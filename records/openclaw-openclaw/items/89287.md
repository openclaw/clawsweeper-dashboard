## Summary

Keep open: the target-aware completion accounting is the right direction, but the patch still lets generated-media automatic payload outcomes satisfy delivery without checking the new target metadata first.

## What This Changes

The branch propagates outbound payload target metadata into delivery status and uses provider/account/thread matching when counting subagent or ACP completion announcements as visibly delivered.

## Best Possible Solution

Land one target-aware completion-delivery path that applies the same provider/account/thread matching to automatic text and generated-media payload outcomes, while preserving legacy targetless compatibility only for the intended non-threaded current-chat cases.
