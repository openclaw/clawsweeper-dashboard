## Summary

Keep open: the target-aware direction is relevant, but current head still lets generated-media automatic payload outcomes count as delivered without checking their new target metadata, and the supplied proof is a synthetic harness rather than a real channel/Gateway run.

## What This Changes

The branch adds outbound delivery target metadata and tightens subagent/ACP completion delivery accounting so completion evidence must match provider, account, channel, and thread targets.

## Best Possible Solution

Filter automatic generated-media payload outcomes through the same target matcher before suppressing fallback, add a wrong-thread media regression, then require live external-route proof or an explicit maintainer proof override before merge.
