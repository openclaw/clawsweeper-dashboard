## What This Changes

The PR changes `session_status` and status rendering tests so the model line can use the persisted runtime session model when no explicit session override exists, with added fallback-display coverage.

## Best Possible Solution

Keep the shipped #55425 behavior for the original symptom and pursue the selected/default-versus-active runtime display in #42690 or an equivalent current-main replacement targeting `src/status`.

## Reproduction Assessment

Yes. The PR defect has a high-confidence static reproduction: use fallback notice fields from a channel/group-selected route that differs from the global default, then the branch passes the wrong selected ref and fallback matching fails.

## Solution Assessment

No. The branch is not the best fix because it edits stale status surfaces and couples fallback preservation to global defaults; the maintainable fix is a current `src/status` split between selected-model semantics and display/runtime identity.
