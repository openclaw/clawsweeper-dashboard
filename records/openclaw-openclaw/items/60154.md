## Summary

Keep open: current main and v2026.5.18 still save Feishu downloaded media into the shared managed inbound bucket, Feishu account config has no scoped inbound storage key, and the open broad media-isolation tracker does not fully replace this Feishu account-boundary request.

## What This Changes

The issue requests an optional per-account Feishu inbound media directory so downloads for one Feishu account are isolated from other accounts and agents.

## Best Possible Solution

Define a managed, backward-compatible account-scoped Feishu inbound media contract and thread that scope through saves, managed references, local access, staging, cleanup, docs, and regression tests without exposing arbitrary host paths by default.

## Reproduction Assessment

Yes, at source level: configure two Feishu accounts and current main still routes Feishu media saves through the shared inbound bucket while the strict account schema has no scoped inbound setting. I did not run a live multi-account Feishu deployment.

## Solution Assessment

Unclear as proposed: an inboundDir option addresses the symptom, but a managed scoped-media contract is safer than arbitrary per-account absolute directories. The best fix needs to cover read paths, staging, cleanup, local-root policy, compatibility, and docs together.
