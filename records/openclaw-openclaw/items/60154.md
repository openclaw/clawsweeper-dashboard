## Summary

Keep open. Current main still saves all Feishu inbound media through the shared managed `inbound` bucket, Feishu account config still has no account-scoped inbound storage setting, and the broader open media-isolation tracker does not replace this Feishu account-boundary request.

## What This Changes

Add per-Feishu-account inbound media storage so each account's downloaded media is isolated from the shared inbound directory.

## Best Possible Solution

Define managed account-scoped inbound buckets, then update Feishu saves plus media references, access roots, sandbox staging, cleanup, docs, and regression tests without exposing arbitrary host paths.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: configure two Feishu accounts and both media download paths still save through `saveMediaBuffer(..., "inbound", ...)`, while the strict Feishu account schema has no account-scoped inbound directory option.

## Solution Assessment

Unclear. The requested `inboundDir` addresses the symptom, but the safer maintainable solution is a managed scoped-media contract rather than arbitrary per-account absolute directories.
