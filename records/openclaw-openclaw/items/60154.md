## Summary

Keep open. Current main still stores Feishu downloads from every configured account in the same managed `inbound` bucket, and the strict Feishu account schema/docs still do not expose an account-level inbound media directory; the linked #19330 tracker is related but broader than this Feishu account-boundary request.

## What This Changes

Request account-specific inbound media storage for multi-account Feishu setups so one account's attachments are not exposed through another agent's media roots.

## Best Possible Solution

Define account-scoped managed inbound buckets under OpenClaw-controlled media storage, then update Feishu saves plus shared media reference, access-root, staging, cleanup, docs, and tests without allowing arbitrary host paths.

## Reproduction Assessment

Yes. A code-level reproduction is high-confidence: configure two Feishu accounts and both download paths still save via `saveMediaBuffer(..., "inbound", ...)`, while `channels.feishu.accounts.<id>.inboundDir` is not accepted by the strict account schema.

## Solution Assessment

Unclear. The requested `inboundDir` fixes the visible symptom, but the safer maintainable direction is a managed account-scoped media-store contract rather than user-configured absolute inbound directories.
