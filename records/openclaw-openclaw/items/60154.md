## Summary

Keep open: current main still saves Feishu inbound media into the shared managed `inbound` bucket, Feishu account config has no account-scoped inbound storage key, and the broader media-isolation tracker does not replace this Feishu account-boundary request.

## What This Changes

Request adding account-level Feishu inbound media storage so downloads from separate configured Feishu accounts do not share the global inbound directory.

## Best Possible Solution

Define a managed, backward-compatible account-scoped inbound media contract for Feishu, then thread that scope through saves, references, local access, staging, cleanup, docs, and regression tests without exposing arbitrary host paths.

## Reproduction Assessment

Yes, at source level: configure two Feishu accounts and current main still routes Feishu media saves through the shared `inbound` bucket while the strict account schema has no scoped inbound setting. I did not run a live multi-account Feishu deployment in this read-only review.

## Solution Assessment

Unclear as proposed. The requested `inboundDir` addresses the symptom, but a managed scoped-media contract is safer and more maintainable than arbitrary per-account absolute directories.
