## Summary

Keep open: current main and v2026.6.10 still save Feishu downloads into the shared inbound media bucket, the strict Feishu account schema has no account-scoped inbound media setting, and related issues are overlapping trackers rather than landed fixes.

## What This Changes

Requests an optional Feishu account-scoped inbound media directory so media downloaded by one configured Feishu account is not stored in the shared global inbound directory.

## Best Possible Solution

Define a managed scoped inbound-media/provenance contract for Feishu account saves and the shared media resolver/staging surfaces instead of only adding an unconstrained absolute inboundDir.

## Reproduction Assessment

Yes at source level: current main still routes Feishu media downloads through the shared inbound bucket and the strict account schema lacks the requested setting. I did not run live multi-account Feishu traffic in this read-only review.

## Solution Assessment

Unclear as proposed: per-account inboundDir is plausible, but the safest fix likely needs a managed scoped media contract across save, resolve, staging, cleanup, and docs.
