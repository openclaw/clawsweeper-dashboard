## Summary

Keep open: current main and v2026.6.10 still route Feishu inbound media through the shared managed inbound bucket, the strict Feishu account schema has no account-scoped media directory setting, and the related open issues overlap but do not settle this Feishu account-boundary product/security decision.

## What This Changes

Request account-scoped Feishu inbound media storage/configuration so separately configured Feishu app accounts do not share one inbound media directory.

## Best Possible Solution

Define a managed scoped inbound-media contract that covers Feishu account saves and the generic media resolver/staging surfaces without adding an unconstrained host-path setting.

## Reproduction Assessment

Yes, at source level: current main still routes Feishu media downloads through the shared inbound bucket and the strict account schema lacks the requested setting. I did not run live multi-account Feishu traffic in this read-only review.

## Solution Assessment

Unclear as proposed: per-account inboundDir is plausible, but the safer fix likely needs a managed scoped media contract across save, resolve, staging, cleanup, docs, and upgrade behavior.
