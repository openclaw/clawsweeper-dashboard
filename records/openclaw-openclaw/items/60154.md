## Summary

Keep open: current main still routes Feishu downloads into the shared inbound media bucket, the strict Feishu account config has no account-scoped inbound storage option, and the broader media-isolation tracker does not fully cover this Feishu account-boundary request.

## What This Changes

Request an optional per-Feishu-account inbound media directory so media downloaded by separate Feishu enterprise apps does not share one global inbound store.

## Best Possible Solution

Define managed account-scoped inbound buckets, then update Feishu saves, managed media references, local roots, sandbox staging, cleanup, docs, and regression tests without exposing arbitrary host paths.

## Reproduction Assessment

Yes, at source level: configure two Feishu accounts and both download paths still call saveMediaBuffer with the shared "inbound" bucket while the strict account schema has no inboundDir setting. I did not run a live Feishu deployment in this read-only review.

## Solution Assessment

Unclear. The requested inboundDir addresses the symptom, but a managed scoped-media contract is safer and more maintainable than arbitrary per-account absolute directories.
