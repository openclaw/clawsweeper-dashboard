## Summary

Keep open. Current main still saves Feishu media for all configured accounts into the shared managed inbound bucket, and neither Feishu account config nor docs expose an account-scoped inbound storage contract; the broader media-isolation tracker is related but does not fully replace this Feishu account-boundary slice.

## What This Changes

The issue requests optional per-Feishu-account inbound media storage so multiple Feishu accounts do not share one downloaded-media directory.

## Best Possible Solution

Define account-scoped managed inbound buckets under OpenClaw-controlled media storage, then update Feishu saves, media references, access roots, sandbox staging, cleanup, docs, and regression tests without allowing arbitrary host paths.

## Reproduction Assessment

Yes for source-level reproduction. Configure two Feishu accounts and both inbound media paths still save through saveMediaBuffer(..., "inbound", ...), while the strict Feishu account schema rejects channels.feishu.accounts.<id>.inboundDir.

## Solution Assessment

No, not as proposed. The requested inboundDir option addresses the symptom, but the safer maintainable fix is a managed account-scoped media-store contract that also covers references, local-root access, staging, cleanup, docs, and tests.
