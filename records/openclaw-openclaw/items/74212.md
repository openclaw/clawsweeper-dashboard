## Summary

Keep open. Current main still masks the OpenAI Codex device-pairing code in remote sessions, and open PR #74258 is the active closing implementation candidate for the same bug.

## What This Changes

OpenAI Codex device-pairing auth should surface the short-lived user code in SSH/remote prompter output while keeping persistent runtime logs redacted.

## Best Possible Solution

Review and land #74258, or replace it with the same provider-local prompter fix plus focused regression coverage and a user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence reproduction exists by running the OpenAI Codex device-code auth method with `isRemote: true` and a mocked `userCode`; current main asserts the placeholder is shown instead of the code.

## Solution Assessment

Yes. The provider already receives `userCode`, so changing only the prompter note while preserving runtime-log redaction is the narrowest maintainable solution.
