## Summary

Keep open: the follow-up fixed the earlier contract/changelog problems and supplied real behavior proof, but the current patch introduces a first-login regression by swallowing Gemini CLI identity/project discovery failures that current main surfaces to the user.

## What This Changes

The PR adds a Google Gemini CLI provider-owned OAuth refresh hook, factors Gemini token exchange/refresh helpers, raises auth-profile resolution failures to warn logs, adds coverage, and updates the changelog.

## Best Possible Solution

Keep the provider-owned refresh design, but separate login and refresh credential-building semantics so initial login still fails with the existing identity/project setup error while refresh may fall back to stored identity.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main lacks a Gemini CLI provider refresh hook, and the PR’s copied live output shows the new hook advancing an expired credential’s expiry. The new review finding is also source-reproducible because `exchangeCodeForTokens()` now uses the same broad identity-discovery catch as refresh.

## Solution Assessment

No, not as currently written. The provider-owned refresh hook is the narrow maintainable fix, but the shared helper should preserve first-login identity/project validation instead of silently creating incomplete credentials.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve first-login identity failures:** `extensions/google/oauth.token.ts:57-60`
  - body: The new shared helper catches every identity-discovery error even when `exchangeCodeForTokens()` calls it with no stored identity. Current main rejects first login when Code Assist/project discovery fails, including the actionable `GOOGLE_CLOUD_PROJECT` guidance; this diff can instead persist a credential with no `email`/`projectId` and defer the failure to runtime. Limit the fallback to refresh calls that already have stored identity, or rethrow when this is an initial login.
