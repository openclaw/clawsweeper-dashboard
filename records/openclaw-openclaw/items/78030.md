## Summary

Keep open; the PR is aimed at the right provider-owned boundary for the Gemini CLI OAuth refresh bug, but it should not merge yet because the new refresh hook does not satisfy the `OAuthCredential` contract and the external PR lacks after-fix real behavior proof.

## What This Changes

The PR adds a Google Gemini CLI OAuth refresh hook, factors token exchange and refresh helpers, adds mocked/unit coverage, and promotes auth-profile resolution failures from debug to warn logging.

## Best Possible Solution

Keep the provider-owned refresh design, but return a complete `OAuthCredential`, add the required changelog entry, and include real expired-token refresh proof before normal maintainer review proceeds.

## Reproduction Assessment

No live high-confidence reproduction was established in this read-only review. Source inspection does show the current-main path: Gemini CLI OAuth has no provider refresh hook, and the pinned shared OAuth registry does not know `google-gemini-cli`.

## Solution Assessment

No, not as currently written. The provider-owned refresh hook is the narrow maintainable direction, but the implementation must satisfy the `OAuthCredential` contract and be proven against a real expired Gemini CLI OAuth profile.
