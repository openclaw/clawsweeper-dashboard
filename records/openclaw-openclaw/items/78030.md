## Summary

Keep open for maintainer review: this PR addresses a source-reproducible Gemini CLI OAuth refresh bug on current main using the existing provider-owned refresh seam, and I found no blocking code or security finding.

## What This Changes

Adds a `google-gemini-cli` provider `refreshOAuth` hook, refactors Google OAuth token grant/refresh helpers, raises auth-profile resolution failures to warnings, and adds focused tests plus a changelog entry.

## Best Possible Solution

Land the provider-owned refresh hook after maintainer/secops review, keeping cooldown policy and broader OAuth diagnostics as separate follow-ups unless owners choose otherwise.

## Reproduction Assessment

Yes, source-reproducible: an expired `google-gemini-cli` OAuth credential reaches `refreshOAuthCredential`, current main has no Google provider `refreshOAuth` hook, and `@mariozechner/pi-ai@0.73.1` has no built-in Gemini CLI OAuth provider.

## Solution Assessment

Yes. The existing provider-owned `refreshOAuth` seam is the narrowest maintainable fix because Google-specific token refresh stays in `extensions/google` while core already routes plugin refresh before the generic `pi-ai` fallback.
