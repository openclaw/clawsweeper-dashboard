## Summary

Keep open: current main still lacks the proposed permission_denied model and payload guards, but the branch is not mergeable as-is because it changes sandbox missing-file semantics, misses current mount-escape messages, and lacks external contributor real behavior proof.

## What This Changes

The PR adds an agent filesystem permission-denied helper, wraps apply_patch and sandbox filesystem boundary failures with reason metadata, and adds write/edit text payload size and NUL guards plus tests.

## Best Possible Solution

Revise this PR rather than close it: keep the structured permission-denied direction, preserve not-found behavior, classify current mount escapes, settle the payload-limit policy, and post real behavior proof.

## Reproduction Assessment

Yes. Source inspection shows current main emits "Sandbox path escapes allowed mounts" while the PR classifier omits that phrase, and current tests require missing sandbox reads to throw ENOENT/no-such-file while the PR wraps all failed boundary opens as permission_denied.

## Solution Assessment

No, not as currently proposed. Centralizing fs permission_denied metadata is reasonable, but the patch should preserve path-not-found semantics, classify current mount-escape messages, and get explicit maintainer agreement on payload limits before merge.
