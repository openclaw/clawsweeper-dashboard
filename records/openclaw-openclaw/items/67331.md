## Best Possible Solution

Request changes, then land a corrected version only after mount escapes classify as workspace-boundary denials, missing sandbox reads preserve not-found semantics, payload limits are accepted, and focused regression plus real-behavior proof are posted.

## Reproduction Assessment

Yes for the PR defects by source inspection: the PR classifier misses the current `Sandbox path escapes allowed mounts` phrase, and `openReadableFile` wraps ordinary path misses as permission_denied. I did not run tests because this review is read-only.

## Solution Assessment

No. Centralizing permission-denied metadata is reasonable, but this patch needs to preserve ENOENT semantics, recognize the current mount-boundary phrase, preserve useful causes, and get maintainer agreement on payload ceilings.
