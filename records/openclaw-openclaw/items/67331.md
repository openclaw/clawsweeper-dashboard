## Summary

Keep open: current main still lacks the structured permission_denied implementation, and the PR has concrete correctness blockers plus missing external-contributor real behavior proof before it can merge.

## What This Changes

The branch adds an agent fs permission-denied helper, wraps apply_patch and sandbox filesystem errors with reason metadata, adds write/edit payload guards, and updates related tests.

## Best Possible Solution

Request changes on this PR, then land a corrected version only after mount escapes classify as workspace-boundary denials, missing sandbox reads preserve not-found semantics, payload limits are accepted, and focused regression plus real-behavior proof are posted.

## Reproduction Assessment

Yes. Source inspection of the PR head shows current `Sandbox path escapes allowed mounts` causes classify as `unknown`, and current-main tests define missing sandbox reads as ENOENT/no-such-file rather than permission_denied.

## Solution Assessment

No, not as currently proposed. Centralizing permission-denied metadata is a reasonable direction, but the patch needs to preserve missing-file semantics, recognize the current mount-boundary phrase, and get maintainer agreement on the payload ceiling.
