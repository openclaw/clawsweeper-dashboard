## Summary

Keep open: current main has not implemented this standardization, and the PR still has concrete correctness blockers plus missing contributor real-behavior proof.

## What This Changes

The PR adds an agent filesystem permission-denied helper, wraps apply_patch and sandbox filesystem errors with reason metadata, adds write/edit payload guards, and updates related tests.

## Best Possible Solution

Request changes, then land a corrected version only after mount escapes classify as workspace-boundary denials, missing sandbox reads preserve not-found semantics, payload limits are accepted, and focused regression plus real-behavior proof are posted.

## Reproduction Assessment

Yes. Source inspection of the PR head shows the current `Sandbox path escapes allowed mounts` cause classifies as `unknown`, and current-main tests define missing sandbox reads as ENOENT/no-such-file rather than permission_denied.

## Solution Assessment

No. Centralizing permission-denied metadata is a reasonable direction, but this patch needs to preserve missing-file semantics, recognize the current mount-boundary phrase, and get maintainer agreement on the payload ceiling.
