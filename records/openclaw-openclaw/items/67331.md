## Summary

Keep open. Current main still lacks the proposed standardized permission-denied helper and mutation payload guards, while this PR has concrete correctness blockers in sandbox not-found semantics and mount-boundary reason classification.

## What This Changes

This PR adds an agent filesystem permission-denied helper, wraps apply_patch and sandbox filesystem boundary errors with standardized metadata, adds 2 MB/NUL text mutation guards, and updates related tests.

## Best Possible Solution

Request changes on this PR, then land it only after mount escapes classify as workspace-boundary denials, missing sandbox reads preserve not-found semantics, payload limits are accepted, and focused host plus sandbox tests cover the behavior.

## Reproduction Assessment

Yes. The blockers are reproducible by focused checks: classify an 'escapes allowed mounts' error through the PR helper, and read a missing sandbox file while asserting ENOENT/no-such-file semantics remain intact.

## Solution Assessment

No, not as currently proposed. Centralizing permission-denied metadata is a maintainable direction, but the branch should preserve existing path-miss behavior and recognize the current mount-boundary message shape before merge.
