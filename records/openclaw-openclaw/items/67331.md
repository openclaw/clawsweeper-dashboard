  - confidence: 0.92
- **[P2] Classify allowed-mount escapes as workspace_boundary:** `src/agents/fs-permission-denied.ts:22-26`
  - body: The classifier does not recognize the bridge message phrase `Sandbox path escapes allowed mounts`, even though the patch wraps local and remote sandbox boundary errors using that exact cause text. Those denials become `reason=unknown`, defeating the standardized reason metadata this PR is adding.
  - confidence: 0.95
- **[P3] Preserve the boundary-open failure cause:** `src/agents/apply-patch.ts:383-386`
  - body: assertBoundaryRead creates a synthetic `Failed boundary read ...` cause for validation failures instead of passing through the boundary-open error when available. That drops symlink, hardlink, or boundary wording the new classifier needs and can still report `reason=unknown` for actionable denials.
  - confidence: 0.78
