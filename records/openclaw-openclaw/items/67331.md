## Summary

Keep open: the central permission_denied model is not on current main, but this PR still has concrete correctness blockers and missing external real-behavior proof before it can merge.

## What This Changes

This PR adds an agent filesystem permission-denied helper, wraps apply_patch and sandbox filesystem boundary errors with standardized reason metadata, adds write/edit text payload guards, and updates related tests.

## Best Possible Solution

Request changes on this PR, then land a corrected version only after sandbox mount escapes classify as workspace-boundary denials, missing sandbox reads preserve not-found semantics, the payload guard limit is accepted, and real-behavior proof is posted.

## Reproduction Assessment

Yes. Source inspection of the PR head shows the classifier sends current `Sandbox path escapes allowed mounts` causes to `unknown`, and current-main tests define missing sandbox reads as ENOENT/no-such-file rather than permission_denied.

## Solution Assessment

No, not as currently proposed. Centralizing permission-denied metadata is a reasonable direction, but the branch needs to preserve existing missing-file semantics, recognize the current mount-boundary phrase, and get maintainer agreement on the payload ceiling.
