Full review comments:

- **[P1] Preserve missing-read errors outside permission_denied:** `src/agents/sandbox/fs-bridge-path-safety.ts:78-84`
  - body: `openRootFile` reports missing paths as `reason: "path"`, and sandbox media/tool callers use `bridge.readFile` directly. Wrapping every failed open here turns ordinary ENOENT/ENOTDIR reads into `E_FS_PERMISSION_DENIED`, which changes existing diagnostics and errno-style handling; only validation/boundary failures should be mapped to permission_denied.
  - confidence: 0.9
- **[P2] Classify allowed-mount escapes as workspace boundaries:** `src/agents/fs-permission-denied.ts:22-29`
  - body: The sandbox bridges now pass messages like `Sandbox path escapes allowed mounts`, but the classifier only checks `escapes sandbox root` and outside-root phrases. Those mount-boundary denials will be emitted as `reason=unknown`, defeating the standardized reason surface this PR introduces.
  - confidence: 0.93
- **[P1] Do not silently cap existing write/edit payloads at 2 MiB:** `src/agents/pi-tools.read.ts:47-48`
  - body: These constants add a new hard rejection path for write content and edit old/new text before the existing tools run. Current write/edit tools do not have this limit, so existing agents that rewrite generated files can start failing without a documented or approved compatibility plan.
  - confidence: 0.86
