## Summary

Keep open: current main still lacks the proposed permission_denied model and mutation guards, but this external PR has concrete correctness blockers in sandbox read semantics and reason classification plus missing real behavior proof before merge.

## What This Changes

This PR adds an agent filesystem permission-denied helper, wraps apply_patch and sandbox filesystem boundary errors with reason metadata, adds write/edit payload guards, and updates related tests.

## Best Possible Solution

Request changes on this PR, then land it only after sandbox mount escapes classify as workspace-boundary denials, missing sandbox reads preserve not-found semantics, payload limits are accepted, focused regressions cover the host and sandbox paths, and real behavior proof is posted.

Yes. Source inspection gives a high-confidence path: classify a current "Sandbox path escapes allowed mounts" error through the PR helper, and read a missing sandbox file while asserting ENOENT/no-such-file semantics remain intact.

## Solution Assessment

No, not as currently proposed. Centralizing permission-denied metadata is a reasonable direction, but the patch should preserve path-miss errors and recognize the current sandbox mount-boundary message before merge.
