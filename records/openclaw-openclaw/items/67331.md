## Summary

Keep open: current main still lacks this permission-denied/error-hardening work, but this PR should not merge as-is because the branch still has concrete filesystem behavior regressions and no contributor real-behavior proof.

## What This Changes

The PR adds an agent filesystem permission_denied helper, wraps apply_patch and sandbox bridge boundary errors with reason metadata, adds write/edit payload size and NUL-byte guards, and updates related tests.

## Best Possible Solution

Request changes on this PR, then land a corrected version only after sandbox mount escapes classify as workspace-boundary denials, missing sandbox reads preserve not-found semantics, payload limits are accepted, and focused regression plus real-behavior proof are posted.

## Reproduction Assessment

Yes. Source inspection shows the PR classifier sends the current `Sandbox path escapes allowed mounts` phrase to `unknown`, and the PR's `openReadableFile` wrapper catches path-miss reads that current main tests expect as ENOENT/no-such-file.

## Solution Assessment

No, not as currently proposed. Centralizing permission-denied metadata is a reasonable direction, but the branch needs to preserve current missing-file semantics, recognize the current mount-boundary phrase, and get maintainer agreement on the payload ceiling.
