Full review comments:

- **[P1] Preserve missing-file read semantics:** `src/agents/sandbox/fs-bridge-path-safety.ts:75-80`
  - body: `openRootFile` distinguishes missing paths as `reason: "path"` for ENOENT/ENOTDIR/ELOOP, but this branch wraps every failed sandbox read as `E_FS_PERMISSION_DENIED`. That turns ordinary missing sandbox reads through `SandboxFsBridge.readFile` into permission-denied failures instead of not-found diagnostics, so only validation/boundary failures should be remapped.
  - confidence: 0.9
- **[P2] Map mount escapes to workspace_boundary:** `src/agents/fs-permission-denied.ts:22-25`
  - body: The sandbox bridge now passes errors whose message is `Sandbox path escapes allowed mounts`, but the classifier only recognizes `escapes sandbox root`, `outside workspace root`, and `outside sandbox root`. Those denials will be emitted as `reason=unknown`, defeating the standardized reason code this PR adds.
  - confidence: 0.93
- **[P1] Do not add an unapproved write/edit size cap:** `src/agents/pi-tools.read.ts:50-51`
  - body: The current write/edit tools accept string content without a documented max size, while this patch adds hard 2 MiB caps before mutation. That can break existing workflows that write or edit larger text files, so the cap needs removal or an explicit maintainer-approved compatibility contract with docs and upgrade proof.
  - confidence: 0.84
