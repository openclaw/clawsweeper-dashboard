## What This Changes

The PR changes the agent image tool to accept bare and `MEDIA:`-prefixed `media://inbound/<id>` references, load them from managed inbound media, and adds image-tool tests plus a changelog entry.

## Best Possible Solution

Rebase or replace this PR with a consolidated fix that preserves managed `media://inbound` references until the shared resolver handles them, covers the image-tool `workspaceDir` path, and includes redacted real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main can pass `media://inbound/<id>` through the normal image tool with `workspaceDir`, where it can be resolved into a malformed workspace `media:/inbound` path before managed-media hydration.

## Solution Assessment

No, not as currently shaped. The underlying bug should be fixed by preserving `media://` until the shared media-reference resolver handles it, not by adding another custom parser while the branch is conflicting and proof-gated.
