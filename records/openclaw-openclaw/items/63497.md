## Summary

The core failure is still valid on current main, but this PR should stay open rather than close because it targets the right non-sandbox image-tool bug while still needing a rebase/replacement through the shared media-reference contract and contributor real behavior proof.

## What This Changes

The PR changes the agent image tool to accept bare and `MEDIA:`-prefixed `media://inbound/<id>` references, adds regression tests, and adds a changelog note.

## Best Possible Solution

Land a rebased or consolidated fix that keeps `media://inbound` references intact until the shared `normalizeMediaReferenceSource`/`resolveInboundMediaReference`/`loadWebMedia` path handles them, with focused regression coverage and redacted real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main passes `workspaceDir` into the image tool and can resolve `media://inbound/<id>` as a workspace-relative path before managed-media hydration runs.

## Solution Assessment

No. The PR targets the right bug, but the maintainable fix should reuse the shared media-reference/loadWebMedia contract instead of introducing a custom URI regex and direct `resolveMediaBufferPath` branch in the image tool.
