## Summary

Keep open. The underlying bug is still source-reproducible on current main, but this PR should not merge as-is because it adds a separate raw media URI parser on the inbound-media boundary and still lacks external real-behavior proof.

## What This Changes

The PR adds image-tool handling and tests for bare and `MEDIA:`-prefixed `media://inbound/<id>` image references.

## Best Possible Solution

Land a rebased or replacement fix that keeps `media://` refs intact or resolves them through the shared media-reference/web-media contract before workspace or sandbox path normalization, with `workspaceDir` regression coverage.

## Reproduction Assessment

Yes, at source level. Normal agent setup passes `workspaceDir` into `createImageTool`, and current main can route `media://inbound/<id>` through workspace path resolution before managed-media hydration.

## Solution Assessment

No. The PR targets the right failure, but the safer fix is to reuse the shared inbound media resolver/loader contract instead of adding a custom raw regex parser on the media access boundary.
