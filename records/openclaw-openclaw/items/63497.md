## Best Possible Solution

Rebase or replace the branch with a shared-resolver fix that keeps managed `media://inbound` refs intact until resolution, covers the `workspaceDir` image-tool path, and includes redacted real behavior proof.

## Reproduction Assessment

Yes, at source level: normal tool setup passes `workspaceDir` into the image tool, and current main can still route `media://inbound/<id>` into the workspace-resolution branch before managed-media hydration. I did not run a live Telegram/ZAI scenario in this read-only review.

## Solution Assessment

No, not as currently shaped. The bug should be fixed through the shared media-reference resolver or by preserving classified media-store URIs before workspace path handling, rather than adding a custom parser in the image tool.
