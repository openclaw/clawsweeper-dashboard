## Summary

Keep open. The linked image-tool bug is still valid on current main, and the PR is useful, but the branch needs rework before merge because it duplicates stale media URI parsing instead of using the shared resolver, misses the workspaceDir production composition in its regression coverage, and has only mock/test proof.

## What This Changes

The PR adds image-tool handling, tests, and a changelog entry for resolving `media://inbound/<id>` and `MEDIA:media://...` references through the inbound media store.

## Best Possible Solution

Land a rebased or replacement patch that keeps managed media refs out of image-tool workspace resolution, uses the shared media-reference/web-media contract, adds workspaceDir regression coverage, and leaves the sandbox bridge fix to https://github.com/openclaw/openclaw/pull/74186.

## Reproduction Assessment

Yes, at source level. Current main passes `workspaceDir` into `createImageTool`, classifies `media://` as supported, and then resolves it as workspace-relative because `isMediaStoreUrl` is not excluded; the path primitive produces the reported `media:/inbound` shape.

## Solution Assessment

No. The PR direction is useful, but the current branch should be rebased or replaced to use the shared inbound media resolver and to cover the workspaceDir agent-session path instead of adding a separate custom parser.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Reuse the shared inbound media resolver:** `src/agents/tools/image-tool.ts:50`
  - body: The added `INBOUND_MEDIA_URI_RE` path resolves the captured ID directly with `resolveMediaBufferPath`, so it does not URL-decode before validation. That lets a URI such as `media://inbound/nested%2Ftiny.png` resolve a literal inbound file even though the shared media-reference contract rejects decoded separators, making the image tool accept a different URI contract than native prompt/media loading. Route this through the shared resolver or pass the normalized ref to `loadWebMedia` instead.
  - confidence: 0.84
- **[P2] Exercise workspaceDir in the regression:** `src/agents/tools/image-tool.test.ts:1381-1384`
  - body: The new regression tests create the image tool without `workspaceDir`, but normal agent sessions provide one through `createOpenClawTools`. Add coverage with `workspaceDir` set so the test fails on the production path that turns `media://inbound/...` into `workspace/media:/inbound/...` on current main.
  - confidence: 0.88
