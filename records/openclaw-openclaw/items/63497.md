## Summary

Keep open. The PR targets a real current-main image-tool bug, but the proposed fix adds a custom raw `media://inbound` parser instead of using the shared media-reference contract, and the PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds image-tool handling, tests, and a changelog entry for bare and `MEDIA:`-prefixed `media://inbound/<id>` image references.

## Best Possible Solution

Land a rebased or replacement fix that preserves or resolves `media://` refs through the shared media-reference/web-media contract before workspace or sandbox path normalization, with `workspaceDir` regression coverage.

## Reproduction Assessment

Yes, at source level. Normal agent setup passes `workspaceDir` into `createImageTool`, and current main can route `media://inbound/<id>` through workspace path resolution before managed-media hydration.

## Solution Assessment

No. The PR targets the right failure, but the safer fix is to reuse the shared inbound media resolver/loader contract instead of adding a custom raw regex parser on the media access boundary.
