## What This Changes

The PR changes gateway status probe budget handling so active configured or explicit remote probes use the caller timeout while inactive remote/local and SSH tunnel probes keep short caps.

## Best Possible Solution

Land the narrow resolver reorder with its regression tests after maintainer review, preserving short caps for inactive/background and SSH tunnel probes.
