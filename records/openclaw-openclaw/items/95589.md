## Summary

Keep open for maintainer review. The PR is a focused managed-npm slice of the stale hashed-import problem and I found no concrete line-level correctness defect, but merging it intentionally changes update/cleanup semantics by retaining old npm generations until maintainers accept or bound that lifecycle policy.

## What This Changes

This PR changes managed npm plugin updates to install into generation-specific project roots, retain replaced generations with OpenClaw marker metadata, and skip retained generations during registry recovery and doctor cleanup.

## Best Possible Solution

Land this managed npm generation-retention slice only after maintainers explicitly accept or bound retained-generation lifetime, while keeping the broader stale-dist lifecycle work on the canonical issue.

## Reproduction Assessment

Yes for source-level review: current main uses a single managed npm project root and can clean up the previous install, while the linked Windows evidence shows the stale hashed import failure shape. I did not run a fresh live package-upgrade reproduction in this read-only pass.

## Solution Assessment

Mostly yes for the managed npm slice: generation-root retention is in the plugin install/registry boundary and avoids deleting live import targets. The unresolved part is whether indefinite retained package trees are the maintainer-approved long-term policy.
