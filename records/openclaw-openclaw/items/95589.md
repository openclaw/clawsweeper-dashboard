## Summary

Keep open for maintainer review: the patch appears to correctly address the managed npm plugin-update slice with sufficient terminal proof, but it intentionally changes upgrade and cleanup semantics by retaining old package generations until maintainers accept or bound that policy.

## What This Changes

The branch installs updated managed npm plugins into generation-specific project roots, marks replaced npm generations as retained, and skips retained generations during registry recovery and doctor cleanup.

## Best Possible Solution

Land the managed npm generation-retention fix after maintainers explicitly accept or bound retained-generation lifetime, while keeping the broader stale hashed-import lifecycle work on the canonical issue.

## Reproduction Assessment

## Solution Assessment

Mostly yes for the managed npm slice: generation-root retention sits in the plugin install/registry boundary and avoids deleting live import targets. The unresolved part is whether indefinite retained package trees are the maintainer-approved long-term policy.
