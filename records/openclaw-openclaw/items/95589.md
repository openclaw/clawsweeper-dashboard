## Summary

Keep this PR open for maintainer review: the patch appears to address the managed npm plugin replacement slice without a discrete review finding, but it intentionally changes upgrade/cleanup semantics by retaining old npm generations until a future cleanup policy exists.

## What This Changes

The branch installs managed npm plugin updates into generation-specific project roots, marks replaced generations as retained, and skips retained generations during registry recovery and doctor cleanup.

## Best Possible Solution

Land the managed npm generation-retention fix only after maintainers explicitly accept or bound retained-generation lifetime, while keeping broader stale dist lifecycle work on the canonical issue.

## Reproduction Assessment

Yes for source-level review: current main can remove the previous managed npm install directory after recording a new install, and the linked Windows logs show the stale import failure shape. I did not run a fresh live reproduction in this read-only pass.

## Solution Assessment

Mostly yes for the managed npm slice: generation-root retention sits in the plugin install/registry boundary and avoids deleting live import targets. The unresolved part is whether indefinite retained package trees are the maintainer-approved long-term policy.
