## Summary

Keep this open: current main and the latest release still send the normal Telegram typing cue only after media resolution and audio preflight can finish, so the requested pre-STT voice-message feedback remains missing.

## What This Changes

The issue requests an immediate Telegram typing cue for voice messages before media download and STT transcription finish.

## Best Possible Solution

Add a Telegram-owned best-effort early chat action after cheap authorization/thread/mention checks but before resolveMedia and transcribeFirstAudio, with regression coverage for ordering and ignored-message cases.

## Reproduction Assessment

Yes. Source inspection shows voice/audio messages can await media resolution and transcribeFirstAudio before the existing sendTyping call is reached.

## Solution Assessment

Yes for a bounded feature. The maintainable path is a Telegram ingress/context-layer cue that preserves current authorization and room-event suppression, not a broad core typing-mode rewrite.
