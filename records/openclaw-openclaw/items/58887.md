## What This Changes

Request to emit a Telegram typing indicator as soon as a voice message is received, before audio download and STT transcription delay user feedback.

## Best Possible Solution

Add a Telegram-plugin early chat-action cue after access/routing accepts an actionable voice or audio message, but before media download or STT preflight can delay feedback.

Yes, from source: an audio-only Telegram DM or mention-gated group voice message downloads media and can await transcribeFirstAudio before the existing early typing cue is sent. I did not run a live Telegram/STT reproduction in this read-only review.

## Solution Assessment

Yes: a plugin-local early chat-action cue on accepted voice/audio inbound messages is the narrowest maintainable fix, as long as it preserves dropped-turn, unauthorized, room-event, and typing-mode suppressions.
