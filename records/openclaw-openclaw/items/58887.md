## Summary

The current Telegram path still has the reported ordering gap: voice/audio media is resolved and preflight transcription can finish before the existing typing cue is sent, so this should stay open as the canonical tracker.

## What This Changes

Request an immediate Telegram typing indicator for voice/audio messages before media download and STT transcription.

## Best Possible Solution

Add a Telegram-owned early typing path for eligible voice/audio messages after cheap authorization/thread resolution but before media download and STT, with regression tests for ordering and ignored-message behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: resolveMedia and then resolveTelegramInboundBody/transcribeFirstAudio can complete before the current context.sendTyping call is reached.

## Solution Assessment

Yes. The best fix is a narrow Telegram handler/context change that reuses existing sendChatAction and thread helpers while preserving current access gates, rather than moving broad policy into core.
