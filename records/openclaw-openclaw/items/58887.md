## What This Changes

The issue requests a Telegram voice-message typing cue as soon as a voice message is received, before STT transcription delays user feedback.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/96278 or a replacement lands the Telegram plugin pre-STT cue with ordering coverage and redacted live Telegram proof; keep broader voice-pipeline work on https://github.com/openclaw/openclaw/issues/48179.

## Solution Assessment

Yes for the narrow Telegram direct-message slice: the maintainable fix belongs in the Telegram plugin after cheap access/config gating and before body/STT. The open linked PR is the current candidate, but it should not close this issue until it lands with live proof.
