## Summary

Keep open. Current main and the latest release still send Telegram typing only after media download and audio preflight transcription for voice-only messages, so the requested pre-STT typing cue remains a real Telegram UX gap.

## What This Changes

The issue requests an immediate Telegram typing indicator for received voice messages before audio download and speech-to-text transcription complete.

## Best Possible Solution

Add a Telegram-plugin-owned best-effort preflight typing cue after cheap access/thread eligibility checks but before media download and transcribeFirstAudio, while preserving existing dropped-turn and sender-authorization behavior.

## Reproduction Assessment

Yes at source level: current main awaits Telegram media download and then awaits transcribeFirstAudio before the first typing cue can be sent for voice-only messages. I did not run a live Telegram timing benchmark in this read-only review.

## Solution Assessment

Yes, the best narrow solution is a Telegram inbound-path typing cue before media/STT work, not a broad voice-pipeline rewrite. The implementation should stay in the Telegram plugin and preserve existing authorization and skipped-message behavior.
