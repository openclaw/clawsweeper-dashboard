## Summary

Keep this PR open: current main still truncates reply directive IDs with UTF-16 `.slice`, while the branch moves the shared sanitizer to code-point truncation and adds focused regression coverage with sufficient terminal proof.

## What This Changes

The PR changes `sanitizeReplyDirectiveId` to truncate by Unicode code point instead of UTF-16 code unit and adds surrogate-pair boundary tests.

## Best Possible Solution

Land the focused sanitizer change after normal maintainer review and exact-head CI; keep the adjacent explicit-ID sanitizer PR separate unless maintainers intentionally combine the two contracts.

Yes, at source level. Current main uses UTF-16 length and slice at the sanitizer boundary, and the read-only Node probe shows a 255-ASCII-prefix plus emoji leaves a dangling high surrogate before the patch.

## Solution Assessment

Yes. Fixing `sanitizeReplyDirectiveId` is the narrow shared boundary for gateway reply directive emission; caller-local truncation would duplicate the reply-ID contract.
