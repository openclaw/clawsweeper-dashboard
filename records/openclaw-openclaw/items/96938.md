## Summary

Keep open: current main still truncates sanitized reply directive IDs with UTF-16 `.slice`, and this focused PR addresses that central bug, but it needs contributor real behavior proof before merge and has one low-priority test-helper defect.

## What This Changes

The PR changes `sanitizeReplyDirectiveId` to truncate by Unicode code points and adds a regression test for emoji at the reply-ID cap.

## Best Possible Solution

Land a focused sanitizer fix after correcting the test helper and adding redacted after-fix proof; keep the adjacent explicit-ID sanitizer work separate unless maintainers choose to merge the two sanitizer contracts together.

## Reproduction Assessment

Yes, at source level. Current main truncates with UTF-16 `.slice(0, 256)`, and a 255-ASCII-prefix plus emoji input leaves a dangling high surrogate before the patch; repo tests were not run because this review is read-only.

## Solution Assessment

Yes, with a small test cleanup. Fixing `sanitizeReplyDirectiveId` is the narrow shared boundary for webchat and transcript reply directive emission; patching individual callers would duplicate the contract.
