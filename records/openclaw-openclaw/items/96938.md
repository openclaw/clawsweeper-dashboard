## Summary

Keep open: current main and v2026.6.10 still truncate reply directive IDs with raw UTF-16 slicing, while this PR fixes the surrogate-pair boundary case with a focused sanitizer change, regression tests, and sufficient terminal proof. I found no blocking patch findings.

## What This Changes

The branch changes `sanitizeReplyDirectiveId` to truncate sanitized reply IDs by Unicode code point and adds surrogate-pair regression tests for the sanitizer.

## Best Possible Solution

Land the focused sanitizer change after maintainer review and required exact-head checks, keeping reply directive ID policy centralized in `sanitizeReplyDirectiveId`.

## Reproduction Assessment

Yes. Current main uses `sanitized.length` and raw `slice`, and a Node probe with 255 ASCII characters followed by an emoji leaves a trailing high surrogate; the PR discussion shows matching terminal proof.

## Solution Assessment

Yes. The shared sanitizer is the right boundary; caller-local fixes would duplicate policy, and `truncateUtf16Safe` would preserve the old code-unit cap by dropping the boundary emoji instead of preserving 256 Unicode code points.
