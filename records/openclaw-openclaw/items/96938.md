## Summary

Keep open: current main and the latest release still truncate reply directive IDs with UTF-16 `slice`, while this focused PR fixes the source-reproducible Unicode boundary bug with no actionable patch findings.

## What This Changes

This PR replaces UTF-16 `slice` truncation in `sanitizeReplyDirectiveId` with code-point truncation and adds surrogate-pair regression tests.

## Best Possible Solution

Land this focused sanitizer fix after maintainer review and exact-head checks, keeping the shared helper as the single reply-ID truncation boundary and reconciling it with the adjacent parsed-ID sanitizer PR if needed.

## Reproduction Assessment

Yes, at source level. Current main uses UTF-16 `length` and `slice`, and the read-only Node probe shows a 255-character prefix plus emoji leaves a dangling high surrogate before the patch.

## Solution Assessment

Yes. Fixing `sanitizeReplyDirectiveId` is the narrow shared boundary for gateway and webchat reply directive emission; caller-local truncation would duplicate the reply-ID contract and miss future callers.
