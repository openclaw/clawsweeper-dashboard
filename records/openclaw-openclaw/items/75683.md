## Summary

Keep open. Current main still returns generated images as managed media for delivery, but does not stage or mount those files into a sandbox-readable workspace path; the remaining work needs a deliberate sandbox/media artifact contract.

## What This Changes

Request that `image_generate` outputs become usable by sandboxed agents for follow-up workflows via workspace-readable files, dual paths, an export tool, or first-class artifact references.

## Best Possible Solution

Add a bounded generated-media artifact/export contract that exposes only the active run's generated outputs to sandbox follow-up tools while preserving managed delivery paths and host-path denials.

## Reproduction Assessment

Yes, by source inspection. `image_generate` returns managed `.openclaw/media/tool-image-generation` paths, while sandbox file access resolves only workspace, agent workspace, and configured bind mounts before throwing the sandbox-root escape error.

## Solution Assessment

Unclear until maintainers choose the contract. A run-scoped export/staging handle is safer than relaxing sandbox resolution to the entire managed media root.
