## Summary

Keep open. Current main still has no workspace-readable generated-media artifact/export contract; it only preserves managed media paths and has a narrower media-param resolver for reply/message attachment paths.

## What This Changes

Expose `image_generate` outputs to sandboxed follow-up workflows through a workspace-readable path, export handle, or reusable artifact reference.

## Best Possible Solution

Add a bounded generated-media artifact/export contract that exposes only the active run's generated outputs to sandbox follow-up tools while preserving managed delivery paths and host-path denials.

## Reproduction Assessment

Yes, by source inspection. Generate an image, then try to read or process the returned managed media path through sandbox file access; current file-bridge mounts do not include the managed media root, while the newer resolver only covers media-param normalization paths.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded export or workspace-staging handle is safer than relaxing sandbox resolution or mounting the whole managed media root.
