## Summary

Keep open. Current main still returns generated images as managed `MEDIA:` paths and does not materialize them into the sandbox workspace for read/exec/OCR/local-processing workflows; the related Telegram delivery report is adjacent, not a canonical replacement for this workspace-access request.

## What This Changes

The issue requests a workspace-readable or first-class export path for `image_generate` outputs returned as `MEDIA:` managed media paths.

## Best Possible Solution

Add a bounded generated-media artifact/export contract that exposes only the active run's generated outputs to sandbox follow-up workflows while preserving managed delivery paths and host-path denials.

## Reproduction Assessment

Yes, by source inspection. `image_generate` returns managed media paths, while sandbox file access resolves only declared workspace/agent/bind mounts, so using the returned host media path with read or exec follows the existing denial path.

## Solution Assessment

Unclear until maintainers choose the contract. A scoped artifact/export handle or active-run workspace staging is safer than broad sandbox access to the whole managed media root.
