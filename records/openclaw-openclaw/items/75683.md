## Summary

Keep open. Current main still separates managed media delivery from sandbox file access, so the report is valid, but the fix needs a maintainer-chosen artifact/export contract instead of a broad sandbox allowlist.

## What This Changes

Expose `image_generate` outputs to sandbox follow-up workflows through a workspace-readable artifact, dual path, export/copy handle, or direct `MEDIA:` reference consumption.

## Best Possible Solution

Keep managed `MEDIA:` delivery and add a bounded generated-artifact/export contract visible to the active sandbox, with regression tests preserving host-path and arbitrary managed-media denials.

## Reproduction Assessment

Yes, by source inspection. `image_generate` returns managed media paths under `tool-image-generation`, while the sandbox fs bridges resolve only workspace, agent-workspace, and configured bind mounts for file access.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded workspace artifact or export handle is safer than widening sandbox file resolution to all managed media paths.
