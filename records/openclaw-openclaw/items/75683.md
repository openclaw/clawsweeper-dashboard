## Summary

Keep open. Current main still returns image_generate outputs as managed media-store paths, while sandbox file access resolves only workspace, agent-workspace, and configured bind mounts; the newer managed-media normalization helps outbound media delivery but not general read/exec/local-processing workflows.

## What This Changes

Requests that image_generate make generated media usable by sandboxed follow-up workflows through a workspace artifact, export handle, or consumable media reference.

## Best Possible Solution

Add a bounded generated-media artifact or export contract that surfaces image_generate outputs to the active sandbox while preserving the managed delivery path and host-path denials.

## Reproduction Assessment

Yes, by source inspection. Run image_generate in a sandboxed session and pass the returned managed media path to read/exec-style sandbox file access; the fs resolver has no managed-media mount and preserves outside-path rejection.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded workspace artifact or explicit export/media-reference handle is safer than relaxing the sandbox fs resolver for managed media paths wholesale.
