## Summary

Keep open. Current main still leaves generated images as managed media paths and sandbox file access does not mount or export that media root, so the reported follow-up workflow remains unsupported.

## What This Changes

Requests a workspace-readable or exportable artifact for `image_generate` outputs so sandboxed follow-up tools can inspect, modify, upload, or archive generated images.

## Best Possible Solution

Add a bounded generated-media artifact/export contract that exposes only the active run's generated outputs to sandboxed follow-up tools while preserving managed delivery paths and host-path denials.

## Reproduction Assessment

Yes, by source inspection. Run `image_generate` in a sandboxed session and pass the returned managed media path to sandbox file access; current main returns a host media path while sandbox reads resolve only workspace, agent, and configured bind mounts.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded artifact/export handle or workspace staging path is safer than relaxing sandbox resolution to the whole managed media root.
