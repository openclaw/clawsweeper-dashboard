## Summary

Keep open. Current `main` still saves `image_generate` outputs as OpenClaw-managed media and returns `MEDIA:` paths, while sandbox file resolution only maps configured workspace, agent, and bind mounts, so the follow-up workflow gap remains real.

## What This Changes

Request that `image_generate` outputs become usable by sandboxed follow-up workflows through workspace materialization, a copy/export handle, or direct artifact-reference support.

## Best Possible Solution

Add a scoped generated-media export or materialization contract that makes explicit generated artifacts available to follow-up sandbox tools without granting broad host media access.

## Reproduction Assessment

Yes, from source inspection: generated images are saved under managed media and returned as `MEDIA:` paths, while sandbox path resolution only maps configured mounts. I did not run live image generation because this read-only review must not create artifacts or depend on provider credentials.

## Solution Assessment

Unclear. The requested direction is valid, but the best fix needs maintainer choice between workspace copy, explicit export tooling, dual-path details, or artifact-reference consumption by specific tools.
