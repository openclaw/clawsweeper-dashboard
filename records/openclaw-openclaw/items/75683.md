## Summary

Keep open. Current main still returns generated images as managed media paths while sandbox file access resolves only mounted workspace, agent-workspace, and configured bind paths, so the reported follow-up workflow remains unsolved.

## What This Changes

Request generated image outputs to be made workspace-readable, directly consumable by follow-up tools, or exportable through a first-class handle.

## Best Possible Solution

Add a bounded generated-media artifact or export contract that exposes a workspace-readable copy or handle to the active sandbox without broadening ordinary host-path access.

## Reproduction Assessment

Yes, by source inspection. Generate an image, then pass the returned managed media path to sandbox read or exec-style file access; the file bridge and container mounts do not expose the managed media store.

## Solution Assessment

Unclear until maintainers choose the contract. A workspace-staged artifact or explicit export handle is safer than relaxing sandbox file resolution for managed media paths broadly.
