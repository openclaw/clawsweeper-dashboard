## Summary

Keep open. Current main still delivers generated images as managed `MEDIA:` attachments, but it does not materialize those outputs into the sandbox workspace or expose them through the sandbox fs bridge for follow-up `read`, `exec`, upload, OCR, or local processing workflows.

## What This Changes

The issue asks for `image_generate` outputs to be available to sandboxed follow-up workflows through workspace staging, dual paths, export handles, or direct artifact references.

## Best Possible Solution

Add a bounded generated-media artifact/export contract that exposes only the active run's generated outputs to sandboxed workflows while preserving managed delivery paths and host-path denial semantics.

## Reproduction Assessment

Yes, by source inspection. Run `image_generate` in a sandboxed session, then pass the returned managed media path to sandbox file access; the fs bridge has no media-root mount and routes outside paths to the sandbox-root denial path.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded artifact/export handle or workspace-staged copy is safer than exposing the whole managed media root to the sandbox.
