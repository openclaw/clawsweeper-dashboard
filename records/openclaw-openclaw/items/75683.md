## Summary

Keep open: current main still returns managed `MEDIA:` paths for generated images while sandbox file access only resolves mounted workspace, agent-workspace, and explicit bind paths, so sandboxed follow-up tools still cannot consume the generated file directly.

## What This Changes

Request that `image_generate` outputs become usable by sandboxed follow-up tools through a workspace-readable copy, dual paths, export handle, or first-class artifact reference.

## Best Possible Solution

Add a bounded generated-media artifact/export contract for active-run outputs that preserves managed delivery paths and host media-root isolation.

## Reproduction Assessment

Yes, by source inspection: `image_generate` returns managed media paths and sandbox reads resolve only mounted paths before denying unmapped host locations. I did not run a live image provider in this read-only review.

## Solution Assessment

Unclear until maintainers choose the contract. A scoped active-run export or staging handle is safer than relaxing sandbox access to the whole managed media root.
