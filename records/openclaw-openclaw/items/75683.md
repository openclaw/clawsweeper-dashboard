## Summary

Keep open. Current main still returns generated images as managed media-store paths while sandbox file follow-up tools only resolve workspace, agent-workspace, and configured bind mounts, so the reported workflow remains blocked.

## What This Changes

Request that image_generate make generated media available to sandbox workspace follow-up tools through workspace materialization, an export handle, or direct consumable media references.

## Best Possible Solution

Keep the existing MEDIA delivery path and add a bounded generated-output export or workspace materialization path that sandbox follow-up tools can read, while preserving denials for unrelated managed media and host paths.

## Reproduction Assessment

Yes. The code-level reproduction is to run image_generate, observe the managed media-store path in the tool output, then pass that path to a sandbox fs bridge or exec-style file operation where it misses all allowed mounts and is rejected as outside the sandbox root.

## Solution Assessment

Yes, the requested direction is appropriate if implemented as a narrow generated-output export/copy or first-class handle. The safer alternative is not to loosen the sandbox fs resolver for all managed-media paths.
