## Summary

Keep open. Current main still returns generated images as managed `MEDIA:` paths for delivery, but sandbox file access only resolves mounted workspace, agent workspace, and configured bind paths, so the reported follow-up workflow remains unsupported.

## What This Changes

The issue requests a workspace-readable or exportable handle for `image_generate` outputs so agents can inspect, modify, upload, or archive generated media in later steps.

## Best Possible Solution

Add a bounded generated-media artifact or export contract that exposes only eligible generated outputs to sandbox follow-up tools while preserving managed delivery paths and host-path denials.

## Reproduction Assessment

Yes, by source inspection. Run `image_generate` in a sandboxed session, then pass the returned managed media path to sandbox file access; the file bridge has no media-root mount and routes outside paths to the sandbox-root denial path.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded artifact/export handle is safer than relaxing sandbox resolution to the whole managed media root.
