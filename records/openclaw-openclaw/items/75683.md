## Summary

Keep open. Current main still saves `image_generate` outputs as managed delivery media while sandbox file tools only resolve workspace, agent-workspace, and configured bind mounts, so follow-up workflows cannot consume the generated file from inside the sandbox.

## What This Changes

Request that `image_generate` outputs be usable by sandboxed follow-up tools through a workspace-readable copy, dual path, export handle, or direct media-reference consumption.

## Best Possible Solution

Keep the managed delivery `MEDIA:` path and add a bounded generated-output artifact or export contract that is visible to the active sandbox, with regression tests preserving managed-media and host-path denials.

## Reproduction Assessment

Yes, by source inspection. Run `image_generate` in a sandboxed session, then pass the returned managed media path to the sandbox fs bridge or exec-style file access; the resolver has no media-root mount and rejects paths outside the sandbox workspace.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded workspace artifact or export handle is the safer direction; relaxing the global sandbox resolver for managed media would be too broad.
