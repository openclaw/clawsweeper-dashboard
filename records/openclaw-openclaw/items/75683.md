## Summary

Keep open. Current main still returns generated images as OpenClaw-managed media paths for delivery, while sandbox file access only resolves configured workspace, agent workspace, and bind mounts, so the follow-up workflow remains unsupported.

## What This Changes

Request generated `image_generate` outputs be made accessible to sandboxed follow-up workflows through workspace staging, dual paths, a copy/export handle, or direct artifact references.

## Best Possible Solution

Add a bounded generated-media artifact/export contract that exposes only the active run's generated outputs to sandboxed follow-up tools while preserving managed delivery paths and existing host-path denials.

## Reproduction Assessment

Yes, by source inspection. Run `image_generate` in a sandboxed session, then pass the returned managed media path to sandbox file access; the bridge has no media-root mount and falls through to the sandbox-root denial path.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded artifact/export handle or active-run workspace staging is maintainable; relaxing sandbox resolution to the whole managed media root would be too broad.
