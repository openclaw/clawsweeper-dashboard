## Summary

Keep open. Current main still returns generated image outputs as managed media paths, while sandbox file access only resolves workspace, agent workspace, and configured bind mounts, so the follow-up workflow remains unsupported.

## What This Changes

Request workspace-readable generated-image artifacts or a first-class export handle so sandboxed follow-up tools can use `image_generate` outputs.

## Best Possible Solution

Add a bounded generated-media artifact/export contract that exposes only the active run's generated outputs to sandbox follow-up tools while preserving managed delivery paths and host-path denials.

## Reproduction Assessment

Yes, by source inspection. `image_generate` returns managed `MEDIA:` paths from `saveMediaBuffer`, while the sandbox fs bridge resolves reads only through workspace, agent, and bind mounts.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded artifact/export handle or active-run workspace staging is safer than relaxing sandbox resolution to the whole managed media root.
