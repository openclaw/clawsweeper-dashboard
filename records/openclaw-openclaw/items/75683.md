## Summary

Keep open. Current main still returns `image_generate` outputs as managed media paths and the sandbox file bridge still mounts only workspace, agent workspace, and configured binds, so sandboxed follow-up reads/uploads remain unsupported.

## What This Changes

Expose `image_generate` outputs to sandboxed follow-up workflows through a workspace-readable path or first-class media export/reference mechanism.

## Best Possible Solution

Add a bounded generated-media artifact/export contract that exposes only the active run's generated outputs inside the sandbox workspace while preserving managed delivery paths and host-path denials.

## Reproduction Assessment

Yes, by source inspection. Generate an image, then pass the returned managed media path to sandbox file access; current mounts do not include that media root, so the bridge rejects or cannot see the host path.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded export/workspace staging handle is safer than relaxing sandbox resolution to the whole managed media root.
