## Summary

Keep open. Current main still exposes `image_generate` outputs as managed media delivery paths, but sandbox file access is limited to mounted workspace roots and configured binds, so the requested follow-up workflow remains unsupported.

## What This Changes

Request generated `image_generate` media to be available to sandboxed agents through a workspace-readable copy, dual path, export tool, or first-class artifact reference.

## Best Possible Solution

Add a bounded generated-media artifact/export contract that exposes active generated outputs to sandboxed follow-up tools while preserving managed delivery paths and host-path denials.

## Reproduction Assessment

Yes, by source inspection. `image_generate` returns managed media paths, while sandbox file reads resolve only mounted workspace roots and reject paths outside that boundary.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded artifact/export handle or workspace staging path is safer than mounting the whole managed media root into sandboxed sessions.
