## Summary

Keep open. Current `main` still delivers generated images through managed `MEDIA:` paths, but sandbox filesystem tools only resolve workspace, agent-workspace, and configured bind mounts, so follow-up workflows cannot reliably read or reuse `image_generate` output files from inside the sandbox.

## What This Changes

The issue requests a workspace-readable or first-class export path for `image_generate` outputs so sandboxed follow-up tools can inspect, modify, upload, or archive generated media.

## Best Possible Solution

Define a bounded generated-media artifact/export contract that exposes only the active run's generated outputs to sandbox follow-up tools while preserving managed delivery paths and host-path denials.

## Reproduction Assessment

Yes, by source inspection. Run `image_generate`, then pass the returned managed media host path to sandbox read/exec or another sandboxed file consumer; that path is outside every fs-bridge mount and falls to the sandbox-root escape path.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded export or workspace-staging mechanism is safer than making the whole managed media root generally visible to sandboxed tools.
