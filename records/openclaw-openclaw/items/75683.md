## Summary

Keep open. Current main still does not provide a workspace-readable or exportable generated-media artifact for `image_generate`; it saves managed media paths and has a narrower resolver for outbound media params, so follow-up `exec`, local processing, OCR, and archiving workflows remain unresolved.

## What This Changes

Expose `image_generate` outputs to sandboxed follow-up workflows through workspace staging, an export handle, or first-class media-reference consumption.

## Best Possible Solution

Add a least-privilege generated-media artifact/export contract that exposes only active-run outputs to sandbox follow-up tools while preserving managed delivery paths and host-path denials.

## Reproduction Assessment

Yes, by source inspection. Generate an image, then try to read or process the returned managed media path through generic sandbox file access; `image_generate` returns managed media paths while the sandbox file bridge does not mount that media root.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded export/workspace-staging handle is safer than relaxing sandbox path checks or mounting the whole managed media root.
