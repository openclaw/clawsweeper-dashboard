## Summary

Keep open. Current main still delivers generated images as managed media paths, while sandbox file tools resolve only workspace, agent-workspace, and configured bind mounts, so follow-up workflows cannot consume the generated file through the sandbox.

## What This Changes

Request generated image outputs to be materialized or exported into a sandbox/workspace-readable form for follow-up tools.

## Best Possible Solution

Add a bounded generated-media artifact or export contract that is visible to the active sandbox while preserving the managed delivery path and existing host-path denials.

## Reproduction Assessment

Yes, by source inspection. Run `image_generate` in a sandboxed session, then pass the returned managed media path to sandbox read/exec-style file access; the resolver has no media-root mount and rejects paths outside the sandbox workspace.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded workspace artifact or export handle is the safer direction; relaxing the sandbox resolver for managed media would be too broad.
