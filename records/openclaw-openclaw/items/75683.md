## Summary

Keep open. Current main still delivers `image_generate` results as managed `MEDIA:` files, and recent managed-media allowlisting only helps outbound delivery/media parameters; sandbox file tools and exec-style access still resolve through workspace, agent, and bind mounts, so the reported follow-up workflow remains unsolved.

## What This Changes

The issue requests that `image_generate` outputs be materialized or exposed so sandboxed follow-up tools can inspect, edit, archive, or upload the generated files.

## Best Possible Solution

Keep managed `MEDIA:` delivery intact and add a bounded generated-artifact/export contract that exposes only the current tool outputs to the active sandbox with regression coverage for host-path denial.

## Reproduction Assessment

Yes, by source inspection. Generate an image in a sandboxed session, then pass the returned managed media path to a sandbox file/read/exec path: the file bridge has no managed-media mount and rejects paths outside workspace, agent workspace, or explicit binds.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded workspace artifact or export handle is safer than broadening the sandbox resolver to all managed media paths.
