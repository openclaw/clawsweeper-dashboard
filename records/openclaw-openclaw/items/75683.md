## Summary

Keep open. Current main still returns `image_generate` outputs as managed media paths, while sandbox file access resolves only workspace, agent-workspace, and configured bind mounts, so follow-up file workflows still need a new artifact/export contract.

## What This Changes

Request that `image_generate` outputs be available to sandboxed follow-up workflows via a workspace-readable file, dual path, export tool, or direct `MEDIA:` artifact consumption.

## Best Possible Solution

Add a bounded generated-media artifact or export contract visible to the active sandbox while preserving the managed delivery path and existing host-path denials.

## Reproduction Assessment

Yes, by source inspection. Run `image_generate` in a sandboxed session, then pass the returned `~/.openclaw/media/tool-image-generation/...` path to sandbox read or exec-style file access; the resolver has no media-root mount and rejects paths outside the sandbox workspace.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded workspace artifact or export handle is the safer direction; relaxing the sandbox resolver for managed media would be too broad.
