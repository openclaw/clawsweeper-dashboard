## Summary

Keep open. Current main still writes `image_generate` results to managed media and exposes those paths for delivery, while sandbox file access is limited to workspace, agent workspace, and configured bind mounts, so the requested follow-up workflow remains unsupported.

## What This Changes

The issue requests making generated image outputs workspace-addressable or exportable so sandboxed follow-up tools can inspect, modify, archive, or upload them.

## Best Possible Solution

Add a bounded generated-media artifact/export contract that exposes only the active run's generated outputs to the sandbox while preserving the managed delivery path and host-path denials.

## Reproduction Assessment

Yes, by source inspection. Run `image_generate` in a sandboxed session, then pass the returned managed media path to sandbox file access; the file bridge has no media-root mount and routes outside paths to the sandbox-root denial path.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded artifact/export handle is the safer direction; relaxing sandbox resolution to the whole managed media root would be too broad.
