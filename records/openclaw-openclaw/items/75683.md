## Summary

Keep open. Current main still does not make `image_generate` outputs generally workspace-readable inside sandboxed follow-up tools; it only preserves managed media for delivery and selected outbound media normalization, so inspect/edit/archive workflows remain unresolved.

## What This Changes

Request a sandbox-readable artifact, export handle, or equivalent path contract for `image_generate` outputs so later tools can reuse generated images.

## Best Possible Solution

Add a bounded generated-output artifact or explicit export contract visible to the active sandbox while keeping managed `MEDIA:` delivery and host-path denials intact.

## Reproduction Assessment

Yes, by source inspection. Run `image_generate` in a sandboxed session and then pass its returned managed media path to sandbox `read` or `exec`; the image tool emits a managed-media path while the sandbox file bridge only resolves workspace, agent, and bind mounts.

## Solution Assessment

Unclear until maintainers choose the contract. The safer direction is a bounded workspace artifact or export handle; relaxing the general sandbox resolver for managed media would be too broad.
