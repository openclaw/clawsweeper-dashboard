## Summary

Keep open. Current main still returns generated images as managed media paths without a sandbox/workspace-readable copy or export handle, while sandbox file tools only resolve workspace, agent-workspace, and configured bind mounts.

## What This Changes

Request that `image_generate` outputs become accessible to sandboxed follow-up workflows through a workspace-readable artifact, paired path, export tool, or direct `MEDIA:` reference consumption.

## Best Possible Solution

Keep managed `MEDIA:` delivery and add a bounded workspace-visible generated artifact or export handle with regression coverage that unrelated managed media and host paths remain denied.

## Reproduction Assessment

Yes, at source level. The reproduction path is to run `image_generate` in a sandboxed session and then pass the returned `/home/.../.openclaw/media/tool-image-generation/...` path to a read, exec, or attachment workflow; current main returns that managed path while the sandbox fs bridge resolves only mounted workspace roots.

## Solution Assessment

Yes. The maintainable fix is a narrow generated-output materialization or export path, not a global sandbox resolver exception for all managed media.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/75683. In sandboxed sessions, `image_generate` currently returns managed `MEDIA:` paths under `.openclaw/media/tool-image-generation` that follow-up file/exec/upload tools cannot read. Preserve the managed `MEDIA:` delivery path, but add a bounded workspace/sandbox-readable generated artifact or export handle for generated outputs, preferably under a deterministic workspace media directory such as `media/generated` or `media/tool-image-generation`. Return the readable path in tool text/details in a way agents can use for follow-up workflows. Do not broaden `resolveSandboxFsPathWithMounts` to allow arbitrary managed media or host paths. Add regression coverage proving the generated output can be read through the sandbox bridge while unrelated managed media and host paths remain denied. Update `docs/tools/image-generation.md` and `CHANGELOG.md` if the user-facing tool result shape changes.
