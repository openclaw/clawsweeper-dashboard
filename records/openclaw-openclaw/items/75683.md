## Summary

Keep open. Current main still returns `image_generate` outputs as managed media-store paths, while sandbox follow-up file tools only resolve mounted workspace roots and configured binds, so the reported follow-up workflow remains blocked.

## What This Changes

Make `image_generate` outputs available to sandboxed follow-up workflows through a workspace-readable artifact path, export handle, or equivalent first-class media reference.

## Best Possible Solution

Keep the existing `MEDIA:` delivery path and add a bounded generated-output export or workspace materialization path that sandbox follow-up tools can read while preserving denials for unrelated managed media and host paths.

## Reproduction Assessment

Yes. The code-level reproduction is to run `image_generate`, observe a managed `tool-image-generation` media path in the tool result, then pass that path to a sandbox file operation where it misses all allowed mounts and is rejected as outside the sandbox root.

## Solution Assessment

Yes, the requested direction is appropriate if implemented as a narrow generated-output copy/export or first-class handle. The safer alternative is not to loosen sandbox filesystem resolution for all managed media paths.
