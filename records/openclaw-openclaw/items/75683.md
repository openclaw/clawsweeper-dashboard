## Summary

Keep open. Current main partially improves outbound delivery for managed tool media, but generated images still are not workspace-addressable for sandbox read/exec/local processing workflows, and the artifact/export contract needs maintainer design.

## What This Changes

Request `image_generate` outputs be materialized in, exported to, or otherwise addressable from the active sandbox workspace for follow-up tools.

## Best Possible Solution

Keep the managed delivery path and add a bounded generated-output artifact or export contract visible to the active sandbox, with tests preserving host-path denials.

## Reproduction Assessment

Yes, by source inspection. Run `image_generate` in a sandboxed session, then pass the returned managed media path to a sandbox read/exec-style file access path; the fs bridge has no managed-media mount and resolves only workspace, agent workspace, and configured binds.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded workspace artifact, export handle, or explicit media-ref consumer is safer than relaxing the global sandbox filesystem resolver.
