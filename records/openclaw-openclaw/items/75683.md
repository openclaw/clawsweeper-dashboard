## Summary

Keep open. Current main still returns generated images as managed delivery media while sandbox file tools resolve only workspace, agent-workspace, and configured bind mounts, so the requested follow-up workflow remains unsupported.

## What This Changes

Requests that `image_generate` outputs be materialized or exported into a sandbox/workspace-readable artifact for follow-up tools.

## Best Possible Solution

Add a bounded generated-media artifact or export contract visible to the active sandbox while preserving managed delivery media and host-path denials.

## Reproduction Assessment

Yes, by source inspection. Generate an image, then pass the returned `.openclaw/media/tool-image-generation` path to sandbox file access; the bridge has no media-root mount and resolves only workspace, agent, or configured bind paths.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded workspace artifact or export handle is the safer direction; relaxing the global sandbox resolver for managed media would be too broad.
