## Summary

Keep open. Current main still treats `image_generate` outputs as managed `MEDIA:` delivery artifacts, while sandbox file access is limited to workspace, agent workspace, and configured bind mounts, so the reported follow-up workflow remains unsupported.

## What This Changes

Expose `image_generate` outputs to sandboxed follow-up workflows through a workspace-readable artifact, dual path, export handle, or direct `MEDIA:` artifact consumption.

## Best Possible Solution

Define a bounded generated-media artifact or export contract that exposes only the active run's generated outputs to sandbox follow-up tools while preserving managed `MEDIA:` delivery and host-path denials.

## Reproduction Assessment

Yes, by source inspection. Current main saves `image_generate` output under managed media and the sandbox read resolver has no media-root mount, so passing the returned managed path to sandbox file access remains outside the mounted workspace boundary.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded artifact/export handle is the safer direction; relaxing sandbox resolution to the whole managed media root would be too broad.
