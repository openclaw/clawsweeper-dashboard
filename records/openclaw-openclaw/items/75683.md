## Summary

Keep open. Current main still returns generated images as managed media paths while sandbox file tools only resolve workspace, agent-workspace, and explicit bind mounts, so the reported follow-up workflows remain unsupported.

## What This Changes

Requests that `image_generate` outputs be materialized or exportable into a sandbox-readable workspace path for follow-up file-tool workflows.

## Best Possible Solution

Add a bounded generated-media artifact/export contract visible to the active sandbox while preserving managed delivery paths and existing host-path denials.

## Reproduction Assessment

Yes. By source inspection, run `image_generate` in a sandboxed session and pass the returned managed `MEDIA:` path to `read` or `exec`; the resolver can only map workspace, agent-workspace, and explicit bind mounts.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded workspace artifact or explicit export/copy handle is safer than adding the managed media root to general sandbox file mounts.
