## Summary

Keep open. Current main still returns `image_generate` outputs as managed media paths that are usable for delivery, while sandbox file tools only resolve workspace, agent-workspace, and configured bind mounts; making generated media workspace-addressable needs an explicit sandbox/media contract.

## What This Changes

Request to make `image_generate` outputs workspace-readable or exportable so sandboxed follow-up tools can inspect, modify, or upload generated media.

## Best Possible Solution

Add a bounded generated-media artifact or export contract that exposes outputs to the active sandbox without broadening arbitrary host-path access, then document the workflow.

## Reproduction Assessment

Yes, by source inspection on current main. `image_generate` returns a managed media-root path, and sandbox file operations resolve only mounted workspace/agent/bind paths before rejecting out-of-sandbox paths.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded workspace artifact or export handle is safer than simply mounting the whole managed media root into sandbox filesystem tools.
