## Summary

Keep open. Current main still returns generated images as managed media-store paths while sandbox read/exec-style file access is limited to configured sandbox mounts, so the reported follow-up workflow remains unsupported. https://github.com/openclaw/openclaw/issues/75683

## What This Changes

Request that `image_generate` outputs be materialized or exported into a sandbox/workspace-readable form for follow-up tools such as inspection, upload, OCR, and archiving.

## Best Possible Solution

Add a bounded generated-media artifact/export contract that exposes generated outputs to the active sandbox without broadening arbitrary host-path access.

## Reproduction Assessment

Yes, by source inspection. `image_generate` returns managed media-store paths, and sandbox file reads resolve only through workspace/agent/bind mounts, so a returned media-cache host path is outside the sandbox file-access contract.

## Solution Assessment

Unclear until maintainers choose the contract. A bounded workspace artifact or explicit export/media-handle flow is safer than teaching generic sandbox reads to accept managed media roots.
