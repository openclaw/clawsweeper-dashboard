## Best Possible Solution

Land one narrow lifecycle-generation guard after session-state maintainer review and required checks, while treating the alternate open candidate PR as redundant if this one merges.

## Solution Assessment

Yes, this is the best narrow fix: lifecycleGeneration is the discriminator between current maintenance candidates and stale pre-restart recovery candidates, while a status-only guard would break existing non-running stale-generation recovery coverage.
