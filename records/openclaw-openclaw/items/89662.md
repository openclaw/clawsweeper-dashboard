## What This Changes

The issue reports the Control UI webchat context usage indicator disappearing after message send and showing a lower token count than `/status` for a local OpenAI-compatible provider session.

## Best Possible Solution

Make the selected webchat session usage pipeline refresh to the same fresh `totalTokens`/`contextTokens` source that `/status` uses after a run completes, while still hiding or marking genuinely unknown stale totals instead of showing old data as fresh.

## Reproduction Assessment

No live high-confidence reproduction was run. The reporter gives concrete steps, and source inspection shows a plausible current-main path where stale session events clear `totalTokens` and make the context notice render nothing.

## Solution Assessment
