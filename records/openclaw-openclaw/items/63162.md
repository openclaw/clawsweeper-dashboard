## Summary

Keep open. Current main still lacks the requested Control UI file explorer toggle/sidebar and the backing arbitrary workspace tree/read API; existing file surfaces are chat attachment upload/drop, transcript artifacts, and the narrow Agents Core Files editor.

## What This Changes

The issue requests a Control UI folder button near Cron/Settings that toggles a collapsible workspace file-tree sidebar with file open, drag-and-drop, context menu, refresh, and search behavior.

## Best Possible Solution

Keep this open until maintainers choose the workspace-browsing product surface, then land a scoped workspace file API plus the selected Control UI browser with search, refresh, open-in-chat behavior, docs, and path-safety coverage.

## Reproduction Assessment

Not applicable as a feature request. Current-main source/docs inspection gives high-confidence absence proof: there is no Control UI explorer toggle/sidebar, no recursive workspace tree UI, and no shipped agents.workspace-style API.

## Solution Assessment

Unclear. A file browser may be useful, but adding only a top-bar button would be premature before the workspace API/security model and UI placement are resolved through #61368/#62417 or an equivalent maintainer design.
