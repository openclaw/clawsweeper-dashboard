## Summary

Keep open. Current main still lacks the requested Control UI file explorer toggle/sidebar and the backing arbitrary workspace tree/read API; the only shipped file surfaces are chat attachment upload/drop and the narrow Agents Core Files editor. Related #61368/#62417 cover broader workspace file management, but they remain open/unmerged and do not settle this issue's chat/top-bar sidebar placement.

## What This Changes

The issue requests a Control UI folder button near Cron/Settings that toggles a collapsible workspace file-tree sidebar with file open, drag-and-drop, context menu, refresh, and search behavior.

## Best Possible Solution

Keep this open until maintainers decide whether workspace browsing belongs in the chat top bar/sidebar, the Agents workspace tab, or both. The durable end state should first land a scoped workspace file API and then add the chosen Control UI browser with search/refresh/open-in-chat behavior and path-safety coverage.
