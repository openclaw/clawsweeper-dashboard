## Summary

Keep this issue open: current main and the latest release still do not populate the first-class task_runs.agent_id field from the executing child agent for cross-agent sessions_spawn task rows, and the remaining fix is already owned by the open closing PR at https://github.com/openclaw/openclaw/pull/88872.

## What This Changes

The issue asks OpenClaw to persist the executing child agent id in task_runs.agent_id for sessions_spawn/subagent task rows.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/88872, then close this issue once the fix is merged and shipped or present on main.

## Reproduction Assessment

Yes. Source tracing reproduces the path: current main builds a childSessionKey from targetAgentId, registers the task without targetAgentId, resolves agentId from owner/requester fields, and persists that value to SQLite.

## Solution Assessment

Yes. The narrow maintainable fix is to pass or infer the child agent during core task creation; plugin-side child_session_key parsing should remain a workaround rather than the canonical attribution field.
