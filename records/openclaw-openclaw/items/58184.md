## Summary

Keep open: the security goal is valid, but the current branch is not mergeable as-is because it still allows arbitrary secure WebSocket egress, drops current required HTTP origins, and regresses documented remote `ws://` gateway use.

## What This Changes

The PR changes the Control UI CSP helper and tests to emit `connect-src 'self' wss:` with optional extra origins instead of the current broader `connect-src` directive.

## Best Possible Solution

Define an explicit Control UI `connect-src` policy that blocks arbitrary WebSocket egress while preserving OpenAI Realtime, tweakcn imports, and supported remote gateway workflows through exact origins, same-origin proxying, or a deliberate documented deprecation.

## Reproduction Assessment

Yes. Static source inspection shows current main allows blanket `ws:`/`wss:` schemes, while the PR head still emits blanket `wss:` and would drop current required origins; I did not run a live browser CSP repro in this read-only review.

## Solution Assessment

No. The current PR is not the best fix because it preserves arbitrary `wss:` egress, removes required `https://api.openai.com` and `https://tweakcn.com` origins, and blocks documented plaintext remote gateway URLs.
