## Summary

Keep this PR open, but not merge-ready: the security-hardening goal is valid, yet the current branch still permits arbitrary `wss:` egress and would conflict with current Control UI CSP behavior for OpenAI Realtime, tweakcn imports, and documented remote gateway URLs.

## What This Changes

The PR changes the Control UI CSP helper and tests to replace blanket `ws:`/`wss:` `connect-src` handling with `connect-src 'self' wss:` plus an unused `extraConnectSrc` option.

## Best Possible Solution

Define an explicit Control UI `connect-src` policy that blocks arbitrary WebSocket egress while preserving supported gatewayUrl, OpenAI Realtime, and tweakcn flows through exact origins, a same-origin relay/proxy, or an intentional documented deprecation.

## Reproduction Assessment

Yes, at source level: current main and the latest release still emit blanket `ws:`/`wss:` sources, and the PR head still emits blanket `wss:`. I did not run a real browser CSP repro, so runtime proof remains missing.

## Solution Assessment

No. The current PR is not the best fix because it leaves arbitrary `wss:` egress while dropping current HTTP connect origins and documented plaintext remote gateway URLs without a maintainer decision.
