## Summary

Keep open: the security-hardening direction is plausible, but the current patch is not mergeable because it still permits arbitrary `wss:` egress, drops current main's required HTTP connect origins, and changes documented remote `ws://` gateway behavior without a maintainer policy decision or real browser proof.

## What This Changes

This PR changes the Control UI CSP builder and tests to remove bare `ws:`, keep `wss:`, and add an optional extra `connect-src` origin parameter.

## Best Possible Solution

Define an explicit Control UI connection policy that defaults to same-origin, preserves required OpenAI and tweakcn origins, and handles remote gateways through validated exact origins, same-origin proxying, or a documented deprecation path with tests and docs.

## Reproduction Assessment

Yes. Static source inspection is enough for the review findings: the PR head emits `connect-src 'self' wss:`, while current main still needs OpenAI/tweakcn HTTP origins and documents/configures remote `ws://` and `wss://` gateway URLs.

## Solution Assessment

No. The current patch keeps the broad `wss:` exfiltration route, drops current required origins, and changes documented remote gateway behavior without a clear security/product policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Remove the blanket wss: source:** `src/gateway/control-ui-csp.ts:58`
  - body: The patched directive still emits `connect-src 'self' wss:`. A scheme source permits WebSocket connections to arbitrary secure hosts, so injected Control UI script could still exfiltrate over `wss://...`; this does not deliver the stated CSP hardening.
  - confidence: 0.94
- **[P2] Keep required HTTP connect origins:** `src/gateway/control-ui-csp.ts:58`
  - body: Current main allows `https://api.openai.com` for browser Realtime offers and `https://tweakcn.com` for theme imports. Replacing the directive with only `'self' wss:` drops both origins and would break those Control UI features after rebase.
  - confidence: 0.9
- **[P2] Preserve documented ws gateway URLs:** `src/gateway/control-ui-csp.ts:58`
  - body: Hardcoding `wss:` while removing `ws:` blocks cross-origin plaintext WebSocket targets, but the UI still accepts `gatewayUrl`, opens it with `new WebSocket(...)`, and the docs still show `gatewayUrl=ws://<gateway-host>:18789`. Preserve this with exact origins or retire it deliberately with docs and tests.
  - confidence: 0.88
