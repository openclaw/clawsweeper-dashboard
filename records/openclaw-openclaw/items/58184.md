## Summary

Keep open: the security-hardening goal is valid, but the current branch is not merge-ready because it leaves arbitrary secure WebSocket egress open and would break supported Control UI connection flows.

## What This Changes

The PR changes the Control UI CSP helper to emit `connect-src 'self' wss:` with an optional `extraConnectSrc` list and adds CSP assertions.

## Best Possible Solution

Define an explicit Control UI `connect-src` policy that blocks arbitrary WebSocket egress while preserving supported gatewayUrl, OpenAI Realtime, Google Live, and tweakcn flows through exact origins, a same-origin relay/proxy, or a deliberate documented deprecation.

## Reproduction Assessment

Yes, at source level: current main and `v2026.5.18` emit blanket `ws:`/`wss:` sources, and the PR head still emits blanket `wss:`. I did not run a real browser CSP repro, so runtime proof remains missing.
