## Summary

Keep this PR open, but it is not merge-ready: the current branch still allows arbitrary `wss:` egress, can drop shipped Control UI connect origins, and can break documented `ws://` remote gateway URLs.

## What This Changes

The PR changes the Control UI CSP builder and tests to replace the blanket `ws:`/`wss:` `connect-src` policy with `connect-src 'self' wss:` plus an optional `extraConnectSrc` list.

## Best Possible Solution

Define an explicit Control UI `connect-src` policy that blocks arbitrary WebSocket egress while preserving supported gatewayUrl, OpenAI Realtime, and tweakcn flows through exact origins, a same-origin relay/proxy, or an intentional documented deprecation.
