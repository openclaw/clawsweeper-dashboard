## Summary

Keep this PR open, but do not merge the current head: it still leaves arbitrary `wss://` egress allowed, conflicts with documented remote `ws://` gateway flows, and is stale against current main's OpenAI Realtime CSP origin.

## What This Changes

The PR changes the Control UI CSP builder and tests to remove `ws:`, keep `wss:`, and add optional extra `connect-src` origins.

## Best Possible Solution

Define an explicit Control UI connection policy that blocks arbitrary WebSocket egress while preserving required OpenAI Realtime calls and supported gatewayUrl flows through exact validated origins or a documented proxy/deprecation path.

## Reproduction Assessment

Yes. Static inspection is enough: PR head emits `connect-src 'self' wss:`, current main opens configured `gatewayUrl` values via `new WebSocket(...)`, and current main fetches OpenAI Realtime offers from `https://api.openai.com`.

## Solution Assessment

No. The patch keeps the main `wss://` exfiltration channel open by scheme and drops current supported origins; an exact-origin policy or same-origin proxy path would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Remove the blanket wss: source:** `src/gateway/control-ui-csp.ts:58`
  - body: The patched directive still emits `connect-src 'self' wss:`. A scheme source permits WebSocket connections to arbitrary secure hosts, so injected Control UI script could still exfiltrate over `wss://...`; this does not deliver the stated CSP hardening.
  - confidence: 0.94
- **[P2] Preserve documented ws:// gateway URLs:** `src/gateway/control-ui-csp.ts:58`
  - body: Hardcoding `wss:` while removing `ws:` blocks cross-origin plaintext gateway targets, but the UI still accepts `gatewayUrl`, opens it with `new WebSocket(...)`, and the docs still show `gatewayUrl=ws://<gateway-host>:18789`. Preserve this with exact origins or retire it deliberately with docs and tests.
  - confidence: 0.9
- **[P2] Keep the OpenAI realtime offer origin:** `src/gateway/control-ui-csp.ts:58`
  - body: Current main added `https://api.openai.com` to `connect-src` because the browser WebRTC transport posts SDP offers there. The PR's replacement directive drops that origin, so reconciling this branch as written would break OpenAI Realtime Talk in the Control UI.
  - confidence: 0.88
