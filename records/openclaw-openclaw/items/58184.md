## Summary

Keep this PR open, but do not merge the current head: the change is useful security hardening work, yet it still permits arbitrary `wss:` connections, blocks documented `ws://` remote-gateway workflows, and is stale against current main's OpenAI Realtime CSP allowance.

## What This Changes

The PR changes the Control UI CSP builder and tests to remove the blanket `ws:` source, keep a blanket `wss:` source, and add an `extraConnectSrc` option.

## Best Possible Solution

Define an explicit Control UI connection policy: default to same-origin, preserve current exact provider origins, and support remote gateways only through narrowly validated exact origins or an intentional documented deprecation path.

## Reproduction Assessment

Yes. Static inspection is enough: the PR head emits `connect-src 'self' wss:`, while current main still accepts `ws://`/`wss://` `gatewayUrl` values and opens them via `new WebSocket(...)`; current main also fetches OpenAI Realtime offers from `https://api.openai.com`.

## Solution Assessment

No. The current patch keeps the secure WebSocket exfiltration channel open by scheme, drops plaintext remote gateway support, and omits a current required HTTP origin; an exact-origin allowlist, same-origin proxy, or explicit deprecation would be safer.
