## Summary

Keep open: the PR is aimed at a real security hardening area, but the current branch still allows arbitrary `wss:` egress, drops required Control UI HTTP origins, and changes documented remote gateway behavior without real browser proof.

## What This Changes

The PR changes the Control UI CSP builder and tests to remove bare `ws:`, keep `wss:`, and add an optional `extraConnectSrc` list.

## Best Possible Solution

Define an explicit Control UI `connect-src` policy that blocks arbitrary WebSocket egress while preserving OpenAI Realtime, tweakcn imports, and documented remote gateway URLs through exact origins, same-origin proxying, or a deliberate docs-backed deprecation.

## Reproduction Assessment

Yes. Static source inspection of the PR head and current main reproduces the review findings: the branch emits `connect-src 'self' wss:` while current main still needs HTTP origins and supports documented `ws://`/`wss://` gateway URLs.

## Solution Assessment

No. The current patch keeps broad secure WebSocket egress, drops required current origins, and changes documented remote gateway behavior without a settled policy.
