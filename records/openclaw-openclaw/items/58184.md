## Summary

Keep open: the branch is not mergeable as-is because it still allows arbitrary `wss:` egress, would lose current required HTTP origins, changes documented remote gateway behavior, and lacks real browser CSP proof.

## What This Changes

The PR changes the Control UI CSP helper to emit `connect-src 'self' wss:` with optional extra origins and adds tests expecting that policy.

## Best Possible Solution

Define an explicit Control UI `connect-src` policy that blocks arbitrary WebSocket egress while preserving OpenAI Realtime, tweakcn imports, and documented remote gateway URLs through exact origins, same-origin proxying, or a deliberate docs-backed deprecation.

## Reproduction Assessment

Yes. Static source inspection of the PR head and current main shows the PR emits `connect-src 'self' wss:` while current main needs HTTP origins and still documents `ws://` and `wss://` remote gateway URLs; I did not run a live browser CSP repro in this read-only review.

## Solution Assessment

No. The patch keeps blanket `wss:`, omits current allowlisted HTTP origins, and removes documented `ws://` remote gateway support, so the maintainable fix needs an explicit allowlist, proxying, or deprecation policy first.
