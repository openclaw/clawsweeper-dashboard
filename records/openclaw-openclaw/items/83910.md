## Summary

Keep open for maintainer product judgment: current main and v2026.5.18 intentionally default isolated `cron add` agentTurn jobs to `announce` in CLI code, docs, and tests, so the requested change is not a confirmed bug fix.

## What This Changes

The issue asks for `openclaw cron add` to omit `params.delivery` unless `--announce` or `--no-deliver` is explicitly passed.

## Best Possible Solution

Keep the documented default unless maintainers decide to change cron delivery semantics, then update CLI behavior, cron defaults, docs, tests, and upgrade guidance together.

## Reproduction Assessment

Yes for the observable behavior: current source and existing tests show isolated `cron add` without delivery flags submits `delivery.mode: "announce"`. The missing part is not reproduction, but agreement that the documented default should change.

## Solution Assessment

No as an automatic bug fix: the suggested fallback-to-undefined patch would contradict the current docs, tests, and core normalizer contract. A safer path is an explicit maintainer decision about the default and compatibility impact.
