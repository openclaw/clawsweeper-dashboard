## Summary

Keep open for maintainer product judgment: current main and the latest release intentionally default isolated cron agent-turn jobs to announce delivery in CLI code, core defaults, docs, and tests, so the requested one-line change is a compatibility-affecting behavior change rather than a confirmed bug fix.

## What This Changes

The issue asks to make isolated `cron add` jobs omit delivery unless `--announce` or `--no-deliver` is explicitly set.

## Best Possible Solution

Keep the documented default unless maintainers intentionally change cron delivery semantics, in which case update the CLI, core defaulting contract, docs, tests, and upgrade guidance together.

## Reproduction Assessment

Yes for the observable behavior: current source and existing tests show isolated `cron add` without delivery flags submits or creates `delivery.mode: "announce"`. The unresolved part is whether that documented default should change.

## Solution Assessment

No as an automatic bug fix: the suggested fallback-to-undefined patch would contradict the current docs, tests, and core normalizer/creation contract. The maintainable path is an explicit product decision with compatibility coverage.
