## Summary

Keep open for explicit maintainer product and compatibility judgment: current main and v2026.5.18 intentionally default isolated `cron add` agentTurn delivery to `announce` across CLI code, docs, tests, and runtime normalization, so this is not a safe automatic bug fix.

## What This Changes

Request changing `cron add` so isolated agentTurn jobs do not submit `delivery.mode: "announce"` unless `--announce` or `--deliver` is explicitly passed.

## Best Possible Solution

Keep the documented default unless maintainers decide to change cron delivery semantics; if they do, update CLI behavior, runtime defaults, docs, tests, and upgrade guidance together while coordinating with https://github.com/openclaw/openclaw/issues/56078.

## Reproduction Assessment

Yes for the observable behavior: current source, tests, docs, and the v2026.5.18 tag show no-flag isolated `cron add` submits announce delivery. The missing part is agreement that the documented default is wrong.

## Solution Assessment

No as an automatic bug fix: changing the fallback to undefined or none would contradict the current docs, tests, and runtime default contract. The safer path is an explicit maintainer compatibility decision, with the concrete no-channel failure handled through the canonical open bug if appropriate.
