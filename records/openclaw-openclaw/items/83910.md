## Summary

Keep open for maintainer product judgment: current main and the latest release intentionally default isolated cron agent-turn delivery to `announce` in CLI code, core normalization, docs, and tests, so the requested fallback-to-undefined change is not a conservative bug cleanup.

## What This Changes

The issue asks OpenClaw to stop defaulting isolated `cron add` agent-turn jobs to `delivery.mode: "announce"` when no delivery flag is passed.

## Best Possible Solution

Keep the documented announce-by-default behavior unless maintainers explicitly choose a default change, then update CLI behavior, core normalization, docs, tests, and upgrade guidance together.

## Reproduction Assessment

Yes for the observable behavior: current source, tests, docs, and the v2026.5.19 tag show isolated `cron add` without delivery flags uses `delivery.mode: "announce"`. The missing piece is maintainer agreement that the documented default should change.

## Solution Assessment

No as an automatic bug fix: changing the fallback to `undefined` would contradict the current CLI docs, regression tests, and core normalizer contract. The maintainable path is an explicit product decision with compatibility coverage.
