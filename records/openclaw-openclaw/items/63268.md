## What This Changes

This PR adds a doctor preflight warning that checks `/root`, `/home/node`, and `/home/openclaw` config paths and prints remediation guidance when doctor appears to target a different config directory.

## Best Possible Solution

Replace the heuristic with a resolver-backed doctor/service config-path comparison, remove destructive deletion advice, and cover root/HOME plus `OPENCLAW_CONFIG_PATH` and `OPENCLAW_STATE_DIR` cases before merge.

## Reproduction Assessment

Yes, source-backed: the paired report gives a concrete root-shell/systemd `HOME` mismatch flow, and current source shows doctor reads and writes through process-env config resolution without a service config-path comparison. I did not run the destructive live root/systemd flow in this read-only review.

## Solution Assessment

No. The warning direction is useful, but this implementation should compare the actual resolver-backed doctor path with verified service or running-gateway context instead of hard-coded home probes and deletion advice.

  - confidence: 0.88

  - confidence: 0.86
