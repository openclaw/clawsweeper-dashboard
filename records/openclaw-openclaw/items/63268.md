## Summary

Keep open: current main still lacks the doctor config-path drift guard, but this PR is not ready to merge because the patch compares hard-coded paths, can point users at stale configs or destructive deletion advice, and lacks inspectable real behavior proof.

## What This Changes

The PR adds a doctor config preflight warning that checks `/root/.openclaw/openclaw.json` against common service-user config files and prints remediation guidance.

## Best Possible Solution

Rework the fix to compare doctor’s actual resolver-backed config path with verified service or running-gateway config context, emit non-destructive guidance, and cover root/systemd drift with focused regression tests.

## Reproduction Assessment

Yes, source-backed. The linked report gives a concrete root-shell/systemd `HOME` mismatch flow, and current source shows doctor reads and writes through process-env config resolution without a doctor-time service config-path comparison.

## Solution Assessment

No. A warning or guard is the right narrow direction, but this implementation should use the existing resolver and verified service/running-gateway context instead of hard-coded home-directory probes and deletion advice.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Resolve the doctor target through the config resolver:** `src/commands/doctor-config-preflight.ts:144`
  - body: This passes `path.join(resolveHomeDir(), ".openclaw")` to the preflight even though doctor reads and writes through the config resolver. With `OPENCLAW_CONFIG_PATH` or `OPENCLAW_STATE_DIR` already pointing at the service-user config, the warning can still treat `/root/.openclaw` as the target and recommend an unnecessary override.
  - confidence: 0.92
- **[P2] Verify the active gateway config before recommending overrides:** `src/commands/doctor-config-preflight.ts:71`
  - body: The check treats the first existing `/home/node` or `/home/openclaw` config as the gateway config without proving the running service loaded it. A stale file at either path can make doctor recommend `OPENCLAW_CONFIG_PATH` for the wrong config, recreating the drift this warning is meant to prevent.
  - confidence: 0.88
- **[P2] Avoid recommending unconditional root config deletion:** `src/commands/doctor-config-preflight.ts:102`
  - body: The warning tells operators to remove `/root/.openclaw/openclaw.json` whenever doctor is using another config path, but it does not verify whether a legitimate root-managed gateway is using that file. Following the advice can delete a live config on multi-instance or root-managed hosts.
  - confidence: 0.84
