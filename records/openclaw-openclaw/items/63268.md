## Summary

Keep open: the linked bug is still valid on current main, and this PR is a plausible implementation path, but the branch still has correctness blockers and lacks after-fix real behavior proof from the contributor.

## What This Changes

The PR adds a doctor preflight warning that checks `/root`, `/home/node`, and `/home/openclaw` config locations and prints remediation guidance for config path drift.

## Best Possible Solution

Rework this into a resolver-backed doctor/service config audit that compares doctor’s actual write path with a verified managed-gateway config path, emits non-destructive guidance, and adds focused regression coverage.

## Reproduction Assessment

Yes. The linked issue provides concrete root-shell/systemd-HOME steps, and source inspection shows current main still lacks a doctor-time service config-path comparison; the PR’s false-positive path is also source-reproducible with `HOME=/root` plus `OPENCLAW_CONFIG_PATH` set correctly.

## Solution Assessment

No. A warning is the right narrow direction, but this implementation compares hard-coded file locations rather than doctor’s actual resolver path and a verified service/running-gateway config path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Resolve doctor target through the config resolver:** `src/commands/doctor-config-preflight.ts:144`
  - body: This preflight derives the doctor target from `resolveHomeDir()` even though doctor reads and writes through the config resolver. With `HOME=/root` and `OPENCLAW_CONFIG_PATH=/home/node/.openclaw/openclaw.json`, doctor is already targeting the service-user file, but this line still compares against `/root/.openclaw` and emits a false mismatch warning.
  - confidence: 0.92
- **[P2] Verify the active service config before recommending overrides:** `src/commands/doctor-config-preflight.ts:71-81`
  - body: The check treats the first existing `/home/node` or `/home/openclaw` config as the gateway config without proving the running service loaded it. A stale file at either path can make doctor recommend `OPENCLAW_CONFIG_PATH` for the wrong config, recreating the drift this warning is meant to prevent.
  - confidence: 0.88
- **[P2] Drop the unconditional root-config deletion advice:** `src/commands/doctor-config-preflight.ts:101-102`
  - body: This message tells operators to delete `/root/.openclaw/openclaw.json` whenever doctor is using another config path, but the patch does not verify that a legitimate root-managed gateway is not using that file. Following the advice can remove a live service config on multi-instance or root-managed hosts.
  - confidence: 0.84
