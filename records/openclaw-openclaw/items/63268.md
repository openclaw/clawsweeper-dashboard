## Summary

This PR should stay open: the underlying doctor/config-path drift problem is real, but the current patch still warns from hard-coded config locations rather than doctor’s actual resolved target and a verified active gateway/service config path, so it can misdirect operators.

## What This Changes

The PR adds a doctor preflight warning that checks `/root`, `/home/node`, and `/home/openclaw` config locations and prints remediation guidance when it sees possible config path drift.

## Best Possible Solution

Implement a resolver-backed doctor/service config audit that compares doctor’s actual write path with a verified managed-gateway config path, emits non-destructive guidance, and adds focused regression coverage.

## Reproduction Assessment

Yes. The linked issue gives concrete root-shell/systemd-HOME steps, and source inspection shows current main still lacks a doctor-time comparison against the active service config path.

## Solution Assessment

No. Warning before clobbering is the right narrow direction, but this patch should use the shared config resolver and a verified service/running-gateway config source instead of hard-coded home-directory probes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Resolve doctor target through the config resolver:** `src/commands/doctor-config-preflight.ts:144`
  - body: This preflight decides whether doctor is targeting root from the passed config directory, but doctor’s real read/write path is resolved by the config resolver. With `HOME=/root` and `OPENCLAW_CONFIG_PATH=/home/node/.openclaw/openclaw.json`, doctor can already be targeting the service-user file while this check still reports a root mismatch.
  - confidence: 0.92
- **[P2] Verify the active service config before recommending overrides:** `src/commands/doctor-config-preflight.ts:71-81`
  - body: The check treats the first existing `/home/node` or `/home/openclaw` config as the gateway config without proving the running service loaded it. A stale file at either path can make doctor recommend `OPENCLAW_CONFIG_PATH` for the wrong config, recreating the drift this warning is meant to prevent.
  - confidence: 0.88
- **[P2] Drop the unconditional root-config deletion advice:** `src/commands/doctor-config-preflight.ts:101-102`
  - body: This warning tells operators to delete `/root/.openclaw/openclaw.json` whenever doctor is using another config path, but the patch does not verify that a legitimate root-managed gateway is not using that file. Following the advice can remove a live service config on multi-instance or root-managed hosts.
  - confidence: 0.84
