## Summary

Keep this PR open: the auto-update fallback bug is still present on current main, but the branch is stale/conflicting, still changes update-command code that main already superseded, and lacks required real behavior proof.

## What This Changes

The PR reorders gateway update and auto-update fallback entrypoint candidates to prefer `dist/index.*` before `dist/entry.*` and adds focused Vitest coverage.

## Best Possible Solution

Wire `runAutoUpdateCommand` through the shared gateway entrypoint resolver, keep focused regression coverage, and leave update-command on the current main implementation.

## Reproduction Assessment

Yes, source-level. On current main, when auto-update runs under Node or Bun without `process.argv[1]` and both `dist/entry.js` and `dist/index.js` exist under the package root, `runAutoUpdateCommand` checks `entry.js` first.

## Solution Assessment

No. The direction is correct, but current main already has a shared index-first resolver, so the remaining auto-update fallback should reuse that helper instead of copying another hardcoded list.
