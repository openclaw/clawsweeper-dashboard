## Summary

This PR should stay open because it addresses a real remaining auto-update fallback bug, but the current branch is stale against main: update-command now already uses the shared index-first resolver, while the auto-update root fallback still needs the same resolver and a changelog entry.

## What This Changes

The PR reorders gateway update fallback entrypoint selection to prefer `dist/index.*` over legacy `dist/entry.*` in update-command and auto-update paths, with focused tests.

## Best Possible Solution

Auto-update root fallback should use the canonical gateway entrypoint resolver so update, install, doctor, and startup recovery share one index-first selection rule while preserving the existing concrete `process.argv[1]` reuse path.

## Reproduction Assessment

Yes. Source inspection on current main shows a Node/Bun auto-update process with no `process.argv[1]` and a package root containing both built entrypoints will enter the root fallback list and select `dist/entry.js` before `dist/index.js`.

## Solution Assessment

No. The direction is correct, but the branch should be narrowed to the remaining auto-update fallback, reuse the shared resolver, drop the superseded update-command hunk, and add the required changelog entry.
