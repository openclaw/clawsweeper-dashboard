## Summary

Keep open. Current main still has the auto-update root fallback choosing `entry.js` before `index.js`, but the update-command half of this branch has been superseded by the shared resolver and the PR still lacks real behavior proof.

## What This Changes

This PR reorders gateway update and auto-update fallback entrypoint candidates to prefer `dist/index.*` before `dist/entry.*` and adds focused unit tests.

## Best Possible Solution

Reuse the shared gateway entrypoint resolver from the auto-update root fallback, keep focused regression coverage, and leave the update-command side to the current main implementation.

## Reproduction Assessment

Yes, source-level. On current main, when auto-update runs under Node or Bun without `process.argv[1]` and both `dist/entry.js` and `dist/index.js` exist under the package root, `runAutoUpdateCommand` checks `entry.js` first.

## Solution Assessment

No. The direction is correct, but current main already has a shared index-first resolver, so the remaining auto-update fallback should call that helper instead of copying another hardcoded list.
