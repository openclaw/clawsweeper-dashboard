## Summary

Keep open: current main still has the auto-update root fallback ordering gap, but this PR needs adaptation because the update-command half is stale and contributor real-behavior proof is still missing.

## What This Changes

This PR reorders gateway update and auto-update entrypoint selection to prefer `dist/index.*` over `dist/entry.*` and adds focused unit tests.

## Best Possible Solution

Auto-update root fallback should reuse the shared gateway entrypoint resolver with focused regression coverage, while update-command stays on current main's centralized resolver.

## Reproduction Assessment

Yes, as a source-level reproduction: current main's no-argv auto-update root fallback checks `dist/entry.js` before `dist/index.js` when both files exist. I did not run a live LaunchAgent/systemd auto-update path, so this is source-reproducible rather than live reproduced.

## Solution Assessment

No. The intended index-first behavior is correct, but the branch should drop the obsolete update-command resolver hunk and route `update-startup.ts` through the shared `resolveGatewayInstallEntrypoint` helper.
