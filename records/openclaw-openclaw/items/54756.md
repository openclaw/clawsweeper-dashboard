## Summary

Keep this PR open: the reported auto-update fallback gap still exists on current main, but the update-command half of the branch is stale after the merged shared gateway entrypoint resolver and the PR still lacks contributor real-behavior proof.

## What This Changes

The branch reorders gateway update and auto-update entrypoint candidates to prefer `dist/index.*` over legacy `dist/entry.*` and adds focused unit tests.

## Best Possible Solution

Auto-update root fallback should reuse the canonical gateway entrypoint resolver with focused regression coverage; update-command should stay on current main's shared resolver.

## Reproduction Assessment

Yes. Source inspection on current main shows the auto-update root fallback still selects `dist/entry.js` before `dist/index.js` when no concrete `process.argv[1]` is available and both files exist.

## Solution Assessment

No. The direction is correct, but the branch should drop the obsolete update-command resolver hunk and have auto-update call the shared `resolveGatewayInstallEntrypoint` helper instead of copying another candidate list.
