## Summary

Keep open: the remaining auto-update fallback bug is still source-reproducible on current main, but the branch is stale against the shared resolver refactor and lacks real runtime proof, so it should not merge as-is.

## What This Changes

The PR reorders gateway update and auto-update entrypoint candidates to prefer `dist/index.*` before legacy `dist/entry.*` and adds focused unit tests.

## Best Possible Solution

Rebase into a narrow fix that makes auto-update reuse the shared gateway entrypoint resolver, drops the stale update-command hunk, keeps focused regression coverage, and includes redacted runtime proof before merge.

## Reproduction Assessment

Yes. On current main, when `process.argv[1]` is absent and both `dist/entry.js` and `dist/index.js` exist under the package root, `runAutoUpdateCommand` checks `entry.js` first; I did not run the live update flow in this read-only review.

## Solution Assessment

No. The direction is right, but current main already has a shared index-first resolver, so the remaining auto-update fallback should call that helper instead of copying another hardcoded list.
