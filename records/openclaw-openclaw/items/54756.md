## Summary

Keep open: current main still has the auto-update root fallback selecting `dist/entry.*` before `dist/index.*`, so this PR has a real remaining fix, but the branch is stale on the update-command side and needs real behavior proof before merge.

## What This Changes

The PR reorders gateway update and auto-update entrypoint fallback candidates to prefer `dist/index.*` over legacy `dist/entry.*`, with focused tests.

## Best Possible Solution

Auto-update root fallback should reuse the canonical gateway entrypoint resolver and keep focused regression coverage; the update-command side should stay on current main's shared resolver.

## Reproduction Assessment

Yes. Source inspection on current main shows that when auto-update runs under Node/Bun without `process.argv[1]` and both built entrypoints exist, `src/infra/update-startup.ts` selects `dist/entry.js` before `dist/index.js`.

## Solution Assessment

No. The direction is correct, but the branch should drop the stale update-command resolver hunk and call the shared `resolveGatewayInstallEntrypoint` helper from the auto-update fallback instead of copying another candidate list.
