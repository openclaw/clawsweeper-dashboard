## Repair Work Prompt

Fix the Gateway CLI handshake silent-failure path reported at https://github.com/openclaw/openclaw/issues/68944. When a `connect.challenge` arrives and `sendConnect()` or connect-frame assembly/signing/request emission throws synchronously, the client must surface an actionable `onConnectError`, close or otherwise settle the socket, and ensure CLI callers reject promptly instead of waiting for a generic timeout. Separate JSON parse failures from event/response handling so non-parse exceptions are not logged only as `gateway client parse error`. Preserve the existing Ed25519 device-auth flow; do not add HMAC fallback or weaken auth. Add focused regression coverage that forces a post-challenge connect assembly failure and proves no hang, plus coverage for the CLI call wrapper if it needs an `onConnectError` hook. Add a single user-facing `CHANGELOG.md` Fixes entry.
