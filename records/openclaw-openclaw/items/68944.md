## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/68944 by handling failures while assembling the Gateway client `connect` request after `connect.challenge`. Keep the scope narrow: wrap/report synchronous failures in `sendConnect()` such as device payload signing or public-key extraction, route them through `onConnectError`, close or otherwise settle the socket/client so CLI calls do not hang until the wrapper timeout, and avoid broad HMAC fallback, protocol changes, or startup migration behavior. Narrow the `handleMessage()` catch so non-JSON parse failures are not mislabeled as parse errors.
