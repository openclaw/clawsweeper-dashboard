## Summary

Keep open: current source still has a plausible MiniMax-only PDF resolution gap, and the docs promise extraction fallback for non-native providers while MiniMax chat models are registered as text-only.

## What This Changes

The issue asks for the PDF tool to allow MiniMax OAuth/API models to use extraction fallback instead of failing with an unknown-model error.

## Best Possible Solution

Repair PDF model selection so MiniMax M2.7 text chat models can be used explicitly and, where appropriate, automatically for non-native PDF extraction fallback, with MiniMax-VL-01 kept on the image-understanding path unless a tested PDF/VLM path is added.

## Reproduction Assessment

No live high-confidence repro was run. Source inspection shows a reproducible MiniMax-only resolution path where PDF auto-selection prefers MiniMax-VL-01 and model resolution happens before extraction fallback, but the exact explicit M2.7 failure still needs a focused regression test or live proof.

## Solution Assessment

Yes for the central direction, with a narrower implementation: make MiniMax text chat models reachable for PDF extraction fallback instead of treating MiniMax-VL-01 as a generic chat/PDF model. Do not broaden MiniMax VLM behavior unless the plugin runtime and tests prove that path.
