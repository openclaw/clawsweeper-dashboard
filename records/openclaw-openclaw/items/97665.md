## Summary

Keep this PR open, but it is not merge-ready: the storage-layer direction is useful, yet the current predicate still breaks a shipped upgrade case for same-origin custom Gateway URLs and the external-PR proof is only a storage test run.

## What This Changes

The PR changes Control UI localStorage load/save behavior to isolate gateway settings by base path, stop writing the shared legacy key, preserve cross-origin legacy custom Gateway migration, and add storage regression tests.

## Best Possible Solution

Use a compatibility-safe storage predicate that rejects stale sibling base-path records while preserving intentional same-origin custom Gateway endpoints, with focused regression tests and real WebSocket-target proof.

## Solution Assessment

No. The storage layer and page-scoped write are the right direction, but the current predicate is too broad and the sibling candidate shows a narrower compatibility shape for same-origin custom endpoints.
