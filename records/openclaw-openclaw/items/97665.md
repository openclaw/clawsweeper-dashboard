## Label Justifications

- `P2`: The PR addresses a normal-priority Control UI routing bug in multi-gateway same-origin deployments.
- `merge-risk: 🚨 compatibility`: The diff changes localStorage migration behavior for existing browser profiles with legacy settings records.
- `merge-risk: 🚨 message-delivery`: The loaded `gatewayUrl` is the WebSocket target, so a migration mistake can send user actions to the wrong Gateway.
- `merge-risk: 🚨 session-state`: The loaded `gatewayUrl` also scopes session selection, so wrong fallback behavior can attach the UI to the wrong session state.

## Summary

Keep open: current main and the latest stable release still have the legacy fallback bug, and this active PR is a viable candidate fix with sufficient proof; the remaining gate is maintainer acceptance of the localStorage upgrade boundary.

## What This Changes

The PR changes Control UI localStorage load/save logic and storage regression tests so same-origin sibling base paths keep independent gateway selections.

## Best Possible Solution

Land one canonical storage-boundary fix for the linked issue that preserves the covered custom endpoint shapes, rejects sibling basePath pollution, and keeps focused tests around both migration outcomes.

## Reproduction Assessment

Yes. On current main, seed the legacy unscoped settings key with Gateway A and load Gateway B without a scoped record; `loadSettings()` can return Gateway A's `gatewayUrl`, which `connectGateway()` uses as the WebSocket target. I did not run tests because this review is read-only.

## Solution Assessment

Yes, with a maintainer compatibility decision. The patch fixes the owner path that selects `gatewayUrl`, adds page-scoped selection for custom endpoints, and covers the sibling and compatibility cases without changing gateway protocol or connection code.
