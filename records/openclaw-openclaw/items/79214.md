## Summary

Keep open: the PR is a useful bug-fix shape, but the latest head still emits cron diagnostic lifecycle events without the `diagnostics.enabled` config gate, which breaks the documented master toggle and dispatch-path parity.

## What This Changes

The PR adds diagnostic lifecycle event emission and a regression test for isolated cron agent turns, plus a changelog entry.

## Best Possible Solution

Land the cron lifecycle events only after they honor `diagnostics.enabled` and test both default-enabled and disabled configurations.

## Reproduction Assessment

Yes. Source inspection shows current main lacks the cron lifecycle calls while the dispatch path emits them through a config gate; a diagnostic subscriber around an isolated cron turn is a high-confidence reproduction path.

## Solution Assessment

No, not yet. Emitting the lifecycle in the cron owner path is the right maintainable direction, but the PR must preserve the existing `diagnostics.enabled: false` contract before merge.

## Review Findings

Overall correctness: patch is incorrect
