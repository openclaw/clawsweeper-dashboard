## Summary

Keep open: the PR addresses a real source-reproducible cron diagnostics gap, but the added lifecycle events still bypass the existing `diagnostics.enabled` config gate used by dispatch and documented as the diagnostics master toggle.

## What This Changes

Adds `message.queued`, `session.state`, and `message.processed` diagnostic lifecycle emission around `runCronIsolatedAgentTurn`, plus a focused regression test and changelog entry.

## Best Possible Solution

Gate the cron lifecycle calls with `isDiagnosticsEnabled(params.cfg)`, keep the enabled-path coverage, and add disabled-config regression coverage before merge.

## Reproduction Assessment

Yes. Current main source lacks lifecycle emission in `runCronIsolatedAgentTurn`, while dispatch emits the same lifecycle through a config gate; a diagnostic subscriber around an isolated cron turn is the focused reproduction path.

## Solution Assessment

No, not yet. Emitting the lifecycle in the cron owner path is the right narrow fix, but the PR must preserve the documented `diagnostics.enabled: false` behavior.

## Review Findings

Overall confidence: 0.9

- **[P2] Guard cron lifecycle events behind diagnostics.enabled:** `src/cron/isolated-agent/run.ts:1120`
