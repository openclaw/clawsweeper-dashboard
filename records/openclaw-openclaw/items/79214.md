## What This Changes

The PR adds queued, session-state, and processed diagnostic lifecycle events around isolated cron agent turns, plus a focused regression test and changelog entry.

## Best Possible Solution

Land the cron lifecycle events after wrapping them in the same `isDiagnosticsEnabled(params.cfg)` gate as dispatch and adding enabled/disabled regression coverage.

## Solution Assessment
