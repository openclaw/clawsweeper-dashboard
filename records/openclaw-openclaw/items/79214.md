## Summary

Keep open: the PR fixes a real cron diagnostics gap and includes sufficient live proof, but the current head emits cron lifecycle events without honoring the existing `diagnostics.enabled: false` contract.

## What This Changes

The PR adds `message.queued`, `session.state`, and `message.processed` diagnostic lifecycle emission around `runCronIsolatedAgentTurn`, a focused regression test, and a changelog entry.

## Best Possible Solution

Gate the cron lifecycle events with `isDiagnosticsEnabled(params.cfg)`, keep the enabled-path coverage, and add disabled-config regression coverage before merge.

## Reproduction Assessment

Yes. Current main source lacks lifecycle emission in `runCronIsolatedAgentTurn`, while dispatch emits the same lifecycle through a config gate; a diagnostic subscriber around an isolated cron turn is the focused reproduction path.

## Solution Assessment

No, not yet. Emitting the lifecycle in the cron owner path is the right narrow fix, but the PR must preserve the documented `diagnostics.enabled: false` behavior.
