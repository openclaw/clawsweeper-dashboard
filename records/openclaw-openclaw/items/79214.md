## Summary

Keep open: this is a plausible bug fix for missing isolated-cron diagnostic lifecycle events, but the current PR head still emits those events without honoring the `diagnostics.enabled` master toggle. The prior Codex review already identified the same gate issue, and the latest diff still has the unconditional calls.

## What This Changes

The PR adds queued, session-state, and processed diagnostic lifecycle events around isolated-cron agent turns, plus a focused regression test and changelog entry.

## Best Possible Solution

Land the cron lifecycle events after wrapping them in the same `isDiagnosticsEnabled(params.cfg)` gate as dispatch and adding enabled/disabled regression coverage.

## Reproduction Assessment

Yes. Current main lacks diagnostic lifecycle calls in `runCronIsolatedAgentTurn`, while the dispatch path emits them through a config gate; source inspection gives a high-confidence reproduction path with a diagnostic subscriber around an isolated cron turn.

## Solution Assessment

No, not yet. Emitting the lifecycle in the cron owner path is the right shape, but the implementation must preserve the `diagnostics.enabled: false` contract before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

- **[P2] Guard cron lifecycle events behind diagnostics.enabled:** `src/cron/isolated-agent/run.ts:1111-1125`
