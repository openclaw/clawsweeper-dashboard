## Summary

Close as external plugin/ClawHub work: current main has no `memory-daymaker` source, OpenClaw’s vision keeps optional plugins outside core, and the existing cron hook/update surface is enough for the plugin to preserve disabled or customized cron fields.

## What This Changes

The issue asks for `memory-daymaker` managed daily cron reconciliation to stop re-enabling disabled jobs and overwriting user-customized description and payload fields on gateway restart.

## Best Possible Solution

Fix `memory-daymaker` in its own plugin package by making its reconcile omit `enabled`, `description`, and `payload.message` when the existing managed job is user-disabled or customized, unless a plugin config change explicitly requires rewriting them.

## Reproduction Assessment

No high-confidence current-main reproduction exists in this repository because `memory-daymaker` is absent from the checkout. The production report is credible external evidence, and current source inspection shows core cron patches can preserve omitted fields.

## Solution Assessment

Yes for the OpenClaw core boundary: the maintainable fix is plugin-local unless a plugin author can show `ctx.getCron` or `cron.update` cannot express the preservation behavior. A shared helper could be proposed separately, but it is not required for this report.
