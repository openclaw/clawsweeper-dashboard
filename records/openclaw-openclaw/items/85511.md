## What This Changes

The PR adds a persisted collapse/expand control for the web UI sidebar’s Recent sessions list, with English strings, settings storage changes, render logic, and CSS styling.

## Best Possible Solution

Keep the PR open and amend it so the new setting is fully persisted, the generated i18n outputs are synced, and the refresh-persistence path has focused proof before merge.

## Reproduction Assessment

Yes for the PR defect by source inspection: the click path stores recentSessionsCollapsed in state, but the settings load and persist paths omit that field. I did not run the PR branch in a browser during this read-only review.

## Solution Assessment

No as-is; the UI pattern matches the existing nav section controls, but the setting needs to be part of the storage contract and the i18n generated outputs need to be synced before this is the maintainable fix.

  - confidence: 0.86
