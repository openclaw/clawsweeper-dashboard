## Summary

Keep open: the error-mapping direction is useful, but the diff gives operators a legacy runtime key that current policy ignores, and the external PR still lacks real gateway behavior proof.

## What This Changes

The PR maps unregistered agent-harness errors to OpenAI-compatible 503 responses for chat completions and adds mocked non-streaming/SSE regression tests.

## Best Possible Solution

Update the error text and tests to point at supported provider/model-scoped runtime policy, keep the 503 mapping if maintainers accept request-time handling, and add redacted real gateway proof before merge.

## Reproduction Assessment

Yes. Source inspection shows `selectAgentHarness` throws the unregistered-harness error and current `openai-http.ts` falls through to generic 500 handling for that unmapped exception.

## Solution Assessment

No. HTTP-boundary mapping is a narrow maintainable direction, but this patch points users at an ignored legacy key; the safer version uses current provider/model runtime policy guidance and proves the real gateway behavior.
