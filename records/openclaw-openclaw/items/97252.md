## Summary

Keep open: this is a focused candidate fix for https://github.com/openclaw/openclaw/issues/97048. Current main and v2026.6.10 still reject generated catalog rows with `audio` or `video`, and this branch covers the previously flagged type surfaces with sufficient terminal proof; I found no blocking patch defect, but the additive public `Model.input` contract is compatibility-sensitive for maintainer acceptance.

## What This Changes

The PR widens model input validation and types to accept `audio`/`video` in generated catalogs, updates OpenAI/list-row handling, and adds registry/model-list regression tests.

## Best Possible Solution

Land the additive schema/type alignment once maintainers accept `audio`/`video` as provider model input metadata, keeping generated catalogs and model-list output consistent.

## Reproduction Assessment

Yes. Source inspection shows generated catalog rows pass through `validateModelsConfig`, whose current main and v2026.6.10 schema only accepts `text` and `image`; the PR body also includes before/after terminal output for the production registry loader path.

## Solution Assessment

Yes. Widening the registry and related model metadata types is the narrowest maintainable fix because clamping would lose provider-reported modality metadata and leave config/catalog contracts split.
