## Summary

Keep open: current main and v2026.6.10 still buffer Anthropic OAuth token responses with an unbounded `response.text()`, while this PR is a focused security hardening that uses the existing bounded reader and adds regression coverage. No blocking patch findings were found.

## What This Changes

The PR replaces Anthropic OAuth token endpoint `response.text()` reads with `readResponseWithLimit` at a 16 MiB cap and adds an oversized refresh-response regression test.

## Best Possible Solution

Merge the focused bounded-read change after exact-head CI completes, keeping the existing helper-based pattern and oversized-response regression test.

## Reproduction Assessment

Yes. Current main and v2026.6.10 visibly call `response.text()` in the Anthropic OAuth token helper, and the PR body shows a terminal proof using an over-limit response stream; I did not rerun tests because this review was read-only.

## Solution Assessment

Yes. Reusing `readResponseWithLimit` at the shared Anthropic OAuth `postJson` boundary is the narrowest maintainable fix because it covers both exchange and refresh while preserving existing parsing and error-shaping behavior.
