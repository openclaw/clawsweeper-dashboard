## Summary

Keep open: current main does not have the exact `agents add` startup override, the PR has focused source/test coverage plus terminal proof, and the remaining question is maintainer acceptance of the compatibility-sensitive plugin preload boundary rather than a concrete patch defect.

## What This Changes

This PR adds an argv-sensitive `agents add` startup policy so no-bind config-only add invocations skip plugin preload while interactive and `--bind` add paths still preload plugins, with focused regression coverage.

## Best Possible Solution

Land the narrow exact startup policy if maintainers accept that plugin import side effects are not a supported contract for no-bind config-only `agents add`; otherwise centralize the flag boundary before merge.

## Reproduction Assessment

Yes. Current-main source shows `agents add` inherits the parent `agents` plugin preload before the action can choose config-only mode, and the PR body provides isolated terminal proof of the affected path after the patch.

## Solution Assessment

Yes, with maintainer acceptance of the compatibility tradeoff. An exact command-catalog startup policy is the narrow place to decide preload before command action execution, and the tests cover no-bind, interactive, `--bind`, equals-form flags, and `--` handling.
