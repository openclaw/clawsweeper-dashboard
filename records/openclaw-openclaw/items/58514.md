## Summary

Keep open: current main and the latest release still contain Google Chat Space/session routing paths that can drop Space messages before a `googlechat:group` session is created, and the linked fix PR remains open rather than merged.

## What This Changes

Google Chat Space/group webhook messages are reported as returning HTTP 200 without creating `googlechat:group` sessions or agent replies, while DMs continue to work.

## Best Possible Solution

Keep this issue as the canonical Google Chat Space ingress/session bug until a merged fix proves that Space @mentions create group sessions and preserve the necessary routing metadata.

Yes at source level: current main still has Google Chat Space handling that depends on legacy `space.type` and mention recognition that can drop Space messages before session creation. I did not run a live Google Chat Space repro in this read-only pass.
