## Summary

Keep open: current main and the latest release still classify Google Chat conversations from the legacy `space.type` field only, while the direct closing fix remains open and unmerged. The issue should stay open until that PR or a replacement lands with proof that Space messages create/process `googlechat:group` sessions without regressing DMs.

## What This Changes

The issue reports that Google Chat Space messages return HTTP 200 but are not processed into group sessions while DMs continue to work.

## Best Possible Solution

Land or replace the linked Google Chat classifier fix with focused tests and live proof that modern Google Chat payloads route DMs as direct sessions and Spaces as group sessions.

## Reproduction Assessment
