## Summary

The reported macOS 15.7.7 behavior matches current and v2026.5.28 source: Voice Wake is intentionally feature-gated to macOS 26+, so this is an unsupported-environment support item rather than a crash or permission regression.

## What This Changes

The issue reports that macOS Voice Wake remains disabled in the Voice settings panel even after microphone and speech-recognition permissions are granted.

## Best Possible Solution

Keep the macOS 26 Voice Wake gate as the current supported behavior; a request to support macOS 15 would need a separate maintainer product decision for an alternate speech backend.

## Reproduction Assessment

Yes from source inspection: on macOS versions below the `voiceWakeSupported` threshold, the settings UI disables the toggle and runtime entry points refuse to start Voice Wake. I did not run a macOS 15 DMG, but the reported disabled state follows directly from current and release-tag source.

## Solution Assessment

Yes for cleanup: current shipped behavior explains why macOS 15.7.7 cannot enable Voice Wake. Supporting that OS would require a new product decision or alternate speech backend rather than a narrow bug fix.
