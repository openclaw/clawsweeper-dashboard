## Summary

Keep open: this is a focused bug-fix PR, but the patch checks the legacy `supportsXHighThinking` hook while current OpenAI/OpenAI Codex thinking support is exposed through `resolveThinkingProfile`, so the reported path can still remain off-only.

## What This Changes

The PR changes `resolveThinkingProfile` to bypass catalog `reasoning: false` only when `resolveProviderXHighThinking` returns true, and adds two regression tests for xhigh override and opt-out preservation.

## Best Possible Solution

Resolve `reasoning: false` conflicts through the provider thinking profile used by OpenAI/OpenAI Codex, with regression coverage for `openai-codex/gpt-5.5` and a no-policy opt-out control.

## Reproduction Assessment

Yes, source-reproducible: current main short-circuits on `context.reasoning === false`, while the OpenAI Codex provider profile includes xhigh for `gpt-5.5`. The linked issue also gives a concrete config and visible rejection path.

## Solution Assessment

No. The patch should consult or normalize the provider thinking profile, because the current OpenAI/OpenAI Codex policy is profile-based rather than `supportsXHighThinking`-based.

## Review Findings

Overall correctness: patch is incorrect

- **[P1] Use the provider profile for the override:** `src/auto-reply/thinking.ts:180-184`
  - body: The real OpenAI/OpenAI Codex provider does not implement `supportsXHighThinking`; it advertises `gpt-5.5` xhigh support through `resolveThinkingProfile`. Please base the override on the provider profile, or make the helper reflect profile-derived xhigh support.
