## Summary

Keep open: current main still has a source-reproducible path for the clean-but-mismatched memory status. The reindex path compares provider/model/providerKey metadata, but status-only managers can still treat any existing metadata as clean while retaining stale vector dimensions.

## What This Changes

The issue reports that switching memory search from OpenAI embeddings to Ollama can leave the SQLite memory index reporting the new provider/model while retaining stale OpenAI vector dimensions and clean status.

## Best Possible Solution

Make status, deep probes, and live reload use the same provider/model/providerKey/scope validation as the full-reindex path, marking mismatched indexes dirty or rebuilding safely before reporting them clean.

## Reproduction Assessment

No end-to-end live reproduction was run. Source inspection gives a high-confidence current-main path for the visible inconsistency: status-only managers treat existing metadata as clean while deep status can initialize the new provider and keep stale vector dimensions.

## Solution Assessment

Yes, the narrow maintainable direction is to reuse or share the existing reindex metadata comparison for status and reload validation. Closing as fixed would be premature because that comparison is not applied to the reported clean status path.
