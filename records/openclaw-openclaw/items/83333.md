## Summary

Keep open: current main still has a source-reproducible path where builtin memory status can combine the newly requested provider with stale index vector dimensions and report the index clean, while the full reindex path performs stronger provider/model/providerKey validation that status does not reuse.

## What This Changes

The issue asks OpenClaw to detect and rebuild or invalidate a builtin memory index after switching `agents.defaults.memorySearch` from OpenAI embeddings to Ollama embeddings instead of reporting a stale mixed-vector index as clean.

## Best Possible Solution

Make status, deep probes, and live reload validate stored index provider/model/providerKey/scope/vector metadata against the active memorySearch settings and mark mismatches dirty or invalid before reporting the index clean.

## Reproduction Assessment

Yes at source level, not yet as a live Docker/Ollama repro. A focused test can seed metadata with old OpenAI vector dimensions, construct a status/deep manager with Ollama settings, and observe current main reporting stale dimensions with `dirty: false`.

## Solution Assessment

Yes, the narrow maintainable fix is to reuse or extract the existing full-reindex metadata comparison for status and reload validation. Adding a new config knob or only changing search ranking would not address the clean-but-invalid status state.
