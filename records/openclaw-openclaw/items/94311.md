## What This Changes

The PR adds lossless-claw config normalization that synthesizes `llm.allowModelOverride` and `llm.allowedModels` from `plugins.entries.lossless-claw.config.summaryModel`, plus regression tests for that normalization.

## Best Possible Solution

Land one canonical fix for the linked LCM bug that preserves explicit `llm.allowModelOverride` and `llm.allowedModels` as the only model-override authorization source, with startup and reload coverage for context-engine LLM calls.

## Reproduction Assessment

No live current-main LCM reproduction was run in this read-only review. The patch defect is source-reproducible because PR-head normalization writes trusted `llm` policy from `summaryModel`, including over an explicit false `llm.allowModelOverride` setting.

## Solution Assessment

No. The reported bug is best solved by preserving the explicit `llm` trust gate and fixing startup/config lifecycle propagation, not by making `summaryModel` a second authorization source.
