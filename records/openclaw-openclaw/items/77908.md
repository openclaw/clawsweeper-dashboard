## Summary

The reported behavior matches the shipped group/channel visible-reply policy: normal assistant final replies in Telegram topics and Discord channels stay private by default unless the agent uses the message tool or `messages.groupChat.visibleReplies` is set to `automatic`.

## What This Changes

The request is for non-main agents bound to Telegram topics or Discord channels to post normal assistant replies back to the originating chat.

## Best Possible Solution

Keep the shipped visible-reply policy and use `message(action=send)` or `messages.groupChat.visibleReplies: "automatic"` for setups that want normal final replies posted to rooms.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction of the reported symptom under the default group/channel visible-reply policy: Telegram topic and Discord channel turns suppress normal final delivery while native commands still reply visibly.

## Solution Assessment

Yes. The narrow maintainable solution is configuration or message-tool delivery; changing the default auto-posting policy for non-main agents would be a product decision, not a bug fix.
