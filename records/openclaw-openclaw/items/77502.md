## Summary

Keep open: current main does need a better externalized-plugin install hint, but this PR points Brave users at the wrong package and changes diagnostic precedence so blocked externalized plugin paths can be reported as not installed.

## What This Changes

The PR adds a config-validation map for externalized plugin IDs, changes missing-plugin warnings for plugin entries and allowlist refs to an install command, updates tests, and adds a changelog entry.

## Best Possible Solution

Keep the install-hint direction, but use canonical install specs from existing package/catalog metadata where possible and preserve blocked-plugin diagnostics before externalized-not-installed hints.

## Reproduction Assessment

Yes. Current main's `pushMissingPluginIssue` path and the existing discord validation test show the generic stale-entry warning for a missing externalized plugin; tests were not run because `pnpm` is unavailable.

## Solution Assessment

No. The direction is narrow and appropriate, but the patch should correct or derive package specs and keep blocked-plugin diagnostics ahead of the new install hint.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use the canonical Brave install spec:** `src/config/validation.ts:51`
  - body: The new externalized-plugin map emits `openclaw plugins install @openclaw/brave` for the `brave` plugin, but the repository package metadata and generated plugin inventory name the install spec `@openclaw/brave-plugin`. Users following this hint will install the wrong package, so this entry should be corrected or sourced from the existing install metadata.
  - confidence: 0.91
- **[P2] Keep blocked-plugin diagnostics ahead of install hints:** `src/config/validation.ts:1521-1527`
  - body: The new externalized lookup runs before `findBlockedPluginDiagnostic`, so an externalized plugin package that is present but blocked by path-safety checks would now get a `plugin not installed` install command instead of the existing `plugin present but blocked` remediation. Please preserve the blocked diagnostic precedence and cover that case in the test.
  - confidence: 0.88
