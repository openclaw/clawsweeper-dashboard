## Summary

Keep open: current main and the latest release still have the hosted installer shell logic that can emit `--before` from npm's computed `before` value while a raw npmrc `min-release-age` policy remains present. The issue is already paired with an open closing PR, so the right cleanup is to keep this issue open until that PR lands or is replaced.

## What This Changes

The issue requests that hosted installer npm calls bypass npm freshness quarantine without emitting a `--before` flag when a raw npmrc `min-release-age` policy is present.

## Best Possible Solution

Land or replace the linked installer fix so shell and PowerShell installers mirror the TypeScript helper's raw npmrc freshness-mode detection, then close this issue after the fix is on main and propagated to the hosted installer.

## Reproduction Assessment

Yes. A scratch npmrc with `min-release-age=7` against current `scripts/install.sh` makes the installer select `--before=<now>` and npm 11.12.1 rejects the mutually exclusive config combination before package resolution.

## Solution Assessment

Yes. The maintainable fix is to port the existing raw npmrc freshness-mode detection from `src/infra/npm-install-env.ts` into the hosted installer scripts and cover that shell/PowerShell path with regression tests.
