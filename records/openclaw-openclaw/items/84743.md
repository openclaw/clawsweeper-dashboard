## Summary

Keep open: current `main` and `v2026.5.19` still have the installer shell branch that can emit `--before=<now>` for a raw npmrc `min-release-age` policy, and npm 11.12.1 reproduces the reported mutual-exclusion failure.

## What This Changes

The issue reports that the hosted installer still fails under npm `min-release-age` because the shell freshness bypass can emit a conflicting `--before` argument.

## Best Possible Solution

Port or mirror the TypeScript helper's raw npmrc freshness-mode detection into the hosted installer scripts, with script-level coverage for raw `min-release-age` and explicit `before` policies.

## Reproduction Assessment

Yes, at source and dependency-contract level: current installer code can select `--before` from npm's derived `before` while the raw npmrc still contains `min-release-age`, and npm 11.12.1 rejects that combination. I did not run the full global install path because it would mutate the machine.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse or mirror the existing raw npmrc detection semantics in the shell installers rather than adding another fallback stack or changing user-facing config.
