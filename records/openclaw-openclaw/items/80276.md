## Summary

Keep open: the feature request is real, but this PR is not mergeable because the setup hook inherits the full process environment, only covers the metadata installer path, and does not prove a real install/update run executing the hook.

## What This Changes

The PR adds `metadata.openclaw.setup` parsing, a `SkillSetupHook` type, post-install setup-script execution after the existing metadata installer succeeds, and integration tests for that path.

## Best Possible Solution

Define one canonical setup-hook contract that isolates env, composes with metadata and ClawHub install/update flows, preserves clear failure semantics, documents the public SKILL.md field, and has redacted real install/update proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the runner inherits `process.env` and the hook is wired only into the metadata installer path. For the linked request itself, bug reproduction is not applicable because this adds a new capability.

## Solution Assessment

No. The direction may be viable, but this implementation is not the best merge candidate until it enforces the declared env contract, covers ClawHub install/update, and settles the security/product contract.
