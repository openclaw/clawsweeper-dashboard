## Summary

Keep this PR open but block merge: it is a plausible implementation attempt for the setup-hook feature, but the diff leaks ambient gateway environment variables to skill scripts, misses the ClawHub install/update flows named by the request, and only supplies test output rather than real behavior proof.

## What This Changes

The PR adds a `metadata.openclaw.setup` field, parses a safe script path, runs the declared script after metadata-installer success, and adds setup-hook tests.

## Best Possible Solution

Define one constrained setup-hook contract that uses a truly isolated allowlisted environment, composes with ClawHub install/update and metadata installers, preserves rollback semantics, documents the new SKILL.md field, and includes real install/update proof.

## Reproduction Assessment

Yes for the PR defects: source inspection shows the proposed runner inherits `process.env` through `runCommandWithTimeout`, and the hook is wired only into the metadata-installer path. As a feature request, current-main bug reproduction is otherwise not applicable.

## Solution Assessment

No. The direction may be viable, but this implementation is not the best merge candidate until it enforces the declared env contract, covers the actual install/update flows, and resolves the feature's security model.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Pass only the declared setup env to scripts:** `src/agents/skills-install.ts:494`
  - body: `runCommandWithTimeout` merges the supplied env over `process.env`, so this still gives every `setup.script` access to unrelated gateway secrets. That contradicts the promised contract of only `SKILL_DIR`, `OPENCLAW_HOOK_KIND`, and allowlisted `requires.env` values.
  - confidence: 0.97
- **[P2] Run setup from the ClawHub install and update flows:** `src/agents/skills-install.ts:628`
  - body: This hook only runs when the gateway metadata installer executes a declared install action. Native `openclaw skills install <slug>` and `skills.update` go through the ClawHub/archive paths, so the flow named by the PR and linked request will not run the setup script.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The diff introduces skill-supplied install-time execution and currently leaks the full gateway process environment to that code.

Concerns:

- **[high] Setup scripts inherit the full host environment:** `src/agents/skills-install.ts:494`
  - body: The PR passes `hookEnv` to `runCommandWithTimeout`, but that helper merges command env over `process.env`; a skill-authored script can therefore read ambient gateway secrets not listed in `requires.env`.
  - confidence: 0.97
