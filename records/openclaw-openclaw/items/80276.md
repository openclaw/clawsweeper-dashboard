## Summary

Keep open: the feature is useful, but this PR still leaks ambient process environment to skill-supplied setup scripts and only wires the metadata installer, leaving native ClawHub install/update paths uncovered. The supplied proof also shows frontmatter parsing only, not a real install/update run executing the hook.

## What This Changes

The PR adds a `metadata.openclaw.setup` frontmatter field, parses a bundle-relative script path, runs it after metadata installer success, and adds focused `skills-install` tests.

## Best Possible Solution

Define one constrained setup-hook contract that isolates the hook environment, composes with metadata and ClawHub install/update flows, preserves rollback semantics, documents the SKILL.md field, and includes redacted real install/update proof.

## Reproduction Assessment

Yes for the PR defects: source inspection shows the proposed runner inherits `process.env` through `runCommandWithTimeout`, and the hook is wired only into the metadata-installer path. For the linked feature request itself, reproduction is not applicable because it adds a new capability.

## Solution Assessment

No. The direction may be viable, but this implementation is not the best merge candidate until it enforces the declared env contract, covers the actual install/update flows, and settles the security/product contract.
