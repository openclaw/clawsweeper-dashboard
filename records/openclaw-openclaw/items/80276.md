## Summary

Keep this PR open for maintainer handling, but do not merge it as-is: source review confirms the patch leaks the ambient process environment to skill-authored setup scripts and only wires the hook into the metadata installer path, while the linked feature request also covers ClawHub install/update behavior. A competing open implementation targets the same feature, but it is not clearly canonical enough to close this PR as superseded without maintainer judgment.

## What This Changes

The PR adds a SKILL.md `metadata.openclaw.setup` field, parses a bundle-relative setup script, runs it after successful metadata-based skill installs, and adds setup-hook tests.

## Best Possible Solution

Define one constrained setup-hook lifecycle that isolates env correctly, composes with ClawHub archive install/update and metadata installers, preserves rollback semantics, documents the SKILL.md contract, and includes real install/update proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the proposed runner passes setup env into a helper that defaults to `process.env`, and native install/update paths go through ClawHub/archive code not touched by the PR. As a feature request against current main, bug reproduction is otherwise not applicable.

## Solution Assessment

No. The setup-hook direction may be viable, but this implementation is not the best merge candidate until it enforces an isolated env contract, covers ClawHub install/update, settles the security model, documents the field, and includes real behavior proof.
