## Summary

Keep this PR open: the patch is not mergeable because the setup hook inherits the ambient process environment and only runs from the metadata installer, not the ClawHub install/update flows named by the linked feature request.

## What This Changes

Adds `metadata.openclaw.setup` parsing, a `SkillSetupHook` type, post-install setup script execution for the metadata installer path, and tests for that hook.

## Best Possible Solution

Land one canonical setup-hook contract that isolates env, covers metadata and ClawHub install/update with rollback, documents SKILL.md semantics, and includes redacted real install/update proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the setup runner inherits `process.env` through `runCommandWithTimeout`, and the hook is wired only into the metadata installer path. For the linked feature request itself, reproduction is not applicable because it adds a new capability.

## Solution Assessment

No. The direction may be viable, but this implementation is not the best merge candidate until it enforces the declared env contract, covers the actual install/update flows, and settles the security/product contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Pass an isolated env to setup scripts:** `src/agents/skills-install.ts:494`
  - body: `runCommandSafely` ultimately calls `runCommandWithTimeout`, whose env handling merges the supplied hook env over `process.env`. A skill-authored setup script can therefore read unrelated gateway secrets, contradicting the advertised minimal env contract.
  - confidence: 0.97
- **[P2] Wire setup into ClawHub install and update:** `src/agents/skills-install.ts:628`
  - body: The hook only runs from `installSkill` after metadata installers. CLI `skills install` and `skills update` use the ClawHub/archive paths, so the primary flows named by the feature request never execute `setup.script`.
  - confidence: 0.9

## Security Review

Status: needs_attention

  - body: The PR passes `hookEnv` as `env`, but the current command runner merges that env over `process.env`; a skill-authored script can read unrelated gateway secrets not listed in `requires.env`.
