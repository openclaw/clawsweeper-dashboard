## Summary

Keep open: this PR targets a real feature gap, but the current patch has blocking issues. It leaks ambient process environment into skill setup scripts and does not wire the hook into the public ClawHub install/update paths described by the PR and linked feature request.

## What This Changes

The branch adds `metadata.openclaw.setup` parsing and runs a bundle-relative setup script after `installSkill` succeeds, with focused tests for that installer path.

## Best Possible Solution

Land one approved setup-hook design that uses a scrubbed hook environment, runs from the public ClawHub install/update flow with clear install/update semantics, documents the trust contract, and includes real CLI proof.

## Reproduction Assessment

Yes for the review findings by source inspection: the PR calls the hook through `runCommandSafely`, and current command execution merges supplied env with `process.env`. The broader request is a feature, not a bug reproduction.

## Solution Assessment

No. A setup hook may be the right product direction, but this implementation needs a truly scrubbed env runner and integration with ClawHub install/update before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Scrub ambient env before running setup hooks:** `src/agents/skills-install.ts:494`
  - body: The hook env built here is passed to `runCommandSafely`, but `runCommandWithTimeout` resolves env by merging `options.env` over `process.env`. That exposes every ambient gateway/CLI variable, including provider tokens and other secrets not listed in `requires.env`, to any skill setup script. The hook runner needs a truly scrubbed base env or an exec option that does not merge `process.env`.
  - confidence: 0.94
- **[P2] Run setup hooks from ClawHub install and update:** `src/agents/skills-install.ts:628`
  - body: This only invokes the hook after `installSkill`'s per-skill dependency installer. The public `openclaw skills install <slug>` and `skills.update` paths go through `installSkillFromClawHub` / `updateSkillsFromClawHub`, so a ClawHub skill declaring `setup.script` is copied or updated without running the hook this PR is meant to add. Wire the archive/ClawHub install and update flow to the same hook with the correct hook kind.
  - confidence: 0.88
