## Summary

This PR should stay open for maintainer review: it adds a new install-time skill execution surface, but the branch still leaks the ambient process environment to skill-authored setup scripts and does not cover the ClawHub install/update paths named by the linked request.

## What This Changes

The PR adds `metadata.openclaw.setup` frontmatter parsing, a `SkillSetupHook` type, setup-script execution after the metadata installer succeeds, and tests for that metadata-install path.

## Best Possible Solution

Define one constrained setup-hook contract that isolates hook environment, composes with metadata and ClawHub install/update flows, preserves rollback semantics, documents the SKILL.md field, and includes real install/update proof.

## Reproduction Assessment

Yes for the PR defects: source inspection shows the proposed runner inherits `process.env`, and the hook is wired only into the metadata installer path. As a feature request, current-main bug reproduction is otherwise not applicable.

## Solution Assessment

No. The direction may be viable, but this implementation is not the best merge candidate until it enforces the declared env contract, covers the actual install/update flows, and settles the security/product contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Pass only the declared setup env to scripts:** `src/agents/skills-install.ts:494`
  - body: `runCommandWithTimeout` merges the supplied env over `process.env`, so this still gives every `setup.script` access to unrelated gateway secrets. That contradicts the promised contract of only `SKILL_DIR`, `OPENCLAW_HOOK_KIND`, and allowlisted `requires.env` values.
  - confidence: 0.97
- **[P2] Run setup from the ClawHub install and update flows:** `src/agents/skills-install.ts:628`
  - body: This hook only runs when the metadata installer executes a declared install action. Native `openclaw skills install <slug>` and `skills.update` use the ClawHub/archive paths, so the flow named by the PR and linked request will not run the setup script.
  - confidence: 0.9

## Security Review
