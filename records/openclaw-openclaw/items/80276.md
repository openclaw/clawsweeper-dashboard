## Summary

Keep open: the PR is a plausible feature direction, but this patch is not merge-ready because setup scripts inherit ambient process secrets and the native ClawHub install/update path named by the request is not wired.

## What This Changes

The branch adds `metadata.openclaw.setup` parsing and runs a bundle-relative setup script after the metadata-driven skill install path succeeds.

## Best Possible Solution

Consolidate on one setup-hook implementation that isolates the hook environment, covers metadata and ClawHub install/update flows with rollback semantics, documents the public SKILL.md contract, and includes redacted real install/update proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the proposed runner inherits `process.env`, and the hook is wired only into the metadata installer path. For the linked feature request itself, current-main bug reproduction is not applicable because this is a new lifecycle feature.

## Solution Assessment

No. The direction may be viable, but this implementation is not the best merge candidate until it enforces the declared env contract, covers ClawHub install/update, and settles the setup-hook security/product contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Pass only the declared setup env to scripts:** `src/agents/skills-install.ts:494`
  - body: `runCommandWithTimeout` merges the supplied `env` over `process.env`, so this still gives every `setup.script` access to unrelated gateway secrets. That contradicts the promised contract of only `SKILL_DIR`, `OPENCLAW_HOOK_KIND`, and allowlisted `requires.env` values.
  - confidence: 0.97
- **[P2] Run setup from the ClawHub install and update flows:** `src/agents/skills-install.ts:628`
  - body: This hook only runs when the metadata installer executes a declared install action. Native `openclaw skills install <slug>` and `skills update` go through the ClawHub/archive paths, so the user-facing flow named by the PR and linked request will not run the setup script.
  - confidence: 0.91

## Security Review
