## Summary

Keep open: the PR targets a real new skill lifecycle feature, but this branch still leaks ambient process environment to setup scripts, misses the native ClawHub install/update paths, and needs stronger real install/update proof plus maintainer security/product review.

## What This Changes

The PR adds `metadata.openclaw.setup` parsing, a setup hook runner after metadata-based skill installs, and tests for hook success, failure, env forwarding, and path handling.

## Best Possible Solution

Consolidate on one canonical setup-hook implementation that isolates env, covers metadata and ClawHub install/update flows, documents the public contract, and includes redacted real install/update proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the setup runner still goes through an env-merging command helper, and the hook is wired only into `installSkill`. For the linked feature request itself, current-main bug reproduction is not applicable because this adds a new capability.

## Solution Assessment

No. The direction may be viable, but this branch is not the best merge candidate until env isolation, ClawHub install/update coverage, docs, trust-model review, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Pass only the declared setup env to scripts:** `src/agents/skills-install.ts:494`
  - body: `runCommandSafely` ultimately calls `runCommandWithTimeout`, whose env resolver merges the supplied env over `process.env`. That means every skill-authored `setup.script` still receives ambient gateway secrets, contradicting the promised minimal env contract.
  - confidence: 0.97
- **[P2] Run setup from ClawHub install and update paths:** `src/agents/skills-install.ts:628`
  - body: This hook runs only after the metadata `installSkill` flow succeeds. The native CLI and gateway `skills.install`/`skills.update` flows go through ClawHub/archive helpers, so the feature named by the PR will not run setup scripts for those installs or updates.
  - confidence: 0.91
- **[P3] Document the setup hook contract before exposing it:** `src/agents/skills/types.ts:37-38`
  - body: The PR adds a new author-facing `metadata.openclaw.setup` field, but the skill docs do not describe timing, env forwarding, failure behavior, rollback semantics, or the security model. Skill authors need that contract before relying on this metadata.
  - confidence: 0.78
