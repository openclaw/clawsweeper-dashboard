## Repair Work Prompt

Fix the unsupported Gateway service platform status crash tracked by https://github.com/openclaw/openclaw/issues/25621. On unsupported `process.platform` values such as `openbsd` or `aix`, `openclaw status` and `openclaw gateway status` must render degraded read-only diagnostics instead of throwing `Gateway service install not supported on ...`. Use the narrowest maintainable approach: either return a conservative unsupported-platform `GatewayService` adapter whose read-only methods report safe unknown/not-installed state while mutating lifecycle methods reject with a clear manual-start message, or guard status-only call sites before resolving managed service adapters. Keep existing darwin/linux/win32 behavior unchanged. Do not add real OpenBSD service-manager support or a generated shell-service template in this PR. Add/update regression tests for unsupported platforms in the resolver/status paths and add a single-line changelog fix entry if the repo's active changelog section requires one for this user-visible behavior.
