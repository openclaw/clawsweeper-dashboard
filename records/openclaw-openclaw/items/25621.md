## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/25621. On unsupported Gateway service platforms such as OpenBSD/aix, `openclaw status` and `openclaw gateway status` must render read-only diagnostics instead of throwing from `resolveGatewayService()`. Keep lifecycle operations (`stage`, `install`, `uninstall`, `stop`, `restart`) clearly rejecting on unsupported platforms; do not add full OpenBSD service-manager support or generic shell-script generation in this repair. A conservative unsupported-platform `GatewayService` adapter is acceptable if its read-only methods return safe unknown/not-installed state, or use status-only guards if that better fits the existing code. Add regression coverage for `resolveGatewayService`, top-level status, and daemon CLI status with a mocked unsupported platform. Do not change macOS launchd, Linux/systemd, or Windows schtasks behavior. Add a `CHANGELOG.md` fix entry if repo policy requires one for the user-visible CLI fix.
