## Summary

Keep open: current main and v2026.5.12 still throw before read-only Gateway status diagnostics can degrade on unsupported platforms, and the focused related implementation PR was closed unmerged.

## What This Changes

Request that OpenBSD and other unsupported-platform status commands avoid crashing when managed Gateway service installation is unavailable.

## Best Possible Solution

Add a conservative unsupported-platform `GatewayService` adapter or status-only guard so read-only status reports unknown/not-installed while lifecycle operations reject clearly.

## Reproduction Assessment

Yes. Source inspection shows unsupported `process.platform` values such as `openbsd` or `aix` throw in `resolveGatewayService()`, and both status paths call it before their fallback handlers can run.

## Solution Assessment

Yes for the status crash: a narrow unsupported-platform adapter or status-only guard is the maintainable fix. Full OpenBSD service-manager support or generated shell templates should stay separate follow-up work.
