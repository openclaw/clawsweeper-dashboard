## Summary

Keep open: current main and v2026.5.7 still throw on unsupported Gateway service platforms before read-only status diagnostics can degrade, and the related implementation PR was closed unmerged.

## What This Changes

The issue requests that `openclaw status` and related diagnostics work on OpenBSD/unsupported Gateway service platforms instead of crashing.

## Best Possible Solution

Add a conservative unsupported-platform service adapter or status-only guard so read-only status reports unknown/not-installed while lifecycle operations reject clearly with manual-start guidance.

## Reproduction Assessment

Yes, source-level with high confidence. On an unsupported `process.platform` such as `openbsd` or `aix`, `resolveGatewayService()` throws and both `openclaw status` and `openclaw gateway status` reach that resolver before their fallback handlers can run.

## Solution Assessment

Yes for the status crash. A narrow unsupported-platform adapter or status-only guard is the maintainable fix; full OpenBSD service-manager support and generated shell templates should be separate follow-up work.

Fix unsupported-platform Gateway service status degradation for https://github.com/openclaw/openclaw/issues/25621. On unsupported `process.platform` values such as `openbsd` or `aix`, read-only diagnostics (`openclaw status`, `openclaw status --json`, `openclaw status --all`, and `openclaw gateway status`) should render degraded unknown/not-installed service state instead of crashing from `resolveGatewayService()`. Keep existing darwin/linux/win32 service behavior unchanged. Do not add full OpenBSD service-manager support, termux-services support, or a generated shell service template in this PR. Lifecycle/mutating operations such as install/start/stop/restart/uninstall must still reject clearly on unsupported platforms with manual-start guidance. Use the closed source PR https://github.com/openclaw/openclaw/pull/25690 only as context; implement against current main with focused tests and a user-visible changelog entry.
