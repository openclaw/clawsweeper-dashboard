## Repair Work Prompt

Fix the unsupported-platform Gateway service status crash reported in https://github.com/openclaw/openclaw/issues/25621. On unsupported `process.platform` values such as `openbsd`/`aix`, `openclaw status` and `openclaw gateway status` should render degraded read-only status instead of throwing from `resolveGatewayService()`. Keep lifecycle operations (`install`, `stage`, `uninstall`, `stop`, `restart`) explicitly unsupported with clear errors; do not add OpenBSD service-manager support or generic shell-template generation in this PR. Likely fix shape: add a conservative unsupported-platform `GatewayService` adapter or guard only status call sites before resolving managed service adapters. Add focused regression coverage for `resolveGatewayService()`, `getDaemonStatusSummary()`, and `gatherDaemonStatus()` on an unsupported platform, and update `CHANGELOG.md` because this is user-visible.
