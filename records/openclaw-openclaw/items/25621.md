## What This Changes

The issue requests that unsupported platforms such as OpenBSD no longer crash Gateway status commands and instead expose a useful unsupported-service fallback.

## Best Possible Solution

Add a conservative unsupported-platform Gateway service adapter or status-only guard so read-only status reports unknown/not-installed while lifecycle operations still reject clearly; leave full OpenBSD service-manager support and shell-template generation as separate product work.
