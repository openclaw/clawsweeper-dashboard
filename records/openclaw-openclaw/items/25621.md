## Summary

Keep open: current main and v2026.5.19 still throw on unsupported platforms before read-only Gateway status diagnostics can degrade, and the focused implementation PR at https://github.com/openclaw/openclaw/pull/25690 was closed unmerged.

## What This Changes

The issue reports that `openclaw status` and related Gateway status commands crash on OpenBSD because managed Gateway service installation is unsupported.

## Best Possible Solution

Add a conservative unsupported-platform `GatewayService` adapter or status-only guard so read-only status reports unknown/not-installed while lifecycle operations reject clearly.

## Reproduction Assessment

Yes at source level. Unsupported `process.platform` values such as `openbsd` or `aix` still throw in `resolveGatewayService()`, and both status paths call it before fallback handlers can run.

## Solution Assessment

Yes for the status crash. A narrow unsupported-platform adapter or status-only guard is the maintainable fix; full OpenBSD service-manager support or generic shell-template generation should remain separate follow-up work.
