## Summary

Keep open: current main and the latest release still throw before read-only Gateway status diagnostics can degrade on unsupported platforms, and the focused related fix PR was closed unmerged.

## What This Changes

The issue asks for unsupported Gateway service platforms such as OpenBSD to stop crashing read-only status commands and to provide clearer fallback service guidance.

## Best Possible Solution

Add a conservative unsupported-platform `GatewayService` adapter or status-only guard so read-only status reports unknown/not-installed while lifecycle operations reject clearly.

## Reproduction Assessment

Yes. On unsupported `process.platform` values such as `openbsd` or `aix`, `resolveGatewayService()` throws and both status paths call it before their fallback handlers can run.

## Solution Assessment

Yes for the status crash: a narrow unsupported-platform adapter or status-only guard is the maintainable fix. Full OpenBSD service-manager support or generated shell templates should be separate follow-up work.
