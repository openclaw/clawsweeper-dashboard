## Summary

Keep open: current main still throws for unsupported Gateway service platforms before read-only status diagnostics can degrade, and the linked implementation PR is closed unmerged.

## What This Changes

Request that unsupported service platforms such as OpenBSD keep read-only status commands usable instead of crashing with `Gateway service install not supported`.

## Best Possible Solution

Add a conservative unsupported-platform `GatewayService` adapter or status-only guard so read-only status reports unknown/not-installed while lifecycle operations reject clearly.

## Reproduction Assessment

Yes, source-level with high confidence. On unsupported `process.platform` values such as `openbsd` or `aix`, `resolveGatewayService()` throws and both status paths call it before their fallback handlers can run.

## Solution Assessment

Yes for the status crash: a narrow unsupported-platform adapter or status-only guard is the maintainable fix. Full OpenBSD service-manager support or generated shell templates should be separate follow-up work.
