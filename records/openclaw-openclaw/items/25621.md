## What This Changes

The issue asks for `openclaw status` and related read-only diagnostics to avoid crashing on OpenBSD when managed Gateway service installation is unsupported.

## Best Possible Solution

Add a conservative unsupported-platform `GatewayService` adapter or status-only guard so read-only status reports unknown/not-installed while lifecycle operations reject clearly.

## Reproduction Assessment

Yes. Source inspection shows unsupported `process.platform` values such as `openbsd` or `aix` throw in `resolveGatewayService()`, and both status paths call it before fallback handlers can run.

## Solution Assessment

Yes for the status crash: a narrow unsupported-platform adapter or status-only guard is the maintainable fix. Full OpenBSD service-manager support or generic shell-template generation should stay separate follow-up work.
