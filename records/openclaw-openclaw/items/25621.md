Keep open: current main and the latest release still throw for unsupported Gateway service platforms before read-only status diagnostics can degrade, and the linked implementation PR was closed unmerged.

## What This Changes

The issue asks for `openclaw status` and Gateway status commands to work on OpenBSD or other unsupported platforms instead of crashing with `Gateway service install not supported`, with broader shell-service guidance as an optional follow-up.

## Best Possible Solution

Add a conservative unsupported-platform `GatewayService` adapter or status-only guard so read-only status reports unknown/not-installed while lifecycle operations reject clearly.

## Reproduction Assessment

## Solution Assessment

Yes for the status crash: a narrow unsupported-platform adapter or status-only guard is the maintainable fix. Full OpenBSD service-manager support or generated shell templates should be separate follow-up work.
