## Summary

Keep open: current main still lacks this warning surface, and the PR has a concrete security flaw because a local skill-owned `.clawhub/origin.json` can suppress the untrusted-source warning. The linked security proposal remains open and needs a maintainer trust-model decision before this branch is repaired or merged.

## What This Changes

The PR adds skill trust metadata, a `skills.load.trustedDirs` config option, and CLI/Control UI/docs/tests that warn when local skills are not trusted.

## Best Possible Solution

Keep the additive status/API/config direction, but suppress local-source warnings only for explicit reviewed directories or provenance verified outside the skill-controlled tree.

## Reproduction Assessment

Yes. Source inspection of the PR head shows `resolveSkillTrustStatus` accepts a local origin marker before local-source classification, and the new test creates that marker inside a temporary workspace skill and expects no warning.

## Solution Assessment

No. Status-surface warnings are useful, but trusting a skill-owned sidecar as ClawHub proof defeats the security goal; explicit trusted dirs or externally verified ClawHub install provenance would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Do not trust local origin files as ClawHub proof:** `src/agents/skills-status.ts:306-307`
