## Summary

Keep open. This PR is an active implementation candidate for a security enhancement, but the current patch has a blocking trust-model bug: a locally writable `.clawhub/origin.json` file can suppress the new untrusted-local warning, which defeats the warning for copied or malicious local skills.

## What This Changes

The PR adds skill trust metadata, `skills.load.trustedDirs`, and CLI/Control UI/docs/tests that warn when loaded local skills lack ClawHub origin metadata or a trusted directory.

## Best Possible Solution

Keep the additive status/UI/config direction, but only suppress local-source warnings for explicit reviewed directories or provenance OpenClaw can verify without trusting files supplied by the local skill itself.

## Reproduction Assessment

Yes. The blocking behavior is source-reproducible on the PR head: a local skill can carry a shape-valid `.clawhub/origin.json`, and the classifier returns trusted ClawHub before local-warning classification.

## Solution Assessment

No, not yet. The additive status warning is a maintainable direction, but accepting a locally writable origin marker as trusted provenance undermines the security goal.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Do not trust local origin files as ClawHub proof:** `src/agents/skills-status.ts:306-307`
  - body: The classifier suppresses the untrusted-local warning as soon as the skill directory contains a shape-valid `.clawhub/origin.json`. That file is under the local skill tree, so a copied or malicious local skill can include it and be reported as trusted ClawHub content without registry or installer provenance, defeating the warning added for sideloaded skills.
  - confidence: 0.88
