## Summary

Keep open: the PR adds a useful warning/status surface and includes real behavior proof, but it still has a blocking security trust bug because a local skill can include a shape-valid `.clawhub/origin.json` and suppress the untrusted-local warning.

## What This Changes

The PR adds skill trust metadata, `skills.load.trustedDirs`, CLI and Control UI warnings, docs, changelog, generated config hashes, and tests for untrusted local skill sources.

## Best Possible Solution

Keep the additive status/API/CLI/UI warning surface, but only suppress local-source warnings for provenance OpenClaw can verify or for directories explicitly configured as reviewed and trusted.

## Reproduction Assessment

Yes. The blocker is source-reproducible on the PR head by creating a local workspace skill with a shape-valid `.clawhub/origin.json`, which the new classifier treats as trusted ClawHub content.

## Solution Assessment

No, not yet. The additive warning surface is maintainable, but a locally writable origin sidecar should not be enough to suppress a security warning without verified ClawHub provenance or an explicit trusted directory.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Do not trust local origin files as ClawHub proof:** `src/agents/skills-status.ts:306-307`
  - body: The classifier suppresses the untrusted-local warning as soon as a skill directory contains a shape-valid `.clawhub/origin.json`. That file lives inside the local skill tree, so a copied or malicious local skill can include it and appear trusted without registry or OpenClaw-owned install provenance, defeating the sideloading warning this PR adds.
  - confidence: 0.88
