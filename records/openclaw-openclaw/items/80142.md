## Summary

Keep open: the branch is useful, but the added security signal can be bypassed because a locally writable `.clawhub/origin.json` sidecar is treated as trusted ClawHub provenance.

## What This Changes

The PR adds skill trust metadata, `skills.load.trustedDirs`, and CLI/Control UI/docs/tests for warning about untrusted local skill sources.

## Best Possible Solution

Keep the additive status/API/config direction, but suppress local-source warnings only for explicit reviewed directories or provenance OpenClaw can verify without trusting files supplied by the local skill itself.

## Reproduction Assessment

Yes. On the PR head, source inspection shows a local `.clawhub/origin.json` marker is accepted before local-source warning classification, and the new test writes that marker into a temporary workspace skill and expects no warning.

## Solution Assessment

No, not yet. The warning surface is maintainable, but a locally writable origin sidecar should not be enough to suppress a security warning without verified ClawHub provenance or an explicit trusted directory.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Do not trust local origin files as ClawHub proof:** `src/agents/skills-status.ts:306-307`
  - body: The classifier suppresses the untrusted-local warning as soon as a skill directory contains a shape-valid `.clawhub/origin.json`. That file lives inside the local skill tree, so a copied or malicious local skill can include it and appear trusted without registry or OpenClaw-owned install provenance, defeating the warning this PR adds for sideloaded skills.
