## Summary

Keep this PR open, but do not merge it yet: the warning surface is useful, while the current trust classifier can be bypassed by a local skill that ships its own origin sidecar.

## What This Changes

The PR adds skill trust metadata, `skills.load.trustedDirs`, and CLI/Control UI/docs/tests for warning about untrusted local skill sources.

## Best Possible Solution

Keep the additive status/API warning surface, but suppress local-source warnings only for explicit reviewed directories or provenance OpenClaw can verify without trusting files supplied by the local skill itself.

## Reproduction Assessment

Yes. On the PR head, source inspection shows a workspace skill can include `.clawhub/origin.json` and be classified as trusted ClawHub content; the new test also encodes that expectation.

## Solution Assessment

No, not yet. The warning/status direction is maintainable, but a locally writable origin sidecar should not suppress the security warning without verified provenance or an explicit trusted directory.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Do not trust local origin files as ClawHub proof:** `src/agents/skills-status.ts:306-307`
  - body: The classifier suppresses the untrusted-local warning as soon as a skill directory contains a shape-valid `.clawhub/origin.json`. That file lives inside the local skill tree, so a copied or malicious local skill can include it and appear trusted without registry or OpenClaw-owned install provenance, defeating the sideloading warning this PR adds.
