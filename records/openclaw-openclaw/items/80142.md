## Summary

Keep open: current main still lacks the requested skill trust status surface, and the PR is a useful direction, but the patch should not merge while a locally writable `.clawhub/origin.json` can suppress the new untrusted-local warning.

## What This Changes

The branch adds skill trust metadata, a `skills.load.trustedDirs` config option, and CLI/Control UI/docs/tests that warn for untrusted local skill sources.

## Best Possible Solution

Keep the additive status/API/config direction, but suppress local-source warnings only for explicit reviewed directories or provenance OpenClaw can verify without trusting files supplied by the local skill itself.

## Reproduction Assessment

Yes. On the PR head, source inspection shows `resolveSkillTrustStatus` accepts a local `.clawhub/origin.json` before local warning classification, and the new test writes that marker into a temporary workspace skill and expects no warning.

## Solution Assessment

No, not yet. The additive status warning is maintainable, but accepting a skill-owned origin sidecar as trusted provenance undermines the security goal; use explicit trusted dirs or verifiable ClawHub install provenance instead.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
