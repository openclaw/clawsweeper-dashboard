## Summary

Keep open: the direction is useful and the PR includes real terminal proof for the local-without-origin warning path, but the proposed trust check has a blocking security false negative because arbitrary local skill directories can self-attest ClawHub origin metadata and suppress the new warning.

## What This Changes

The branch adds skill trust metadata, local-source warnings in CLI and Control UI status surfaces, a `skills.load.trustedDirs` config field, docs, changelog, and focused tests.

## Best Possible Solution

Keep the additive warning surface, but only suppress local-source warnings for provenance a local skill cannot self-assert, or require explicit `skills.load.trustedDirs`; maintainers should also decide whether the linked security issue needs enable/invoke confirmation beyond status rendering.

## Reproduction Assessment

Yes. By source inspection, any local skill directory containing syntactically valid `.clawhub/origin.json` follows the PR's origin-marker path and skips `untrustedLocalSource`, and the added test constructs exactly that local marker shape.

## Solution Assessment

No. A status-layer warning is a reasonable narrow start, but the trust suppression should not accept self-attested metadata inside the skill tree; tie it to installer/lock provenance outside the loaded skill or leave those local sources warned until explicitly trusted.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Require real ClawHub provenance before suppressing warnings:** `src/agents/skills-status.ts:306-307`

That defeats the linked security goal for local/third-party skill directories; require provenance outside the skill tree, such as a trusted install/lock record, or keep origin-only local skills warned.
