## Summary

Keep open: the PR addresses a real security proposal and has sufficient live proof, but the current diff still lets a locally writable `.clawhub/origin.json` suppress the untrusted-local warning it adds.

## What This Changes

The PR adds skill trust metadata, `skills.load.trustedDirs`, and CLI/Control UI/docs/tests that warn when local skill sources are not explicitly trusted.

## Best Possible Solution

Keep the additive status/API warning surface, but only suppress local-source warnings for explicit reviewed directories or provenance OpenClaw can verify without trusting files supplied by the local skill itself.

## Reproduction Assessment

Yes. On the PR head, a workspace skill can include a shape-valid `.clawhub/origin.json`; `resolveSkillTrustStatus` returns `clawhub` before local-warning classification, and the new test expects that bypass.

## Solution Assessment

No, not yet. The additive status warning is maintainable, but accepting a locally writable origin marker as trusted provenance undermines the security goal.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Do not trust local origin files as ClawHub proof:** `src/agents/skills-status.ts:306-307`
  - body: The classifier suppresses the untrusted-local warning as soon as a skill directory contains a shape-valid `.clawhub/origin.json`. That file lives inside the local skill tree, so a copied or malicious local skill can include it and appear trusted without registry or OpenClaw-owned install provenance, defeating the warning this PR adds for sideloaded skills.
  - confidence: 0.88
