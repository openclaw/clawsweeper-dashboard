## Summary

Keep open: current main still returns only snapshot-provided channel ids for the Channels page, and open PR #44165 already tracks the narrow UI fix with closing syntax.

## What This Changes

Request that the Control UI Channels page render built-in channel cards alongside custom channel plugins even when `channels.status` only reports the custom plugin.

## Best Possible Solution

Review and land or close PR #44165; if it lands, the issue can resolve through the normal closing reference.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: a snapshot with only custom `channelMeta` or `channelOrder` makes `resolveChannelOrder` return only that custom list, so built-in cards are skipped.

## Solution Assessment

Yes. The narrow maintainable solution is UI-only: merge snapshot-provided ids with the built-in discovery ids while leaving backend plugin loading semantics unchanged.
