## Label Justifications

- `P2`: The issue hides supported channel configuration choices in the Control UI but does not break active message delivery or core runtime availability.
- `impact:other`: Users can lose visibility and configuration access for built-in channel cards after enabling only a custom channel plugin.

## Summary

Keep open: current main and v2026.6.6 still treat snapshot channel metadata/order as the complete card list, so a custom-only channel snapshot can hide built-in channel cards; an open linked PR already owns the candidate fix.

## What This Changes

Show built-in channel cards in the Control UI even when the gateway snapshot contains only custom channel plugins.

## Best Possible Solution

Land or replace https://github.com/BingqingLyu/openclaw/pull/544 so snapshot-provided channels stay first, missing built-in channel cards are appended once, and regression coverage is added without changing backend plugin loading.

## Reproduction Assessment

Yes at source level: a snapshot whose channelMeta contains only a custom channel makes resolveChannelOrder return only that id, and renderChannels maps only that returned list to cards. I did not run a live Control UI browser scenario in this read-only review.

## Solution Assessment

Yes, the frontend resolver remains the narrowest maintainable fix: preserve backend loaded-plugin semantics, merge snapshot order with the built-in discovery list, and test custom-only plus mixed-order snapshots.
