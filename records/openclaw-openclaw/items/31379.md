## Label Justifications

- `P2`: The bug hides supported channel configuration choices in the Control UI but does not break active message delivery or core runtime availability.
- `impact:other`: Users can lose visibility and configuration access for built-in channel cards after enabling only a custom channel plugin.

## Summary

Keep open: current main and v2026.6.5 still treat snapshot channel metadata as the complete card list, so a custom-only snapshot can hide built-in channel cards; an open linked PR already owns the candidate fix.

## What This Changes

The issue requests that the Control UI Channels settings view render built-in channel cards even when a gateway snapshot contains only custom channel plugin metadata.

## Best Possible Solution

Land or replace the open UI-only resolver fix so snapshot IDs stay first, missing built-in channel IDs are appended once, and custom-only plus mixed-order regression coverage is added without changing backend plugin loading.

## Reproduction Assessment

Yes at source level: a snapshot containing only a custom channel ID passes through the current resolver unchanged and renderChannels maps only that returned ID list to cards. I did not run a live Control UI gateway scenario in this read-only review.

## Solution Assessment

Yes, the frontend resolver is the narrowest owner boundary: preserve backend loaded-plugin semantics, merge snapshot order with the built-in discovery list, and test custom-only plus mixed-order snapshots.
