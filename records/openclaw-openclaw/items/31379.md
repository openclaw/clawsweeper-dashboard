## Summary

Keep open: current main and v2026.5.12 still treat nonempty channel metadata as the complete Channels page render list, and the active open fix PR references this issue with closing syntax, so this should close only after that PR or a replacement lands.

## What This Changes

The issue requests that the Control UI Channels page keep built-in channel cards visible when a custom channel plugin is the only loaded channel.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/44165, or a replacement that preserves custom snapshot IDs first and appends de-duplicated built-in IDs from the catalog-backed channel order.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a custom-only `channelMeta` or `channelOrder` snapshot makes `resolveChannelOrder()` return only that custom ID, and `renderChannels()` renders exactly those IDs.

## Solution Assessment

Yes, on the UI-only boundary. The maintainable fix should append de-duplicated built-in channel IDs after snapshot-provided IDs, preferably from the catalog-backed order, without changing backend plugin loading semantics.
