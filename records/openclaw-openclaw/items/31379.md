## Summary

Keep open: current main and the latest release still treat `channelMeta` or `channelOrder` as the complete Channels page render list, and the open linked PR already owns the focused UI-only fix path.

## What This Changes

The issue requests that the Control UI Channels settings page keep showing built-in channel cards when a custom channel plugin is the only loaded channel in the gateway snapshot.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44165 or an equivalent UI-only resolver fix that preserves snapshot/custom IDs first, appends de-duplicated built-in IDs, and leaves backend plugin loading unchanged.

## Reproduction Assessment

Yes. Current main's resolver returns only snapshot `channelMeta` or `channelOrder` IDs, and `channels.status` can supply a custom-only list from loaded plugins, so the source path reproduces the custom-only render list.

## Solution Assessment

Yes. The narrow maintainable fix is to merge snapshot-provided IDs with de-duplicated built-in IDs in the UI resolver while leaving gateway plugin loading unchanged.
