## Summary

Keep this open: the checked-out main branch still treats `channelMeta` or `channelOrder` as the complete Channels page render list, while the backend status payload is still built from loaded channel plugins only. PR #44165 is an open, unmerged implementation candidate that explicitly links this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for the Control UI Channels settings page to show built-in channel cards alongside custom channel plugins even when the backend snapshot only contains the custom channel.

## Best Possible Solution

Land PR #44165 or an equivalent Control UI fix that merges snapshot-provided channel IDs with the built-in channel order, preserves snapshot/custom order first, dedupes IDs, keeps enabled-channel promotion, and adds focused regression tests without changing backend plugin loading semantics.
