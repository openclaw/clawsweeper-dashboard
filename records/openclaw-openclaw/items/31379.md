## What This Changes

The issue asks the Control UI Channels page to keep built-in channel cards visible when `channels.status` only reports a custom channel plugin.

## Best Possible Solution

Review and land or close PR #44165; if it lands, the closing reference can resolve this issue through the normal GitHub flow.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a snapshot containing only custom `channelMeta` or `channelOrder` makes `resolveChannelOrder` return only that custom list, so built-in fallback cards are skipped.

## Solution Assessment

Yes. The narrow maintainable fix is UI-only: merge snapshot-provided ids with the built-in discovery ids while leaving backend plugin loading semantics unchanged.
