## Summary

Keep open: current main still treats a nonempty channel snapshot as the complete Channels settings render list, so a custom-only channel snapshot can hide built-in cards. The active fix path is the open linked PR at https://github.com/openclaw/openclaw/pull/44165, which references this issue with closing syntax.

## What This Changes

The issue asks the Control UI Channels page to render built-in channel cards alongside custom channel plugin cards even when the backend snapshot only contains custom channels.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44165 or a replacement UI-only fix that preserves snapshot/custom IDs first and appends de-duplicated built-in channel IDs without changing backend plugin loading.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a custom-only `channelMeta` or `channelOrder` snapshot makes `resolveChannelOrder()` return only that custom ID, and `renderChannels()` renders only the returned IDs.

## Solution Assessment

Yes. The narrowest maintainable fix is the UI-only path already represented by the linked PR: merge snapshot-provided IDs with de-duplicated built-in channel IDs while leaving backend plugin loading unchanged.
