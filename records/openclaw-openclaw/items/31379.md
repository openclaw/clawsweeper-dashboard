## Label Justifications

- `P2`: The issue affects user-facing Control UI channel discovery with limited blast radius and does not break active delivery or core runtime availability.
- `impact:other`: Users can lose visibility of built-in channel setup cards after enabling only a custom channel plugin.

## Summary

Keep open: current main and the latest release still treat `channelMeta` or `channelOrder` as the complete Channels card list, so a custom-only channel snapshot can still hide built-in cards. The open linked implementation candidate has not landed in OpenClaw main, so this issue should wait for that fix to land or be replaced.

## What This Changes

The issue requests that the Channels settings page render built-in channel cards even when a custom channel plugin is the only loaded channel plugin.

## Best Possible Solution

Land or replace the UI resolver fix so snapshot-provided channels stay first, missing built-in channel cards are appended once, and regression coverage proves custom-only plus mixed-order snapshots without changing backend status semantics.

## Reproduction Assessment

Yes, source-level: current main maps exactly the resolver output to channel cards, and the resolver returns a custom-only `channelMeta` or `channelOrder` list as complete. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

Yes, the frontend resolver merge remains the narrowest maintainable solution: preserve backend loaded-plugin semantics, keep custom channels first, and append missing built-in discovery cards with de-duplication.
