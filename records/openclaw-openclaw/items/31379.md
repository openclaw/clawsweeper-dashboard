## Summary

Keep open: current main still treats a nonempty channel snapshot as the complete Control UI render list, so a custom-only snapshot still hides bundled channel cards. The active implementation path at https://github.com/openclaw/openclaw/pull/44165 references this issue with closing syntax, so this should remain open until a corrected PR lands or that path is closed.

## What This Changes

The issue asks for the Channels settings page to show custom channel plugins and built-in channel cards together when the gateway snapshot only reports the custom channel.

## Best Possible Solution

Land a corrected UI-only fix that preserves snapshot-provided custom IDs first, appends de-duped catalog-backed bundled channel IDs, and adds focused resolver coverage plus real Control UI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a custom-only channelMeta or channelOrder snapshot makes resolveChannelOrder() return only those IDs, and renderChannels() renders only that returned list.

## Solution Assessment

Yes on the UI-only boundary, but the maintainable implementation should use the current catalog-backed bundled channel order rather than a frozen inline list. Backend plugin loading should remain unchanged for this display bug.
