## Summary

Current main still has the reported Channels Control UI behavior, and the issue is already paired with open, unmerged PR #44165 using closing syntax. It should stay open until that PR or an equivalent UI/test fix lands.

## What This Changes

The issue requests that the Channels settings page always render built-in channel cards alongside custom channel plugins when the backend snapshot only reports loaded/custom channels.

## Best Possible Solution

Review and land PR #44165, or an equivalent Control UI change, so `resolveChannelOrder()` preserves snapshot-provided custom/loaded ids first, appends the built-in channel ids with dedupe, keeps enabled-channel promotion intact, and adds regression tests for custom-only `channelMeta`, custom-only `channelOrder`, and empty-snapshot fallback.

## Reproduction Assessment

Yes. A `channels.status` snapshot whose `channelMeta` or `channelOrder` contains only a custom id such as `icenter` will make current `resolveChannelOrder()` return only that id, so `renderChannels()` renders no built-in channel cards.

## Solution Assessment

Yes. The narrow frontend merge described by the issue and PR #44165 is the maintainable fix because the backend status payload intentionally reflects loaded plugins, while the UI already owns the discovery/card ordering behavior.
