## Summary

Keep open: current main and the latest release still treat a nonempty channel snapshot as the complete Channels page render list, so a custom-only snapshot can still hide built-in channel cards. An open implementation PR already references this issue with closing syntax, so this should stay open until that PR or a replacement lands.

## What This Changes

The issue requests that the Control UI Channels settings page show built-in channel cards alongside custom channel plugins even when only custom channels are loaded.

## Best Possible Solution

Land a corrected UI-only fix through https://github.com/openclaw/openclaw/pull/44165 or a replacement that preserves snapshot/custom IDs first and appends de-duplicated bundled IDs from the current channel catalog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a custom-only channelMeta or channelOrder snapshot makes resolveChannelOrder() return only those IDs, and renderChannels() renders only that returned list.

## Solution Assessment

Yes on the UI-only boundary, but the maintainable implementation should derive appended built-in IDs from the current catalog-backed order rather than a frozen inline list. Backend plugin loading should remain unchanged for this display bug.
