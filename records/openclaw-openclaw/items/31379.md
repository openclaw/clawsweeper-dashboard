## Summary

Keep open: current main and v2026.5.20 still treat `channelMeta` or `channelOrder` as the complete Channels page render list, and the open linked PR at https://github.com/openclaw/openclaw/pull/44165 owns the focused UI-only fix path.

## What This Changes

Show built-in channel cards in the Control UI even when the gateway snapshot only contains custom channel plugins.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44165 or an equivalent UI-only resolver fix that preserves snapshot/custom IDs first, appends de-duplicated built-in IDs, and leaves backend plugin loading unchanged.

## Reproduction Assessment

Yes. Current source inspection shows the resolver returns only snapshot `channelMeta` or `channelOrder`, while `channels.status` can supply a custom-only list from loaded plugins.

## Solution Assessment

Yes. The narrow maintainable fix is to merge snapshot-provided IDs with de-duplicated built-in IDs in the UI resolver while leaving gateway plugin loading unchanged.
