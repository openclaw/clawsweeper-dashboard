## Summary

Keep open: current main and the latest release still treat `channelMeta` or `channelOrder` as the complete Channels page render list, and an open linked PR already owns the UI-only fix path.

## What This Changes

The issue asks the Control UI Channels page to render built-in channel cards even when the gateway snapshot only lists enabled custom channel plugins.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44165 or an equivalent UI-only resolver fix that preserves snapshot/custom IDs first, appends de-duplicated built-in IDs, and leaves backend plugin loading unchanged.

## Reproduction Assessment

Yes, source-level. Current main's resolver returns only snapshot `channelMeta` or `channelOrder` IDs, and `channels.status` can supply a custom-only list from loaded plugins, so a custom-only plugin snapshot renders only that card.

## Solution Assessment

Yes. The narrow maintainable solution is a UI resolver change that merges snapshot-provided IDs with de-duplicated built-in IDs while leaving gateway plugin loading unchanged.
