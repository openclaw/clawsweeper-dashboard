## Summary

Keep open: current main and v2026.5.12 still treat a nonempty channel snapshot as the complete Channels page render list, so a custom-only snapshot can still hide built-in channel cards; https://github.com/openclaw/openclaw/pull/44165 is the active open fix path and already references this issue with closing syntax.

## What This Changes

The issue requests that the Control UI Channels settings page always render built-in channel cards alongside enabled custom channel plugins.

## Best Possible Solution

Land a corrected UI-only fix through https://github.com/openclaw/openclaw/pull/44165 or a replacement that preserves snapshot/custom IDs first and appends de-duplicated built-in IDs from the current channel catalog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a custom-only `channelMeta` or `channelOrder` snapshot makes `resolveChannelOrder()` return only that custom ID, and `renderChannels()` renders only the returned IDs.

## Solution Assessment

Yes on the requested UI-only boundary. The maintainable fix should leave backend plugin loading unchanged and derive appended built-in IDs from the catalog-backed channel order rather than a frozen inline list.
