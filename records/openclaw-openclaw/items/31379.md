## Summary

Keep open: current main and the latest release still treat `channelMeta` or `channelOrder` as the complete Channels page render list, and the open closing PR at https://github.com/openclaw/openclaw/pull/44165 already owns the focused fix path.

## What This Changes

The issue requests that the Control UI Channels settings page render built-in channel cards even when the gateway snapshot only includes a custom channel plugin.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44165 or an equivalent UI-only resolver fix that preserves snapshot/custom IDs first, appends de-duplicated built-in IDs, and leaves backend plugin loading unchanged.

## Reproduction Assessment

Yes. A custom-only loaded plugin registry can produce a `channels.status` snapshot whose `channelMeta` or `channelOrder` contains only the custom ID, and current `resolveChannelOrder` returns that list as the full render list.

## Solution Assessment

Yes. The narrow maintainable solution is to merge snapshot-provided IDs with de-duplicated built-in channel IDs in the UI resolver while leaving gateway plugin loading unchanged.
