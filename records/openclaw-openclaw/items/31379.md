## Summary

Keep open: current main and the latest release still treat a nonempty channel snapshot as the complete Channels page render list, so a custom-only snapshot can hide built-in channel cards. The active fix path is https://github.com/openclaw/openclaw/pull/44165, which references this issue with closing syntax, so this issue should stay open until that PR or a replacement lands.

## What This Changes

The issue requests that the Control UI Channels settings page keep showing built-in channel cards when a custom channel plugin is the only loaded channel in the backend snapshot.

## Best Possible Solution

Land a corrected UI-only fix through https://github.com/openclaw/openclaw/pull/44165 or a replacement that preserves snapshot/custom IDs first and appends de-duplicated built-in IDs, preferably from the catalog-backed channel order.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a custom-only channelMeta or channelOrder snapshot makes current main return only that custom ID, and renderChannels() renders only the returned IDs.

## Solution Assessment

Yes on the requested UI-only boundary. The maintainable fix should leave backend plugin loading unchanged and append de-duplicated built-in IDs to the snapshot-provided order.
