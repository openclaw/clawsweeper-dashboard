## Label Justifications

- `P2`: The issue affects user-facing channel discovery in the Control UI but does not break active message delivery or core runtime availability.
- `impact:other`: Users can lose visibility of built-in channel setup cards after enabling only a custom channel plugin.

## Summary

Keep open: current main and the latest release still treat snapshot `channelMeta` or `channelOrder` as the complete Channels card list, so a custom-only channel snapshot can still hide built-in channel cards. A linked open implementation candidate exists, so this issue should wait for that fix path to land or be replaced.

## What This Changes

Requested a Control UI resolver change so the Channels settings page appends built-in channel cards after snapshot-provided custom channels.

## Best Possible Solution

Land or replace the linked resolver PR so snapshot-provided channels stay first, missing built-in channel cards are appended once, and regression coverage proves custom-only plus mixed-order snapshots without changing backend plugin loading.

## Reproduction Assessment

Yes, source-level: current main maps exactly the resolver output to cards, and the resolver returns custom-only snapshot metadata/order as the complete list. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

Yes, the frontend resolver merge remains the narrowest maintainable solution: preserve backend loaded-plugin semantics, keep custom channels first, and append missing built-in discovery cards with de-duplication.
