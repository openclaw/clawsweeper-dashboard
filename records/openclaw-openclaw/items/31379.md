## Summary

Keep open: current main and the latest release still treat a custom-only `channelMeta` or `channelOrder` snapshot as the complete Channels card list, so built-in cards can still disappear. A linked open fork PR with closing syntax proposes the right UI resolver fix, so this issue should stay open until maintainers land, adopt, or replace that candidate.

## What This Changes

The issue asks for the Control UI Channels settings page to show built-in channel cards even when the gateway snapshot only lists custom channel plugins.

## Best Possible Solution

Land or replace the linked UI resolver fix so snapshot-provided custom channels stay first, missing built-in discovery cards are appended once, and focused regression coverage proves custom-only and mixed-order snapshots without changing backend status semantics.

## Reproduction Assessment

Yes, source-level: `renderChannels()` maps exactly the `resolveChannelOrder()` output to rendered cards, and the resolver returns a custom-only `channelMeta` or `channelOrder` list as complete. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

Yes, the narrow frontend resolver merge is the best maintainable shape: preserve backend loaded-plugin status semantics, keep custom channels first, and append missing built-in discovery cards with de-duplication.
