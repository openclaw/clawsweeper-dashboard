## Summary

Keep open: current main still treats `channelMeta`/`channelOrder` as the complete render list, so a custom-only channel snapshot hides built-in channel cards. Open PR #44165 already targets this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Show built-in channel cards in the Control UI even when `channels.status` only reports a custom channel plugin.

## Best Possible Solution

Land #44165 or an equivalent narrow UI/test fix that appends the built-in channel ids with dedupe while preserving snapshot-provided custom ids first and enabled-channel promotion.

## Reproduction Assessment

Yes. Source inspection on current main shows a `channels.status` snapshot with only `channelMeta: [{ id: "icenter", ... }]` or `channelOrder: ["icenter"]` makes `resolveChannelOrder()` return only that custom id, so built-in cards are not rendered.

## Solution Assessment

Yes. A narrow frontend merge/dedupe fix is the best path because the backend status payload intentionally reflects loaded plugins, while the Control UI owns discovery card ordering and already has built-in fallback behavior.
