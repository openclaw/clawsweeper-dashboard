## Summary

Keep open: current main still uses snapshot-only channel ordering, and open PR #44165 explicitly fixes and closes this issue, so it should not be closed before that PR is resolved.

## What This Changes

The issue requests that the Control UI Channels settings page continue to render built-in channel cards when the backend status snapshot only lists custom channel plugins.

## Best Possible Solution

Review and land or close PR #44165; if landed, GitHub can close this issue through the PR's closing reference.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a snapshot containing only custom `channelMeta` or `channelOrder` makes `resolveChannelOrder` return only those ids, bypassing the built-in fallback list.

## Solution Assessment

Yes. The narrow maintainable direction is a UI-only merge of snapshot-provided channel ids with the built-in discovery ids, preserving backend loading behavior and custom-first order.
