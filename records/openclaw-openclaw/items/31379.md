## What This Changes

The issue asks the Control UI Channels settings page to append built-in channel cards when the gateway snapshot lists only custom channel plugins.

## Best Possible Solution

Land a focused upstream UI resolver fix that preserves snapshot/custom IDs first, appends de-duplicated built-in channel IDs, and leaves backend plugin loading semantics unchanged.

## Reproduction Assessment

Yes, source-level. A custom-only channels.status snapshot with channelMeta or channelOrder containing only the custom ID reaches resolveChannelOrder(), which returns that list as the full rendered card set.

## Solution Assessment

Yes. The narrow maintainable fix is to merge snapshot-provided IDs with de-duplicated built-in IDs in the UI resolver while leaving gateway plugin loading and channels.status semantics unchanged.
