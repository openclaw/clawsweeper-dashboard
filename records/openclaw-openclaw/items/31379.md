## Summary

Keep open: current main and the latest release still treat a nonempty channel snapshot as the complete Channels page render list, so a custom-only snapshot can still hide built-in channel cards. The active fix path is the open linked PR at https://github.com/openclaw/openclaw/pull/44165, which references this issue with closing syntax.

## What This Changes

The issue requests that the Control UI Channels settings page continue to show built-in channel cards when a custom channel plugin is the only loaded channel in the backend snapshot.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44165 or a replacement UI-only fix that preserves snapshot/custom IDs first and appends de-duplicated built-in channel IDs without changing backend plugin loading.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a custom-only `channelMeta` or `channelOrder` snapshot makes `resolveChannelOrder()` return only that custom ID, and `renderChannels()` renders only the returned IDs.

## Solution Assessment

Yes. The best fix is the UI-only approach already represented by the linked PR: merge snapshot-provided IDs with de-duplicated built-in IDs while leaving backend plugin loading unchanged.
