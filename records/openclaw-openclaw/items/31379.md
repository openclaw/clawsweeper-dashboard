## Summary

Keep open: current main still treats a nonempty gateway channel snapshot as the complete render list, so a custom-only snapshot hides bundled channel cards. The open linked implementation path at https://github.com/openclaw/openclaw/pull/44165 still references this issue with closing syntax, so the issue should stay open until a corrected PR lands.

## What This Changes

The issue requests that the Control UI Channels settings page show built-in channel cards alongside custom channel plugins.

## Best Possible Solution

Land a corrected UI-only fix, likely through https://github.com/openclaw/openclaw/pull/44165 or a replacement, that preserves snapshot-provided IDs first, appends de-duped catalog-backed bundled channel IDs, and adds regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a custom-only channelMeta or channelOrder snapshot makes resolveChannelOrder() return only those IDs, and renderChannels() renders only that list.

## Solution Assessment

Yes for the current routing: keep this issue open while the linked implementation PR is reviewed. The maintainable fix should merge snapshot IDs with a catalog-backed bundled channel order rather than changing backend plugin loading or freezing an old inline list.
