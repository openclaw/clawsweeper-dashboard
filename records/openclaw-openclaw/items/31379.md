## Summary

Keep open: current main and the latest release still treat a nonempty Channels status snapshot as the complete render list, so a custom-only snapshot can hide built-in channel cards. A live PR at https://github.com/openclaw/openclaw/pull/44165 references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that the Control UI Channels page always show built-in channel cards alongside custom channel plugins.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44165 or an equivalent UI-only fix that preserves snapshot/custom IDs first and appends de-duplicated built-in channel IDs without changing backend plugin loading.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a custom-only `channelMeta` or `channelOrder` snapshot makes `resolveChannelOrder()` return only that custom ID, and `renderChannels()` renders only the returned IDs.

## Solution Assessment

Yes. The narrow maintainable solution is a UI-only resolver change that merges snapshot-provided IDs with de-duplicated built-in IDs while leaving backend plugin loading unchanged.
