## Summary

Keep open: current main still renders only the channel IDs present in the gateway snapshot, and an open linked pull request is already the active implementation path for this issue.

## What This Changes

Show bundled channel cards on the Control UI Channels page alongside loaded/custom plugin channels, even when the gateway snapshot contains only custom channel metadata.

## Best Possible Solution

Land a corrected UI-only fix, likely through https://github.com/openclaw/openclaw/pull/44165, that preserves snapshot-provided custom/loaded IDs first, appends de-duped catalog-backed bundled channel IDs, and adds regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: a custom-only channelMeta or channelOrder snapshot makes resolveChannelOrder() return only those IDs, and renderChannels() renders only that list.

## Solution Assessment

Yes for the current routing: keep this issue open while the linked implementation PR is reviewed. The maintainable fix should merge snapshot IDs with the catalog-backed bundled channel order rather than freezing a legacy list.
