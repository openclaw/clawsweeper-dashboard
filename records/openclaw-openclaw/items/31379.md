## Summary

Keep open: current main and the latest release still treat `channelMeta` or `channelOrder` as the complete Channels page render list, so a custom-only gateway snapshot can hide the built-in channel cards. https://github.com/openclaw/openclaw/pull/44165 is an open linked fix path with closing syntax, so this issue should wait for that PR or an equivalent fix to land.

## What This Changes

The issue asks the Control UI Channels page to show built-in channel cards even when a custom channel plugin is the only loaded channel in the gateway snapshot.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44165 or an equivalent UI-only resolver fix that preserves snapshot/custom IDs first, appends de-duplicated built-in channel IDs, and leaves backend plugin loading unchanged.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `channels.status` can be built from only loaded custom channel plugins, then current `resolveChannelOrder()` returns only that custom `channelMeta` or `channelOrder` list and `renderChannels()` renders only those IDs.

## Solution Assessment

Yes. The narrow maintainable solution is a UI-only resolver change that appends de-duplicated built-in IDs to snapshot-provided IDs while leaving gateway plugin loading unchanged; https://github.com/openclaw/openclaw/pull/44165 follows that shape but has not landed.
