## What This Changes

Requests that the Control UI Channels settings page render built-in channel cards alongside custom channel plugin cards even when the status snapshot only lists custom channels.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44165 or an equivalent UI-only fix that preserves snapshot/custom IDs first and appends de-duplicated built-in channel IDs without changing backend plugin loading.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a custom-only channelMeta or channelOrder snapshot makes resolveChannelOrder() return only custom IDs, and renderChannels() renders only those IDs.

## Solution Assessment

Yes. The narrow maintainable fix is a UI-only resolver change that merges snapshot-provided IDs with de-duplicated built-in IDs while leaving gateway plugin loading unchanged.
