## Label Justifications

- `P2`: The issue hides supported channel configuration choices but does not disrupt active channel delivery or core runtime availability.
- `impact:other`: Users can lose visibility and configuration access for supported built-in channels in the Control UI.

## Summary

Current main and v2026.6.5 still treat snapshot-provided channel metadata as the complete card list, so a custom-only snapshot continues to hide built-in channel cards. The issue remains necessary and is paired with an open implementation PR.

## What This Changes

Keep built-in channel cards visible in the Control UI when the gateway snapshot contains custom channel plugins.

## Best Possible Solution

Land one focused target-repository change that preserves snapshot order, appends deduplicated built-in channel IDs for discovery, and adds custom-only and mixed-order regression coverage without changing backend plugin-loading semantics.

## Reproduction Assessment

Yes at source level: a snapshot containing only a custom channel ID passes through the current resolver unchanged and produces only that card. This read-only review did not execute the Control UI in a live gateway setup.

## Solution Assessment

Yes. The frontend owns discovery-card composition, so merging snapshot IDs with the existing built-in fallback list is narrower and more maintainable than changing the backend’s loaded-plugin catalog contract.
