## Summary

Keep open: current main and the latest release still treat `channelMeta` or `channelOrder` as the complete render list, so a custom-only gateway snapshot hides the built-in channel cards. https://github.com/openclaw/openclaw/pull/44165 is an open linked fix path with closing syntax, so this issue should wait for that PR or an equivalent fix to land.

## What This Changes

The issue requests that the Control UI Channels settings page keep showing built-in channel cards when a custom channel plugin is the only channel reported by the gateway snapshot.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44165 or an equivalent UI-only resolver fix that preserves snapshot/custom IDs first, appends de-duplicated built-in channel IDs, and leaves backend plugin loading unchanged.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a custom-only `channelMeta` or `channelOrder` snapshot makes `resolveChannelOrder()` return only custom IDs, and `renderChannels()` renders only those IDs.

## Solution Assessment

Yes. The narrow maintainable solution is a UI-only resolver change that merges snapshot-provided IDs with de-duplicated built-in IDs while leaving gateway plugin loading unchanged.
