## Summary

Keep open: current main still renders only the channel IDs provided by the gateway snapshot, and an open linked pull request already carries the active implementation path for this issue.

## What This Changes

The issue requests that the Control UI Channels settings page keep built-in channel cards visible when a custom channel plugin is the only loaded channel.

## Best Possible Solution

Land a corrected UI-only fix that merges snapshot-provided custom or loaded IDs with catalog-backed bundled channel IDs, preserves snapshot order first, de-dupes IDs, and adds regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a snapshot with only custom channelMeta or channelOrder makes resolveChannelOrder return only that custom list, and renderChannels maps only those IDs into cards.

## Solution Assessment

Yes for keeping this issue open. The narrow maintainable fix is UI-only, but the appended built-in IDs should come from the current catalog-backed channel source rather than a frozen legacy list.
