## Summary

Keep open: current main and the latest release still treat a nonempty Channels status snapshot as the complete render list, so a custom-only channel snapshot can hide built-in channel cards. The active path is the open linked PR at https://github.com/openclaw/openclaw/pull/44165, which references this issue with closing syntax, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that the Control UI Channels settings page show built-in channel cards even when a custom channel plugin is the only loaded channel.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/44165 or an equivalent UI-only fix that preserves snapshot/custom IDs first and appends de-duplicated built-in channel IDs without changing backend plugin loading.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a custom-only `channelMeta` or `channelOrder` snapshot makes `resolveChannelOrder()` return only that custom ID, and `renderChannels()` renders only the returned IDs.

## Solution Assessment

Yes. The narrowest maintainable fix is the linked UI-only approach: merge snapshot-provided IDs with de-duplicated built-in IDs while leaving backend plugin loading unchanged.
