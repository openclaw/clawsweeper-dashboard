## Label Justifications

- `P2`: The issue affects user-facing channel discovery in the Control UI but does not break active message delivery or core runtime availability.
- `impact:other`: Users can lose visibility of built-in channel setup cards after enabling only a custom channel plugin.

## Summary

Keep open: current main and the latest release still let a custom-only `channelMeta` or `channelOrder` snapshot hide all built-in channel cards, and the open linked implementation candidate should land or be replaced before this issue closes.

## What This Changes

The issue asks for the Control UI Channels settings page to render built-in channel cards alongside custom channel plugins even when the gateway snapshot only reports the custom plugin.

## Best Possible Solution

Land or replace the linked UI resolver fix so snapshot-provided custom channels stay first, missing built-in channel cards are appended once, and regression coverage proves custom-only plus mixed-order snapshots without changing backend status semantics.

## Reproduction Assessment

Yes, source-level: current `renderChannels()` maps exactly the ids returned by `resolveChannelOrder()`, and that resolver treats a custom-only snapshot list as complete. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

Yes, the frontend resolver merge is the narrowest maintainable fix: preserve backend loaded-plugin semantics, keep custom channels first, and append missing built-in discovery cards with de-duplication.
