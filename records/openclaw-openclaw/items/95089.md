## What This Changes

Requests a CI routing change so Markdown-only pull requests avoid Blacksmith-backed `preflight` and `security-fast` while preserving necessary lightweight docs and workflow checks.

## Best Possible Solution

Add a conservative GitHub-hosted PR scope gate or equivalent narrow trigger change before Blacksmith jobs, with guard coverage proving root Markdown-only PRs skip Blacksmith while docs, workflow, and security-sensitive paths still receive the intended checks.

## Solution Assessment
