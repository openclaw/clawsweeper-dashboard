## Summary

Keep this issue open: current main and the latest release still route Markdown-only pull-request CI into Blacksmith-backed `preflight` and `security-fast` before docs-only detection can affect runner choice, and open closing PRs now own the implementation decision.

## What This Changes

Request to change PR CI routing so Markdown-only docs pull requests do not reserve Blacksmith-backed jobs while preserving necessary lightweight, docs, and security checks.

## Best Possible Solution

Review and land the early hosted scope-gate strategy in https://github.com/openclaw/openclaw/pull/95098, or choose an equivalent strategy that preserves docs and security coverage, then close this issue after the selected fix merges.

## Reproduction Assessment

Yes. Current-main source shows Markdown-only PRs still start CI and select Blacksmith for `preflight` and `security-fast` before docs-only detection, and the linked historical run shows those jobs requested the Blacksmith label with no runner assigned.

## Solution Assessment

Yes for the issue direction, but the best current implementation is an early hosted scope gate rather than broad trigger-level `paths-ignore`. The strongest open candidate is https://github.com/openclaw/openclaw/pull/95098 because it preserves CI/security jobs while changing runner selection.
