## Summary

Keep this issue open: current main and v2026.6.10 still route Markdown-only pull-request CI through Blacksmith-backed `preflight` and `security-fast` before docs-only detection can help, and multiple open closing PRs already target the fix path.

## What This Changes

Request to change OpenClaw CI routing so Markdown-only pull requests do not reserve Blacksmith runners while preserving needed lightweight docs/security checks.

## Best Possible Solution

Review and land one maintainer-selected workflow-routing fix, preferably an early hosted scope gate that preserves docs and security coverage while moving Markdown-only PR routing off Blacksmith.

## Reproduction Assessment

Yes. Source inspection shows a root Markdown-only PR is not ignored by the pull-request trigger and that `preflight` plus `security-fast` select Blacksmith before docs-only detection runs; the issue also includes a concrete queued CI run example.

## Solution Assessment

Unclear pending maintainer choice. An early hosted scope gate appears safer than broad trigger-level ignores because it can preserve docs and security coverage while changing runner selection.
