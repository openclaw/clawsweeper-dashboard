## Summary

Keep this issue open: current main still routes Markdown-only pull-request `preflight` and `security-fast` jobs to Blacksmith before docs-only detection can affect runner choice, and several open fix PRs now target the issue.

## What This Changes

The issue asks for Markdown-only PRs to avoid Blacksmith-backed CI jobs while preserving necessary lightweight docs, workflow, and security checks.

## Best Possible Solution

Land one maintainer-approved routing fix, with https://github.com/openclaw/openclaw/pull/95098 currently the strongest proof-backed candidate, then close this issue and retire alternate fix PRs as superseded.

## Reproduction Assessment

Yes. Current source shows Markdown-only PRs still enter CI and choose Blacksmith for `preflight` and `security-fast` before docs-only detection, and the linked docs-only run confirms those jobs requested the Blacksmith label.

## Solution Assessment

Yes, the best current direction is an early docs-scope gate that reuses the existing detector before runner selection; trigger-level `paths-ignore` alternatives need maintainer acceptance because they can suppress checks more broadly.
