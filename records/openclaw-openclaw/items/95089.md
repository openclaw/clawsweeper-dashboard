## Summary

Keep this issue open: current main and the latest release still route Markdown-only pull-request CI through Blacksmith-backed `preflight` and `security-fast` before docs-only detection can help. The issue is now paired with open fixing PRs, especially https://github.com/openclaw/openclaw/pull/95098, so the next step is maintainer strategy selection rather than another repair job.

## What This Changes

Route Markdown-only pull-request CI away from Blacksmith-backed runners without dropping necessary docs or security checks.

## Best Possible Solution

Land the proof-positive early scope-gate approach in https://github.com/openclaw/openclaw/pull/95098, or explicitly choose another strategy that preserves docs and security coverage, then close this issue after the selected fix merges.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.9 shows Markdown-only PRs still start CI and select Blacksmith for `preflight` and `security-fast` before docs-only detection runs; the issue also includes an observed queued-run example.

## Solution Assessment

Yes for the requested direction, but the best implementation appears to be an early hosted scope gate rather than broad trigger-level ignores because it preserves docs and security jobs while changing runner selection.
