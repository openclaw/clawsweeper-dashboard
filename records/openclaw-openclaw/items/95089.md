Keep open: current main and the latest release still route Markdown-only pull-request CI through Blacksmith-backed jobs before docs-only detection can affect runner choice, and open linked PRs already cover the implementation path.

## What This Changes

Request to route Markdown-only pull-request CI away from Blacksmith runners while preserving necessary docs and security checks.

## Best Possible Solution

Land the maintainer-selected CI-routing fix, preferably the proof-positive early scope-gate approach in https://github.com/openclaw/openclaw/pull/95098 if maintainers accept that graph change, then close this issue and retire duplicate branches.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.9 shows root Markdown-only pull requests still trigger CI, and preflight plus security-fast select Blacksmith before docs-only detection can influence routing.

## Solution Assessment

Yes for the early hosted scope-gate direction: it reuses the existing docs detector before runner selection and preserves docs/security jobs. Broad paths-ignore alternatives are riskier because they can suppress checks instead of rerouting them.
