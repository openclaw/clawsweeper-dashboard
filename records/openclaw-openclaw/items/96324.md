## Summary

Keep open: current main still has the targeted unbounded Google JSON reads, so the hardening is useful, but this PR head is not merge-ready because the OAuth project parser discards required payloads, duplicates an existing bounded JSON helper, and lacks after-fix real behavior proof.

## What This Changes

The PR replaces Google plugin success-body JSON reads with `readResponseWithLimit` plus `JSON.parse` under 1 MiB caps across embedding batch, image generation, media understanding, OAuth, and speech paths.

## Best Possible Solution

Use the existing provider JSON reader or a narrow shared Google wrapper, preserve the original OAuth payload assignments, and add focused oversized-stream proof plus cap rationale before merge.

## Reproduction Assessment

Yes. ## Solution Assessment

No. The bounded-read direction is appropriate, but this is not the best mergeable fix while it drops OAuth payload assignments, duplicates the existing bounded JSON helper, and lacks runtime proof.
