## Summary

Keep open: the Google bounded-read hardening is valid, but this PR is not merge-ready because the branch currently fails TypeScript/lint with duplicate imports, missing constants, and dropped OAuth payload assignments.

## What This Changes

The branch replaces several Google plugin `response.json()` success-body reads with `readResponseWithLimit` and 1 MiB JSON caps across embedding, image, media, OAuth, and speech providers.

## Best Possible Solution

Repair the patch with a small shared Google JSON read helper or consistent local helper calls that preserve existing assignments, add focused oversized-stream coverage/proof for the changed paths, and keep the hardening scoped to the Google plugin.

## Reproduction Assessment

Source-level yes: current main has unbounded `await response.json()` reads on the Google plugin success paths this PR targets. I did not run a live oversized-response repro for these exact paths, and the PR does not include one.

## Solution Assessment

No: using `readResponseWithLimit` through the plugin SDK is the right shape, but this implementation is not the best fix until it preserves payload variables, declares or centralizes caps, removes duplicate imports, and proves the runtime behavior.
