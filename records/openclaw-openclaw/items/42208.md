## Summary

Keep open: this PR is a useful security direction, but it is not mergeable as written because the new guard is not wired into production tool dispatch, misses current messaging send tools, and leaves the existing web_search wrapper test inconsistent with the behavior change.

## What This Changes

The PR adds a security guard helper and tests for destructive tool calls in external-content hook sessions, and changes web_search wrapping to include the long security warning.

## Best Possible Solution

Implement the external-content approval boundary through the existing before_tool_call/trusted-policy runtime path, cover OpenClaw's actual mutating and messaging tools, preserve the break-glass behavior intentionally, and update regression tests for hook sessions and web_search wrapping.

## Reproduction Assessment

Yes. Source inspection reproduces the PR defects: the new guard has no production call site, the current web_search test contradicts the wrapper change, and the proposed outbound messaging patterns miss current OpenClaw send tools.

## Solution Assessment

No. A standalone helper plus unit tests is not the best fix unless it is integrated into the existing runtime hook/approval path and uses the current tool taxonomy.
