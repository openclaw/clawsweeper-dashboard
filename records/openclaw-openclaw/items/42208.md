## Summary

Keep open: the security goal is plausible, but this PR is not mergeable because the new guard is not wired into production dispatch, its destructive-tool matcher is narrower than current mutating-tool handling, the `web_search` warning change conflicts with the current wrapper contract/tests, and the external PR has no real hook-session proof.

## What This Changes

The PR adds a new `src/security/tool-call-guard.ts` helper/test and makes `wrapWebContent` include the external-content warning for `web_search` as well as `web_fetch`.

## Best Possible Solution

Implement the gate through the existing `before_tool_call` or trusted-policy approval path, reuse the shared mutating-tool classifier, update the `web_search` wrapper contract/tests deliberately, and validate with redacted hook-session proof.

## Reproduction Assessment

Yes for the PR defects. Source inspection of the PR diff and current main shows the guard is added as a helper/test only, current dispatch already has an approval seam, and the current `web_search` wrapper test still asserts no long security warning.

## Solution Assessment

No. A standalone helper is not the best way to solve this unless it is integrated into the existing approval path and aligned with OpenClaw's current mutating-tool taxonomy.

## Review Findings

Full review comments:

  - body: `evaluateToolCallGuard` returns `requiresApproval`, but the PR never calls it from agent, gateway, MCP, or tool execution before a selected tool runs. Hook-originated destructive calls would still execute without this gate, so the advertised hard boundary is unenforced.
  - confidence: 0.96
- **[P1] Reuse the shared mutating-tool classifier:** `src/security/tool-call-guard.ts:13-29`

  - confidence: 0.92
- **[P2] Update the web_search wrapper contract:** `src/security/external-content.ts:339-340`
  - body: Forcing warnings for `web_search` conflicts with the current colocated test and documented wrapper behavior that only `web_fetch` carries the long `SECURITY NOTICE`. Either update the contract/tests/docs for that behavior change or keep the prior warning policy.
  - confidence: 0.9
