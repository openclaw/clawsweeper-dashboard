## Summary

Keep open: this is a plausible security hardening PR, but the branch still does not enforce the promised boundary at runtime, misses OpenClaw's current mutating send surfaces, changes the web_search wrapper contract without updating tests, and has no real hook-session proof.

## What This Changes

The PR adds an external-content tool-call guard/test and changes `wrapWebContent` so `web_search` output includes the external-content warning.

## Best Possible Solution

Implement the boundary through the existing trusted-policy or `before_tool_call` approval path, reuse the central mutating-tool classifier, update the web_search contract tests/docs, and require redacted real hook-session proof before merge.

## Reproduction Assessment

Yes. Source inspection reproduces the review defects: the guard is only a helper/test in the PR, current dispatch already has a separate approval seam, the matcher is narrower than current mutating sends, and the web_search test contract conflicts with the warning change.

## Solution Assessment

No. A standalone helper is not the best fix unless it is wired into the existing approval path and aligned with OpenClaw's current mutating-tool taxonomy.
