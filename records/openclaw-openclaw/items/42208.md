## Summary

Keep open. The security goal is valid, but this PR is not mergeable as submitted because the new guard is not connected to runtime tool dispatch, duplicates a narrower mutating-tool classifier, uses brittle session-key provenance, and changes web_search warning behavior without updating the existing expectation.

## What This Changes

The PR adds a new security tool-call guard with tests and changes `wrapWebContent` so `web_search` results receive the detailed external-content warning.

## Best Possible Solution

Implement the policy inside the existing trusted before-tool-call path, using explicit external-content provenance plus `isMutatingToolCall`, then route mandatory approval through the established plugin approval gateway while preserving deliberate unsafe-content bypass semantics.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main wraps hook content before the model turn, `runBeforeToolCallHook` has no external-content approval policy, and the PR adds a helper that is not invoked by production dispatch.

## Solution Assessment

No. The proposed direction is useful, but the submitted implementation is not the best fix because it is unintegrated, uses a narrower classifier than current main already has, and relies on raw session-key prefixes instead of preserved provenance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Wire the guard into tool execution:** `src/security/tool-call-guard.ts:46-76`
  - body: The new `evaluateToolCallGuard` helper is never called from the production before-tool-call or tool-dispatch path, so destructive tool calls from external hook content would still execute without the promised human approval boundary.
  - confidence: 0.97
- **[P2] Reuse the existing mutating-tool classifier:** `src/security/tool-call-guard.ts:13-29`
  - body: The hard-coded destructive regex misses current mutating surfaces such as `message`, `sessions_send`, action-sensitive `cron`/`gateway`/`canvas` calls, and `process` actions. A security gate should align with `isMutatingToolCall` instead of maintaining a smaller parallel list.
  - confidence: 0.91
- **[P2] Carry hook provenance instead of matching raw keys:** `src/security/tool-call-guard.ts:55-57`
  - body: Cron normalizes hook sessions under agent-scoped keys and can also preserve external-content provenance separately, while this guard only checks raw `hook:*` keys. Once integrated, common hook runs could bypass approval unless provenance is passed explicitly.
  - confidence: 0.89
- **[P2] Update the web_search warning expectation:** `src/security/external-content.ts:343-344`
  - body: Changing `wrapWebContent(..., "web_search")` to always include the long warning conflicts with the existing test that asserts web search wrapping does not contain `SECURITY NOTICE`, so the focused test expectation needs to change with this behavior.
  - confidence: 0.94
