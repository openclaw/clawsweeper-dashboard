## Summary

Keep this PR open, but do not merge it as-is: the security goal is valid, while the proposed guard is not wired into production dispatch, misses current outbound messaging tools, and leaves the web_search wrapper test contract inconsistent.

## What This Changes

The PR adds a security tool-call guard and tests for hook sessions, and changes web_search wrapping to include the detailed external-content warning.

## Best Possible Solution

Implement the approval boundary through the existing before_tool_call or trusted-tool-policy execution path, cover OpenClaw's actual mutating and outbound-send tools, preserve the break-glass behavior intentionally, and update focused regression tests.

## Reproduction Assessment

Yes. Source inspection reproduces the PR defects: the new guard is absent from production dispatch, current web_search tests contradict the wrapper change, and current messaging sends use tool names the branch does not match.

## Solution Assessment

No. A standalone helper plus unit tests is not the best security fix unless it is integrated into the existing runtime approval path and uses the current tool taxonomy.
