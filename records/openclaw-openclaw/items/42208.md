## Summary

Keep open: this is a plausible security hardening direction, but the patch is not mergeable because the new guard is not wired into production dispatch, its tool matcher is narrower than OpenClaw's current mutating-tool taxonomy, the web_search wrapper contract is not updated, and the external PR lacks real behavior proof.

## What This Changes

The PR adds a security guard helper and tests for destructive tool calls in external hook sessions, and changes `wrapWebContent` so `web_search` content includes the external-content warning.

## Best Possible Solution

Implement the gate through the existing `before_tool_call` or trusted-policy approval path, reuse central mutating-tool classification, update the web_search warning contract, and validate with a redacted hook-session run.

## Reproduction Assessment

Yes. Source inspection against PR head 33e2409 and current main reproduces the review defects: the guard is only introduced in the new helper/test, the diff does not touch production dispatch, and current main has a broader mutating-tool classifier plus an existing approval seam.

## Solution Assessment

No. A standalone helper is not the best fix unless it is wired into the existing approval path and aligned with OpenClaw's current mutating-tool taxonomy.
