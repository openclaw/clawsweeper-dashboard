Allow operators to customize or suppress pairing-challenge text sent to unknown DM senders so branding and CLI approval hints are not exposed.

## Best Possible Solution

Land one maintainer-approved response contract for unpaired DM pairing replies, wired through shared pairing helpers, config schemas, docs, and the relevant channel call sites, then close this issue after the chosen PR merges.

Unclear. A configurable reply surface is a plausible fix, but maintainers need to choose between granular message templates and broader silent/code-only/branded response modes before landing a coherent schema and channel contract.
