## Summary

## What This Changes

The PR adds a memory-core phase-signal helper that filters REM inputs to pending light-staged short-term entries, records REM-considered keys, and adds short-term promotion regression coverage.

## Best Possible Solution

Thread same-sweep light-staged keys into REM selection, keep the standalone REM fallback when no handoff yields candidates, and prove the old-high-recall versus fresh-light-staged path with a real workspace command output.

No. The PR touches the right runtime path, but the pending-light helper can still include historical unconsumed staged entries; a same-sweep handoff or explicit recency window is the safer fix.
