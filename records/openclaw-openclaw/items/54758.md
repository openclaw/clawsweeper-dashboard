## Summary

Keep this PR open: current main and the latest release still flush pending Control UI RPCs with a generic gateway-closed Error, while the PR narrowly preserves structured connect-failure details and has focused test plus real-environment log proof. The remaining blocker is maintainer acceptance of the intentional pending-request error-shape compatibility change.

## What This Changes

The PR changes Control UI WebSocket close handling to reject pending RPC promises with GatewayRequestError when a structured connect failure is pending, and adds a node regression test.

## Best Possible Solution

Land the narrow GatewayRequestError propagation after Control UI owners accept the pending-request error-shape compatibility change; if they do not, preserve the generic rejection contract while exposing structured metadata through an agreed path.

## Reproduction Assessment

Yes. Source inspection shows current main stores pendingConnectError but flushes pending promises with a generic Error, and the PR adds a focused test that drives the failed connect response plus pending request rejection path.

## Solution Assessment

Yes, with one maintainer decision outstanding. Reusing the existing GatewayRequestError at the flushPending boundary is the narrowest fix because pending RPC promises otherwise never receive the structured connect details.
