## What This Changes

This PR changes `GatewayBrowserClient` so close handling can reject pending Control UI RPCs with structured connect-handshake errors and adds a mocked node unit regression test.

## Best Possible Solution

Preserve the original `GatewayRequestError` instance or store raw gateway error fields for pending-request flushes, keep ordinary closes generic, and land only after redacted live Control UI/gateway proof.

## Reproduction Assessment

Yes. Source inspection shows current main stores structured connect failure details but flushes pending requests with a generic close `Error`; this review did not run tests because it was read-only.

## Solution Assessment

No. The branch should not construct a new `GatewayRequestError` from an already formatted stored error; it should preserve the original error instance or store raw unformatted gateway error fields.
