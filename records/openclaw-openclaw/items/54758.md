## What This Changes

The PR changes `GatewayBrowserClient` to flush pending RPCs with the stored structured connect error on WebSocket close and adds a node unit regression test.

## Best Possible Solution

Land the narrow error propagation and regression test after the contributor adds redacted real behavior proof for a live Control UI/gateway pending request.

## Reproduction Assessment

Yes, by source inspection. Current main can enqueue a request on an open WebSocket before the connect handshake fails, stores `pendingConnectError`, and then flushes that pending request with `Error("gateway closed ...")`.

## Solution Assessment

Yes, functionally. Reusing the existing `pendingConnectError` in the close handler is the narrowest maintainable fix, and non-connect closes stay on the generic close-error path.
