## Summary

The remaining decision is maintainer acceptance of the compatibility-sensitive error-shape change and normal merge/check readiness.

## What This Changes

The branch changes `GatewayBrowserClient` close handling to reject pending RPCs with `GatewayRequestError` when a structured connect error exists, and adds a regression test for that path.

## Best Possible Solution

Land the focused typed-error propagation after Control UI owners accept the pending-request error-shape compatibility change and required checks are acceptable.

## Solution Assessment

Yes, with a maintainer compatibility call: reusing the existing `GatewayRequestError` for the already-captured connect error is the narrowest maintainable fix. Changing only UI formatting would not help pending RPC callers that need structured rejection metadata.
