The branch changes `GatewayBrowserClient` close handling to reject pending requests with a preserved structured `GatewayRequestError` after connect-handshake failures and adds a node unit regression test.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes, functionally. Reusing the existing `pendingConnectError` to select the close-time flush error is the narrowest maintainable fix, and non-connect closes still keep the generic error path.
