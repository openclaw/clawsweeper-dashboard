## What This Changes

The branch changes `GatewayBrowserClient` so WebSocket close flushes pending Control UI RPCs with structured connect-handshake errors and adds a mocked node unit regression test.

## Best Possible Solution

Carry the original structured connect failure, or store raw unformatted gateway error info, so pending requests preserve `gatewayCode` and `details` without reformatting; land only after redacted real Control UI/gateway proof and conscious compatibility acceptance.

## Reproduction Assessment

Yes. Source inspection shows current main stores structured connect failure details but flushes pending requests with a generic close `Error`; the PR-specific double-formatting defect is also source-reproducible from the stored formatted message plus the `GatewayRequestError` constructor path.

## Solution Assessment

No. The fix direction is right, but wrapping the stored `connectError` in a new `GatewayRequestError` can double-format protocol mismatch messages; carrying the original error or raw unformatted fields is safer.
