## Summary

This PR is still necessary because current `main` still drops structured connect failure details when flushing pending Control UI RPC requests, while the branch preserves those details through `GatewayRequestError`. The patch looks narrowly correct, but it should stay in maintainer review because it intentionally changes the pending-request rejection shape for structured connect failures.

## What This Changes

This PR changes `GatewayBrowserClient` close handling to reject pending requests with `GatewayRequestError` when a structured connect failure exists, and adds a node regression test for that path.

## Best Possible Solution

Land the narrow structured-error propagation after a Control UI owner accepts the pending RPC rejection-shape compatibility change and the branch is confirmed clean against current `main`.

## Reproduction Assessment

Yes from source inspection: current `main` stores `pendingConnectError` before close but flushes pending promises with a generic close Error. I did not run the test in this read-only review.

## Solution Assessment

Yes, with a maintainer compatibility decision outstanding. Reusing the existing `GatewayRequestError` at the pending flush boundary is the narrow owner-local fix because pending RPC promises otherwise never receive the structured connect details.
