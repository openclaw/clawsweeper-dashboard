## What This Changes

The PR changes Control UI GatewayBrowserClient close handling to reject pending requests with GatewayRequestError when a structured connect failure is pending, and adds a node regression test.

## Best Possible Solution

Land the narrow structured-error propagation after a current-base conflict refresh and Control UI owner acceptance of GatewayRequestError for pending RPC promises during structured connect failures.

## Reproduction Assessment

## Solution Assessment

Yes, with owner acceptance of the compatibility change. Reusing GatewayRequestError at the existing flushPending boundary is the narrowest local fix because pending request callers otherwise never receive the structured connect details already used by close/UI paths.
