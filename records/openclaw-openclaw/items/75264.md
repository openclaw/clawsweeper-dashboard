## Summary

Keep open. Current main still matches the reported failure shape: the bundled Claude CLI backend exposes Claude-native file tools while the OpenClaw MCP loopback explicitly excludes native read/write/edit/exec/process tools, the live-session watchdog can still abort after minute-scale silence, and Telegram still converts dispatch failures to the generic fallback.

## What This Changes

Investigate and fix Telegram-routed Claude CLI wrapper-path turns that stall on a native file Read call and only surface a generic Telegram error instead of executing or failing fast.

## Best Possible Solution

Define the supported boundary for Claude CLI native file tools versus OpenClaw MCP/ACP-hosted tool execution, then either fail fast with sanitized diagnostics or route wrapper execution through the intended ACP/node path.

## Reproduction Assessment

Unclear. The issue gives detailed live steps and current source matches the reported failure shape, but reproducing it with high confidence requires the reporter's Telegram, Claude CLI, macOS wrapper, auth, and node approval setup.

## Solution Assessment

Unclear. There is no proposed patch, and the narrowest maintainable fix depends on whether maintainers want to intercept Claude-native file tools, improve timeout diagnostics, or document the raw CLI backend path as unsupported for wrapper execution.
