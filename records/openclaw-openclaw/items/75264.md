## Summary

Keep open. Current main still matches the reported failure shape: Claude CLI native file tools are outside OpenClaw's MCP loopback, live-session timeouts can still produce a minute-scale abort, and Telegram still maps dispatch failures to the generic fallback; the related gateway CPU-spin work does not prove this native Read path is fixed.

## What This Changes

The report asks OpenClaw to avoid a long Claude CLI/Telegram stall on native file Read of an approved wrapper path by executing through the supported path or failing fast with a clear sanitized tool error.

## Best Possible Solution

Define the supported boundary for claude-cli native file tools versus OpenClaw MCP/ACP/node execution, then either fail fast with sanitized diagnostics or document and route wrapper execution through the intended ACP/node path.
