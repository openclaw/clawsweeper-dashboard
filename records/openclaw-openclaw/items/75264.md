## Summary

Keep open. Current main still has the reported failure boundary: Claude CLI native file tools remain outside OpenClaw's loopback MCP policy path, watchdog timeout behavior can still convert a silent CLI/native-tool stall into a minute-scale failure, and Telegram still sends the generic dispatch fallback instead of a per-tool diagnostic.

## What This Changes

Investigate a Telegram-triggered Claude CLI backend stall during a native file Read of an approved wrapper path, where Telegram only receives a generic error.

## Best Possible Solution

Define the supported route for Telegram-triggered wrapper execution on Claude CLI, then either fail fast with sanitized diagnostics for unsupported native Read paths or document and route the workflow through ACP/node exec where OpenClaw owns the tool policy.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. Source inspection confirms the same structural path, but the exact stall requires the reporter's Telegram, Claude CLI, macOS wrapper, auth profile, and node approval environment.

## Solution Assessment

Unclear. There is no proposed patch, and the narrowest maintainable fix depends on whether maintainers want to intercept or diagnose Claude native file tools, improve Telegram timeout disclosure, or declare raw CLI-backend wrapper execution out of scope in favor of ACP/node exec.
