## Summary

Keep open. Current main still matches the reported failure shape: Claude CLI native file tools are outside OpenClaw's loopback tool path, Claude live turns can still abort on timeout/no-output timers, and Telegram still converts dispatch failures into the generic user-facing fallback.

## What This Changes

The issue asks OpenClaw to avoid a multi-minute Telegram-routed Claude CLI stall on a wrapper-path `Read` and return either wrapper output or a clear sanitized denial/error.

## Best Possible Solution

Define the supported Claude CLI native-tool boundary, then either make unsupported native `Read` fail fast with sanitized diagnostics or route wrapper execution through an OpenClaw-owned exec path that can log, approve, and return results predictably.
