## Summary

Keep open. Current main still has the reported structural boundary: Claude CLI native file tools remain outside OpenClaw's MCP loopback policy, the loopback excludes `read`, a silent Claude live-session stall can still become a timeout, and Telegram still sends a generic dispatch fallback.

## What This Changes

The issue asks OpenClaw to avoid a multi-minute Claude CLI plus Telegram stall on a wrapper-path `Read` failure by either executing the approved wrapper or failing fast with a clear tool error.

## Best Possible Solution

Define the supported Claude CLI route for Telegram-triggered wrapper execution, then implement sanitized fast-failure diagnostics or route the workflow through ACP/node exec where OpenClaw owns policy.

## Reproduction Assessment

Unclear. The issue gives detailed live reproduction steps and current main still has the same structural failure shape, but high-confidence reproduction needs the reporter's Telegram, Claude CLI, macOS wrapper, auth profile, and node approval environment.

## Solution Assessment

Unclear. There is no patch, and the narrowest maintainable solution depends on whether maintainers want to intercept/diagnose Claude native file tools, improve Telegram timeout disclosure, or document ACP/node exec as the supported route.
