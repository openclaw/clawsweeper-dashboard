## Summary

Keep open. Current main still matches the reported failure boundary: Claude CLI keeps native file tools outside OpenClaw's MCP loopback policy path, watchdog timeout behavior can still turn a silent CLI/native-tool stall into a minute-scale failure, and Telegram still sends the generic dispatch fallback rather than a per-tool diagnostic.

## What This Changes

The issue asks for Telegram-triggered Claude CLI wrapper-path turns to either execute the approved wrapper or fail fast with clear diagnostics instead of stalling and returning only a generic Telegram error.

## Best Possible Solution

Define the supported route for Telegram-triggered wrapper execution on Claude CLI, then either fail fast with sanitized diagnostics for unsupported native Read paths or document and route the workflow through ACP/node exec where OpenClaw owns the tool policy.

## Solution Assessment
