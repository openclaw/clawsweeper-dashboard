## Summary

Keep open. Current main still matches the reported failure shape: Claude CLI native `Read` is outside OpenClaw's MCP loopback tool path, Claude live-session watchdogs can still turn a quiet native-tool stall into a timeout, and Telegram still projects dispatch failures as the generic fallback; related #72338 liveness fixes do not prove this wrapper-path/native-Read case is resolved.

## What This Changes

The issue asks OpenClaw to make Claude CLI plus Telegram wrapper-path turns either execute approved wrappers or fail fast with a clear, sanitized tool error instead of stalling and sending only a generic Telegram error.

## Best Possible Solution

Define the supported boundary for Claude CLI native file tools versus OpenClaw MCP/ACP-hosted execution, then either produce a sanitized fast failure for unsupported native reads or document and route wrapper execution through the intended ACP/node path.

## Solution Assessment
