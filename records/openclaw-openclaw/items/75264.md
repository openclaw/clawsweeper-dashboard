## Summary

Keep open. Current main still has the same structural failure boundary: Claude CLI keeps native file tools outside OpenClaw's MCP loopback policy path, the CLI watchdog can turn a silent native-tool stall into a minute-scale timeout, and Telegram still emits a generic dispatch fallback.

## What This Changes

The issue asks for Telegram-routed Claude CLI turns against an approved wrapper path to either succeed or fail fast with a clear tool error instead of stalling and returning a generic Telegram error.

## Best Possible Solution

Define the supported route for Telegram-triggered wrapper execution on Claude CLI, then either fail fast with sanitized diagnostics for unsupported native Read paths or route the workflow through ACP/node exec where OpenClaw owns tool policy.

## Reproduction Assessment

Unclear as an end-to-end live reproduction. Source inspection confirms current main still has the same structural path, but reproducing the exact stall requires the reporter's Telegram, Claude CLI, macOS wrapper, auth profile, and node approval setup.

## Solution Assessment

Unclear. There is no proposed patch, and the narrowest maintainable fix depends on whether maintainers want to intercept Claude native file tools, improve Telegram timeout disclosure, or declare this CLI-backend wrapper route unsupported.
