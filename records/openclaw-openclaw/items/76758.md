## Summary

Keep open. Current main still lacks a host-side Clock, frozen RPC snapshot replay contract, or reusable plugin tool-output replay harness, and the request would add public SDK/runtime semantics rather than clean up an obsolete item.

## What This Changes

The issue requests a deterministic plugin tool-call replay contract covering Clock injection, frozen host RPC responses during replay, and reusable record/replay test helpers.

## Best Possible Solution

Keep this open for a versioned plugin SDK/runtime replay design that standardizes time access and records/replays allowed host RPC responses after the gateway accessor and type surfaces are settled.
