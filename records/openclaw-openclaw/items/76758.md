## Summary

Keep open. Current main still lacks the requested host-side replay clock, frozen plugin RPC snapshot contract, and reusable plugin tool output replay harness; this is a new public SDK/runtime design rather than conservative cleanup.

## What This Changes

Define a host-side plugin replay contract with SDK Clock injection, frozen gateway RPC snapshots, and reusable record/replay helpers for deterministic plugin tool outputs.

## Best Possible Solution

Keep this open for a versioned plugin SDK/runtime design that coordinates replay clocks, gateway snapshot semantics, public type exports, docs, and focused contract tests.

## Reproduction Assessment

Not applicable; this is an RFC-shaped feature request, not a broken existing behavior with an established contract. Source and docs inspection confirm the requested host-side plugin replay contract is absent on current main.

## Solution Assessment

Unclear; the direction is plausible, but the final Clock, frozen RPC, and replay harness shape should be chosen as a public SDK/runtime design coordinated with the related gateway accessor work.
