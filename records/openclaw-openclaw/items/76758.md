## Summary

Keep open. Current main still has provider transcript replay policy and live plugin tool execution, but no host-side Clock, frozen plugin tool replay, or frozen gateway RPC snapshot contract; this is public SDK/runtime design work rather than conservative cleanup.

## What This Changes

Request a host-side plugin tool-call replay contract with SDK Clock injection, frozen host RPC snapshots, and reusable record/replay test helpers.

## Best Possible Solution

Keep this open for a maintainer design pass that defines a versioned plugin replay contract covering runtime clock injection, gateway RPC snapshotting, and plugin test helpers.

## Reproduction Assessment

Not applicable; this is an RFC-shaped feature request, not a broken existing behavior with an established contract. Source and docs inspection confirm the requested host-side replay contract is absent on current main.

## Solution Assessment

Unclear; the Clock plus frozen RPC direction is plausible, but the narrow maintainable path should be chosen as a public SDK/runtime design coordinated with #76756 and #76759.
