## Summary

Keep open. Current main still lacks a host-side Clock, frozen plugin RPC snapshot contract, or reusable plugin tool-call replay harness, and the request is a new public SDK/runtime design tied to the still-open gateway accessor work.

## What This Changes

Request a standard plugin-sdk Clock, frozen host RPC results during replay, and a reusable record/replay harness for deterministic plugin tool outputs.

## Best Possible Solution

Keep this as an SDK design/RFC item and define a versioned replay contract alongside the gateway RPC accessor and public type export work.

## Reproduction Assessment

Not applicable; this is an RFC-shaped feature request, not a broken existing behavior with an established contract. Source and docs inspection confirm the requested host-side Clock and frozen RPC replay contract is absent on current main.

## Solution Assessment

Unclear; the Clock plus frozen RPC direction is plausible, but the final shape should be chosen with public SDK versioning and #76756 rather than as an isolated small fix.
