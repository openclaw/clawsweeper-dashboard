## Summary

Keep open. Current main still lacks a host-side Clock, frozen RPC snapshot replay path, or plugin tool output record/replay harness; the existing replay surfaces are provider transcript policy, so this is public SDK/runtime design work rather than conservative cleanup.

## What This Changes

Request a public plugin SDK/runtime contract for deterministic plugin tool-call replay, including Clock injection, frozen host RPC snapshots, and a reusable record/replay harness.

## Best Possible Solution

Keep this open for a maintainer design pass that defines a versioned plugin replay contract coordinated with gateway accessors and public SDK exports.
