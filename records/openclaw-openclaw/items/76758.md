## Summary

Keep open. Current main still lacks a host-side Clock or frozen RPC snapshot replay contract for plugin tool calls; the existing replay surfaces are provider transcript policies, so this is public SDK/runtime design rather than conservative cleanup.

## What This Changes

The issue requests a deterministic plugin tool-call replay contract covering Clock injection, frozen host RPC snapshots during replay, and a reusable record/replay test harness.

## Best Possible Solution

Keep this open for a maintainer design pass that defines a versioned plugin replay contract coordinated with gateway accessors, SDK exports, and regression harness expectations.

## Reproduction Assessment

Not applicable; this is an RFC-shaped feature request, not a broken existing behavior with an established contract. Source and docs inspection confirm the requested host-side replay contract is absent on current main.

## Solution Assessment

Unclear; the direction is plausible, but the Clock, frozen RPC, and byte-replay harness shape should be chosen as public SDK/runtime design rather than an isolated small fix.
