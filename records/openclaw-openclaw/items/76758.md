## Summary

Keep open. Current main still lacks a host Clock or frozen RPC snapshot contract for plugin tool-call replay, and the request is a new public plugin SDK/runtime design tied to #76756 rather than conservative cleanup.

## What This Changes

The issue requests deterministic plugin tool-call replay via SDK Clock injection, frozen host RPC snapshots, and reusable record/replay test helpers.

## Best Possible Solution

Define a versioned plugin replay contract after settling gateway RPC access, then add SDK types, host injection, recording semantics, docs, and contract tests together.

## Reproduction Assessment

Not applicable; this is an RFC-shaped feature request, not a broken existing behavior with an established contract. Source and docs inspection show the requested host-side replay contract is absent today.

## Solution Assessment

Unclear; Clock injection is plausible, but the frozen RPC and replay harness shape should be chosen as a public SDK/runtime API design coordinated with #76756, not as an isolated helper export.
