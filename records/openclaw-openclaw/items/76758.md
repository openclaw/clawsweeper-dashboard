## Summary

Keep open. Current main still lacks a host-side Clock, frozen gateway RPC snapshot, or tool-output replay contract for plugin tool calls; this is public SDK/runtime design rather than conservative cleanup.

## What This Changes

Request a plugin SDK/runtime contract for deterministic plugin tool-call record/replay, including Clock injection, frozen gateway RPC results, and reusable replay tests.

## Best Possible Solution

Keep this open for a maintainer design pass that defines an additive, versioned plugin replay contract with SDK types, runtime wiring, docs, and focused contract tests.

## Reproduction Assessment

Not applicable; this is an RFC-shaped feature request, not broken behavior under an existing contract. Source and docs inspection confirm the requested host-side replay contract is absent on current main.

## Solution Assessment

Unclear; the direction is plausible, but standardizing Clock injection, frozen gateway RPC snapshots, and byte-replay tests should be handled as public SDK/runtime design rather than a small cleanup fix.
