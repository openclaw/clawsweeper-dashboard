## Summary

Keep open. Current main still lacks a host-side Clock/frozen RPC replay contract for plugin tool calls; existing replay support is provider transcript policy, so this is public SDK/runtime design rather than conservative cleanup.

## What This Changes

Request to add a public host-side plugin replay contract covering Clock injection, frozen gateway RPC snapshots, and a reusable record/replay tool-output harness.

## Best Possible Solution

Define a versioned plugin runtime replay contract together with gateway accessors/types, then implement SDK exports, runtime injection, snapshot recording/replay, docs, and contract tests.

## Reproduction Assessment

Not applicable; this is an RFC-shaped feature request, not a broken behavior with an established current contract. Source and docs inspection show the requested host-side replay contract is absent on current main.

## Solution Assessment

Unclear; the direction is plausible, but the Clock, frozen RPC, and byte-replay harness shape should be chosen through maintainer SDK/runtime design rather than a narrow cleanup patch.
