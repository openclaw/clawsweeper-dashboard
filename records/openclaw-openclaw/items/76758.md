## Summary

Keep open: current main and v2026.6.10 still lack a host-side deterministic plugin tool-call replay contract. The remaining work changes public Plugin SDK, plugin runtime injection, Gateway replay semantics, and security boundaries, so it is still necessary but needs maintainer product/security review before implementation.

## What This Changes

The issue requests a Plugin SDK replay contract with host-injected clocks, frozen recorded Gateway RPC observations, and reusable plugin replay tests.

## Best Possible Solution

Design a narrow plugin SDK replay contract that provides deterministic time and recorded read-only host observations, with docs and contract tests, after maintainers choose the Gateway allowlist and replay/security boundary.

## Reproduction Assessment

Yes, at source level. Current main lacks a PluginRuntime clock/replay snapshot contract while the live Gateway dispatcher is route-scoped and not replay-frozen; I did not run a replay scenario because the missing surface is an absent SDK feature.

## Solution Assessment

Unclear until maintainers choose the public SDK and security boundary. The proposed Clock plus frozen-snapshot direction is plausible, but it should be designed with the related Gateway RPC and type-export issues rather than queued as a blind fix.
