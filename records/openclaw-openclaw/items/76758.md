## Summary

Keep open. Current main still lacks a host-side Clock, frozen gateway-RPC snapshot, or plugin tool-call replay contract; the existing replay surface is provider transcript policy, and the provided discussion links this request to still-open gateway accessor and SDK type-export design work.

## What This Changes

Request standardizes a host-injected plugin Clock, frozen replay snapshots for curated gateway RPCs, and a reusable byte-stable plugin tool replay harness.

## Best Possible Solution

Keep this open for a maintainer design pass that defines a versioned plugin replay contract coordinated with gateway accessors and public SDK type exports before implementation.

## Reproduction Assessment

Not applicable; this is an RFC-shaped feature request rather than broken behavior under an existing contract. Source and docs inspection confirm the requested host-side replay contract is absent on current main.

## Solution Assessment

Unclear; the direction is plausible, but the Clock, frozen RPC, and replay harness shape should be chosen as a public SDK/runtime design rather than a small autonomous fix.
