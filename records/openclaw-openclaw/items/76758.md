## Summary

Keep open. Current main still lacks a host-side Clock or frozen host-RPC snapshot contract for deterministic plugin tool-call replay; the existing replay surfaces are provider transcript policy, SDK event replay, or test timer helpers rather than plugin tool output replay semantics.

## What This Changes

The issue requests a public SDK/runtime contract for deterministic plugin tool-call replay, including Clock injection, frozen host RPC results, and reusable record/replay harness coverage.

## Best Possible Solution

Keep this open for a maintainer-led SDK/runtime design that defines versioned replay semantics and coordinates Clock injection with curated gateway accessors and plugin replay tests.

## Reproduction Assessment

Not applicable; this is an RFC-shaped feature request rather than broken behavior under an existing contract. Source and docs inspection confirm the requested host-side replay contract is absent on current main.

## Solution Assessment

Unclear; the proposed direction is plausible, but the specific Clock, frozen RPC, and byte-replay harness design should be chosen as a public SDK/runtime contract rather than a narrow bug fix.
