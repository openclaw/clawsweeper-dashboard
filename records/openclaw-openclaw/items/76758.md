## Summary

Keep open. Current main still lacks a host-side Clock or frozen RPC snapshot replay contract for plugin tool calls; existing replay support is provider transcript policy, and plugin tool execution remains live, so this is public SDK/runtime design rather than conservative cleanup.

## What This Changes

The issue requests a host-side plugin SDK/runtime contract for deterministic plugin tool-call replay using injected clocks, frozen host RPC snapshots, and reusable replay tests.

## Best Possible Solution

Keep this open for a maintainer design pass that defines a versioned plugin replay contract coordinated with gateway accessors and public SDK type exports before implementation.

## Reproduction Assessment

Not applicable; this is an RFC-shaped feature request, not a broken existing behavior with an established contract. Source and docs inspection confirm the requested host-side replay contract is absent on current main.

## Solution Assessment

Unclear; the direction is plausible, but the specific Clock, frozen RPC, and byte-replay harness shape should be chosen as a public SDK/runtime design rather than an isolated small fix.
