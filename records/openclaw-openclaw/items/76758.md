## Security Review

Status: needs_attention

Summary: The issue itself is security-sensitive because a plugin-visible Gateway replay surface could expose host state or control methods without a narrow allowlist and recorded-value contract.

Concerns:

- **[medium] Constrain replayed Gateway access:** `src/gateway/methods/core-descriptors.ts:21`
  - body: The core Gateway method table includes read, write, and admin methods, so any plugin replay contract should expose only intentional read-only observations and should not hand plugin tools raw Gateway dispatch.

  - body: The existing plugin Gateway dispatcher is scoped to authenticated HTTP routes and live in-process dispatch; using that directly for tool-call replay would not provide frozen recorded responses and would blur the plugin privilege boundary.
