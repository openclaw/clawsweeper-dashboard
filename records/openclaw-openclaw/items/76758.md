## Summary

Keep open: current main and v2026.6.8 still lack a host-side deterministic plugin tool-call replay contract, and the remaining work is a public plugin SDK plus Gateway security design decision rather than cleanup.

## What This Changes

The issue requests a host-side deterministic replay contract for plugin tool calls, including plugin SDK clock injection, frozen host RPC snapshots, and replay harness coverage.

## Best Possible Solution

Design a narrow plugin SDK replay contract that gives plugin tools deterministic time and recorded read-only host observations, with docs and contract tests, after maintainers choose the Gateway allowlist and replay/security boundary.

## Reproduction Assessment

Yes at source level: the current plugin runtime and live Gateway dispatcher lack a deterministic plugin tool-call replay clock and frozen snapshot contract. I did not run a replay scenario because this is an absent SDK feature rather than a broken documented runtime behavior.

## Solution Assessment

Unclear until maintainers choose the public SDK and security boundary. The proposed Clock plus frozen-snapshot direction is plausible, but it should be designed with the related Gateway RPC and type-export issues instead of queued as a blind fix.
