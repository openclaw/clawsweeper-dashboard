## Summary

Keep open. Current main still lacks a host-side plugin tool-call replay contract; existing replay support is provider transcript policy, while plugin tools and gateway RPC reads remain live runtime behavior, so this needs public SDK/runtime design rather than conservative cleanup.

## What This Changes

Request a public plugin SDK/runtime contract for deterministic plugin tool-call replay via injected Clock, frozen gateway RPC snapshots, and a reusable record/replay test harness.

## Best Possible Solution

Keep this open for a versioned plugin replay design that coordinates Clock injection, recorded tool/RPC snapshots, SDK exports, docs, and plugin test harness coverage.

## Reproduction Assessment

Not applicable; this is an RFC-shaped feature request, not a broken behavior with an established current contract. Source and docs inspection show the requested host-side Clock and frozen plugin tool/RPC replay contract are absent on current main.

## Solution Assessment

Unclear; the direction is plausible, but the exact Clock, snapshot, and RPC accessor semantics should be chosen as public SDK/runtime design, likely coordinated with #76756 and #76759.
