## Summary

Keep open. Current main still has provider transcript replay helpers and live plugin tool execution, but no host-side Clock, frozen RPC snapshot, or plugin tool-call byte replay contract; this is public SDK/runtime design work tied to the still-open gateway RPC accessor proposal, not conservative cleanup.

## What This Changes

Request a public SDK/runtime contract for deterministic plugin tool-call replay, including host-injected clocks, frozen host RPC snapshots, and replay test helpers.

## Best Possible Solution

Define a versioned plugin replay contract in the SDK/runtime layer, then align docs and tests with the gateway RPC accessor work so plugin authors get one deterministic pattern instead of per-plugin conventions.

## Reproduction Assessment

Not applicable; this is an RFC-shaped feature request, not a broken existing behavior with an established contract. Source and docs inspection confirm the requested host-side replay contract is not present today.

## Solution Assessment

Unclear; the direction is plausible, but the Clock plus frozen RPC shape should be chosen through public SDK/runtime design with #76756 rather than implemented as an isolated small fix.
