## Summary

Keep open. Current main still lacks a host-side Clock, replay mode, frozen plugin RPC snapshot contract, or reusable plugin tool-output replay harness, so this remains a public SDK/runtime design request rather than conservative cleanup.

## What This Changes

The issue requests a public SDK/runtime contract for deterministic plugin tool-call replay using a host-injected Clock and frozen host RPC snapshots.

## Best Possible Solution

Keep this open for a maintainer design pass that defines a versioned plugin replay contract and coordinates it with the gateway RPC accessor and public SDK type work.

## Reproduction Assessment

Not applicable; this is an RFC-shaped feature request, not a broken current contract. Current source and docs inspection confirm the requested host-side Clock and frozen RPC replay surface is not present today.

## Solution Assessment

Unclear; the proposed Clock plus frozen RPC direction is plausible, but the best shape should be chosen with public SDK versioning and #76756 rather than implemented as an isolated small fix.
