## Best Possible Solution

Land the decoupling with all policy expectations aligned and a changelog/docs note telling operators to deny `apply_patch` explicitly when they want patch writes blocked.

## Solution Assessment

No, not yet. Removing the hidden deny coupling is the narrow code direction, but the branch must update remaining policy expectations and document the permission-widening behavior before merge.
