## Summary

Keep open: the PR targets a real Codex app-server OAuth harness path and has sufficient live proof, but the current branch is not mergeable as written because its runner change passes a broader merged auth store into plugin-owned harnesses. Current main already landed the safer scoped-store projection in https://github.com/openclaw/openclaw/pull/80017, so this PR should be repaired to preserve that boundary while carrying over any remaining app-server lookup and live-harness changes.

## What This Changes

The branch scopes Codex app-server OAuth profile lookup to Codex external CLI providers, forwards harness auth context from the embedded runner, and makes live Codex harness command probes opt-in.

## Best Possible Solution

Preserve current main's one-profile auth-store projection, add scoped Codex app-server external CLI lookup only at the app-server boundary, and keep the live command probe opt-in with targeted regression coverage.

## Reproduction Assessment

Yes. Source inspection shows the app-server bridge still loads auth profiles without explicit external CLI scope, while external CLI sync only imports CLI credentials by default when the matching profile already exists; I did not live-run the harness in this read-only review.

## Solution Assessment

No, not as written. The scoped app-server lookup is a maintainable direction, but the runner hunk should preserve current main's scoped projection instead of passing a full merged auth store to plugin-owned harnesses.
