## Summary

Keep this PR open: current main still lacks the rich-method-unavailable fallback, but this PR head is not merge-ready because long legacy preview fallback can still hide the suffix, long rich edit fallback is still unbounded for legacy edit limits, and current-head real Telegram proof is missing.

## What This Changes

The PR adds Telegram fallbacks from rich Bot API send, edit, and draft-preview paths to legacy sendMessage/editMessageText when rich methods are unavailable.

## Best Possible Solution

Land one Telegram-plugin fallback that preserves complete final text, bounds legacy edit behavior, keeps rich as the preferred path, and includes redacted current-head Telegram proof for the rich-method-unavailable path.

## Reproduction Assessment

Yes, source-level: enable Telegram richMessages, make sendRichMessage unavailable, and stream more than 4096 characters. PR-head source then caps the legacy preview while recording the full text as delivered, so stop() can skip the hidden suffix.

## Solution Assessment

No as-is. The Telegram plugin is the right boundary, but the patch must treat capped legacy previews as partial delivery, bound or explicitly scope legacy edit fallback, and refresh current-head real Telegram proof before it is the best fix.
