Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve suffix text after capped legacy previews:** `extensions/telegram/src/draft-stream.ts:127-151`
  - body: When rich previews fall back to legacy delivery, this helper caps the visible text to 4096 characters, but the surrounding stream code records the full preview as delivered after the send/edit succeeds. A long reply can then finish without ever sending the suffix; keep delivery state aligned with the text Telegram actually received or force a continuation for the remaining text.
  - confidence: 0.94
- **[P1] Narrow the rich-method-unavailable matcher:** `extensions/telegram/src/network-errors.ts:294-295`
  - body: This regex treats any `Call to 'sendRichMessage' failed! (400:` as method-unavailable before checking for missing-method text. Rich validation errors such as invalid rich URLs can be retried through legacy delivery, and if a later rich chunk fails the full legacy retry can duplicate already delivered content; match only actual missing/unsupported method errors and add negative tests.
  - confidence: 0.91
- **[P2] Define the long rich-edit fallback:** `extensions/telegram/src/send.ts:1682`
  - body: When rich edit is unavailable, the fallback sends the already-built full text through legacy `editMessageText`, but legacy Telegram text edits are capped at the shorter text limit. Long rich edits can still fail here; scope the fallback to short edits, deliberately truncate, or switch to a follow-up send path with tests.
  - confidence: 0.86
