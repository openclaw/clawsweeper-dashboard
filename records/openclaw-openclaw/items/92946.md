- **[P1] Preserve the suffix after capped legacy previews:** `extensions/telegram/src/draft-stream.ts:139`
  - body: When rich previews fall back to legacy delivery, `capTelegramLegacyTransportPreview` keeps only the first 4096-character chunk, but the caller records the full text as delivered. A long stream under the rich limit can therefore stop with the suffix never sent; keep delivery accounting aligned with what was actually visible or force finalization for the remaining text.
  - confidence: 0.94
- **[P2] Narrow the rich-unavailable matcher:** `extensions/telegram/src/network-errors.ts:295`
  - body: `RICH_METHOD_UNAVAILABLE_RE` matches `Call to 'sendRichMessage' failed! (400:` before checking for a missing-method response, so any 400 from sendRichMessage can be treated as an unavailable API and retried through legacy delivery. Restrict this to actual method-missing/not-found rich-method errors and add negative tests for non-method 400s.
  - confidence: 0.9
- **[P2] Define the long rich-edit fallback path:** `extensions/telegram/src/send.ts:1682`
  - body: When rich edit is unavailable, the fallback sends the full rich-sized text through legacy `editMessageText`, whose `text` parameter is capped at 4096 characters after entity parsing. Long rich edits can still fail unless the PR scopes edits to short text, truncates deliberately, or switches to a follow-up send path.
  - confidence: 0.86
