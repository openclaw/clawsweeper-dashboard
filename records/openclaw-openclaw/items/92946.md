Full review comments:

- **[P1] Preserve the suffix after capped legacy previews:** `extensions/telegram/src/draft-stream.ts:139`
  - body: When rich previews fall back to legacy delivery, this helper truncates the preview to 4096 characters, but the caller still records the full text as delivered after the send. A long stream under the rich limit can therefore stop with the suffix never sent; keep delivery accounting aligned with what was actually visible or force finalization for the remaining text.
  - confidence: 0.94
- **[P2] Narrow the rich-unavailable matcher:** `extensions/telegram/src/network-errors.ts:295`
  - body: This regex treats any `Call to 'sendRichMessage' failed! (400:` as method-unavailable before checking for missing-method text. A non-method rich validation error can be silently retried through legacy delivery, so narrow it to actual method-missing/not-found errors and add negative tests for ordinary 400s.
  - confidence: 0.9
- **[P2] Define the long rich-edit fallback path:** `extensions/telegram/src/send.ts:1682`
  - body: When rich edit is unavailable, the fallback sends the already-built rich-sized text through legacy `editMessageText`, but Telegram's legacy text edit is capped at 4096 characters after entity parsing. Long rich edits can still fail; scope the fallback to short edits, deliberately truncate, or switch to a follow-up send path.
  - confidence: 0.86
