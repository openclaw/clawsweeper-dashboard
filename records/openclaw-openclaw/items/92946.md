Full review comments:

- **[P1] Preserve the suffix after capped legacy previews:** `extensions/telegram/src/draft-stream.ts:127-151`
  - body: When rich previews fall back to legacy delivery, the new cap helper sends only the first 4096-character legacy preview, but the caller records the full text as delivered after the send succeeds. A long stream can then stop without sending the suffix; keep delivery accounting aligned with what Telegram actually received or force finalization for the remaining text.
  - confidence: 0.94
- **[P2] Narrow the rich-unavailable matcher:** `extensions/telegram/src/network-errors.ts:294-295`
  - body: This regex treats any `Call to 'sendRichMessage' failed! (400:` as method-unavailable before checking for missing-method text. A normal rich validation error can be retried through legacy delivery, and if a later rich chunk fails the full fallback can duplicate already delivered content; narrow it to actual missing-method/not-found errors and add negative tests.
  - confidence: 0.91
- **[P2] Define the long rich-edit fallback path:** `extensions/telegram/src/send.ts:1682`
  - body: When rich edit is unavailable, the fallback sends the already-built rich-sized text through legacy `editMessageText`, but legacy Telegram text edits are bounded by the shorter text limit. Long rich edits can still fail; scope the fallback to short edits, deliberately truncate, or switch to a follow-up send path.
  - confidence: 0.86
