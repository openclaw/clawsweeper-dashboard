  - body: The PR reads `config.tui.showTimestamps`, but current config validation is strict and has no top-level `tui` schema.   - confidence: 0.96

  - confidence: 0.9
- **[P2] Render stored message times for history:** `src/tui/components/chat-log.ts:9-14`
  - body: `formatTimestamp()` always uses the current local clock, while `chat.history` entries already carry timestamps and the TUI replay path currently passes only text into ChatLog. Reopened sessions would show reload time instead of message time unless the renderer accepts and formats the stored timestamp.
  - confidence: 0.86
