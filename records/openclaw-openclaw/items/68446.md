## Summary

Keep this PR open and do not merge as-is. The branch addresses the main WhatsApp group-policy bypass direction, but the same DM-to-group authorization boundary still appears in the command authorization path, and the PR discussion already includes a later security review flagging that remaining gap.

## What This Changes

This PR changes WhatsApp inbound policy resolution to stop using DM `allowFrom` as the resolved group allowlist for the conversation-policy bypass and adds regression coverage for fail-closed group allowlist behavior.

## Best Possible Solution

Carry the strict WhatsApp group sender boundary through both normal group gating and command authorization, add a focused group-command regression test, and update the WhatsApp access-control docs if the `allowFrom` fallback is intentionally retired for this channel.

## Reproduction Assessment

Yes. A focused source-level reproduction can call `resolveWhatsAppCommandAuthorized` for a group control command with `groupPolicy: "allowlist"`, DM `allowFrom`, and no `groupAllowFrom`; the current call passes `policy.dmAllowFrom` into the shared command gate without disabling group fallback.

## Solution Assessment

No. The proposed patch is the right direction for the conversation-policy sender-filter bypass, but it is not complete because command authorization still uses the shared fallback-sensitive helper without the stricter WhatsApp group boundary.

## Review Findings
