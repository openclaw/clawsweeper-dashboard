## Summary

Keep this PR open and do not merge as-is. The branch fixes the main WhatsApp group auto-reply sender-filter bypass, but the same DM-to-group fallback remains in command authorization, and a bot review already flagged that after the follow-up commit. Reviewed item: https://github.com/openclaw/openclaw/pull/68446

## What This Changes

This PR changes WhatsApp inbound policy resolution so DM `allowFrom` entries no longer seed group allowlists and adds regression coverage for group allowlist fail-closed behavior.

## Best Possible Solution

Carry the WhatsApp-specific strict group sender boundary through both normal group gating and command authorization: pass `groupAllowFromFallbackToAllowFrom: false` into `resolveDmGroupAccessWithCommandGate`, add a focused group-command regression test, and update the WhatsApp access-control docs if the fallback is intentionally retired for this channel.

## Reproduction Assessment

Yes. By source inspection, a group command with `groupPolicy: "allowlist"`, a DM `allowFrom` entry, no `groupAllowFrom`, and a sender matching that DM entry still reaches `resolveDmGroupAccessWithCommandGate` without the fallback-disabled flag, so the helper can authorize it through the DM-derived group list.

## Solution Assessment

No. The proposed fix is the right direction for `resolveConversationGroupPolicy`, but it is not complete because the command authorization path uses the same shared fallback contract and remains on the default fallback behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Disable DM fallback for command authorization:** `extensions/whatsapp/src/inbound-policy.ts:103`
  - body: This only disables `allowFrom` fallback for the conversation-policy list. `resolveWhatsAppCommandAuthorized` below still calls `resolveDmGroupAccessWithCommandGate` without `groupAllowFromFallbackToAllowFrom: false`, so when `groupPolicy` is `allowlist` and `groupAllowFrom` is absent, a group sender matching a DM `allowFrom` entry can still authorize control commands from an otherwise unallowlisted group. Pass the same flag into the command gate and cover that group-command case.
  - confidence: 0.91
