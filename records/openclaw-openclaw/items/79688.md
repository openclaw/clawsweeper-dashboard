## Summary

Close as implemented: current main drops empty Discord message events before dispatch and strips stale trailing assistant prefill turns at the provider boundary, so the reported assistant-tail/no-user-payload failure no longer reaches Anthropic-class transports.

## What This Changes

Prevent Discord inbound events with no user text from driving an infinite retry loop when the session transcript ends on an assistant turn for prefill-strict models.

## Best Possible Solution

Keep the shipped provider-boundary assistant-tail sanitizer plus Discord empty-inbound guards; affected 2026.5.2 installs should upgrade to a release containing the fix.

## Reproduction Assessment

Yes, at source/test level: the regression test covers an Anthropic replay ending in an assistant turn, and the issue logs match the same provider rejection. I did not run the reporter's live Discord/custom-provider setup.

## Solution Assessment

Yes. Stripping stale assistant tails at the provider boundary is the narrow durable fix because it covers Discord reactions, system events, runtime-context short circuits, and other no-user-turn paths, while Discord also now drops empty message events before dispatch.
