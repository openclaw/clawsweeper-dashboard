## Summary

Keep open: current main still lacks the requested transient-error context, and the PR is useful, but the proposed patch has blocking regressions around failover classification and chat-visible request IDs.

## What This Changes

The branch enriches pi-embedded overloaded, rate-limit, and transport error formatting with provider/model/profile/trigger/session/request context and threads profile/trigger through runner payload construction and tests.

## Best Possible Solution

Land a rebased context-enrichment implementation that preserves existing transient failover behavior, omits or hashes provider request IDs in chat-visible copy, restores focused regression coverage, and adds a user-facing changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows the requested context is absent, and the provided PR diff plus existing tests give a high-confidence path for the failover and request-id regressions without needing a live provider.

## Solution Assessment

No. The direction is useful, but this patch is not the best solution until it preserves broad transient api_error classification and follows the existing request-id redaction policy for user-facing output.
