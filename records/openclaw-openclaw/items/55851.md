## What This Changes

This PR enriches pi-embedded rate-limit, overload, and transport error messages with provider/model/profile/trigger/session/request context and threads profile/trigger through runner payload formatting while loosening related assertions.

## Best Possible Solution

Land or replace this with a rebased shared-formatter change that adds sanitized active provider/model/profile/trigger context while preserving broad transient failover and request-id privacy.

Yes for the PR blockers by source inspection: PR head appends raw req= values and narrows api_error retry matching, while current-main tests assert request-id suppression and broader transient classification. I did not run live provider or Telegram proof.

## Solution Assessment

No, not as-is. The maintainable path is a rebased shared formatter implementation that omits raw request IDs, preserves retry/failover classification, and includes real Telegram or equivalent runtime proof.
