## Summary

Keep open. Current main and the latest release still expose `session_status` as rendered status text plus minimal metadata, while the merged context-budget work is adjacent reporting-only work and does not provide the requested structured session, last-turn, token, cost, and budget usage API.

## What This Changes

Request to expose structured per-session, last-turn, cost, and budget token usage through `session_status` so agents can budget work programmatically.

## Best Possible Solution

Define an additive documented agent-facing usage contract on `session_status.details` or a dedicated usage/budget tool, reusing transcript and gateway usage sources while preserving source, availability, cost, and privacy semantics.

## Reproduction Assessment

Yes as a source-level feature-gap check, not as a bug reproduction. Current main and `v2026.5.22` accept only `sessionKey` and `model` and return rendered `statusText` without a structured `usage` object.

## Solution Assessment

Unclear until maintainers choose the public API shape and privacy semantics. Extending `session_status.details` is plausible, but a separate usage or budget tool may be cleaner if display status and machine-readable accounting should stay separate.
