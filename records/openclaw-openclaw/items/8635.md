## Summary

Keep open. Current main and the latest release still expose `session_status` as rendered status text plus minimal metadata; structured token, cost, session, last-turn, and budget/window usage data remains outside the agent-facing tool contract.

## What This Changes

The issue requests a structured token and cost usage API on the agent-facing `session_status` tool.

## Best Possible Solution

Define an additive documented agent-facing usage contract, either under `session_status.details.usage` or a dedicated usage tool, backed by transcript/session usage data and explicit availability/source metadata while preserving current visibility guards.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection give a high-confidence current-behavior check: `session_status` returns rendered `statusText` and does not expose the requested structured token/cost usage object.

## Solution Assessment

Unclear until maintainers choose the public API shape and privacy semantics. The requested structured usage direction is maintainable, but the exact fields, source precedence, and visibility rules need product/API review before implementation.
