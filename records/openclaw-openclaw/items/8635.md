## Summary

Keep open. Current main and the latest checked release still expose `session_status` as rendered status text plus minimal metadata, while the requested structured session, last-turn, token, cost, and budget usage contract is absent; adjacent usage issues do not supersede this broader agent-facing API request.

## What This Changes

Expose structured per-session and last-turn token, cost, and budget usage data through `session_status` so agents can budget work without parsing transcript JSONL.

## Best Possible Solution

Define an additive documented agent-facing usage contract, likely `session_status.details.usage` or an equivalent supported tool, backed by transcript usage, `sessions.usage`, and provider-window data while preserving existing session visibility and privacy checks.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source and docs inspection provide a high-confidence current-behavior check: `session_status` returns rendered `statusText` and no structured `details.usage`.

## Solution Assessment

Unclear until maintainers choose the public schema and privacy boundary. A stable sanitized usage payload is the maintainable direction; parsing transcript JSONL or rendered status text should remain a workaround, not the contract.
