## Summary

Keep open: current main still exposes session_status as a rendered status-card tool result, while raw session, last-turn, cost, and quota/budget usage data remains in separate Gateway/status/transcript surfaces. The merged https://github.com/openclaw/openclaw/pull/84830 and latest release add adjacent context-budget display, but not the broader agent-facing structured usage API requested here.

## What This Changes

Expose raw per-session and last-turn token, cost, and budget usage data through the agent-facing session_status tool.

## Best Possible Solution

Define a documented structured agent-facing usage surface under existing session visibility checks, with explicit measured, estimated, stale, unavailable, source, cost, and quota semantics for session totals and last-turn usage.

## Reproduction Assessment

Yes. At source level, current session_status schema and returned details have no structured usage object, while structured usage data exists under separate Gateway usage RPCs and rendered status text.

Unclear. The requested capability is reasonable, but maintainers still need to choose the stable response schema, freshness rules, cost-source semantics, quota-window availability metadata, and privacy boundaries before implementation.
