## Summary

Keep open: current main still lacks a structured agent-facing `session_status` usage contract; token/cost data is either rendered into status text or exposed through separate operator gateway surfaces. The related budget request at https://github.com/openclaw/openclaw/issues/57053 is narrower and does not supersede this broader session and last-turn usage API request.

## What This Changes

Expose per-session, last-turn, cost, and budget token usage data through the `session_status` tool so agents can programmatically track capacity.

## Best Possible Solution

Define an additive documented agent-facing usage contract backed by `sessions.usage` and transcript/provider usage parsers, with explicit source/unavailable metadata and existing visibility checks preserved.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-behavior check: `session_status` returns rendered `statusText` and minimal metadata, while structured usage is only available on separate gateway/operator surfaces or as display text.

## Solution Assessment

Unclear until maintainers choose the public API contract. Reusing `sessions.usage` and transcript/provider usage parsers with visibility and availability metadata is safer than asking agents to parse status text or JSONL files directly.
