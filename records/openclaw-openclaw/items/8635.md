## Summary

Keep open: current main and the latest release still expose `session_status` as rendered status text plus minimal metadata, while structured token/cost/session usage remains on separate gateway usage surfaces. The discussion and the closed related report at https://github.com/openclaw/openclaw/issues/39400 make this the canonical API/product decision for an agent-facing usage contract.

## What This Changes

The issue requests adding raw per-session, last-turn, cost, and quota/budget usage fields to the agent-facing `session_status` tool.

## Best Possible Solution

Define an additive documented agent-facing usage contract, backed by `sessions.usage` and transcript parsers with explicit source/unavailable metadata and existing visibility checks preserved.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives a high-confidence current-behavior check: `session_status` has no structured usage fields, while raw usage data is available through separate gateway RPCs.

## Solution Assessment

Unclear until maintainers choose the public API contract. Reusing `sessions.usage` with visibility and availability metadata is safer than asking agents to parse rendered status text or transcript files directly.
