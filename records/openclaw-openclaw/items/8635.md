## Summary

Keep open: current main and v2026.6.6 still expose `session_status` as a rendered `/status`-style result, while structured session, last-turn, cost, and quota/budget usage data remains on separate status, transcript, and Gateway usage surfaces.

## What This Changes

Expose raw per-session, last-turn, cost, and quota/budget usage data through the agent-facing `session_status` tool.

## Best Possible Solution

Define a documented structured agent-facing usage surface under the existing session visibility checks, with explicit measured, estimated, stale, unavailable, and source metadata for session totals, last-turn usage, costs, and quota windows.

## Reproduction Assessment

Yes, at source level: current `session_status` schema and return details can be inspected to show structured usage fields are absent while related raw usage data exists only as rendered status text or separate Gateway methods.

## Solution Assessment

Unclear. The requested capability is reasonable, but the sample response is not yet a proven best design because maintainers still need to choose freshness, privacy, quota, last-turn, and cost-source semantics for a stable public contract.
