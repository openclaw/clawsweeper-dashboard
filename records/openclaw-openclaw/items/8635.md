## Summary

Keep open: current main and v2026.5.12 still lack a structured agent-facing usage payload on `session_status`; the reusable structured data exists on separate gateway usage RPCs, and the related open issue is narrower rather than superseding this broader API request.

## What This Changes

The issue requests structured token, cost, last-turn, and budget usage data in the agent-facing `session_status` tool.

## Best Possible Solution

Define an additive documented agent-facing usage contract, backed by `sessions.usage` and transcript/provider usage parsers with explicit source/unavailable metadata and existing visibility checks preserved.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives a high-confidence current-behavior check: `session_status` returns rendered status text and minimal metadata, while structured usage data is only available on separate gateway RPCs.

## Solution Assessment

Unclear until maintainers choose the public API contract. Reusing `sessions.usage` and transcript/provider usage parsers with visibility and availability metadata is safer than asking agents to parse status text or JSONL files directly.
