## Summary

Keep open. Current main still exposes `session_status` as rendered status text plus minimal metadata, while structured session/last-turn token and cost data remains on separate Gateway usage RPCs rather than an agent-facing `session_status` contract.

## What This Changes

Expose raw per-session, last-turn, cost, and budget usage data through the agent-facing `session_status` tool.

## Best Possible Solution

Define an additive documented structured usage contract on `session_status.details` or an equivalent agent-facing tool, backed by the existing `sessions.usage` and transcript usage parsers while preserving current visibility and privacy checks.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection give a high-confidence current-behavior check: `session_status` returns rendered `statusText` and does not expose the requested structured token/cost/budget fields.

## Solution Assessment

Unclear until maintainers choose the public API shape and semantics. Structured usage data is a maintainable direction, but this should be an additive documented contract rather than an automatic local patch.
