## Summary

Keep open. Current main still exposes the agent-facing status tool as a rendered status-card result with minimal structured details, while raw token, cost, last-turn, and provider-window data remain text-only or available only through separate gateway usage surfaces.

## What This Changes

The issue requests structured per-session token, cost, last-turn, and budget usage data through the agent-facing `session_status` tool.

## Best Possible Solution

Define a documented additive `session_status.details.usage` contract, or an equivalent supported agent tool, that reuses transcript, `sessions.usage`, and provider-usage data while preserving visibility checks and availability metadata.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection gives a high-confidence current-behavior check: `session_status` returns rendered `statusText` and does not expose structured `details.usage`.

## Solution Assessment

Unclear until maintainers choose the public schema and privacy boundary. A stable sanitized usage payload is more maintainable than requiring agents to parse session JSONL or rendered status text directly.
