## Summary

Keep open. Current `main` and the latest release still expose `session_status` as rendered status text plus minimal metadata, while raw session, last-turn, cost, and budget usage remain unavailable through a documented agent-facing structured contract; the open context-budget PR is adjacent but does not close this broader request.

## What This Changes

The issue requests structured token, cost, last-turn, and budget usage data through the agent-facing `session_status` tool.

## Best Possible Solution

Define an additive documented agent-facing usage contract, either on `session_status.details` or a dedicated usage/budget tool, backed by visibility-aware session/gateway usage data with explicit source, availability, cost, and budget semantics.

## Reproduction Assessment

Yes as a source-level behavior check, not as a bug reproduction: current `main` and `v2026.5.20` accept only `sessionKey`/`model` and return rendered `statusText` without a structured usage object.

## Solution Assessment

Unclear until maintainers choose the public API shape and privacy semantics. Extending `session_status.details` is plausible, but a separate usage or budget tool may be cleaner if maintainers want to separate display status from machine-readable accounting.

## Risks / Open Questions

- The public API shape still needs maintainer judgment because session totals, last-turn usage, provider quota windows, context headroom, user-defined budgets, and elapsed/remaining time are related but distinct contracts.
- Exposing transcript totals, costs, or provider quota metadata to agents must preserve existing session visibility, sandbox, cross-agent access, and privacy boundaries.
- Provider quota windows can be unavailable, auth-dependent, or source-dependent, so a stable API needs explicit availability/source metadata rather than silent omission.
- Adjacent open work such as https://github.com/openclaw/openclaw/issues/57053 and https://github.com/openclaw/openclaw/pull/84830 should be coordinated so context-budget and token-usage fields do not grow conflicting meanings.
