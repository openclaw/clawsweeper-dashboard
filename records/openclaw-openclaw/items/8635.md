## Summary

Keep open. Current main and v2026.5.20 still expose `session_status` as rendered status text with minimal metadata, while machine-readable session, last-turn, cost, and budget usage remains outside the agent-facing tool contract.

## What This Changes

The issue requests structured token, cost, and budget usage data from `session_status` so agents can budget work without parsing transcript JSONL files.

## Best Possible Solution

Define an additive documented agent-facing usage contract backed by existing transcript and `sessions.usage` data, with explicit availability/source metadata and the same visibility guards as current session tools.

## Reproduction Assessment

Yes as a source-level feature-gap check, not as a bug reproduction: current main and v2026.5.20 accept only `sessionKey`/`model` and return rendered `statusText` without a structured usage object.

## Solution Assessment

Unclear until maintainers choose the public API shape and privacy semantics. Extending `session_status.details` is plausible, but a separate usage or budget tool may be cleaner if maintainers want to separate display status from machine-readable accounting.
