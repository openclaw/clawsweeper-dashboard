## Summary

Keep open. Current main still exposes `session_status` as a rendered status-card result with `statusText`, while the requested structured session, last-turn, token, cost, and budget usage contract is still absent and not fully owned by adjacent narrower usage issues.

## What This Changes

Expose raw per-session and last-turn token, cost, and budget usage data through the agent-facing `session_status` tool instead of requiring transcript JSONL parsing.

## Best Possible Solution

Define a documented additive `session_status.details.usage` contract, or an equivalent supported agent-facing usage tool, backed by transcript usage, `sessions.usage`, and provider-window data while preserving existing visibility checks.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection gives a high-confidence current-behavior check: `session_status` returns rendered `statusText` and does not expose structured `details.usage`.

## Solution Assessment

Unclear until maintainers choose the public schema and privacy boundary. A stable sanitized usage payload is more maintainable than requiring agents to parse session JSONL or rendered status text directly.
