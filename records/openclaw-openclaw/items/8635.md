## Summary

Keep this issue open. Current main and the latest release still expose session_status as a /status-style text card with route/model details, while raw structured session/last-turn token and cost data remains on separate gateway usage surfaces; the merged context-budget work is useful but only partially overlaps this broader API request.

## What This Changes

The issue requests structured per-session, last-turn, cost, and budget usage data from the agent-facing session_status tool.

## Best Possible Solution

Define one documented structured usage result behind the existing session-visibility checks, with explicit measured, estimated, stale, unavailable, and source metadata for session totals, latest-turn usage, costs, and quota windows.

## Reproduction Assessment

Yes, at source level. Inspecting the current session_status schema and return object shows that structured usage fields are absent while related usage data exists only as rendered status text or separate gateway methods.

## Solution Assessment

Unclear. The requested capability is reasonable, but the exact sample object is not yet a proven best design because maintainers still need to choose freshness, privacy, quota, and last-turn semantics for a stable public contract.
