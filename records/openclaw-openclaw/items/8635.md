## Summary

Keep open. Current main and the latest shipped release still expose `session_status` as a rendered status-card response, while structured session token/cost/last-turn usage remains available only through gateway usage RPCs; the adjacent budget issue and context-budget PR do not cover this broader agent-facing usage API request.

## What This Changes

Add structured token, cost, last-turn, and budget usage data to the agent-facing `session_status` tool so agents can track consumption without parsing session JSONL files.

## Best Possible Solution

Define an additive documented agent-facing usage contract, either on `session_status.details` or a dedicated usage/budget tool, backed by visibility-aware session, transcript, provider-window, cost, and availability semantics.

## Reproduction Assessment

Yes as a source-level feature-gap check, not as a bug reproduction: current main and `v2026.5.20` accept only `sessionKey`/`model` and return rendered `statusText` without a structured usage object.

## Solution Assessment

Unclear until maintainers choose the public API shape and privacy semantics. Extending `session_status.details` is plausible, but a separate usage or budget tool may be cleaner if maintainers want to separate display status from machine-readable accounting.
