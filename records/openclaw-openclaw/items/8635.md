## Summary

Keep open. Current main still exposes `session_status` as rendered status text with minimal metadata, while the requested structured session, last-turn, token, cost, and budget usage contract is still absent from the agent-facing tool.

## What This Changes

The issue requests structured token and cost usage data through the agent-facing `session_status` tool.

## Best Possible Solution

Define an additive documented agent-facing usage contract, likely `session_status.details.usage` or an equivalent supported tool, backed by transcript usage, `sessions.usage`, and provider-window data while preserving current visibility checks.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source and docs inspection provide a high-confidence current-behavior check that `session_status` returns rendered `statusText` and no structured `details.usage`.

## Solution Assessment

Unclear until maintainers choose the public schema and privacy boundary. A stable sanitized usage payload is the maintainable direction; parsing transcript JSONL or rendered status text should remain a workaround, not the contract.
