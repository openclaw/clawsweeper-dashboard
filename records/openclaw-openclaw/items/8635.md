## Summary

Keep open. Current main and the latest release still expose `session_status` as a rendered status card plus minimal metadata, while the requested structured agent-facing token/cost/session/last-turn usage API remains absent.

## What This Changes

Request to expose raw per-session, last-turn, cost, and budget/window usage data through `session_status` so agents can budget token consumption without parsing transcript JSONL files.

## Best Possible Solution

Define an additive documented agent-facing usage contract on `session_status.details` or a dedicated usage/budget tool, reusing existing gateway/transcript usage sources while preserving visibility, source, availability, and privacy semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; yes as a source-level feature-gap check. Current main and `v2026.5.22` accept only `sessionKey`/`model` and return rendered `statusText` without a structured `usage` object.

## Solution Assessment

Unclear until maintainers choose the public API shape and privacy semantics. Extending `session_status.details` is plausible, but a separate usage or budget tool may be cleaner if maintainers want display status and machine-readable accounting to stay separate.
