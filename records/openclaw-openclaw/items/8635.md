## Summary

Keep open. Current main still exposes `session_status` as a rendered status-card tool result with minimal structured metadata, while raw session, last-turn, cost, and provider-window usage remain formatted text or separate gateway usage RPC data.

## What This Changes

The issue requests structured token, cost, and budget usage fields in the agent-facing `session_status` response.

## Best Possible Solution

Define a documented additive `session_status.details.usage` contract, or an equivalent supported agent-facing usage tool, backed by transcript usage, `sessions.usage`, and provider-window data with visibility checks and availability metadata preserved.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection gives a high-confidence current-behavior check: `session_status` returns rendered `statusText` and does not expose structured `details.usage`.

## Solution Assessment

Unclear until maintainers choose the public schema and privacy boundary. A stable sanitized usage payload is more maintainable than requiring agents to parse session JSONL or rendered status text directly.
