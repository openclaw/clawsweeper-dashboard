## Summary

Keep open. Current main still exposes `session_status` as rendered status text plus minimal details, while the requested structured session, last-turn, token, cost, and budget usage contract is absent; the adjacent https://github.com/openclaw/openclaw/issues/39400 is narrower and does not supersede this broader API request.

## What This Changes

The issue requests raw per-session and last-turn token/cost/budget usage fields from the agent-facing `session_status` result.

## Best Possible Solution

Define a documented additive `session_status.details.usage` contract, or an equivalent supported agent-facing usage tool, backed by transcript usage, `sessions.usage`, and provider-window data while preserving existing visibility checks.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source and docs inspection provide a high-confidence current-behavior check: `session_status` returns rendered `statusText` and does not expose structured `details.usage`.

## Solution Assessment

Unclear until maintainers choose the public schema and privacy boundary. A stable sanitized usage payload is more maintainable than requiring agents to parse session JSONL or rendered status text directly.
