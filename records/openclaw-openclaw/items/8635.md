## Summary

Keep open. Current main and the latest stable release still expose `session_status` as rendered status text plus minimal metadata; the merged context-budget work and existing gateway usage RPCs are adjacent, but they do not provide the requested agent-facing structured session, last-turn, token, cost, and budget usage API.

## What This Changes

The issue requests a structured token and cost usage API on the agent-facing `session_status` tool.

## Best Possible Solution

Define one additive, documented, visibility-aware agent-facing usage contract on `session_status.details` or a dedicated usage tool, reusing existing transcript and `sessions.usage` sources without changing the rendered status card semantics.

## Reproduction Assessment

Yes as a source-level feature-gap check: current main and `v2026.5.22` accept only `sessionKey`/`model` and return rendered `statusText` without structured token, cost, last-turn, or budget usage fields. This is not a bug reproduction because the request is for a new API capability.

## Solution Assessment

Unclear until maintainers choose the public API shape and semantics. Structured usage data is the maintainable direction, but the exact fields, data sources, availability metadata, and privacy rules need product/API judgment first.
