## Summary

Keep open. Current main still returns a rendered `/status`-style `statusText` from `session_status`; structured session/last-turn token and cost data exists in transcript and gateway usage surfaces, but not as a documented agent-facing `session_status` contract. Related #39400 is adjacent quota-window stability work, not a full substitute for this API request.

## What This Changes

Expose structured per-session, last-turn, cost, and budget/window usage data through the agent-facing `session_status` tool.

## Best Possible Solution

Add a documented structured agent-facing usage contract, likely on `session_status.details.usage` or an equivalent supported tool, reusing existing transcript and `sessions.usage` parsers while preserving visibility checks.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection on current main, which shows no structured usage payload in `session_status`.

## Solution Assessment

Unclear until maintainers choose the public schema and privacy boundary. The maintainable direction is a sanitized structured contract on `session_status` or an equivalent supported tool, not agent-side transcript parsing.
