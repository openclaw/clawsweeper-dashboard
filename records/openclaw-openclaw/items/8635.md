## Summary

Keep open: current main still does not expose structured token, cost, last-turn, or budget data through the agent-facing `session_status` result; it renders usage into status text while structured usage remains on separate gateway/operator surfaces.

## What This Changes

Request to expose raw per-session, last-turn, cost, and budget token usage data through the agent-facing `session_status` tool.

## Best Possible Solution

Define an additive documented agent-facing usage contract backed by `sessions.usage` and transcript/provider usage parsers, with explicit source/unavailable metadata and existing visibility checks preserved.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection on current main gives a high-confidence behavior check: `session_status` returns rendered `statusText` and minimal metadata, while structured usage is available only through separate gateway/operator surfaces or display text.

## Solution Assessment

Unclear until maintainers choose the public API contract. Reusing `sessions.usage` plus transcript/provider usage parsers with visibility and availability metadata is safer than asking agents to parse status text or JSONL files directly.
