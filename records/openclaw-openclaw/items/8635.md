## Summary

Keep open. Current main includes the merged estimated context-budget status work, but `session_status` still returns a rendered status card plus minimal metadata rather than a structured agent-facing API for session totals, last-turn usage, cost, and budget/window data.

## What This Changes

Request exposing per-session, last-turn, cost, and budget token usage as structured fields through `session_status` so agents can budget usage without parsing JSONL transcripts.

## Best Possible Solution

Define an additive documented agent-facing usage contract on `session_status.details` or a dedicated usage/budget tool, reusing `sessions.usage` and transcript parsers while preserving visibility, source, availability, and privacy semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; yes as a source-level feature-gap check. Current main and `v2026.5.20` accept only `sessionKey`/`model` and return rendered `statusText` without a structured `usage` object.

## Solution Assessment

Unclear until maintainers choose the public API shape and privacy semantics. Extending `session_status.details` is plausible, but a separate usage or budget tool may be cleaner if maintainers want display status and machine-readable accounting to stay separate.
