## Summary

Keep this issue open. Current main provides rendered status output, structured gateway usage reporting, and an estimated context-budget snapshot, but it still lacks a documented structured usage contract available to agent callers; the narrower OpenRouter cost issue does not supersede the broader session and latest-turn request.

## What This Changes

Expose structured session totals, latest-turn token and cost usage, and provider budget windows to agents through `session_status` or an equivalent supported tool.

## Best Possible Solution

Define one documented structured usage result behind existing session-visibility checks, reusing current transcript and provider-usage parsers while explicitly distinguishing measured, estimated, stale, unsupported, and temporarily unavailable values.

## Reproduction Assessment

Yes, at source level. The current `session_status` result can be inspected directly to confirm that structured usage fields are absent while related information exists only in rendered status output or separate gateway methods.

## Solution Assessment

The requested capability is appropriate, but the exact proposed object is not yet the best proven design. Maintainers should first define one stable contract across transcript totals, latest-turn usage, estimated context budget, provider costs, quota-window freshness, and access boundaries.
