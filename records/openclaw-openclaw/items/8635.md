## Summary

Keep open. Current main and the latest release still expose `session_status` as a rendered status card plus minimal metadata; the merged context-budget work is adjacent but does not provide the requested structured session, last-turn, token, cost, and budget usage API.

## What This Changes

Request to expose structured token, cost, and budget usage data through the agent-facing `session_status` tool.

## Best Possible Solution

Define an additive documented agent-facing usage contract on `session_status.details` or a dedicated usage/budget tool, reusing existing transcript and gateway usage sources while preserving source, availability, cost, and privacy semantics.

## Reproduction Assessment

Yes as a source-level feature-gap check, not as a runtime bug reproduction. Current main and `v2026.5.22` accept only `sessionKey` and `model` and return rendered `statusText` without a structured `usage` object.

## Solution Assessment

Unclear until maintainers choose the public API shape and privacy semantics. Extending `session_status.details` is plausible, but a separate usage or budget tool may be cleaner if display status and machine-readable accounting should stay separate.
