## Summary

Keep open. Current main still exposes `session_status` as a rendered status-card result, while raw token, cost, last-turn, and provider-window data remain separate or text-only surfaces; related https://github.com/openclaw/openclaw/issues/39400 is adjacent but narrower.

## What This Changes

Expose structured per-session and last-turn token usage, cost, and budget/window data through the agent-facing `session_status` tool.

## Best Possible Solution

Add a documented additive structured usage contract, likely `session_status.details.usage` or an equivalent supported agent tool, reusing transcript and `sessions.usage` parsers while preserving visibility checks.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection gives a high-confidence verification path: current `session_status` renders usage into `statusText` and does not expose structured `details.usage`.

## Solution Assessment

Unclear until maintainers choose the public schema and privacy boundary. A stable sanitized `details.usage` contract or equivalent supported tool is more maintainable than requiring agents to parse session JSONL directly.
