## Summary

Keep open. Current main still exposes `session_status` as a rendered status card with minimal metadata, and the recent context-budget work is adjacent but does not provide the requested structured session, last-turn, token, cost, and budget usage API.

## What This Changes

Request to expose raw per-session and last-turn token, cost, and budget usage data through `session_status` or an equivalent agent-facing API.

## Best Possible Solution

Define an additive documented agent-facing usage contract on `session_status.details` or a dedicated usage/budget tool, reusing gateway/transcript usage sources while preserving visibility, source, availability, and privacy semantics.

## Reproduction Assessment

Yes as a source-level feature-gap check, not as a bug reproduction. Current main and `v2026.5.22` accept only `sessionKey` and `model` and return rendered `statusText` without a structured `usage` object.

## Solution Assessment

Unclear until maintainers choose the public API shape and privacy semantics. Extending `session_status.details` is plausible, but a separate usage or budget tool may be cleaner if display status and machine-readable accounting should stay separate.
