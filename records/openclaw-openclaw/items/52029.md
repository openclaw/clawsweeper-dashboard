## Summary

Keep open. Current main still lacks a heartbeat-scoped tool-disable config, while the lower-level `disableTools` runtime seam already exists, so this is a narrow safety/cost feature rather than a runtime redesign.

## What This Changes

Add a heartbeat-scoped config option that omits tool definitions during heartbeat agent turns for local/small models.

## Best Possible Solution

Add a documented `heartbeat.tools: false` boolean for defaults and per-agent heartbeat config that maps heartbeat turns to the existing `disableTools` run option without changing normal user turns or global tool policy.

## Reproduction Assessment

Yes. Source inspection on current main is high confidence: the strict heartbeat schema rejects the requested key and ordinary `runHeartbeatOnce` reply options do not set `disableTools` except for due-commitment heartbeats.

## Solution Assessment

Yes. Plumbing a boolean heartbeat config into the existing `disableTools` seam is the narrowest maintainable path; an allowlist can remain a later extension if maintainers want it.
