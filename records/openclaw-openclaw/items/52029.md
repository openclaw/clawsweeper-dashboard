## Summary

Keep open. Current main still lacks a heartbeat-scoped tool-disable config, while the existing run-level `disableTools` seam means this is a narrow config-plumbing feature rather than a runtime redesign.

## What This Changes

Requests a heartbeat-scoped config option such as `agents.defaults.heartbeat.tools: false` to omit tool definitions during heartbeat turns for local or small models.

## Best Possible Solution

Add a documented heartbeat-scoped `tools: false` boolean that maps ordinary heartbeat turns to the existing `disableTools` run option, leaving normal user turns and existing global/per-agent tool policies unchanged.

## Reproduction Assessment

Yes. A source-level current-main check is high confidence: strict `HeartbeatSchema` rejects the requested key and `runHeartbeatOnce` does not set `disableTools` for ordinary heartbeat runs.

## Solution Assessment

Yes. Plumbing a boolean heartbeat config into the existing `disableTools` seam is the narrowest maintainable solution; an allowlist can remain a later extension if needed.
