## Summary

Keep open. Current main and the latest release still lack a heartbeat-scoped tool-disable config; ordinary heartbeat turns only pass `disableTools` for due-commitment heartbeats, while the schema, public types, docs, and inventory have no `heartbeat.tools` option.

## What This Changes

Request adds a heartbeat-scoped config option, such as `agents.defaults.heartbeat.tools: false`, to omit tool definitions from heartbeat model turns.

## Best Possible Solution

Add a documented `heartbeat.tools: false` boolean for defaults and per-agent heartbeat config that maps heartbeat turns to the existing `disableTools` run option without changing normal user turns or global tool policy.

## Reproduction Assessment

Yes for the requested config gap: current-main source shows a strict heartbeat schema with no `tools` key, and ordinary heartbeat reply options do not set `disableTools` except for due commitments. I did not reproduce the reported local-model tool loop live.

## Solution Assessment

Yes. Plumbing a boolean heartbeat config into the existing `disableTools` seam is the narrowest maintainable path; a heartbeat tool allowlist can remain a later extension if maintainers want it.
