## Summary

Keep open: current main and the latest release still lack a heartbeat-scoped tool toggle, and ordinary heartbeat turns only set `disableTools` for due-commitment delivery. The issue is a narrow feature/config addition with a clear existing runtime seam.

## What This Changes

The issue requests `agents.defaults.heartbeat.tools: false` and per-agent equivalent support so heartbeat turns can omit tool definitions for local/small models.

## Best Possible Solution

Add documented `heartbeat.tools: false` support for defaults and per-agent heartbeat config, map it to `disableTools: true` with an empty skill filter during heartbeat reply construction, and preserve the current tool-capable default behavior.

## Reproduction Assessment

Yes for the config gap: current-main source shows a strict heartbeat schema with no `tools` key, and ordinary heartbeat reply options do not set `disableTools` except for due commitments. I did not live-reproduce the reported local-model tool loop.

## Solution Assessment

Yes. Plumbing a boolean heartbeat config into the existing `disableTools` seam is the narrowest maintainable solution; a richer allowlist can be a later extension if maintainers want it.
