## Summary

Keep open. Current main still lacks a heartbeat-scoped tool-disable config, while the runtime already has a lower-level `disableTools` seam that makes this a narrow config-plumbing feature.

## What This Changes

Add a heartbeat-scoped config option, proposed as `agents.defaults.heartbeat.tools: false`, to omit tool definitions from heartbeat turns.

## Best Possible Solution

Add a documented heartbeat boolean that maps ordinary heartbeat turns to `disableTools`, while leaving normal user turns, existing due-commitment behavior, and global/per-agent tool policies unchanged.

## Reproduction Assessment

Yes. A source-level current-main check is high confidence: strict `HeartbeatSchema` rejects the requested key and `runHeartbeatOnce` does not set `disableTools` for ordinary heartbeat runs.

## Solution Assessment

Yes. Plumbing a heartbeat-scoped boolean into the existing `disableTools` seam is the narrowest maintainable solution; an allowlist can remain a later extension if maintainers want it.
