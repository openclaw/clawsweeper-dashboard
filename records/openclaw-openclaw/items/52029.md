## Summary

Keep open: current main and v2026.5.19 still lack a heartbeat-scoped hard no-tools config, and the related open PR at https://github.com/openclaw/openclaw/pull/70518 only proposes a heartbeat skill allowlist rather than a full execution-tool disable mode.

## What This Changes

Request a heartbeat config option that can omit model-visible tools during heartbeat runs.

## Best Possible Solution

Add documented `heartbeat.tools: false` or a maintainer-approved equivalent that sets `disableTools: true` for opted-in ordinary heartbeat turns, preserves current default tool-capable behavior, and does not regress due-commitment no-tools handling.

## Reproduction Assessment

Yes for the config gap: current-main source shows a strict heartbeat schema with no hard tool-policy key, and ordinary heartbeat reply options only set `disableTools` for due commitments. I did not live-reproduce the reported local-model tool loop.

## Solution Assessment

Yes for the implementation seam: plumbing an opt-in heartbeat config into the existing `disableTools` path is the narrow maintainable fix. The public option name and boolean-versus-allowlist semantics still need maintainer acceptance.
