## Summary

Keep open: current main still has no heartbeat-scoped config knob for disabling tools, and ordinary heartbeat turns only set `disableTools` for due-commitment delivery. The request is a narrow feature/config addition with a clear existing runtime seam.

## What This Changes

Add a `heartbeat.tools` option that can disable tool definitions during heartbeat agent turns.

## Best Possible Solution

Add documented `heartbeat.tools: false` support for defaults and per-agent heartbeat config, map it to `disableTools: true` and an empty skill filter for heartbeat reply options, and leave the default heartbeat behavior tool-capable.

## Reproduction Assessment

Yes for the config gap: current-main source shows a strict heartbeat schema with no `tools` key, and ordinary heartbeat reply options do not set `disableTools` except for due commitments. I did not live-reproduce the reported local-model tool loop.

## Solution Assessment

Yes. Plumbing a boolean heartbeat config into the existing `disableTools` seam is the narrowest maintainable solution; a heartbeat allowlist can be a later extension if maintainers want it.
