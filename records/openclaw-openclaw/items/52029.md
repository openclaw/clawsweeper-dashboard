## Best Possible Solution

Decide the heartbeat tool-policy contract, then land a backward-compatible schema, runner, docs, and regression-test change that lets operators explicitly run heartbeats with no tools or a narrowly approved tool set.

Unclear as proposed: a `tools: false` boolean is plausible, but maintainers still need to choose whether a boolean, allowlist, or another policy best fits heartbeat response tools, commitments, compatibility, and security expectations.
