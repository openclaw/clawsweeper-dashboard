## What This Changes

Prevent WebChat from showing both the raw assistant `MEDIA:` reply and the gateway-injected normalized media transcript message for one assistant response.

## Best Possible Solution

Use the existing PR as the canonical tracker, review its transcript-replacement approach, and land or close that PR rather than split the same WebChat MEDIA fix across another issue.

## Reproduction Assessment

Yes. Source inspection shows current main can expose two assistant entries for one MEDIA reply: the UI renders the raw provider `MEDIA:` text, while the gateway agent-run media supplement appends a normalized `gateway-injected` transcript message.

## Solution Assessment

Yes. Replacing the raw transcript turn in the gateway, as the canonical PR does, is narrower and more maintainable than UI-only dedupe because it keeps `chat.history`, replay hygiene, and persisted media content aligned.
