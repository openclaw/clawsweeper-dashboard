## Summary

Keep open. The reported MEDIA duplicate is still source-reproducible on current main, and the closest open fix path is https://github.com/openclaw/openclaw/pull/75727, which has not landed yet.

## What This Changes

Avoid duplicate WebChat assistant delivery when an assistant reply contains a MEDIA directive by showing only one normalized media/text assistant message.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/75727 with a narrow gateway transcript fix that rewrites the current raw MEDIA assistant turn into structured media content instead of appending a second `:assistant-media` turn, with focused regression coverage.

## Reproduction Assessment

Yes, source-level. Current main still appends a gateway-injected assistant-media transcript for media-bearing agent replies while WebChat can render the raw provider-authored MEDIA assistant turn, and attachment-bearing duplicates are not collapsed by the UI text-only duplicate guard.

## Solution Assessment

Yes. The maintainable fix is to reconcile the two transcript owners at the gateway by replacing or suppressing one matching MEDIA turn, not by adding UI-only duplicate hiding.
