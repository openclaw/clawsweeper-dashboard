## Summary

Keep open: current main still has the source shape that can produce a raw assistant MEDIA transcript row plus a separate `:assistant-media` gateway row, and https://github.com/openclaw/openclaw/pull/75727 is only an open, conflicting candidate fix rather than a merged implementation.

## What This Changes

The issue requests a WebChat fix so a single assistant reply containing a `MEDIA:` directive is delivered once instead of as both the raw provider message and a gateway-injected media message.

## Best Possible Solution

Land a current-main fix that replaces or suppresses the raw MEDIA transcript turn when the gateway normalizes the same reply, with focused transcript and WebChat rendering coverage.

## Reproduction Assessment

Yes at source level: current main appends a gateway `:assistant-media` row for media-bearing agent payloads while the UI can render the raw MEDIA row as an attachment too. I did not run a fresh live WebChat session, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

Yes on direction: the best fix is at the Gateway transcript/media handoff, replacing or suppressing the matching raw assistant row rather than relying on UI-only dedupe. The open candidate PR follows that direction but is not merge-ready as-is.
