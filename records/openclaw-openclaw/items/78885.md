## Summary

Keep open. Current main still has the source-level duplicate path: WebChat can render the provider-authored raw `MEDIA:` assistant turn, while the Gateway can append a separate `:assistant-media` transcript entry for the same media-bearing agent reply; the closest fix, https://github.com/openclaw/openclaw/pull/75727, remains open.

## What This Changes

The issue requests preventing WebChat from delivering both the raw `MEDIA:` assistant message and the gateway-injected media message for one assistant reply.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/75727 with a narrow gateway transcript fix that replaces or suppresses the current raw `MEDIA:` assistant turn when the normalized media-bearing reply represents the same assistant message, with focused regression coverage.

## Solution Assessment

Yes. The maintainable fix belongs in the gateway transcript/media persistence seam by reconciling the raw and normalized assistant turns, not by adding a UI-only duplicate hiding rule.
