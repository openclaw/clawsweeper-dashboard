## Summary

## What This Changes

The issue asks OpenClaw to deliver a WebChat `MEDIA:` assistant reply once instead of showing both the raw provider message and a gateway-injected media message.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/75727 with a narrow gateway transcript reconciliation that rewrites or suppresses the matching raw `MEDIA:` assistant turn instead of appending a second visible media turn.

## Reproduction Assessment

Yes, source-level. Current main still parses assistant `MEDIA:` directives into media-bearing reply payloads, appends a separate WebChat `:assistant-media` transcript entry, and renders raw `MEDIA:` text as attachments while the duplicate guard ignores attachment-bearing messages.

## Solution Assessment

Yes. The maintainable fix belongs at the gateway transcript boundary by reconciling the raw `MEDIA:` assistant turn with the normalized media message; UI-only duplicate hiding would leave the transcript contract wrong.
