This PR adds opt-in reset-family transcript lookup to Gateway and embedded `chat.history`, forwards it through `sessions_history` with a `resetAncestors` alias, and updates protocol schema, docs, generated Swift models, tests, and prompt snapshots.

## Best Possible Solution

Keep the bounded/header-checked archive resolver, reserve current-session targets before capped ancestors in Gateway and embedded mode, settle one public selector contract, and require standalone default-versus-family history proof before merge.

No. The owner boundary is plausible, but this is not the best fix until current-session history is reserved under the cap and maintainers settle the public selector/read-scope contract.
