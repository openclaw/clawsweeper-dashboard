Full review comments:

- **[P1] Preserve owned topic ancestor archives:** `src/gateway/server-methods/chat.ts:2529-2533`
  - body: For ancestor family sessions, this calls the bounded archive helper without the ancestor's `sessionFile`. That helper only searches reset archives for known transcript basenames, so an owned archive such as `<ancestor>-topic-456.jsonl.reset.*` is missed even though topic transcript filenames are established and the PR's proof claims that case.
  - confidence: 0.88
- **[P1] Use the bounded archive resolver in embedded history:** `src/agents/tools/embedded-gateway-stub.ts:206-209`
  - body: Embedded `chat.history(includeFamily)` still scans the sessions directory by filename prefix and passes every match to the transcript reader. That bypasses the server helper's bounded/header-checked ownership checks, so embedded mode can include colliding or headerless reset archives.
  - confidence: 0.91
- **[P2] Bound family history reads before opening transcripts:** `src/gateway/server-methods/chat.ts:2680-2689`
  - body: The resolver builds targets for every stored family session id and then reads them all with `Promise.all` before the final response byte budget is enforced. A long reset lineage can turn one opt-in history call into many concurrent file reads.
  - confidence: 0.78
