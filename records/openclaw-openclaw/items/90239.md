## What This Changes

The PR adds an opt-in `includeFamily` history path to Gateway `chat.history` and `sessions_history`, with shared reset-archive resolution, Swift/protocol/docs/prompt snapshot updates, and regression coverage.

## Best Possible Solution

Land only after maintainers accept `includeFamily` as the single opt-in API/model-facing selector, exact-head required/security checks pass, and the automatic carryover-summary behavior remains separate.

## Reproduction Assessment

Yes at source level: current main's `chat.history` reads only the current session target and `sessions_history` has no family selector. I did not run a live transport reproduction.

## Solution Assessment

Yes, if maintainers accept the new public selector: bounded opt-in family lookup with header-checked archives is cleaner than automatic startup loading or broad archive scans. The safer alternative, if the API is not accepted, is to keep the resolver internal and defer model-facing exposure.
