- **[P1] Preserve topic archive lookup for family ancestors:** `src/gateway/server-methods/chat.ts:2529-2534`
  - body: For ancestor family sessions, this passes the session id but no ancestor `sessionFile` into the bounded archive helper. That helper only searches archives for known transcript basenames, so an owned reset archive like `<ancestor>-topic-456.jsonl.reset.*` is missed even though topic transcript filenames are established and the PR body's smoke proof uses that shape.
  - confidence: 0.88
- **[P1] Use the bounded archive resolver in embedded history:** `src/agents/tools/embedded-gateway-stub.ts:206-209`
  - body: Embedded `chat.history(includeFamily)` still scans the sessions directory by raw filename prefix and passes every match to the transcript reader. That bypasses the server helper's bounded/header-checked ownership checks, so embedded mode can include colliding or headerless same-prefix reset archives.
  - confidence: 0.9
- **[P2] Bound family history targets before reading transcripts:** `src/gateway/server-methods/chat.ts:2680-2690`
  - body: The resolver builds targets for every stored family session id and then reads them all with `Promise.all` before the final response byte budget is enforced. Since `usageFamilySessionIds` is accumulated without a cap, a long reset lineage can turn one opt-in history call into many concurrent file reads.
  - confidence: 0.78
