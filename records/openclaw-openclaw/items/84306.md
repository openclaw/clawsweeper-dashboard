## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Keep chat abort registrations on the canonical session key:** `src/gateway/server-methods/chat.ts:3220`
  - body: This stores the raw request key for every non-global `chat.send`. After preprocessing, the same run is added to chat run state with the canonical `sessionKey`; `abortChatRunById` then uses `active.sessionKey` for both aborted delivery and `removeChatRun`, while the registry removes only entries whose session key matches. For canonicalized aliases, aborting by run id would emit the aborted frame to the alias and fail to remove the canonical active-run entry immediately. Keep the abort entry on the canonical `sessionKey` and rely on the abort handler's existing raw/canonical alias check for authorization.
  - confidence: 0.9
