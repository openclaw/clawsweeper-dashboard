## Summary

Keep this PR open: current main still has the attachment-preprocessing abort race, and the branch is a meaningful fix path, but it is not merge-ready because terminal `chat.send` ACK handling can leave the Control UI's optimistic local run active and the revised terminal-ACK/session behavior still needs stronger real proof.

## What This Changes

The PR moves Gateway `chat.send` abort registration before attachment preprocessing, adds terminal `timeout`/`error` ACK handling and dedupe behavior, updates Control UI terminal-ACK adoption logic, widens a forwarded skills revision result schema, documents the ACK change, and adds regression tests.

## Best Possible Solution

Land a version that keeps the early abort registration and terminal dedupe semantics, clears Control UI local state on terminal ACKs, documents the widened ACK contract, and includes redacted real Gateway/WebSocket or browser proof for the changed terminal behavior.

## Reproduction Assessment

Yes. Source inspection shows current main registers the chat abort controller only after awaited attachment preprocessing, leaving the reported immediate-abort race; the PR adds deterministic regression coverage, but I did not run tests in this read-only review.

## Solution Assessment

No. The server-side fix is a plausible owner-boundary repair, but the proposed Control UI terminal-ACK path is incomplete because it skips adoption without clearing the local optimistic run, and the public ACK widening needs explicit compatibility review.
