The PR moves Gateway `chat.send` abort-controller registration before awaited attachment preprocessing, adds guarded pre-dispatch media cleanup, and adds regression tests for aborting attachment sends during preprocessing.

## Reproduction Assessment

Yes. Current main has a source-level race where `chat.abort` can return `aborted:false` before `chat.send` registers the run after awaited attachment preprocessing; the PR body supplies after-fix live Gateway proof for the same timing window.

## Solution Assessment

Yes. Registering after validation, send-policy, idempotency, and active-dedupe checks but before awaited preprocessing is the narrow maintainable fix, and the PR preserves public response shapes while cleaning pre-dispatch media.
