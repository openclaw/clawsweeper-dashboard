- **[P2] Preserve current hook context merging on rebase:** `src/agents/pi-embedded-runner/run/attempt.prompt-helpers.ts:93-96`
  - body: The branch resolver is based on an older helper and its return path only carries prompt-build/legacy fields. Current main also merges drained next-turn injections, `agent_turn_prepare`, heartbeat contribution, and `appendContext`; resolving conflicts by taking this branch's block would drop those prompt mutations on affected runs.
  - confidence: 0.88
- **[P2] Expose client tools to prompt-build hooks:** `src/agents/pi-embedded-runner/run/attempt.ts:1839-1844`
  - body: `availableTools` is built from `effectiveTools`, but client-hosted tools are registered separately through `clientToolDefs`/`allCustomTools`. A plugin deriving `toolsAllow` from the exposed catalog cannot include those model-visible tools, and `setActiveToolsByName(...)` can remove tools the hook never saw.
  - confidence: 0.86
- **[P2] Filter snapshot skills without live workspace reload:** `src/agents/pi-embedded-runner/run/attempt.ts:1927-1938`
  - body: For snapshot-backed runs, this reloads current workspace skill entries before filtering. That can inject newer on-disk skill content or drop skills that existed in the session snapshot, breaking the snapshot consistency current main preserves.
  - confidence: 0.78
- **[P2] Preserve hook prompt edits when filtering skills:** `src/agents/pi-embedded-runner/run/attempt.ts:1946-1952`
  - body: When `skillsAllow` is set, this rebuilds `systemPromptText` from `appendPromptParams` after hook system-prompt mutations were already applied. That can drop same-turn `systemPrompt`/prepend/append system context and bypass the normal provider system-prompt transform path.
  - confidence: 0.9
