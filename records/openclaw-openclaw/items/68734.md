## Summary

Keep open: the idea has useful real-world support, but this PR is not merge-ready because the current patch can drop prompt-boundary guidance, mis-handle client-hosted tools, and conflicts with an overlapping public hook proposal.

## What This Changes

The branch adds before_prompt_build toolsAllow/skillsAllow hook fields, passes hook-visible tool and skill catalogs, narrows active session tools, and rebuilds the skill prompt catalog per turn.

## Best Possible Solution

Land one maintainer-approved dynamic narrowing contract that filters tools and skills through the normal prompt/session pipeline while covering client tools, snapshots, provider transforms, and compatibility tests.

## Reproduction Assessment

Yes for the review findings at source level: the PR head shows the skillsAllow rebuild after hook prompt mutations, and current main shows client tools plus skill snapshots are assembled outside the PR's observed lists. I did not run a live gateway repro in this read-only review.

## Solution Assessment

No, not as submitted. The useful capability should be implemented through one approved plugin contract and should reuse the normal prompt/session assembly path rather than rebuilding parallel prompt state inside the hook branch.
