## Summary

## What This Changes

The PR adds before_prompt_build tool/skill catalogs and hook-returned toolsAllow/skillsAllow filtering in the embedded runner, with focused hook tests.

## Best Possible Solution

Land a documented hook seam that exposes the complete model-visible tool catalog and resolved skill catalog, then narrows through existing policy, session, snapshot, and provider prompt-builder paths without losing prompt mutations.

## Reproduction Assessment

Yes. Source inspection of current main shows before_prompt_build receives only prompt/messages and cannot return toolsAllow or skillsAllow, so the missing capability is reproducible from the current hook contract and call path.

## Solution Assessment

No, not as written. The hook seam is a plausible direction, but PR head needs fixes for client-hosted tools, snapshot-backed skills, prompt mutation preservation, and rebase safety before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Expose client-hosted tools to prompt hooks:** `src/agents/pi-embedded-runner/run/attempt.ts:1839-1840`
  - body: `availableTools` is derived from `effectiveTools`, but client-hosted tools are added separately through `clientToolDefs` into `allCustomTools` and sent to the model. A plugin that mirrors the exposed list cannot preserve those tools in `toolsAllow`, so `setActiveToolsByName()` can remove model-visible tools the hook never exposed.
  - confidence: 0.86
- **[P2] Reuse snapshot skills when filtering:** `src/agents/pi-embedded-runner/run/attempt.ts:1927-1938`
  - body: In snapshot-backed runs this branch reloads current workspace skill entries and rebuilds from live disk state. Current `resolveSkillsPromptForRun()` uses the snapshot prompt, so this can inject newer skill content or drop snapshot-only skills during a run that should stay snapshot-consistent.
  - confidence: 0.8
- **[P2] Preserve prompt mutations when narrowing skills:** `src/agents/pi-embedded-runner/run/attempt.ts:1946-1952`
  - body: When `skillsAllow` is set, this rebuilds from `appendPromptParams` and applies the result after hook system-prompt mutations. Returning `skillsAllow` with `systemPrompt` or prepend/append system context can lose those prompt edits and bypass the normal provider transform path.
  - confidence: 0.9
