## Summary

Keep open: this is an active external feature PR for a real plugin-owned prompt-surface gap, and the body includes plausible live-run evidence, but current main does not already implement the hook narrowing contract and the branch still has correctness issues in prompt rebuild, client-tool exposure, and snapshot skill handling.

## What This Changes

The PR extends `before_prompt_build` hooks with tool/skill catalogs plus `toolsAllow` and `skillsAllow`, then wires embedded-runner filtering for active tools and skill prompt content with focused tests.

## Best Possible Solution

Rebase onto current main, keep one plugin-owned dynamic narrowing seam, expose every model-visible tool and already-resolved skill source to hooks, and rebuild narrowed prompts through the normal prompt composition/provider-transform path with regression coverage.

## Reproduction Assessment

Yes. Source inspection of current main and PR head gives a high-confidence reproduction path for the review findings: current main lacks the contract, while PR-head code shows the prompt rebuild, client-tool, and snapshot-skill edge cases without needing a live gateway.

## Solution Assessment

No, not as written. The feature direction is plausible and has real deployment support, but the current patch should not merge until it preserves prompt mutations/provider transforms, exposes client tools, and filters snapshot-backed skills without live reload.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve hook prompt edits when filtering skills:** `src/agents/pi-embedded-runner/run/attempt.ts:1946-1952`
  - body: When `skillsAllow` is set, this branch rebuilds `systemPromptText` from `buildEmbeddedSystemPrompt(appendPromptParams)` after hook prompt mutations have already been applied. That can drop same-turn `systemPrompt`/prepend/append system context and bypass the normal provider system-prompt transform path.
  - confidence: 0.9
- **[P2] Expose client tools to prompt-build hooks:** `src/agents/pi-embedded-runner/run/attempt.ts:1839`
  - body: `availableTools` is built from `effectiveTools`, but client-hosted tools are registered separately through `clientToolDefs`/`allCustomTools`. A plugin deriving `toolsAllow` from the exposed catalog cannot include those model-visible tools, and the active-tool update can then remove tools the hook never saw.
  - confidence: 0.86
- **[P2] Filter snapshot skills without live workspace reload:** `src/agents/pi-embedded-runner/run/attempt.ts:1927-1938`
  - body: For snapshot-backed runs, this branch reloads current workspace skill entries before filtering. That can inject newer on-disk skill content or drop skills that existed in the session snapshot, breaking the snapshot consistency preserved by the current skills runtime.
  - confidence: 0.78
