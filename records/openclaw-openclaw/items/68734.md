## Summary

Keep open: current main still lacks dynamic prompt-hook tool/skill narrowing, and the discussion includes useful real-world MCP token-cost evidence, but this PR is not mergeable as-is because the PR head still has correctness blockers and conflicts with current prompt/session code.

## What This Changes

The PR extends `before_prompt_build` hook contracts to expose available tools and skills, lets hook results return `toolsAllow` and `skillsAllow`, wires those allowlists into embedded runner prompt/session assembly, and adds focused tests.

## Best Possible Solution

Land a documented hook seam that exposes the complete model-visible tool catalog and resolved skill catalog, then narrows through existing policy, session, snapshot, and prompt-builder paths without breaking current prompt mutations.

## Reproduction Assessment

Yes. Source inspection shows current main has no tool or skill catalogs on `before_prompt_build`, and the PR-head blockers are traceable from the current client-tool, snapshot-skill, and prompt-builder paths.

## Solution Assessment

No, not as written. The hook seam is plausible, but the implementation needs a rebase and fixes so it exposes the full model-visible tool catalog and rebuilds narrowed skill prompts through existing snapshot and provider-transform behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Expose client-hosted tools to prompt hooks:** `src/agents/pi-embedded-runner/run/attempt.ts:1839`
  - body: `availableTools` is derived from `effectiveTools`, but the runner adds `clientToolDefs` separately into `allCustomTools` before session creation. A plugin that mirrors the exposed list cannot preserve client-hosted tools in `toolsAllow`, so `setActiveToolsByName()` can remove model-visible tools the hook never saw.
  - confidence: 0.86
- **[P2] Filter snapshot skills without reloading live entries:** `src/agents/pi-embedded-runner/run/attempt.ts:1927-1938`
  - body: In snapshot-backed runs, this branch calls `loadWorkspaceSkillEntries()` and rebuilds from current disk state. Current `resolveSkillsPromptForRun()` returns `skillsSnapshot.prompt`, so this can inject newer skill content or drop snapshot skills, breaking snapshot consistency.
  - confidence: 0.8
- **[P2] Preserve prompt mutations when rebuilding skills:** `src/agents/pi-embedded-runner/run/attempt.ts:1946-1952`
  - body: When `skillsAllow` is set, this rebuilds from `appendPromptParams` and applies the result directly after hook system-prompt mutations. Returning `skillsAllow` with `systemPrompt` or prepend/append context can lose those prompt edits and bypass the normal provider transform path.
  - confidence: 0.9
