## Summary

Keep open. Current main still lacks the requested prompt-hook tool/skill narrowing surface, and the PR has useful real-run evidence plus supporting discussion, but the branch needs maintainer API review, a current-main rebase, and correctness fixes before merge.

## What This Changes

The PR extends `before_prompt_build` hook contracts with available tool/skill catalogs and optional `toolsAllow`/`skillsAllow`, then applies those allowlists in the embedded runner with focused tests.

## Best Possible Solution

Land a documented hook seam that exposes the complete model-visible tool catalog and resolved skill catalog, then narrows through existing policy, session, snapshot, and provider prompt-builder paths without losing current prompt mutations.

## Reproduction Assessment

Yes. Source inspection of current main shows the hook contract and call path do not expose tool or skill catalogs and cannot return `toolsAllow` or `skillsAllow` from `before_prompt_build`.

## Solution Assessment

No, not as written. The hook seam is a plausible direction, but PR head needs fixes for client-hosted tools, snapshot-backed skills, prompt mutation preservation, and current-main rebase safety before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Expose client-hosted tools to prompt hooks:** `src/agents/pi-embedded-runner/run/attempt.ts:1839`
  - body: `availableTools` is derived from `effectiveTools`, but client-hosted tools are later added through `clientToolDefs` into `allCustomTools` and sent to the model. A plugin that mirrors the exposed list cannot preserve those tools in `toolsAllow`, so `setActiveToolsByName()` can remove model-visible tools the hook never exposed.
  - confidence: 0.86
- **[P2] Reuse snapshot skills when filtering:** `src/agents/pi-embedded-runner/run/attempt.ts:1927-1938`
  - body: In snapshot-backed runs this branch reloads current workspace skill entries and rebuilds from live disk state. Current `resolveSkillsPromptForRun()` returns the snapshot prompt, so this can inject newer skill content or drop snapshot-only skills during a run that should stay snapshot-consistent.
  - confidence: 0.8
- **[P2] Preserve prompt mutations when narrowing skills:** `src/agents/pi-embedded-runner/run/attempt.ts:1946-1952`
  - body: When `skillsAllow` is set, this rebuilds from `appendPromptParams` and applies the result after hook system-prompt mutations. Returning `skillsAllow` with `systemPrompt` or prepend/append system context can lose those prompt edits and bypass the normal provider transform path.
  - confidence: 0.9
