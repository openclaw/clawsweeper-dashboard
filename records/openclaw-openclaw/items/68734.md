## Summary

Keep open: the feature is plausible and has useful live-proof signal, but source review found merge-blocking prompt-pipeline and hosted-tool wiring issues in the proposed hook implementation.

## What This Changes

The PR adds optional `toolsAllow` and `skillsAllow` results plus `availableTools` and `availableSkills` hook inputs to `before_prompt_build`, wiring them into embedded agent tool selection and skill prompt rebuilding with focused tests.

## Best Possible Solution

Land one maintainer-approved hook contract that can narrow the prompt surface per turn while preserving the existing system-prompt/provider-transform pipeline and every model-visible tool class, with regression tests for hosted tools, skill filtering, and combined hook prompt mutations.

## Reproduction Assessment

Yes for the review findings from source inspection. A hook returning `skillsAllow` plus prompt mutations enters the new rebuild branch that replaces the already-mutated prompt, and a hook deriving `toolsAllow` from `availableTools` cannot see `clientToolDefs` because they are added through a separate session path.

## Solution Assessment

No, not as implemented. The capability is useful, but the implementation should reuse the normal prompt-building/provider-transform path, include all model-visible tool classes, and resolve the public API direction relative to the open tool-resolve proposal.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Preserve hook prompt mutations when filtering skills:** `src/agents/pi-embedded-runner/run/attempt.ts:1952-1953`
  - body: When `skillsAllow` is set, this rebuilds `systemPromptText` from `buildEmbeddedSystemPrompt(appendPromptParams)` after the hook contexts were already applied. That drops `hookResult.systemPrompt`, prepend/append system context, and the normal `transformProviderSystemPrompt` path, so a plugin that narrows skills and also adds system guidance silently loses that guidance for the turn. Apply skill narrowing before the existing prompt pipeline or rebuild through the same helper used for the normal prompt.
  - confidence: 0.88
- **[P2] Include client-hosted tools in hook allowlists:** `src/agents/pi-embedded-runner/run/attempt.ts:1840`
  - body: `availableTools` is populated from `effectiveTools`, but client-hosted tools are added separately as `clientToolDefs` and then included in `allCustomTools` for the session. A classifier cannot include tools it never sees, and passing that allowlist to `setActiveToolsByName(...)` can remove hosted tools from the active session. Include hosted tool names in the hook-visible catalog or preserve the existing hosted-tool allowlist when narrowing.
  - confidence: 0.84
