## What This Changes

The PR extends `before_prompt_build` with tool and skill catalogs plus `toolsAllow`/`skillsAllow`, then wires those allowlists into embedded-runner tool/session and skills-prompt assembly with focused tests.

## Best Possible Solution

Land a documented hook seam that exposes the complete model-visible tool catalog and resolved skill catalog, then narrows only through existing policy, session, snapshot, and prompt-builder paths.

## Reproduction Assessment

Yes. Source inspection shows current main lacks tool/skill catalogs on `before_prompt_build`, and the PR-head blockers are traceable through the current client-tool, snapshot-skill, and prompt-builder paths.

## Solution Assessment

No, not as written. The hook seam is plausible, but the implementation needs a rebase and fixes so it exposes the full model-visible tool catalog and rebuilds narrowed skill prompts through the existing snapshot and provider-transform behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Include client-hosted tools in the hook catalog:** `src/agents/pi-embedded-runner/run/attempt.ts:1839`
  - body: `availableTools` is derived from `effectiveTools`, but client-hosted tools are added separately through `clientToolDefs` into `allCustomTools`. A plugin that mirrors the exposed list cannot preserve those tools in `toolsAllow`, so `setActiveToolsByName()` can remove model-visible tools the hook never saw.
  - confidence: 0.86
- **[P2] Filter snapshot skills without reloading live entries:** `src/agents/pi-embedded-runner/run/attempt.ts:1927-1938`
  - body: In snapshot-backed runs, this branch reloads current workspace skill entries and rebuilds from disk. Current `resolveSkillsPromptForRun()` preserves `skillsSnapshot.prompt`, so this can inject newer skill content or drop snapshot skills and break snapshot consistency.
  - confidence: 0.82
- **[P2] Rebuild filtered skills through the normal prompt path:** `src/agents/pi-embedded-runner/run/attempt.ts:1946-1952`
  - body: When `skillsAllow` is set, this rebuilds directly from `appendPromptParams` after hook system-prompt mutations have already been applied. Returning `skillsAllow` with `systemPrompt` or prepend/append system context can lose those edits and skip the provider transform path.
  - confidence: 0.9
