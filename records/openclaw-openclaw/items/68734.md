## Summary

Keep open. The feature direction is useful and backed by real-run logs, but current main still lacks this public hook-narrowing API and the PR head has prompt/session correctness blockers that need maintainer review before merge.

## What This Changes

The PR extends `before_prompt_build` so plugins can see available tool and skill catalogs and return `toolsAllow`/`skillsAllow` to narrow the per-turn model prompt surface.

## Best Possible Solution

Land a rebased, documented hook-narrowing contract that exposes complete model-visible catalogs and applies filters through canonical session and prompt assembly while preserving snapshots, hook mutations, provider transforms, and focused tests.

## Reproduction Assessment

Yes, for the PR defects by source inspection rather than a current-main bug reproduction. Current main lacks the proposed hook fields, and the PR head shows omitted client-hosted tools, snapshot reloads, and raw prompt rebuilds that would misbehave after merge.

## Solution Assessment

No, not as written. The hook seam is plausible, but the implementation needs to be rebased into current `buildAttemptSystemPrompt` and session-tool assembly rather than applying post-hoc prompt rebuilds.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Include client-hosted tools in hook catalogs:** `src/agents/pi-embedded-runner/run/attempt.ts:1839`
  - body: `availableTools` is derived from `effectiveTools`, but client-hosted tools are registered separately in `allCustomTools`. A plugin that mirrors the exposed catalog cannot include those client tool names, so `setActiveToolsByName(...)` can remove tools the hook never had a chance to preserve.
  - confidence: 0.88
- **[P2] Filter snapshot skills from the captured snapshot:** `src/agents/pi-embedded-runner/run/attempt.ts:1927-1938`
  - body: For snapshot-backed runs this reloads current workspace skill entries from disk and filters those, instead of filtering the captured `skillsSnapshot`. Snapshot runs should stay tied to captured skill content, otherwise filtering can inject newer skills or drop snapshot-only entries.
  - confidence: 0.86
- **[P2] Preserve hook and provider prompt transforms:** `src/agents/pi-embedded-runner/run/attempt.ts:1946-1952`
  - body: When `skillsAllow` is set, this rebuilds from `appendPromptParams` and applies it directly. That can drop earlier hook system-prompt mutations and bypass the provider transform path used by normal prompt assembly.
  - confidence: 0.9
