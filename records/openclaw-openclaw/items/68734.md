## Summary

This is an active external feature PR for a new plugin hook contract, and current main does not already provide the requested dynamic prompt-hook narrowing. The direction has useful deployment evidence, but the branch still needs correctness fixes around client-hosted tools, snapshot-backed skills, and preserving hook/provider prompt mutations before it can merge.

## What This Changes

The PR extends `before_prompt_build` hook events and results with available tool/skill catalogs plus `toolsAllow` and `skillsAllow`, then wires embedded-runner logic and tests to narrow per-turn model tool and skill prompt surfaces.

## Best Possible Solution

Rebase the feature onto current main, keep the plugin-owned narrowing contract, expose every model-visible tool and already-resolved skill source to the hook, and rebuild narrowed prompts through the normal prompt composition/provider-transform path with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main lacks the hook fields and runner wiring, and PR-head edge cases are traceable from the diff against the current embedded-runner tool and skill paths.

## Solution Assessment

No, not as written. The feature direction is plausible and supported by real deployment evidence, but the current patch is not the narrow maintainable solution until the client-tool, snapshot-skill, and prompt-rebuild issues are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve hook prompt edits when filtering skills:** `src/agents/pi-embedded-runner/run/attempt.ts:1946-1952`
  - body: When `skillsAllow` is set, the branch rebuilds `systemPromptText` from `buildEmbeddedSystemPrompt(appendPromptParams)` after hook prompt mutations have already been applied. That can drop `systemPrompt`/prepend/append system context changes for the same turn and bypass the normal provider system-prompt transform path.
  - confidence: 0.9
- **[P2] Expose client tools to prompt-build hooks:** `src/agents/pi-embedded-runner/run/attempt.ts:1839-1840`
  - body: `availableTools` is built from `effectiveTools`, but client-hosted tools are registered separately through `clientToolDefs`/`allCustomTools`. A plugin deriving `toolsAllow` from the exposed catalog cannot include those model-visible tools, and the active-tool update can then remove tools the hook never saw.
  - confidence: 0.86
- **[P2] Filter snapshot skills without live workspace reload:** `src/agents/pi-embedded-runner/run/attempt.ts:1927-1938`
  - body: For snapshot-backed runs, the branch reloads current workspace skill entries before filtering. That can inject newer on-disk skill content or drop skills that existed in the session snapshot, breaking the snapshot consistency preserved by the current skills runtime.
  - confidence: 0.78
