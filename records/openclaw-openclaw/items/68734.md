## Summary

Keep open. Current main does not implement hook-driven tool or skill narrowing, and the PR addresses a real token-cost use case, but the branch still has blocking prompt/session correctness issues in a new public plugin API surface.

## What This Changes

This PR extends `before_prompt_build` with available tool/skill catalogs and `toolsAllow`/`skillsAllow`, then applies hook allowlists in embedded runner prompt/session paths with focused tests.

## Best Possible Solution

Land a rebased, documented hook implementation that exposes the complete model-visible catalogs and applies allowlists through canonical prompt/session assembly while preserving snapshots, provider transforms, and hook prompt mutations.

## Reproduction Assessment

Yes, as source-level PR-defect checks rather than a current-main bug reproduction. Current main lacks this capability, while the PR head shows catalog, snapshot, and prompt rebuild paths that would misbehave.

## Solution Assessment

No, not as written. The hook seam is plausible, but the implementation should be rebased onto current main and feed allowlists through the existing prompt/session helpers rather than rebuilding older assembled state afterward.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Include client-hosted tools in hook catalogs:** `src/agents/pi-embedded-runner/run/attempt.ts:1839`
  - body: `availableTools` is derived from `effectiveTools`, but client-hosted tools are appended later through `clientToolDefs` into `allCustomTools`. A plugin that mirrors the catalog can therefore return an allowlist that drops client-hosted tools it never had a chance to see.
  - confidence: 0.88
- **[P2] Filter snapshot skills from the captured snapshot:** `src/agents/pi-embedded-runner/run/attempt.ts:1927-1938`
  - body: For snapshot-backed runs this reloads current workspace skill entries from disk and filters those, instead of filtering the captured `skillsSnapshot`. Snapshot runs should stay tied to captured skill content, otherwise newer workspace skills can be injected or snapshot-only entries can disappear.
  - confidence: 0.86
- **[P2] Preserve prompt transforms when filtering skills:** `src/agents/pi-embedded-runner/run/attempt.ts:1946-1952`
  - body: When `skillsAllow` is set, this rebuilds from the original append-prompt params and applies it directly. That can drop earlier `systemPrompt` or prepend/append system-context hook mutations and bypass the provider transform used for normal prompt assembly.
  - confidence: 0.9
