## Summary

Keep open. Current main does not implement the requested hook-side tool/skill narrowing, and the PR has useful real deployment context, but the branch is not correct enough to merge because it can miss model-visible client tools, break snapshot-backed skill consistency, and lose prompt mutations/provider transforms.

## What This Changes

The PR extends before_prompt_build hook contracts and embedded-runner wiring so plugins can return per-turn tool and skill allowlists.

## Best Possible Solution

Rebase onto current main, preserve the newer prompt-helper composition and snapshot semantics, expose every model-visible tool and resolved skill source to hooks, and rebuild narrowed prompts through the normal provider-transform path with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main lacks hook catalogs and allowlists, and the PR-head client-tool, snapshot-skill, stale-helper, and prompt-rebuild issues are traceable from the diff and current-main code without running tests.

## Solution Assessment

No, not as written. The feature direction is plausible and supported by deployment evidence, but the current branch is not the narrowest maintainable solution until it fixes the client-tool catalog, snapshot consistency, prompt rebuild, and rebase issues.

- **[P2] Preserve current hook context merging on rebase:** `src/agents/pi-embedded-runner/run/attempt.prompt-helpers.ts:93-96`

  - confidence: 0.88
- **[P2] Expose client tools to prompt-build hooks:** `src/agents/pi-embedded-runner/run/attempt.ts:1839-1844`
  - body: availableTools is built from effectiveTools, but client-hosted tools are registered separately through clientToolDefs/allCustomTools. A plugin deriving toolsAllow from the exposed catalog cannot include those model-visible tools, and setActiveToolsByName(...) can remove tools the hook never saw.
  - confidence: 0.86
- **[P2] Filter snapshot skills without live workspace reload:** `src/agents/pi-embedded-runner/run/attempt.ts:1927-1938`
  - body: For snapshot-backed runs, this reloads current workspace skill entries before filtering. That can inject newer on-disk skill content or drop skills that existed in the session snapshot, breaking the snapshot consistency current main preserves.
  - confidence: 0.78
- **[P2] Preserve hook prompt edits when filtering skills:** `src/agents/pi-embedded-runner/run/attempt.ts:1946-1952`
  - body: When skillsAllow is set, this rebuilds systemPromptText from appendPromptParams after hook system-prompt mutations were already applied. That can drop same-turn systemPrompt/prepend/append system context and bypass the normal provider system-prompt transform path.
  - confidence: 0.9
