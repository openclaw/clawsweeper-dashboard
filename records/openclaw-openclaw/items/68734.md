## Summary

Keep open: this is a recent external feature PR with real deployment support, but current main does not already provide the requested hook-based narrowing and the PR still has correctness issues in the runtime wiring.

## What This Changes

The branch extends prompt-build hooks so plugins can see available tools/skills and return per-turn tool and skill allow-lists.

## Best Possible Solution

Rebase onto current main, keep one maintained plugin-owned narrowing contract, expose all model-visible tools and already-resolved skills to hooks, and rebuild narrowed prompts through the normal prompt composition/provider-transform path.

## Reproduction Assessment

Yes for review. Source inspection shows current main lacks the hook catalogs and allow-list fields, and PR-head source inspection shows the client-tool, snapshot-skill, and prompt-rebuild edge cases without needing a local test run.

## Solution Assessment

No, not as written. The direction is plausible, but the patch should include client-hosted tools, preserve snapshot skill consistency, and rebuild filtered skill prompts through the normal transformed system-prompt path before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve hook prompt edits when filtering skills:** `src/agents/pi-embedded-runner/run/attempt.ts:1946-1952`
  - body: When `skillsAllow` is set, this branch rebuilds `systemPromptText` from `buildEmbeddedSystemPrompt(appendPromptParams)` after hook prompt mutations have already been applied. That can drop `systemPrompt` or prepend/append system context changes for the same turn and bypass the normal provider system-prompt transform path.
  - confidence: 0.9
- **[P2] Expose client tools to prompt-build hooks:** `src/agents/pi-embedded-runner/run/attempt.ts:1839-1840`
  - body: `availableTools` is built from `effectiveTools`, but client-hosted tools are registered separately through `clientToolDefs` and `allCustomTools`. A plugin deriving `toolsAllow` from the exposed catalog cannot include those model-visible tools, and the active-tool update can remove tools the hook never saw.
  - confidence: 0.86
- **[P2] Filter snapshot skills without live workspace reload:** `src/agents/pi-embedded-runner/run/attempt.ts:1927-1938`
  - body: For snapshot-backed runs, this branch reloads current workspace skill entries before filtering. That can inject newer on-disk skill content or drop skills that existed in the session snapshot, breaking the snapshot consistency preserved by the current skills runtime.
  - confidence: 0.78
