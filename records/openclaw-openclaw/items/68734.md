## Summary

Keep open. The capability gap is real and the discussion includes concrete MCP-heavy deployment evidence, but this PR is not merge-ready because it has blocking prompt/tool/skill path bugs and currently conflicts with main.

## What This Changes

The PR extends before_prompt_build hooks to expose tool and skill catalogs and return toolsAllow/skillsAllow filters, then wires those filters through the embedded runner with focused tests.

## Best Possible Solution

Land a documented hook seam that exposes the complete model-visible tool catalog and resolved skill catalog, then narrows through existing policy, session, snapshot, and provider prompt-builder paths without losing hook prompt mutations.

## Reproduction Assessment

Yes. Source inspection of current main shows before_prompt_build receives only prompt/messages and cannot return tool or skill allowlists, so the feature gap is verifiable without a live gateway.

## Solution Assessment

No, not as written. The hook seam is plausible, but the current patch must handle client-hosted tools, snapshot-backed skills, and provider/hook prompt mutation paths before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Expose client-hosted tools to prompt hooks:** `src/agents/pi-embedded-runner/run/attempt.ts:1839`
  - body: availableTools is derived from effectiveTools, but client-hosted tools are registered separately in allCustomTools and sent to the model. A plugin mirroring the exposed list into toolsAllow cannot preserve those tools, so setActiveToolsByName() can remove tools the hook never exposed.
  - confidence: 0.86
- **[P2] Reuse snapshot skills when filtering:** `src/agents/pi-embedded-runner/run/attempt.ts:1927-1938`
  - body: In snapshot-backed runs this branch reloads current workspace skill entries and rebuilds from live disk state. That can inject newer skill content or drop snapshot-only skills during a run that should remain snapshot-consistent.
  - confidence: 0.8
- **[P2] Preserve prompt mutations when narrowing skills:** `src/agents/pi-embedded-runner/run/attempt.ts:1946-1952`
  - body: When skillsAllow is set, this rebuilds systemPromptText from appendPromptParams after hook system-prompt mutations were applied. That can drop systemPrompt/prepend/append edits and bypass the normal provider transform path for the turn.
  - confidence: 0.9
