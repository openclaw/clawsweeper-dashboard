## Summary

Keep open: the PR addresses a valid prompt-surface limitation, but the current branch still has correctness issues around client-hosted tools, snapshot-backed skills, and prompt mutation preservation, plus a public hook API decision that needs maintainer review.

## What This Changes

The PR extends `before_prompt_build` with tool and skill catalogs plus optional `toolsAllow` and `skillsAllow`, then applies those allowlists in the embedded runner with focused hook tests.

## Best Possible Solution

Land one documented, backward-compatible prompt-surface narrowing seam that exposes the complete model-visible tool catalog and resolved skill catalog while preserving policy, snapshots, provider transforms, and hook prompt mutations.

## Reproduction Assessment

Yes. Source inspection of current main shows `before_prompt_build` receives only prompt/messages and its result type cannot return tool or skill allowlists, so the motivating limitation is source-reproducible.

## Solution Assessment

No, not as written. The hook seam is plausible, but this branch needs fixes for client-hosted tools, snapshot skill filtering, prompt mutation preservation, and maintainer API-boundary review before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Expose client-hosted tools to prompt hooks:** `src/agents/pi-embedded-runner/run/attempt.ts:1839-1840`
  - body: `availableTools` is derived from `effectiveTools`, but client-hosted tools are added separately through `clientToolDefs` into `allCustomTools` and sent to the model. A plugin that mirrors the exposed list cannot include those tools in `toolsAllow`, so `setActiveToolsByName()` can remove model-visible tools the hook never exposed.
  - confidence: 0.86
- **[P2] Reuse snapshot skills when filtering:** `src/agents/pi-embedded-runner/run/attempt.ts:1927-1938`
  - body: In snapshot-backed runs this branch reloads current workspace skill entries and rebuilds from live disk state. Current `resolveSkillsPromptForRun()` returns the snapshot prompt, so this can inject newer skill content or drop snapshot-only skills during a run that should stay snapshot-consistent.
  - confidence: 0.82
- **[P2] Preserve hook prompt mutations when narrowing skills:** `src/agents/pi-embedded-runner/run/attempt.ts:1946-1952`
  - body: When `skillsAllow` is set, this rebuilds from `appendPromptParams` and applies the result after hook system-prompt mutations. Returning `skillsAllow` with `systemPrompt` or prepend/append system context can lose those prompt edits and bypass the normal provider transform path.
  - confidence: 0.9
