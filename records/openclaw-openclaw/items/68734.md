## Summary

Keep open: current main has not implemented the hook allowlist contract, and this PR remains useful, but the current head still has source-backed P2 correctness gaps around client tools, snapshot skills, and prompt rebuilding.

## What This Changes

The PR extends `before_prompt_build` with tool and skill catalogs plus `toolsAllow`/`skillsAllow`, wires those allowlists into the embedded runner, and adds focused hook tests.

## Best Possible Solution

Land a documented hook seam that exposes the complete model-visible tool catalog and resolved skill catalog, then narrows through existing policy, session, and prompt-builder seams without breaking snapshots, provider transforms, or hook prompt mutations.

## Reproduction Assessment

Yes. Source inspection on current main shows the hook receives only `prompt` and `messages`, and the PR-head blockers are traceable from the diff plus current client-tool, skill snapshot, and prompt-builder paths.

## Solution Assessment

No, not as written. The hook seam is plausible, but the implementation should be rebased onto current main and fixed to expose the complete tool catalog, preserve snapshot skill data, and rebuild skills through the normal prompt mutation and provider-transform path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Expose client-hosted tools to prompt hooks:** `src/agents/pi-embedded-runner/run/attempt.ts:1839`
  - body: `availableTools` is built from `effectiveTools`, but client-hosted tools are added separately through `clientToolDefs` and included in `allCustomTools`. A plugin deriving `toolsAllow` from the exposed catalog cannot keep those model-visible tools, and `setActiveToolsByName(...)` can remove tools the hook never saw.
  - confidence: 0.86
- **[P2] Filter snapshot skills from snapshot data:** `src/agents/pi-embedded-runner/run/attempt.ts:1927-1938`
  - body: For snapshot-backed runs, this reloads current workspace skill entries before filtering. That can inject newer on-disk skill content or drop skills that existed in the session snapshot, breaking the snapshot consistency current main preserves.
  - confidence: 0.8
- **[P2] Preserve prompt mutations when rebuilding skills:** `src/agents/pi-embedded-runner/run/attempt.ts:1946-1952`
  - body: When `skillsAllow` is set, this rebuilds `systemPromptText` from the original prompt params after hook system-prompt mutations were applied. Runs that return `skillsAllow` together with `systemPrompt`, prepend, or append system context can silently lose those same-turn prompt edits and provider transforms.
  - confidence: 0.9
