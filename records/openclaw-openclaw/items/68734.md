## Summary

Keep open: current main still lacks hook-owned dynamic tool or skill narrowing, and the discussion includes real deployment evidence that this matters. The PR is a plausible implementation direction with real after-fix logs, but it still has blocking correctness issues around client-hosted tools, snapshot-backed skills, and prompt rebuild semantics.

## What This Changes

The branch extends `before_prompt_build` hooks to receive tool and skill catalogs and return `toolsAllow`/`skillsAllow`, wiring those allowlists into embedded agent tool selection and skills prompt rebuilding with focused tests.

## Best Possible Solution

Land a documented narrowing seam that exposes the complete model-visible tool catalog and a snapshot-consistent skill catalog, then applies filtering through the existing session, provider transform, and prompt mutation paths without broadening access.

## Reproduction Assessment

Yes. Source inspection of current main shows `before_prompt_build` receives only prompt/messages and cannot return tool or skill allowlists, so the feature gap is reproducible without a live gateway.

## Solution Assessment

No, not as written. The hook seam is plausible, but the current patch must include client-hosted tools, preserve snapshot-backed skills, and rebuild filtered prompts through the normal mutation/provider transform path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Expose client-hosted tools to prompt hooks:** `src/agents/pi-embedded-runner/run/attempt.ts:1839`
  - body: `availableTools` is derived from `effectiveTools`, but client-hosted tools are added later through `clientToolDefs` into `allCustomTools` and sent to the model. A plugin that mirrors the exposed list cannot include those tools in `toolsAllow`, so `setActiveToolsByName()` can remove model-visible client tools the hook never exposed.
  - confidence: 0.86
- **[P2] Reuse snapshot skills when filtering:** `src/agents/pi-embedded-runner/run/attempt.ts:1927-1938`
  - body: In snapshot-backed runs this branch reloads current workspace skill entries and rebuilds from live disk state. Current `resolveSkillsPromptForRun()` returns the snapshot prompt before considering live entries, so this can inject newer skill content or drop snapshot-only skills during a run that should stay snapshot-consistent.
  - confidence: 0.82
- **[P2] Preserve prompt mutations when narrowing skills:** `src/agents/pi-embedded-runner/run/attempt.ts:1946-1952`
  - body: When `skillsAllow` is set, this rebuilds from `appendPromptParams` and applies the result after hook system-prompt mutations. Returning `skillsAllow` with `systemPrompt` or prepend/append system context can lose those prompt edits and bypass the normal provider transform path.
  - confidence: 0.9
