## What This Changes

The PR adds a result-marker guard and focused unit tests so task completion summaries that start with progress narration can be accepted when they include final result text.

## Best Possible Solution

Keep the fix in the shared task completion contract, but constrain result-marker acceptance to actual final-report contexts while preserving blocks for future or conditional progress narration, then require real task-flow proof before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path. Current main blocks comma-joined progress-prefixed summaries, while the PR regex would also accept conditional progress-only text containing `tests passed` or `done`.

## Solution Assessment

No, not as written. The shared classifier is the right layer, but the marker rescue needs to be contextual rather than an early anywhere-in-text override.
