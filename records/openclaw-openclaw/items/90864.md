## Summary

Keep open: this is a useful classifier fix, but the current diff is not merge-ready because the broad marker guard can classify future or conditional progress text as a successful required completion.

## What This Changes

The PR adds a result-marker guard and focused tests so task completion summaries with later result/report/verification markers are not treated as progress-only.

## Best Possible Solution

Keep the classifier-only direction, but narrow result markers to explicit result/report sections or clearly completed-result clauses and add counterexamples for future or conditional progress text.

## Reproduction Assessment

Yes. Source inspection shows current main still has a classifier path that can block single-sentence progress-prefixed reports with later result content, and the PR's new regex has a direct false-positive path for conditional/future progress text.

## Solution Assessment

No. The classifier is the right layer, but this implementation is too broad; the safer fix is a narrower result-marker guard plus conditional/future counterexample coverage.
