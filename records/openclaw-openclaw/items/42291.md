## Summary

Keep open: current main and `v2026.6.8` still run provider auth login against the local agent auth store with no remote node-host guard, and the only open closing PR is conflicting and not maintainer-editable.

## What This Changes

The issue reports that `openclaw models auth login` on a remote node host writes OAuth credentials to the node-local auth store instead of the gateway that performs model calls.

## Best Possible Solution

Land a focused preflight that detects remote node-host gateway config and blocks or warns before OAuth starts, with regression coverage for non-loopback and supported local cases.

## Reproduction Assessment

Yes, at source level: current main resolves a local `agentDir`, runs provider auth, and upserts returned credentials there while node-host config can point at a non-loopback gateway.

## Solution Assessment

Yes, blocking or warning before auth starts on remote node hosts is the narrowest maintainable fix. Automatic gateway credential forwarding would be a larger API design.
