## Summary

Keep open: current main still writes `models auth login` credentials through the local agent auth store, and the active fix path is the still-open closing PR. This issue should close only after that PR or an equivalent replacement lands.

## What This Changes

The issue asks for node-host `models auth login` to avoid silently storing OAuth credentials locally when model calls use a remote gateway.

## Best Possible Solution

Land, rebase, or replace https://github.com/openclaw/openclaw/pull/42381 with a focused guard that blocks non-loopback node-host auth-login writes using shared loopback semantics and clear gateway-host guidance; defer credential sync to an explicit API design.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `models auth login` resolves a local agent directory and persists profiles there, while node-host config can point at a remote gateway that owns model-call state; I did not run a live OAuth flow.

## Solution Assessment

Yes. Keeping this issue open while the linked implementation PR is reviewed, rebased, merged, or replaced is the best workflow; the narrow maintainable fix is a remote node-host guard with actionable guidance, not implicit gateway credential sync.
