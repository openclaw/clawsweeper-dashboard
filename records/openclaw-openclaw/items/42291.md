## Summary

Keep open: current main still lets `models auth login` resolve and write provider credentials through the local agent auth store, while an open closing PR remains the active fix path for blocking remote node-host writes.

## What This Changes

The issue reports that running `openclaw models auth login` on a node host connected to a remote gateway writes OAuth credentials to the node-local auth store instead of the gateway-owned store.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/42381 with a focused pre-auth guard that blocks non-loopback node-host auth-login writes before provider auth starts.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: a node host can persist a non-loopback gateway in `node.json`, and current `models auth login` still resolves a local `agentDir` and writes auth profiles without checking that node-host state.

## Solution Assessment

Yes. A pre-auth remote-node guard with gateway-host guidance is the narrow maintainable fix; automatic credential sync would need a larger gateway auth-profile API and token-safety design.
