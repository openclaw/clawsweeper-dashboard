## Summary

Keep open: current main and the latest release still persist provider auth profiles through the local agent store with no remote node-host guard, and the open closing PR has not merged.

## What This Changes

The issue asks `openclaw models auth login` on a remote node host to avoid silently writing OAuth credentials to the node-local auth store when model calls run on the gateway.

## Best Possible Solution

Land a focused auth-login preflight that detects remote node-host gateway config with shared loopback semantics and blocks before OAuth starts, with regression coverage for remote and loopback cases.

## Reproduction Assessment

Yes, at source level. Current main resolves a local `agentDir`, runs provider auth, and upserts returned credentials there while node-host config can point at a non-loopback gateway.

## Solution Assessment

No shipped/current solution exists yet. The narrowest maintainable fix is to block or warn before auth starts on remote node hosts unless maintainers explicitly design a gateway credential-forwarding API.
