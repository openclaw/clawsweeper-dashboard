## Summary

Keep open: current main still runs `models auth login` through local auth-profile persistence with no remote node-host guard, and the open closing PR remains the active fix path.

## What This Changes

`openclaw models auth login` on a remote node host should not silently write OAuth credentials to the node-local auth store when model calls run on the gateway.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/42381 with a focused pre-auth guard that blocks non-loopback node-host auth-login writes before provider auth starts.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: configure a node host with a non-loopback saved gateway and run `models auth login`; current main proceeds into local provider auth persistence with no node-host guard.

## Solution Assessment

Yes. A pre-auth remote-node guard with clear gateway-host guidance is the narrow maintainable fix; automatic credential sync would need a larger gateway auth-profile API and token-safety design.
