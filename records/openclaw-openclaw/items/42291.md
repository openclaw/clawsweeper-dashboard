## Summary

Keep open: current main still runs provider auth and persists returned profiles through the local agent auth store, while the open closing PR remains unmerged and currently needs repair before it can resolve the issue.

## What This Changes

The issue asks `openclaw models auth login` to avoid silently storing OAuth credentials on a node host when model calls use a remote gateway.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/42381 with a focused pre-auth guard that blocks non-loopback node-host auth login before any provider token exchange and directs the user to authenticate on the gateway host.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a node host can persist a non-loopback gateway in `node.json`, while current `models auth login` still writes provider profiles through the local `agentDir`.

## Solution Assessment

Yes. A pre-auth remote-node guard with gateway-host guidance is the narrow maintainable fix; automatic credential sync would require a larger gateway auth-profile API and token-safety design.
