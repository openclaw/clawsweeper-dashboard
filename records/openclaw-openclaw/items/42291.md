## What This Changes

Ensure `openclaw models auth login` on a remote node host does not silently write OAuth credentials only to the node-local auth store.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/42381 with a focused guard that blocks non-loopback node-host auth-login writes and gives clear gateway-host guidance; gateway credential sync should wait for an explicit API/product design.
