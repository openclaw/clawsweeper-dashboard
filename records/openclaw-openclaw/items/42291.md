## Summary

Keep open: current main still lets `models auth login` run provider auth and persist credentials through the local agent auth store without checking remote node-host state, and the open closing PR remains the active fix path.

## What This Changes

The issue asks OpenClaw to stop `openclaw models auth login` on a remote node host from silently writing OAuth credentials to the node-local auth store instead of the gateway.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/42381 with a focused pre-auth guard that blocks non-loopback node-host auth-login before provider auth starts and tells the user to run auth on the gateway host.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: node-host config can persist a non-loopback gateway in `node.json`, while current `models auth login` resolves a local `agentDir` and writes auth profiles without checking that node-host state.

## Solution Assessment

Yes. A pre-auth remote-node guard with gateway-host guidance is the narrow maintainable fix; automatic credential sync would require a larger gateway auth-profile API and token-safety design.
