## Summary

Current main still has the reported local OAuth persistence path, and an open PR already uses closing syntax for this issue, so this should stay open until that PR is reviewed, rebased, merged, or replaced.

## What This Changes

The issue asks `openclaw models auth login --provider openai-codex` to avoid silently writing OAuth credentials to a node-local auth store when model execution uses a remote gateway.

## Best Possible Solution

Review and land PR #42381, or replace it with an equivalent narrow fix that blocks remote node-host OAuth login with clear guidance unless maintainers intentionally add a gateway credential-sync API.

## Reproduction Assessment

Yes, from source inspection rather than a live remote setup. Current main resolves a local agent directory and persists OAuth profiles there without checking remote node-host mode.

## Solution Assessment

Yes. Keeping this issue open is the right workflow because an open PR already references it with closing syntax; the narrow maintainable fix is a guard and actionable error, unless maintainers choose a larger gateway credential-sync feature.
