## Summary

Keep open: current main still resolves and persists `models auth login` credentials through the local agent store with no remote node-host guard, and an open closing PR remains the active fix path.

## What This Changes

The issue asks `openclaw models auth login` to avoid silently writing provider OAuth credentials to a node host when model calls run through a remote gateway.

## Best Possible Solution

Repair and land https://github.com/openclaw/openclaw/pull/42381, or replace it with an equivalent focused guard that blocks non-loopback node-host auth-login writes before provider auth starts.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: configure a node host with a non-loopback gateway and run `models auth login`; current code proceeds to local provider auth persistence. I did not run a live browser OAuth flow.

## Solution Assessment

Yes, the narrow maintainable solution is a pre-auth remote node-host guard with clear gateway-host guidance. Implicit credential sync would need a larger gateway auth API design and should not be the first fix.
