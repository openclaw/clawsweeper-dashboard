## Summary

Keep open: current main and the latest release still route provider auth login through the local agent auth store, and the focused closing PR is open and conflicting rather than merged.

## What This Changes

The issue reports that `openclaw models auth login` on a remote node host stores OAuth credentials in the node-local auth store instead of the gateway-owned auth state.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/42381 with a focused pre-auth guard that blocks non-loopback node-host auth login before provider auth and tells users to run auth on the gateway host.

## Reproduction Assessment

Yes by source inspection: a remote node host can persist non-loopback gateway state in `node.json`, while current `models auth login` still writes returned provider profiles through the local `agentDir`. I did not run a live OAuth flow in this read-only review.

## Solution Assessment

Yes in direction: a pre-auth remote-node guard is the narrow maintainable fix; automatic credential syncing would require a larger gateway auth-profile API and token-safety design.
