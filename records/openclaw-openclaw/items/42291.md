## Summary

Current main still has the reported local OAuth persistence path, and an open linked PR already claims to fix it, so the issue should stay open until that PR is reviewed and merged or replaced.

## What This Changes

The issue requests that `openclaw models auth login` avoid silently writing OAuth credentials to node-local storage when model execution uses a remote gateway.

## Best Possible Solution

Review and land PR #42381, or replace it with an equivalent narrow fix that blocks remote node-host OAuth login with a clear error unless a deliberate gateway credential-sync API is added.

## Reproduction Assessment

Yes from source inspection, though not from a live remote setup. Current main resolves a local `agentDir` and persists OAuth profiles with `upsertAuthProfile` against that directory without checking `gateway.remote.url` or node-host mode.

## Solution Assessment

Yes. Keeping this issue open is the correct workflow because PR #42381 already uses closing syntax; the narrow maintainable fix is a guard with actionable guidance unless maintainers intentionally add a gateway-side credential sync path.
