## What This Changes

The issue reports malformed local GLM/Z.AI tool parameters, especially `process.action: {}`, causing repeated process-tool failures or an agent retry loop.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81157, or an equivalent narrow runtime guard, so invalid `process.action` values fail before session handling while provider-facing schemas stay stable.

## Reproduction Assessment

No high-confidence live GLM reproduction was run in this read-only review. Source inspection does show current main has no early invalid-action guard for malformed `process.action` values before session-id handling.

## Solution Assessment

Yes. The linked runtime boundary validation is the narrowest maintainable fix shape; blanket parameter cleaning or changing the provider-facing schema would be broader and riskier without a raw provider fixture.
