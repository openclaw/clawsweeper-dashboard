## What This Changes

The issue reports malformed tool-call parameters from a local GLM/Z.AI model, especially `process({ action: {} })`, leading to repeated process tool invocation failures instead of a shape-aware rejection.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81157, or an equivalent narrow process-action boundary guard, and track any remaining provider-specific edit-parameter suffix corruption separately with raw tool-call evidence.

## Reproduction Assessment

Yes at source level, but not by a fresh live run in this review. Current `main` still casts `process` args and checks `sessionId` before rejecting invalid action shapes, matching the reported `action: {}` retry path.

## Solution Assessment

Yes for the high-confidence process-action part of the report. The narrowest maintainable fix is an early runtime guard in `createProcessTool().execute`, which is the linked PR's approach; broad provider-side cleanup would need stronger raw provider evidence.
