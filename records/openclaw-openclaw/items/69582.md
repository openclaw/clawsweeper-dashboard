The issue asks OpenClaw to stop malformed `{}` tool parameters from causing repeated tool invocations, especially `process.action` becoming an object under local/glm-5.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81157 if it remains correct, keeping provider-wide malformed-argument cleanup for a raw-fixture-backed follow-up.

## Solution Assessment

Yes for the linked PR's narrow direction: validating `process.action` before session lookup is the smallest maintainable fix for the verified symptom. Blanket stripping trailing `{}` from arbitrary strings is not the best fix without provider transcript evidence.
