## Summary

Keep open. Current main still lacks a schema-aware guard for the reported malformed `process(action={})` symptom, and the existing malformed stream repair does not safely cover valid-but-wrong typed arguments or literal `{}` inside string values.

## What This Changes

Requested cleanup for GLM tool calls where `{}` contaminates edit/process parameters and can lead to repeated failed tool invocations.

## Best Possible Solution

Add narrow schema-aware validation at the OpenClaw tool invocation boundary for invalid `process.action` types/values, and only add GLM/Z.AI-specific argument repair if a raw streamed-argument fixture proves it is a provider parse artifact.

## Reproduction Assessment

Unclear for the full GLM injection because no raw provider transcript or minimal live fixture is present. There is a high-confidence source reproduction for the reported `process(action={})` symptom: current main can route the malformed object to the generic missing-`sessionId` failure path.

## Solution Assessment

No, blanket parameter cleaning is not the best fix because `{}` can be valid user text. The safer approach is targeted runtime validation with non-retry guidance, plus fixture-backed provider repair only for malformed whole tool-call payloads.
