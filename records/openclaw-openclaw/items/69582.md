## What This Changes

The issue reports malformed `{}` tool parameters, especially `process.action: {}`, causing repeated tool-invocation retries with local/glm-5.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81157 if it remains correct, then handle any broader provider-side `{}` cleanup only with raw fixture evidence.

## Reproduction Assessment

Unclear for the full local/glm-5 infinite-loop report because no raw provider/tool-call transcript is attached. The narrow `process(action={})` boundary is source-reproducible because current main routes malformed non-`list` actions through session handling before action-shape feedback.

## Solution Assessment

Yes for the linked PR's narrow direction: validating `process.action` before session lookup is the smallest maintainable fix for the verified symptom. Blanket stripping trailing `{}` from arbitrary strings is not the best fix without provider transcript evidence.
