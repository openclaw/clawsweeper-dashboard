## Summary

Keep open. Current main and the latest release still lack the process-tool boundary validation for malformed `process.action`, and the linked fix remains an open implementation path rather than merged behavior.

## What This Changes

The issue reports malformed local-model tool arguments such as `process({ action: {} })` causing misleading tool errors and retry loops instead of a clear invalid-argument refusal.

## Best Possible Solution

Review and land the linked PR, or an equivalent small runtime boundary guard, so invalid `process.action` values fail before `sessionId` handling while preserving provider-safe tool schemas.

## Reproduction Assessment

No high-confidence end-to-end GLM reproduction was run for this review. Source inspection does show a clear malformed-action path where current main handles `process({ action: {} })` without an early invalid-action refusal.

## Solution Assessment

Yes, the linked runtime boundary validation is the best known fix shape. Blanket parameter cleaning or schema-union changes would be broader and less provider-safe than rejecting invalid `action` values inside the process tool before session handling.
