## Summary

Keep open: the PR is a focused CLI UX change, but it currently wires suggestions only into Commander parse-error output while the advertised `openclaw upate` and `openclaw upgrade` flows stop earlier in `run-main.ts`; the external proof is also only unit/lint output, not real CLI behavior.

## What This Changes

Adds a Levenshtein-based CLI command suggester, tests it in the parse-error formatter, and passes registered root command names into Commander error formatting.

## Best Possible Solution

Share the command suggestion helper with both Commander parse errors and `resolveUnownedCliPrimaryMessage()`, preserving plugin-policy diagnostics and adding `run-main.exit.test.ts` coverage for typo, alias, and no-suggestion cases.

## Reproduction Assessment

Yes, by source inspection: current main rejects unowned roots before Commander and existing tests assert `buildProgramMock` is not called on that path. I did not run the CLI because this review was constrained to read-only inspection.

## Solution Assessment

No, not yet: the helper shape is reasonable, but wiring only Commander `outputError` misses the early unknown-primary path named by the linked request. The maintainable fix is to use one helper from both diagnostic paths.
