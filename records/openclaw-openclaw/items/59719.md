## Summary

Keep open: the bug direction is valid, but this PR is stale/conflicting and still has blocking correctness issues around broad CLI liveness, owner-task session overrides, delayed background yield, and missing real behavior proof.

## What This Changes

The PR links background exec/process sessions to runtime:"cli" task records, updates process remove/kill visibility, and derives session/status rows from linked task state with regression tests.

## Best Possible Solution

Rebase onto current main and implement exec-owned CLI task tracking that preserves runtime-specific task maintenance, scopes session row derivation to the linked exec task, refreshes heartbeats for all output, resolves background yield immediately, and includes redacted real behavior proof.

## Reproduction Assessment

Yes, source-reproducible. Current main's background yield path only marks the process backgrounded and resolves the running response, and the bash exec paths do not create a linked task-registry record.

## Solution Assessment

No. The direction is appropriate, but this branch is not the best implementation until CLI liveness and session-row overrides are scoped to exec-owned task records and the background yield path is not delayed by task writes.

- **[P1] Scope CLI orphan checks to exec-owned tasks:** `src/tasks/task-registry.maintenance.ts:75`
  - body: This routes every runtime:"cli" task with a runId through isCliRunActive, but gateway wires that hook to bash-process-registry liveness. Gateway agent and media tasks create runtime:"cli" rows with non-process run IDs, so live work can be marked lost after the short CLI grace.
  - confidence: 0.91

  - body: reportCliTaskProgress updates lastEventAt only inside if (summary), while ordinary ongoing output calls the helper without a summary. A noisy long command can keep running with stale task/session activity and be treated as inactive.
  - confidence: 0.86
- **[P2] Filter session rows to the linked exec task:** `src/gateway/session-utils.ts:1216-1221`
  - body: buildGatewaySessionRow snapshots all tasks for the owner key and lets the active/focused owner task override status and timing. An unrelated ACP, subagent, or media task owned by the same session can make a done row look running or failed.
  - confidence: 0.83
- **[P2] Resolve background yields before task writes:** `src/agents/bash-tools.exec.ts:1651`
  - body: onYieldNow awaits a dynamic import and task-registry write before resolving the running tool result. Slow storage can push the user-visible background response past the configured yield window even though the process has already been backgrounded.
  - confidence: 0.76
