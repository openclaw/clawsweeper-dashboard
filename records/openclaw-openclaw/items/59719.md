## Summary

Keep open: current main still lacks the central exec-linked CLI task behavior, but this PR is not merge-ready because the branch has source-confirmed session/task-liveness bugs, is currently conflicting, and still lacks after-fix real behavior proof.

## What This Changes

The PR links background exec/process runs to runtime:"cli" task records, changes process removal visibility, derives session/status state from linked tasks, adds CLI orphan maintenance, and expands regression coverage.

## Best Possible Solution

Rebase and land a scoped exec-owned CLI task implementation that preserves current main's runtime-aware task maintenance and only lets linked exec tasks drive session rows.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main backgrounds exec runs without creating a linked task-registry row, so status surfaces cannot derive liveness from that missing record. I did not run a live repro in this read-only review.

## Solution Assessment

No. The direction is appropriate, but this branch is not the best implementation until CLI liveness and session-row overrides are scoped to exec-owned task records and background yield latency is preserved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Scope CLI liveness checks to exec-owned tasks:** `src/tasks/task-registry.maintenance.ts:75`
  - body: This sends every runtime:"cli" task with a runId through isCliRunActive, but gateway wires that hook to the bash process registry. Existing gateway agent and media-generation CLI tasks use non-process run IDs, so live work can be marked lost after the short CLI grace window.
  - confidence: 0.91
- **[P2] Update CLI task activity on every output chunk:** `src/agents/bash-tools.exec-runtime.ts:678-690`
  - body: lastEventAt is only written when a progress summary is present, while normal ongoing stdout/stderr calls this helper without a summary. A noisy long command can keep running with stale task activity and disappear from active status filters.
  - confidence: 0.86
- **[P2] Filter session rows to the linked exec task:** `src/gateway/session-utils.ts:1216-1221`
  - body: The row builder snapshots all tasks for the owner key and lets the active/focused owner task override the session row. An unrelated ACP, media, or gateway task owned by the same session can make a done row look running or failed.
  - confidence: 0.83
- **[P2] Resolve the background yield before task writes:** `src/agents/bash-tools.exec.ts:1651-1652`
  - body: onYieldNow awaits a dynamic import and task-registry write before returning the running tool result. Slow storage can push the visible background response past the configured yield window even though the process has already been backgrounded.
  - confidence: 0.76
