Full review comments:

- **[P1] Scope CLI orphan checks to exec-owned tasks:** `src/tasks/task-registry.maintenance.ts:75`
  - body: This routes every runtime:"cli" task with a runId through the bash-process liveness observer, but current main also creates CLI tasks for gateway agent work and other producers whose run ids are not process session ids. Those live tasks can be marked lost after the short CLI grace, so preserve the existing runtime-aware checks and apply this observer only to exec-owned task records.
  - confidence: 0.91
- **[P2] Restrict session overrides to linked exec tasks:** `src/gateway/session-utils.ts:1217`
  - body: The new mapping reads every task owned by the session and lets the focused owner task replace the row status and timestamps. An unrelated owner task can make a completed session appear running, failed, or killed, so this override needs to select only the linked background exec task this PR creates.
  - confidence: 0.87
- **[P2] Refresh CLI task heartbeat on every output chunk:** `src/agents/bash-tools.exec-runtime.ts:678-682`
  - body: `recordTaskRunProgressByRunId` only runs when `summary` is present, but normal output chunks after the first one pass `undefined`. A long-running command that keeps printing output can stop advancing `lastEventAt`, making status and active-session filtering treat live work as stale.
  - confidence: 0.85
- **[P3] Return the background response before task creation:** `src/agents/bash-tools.exec.ts:1651`
  - body: `onYieldNow` awaits the dynamic import and task-registry write before resolving the background response. On a slow registry path this can extend the configured yield window, so start task creation asynchronously and keep the promise for terminal-state synchronization.
  - confidence: 0.74
