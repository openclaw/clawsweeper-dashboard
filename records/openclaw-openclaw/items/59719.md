## Summary

Keep open: current main still does not create linked task-registry records for background exec sessions, but this branch is not merge-ready because it can break existing CLI task liveness/session state and still lacks real behavior proof.

## What This Changes

The branch adds runtime:"cli" task creation/progress/finalization for background exec sessions, changes process remove/kill visibility, wires a CLI liveness hook into task maintenance, and lets gateway session rows derive status from task records.

## Best Possible Solution

Rebase onto current main and land a narrower exec-owned CLI task lifecycle that preserves existing run-context liveness for other CLI producers, only drives session rows from linked exec tasks, refreshes heartbeats on all output, and includes redacted real behavior proof.

## Reproduction Assessment

Yes, source-reproducible. Current main's background exec yield path only calls markBackgrounded and resolves the running result, and the exec runtime has no task-registry lifecycle writes for those background sessions.

## Solution Assessment

No, not yet. The direction is appropriate, but the branch needs to be rebased and scoped so exec-owned CLI tasks do not override or break the existing run-context liveness contract for other CLI task producers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Scope CLI liveness to exec-owned tasks:** `src/tasks/task-registry.maintenance.ts:75`
  - body: This routes every runtime:"cli" task with a runId through isCliRunActive, but gateway wires that hook to the bash process registry. Existing gateway-agent and media-generation CLI tasks use non-process run IDs, so live work can be marked lost after the short CLI grace.
  - confidence: 0.91
- **[P2] Record heartbeats for every CLI output chunk:** `src/agents/bash-tools.exec-runtime.ts:678-690`
  - body: reportCliTaskProgress only writes lastEventAt when a summary is present. Normal ongoing output after the first chunk passes no summary, so a noisy long-running command can keep running while its task/session activity timestamp goes stale.
  - confidence: 0.86
- **[P2] Filter session rows to the linked exec task:** `src/gateway/session-utils.ts:1216-1221`
  - body: buildGatewaySessionRow snapshots all tasks for the owner key and lets the active/focused owner task override status and timing. An unrelated ACP, subagent, or media task owned by the same session can make the row look running or failed.
  - confidence: 0.83
- **[P2] Resolve background yields before task writes:** `src/agents/bash-tools.exec.ts:1651`
  - body: onYieldNow awaits a dynamic import and task-registry write before resolving the running tool result. Slow storage can push the user-visible background response past the configured yield window even though the process is already backgrounded.
  - confidence: 0.76
