## Summary

Keep open. Current main still lacks exec-linked task creation for background exec yield, but this PR is not merge-ready because it routes all CLI task liveness through bash-process sessions, leaves heartbeat/session-row scoping bugs, and lacks after-fix real behavior proof.

## What This Changes

The PR links background exec/process runs to CLI task records and updates process, task-maintenance, and session-status surfaces around liveness, cancellation, and terminal state.

## Best Possible Solution

Land a current-main implementation that creates exec-owned CLI task records and updates heartbeats/terminal state while preserving runtime-specific liveness for gateway-agent, media, cron, ACP, and subagent tasks.

## Reproduction Assessment

Yes. Source inspection shows current main's background yield path only marks the process session backgrounded and resolves the running response, while the PR's blockers are also source-reproducible against current non-exec CLI task producers.

## Solution Assessment

No. The product direction is reasonable, but this branch is not the best fix because it broadens CLI liveness and session-row derivation beyond exec-owned tasks instead of preserving current runtime-aware task contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restrict CLI liveness to exec-backed task records:** `src/tasks/task-registry.maintenance.ts:75`
  - body: This sends every runtime:"cli" task through the bash-process liveness hook. Current main has gateway-agent and media CLI tasks whose run IDs are not process session IDs, so this can mark live non-exec work lost after the short CLI grace; scope the hook to exec-created task records or keep the runtime-specific checks.
  - confidence: 0.91
- **[P2] Refresh task activity on every output chunk:** `src/agents/bash-tools.exec-runtime.ts:678-696`
  - body: reportCliTaskProgress only writes lastEventAt when a summary is present, but ordinary ongoing chunks after the first output pass undefined. A live command that keeps producing output can therefore look stale in task/session status; always record the heartbeat timestamp and gate only the human summary text.
  - confidence: 0.86
- **[P2] Filter session-row overrides to linked exec tasks:** `src/gateway/session-utils.ts:1216-1221`
  - body: buildGatewaySessionRow snapshots all tasks owned by the session, so an unrelated active or recent owner task can override the session row's status and timing. Limit this path to the specific linked background exec CLI task shape this PR is adding.
  - confidence: 0.83
- **[P2] Resolve background yield before task writes:** `src/agents/bash-tools.exec.ts:1651`
  - body: onYieldNow awaits dynamic import and task creation before resolving the background tool result. If task storage is slow or contended, the command can exceed the configured yield window even though the process is already backgrounded; start task creation asynchronously and use the stored promise for exit races.
  - confidence: 0.76
