## Summary

Keep open: the branch addresses a real background exec liveness gap, but it is not merge-ready because it lacks real behavior proof, is conflicted against current main, and introduces task/session liveness regressions that can misstate live work.

## What This Changes

The PR links background exec/process sessions to runtime:"cli" task records, changes process removal/cancellation visibility, adds a CLI liveness observer for task maintenance, and derives gateway session rows from task state.

## Best Possible Solution

Rebase on current main, preserve runtime-aware CLI task maintenance, add scoped exec-owned CLI task linkage/status overrides, refresh heartbeats on all output, and require redacted real behavior proof.

## Reproduction Assessment

Yes, source-reproducible: current main's background exec yield path only calls markBackgrounded and resolveRunning without creating a task-registry record. I did not run a live background command in this read-only review.

## Solution Assessment

No, not as currently proposed: the direction is useful, but the branch must be rebased and scoped to exec-owned CLI tasks while preserving current main's runtime-aware maintenance behavior.

## Review Findings

Full review comments:

- **[P1] Scope CLI orphan checks to exec-owned tasks:** `src/tasks/task-registry.maintenance.ts:75-76`
  - body: This branch routes every `runtime:"cli"` task through a bash-process `isCliRunActive` observer. Current main also creates CLI tasks for gateway/chat/media runs, so live non-exec CLI work can be marked `lost` after the new 5s grace when bash-process-registry does not know its run id.
  - confidence: 0.91
- **[P2] Restrict session overrides to linked exec tasks:** `src/gateway/session-utils.ts:1217-1220`
  - body: The new row mapping takes the first active/focused task for the owner key, regardless of whether it is the linked background exec task. An unrelated owner task can override a completed session's status/timing and keep it visible as running or failed.
  - confidence: 0.88
- **[P2] Refresh CLI task heartbeats for output chunks:** `src/agents/bash-tools.exec-runtime.ts:682-695`
  - body: `reportCliTaskProgress` only writes `lastEventAt` when a summary is present; ordinary output after the first chunk passes `undefined`, so a long-running active command can look stale in task/session status.
  - confidence: 0.86
- **[P2] Preserve configured background yield latency:** `src/agents/bash-tools.exec.ts:1651-1653`
  - body: `onYieldNow` awaits a dynamic import and task write before resolving the background response, so a slow registry path can extend the configured yield window. Start task creation asynchronously and keep the promise for terminal synchronization.
  - confidence: 0.78
