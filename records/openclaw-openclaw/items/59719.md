## Summary

Keep open: the current-main liveness gap is still source-reproducible, but the PR head is not yet safe to merge because it broadens CLI liveness/session-row behavior beyond exec-owned tasks and still lacks external real behavior proof.

## What This Changes

The PR links background exec/process runs to runtime:"cli" task records, changes process remove/kill visibility, derives session rows from task state, and adds regression coverage.

## Best Possible Solution

Land a rebased implementation that creates exec-owned CLI task records while preserving runtime-specific liveness for gateway-agent, media, cron, ACP, and subagent tasks.

## Reproduction Assessment

Yes. Source inspection on current main shows the background exec yield path only marks the process session backgrounded and resolves the running response, with no linked task-registry record creation.

## Solution Assessment

No. The direction is appropriate, but this branch is not yet the best implementation because it broadens CLI liveness and session-row derivation beyond exec-owned tasks and delays the background response on task writes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Scope CLI orphan checks to exec tasks:** `src/tasks/task-registry.maintenance.ts:75`
  - body: This sends every runtime:"cli" task through isCliRunActive(runId), but the gateway observer is bash-process-registry liveness. Gateway-agent and media CLI task run IDs are not bash process sessions, so live work can be marked lost after the short CLI grace; scope this hook to exec-created task records or preserve the runtime-aware run-context checks.
  - confidence: 0.91
- **[P2] Refresh CLI task heartbeats on every output:** `src/agents/bash-tools.exec-runtime.ts:678-696`
  - body: lastEventAt is only written when summary is present, while ordinary continued output can call this path without a summary. A live command that keeps producing output can leave task/session activity stale, so record the heartbeat timestamp for every accepted output chunk and gate only the human summary text.
  - confidence: 0.86
- **[P2] Filter session rows to linked exec tasks:** `src/gateway/session-utils.ts:1216-1221`
  - body: The new session-row override snapshots all tasks owned by the session key, so an unrelated active or recent owner task can override the row status and timing. Limit this path to the linked background-exec CLI task shape introduced by this PR.
  - confidence: 0.83
- **[P2] Resolve background yield before task writes:** `src/agents/bash-tools.exec.ts:1651`
  - body: onYieldNow awaits a dynamic import and task-registry write before resolving the running tool result. If task storage is slow or contended, the command can exceed the configured yield window even though the process is already backgrounded; start task creation asynchronously and retain the promise for exit races.
  - confidence: 0.76
