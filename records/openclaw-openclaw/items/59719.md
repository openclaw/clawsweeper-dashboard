Full review comments:

- **[P1] Restrict CLI orphan checks to exec-backed runs:** `src/tasks/task-registry.maintenance.ts:75`
  - body: This hook is wired to the bash process registry, but it is used for every runtime:"cli" task. CLI tasks created by gateway agent runs or media/tool runtimes do not live in that registry, so after the 5s grace they can be marked lost while still active. Keep the newer runtime-context based liveness from current main or gate this path to exec-backed task IDs only.
  - confidence: 0.93
- **[P2] Scope session status overrides to linked CLI tasks:** `src/gateway/session-utils.ts:1217-1223`
  - body: The new row logic snapshots all tasks for the session owner and lets the focused task override status, timing, and updatedAt. Any unrelated active or recent owner task can make a completed session appear running/failed, so this should filter to the background exec CLI task relationship this PR is adding.
  - confidence: 0.88
- **[P2] Refresh task heartbeats for every output chunk:** `src/agents/bash-tools.exec-runtime.ts:678-691`
  - body: After the first output, normal chunks call reportCliTaskProgress with no summary, so lastEventAt is not written to the task registry. A long command that keeps producing output can still look stale in task/session views; record a heartbeat for output even when the visible progressSummary does not change.
  - confidence: 0.86
