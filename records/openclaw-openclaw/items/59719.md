## Summary

Keep open. The PR is a plausible fix direction for background exec liveness, but the current branch still has correctness blockers, is merge-conflicting with current main, and lacks contributor-provided after-fix real behavior proof.

## What This Changes

The PR links background exec/process sessions to runtime:"cli" task records, updates process removal/cancellation visibility, derives session rows from task state, and adds task-registry/process regression tests.

## Best Possible Solution

Land a rebased implementation that creates exec-owned CLI task records and status updates while preserving current runtime-specific liveness for gateway-agent, media, cron, ACP, and subagent tasks.

## Reproduction Assessment

Yes. Source inspection shows current main's background exec yield path only marks the process session backgrounded and resolves the running response, with no task-registry record creation in that path.

## Solution Assessment

No. The direction is reasonable, but this branch is not yet the best fix because it broadens CLI liveness/session-row derivation beyond exec-owned tasks, misses output heartbeats, and delays the background response on task writes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restrict CLI orphan checks to exec-backed run IDs:** `src/tasks/task-registry.maintenance.ts:75`
  - body: This treats every runtime:"cli" task as backed only by isCliRunActive(runId), but the gateway hook only knows bash process sessions. Gateway-agent and media CLI tasks use other run IDs, so active work can be reconciled to lost after the short CLI grace; scope this path to exec-created records or preserve the current runtime-aware checks.
  - confidence: 0.91
- **[P2] Refresh task activity on every output chunk:** `src/agents/bash-tools.exec-runtime.ts:678-696`
  - body: lastEventAt is only written when a summary is present, but ordinary ongoing output after the first chunk passes undefined. A live command that keeps producing output can leave task/session updatedAt stale, so record the heartbeat timestamp on every accepted output chunk and gate only the human summary text.
  - confidence: 0.86
- **[P2] Filter session-row overrides to linked exec tasks:** `src/gateway/session-utils.ts:1216-1221`
  - body: buildGatewaySessionRow snapshots all tasks owned by the session key, so an unrelated active or recent owner task can override the row's status and timing. Limit this override to the linked background exec CLI task shape this PR introduces.
  - confidence: 0.83
- **[P2] Resolve background yield before task writes:** `src/agents/bash-tools.exec.ts:1651`
  - body: onYieldNow awaits dynamic import and task creation before resolving the background tool result. If task storage is slow or contended, the command can exceed the configured yield window even though the process is already backgrounded; start task creation asynchronously and keep the promise for exit races.
  - confidence: 0.76
