## Summary

Keep open: current main still lacks the central exec-linked CLI task creation, and this branch remains a plausible fix direction, but it has blocking correctness findings, is conflicting with current main, and still lacks contributor-supplied real behavior proof.

## What This Changes

The branch wires background exec/process sessions into runtime:"cli" task records, process removal/cancellation state, task maintenance, gateway session row status, and related tests.

## Best Possible Solution

Land a rebased implementation that creates exec-owned CLI task records and status updates while preserving runtime-specific liveness for gateway-agent, media, cron, ACP, and subagent tasks.

## Reproduction Assessment

Yes. Source inspection shows current main's background exec yield path only marks the process as backgrounded and resolves the running response, while the PR's regressions are visible from the branch source.

## Solution Assessment

No. The direction is reasonable, but this branch is not yet the best fix because it broadens CLI liveness/session-row derivation beyond exec-owned tasks and delays the background response on task writes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restrict CLI liveness to exec-backed task records:** `src/tasks/task-registry.maintenance.ts:75`
  - body: This routes every runtime:"cli" task through isCliRunActive(), but the gateway hook only knows bash process session ids. Gateway-agent and media CLI tasks use other run ids, so active work can be reconciled to lost after the short CLI grace; scope this path to exec-created records or preserve the current runtime-aware checks.
  - confidence: 0.91
- **[P2] Refresh task activity on every output chunk:** `src/agents/bash-tools.exec-runtime.ts:678-696`
  - body: lastEventAt is only written when summary is present, but ordinary ongoing output after the first chunk passes undefined. A live command that keeps producing output can leave task/session updatedAt stale, so record the heartbeat timestamp on every accepted output chunk and gate only the human summary text.
  - confidence: 0.86
- **[P2] Filter session-row overrides to linked exec tasks:** `src/gateway/session-utils.ts:1216-1221`
  - body: buildGatewaySessionRow snapshots all tasks owned by the session key, so an unrelated active or recent owner task can override the row's status and timing. Limit this override to the linked background exec CLI task shape this PR introduces.
  - confidence: 0.83
- **[P2] Resolve background yield before task writes:** `src/agents/bash-tools.exec.ts:1651`
  - body: onYieldNow awaits dynamic import and task creation before resolving the background tool result. If task storage is slow or contended, the command can exceed the configured yield window even though the process is already backgrounded; start task creation asynchronously and keep the promise for exit races.
  - confidence: 0.76
