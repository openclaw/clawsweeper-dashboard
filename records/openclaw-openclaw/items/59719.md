## Summary

Keep open: the bug-fix direction is useful and current main still lacks linked task tracking for background exec, but this branch has correctness blockers around CLI liveness scoping, session-row overrides, and missing real behavior proof.

## What This Changes

The branch wires background exec/process sessions into runtime:"cli" task records, updates process remove/kill visibility, and derives session/status state from linked tasks with regression tests.

## Best Possible Solution

Rebase and land a scoped implementation that creates exec-owned CLI task records while preserving runtime-specific liveness and status derivation for other task producers.

## Reproduction Assessment

Yes. Source inspection on current main shows the background exec yield path only marks the process backgrounded and returns the running response, with no linked task-registry record creation.

## Solution Assessment

No. The direction is appropriate, but this branch is not the best implementation until CLI liveness and session-row overrides are scoped to exec-owned task records and the yield response is not delayed by task writes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Scope CLI orphan checks to exec-owned tasks:** `src/tasks/task-registry.maintenance.ts:75`
  - body: This routes every runtime:"cli" task to isCliRunActive(runId), but gateway wires that hook to bash-process-registry liveness. Gateway agent and media tasks create runtime:"cli" rows with non-process run IDs, so live work can be marked lost after the 5s CLI grace.
  - confidence: 0.91
- **[P2] Record CLI task heartbeats for every output chunk:** `src/agents/bash-tools.exec-runtime.ts:678-690`
  - body: reportCliTaskProgress updates lastEventAt only inside if (summary), while ordinary ongoing output calls the helper without a summary. A noisy long command can keep running with stale task/session activity and be filtered as inactive.
  - confidence: 0.86
- **[P2] Filter session rows to the linked exec task:** `src/gateway/session-utils.ts:1216-1221`
  - body: buildGatewaySessionRow snapshots all tasks for the owner key and lets the active/focused owner task override status and timing for the session. An unrelated ACP, subagent, or media task owned by the same session can make a done row look running or failed.
  - confidence: 0.83
- **[P2] Resolve background yields before task writes:** `src/agents/bash-tools.exec.ts:1651`
  - body: onYieldNow waits for a dynamic import and task-registry write before resolving the running tool result. Slow storage can push the user-visible background response past the configured yield window even though the process has already been backgrounded.
  - confidence: 0.76
