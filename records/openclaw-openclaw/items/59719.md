Full review comments:

- **[P1] Scope CLI liveness to exec-owned task records:** `src/tasks/task-registry.maintenance.ts:75`
  - body: This routes every runtime:"cli" task with a runId through the bash-process liveness observer, but current main also creates CLI tasks for gateway agent runs and media-generation tools. Those tasks can be reported missing and marked lost even while their real run context is active, so preserve the existing runtime-aware checks and apply bash-process liveness only to the exec task records created here.
  - confidence: 0.9
- **[P2] Filter session row overrides to background exec tasks:** `src/gateway/session-utils.ts:1217`
  - body: The new session-row mapping reads every task owned by the session and lets the focused owner task replace status, timestamps, and runtime fields. An unrelated owner task can make a completed session appear running, failed, or killed, so the override needs to select only the linked background exec task records this PR creates.
  - confidence: 0.86
- **[P2] Refresh CLI task heartbeat for every output chunk:** `src/agents/bash-tools.exec-runtime.ts:678-682`
  - body: recordTaskRunProgressByRunId is only called when summary is present, but normal output chunks after the first one pass undefined. A long-running command that keeps printing output can stop advancing lastEventAt, making status and active-session filtering treat live work as stale.
  - confidence: 0.84
- **[P3] Resolve the background response before creating the task:** `src/agents/bash-tools.exec.ts:1651`
  - body: onYieldNow awaits the dynamic import and task-registry write before resolving the background response. That can extend the configured yield window on a slow registry path; start task creation asynchronously and keep the promise for terminal-state synchronization.
  - confidence: 0.74
