The branch creates linked runtime:"cli" tasks for background exec/process sessions, changes process removal/cancel visibility, maps session rows from task state, and changes CLI task-maintenance reconciliation.

## Best Possible Solution

Rebase on current main, preserve runtime-aware task maintenance, add scoped exec-owned CLI task linkage with output heartbeat refreshes, and require redacted real behavior proof before merge.

## Reproduction Assessment

## Solution Assessment

No, not as currently proposed. The direction is useful, but the branch must be rebased and scoped to exec-owned CLI tasks while preserving current main's runtime-aware maintenance behavior.

  - confidence: 0.88

  - body: reportCliTaskProgress only records lastEventAt when a summary is present, but normal output after the first chunk passes no summary. Long-running commands that keep producing output can stop refreshing the task heartbeat and look stale in status surfaces.
  - confidence: 0.86
- **[P2] Preserve the configured background yield latency:** `src/agents/bash-tools.exec.ts:1651`
  - body: onYieldNow awaits dynamic import and task creation before returning the background response, so a slow registry path can extend the configured yield window. Start task creation asynchronously and keep the promise for terminal-state synchronization.
  - confidence: 0.78
