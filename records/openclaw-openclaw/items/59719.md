## Summary

Keep this PR open, but it is not ready to merge: the background exec liveness gap is real on current main, while this branch still regresses existing CLI task liveness/session-row contracts and lacks external after-fix real behavior proof.

## What This Changes

The PR links background exec/process sessions to `runtime:"cli"` task records and updates process removal, task maintenance, and session-row status behavior around those linked tasks.

## Best Possible Solution

Land a current-main implementation that creates exec-owned CLI task records and updates heartbeats/terminal state while preserving runtime-specific liveness for gateway-agent, media, cron, ACP, and subagent tasks.

## Reproduction Assessment

Yes. Source inspection shows current main's background exec yield path only marks the process session backgrounded and returns a running process handle, with no linked task record created for status/task surfaces.

## Solution Assessment

No. The product direction is reasonable, but this branch is not the best fix because it broadens CLI liveness and session-row derivation beyond exec-owned task records instead of preserving current runtime-aware task contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restrict CLI orphan checks to exec-backed run IDs:** `src/tasks/task-registry.maintenance.ts:75`
  - body: This sends every `runtime:"cli"` task through the bash-process liveness hook. Current main also has gateway-agent and media CLI task records whose run IDs are not process session IDs, so they can be marked `lost` after the short CLI grace while their real work is still active; scope this hook to exec-created task records or keep runtime-specific checks.
  - confidence: 0.91
- **[P2] Refresh task activity on every output chunk:** `src/agents/bash-tools.exec-runtime.ts:678-696`
  - body: `reportCliTaskProgress` only writes `lastEventAt` when a summary is present, but ordinary ongoing output after the first chunk passes no summary. A live command that keeps producing output can therefore look stale in task/session status; always update the heartbeat timestamp and gate only the human summary text.
  - confidence: 0.86
- **[P2] Filter session-row overrides to linked exec tasks:** `src/gateway/session-utils.ts:1216-1221`
  - body: `buildGatewaySessionRow` snapshots all tasks owned by the session, so an unrelated active or recent owner task can override the session row's status and timing. Limit this to the specific linked background exec CLI task shape added by this PR.
  - confidence: 0.84
- **[P2] Resolve background yield before task writes:** `src/agents/bash-tools.exec.ts:1651`
  - body: `onYieldNow` waits for dynamic import and task creation before returning the background tool result. If task storage is slow or contended, the command can exceed the configured yield window even though the process is already backgrounded; start the write asynchronously and keep the stored promise for exit races.
  - confidence: 0.76
