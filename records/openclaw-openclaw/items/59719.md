## Summary

Keep open. The underlying bug is valid and current main still lacks linked task creation for background exec, but this PR is not mergeable as-is because its CLI liveness and session-row mapping are too broad and stale against current task-maintenance behavior.

## What This Changes

The PR adds linked `runtime:"cli"` task records for background exec/process runs, changes process remove/kill visibility, derives session rows from task state, adds CLI liveness reconciliation, and expands regression tests.

## Best Possible Solution

Land a current-main implementation that creates and updates exec-owned CLI task records while preserving runtime-specific task maintenance and scoping session-row overrides to the linked exec task.

## Reproduction Assessment

Yes. There is a high-confidence code-level reproduction for the review blockers: existing gateway-agent and media `runtime:"cli"` tasks use non-process run IDs, and the PR routes them through bash-process liveness; the current-main background exec yield path also visibly lacks task creation.

## Solution Assessment

No. The product direction is reasonable, but this implementation is not the best fix because it broadens CLI task liveness and session status derivation beyond exec-owned tasks instead of using the current runtime-aware task contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restrict CLI liveness to exec-backed task records:** `src/tasks/task-registry.maintenance.ts:75`
  - body: This sends every `runtime:"cli"` task through the bash-process liveness hook. Current main has gateway-agent and media CLI tasks whose run IDs are not process session IDs, so this can mark live non-exec work `lost` after the short CLI grace. Scope the hook to exec-created task records or keep the existing runtime-specific checks for other CLI producers.
  - confidence: 0.91
- **[P2] Refresh task activity on every output chunk:** `src/agents/bash-tools.exec-runtime.ts:678-696`
  - body: `reportCliTaskProgress` only writes `lastEventAt` when `summary` is present, but normal ongoing chunks after the first output pass `undefined`. A live command that keeps producing output can therefore look stale in task/session status. Always record the heartbeat timestamp and gate only the human summary text.
  - confidence: 0.86
- **[P2] Filter session-row overrides to linked exec tasks:** `src/gateway/session-utils.ts:1216-1221`
  - body: `buildGatewaySessionRow` snapshots all tasks owned by the session, so an unrelated active or recent owner task can override the session row's status and timing. Limit this path to the specific linked background exec CLI task shape this PR is adding.
  - confidence: 0.83
- **[P2] Do not block background yield on task writes:** `src/agents/bash-tools.exec.ts:1651`
  - body: `onYieldNow` awaits dynamic import and task creation before resolving the background tool result. If task storage is slow or contended, the command can exceed the configured yield window even though the process is already backgrounded. Start task creation asynchronously and let the stored promise handle exit races.
  - confidence: 0.76
