## Summary

Keep open: current main and v2026.5.22 have adjacent Telegram throttling, retry, chunking, and durable batch primitives, but no opt-in outbound notification digest aggregation; the remaining work needs maintainer delivery-policy decisions.

## What This Changes

Add configurable Telegram outbound notification batching so many short automation events can be grouped into digest messages instead of sent immediately one-by-one.

## Best Possible Solution

Define a Telegram-owned digest aggregation contract that composes with the existing throttler, retry_after handling, durable delivery, media behavior, priority bypasses, shutdown flushing, docs, and focused tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source, docs, GitHub search, and v2026.5.22 inspection give high confidence that configurable Telegram outbound digest aggregation is not implemented today.

## Solution Assessment

Unclear. Native Telegram digest aggregation may be reasonable, but maintainers need to decide config scope, priority bypasses, media/durability behavior, shutdown flushing, and coordination with https://github.com/openclaw/openclaw/issues/69165.
