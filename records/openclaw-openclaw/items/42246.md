## Summary

Keep open: current main still has adjacent Telegram retry, throttling, durable delivery, and batch-plan metadata, but it does not implement configurable digest aggregation for independent outbound Telegram notifications.

## What This Changes

The issue requests opt-in Telegram notification batching that aggregates many outbound events into digest messages with configurable flush and grouping policy.

## Best Possible Solution

Define a Telegram plugin-owned aggregation design that composes with transport throttling, retry_after handling, durable delivery, media, priority rules, and shutdown semantics before adding schema, docs, and focused tests.
