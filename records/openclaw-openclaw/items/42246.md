## Summary

Keep open: current main and latest release v2026.6.10 still lack configurable outbound Telegram digest aggregation. Existing Telegram delivery supports direct sends, durable payload batches, chunking, retry/throttling, and media-group handling, but it does not collect unrelated outbound notifications into timed grouped digest messages.

## What This Changes

The issue requests an opt-in Telegram outbound notification batching layer that groups many short events into digest messages over a configurable window.

## Best Possible Solution

Define a maintainer-approved notification or digest policy seam above final Telegram transport, persist pending digest state in SQLite when durability is required, and send completed digest payloads through the existing durable Telegram delivery and throttling paths.

## Reproduction Assessment

Yes, at source level: current Telegram send paths deliver caller-supplied payloads directly through durable delivery and throttling, with no timed digest aggregator or batching config. I did not run a live Telegram scenario in this read-only review.

## Solution Assessment

No, not as a Telegram-local ring buffer alone. The safer shape is a maintainer-approved notification/digest policy seam above final Telegram transport so urgency, source grouping, media, thread targets, and durability semantics are decided before delivery.
