## Summary

Keep open: current main and the latest release have adjacent Telegram throttling, retry-after-aware retries, chunking, and durable rendered-batch delivery, but they still do not provide the requested configurable Telegram digest aggregation layer. The remaining work is a new feature/config surface that needs maintainer product design before implementation.

## What This Changes

Request an opt-in Telegram notification aggregator that batches high-volume outbound events into digest messages with configurable interval, batch size, grouping, and shutdown flush behavior.

## Best Possible Solution

Define a Telegram-owned digest aggregation contract that composes with current throttling, retry, durable delivery, media handling, priority rules, and shutdown behavior, then add opt-in config, docs, focused tests, and real Telegram proof.
