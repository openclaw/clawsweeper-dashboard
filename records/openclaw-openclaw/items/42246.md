## Summary

Keep open: current main has Telegram direct sending, retry, preview/progress controls, and write-ahead recovery, but no configurable digest/batching layer; related #69165 is complementary outbound queue/rate-limit work rather than a replacement.

## What This Changes

Request configurable Telegram outbound batching that aggregates many short notifications into digest messages using interval, max-message, grouping, and shutdown-flush settings.

## Best Possible Solution

Define a Telegram plugin-owned outbound aggregation design that composes with retry/rate limiting and preview/progress delivery, then add schema, docs, and focused tests for the chosen semantics.

## Reproduction Assessment

Not applicable. This is an enhancement request, and source inspection is enough to show current main lacks the requested batching configuration and digest behavior.

## Solution Assessment

Unclear. A native Telegram aggregation layer may be reasonable, but the exact implementation needs maintainer decisions about config shape, urgent-error bypass, media payloads, persistence, and coordination with #69165.
