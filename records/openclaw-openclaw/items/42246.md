## Summary

Keep open: current main still lacks configurable outbound Telegram notification digesting. The code has adjacent retry, throttling, durable delivery, chunking, and rendered-batch helpers, but nothing collects independent outbound notifications over a configurable window and formats them into digest messages.

## What This Changes

Request a Telegram notification batching layer that aggregates many outbound events into configurable digest messages instead of sending each event immediately.

## Best Possible Solution

Define a Telegram plugin-owned aggregation design that composes with transport throttling, retries, media, streaming previews, priority rules, and shutdown semantics before adding schema, docs, and focused tests.

## Reproduction Assessment

Not applicable. This is a feature/config request, and current-main source inspection shows the requested digest configuration and aggregation layer are absent.

## Solution Assessment

Unclear. A native Telegram aggregation layer may be reasonable, but the exact config shape and delivery guarantees should be designed alongside https://github.com/openclaw/openclaw/issues/69165 before implementation.
