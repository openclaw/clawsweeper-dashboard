## Summary

Keep open: current main has adjacent Telegram throttling, retry_after retry handling, and durable delivery recovery, but it still does not implement configurable digest aggregation for independent outbound Telegram notifications. ## What This Changes

Add an opt-in Telegram notification batching layer that groups bursts of outbound events into digest messages with configurable timing, size, grouping, and shutdown flush behavior.

## Best Possible Solution
