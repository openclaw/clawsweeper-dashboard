## Summary

Keep open: current main and v2026.5.22 have Telegram text chunking, API throttling, retry handling, and durable delivery replay, but not configurable outbound notification digest aggregation. The remaining work is a Telegram delivery-policy feature that needs maintainer decisions before implementation.

## What This Changes

Request configurable Telegram outbound notification batching so bursts of short OpenClaw events can be grouped into digest messages instead of sent immediately one by one.

## Best Possible Solution

Define a Telegram-plugin-owned digest aggregation contract that composes with current throttling, retry_after handling, durable final delivery, media behavior, priority bypasses, shutdown flushing, and docs/tests before implementation.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source, docs, and latest-release inspection give high confidence that configurable Telegram outbound digest aggregation is not implemented today.

## Solution Assessment

Unclear. Native Telegram digest aggregation may be reasonable, but maintainers need to decide config scope, priority bypasses, media and durability behavior, shutdown flushing, and coordination with https://github.com/openclaw/openclaw/issues/69165.
