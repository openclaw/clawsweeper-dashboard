## Summary

Keep open: current main still lacks configurable Telegram digest or aggregation behavior, and related #69165 tracks complementary outbound queue/rate-limit work rather than combining multiple notifications into fewer messages.

## What This Changes

Request configurable Telegram notification batching so bursts of outbound events can be grouped into digest messages.

## Best Possible Solution

Define a Telegram plugin-owned aggregation design that composes with retry/rate limiting, preview/progress delivery, media, and shutdown semantics, then add schema, docs, and focused tests for the chosen behavior.
