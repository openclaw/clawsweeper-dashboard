## Summary

Keep open: current main has adjacent Telegram throttling, retry_after-aware retry, and durable delivery recovery, but it still lacks configurable digest aggregation for independent outbound Telegram notifications. This remains a feature/product decision rather than a closeable cleanup item.

## What This Changes

Add opt-in Telegram outbound notification batching so multiple short automation events can be grouped into digest messages with configurable flush and grouping rules.

## Best Possible Solution

Define a Telegram-owned digest aggregation contract that composes with transport throttling, retry_after handling, durable delivery recovery, media handling, priority rules, and shutdown semantics before adding config schema, docs, and tests.

Not applicable as a strict bug reproduction because this is a feature request. Source inspection on current main gives high confidence that the requested digest configuration and aggregation layer are absent.

## Solution Assessment

Unclear. Native Telegram digest aggregation may be reasonable, but the exact delivery guarantees, config shape, and interaction with the open outbound queue request need maintainer product design first.
