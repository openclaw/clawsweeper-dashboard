## Summary

## What This Changes

The branch scopes Discord command-deploy cache keys by application id, serializes and merges shared cache persistence, adds multi-application regression tests, and changes CHANGELOG.md.

## Best Possible Solution

Land the focused app-id cache namespace and serialized persistence fix after rebasing/restoring the changelog, fixing the lock cleanup, and adding redacted live multi-bot Discord proof.

## Reproduction Assessment

Yes by source inspection. Current main uses a shared `command-deploy-cache.json` and unscoped keys, so a second Discord application with the same command hash can skip its own REST reconcile when `global:reconcile` already matches.

## Solution Assessment

Yes for the core direction, but not for this exact head as-is. Application-id cache keys plus serialized read-merge-write persistence is the narrow fix; the branch still needs proof, changelog cleanup, and a small lock cleanup correction.
