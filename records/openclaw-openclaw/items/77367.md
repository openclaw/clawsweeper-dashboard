## What This Changes

The branch scopes Discord command-deploy cache keys by application id, serializes cache persistence, adds Discord deployer regression tests, and adds a changelog entry.

## Best Possible Solution

Rebase and land the application-id cache namespace plus serialized read-merge-write persistence after redacted live multi-bot Discord proof is supplied, while preserving private file-store semantics and fixing lock cleanup.

## Reproduction Assessment

Yes. Source inspection on current main shows unscoped cache keys in a shared `command-deploy-cache.json` can make a later application skip REST reconciliation, and the linked bug reports provide concrete multi-bot Discord steps; I did not run a live Discord setup.

## Solution Assessment
