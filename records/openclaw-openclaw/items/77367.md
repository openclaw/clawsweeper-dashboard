## Summary

Keep open: current main still has the unscoped Discord deploy-cache behavior, and this PR is a focused fix with matching regression coverage and changelog text. I found no blocking review findings in the proposed patch.

## What This Changes

The PR scopes Discord command-deploy cache keys by Discord application id, adds multi-application regression tests, and adds a changelog entry for the slash-command fix.

## Best Possible Solution

Land the focused Discord cache-key scoping with the regression tests and changelog entry once normal maintainer review and required checks pass.

## Reproduction Assessment

Yes. Source inspection on current main shows Discord accounts share `discord/command-deploy-cache.json`, cache keys are only command-scope strings, and `putCommandSetIfChanged` skips REST reconciliation when the hash matches.

## Solution Assessment

Yes. Scoping the existing persisted cache keys by resolved `clientId` is the narrow maintainable fix because it preserves same-application cache skips while preventing cross-application skips.
