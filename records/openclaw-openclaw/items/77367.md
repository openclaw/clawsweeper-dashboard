## What This Changes

The PR scopes Discord command-deploy cache keys by application id, serializes shared cache persistence, adds multi-application cache regression tests, and changes the changelog.

## Best Possible Solution

Rebase cleanly, keep only the Discord cache/test changes plus a minimal changelog entry, fix the type and mutex cleanup issues, then land after redacted live multi-bot Discord proof.

Yes. Source inspection on current main plus live issue reports show two Discord accounts can share one `command-deploy-cache.json` entry under `global:reconcile`, causing later applications with the same command hash to skip REST reconciliation.

## Solution Assessment

Yes for the core direction, but no for the current PR head. Application-id cache keys plus serialized read-merge-write persistence is the narrow maintainable fix; the branch still needs rebase cleanup, CI fixes, mutex cleanup, and real Discord proof.
