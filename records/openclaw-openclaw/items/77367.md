## Summary

Keep open: the fix is directionally correct, but the latest head still has a blocking race where concurrent Discord account deployers can overwrite each other's app-scoped cache entries in the shared JSON file.

## What This Changes

The PR namespaces Discord slash-command deploy-cache keys by application id, adds multi-application regression tests, and records a Discord changelog fix.

## Best Possible Solution

Keep the application-id cache namespace, then make shared deploy-cache persistence merge current on-disk hashes or serialize writes so concurrent account deployments preserve every application's entry.

## Reproduction Assessment

Yes. Source inspection shows current main shares one cache file with bare command-scope keys and skips REST reconcile on a matching hash; the PR's remaining race is also source-reproducible from concurrent background deploys writing non-merged in-memory maps.

## Solution Assessment

No. Application-id key scoping is the right narrow fix for the original bug, but it needs a small persistence repair so concurrent writers do not drop each other's scoped entries.
