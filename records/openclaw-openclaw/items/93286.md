## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve supported HTML entities before sending:** `extensions/telegram/src/rich-message.ts:169`
  - body: This now sends explicit `textMode: "html"` payloads through `sanitizeTelegramRichHtml`, but that sanitizer only recognizes the five legacy named entities. Telegram rich HTML also supports entities such as `&nbsp;`, `&hellip;`, `&mdash;`, `&ndash;`, and curly quotes, so existing explicit rich HTML like `A&nbsp;B` becomes `A&amp;nbsp;B` and renders literally. Keep Telegram-supported entities intact, and make sure sanitization happens before any length-based split if escaping can expand the payload.
  - confidence: 0.88
- **[P2] Keep Markdown table cell formatting in rich HTML:** `extensions/telegram/src/format.ts:725-726`
  - body: Markdown tables are now converted to HTML through string-only table metadata, and each cell is escaped as plain text. That drops Telegram-supported inline formatting and alignment from cells such as `**42** <sup>ms</sup>` that raw rich Markdown could preserve. Carry table-cell inline spans/alignment through `MarkdownTableMeta`, or leave those tables on a path that preserves Telegram's rich Markdown semantics.
  - confidence: 0.82
