## Summary

Keep open: the PR targets a real Telegram `/reasoning on` message-loss bug, but it should not merge as-is because the new allowlist includes Slack even though Slack still drops reasoning payloads at transform time, which can mark final delivery failed and leave pending-final bookkeeping uncleared. ## What This Changes

The branch changes shared reasoning suppression so `isReasoning` block and final payloads pass through for Telegram, Discord, Slack, and MSTeams, and adds two Telegram dispatch regression tests.

## Best Possible Solution

Keep the fix narrow to Telegram's existing reasoning lane, preserve non-Telegram suppression until each channel has proven durable reasoning handling, and require live Telegram proof before merge.

## Reproduction Assessment

Yes, source-level: current main emits durable `isReasoning` block/final payloads for `/reasoning on` and then suppresses them before Telegram delivery. I did not run a live Telegram plus reasoning-model session in this read-only review.

## Solution Assessment
