## Summary

Keep open: current main still has the mid-turn reset path, and the PR’s predicate change is useful, but the submitted head still has blocking transcript-permission, portability, wrapper, and real-behavior-proof gaps.

## What This Changes

This PR narrows embedded session initialization resets to header-only transcripts, archives before that reset, adds focused unit coverage, and adds a shell verification wrapper.

## Best Possible Solution

Amend the PR to keep the hasAnyMessage guard, recreate reset transcripts with mode 0o600 and coverage, use portable test paths plus the repo test wrapper, and add redacted real restart proof.

## Reproduction Assessment

Yes. Source inspection shows a session header plus user message and no assistant reply flows through the embedded attempt path into the current !hasAssistant reset branch on main.

## Solution Assessment

No as submitted. The predicate change is the narrow maintainable fix for the data-loss bug, but the archive/reset implementation must preserve transcript privacy and the support files need cleanup first.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve private mode when recreating transcripts:** `src/agents/pi-embedded-runner/session-manager-init.ts:56`
  - body: After archiveFileOnDisk renames the existing transcript, this write creates a new path using Node's default file mode instead of the transcript 0o600 contract. Recreate it with mode 0o600 or chmod it before future transcript content can be appended.
  - confidence: 0.91

  - body: The new test hardcodes /tmp as the mkdtemp parent and builds the session path by string concatenation. Use os.tmpdir() and path.join() so this regression test remains portable outside Unix-like runners.
  - confidence: 0.84

  - body: The helper invokes raw pnpm vitest run, bypassing the repo's pnpm test routing and Vitest process handling. Use pnpm test src/agents/pi-embedded-runner/session-manager-init.test.ts or remove the wrapper.
  - confidence: 0.79

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete transcript-permission regression in the archive/reset path until the recreated file is forced back to 0o600.

Concerns:

- **[medium] Transcript reset recreates file without private mode:** `src/agents/pi-embedded-runner/session-manager-init.ts:56`
  - body: archiveFileOnDisk renames the existing transcript, then the PR recreates the original path with the string-encoding writeFile overload. Because normal transcript creation uses 0o600, later transcript content could be appended to a less-private file after a header-only reset.
  - confidence: 0.91
