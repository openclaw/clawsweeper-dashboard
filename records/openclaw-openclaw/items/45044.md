## Summary

Current main still has the reset branch that can clear a session file containing a user message but no assistant reply, so the PR is addressing a real data-loss path. The predicate change is the right narrow direction and the Greptile/Ryce discussion supports that, but the patch should not merge as submitted because the new archive/reset path can recreate private transcripts with default permissions and the added test/script coverage needs portability and repo-wrapper fixes.

## What This Changes

The PR narrows embedded session initialization resets to header-only transcripts, archives before clearing that reset case, adds session-manager-init unit coverage, and adds a shell verification wrapper.

## Best Possible Solution

Amend this PR so the `hasAnyMessage` guard remains, the header-only archive/reset path recreates or preserves transcripts at `0o600`, the tests use `os.tmpdir()` plus `path.join()`, the helper script uses the repo test wrapper or is removed, and a single-line changelog fix entry is added before targeted tests and the changed gate run.

## Reproduction Assessment

Yes. A transcript state of header plus user message and no assistant reply maps directly to current main's `hadSessionFile && header && !hasAssistant` reset branch, and the PR adds a unit case for that state.

## Solution Assessment

No as submitted. The predicate change is the narrow maintainable fix for the data-loss bug, but the archive-before-write addition needs explicit private mode handling and the support files need repo-portability cleanup before merge.

- **[P1] Preserve private mode when recreating transcripts:** `src/agents/pi-embedded-runner/session-manager-init.ts:56`
  - body: After `archiveFileOnDisk` renames the existing transcript, this write creates the replacement path with the process default mode instead of the transcript's existing private `0o600` mode. Recreate the file with `mode: 0o600` or restore the archived file's mode before future transcript content is appended.
  - confidence: 0.9

  - body: The new tests hardcode `/tmp` as the `mkdtemp` parent and build paths with string concatenation. Windows CI can fail before the session-preservation assertions run, so use `os.tmpdir()` with `path.join()`.
  - confidence: 0.86

  - body: The verification helper invokes raw `pnpm vitest run`, bypassing the repo's documented `pnpm test` wrapper and Vitest process handling. Use `pnpm test src/agents/pi-embedded-runner/session-manager-init.test.ts` or remove the wrapper.

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete transcript privacy regression in the archive/reset path.

Concerns:

- **[medium] Transcript reset recreates file without private mode:** `src/agents/pi-embedded-runner/session-manager-init.ts:56`
  - body: The PR archives the existing transcript with a rename and then recreates the original path using the string-encoding `writeFile` overload. Since normal transcript creation uses `0o600`, later transcript content could be appended to a less-private file after a header-only reset.
