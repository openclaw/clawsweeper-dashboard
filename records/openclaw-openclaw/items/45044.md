## Summary

Keep this PR open. Current main still has the mid-turn transcript-loss behavior, and the PR targets the right predicate, but the submitted diff introduces a transcript privacy regression in the archive/reset path and has smaller portability/wrapper issues that should be amended before merge.

## What This Changes

The PR narrows embedded session initialization resets to header-only transcripts, archives before clearing that reset case, adds unit coverage for mid-turn gateway restart cases, and adds a shell verification wrapper.

## Best Possible Solution

Amend this PR so the hasAnyMessage guard remains, the header-only archive/reset path preserves private 0o600 permissions when recreating the transcript, the new fixtures use os.tmpdir() with path.join(), and the helper script uses the repo test wrapper or is removed.

## Reproduction Assessment

Yes. A session file represented as header plus user message and no assistant maps directly to current main's hadSessionFile && header && !hasAssistant reset branch, and the PR adds a unit case for that exact state.

## Solution Assessment

No as submitted. The predicate change is the narrow maintainable fix for the data-loss bug, but the archive-before-write addition needs explicit private file mode handling before this is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve private mode when recreating transcripts:** `src/agents/pi-embedded-runner/session-manager-init.ts:56`
  - body: The new archive step renames the existing transcript before this write runs, so `fs.writeFile(..., "utf-8")` creates a replacement file with default process permissions instead of preserving the existing private transcript mode. Recreate the file with `mode: 0o600` or otherwise restore the archived file's mode before future transcript content is appended.
  - confidence: 0.9
- **[P2] Use portable temp paths in the tests:** `src/agents/pi-embedded-runner/session-manager-init.test.ts:10-11`
  - body: The new test suite hardcodes `/tmp` as the mkdtemp parent and concatenates paths manually. Windows-style environments may not have that parent, so the test can fail before it verifies the session-preservation behavior; use `os.tmpdir()` and `path.join()`.
  - confidence: 0.86
- **[P3] Use the repo test wrapper:** `scripts/verify-session-init-fix.sh:13`
  - body: The verification helper invokes raw `pnpm vitest run`, bypassing the repo's documented `pnpm test` wrapper and Vitest process handling. Use `pnpm test src/agents/pi-embedded-runner/session-manager-init.test.ts` or remove the wrapper.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The PR introduces a transcript privacy concern by archiving the original private file and recreating the transcript path without an explicit private mode.

Concerns:

- **[medium] Transcript reset recreates file without private mode:** `src/agents/pi-embedded-runner/session-manager-init.ts:56`
  - body: After archiveFileOnDisk renames the original transcript, the following write creates a new file using default permissions. Future transcript content can then be appended to a less-private file than the 0o600 creation paths intend.
