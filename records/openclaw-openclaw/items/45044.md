## What This Changes

The PR changes embedded session initialization to preserve message-bearing transcripts on restart, archive header-only resets, and adds regression tests plus a verification script.

## Best Possible Solution

Land a narrow version of the guard change that preserves `0o600`, keeps portable regression coverage through repo wrappers, and includes redacted real restart proof.

## Reproduction Assessment

Yes, from source inspection. A pre-existing transcript containing a session header plus a user message but no assistant message reaches `prepareSessionManagerForRun` with `hadSessionFile: true`, and current main’s `!hasAssistant` branch truncates it.

## Solution Assessment

No as submitted. The predicate change is the narrow maintainable fix, but the archive/recreate path must preserve transcript privacy and the support files need cleanup plus real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve private mode when recreating transcripts:** `src/agents/pi-embedded-runner/session-manager-init.ts:56`
  - body: After `archiveFileOnDisk` renames the existing transcript, this write recreates the original path without the transcript file’s `0o600` mode contract. Pass write options with `mode: 0o600` or chmod the recreated file so later session content is not appended to a less-private file.
  - confidence: 0.91
- **[P2] Use portable temp paths in the tests:** `src/agents/pi-embedded-runner/session-manager-init.test.ts:10-11`
  - body: The new tests hardcode `/tmp` and concatenate the session path, which makes the regression test depend on a POSIX temp root instead of Node’s runner temp directory. Use `os.tmpdir()` with `path.join()` so it remains reliable on Windows and custom runners.
  - confidence: 0.82
- **[P3] Use the repo test wrapper:** `scripts/verify-session-init-fix.sh:13`
  - body: This helper invokes raw `pnpm vitest run`, bypassing the repo’s test routing and process handling described in the scripts guide. Use `pnpm test src/agents/pi-embedded-runner/session-manager-init.test.ts` or remove the wrapper.
  - confidence: 0.8
