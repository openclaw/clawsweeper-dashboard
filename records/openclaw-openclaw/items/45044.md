## Summary

Keep open. Current main still has the destructive mid-turn reset path, and this PR targets the right predicate, but the submitted diff should not merge until it preserves transcript file mode and cleans up the portable test, wrapper, and changelog issues.

## What This Changes

The PR changes embedded session initialization to reset only header-only transcripts, archives before that reset, adds regression tests, and adds a shell verification wrapper.

## Best Possible Solution

Amend this PR in place: keep the `hasAnyMessage` guard, recreate reset transcripts at `0o600`, use portable temp paths and the repo test wrapper, add the active changelog entry, then run targeted and changed-gate validation.

## Reproduction Assessment

Yes from source inspection. On current main, a reopened transcript containing a session header plus a user message and no assistant enters the `hadSessionFile && header && !hasAssistant` branch and is cleared in `prepareSessionManagerForRun`.

## Solution Assessment

No as submitted. The `hasAnyMessage` predicate is the narrow maintainable fix for the data-loss bug, but the archive/reset path must preserve transcript privacy and the support files need cleanup before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve private mode when recreating transcripts:** `src/agents/pi-embedded-runner/session-manager-init.ts:56`
  - body: After `archiveFileOnDisk` renames the existing transcript, this write recreates the original path with Node's default file mode instead of the transcript `0o600` contract. Later transcript content can be appended to a group/world-readable file depending on umask, so recreate the file with `mode: 0o600` or preserve the restrictive mode.
  - confidence: 0.9
- **[P2] Use portable temp paths in the tests:** `src/agents/pi-embedded-runner/session-manager-init.test.ts:10-11`
  - body: The new tests hardcode `/tmp` as the `mkdtemp` parent and build paths with string concatenation. Windows runners do not guarantee `/tmp` exists, so use `os.tmpdir()` and `path.join()` before the regression assertions run.
  - confidence: 0.86
- **[P3] Use the repo test wrapper:** `scripts/verify-session-init-fix.sh:13`
  - body: The verification helper invokes raw `pnpm vitest run`, bypassing the repo's documented `pnpm test` wrapper and Vitest process handling. Use `pnpm test src/agents/pi-embedded-runner/session-manager-init.test.ts` or remove the wrapper.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `src/agents/pi-embedded-runner/session-manager-init.ts:47`
  - body: This changes user-facing data-loss behavior, but the PR does not add a `CHANGELOG.md` Fixes entry. Add a single-line entry for the active version and credit the non-bot contributor before merge.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces a transcript privacy regression in the header-only archive/reset path.

Concerns:

- **[medium] Transcript reset recreates file without private mode:** `src/agents/pi-embedded-runner/session-manager-init.ts:56`
  - body: The PR archives the existing transcript with `renameSync` and then recreates the original path using the string-encoding `writeFile` overload. Normal transcript creation uses `0o600`, so later transcript content could be appended to a less-private file after a header-only reset.
