## Summary

Keep open: current main still has the mid-turn transcript reset bug, and this PR has the right core predicate change, but it needs changes before merge because the added archive/reset path can recreate private transcripts with default file permissions.

## What This Changes

The PR changes embedded session initialization to reset only header-only transcripts, adds archive-before-reset behavior, adds session-manager-init unit tests, and adds a verification script.

## Best Possible Solution

Amend this PR so the `hasAnyMessage` guard remains, the header-only archive/reset path recreates transcripts with `0o600`, tests are portable, repo test wrappers are used, and the active changelog records the user-facing data-loss fix.

## Reproduction Assessment

Yes. Source inspection on current main shows a reopened transcript with a session header plus a user message and no assistant reply satisfies the `!hasAssistant` reset branch after the gateway attempt path calls `prepareSessionManagerForRun`.

## Solution Assessment

No as submitted. The predicate change is the narrow maintainable fix for the data-loss bug, but the archive/reset path must preserve transcript privacy and the support/test/changelog surface needs cleanup before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve private mode when recreating transcripts:** `src/agents/pi-embedded-runner/session-manager-init.ts:56`
  - body: After `archiveFileOnDisk` renames the existing transcript, this write recreates the original path without the transcript's `0o600` mode. Recreate the file with `{ encoding: "utf-8", mode: 0o600 }`, or otherwise preserve the restrictive mode, before later transcript content is appended.
  - confidence: 0.9

  - body: The new tests hardcode `/tmp` as the `mkdtemp` parent and build the file path with string concatenation. A targeted run on Windows can fail before exercising the regression assertions, so use `os.tmpdir()` with `path.join()`.
  - confidence: 0.86

  - body: This helper invokes raw `pnpm vitest run`, bypassing the repo's documented `pnpm test` wrapper and Vitest process handling. Use `pnpm test src/agents/pi-embedded-runner/session-manager-init.test.ts` or remove the wrapper.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete transcript-permission regression in the header-only archive/reset path.

Concerns:

- **[medium] Transcript reset recreates file without private mode:** `src/agents/pi-embedded-runner/session-manager-init.ts:56`
  - body: The PR archives the existing transcript with `renameSync` and then recreates the original path using the string-encoding `writeFile` overload. Since normal transcript creation uses `0o600`, later transcript content could be appended to a less-private file after a header-only reset.
