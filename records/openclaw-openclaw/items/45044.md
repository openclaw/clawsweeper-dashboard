## Summary

Keep open: current main still has the mid-turn transcript reset path, and the PR’s core predicate change is useful, but the submitted patch has concrete correctness/security blockers and only test/build proof so far.

## What This Changes

This PR narrows embedded session initialization resets to header-only transcripts, archives before clearing that header-only case, adds focused unit tests, and adds a shell verification wrapper.

## Best Possible Solution

Amend the PR to keep the hasAnyMessage guard, recreate reset transcripts with mode 0o600 and coverage, use portable test paths plus the repo test wrapper, and add redacted real restart proof.

## Reproduction Assessment

Yes. Source inspection shows an existing transcript with a session header plus a user message and no assistant reply flows through run/attempt into prepareSessionManagerForRun and matches the current !hasAssistant reset branch.

## Solution Assessment

No as submitted. The predicate change is the narrow maintainable fix for the data-loss bug, but the archive/reset implementation must preserve transcript privacy and the support files need cleanup first.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve private mode when recreating transcripts:** `src/agents/pi-embedded-runner/session-manager-init.ts:56`
  - body: After archiveFileOnDisk renames the existing transcript, this write recreates the original path with Node's default file mode instead of the transcript 0o600 contract. Recreate the file with mode: 0o600 or chmod it before later transcript content can be appended.
  - confidence: 0.91
- **[P2] Use portable temp paths in the tests:** `src/agents/pi-embedded-runner/session-manager-init.test.ts:10-11`
  - body: The new tests hardcode /tmp as the mkdtemp parent and build the session path by string concatenation. Use os.tmpdir() and path.join() so the regression test can run reliably on Windows shards too.
  - confidence: 0.84
- **[P3] Use the repo test wrapper:** `scripts/verify-session-init-fix.sh:13`
  - body: The helper invokes raw pnpm vitest run, bypassing the repo's pnpm test routing and process handling. Use pnpm test src/agents/pi-embedded-runner/session-manager-init.test.ts or remove the wrapper.
  - confidence: 0.79

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete transcript-permission regression in the archive/reset path until the recreated file is forced back to 0o600.

Concerns:

- **[medium] Transcript reset recreates file without private mode:** `src/agents/pi-embedded-runner/session-manager-init.ts:56`
  - body: The PR archives the existing transcript with renameSync and then recreates the original path using the string-encoding writeFile overload. Because normal transcript creation uses 0o600, later transcript content could be appended to a less-private file after a header-only reset.
