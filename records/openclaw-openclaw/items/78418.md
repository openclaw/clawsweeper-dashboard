## Summary

Keep open: current `main` still violates the documented `gateway.reload.mode="off"` contract because the runtime config reloader starts chokidar and reads watcher snapshots before the later action-only mode guard. The destructive auto-clobber part is already addressed, but the remaining watcher opt-out is a narrow source-backed bug, not a stale feature request.

## What This Changes

Requested a supported way for operators to disable runtime config-file watching and validation/clobber behavior while preserving startup validation.

## Best Possible Solution

Make the documented `gateway.reload.mode="off"` contract suppress external runtime watcher processing while preserving startup validation and internal config-write bookkeeping.

## Reproduction Assessment

Yes, at source level: current main still creates the chokidar watcher and reads snapshots from watcher events before the `gateway.reload.mode="off"` action guard. I did not run a live gateway because this review is read-only.

## Solution Assessment

Yes. The narrowest maintainable fix is to honor the existing documented `off` mode instead of adding a second public config field or environment variable.
