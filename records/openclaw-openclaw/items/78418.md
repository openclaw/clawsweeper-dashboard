## Summary

Keep open: current main no longer auto-restores invalid external edits, but it still starts the runtime config watcher and reads/validates changes even when `gateway.reload.mode="off"`. The public docs now say `off` disables file watching, so the remaining scope is a narrow behavior/docs-contract bug rather than stale cleanup.

## What This Changes

Request to let operators disable the gateway runtime config watcher so config edits remain operator-controlled until manual validation or restart.

## Best Possible Solution

Honor the existing documented `gateway.reload.mode="off"` contract by preventing runtime config watching/validation while preserving startup validation and doctor-owned recovery.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: start the gateway reloader with `gateway.reload.mode="off"`, and current main still creates the chokidar watcher and reads invalid external edits before any off-mode action check.

## Solution Assessment

Yes. The narrowest maintainable fix is to make the existing `off` mode match the docs, rather than adding a new `gateway.fileWatch` key or environment variable unless maintainers later want an additional alias.
