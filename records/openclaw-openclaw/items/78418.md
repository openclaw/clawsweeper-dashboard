## Summary

Keep open: current main has fixed the destructive auto-restore part, but it still starts the runtime config watcher and reads/validates external edits before the later `gateway.reload.mode="off"` action guard. That leaves a real docs/source contract mismatch, and I found no canonical open or merged PR that owns the remaining watcher opt-out.

## What This Changes

Allow operators to fully disable runtime config-file watching/validation while preserving startup config validation.

## Best Possible Solution

Honor the existing `gateway.reload.mode="off"` contract by suppressing external runtime file-watcher activity while preserving startup validation and OpenClaw-owned config-write side effects.

## Reproduction Assessment

Yes, at source level: current main starts `chokidar.watch`, and watcher events call `opts.readSnapshot()` before the `gateway.reload.mode="off"` guard can skip reload actions. I did not run a live gateway because this review is read-only.

## Solution Assessment

Yes. The narrowest maintainable fix is to make the existing documented `off` mode suppress external watcher processing instead of adding a new public config field or environment variable.
