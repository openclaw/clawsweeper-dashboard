## Summary

Keep open: current main appears to have removed the destructive auto-restore/clobber path for invalid external edits, but the requested operator opt-out from the watcher/validation loop is still not implemented and needs a config-contract decision.

## What This Changes

Request a supported config/env way to disable runtime config file watching and validation so intentional intermediate config edits are left under operator control.

## Best Possible Solution

Decide whether `gateway.reload.mode=off` should stop the chokidar watcher itself or only suppress apply/restart actions; if a full opt-out is desired, add documented schema/env semantics around the existing reloader startup path.

## Reproduction Assessment

Yes for the remaining behavior: source inspection shows the watcher still starts and invalid snapshots are handled before the `gateway.reload.mode=off` action check. No for the destructive auto-clobber portion on current main, because tests and docs show invalid external edits are skipped without recovery.

## Solution Assessment

Unclear; adding a new `gateway.fileWatch.enabled`/env knob is plausible, but maintainers should first decide whether the existing reload mode should own watcher lifecycle semantics or whether the no-clobber behavior already covers the operator need.
