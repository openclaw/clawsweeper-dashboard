## Summary

Keep open: current main has removed the destructive external-edit restore path, but `gateway.reload.mode="off"` still starts the chokidar watcher and reads/validates external config edits before the later off-mode action guard. That still conflicts with the Gateway docs, which say `off` disables file watching.

## What This Changes

The issue asks for a supported way to disable runtime config-file watching and validation while keeping gateway startup validation intact.

## Best Possible Solution

Honor the existing `gateway.reload.mode="off"` contract by suppressing external runtime watcher processing while preserving startup validation and OpenClaw-owned config-write side effects.

## Reproduction Assessment

Yes, at source level: current main registers a chokidar watcher and watcher events call `readSnapshot()` before the `gateway.reload.mode="off"` action guard. I did not run a live gateway in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is to make the existing documented `off` mode suppress external watcher processing instead of adding a new public config field or environment variable.
