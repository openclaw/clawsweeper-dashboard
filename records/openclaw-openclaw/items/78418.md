## Summary

Keep open: current main has removed the destructive auto-restore path, but the remaining watcher opt-out still is not honored. The documented `gateway.reload.mode=off` behavior says file watching is disabled, while the reloader still starts chokidar and reads/validates external edits before the later action guard.

## What This Changes

Request to let operators disable runtime config-file watching and validation so direct edits stay on disk until manual validation or restart.

## Best Possible Solution

Honor the existing `gateway.reload.mode=off` contract by preventing the runtime file watcher from starting or continuing when that mode is active, while keeping startup validation unchanged.

## Reproduction Assessment

Yes, at source level: with `gateway.reload.mode=off`, the reloader still creates chokidar and reads/validates external edits before the later action-only guard. I did not run a live gateway because this review was read-only.

## Solution Assessment

Yes. The narrowest maintainable fix is to make the existing documented `off` mode actually disable the watcher, instead of adding a second public config field or env var.
