## Summary

Keep open: current main has removed the destructive auto-restore behavior, but the documented `gateway.reload.mode="off"` contract is still not honored because the runtime reloader starts the chokidar watcher and reads/validates external edits before the later mode-off action guard. The reporter’s follow-up narrowed the request to that remaining watcher opt-out, and I found no merged or open PR that already owns this exact fix.

## What This Changes

Request a supported way for operators to fully disable runtime config-file watching/validation while leaving gateway startup validation unchanged.

## Best Possible Solution

Honor the existing `gateway.reload.mode="off"` contract by suppressing external runtime file-watcher activity while preserving startup validation and OpenClaw-owned config-write side effects.

## Reproduction Assessment

Yes, at source level: `startGatewayConfigReloader` always creates the chokidar watcher, and watcher events read/validate snapshots before the mode-off guard can skip reload actions. I did not execute a live gateway because this review was read-only.

## Solution Assessment

Yes. The narrowest maintainable fix is to make the documented `off` mode actually disable external watcher processing instead of adding a second public config field or environment variable.
