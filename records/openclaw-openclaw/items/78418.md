## Summary

Keep open: current main fixed the destructive auto-restore portion, but `gateway.reload.mode="off"` still does not match the documented contract because the runtime watcher starts and reads/validates external edits before the off-mode guard.

## What This Changes

The issue requests a supported way for operators to disable runtime config-file watching and validation while preserving startup validation.

## Best Possible Solution

Honor `gateway.reload.mode="off"` by suppressing external watcher processing while preserving startup validation and intentional OpenClaw-owned config-write follow-ups.

## Reproduction Assessment

Yes, at source level: current main starts `chokidar.watch`, and watcher events call `readSnapshot()` before the later `gateway.reload.mode="off"` guard can skip reload actions. I did not run a live gateway in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is to make the existing documented `off` mode suppress external runtime watcher processing instead of adding a new public config field or environment variable.
