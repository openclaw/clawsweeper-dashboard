## Summary

Keep open: current main and the latest release no longer auto-restore invalid external edits, but `gateway.reload.mode="off"` still starts the runtime watcher and reads/validates external file changes before the off-mode guard, contradicting the documented contract.

## What This Changes

The issue requests a supported runtime opt-out for the gateway config watcher so operator-owned edits are not processed while the gateway is running.

## Best Possible Solution

Honor the existing `gateway.reload.mode="off"` contract by suppressing external watcher-triggered reads, validation, and reload planning while preserving startup validation and intentional OpenClaw-owned config-write follow-ups.

## Reproduction Assessment

Yes, at source level: with reload mode off, current main still registers chokidar handlers and watcher events call `readSnapshot()` before the off-mode guard can skip reload actions. I did not run a live gateway in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is to make the existing documented `off` mode suppress external runtime watcher processing instead of adding a new public config field or environment variable.
