## Summary

Keep open: current main and the latest release still start the config-file watcher and read/validate external edits when reload mode is off, even though the docs say `off` disables file watching. The destructive auto-restore part is already fixed, so the remaining work is a narrower source/docs contract bug.

## What This Changes

The issue asks for a supported runtime opt-out from config-file watcher processing and validation while preserving startup validation.

## Best Possible Solution

Honor the existing `gateway.reload.mode="off"` contract by suppressing external runtime watcher processing while keeping startup validation and intentional OpenClaw-owned config-write follow-ups intact.

## Reproduction Assessment

Yes, at source level: current main starts `chokidar.watch`, and watcher events call `readSnapshot()` before the later `gateway.reload.mode="off"` guard can skip reload actions. I did not run a live gateway in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is to make the existing documented `off` mode suppress external runtime watcher processing instead of adding a new public config field.
