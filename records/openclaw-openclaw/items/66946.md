## Summary

Keep open: current main still contains the duplicated Control UI bootstrap path behavior, and the two related fix PRs were closed without merging.

## What This Changes

The issue reports that loading Control UI at `/__openclaw__/` builds `/__openclaw__/__openclaw/control-ui-config.json`, returns 404, and prevents chat from working.

## Best Possible Solution

Land a focused Control UI/gateway fix that makes the bootstrap config endpoint consistently base-path-relative across the shared contract, frontend loader, gateway handler, Vite dev stub, tests, docs, and changelog.

## Reproduction Assessment

Yes. The static reproduction is high-confidence: current main can infer `/__openclaw__` from the reported entry path, prepend it to a constant that already contains `/__openclaw`, and then miss the gateway bootstrap endpoint with a `.json` 404.

## Solution Assessment

Yes. The narrow maintainable direction is to make only the bootstrap config contract base-path-relative while preserving existing Control UI auth, static asset routing, plugin webhook passthrough, and gateway probe behavior.
