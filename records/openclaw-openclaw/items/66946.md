The issue reports that opening Control UI under `/__openclaw__/` can request `/__openclaw__/__openclaw/control-ui-config.json`, returning 404 and breaking chat startup.

## Best Possible Solution

Land a focused Control UI/gateway fix that makes the bootstrap config endpoint base-path-relative, keeps the frontend loader, gateway handler, Vite dev stub, tests, docs, and changelog aligned, and preserves existing auth/static-routing behavior.

## Solution Assessment
