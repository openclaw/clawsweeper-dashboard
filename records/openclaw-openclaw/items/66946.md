## Summary

Keep open: current main and v2026.5.18 still contain the Control UI bootstrap URL composition that can produce the reported duplicated `__openclaw__` config URL. The directly linked fix PRs were closed unmerged, and the merged related auth PR did not change the endpoint path contract.

## What This Changes

The issue reports that opening Control UI under `/__openclaw__/` makes the frontend request `/__openclaw__/__openclaw/control-ui-config.json`, causing a 404 and breaking chat.

## Best Possible Solution

Land a focused Control UI/gateway fix that makes the bootstrap config endpoint base-path-safe, keeps the frontend loader, gateway route, Vite dev stub, docs, and tests in agreement, and preserves bootstrap auth behavior.

## Reproduction Assessment

Yes, at source level. Current main can infer `/__openclaw__` from the reported entry path, then fetch `${basePath}${CONTROL_UI_BOOTSTRAP_CONFIG_PATH}`, while the gateway does not serve the resulting duplicated JSON URL.

## Solution Assessment

Yes. The narrow maintainable fix is to align the shared bootstrap endpoint contract with base-path handling and add regression coverage for the reported `/__openclaw__/` entry path while preserving auth.
