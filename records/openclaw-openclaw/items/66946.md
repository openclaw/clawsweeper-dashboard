## Summary

Keep open: current main still contains the duplicated Control UI bootstrap path behavior, and the related fix PRs were closed without merging.

## What This Changes

Fix Control UI bootstrap config fetching so a `/__openclaw__` base path does not produce `/__openclaw__/__openclaw/control-ui-config.json` and 404.

## Best Possible Solution

Make the shared bootstrap config endpoint base-path-relative and keep the frontend loader, gateway handler, Vite dev stub, tests, docs, and changelog aligned.

## Reproduction Assessment

Yes, from source inspection. On current main, `/__openclaw__/` can infer `basePath` as `/__openclaw__`, the frontend then requests `/__openclaw__/__openclaw/control-ui-config.json`, and the gateway falls through to a `.json` 404 when that route does not match the intended bootstrap endpoint.

## Solution Assessment

Yes. The narrow maintainable fix is to make only the bootstrap config path contract base-path-relative while preserving existing Control UI auth, static asset routing, plugin webhook passthrough, and gateway probe behavior.
