## Summary

Keep open: current main and the latest release still contain the Control UI bootstrap URL composition that can produce the reported duplicated `__openclaw__` config URL. The directly linked fix PRs were closed unmerged, and the merged related PR only hardened auth for the same endpoint.

## What This Changes

The issue reports that opening Control UI at `/__openclaw__/` makes the frontend request `/__openclaw__/__openclaw/control-ui-config.json`, returning 404 and blocking chat.

## Best Possible Solution

Land a focused Control UI/gateway fix that makes the bootstrap config endpoint base-path-safe for the reported entry path while keeping the documented endpoint and auth behavior intact.

## Reproduction Assessment

Yes. Source inspection on current main shows `/__openclaw__/` can infer `/__openclaw__`, the frontend then fetches `/__openclaw__/__openclaw/control-ui-config.json`, and the gateway does not serve that URL as the bootstrap endpoint.

## Solution Assessment

Yes. The narrow maintainable fix is to align the Control UI bootstrap URL and gateway route contract with base-path handling, with regression coverage for the reported `/__openclaw__/` entry path and configured base paths.
