## Summary

Keep open: current main still builds the duplicated Control UI bootstrap config URL for the reported `/__openclaw__/` entry path, and the latest release still contains the same contract. The two directly linked fix PRs were closed unmerged, and the only open search hit is a broader bootstrap-auth PR, not a canonical fix for this path bug.

## What This Changes

The issue reports that opening Control UI at `/__openclaw__/` fetches a duplicated bootstrap config URL and breaks chat.

## Best Possible Solution

Keep the bootstrap config endpoint single-sourced and base-path-aware so `/__openclaw__/` loads the authenticated runtime config in production and dev without changing Control UI auth policy.

## Reproduction Assessment

Yes, by source inspection rather than live execution: `/__openclaw__/` can infer `/__openclaw__`, the frontend then fetches `/__openclaw__/__openclaw/control-ui-config.json`, and the gateway treats that `.json` URL as a missing static asset. I did not run a live gateway because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to align the frontend, gateway route, dev stub, tests, docs, and changelog around a base-path-aware bootstrap config URL while preserving the auth gate from https://github.com/openclaw/openclaw/pull/70247.
