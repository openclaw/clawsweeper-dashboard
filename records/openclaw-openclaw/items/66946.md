## Summary

Keep open: current main and the latest release still encode the duplicated Control UI bootstrap config path, so the reported 404 remains source-reproducible; the two related fix PRs were closed without merging.

## What This Changes

The issue asks OpenClaw to fix a Control UI bootstrap config URL that duplicates the `__openclaw__` path prefix and returns 404, preventing chat from loading.

## Best Possible Solution

Land a focused Control UI/gateway fix that makes the bootstrap config endpoint base-path-relative, updates the dev stub, tests, docs, and changelog, and preserves existing auth and static-routing behavior.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction path. Opening the UI under `/__openclaw__/` can infer `/__openclaw__`, the frontend fetches `/__openclaw__/__openclaw/control-ui-config.json`, and the gateway falls through to a `.json` 404 when that URL is not the bootstrap endpoint.

## Solution Assessment

Yes. The narrow maintainable solution is to make the bootstrap config path contract base-path-relative and keep the frontend loader, gateway handler, Vite dev stub, tests, and docs aligned without changing unrelated Control UI routing or auth policy.
