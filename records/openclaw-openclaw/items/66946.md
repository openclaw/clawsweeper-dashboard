## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/66946. Current main has `CONTROL_UI_BOOTSTRAP_CONFIG_PATH = "/__openclaw/control-ui-config.json"`; both the Control UI frontend and gateway prepend `basePath` to that constant. When the UI is opened under `/__openclaw__/`, the frontend can infer basePath `/__openclaw__` and request `/__openclaw__/__openclaw/control-ui-config.json`, which misses the gateway bootstrap endpoint and returns 404 as a missing `.json` asset. Make the bootstrap config endpoint contract base-path-relative, likely `/control-ui-config.json`, or otherwise normalize the contract so frontend and gateway agree. Update the Vite dev stub, add/adjust regression coverage for basePath `/__openclaw__` and the no-basePath path, update `docs/web/control-ui.md`, and add a user-facing changelog fix entry. Preserve the auth gating added by https://github.com/openclaw/openclaw/pull/70247, the SPA/static asset fallback behavior, and unrelated Control UI namespace routes.
