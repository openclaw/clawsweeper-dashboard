## Summary

Keep open. Current main still has a source-confirmed Control UI bootstrap path mismatch for the reported `/__openclaw__/` entrypoint: the frontend can infer `/__openclaw__` as its base path and request `/__openclaw__/__openclaw/control-ui-config.json`, while the root-mounted gateway only treats `/__openclaw/control-ui-config.json` as the bootstrap JSON endpoint and returns 404 for JSON-looking static misses. The related fix PRs were closed unmerged, and issue-number search did not find an open replacement PR that closes this issue.

## What This Changes

The issue reports that opening Control UI under `/__openclaw__/` makes the frontend request a duplicated bootstrap config path, causing a 404 and leaving chat unusable.

## Best Possible Solution

Land a focused Control UI/gateway path-contract fix that makes the bootstrap config endpoint agree across inferred/configured base paths, gateway routing, frontend loading, Vite dev stubs, tests, and public docs, with explicit coverage for the reported `/__openclaw__/` entrypoint and configured `gateway.controlUi.basePath: "/__openclaw__"`.

## Reproduction Assessment

Yes. Current source gives a high-confidence static reproduction path: opening `/__openclaw__/` can infer `/__openclaw__` as `basePath`, the frontend requests `/__openclaw__/__openclaw/control-ui-config.json`, and the default gateway does not treat that JSON URL as the bootstrap endpoint.

## Solution Assessment

Yes, the best direction is a narrow gateway/UI path-contract fix. The patch should prove the reported route and configured-basePath cases rather than only changing one string or broadening unrelated Control UI routing/auth behavior.
