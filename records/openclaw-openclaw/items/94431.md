## What This Changes

The branch adds a lazy CLI reparse helper that hoists parent or ancestor command options before lazy subcommands, plus focused CLI and browser regression tests.

## Best Possible Solution

Land the parent-option hoist only after it preserves missing-value behavior and pins the malformed cases with helper and browser lazy tests.

## Reproduction Assessment

Yes. Source inspection shows current main and `v2026.6.8` still put `--browser-profile` on the browser parent while lazy reparse has no parent-option hoist, and the PR body supplies before/after terminal proof for the reported order.

## Solution Assessment

No, not yet. The reparse layer is a plausible owner boundary, but the helper must not hoist value options when no value follows because Commander will then consume the subcommand or next flag as the parent option value.
