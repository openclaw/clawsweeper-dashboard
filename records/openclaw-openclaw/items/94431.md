## Best Possible Solution

Keep the reparse-level parent-option hoist, but require an actual non-empty value before hoisting value options so Commander keeps its normal missing-value/error behavior.

## Reproduction Assessment

## Solution Assessment

No, not yet. The reparse layer is the right place for the fix, but the helper must not hoist value options when no real value follows because Commander will consume the subcommand or next flag as that option's value.
