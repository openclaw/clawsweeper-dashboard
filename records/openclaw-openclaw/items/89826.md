Full review comments:

- **[P1] Reuse the existing context usage surface:** `ui/src/ui/views/chat.ts:2266`
  - body: Current main already renders `renderContextNotice` in the composer from the same session token fields, including a low-usage meter, high-usage warning, compact action, and stale-token suppression. - **[P2] Back the new overview cards with real agent data:** `ui/src/ui/views/agents-panels-overview.ts:262`
  - body: These new cards render static values for every non-loading agent, so agents without a memory corpus or `SOUL.md` still appear to have them. `renderAgentOverview` already receives config and `agentFilesList`; use those real inputs, show an unavailable state, or omit these cards until the data exists.
  - confidence: 0.86

  - body: The sidebar recent sessions render before the chat view in `app-render.ts`, but `sessionHasKnownToolErrors` is populated later as a side effect of `renderChat`. That means the new dot can miss the first render of a failed tool call and cannot know about sessions that have not been rendered locally; derive this from session data or a controller-owned cache before rendering the sidebar.
  - confidence: 0.82
