## Summary

Keep open: current main still has the reported doctor/security mismatch, and the same-author linked fix PR is open rather than merged. ## What This Changes

The issue reports that `openclaw doctor --lint --json` classifies a complete `gateway.auth.mode="trusted-proxy"` Gateway setup as unauthenticated instead of as trusted-proxy-authenticated with exposure warnings.

## Best Possible Solution

Review and land or intentionally close the focused linked PR; keep this issue open until that implementation path is resolved.

## Reproduction Assessment

Yes. Source inspection shows a LAN-bound Gateway with `gateway.auth.mode="trusted-proxy"`, a `trustedProxy.userHeader`, and `gateway.trustedProxies` still has no token/password `hasSharedSecret`, so current doctor/security takes the unauthenticated critical path.

## Solution Assessment

Yes. The narrowest maintainable fix is in `src/commands/doctor-security.ts`: align the exposure classification with the existing runtime/docs trusted-proxy auth boundary while keeping unsafe trusted-proxy cases critical.
