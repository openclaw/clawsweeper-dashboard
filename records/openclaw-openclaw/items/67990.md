## Summary

Keep open. Current main implements several partial backup hardening slices, but the broader design is still not complete: there is no user-configurable exclusion policy, no restore/import command surface, and no service-aware gateway quiesce lifecycle; related open items and an open exclusion PR cover only parts of the proposal.

## What This Changes

The issue proposes a broader backup architecture with configurable exclusions, symlink-aware traversal, service-aware lifecycle, tar.gz archives, and config-only/full modes.

## Best Possible Solution

Keep this as the broad backup-design tracker until maintainers intentionally split or approve the remaining exclusion-policy, restore/import, archive-safety, and optional gateway-quiesce contracts.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a single bug reproduction because this is a design proposal; the missing-feature check is high-confidence from current source, docs, tests, and related live GitHub state.

## Solution Assessment

Is this the best way to solve the issue? No, not as one automated patch or a direct port of the standalone tool; the maintainable path is to keep current TypeScript backup primitives and split public exclusion, restore/import, archive-safety, and quiesce semantics into reviewed slices.
