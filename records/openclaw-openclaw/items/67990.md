## Summary

Keep open. Current main implements several backup hardening slices, including symlink/source de-duplication, volatile-file skipping/retry, and installed-plugin `node_modules` pruning, but the broader proposal is still not complete: configurable user exclusions, native restore/import, and service-aware gateway quiesce semantics remain unresolved and product/security-sensitive.

## What This Changes

The issue proposes a broader backup design with configurable exclusions, service-aware archiving, symlink-aware traversal, tar.gz archives, and config-only/full modes.

## Best Possible Solution

Keep this as the broad backup-design tracker while maintainers split and review the remaining exclusion-policy, restore/import, archive-safety, and gateway-quiesce contracts in focused slices.

## Reproduction Assessment

Not applicable as a single bug reproduction; this is a design proposal. The missing-feature check is high-confidence from current source, docs, tests, and related live GitHub state.

## Solution Assessment

No, not as one automated patch or a direct port of the standalone tool. The maintainable path is to build on the current TypeScript backup primitives and split exclusion policy, restore/import, archive safety, and quiesce semantics into reviewed slices.
