## Summary

Keep open. Current main only has the shipped partial that skips installed plugin `extensions/*/node_modules`; the broader request still lacks configurable exclusion rules, general recursive filtering, service-aware quiescing, live-file retry/skip handling on main, and restore/import/full-mode surfaces. Related #72251 is an open partial live-backup race PR, not a superseding implementation.

## What This Changes

The issue proposes improving `openclaw backup create` with configurable exclusions, symlink-aware traversal, service-aware archiving, tar.gz restore guidance, and separate config-only/full backup modes.

## Best Possible Solution

Keep this as the broader backup design tracker and split implementation into reviewed slices: documented exclusion policy/config, general archive filtering, live-file resilience, explicit quiesce semantics if desired, and restore/import disaster-recovery behavior coordinated with #13616, #54242, #67417, and #72251.

## Reproduction Assessment

Not applicable as a single reproduction because this is a design proposal. Current-main source/docs inspection gives high-confidence proof that the central requested surfaces are still absent, while related race reports provide their own concrete repro paths.

## Solution Assessment

No, not as one automated implementation. The direction is valid, but the maintainable path is to preserve the shipped backup primitives and make separate maintainer-approved changes for filtering, live-file behavior, service lifecycle, and restore/import scope.
