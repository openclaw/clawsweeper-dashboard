## Summary

Keep open. Current main now covers several slices of the proposal, including canonical source de-duplication for symlinked workspaces, installed-plugin node_modules pruning, and known volatile-file skips/retries, but the central design is still not complete: there is no user-configurable exclusion policy, restore/import command, or service-aware gateway quiesce lifecycle; the exclusion slice is still tracked separately by https://github.com/openclaw/openclaw/issues/40786 and https://github.com/openclaw/openclaw/pull/44288.

## What This Changes

The issue proposes a broader backup architecture with configurable exclusions, service-aware archiving, symlink-aware traversal, tar.gz disaster-recovery archives, and config/full backup modes.

## Best Possible Solution

Keep this as a backup-design tracker only until maintainers either approve a coordinated design or split the remaining work into narrower canonical issues for exclusion policy, restore/import, deleted-file handling, hardlink normalization, and optional gateway quiescence.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a single bug reproduction because this is a design proposal; the missing-feature check is high-confidence from current main source, docs, tests, and live related issue/PR state.

## Solution Assessment

Is this the best way to solve the issue? No, not as one automated patch or a direct port of the standalone implementation; the maintainable path is to preserve the current TypeScript backup primitives while maintainers define the public exclusion, restore, and quiesce contracts in smaller slices.
