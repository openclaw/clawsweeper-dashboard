## Summary

Keep open. Current main has backup create/verify and a narrow installed-plugin `node_modules` prune, but the broader design request remains unimplemented: configurable exclusions, service-aware gateway quiescence, general live-file resilience, restore/import behavior, and a public backup policy contract are still missing.

## What This Changes

The issue proposes a broader backup architecture with configurable exclusion rules, service-aware archiving, symlink-safe traversal, tar.gz archives, and config-only/full backup modes.

## Best Possible Solution

Keep this as the broad backup design tracker and split maintainer-approved follow-ups for exclusion config/API, archive filtering and symlink/volatile-file semantics, optional service quiescence, and restore/import disaster-recovery behavior coordinated with #13616, #54242, #67417, and #72251.

## Reproduction Assessment

Not applicable: this is a design proposal rather than one current-main bug reproduction. Source, docs, tests, changelog, and related-item context give high-confidence proof that the requested configurable exclusions, service lifecycle, live-file resilience, and restore/import surfaces remain absent.

## Solution Assessment

No, not as one automated patch or a direct port of the standalone implementation. The maintainable route is to keep the existing TypeScript backup primitives and have maintainers define the public exclusion config, quiesce semantics, and restore/import contract before implementation.
