## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use the resolved OpenRouter usage type:** `src/commands/models/list.status-command.ts:346`
  - body: `ReturnType<typeof loadOpenRouterMeteredUsage>` is the Promise type from the async loader, but the variable is assigned the awaited result and later read as an object. This will fail type checking; type it as `Awaited<ReturnType<...>>` or the concrete resolved usage type.
  - confidence: 0.96
- **[P2] Gate OpenRouter billing fetches to JSON output:** `src/commands/models/list.status-command.ts:376-380`
  - body: The default rich `models status` path enters `!opts.plain` and calls `loadOpenRouterMeteredUsage`, but `openrouterUsage` is only rendered inside the later `opts.json` block. Users with an OpenRouter key would pay a remote billing request and timeout risk for output that never displays that data.
  - confidence: 0.94
- **[P2] Report OpenRouter HTTP failures as errors:** `src/infra/provider-usage.openrouter.ts:193-194`
  - body: Non-2xx `/auth/key` responses are collapsed to `null`, and the caller only treats thrown exceptions as failures. A revoked or unauthorized key can therefore produce `status: "ok"` with missing key data instead of an actionable provider error in the JSON status output.
  - confidence: 0.9
- **[P2] Mark partial key fetch failures in provider status:** `src/infra/provider-usage.openrouter.ts:272-280`
  - body: When at least one key succeeds and another key request throws, `keyErrors.length > 0` still leaves `status` as `ok` unless the credits request also failed. That silently hides partial OpenRouter failures from consumers that rely on `auth.providerUsage.openrouter.status`.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `src/commands/models/list.status-command.ts:423-424`
  - body: This PR adds a user-facing `feat` to `openclaw models status --json`, but the diff does not update `CHANGELOG.md`. OpenClaw policy requires changelog coverage for user-facing feature changes.
  - confidence: 0.84
