## Summary

Keep open. Current main has generic live-check, docs/source lookup, config-schema lookup, and status-version guidance, but it still does not implement version-aware agent self-knowledge, agent-readable change notes, a general `openclaw explain <topic>` surface, or a known-behavior registry.

## What This Changes

Request that OpenClaw add version-aware self-knowledge so agents can distinguish intentional upgrade behavior changes from bugs.

## Best Possible Solution

Define an upgrade-diagnostics contract that combines the running version, current docs/source/config schema, and a maintained release/change-note source before adding any general explain command or registry.

## Reproduction Assessment

Not applicable. This is a broad feature/product request rather than a single broken behavior, and the current-main gap is source-verifiable from the prompt, docs CLI, status, and docs inventory surfaces.

## Solution Assessment

Unclear. The direction is valid, but the best implementation needs maintainer judgment between prompt injection, docs/changelog retrieval, focused explain commands, and a registry so OpenClaw does not add stale or cache-hostile context.
