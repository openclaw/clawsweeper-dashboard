## Summary

Keep open. Current main has generic live-check, docs/source lookup, config-schema lookup, and status-version guidance, but it still lacks version-aware agent self-knowledge, agent-readable release/change notes, a general `openclaw explain <topic>` surface, or a known-behavior registry.

## What This Changes

The issue requests version-aware agent self-knowledge so OpenClaw can distinguish intentional upgrade behavior changes from bugs.

## Best Possible Solution

Define an upgrade-diagnostics contract that combines the running version, authoritative current docs/source/config schema, and a maintained release/change-note source before adding prompt injection, a general explain command, or a behavior registry.

## Reproduction Assessment

Not applicable. This is a broad feature/product request rather than a single broken behavior, and the current-main gap is source-verifiable from the prompt, docs CLI, status, and docs inventory surfaces.

## Solution Assessment

Unclear. The direction is valid, but the best implementation needs maintainer judgment between prompt injection, docs/changelog retrieval, focused explain commands, and a registry so OpenClaw does not add stale or cache-hostile context.
