## Summary

Keep open. Current main has generic live-check guidance, docs/source lookup, config-schema lookup, and status-version output, but it still does not provide version-aware agent self-knowledge, agent-readable change notes, a general `openclaw explain <topic>` command, or a known-behavior registry.

## What This Changes

Request version-aware agent self-knowledge, agent-readable change notes, an `openclaw explain <topic>` surface, and a known-behavior registry so upgrade diagnostics can distinguish intentional changes from bugs.

## Best Possible Solution

Define a core upgrade-diagnostics contract that combines the running version, current docs/source/config schema, and a maintained release/change-note source before adding a general explain command or registry.

## Reproduction Assessment

Not applicable. This is a broad feature/product request rather than a single broken behavior, and the current-main gap is source-verifiable from the prompt, docs CLI, status, and config-schema surfaces.

## Solution Assessment

Unclear. The direction is valid, but maintainers need to choose between prompt injection, docs/changelog retrieval, focused explain commands, and a registry so the result stays accurate and cache-conscious.
