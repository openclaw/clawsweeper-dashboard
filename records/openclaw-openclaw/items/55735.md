## Summary

Keep open. Current main has generic live-check guidance, docs/source fallback, config-schema lookup, and status-version surfaces, but it still does not provide the requested version-aware self-knowledge, agent-readable change notes, general `openclaw explain <topic>` command, or known-behavior registry.

## What This Changes

The issue requests version-aware agent self-knowledge and authoritative upgrade-diagnostics surfaces so intentional behavior changes are not misdiagnosed after upgrades.

## Best Possible Solution

Define an upgrade-diagnostics contract that combines the running version, authoritative current docs/source/config schema, and a maintained release/change-note source before adding prompt injection, an explain command, or a behavior registry.

## Reproduction Assessment

Not applicable. This is a broad feature/product request rather than a single failing current-main behavior, and the remaining gap is source-verifiable from the prompt, docs, CLI, and gateway-tool surfaces.

## Solution Assessment

Unclear. The direction is valid, but maintainers need to choose between prompt injection, docs/changelog retrieval, focused explain commands, and a registry so OpenClaw does not add stale or cache-hostile context.
