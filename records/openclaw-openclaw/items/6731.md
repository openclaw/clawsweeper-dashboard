## Summary

Keep open. Current main and v2026.5.22 still document and implement trusted-operator host exec as `security="full"` and `ask="off"` by default, while this issue asks for a hard/default dangerous-operation gate or explicit safe/unsafe security posture that needs maintainer product and security review.

## What This Changes

The issue requests a safe/unsafe ClawdBot posture with hard platform enforcement for risky host operations rather than relying only on prompts or manual policy.

## Best Possible Solution

Have maintainers choose the core exec/tool-policy stance, then either implement a focused native/default destructive-operation gate with Windows PowerShell classifications, compatibility behavior, docs, and regression tests, or explicitly document the current opt-in model as final.
