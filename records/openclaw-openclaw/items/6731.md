## Summary

## Best Possible Solution

Have maintainers choose the core exec/tool-policy stance, then either implement a focused native/default destructive-operation gate with Windows PowerShell classifications, compatibility behavior, docs, and regression tests, or explicitly document the current opt-in model as final and route related PRs accordingly.
