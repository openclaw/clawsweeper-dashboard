## Summary

Keep open. Current main and the latest shipped tag still preserve trusted-operator host exec as `security="full"` and `ask="off"` by default, while the request is for a hard/default dangerous-operation gate or explicit safe/unsafe security stance that needs product and security review.

## Best Possible Solution

Have maintainers choose the core exec/tool-policy stance, then either implement a focused native/default destructive-operation gate with Windows PowerShell classifications, compatibility behavior, docs, and regression tests, or explicitly document the current opt-in model as final.
