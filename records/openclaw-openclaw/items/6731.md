## Summary

Keep open. Current main and the latest release still preserve trusted-operator host exec as `security="full"` and `ask="off"` by default, while this issue asks for a hard/default dangerous-operation gate or explicit safe/unsafe security stance that needs product and security review.

## What This Changes

The issue asks for a safe/unsafe ClawdBot mode or native hard gate for destructive host-exec operations such as Windows PowerShell removals, credential revocation, and config overwrites.

## Best Possible Solution

Have maintainers choose the core exec/tool-policy stance, then either implement a focused native/default destructive-operation gate with Windows PowerShell classifications, compatibility behavior, docs, and regression tests, or explicitly document the current opt-in model as final.
