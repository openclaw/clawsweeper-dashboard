## Summary

Keep open. Current main and v2026.5.22 still preserve trusted-operator host exec as `security="full"` and `ask="off"` by default, while this issue asks for a hard/default destructive-operation gate or explicit safe/unsafe posture that needs product and security review.

## What This Changes

Requests a safe/unsafe ClawdBot posture with sandboxed safe mode and hard gates for destructive or prompt-injected host operations.

## Best Possible Solution

Have maintainers choose the core exec/tool-policy stance, then either implement a focused native/default destructive-operation gate with Windows PowerShell classifications, compatibility behavior, docs, and regression tests, or explicitly document the current opt-in model as final.
