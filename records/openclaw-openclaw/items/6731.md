## Summary

Keep open: current main and the latest release still document and implement explicit full/YOLO host exec without a separate destructive-operation classifier, while related narrower issues do not replace the broader safe-default product decision.

## What This Changes

The issue requests a core safe/unsafe ClawdBot mode or hard enforcement layer that prevents destructive host operations and prompt-injection-driven privileged actions by default.

## Best Possible Solution

Keep this as the core umbrella until maintainers define the safe-default and explicit high-power contracts, then split implementation into threat-modeled exec/tool policy, sandbox, approval UX, docs, and cross-platform regression coverage.

## Reproduction Assessment

Yes. Current source and shipped docs establish that explicit full/YOLO host execution runs with approvals off and without a separate destructive-command classifier; a dangerous live command is unnecessary and inappropriate for reproduction.

## Solution Assessment

No, not as a Rust rewrite or a simple dangerousOps list. The maintainable solution is a product-defined safe-default and upgrade contract built on existing exec modes, sandboxing, and approval runtime, with reliable enforcement and cross-platform tests.
