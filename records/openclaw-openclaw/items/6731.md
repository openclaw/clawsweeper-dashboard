## Summary

Keep open. Current main still documents and implements permissive trusted-operator host exec by default, with hard controls available only through opt-in exec policy, sandboxing, and approvals; that does not satisfy the requested native hard/default dangerous-operation gate.

## What This Changes

The issue requests a safe/unsafe ClawdBot mode or equivalent hard platform gate for sandboxing, destructive host operations, and prompt-injection risk.

## Best Possible Solution

Keep this open until maintainers either implement a core hard/default dangerous-operation gate with docs and tests, or explicitly decide and document that the current opt-in exec approvals, sandboxing, and tool-policy model is the final stance.

No automatic or cleanup-close path is the best solution here. The maintainable path is a maintainer/security decision about default safe mode, dangerous-command classification, approval UX, sandbox interaction, and regression coverage.
