## Best Possible Solution

Preserve existing lightweight cron workflows by making the valid Haiku target explicit as `anthropic/claude-haiku-4-5` and deciding whether applicable upgrades should backfill that allowlist entry or surface a clear doctor/docs remediation instead of suggesting any `4-7` Haiku slug.
