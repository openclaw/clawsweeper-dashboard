## What This Changes

The PR exports the daemon service-env preservation helper, adds a three-key `OPENCLAW_*` preservation allowlist for root/container opt-ins, and adds helper tests for preserve/drop cases.

## Best Possible Solution

Confirm which root/container env keys are intended durable service configuration; if they are, land the narrow allowlist with install-plan coverage and real Linux service-regeneration proof.

## Solution Assessment

Unclear: the allowlist is a small code change, but persisting root/container control env vars is a policy decision. The safer path is maintainer-confirmed key selection plus proof through the service regeneration path rather than only direct helper output.
