## Summary

Keep open: the reported `/v1/responses` compatibility bug is valid on current main, but this PR still widens the schema to accept unsupported structured text controls and lacks patched-gateway real behavior proof.

## What This Changes

The branch adds a `text` object to `CreateResponseBodySchema`, schema tests, and a changelog entry for OpenAI SDK 6.x/n8n `/v1/responses` compatibility.

## Best Possible Solution

Accept the SDK-default `text.format: { type: "text" }` shape, reject unsupported structured text controls until runtime enforces them, and add focused HTTP/schema regression proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: POST `/v1/responses` with a top-level `text` field reaches the strict current-main schema, lacks an allowed key, and returns HTTP 400 before agent dispatch.

## Solution Assessment

No. The narrow schema-parity direction is right, but accepting `json_schema`, `json_object`, and `verbosity` before the runtime honors those contracts is not the best implementation.

## Review Findings

Overall correctness: patch is incorrect
