## Summary

## What This Changes

The PR widens the OpenAI-compatible Responses request schema to accept a top-level `text` field, adds schema tests, and edits `CHANGELOG.md`.

## Best Possible Solution

Land a narrow Gateway schema/test fix that accepts the SDK-default `text` request while either rejecting or implementing non-default text controls, removes the changelog edit, and includes redacted patched-Gateway proof.

## Reproduction Assessment

Yes, at source level: current main's strict `/v1/responses` schema has no top-level `text`, and the handler returns HTTP 400 on schema failure before dispatch. I did not run the Gateway because this cleanup review is read-only.

## Solution Assessment

No, not as submitted. Accepting the SDK-default `text.format.type === "text"` object is the right narrow bug fix, but accepting structured-output and verbosity controls without forwarding or rejecting them is not the safest API contract.
