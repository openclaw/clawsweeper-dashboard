## Best Possible Solution

Land a narrow Gateway schema/test fix that accepts the SDK-default `text` request, rejects or implements non-default text controls, removes the changelog edit, and includes redacted patched-Gateway proof.

## Solution Assessment

No, not as submitted. Accepting the SDK-default `text` object is the right narrow bug fix, but accepting structured-output and verbosity controls without forwarding or rejecting them is not the safest API contract.
