## Best Possible Solution

Land a narrow Gateway schema/test fix that accepts the SDK-default `text` shape while rejecting or implementing non-default text controls, removes the changelog edit, and includes redacted patched-Gateway proof.

## Reproduction Assessment

Yes, at source level. Current main has a strict `/v1/responses` body schema without `text`, and the handler returns HTTP 400 before dispatch when schema parsing fails.

## Solution Assessment

No, not as submitted. Accepting the SDK-default `text` object is the right narrow fix, but accepting structured-output and verbosity controls without forwarding or rejecting them is not the safest API contract.
