## Summary

Keep this PR open: it targets a real `/v1/responses` compatibility bug, but the current patch still accepts structured-output formats that the gateway never enforces, and the external-PR real behavior proof is still schema-only after the fix.

## What This Changes

The branch adds a top-level `text` object to `CreateResponseBodySchema`, adds schema unit tests for `text.format` and `verbosity`, and records a changelog fix.

## Best Possible Solution

Accept the SDK-default `text.format: { type: "text" }` request shape now, reject or fully implement `json_schema` and `json_object` before accepting them, and add redacted patched-gateway proof.

## Reproduction Assessment

Yes. Current main is source-reproducible with high confidence: the strict request schema lacks `text`, and the HTTP handler turns schema parse failure into a 400 before dispatch.

## Solution Assessment

No. The narrow maintainable fix is to accept the SDK-default `text` shape needed for OpenAI SDK 6.x parity while continuing to reject structured-output formats until the runtime can enforce those contracts.
