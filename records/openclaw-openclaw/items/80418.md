## Summary

## Best Possible Solution

Land a narrow Gateway schema and HTTP-test fix that accepts the SDK-default `text` shape while rejecting or implementing non-default text controls, includes redacted patched-endpoint proof, and removes the changelog edit.

## Reproduction Assessment

Yes, source-level: current main's strict `CreateResponseBodySchema` has no top-level `text` field, and `handleOpenResponsesHttpRequest` returns HTTP 400 on schema parse failure before dispatch. I did not run the Gateway because this cleanup review is read-only.

## Solution Assessment

No, not as submitted. Accepting the SDK-default `text` object is the right narrow bug fix, but accepting structured-output and verbosity controls without forwarding or rejecting them is not the safest API contract.
