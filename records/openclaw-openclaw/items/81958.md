## Summary

Keep open: this fresh external Control UI shortcut PR is aimed at a maintainer-scoped enhancement, but source review found guard holes in the proposed dispatcher and the supplied live proof does not cover the Esc behavior changed by the patch.

## What This Changes

The PR adds document-level Control UI shortcuts for `/`, `N`, and `Esc`, plus focused browser test coverage for `/` and `N`.

## Best Possible Solution

Land a single guarded dashboard shortcut path that shares one editable-target predicate, only triggers the new-message shortcut when Chat can actually handle it, and includes focused tests plus live proof for `/`, `N`, and `Esc`.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path for the review findings: the `N` branch is keyed only on `chatNewMessagesBelow`, `scheduleChatScroll` falls back to the document scroller without `.chat-thread`, and a separate mobile Escape listener remains outside the new editable guard. I did not execute the branch because this was a read-only review.

## Solution Assessment

No. A document-level dispatcher is the right direction, but the current patch needs the editable guard applied consistently, the `N` shortcut tied to the visible/reachable chat affordance, and a broader editable predicate before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Cover plaintext contenteditable targets:** `ui/src/ui/app.ts:141`
  - body: The editable guard only matches empty or `true` contenteditable attributes. `contenteditable="plaintext-only"` is still a valid editable region, so `/`, `N`, and `Esc` can fire while a user is typing there; use `isContentEditable` or exclude only `contenteditable="false"`.
  - confidence: 0.9
- **[P2] Keep N tied to the chat affordance:** `ui/src/ui/app.ts:653`
  - body: This branch fires on any tab as long as `chatNewMessagesBelow` is true. Outside Chat there is no New messages button, and `scrollToBottom()` falls back to the document scroller when `.chat-thread` is absent, so `N` can scroll the current dashboard page instead of reaching chat messages; gate it on Chat or switch to Chat before scrolling.
  - confidence: 0.86
- **[P2] Apply the editable guard to mobile Escape handling:** `ui/src/ui/app.ts:670-672`
  - body: The new dispatcher returns for editable targets, but the existing document-level mobile-controls Escape listener still runs afterward. With mobile controls open, Escape from an editable target can still prevent default and close controls; fold that path into the guarded dispatcher or apply the same guard there.
  - confidence: 0.78
