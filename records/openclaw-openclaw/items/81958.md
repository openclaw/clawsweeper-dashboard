## Summary

Keep open. The latest PR fixes two earlier shortcut bugs for non-chat `N` and mobile Escape, but the editable-target guard still misses a valid contenteditable mode, and the supplied real behavior proof still does not demonstrate the Esc path after the fixup.

## What This Changes

The PR adds document-level Control UI shortcuts for `/`, `N`, and `Escape` plus focused browser coverage in `ui/src/ui/app.ts` and `ui/src/ui/navigation.browser.test.ts`.

## Best Possible Solution

Land one guarded Control UI shortcut dispatcher only after the editable predicate covers all editable targets and the contributor supplies redacted live proof for `/`, `N`, and `Esc`.

## Reproduction Assessment

Yes for the review finding: source inspection shows the new guard omits `contenteditable="plaintext-only"`, which the platform treats as editable. I did not execute the PR branch because this was a read-only review.

## Solution Assessment

No. The dispatcher direction is right, but the guard should use `isContentEditable` or otherwise cover every contenteditable value except false before this is the narrow maintainable fix.
