## Summary

Keep open. The restart-recovery direction is plausible, but the patch is not merge-ready because the new tool-free announce handoff flag only reaches the embedded runner while CLI-backed sessions still branch into `runCliAgent` without `disableTools`; the PR also lacks real Telegram/gateway behavior proof for the topic restart path.

## What This Changes

The PR adds announce run-id/session markers, restart-recovery reconciliation for interrupted announce runs, a shared subagent announce handoff classifier, embedded-runner tool gating, and regression tests.

## Best Possible Solution

Apply the announce handoff classifier consistently across every runtime branch, including CLI fail-closed/native-tool paths, preserve normal human-turn recovery, and require redacted real Telegram restart proof plus focused regression tests.
