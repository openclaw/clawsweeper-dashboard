## Summary

Keep open: current main and v2026.5.12 still compute Discord startup delay from the enabled account list order, which is alphabetical, instead of promoting the configured default account first.

## What This Changes

The issue requests Discord startup ordering that starts the configured default/primary account before secondary accounts while preserving the existing stagger.

## Best Possible Solution

Order Discord startup-delay candidates with the resolved configured default account first, then preserve the current enabled/configured filtering and 10s stagger for the remaining accounts.

## Reproduction Assessment

Yes, by source inspection: configure enabled Discord accounts such as `billy`, `farber`, and `main` with `channels.discord.defaultAccount: "main"`; the sorted account list gives `main` a nonzero startup index. I did not run a live Discord gateway restart in this read-only review.

## Solution Assessment

Yes: fixing Discord's startup-delay ordering to use the existing default-account resolver is the narrowest maintainable solution. The repair should avoid hard-coded aliases and should not change the stagger duration or generic account-list ordering for other callers.
