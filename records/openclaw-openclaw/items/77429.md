## Summary

Keep open: current main still computes Discord account startup delay from the sorted enabled account list, so a configured default account can still be delayed behind alphabetically earlier specialist accounts.

## What This Changes

The issue requests Discord multi-account startup ordering to launch the configured default/main account before specialist accounts while preserving the existing stagger.

## Best Possible Solution

Order Discord startup-delay candidates with the resolved configured default account first, keep the current enabled/configured filtering and stable ordering for the rest, and add focused regression coverage.

## Reproduction Assessment

Yes, by source inspection: configure enabled Discord accounts such as `billy`, `farber`, `lumbergh`, `main`, `maverick`, and `scout` with `channels.discord.defaultAccount: "main"`; the sorted account list gives `main` index 3, so `resolveDiscordStartupDelayMs` returns a 30s delay. I did not run live Discord startup in this read-only review.

## Solution Assessment

Yes: fixing the ordering inside Discord's startup-delay calculation is the narrowest maintainable solution. The repair should use the existing `defaultAccount` resolver and avoid hard-coded account aliases.
