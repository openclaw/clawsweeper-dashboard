## Summary

Keep open: current main still computes Discord startup delay from sorted enabled account IDs, so a configured default account such as `main` can start after alphabetically earlier specialist accounts.

## What This Changes

The issue requests that Discord multi-account startup place the configured default/main account first while keeping the existing stagger for secondary accounts.

## Best Possible Solution

Order Discord startup-delay candidates with the resolved `channels.discord.defaultAccount` first, keep the remaining enabled accounts deterministic, and add regression coverage in the Discord channel tests.

## Reproduction Assessment

Yes, by source inspection: with accounts `billy`, `farber`, `lumbergh`, `main`, `maverick`, and `scout`, the sorted account list gives `main` index 3 and a 30s delay. I did not run a live Discord multi-account restart in this read-only review.

## Solution Assessment

Yes: fixing Discord's startup-delay ordering to use the existing default-account resolver is the narrowest maintainable solution. The repair should avoid hard-coded aliases such as `main` or `billy`.
