## Summary

Keep open: current main still computes Discord startup delay from the sorted enabled account list and does not promote `channels.discord.defaultAccount`, so the reported delayed `main` account is source-reproducible.

## What This Changes

Discord multi-account startup should start the configured default or primary account before specialist accounts while preserving the existing stagger.

## Best Possible Solution

Use the existing Discord default-account resolver to put the resolved default account first in the startup-delay candidate list, keep deterministic ordering for the remaining eligible accounts, and add regression coverage.

## Reproduction Assessment

Yes, by source inspection: with accounts `billy`, `farber`, `lumbergh`, `main`, `maverick`, and `scout`, the sorted helper makes `main` index 3 and `resolveDiscordStartupDelayMs` returns a 30s delay. I did not run a live Discord restart in this read-only review.

## Solution Assessment

Yes: fixing Discord's startup-delay ordering to derive priority from `defaultAccount` is the narrowest maintainable repair. A hard-coded alias list would be less maintainable and would not respect the existing config contract.
