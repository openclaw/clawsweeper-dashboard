## Summary

Keep open: current main still calculates Discord startup delay from the sorted enabled account list, so a configured primary/default account such as `main` can be delayed behind alphabetically earlier specialist accounts.

## What This Changes

Request that Discord multi-account startup prioritize the configured default/main account before staggering secondary accounts.

## Best Possible Solution

Use the existing Discord default-account resolver to order the startup-delay candidate list with the resolved default account first, then keep deterministic ordering for the remaining enabled/configured accounts and add regression coverage.

## Reproduction Assessment

Yes, by source inspection: with accounts `billy`, `farber`, `lumbergh`, `main`, `maverick`, and `scout`, the sorted account list gives `main` index 3 and a 30s delay. I did not run live Discord startup.

## Solution Assessment

Yes: fixing the ordering inside Discord's startup-delay calculation is the narrowest maintainable solution. It should use the existing `defaultAccount` contract rather than hard-coded account aliases.
