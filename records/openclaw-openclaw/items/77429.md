## Summary

Keep open: current main still derives Discord startup delay from the sorted account list, so a configured `channels.discord.defaultAccount` can start after alphabetically earlier specialist accounts.

## What This Changes

The issue requests that Discord multi-account startup start the configured default or primary Discord account before staggered secondary accounts.

## Best Possible Solution

Rank enabled/configured Discord startup accounts with `resolveDefaultDiscordAccountId(cfg)` first, preserve the existing 10s stagger for the remaining accounts, and avoid hard-coded account aliases.

## Reproduction Assessment

Yes, from source inspection: with accounts such as `billy`, `farber`, `lumbergh`, and `main` plus `defaultAccount: "main"`, the delay list is alphabetically sorted, so `main` is not index 0. I did not run a live Discord restart in this read-only review.

## Solution Assessment

Yes, using the already configured default account as the first startup rank is the narrowest maintainable fix. The safer implementation avoids hard-coded names and keeps the current stagger interval unchanged.
