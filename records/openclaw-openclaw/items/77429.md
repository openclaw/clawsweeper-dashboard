## Summary

Keep open: current main still computes Discord startup delay from the sorted enabled account list, so a configured default account such as `main` can still start after alphabetically earlier specialist accounts.

## What This Changes

Request that Discord multi-account startup start the configured default or primary account before staggered specialist accounts.

## Best Possible Solution

Order Discord startup-delay candidates with the resolved configured default account first, then keep the current enabled/configured filtering and 10s stagger for the remaining accounts.

## Reproduction Assessment

Yes, by source inspection: configure enabled Discord accounts such as `billy`, `farber`, and `main` with `channels.discord.defaultAccount: "main"`; current main gives `main` a nonzero startup index because the account list is sorted before delay calculation. I did not run a live Discord restart in this read-only review.

## Solution Assessment

Yes: fixing Discord's startup-delay ordering to use the existing default-account resolver is the narrowest maintainable solution. The repair should avoid hard-coded account aliases and should not change the stagger duration or shared account-list ordering semantics for other callers.
