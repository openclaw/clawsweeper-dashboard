## Summary

Keep open: current main and the latest release still compute Discord startup delay from the sorted enabled account list, so a configured default account such as `main` can still be delayed behind alphabetically earlier specialist accounts.

## What This Changes

The issue requests that Discord multi-account startup place the configured default/primary account first while preserving the existing 10s stagger for the remaining accounts.

## Best Possible Solution

Order Discord startup-delay candidates with the resolved configured default account first, then keep the current enabled/configured filtering and 10s stagger for remaining accounts without adding new config or hard-coded account aliases.

Yes, by source inspection: configure enabled Discord accounts such as `billy`, `farber`, and `main` with `channels.discord.defaultAccount: "main"`; the current startup-delay code can assign `main` a nonzero index because account ids are sorted before delay calculation. ## Solution Assessment

Yes: using the existing Discord default-account resolver for startup ordering is the narrowest maintainable fix. The repair should not hard-code aliases, change the 10s stagger, or alter shared account-list ordering for other callers.
