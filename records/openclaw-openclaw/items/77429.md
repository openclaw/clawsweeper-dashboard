## Summary

Keep open: current main still computes Discord startup delay from the sorted enabled account list and does not promote the configured `defaultAccount`, so a primary `main` account can be delayed behind alphabetically earlier specialist accounts.

## What This Changes

Request Discord multi-account startup to start the configured default/main account first while preserving the existing 10s stagger for secondary accounts.

## Best Possible Solution

Use the existing Discord default-account resolver to put the resolved default account first in the startup-delay candidate list, keep deterministic ordering for the remaining enabled/configured accounts, and add regression coverage.

## Reproduction Assessment

Yes, by source inspection: with enabled accounts `billy`, `farber`, `lumbergh`, `main`, `maverick`, and `scout`, `listDiscordAccountIds` sorts alphabetically and `resolveDiscordStartupDelayMs` assigns `main` index 3. I did not run a live Discord startup in this read-only review.

## Solution Assessment

Yes: changing Discord's startup-delay ordering to use the existing `defaultAccount` contract is the narrowest maintainable fix. The repair should avoid hard-coded aliases such as `main` or `billy`.
