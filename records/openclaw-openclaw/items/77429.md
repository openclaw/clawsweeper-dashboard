## Summary

Keep open: current main still derives Discord startup delay from the sorted enabled account list, so a configured Discord `defaultAccount` can start after alphabetically earlier specialist accounts.

## What This Changes

The issue asks Discord multi-account startup to start the configured default/main account before staggered secondary accounts.

## Best Possible Solution

Start the resolved Discord default account at delay 0, keep the current 10s stagger and filtering for the remaining accounts, and cover the behavior with a focused regression test.

## Reproduction Assessment

Yes, by source inspection: with enabled accounts `billy`, `farber`, `lumbergh`, `main`, `maverick`, and `scout`, `listDiscordAccountIds` sorts alphabetically and `resolveDiscordStartupDelayMs` gives `main` index 3, producing a 30s delay. I did not run live Discord startup in this read-only review.

## Solution Assessment

Yes: fixing the ordering inside Discord's startup-delay calculation is the narrowest maintainable solution. The repair should use the existing `defaultAccount` contract and avoid hard-coded account aliases.
