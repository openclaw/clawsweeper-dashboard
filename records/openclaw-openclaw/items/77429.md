## Summary

Keep open: current main still computes Discord startup delay from the sorted enabled account list rather than promoting the configured default account, and the closed related report points back here as the canonical remaining bug.

## What This Changes

Requests Discord multi-account startup to start the configured default/main account before secondary accounts while preserving the existing stagger.

## Best Possible Solution

Order Discord startup-delay candidates with the resolved configured default account first, then preserve the existing enabled/configured filtering and 10s stagger for the remaining accounts.

## Reproduction Assessment

Yes, by source inspection: configure enabled Discord accounts such as `billy`, `farber`, and `main` with `channels.discord.defaultAccount: "main"`; current main gives `main` a nonzero startup index because the account list is sorted before delay calculation. I did not run a live Discord restart in this read-only review.

## Solution Assessment

Yes: using the existing Discord default-account resolver for startup ordering is the narrowest maintainable fix. The repair should avoid hard-coded account aliases and should not change the stagger duration or unrelated routing semantics.
