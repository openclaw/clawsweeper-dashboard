## Summary

Keep open: current main still delays Discord startup by the enabled account's alphabetically sorted index, so a configured `defaultAccount` such as `main` can wait behind secondary accounts. The repair is narrow and belongs in the Discord plugin startup-delay path.

## What This Changes

Make Discord multi-account startup prioritize the configured default/main account before staggered secondary accounts.

## Best Possible Solution

Prioritize the resolved Discord default account when building the startup-delay list, preserve the existing 10s stagger and filtering for the rest, and add focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible: with enabled accounts `billy`, `farber`, `lumbergh`, `main`, `maverick`, and `scout`, `listDiscordAccountIds` sorts alphabetically and `resolveDiscordStartupDelayMs` gives `main` index 3, producing a 30s delay. I did not run live Discord startup in this read-only review.

## Solution Assessment

Yes: fixing Discord's startup-delay candidate ordering to put `resolveDefaultDiscordAccountId(cfg)` first is the narrowest maintainable fix. The repair should avoid hard-coded aliases and keep the current enabled/configured filtering and deterministic ordering for non-default accounts.
