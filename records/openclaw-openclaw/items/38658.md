## Summary

Keep open: the PR targets a real Slack context behavior, but it is not ready to merge against current main because Slack now lives under `extensions/slack`, current docs/tests describe group history as pending-only, and the diff appends retained history after async dispatch where adjacent channel turns can be stored out of order.

## What This Changes

This PR changes Slack `requireMention=false` room handling so processed channel messages are retained in the group history window instead of clearing history after each dispatch.

## Best Possible Solution

Decide whether Slack group history should include processed `requireMention=false` turns; if yes, implement it in `extensions/slack` with ingress-ordered retention, updated tests, docs alignment, and a changelog entry.

## Reproduction Assessment

Yes. Source inspection shows current main clears Slack room history after dispatch and the existing `requireMention=false` monitor test asserts the second Body excludes the first processed message, while docs describe group history as pending-only.

## Solution Assessment

No. The PR's direction may be useful, but the current diff is stale and records retained history after async dispatch, which can reorder adjacent channel messages; a safer implementation would record in ingress order or sort retained history by timestamp and update the current docs/tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Record retained history in ingress order:** `src/slack/monitor/message-handler/dispatch.ts:552`
  - body: Both new `requireMention=false` paths append to `channelHistories` only after dispatch finishes. Slack top-level messages use debounce keys that include the message timestamp, so adjacent channel turns can dispatch concurrently; a slower earlier turn can be appended after a faster later one, and history replay uses insertion order. Record the retained entry before the async reply work, after building the current Body, or sort by timestamp before formatting.
  - confidence: 0.86
