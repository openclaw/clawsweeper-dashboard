## Summary

Keep this PR open: current main still clears Slack room history for accepted `requireMention=false` messages, but the PR changes documented pending-only group-history behavior and its post-dispatch retention can reorder concurrent Slack channel context.

## What This Changes

The PR threads Slack mention policy into prepared messages and retains a sliding history window for processed `requireMention=false` channel messages instead of clearing it after dispatch.

## Best Possible Solution

Decide the group-history contract first; if always-on Slack channels should retain recent context, rebase the change into `extensions/slack`, preserve arrival order by recording before async dispatch or sorting retained history by timestamp, and update docs, tests, and changelog together.

## Reproduction Assessment

Yes. Source inspection on current main shows two accepted `requireMention=false` Slack channel messages will not retain the first message for the second prompt, and the existing Slack tool-result test asserts that behavior.

## Solution Assessment

No. The PR addresses a real context gap, but post-dispatch recording is not the safest implementation and the docs still define group history as pending-only; ingress-time recording or timestamp-ordered formatting after maintainer approval would be more maintainable.

## Review Findings
