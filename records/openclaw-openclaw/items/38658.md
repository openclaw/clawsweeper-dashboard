## Summary

Keep open: the branch addresses a real Slack context gap and the author already fixed the earlier no-reply-path review concern, but it still changes the documented pending-only group-history contract and records retained history after async dispatch, which can reorder context under overlapping channel turns.

## What This Changes

The PR threads Slack mention policy into prepared messages and changes requireMention=false Slack channel turns to append processed messages to a sliding history window instead of clearing it, with tests and a changelog entry.

## Best Possible Solution

Decide the group-history contract first; if always-on Slack channels should retain recent context, implement it under extensions/slack with ingress-order recording or timestamp-ordered formatting, and align docs, tests, and changelog together.

## Reproduction Assessment

Yes for the observable context gap: source inspection on current main shows requireMention=false processed Slack room messages are cleared after dispatch and not retained for the next prompt. This does not prove a contract violation because current docs and tests define processed room history as not retained.

## Solution Assessment

No. This PR moves toward the desired behavior, but post-dispatch recording is not the safest implementation and the change needs an explicit decision to revise the pending-only group-history contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Preserve ingress order before retaining Slack history:** `src/slack/monitor/message-handler/dispatch.ts:550`
