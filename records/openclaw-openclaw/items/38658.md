## Summary

Keep open. Current main has not implemented the requested Slack history retention, but the PR changes documented pending-only group-history semantics, has a concrete ordering bug in the proposed retention path, and still lacks real Slack workspace proof.

## What This Changes

The branch threads `requireMention` through prepared Slack messages, conditionally retains processed `requireMention=false` room messages in history, updates Slack history tests, and adds a changelog entry.

## Best Possible Solution

Decide the Slack always-on channel context policy first, then implement any approved retention in `extensions/slack` with ingress-time or timestamp-ordered history, updated docs/tests/changelog, and redacted live Slack proof.

## Reproduction Assessment

Yes, by source inspection: current main clears Slack room history after processed turns and the existing Slack test asserts a second `requireMention=false` channel prompt excludes the first message. I did not establish a live Slack workspace reproduction.

## Solution Assessment

No. The direction may be useful, but this PR records retained history only after async dispatch and changes documented pending-only group-history semantics; ingress-time or timestamp-sorted retention after maintainer approval would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Preserve ingress order for retained Slack history:** `src/slack/monitor/message-handler/dispatch.ts:550`
  - body: The PR appends `requireMention=false` messages only after dispatch finishes. Top-level Slack channel messages use per-message debounce keys, so overlapping turns can complete out of order; `buildPendingHistoryContextFromMap` then emits insertion order and can feed the next prompt reversed context. Record at ingress or sort by Slack timestamp before formatting.
  - confidence: 0.82
