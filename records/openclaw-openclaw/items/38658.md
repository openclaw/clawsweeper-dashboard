## Summary

Keep this PR open, but not merge-ready: current main still clears processed Slack room history, while this branch changes documented pending-only group-history semantics, records retained messages after async dispatch, and has no real Slack workspace proof.

## What This Changes

The PR threads `requireMention` through Slack prepared messages and records processed `requireMention=false` channel messages into the history window instead of clearing it, with tests and a changelog entry.

## Best Possible Solution

Decide the Slack group-history product contract first, then implement any accepted retention in `extensions/slack` with ingress-time or timestamp-sorted ordering plus docs, tests, changelog, and redacted live Slack proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main clears Slack room history after processed turns, and the existing Slack tool-result test asserts a second `requireMention=false` channel prompt excludes the first message. I did not run a live Slack workspace reproduction.

## Solution Assessment

No. The direction may be useful, but this PR records retained history after async dispatch and conflicts with the current pending-only group-history docs; ingress-time retention or timestamp-sorted formatting after maintainer approval is safer.

- **[P2] Preserve ingress order for retained Slack history:** `src/slack/monitor/message-handler/dispatch.ts:550`
