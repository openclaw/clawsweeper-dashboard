Full review comments:

- **[P1] Keep node role out of operator methods:** `src/gateway/method-scopes.ts:31-47`
  - body: `NODE_ALSO_ALLOWED` grants node clients methods such as `config.schema`, `chat.send`, `chat.history`, `talk.mode`, `sessions.list`, and `tts.providers`; because `isRoleAuthorizedForMethod` returns true for this set before operator scope checks, node clients can bypass the current gateway trust boundary.

- **[P2] Move preprocessing into speech-core:** `src/tts/tts.ts:854-857`
  - body: Current main's `src/tts/tts.ts` is a facade over the plugin SDK TTS runtime, and active auto-TTS runs in `extensions/speech-core/src/tts.ts`. Adding preprocessing here will not affect the rebased runtime path.
  - confidence: 0.88
- **[P2] Mirror notts handling in the shell filter:** `scripts/tts-filter.sh:24-40`
  - body: The new shell filter handles only `<tts>`, so `<notts>visual</notts>` would print raw tags in text output and still include visual-only content in `VOICE_CLEAN`, contradicting the new tag semantics.
  - confidence: 0.93
- **[P2] Gate tag-triggered TTS on tag processing:** `src/tts/tts.ts:871-872`
  - body: The tagged-mode trigger checks only for `<tts>` and ignores the `processTtsTags` setting. A `<notts>`-only message will not trigger audio, while disabled tag processing can still force audio and speak markup.
  - confidence: 0.86
