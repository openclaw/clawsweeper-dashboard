## Review Findings

Overall correctness: not a patch

Overall confidence: 0.87

Full review comments:

- **[P2] Mirror `<notts>` handling in the shell filter:** `scripts/tts-filter.sh:24-40`
  - body: The added script claims to provide the same text/voice split, but `TEXT_CLEAN` only removes `<tts>` blocks and `VOICE_CLEAN` only unwraps `<tts>`. A message with `<notts>table</notts>` would show raw tags in text output and still speak the visual-only block, so the script needs the same `<notts>` unwrap/remove rules as the TypeScript path or should be dropped.
  - confidence: 0.94
- **[P2] Align tagged-mode gating with tag processing:** `src/tts/tts.ts:871-872`
  - body: `hasTtsTags` only checks `<tts>` and ignores `processTtsTags`. In `auto: "tagged"`, `<notts>`-only messages would skip TTS even though the author marked a speech-aware split, while `processTtsTags: false` would still let literal `<tts>` markup force audio generation. Gate this on the resolved tag-processing setting and include `<notts>` if the feature remains.
  - confidence: 0.9
