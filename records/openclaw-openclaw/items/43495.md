## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep node role out of operator RPCs:** `src/gateway/role-policy.ts:21`
  - body: This allow-list returns true for node callers on methods like `chat.send`, `config.get`, `sessions.preview`, and `talk.mode`. `authorizeGatewayMethod` skips operator scope checks once the role is `node`, so paired node clients would gain read/write UI RPCs without `operator.read`/`operator.write`; current tests explicitly reject `node` calling `status`.
  - confidence: 0.92
- **[P2] Mirror `<notts>` filtering in the shell helper:** `scripts/tts-filter.sh:22-44`
  - body: The script claims to mirror the TS preprocessing, but `TEXT_CLEAN` only removes `<tts>` blocks and `VOICE_CLEAN` only unwraps `<tts>`, so `<notts>...</notts>` tags remain visible and their visual-only content is spoken. Add the same strip-tags/remove-block logic here or drop the duplicate helper.
  - confidence: 0.94
- **[P2] Make tagged-mode gating follow tag processing:** `src/tts/tts.ts:869-870`
  - body: The new tagged-mode trigger checks only raw `<tts>` and ignores both `<notts>` and `processTtsTags: false`. In `auto: "tagged"`, a `<notts>`-only message will not synthesize, while a config that disables tag processing can still synthesize literal tag markup.
  - confidence: 0.88
