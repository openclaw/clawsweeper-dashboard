## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep node role access out of operator methods:** `src/gateway/role-policy.ts:21-23`
  - body: This lets any `node` role call the new allowlist, including read/write methods such as `config.get`, `chat.send`, and `status`, without the normal operator scopes. Current main deliberately restricts node role sessions to node methods, and the role-policy tests assert `node` cannot call `status`; this authorization widening is unrelated to the TTS feature and should be split or redesigned with explicit security review.
  - confidence: 0.9
- **[P2] Mirror `<notts>` handling in the shell filter:** `scripts/tts-filter.sh:23-44`
  - body: The new shell filter still only strips `<tts>` for chat text and only unwraps `<tts>` for voice text. A message containing `<notts>table</notts>` would show raw tags in the text output and speak the visual-only content, contradicting the feature semantics implemented in the TypeScript path.
  - confidence: 0.92
- **[P2] Gate tagged mode on processed speech tags:** `src/tts/tts.ts:869-874`
  - body: In tagged auto mode this only detects `<tts>`, so a reply that uses only `<notts>` around visual content plus ordinary speakable text outside will skip audio. The check also ignores the `processTtsTags=false` setting, where angle tags are supposed to remain literal text, so resolve the setting first and include `<notts>` only when tag processing is enabled.
  - confidence: 0.86
