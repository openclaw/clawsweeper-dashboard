## Summary

Keep open: the patch addresses the canonical `\OpenClaw Gateway` false positive, but it also hides slash-prefixed copy or stale OpenClaw tasks from status/doctor extra-service detection, and the external PR still lacks native Windows status/doctor proof.

## What This Changes

The PR strips leading slash/backslash prefixes before Windows scheduled-task name matching and adds regression coverage for slash-prefixed canonical gateway task names.

## Best Possible Solution

Strip root-folder prefixes, then ignore only exact canonical default/profile Windows task names while keeping copy/stale task detection, with a regression for prefixed copy tasks and redacted native Windows proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main parses raw `schtasks` names, so `\OpenClaw Gateway` misses the managed-name check and is reported during the deep Windows scan. I did not run a native Windows Task Scheduler repro.

## Solution Assessment

No. Prefix stripping is the right direction, but applying it before the broad `startsWith("openclaw gateway")` match hides prefixed copy/stale OpenClaw tasks; exact default/profile matching is the safer fix.
