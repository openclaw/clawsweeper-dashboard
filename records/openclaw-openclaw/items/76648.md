## Summary

Keep open. Current source has an OpenCode provider hook for Claude thinking profiles, but the lightweight thinking resolver used by directive/session paths only sees active providers or `provider-policy-api.js`, and OpenCode has no such policy artifact, matching the reported downgrade path.

## What This Changes

The issue reports that `opencode/claude-opus-4-7` downgrades `/think xhigh` to `high` even though the proxied Anthropic Opus 4.7 profile should allow `xhigh`.

## Best Possible Solution

Keep the OpenCode-owned Claude thinking profile available through the same lightweight policy surface used by directive and session validation, with regression coverage for the OpenCode route rather than another core special case.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection does identify a current-main remap path where `opencode/claude-opus-4-7` can lose `xhigh` when the OpenCode provider is not active and no policy artifact supplies its profile.

## Solution Assessment

Yes, the best fix appears to be a narrow OpenCode/provider-policy repair: expose the Claude thinking profile through OpenCode's lightweight provider policy surface and add coverage for directive/session validation.
