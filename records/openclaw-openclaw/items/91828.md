## Best Possible Solution

## Solution Assessment

Yes, this is an acceptable narrow fix: caller-side `path-mismatch` retry preserves the fs-safe boundary, and coalescing at the imported-source sync boundary covers the gateway load path without duplicating bridge logic. The remaining question is maintainer acceptance of the deliberate concurrent freshness tradeoff.
