## Summary

Keep open: the retry and same-vault coalescing approach is useful and well-proven, but the PR introduces an incomplete coalescing key that can share one bridge import/index result across different app configs. The remaining blocker is a narrow source-sync repair plus maintainer acceptance of the bounded coalescing behavior.

## What This Changes

The PR adds bounded `path-mismatch` retry handling for memory-wiki imported source page writes and coalesces concurrent bridge imported-source sync plus index refresh work.

## Best Possible Solution

Keep the bounded retry and imported-source-boundary coalescing shape, but key coalescing on the full artifact-discovery inputs or resolved workspaces before maintainers accept the session-state tradeoff.

## Reproduction Assessment

Yes, source-level reproduction is clear: current main routes visible wiki gateway calls through imported-source sync without retry/coalescing, and fs-safe can raise `path-mismatch` during verified reads/writes. This read-only review did not run the live load reproduction.

## Solution Assessment

No, not yet: caller-side `path-mismatch` retry is a good bounded layer, but the coalescing key is incomplete for the public-artifact discovery contract. The safer fix is to key on resolved memory workspaces or include the missing default workspace input plus a regression test.
