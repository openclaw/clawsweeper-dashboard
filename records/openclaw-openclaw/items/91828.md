## Summary

This PR is useful but not merge-ready: the retry path can recover the file write while reporting the import as skipped, which would suppress the downstream ingest log and index refresh for the exact path-mismatch case it is trying to fix.

## What This Changes

The PR adds bounded path-mismatch retries for imported memory-wiki source page writes and coalesces concurrent bridge source syncs for the same vault/config.

## Best Possible Solution

Keep the retry and coalescing direction, but make the retry path preserve the original write intent so successful recovered writes still trigger ingest logging and index refresh.

## Reproduction Assessment

Yes, from source inspection: fs-safe can throw path-mismatch after an atomic write, and the PR retry then recomputes existing content after the write and can report changed:false. I did not run tests because this review is read-only.

## Solution Assessment

No, not yet. The retry/coalescing shape is a reasonable narrow fix, but it must preserve changed/created accounting so downstream logs and index refresh remain correct.
