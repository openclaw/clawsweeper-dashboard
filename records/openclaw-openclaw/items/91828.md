## Label Justifications

- `P1`: The PR addresses a user-facing memory workflow that can fail gateway calls and contribute to severe process memory pressure under concurrent polling.
- `merge-risk: 🚨 session-state`: The new in-flight map intentionally shares imported memory and index-refresh results among concurrent callers based on a configuration-derived key.

## Summary

This PR remains necessary because current main still propagates transient imported-page path mismatches and independently runs concurrent bridge imports. The proposed plugin-local retry and single source-sync coalescing boundary are well targeted, preserve fs-safe refusal behavior, and have strong focused and production-path proof; no blocking correctness defect was found.

## What This Changes

The PR adds bounded recovery for transient fs-safe path mismatches and coalesces equivalent concurrent memory-wiki bridge imports through downstream index refresh.

## Best Possible Solution

Keep the retry policy in the memory-wiki writer, retry only transient path-mismatch failures, preserve every fs-safe symlink and alias refusal, and use one complete bridge-import-plus-index-refresh operation per behavior-equivalent configuration.

## Reproduction Assessment

Yes at source level. Current main exposes both the propagated path-mismatch boundary and duplicate concurrent sync entry points, while the supplied after-fix outputs exercise retry recovery and 60 concurrent production gateway calls; this read-only review did not independently rerun them.

## Solution Assessment

Yes. Caller-local bounded retry avoids weakening fs-safe globally, and coalescing at the imported-source boundary covers both expensive bridge discovery and downstream index refresh without adding competing cache paths.
