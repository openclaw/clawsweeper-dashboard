## What This Changes

The PR changes gateway TLS schema and runtime loading so empty or whitespace-only certPath/keyPath values are rejected or defaulted, with focused schema and runtime tests.

## Best Possible Solution

Keep the blank-path guard, but reject or default only fully blank values while preserving exact non-empty cert/key path strings and covering that compatibility case in tests.

## Solution Assessment

No as written. Schema plus runtime is the right boundary, but the implementation should preserve exact non-empty filesystem path strings instead of trimming them.
