## Summary

Keep open. The PR is a useful direct-attempt guard, but it does not cover the production fallback auth gate that the linked external-CLI harness issue describes, and it still lacks real behavior proof from a CLI-owned transport setup.

## What This Changes

The PR adds an early return in CLI attempt auth-profile selection to skip automatic harness auth-profile resolution when auth forwarding is disabled, plus a focused malformed-auth-store regression test.

## Best Possible Solution

Move the owned-transport auth skip into the production model-fallback precheck, preserve explicit user auth-profile forwarding, and prove it with regression coverage plus a redacted real CLI-owned transport run.

## Reproduction Assessment

Yes, source-reproducible: normal agent runs enter `runWithModelFallback`, which checks provider auth profiles and cooldown state before `runAgentAttempt`. The added test bypasses that production wrapper by calling `runAgentAttempt` directly.

## Solution Assessment

No. The early return is a useful local guard, but the narrow maintainable fix needs to align the production fallback precheck with transport ownership so stale OpenClaw auth state cannot block the harness before this helper runs.
