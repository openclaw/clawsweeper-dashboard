## Summary

Keep this PR open: it targets the right auth-selection area, but the patch only bypasses attempt-level automatic profile resolution and does not cover the pre-attempt `runWithModelFallback` auth cooldown gate that can still block canonical CLI-owned runtimes before `runAgentAttempt` is called. The supplied proof is also a mocked Vitest run rather than real CLI-owned transport behavior.

## What This Changes

The branch adds an early return in `resolveHarnessAuthProfileSelection` when harness auth forwarding is disabled and adds a direct `runAgentAttempt` regression test for malformed `auth-profiles.json`.

## Best Possible Solution

Apply the auth bypass at the production fallback boundary for CLI-owned runtime candidates, keep explicit user-selected profile forwarding intact, and add regression coverage that enters through `runWithModelFallback` or `agent-command` before asking for real CLI proof.

## Reproduction Assessment

Yes, from source inspection: canonical `anthropic/...` plus `agentRuntime.id: "claude-cli"` enters `runWithModelFallback`, which still resolves auth profile cooldown state before invoking `runAgentAttempt`. I did not execute a live repro because this review is read-only.

## Solution Assessment

No. The attempt-level early return is useful but not the narrow complete fix; the bypass also needs to happen before the fallback auth precheck, with production-path regression coverage.
