## Summary

Keep open. The patch is useful but incomplete because it skips auth-store reading only inside `runAgentAttempt`, while normal agent commands can still be stopped earlier by `runWithModelFallback`'s auth cooldown gate; the supplied proof is also a mocked Vitest run rather than a real CLI-owned transport run.

## What This Changes

The branch adds an early return to skip automatic harness auth-profile selection when auth forwarding is disabled, plus a direct `runAgentAttempt` regression test for malformed `auth-profiles.json`.

## Best Possible Solution

Move or mirror the owned-transport auth skip into the production model-fallback precheck, preserve explicit user auth-profile forwarding, and verify both the production path and a redacted real CLI-owned transport run before merge.

## Reproduction Assessment

Yes. Source inspection shows normal agent runs enter `runWithModelFallback`, which resolves provider auth profiles and cooldown state before calling `runAgentAttempt`; the PR's new test bypasses that wrapper.

## Solution Assessment

No. The early return is a useful local guard, but the maintainable fix needs the production fallback precheck to honor transport ownership before stale OpenClaw auth state can skip the harness.
