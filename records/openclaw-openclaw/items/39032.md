## Summary

Keep open. Current main still allows failed tool/toolResult text to become subagent completion fallback output, and the childCompletionFindings path still bypasses the normal ANNOUNCE_SKIP/NO_REPLY and reply-cleanup handling. The direct fix PR #39367 is closed unmerged, while #59649 and #74113 cover adjacent leak/race paths rather than this failed-tool filtering gap.

## What This Changes

The issue requests that subagent completion announcements exclude internal tool-failure reasoning and debugging text instead of forwarding failed tool output to requester sessions.

## Best Possible Solution

Land a focused agents fix that filters failed tool/toolResult messages before they can populate subagent fallback output, keeps successful tool-result fallback intact, and applies the same ANNOUNCE_SKIP/NO_REPLY and reply-classification cleanup to child aggregate completion output.

## Repair Work Prompt

Fix the subagent completion output leak described in this issue. In `src/agents/subagent-announce-output.ts`, prevent failed `tool` / `toolResult` messages from becoming `latestRawText` or selected fallback output by detecting top-level `isError: true`, `is_error: true`, and error-like details/status fields, while preserving successful tool-result fallback behavior. In `src/agents/subagent-announce.ts`, make the `childCompletionFindings` path honor the same ANNOUNCE_SKIP / NO_REPLY suppression and `stripAndClassifyReply` cleanup semantics as the normal reply branch before building the task completion result. Add regression coverage for failed tool-result filtering, successful tool-result fallback preservation, and child aggregate skip/silent behavior. Coordinate with #74073/#74113 so this patch does not replace the separate post-compaction retry fix. Add a user-facing changelog fix entry crediting the reporter(s); do not change provider APIs, plugin APIs, or broad subagent lifecycle behavior.
