## Label Justifications

- `P1`: The issue is a high-priority user-facing agent workflow bug that can disclose internal subagent/tool output to requester sessions.
- `impact:security`: The report is about information disclosure of internal tool errors, session keys, and implementation details.
- `impact:session-state`: The leak occurs through subagent completion/session handoff state becoming visible in the parent session.

## Summary

Keep open: current main and v2026.5.12 still allow raw child `tool`/`toolResult` text to become subagent completion output, and the focused fix path is the still-open maintainer-labeled PR at https://github.com/openclaw/openclaw/pull/80110.

## What This Changes

Prevent subagent completion announcements from forwarding internal tool-failure/toolResult reasoning to requester sessions.

## Best Possible Solution

Land or rebase https://github.com/openclaw/openclaw/pull/80110, or an equivalent selector/docs/test fix, and add coverage for the `childCompletionFindings` cleanup variant before closing this issue.

## Reproduction Assessment

Yes, at source level. A child history with empty assistant text followed by `toolResult` or `tool` text populates `latestRawText`, the selector returns it, and current e2e coverage still asserts that raw tool text is announced.

## Solution Assessment

Yes for the central leak path. The narrow maintainable fix is to stop selecting raw tool/toolResult output as completion text and fall back to post-compaction assistant text or `(no output)`, while adding coverage for the nested `childCompletionFindings` variant.
