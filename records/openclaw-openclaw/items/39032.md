## Summary

Keep open. Current main still lets failed tool/toolResult content be selected as subagent fallback output, and the child completion aggregate path still bypasses the normal ANNOUNCE_SKIP/NO_REPLY cleanup. The direct fix PR #39367 is closed unmerged, while #74113 targets the adjacent post-compaction retry race rather than this failed-tool filtering gap.

## What This Changes

The issue requests that subagent completion announcements stop forwarding failed tool/toolResult reasoning or other internal failure text from child sessions to requester sessions.

## Best Possible Solution

Land a focused agents fix that filters failed tool/toolResult messages before they can populate subagent fallback output, keeps successful tool-result fallback behavior, and applies the same ANNOUNCE_SKIP/NO_REPLY and reply-classification cleanup to child completion aggregates before they become task_completion.result.

## Reproduction Assessment

Yes. A unit fixture with a child transcript containing only a failed role=toolResult/tool message and no later assistant text should reproduce the fallback leak path; a registry fixture with child frozenResultText set to ANNOUNCE_SKIP or raw tool JSON should reproduce the childCompletionFindings bypass.

## Solution Assessment

Yes. The narrowest maintainable fix is in the subagent announce/output pipeline, not channel-specific tooling: filter failed tool results at extraction/selection time and normalize child aggregate text through the same silent-token cleanup as ordinary replies.
