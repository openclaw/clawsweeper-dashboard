## Summary

Keep open. Current main still allows subagent completion selection to fall back to raw `tool`/`toolResult` text, and the same behavior is present in the latest release; the focused fix is still open at https://github.com/openclaw/openclaw/pull/80110.

## What This Changes

The issue reports that subagent completion announcements can expose internal tool-failure reasoning or raw child tool output to the requester session.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80110 or an equivalent selector/docs/test fix so tool-only child histories produce no completion text, while leaving stale delivery/history replay work to its narrower follow-up issue.

## Reproduction Assessment

Yes, at source level. A child transcript with empty assistant output and a trailing `toolResult` or `tool` message can populate `latestRawText`, and current `selectSubagentOutputText` returns that fallback.

## Solution Assessment

Yes. The narrow maintainable fix is to stop the selector from treating raw tool/toolResult text as completion text and update the docs/regression tests; channel-specific `message` validation would only block one trigger.
