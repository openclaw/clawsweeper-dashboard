## Summary

Keep open: current main still allows raw child tool/toolResult text to become subagent completion output, and the focused fix path is the open protected-label PR at https://github.com/openclaw/openclaw/pull/80110.

## What This Changes

Prevent subagent completion announcements from forwarding internal tool-failure or raw tool output text to the requester session.

## Best Possible Solution

Land or rebase https://github.com/openclaw/openclaw/pull/80110, or an equivalent selector/docs/test fix, while tracking stale-delivery and unrelated-history variants separately.

## Reproduction Assessment

Yes, at source level. A child history with empty assistant text followed by a toolResult/tool message populates latestRawText, and current tests still assert that raw tool text is announced.

## Solution Assessment

Yes. The narrow maintainable fix is to stop treating raw tool/toolResult text as completion output and update docs plus regression tests; channel-specific message validation would only block one trigger.
