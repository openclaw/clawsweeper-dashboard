## Summary

Keep open. Current main still has a source-level leak path in the subagent completion extractor: failed `toolResult`/`tool` text is still eligible as fallback completion output, while the exact fix PR was closed unmerged and the related hardening PR did not touch this path.

## What This Changes

The issue asks OpenClaw to stop forwarding subagent failed-tool reasoning or internal error text into requester-session completion announcements.

## Best Possible Solution

Filter failed `tool`/`toolResult` messages before subagent completion selection, apply equivalent cleanup to child completion aggregates, and add regression coverage that preserves successful tool fallback while blocking failed-tool/internal text.

## Reproduction Assessment

Yes at source level: a child transcript ending in a failed `toolResult` or `tool` text can still populate `latestRawText` and be selected by `readSubagentOutput`. I did not run a live channel reproduction in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the subagent announce/output extraction and child aggregation pipeline, not in a channel-specific `message` tool gate.
