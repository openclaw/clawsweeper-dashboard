## Label Justifications

- `P1`: This is a high-priority agent workflow bug that can disclose internal subagent/tool output to requester sessions.
- `impact:security`: The report concerns information disclosure of internal tool errors, session keys, implementation details, or raw child output.
- `impact:session-state`: The leak occurs through subagent completion/session handoff state becoming visible in the parent/requester session.

## Summary

Keep open: current main and the latest release still allow raw subagent `tool`/`toolResult` text to become completion output when no assistant-authored result exists. The earlier closed fix attempt at https://github.com/openclaw/openclaw/pull/39367 did not land, and the focused current fix path is still the open, conflicting PR at https://github.com/openclaw/openclaw/pull/80110.

## What This Changes

The issue asks OpenClaw to prevent subagent completion announcements from forwarding internal tool-failure reasoning or raw child tool output to the requester session.

## Best Possible Solution

Rebase and land https://github.com/openclaw/openclaw/pull/80110 or an equivalent selector/docs/test fix, with coverage for the nested `childCompletionFindings` cleanup path before closing this issue.

## Reproduction Assessment

Yes, at source level. A child history with empty assistant text followed by `toolResult` or `tool` text populates `latestRawText`, the selector returns it, and current e2e coverage still expects that text in the requester handoff.

## Solution Assessment

Yes for the active fix direction. The maintainable fix is to stop treating raw tool/toolResult text as completion output and fall back to post-compaction assistant text, partial-progress text, or `(no output)` instead.
