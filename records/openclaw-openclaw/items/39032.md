## Summary

Keep open: current main and the latest shipped release still allow tool/toolResult text to become subagent completion output when no assistant-authored result exists, and the focused fix path is still an open protected-label PR.

## What This Changes

The issue requests preventing subagent completion announcements from forwarding internal tool-failure or raw tool-output reasoning into the requester session.

## Best Possible Solution

Land or rebase https://github.com/openclaw/openclaw/pull/80110, or an equivalent selector/docs/test fix, and include coverage for the nested `childCompletionFindings` cleanup variant before closing this issue.

## Reproduction Assessment

Yes, at source level. A child history with empty assistant text followed by `toolResult` or `tool` text populates `latestRawText`, the selector returns it, and current e2e coverage still expects that text in the requester handoff.

## Solution Assessment

Yes for the central leak path. The maintainable fix is to stop selecting raw tool/toolResult output as completion text and instead fall back to post-compaction assistant text or `(no output)`, with coverage for nested child findings.
