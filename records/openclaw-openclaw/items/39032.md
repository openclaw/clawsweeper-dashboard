## Summary

Keep open: current main and the latest release still allow raw child `tool`/`toolResult` text to be selected as subagent completion output when no assistant-authored result exists, and the focused fix is still open/unmerged.

## What This Changes

The issue reports that subagent completion announcements can expose internal tool-failure or raw child output to the requester session.

## Best Possible Solution

Rebase and land https://github.com/openclaw/openclaw/pull/80110 or an equivalent selector/docs/test fix, with coverage for the nested `childCompletionFindings` cleanup path; keep the lost-context variant tracked through https://github.com/openclaw/openclaw/issues/81359 and its paired PR.

## Reproduction Assessment

Yes, at source level. A child history with empty assistant output followed by `toolResult` or `tool` text populates `latestRawText`, and current main still returns that value as completion output.

## Solution Assessment

Yes for the active fix direction. The narrow maintainable fix is to stop treating raw tool/toolResult text as completion output and fall back to assistant text, post-compaction assistant text, partial-progress text, or `(no output)`.
