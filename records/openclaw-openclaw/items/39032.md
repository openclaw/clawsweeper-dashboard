## Summary

Keep open: current main still has the source-level leak path where raw child tool/toolResult text can become subagent completion output, and the focused maintainer-labeled fix remains open and conflicting at https://github.com/openclaw/openclaw/pull/80110.

## What This Changes

The issue asks OpenClaw to stop forwarding subagent tool-failure/debugging text into requester-session completion announcements.

## Best Possible Solution

Land or rebase https://github.com/openclaw/openclaw/pull/80110, or an equivalent selector/docs/test fix, so tool-only child histories produce no raw completion text while stale delivery and lost-context variants remain tracked separately.

## Reproduction Assessment

Yes, at source level. A child history with empty assistant output followed by a toolResult/tool message populates latestRawText, and current tests expect that raw tool text to be announced.

## Solution Assessment

Yes. The narrow maintainable fix is to stop treating raw tool/toolResult text as completion output and update the docs and regression tests; channel-specific message validation would only block one trigger.
