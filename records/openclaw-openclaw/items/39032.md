## Summary

Keep open. Current main still has the source-level leak path: tool/toolResult text can be selected as subagent completion output without checking error flags, and an open maintainer replacement PR addresses the raw-output selector but has not landed yet.

## What This Changes

The issue asks OpenClaw to prevent failed subagent tool output and internal tool-failure reasoning from being forwarded to the requester session as completion text.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80110 or an equivalent focused selector fix, ensure errored tool results and child completion aggregates cannot bypass cleanup, and keep regression coverage for tool-only and failed-tool histories.

## Reproduction Assessment

Yes at source level. A child transcript ending in failed `toolResult`/`tool` text can still populate `latestRawText`, and that value is returned as the completion fallback on current main.

## Solution Assessment

Yes, the best fix belongs in the subagent announce/output selection pipeline. A channel-specific `message` tool gate would only block one trigger and would not protect other failed tool or child aggregate paths.
