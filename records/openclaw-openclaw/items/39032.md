## Summary

Keep open. Current main still lets subagent completion selection fall back to raw `tool`/`toolResult` text, and the latest shipped release has the same path; the focused fix remains open at https://github.com/openclaw/openclaw/pull/80110.

## What This Changes

The issue asks OpenClaw to prevent subagent completion announcements from forwarding internal tool-failure or raw tool output into requester sessions.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80110 or an equivalent selector/docs/test fix so tool-only child histories produce no completion text, while leaving stale delivery/history replay work to its narrower follow-up issue.

## Reproduction Assessment

Yes, at source level. A child transcript with empty assistant output and a trailing `toolResult` or `tool` message can populate `latestRawText`, and current `selectSubagentOutputText` returns that fallback.

## Solution Assessment

Yes. The narrow maintainable fix is to stop the subagent output selector from treating raw tool/toolResult text as completion text, with docs and regression coverage; channel-specific `message` validation would only block one trigger.
