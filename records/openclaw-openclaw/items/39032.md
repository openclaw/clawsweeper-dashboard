## Summary

Keep open. Current main still has the source-level leak path, the latest release still contains it, and the focused fix remains in an open maintainer-labeled PR at https://github.com/openclaw/openclaw/pull/80110.

## What This Changes

The issue asks OpenClaw to prevent subagent completion announcements from forwarding internal tool-failure or raw tool output back to the requester session.

## Best Possible Solution

Land or rebase https://github.com/openclaw/openclaw/pull/80110, or an equivalent selector/docs/test fix, so tool-only child histories produce no completion text while stale delivery/history replay remains tracked separately.

## Reproduction Assessment

Yes, at source level. A child transcript with empty assistant output followed by a `toolResult` or `tool` message can populate `latestRawText`, and current `selectSubagentOutputText` returns that fallback.

## Solution Assessment

Yes. The narrow maintainable fix is to stop treating raw tool/toolResult text as completion output and update the docs and regression tests; channel-specific `message` validation would only block one trigger.
