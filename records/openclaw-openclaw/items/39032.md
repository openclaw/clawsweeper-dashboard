## Label Justifications

- `P1`: This is a high-priority agent workflow bug that can disclose internal subagent/tool output to requester sessions.
- `impact:security`: The report concerns information disclosure of internal tool errors, session keys, implementation details, or raw child output.
- `impact:session-state`: The leak occurs through subagent completion/session handoff state becoming visible in the parent/requester session.

## Summary

Keep open: current main still allows subagent `tool`/`toolResult` text to become completion output when no assistant-authored result exists, and the focused fix path is still the open PR at https://github.com/openclaw/openclaw/pull/80110.

## What This Changes

Prevent subagent completion announcements from forwarding internal tool-failure or raw tool output reasoning into the requester session.

## Best Possible Solution

Land or rebase https://github.com/openclaw/openclaw/pull/80110, or an equivalent selector/docs/test fix, and include coverage for the nested `childCompletionFindings` cleanup path before closing this issue.

## Reproduction Assessment

Yes, at source level. A child history with empty assistant text followed by `toolResult` or `tool` text populates `latestRawText`, the selector returns it, and current e2e coverage still expects that text in the requester handoff.

## Solution Assessment

Yes for the central leak path. The maintainable fix is to stop selecting raw tool/toolResult output as completion text and instead fall back to post-compaction assistant text or `(no output)`, with nested child-findings coverage before closure.
