## Summary

Keep open: current main still lets subagent completion selection extract tool/toolResult content and return `latestRawText` when no assistant text exists, and the focused fix path remains the open protected-label PR at https://github.com/openclaw/openclaw/pull/80110. The issue discussion is useful: the earlier unmerged PR targeted error tool results, and a later reporter points to a possible `childCompletionFindings` variant that maintainers should check before closing.

## What This Changes

The issue asks OpenClaw to stop subagent completion announcements from forwarding internal tool-failure reasoning or raw tool output to the requester session.

## Best Possible Solution

Land or rebase https://github.com/openclaw/openclaw/pull/80110, or an equivalent selector/docs/test fix, and include coverage for the `childCompletionFindings` variant if it still reproduces.

## Reproduction Assessment

Yes, at source level. A child history with empty assistant text followed by a `toolResult`/`tool` message populates `latestRawText`, the selector returns it, and current e2e coverage still asserts that raw tool output is announced.

## Solution Assessment

Yes for the central leak. The narrow maintainable fix is to stop treating raw tool/toolResult text as completion output and update docs plus regression tests; channel-specific `message` validation would only block one trigger.
