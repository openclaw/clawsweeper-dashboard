## Summary

Keep open. Current main still has source-level leak paths in the subagent announce/output pipeline: failed tool output can still be selected as fallback completion text, and nested child completion findings bypass the normal silent-token and reply cleanup path; the direct fix PR is closed unmerged, and the merged related hardening PR did not touch these central files.

## What This Changes

The issue requests that subagent completion announcements exclude internal tool-failure/debugging text and silent control tokens from requester-visible output.

## Best Possible Solution

Filter failed tool/toolResult messages before output selection, run child completion aggregates through equivalent silent-token and reply cleanup, and add regression coverage that preserves successful tool fallback and nested child-result aggregation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence fixture path: a child transcript ending in a failed `toolResult` can populate `latestRawText`, and nested `frozenResultText` can bypass the cleanup branch via `childCompletionFindings`.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the subagent announce/output pipeline, not in a channel-specific `message` tool gate, because the leak is selected during completion extraction and aggregation.
