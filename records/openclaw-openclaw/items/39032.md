## Summary

Keep open. Current main still has source-level leak paths: failed tool/toolResult text can be selected as fallback output, and child completion aggregation can bypass ANNOUNCE_SKIP/NO_REPLY cleanup; #39367 closed unmerged, while #59649 hardened adjacent task/status surfaces rather than this announce path.

## What This Changes

Ensure subagent completion announcements do not forward failed tool reasoning, debugging text, or silent child results to requester sessions.

## Best Possible Solution

Filter failed tool/toolResult messages before fallback selection, run child completion aggregates through the same silent-token/internal-text cleanup, and add regression coverage that still preserves successful tool-output fallback.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence fixture path: a child transcript ending with only a failed toolResult/tool message can populate fallback output, and raw `frozenResultText` can populate child completion findings without cleanup.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the subagent announce/output pipeline rather than a channel-specific `message` gate, because the leak is selected during completion extraction and aggregation.
