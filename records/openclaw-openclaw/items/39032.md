## Summary

Keep open. Current main still allows failed `toolResult`/`tool` content to become subagent fallback output, and child completion aggregation can bypass the normal `ANNOUNCE_SKIP`/`NO_REPLY` cleanup; the direct fix PR is closed unmerged and the open related PR targets an adjacent retry race.

## What This Changes

Prevent subagent completion announcements from forwarding internal tool-failure reasoning or raw failed tool output to requester sessions.

## Best Possible Solution

Land a focused agents fix that filters failed tool/toolResult messages before fallback selection, normalizes child completion aggregates through the same silent-token cleanup, and adds regression coverage for both paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence fixture path: a child transcript ending with only a failed `toolResult`/`tool` message can populate `latestRawText`, and a registry row with raw `frozenResultText` can populate `childCompletionFindings` without cleanup.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the subagent announce/output pipeline, not in a channel-specific `message` tool gate, because the leak is selected during completion extraction and aggregation.
