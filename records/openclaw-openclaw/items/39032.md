## Label Justifications

- `impact:security`: The report concerns information disclosure of internal tool errors, session keys, implementation details, and raw child output.

## Summary

Keep open: current main and the latest release still let raw child `tool`/`toolResult` text become subagent completion output when assistant text is absent, and the focused fix is still open and conflicting.

## What This Changes

The issue asks OpenClaw to stop subagent completion announcements from forwarding internal tool-failure reasoning or raw tool output into requester sessions.

## Best Possible Solution

Rebase and land `https://github.com/openclaw/openclaw/pull/80110` or an equivalent selector/docs/test fix, with coverage for the nested `childCompletionFindings` cleanup path; keep the lost-context variant tracked through `https://github.com/openclaw/openclaw/issues/81359`.

## Reproduction Assessment

Yes, at source level. A child history with empty assistant output followed by `toolResult` or `tool` text populates `latestRawText`, and current main still returns that value as completion output.

## Solution Assessment

Yes for the active repair direction. The narrow maintainable fix is to stop treating raw tool/toolResult text as completion output and fall back to assistant text, post-compaction assistant text, partial-progress text, or `(no output)`.
