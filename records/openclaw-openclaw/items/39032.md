## Summary

Keep open: current main and the latest release still allow raw child `tool`/`toolResult` text to become subagent completion output, while the focused fix path is the open maintainer-labeled PR at https://github.com/openclaw/openclaw/pull/80110.

## What This Changes

The issue asks OpenClaw to prevent subagent completion announcements from forwarding internal tool-failure or raw tool-output reasoning to the requester session.

## Best Possible Solution

Land or rebase https://github.com/openclaw/openclaw/pull/80110, or an equivalent selector/docs/test fix, and add coverage for the nested `childCompletionFindings` variant before closing this issue.

## Reproduction Assessment

Yes, at source level. A child history with empty assistant text followed by `toolResult` or `tool` text populates `latestRawText`, the selector returns it, and current e2e coverage still asserts that raw tool text is announced.

## Solution Assessment

Yes for the central leak path: the narrow maintainable solution is to stop selecting raw tool/toolResult output as completion text and fall back to post-compaction assistant text or `(no output)`. Channel-specific `message` validation would only block one trigger.
