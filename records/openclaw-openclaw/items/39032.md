## Summary

Keep open. Current main still allows tool/toolResult text to become subagent completion output through the raw fallback path, and the focused replacement PR that suppresses raw subagent tool output is still open rather than merged.

## What This Changes

The issue asks OpenClaw to prevent subagent completion announcements from forwarding internal tool-failure or raw tool output into the requester session.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80110 or an equivalent focused selector fix, and keep regression coverage for tool-only, failed-tool, and child aggregate completion histories.

## Reproduction Assessment

Yes, at source level. A child transcript ending in `toolResult` or `tool` text can still populate `latestRawText`, and `selectSubagentOutputText` returns that fallback on current main.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the subagent announce/output selection pipeline; a channel-specific `message` gate would only block one trigger and would not protect other tool-output or child aggregate paths.
