## Summary

Keep open. Current main and the latest shipped release still allow raw `tool`/`toolResult` text to become subagent completion output, while the focused replacement fix remains open at https://github.com/openclaw/openclaw/pull/80110.

## What This Changes

Prevent subagent completion announcements from forwarding internal tool-failure or raw tool-output text to the requester session.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80110 or an equivalent selector/docs/test fix so tool-only child histories produce no completion text, and keep stale subagent delivery/history replay work tracked separately in https://github.com/openclaw/openclaw/issues/78055.

## Reproduction Assessment

Yes, at source level. A child transcript with no assistant text and a trailing `toolResult` or `tool` message can still populate `latestRawText`, and `selectSubagentOutputText` returns that fallback on current main.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the subagent announce output selector with docs and regression tests; a channel-specific `message` gate would only block one trigger.
