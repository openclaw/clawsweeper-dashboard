## Summary

Keep open. Current main still allows subagent completion selection to fall back to raw tool/toolResult text, and the focused replacement fix is still an open protected-label PR rather than merged.

## What This Changes

The issue asks OpenClaw to stop subagent completion announcements from forwarding internal tool-failure reasoning or raw tool output to the requester session.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80110 or an equivalent selector/docs/test fix, and keep broader stale subagent delivery work tracked separately if it still reproduces.

## Reproduction Assessment

Yes, at source level. A child transcript with no assistant text and a trailing `toolResult` or `tool` message can still populate `latestRawText`, and `selectSubagentOutputText` returns that fallback on current main.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the subagent announce output selector; a channel-specific `message` gate would only block one trigger and would not protect other raw tool-output paths.
