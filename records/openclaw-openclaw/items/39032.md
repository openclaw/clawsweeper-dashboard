## Summary

Keep open. Current main still has the source-level leak path: failed `toolResult`/`tool` content is eligible for subagent completion output, the direct fix PR was closed unmerged, and related broader subagent issues do not replace this exact error-flag guard.

## What This Changes

The issue requests preventing subagent completion announcements from forwarding internal tool-failure reasoning or error text into requester sessions.

## Best Possible Solution

Filter errored tool/toolResult messages before fallback selection, apply equivalent silent-token and internal-context cleanup to child completion aggregates, and add focused regression coverage for both paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence fixture path: a child transcript ending in failed `toolResult` text can populate `latestRawText`, and nested `frozenResultText` can bypass the normal reply cleanup path via `childCompletionFindings`.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the subagent announce/output pipeline, not in a channel-specific `message` tool gate, because the unsafe text is selected during completion extraction and aggregation.
