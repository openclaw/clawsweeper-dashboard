## Summary

Keep open: current `main` and the latest release still have a cancellable `message_sending` hook, but shared outbound delivery does not pass session/run identity into that hook and there is no bounded transcript/tool-call/result evidence contract for deterministic verification.

## What This Changes

The issue requests an outbound pre-delivery hook context that includes session/run identity and enough deterministic evidence to annotate or block outbound messages before users see unsupported claims.

## Best Possible Solution

Define one additive, permission-aware `message_sending` contract that consistently threads available session/run identity plus a bounded, redacted recent evidence view across shared outbound and direct reply paths, then resolve the overlapping PRs against that contract.

## Reproduction Assessment

Yes, at source level: current `main` and `v2026.5.20` show shared outbound `message_sending` lacks `sessionKey`, `runId`, and verification evidence. This is an API feature gap rather than a failing documented behavior test.

## Solution Assessment

No for exposing raw session tool history exactly as proposed. The safer maintainable direction is a bounded, redacted, permission-aware evidence contract coordinated with the open session-identity, outbound-send, and agent-context PRs.
