## Summary

Current main implements the sessionKey subset, but the central deterministic-verification capability remains missing: outbound message hooks still lack runId and bounded transcript/tool-call/result evidence. The overlapping open PR is draft, conflicting, and narrower than this issue, so it is not a safe canonical replacement.

## What This Changes

Expose outbound hook session/run identity and bounded execution evidence so plugins can deterministically inspect, rewrite, or block messages before delivery.

## Best Possible Solution

Extend the capability-gated finalization lifecycle with the minimum bounded verification evidence and canonical run/session correlation, adding outbound runId only for delivery-policy cases that cannot use that boundary.

## Reproduction Assessment

Yes at source level: current main explicitly documents that outbound runId is unplumbed, and the message_sending event has no transcript/tool-result evidence. This is a reproducible capability gap rather than broken existing documented behavior.

## Solution Assessment

No, the standalone evidence-rich message_sending design is not yet established as the best solution because before_agent_finalize already owns gated final-answer inspection and revision semantics; maintainers should consolidate around that boundary first.
