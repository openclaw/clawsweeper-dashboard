## Summary

Keep open. Current main has a `message_sending` hook that can rewrite or cancel outbound content, but shared outbound delivery still drops session/run identity and there is no bounded transcript/tool-call/result evidence contract for deterministic verification.

## What This Changes

Request an outbound pre-delivery hook context that exposes outbound text, session identity, and session tool-call evidence so plugins can deterministically annotate, block, or retry messages.

## Best Possible Solution

Define one additive plugin hook contract for outbound verification: thread available `sessionKey` and `runId` through shared outbound and direct reply paths, expose a bounded redacted recent evidence view, then update docs and regression tests.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a `message_sending` hook invoked by shared outbound delivery receives only route context, while the event type lacks the requested verification evidence payload.

## Solution Assessment

Unclear as specified. The maintainable solution is an explicit bounded/redacted hook-context contract plus targeted plumbing/tests, not unbounded raw session history or legacy metadata dependence.
