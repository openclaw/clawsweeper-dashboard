## Summary

Source review still supports keeping this open: current main has a cancellable `message_sending` hook, but shared outbound delivery still does not pass session/run identity into that hook and there is no bounded transcript/tool-call/result evidence contract. The related open PRs cover overlapping session-identity slices, not the full deterministic verification API and privacy decision requested here.

## What This Changes

The issue requests a pre-delivery outbound message hook context that exposes session/run identity and recent tool/result evidence so plugins can deterministically verify, annotate, or block outbound messages.

## Best Possible Solution

Define one additive, permission-aware plugin hook contract that consistently threads available session/run identity and exposes a bounded, redacted recent verification evidence view across shared outbound and direct reply delivery, with docs and regression tests.

## Reproduction Assessment

Yes for the missing API surface: source inspection on current main shows the shared outbound hook receives only routing context, `OutboundSessionContext` lacks `runId`, and the sending event type lacks transcript/tool-call/tool-result evidence. This is a feature/API gap, not a broken documented behavior repro.

## Solution Assessment

No for exposing raw session tool history exactly as proposed. The safer maintainable solution is a bounded, redacted, permission-aware evidence contract coordinated with the overlapping session-identity PRs.
