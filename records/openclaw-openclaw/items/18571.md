## Summary

Keep open. Current main has a `message_sending` hook that can rewrite or cancel outbound content, and the message context type now includes optional `sessionKey`/`runId`, but the shared outbound delivery path still invokes the hook with only routing fields and there is no bounded transcript/tool-call/result evidence payload for deterministic verification. The related open PRs in the provided context overlap the session identity subset, while the central run identity plus evidence contract remains unresolved.

## What This Changes

Expose a pre-delivery outbound message hook with message text, channel/recipient, stable session/run identity, and bounded session tool-call/result evidence so plugins can annotate or block outbound replies before users see them.

## Best Possible Solution

Consolidate the related PRs into one additive plugin hook contract that consistently threads sessionKey/runId through shared outbound, direct reply, and channel-specific delivery paths, then add a bounded recent transcript/tool-call/result evidence view with docs and regression tests.
