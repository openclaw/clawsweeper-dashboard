## Summary

Keep open. Current main still has only the cancellable outbound `message_sending` surface plus optional correlation types; it does not consistently thread outbound session/run identity and does not expose the bounded tool-call/result evidence needed for deterministic verification.

## What This Changes

Expose outbound hook context that lets plugins correlate and verify outgoing messages before delivery using session/run identity and bounded recent evidence.

## Best Possible Solution

Define one additive plugin hook contract that threads available session/run identity consistently and exposes a bounded, redacted recent evidence view across shared outbound and direct reply delivery, with docs and regression tests.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the missing capability: shared outbound `message_sending` receives only route context, and the public event type has no transcript/tool-call/tool-result evidence fields.

## Solution Assessment

No, not as raw session tool history. The maintainable solution is a deliberately bounded and redacted evidence contract with explicit plugin permission semantics, coordinated with the overlapping session-identity PRs.
