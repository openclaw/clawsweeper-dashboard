## Summary

Keep open. Current main and v2026.6.8 have the sessionKey subset and the newer reply_payload_sending hook, but the requested outbound pre-delivery verification contract is still missing runId on message_sending and any bounded tool-call/result evidence view.

## What This Changes

The issue requests an outbound pre-delivery hook/context that exposes outbound text, session/run identity, and bounded verification evidence so plugins can deterministically annotate or block messages.

## Best Possible Solution

Keep this issue open for a maintainer-approved additive plugin-hook contract that threads outbound runId and a bounded, redacted same-turn evidence view through shared delivery paths with docs and regression tests.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.8 shows message_sending can rewrite/cancel and gets ctx.sessionKey, but it still has no outbound runId or bounded tool-call/result evidence.

## Solution Assessment

No. The existing sessionKey and reply_payload_sending work is useful but partial; the best fix is a maintainer-approved public plugin SDK contract for outbound run identity and redacted evidence, not a piecemeal hook-field patch.
