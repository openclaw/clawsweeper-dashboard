## Summary

Keep open. Current main has the `sessionKey` subset and the newer `reply_payload_sending` hook, but outbound `message_sending` still lacks `runId` and any bounded transcript/tool-call/result evidence for deterministic pre-delivery verification.

## What This Changes

The issue requests outbound hook context that lets plugins correlate a reply to its session/run, inspect recent execution evidence, and annotate or block the message before delivery.

## Best Possible Solution

Choose a maintainer-approved verification boundary, then add a least-privilege plugin contract for run correlation and bounded evidence across shared outbound delivery and direct reply paths, with docs and regression tests.

## Reproduction Assessment

Yes at source level: current main documents outbound `runId` as unplumbed and `message_sending` has no bounded transcript/tool-call/result evidence payload. This is a capability gap rather than a failing existing documented behavior.

## Solution Assessment

No, not as a standalone evidence-rich `message_sending` design yet. `before_agent_finalize` and `reply_payload_sending` already cover adjacent phases, so maintainers should decide the canonical boundary and least-privilege data shape first.
