## Summary

Keep open. Current main implements the useful `sessionKey` subset, but outbound `message_sending` still lacks `runId` and any bounded transcript/tool-call/result evidence contract for deterministic pre-delivery verification.

## What This Changes

Request to expose outbound pre-delivery hook context with message text, session/run identity, recent tool-call/result evidence, and block/retry control for deterministic verification.

## Best Possible Solution

Choose a least-privilege verification boundary, then add run correlation and bounded evidence consistently across shared outbound delivery and direct reply paths with docs and regression tests.

## Reproduction Assessment

Yes, at source level: current main documents outbound `runId` as unplumbed and `message_sending` has no bounded transcript/tool-call/result evidence payload. This is a capability gap rather than a failing existing documented behavior.

## Solution Assessment

No, not as a standalone unbounded `message_sending` evidence dump. Existing `before_agent_finalize` and `reply_payload_sending` cover adjacent phases, so maintainers should decide the canonical boundary and data shape first.
