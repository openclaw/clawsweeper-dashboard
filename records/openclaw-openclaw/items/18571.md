## Summary

Keep open: current main still has a cancellable `message_sending` hook, but it does not provide the requested session/run identity or bounded tool-call/result evidence needed for deterministic verification before delivery.

## What This Changes

Request to expose outbound pre-send hook context with message text, sessionKey/runId, and session tool-call/result evidence so plugins can annotate or block delivery deterministically.

## Best Possible Solution

Define one additive, permission-aware `message_sending` contract that consistently threads available session/run identity plus a bounded, redacted recent evidence view across outbound and direct reply paths.

## Reproduction Assessment

Yes, at source level: current main and v2026.5.20 show shared outbound `message_sending` lacks `sessionKey`, `runId`, and verification evidence. This is an API feature gap rather than a failing documented behavior test.

## Solution Assessment

No for exposing raw session tool history exactly as proposed. The safer maintainable direction is a bounded, redacted, permission-aware evidence contract coordinated with the open session-identity and outbound-send work.
