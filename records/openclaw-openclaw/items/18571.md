## Summary

Keep open: current main and v2026.5.20 still have a cancellable `message_sending` hook, but shared outbound delivery does not pass session/run identity and the public sending event has no bounded tool-call/result evidence for deterministic verification.

## What This Changes

The issue requests a pre-delivery outbound message hook context with outbound text, routing, session/run identity, recent tool evidence, and the ability to annotate or block delivery.

## Best Possible Solution

Define one additive, permission-aware `message_sending` contract that consistently threads available session/run identity plus a bounded, redacted recent evidence view across outbound and direct reply paths.

## Reproduction Assessment

Yes, at source level: current main and v2026.5.20 show shared outbound `message_sending` lacks `sessionKey`, `runId`, and verification evidence. This is an API feature gap rather than a failing documented behavior test.

## Solution Assessment

No, not as raw session tool history exactly as proposed. The safer maintainable direction is a bounded, redacted, permission-aware evidence contract coordinated with the open session-identity and outbound-send work.
