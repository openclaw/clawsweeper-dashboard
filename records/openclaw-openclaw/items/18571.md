## Summary

Keep open. Current main and the latest release have the sessionKey slice and the newer reply_payload_sending hook, but the requested outbound pre-delivery verification contract is still missing runId on message_sending and any bounded tool-call/result evidence view.

## What This Changes

The issue requests outbound pre-send hook context with session/run identity, outbound text, and bounded tool-call/result evidence so plugins can deterministically verify or block messages before delivery.

## Best Possible Solution

Define a maintainer-approved, additive plugin-hook contract for outbound run identity and redacted same-turn verification evidence, then wire it through shared delivery paths with SDK docs, baseline updates, and regression tests.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.10 shows message_sending can rewrite/cancel and gets ctx.sessionKey, but it still has no outbound runId or bounded tool-call/result evidence.

## Solution Assessment

No, not as a narrow automated patch. The sessionKey and reply_payload_sending work is useful but partial; the best fix is a maintainer-approved public plugin SDK contract for outbound run identity and redacted evidence.
