## Summary

Keep open: current main and v2026.5.12 have a cancellable `message_sending` hook, but shared outbound delivery still passes only routing context, `OutboundSessionContext` has no `runId`, and the public sending event has no bounded transcript/tool-call/tool-result evidence view. The linked PRs cover session-identity slices, not the full deterministic verification contract.

## What This Changes

The issue requests a pre-send outbound hook that can inspect outbound text, correlate by session/run identity, access bounded recent tool-call evidence, and cancel or annotate messages before delivery.

## Best Possible Solution

Define one additive, permission-aware plugin hook contract that consistently threads available session/run identity and exposes a bounded, redacted recent evidence view across shared outbound and direct reply delivery, with docs and regression tests.

## Reproduction Assessment

Yes for the missing API surface: source inspection on current main shows the shared outbound hook receives only routing context, `OutboundSessionContext` lacks `runId`, and the event type lacks transcript/tool-call/tool-result evidence. This is a feature/API gap rather than a broken documented behavior repro.

## Solution Assessment

No for exposing raw session tool history exactly as proposed. The safer maintainable solution is a bounded, redacted, permission-aware evidence contract coordinated with the overlapping session-identity PRs.
