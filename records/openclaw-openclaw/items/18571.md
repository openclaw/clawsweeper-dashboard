## Summary

Keep open: current main and v2026.5.12 have a cancellable `message_sending` hook, but shared outbound delivery still omits available session/run identity and there is no bounded transcript/tool-call/result evidence contract for deterministic pre-send verification. Prior issue discussion and the related open PRs support keeping this as a maintainer API, product, and security decision rather than closing it as implemented.

## What This Changes

Request to expose outbound pre-send hook context with outbound text, session/run identity, and bounded tool-call/result evidence so plugins can annotate or block unverifiable agent claims.

## Best Possible Solution

Define one additive, permission-aware plugin hook contract that threads available session/run identity consistently and exposes a bounded, redacted recent verification evidence view across shared outbound and direct reply delivery, with docs and regression tests.

## Reproduction Assessment

Yes for the missing API surface: source inspection on current main shows shared outbound `message_sending` receives only routing context, `OutboundSessionContext` lacks `runId`, and the sending event type lacks transcript/tool-call/tool-result evidence. This is a feature/API gap rather than a broken documented behavior repro.

## Solution Assessment

No for exposing raw session tool history exactly as proposed. The safer maintainable solution is a bounded, redacted, permission-aware evidence contract coordinated with the overlapping session-identity and outbound-send PRs.
