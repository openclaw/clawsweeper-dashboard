## Summary

Keep open: current `main` and `v2026.6.5` have cancellable `message_sending` and outbound `sessionKey` correlation, but they still do not expose outbound `runId` or a bounded tool-call/result evidence view needed for deterministic pre-delivery verification.

## What This Changes

The issue requests a pre-delivery outbound message hook context that exposes `sessionKey`/`runId` plus bounded tool-call or transcript evidence so verification plugins can block or annotate unsupported agent claims before delivery.

## Best Possible Solution

Define one additive, permission-aware outbound verification contract that threads available run identity and a bounded recent evidence view across shared outbound delivery and direct reply paths, with docs and regression tests.

## Reproduction Assessment

Yes, at source level: current `main` and `v2026.6.5` show `message_sending` gets outbound content and optional `sessionKey`, but not outbound `runId` or bounded tool-call/result evidence. This is an API feature gap rather than a failing documented runtime behavior test.

## Solution Assessment

No, not as raw session tool history exactly as proposed. The safer maintainable path is a bounded, redacted, permission-aware evidence contract coordinated with existing hook correlation surfaces.
