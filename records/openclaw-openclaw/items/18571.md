## Summary

Keep open. Current main has a cancellable `message_sending` hook and typed optional correlation fields, but shared outbound delivery still does not thread session/run identity into that hook and there is no bounded transcript/tool-call/result evidence contract for deterministic verification.

## What This Changes

Request an outbound pre-delivery hook contract that exposes outbound text, session/run identity, and bounded verification evidence so plugins can annotate, block, or retry messages before delivery.

## Best Possible Solution

Define one additive plugin hook contract for outbound verification: thread available session/run identity consistently and expose a bounded, redacted recent evidence view across shared outbound and direct reply delivery paths, with docs and regression tests.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: shared outbound `message_sending` is invoked with only route context, and the public event type lacks transcript/tool-call/tool-result evidence fields.

## Solution Assessment

No as phrased if it means raw session tool history. The maintainable direction is a deliberately bounded and redacted evidence contract with explicit plugin permission semantics, coordinated with the overlapping session-identity PRs.
