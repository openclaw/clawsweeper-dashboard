## Summary

Keep open. Current main partially solves the correlation problem by threading canonical sessionKey into outbound message hooks, but the central request still remains: outbound message_sending lacks runId and bounded verification evidence for deterministic same-turn policy decisions.

## What This Changes

The issue requests a plugin hook contract that lets outbound message_sending observe same-turn run identity and bounded evidence needed to verify or block content before send.

## Best Possible Solution

Define a maintainer-approved additive hook contract for outbound run identity plus redacted, bounded same-turn evidence, then wire the shared delivery and direct-reply paths with docs and focused tests.
