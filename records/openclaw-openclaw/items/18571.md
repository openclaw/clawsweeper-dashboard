## Summary

Keep open. Current main and the latest release have the sessionKey subset and the newer reply_payload_sending hook, but outbound message_sending still lacks outbound runId and any bounded transcript/tool-call/result evidence for deterministic pre-delivery verification.

## What This Changes

The issue requests a pre-delivery outbound message hook with session/run identity and bounded session evidence so plugins can deterministically verify, annotate, or block agent messages before users see them.

## Best Possible Solution

Choose a maintainer-approved verification boundary, then add a least-privilege plugin contract for run correlation and bounded evidence across shared outbound delivery and direct reply paths, with docs and regression tests.

## Reproduction Assessment

Yes at source level: current main documents outbound runId as unplumbed and message_sending has no bounded transcript/tool-call/result evidence payload. This is a capability gap rather than a failing existing documented behavior.

## Solution Assessment

Unclear as an implementation choice: adding a hook/evidence surface is the right class of solution, but maintainers still need to choose the bounded evidence shape, security contract, and canonical boundary before code should change.
