## Label Justifications

- `P2`: The issue is a valid plugin API enhancement with meaningful operational value, but it is not an urgent core runtime outage.
- `impact:session-state`: The requested API depends on reliable session/run correlation and recent tool activity for outbound verification.
- `impact:security`: The requested hook context could expose sensitive transcript, command, and tool-result data to plugins before delivery.

## Summary

Keep open: current main and the latest release provide cancellable `message_sending` and session-key correlation, but they still do not expose outbound `runId` or bounded tool-call/result evidence for deterministic pre-delivery verification.

## What This Changes

Request to expose outbound pre-delivery hook context with message text, session/run identity, recent tool-call/result evidence, and annotate/block behavior for deterministic verification.

## Best Possible Solution

Define one additive, permission-aware outbound verification contract that threads available run identity and a bounded recent evidence view across shared outbound delivery and direct reply paths, with docs and regression tests.

## Reproduction Assessment

Yes, at source level: current main and `v2026.6.5` show `message_sending` gets content and optional `sessionKey`, but not outbound `runId` or bounded tool-call/result evidence. This is an API feature gap rather than a failing documented runtime behavior test.

## Solution Assessment

No, not as raw session tool history exactly as proposed. The safer maintainable path is a bounded, redacted, permission-aware evidence contract coordinated with the existing hook correlation surfaces.

## Risks / Open Questions

- The current sessionKey-only contract cannot disambiguate concurrent turns in the same session, so it is not enough for deterministic run-level verification.
- Open related PR https://github.com/openclaw/openclaw/pull/63007 covers only part of the identity/cancellation surface; landing partial API slices without one canonical contract could create conflicting plugin semantics.
