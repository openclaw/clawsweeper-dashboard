## Summary

Keep open: current main has a cancellable `message_sending` hook, but it still does not provide outbound `runId` plumbing or a bounded transcript/tool-call/tool-result evidence contract for deterministic pre-delivery verification. The related open PRs only cover session-identity slices, not the full requested verification API.

## What This Changes

Expose session/run identity and recent verification evidence to outbound pre-send message hooks so plugins can annotate or block delivery deterministically.

## Best Possible Solution

Define one additive, permission-aware plugin hook contract that consistently threads available session/run identity and exposes a bounded, redacted recent evidence view across shared outbound and direct reply delivery, with docs and regression tests.

## Reproduction Assessment

Yes for the missing current-main API surface: source inspection shows shared outbound `message_sending` receives only routing context, `OutboundSessionContext` lacks `runId`, and the event type lacks transcript/tool-call/tool-result evidence. This is a feature/API gap rather than a broken documented behavior repro.

## Solution Assessment

No for exposing raw session tool history exactly as proposed. The safer solution is a bounded, redacted, permission-aware evidence contract coordinated with the overlapping session-identity PRs.
