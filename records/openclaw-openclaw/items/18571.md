## Summary

Keep open: current main and v2026.5.22 have a cancellable `message_sending` hook, but the requested outbound session/run identity plus bounded tool-call/result evidence contract is still missing, and related PRs only cover subsets or remain unmerged.

## What This Changes

Request to expose outbound text, session/run identity, and recent tool-call/result evidence to a pre-send message hook so plugins can annotate or cancel delivery for deterministic verification.

## Best Possible Solution

Define one additive, permission-aware `message_sending` or outbound-payload contract that consistently threads available session/run identity plus a bounded, redacted recent evidence view across shared outbound and direct reply paths.

## Reproduction Assessment

Yes, at source level: current main and v2026.5.22 show shared outbound `message_sending` lacks `sessionKey`, `runId`, and verification evidence. This is an API feature gap rather than a failing documented behavior test.

## Solution Assessment

No, not as raw session tool history exactly as proposed. The safer maintainable direction is a bounded, redacted, permission-aware evidence contract coordinated with the overlapping hook-context PRs.
