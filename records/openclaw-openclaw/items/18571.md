## Summary

Keep open: current main and the latest release still have a cancellable `message_sending` hook, but shared outbound delivery passes only routing context and the public sending event has no bounded tool-call/result evidence for deterministic verification.

## What This Changes

The issue requests a pre-delivery outbound message hook context that exposes outbound text, session/run identity, and bounded tool-call/result evidence so plugins can deterministically verify or block agent claims before delivery.

## Best Possible Solution

Define one additive, permission-aware `message_sending` or outbound-payload contract that consistently threads available session/run identity plus a bounded, redacted recent evidence view across outbound and direct reply paths.

## Reproduction Assessment

Yes, at source level: current main and `v2026.5.20` show shared outbound `message_sending` lacks `sessionKey`, `runId`, and verification evidence. This is an API feature gap rather than a failing documented behavior test.

## Solution Assessment

No, not as raw session tool history exactly as proposed. The safer maintainable direction is a bounded, redacted, permission-aware evidence contract coordinated with the overlapping hook-context PRs.
