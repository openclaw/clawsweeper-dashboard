## Summary

Keep open. Current main has a pre-send `message_sending` hook, but shared outbound delivery still calls it with only route context and the public event has no bounded transcript/tool-call/result evidence for deterministic verification; overlapping PRs cover only parts of the session-identity subset and remain unmerged.

## What This Changes

The issue requests a pre-delivery outbound message hook context that exposes session/run identity plus bounded message and tool-call evidence for deterministic verification.

## Best Possible Solution

Define one additive plugin hook contract for outbound verification: thread available session/run identity consistently and expose a bounded, redacted recent evidence view across shared outbound and direct reply delivery paths, with docs and regression tests.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: `message_sending` in shared outbound receives only route context, and the event type lacks the requested verification evidence payload.

## Solution Assessment

Unclear as specified. The maintainable solution is a deliberately bounded and redacted hook evidence contract, not unbounded raw session/tool history exposure.
