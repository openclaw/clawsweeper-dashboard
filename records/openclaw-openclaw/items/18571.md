## Summary

Keep open: current main has a cancellable `message_sending` hook, but shared outbound delivery still does not pass `sessionKey` or `runId` into that hook context and there is still no bounded transcript/tool-call/result evidence contract for deterministic verification. The related PRs are still open and cover only parts of the identity or outbound-send problem, so this still needs a maintainer API and security decision.

## What This Changes

The issue requests a pre-delivery outbound message hook/context that exposes outbound text, routing, session/run identity, and bounded tool/transcript evidence so plugins can deterministically annotate or block unverified agent claims.

## Best Possible Solution

Define one additive, permission-aware message hook contract that consistently carries available session/run identity and a bounded redacted recent evidence view across shared outbound and direct reply paths.

## Reproduction Assessment

Yes, at source level: current main shows shared outbound `message_sending` lacks `sessionKey`, `runId`, and verification evidence, while the latest release has the same gap. This is an API feature gap rather than a failing documented behavior test.

## Solution Assessment

No for exposing raw session tool history exactly as proposed. The safer maintainable solution is a bounded, redacted, permission-aware evidence contract coordinated with the overlapping session-identity and outbound-send work.
