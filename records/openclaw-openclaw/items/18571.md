## Summary

Keep open. Current main has a `message_sending` hook that can rewrite or cancel outbound content, but shared outbound delivery still invokes it with only route context, and the hook contract has no bounded transcript/tool-call/result evidence for deterministic verification.

## What This Changes

Request a pre-delivery outbound message hook that exposes outgoing text, session/run identity, recent tool-call evidence, and the ability to annotate or block delivery.

## Best Possible Solution

Define one additive plugin hook contract for outbound verification: keep rewrite/cancel semantics, thread available `sessionKey` and `runId` consistently, expose a bounded redacted recent evidence view, and cover shared outbound plus direct reply paths in docs and tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a `message_sending` hook in shared outbound delivery receives only route context, while the event type lacks the requested verification evidence payload.

## Solution Assessment

Unclear as specified. The maintainable solution is a deliberately bounded and redacted hook evidence contract, not unbounded raw session history exposure.
