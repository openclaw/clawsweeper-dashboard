## Summary

Keep open: current main has cancellable `message_sending`, but it still lacks outbound `runId` plumbing and any bounded transcript/tool-call/tool-result evidence contract needed for deterministic pre-delivery verification.

## What This Changes

Request to expose pre-delivery outbound message hook context with outbound text, session/run identity, recent tool evidence, and cancellation semantics for deterministic verification.

## Best Possible Solution

Define one additive plugin hook contract that threads available session/run identity consistently and exposes a bounded, redacted recent evidence view across shared outbound and direct reply delivery, with docs and regression tests.

## Reproduction Assessment

Yes for the current missing surface: source inspection on current main shows shared outbound `message_sending` receives only routing context, `OutboundSessionContext` has no `runId`, and the event type lacks transcript/tool-call/tool-result evidence.

## Solution Assessment

No, not as raw session tool history. The maintainable solution is a bounded, redacted, permission-aware evidence contract coordinated with the existing session-identity PRs.
