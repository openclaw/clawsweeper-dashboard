## Summary

Keep open. Current main and v2026.6.8 have the sessionKey subset from the merged outbound hook PR, but outbound message_sending still lacks outbound runId and any bounded transcript/tool-call/result evidence for deterministic pre-delivery verification.

## What This Changes

The issue requests a pre-delivery outbound message hook with session/run identity and bounded verification context so plugins can inspect, annotate, or block agent messages deterministically.

## Best Possible Solution

Choose a maintainer-approved verification boundary, then add outbound run correlation and a bounded least-privilege evidence view across shared delivery and direct reply paths with docs and regression tests.

## Reproduction Assessment

Yes, at source level: current main documents outbound runId as unplumbed and message_sending has no bounded transcript/tool-call/result evidence payload. This is a capability gap rather than a failing existing documented behavior.

## Solution Assessment

Unclear as an implementation choice: adding a hook/evidence surface is the right class of solution, but maintainers still need to choose the bounded evidence shape, access rules, and canonical boundary before code should change.
