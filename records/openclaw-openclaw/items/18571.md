## Summary

Keep open. Current main and the latest release cover the sessionKey subset, but outbound message hooks still lack runId and bounded transcript/tool-call/result evidence, so the deterministic pre-delivery verification request remains an unresolved product/security design item.

## What This Changes

Request to add a pre-delivery outbound message hook/context that exposes outbound text plus session/run/tool evidence so plugins can deterministically annotate, block, or retry messages before delivery.

## Best Possible Solution

Define one least-privilege verification boundary: either extend before_agent_finalize or add outbound run/evidence context with permission gating, docs, and tests, while preserving the sessionKey behavior from https://github.com/openclaw/openclaw/pull/73706.

## Reproduction Assessment

Yes, at source level: current main documents outbound runId as unplumbed and message_sending has no bounded transcript/tool-call/result evidence payload. This is a capability gap, not a failing existing documented behavior.

## Solution Assessment

No, not yet. A standalone evidence-rich message_sending hook is not established as the best solution because before_agent_finalize already owns final-answer inspection and bounded revision semantics, so maintainers should choose the canonical boundary first.
