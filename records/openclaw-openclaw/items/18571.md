## Summary

Keep open: current main has a cancellable `message_sending` hook, but it still does not provide outbound `runId` plumbing or any bounded transcript/tool-call/tool-result evidence contract for deterministic pre-delivery verification. The related open PRs cover parts of session identity, but not the full requested run identity plus evidence surface.

## What This Changes

Request to expose pre-delivery outbound hook context with message text, channel/recipient, session/run identity, and session tool-call evidence so plugins can annotate or block delivery.

## Best Possible Solution

Define one additive plugin hook contract that consistently threads available session/run identity and exposes a bounded, redacted recent evidence view across shared outbound and direct reply delivery, with docs and regression tests.

## Reproduction Assessment

Yes for the missing current-main surface: source inspection shows shared outbound `message_sending` receives only routing context, `OutboundSessionContext` lacks `runId`, and the event type lacks transcript/tool-call/tool-result evidence. This is a feature/API gap rather than a broken documented behavior repro.

## Solution Assessment

No, not as raw session tool history. The maintainable path is a bounded, redacted, permission-aware evidence contract coordinated with https://github.com/openclaw/openclaw/pull/73706, https://github.com/openclaw/openclaw/pull/44011, and https://github.com/openclaw/openclaw/pull/63007.
