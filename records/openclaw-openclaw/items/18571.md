## Summary

Keep open: current main and v2026.5.12 still provide a cancellable `message_sending` hook, but not the requested outbound run identity plus bounded verification evidence contract. The related open PRs cover session identity, agent identity, or gateway denial slices only, so this still needs maintainer API and security design.

## What This Changes

The issue requests an outbound pre-send hook context that exposes session/run identity and recent tool-call evidence so plugins can verify or block outbound claims before delivery.

## Best Possible Solution

Define one additive, permission-aware message hook contract that consistently threads available session/run identity and exposes a bounded redacted recent evidence view across shared outbound and direct reply paths.

## Reproduction Assessment

Yes, at source level: current main shows the outbound hook event and context lack the requested run identity and verification evidence, and v2026.5.12 has the same gap. This is a feature/API contract gap rather than a failing documented behavior test.

## Solution Assessment

No for exposing raw session tool history exactly as proposed. The safer maintainable solution is a bounded, redacted, permission-aware evidence contract coordinated with the overlapping session-identity and outbound-send work.
