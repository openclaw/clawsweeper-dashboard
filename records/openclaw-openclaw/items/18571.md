## Summary

Keep open. Current main has the sessionKey subset and an existing before_agent_finalize boundary, but outbound message_sending still lacks runId and bounded transcript/tool-call/result evidence, so the requested deterministic verification capability remains unresolved.

## What This Changes

Request to expose session/run identity and bounded execution evidence to a pre-delivery outbound message hook so plugins can verify, annotate, or block messages.

## Best Possible Solution

Keep the issue open until maintainers choose the canonical verification boundary and least-privilege hook payload, with outbound run/session correlation only where it is needed beyond the existing finalization hook.

## Reproduction Assessment

Yes at source level: current main explicitly documents that outbound runId is unplumbed, and message_sending has no bounded transcript/tool-result evidence payload. This is a reproducible capability gap rather than broken existing documented behavior.

## Solution Assessment

No, the standalone evidence-rich message_sending design is not yet established as the best solution because before_agent_finalize already owns gated final-answer inspection and revision semantics. Maintainers should choose whether to extend that boundary, add outbound run correlation, or define a new delivery-policy contract.
