## Summary

Keep open: current main and v2026.5.12 have a cancellable `message_sending` hook, but they still do not provide outbound `runId` plumbing or a bounded transcript/tool-call/tool-result evidence contract for deterministic pre-delivery verification. The related open PRs only cover session-identity slices and do not resolve the full requested API/privacy contract.

## What This Changes

Requests pre-delivery outbound hook context with outbound text, session/run identity, and tool-call/result evidence so plugins can verify, annotate, or block messages before delivery.

## Best Possible Solution

Define one additive, permission-aware plugin hook contract that consistently threads available session/run identity and exposes a bounded, redacted recent evidence view across shared outbound and direct reply delivery, with docs and regression tests.

## Reproduction Assessment

Yes for the missing current-main API surface: source inspection shows shared outbound `message_sending` receives only routing context, `OutboundSessionContext` lacks `runId`, and the event type lacks transcript/tool-call/tool-result evidence. This is a feature/API gap rather than a broken documented behavior repro.

## Solution Assessment

No for exposing raw session tool history exactly as proposed. The safer path is a bounded, redacted, permission-aware evidence contract coordinated with the overlapping session-identity PRs.
