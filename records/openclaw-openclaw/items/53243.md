## Summary

Keep this PR open, but it is not ready to merge: the branch still has a serialized-byte budgeting bug, current main moved the durable truncation path into the shared chat display projection helper, and the contributor has not provided real after-fix behavior proof.

## What This Changes

The branch raises the `chat.history` text cap to 64,000, adds raw UTF-8 byte-aware truncation in `src/gateway/server-methods/chat.ts`, and adds gateway tests for long ASCII and emoji-heavy assistant history.

## Best Possible Solution

Implement the truncation behavior in `src/gateway/chat-display-projection.ts`, budget visible prefixes against serialized message size, and align the gateway default/config and Control UI request cap with focused gateway and UI regression coverage.

## Reproduction Assessment

Yes, source-reproducible. Current main still truncates projected `chat.history` text before the serialized byte guard, and a focused byte calculation shows the PR fallback can still overflow the later JSON-byte cap for escape-heavy text.

## Solution Assessment

No. The branch fixes part of the symptom, but the durable fix should live in the shared projection path and truncate against serialized message bytes while maintainers decide the default and UI request caps.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

- **[P2] Measure history text budget with serialized bytes:** `src/gateway/server-methods/chat.ts:425-427`
  - body: The fallback trims candidates with `Buffer.byteLength(candidate, "utf8")`, but the later oversized gate measures the full `JSON.stringify(message)` byte size.
