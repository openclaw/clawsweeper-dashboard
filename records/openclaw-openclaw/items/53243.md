## Summary

Keep open: the branch addresses a valid truncation bug, but the proposed fallback still budgets raw text bytes while the gateway enforces JSON-serialized byte limits, and required real behavior proof is missing.

## What This Changes

The PR raises the legacy `chat.history` text cap to 64,000, adds raw UTF-8 byte-aware truncation, and adds gateway tests for long ASCII and emoji-heavy assistant history.

## Best Possible Solution

Implement the fix in `src/gateway/chat-display-projection.ts`, preserve text that fits the serialized history budget, truncate byte-heavy or JSON-escape-heavy text to a visible prefix, and align gateway/config/UI cap policy with focused tests.

## Reproduction Assessment

Yes, source-reproducible. A 50k ASCII assistant message can fit under the 128 KiB per-message byte guard, but current main applies the 8,000 default projection cap, or the Control UI's 4,000 RPC cap, before the serialized byte guard runs.

## Solution Assessment

No. The branch edits the older helper and measures raw text bytes, while current main projects through `chat-display-projection.ts` and the hard omission path measures JSON-serialized message bytes.
