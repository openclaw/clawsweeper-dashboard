## Summary

Keep open: the user problem remains relevant, but this branch is stale against current main and still has a correctness bug because its raw-byte fallback can collapse JSON-escape-heavy messages to the oversized placeholder.

## What This Changes

This PR changes `chat.history` truncation by raising the old gateway text cap to 64,000, adding a raw UTF-8 byte fallback, and adding long ASCII and emoji regression tests.

## Best Possible Solution

Move the fix into `src/gateway/chat-display-projection.ts`, choose and document the default limit, and truncate against serialized message bytes before the hard placeholder guard.

## Reproduction Assessment

Yes. Source inspection shows default Control UI `chat.history` still goes through the 8,000-character shared projection limit, and a small Node shell check demonstrates the PR-specific JSON-escaping defect without running mutating tests.

## Solution Assessment

No. The PR addresses the right user problem, but the maintainable fix should live in the shared projection path and use the same serialized-byte measurement as the oversized guard.

## Review Findings
