## Summary

Keep open. The PR targets a real chat.history truncation problem, but current main moved the truncation behavior into the shared projection helper, the branch is conflicting, and the proposed raw-byte fallback can still miss the serialized-byte budget that triggers the placeholder.

## What This Changes

The PR raises the old chat.history text cap to 64,000, adds raw UTF-8 byte-aware truncation in the gateway chat method, and adds gateway regression tests for long ASCII and emoji-heavy assistant history.

## Best Possible Solution

Implement serialized-message-byte-aware truncation in src/gateway/chat-display-projection.ts, then deliberately align the gateway default, config/docs, Control UI request cap, and focused gateway/UI regression coverage.

## Reproduction Assessment

Yes, source-reproducible. Current main still truncates projected chat.history text before the serialized-byte guard, and the PR-specific raw-byte fallback can exceed the JSON hard cap for escape-heavy text.

## Solution Assessment

No. The patch fixes part of the symptom, but the maintainable fix should live in the shared projection path and budget prefixes against serialized message bytes while aligning the UI/config policy.
