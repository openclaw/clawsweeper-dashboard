## Summary

Keep open. The paired bug is still unresolved on current main, but this branch needs rebase or replacement against the shared projection helper, has a serialized-byte budgeting defect, and lacks contributor real behavior proof before merge.

## What This Changes

The branch raises the legacy `chat.history` text cap to 64,000, adds raw UTF-8 byte-aware truncation, and adds gateway tests for long ASCII and emoji-heavy assistant history.

## Best Possible Solution

Rework the fix in `src/gateway/chat-display-projection.ts` so visible prefixes are budgeted against serialized per-message bytes, then align the gateway default/config and Control UI `maxChars` policy with focused tests.

## Reproduction Assessment

Yes, source-reproducible: current main slices projected text by `maxChars` before the serialized-byte guard, and a 50k ASCII message fits under the 128 KiB JSON budget while exceeding the default cap. I did not run a live Gateway/WebChat session in this read-only review.

## Solution Assessment

No. The PR fixes part of the symptom in the older server-methods helper, but current main centralizes projection elsewhere and the new raw-byte fallback can still hit the placeholder for JSON-escape-heavy text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

- **[P2] Budget history prefixes against serialized bytes:** `src/gateway/server-methods/chat.ts:425-427`
