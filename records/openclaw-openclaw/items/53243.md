## Summary

Keep open: this PR still targets a real chat-history truncation gap, but the branch is conflicting, edits an obsolete helper path, and leaves a serialized-byte budgeting bug plus missing real behavior proof.

## What This Changes

The branch raises the legacy `chat.history` text cap to 64,000, adds raw UTF-8 byte-aware truncation, and adds gateway tests for long ASCII and emoji-heavy assistant history.

## Best Possible Solution

Implement the behavior in `src/gateway/chat-display-projection.ts`, preserve long assistant text that fits the serialized history budget, visibly truncate byte-heavy or JSON-escape-heavy text before placeholder replacement, and align the chosen default/config/UI policy with focused gateway and UI coverage.

## Reproduction Assessment

Yes. Source inspection shows current main truncates chat-history text by `maxChars` before the serialized per-message byte guard, so a 50k ASCII assistant message can be sliced even though it fits the 128 KiB budget; I did not run a live Gateway/WebChat repro in this read-only review.

## Solution Assessment

No. The PR edits the old server-methods helper and budgets raw UTF-8 text bytes, while current main uses `chat-display-projection.ts` and the hard gate measures JSON-serialized message bytes.
