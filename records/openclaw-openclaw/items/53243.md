## Summary

Keep open. The PR targets a real chat.history truncation problem, but it still has a blocking serialized-byte budgeting defect, lacks required real behavior proof, and current main moved the truncation path into the shared projection helper.

## What This Changes

The branch raises the old gateway chat.history text cap to 64,000, adds raw UTF-8 byte-aware truncation in server-methods/chat.ts, and adds gateway tests for long ASCII and emoji-heavy assistant history.

## Best Possible Solution

Implement serialized-message-byte-aware truncation in the shared chat display projection path, then align the chosen default/config/UI request cap with focused gateway and UI regression coverage.

## Reproduction Assessment

Yes, source-reproducible. Current main still truncates projected chat.history text before the serialized byte guard, and the PR-specific fallback can still serialize above the hard cap for escape-heavy text.

## Solution Assessment

No. The branch fixes part of the symptom, but the maintainable fix should live in the shared projection path and budget prefixes against serialized message bytes while aligning the default/config/UI cap policy.
