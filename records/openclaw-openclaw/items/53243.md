## Summary

Keep open. The PR is the active companion to an open bug report, but it is not merge-ready: current main moved the truncation helper, the branch budgets raw UTF-8 text instead of serialized message bytes, and the external contributor has not provided real after-fix behavior proof.

## What This Changes

The PR raises the old gateway chat.history text cap to 64,000, adds raw UTF-8 byte-aware truncation, and adds gateway regression tests for long ASCII and emoji-heavy assistant text.

## Best Possible Solution

Rebase or replace the PR with a serialized-byte-aware fix in `src/gateway/chat-display-projection.ts`, align the chosen gateway/UI/config caps, and cover the gateway plus Control UI history paths.

## Reproduction Assessment

Yes, source-reproducible. Current main still truncates projected chat.history text before the serialized-byte guard, and a Node byte check shows the PR-specific raw-byte fallback can still exceed the JSON hard cap for escape-heavy text.

## Solution Assessment

No. The patch fixes part of the symptom, but the maintainable fix should live in the shared projection path, budget against serialized message bytes, and deliberately align the Control UI/config limits.

## Review Findings
