## Summary

Keep this PR open because it targets a real Discord message-action regression, but the current patch is not merge-ready: the shared migration now prefixes more than bare IDs and can rewrite already explicit targets into invalid delivery targets.

## What This Changes

The branch changes shared message-action normalization to prefix legacy channelId values when migrating to target/to, and also carries deterministic media-generation completion reply-instruction changes with tests.

## Best Possible Solution

Keep the channelId compatibility fix, but narrow it so only truly bare channel IDs gain channel intent while existing explicit prefixes and non-Discord channel target formats continue to round-trip.

## Reproduction Assessment

Yes. Current main source shows channelId is migrated into target/to before resolveActionTarget, while only the original channelId path supplied preferredKind: "group"; the Discord parser then throws for bare numeric targets without defaultKind.

## Solution Assessment

No. The direction is right, but the implementation should preserve already explicit/provider-qualified targets or carry channel intent through resolution instead of prefixing every non-channel value in a generic path.
