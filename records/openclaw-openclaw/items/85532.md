## Summary

Keep open: current main still classifies queued embedded runs with item-level terminal-looking Codex app-server progress as stalled and can allow active abort recovery after the abort threshold. GitHub also shows https://github.com/openclaw/openclaw/pull/85623 open with closing syntax for this issue, so this issue should remain open until that fix path lands or is explicitly closed.

## What This Changes

The issue reports diagnostic stuck-session recovery aborting active embedded Codex runs when low-level app-server progress looks terminal while task-level work may still be progressing.

## Best Possible Solution

Land a focused fix that gates terminal-looking progress on real staleness or durable run state, preserves recovery for true orphans, and closes this issue only after the linked implementation path merges with coverage and proof.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main classifies `rawResponseItem/completed` behind queued embedded work as stalled and the heartbeat test expects active abort recovery after the threshold; no live gateway reproduction was run in this read-only review.

## Solution Assessment

Yes for the general fix direction, but not closed yet. The narrow maintainable path is to gate item-level terminal-looking progress on true staleness or durable run/session completion, then review and land the open linked PR or replace it with an equivalent focused fix.
