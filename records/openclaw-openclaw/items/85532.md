## Summary

Keep open: current main still classifies queued embedded runs as stalled on item-level terminal-looking Codex app-server progress, and an open closing PR is already the implementation candidate. This issue should close only after that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw diagnostics to distinguish item-level terminal Codex app-server progress from durable run completion before aborting active embedded direct/group runs.

## Best Possible Solution

Review and land the linked closing PR or an equivalent core diagnostic change that treats recent item-level terminal progress as active progress and only permits abort after a durable terminal/no-progress invariant, with focused regression coverage.

## Reproduction Assessment

Yes. The source path is high-confidence: current main classifies a queued embedded run with rawResponseItem/completed as stalled, and the existing test expects active abort recovery after the threshold; no live gateway replay was run in this read-only review.

## Solution Assessment

Yes for the first repair. The narrow maintainable fix is to add a stale-progress or durable-terminal guard before stalled classification/active abort, and the open closing PR follows that shape.
