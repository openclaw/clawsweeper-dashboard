## Summary

Keep open: the reporter's log pattern matches current main and v2026.5.20 source/test behavior, where item-level terminal-looking Codex progress can classify queued embedded work as stalled and permit active abort once the session-age threshold is reached.

## What This Changes

The issue asks OpenClaw to stop diagnostic stuck-session recovery from aborting active embedded Codex runs solely because the latest app-server progress event looks terminal at the item level.

## Best Possible Solution

Gate active embedded-run aborts on durable run/session terminal state or a true no-progress invariant, then add regression coverage for item-level terminal progress followed by later non-terminal assistant/tool/browser progress with queued work.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main and v2026.5.20 tests exercise `rawResponseItem/completed` behind queued embedded work and expect active abort recovery after the abort threshold.

## Solution Assessment

Yes for the first repair. The narrowest maintainable fix is to stop item-level terminal notifications from authorizing active embedded-run abort unless the run/session is durably terminal or progress is actually stale; a separate `system_aborted` UI/API state can follow after the abort boundary is corrected.
