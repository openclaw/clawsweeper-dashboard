## Summary

Keep open: current main still has a source-proven path that classifies fresh item-level terminal Codex progress as stalled, and the open closing PR at https://github.com/openclaw/openclaw/pull/85623 should be reviewed or rejected before this issue closes.

## What This Changes

The issue asks OpenClaw diagnostic recovery to stop aborting active embedded Codex runs solely because the latest app-server progress event looks item-terminal.

## Best Possible Solution

Land a narrow diagnostic classifier fix that requires stale progress or durable run/session terminal evidence before active embedded-run abort recovery, then close this issue from the merged PR.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence because current main's classifier and test both classify fresh `rawResponseItem/completed` progress behind queued embedded work as stalled, and recovery eligibility then permits active abort after the abort threshold.

## Solution Assessment

Yes for the first repair. The narrowest maintainable fix is to stop item-level terminal notifications from authorizing active embedded-run abort unless progress is actually stale or the run/session is durably terminal; the linked PR at https://github.com/openclaw/openclaw/pull/85623 takes that shape.
