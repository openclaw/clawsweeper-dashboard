## Summary

Keep open: this is a plausible fix for the linked voice-call status bug, but it is not merge-ready because it widens active-call manager getters across non-status paths and can return the oldest persisted snapshot instead of the final transcript-bearing state.

## What This Changes

The PR changes voice-call manager lookup methods to fall back to persisted call records and adds store lookup helpers plus manager restore regression tests.

## Best Possible Solution

Keep active manager lookups active-only, add a status/read-only persisted lookup that returns the latest matching call by call id or provider id, and prove it with multi-snapshot tests plus redacted completed-call status output.

## Reproduction Assessment

Yes. Source inspection shows current status paths read active manager maps only, while terminal calls are persisted and then removed from those maps; I did not place a live call in this read-only review.

## Solution Assessment

No. The PR is in the right plugin area, but changing active manager getters is too broad and the new store helper should return the latest persisted snapshot rather than the first matching event.

- **[P2] Keep active lookups active-only:** `extensions/voice-call/src/manager.ts:435-449`
  - body: `getCall()` and `getCallByProviderCallId()` are used as active-call gates by message, media-stream, realtime, and auto-response paths. Falling back to persisted completed records here lets stale calls pass those gates outside `get_status`; keep these manager methods active-only and put the persisted fallback on the status/read path instead.
  - confidence: 0.9
- **[P2] Return the newest persisted call snapshot:** `extensions/voice-call/src/manager/store.ts:403`
  - body: `readCallRecordEvents()` returns every snapshot in oldest-to-newest order, and one call is persisted repeatedly as provider ids, transcript entries, and terminal state arrive. Using `find()` returns the oldest matching snapshot, so a completed evicted call can still come back without the final transcript or terminal state; search newest-first or collapse by id and add a multi-snapshot test.
  - confidence: 0.92
