## Summary

Keep open: the status-only persisted fallback is the right repair shape, but this patch is not merge-ready because it removes the read-scoped status redaction, drops the manager `coreSession` path, and still has mock-only proof.

## What This Changes

This PR adds persisted voice-call store lookup helpers and wires gateway, tool, legacy, and CLI status lookups to fall back to stored call records after active manager misses.

## Best Possible Solution

Keep the status-only persisted lookup, but preserve `toVoiceCallStatus`, keep `coreSession` wired through `CallManager`, fix lint/types, and add redacted real runtime or CLI/gateway proof for the completed-and-evicted call path.

## Solution Assessment

No, not in the current patch form. The status-only persisted fallback is the right boundary, but the implementation must preserve redaction and session wiring instead of returning raw records or dropping `coreSession`.
