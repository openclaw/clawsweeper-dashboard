## Summary

## What This Changes

The issue reports 6-7s per-turn agent prep latency in `bundle-tools` and asks for lazy loading, warm caching, or a fast path for MCP/tool initialization.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/94230 or an equivalent bundle-MCP catalog optimization is reviewed, merged, and backed by before/after latency proof.

## Reproduction Assessment

Yes at source level: current main and v2026.6.8 await connect and tools/list inside a per-server loop that can add per-server delays. I did not run a live timing reproduction with the reporter's exact MCP setup.

## Solution Assessment

Unclear whether the broader lazy-loading/cache request is the final best design, but the open PR's parallelization is the narrow active fix for the proven sequential bottleneck. Closing before that PR merges would be premature.
