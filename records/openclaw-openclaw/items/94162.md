## What This Changes

The issue asks OpenClaw to remove repeated 6-7s bundle-tools latency caused by eager MCP server and tool preparation on each agent request.

## Best Possible Solution

Review and land a bounded startup-latency fix with slow-MCP proof, likely through https://github.com/openclaw/openclaw/pull/94230 or a narrowed successor, while tracking broader lazy loading and fast-path work separately.

## Reproduction Assessment

Yes from source for the central behavior: current main awaits each configured MCP server connection and tools/list call inside one loop on the bundle-tools path. I did not run a live multi-server latency reproduction in this read-only review.

## Solution Assessment

Unclear as a final product direction: parallelizing independent startup is a plausible bounded mitigation, but the requested lazy-loading, warm-pooling, and simple-chat fast path need maintainer choice and proof.
