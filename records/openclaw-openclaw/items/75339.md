## Summary

Keep this PR open. The reported parser bug is real and the shared stream-parser direction is appropriate, but the latest head still has a P2 normalizer regression and the external PR only provides test/mock proof, not after-fix real behavior proof.

## What This Changes

This PR adds a shared structured-content normalizer for OpenAI-compatible completions streams, routes typed `delta.content` and reasoning blocks to text/thinking deltas, adds regression tests, and records a changelog fix for Mistral `[object Object]` replies.

## Best Possible Solution

Fix the normalizer fallback ordering, add a regression case for mixed non-string fields plus fallback string content/thinking, provide after-fix real Mistral or equivalent OpenAI-compatible output proof, then land the shared parser fix and reconcile #68418.

## Reproduction Assessment

Yes. Source inspection on current main shows array-shaped `choice.delta.content` reaches the text append/concatenation path, and the linked issue/PR context provides the Mistral typed-block payload shape; I did not run a live Mistral call in this read-only review.

## Solution Assessment

No for the current PR revision. Normalizing at the shared OpenAI-compatible stream parser is the right layer, but this patch should preserve fallback string fields after empty nested structures and include real behavior proof before merge.
