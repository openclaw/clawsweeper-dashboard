## Summary

Keep open. Current main and v2026.5.22 only provide diagnostics observe/abort/release recovery, while the requested steer-then-nudge-then-abort ladder is a new configurable recovery mode that needs maintainer design before implementation.

## What This Changes

Requests an opt-in diagnostics stuck-session recovery ladder that steers, then nudges, then aborts stalled embedded runs.

## Best Possible Solution

Design an opt-in diagnostics recovery ladder in the core diagnostics/session recovery layer, reuse existing sessions.steer and sessions.send semantics, and prove delivery stays scoped to the original session before abort fallback runs.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing contract. Source inspection does confirm current behavior lacks the requested ladder: config has no stuckSessionRecovery object and recovery actions are observe/abort/release only.

## Solution Assessment

Unclear until maintainers decide the product shape. The ladder is plausible and aligned with existing sessions.steer/sessions.send seams, but opt-in defaults, message templates, recheck timing, and delivery safety need review before a repair PR.
