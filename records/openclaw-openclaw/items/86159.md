## Summary

Keep open: current main and v2026.6.6 still provide manual `sessions.send`, `sessions.steer`, and `sessions.abort` RPCs, while diagnostics recovery remains observe/abort/release-only. The requested steer/nudge/abort ladder is new core recovery policy and config surface, so it needs maintainer product review before implementation.

## What This Changes

The issue requests a configurable diagnostics stuck-session recovery ladder that steers, nudges, and only then aborts stalled embedded runs.

## Best Possible Solution

Design an opt-in diagnostics-owned recovery policy that preserves today’s abort-compatible default, reuses the existing session send/steer semantics safely, and updates config/schema/docs/tests around ordering and delivery targeting.

## Reproduction Assessment

Not applicable as a feature request. Source inspection against current main and v2026.6.6 gives high confidence that the proposed config and steer/nudge actions do not exist today.

## Solution Assessment

Unclear until maintainers choose the recovery policy. The direction is plausible, but the safest shape is an explicit opt-in ladder with current abort-drain behavior preserved as the default and channel delivery safety tested.
