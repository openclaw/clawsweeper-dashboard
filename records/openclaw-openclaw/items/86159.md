## Summary

Keep open: current main and v2026.6.9 still expose manual session control plus observe/release/abort diagnostics recovery, but not the requested automatic steer/nudge/abort ladder or `diagnostics.stuckSessionRecovery` config. This remains a core recovery-policy feature that needs maintainer product direction before implementation.

## What This Changes

Requested configurable diagnostics recovery that escalates stuck embedded runs from steer to nudge to abort instead of only observing or abort-draining.

## Best Possible Solution

Maintainers should choose an explicit opt-in diagnostics recovery policy that preserves today’s abort-compatible default, reuses existing session send/steer/abort semantics, and adds focused tests for recheck timing, recovery events, and channel-target safety.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that the proposed config and steer/nudge recovery actions are absent from current main and v2026.6.9.

## Solution Assessment

Unclear until maintainers choose the recovery policy. The ladder is plausible, but the safer shape is an opt-in mode with today’s abort-drain behavior preserved by default and channel-target safety tested.
