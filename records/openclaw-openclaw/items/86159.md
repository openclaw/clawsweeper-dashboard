Keep open: current main and v2026.6.8 still expose only manual `sessions.send`/`sessions.steer`/`sessions.abort` RPCs plus diagnostics observe/release/abort recovery. The requested automatic steer/nudge/abort ladder and `stuckSessionRecovery` config remain a new core recovery policy that needs maintainer product direction.

## What This Changes

Request an automatic diagnostics stuck-session recovery ladder that steers, nudges, then aborts stalled sessions through configurable diagnostics settings.

## Best Possible Solution

Maintainers should choose an opt-in diagnostics recovery policy that preserves today’s abort-compatible default, reuses existing session send/steer semantics, and tests channel delivery safety before implementation.
