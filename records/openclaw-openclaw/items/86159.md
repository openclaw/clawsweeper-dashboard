## Summary

Keep this open: current main still has only manual `sessions.steer`, `sessions.send`, and `sessions.abort` primitives plus diagnostics recovery that observes, releases lanes, or abort-drains. The requested steer/nudge/abort ladder is a new diagnostics policy and config surface, so it needs maintainer product review rather than cleanup closure.

## What This Changes

The issue requests a configurable diagnostics recovery ladder for stuck sessions that tries steer, then nudge, then abort instead of abort-only recovery.

## Best Possible Solution

Design an explicit opt-in diagnostics recovery policy that preserves today’s abort-compatible default, reuses the existing session control semantics internally, and adds schema/docs/tests for recovery ordering, recheck timing, idempotency, and external-channel delivery safety.

## Reproduction Assessment

Yes, at source level: current config, recovery outcome types, heartbeat gating, and runtime logic show no steer/nudge ladder or proposed config keys. A live stalled-session repro was not needed to verify that the requested feature is absent on current main.

## Solution Assessment

Unclear until maintainers choose the policy. The requested ladder is plausible, but the safest implementation should be explicit and opt-in with today’s abort-drain behavior preserved as the default.
