## Summary

Keep open: current main and v2026.6.8 still implement diagnostics recovery as observe/abort/release only, while the requested steer/nudge/abort ladder would add new core policy and config surface that needs maintainer product review.

## What This Changes

The issue requests an opt-in diagnostics stuck-session recovery ladder that steers, then nudges, then aborts stalled sessions instead of going straight from observe-only to abort-drain.

## Best Possible Solution

Design an opt-in diagnostics recovery policy that preserves today’s abort-compatible default, reuses existing session send/steer semantics safely, and updates config/schema/docs/tests around ordering, rechecks, and delivery targeting.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that the requested config object and steer/nudge recovery actions do not exist on current main or v2026.6.8.

## Solution Assessment

Unclear until maintainers choose the product policy. The direction is plausible, but the safest shape is an explicit opt-in ladder with current abort-drain behavior preserved as the default and channel delivery safety tested.
