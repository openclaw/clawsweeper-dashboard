## Summary

This should stay open: current main has the manual `sessions.steer`, `sessions.send`, and `sessions.abort` RPCs, but diagnostics recovery still only observes, abort-drains, releases lanes, or no-ops. The requested configurable steer/nudge/abort ladder is new core recovery policy and config surface, so it needs maintainer product/design review before implementation.

## What This Changes

Add a configurable diagnostics stuck-session recovery ladder that tries steer, then nudge, then abort-drain instead of only observing until the abort threshold.

## Best Possible Solution

Design an opt-in diagnostics recovery policy that keeps today’s abort-compatible default, reuses the existing session RPC semantics internally, and adds focused config/schema/docs/tests for recovery ordering and channel-target safety.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks the proposed ladder and only supports the existing observe/abort/release recovery paths.

## Solution Assessment

Unclear until maintainers choose the policy. The request is plausible, but the safest implementation would be an explicit opt-in ladder with today’s abort-drain behavior preserved as the default.
