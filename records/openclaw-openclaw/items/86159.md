## Summary

Keep open: current main and v2026.6.9 still provide manual `sessions.send`/`sessions.steer` plus diagnostics observe/release/abort recovery, but not the requested automatic steer/nudge/abort ladder or `diagnostics.stuckSessionRecovery` config. This remains a core recovery-policy and config-surface feature that needs maintainer product direction.

## What This Changes

The issue requests a configurable diagnostics recovery ladder that tries session steering, then a nudge message, then abort-drain for stuck embedded runs.

## Best Possible Solution

Maintainers should choose an explicit opt-in diagnostics recovery policy that preserves today’s abort-compatible default, reuses existing session send/steer/abort semantics, and adds focused tests for recheck timing, recovery events, and channel-target safety.

## Reproduction Assessment

Not applicable: this is a feature request for a new automatic recovery mode and config object, not a report that current documented behavior is broken. Source inspection gives high confidence that the proposed mode is absent from current main and v2026.6.9.

## Solution Assessment

Unclear until maintainers choose the recovery policy. The ladder is plausible, but the safer shape is an opt-in mode with today’s abort-drain behavior preserved by default and channel-target safety tested.
