## Summary

Keep open: current main and v2026.6.9 still provide manual send/steer plus diagnostics observe/release/abort recovery, but not the requested automatic steer/nudge/abort ladder or `diagnostics.stuckSessionRecovery` config.

## What This Changes

The issue requests a configurable diagnostics recovery ladder that steers a stalled session, nudges it after a recheck, and abort-drains only as the final fallback.

## Best Possible Solution

Maintainers should choose an explicit opt-in recovery policy that keeps today’s abort-compatible default, reuses existing session send/steer/abort semantics, and proves recheck timing plus channel-target safety.

## Reproduction Assessment

Not applicable as a feature request; source inspection is enough to verify the proposed config object and steer/nudge actions are not present on current main or v2026.6.9.

## Solution Assessment

Unclear until maintainers choose the recovery policy. The ladder is plausible, but the maintainable shape should be opt-in and preserve today’s abort-drain behavior by default.
