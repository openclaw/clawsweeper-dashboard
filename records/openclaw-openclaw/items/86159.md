## Summary

Keep this open: current main still has manual `sessions.steer`, `sessions.send`, and `sessions.abort`, but diagnostics recovery itself remains observe/release/abort-only and the requested configurable steer/nudge/abort policy is not implemented. The remaining work is a product/config decision, not conservative cleanup.

## What This Changes

The issue requests an opt-in diagnostics recovery ladder that steers, then nudges, then aborts stalled embedded runs, with config, rechecks, and structured recovery events.

## Best Possible Solution

Design an explicit opt-in diagnostics recovery policy that preserves today’s abort/release default, reuses the existing session-control semantics internally, and adds config/schema/docs/tests for ordering and channel-target safety.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks the proposed ladder and only supports observe/release/abort recovery paths.

## Solution Assessment

Unclear until maintainers choose the policy. The maintainable direction is likely an explicit opt-in ladder that keeps today’s abort-compatible behavior as the default and proves channel-target safety.
