## Summary

Keep open: current main still has no cron preflight or run-log diagnostic for the case where `toolsAllow` explicitly requests `web_search` but the runtime has no enabled search provider, and the related closed issues fixed different configured-provider selection bugs rather than this disabled-provider warning gap.

## What This Changes

The issue requests an operator-visible diagnostic or failure when a cron `agentTurn` allows `web_search` while no web search provider plugin is enabled.

## Best Possible Solution

Add a narrow non-fatal cron/embedded-run diagnostic for explicitly requested `web_search` when no enabled search provider is available, while keeping provider enablement plugin-owned and avoiding new config surface.

## Reproduction Assessment

Yes, at source level: `web_search` remains callable unless config disables it, and the no-provider error only appears inside `runWebSearch` when the tool executes. I did not run a live cron job because this review is read-only.

## Solution Assessment

Yes, a focused diagnostic at the cron or embedded-run preflight/diagnostics boundary is the narrowest maintainable fix. Failing every run hard would be more compatibility-sensitive, and changing the model prompt would not give operators a reliable signal.
