## Summary

Keep this PR open, but do not merge it as written. Current main still returns a hard `timeout` after a successful `agent` start when `agent.wait` times out, and the PR's compensation direction matches the linked production reports; however, its accepted-no-reply branch returns before starting the existing background A2A announce flow, which preserves part of the unobserved-run failure mode.

## What This Changes

The PR adds `compensateAfterWaitTimeout()` and focused tests so `sessions_send` re-reads target history after an `agent.wait` timeout and can return `ok` for a late reply or `accepted` for an accepted run with no visible reply yet.

## Best Possible Solution

Revise this PR so `sessions_send` still distinguishes wait timeout from send failure, but preserves the existing background A2A delivery contract: late replies return `ok`, accepted still-running runs return `accepted`, and accepted runs without an inline reply start `runSessionsSendA2AFlow` with the run id so the eventual result can still announce. Add focused coverage and a changelog entry before merge.

## Reproduction Assessment

Yes. A focused mock path reproduces both sides: on current main, `agent` success plus `agent.wait` timeout returns a hard `timeout`; on the PR head, a compensation result with `accepted: true` and no `newReply` returns before any A2A call is started.

## Solution Assessment

No as written. The compensation helper is a narrow fit for the misclassification bug, but the accepted-no-reply branch needs to preserve the background announce/follow-up behavior before this is the best complete fix.
