## Summary

Keep this PR open, but it is not merge-ready. The compensation idea is still a meaningful candidate for the linked `sessions_send` timeout bug, but the branch is conflicting with current main and places history compensation before current pending-error and terminal-timeout guards, which can mask hard timeout diagnostics as `ok` or `accepted`.

## Best Possible Solution
