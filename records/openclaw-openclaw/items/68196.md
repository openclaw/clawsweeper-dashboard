## Summary

The current main branch still has a real `sessions_send` timeout edge that this PR is trying to address, but the branch is dirty against main and the patch conflicts with newer timeout/A2A handling by masking terminal timeouts and skipping the delayed follow-up path.

## What This Changes

The PR adds a `compensateAfterWaitTimeout` helper plus `sessions_send` timeout handling and focused tests to return a late reply or accepted state after an `agent.wait` timeout.

## Best Possible Solution

Rebase the patch onto current main and make compensation preserve terminal timeouts, return `ok` only for a real new post-baseline reply, and keep delayed A2A alive when there is no immediate reply.

## Reproduction Assessment

Yes, from source inspection: `agent.wait` can return a non-terminal timeout after a successful start, and current tests model both delayed reply recovery and terminal timeout preservation. I did not run tests because this was a read-only review.

## Solution Assessment

No as written: post-timeout compensation is the right boundary, but this branch must be rebased and must preserve current terminal-timeout and delayed-A2A contracts before it is the maintainable fix.
