## Summary

Keep open. The PR targets a real sessions_send timeout bug, but current main has overlapping accepted/pending A2A handling, the PR still adds a distinct late-reply compensation path, and the proposed patch has blocking timeout/A2A regressions plus missing external real behavior proof.

## What This Changes

Adds post-timeout sessions_send compensation by reading target session history after agent.wait timeout, returning ok or accepted, with focused run-wait and sessions tests.

## Best Possible Solution

Rebase and consolidate this PR onto current main, keeping terminal timeout preservation and delayed A2A delivery while adding late-reply compensation only when it has concrete post-baseline evidence.

## Reproduction Assessment

Yes. Focused mocks can reproduce the original bug by making agent start succeed and the first agent.wait return a non-terminal timeout; PR regressions are source-reproducible with terminal timeout metadata and accepted/no-reply compensation cases.

## Solution Assessment

No as written. The compensation direction is narrow and useful, but the patch must preserve terminal timeout results and keep the delayed A2A flow alive before it is the best maintainable fix.
