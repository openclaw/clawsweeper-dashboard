## Summary

Keep this PR open for human review: the branch still contains correctness and security-sensitive delivery regressions, and the requested raw full-reply delivery needs maintainer product judgment plus real threaded setup proof before merge.

## What This Changes

The branch sends the final A2A `latestReply` directly to the resolved announce target, moves announce generation into the requester session, suppresses duplicate announce sends, and passes `threadId`.

## Best Possible Solution

Define one policy-gated final-delivery path after ping-pong that can post the intended final content only after control-token filtering, while preserving lanes, provenance, `threadId`, docs, tests, and redacted real thread proof.

## Reproduction Assessment

Yes for the PR-introduced defects: PR-head source shows direct `latestReply.trim()` channel delivery while current main gates external delivery through announce suppression and non-deliverable-token checks. The original threaded ack complaint still lacks after-fix real setup proof.

## Solution Assessment

No. Direct raw sends are not the narrowest maintainable solution because they bypass the existing announce policy boundary; a single policy-gated final-delivery decision after ping-pong is safer.
