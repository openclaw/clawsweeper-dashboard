## Summary

Keep open: current main has not implemented raw full-reply thread delivery, and this PR is not mergeable as-is because it bypasses announce suppression/security policy and conflicts with current main A2A safeguards.

## What This Changes

The PR changes the A2A `sessions_send` flow to send `latestReply` directly to the resolved announce target, run the announce step in the requester session, suppress duplicate announce delivery, and pass `threadId` in the fallback send.

## Best Possible Solution

Define one final A2A delivery path that gates external posting on announce suppression, preserves current main provenance and lane contracts, sends at most one approved message, and includes focused tests plus a changelog entry.

## Reproduction Assessment

Yes. A mocked A2A send with an announce target, non-empty `latestReply`, and an announce step returning `ANNOUNCE_SKIP` follows the PR branch's direct-send path before the skip check; current main has source and test coverage for suppression before external delivery.

## Solution Assessment

No. Posting `latestReply` directly before policy evaluation is not the narrowest maintainable fix; the safer path is to choose the allowed final content after suppression/provenance checks and then perform exactly one external send.
