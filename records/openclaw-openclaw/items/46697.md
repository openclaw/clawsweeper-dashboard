## Summary

Keep open: current main has not implemented raw full-reply forwarding, and this PR is not safe to merge as-is because it bypasses announce suppression/control-token policy and regresses current lane/provenance contracts.

## What This Changes

The PR changes `src/agents/tools/sessions-send-tool.a2a.ts` to send the target session's final `latestReply` directly to the resolved announce target, run the announce step in the requester session, suppress duplicate announce delivery, and pass `threadId` in the fallback send.

## Best Possible Solution

Define one approved A2A external-delivery path that chooses final content after ping-pong and suppression checks, preserves current provenance and per-session lane behavior, sends at most one message, and includes focused regression tests plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection shows current main sends only the policy-gated announce reply, while PR head sends `latestReply` directly before announce suppression can run; the existing tests also cover suppression of exact control replies.

## Solution Assessment

No. Posting raw `latestReply` directly is not the safest maintainable fix because it bypasses current suppression/provenance contracts; a better solution would make policy select the one allowed final message before any external send.
