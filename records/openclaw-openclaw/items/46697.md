## What This Changes

The branch modifies `src/agents/tools/sessions-send-tool.a2a.ts` to post `latestReply` directly to an announce target after ping-pong, run the announce step in the requester session, suppress later announce delivery after direct send succeeds, and pass `threadId`.

## Best Possible Solution

Keep A2A visible delivery behind one policy-gated final-delivery path after ping-pong that filters control tokens, honors announce suppression, preserves threadId/provenance/per-session lanes, and is backed by focused tests plus redacted real threaded proof.

## Reproduction Assessment

Yes for the PR-introduced defects: PR-head source shows raw direct `latestReply` channel delivery, missing non-deliverable filtering, and shared-lane requester announce behavior. The original after-fix threaded Discord/Cursor scenario still lacks real behavior proof.

## Solution Assessment

No. Direct raw full-reply sends are not the narrowest maintainable fix because they bypass the current announce policy boundary; a policy-gated final delivery decision after ping-pong is safer.
