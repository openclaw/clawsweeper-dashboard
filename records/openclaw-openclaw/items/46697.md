## Summary

Keep open: the PR addresses a plausible A2A thread UX gap, but the proposed patch changes the external delivery contract by sending raw target replies directly to the channel before the existing announce policy can approve or suppress delivery.

## What This Changes

The PR changes `src/agents/tools/sessions-send-tool.a2a.ts` to send `latestReply` directly to the resolved announce target, run the announce step in the requester session, suppress duplicate announce sends, and pass `threadId`.

## Best Possible Solution

Define one approved A2A external-delivery path that chooses final content after ping-pong, applies suppression/control-token/provenance policy once, preserves per-session lanes and `threadId`, and has focused regression tests plus real behavior proof.

## Reproduction Assessment

Yes. Source inspection shows PR head directly sends `latestReply.trim()` to the announce target, while current main and tests route external delivery through the announce reply gate and non-deliverable token checks.

## Solution Assessment

No. Direct raw sends are not the narrowest safe fix because they bypass the existing announce policy boundary; a maintainable fix should make one policy-gated final delivery decision after ping-pong and suppression checks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Honor announce suppression before full-reply delivery:** `src/agents/tools/sessions-send-tool.a2a.ts:104-118`
  - body: The new direct send publishes `latestReply` to the announce target before the announce step can return `ANNOUNCE_SKIP`. Current main only sends externally after the skip and non-deliverable checks, so suppressed content can become visible in a linked thread.
  - confidence: 0.94
- **[P1] Filter control replies before full-reply sends:** `src/agents/tools/sessions-send-tool.a2a.ts:104-110`
  - body: This path sends any non-empty `latestReply` directly to the channel. Current main drops exact control replies such as `NO_REPLY`, `HEARTBEAT_OK`, `ANNOUNCE_SKIP`, and `REPLY_SKIP` before A2A delivery, so the PR can leak control tokens into user-visible chat.
  - confidence: 0.91
- **[P2] Preserve session-scoped nested lanes:** `src/agents/tools/sessions-send-tool.a2a.ts:147`
  - body: The requester announce step uses the shared `AGENT_LANE_NESTED` lane. Current main scopes nested A2A work with `resolveNestedAgentLaneForSession`, so this can reintroduce cross-session blocking that the later lane fix removed.
  - confidence: 0.86
- **[P2] Use target-channel provenance for requester announce:** `src/agents/tools/sessions-send-tool.a2a.ts:148-149`
  - body: The requester announce step sets `sourceSessionKey` to the target session while leaving `sourceChannel` as the requester channel. Cross-channel A2A prompts can then label the foreign target reply as coming from the wrong channel.
  - confidence: 0.8
- **[P3] Skip unused announce runs after full-reply sends:** `src/agents/tools/sessions-send-tool.a2a.ts:141-147`
  - body: When full-reply delivery succeeds, `fullReplyPosted` later prevents the announce result from being sent, but the PR still runs another agent step. That adds latency and records an announce turn that cannot affect output.
  - confidence: 0.76
