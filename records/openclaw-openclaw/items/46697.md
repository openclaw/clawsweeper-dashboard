## Summary

Keep open. The PR targets a real `sessions_send` A2A thread-content gap, but it is not mergeable as-is because it posts the raw final reply to an external announce target before the announce policy can return `ANNOUNCE_SKIP`, and it regresses current-main lane/provenance contracts around nested A2A work.

## What This Changes

The PR modifies `src/agents/tools/sessions-send-tool.a2a.ts` to send `latestReply` directly to the resolved announce target after ping-pong, run the announce step in the requester session, suppress fallback announce delivery after direct delivery, and pass `threadId`.

## Best Possible Solution

Define one final A2A delivery path that preserves announce suppression, target-channel provenance, session-scoped nested lanes, transcript hygiene, tests for full-reply delivery and skip behavior, and a changelog entry.

## Reproduction Assessment

Yes. Use an A2A send with a resolved announce target, a non-empty `latestReply`, and an announce step that returns `ANNOUNCE_SKIP`; current main suppresses the gateway send, while the PR branch sends the raw reply before observing the skip.

## Solution Assessment

No. Posting the final full reply may be the right product direction, but this patch bypasses the existing policy gate and should be revised so the allowed final content is chosen before exactly one external send occurs.

- **[P1] Honor announce suppression before full-reply delivery:** `src/agents/tools/sessions-send-tool.a2a.ts:104-118`
  - body: The new direct send publishes `latestReply` to the announce target before the announce step can return `ANNOUNCE_SKIP`. Current main only sends externally after that skip check, with tests asserting no `send` call, so this can leak content into a visible thread that policy meant to silence.
  - confidence: 0.93

  - body: The requester-side announce run uses the shared `AGENT_LANE_NESTED` lane. Current main scopes nested lanes with `resolveNestedAgentLaneForSession`, so concurrent A2A announces for different sessions can block each other again unless this path keeps the session-scoped lane contract.
  - confidence: 0.82
- **[P2] Use target-channel provenance for requester announce:** `src/agents/tools/sessions-send-tool.a2a.ts:148-149`
  - body: This announce step sets `sourceSessionKey` to the target session but keeps `sourceChannel` as the requester channel. When requester and target channels differ, downstream prompt/provenance text describes the target content as coming from the wrong channel, which can affect routing and formatting decisions.
  - confidence: 0.8

  - body: When the direct full-reply send succeeds, `fullReplyPosted` prevents the announce reply from being delivered, but the code still runs a requester agent step. That adds latency/cost and records a synthetic announce turn without affecting output.
  - confidence: 0.74
