## Summary

Keep this PR open, but do not merge the current patch. The underlying request is still plausible because current main posts the model-written announce reply rather than the target session's full reply, but this implementation bypasses the existing ANNOUNCE_SKIP suppression contract and regresses current-main lane/provenance behavior around the requester-side announce step.

## What This Changes

The PR changes the sessions_send A2A flow to post the target session's latest reply directly to the resolved announce target after ping-pong, run the announce step in the requester session, suppress fallback announce delivery after direct full-reply delivery, and pass threadId on fallback sends.

## Best Possible Solution

Define the A2A delivery contract explicitly: a visible announce target should receive exactly one allowed final post after ping-pong completes, while preserving ANNOUNCE_SKIP suppression, target-channel provenance, per-session nested lanes, transcript hygiene, focused tests, and a changelog entry.

## Reproduction Assessment

Yes. Against the PR diff, use an announce target, a non-empty latestReply, and an announce step that would return ANNOUNCE_SKIP; the direct full-reply send runs before that skip can suppress external delivery.

## Solution Assessment

No. Posting the final full reply may be a reasonable direction, but this patch sends before the announce policy decision, keeps inconsistent provenance, and reintroduces a shared nested lane; the safer solution is to decide suppression/provenance first and then perform one final external send.
