## Summary

Keep open. The PR targets a real A2A thread-delivery complaint, but the branch adds a raw external full-reply send that bypasses current announce suppression/control-token policy, is stale behind later sessions_send hardening, and has no after-fix real behavior proof.

## What This Changes

The PR modifies `src/agents/tools/sessions-send-tool.a2a.ts` to send the target session's full `latestReply` directly to the resolved announce target, run the announce step in the requester session, suppress duplicate announce sends, and pass `threadId`.

## Best Possible Solution

Define one approved A2A external-delivery path that chooses final content after ping-pong, applies suppression/control-token/provenance policy once, preserves per-session lanes and `threadId`, and is backed by focused regression tests plus redacted real behavior proof.

## Reproduction Assessment

Yes for the PR-introduced defects: source inspection of PR head shows direct `latestReply.trim()` channel delivery, while current main gates external delivery through announce suppression and non-deliverable token checks. The original live thread complaint is not proven by real after-fix evidence in the PR.

## Solution Assessment

No. Direct raw sends are not the narrowest maintainable fix because they bypass the existing announce policy boundary; the safer path is a single policy-gated final-delivery decision after ping-pong.
