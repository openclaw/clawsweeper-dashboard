## Summary

Keep this PR open for maintainer review, but it is not safe to merge as-is: the branch tries to fix a real threaded A2A delivery pain point, yet PR head bypasses current announce-suppression and control-token delivery policy, moves announce work onto an unscoped lane, and still lacks after-fix real threaded proof.

## What This Changes

The PR changes `src/agents/tools/sessions-send-tool.a2a.ts` to send the final `latestReply` directly to the announce target, move announce generation into the requester session, suppress duplicate announce sends, and pass `threadId`.

## Best Possible Solution

Keep one policy-gated final A2A delivery path after ping-pong that filters control tokens, honors announce suppression, preserves per-session lanes/provenance/threadId, and has focused tests plus real threaded proof.

## Reproduction Assessment

Yes for the PR-introduced defects: PR-head source shows raw direct `latestReply` channel delivery and shared-lane requester announce behavior, while current main gates external delivery and scopes nested lanes. The original threaded ack complaint still lacks after-fix real setup proof.

## Solution Assessment

No. Direct raw full-reply sends are not the narrowest maintainable solution because they bypass the current announce policy boundary; a policy-gated final delivery decision after ping-pong is safer.
