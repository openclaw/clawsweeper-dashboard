## Summary

Keep open for maintainer review: the PR addresses a plausible threaded A2A annoyance, but the current patch bypasses existing announce-delivery policy and current main has since added related hardening that this branch would need to preserve.

## What This Changes

The PR changes `runSessionsSendA2AFlow` to send `latestReply` directly to the resolved announce target, move announce generation to the requester session, suppress duplicate announce sends, and pass `threadId`.

## Best Possible Solution

Define one policy-gated A2A external-delivery path that chooses final content after ping-pong, filters control tokens, preserves target thread routing plus session-scoped lane/provenance, and ships with focused tests and redacted real thread proof.

## Reproduction Assessment

Yes for the PR-introduced defects: source inspection of PR head shows direct `latestReply.trim()` channel delivery while current main gates external delivery through announce suppression and non-deliverable-token checks. The original threaded-reply complaint still lacks after-fix real setup proof in this PR.

## Solution Assessment

No. Direct raw sends are not the narrowest maintainable solution because they bypass the existing announce policy boundary; a single policy-gated final-delivery decision after ping-pong is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Honor announce suppression before full-reply delivery:** `src/agents/tools/sessions-send-tool.a2a.ts:106-110`
  - body: The new direct send publishes `latestReply` to the announce target before the announce step can return `ANNOUNCE_SKIP`. Current main only sends externally after skip checks, so content intended to remain silent can become visible in the linked thread or channel.
  - confidence: 0.94
- **[P1] Drop control replies before full-reply sends:** `src/agents/tools/sessions-send-tool.a2a.ts:104-110`
  - body: This path sends any non-empty `latestReply` directly to the channel. Because PR head lacks current-main non-deliverable reply guards, `NO_REPLY`, `HEARTBEAT_OK`, `ANNOUNCE_SKIP`, or `REPLY_SKIP` can be posted to user-visible chat instead of being filtered.
  - confidence: 0.91
- **[P2] Preserve session-scoped nested lanes:** `src/agents/tools/sessions-send-tool.a2a.ts:147`
  - body: The moved requester announce step uses the shared nested lane. Current main scopes nested A2A work with `resolveNestedAgentLaneForSession`, so this can reintroduce cross-session blocking for unrelated sessions.
  - confidence: 0.86
- **[P2] Use target-channel provenance for requester announce:** `src/agents/tools/sessions-send-tool.a2a.ts:148-149`
  - body: This announce step sets `sourceSessionKey` to the target session but keeps `sourceChannel` as the requester channel. Cross-channel A2A prompts can label the foreign target reply as coming from the wrong channel.
  - confidence: 0.8
- **[P3] Skip unused announce runs after full-reply sends:** `src/agents/tools/sessions-send-tool.a2a.ts:141-147`
  - body: When full-reply delivery succeeds, `fullReplyPosted` prevents the announce result from being sent, but the PR still runs another agent step. That adds latency and records an announce turn that cannot affect output.
  - confidence: 0.76
