## What This Changes

The PR changes `runSessionsSendA2AFlow` to send `latestReply` directly to an announce target after ping-pong, run the announce step in the requester session, and skip announce delivery when the direct full-reply send succeeds.

## Best Possible Solution

Reimplement the useful full-reply behavior on current main as a small A2A delivery fix that preserves final post-ping-pong content without bypassing `ANNOUNCE_SKIP`, non-deliverable filtering, provenance, or current delayed-reply safeguards, then require redacted real transport proof.

## Reproduction Assessment
