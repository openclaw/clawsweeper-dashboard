## Summary

Keep open for human review: the branch targets a real A2A thread-delivery complaint, but its raw full-reply delivery bypasses current announce/control-token policy and still lacks after-fix real behavior proof.

## What This Changes

The PR changes `src/agents/tools/sessions-send-tool.a2a.ts` to send the target session's final `latestReply` directly to the resolved announce target, run the announce step in the requester session, suppress duplicate announce sends, and pass `threadId`.

## Best Possible Solution

Define one policy-gated A2A external-delivery path that selects final content after ping-pong, preserves session-scoped lanes/provenance/threadId, filters control tokens, and is covered by focused regression tests plus redacted live/thread proof.

## Reproduction Assessment

Yes for the PR-introduced defects: source inspection of PR head shows direct `latestReply.trim()` channel delivery while current main gates external delivery through announce suppression and non-deliverable token checks. The original live thread complaint still lacks after-fix real setup proof in this PR.

## Solution Assessment

No. Direct raw sends are not the narrowest maintainable fix because they bypass the existing announce policy boundary; a single policy-gated final-delivery decision after ping-pong is safer.

Full review comments:

- **[P1] Honor announce suppression before full-reply delivery:** `src/agents/tools/sessions-send-tool.a2a.ts:104-110`
  - body: The new direct send publishes `latestReply` to the announce target before the announce step can return `ANNOUNCE_SKIP`. Current main only sends externally after skip checks, so content that should remain silent can become visible in the linked thread.
  - confidence: 0.94
- **[P1] Filter control replies before full-reply sends:** `src/agents/tools/sessions-send-tool.a2a.ts:104-110`
  - body: This path sends any non-empty `latestReply` directly to the channel. If the target or ping-pong loop returns `NO_REPLY`, `HEARTBEAT_OK`, `ANNOUNCE_SKIP`, or `REPLY_SKIP`, the token can be posted to user-visible chat instead of being dropped.
  - confidence: 0.91
- **[P2] Preserve session-scoped nested lanes:** `src/agents/tools/sessions-send-tool.a2a.ts:147`
  - body: The requester announce step uses the shared nested lane. Current main scopes nested A2A work with `resolveNestedAgentLaneForSession`, so this can reintroduce cross-session blocking for unrelated sessions.
  - confidence: 0.86
- **[P2] Use target-channel provenance for requester announce:** `src/agents/tools/sessions-send-tool.a2a.ts:148-149`
  - body: The requester announce step sets `sourceSessionKey` to the target session while keeping `sourceChannel` as the requester channel. Cross-channel A2A prompts can label the foreign target reply as coming from the wrong channel.
  - confidence: 0.8
- **[P3] Skip unused announce runs after full-reply sends:** `src/agents/tools/sessions-send-tool.a2a.ts:141-147`
  - body: When full-reply delivery succeeds, `fullReplyPosted` prevents the announce result from being sent, but the PR still runs another agent step. That adds latency and records an announce turn that cannot affect output.
  - confidence: 0.76
