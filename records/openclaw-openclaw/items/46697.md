## Summary

## What This Changes

The PR changes `src/agents/tools/sessions-send-tool.a2a.ts` to post `latestReply` directly to an announce target after ping-pong, move the announce step into the requester session, suppress the announce send after direct delivery succeeds, and pass `threadId`.

## Best Possible Solution

Keep any full-reply behavior behind one final policy-gated A2A delivery path after ping-pong that filters control tokens, honors announce suppression, preserves threadId/provenance/per-session lanes, and is backed by focused tests plus redacted real threaded proof.
