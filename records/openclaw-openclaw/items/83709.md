## Summary

Keep open: current main and the latest release still omit Mistral's prompt-cache-key compat opt-in, while the shared OpenAI-compatible completions transport only sends `prompt_cache_key` when `compat.supportsPromptCacheKey === true`; the issue is clear enough for a focused provider fix.

## What This Changes

Request to enable Mistral chat-completions prompt caching by adding `supportsPromptCacheKey` to its provider transport compat and aligning coverage/accounting.

## Best Possible Solution

Add Mistral-owned `supportsPromptCacheKey: true`, update focused provider/transport tests, and align Mistral cache-read pricing/docs with the upstream cached-token billing contract.

## Reproduction Assessment

Yes, from source inspection rather than a live request: Mistral's provider compat omits the explicit flag, and the transport gate requires that flag before it sends `prompt_cache_key`. The upstream Mistral docs confirm the request and usage fields exist.

## Solution Assessment

Yes, the maintainable fix is at the provider-owned compat boundary with focused tests; only adding the flag without Mistral cache-read pricing coverage would be incomplete.
