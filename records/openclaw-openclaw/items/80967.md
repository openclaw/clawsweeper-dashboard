## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep profile suffixes behind profile trust:** `src/plugins/runtime/runtime-llm.runtime.ts:611-623`
  - body: The new `allowProfileOverride` key is described as gating auth-profile steering, but this branch treats a `provider/model@profile` suffix as covered when the caller only has model-override trust. That lets a plugin trusted to choose a model also choose a stored credential profile/account, so require `allowProfileOverride` for suffixes unless maintainers explicitly accept that broader boundary.
  - confidence: 0.86
- **[P2] Regenerate the config docs baseline:** `src/config/zod-schema.ts:220`
  - body: Adding `plugins.entries.<id>.llm.allowProfileOverride` changes the public config schema, but the PR does not update `docs/.generated/config-baseline.sha256`. The config baseline guard says intentional config-surface changes must run `pnpm config:docs:gen` and commit the updated hash, so config docs checks or release preflight can fail until this is refreshed.
  - confidence: 0.91

## Security Review
