## Summary

Keep open. Current main still exposes terminal-looking `status` values through persisted session entries, gateway session rows, and the agent-facing `sessions_list` output without explicit resumability or last-turn status-scope metadata; the docs and default prompt still do not give the requested orchestration guidance.

## What This Changes

The issue asks OpenClaw to clarify that session `status` values like `failed`, `timeout`, and `done` describe the last run or turn, not whether the session can be resumed with `sessions_send`.

## Best Possible Solution

Add backward-compatible derived metadata to session list rows and agent tool output, then document and prompt that `status` is a last-run/turn signal while preserving existing status values for clients.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to return or persist a session row with `status: "failed"`, `"timeout"`, or `"done"`; current main forwards that bare value through gateway and `sessions_list` without a resumability or status-scope field.

## Solution Assessment

Yes. An additive contract plus docs and prompt guidance is the narrowest maintainable fix; renaming the existing status values would be clearer conceptually but is a breaking API/storage change.
