## Label Justifications

- `P2`: The issue describes a real but bounded session-orchestration behavior that can waste model calls and context without making core runtime unusable.
- `impact:session-state`: The reported failure mode causes agents to abandon resumable sessions and lose continuity with accumulated session context.

## Summary

Keep open. Current main still exposes terminal-looking session statuses through persistence, gateway rows, and agent-facing `sessions_list` without explicit resumability/status-scope metadata, while `sessions.send` can still target and reactivate completed subagent sessions.

## What This Changes

The issue asks OpenClaw to make session status semantics clear enough that agents resume existing sessions instead of spawning duplicates after `failed`, `timeout`, or `done` last-turn states.

## Best Possible Solution

Preserve the legacy `status` union for compatibility, add explicit agent-visible resumability/status-scope or targetability metadata, and update prompts, docs, and regression tests around that contract.

## Reproduction Assessment

Yes, at source level. Current `sessions_list` can still return `done`, `failed`, or `timeout` without resumability context, while `sessions.send` can send to an existing session key and reactivate completed subagent sessions.

## Solution Assessment

No for the proposed breaking rename. The maintainable fix is additive metadata and guidance that preserves the existing status union while making resumability explicit for agents and clients.
