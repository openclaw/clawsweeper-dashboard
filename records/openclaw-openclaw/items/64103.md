## Summary

Keep open: current main and the latest release still expose ambiguous last-turn session statuses without a `resumable` signal or model-facing status semantics, and the open linked PR is the active candidate fix but has not merged.

## What This Changes

Session rows should avoid misleading agents that last-turn statuses such as `failed`, `timeout`, and `done` mean a session cannot be resumed.

## Best Possible Solution

Review and land, rebase, or replace the open linked PR with a backward-compatible resumability/status-semantics fix that preserves existing status values while making resume behavior explicit to agents.

## Reproduction Assessment

Yes from source inspection, though not from a live run in this read-only review: current main exposes last-turn `status` values without a `resumable` field or model-facing semantics guidance, while `sessions_send` can reactivate ended subagent sessions.

## Solution Assessment

The additive `resumable` field plus prompt/tool guidance is a safer fix than renaming shipped status values, but maintainers still need to confirm the exact session-state contract before the open PR lands.
