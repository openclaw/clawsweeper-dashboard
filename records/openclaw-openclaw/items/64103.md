## Summary

Keep open: current main and v2026.6.10 still expose terminal-looking session `status` values without a `resumable` signal or model-facing status semantics, and the open linked PR is the active but unmerged fix path.

## What This Changes

The issue asks OpenClaw to make session-list status semantics unambiguous so agents resume existing sessions instead of spawning duplicate sub-agent sessions.

## Best Possible Solution

Refresh and review, land, or replace the open linked PR with a backward-compatible resumability/status-semantics fix that preserves existing status values unless maintainers explicitly approve a breaking rename.

## Reproduction Assessment

Yes from source inspection, though not from a live run in this read-only review: current main exposes terminal-looking statuses without a resume signal, while `sessions_send` can reactivate ended subagent sessions.

## Solution Assessment

Yes, conditionally: additive `resumable` metadata plus tool/prompt guidance is safer than renaming shipped status values, but maintainers still need to confirm the exact session-state contract before the linked PR lands.
