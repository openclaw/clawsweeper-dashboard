## Summary

Keep open: current main and the latest release still expose terminal-looking session `status` values without a resumability signal or model-facing status semantics, and https://github.com/openclaw/openclaw/pull/89101 is the open closing PR that currently owns the fix path.

## What This Changes

The issue asks OpenClaw to clarify session status semantics so orchestrator agents resume existing sub-agent sessions instead of spawning duplicates after last-turn `done`, `failed`, or `timeout` statuses.

## Best Possible Solution

Review, refresh, and land or replace https://github.com/openclaw/openclaw/pull/89101 with a backward-compatible resumability/status-semantics fix, preserving existing status values unless maintainers explicitly approve a breaking rename.

## Reproduction Assessment

Yes, from source inspection: current main exposes terminal-looking statuses without a resume signal, while `sessions_send` can reactivate ended sub-agent sessions. I did not run a live orchestrator reproduction in this read-only review.

## Solution Assessment

Yes, conditionally: additive `resumable` metadata plus tool/prompt guidance is safer than renaming shipped status values, but maintainers still need to confirm the session-state contract before the linked PR lands.
