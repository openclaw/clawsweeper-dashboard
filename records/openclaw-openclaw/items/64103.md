## Label Justifications

- `P2`: The issue describes a bounded session-state bug that can cause duplicate subagent sessions and paid-token waste without evidence of an emergency runtime outage.
- `impact:session-state`: The reported failure mode is agents misreading session state, abandoning resumable context, and spawning duplicate sessions.

## Summary

Keep open: current main still exposes terminal-looking session `status` values without an explicit resumability signal, and the open linked fix at https://github.com/openclaw/openclaw/pull/89101 has not merged yet.

## What This Changes

The issue requests clearer session-list status semantics so agents do not treat last-turn states as terminal and spawn duplicate subagent sessions.

## Best Possible Solution

Land a maintainer-approved additive session-list contract, likely `resumable` metadata plus model-facing guidance, through the linked PR or a refreshed equivalent.

## Reproduction Assessment

Yes, at source-contract level: current main maps ended run outcomes to `done`, `failed`, `timeout`, and `killed` while the reactivation path accepts ended subagent runs. I did not run the full orchestrator duplicate-spawn live scenario.

## Solution Assessment

Yes, if maintainers accept the contract: additive resumability metadata plus guidance is a narrower compatible fix than renaming shipped status values. The linked PR should be reviewed and refreshed rather than starting a second fix path.
