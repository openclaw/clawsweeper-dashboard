## Summary

Keep this issue open: current main and the latest release still expose terminal-looking session `status` values without a resumability signal, and https://github.com/openclaw/openclaw/pull/89101 is the active linked fix path with closing syntax.

## What This Changes

The issue asks OpenClaw to clarify that session `status` values describe the last communication turn, not permanent session lifecycle, so agents resume via `sessions_send` instead of spawning duplicates.

## Best Possible Solution

Land or explicitly replace the linked additive `resumable` PR, then close this issue once the fix reaches main.

## Reproduction Assessment

We have a high-confidence source-level reproduction, not a deterministic live orchestrator transcript: current main exposes terminal-looking `status` values without resumability metadata while the runtime can reactivate ended subagent runs.

## Solution Assessment
