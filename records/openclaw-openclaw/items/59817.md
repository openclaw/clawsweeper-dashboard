## Summary

Keep open. Current main still has the reported degraded Discord DM path: inbound preflight only marks a message as a DM when channel-info fetch returns ChannelType.DM, lookup failures return null, and routing then builds a channel peer/session key. The affected files are unchanged since the prior ClawSweeper review that identified the same gap.

## What This Changes

Discord inbound DM routing should keep one stable direct session key when channel-info lookup fails during network or API degradation.

## Best Possible Solution

Add a narrow Discord-owned preflight classification helper that prefers fetched channelInfo when available, then falls back to trusted one-to-one DM payload signals during channel-info failures. The fallback should route degraded DMs by author id and preserve existing guild, group-DM, thread, and binding behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an inbound Discord MESSAGE_CREATE payload with no guild_id and a normal user author while client.fetchChannel returns null or throws; current preflight sets isDirectMessage false and routes through the channel path.

## Solution Assessment

Yes. The best fix is a narrow preflight classification change in the Discord plugin; more aggressive session-key normalization alone is insufficient because the bad path marks ChatType as channel before the normalizer can canonicalize direct keys.
