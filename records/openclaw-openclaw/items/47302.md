## Summary

Keep open: the idea is useful, but this branch is not merge-ready because current main already has `doctor --lint` as the structured automation surface, the branch is currently conflicting, and source inspection still shows false-clean validation paths for unresolved env refs, malformed primary models, and fallback refs.

## What This Changes

The PR adds a separate `openclaw doctor --check-config` flag plus a new deep config validation module and formatter-only tests.

## Best Possible Solution

Move the deep config checks into the doctor health-check registry under `doctor --lint`, using shared JSON/severity/skip/only behavior and tests for env, primary model, fallback, TTS, and channel cases.

## Reproduction Assessment

Yes, for the review findings by source inspection: unresolved `${ENV}` warnings, malformed `agents.defaults.model` values such as `openai/`, and parseable unknown fallbacks each have clear paths through the new code. I did not execute the CLI in this read-only review.

## Solution Assessment

No. The maintainable path is to port these diagnostics into `doctor --lint` rather than adding a parallel flag, then fix the false-clean validation cases before merge.

- **[P1] Use the existing doctor lint health-check path:** `src/cli/program/register.maintenance.ts:29-32`
  - body: Current main already exposes `openclaw doctor --lint` as the read-only CI/preflight surface with JSON output, severity thresholds, and `--only`/`--skip`. Adding `--check-config` here creates a competing automation contract and this branch is already conflicting with that path, so the deep checks should be registered as health checks under `doctor --lint` instead.
  - confidence: 0.91
- **[P1] Fail unresolved env substitutions:** `src/commands/doctor-check-config.ts:72`
  - body: `readConfigFileSnapshot` preserves missing `${ENV}` placeholders and reports them in `snapshot.warnings`, but this code records those as `warn` and `hasFailures` ignores warnings. A config with an unresolved credential placeholder can still exit 0, which misses the silent degradation this command is meant to catch.
  - confidence: 0.86
- **[P2] Reject malformed configured primary refs:** `src/commands/doctor-check-config.ts:153-154`
  - body: When a configured primary model includes `/` but cannot be parsed, such as `openai/`, `resolveConfiguredModelRef` can fall back before this check runs. Because the mismatch branch only fires when `parseModelRef` returns a parsed value, the malformed configured ref falls through to the final OK result.
  - confidence: 0.84
- **[P2] Validate fallback refs against runtime status:** `src/commands/doctor-check-config.ts:279-280`
  - body: `resolveModelRefFromString` only proves the fallback string can be parsed or resolved as an alias; it does not prove the provider/model is in the configured/catalog runtime set. That means an unknown but syntactically valid fallback can be reported as validated, which misses the provider-definition failure this PR says it closes.
  - confidence: 0.88
