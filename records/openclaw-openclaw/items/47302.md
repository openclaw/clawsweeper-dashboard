## Summary

Keep open: this is a useful feature PR, but the latest head still does not fail closed for several broken configured states that the new command is meant to catch.

## What This Changes

The branch adds an `openclaw doctor --check-config` option, a new deep config diagnostic module for schema/model/fallback/TTS/channel checks, and formatter tests.

## Best Possible Solution

Land a revised flag that fails missing-env and malformed model inputs, validates fallback refs through runtime model status without requiring explicit `models.providers`, adds behavior coverage, updates docs, and includes redacted CLI proof for passing and failing configs.

## Reproduction Assessment

Yes for the review blockers by source inspection: missing env placeholders, malformed primary refs, and syntactic-only fallback refs follow clear paths to warning or OK output. I did not execute those configs in this read-only review.

## Solution Assessment

No. The feature direction is useful, but the current implementation is not the best mergeable form because it does not fail closed for broken configured values that the command is intended to catch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Fail missing env substitutions:** `src/commands/doctor-check-config.ts:72-73`
  - body: `readConfigFileSnapshot` preserves missing `${VAR}` placeholders as warnings, but this branch records those as `warn` and `hasFailures` only checks `fail`. For a configured TTS key such as `${OPENAI_API_KEY}`, the provider can still see a non-empty placeholder and the command can exit 0 while the feature is unavailable.
  - confidence: 0.9
- **[P2] Fail malformed primary model refs:** `src/commands/doctor-check-config.ts:153-154`
  - body: When a configured primary contains a slash but is malformed, such as `anthropic/`, `parseModelRef` returns null and this condition skips the warning. Since `resolveConfiguredModelRef` already fell back to the default, the command can report `Primary model` as OK for an invalid configured value.
  - confidence: 0.9
- **[P2] Validate fallback refs against runtime status:** `src/commands/doctor-check-config.ts:279-280`
  - body: `resolveModelRefFromString` only proves that the fallback string can be parsed or aliased, so a syntactic ref like `missing-provider/k2p5` is counted in the validated total without proving the fallback can run. That misses broken fallback chains this diagnostic is supposed to surface.
  - confidence: 0.84
