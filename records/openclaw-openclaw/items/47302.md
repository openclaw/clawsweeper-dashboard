## Summary

Keep open: current main does not contain this feature, and the latest PR head still has source-reproducible fail-closed gaps that would let broken config pass the new CI-style diagnostic.

## What This Changes

The PR adds `openclaw doctor --check-config`, a new doctor CLI flag and diagnostic module with formatter tests for schema, model, fallback, TTS, and channel validation.

## Best Possible Solution

Land a revised implementation that fails closed for broken configured model/fallback/TTS inputs, adds subsystem and CLI coverage, and includes redacted real CLI proof for configured success and intentional failure cases.

## Reproduction Assessment

Yes. The blockers are source-reproducible with configs using an unset `${OPENAI_API_KEY}`, `agents.defaults.model: "anthropic/"`, and a fallback like `missing-provider/k2p5`; this read-only review did not execute those configs.

## Solution Assessment

No. The feature direction is useful, but this implementation still reports some broken config states as warning or OK, so it is not yet the narrowest maintainable solution for CI-gated deep validation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Fail unresolved env substitutions:** `src/commands/doctor-check-config.ts:68-73`
  - body: `readConfigFileSnapshot` preserves missing `${VAR}` placeholders as warnings, but this path emits them as `warn` and only exits non-zero for `fail`. With a missing TTS API-key env var, the placeholder remains a non-empty string, so the provider can still look configured while `--check-config` exits 0.
  - confidence: 0.9
- **[P2] Fail malformed configured primary refs:** `src/commands/doctor-check-config.ts:153-154`
  - body: For a malformed primary such as `anthropic/`, `resolveConfiguredModelRef` falls back to the default model, then `parseModelRef(rawConfigured, DEFAULT_PROVIDER)` returns null and skips the mismatch warning. The command can report the fallback default as the OK primary even though the configured value is invalid.
  - confidence: 0.9
- **[P2] Validate fallback refs against availability:** `src/commands/doctor-check-config.ts:279-280`
  - body: `resolveModelRefFromString` only parses and resolves aliases; it accepts a syntactic value like `missing-provider/k2p5`. The loop then counts that fallback as validated even though the command has not shown the provider/model is available under the current runtime contract.
  - confidence: 0.84
