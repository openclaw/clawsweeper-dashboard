## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Use the current TTS provider-order API:** `src/commands/doctor-check-config.ts:18`
  - body: The added module imports `TTS_PROVIDERS` from `../tts/tts.js`, but current main's TTS facade no longer exports that symbol. After rebasing, this will fail typecheck before `doctor --check-config` can run; use the config-aware `resolveTtsProviderOrder(ttsConfig.provider, cfg)` or provider registry path with `isTtsProviderConfigured(..., cfg)` instead.
  - confidence: 0.96
- **[P2] Fail invalid primary refs before fallback resolution masks them:** `src/commands/doctor-check-config.ts:152-154`
  - body: `resolveConfiguredModelRef` falls back to the default model when a slash-qualified primary ref cannot be resolved, but this check only warns when `parseModelRef(rawConfigured, ...)` returns a different parsed ref. If the user configures a malformed ref like `anthropic/`, `parseModelRef` returns null and the code falls through to a green `Primary model` result for the fallback default, missing the broken config this command is meant to catch.
  - confidence: 0.89
