Full review comments:

- **[P1] Use the exported TTS provider order helper:** `src/commands/doctor-check-config.ts:18`
  - body: `../tts/tts.js` does not export `TTS_PROVIDERS` on current main, so this new module will fail typecheck/build when included. Use the current registry-backed `resolveTtsProviderOrder` API to enumerate fallback providers.
  - confidence: 0.98
- **[P2] Fail unresolved active config inputs:** `src/commands/doctor-check-config.ts:72`
  - body: Snapshot warnings for missing `${ENV_VAR}` substitutions are emitted as warnings, while `hasFailures` only counts `fail`. An unresolved API key used by an active provider can therefore still produce exit 0, missing the broken config this CI-gating mode is meant to catch.
  - confidence: 0.9
- **[P2] Load the model catalog in read-only mode:** `src/commands/doctor-check-config.ts:183`
  - body: `loadModelCatalog({ config: cfg })` takes the writable path on current main and can call `ensureOpenClawModelsJson`. A doctor validation mode should pass `readOnly: true` so checking config does not mutate user state.
  - confidence: 0.93
- **[P2] Reject malformed primary refs before fallback:** `src/commands/doctor-check-config.ts:153-154`
  - body: When `agents.defaults.model` is a malformed qualified ref, `resolveConfiguredModelRef` can fall back to the default model. The check can then report the fallback as healthy instead of flagging the broken explicit primary model.
  - confidence: 0.88
- **[P2] Validate fallback refs against catalog availability:** `src/commands/doctor-check-config.ts:279`
  - body: `resolveModelRefFromString` proves a fallback string can be parsed, but not that the provider/model is actually available in the runtime catalog. A missing provider can still be counted as validated, leaving the fallback-validation request incomplete.
  - confidence: 0.88
- **[P3] Document the new doctor flag and changelog entry:** `src/cli/program/register.maintenance.ts:28-32`
  - body: This user-facing feature adds a new CLI option, but the PR does not update the doctor docs or `CHANGELOG.md`. Add both so the released CLI surface and release notes match the new behavior.
  - confidence: 0.84
