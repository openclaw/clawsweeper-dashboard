## Summary

Keep open: current main still lacks the proposed deep config doctor flag, but this PR is not mergeable as-is because it relies on a removed TTS provider export, still lets unresolved env-backed TTS secrets exit 0, and has no real after-fix behavior proof.

## What This Changes

The PR adds `openclaw doctor --check-config` with schema, model, fallback, TTS, and channel config validation plus formatter tests.

## Best Possible Solution

Keep the PR open, rebase it onto current main, use the current speech provider APIs, make unresolved env-backed provider failures affect exit status, add focused subsystem tests, and provide redacted real CLI proof.

## Reproduction Assessment

Yes for the PR defects by source inspection. Current main does not export `TTS_PROVIDERS`, and the missing-env path is traceable from config snapshot warnings through OpenAI TTS's truthy apiKey check to the PR's warn-only result handling.

## Solution Assessment

No. The feature direction is plausible, but this implementation should first align with the current TTS provider registry and make broken env-backed provider config fail the new CI-oriented command.
