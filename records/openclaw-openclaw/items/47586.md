## Summary

Keep open. Current main still matches memory-retention replies as reminder commitments, and the issue is already paired with open PR #47589 using closing syntax, so it should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to stop appending the unscheduled-reminder note for memory-retention wording like "I'll remember that" and to tighten cron suppression around reminders created in the current turn.

## Best Possible Solution

Land PR #47589 or an equivalent focused fix that removes bare `remember` from the commitment matcher, preserves explicit reminder intent such as `remember to ...`, and adds direct guard tests for memory-retention phrases. Treat the deeper cron-suppression behavior as a scoped follow-up unless maintainers intentionally fold it into the same change.

## Reproduction Assessment

Yes. The checked-in regex matches the reported memory phrases, and the reply pipeline appends the note when `successfulCronAdds` is zero and no same-session cron suppresses it.

## Solution Assessment

Yes for the memory false-positive portion: narrowing the matcher and adding focused tests is the narrowest maintainable fix. The cron-suppression portion is related but broader and should be scoped against #43292/#52972 during maintainer review.
