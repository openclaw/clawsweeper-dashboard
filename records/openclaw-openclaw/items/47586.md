## Summary

Keep this issue open: current main still source-reproduces the plain-memory false positive, and the open linked PRs are fix candidates rather than merged implementation evidence.

## What This Changes

The issue asks OpenClaw to stop treating plain memory-retention phrases like "I'll remember your preference" as unscheduled reminder commitments, while preserving real temporal reminder detection.

## Best Possible Solution

Land or replace one focused reminder-guard fix that removes the bare-memory match, preserves temporal forms such as `remember to`, and adds positive and negative guard coverage; keep cron scheduling/suppression work tracked on the related cron-state issues.

## Reproduction Assessment

Yes at source level: the current regex still matches plain memory-retention statements such as `I'll remember your preference`, and the caller appends the note when no cron signal suppresses it. I did not run a live channel scenario in this read-only review.

## Solution Assessment

Yes for the central false-positive bug: the narrowest maintainable fix is in `agent-runner-reminder-guard.ts` plus focused regression tests. The cron suppression scoping described in the body is related but has distinct state-tracking reports and should not block the regex fix.
