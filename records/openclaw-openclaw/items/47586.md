## Summary

Current main and the latest release still reproduce the false positive because bare `remember` remains in the active commitment matcher. The same-author linked fix PR is open and unmerged, so this issue should remain open until that implementation is landed, replaced, or closed; the broader cron-state concerns have narrower related reports.

## What This Changes

Prevent memory-retention statements such as “I’ll remember your preference” from triggering the unscheduled-reminder disclaimer.

## Best Possible Solution

Land the focused matcher correction from https://github.com/openclaw/openclaw/pull/47589, then address compaction, isolated-session matching, and current-turn cron evidence through the narrower related reminder-state issues.

## Reproduction Assessment

Yes. On current main, `hasUnbackedReminderCommitment("I'll remember your preference")` matches the bare-`remember` regex, and the caller appends the disclaimer whenever its cron suppression conditions are false.

## Solution Assessment

Yes for the linked PR's narrow matcher change, which directly removes this false-positive class while preserving “remember to” commitments. The broader cron-state changes should not be bundled without resolving the separate compaction and isolated-session invariants.
