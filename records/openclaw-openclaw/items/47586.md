## Summary

Keep open. Current main still treats bare memory-retention phrases such as "I'll remember that" as reminder commitments, and this issue is paired with open PR #47589 using closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to stop appending the unscheduled-reminder note for memory-retention wording like "I'll remember that" and to make cron suppression reflect reminders created in the current turn.

## Best Possible Solution

Land PR #47589 or an equivalent focused fix that removes bare `remember` from the commitment matcher, preserves explicit reminder intent such as `remember to ...`, and adds direct guard coverage for memory-retention phrases. Track the broader cron-suppression behavior in the existing cron-note issues unless maintainers intentionally fold it into the same change.

## Reproduction Assessment

Yes. The current regex directly matches the reported memory phrases, and the reply path appends the note when `successfulCronAdds` is zero and no same-session cron suppresses it.

## Solution Assessment

Yes for the memory false-positive portion: narrowing the matcher and adding focused tests is the smallest maintainable fix. The cron-suppression portion is related but broader, so the safer path is to scope it against #43292/#52972 during maintainer review.
