## Summary

Keep open: current main still treats bare memory-retention language like "I'll remember your preference" as an unbacked reminder commitment, and the issue is already paired with an open linked PR that targets the matcher and test coverage. The broader cron-suppression complaint remains real but overlaps separate open reminder-note issues, so this issue should stay open until the linked PR lands or maintainers choose a different canonical fix.

## What This Changes

The issue asks OpenClaw to stop appending the unscheduled-reminder note for memory/knowledge-retention phrases and to scope cron suppression to reminders created in the current turn.

## Best Possible Solution

Review and land a narrow matcher/test fix for memory-retention phrases, then handle current-turn cron suppression through the existing canonical reminder-note issues if maintainers want that broader behavior changed.

## Reproduction Assessment

Yes, by source inspection: current main still matches `I'll remember` through the broad reminder commitment regex, and the guard appends the note when no successful cron add or covered cron exists. I did not run tests because this review is read-only, but the code path is direct.

## Solution Assessment

Yes for the central false positive: narrowing bare `remember` and adding focused tests is the maintainable fix. The cron-suppression part should be handled as a separate scoped reminder-state fix rather than folded into a duplicate repair job for this issue.
