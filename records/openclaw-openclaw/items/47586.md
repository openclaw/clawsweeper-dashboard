## Summary

Keep open: current main and the latest release still include the bare `remember` reminder-guard matcher, and an open linked PR explicitly targets this issue but is not merged.

## What This Changes

The issue asks OpenClaw to stop appending the unscheduled-reminder disclaimer for memory-retention phrases like “I'll remember that” and to avoid blunt cron suppression behavior.

## Best Possible Solution

Land or replace the linked reminder-guard matcher fix, then track any remaining cron-suppression behavior in the narrower scheduling reports.

## Reproduction Assessment

Yes at source level: `I'll remember your preference` matches the current bare `remember` regex, and the reply path appends the guard note when no cron add or covered session cron suppresses it.

## Solution Assessment

Yes for the central bug: narrowing `remember` matching while preserving reminder-intent phrases is the narrow maintainable fix; current-turn cron scoping is related but broader and overlaps other open scheduling reports.
