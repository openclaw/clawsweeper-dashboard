## Summary

Keep open: current main and v2026.6.8 still match bare `remember` in the reminder-commitment regex, so plain memory statements can still receive the unscheduled-reminder note. The focused fix path is already represented by https://github.com/openclaw/openclaw/pull/93862, while https://github.com/openclaw/openclaw/pull/47589 is an older conflicting attempt, so this issue should stay open until a linked PR lands.

## What This Changes

The issue asks OpenClaw to stop treating plain memory-retention phrases such as “I'll remember that” as unscheduled reminder commitments while preserving real reminder phrases.

## Best Possible Solution

Land one focused reminder-guard fix that removes bare memory promises from the commitment matcher, keeps temporal phrases such as “remember to” covered, and leaves the deeper cron-suppression work on the existing cron-note trackers.

## Reproduction Assessment

Yes, at source level: the current regex matches bare “I'll remember ...” text, and the reply pipeline appends the note when no cron add is recorded. I did not run a live channel or agent turn in this read-only review.

## Solution Assessment

Yes on direction: the narrow maintainable fix is to change the matcher and add regression coverage for memory statements versus temporal reminder commitments. The current focused linked PR is the right shape to review before creating any duplicate repair work.
