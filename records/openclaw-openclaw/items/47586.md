## Summary

Keep open: current main and the latest release still contain the broad bare `remember` matcher, and open linked PRs already own the proposed fix path, so this issue should stay paired with PR review rather than be cleaned up.

## What This Changes

The issue asks OpenClaw to stop appending the unscheduled-reminder note when an agent says plain memory-retention phrases like `I'll remember that`.

## Best Possible Solution

Review and land one focused matcher fix with regression tests for plain memory statements, then leave the cron-state suppression problems to their narrower linked items.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: on current main, `hasUnbackedReminderCommitment` still matches bare `remember`, and the reply runner appends the guard note when no cron coverage suppresses the match. I did not run a live agent turn in this read-only review.

## Solution Assessment

Yes for the central issue: a narrow matcher change plus positive and negative regression tests is the best maintainable fix for memory false positives. The separate cron-suppression concern should remain in the related cron-state issues and PRs rather than broadening this exact fix.
