## Summary

Keep open: current main and the latest release still treat bare memory-retention `remember` replies as reminder commitments, and this issue has an open closing PR that should be reviewed or replaced before the issue closes.

## What This Changes

The issue asks OpenClaw to stop appending the unscheduled-reminder disclaimer to memory-retention replies like "I'll remember that" while leaving true temporal reminder commitments covered.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/47589 with a current-main-compatible matcher narrowing and regression tests, while leaving broader cron-suppression fixes to the separate open reminder-guard issues.

## Reproduction Assessment

Yes. Current main is source-reproducible: the matcher still includes bare `remember`, and the reply pipeline appends the note when no cron add or same-session cron coverage suppresses it.

## Solution Assessment

Yes for the central matcher bug. Removing bare `remember` while preserving explicit reminder phrasing such as `remember to` is the narrow maintainable fix; broader cron behavior should remain in the related cron trackers.
