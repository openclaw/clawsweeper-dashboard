## Summary

Keep this open: current main and the latest release still classify bare `I'll remember ...` memory statements as reminder commitments, and the open linked fix PR has not merged.

## What This Changes

The issue asks OpenClaw to stop appending the unscheduled-reminder note for memory-retention replies such as `I'll remember your preference` while preserving real reminder warnings.

## Best Possible Solution

Refresh and land the linked matcher fix with focused tests and real reply-path proof, while leaving broader cron-created-this-turn behavior to the separate reminder-guard issues.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, source-level: the current matcher returns true for memory-retention strings such as `I'll remember your preference`, and the reply pipeline appends the note on that predicate when cron suppression does not apply.

## Solution Assessment

Is this the best way to solve the issue? Yes for the central matcher bug: removing bare `remember` while keeping explicit reminder phrases such as `remember to` is the narrow maintainable direction; broader cron suppression should stay in related reminder-guard work.
