## What This Changes

The issue asks for the reminder guard to stop treating memory/knowledge “I’ll remember” statements as unscheduled reminder promises, and also asks for tighter cron suppression scoping.

## Best Possible Solution

Land or replace the focused reminder-guard fix that stops bare memory promises from triggering the note, while keeping scheduled-cron suppression fixes tracked in the related cron-specific issues.

## Reproduction Assessment

Yes from source and a focused regex probe: current main’s pattern matches `I'll remember your preference`, and the reply path appends the note when no cron add or existing session cron covers it. I did not run the full agent flow because this was a read-only review.

## Solution Assessment

Yes for the central bug: narrowing the reminder-guard matcher in `agent-runner-reminder-guard.ts` is the right layer for plain memory false positives. The broader cron-suppression behavior should be handled through the related scheduled-cron issues unless maintainers intentionally combine the work.
