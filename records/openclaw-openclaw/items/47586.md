## What This Changes

The issue asks OpenClaw to narrow the reminder guard so memory-retention statements do not trigger the unscheduled-reminder note, while preserving real temporal reminder detection.

## Best Possible Solution

Land or replace the focused linked PR that narrows plain memory promises out of the matcher, while keeping cron-success and compaction/session fixes on their separate related issue tracks.

## Reproduction Assessment

Yes, at source level: current main's regex matches bare `remember`, and the reply pipeline appends the note when that match has zero successful cron adds. I did not run a live agent turn because this was a read-only review.

## Solution Assessment

Yes on direction: the narrowest maintainable fix is to adjust the existing reminder-guard matcher and add focused guard tests, as the open linked PR attempts. The separate cron suppression issue should stay on its own track.
