## Summary

Keep open. Current main still treats bare memory phrasing like `I will remember your preference` as an unbacked reminder commitment, and open PR #47589 is already the closing-linked implementation candidate.

## What This Changes

The issue asks to narrow the reminder guard so memory-retention `I'll remember ...` replies do not trigger the unscheduled-reminder disclaimer, with cron suppression scoped more precisely.

## Best Possible Solution

Review and land PR #47589 or an equivalent focused matcher-and-test fix, while leaving deeper current-turn cron suppression to #43292/#52972 unless maintainers intentionally expand the scope.

## Reproduction Assessment

Yes, at source level. Current main's regex matches the reported memory phrases, and the reply pipeline appends the disclaimer when no cron add or session cron coverage suppresses it.

## Solution Assessment

Yes for the memory false-positive portion. Narrowing bare `remember` to explicit reminder intent and adding direct regression tests is the smallest maintainable fix; cron-suppression changes should remain separately scoped unless maintainers choose otherwise.
