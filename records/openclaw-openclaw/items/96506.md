## Summary

Keep open for maintainer review. The patch appears to fix a real ClawHub pinned-version compatibility bug, with source, live endpoint, test, and proof support; the remaining merge question is the deliberate fail-closed behavior when selected-version metadata cannot be fetched.

## What This Changes

The PR changes ClawHub plugin install resolution to recover selected-version compatibility for sparse non-latest artifact metadata, restrict package-level compatibility fallback to latest installs, fail closed on recovery errors, and add regression tests.

## Best Possible Solution

Land the resolver fix once maintainers accept fail-closed selected-version recovery; longer term, ClawHub should include selected-version compatibility in artifact responses so the client does not need the extra recovery request.

## Reproduction Assessment

Yes, at source level. Current main falls back to latest package compatibility after sparse pinned artifact metadata, and live ClawHub endpoints show the pinned artifact response lacks compatibility while the pinned version endpoint has selected-version compatibility.

## Solution Assessment

Yes, this is the right layer for the fix. The ClawHub install entry point funnels through the resolver before compatibility validation and download, so narrowing the fallback there avoids duplicating policy in callers; maintainers still need to accept the fail-closed recovery behavior.
