## Summary

Keep open: the reported pinned-version bug is credible, but this patch is too broad because it also drops the package-level latest compatibility guard for unpinned/latest ClawHub installs, and the PR has only test evidence rather than real CLI behavior proof.

## What This Changes

This PR removes package-detail compatibility fallback from ClawHub version resolution and adds a regression test for a pinned older plugin version without version-specific compatibility metadata.

## Best Possible Solution

Keep the fix, but narrow it so package-level compatibility is still used when the resolved version is the package latest, while older pinned versions rely only on version-specific compatibility or no restriction.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a pinned ClawHub version without version-specific compatibility currently falls through to package-level latest compatibility and can be rejected by validateClawHubPluginPackage; I did not run a live ClawHub install in this read-only review.

## Solution Assessment

No. The PR is a plausible fix for older pinned versions, but the best fix should preserve the package-level fallback for the resolved latest version and add a regression test proving latest installs still enforce compatibility when version metadata is sparse.
