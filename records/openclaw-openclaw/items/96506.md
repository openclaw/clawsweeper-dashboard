## Summary

Keep open. The PR fixes a real ClawHub install bug: current main and the latest release can still apply latest package compatibility to a pinned older artifact response, while the patch has focused tests and sufficient CLI proof; the remaining blocker is maintainer acceptance of the new fail-closed recovery behavior.

## What This Changes

The PR changes ClawHub plugin install resolution to recover selected-version compatibility for sparse pinned artifact metadata, restrict package-level compatibility fallback to latest installs, fail closed on recovery errors, and add regression tests.

## Best Possible Solution

Land this resolver change after maintainers explicitly accept fail-closed selected-version recovery; otherwise adjust the fallback policy and tests to match the chosen compatibility contract.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.10 fall back to package-level compatibility for sparse pinned artifact metadata, and live ClawHub endpoints show the pinned artifact response lacks compatibility while the version endpoint has selected-version compatibility.

## Solution Assessment

Yes, this is the right resolver boundary, but merge should wait for maintainer acceptance of fail-closed recovery. The install entry point centralizes compatibility resolution before validation and download, so fixing it there avoids duplicate policy in callers.
