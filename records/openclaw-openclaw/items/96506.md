## What This Changes

The PR changes ClawHub plugin install resolution to use package-level compatibility only for the resolved latest version and adds tests for pinned older and unpinned latest installs with sparse artifact compatibility.

## Best Possible Solution

Recover selected-version compatibility from the version metadata when artifact metadata is sparse, keep package-level fallback only for the selected latest version, and add after-fix real CLI proof.

## Reproduction Assessment

Yes, by source inspection plus live ClawHub API responses: the artifact endpoint can omit compatibility while the selected version endpoint still advertises it. I did not run the local CLI in this read-only review, and the PR still lacks after-fix real CLI proof.

## Solution Assessment

No. The latest-only package fallback is a plausible mitigation for the original pinned symptom, but the best fix must preserve selected-version compatibility rather than treating sparse artifact metadata as unrestricted.
