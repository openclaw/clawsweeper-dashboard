## Summary

Keep open: this is a useful ClawHub install bug fix, but the current branch is not merge-ready because the selected-version compatibility recovery fails open if the version endpoint errors, and after-fix real CLI proof is still missing.

## What This Changes

The PR changes ClawHub plugin install resolution so pinned older versions use artifact compatibility, then selected-version compatibility, then package-level compatibility only for latest, with regression tests.

## Best Possible Solution

Keep selected-version compatibility recovery, but fail closed or return a mapped version error when selected-version metadata cannot be fetched; keep package-level fallback only for latest installs and add redacted after-fix CLI proof.

## Reproduction Assessment

Yes, from source inspection and live API shape rather than a local CLI run. Current main falls back to latest package compatibility, and live ClawHub shows sparse artifact metadata while the selected version endpoint carries the older version's compatibility.

## Solution Assessment

No. The branch is in the right resolver boundary and now recovers selected-version compatibility when the request succeeds, but swallowing recovery errors is not the safest fix because the version endpoint is the source of truth when artifact metadata is sparse.
