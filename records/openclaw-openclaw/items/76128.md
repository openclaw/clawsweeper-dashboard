## Summary

Keep open. Current main still lets `openclaw models set` update the global default while an explicit default-agent model remains the effective runtime model; the separate `models status --agent` source labels help diagnosis but do not warn at write time.

## What This Changes

The issue requests that default-model changes detect, warn about, or explicitly reconcile `agents.list.main.model` overrides that keep the active default agent on an older model.

## Best Possible Solution

Keep per-agent model override precedence intact, but make `openclaw models set` surface explicit default-agent overrides that will still win after the default change, with a clear path to update or reset them.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: configure `agents.defaults.model.primary` to a new model while `agents.list` contains `id: "main"` with an older `model.primary`; current resolution uses the agent model first while `models set` only writes defaults.

## Solution Assessment

Yes for a narrow warning/reporting fix. Automatic reconciliation or precedence changes would be broader than needed because explicit per-agent model overrides are current documented and tested behavior.
