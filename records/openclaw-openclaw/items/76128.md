## Summary

Keep open. Current main still has the reported shadowing shape: `models set` updates the global default, while runtime/model-status paths can still prefer an explicit default-agent `agents.list[].model`; the remaining decision is the right product contract for warning vs. mutating explicit agent overrides.

## What This Changes

This issue requests that default-model changes stop silently leaving the active default agent on an older explicit per-agent model override.

## Best Possible Solution

Make the default-model command and diagnostics expose the effective active-agent result; at minimum, warn when the default agent has an explicit `agents.list[].model` that will continue to win after `models set`.

## Reproduction Assessment

Yes, at source level: `modelsSetCommand` updates `agents.defaults.model`, while runtime/status/list surfaces resolve the effective agent model by checking `agents.list[].model` before the default. I did not run the CLI because this is a read-only cleanup review and tests/commands could write artifacts.

## Solution Assessment

Unclear as a product decision. A warning is the narrowest low-risk repair, while automatically mutating explicit per-agent overrides would be more surprising and should be maintainer-approved.
