## Summary

Keep open. Current main and v2026.6.6 still have the reported shadowing shape: `models set` updates only the global default, explicit `agents.list[].model` still wins for the agent, and the existing source visibility is only available through a separate status path.

## What This Changes

The issue asks OpenClaw to warn, reconcile, or expose when `models set` changes the global default but the active default agent remains pinned by an explicit per-agent model override.

## Best Possible Solution

Maintainers should choose the explicit-agent override contract, then implement a narrow non-destructive path such as a `models set` warning plus status or doctor guidance before considering any automatic per-agent mutation.

## Reproduction Assessment

Yes. Source inspection shows `models set` writes `agents.defaults.model`, while `resolveAgentEffectiveModelPrimary` still chooses an explicit `agents.list[].model` before the default; I did not run the CLI because this review is read-only.

## Solution Assessment

Unclear. A warning or diagnostic path looks safer than silently mutating explicit agent pins, but maintainers need to choose whether `models set` should only report shadowing, offer an explicit action, or coordinate with doctor/status reconciliation.
