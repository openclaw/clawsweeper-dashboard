## Summary

Keep open. Current main still lets `openclaw models set` update only `agents.defaults.model` while an explicit `agents.list` model for the default agent continues to win at runtime; `models status --agent` helps diagnose the source, but the write path still gives no warning.

## What This Changes

Request that default-model changes warn about or reconcile a default-agent `agents.list[].model.primary` override that keeps the active agent on the old model.

## Best Possible Solution

Keep override precedence intact, but make `openclaw models set` detect explicit default-agent model overrides that still point elsewhere and print a clear warning with the affected agent id and config path; any rewrite should require an explicit opt-in flag.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a config with `agents.defaults.model.primary` set to the new model and `agents.list` containing `id: "main"` with its own old `model.primary`; the current resolver returns the explicit agent model first.

## Solution Assessment

Yes for a narrow warning/reporting fix. Automatic reconciliation or precedence changes would need maintainer product judgment because they would alter documented per-agent override semantics.

## Repair Work Prompt

Fix issue https://github.com/openclaw/openclaw/issues/76128 by adding a narrow warning/reporting path for `openclaw models set <model>` when the configured default agent has an explicit `agents.list[].model.primary` that will continue to shadow the newly written `agents.defaults.model.primary`. Preserve current per-agent override precedence and do not silently rewrite `agents.list[].model` without an explicit new opt-in flag. Likely implementation: after `applyDefaultModelPrimaryUpdate`, inspect the configured default agent via `resolveDefaultAgentId`/`resolveAgentExplicitModelPrimary` and warn when the explicit primary is non-empty and differs from the new default. Include the agent id and config path (`agents.list[].model.primary`) in the message; mention that `openclaw models status --agent <id>` shows the source. Add focused coverage for the warning and for no warning when the default agent inherits defaults or already matches. Update user-facing docs and CHANGELOG.md because this changes CLI output. Do not redesign session override handling, subagent precedence, or global model reconciliation in this PR.
