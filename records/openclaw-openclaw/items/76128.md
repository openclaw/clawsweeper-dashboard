## Summary

Keep open. Current main and the latest release still have the reported shadowing shape: `openclaw models set` updates the global default only, while an explicit main/default-agent model override remains the effective runtime model and no source-aware warning or reconciliation path has been chosen.

## What This Changes

The issue requests source-aware handling when changing the global default model while the active main/default agent has an explicit `agents.list[].model` override.

## Best Possible Solution

Maintainers should choose a source-aware, non-destructive contract first, likely a `models set` warning plus status or doctor guidance before any explicit opt-in agent mutation.

## Reproduction Assessment

Yes. Source inspection shows `models set` writes only `agents.defaults.model`, while `resolveAgentEffectiveModelPrimary` still selects an explicit `agents.list[].model` before the default; I did not run the CLI because this review is read-only.

## Solution Assessment

Unclear. A warning or source-chain diagnostic looks safer than silently mutating explicit agent pins, but maintainers need to choose whether the command should report shadowing, offer an explicit action, or delegate reconciliation to doctor/status.
