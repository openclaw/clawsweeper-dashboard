## Summary

Keep open. Current main and v2026.6.10 still let `openclaw models set` update the global default while an explicit main/default agent model remains the effective runtime model; the remaining fix needs maintainer policy for warning, source visibility, doctor reporting, or explicit opt-in mutation.

## What This Changes

The issue requests that changing the default model not silently leave the main/default agent using an older explicit `agents.list[].model` override.

## Best Possible Solution

Keep the current precedence until maintainers choose a non-destructive source-aware contract, likely a `models set` warning plus status or doctor guidance before any explicit opt-in agent mutation.

## Reproduction Assessment

Yes. Source inspection shows `models set` writes only `agents.defaults.model`, while effective agent model resolution still selects an explicit `agents.list[].model` before the global default; I did not run the CLI because this pass is read-only.

## Solution Assessment

Unclear. A warning or source-chain diagnostic is safer than silently mutating explicit agent pins, but maintainers need to choose whether this belongs in `models set`, doctor, status, or an explicit opt-in mutation path.
