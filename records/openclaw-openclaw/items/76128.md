## Summary

Keep open. Current main still allows `models set` to report a successful global-default update while an explicit default-agent model override remains the effective runtime model; `models status --agent` helps diagnose it, but the write path does not warn about the shadowing override.

## What This Changes

Changing the default model should not silently leave the active/default agent pinned to an older `agents.list[].model.primary` override.

## Best Possible Solution

Keep agent override precedence intact, but make the global default-model write path clearly warn when the configured default agent will still resolve from an explicit per-agent model.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: configure `agents.defaults.model.primary` to a new value while `agents.list` contains the active/default agent with its own older `model.primary`; current resolution reads the agent value first.

## Solution Assessment

Yes for a narrow warning or diagnostic repair. Automatic reconciliation should be opt-in or separately reviewed because it would change existing per-agent override semantics.

## Repair Work Prompt

Create a narrow fix for https://github.com/openclaw/openclaw/issues/76128. Current `openclaw models set <model>` updates `agents.defaults.model.primary` but can leave the configured default/active agent pinned by an explicit `agents.list[].model.primary`. Do not change model precedence and do not silently rewrite all agent overrides. Preferred fix: after resolving/writing the new global default, detect the configured default agent via the existing agent-scope helpers; if that agent has an explicit model primary that differs from the new default, print a clear warning listing the agent id and `agents.list[].model.primary`, with guidance to run `openclaw models status --agent <id>` or update the per-agent config. Add tests showing no warning when there is no explicit agent model or it matches, and a warning when `main`/default shadows the new global default. Update docs/changelog for the user-facing warning. Keep this scoped to issue #76128; do not solve the broader reconcile proposal #28965 or session override issues #51251/#74284 in this PR.
