## Summary

Keep open. Current main still lets `openclaw models set` report a successful global default update while an explicit default-agent `agents.list[].model.primary` continues to win for the active agent, and the later comment correctly notes the same old model can also flow into spawned subagents through the caller-model fallback.

## What This Changes

Request `openclaw models set` not to silently leave the main/default agent on an older explicit agent-level model after changing the global default.

## Best Possible Solution

Keep per-agent override precedence intact, but make `openclaw models set` warn when the configured default agent has an explicit model primary that still shadows the newly selected global default.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: configure `agents.defaults.model.primary` to the new model while `agents.list` contains the default `main` agent with its own older `model.primary`; current `models set` writes only the default, and effective resolution still chooses the agent primary.

## Solution Assessment

Yes for a narrow warning/reporting repair. Automatic reconciliation or precedence changes would be riskier because documented per-agent overrides should remain explicit and user-controlled.
