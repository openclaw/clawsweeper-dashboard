## Summary

Keep open. Current `main` still lets `openclaw models set` update and report the global default while an explicit default-agent `agents.list[].model.primary` remains authoritative for the active agent, and the later thread comment is correct that the same pinned model can flow into spawned subagents through the caller-model fallback.

## What This Changes

The issue requests that default-model changes surface or reconcile agent-level model overrides so the active `main` agent does not silently keep using an older explicit model.

## Best Possible Solution

Keep per-agent model precedence intact, but make `openclaw models set` warn when the configured default agent has an explicit model primary that will still shadow the newly selected global default, including the subagent inheritance implication where practical.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `agents.defaults.model.primary` to a new model while the default `main` agent has its own older `agents.list[].model.primary`; current `models set` writes only the default, and effective resolution still chooses the agent primary.

## Solution Assessment

Yes for a narrow diagnostic repair. A warning/reporting change is safer than automatic reconciliation or precedence changes because the docs define per-agent model overrides as intentional configuration.
