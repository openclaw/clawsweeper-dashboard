## Summary

Keep open. Current main and the latest release still let `openclaw models set` update only the global default while an explicit main-agent model remains the effective runtime model; current `models status --agent` source labels help, but they do not address the silent write-path confusion.

## What This Changes

The issue reports that changing the global default model can leave the main/default agent using an older explicit agent-level model override.

## Best Possible Solution

Add a maintainer-approved visibility path for default-model writes, such as warning from `openclaw models set` when the default agent has an explicit model override, without silently rewriting intentional per-agent pins.

## Reproduction Assessment

Yes, from source: `openclaw models set` writes `agents.defaults.model`, while agent resolution still prefers `agents.list[].model` for the requested agent. I did not run the CLI because this review is read-only and the command mutates config.

## Solution Assessment

Unclear as an implementation choice: warning/status/doctor visibility is safer than mutating agent overrides, but maintainers need to choose the policy for intentional main-agent pins.
