## Summary

Keep open. Current main and the latest release still have the reported shadowing shape: `models set` writes the global default, while explicit `agents.list[].model` remains a higher-precedence effective model for the agent; current `models status --agent` adds some source visibility, but it does not warn or reconcile during the model-change operation.

## What This Changes

Request warning, reconciliation, or clearer status when `openclaw models set` changes the global default but an explicit default-agent model keeps runtime traffic on the old model.

## Best Possible Solution

Pick the intended contract for explicit default-agent model overrides, then implement one narrow path such as a `models set` warning plus status/doctor guidance without silently rewriting intentional per-agent config.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source inspection: with `agents.defaults.model.primary` changed by `models set` and `agents.list[].id == "main"` carrying its own `model.primary`, current resolution still chooses the agent-level value; I did not run the CLI because this review is read-only.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose warning versus mutation versus doctor/status diagnostics; the safest implementation should preserve intentional per-agent pins while making shadowing visible at the time of change.
