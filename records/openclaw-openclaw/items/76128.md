## Summary

Keep open. Current main still lets `openclaw models set` update and report the global default while an explicit default-agent model can continue to win, and the follow-up comment’s subagent-inheritance concern matches the current source path.

## What This Changes

The issue asks `openclaw models set` to warn, reconcile, or diagnose explicit `agents.list[].model.primary` overrides that leave the main/default agent on the old model.

## Best Possible Solution

Keep per-agent precedence intact, but have `openclaw models set` warn when the configured default/active agent has an explicit model primary that will continue to shadow the new global default.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `agents.defaults.model.primary` to a new model while `agents.list` contains the default `main` agent with an older explicit `model.primary`; current `models set` writes only the default, and runtime resolution still chooses the agent primary.

## Solution Assessment

Yes for a narrow warning or diagnostic repair. Updating agent overrides automatically, adding flags, or changing model precedence would be broader and riskier than surfacing the shadowing state at the command boundary.

## Repair Work Prompt

Fix the model-shadowing confusion from https://github.com/openclaw/openclaw/issues/76128. Keep per-agent model override precedence unchanged. Update `openclaw models set <model>` so that after writing `agents.defaults.model.primary`, it detects when the configured default/active agent has an explicit `agents.list[].model.primary` that differs from the new default and warns that the agent override will continue to win. Prefer a concise warning that points users to `openclaw models status --agent <id>` or the exact agent config path. Do not add new flags, do not silently rewrite `agents.list[].model`, and do not change subagent precedence. Add focused regression coverage for a default `main` agent with an old explicit primary, and update CLI docs/changelog if the warning is user-facing.
