## Summary

Keep open. Current main still lets `models set` report a successful global-default change while an explicit default-agent model override continues to win at runtime; the existing `models status --agent` source labeling helps diagnosis but does not warn during the write that creates the confusion.

## What This Changes

Request that default-model changes warn about or reconcile explicit `agents.list.main.model` overrides that keep the active/default agent on the old model.

## Best Possible Solution

Keep agent override precedence intact, but make `openclaw models set` detect explicit default-agent overrides that still point elsewhere and print a clear warning with the affected agent ids and the command/config path to update them.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: configure `agents.defaults.model.primary` to a new model while `agents.list` contains `id: "main"` with its own old `model.primary`; current resolution uses the agent model first.

## Solution Assessment

Yes for a narrow warning/reporting fix. Automatic reconciliation or precedence changes would need explicit user opt-in or maintainer product judgment because per-agent model overrides are documented behavior.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/76128 with a narrow warning/reporting change. Current behavior: `openclaw models set <provider/model>` updates `agents.defaults.model.primary` but does not warn when configured agents, especially the configured default agent or `main`, have an explicit `agents.list[].model.primary` that will continue to win at runtime. Implement a conservative diagnostic: after resolving and applying the new default, inspect configured agent entries for explicit `model` primaries that differ from the newly written default and print a clear warning listing affected agent ids and model refs. Prefer warning/reporting over automatic mutation; do not change `resolveAgentEffectiveModelPrimary` precedence and do not silently rewrite `agents.list[].model`. Add focused coverage for `modelsSetCommand` with an explicit default-agent override, preserve existing normalization/allowlist behavior, and update user-facing docs/changelog if the warning is user-visible.
