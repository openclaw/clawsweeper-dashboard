## Summary

Keep open. Current main and the latest release still let `openclaw models set` update only `agents.defaults.model`, while an explicit `agents.list[].model` remains the effective model for the main/default agent; the remaining fix needs maintainer semantics for warning, source visibility, doctor guidance, or explicit opt-in mutation.

## What This Changes

Changing the default model should not silently leave the active main/default agent pinned to an older explicit agent-level model override.

## Best Possible Solution

Keep the current precedence until maintainers choose a non-destructive source-aware contract, likely a `models set` warning plus status or doctor guidance before any explicit opt-in agent mutation.

## Reproduction Assessment

Yes. Source inspection shows `models set` writes only the global default while effective model resolution still prefers an explicit `agents.list[].model`; no CLI execution was run because this review is read-only.

## Solution Assessment

Unclear. A warning or source-chain diagnostic is safer than silent mutation of explicit agent pins, but maintainers need to choose whether this belongs in `models set`, doctor, status, or an explicit opt-in mutation path.
