## Summary

Keep open. Current main and the latest release still update only the global default during `models set`, while explicit agent-level model config remains higher precedence; the remaining fix is a compatibility-sensitive UX/product decision rather than cleanup-closeable work.

## What This Changes

The issue asks OpenClaw to warn, reconcile, or otherwise expose stale default-agent model overrides when changing the global default model.

## Best Possible Solution

Pick the public contract for default-agent overrides, then add one narrow visible path such as a `models set` warning, explicit reset/include flag, or doctor/status diagnostic without silently rewriting intentional per-agent config.

## Reproduction Assessment

Yes. Source inspection on current main shows `models set` updates `agents.defaults.model`, while effective model resolution still lets an explicit `agents.list[].model` primary win over that default.

## Solution Assessment

Unclear. Warning, explicit flag, doctor detection, and status-source improvements are all plausible, but maintainers need to choose the compatibility contract for existing per-agent pins.
