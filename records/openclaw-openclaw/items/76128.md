## Summary

Keep open. Current main and the latest release still have the reported shadowing shape: `models set` updates only the global default, explicit `agents.list[].model` still wins for the active agent, and the existing source visibility is only available through a separate status path.

## What This Changes

The issue asks OpenClaw to warn, reconcile, or expose source information when a default-model change is shadowed by an explicit main/default agent model override.

## Best Possible Solution

Keep explicit per-agent override precedence, but add a maintainer-chosen non-destructive warning or diagnostic path that makes shadowing visible when the default model changes.

## Reproduction Assessment

Yes. Source inspection shows `models set` writes `agents.defaults.model`, while the agent resolver still chooses an explicit `agents.list[].model` before the default; I did not run the CLI because this review is read-only.

## Solution Assessment

Unclear. A warning or diagnostic path looks safer than silently mutating explicit agent pins, but maintainers need to choose whether `models set`, doctor, status, or an explicit flag owns the behavior.
