## Summary

This PR should stay open: the underlying duplicate-reply bug is source-reproducible, but the proposed prompt-only fix is too broad because it removes required `message_tool_only` guidance for default Discord group/channel replies and still lacks after-fix live Discord proof.

## What This Changes

The PR updates the Codex embedded developer instruction, Discord skill overview, and changelog to route normal current-conversation replies through final assistant output while reserving the `message` tool for out-of-band Discord actions.

## Best Possible Solution

Keep the prompt-only fix boundary, but make both Codex and Discord guidance delivery-mode aware: final assistant output for automatic current-reply delivery, and `message(action="send")` for `message_tool_only` visible replies.

## Reproduction Assessment

Yes for review purposes by source inspection: current main can route autoThread final delivery to the created thread while the current prompt/skill guidance can still steer the agent to also call `message.send`. I did not run a live Discord session.

## Solution Assessment

No. Prompt and skill wording are the right narrow surfaces, but this patch is not delivery-mode aware and conflicts with the existing `message_tool_only` contract for visible Discord group/channel replies.
