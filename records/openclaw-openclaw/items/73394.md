## Summary

Keep open. The PR targets a real Discord autoThread duplicate-reply path, but the patch makes final assistant delivery unconditional even though current main still supports default tool-only visible replies for group/channel turns, and the contributor has not provided after-fix live Discord proof.

## What This Changes

The PR changes Codex developer instructions, Discord skill text, and the changelog to reserve the message tool for sideband actions and use final assistant output for current-conversation replies.

## Best Possible Solution

Make the Codex and Discord guidance explicitly delivery-mode aware: use final assistant output when OpenClaw will deliver the current reply automatically, and preserve message(action="send") guidance for message_tool_only visible turns.

## Reproduction Assessment

Yes for review purposes. Source inspection on current main shows autoThread final delivery can target the created thread while current prompt and skill text can still steer the agent to also call message.send; I did not run a live Discord session.

## Solution Assessment

No. Prompt and skill wording is the right narrow surface, but this patch is not delivery-mode aware and would undercut the documented tool-only visible-reply mode.
