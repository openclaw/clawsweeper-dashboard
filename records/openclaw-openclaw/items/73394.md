## Summary

Keep open: the PR is unsafe as written because current main defaults Discord guild/channel turns to tool-only visible delivery, while the proposed prompt text tells agents to rely on final assistant delivery for the current conversation.

## What This Changes

The PR updates the Codex embedded developer instructions, bundled Discord skill text, and changelog to discourage same-conversation `message.send` replies for Discord autoThread mentions.

## Best Possible Solution

Make the Codex and Discord guidance delivery-mode aware: preserve `message(action=send)` for visible output in `message_tool_only` turns, and warn against duplicate same-conversation sends only when automatic final delivery is active.

## Reproduction Assessment

Yes. The linked bug report gives concrete Discord autoThread steps, and current-main source/tests show a high-confidence regression path where default guild/channel turns are `message_tool_only` and final source delivery is suppressed.

## Solution Assessment

No. Prompt-only is a reasonable boundary, but this wording is not the best fix because it contradicts `sourceReplyDeliveryMode`; conditional guidance is the safer maintainable path.
