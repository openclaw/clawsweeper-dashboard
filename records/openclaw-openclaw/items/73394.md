## Summary

Keep open. The PR targets a real Discord autoThread duplicate-reply problem, but the proposed prompt wording is unsafe on current main because Discord guild/channel turns default to message-tool-only visible delivery while automatic final source delivery is suppressed.

## What This Changes

The PR updates the changelog, Codex embedded developer instructions, and the bundled Discord skill so normal current-conversation replies use the final assistant message while the `message` tool is reserved for explicit out-of-band Discord actions.

## Best Possible Solution

Make the static Codex and Discord skill guidance defer to the current visible-reply policy: use the final assistant message only when automatic final delivery is active for the current conversation, keep `message(action=send)` for visible output when final replies are private/message-tool-only, and continue to reserve the message tool for explicit cross-conversation or Discord-specific actions.

## Reproduction Assessment

Yes. The original duplicate path is documented in #73278 against OpenClaw 2026.4.26, and the PR regression has a current-main code/test proof: default Discord guild/channel turns are `message_tool_only` and automatic final delivery is suppressed.

## Solution Assessment

No. A prompt-only fix is plausible, but this version unconditionally relies on final assistant delivery and therefore conflicts with current-main message-tool-only Discord channel policy.
