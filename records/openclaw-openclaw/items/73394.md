## Summary

Keep open. The PR targets a real Discord autoThread duplicate-reply bug, but the proposed prompt wording is still too broad for current main's tool-only group/channel delivery mode and the external PR lacks after-fix real Discord proof.

## What This Changes

The PR changes Codex embedded developer instructions, the bundled Discord skill, and the changelog to reserve the message tool for out-of-band Discord actions and route normal current-conversation replies through final assistant output.

## Best Possible Solution

Make the Codex and Discord guidance explicitly delivery-mode aware: use final assistant output when automatic source delivery will post to the auto-created thread, and preserve message(action="send") guidance for tool-only visible Discord turns.

## Reproduction Assessment

Yes for review purposes. Source inspection shows autoThread final delivery can target the created thread while the current prompt/skill text can still steer an agent to also call message.send; I did not run a live Discord session in this read-only review.

## Solution Assessment

No. Prompt-only may be the right boundary, but this version is not the best fix because it makes final delivery unconditional instead of respecting sourceReplyDeliveryMode.
