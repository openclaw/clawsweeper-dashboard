## Summary

Keep this PR open: it addresses a real Discord autoThread duplicate-reply bug, but the patch is not mergeable because its Discord skill guidance contradicts current `message_tool_only` source-delivery behavior for default guild/channel turns.

## What This Changes

The PR rewrites Codex embedded developer instructions, Discord skill guidance, and the changelog so current-conversation replies use final assistant delivery while explicit Discord operations use the `message` tool.

## Best Possible Solution

Make the Codex and Discord prompt guidance conditional on `sourceReplyDeliveryMode`: automatic turns should avoid duplicate same-conversation sends, while `message_tool_only` turns must still use `message(action=send)` for visible source-channel output.

## Reproduction Assessment

Yes. #73278 gives concrete OpenClaw 2026.4.26 Discord autoThread steps, and current-main source/tests provide a high-confidence regression path through default Discord `message_tool_only` channel delivery.

## Solution Assessment

No. Prompt-only may be the right boundary, but this wording is not the best fix because it treats final assistant delivery as universal instead of delivery-mode dependent.

## Review Findings
