## Summary

Current main already turns the reported ACP no-terminal-event path into an `ACP_TURN_FAILED` rejection and a terminal failed session lifecycle state; the fix is shipped in `v2026.4.26` and later.

## What This Changes

Reject ACP `runTurn` when an ACP child exits or stream closes before a terminal event, so gateway session lifecycle becomes failed instead of running forever.

## Best Possible Solution

Keep the shipped generic ACP terminal-event and timeout handling; ask for a fresh report only if a current `2026.5.x` install still leaves ACP sessions running after a child exits.

## Reproduction Assessment

Yes, source-level: current tests simulate an ACP stream that ends without `done` and assert `runTurn` rejects and the session becomes error. I did not run tests because this was a read-only review.

## Solution Assessment

Yes. The current fix is at the maintainable generic boundary: the ACP control plane rejects non-terminal streams, `agent-command` emits lifecycle error, and gateway session persistence maps that to `failed` without adding provider-specific policy.
