## Summary

Current main and shipped releases after the reported 2026.4.5 build already handle this failure shape: ACP turns now reject on missing terminal events or backend failures and persist the session as errored instead of leaving it running.

## What This Changes

Report asks ACP runtime dispatch to reject and mark sessions failed when an ACP child process exits or closes stdout before sending a terminal event.

## Best Possible Solution

Keep the shipped ACP manager terminal-event guard and acpx 0.6.1 runtime behavior; deployments still on 2026.4.5 should upgrade to v2026.4.26 or newer.

## Reproduction Assessment

No current-main failing reproduction remains. The 2026.4.5 report is source-plausible, but current main has dependency/runtime guards and regression coverage for backend exit or stream end before a terminal done event.

## Solution Assessment

Yes. Handling this at the ACP manager/runtime boundary is the narrow maintainable fix because all ACP backends share the same terminal-event and session-state contract.
