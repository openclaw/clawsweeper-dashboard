## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Apply pending reads against live state:** `ui/src/ui/views/chat.ts:597-600`
  - body: `props` is a render-time snapshot, so concurrent reads can all compute from the same `attachmentReadPending` value. With two files starting from 0, both increments can set 1 and one `loadend` can set 0 while another read is still active, re-enabling send before every attachment is available. Report deltas or use an updater that adds against current app state.
  - confidence: 0.91
- **[P2] Let override commands bypass pending attachment reads:** `ui/src/ui/app-chat.ts:273-275`
  - body: This early return runs before command handling and also applies to `messageOverride` sends, which never include composer attachments. Actions such as the compact control call `handleSendChat("/compact", ...)`, so they become enabled no-ops while an image is loading. Limit the guard to composer sends that would include attachments, or route local/override commands first.
  - confidence: 0.86
