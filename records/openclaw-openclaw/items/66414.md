## Review Findings

Overall confidence: 0.88

Full review comments:

- **[P1] Apply pending read deltas to live state:** `ui/src/ui/views/chat.ts:600-602`
  - body: `props` is the render-time snapshot captured when the handler ran, so concurrent reads can all compute `next` from the same stale count. That can re-enable send while reads are still in flight or leave the composer stuck disabled; move the delta application to the app/controller state owner or use an equivalent live-state updater.
  - confidence: 0.91
- **[P2] Do not block override commands during reads:** `ui/src/ui/app-chat.ts:276-278`
  - body: This guard runs before command handling even when `messageOverride` is set, while `attachmentsToSend` is empty for override sends. During an image read, button-driven commands such as starting a new session or compacting can silently no-op even though they do not depend on the pending attachment.
  - confidence: 0.82
- **[P3] Name the pending state for assistive tech:** `ui/src/ui/views/chat.ts:1693`
  - body: When the pending-read state disables the send button, the title changes to "Processing attachment..." but the accessible name remains "Send message" or "Queue message". Update the aria-label alongside the disabled/title state so screen-reader users get the same reason.
  - confidence: 0.78
