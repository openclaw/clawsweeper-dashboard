Full review comments:

- **[P1] Apply pending-read deltas to live state:** `ui/src/ui/views/chat.ts:602-603`
  - body: `props` is the render snapshot that created the FileReader handlers, so two concurrent reads can both increment from the same old value and the first `loadend` can set the count back to zero while another read is still running. That re-enables send early and leaves the attachment-drop race this PR is meant to fix.
  - confidence: 0.95
- **[P2] Let override sends bypass attachment-read waits:** `ui/src/ui/app-chat.ts:276`
  - body: This guard runs before command handling and also blocks `messageOverride` calls, even though those sends intentionally use no attachments. While a file is reading, actions such as compact or other programmatic chat sends can become silent no-ops instead of executing.
  - confidence: 0.84
- **[P3] Reflect the processing state in the button label:** `ui/src/ui/views/chat.ts:1693`
  - body: When the PR disables the send button for pending attachment reads, the title changes but the accessible name still says send or queue. Screen-reader users do not get the reason the control is disabled.
  - confidence: 0.8
