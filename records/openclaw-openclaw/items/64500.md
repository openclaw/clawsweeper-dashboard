## Summary

Keep open. Current main and the latest shipped tag still preserve the reported gap: changing-output ping-pong loops stay warning-only, and the global breaker is still fed by a same-tool/same-argument no-progress streak. The open session-global breaker PR is useful adjacent work, but it does not close this issue's pair-aware changing-output escalation problem.

## What This Changes

The issue asks OpenClaw to stop sustained A/B tool ping-pong loops by enforcing circuit breaking at the pair level instead of only on the tool that most recently crossed a threshold.

## Best Possible Solution

Add run/session-scoped pair-aware loop enforcement to the existing detector and before-tool hook so sustained offending A/B signatures can block both sides, while preserving conservative progress safeguards for legitimate alternating work.

## Reproduction Assessment

Yes, at source level. Alternating A/B calls with changing per-side result hashes still reach the ping-pong warning branch rather than the critical branch, and the hook only vetoes the current attempted call when a critical detector fires.

## Solution Assessment

Unclear until maintainers choose the false-positive tradeoff. Pair-signature-aware enforcement is the narrowest maintainable direction, while simply dropping `noProgressEvidence` from all critical ping-pong cases is riskier.
