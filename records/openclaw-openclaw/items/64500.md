## Summary

Keep open. Current main still has the source-backed pair-enforcement gap: sustained A/B alternation with changing per-side result hashes remains warning-only, and the before-tool-call hook only vetoes the current invocation rather than persisting a paired-tool block.

## What This Changes

The issue requests pair-aware tool-loop circuit breaker enforcement so ping-pong loops block both tools or the alternating pair instead of only the current repeated tool.

## Best Possible Solution

Add pair-aware enforcement to the existing loop detector/hook path so a sustained offending A/B signature pair trips within the active run and blocks both sides without changing defaults or adding new config.

## Reproduction Assessment

Yes. A focused source-level reproduction is to record alternating A/B calls with changing per-side result hashes: current code and tests show this reaches ping-pong warning at the critical threshold but not critical blocking, and the hook has no pair-block state.

## Solution Assessment

Yes. Pair-aware enforcement using the existing thresholds and run-scoped history is the narrowest maintainable direction; it is safer than simply removing all progress evidence checks without constraining the offending pair.
