## Summary

Keep open. Current main still has the reported pair-aware enforcement gap: ping-pong critical blocking requires stable no-progress hashes, while the global breaker and before-tool hook remain scoped to the current tool call rather than the alternating pair.

## What This Changes

The issue requests pair-aware circuit-breaker behavior so sustained alternating tool loops block both tools in a detected pair rather than only the current tool.

## Best Possible Solution

Implement pair-aware loop enforcement in the agent tool-loop detector and before-tool-call hook: track the alternating pair/signatures within the active run/session, trip a conservative pair-scoped breaker for sustained A/B loops, block or suppress both sides of the offending pair, and add regression coverage for variable-output ping-pong loops without changing unrelated generic-repeat or unknown-tool behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an alternating A/B tool sequence with changing per-side result hashes: current main reaches ping-pong warning at the threshold but does not critical-block, and the hook has no stored pair block for the alternate tool.

## Solution Assessment

Yes, pair-aware enforcement is the narrow maintainable direction. The safer variant is a pair-keyed breaker or pair block state scoped to the active run/session, rather than bluntly denying tools globally or weakening all no-progress heuristics without false-positive controls.
