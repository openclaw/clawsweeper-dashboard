## Summary

Keep open: current main and v2026.6.6 reduce some cron workarounds with command jobs and fresh isolated sessions, but the requested scheduler/heartbeat elevation trust model is still missing.

## What This Changes

Request per-job and heartbeat-scoped elevated exec authorization so scheduled/internal events do not rely on wildcard synthetic-provider allowlists.

## Best Possible Solution

Add a maintainer-reviewed scheduler and heartbeat trust model that stores approved elevation intent, defaults untrusted scheduled work to non-elevated, and migrates/docs existing wildcard setups safely.

Unclear as an implementation, but the direction is valid. The best fix needs maintainer security design for persisted trust, owner approval, upgrade behavior, and docs rather than a small automatic boolean addition.
