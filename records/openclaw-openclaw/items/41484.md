## Summary

Keep open: current main has partial cron safety mitigations, but it does not implement per-job elevated/provenance scoping and this is a security-sensitive feature that needs maintainer design.

## What This Changes

Request adding per-job elevation controls so selected cron and heartbeat automation can run elevated without granting wildcard elevated access to all internal cron-event/heartbeat runs.

## Best Possible Solution

Define one fail-closed cron/heartbeat elevation trust contract, then add the schema, runtime checks, docs, migration/doctor guidance, and regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection confirms the requested per-job elevated field/provenance is absent while current elevated access is resolved through provider allowlists and session state.

## Solution Assessment

Unclear. The proposed direction is plausible, but the maintainable fix needs a security/product decision for cron job ownership, heartbeat provenance, compatibility, and fail-closed defaults before implementation.
