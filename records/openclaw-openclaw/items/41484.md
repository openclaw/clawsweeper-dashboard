## Summary

Keep open. Current main still lacks a cron or heartbeat elevated-provenance contract, and the requested change affects the elevated host-exec authorization boundary rather than a narrow bug fix.

## What This Changes

Request per-job elevated-exec scoping for cron jobs and heartbeat so unattended automation can be approved without wildcard internal-event allowlists.

## Best Possible Solution

Design one canonical cron and heartbeat elevated-provenance contract, then update protocol schema, runtime types, cron service paths, elevated resolution, heartbeat handling, docs, doctor or migration behavior, and regression tests together.

## Reproduction Assessment

Yes, at source level. Current main has no per-job elevated/provenance field in cron schema/types or persistence, and elevated availability still resolves from provider/sender allowlists.

## Solution Assessment

Unclear. A per-job flag is plausible, but the maintainable fix needs an explicit privilege-provenance design for approval, update, revocation, migration, and heartbeat authority.
