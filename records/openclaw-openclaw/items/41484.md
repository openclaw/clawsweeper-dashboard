Keep open. Current main and v2026.5.22 still lack per-job cron elevated authority and heartbeat provenance, so this remains a security-sensitive feature/design request rather than cleanup.

## What This Changes

The issue requests per-job elevated execution scoping for cron jobs and heartbeat so unattended automation can run with approved elevated privileges without wildcard internal-event allowlists.

## Best Possible Solution

Define one canonical elevated-provenance contract for cron and heartbeat, then update protocol schema, persistence, elevated resolution, docs, migration or doctor behavior, and regression coverage together.

## Reproduction Assessment

Yes, at source level. Current main has no cron elevated/provenance field in protocol, runtime types, create/patch persistence, or heartbeat/elevated resolution.

## Solution Assessment

Unclear. A per-job flag is plausible, but maintainers need to choose the approval, update, revocation, audit, migration, and heartbeat provenance contract before implementation.
