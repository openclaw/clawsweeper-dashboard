## Summary

Keep open. Current main has cron and heartbeat hardening, but it still does not implement the requested persisted per-job elevated/trust flag, owner-approved mutation rule, or heartbeat provenance/explicit elevated metadata.

## What This Changes

The issue requests per-job elevated exec scoping for cron jobs and heartbeat so privileged scheduled runs can be explicitly approved without relying on broad internal `cron-event` or `heartbeat` elevated allowlists.

## Best Possible Solution

Design and implement an additive cron/heartbeat trust model that persists elevated/provenance metadata, authorizes setting `elevated: true` only from an approved owner/elevated sender, avoids broad internal event-source allowlists, and documents/migrates the resulting behavior without weakening existing scheduled jobs.
