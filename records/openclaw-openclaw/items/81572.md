## Summary

Keep open: current main still buffers due-job outcomes until every worker in the batch finishes, while this PR contains a focused fix, regression coverage, sufficient live proof, and no blocking review findings.

## What This Changes

This PR changes cron timer due-batch processing to persist each completed job outcome immediately, preserve running markers for unapplied batch jobs, and adjust related cron and gateway tests.

## Best Possible Solution

Land the focused cron timer persistence fix with its regression tests after normal maintainer review and fresh validation against current main.

## Reproduction Assessment

Yes, from source inspection and the PR's focused regression scenario: current main buffers completed due-job outcomes until Promise.all resolves, so a completed deleteAfterRun job can remain persisted while a later same-batch job is still running. I did not execute tests in this read-only review.

## Solution Assessment

Yes. The PR moves the existing applyOutcomeToStoredJob and persist path to each completed result under the cron store lock, and the preserveRunningJobIds addition handles the long-batch stuck-marker edge case.
