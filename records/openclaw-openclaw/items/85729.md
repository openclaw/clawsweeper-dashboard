## Summary

Keep open. Current main already implements `config.schema.lookup` for Gateway RPC and the embedded agent `gateway` tool, but the ACPX/OpenClaw-tools MCP bridge that would expose built-in tools to external harnesses only publishes `cron`; fixing this bridge/Cline flow needs a scoped tool-surface decision rather than cleanup closure.

## What This Changes

Request that Cline bridge config tasks receive `config.schema.lookup` or prevalidated schema context instead of exiting without a result.

## Best Possible Solution

Add a narrow read-only bridge-visible config schema lookup path with explicit permission docs and failure reporting, or keep Cline-specific prevalidation in the external wrapper if that bridge stays outside OpenClaw.

## Reproduction Assessment

No, not for the full Cline wrapper flow: this checkout does not contain `bridge-cli.mjs` or the reported `.result.md` path. Source inspection does show the likely gap: the ACPX OpenClaw-tools MCP bridge exposes only `cron`, while `config.schema.lookup` exists elsewhere.

## Solution Assessment

Unclear. The maintainable direction is probably a narrow read-only config-schema lookup bridge, not exposing the full `gateway` tool surface to external harnesses without an explicit security/product decision.
