## Summary

Keep open: current main and the latest release still write root/include config before runtime SecretRef refresh can reject missing active refs, and the closing pull request is open rather than merged.

## What This Changes

The issue reports that model-assisted Moonshot/Kimi setup can persist an env SecretRef visible to the interactive shell but not to the daemon, leaving the gateway unable to start.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84454 or an equivalent maintainer-approved provider SecretRef activation policy that preflights daemon-visible refs before persistence and preserves rollback/recovery.

## Reproduction Assessment

Yes for the deterministic lower-level path: an active models.providers.<id>.apiKey env SecretRef with the env var absent from the daemon runtime is collected, resolution throws, and startup wraps it as a fatal secret failure. The exact model-assisted shell recommendation was not live-reproduced.

## Solution Assessment

Unclear until maintainers accept the policy. The linked PR is a focused implementation path for pre-commit SecretRef preflight and rollback, but fail-closed behavior for active provider refs needs maintainer approval before this issue can close.
