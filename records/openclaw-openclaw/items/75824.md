## Summary

Keep open. The report is fresh, corroborated across macOS, WSL2, and Debian, and current main only adds an explicit SSE escape path while the default direct OpenAI route remains WebSocket-first and the Web UI still clears failed live turns without appending an inline assistant error.

## What This Changes

Fix direct OpenAI API-key embedded runs where Web UI or channel turns time out with no assistant output even though direct OpenAI Responses API calls succeed.

## Best Possible Solution

Fix the direct OpenAI Responses path so default API-key embedded runs consume generated output or reliably fall back to SSE, and land inline Web UI error rendering so failed turns remain visible.

## Reproduction Assessment

Yes. A high-confidence user-level reproduction is to configure direct OpenAI API-key routing, send a simple Web UI or channel prompt, and compare OpenClaw's idle-timeout/empty assistant text with a successful direct OpenAI API or SDK streaming call from the same host.

## Solution Assessment

Not applicable as a patch review. The current explicit-SSE handling is a useful mitigation, but the maintainable fix should address the default OpenAI auto transport path and preserve visible error state in the Web UI.
