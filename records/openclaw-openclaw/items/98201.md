## What This Changes

The PR changes diagnostics-otel log export so withheld log message bodies use `[message redacted]`, adds an internal `openclaw.log.body_redacted` marker, blocks caller spoofing of that marker, and updates OTEL docs/tests.

## Best Possible Solution

Land the diagnostics-otel builder/docs/test change only after maintainers accept the exported log body compatibility change and focused OTEL log validation passes.

## Reproduction Assessment

Yes by source inspection: current main's diagnostic log builder emits `body: "log"` whenever log body capture is disabled, and existing focused tests assert that placeholder. I did not run the test suite because this review is read-only and the PR body already notes the focused Vitest proof was not completed.

## Solution Assessment

Yes, with a compatibility caveat: changing the shared log-record builder fixes both OTLP and stdout exports, and blocking the reserved marker in caller attributes is the right boundary for spoof prevention. The safer merge shape is to keep this narrow patch and make the exported-output change an explicit maintainer decision before landing.
