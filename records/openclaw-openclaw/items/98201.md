## What This Changes

The PR changes diagnostics-otel OTLP/stdout log export so withheld log bodies emit `[message redacted]` with a reserved `openclaw.log.body_redacted` marker, plus focused tests and docs.

## Best Possible Solution

Land this narrow builder/docs/test change after maintainers explicitly accept the exported-output compatibility change; otherwise preserve the old body value and add only a marker or transition path.

## Reproduction Assessment

Yes by source inspection: current main and the latest release emit `body: "log"` when log body capture is disabled, and current focused tests assert that placeholder. I did not run tests in this read-only cleanup review, but the PR body lists focused local and Testbox diagnostics-otel validation.

## Solution Assessment

Yes, with a compatibility caveat: changing the shared log-record builder is the right layer because both OTLP and stdout use that built record, and blocking caller spoofing protects the reserved marker. The safer alternative, if maintainers reject the output change, is to keep `body: "log"` and add only a documented marker or transition path.
