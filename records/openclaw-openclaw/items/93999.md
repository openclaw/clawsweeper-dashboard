## What This Changes

The PR adds `starting up` as a busy TUI activity status during post-connect initialization and adds unit plus fake-backend PTY coverage for that loader.

## Best Possible Solution

Keep the narrow startup-loader fix, but make startup status own only the startup-idle window, preserve reconnect/history busy states, and require redacted real TUI/PTTY/Hatch proof before merge.

## Solution Assessment
