## Summary

Keep this PR open: it is a focused Codex plugin binding fix with useful live proof, but the new plugin/read fallback can bind auth-required apps when app/list is unavailable, so it needs a narrow auth-readiness guard before merge.

## What This Changes

The PR updates the Codex plugin harness to resolve live config at attempt time, refresh and log Codex app inventory recovery, sanitize app/list diagnostics, and fall back to plugin/read-owned app ids when app inventory is unavailable.

## Best Possible Solution

Keep the live-config and inventory-recovery changes, but only bind plugin/read fallback apps when plugin detail says they do not need auth and add regression coverage for the auth-required fallback case.

## Reproduction Assessment

Yes, at source/proof level: current main fails closed when app/list is unavailable, and the PR body includes a live gateway after-fix proof for Google Calendar. I did not execute a local repro because this review is read-only.

## Solution Assessment

No, not as-is: resolving live config and refreshing inventory are good fits, but the plugin/read fallback should preserve the auth-readiness boundary before it is the best fix.
