## Summary

Keep open. Current main still has Azure Speech as TTS-only, and this PR is a useful plugin-owned implementation, but the latest head is not merge-ready because it leaves a close-during-connect lifecycle race plus missing release/config documentation artifacts.

## What This Changes

The PR adds Azure Speech as a Voice Call realtime transcription provider with provider/session code, manifest metadata, docs, tests, and a Microsoft Speech SDK runtime dependency.

## Best Possible Solution

Fix the Azure session so pending startup is aborted and cleaned up when close() wins the race, commit the regenerated config baseline and a user-facing changelog entry, then proceed with maintainer dependency/live-provider review for the bundled plugin feature.

## Reproduction Assessment

Yes for the review blockers. The close race is reproducible by starting connect(), closing before the SDK start callback resolves, and observing that the PR code continues to create/start the recognizer; the missing artifacts are reproducible from the PR file list plus the manifest schema and changelog policy.

## Solution Assessment

No, the PR is not the best mergeable form as-is. The provider direction is maintainable and plugin-owned, but it needs lifecycle cancellation, generated config docs, and release notes before normal maintainer review can safely decide the dependency and live behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Stop startup when the session has been closed:** `extensions/azure-speech/realtime-transcription-session.ts:245-246`
  - body: Voice Call starts `sttSession.connect()` without awaiting it and calls `close()` when the stream stops. If the call closes while Azure SDK load/start is still pending, `teardown()` sets `closing` before any recognizer exists, but `connect()` continues, starts continuous recognition, sets `connected`, and never closes the new recognizer. Check `closing` after the awaited startup steps and close any created stream/recognizer so aborted calls do not leave Azure recognition running.
  - confidence: 0.89
- **[P2] Regenerate the config docs baseline:** `extensions/azure-speech/openclaw.plugin.json:67-94`
  - body: This manifest adds Azure Speech realtime STT configSchema keys, but the latest PR file list no longer includes `docs/.generated/config-baseline.sha256`. The baseline loader reads bundled plugin manifests from `extensions/`, so `pnpm config:docs:check` should report drift until the hash is regenerated and committed.
  - confidence: 0.92
- **[P2] Add the required changelog entry:** `extensions/azure-speech/index.ts:11`
  - body: Registering a bundled realtime transcription provider is a user-facing feature, but the latest PR file list does not include `CHANGELOG.md`. OpenClaw policy requires a single-line changelog entry for user-facing `feat` changes so release notes do not miss the new provider.
  - confidence: 0.9
