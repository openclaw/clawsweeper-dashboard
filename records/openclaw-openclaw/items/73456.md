## Summary

Keep open. Current main still lacks Azure Speech realtime transcription for Voice Call, and this PR is on the right plugin-owned seam, but the latest head is not merge-ready: it can leave an Azure recognizer running if the media stream closes while `connect()` is still pending, and it is still missing required changelog and generated config baseline updates.

## What This Changes

The PR adds an Azure Speech Voice Call realtime transcription provider with provider/session code, manifest metadata, docs, tests, and a `microsoft-cognitiveservices-speech-sdk` runtime dependency, plus incidental formatting-only changes outside Azure Speech.

## Best Possible Solution

Keep the provider in the Azure Speech plugin, fix the session lifecycle so pending connects cannot outlive `close()`, add the required changelog and regenerated config baseline artifacts, rebase onto current main, then run the focused Azure Speech, Voice Call, config/schema, extension type, and lint checks before maintainer dependency approval.

## Reproduction Assessment

Yes for the review blockers. Source inspection shows current main lacks the provider, the PR session can be closed while Voice Call’s background `connect()` is still pending, and the PR file list lacks the required changelog and generated baseline artifacts.

## Solution Assessment

No, the PR is not the best mergeable form as-is. The plugin-owned provider direction is appropriate, but the narrow maintainable fix is to close the async session lifecycle gap and commit the repository-required release/config artifacts before normal maintainer review.
