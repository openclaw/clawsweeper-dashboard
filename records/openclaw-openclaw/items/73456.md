## Summary

Keep open: the feature is in-scope and current main still lacks Azure Speech realtime transcription, but the PR is not merge-ready because packaged OpenClaw installs would not have the lazy-loaded Speech SDK available at runtime.

## What This Changes

The PR adds an Azure Speech realtime transcription provider for Voice Call with config normalization, session lifecycle tests, live-test coverage, docs, changelog, and a lazy-loaded Microsoft Speech SDK dependency.

## Best Possible Solution

Keep the extension-owned provider approach, internalize the Speech SDK for packaged bundled runtime, rerun changed/package validation, and then proceed with maintainer review.

## Reproduction Assessment

Yes for the packaging regression: source inspection shows packaged bundled plugins skip plugin node_modules and the PR head root package lacks the SDK dependency that the provider lazy-imports. Live Azure transcript quality remains a credentialed validation step, not the blocker found here.

## Solution Assessment

The extension-owned provider design is the right boundary, but this PR is not the best complete solution as-is because the runtime dependency is only declared plugin-locally. The safer fix is to add the SDK to root runtime dependencies as well as the plugin manifest dependency metadata.

## Review Findings

Overall correctness: patch is incorrect
