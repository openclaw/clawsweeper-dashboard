## Summary

Keep open: the provider work is in scope and current main still lacks Azure Speech realtime STT, but the PR is not merge-ready because the new Speech SDK is declared only plugin-locally while packaged bundled plugins do not carry or install plugin-local node_modules.

## What This Changes

The PR adds Azure Speech as a Voice Call realtime transcription provider with plugin registration, manifest/config schema entries, provider/session implementation, tests, live-test coverage, docs, changelog, and a Microsoft Speech SDK dependency.

## Best Possible Solution

Keep the provider extension-owned, make the lazy SDK dependency resolvable in packaged bundled installs, then rerun Azure Speech, Voice Call, dependency-ownership, docs, changed-gate, and package-acceptance validation.

## Reproduction Assessment

Yes for the blocking packaging path: source inspection shows bundled runtime staging skips plugin node_modules and postinstall does not install plugin package dependencies, while the PR declares the SDK only plugin-locally. I did not run package acceptance in this read-only review.

## Solution Assessment

No, not as-is. The extension-owned provider design is the narrow maintainable direction, but the SDK dependency must be root-resolvable for packaged bundled installs or moved behind an explicit downloadable plugin path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Add the Speech SDK to root runtime dependencies:** `extensions/azure-speech/package.json:8`
  - body: The provider lazy-imports `microsoft-cognitiveservices-speech-sdk`, but the PR declares it only in the Azure Speech plugin package. Azure Speech is bundled in the root package, bundled runtime staging skips plugin `node_modules`, and package postinstall does not install plugin package deps, so npm/global installs can select Azure STT and then fail with module-not-found. Please add the SDK to root `package.json`/`pnpm-lock.yaml` and any required dependency ownership metadata, or make this an explicit downloadable plugin path.
  - confidence: 0.9
