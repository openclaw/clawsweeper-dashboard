## Summary

Keep open. Current main still lacks Azure Speech realtime transcription and this PR is in-scope, but the patch remains incorrect because packaged core installs will not resolve the new Speech SDK when Azure STT is selected.

## What This Changes

The PR adds an Azure Speech realtime transcription provider to the bundled Azure Speech plugin, with provider/session code, tests, docs, changelog, manifest schema, and a Microsoft Speech SDK dependency.

## Best Possible Solution

Keep the extension-owned Azure Speech STT implementation, internalize the SDK for packaged OpenClaw runtime, rerun changed and package validation, then continue maintainer review.

## Reproduction Assessment

Yes for the packaging blocker: PR-head source imports the SDK at runtime, root package.json lacks it, Azure Speech is included in the core package, and packaged extension node_modules are omitted. I did not run live Azure STT because the source-level packaging failure is already sufficient.

## Solution Assessment

No. The provider boundary is the right shape, but the complete solution must either add the SDK to root runtime dependencies for the included bundled plugin or externalize/install-manage Azure Speech instead of relying on plugin-local deps in the core package.
