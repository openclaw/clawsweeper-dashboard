## Summary

Keep open. Current main does not implement Azure Speech realtime transcription, and the extension-owned provider direction fits the existing Voice Call provider seam, but the patch is not merge-ready because packaged installs cannot resolve the SDK, close-during-start cleanup can leak Azure resources, and generated plugin inventory docs are stale.

## What This Changes

The PR adds Azure Speech as a Voice Call realtime transcription provider with plugin registration, manifest/config schema, provider/session code, unit/live tests, docs, changelog, and the Microsoft Speech SDK dependency.

## Best Possible Solution

Keep the provider extension-owned, make its SDK dependency resolvable in packaged bundled installs, preserve async cleanup until Azure resources are actually closed, and regenerate generated plugin inventory docs.

## Reproduction Assessment

Yes. Source inspection reproduces the blocking paths: packaged installs exclude plugin node_modules while the PR only adds the SDK plugin-locally, and the close-during-start path clears Azure SDK refs before the async stop callback can close them.

## Solution Assessment

No, not as-is. The provider design is the narrow maintainable direction, but the SDK needs root-resolvable packaging/ownership metadata and the session cleanup must retain resources until stop completion.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Add the Speech SDK to root runtime dependencies:** `extensions/azure-speech/package.json:8`
  - body: The provider lazy-imports `microsoft-cognitiveservices-speech-sdk`, but the PR declares it only in the Azure Speech plugin package. Bundled packaged installs exclude plugin `node_modules`, and postinstall does not install plugin package dependencies, so selecting Azure STT from a normal install can fail with module-not-found. Add the SDK to root runtime dependencies plus ownership metadata, or make this an explicit external plugin path.
  - confidence: 0.92
- **[P2] Keep Azure resources until the stop callback closes them:** `extensions/azure-speech/realtime-transcription-session.ts:282-283`
  - body: When `close()` lands while `startContinuousRecognitionAsync` is pending, this branch registers `closeStreamAndRecognizer` as the stop callback and then immediately clears `recognizer` and `pushStream`. The SDK invokes stop completion through promise callbacks, so the later callback can see `undefined` and skip closing the stream/recognizer. Capture the resources or clear them only after the stop callback/error path closes them, and make the regression test use an async stop callback.
  - confidence: 0.87
- **[P2] Regenerate plugin inventory docs for the new contract:** `extensions/azure-speech/openclaw.plugin.json:28`
  - body: The manifest now advertises `realtimeTranscriptionProviders`, but the generated plugin inventory/reference docs are not updated in the PR and still list Azure Speech as TTS-only on main. `pnpm plugins:inventory:check` is part of release checks, so regenerate and commit the generated plugin docs after the manifest change.
  - confidence: 0.86
