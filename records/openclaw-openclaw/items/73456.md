## Summary

Keep open: the Azure Speech realtime transcription provider is an in-scope extension-owned addition, but the PR is still not merge-ready because packaged OpenClaw will not install the SDK where the bundled runtime can resolve it.

## What This Changes

The branch adds an Azure Speech realtime transcription provider for Voice Call, with provider/session code, tests, docs, changelog, manifest metadata, and a plugin-local Microsoft Speech SDK dependency.

## Best Possible Solution

Keep the extension-owned Azure provider design, internalize the Speech SDK for packaged bundled runtime, rerun dependency/package validation, and then continue maintainer review.

## Reproduction Assessment

Yes for the packaging regression: PR head lazy-imports microsoft-cognitiveservices-speech-sdk, root package.json lacks it, and packaged bundled plugins skip plugin node_modules. Selecting Azure STT from a global/package install should fail with module-not-found until the SDK is root-internalized.

## Solution Assessment

No: the provider boundary is right, but the dependency placement is incomplete for packaged runtime. The safer fix is to keep the plugin-local declaration and also add the SDK to root dependencies/lock as an internalized bundled runtime dependency.
