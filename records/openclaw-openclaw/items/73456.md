## Summary

Keep open. Current main still lacks Azure Speech as a Voice Call realtime transcription provider, and the plugin-owned implementation direction matches the existing provider registry. The latest head appears to have fixed the earlier Greptile lazy-connect frame-loss issue, but the PR remains not merge-ready because the session lifecycle can leave Azure recognizer resources running after a pending connect is closed, and required generated config baseline plus changelog artifacts are absent from the PR file list.

## What This Changes

The PR adds an Azure Speech realtime transcription provider for Voice Call, with provider/session code, manifest metadata, docs, tests, and a Microsoft Speech SDK runtime dependency.

## Best Possible Solution

Keep the Azure Speech realtime STT provider in the Azure Speech plugin, repair the pending-connect close path so closed sessions cannot become connected or leak SDK resources, include the generated config baseline and changelog entry, then land after focused Azure Speech, Voice Call, config/schema, extension type, lint, and dependency review gates are clean.

## Reproduction Assessment

Yes for the review blockers. Source inspection shows Voice Call starts transcription connect in the background and the PR session does not re-check closed state while async connect completes; GitHub's PR file list also verifies the required changelog and generated config baseline files are absent.

## Solution Assessment

No, the PR is not the best mergeable form as-is. The feature belongs in the Azure Speech plugin, but the maintainable solution needs the session lifecycle guard plus repository-required release/config artifacts before normal maintainer review.
