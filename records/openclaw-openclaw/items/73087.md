## Summary

Current main already implements the useful macOS LaunchAgent PATH behavior: macOS defaults use a canonical system PATH that excludes shell version-manager/user dirs, and the shipped fix covers the false required-dir warning path. The PR branch is now stale and has review findings in its diagnostic wording, so it is not useful to merge as-is.

## What This Changes

The PR filters macOS gateway service PATH missing-dir audits for shell version-manager entries, changes the missing-dir diagnostic wording/detail, and adds macOS LaunchAgent PATH audit tests.

## Best Possible Solution

Keep the current shipped/current-main service PATH contract and close this stale branch rather than merging its conflicting diagnostic wording.

## Reproduction Assessment

No current-main failing reproduction remains. Source inspection shows current main omits macOS user/version-manager dirs from the default LaunchAgent PATH, and regression tests accept the canonical macOS PATH without gatewayPathMissingDirs.

## Solution Assessment

Yes. Current main is the cleaner solution because it changes the macOS service PATH contract at the source and leaves Linux diagnostics platform-neutral, while this PR filters later and changes wording globally.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep Linux PATH wording platform-neutral:** `src/daemon/service-audit.ts:476`
  - body: This message is emitted for all platforms, but on Linux the missing list can still contain env-configured version-manager paths such as NVM_DIR or FNM_DIR. Calling those entries non-version-manager dirs would make Linux diagnostics self-contradictory.
  - confidence: 0.92
- **[P2] Align the macOS test with existence-gated fallbacks:** `src/daemon/service-audit.test.ts:136-140`
  - body: The new expected message includes pnpm fallback paths without configuring env or directory existence, but the service path resolver only adds hard-coded macOS pnpm/fnm fallback dirs when those dirs exist. That makes the test brittle on normal CI.
  - confidence: 0.86
- **[P3] Constrain nvm matching to a path segment:** `src/daemon/service-audit.ts:420`
  - body: The helper matches `/.nvm` without requiring a segment boundary, unlike the other markers. A path like `/Users/test/.nvmfoo` would be skipped from the missing-dir audit even though it is not under `.nvm`.
  - confidence: 0.78
