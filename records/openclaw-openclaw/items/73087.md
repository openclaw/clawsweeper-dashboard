## Summary

Current main already implements the intended macOS LaunchAgent PATH audit behavior through the shared service PATH contract, while the PR branch is now stale and still carries review defects that would regress Linux wording if merged as-is.

## What This Changes

The PR filters macOS gateway service PATH missing-dir audits to skip version-manager entries, changes the missing-dir diagnostic wording/detail, and adds macOS LaunchAgent PATH audit tests.

## Best Possible Solution

Close this PR and keep current main's canonical macOS LaunchAgent PATH/audit behavior; any remaining copy tweak should be handled as a separate narrow issue only if missing system-dir wording is still misleading.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: on current main, a darwin audit expects only canonical system PATH entries and no longer reports absent user/version-manager dirs, while actual version-manager entries in the service PATH still trigger the non-minimal warning.

## Solution Assessment

Yes. Current main is the better solution because it fixes the macOS false-positive at the shared service-env/audit contract and preserves Linux's platform-neutral missing-dir semantics instead of applying a global non-version-manager label.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep Linux PATH wording platform-neutral:** `src/daemon/service-audit.ts:476`
  - body: The new message is emitted for all platforms, but Linux `expectedForMissing` remains the full expected PATH and can include env-configured NVM/FNM roots. A missing `/opt/nvm/current/bin` would be labeled a `non-version-manager` dir, which regresses the Linux diagnostic.
  - confidence: 0.92
- **[P2] Align the macOS test with existence-gated fallbacks:** `src/daemon/service-audit.test.ts:136-140`
  - body: The new assertion expects pnpm fallback dirs without configuring env or directory existence, but the service path resolver only adds those hard-coded macOS fallbacks when they exist. On normal CI those paths are absent, so this test can fail instead of documenting the resolver contract.
  - confidence: 0.86
- **[P3] Constrain nvm matching to a path segment:** `src/daemon/service-audit.ts:420`
  - body: The helper matches `/.nvm` without a segment boundary, unlike the other markers. A path such as `/Users/test/.nvmfoo` would be treated as version-manager-managed and skipped from the missing-dir audit even though it is not under `.nvm`.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `src/daemon/service-audit.ts:476`
  - body: This changes a user-facing gateway diagnostic, so the branch would need an Unreleased changelog entry before it could merge. The branch diff only touches the audit implementation and tests.
  - confidence: 0.9
