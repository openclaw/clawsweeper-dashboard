## Summary

Keep open. Current main still preserves user-supplied model casing on the local and gateway `infer model run` paths, and open closing PRs already target this bug.

## What This Changes

The issue asks OpenClaw to reject or canonicalize case-mismatched `--model <provider>/<model>` values before provider dispatch.

## Best Possible Solution

Land or consolidate a catalog-aware fix that sends canonical ids for unique known model matches on both transports, rejects true unknowns clearly, and adds focused regression coverage.

## Reproduction Assessment

Yes. The issue provides a deterministic CLI matrix, and current-main source inspection confirms mixed-case model values can still flow to local and gateway dispatch without live provider validation.

## Solution Assessment

Yes, catalog-aware canonicalization before dispatch is the narrowest maintainable direction. Blind lowercasing is riskier because third-party or custom model ids may be intentionally case-sensitive.
