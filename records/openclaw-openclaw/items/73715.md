## Summary

Keep open. Current main still forwards user-supplied model casing through the `infer model run` local and gateway paths, and two open unmerged PRs already target the same bug, so this should stay open until maintainers choose and land one fix.

## What This Changes

The issue asks OpenClaw to reject or canonicalize case-mismatched `infer model run --model <provider/model>` model IDs before dispatch so users do not see misleading provider no-output errors.

## Best Possible Solution

Land one consolidated fix that canonicalizes unique known catalog matches before both local and gateway dispatch, preserves or clearly rejects unmatched custom refs, and adds focused regression coverage in `src/cli/capability-cli.test.ts`.

## Reproduction Assessment

Yes. The issue provides a deterministic CLI matrix, and current-main source inspection confirms the mixed-case model value still reaches local and gateway dispatch without needing a live provider run.

## Solution Assessment

Unclear until a PR is selected. Catalog-aware canonicalization before dispatch is safer than blanket lowercasing because it avoids changing unknown custom model ids that may be case-sensitive.
