## Summary

Keep open. Current main still preserves user-supplied model casing on both local and gateway `infer model run` paths, and open closing PRs #73717 and #74945 already target the bug, so this should remain open until one fix lands or the PRs are consolidated.

## What This Changes

The issue requests that case-mismatched `infer model run --model` refs be canonicalized or rejected before provider dispatch instead of surfacing a misleading empty-output provider error.

## Best Possible Solution

Land one consolidated fix that canonicalizes unique known catalog matches before both local and gateway dispatch, preserves or deliberately rejects unknown custom refs, and adds focused mixed-case model-ref regression coverage.

## Reproduction Assessment

Yes. The issue provides a deterministic CLI matrix, and current-main source inspection confirms mixed-case model values still flow to local and gateway dispatch without a live provider run.

## Solution Assessment

Unclear for the open PRs until maintainers choose an approach. Catalog-aware canonicalization looks safer than unconditional lowercasing because it fixes known models without silently rewriting arbitrary custom ids.
