## Summary

Keep open. Current main still preserves the user-supplied model casing through both local and gateway `infer model run` paths, and open closing PRs #73717 and #74945 already target this bug, so the issue should remain open until one consolidated fix lands.

## What This Changes

Reject or canonicalize case-mismatched `infer model run --model` ids before provider dispatch so users do not get misleading no-output errors.

## Best Possible Solution

Land one catalog-aware fix that canonicalizes unique known model matches before both local and gateway dispatch, while preserving auth-profile suffixes and custom mixed-case refs.

## Reproduction Assessment

Yes. The issue provides a deterministic CLI matrix, and current-main source inspection confirms mixed-case model values still flow to local and gateway dispatch without a live provider run.

## Solution Assessment

Yes for fixing this at the CLI/catalog boundary. Catalog-aware unique-match canonicalization is safer than unconditional lowercasing because it fixes known catalog models without silently rewriting arbitrary custom ids.
