## Summary

Keep open. Current main still preserves the user-supplied model casing in both local and gateway `infer model run` paths, and the issue is paired with two open unmerged closing PRs, so it should stay open until maintainers choose and land a fix.

## What This Changes

The issue requests that case-mismatched `--model <provider>/<model>` refs in `infer model run` be canonicalized or rejected before provider dispatch.

## Best Possible Solution

Land one consolidated fix that either canonicalizes unique known catalog matches before both local and gateway dispatch or rejects case-only mismatches before provider calls, with focused regression coverage in `src/cli/capability-cli.test.ts` and preservation of unmatched custom refs unless maintainers explicitly choose a stricter policy.

## Reproduction Assessment

Yes. The issue provides a deterministic CLI matrix, and current-main source inspection confirms the mixed-case model value is still forwarded through local and gateway paths without needing a live provider call.

## Solution Assessment

Unclear because no fix is landed yet. A catalog-aware canonicalization or fail-fast unknown-model check is safer than blanket lowercasing all model IDs, especially for custom providers whose IDs may be case-sensitive.
