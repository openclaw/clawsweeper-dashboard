## Summary

Keep open. Current main still preserves user-supplied model casing on the local and gateway `infer model run` paths, and open closing PRs already target this bug.

## What This Changes

The issue asks `infer model run --model <provider>/<model>` to canonicalize or reject case-mismatched model ids before provider dispatch.

## Best Possible Solution

Land one consolidated fix that canonicalizes unique known catalog matches before both local and gateway dispatch while preserving or clearly rejecting unknown custom refs.

## Reproduction Assessment

Yes. The issue provides a deterministic CLI matrix, and current-main source inspection confirms mixed-case model values still flow to dispatch without a live provider run.

## Solution Assessment

No, current main is not the best solution because it still forwards raw casing. A catalog-aware canonicalization fix, such as #73717 or an equivalent, is safer than blindly lowercasing every custom model string.
