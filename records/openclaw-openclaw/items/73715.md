## Summary

Keep open. Current main still preserves user-supplied model casing on the `infer model run` local and gateway paths, and two open closing PRs already target the bug, so this should remain open until maintainers land or consolidate a fix.

## What This Changes

Handle case-mismatched `infer model run --model <provider>/<model>` ids by canonicalizing known models or rejecting them before provider dispatch.

## Best Possible Solution

Land one consolidated catalog-aware fix that canonicalizes unique known model matches before both local and gateway dispatch, or rejects case-only mismatches before provider calls, with focused regression coverage.

## Reproduction Assessment

Yes. The report provides a deterministic CLI matrix, and current-main source inspection confirms mixed-case model values still flow toward local and gateway dispatch without a live provider run.

## Solution Assessment

Yes for the requested direction, but unconditional lowercasing is not the safest universal policy. Catalog-aware unique-match canonicalization, or a fail-fast `Unknown model` error for case-only mismatches, preserves custom ids better.
