## Summary

Keep open. Current main still preserves caller-supplied model casing on both local and gateway `infer model run` paths, and open closing PRs already target the same bug.

## What This Changes

Reject or canonicalize case-mismatched `infer model run --model` model ids before dispatch so users get a clear model-resolution result instead of a downstream no-text provider error.

## Best Possible Solution

Land one consolidated fix that canonicalizes unique known catalog matches, or rejects case-only mismatches with `Unknown model`, before both local and gateway dispatch, with focused regression coverage in `src/cli/capability-cli.test.ts`.

## Reproduction Assessment

Yes. The issue includes a deterministic CLI matrix, and source inspection on current main confirms mixed-case model values still flow into local and gateway dispatch without canonicalization.

## Solution Assessment

Unclear for the open PRs until review. Catalog-aware canonicalization is the narrowest maintainable direction because it fixes known model refs without silently rewriting arbitrary custom provider model ids.
