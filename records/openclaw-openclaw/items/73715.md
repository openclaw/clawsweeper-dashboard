## What This Changes

The issue requests that `infer model run --model <provider>/<model>` either canonicalize case-only matches for known catalog model IDs or reject mismatched model casing before dispatching to a provider.

## Best Possible Solution

Land one consolidated fix that canonicalizes unique known catalog matches before both local and gateway dispatch, preserves unmatched custom refs, and adds focused `src/cli/capability-cli.test.ts` coverage for mixed-case provider/model inputs and custom mixed-case preservation.

## Reproduction Assessment

Yes. The issue provides a deterministic CLI matrix, and current-main source inspection confirms the raw mixed-case model ID is still forwarded on the local and gateway paths without needing a live provider run.

## Solution Assessment

Unclear until a PR is selected. The narrowest maintainable direction is catalog-aware canonicalization for unique known matches; blanket lowercasing is riskier because some provider docs and custom provider IDs can be case-sensitive.
