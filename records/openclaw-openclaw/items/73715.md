## What This Changes

The issue requests that `infer model run --model <provider>/<model>` either canonicalize case-only model-id mismatches before provider dispatch or reject them upfront with a clear unknown-model error.

## Best Possible Solution

Land one consolidated fix that canonicalizes only unique known catalog matches before both local and gateway dispatch, preserves unmatched custom refs, and adds focused regression coverage around the CLI override-to-dispatch seam.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. The issue provides a deterministic CLI matrix, and current source inspection confirms the raw mixed-case model id can still be forwarded without needing a live provider run.

## Solution Assessment

Is this the best way to solve the issue? Unclear until a PR is selected. The safer maintainable direction is catalog-aware canonicalization or clear upfront rejection, not unconditional lowercasing of every CLI model id.
