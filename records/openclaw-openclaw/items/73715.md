## Summary

Keep open. Current main still forwards the user-supplied model casing on both local and gateway `infer model run` paths, and two open closing PRs already target the bug, so the issue should remain open until maintainers land or consolidate a fix.

## What This Changes

The issue asks for `infer model run --model <provider>/<model>` to either canonicalize case-only model-id mismatches before dispatch or reject them upfront with a clear unknown-model error.

## Best Possible Solution

Land one consolidated catalog-aware fix that canonicalizes unique known model matches before both local and gateway dispatch, or rejects case-only mismatches before provider calls, with focused regression coverage.

## Reproduction Assessment

Yes. The issue provides a deterministic CLI matrix, and current-main source inspection confirms mixed-case model values still flow to local and gateway dispatch without a live provider run.

## Solution Assessment

Yes for the requested direction, but the exact PR should be chosen carefully. Catalog-aware unique-match canonicalization is safer than unconditional lowercasing because it fixes known models without silently rewriting arbitrary custom ids.
