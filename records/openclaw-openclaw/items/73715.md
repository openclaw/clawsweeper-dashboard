## Summary

Keep open. Current main still preserves the user-supplied model casing on both local and gateway `infer model run` paths, and two open unmerged PRs already target this issue with closing syntax, so maintainers should review and consolidate those candidates rather than close the report now.

## What This Changes

The issue asks for `infer model run --model <provider>/<model>` to either canonicalize case-only model id mismatches before dispatch or reject them upfront with a clear unknown-model error.

## Best Possible Solution

Land one consolidated fix that canonicalizes unique known catalog matches before both local and gateway dispatch, while preserving or clearly rejecting unmatched custom refs, and add focused regression coverage in `src/cli/capability-cli.test.ts`.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. The issue provides a deterministic CLI matrix, and current source inspection confirms the raw mixed-case model id is still forwarded without needing a live provider run.

## Solution Assessment

Is this the best way to solve the issue? Unclear until a PR is selected. The safer maintainable direction is unique catalog-match canonicalization; blanket lowercasing is riskier for custom providers or model ids that intentionally use mixed case.
