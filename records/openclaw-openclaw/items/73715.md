## Summary

Keep open. Current main still forwards user-supplied model casing through both local and gateway `infer model run` paths, and two open closing PRs already target this bug, so the issue should stay open until maintainers land or consolidate a fix.

## What This Changes

Reject or canonicalize case-mismatched `--model` model ids before `infer model run` dispatch so users do not see provider-level empty-output errors.

## Best Possible Solution

Land one consolidated catalog-aware fix that canonicalizes unique known model matches before both local and gateway dispatch, or rejects case-only mismatches before provider calls, with focused regression coverage.

## Reproduction Assessment

Yes. The issue provides a deterministic CLI matrix, and current-main source inspection confirms mixed-case model values still flow to local and gateway dispatch without a live provider run.

## Solution Assessment

Yes for fixing this at the CLI/catalog boundary. Catalog-aware unique-match canonicalization is safer than unconditional lowercasing because it fixes known models without silently rewriting arbitrary custom ids.
