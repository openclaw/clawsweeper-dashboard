## Summary

Keep open. Current main still preserves user-supplied model casing on both the local and gateway `infer model run` paths, and two open unmerged closing PRs already target this bug, so this issue should remain open until maintainers land or consolidate one fix.

## What This Changes

The issue requests fail-fast or canonicalized handling for mixed-case `infer model run --model` model IDs so case-mismatched catalog models do not reach providers as misleading no-output errors.

## Best Possible Solution

Land one consolidated fix that canonicalizes unique known catalog matches before both local and gateway dispatch, preserves or rejects unknown custom refs deliberately, and adds focused regression coverage for mixed-case model refs.

## Reproduction Assessment

Yes. The issue provides a deterministic CLI matrix, and current-main source inspection confirms mixed-case model values still flow to local and gateway dispatch without a live provider run.

## Solution Assessment

Unclear for the open PRs until maintainers choose an approach. Catalog-aware canonicalization is the narrowest maintainable direction because it fixes known models without silently rewriting arbitrary custom ids.
