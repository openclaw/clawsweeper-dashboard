## Summary

Keep open. Current main still preserves the user-supplied model casing on the local and gateway `infer model run` paths, and open unmerged PRs #73717 and #74945 already target this issue, so the report should stay open until maintainers choose and land a consolidated fix.

## What This Changes

The issue requests that `openclaw infer model run --model <provider>/<model>` either canonicalize case-only model ID mismatches before dispatch or reject them upfront with a clear unknown-model error.

## Best Possible Solution

Land one consolidated fix that canonicalizes unique known catalog matches before both local and gateway dispatch, while preserving or clearly rejecting unmatched custom refs, and add focused regression coverage in `src/cli/capability-cli.test.ts`.

## Reproduction Assessment

Yes. The issue provides a deterministic CLI matrix, and current source inspection confirms the raw mixed-case model id is still forwarded without needing a live provider run.

## Solution Assessment

Unclear until a PR is selected. The safer maintainable direction is unique catalog-match canonicalization; blanket lowercasing is riskier for custom providers or model ids that intentionally use mixed case.
