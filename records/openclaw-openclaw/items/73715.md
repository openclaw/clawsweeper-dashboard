## Summary

Keep open. Current main still preserves the user-supplied model casing on both local and gateway `infer model run` paths, and two open unmerged closing PRs already target this bug.

## What This Changes

The issue asks OpenClaw to canonicalize or reject case-mismatched `infer model run --model <provider/model>` values before dispatch so users do not get misleading provider no-output errors.

## Best Possible Solution

Land one consolidated fix, preferably catalog-aware canonicalization for unique known matches before both local and gateway dispatch, with focused regression coverage for mixed-case known model refs and unknown custom refs.

## Reproduction Assessment

Yes. The issue provides a deterministic CLI matrix, and current-main source inspection confirms mixed-case model values still flow into local and gateway dispatch without catalog-aware canonicalization.

## Solution Assessment

Unclear for the open PRs until maintainers choose the exact semantics. Catalog-aware canonicalization is the narrowest maintainable direction because it fixes known catalog models without silently rewriting arbitrary custom model ids.
