## Summary

Keep open for normal PR review: current main does not externalize the Feishu/Lark SDK, and this one-line patch matches the existing tsdown `neverBundle` mechanism with no definite blocking correctness issue found.

## What This Changes

The PR adds `@larksuiteoapi/node-sdk` to `tsdown.config.ts`'s explicit `neverBundle` dependency list so tsdown treats the Feishu/Lark SDK as an external runtime dependency.

## Best Possible Solution

Merge the targeted externalization if CI/package validation confirms the Feishu plugin package resolves its declared SDK dependency at runtime; a direct tsdown-config assertion for the new dependency would make the guardrail stronger.

## Reproduction Assessment

Partial: current main's `neverBundle` predicate cannot match `@larksuiteoapi/node-sdk` because the dependency is absent from the explicit list, and the PR patch would make that static check pass. There is not a high-confidence artifact reproduction in the PR because no failing packaged build or runtime log was provided.

## Solution Assessment

Yes for the stated build boundary: adding the SDK to the existing `deps.neverBundle` allowlist is the narrowest change and avoids changing plugin packaging or dependency ownership. The safer follow-through is targeted CI/package validation, optionally with a direct test assertion for this dependency.
