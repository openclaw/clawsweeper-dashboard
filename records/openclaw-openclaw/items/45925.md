## Summary

Keep this PR open, but do not merge it as-is. Current `main` still has the cron-session heartbeat targeting bug, and the PR fixes the right surface, but the latest diff drops the originating agent target for cron-originated exec wakes and can fan out heartbeat runs to unrelated agents.

## What This Changes

The PR changes heartbeat wake scoping and forced heartbeat session resolution so cron session keys fall back to normal heartbeat sessions, with helper and `runHeartbeatOnce` regression tests.

## Best Possible Solution

Land a revised fix that keeps heartbeat prompts out of isolated cron run sessions while preserving the originating agent boundary, with multi-agent regression coverage and a changelog entry.

## Reproduction Assessment

Yes, at source level. Current `main` forwards cron session keys from exec completion into heartbeat targeting, and the PR diff changes that helper to return no `sessionKey` or `agentId`, which the runner treats as an unscoped wake.

## Solution Assessment

No. Blocking direct cron-session heartbeat execution is the right direction, but dropping all target context is not the narrowest maintainable fix; the patch should preserve the parsed agent target or map to that agent's main heartbeat session.
