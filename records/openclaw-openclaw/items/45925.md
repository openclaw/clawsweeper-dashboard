## Summary

Keep open: the PR targets a real main-branch heartbeat/session bug, but the current diff still loses the originating agent target when it strips cron session keys, so a cron exec wake can become an unscoped heartbeat wake across enabled agents.

## What This Changes

The PR changes heartbeat wake scoping and forced heartbeat session resolution so cron session keys are not used as heartbeat targets, with focused helper and heartbeat fallback tests.

## Best Possible Solution

Revise the patch so heartbeat never targets isolated cron run sessions while preserving the originating agent boundary, with multi-agent regression coverage for cron exec wake routing.

## Reproduction Assessment

Yes. Source inspection plus the PR diff give a high-confidence path: a cron session key passed through the changed helper produces no target, and the existing runner treats no `sessionKey`/`agentId` as a broadcast over enabled heartbeat agents.

## Solution Assessment

No. Blocking direct cron-session heartbeat execution is the right direction, but dropping all target context is not the narrowest maintainable fix; preserving the parsed agent target or mapping to that agent's main heartbeat session is safer.
