## Summary

Keep open: current main still allows cron session keys through heartbeat wake scoping, and this PR targets a real runtime bug, but the proposed helper change drops all wake target context for cron exec completions and can turn an agent-scoped wake into an unscoped multi-agent heartbeat run.

## What This Changes

The PR changes heartbeat wake scoping and forced heartbeat session resolution so cron session keys are not used as heartbeat targets, with focused regression tests.

## Best Possible Solution

Route cron-originated heartbeat wakes to the originating agent's normal heartbeat session or agent target, never the isolated cron run session, and cover the multi-agent exec wake case before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the PR drops the cron `sessionKey`, exec completion does not pass `agentId`, and the runner treats a wake with neither target as a fan-out over enabled agents.

## Solution Assessment

No. Blocking direct heartbeat execution inside isolated cron run sessions is the right direction, but dropping all target context is not the narrowest maintainable fix; preserve the parsed agent target or map to that agent's main heartbeat session.

## Review Findings
