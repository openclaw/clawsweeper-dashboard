## Summary

Close as superseded: a newer open PR covers the same cron-run heartbeat problem with the missing agent-scope preservation and broader enqueue-site coverage, while this patch still drops wake targeting for cron-originated exec completions.

## What This Changes

The branch drops cron session keys from heartbeat wake options, ignores forced cron session keys during heartbeat resolution, and adds focused helper and heartbeat fallback tests.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/80214 as the canonical fix, and avoid merging this stale branch because its remaining useful intent is covered there.

## Reproduction Assessment

Yes. Source inspection shows exec completion forwards only the session key through scopedHeartbeatWakeOptions, this PR drops that target for cron keys, and the runner treats a wake with no sessionKey or agentId as an all-agent wake.

## Solution Assessment

No. Blocking direct cron-run heartbeat targeting is the right direction, but this patch drops the originating agent boundary; https://github.com/openclaw/openclaw/pull/80214 uses the safer remap-to-durable-queue approach.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve agent scope when dropping cron keys:** `src/routing/session-key.ts:46`
  - body: Returning wakeOptions unchanged for cron session keys removes the only targeting data exec completion passes to requestHeartbeat. Without sessionKey or agentId, startHeartbeatRunner treats the wake as unscoped and can run heartbeat for every enabled agent instead of the originating cron job's agent.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff has no dependency or CI supply-chain changes, but it introduces a concrete cross-agent heartbeat routing risk.

Concerns:

- **[medium] Cron wake can lose agent isolation:** `src/routing/session-key.ts:46`
  - body: Dropping a cron session key without replacing it with agentId or an agent-main session key can turn a cron-originated exec completion into an unscoped wake that runs unrelated agents' heartbeat prompts and sends.
  - confidence: 0.88
