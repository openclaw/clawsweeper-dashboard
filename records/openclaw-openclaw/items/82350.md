## Summary

Close: the reported Codex hook-relay bypass matches a deprecated feature flag that was replaced on current main by PR 82078; the fix is not in v2026.5.12, but main now sends the stable hook flag and PreToolUse relay config through the Codex thread config.

## What This Changes

The issue reports that Codex-harness agents do not receive `hooks.PreToolUse`, causing plugin `before_tool_call` enforcement to be bypassed.

## Best Possible Solution

Keep the current main fix, ship it in the next release, and leave the separate Anthropic-harness report at https://github.com/openclaw/openclaw/issues/76201 open for its distinct runtime path.

## Reproduction Assessment

Yes. The report’s v2026.5.12 symptom matches the old `features.codex_hooks` key that current main replaced with `features.hooks`, and current tests assert `hooks.PreToolUse` reaches `thread/start`; I did not live-run the installed harness in this read-only review.

## Solution Assessment

Yes. Switching the generated Codex thread config to the stable `features.hooks` flag is the narrow maintainable fix for this transport drop, and it aligns with the Codex 0.130.0 app-server contract.
