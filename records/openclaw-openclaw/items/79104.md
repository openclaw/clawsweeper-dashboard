## Summary

Keep open: current main still suppresses the accepted-message early typing cue for Discord guild/channel messages, while the group-channel docs say instant typing should apply. The related open fix PR is the right implementation path, but it is not merged yet, so this issue remains actionable.

## What This Changes

The issue asks for accepted Discord guild/channel messages with `typingMode: "instant"` to show native typing promptly instead of waiting several seconds.

## Best Possible Solution

Land a maintainer-approved Discord typing lifecycle fix that starts accepted guild-channel typing feedback according to `typingMode` and keeps cleanup/refresh behavior covered by focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: an accepted guild preflight context reaches `queueAcceptedDiscordTypingCue`, the helper rejects guild messages, and the queue test asserts no early typing call while processing continues.

## Solution Assessment

Yes. The narrowest maintainable fix is in the Discord accepted typing lifecycle/gate and its queue/process tests, not in core debounce configuration or a new user setting.
