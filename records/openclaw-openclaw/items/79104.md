## Summary

Keep open: current main still excludes accepted Discord guild messages from the early typing cue, while the group-channel docs say instant typing should apply for group/tool-only turns, so the reported silent pre-dispatch gap remains plausible and actionable.

## What This Changes

Request that Discord guild/channel messages send an immediate typing indicator when `agents.defaults.typingMode` is `instant` and no inbound debounce is configured.

## Best Possible Solution

Honor `typingMode: instant` for accepted Discord guild/channel messages by sending the same best-effort early typing cue used for DMs, while retaining suppression for rejected, empty, aborted, and non-instant turns.

## Reproduction Assessment

Yes from source inspection, though not live: an accepted guild preflight context reaches the early typing cue call, but `shouldSendAcceptedDiscordTypingCue` rejects guild messages and the queue test asserts that no typing is sent. The report's timing gap is consistent with that missing pre-run signal.

## Solution Assessment

Yes, the narrowest maintainable fix is in the Discord accepted-typing gate and its queue tests, not in core typing mode or debounce settings. Core already starts typing at agent-run start, while this issue concerns the earlier accepted-message liveness cue before queued processing begins.
