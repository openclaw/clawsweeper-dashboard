## Summary

Keep open: current main still blocks the accepted Discord pre-dispatch typing cue for guild messages, and the related fix PR is still unmerged. The docs and recent Discord confirmations support this as a real plugin bug, not cleanup-ready stale work.

## What This Changes

Discord guild/channel messages should show an immediate typing indicator once accepted when `typingMode: "instant"` is configured.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/76091 or an equivalent Discord plugin fix that starts and carries accepted typing feedback for guild-channel messages while preserving `typingMode` opt-outs and cleanup paths.

## Reproduction Assessment

Yes from source inspection, though not from an independent live Discord run. An accepted guild preflight context reaches `queueAcceptedDiscordTypingCue`, but the helper rejects guild contexts and the queue test asserts no early typing call.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Discord accepted-typing lifecycle, not in core debounce or new configuration, and the open related PR is the active implementation path.
