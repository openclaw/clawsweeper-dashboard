## Summary

Keep open: current main and the latest tagged release still suppress the Discord accepted guild-channel early typing cue before checking `typingMode`, and the related Discord typing lifecycle PR is open rather than merged.

## What This Changes

The issue reports delayed Discord guild-channel typing indicators despite `agents.defaults.typingMode = "instant"` and no configured inbound debounce.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/76091 with a focused Discord plugin fix that starts accepted guild typing feedback for `typingMode: "instant"` while preserving rejected, aborted, empty, and opt-out cases.

## Reproduction Assessment

Yes from source inspection: an accepted guild preflight context reaches `queueAcceptedDiscordTypingCue`, but the helper rejects guild contexts before checking `typingMode`, and the current queue test asserts no early typing call. I did not run a live Discord guild reproduction in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is to fix the Discord accepted-typing lifecycle, preferably through the existing related PR or a smaller replacement, rather than adding new configuration or changing core typing policy.
