## Summary

Keep open: current main still blocks accepted Discord guild messages from the early typing cue despite group typing docs, and the active fix is the open Discord typing lifecycle PR rather than an already-merged change.

## What This Changes

Discord guild/channel messages should show immediate typing feedback when accepted with `typingMode: "instant"`, instead of waiting silently before the reply pipeline starts.

## Best Possible Solution

Land a reviewed Discord-owned typing lifecycle fix that starts accepted guild typing feedback after preflight and carries or cleans it through queued processing without changing global typing-mode semantics.

## Reproduction Assessment

Yes from source inspection. An accepted guild preflight context reaches `queueAcceptedDiscordTypingCue`, but current main rejects guild messages before checking `typingMode`, and the queue test asserts that no early typing call is sent.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Discord plugin typing lifecycle and tests; https://github.com/openclaw/openclaw/pull/76091 is the active implementation path to review rather than opening a duplicate repair.
