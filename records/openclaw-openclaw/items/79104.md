## Summary

Keep open: current main still rejects accepted Discord guild messages from the early typing cue and the queue test asserts that suppression, while group-channel docs say instant typing should apply. This remains a narrow, actionable Discord plugin bug rather than a cleanup close.

## What This Changes

The issue asks for Discord guild/channel messages with `typingMode: "instant"` to show a native typing indicator immediately after the message is accepted for processing.

## Best Possible Solution

Honor `typingMode: "instant"` for accepted Discord guild/channel messages in the Discord plugin’s early typing gate, while leaving core typing semantics and refresh/cleanup behavior unchanged.

## Reproduction Assessment

Yes from source inspection, though not from a live Discord run: an accepted guild preflight context reaches `queueAcceptedDiscordTypingCue`, but the helper rejects guild messages and the queue test asserts no early typing call. That is a high-confidence current-main reproduction of the missing pre-dispatch cue.

## Solution Assessment

Yes. The narrow maintainable fix is in the Discord accepted-typing gate and its queue tests, not in core debounce or typing-mode configuration.
