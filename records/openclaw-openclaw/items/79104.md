## Summary

Keep open: current main still excludes Discord guild/channel contexts from the accepted-message early typing cue, while the docs say group typing can be instant and an active implementation PR is still unmerged.

## What This Changes

Request that accepted Discord guild/channel messages honor `typingMode: "instant"` by showing typing promptly instead of waiting for later reply processing.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/76091 or an equivalent narrow Discord typing lifecycle fix, then close this issue once the change reaches main.

## Reproduction Assessment

Yes from source inspection, though not from a live Discord run: accepted guild preflight reaches the early typing helper, but that helper rejects guild contexts before checking `typingMode`. The current queue test also asserts this suppression.

## Solution Assessment

Yes. The maintainable fix belongs in the Discord accepted typing lifecycle and its queue/process tests, not in a new config option or broad core debounce change.
