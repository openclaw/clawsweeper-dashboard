## Summary

Keep open: the PR targets a real Agent Teams lifecycle bug, but the current patch can falsely convert non-terminal child activity into `ok` and can skip the existing lost-context fallback if output lookup fails.

## What This Changes

This PR adds a lost-context outcome resolver, wires the subagent sweeper to use it before completing stale runs, and adds focused regression tests.

## Best Possible Solution

Keep the sweeper reconciliation, but base recovery on a terminal child-result predicate that rejects tool-only, `sessions_yield` wait, and `toolUse` pre-tool states, catches read failures, and is backed by real Agent Teams proof.

## Reproduction Assessment

Yes, source-level: current main has a stale-run lost-context fallback, and the linked issue shows the announce path can still carry child output beside that failure status. I did not execute a live gateway reproduction in this read-only pass.

## Solution Assessment

No, not as submitted: the fix location is plausible, but using the announce display helper as the success predicate is too broad and read failures need to fall back to the existing lost-context outcome.

Full review comments:

- **[P1] Require terminal child output before marking lost context ok:** `src/agents/subagent-lost-context-completion.ts:16-18`
  - body: `readSubagentOutput` is an announce-display helper, not a terminal-success predicate: it can return synthetic tool-call summaries or pre-tool text from a still-working assistant turn. This truthy check can therefore complete a stale unfinished child as `ok`, so recovery should use a classified terminal-result predicate and keep lost-context for non-results.
  - confidence: 0.88
- **[P2] Preserve the fallback when output lookup fails:** `src/agents/subagent-lost-context-completion.ts:16`
  - body: `readSubagentOutput` reaches `chat.history` and can reject. Because this await happens before `completeSubagentRunWithRecovery`, a lookup failure now aborts the sweep instead of recording the existing lost-context fallback; catch read failures and return the error outcome.
  - confidence: 0.82
