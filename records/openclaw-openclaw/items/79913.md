## Summary

Keep open: current main and the latest release still lack the spawn-allowlist env bridge, but this patch is not merge-ready because it lets host env replace an explicit default `sessions_spawn` allowlist.

## What This Changes

The PR adds a config-load env overlay that parses `OPENCLAW_SPAWN_ALLOWLIST`/`SPAWN_ALLOWLIST` into `agents.defaults.subagents.allowAgents` and adds focused unit tests.

## Best Possible Solution

Land a fallback-only env bridge that fills a missing default allowlist for env-only Docker deployments, preserves explicit JSON and per-agent allowlists, keeps configured-agent wildcard limits, and documents precedence.

## Reproduction Assessment

Yes, source-reproducible: current main and `v2026.6.10` have no `SPAWN_ALLOWLIST` reader, while native and ACP spawn gates only receive configured `subagents.allowAgents`. I did not run the Docker Compose scenario in this read-only review.

## Solution Assessment

No as written. A config-load fallback is the right narrow layer for env-only Docker deployments, but the patch should not override explicit default allowlists and should document the new env surface.
