## Summary

Keep open: current main and the latest release still do not read the spawn allowlist env vars, but the submitted patch is not merge-ready because it can overwrite explicit `agents.defaults.subagents.allowAgents` and broaden `sessions_spawn` authorization.

## What This Changes

The PR adds `OPENCLAW_SPAWN_ALLOWLIST`/`SPAWN_ALLOWLIST` parsing during config load and writes the parsed value to `agents.defaults.subagents.allowAgents`.

## Best Possible Solution

Keep this focused branch, but change the env overlay to fill `agents.defaults.subagents.allowAgents` only when no explicit default is configured, then document the env fallback precedence.

## Reproduction Assessment

Yes, source-reproducible: current main and `v2026.6.9` have no spawn allowlist env reader, while native and ACP spawn authorization consume only config-backed allowlists. I did not run the Docker Compose scenario in this read-only review.

## Solution Assessment

No as submitted. Config-load materialization is a good layer for env-only Docker deployments, but env should be a missing-default fallback unless maintainers explicitly choose and document authoritative env precedence.
