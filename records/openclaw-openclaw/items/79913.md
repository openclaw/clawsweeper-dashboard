## Summary

This PR is still useful because current main and v2026.6.9 do not read `SPAWN_ALLOWLIST`, but it needs changes before merge because the env overlay currently replaces explicit default `sessions_spawn` allowlists.

## What This Changes

The branch adds a config-load helper and tests that parse `OPENCLAW_SPAWN_ALLOWLIST`/`SPAWN_ALLOWLIST` into `agents.defaults.subagents.allowAgents`.

## Best Possible Solution

Land a fallback-only env bridge that fills a missing default allowlist for env-only Docker deployments, preserves explicit JSON and per-agent allowlists, keeps configured-agent wildcard limits, and documents precedence.

## Reproduction Assessment

Yes, source-reproducible: current main and v2026.6.9 have no `SPAWN_ALLOWLIST` reader, and native/ACP spawn gates only receive configured `subagents.allowAgents`. I did not run the Docker Compose scenario in this read-only review.

## Solution Assessment

No as written. A config-load fallback is the right narrow layer for env-only Docker deployments, but the patch should not override explicit default allowlists and should document the new env surface.
