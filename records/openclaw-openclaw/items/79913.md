## Summary

Keep open: the PR targets a valid source-reproducible Docker spawn allowlist gap, but the current diff lets an environment value replace explicit default spawn allowlist config and can broaden `sessions_spawn` authorization.

## What This Changes

The PR parses `OPENCLAW_SPAWN_ALLOWLIST`/`SPAWN_ALLOWLIST` during config load, writes the result to `agents.defaults.subagents.allowAgents`, and adds focused unit coverage.

## Best Possible Solution

Land one canonical implementation that maps the env allowlist into the documented subagent allowlist contract without overriding explicit config, with docs/tests and Docker proof.

## Reproduction Assessment

Yes, source-reproducible: current main only passes per-agent/default config allowlists into `sessions_spawn`, ACP spawn, and `agents_list`, and no `SPAWN_ALLOWLIST` reader exists. That matches the linked Docker/Coolify report when the operator supplies only the env allowlist.

## Solution Assessment

No. The parser and config-load hook are plausible, but env-overrides-config can widen the default spawn authorization boundary; fallback-only precedence is the safer maintainable path unless maintainers explicitly approve env-wins semantics.
