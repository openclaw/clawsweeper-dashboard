## Summary

Keep this PR open, but it still needs changes before merge: the branch fixes a plausible Docker env allowlist gap, yet it lets an env variable override explicit spawn authorization defaults and overlaps with an open fallback-only implementation.

## What This Changes

The PR adds a config-load helper that parses `OPENCLAW_SPAWN_ALLOWLIST`/`SPAWN_ALLOWLIST` into `agents.defaults.subagents.allowAgents` and tests env parsing plus overlay behavior.

## Best Possible Solution

Land one canonical env fallback contract that preserves explicit per-agent/default config precedence unless maintainers deliberately approve and document env-wins authorization semantics.

## Reproduction Assessment

Yes. Source inspection shows current main has no `SPAWN_ALLOWLIST` reader and the spawn/list paths only consume config-backed allowlists, which matches the linked Docker `allowed: none` failure path.

## Solution Assessment

No. The parser and load hook are plausible, but writing env values over explicit default config is not the safest authorization boundary; fallback-only precedence or explicit maintainer approval is needed.
