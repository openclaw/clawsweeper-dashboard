## Summary

Keep open: the linked Docker/Coolify spawn-allowlist symptom is source-reproducible on current main, but this PR should not merge as-is because its env overlay can replace an explicit default allowlist and broaden `sessions_spawn` authorization. The work also overlaps with https://github.com/openclaw/openclaw/pull/79538, which implements the safer fallback-only direction.

## What This Changes

The PR adds a config-load helper that parses `OPENCLAW_SPAWN_ALLOWLIST` or `SPAWN_ALLOWLIST` into `agents.defaults.subagents.allowAgents` and adds focused parsing/overlay tests.

## Best Possible Solution

Land one canonical implementation that treats `SPAWN_ALLOWLIST` as a documented fallback after explicit per-agent/default config, with regression coverage and maintainer-approved precedence.

## Reproduction Assessment

Yes, source-level. Current main has no `SPAWN_ALLOWLIST` reader, and direct spawn, ACP spawn, and `agents_list` only consult config-backed `subagents.allowAgents`, matching the linked env-only failure path.

## Solution Assessment

No. The parser and load hook are plausible, but env-overrides-config is not the narrowest maintainable authorization policy; fallback-only precedence or an explicit maintainer-approved env-wins contract is safer.
