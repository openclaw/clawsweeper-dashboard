## Summary

## What This Changes

The PR adds a config-load env overlay that parses `OPENCLAW_SPAWN_ALLOWLIST` or `SPAWN_ALLOWLIST` into `agents.defaults.subagents.allowAgents`, plus parser and overlay tests.

## Best Possible Solution

Map the env allowlist into the runtime default only when no explicit per-agent/default config allowlist exists, document the env contract, and prove Docker `sessions_spawn` succeeds against a configured target.

## Reproduction Assessment

Yes, source-reproducible. Current main and the latest release only read config-backed `subagents.allowAgents`, and no `SPAWN_ALLOWLIST` reader exists on the inspected spawn/list paths.

## Solution Assessment

No. The config-load seam is plausible, but env-overrides-config is not the safest authorization contract; fallback-only precedence plus docs and direct spawn proof is the narrower maintainable path.
