## Summary

Keep this PR open for maintainer review: it targets the reported Docker allowlist gap, but the current patch makes an env variable override explicit default spawn authorization config and overlaps an open broader PR with different fallback semantics.

## What This Changes

The branch adds a config-load helper and tests that parse `OPENCLAW_SPAWN_ALLOWLIST`/`SPAWN_ALLOWLIST` into `agents.defaults.subagents.allowAgents`.

## Best Possible Solution

Support the env mapping through a documented and tested contract that preserves explicit per-agent/default config precedence unless maintainers deliberately approve env-wins authorization semantics.

## Reproduction Assessment

Yes, at source level: current main has no `SPAWN_ALLOWLIST` reader, and both `sessions_spawn` and `agents_list` read only config-backed allowlists. I did not run a Docker repro in this read-only review.

## Solution Assessment

No. The parser and config-load hook are plausible, but env-overrides-config can broaden spawn authorization; fallback-only precedence is the safer maintainable path unless maintainers approve and document env-wins semantics.
