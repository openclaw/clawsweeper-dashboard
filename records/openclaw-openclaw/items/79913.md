## Summary

Keep open: the Docker/Coolify `sessions_spawn` env allowlist bug is still not fixed on current main or the latest release, and this focused PR is a useful candidate, but it is not merge-ready because the patch lets host env replace explicit default spawn allowlists at an authorization boundary and does not document the new env surface.

## What This Changes

The PR adds a config-load helper and tests that parse `OPENCLAW_SPAWN_ALLOWLIST`/`SPAWN_ALLOWLIST` into `agents.defaults.subagents.allowAgents` during runtime config finalization.

## Best Possible Solution

Land a fallback-only env bridge that fills a missing default spawn allowlist for env-only Docker deployments, preserves explicit JSON and per-agent allowlists, keeps configured-agent wildcard limits, and documents the env formats and precedence.

## Reproduction Assessment

Yes. Current main and `v2026.6.10` have no `SPAWN_ALLOWLIST`/`OPENCLAW_SPAWN_ALLOWLIST` reader, while native and ACP spawn gates only consume configured `subagents.allowAgents`; I did not run the Docker Compose scenario in this read-only review.

## Solution Assessment

No as written. Config-load fallback is the right narrow layer for env-only Docker deployments, but the patch should be fallback-only rather than overriding explicit default allowlists, and it needs docs for the new env surface.
