## Summary

Keep this PR open because the linked Docker sessions_spawn regression is still real on current main, but the submitted patch is not merge-ready: it lets a host/container env var replace an explicit default spawn allowlist and can broaden authorization.

## What This Changes

The PR parses `OPENCLAW_SPAWN_ALLOWLIST`/`SPAWN_ALLOWLIST` during config load, writes the result to `agents.defaults.subagents.allowAgents`, and adds parser/overlay tests.

## Best Possible Solution

Keep the focused env parser, but apply it only as a fallback when no default `allowAgents` is configured, preserve per-agent/default JSON precedence, and add regression coverage for the non-overwrite case.

## Reproduction Assessment

Yes. Current main has no `SPAWN_ALLOWLIST` reader, and the native/ACP sessions_spawn gates only receive `allowAgents` from per-agent or default config; I did not run the Docker Compose scenario in this read-only review.

## Solution Assessment

No, not as submitted. Reading the env var during config load is a reasonable narrow layer, but the implementation should preserve explicit default config and use env only as a missing-default fallback.
