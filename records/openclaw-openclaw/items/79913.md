## Summary

Keep open: current main still ignores the reported spawn allowlist env var, but this PR is not merge-ready because it lets an env variable replace explicit default spawn authorization config. The related open fallback-only implementation means maintainers should choose the canonical precedence before merging.

## What This Changes

The PR adds a config-load helper that parses `OPENCLAW_SPAWN_ALLOWLIST`/`SPAWN_ALLOWLIST`, writes it to `agents.defaults.subagents.allowAgents`, and adds parser/overlay tests.

## Best Possible Solution

Add a documented env fallback that only applies when per-agent and default config allowlists are absent, or explicitly decide and document env override semantics before merging.

## Reproduction Assessment

Yes. Source inspection shows current main has no `SPAWN_ALLOWLIST` reader and both direct and ACP `sessions_spawn` authorization paths consult only config-backed allowlists; I did not run a live Docker reproduction in this read-only review.

## Solution Assessment

No. The env parser is plausible, but applying it during config load overwrites explicit default spawn authorization config; fallback-only precedence or an explicit maintainer-approved override contract is safer.
