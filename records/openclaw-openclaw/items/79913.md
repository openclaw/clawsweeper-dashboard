## Summary

Keep open: the PR addresses a still-missing env-driven sessions_spawn allowlist path, but the patch is not merge-ready because it lets container env overwrite an explicit default allowlist on an authorization-sensitive path.

## Best Possible Solution

Keep the focused config-load env parser, but apply it only when no default allowAgents is configured, preserve explicit JSON precedence, document the env fallback, and add regression coverage for the non-overwrite case.

## Solution Assessment

No, not as submitted. Config-load materialization is the right layer, but env must be a missing-default fallback rather than an override of explicit allowlist policy.
