## Summary

The PR adds a config-load helper that reads `OPENCLAW_SPAWN_ALLOWLIST`/`SPAWN_ALLOWLIST` into `agents.defaults.subagents.allowAgents`, plus focused parser and overlay tests.

## Best Possible Solution

Land a narrow fallback-only env bridge: read the env var for env-only Docker deployments when the default allowlist is absent, preserve explicit JSON/per-agent allowlists, keep configured-agent wildcard limits, and document the precedence.

## Reproduction Assessment

Yes. Current main has no `SPAWN_ALLOWLIST`/`OPENCLAW_SPAWN_ALLOWLIST` reader, and both native and ACP spawn gates only receive configured `subagents.allowAgents`, matching the linked Docker report's `allowed: none` failure path.

## Solution Assessment
