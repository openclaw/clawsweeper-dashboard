Keep this PR open but not merge-ready: it targets a real Docker/Coolify sessions_spawn allowlist bug, but the current patch still lets host env replace an explicit default allowlist and needs docs for the new authorization-affecting env surface.

## What This Changes

The PR adds a config-load helper that reads OPENCLAW_SPAWN_ALLOWLIST or SPAWN_ALLOWLIST into agents.defaults.subagents.allowAgents, plus parser and overlay tests.

## Best Possible Solution

Land a fallback-only env bridge: env should populate the default allowlist only when explicit default config is absent, per-agent allowlists must still win, wildcard limits must stay configured-agent-only, and docs should state formats and precedence.

## Reproduction Assessment

Yes. Source inspection shows current main has no SPAWN_ALLOWLIST reader, while native spawn, ACP spawn, and agents_list only consume configured allowAgents, matching the reported allowed-none failure path.

## Solution Assessment

No as written. Config-load fallback is the right narrow layer for env-only Docker deployments, but replacing explicit JSON defaults is not the safest fix and needs either fallback-only behavior or an explicit maintainer security decision.
