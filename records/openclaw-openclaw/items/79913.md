## Summary

Keep this PR open, but do not merge it as-is: the implementation proves the Docker env path is plausible, yet the patch makes an env var override explicit spawn authorization config and overlaps with another open fallback-only implementation.

## What This Changes

The PR adds a config-load helper that parses `OPENCLAW_SPAWN_ALLOWLIST`/`SPAWN_ALLOWLIST` into `agents.defaults.subagents.allowAgents` with focused parser and overlay tests.

## Best Possible Solution

Support `SPAWN_ALLOWLIST` through a documented fallback-only contract that preserves explicit per-agent/default config precedence, keeps `agents_list` aligned with `sessions_spawn`, and includes regression coverage for the Docker env path.

## Reproduction Assessment

Yes, source-reproducible. Current main has no `SPAWN_ALLOWLIST` reader, while `sessions_spawn` and `agents_list` only use config-backed `allowAgents`, matching the linked Docker report if env support is accepted as intended behavior.

## Solution Assessment

No. The parser and load hook are plausible, but env-overrides-config can broaden a spawn authorization default; fallback-only precedence is the safer maintainable path unless maintainers explicitly approve env-wins semantics.
