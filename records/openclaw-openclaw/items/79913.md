## Summary

Keep open. The PR targets a real source-reproducible gap, but the current patch should not merge because the env overlay can replace an explicit default spawn allowlist and broaden `sessions_spawn` authorization.

## What This Changes

The PR adds `OPENCLAW_SPAWN_ALLOWLIST`/`SPAWN_ALLOWLIST` parsing during config load, maps it to `agents.defaults.subagents.allowAgents`, and adds parser/unit coverage.

## Best Possible Solution

Land one documented env allowlist implementation that preserves explicit config precedence across direct spawn, ACP spawn, and `agents_list`, or explicitly decide that operators should use `openclaw.json` only.

## Reproduction Assessment

Yes. Source inspection shows current main has no spawn allowlist env reader and the `sessions_spawn` paths only consult config-backed `subagents.allowAgents`, matching the linked Docker symptom.

## Solution Assessment

No. The narrow parser is plausible, but env-overrides-config is not the safest default for a spawn authorization allowlist; fallback-only behavior or explicit maintainer-approved override semantics would be safer.
