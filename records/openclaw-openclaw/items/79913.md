## Summary

This should stay open for maintainer review: current main still does not support the env allowlist path, but the proposed fix is not safe to merge as-is because env can override explicit spawn authorization config.

## What This Changes

The branch adds a config-load helper that parses `OPENCLAW_SPAWN_ALLOWLIST`/`SPAWN_ALLOWLIST` into `agents.defaults.subagents.allowAgents`, plus parser/unit coverage.

## Best Possible Solution

Add a documented, maintainer-approved env allowlist mapping only if desired, preserving explicit per-agent/default config precedence and covering direct spawn, ACP spawn, and `agents_list` consistently.

## Reproduction Assessment

Yes. Source inspection shows current main has no `SPAWN_ALLOWLIST` reader and the `sessions_spawn` authorization paths only consult config-backed `subagents.allowAgents`, matching the linked Docker/Coolify symptom.

## Solution Assessment

No. The env mapping may be useful, but this implementation is not the best way as-is because it overwrites explicit config; fallback-only behavior or documented maintainer-approved env precedence is safer.
