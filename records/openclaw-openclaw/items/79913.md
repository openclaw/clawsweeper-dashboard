## Summary

Keep open: the linked Docker failure is still source-reproducible on current main, and this PR is a plausible narrow fix, but the current env-wins implementation can broaden `sessions_spawn` authorization and overlaps with https://github.com/openclaw/openclaw/pull/79538.

## What This Changes

Adds a config-load helper that parses `OPENCLAW_SPAWN_ALLOWLIST`/`SPAWN_ALLOWLIST` into `agents.defaults.subagents.allowAgents`, wires it into config finalization, and tests parsing plus overlay behavior.

## Best Possible Solution

Land one canonical implementation that treats `SPAWN_ALLOWLIST` as a documented fallback when no per-agent/default `allowAgents` config exists, and keeps `agents_list` aligned with `sessions_spawn`.

## Reproduction Assessment

Yes by source inspection: current main has no `SPAWN_ALLOWLIST` reader, and both `sessions_spawn` and `agents_list` pass only per-agent/default config `allowAgents`, so the env-only Docker setup reaches the reported `allowed: none` path. I did not run a container repro in this read-only review.

## Solution Assessment

No. The parser and load hook are plausible, but env-overrides-config can broaden a spawn authorization default; fallback-only precedence with docs/tests is the safer maintainable path unless maintainers explicitly approve env-wins.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

- **[P2] Preserve configured spawn allowlists:** `src/config/spawn-allowlist-env.ts:52`
