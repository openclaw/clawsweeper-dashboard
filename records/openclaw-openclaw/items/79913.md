## Summary

Keep this PR open, but it is not merge-ready because the proposed env overlay can broaden the default subagent spawn allowlist by overriding explicit config; the related discussion also shows maintainers still need to choose the env/config precedence and canonical implementation path.

## What This Changes

The PR adds a config-load helper that parses `OPENCLAW_SPAWN_ALLOWLIST`/`SPAWN_ALLOWLIST` into `agents.defaults.subagents.allowAgents` and unit tests for parsing and overlay behavior.

## Best Possible Solution

Support the env mapping only through a documented and tested contract that preserves explicit per-agent/default config precedence unless maintainers deliberately approve env-wins authorization semantics.

## Reproduction Assessment

Yes. Source inspection of current main shows `sessions_spawn` and `agents_list` only read config-backed allowlists and no `SPAWN_ALLOWLIST` reader exists, matching the linked Docker report's `allowed: none` symptom if env support is accepted as intended behavior.

## Solution Assessment

No. The parser and load hook are plausible, but env-overrides-config can broaden a spawn authorization default; fallback-only precedence is the safer maintainable path unless maintainers explicitly approve the override policy.
