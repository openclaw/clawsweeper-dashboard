## Summary

Current main still has the source-backed failure path, and an open PR explicitly closes this issue. Keep this issue open while that PR is reviewed, repaired, or replaced rather than closing it ahead of a merged fix.

## What This Changes

The issue reports that gateway auto-update children inherit gateway service marker environment variables and are then refused by the package-update guard.

## Best Possible Solution

Land or replace the linked closing PR with a service-safe auto-update handoff that clears inherited gateway service state and avoids the live gateway process tree.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: service env injects the markers, the auto-update child does not clear them, env merge preserves inherited values, and the update command blocks package updates from that service context.

## Solution Assessment

Yes for keeping the issue open behind the linked closing PR; no for closing it now. The narrowest maintainable fix should preserve the operator safety guard while making the auto-update handoff run outside the blocked service context.
