## Summary

Current main still has the reported source-backed failure path, and an open linked PR explicitly closes this issue. Keep this issue open while that PR is reviewed, repaired, or replaced rather than closing it ahead of the fix.

## What This Changes

The issue reports that managed gateway auto-update inherits service marker environment variables into the spawned updater, causing the package-update guard to reject the update.

## Best Possible Solution

Land or replace the linked closing PR with a service-safe auto-update handoff that clears inherited gateway service state and routes package replacement outside the live gateway process tree.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: the managed service sets marker env, the auto-update child inherits it, env merge preserves it, and the update command blocks package updates from that service context.

## Solution Assessment

Yes for keeping the issue open behind the linked PR; no for closing it now. The narrowest maintainable fix should preserve the operator safety guard while giving trusted auto-update a service-safe handoff.
