## Best Possible Solution

## Reproduction Assessment

Yes for the blocking PR regression by source inspection: setup writes host OPENCLAW_* paths for bind mounts, and PR-head Compose imports .env without repinning those variables to container paths. I did not run a live Docker smoke in this read-only review.

## Solution Assessment

No, not as-is: the Docker baseline direction may be useful, but this patch removes a shipped path-protection contract, leaves public docs and exact-head proof incomplete, and bundles unrelated fixes into one dirty branch.
