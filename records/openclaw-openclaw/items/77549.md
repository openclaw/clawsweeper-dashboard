Full review comments:

- **[P1] Preserve container-side Docker path overrides:** `docker-compose.yml:9`
  - body: The PR head drops the Compose environment overrides for `OPENCLAW_CONFIG_DIR` and `OPENCLAW_WORKSPACE_DIR` even though the bind-mount sources still come from host-side `.env` values. That reopens the Docker path-leak failure fixed by https://github.com/openclaw/openclaw/pull/77446, where runtime code could try to create host paths such as `/Users/...` inside Linux Docker.
  - confidence: 0.9
- **[P1] Keep channel validation strict by default:** `Dockerfile:200`
  - body: Setting `OPENCLAW_LENIENT_CHANNEL_CONFIG=1` in the image makes invalid channel config validate with warnings and disables or removes the channel config, so an existing Docker deployment can upgrade from a loud config failure to lost channel delivery. Keep strict validation as the default and handle known legacy shapes through doctor or an explicit opt-in.
  - confidence: 0.88
- **[P2] Keep outbound loading on the bootstrap registry path:** `src/channels/plugins/outbound/load.ts:19`
  - body: `loadChannelOutboundAdapter` is the lightweight send-primitive loader, and current outbound delivery already retries through `bootstrapOutboundChannelPlugin` after a registry miss. Falling back to `getBundledChannelPlugin(id)?.outbound` here materializes the full bundled channel plugin on the cheap send path; repair the registry/bootstrap seed or add an outbound-only facade instead.
  - confidence: 0.86
