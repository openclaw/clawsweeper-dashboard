## Summary

Keep open: the PR is useful, but the current head splits the Docker runtime contract by switching the image to `openclaw` uid 1001 while Compose/setup/docs still use `node` uid 1000 and `/home/node`. That should be fixed before merge, along with the required changelog/docs coverage.

## What This Changes

The PR expands the hosted Docker image baseline, adds hosted Docker update/channel-validation env defaults, and emits ACP pre-agent message hooks with focused tests.

## Best Possible Solution

Land a version that keeps one coherent Docker user/home/uid contract across Dockerfile, Compose, setup scripts, docs, and regression tests, then add the required user-facing changelog/docs for the new hosted defaults.

## Reproduction Assessment

Not applicable as a user bug report, but the main review finding is source-reproducible: the PR head runs the image as `openclaw` uid 1001 while unchanged Docker setup still mounts and chowns state under `/home/node` for uid 1000.

## Solution Assessment

No: the branch is not merge-ready because the Docker runtime ownership change is not carried through the Compose/setup/docs surfaces. The narrow maintainable fix is to align those surfaces around either the new `openclaw` contract or the existing `node` contract before changing image defaults.
