- **[P1] Preserve container-side Docker path overrides:** `docker-compose.yml:9`
  - body: The PR changes HOME/mount targets to `/home/openclaw` but removes the current `OPENCLAW_STATE_DIR`, `OPENCLAW_CONFIG_PATH`, `OPENCLAW_CONFIG_DIR`, and `OPENCLAW_WORKSPACE_DIR` overrides on both services while volume sources still come from host-side `.env` variables.   - confidence: 0.9
- **[P1] Keep channel validation strict by default:** `Dockerfile:200`
  - body: Setting `OPENCLAW_LENIENT_CHANNEL_CONFIG=1` in the runtime image changes invalid channel config from a startup validation failure into warnings plus removed or disabled channel config. Existing Docker deployments can lose channel delivery silently on upgrade; keep strict validation as default and repair known legacy shapes through doctor or an explicit opt-in.
  - confidence: 0.88
- **[P2] Keep outbound loading on the bootstrap registry path:** `src/channels/plugins/outbound/load.ts:19`
  - body: This loader is documented as the cheap outbound send-primitive path, and current delivery already bootstraps runtime plugins after a registry miss in `src/infra/outbound/deliver.ts`. The new bundled fallback imports full channel runtime from the lightweight loader, so fix the registry/bootstrap path or add an outbound-only facade instead.
  - confidence: 0.86
