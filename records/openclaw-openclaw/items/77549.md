## Summary

## What This Changes

The PR expands the Docker runtime image/tooling baseline, changes the container home/uid and hosted env defaults, adds lenient config/model compatibility, emits ACP pre-agent hooks, and updates related tests.

## Best Possible Solution

Keep the hosted Docker baseline only after restoring /home/openclaw container env pins, updating public Docker/updating docs and CHANGELOG.md, and attaching real Docker setup proof for maintainer review.

## Reproduction Assessment

Yes for the blocking PR regression by source inspection: setup writes host OPENCLAW_* dirs to .env, and PR-head Compose imports that env_file without overriding those vars to /home/openclaw inside the service. I did not run a live Docker smoke in this read-only review.

## Solution Assessment

No, not as-is: the hosted baseline direction may be useful, but the current patch removes an existing Docker path-protection contract and leaves user-facing docs/changelog plus real setup proof incomplete.
