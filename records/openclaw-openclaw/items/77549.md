## What This Changes

## Best Possible Solution

Restore the Compose env pins with /home/openclaw paths, complete legacy allowlist handling across all model surfaces, then update Docker/updating docs and CHANGELOG.md before maintainer review of the hosted baseline.

## Reproduction Assessment

Yes for the blocking PR regression: source inspection shows setup.sh writes host paths into .env while PR-head Compose imports that file without overriding the runtime paths back to container paths. I did not run a live Docker smoke in this read-only review.

## Solution Assessment
