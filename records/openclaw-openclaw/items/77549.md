## Summary

Keep this PR open: the latest head still needs docs/changelog coverage and now drops the Compose runtime env overrides that keep host `.env` paths from leaking into the Linux container.

## What This Changes

The PR expands the hosted Docker image baseline, switches the runtime user/home to `openclaw` uid 1001, adds hosted config/update guards, and emits ACP pre-agent message hooks.

## Best Possible Solution

Restore the Compose container-side config/workspace env pins with the new `/home/openclaw` paths, then add the required Docker docs and changelog before landing.

## Reproduction Assessment

Yes for the blocking PR regression: source inspection shows setup writes host paths into `.env`, while PR-head Compose imports that file without overriding the runtime paths back to container paths. No live Docker smoke was run in this read-only review.

## Solution Assessment

No, not complete as-is: the Docker baseline direction may be acceptable, but the narrow maintainable repair is to preserve the existing Compose path-protection contract and document the new hosted defaults.
