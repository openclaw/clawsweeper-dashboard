## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Restore the container-side runtime path pins:** `docker-compose.yml:9-10`
  - body: Both services still import `.env`, but the branch no longer overrides `OPENCLAW_HOME`, `OPENCLAW_STATE_DIR`, `OPENCLAW_CONFIG_PATH`, `OPENCLAW_CONFIG_DIR`, or `OPENCLAW_WORKSPACE_DIR` inside the container. Because `scripts/docker/setup.sh` writes host paths into `.env`, macOS values such as `/Users/...` can leak back into Linux runtime path resolution and reintroduce the first-reply EACCES failure fixed by https://github.com/openclaw/openclaw/pull/77446.
  - confidence: 0.96
- **[P2] Preserve the auth-profile secret mount:** `docker-compose.yml:46-47`
  - body: The branch rewrites the Compose volumes to `/home/openclaw` but drops the current `OPENCLAW_AUTH_PROFILE_SECRET_DIR` mount into the container config directory. Docker setups that rely on mounted auth-profile secrets would lose those secrets inside both services after merge.
  - confidence: 0.9
- **[P3] Update Docker docs for the new runtime home:** `Dockerfile:193-203`
  - body: The image now runs as user `openclaw` with home `/home/openclaw`, uid 1001, and writable npm/pip homes, but the Docker docs still tell users that `OPENCLAW_HOME_VOLUME` persists `/home/node`. Update the public Docker docs so mount, volume, and permission guidance matches the new runtime contract.
  - confidence: 0.91
