## Summary

Keep open: the hosted Docker direction may be useful, but the current branch still has a source-proven Compose regression, insufficient real Docker proof, stale Docker docs, dirty/unmergeable state, and broad product scope needing maintainer review.

## What This Changes

This PR expands the Docker/GHCR runtime image baseline and Compose settings, adds hosted config/update guards and compatibility behavior, emits ACP pre-agent hooks, and includes web-search, tool-error, and Telegram route fixes with tests.

## Best Possible Solution

Land a focused, maintainer-approved hosted Docker baseline after restoring container path pins, updating Docker docs, and attaching exact-head Docker smoke proof; split unrelated Telegram/web-search/tool-error changes if they should proceed independently.

## Reproduction Assessment

Yes for the blocking PR regression by source inspection: scripts/docker/setup.sh writes host OPENCLAW_* paths into .env, and PR-head docker-compose.yml imports that env_file without repinning those variables to /home/openclaw inside the services. I did not run a live Docker smoke in this read-only review.

## Solution Assessment

No, not as-is: the Docker baseline direction may be useful, but the patch removes an existing path-protection contract and leaves public docs plus exact-head real behavior proof incomplete.
