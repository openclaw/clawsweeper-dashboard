## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve the container path pins on rebase:** `docker-compose.yml:9-10`
  - body: Current main pins OPENCLAW_HOME, OPENCLAW_STATE_DIR, OPENCLAW_CONFIG_PATH, OPENCLAW_CONFIG_DIR, and OPENCLAW_WORKSPACE_DIR inside the container so host .env paths are only bind-mount sources. This branch switches the service to /home/openclaw but omits those overrides, which would reintroduce the Docker first-reply host-path failure that current main fixed; preserve those pins with the new home instead of only changing HOME and volumes.
  - confidence: 0.9
- **[P1] Keep lenient channel validation out of the default image:** `Dockerfile:200-202`
  - body: Because the Dockerfile sets OPENCLAW_LENIENT_CHANNEL_CONFIG=1, any channel schema error takes the new branch that replaces that channel with { enabled: false }. A stale or typoed Telegram/Discord config would boot with the channel disabled instead of failing loudly or going through doctor migration, so this needs to stay opt-in or be handled by a narrow migration before becoming a hosted default.
  - confidence: 0.88
- **[P1] Bound pre-agent hook execution before awaiting it:** `src/auto-reply/reply/message-preprocess-hooks.ts:57-58`
  - body: triggerInternalHook awaits registered handlers serially and has no timeout, so awaiting it here lets a slow or hung message hook block get-reply/ACP before the agent turn. Keep the old non-blocking behavior for ordinary hooks or add a bounded mutation-specific hook path before putting this on the reply hot path.
  - confidence: 0.86
