  - body: The PR changes the Compose container HOME and mount targets but drops the container-side OPENCLAW_CONFIG_DIR, OPENCLAW_WORKSPACE_DIR, state, and config-path overrides while bind-mount sources still come from host-side .env values.   - confidence: 0.9
- **[P1] Keep channel validation strict by default:** `Dockerfile:200`
  - body: Setting OPENCLAW_LENIENT_CHANNEL_CONFIG=1 in the image makes invalid channel config validate with warnings and then removes or disables the channel config. Existing Docker deployments can upgrade from a loud config failure to silently lost channel delivery, so keep strict validation as the default and handle known legacy shapes through doctor or an explicit opt-in.
  - confidence: 0.88
- **[P2] Keep outbound loading on the bootstrap registry path:** `src/channels/plugins/outbound/load.ts:19`
  - body: loadChannelOutboundAdapter is the lightweight send-primitive loader, and current outbound delivery already retries through bootstrapOutboundChannelPlugin after a registry miss. Falling back to getBundledChannelPlugin(id)?.outbound here materializes full bundled channel plugins on the cheap send path; repair the registry/bootstrap seed or add an outbound-only facade instead.
  - confidence: 0.86
