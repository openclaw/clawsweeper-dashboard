Full review comments:

- **[P2] Restore the container-side state path overrides:** `docker-compose.yml:9-11`
  - body: This service still imports `.env`, but the branch no longer pins `OPENCLAW_HOME`, `OPENCLAW_STATE_DIR`, `OPENCLAW_CONFIG_PATH`, `OPENCLAW_CONFIG_DIR`, or `OPENCLAW_WORKSPACE_DIR` in either service environment.   - confidence: 0.96

  - body: The branch rewrites the Compose volume targets for `/home/openclaw` but does not carry forward current main's `OPENCLAW_AUTH_PROFILE_SECRET_DIR` mount into the container config directory. Docker setups that rely on mounted auth-profile secrets would lose those secrets inside both services after this branch is merged or rebased without restoring the mount.
  - confidence: 0.88
- **[P3] Update Docker docs for the new runtime home:** `Dockerfile:193-203`
  - body: The image now runs as `openclaw` with home `/home/openclaw`, uid 1001, writable npm/pip homes, and hosted update defaults, while the Docker docs still tell users that `OPENCLAW_HOME_VOLUME` persists `/home/node`. Update the public Docker docs so mount, volume, and permission guidance matches the new runtime contract.
  - confidence: 0.91
