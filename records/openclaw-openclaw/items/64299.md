## Summary

Keep open. Current main still leaves the Feishu document write/append 400 report actionable because the Feishu docs/setup guidance still omit the reporter-verified conversion scope, and the small-document Descendant API payload builder can still overwrite normalized child arrays with raw converted block data.

## What This Changes

The issue asks for `feishu_doc` write and append to accept valid markdown instead of returning 400, including the required Feishu document conversion setup.

## Best Possible Solution

Fix this in the Feishu plugin and docs by documenting the document conversion scopes, preserving normalized descendant `children` after object spread, and adding focused regression coverage for string-valued converted children.

## Reproduction Assessment

Yes. The reporter live-verified the missing `docx:document.block:convert` scope, and source inspection shows a converted block with `children: "id"` can flow into `toDescendantBlock()` and overwrite the normalized array before the Descendant API request.

## Solution Assessment

Yes. A narrow Feishu plugin/docs repair is the best path; no core API, channel routing, or product decision is needed before fixing the setup guidance and payload normalization.
