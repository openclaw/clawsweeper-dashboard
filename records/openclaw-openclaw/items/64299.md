## Summary

Keep open. Current main still leaves the Feishu document write/append 400 report actionable: write and append still depend on Feishu markdown conversion, setup guidance still omits the reporter-verified conversion scope, and the small-document descendant mapper can still overwrite normalized child arrays with raw converted block data.

## What This Changes

The issue requests fixing Feishu document `write` and `append` so valid markdown content can be converted and inserted without 400 responses.

## Best Possible Solution

Keep the repair in the Feishu plugin: document the required docx conversion scopes in public and manual setup guidance, normalize descendant payload children after object spread for all converted blocks, and add focused regression coverage for string-valued converted children.

Yes. The permission failure was live-verified by the reporter by adding `docx:document.block:convert`, and the payload failure has a source-level reproduction by mocking converted markdown blocks with `children` as a string and inspecting the Descendant API payload.
