Keep open. Current main still has a Feishu document write/append payload bug for converted blocks whose `children` value is a string, and the manual setup guidance still does not name the conversion scope that the reporter verified was required.

Fix the Feishu plugin by preserving normalized descendant `children`, add focused regression coverage, and document the document-conversion scope in the Feishu setup path without broadening core contracts.
