## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/64299. The observable bug is that `feishu_doc` `write` and `append` can return 400 for valid Markdown because setup guidance omits the Feishu `docx:document.block:convert` permission and the Descendant API payload builder can overwrite normalized `children: string[]` with raw converted block data. Keep the fix scoped to the Feishu plugin/docs: preserve normalized descendant `children` after spreading raw block fields in both normal and batch descendant insertion paths as needed; add regression coverage for converted blocks whose `children` is a single string; update public Feishu setup/docs and setup wizard guidance to include the document conversion/document scopes; add a user-facing CHANGELOG.md fix entry. Do not change core plugin APIs, channel routing, auth storage, provider behavior, or add a new config option. Related refs: #64299, #64331, #64336, #29411, #40524.
