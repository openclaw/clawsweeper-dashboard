## Summary

Keep open: current main still does not honor structured `server_error` payloads for the documented cron retry reason, and this PR is a plausible narrow fix, but it still needs required after-fix real behavior proof before merge.

## What This Changes

## Best Possible Solution

Finish the narrow classifier/protocol retry fix, keep the broader provider-outage behavior tracked separately, and merge only after redacted real cron evidence confirms `lastErrorReason: "server_error"` and the retry schedule in a real setup.
