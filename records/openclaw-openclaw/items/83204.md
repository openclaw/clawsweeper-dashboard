## Summary

Current main partially addresses this class by routing unqualified `/stop` and status commands to a Telegram control lane, but the observed `/status@botname` form still appears actionable when bot metadata is unavailable, so this should stay open.

## What This Changes

The issue asks for Telegram control commands such as `/stop` and `/status` to bypass a blocked per-topic queue so they can inspect or cancel stuck runs.

## Best Possible Solution

Extend the Telegram control-lane classifier so read-only status-category commands with a Telegram bot suffix are recognized in isolated ingress even when cached/probed bot metadata is unavailable, while keeping mutating commands on the normal topic lane unless explicitly safe.

## Reproduction Assessment
