## Summary

Keep this PR open: the snapshot fast path matches the plugin metadata caching direction, but the patch trusts the current snapshot even when that snapshot carries registry diagnostics, while neighboring current-snapshot reuse paths deliberately fall back in that case.

## What This Changes

The PR changes the plugin SDK facade manifest lookup to prefer the current in-memory plugin metadata snapshot before falling back to filesystem registry loading, and adds facade runtime tests for Windows path matching plus config-mismatch fallback.

## Best Possible Solution

Keep the snapshot fast path, but only reuse a compatible current snapshot when it is clean; otherwise preserve the existing filesystem registry fallback.

## Reproduction Assessment

Yes for the review finding: source inspection shows the new facade fast path lacks the registry-diagnostics guard that sibling current-snapshot reuse helpers already require.

## Solution Assessment

No: the fast path is the right direction, but the maintainable solution should mirror existing current-snapshot reuse semantics and fall back when registry diagnostics are present.
