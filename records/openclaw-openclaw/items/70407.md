## Summary

Keep open: current main still rejects legacy string-shaped `memory-core` `dreaming.storage` values and has no targeted migration before the generic invalid-plugin quarantine path, so the reported upgrade regression remains actionable.

## What This Changes

The issue asks for an upgrade-safe migration or clearer repair path for legacy `plugins.entries.memory-core.config.dreaming.storage` string values after the schema moved to an object shape.

## Best Possible Solution

Add a focused doctor legacy migration for `memory-core` that rewrites `dreaming.storage` strings (`inline`, `separate`, `both`) to the canonical object shape before invalid-plugin quarantine, while keeping the manifest schema strict.

## Reproduction Assessment

Yes. A config containing `plugins.entries.memory-core.config.dreaming.storage: "both"` has a high-confidence source-level reproduction path: the manifest requires an object, validation reports plugin config errors, and no current migration converts the string before quarantine.

## Solution Assessment

Yes. The narrowest maintainable fix is a targeted memory-core doctor migration plus regression tests; globally relaxing plugin schema validation would be broader and could hide real plugin config errors.
