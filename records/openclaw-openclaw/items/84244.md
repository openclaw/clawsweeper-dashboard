## Summary

Keep open; this PR is a small, owner-local fix for a source-reproducible Twitch test-isolation gap, and I found no blocking correctness or security findings. Current main keeps the Twitch manager registry in module state while repo Vitest defaults to non-isolated runs, and the sibling manager already exposes `clearForTest`.

## What This Changes

This PR adds a test-only `clearRegistryForTest()` export to the Twitch client-manager registry so tests can clear the module-scoped manager cache.

## Best Possible Solution

Merge the additive helper after normal CI or focused Twitch extension proof so tests that exercise `getOrCreateClientManager` can clear registry state in `afterEach`.

## Reproduction Assessment

Yes, from source inspection rather than an executed failing test. Current main uses a module-scoped registry under a documented non-isolated Vitest default, so two tests using the same account id can share a stale manager unless the registry is cleared.

## Solution Assessment

Yes. A test-only clear helper mirrors the existing `TwitchClientManager.clearForTest` pattern and preserves production cache behavior while giving future registry-factory tests an explicit reset hook.
