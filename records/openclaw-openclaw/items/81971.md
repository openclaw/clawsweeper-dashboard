## Best Possible Solution

Keep the PR open, align the bridge policy resolver with the existing Claude arg normalization and exec defaults, and require live Claude CLI proof before merge.

## Reproduction Assessment

Yes from source inspection: current main returns early for Claude live non-`result` frames, so `control_request` frames cannot receive `control_response` writes. The linked report adds live timeout symptoms, but I did not run a live Claude session in this read-only review.

## Solution Assessment

No. The control-response handler is the right narrow surface, but its default policy fallback must match the existing full/off exec and Claude `bypassPermissions` defaults before this is mergeable.
