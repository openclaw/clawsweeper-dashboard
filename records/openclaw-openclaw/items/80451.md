## Summary

Keep open: current main still builds configured sibling Gemini candidates, classifies HTTP 429/RESOURCE_EXHAUSTED as `rate_limit`, and has no billing-account-scoped suppression. The adjacent quota and billing-classification PRs are still open and do not supersede this issue's same-account fallback gap.

## What This Changes

Request account-scoped failover suppression so Gemini sibling models sharing a Google billing account are skipped after a monthly-cap RESOURCE_EXHAUSTED failure.

## Best Possible Solution

Define a provider/account-scope contract for terminal billing or quota failures, then carry that scope through one fallback run so Gemini siblings sharing the resolved account are skipped while cross-provider fallback remains available.

## Reproduction Assessment

Yes, source-level: current main builds the configured sibling candidates, treats Google RESOURCE_EXHAUSTED/HTTP 429 as `rate_limit`, and has no per-run billing-account suppression. A live exhausted Google billing account would still be needed to verify the reporter's exact journal volume.

## Solution Assessment

Unclear: the requested direction is reasonable, but the best maintainable fix depends on whether OpenClaw keys this by auth-profile account metadata, provider-owned billing scope, or a new explicit account-scope contract. A provider-only skip would be a risky overcorrection for multi-account setups.
