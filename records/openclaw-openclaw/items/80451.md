## Summary

Keep open: current main still lacks billing-account-scoped suppression for Gemini sibling fallbacks. Related work covers host-level quota circuit breaking, session suspension, or budget-error classification, but none defines the account-scope key needed to skip only same-account candidates without overblocking separate Google accounts/projects.

## What This Changes

The issue asks OpenClaw to skip configured Gemini sibling models that share a billing account after one model hits an account-scoped spending cap.

## Best Possible Solution

Define an additive provider/account-scope contract for terminal billing or quota failures, classify Google monthly spending-cap responses into that terminal class, and skip only candidates sharing the resolved account scope during the same fallback run.

## Reproduction Assessment

Yes, at source level: configured sibling candidates remain in the fallback chain, the loop checks provider/profile/model cooldown only, and Google 429/RESOURCE_EXHAUSTED still lands in rate_limit. A live exhausted Google billing account would be needed only to reproduce the reporter's exact journal volume.

## Solution Assessment

Unclear as an implementation plan: the requested direction is reasonable, but a durable fix needs maintainer agreement on the account-scope key. A provider-only skip would be too broad for multi-account or multi-project Google setups.
