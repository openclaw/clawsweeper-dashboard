## Summary

Keep open: current main still has no billing-account or quota-bucket identity in the model fallback path, and the adjacent quota/classification work does not supersede this same-account sibling suppression request.

## What This Changes

The issue asks OpenClaw to skip Gemini sibling models that share a Google billing account after a billing-cap `RESOURCE_EXHAUSTED` failure.

## Best Possible Solution

Define an additive provider/account or quota-bucket scope contract, classify native Google billing-cap errors into that scope, and suppress only matching sibling candidates while preserving cross-provider fallback.

## Reproduction Assessment

Yes, source-level: current main builds configured sibling candidates, classifies Google `RESOURCE_EXHAUSTED`/HTTP 429 as `rate_limit`, and has no billing-account suppression. A live exhausted Google billing account would still be needed to verify the reporter's exact journal volume.

## Solution Assessment

Unclear: the requested direction is reasonable, but the maintainable fix depends on whether OpenClaw keys this by auth profile, explicit billing account metadata, provider-owned quota bucket, or another additive contract.
