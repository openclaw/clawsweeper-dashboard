## Summary

Keep open: current main still lacks billing-account-scoped failover suppression, and the related circuit-breaker/classification work does not cover the within-run sibling Gemini scope. The durable fix needs maintainer API/product direction before implementation.

## What This Changes

Request that failover skip same-billing-account sibling Gemini models after a billing-account cap error instead of contacting each configured sibling.

## Best Possible Solution

Define a small provider/account-scope contract for terminal billing or quota failures, carry that scope through one fallback run, and skip only candidates sharing the resolved account scope with focused Gemini-to-OpenAI-to-Gemini coverage.

## Reproduction Assessment

Yes, from source inspection: current main builds the configured sibling candidates, maps Google `RESOURCE_EXHAUSTED`/HTTP 429 to `rate_limit`, and has no per-run billing-account suppression. A live exhausted Google billing account is still needed to verify the reporter's exact journal volume.

## Solution Assessment

Unclear: the requested direction is reasonable, but the right scope key needs maintainer API/product judgment. A provider-only skip would be a risky overcorrection for multi-account Google setups.
