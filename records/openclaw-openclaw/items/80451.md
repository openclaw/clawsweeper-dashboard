Keep this open: current main still has a source-proven same-account failover gap, but the durable fix needs maintainer/API direction for identifying billing-account scope rather than a conservative cleanup close.

## What This Changes

The issue requests account-scoped failover suppression so Gemini sibling models sharing the same Google billing account are skipped after a billing-cap RESOURCE_EXHAUSTED failure.

## Best Possible Solution

Define a small provider/auth billing-account scope contract, then carry that scope through one fallback run so only candidates sharing the resolved account scope are skipped, with focused Gemini-to-OpenAI-to-Gemini coverage.
