## Summary

Keep open. Current main still lacks a billing-account-scoped failover contract: configured Gemini siblings remain normal fallback candidates, Google `RESOURCE_EXHAUSTED`/HTTP 429 still lands in the `rate_limit` lane, and the nearby cooldown logic is provider/profile/model scoped rather than billing-account scoped.

## What This Changes

The issue requests skipping sibling Gemini fallback candidates that share a billing account after a billing-account spending-cap failure.

## Best Possible Solution

Define a small provider/account-scope contract for terminal billing or quota failures, then carry that scope through a single fallback run so only candidates sharing the resolved account scope are skipped.

## Reproduction Assessment

Yes from source inspection: current main builds the configured sibling candidates and has no per-run billing-account suppression after the first `RESOURCE_EXHAUSTED` failure. A live exhausted Google billing account would still be needed to verify the reporter's exact journal volume.

## Solution Assessment

Unclear: the requested direction is reasonable, but the narrow maintainable fix depends on how OpenClaw identifies shared billing-account scope across provider models and auth profiles. A broad provider-only skip would be riskier than an account-scoped contract.
