## What This Changes

The PR changes the gateway watch regression script and its focused tests to improve shell selection, child-exit diagnostics, bounded failure log tails, and duplicate dist-runtime warning gating.

## Best Possible Solution

Fail every unplanned child exit before the watch window completes, keep the normal stop-phase SIGTERM path non-failing, and retain the bounded failure log diagnostics.

## Reproduction Assessment

Yes, source-reproducible. On the PR head, a child exit during settle or idle sets `exit` and skips the rest of the window, while the failure collector only fails exits that happened before readiness.

## Solution Assessment

No, not yet. The guard script is the right owner boundary, but the fix needs to distinguish expected stop-phase termination from unplanned exits before the idle window completes.
