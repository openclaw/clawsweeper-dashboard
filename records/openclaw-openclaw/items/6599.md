## Summary

Keep open: current main still lacks the requested primary-skipping fallback-chain test command, and the related open health-check work is broader than this targeted `/models test-fallback` request.

## What This Changes

Request adding a `/models test-fallback` diagnostic that can intentionally skip the primary model and verify the configured fallback chain before a real provider failure.

## Best Possible Solution

Ship one shared fallback-chain probe behind the chosen command surface that reuses real model/auth resolution, skips the primary by design, supports first and all fallback modes, reports the answering model, and stays side-effect-free by default.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, related PR inspection, and search show the requested command and an equivalent primary-skipping fallback-chain probe are absent on current main.

## Solution Assessment

Unclear until maintainers choose the product surface. A shared probe service reused by the selected CLI or slash UX is safer than a one-off parser branch because fallback ordering, auth, timeout, rate-limit, and state-mutation policy need one source of truth.
