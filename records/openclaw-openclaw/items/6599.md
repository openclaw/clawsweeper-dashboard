Implement the feature requested in https://github.com/openclaw/openclaw/issues/6599. Add a first-class fallback-chain probe for the model command surface: `/models test-fallback` should intentionally skip the primary model, try the first configured fallback, and report success/failure plus the model that responded; an all-chain mode should test every configured fallback. Follow the useful source work in closed PR #43342 for intent, but implement against current main. Reuse OpenClaw's real model/auth fallback resolution path instead of inventing provider-specific curl logic. Add CLI parity as `openclaw models test-fallback` if following the already-scoped maintainer acceptance criteria, with JSON/plain output consistent with existing model commands. Update focused tests and docs, and add a single-line user-facing changelog entry. Do not broaden into the #63145 per-model health-check feature beyond what is needed for primary-skipping fallback-chain verification.
