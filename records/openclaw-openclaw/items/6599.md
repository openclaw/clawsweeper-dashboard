## Summary

Keep open. Current main still lacks the requested primary-skipping fallback-chain probe; the direct implementation PR is closed unmerged, and the broader per-model health-check request is not an exact replacement.

## What This Changes

The issue asks for a `/models test-fallback` command that verifies fallback configuration by intentionally probing fallback candidates before a real provider failure.

## Best Possible Solution

Ship a focused fallback-chain probe that reuses OpenClaw’s real model/auth resolution, starts at configured fallback candidates, reports which model answered, supports first-fallback and all-fallback modes, and documents how it differs from auth-profile probes and broad per-model health checks.

## Solution Assessment

Implement the missing fallback-chain test command for https://github.com/openclaw/openclaw/issues/6599. Add a first-class `/models test-fallback` slash command that intentionally skips the active primary and probes configured fallback candidates using the real model/auth execution path. Support default first-fallback mode plus an `all`/`--all` style mode for the full chain, report success/failure and the responding model, and ensure each probe is pinned so a later fallback cannot mask the candidate being tested.
