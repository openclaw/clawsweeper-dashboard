## Summary

Keep open: current main and the latest release still lack a primary-skipping fallback-chain test command, and the directly related implementation PRs are closed unmerged while the open health-check issue is broader than this targeted request.

## What This Changes

Add a `/models test-fallback` style command that intentionally skips the primary model and verifies configured fallback models before a real outage.

## Best Possible Solution

Ship one shared, side-effect-free fallback-chain probe behind the chosen command surface that reuses real model/auth resolution, skips the primary by design, supports first and all fallback modes, and reports the answering model.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, release, and related-item inspection show the requested command and an equivalent primary-skipping fallback-chain probe are absent.

## Solution Assessment

Unclear until maintainers choose the product surface. A shared probe service reused by the selected CLI or slash UX is safer than a one-off parser branch because fallback ordering, auth, timeout, rate-limit, and state-mutation policy need one source of truth.
