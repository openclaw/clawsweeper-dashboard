## Summary

Keep open: current main and the latest release still lack a primary-skipping fallback-chain test command, and the direct implementation PR is closed unmerged while the open per-model health-check work is broader and does not replace the requested `/models test-fallback` behavior.

## What This Changes

Request to add a `/models test-fallback` command that proactively verifies configured model fallbacks by skipping the primary and reporting which fallback responds.

## Best Possible Solution

Ship one shared, side-effect-free fallback-chain probe behind the chosen command surface that reuses real model/auth resolution, intentionally skips the primary, supports first and all fallback modes, and reports the answering model.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, release, and related-item inspection show the requested command and an equivalent primary-skipping fallback-chain probe are absent.

## Solution Assessment

Unclear until maintainers choose the product surface. A shared probe service reused by the selected CLI or slash UX is safer than a one-off parser branch because fallback ordering, auth, timeout, rate-limit, and state-mutation policy need one source of truth.
