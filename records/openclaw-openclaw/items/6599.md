## Summary

Keep open: current main and the latest release still lack a primary-skipping fallback-chain test command, and the direct implementation attempt is closed unmerged while the broader health-check issue does not fully replace this targeted slash-command request.

## What This Changes

The issue requests a `/models test-fallback` command that proactively verifies configured fallback models by intentionally skipping the primary model.

## Best Possible Solution

Ship one shared, side-effect-free fallback-chain probe behind the chosen command surface that reuses real model/auth resolution, skips the primary by design, supports first-fallback and all-fallback modes, and reports the answering model with clear live-probe warnings.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, release-tag, and related-item inspection show the requested command and equivalent primary-skipping fallback-chain probe are absent.

## Solution Assessment

Unclear until maintainers choose the product surface and live-probe semantics. A shared probe service reused by the selected CLI or slash UX is safer than a one-off parser branch because fallback ordering, auth, timeout, rate-limit, and state-mutation policy need one source of truth.
