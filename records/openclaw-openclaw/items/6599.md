## Summary

Keep open. Current main still lacks the requested primary-skipping fallback-chain probe: the CLI and slash `/models` surfaces provide listing, status, fallback management, and auth probes, but no `test-fallback` command, and the direct implementation PR is closed unmerged.

## What This Changes

The issue requests a command that proactively verifies the configured model fallback chain by skipping the primary model and probing fallback candidates.

## Best Possible Solution

Ship a focused fallback-chain probe that reuses OpenClaw’s real model/auth request path, starts from configured fallback candidates, reports which model answered, supports first-fallback and all-fallback modes, and documents how it differs from auth-profile probes and broader per-model health checks.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection on current main gives high-confidence evidence that no `/models test-fallback` or `openclaw models test-fallback` surface exists.

## Solution Assessment

Yes. A narrow first-class fallback-chain probe is the best fit; the broader request at https://github.com/openclaw/openclaw/issues/63145 would not by itself prove the primary-skipping fallback behavior requested here.
