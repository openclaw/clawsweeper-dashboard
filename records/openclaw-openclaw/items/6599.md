## Summary

Keep open: current main and the latest release still lack a first-class fallback-chain test command, the direct implementation PR is closed unmerged, and the broader per-model health-check issue does not replace this targeted `/models test-fallback` request.

## What This Changes

The issue requests a `/models test-fallback` command that intentionally skips the primary model and live-probes configured fallback models before a real outage.

## Best Possible Solution

Ship a maintainer-approved fallback-chain probe that reuses real OpenClaw model/auth resolution, intentionally starts at configured fallback candidates, reports the responding model, supports first-fallback and all-fallback modes, and clearly distinguishes itself from auth-profile probes and broad per-model health checks.

## Reproduction Assessment

Yes for the missing feature: current source shows no `/models test-fallback` parser action, slash handler branch, or CLI subcommand. The existing `models status --probe` path is an auth-profile probe, not a primary-skipping fallback-chain test.

## Solution Assessment

Unclear as an immediate implementation path: the requested behavior is useful, but maintainers should first choose its relationship to the broader `models status --probe --all` idea and decide whether CLI, slash command, or both are canonical.
