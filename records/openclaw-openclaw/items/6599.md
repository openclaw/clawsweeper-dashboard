## Summary

Keep open. Current main still lacks a primary-skipping fallback-chain test command, the direct implementation attempt is closed unmerged, and the open per-model health probe work is adjacent rather than a complete replacement.

## What This Changes

Request a `/models test-fallback` command that proactively probes configured fallback models by skipping the primary and optionally testing the full chain.

## Best Possible Solution

Ship a focused fallback-chain probe that reuses the real fallback/auth execution path, starts at configured fallback candidates, reports which model answered, supports first-fallback and all-fallback modes, and documents how it differs from auth-profile probes and per-model health checks.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection on current main confirm that neither `/models test-fallback` nor `openclaw models test-fallback` exists.

## Solution Assessment

Unclear until maintainers settle the command shape and live-probe semantics. The maintainable direction is to reuse the real fallback/auth path and keep this distinct from `models status --probe` auth checks and the broader per-model health-check proposal.
