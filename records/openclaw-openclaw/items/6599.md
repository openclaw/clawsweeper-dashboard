## Summary

Keep open. Current main still does not provide the requested fallback-chain test command, and the nearest open work tracks broader per-model health probing rather than this primary-skipping fallback-chain verification flow.

## What This Changes

Add a `/models test-fallback` slash command, with an optional all-fallbacks mode, to proactively verify configured model fallback chains without waiting for a real provider failure.

## Best Possible Solution

Ship a focused fallback-chain probe that reuses OpenClaw’s real model/auth execution path, starts at configured fallback candidates, reports which model answered, supports first-fallback and all-fallback modes, and documents how it differs from auth-profile probes and broader per-model health checks.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection on current main shows no `/models test-fallback` or `openclaw models test-fallback` surface exists.

## Solution Assessment

Yes, with a small product decision on command shape. A narrow fallback-chain probe is the maintainable direction because `models status --probe` and the open `--probe --all` proposal check configured model reachability, not the requested primary-skipping fallback-chain behavior.
