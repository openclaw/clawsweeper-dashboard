## Summary

Keep open: current main and the latest release still lack a primary-skipping fallback-chain test command, the direct implementation attempts are closed unmerged, and the broader health-check issue does not fully replace this targeted request.

## What This Changes

The issue requests a `/models test-fallback` command that intentionally skips the primary model and probes one or all configured fallback candidates before a real provider failure occurs.

## Best Possible Solution

Ship one shared, read-only fallback-chain probe that reuses real model/auth resolution, skips the primary, supports first and all fallback modes, reports the answering model, and documents how it differs from auth-profile probes and broad per-model health checks.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, release, and related-item checks show the requested command and an equivalent primary-skipping fallback-chain probe are absent.

## Solution Assessment

Unclear until maintainers choose the product surface and live-probe semantics. A shared side-effect-free probe service reused by the selected CLI or slash UX is safer than a one-off parser branch.
