## Summary

Keep open. Current main still lacks a command that deliberately exercises configured fallback candidates; the existing probe checks provider/auth-profile reachability, while the closed implementation PR was never merged and the related open health-check issue covers a broader per-model diagnostic.

## What This Changes

Add a command that deliberately exercises configured fallback models and reports whether the fallback chain is usable before an outage.

## Best Possible Solution

Provide one canonical model-diagnostic implementation that selects configured fallback candidates through the real fallback/auth resolution path, can test the first candidate or the full ordered chain, reports each responding model and failure classification, and exposes consistent CLI and chat presentation without changing configuration.

## Reproduction Assessment

Yes at the source level: current command registration, slash parsing, auth-probe execution, and fallback candidate resolution establish that no dedicated fallback-chain test path exists. This is a feature gap rather than a failing existing behavior.

## Solution Assessment

Unclear. Exercising configured fallbacks is useful, but maintainers should first decide whether the user-facing surface is a dedicated `/models` action, an extension of `openclaw models status --probe`, or both backed by one canonical implementation.
