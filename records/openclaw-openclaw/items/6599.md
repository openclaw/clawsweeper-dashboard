## Summary

Keep open: current main still lacks a `/models test-fallback` command or equivalent primary-skipping fallback-chain probe. The direct implementation attempt is closed unmerged, and the open per-model health-check work is broader CLI probing rather than the requested slash/first-fallback behavior.

## What This Changes

Request adds a `/models test-fallback` command to proactively verify configured model fallback candidates without waiting for a real provider failure.

## Best Possible Solution

Implement one shared fallback-chain probe behind the chosen command surface so first-fallback and full-chain checks reuse real model/auth resolution, report the responding model, and avoid config/session/cooldown side effects by default.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, related PR inspection, and search show the requested command and an equivalent primary-skipping fallback-chain probe are absent on current main.

## Solution Assessment

Unclear until maintainers choose the product surface. A shared probe service reused by the selected CLI or slash UX is safer than a one-off parser branch because fallback ordering, auth, timeout, rate-limit, and state-mutation policy need one source of truth.
