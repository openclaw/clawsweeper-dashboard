## Summary

Keep open. Current main still lacks the requested first-class fallback-chain test command, and the directly related implementation PR is closed unmerged while broader health-check requests do not replace this primary-skipping probe.

## What This Changes

Add a `/models test-fallback` command that proactively verifies configured fallback models, with an option to test the full fallback chain.

## Best Possible Solution

Ship a focused fallback-chain probe that reuses the real model/auth fallback path, starts from configured fallback candidates, reports which model answered, supports first-fallback and all-fallback modes, and documents how it differs from auth-profile probes and broad per-model health checks.

Implement issue #6599 as a focused fallback-chain probe. Add `/models test-fallback` for the slash command surface and the matching `openclaw models test-fallback` CLI surface if keeping model command parity; default to testing the first configured fallback and support an `all`/`--all` mode for the full configured chain. Reuse the real model/auth fallback resolution path, but intentionally start from configured fallback candidates rather than the primary, and pin each probe attempt so later fallbacks do not mask a failed candidate.
