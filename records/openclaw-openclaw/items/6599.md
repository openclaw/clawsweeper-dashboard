## What This Changes

The issue requests a command that proactively verifies configured model fallback chains by intentionally skipping the primary model and probing fallback candidates.

## Best Possible Solution

Ship a focused fallback-chain probe that reuses the real model/auth execution path, starts at configured fallback candidates, reports which model answered, supports first-fallback and all-fallback modes, and documents how it differs from auth-profile probes and broad per-model health checks.
