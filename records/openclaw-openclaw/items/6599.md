## Summary

Keep open. Current main still lacks the requested fallback-chain test command, the direct implementation PR is closed unmerged, and the nearest open work is a broader per-model health probe rather than this primary-skipping fallback-chain check.

## What This Changes

Request to add `/models test-fallback` to intentionally skip the primary model and verify configured fallback models, optionally across the full chain.

## Best Possible Solution

Ship a focused fallback-chain probe that reuses OpenClaw’s real model/auth execution path, starts at configured fallback candidates, reports which model answered, supports first-fallback and all-fallback modes, and documents how it differs from auth-profile probes and per-model health checks.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection on current main show no `/models test-fallback` or `openclaw models test-fallback` surface exists.

## Solution Assessment

Yes, the requested fallback-chain probe is a maintainable direction, but the exact command UX should be settled before implementation. The broader `models status --probe --all` proposal is useful adjacent work, not a complete substitute for primary-skipping fallback-chain verification.
