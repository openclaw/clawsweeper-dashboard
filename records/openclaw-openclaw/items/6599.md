## Summary

Keep open: current main and the latest release still lack a first-class fallback-chain test command, and the direct implementation attempts are closed unmerged while broader model-health issues do not replace this targeted request.

## What This Changes

The issue requests a `/models test-fallback` command that intentionally skips the primary model and probes configured fallback models before a real provider failure.

## Best Possible Solution

Define one canonical fallback-chain probe that reuses the real model/auth fallback path, intentionally starts at configured fallback candidates, reports the responding model, supports first-fallback and all-fallback modes, and clearly distinguishes itself from broader model-health checks.

## Reproduction Assessment

Yes. At source level, current main has no `models test-fallback` CLI command or `/models test-fallback` action, and the existing probe path is provider/auth oriented rather than a primary-skipping fallback-chain test.

## Solution Assessment

Unclear as an exact implementation until maintainers choose the canonical probe UX. A targeted fallback-chain probe is plausible, but it should be reconciled with broader configured-model health checks before implementation is queued.
