## Summary

Keep open. Current main still lacks a primary-skipping fallback-chain test command, the direct implementation attempt is closed unmerged, and the nearest open per-model health probe work is adjacent rather than a complete substitute.

## What This Changes

Request adds a proactive `/models test-fallback` command that skips the primary model and probes configured fallback candidates, optionally across the full chain.

## Best Possible Solution

Ship a focused fallback-chain probe that reuses OpenClaw's real fallback/auth execution path, starts at configured fallback candidates, supports first-fallback and all-fallback modes, and documents how it differs from auth-profile probes and per-model health checks.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection on current main confirm that neither `/models test-fallback` nor `openclaw models test-fallback` exists.

## Solution Assessment

Unclear until the maintainer-facing command shape is settled. A first-class fallback-chain probe is the right direction, but the adjacent `models status --probe --all` work is broader health checking and not a complete primary-skipping fallback-chain test.
