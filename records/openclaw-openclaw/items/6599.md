## Summary

Keep open. Current main still does not provide a primary-skipping fallback-chain test command, and the only direct implementation attempts are closed unmerged while the open per-model health-check request is adjacent rather than a complete replacement.

## What This Changes

The issue requests a `/models test-fallback` or equivalent command that proactively exercises configured fallback models without waiting for a real provider failure.

## Best Possible Solution

Ship a focused fallback-chain probe that reuses the real model/auth execution path, starts at configured fallback candidates, reports which model answered, supports first-fallback and all-fallback modes, and documents how it differs from `models status --probe` and broader per-model health checks.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection on current main confirm that neither `/models test-fallback` nor `openclaw models test-fallback` exists.

## Solution Assessment

Unclear until maintainers settle the command shape and live-probe semantics. The maintainable direction is to reuse the existing fallback/auth path rather than adding an independent provider-specific probing loop.
