## Summary

Keep open: current main and v2026.6.5 still lack the requested primary-skipping fallback-chain test command, the direct implementation PR is closed unmerged, and the broader per-model health-check issue does not fully replace this targeted slash/command request.

## What This Changes

Request adding a `/models test-fallback` command that proactively skips the primary model and probes configured fallback candidates, with an optional all-fallback mode.

## Best Possible Solution

Ship a shared, read-only fallback-chain probe that reuses real model/auth resolution, skips the primary, supports first-fallback and all-fallback modes, reports the answering model, and documents how it differs from auth-profile probes and broad per-model health checks.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the product surface and live-probe semantics; a shared side-effect-free probe service reused by the selected CLI or slash UX is safer than a one-off parser branch.
