## Summary

Keep open: current main and v2026.6.5 still lack a primary-skipping fallback-chain test command, the direct implementation PR at https://github.com/openclaw/openclaw/pull/43342 is closed unmerged, and the broader health-check issue at https://github.com/openclaw/openclaw/issues/63145 does not exactly replace the requested `/models test-fallback` behavior.

## What This Changes

Request to add a `/models test-fallback` command that intentionally skips the primary model and probes configured fallback candidates before a real outage.

## Best Possible Solution

Ship one shared, read-only fallback-chain probe that reuses real model/auth fallback resolution, intentionally skips the primary, supports first and all fallback modes, reports the answering model, and documents how it differs from auth-profile probes and broader per-model health checks.

## Solution Assessment
