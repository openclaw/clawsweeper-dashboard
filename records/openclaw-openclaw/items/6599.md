## Summary

Keep open: current main still lacks the requested fallback-chain test command or an equivalent primary-skipping probe. The direct implementation PR at https://github.com/openclaw/openclaw/pull/43342 is closed unmerged, and the open per-model health-check work at https://github.com/openclaw/openclaw/pull/63259 overlaps the diagnostics goal but does not settle the requested slash/first-fallback behavior.

## What This Changes

Request to add a `/models test-fallback` command that intentionally skips the primary model and probes one or all configured fallback models.

## Best Possible Solution

Ship one shared fallback-chain probe behind the chosen slash/CLI surface that reuses real model/auth resolution, skips the primary by design, supports first/all fallback modes, reports the answering model, and avoids config/session/cooldown side effects by default.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, related PR inspection, and search show the requested command and an equivalent primary-skipping fallback-chain probe are absent on current main.

## Solution Assessment

Unclear until maintainers choose the product surface. A shared probe service reused by the selected CLI or slash UX is safer than a one-off parser branch because fallback ordering, auth, timeout, rate-limit, and state-mutation policy need one source of truth.
