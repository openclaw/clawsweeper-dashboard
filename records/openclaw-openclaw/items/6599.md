## Summary

Keep open. Current main still lacks a first-class fallback-chain test command: CLI and slash-command surfaces expose model listing/status/fallback management and auth probes, but nothing that intentionally skips the primary and walks configured fallbacks.

## What This Changes

Add a `/models test-fallback` command, with an optional all-fallback mode, to proactively verify configured model fallback chains.

## Best Possible Solution

Ship a focused fallback-chain probe that reuses the real model/auth fallback path, starts at configured fallback candidates, reports which model answered, supports first-fallback and all-fallback modes, and documents how it differs from auth-profile probes and broader per-model health checks.

## Reproduction Assessment

Not applicable as a bug reproduction. This is a feature request, and the current-main source/docs checks give a high-confidence way to verify the requested command is absent.

## Solution Assessment

Yes. A narrow first-class fallback-chain probe is the best fit; broad `models status --probe --all` health checking remains a related but different feature and would not by itself test the primary-skipping fallback path.
