## Summary

Keep open: current main still lacks the requested fallback-chain test command, and the existing model status probe checks auth/profile reachability rather than intentionally starting at configured fallback candidates.

## What This Changes

Request a `/models test-fallback` command that skips the primary model, probes configured fallback models, and reports which model responds.

## Best Possible Solution

Implement one shared fallback-probe service behind the selected command surface so first-fallback and full-chain checks reuse real model/auth resolution, report the responding model, and avoid mutating config, session overrides, or cooldown state.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, related PRs, and current-main inspection show the requested command and primary-skipping probe path are absent.

## Solution Assessment

Unclear until maintainers choose the product surface. A shared probe service reused by the selected CLI or slash UX is safer than a one-off parser branch because fallback, auth, timeout, and state-mutation policy need one source of truth.
