## Summary

Keep open: current main and v2026.5.22 still lack a primary-skipping `/models test-fallback` path, the direct implementation attempts are closed unmerged, and the open `models status --probe --all` work is a broader CLI health-check path rather than this slash-command fallback-chain probe.

## What This Changes

The issue requests a `/models test-fallback` command that intentionally skips the primary model and probes configured fallback models before a real outage.

## Best Possible Solution

Ship one shared, side-effect-free fallback-chain probe behind the chosen command surface that reuses real model/auth resolution, intentionally skips the primary, supports first-fallback and all-fallback modes, and reports which model answered.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, release, and related-item inspection show the requested command and an equivalent primary-skipping fallback-chain probe are absent.

## Solution Assessment

Unclear until maintainers choose the product surface and live-probe semantics. A shared probe service reused by the selected CLI or slash UX is safer than a one-off parser branch because fallback ordering, auth, timeout, rate-limit, and state-mutation policy need one source of truth.
