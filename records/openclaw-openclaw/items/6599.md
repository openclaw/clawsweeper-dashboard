## Summary

Keep open: current main and the latest release still lack a primary-skipping fallback-chain test command, the direct implementation attempt at https://github.com/openclaw/openclaw/pull/43342 is closed unmerged, and https://github.com/openclaw/openclaw/issues/63145 is broader per-model health checking rather than a replacement for this targeted workflow.

## What This Changes

Request a `/models test-fallback` command that intentionally skips the primary model and verifies configured fallback models before a real provider failure.

## Best Possible Solution

Ship one shared, read-only fallback-chain probe that reuses real model/auth resolution, skips the primary, supports first-fallback and all-fallback modes, reports the answering model, and documents how it differs from auth-profile probes and broad per-model health checks.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the product surface and live-probe semantics; a shared side-effect-free probe service reused by the selected CLI or slash UX is safer than a one-off parser branch.
