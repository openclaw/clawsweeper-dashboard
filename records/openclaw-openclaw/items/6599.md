## Summary

Keep open. Current main still lacks the requested primary-skipping fallback-chain test command, and the related open per-model probe work does not fully replace this slash-command workflow.

## What This Changes

Add a `/models test-fallback`-style command that intentionally skips the primary model and probes configured fallback models.

## Best Possible Solution

Define the boundary with `models status --probe --all`; if distinct, implement a first-class fallback-chain probe that reuses OpenClaw’s real fallback/auth path and documents its live-request behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection on current main confirm that neither `/models test-fallback` nor `openclaw models test-fallback` exists.

## Solution Assessment

Unclear until maintainers settle whether a targeted primary-skipping fallback-chain probe should exist alongside broader per-model probing. If it does, the maintainable path is to reuse the real fallback/auth resolution path rather than adding a separate provider-specific probe loop.
