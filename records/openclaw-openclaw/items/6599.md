## Summary

Keep open. Current main still lacks a proactive fallback-chain test command on both the CLI and `/models` chat surfaces, and the direct implementation PR is closed unmerged rather than superseding this issue.

## What This Changes

Add a `/models test-fallback` or equivalent models command that intentionally starts at configured fallback candidates and reports whether the fallback chain is usable.

## Best Possible Solution

Add a first-class fallback-chain probe that reuses OpenClaw’s real fallback/auth resolution, starts at configured fallback candidates, reports per-candidate outcomes and the responding model, and documents how it differs from auth-profile probes and broader model health checks.

## Solution Assessment

Yes. A focused fallback-chain probe is the narrow maintainable direction; #63145 may share probe infrastructure, but its broader per-model health check does not replace the requested `/models test-fallback` workflow.
