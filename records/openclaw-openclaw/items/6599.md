## What This Changes

The issue requests a `/models test-fallback` command that skips the primary model, probes configured fallback models, and reports which fallback succeeds or fails.

## Best Possible Solution

Design one canonical fallback-chain probe that reuses existing model/auth/fallback resolution, supports first-fallback and full-chain modes, reports the answered model clearly, avoids mutating config/session state, and documents how it differs from auth-profile probes and broader per-model health checks.

## Reproduction Assessment

Yes, source inspection is enough: current main and `v2026.6.6` expose auth/profile probes and fallback configuration, but no command path that skips the primary and probes the fallback chain.

## Solution Assessment

Unclear until maintainers choose the canonical command shape; the best implementation should share the existing probe and fallback-resolution backend instead of adding a separate duplicated health-check path.
