## Summary

Keep open: current main and v2026.6.5 still lack an operator-facing fallback-chain test; existing probes either validate provider authentication or retry the primary after automatic failover, so they do not solve the requested proactive fallback validation.

## What This Changes

Add a command that deliberately skips the primary model and tests the first or all configured fallback candidates.

## Best Possible Solution

Provide one canonical live model-health interface that resolves fallback candidates exactly as runtime does, can target the first or entire fallback chain, reports the responding model and structured failures, and prevents probes from mutating normal session or cooldown state.

## Reproduction Assessment

Yes at source level: current main and v2026.6.5 can be inspected directly, and neither exposes a command that intentionally begins with configured fallback candidates.

## Solution Assessment

Unclear: reusing canonical fallback and authentication resolution is the maintainable implementation boundary, but maintainers must choose whether the public surface belongs under `models status`, a dedicated CLI command, `/models`, or a coordinated combination.
