Keep open: current main still lacks a first-class fallback-chain test command, and the remaining work is a new model/auth command surface that needs maintainer product review.

## What This Changes

Request a `/models test-fallback` command that intentionally skips the primary model and probes configured fallback candidates.

## Best Possible Solution

Choose one canonical fallback-probe UX, then implement it through shared model/auth resolution so first-fallback and full-chain checks report the responding model without duplicating normal failover policy.

## Solution Assessment
