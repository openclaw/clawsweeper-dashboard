## Summary

Keep open: current main and v2026.5.22 still lack a primary-skipping fallback-chain test command, the direct implementation attempt at https://github.com/openclaw/openclaw/pull/43342 is closed unmerged, and the open per-model health-check work at https://github.com/openclaw/openclaw/issues/63145 is broader rather than an exact replacement.

## What This Changes

The issue asks for a `/models test-fallback` command that intentionally skips the primary model and probes configured fallback candidates before a real provider failure occurs.

## Best Possible Solution

Ship one shared, side-effect-conscious fallback-chain probe behind the chosen command surface that reuses real model/auth resolution, intentionally skips the primary, supports first-fallback and all-fallback modes, and reports which fallback answered.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, release, and related-item inspection show the requested command and an equivalent primary-skipping fallback-chain probe are absent.

## Solution Assessment

Unclear until maintainers choose the product surface. A shared probe service reused by the selected CLI or slash UX is safer than a one-off parser branch because fallback ordering, auth, timeout, rate-limit, and state-mutation policy need one source of truth.
