## Summary

Keep open: current main and the latest release still lack a primary-skipping fallback-chain test command, and the direct implementation attempts are closed unmerged while the broader health-check issue does not exactly replace this targeted request.

## What This Changes

The issue requests a `/models test-fallback` command, with an optional all-fallback mode, to proactively verify configured model fallback candidates and auth reachability.

## Best Possible Solution

Ship one shared, read-only fallback-chain probe that reuses real model/auth fallback resolution, intentionally skips the primary, supports first and all fallback modes, reports the answering model, and documents how it differs from auth-profile probes and broad per-model health checks.

## Solution Assessment

Unclear until maintainers choose the product surface and live-probe semantics. A shared side-effect-free probe service reused by the selected CLI or slash UX is safer than a one-off parser branch.
