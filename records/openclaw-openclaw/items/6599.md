## Summary

Keep open. Current main still has no proactive fallback-chain test command: `/models` only lists providers/models, `models status --probe` remains auth-profile oriented, and the direct implementation PR is closed unmerged while the broader per-model health issue is not an exact substitute.

## What This Changes

Request to add a `/models test-fallback` command that intentionally skips the primary model and probes configured fallback candidates, optionally the whole chain.

## Best Possible Solution

Ship a shared fallback-chain probe that reuses OpenClaw’s real model/auth fallback resolution, starts at configured fallback candidates, reports the responding model and per-candidate results, and documents how it differs from auth-profile probes and broader health checks.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is static inspection of current main’s CLI registration, slash parser/handler, docs, tests, and fallback runtime, all of which show the requested command is absent.

## Solution Assessment

Yes. A first-class fallback-chain probe is the narrow maintainable direction; folding it into `models status --probe` would blur auth-profile reachability with configured fallback-chain execution unless maintainers intentionally merge it with the broader #63145 design.
