## Summary

Keep open. Current main still lacks the requested proactive fallback-chain test command; the existing model surfaces only list/configure/probe auth, and the related implementation PR is closed unmerged while the later health-check request is broader rather than a substitute.

## What This Changes

The issue requests a `/models test-fallback` command that intentionally skips the primary model and verifies configured fallback models before a real provider failure.

## Best Possible Solution

Ship a shared fallback-chain probe that reuses OpenClaw’s real model/auth fallback resolution, starts at configured fallback candidates, reports per-candidate results and the responding model, supports first-fallback and full-chain modes, and documents how it differs from auth-profile probes and broader model health checks.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is static inspection of current main’s CLI registration, slash parser/handler, docs, tests, and fallback runtime, all of which show the requested command is absent.

## Solution Assessment

Yes. A first-class fallback-chain probe is the narrow maintainable direction; folding it into `models status --probe` would blur auth/profile reachability with configured fallback-chain execution unless maintainers explicitly merge it with the broader #63145 health-check design.
