## What This Changes

Request to add a `/models test-fallback` diagnostics command that skips the primary model and probes one or all configured fallback candidates.

## Best Possible Solution

Ship one shared, read-only fallback-chain probe that reuses real model/auth resolution, intentionally skips the primary, supports first and all fallback modes, and reports the answering model without mutating state.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, release, and related-item checks show the requested command and an equivalent primary-skipping fallback-chain probe are absent.

## Solution Assessment

Unclear until maintainers choose the product surface. A shared side-effect-free probe service reused by the selected CLI or slash UX is safer than a one-off parser branch because fallback ordering, auth, timeout, rate-limit, and state-mutation policy need one source of truth.
