## Summary

Keep open. Current main still lacks a user-invoked fallback-chain test command: neither the CLI nor slash `/models` surface has `test-fallback`, existing `models status --probe` remains auth/profile oriented, and the real fallback resolver begins with the primary model rather than intentionally starting at configured fallback candidates.

## What This Changes

The issue requests a `/models test-fallback` style command that skips the primary model and proactively verifies one or all configured fallback models.

## Best Possible Solution

Define how the targeted primary-skipping fallback probe relates to `models status --probe --all`, then ship a first-class command only if maintainers still want that distinct workflow.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection on current main confirm that neither `/models test-fallback` nor `openclaw models test-fallback` exists.

## Solution Assessment

Unclear until maintainers settle the command shape and its relationship to broader per-model health probing. The maintainable implementation would reuse OpenClaw’s real fallback/auth resolution path rather than adding a separate provider-specific probe loop.
