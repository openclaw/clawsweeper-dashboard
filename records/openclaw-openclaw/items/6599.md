## Summary

Keep open: current main still lacks a first-class fallback-chain test command, and the remaining work is a new live model/auth probing surface that needs maintainer product guidance before implementation.

## What This Changes

Request to add a `/models test-fallback` or equivalent command that intentionally skips the primary model and probes configured fallback candidates.

## Best Possible Solution

Choose one canonical fallback-probe UX, then implement it through shared model/auth fallback resolution so first-fallback and full-chain checks report the responding model without mutating session or config state.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, release-tag, and related-PR inspection give high-confidence proof that no `/models test-fallback` command or primary-skipping fallback probe exists on current main or v2026.5.12.

## Solution Assessment

Unclear until maintainers choose the product surface. A shared probe service reused by the chosen CLI/slash surface is safer than a one-off slash-only implementation because it avoids duplicating fallback and auth routing policy.
