## Summary

Keep open: current main still lacks a first-class fallback-chain test command, and the remaining work is a new product/UX surface rather than an already-shipped fix.

## What This Changes

The issue requests a proactive `/models test-fallback` command that intentionally skips the primary model and verifies configured fallback candidates before a real provider failure.

## Best Possible Solution

Choose the command surface deliberately, then implement a shared fallback-chain probe that reuses real model/auth resolution, starts at configured fallback candidates, reports the responding model, supports first-only and full-chain modes, and documents how it differs from auth-profile probes.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high-confidence proof that current main has no `/models test-fallback` parser branch, CLI subcommand, docs entry, or primary-skipping fallback probe path.

## Solution Assessment

Unclear until maintainers choose the product surface. The requested command is a plausible narrow solution, but the overlap with `models status --probe --all` means the best implementation may be a shared probe service exposed through one or both command surfaces.
