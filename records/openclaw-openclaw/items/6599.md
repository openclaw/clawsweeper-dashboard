## Summary

Keep open: current main still lacks the requested `/models test-fallback` or equivalent primary-skipping fallback-chain probe, and the related open health-check work is broader CLI probing rather than this exact slash-command workflow.

## What This Changes

Add a command, proposed as `/models test-fallback`, that intentionally skips the primary model and verifies configured fallback models before a real outage.

## Best Possible Solution

Implement one shared fallback-probe service behind the chosen command surface so first-fallback and full-chain checks reuse real model/auth resolution, report the responding model, and avoid mutating config, session overrides, or cooldown state.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, live issue context, and related PR inspection show the requested command and primary-skipping probe path are absent on current main.

## Solution Assessment

Unclear until maintainers choose the product surface. A shared probe service reused by the selected CLI or slash UX is safer than a one-off parser branch because fallback, auth, timeout, rate-limit, and state-mutation policy need one source of truth.
