## Summary

Keep open. Current main still lacks the requested primary-skipping fallback-chain test command, and the open broader per-model probe request does not fully replace the slash-command and first-fallback workflow described here.

## What This Changes

Add a `/models test-fallback` command that intentionally skips the primary model and probes fallback candidates, optionally the full chain.

## Best Possible Solution

Define the boundary with https://github.com/openclaw/openclaw/issues/63145; if this remains distinct, implement a read-only fallback-chain probe that reuses real fallback/auth resolution, starts at configured fallback candidates, reports the responding model, and documents its live-request behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection on current main confirm that neither `/models test-fallback` nor `openclaw models test-fallback` exists.

## Solution Assessment

Unclear until maintainers settle whether a targeted primary-skipping fallback-chain probe should exist beside broader per-model health checks. If it does, the maintainable path is to reuse the real fallback/auth resolution path rather than adding a separate provider-specific probe loop.
