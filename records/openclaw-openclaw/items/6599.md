## Summary

Keep open: current main and the latest release still lack a `/models test-fallback` or equivalent primary-skipping fallback-chain probe, and the direct implementation attempts are closed unmerged while the broader per-model health-check issue does not fully replace this targeted request.

## What This Changes

Request a `/models test-fallback` command that intentionally skips the primary model and probes one or all configured fallback models to verify auth and reachability before an outage.

## Best Possible Solution

Ship one shared, read-only fallback-chain probe that reuses real model/auth resolution, skips the primary by design, supports first-fallback and all-fallback modes, reports the answering model, and documents how it differs from auth-profile probes and broad per-model health checks.

## Reproduction Assessment

Yes for source behavior: current main can be inspected to show the `/models` parser, slash handler, CLI registration, docs, and latest release have no `test-fallback` or equivalent primary-skipping fallback-chain probe. This is not a bug reproduction because the item requests a new diagnostic capability.

## Solution Assessment

Unclear until maintainers choose the product surface and live-probe semantics. A shared side-effect-free probe service reused by the selected CLI or slash UX is safer than a one-off parser branch or a separate provider-call implementation.
