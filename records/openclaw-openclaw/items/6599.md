## Summary

Keep open: current main still lacks a first-class fallback-chain test command, the direct implementation PR is closed unmerged, and the remaining work needs maintainer product direction for live probe UX and policy.

## What This Changes

Add a `/models test-fallback` command that intentionally skips the primary model and probes configured fallback candidates, optionally across the full chain.

## Best Possible Solution

Choose one canonical fallback-probe UX and implement it through shared model/auth fallback resolution so first-fallback and full-chain checks report the responding model without changing config, session overrides, or cooldown state.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, related-PR, and current-main inspection show the requested command and primary-skipping probe path are absent.

## Solution Assessment

Unclear until maintainers choose the product surface. A shared probe service reused by the selected CLI or slash UX is safer than a one-off command because it keeps fallback, auth, timeout, and state-mutation policy in one place.
