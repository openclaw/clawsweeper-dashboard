## Summary

Keep open: current main still lacks a first-class fallback-chain test command, and the remaining work is a new live model/auth probe surface that needs maintainer product review.

## What This Changes

Add a `/models test-fallback` command to intentionally skip the primary model, probe configured fallback candidates, and optionally test the whole fallback chain.

## Best Possible Solution

Choose one canonical fallback-probe UX, then implement it through shared model/auth fallback resolution so first-fallback and full-chain checks report the responding model without mutating session or config state.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high-confidence proof that no `/models test-fallback` command or primary-skipping fallback probe exists.

## Solution Assessment

Unclear until maintainers choose the product surface. A shared probe service reused by the chosen CLI/slash surface looks safer than a one-off slash-only implementation because it avoids duplicating fallback and auth routing policy.
