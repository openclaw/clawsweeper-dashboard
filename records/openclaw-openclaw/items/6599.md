## Summary

Keep open: current main and the latest release still lack a primary-skipping fallback-chain test command, and the open per-model health-check work is related but broader than this targeted `/models test-fallback` request.

## What This Changes

Add a `/models test-fallback` command that skips the primary model and probes the configured fallback chain, optionally across all fallbacks.

## Best Possible Solution

Ship one shared fallback-chain probe behind the chosen command surface that reuses real model/auth resolution, skips the primary by design, supports first and all fallback modes, reports the answering model, and stays side-effect-free by default.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, release, and related-item inspection show the requested command and an equivalent primary-skipping fallback-chain probe are absent.

## Solution Assessment

Unclear until maintainers choose the product surface. A shared probe service reused by the selected CLI or slash UX is safer than a one-off parser branch because fallback ordering, auth, timeout, rate-limit, and state-mutation policy need one source of truth.
