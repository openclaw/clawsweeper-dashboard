## Summary

Keep open. Current main still lacks the requested user-invoked fallback-chain test command: the CLI has status/list/set/fallback management/auth probes, the slash `/models` parser only dispatches provider/list/add flows, and the existing live probe path remains auth/profile oriented rather than a primary-skipping fallback-chain walk.

## What This Changes

The issue requests a proactive `/models test-fallback` or equivalent models CLI command that verifies configured fallback models before a real provider failure.

## Best Possible Solution

Define how this targeted fallback-chain probe relates to `models status --probe --all`, then ship the narrow command only if maintainers still want a primary-skipping fallback verification workflow.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection on current main confirm that neither `/models test-fallback` nor `openclaw models test-fallback` exists.

## Solution Assessment

Unclear until maintainers settle the command shape and relationship to `models status --probe --all`. The maintainable direction is to reuse the real fallback/auth execution path rather than adding an independent provider-specific probe loop.
