## Summary

Keep open: current main still does not provide a first-class fallback-chain test command, and the closest open work is a broader CLI `models status --probe --all` health-check path rather than the requested `/models test-fallback` primary-skipping UX.

## What This Changes

The issue requests a `/models test-fallback` command that proactively verifies configured fallback models without waiting for a provider failure.

## Best Possible Solution

Choose one canonical fallback-probe UX and implement it through shared model/auth fallback resolution so first-fallback and full-chain checks report the responding model without changing config, session overrides, or cooldown state.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, related-PR, and current-main inspection show the requested command and primary-skipping probe path are absent.

## Solution Assessment

Unclear until maintainers choose the product surface. A shared probe service reused by the selected CLI or slash UX is safer than a one-off parser branch because fallback, auth, timeout, and state-mutation policy need one source of truth.
