## Summary

Keep open. Current main still lacks the requested proactive fallback-chain test command: neither the CLI nor `/models` slash-command surface registers `test-fallback`, `models status --probe` remains an auth-profile probe, and the fallback runtime starts with the effective primary before configured fallbacks. The direct implementation PR #43342 is closed and unmerged; #63145 is related but broader per-model health checking rather than an exact substitute.

## What This Changes

Add a `/models test-fallback` command that intentionally skips the primary model, probes configured fallback candidates, and reports which fallback responds, with an option to test the full chain.

## Best Possible Solution

Ship a shared fallback-chain probe that reuses OpenClaw’s real model/auth resolution, intentionally starts at configured fallback candidates, reports the responding provider/model and per-candidate failures, supports first-fallback and full-chain modes, and clearly distinguishes itself from auth-profile probes and broader model health checks.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is static inspection of current main’s CLI registration, slash parser/handler, docs, tests, and fallback runtime, which shows the requested command is absent.

## Solution Assessment

Yes. A first-class fallback-chain probe is the narrow maintainable direction; silently folding this into the existing auth-profile `models status --probe` behavior would blur auth reachability with configured fallback-chain execution unless maintainers deliberately merge it with #63145.
