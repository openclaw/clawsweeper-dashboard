## Summary

Keep open: current main still lacks persisted cron lifecycle hooks or a deterministic pre-run gate, and the remaining work is active product/security design with open implementation candidates rather than cleanup-closeable stale work.

## What This Changes

Request adds pre/post/onError cron hooks and shell/webhook/agent/system hook kinds so scheduled jobs can gate or extend execution before and after the agent run.

## Best Possible Solution

Settle the core/plugin and security contract, then land or explicitly supersede a hardened additive cron lifecycle or pre-run gate with protocol, config, CLI/UI, docs, tests, and clear trust boundaries.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. Source inspection on current main verifies the requested persisted cron hook/pre-run gate contract is absent from storage/runtime types, Gateway schemas, config validation, and scheduled/startup/manual execution paths.

## Solution Assessment
