Status: needs_attention

Summary: The requested cron hook system is security-sensitive because it would store unattended script and webhook execution around scheduled jobs.

  - body: Persisted cron hooks could run local processes or outbound webhooks outside an interactive turn, so maintainers need explicit rules for admin scope, cwd/path/env/secret handling, approval interaction, timeout/cancellation, resource limits, logging, and skip/error semantics.
