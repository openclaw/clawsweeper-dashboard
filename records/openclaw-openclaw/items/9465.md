## Summary

Keep open. Current main and v2026.6.6 now include command cron jobs for direct deterministic scripts, but the requested persisted cron hook lifecycle is still absent from the cron job shape, Gateway schema, and scheduled execution path; the prior hook implementation PRs are closed unmerged, so this remains product/security design work rather than cleanup-ready stale work.

## What This Changes

Requested a cron job hook lifecycle for pre, post, and error custom logic, including shell/webhook/agentTurn hooks and deterministic pre-run gates that can skip model calls.

## Best Possible Solution

Define a maintainer-approved cron lifecycle or gate contract that preserves the shipped command-payload path, adds only the necessary persisted/API/CLI/UI/docs surfaces, and includes explicit security semantics for shell and webhook execution.

## Reproduction Assessment

Not applicable as a bug reproduction. The feature gap is source-verifiable: current main and v2026.6.6 omit persisted cron hook fields and execution phases while retaining only command payloads and observe-only cron_changed hooks.

## Solution Assessment

No, the existing command payload is not the complete solution for this issue. It is the right shipped path for direct no-LLM scripts, but the requested hook lifecycle still needs an owner-reviewed API and security model before implementation.
