## Summary

Keep open. Current main and the latest stable release have command cron jobs for direct no-LLM scripts, but they still do not provide persisted pre/post/onError cron hooks, a pre-run skip gate, or lifecycle hook actions; the remaining work is product and security sensitive.

## What This Changes

Request adds cron job lifecycle hooks that can run custom pre/post/error logic such as shell gates, webhooks, forced reminders, escalation, and conditional execution.

## Best Possible Solution

Define and land a maintainer-approved, additive cron hook lifecycle or narrower pre-run gate with a clear trust model across cron types, protocol, scheduler paths, CLI/UI, docs, and tests.

## Reproduction Assessment

Yes for source-level absence: current main and v2026.6.6 schemas and runtime paths do not accept or execute cron hook fields; this is feature absence, not a broken existing contract.

## Solution Assessment

Unclear. Command payloads are now the right solution for deterministic no-LLM scripts, but the broader pre/post/onError lifecycle, conditional skip, escalation, and webhook/agentTurn hook semantics still need maintainer design.
