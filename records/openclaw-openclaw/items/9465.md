## Summary

Keep open. Current main still lacks persisted cron lifecycle hooks or a deterministic pre-execution gate, and the related implementation PR plus maintainer discussion make this active product/security design work rather than cleanup-ready.

## What This Changes

Request adds cron pre/post/error hook support so jobs can run conditional logic such as shell or webhook gates before and after scheduled execution.

## Best Possible Solution

Review and either land or supersede #45465 with an additive cron hook lifecycle covering persisted job/config types, Gateway protocol, scheduled/startup/manual execution, CLI, Control UI, docs, tests, and an operator-controlled security model.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. There is a high-confidence source verification path: current main has no persisted hooks field and no hook phase before executeJobCoreWithTimeout.

## Solution Assessment

Unclear for the exact proposed schema because this is active product and security design work. The maintainable path is a maintainer-reviewed lifecycle design through #45465 or a successor, not a cleanup close or separate automated repair branch.

## Security Review

Status: needs_attention

Summary: The requested shell/webhook cron lifecycle is security-sensitive and needs explicit design review before implementation.

Concerns:

- **[medium] Define cron hook execution boundaries:**
  - body: Cron shell and webhook hooks would run local commands or outbound network calls around scheduled jobs. The design needs clear operator controls for permissions, timeouts, logging, skip behavior, failure handling, and secret exposure so hooks do not become an implicit privileged execution path.
  - confidence: 0.9
