## Summary

Keep open. Current main still lacks persisted cron lifecycle hooks or a deterministic pre-execution gate, and the related open implementation plus maintainer discussion make this active product/security design work rather than cleanup-ready.

## What This Changes

The issue requests cron job lifecycle hooks, including pre/post/error phases and deterministic pre-execution gating for user logic before model execution.

## Best Possible Solution

Add a maintainer-reviewed cron hook lifecycle spanning persisted job/config types, Gateway protocol, scheduled/startup/manual execution, CLI, Control UI, docs, tests, and explicit security semantics.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The missing feature is source-verifiable on current main: cron persisted types and Gateway schemas have no hooks field, and scheduled/manual runs call executeJobCoreWithTimeout without a pre-hook phase.

## Solution Assessment

Unclear for the exact proposed schema because this is active product and security design work. The maintainable path is to review and land or supersede #45465 rather than close this issue or start a separate automated repair branch.

## Security Review

Status: needs_attention

Summary: The requested shell/webhook cron lifecycle is security-sensitive and needs explicit maintainer design review before implementation.

Concerns:

- **[medium] Define cron hook execution boundaries:**
  - body: Shell and webhook hooks would run local commands or outbound network calls around scheduled jobs. The design needs clear operator controls for permissions, timeouts, logging, skip behavior, failure handling, and secret exposure before becoming a privileged execution path.
