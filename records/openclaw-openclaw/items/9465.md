## Summary

Keep open. Current main and the latest release include command cron jobs for direct zero-LLM script runs, but they still do not implement persisted pre/post/onError cron hooks or a conditional pre-agent gate; the remaining shell/webhook lifecycle semantics need maintainer product and security review.

## What This Changes

The issue requests persisted cron lifecycle hooks, including pre/post/error phases for agent turns, system events, shell commands, and webhooks.

## Best Possible Solution

Design and land a maintainer-approved cron hook or gate API across persisted job/schema, scheduled and manual execution paths, CLI/UI/docs/tests, with an explicit trust model and reuse of command payloads or plugin hooks where those already fit.

## Reproduction Assessment

Yes for source-level current-state verification: the persisted cron shape, protocol schemas, scheduled path, and manual path do not implement the requested lifecycle hooks. This is a feature request rather than a broken existing behavior reproduction.

## Solution Assessment

Unclear as proposed. The user need is real, but the broad shell/webhook lifecycle design needs a maintainer choice between command payloads, plugin hooks, and a persisted hook API before implementation.
