## Summary

Keep open. Current main still lacks the requested persisted cron hook lifecycle, and the active related implementation remains an open product/security review item rather than cleanup-ready stale work.

## What This Changes

Add configurable pre/post/error cron-job hooks that can run agent turns, system events, shell commands, or webhooks around scheduled execution.

## Best Possible Solution

Land or explicitly supersede #45465 with an additive cron hook lifecycle covering persisted job/config types, Gateway protocol, scheduled/startup/manual execution, CLI, Control UI, docs, tests, and a clear security model.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The missing feature is source-verifiable on current main: cron persisted types and Gateway schemas have no hooks field, and scheduled/manual runs call the core executor without a hook phase.

## Solution Assessment

Unclear for the exact proposed schema because this is active product and security design work. The maintainable path is to review and land or supersede #45465 rather than close this issue or start a separate automated repair branch.
