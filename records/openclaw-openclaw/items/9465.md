## Summary

Keep open: current main still lacks persisted cron hooks or a deterministic pre-run gate, and the live discussion plus related open PRs make this active product/security design work rather than cleanup-ready stale work.

## What This Changes

Request to add cron job lifecycle hooks or deterministic pre-run gates so scheduled jobs can run custom logic before or after execution and skip unnecessary model calls.

## Best Possible Solution

Settle one hardened additive cron lifecycle or pre-run gate contract, then land or explicitly supersede the existing open PRs with aligned persisted types, Gateway schemas, scheduled/startup/manual runtime paths, CLI/UI/docs, tests, and real behavior proof.
