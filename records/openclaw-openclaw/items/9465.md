## Summary

Keep open: current main still lacks persisted cron hooks or a deterministic pre-run gate, and the related open PRs plus unattended shell/webhook semantics make this active product and security design work rather than cleanup-ready stale work.

## What This Changes

The issue requests cron lifecycle hooks or pre-run gates so scheduled jobs can run deterministic logic before and after agent execution.

## Best Possible Solution

Select one hardened additive cron lifecycle or pre-run gate contract, then land or explicitly supersede the open implementation PRs with aligned persisted types, Gateway schemas, runtime paths, CLI/UI/docs, tests, and real behavior proof.
