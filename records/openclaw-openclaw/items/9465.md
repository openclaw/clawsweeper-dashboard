## Summary

Keep open. Current main still lacks persisted or executable cron lifecycle hooks, and the active implementation path remains product/security-sensitive rather than cleanup-ready.

## What This Changes

Request to add pre/post/error cron job hooks so scheduled jobs can run custom logic, skip work before model calls, and handle escalation or notification workflows.

## Best Possible Solution

Land or explicitly supersede the open implementation candidate with an additive lifecycle hook API, tests, changelog, UI/CLI/protocol coverage, docs, and a documented security model.
