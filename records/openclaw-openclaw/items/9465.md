## Summary

Keep open. Current main and v2026.6.10 support direct command cron payloads for some zero-LLM jobs, but they still do not implement the requested persisted pre/post/onError cron hooks or a conditional pre-agent gate; all known implementation PRs are closed unmerged and the remaining API/trust model needs maintainer review.

## What This Changes

Requested a persisted cron lifecycle hook system that can run pre/post/error logic such as shell gates, webhooks, and agent turns around scheduled execution.

## Best Possible Solution

Design and land a maintainer-approved cron hook or gate API with an explicit trust model across persisted job/schema, scheduled/startup/manual execution paths, CLI/UI/docs/tests, while reusing existing command payloads and plugin hooks where they already fit.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Yes for current-state verification: source inspection shows the requested persisted hook fields and execution phases are absent on current main and v2026.6.10.

## Solution Assessment

Unclear as proposed. The user need is real, but the broad shell/webhook lifecycle design needs a maintainer choice between command payloads, plugin hooks, and a persisted hook API before implementation.
