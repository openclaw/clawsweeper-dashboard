## Summary

Keep open. Current main still lacks the requested persisted cron hook lifecycle, #45465 remains the active open implementation candidate, and shell/webhook hook execution needs maintainer product and security review before this can be merged or replaced safely.

## What This Changes

The issue asks for configurable pre, post, and error hooks on cron jobs so users can gate, retry, escalate, or run custom processing around scheduled execution.

## Best Possible Solution

Land or explicitly supersede #45465 with an additive cron hook lifecycle covering persisted job/config types, Gateway protocol, scheduled/startup/manual execution, CLI, Control UI, docs, tests, and a clear security model.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The missing feature is source-verifiable on current main: persisted cron types and Gateway schemas have no hook field, and scheduled/startup/manual paths call the core executor without hook phases.

## Solution Assessment

Unclear for the exact proposed schema because this is active product and security design work. The maintainable path is to review and land or supersede #45465 rather than close this issue or start a separate automated repair branch.
