- `impact:session-state`: Cron hooks or gates would change scheduled-run state, skips, retries, failure counts, and whether an agent session is started.

## Summary

Keep this open: current main and v2026.6.6 added direct command cron payloads, but they still do not provide persisted lifecycle hooks or a pre-model gate around agentTurn cron jobs, so #9465 remains the canonical product/security design item.

## What This Changes

The issue requests cron job lifecycle hooks, including pre-run gating before LLM execution and post/error hooks for scheduled jobs.

## Best Possible Solution

Keep #9465 as the canonical design item and choose one additive, security-reviewed cron hook or gate contract that covers scheduled and manual runs, protocol/types, CLI/UI/docs, and upgrade behavior.

## Reproduction Assessment

Yes at source level: current types, protocol schemas, runtime dispatch, and docs expose command payloads but no persisted lifecycle hook or pre-model gate for agentTurn cron jobs.

## Solution Assessment

Unclear as implementation direction: direct command jobs are the right solution for pure no-LLM execution, while conditional agentTurn gating and lifecycle hooks still need a maintainer-selected contract.
