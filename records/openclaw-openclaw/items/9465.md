## Summary

Keep open: current main and v2026.5.22 still do not provide persisted cron lifecycle hooks or a deterministic pre-run gate, and the linked open PRs show this is active product/security design work rather than conservative cleanup.

## What This Changes

Add persisted cron job hooks or pre-run gates so custom pre/post/error logic can run around scheduled execution, including deterministic shell or webhook gating before agent execution.

## Best Possible Solution

Select one hardened additive cron hook or pre-run gate contract, then land or explicitly supersede the open implementation PRs with aligned persisted types, Gateway schemas, scheduled/startup/manual execution paths, CLI/UI/docs, tests, security policy, and real behavior proof.
