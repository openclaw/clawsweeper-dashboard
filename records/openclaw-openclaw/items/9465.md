## Summary

Keep open. Current main still lacks persisted cron lifecycle hooks or a deterministic pre-execution gate, and the related open implementation PR plus maintainer discussion make this active product/security design work rather than cleanup-ready.

## What This Changes

Request adds persisted cron job hooks for pre/post/error phases, including shell and webhook hooks that can gate, augment, retry, or escalate scheduled execution.

## Best Possible Solution

Review and either land or supersede #45465 with an additive cron hook lifecycle covering persisted job/config types, Gateway protocol, scheduled/startup/manual execution, CLI, Control UI, docs, tests, and an operator-controlled security model.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. There is a high-confidence verification path: current source and schema inspection shows no persisted hooks field and no hook phase before executeJobCoreWithTimeout.

## Solution Assessment

Unclear for the exact proposed schema because this is active product and security design work. The maintainable path is a maintainer-reviewed lifecycle design through #45465 or a successor, not a cleanup close or separate automated repair branch.
