## Summary

Keep open. Current main and v2026.6.8 support direct command cron payloads, but they still do not implement the requested persisted pre/post/onError cron hooks or a conditional pre-agent gate; the remaining work needs maintainer product and security review.

## What This Changes

The issue requests cron lifecycle hooks for pre-run gating, post-run processing, and error handling around scheduled jobs.

## Best Possible Solution

Design and land a maintainer-approved cron hook or gate API with an explicit trust model across persisted job/schema, scheduled/startup/manual execution paths, CLI/UI/docs/tests, while reusing command payloads or plugin hooks for cases they already cover.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request, and source inspection gives a high-confidence current-main check that persisted hook fields and hook execution phases are absent.

## Solution Assessment

Unclear as proposed. The user need is real, but the broad shell/webhook lifecycle design needs a maintainer choice between command payloads, plugin hooks, and a persisted hook API before implementation.
