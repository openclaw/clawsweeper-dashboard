- `impact:security`: The requested hook kinds include shell commands and webhooks that would run unattended from persisted cron configuration.

## Summary

Keep open: current main and the latest release support command-backed cron jobs, but they still do not provide persisted pre/post/error cron hooks or a pre-model gate around agentTurn jobs. The remaining work is valid feature and security design, not cleanup-ready stale work.

## What This Changes

Requests a persisted cron hook system with pre/post/onError phases, including shell, webhook, agentTurn, and systemEvent hook actions around scheduled job execution.

## Best Possible Solution

Keep this as the canonical design item and choose one additive, security-reviewed cron hook or gate contract spanning persisted job/config types, protocol schemas, scheduled/manual/startup execution, CLI/UI/docs, tests, and upgrade behavior.

## Reproduction Assessment

Yes at source level: current cron types, Gateway schemas, runtime execution paths, and docs expose command payloads but no persisted lifecycle hook or pre-model gate for agentTurn cron jobs.

## Solution Assessment

Unclear as an implementation direction. Command payloads are the right solution for pure no-LLM scripts, while conditional agentTurn gating and lifecycle hooks still need a maintainer-selected security and product contract.
