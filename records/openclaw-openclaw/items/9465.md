## Summary

Keep open: current main and v2026.6.6 still lack persisted cron pre/post/onError hooks or a pre-model gate around agentTurn jobs. Command cron now covers pure deterministic no-LLM scripts, but the requested lifecycle and conditional gate behavior remains security-sensitive product design work.

## What This Changes

Add a cron job hooks system for custom pre, post, and error logic around scheduled job execution.

## Best Possible Solution

Keep this as the canonical design item until maintainers choose one additive, security-reviewed cron hook or gate contract across persisted types, protocol, runtime, CLI/UI/docs, tests, and any needed plugin API boundary.

## Reproduction Assessment

Yes at source level: current cron types, Gateway schemas, runtime execution paths, and docs expose command payloads but no persisted lifecycle hook or pre-model gate for agentTurn cron jobs.

## Solution Assessment

Unclear as an implementation direction. Command payloads are the best shipped solution for pure deterministic scripts, while conditional agentTurn gating and lifecycle hooks still need a maintainer-selected security and product contract.
