## Summary

Keep open. Current main still lacks persisted or executable cron lifecycle hooks, and the active implementation path is product/security-sensitive rather than cleanup-ready.

## What This Changes

The issue requests configurable cron job lifecycle hooks for pre-run gates, post-run processing, error handling, retries, escalation, shell commands, and webhooks.

## Best Possible Solution

Land or explicitly supersede the open implementation candidate with an additive lifecycle hook API, docs, tests, changelog, UI/CLI/protocol coverage, and a documented security model.
