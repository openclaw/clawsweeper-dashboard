## Summary

Keep open: current main and v2026.6.6 now cover direct zero-LLM command cron jobs, but they still do not provide the requested persisted pre/post/on-error cron hook lifecycle or pre-run gate around an existing agent turn. The remaining work is product- and security-sensitive, and the recent implementation attempts were closed unmerged as superseded by this canonical design issue.

## What This Changes

Request a cron job hook system with pre, post, and error phases that can run agent turns, system events, shell commands, or webhooks around scheduled execution.

## Best Possible Solution

Keep this issue as the canonical cron hook/gate design item and land only a maintainer-approved, additive API with a clear security model across persisted types, protocol, runtime paths, CLI/UI, docs, and tests.

## Reproduction Assessment

Unclear. The broad hook object is plausible, but the best maintainable shape should reuse shipped command jobs where enough and add only the minimal core or plugin API needed for gated agent turns after security review.
