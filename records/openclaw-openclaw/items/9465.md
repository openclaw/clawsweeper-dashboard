## Summary

Keep open: current main still lacks persisted cron lifecycle hooks or a deterministic pre-run gate, and the linked open implementation PRs show this remains active product/security design work rather than cleanup-ready stale work.

## What This Changes

Request a persisted cron job hook system with pre, post, and error phases, including deterministic shell/webhook-style pre-run gates that can skip model execution.

## Best Possible Solution

Settle the core/plugin and security contract, then land or explicitly supersede a hardened additive cron lifecycle or pre-run gate through https://github.com/openclaw/openclaw/pull/45465 and https://github.com/openclaw/openclaw/pull/62195 while coordinating with https://github.com/openclaw/openclaw/issues/18160 for direct exec-style cron work.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. Source inspection on current main verifies the requested persisted cron hook and pre-run gate contract is absent from storage/runtime types, Gateway schemas, config validation, and scheduled/startup/manual execution paths.

## Solution Assessment

Unclear for the exact proposed schema. The narrow maintainable path is maintainer and secops review of one cron lifecycle or pre-run gate contract, not another competing automated implementation while the open PRs remain unresolved.
