## Summary

Keep open. Current main and v2026.6.10 support command cron jobs for script-only zero-LLM runs, but they still do not implement persisted pre/post/onError cron hooks or a conditional pre-agent gate; known implementation PRs are closed unmerged and the remaining shell/webhook semantics need maintainer product and security review.

## What This Changes

The issue requests cron lifecycle hooks that can run custom pre, post, and error logic around scheduled job execution.

## Best Possible Solution

Design and land a maintainer-approved cron hook or gate API across persisted job/schema, scheduled and manual execution paths, CLI/UI/docs/tests, with an explicit trust model and reuse of command payloads or plugin hooks where those already fit.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Yes for current-state verification: source inspection shows the requested persisted hook fields and execution phases are absent on current main and v2026.6.10.

## Solution Assessment

Unclear as proposed. The user need is real, but the broad shell/webhook lifecycle design needs a maintainer choice between command payloads, plugin hooks, and a persisted hook API before implementation.
