## Summary

Keep open: current main and the latest release support direct command cron payloads, but they still do not provide persisted pre/post/error lifecycle hooks or a pre-model gate around agentTurn cron jobs. The remaining request is valid product/security design work, not cleanup-ready stale work.

## What This Changes

Request adds cron job lifecycle hooks for pre/post/error custom logic such as shell or webhook gates, forced reminders, escalation, and conditional execution.

## Best Possible Solution

Keep this as the canonical design item and choose one additive, security-reviewed cron hook or gate contract spanning persisted job/config types, protocol schemas, scheduled/manual/startup execution, CLI/UI/docs, tests, and upgrade behavior.

## Reproduction Assessment

Yes at source level: current cron types, Gateway schemas, runtime execution paths, and docs expose command payloads but no persisted lifecycle hook or pre-model gate for agentTurn cron jobs.

## Solution Assessment

Unclear as an implementation direction. Command payloads are the right solution for pure no-LLM scripts, while conditional agentTurn gating and lifecycle hooks still need a maintainer-selected security and product contract.
