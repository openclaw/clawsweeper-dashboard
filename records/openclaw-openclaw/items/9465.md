## Summary

Keep open: current main still lacks persisted cron lifecycle hooks or a deterministic pre-run gate, and the active related PRs show this is still product/security design work rather than cleanup-ready stale work.

## What This Changes

The issue requests cron job lifecycle hooks, especially pre-run gates that can skip agent execution before an LLM call.

## Best Possible Solution

Settle the core/plugin and security contract, then land or explicitly supersede a hardened additive cron lifecycle or pre-run gate covering persisted types, Gateway protocol, config validation, scheduled/startup/manual runtime coverage, CLI/UI, docs, tests, and trust boundaries.
