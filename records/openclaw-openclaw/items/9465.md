## Summary

Keep open: current main still lacks persisted cron lifecycle hooks or a deterministic pre-run gate, and this issue remains the canonical tracker for the hook/gating work while the open implementation candidate still needs product and security review.

## What This Changes

Request to add cron job pre/post/error hooks, including shell and webhook-style gates, so scheduled jobs can run custom logic before or after the main payload.

## Best Possible Solution

Settle the core/plugin and security contract, then land or explicitly supersede a hardened additive cron lifecycle or pre-run gate with protocol, config, CLI/UI, docs, tests, and clear trust boundaries.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. Source inspection on current main verifies the requested persisted cron hook contract is absent from storage/runtime types, Gateway schemas, config validation, and scheduled/startup/manual execution paths.

## Solution Assessment

Unclear for the exact proposed schema. The maintainable path is owner and secops review of the lifecycle or pre-run gate contract, likely by landing or explicitly superseding https://github.com/openclaw/openclaw/pull/45465.
