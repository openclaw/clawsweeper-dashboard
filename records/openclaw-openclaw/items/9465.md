## Summary

Keep open. Current main and the latest release still lack persisted cron pre/post/onError hook fields and hook execution phases; the newer command cron payload covers deterministic no-LLM scripts, but not conditional pre-agent gating or a full lifecycle hook contract.

## What This Changes

Request to add persisted cron job lifecycle hooks for pre-run gates, post-run work, error handling, shell/webhook calls, and conditional scheduled execution.

## Best Possible Solution

Design and land a maintainer-approved cron hook or gate API with an explicit trust model across persisted job/schema, scheduled/startup/manual execution paths, CLI/UI/docs/tests, while reusing command payloads or plugin hooks for the cases they already cover.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence current-main check that persisted cron hooks and hook execution phases are absent.

## Solution Assessment

Unclear as proposed. The user need is real, but the broad shell/webhook lifecycle design needs a maintainer choice between command payloads, plugin hooks, and a persisted hook API before implementation.
