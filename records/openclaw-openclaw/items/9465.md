## Summary

Keep open: current main still lacks the persisted cron hook lifecycle, and the remaining work is active product/security design rather than cleanup-ready.

## What This Changes

The issue requests cron job lifecycle hooks for pre/post/error custom logic, including shell gates, agent turns, system events, and webhooks around scheduled execution.

## Best Possible Solution

Settle the core-versus-plugin and security contract, then land or supersede a hardened additive lifecycle hook design with protocol, config, CLI/UI, docs, tests, and a documented trust boundary.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. Source inspection on current main verifies the requested persisted cron hook contract is absent from storage/runtime types, Gateway schemas, config validation, CLI/UI surfaces, and scheduled/startup/manual execution paths.
