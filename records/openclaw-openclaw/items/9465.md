## Summary

Keep open. Current main still lacks executable or persisted cron lifecycle hooks, and the active implementation candidate still needs maintainer product/security review around scheduled shell/webhook execution before this can land.

## What This Changes

The issue requests configurable cron lifecycle hooks around scheduled job execution for deterministic pre-run gates, post/error handling, reminders, escalation, and external processing.

## Best Possible Solution

Land or explicitly supersede https://github.com/openclaw/openclaw/pull/45465 with an additive cron hook lifecycle covering persisted job/config types, Gateway protocol, scheduled/startup/manual execution, CLI, Control UI, docs, tests, and a documented security model.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. Source inspection on current main shows the requested hook contract is absent from cron storage/runtime types, Gateway schemas, config validation, UI types/forms, and scheduled/startup/manual execution paths.

## Solution Assessment

Unclear for the exact proposed schema because this is active product and security design work. The maintainable path is to review, harden, and land or supersede the open implementation candidate rather than close this issue or start a separate automated repair branch.
