## Summary

Keep open. Current main and v2026.6.6 still lack persisted cron pre/post/onError hook fields and hook execution phases; command payloads now cover direct deterministic no-LLM scripts, but not conditional pre-agent gating or a full cron lifecycle hook contract.

## What This Changes

The issue requests cron job lifecycle hooks for pre-run gating, post-run/on-error actions, shell/webhook execution, and conditional notification flows.

## Best Possible Solution

Design and land a maintainer-approved cron hook or gate API with an explicit trust model across persisted job/schema, scheduled/startup/manual execution paths, CLI/UI/docs/tests, while reusing command payloads or plugin hooks for the cases they already cover.
