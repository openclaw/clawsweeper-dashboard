## Summary

Keep open. The issue asks for browser upload/file chooser flows to avoid stale split upload-then-click failures and preserve action-specific timeout errors.

## Best Possible Solution

Keep this as the canonical browser upload reliability issue until maintainers land or replace the timeout-budget work and define an atomic upload-trigger/stale-ref error contract with regression coverage and real managed-browser proof.

## Solution Assessment
