Keep open. Current main still exposes the stale-ref upload path: file-chooser arming can be followed by a click on the original ref, and the open related timeout PR covers only the client timeout budget.

Make browser file-chooser uploads robust on dynamic pages by preferring an atomic upload+trigger flow and preserving action-specific timeout or stale-ref errors.

## Best Possible Solution

Keep this open until browser upload-with-trigger has an agreed atomic, abort-aware path that preserves stale-ref errors and has focused dynamic-page regression coverage.
