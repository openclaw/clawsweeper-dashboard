Keep open. Current main still exposes the split upload-arming plus stale-ref click path, the file-chooser client call still uses a fixed 20s deadline, and the related open timeout PR only addresses one slice of the broader upload-trigger/error-preservation problem.

The issue asks browser upload/file chooser flows to avoid split stale-ref failures, keep action errors specific, and prefer an atomic upload-plus-trigger path.

## Best Possible Solution
