## Label Justifications

- `P2`: This is a real but bounded browser automation reliability and diagnostics defect that does not make the core runtime broadly unusable.
- `impact:session-state`: Dynamic rerenders can invalidate stored browser refs between requests, causing the automation's target state to drift from the live page.

## Summary

The issue remains necessary: current main and the latest release provide an atomic upload-plus-trigger request, but the file-chooser client still uses a fixed 20-second outer timeout and the HTTP timeout path still reports browser-control-service unreachability. The closed related PRs are not safe canonical replacements because the direct timeout fixes were not merged, while the merged batch/selector work only reduces some stale-ref exposure and does not resolve this upload path.

## What This Changes

Fix browser file-chooser uploads so dynamic-page ref staleness is avoided through the atomic upload path and in-route action timeouts are not misreported as control-service outages.

## Best Possible Solution

Keep one canonical target-bearing upload request for normal agent use, retain arm-only mode only if maintainers consider it a supported public workflow, align the client budget with the chooser/action operation, and distinguish local action expiry from an unreachable browser control service with focused contract tests and live upload proof.

## Reproduction Assessment

Yes, from source with high confidence: current main still has a fixed 20-second file-chooser client timeout and generic HTTP timeout rewriting, and separate ref-based requests can cross a page rerender boundary. A live current-main dynamic-page reproduction was not executed during this read-only review.

## Solution Assessment

No, a longer timeout alone is not the best complete solution. The narrow maintainable direction is to use the existing atomic upload target path consistently and preserve action-specific failure context, with maintainers deciding whether arm-only upload remains a supported agent workflow.
