## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Is this the best way to solve the issue? Not yet: reading the Windows build directory or ProductVersion metadata is the right owner-local direction, but the patch should preserve the existing fallback and needs real Windows proof.

- **[P1] Preserve the Windows version fallback:** `extensions/browser/src/browser/chrome.executables.ts:749`
