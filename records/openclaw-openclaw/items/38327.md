## What This Changes

The issue reports a regression where `google-vertex/gemini-3.1-pro-preview` causes embedded agent replies to fail with `Cannot convert undefined or null to object` after upgrading to 2026.3.2.

## Best Possible Solution

Land or reject https://github.com/openclaw/openclaw/pull/83991 for the sanitizer guard with route proof, then track a separate dependency/auth-stack item only if the gaxios/native-fetch failure is still validated.

## Reproduction Assessment

Yes, at source level: current main has a concrete path where `google-vertex` and `google-gemini-cli` share the Google stream wrapper while bypassing the sanitizer guarded only for `google-generative-ai`. I did not run a live Vertex credential flow in this read-only review.

## Solution Assessment

Yes for the sanitizer gap: widening the existing Google-family predicate with regression coverage is the narrow maintainable fix. The possible gaxios/native-fetch path should remain separate unless maintainers validate that dependency/auth-stack failure on current main.
