## Summary

Keep open. Current main and the latest checked release still have a `google-generative-ai`-only Google thinking payload guard, while Vertex and Gemini CLI are routed through the same Google wrapper family; open PR #73416 is the focused closing candidate and should be reviewed before this issue is closed.

## What This Changes

The issue asks OpenClaw to stop embedded Google Vertex/Gemini 3.1 Pro preview runs from crashing with `Cannot convert undefined or null to object` after the 2026.3.2 update.

## Best Possible Solution

Keep this issue open until PR #73416 is reviewed and either landed or rejected. The desired end state is a narrow Google wrapper guard fix with regression coverage for `google-vertex/gemini-3.1-pro-preview`, `google-vertex/gemini-3-flash-preview`, and `google-gemini-cli`; any gaxios/native-fetch fallback should be split into its own dependency-patch review.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to run the Google thinking wrapper with `model.api: "google-vertex"` or `"google-gemini-cli"` and a Gemini 3 payload containing `thinkingConfig.thinkingBudget: -1`; current main leaves that payload unsanitized because the guard only matches `google-generative-ai`.

## Solution Assessment

Yes for the wrapper root cause: widening the guard to the Google API shapes that already flow through this wrapper is the narrowest maintainable fix. The gaxios/native-fetch concern should not be bundled into this issue's primary fix without separate dependency and supply-chain review.
