The issue reports a 2026.3.2 regression where `google-vertex/gemini-3.1-pro-preview` embedded agent runs fail on every message with `Cannot convert undefined or null to object`.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/83991 for the sanitizer guard, then keep this issue open only until the gaxios/native-fetch auth-stack concern is either validated into a separate narrow fix or ruled out on current main.

Yes, at source level: current main routes Google stream-family payloads through a guard that only admits `google-generative-ai`, while current Google model tests show `google-vertex` can resolve through `google-gemini-cli`. ## Solution Assessment
