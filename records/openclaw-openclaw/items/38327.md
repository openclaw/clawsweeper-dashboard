## Summary

Keep open: current main and the latest release still run the Google thinking payload sanitizer only for `google-generative-ai`, while the active sanitizer fix is still an open PR rather than merged or shipped.

## What This Changes

The issue reports that Google Vertex Gemini preview models crash embedded agent runs with `Cannot convert undefined or null to object` after upgrading to OpenClaw 2026.3.2.

## Best Possible Solution

Land or reject https://github.com/openclaw/openclaw/pull/83991 for the sanitizer guard with route proof, then split any separately validated gaxios/native-fetch failure into its own dependency/auth item.

## Reproduction Assessment

Yes, at source level: current main routes Google stream-family payloads through a guard that only admits `google-generative-ai`, while current Google model tests show `google-gemini-cli` API ids on the affected Vertex provider path. I did not run a live Vertex credential flow in this read-only review.

## Solution Assessment

Yes for the sanitizer gap: widening the existing Google-family predicate with focused regression coverage is the narrow maintainable fix. It should not be treated as a complete closure for any separately validated gaxios/native-fetch auth-stack failure.
