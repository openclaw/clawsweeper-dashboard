## Summary

Keep open: current main and v2026.5.12 still have the Google thinking wrapper guard limited to `google-generative-ai`, while the reported `google-vertex` and `google-gemini-cli` paths share the wrapper and remain source-reproducible; the focused closing PR was closed unmerged.

## What This Changes

The issue reports that `google-vertex/gemini-3.1-pro-preview` embedded-agent messages crash with `Cannot convert undefined or null to object` after upgrading to 2026.3.2.

## Best Possible Solution

Land a narrow Google API guard widening with wrapper regression tests, then handle the gaxios/native-fetch failure as a separately approved dependency fix if it still reproduces.

## Reproduction Assessment

Yes. Source inspection shows `google-vertex` and `google-gemini-cli` models reach the shared Google wrapper but bypass sanitization because current main only matches `google-generative-ai`; several users also confirmed the Vertex crash on 2026.4.22 and 2026.4.23. I did not run a live Vertex credential repro.

## Solution Assessment

Yes for the wrapper gap. The narrow maintainable fix is to widen the existing Google-family API guard in `src/plugin-sdk/provider-stream-shared.ts` and add wrapper-level regression tests; the gaxios workaround should remain separate unless explicitly approved.
