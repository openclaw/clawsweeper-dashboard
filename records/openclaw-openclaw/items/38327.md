## Summary

Keep open: current main and v2026.5.2 still have the narrow Google thinking sanitizer guard, and #73416 is the open closing PR rather than a shipped fix.

## What This Changes

Google Vertex/Gemini CLI messages can crash after 2026.3.2 with `Cannot convert undefined or null to object` when Gemini thinking payloads are not sanitized.

## Best Possible Solution

Land a narrow guard widening with regression coverage for Google Vertex and Gemini CLI, then track any remaining gaxios native-fetch failure separately.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send a Gemini 3/3.1 payload with negative `thinkingBudget` through a `google-vertex` or `google-gemini-cli` model; current main does not run the sanitizer for those `model.api` values.

## Solution Assessment

Yes for the wrapper gap. #73416's narrow API guard widening plus regression tests is the maintainable fix; the gaxios/native-fetch fallback should stay separate unless maintainers approve dependency patching.
