## Summary

Keep open: current main and the latest checked release still only sanitize Google thinking payloads for `google-generative-ai`, while `google-vertex` and `google-gemini-cli` use the same Google wrapper family and PR #73416 is still the unmerged closing candidate.

## What This Changes

Fix the embedded-agent crash reported with `google-vertex/gemini-3.1-pro-preview` and related Vertex Gemini models after 2026.3.2.

## Best Possible Solution

Land a narrow Google wrapper guard fix with regression coverage for Vertex and Gemini CLI, then split any still-valid gaxios native-fetch fallback into its own dependency-review item.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: send a Gemini 3 payload with `thinkingConfig.thinkingBudget: -1` through a `google-vertex` or `google-gemini-cli` model, and current main skips the sanitizer because the guard only matches `google-generative-ai`.

## Solution Assessment

Yes for the wrapper root cause. PR #73416's guard widening plus targeted regression tests is the narrow maintainable fix; the gaxios fallback should remain a separate review if still needed.
