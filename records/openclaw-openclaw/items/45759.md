## Summary

Keep open. Current main still leaves the reported outage surface: the shared keepalive loop has no tick-level breaker, Telegram typing still goes through the normal retry runner, and the existing sendChatAction guard only suspends repeated 401s while transient failures remain covered by overlapping unmerged work.

## What This Changes

The issue asks OpenClaw to make Telegram typing/sendChatAction failures best-effort so network outages cannot stall or crash the gateway.

## Best Possible Solution

Land one canonical runtime fix that treats typing indicators as best-effort: bound repeated keepalive failures, make Telegram typing fail cheaply or enter transient cooldown, preserve 401 suspension and normal message delivery retries, and add focused regression coverage.

## Reproduction Assessment

Yes. The issue provides concrete network-outage steps and production log signatures, and current main still has source paths that can repeatedly invoke expensive Telegram typing/sendChatAction failures under outage conditions.

## Solution Assessment

Unclear whether the open transient-cooldown PR alone is sufficient. The maintainable solution should cover both the Telegram-specific retry/cooldown path and the shared keepalive lifecycle edge without changing reliable delivery semantics for real messages.
