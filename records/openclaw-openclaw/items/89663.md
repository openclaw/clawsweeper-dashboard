## What This Changes

The PR changes the Slack outbound adapter so adapter-routed sends no longer convert `OutboundIdentity` into per-message `username`, `icon_url`, or `icon_emoji` fields for `chat.postMessage`.

## Best Possible Solution

Support multi-bot bot-profile attribution without breaking the shipped active-agent overlay, likely by landing a documented Slack authorship policy or config path with tests and real Slack proof.

## Reproduction Assessment

Yes from source for the compatibility impact: current main maps outbound identity into Slack `username`/icon fields and the PR removes that adapter path. The reported multi-bot visual fix is plausible but still lacks inspectable live Slack proof in this PR.

## Solution Assessment

No as-is. Unconditionally dropping identity is too broad; the maintainable fix should preserve or explicitly migrate the documented active-agent identity behavior while giving multi-bot deployments a clear supported mode.
