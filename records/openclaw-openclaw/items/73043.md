## Summary

Keep open. Current main still sends `claude-cli` simple-completion requests into the pi-ai Anthropic HTTP path without a Claude CLI route or a clear unsupported-runtime guard, and no canonical merged or open fix was found.

## What This Changes

The issue asks OpenClaw to make Lossless/pi-ai simple completion with `claude-cli/claude-sonnet-4-6` either route through Claude CLI successfully or fail with a specific unsupported transport error.

## Best Possible Solution

Define the SDK simple-completion contract for CLI runtimes, then either add a Claude CLI one-shot path or reject CLI runtime models with a specific unsupported-runtime error and regression coverage.

## Reproduction Assessment

Yes, by source inspection rather than a live macOS run. The reported model still reaches pi-ai simple completion without a Claude CLI dispatch path or unsupported-runtime guard, while pi-ai’s Anthropic stream uses `model.baseUrl` as an HTTP SDK base URL.

## Solution Assessment

Unclear. Passing `cli://claude` through the generic pi-ai Anthropic path is not maintainable, but maintainers should choose whether this API should support CLI one-shot completions or fail fast with a clear unsupported-runtime error.
