## Summary

Keep open. Current main still has no simple-completion CLI dispatch or fail-fast guard for `claude-cli`/`cli://claude`; the OpenClaw simple-completion wrapper prepares the model and delegates to pi-ai, while Claude CLI is registered as a separate agent runtime/backend and synthetic auth surface.

## What This Changes

The issue asks OpenClaw to make Lossless/pi-ai simple completions with `claude-cli/claude-sonnet-4-6` either route through Claude CLI or fail with a clear unsupported-transport error.

## Best Possible Solution

Define the plugin SDK simple-completion contract for CLI runtimes, then either add a Claude CLI one-shot simple-completion adapter or reject CLI-runtime models before pi-ai with a typed unsupported-runtime error and focused regression tests.

## Reproduction Assessment

Yes, by source inspection rather than a live macOS run: the reported `claude-cli` model has no OpenClaw simple-completion CLI route or guard and falls through toward pi-ai's Anthropic path. A live harness would only confirm the exact external error string.

## Solution Assessment

Unclear. Letting `cli://claude` fall through to pi-ai is not the best solution, but maintainers need to choose between adding CLI simple-completion support and declaring CLI runtimes unsupported for this SDK surface.
