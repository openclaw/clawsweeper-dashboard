## Summary

Keep open. Current main still lacks a Claude CLI simple-completion dispatch or fail-fast guard, so `claude-cli`/`cli://claude` can still fall through to pi-ai's Anthropic HTTP-style provider instead of the registered CLI backend.

## What This Changes

The issue asks OpenClaw to make Lossless/pi-ai simple completions against `claude-cli/claude-sonnet-4-6` either work through Claude CLI or fail with a clear unsupported-runtime error.

## Best Possible Solution

Define the SDK simple-completion contract for CLI runtimes, then add focused coverage for either Claude CLI one-shot completion or a clear unsupported-runtime error that plugin callers can surface.

## Reproduction Assessment

Yes. The issue provides a concrete harness and passing controls, and current-main source inspection confirms there is still no Claude CLI dispatch or guard on the simple-completion path.

## Solution Assessment

Unclear. Letting `cli://claude` pass through pi-ai unchanged is not the best solution; maintainers should choose explicit CLI backend support or a fail-fast unsupported-runtime contract before implementation.
