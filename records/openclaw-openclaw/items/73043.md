## Summary

Keep open. Current main still does not route `claude-cli`/`cli://claude` simple completions through the Claude CLI backend, and it also lacks an early explicit unsupported-runtime error for that model shape.

## What This Changes

The issue asks OpenClaw to either support Lossless/pi-ai simple completions for `claude-cli/claude-sonnet-4-6` or fail with a clear unsupported-runtime error instead of a generic connection error.

## Best Possible Solution

Define the simple-completion contract for CLI runtime models in the OpenClaw plugin-SDK/runtime path: either add a first-class non-mutating Claude CLI one-shot helper, or detect CLI runtime/baseUrl inputs and return a typed unsupported-runtime error with docs guidance.

## Reproduction Assessment

Yes. The issue includes a concrete harness, and static current-main inspection shows the relevant simple-completion path still passes the model into pi-ai completion without a Claude CLI backend branch.

## Solution Assessment

Unclear. Passing `cli://claude` through pi-ai is not the maintainable solution; the maintainable direction is an explicit OpenClaw runtime contract or a deliberate fail-closed guard.
