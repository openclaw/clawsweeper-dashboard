## Summary

Keep open. Current main still lets `claude-cli` / `cli://claude` pass through the simple-completion path into pi-ai's Anthropic HTTP provider instead of dispatching to the CLI backend or failing fast with a clear unsupported-runtime error.

## What This Changes

The issue asks OpenClaw's plugin SDK simple-completion path to either support the synthetic `claude-cli` Sonnet transport or return a clear transport/provider error.

## Best Possible Solution

Define and enforce the SDK simple-completion contract for CLI runtimes, with either a Claude CLI one-shot path or a typed unsupported-runtime error covered by focused regression tests.

## Reproduction Assessment

Yes, by source inspection rather than a local live run. The provided harness targets `completeSimple` with `claude-cli/claude-sonnet-4-6`, and current main still has no simple-completion CLI dispatch or guard before pi-ai handles the `anthropic-messages` model.

## Solution Assessment

Unclear. Letting `cli://claude` fall through to pi-ai's Anthropic HTTP provider is not the best solution; maintainers should choose explicit CLI backend support or a fail-fast unsupported-runtime contract before implementation.
