## Summary

Keep open. Current main still routes simple-completion calls through pi-ai without a Claude CLI backend dispatch or explicit unsupported-runtime guard, so the reported `claude-cli` / `cli://claude` failure remains actionable.

## What This Changes

Allow Lossless/pi-ai simple completions using `claude-cli/claude-sonnet-4-6` to either use OpenClaw's Claude CLI transport or fail with a specific unsupported-transport error.

## Best Possible Solution

Define and document the SDK simple-completion contract for CLI runtime models, with focused tests covering either first-class Claude CLI one-shot completion or a clear unsupported-runtime error.

## Reproduction Assessment

Yes. The issue provides a concrete harness and controls, and current-main source inspection shows the same missing CLI backend dispatch or guard; this host lacks a `claude` binary for a live macOS-style run.

## Solution Assessment

Unclear. Passing `cli://claude` through pi-ai unchanged is not the best path; maintainers should choose explicit CLI backend support or a fail-fast unsupported-runtime contract before implementation.
