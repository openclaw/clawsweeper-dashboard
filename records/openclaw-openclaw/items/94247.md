- **[P1] Carry the separator through delivered block replies:** `src/agents/embedded-agent-block-chunker.ts:188-190`
  - body: The new boundary is stored as trailing whitespace on the chunk, but the subscriber caller still passes chunk text through `trimEnd()` before `onBlockReply`. A flushed heading/body split can therefore still deliver `# Title` and `First paragraph.` without the blank line, so preserve this boundary through the delivery path or explicit metadata and cover that subscription path.
  - confidence: 0.91
- **[P2] Keep separator-inclusive chunks within maxChars:** `src/agents/embedded-agent-block-chunker.ts:188-190`
  - body: The fit check still accepts `paragraphBreak.index - start <= paragraphLimit`, but this patch emits through `paragraphBreak.index + paragraphBreak.length`. When the separator starts exactly at the limit, this can emit `maxChars + 2` characters and bypass the provider/channel text-limit bound; include the separator in the fit check or fall back without losing the boundary.
  - confidence: 0.92
