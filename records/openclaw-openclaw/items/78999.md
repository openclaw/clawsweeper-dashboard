## Summary

Keep open: this is a recent, concrete regression in a documented WeChat login path, and current main does not contain a proven fix or a canonical open replacement. The code path is partly external-plugin-owned, but OpenClaw still pins and invokes that plugin, so closing as external-only would be premature without isolating the failing fetch to Tencent's package or the host network runtime.

## What This Changes

The issue reports that the WeChat plugin fails QR login fetches on OpenClaw 2026.5.6 with Node.js 25.x after upgrading from v2026.4.15.

## Best Possible Solution

Reproduce the Weixin QR-login failure on macOS with current main or v2026.5.6, Node 25, and the pinned plugin; then fix it either in the external plugin package or in a generic OpenClaw host-network/plugin SDK seam, updating the catalog/docs/changelog if the pinned package changes.

No, we do not have a high-confidence current-main reproduction path from this review. The report is detailed, but reproducing requires a live WeChat/Tencent iLink login setup with the external plugin and Node 25.
