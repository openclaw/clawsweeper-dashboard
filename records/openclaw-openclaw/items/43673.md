## Summary

Keep open: current main and v2026.6.10 have useful multi-agent primitives, `openclaw agents add`, Gateway agent/workspace RPCs, per-agent workspaces, sender-scoped tool policy, and multi-user session-scope guidance, but they still do not provide the requested team scaffold command, declarative deployment manifest, or first-class role/tenant-backed RBAC model. This should remain a maintainer-owned product/security umbrella rather than a cleanup close.

## What This Changes

The issue requests first-class org/team deployment support: workspace scaffolding, per-session or per-role tool policy/RBAC, deployment manifests, and org deployment documentation.

## Best Possible Solution

Keep the current per-agent workspace and sender-policy primitives, then split the umbrella into maintainer-approved designs for tenant/session RBAC and a separate decision on whether team scaffolds/manifests belong in core, docs, plugins, or external deployment tooling.
