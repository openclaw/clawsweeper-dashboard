## Summary

Keep open. Current main has adjacent multi-agent routing, per-agent tool/sandbox policy, group sender policy plumbing, and delegate/org guidance, but it still lacks the requested first-class team workspace scaffolding, declarative deployment manifest, and session/operator-scoped RBAC; #49183 and #53530 are useful narrower building blocks, not a complete replacement.

## What This Changes

Request first-class org/team deployment support through workspace scaffolding, per-session tool/RBAC policies, declarative deployment manifests, and official org deployment documentation.

## Best Possible Solution

Keep this as a maintainer-scoped umbrella or convert it into an RFC, then split accepted work into narrower issues for RBAC semantics, scaffold CLI, manifest schema, and org deployment docs.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection of the CLI, config types, tool-policy resolution, group policy helpers, and security docs, which shows adjacent support but not the requested first-class org/team model.

## Solution Assessment

Unclear that one implementation issue is the best way to solve this. A safer path is maintainer product/security design first, then smaller PRs that align Gateway operator scopes, channel sender identity, agent tool policy, CLI scaffolding, and manifest config.
