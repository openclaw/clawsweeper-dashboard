## Summary

Keep open. Current main has meaningful adjacent support for isolated agents, per-agent tool/sandbox policy, group `toolsBySender`, and delegate-architecture docs, but it still does not provide the requested first-class team scaffold command, deployment manifest, or session/operator-scoped RBAC model. The related RFC and open PR are useful smaller slices, not a complete replacement for this umbrella request.

## What This Changes

The issue requests first-class org/team deployment support through workspace scaffolding, per-session RBAC-style tool policies, declarative deployment manifests, and official org deployment guidance.

## Best Possible Solution

Keep this as a maintainer-scoped umbrella or convert it into an RFC, then split accepted work into narrower issues for RBAC semantics, team/workspace scaffolding, manifest schema, and org deployment docs. The current group/sender policy plumbing and the open direct-session policy PR can be evaluated as incremental building blocks.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source and docs inspection of CLI commands, config types, tool-policy resolution, group policy helpers, Gateway scope/security docs, and related discussion, which shows adjacent support but not the requested first-class org/team deployment model.

## Solution Assessment

Unclear that one implementation issue is the best way to solve this. The maintainable path is explicit maintainer product/security design followed by smaller PRs, especially for any RBAC subset that must align Gateway operator scopes, channel sender identity, and agent tool-policy semantics.
