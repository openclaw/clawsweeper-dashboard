## Summary

Keep open. Current main has manual multi-agent setup, Feishu-specific dynamic agent creation, and merged per-sender tool policy primitives, but it still does not provide the requested first-class org/team surface: a team workspace CLI, declarative deployment manifest, or reviewed per-session/scope RBAC contract.

## What This Changes

Request first-class org/team deployment support with workspace scaffolding, per-session RBAC/tool policies, declarative manifests, and official deployment guidance.

## Best Possible Solution

Split the umbrella into reviewed tracks: keep scope/session tool RBAC under https://github.com/openclaw/openclaw/issues/49183, lane/runtime isolation under https://github.com/openclaw/openclaw/issues/43235 and https://github.com/openclaw/openclaw/issues/42026, and make a separate product decision on whether team workspace templates and manifests belong in core or plugin distribution.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a broad feature and security/config request. Source and docs inspection give high confidence that current main lacks a first-class team workspace command, deployment manifest, and scope/session-keyed RBAC contract.

## Solution Assessment

No as one automated change. The maintainable path is to split the request into product/security-reviewed slices and reuse the existing multi-agent, sender-policy, Gateway-scope, plugin, and channel-specific dynamic-agent surfaces where they fit.
