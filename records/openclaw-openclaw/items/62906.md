The issue requests preventing repeated identical or near-identical assistant responses during a single OpenClaw interaction, reported on macOS WebChat with an Omlx/Qwen model.

## Best Possible Solution

Capture a current-main reproduction with raw WebChat transcript, Gateway chat events, agent events, provider/model config, and the exact tool/approval sequence, then fix the confirmed layer and coordinate with #63411/#39961 only if the root cause is pure streamed text repetition.

## Solution Assessment
