## Summary

Current main and the latest release already keep `before_prompt_build` `prependContext` out of the system prompt, with source and regression tests covering both the embedded runner and CLI prompt paths.

## What This Changes

The issue reports that `before_prompt_build` hook `prependContext` output moved from the user prompt in 2026.4.12 to the system prompt in 2026.5.19.

## Best Possible Solution

Keep the current v2026.5.22/current-main behavior and ask reporters on 2026.5.19 to update; reopen only with evidence that a current release still places `prependContext` in the system prompt.

## Reproduction Assessment

No for current main; source and regression tests show `prependContext` is not routed into the system prompt, and the latest release contains the same proof path.

## Solution Assessment

Yes; the current implementation keeps dynamic `prependContext` on the run prompt/context path while reserving `prependSystemContext` and `appendSystemContext` for system-prompt text.
