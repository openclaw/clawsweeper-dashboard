## Best Possible Solution

Keep the narrow runSessionKey alias-write approach, adjust the test to respect pre-transcript non-resumable persistence, and merge only after real behavior proof or maintainer validation.

## Solution Assessment

Is this the best way to solve the issue? The alias-write implementation is the narrowest maintainable fix, but this patch is not ready because the new test conflicts with existing pre-transcript behavior and real behavior proof is absent.
