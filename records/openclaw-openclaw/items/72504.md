## Summary

Keep open. Current `main` and the latest tagged release still strip the receiving Feishu bot's own mention before building agent-facing content, and open PR https://github.com/openclaw/openclaw/pull/72661 explicitly closes this issue while https://github.com/openclaw/openclaw/pull/72532 remains an alternate active fix path.

## What This Changes

Feishu group messages that mention multiple bots should preserve the receiving bot's own mention in agent-facing content so each bot can respond instead of returning `NO_REPLY`.

## Best Possible Solution

Land one focused Feishu plugin fix, preferably through the canonical open fix path, that preserves semantic inline self-mentions while keeping leading mention-prefixed slash commands command-clean with regression coverage and real Feishu proof.

## Reproduction Assessment

Yes, at source level. A Feishu group text event containing Bot A and Bot B mention placeholders, parsed with Bot A's `open_id`, will have Bot A's placeholder removed from `ctx.content` before the agent body is built.

## Solution Assessment

No for the issue's blanket suggestion to remove `botStripId` entirely. The narrower maintainable fix is to preserve inline self-mention context while keeping leading addressing prefixes and slash commands clean, or to separate command-probe text from agent-facing content.
