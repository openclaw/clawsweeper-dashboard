## Summary

Keep open: current main and latest release still strip the receiving Feishu bot's own mention, and the open linked PRs are candidate fixes rather than merged implementations, so this issue remains the canonical tracker.

## What This Changes

The issue asks Feishu to preserve the receiving bot's own <at> mention in multi-bot group messages so explicitly mentioned bots do not return NO_REPLY.

## Best Possible Solution

Land one canonical Feishu fix that preserves model-visible self-mentions while keeping command-facing text slash-clean, with focused tests and redacted Feishu proof, then close this issue.

## Reproduction Assessment

Yes at source level: a Feishu group event with the receiving bot plus another bot in mentions flows through parseFeishuMessageEvent into normalizeMentions with botOpenId, which removes the receiving bot's own tag before the agent-visible content is built. I did not run a live Feishu tenant reproduction.

## Solution Assessment

No landed solution exists yet. The direct remove-all stripping request is not the complete best fix because current tests protect mention-prefixed slash commands; the maintainable fix should split model-visible self-mention context from slash-clean command text.
