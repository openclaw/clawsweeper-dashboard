## Summary

Keep open. Current main still strips the receiving Feishu bot's own mention from the agent-facing content, and open PR #72661 explicitly closes this issue while #72532 remains an alternate active fix path.

## What This Changes

Preserve Feishu self-mention context in multi-bot group messages so explicitly mentioned bots do not return NO_REPLY.

## Best Possible Solution

Land one focused Feishu fix, preferably through #72661 or a replacement, that preserves semantic self-mention context while keeping mention-prefixed slash commands covered by regression tests and a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a group text event containing Bot A and Bot B mention placeholders parsed with Bot A's open_id; current normalizeMentions removes Bot A's placeholder before BodyForAgent is built.

## Solution Assessment

Unclear for the issue's suggested blanket removal of botStripId. The narrower maintainable path is to strip only the leading addressing prefix, or otherwise separate command probing from agent-facing mention context.
