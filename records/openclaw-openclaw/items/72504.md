## Summary

Keep open. Current main still removes the receiving Feishu bot's own mention from agent-facing content, and open PR #72661 explicitly closes this issue while #72532 remains an alternate active fix path.

## What This Changes

The issue reports that Feishu inbound message normalization removes the receiving bot's own mention, causing multi-bot group prompts to lose self-addressing context and return NO_REPLY.

## Best Possible Solution

Land one focused Feishu fix, preferably through #72661 or a replacement, that preserves semantic self-mention context while keeping mention-prefixed slash commands covered by regression tests and a changelog entry.

## Reproduction Assessment

Yes, at source level. A Feishu group text event containing Bot A and Bot B mention placeholders, parsed with Bot A's open_id, will have Bot A's placeholder removed before the agent body is built.

## Solution Assessment

Unclear for the issue's suggested blanket removal of botStripId. The narrower maintainable path is to strip only the leading addressing prefix, or otherwise separate command probing from agent-facing mention context.
