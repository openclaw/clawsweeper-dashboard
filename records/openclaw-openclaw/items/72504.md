## Summary

Keep open. Current main still removes the receiving Feishu bot's own mention before constructing the model-facing prompt, and the issue is paired with open implementation work, including an open PR with closing syntax, so it should remain open until that fix or a replacement lands.

## What This Changes

The issue requests that Feishu group mention normalization preserve the receiving bot's own `<at>` context in multi-bot group mentions so each addressed bot can identify itself and reply.

## Best Possible Solution

Land one focused Feishu patch that decouples command probing from model-facing content: preserve the self-mention context needed by the agent in multi-bot group prompts, keep mention-prefixed slash commands working, and add regression tests for both the multi-bot prompt and existing #35994 command cases.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Feishu group text event containing Bot A and Bot B mention placeholders while parsing with Bot A's `open_id`; current `normalizeMentions` removes Bot A's placeholder before `buildFeishuAgentBody` constructs `BodyForAgent`.

## Solution Assessment

Unclear for the issue's broad suggested removal of all `botStripId` behavior. The safer solution is a narrow Feishu change that preserves model-facing self-mention tags while retaining the command-probe normalization that protects mention-prefixed slash commands.
