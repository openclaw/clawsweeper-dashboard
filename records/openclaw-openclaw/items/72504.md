## Summary

Keep open. Current main still strips the receiving Feishu bot's own mention before building the agent-facing body, and open PR #72661 explicitly closes this issue with #72532 as an alternate active implementation, so this should wait for one focused fix to land.

## What This Changes

The issue asks Feishu to preserve the receiving bot's own mention in multi-bot group messages so explicitly mentioned bots do not return NO_REPLY.

## Best Possible Solution

Land one focused Feishu fix, preferably through #72661 or a replacement, that preserves inline self-mention context while keeping mention-prefixed slash commands covered by tests and changelog.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Feishu group text event containing Bot A and Bot B mention placeholders while parsing with Bot A's `open_id`; current `normalizeMentions` removes Bot A's placeholder before `BodyForAgent` is constructed.

## Solution Assessment

Unclear for the issue's suggested removal of all `botStripId` behavior. The safer direction is to preserve semantic self-mentions in agent-facing content while stripping only the leading addressing prefix or keeping command probing independent so #35994 remains protected.
