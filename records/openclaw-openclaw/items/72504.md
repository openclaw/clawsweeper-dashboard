## Summary

Keep open. Current main still removes the receiving Feishu bot’s own mention before building `BodyForAgent`, and open PR #72661 explicitly closes this issue while #72532 remains alternate active work, so cleanup should wait for one focused fix to land.

## What This Changes

Preserve a receiving Feishu bot’s own mention in multi-bot group prompts so each addressed bot can respond instead of returning `NO_REPLY`.

## Best Possible Solution

Land one focused Feishu fix that preserves model-facing self-mention context except where only a leading addressing prefix must be removed for slash-command compatibility, with regression tests and a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Feishu group text event with Bot A and Bot B mention placeholders parsed with Bot A’s `open_id`; current `normalizeMentions` removes Bot A’s placeholder before `BodyForAgent` is built.

## Solution Assessment

Unclear for the issue’s suggested removal of all `botStripId` behavior. The safer maintainable direction is the narrower design from #72661: preserve inline/semantic self mentions while stripping only the leading addressing prefix needed for group slash commands.
