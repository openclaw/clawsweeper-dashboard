## Summary

Keep open. Current main still strips the receiving Feishu bot's own mention from agent-facing content, and open PR #72661 explicitly closes this issue while #72532 remains an alternate active fix path.

## What This Changes

Feishu group messages that mention multiple bots should preserve the receiving bot's own mention in agent-facing content so each bot can respond.

## Best Possible Solution

Land one focused Feishu fix, preferably #72661 or a replacement, that strips only leading addressing prefixes while preserving inline self-mention context, with regression coverage and a changelog entry.

## Reproduction Assessment

Yes, at source level. In current main, a Feishu text event with mentions for Bot A and Bot B is parsed with Bot A's open_id, so normalizeMentions removes Bot A's placeholder before ctx.content reaches buildFeishuAgentBody.

## Solution Assessment

Unclear for the issue's suggested blanket removal. The narrower maintainable fix is to preserve inline semantic self-mentions while still stripping only the leading addressing prefix needed for group slash-command probing.
