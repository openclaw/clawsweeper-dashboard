## Summary

Keep open. Current main still strips the receiving Feishu bot's own mention before building the agent-facing prompt, and open PR #72661 explicitly tracks a focused fix while #72532 remains an alternate implementation candidate.

## What This Changes

The issue requests preserving the receiving Feishu bot's own `<at>` mention in inbound group content so multi-bot @mentions do not lead every bot to return `NO_REPLY`.

## Best Possible Solution

Land one focused Feishu patch that preserves self-mention context in the model-facing body for multi-bot group prompts, keeps mention-prefixed slash commands working through command-probe normalization, and adds regression coverage for both paths.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Feishu group text event containing Bot A and Bot B mention placeholders parsed with Bot A's `open_id`; current `normalizeMentions` removes Bot A's placeholder before `BodyForAgent` is constructed.

## Solution Assessment

Unclear for the issue's suggested removal of all `botStripId` behavior. The safer solution is a narrow Feishu change that preserves model-facing self-mention tags while retaining the command-probe normalization that protects #35994 slash-command cases.
