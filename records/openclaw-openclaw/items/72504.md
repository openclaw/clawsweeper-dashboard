- `impact:message-loss`: The reported failure suppresses expected Feishu bot replies and turns explicit mentions into silent NO_REPLY outcomes.

## Summary

Keep open. Current main and v2026.5.18 still strip the receiving Feishu bot’s own mention from agent-facing content, and open PR https://github.com/openclaw/openclaw/pull/72661 explicitly closes this issue while https://github.com/openclaw/openclaw/pull/72532 overlaps the same root cause.

## What This Changes

Feishu group chats should preserve enough bot self-mention context that multiple explicitly mentioned bots do not silently return NO_REPLY.

## Best Possible Solution

Land one focused Feishu plugin fix that preserves semantic inline self-mentions while keeping leading mention-prefixed slash commands command-clean, with regression coverage and redacted live Feishu proof.

## Reproduction Assessment

Yes, at source level. A Feishu group text event with Bot A and Bot B mention placeholders, parsed with Bot A's `open_id`, will remove Bot A's placeholder from `ctx.content` before the agent body is built.

## Solution Assessment

No for the issue's blanket suggestion to remove `botStripId` entirely. The narrower maintainable fix is to preserve inline self-mention context while keeping leading addressing prefixes and slash commands clean.
