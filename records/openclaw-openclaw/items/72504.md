## Summary

Keep open. Current main still strips the receiving Feishu bot's own mention from agent-facing content, and open PR #72661 explicitly closes this issue while #72532 remains an alternate active fix path.

## What This Changes

The issue reports that Feishu group messages mentioning multiple bots can lead each bot to return NO_REPLY because its own `<at>` mention is removed before the agent sees the prompt.

## Best Possible Solution

Review and land #72661 or an equivalent Feishu-owned fix that preserves semantic self-mention context while keeping group slash-command probing covered by regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is a group text event with Bot A and Bot B mention placeholders parsed with Bot A's `open_id`; current normalization removes Bot A's placeholder before `BodyForAgent` is built.

## Solution Assessment

Unclear for the issue's blanket removal suggestion. The safest path is a focused Feishu fix that keeps #35994 command behavior covered while ensuring inline or multi-bot self-mentions remain visible to the agent.
