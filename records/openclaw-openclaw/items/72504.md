## Summary

Keep open. Current main still strips the receiving Feishu bot's own mention before building the agent-facing body, and open PR #72661 explicitly closes this issue while #72532 remains alternate active work.

## What This Changes

Preserve the receiving Feishu bot's own `<at>` mention in multi-bot Feishu group messages so agent routing does not return `NO_REPLY`.

## Best Possible Solution

Land one focused Feishu fix, likely through #72661 or a replacement, that strips only addressing-prefix self mentions for command compatibility while preserving inline self-mention tags in agent content.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Feishu group text event containing Bot A and Bot B mention placeholders parsed with Bot A's `open_id`; current `normalizeMentions` removes Bot A's placeholder before `BodyForAgent` is constructed.

## Solution Assessment

Unclear for the issue's suggested removal of all `botStripId` behavior. The narrower leading-prefix approach is safer because it preserves the #35994 slash-command behavior while keeping semantic self-mentions visible to the model.
