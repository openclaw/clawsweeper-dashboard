## Summary

Keep open. Current main still strips the receiving Feishu bot's own mention before constructing the agent-facing body, and open PR #72661 explicitly closes this issue while #72532 remains an alternate active implementation, so this should wait for one focused fix to land.

## What This Changes

Preserve Feishu self-mention context in multi-bot group prompts so explicitly mentioned bots do not return NO_REPLY.

## Best Possible Solution

Land one focused Feishu fix, preferably through #72661 or a replacement, that preserves model-facing self-mention context where it matters while keeping mention-prefixed slash commands covered by regression tests and a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Feishu group text event with Bot A and Bot B mention placeholders while parsing with Bot A's open_id; current normalizeMentions removes Bot A's placeholder before BodyForAgent is constructed.

## Solution Assessment

Unclear for the issue's suggested removal of all botStripId behavior. The safer direction is to strip only the leading addressing prefix or otherwise keep command probing independent while preserving semantic self-mentions in agent-facing content.
