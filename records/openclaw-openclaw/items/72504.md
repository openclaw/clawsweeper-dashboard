## Summary

Keep open. Current main still strips the receiving Feishu bot's own mention before building the agent-facing body, and open PR #72661 directly targets this issue with #72532 as an alternate active implementation.

## What This Changes

The issue requests preserving Feishu self-mention context in multi-bot group messages so each mentioned bot can see it was addressed instead of returning NO_REPLY.

## Best Possible Solution

Land one focused Feishu fix that preserves inline/self mention tags in agent-facing content while keeping leading mention-prefixed slash commands covered by regression tests and a changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is a Feishu group text event containing Bot A and Bot B mention placeholders parsed with Bot A's `open_id`; current `normalizeMentions` removes Bot A's placeholder before `BodyForAgent` is built.

## Solution Assessment

Unclear for the issue's suggested blanket removal. The narrower leading-prefix-only direction appears safer because command probing already has a separate normalization path while existing tests preserve #35994 behavior.
