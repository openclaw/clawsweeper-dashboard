## Summary

Keep open. Current main still removes the receiving Feishu bot's own mention from agent-facing text, and an open closing PR already targets the issue, so this should wait for that implementation path instead of being swept closed.

## What This Changes

Preserve Feishu self-mention context in multi-bot group messages so each explicitly mentioned bot can respond instead of producing NO_REPLY.

## Best Possible Solution

Land one focused Feishu fix, likely through #72661 or an equivalent replacement, that preserves inline self-mention context while keeping leading mention-prefixed slash commands covered by regression tests and a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a group text event with Bot A and Bot B mention placeholders parsed with Bot A's open_id; current normalizeMentions removes Bot A's placeholder before BodyForAgent is built.

## Solution Assessment

Unclear for the issue's blanket removal proposal. The narrower maintainable fix is to strip only the leading addressing prefix or otherwise separate command probing from agent-facing mention context.
