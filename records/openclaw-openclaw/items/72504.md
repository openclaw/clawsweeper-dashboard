## Summary

Keep open. Current main still has the reported Feishu mention-normalization behavior, and an open PR with explicit closing syntax is already the focused implementation path for this issue.

## What This Changes

The issue requests preserving a Feishu bot's own non-addressing <at> mention in agent-facing content so multi-bot group mentions do not route to NO_REPLY.

## Best Possible Solution

Land one focused Feishu fix, preferably through #72661 or an equivalent replacement, that preserves inline self-mention context while keeping leading-address slash commands covered by regression tests and a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a group text event containing Bot A and Bot B mention placeholders parsed with Bot A's open_id; current normalizeMentions removes Bot A's placeholder before BodyForAgent is built.

## Solution Assessment

Unclear for the issue's suggested blanket removal of botStripId. The narrower maintainable fix is to strip only the leading addressing prefix, or otherwise keep command probing separate from agent-facing mention context.
