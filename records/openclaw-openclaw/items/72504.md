## Summary

Keep open. Current main still removes the receiving Feishu bot's own mention from agent-facing content, and an open closing PR plus an alternate open PR are already handling the implementation path.

## What This Changes

The issue requests a Feishu fix so multi-bot group mentions preserve each receiving bot's own `<at>` context instead of causing `NO_REPLY`.

## Best Possible Solution

Land one focused Feishu plugin fix, preferably through the canonical open fix path, that preserves semantic inline self-mentions while keeping leading addressing prefixes and slash commands covered by regression tests.

## Reproduction Assessment

Yes, at source level. A Feishu group text event with Bot A and Bot B mention placeholders, parsed with Bot A's `open_id`, will have Bot A's placeholder removed from `ctx.content` before the agent body is built.

## Solution Assessment

No for the issue's blanket suggested removal as-is. The narrower maintainable direction is to preserve inline self-mention context while keeping leading addressing prefixes slash-clean, or to separate command probing from agent-facing content.
