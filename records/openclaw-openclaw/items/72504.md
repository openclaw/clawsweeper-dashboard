## Summary

Keep open. Current main still strips the receiving Feishu bot's own mention from agent-facing content, and open PR #72661 explicitly closes this issue while #72532 remains an alternate open fix path.

## What This Changes

The issue asks Feishu inbound parsing to preserve the receiving bot's own `<at>` mention in multi-bot group messages so explicitly mentioned bots do not go silent.

## Best Possible Solution

Land one focused Feishu fix, preferably through #72661 or a replacement, that preserves semantic self-mention context while keeping leading-address slash commands covered by regression tests and a changelog entry.

## Reproduction Assessment

Yes. On current main, a Feishu group event with Bot A and Bot B in `event.message.mentions` parsed with Bot A's `open_id` sets `mentionedBot=true` but removes Bot A's placeholder from `ctx.content` before the agent body is built.

## Solution Assessment

Unclear for the issue's suggested blanket removal. The safer maintainable path is to strip only a leading addressing prefix, or otherwise separate command probing from agent-facing mention context so #35994 slash commands keep working.
