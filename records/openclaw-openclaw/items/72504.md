## Summary

Keep open. Current main still removes the receiving Feishu bot's own mention from the agent-facing message body, and open PR #72661 explicitly closes this issue while #72532 remains an alternate active fix, so the issue should wait for one implementation to land.

## What This Changes

The issue asks Feishu to preserve the receiving bot's own @ mention in multi-bot group messages so each addressed bot has enough agent-facing context to respond instead of returning NO_REPLY.

## Best Possible Solution

Land one focused Feishu fix, preferably through #72661 or a replacement, that preserves semantic self-mentions in BodyForAgent while keeping mention-prefixed slash commands covered by regression tests and a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Feishu group text event containing Bot A and Bot B mention placeholders parsed with Bot A's open_id; current normalizeMentions removes Bot A's placeholder before BodyForAgent is constructed.

## Solution Assessment

Unclear for the issue's suggested removal of all botStripId behavior. The safer direction is to strip only the leading addressing prefix or otherwise keep command probing independent while preserving semantic self-mentions in agent-facing content.
