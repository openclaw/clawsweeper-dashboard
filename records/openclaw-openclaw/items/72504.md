## Summary

Keep open. Current main still strips the receiving Feishu bot's own mention from agent-facing content, and open PR #72661 explicitly closes this issue while #72532 remains an active alternate fix path.

## What This Changes

Feishu group messages that @mention multiple bots should preserve each receiving bot's own mention so the agent does not answer with NO_REPLY.

## Best Possible Solution

Land one focused Feishu fix, likely #72661 or a replacement, that keeps slash-command prefix handling while preserving inline and multi-bot self-mentions with regression coverage and a changelog entry.

## Reproduction Assessment

Yes, at source level. A text event with mentions for Bot A and Bot B parsed with Bot A's `open_id` enters `normalizeMentions(..., botStripId=BotA)`, which removes Bot A's placeholder before `buildFeishuAgentBody` dispatches `ctx.content`.

## Solution Assessment

Unclear for the issue's suggested blanket removal. The safer maintainable path is to preserve semantic self-mention context while stripping only a leading addressing prefix or using the command-probe path for slash commands.
