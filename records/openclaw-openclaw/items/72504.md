## Summary

Keep open. Current main still removes the receiving Feishu bot's own mention from agent-facing content, and open PR #72661 explicitly closes this issue while #72532 remains an alternate active fix path.

## What This Changes

The issue asks Feishu mention normalization to preserve the receiving bot's own `<at>` tag so multi-bot group mentions do not result in `NO_REPLY`.

## Best Possible Solution

Preserve semantic self-mention context in Feishu agent content while keeping leading mention slash-command normalization covered by regression tests and a changelog entry.

## Reproduction Assessment

Yes, at source level. A Feishu group text event containing Bot A and Bot B mention placeholders, parsed with Bot A's `open_id`, will have Bot A's placeholder removed before the agent body is built.

## Solution Assessment

No for the issue's suggested blanket removal. The safer maintainable fix is to strip only the leading addressing prefix, or otherwise separate command probing from agent-facing mention context.
