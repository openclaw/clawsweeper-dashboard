## Summary

Keep this PR open, but it is not merge-ready: current main still lacks the requested passthrough behavior, while the PR has source-reproducible config-contract defects and missing real behavior proof.

## What This Changes

The PR changes `agents.defaults.models` entries to accept arbitrary passthrough fields, merges those fields into provider model definitions during model defaulting, widens the config type, and adds focused tests.

## Best Possible Solution

Land a rebased, maintainer-approved config-contract change that applies passthrough after normalized model refs, excludes known agent-model fields, updates docs/help, and includes focused tests plus redacted runtime proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the raw passthrough lookup happens before current-main model-default normalization, and the known-key filter omits current `agentRuntime`. I did not run tests because this review is read-only.

## Solution Assessment

No, not yet. The passthrough direction may be useful, but the patch should normalize lookup keys, keep known agent-model fields out of provider-model passthrough, and document the expanded config contract before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Normalize model-default keys before passthrough lookup:** `src/config/defaults.ts:265-266`
  - body: The new override lookup indexes `agents.defaults.models` with raw `${providerId}/${raw.id}` before the existing config normalization pass. Valid keys that normalize later, such as provider aliases like `bedrock/...` or retired model ids, will still resolve for other model-default behavior but silently skip the new `contextWindow`/`maxTokens` passthrough; normalize the defaults map or lookup key before extracting overrides.
  - confidence: 0.9
- **[P2] Keep agentRuntime out of provider-model passthrough:** `src/config/defaults.ts:48`
  - body: Current main has `agents.defaults.models.*.agentRuntime` as a known per-model runtime policy, but the new known-key set only excludes `alias`, `params`, and `streaming`. When this PR is rebased, `agentRuntime` can be merged into the global provider model definition and affect other agents using the same model, so exclude all known agent-model entry fields before forwarding passthrough metadata.
  - confidence: 0.86
- **[P3] Document the model-entry passthrough contract:** `src/config/zod-schema.agent-defaults.ts:35`
  - body: Changing the schema to `.passthrough()` expands the public config contract, but the PR does not update schema help or user docs to explain which model metadata belongs on `agents.defaults.models` versus `models.providers`. Add help/docs with the schema change so users do not have to infer the new behavior from source.
  - confidence: 0.82
