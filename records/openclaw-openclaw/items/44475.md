## Summary

Keep open: current main still rejects passthrough fields at `agents.defaults.models`, and this PR is useful but not merge-ready because its lookup misses normalized model refs and the public schema/docs/changelog are not refreshed.

## What This Changes

The PR relaxes `agents.defaults.models` entries to accept unknown fields, merges those fields into provider model definitions during `applyModelDefaults()`, updates the TypeScript type, and adds focused passthrough tests.

## Best Possible Solution

Land a revised config change that keeps the passthrough scope but normalizes model-ref lookup, regenerates schema/docs metadata, adds the required changelog entry, and preserves focused regression coverage.

## Reproduction Assessment

Yes. Source inspection of the PR diff shows the new passthrough merge uses an exact key lookup, while existing allowlist handling normalizes refs; a valid `bedrock/<model>` entry for `amazon-bedrock` would be accepted elsewhere but missed by the new code.

## Solution Assessment

No, not yet. The passthrough direction is plausible after the mutation-preservation fix, but the branch should reuse the normalized model-ref contract and update generated public config surfaces before merge.
