Land a rebased config-contract change that normalizes passthrough lookup keys, keeps agent model fields separate from provider metadata, updates docs/help, and includes focused tests plus redacted runtime proof.

Full review comments:

- **[P2] Normalize defaults keys before passthrough lookup:** `src/config/defaults.ts:265-266`
  - body: The new override lookup indexes `agents.defaults.models` with raw `${providerId}/${raw.id}` before the existing config-normalization pass. Valid keys that normalize elsewhere, such as provider aliases or retired Gemini ids, can still resolve for other model-default behavior but silently skip the new `contextWindow`/`maxTokens` passthrough, so normalize the defaults map or lookup key before extracting overrides.
  - confidence: 0.9
- **[P2] Exclude agentRuntime from provider-model passthrough:** `src/config/defaults.ts:49`
  - body: Current main has `agents.defaults.models.*.agentRuntime` as a known per-model runtime-policy field, but the new known-key set only excludes `alias`, `params`, and `streaming`. When rebased, forwarding `agentRuntime` through the provider-model metadata merge blurs the separate agent runtime-policy contract, so exclude all known agent-model entry fields before passing metadata through.
  - confidence: 0.84
- **[P3] Document model-entry passthrough before widening schema:** `src/config/zod-schema.agent-defaults.ts:35`
  - body: Changing this schema to `.passthrough()` expands a public config contract, but the PR does not update user docs or schema help to explain which model metadata belongs on `agents.defaults.models` versus `models.providers`. Add docs/help with the schema change so users do not have to infer the new behavior from source.
  - confidence: 0.82
