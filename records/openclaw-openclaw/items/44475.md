## Summary

Keep open: current main still lacks the requested passthrough config behavior, and the PR is useful but not merge-ready because the new lookup bypasses existing model-ref normalization and the public config/schema surfaces are incomplete.

## What This Changes

The PR changes `agents.defaults.models` entries to accept unknown fields, merges those fields into provider model definitions in `applyModelDefaults()`, widens the TypeScript type, and adds focused passthrough tests.

## Best Possible Solution

Land a rebased config change that preserves the passthrough scope, normalizes model-ref lookup, updates generated schema/docs metadata, adds a user-facing changelog entry, and keeps focused regression coverage.

## Reproduction Assessment

Yes. Source inspection of the PR diff shows the new passthrough merge uses an exact key lookup, while current model-ref helpers normalize supported provider aliases such as `bedrock` to `amazon-bedrock`.

## Solution Assessment

No, not yet. The passthrough approach is plausible after the mutation fix, but it should reuse the normalized model-ref contract and update the generated/public config surfaces before merge.
