## Summary

Keep this PR open. The requested `agents.defaults.models` passthrough behavior is still not implemented on checked-out `main`, and the PR is useful source work, but the patch is not ready because it bypasses the existing normalized model-ref contract and leaves generated config schema/docs stale.

## What This Changes

The PR relaxes `agents.defaults.models` entries to accept unknown fields, merges those fields into provider model definitions during `applyModelDefaults()`, updates the TypeScript entry type, and adds focused passthrough tests.

## Best Possible Solution

Land a revised config change that treats passthrough model metadata as a public config contract: normalize override lookup the same way allowlists and aliases are normalized, preserve deterministic provider model defaulting, regenerate schema/baseline/docs, add a changelog entry, and keep the scope limited to this config surface.

## Reproduction Assessment

Yes. On the PR branch, configure `models.providers.amazon-bedrock` with a model and set `agents.defaults.models["bedrock/<model>"].contextWindow`; current allowlist helpers normalize that key to `amazon-bedrock/<model>`, but the PR’s exact lookup drops the override.

## Solution Assessment

No, not yet. The passthrough direction is plausible after the mutation-preservation fix, but the branch should reuse the normalized model-ref contract and refresh generated schema/docs before merge.
