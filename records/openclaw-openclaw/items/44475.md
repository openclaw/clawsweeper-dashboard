This PR should stay open: current main still lacks the requested passthrough config behavior, but the branch is conflicting, missing real behavior proof, and still has a source-reproducible normalization bug before it can merge.

## What This Changes

The branch relaxes `agents.defaults.models` entries to passthrough arbitrary fields, merges those extras into provider model definitions during defaulting, widens the type, and adds focused tests.

## Best Possible Solution

Land a rebased config-contract change that normalizes model-default keys before applying passthrough overrides, preserves current model-entry semantics, updates docs/help, and includes focused tests plus redacted runtime proof.

Yes for the blocking PR defect: source inspection of the PR head shows an exact raw model-default lookup before current-main normalization, so normalized refs can skip passthrough overrides. I did not run tests because this review is read-only.

## Solution Assessment

No, not yet. The passthrough direction may be useful, but the implementation should reuse normalized model-default keys and move the public config docs/help with the schema change.
