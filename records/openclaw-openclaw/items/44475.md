The PR relaxes `agents.defaults.models` entries to passthrough unknown fields, merges non-known fields into provider model definitions in `applyModelDefaults()`, widens the config type, and adds model-default tests.

## Best Possible Solution

Land a rebased config change that accepts model metadata passthrough via normalized model refs, preserves current agent-runtime/default semantics, updates docs/baselines/changelog, and includes focused tests plus real behavior proof.

## Solution Assessment
