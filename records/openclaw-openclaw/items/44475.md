## Summary

Keep open: current main still does not implement agent-model passthrough fields, but this PR is not merge-ready because the diff misses current model-ref normalization, must be reconciled with `agentRuntime`, and lacks after-fix real behavior proof.

## What This Changes

The branch widens `agents.defaults.models` entries to `.passthrough()`, merges extra fields into provider model definitions in `applyModelDefaults()`, updates the exported type, and adds config-default tests.

## Best Possible Solution

Land a rebased, maintainer-approved config-contract change that normalizes passthrough lookup keys, preserves known agent-model fields, updates docs/help, and includes focused regression tests plus redacted runtime proof.

## Reproduction Assessment

Yes for the PR review defects by source inspection: the PR performs raw passthrough lookup before current-main model-default normalization, and its known-key filter omits current `agentRuntime`. I did not run tests because this is a read-only review.

## Solution Assessment

No, not yet. The passthrough direction may be useful, but the patch should normalize lookup keys, protect known agent-model fields, document the expanded public config contract, and show real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Normalize defaults keys before passthrough lookup:** `src/config/defaults.ts:265-266`
  - body: The new override lookup indexes `agents.defaults.models` with raw `${providerId}/${raw.id}` before the existing config normalization pass. Valid keys that normalize elsewhere, such as provider aliases like `bedrock/...` or retired Gemini ids, will still resolve for other model-default behavior but silently skip the new `contextWindow`/`maxTokens` passthrough; normalize the defaults map or lookup key before extracting overrides.
  - confidence: 0.9
- **[P2] Exclude agentRuntime from provider-model passthrough:** `src/config/defaults.ts:49`
  - body: Current main has `agents.defaults.models.*.agentRuntime` as a known per-model runtime policy, but the new known-key set only excludes `alias`, `params`, and `streaming`. When this PR is rebased, `agentRuntime` can be merged into the global provider model definition and affect other agents using the same model, so exclude all known agent-model entry fields before forwarding passthrough metadata.
  - confidence: 0.86
- **[P3] Document model-entry passthrough before widening schema:** `src/config/zod-schema.agent-defaults.ts:35`
  - body: Changing the schema to `.passthrough()` expands the public config contract, but the PR does not update schema help or user docs to explain which model metadata belongs on `agents.defaults.models` versus `models.providers`. Add help/docs with the schema change so users do not have to infer the new behavior from source.
  - confidence: 0.82
