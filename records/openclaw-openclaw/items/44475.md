## Summary

This PR still addresses a real missing config capability on current main, but it should not merge as-is because the passthrough lookup bypasses existing model-ref normalization and the generated/user-facing config surfaces are not aligned.

## What This Changes

The PR allows unknown fields in `agents.defaults.models` entries, forwards those fields into provider model definitions during `applyModelDefaults()`, updates the entry type, and adds focused config tests.

## Best Possible Solution

Land a revised config change that reuses existing model-ref normalization and keeps runtime behavior, generated schema, docs, changelog, and regression coverage aligned.

## Reproduction Assessment

Yes. Source inspection is enough for the PR blocker: the diff uses an exact `${providerId}/${raw.id}` lookup, while current allowlist code normalizes refs and provider aliases before matching.

## Solution Assessment

No, not yet. The passthrough direction is plausible after the mutation fix, but the merge should normalize lookup keys and update generated/user-facing config surfaces first.
