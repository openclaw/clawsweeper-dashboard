Full review comments:

- **[P1] Validate model-default metadata before merging:** `src/config/zod-schema.agent-defaults.ts:35`
  - body: `agents.defaults.models` is currently a strict, schema-backed config surface. Changing the entry to `.passthrough()` lets malformed metadata such as `contextWindow: "1000000"`, `maxTokens: "large"`, or unexpected provider/model fields pass validation; `applyModelDefaults()` then defaults or forwards them instead of producing a config error, so validation guarantees and upgrade diagnostics are lost. Please add explicit/schema-backed fields or reuse the provider model metadata schema before merging.
  - confidence: 0.88
- **[P2] Normalize model refs before override lookup:** `src/config/defaults.ts:265-266`
  - body: The override map is read with exact `${providerId}/${raw.id}` keys before current main's `normalizeAgentModelMapForConfig()` normalization. A valid config keyed as `bedrock/...` will normalize to `amazon-bedrock/...` for selection, but this lookup will miss it, so passthrough metadata applies inconsistently for provider aliases. Build a normalized defaults map or normalize both sides before merging.
  - confidence: 0.91
