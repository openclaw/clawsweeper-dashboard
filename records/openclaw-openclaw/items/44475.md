## Summary

Keep open: current main still rejects and does not apply `agents.defaults.models` metadata passthrough, while this PR remains a useful implementation candidate but needs focused fixes before merge.

## What This Changes

The PR changes `agents.defaults.models` entries to passthrough unknown fields, merges non-`alias`/`params`/`streaming` keys into provider model definitions during `applyModelDefaults()`, updates the TypeScript type, and adds focused tests.

## Best Possible Solution

Land a revised config change that preserves the passthrough scope, reuses normalized model-ref lookup, updates generated schema/docs metadata and user docs, adds a changelog entry, and keeps focused regression coverage.

## Reproduction Assessment

Yes. Source inspection is enough: the PR's passthrough merge uses an exact map key while current allowlist/model selection code normalizes refs such as `bedrock/...` to `amazon-bedrock/...`.

## Solution Assessment

No, not yet. The passthrough approach is plausible, especially after the mutation-preservation fix, but the branch should normalize lookup keys and update generated/user-facing config surfaces before merge.
