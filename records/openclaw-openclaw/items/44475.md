## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Normalize defaults keys before passthrough lookup:** `src/config/defaults.ts:265-266`
  - body: The new override lookup indexes `agents.defaults.models` with raw `${providerId}/${raw.id}` before current-main normalization. Valid refs that normalize elsewhere, such as `bedrock/...` to `amazon-bedrock/...` or retired Gemini ids, will keep working for existing model-default behavior but silently skip the new `contextWindow`/`maxTokens` passthrough; normalize the defaults map or lookup key before extracting overrides.
  - confidence: 0.9
- **[P2] Exclude agentRuntime from provider-model passthrough:** `src/config/defaults.ts:49`
  - body: Current main has `agents.defaults.models.*.agentRuntime` as a known per-model runtime policy, but the new known-key set excludes only `alias`, `params`, and `streaming`. When rebased, an `agentRuntime` entry can be copied into the global provider model definition and affect other agents using that provider model, so filter all known agent-model entry fields before forwarding provider metadata.
  - confidence: 0.87
- **[P2] Validate known model metadata before accepting passthrough:** `src/config/zod-schema.agent-defaults.ts:35`
  - body: Switching the entry schema directly to `.passthrough()` accepts bad known metadata such as `contextWindow: "large"` or negative `maxTokens`, even though provider model schema rejects those fields. `applyModelDefaults()` then treats non-positive or non-number values as missing and falls back to defaults, so a typo can be accepted and ignored instead of surfacing a config error; add explicit schema validation for known model metadata before forwarding unknown fields.
  - confidence: 0.82
- **[P3] Document model-entry passthrough before widening schema:** `src/config/zod-schema.agent-defaults.ts:35`
  - body: This change expands the public config contract, but the PR does not update config help or user docs to explain which model metadata belongs on `agents.defaults.models` versus `models.providers`. Add the help/docs with the schema change so users do not have to infer supported behavior from source.
  - confidence: 0.8
