Land a maintainer-approved config contract that validates known model metadata, normalizes model-default lookups, preserves scoped runtime policy, updates docs/tests, and includes redacted real setup proof.

I did not run tests because this review is read-only.

- **[P2] Normalize defaults keys before passthrough lookup:** `src/config/defaults.ts:265-266`
  - body: The new override lookup indexes `agents.defaults.models` with raw `${providerId}/${raw.id}` before current-main normalization.   - confidence: 0.9
- **[P2] Exclude agentRuntime from provider-model passthrough:** `src/config/defaults.ts:49`
  - body: Current main has `agents.defaults.models.*.agentRuntime` as scoped per-model runtime policy, but the new known-key set excludes only `alias`, `params`, and `streaming`. When rebased, an `agentRuntime` entry can be copied into the global provider model definition and affect other agents using that provider model.
  - confidence: 0.87
- **[P2] Validate known model metadata before passthrough:** `src/config/zod-schema.agent-defaults.ts:35`
  - body: Switching the entry schema directly to `.passthrough()` accepts bad known metadata such as `contextWindow: "large"` or negative `maxTokens`, even though provider model schema rejects those fields. Add explicit validation for supported provider-model metadata before forwarding unknown fields.
  - confidence: 0.82
