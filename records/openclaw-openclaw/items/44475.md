## Summary

Keep this PR open. Current main still has the narrower `agents.defaults.models` contract, and the PR is a useful implementation candidate, but it is not ready to merge because passthrough lookup bypasses the existing normalized model-ref contract and the public generated schema/docs are not refreshed.

## What This Changes

The PR relaxes `agents.defaults.models` entries to passthrough unknown fields, merges those fields into provider model definitions during `applyModelDefaults()`, updates the TypeScript entry type, and adds focused tests for passthrough metadata.

## Best Possible Solution

Land a revised config change that treats passthrough model metadata as a public config contract: normalize model-default override lookup the same way allowlists and aliases are normalized, keep provider model defaulting deterministic, and refresh generated schema, docs, changelog, and focused tests before maintainer merge.

## Reproduction Assessment

Yes. On the PR branch, configure a provider under `models.providers.amazon-bedrock` and set `agents.defaults.models["bedrock/<model>"].contextWindow`; existing allowlist/model-selection helpers normalize that key to `amazon-bedrock/<model>`, but the PR’s exact lookup misses it and drops the override.

## Solution Assessment

No, not yet. The passthrough direction is plausible after the mutation-preservation fix, but the implementation should reuse the normalized model-ref contract and update generated schema/docs before merge.
