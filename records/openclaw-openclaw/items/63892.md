## Summary

Keep open: current main still returns from the fresh-token preflight path before evaluating token-threshold compaction, and related open PRs are already tracking the same fix.

## What This Changes

The issue reports that proactive compaction stops re-firing after the first checkpoint because fresh persisted token totals bypass the preflight threshold check.

## Best Possible Solution

Land or replace a focused preflight fix so fresh persisted totals at or above the computed threshold reach the existing compaction gate, with regression coverage for fresh above-threshold tokens and no transcript-byte trigger.

## Reproduction Assessment

Yes. A deterministic source-level reproduction is a session entry with `totalTokensFresh: true`, a fresh `totalTokens` above `contextWindow - reserveTokensFloor - softThresholdTokens`, and no active transcript-byte trigger; current main returns before the threshold helper runs.

## Solution Assessment

Yes. The narrowest maintainable fix is to evaluate the existing threshold gate for fresh persisted totals before returning; changing `compactionCount` semantics would address the original hypothesis, not the confirmed root cause.
