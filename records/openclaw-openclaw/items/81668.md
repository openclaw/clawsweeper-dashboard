The PR updates Codex OAuth auth-profile eligibility, app-server auth refresh coverage, failover classification, and changelog text so oauthRef-backed Codex profiles stay selectable and refresh failures stop retrying.

## Best Possible Solution

Land the oauthRef eligibility and app-server selection fix, but narrow the failover change so only known permanent OAuth refresh reasons enter `auth_permanent` backoff.

## Reproduction Assessment

Yes, source-level. Current main rejects oauthRef-only Codex OAuth profiles as `missing_credential`, and the current failover classifier does not map known OAuth refresh/logout strings to `auth_permanent`.

## Solution Assessment

No, not as currently patched. The oauthRef fix is narrow and well-targeted, but the failover change should inspect the refresh-failure reason before treating the failure as high-confidence `auth_permanent`.
