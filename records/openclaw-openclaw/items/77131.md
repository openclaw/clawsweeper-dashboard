## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Keep an error listener while aborting timed-out sockets:** `src/gateway/shared-auth.test-helpers.ts:32-33`
  - body: When the open timeout fires, `cleanup()` removes the `error` listener before `ws.close()`. In ws 8.x, closing a CONNECTING socket aborts the handshake and emits an `error` on the next tick, so this can crash the Vitest worker instead of producing the bounded failure this helper is meant to add. Keep a one-shot error listener or otherwise drain that expected abort error while closing.
  - confidence: 0.88
