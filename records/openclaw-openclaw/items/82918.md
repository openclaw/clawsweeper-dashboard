## Summary

## What This Changes

This PR adds an early pipe-buffer guard to `scripts/install.sh` and a standalone shell invocation-matrix helper for piped, direct, command-string, process-substitution, and `/dev/null` stdin modes.

## Best Possible Solution

Land the installer-side buffering approach after maintainers either wire the invocation matrix into maintained validation or record explicit WSL2/macOS proof for the hosted installer path.

## Reproduction Assessment

No, not for a fresh live reproduction of the exact WSL2 hang in this review. The linked issue supplies concrete repeated logs, current main still documents the vulnerable piped path, and source inspection shows child commands can inherit the script stream.

## Solution Assessment

Mostly yes: an installer-side guard is the right boundary because docs-only invocation changes would not protect the supported `curl | bash` path. The safer merge path is to keep the guard narrow and add maintained invocation coverage or platform proof.
