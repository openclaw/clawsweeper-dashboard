## Summary

Keep open: the installer pipe-buffering fix targets a real documented `curl | bash` failure mode, but the current branch is not merge-ready because its cleanup path trusts an inherited environment variable and can remove an unrelated writable file.

## What This Changes

The PR adds a Bash pipe guard to `scripts/install.sh` so piped `curl | bash` execution buffers the script to a temporary file and re-execs Bash from that file.

## Best Possible Solution

Land a hardened pipe guard that deletes only its own buffered temp script, preserves documented piped arguments and direct execution, and adds focused installer regression tests.

## Reproduction Assessment

No, not as a fresh live WSL2 current-main reproduction of the original hang. The reporter logs, current documented pipe command, and current child-stdin source path make the bug source-reproducible, and the PR-introduced cleanup defect is source-verifiable from the diff.

## Solution Assessment

No, not in this exact form. Buffering the piped script is a plausible installer-level fix, but the cleanup marker must be validated and covered by focused tests before this is the best mergeable solution.
