## What This Changes

The PR adds a top-of-file pipe guard to `scripts/install.sh` that buffers piped stdin to a temp file and re-execs it, plus a changelog fix entry for the installer hang.

## Best Possible Solution

Land an installer-side stdin isolation fix with focused proof for the documented piped install path, then keep public install examples aligned with the tested behavior.

## Reproduction Assessment

No, not as a fresh end-to-end WSL2 reproduction in this review. The linked issue has concrete repeated logs, and current source plus prior review context make the pipe-stdin failure class source-reproducible.

## Solution Assessment

Yes, the installer-side pipe guard is a better fit than the earlier docs-only `bash -s --` workaround because it changes the process reading model before interactive or child commands run. A focused regression or smoke check would make the landing stronger.
