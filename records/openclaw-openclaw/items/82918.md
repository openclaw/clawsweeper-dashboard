## Summary

Keep open: the pipe-buffering approach is a plausible fix for the documented `curl | bash` stdin corruption, but this patch trusts an inherited cleanup marker and can remove an unrelated writable path before merge.

## What This Changes

The PR adds a top-of-file `scripts/install.sh` guard that buffers piped installer stdin to a temp file and re-execs Bash for `curl | bash` installs.

## Best Possible Solution

Land a hardened pipe guard that only cleans up its own buffered temp script, preserves piped arguments and direct execution, and adds focused installer regression tests.

## Reproduction Assessment

No for a high-confidence live current-main reproduction of the original WSL2 hang; the reporter logs, documented piped install command, and current source make the failure mode plausible. The PR-introduced cleanup defect is source-reproducible from the added lines.
