## Summary

Keep open: this is a focused, recent installer fix for a real `curl | bash` stdin-corruption report, but the current guard is too broad and can hang command-string execution before the installer starts.

## What This Changes

Adds a top-of-file `scripts/install.sh` guard that buffers piped stdin to a temp file and re-execs Bash so child processes cannot consume the script stream.

## Best Possible Solution

Keep the pipe-buffer approach, but gate it on actual stdin-script execution and cover the installer invocation matrix before landing.

## Reproduction Assessment

No high-confidence live WSL2 reproduction was run in this review. Current main still has an unbuffered `curl | bash` path, the related issue has truncation logs, and the PR proof demonstrates the intended Linux Bash path, so the original bug is source-reproducible with medium confidence.

## Solution Assessment

No, not as-is. The pipe-buffer/re-exec shape is a narrow fix for stdin stealing, but the `BASH_SOURCE`-only trigger also matches `bash -c`, so the guard needs tighter detection and focused regression coverage before merge.
