## Best Possible Solution

Keep the installer-side pipe guard, but validate the buffered marker against the current script path before cleanup, add focused installer tests for inherited-marker safety and piped argument preservation, then refresh cleanly onto current `main`.

## Reproduction Assessment

Partly: the original WSL2 hang was not rerun here, but current source and the linked report support the `curl | bash` stdin-consumption failure mode, and the new marker-deletion defect is directly source-reproducible from the diff.

## Solution Assessment

Mostly: buffering piped stdin is a maintainable installer-side fix for this failure mode, but the cleanup marker must be hardened and the branch refreshed before it is the best mergeable solution.
