## Summary

This focused installer PR should stay open but needs changes before merge: the pipe-buffering approach addresses a plausible documented `curl | bash` failure, yet the cleanup path currently trusts an inherited environment variable and can remove an unrelated writable file.

## What This Changes

The PR adds a top-of-file `scripts/install.sh` pipe guard that buffers piped installer input to a temp file and re-executes Bash from that file.

## Best Possible Solution

Land a hardened pipe guard that only cleans up its own buffered temp script, preserves piped arguments and existing installer behavior, and adds focused regression tests.

## Reproduction Assessment
