## Summary

## What This Changes

The PR adds an early stdin pipe-buffer guard to `scripts/install.sh` and a standalone shell invocation harness for piped, direct, command-string, process-substitution, and `/dev/null` stdin modes.

## Best Possible Solution

Land the installer-side pipe guard after exact-head checks finish and maintainers either accept the supplied Linux terminal proof or require WSL2/macOS or maintained invocation-matrix proof for the public installer path.

## Reproduction Assessment

No, I did not establish a fresh live current-main reproduction of the exact WSL2 hang. The path is source-reproducible from current docs plus installer code, and the linked issue supplies repeated logs showing script-stream corruption around `warn_shell_path_missing_dir`.

## Solution Assessment

Mostly yes: an installer-side guard is the right boundary because docs-only invocation changes do not protect the supported raw `curl | bash` path. The safer merge path is to keep this guard narrow and add maintained invocation coverage or platform proof if maintainers want higher confidence.
