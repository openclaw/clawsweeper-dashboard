## Summary

Keep open: current main still isolates the Codex app-server HOME under the per-agent Codex home and has no first-class `homeDir` or `OPENCLAW_CODEX_APP_SERVER_HOME_DIR` override. The linked open implementation PR, https://github.com/openclaw/openclaw/pull/80318, uses closing syntax for this issue, so this issue should stay open until that PR is reviewed, landed, or replaced.

## What This Changes

The issue requests a trusted-local Codex app-server `HOME` override via config/env while leaving per-agent `CODEX_HOME` isolation intact.

## Best Possible Solution

Land a narrow, explicit trusted-local `appServer.homeDir` and env fallback through the linked PR or a replacement, with schema, manifest, docs, focused tests, and real app-server/local-auth proof while preserving default HOME isolation.

## Reproduction Assessment

Yes. Source inspection of current main shows stdio app-server launches set `HOME` to `<agentDir>/codex-home/home` unless `startOptions.env.HOME` is already supplied, and existing tests assert that default isolation; I did not run a live `gh auth status` reproduction in this read-only review.

## Solution Assessment

Unclear as a final implementation, but the requested explicit opt-in override is a plausible narrow direction. The best path is to review and land https://github.com/openclaw/openclaw/pull/80318 after its schema/docs/proof blockers are resolved, rather than closing this issue before merge.
