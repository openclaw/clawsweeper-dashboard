## Summary

Keep open: current main intentionally isolates the Codex app-server HOME under the per-agent Codex home, and there is no first-class trusted-local HOME override yet.

## What This Changes

Add a trusted-local Codex app-server HOME override while preserving per-agent CODEX_HOME isolation.

## Best Possible Solution

Define a gated, documented trusted-local stdio app-server HOME override in the Codex plugin config/env surface, keep default isolation unchanged, and add schema, docs, and auth-bridge tests.

## Reproduction Assessment

Yes, source inspection and existing tests give a high-confidence reproduction of the current behavior: local app-server launches set HOME to the per-agent codex-home/home path unless startOptions.env already carries HOME. I did not run a live gh-auth reproduction in this read-only sweep.

## Solution Assessment

Unclear. A first-class explicit override is a plausible narrow shape, but because docs and tests intentionally reserve HOME for isolation, maintainers should decide the trusted-local policy and exact config/env contract before implementation.
