## Summary

Keep open: current main still intentionally isolates the Codex app-server HOME under the per-agent Codex home and does not expose the requested trusted-local HOME override. Adding that override changes a deliberate isolation boundary, so this needs maintainer policy review before implementation.

## What This Changes

Add a trusted-local Codex app-server HOME override through config/env while keeping CODEX_HOME isolated and preserving the default isolated HOME behavior.

## Best Possible Solution

Define a gated, documented trusted-local stdio app-server HOME override in the Codex plugin config/env surface, keep default isolation unchanged, preserve isolated CODEX_HOME, and update schema, docs, and auth-bridge tests together.

## Reproduction Assessment

Yes. Source inspection and existing tests give a high-confidence reproduction of the current behavior: local stdio app-server launches set HOME to the per-agent codex-home/home path unless internal startOptions.env already supplies HOME.

## Solution Assessment

Unclear. The proposed explicit override is a plausible narrow shape, but maintainers should decide the trusted-local policy and exact config/env contract because current docs and tests intentionally reserve HOME for isolation.
