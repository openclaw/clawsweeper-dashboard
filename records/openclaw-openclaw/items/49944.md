## Summary

Keep open. Current main still has the reported Control UI bug: `chat.history` returns the effective verbose level, but the Control UI history loader does not store it and the chat render still passes `autoExpandToolCalls: false`; https://github.com/openclaw/openclaw/pull/74398 remains open with closing syntax for this issue.

## What This Changes

Make Control UI expand tool cards by default when the effective `verboseLevel` or `agents.defaults.verboseDefault` is `full`.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/74398 so Control UI persists the effective `chat.history.verboseLevel`, derives tool-card auto-expansion from `full`, and clears stale verbose state on the same lifecycle paths as thinking state.

## Reproduction Assessment

Yes. The source path is high-confidence: configure `agents.defaults.verboseDefault = "full"`, load Control UI chat history, and current main receives `verboseLevel` from `chat.history` but discards it before rendering with `autoExpandToolCalls: false`.

## Solution Assessment

Yes. The narrow maintainable solution is Control UI state and render plumbing that consumes the already-returned effective verbose level; the open linked PR targets that boundary without changing the gateway contract.
