## Summary

Keep open. Current main still has the reported Control UI gap: the gateway returns an effective verbose level, but the Control UI history/render path does not preserve it and still disables tool-card auto-expansion. The discussion has an active follow-up PR, #74398, with closing syntax for the same bug, so this issue should wait for that PR to be reviewed, landed, replaced, or closed.

## What This Changes

The issue requests that Control UI/WebChat tool cards expand by default when the effective agent verbose level is `full`.

## Best Possible Solution

Carry the effective `chat.history.verboseLevel` through Control UI state, derive `autoExpandToolCalls` from `verboseLevel === "full"`, clear stale verbose state on the same paths as thinking state, preserve manual expansion toggles, and cover the controller/render path with focused regression tests plus a user-facing changelog entry.

## Reproduction Assessment

Yes. The static source path is enough for high-confidence reproduction: with `verboseDefault` set to `full`, the gateway returns `verboseLevel`, but the Control UI drops it and renders chat with `autoExpandToolCalls: false`.

## Solution Assessment

Yes. The narrow maintainable fix is Control UI state/render plumbing; #74398 targets that boundary, so the best next step is normal review, landing, replacement, or closure of that PR rather than closing this issue now.
