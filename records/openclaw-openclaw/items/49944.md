## What This Changes

Control UI should expand tool cards by default when `agents.defaults.verboseDefault` is `full`.

## Best Possible Solution

Review and land or replace #74398 so Control UI consumes `chat.history.verboseLevel` and feeds the existing `autoExpandToolCalls` seam while preserving manual tool-card toggles.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: configure verbose `full`, request Control UI history, and current main returns `verboseLevel` from the gateway but discards it before rendering with `autoExpandToolCalls: false`.

## Solution Assessment

Yes. The narrow maintainable fix is Control UI state/render plumbing only, and the linked PR follows that boundary rather than changing gateway metadata or disclosure-state helpers.
