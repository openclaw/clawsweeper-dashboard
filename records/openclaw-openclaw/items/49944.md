## Summary

Keep open. Current main and the latest release still drop the effective verbose level before Control UI rendering and still pass `autoExpandToolCalls: false`; https://github.com/openclaw/openclaw/pull/74398 is open, unmerged, and owns the implementation path for this bug.

## What This Changes

Control UI tool cards should default to expanded when the effective session verbose level from `agents.defaults.verboseDefault` is `full`.

## Best Possible Solution

Review, rebase, and land or replace https://github.com/openclaw/openclaw/pull/74398 so Control UI preserves `chat.history.sessionInfo.verboseLevel`, derives tool-card auto-expansion from `full`, and keeps manual disclosure behavior covered by focused tests.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: `chat.history` exposes the effective verbose level, but current Control UI stores only thinking level and renders with `autoExpandToolCalls: false`. I did not run a live browser session in this read-only review.

## Solution Assessment

Yes, the best fix direction is narrow Control UI plumbing: consume the already-returned effective verbose level and feed the existing expansion helper. The open linked PR follows that boundary but needs normal review, rebase, and proof before this issue closes.
