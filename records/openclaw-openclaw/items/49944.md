## Summary

Keep open. Current main still has the source-level bug: `chat.history` returns the effective verbose level, but Control UI does not store it and still renders chat with tool auto-expansion disabled; the linked fix PR at https://github.com/openclaw/openclaw/pull/74398 remains open with closing syntax for this issue.

## What This Changes

Honor `agents.defaults.verboseDefault = "full"` in Control UI so new chat tool cards expand by default.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/74398 so Control UI stores `chat.history.verboseLevel`, derives tool-card auto-expansion from `full`, clears stale verbose state on session/history resets, and keeps the existing gateway contract unchanged.

## Reproduction Assessment

Yes. The source path is high-confidence: configure `agents.defaults.verboseDefault = "full"`, load Control UI chat history, and current main receives `verboseLevel` from `chat.history` but discards it before rendering with `autoExpandToolCalls: false`.

## Solution Assessment

Yes. The narrow maintainable fix is Control UI state/render plumbing that consumes the already-returned effective verbose level; the open linked PR targets that boundary, so this issue should remain open until that PR lands, is replaced, or is closed.
