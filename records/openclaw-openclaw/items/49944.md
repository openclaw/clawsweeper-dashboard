## Summary

Keep open. Current main still drops the effective `chat.history.verboseLevel` before rendering Control UI chat and still hardcodes tool-card auto-expansion off; open PR #74398 already has closing syntax for the same bug, so this issue should remain open until that PR lands, is replaced, or is closed.

## What This Changes

The issue requests that Control UI tool cards expand by default when `agents.defaults.verboseDefault` resolves to `full`.

## Best Possible Solution

Carry `chat.history.verboseLevel` through Control UI state, derive `autoExpandToolCalls` from `verboseLevel === "full"`, reset stale verbose state with the same lifecycle paths as thinking state, and land or replace #74398 with focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes. The source-level path is high confidence: configure verbose `full`, load Control UI chat history, and current main receives `verboseLevel` from `chat.history` but discards it before rendering with `autoExpandToolCalls: false`.

## Solution Assessment

Yes. The narrow maintainable fix is Control UI state/render plumbing only; #74398 already targets that path, so the best next action is to review, land, replace, or close that PR rather than close this issue now.
