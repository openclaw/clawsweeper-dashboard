## Summary

Keep open. Current main and the latest release still expose the effective verbose level from `chat.history`, but Control UI drops that value before rendering and still passes `autoExpandToolCalls: false`; https://github.com/openclaw/openclaw/pull/74398 is open, unmerged, and owns the focused fix path.

## What This Changes

The issue requests that Control UI expand tool cards by default when the effective verbose level is `full`, including `agents.defaults.verboseDefault`.

## Best Possible Solution

Review, rebase, and land or replace https://github.com/openclaw/openclaw/pull/74398 so Control UI preserves `chat.history.verboseLevel`, derives tool auto-expansion from `full`, and keeps manual disclosure behavior covered by focused tests.

## Reproduction Assessment

Yes. Source inspection shows `chat.history` returns the effective verbose level, while current Control UI does not store it and still renders with `autoExpandToolCalls: false`; I did not run a live browser reproduction in this read-only review.

## Solution Assessment

Yes. The best fix is narrow Control UI plumbing that consumes the already-returned effective verbose level and feeds the existing expansion helper; the linked PR follows that boundary but needs rebase, review, and proof before this issue closes.
