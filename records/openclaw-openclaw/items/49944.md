## What This Changes

The issue asks Control UI to expand tool cards by default when the effective verbose setting from `agents.defaults.verboseDefault` is `full`.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/74398 so Control UI stores the effective `chat.history.verboseLevel`, derives tool-card auto-expansion from `full`, and preserves the existing manual disclosure behavior.

## Reproduction Assessment

Yes, at source level. Configure `agents.defaults.verboseDefault = "full"`, load Control UI chat history, and current main returns `verboseLevel` from `chat.history` while the UI drops it and renders with `autoExpandToolCalls: false`.

## Solution Assessment

Yes. The narrow maintainable fix is to consume the already-returned effective verbose level in Control UI state and feed the existing `autoExpandToolCalls` render path, which is exactly the boundary targeted by the open linked PR.
