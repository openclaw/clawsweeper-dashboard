## Summary

Keep open. Current main still has the Control UI data-flow gap: the gateway returns the effective verbose level, but the UI history loader does not store it and the chat render still disables tool-card auto-expansion; open PR #74398 already tracks a focused fix with closing syntax.

## What This Changes

The issue asks Control UI to expand tool cards by default when the effective verbose level is `full`.

## Best Possible Solution

Carry `chat.history.verboseLevel` through Control UI state, derive `autoExpandToolCalls` from `verboseLevel === "full"`, clear stale verbose state with the same lifecycle rules as thinking state, and land or replace the focused fix in #74398 with regression coverage and a changelog entry.

## Reproduction Assessment

Yes. The source-level path is high confidence: configure verbose `full`, load Control UI chat history, and current main receives `verboseLevel` from `chat.history` but discards it before rendering with `autoExpandToolCalls: false`.

## Solution Assessment

Yes. The narrow maintainable fix is Control UI state/render plumbing only; #74398 targets that boundary, so the best next action is to review, land, replace, or close that PR rather than close this issue now.
