## Summary

Keep open. Current main and latest release still match the report: the gateway returns the effective verbose level, but Control UI drops it before rendering and still hard-codes tool-card auto-expansion off; https://github.com/openclaw/openclaw/pull/74398 is the open, unmerged implementation path for this bug.

## What This Changes

Control UI should expand tool cards by default when the effective session verbose level is `full`, including `agents.defaults.verboseDefault = "full"`.

## Best Possible Solution

Review, rebase, and land or replace https://github.com/openclaw/openclaw/pull/74398 so Control UI stores `chat.history.verboseLevel`, derives tool-card auto-expansion from `full`, preserves manual disclosure behavior, and has focused regression coverage plus live UI proof.

## Reproduction Assessment

Yes, at source level. Configure `agents.defaults.verboseDefault = "full"`, load Control UI chat history, and current main returns `verboseLevel` from `chat.history` while `loadChatHistory` drops it and render still passes `autoExpandToolCalls: false`.

## Solution Assessment

Yes. The narrow maintainable solution is to consume the already-returned effective verbose level in Control UI state and feed the existing tool expansion path; the open linked PR follows that boundary.
