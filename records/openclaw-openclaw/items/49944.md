## Summary

Keep open. Current main still has the reported Control UI data-flow gap: `chat.history` returns the effective `verboseLevel`, but the Control UI history loader only stores `thinkingLevel` and the top-level chat render still passes `autoExpandToolCalls: false`. The issue is also paired with open, unmerged PR #74398 using closing syntax, so it should remain open until that PR is reviewed, landed, replaced, or closed.

## What This Changes

The issue requests that Control UI tool cards open by default when `agents.defaults.verboseDefault` or the effective session verbose level is `full`.

## Best Possible Solution

Carry the effective `chat.history.verboseLevel` through Control UI state, derive tool-card auto-expansion from `verboseLevel === "full"`, clear stale verbose state on the same lifecycle paths as thinking state, preserve manual expansion toggles, and cover the controller/render path with focused regression tests and a changelog entry.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: configure verbose `full`, load Control UI chat history, and the gateway returns `verboseLevel` while the Control UI discards it and renders with `autoExpandToolCalls: false`.

## Solution Assessment

Yes. The narrow maintainable fix is Control UI state/render plumbing only; #74398 targets that boundary, so the best next action is to review, land, replace, or close that PR rather than close this issue now.
