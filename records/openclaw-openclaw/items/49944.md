## Summary

Keep open. Current main and the latest release still return the effective verbose level from `chat.history` but drop it before Control UI rendering, and the open linked PR owns the exact fix path.

## What This Changes

Control UI tool cards should open by default when `agents.defaults.verboseDefault` makes the effective verbose level `full`.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/74398 so Control UI preserves `chat.history.verboseLevel`, derives tool auto-expansion from `full`, and keeps existing manual disclosure behavior intact.

## Reproduction Assessment

Yes, at source level. Configure `agents.defaults.verboseDefault = "full"`, load Control UI chat history, and current main returns `verboseLevel` from `chat.history` while `loadChatHistory` drops it and render still passes `autoExpandToolCalls: false`.

## Solution Assessment

Yes. The narrow maintainable fix is Control UI state/render plumbing that consumes the already-returned effective verbose level, which is the boundary of the open linked PR.
