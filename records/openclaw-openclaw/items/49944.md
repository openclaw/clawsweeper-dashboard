## Summary

Keep open. Current main still returns the effective verbose level from `chat.history` but the Control UI history loader discards it and the app render still passes `autoExpandToolCalls: false`; the linked implementation PR remains open and unmerged.

## What This Changes

The issue reports that Control UI tool cards stay collapsed even when `agents.defaults.verboseDefault` is set to `full`.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/74398 so Control UI persists `chat.history.verboseLevel`, derives tool-card auto-expansion from `full`, and keeps manual disclosure behavior intact.

## Reproduction Assessment

Yes. Configure `agents.defaults.verboseDefault = "full"`, load Control UI chat history, and current main returns `verboseLevel` from `chat.history` while the UI drops it and renders with `autoExpandToolCalls: false`; I did not run a live browser session.

## Solution Assessment

Yes. The narrow maintainable fix is Control UI state/render plumbing that consumes the already-returned effective verbose level, which is exactly the boundary targeted by the open linked PR.
