## What This Changes

Control UI should expand tool cards by default when the effective verbose level from `agents.defaults.verboseDefault` is `full`.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/74398 so Control UI persists `chat.history.verboseLevel`, derives `autoExpandToolCalls` from `full`, and preserves manual disclosure behavior.

## Reproduction Assessment

Yes, at source level. Configure `agents.defaults.verboseDefault = "full"`, load Control UI chat history, and current main returns `verboseLevel` from `chat.history` while the UI drops it and renders with `autoExpandToolCalls: false`.

## Solution Assessment

Yes. The narrow maintainable fix is Control UI state/render plumbing that consumes the already-returned effective verbose level, which is the boundary targeted by the open linked PR.
