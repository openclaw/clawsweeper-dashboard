## Summary

Keep open. Current main still matches the reported Control UI bug: the gateway returns the effective verbose level, but the UI history loader drops it and the top-level render still forces tool-card auto-expansion off; the open linked implementation PR should be reviewed, rebased, landed, replaced, or closed before this issue is resolved.

## What This Changes

The issue asks Control UI to expand tool cards by default when the effective verbose level from `agents.defaults.verboseDefault` is `full`.

## Best Possible Solution

Carry the effective `chat.history.verboseLevel` through Control UI state, derive `autoExpandToolCalls` from `verboseLevel === "full"`, preserve manual tool-card toggles, and land that path through the existing linked PR or a focused replacement.

## Reproduction Assessment

Yes, at source level. Configure `agents.defaults.verboseDefault = "full"`, load Control UI chat history, and current main returns `verboseLevel` from `chat.history` while the UI drops it and renders with `autoExpandToolCalls: false`.

## Solution Assessment

Yes. The best path is the narrow Control UI state/render plumbing already targeted by https://github.com/openclaw/openclaw/pull/74398, not a new config option or duplicate repair branch.
