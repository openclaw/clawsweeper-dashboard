## Summary

Keep open. Current main and the latest release still drop `chat.history.verboseLevel` in Control UI and render chat with `autoExpandToolCalls: false`; https://github.com/openclaw/openclaw/pull/74398 is the open closing PR for this issue but remains unmerged and conflicting.

## What This Changes

Control UI should expand tool cards by default when the effective verbose level is `full`.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/74398 so Control UI stores the effective `chat.history.verboseLevel`, derives tool-card auto-expansion from `full`, and preserves the existing manual disclosure behavior.

## Reproduction Assessment

Yes. The high-confidence source path is to configure `agents.defaults.verboseDefault = "full"`, load Control UI chat history, and observe that current main returns `verboseLevel` from `chat.history` but drops it before rendering with `autoExpandToolCalls: false`.

## Solution Assessment

Yes. The best fix direction is narrow Control UI state/render plumbing that consumes the already-returned effective verbose level; the linked PR targets that boundary without changing the gateway contract.
