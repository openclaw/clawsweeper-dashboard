## Summary

Keep open. Current main still strips durable managed env values before the macOS launchd env-file writer can export them, and open PR #75507 is already the direct closing fix candidate.

## What This Changes

The issue asks macOS launchd service-env regeneration to export actual durable managed env values instead of only listing them in `OPENCLAW_SERVICE_MANAGED_ENV_KEYS`.

## Best Possible Solution

Land #75507 or an equivalent narrow daemon install fix that preserves launchd owner-only env-file values while keeping systemd behavior and plist secret avoidance intact.

## Reproduction Assessment

Yes. Source inspection on current main shows durable `.env` values are collected, listed as managed keys, then deleted before launchd serializes only the remaining install environment into the owner-only env file.

## Solution Assessment

Yes. The maintainable fix belongs in the daemon install to launchd env-file handoff, not in gateway auth resolution or by making the wrapper read unrelated files directly.
