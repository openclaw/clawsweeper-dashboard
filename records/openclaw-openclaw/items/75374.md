## Summary

Keep open. Current main still has the launchd managed-env handoff that can turn durable values into `OPENCLAW_SERVICE_MANAGED_ENV_KEYS` metadata while removing the actual key values before the owner-only launchd env file is generated, and the issue is already paired with open closing PR #75507.

## What This Changes

The issue reports that macOS launchd service-env regeneration records managed environment key names but omits the exported values, causing env-backed gateway auth tokens to be unavailable to the gateway service.

## Best Possible Solution

The desired end state is a narrow daemon service-env fix that keeps durable managed values available to the launchd owner-only env file, keeps secrets out of LaunchAgent plist metadata, preserves existing systemd behavior, and includes focused regression coverage plus a user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: put a managed key in state-dir `.env`, build a gateway install plan, observe the durable value being deleted by `writeManagedServiceEnvKeysToEnvironment`, and then observe launchd serializing only the remaining environment entries.

## Solution Assessment

Yes. The best fix belongs in the install-plan-to-launchd env-file handoff, not in gateway auth resolution; the linked open PR's stated boundary matches that direction, so this issue should stay open until that PR or an equivalent fix lands.
