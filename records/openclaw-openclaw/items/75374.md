## Summary

Keep open. Current main still source-reproduces the macOS launchd service-env handoff bug, and open PR #75507 is already the closing fix candidate, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that macOS launchd service-env regeneration records managed env key names without exporting their values, causing env-backed gateway auth tokens and other managed keys to be unavailable at service start.

## Best Possible Solution

Land #75507 or an equivalent narrow daemon install fix that keeps durable managed values available to the launchd env-file writer while preserving systemd behavior and plist secret avoidance.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: durable `.env` values are collected, listed in `OPENCLAW_SERVICE_MANAGED_ENV_KEYS`, deleted from the install-plan environment, and therefore never reach the launchd env-file writer.

## Solution Assessment

Yes for the linked repair direction, but no for closing this issue before merge. The maintainable fix belongs in the daemon install to launchd env-file handoff, not in gateway auth resolution or a wrapper-only workaround.
