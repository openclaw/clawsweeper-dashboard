## Summary

Keep open. Current main still matches the reported macOS launchd service-env failure, and open PR #75507 is the direct closing fix candidate rather than a reason to close this issue before merge.

## What This Changes

The issue reports that macOS launchd service-env regeneration writes `OPENCLAW_SERVICE_MANAGED_ENV_KEYS` without exporting the listed managed values, causing env-backed gateway auth tokens to be missing at service start.

## Best Possible Solution

Land #75507 or an equivalent narrow daemon install fix that keeps durable managed values available to the launchd owner-only env-file writer without serializing secrets into the LaunchAgent plist.

## Reproduction Assessment

Yes. Source inspection on current main shows `.env` values are collected and listed as managed, then deleted before launchd serializes the owner-only env file from the remaining environment.

## Solution Assessment

Yes for the linked repair direction; the smallest maintainable fix is in the daemon install to launchd env-file handoff, not in gateway auth resolution or the wrapper alone.
