## Summary

Keep open. Current main still has the launchd managed-env handoff that can list durable managed keys while omitting their values, and open PR #75507 is already paired as the closing fix candidate.

## What This Changes

The issue asks for macOS launchd service-env regeneration to export actual managed environment values, not only `OPENCLAW_SERVICE_MANAGED_ENV_KEYS`, so env-backed gateway auth tokens survive service restart/install.

## Best Possible Solution

Review and land #75507 or an equivalent narrow fix that keeps durable managed values available to the owner-only launchd env file, keeps secrets out of LaunchAgent plist metadata, preserves current systemd behavior, and adds focused regression coverage plus a user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: state-dir `.env` values enter `durableEnvironment`, are removed by `writeManagedServiceEnvKeysToEnvironment`, and launchd serializes only the remaining plan environment into the sourced env file.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the launchd service-env regeneration/handoff path, not in gateway auth resolution; the linked open PR’s stated boundary matches that direction and should be reviewed or superseded before this issue is closed.
