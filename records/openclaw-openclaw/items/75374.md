## Summary

Keep open. Current main still matches the reported macOS launchd service-env failure, and open PR #75507 is the direct closing fix candidate rather than a reason to close this issue before merge.

## What This Changes

The issue asks for managed service environment values from the state-dir `.env` to be exported into the macOS LaunchAgent owner-only env file, not only listed in `OPENCLAW_SERVICE_MANAGED_ENV_KEYS`.

## Best Possible Solution

Land #75507 or an equivalent narrow daemon install fix that preserves systemd behavior while keeping durable managed values available to the launchd env-file writer.

## Reproduction Assessment

Yes. Source inspection on current main shows a durable `.env` key is collected and listed in `OPENCLAW_SERVICE_MANAGED_ENV_KEYS`, then deleted before launchd serializes the owner-only env file.

## Solution Assessment

Yes for the linked repair direction, but not for closing this issue before merge. The narrow maintainable fix belongs in the daemon install to launchd env-file handoff, not in gateway auth resolution or the wrapper alone.
