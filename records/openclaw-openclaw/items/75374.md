## Summary

Keep open. Current main still matches the reported macOS launchd service-env failure, and open PR #75507 is already the closing fix candidate, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that macOS LaunchAgent service-env regeneration export actual managed env values, not only `OPENCLAW_SERVICE_MANAGED_ENV_KEYS`.

## Best Possible Solution

Land #75507 or an equivalent narrow daemon install fix that keeps durable managed values available to the launchd env-file writer while preserving systemd behavior and plist secret avoidance.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: a durable `.env` key is collected, listed in `OPENCLAW_SERVICE_MANAGED_ENV_KEYS`, deleted from the environment, and therefore never reaches the launchd env-file writer.

## Solution Assessment

Yes for the linked repair direction, but no for closing this issue before merge. The maintainable fix belongs in the daemon install to launchd env-file handoff rather than in gateway auth resolution or the wrapper script alone.
