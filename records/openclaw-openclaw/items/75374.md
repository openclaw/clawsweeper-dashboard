## Summary

Current main still has the macOS launchd service-env regression: durable managed values are deleted before the launchd env-file writer serializes the service environment, and #75507 is open but unmerged as the closing fix candidate.

## What This Changes

The issue requests that launchd service-env regeneration export actual managed key values from durable `.env` sources, not only `OPENCLAW_SERVICE_MANAGED_ENV_KEYS` metadata.

## Best Possible Solution

Land #75507 or an equivalent narrow fix that retains durable managed values for the launchd owner-only env file, preserves systemd metadata behavior, and adds focused daemon install plus launchd regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a state-dir `.env` value enters `durableEnvironment`, is converted into managed-key metadata, is deleted by the managed-env writer, and therefore never reaches the launchd env-file writer.

## Solution Assessment

Yes for the linked repair direction, but not for closing this issue before merge. The narrow maintainable fix belongs in the daemon install to launchd env-file handoff, not in gateway auth token resolution.
