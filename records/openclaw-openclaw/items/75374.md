## Summary

Keep open: current main still strips durable managed env values before launchd serializes the env file, and open PR #75507 is already the closing fix candidate.

## What This Changes

The issue requests that macOS launchd service-env regeneration export actual managed key values from state-dir `.env` so env-backed gateway auth tokens and API keys are present at service startup.

## Best Possible Solution

Land #75507 or an equivalent narrow daemon install fix that retains durable managed values for launchd's owner-only env file, preserves systemd behavior, adds regression coverage, and includes the required user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a state-dir `.env` value enters `durableEnvironment`, is removed by `writeManagedServiceEnvKeysToEnvironment`, and launchd serializes only the reduced environment into the sourced env file.

## Solution Assessment

Yes for the linked repair direction, but not for closing this issue before merge. The maintainable fix belongs in the daemon install to launchd env-file handoff while keeping secrets out of the LaunchAgent plist.
