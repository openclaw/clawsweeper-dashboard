## Summary

Keep open. Current main still drops durable managed values before the launchd env-file writer serializes the service environment, and open PR #75507 is the closing implementation candidate, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that macOS launchd service-env regeneration records managed env key names but omits the corresponding exported values, leaving env-backed gateway auth tokens missing at service boot.

## Best Possible Solution

Land #75507 or an equivalent narrow launchd handoff fix that retains durable managed values for the owner-only env file, preserves systemd behavior, and updates the focused regression tests and changelog.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: a durable state-dir .env value enters durableEnvironment, is removed by the managed-key writer, and launchd serializes only the reduced environment into the sourced env file; the issue and follow-up comment also confirm it on v2026.4.29 macOS launchd installs.

## Solution Assessment

Yes for the linked repair direction, but no for closing this issue before merge. The narrow maintainable fix belongs in the daemon install to launchd env-file handoff rather than in gateway auth resolution.
