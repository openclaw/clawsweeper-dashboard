## Summary

Keep open. Current main still has the launchd managed-env handoff that can list durable managed keys while omitting their values, and open PR #75507 is already paired as the closing fix candidate.

## What This Changes

The issue asks for macOS launchd service-env regeneration to export actual managed env values from the state-dir `.env` so env-backed gateway tokens survive service install or restart.

## Best Possible Solution

Review and land #75507 or an equivalent narrow fix that keeps durable managed values available to the launchd owner-only env file, preserves systemd behavior, prevents token values from entering plist metadata, and adds focused regression coverage plus a user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a durable state-dir `.env` value enters `durableEnvironment`, is removed by `writeManagedServiceEnvKeysToEnvironment`, and launchd serializes only the reduced environment into the sourced env file.

## Solution Assessment

Yes for the linked repair direction, but no for closing this issue before merge. The narrow maintainable fix belongs in the launchd service-env regeneration/handoff path rather than gateway auth resolution.
