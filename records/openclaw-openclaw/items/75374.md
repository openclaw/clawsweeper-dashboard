## Summary

Keep open. Current main still drops durable managed values before the launchd env-file writer sees them, and open PR #75507 is already the closing fix candidate, so this issue should not be closed before that PR or an equivalent fix lands.

## What This Changes

The issue asks for macOS launchd service-env regeneration to export the actual managed environment values from durable `.env` sources so env-backed gateway auth tokens survive service install and restart.

## Best Possible Solution

Review and land #75507 or an equivalent narrow launchd handoff fix that retains durable managed values for the owner-only env file, preserves systemd behavior, adds focused regression tests, and includes the required user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a durable state-dir `.env` value enters `durableEnvironment`, is removed by the managed-key writer, and launchd serializes only the reduced environment into the sourced env file.

## Solution Assessment

Yes for the linked repair direction, but no for closing this issue before merge. The maintainable fix belongs in the daemon install to launchd env-file handoff rather than in gateway auth resolution.
