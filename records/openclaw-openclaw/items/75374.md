## Summary

Keep open. Current main still drops durable managed environment values before the macOS launchd env-file writer sees them, and open PR #75507 is already the paired closing fix candidate.

## What This Changes

The issue requests that launchd service-env regeneration export the actual managed values from the durable state-dir `.env`, not only `OPENCLAW_SERVICE_MANAGED_ENV_KEYS`, so env-backed gateway auth tokens are present when the service boots.

## Best Possible Solution

Review and land PR #75507 or an equivalent narrow fix that preserves systemd behavior, keeps token values out of plist metadata, writes durable managed values to the launchd owner-only env file, and adds focused regression coverage plus a user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a durable `.env` value enters `durableEnvironment`, is removed by `writeManagedServiceEnvKeysToEnvironment`, and launchd serializes only the reduced environment into the sourced env file.

## Solution Assessment

Yes for the linked repair direction, but no for closing before merge. The maintainable fix belongs in the launchd service-env regeneration and install-plan handoff, not in gateway auth resolution.
