## Summary

Keep open. Current main still strips durable managed env values before the launchd owner-only env file is written, and the linked closing PR https://github.com/openclaw/openclaw/pull/75507 is still open and unmerged.

## What This Changes

Ensure macOS launchd service-env regeneration writes actual managed env exports from durable env sources, not only the `OPENCLAW_SERVICE_MANAGED_ENV_KEYS` metadata list, so env-backed gateway auth tokens survive service restarts.

## Best Possible Solution

Review and land #75507 or an equivalent narrow launchd handoff fix that keeps durable managed values available to the owner-only env file while preserving systemd behavior and plist secret avoidance.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a durable state-dir `.env` key enters `durableEnvironment`, is deleted by the managed-key writer, and launchd writes only the reduced environment to the sourced env file.

## Solution Assessment

Yes for the linked repair direction, but not for closing this issue before merge. The maintainable fix belongs in the daemon install to launchd env-file handoff rather than in gateway auth resolution.
