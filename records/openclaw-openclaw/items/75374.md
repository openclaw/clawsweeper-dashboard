## Summary

Keep open. Current main still has the reported macOS launchd handoff bug, and open PR #75507 is already the closing fix candidate, so this issue should wait for that PR or an equivalent repair to merge.

## What This Changes

The issue reports that macOS launchd service-env regeneration records managed key names but drops the exported values, leaving env-backed gateway auth tokens unavailable at service start.

## Best Possible Solution

Land #75507 or an equivalent narrow daemon install fix that preserves durable managed values for the launchd owner-only env-file writer while keeping systemd behavior and plist secret avoidance intact.

## Reproduction Assessment

Yes. Source inspection on current main shows durable `.env`/config values are merged, listed in `OPENCLAW_SERVICE_MANAGED_ENV_KEYS`, then deleted before launchd serializes the env file; the current test suite still asserts that omission for a dotenv-backed key.

## Solution Assessment

Yes for the linked repair direction, but not for closing before merge. The maintainable fix belongs in the daemon install to launchd env-file handoff, with tests proving the plist still does not embed secret values.
