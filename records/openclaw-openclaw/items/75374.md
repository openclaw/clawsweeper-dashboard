## Summary

Keep open. Current main still has the launchd-specific service-env handoff described in the report: durable state-dir `.env` values are converted into managed-key metadata, the actual key values are removed from the install environment, and the LaunchAgent writer only serializes the remaining entries into the sourced env file. The issue is narrow, reproducible from source, and suitable for a focused fix PR.

## What This Changes

Fix macOS LaunchAgent service-env regeneration so keys listed in `OPENCLAW_SERVICE_MANAGED_ENV_KEYS` are exported with their actual managed values, preventing gateway auth token SecretRefs from resolving empty in the service process.

## Best Possible Solution

Add a macOS launchd service-env repair that rehydrates durable state-dir `.env` and config-managed values listed in `OPENCLAW_SERVICE_MANAGED_ENV_KEYS` into the owner-only `service-env/*.env` file, preserves `0600` file and `0700` wrapper/dir permissions, keeps secrets out of plist `EnvironmentVariables`, and leaves systemd behavior unchanged except for shared helper tests if needed.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: put a managed token in state-dir `.env`, build the gateway install environment, observe the actual key is removed while `OPENCLAW_SERVICE_MANAGED_ENV_KEYS` remains, then follow launchd's writer path, which only serializes the remaining entries.

## Solution Assessment

Yes. The best fix belongs in the launchd service-env regeneration/handoff path, not in gateway auth resolution, because auth is correctly failing when the service process lacks the configured SecretRef value.

## Repair Work Prompt

Fix the macOS LaunchAgent service-env regeneration bug described in openclaw/openclaw#75374. When durable state-dir `.env` values or config-managed env values are listed in `OPENCLAW_SERVICE_MANAGED_ENV_KEYS`, the launchd owner-only env file must contain the actual `export KEY='value'` lines so `gateway.auth.token = "${OPENCLAW_GATEWAY_AUTH_TOKEN}"` and similar SecretRefs work after `openclaw gateway install --force` and restart. Keep secrets out of LaunchAgent plist `EnvironmentVariables`; preserve `service-env` directory/file/wrapper permissions; do not regress systemd's `EnvironmentFile` behavior; do not add startup/load-time core migrations. Likely fix boundary is `src/daemon/launchd.ts` plus shared helper use from `src/config/state-dir-dotenv.ts` or `src/commands/daemon-install-helpers.ts` if needed. Add focused regression coverage for a state-dir `.env` managed key reaching the launchd env file while remaining absent from the plist. Add a changelog entry if the repo policy treats this user-facing gateway fix as requiring one, crediting the reporter `@adzendo` and any non-bot contributors.
