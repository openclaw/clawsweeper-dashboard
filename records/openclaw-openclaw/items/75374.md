## Summary

Keep open. Current main still removes durable managed values before the launchd env-file writer sees them, and open PR #75507 is already the closing fix candidate, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that macOS launchd service-env regeneration writes `OPENCLAW_SERVICE_MANAGED_ENV_KEYS` without exporting the actual managed environment values, causing env-backed gateway auth tokens to be missing at service startup.

## Best Possible Solution

Review and land #75507 or an equivalent narrow launchd handoff fix that preserves durable managed values for the owner-only env file while keeping secrets out of LaunchAgent plist metadata and preserving systemd behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a durable state-dir `.env` value enters `durableEnvironment`, is removed by the managed-key writer, and launchd serializes only the reduced environment into the sourced env file.

## Solution Assessment

Yes for the linked repair direction, but not for closing this issue before merge. The maintainable fix belongs in the daemon install to launchd env-file handoff rather than in gateway auth resolution.
