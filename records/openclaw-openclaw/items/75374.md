## Summary

Keep open: current main still has the macOS launchd handoff bug, and #75507 is an open unmerged closing PR, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that macOS launchd service-env regeneration records managed key names but omits the actual dotenv-backed exports, leaving gateway auth tokens and other managed secrets unavailable at service start.

## Best Possible Solution

Review and land #75507 or an equivalent narrow launchd handoff fix that retains durable managed values for the owner-only env file, preserves systemd behavior, adds focused regression coverage, and includes the user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a state-dir .env managed key enters durableEnvironment, is deleted when OPENCLAW_SERVICE_MANAGED_ENV_KEYS is written, and launchd writes only the resulting reduced environment to the owner-only env file.

## Solution Assessment

Yes for #75507's direction, but no for closing this issue before merge. The narrow maintainable fix belongs in the daemon install to launchd env-file handoff rather than gateway auth resolution.
