## Summary

Keep open. Current main still removes durable managed env values before the launchd env-file writer serializes them, and open PR #75507 is already the closing fix candidate, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that launchd service-env regeneration export actual managed key values from durable env sources, not only the OPENCLAW_SERVICE_MANAGED_ENV_KEYS metadata list.

## Best Possible Solution

Land #75507 or an equivalent narrow daemon install fix that retains durable managed values for launchd's owner-only env file, preserves systemd behavior, adds regression coverage, and includes the required user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a state-dir .env value enters durableEnvironment, is removed by writeManagedServiceEnvKeysToEnvironment, and launchd serializes only the reduced environment into the sourced env file.

## Solution Assessment

Yes for the linked repair direction, but not for closing this issue before merge. The maintainable fix belongs in the daemon install to launchd env-file handoff rather than in gateway auth resolution.
