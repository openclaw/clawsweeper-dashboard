## Summary

Keep open. Current main still has the launchd managed-env handoff that can record managed key names while omitting their values, and an open closing PR (#75507) is already paired with this issue, so the right cleanup point is after that PR or an equivalent fix lands.

## What This Changes

Ensure macOS launchd service-env generation exports the actual managed environment values listed in `OPENCLAW_SERVICE_MANAGED_ENV_KEYS` so env-backed gateway auth tokens are present when the service starts.

## Best Possible Solution

Review and land #75507 or an equivalent narrow fix that keeps durable managed values available to the owner-only launchd env file, keeps secrets out of LaunchAgent plist metadata, preserves current systemd behavior, and adds focused regression coverage plus a user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: state-dir `.env` values enter `durableEnvironment`, are removed by `writeManagedServiceEnvKeysToEnvironment`, and launchd serializes only the remaining plan environment into the sourced env file.

## Solution Assessment

Yes. The best fix is in the launchd service-env regeneration/handoff path, not gateway auth resolution; the linked open PR’s stated boundary matches that direction and should be reviewed or superseded before this issue is closed.
