## Summary

Keep open. Current main still has the reported launchd managed-env handoff gap, and open PR #75507 is already the closing fix candidate, so this issue should remain open until that PR or an equivalent fix merges.

## What This Changes

The issue requests that macOS service-env regeneration export the actual managed environment values, not only `OPENCLAW_SERVICE_MANAGED_ENV_KEYS`, so env-backed gateway auth tokens survive launchd startup.

## Best Possible Solution

Review and land #75507 or an equivalent narrow launchd handoff fix that retains durable managed values for the owner-only env file, preserves systemd behavior, adds regression coverage, and includes the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a state-dir `.env` value enters `durableEnvironment`, is removed by the managed-key writer, and launchd serializes only the reduced environment into the sourced env file.

## Solution Assessment

Yes for the linked repair direction, but no for closing this issue before merge. The maintainable fix belongs in the daemon install to launchd env-file handoff rather than in gateway auth resolution.
