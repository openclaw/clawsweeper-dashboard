## Summary

Keep open. Current main still has the reported macOS launchd service-env handoff bug, and open PR #75507 is the active closing fix candidate rather than a reason to close before merge.

## What This Changes

The issue asks for macOS launchd service-env regeneration to write actual managed `.env` values, not only `OPENCLAW_SERVICE_MANAGED_ENV_KEYS`, so env-backed gateway auth tokens survive service restarts.

## Best Possible Solution

Land #75507 or an equivalent narrow daemon install fix that keeps durable managed values available to the launchd env-file writer while preserving systemd behavior and plist secret avoidance.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a durable `.env` key is collected, listed in `OPENCLAW_SERVICE_MANAGED_ENV_KEYS`, deleted from the install-plan environment, and never reaches the launchd env-file writer.

## Solution Assessment

Yes for the linked repair direction, but not for closing this issue before merge. The maintainable fix belongs in the daemon install to launchd env-file handoff, not in gateway auth resolution or the wrapper alone.
