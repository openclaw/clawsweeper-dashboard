## Summary

Keep open. Current main still exhibits the launchd service-env handoff bug, and open PR #75507 is already the closing fix candidate, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to export durable managed service environment values into the macOS launchd owner-only env file so env-backed gateway auth tokens survive service regeneration.

## Best Possible Solution

Review and land #75507 or an equivalent narrow launchd handoff fix that preserves durable managed values for env-file serialization while keeping plist metadata secret-free.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a state-dir .env value enters durableEnvironment, is removed by writeManagedServiceEnvKeysToEnvironment after metadata generation, and launchd then writes only the reduced environment into the sourced env file.

## Solution Assessment

Yes for the linked repair direction, but no for closing this issue before merge. The maintainable fix belongs at the daemon install to launchd env-file handoff, not in gateway auth token resolution.
