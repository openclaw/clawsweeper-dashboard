## Summary

Keep open. Current main still has the launchd managed-env handoff that can list durable managed keys while deleting their values before the owner-only env file is generated, and open PR #75507 is already the paired closing fix candidate.

## What This Changes

The issue requests fixing macOS launchd service environment regeneration so managed keys from durable service env sources are exported with their actual values instead of only listed in `OPENCLAW_SERVICE_MANAGED_ENV_KEYS`.

## Best Possible Solution

Review and land #75507 or an equivalent narrow launchd handoff fix that exports durable managed values into the owner-only env file, keeps those values out of LaunchAgent plist metadata, preserves current systemd behavior, and adds focused regression coverage plus a user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: put a durable key in the state-dir `.env`, build a gateway install plan, and the current code collects the value, deletes it while writing `OPENCLAW_SERVICE_MANAGED_ENV_KEYS`, then lets launchd serialize only the remaining environment.

## Solution Assessment

Yes. The narrow maintainable solution belongs in the daemon install to launchd env-file handoff, not in gateway auth resolution; the open PR's stated boundary matches that direction and should be reviewed or superseded before this issue closes.
