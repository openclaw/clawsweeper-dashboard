## Summary

Keep open. Current main still drops durable managed env values before launchd serializes the owner-only env file, and open PR #75507 is already the closing fix candidate for the same bug.

## What This Changes

The issue asks macOS launchd service-env regeneration to export actual managed env values from the durable `.env` source alongside `OPENCLAW_SERVICE_MANAGED_ENV_KEYS` so gateway auth and other env SecretRefs survive service restarts.

## Best Possible Solution

Land #75507 or an equivalent narrow daemon install fix that keeps durable managed values available to launchd's owner-only env-file writer, preserves systemd behavior, and adds regression coverage plus the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a durable `.env` key enters the install plan, is removed by the managed-key writer, and launchd serializes only the reduced environment into the sourced env file; the second issue comment also confirms the macOS end-to-end failure.

## Solution Assessment

Yes for the linked repair direction, but not for closing before merge. The maintainable fix belongs at the daemon install to launchd env-file handoff, not in gateway auth resolution or a wrapper-side workaround.
