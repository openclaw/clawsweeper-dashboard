## Summary

Keep open. Current main still strips durable managed env values before the launchd env-file writer serializes them, and open PR #75507 is already the unmerged closing candidate for the reported macOS launchd failure.

## What This Changes

Fix macOS launchd service-env regeneration so managed keys listed in OPENCLAW_SERVICE_MANAGED_ENV_KEYS are also exported into the owner-only env file.

## Best Possible Solution

Land #75507 or an equivalent narrow daemon install fix that keeps durable managed values available to launchd's owner-only env-file writer while preserving systemd's current metadata behavior and adding regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a state-dir .env value enters durableEnvironment, is removed by writeManagedServiceEnvKeysToEnvironment, and launchd serializes only the reduced environment into the sourced env file.

## Solution Assessment

Yes for the linked repair direction, but not for closing this issue before merge. The maintainable fix belongs in the daemon install to launchd env-file handoff rather than in gateway auth resolution.
