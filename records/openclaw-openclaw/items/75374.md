## Summary

Keep open. Current main still matches the reported macOS launchd service-env failure, and open PR #75507 is already the direct closing fix candidate.

## What This Changes

The issue reports that macOS launchd service-env regeneration lists managed keys but omits their exported values, leaving env-backed gateway auth tokens unavailable at service boot.

## Best Possible Solution

Land #75507 or an equivalent narrow daemon install fix that keeps durable managed values available to the launchd env-file writer while preserving systemd behavior and plist secret avoidance.

## Reproduction Assessment

Yes. Source inspection on current main shows a durable `.env` key is collected and listed in `OPENCLAW_SERVICE_MANAGED_ENV_KEYS`, then deleted before launchd serializes `params.environment` into the owner-only env file.

## Solution Assessment

Yes for the linked repair direction, but not for closing this issue before merge. The maintainable fix belongs in the daemon install to launchd env-file handoff rather than in gateway auth resolution or the wrapper alone.
