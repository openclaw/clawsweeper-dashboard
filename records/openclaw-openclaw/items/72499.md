## Summary

Keep open: current main still has the followup, model-probe, and slug helper gaps, while this external PR is the active implementation but remains conflicting, lacks real behavior proof, and still has a cron tool-policy/security blocker.

## What This Changes

The PR routes CLI-backed followup, model status probe, LLM slug generation, and cron helper paths through CLI execution and adds MCP routing/message-send tracking.

## Best Possible Solution

Land a rebased or replacement change that routes followups, probes, and slug generation through the shared CLI dispatch pattern while preserving current-main cron tool-policy fail-closed behavior.

## Reproduction Assessment

Yes, at source level. Current main still has direct runEmbeddedPiAgent calls for queued followups, model auth probes, and LLM slug generation after provider/model selection; no live CLI-backed run was executed in this read-only review.

## Solution Assessment

No, not as-is. Routing helper paths through runCliAgent is the right direction, but the branch must be rebased and the cron CLI path must preserve or fail closed on tool/message policy before merge.
