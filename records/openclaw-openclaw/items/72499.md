## Summary

Keep open. The PR still targets real CLI-backed helper-path gaps, but the current PR head is not merge-ready because its cron CLI alias path drops existing cron runtime tool allowlist and message-tool policy, and it now overlaps newer cron work on main.

## What This Changes

The PR routes queued followups, model status probes, LLM slug generation, and canonical-provider cron runs through CLI-backed execution and adds session/MCP routing plus message-send tracking coverage.

## Best Possible Solution

Rebase the PR onto current main, keep the non-cron helper dispatch fixes, and make any cron CLI path policy-equivalent to the embedded path before merge.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: configure a cron agent turn whose canonical provider resolves to a CLI runtime and set toolsAllow to a restrictive list or disable message delivery; PR head enters runCliAgent without the embedded path’s tool-policy fields.

## Solution Assessment

No. Routing CLI-backed helper paths through runCliAgent is the right direction, but this patch is not the best implementation until the cron CLI path preserves runtime toolsAllow and message-tool policy or stays on the embedded path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve cron tool policy on CLI alias runs:** `src/cron/isolated-agent/run-executor.ts:170-205`
  - body: This CLI branch forwards only the owner-only cron grant, while the embedded branch still receives toolsAllow plus requireExplicitMessageTarget, disableMessageTool, and forceMessageTool. A cron job configured with a restrictive toolsAllow list or with message delivery disabled can now enter the CLI alias path with a broader bundled MCP/message tool surface than configured. Add equivalent CLI-side enforcement or keep these cron runs on the embedded path until policy is preserved.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The PR can weaken per-job cron tool restrictions for canonical-provider cron runs that resolve to CLI-backed runtimes.

Concerns:

- **[medium] Cron CLI alias path can bypass tool restrictions:** `src/cron/isolated-agent/run-executor.ts:170`
  - body: The new CLI cron branch does not carry the runtime toolsAllow allowlist or message-tool disable/require flags that the embedded branch enforces. Because bundled MCP tools are exposed through the CLI loopback, restrictive cron jobs can receive a broader tool surface than configured.
  - confidence: 0.88
