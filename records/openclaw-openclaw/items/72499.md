## Summary

Keep open. The PR targets a real source-level bug that still exists on current main for followups, model probes, and slug generation, but the branch is not merge-ready because it lacks real behavior proof and its cron CLI path does not preserve restricted cron tool policy.

## Best Possible Solution

Land a rebased or replacement change that routes followups, probes, and slug generation through shared CLI dispatch while preserving fail-closed cron tool policy.

Yes, at source level. Current main still has direct embedded-runner calls for queued followups, model auth probes, and LLM slug generation after provider/model selection; no live CLI-backed helper run was executed in this read-only review.

## Solution Assessment

No, not as-is. Reusing `runCliAgent()` is the right direction, but the branch needs a rebase and must preserve restricted cron tool policy before it is mergeable.
