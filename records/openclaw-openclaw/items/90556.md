## Summary

Keep open: current main and the latest release still do not render first-output, total response latency, token count, or throughput under assistant messages in the Chat UI. Current code has diagnostic chat-send timing for first assistant events, and an open related PR partially explores more timing phases, but the durable UI/history contract remains undecided and unimplemented.

## What This Changes

Request to display assistant response latency and optional token/throughput metrics under each Chat UI assistant message.

## Best Possible Solution

Define one canonical assistant-turn timing contract that works for live events and history reloads, then render it through the existing compact assistant metadata footer.

## Reproduction Assessment

No runtime bug reproduction applies because this is a feature request. Source inspection gives high confidence that current main lacks per-message first-output or total-latency metadata in the chat stream/history/UI rendering path.

## Solution Assessment

Unclear pending maintainer product direction. The narrow maintainable path is to choose the timing metadata owner first, not to derive a visible message footer only from transient client diagnostics.
