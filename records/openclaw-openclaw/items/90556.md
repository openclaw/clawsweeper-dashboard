## Summary

Keep open: current main has transient Control UI chat-send timing instrumentation and existing assistant message context metadata, but it does not render first-token latency, total latency, token count plus throughput under each assistant message. The related items in the provided context are closed and cover different latency/API problems, so there is no canonical duplicate for this feature request.

## What This Changes

The issue requests showing first-token latency, total response latency, token count, and throughput under assistant messages in the Control UI chat surface.

## Best Possible Solution

Add a small, additive chat timing metadata contract that preserves current chat-event compatibility, attaches durable enough timing/usage data to assistant turns, and renders it through the existing message metadata footer with focused UI, gateway, and protocol tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new UI feature request. Source inspection confirms current main lacks the requested under-message latency display while only buffering timing as performance telemetry.

## Solution Assessment

Unclear until maintainers choose the timing contract, but the best direction is not to scrape Gateway logs; it should flow through a typed live/history chat metadata path and reuse existing timing and usage data where possible.
