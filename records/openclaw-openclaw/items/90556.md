## Summary

Keep open: current main and v2026.6.9 still do not expose durable per-message first-output latency, total latency, token count, or throughput for Chat UI rendering. The existing timing path is diagnostic-only, and the open related PR is partial and currently not a complete UI/history solution.

## What This Changes

Request to display response latency, and optionally token count and throughput, under each assistant message in the Chat/Control UI.

## Best Possible Solution

Define a canonical assistant-turn timing metadata contract that works for live streaming and history reloads, then render it through the existing assistant metadata footer with focused gateway and UI coverage.

## Reproduction Assessment

No runtime bug reproduction applies because this is a feature request. Source inspection gives high confidence that current main lacks per-message latency metadata in the chat stream, history payload, and grouped assistant footer.

## Solution Assessment

Unclear pending maintainer product direction. The maintainable path is to choose the timing metadata owner first, rather than deriving a visible footer only from transient diagnostic timing events.
