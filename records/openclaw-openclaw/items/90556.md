## Best Possible Solution

Define one canonical assistant-turn timing contract that works for live events and history reloads, then render it through the existing assistant metadata footer after maintainer direction.

## Solution Assessment

Unclear pending maintainer product direction. The maintainable path is to choose the timing metadata owner first, then render from that canonical live/history contract rather than deriving a footer only from transient diagnostics.
