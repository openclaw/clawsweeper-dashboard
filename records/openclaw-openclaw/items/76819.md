## Summary

Keep open. The PR targets a real WebChat transcript persistence regression, but the latest diff still needs revision because it appends a gateway-injected assistant message for agent-run final text without checking whether Pi already wrote the normal assistant turn, which can duplicate `chat.history`.

## What This Changes

The PR adds a silent gateway transcript append path for WebChat agent-run final text, threads a `silent` option through injected transcript appends, updates regression tests, and adds a changelog entry.

## Best Possible Solution

Preserve the silent persistence idea only as a precise fallback that activates when the transcript is missing the final assistant turn, or fix the Pi/session mirror path that should write that turn normally.

## Reproduction Assessment

Yes. The original gap is source-reproducible from the current `agentRunStarted` branch and the PR's focused test fixture; the PR blocker is also source-reproducible because the new fallback has no check for an existing Pi assistant turn.

## Solution Assessment

No. The silent append avoids the earlier UI reload side effect, but the maintainable fix must first verify that Pi did not already persist the assistant `message_end` before appending a gateway-injected duplicate.
