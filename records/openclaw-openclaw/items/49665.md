The issue reports that the Control UI Channels page can render blank when a lazy-loaded view import rejects, leaving no visible error or retry path.

## Best Possible Solution

Keep the shipped createLazyView/renderLazyView implementation and close catch-only duplicates for this symptom.

## Reproduction Assessment

## Solution Assessment

Yes. The shipped shared helper is stronger than a bare catch because it surfaces failure state, avoids automatic retry loops while hasError is set, and has focused browser coverage.
