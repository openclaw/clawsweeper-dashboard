## Summary

Keep this issue open: current main and v2026.6.8 still have Telegram pin/unpin helper plumbing, but the agent-facing Telegram message action discovery, dispatch map, runtime aliases, and action gates do not expose `pin` or `unpin`. The referenced implementation commit was not merged into any branch or tag, so this remains a real product/API decision rather than implemented cleanup.

## What This Changes

Expose Telegram `pin` and `unpin` as message tool actions so agents can programmatically manage pinned Telegram status messages.

## Best Possible Solution

Add Telegram-plugin-owned `pin` and `unpin` message actions after maintainer approval, with explicit gating, safe message-id semantics, docs, focused tests, and live Telegram proof.

## Reproduction Assessment

Yes. At source level, configured Telegram advertises and dispatches several message actions but not `pin` or `unpin`; a live Telegram probe is not useful until an action surface exists to exercise.

## Solution Assessment

Unclear until maintainers choose the product contract. The maintainable path is Telegram-plugin-owned action discovery and runtime dispatch using the existing helpers, not a core workaround or the unmerged referenced commit as-is.
