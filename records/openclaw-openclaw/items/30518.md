- `P2`: This is a well-defined improvement to an existing Telegram workflow with limited blast radius and no active outage.
- `impact:other`: Agents maintaining pinned Telegram dashboards currently require manual human intervention because the action is absent.

## Summary

Current main still lacks Telegram pin/unpin dispatch even though the shared message-action contract already supports both actions. The issue remains useful, but the action-gating/default and unpin scope should receive maintainer product review before implementation.

## What This Changes

Add Telegram message-tool actions for pinning and unpinning a specified message so agents can manage pinned status messages without manual intervention.

## Best Possible Solution

Implement required-message `pin` and `unpin` through the existing typed channel-action contract inside the Telegram plugin, preserve account-scoped action gating, and treat unpin-all as a separate explicitly named capability if it is desired later.

## Reproduction Assessment

Yes, at high confidence from current source and v2026.6.5: the shared contract accepts `pin` and `unpin`, but Telegram neither advertises nor dispatches them.

## Solution Assessment

The requested direction is sound, but the best implementation should use the current Telegram plugin boundary and require a message ID for both actions; maintainers must first choose the permission-gating/default policy.

## Risks / Open Questions

- Pinning is a privileged, state-changing chat action; maintainers should decide whether it is enabled by default, controlled only by the shared action allowlist, or receives a Telegram-specific action gate.
- Telegram supports both unpinning one message and unpinning all messages, while sibling channel `unpin` actions require a message ID; silently overloading omission to mean unpin-all would be an unsafe semantic difference.
