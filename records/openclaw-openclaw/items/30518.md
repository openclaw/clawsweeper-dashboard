## Summary

Keep this open: current main and the latest release still omit standalone Telegram `pin`/`unpin` from the agent-facing message-action descriptor and runtime aliases, even though lower-level Telegram pin/unpin helpers exist. The remaining blocker is maintainer approval for the new chat-administration write action, including its action gate/default and notification semantics.

## What This Changes

Request to expose Telegram message pin and unpin as agent/CLI message actions so agents can maintain pinned status messages without manual intervention.

## Best Possible Solution

Add the feature in the Telegram plugin action layer using the existing send helpers, after maintainers approve the gate/default/notification contract and require focused docs, tests, and live Telegram proof.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the missing exposure: current Telegram descriptor and runtime aliases omit standalone `pin`/`unpin` while docs and release state match that absence.

## Solution Assessment

Unclear until maintainers decide the product contract. The maintainable implementation should stay in the Telegram plugin action layer and reuse existing helpers, but the gate/default/notification semantics need approval first.
