## Summary

Keep open: current main and the latest release still expose only Telegram delivery-time pinning plus low-level pin/unpin helpers; standalone Telegram message-tool `pin` and `unpin` discovery, gating, and dispatch remain missing, and the prior implementation PRs were closed unmerged.

## What This Changes

The issue requests exposing Telegram Bot API `pinChatMessage` and `unpinChatMessage` as standalone Telegram message-tool `pin` and `unpin` actions.

## Best Possible Solution

Add Telegram-plugin-owned `pin` and `unpin` discovery plus runtime dispatch behind a maintainer-approved gate, reuse the existing helpers, update docs/tests, and validate with live Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source, docs, latest-release, and dependency checks give high confidence that standalone Telegram `pin` and `unpin` action routing is still missing.

## Solution Assessment

Yes, if maintainers approve the new write-action surface. The narrow maintainable path is plugin-local discovery, dispatch, and gating that reuses existing helpers without changing core channel contracts.
