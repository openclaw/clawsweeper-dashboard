## Summary

Keep open: current main has Telegram delivery-pin support and low-level Bot API helpers, but it still does not expose standalone Telegram `pin`/`unpin` message actions through discovery, runtime aliases/dispatch, config gating, or docs; the related implementation PRs are closed unmerged.

## What This Changes

Expose Telegram message-tool `pin` and `unpin` actions so agents can pin and unpin existing Telegram messages programmatically.

## Best Possible Solution

Add Telegram-plugin-owned `pin`/`unpin` action discovery and runtime dispatch behind a consistent action gate, reuse the existing send helpers, document it, add focused regression coverage, and leave pinned-message reads to https://github.com/openclaw/openclaw/issues/38580.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request, not broken documented behavior. Source inspection on current main gives high confidence that standalone Telegram `pin`/`unpin` action routing is still missing.

## Solution Assessment

Yes. The narrow maintainable solution is Telegram plugin-local action discovery and runtime dispatch that reuses existing Bot API helpers, adds a consistent action gate, and avoids changing core channel contracts beyond the already-present shared action names.
