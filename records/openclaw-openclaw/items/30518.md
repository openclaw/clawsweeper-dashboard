## Summary

Keep open: current main still exposes Telegram pinned delivery for newly sent messages and low-level Bot API helpers, but it does not advertise or execute manual Telegram `pin`/`unpin` message tool actions for existing messages. Prior closed PRs attempted this feature, but the current extension-owned implementation still lacks the wiring.

## What This Changes

Expose Telegram Bot API pin/unpin operations as message tool actions for existing Telegram messages.

## Best Possible Solution

Add Telegram plugin-local pin/unpin discovery and runtime wiring using the existing helpers, with matching schema/config docs, regression tests, and a user-facing changelog entry.

## Solution Assessment
