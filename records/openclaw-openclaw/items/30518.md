## Summary

Keep open: current main and the latest release have Telegram Bot API helpers plus delivery-time pinning for newly sent messages, but standalone Telegram `message(action=pin)` and `message(action=unpin)` are still absent from discovery and dispatch. The two direct implementation PRs for this request were closed unmerged, so no merged item supersedes the remaining work.

## What This Changes

The issue requests Telegram pin and unpin message actions so agents can maintain pinned dashboard messages without manual intervention.

## Best Possible Solution

Add plugin-owned Telegram `pin` and `unpin` message actions that advertise through `extensions/telegram/src/channel-actions.ts`, route through `extensions/telegram/src/action-runtime.ts`, reuse the existing send helpers, and include focused source plus live Telegram validation.

## Reproduction Assessment

Yes. Current main and v2026.6.6 expose only delivery-time Telegram pinning; standalone Telegram pin/unpin is absent from discovery, dispatch aliases, runtime branches, and docs-visible action support.

## Solution Assessment

Yes, if kept narrow. The best implementation is to extend the Telegram plugin's existing message action adapter/runtime and reuse its send helpers, with a maintainer decision on action gating before adding any new config surface.
