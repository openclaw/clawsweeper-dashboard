## Summary

Keep open: current main still lacks standalone Telegram `pin` and `unpin` message-tool action discovery and runtime routing, even though delivery pinning and low-level helper functions exist. The remaining work is a new user-visible Telegram action surface with gating, so the current maintainer/product-decision labels still fit.

## What This Changes

Request to expose Telegram Bot API pinChatMessage/unpinChatMessage as standalone Telegram message-tool `pin` and `unpin` actions for programmatic pinned status dashboards.

## Best Possible Solution

Add Telegram-plugin-owned `pin` and `unpin` discovery plus runtime dispatch behind a maintainer-approved action gate, reuse the existing helpers, document Bot API permissions, and require live Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection on current main gives high confidence that standalone Telegram `pin` and `unpin` action routing is still missing.

## Solution Assessment

Yes, if maintainers accept the new action surface. The narrow maintainable path is Telegram plugin-local discovery and dispatch that reuses existing helpers and adds a consistent action gate without changing core channel contracts.
