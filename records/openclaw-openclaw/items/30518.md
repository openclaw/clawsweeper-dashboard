## Summary

Keep open: current main still supports Telegram delivery pinning and low-level Bot API helpers, but standalone Telegram `pin`/`unpin` message actions are not advertised, routed, gated, or documented. The prior implementation PRs are closed unmerged, and the open pinned-message read request is related but distinct.

## What This Changes

Request to expose Telegram `pin` and `unpin` as message actions so agents can programmatically maintain pinned dashboard/status messages.

## Best Possible Solution

Add Telegram-plugin-owned `pin`/`unpin` discovery and runtime dispatch behind a consistent `actions.pins` gate, reuse the existing send helpers, document the actions, add focused regression coverage, and leave pinned-message reads to https://github.com/openclaw/openclaw/issues/38580.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection on current main gives high confidence that standalone Telegram `pin`/`unpin` action routing is still missing.

## Solution Assessment

Yes, if maintainers accept the new action surface. The narrow maintainable solution is Telegram plugin-local action discovery and runtime dispatch that reuses existing Bot API helpers and adds a consistent action gate without changing core channel contracts.
