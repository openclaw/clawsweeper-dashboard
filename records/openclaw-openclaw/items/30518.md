## Summary

Current main still lacks the requested manual Telegram `pin` and `unpin` message actions. The repo has shared action names and low-level Telegram Bot API helpers, but the bundled Telegram plugin does not yet advertise, map, alias, dispatch, gate, document, or test pin/unpin as user-invoked message actions.

## What This Changes

Expose Telegram `pin` and `unpin` as manual message actions so agents can programmatically maintain pinned Telegram messages.

## Best Possible Solution

Add Telegram-owned manual `pin` and `unpin` support in the bundled plugin by wiring the existing Bot API helpers through action discovery, action mapping, runtime aliases, runtime dispatch, optional config gates, docs, focused tests, and a user-facing changelog entry crediting the reporter.

## Reproduction Assessment

Yes. On current main, inspecting the Telegram plugin action map/discovery/runtime is enough to reproduce the gap: `pin` and `unpin` are not advertised or mapped and would reach the unsupported-action branch.

## Solution Assessment

Yes. The narrowest maintainable solution is plugin-owned wiring around `pinMessageTelegram` and `unpinMessageTelegram`, keeping send-time `delivery.pin` separate and avoiding new core action names.
