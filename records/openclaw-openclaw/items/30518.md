## Summary

Keep open: current main still has Telegram delivery pinning and low-level pin/unpin helpers, but it does not expose standalone Telegram `pin` or `unpin` message-tool actions. The remaining decision is user-visible action surface and gating, which matches the current maintainer/product-review labels.

## What This Changes

Add Telegram message-tool actions for pinning and unpinning existing messages.

## Best Possible Solution

Add Telegram-plugin-owned `pin` and `unpin` discovery and runtime dispatch behind a maintainer-approved action gate, reuse the existing helpers, document permissions and usage, and keep pinned-message read/list work separate.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection on current main gives high confidence that standalone Telegram `pin` and `unpin` action routing is still missing.

## Solution Assessment

Yes, if maintainers accept the new action surface. The narrow maintainable path is Telegram plugin-local discovery and dispatch that reuses existing Bot API helpers and adds a consistent action gate without changing core channel contracts.
