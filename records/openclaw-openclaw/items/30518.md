## Summary

Keep open: current main has Telegram send-layer pin/unpin helpers and pinned-delivery support, but it still does not expose agent-facing Telegram `pin` or `unpin` message actions through discovery or dispatch. The remaining work is real and should wait for maintainer product/security direction on gating, defaults, and unpin semantics.

## What This Changes

Request to expose Telegram message-tool `pin` and `unpin` actions so agents can programmatically pin or unpin existing Telegram messages.

## Best Possible Solution

Add Telegram-plugin-owned `pin` and `unpin` message actions with explicit gating, conservative unpin semantics, docs, focused action/runtime tests, and live Telegram proof after maintainer product/security approval.

## Reproduction Assessment

Yes, at source level: current Telegram discovery, action mapping, and runtime aliases omit `pin` and `unpin`, while the latest release has the same standalone action gap. I did not run a live Telegram probe because there is no current action surface to exercise.

## Solution Assessment

Unclear until maintainers choose the privileged action contract. The likely best fix is a Telegram-plugin-owned action surface with a dedicated gate and conservative unpin semantics, not a core workaround or delivery-pin-only behavior.
