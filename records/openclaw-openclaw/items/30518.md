## Summary

Keep open: current main still lacks standalone Telegram `pin`/`unpin` message action discovery and dispatch, even though shared action names and low-level Telegram Bot API helpers already exist. Prior focused implementation PRs were closed unmerged, and there is no open linked PR ready to land.

## What This Changes

Request to expose Telegram `pin` and `unpin` as message actions so agents can programmatically maintain pinned status messages.

## Best Possible Solution

Add Telegram plugin-owned `pin`/`unpin` discovery and runtime dispatch around the existing helpers, with a consistent `actions.pins` gate, tests, docs, and changelog; keep pinned-message reads/listing scoped to https://github.com/openclaw/openclaw/issues/38580.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request, not broken documented behavior. Source inspection gives high confidence that current main still lacks standalone Telegram `pin`/`unpin` action routing while retaining low-level helper support.

## Solution Assessment

Yes. The narrow maintainable path is plugin-local Telegram action wiring and regression coverage, without adding Telegram-specific behavior to core or bundling pinned-message reads into this request.
