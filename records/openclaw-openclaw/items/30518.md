## Summary

Keep open: current main has Telegram delivery pinning and low-level Bot API helpers, but standalone Telegram `pin`/`unpin` message actions are still not advertised, routed, gated, or documented. The closed unmerged implementation PR shows this was attempted, and the open pinned-message read request is related but distinct.

## What This Changes

Request to expose Telegram Bot API pin and unpin as first-class Telegram message tool actions.

## Best Possible Solution

Add Telegram-plugin-owned `pin`/`unpin` discovery and runtime dispatch behind a maintainer-approved action gate, reuse the existing helpers, document permissions and usage, and leave pinned-message reads tracked separately.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection on current main gives high confidence that standalone Telegram `pin`/`unpin` action routing is still missing.

## Solution Assessment

Yes, if maintainers accept the new action surface. The narrow maintainable solution is Telegram plugin-local action discovery and runtime dispatch that reuses existing Bot API helpers and adds a consistent action gate without changing core channel contracts.
