## Summary

Keep open. Current main still lacks manual Telegram `pin`/`unpin` message actions: the shared names and low-level Telegram helpers exist, but the Telegram plugin does not advertise, map, alias, dispatch, gate, document, or test those manual actions; prior closing PRs are closed unmerged.

## What This Changes

Expose Telegram Bot API `pinChatMessage` and `unpinChatMessage` as manual Telegram message `pin` and `unpin` actions.

## Best Possible Solution

Add Telegram-owned manual `pin` and `unpin` support in the bundled plugin, reusing existing Bot API helpers while keeping send-time `delivery.pin` behavior separate.

## Reproduction Assessment

Yes. As a source-level capability gap, current main clearly shows `pin` and `unpin` are absent from Telegram action discovery, mapping, aliases, and dispatch, while unsupported actions fail in the Telegram adapter.

## Solution Assessment

Yes. The narrow maintainable path is plugin-owned wiring around existing `pinMessageTelegram` and `unpinMessageTelegram`, plus action gating, docs, and focused tests; no core action-name change is needed.
