## Summary

Keep open: current main has Telegram delivery pinning and low-level pin helpers, but it still lacks standalone Telegram `pin`/`unpin` message-tool discovery, runtime routing, and an approved action gate.

## What This Changes

Expose standalone `pin` and `unpin` message-tool actions for Telegram so agents can programmatically pin status-dashboard messages.

## Best Possible Solution

Add Telegram-plugin-owned `pin` and `unpin` discovery plus runtime dispatch behind a maintainer-approved action gate, reuse the existing helpers, document Bot API permission expectations, and require live Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection on current main gives high confidence that standalone Telegram `pin` and `unpin` action routing is still missing.

## Solution Assessment

Yes, if maintainers accept the new action surface. The narrow maintainable path is Telegram plugin-local discovery and dispatch that reuses existing helpers and adds a consistent action gate without changing core channel contracts.
