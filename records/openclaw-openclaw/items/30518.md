## Summary

Current main and v2026.6.6 still expose only delivery-time Telegram pinning, not standalone pin/unpin actions for existing messages. The request remains useful, but maintainers must choose the privileged-action gate/default before implementation; likely routing is @steipete, @gumadeiras, and @mbelinky.

## What This Changes

Add Telegram message-tool actions to pin and unpin existing messages programmatically.

## Best Possible Solution

Implement message-ID-required pin and unpin inside the Telegram plugin through the existing typed action contract, preserve account-scoped action gating, and reserve unpin-all for a separately named capability if maintainers want it.

## Reproduction Assessment

Yes. Current main and v2026.6.6 clearly show delivery-time pinning but no standalone Telegram pin/unpin advertisement or dispatch for existing messages.

## Solution Assessment

Partly yes: the existing Telegram plugin action boundary and required-message-ID precedent are the narrowest maintainable solution, but maintainers must first decide the permission gate and default policy.
