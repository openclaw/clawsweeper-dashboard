## Summary

Keep this issue open. Current main and the latest release have Telegram delivery-pin helpers, but the Telegram message action discovery and dispatch surface still do not expose `pin` or `unpin`, so the requested agent-facing action remains a real feature/product decision.

## What This Changes

Expose Telegram `pin` and `unpin` as message tool actions so agents can programmatically manage pinned Telegram messages.

## Best Possible Solution

Add plugin-owned Telegram pin/unpin message actions after maintainer approval, with explicit gating, safe unpin semantics, docs, and focused Telegram action tests.

## Reproduction Assessment

Yes, at source level: configured Telegram advertises send/poll/react/delete/edit/topic actions but not pin/unpin, and unmapped actions throw before dispatch. I did not run a live Telegram probe because there is no current action surface to exercise.

## Solution Assessment

Unclear until maintainers choose the product contract. The maintainable path is Telegram-plugin-owned action discovery and runtime dispatch, not a core workaround or reuse of delivery-pin alone.
