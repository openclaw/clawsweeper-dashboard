## Summary

Current main and v2026.6.5 ship Telegram pin/unpin API helpers, but the agent-facing Telegram message-action descriptor and dispatcher still do not expose standalone pin or unpin actions. The request remains valid and focused; however, adding a chat-administration write action requires maintainer approval for its action-gate key and secure default before implementation.

## What This Changes

Expose Telegram pin and unpin as standalone message-tool actions for programmatically maintaining pinned Telegram messages.

## Best Possible Solution

Add typed pin and unpin actions to the current Telegram plugin descriptor and runtime, reuse the shipped API helpers, require an explicit secure action gate at global and account scope, and document Telegram administrator-permission and notification behavior.

## Reproduction Assessment

Yes at the source level: current main rejects pin and unpin because the Telegram action descriptor/dispatcher omits those names, while the underlying helpers already exist. A live Telegram run is unnecessary to establish this missing feature surface.

## Solution Assessment

The capability is appropriate, but the unmerged old core-path patch is not the best current solution. The maintainable fix is a plugin-owned typed action with an explicit authorization/default decision and focused live Telegram proof.
