## Summary

Keep open: current main and v2026.6.5 have Telegram delivery-pin support and low-level pin/unpin helpers, but the Telegram plugin still does not expose standalone pin or unpin message actions through discovery or dispatch. The remaining blocker is the maintainer product/security decision for a new administrative write-action gate and default.

## What This Changes

Request to expose Telegram pin and unpin as standalone message actions for agents that maintain pinned status dashboards.

## Best Possible Solution

Add Telegram plugin-owned typed pin and unpin actions that reuse the existing helpers, introduce an explicit secure gate/default, document administrator-permission and notification behavior, and prove the flow with focused tests plus live Telegram evidence.

## Reproduction Assessment

Yes at the source level: current main accepts pin/unpin as generic message action names, but Telegram's plugin descriptor and action map do not expose or dispatch them.

## Solution Assessment

Yes, the right shape is a Telegram plugin-owned action implementation, not a core workaround; the open question is the secure gate/default and notification behavior maintainers want.
