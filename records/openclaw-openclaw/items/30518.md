## Summary

Keep open: current main and the latest release still have lower-level Telegram pin/unpin helpers and pinned-delivery support, but they do not expose standalone Telegram `pin` or `unpin` message actions through discovery or dispatch. The remaining work is real and needs maintainer product/security direction before an implementation PR is queued.

## What This Changes

The issue asks OpenClaw to expose Telegram Bot API pinChatMessage and unpinChatMessage as agent-facing pin/unpin message actions so agents can maintain pinned status dashboards.

## Best Possible Solution

Add Telegram-plugin-owned `pin` and `unpin` message actions with explicit gating, conservative unpin semantics, docs, focused action/runtime tests, and live Telegram proof after maintainer product/security approval.

## Reproduction Assessment

Yes, at source level: current Telegram discovery, action mapping, and runtime aliases omit `pin` and `unpin`, while current main and `v2026.6.10` only provide send-layer helpers and pinned delivery. I did not run a live Telegram probe because there is no standalone action surface to exercise.

## Solution Assessment

Unclear until maintainers choose the privileged action contract. The likely best fix is a Telegram-plugin-owned action surface with a dedicated gate and conservative unpin semantics, not a core workaround or delivery-pin-only behavior.
