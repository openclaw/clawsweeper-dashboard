## Summary

Keep this issue open. Current main still has the shared `pin`/`unpin` action names and Telegram Bot API transport helpers, but the bundled Telegram plugin does not expose manual `pin` or `unpin` through action discovery, adapter mapping, runtime aliases, runtime dispatch, config gates, docs, or contract tests.

## What This Changes

Expose Telegram `pin` and `unpin` message actions through the message tool, backed by Telegram Bot API `pinChatMessage` and `unpinChatMessage`.

## Best Possible Solution

Implement the feature inside the bundled Telegram plugin boundary: advertise and map `pin`/`unpin`, dispatch to the existing Telegram helpers, keep send-time `delivery.pin` separate, document the manual actions, and cover the behavior with focused Telegram action/runtime tests plus message-tool contract coverage.
