Request to expose Telegram message-tool `pin` and `unpin` actions for agents that maintain pinned status messages.

## Best Possible Solution

Add Telegram-plugin-owned `pin` and `unpin` discovery plus runtime dispatch behind a maintainer-approved action gate, reuse the existing helpers, document Bot API permission expectations, and require live Telegram proof.

## Solution Assessment

Yes, if maintainers accept the new write-action surface. The narrow maintainable path is Telegram plugin-local discovery and dispatch that reuses existing helpers and adds a consistent action gate without changing core channel contracts.
