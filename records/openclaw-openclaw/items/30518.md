## Summary

Keep open: current main and v2026.5.20 still expose only Telegram delivery-time pinning plus low-level helpers, not standalone Telegram message-tool pin/unpin actions. The request is valid, but it adds a new Telegram write-action surface and still needs maintainer approval for the public gate/default behavior before implementation.

## What This Changes

Expose Telegram message-tool `pin` and `unpin` actions so agents can pin or unpin existing Telegram messages programmatically.

## Best Possible Solution

Add Telegram-plugin-owned `pin` and `unpin` discovery plus runtime dispatch behind a maintainer-approved action gate, reuse the existing helpers, update docs/tests, and validate with live Telegram proof.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source, docs, latest-release, and dependency checks give high confidence that standalone Telegram `pin` and `unpin` action routing is still missing.

## Solution Assessment

Yes, if maintainers approve the new write-action surface. The narrow maintainable path is plugin-local discovery, dispatch, and gating that reuses existing helpers without changing core channel contracts.
