## Summary

Keep open. Current main still lacks content-aware write/edit dedupe and a general run/session tool-call budget; the existing loop detector is opt-in and pattern-based, while the related #75924 work is narrower consecutive-error detection.

## What This Changes

Request adds core agent safeguards for duplicate file writes, repeated identical operations, and runaway tool-call iteration limits.

## Best Possible Solution

Keep this as the canonical design item for a core agent safeguard that combines run-scoped write/edit content fingerprinting with a documented tool-call budget aligned to existing loop detection.

## Reproduction Assessment

Yes for the missing safeguards: current source and docs show no content-hash write/edit dedupe, no requested safeguard config fields, and max_tool_calls remains ignored. The original 382-file model loop is only partially reproducible without the full transcript.

## Solution Assessment

Unclear. The requested direction is sound, but a same-target existing-file hash check would miss the reported versioned-filename loop, so the maintainable fix should define cross-path fingerprint scope and explicit run-budget semantics first.
