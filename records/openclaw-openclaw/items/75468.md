## Summary

Keep open. Current main still lacks the requested content-aware write/edit dedupe and a general run/session tool-call budget, and the linked consecutive-error work is a narrower adjacent guard rather than a substitute for this design item.

## What This Changes

The issue requests core safeguards for repeated identical file writes and a general run/session tool-call limit to stop autonomous agent loops.

## Best Possible Solution

Keep this as the canonical core safeguard design item, coordinate it with the narrower consecutive-error PR, and define a run-scoped content fingerprint plus explicit tool-call budget contract before implementation.

## Reproduction Assessment

Yes for the missing safeguards: current source and docs show no content-hash write/edit dedupe, no requested safeguard config fields, and max_tool_calls remains ignored. The original 382-file model loop is only partially reproducible without the full transcript.

## Solution Assessment

Unclear. The requested direction is sound, but same-target hash dedupe alone would miss the reported versioned-filename loop, so the maintainable solution should define cross-path fingerprint scope and run-budget semantics first.
