## Summary

Keep open. Current main still lacks the requested content-aware write/edit safeguard and a general run/session tool-call cap; the existing loop detector is opt-in, tool-args based, and does not catch same-content writes under versioned filenames.

## What This Changes

Add core agent safeguards for duplicate file writes and runaway tool execution loops.

## Best Possible Solution

Keep this as the canonical core-agent safeguard design item and define a run-scoped content-fingerprint guard plus documented tool-call budget semantics in the existing loop-detection surface.

## Reproduction Assessment

Yes for the missing safeguards: source and docs show no content-hash write/edit dedupe, no requested safeguard config fields, and max_tool_calls is accepted but ignored. The original model loop is only partially reproducible without the full transcript.

## Solution Assessment

Unclear. The requested direction is sound, but the literal same-path hash comparison would miss the versioned-filename failure mode, so maintainers should first settle cross-path fingerprint scope and run-budget behavior.
