## Summary

Keep open. Current main still lacks content-aware write/edit dedupe and a general run/session tool-call budget; existing loop detection is opt-in and pattern-based, while related #75924 is narrower consecutive-error work.

## What This Changes

Request adds content-hash dedup for file write/edit operations plus session/run-level tool-call limits to stop runaway autonomous agent loops.

## Best Possible Solution

Keep this as the canonical design item for core agent safeguards combining run-scoped write/edit content fingerprinting with a documented tool-call budget wired through the existing loop-detection/config surfaces.

## Reproduction Assessment

Yes for the missing safeguards: current source shows direct write/edit execution, no content-fingerprint guard, and no enforced run/session tool-call budget. The original 382-file model loop is only partially reproducible without the full transcript.

## Solution Assessment

Unclear. The requested direction is useful, but same-target file comparison would miss the reported versioned-filename loop, so the best fix should define cross-path fingerprint scope and run-budget semantics first.
