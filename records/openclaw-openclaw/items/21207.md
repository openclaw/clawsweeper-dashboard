## Summary

Keep open. Current main still lacks the three requested native product surfaces: batched read/write/search tool inputs or gateway aggregation, AI-callable named context block save/load/unload/list tools, and submitted-plan step enforcement. Existing `parallel_tool_calls`, `update_plan`, strict-agentic retries, `/context`, `/compact`, and context-engine lifecycle hooks are adjacent groundwork rather than the requested API and runner behavior, which matches the prior maintainer/Codex comment that already kept this issue open.

## What This Changes

The issue requests native batch tool calls, AI-controlled context block management, and opt-in plan submission with step enforcement for multi-step workflows.

## Best Possible Solution

Keep this open and split it into maintainer-reviewed tracks: a native or gateway-level batch contract for file and web-search tools, a deliberate context-block API if AI-controlled local block storage is desired, and a runner-owned opt-in submitted-plan state machine if strict step gating becomes product policy.
