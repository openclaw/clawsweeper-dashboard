## Summary

Keep open for maintainer product design: current main and v2026.6.5 provide parallel-call transport capability, scalar file/search tools, lifecycle-based context engines, compaction, `update_plan`, and strict-agentic completion checks, but still do not provide the three requested native product surfaces. No canonical replacement issue or PR was found, and the existing discussion explicitly treats those mechanisms as groundwork rather than closure.

## What This Changes

Requests native batched file/search tool inputs, AI-managed named context blocks, and opt-in enforced submitted-plan execution for multi-step agent workflows.

## Best Possible Solution

Split the umbrella into separately approved designs: tool-owner batching with bounded result semantics, context-engine/plugin-owned selective context state using canonical SQLite storage when persistence is needed, and runner-owned plan enforcement with explicit recovery behavior and a justified minimal configuration surface.

## Solution Assessment
