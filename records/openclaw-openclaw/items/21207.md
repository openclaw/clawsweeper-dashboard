## Summary

Keep open. Current main and v2026.6.6 still lack the three requested native contracts: read/write/web-search tools remain single-call schemas, context engines expose lifecycle and compaction hooks rather than AI-callable named block tools, and `update_plan` is advisory progress tracking rather than submitted-plan enforcement.

## What This Changes

The issue requests native batch tool calls, AI-controlled context block save/load/unload/list tools, and opt-in submitted-plan enforcement for multi-step agent workflows.

## Best Possible Solution

Keep this as a product-design umbrella, then split accepted work into separate maintainer-reviewed designs for batch-capable tool schemas, context-engine-managed selective state, and runner-owned plan gating.
