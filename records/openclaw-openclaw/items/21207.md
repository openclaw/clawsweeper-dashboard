## Summary

Keep open because current main and the latest release still provide adjacent mechanisms only: scalar file/search tool inputs, lifecycle-based context engines, provider parallel-tool-call params, strict-agentic retries, and `update_plan`, but not native batched tool schemas, AI-callable context block save/load/unload/list tools, or submitted-plan enforcement.

## What This Changes

Request native batch file/search tool calls, AI-managed named context blocks, and opt-in plan submission/step enforcement for multi-step agent workflows.

## Best Possible Solution

Split this into maintainer-reviewed design tracks for native tool batching, explicit context-block APIs, and runner-owned opt-in plan enforcement, each with schemas, compatibility rules, docs, and focused tests.

Do we have a high-confidence way to reproduce the issue? Not applicable; this is a feature request, and current-main source inspection confirms the requested batch parameters and named context/plan tools are absent.

## Solution Assessment

Is this the best way to solve the issue? No, not as one combined implementation. The maintainable path is to split batching, context block management, and plan enforcement into explicit product/API design tracks before implementation.
