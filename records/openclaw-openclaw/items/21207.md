## Summary

Keep open. Current main still lacks the requested native batch tool inputs, AI-callable named context block tools, and submitted-plan enforcement; existing `update_plan`, strict-agentic retries, and context-engine lifecycle hooks are adjacent groundwork, not the requested product surface.

## What This Changes

Request native/gateway batched tool calls, AI-callable context block save/load/unload/list tools, and opt-in system plan enforcement with settings.

## Best Possible Solution

Split this into maintainer-reviewed tracks for native batching, a deliberate context-block API, and opt-in runner-owned plan enforcement, with schema/docs/tests for any accepted surface.
