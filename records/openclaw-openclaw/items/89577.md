## Summary

Keep open: this is useful Canvas A2UI security hardening, but the current patch is not merge-ready because it updates only the source helper while current main also ships a tracked native CanvasA2UI bundle that still uses the old `Math.random()` fallback.

## What This Changes

The PR changes the Canvas A2UI browser bootstrap so action IDs keep `crypto.randomUUID()`, fall back to 16 bytes from `crypto.getRandomValues()`, and throw when Web Crypto randomness is unavailable.

## Best Possible Solution

Update the source and every shipped/generated A2UI runtime asset from a clear generation path, prove the browser and native-local asset behavior, then let maintainers decide the no-Web-Crypto fail-closed support boundary.

## Reproduction Assessment

Yes. Source inspection shows current main falls back to `Math.random()` in the A2UI source helper and the tracked native CanvasA2UI bundle; the PR body also supplies Chrome/Playwright output for the generated source path.

## Solution Assessment

No. The helper shape is a good narrow fix, but the PR is one-sided against current main because it leaves the tracked native A2UI bundle with the same weak fallback.
