## Summary

Keep open: current main and v2026.6.9 still use the weak Canvas A2UI `Math.random()` fallback, and this PR is a focused security hardening with sufficient browser proof; merge still needs maintainer acceptance of the no-Web-Crypto fail-closed compatibility tradeoff.

## What This Changes

The PR changes the bundled Canvas A2UI host action-id helper to replace the `Math.random()` fallback with `crypto.getRandomValues()` and a no-Web-Crypto throw.

## Best Possible Solution

Land the bundled A2UI host hardening if maintainers accept no-Web-Crypto clients failing closed; otherwise add an explicit supported compatibility path or pause the security tradeoff.

## Reproduction Assessment

Yes. I did not execute a local repro in this read-only review, but current main and v2026.6.9 visibly fall back to `Math.random()` in `#makeActionId()`, and the PR body supplies after-fix Chrome/Playwright live output for the `getRandomValues(16)` path.

## Solution Assessment

Yes for the bundled A2UI host path: Web Crypto randomness with fail-closed behavior is the narrow security fix. It is not a whole-Canvas action-ID hardening, because the generic injected helper remains a separate path.
