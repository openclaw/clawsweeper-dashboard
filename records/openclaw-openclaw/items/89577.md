## Summary

Keep open for maintainer review: the patch appears correct and is not obsolete, but it intentionally changes the Canvas A2UI support floor by failing closed when Web Crypto randomness is unavailable.

## What This Changes

The PR replaces the Canvas A2UI `Math.random()` action ID fallback with Web Crypto `getRandomValues` and throws when no Web Crypto random source exists.

## Best Possible Solution

Land the bundled A2UI host hardening after maintainers accept Web Crypto as the support floor, with a focused fallback-contract test if they want stronger future regression coverage.

## Reproduction Assessment

Yes, from source inspection. Current main falls back to `Math.random` when `crypto.randomUUID` is absent, and the PR body supplies after-fix Chrome/Playwright live output for the `getRandomValues` path.

## Solution Assessment

Yes for the bundled A2UI host path: Web Crypto randomness with fail-closed behavior is the narrow security fix. Maintainers still need to accept the compatibility tradeoff, and the generic injected helper remains a separate existing ID path.
