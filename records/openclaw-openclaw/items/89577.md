## Summary

Keep this PR open: it is a narrow, likely-correct Canvas security hardening, but it intentionally changes fallback behavior for obsolete no-Web-Crypto WebViews, so maintainer acceptance is still required before merge.

## What This Changes

The PR changes Canvas A2UI browser action ID generation to use Web Crypto randomness instead of a Math.random fallback.

## Best Possible Solution

Land this Web Crypto-only A2UI host path if maintainers accept fail-closed behavior for no-Web-Crypto WebViews; track broader Canvas bridge ID hardening separately.

## Reproduction Assessment

Yes. Current main falls back to `Math.random()` in `#makeActionId()`, and the action ID is then sent through the Canvas A2UI bridge path.

## Solution Assessment

Yes for the bundled A2UI host path: Web Crypto randomness with fail-closed behavior is the narrow security fix. It is not whole-Canvas ID hardening because the injected helper remains a separate existing path.
