## Summary

Keep open: the security-hardening direction is sound, but the branch only updates the A2UI source bootstrap while the tracked app-owned A2UI bundle used by native mobile action delivery still contains the old `Math.random()` fallback.

## What This Changes

The PR changes the Canvas A2UI source bootstrap action ID generator to prefer `crypto.randomUUID()`, use a `getRandomValues()` hex fallback, and throw when Web Crypto random values are unavailable.

## Best Possible Solution

Update the canonical source and every tracked app-owned A2UI runtime asset that native mobile loads, add focused fallback/fail-closed coverage, then have maintainers explicitly accept the Web Crypto-only compatibility policy.

## Reproduction Assessment

Yes by source inspection: current main's source bootstrap and the checked-in app-owned bundle both fall back to `Math.random()` when `crypto.randomUUID()` is unavailable. I did not run browser or device proof in this read-only review.

## Solution Assessment

No, not yet: the source helper change is the right direction, but it misses the tracked app-owned A2UI bundle that native mobile uses for action-capable delivery. The maintainable fix is to update that shipped asset or its generation path along with regression coverage.
