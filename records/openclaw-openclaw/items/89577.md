## Summary

Keep open for maintainer review: this PR is still necessary because current main and the latest release still use the `Math.random()` fallback, and the patch looks correct but carries an intentional no-Web-Crypto fail-closed compatibility tradeoff.

## What This Changes

The branch replaces the bundled Canvas A2UI host action ID `Math.random()` fallback with `crypto.randomUUID()`/`crypto.getRandomValues()` and throws when Web Crypto randomness is unavailable.

## Best Possible Solution

Land the bundled A2UI host hardening if maintainers accept the no-Web-Crypto fail-closed behavior, and track broader Canvas helper ID hardening separately if needed.

## Reproduction Assessment

Yes. Source inspection shows current main and `v2026.6.8` still fall back to `Math.random()` in `#makeActionId()`, and the PR body supplies after-fix Chrome/Playwright live output for the `getRandomValues(16)` path.

## Solution Assessment

Yes for the bundled A2UI host path: Web Crypto randomness with fail-closed behavior is the narrow security fix. It is not a complete Canvas bridge-ID hardening because the injected generic helper remains a separate existing path.
