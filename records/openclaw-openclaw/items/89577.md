## Summary

Keep open for maintainer review: the patch is a narrow, source-backed security hardening, but it intentionally changes no-Web-Crypto environments from predictable ID generation to a thrown error before A2UI action delivery.

## What This Changes

The PR changes the Canvas A2UI browser bootstrap action ID generator to keep `crypto.randomUUID()`, use `crypto.getRandomValues()` for the fallback suffix, and throw when Web Crypto randomness is unavailable.

## Best Possible Solution

Land this Web Crypto-only A2UI host path if maintainers accept fail-closed behavior for no-Web-Crypto WebViews; track any broader Canvas bridge ID hardening separately.

## Reproduction Assessment

Yes. Current main visibly falls back to `Math.random()` in `#makeActionId()`, and the PR body supplies after-fix Chrome/Playwright live output for the `getRandomValues(16)` path.

## Solution Assessment

Yes for the bundled A2UI host path: using Web Crypto randomness and failing closed is the narrow maintainable security fix. It is not a complete Canvas bridge-ID hardening because the injected generic helper in `a2ui-shared.ts` remains separate existing behavior.
