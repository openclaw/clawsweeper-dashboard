## Summary

Keep open: the PR is a useful security hardening direction, but it currently hardens only the Canvas A2UI source bootstrap while the checked-in app-bundled A2UI resource on current main still ships the old `Math.random()` fallback on the action-capable native path.

## What This Changes

The PR changes the Canvas A2UI browser bootstrap action ID helper to prefer `crypto.randomUUID()`, fall back to `crypto.getRandomValues()`, and throw when Web Crypto randomness is unavailable.

## Best Possible Solution

Apply the secure action-ID generator to every shipped or action-capable Canvas A2UI bundle, then have maintainers explicitly accept the Web Crypto fail-closed compatibility tradeoff before merge.

## Reproduction Assessment

Yes. Current main source and the checked-in app-shared bundle both visibly use the `Math.random()` fallback, and the PR body supplies after-fix browser live output for the extension source path.

## Solution Assessment

No. The Web Crypto-only helper is the right narrow shape for the edited source file, but the PR is incomplete until the app-bundled A2UI resource or its generation path is updated too.
