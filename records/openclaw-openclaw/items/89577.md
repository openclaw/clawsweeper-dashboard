## Summary

Keep open: this is a narrow, useful security-hardening PR, and current main plus the latest release still use the weak fallback. The remaining decision is whether maintainers accept fail-closed behavior for very old WebViews without Web Crypto.

## What This Changes

The PR changes Canvas A2UI browser bootstrap action ID generation to keep `crypto.randomUUID()`, use a 16-byte `crypto.getRandomValues()` fallback, and throw when Web Crypto randomness is unavailable.

## Best Possible Solution

Land the Web Crypto-only bundled A2UI host change if maintainers accept fail-closed behavior for no-Web-Crypto WebViews, and track broader Canvas helper ID hardening separately if desired.

## Reproduction Assessment

Yes. Current main and `v2026.6.6` visibly fall back to `Math.random()` in `#makeActionId()`, and the PR body supplies after-fix Chrome/Playwright live output for the `getRandomValues(16)` path.

## Solution Assessment

Yes for the bundled A2UI host path: Web Crypto randomness with fail-closed behavior is the narrow maintainable security fix. It is not a complete Canvas bridge-ID hardening because the injected generic helper remains a separate existing path.
