## Summary

Keep open for maintainer review: the patch is a focused security hardening for the bundled Canvas A2UI host, but merging it intentionally changes no-Web-Crypto environments from predictable IDs to fail-closed action delivery.

## What This Changes

The PR replaces the Canvas A2UI host `Math.random()` fallback action ID path with Web Crypto `getRandomValues()` and throws when Web Crypto randomness is unavailable.

## Best Possible Solution

Land the bundled A2UI host hardening if maintainers accept the no-Web-Crypto fail-closed behavior, and track broader Canvas helper ID hardening separately if needed.

## Reproduction Assessment

Yes. Current main and v2026.6.8 still fall back to `Math.random()` in `#makeActionId()`, and the PR body supplies after-fix Chrome/Playwright live output for the `getRandomValues(16)` path.

## Solution Assessment

Yes for the bundled A2UI host path: Web Crypto randomness with fail-closed behavior is the narrow security fix. Maintainers still need to accept the compatibility tradeoff, and the generic injected helper remains a separate existing ID path.
