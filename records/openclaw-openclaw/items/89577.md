## Summary

Keep this PR open for maintainer review: it is a narrow, correct Canvas A2UI security hardening change that is not on current main or the latest release, but merging it requires accepting the fail-closed behavior for very old WebViews without Web Crypto.

## What This Changes

The PR replaces the Canvas A2UI `Math.random()` action-ID fallback with `crypto.randomUUID()` or a 16-byte `crypto.getRandomValues()` hex fallback, and throws when Web Crypto randomness is unavailable.

## Best Possible Solution

Land this hardening if maintainers accept Web Crypto as the Canvas A2UI support floor, and track any broader helper-ID hardening as a separate Canvas follow-up.

## Reproduction Assessment

Yes. Source inspection shows current main falls back to `Math.random()` when `crypto.randomUUID()` is absent, and the PR body supplies after-fix Chrome/Playwright live output for the `getRandomValues(16)` path.

## Solution Assessment

Yes for the bundled A2UI host path. Web Crypto randomness with fail-closed behavior is the narrow security fix, but maintainers still need to accept the legacy-WebView compatibility tradeoff.
