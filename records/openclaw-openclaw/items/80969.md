## Best Possible Solution

Land this focused delete-after-success cleanup with the included regression coverage, while tracking stale-session reachability and future SQLite migration behavior separately.

## Solution Assessment

Yes. Deleting the backup only after `replaceFileAtomic` succeeds preserves the crash-safety window while stopping success-path disk growth; keeping failed atomic replacement behavior out of scope is a reasonable boundary.
