## Summary

Keep open: current main still retains a full `.bak-*` file on every successful session-file repair, and this PR is a focused, source-backed fix with sufficient real filesystem proof and no blocking review findings.

## What This Changes

The PR removes successful session-file repair `.bak-*` backups after atomic replacement, returns `backupPath` only on cleanup failure, and updates tests, transcript hygiene docs, and the changelog.

## Best Possible Solution

Land the focused cleanup after normal maintainer checks, preserve unsuccessful-repair crash-safety backups, and track ended-session reachability separately if it still reproduces.

## Reproduction Assessment

Yes. Source inspection shows current main writes and returns a unique retained `.bak-*` on every successful repair; the PR body also supplies a real filesystem baseline where main accumulates five backups and the patched branch retains zero.

## Solution Assessment

Yes. Deleting the backup only after atomic replace succeeds, while reporting it if cleanup fails, is narrower than TTL/rotation/config work and preserves the failure-path postmortem contract.
