## Summary

Keep open; current main still has the retained-backup behavior and this PR is the active focused fix for the linked session-repair disk-growth bug. The patch matches the implicated code path, has sufficient terminal proof, and I found no blocking correctness or security findings.

## What This Changes

This PR deletes successful transient session-repair `.bak-<pid>-<ts>` backups after atomic replacement, returns `backupPath` only when cleanup fails, and updates focused tests, transcript hygiene docs, and changelog text.

## Best Possible Solution

Land the focused delete-after-success contract with the included regression coverage, keeping broader SQLite migration cleanup separate.

## Reproduction Assessment

Yes. A malformed JSONL tail repaired repeatedly through `repairSessionFileIfNeeded` has a clear current-main source path that leaves one retained `.bak-*` per successful repair; the PR body also supplies a baseline-versus-patched terminal proof.

## Solution Assessment

Yes. Deleting the backup only after `replaceFileAtomic` succeeds preserves the crash-safety window while stopping success-path disk growth; a new cap or config knob would be broader than this bug requires.
