Keep open for normal maintainer review: current main still retains a `.bak-*` file after every successful session repair, while this PR applies a focused delete-after-success fix with regression coverage, sufficient real behavior proof, and no blocking review findings.

## What This Changes

The PR deletes transient session-file repair backups after a successful atomic replace, reports `backupPath` only when cleanup fails, and updates tests, transcript hygiene docs, and the changelog.

## Best Possible Solution

Land the focused cleanup with the included regression coverage, while keeping failed atomic-replace behavior and stale ended-session reachability as separate follow-up work.

## Reproduction Assessment

Yes. Current main has a clear source path: repeated successful calls write and retain a fresh `.bak-*` sibling, and the PR body supplies a real filesystem baseline showing the accumulation.

## Solution Assessment

Yes. Deleting the backup only after `replaceFileAtomic` succeeds preserves the crash-safety window while stopping success-path disk growth; a TTL, rotation policy, or config knob would be broader than this bug requires.
