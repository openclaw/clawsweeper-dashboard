I found no blocking correctness or security findings; the remaining action is maintainer review and normal merge gating.

## What This Changes

The PR deletes transient session-file repair backups after successful atomic replacement, returns `backupPath` only when cleanup fails, and updates tests, transcript hygiene docs, and the changelog.

## Best Possible Solution

Land this focused cleanup with its regression coverage, while tracking any stale ended-session reachability as a separate follow-up if maintainers want that investigated.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction path: repeatedly repair a session file that keeps needing cleanup and each successful call writes a unique retained `.bak-*` sibling; the PR body also supplies real filesystem before/after terminal proof.

## Solution Assessment

Yes. Deleting the backup only after `replaceFileAtomic` succeeds preserves the crash-safety window, and retaining/reporting it when unlink cleanup fails is narrower than adding TTL, rotation, or a config knob.
