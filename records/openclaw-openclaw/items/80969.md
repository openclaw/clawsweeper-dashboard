## What This Changes

This PR deletes successful session-file repair backups after atomic replacement, returns `backupPath` only when cleanup fails, and updates focused tests, transcript hygiene docs, and changelog text.

## Best Possible Solution

Land the focused delete-after-success contract with the included regression coverage, keeping broader stale-session reachability and SQLite migration cleanup separate.

## Reproduction Assessment

Yes, at source level. Current main writes and returns a unique `.bak-*` path after every successful repair, both run-attempt and compaction can re-enter that repair path, and the PR body adds a baseline-versus-patched terminal proof.

## Solution Assessment

Yes. Deleting the backup only after `replaceFileAtomic` succeeds preserves the crash-safety window while stopping success-path disk growth; adding a config knob or broader rotation policy would be more surface than this bug requires.
