## Summary

Keep open: current main and the latest release still return raw MSTeams `file.download.info` `downloadUrl` values, while the existing Graph shares rewrite only covers the `contentUrl` branch. The open linked PR https://github.com/openclaw/openclaw/pull/85845 owns the exact narrow fix, so this issue should remain open until that PR lands or is intentionally replaced.

## What This Changes

MSTeams DM `file.download.info` SharePoint/OneDrive `downloadUrl` attachments should be routed through the existing Graph shares download path instead of being fetched directly.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/85845, or an equivalent narrow patch that reuses the existing Graph shares helper and adds the missing regression test, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection on current main shows the `file.download.info` branch returns the raw `downloadUrl`, while the sibling `contentUrl` branch rewrites SharePoint/OneDrive links through Graph shares; I did not run a live Teams tenant in this read-only review.

## Solution Assessment
