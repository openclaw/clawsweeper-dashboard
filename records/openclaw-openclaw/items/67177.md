## Summary

Keep open: current main and the latest release still return the raw MSTeams `file.download.info` `downloadUrl`, while the open linked PR is the active implementation candidate rather than a merged fix.

## What This Changes

Issue requests that MSTeams `file.download.info` DM attachment `downloadUrl` values be rewritten through the existing Microsoft Graph shares endpoint so inbound files download instead of degrading to placeholders.

## Best Possible Solution

Review and land, or intentionally replace, https://github.com/openclaw/openclaw/pull/85845, then close this issue once main contains the resolver rewrite and regression coverage.

## Reproduction Assessment

Yes, at source level. Current main treats `file.download.info` attachments as downloadable, then `resolveDownloadCandidate()` returns the raw SharePoint/OneDrive `downloadUrl` before the direct download path runs.

## Solution Assessment

Yes. Reusing `tryBuildGraphSharesUrlForSharedLink()` in the missing `file.download.info` branch, with fallback to the original URL for non-shared links, is the narrow maintainable fix and matches the open linked PR.
