## Summary

Keep open: current main and v2026.6.1 still return the raw MSTeams `file.download.info` `downloadUrl`, while the existing Graph shares rewrite only runs for `contentUrl`. The linked open PR https://github.com/openclaw/openclaw/pull/85845 directly owns the narrow fix, so this issue should stay open until that PR lands or is intentionally replaced.

## What This Changes

The issue asks for MSTeams inbound `file.download.info` attachments to rewrite SharePoint/OneDrive `downloadUrl` values through Graph shares so DM uploads download instead of becoming placeholders.

## Best Possible Solution

Land or replace the narrow MSTeams resolver fix that routes eligible `file.download.info` `downloadUrl` values through the existing Graph shares helper with regression coverage, then close this issue as implemented.
