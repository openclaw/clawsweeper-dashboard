## Summary

Keep open. The Feishu multi-bot bug is valid and this PR is pointed at the right owner surface, but the current patch would regress no-space mention-prefixed p2p slash commands and still needs the required changelog entry before merge.

## What This Changes

This PR changes Feishu mention normalization to strip the receiving bot's own mention only when it appears as a leading addressing prefix, and adds regression coverage for preserving inline self-mentions in multi-bot group messages.

## Best Possible Solution

Land a corrected Feishu plugin fix that strips the receiving bot's self-mention only when it is a leading addressing prefix followed by whitespace, slash, or end-of-text, preserves inline self-mentions for multi-bot routing, and documents the user-facing fix in the changelog.

## Reproduction Assessment

Yes. The current-main bug is reproducible from source with a Feishu group text event mentioning BotA and BotB: normalizeMentions removes the receiving bot's own tag before ctx.content reaches the agent. The PR regression is reproducible with a p2p text event like @_bot_1/model because p2p command probing uses ctx.content directly.

## Solution Assessment

No, not as written. The leading-prefix-only direction is the narrow maintainable fix, but the boundary should include slash and the PR needs a changelog entry before it is ready to land.
