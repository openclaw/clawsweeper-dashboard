## Summary

Keep open: the fix direction is useful, but this PR still risks breaking no-space Feishu direct slash commands because its leading self-mention boundary excludes `/`, and it only provides test/CI proof rather than real after-fix behavior evidence.

## What This Changes

The branch changes Feishu mention normalization to strip only a leading receiving-bot mention and adds regression tests for preserving inline self-mentions in multi-bot group content.

## Best Possible Solution

Land a corrected Feishu plugin fix that treats whitespace, slash, or end-of-text as the leading self-mention delimiter, preserves inline self-mentions, and covers both group and direct slash commands.

## Reproduction Assessment

Yes. Source inspection shows current main strips inline receiving-bot self-mentions, and the PR regression is reproducible with a direct Feishu text body like `@_bot_1/model` because the PR would leave `<at>Bot</at>/model` before direct command probing.

## Solution Assessment

No, not as written. Leading-prefix-only stripping is the narrow maintainable direction, but the delimiter should include slash and the contributor still needs after-fix real behavior proof.
