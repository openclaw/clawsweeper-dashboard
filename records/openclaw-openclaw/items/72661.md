## Summary

Keep open. Current main still strips every receiving-bot self-mention in Feishu content, so the linked multi-bot bug is real and not implemented on main, but this PR is not ready because its leading-prefix guard excludes slash-prefixed commands in p2p messages and it lacks the required user-facing changelog entry.

## What This Changes

The PR changes Feishu mention normalization to strip the receiving bot's own mention only as a leading addressing prefix and adds regression tests that preserve inline self-mentions in multi-bot group messages.

## Best Possible Solution

Land a corrected Feishu plugin fix that strips the receiving bot's self-mention only when it is a leading addressing prefix followed by whitespace, slash, or end-of-text, preserves inline self-mentions for multi-bot routing, covers both group and p2p command forms, and records the user-facing fix in CHANGELOG.md.

## Reproduction Assessment

Yes. The current-main bug is reproducible from source with a Feishu group text event mentioning BotA and BotB: normalizeMentions removes the receiving bot's own tag before ctx.content reaches the agent. The PR regression is reproducible from the diff with a p2p text event like @_bot_1/model, because p2p command probing uses ctx.content without the group normalizer.

## Solution Assessment

No, not as written. The leading-prefix-only direction is the narrow maintainable fix, but the boundary should include slash and the PR needs changelog and regression coverage before merge.
