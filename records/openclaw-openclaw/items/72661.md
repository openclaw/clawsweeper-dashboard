## Summary

Keep open. The PR targets a valid Feishu bug that current main still has, but the branch is not merge-ready because its leading-mention boundary misses compact direct slash commands and the external contributor has only provided tests/typecheck rather than real Feishu proof.

## What This Changes

The branch changes Feishu mention normalization to strip only a leading receiving-bot mention and adds regression tests that preserve inline self-mentions.

## Best Possible Solution

Land one canonical Feishu plugin fix that preserves semantic inline self-mentions while keeping leading mention-prefixed slash commands command-clean, with regression coverage and redacted real Feishu proof.

## Reproduction Assessment

Yes, at source level. Current main passes `botOpenId` into `normalizeMentions`, which globally removes the receiving bot mention before `buildFeishuAgentBody` sends `ctx.content` to the agent; I did not run a live Feishu tenant reproduction in this read-only review.

## Solution Assessment

No, not as written. Leading-prefix-only stripping is the right direction, but the boundary should also include slash for compact commands and the contributor still needs after-fix real behavior proof.
