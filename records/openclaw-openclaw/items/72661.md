## Summary

Keep this PR open. Current main still strips every receiving-bot self-mention, so the underlying Feishu multi-bot bug remains real, but the PR branch still breaks an existing no-space mention-prefixed slash-command shape and is missing the required user-facing changelog entry.

## What This Changes

The PR changes Feishu mention normalization so the receiving bot's own mention is stripped only as a leading addressing prefix, and adds regression tests for preserving inline self-mentions in multi-bot group messages.

## Best Possible Solution

Land a corrected version that strips the receiving bot's self-mention only when it is a leading addressing prefix followed by whitespace, slash, or end-of-text, while preserving inline self-mentions for multi-bot routing and documenting the user-facing fix in the changelog.

## Reproduction Assessment

Yes. The current-main bug is reproducible from source by parsing a group Feishu text event with BotA and BotB both mentioned; normalizeMentions removes the receiving bot's own tag before the model sees the content. The PR regression is reproducible with a p2p text event like @_bot_1/model because direct command probing uses ctx.content without group mention-probe normalization.

## Solution Assessment

No, not as written. The leading-prefix-only direction is the narrow maintainable fix, but the boundary should include slash and the PR needs a changelog entry before landing.
