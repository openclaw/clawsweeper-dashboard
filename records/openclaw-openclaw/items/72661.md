## Summary

Keep this PR open. The PR targets a real Feishu multi-bot bug that is still present on current main, and the existing discussion correctly identifies the useful leading-prefix direction, but the current diff still regresses no-space mention-prefixed direct commands and lacks the required user-facing changelog entry.

## What This Changes

The PR changes Feishu mention normalization so the receiving bot's own mention is stripped only as a leading addressing prefix, and adds regression tests for preserving inline self-mentions in multi-bot group messages.

## Best Possible Solution

Land a corrected Feishu plugin fix that strips the receiving bot's self-mention only when it is a leading addressing prefix followed by whitespace, slash, or end-of-text, while preserving inline self-mentions for multi-bot routing and documenting the user-facing fix in the changelog.

## Reproduction Assessment

Yes. The current-main bug is reproducible from source by parsing a group Feishu text event that mentions BotA and BotB: normalizeMentions removes the receiving bot's own tag before the agent sees ctx.content. The PR regression is reproducible with a p2p text event like @_bot_1/model because direct command probing uses ctx.content without the group mention-probe normalizer.

## Solution Assessment

No, not as written. The leading-prefix-only direction is the narrow maintainable fix, but the boundary should include slash and the PR needs a changelog entry before it is ready to land.
