## Summary

Keep open. The PR targets a real current-main Feishu bug and the leading-prefix approach is directionally better than preserving every self-mention, but the current diff still regresses no-space mention-prefixed DM slash commands and is missing the required user-facing changelog entry.

## What This Changes

The PR changes Feishu mention normalization to strip the receiving bot's self-mention only as a leading address prefix and adds regression tests for preserving inline self-mentions in multi-bot messages.

## Best Possible Solution

Land a corrected Feishu plugin fix that strips the receiving bot's self-mention only when it is a leading prefix followed by whitespace, slash, or end-of-text, preserves inline self-mentions for multi-bot routing, keeps the #35994 slash-command behavior covered, and adds the required changelog entry.

## Reproduction Assessment

Yes. The current-main bug is reproducible from source with a Feishu group text event that mentions BotA and BotB together, because normalizeMentions removes the receiving bot's own tag anywhere in the body. The PR regression is reproducible with a p2p text event like @_bot_1/model, where the proposed boundary leaves a leading <at> tag before /model and direct-message command probing no longer sees a slash command.

## Solution Assessment

No, not as written. The leading-prefix-only direction is the narrow maintainable solution, but the boundary needs to include slash and the PR needs the user-facing changelog entry before it is ready to land.
