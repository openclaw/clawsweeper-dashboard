## Summary

Keep open. The PR targets a real Feishu multi-bot mention bug, but the current diff regresses compact direct mention-prefixed commands and the external contributor has not posted inspectable after-fix Feishu behavior proof.

## What This Changes

The PR changes Feishu mention normalization to strip only a leading receiving-bot mention and adds regression tests for preserving inline self-mentions in multi-bot group content.

## Best Possible Solution

Land one corrected Feishu plugin fix that preserves inline self-mentions while treating whitespace, slash, or end-of-text as the leading self-mention delimiter, with regression coverage and real Feishu proof.

## Reproduction Assessment

Yes. Source inspection shows current main strips inline receiving-bot self-mentions, and the PR regression follows from a direct Feishu body like @_bot_1/model leaving <at>Bot</at>/model before direct command probing.

## Solution Assessment

No, not as written. Leading-prefix-only stripping is the right direction, but the delimiter should include slash and the contributor still needs after-fix real behavior proof.
