## Summary

Keep open. The Feishu multi-bot bug is still real on current main, but this PR as written has a blocking compact direct slash-command regression and still lacks inspectable after-fix Feishu behavior proof.

## What This Changes

The PR changes Feishu mention normalization to strip only a leading receiving-bot mention and adds tests for preserving inline self-mentions.

## Best Possible Solution

Land one corrected Feishu plugin fix that preserves inline self-mentions while keeping leading whitespace, slash, and end-of-text self-mention prefixes command-clean, with direct and group regression coverage plus real Feishu proof.

## Reproduction Assessment

Yes, source-level. Current main strips inline receiving-bot self-mentions, and this PR’s regression follows from a direct Feishu body like @_bot_1/model leaving <at>Bot</at>/model before direct command probing.

## Solution Assessment

No, not as written. Leading-prefix-only stripping is the right direction, but the delimiter should include slash and the contributor still needs after-fix real behavior proof.
