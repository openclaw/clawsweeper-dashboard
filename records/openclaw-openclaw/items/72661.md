## Summary

Keep open: current main still strips every receiving-bot self-mention, and this PR is the right narrow direction but still regresses no-space mention-prefixed DM slash commands and lacks the required user-facing changelog entry.

## What This Changes

The PR changes Feishu mention normalization to strip only a leading receiving-bot self-mention and adds regression tests that preserve inline self-mentions in multi-bot messages.

## Best Possible Solution

Land a corrected Feishu plugin fix that treats whitespace, slash, or end-of-text as valid delimiters after the leading self-mention, preserves inline self-mentions for multi-bot routing, adds p2p no-space command coverage, and records the user-facing fix in CHANGELOG.md.

## Reproduction Assessment

Yes. The current-main bug is reproducible at the parseFeishuMessageEvent/normalizeMentions level with a group message such as `Hey @_bot_a @_bot_b please coordinate`, and the PR regression is reproducible with a p2p message such as `@_bot_1/model`.

## Solution Assessment

No, not as written. The leading-prefix-only approach is the narrow maintainable fix, but the delimiter must include `/` and the PR needs the required changelog entry before it is ready to land.
