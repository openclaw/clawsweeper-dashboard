## Summary

Keep open: the provider-hook bug is still source-reproducible on current main, but this PR is not mergeable as-is because it edits the old `provider-runtime.ts` helper body while current main uses `provider-hook-runtime.ts`, and it still lacks after-fix real behavior proof.

## What This Changes

The PR changes provider stream wrapping so a primary provider and matching `hookAliases` providers can chain `wrapStreamFn`, with regression coverage that skips unrelated providers.

## Best Possible Solution

Port wrapper composition into `src/plugins/provider-hook-runtime.ts`, preserve provider matching, stable dedupe, and `runtimeHandle` semantics, keep focused regression coverage, and collect redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main shows the active helper invokes one provider wrapper while `extra-params.ts` calls the helper once, so a primary provider plus matching `hookAliases` wrapper cannot compose there.

## Solution Assessment

No. The intended composition is the right narrow fix, but this branch edits the old helper body instead of the active `provider-hook-runtime.ts` helper used by current main.
