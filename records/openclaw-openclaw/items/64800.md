## What This Changes

The PR changes `wrapProviderStreamFn` in `src/plugins/provider-runtime.ts` to chain primary and matching `hookAliases` provider wrappers and adds a regression test that skips unrelated providers.

## Best Possible Solution

Port the composition logic into `src/plugins/provider-hook-runtime.ts`, preserve provider matching, stable dedupe, and runtime-handle semantics, keep the focused regression test, and attach redacted real runtime proof.

## Reproduction Assessment

Yes, source-reproducible: register a primary provider with `wrapStreamFn` plus a matching `hookAliases` provider with its own wrapper. Current main's active helper resolves only the runtime owner wrapper and never iterates the alias provider.

## Solution Assessment

No. The requested behavior is the narrow maintainable fix, but this branch changes a stale `provider-runtime.ts` body instead of the active `provider-hook-runtime.ts` helper used on current main.
