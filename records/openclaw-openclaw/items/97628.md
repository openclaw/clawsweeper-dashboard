## Summary

Keep open: current main still buffers Google OAuth helper responses with `arrayBuffer()`, and this PR is a narrow, proof-backed helper-boundary cap, but maintainers should explicitly accept the new oversized-response auth failure mode before merge.

## What This Changes

The PR replaces Google OAuth `fetchWithTimeout`'s unbounded `arrayBuffer()` read with a 16 MiB `readResponseWithLimit` cap and adds mocked plus loopback tests for oversized and normal responses.

## Best Possible Solution

Land the helper-boundary cap if maintainers accept the oversized-response failure mode, while keeping https://github.com/openclaw/openclaw/pull/97587 as the separate parser-side hardening track.

## Reproduction Assessment

Yes. Source inspection of current main shows `fetchWithTimeout` drains `response.arrayBuffer()` before any caller-side cap can run, and the OAuth project/token paths all use that helper.

## Solution Assessment

Yes, with maintainer acceptance of the failure mode. The shared fetch helper is the right owner boundary for the network-read cap, while https://github.com/openclaw/openclaw/pull/97587 remains the separate parser-cap layer.
