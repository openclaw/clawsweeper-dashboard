## Summary

## What This Changes

The PR replaces Google OAuth `fetchWithTimeout`'s unbounded `response.arrayBuffer()` read with a 16 MiB `readResponseWithLimit` cap and adds mocked plus loopback tests.

## Best Possible Solution

Land the helper-boundary cap after repairing the test defects and explicitly accepting the oversized-response fail-closed behavior; keep https://github.com/openclaw/openclaw/pull/97587 as the separate parser-cap track.

## Reproduction Assessment

Yes. Source inspection of current `main` shows `fetchWithTimeout` drains `response.arrayBuffer()` before any caller-side cap can run, and the OAuth project/token paths all call that helper.

## Solution Assessment

Mostly yes. The shared fetch helper is the right owner boundary for the network-read cap, but the branch needs the narrow test repairs before it is the best mergeable version.
