Full review comments:

- **[P1] Use the service-worker Client.postMessage signature:** `ui/public/sw.js:48`
  - body: `Client.postMessage` from a service worker accepts the message plus a transfer list or options object, not a Window-style target origin. Passing `"*"` here can throw during activation before stale tabs receive `sw-updated`.
  - confidence: 0.95
- **[P1] Await the activation broadcast after claiming clients:** `ui/public/sw.js:46-50`
  - body: The new `matchAll`/`postMessage` work is not part of `event.waitUntil` and can run before `clients.claim()` resolves. Since `includeUncontrolled` defaults to false, the stale windows this fix is meant to reload can be missed.
  - confidence: 0.9
