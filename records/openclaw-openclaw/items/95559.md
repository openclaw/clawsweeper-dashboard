## Summary

Keep open: the PR targets a real linked Control UI lockout path, but it is not merge-ready because the service-worker Client.postMessage call uses a Window-style targetOrigin argument, the broadcast is outside the activation lifetime, and no real stale-browser upgrade proof is provided.

## What This Changes

The PR adds a service-worker activation broadcast and a Control UI listener that reloads the page when a `sw-updated` message is received.

## Best Possible Solution

Use the valid Client.postMessage API inside the awaited activation flow after claim/cache cleanup, guard notification to real prior-version upgrades, add focused coverage, and prove a stale token-auth browser profile reloads once.

## Reproduction Assessment

Yes at source level: stale Control UI protocol ranges are rejected by the gateway and the linked issue provides concrete cloud symptoms. I did not establish a live Zeabur/Chrome stale-service-worker reproduction in this read-only review.

## Solution Assessment

No. Auto-reload is a plausible direction, but this implementation is not the best fix until the Client.postMessage API use, activation sequencing, upgrade guard, tests, and real browser proof are addressed.

## Review Findings

Overall confidence: 0.92

Full review comments:

- **[P1] Use the Client.postMessage API:** `ui/public/sw.js:48`
  - body: In this service worker, `client` is a `Client`, not a `Window`; its `postMessage` overload accepts a transfer list or structured-serialize options, not a `targetOrigin`. Passing `"*"` can make the update notification fail, so the stale tab may never reload.
  - confidence: 0.96
- **[P1] Chain the update broadcast into activation:** `ui/public/sw.js:46-50`
  - body: The new `matchAll()` is launched after `event.waitUntil(...)`, so activation can finish or the worker can be stopped before clients are notified. Move claim, cache cleanup, and notification into one awaited activation path, and guard it so fresh installs or same-version activations do not reload unnecessarily.
  - confidence: 0.9
