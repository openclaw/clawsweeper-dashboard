  - body: `Client.postMessage` accepts a message plus transfer/options, not a Window.postMessage-style target-origin string. Passing `"*"` here can throw or fail overload conversion when a client is found, so the stale tab may never receive the update notification.
  - confidence: 0.95

  - body: The new `matchAll().then(...)` promise is started outside `event.waitUntil()` and `matchAll` omits `includeUncontrolled`, so it races `clients.claim()` and can miss the old tab this fix is meant to reload.
  - confidence: 0.9
