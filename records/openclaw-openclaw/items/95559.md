## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use the service-worker Client.postMessage signature:** `ui/public/sw.js:48`
  - body: `Client.postMessage` in a service worker accepts the message plus an optional transfer list or options object, not a Window-style target origin. Passing `"*"` here can fail during activation before stale tabs receive `sw-updated`, so use `client.postMessage({ type: "sw-updated", version: CACHE_VERSION })` or valid transfer/options instead. ([developer.mozilla.org](https://developer.mozilla.org/en-US/docs/Web/API/Client/postMessage))
  - confidence: 0.95
- **[P1] Sequence the activation broadcast inside waitUntil:** `ui/public/sw.js:46-50`
  - body: The new `matchAll`/`postMessage` work is not part of `event.waitUntil`, and `matchAll` without `includeUncontrolled` only returns controlled clients by default. That can let activation finish before the broadcast completes or miss the stale windows this fix is meant to reload. ([developer.mozilla.org](https://developer.mozilla.org/en-US/docs/Web/API/ExtendableEvent/waitUntil))
  - confidence: 0.91
