## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use the service-worker Client.postMessage signature:** `ui/public/sw.js:48`
  - body: `Client.postMessage` from a service worker accepts the message plus a transfer list or options object, not a Window-style target origin. Passing `"*"` can throw before the `sw-updated` notification is delivered, leaving stale tabs stuck after activation.
  - confidence: 0.95
- **[P1] Await the update broadcast during activation:** `ui/public/sw.js:46-50`
  - body: The new `matchAll()`/`postMessage()` work runs outside `event.waitUntil()` and starts before `clients.claim()` resolves. Since `includeUncontrolled` defaults false, activation can finish or miss the stale windows this fix is meant to reload.
  - confidence: 0.9
