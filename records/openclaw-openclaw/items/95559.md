Full review comments:

- **[P1] Use the service-worker Client.postMessage signature:** `ui/public/sw.js:48`
  - body: `Client.postMessage` in a service worker accepts the message plus an optional transfer list or structured-serialize options object, not a Window-style target origin. Passing `"*"` here can fail during activation before stale tabs receive `sw-updated`, so use the one-argument call or a valid options object.
  - confidence: 0.95
- **[P1] Keep the broadcast inside the activation promise:** `ui/public/sw.js:46-50`
  - body: The new `matchAll`/`postMessage` work is outside `event.waitUntil` and races `clients.claim()`. Because `includeUncontrolled` defaults to false, the stale windows this fix is meant to reload can be missed or the worker can be terminated before the broadcast completes.
  - confidence: 0.91
