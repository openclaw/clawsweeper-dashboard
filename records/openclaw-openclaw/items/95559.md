## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use the service-worker Client.postMessage signature:** `ui/public/sw.js:48`
  - body: `Client.postMessage` from a service worker accepts the message plus a transfer list or options object, not a Window-style target origin. Passing `"*"` here can make the activation broadcast fail before stale tabs receive `sw-updated`.
  - confidence: 0.95
- **[P1] Await the activation broadcast after claiming clients:** `ui/public/sw.js:46-50`
  - body: The new `matchAll().then(...)` work runs outside `event.waitUntil` and is not sequenced after `clients.claim()`. Because `includeUncontrolled` defaults false, activation can finish or skip the stale windows this fix is intended to reload.
  - confidence: 0.9
