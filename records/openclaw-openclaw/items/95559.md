## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use the service-worker Client.postMessage signature:** `ui/public/sw.js:48`
  - body: `Client.postMessage` in a service worker does not take a target-origin string; the second argument is a transfer list or options object. Passing `"*"` here is not the Window `postMessage` API and can prevent the update notification from reaching clients.
  - confidence: 0.95
- **[P1] Await the update broadcast during activation:** `ui/public/sw.js:46-50`
  - body: The new `matchAll()` promise runs outside `event.waitUntil()` and races the existing `clients.claim()`. Because `matchAll()` defaults to controlled clients only, this can miss the stale tabs that need the reload and let activation finish without delivering the message.
  - confidence: 0.9
