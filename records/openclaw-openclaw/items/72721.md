## Summary

Keep open. Current main and v2026.6.8 still route Feishu bot identity probes through the Lark SDK/Axios HTTP path; recent code bounds and softens the startup timeout, but it does not prove the Alibaba Cloud ECS / Node.js 24 timeout is fixed. ## What This Changes

The issue reports Feishu websocket-mode gateway startup/status timing out on the Lark SDK Axios request to `open.feishu.cn` while native Node HTTPS succeeds in the same environment.

## Best Possible Solution

Keep this as the canonical Feishu SDK timeout tracker and land a focused Feishu-plugin fix that makes SDK HTTP egress/startup behavior reliable without weakening proxy policy, with targeted tests and redacted real Feishu proof.

## Reproduction Assessment
