## Summary

Keep open. Current main still sends the reported Feishu bot ping through the Lark SDK axios-backed HTTP instance, WebSocket mode remains the documented default, and the merged code only converts probe hangs into bounded/non-fatal timeout handling rather than proving the Alibaba Cloud ECS / Node.js 24 axios transport hang is fixed; related PR #67466 is still only an open mitigation candidate. https://github.com/openclaw/openclaw/issues/72721

## What This Changes

The issue requests a Feishu plugin fix for Gateway WebSocket mode where the Lark SDK axios transport times out on the bot ping from Alibaba Cloud ECS with Node.js 24 while native HTTPS succeeds.

## Best Possible Solution

Fix this inside the Feishu plugin with a deterministic transport and timeout policy for startup identity probing, setup/status probing, and SDK HTTP calls while keeping WebSocket as a supported default and webhook mode as an operational workaround.

## Reproduction Assessment

Unclear. Not applicable for a patch on this issue.
