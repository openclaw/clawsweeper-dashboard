## Summary

Keep open. Current main still routes the reported Feishu ping and startup/status probe paths through the Lark SDK Axios-backed HTTP instance, and the related startup-latency PR remains only an open, unmerged implementation candidate.

## What This Changes

The issue reports Feishu websocket-mode gateway startup timing out in the Lark SDK Axios-backed `openclaw_bot/ping` call on Alibaba Cloud ECS with Node.js 24.

## Best Possible Solution

Keep the fix plugin-local: bound and degrade Feishu startup/status probes first, then bypass or replace the SDK Axios path for affected Feishu API calls only if live or deterministic dependency proof confirms that transport is the root cause.

## Reproduction Assessment

No. Source inspection confirms current main still executes the reported SDK-backed ping/probe path, but I did not live-reproduce the Alibaba Cloud ECS / Node.js 24 Axios hang with Feishu credentials.

## Solution Assessment

Not applicable for a patch on this issue. The narrow maintainable direction is to finish or replace the Feishu-local startup/status timeout work first, then change the SDK transport only with stronger proof.
