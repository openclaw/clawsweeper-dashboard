## Summary

Keep open. Current main still sends the reported Feishu `openclaw_bot/ping` probe through the Lark SDK Axios-backed HTTP path, WebSocket remains the documented default, and the open Feishu startup-latency PR is not merged.

## What This Changes

The issue reports Feishu websocket-mode gateway startup hanging until the Lark SDK Axios call to `/open-apis/bot/v1/openclaw_bot/ping` times out on Alibaba Cloud ECS with Node.js 24.

## Best Possible Solution

Keep the fix plugin-local: bound and degrade Feishu startup/status probes, then bypass or replace the SDK Axios path for affected Feishu API calls only if live or deterministic dependency proof confirms that transport is the root cause.

## Reproduction Assessment

Unclear. The exact Alibaba Cloud ECS / Node.js 24 Axios hang was not live-reproduced here, but current source still exercises the same SDK-backed ping endpoint and startup/status probe surfaces.

## Solution Assessment
