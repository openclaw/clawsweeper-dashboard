## Summary

Keep open. Current main still routes the reported Feishu bot ping through the Lark SDK axios-backed HTTP instance, websocket remains the documented default, and the related Feishu startup/status timeout PR is still open rather than merged as a fix.

## What This Changes

The issue reports Feishu websocket-mode gateway startup hanging until the Lark SDK axios call to `/open-apis/bot/v1/openclaw_bot/ping` times out on Alibaba Cloud ECS / Node.js 24 while native HTTPS succeeds.

## Best Possible Solution

Keep the fix plugin-local: bound and degrade Feishu startup/status probes while preserving monitor lifecycle, then bypass or replace the SDK axios path for affected Feishu API calls only with stronger live or deterministic dependency proof.

## Reproduction Assessment

No high-confidence live reproduction was established for the Alibaba Cloud ECS / Node.js 24 timeout. The current source still exercises the same SDK-backed ping endpoint and startup/status probe surfaces, so the report remains source-reproducible but environment-dependent.

## Solution Assessment

Not applicable for a patch on this issue. The narrow maintainable direction is to review or replace the Feishu-local startup/status timeout work in #67466 before opening another repair lane, then address the SDK transport only with stronger proof.
