## Summary

Keep open. Current main still routes the reported Feishu websocket/status probe through the SDK-backed `openclaw_bot/ping` path, and websocket mode remains documented as the default supported Feishu transport. The related Feishu timeout/startup PR is open and unmerged, so this issue remains useful canonical field evidence for the Alibaba Cloud ECS / Node 24 Axios timeout shape.

## What This Changes

The issue reports Feishu websocket mode timing out in the SDK-backed `openclaw_bot/ping` request from the gateway on Alibaba Cloud ECS with Node.js 24, while webhook mode and native Node HTTPS remain usable.

## Best Possible Solution

Fix the Feishu plugin locally with a consistent transport and timeout policy for startup identity probing, setup/status probing, and SDK HTTP calls, while preserving websocket mode as a supported default and webhook mode as an operational workaround. Review, repair, or replace #67466 only after the fix is lifecycle-safe and covered by a deterministic SDK transport fixture or redacted live Feishu proof.
