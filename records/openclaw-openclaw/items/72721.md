## Summary

Keep open. Current main still routes the Feishu startup/status probe through the Lark SDK HTTP path and the reported `openclaw_bot/ping` endpoint, while existing timeout handling only bounds or degrades the call instead of proving the Alibaba Cloud ECS / Node.js 24 axios hang is fixed.

## What This Changes

The issue reports Feishu websocket gateway startup calls timing out through `@larksuiteoapi/node-sdk` axios on Alibaba Cloud ECS with Node.js 24 while native HTTPS and webhook mode work.

## Best Possible Solution

Keep the fix plugin-local: bound and degrade Feishu startup/status probes, then bypass or replace the SDK axios path for affected Feishu API calls only if live or dependency proof confirms that transport is the root cause.

## Reproduction Assessment

No high-confidence live reproduction was established on current main. The field logs and current source identify the same SDK-backed ping path, but this review did not run the Alibaba Cloud ECS / Node.js 24 / Feishu-credential environment.
