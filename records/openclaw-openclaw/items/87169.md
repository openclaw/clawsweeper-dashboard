## What This Changes

The PR adds `channels.msteams.graphTenantId` to Teams config schema/types/docs/metadata and changes `resolveGraphToken` to use it for Graph token acquisition.

## Best Possible Solution

Land one canonical Teams Graph-tenant implementation that routes every Teams Graph caller through a shared tenant-aware token-provider path, keeps bot tokens on `tenantId`, documents the config surface, and ships with cross-tenant proof.

## Reproduction Assessment

## Solution Assessment

No. The safer shape is to thread `graphTenantId` through the shared Teams token-provider path, using the upstream `getAppGraphToken(tenantId?)` contract, rather than adding a one-off credential clone in only one helper.
