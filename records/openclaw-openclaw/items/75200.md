## Summary

Keep this PR open: it still has blocking runtime regressions in the proposed eager Active Memory path, so it needs an author update and maintainer review before merge.

## What This Changes

The branch moves Active Memory recall to an eager `agent_turn_prepare` path, adds a memory-only embedded run mode, filters tool construction by runtime `toolsAllow`, adds QMD timeout/fallback instrumentation, and updates related tests.

## Best Possible Solution

Preserve the narrow tool-construction and QMD timeout improvements, but run eager Active Memory with real prompt history and share the normal embedded provider/transcript setup for memory-only runs.

## Reproduction Assessment

Yes. The linked bug has direct-chat steps and timeout logs, and the PR regressions are reproducible by source inspection of the PR head: eager hooks receive an empty message list, and memory-only attempts bypass the shared setup path.

## Solution Assessment

No. The direction is useful, but the implementation should not trade away recent-context query behavior or provider compatibility; the safer fix is to factor shared setup and only optimize the tool surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Pass real history to eager turn-prepare hooks:** `src/agents/pi-embedded-runner/run.ts:692`
  - body: Active Memory is now registered as an eager `agent_turn_prepare` hook, but the eager call is started with `messages: []`. That makes the default `recent` query mode behave like `message` mode and also breaks `full` mode because the hook no longer receives the conversation tail it builds the memory query from.
  - confidence: 0.9
- **[P2] Reuse normal embedded setup for memory-only runs:** `src/agents/pi-embedded-runner/run/attempt.ts:1191-1200`
  - body: The new `memory-only` branch returns before the normal embedded attempt setup applies provider extra params and transcript/tool-call compatibility wrappers. Active Memory can inherit the current session model, so providers that rely on those wrappers can regress only for these recall runs; keep the tool filtering but share the setup before prompting.
  - confidence: 0.86
