## Summary

Keep open. Current main and the latest release still route task registry restore through the shared SQLite state database without a corruption quarantine/recreate path, and the open linked fix PR is the active implementation candidate.

## What This Changes

The issue reports that gateway startup cannot recover from a corrupted task registry SQLite database, originally `tasks/runs.sqlite`, causing durable task restore failures.

## Best Possible Solution

Review and land a current-main-compatible corruption recovery path through https://github.com/openclaw/openclaw/pull/86744, or replace it with an equivalent narrow PR if that branch stays dirty.

## Reproduction Assessment

Yes, there is a source-level reproduction path: corrupt the current shared SQLite state database used by task tables and call `loadTaskRegistryStateFromSqlite()`, which opens the database and rethrows SQLite/schema errors. I did not execute the repro because this cleanup review is read-only.

## Solution Assessment

Yes, a narrow quarantine/recreate recovery is the maintainable direction, but it must be adapted to the current shared state database layout. The issue should remain open until the linked fix PR merges or is replaced.
