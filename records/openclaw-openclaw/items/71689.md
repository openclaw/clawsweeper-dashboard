## Summary

Keep open. Current main still opens and restores `tasks/runs.sqlite` directly and only logs restore failures, while the active fix is still an open draft PR at https://github.com/openclaw/openclaw/pull/77196.

## What This Changes

The issue asks OpenClaw to recover gracefully when the durable task registry SQLite database at `tasks/runs.sqlite` is malformed or corrupt.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77196, or an equivalent narrow fix that preserves corrupt DB sidecars, recreates a clean task store, and leaves salvage/root-cause prevention to deliberate follow-up work.

## Solution Assessment

Yes for the availability-first direction. Quarantine-and-recreate is the narrow maintainable fix for startup availability; automatic `.recover`, payload caps, or root-cause prevention should be separate maintainer-scoped follow-ups.
