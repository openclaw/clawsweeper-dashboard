Keep open. Current main and the latest release still route task registry restore through the shared SQLite state database without a malformed-image quarantine/recreate path, and the open linked fix PR is the active implementation candidate.

## What This Changes

## Best Possible Solution

Review and either land the open shared-state recovery PR or replace it with an equivalent current-main-compatible fix that preserves corrupted SQLite files before starting with fresh state.

## Solution Assessment

Yes at the boundary, but the policy needs maintainer acceptance. The durable fix belongs in the canonical shared state database opener rather than reviving the old `tasks/runs.sqlite` sidecar path.
