## Summary

Keep open. Current main still has the Dockerfile curl-piped Bun installer, and this PR is the right hardening direction, but it needs a small update before merge because main now uses Bun 1.3.13 while the PR pins 1.3.9.

## What This Changes

The PR updates the root Dockerfile to copy Bun from a digest-pinned `oven/bun` source stage and expands Docker image digest tests to cover ARG-backed `FROM` stages.

## Best Possible Solution

Land a reviewed Docker hardening change that removes the remote Bun installer, pins the Bun source image to the current CI Bun version by digest, keeps the digest test aligned with Docker ARG scope, includes the required changelog entry, and has Docker build proof.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the reported condition at `Dockerfile:47`, where the build still executes `https://bun.sh/install` through `bash`.

## Solution Assessment

Not as-is. The Dockerfile direction is the narrow maintainable fix, but the branch should be rebased or updated to Bun 1.3.13 with a matching digest and a tighter ARG-scope test before merge.
