## Summary

Keep open. Current main still runs the Bun installer through `curl | bash`, while this unmerged PR removes that path with a digest-pinned Bun source image and directly closes the linked Docker supply-chain report. The direction is sound, but the PR still needs maintainer/security review plus small nonblocking cleanup around changelog coverage and ARG-scope modeling in the new digest test.

## What This Changes

The PR changes the root Dockerfile to copy Bun 1.3.9 from a digest-pinned `oven/bun` stage instead of executing `bun.sh/install`, and expands Docker image digest tests to cover all ARG-backed `FROM` stages.

## Best Possible Solution

Land a reviewed Docker hardening change that keeps the Bun source image digest-pinned, records the user-facing Docker fix in the changelog, models Docker ARG scope accurately in the digest test, and closes #74356 through the merge after targeted tests and Docker build proof pass.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the reported issue: `Dockerfile:47` still executes `https://bun.sh/install` through `bash` during Docker builds.

## Solution Assessment

Yes, with small cleanup. Copying Bun from a digest-pinned official image stage is the narrow maintainable fix for the supply-chain issue; the test helper should model Docker ARG scope and the PR should add the required release-note entry.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.84

Full review comments:

- **[P3] Limit ARG defaults to global Dockerfile scope:** `src/docker-image-digests.test.ts:36-45`
  - body: Docker only lets `FROM` use ARGs declared before the first `FROM`, but this helper now collects every ARG in every stage. A future stage-local ARG could make the digest assertion pass even though Docker would not resolve that later `FROM` from the same default. Stop collecting at the first `FROM`, or model Docker ARG scope explicitly.
