## Summary

Keep open. Current main still has the Dockerfile `curl | bash` Bun installer, while this PR is a valid unmerged hardening patch with one low-priority test-helper issue and remaining Docker build proof to complete.

## What This Changes

The PR removes the root Dockerfile Bun curl-pipe installer, copies Bun 1.3.13 from a digest-pinned `oven/bun` source stage, adds a changelog entry, and extends Docker digest tests for ARG-backed `FROM` stages.

## Best Possible Solution

Land the Docker hardening change after the ARG-scope test note is addressed or accepted and a root Docker image build/install-smoke confirms the copied Bun binary works, then let #74356 close through the merge.

## Reproduction Assessment

Yes. Static inspection of current main reproduces the issue at `Dockerfile:47`, where the build stage executes `https://bun.sh/install` through `bash`; no live Docker build was needed to confirm that source-level behavior.

## Solution Assessment

Yes, with a small test-helper caveat. Copying Bun from a digest-pinned official image aligned with CI is the narrow maintainable fix for the live Dockerfile risk; the digest test should model Docker ARG scope more accurately.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.88

Full review comments:

- **[P3] Model Docker ARG scope in the digest helper:** `src/docker-image-digests.test.ts:36-43`
  - body: `resolveArgDefaults` now records ARG defaults from the whole Dockerfile, but Docker only lets `FROM` resolve ARGs declared before the first `FROM`. A future stage-local ARG could make this test resolve an image value Docker would not resolve, so limit defaults to global ARGs or model per-stage scope explicitly.
  - confidence: 0.74
