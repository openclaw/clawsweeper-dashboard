## What This Changes

The PR updates `parseIdentityMarkdown` to extract trailing avatar sources from descriptive `Avatar:` lines and adds one parser regression test.

## Best Possible Solution

Preserve direct URL, data URI, and valid direct avatar path values unchanged before attempting descriptive-tail extraction, then add focused tests for descriptive text and dash-containing direct filenames plus the required changelog entry.

## Reproduction Assessment

Yes. With the PR helper, `- Avatar: avatars/hawk - final.png` or `- Avatar: avatars/hawk—final.png` can be split to `final.png`, which `looksLikeAvatarPath` accepts; current main preserves the full value.

## Solution Assessment

No. The intended behavior is reasonable, but the implementation should first recognize direct avatar values and only extract a trailing source from clearly descriptive prose.
