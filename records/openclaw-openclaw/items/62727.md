## Summary

Keep open: the PR targets a plausible identity parser gap, but the current diff can regress direct avatar values containing dash characters and still lacks contributor real behavior proof.

## What This Changes

The PR adds an avatar-specific tail extractor to `parseIdentityMarkdown` and a regression test for a descriptive `IDENTITY.md` avatar line ending in a workspace-relative path.

## Best Possible Solution

Preserve whole direct URL, data URI, and workspace-relative avatar values before applying a descriptive-tail heuristic, with coverage for both descriptive prose and dash-containing filenames.

## Reproduction Assessment

Yes, at source level. The PR helper scans before direct-value recognition while `looksLikeAvatarPath` accepts a truncated image filename tail; current main keeps the full direct avatar string.

## Solution Assessment

No. The intended parser tolerance is reasonable, but the implementation should first preserve direct avatar sources and only extract from clearly descriptive prose.
