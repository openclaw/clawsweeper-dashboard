## Summary

Keep open: the descriptive-avatar parsing idea is useful, but the current branch is not mergeable because it can truncate already-valid direct avatar paths and still lacks contributor real behavior proof.

## What This Changes

The PR changes `IDENTITY.md` avatar parsing to extract a trailing avatar path from descriptive avatar lines and adds parser regression coverage.

## Best Possible Solution

Preserve direct URL, data URI, and local path avatar values first, then parse only clearly descriptive prose tails with focused regression coverage.

## Reproduction Assessment

Yes for the review finding: source inspection shows the PR applies dash-tail extraction to all avatar values, so direct filenames like `avatars/hawk - final.png` can be truncated. For the requested descriptive-line tolerance, current main simply preserves the full value and the missing behavior is source-visible rather than a documented existing contract failure.

## Solution Assessment

No. The parser tolerance can be useful, but this implementation should first preserve already-valid direct avatar values and only then apply a constrained descriptive-tail heuristic.
