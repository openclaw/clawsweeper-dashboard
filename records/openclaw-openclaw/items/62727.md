## What This Changes

The PR adds avatar-specific IDENTITY.md parsing that extracts a trailing URL/path/data URI from descriptive Avatar lines and adds one parser regression test.

## Best Possible Solution

Preserve whole-value avatar URLs, data URIs, and workspace paths before applying descriptive-tail extraction, then cover both descriptive-tail parsing and dash-containing direct filenames with tests and real behavior proof.

## Reproduction Assessment

Yes for the PR-introduced regression: source inspection shows the separator scan can turn `avatars/hawk - final.png` or `avatars/hawk—final.png` into `final.png`, which still passes the avatar-path heuristic. I did not run tests in this read-only review.

## Solution Assessment

No. The maintainable path is to accept already-valid direct avatar values first, then use descriptive-tail extraction only as a fallback for non-direct descriptive Avatar text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve direct avatar values before tail extraction:** `src/agents/identity-file.ts:45`
