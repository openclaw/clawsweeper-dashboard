## Summary

Keep open: this is a plausible identity-avatar parser improvement, but the current PR head can truncate valid direct avatar paths with dash-separated filenames, and the contributor has not posted after-fix real behavior proof.

## What This Changes

The PR adds avatar-specific extraction for descriptive `IDENTITY.md` avatar lines and a parser regression test for a trailing workspace-relative avatar path.

## Best Possible Solution

Preserve direct URL, data URI, and workspace-relative avatar values unchanged before applying a descriptive-tail heuristic, then cover both descriptive prose and dash-containing filenames.

## Reproduction Assessment

Yes, at source level. The PR helper can reduce direct values such as `avatars/hawk - final.png` or `avatars/hawk—final.png` to `final.png`, while current main preserves the full direct value.

## Solution Assessment

No. The intended descriptive-tail extraction is reasonable, but the implementation should first recognize direct avatar sources and only extract from clearly descriptive prose.
