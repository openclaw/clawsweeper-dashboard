The branch adds avatar-specific IDENTITY.md parsing that extracts a trailing avatar URL/path/data URI from descriptive Avatar lines and adds parser regression coverage.

## Best Possible Solution

Preserve already-valid avatar URLs, data URIs, and workspace paths before applying any descriptive-tail heuristic, add dash-filename regression coverage, then provide redacted Control UI or terminal proof.

## Reproduction Assessment

Yes for the PR-introduced compatibility regression by source inspection: the PR scans dash-like separators on every avatar value, and looksLikeAvatarPath accepts a tail such as final.png. I did not execute tests in this read-only review.

## Solution Assessment

No. The useful parser expansion should preserve already-valid avatar values first, then apply descriptive-tail extraction only to otherwise descriptive text.
