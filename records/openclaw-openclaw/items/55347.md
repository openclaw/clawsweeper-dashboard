## Summary

Keep this open. Current main has health probes, channel-level health restarts, doctor/config recovery, and restart sentinel notices, but it still lacks the requested native gateway self-heal workflow for process supervision, bounded self-restarts, restart-time validation/repair summaries, dead-man alerts, and configurable channel recovery briefs.

## What This Changes

The issue requests a native `gateway.selfHeal` workflow that monitors gateway health, validates or repairs config before restart, bounds restart attempts, and posts recovery briefs to a configured channel.

## Best Possible Solution

Keep this as the umbrella until maintainers either accept a complete native workflow or split canonical follow-ups for process supervision, bounded restart behavior, restart-time repair diagnostics, and user-visible recovery notification.
