## Summary

Keep open. The report is concrete and still matches current `main`: Telegram startup can perform `getMe`, command-menu sync, and `deleteWebhook`/`getWebhookInfo` work before the first polling cycle, and I found no implemented fix for this high-RTT startup failure.

## What This Changes

Fix the Telegram polling regression where high-RTT hosts time out during startup setup and never reach long polling.

## Best Possible Solution

Harden Telegram startup so bounded `getMe`, command-menu, and webhook-cleanup failures either retry with visible diagnostics or fall through to polling only when cleanup is already safe, without adding a new config knob unless maintainers explicitly choose that product surface.
