## Summary

Keep open. The report is recent and concrete, and current main still routes Telegram long polling through pre-`getUpdates` setup work; existing hardening covers parts of timeout/retry behavior, but it does not prove the reported high-RTT startup silence is fixed.

## What This Changes

Telegram polling on 2026.5.2 reportedly never reaches `getUpdates` on high-RTT VPS hosts after a pre-poll `getMe` timeout, regressing from 2026.4.29.

## Best Possible Solution

Keep this issue open and harden Telegram startup so pre-poll `getMe`, command-menu sync, and webhook cleanup timeouts either recover with visible diagnostics or safely reach long polling without adding a new config knob by default.
