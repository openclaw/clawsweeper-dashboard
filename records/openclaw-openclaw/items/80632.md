## Summary

Keep open: the Slack mission-thread goal is plausible and the PR supplies real Slack smoke output, but the current patch still has blocking correctness issues in mission detection and registry write ordering.

## What This Changes

The PR adds a Slack mission-thread detector and JSON registry, then wires inbound preparation, dispatcher replies, sendMessage actions, outbound sends, and threading tests to resolve mission-bound Slack messages to canonical threads or fail closed.

## Best Possible Solution

Keep the Slack-owned routing fix, but derive mission identity from an explicit trusted contract, persist canonical thread state only after the inbound Slack message is accepted, and cover the fail-closed paths with focused Slack plugin regressions.

## Reproduction Assessment

Yes for the PR defects: source inspection of the current PR head shows the registry write precedes ingress gates and the detector matches ordinary branch-like text that the new send paths fail closed on. I did not independently live-reproduce the original Slack misrouting claim beyond the PR's supplied live output.

## Solution Assessment

No. The patch is in the right Slack plugin area, but mission detection and persistence need a narrower contract before this is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Persist mission threads only after ingress accepts the message:** `extensions/slack/src/monitor/message-handler/prepare.ts:548-553`
  - body: The registry write runs before the ambiguous-thread, mention, sender, bot, and command ingress gates below it. A room message that matches a mission-like string can be saved as canonical even when this handler later returns null, so later sends may route into a Slack thread OpenClaw intentionally ignored.
  - confidence: 0.88
- **[P2] Require an explicit mission marker before fail-closed routing:** `extensions/slack/src/mission-threads.ts:21-24`
  - body: The detector treats ordinary branch text such as fix/foo or feat/bar, plus the hard-coded THREADING TEST phrase, as mission scope. Because the action and outbound paths now throw when a matched mission has no registry entry, normal Slack messages mentioning branch names can fail instead of sending top-level.
  - confidence: 0.9
