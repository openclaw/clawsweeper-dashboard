## Summary

Keep open: the PR addresses a real Slack mission-thread routing gap and includes sufficient live proof, but the patch still misses the exported Slack channel outbound text/media path used by durable routed sends.

## What This Changes

The PR adds Slack mission detection, a file-backed mission-thread registry, routing updates in Slack prepare/dispatch/tool/outbound paths, and focused regression tests.

## Best Possible Solution

Centralize mission-thread resolution and fail-closed behavior at the shared Slack send context or exported slackChannelOutbound text/media paths so payload, text, media, tool, and dispatcher sends share one invariant.

## Reproduction Assessment

Yes. The blocking PR defect is source-reproducible: generic durable Slack text delivery can reach slackChannelOutbound.sendText through the channel message adapter and still send mission text without consulting the mission-thread registry when no explicit thread is supplied.

## Solution Assessment

No. The registry and mission-detection direction are reasonable, but the patch should cover the exported Slack ChannelOutboundAdapter text/media paths rather than only the lazy outbound helper.
