## Summary

Keep this PR open: it is a focused Slack bug fix with supplied live proof, but the current branch still misses a reachable exported Slack outbound text/media path, so mission-bound generic sends can still escape the canonical thread.

## What This Changes

The PR adds Slack mission-thread detection, a file-backed mission-thread registry, dispatcher/action/outbound routing changes, and Slack regression tests for canonical mission threads.

## Best Possible Solution

Put the mission-thread resolution and fail-closed invariant at a shared Slack send context or at the exported slackChannelOutbound text/media paths so payload, tool, dispatcher, and durable routed sends share one canonical thread contract.

## Reproduction Assessment

Yes, for the blocking PR defect by source inspection. Generic outbound delivery reaches the exported Slack sendText/sendMedia paths, and current channel.ts only resolves explicit reply/thread IDs before sending.

## Solution Assessment

No. The registry and dispatcher/tool direction are appropriate, but the invariant needs to cover the exported Slack outbound text/media paths rather than only the lazy outbound helper.
