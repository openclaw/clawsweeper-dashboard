## What This Changes

The PR adds Slack mission-id detection, a file-backed mission-to-thread registry, and mission-thread resolution for inbound preparation, dispatcher replies, Slack tool sends, and outbound payload sends.

## Best Possible Solution

Keep the mission-thread invariant, but store it in SQLite-backed plugin state, route every Slack send surface through one shared mission-thread resolver, and make the fail-closed upgrade behavior an explicit maintainer decision.
