## Summary

Keep open: this is a useful Slack bug fix with sufficient live proof, but source inspection confirms the exported Slack channel-message text/media path still bypasses the new mission-thread registry and fail-closed guard.

## What This Changes

The branch adds Slack mission-thread detection, file-backed thread persistence, canonical-thread routing for prepared Slack mission replies/tools/outbound payloads, and regression tests.

## Best Possible Solution

Apply mission-thread resolution and the fail-closed invariant at the shared Slack send context or exported slackChannelOutbound text/media paths so every durable Slack write path uses the same canonical thread contract.

## Reproduction Assessment

Yes, for the blocking PR defect by source inspection: generic routed Slack text delivery reaches slackChannelOutbound.sendText through the channel message adapter, and that path still sends without consulting the mission-thread registry when no explicit thread is present.

## Solution Assessment

No. The registry and dispatcher/tool direction are appropriate, but this implementation is incomplete until the exported Slack channel-message text/media path shares the same mission-thread lookup and fail-closed behavior.
