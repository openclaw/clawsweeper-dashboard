## Summary

Keep this PR open: current main still lacks mission-specific Slack canonical-thread routing, but this branch is not merge-ready because it adds durable JSON sidecar state, only guards some send paths, and changes fallback behavior without a settled upgrade decision.

## What This Changes

The PR adds Slack mission-thread detection, file-backed mission-to-thread persistence, and mission-scoped thread resolution for inbound, dispatcher, tool, and outbound Slack sends.

## Best Possible Solution

Keep the mission-thread invariant, but implement it through SQLite plugin state, one shared resolver across every Slack send surface, and an explicit maintainer decision for any fail-closed upgrade behavior.

## Solution Assessment

No. The invariant is plausible, but the best fix needs SQLite-backed plugin state, one shared mission-thread resolver across all Slack send surfaces, and a deliberate fail-closed upgrade decision.
