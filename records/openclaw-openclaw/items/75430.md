## What This Changes

The issue reports that `openclaw tui` in v2026.4.29 hangs at startup with about 100% CPU while repeatedly reloading bundled provider plugins, with `OPENCLAW_DISABLE_BUNDLED_PLUGINS=1` as a TUI-only workaround.

## Best Possible Solution

Keep this open until current main has a focused regression or live proof that gateway-backed TUI reaches first render without loading or repeatedly reloading bundled provider plugin runtime in the TUI client. If a bug remains, fix the narrow TUI gateway startup/import path or provider/runtime-deps cache path without globally disabling bundled plugins or changing gateway provider loading.
