## Summary

Keep open: the linked auth-profile precedence bug is real, but the latest PR head still has two blocking regressions and the external real-behavior proof is stale after force-pushes.

## What This Changes

The PR changes auth-profile store merging to prefer selected-agent provider profiles, adds focused auth coverage, exports shared test helpers, adjusts channel-add catalog lookup, and changes one cron persisted payload validator.

## Best Possible Solution

Land a focused auth merge fix that makes selected-agent same-provider credentials local-first only when no explicit local order excludes them, preserves inherited main fallback, and keeps unrelated cron/channel/test-helper changes separate unless independently justified.

## Reproduction Assessment

Yes. Source inspection shows current main merges main and requested-agent auth stores before runtime order resolution, so same-provider different-profile IDs can leave inherited main auth ahead of the selected-agent profile.

## Solution Assessment

No. The shared merge boundary is the right place to fix the bug, but this PR must preserve explicit selected-agent provider order and should not loosen unrelated persisted cron payload validation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve explicit local auth order:** `src/agents/auth-profiles/persisted.ts:376-380`
  - body: When the selected-agent store already has an order entry for a provider, current docs define omitted profiles as excluded. This appends every selected-agent profile after the explicit order, so a local credential intentionally omitted from order can re-enter runtime auth selection before inherited fallback; only synthesize local-first order when there is no selected-agent order entry.
  - confidence: 0.92
- **[P2] Keep blank agentTurn payloads invalid:** `src/cron/persisted-shape.ts:67`
  - body: The loader currently skips persisted agentTurn jobs whose message is empty or whitespace-only. After this change, blank strings are accepted and can be passed to the isolated agent job as the prompt, so malformed persisted jobs can run instead of being skipped and warned.
  - confidence: 0.86
