## Summary

Keep open: this PR addresses a valid auth-profile precedence bug, but the current patch can broaden explicit selected-agent auth order by reintroducing local profiles that the agent order intentionally omitted.

## What This Changes

The PR changes auth-profile store merging and regression coverage so selected-agent provider profiles can precede inherited main-agent profiles for the same provider.

## Best Possible Solution

Keep the shared merge fix, but only synthesize local-first fallback order when the selected agent has no explicit order for that provider; explicit selected-agent order must stay authoritative.

## Reproduction Assessment

Yes. Source inspection shows current main merges the main auth store with the requested agent store before resolveAuthProfileOrder honors stored provider order, and the PR body supplies terminal proof for the intended local-first path.

## Solution Assessment

No. The shared merge boundary is the right place, but this patch appends every local provider profile even when a local explicit order intentionally omitted it; preserving explicit local order is the safer fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

- **[P2] Preserve explicit local auth order:** `src/agents/auth-profiles/persisted.ts:813`
