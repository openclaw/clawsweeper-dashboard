## Summary

Keep open: the PR targets a real selected-agent auth precedence bug, but current head still broadens explicit local auth order and includes unrelated cron persisted-shape behavior changes that should not merge as-is.

## What This Changes

The PR changes auth-profile merging so selected-agent provider profiles precede inherited main-agent profiles, adds auth regression coverage, and includes cron/pairing test-stabilization changes.

## Best Possible Solution

Keep the shared auth merge fix, preserve explicit selected-agent order as authoritative, remove or separately justify the cron changes, and refresh current-head proof after the patch is updated.

## Reproduction Assessment

Yes. Source inspection of current main shows non-main agent auth merges main and selected-agent stores in a way that can keep inherited same-provider profiles first; the explicit-order regression in this PR is also source-reproducible from the new merge helper.

## Solution Assessment

No. The merge boundary is the right place, but the helper should only synthesize local-first fallback order when the selected agent lacks an explicit order for that provider, and unrelated cron validation changes should not be bundled into this auth fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve explicit local auth order:** `src/agents/auth-profiles/persisted.ts:811-814`
  - body: When the selected-agent store already has an order entry for a provider, that order is the authoritative filter. This helper appends every selected-agent profile for the provider after the explicit order, so a local profile intentionally omitted from the order can still be tried at runtime. Only synthesize local-first fallback order when there is no selected-agent order entry for that provider.
  - confidence: 0.92
- **[P2] Keep blank cron payloads invalid:** `src/cron/persisted-shape.ts:55-61`
  - body: The persisted loader uses this shape check to skip non-canonical jobs before they enter the scheduler. Accepting whitespace-only text/message loads invalid persisted jobs; main jobs are only skipped later, and isolated agentTurn jobs can be passed to the runner with an empty prompt. Keep the trim validation or replace it with an equivalent non-empty persisted payload check.
  - confidence: 0.84
