## Summary

Keep open: the PR targets a valid auth-provider regression, but the latest diff still has blocking auth-order and cron-validator regressions, and the contributor proof does not match the latest head.

## What This Changes

This PR changes auth-profile merging so selected-agent profiles can precede inherited main profiles, adds focused auth coverage, and also changes channel-add lookup plus cron persisted-job validation/tests.

## Best Possible Solution

Land a focused auth-profile merge fix that makes selected-agent same-provider profiles local-first only when no explicit selected-agent order excludes them, preserves inherited main fallback, and keeps unrelated cron/channel changes separate unless independently justified.

## Reproduction Assessment

Yes by source inspection. Current main merges main and selected-agent stores before auth-order resolution, so different same-provider profile IDs can leave inherited main auth ahead of the selected-agent profile.

## Solution Assessment

No. The shared merge boundary is the right place to fix the bug, but this patch must preserve explicit selected-agent auth-order exclusions, keep blank cron payloads invalid, and refresh proof for the latest head.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve explicit local auth order:** `src/agents/auth-profiles/persisted.ts:376-379`
  - body: When the selected-agent store already has an order entry for a provider, omitted profiles are intentionally excluded. Appending every selected-agent profile after that explicit order lets excluded local credentials re-enter runtime auth selection; only synthesize local-first order when the selected-agent store has no provider order entry.
  - confidence: 0.92
- **[P2] Keep blank agentTurn payloads invalid:** `src/cron/persisted-shape.ts:67`
  - body: The loader currently skips persisted agentTurn jobs whose message is empty or whitespace-only. After this change, blank strings are accepted and can be passed to the isolated agent job as the prompt, so malformed persisted jobs can run instead of being skipped and warned.
  - confidence: 0.86
