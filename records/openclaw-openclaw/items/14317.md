## Summary

Keep open. Current main still lacks Slack `groupPolicy: "private"` in the config contract, Slack routing, slash-command routing, docs, and exposure-audit handling; the requested access-control change is clear but needs maintainer/security product approval.

## What This Changes

Add a Slack `groupPolicy: "private"` mode that allows invited private channels while blocking public channels unless explicitly allowlisted.

## Best Possible Solution

Approve or reject the Slack private-channel trust model; if approved, implement a Slack-scoped policy across config types/schema/metadata, message and slash routing, exposure audit, docs, and focused tests with Slack Connect caveats.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction; source inspection shows `groupPolicy: "private"` is not accepted by the current type/schema contract and Slack routing has no privacy-aware branch.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers approve the Slack private-channel trust model; if approved, the maintainable path is a Slack-aware policy with routing, audit, docs, and tests rather than an enum-only change.
