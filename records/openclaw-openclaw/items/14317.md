## Summary

Keep open. Current main still lacks Slack `groupPolicy: "private"` in the shared config contract, Slack schema/docs, normal message routing, and slash-command routing; the request is concrete but changes Slack access-control semantics and needs maintainer/security product approval.

## What This Changes

Add a Slack-only `groupPolicy: "private"` mode that accepts private channels by membership while requiring explicit allowlist entries for public channels.

## Best Possible Solution

If approved, implement a Slack-scoped private policy across config types/schema/metadata, normal and slash routing, security audit/docs, and focused tests with Slack Connect caveats.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction; source inspection shows current main rejects `groupPolicy: "private"` in config and has no privacy-aware Slack routing branch.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers approve the Slack trust model; if approved, the narrow maintainable path is a Slack-specific policy with privacy metadata and audit/docs coverage, not a generic enum-only change.
