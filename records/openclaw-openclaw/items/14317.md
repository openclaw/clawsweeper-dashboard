Keep open: current main still rejects Slack `groupPolicy: "private"`, and Slack normal-message routing, slash-command routing, docs, and tests still lack a private-channel policy branch. The implementation shape is clear, but this is a Slack access-control feature that needs maintainer/security approval before automation should change the trust boundary.

Add a Slack-only `groupPolicy: "private"` mode that allows private channels by membership while still permitting explicitly allowlisted public channels.

## Best Possible Solution

If approved, implement a Slack-scoped private policy across config schema/types/metadata, normal and slash routing, security audit guidance, docs, and focused tests without changing other channel semantics or Slack defaults unless maintainers choose that explicitly.
