## Summary

Keep open. Current main still lacks Slack `groupPolicy: "private"` in the config contract and Slack routing paths, no fixing PR owns the work, and the requested policy changes Slack authorization and audit semantics.

## What This Changes

Add a Slack-only `groupPolicy: "private"` mode that allows invited private channels while keeping public channels gated by explicit allowlist entries.

## Best Possible Solution

If maintainers approve the trust boundary, implement a Slack-scoped private policy across config, normal and slash routing, audit/docs, and focused tests with Slack Connect caveats.

## Solution Assessment
