## Summary

Keep open: current main and v2026.6.5 still lack the requested Slack private-channel policy, although the original concern about Slack defaulting to `open` is obsolete because missing Slack config now fails closed to `allowlist`. The remaining request is a concrete ergonomics improvement, but using private-channel membership as authorization—including Slack Connect cases—requires an explicit maintainer/security product decision.

## What This Changes

Add a Slack-only `groupPolicy: "private"` mode that permits invited private channels while rejecting public channels unless explicitly allowlisted.

## Best Possible Solution

If maintainers approve private-channel membership as a trust signal, add an opt-in Slack-owned policy decision shared by normal messages and slash commands, retain the existing fail-closed `allowlist` default, support explicit public-channel exceptions, and document Slack Connect limitations.
