## Summary

Keep open. Current main still lacks Slack `groupPolicy: "private"` in the shared config contract, Slack schema/docs, normal message routing, slash-command routing, and security guidance; the request is plausible but needs a maintainer security/product decision before changing the Slack trust boundary.

## What This Changes

Add a Slack `groupPolicy: "private"` mode that allows invited private channels while requiring explicit allowlist entries for public channels.

## Best Possible Solution

If maintainers approve the trust boundary, implement a Slack-scoped private policy across config/schema/docs, normal and slash routing, action target checks, security audit guidance, metadata, and focused tests while documenting Slack Connect caveats.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high-confidence proof that current main does not accept or enforce `groupPolicy: "private"` for Slack.

## Solution Assessment

Unclear until maintainers approve the security model. The maintainable path is a Slack-scoped policy change with aligned routing, audit, docs, and tests rather than a broad shared enum change that leaves sibling channels unable to enforce the value.
