## Summary

Keep open. Current main and the latest release still only support Slack groupPolicy values open, allowlist, and disabled; the requested private-channel trust boundary remains missing and needs maintainer security/product approval before implementation.

## What This Changes

Request to add a Slack groupPolicy value private that admits invited private channels while allowing explicitly configured public-channel exceptions.

## Best Possible Solution

Approve or reject the Slack private-channel trust model; if approved, implement it as a Slack-scoped policy across config/schema/docs, routing, read-target checks, security guidance, metadata, and focused tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the missing capability: source inspection shows current main rejects or ignores groupPolicy="private" across schema, types, normal routing, slash routing, action reads, security guidance, and docs; this is not a bug reproduction path.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers approve the security model; the maintainable path is a Slack-scoped policy change rather than a broad shared enum change that leaves sibling channels unable to enforce the new value.
