## Summary

Keep open. Current main still lacks Slack `groupPolicy: "private"` in the shared config contract, Slack schema/docs/metadata, normal and slash-command routing, and exposure-audit classification; prior maintainer review comments that kept this open still match the current checkout.

## What This Changes

Add a Slack `groupPolicy: "private"` option that allows invited private channels, blocks public channels unless allowlisted, and updates security-audit behavior.

## Best Possible Solution

After approval, implement a Slack-scoped private policy end to end: config types/schema/metadata, normal and slash-command authorization using Slack privacy metadata, public-channel allowlist fallback, exposure-audit classification, docs with Slack Connect caveats, and focused tests.

## Reproduction Assessment

Yes. A source-level reproduction is enough: current main rejects `groupPolicy: "private"` in the shared schema and has no private-channel branch in normal or slash-command Slack routing, so no live Slack workspace is needed to verify the missing feature.

## Solution Assessment

Unclear until maintainers approve the trust model. If approved, the narrow maintainable solution is a Slack-specific `private` policy with explicit audit/docs coverage, not a generic enum addition that relies on existing policy fallthrough.
