## Summary

Keep open. Current main still lacks Slack `groupPolicy: "private"`, no fixing PR or duplicate owns the work, and the requested behavior changes Slack access-control and audit semantics rather than fitting conservative cleanup.

## What This Changes

Add a Slack `groupPolicy: "private"` option that allows invited private channels while ignoring public channels unless they are explicitly allowlisted.

## Best Possible Solution

If maintainers approve the trust boundary, implement a Slack-scoped private policy across config, normal and slash routing, audit/docs, and focused tests with Slack Connect caveats.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers approve the Slack trust model; if approved, the narrow maintainable path is a Slack-specific policy branch with audit and docs coverage, not an enum-only change.
