Keep open: current main and the latest shipped tag still expose only `open`, `disabled`, and `allowlist` for Slack channel group policy, and the Slack normal/slash routing paths have no private-channel authorization branch. Request to add a Slack `groupPolicy: "private"` mode that permits invited private channels while ignoring public channels unless explicitly allowlisted.

If maintainers approve the trust boundary, implement a Slack-scoped private policy across config schema/types/metadata, normal and slash routing, security audit guidance, docs, and focused tests while leaving existing defaults unchanged unless separately approved.
