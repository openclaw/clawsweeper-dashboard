## Summary

Keep open. Current main still does not implement Slack `groupPolicy: "private"`, and the requested change affects Slack authorization and security-audit semantics rather than being obsolete cleanup.

## What This Changes

Add a Slack `groupPolicy: "private"` mode that allows invited private channels, blocks public channels unless explicitly allowlisted, and updates audit/docs behavior.

## Best Possible Solution

If maintainers approve the trust boundary, implement a Slack-scoped private policy across config types/schema/metadata, normal and slash-command routing, exposure audit/docs, and focused regression tests with a Slack Connect caveat.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence verification that current main rejects `groupPolicy: "private"` and has no private-channel authorization path.

## Solution Assessment

Unclear until maintainers approve the Slack trust model. If approved, the maintainable path is a Slack-specific policy branch with audit and docs coverage, not a generic enum addition that falls through existing policy logic.
