## Summary

Keep open. Current main still lacks Slack `groupPolicy: "private"` in the shared config contract, Slack schema/docs, normal message routing, slash-command routing, and exposure-audit handling; the existing review comments remain accurate and this security-sensitive authorization change needs maintainer/product approval.

## What This Changes

Add a Slack `groupPolicy: "private"` mode that allows private channels by default while blocking public channels unless explicitly allowlisted.

## Best Possible Solution

Approve or reject the Slack private-channel trust model; if approved, implement a Slack-scoped policy across config schema/types/metadata, routing, audit, docs, and focused tests with explicit Slack Connect caveats.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction; source inspection shows `groupPolicy: "private"` is not accepted by the type/schema contract and Slack routing has no privacy-aware branch.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers approve the Slack private-channel trust model; if approved, the maintainable path is a Slack-aware policy with audit/docs/tests, not an enum-only change.
