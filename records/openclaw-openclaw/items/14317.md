## Summary

Keep open. Current main still lacks Slack `groupPolicy: "private"` in config types/schema, Slack docs, normal message routing, slash-command routing, and audit handling; the request is a security-sensitive Slack authorization feature that needs maintainer/product approval.

## What This Changes

Add a Slack-only `groupPolicy: "private"` option that permits private channels by membership while allowing explicitly configured public channels.

## Best Possible Solution

Approve or reject the Slack private-channel trust model; if approved, implement a Slack-scoped policy across config schema/types/metadata, routing, audit, docs, and focused tests with explicit Slack Connect caveats.

## Solution Assessment
