## Summary

Keep open. Current main still does not implement Slack `groupPolicy: "private"`, and the remaining work changes Slack authorization and security-audit semantics rather than cleaning up an obsolete issue.

## What This Changes

Request to add a Slack-specific `groupPolicy: "private"` mode that allows private Slack channels plus explicitly allowlisted public channels while blocking other public channels.

## Best Possible Solution

If maintainers approve the trust boundary, implement a Slack-scoped private policy end to end across config schema/types/metadata, normal and slash-command routing, exposure audit guidance, docs with Slack Connect caveats, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection still gives high-confidence verification that current main rejects `groupPolicy: "private"` and has no private-channel authorization path.

## Solution Assessment

Unclear until maintainers approve the Slack trust model. If approved, the maintainable path is a Slack-specific policy with explicit public-allowlist fallback and audit/docs coverage, not a generic enum addition that falls through existing policy logic.
