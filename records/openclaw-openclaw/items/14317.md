## Summary

Keep open: current main and v2026.5.22 still accept only `open`, `disabled`, and `allowlist` for Slack group policy, and Slack routing has no private-channel policy branch. Implementing this needs maintainer/security approval because it would make Slack privacy metadata part of the authorization boundary.

## What This Changes

Add a Slack `groupPolicy: "private"` mode that allows invited private channels while ignoring public channels unless explicitly allowlisted.

## Best Possible Solution

If maintainers approve the trust boundary, implement a Slack-scoped private policy across config schema/types/metadata, normal/slash/action routing, security audit guidance, docs, and focused tests while leaving existing defaults unchanged unless separately approved.
