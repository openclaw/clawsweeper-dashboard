## Summary

Keep open. Current main still lacks Slack `groupPolicy: "private"` in the shared config contract and Slack routing paths, and prior review comments already identified the same unresolved access-control decision.

## What This Changes

The issue requests a Slack-only `groupPolicy: "private"` option that allows private Slack channels by membership while denying public channels unless explicitly allowlisted.

## Best Possible Solution

If approved, implement a Slack-scoped private policy across config types/schema/metadata, normal and slash routing, security audit/docs, and focused tests with explicit Slack Connect caveats.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction; source inspection confirms current main rejects `groupPolicy: "private"` and has no privacy-aware Slack routing branch.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers approve the Slack trust model; if approved, the maintainable path is a Slack-specific policy with privacy metadata, audit classification, docs, and tests, not a generic enum-only change.
