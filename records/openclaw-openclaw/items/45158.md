- `impact:session-state`: Wrong routing can mix transcript, memory, and per-agent session state across users configured for isolation.

## Summary

Keep open: current main still has a source-reproducible Feishu/multi-account routing pitfall where bare bindings can miss named accounts and fall back to one agent session. The related account-scope PR is still open and unmerged, and the remaining fix needs maintainer approval because wildcard/default account behavior affects isolation semantics.

## What This Changes

The issue asks for Feishu multi-agent routing to honor configured per-user/per-account bindings instead of sending all messages to one agent session.

## Best Possible Solution

Resolve this through a maintainer-approved Feishu/account-scope binding path that preserves explicit account isolation, documents `peer` and `accountId` usage, adds focused resolver/CLI/doctor coverage, and includes redacted Feishu proof.

## Reproduction Assessment

Yes for the likely source-level failure: current routing code and tests show omitted-account bindings miss named accounts and fall back to `main`. The exact seven-account Feishu setup was not live-reproduced because the full config and credentials are unavailable.

## Solution Assessment

Unclear as a final product fix. Explicit `peer` and `accountId` guidance is necessary, but automatically treating bare multi-account bindings as wildcard routing changes isolation semantics and needs maintainer approval.
