## Summary

Keep open: current main still has an outbound/message-tool channel gap, because Feishu and Telegram do not provide the sanitizer hook that safer sibling channels use, and no canonical replacement issue or PR now owns the remaining work.

## What This Changes

The issue requests applying assistant-visible text sanitization to Feishu and other non-Discord outbound channel sends so internal tool-call/XML scaffolding cannot leak to users.

## Best Possible Solution

Keep this issue open and implement one canonical assistant-visible outbound sanitizer path, with focused Feishu and Telegram coverage plus formatting-preservation tests.

## Reproduction Assessment

Yes from source: Feishu and Telegram outbound paths lack the full assistant-visible sanitizer hook, and shared delivery only calls channel sanitization when the adapter defines it. I did not run a live Feishu or Telegram account reproduction in this read-only review.

## Solution Assessment

Yes on direction but not on exact placement: a shared sanitizer is more maintainable than per-channel config, but maintainers should choose the boundary to avoid double-sanitizing normal final replies or breaking channel formatting.
