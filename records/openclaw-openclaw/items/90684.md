## Label Justifications

- `impact:security`: The report concerns exposure of internal tool-call, command, and reasoning scaffolding in outbound user messages.

## Summary

Keep open: current main sanitizes ordinary final reply text and strips some shared runtime scaffolding before outbound delivery, but the direct/message-tool outbound path still relies on per-channel sanitizer hooks for the fuller assistant-visible cleanup, and Feishu/Telegram do not provide those hooks.

## What This Changes

Request that Feishu and other non-Discord outbound channels apply the canonical assistant-visible text sanitizer before sending user-visible text.

## Best Possible Solution

Choose one canonical outbound sanitization boundary: either a shared delivery default for model-originated sends or audited per-channel sanitizeText hooks for Feishu, Telegram, and similar adapters, with regression coverage for direct/message-tool outbound delivery.

## Reproduction Assessment

Yes, from source inspection: direct outbound delivery normalizes payloads through shared scaffolding stripping and optional channel sanitizeText hooks, while Feishu and Telegram lack the full assistant-visible hook. I did not run a live Feishu/Telegram channel reproduction in this read-only review.

## Solution Assessment

Unclear as a final design: the issue is valid, but the best repair needs a maintainer boundary choice between a shared model-originated outbound sanitizer and audited per-channel hooks. Adding a new config flag is less attractive unless maintainers explicitly want opt-out behavior.
