## Summary

Keep open: current main and v2026.6.8 still leave Feishu/direct outbound paths dependent on optional per-channel sanitizer hooks, while Feishu and Telegram lack the full assistant-visible sanitizer hook and open related PRs do not yet close the broad cross-channel boundary.

## What This Changes

Request Feishu and other non-Discord outbound channel paths to apply canonical assistant-visible sanitization before sending user-visible text.

## Best Possible Solution

Maintainers should choose one canonical outbound sanitizer boundary, cover Feishu/direct message-tool and at least one sibling non-Discord path, and avoid a new config toggle unless an opt-out policy is explicitly wanted.

## Reproduction Assessment

Yes at source level: shared delivery only applies full assistant-visible cleanup when a channel provides sanitizeText, and Feishu/Telegram do not; a live channel repro remains missing.

## Solution Assessment

The issue direction is valid, but the best implementation needs maintainer choice between a shared model-originated outbound sanitizer and audited per-channel hooks; the suggested config-flag approach is not clearly the best default.
