## Label Justifications

- `P1`: Internal tool-call scaffolding can leak into visible messages across configured chat channels, making this an urgent channel-delivery safety bug.
- `impact:security`: The issue is about preventing internal tool XML, commands, and operational context from being exposed to chat participants.

## Summary

Keep open: current main and latest release still leave the central outbound/message-tool gap. Normal final replies are sanitized earlier, but shared channel delivery only calls an adapter sanitizer when one exists, and Feishu/Telegram still do not provide the assistant-visible sanitizer hook.

## What This Changes

Request that Feishu and other non-Discord outbound channel sends strip internal assistant XML/tool-call scaffolding before user-visible delivery.

## Best Possible Solution

Add a shared outbound visible-text sanitization step, or consistently wire sanitizeAssistantVisibleText into affected channel adapters, with focused Feishu and Telegram message-tool tests plus formatting-preservation cases.

## Reproduction Assessment

Yes at source level: message-tool/outbound payload text containing tool-call XML reaches Feishu or Telegram delivery because those adapters lack sanitizeText and shared delivery only calls that hook when present. I did not run a live transport account reproduction.

## Solution Assessment

Yes for the requested sanitizer direction, but the best fix is not a new config flag; use a shared/default outbound sanitizer or consistent adapter hooks with regression coverage for affected channels.
