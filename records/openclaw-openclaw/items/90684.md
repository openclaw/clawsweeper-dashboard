## Summary

Keep open: current main partially strips several tool-call leak shapes, but Feishu and some sibling outbound paths still do not consistently run the canonical assistant-visible sanitizer, and open related work covers narrower remaining sanitizer gaps.

## What This Changes

The issue asks OpenClaw to apply the assistant-visible text sanitizer to Feishu and other non-Discord outbound channel messages so internal tool-call, reasoning, and runtime scaffolding is not sent to users.

## Best Possible Solution

Keep this as the cross-channel sanitizer umbrella until a maintainer-approved shared final-delivery path applies the canonical assistant-visible sanitizer, with focused channel-path proof and no new config unless maintainers explicitly choose an opt-out.

## Reproduction Assessment

Yes at source level: current main shows the canonical delivery sanitizer is broader than the final-reply sanitizer, Feishu has no outbound sanitizeText hook, and low-level Feishu send/card helpers do not sanitize text themselves. I did not run a live Feishu channel reproduction.

## Solution Assessment

No for the per-channel config proposal; the better fix is a default-on shared delivery sanitizer boundary with focused tests for Feishu and sibling channels, avoiding a new opt-out config unless maintainers choose that product direction.
