## Label Justifications

- `P1`: Internal tool-call scaffolding can leak into visible messages across configured chat channels, making this an urgent channel-delivery safety bug.
- `impact:security`: The issue is about preventing internal tool XML, commands, and operational context from being exposed to chat participants.

## Summary

Keep open: current main and the latest release still leave the Feishu outbound/message-tool path without the assistant-visible sanitizer, while sibling channels show this class is fixed per adapter or sanitizer boundary. Related issues track broader audience-boundary leaks, but none safely supersedes this Feishu/non-Discord XML sanitizer gap.

## What This Changes

Feishu and other non-Discord channel outbound text should apply the existing assistant-visible sanitizer so internal tool-call XML does not reach chat users.

## Best Possible Solution

Make assistant-visible sanitization a consistent outbound delivery boundary, either shared or explicitly wired into affected adapters, with Feishu and Telegram regression coverage plus formatting-preservation cases.

## Reproduction Assessment

Yes, at source level: Feishu outbound lacks sanitizeText, Telegram still intentionally has no sanitizeText hook, and shared delivery only calls the channel sanitizer when present. I did not run a live Feishu or Telegram account reproduction in this read-only pass.

## Solution Assessment

Yes on the sanitizer direction, but not on adding a new config flag as the primary fix. The safer maintainable path is to reuse the existing assistant-visible sanitizer in the shared outbound/channel-adapter boundary while preserving channel formatting contracts.
