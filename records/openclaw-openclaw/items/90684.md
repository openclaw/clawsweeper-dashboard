## Label Justifications

- `P1`: Internal tool-call scaffolding can leak into visible chat messages across configured non-Discord channels, making this an urgent channel-delivery safety bug.
- `impact:security`: The issue is about preventing internal tool XML, commands, reasoning, or operational context from being exposed to chat participants.

## Summary

Keep open: current main still leaves Feishu and Telegram outbound/message-tool delivery dependent on per-channel sanitizer hooks, and those adapters lack `sanitizeText`, so internal tool-call XML can still reach chat users.

## What This Changes

The issue requests applying `sanitizeAssistantVisibleText()` or equivalent assistant-visible sanitization to Feishu and other non-Discord outbound channel messages.

## Best Possible Solution

Apply the canonical assistant-visible sanitizer at the shared outbound/plugin boundary or through an audited set of affected channel adapters, with Feishu and Telegram regression coverage and no new user-facing config unless maintainers explicitly choose opt-out behavior.

## Reproduction Assessment

Yes at source level: Feishu and Telegram outbound adapters can receive payload text without a `sanitizeText` hook, and shared delivery only applies that hook when present. I did not run live Feishu or Telegram credentials in this read-only review.

## Solution Assessment

No for the proposed config-flag options; the maintainable direction is a shared outbound/plugin-boundary sanitizer or audited per-adapter use of the existing sanitizer. The exact placement needs maintainer judgment because it affects plugin/channel formatting contracts.
