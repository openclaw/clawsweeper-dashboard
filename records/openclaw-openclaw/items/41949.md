## Summary

Keep open: current main has snapshot caps, an opt-in efficient browser snapshot mode, and generic tool-result/context guards, but it still does not make compact goal-shaped browser reads the normal contract or return compact post-action change reports. The remaining work needs maintainer product review and live browser proof before it is safe to automate.

## What This Changes

The issue requests browser-tool context-budget handling so large or noisy page content does not overwhelm model context during multi-step browser interactions.

## Best Possible Solution

Define the browser-agent read contract, make compact or task-scoped reads the normal path with explicit raw/large-read escape hatches, add compact action-diff responses, and prove the behavior on a real content-heavy browser flow.

## Reproduction Assessment

No high-confidence live reproduction was established; this read-only review did not run a multi-step browser task on a noisy site. Source inspection does show current-main paths where browser snapshots are bounded but not task-scoped by default, and action responses do not include compact change reports.

## Solution Assessment

Unclear. Compact, goal-shaped browser reads with explicit raw-read escape hatches and additive action diffs look maintainable, but changing defaults and browser-agent contracts needs maintainer product judgment plus live proof.
