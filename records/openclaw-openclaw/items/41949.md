## Summary

Keep open: current main and the latest release have important browser snapshot and generic tool-result caps, but the requested browser-specific, task-aware context reduction is still not a shipped default or defined contract.

## What This Changes

Request browser interactions to inject smaller, task-relevant page state into model context instead of repeated large page snapshots.

## Best Possible Solution

Define the browser-agent snapshot policy, then make agent-driven snapshots compact or task-scoped by default with regression tests and docs while preserving explicit raw/debug capture paths.

## Reproduction Assessment

No. The report gives plausible generic steps, but current main already has per-snapshot and tool-result caps, and I did not establish a failing current-main browser flow on a concrete site/model/session.

## Solution Assessment

Unclear. The direction is reasonable, but choosing task-aware extraction, efficient-by-default behavior, pagination, or summarization changes the browser tool contract and needs maintainer product review before implementation.
