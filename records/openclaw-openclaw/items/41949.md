## Summary

Keep open. Current main has useful mitigations for browser and tool-output size, but it still does not define the requested default compact, goal-shaped browser read and action-feedback contract; this remains a maintainer product decision rather than a safe autonomous bug fix.

## What This Changes

The issue requests context-budget-aware browser interaction output so multi-step browser tasks do not inject excessive page content into model context.

## Best Possible Solution

Define the browser-read and action-feedback contract first: make compact, goal-shaped output the intended default for interactive browser work, keep raw/full reads explicit, and prove it with focused browser-route tests plus one noisy-page live flow.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection does confirm the remaining contract gap: default browser reads are capped but not universally compact or goal-shaped, and action responses do not yet have a compact change-report contract.

## Solution Assessment

No, not as an autonomous bug patch yet. The maintainable solution needs a maintainer decision on default compact browser reads, explicit raw/full-read escape hatches, and whether action responses should grow a compact diff contract.
