## Summary

Keep open: current main and the shipped v2026.5.12 release still provide fixed browser snapshot caps, opt-in efficient snapshots, and generic tool-result truncation, but not a browser-specific default or task-scoped extraction policy for multi-step interaction quality. The recent FSB comment supports the requested direction, but the remaining choice is a maintainer product decision with live browser proof still missing.

## What This Changes

The issue asks for browser interactions to become context-budget-aware so large page snapshots do not crowd out multi-step agent reasoning.

## Best Possible Solution

Define the browser-agent snapshot policy, then make agent-facing snapshots compact or task-scoped by default with explicit raw snapshot escape hatches, regression tests, docs, and live proof on a noisy multi-step browser task.

## Reproduction Assessment

No high-confidence live reproduction was established because I did not run a multi-step browser task on a content-heavy site. Source review does show current-main paths where ordinary or explicit AI snapshots still carry fixed-size browser dumps and rely on generic truncation rather than task-scoped extraction.

## Solution Assessment

Unclear. Compact, goal-shaped browser reads with raw reads as an explicit escape hatch look like the right direction, but choosing the default behavior, action-diff shape, and compatibility escape hatches needs maintainer product judgment.
