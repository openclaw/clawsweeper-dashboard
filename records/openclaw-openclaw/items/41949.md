## Summary

Keep open: current main has browser snapshot caps, an opt-in efficient mode, and generic tool-result/context guards, but compact goal-shaped browser reads are still conditional rather than the normal browser-agent contract, and action responses do not yet provide compact change reports. The FSB comment and linked implementation context support the remaining direction: compact default reads, explicit raw-page escape hatches, and small post-action diffs.

## What This Changes

The issue requests browser-tool context handling that avoids injecting excessive page content into model context during multi-step interactions.

## Best Possible Solution

Define the browser-agent read contract, make compact or task-scoped reads the normal path with explicit raw/large-read escape hatches, add compact post-action change reports, and prove the behavior on a real content-heavy browser flow.

## Reproduction Assessment

No, not as a high-confidence live reproduction. I did not run a noisy multi-step browser task, but source inspection shows current main still has bounded raw AI snapshots as the default path and compact/interactive extraction only when efficient mode or role-snapshot options are selected.

## Solution Assessment

Unclear pending maintainer product review. Compact, goal-shaped browser reads with explicit raw-read escape hatches and action diffs look like the maintainable direction, but changing the default browser-agent contract needs live proof and compatibility judgment.
