## Summary

Keep open. Current main and v2026.5.7 still allow normal browser AI snapshots to return up to 40,000 characters as model-visible tool text, while efficient/compact snapshot behavior remains opt-in, so the requested browser-specific context budget policy is not implemented.

## What This Changes

The issue requests browser interactions to send only context-budget-aware, task-relevant page state instead of large raw page snapshots that can overwhelm the model transcript.

## Best Possible Solution

Define a browser-owned budget policy that defaults agent-facing snapshots to compact actionable output where appropriate, applies consistent limits across snapshot backends, and keeps full page detail behind explicit detail-fetch controls.

## Reproduction Assessment

No high-confidence end-to-end reproduction is available because the issue lacks a specific URL, transcript, token trace, model, and session config. Source inspection does show a current-main and v2026.5.7 path where large browser snapshot text can still enter the model-visible tool transcript.

## Solution Assessment

Unclear because no patch is proposed and compactness-versus-fidelity defaults are a product decision. The maintainable direction is a browser-specific budget policy at snapshot generation time, not only generic context pruning after tool output already exists.
