## Summary

Keep open: current main has bounded browser snapshots, an opt-in efficient mode, and generic tool-result truncation, but it still does not make compact goal-shaped browser reads the default or return compact post-action change reports. The remaining behavior needs maintainer product review and live browser proof before a fix shape is chosen.

## What This Changes

The issue requests context-budget-aware browser interactions with task-relevant page reads, explicit raw/large-read escape hatches, and compact post-action state changes.

## Best Possible Solution

Define the browser-agent read contract, make compact or task-scoped reads the normal path with explicit raw/large-read escape hatches, add compact action-diff responses, and prove the behavior on a real content-heavy browser flow.

## Reproduction Assessment

No high-confidence live reproduction was established; this read-only review did not run a multi-step browser task on a noisy site. Source inspection does show current-main paths where browser snapshots are bounded but not task-scoped by default, and action responses do not include compact change reports.

## Solution Assessment

Unclear. Compact, goal-shaped browser reads with explicit raw-read escape hatches and additive action diffs look like a maintainable direction, but the default behavior and compatibility policy need maintainer product judgment plus live proof.
