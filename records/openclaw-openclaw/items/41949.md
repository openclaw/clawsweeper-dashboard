## Summary

Keep open: current main still bounds browser snapshots and generic tool results, but it does not implement default compact, goal-shaped browser reads or action diffs. The FSB comment is useful prior art for the likely direction, while the remaining choice needs maintainer product judgment and live browser proof.

## What This Changes

Request context-budget-aware browser interactions that avoid repeatedly injecting large full-page snapshots into model context.

## Best Possible Solution

Define the browser-agent snapshot policy, then make agent-facing reads compact or task-scoped by default with explicit raw snapshot escape hatches, regression tests, docs, and live proof on a noisy multi-step browser task.

No high-confidence live reproduction was established because this review did not run a multi-step browser task on a content-heavy site. Source review does show current-main paths where ordinary or explicit AI snapshots still carry fixed-size browser dumps and rely on generic truncation rather than task-scoped extraction.

## Solution Assessment

Unclear. Compact, goal-shaped browser reads with raw reads as an explicit escape hatch look like the right direction, but choosing the default behavior, action-diff shape, and compatibility escape hatches needs maintainer product judgment.
