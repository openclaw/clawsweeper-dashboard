## Summary

Keep open. Current main and the latest release have snapshot caps plus an optional efficient preset, but normal browser snapshots can still return large wrapped page text as model-visible tool output, so the browser-specific context-budget behavior requested here is not implemented.

## What This Changes

Make browser interaction snapshots context-budget aware so page state is compacted or filtered instead of carrying large page content through model context.

## Best Possible Solution

Define a browser-owned budget policy that defaults agent-facing snapshots to compact actionable output where appropriate, applies consistent limits across snapshot backends, and keeps full page detail behind explicit detail-fetch controls.

## Reproduction Assessment

No high-confidence end-to-end reproduction is available because the issue lacks a specific URL, transcript, token trace, model, and session config. Source inspection does show a current-main and v2026.5.7 path where large browser snapshot text can still enter the model-visible tool transcript.

## Solution Assessment

Unclear because no patch is proposed and compactness-versus-fidelity defaults are a product decision. The maintainable direction is a browser-specific budget policy at snapshot generation time, not only generic context pruning after tool output exists.
