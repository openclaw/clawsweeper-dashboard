## Summary

Keep open: current main bounds browser snapshots and generic tool results, but it still lacks a browser-specific task-aware/default context policy, and no concrete failing current-main browser flow was reproduced.

## What This Changes

Request browser interactions to limit or summarize page snapshots so large pages do not consume model context during multi-step tasks.

## Best Possible Solution

Define the browser-agent snapshot policy, then make agent-driven snapshots compact or task-scoped by default with explicit raw capture escape hatches, regression tests, and docs.

## Reproduction Assessment

No for a concrete bug reproduction. Source review shows current main is bounded but not task-scoped, while the report does not provide a specific site/model/session that still exhausts context on current main.

## Solution Assessment

Unclear. The requested direction is reasonable, but choosing efficient-by-default snapshots, relevant-element extraction, pagination, or summarization changes the browser tool contract and needs maintainer product review.
