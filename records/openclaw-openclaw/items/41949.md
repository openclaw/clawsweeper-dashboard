## Summary

Keep open: current main has browser snapshot caps and opt-in efficient snapshots, but it does not yet provide a task-aware/default browser context policy that solves the reported quality problem.

## What This Changes

Request browser interactions to avoid filling model context with large full-page snapshots by using task-aware, summarized, or budgeted page state.

## Best Possible Solution

Define the browser-agent snapshot policy, then make agent-driven snapshots compact or task-scoped by default with explicit raw capture escape hatches, regression tests, and docs.

## Reproduction Assessment

No high-confidence end-to-end reproduction is available. Source review confirms current main is bounded but still not task-scoped by default, while the report lacks a specific site, model, and session trace that still exhausts context on current main.

## Solution Assessment

Unclear. The requested direction is reasonable, but choosing efficient-by-default snapshots, relevant-element extraction, pagination, or summarization changes the browser tool contract and needs maintainer product review.
