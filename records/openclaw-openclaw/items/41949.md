## Summary

Keep open. Current main and the latest release have useful browser snapshot caps, efficient mode, and generic tool-result truncation, but they still do not define the requested default compact, goal-shaped browser read and compact action-feedback contract.

## What This Changes

The issue asks for browser interactions to use context-budget-aware, task-relevant page state instead of repeatedly injecting large browser snapshots during multi-step tasks.

## Best Possible Solution

Define the browser-read and action-feedback contract first: compact, goal-shaped output as the intended interactive default, explicit full/raw reads for escape hatches, and proof across Playwright, Chrome MCP, raw CDP, and one noisy-page live flow.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection confirms the remaining contract gap: current browser snapshots are capped and configurable, but not universally compact, goal-shaped, or paired with compact action diffs.

## Solution Assessment

No, not as an autonomous bug patch. The maintainable path needs a maintainer decision on default compact browser reads, explicit full/raw escape hatches, and whether action responses should grow a compact change-report contract.
