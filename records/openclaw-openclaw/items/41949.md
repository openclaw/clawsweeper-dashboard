## Summary

Keep open. Current main has real mitigations for oversized browser/tool output, but it still leaves the core browser-read contract as a product decision: compact, goal-shaped browser reads and action change reports are not the default behavior, and the FSB comment is useful prior art for that decision.

## What This Changes

The issue requests browser interactions to be context-budget aware by using compact, task-relevant browser state instead of repeatedly injecting large page content into model context.

## Best Possible Solution

Define a browser-plugin read contract that defaults interactive work to compact, goal-shaped snapshots and small action feedback, keeps full/raw reads explicit, and proves it with focused route/tool tests plus one live noisy-page flow.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection does confirm the remaining contract gap: efficient, compact browser reads are explicit or configuration-driven rather than the default browser interaction behavior.

## Solution Assessment

No, not as an autonomous bug patch yet. The maintainable solution needs a maintainer decision on the default compact browser read contract, raw/full-read escape hatches, and whether action responses should grow a small diff contract.
