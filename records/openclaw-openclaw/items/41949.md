## Summary

Keep open: current main and the latest release partially mitigate browser context pressure with snapshot caps, efficient snapshot mode, and generic tool-result guards, but explicit AI snapshots still carry bounded raw page content and action responses do not provide compact change reports. The FSB field comment supports the same remaining direction: compact goal-shaped reads, explicit raw-page escape hatches, and small post-action diffs.

## What This Changes

The issue asks for browser interactions to become context-budget aware by using compact or task-relevant page state instead of repeatedly injecting large raw page snapshots into model context.

## Best Possible Solution

Define the browser-agent read contract, make compact or task-scoped reads the normal path with explicit raw/large-read escape hatches, add compact post-action change reports, and validate the result on a real content-heavy multi-step browser flow.

## Reproduction Assessment

No, not as a high-confidence live reproduction. Source inspection confirms bounded raw AI snapshots remain available and compact extraction is conditional, but I did not run a noisy multi-step browser task that demonstrates context exhaustion on current main.

## Solution Assessment

Unclear pending maintainer product review. Compact, goal-shaped browser reads with raw-read escape hatches and action diffs look like the maintainable direction, but changing browser defaults and action response semantics needs compatibility judgment and live proof.
