## Summary

Keep open. Current main has snapshot caps and an optional efficient preset, but it still does not implement browser-specific context-budget-aware behavior because large wrapped snapshot text can still be returned directly into the tool transcript and compact mode remains conditional.

## What This Changes

The issue requests browser-specific context-budget-aware snapshot handling so large page content does not fill model context during multi-step browser tasks.

## Best Possible Solution

Define a browser-owned budget policy that defaults normal agent-facing snapshots to compact/actionable output where appropriate, applies consistent limits across snapshot backends, and keeps full page detail behind explicit detail-fetch controls.

## Reproduction Assessment

No high-confidence end-to-end reproduction is available because the issue lacks a specific URL, transcript, token trace, model, and session config. Source inspection does show a current-main path where browser snapshot text can still enter the tool transcript as large model-visible content.

## Solution Assessment

Unclear because no patch is proposed and the compactness-versus-fidelity default is a product decision. The maintainable direction is a browser-specific budget policy at snapshot generation time, not only generic context pruning after tool output has already been produced.
