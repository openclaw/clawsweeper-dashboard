## Summary

Keep open. Current main has meaningful mitigations, including optional efficient browser snapshots, generic tool-result caps, and session pruning, but it still does not define the default compact, goal-shaped browser read/action-diff contract requested here.

## What This Changes

The issue requests context-budget-aware browser interactions so large or noisy pages do not flood model context with raw page or snapshot content during multi-step browser work.

## Best Possible Solution

Define a maintainer-approved browser interaction contract: compact, goal-shaped reads by default; explicit raw/full snapshots on demand; compact action-change reports after interactions; and live proof on a noisy multi-step page.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows browser snapshots can still contribute large wrapped page-content blocks and only optional efficient/context-pruning mitigations exist, but the exact multi-step degradation needs a live current-main browser proof.

## Solution Assessment

Unclear until maintainers choose the contract. Existing efficient snapshots, tool-result caps, and session pruning are useful mitigations, but the requested default compact read/action-diff behavior is a browser product/API decision rather than a narrow bug patch.
