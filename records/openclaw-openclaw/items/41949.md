## Summary

Keep open: current main and the latest release still provide fixed browser snapshot caps, opt-in efficient snapshots, and generic tool-result truncation, but they do not implement a browser-specific default or task-scoped extraction policy for multi-step page interaction quality. The remaining decision is product/behavioral, not a narrow automated bug fix.

## What This Changes

Request context-budget-aware browser interactions so large page snapshots do not crowd out multi-step agent reasoning.

## Best Possible Solution

Define the browser-agent snapshot policy, then make agent-facing snapshots compact or task-scoped by default with explicit raw snapshot escape hatches, regression tests, and docs.

## Reproduction Assessment

No high-confidence end-to-end reproduction is available because I did not run a live multi-step browser task. Source review does show a clear current-main path where ordinary and explicit AI snapshots can still contribute fixed-size browser dumps while repeated results rely on generic truncation instead of task-scoped extraction.

## Solution Assessment

Unclear. The requested direction is reasonable, but choosing efficient-by-default snapshots, task-aware extraction, pagination, or summarization changes the browser tool contract and should be a maintainer product decision.
