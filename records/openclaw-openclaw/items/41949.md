## Summary

Keep open. Current main has useful mitigations, including optional efficient snapshots and generic tool-result caps, but it still does not implement the requested default compact, goal-shaped browser read/action-diff contract.

## What This Changes

The issue requests browser interaction output that is context-budget aware by default, with compact task-relevant page state instead of repeated large page snapshots.

## Best Possible Solution

Define a maintainer-approved browser interaction contract: compact, goal-shaped reads by default, explicit raw/full snapshots on demand, and compact action-change reports after browser actions.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection shows browser snapshots can still inject capped but large wrapped page-content blocks, while compact/efficient behavior is optional or configurable.

## Solution Assessment

Unclear until maintainers choose the product contract. Existing caps and efficient mode are useful mitigations, but the requested default compact read/action-diff behavior is not a narrow bug patch.
