## Summary

Keep open: current main has browser snapshot caps, an opt-in efficient mode, and generic tool-result/context guards, but compact goal-shaped browser reads are still conditional rather than the default browser-agent contract, and action responses do not yet provide compact change reports.

## What This Changes

The issue requests browser interaction context handling that avoids injecting excessive page content into model context during multi-step browser tasks.

## Best Possible Solution

Define the browser-agent read contract, make compact or task-scoped reads the normal path with explicit raw/large-read escape hatches, add compact post-action change reports, and prove the behavior on a real content-heavy browser flow.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows current main still has bounded raw AI snapshots as the default or explicit AI path, while compact role snapshots require efficient mode or explicit role-snapshot options.

## Solution Assessment

Unclear pending maintainer product review. Compact, goal-shaped browser reads with explicit raw-read escape hatches and action diffs look like the maintainable direction, but changing the default browser-agent contract needs live proof and compatibility judgment.
