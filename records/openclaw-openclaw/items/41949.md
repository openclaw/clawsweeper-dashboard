## Summary

Keep open: current main has partial mitigations (bounded AI snapshots, an opt-in efficient snapshot mode, and generic tool-result overflow guards), but it does not yet implement the requested browser-specific, task-aware context-budget behavior by default.

## What This Changes

Request that the browser tool become context-budget aware so multi-step browser interactions avoid injecting large page snapshots into model context.

## Best Possible Solution

Define the browser-owned context policy first, then implement bounded task-relevant snapshot output or safer defaults with focused browser and agent-loop regression coverage.

## Reproduction Assessment

No high-confidence live reproduction is established. Source inspection confirms browser snapshots can still contribute large bounded text payloads, but the report lacks a concrete site/model/transcript path proving downstream degradation on current main.

## Solution Assessment

No, the existing generic guard plus opt-in efficient mode is only a partial mitigation. The durable solution needs an explicit browser policy for default compact snapshots, task-scoped extraction, or context-aware browser result shaping.
