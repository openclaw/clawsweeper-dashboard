## Summary

Keep open. Current main has meaningful mitigations, including capped AI snapshots, optional efficient browser snapshots, context pruning, and tool-result truncation, but it still does not make compact, goal-shaped browser reads the default or define a browser action change-report contract; the FSB comment is useful prior art for the product decision.

## What This Changes

The issue requests context-budget-aware browser interactions so large page snapshots do not consume the model context during multi-step browser tasks.

## Best Possible Solution

Define the browser read contract first: default to compact, goal-shaped snapshots/action feedback for interactive tasks, keep full/raw reads explicit, and prove the behavior with focused browser-route tests plus one live noisy-page flow.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection does confirm the remaining contract gap: default browser snapshots use the normal AI snapshot path and efficient, compact output is only explicit or configuration-driven.

## Solution Assessment

No, not as an autonomous bug patch yet. The maintainable solution needs a maintainer decision on default compact browser reads, explicit raw/full-read escape hatches, and whether action responses should grow a small diff contract.
