## Summary

Keep open: current main has browser snapshot caps, efficient snapshot mode, generic tool-result truncation, and an opt-in mid-turn pressure check, but it still has no browser-specific default or task-scoped context policy that resolves the reported quality problem. The remaining work needs maintainer product judgment and likely live repro data, so it is not a close or automated fix candidate.

## What This Changes

The issue requests browser-specific context-budget handling so interactive browser tasks send compact, task-relevant page state instead of large page snapshots.

## Best Possible Solution

Define the browser-agent snapshot policy, then make agent-facing snapshots compact or task-scoped by default with explicit raw snapshot escape hatches, regression tests, and docs.

## Reproduction Assessment

No high-confidence end-to-end reproduction is available; I did not run a live multi-step browser session. Source review shows ordinary and explicit AI snapshots can still use the default cap path while repeated browser results rely on generic truncation rather than task-scoped browser extraction.

## Solution Assessment

Unclear. The requested direction is reasonable, but choosing efficient-by-default snapshots, task-aware extraction, pagination, or summarization changes the browser tool contract and needs maintainer product review.
