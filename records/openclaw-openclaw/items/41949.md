## Summary

Keep open. Current main has explicit snapshot caps and an optional efficient preset, but it still does not implement the requested browser-specific context-budget-aware behavior: normal AI snapshots can be large model-visible text, efficient mode is conditional, and backend coverage is uneven.

## What This Changes

Request browser interactions use compact, task-relevant, context-budget-aware page state instead of large raw or full-page snapshots during multi-step browser tasks.

## Best Possible Solution

Define a browser-owned budget policy that defaults normal agent-facing snapshots to compact/actionable output where appropriate, applies consistent limits across snapshot backends, and keeps full page detail behind explicit detail-fetch controls.

## Reproduction Assessment

No high-confidence end-to-end reproduction is available because the issue lacks a specific URL, transcript, token trace, or model/session config. Source inspection does show a clear current-main path where browser snapshots can still enter the tool transcript as large text.

## Solution Assessment

Unclear because no patch is proposed and the default compactness versus ref-fidelity tradeoff is a product decision. The maintainable direction is a browser-specific budget policy at snapshot generation time, not only generic context pruning after tool output has already been produced.
