## Summary

Keep open: current main has important mitigations for browser output size, including AI snapshot caps, an opt-in efficient snapshot mode, and generic tool-result context guards, but the browser-agent contract still defaults to bounded raw AI snapshots unless efficient/role options are selected. The FSB-linked comment adds useful prior art for the remaining direction: compact goal-shaped reads, explicit raw-read escape hatches, and small post-action change reports.

## What This Changes

The issue asks for browser interactions to become context-budget aware so noisy page content does not fill the model context during multi-step browser tasks.

## Best Possible Solution

Define the browser-agent read contract, make compact or task-scoped reads the normal path with explicit raw/large-read escape hatches, add compact post-action change reports, and prove the behavior on a real content-heavy browser flow.

## Reproduction Assessment

No, not as a high-confidence live reproduction. Source inspection shows the relevant current-main path: bounded raw AI snapshots remain the default and compact/interactive extraction is conditional, but I did not run a noisy multi-step browser task.

## Solution Assessment

Unclear pending maintainer product review. Compact, goal-shaped browser reads with explicit raw-read escape hatches and action diffs look like the maintainable direction, but changing the default browser-agent contract needs live proof and compatibility judgment.
