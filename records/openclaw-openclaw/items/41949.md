## Summary

Keep open: current main and the latest release have browser snapshot caps, efficient/compact snapshot options, and generic tool-result context guards, but the normal browser-agent read contract still allows bounded raw AI snapshots and action responses still do not provide compact change reports. That leaves the central request unresolved and needing maintainer product/API review plus live browser proof.

## What This Changes

The issue asks for browser interactions to avoid exhausting model context by using compact, task-relevant page state instead of repeatedly injecting large raw page snapshots.

## Best Possible Solution

Define the browser-agent read contract, make compact or task-scoped reads the normal path with explicit raw/large-read escape hatches, add compact post-action change reports, and validate it on a real content-heavy browser flow.

## Reproduction Assessment

No, not as a high-confidence live reproduction. Source inspection shows bounded raw AI snapshots remain the default path and compact extraction is conditional, but I did not run a noisy multi-step browser task that demonstrates context exhaustion on current main.

## Solution Assessment

Unclear pending maintainer product review. Compact, goal-shaped browser reads with raw-read escape hatches and action diffs look like the maintainable direction, but changing browser defaults and action response semantics needs compatibility judgment and live proof.
