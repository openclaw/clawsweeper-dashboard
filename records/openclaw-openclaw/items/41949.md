## Summary

Keep open: current main has browser snapshot caps, configurable efficient snapshots, and generic tool-result truncation, but it still does not define a browser-specific, task-scoped/default context policy that resolves the reported quality problem.

## What This Changes

Request browser interactions to avoid exhausting model context by sending compact, task-relevant page state instead of large raw snapshots.

## Best Possible Solution

Define the browser-agent snapshot policy, then make agent-facing snapshots compact or task-scoped by default with explicit raw snapshot escape hatches, regression tests, and docs.

## Reproduction Assessment

No high-confidence end-to-end reproduction is available. Source review shows the relevant current-main path: ordinary and explicit AI snapshots can still carry up to the default AI snapshot cap unless efficient or role-snapshot features are requested.

## Solution Assessment

Unclear. The requested direction is reasonable, but choosing efficient-by-default snapshots, task-aware extraction, pagination, or summarization changes the browser tool contract and needs maintainer product review.
