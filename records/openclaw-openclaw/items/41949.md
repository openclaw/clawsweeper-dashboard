## Summary

Keep open. Current main and v2026.6.10 have useful mitigations, including optional efficient snapshots and generic tool-result caps, but they still do not provide the default compact, goal-shaped browser read plus action-change-report contract requested here.

## What This Changes

The issue asks for browser interactions to avoid pushing large raw page snapshots into model context by using compact, task-relevant browser state by default.

## Best Possible Solution

Define a maintainer-approved browser interaction contract: compact goal-shaped reads by default, explicit raw/full snapshots on demand, and compact action-change reports after UI mutations.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows repeated browser snapshots can still contribute large wrapped page-content blocks and current mitigations are optional or generic, but the downstream wrong-action degradation still needs current-main live proof.

## Solution Assessment

Unclear until maintainers choose the browser read contract. Existing efficient snapshots, tool-result caps, and pruning are useful mitigations, but they are not the default compact read/action-diff behavior requested here.
