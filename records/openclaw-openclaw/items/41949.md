## Summary

Keep open. Current main has stronger generic tool-result caps and compaction prechecks, but it still does not implement the browser-specific, context-budget-aware snapshot policy requested here: the default AI browser snapshot path remains broad, compact mode is optional or conditional, and normal snapshot output still returns model-visible text.

## What This Changes

The issue requests browser interactions to become context-budget aware so large page snapshots do not flood the model transcript during multi-step browser tasks.

## Best Possible Solution

Implement a browser-owned context budget policy that defaults agent-facing snapshots to compact/actionable output where appropriate, applies explicit max-character/detail limits consistently across snapshot backends, adapts under context pressure, and keeps full page state available through explicit follow-up detail fetches rather than normal tool-result text.

## Reproduction Assessment

No high-confidence end-to-end reproduction is available because the report does not provide a specific URL, transcript, token trace, model, or browser profile. Source and tests do provide a high-confidence static path showing large browser snapshot text can still enter model-visible tool context.

## Solution Assessment

Unclear because there is no proposed patch to review. The current generic tool-result caps are a useful mitigation, but the maintainable fix should live in the browser snapshot policy rather than relying only on downstream transcript truncation after the browser result has already been produced.
