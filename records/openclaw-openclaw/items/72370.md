## Summary

Keep open. Current main still preserves the documented no-override policy for workspace hooks, but an eventless managed hook can win a same-name collision, register no handlers, and still be considered loadable by the status/check path.

## What This Changes

The issue asks OpenClaw to handle workspace hooks that collide with same-named managed hooks without silently replacing user hook logic with eventless no-op managed hooks.

## Best Possible Solution

Preserve managed-hook precedence, but carry collision and eventless-hook state into hooks list/info/check and status surfaces so enabled no-op winners are reported with an actionable blocked reason.

## Reproduction Assessment

Yes, by source inspection. A managed hook and workspace hook with the same name resolves to the managed hook, and if that managed hook has no metadata events, the loader registers nothing while status can still mark it loadable.

## Solution Assessment

Yes. A diagnostics-focused fix is the narrowest maintainable path because the no-override precedence is documented and covered by tests, while the remaining broken behavior is the readiness/status contract.

## Repair Work Prompt

Fix the hook collision/eventless readiness diagnostics for https://github.com/openclaw/openclaw/issues/72370. Preserve the documented no-override policy: workspace hooks must not override bundled, managed, or plugin-provided hooks with the same name. Instead, make the status/check/list/info path report an enabled managed hook with no metadata events as not ready/loadable, and carry enough collision context from workspace discovery or policy resolution so users see an actionable reason when their workspace hook was ignored due to a same-name managed winner. Keep the change focused on hook policy/loader/status/CLI diagnostics; do not add a new config option or change hook precedence. Add/update regression coverage for a same-name managed hook with empty metadata events plus a workspace hook with events, and update docs/CHANGELOG if user-visible CLI/status output changes.
