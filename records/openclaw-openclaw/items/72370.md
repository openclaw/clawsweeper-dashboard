## Summary

Keep open. Current main still preserves managed/bundled/plugin hook precedence and only logs same-name workspace collisions, while status/check output does not retain ignored-collision or empty-events state, so the silent-disable diagnostic problem remains valid.

## What This Changes

Request that OpenClaw surface actionable diagnostics when a same-name workspace hook is ignored in favor of a managed/bundled/plugin hook, especially when the winning hook has no metadata events.

## Best Possible Solution

Keep managed-hook precedence, but carry ignored-collision and empty-events state into hook status/check output, CLI JSON/text, docs, and tests so users can see the winning hook and the rename/remove path.

## Reproduction Assessment

Yes. A source-level reproduction is a managed hook and workspace hook with the same name where the managed hook is enabled but has no metadata events; current main keeps the managed entry, skips registration, and still reports readiness from status inputs that ignore events/collisions.

## Solution Assessment

Yes. A diagnostics-focused fix is the narrowest maintainable solution because it addresses the silent failure without weakening the managed-hook precedence policy.
