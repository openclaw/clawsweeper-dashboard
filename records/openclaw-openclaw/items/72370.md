## Summary

Keep open. Current main still enforces managed-hook precedence and drops same-name workspace collisions after a startup warning, while eventless managed hooks can still appear ready in hook status/check output, so the diagnostics bug remains valid.

## What This Changes

The issue requests actionable diagnostics when same-name workspace hooks are rejected in favor of managed hooks, especially when the winning managed hook has no metadata events and therefore registers no handler.

## Best Possible Solution

Keep the documented managed-hook precedence, but make ignored collisions and enabled hooks with no metadata events visible as not-ready/actionable in hook status, hook check/list/info JSON and text, docs, and regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a managed hook and workspace hook with the same name where the managed hook is enabled but has no metadata events; current main keeps the managed entry, skips registration, and still computes it as loadable.

## Solution Assessment

Yes. A diagnostics-focused repair is the narrowest maintainable fix because it preserves the documented no-override policy while making the effective no-op state visible and actionable.
