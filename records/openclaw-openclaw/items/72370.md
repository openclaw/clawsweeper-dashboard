## Summary

Keep open. Current main still enforces documented managed-hook precedence and drops same-name workspace collisions after a startup warning, while enabled managed hooks with empty metadata events can still look loadable in hook status surfaces, so the diagnostics bug remains valid.

## What This Changes

The issue asks OpenClaw to stop silently disabling same-name workspace hooks, or at least surface ignored collisions and eventless managed-hook replacements through actionable hook diagnostics.

## Best Possible Solution

Preserve documented no-override precedence, but carry collision and eventless-hook state into hook status/check/list/info and docs so an enabled no-op winner is reported as not ready/actionable.

## Reproduction Assessment

Yes. A high-confidence source path is a managed hook and workspace hook with the same name where the managed hook is enabled but has no metadata events: policy keeps the managed entry, loader registers no events, and status still computes it as loadable.

## Solution Assessment

Yes. A diagnostics-focused repair is the narrowest maintainable solution because it keeps the documented managed-hook precedence while making the effective no-op state visible and actionable.
