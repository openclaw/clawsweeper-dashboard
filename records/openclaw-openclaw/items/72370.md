## Summary

Keep open. Current main still preserves managed-hook precedence, which matches the documented trust boundary, but it still drops ignored workspace-collision details and treats enabled hooks with empty metadata events as loadable in status/check output.

## What This Changes

The issue reports workspace hook name collisions being ignored in favor of managed hooks, with eventless managed hooks then behaving like no-ops without surfaced diagnostics.

## Best Possible Solution

Preserve managed-over-workspace precedence, but carry collision and empty-events state into hook diagnostics so ignored workspace hooks and eventless winners are visible and actionable.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a managed hook and workspace hook with the same name where the managed hook is enabled but has no metadata events; current main keeps the managed entry, skips handler registration, and reports no collision or no-events blocker through status/check data.

## Solution Assessment

Yes. A diagnostics-focused fix is the narrowest maintainable path because it addresses the silent failure without weakening the documented managed-hook precedence policy.
