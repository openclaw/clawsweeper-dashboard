## Summary

Keep open. Current main still preserves the documented managed-hook precedence but does not make the effective no-op winner visible as a readiness problem, so the diagnostics bug remains actionable.

## What This Changes

The issue reports workspace-defined hooks being rejected on same-name managed-hook collisions while eventless managed hooks can replace the workspace behavior without clear status diagnostics.

## Best Possible Solution

Preserve documented no-override precedence, but carry collision and no-events state into hook status/check/list/info and docs so an enabled no-op winner is reported as not ready and actionable.

## Reproduction Assessment

Yes. A high-confidence source path exists: a same-name managed/workspace collision keeps the managed entry, the loader skips registration when that managed entry has no metadata events, and status still computes it as loadable.

## Solution Assessment

Yes. A diagnostics-focused repair is the narrowest maintainable solution because it keeps the documented managed-hook precedence while making the effective no-op state visible and actionable.
