## Summary

Keep open. Current main still preserves managed-hook precedence, drops same-name workspace hook collisions after a startup warning, and reports enabled hooks with missing metadata events as loadable, so the silent-disable diagnostics bug remains valid.

## What This Changes

The issue requests actionable handling when workspace-defined hooks share names with managed hooks and the winning managed hook has no metadata events.

## Best Possible Solution

Preserve managed-hook precedence, but carry ignored-collision and empty-events diagnostics into hook status/check/list/info output with clear rename, remove, or metadata-fix guidance.

## Reproduction Assessment

Yes. A source-level repro is a managed hook and a workspace hook with the same name where the managed entry is enabled but has no metadata events; current main keeps the managed entry, skips handler registration, and still reports it as loadable.

## Solution Assessment

Yes. A diagnostics-focused fix is the narrowest maintainable solution because it addresses the silent failure without weakening the managed-hook trust and precedence model.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/72370. Current main drops ignored same-name workspace hook collisions after loadWorkspaceHookEntries logs a warning, and buildWorkspaceHookStatus treats enabled hooks with empty or missing metadata.openclaw.events as loadable even though loadInternalHooks skips registration. Preserve the existing policy that workspace hooks cannot override managed/plugin/bundled hooks. Add a status/report model that carries ignored collision diagnostics and marks or warns enabled empty-event winners so `openclaw hooks list`, `info`, and `check` text/JSON make the winning hook source/path and the ignored workspace hook clear, with actionable rename/remove/fix-metadata guidance. Update docs/automation/hooks.md, docs/cli/hooks.md, and CHANGELOG.md because this changes user-facing diagnostics. Add focused tests for policy/workspace collision diagnostics, status loadability/blocked reason for empty events, loader behavior, and CLI JSON/text output. Do not change unrelated plugin SDK hook behavior, hook precedence, or compaction behavior.
