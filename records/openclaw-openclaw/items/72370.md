## Summary

Keep open. Current main intentionally preserves managed-hook precedence, but the silent-disable diagnostics problem remains: ignored workspace collisions are warning-only, discarded before status reporting, and eventless winning hooks still appear loadable to CLI/status checks.

## What This Changes

Workspace-defined hooks that collide with managed hooks should be surfaced clearly instead of being quietly replaced by managed entries that may have no metadata events.

## Best Possible Solution

Keep managed-hook precedence, but carry collision and empty-events state into hook status/check output, CLI JSON/text, docs, and focused tests so users can identify the winning hook and the rename/remove path.

## Reproduction Assessment

Yes. A source-level reproduction is a managed hook and workspace hook with the same name where the managed hook is enabled but has no metadata events; current main keeps the managed entry, skips registration, and has no status blocker for that condition.

## Solution Assessment

Yes. A diagnostics-focused fix is the narrowest maintainable solution because it addresses the silent failure without weakening the existing managed-hook precedence policy.
