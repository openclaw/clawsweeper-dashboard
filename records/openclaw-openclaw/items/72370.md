## Summary

Keep open. Current main still preserves managed-hook precedence, but ignored workspace collisions are only logged and enabled hooks with empty metadata events still look loadable to status/CLI diagnostics, so the observable silent-disable problem remains valid.

## What This Changes

The issue reports workspace hooks being ignored when their names collide with managed hooks, leaving eventless managed hooks active without visible status diagnostics.

## Best Possible Solution

Keep the existing precedence policy, but carry collision and metadata-event state into hook status/check output, CLI JSON, docs, and tests so users can see the ignored workspace hook, the winning hook, and the rename/remove path.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a managed hook and workspace hook with the same name where the managed hook is enabled but has no metadata events; current main keeps the managed entry, skips handler registration, and exposes no collision or empty-events blocker in status/check data.

## Solution Assessment

Yes. A diagnostics-focused fix is the narrowest maintainable solution because it addresses the silent failure without weakening the managed-hook precedence policy.
