## Summary

Keep open. Current main still preserves managed-hook precedence, drops same-name workspace hooks with only a startup warning, and reports eventless managed winners as loadable in status/CLI data, so the silent-disable diagnostics problem remains valid.

## What This Changes

The issue asks OpenClaw to make same-name workspace hook rejection and eventless managed-hook replacement visible and actionable instead of silently disabling custom workspace logic.

## Best Possible Solution

Keep managed-hook precedence, but carry ignored-collision and empty-events state into hook status/check output, CLI JSON/text, docs, and focused tests so users can see the winning hook and the rename or remove path.

## Reproduction Assessment

Yes. A source-level repro is a managed hook and workspace hook with the same name where the managed hook is enabled but has no metadata events; current main keeps the managed entry, skips handler registration, and still marks it loadable in status data.

## Solution Assessment

Yes. A diagnostics-focused fix is the narrowest maintainable solution because it addresses the silent failure without weakening the managed-hook precedence policy.
