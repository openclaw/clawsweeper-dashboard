## Summary

Keep open. Current main still preserves managed-hook precedence, drops ignored same-name workspace hooks after only a startup warning, and can report an enabled hook with no metadata events as loadable, so the diagnostics bug remains actionable.

## What This Changes

Workspace hook collisions with managed hooks should be surfaced as actionable hook status/check diagnostics, especially when the winning managed hook has no metadata events.

## Best Possible Solution

Keep managed-hook precedence, but carry ignored-collision and empty-events state into hook status/check output, CLI JSON/text, docs, and focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a managed hook and workspace hook with the same name where the managed hook is enabled but has no metadata events; current main keeps the managed entry, registers no handler, and still has status data that can mark it loadable.

## Solution Assessment

Yes. A diagnostics-focused repair is the narrowest maintainable solution because reversing workspace-vs-managed precedence would conflict with the documented hook discovery policy.
