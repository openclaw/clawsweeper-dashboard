## Summary

Keep open. Current main still enforces managed-hook precedence and drops same-name workspace collisions after a startup warning, while enabled managed hooks with no metadata events can still be treated as ready by hook status/check output.

## What This Changes

The issue reports that same-name workspace hooks are ignored in favor of openclaw-managed hooks, and eventless managed hooks then appear ready while registering no handlers.

## Best Possible Solution

Keep managed-hook precedence, but carry ignored-collision and empty-events state into hook status/check output, CLI JSON/text, docs, and focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a managed hook and workspace hook with the same name where the managed hook is enabled but has no metadata events; current main keeps the managed entry, skips handler registration, and still marks it loadable in status.

## Solution Assessment

Yes. A diagnostics-focused repair is the narrowest maintainable solution because docs already define the managed-hook precedence policy, so the fix should make the disabled workspace hook and eventless winner visible instead of allowing override.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/72370 as a diagnostics bug. Preserve the documented rule that workspace hooks cannot override bundled, managed, or plugin hooks with the same name. Add a narrow mechanism to carry ignored hook collisions and enabled hooks with empty metadata events into hook status/check output. `openclaw hooks list`, `openclaw hooks info`, and `openclaw hooks check` should make these cases actionable in text and JSON instead of reporting the winning eventless hook as ready. Add focused tests for same-name managed/workspace collision diagnostics and eventless enabled hooks, update docs for the new diagnostic behavior, and add a single-line user-facing CHANGELOG.md fix entry. Do not add a config option, do not allow workspace override of managed hooks, and do not broaden hook loading policy.
