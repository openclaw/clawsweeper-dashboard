## Summary

Keep open. Current main still preserves managed-hook precedence, drops ignored same-name workspace collisions after a startup warning, and reports enabled managed hooks with no metadata events as loadable, so the silent-disable diagnostics problem remains valid.

## What This Changes

The issue asks OpenClaw to make workspace hook collisions with managed hooks actionable instead of silently leaving same-name workspace hook logic disabled behind eventless managed hook entries.

## Best Possible Solution

Keep managed-hook precedence, but carry ignored-collision and empty-events state into hook status/check output, CLI JSON/text, docs, and focused regression tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a managed hook and workspace hook with the same name where the managed hook is enabled but has no metadata events; current main keeps the managed entry, skips handler registration, and still marks it loadable in status.

## Solution Assessment

Yes. A diagnostics-focused repair is the narrowest maintainable path because it fixes the silent failure without weakening the documented managed-hook precedence policy.
