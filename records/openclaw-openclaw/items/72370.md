## Summary

Keep open. Current main still documents and enforces managed-hook precedence over same-name workspace hooks, but it still drops the ignored workspace collision from diagnostics and can mark an enabled managed hook with no metadata events as ready, so the actionable status/check bug remains.

## What This Changes

Workspace hooks with names colliding with managed hooks are ignored at startup, and eventless managed winners can effectively replace the workspace logic without being surfaced as not ready.

## Best Possible Solution

Preserve the documented no-override precedence, but carry collision and eventless-hook state into hook status/check/list/info and docs so an enabled no-op winner is reported as not ready with an actionable explanation.

## Reproduction Assessment

Yes. A high-confidence source path is a managed hook and workspace hook with the same name where the managed hook is enabled but has no metadata events: policy keeps the managed entry, the loader registers no events, and status still computes it as loadable.

## Solution Assessment

Yes. A diagnostics-focused repair is the narrowest maintainable solution because it keeps the documented managed-hook precedence while making the effective no-op state visible and actionable.
