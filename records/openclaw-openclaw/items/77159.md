## Summary

Close because the provided manifest and stale allowlist entry point to external plugin packaging/configuration, not an OpenClaw source bug: current OpenClaw documents the `channelConfigs` contract, emits the reported diagnostic for non-bundled manifests missing it, and ships a repo-owned Feishu plugin alternative.

## What This Changes

The issue asks whether the external openclaw-lark Feishu channel manifest should add `channelConfigs` metadata so OpenClaw 2026.5.2 can load `channels.feishu` configuration.

## Best Possible Solution

Keep the OpenClaw manifest contract unchanged; update the external `openclaw-lark` package to ship inline JSON Schema under `openclaw.plugin.json#channelConfigs` or switch the install/config to the supported `@openclaw/feishu` plugin and remove stale `plugins.allow` entries.

## Reproduction Assessment

No high-confidence full runtime reproduction is available from the repo alone because the failing `openclaw-lark` package and local config are external. Source inspection does reproduce the reported diagnostic paths for missing `channelConfigs` metadata and stale plugin allowlist entries.

## Solution Assessment

Yes for the concrete manifest question: the narrow fix belongs in the external plugin manifest as inline JSON Schema under `channelConfigs`, not as a Zod reference in OpenClaw core. If the user wants a repo-supported path, use the existing `@openclaw/feishu` channel plugin.
