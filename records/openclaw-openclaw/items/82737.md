## Summary

Keep open: the PR targets a real TUI footer bug, but the new fallback can report cumulative input/output counters as live context-window usage, which conflicts with the existing session freshness contract.

## What This Changes

This PR adds a TUI footer token-total helper, routes the footer through it, adds formatter coverage, and adds a small repro script for missing usage counters.

## Best Possible Solution

Keep the fix at the TUI/session contract boundary: render an explicit fresh `totalTokens` when available, and preserve unknown or stale context usage instead of deriving it from cumulative input/output counters.

## Reproduction Assessment

Yes. Current main still has a source-reachable footer path where `formatTokens(null, 200000)` renders `tokens ?/200k`, and the linked issue documents the TUI receiving a known context window without a displayed total.

## Solution Assessment

No. The proposed helper is too broad because it treats `inputTokens + outputTokens` as a context snapshot; the safer solution is to propagate/use fresh context totals and keep stale or missing totals unknown.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
