## Summary

Keep open: the patch addresses the visible TUI `?/ctx` footer path, but it still turns explicitly stale or unknown session-list totals into fresh-looking `0%` usage, which conflicts with the existing `totalTokensFresh` contract.

## What This Changes

This PR adds a TUI footer token-total helper, wires the footer through it, and adds formatter tests plus a local repro script.

## Best Possible Solution

Thread `totalTokensFresh` through TUI session state and footer formatting so stale unknown totals stay unknown while genuine context-known zero/default cases can render `0/ctx`.

## Reproduction Assessment

Yes, source-level. Current main can pass a known context with a missing total into the TUI footer, and the PR-introduced stale-total regression is reachable from a `sessions.list` row with `totalTokensFresh=false`.

## Solution Assessment

No. The helper is the right narrow surface, but this version needs the existing freshness metadata before it can safely default missing totals to zero.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve stale unknown totals before zeroing the footer:** `src/tui/tui-formatters.ts:438-440`

Thread `totalTokensFresh` through TUI state and return `null` when it is explicitly false.
