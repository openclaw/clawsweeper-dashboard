## Summary

Keep open: current main still has the reported TUI footer `?/ctx` path, but this draft PR still collapses explicitly stale or unknown session totals into fresh-looking zero usage.

## What This Changes

The PR adds a TUI footer token-total helper, wires the footer through it, adds formatter tests, and adds a local repro script for context-known sessions with missing totals.

## Best Possible Solution

Thread `totalTokensFresh` through the TUI session types/state and make the footer show explicit fresh totals, `0/ctx` for non-stale context-known missing totals, and unknown for explicitly stale rows.

## Reproduction Assessment

Yes, source-reproducible. Current main sends a null total with known context to the footer formatter, and the PR-introduced stale-total case is reachable from `sessions.list` rows with `totalTokensFresh=false`, no fresh total, and known context.

## Solution Assessment

No. The helper is the right narrow surface, but it needs TUI freshness metadata before it can safely default missing totals to zero.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve stale unknown totals before zeroing footer:** `src/tui/tui-formatters.ts:438-439`
  - body: `resolveSessionFooterTokenTotal` returns `0` for every known-context row with no `totalTokens`, but `sessions.list` can explicitly report `totalTokensFresh=false` with no fresh total. Since TUI state does not carry that flag, stale/unknown context usage will render as fresh `tokens 0/ctx`; thread freshness through the TUI state and keep those rows unknown.
  - confidence: 0.9
