## Summary

Keep open: the cleaned-up PR is focused on the TUI footer, but the helper now converts every known-context missing total to zero, which drops the existing stale/unknown token freshness contract.

## What This Changes

This PR adds a TUI footer helper that formats known-context, missing-total sessions as `0/ctx`, wires it into the footer, and adds formatter tests plus a repro script.

## Best Possible Solution

Thread `totalTokensFresh` through the TUI session state and make the footer show `0/ctx` only for true context-known no-usage/default cases while keeping stale missing totals as `?/ctx`.

## Reproduction Assessment

Yes. Current main source shows the footer passes a null total with known context into `formatTokens`, and the PR-introduced stale-total regression is source-reachable because session storage can emit `totalTokensFresh=false` with no `totalTokens`.

## Solution Assessment

No. The footer helper is the right narrow surface, but it needs to carry freshness metadata instead of treating every missing total with a known context as zero.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve stale unknown token totals before zeroing the footer:** `src/tui/tui-formatters.ts:439`
  - body: This helper returns `0` whenever `contextTokens` is known and `totalTokens` is absent. Session rows can intentionally omit `totalTokens` when `totalTokensFresh=false` while still carrying context and cumulative counters; existing status rendering preserves that as `?/ctx`. Thread `totalTokensFresh` into TUI session state and keep stale rows unknown, while still showing `0/ctx` for the true context-only/no-usage case.
  - confidence: 0.87
