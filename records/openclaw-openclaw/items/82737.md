## Best Possible Solution

Keep the canonical issue open for a session-data fix that supplies or derives real fresh/transcript/input-output totals for the footer while preserving `?/<ctx>` when usage is genuinely unknown.

## Reproduction Assessment

Yes, at source level: current main can render `tokens ?/200k` when `formatTokens` receives a known context and a nullish total, and the TUI footer passes those nullable fields directly from `sessionInfo`. I did not run an interactive TUI session.

## Solution Assessment

No. The PR converts unknown context usage to zero in the display layer; the safer fix is to carry a real fresh/transcript-derived or input/output-derived total to the footer and leave unknown totals unknown.
