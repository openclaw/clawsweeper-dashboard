## Summary

Keep open: this is an active, focused fix for a source-reproducible TUI footer bug, but the current head still has a P2 correctness issue because it ignores `totalTokensFresh` and can render stale unknown usage as fresh zero usage.

## What This Changes

The PR adds a TUI footer token-total helper, routes footer formatting through it, and adds formatter tests plus a local proof script for context-known sessions with missing totals.

## Best Possible Solution

Thread `totalTokensFresh` from `sessions.list` into TUI session state and return `0/ctx` only when context is known and freshness is not explicitly stale; stale missing totals should remain unknown.

## Reproduction Assessment

Yes, at source level. Current main passes a null total with a known context into `formatTokens`, and the PR-introduced stale case is source-reproducible from a `sessions.list` row with `totalTokensFresh=false`, known context, retained input/output counters, and no fresh total.

## Solution Assessment

No. The helper is the right narrow surface, but the current implementation needs the existing freshness metadata before it can safely default missing totals to zero.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve stale missing totals in the footer:** `src/tui/tui-formatters.ts:438-439`
  - body: `resolveSessionFooterTokenTotal` returns `0` whenever `contextTokens` is known and `totalTokens` is absent, but `sessions.list` can also mark those totals as stale with `totalTokensFresh=false`. Since the TUI still drops that flag, stale rows with retained input/output counters render as fresh `tokens 0/ctx`; thread the freshness flag through TUI state and keep explicitly stale missing totals unknown.
  - confidence: 0.9
