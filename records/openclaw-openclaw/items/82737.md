## Summary

Keep open: the context-only footer fix addresses the linked TUI display bug, but the branch should not merge as-is because it can turn stale cumulative input/output counters into a misleading context-pressure percentage.

## What This Changes

The PR adds a TUI footer token-total helper, wires the footer through it, adds formatter tests, and adds a local repro script.

## Best Possible Solution

Land a narrowed TUI footer fix that uses fresh `totalTokens` or context-only zero, with regression coverage for stale usage rows.

## Reproduction Assessment

Yes. Source inspection shows current main passes a null total plus known context into `formatTokens`, which renders `tokens ?/200k`; the linked issue provides the user-facing TUI symptom.

## Solution Assessment

No. The context-only `0/ctx` direction is right, but the PR should skip or explicitly gate the input/output fallback because those counters can remain present when context totals are stale.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Skip cumulative input/output as context total:** `src/tui/tui-formatters.ts:438-439`
  - body: This derives a footer context total from `inputTokens + outputTokens` whenever `totalTokens` is absent. Current session rows can retain those cumulative counters while marking context totals stale with `totalTokensFresh=false`, and other UI paths suppress context pressure in that state, so this can render a misleading `tokens X/ctx` percentage. Limit this helper to explicit totals and the context-only zero fallback, or carry freshness through before using the sum.
  - confidence: 0.88
