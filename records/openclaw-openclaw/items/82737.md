## Summary

Keep open: this is a valid, focused TUI footer bug fix, but the current patch still drops the existing freshness signal and can render explicitly stale or unknown context usage as fresh `tokens 0/ctx`. The remaining repair is narrow and source-backed.

## What This Changes

The PR adds a TUI footer token-total helper, routes the footer token display through it, and adds formatter tests plus a repro script for context-known sessions with missing totals.

## Best Possible Solution

Keep the helper, but thread `totalTokensFresh` through TUI session state and tests so explicit stale/unknown rows stay unknown while non-stale context-only rows can render `0/ctx`.

## Reproduction Assessment

Yes, at source level. Current main passes a null total with a known context into `formatTokens`, which renders the reported `tokens ?/200k` footer shape.

## Solution Assessment

No, not as submitted. The helper is the right narrow surface, but it needs the existing freshness metadata before defaulting missing totals to zero.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve explicit stale totals before zeroing the footer:** `src/tui/tui-formatters.ts:438-439`
  - body: `resolveSessionFooterTokenTotal` returns `0` for any row with known `contextTokens` and absent `totalTokens`, but `sessions.list` can mark those totals as stale or unknown with `totalTokensFresh=false`. Since TUI state still drops that flag, the footer can render `tokens 0/ctx` for explicitly stale context usage; carry the freshness flag through and keep those rows unknown before defaulting to zero.
  - confidence: 0.9
