## Summary

This PR addresses a real TUI footer bug, but the latest diff is not merge-ready because the helper still turns stale/unknown context usage into `tokens 0/ctx` instead of preserving the existing `totalTokensFresh=false` contract.

## What This Changes

The PR adds a TUI footer token-total helper, wires the footer through it, and adds formatter tests plus a repro script for context-known sessions with missing totals.

## Best Possible Solution

Thread `totalTokensFresh` through TUI session state and return `0/ctx` only for context-known rows that are not explicitly stale, while keeping stale missing totals unknown.

## Reproduction Assessment

Yes. Current main source shows the reported footer path, and the PR-introduced stale-total regression is source-reproducible from a `sessions.list` row with `totalTokensFresh=false`, stale counters, known context, and no `totalTokens`.

## Solution Assessment

No. The helper is the right narrow surface, but the current version needs the existing freshness metadata before it can safely default missing totals to zero.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve stale unknown totals before zeroing the footer:** `src/tui/tui-formatters.ts:438-439`
  - body: `resolveSessionFooterTokenTotal` returns `0` for any row with known `contextTokens` and absent `totalTokens`, but `sessions.list` can also return stale rows with `totalTokensFresh=false`, stale counters, and no current total. Since TUI state does not carry that flag, this renders unknown context usage as fresh `tokens 0/ctx`; thread the freshness flag through and keep those rows unknown.
  - confidence: 0.9
