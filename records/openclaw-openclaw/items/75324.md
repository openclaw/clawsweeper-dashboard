## Summary

Keep open. Current main still sends Discord message-action discovery through strict Discord account/token resolution, so a raw SecretRef can still throw the reported unresolved SecretRef error; the issue is also paired with open closing PR #75424, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

Fix Discord SecretRef handling so a valid `channels.discord.token` SecretRef that passes `openclaw secrets audit` does not fail during Discord message-action discovery or runtime message operations.

## Best Possible Solution

Update Discord action discovery to use config-only account inspection so configured SecretRefs count as credentials without reading token values, keep strict token resolution for actual runtime auth/send paths, and add regression coverage for top-level and account-scoped Discord token SecretRefs.

## Reproduction Assessment

Yes. The issue provides concrete config and the current source reproduces the failure statically: raw SecretRef config can reach `describeDiscordMessageTool` -> `resolveDiscordAccount` -> `resolveDiscordToken` -> `normalizeResolvedSecretInputString`, which throws the reported error.

## Solution Assessment

Yes. The best current path is to review and land #75424 or an equivalent narrow fix; closing this issue before merge would lose the issue/PR pairing and a separate repair job would duplicate an open implementation candidate.
