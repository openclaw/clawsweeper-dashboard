## Summary

Keep open. Current main still routes Discord message-action discovery through strict Discord account/token resolution, so a raw SecretRef-backed token can still produce the reported unresolved SecretRef failure. The issue is also explicitly paired with open closing PR #75424, so the right cleanup is to review and land or close that PR before closing this issue.

## What This Changes

The issue reports that Discord runtime action discovery and message sends can encounter a raw `channels.discord.token` SecretRef even after `openclaw secrets audit` reports no unresolved refs.

## Best Possible Solution

Land #75424 or an equivalent narrowly reviewed fix that uses config-only Discord credential inspection for action discovery, preserves strict token resolution for actual runtime auth/send paths, and adds regression coverage for top-level and account-scoped Discord token SecretRefs.

Yes. The issue gives concrete config and the current source path is a high-confidence static reproduction: raw SecretRef config can flow through `describeDiscordMessageTool` to `resolveDiscordAccount`, `resolveDiscordToken`, and strict SecretRef normalization.
