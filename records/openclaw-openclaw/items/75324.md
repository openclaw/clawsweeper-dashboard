## Summary

Keep open. Current main still has the reported Discord SecretRef failure path in source, and #75424 is an open closing PR for the same issue, so closing this before that PR lands would be premature.

## What This Changes

The issue reports Discord message-action discovery and runtime message sends failing on unresolved `channels.discord.token` SecretRefs even after `openclaw secrets audit` reports them resolved.

## Best Possible Solution

Keep this issue open until #75424 or an equivalent fix lands. The end state should make Discord action discovery inspect configured credentials without resolving token values, while actual send/auth paths continue to resolve the selected Discord channel/account SecretRefs strictly before runtime use.
