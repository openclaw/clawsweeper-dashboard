## Summary

Keep open. Current main still strict-resolves the Discord token during message-action discovery, so a raw SecretRef can still reach the throwing token normalization path; the issue is also paired with open closing PR #75424, so it should not be closed before that PR or an equivalent fix lands.

## What This Changes

Discord message-action discovery and runtime message handling should support `channels.discord.token` SecretRefs consistently with `secrets audit` instead of failing with an unresolved SecretRef at runtime.

## Best Possible Solution

Keep this issue open until #75424 or an equivalent patch lands. The desired end state is Discord action discovery using config-only credential inspection for top-level and account-scoped SecretRefs, while actual send/auth paths keep strict scoped SecretRef resolution and regression coverage.
