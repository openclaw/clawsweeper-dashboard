## Summary

Keep open. The docs gap is real, but this PR should not merge as written because it presents a fresh agent session as an alternative after installing `gh`, while current main can keep a cached missing-binary result inside the running gateway process.

## What This Changes

The branch adds a new `After installing or authenticating gh` section to `skills/github/SKILL.md` with gateway restart/status commands and Linux/systemd PATH troubleshooting.

## Best Possible Solution

Keep the docs-only fix, but make gateway restart the required path after installing `gh` or changing the service PATH, and reserve fresh-session guidance for already-visible auth/config snapshot changes.

## Reproduction Assessment

Yes. Current main reproduces the documentation gap, and source plus test inspection shows the fresh-session alternative does not clear a cached missing binary in the running gateway process.

## Solution Assessment

No, not as written. A docs-only change is the right scope, but the wording should distinguish install/PATH changes from auth or session-snapshot changes so it matches the runtime contract.
