## Best Possible Solution

Land the token-independent full/off fast path while retaining explicit `requireSocket` mediation and the canonical mode-0600 approvals writer.

## Reproduction Assessment

Yes at source level: a persisted socket token prevents the documented full/off fast return and diverts node execution into socket-backed approval handling. I did not independently run the contributor's live node setup, so this is source-reproducible rather than directly reproduced.

## Solution Assessment

Yes. Removing only the token predicate is the narrowest fix because effective full/off policy requires no prompt, while explicit `requireSocket` callers still force socket initialization and canonical writes remain permission-hardened.
