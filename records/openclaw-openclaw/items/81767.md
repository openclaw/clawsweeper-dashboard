## What This Changes

The PR adds a payment-security cookbook page for SpendEnvelope and `before_tool_call` patterns, then adds a top-level Cookbook tab in `docs/docs.json`.

## Best Possible Solution

Rewrite this as a sourced, current-API docs recipe, preferably under the existing plugin hooks or capability-cookbook structure unless maintainers explicitly want a new Cookbook tab.

## Reproduction Assessment

Not applicable as a user bug, but the PR defects are high-confidence: CI reproduced the docs formatting failure, and current source/docs contradict the hook API used in the cookbook sample.

## Solution Assessment

No. The maintainable path is to ground the recipe in the current `api.on("before_tool_call")` contract, remove or qualify unverified security/package claims, and pass docs validation with real rendered or terminal proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use the plugin hook API that exists:** `docs/cookbook/payment-security-patterns.md:167-193`
  - body: The sample imports `BeforeToolCallHook` and `ToolCall` from `@openclaw/sdk` and treats the hook as `(call, ctx)` with `call.metadata` and `ctx.nonceStore`. Current OpenClaw plugin hooks use `api.on("before_tool_call", ...)` with `event.toolName`, `event.params`, and no runtime-provided nonce store, so readers cannot copy this example into a real plugin.
  - confidence: 0.94
- **[P2] Avoid endorsing an unverified payment plugin:** `docs/cookbook/payment-security-patterns.md:112-120`
  - body: This section tells payment-skill authors to install `@pqsafe/openclaw` and calls it a production-ready verifier for payment mandates. Since this is a payment authorization path, the docs should not endorse an external package until its package source, contract, support status, and threat model are verified or clearly framed as an unendorsed example.
  - confidence: 0.83
- **[P2] Do not describe unpublished standards as published:** `docs/cookbook/payment-security-patterns.md:299-321`
  - body: The page says the FIDO AP2-PQ profile was published on 28 April 2026, but this same section says the AP2-PQ RFC link will be added when it is formally published. Security guidance should either cite the published profile or describe the alignment as preliminary.
  - confidence: 0.86
- **[P3] Format the new cookbook page:** `docs/cookbook/payment-security-patterns.md:1`
  - body: The PR's `check-docs` job fails in `pnpm check:docs` at `format:docs:check` and names this new markdown file, so the docs gate remains red until the page is formatted with the repo's docs formatter.
  - confidence: 0.96
