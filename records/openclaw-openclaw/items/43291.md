## Summary

This PR targets a real current-main bug, but it needs changes before merge because the branch is stale against the current CLI prompt assembly path. Current main builds CLI system prompts inside prepareCliRunContext, while this PR forwards sessionKey at the older runCliAgent prompt-build call, so the cron/subagent heartbeat prompt leak would remain unfixed after the current code path is used.

## What This Changes

The PR adds session-key-aware minimal/full prompt selection to the CLI system-prompt helper, forwards sessionKey from runCliAgent, moves/re-exports a prompt-mode resolver, and adds helper-level tests.

## Best Possible Solution

Carry the cron/subagent prompt-mode decision through the current prepareCliRunContext or buildSystemPrompt path, preserve normal-session and systemPromptOverride behavior, and add prepare-level regression coverage before merging.

## Reproduction Assessment

Yes. Source tracing gives a high-confidence reproduction path: isolated cron passes a cron sessionKey to runCliAgent, prepareCliRunContext builds the CLI system prompt without sessionKey or promptMode, and buildAgentSystemPrompt defaults to full mode with the Heartbeats section.

## Solution Assessment

No. The PR direction is right, but the current diff wires sessionKey through an obsolete prompt-construction location; the narrower maintainable fix is to derive or pass promptMode in the current prepareCliRunContext prompt assembly path and test that path directly.
