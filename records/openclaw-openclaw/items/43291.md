## Summary

Keep open: the reported CLI-backend heartbeat leak is still source-reproducible on current main, but this PR no longer fixes the active prompt path because CLI system prompts are now built in prepareCliRunContext, not at the runCliAgent call this diff wires.

## What This Changes

The PR adds session-key-derived prompt-mode selection to CLI runner system prompt construction, moves the helper toward session-key routing, and adds helper tests for cron/subagent heartbeat suppression.

## Best Possible Solution

Carry session-key-derived promptMode through prepareCliRunContext/buildCliAgentSystemPrompt, preserve systemPromptOverride behavior, and add prepare-level regression coverage for cron and subagent CLI sessions.

## Reproduction Assessment

Yes. Source tracing shows isolated cron passes a cron sessionKey to runCliAgent, runCliAgent prepares the CLI context, and prepareCliRunContext builds a full system prompt with heartbeatPrompt but without promptMode.

## Solution Assessment

No. The intended direction is right, but the current diff wires sessionKey through an obsolete prompt-construction location instead of the active prepareCliRunContext path.
