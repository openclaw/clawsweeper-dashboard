## Summary

Keep open: this is a valid CLI-backend heartbeat prompt bug, but the branch no longer fixes the active current-main path and the external PR still lacks real behavior proof before merge.

## What This Changes

The branch adds session-key-derived minimal prompt selection to CLI prompt helper wiring, moves the prompt-mode helper toward shared session-key routing, and adds helper-level tests for cron/subagent heartbeat suppression.

## Best Possible Solution

Carry session-derived promptMode through `prepareCliRunContext` and `buildCliAgentSystemPrompt`, preserve `systemPromptOverride` behavior, and add prepare-level cron/subagent CLI prompt regression coverage.

## Reproduction Assessment

Yes. Source tracing shows isolated cron passes `params.runSessionKey` into `runCliAgent`, while `prepareCliRunContext` builds the active CLI system prompt without deriving `promptMode` from that key.

## Solution Assessment

No. The intended direction is right, but this PR wires `sessionKey` into an obsolete helper path instead of the active `prepareCliRunContext` prompt construction path.
