## Summary

## What This Changes

The PR forwards `sessionKey` into the old CLI prompt builder, shares cron/subagent prompt-mode resolution, and adds prompt tests for heartbeat suppression.

## Best Possible Solution

Update the fix on top of current main so the CLI backend derives minimal prompt mode from cron/subagent session keys in the active prepare/helper path, add focused regression coverage there, and provide redacted real CLI-backend run proof.

## Reproduction Assessment

Yes, at source level. A CLI backend run with a cron or subagent `sessionKey` and a resolved `heartbeatPrompt` still reaches `buildAgentSystemPrompt` without `promptMode: "minimal"`, so the default full mode can emit `## Heartbeats` and `HEARTBEAT_OK`.

## Solution Assessment

Yes in direction, but not as-is on the current branch. The maintainable fix is to apply the same prompt-mode contract to the current CLI prepare/helper path rather than the older `runCliAgent` prompt-building call site.
