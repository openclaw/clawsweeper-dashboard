## Summary

Keep open: the underlying cron/subagent CLI-backend heartbeat bug is source-reproducible on current main, but this PR does not fix the active prompt-construction path and still lacks after-fix real behavior proof.

## What This Changes

The branch adds session-key-based minimal prompt selection to the CLI runner system-prompt helper, forwards the key from runCliAgent, moves the prompt-mode helper into routing/session-key, and adds helper tests.

## Best Possible Solution

Retarget the fix to `prepareCliRunContext` and `buildCliAgentSystemPrompt` so cron/subagent CLI sessions use minimal prompt mode and avoid heartbeat guidance, then add prepare-level regression coverage and redacted real CLI-backend proof.

## Reproduction Assessment

Yes, source-reproducible: the cron executor passes a cron session key into `runCliAgent`, while current main prepares the active CLI system prompt without deriving `promptMode` from that key. A live cron run was not executed in this read-only review.

## Solution Assessment

No. The intended policy is right, but the patch updates an obsolete run-level helper path instead of deriving prompt mode where current main actually prepares CLI system prompts.
