## Summary

Keep open: the bug is source-reproducible and the intended policy is valid, but this PR is still aimed at an obsolete run-level prompt helper path and lacks required real behavior proof from the contributor.

## What This Changes

The PR adds session-key-based minimal prompt mode selection to the CLI runner system prompt helper, forwards `sessionKey` from `runCliAgent`, moves prompt-mode resolution toward routing/session-key, and adds helper tests for cron/subagent heartbeat suppression.

## Best Possible Solution

Derive session-based prompt mode in `prepareCliRunContext`/`buildCliAgentSystemPrompt`, preserve `systemPromptOverride`, add prepare-level cron/subagent regression coverage, and collect redacted real CLI-backend behavior proof.

## Reproduction Assessment

Yes, source-reproducible. `createCronPromptExecutor` passes `params.runSessionKey` into `runCliAgent`, while current `prepareCliRunContext` builds the active CLI system prompt without deriving `promptMode` from that key; no live cron run was executed in this read-only review.

## Solution Assessment

No. The intended policy is correct, but this PR forwards `sessionKey` into an obsolete helper/call-site path instead of deriving prompt mode where the active CLI system prompt is prepared.
