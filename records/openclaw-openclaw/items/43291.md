## Summary

Keep this PR open, but it is not merge-ready. Current main still has the CLI-backend cron/subagent heartbeat prompt bug, while the branch targets an old prompt-construction call instead of the active prepareCliRunContext path.

## What This Changes

The PR adds session-key-aware minimal/full prompt selection to CLI system prompt construction, shares the prompt-mode resolver, forwards the session key from the CLI runner, and adds helper-level prompt tests.

## Best Possible Solution

Carry the cron/subagent session-key prompt-mode decision through the current CLI prepare flow, preserve normal-session and systemPromptOverride behavior, and cover the active prepareCliRunContext path with regression tests.

## Reproduction Assessment

Yes. Static current-main tracing is high-confidence: isolated cron passes a cron sessionKey into runCliAgent, runCliAgent delegates to prepareCliRunContext, and prepareCliRunContext builds a full prompt without passing sessionKey or promptMode.

## Solution Assessment

No. The intended direction is right, but the patch wires sessionKey through an obsolete prompt-construction site and tests only the helper; the safer fix is to derive or pass promptMode in prepareCliRunContext and test that active path.
