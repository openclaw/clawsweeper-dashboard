## Summary

Keep open: the bug is valid and the PR direction is useful, but the branch still misses current main's active CLI prompt-construction path and lacks after-fix real behavior proof.

## What This Changes

The branch adds session-key-derived prompt-mode selection to the CLI prompt helper, forwards `sessionKey` from `runCliAgent`, moves the helper toward routing/session-key, and adds helper tests.

## Best Possible Solution

Carry session-key-derived `promptMode` through `prepareCliRunContext` and `buildCliAgentSystemPrompt`, preserve `systemPromptOverride` behavior, and cover cron/subagent CLI prompts at the prepare-level path.

## Reproduction Assessment

Yes, by source tracing. Isolated cron creates an agent-scoped cron run session key and passes it into `runCliAgent`, while the active prepare path still omits `promptMode`, causing the prompt builder to default to full mode.

## Solution Assessment

No. The helper-level approach was reasonable before the CLI runner refactor, but current main needs the fix in `prepareCliRunContext`/`buildCliAgentSystemPrompt` rather than the obsolete `runCliAgent` prompt-construction call.
