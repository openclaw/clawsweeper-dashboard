## Summary

Keep open: the underlying CLI-backend heartbeat prompt bug remains source-reproducible on current main, but this branch wires the obsolete helper path and only provides unit-test proof.

## What This Changes

The branch adds session-key-derived minimal prompt selection to CLI system prompt helper wiring, moves `resolvePromptModeForSession` toward routing/session-key, and adds helper tests for cron/subagent heartbeat suppression.

## Best Possible Solution

Carry session-derived `promptMode` through `prepareCliRunContext` and `buildCliAgentSystemPrompt`, preserve `systemPromptOverride`, and add prepare-level cron/subagent CLI prompt regression coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes. Source tracing shows isolated cron passes `params.runSessionKey` into `runCliAgent`, while `prepareCliRunContext` builds the active CLI system prompt without deriving `promptMode` from that key; no live cron run was executed in this read-only review.

## Solution Assessment

No. The intended policy is correct, but this PR wires `sessionKey` into an obsolete helper path instead of the current prepare-time CLI prompt construction path.
