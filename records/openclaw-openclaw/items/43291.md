## Summary

Keep open: the reported bug is still credible, but this PR is not mergeable as-is because it wires sessionKey into a stale prompt-construction path while current main builds CLI system prompts in prepareCliRunContext; it also lacks real behavior proof.

## What This Changes

The branch adds sessionKey-derived minimal prompt selection to CLI prompt helper construction, moves the prompt-mode helper toward session-key routing, forwards sessionKey from runCliAgent, and adds helper tests.

## Best Possible Solution

Carry session-key-derived promptMode through prepareCliRunContext/buildCliAgentSystemPrompt, preserve systemPromptOverride behavior, and add prepare-level cron/subagent CLI prompt regression coverage.

## Reproduction Assessment

Yes. Source tracing shows isolated cron passes a cron sessionKey into runCliAgent, while prepareCliRunContext builds a full CLI system prompt without deriving minimal prompt mode from that key.

## Solution Assessment

No. The intended direction is right, but this PR wires sessionKey into an obsolete prompt-construction location instead of the active prepareCliRunContext path.
