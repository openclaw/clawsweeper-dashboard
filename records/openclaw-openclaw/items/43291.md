## What This Changes

The branch adds session-key-derived minimal prompt selection to CLI system prompt helper wiring, moves prompt-mode resolution toward routing/session-key, and adds helper tests for cron/subagent heartbeat suppression.

## Best Possible Solution

Derive session-based prompt mode in `prepareCliRunContext`/`buildCliAgentSystemPrompt`, preserve `systemPromptOverride`, add prepare-level cron/subagent regression coverage, and collect redacted real CLI-backend behavior proof.

## Reproduction Assessment

## Solution Assessment

No. The intended policy is correct, but this PR wires `sessionKey` into an obsolete helper/call-site path instead of deriving prompt mode where the active CLI system prompt is prepared.
