## Summary

Keep this PR open, but it is not merge-ready. The reported cron/subagent CLI-backend prompt bug still exists on current main, and the PR’s intended fix targets an old prompt-construction site instead of the active prepareCliRunContext path.

## What This Changes

The PR adds session-key-aware minimal/full prompt selection to the CLI system-prompt helper, forwards the session key from runCliAgent, shares the embedded prompt-mode resolver, and adds helper-level tests.

## Best Possible Solution

Carry the cron/subagent prompt-mode decision through the active CLI preparation flow, preserve normal-session and systemPromptOverride behavior, and add regression coverage that exercises prepareCliRunContext rather than only the lower-level helper.

## Reproduction Assessment

Yes. Static tracing gives a high-confidence reproduction path: isolated cron passes a cron sessionKey into runCliAgent, runCliAgent calls prepareCliRunContext, and that active prompt build still uses full mode unless explicitly changed.

## Solution Assessment

No. The PR’s direction is right, but the branch wires sessionKey through an obsolete prompt-build location; the safer fix is to derive or pass promptMode in prepareCliRunContext and cover that active path with tests.
