The branch adds session-key-derived minimal prompt mode to the CLI runner helper, forwards `sessionKey` from `runCliAgent`, moves `resolvePromptModeForSession` toward `routing/session-key`, and adds helper tests.

## Best Possible Solution

Derive session/trigger-based heartbeat suppression in the active CLI prepare/system-prompt path, preserve `systemPromptOverride`, add prepare-level regression coverage, and collect redacted real CLI-backend proof.

## Solution Assessment

No. The intended policy is right, but the branch should be retargeted to the active prepare/helper path instead of the stale direct `buildSystemPrompt` call site.
