## What This Changes

The issue reports local memory embeddings breaking across upgrades because the extracted llama.cpp provider was not migrated/installed cleanly, error text was misleading, local model identity could mismatch existing indexes, and CLI indexing may not match gateway plugin behavior.

## Best Possible Solution

Settle the local embedding upgrade contract, then either add a tested doctor migration or normalization for equivalent old local model identities, or document and prove the intentional rebuild-only behavior with clear CLI/gateway guidance.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only pass.
