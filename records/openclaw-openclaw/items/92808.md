## Summary

Keep open: current main and v2026.6.6 now have local-provider guidance and doctor repair for the extracted llama.cpp plugin, but the reported upgrade compatibility around local model identity is still unresolved and needs maintainer/live-upgrade proof.

## What This Changes

The issue asks OpenClaw to preserve or repair local embedding provider upgrades, surface accurate diagnostics, and provide a reliable local memory reindex path after the llama.cpp provider moved to a plugin.

## Best Possible Solution

Settle the local embedding upgrade contract, then either add a tested doctor migration or normalization for equivalent old local model identities, or keep rebuild-only behavior with clearer upgrade guidance and live CLI/gateway proof.

No high-confidence live reproduction was established in this read-only pass. Source inspection gives a medium-confidence path for the remaining problem: stored index meta.model still mismatches when the current local provider model identity changes between an hf: URI and a filesystem path.
