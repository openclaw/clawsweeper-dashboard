## Summary

Keep open: the security hardening is still useful, but this branch applies it to stale `src/web` WhatsApp files that current main no longer uses; the active `extensions/whatsapp` readers still follow symlinks and the PR lacks real behavior proof.

## What This Changes

The PR adds lstat-based symlink rejection and `0o700`/chmod auth-directory hardening to the old `src/web` WhatsApp credential/session files.

## Best Possible Solution

Port the hardening into the active `extensions/whatsapp` credential and session paths, centralize lstat-based credential reads, and add focused symlink and directory-mode regression tests.

## Reproduction Assessment

Yes. Source inspection on current main is enough: a symlinked `creds.json` with a regular-file target is followed by `stat`/`readFile` in `extensions/whatsapp/src/auth-store.ts` and `statSync` in `extensions/whatsapp/src/creds-files.ts`; no live run was performed in this read-only review.

## Solution Assessment

No. The PR addresses the right hardening goal, but it applies the change to stale `src/web` files; the maintainable solution is to move the lstat rejection and directory mode/chmod hardening into `extensions/whatsapp` with regression coverage.
