## Summary

Keep open: the hardening is useful, but the branch applies it to stale `src/web` paths while current main's active WhatsApp plugin still has credential reads and session setup that the PR does not affect.

## What This Changes

The PR adds symlink rejection and auth-directory mode/chmod hardening to WhatsApp credential/session files under `src/web`.

## Best Possible Solution

Retarget the hardening to `extensions/whatsapp` credential helpers and session setup, with focused symlink and directory-mode regression coverage.

## Reproduction Assessment

Yes. By source and dependency proof, current main's active credential readers call `fs.stat`/`statSync` and `readFile`/`readFileSync` on `creds.json`, and a Node probe confirmed those calls follow symlinks while `lstat` detects them.

## Solution Assessment

No. The helper approach is reasonable in isolation, but applying it under removed `src/web` files is not the maintainable fix; the checks need to cover the active `extensions/whatsapp` paths.
