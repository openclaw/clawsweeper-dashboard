## Summary

Keep open: the MCP readiness feature is not on current main, but the PR still has a readiness precedence bug, a smoke-script cwd bug, missing real behavior proof, and conflicts with newer main changes.

## What This Changes

This PR adds MCP metadata parsing for skills, filters prompt-advertised MCP-backed skills through readiness files, adds readiness prompt tests, and introduces an mcporter readiness smoke script while touching adjacent gateway and qa-channel setup plumbing.

## Best Possible Solution

Land a rebased MCP readiness gate only after skill-level readiness records are authoritative, the smoke script uses the correct global cwd, and freshness/owner semantics are documented or intentionally omitted.

## Reproduction Assessment

Yes. From PR-head source inspection, a readiness file with skills[name].callable false and servers[server].callable true reaches the server fallback in isMcpSkillRuntimeReady and would advertise the skill.

## Solution Assessment

No. The readiness-gate direction is plausible, but this patch is not mergeable until the precedence and smoke cwd defects are fixed and the branch is rebased against current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Short-circuit skill readiness denials:** `src/agents/skills/workspace.ts:174-180`
  - body: When a readiness file has a skill-level record that is not callable for this agent, the new code still checks the server record and can return true. That lets a server-level grant advertise a skill whose own entry denies callability; treat an existing skill record as authoritative before falling back to the server.
  - confidence: 0.94
- **[P2] Use home cwd for global mcporter config:** `scripts/mcp-readiness-smoke.mjs:58-60`
  - body: The global ~/.mcporter/mcporter.json case falls back to the OpenClaw home as cwd before running mcporter config list. That verifies global mcporter from the state directory instead of the user's home context, so use os.homedir() for the non-workspace branch.
  - confidence: 0.86
