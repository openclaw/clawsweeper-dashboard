## Summary

Current main still lacks MCP readiness metadata/gating for advertised skills, but the PR head is not correct yet: explicit skill-level denials can be bypassed, the smoke script does not invoke mcporter's JSON mode correctly, the global config cwd is wrong, and a required changelog entry is missing.

## What This Changes

The PR adds MCP metadata parsing and readiness-file filtering for MCP-backed skill advertisements, focused skill tests, a mcporter readiness smoke script, and adjacent gateway/QA-channel CI fixes.

## Best Possible Solution

Merge a rebased readiness gate where skill-level readiness records are authoritative, mcporter smoke verification uses the supported CLI contract and correct cwd, user-visible behavior is documented in the changelog, and CI is green.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. ## Solution Assessment

Is this the best way to solve the issue? No. The product direction is plausible, but the current patch is not the best merge shape until denial precedence, smoke verification, changelog coverage, and stale-branch cleanup are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Make skill readiness records authoritative:** `src/agents/skills/workspace.ts:175-180`
  - body: When a readiness file has a skill-level record that is not callable, this code falls through to the server record and can still return true. That lets `servers[server].callable:true` advertise a skill whose own readiness entry explicitly denies callability; treat an existing skill record as authoritative before checking server fallback.
  - confidence: 0.95
- **[P2] Use mcporter's JSON flag:** `scripts/mcp-readiness-smoke.mjs:40`
  - body: `mcporter config list` supports `--json`, not `--output json`; upstream parses this invocation as a text-mode filter. As written, `--verify-callable` does not get the machine-readable server list the smoke needs, so the readiness mismatch check can silently verify the wrong thing.
  - confidence: 0.9
- **[P2] Use home cwd for global mcporter config:** `scripts/mcp-readiness-smoke.mjs:59-62`
  - body: The global `~/.mcporter/mcporter.json` row runs mcporter from OpenClaw home. That is a different cwd from the home-level mcporter config path, so use `os.homedir()` for the non-workspace branch while keeping workspace config checks rooted in their workspace.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/agents/skills/workspace.ts:871-875`
  - body: This changes user-visible skill advertisement behavior by hiding MCP-backed skills until readiness says they are callable, but the PR does not update `CHANGELOG.md`. Add a single-line entry under the active release section before merge.
  - confidence: 0.82
