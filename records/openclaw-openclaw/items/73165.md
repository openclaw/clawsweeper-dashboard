## Summary

Keep open: current main still lacks MCP readiness gating for advertised skills, and this PR is useful but not merge-ready because the head still has a readiness-precedence bug, a smoke-script cwd bug, and no after-fix real behavior proof.

## What This Changes

The PR adds MCP metadata parsing for skills, gates prompt-advertised MCP-backed skills on readiness files, adds readiness tests, and introduces a mcporter readiness smoke script while touching adjacent gateway and qa-channel setup plumbing.

## Best Possible Solution

Keep the PR open, repair the precedence and smoke cwd defects, add focused denial coverage, provide redacted real-output proof, and rebase away adjacent churn already covered on main.

## Reproduction Assessment

Yes. From source inspection of the PR head, a readiness file with skills[name].callable false and servers[server].callable true reaches the server fallback and would advertise the skill.

## Solution Assessment

No. The readiness-gate direction is plausible, but the patch is not the best mergeable form until skill records are authoritative over server fallback and the global mcporter smoke path uses the user's home cwd.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Short-circuit skill readiness denials:** `src/agents/skills/workspace.ts:174-180`
  - body: When a readiness file has a skill-level record that is not callable for this agent, the new code still checks the server record and can return true. That lets a server-level grant advertise a skill whose own entry denies callability; treat an existing skill record as authoritative before falling back to the server.
  - confidence: 0.94
- **[P2] Use home cwd for global mcporter config:** `scripts/mcp-readiness-smoke.mjs:58-60`
  - body: The global ~/.mcporter/mcporter.json case falls back to the OpenClaw home as cwd before running mcporter config list. That verifies global mcporter from the state directory instead of the user's home context, so use os.homedir() for the non-workspace branch.
  - confidence: 0.86
