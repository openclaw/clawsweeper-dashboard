## Summary

Keep open. Current main has not shipped MCP readiness gating, and the PR still has blocking logic and smoke-script issues: explicit skill-level denials can be overridden by server-level readiness, and global mcporter smoke verification runs from the OpenClaw home instead of the user home.

## What This Changes

The PR adds MCP metadata parsing and readiness-file gating for MCP-backed advertised skills, a mcporter readiness smoke script, focused skills tests, and adjacent CI/helper changes around gateway prewarm and qa-channel setup.

## Best Possible Solution

Merge a rebased, narrower readiness gate where skill-level readiness records are authoritative, denial precedence has regression coverage, the smoke cwd is correct, and the user-facing behavior has a changelog entry.

## Reproduction Assessment

Yes. On PR head, a readiness file with a skill-level callable:false record and a server-level callable:true record reaches the server fallback in isMcpSkillRuntimeReady and advertises the skill.

## Solution Assessment

No. The core direction is plausible, but this patch is not the best merge shape until explicit skill records win over server fallback and the mcporter smoke script uses the correct global cwd.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Short-circuit skill readiness denials:** `src/agents/skills/workspace.ts:176-180`
  - body: When a readiness file has a skill-level record that is not callable, this code falls through to the server record and can still return true. That lets servers[server].callable:true advertise a skill whose own readiness entry explicitly denies callability; treat an existing skill record as authoritative before checking the server fallback.
  - confidence: 0.94

  - confidence: 0.86

  - confidence: 0.82
