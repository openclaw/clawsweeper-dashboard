## Summary

Keep open. Current main still does not implement the MCP readiness gate for advertised skills, and the PR remains useful but not merge-ready because the proposed gate can bypass an explicit skill-level denial and the new smoke script verifies the global mcporter config from the wrong working directory.

## What This Changes

The PR adds MCP skill metadata parsing, reads `mcp-readiness.json`, filters MCP-backed advertised skills until readiness says they are callable, adds focused skills tests, and adds an opt-in mcporter readiness smoke script with adjacent test and boundary updates.

## Best Possible Solution

Keep the PR open as useful source work, but merge only a narrower or rebased version where skill-level readiness records are authoritative, denial precedence has regression coverage, the smoke script uses the correct cwd for global mcporter config, and readiness freshness semantics are documented or intentionally left out.

## Reproduction Assessment

Yes. On PR head, a readiness file with `skills["mcp-memory"] = { declared: true, callable: false }` and `servers["memory"] = { declared: true, callable: true }` reaches the server fallback at `src/agents/skills/workspace.ts:179` and returns `true`.

## Solution Assessment

No. The core readiness gate is a plausible direction, but this patch is not the best merge shape until explicit skill-level denials win, the smoke-script cwd bug is fixed, and stale unrelated branch changes are split or rebased.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Short-circuit skill readiness denials:** `src/agents/skills/workspace.ts:176-180`
  - body: When a readiness file has a skill-level record that is not callable, this code falls through to the server record and can still return `true`. That lets `servers[server].callable: true` advertise a skill whose own readiness entry explicitly says it is not callable; treat an existing skill record as authoritative before checking the server fallback.
  - confidence: 0.94
- **[P2] Use home cwd for global mcporter config:** `scripts/mcp-readiness-smoke.mjs:59-62`
  - body: The script is iterating the global `~/.mcporter/mcporter.json` entry, but the non-workspace branch runs `mcporter config list` from OpenClaw home. That makes global smoke verification run from a different working directory than the config it reports, so use `os.homedir()` for the global row.
  - confidence: 0.86
