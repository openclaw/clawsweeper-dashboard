## What This Changes

The PR adds MCP metadata parsing and readiness-file gating for prompt-advertised skills, a dry-run mcporter readiness smoke script, focused skills tests, and adjacent gateway, qa-channel, and package-boundary updates.

## Best Possible Solution

Keep the PR open as useful source work, but merge only a rebased or replacement patch where skill-level readiness records are authoritative, the mcporter smoke script uses the correct cwd for user-home config, regression coverage locks denial precedence, and stale adjacent gateway/qa-channel changes are preserved or split away.

## Reproduction Assessment

Yes. On PR head, a readiness file with a non-callable skill record and a callable matching server record follows the server fallback at src/agents/skills/workspace.ts:179 and advertises the skill.

## Solution Assessment

No. The readiness-gate direction is plausible, but this patch is not the best merge shape until explicit skill-level records short-circuit server fallback and the smoke script uses os.homedir() for the user-home config row.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Short-circuit skill readiness denials:** `src/agents/skills/workspace.ts:176-180`
  - body: When a readiness file has a skill-level record that is not callable, this code falls through to the server record and can still return true. That lets a callable server record advertise a skill whose own readiness entry explicitly says it is not callable; treat an existing skill record as authoritative before checking server fallback.
  - confidence: 0.94
- **[P2] Use home cwd for global mcporter config:** `scripts/mcp-readiness-smoke.mjs:59-62`
  - body: The user-home mcporter config row runs mcporter config list from OpenClaw home. That makes global smoke verification use a different working directory than the config it reports, so use os.homedir() for the non-workspace row.
  - confidence: 0.86
