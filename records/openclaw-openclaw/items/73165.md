Keep open. Current main still lacks MCP readiness gating for advertised skills, and the PR head has a denial-precedence bug plus a smoke-script cwd mismatch that should be repaired before merge.

## What This Changes

The PR adds MCP skill metadata/readiness parsing, gates prompt-advertised MCP-backed skills on readiness files, adds a mcporter readiness smoke script, and adjusts adjacent gateway/qa-channel test/build plumbing.

## Best Possible Solution

Repair and rebase the PR so skill-level readiness entries are authoritative, global mcporter smoke uses the correct cwd, focused regression coverage and changelog are added, and obsolete adjacent churn is dropped.

## Reproduction Assessment

Yes, from source inspection of the PR head. A readiness file with skills[name].callable false and servers[server].callable true reaches the server fallback and would advertise the skill.

## Solution Assessment

No. The direction is plausible, but the patch is not merge-ready until explicit skill records win over server fallback and the smoke script uses the right global cwd.

Full review comments:

- **[P1] Short-circuit skill readiness denials:** `src/agents/skills/workspace.ts:174-180`
  - body: When a readiness file has a skill-level record that is not callable, this code falls through to the server record and can still return true. That lets servers[server].callable:true advertise a skill whose own readiness entry explicitly denies callability; treat an existing skill record as authoritative before checking the server fallback.
  - confidence: 0.94

  - body: The global ~/.mcporter/mcporter.json row runs mcporter config list from the OpenClaw home directory. That makes global smoke verification run from a different cwd than the global mcporter config expects, so use os.homedir() for this non-workspace branch.
  - confidence: 0.86

  - body: This changes user-visible skill advertisement behavior by hiding MCP-backed skills until readiness says they are callable, but the PR does not update CHANGELOG.md. Add a single-line entry under the active release section before merge.
