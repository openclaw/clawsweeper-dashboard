## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Short-circuit skill readiness denials:** `src/agents/skills/workspace.ts:175-180`
  - body: When a skill-level readiness record exists but is not callable, this code falls through to the server record and can still return true. That lets servers[server].callable:true advertise a skill whose own readiness entry explicitly denies callability; treat an existing skill record as authoritative before checking the server fallback and add regression coverage.
  - confidence: 0.94
- **[P2] Use home cwd for global mcporter config:** `scripts/mcp-readiness-smoke.mjs:59-62`
  - body: The global ~/.mcporter/mcporter.json path currently runs mcporter config list from OPENCLAW_HOME/root. That is not the cwd a global mcporter config expects, so use os.homedir() for this non-workspace branch while keeping workspace configs rooted at their workspace.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/agents/skills/workspace.ts:871-875`
  - body: This changes user-visible skill advertisement behavior by hiding MCP-backed skills until readiness says they are callable, but the PR head has no CHANGELOG.md entry for the behavior. Add a single-line entry under the active release section before merge.
  - confidence: 0.82
