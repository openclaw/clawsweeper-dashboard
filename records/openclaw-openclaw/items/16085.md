## Summary

Keep open: current main still lacks bbernhard REST/WebSocket Signal support, and this PR is the canonical implementation candidate, but it is merge-conflicting, lacks inspectable real container proof, and still has a credentialed URL logging bug in the new container client.

## What This Changes

The PR adds a Signal native/container adapter, a bbernhard REST/WebSocket client, `channels.signal.apiMode`, docs, tests, a changelog fragment, config baseline updates, and a plugin-local `ws` dependency.

## Best Possible Solution

Update the existing branch to reject or sanitize credentialed container base URLs, rebase it cleanly, keep docs/config aligned, and attach redacted live bbernhard container proof before maintainer review/merge.

## Reproduction Assessment

Yes. Source inspection shows main only uses native `/api/v1/rpc`, `/api/v1/check`, and `/api/v1/events`, while bbernhard docs show `/v2/send` plus WebSocket `/v1/receive/<number>`; I did not run a live container in this read-only review.

## Solution Assessment

No, not yet. The plugin-local adapter direction is maintainable, but this branch should fix the credential logging path, resolve conflicts, and include inspectable live container proof before merge.
