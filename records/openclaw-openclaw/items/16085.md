## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reject credentialed Signal container base URLs:** `extensions/signal/src/client-container.ts:170`
  - body: The native Signal client rejects `httpUrl` values with username/password, but the new container path accepts them and builds the receive log URL from the full authority. With `http://user:pass@host`, this line can still print credentials; parse and reject credentialed base URLs or build the log URL from a sanitized URL.
  - confidence: 0.93
- **[P3] Format group ids before container typing calls:** `extensions/signal/src/client-container.ts:477-479`
  - body: When replying in a Signal group, `sendTypingSignal()` passes `params.groupId` into container mode, but this fallback sends the raw group id as `recipient`. The bbernhard client only treats values with the `group.` prefix as groups, so group typing indicators are sent as direct recipients and can fail; use `formatGroupIdForContainer(groupId)` here.
  - confidence: 0.86
