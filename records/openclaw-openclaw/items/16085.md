## Summary

Keep open. Current main still lacks Signal container REST/WebSocket support, and the latest PR head remains a useful implementation candidate, but it is not mergeable as-is because two concrete regressions remain in the proposed container client.

## What This Changes

The PR adds a Signal native/container adapter, a bbernhard REST/WebSocket container client, Signal caller rewiring, `channels.signal.apiMode` config/docs/tests/changelog coverage, and a direct `ws` runtime dependency for the Signal plugin.

## Best Possible Solution

Land a plugin-local Signal transport adapter that preserves native JSON-RPC/SSE behavior, parses bbernhard REST responses correctly, avoids logging account identifiers in normal logs, keeps config/schema/docs aligned, and ships after targeted Signal tests, changed-gate proof, and live container smoke or maintainer waiver.

## Reproduction Assessment

Yes. The original user problem is source-verifiable against current main because OpenClaw only calls native `/api/v1/*` endpoints, while bbernhard container support requires `/v2/send` and WebSocket `/v1/receive/{number}`. The PR regressions are also statically reproducible from the latest PR source.

## Solution Assessment

No. The adapter direction is plausible, but the current branch is not the best mergeable solution until it preserves `/v2/send` 201 response timestamps and redacts the account-bearing WebSocket URL log.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Parse Created send responses:** `extensions/signal/src/client-container.ts:118-120`
  - body: The bbernhard `/v2/send` contract returns `201 Created` with a JSON timestamp, but `containerRestRequest` returns `undefined` for every 201 before reading the body. Container sends will therefore report `messageId: "unknown"` even when the API returned the real timestamp; only no-content statuses should skip parsing.
  - confidence: 0.9
- **[P3] Redact the WebSocket receive URL:** `extensions/signal/src/client-container.ts:173`
  - body: This logs the full receive URL in normal gateway logs, including the Signal account embedded in `/v1/receive/{account}`. Log a redacted endpoint or move the full URL behind verbose diagnostics so routine startup logs do not expose account identifiers.
  - confidence: 0.84
