## Summary

Keep open. Current main still lacks bbernhard REST/WebSocket Signal container support, so this is not closable as implemented; the PR is a real implementation candidate for the linked container-support request, but the current head is dirty and has blocking correctness, security, config-boundary, and schema issues before merge.

## What This Changes

The PR adds a Signal native/container adapter, a bbernhard REST/WebSocket client, caller rewiring, `channels.signal.apiMode` docs/types/tests, a changelog fragment, and a direct `ws` runtime dependency.

## Best Possible Solution

Ship a plugin-local Signal transport adapter that preserves native JSON-RPC/SSE behavior, adds documented bbernhard REST/WebSocket mode through accepted SDK runtime/config seams, aligns strict schema/generated metadata/docs, removes sensitive logging, and has targeted tests plus live container proof or an explicit maintainer waiver.

## Reproduction Assessment

Yes. Current main can be checked from source against a bbernhard container path mismatch: OpenClaw uses `/api/v1/rpc`, `/api/v1/check`, and `/api/v1/events`, while upstream container docs use `/v2/send` and WebSocket `/v1/receive/<number>`; this read-only review did not run a live container smoke.

## Solution Assessment

No. The adapter direction is plausible, but this branch is not the best mergeable form until the native SSE timeout regression, startup retry behavior, raw logging, SDK boundary imports, schema/generated metadata, dirty merge state, and live-proof gaps are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Preserve the idle SSE timeout override:** `extensions/signal/src/sse-reconnect.ts:53-65`
  - body: Current main keeps quiet native Signal SSE streams open by passing `timeoutMs: 0` from the monitor. This PR's `runSignalSseLoop` call no longer accepts or forwards a timeout, so native mode falls back to the 10s default through the adapter and would reintroduce idle reconnect churn for healthy `/api/v1/events` streams.
  - confidence: 0.91
- **[P1] Keep startup Signal checks retryable:** `extensions/signal/src/client-adapter.ts:107-111`
  - body: `signalCheck()` now resolves auto mode before returning. While an auto-started native daemon is still booting, both probes can fail and `detectSignalApiMode()` throws; `waitForTransportReady()` does not catch exceptions from its check callback, so startup can fail immediately instead of polling until the daemon is ready.
  - confidence: 0.9
- **[P1] Remove raw Signal payload logging:** `extensions/signal/src/client-container.ts:202`
  - body: `streamContainerEvents` logs the first 200 characters of every inbound WebSocket frame before parsing it. Those frames can contain private Signal message text plus sender, group, and attachment metadata, so normal container receive would write sensitive content to gateway logs.
  - confidence: 0.94
- **[P2] Use the accepted config seam:** `extensions/signal/src/client-adapter.ts:9`
  - body: This production bundled plugin file imports the broad `openclaw/plugin-sdk/config-runtime` barrel and calls ambient `loadConfig()`. Repo guards reject that broad subpath and ambient config access in bundled plugin production code; pass the loaded config or use the accepted runtime config seam instead.
  - confidence: 0.91
- **[P2] Import fetch helpers from the narrow SDK subpath:** `extensions/signal/src/client-container.ts:11`
  - body: The new container client imports `resolveFetch` from the broad `openclaw/plugin-sdk/infra-runtime` barrel. The bundled-plugin import guard rejects that broad subpath, and existing plugins use `openclaw/plugin-sdk/fetch-runtime` for this helper.
  - confidence: 0.88
- **[P2] Validate the apiMode setting:** `src/config/types.signal.ts:68`
  - body: `apiMode` is added to the public Signal config type and docs, but the strict `SignalConfigSchema` still does not accept this key. A user setting `channels.signal.apiMode: "container"` can fail config validation before the adapter sees it.
  - confidence: 0.92
