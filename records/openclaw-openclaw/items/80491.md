## Summary

Keep open: the PR targets a valid linked bug, but the new routing branch would bypass existing media-directive preservation for phase-tagged final answers and the external PR still lacks sufficient after-fix real behavior proof.

## What This Changes

The PR changes the embedded run payload builder to prefer phase-filtered assistant text when the last assistant message contains phase-tagged text blocks.

## Best Possible Solution

Keep the phase-aware routing direction, but route through the existing media-preserving final-answer source and add focused regression coverage for single-entry raw streamed commentary plus phase-tagged final answers.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence helper-level path for both the linked bug and this PR regression: construct `buildEmbeddedRunPayloads` with a single raw streamed assistant text and a phase-tagged `lastAssistant`; the proposed branch then changes which final source is delivered.

## Solution Assessment

No. The PR's direction is narrow and appropriate, but the inserted branch is ordered before the existing raw-media fallback and should preserve that contract while adding tests.
