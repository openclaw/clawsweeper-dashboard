- `impact:security`: The reported failure crosses a sensitive information boundary by exposing internal thinking text to chat users.

## Summary

Keep open: current main and the latest release still use a narrower streaming thinking-tag scanner than the final sanitizer, so attribute-bearing thinking tags can reach user-visible block delivery. Related broader reasoning-default and interleaved-stream issues remain open, but they do not fully supersede this focused streaming sanitizer gap.

## What This Changes

The issue asks OpenClaw to stop sending agent internal thinking/planning text in user-visible chat replies after the 2026.4.9 upgrade.

## Best Possible Solution

Unify streaming block-reply thinking-tag parsing with the canonical final sanitizer, add focused coverage for attributed and split thinking tags before channel delivery, and keep broader reasoning-default policy in https://github.com/openclaw/openclaw/issues/73182.

## Reproduction Assessment

Yes. Source inspection shows current main's streaming scanner accepts fewer thinking-tag shapes than the final sanitizer, so an attributed thinking tag can bypass streaming cleanup before channel delivery.

## Solution Assessment

No for closing this issue. Current main and v2026.6.6 still leave the focused streaming sanitizer gap; the best fix is to share or align the reasoning-tag parser used by streaming and final cleanup.
