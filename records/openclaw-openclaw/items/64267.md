## Summary

Keep open: current main and the latest release still leave the focused streaming sanitizer gap for attribute-bearing thinking tags, even though bare and namespaced tags are covered elsewhere.

## What This Changes

The issue reports internal reasoning/thinking text leaking into user-visible channel replies after OpenClaw 2026.4.9.

## Best Possible Solution

Keep this focused sanitizer issue open until the streaming parser is aligned with the canonical reasoning-tag grammar and covered for attributed tags across chunk boundaries.

## Reproduction Assessment

Yes, source-level: current `main` and `v2026.6.8` still have a streaming scanner that only accepts bare thinking tags, while final-message sanitization accepts attributed tags. A live WeChat run was not performed in this read-only review.

## Solution Assessment

Yes for the narrow leak: the best fix is to align or share the streaming scanner with the canonical reasoning-tag parser and add focused streaming tests. Broader default reasoning policy belongs in https://github.com/openclaw/openclaw/issues/73182.
