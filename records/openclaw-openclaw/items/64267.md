## Summary

Keep open: current main and the latest release still use a narrower streaming thinking-tag scanner than the final sanitizer, so attributed thinking tags can still reach user-visible channel delivery. The broader reasoning-default issues are related but do not fully supersede this specific streaming sanitizer gap.

## What This Changes

The issue reports internal agent thinking/planning text being exposed to chat users after upgrading OpenClaw, with discussion tracing it to streaming reasoning-tag cleanup.

## Best Possible Solution

Unify streaming block-reply thinking-tag parsing with the canonical final sanitizer and add focused coverage for attributed and split thinking tags before channel delivery.

## Reproduction Assessment

Yes. Source inspection shows current main's block-reply streaming scanner accepts fewer thinking-tag shapes than the final sanitizer, so attributed thinking tags can bypass streaming cleanup before channel delivery.

## Solution Assessment

No for closing this issue: current main and v2026.6.6 still leave the focused streaming sanitizer gap. The best fix is to share or align the reasoning-tag parser used by streaming and final cleanup.
