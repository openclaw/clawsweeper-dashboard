## Summary

Keep open: current main and the latest release still leave a source-level embedded block-streaming sanitizer gap for attribute-bearing thinking tags, while the shared final sanitizer and partitioner already accept those tags.

## What This Changes

The issue reports internal agent reasoning/thinking text being sent to users after upgrading to OpenClaw 2026.4.9, with WeChat as the observed channel and multiple models affected.

## Best Possible Solution

Align embedded block-streaming tag stripping with the shared reasoning-tag grammar or partitioner, add streaming coverage for attributed and split tags, and keep provider default policy in https://github.com/openclaw/openclaw/issues/73182.

## Reproduction Assessment

Yes, source-reproducible: current main's embedded block-streaming scanner does not recognize attribute-bearing opening thinking tags while the shared sanitizer does. I did not run a live WeChat scenario.

## Solution Assessment

Yes for the narrow leak: the maintainable fix is to share or align the streaming scanner with the canonical reasoning-tag parser and add focused streaming tests. Broader default reasoning policy belongs in https://github.com/openclaw/openclaw/issues/73182.
