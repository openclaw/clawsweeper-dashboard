- `P1`: Hidden assistant reasoning can still reach live channel replies, which is an urgent user-facing privacy and channel workflow failure.
- `impact:security`: The issue is about preserving the boundary between hidden assistant reasoning and user-visible channel content.

## Summary

Keep open: current main has partially addressed bare and namespaced thinking tags, but the streaming scanner still does not match the shared final sanitizer for attributed thinking tags, so a source-level user-visible leak path remains.

## What This Changes

The issue asks OpenClaw to stop exposing hidden assistant thinking/planning text in user-visible channel replies after the 2026.4.9 upgrade.

## Best Possible Solution

Use one shared, code-region-aware reasoning/final-tag parser for streaming delivery and final sanitization, then add focused regression tests and redacted real channel proof for attributed, namespaced, split-chunk, and malformed cases.

## Reproduction Assessment

Yes, at source level: current main's streaming scanner misses attributed thinking openings while the final sanitizer recognizes them, leaving an early streamed chunk eligible for user-visible delivery. I did not establish a live WeChat reproduction in this read-only pass.

## Solution Assessment

No current main is not complete: the namespaced-tag fix was useful, but the parser contract is still split between streaming and final sanitization. The best fix is a shared parser boundary, not a WeChat-specific filter.
