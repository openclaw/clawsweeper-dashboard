## Summary

Keep open: the reported leak is still source-reproducible on current main and the latest shipped tag because block-reply streaming scans a narrower reasoning-tag shape than the shared final sanitizer, so attributed reasoning tags can reach channel delivery before final cleanup.

## What This Changes

The issue reports OpenClaw 2026.4.9 leaking hidden agent thinking/planning text into user-visible channel replies across multiple providers.

## Best Possible Solution

Keep this open until block-reply streaming uses the same attribute-aware reasoning parser as final sanitization, with regression coverage for attributed namespaced tags across chunk boundaries and real channel proof.

## Reproduction Assessment

Yes, from source: an attributed tag such as `<thinking hidden='1'>` is accepted by the final sanitizer but not by the block-reply streaming scanner before emit. I did not run a live channel reproduction because this review is read-only.

## Solution Assessment

Yes, the best fix is to align the streaming block-reply parser with the shared attribute-aware reasoning parser and add focused regression coverage. A reasoning-default policy change should stay separate from this parser repair.
