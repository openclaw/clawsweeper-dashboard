## Summary

Keep open: current main and v2026.5.7 still have a source-level streaming path where attributed thinking tags are not recognized before channel-visible delivery, while the shared sanitizer treats those same tags as hidden.

## What This Changes

Investigate and fix OpenClaw leaking hidden agent thinking or planning text into user-visible channel replies after 2026.4.9.

## Best Possible Solution

Centralize or share a safe reasoning/final tag parser across embedded streaming, extraction, promotion, and final sanitization, with regression coverage for attributed, namespaced, self-closing, malformed, code-span, and split-chunk cases.

## Reproduction Assessment

Yes, at source level. The current streaming scanner misses attributed thinking openings, and a split chunk beginning with `<think hidden="1">internal` remains deliverable before the later close tag is processed.

## Solution Assessment

Yes, the right direction is a core parser/sanitizer fix rather than a WeChat-specific patch. Maintainers should reconcile the overlapping parser PRs and keep the strict suppression path fail-closed for malformed tags.
