## Summary

Keep open. Current main and the latest shipped tag still route embedded block streaming through a reasoning-tag scanner that only recognizes bare tags; the shared final sanitizer already treats attributed and namespaced tags as hidden, so attributed `<think>` or `<antml:thinking>` openings can still leak during streaming before final cleanup.

## What This Changes

The issue asks OpenClaw to stop exposing agent internal reasoning/planning text in user-facing channel replies after the reporter upgraded to OpenClaw 2026.4.9.

## Best Possible Solution

Align every embedded streaming, extraction, and tag-promotion helper with the shared reasoning-tag grammar, including attributes, `antml:` prefixes, case-insensitivity, code-span handling, and partial chunk buffering; then add focused streaming and promotion regression tests for attributed tags.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a streamed assistant delta beginning with `<thinking hidden="1">` or `<antml:thinking hidden="1">` before the closing tag; current main misses that opening tag in the streaming scanner while the final sanitizer recognizes it.

## Solution Assessment

Yes. The narrow maintainable fix is in core sanitizer utilities and embedded streaming tests; patching the external WeChat plugin would be the wrong boundary because the block reply text can already be leaked before channel delivery.
