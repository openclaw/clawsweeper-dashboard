## Summary

Keep open: current main and v2026.5.12 still have a narrower embedded streaming thinking-tag scanner than the shared final sanitizer, so attributed thinking tags can pass through block streaming before final cleanup suppresses them.

## What This Changes

The issue reports user-visible internal thinking/planning text after upgrading to OpenClaw 2026.4.9, with discussion tracing it to a mismatch between streaming and final reasoning-tag sanitizers.

## Best Possible Solution

Use one shared, fail-closed reasoning/final-tag parser across streaming delivery and final sanitization, with regression coverage for attributed, namespaced, malformed, code-span, and split-chunk cases.

## Reproduction Assessment

Yes, at source level: the current streaming scanner misses attributed thinking openings while the shared final sanitizer recognizes them, leaving early block chunks eligible for delivery. I did not establish a live WeChat reproduction in this read-only pass.

## Solution Assessment

Yes, the best direction is a core parser/sanitizer alignment rather than a WeChat-specific filter. The open related parser work should be reconciled with strict, fail-closed delimiter handling before this issue is closed.
