## Summary

Keep open: current main and the latest release still leave a source-level block-streaming sanitizer gap for attribute-bearing thinking tags, while the broader final sanitizer already accepts those tags; the related default-reasoning issue only partially overlaps this concrete parser mismatch.

## What This Changes

The issue asks OpenClaw to stop exposing internal agent thinking/planning text in user-visible channel replies, especially after the 2026.4.9 upgrade path.

## Best Possible Solution

Align embedded block-streaming reasoning-tag parsing with the shared reasoning-tag grammar or partitioner, add attributed and split-tag streaming coverage, and keep provider default policy in the related issue.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.8 still use a block-streaming scanner that misses attributed thinking tags, while final-message sanitization accepts them. A live WeChat run was not performed in this read-only review.

## Solution Assessment

Yes for the narrow leak: the best fix is to share or align the streaming scanner with the canonical reasoning-tag parser and add focused streaming tests. Broader default reasoning policy belongs in https://github.com/openclaw/openclaw/issues/73182.
