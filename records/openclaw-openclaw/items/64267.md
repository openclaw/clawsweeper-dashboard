## Summary

Keep open: current main and the latest release still have a source-level embedded streaming path where attributed thinking tags are not recognized before channel-visible delivery, while the shared final sanitizer treats those same tags as hidden. The issue discussion already identified this mismatch, and the related open parser PRs focus on overlapping final-tag work rather than a merged, safe fix for this exact thinking-tag suppression gap.

## What This Changes

The issue reports hidden agent reasoning text being delivered to users during streamed channel responses, especially after upgrading around OpenClaw 2026.4.9 with the WeChat plugin.

## Best Possible Solution

Centralize or share a fail-closed reasoning/final tag parser across embedded streaming, extraction, promotion, and final sanitization, with regression coverage for attributed, namespaced, self-closing, malformed, code-span, and split-chunk cases.

## Reproduction Assessment

Yes, at source level. The current streaming scanner misses attributed thinking openings such as `<think hidden="1">`, and a chunk that begins with that text remains eligible for delivery before a later close tag is processed.

## Solution Assessment

Yes, the best direction is a core parser/sanitizer fix rather than a WeChat-specific filter. The implementation should be reconciled with the open parser PRs and keep strict suppression fail-closed for malformed tags.
