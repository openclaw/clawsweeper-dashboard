## Summary

Keep open: current main and v2026.6.10 still have a source-level mismatch where the embedded streaming scanner does not accept attribute-bearing thinking tags, while the shared final sanitizer already strips them; this remains a core privacy/security bug, not just a WeChat plugin issue.

## What This Changes

The issue reports that OpenClaw 2026.4.9 can send internal agent thinking/planning text to chat users, especially through streaming channel delivery.

## Best Possible Solution

Unify the embedded streaming scanner with the shared reasoning-tag grammar, including attributes and known namespaces, then add focused regression tests for streamed attribute-bearing tags across chunk boundaries.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main's embedded streaming scanner misses attribute-bearing thinking tags while the shared final sanitizer and tests accept them. I did not run a live WeChat channel reproduction in this read-only pass.

## Solution Assessment

Yes, the best fix belongs in the core embedded/shared sanitizer boundary rather than in the WeChat plugin, because channels can receive already-streamed visible text from this path.
