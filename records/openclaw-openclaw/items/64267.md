## Summary

Keep open: current main and v2026.5.12 still have an embedded streaming scanner that misses attributed thinking tags before channel-visible block reply delivery, while the shared final sanitizer treats those tags as hidden.

## What This Changes

The issue reports hidden agent reasoning text leaking into user-visible chat responses after the OpenClaw 2026.4.9 upgrade path.

## Best Possible Solution

Use one shared, fail-closed reasoning/final-tag parser across streaming delivery and final sanitization, with regression coverage for attributed, namespaced, malformed, code-span, and split-chunk cases.

## Reproduction Assessment

Yes, at source level. The current streaming regex misses attributed thinking openings while the shared final sanitizer recognizes them, leaving an early stream chunk eligible for delivery before a later close tag or final sanitizer can suppress it.

## Solution Assessment

Yes, the best direction is a core parser/sanitizer alignment rather than a WeChat-specific filter. The fix should be reconciled with the overlapping parser and delivery-normalization PRs and should keep malformed or split reasoning tags fail-closed.
