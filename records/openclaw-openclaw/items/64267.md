## Summary

Keep this open. Current main still has the core streaming sanitizer mismatch identified in the discussion: final-message sanitization accepts attributed and namespaced reasoning tags, but the streaming/block scanner only recognizes bare tags, so attributed openings can be emitted before the closing tag or final sanitizer catches up.

## What This Changes

The issue asks OpenClaw to stop exposing agent internal reasoning/thinking text in user-visible channel replies after the 2026.4.x upgrade, with openclaw-weixin as the reported channel surface.

## Best Possible Solution

Align every embedded streaming and tag-promotion helper with the shared reasoning-tag grammar, including attributes, `antml:` prefixes, case-insensitivity, and partial chunk handling, then add focused regression coverage for attributed tags on the streaming and promotion paths.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a streamed assistant delta that starts with `<thinking hidden="1">...` or `<antml:thinking hidden="1">...` before the closing tag; current `THINKING_TAG_SCAN_RE` does not match the opening tag, so `stripBlockTags` can emit that text before final sanitization.

## Solution Assessment

Yes, a narrow core sanitizer fix is the best path. Patching openclaw-weixin would be the wrong boundary because the WeChat plugin is external and the leak path is in core block streaming before channel delivery.
