## Label Justifications

- `impact:security`: The report crosses a sensitive information boundary by sending internal thinking text to chat users.

## Summary

Keep open: current main and v2026.6.10 still leave a source-level block-streaming sanitizer mismatch for attributed thinking tags, while final-message sanitization already handles those tags. The related reasoning-default and channel-specific issues overlap, but they do not replace this narrow core streaming parser gap.

## What This Changes

The issue asks OpenClaw to stop exposing agent internal thinking/planning text in user-visible channel replies after the 2026.4.9 upgrade.

## Best Possible Solution

Align embedded block-streaming reasoning-tag parsing with the shared reasoning-tag grammar or partitioner, add attributed and split-tag streaming coverage, and keep provider default policy in the related issue.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.10 still use a block-streaming scanner that misses attributed opening tags while final-message sanitization accepts them. A live WeChat run was not performed in this read-only review.

## Solution Assessment

Yes for the narrow leak: the maintainable fix is to share or align the streaming scanner with the canonical reasoning-tag parser and add focused streaming tests. Broader default reasoning policy belongs in https://github.com/openclaw/openclaw/issues/73182.
