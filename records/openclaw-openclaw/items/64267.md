## Summary

Keep open: current main and the latest release still have a source-level leak path because embedded streaming scans only bare reasoning tags while the final sanitizer treats attributed and namespaced tags as hidden.

## What This Changes

Investigate and prevent agent thinking/planning text from being delivered to users across models and chat channels.

## Best Possible Solution

Unify embedded streaming, extraction, and promotion with the shared reasoning-tag grammar, then add regression coverage for attributed, namespaced, self-closing, code-span, and split-chunk cases on current main.

## Reproduction Assessment

Yes, at source level. A streamed assistant delta beginning with an attributed reasoning tag such as `<think hidden="1">internal</think>Visible` will miss the opening tag in the embedded streaming scanner while final cleanup treats that tag shape as hidden.

## Solution Assessment

Yes. The narrow maintainable repair is to share or mirror the canonical reasoning-tag grammar in the embedded streaming and promotion helpers, not to patch individual channel plugins such as WeChat.
