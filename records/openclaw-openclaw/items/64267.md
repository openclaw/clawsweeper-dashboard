## Summary

Keep open: current main still has a source-level leak path where attributed thinking tags can be streamed before the final sanitizer sees the complete message. The related PRs are open/partial and still need maintainer reconciliation, so this is not implemented or safely superseded yet.

## What This Changes

The issue reports hidden agent reasoning text leaking into user-visible channel replies after upgrading OpenClaw.

## Best Possible Solution

Use one shared, constrained, fail-closed reasoning/final-tag parser across streaming delivery and final sanitization, with regression coverage for attributed, namespaced, malformed, code-span, and split-chunk cases.

## Reproduction Assessment

Yes, at source level: current main’s streaming scanner misses attributed thinking openings while the final sanitizer recognizes them, leaving an early streamed chunk eligible for user-visible delivery. I did not establish a live WeChat reproduction in this read-only pass.

## Solution Assessment

Yes, the best direction is a core parser/sanitizer alignment rather than a WeChat-specific filter. The open related parser work should be reconciled with strict delimiter handling and real channel proof before this issue is closed.
