## Label Justifications

- `P1`: Hidden assistant reasoning can still reach live channel replies, which is an urgent privacy and user-facing channel workflow failure.
- `impact:security`: The issue is about preserving the boundary between hidden assistant reasoning and user-visible channel content.

## Summary

Keep open: current main and the latest shipped release still have a source-level path where attributed thinking tags can bypass the streaming scanner even though final sanitization recognizes them, so hidden reasoning can be delivered before final cleanup.

## What This Changes

The issue reports hidden agent reasoning text leaking into user-visible channel replies after upgrading to OpenClaw 2026.4.9, with WeChat observed and multiple models affected.

## Best Possible Solution

Use one shared, code-region-aware reasoning-tag parser for streaming and final delivery, accept namespaced and attributed tags consistently, and prove the fix with focused tests plus redacted real channel output.

## Reproduction Assessment

Yes, at source level: current main's streaming scanner misses attributed thinking openings while the final sanitizer recognizes them, leaving early streamed chunks eligible for user-visible delivery. I did not run a live WeChat reproduction in this read-only pass.

## Solution Assessment

No, current main is not complete. The useful antml fix should stay, but the maintainable fix is to share the final sanitizer's parser contract with streaming rather than keep separate regexes drifting.
