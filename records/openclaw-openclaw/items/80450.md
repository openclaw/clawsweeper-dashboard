## What This Changes

The issue requests an explicit final-summary ownership contract for detached or migrated long-running subagent work so the full completion is delivered by exactly one owner surface.

## Best Possible Solution

Settle the owner-mode contract first, then land a focused implementation that preserves current default requester/thread behavior, persists the selected owner, documents the public API, and proves only the selected owner receives the full final summary.
