## Summary

Keep open: current main has requester-origin/thread-bound completion routing and an internal session-key ownership seam, but no explicit final-summary owner mode contract. The linked implementation path is the open PR at https://github.com/openclaw/openclaw/pull/80544, so this issue should stay open until maintainers review, land, or reject that design.

## What This Changes

Request an explicit final-summary ownership contract for detached or migrated long-running subagent work so only one surface publishes the full completion and child identity survives migration.

## Best Possible Solution

Review and either land a compatibility-safe version of https://github.com/openclaw/openclaw/pull/80544 or define a narrower native subagent completion-owner contract that preserves legacy requester/thread routes and makes ACP scope explicit.

## Reproduction Assessment

Not applicable as a strict bug reproduction. The issue reports a real Discord longwork migration observation but explicitly lacks a minimal upstream runtime test; source inspection confirms the missing first-class owner contract rather than a narrow failing path.

## Solution Assessment

Unclear until maintainer API review. A narrow core subagent spawn/registry/delivery contract looks like the right direction, but it must avoid duplicating the existing `completionOwnerKey` session-key seam and preserve legacy routing behavior.
