## Summary

Keep open: current main and v2026.6.6 still have requester-session-key completion routing, not the explicit final-summary owner contract requested here. The linked implementation PR was closed unmerged, so the remaining work is maintainer API/product review rather than cleanup closure.

## What This Changes

Request to add an explicit final-summary ownership contract for detached or migrated long-running subagent work so only one surface publishes the full final summary.

## Best Possible Solution

Settle the owner-mode contract first, then land a focused implementation that preserves the default requester/thread behavior, documents the public API, persists the owner safely, and proves one full final reaches only the selected owner.

## Reproduction Assessment

No high-confidence current-main reproduction is available: the issue says there is no minimal upstream runtime test case, and I did not reproduce the Discord migration scenario. Source inspection does confirm the requested owner-mode contract is absent on current main and v2026.6.6.

## Solution Assessment

Unclear: a first-class final-summary ownership contract is the right class of solution, but the exact model-facing/plugin-facing shape still needs maintainer decision. The prior broad PR was closed unmerged, so the best path is a smaller recut after API/product signoff.
