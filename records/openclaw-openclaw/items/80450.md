## Summary

Keep open: current main still lacks a first-class final-summary owner for detached or migrated subagent work. The related open PRs cover adjacent same-turn duplicate delivery and hook metadata, not this multi-surface delivery policy.

## What This Changes

Request a persisted final-delivery ownership contract for detached or migrated longwork so only one surface publishes the full child completion.

## Best Possible Solution

Define and persist a subagent delivery contract at spawn or migration time with completion-owner and bridge metadata, then enforce one full-final owner while allowing short pointer/status notes elsewhere.

## Reproduction Assessment

No, not as a bug reproduction. The issue is a feature/design request and explicitly says there is no minimal upstream runtime test case yet; source inspection confirms the relevant routing surfaces and missing owner contract.

## Solution Assessment

Unclear. The proposed completion-owner shape is plausible, but the exact modes, bridge-back semantics, and migration payload need a core delivery design decision before implementation.
