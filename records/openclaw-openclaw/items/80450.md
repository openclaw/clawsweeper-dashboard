## Summary

Keep open: current main and v2026.6.8 still only have requester-session-key completion routing, not the explicit final-summary owner-mode contract requested here. The linked implementation PR was closed unmerged, so the remaining work is maintainer API/product review rather than cleanup closure.

## What This Changes

The issue requests a first-class detached longwork final-summary ownership contract so exactly one surface owns the full child completion while other surfaces receive only pointer/status updates.

## Best Possible Solution

Settle the owner-mode contract first, then land a focused implementation that preserves current default requester/thread behavior, persists the selected owner, documents the public API, and proves only the selected owner receives the full final summary.

## Reproduction Assessment
