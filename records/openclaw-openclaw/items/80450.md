## Summary

Keep open: current main has requester/session-based subagent completion routing and a recent `completionOwnerKey` helper, but not the explicit final-summary ownership modes requested here. The linked implementation PR is still open and uses closing syntax, so this issue should stay open until that PR is reviewed, landed, or intentionally closed.

## What This Changes

Request an explicit detached longwork completion-ownership contract so exactly one surface owns the full final subagent summary across origin, work, and update threads.

## Best Possible Solution

Review the linked native completion-ownership PR as the implementation path, or replace it with a maintainer-approved contract that persists final-owner mode and enforces one full final summary across announce delivery and retry state.

## Reproduction Assessment

No, not as a bug reproduction. This is a feature/design request, the issue says there is no minimal upstream runtime test case, and the current source check only verifies the missing owner-mode contract.

## Solution Assessment

Unclear until maintainer review of the open implementation PR. The proposed mode-based contract fits the affected subagent routing surface, but mode names, ACP scope, bridge semantics, and non-owner pointer behavior need architecture review before this issue can close.
