## Summary

Keep open: current main has requester-origin routing and direct/queued subagent completion delivery, but no persisted single final-summary owner contract. The related pull requests are adjacent and still open, so this is a fresh core delivery-policy design request rather than cleanup-closeable work.

## What This Changes

Request an explicit final-summary ownership contract for detached or migrated long-running subagent work so only one surface publishes the full completion.

## Best Possible Solution

Define a core subagent/task delivery contract that persists final-owner and bridge metadata at spawn or migration time, carries exact child identity, and makes announce delivery, hooks, docs, and regression tests enforce one full final-summary owner with pointer/status notes elsewhere.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a feature/design request, and the issue explicitly says there is no minimal upstream runtime test yet. Source inspection confirms the adjacent routing surfaces exist, but not a high-confidence current-main failing scenario.

## Solution Assessment

Unclear: the proposed completion-owner contract is plausible and belongs near the core subagent/task delivery path, but the exact owner modes and bridge semantics need maintainer design before a narrow fix PR is safe.
