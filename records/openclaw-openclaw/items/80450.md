## Summary

Keep open: current main has subagent completion routing, requester origin, direct/queued dispatch, and child-session lookup, but I found no explicit single final-summary owner contract. This is a fresh core delivery-policy feature request, not an implemented cleanup or duplicate of the adjacent open work.

## What This Changes

The issue requests a first-class final-summary ownership contract for detached or migrated long-running work across origin, work, and update threads.

## Best Possible Solution

Define a core subagent/task delivery contract that persists final-owner and bridge metadata at spawn or migration time, then make announce delivery, hooks, and regression tests enforce one full final-summary owner while allowing short pointer/status notes elsewhere.

Not applicable as a strict bug reproduction: this is a feature/design request, and the issue explicitly says there is no minimal upstream runtime test yet. Source inspection confirms related routing surfaces exist, but not a high-confidence current-main failing scenario.
