## Summary

Keep open: current main has subagent requester routing, completion delivery hooks, and durable pending-final replay, but no explicit single final-summary owner contract for detached or migrated longwork. The two linked PRs remain open and address adjacent hook metadata and same-turn duplicate delivery, not this cross-surface ownership policy.

## What This Changes

Make detached or migrated long-running subagent work persist a single final-summary owner so full completions are not delivered to multiple surfaces.

## Best Possible Solution

Define and persist a core subagent final-delivery ownership contract at spawn or migration time, then enforce one full-summary owner across announce delivery, hooks, and regression coverage.

## Reproduction Assessment

No, not as a bug reproduction. This is a feature/design request, the issue says there is no minimal upstream runtime test case yet, and source inspection only confirms the missing owner contract.

## Solution Assessment

Unclear. The proposed completion-owner shape is plausible and belongs near the subagent spawn/delivery contract, but the concrete modes and bridge semantics need maintainer design before implementation.
