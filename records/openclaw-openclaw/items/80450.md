## Summary

Keep open: current main still lacks an explicit final-summary ownership contract, and the open implementation PR that references this issue with closing syntax has not merged.

## What This Changes

Detached or migrated long-running subagent work should have one explicit owner for the full final summary so multiple surfaces do not duplicate completion delivery.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/80544 after maintainer approval and live proof, or replace it with an approved core subagent delivery contract that enforces exactly one full-final owner.

## Reproduction Assessment

Not applicable as a strict bug reproduction. This is a feature/design request, and the issue body says there is no minimal upstream runtime test case yet; source inspection confirms only that current main lacks the owner contract.

## Solution Assessment

Unclear. An explicit owner contract is the right class of solution and the open PR is the current candidate, but the owner vocabulary and bridge semantics need maintainer approval before this issue should close.
