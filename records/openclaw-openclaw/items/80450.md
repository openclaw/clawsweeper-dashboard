## Summary

Keep open: current main has subagent completion routing, requester-origin delivery, hooks, retry state, and child-session lookup, but no explicit single final-summary owner contract. The related open PRs address adjacent same-turn duplicate delivery and hook metadata, not this multi-surface delivery policy.

## What This Changes

Request an explicit final-summary ownership contract for detached or migrated long-running subagent work so only one surface publishes the full completion summary.

## Best Possible Solution

Define a core subagent/task delivery contract that persists final-owner and bridge metadata at spawn or migration time, then enforce one full final-summary owner across announce delivery, hooks, and regression tests while allowing short pointer/status notes elsewhere.

## Reproduction Assessment

No, not as a bug reproduction. This is a feature/design request and the issue explicitly says there is no minimal upstream runtime test case yet; source inspection only confirms the relevant routing surfaces and missing owner contract.

## Solution Assessment

Unclear. The proposed completion-owner shape is plausible and belongs near the core subagent/task delivery contract, but the exact modes and bridge semantics need maintainer design before implementation.
