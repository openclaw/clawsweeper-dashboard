## Summary

Keep open for normal maintainer review: current main still has the SDK default-thinking mismatch, and this PR is a focused fix with no blocking code findings found. The remaining review concern is merge-time compatibility/proof, because it intentionally changes the SDK no-config fallback for providers that declare thinking off by default and does not include a live Ollama round-trip.

## What This Changes

The branch makes `createAgentSession` resolve provider-specific thinking defaults, adds SDK default tests, and extends thinking-default coverage for provider `defaultLevel: "off"`.

## Best Possible Solution

Merge the provider-default SDK fix once maintainers accept the fallback behavior change, with explicit `thinkingLevel` and settings defaults remaining the opt-in path for users who want Ollama thinking enabled.

## Reproduction Assessment

Yes for a source-level path: current main defaults SDK sessions to `medium`, and the Ollama wrapper forwards non-off thinking as native `think`. A live current-main Ollama Gemma/Qwen repro was not run in this read-only review.

## Solution Assessment

Yes, this looks like the right layer: the SDK session factory should consume the same provider thinking policy used by TUI/auto-reply defaults. Changing only the Ollama stream wrapper would leave the SDK default inconsistent with provider policy.
