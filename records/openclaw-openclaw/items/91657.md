## Summary

Keep open: the branch targets a real Ollama SDK thinking-default bug, but it is not merge-ready because the new provider-default path can turn provider `defaultLevel: "adaptive"` into `"off"` through the SDK clamp instead of preserving the previous usable default.

## What This Changes

The branch changes `createAgentSession` to resolve provider-specific thinking defaults, adds SDK coverage for provider/default precedence and legacy sessions, and extends thinking-helper coverage for provider `defaultLevel: "off"`.

## Best Possible Solution

Keep the Ollama provider-default fix, but preserve or explicitly support non-SDK levels such as `adaptive` before using provider defaults in `createAgentSession`, with regression coverage for both Ollama `off` and adaptive-provider defaults.

## Reproduction Assessment

Yes, by source inspection: current `main` hard-codes the SDK fallback to `medium`, while Ollama declares provider default `off` and injects native `think` when the runtime thinking level is non-off. I did not run the live Ollama 0.9 round trip in this read-only review.

## Solution Assessment

No, not as written: using provider defaults in the SDK is the right direction for Ollama, but the implementation casts broader provider defaults through a narrower SDK level type and can turn adaptive defaults into `off`.

- **[P1] Do not clamp adaptive provider defaults to off:** `src/agents/sessions/sdk.ts:275-280`

Preserve the prior SDK fallback for unsupported defaults or add full adaptive support plus a regression test.
