## Summary

Keep open: the Ollama SDK-session fix is still needed, but this patch still has a blocking compatibility regression because it can turn provider defaults such as `adaptive` into `off` for SDK-created sessions.

## What This Changes

The PR makes SDK session creation use `resolveThinkingDefaultForModel` for the model-specific fallback and adds SDK/thinking tests for Ollama `defaultLevel: "off"` behavior.

## Best Possible Solution

Keep the provider-default fix, but make the SDK fallback handle the full provider `ThinkLevel` contract so Ollama `off` works without clamping `adaptive` defaults to `off`, and cover both cases in SDK tests.

## Reproduction Assessment

Yes at source level: current main uses the global SDK `DEFAULT_THINKING_LEVEL`, while the Ollama provider policy declares `defaultLevel: "off"` for reasoning models. I did not run a live Ollama round trip in this read-only review.

## Solution Assessment

No: using the provider default is the right direction, but this implementation casts the broader provider `ThinkLevel` into the narrower Agent `ThinkingLevel` and clamp path. The safer repair is a small bridge that preserves adaptive defaults plus an SDK regression test.
