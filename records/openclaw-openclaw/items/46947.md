The PR changes `ui/src/styles/chat/sidebar.css` so sidebar markdown inline images use the same 420px width and 320px height caps as main chat markdown images.

## Best Possible Solution

Land this CSS cap or an equivalent sidebar markdown image constraint after contributor-provided visual proof and current green checks, then leave any separate extended-session or virtual-scroll image growth behavior on the linked issue if it still reproduces.

## Reproduction Assessment

## Solution Assessment

Is this the best way to solve the issue? Yes for the known sidebar mismatch: matching the existing main-chat image cap in the sidebar selector is narrow and keeps renderer behavior unchanged. It still needs real after-fix proof and current validation before merge.
