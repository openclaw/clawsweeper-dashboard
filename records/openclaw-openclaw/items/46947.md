## Summary

Current main still has the loose sidebar markdown image sizing this PR targets, while the main chat image rule already uses the proposed cap; the patch is narrow and I found no discrete correctness or security finding, but it still needs maintainer UI confirmation and fresh CI before merge.

## What This Changes

This PR changes the sidebar markdown inline-image CSS to cap image width at `min(100%, 420px)` and height at `320px`, matching the main chat image sizing rule.

## Best Possible Solution

Land this CSS cap, or an equivalent shared markdown image sizing rule, after a focused Control UI visual check with inline base64 markdown images in the sidebar; then use the linked regression to decide whether any separate main-chat image growth remains.

## Reproduction Assessment

Yes for the sidebar path: render a base64 markdown image through the markdown sidebar on current main and the image gets `.markdown-inline-image` inside `.sidebar-markdown`, where CSS allows full sidebar width and 420px height. The broader extended-session symptom from #45846 was not live-reproduced in this read-only review.

## Solution Assessment

Yes for the sidebar inconsistency. A CSS-only cap on the existing selector is the narrowest maintainable fix because it aligns sidebar image sizing with the already-capped main chat rule without changing markdown rendering or sanitizer behavior.
