## Summary

Keep open. The feature is not implemented on current main, and this external PR still has source-proven correctness blockers plus missing real behavior proof before it can merge.

## What This Changes

The PR adds optional `labelTemplate` fields to global and per-agent subagent config and uses them to generate fallback `sessions_spawn` labels when `label` is omitted.

## Best Possible Solution

Land a target-aware implementation that renders labels after resolving the effective native or ACP target, uses requester-scoped per-agent policy, documents the config contract, and includes focused coverage plus contributor-provided runtime proof.

## Reproduction Assessment

Yes, by source inspection. Configure `agents.defaults.subagents.labelTemplate` as `{agentId}-{taskPreview}` and call `sessions_spawn` without `agentId`; this PR renders `{agentId}` before native spawn defaults to the requester or ACP uses `acp.defaultAgent`.

## Solution Assessment

No, not as written. The feature direction is plausible, but the patch should move rendering to a target-aware point, resolve per-agent template policy from the requester, and update docs/tests before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Render labels after resolving the target agent:** `src/agents/tools/sessions-spawn-tool.ts:225-226`
  - body: The fallback label is computed from raw `requestedAgentId` before native or ACP spawn resolves the actual target. When `agentId` is omitted, native spawn defaults to the requester and ACP can use `acp.defaultAgent`, but `{agentId}` has already become empty, producing labels like `-Score all candidates`.
  - confidence: 0.91
- **[P2] Resolve labelTemplate from the requester agent:** `src/agents/tools/sessions-spawn-tool.ts:218-220`
  - body: This looks up `agents.list[].subagents.labelTemplate` by the requested child target, while existing subagent policy knobs such as `allowAgents` and `requireAgentId` are requester-scoped. A coordinator's per-agent label convention can be ignored, and the target can accidentally dictate the caller's label convention.
  - confidence: 0.84
- **[P3] Document the labelTemplate config:** `src/config/types.agent-defaults.ts:414-415`
  - body: This adds an operator-facing config key, but the public config docs do not describe the default or per-agent paths, token behavior, or precedence. Users would not be able to discover the option from the published docs after release.
  - confidence: 0.78
