## Summary

Keep open. Current main still has no `labelTemplate` support, so the PR is not obsolete, but the proposed branch is not ready because it renders labels before native/ACP target resolution and leaves generated config, docs, and changelog surfaces stale.

## What This Changes

The PR adds optional `labelTemplate` fields under default and per-agent subagent config and uses them to synthesize a `sessions_spawn` label when no explicit label is supplied.

## Best Possible Solution

Land a narrowed version that renders templates only after the effective native or ACP target agent is known, preserves explicit-label precedence, regenerates public config artifacts, documents the new operator-facing key, adds a changelog entry, and covers default, explicit-target, explicit-label, duplicate-token, and schema cases.

## Reproduction Assessment

Yes. Configure `agents.defaults.subagents.labelTemplate` as `{agentId}-{taskPreview}` and call `sessions_spawn` from `agent:main` without `agentId`; this PR would substitute an empty `{agentId}` before native spawning resolves the target to `main`. The generated schema drift is also reproducible by applying the PR and running the schema check.

## Solution Assessment

No. The feature direction is reasonable and narrow, but the current patch renders too early and omits required generated/docs/release surfaces; the safer fix is a small target-aware rendering helper or rendering inside the resolved native and ACP spawn paths.
