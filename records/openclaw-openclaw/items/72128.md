## Summary

Keep open. The feature is not implemented on current main and the PR is still useful, but the branch still renders fallback labels before the effective spawn target is resolved, leaves requester-scoped override behavior wrong, lacks docs, and has no after-fix real behavior proof.

## What This Changes

The PR adds optional `labelTemplate` fields to global and per-agent subagent config and uses them to synthesize a fallback `sessions_spawn` label from `{agentId}` and `{taskPreview}` when `label` is omitted.

## Best Possible Solution

Land a target-aware implementation that renders templates after resolving the effective native or ACP target, uses the requester-scoped override consistently, documents the new config contract, and includes focused coverage plus real runtime proof.

## Reproduction Assessment

Yes, by source inspection. Configure `agents.defaults.subagents.labelTemplate` as `{agentId}-{taskPreview}` and call `sessions_spawn` without `agentId`; this PR substitutes an empty `{agentId}` before native spawn defaults the target to the requester or ACP resolves `acp.defaultAgent`.

## Solution Assessment

No, not as written. The feature direction is plausible, but the implementation should move rendering to a target-aware point, resolve per-agent template policy from the requester, and update docs/tests before merge.
