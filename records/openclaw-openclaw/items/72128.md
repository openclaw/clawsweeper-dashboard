## Summary

Keep open. The PR is useful and already fixed the duplicate-token edge case with `replaceAll`, but it is not mergeable because label rendering still happens before the effective target agent is resolved, and the public config schema/docs/changelog work is incomplete.

## What This Changes

The PR adds optional `labelTemplate` config under agent default and per-agent subagent settings, then uses it to synthesize a `sessions_spawn` label when no explicit label is provided.

## Best Possible Solution

Land a narrowed version that renders the template after resolving the effective native or ACP target agent, preserves explicit-label precedence and existing spawn behavior, regenerates schema artifacts, updates subagent docs and changelog, and adds focused coverage for default, explicit-target, explicit-label, duplicate-token, and generated-schema cases.

## Reproduction Assessment

Yes. Configure `agents.defaults.subagents.labelTemplate` as `{agentId}-{taskPreview}` and call `sessions_spawn` from `agent:main` without `agentId`; this PR would substitute an empty `{agentId}` before `spawnSubagentDirect` resolves the target to `main`.

## Solution Assessment

No. The proposed direction is maintainable, but the current patch renders too early and leaves public schema/docs/release surfaces stale; the safer fix is to share a small target-aware label-rendering helper or move rendering into the resolved spawn paths.
