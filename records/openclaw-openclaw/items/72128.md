## What This Changes

The PR adds optional `labelTemplate` settings under global and per-agent subagent config, validates them in Zod schemas, and uses them as a fallback `sessions_spawn` label when `label` is omitted.

## Best Possible Solution

Land a target-aware, documented implementation that renders fallback labels after the actual native or ACP target is known, defines per-agent scope and precedence, covers explicit/default/native/ACP cases with focused tests, and includes redacted real behavior proof.

## Reproduction Assessment

Yes for the main review blocker by source inspection: configure a template with `{agentId}` and call `sessions_spawn` without `agentId`; the PR renders the token before native requester fallback or ACP `defaultAgent` resolution. I did not run a live spawn because this review was read-only.

## Solution Assessment

No, not as written. The maintainable implementation should render labels at or after the spawn path resolves the actual target, clarify whether per-agent templates are requester- or target-scoped, document the public config, and test the supported precedence matrix.
