## Summary

Keep open. Current main still lacks configurable `sessions_spawn` label templates, and this PR is useful source work, but the current patch renders labels before the effective native/ACP target is known and leaves generated schema, docs, and changelog surfaces stale.

## What This Changes

The PR adds optional `labelTemplate` fields under global and per-agent subagent config and uses them to synthesize a `sessions_spawn` label when no explicit label is supplied.

## Best Possible Solution

Land a narrowed version that renders templates after resolving the effective native or ACP target, preserves explicit-label precedence, regenerates config artifacts, updates subagent docs and changelog, and adds focused coverage for default-target and explicit-target cases.

## Reproduction Assessment

Yes. Configure `agents.defaults.subagents.labelTemplate` as `{agentId}-{taskPreview}` and call `sessions_spawn` without `agentId` from an agent session; this PR would substitute an empty `{agentId}` before native spawn resolves the requester as the target.

## Solution Assessment

No, not as written. The feature direction is maintainable, but the current implementation should move template rendering to a target-aware point and update generated/public surfaces before merge.
