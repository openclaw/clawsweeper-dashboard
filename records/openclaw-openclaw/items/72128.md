## Summary

This PR should stay open because the feature is not on current main and the branch still has blocking review findings around target-aware label rendering, requester-scoped config, missing docs, and missing contributor real-behavior proof.

## What This Changes

The PR adds optional `subagents.labelTemplate` config fields and uses `{agentId}` / `{taskPreview}` to generate fallback `sessions_spawn` labels when no explicit label is provided.

## Best Possible Solution

Land a target-aware implementation that renders after effective native or ACP target resolution, applies requester-scoped template policy consistently with existing subagent settings, documents the config contract, and includes focused coverage plus contributor runtime proof.

## Reproduction Assessment

Yes, by source inspection for the review blockers. Configure `agents.defaults.subagents.labelTemplate` as `{agentId}-{taskPreview}` and call `sessions_spawn` without `agentId`; the PR renders `{agentId}` before native spawn defaults to the requester or ACP uses `acp.defaultAgent`.

## Solution Assessment

No, not as written. The feature direction is plausible, but the patch should move label rendering to a target-aware point, resolve per-agent policy from the requester, and add docs before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Render labels after resolving the target agent:** `src/agents/tools/sessions-spawn-tool.ts:227-229`
  - body: The fallback label is computed from raw `requestedAgentId` before native or ACP spawn resolves the actual target. When `agentId` is omitted, native spawn defaults to the requester and ACP can use `acp.defaultAgent`, but `{agentId}` has already become empty, producing labels like `-Score all candidates`.
  - confidence: 0.91
- **[P2] Resolve labelTemplate from the requester agent:** `src/agents/tools/sessions-spawn-tool.ts:220-222`
  - body: This looks up `agents.list[].subagents.labelTemplate` by the requested child target, while existing subagent policy knobs such as `allowAgents` and `requireAgentId` are requester-scoped. A coordinator's per-agent label convention can be ignored, and the target can accidentally dictate the caller's label convention.
  - confidence: 0.84
- **[P3] Document the labelTemplate config:** `src/config/types.agent-defaults.ts:415-416`
  - body: This adds an operator-facing config key, but the public config docs do not describe the default or per-agent paths, token behavior, or precedence. Users would not be able to discover the option from the published docs after release.
  - confidence: 0.78
