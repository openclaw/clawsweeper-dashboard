## Summary

Keep open. The feature is still not on current main and this PR remains useful, but the current head still renders templates before effective target resolution, leaves generated/public config surfaces stale, and has CI-blocking schema and lint failures.

## What This Changes

The PR adds optional `labelTemplate` fields to global and per-agent subagent config and uses them to synthesize fallback `sessions_spawn` labels from `{agentId}` and `{taskPreview}` tokens.

## Best Possible Solution

Land a narrowed version that renders fallback labels after resolving the effective native or ACP target, keeps explicit labels authoritative, updates generated config artifacts/docs/changelog, and adds focused tests for explicit-target and default-target spawns.

## Reproduction Assessment

Yes. Configure `agents.defaults.subagents.labelTemplate` as `{agentId}-{taskPreview}` and call `sessions_spawn` without `agentId`; the PR substitutes an empty `{agentId}` before native or ACP spawn resolves the actual target.

## Solution Assessment

No, not as written. The feature direction is maintainable, but label rendering needs to happen at a target-aware point and the public config/docs surfaces must be regenerated before merge.
