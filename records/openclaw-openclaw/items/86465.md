The issue asks for `exec` command strings containing `$word` patterns to pass through unchanged or for the substitution/limitation to be documented.

## Best Possible Solution

Define the Windows exec command-fidelity contract, then either preserve literal payloads in the default Windows exec path with focused Windows proof and docs or explicitly document the PowerShell limitation and supported workaround.
