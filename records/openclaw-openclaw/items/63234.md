## Repair Work Prompt

Fix the plugin loader false-positive invalid-config spam tracked by https://github.com/openclaw/openclaw/issues/63234. Current source still validates missing `entry?.config` as `{}` in `src/plugins/loader.ts`, and both runtime and CLI metadata loader paths can emit `[plugins] <id> invalid config: ...` for strict schemas before a valid plugin config is materialized. Implement a narrow generic fix that preserves or defers valid `plugins.entries.<id>.config` across gateway startup, non-activating metadata/snapshot loads, tools catalog, and post-ready plugin tool resolution. Do not add a new config option, do not weaken real activation validation for explicit invalid config, and do not add UnraidClaw-specific product literals. Add focused regression coverage with a synthetic required-config plugin proving valid wrapped config is not later observed as `{}` and repeated false invalid-config logs are not emitted. Do not edit CHANGELOG.md.
