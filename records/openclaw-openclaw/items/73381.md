## Summary

Keep open. Current main still rejects Gateway `agent` requests when the effective session send policy is `deny` before checking whether the request asked for external delivery, and open PR #73500 is already the linked implementation candidate.

## What This Changes

Non-delivery Gateway `agent` invocations should bypass outbound `session.sendPolicy=deny` enforcement while explicit external delivery remains blocked.

## Best Possible Solution

Land PR #73500 or an equivalent narrow patch that gates Gateway agent send-policy denial on `deliver: true` and keeps explicit-delivery denial covered by tests.

## Reproduction Assessment

Yes. A Gateway `agent` request whose effective session policy resolves to `deny` and whose `deliver` field is omitted or false follows the current source path that rejects before delivery intent is evaluated.

## Solution Assessment

Yes. Gating the rejection on explicit delivery is the narrowest maintainable fix because it matches `agentCommand` behavior and the protocolâ€™s opt-in delivery contract.
