## Summary

Keep open. Current main still enforces session `sendPolicy=deny` in the Gateway `agent` handler before checking whether the request asked for external delivery, and open PR #73500 is already linked as the narrow implementation candidate.

## What This Changes

The issue requests that non-delivery Gateway `agent` invocations bypass outbound session send-policy denial while explicit delivery remains blocked.

## Best Possible Solution

Land PR #73500 or an equivalent patch that gates send-policy denial on `deliver: true`, preserves denial for explicit delivery, and adds regression coverage for both paths.

## Reproduction Assessment

Yes. A Gateway `agent` request with an effective `sendPolicy` of `deny` and `deliver` omitted or false follows the current source path that rejects before delivery planning.

## Solution Assessment

Yes. The linked fix direction is the narrowest maintainable solution because it matches embedded agent behavior and the protocol docs while keeping outbound-delivery protection intact.
