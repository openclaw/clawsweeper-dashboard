## Summary

Keep open. Current `main` still rejects Gateway `agent` requests when the effective session send policy is `deny` before checking whether outbound delivery was requested, and open PR #73500 is already the linked implementation candidate for this exact bug.

## What This Changes

The issue requests that Gateway agent invocations without explicit external delivery bypass session send-policy denial while preserving denial for delivery requests.

## Best Possible Solution

Land PR #73500 or an equivalent patch that gates Gateway `agent` send-policy denial on explicit `deliver: true`, preserves explicit-delivery blocking, and adds regression coverage for both paths.

## Reproduction Assessment

Yes. A Gateway `agent` request with effective `sendPolicy=deny` and `deliver` omitted or false follows the current source path that rejects before delivery intent is evaluated.

## Solution Assessment

Yes. Gating the policy rejection on explicit delivery is the narrowest maintainable fix because it matches `agentCommand` behavior and the protocol’s opt-in delivery contract.
