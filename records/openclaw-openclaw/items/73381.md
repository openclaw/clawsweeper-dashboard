## Summary

Keep open. Current main still enforces `session.sendPolicy=deny` before the Gateway `agent` handler determines whether external delivery was requested, and PR #73500 is the open closing implementation candidate for the narrow fix.

## What This Changes

The issue requests that non-delivery Gateway `agent` invocations remain allowed under outbound `session.sendPolicy=deny` while explicit delivery stays blocked.

## Best Possible Solution

Gate the Gateway `agent` send-policy denial on explicit external delivery and add regression coverage for both non-delivery allowed and `deliver: true` denied cases.

## Reproduction Assessment

Yes. A Gateway `agent` request with an effective `sendPolicy` of `deny` and `deliver` omitted or false follows the current source path that returns `INVALID_REQUEST` before delivery intent is computed.

## Solution Assessment

Yes. The linked fix direction is the narrowest maintainable path because it mirrors the embedded `agentCommand` behavior while preserving outbound-delivery protection.
