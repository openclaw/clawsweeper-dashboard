## Summary

Keep open. Current main still rejects Gateway `agent` requests on effective `sendPolicy=deny` before it checks whether outbound delivery was requested, and open PR #76317 is already the implementation candidate carrying forward #73500.

## What This Changes

The issue asks Gateway `agent` invocations without external delivery to bypass outbound `session.sendPolicy=deny` enforcement while preserving denial for explicit delivery.

## Best Possible Solution

Land #76317 or an equivalent patch that gates Gateway agent send-policy denial on `deliver: true` and keeps regression coverage for both non-delivery allow-through and explicit-delivery denial.

## Reproduction Assessment

Yes. On current main, a Gateway `agent` request with effective `sendPolicy=deny` reaches the deny response before `request.deliver === true` is computed, and the CLI sends `deliver: false` when `--deliver` is omitted.

## Solution Assessment

Yes. Gating the Gateway policy rejection on explicit delivery matches `agentCommand` behavior and the protocol’s opt-in delivery contract without weakening actual outbound delivery denial.
