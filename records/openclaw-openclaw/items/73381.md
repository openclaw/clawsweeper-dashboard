## Summary

Keep open. Current `origin/main` still rejects Gateway `agent` requests on `sendPolicy=deny` before it checks whether the request actually asked for external delivery, and the issue is already paired with open, unmerged PR #73500 that uses closing syntax for this bug.

## What This Changes

Request to allow non-delivery Gateway agent invocations when `session.sendPolicy=deny` while still blocking explicit external delivery sends.

## Best Possible Solution

Land PR #73500 or an equivalent focused change that gates the Gateway `agent` send-policy denial on `request.deliver === true`, with regression coverage for both non-delivery allow and explicit-delivery denial.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a Gateway `agent` request with `deliver` omitted or false and an effective `sendPolicy` of `deny` reaches the unconditional rejection before the handler computes delivery intent.

## Solution Assessment

Yes. Gating the Gateway policy check on explicit delivery is the narrowest maintainable fix because it matches the embedded agent path while preserving the outbound-delivery safety check.
