## Summary

Keep open. Current main still rejects Gateway `agent` requests with an effective `sendPolicy=deny` before checking whether the request asked for external delivery, and the issue is already paired with open closing PR #73500 for the focused fix.

## What This Changes

A Gateway `agent` request without explicit delivery should run even when its session send policy is deny, while explicit delivery should remain blocked.

## Best Possible Solution

Land PR #73500 or an equivalent focused change that gates the Gateway send-policy rejection on `request.deliver === true`, while adding regression coverage for non-delivery allow and explicit-delivery denial.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists: invoke the Gateway `agent` handler with `deliver` omitted or false and an effective denied send policy; current main reaches the rejection before delivery intent is considered.

## Solution Assessment

Yes. Mirroring the embedded `agentCommand` gate in the Gateway handler is the narrowest maintainable fix because it preserves outbound-delivery protection while allowing internal non-delivery runs.
