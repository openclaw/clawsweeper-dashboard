## Summary

Keep open. Current main still rejects Gateway `agent` requests when `resolveSendPolicy` returns `deny` before the handler checks `request.deliver`, and open unmerged PR #73500 is already linked with closing syntax for the narrow fix.

## What This Changes

The issue asks for non-delivery Gateway `agent` invocations to bypass outbound session send-policy denial while preserving denial for explicit external delivery.

## Best Possible Solution

Gateway `agent` should enforce `sendPolicy=deny` only for explicit external delivery, keep denial intact for `deliver: true`, and carry regression tests for both paths.

## Reproduction Assessment

Yes. Invoke the Gateway `agent` method with an effective session `sendPolicy` of `deny` and `deliver` omitted or false; current main returns `INVALID_REQUEST` before reaching the delivery-planning code.

## Solution Assessment

Yes. The linked fix direction is the narrowest maintainable path because it matches the embedded agent behavior while preserving outbound-delivery protection.
