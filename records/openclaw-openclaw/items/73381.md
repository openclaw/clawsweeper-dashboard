## Summary

Current main implements the requested split: Gateway `agent` requests only enforce `session.sendPolicy=deny` when `deliver: true`, and regression coverage now verifies both the non-delivery allowed path and explicit-delivery blocked path. The fix is on main via #76317 but appears unreleased after v2026.5.2.

## What This Changes

The issue asks for non-delivery Gateway agent invocations to bypass outbound send-policy denial while preserving denial for explicit external delivery.

## Best Possible Solution

Keep the main-branch implementation and let the next release carry the existing changelog entry so packaged users receive the fixed Gateway behavior.

## Reproduction Assessment

No current-main failure reproduces: the reported path is now covered by source and tests, with non-delivery deny-policy requests allowed and explicit `deliver: true` requests blocked. The original pre-fix bug has a high-confidence reproduction shape from the issue and prior review.

## Solution Assessment

Yes. Gating the Gateway send-policy denial on explicit delivery is the narrowest maintainable fix because it matches `agentCommand` behavior and the protocol's opt-in delivery contract.
