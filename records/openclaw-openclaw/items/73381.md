## Summary

Keep open. Current main still rejects Gateway `agent` requests on effective `sendPolicy=deny` before checking whether external delivery was requested, and open PR #73500 is already the linked implementation candidate.

## What This Changes

The issue requests that non-delivery Gateway `agent` invocations run even when outbound session delivery is blocked by `sendPolicy=deny`.

## Best Possible Solution

Land PR #73500 or an equivalent patch that gates Gateway agent send-policy denial on `deliver: true` while preserving explicit-delivery denial and adding regression coverage for both paths.

## Reproduction Assessment

Yes. A Gateway `agent` request with effective `sendPolicy=deny` and `deliver` omitted or false follows the current source path that rejects before delivery intent is evaluated.

## Solution Assessment

Yes. Gating the policy rejection on explicit delivery is the narrowest maintainable fix because it matches `agentCommand` behavior and the protocol’s opt-in delivery contract.
