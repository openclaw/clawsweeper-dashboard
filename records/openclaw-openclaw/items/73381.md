## Summary

Keep open. Current main still enforces `session.sendPolicy=deny` in the Gateway `agent` handler before it computes whether the request is delivering externally, and the issue is paired with open, unmerged PR #73500 that uses closing syntax for this bug.

## What This Changes

The issue asks for non-delivery Gateway `agent` invocations to run even when the session send policy is `deny`, while preserving the block for explicit external delivery.

## Best Possible Solution

Review and land PR #73500 or an equivalent focused change that gates the Gateway `sendPolicy=deny` rejection on `request.deliver === true`, with regression coverage for both non-delivery allow and explicit-delivery denial.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a Gateway `agent` request with `deliver` omitted or false and an effective `sendPolicy` of `deny` reaches the unconditional rejection before the handler computes delivery intent.

## Solution Assessment

Yes. The linked fix direction is the narrowest maintainable solution because it matches the embedded agent path while preserving the outbound-delivery safety check for `deliver: true`.
