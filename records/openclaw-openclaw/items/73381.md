## Summary

Keep open. Current main still rejects Gateway `agent` requests on `sendPolicy=deny` before checking whether the request actually wants external delivery, and this issue is already paired with open, unmerged PR #73500 that uses closing syntax for the same bug.

## What This Changes

The issue requests that Gateway `agent` invocations without external delivery bypass outbound `session.sendPolicy=deny` while preserving the denial for explicit delivery requests.

## Best Possible Solution

Review and land PR #73500 or an equivalent focused change that gates the Gateway `sendPolicy=deny` rejection on `request.deliver === true`, with regression coverage for both non-delivery allow and explicit-delivery denial.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: call the Gateway `agent` method with `deliver` omitted or false while the effective session policy resolves to `deny`; current main reaches the rejection before it computes delivery intent.

## Solution Assessment

Yes. The narrowest maintainable fix is to mirror the embedded agent path by enforcing this policy only for explicit delivery, while keeping the denial intact for `deliver: true`.
