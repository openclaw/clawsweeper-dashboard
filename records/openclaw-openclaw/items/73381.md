## Summary

Keep open. Current main still enforces `sendPolicy=deny` in the Gateway `agent` handler before it checks whether the request asked for external delivery, and the issue is already paired with open, unmerged PR #73500 that uses closing syntax for this bug.

## What This Changes

Prevent non-delivery Gateway `agent` invocations from being blocked by outbound session send policy.

## Best Possible Solution

Review and land PR #73500 or an equivalent focused change that only enforces `session.sendPolicy=deny` when `request.deliver === true`, with regression coverage for non-delivery allow and explicit-delivery denial.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a Gateway `agent` request with `deliver` omitted or false and an effective `sendPolicy` of `deny` reaches the unconditional rejection before the handler computes delivery intent.

## Solution Assessment

Yes. Gating the Gateway policy check on `request.deliver === true` is the narrowest maintainable fix because it matches the embedded agent path while preserving the outbound-delivery safety check.
