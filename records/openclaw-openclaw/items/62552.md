## Summary

Keep open: the ACP fallback-key fix is still useful on current main, but the PR has a blocking cron-task maintenance regression, misses the matching ACP docs update, and still lacks contributor real-behavior proof.

## What This Changes

The PR changes ACP fallback sessions from `acp:<uuid>` to `acp-bridge:<uuid>`, adds raw/canonical pending-prompt matching, and treats terminal child sessions as missing during task maintenance.

## Best Possible Solution

Land a rebased or replacement fix that keeps ACP bridge fallback sessions out of the ACP runtime namespace, updates ACP docs/tests, and preserves cron runtime authority while leaving broader CLI zombie-task cleanup to https://github.com/openclaw/openclaw/issues/73121.

## Reproduction Assessment

Yes, source-level. Current main still uses `acp:<uuid>` for metadata-less ACP bridge sessions, and the PR regression follows from letting terminal child-session state override `isCronJobActive(jobId)`.

## Solution Assessment

No. The ACP fallback-prefix direction is maintainable, but the task-maintenance hunk is not the best fix because cron runtime ownership should remain authoritative and the docs need to move with the runtime prefix change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve active cron ownership:** `src/tasks/task-registry.maintenance.ts:159-170`
  - body: Including `cron` in this child-session branch makes active cron tasks depend on a stored child session row. If that row is terminal or missing, maintenance can mark the task lost even while `isCronJobActive(jobId)` is true, so keep the cron runtime short-circuit or scope terminal-session handling away from cron ownership.
  - confidence: 0.9
- **[P3] Update the ACP session-key docs:** `src/acp/translator.ts:540`
  - body: Changing the fallback to `acp-bridge:<uuid>` leaves `docs/cli/acp.md` documenting isolated ACP keys as `acp:`. Update the docs with the runtime change so users do not rely on the old prefix.
  - confidence: 0.83
