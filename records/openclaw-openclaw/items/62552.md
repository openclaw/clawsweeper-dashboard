## Summary

Keep open. Current main still lacks the ACP bridge key fixes, and this PR is useful source work, but the task-maintenance hunk can regress active cron liveness and the branch also needs docs and changelog follow-through before merge.

## What This Changes

The PR changes ACP bridge fallback sessions to `acp-bridge:<uuid>`, matches pending ACP prompts across raw and Gateway-canonical session keys, and treats terminal child sessions as missing during task registry maintenance.

## Best Possible Solution

Land a rebased or replacement patch that keeps ACP bridge fallback sessions on a non-ACP-runtime prefix and matches Gateway-canonical keys, while preserving active cron runtime authority and updating ACP docs, changelog, and focused regressions.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main creates fallback ACP sessions as `acp:<uuid>` and only exact-matches pending prompt session keys, while the PR regression follows from an active cron task whose terminal or missing child session would override `isCronJobActive(jobId)`.

## Solution Assessment

No. The ACP bridge key direction is maintainable, but the cron maintenance hunk is not the best fix because child-session state must not override authoritative active-cron ownership.
