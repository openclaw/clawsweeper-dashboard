## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Do not gate active cron tasks on child session rows:** `src/tasks/task-registry.maintenance.ts:162-173`
  - body: After `isCronJobActive(jobId)` succeeds, this falls through and requires the cron task's `childSessionKey` to point at a non-terminal session row. Current cron execution records isolated tasks against the backing key while the actual execution uses a run-scoped `:run:<sessionId>` key, so an active cron job can be marked `lost` just because the backing row is missing or terminal. Keep cron liveness tied to the cron runtime/run-log recovery path or prove the child key is the executing run.
  - confidence: 0.88
- **[P1] Use the shared session-key matcher:** `src/acp/translator.ts:411-426`
  - body: This local comparator duplicates the existing request-key alias contract in `src/routing/session-key.ts` and strips `agent:<id>:` outside the shared parser. Pending prompt matching is a session boundary, so this should use or extend the shared helper instead of creating a second normalization rule that can drift from Gateway routing.
  - confidence: 0.78
