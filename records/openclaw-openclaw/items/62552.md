## Summary

## What This Changes

The PR changes metadata-free ACP bridge fallback sessions to `acp-bridge:*`, adds raw/canonical pending-prompt matching, and treats terminal child sessions as missing during task maintenance for cron/cli/subagent tasks.

## Best Possible Solution

Land a rebased, focused ACP bridge fix that keeps metadata-free bridge sessions out of ACP runtime key space, scopes raw/canonical aliasing to the ACP bridge case or current run-id reconciliation, updates docs/tests, and leaves cron liveness owned by the cron runtime.
