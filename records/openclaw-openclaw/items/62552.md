## Summary

Keep open. Current main still uses the old ACP bridge fallback prefix, but this PR has a blocking cron-maintenance regression and the external contributor has not provided real after-fix behavior proof.

## What This Changes

The PR changes ACP fallback session keys to `acp-bridge:<uuid>`, matches pending ACP prompts across raw and Gateway-canonical session keys, and treats terminal child sessions as missing during task maintenance.

## Best Possible Solution

Land a rebased or replacement fix that keeps ACP bridge fallback sessions out of the ACP runtime namespace, preserves cron runtime authority, updates ACP docs/tests, and leaves broader CLI zombie-task cleanup to https://github.com/openclaw/openclaw/issues/73121.

## Reproduction Assessment

Yes, source-level. Current main still creates metadata-less ACP bridge sessions with `acp:` and has no `acp-bridge` implementation, while the PR cron regression follows from routing active cron tasks into child-session terminal-state checks.

## Solution Assessment

No. The ACP fallback-prefix direction is maintainable, but the task-maintenance hunk should not override cron runtime ownership and the public ACP docs need to be updated with the new prefix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve active cron ownership:** `src/tasks/task-registry.maintenance.ts:159-170`
  - body: Including `cron` in this child-session branch makes an active cron task depend on a stored child session row. If that row is terminal or missing, maintenance can mark the task lost even while `isCronJobActive(jobId)` is true, so keep the cron runtime short-circuit or scope terminal-session handling away from cron ownership.
  - confidence: 0.9
- **[P3] Update the ACP session-key docs:** `src/acp/translator.ts:540`
  - body: Changing the fallback to `acp-bridge:<uuid>` leaves `docs/cli/acp.md` documenting default isolated ACP keys as `acp:`. Update the docs with the runtime change so users do not rely on the old prefix.
  - confidence: 0.84
