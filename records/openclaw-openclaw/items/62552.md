## Summary

Keep open: the ACP bridge-key fix is still useful, but this PR is not merge-ready because it conflicts with current main, lacks real ACP/acpx behavior proof, includes an unsafe cron task liveness change, and leaves the ACP key docs stale.

## What This Changes

The branch changes metadata-free ACP bridge fallback sessions to `acp-bridge:*`, adds raw/canonical pending-prompt matching, and extends task maintenance to treat terminal child sessions as missing for cron/cli/subagent tasks.

## Best Possible Solution

Land a rebased, narrower ACP bridge-key fix that separates bridge fallback keys from ACP runtime keys, preserves current run-id reconciliation, updates ACP docs/tests, and leaves active cron liveness owned by the cron runtime.

## Reproduction Assessment

Yes, at source level: current main still creates metadata-free ACP bridge sessions with `acp:<uuid>`, and ACP runtime resolution treats ACP-shaped keys without stored ACP metadata as stale. I did not run a live ACP/acpx session in this read-only review.

## Solution Assessment

No for this PR as written. The bridge prefix split is the right direction, but the cron liveness hunk is unsafe against current main, the branch is conflicted, and docs plus real behavior proof are still missing.
