## Summary

Keep open. This targets a real embedded Pi hot path, but the patch is not mergeable because it bypasses private Pi resource-loader internals, has a Vitest hoisting hazard, and lacks after-fix real benchmark proof.

## What This Changes

The PR makes embedded Pi resource-loader creation async, skips `DefaultResourceLoader.reload()` in attempt and compact paths, directly loads inline extension factories, and rewrites the related tests.

## Best Possible Solution

Use a public-contract embedded resource-loader fast path, or an upstream Pi API, that avoids package discovery while preserving reload semantics, inline extensions, conflict/source-info handling, and real timing proof.

## Reproduction Assessment

No high-confidence real reproduction was run. Source inspection confirms current main awaits the dependency `reload()` path and the dependency calls `packageManager.resolve()`, but the PR provides no after-fix benchmark proof.

## Solution Assessment

No. Directly invoking private `DefaultResourceLoader` internals with `any` is not the narrowest maintainable solution; a public fast path or upstream contract with real timing proof is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Avoid private Pi loader internals:** `src/agents/pi-embedded-runner/resource-loader.ts:49-52`
  - body: `extensionsResult` and `loadExtensionFactories` are private in the published Pi declarations, but this fast path calls and mutates them through `any`. That bypasses the public reload contract and can break on dependency updates; use a public API/upstream fast path or keep the reload contract intact.
  - confidence: 0.88
- **[P2] Hoist the shared mock state:** `src/agents/pi-embedded-runner/resource-loader.test.ts:24`
  - body: Vitest hoists `vi.mock()` factories, so this factory can run before `mockLoadExtensionFactories` is initialized. Put the shared mock in `vi.hoisted()` or keep the mock state inside a hoisted object before referencing it from the module factory.
  - confidence: 0.86
