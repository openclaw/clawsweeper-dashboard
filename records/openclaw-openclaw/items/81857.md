## Summary

Keep open; the reported bug is plausible and has real proof, but the current patch introduces a blocking regression by filtering selected plugins before the existing standalone cold-load path can materialize plugin tools that are not already in active/channel registries.

## What This Changes

The PR filters plugin tool lookup to plugin ids already present in active runtime or channel registries and adds a regression test for loaded plugins with unloaded manifest siblings.

## Best Possible Solution

Keep the fix direction, but scope only the active/channel registry lookup to loaded plugin ids while preserving the full manifest-selected runtimePluginIds for standalone cold-load fallback and retry coverage.

## Reproduction Assessment

Yes for the reported behavior: the strict loaded-registry lookup path is clear from source, and the PR supplies before/after Telegram screenshots showing Active Memory changing from unavailable to ok. I did not run a live gateway in this read-only review.

## Solution Assessment

No. The proposed direction is right, but this implementation filters selected plugin ids too early and bypasses the existing cold-load fallback for plugins that are selected but not already present in active/channel registries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve the cold-load fallback for missing plugin ids:** `src/plugins/tools.ts:966-968`
  - body: Filtering `runtimePluginIds` down to ids already present in active/channel registries means a selected plugin tool that is not preloaded gets dropped before `ensureStandaloneRuntimePluginRegistryLoaded` can run. Current main has regression coverage for config-origin/path plugin tools from https://github.com/openclaw/openclaw/issues/76598; with this early return and the narrowed `loadOptions`, those tools would disappear again instead of being cold-loaded. Keep the full selected id set for the standalone load/retry path and only narrow the loaded-registry check.
  - confidence: 0.91
