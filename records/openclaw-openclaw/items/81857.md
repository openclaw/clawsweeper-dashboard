## Summary

Keep open. The loaded-plugin fix is useful and the before/after proof is strong, but PR head still reads process-global active/channel registries before the resolver's workspace-compatible checks, so stale registry state can influence which plugin tools become callable.

## Best Possible Solution

Keep the loaded-sibling fix, but build the loaded-id prefilter through workspace-compatible registry lookups and add stale active/pinned registry regression coverage before merge.
