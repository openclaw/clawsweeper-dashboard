## Summary

Keep open: the archive-first direction is useful and has real terminal proof, but current head still has a source-level P2 crash-window gap before merge.

## What This Changes

The PR archives short-term promotion details under quarterly `memory/archived/` dumps, keeps `MEMORY.md` as a compact pointer, migrates legacy promotion sections, and updates docs, CLI output, events, and tests.

## Best Possible Solution

Keep the archive-first storage design, but make promotion apply rewrite the compact pointer when selected candidates are reconciled from existing archives and cover that crash window with a focused regression.

## Reproduction Assessment

Yes, at source level. Seed an archive with a selected candidate marker, leave `MEMORY.md` without the compact pointer and the store unpromoted, then run apply; PR head classifies the candidate as `alreadyWritten`, skips the pointer rewrite block, and marks it promoted.

## Solution Assessment

No, not yet. The archive-first approach is the maintainable direction, but the current self-heal condition needs to account for already-archived selected candidates and point `MEMORY.md` at their existing archive.
