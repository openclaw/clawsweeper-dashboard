## Summary

Keep open: the latest PR head still has a source-reproducible data-loss regression where an unmarked `Promoted From Short-Term Memory` section can be overwritten by the new pointer section instead of being preserved or archived.

## What This Changes

The PR archives short-term promotion details into quarterly `memory/archived` dumps, keeps `MEMORY.md` as a compact pointer, updates memory promotion events, CLI/docs/changelog text, and adjusts memory-core tests.

## Best Possible Solution

Keep the archive-first design only if migration and pointer rewriting preserve or intentionally archive every existing promotion-heading section without deleting user-authored or pre-marker content.

## Reproduction Assessment

Yes. Seed `MEMORY.md` with an unmarked `## Promoted From Short-Term Memory` block, trigger an archive write with one selected candidate, and the latest PR source shows extraction skips that block while the pointer helper replaces it.

## Solution Assessment

No as submitted. Archive-first storage may be a maintainable direction, but the migration must be marker-aware or insertion-only so unarchived content is not deleted.
