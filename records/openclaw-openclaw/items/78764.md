## Summary

Keep this PR open: the archive-first memory promotion direction is useful, current main does not already provide it, and the latest head appears to address the earlier self-heal blocker. ## What This Changes

The branch moves detailed short-term promotion writes from root `MEMORY.md` into quarterly archives, updates the compact pointer/docs/help/events, and removes the old root-size budget helper.

## Best Possible Solution

Keep the archive-first design, align CLI text/JSON with archive-only migrations, then land after required CI and maintainer review are satisfied.

## Reproduction Assessment

Yes. The core archive flow has supplied terminal proof, and the remaining CLI reporting gap is source-reproducible from `runMemoryPromote` only treating `applied > 0` as an archive success.

## Solution Assessment

Mostly yes. Archiving detailed excerpts while keeping `MEMORY.md` as a compact pointer is the maintainable direction, but the CLI should also report archive-only migrations instead of looking like a no-op.
