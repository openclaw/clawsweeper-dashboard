## Summary

## Best Possible Solution

Land archive-first promotion only after preserving the existing timestamp and snippet-cap contracts, exposing archive destinations to JSON callers, and getting owner acceptance for the durable `MEMORY.md` to archive-pointer migration.

## Solution Assessment

No: archive-first promotion is a plausible direction, and the crash-window dedupe follow-up is useful, but this is not the best merge shape until it preserves the existing timestamp/cap contracts and updates machine-readable output.
