## Summary

Keep open: the archive-first direction is well covered and the previous docs inconsistency is fixed, but the patch can leave existing detailed root `MEMORY.md` promotion sections unmigrated when there are no fresh promotion candidates.

## What This Changes

The PR changes memory-core promotion to write detailed short-term promotion excerpts to quarterly archives, keep a compact pointer in `MEMORY.md`, and update related events, CLI copy, docs, tests, and changelog.

## Best Possible Solution

Run root-section migration independently of fresh candidate selection, add a regression test with existing promoted store entries, then land the archive-first behavior if maintainers accept the storage change.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an old `MEMORY.md` detailed promotion section plus store entries already marked `promotedAt` makes apply return before migration when there are no new candidates.

## Solution Assessment

No. The archive-and-pointer approach is maintainable, but the current implementation needs a separate migration path for existing detailed root sections that does not depend on selecting fresh candidates.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82
