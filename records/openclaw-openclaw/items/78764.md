## Summary

Keep open: the archive-first direction is coherent and the supplied terminal proof covers the main behavior, but the latest head still has CLI reporting gaps for migration-only applies and JSON consumers, and exact-head checks are not green yet.

## What This Changes

This PR changes short-term memory promotion to archive detailed excerpts under quarterly `memory/archived/` dumps while keeping `MEMORY.md` as a compact pointer, with docs, CLI text, event metadata, and tests updated.

## Best Possible Solution

Keep the archive-first implementation local to `memory-core`, but fix the CLI apply surface so both human and JSON output accurately report archive destinations and migration-only writes before maintainers decide whether to land the storage policy change.

## Reproduction Assessment

Yes for the behavior being changed: source inspection on current main shows detailed short-term promotion snippets are appended directly to `MEMORY.md`. This is not a strict broken-contract bug because existing docs describe the append behavior.

## Solution Assessment

No for the latest patch as-is. The archive-first storage design is a plausible owner-local solution, but the CLI output and JSON apply contract need to reflect migration-only archive writes before this is the best mergeable version.
