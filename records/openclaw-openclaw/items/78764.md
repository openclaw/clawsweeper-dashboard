## Summary

Keep open: this PR is a plausible archive-first memory-core improvement, but the latest head still leaves a blocking self-heal pointer bug when migrated legacy detail already lives in an older archive.

## What This Changes

This PR changes memory-core short-term promotion to archive detailed promoted excerpts under `memory/archived/YYYY-Q#/`, keep a compact `MEMORY.md` pointer, migrate existing marked root sections, and update docs, CLI text, events, changelog, and tests.

## Best Possible Solution

Keep archive-first promotion only if memory-core owners want that storage contract, and make self-heal point `MEMORY.md` at an archive path that actually exists and contains the preserved promoted detail.

## Reproduction Assessment

Yes for the PR defect by source inspection: seed `MEMORY.md` with a marked legacy promotion already present in an older archive, then run apply on a later day with no new candidates. The code skips the current archive write but rewrites the pointer to that nonexistent current path.

## Solution Assessment

No, not yet. The archive dedupe repair prevents duplicate archive writes, but the narrow maintainable fix also needs to preserve a truthful latest-archive pointer and assert that target in the regression.
