## Summary

Keep open: the archive-first direction is useful and the contributor supplied real behavior proof, but latest head still has a crash-window pointer bug that can leave `MEMORY.md` pointing at an archive file that was never created.

## What This Changes

This PR moves detailed short-term promotion writes from `MEMORY.md` into quarterly archive dumps, keeps `MEMORY.md` as a compact latest-archive pointer, migrates legacy promotion sections, updates docs/help/events, and removes the old size-budget helper.

## Best Possible Solution

Keep the archive-first design only if memory-core owners accept the storage-contract change, with self-heal preserving dedupe and an accurate pointer to an existing archive.

## Reproduction Assessment

Yes for the PR finding from source inspection: seed `MEMORY.md` with a legacy promotion section, seed the same marker in an older archive, then run apply on a later date with no new candidates. The PR's own test models this and expects no current archive file, while the implementation writes the pointer using the current archive path.

## Solution Assessment

No, not yet. Archive-first storage is plausible, but the crash-window self-heal needs to point `MEMORY.md` at the existing archive that contains the migrated detail before this is ready for owner review and merge.
