## Summary

Keep open: this active external PR is a coherent archive-first memory-core implementation, but it changes user-facing storage policy and still needs maintainer choice plus exact-head CI handling. I found one low-priority CLI feedback issue and no blocking correctness or security defect.

## What This Changes

The PR changes short-term memory promotion to archive detailed dumps under quarterly `memory/archived/` files, keep `MEMORY.md` as a compact pointer, and updates events, docs, CLI copy, tests, and changelog.

## Best Possible Solution

Land this PR only if maintainers choose archive-first promotion as the canonical storage policy; otherwise route the bounded-memory fix through the narrower canonical work in https://github.com/openclaw/openclaw/issues/73691 / https://github.com/openclaw/openclaw/pull/74088.

## Solution Assessment
