## Summary

Keep open for maintainer review: this PR contains meaningful archive-first memory storage work that is not on current main, while the original growth bug is already addressed by the shipped bounded-compaction fix in https://github.com/openclaw/openclaw/pull/74088. Merging this would replace the current MEMORY.md storage contract with a different archive-backed design, so it needs owner/product judgment rather than conservative cleanup closure.

## What This Changes

This PR moves detailed short-term promotion writes from root MEMORY.md to quarterly memory/archived dumps, leaves a compact pointer in MEMORY.md, migrates marked root promotion sections, adds archive fields to promotion events, and updates docs, help text, tests, and changelog.

## Best Possible Solution

Adopt archive-first storage only if memory-core owners want that storage contract; otherwise keep the shipped compaction fix and close this PR by maintainer decision.

## Reproduction Assessment

Not applicable; this PR changes the storage design rather than proving a current-main bug. The original MEMORY.md growth issue is already fixed on main by bounded compaction, and the PR body supplies after-fix terminal proof for the proposed archive-first behavior.

## Solution Assessment

Unclear as a product decision: archive-first storage is a plausible and well-tested direction if maintainers want durable promotion history outside MEMORY.md. It is not the narrowest fix for the already-shipped growth bug because https://github.com/openclaw/openclaw/pull/74088 already delivered bounded compaction.
