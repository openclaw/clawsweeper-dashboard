## Summary

Needs changes before merge: the latest PR head still can drop unmarked `Promoted From Short-Term Memory` sections while moving marked promotion details into archives. The supplied terminal proof is enough for the happy path, but this migration edge remains unsafe.

## What This Changes

The branch changes memory-core promotion to archive detailed short-term excerpts under quarterly `memory/archived` dumps, keep a compact `MEMORY.md` pointer, update docs/help/events/tests, and remove the budget compaction helper.

## Best Possible Solution

Make the migration marker-aware so only archived marker-backed sections are replaced, preserve unmarked root content, then have memory-core owners decide whether archive-first storage should supersede the shipped budget cap.

## Reproduction Assessment

Yes. Source inspection at PR head gives a high-confidence reproduction path: put an unmarked `## Promoted From Short-Term Memory` section in `MEMORY.md`, run an archive write, and the helper will replace that section even though it was skipped by the archive extractor.

## Solution Assessment

No as submitted. Archive-first may be a maintainable direction, but the pointer rewrite must preserve unmarked sections or only replace sections that were actually migrated into the archive.
