## Summary

Keep this active PR open: the supplied proof and latest follow-up address the legacy migration dedupe concern, but source review found one remaining archive-first crash-window hole that can leave `MEMORY.md` without its compact archive pointer.

## What This Changes

The PR changes short-term memory promotion to archive detailed excerpts under `memory/archived/YYYY-Q#/` while keeping `MEMORY.md` as a compact latest-archive pointer.

## Best Possible Solution

Keep the archive-first design, but make apply self-heal `MEMORY.md` when selected candidates are reconciled from existing archives and add a regression for that crash window.

## Reproduction Assessment

Yes, at source level: if an archive contains a selected candidate marker but `MEMORY.md` lacks the pointer, the PR classifies the candidate as already written, skips the archive/pointer block, then marks it promoted. I did not execute the path because this review is read-only.

## Solution Assessment

No, not yet. The archive-first approach is the right maintainable direction, but the current condition should also rewrite the compact pointer when selected candidates are reconciled from existing archive markers.
