## Summary

Keep open for maintainer review: this is a plausible archive-first memory storage change, but current main has already shipped a compaction-based fix for the same growth problem, so maintainers should choose whether the archive policy replaces or complements that path.

## What This Changes

The PR moves detailed short-term promotion output into quarterly `memory/archived` dumps, keeps `MEMORY.md` as a compact pointer, and updates CLI copy, docs, events, tests, changelog, and generated Swift protocol output.

## Best Possible Solution

Choose whether archive-first promotion should replace or complement the shipped compaction strategy, then land a rebased version with idempotent migration and aligned docs/events/tests.

## Reproduction Assessment

Yes for the behavior being changed: current main source shows detailed promotion sections still write to `MEMORY.md`, with a budget compaction layer added by the merged growth fix. This is not a strict current-contract bug reproduction because current main already addresses unbounded growth through compaction.

## Solution Assessment

Unclear pending maintainer policy. Archive-first is technically coherent and preserves durable details, but current main already has the narrower compaction fix, so maintainers should decide whether this broader retention layout is the preferred end state.
