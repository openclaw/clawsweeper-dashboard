## Summary

Keep this PR open: archive-first promotion is useful and the supplied terminal proof covers the main flow, but the latest head still has a correctness blocker in the crash-window self-heal path. A legacy root section whose marker already exists only in an older archive can be removed from MEMORY.md while the compact pointer is rewritten to a current-day archive file that was never created.

## What This Changes

This PR changes short-term memory promotion to archive detailed promotion excerpts outside root MEMORY.md while leaving a compact pointer and updating docs, CLI text, events, and tests.

## Best Possible Solution

Keep the archive-first design only if memory-core owners accept the storage-contract change, with self-heal preserving dedupe and an accurate pointer to an existing archive.

## Reproduction Assessment

Yes for the PR defect from source inspection: seed MEMORY.md with a legacy promoted section whose marker already exists only in an older archive, then run apply on a later date with no new candidates. The current code filters out all archive writes but still rewrites the root pointer to the later, nonexistent archive path.

## Solution Assessment

No, not yet. Archive-first storage is plausible, but the self-heal path needs to point MEMORY.md at the existing archive that contains the migrated detail before this is ready to merge.
