## Summary

Keep open: the archive-first direction has sufficient terminal proof and no supply-chain concern, but latest PR head still has a P2 data-loss bug where unmarked promotion sections can be replaced by the pointer without being archived.

## What This Changes

This PR moves detailed memory-core short-term promotion writes from root `MEMORY.md` into quarterly `memory/archived` dump files with a compact pointer, updates CLI/docs/events/tests, and removes the old root budget compaction helper.

## Best Possible Solution

Make the migration and pointer rewrite marker-aware so only marker-backed detailed promotion sections are archived/removed while unmarked root content survives, then let memory-core owners decide whether archive-first storage should replace budgeted root promotion.

## Reproduction Assessment

Yes, source-reproducible. At PR head, unmarked sections are skipped by the extractor but can still be replaced by the pointer helper during an archive write; I did not mutate a workspace in this read-only review.

## Solution Assessment

No as submitted. Archive-first storage is a plausible maintainable direction, but the current rewrite must preserve or explicitly migrate only marker-backed promotion sections before the patch is safe.
