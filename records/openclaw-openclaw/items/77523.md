## Summary

Keep open: this PR targets a real MiniMax Portal OAuth refresh bug on current main, but the latest diff still routes legacy CN profiles with missing credential metadata to the Global endpoint and only provides mocked/unit-test proof rather than a real after-fix refresh.

## What This Changes

The PR adds a MiniMax Portal OAuth refresh-token flow, stores `resourceUrl` on new OAuth credentials, wires `refreshOAuth`, adds provider tests, and updates the changelog plus raw-fetch allowlist offsets.

## Best Possible Solution

Land a provider-owned MiniMax Portal refresh hook after it safely handles legacy profiles without `resourceUrl`, keeps OAuth implementation behind the lazy runtime boundary, and includes real after-fix refresh evidence.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: expired `minimax-portal` OAuth credentials call the plugin refresh seam, find no MiniMax hook, then fall through to pi-ai built-ins that omit MiniMax.

## Solution Assessment

No. The provider-owned `refreshOAuth` hook is the right direction, but this diff is not the safest fix until missing legacy region metadata is handled without defaulting to Global and the lazy OAuth boundary is preserved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve the legacy MiniMax OAuth region:** `extensions/minimax/provider-registration.ts:289`
  - body: Existing MiniMax OAuth profiles from current releases do not store `resourceUrl`; their CN/Global choice lives in provider config. This line turns missing metadata into Global, so upgraded CN profiles send refresh tokens to `api.minimax.io` and still fail. Derive the legacy region from a reliable source, backfill metadata, or fail safely with reauth instead of choosing the wrong host.
  - confidence: 0.91
- **[P3] Keep the resolver behind the lazy OAuth boundary:** `extensions/minimax/provider-registration.ts:22`
  - body: Provider registration currently reaches OAuth code through `oauth.runtime.ts`. Adding a value import from `./oauth.js` pulls the OAuth implementation into the eager registration path while refresh still dynamically imports the runtime module. Move the resolver behind the runtime export or into a lightweight helper that does not import the full OAuth implementation.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff adds no dependencies, but missing legacy region metadata can submit CN refresh tokens to the Global MiniMax endpoint.
