## Summary

Keep open: current main still routes deep plugin code-safety scans through the scanner without enabling test-file exclusion, and active PRs are still open rather than merged.

## What This Changes

The issue asks OpenClaw to stop reporting critical `env-harvesting` findings from bundled plugin test/spec/mock files that are not runtime plugin code.

## Best Possible Solution

Review and land one focused PR that excludes non-runtime test/spec/mock paths from deep plugin code-safety scans while preserving runtime-reachable coverage, then track string-literal masking separately if still needed.

## Reproduction Assessment

Yes. Source inspection shows current main still calls the deep scanner without `excludeTestFiles`, while the scanner default includes test files; the issue body gives concrete Discord test-file false-positive examples.

## Solution Assessment

Yes. Reusing the existing scanner exclusion option is the narrow fix, but the issue should stay open until one linked implementation PR is merged and runtime-reachable scan coverage is preserved.
