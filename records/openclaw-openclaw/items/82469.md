## Summary

Keep open: current main partially addresses the old whole-file env-harvesting problem with an 8-line context window, but the plugin deep-audit path still scans test files and the heuristic source still preserves string contents, so the reported false-positive class is not fully fixed or superseded.

## What This Changes

The issue asks for `plugins.code_safety` to stop raising critical env-harvesting false positives from bundled plugin test files and string-literal context.

## Best Possible Solution

Align deep plugin audit with the install scanner by excluding non-runtime test/mock/spec paths while still force-scanning manifest entries, and mask string literal contents for heuristic context with focused regression coverage.
