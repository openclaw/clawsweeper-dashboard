## Summary

Keep open: current main still routes deep plugin code-safety scans through the scanner without `excludeTestFiles`, the scanner default includes test files, and live GitHub shows an open implementation PR rather than a merged fix.

## What This Changes

The issue asks OpenClaw to stop reporting critical `env-harvesting` findings from bundled plugin test/spec/mock files during deep plugin code-safety audits, with optional follow-up hardening for string-literal and context-window matching.

## Best Possible Solution

Review and land one focused implementation, likely https://github.com/openclaw/openclaw/pull/82535 or a successor, that applies the existing test-file exclusion to deep plugin and skill code-safety scans while preserving explicit runtime entry scanning.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence because current main still scans plugin roots without `excludeTestFiles`, and the reported Discord test files still contain the env and network-context patterns that can trigger `env-harvesting`.

## Solution Assessment

Yes. Reusing the existing scanner `excludeTestFiles` option in the deep-audit caller is the narrow maintainable fix; the issue should stay open until the linked implementation PR or a successor is merged.
