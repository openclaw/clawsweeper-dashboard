## Summary

Keep open: current main still scans deep-audit plugin and skill directories without passing `excludeTestFiles`, and the only live implementation candidate is still an open PR rather than a merged fix.

## What This Changes

The issue asks for plugin code-safety deep audits to stop reporting critical `env-harvesting` false positives from bundled plugin test/spec/mock files and string-literal matches.

## Best Possible Solution

Review and land one focused implementation, likely https://github.com/openclaw/openclaw/pull/82535 or an equivalent replacement, that excludes non-runtime test/spec/mock paths while preserving manifest-entry scanning; track string-literal masking separately if it still reproduces.

## Reproduction Assessment

Yes. Source inspection shows current main still calls the scanner without `excludeTestFiles`, while the scanner default includes test files; I did not run a live audit because this review is read-only.

## Solution Assessment

Yes. Reusing the existing scanner test-exclusion option in the deep audit caller is the narrow maintainable fix for the central false positive, but the issue should stay open until a PR lands.
