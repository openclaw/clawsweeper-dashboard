## Summary

Keep open: current main appears to have mitigated the exact whole-file/string-literal Discord examples with the eight-line env-harvesting window, but the deep plugin audit still scans plugin test files because it never passes the scanner's existing test-file exclusion option.

## What This Changes

The issue requests narrowing deep plugin code-safety scanning so bundled plugin test/spec/mock files do not produce critical env-harvesting false positives from test-only process.env and network/mock code.

## Best Possible Solution

Make plugin deep-audit tree scans skip test/spec/mock files by default while still force-scanning manifest-declared entries, and add regression coverage for both behaviors.

## Reproduction Assessment

Yes, from source: collectPluginsCodeSafetyFindings scans plugin roots through getCodeSafetySummary without excludeTestFiles, while the scanner defaults that option to false. I did not execute the audit in this read-only review, so the structured status is source_reproducible rather than reproduced.

## Solution Assessment

Yes. The narrow fix is to pass excludeTestFiles: true for normal plugin deep-audit scans, include that option in any relevant cache key, and keep includeFiles as the escape hatch for manifest-declared runtime entries.
