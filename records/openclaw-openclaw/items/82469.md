## Summary

Keep open: current main still runs deep plugin and installed-skill code-safety scans without `excludeTestFiles`, while the scanner already supports that boundary and the active fix PR has not merged.

## What This Changes

The issue asks for deep plugin code-safety scans to avoid treating non-runtime test/spec/mock files and literal/mock patterns as critical `env-harvesting` findings.

## Best Possible Solution

Land one focused implementation that excludes non-runtime test/spec/mock paths from deep plugin and skill code-safety scans while preserving manifest/runtime-reachable coverage, then track string-literal masking separately if it still reproduces.

## Reproduction Assessment

Yes. Source inspection shows current main still calls the deep code-safety scanner without `excludeTestFiles`, while the scanner defaults that option off; I did not run a live audit because this review is read-only.

## Solution Assessment

Yes. Reusing the existing scanner test-exclusion option in the deep-audit helper is the narrow maintainable direction, provided the final patch keeps explicit/runtime-reachable entry coverage intact.
