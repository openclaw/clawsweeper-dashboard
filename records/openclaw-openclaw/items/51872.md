## Summary

Keep open: current main does not implement transport-level proxy auth for mobile Gateway connections, and open PR #73168 explicitly tracks part of the requested Android work with closing syntax.

## What This Changes

Request adds transport-level proxy authentication metadata to QR/bootstrap flows and mobile clients so Android/iOS can send proxy-required credentials before the Gateway WebSocket upgrade.

## Best Possible Solution

Keep the issue open until #73168 is reviewed or replaced, then ensure the accepted design covers the intended platform scope and setup-code contract without weakening Gateway TLS guarantees.

## Reproduction Assessment

Not applicable as a feature request. Source inspection shows current setup/mobile connection paths do not carry transport-level proxy auth headers or metadata.

## Solution Assessment
