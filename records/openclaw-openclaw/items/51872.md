## What This Changes

Request support for reverse-proxy transport authentication credentials in QR/setup payloads and Android/iOS gateway connections before the WebSocket upgrade.

## Best Possible Solution

Review and revise https://github.com/openclaw/openclaw/pull/73168, or replace it with a cross-platform design covering setup-code/manual config, Android/iOS pre-upgrade headers, secret storage/redaction, origin scoping, and OIDC/TLS policy.

Yes for the current capability gap by source inspection. The setup-code, QR CLI, Android config, Android WebSocket, and iOS shared WebSocket paths have no transport-auth credential path before the gateway connect frame.

Unclear. The requested direction is valid, but the maintainable fix needs maintainer/security decisions on schema shape, credential storage, origin-scoped header attachment, OIDC flow, and TLS trust policy.
