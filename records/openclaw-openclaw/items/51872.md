## Summary

Keep this issue open. Current main still lacks a mobile transport-auth path before the WebSocket upgrade, and https://github.com/openclaw/openclaw/pull/73168 is an open closing PR that should be reviewed, revised, landed, or closed first.

## What This Changes

The issue requests transport-level proxy authentication support so mobile clients can include reverse-proxy credentials before the gateway WebSocket upgrade.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/73168 as the current candidate, or replace it with an additive cross-platform design covering setup-code/manual config, Android/iOS pre-upgrade headers, secret storage/redaction, and OIDC/TLS policy.

## Reproduction Assessment

Yes. A high-confidence source-level check shows current setup-code, QR CLI, Android config models, Android WebSocket construction, and iOS shared WebSocket construction have no transport-auth credential path before the gateway connect frame.

## Solution Assessment

Unclear. The requested direction is valid, but the best maintainable solution needs maintainer/security judgment on the cross-platform schema, credential storage, header scoping, and whether Android-only bearer/basic support should land before broader proxy/OIDC support.
