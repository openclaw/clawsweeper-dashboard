## Summary

Keep this issue open. Current main still lacks mobile reverse-proxy credentials before the WebSocket upgrade, and open PR #73168 is an unmerged implementation candidate that uses closing syntax for this issue.

## What This Changes

Request mobile setup-code/manual configuration support for transport-level proxy credentials before Android/iOS gateway WebSocket upgrades.

## Best Possible Solution

Review PR #73168 as the current candidate, or replace it with an additive cross-platform transport-auth design covering setup-code/manual config, Android/iOS WebSocket construction, and secret-handling safeguards.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current setup-code and mobile config models have no proxy-auth fields, and Android/iOS create the WebSocket upgrade without caller-provided transport auth headers.

## Solution Assessment

Unclear. The requested direction is valid, but maintainers still need to decide whether to land Android-only bearer/basic support first or require a broader cross-platform schema that covers custom headers, cookies, and OIDC.
