## Summary

Keep open: current main and the latest release still match the reported timeout path for a Docker/self-hosted llama.cpp provider using a bare hostname, because the explicit provider timeout is converted to model request metadata but then clamped by the non-local idle-watchdog branch.

## What This Changes

The issue asks OpenClaw to honor the supported provider timeout for long local llama.cpp prefill instead of rejecting the legacy idle-timeout key and aborting at the default idle watchdog.

## Best Possible Solution

Keep the provider-scoped timeout contract and make explicit `models.providers.<id>.timeoutSeconds` reach the model idle watchdog for self-hosted/local OpenAI-compatible providers, including Docker bare-host aliases, without restoring `agents.defaults.llm`.

## Reproduction Assessment

Yes for the source-level mismatch: with `modelRequestTimeoutMs=14400000` and `baseUrl=http://cerebro-mac:8080/v1`, current main classifies the host as non-local and caps the idle watchdog at 120s. I did not run the full Docker and llama.cpp prefill path.

## Solution Assessment

Yes. The narrow maintainable fix is to align the existing provider-timeout contract with the idle watchdog for this self-hosted provider path, not to reintroduce the retired legacy key or add a new timeout hierarchy here.
