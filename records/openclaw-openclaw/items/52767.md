## Summary

Keep open. Current main and the latest release still run a non-empty `BOOT.md` on every configured gateway startup; the source path is clear, but changing that to durable run-once behavior would change the documented boot-md startup contract and needs maintainer product judgment.

## What This Changes

The issue requests changing the bundled boot-md startup hook so `BOOT.md` is not re-executed on every gateway restart, with durable idempotency and safer boot-session behavior.

## Best Possible Solution

Define BOOT.md's contract explicitly, then either add a compatibility-safe once/reset mode with isolated boot execution or keep repeatable startup behavior and steer one-time setup to BOOTSTRAP.md or idempotent BOOT.md checks.

## Reproduction Assessment

Yes, source-level. Enable boot-md, keep a non-empty BOOT.md in the resolved workspace, and restart the gateway twice; each startup dispatches `gateway:startup`, creates a fresh handler-local `seenWorkspaces` Set, and `runBootOnce` has no persisted completion check before `agentCommand`.

## Solution Assessment

Unclear. A sentinel file would satisfy durable run-once behavior, but it would change the documented startup checklist contract, so maintainers should first choose repeatable versus once-per-workspace semantics, reset behavior, and boot-session isolation.
