## Summary

Keep open. Current main and the latest release still execute a non-empty `BOOT.md` on every configured gateway startup; recent boot-session isolation work reduced session pollution, but it did not add durable run-once semantics, and docs/tests still describe `BOOT.md` as a repeatable startup checklist.

## What This Changes

The issue requests changing the bundled `boot-md` hook so `BOOT.md` is not re-executed unconditionally on every gateway restart.

## Best Possible Solution

Define and document the BOOT.md contract, then either add a compatibility-safe once/reset mode with regression coverage or keep repeatable startup behavior and steer one-time setup to BOOTSTRAP.md/idempotent BOOT.md guidance.

## Reproduction Assessment

Yes, source-level. Enable boot-md, keep a non-empty `BOOT.md` in the resolved workspace, and restart the gateway twice; each startup dispatches `gateway:startup`, creates a fresh handler-local `seenWorkspaces` set, and `runBootOnce` has no persisted completion check before `agentCommand`.

## Solution Assessment

Unclear. A sentinel or run-once mode would address repeated execution, but it would change the documented startup checklist contract; maintainers should choose repeatable versus once/reset semantics before a fix PR.
