## Summary

Keep open. Current main already handles the original doctor false-positive by excluding trajectory runtime artifacts, but this PR has evolved into a narrower directory-aware disambiguation that current main does not fully implement; the PR head still locks in a P2 no-context helper regression and has failing check runs, so it is not ready to merge or close as implemented.

## What This Changes

The PR updates session artifact classifiers and doctor, usage, disk-budget, and memory-host callers to treat `.trajectory.jsonl` as a sidecar only when a matching primary transcript is present, adds regression tests, and also carries node-host exec workdir handling changes.

## Best Possible Solution

Land a narrowed session-artifact fix that keeps no-context/public helper behavior conservative, makes directory-aware sidecar disambiguation explicit where a sibling filename set is available, consistently handles or reserves `.trajectory`-suffixed session IDs, and splits the node exec workdir behavior into a separate review if it is still needed.

## Reproduction Assessment

Yes. The linked bug has concrete doctor output for the original false positive, and the PR finding is reproducible directly from the PR head helpers: call `isUsageCountedSessionTranscriptFileName("abc.trajectory.jsonl")` without a filename set and it returns true, as the PR tests assert.

## Solution Assessment

No. Passing a directory filename set through the scanned call sites is the right direction, but making that context optional leaves exported and future no-context callers unsafe; a required-context classifier or conservative default is the safer maintainable fix.
