## Summary

Keep open for maintainer review. The PR addresses a real doctor/session artifact bug, but the current branch is not merge-ready because it makes no-context transcript helpers treat `*.trajectory.jsonl` as primary and usage-counted again, mixes in node-host exec behavior, and is currently dirty with failing check runs.

## What This Changes

The PR changes session artifact helpers and doctor/usage/disk-budget/memory-host scanners to treat `.trajectory.jsonl` as a sidecar only when a matching primary transcript exists, and also passes explicit node workdir state through the node-host exec path.

## Best Possible Solution

Land a narrowed session-artifact fix that makes directory-aware sidecar disambiguation explicit, keeps no-context/public helper behavior conservative, preserves valid dotted session IDs only where directory context is available, covers doctor/usage/disk-budget/memory-host behavior, and splits the node exec workdir change into a separate PR if still needed.

## Reproduction Assessment

Yes. The PR diff itself provides a high-confidence reproduction: no-context calls to the helper now classify `abc.trajectory.jsonl` as primary and usage-counted, and the linked #70680 report gives concrete doctor output showing why broad classification is dangerous.

## Solution Assessment

No. The directory-scanned call sites move in the right direction, but making context optional leaves exported and future no-context callers able to reintroduce the original maintenance/usage-integrity problem; a separate required-context classifier or conservative default is safer.
