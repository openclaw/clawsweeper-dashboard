## What This Changes

The issue reports a macOS Gateway regression where explicit `gateway.bind: "loopback"` can resolve to `0.0.0.0` and then fail startup under the loopback runtime guard.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81306 or an equivalent narrow patch that pins explicit `loopback` to `127.0.0.1`, keeps the runtime guard, and preserves other bind-mode fallbacks.

## Reproduction Assessment

## Solution Assessment

Yes: pinning explicit `loopback` to `127.0.0.1` while preserving `auto`, `tailnet`, `custom`, and the safety guard is the narrow maintainable fix. The issue should stay open until the linked PR or an equivalent patch merges.
