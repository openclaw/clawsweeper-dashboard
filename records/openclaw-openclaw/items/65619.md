## What This Changes

The issue requests fixing a macOS Gateway regression where explicit `gateway.bind="loopback"` can resolve to `0.0.0.0` and fail startup.

## Best Possible Solution

## Reproduction Assessment

Yes, source-level: if the current `127.0.0.1` bind preflight returns false, explicit loopback resolves to `0.0.0.0` and the runtime guard throws the reported startup error. I did not run a live macOS Gateway foreground reproduction in this read-only review.

## Solution Assessment
