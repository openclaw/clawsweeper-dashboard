## Summary

Keep open: current `origin/main` still drops Feishu card-action sibling fields before dispatch, and open implementation PRs already cover the active fix path, including one that closes this issue.

## What This Changes

Pass Feishu card-action sibling fields such as `form_value`, `input_value`, `name`, `option`, and `options` through to the synthetic agent message while preserving simple button behavior.

## Best Possible Solution

Land or replace the linked Feishu PR with an end-to-end parser, decoder, fallback, handler, changelog, and regression-test fix, then close this issue after merge.

## Reproduction Assessment

Yes, source-level. A raw `card.action.trigger` payload with sibling form/select/input fields is normalized to `event.action.value` and `event.action.tag` only, and fallback text then serializes only `action.value`.

## Solution Assessment

Yes at the direction level. The narrow maintainable fix is to preserve the fields at the Feishu parser/event boundary and carry them through fallback dispatch; a handler-only patch is insufficient because production callbacks are normalized first.
