## Summary

Keep open: current main and v2026.6.6 still normalize Feishu card actions to `action.value`/`tag` only, while an open closing PR already owns this issue’s implementation path and still needs a maintainer-selected payload contract.

## What This Changes

The issue asks Feishu card action callbacks to pass through form, input, name, option, and multi-select fields instead of sending only the button value.

## Best Possible Solution

Land one Feishu plugin patch that preserves raw callback fields at the parser boundary, carries them through decoder/handler without bypassing structured-action safeguards, keeps command routing compatible, and covers it with focused tests plus redacted live Feishu proof.

## Reproduction Assessment

Yes by source inspection: live `card.action.trigger` callbacks are normalized to `value`/`tag` only, and fallback dispatch reads only `action.value`. I did not run a live Feishu callback in this read-only review.

## Solution Assessment

No implementation is on main yet. The best fix is plugin-local but must start at the parser boundary and preserve current structured-action validation, replay, and command-routing behavior.
