## Summary

Keep open. Current main and latest v2026.5.3 still drop Feishu card-action fields outside `action.value`, and open PR #43953 directly closes this issue, so the issue should remain open until that PR or a superseding Feishu fix lands.

## What This Changes

Request to pass Feishu card-action `form_value`, `input_value`, `name`, `option`, and `options` into the synthetic message while keeping simple button value behavior unchanged.

## Best Possible Solution

Review and land or supersede the linked Feishu fix so the parser, event type, decoder/fallback, handler, and regression tests preserve the requested fields without changing value-only button behavior.

## Reproduction Assessment

Yes, as a source-level reproduction: a `card.action.trigger` payload with `action.form_value` or `action.input_value` is parsed into `action.value` and `action.tag` only, leaving the handler no access to those fields.

## Solution Assessment

Yes, the narrow maintainable fix belongs in the Feishu plugin parser/type/decoder/fallback/handler path with regression coverage. A handler-only change would be incomplete because the live parser drops the sibling fields first.
