## Summary

Keep open: current main and v2026.6.6 still drop the requested Feishu card-action fields before agent dispatch, and https://github.com/openclaw/openclaw/pull/43953 remains an open closing PR that should be reviewed or replaced before this issue closes.

## What This Changes

The issue requests Feishu card-action callbacks preserve form_value, input_value, name, option, and options in the synthetic message sent to agents while keeping simple button clicks compatible.

## Best Possible Solution

Land one Feishu plugin patch that preserves raw callback fields at the parser boundary, carries them through the decoder and handler without bypassing structured-action safeguards, keeps command routing compatible, and covers it with focused tests plus redacted live Feishu proof.

## Reproduction Assessment

Yes, at source level: live card.action.trigger callbacks are normalized to action.value/tag only and fallback dispatch reads only action.value. I did not run a live Feishu callback in this read-only review.

## Solution Assessment

No implementation has landed on main yet. The requested direction is valid, but the maintainable fix should start at the raw parser boundary and preserve current structured-action validation, replay, and command-routing behavior.
