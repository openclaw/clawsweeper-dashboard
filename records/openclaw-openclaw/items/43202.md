## Summary

Keep open: current `main` and the latest release still normalize Feishu card actions down to `action.value`/`tag`, and the issue is already paired with active implementation PRs, including https://github.com/openclaw/openclaw/pull/43953 with closing syntax.

## What This Changes

The issue asks the Feishu plugin to preserve card callback fields such as `form_value`, `input_value`, `name`, `option`, and `options` instead of only forwarding `action.value`.

## Best Possible Solution

Review, land, or replace one canonical Feishu plugin patch that preserves the missing fields at the parser boundary, carries them through fallback/handler dispatch, adds regression coverage, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes, source-level. A raw `card.action.trigger` payload with sibling form/input/select fields is normalized to only `action.value` and `action.tag`, and fallback text then serializes only `action.value`.

## Solution Assessment

No, not as a handler-only change. The durable fix should start at `parseFeishuCardActionEventPayload` and then use one selected payload contract across the overlapping Feishu PRs.
