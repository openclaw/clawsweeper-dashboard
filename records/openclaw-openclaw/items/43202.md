## Label Justifications

- `P2`: Feishu interactive card data loss affects a real plugin workflow but is scoped to one channel/plugin area.
- `impact:data-loss`: Structured form and selector values from Feishu callbacks are discarded before the agent can use them.
- `impact:message-loss`: The missing callback fields are part of the inbound Feishu card-action message payload.

## Summary

Keep open: current main and v2026.5.19 still drop Feishu card-action sibling fields, and this issue is already paired with open implementation PRs that need maintainer selection rather than early closure.

## What This Changes

Requests that Feishu card-action callbacks preserve fields such as form_value, input_value, name, option, and options in the synthetic message sent to agents.

## Best Possible Solution

Select one canonical Feishu plugin patch that preserves supported sibling action fields at the parser boundary, carries them through decode/fallback/handler dispatch, adds regression coverage, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes, source-level. A raw card.action.trigger payload with form/input/select sibling fields is normalized to only action.value and action.tag, then fallback dispatch serializes only action.value.

## Solution Assessment

No for a handler-only fix. The durable fix should preserve fields at parseFeishuCardActionEventPayload and align the event type, decoder/fallback behavior, handler dispatch, tests, and selected payload contract.
