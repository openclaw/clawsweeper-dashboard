## Summary

Keep open. Current main and v2026.6.10 still reduce Feishu card.action.trigger callbacks to action.value/tag, and https://github.com/openclaw/openclaw/pull/43953 is the open closing-fix candidate that should be reviewed or refreshed before this issue closes.

## What This Changes

Request pass-through of Feishu card action form, input, name, option, and options fields into the synthetic agent message while preserving simple button behavior.

## Best Possible Solution

Land or replace the Feishu-plugin fix that preserves raw form/input/select fields through the parser and fallback path with focused tests and redacted real Feishu proof.

Yes from source: the current Feishu card-action parser returns only action.value/tag and the fallback text builder reads only action.value. I did not run a live Feishu callback in this read-only review.

## Solution Assessment

Yes on owner boundary: preserving these fields belongs in the Feishu plugin parser/fallback/handler with focused tests. The exact payload contract still needs maintainer review, so the open linked PR should be reviewed or refreshed rather than duplicating the work.
