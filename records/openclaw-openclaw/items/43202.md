## Label Justifications

- `P2`: Feishu interactive-card payload loss affects real plugin workflows but is scoped to one channel plugin.
- `impact:data-loss`: Structured form, input, and selector values from Feishu callbacks are discarded before agents can use them.
- `impact:message-loss`: The synthetic channel message created from the card callback omits part of the inbound user interaction payload.

## Summary

Keep open: current main and latest v2026.5.20 still drop Feishu card-action sibling fields before agent dispatch; open Feishu PRs already target this behavior, so this issue should stay open until maintainers land one canonical payload contract.

## What This Changes

The issue asks the Feishu plugin to pass through card-action form, input, selector, and component fields instead of only forwarding action.value.

## Best Possible Solution

Land one canonical Feishu plugin patch that preserves selected action fields from parser through decoder/fallback/handler dispatch, keeps legacy command handling compatible, adds regression coverage and live proof, then close this issue after merge.

## Reproduction Assessment

Yes, source-level. A raw card.action.trigger payload with form/input/select sibling fields is normalized to only action.value and action.tag, then fallback dispatch serializes only action.value.

## Solution Assessment

No final implementation is on main yet. A handler-only change would be incomplete; the durable fix should start at parseFeishuCardActionEventPayload and align the event type, decoder/fallback behavior, handler dispatch, tests, and selected payload shape.
