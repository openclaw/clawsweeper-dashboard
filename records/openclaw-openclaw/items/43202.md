## Summary

Keep open. Current main and the latest release still type, parse, decode, and dispatch Feishu card actions through `action.value` only, while the supplied context shows open implementation PRs that directly target this issue.

## What This Changes

Request to pass Feishu card action sibling fields such as `form_value`, `input_value`, `name`, `option`, and `options` through to the synthetic agent message while preserving simple button behavior.

## Best Possible Solution

Review and land or supersede the open Feishu fix PR so the parser, event type, decoder/fallback, handler, and regression tests preserve the requested fields without changing value-only button behavior.

## Reproduction Assessment

Yes, as a source-level reproduction: a `card.action.trigger` payload with `action.form_value` or `action.input_value` is narrowed by `parseFeishuCardActionEventPayload` to `action.value` and `action.tag`, after which the handler has no access to the extra fields.

## Solution Assessment

Yes, the narrow maintainable fix belongs in the Feishu plugin parser/type/decoder/fallback/handler path with regression coverage. A handler-only change would be incomplete because the live parser currently drops the sibling fields first.
