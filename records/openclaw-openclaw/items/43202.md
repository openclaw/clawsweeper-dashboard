## Summary

Keep open: current main and v2026.5.12 still drop Feishu card-action sibling fields before the agent sees them, and this issue is paired with open implementation work that uses closing syntax. The remaining action is to review, select, and land one Feishu payload-contract fix rather than close this issue early.

## What This Changes

Pass Feishu card-action sibling fields such as form_value, input_value, name, option, and options through to the synthetic agent message while preserving simple button-click behavior.

## Best Possible Solution

Select and land one canonical Feishu plugin patch that preserves supported sibling action fields at the parser boundary, carries them through decoder/fallback/handler dispatch, adds focused regression coverage, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes, source-level. A raw card.action.trigger payload with form/input/select sibling fields is normalized to only action.value and action.tag, then fallback dispatch serializes only action.value.

## Solution Assessment

No for the handler-only proposed fix. The maintainable fix should preserve these fields in parseFeishuCardActionEventPayload, then align the event type, decoder/fallback behavior, handler dispatch, tests, and the selected payload contract.
