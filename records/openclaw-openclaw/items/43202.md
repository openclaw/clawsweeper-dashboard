## Summary

Keep open: current main still strips Feishu card-action fields outside `action.value`, and open implementation PRs already cover the active fix path, including one that closes this issue.

## What This Changes

Pass through Feishu card-action `form_value`, `input_value`, `name`, `option`, and `options` fields to the synthetic agent message while preserving simple button behavior.

## Best Possible Solution

Land or replace a single canonical Feishu plugin patch that preserves the missing fields at the raw parser boundary, carries them through fallback/handler dispatch with regression tests, and then close this issue after merge.

## Reproduction Assessment

Yes, source-level. A raw `card.action.trigger` payload with sibling form/input/select fields is normalized to `action.value` and `action.tag` only, and fallback text then serializes only `action.value`.

## Solution Assessment

Yes at the direction level, but the fix should start at the Feishu parser/event boundary rather than only in the handler. The exact structured JSON versus command-text payload contract still needs maintainer selection across the open PRs.
