## Summary

Keep open. Current main and latest release still narrow Feishu card actions to `action.value`/`tag` before dispatch, and open PR #43953 directly closes this issue with overlapping work in #42908.

## What This Changes

Pass Feishu card action fields such as `form_value`, `input_value`, `name`, `option`, and `options` through to the synthetic agent message while preserving button-only behavior.

## Best Possible Solution

Review and land or supersede the linked Feishu fix PR so the parser, event type, decoder/fallback, handler, and regression tests preserve the requested fields without changing value-only button behavior.

## Reproduction Assessment

Yes, as a source-level reproduction: a `card.action.trigger` payload with `action.form_value` or `action.input_value` is narrowed by the parser to `action.value` and `action.tag`, leaving the handler no access to those fields.

## Solution Assessment

Yes, the narrow maintainable fix belongs in the Feishu plugin parser/type/decoder/fallback/handler path with regression coverage. A handler-only change would be incomplete because the live parser currently drops the sibling fields first.
