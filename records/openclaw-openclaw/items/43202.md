- `impact:data-loss`: Structured form/select values from Feishu callbacks are discarded before the agent can use them.
- `impact:message-loss`: The missing callback fields are part of the inbound Feishu card-action message payload.

## Summary

Keep open: current main and the latest tagged release still drop Feishu card-action sibling fields, and this issue is paired with open implementation PRs that need maintainer selection rather than early closure.

## What This Changes

The issue asks for Feishu card-action callbacks to preserve form_value, input_value, name, option, options, and similar action sibling fields so agents receive interactive card form/select data.

## Best Possible Solution

Select one canonical Feishu plugin patch that preserves supported sibling action fields at parseFeishuCardActionEventPayload, carries them through fallback/handler dispatch, adds regression coverage, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes, source-level. A raw card.action.trigger payload with form/input/select sibling fields is normalized to only action.value and action.tag, then fallback dispatch serializes only action.value.

## Solution Assessment

No for a handler-only fix. The durable fix should preserve fields at parseFeishuCardActionEventPayload and align the event type, decoder/fallback behavior, handler dispatch, tests, and selected payload contract.
