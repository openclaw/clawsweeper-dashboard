## Summary

Keep open: current `main` and the latest release still drop Feishu card-action sibling fields before dispatch, and this issue is already paired with active implementation PRs, including https://github.com/openclaw/openclaw/pull/43953 with closing syntax.

## What This Changes

Feishu card action callbacks should pass through `form_value`, `input_value`, `name`, `option`, `options`, and similar action fields to the synthetic agent message while preserving simple button behavior.

## Best Possible Solution

Review, land, or replace one canonical Feishu plugin patch that preserves sibling fields at the parser boundary through fallback/dispatch, with focused regression coverage and redacted live Feishu proof.

## Reproduction Assessment

Yes, source-level. A raw `card.action.trigger` payload with sibling form/input/select fields is normalized to only `action.value` and `action.tag`, and fallback text then serializes only `action.value`.

## Solution Assessment

No, not as a handler-only change. The durable fix should preserve fields in `parseFeishuCardActionEventPayload` before `handleFeishuCardAction` and then use one selected payload contract for parser, fallback, handler, and tests.
