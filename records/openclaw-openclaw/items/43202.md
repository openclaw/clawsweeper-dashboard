- `impact:data-loss`: Feishu callback fields containing form/input/selection data are silently discarded before agents can consume them.
- `impact:message-loss`: The synthetic card-action message can omit meaningful parts of the user's Feishu card interaction payload.

## Summary

Keep open. Current main and the latest release still strip Feishu card callback fields down to `action.value` and `tag`, while https://github.com/openclaw/openclaw/pull/43953 explicitly closes this issue but remains open and conflicting.

## What This Changes

The issue requests preserving Feishu card callback fields such as `form_value`, `input_value`, `name`, `option`, and `options` in the synthetic agent message.

## Best Possible Solution

Land or replace one narrow Feishu plugin fix that preserves the raw callback fields at the parser boundary, forwards them in a deliberate agent-facing shape, and keeps plain button behavior unchanged.

## Reproduction Assessment

Yes, source-level. Current main and `v2026.6.6` normalize card-action callbacks to `action.value`/`tag` and build fallback text from `action.value` only; I did not run a live Feishu tenant callback in this read-only review.

## Solution Assessment

Mostly. The issue points at the right Feishu plugin boundary, but maintainers still need to choose the permanent payload shape before landing one of the linked PRs or a replacement.
