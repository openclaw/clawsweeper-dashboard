## Summary

Keep open. Current main and the latest release still normalize Feishu card callbacks down to `action.value` and `action.tag`, while the linked fixing PR remains open and needs maintainer refresh or replacement before this issue can close.

## What This Changes

Pass Feishu card-action `form_value`, `input_value`, `name`, `option`, and `options` fields through to the synthetic agent message while preserving plain button behavior.

## Best Possible Solution

Land or replace one narrow Feishu plugin fix that preserves the raw callback fields at the parser boundary, forwards them in a deliberate agent-facing shape, and keeps plain button behavior unchanged.

## Reproduction Assessment

Yes, source-level: `parseFeishuCardActionEventPayload` returns only `action.value` and `action.tag`, and `buildFeishuCardActionTextFallback` serializes only `action.value`. I did not run a live Feishu tenant callback in this read-only review.

## Solution Assessment

No, closing is not the best solution because current main and the latest release still drop the requested fields. The right path is a refreshed or replacement Feishu plugin PR that preserves the raw callback fields and settles the payload shape.
