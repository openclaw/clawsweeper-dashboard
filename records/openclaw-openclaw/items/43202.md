## Label Justifications

- `P2`: Feishu interactive card payload loss affects real plugin workflows but is scoped to one channel plugin.
- `impact:data-loss`: Structured form and selector values from Feishu callbacks are discarded before plugins or agents can use them.
- `impact:message-loss`: The synthetic channel message created from the card callback omits part of the inbound user interaction payload.

## Summary

Keep open: current main and v2026.5.20 still drop the requested Feishu card-action sibling fields, and open implementation PRs already target this issue so maintainers need to choose the canonical payload contract before it can close.

## What This Changes

Pass Feishu card.action.trigger sibling fields such as form_value, input_value, name, option, and options through to the agent-visible synthetic message while preserving simple button behavior.

## Best Possible Solution

Land one canonical Feishu plugin patch that preserves selected sibling action fields at the parser boundary, carries them through decoder/fallback/handler dispatch, keeps legacy command handling compatible, adds regression coverage, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes, source-level. A raw card.action.trigger payload with form/input/select sibling fields is normalized to only action.value and action.tag, then fallback dispatch serializes only action.value.

## Solution Assessment

No final implementation is on main yet. A handler-only change would be incomplete; the maintainable fix should start at parseFeishuCardActionEventPayload and align the event type, decoder/fallback behavior, handler dispatch, tests, and selected payload shape.
