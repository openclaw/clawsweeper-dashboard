## Summary

Keep open: the warning direction is useful and the PR includes real CLI/API behavior proof, but the current trust check suppresses the new warning when a local skill carries a self-authored `.clawhub/origin.json`, which undercuts the security goal for sideloaded skills.

## What This Changes

The PR adds skill trust fields, a `skills.load.trustedDirs` config option, CLI/Control UI warning rendering, docs, changelog, and tests for untrusted local skill sources.

## Best Possible Solution

Keep the status/API warning path, but only suppress local-source warnings for provenance OpenClaw can verify or for explicitly configured reviewed directories.

## Reproduction Assessment

Yes. The blocker is source-reproducible on the PR head: `resolveSkillTrustStatus` trusts a local `.clawhub/origin.json` before local-source checks, and the new test demonstrates that such a local marker suppresses the warning.

## Solution Assessment

No, not yet. The additive status surface is maintainable, but the trust decision should not treat a locally writable origin marker as sufficient proof of ClawHub provenance for a security warning.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Do not trust local origin files as ClawHub proof:** `src/agents/skills-status.ts:306-307`
  - body: The new classifier suppresses the untrusted-local warning as soon as a skill directory contains a shape-valid `.clawhub/origin.json`. That file lives inside the same local skill tree, so a copied or malicious local skill can include it and appear trusted without registry/source verification, defeating the warning this PR adds for sideloaded skills. Keep warning for local skills unless the provenance is verified by an OpenClaw-owned install record or another trusted source.
  - confidence: 0.88
