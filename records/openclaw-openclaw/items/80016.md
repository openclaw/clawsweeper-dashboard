## Summary

Keep open. The warning/status direction addresses the linked security request, but the patch currently lets locally writable `.clawhub/origin.json` metadata suppress the warning without verified provenance, weakening the new security signal.

## What This Changes

The PR adds skill trust metadata, a `skills.load.trustedDirs` config option, CLI and Control UI warnings, docs, changelog, generated config baseline hash updates, and tests for untrusted local skill sources.

## Best Possible Solution

Keep the additive status and warning surface, but suppress warnings only for verified OpenClaw/ClawHub provenance or explicitly configured reviewed directories.

## Reproduction Assessment

Yes. On the PR head, a local skill can include a shape-valid `.clawhub/origin.json`; `readClawHubOriginMarker` returns true and `resolveSkillTrustStatus` marks it trusted before local-source warning logic.

## Solution Assessment

No, not yet. Status-layer warnings are a good narrow surface, but trusting a locally writable origin marker is not the right security boundary without verified provenance or an explicit trusted directory.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Do not trust local origin files as ClawHub proof:** `src/agents/skills-status.ts:306-307`
  - body: The new classifier suppresses the untrusted-local warning as soon as a skill directory contains a shape-valid `.clawhub/origin.json`. That file is inside the local skill tree, so a copied or malicious local skill can include it and be reported as trusted without registry or install-record verification. Keep the warning for local skills unless provenance is verified or the directory is explicitly trusted.
  - confidence: 0.88
