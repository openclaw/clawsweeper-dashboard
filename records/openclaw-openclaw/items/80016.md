## Summary

Keep this PR open. The warning/status direction is useful and the supplied live output is enough real behavior proof, but the current patch still lets locally writable ClawHub origin metadata suppress the untrusted-local warning, which is the central security decision for the linked request.

## What This Changes

The PR adds skill trust metadata, a `skills.load.trustedDirs` config option, and CLI/Control UI/docs/tests that warn for untrusted local skill sources.

## Best Possible Solution

Keep the status/API warning surface, but only suppress local-source warnings for provenance OpenClaw can verify or for directories explicitly configured as reviewed and trusted.

## Reproduction Assessment

Yes. The blocker is source-reproducible on the PR head: a local `.clawhub/origin.json` marker is accepted before local-source warning classification, and the new test encodes that bypass as expected behavior.

## Solution Assessment

No, not yet. The additive status surface is maintainable, but a locally writable origin marker should not be enough to suppress a security warning without verified ClawHub provenance or an explicit trusted directory.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Do not trust local origin files as ClawHub proof:** `src/agents/skills-status.ts:306-307`
  - body: The classifier suppresses the untrusted-local warning as soon as a skill directory contains a shape-valid `.clawhub/origin.json`. That file lives inside the local skill tree, so a copied or malicious local skill can include it and appear trusted without registry or OpenClaw-owned install provenance, defeating the warning this PR adds for sideloaded skills.
