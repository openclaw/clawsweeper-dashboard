## Summary

Keep open: the PR is focused and useful, and its text proof is sufficient for the status/CLI path, but the new ClawHub trust check can be bypassed by a local skill that ships a forgeable origin marker.

## What This Changes

The PR adds skill trust metadata plus CLI, Control UI, config, docs, tests, and changelog updates for warning on untrusted local skill sources.

## Best Possible Solution

Keep the additive status/API direction, but only suppress local warnings for non-self-asserted ClawHub provenance or explicit trustedDirs, with negative tests for forged and mismatched origin metadata.

## Reproduction Assessment

Yes for the blocking review finding: source inspection of the PR head shows any local skill can include a shaped `.clawhub/origin.json` and be classified as `clawhub`. I did not run local tests because this review is read-only and `pnpm` is unavailable in the checkout environment.

## Solution Assessment

No. Status-layer warnings are a reasonable maintainable layer, but the current implementation treats self-supplied local metadata as trusted ClawHub provenance, which undercuts the warning it adds.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Treat forgeable ClawHub markers as untrusted:** `src/agents/skills-status.ts:269-276`
