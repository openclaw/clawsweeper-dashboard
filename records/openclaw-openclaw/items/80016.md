## Summary

Keep open. The warning/status surface is useful and the supplied live output is adequate, but the branch still lets a local skill self-attest as ClawHub by adding `.clawhub/origin.json`, which undermines the security warning this PR adds.

## What This Changes

The PR adds skill trust metadata, a `skills.load.trustedDirs` config option, and CLI/Control UI/docs/tests that warn for untrusted local skill sources.

## Best Possible Solution

Keep the additive status/API warning, but suppress local-source warnings only for verifiable ClawHub provenance or explicitly configured reviewed directories, then decide separately whether install/load confirmation is required.

## Reproduction Assessment

Yes. On the PR head, source inspection and the new test show a workspace skill can write shape-valid `.clawhub/origin.json` and be classified as trusted ClawHub with no warning.

## Solution Assessment

No, not yet. The status surface is maintainable, but the current classifier makes the security signal self-attestable; verified provenance or explicit `skills.load.trustedDirs` is the safer boundary.
