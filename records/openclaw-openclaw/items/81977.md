## Summary

Current main already implements the requested Gateway client-tool loop: `/v1/chat/completions` parses client `tools`, passes them into the agent run as delegated client tools, and returns pending calls as OpenAI-shaped `tool_calls`. The proof commit is on main only; no release tag in the checkout contains it yet.

## What This Changes

Request to make Gateway `/v1/chat/completions` preserve client-provided function tools and return resulting `tool_calls` for client-side execution.

## Best Possible Solution

Keep the current main implementation; if a tagged release still fails, track that as a release-version-specific bug with the exact OpenClaw version and Gateway response body.

## Reproduction Assessment

Not applicable as a feature cleanup item. I did not run a live Gateway request, but current source, docs, and regression tests show the requested client-tool behavior is already implemented on main.

## Solution Assessment

Yes. The current implementation is the narrow maintainable solution: it delegates client tools back to the caller and returns OpenAI-compatible `tool_calls` without adding a separate `toolMode` config/header.
