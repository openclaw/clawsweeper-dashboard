## What This Changes

The issue asks for `isolatedSession=true` heartbeat ticks to be truly fresh/bounded and not replay prior heartbeat context-engine state.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/84248; the desired end state is a per-tick context-engine identity for isolated heartbeats while keeping the stable delivery `SessionKey`.

## Reproduction Assessment

No executed current-main overflow reproduction was run in this read-only review. Source inspection gives a high-confidence path: isolated heartbeats get fresh session ids while context-engine lifecycle calls still receive the stable `<base>:heartbeat` session key.

## Solution Assessment

Yes. The narrow maintainable fix is to honor the existing docs by giving isolated heartbeat context-engine state a fresh per-tick identity while preserving stable delivery routing, not to add a broad new config knob first.
