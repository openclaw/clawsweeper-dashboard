## What This Changes

The issue reports that `isolatedSession=true` heartbeat ticks can reuse prior heartbeat context-engine state despite the documented fresh-session contract.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84248, or an equivalent fix that gives isolated heartbeat ticks a fresh context-engine identity while preserving stable heartbeat delivery routing; close this issue only after that fix reaches main.

## Reproduction Assessment

No executed current-main live reproduction was run in this read-only review. Source inspection gives a clear reproduction path once prior heartbeat context accumulates: current main creates fresh heartbeat session ids but forwards the stable `:heartbeat` key into context-engine lifecycle calls.

## Solution Assessment

Yes, the best direction is to honor the existing docs by isolating context-engine state per heartbeat tick while keeping the stable routing key. The linked PR implements that direction, so the issue should wait for that PR to land or be rejected.
