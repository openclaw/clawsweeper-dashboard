## Summary

Keep open: current main still matches the reported source path, with isolated heartbeat runs using a stable heartbeat session key for context-engine assembly despite docs promising no prior conversation history; the linked fix PR is open and should be reviewed before closing the issue.

## What This Changes

The issue requests that heartbeat runs with `isolatedSession=true` stop replaying prior heartbeat context so repeated context-overflow and restart loops cannot accumulate.

## Best Possible Solution

Review and land the linked fix PR or an equivalent patch that gives isolated heartbeat ticks a fresh context-engine-only identity while preserving the stable heartbeat delivery key and regression coverage.

## Reproduction Assessment

No executed current-main reproduction was established in this read-only review. Source inspection gives a clear path: isolated heartbeat ticks get fresh session ids while context-engine lifecycle calls still receive the stable `<base>:heartbeat` session key.

## Solution Assessment

Yes: the narrow maintainable direction is to honor the existing docs by splitting context-engine identity from stable delivery routing, which is what the linked PR attempts.
