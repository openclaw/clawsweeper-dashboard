## Summary

Keep open: current main still shows a credible plugin SDK lifecycle failure for memory corpus supplements registered from a service start path, and two open PRs already explicitly target this issue, so it should remain open until maintainers review and land or reject the fix direction.

## What This Changes

The issue reports that a non-bundled plugin's `MemoryCorpusSupplement` registered from `api.registerService({ start })` disappears before `memory_search` after a lazy plugin re-register.

## Best Possible Solution

Settle the SDK lifecycle and land one regression-tested loader or SDK fix that keeps legitimate memory corpus supplement registrations available to `memory_search`.

## Reproduction Assessment

Yes, by source inspection. A plugin can register a service during synchronous `register()`, but the service starts later with a captured guarded API, and `memory_search` returns no supplement results when `listMemoryCorpusSupplements()` is empty.

## Solution Assessment

Unclear. The open state-preservation PRs are plausible, but the final fix should also account for the guarded API/service-start lifecycle or explicitly document and diagnose register-only usage.
