## Summary

Keep open: current main still has the loader path that can clear memory corpus supplements during an activating reload, and open PR #77065 explicitly uses closing syntax for this issue but has not merged.

## What This Changes

The issue requests that `registerMemoryCorpusSupplement` registrations remain available to `memory_search` after lazy plugin re-registration, including registrations that are not re-added on a later activating load.

## Best Possible Solution

Review and land #77065 if its restore semantics match the intended plugin SDK lifecycle; otherwise document the supported registration window and add a runtime-safe corpus supplement registration path with regression coverage.

## Reproduction Assessment

Yes, via source inspection rather than a live run. An activating reload clears memory plugin state, service registration is idempotent, and `memory_search` returns no supplement results when the corpus supplement list is empty.

## Solution Assessment

Unclear. Preserving still-active plugin supplements across activating reloads appears like the narrow fix, but maintainers should review #77065's restore semantics against the intended service/runtime registration contract before this issue closes.
