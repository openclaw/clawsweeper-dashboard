## Summary

Keep open: current main still leaves the service-start memory supplement lifecycle unresolved, and open PR #77065 explicitly targets this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that memory corpus supplements registered from a plugin service start callback can disappear or remain unavailable after later plugin loading triggered by lazy memory_search use.

## Best Possible Solution

Review and land PR #77065 or an equivalent lifecycle-safe fix with regression coverage, ensuring the final behavior either supports service-start corpus registration or clearly diagnoses that it must happen during synchronous plugin registration.

## Reproduction Assessment

Yes. Source inspection shows service start callbacks run after guarded plugin registration closes, and activating plugin loads can clear memory supplement state before memory_search reads the current supplement list.

## Solution Assessment

Unclear. PR #77065 targets the activating reload wipe, but maintainers should verify it also covers or intentionally documents the service.start registration lifecycle before treating the issue as fully fixed.
