## What This Changes

The issue reports that memory corpus supplements registered from a plugin service start callback disappear after lazy non-activating plugin re-registration, causing `memory_search` for `wiki` or `all` to return no supplement results.

## Best Possible Solution

Review and land one canonical fix or SDK-contract clarification that preserves valid memory corpus supplement registrations across lazy reloads and adds regression coverage for the service-start scenario.

## Reproduction Assessment

Yes. Source inspection plus the reporter's trace gives a high-confidence reproduction path, but I did not run a live plugin reproduction in this read-only review.

## Solution Assessment

Unclear. The narrow code fix depends on whether maintainers want `registerMemoryCorpusSupplement` to be valid from `service.start()` or want docs/diagnostics to require synchronous registration inside `register(api)`.
