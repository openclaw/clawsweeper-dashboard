## Summary

Keep this PR open, but the source-only change is not mergeable as written because it collapses the default-vs-gateway-bindable registry boundary that current tests and runtime code intentionally preserve.

## What This Changes

The PR removes `runtimeSubagentMode` from the plugin loader cache key in `src/plugins/loader.ts` so equivalent plugin loads can reuse cached registries across gateway-binding modes.

## Best Possible Solution

Implement asymmetric reuse that can reuse a gateway-bindable registry for default callers when safe, but never serve a default-built registry to a gateway-bindable load; add regression coverage and a changelog entry.

## Reproduction Assessment

Yes. The focused reproduction is static and test-backed: remove `runtimeSubagentMode` from the cache key, then the default-mode compatibility test and the cache-miss expectation for gateway binding no longer match the runtime boundary.

## Solution Assessment

No. The proposed one-line cache-key collapse is too broad because `allowGatewaySubagentBinding` changes the runtime object created for plugins, especially subagent and node late binding.
