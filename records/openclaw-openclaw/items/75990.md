## Summary

Keep open. Current main does appear to have regressed the earlier cache-key fix, but this PR is not merge-ready because removing the key dimension also changes active-registry compatibility and leaves tests/changelog out of sync.

## What This Changes

The PR removes `runtimeSubagentMode` from the plugin loader registry cache-key string and adds an explanatory comment in `src/plugins/loader.ts`.

## Best Possible Solution

Keep the cache reuse fix, but preserve the active runtime-mode invariant, restore the #61854-style regression tests, and add a single Unreleased changelog fix entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: after applying this diff, `getCompatibleActivePluginRegistry()` can match a gateway-bindable request against a default-mode active registry by cache key before runtime-mode checks, and the existing loader/runtime-registry tests show the affected invariants.

## Solution Assessment

No. Excluding `runtimeSubagentMode` from the loader cache key is the right direction for redundant imports, but this patch also needs an active-registry compatibility guard or activation path so gateway binding state is not skipped.
