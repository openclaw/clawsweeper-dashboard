## Summary

Keep open. The performance problem is credible, but this PR’s one-line cache-key collapse breaks the current gateway-bindable runtime boundary and leaves focused tests plus the required changelog update behind.

## What This Changes

The branch removes `runtimeSubagentMode` from `buildCacheKey()` in `src/plugins/loader.ts` and adds comments explaining the intended cache reuse.

## Best Possible Solution

Implement asymmetric registry reuse that reduces duplicate plugin reloads while never serving a default-built registry to a gateway-bindable caller, then cover that boundary in tests and changelog.

## Reproduction Assessment

Yes. The patch-level reproduction is static and test-backed: collapsing the key makes default and gateway-bindable cache contexts match, contradicting current tests that require default-to-gateway reuse to miss.

## Solution Assessment

No. Removing the key dimension is too broad because `allowGatewaySubagentBinding` changes the plugin runtime object; the safer fix is asymmetric reuse from gateway-bindable to default only when compatibility checks prove it safe.
