## Summary

Keep open. The PR targets a credible duplicate plugin-load regression, but the one-line cache-key collapse is too broad because gateway-bindable plugin runtimes are not symmetric with default runtimes and current tests encode that boundary.

## What This Changes

The PR removes `runtimeSubagentMode` from the plugin loader cache-key string in `src/plugins/loader.ts` to dedupe registry loads across `allowGatewaySubagentBinding` modes.

## Best Possible Solution

Implement asymmetric reuse that can reuse a gateway-bindable registry for default or snapshot callers when scope matches, but never serve a default-built registry to a gateway-bindable load; update focused tests and changelog.

## Reproduction Assessment

Yes. Apply this PR's cache-key collapse and the current default-mode compatibility test plus the gateway-binding cache-miss expectation no longer match the runtime boundary.

## Solution Assessment

No. The proposed cache-key collapse is not the narrowest maintainable fix because `allowGatewaySubagentBinding` changes plugin runtime subagent and node behavior; a compatibility-aware cache reuse path is safer.
