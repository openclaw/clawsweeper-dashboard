## Summary

Keep open because current main only partially addresses the burst side with centralized event-wake cooldowns; the generated macOS launchd plist still has no App Nap mitigation, and the issue remains paired with same-author open PRs covering the proposed launchd and scheduler surfaces.

## What This Changes

Issue requests heartbeat scheduling changes so non-interval wake reasons respect interval spacing and macOS launchd/App Nap idle gaps are mitigated.

## Best Possible Solution

Keep this issue as the canonical report while maintainers review #62308 for the remaining macOS launchd/App Nap mitigation and reconcile or close #62310 against the centralized cooldown already on main.

## Reproduction Assessment

Partially yes. Source and regression tests show the old exec-event burst path is now fixed, while the remaining launchd/App Nap gap is source-visible from the missing plist key but was not live-reproduced on macOS in this review.

## Solution Assessment

Partially. The maintainable direction is a centralized cooldown for event-driven wakes plus a separately reviewed launchd mitigation; a blanket interval gate for every non-interval reason would conflict with current wake-now contracts.
