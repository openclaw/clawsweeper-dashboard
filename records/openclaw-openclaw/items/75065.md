Full review comments:

- **[P2] Move the id schema to the current protocol package:** `src/gateway/protocol/schema/cron.ts:302`
  - body: This adds `id` under `src/gateway/protocol`, but current main's cron validators are generated from `packages/gateway-protocol`. Rebase the change onto the current package schema and validator test, otherwise the Gateway create contract will not actually accept the new field on the maintained path.
  - confidence: 0.86
- **[P2] Preserve Swift initializer compatibility:** `apps/shared/OpenClawKit/Sources/OpenClawProtocol/GatewayModels.swift:4145`
  - body: Adding `id` as a new required leading parameter changes the public `CronAddParams` initializer signature, so existing Swift callers that construct cron.add params without an id will no longer compile. Keep the change additive with `id: String? = nil` or a preserved overload in the generated Swift model copies.
  - confidence: 0.87
- **[P2] Return duplicate IDs as invalid cron.add requests:** `src/cron/service/ops.ts:326`
  - body: This duplicate-id path throws a plain `Error`, but the current cron.add handler only maps known validation errors to `INVALID_REQUEST`; the new message is not in that allowlist. A user-supplied duplicate id can therefore escape as an internal RPC failure instead of the actionable validation error promised by the PR.
  - confidence: 0.86
