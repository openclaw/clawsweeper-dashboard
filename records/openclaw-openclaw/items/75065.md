Full review comments:

- **[P1] Move cron.add id schema to the current protocol package:** `src/gateway/protocol/schema/cron.ts:298`
  - body: Current main imports cron validators from `packages/gateway-protocol`, and `src/gateway/protocol` no longer exists. Adding `id` only under the old path means the live `cron.add` schema will still reject the new field after this branch is refreshed, so the schema and validator tests need to move to the current package surface.
  - confidence: 0.9
- **[P2] Preserve Swift initializer compatibility:** `apps/shared/OpenClawKit/Sources/OpenClawProtocol/GatewayModels.swift:4142`
  - body: Adding `id` as a new required leading parameter changes the public `CronAddParams` initializer signature, so existing Swift callers that construct cron.add params without an id will stop compiling. Keep this additive with a default `nil` value or a compatibility overload in the current Swift protocol model.
  - confidence: 0.88
- **[P2] Return duplicate IDs as invalid cron.add requests:** `src/cron/service/ops.ts:326`
  - body: This duplicate-id path throws a plain `Error`, but the current Gateway cron.add handler only converts TypeError, RangeError, or recognized cron invalid-request errors into `INVALID_REQUEST`; the new already-exists message is not recognized and can escape as an internal RPC failure. Throw an accepted validation error or extend the classifier with coverage.
  - confidence: 0.87
