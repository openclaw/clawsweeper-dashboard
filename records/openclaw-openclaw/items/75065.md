## Summary

Keep open. Current main still lacks stable cron job IDs and duplicate-ID validation, and this PR is the active replacement for the closed unmerged #40218, but the current patch still has two merge blockers: it makes the generated Swift `CronAddParams` initializer source-incompatible and it omits the required user-facing changelog entry.

## What This Changes

The PR adds optional stable cron job IDs and duplicate-ID rejection across cron service creation, CLI/gateway protocol inputs, docs, generated Swift models, and targeted tests.

## Best Possible Solution

Land the additive stable cron-id feature after making generated Swift optional fields source-compatible, regenerating both Swift model copies, adding the Unreleased changelog line with contributor credit, and validating cron service, gateway protocol, CLI, and macOS client surfaces together.

## Reproduction Assessment

Yes. Current-main source inspection reproduces the missing behavior: `createJob` always uses a UUID, `add` does not guard duplicate IDs, and `CronAddParamsSchema` rejects a top-level `id`. The PR blocker is reproducible from the patch plus `scripts/protocol-gen-swift.ts`, which emits optional Swift initializer parameters without defaults.

## Solution Assessment

No. The TypeScript cron service/gateway direction is narrow and maintainable, but the current PR is not yet the best merge shape because an optional wire field should not require existing Swift callers to pass a new initializer argument, and the user-facing CLI/API feature needs a changelog entry.
