## Summary

Keep open. Current main does not implement stable cron job IDs or duplicate-ID rejection, and the PR is useful, but the latest provided diff still has merge-blocking compatibility and validation issues plus the required user-facing changelog entry is missing.

## What This Changes

The PR adds optional cron job IDs through cron service creation, gateway schema, CLI `cron add --id`, agent cron-tool recovery, docs, generated Swift protocol models, and regression tests, plus duplicate-id rejection in `CronService.add`.

## Best Possible Solution

Land the additive stable cron-id feature after preserving Swift protocol source compatibility, making duplicate-id rejection a first-class Gateway validation error, adding the Unreleased changelog entry with contributor credit, and validating the cron service, gateway protocol, CLI, generated Swift models, and docs together.

## Reproduction Assessment

Yes. Current-main source inspection reproduces the missing behavior, and the PR blockers are reproducible from the provided latest diff plus current `scripts/protocol-gen-swift.ts` and `src/gateway/server-methods/cron.ts` behavior.

## Solution Assessment

No. The TypeScript service, CLI, gateway schema, and docs direction is narrow and maintainable, but this is not the best merge shape until Swift initializer compatibility, duplicate Gateway error mapping, and the changelog requirement are fixed.
