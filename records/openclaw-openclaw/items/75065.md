Full review comments:

- **[P2] Preserve Swift initializer compatibility:** `apps/shared/OpenClawKit/Sources/OpenClawProtocol/GatewayModels.swift:4142`
  - body: Adding `id: String?` as a required initializer argument changes the public `CronAddParams(...)` call signature even though the wire field is optional. Default generated optional initializer parameters to `nil` or add a compatibility overload, then regenerate the current Swift model copies so existing callers keep compiling.
  - confidence: 0.9
- **[P2] Return duplicate IDs as invalid cron.add requests:** `src/cron/service/ops.ts:326`
  - body: The duplicate guard throws a plain `Error`, but Gateway `cron.add` only converts `TypeError` and `RangeError` from `context.cron.add` into `INVALID_REQUEST`. A duplicate ID through Gateway or CLI will surface as an internal failure rather than the promised validation response unless this case is mapped or thrown as a validation error.
  - confidence: 0.86
