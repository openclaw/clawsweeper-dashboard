## Summary

Keep open. The stable cron job id feature is useful and current main still lacks it, but this PR is not merge-ready because it makes the generated Swift `CronAddParams` initializer source-incompatible for an optional wire field and it omits the required user-facing changelog entry.

This PR adds optional custom cron job ids and duplicate-id rejection across cron creation, gateway schema/validators, CLI/tool normalization, docs, generated Swift protocol models, and targeted tests.

## Best Possible Solution

Keep the additive cron stable-id behavior, but make generated Swift optional protocol fields source-compatible, regenerate both Swift model copies, add the Unreleased changelog entry, and validate the cron service, gateway protocol, CLI, and macOS client surfaces together.

Yes. Current-main source inspection reproduces the missing behavior, and the PR blocker is visible from the generated Swift initializer plus the generator line that emits optional parameters without defaults.

## Solution Assessment
