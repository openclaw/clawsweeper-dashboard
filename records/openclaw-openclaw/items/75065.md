- **[P2] Preserve Swift initializer compatibility:** `apps/shared/OpenClawKit/Sources/OpenClawProtocol/GatewayModels.swift:4145`
  - body: Adding `id` as a new required leading parameter changes the public `CronAddParams` initializer signature, so existing Swift callers that construct cron.add params without an id will no longer compile. Keep this additive by giving the new parameter a default `nil` value or preserving an overload in both generated Swift model copies.
  - confidence: 0.87

  - body: This duplicate-id path throws a plain `Error`, but the Gateway cron.add handler only converts `TypeError` and `RangeError` from `context.cron.add` into `INVALID_REQUEST`; other errors are rethrown. A user-supplied duplicate id would therefore escape as an internal RPC failure instead of the actionable validation error promised by the PR.
  - confidence: 0.86
