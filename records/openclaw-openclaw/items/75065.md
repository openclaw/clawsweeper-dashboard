## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep selector id out of recovered update patches:** `src/agents/tools/cron-tool.ts:62`
  - body: Adding `id` to the recoverable cron object keys makes flat `action: "update"` calls recover the top-level selector id into the patch as well. The tool already reads top-level `id` as the job selector, and `cron.update` rejects immutable/unknown `patch.id`, so existing flat update calls like `{ action: "update", id, enabled: false }` would start failing instead of updating the job. Scope custom create ids to `action: "add"` job recovery or nested `job.id` so selector ids do not leak into patches.
  - confidence: 0.9
