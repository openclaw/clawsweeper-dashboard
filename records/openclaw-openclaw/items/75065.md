Full review comments:

- **[P1] Preserve Swift initializer compatibility:** `apps/shared/OpenClawKit/Sources/OpenClawProtocol/GatewayModels.swift:4143`
  - body: Adding `id` as a new required leading parameter changes the public `CronAddParams` initializer signature, so existing Swift callers that construct cron.add params without an id will no longer compile. Keep this additive by giving the new parameter a default `nil` value or preserving an overload in the generated Swift model copies after rebasing.
  - confidence: 0.87
- **[P2] Return duplicate IDs as invalid cron.add requests:** `src/cron/service/ops.ts:326`
  - body: This duplicate-id path throws a plain `Error`, but the Gateway cron.add handler only maps `TypeError`, `RangeError`, and known invalid-request messages to `INVALID_REQUEST`; other errors are rethrown. A user-supplied duplicate id can therefore escape as an internal RPC failure instead of the validation error promised by the PR.
  - confidence: 0.86
