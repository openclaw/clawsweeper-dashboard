## Summary

Keep open: current main still lacks stable cron add ids, but the PR branch has three merge-blocking issues around Swift source compatibility, Gateway error mapping, and the required changelog entry.

## What This Changes

The PR adds optional custom cron job ids and duplicate-id rejection across the cron service, Gateway protocol, CLI, agent cron-tool recovery, docs, generated Swift models, and tests.

## Best Possible Solution

Land the additive stable cron-id feature after preserving generated Swift source compatibility, making duplicate-id rejection a first-class Gateway validation error, adding the Unreleased changelog entry, and keeping duplicate-name replacement UX in a separate issue.

## Reproduction Assessment

Yes. Source inspection on current main shows UUID-only creation, no add-time id collision guard, and no Gateway `cron.add` id field; the PR blockers are reproducible from the provided head diff plus current Gateway handler and Swift generator behavior.

## Solution Assessment

No. The feature direction is narrow and maintainable, but this PR is not the best merge shape until the generated Swift initializer remains source-compatible, duplicate id errors map to `INVALID_REQUEST`, and the changelog policy is satisfied.
