## Summary

Keep open. This PR is a relevant fix candidate for the Telegram account-level `agentId` regression, but it is not merge-ready because it lacks after-fix real behavior proof and still has concrete rebase/metadata/test issues against current main.

## What This Changes

The PR adds Telegram account-level `agentId` typing and schema support, routes it as an account default when no explicit binding matches, rejects top-level `channels.telegram.agentId`, and adds schema/routing tests.

## Best Possible Solution

Land this PR or an equivalent focused patch only after maintainers accept the compatibility direction, the branch is rebased onto current Telegram tests, generated metadata/docs are aligned, and real Telegram or gateway proof is provided.

## Reproduction Assessment

Yes, at source level. Current main rejects `channels.telegram.accounts.<id>.agentId` through a strict account schema and does not consume that field in Telegram routing, matching the linked regression without needing a live gateway run.

## Solution Assessment

No, not as submitted. The compatibility direction is plausible, but the patch needs metadata/test/import fixes, docs or wording alignment, maintainer acceptance of the account-level config contract, and real behavior proof before it is the maintainable solution.
