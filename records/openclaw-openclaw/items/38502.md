## Summary

Keep open. The PR addresses a real TUI UX mismatch, but the current patch is not merge-ready because it only keys the hint off `activeChatRunId`, missing the fast submit window while `pendingOptimisticUserMessage` is true, and it lacks the required changelog entry.

## What This Changes

The PR appends a TUI Ctrl+C status hint when `activeChatRunId` is set so users know the run remains active and `/abort` stops it.

## Best Possible Solution

TUI should present one consistent cancellation policy: for this text-only PR, show accurate status for both pending optimistic submissions and bound active runs, with tests and changelog; leave abort-first semantics to #44223.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to submit a TUI message and press Ctrl+C before the first chat event arrives; `pendingOptimisticUserMessage` is true, `activeChatRunId` is still null, and this PR's hint condition does not fire while the run can continue.

## Solution Assessment

No as written. A text-only hint is maintainable if Ctrl+C remains clear/exit, but this patch must cover the pending submission window with accurate wording or earlier safe run binding, plus release-note coverage.
