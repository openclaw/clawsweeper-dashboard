## Summary

Keep open. Current main still has the old Ctrl+C clear/warn status text, and this PR is useful but incomplete because its hint only checks `activeChatRunId`, missing the pending optimistic send window described by the linked bug; it also needs the required changelog entry.

## What This Changes

This PR appends a `/abort` run-still-active hint to TUI Ctrl+C clear/warn status text when `activeChatRunId` is set.

## Best Possible Solution

Repair this text-only PR so the hint covers both pending optimistic sends and bound active runs, or choose #44223 if maintainers want Ctrl+C to abort active runs instead.

## Reproduction Assessment

Yes. Source inspection shows `sendMessage` marks a pending optimistic message before `activeChatRunId` is assigned, so a quick Ctrl+C can still miss this PR's active-run hint.

## Solution Assessment

No as written. A status-only clarification is a narrow maintainable fix under the current documented policy, but the predicate must include pending optimistic submissions and the PR needs changelog coverage.
