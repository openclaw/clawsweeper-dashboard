## Summary

Keep open: current main still lacks the active-run Ctrl+C hint, and this PR's text-only approach is close but incomplete because it misses the pending optimistic-submit window and lacks the required changelog entry.

## What This Changes

This PR appends a TUI Ctrl+C status hint in `src/tui/tui.ts` when `activeChatRunId` is set, without changing abort behavior or keybindings.

## Best Possible Solution

TUI should present one consistent cancellation policy: either keep Ctrl+C as clear/exit and include the hint for both pending optimistic submissions and bound active runs, or land the abort-first behavior from #44223 with docs and tests aligned.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to submit a TUI message and press Ctrl+C before the first chat event arrives: `pendingOptimisticUserMessage` is true, `activeChatRunId` is still null, and this PR's hint condition does not fire while the run can continue.

## Solution Assessment

No as written. A text-only hint is a narrow maintainable fix if Ctrl+C should remain clear/exit, but this diff misses the reported fast-submit timing and needs changelog coverage.
