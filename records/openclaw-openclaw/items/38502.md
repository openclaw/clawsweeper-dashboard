## Summary

Keep open; the patch still misses the quick Enter/Ctrl+C window where a submitted message is pending before a run id is bound, and the user-facing status change needs a changelog entry.

## What This Changes

This PR appends a run-still-active `/abort` hint to TUI Ctrl+C clear/warn status messages when `activeChatRunId` is set.

## Best Possible Solution

Keep one TUI cancellation policy: either repair this text-only PR so the hint covers both pending optimistic sends and bound active runs, or choose #44223 as the explicit Ctrl+C abort behavior change.

## Reproduction Assessment

Yes. Source inspection shows `sendMessage` marks `pendingOptimisticUserMessage` before `activeChatRunId` is assigned, while this PR's hint only checks `activeChatRunId`.

## Solution Assessment

No as written. A text-only hint is a narrow maintainable fix for the current Ctrl+C policy, but it needs to cover the pending optimistic state and include the required changelog entry.
