## Summary

Keep open: the PR is a useful text-only TUI clarification, but the current patch still misses the Enter-then-Ctrl+C window where a send is pending before `activeChatRunId` is assigned, and it also lacks the required user-facing changelog entry.

## What This Changes

This PR appends a run-still-active `/abort` hint to TUI Ctrl+C clear/warn status messages when `activeChatRunId` is set.

## Best Possible Solution

Keep TUI cancellation policy internally consistent: Ctrl+C status should not imply abort while a send is pending or active unless maintainers choose the broader Ctrl+C abort behavior in #44223.

## Reproduction Assessment

Yes. Source inspection shows `sendMessage` marks `pendingOptimisticUserMessage` before `activeChatRunId` is assigned, while this PR's hint checks only `activeChatRunId`.

## Solution Assessment

No as written. A text-only hint is a narrow maintainable fix if Ctrl+C remains clear/exit, but the predicate must cover pending optimistic sends and the PR needs the required changelog entry.
