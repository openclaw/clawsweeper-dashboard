## Summary

Keep open. The PR addresses a real TUI status mismatch, but the diff only keys the hint off `activeChatRunId` and still misses the fast submit window before the first chat event binds the run; it also lacks the required user-facing changelog entry.

## What This Changes

The PR appends a TUI Ctrl+C status hint when `activeChatRunId` is set, without changing abort behavior or keybindings.

## Best Possible Solution

Either complete this text-only policy by hinting for both bound active runs and pending optimistic submissions with tests/changelog, or explicitly supersede it by landing the broader Ctrl+C abort behavior from #44223.

## Reproduction Assessment

Yes. Submit a TUI message and press Ctrl+C before the first chat event; `pendingOptimisticUserMessage` is true while `activeChatRunId` remains null, so this PR's active-run hint does not fire.

## Solution Assessment

No as written. Text-only clarification is a narrow maintainable solution for current documented semantics, but this diff must cover pending optimistic sends and include changelog/test proof.
