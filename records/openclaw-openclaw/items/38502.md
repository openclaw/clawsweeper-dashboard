## Summary

Keep open. Current main still has the original Ctrl+C clear/warn/exit messaging, and this PR's text-only fix is incomplete because it keys the hint only on `activeChatRunId`, missing the fast-submit window where a run is pending but not yet bound to an active run id. The PR also needs the required changelog entry for a user-visible TUI fix.

## What This Changes

This PR appends a TUI Ctrl+C status hint in `src/tui/tui.ts` when `activeChatRunId` is set, without changing abort behavior or keybindings.

## Best Possible Solution

TUI should present one consistent cancellation policy: either keep Ctrl+C as clear/exit and include the run-active hint for both pending optimistic submissions and bound active runs, or land the abort-first behavior in #44223 with tests and docs aligned.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to submit a TUI message and press Ctrl+C before the first chat event arrives: `pendingOptimisticUserMessage` is true, `activeChatRunId` is still null, and this PR's hint condition does not fire while the run can continue.

## Solution Assessment

No as written. A text-only hint is a narrow maintainable fix if Ctrl+C should remain clear/exit, but this diff misses the reported fast-submit timing and needs changelog coverage; #44223 may be the better direction if maintainers want Ctrl+C to abort active runs.
