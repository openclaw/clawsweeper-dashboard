## Summary

Keep open. The PR is aimed at a valid TUI UX mismatch, but the current patch misses the fast submit window before the gateway event binds `activeChatRunId`, and it still lacks the required user-facing changelog entry.

## What This Changes

The PR changes `src/tui/tui.ts` to append a Ctrl+C status hint when `activeChatRunId` is set, without changing abort behavior or keybindings.

## Best Possible Solution

TUI should expose one consistent cancellation policy: either keep Ctrl+C as clear/exit and show accurate status for both pending and bound runs, or deliberately land the broader abort-first behavior with tests and docs aligned.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to submit a TUI message and press Ctrl+C before the first chat event arrives; `pendingOptimisticUserMessage` is true, `activeChatRunId` is still null, and this PR's hint condition does not fire while the run can continue.

## Solution Assessment

No as written. A text-only hint is a narrow maintainable fix if Ctrl+C remains clear/exit, but the predicate must include pending submissions and the user-visible change needs changelog coverage.
