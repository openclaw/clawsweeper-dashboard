Keep open: current main and v2026.6.10 still use transcription-only media understanding for inbound audio, and the open native audio/video PR is a candidate but remains blocked by content-contract, provider/auth, security, and proof concerns.

Request adding opt-in native audio handling so voice notes can be sent to audio-capable models as raw media instead of only STT transcripts.

## Best Possible Solution

## Solution Assessment
