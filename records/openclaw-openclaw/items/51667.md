## Label Justifications

- `P2`: This is a clear provider/media feature with real user value, but it is not an urgent regression, crash, data-loss event, or active security incident.
- `impact:session-state`: Skipping STT changes what enters the durable transcript and can affect replay, search, and session continuity.

- `impact:auth-provider`: The implementation depends on provider capability metadata, model routing, and preserving provider auth/transport controls.

## Summary

Keep open: current main and v2026.6.8 still route inbound audio through transcript-oriented media understanding, while native model audio remains a cross-cutting provider/content/security decision with an open but blocked implementation attempt.

## What This Changes

Request native raw-audio input for omni-modal models so voice notes can bypass STT transcription when explicitly enabled and supported by the active provider/model.

## Best Possible Solution

Define a shared typed native-audio content/provider contract with explicit opt-in egress, capability checks, STT fallback, transcript or hash audit semantics, docs, tests, and live channel proof before landing this feature.

## Reproduction Assessment

Yes. Source inspection on current main shows model turns support text/image content only, agent-turn attachments are image-only, and audio media understanding writes audio.transcription outputs rather than native model audio parts.

## Solution Assessment

No, the proposed config flag alone is not the best complete solution. The maintainable path needs a shared typed audio content contract, provider serializers/capability gates, fallback semantics, payload limits, and transcript/audit handling before exposing a public opt-in.
