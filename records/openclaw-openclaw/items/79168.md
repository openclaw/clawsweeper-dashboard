## Summary

Keep open: current main still lacks the requested configurable pre-model tool-output prompt-injection scanner/blocker; existing wrapping, pre-call hooks, observation hooks, and bundled-only middleware do not settle the remaining security policy/API question.

## What This Changes

Request a configurable content-based prompt-injection scanning layer for tool outputs before those results are fed back to the model.

## Best Possible Solution

Have maintainers choose the trust boundary, then implement either a documented opt-in bundled scanner/middleware with conservative defaults or a narrow trusted pre-model tool-result gate that plugins can use safely.

## Reproduction Assessment

Not applicable: this is a security hardening feature request, not a broken existing contract. Source inspection shows the requested configurable tool-output scan/block policy is absent on current main.

## Solution Assessment

Unclear: the requested direction is valid, but after_tool_call alone is observation-only and external plugins cannot register the pre-model result middleware. The maintainable solution needs a deliberate bundled-policy or trusted-API decision first.
