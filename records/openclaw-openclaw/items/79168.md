## Summary

Keep open: current main still lacks a configurable pre-model tool-output prompt-injection scanner/blocker; existing wrapping, logging, before_tool_call blocking, and bundled-only middleware are useful but leave a security contract and product/API decision unresolved.

## What This Changes

Request a configurable content-based prompt-injection scanner and block/warn/redact policy for tool output before the model sees it.

## Best Possible Solution

Have maintainers define the security contract, then implement a tested opt-in bundled scanner or a narrow versioned pre-model tool-result gate with documented policy and SDK boundaries.

## Reproduction Assessment

Not applicable: this is a security hardening feature request, not a broken existing contract. Source inspection verifies the requested configurable scanner/gate is absent on current main.

## Solution Assessment

Unclear: the direction is valid, but the exact fix needs maintainer security/product judgment. after_tool_call is observation-only for ordinary plugins, while the existing awaited tool-result middleware is intentionally bundled-only.
