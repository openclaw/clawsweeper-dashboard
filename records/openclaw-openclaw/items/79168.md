## Summary

Keep open: current main has external-content wrapping/logging, a before_agent_run block gate, and a trusted bundled tool-result middleware seam, but it still lacks the requested configurable content scanner/blocker for tool outputs and needs a maintainer-defined security contract.

## What This Changes

Request content-based prompt-injection scanning for tool outputs with configurable block, warn, or redact behavior before results reach the model.

## Best Possible Solution

Define the security contract first, then implement a trusted bundled/core pre-model tool-result scanner with upgrade-safe defaults, explicit config, and coverage across OpenClaw and Codex runtimes.

## Reproduction Assessment

Yes for source proof: current code shows detection/logging and wrapping but no configurable tool-output scanner/blocker before model submission. This is not a runnable regression reproduction because the item asks for a new security capability and policy.

## Solution Assessment

Unclear until maintainers define the security contract. The likely right direction is a trusted bundled/core result-layer gate, not an external plugin-only workaround, because that seam can rewrite high-trust tool output before model use.
