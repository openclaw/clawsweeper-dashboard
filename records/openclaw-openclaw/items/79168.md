## Summary

Keep open: current main and v2026.6.6 shipped the trusted pre-model tool-result middleware seam, but the requested built-in or officially supported prompt-injection scanner, severity policy, and block/warn/redact contract are still missing.

## What This Changes

Requested content-based prompt-injection scanning and configurable blocking or warning for tool output before it reaches the model.

## Best Possible Solution

Define the security contract first, then implement the scanner as a trusted tool-result middleware or official reference plugin using the shipped middleware seam.

## Reproduction Assessment

Yes. Source inspection shows current main can detect, log, wrap, and provide a pre-model middleware seam, but it does not include the requested scanner policy or block/warn/redact implementation.

## Solution Assessment

Partially. The shipped tool-result middleware is the right owner boundary, but the scanner action, severity threshold, default, and plugin-vs-core shape need maintainer security/product decisions before implementation.
