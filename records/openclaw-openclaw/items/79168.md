## Summary

Keep open: current main still lacks a configurable pre-model tool-output prompt-injection scanner/blocker, and the latest reporter comment confirms this needs a maintainer-defined security contract before implementation.

## What This Changes

Request a built-in or officially supported, opt-in content scanner that can warn, redact, or block suspicious tool output before it reaches the model.

## Best Possible Solution

Maintainers should define the security boundary, then implement either a trusted opt-in bundled scanner/middleware or a narrow pre-model tool-result gate with documented severity policy, config semantics, and regression coverage.

## Reproduction Assessment

Not applicable: this is a security hardening feature request, not a broken existing contract. Source inspection on current main confirms the requested configurable tool-output scan/block policy is absent.

## Solution Assessment

Unclear: the requested direction is valid, but the best implementation requires maintainer security/product judgment. after_tool_call alone is observation-only, while the current pre-model middleware seam is intentionally bundled-only.
