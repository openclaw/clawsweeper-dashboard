## Summary

Keep open: current main still lacks the requested configurable, pre-model tool-output prompt-injection scanner/blocker; the existing wrapping, logging, hooks, and bundled-only middleware are useful but leave a security policy/API decision unresolved.

## What This Changes

Request built-in or officially supported configurable scanning and blocking of tool outputs for prompt-injection patterns before model submission.

## Best Possible Solution

Have maintainers choose the security boundary, then implement either a trusted opt-in bundled scanner/middleware with documented severity policy or a narrow pre-model tool-result gate for vetted plugins.

## Solution Assessment
