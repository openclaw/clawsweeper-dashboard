## What This Changes

The issue requests configurable content-based prompt-injection scanning or blocking for tool output before it is submitted to the model.

## Best Possible Solution

Define the security contract first, then implement a tested opt-in bundled scanner or a narrow versioned pre-model tool-result gate with documented policy, config, and SDK boundaries.

## Solution Assessment

Unclear: the requested direction is valid, but the exact implementation needs maintainer security/product judgment. A bundled scanner or tightly specified pre-model gate is safer than making after_tool_call broadly modifying without a trust contract.
