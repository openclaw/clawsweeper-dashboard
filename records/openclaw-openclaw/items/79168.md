## Summary

Keep open: current main still lacks the requested configurable pre-model tool-output prompt-injection scanner/blocker. Existing wrapping, observation hooks, and bundled-only middleware help, but the remaining change is a security/product/API decision rather than conservative cleanup.

## What This Changes

Request configurable built-in or officially supported content scanning for tool outputs before they are fed back to the model.

## Best Possible Solution

Define the security boundary, then implement either an opt-in trusted bundled scanner/middleware with documented severity policy or a narrow pre-model tool-result gate that can block/redact before model replay.

## Reproduction Assessment

Not applicable: this is a security hardening feature request, not a broken existing contract. Source inspection of current main shows the requested configurable tool-output scan/block policy is absent.

## Solution Assessment

Unclear: the requested direction is valid, but the exact fix needs maintainer security/product judgment. after_tool_call alone is observation-only; trusted bundled middleware or a new narrow pre-model tool-result gate is the safer implementation point.
