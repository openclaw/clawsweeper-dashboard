The PR adds a typed `agent_start` plugin hook, threads parent run IDs through agent/subagent/Gateway paths, extends `llm_input` with prompt split fields, and updates Gateway protocol models and hook tests.

## Best Possible Solution

Land an additive, documented tracing contract that forwards parent run identity through both spawn and delivery-back Gateway calls, keeps generated protocol clients/docs/tests aligned, and includes redacted real tracing proof.

No high-confidence product bug reproduction applies because this is a new tracing API. The remaining PR defect is source-reproducible: `requesterRunId` reaches `deliverSubagentAnnouncement` but not the queued/direct Gateway agent params.

## Solution Assessment

No; the typed hook direction is plausible, but this PR is not the best mergeable form until it forwards completion delivery-back identity, documents the public API, and includes external real behavior proof.
