## Summary

Keep open: the requested core tracing seam is not on current main, and this PR still has a blocking propagation gap plus external real-behavior proof limited to tests/typecheck output.

## What This Changes

The PR adds an `agent_start` plugin hook, `spawnedByRunId` Gateway/subagent plumbing, and `userPrompt`/`prependedContext` fields on `llm_input`.

## Best Possible Solution

Fix the missing requester-run-id propagation through both direct and queued completion paths, add focused coverage for those Gateway params, and align public hook/protocol docs before landing the API seam.

## Reproduction Assessment

Yes, from source inspection of the PR. `requesterRunId` is accepted by the announce delivery API but is not included in the queued or direct Gateway `agent` calls, so completion re-invocations cannot receive `spawnedByRunId`.

## Solution Assessment

No, not yet. The additive hook direction is plausible, but the implementation does not fully carry the parent run id through the completion-delivery path that the PR says it fixes.
