## Summary

Keep open: the tracing seam is plausible, but the PR currently adds a new lifecycle hook without the existing conversation-access guard and awaits it without a default timeout, which creates security and availability blockers before merge.

## What This Changes

The PR adds an additive typed `agent_start` plugin hook, `llm_input` prompt split fields, and run-id propagation through subagent and Gateway paths for trace-tree correlation.

## Best Possible Solution

Land the core seam only after `agent_start` shares the existing conversation-access policy, awaited execution is bounded, and the public plugin/protocol documentation stays aligned.

## Reproduction Assessment

Yes for the review blockers: source inspection shows the PR adds `agent_start` without adding it to the conversation hook classification, and the awaited hook path has no default timeout. The underlying trace-tree feature itself is a new capability, so normal bug reproduction is not applicable.

## Solution Assessment

No. The additive hook direction is plausible, but this implementation should first preserve the existing conversation-access boundary and prevent a hung tracing hook from blocking model submission indefinitely.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Gate agent_start as conversation access:** `src/plugins/hook-types.ts:74`
  - body: The registry only enforces `allowConversationAccess` for hook names in `CONVERSATION_HOOK_NAMES`. Since this new lifecycle hook exposes run/session/lineage metadata, a non-bundled plugin could register it without the explicit conversation-access opt-in that gates hooks like `llm_input` and `agent_end`; add `agent_start` to that list and cover the registration-policy case.
  - confidence: 0.88
- **[P2] Bound awaited agent_start hooks:** `src/agents/pi-embedded-runner/run/attempt.ts:3024`
  - body: This awaits `runAgentStart` before the model call, but the new void hook has no default timeout; `runVoidHook` awaits an unresolved handler forever when no timeout is set. A hung tracing plugin would stall the agent before `llm_input`, so give `agent_start` a default timeout or otherwise bound this awaited path.
  - confidence: 0.84
