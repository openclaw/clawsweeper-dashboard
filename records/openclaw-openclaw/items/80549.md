## Summary

Keep open: the hook direction is plausible and paired with an open feature issue, but the patch has blocking implementation/protocol gaps and only test/typecheck proof rather than real trace behavior proof.

## What This Changes

The PR adds a typed plugin `agent_start` hook, emits it before `llm_input`, and threads parent run IDs through subagent and Gateway paths for tracing.

## Best Possible Solution

Finish the core API seam by forwarding parent run identity through direct and queued subagent announce calls, regenerating protocol artifacts and docs, and adding redacted real trace/log proof before merge.

## Reproduction Assessment

No high-confidence bug reproduction exists because this is a new hook/API feature, not a broken documented behavior. Source inspection does confirm current main lacks `agent_start` and the parent run-id Gateway path the PR proposes.

## Solution Assessment

No; the typed hook is a plausible direction, but this patch is not yet the best mergeable form because it misses requester run-id forwarding and generated protocol artifacts. The maintainable fix should complete those contract paths and document the public hook.
