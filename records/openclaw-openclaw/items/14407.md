## Summary

Keep open: current main still has partial Control UI reveal support, but the requested cross-platform sensitive-input reveal pattern is missing on Skills/API-key and native credential-entry surfaces, and persisted cleartext visibility needs product/security review.

## What This Changes

The issue asks OpenClaw to add show/hide password or secret toggles across web, desktop, iOS, and Android, including a persisted visibility preference.

## Best Possible Solution

Define reusable transient sensitive-input reveal components for approved password/token/API-key fields on Control UI, macOS, iOS, and Android, and only add persisted visibility if product/security explicitly approve it.

## Reproduction Assessment

Not applicable. This is a feature request; source inspection against origin/main shows partial web reveal controls but no cross-platform reveal pattern across Skills API-key editing and native credential-entry surfaces.

## Solution Assessment

No, not as proposed. A reveal toggle is a reasonable direction, but persisted cleartext visibility is not the safest default; reusable transient controls plus an approved field inventory are the narrower maintainable path.
