## Summary

Keep open: current main and the latest release still only partially support credential reveal controls. Web Overview has token/password toggles, but web Skills, macOS Skills, iOS Skills, and Android credential fields still have masked-only or plain shared text fields, and the linked implementation PR is closed unmerged.

## What This Changes

The issue requests cross-platform password, token, and API-key visibility toggles with clear-text reveal controls and a persisted visibility preference.

## Best Possible Solution

Implement reusable masked-by-default sensitive-input reveal controls per UI stack, keep reveal state ephemeral by default, and add focused web/native coverage for the touched credential fields.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a failing-bug reproduction, but yes for source verification: current main still has specific masked-only credential fields without reveal controls.

## Solution Assessment

Is this the best way to solve the issue? No, not with persisted clear-text visibility as the default; the safer implementation is a reusable masked-by-default reveal control with a separate maintainer decision for persistence.
