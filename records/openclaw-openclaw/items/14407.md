## Summary

Keep open. Current main still has only partial web reveal support; the web Skills API-key editor, macOS/iOS SwiftUI credential fields, and Android Compose credential fields still lack a consistent visibility toggle, and the linked implementation attempts are closed and unmerged.

## What This Changes

Requested cross-platform password, token, and API-key reveal toggles for web, desktop, iOS, and Android credential entry surfaces.

## Best Possible Solution

Define reusable sensitive-input reveal components for web, SwiftUI, and Android Compose, apply them to credential fields, and keep reveal state session-local unless maintainers approve persistence.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The feature gap is high-confidence by source inspection because current main still has credential fields without reveal controls across web Skills, SwiftUI, and Android Compose surfaces.

## Solution Assessment

No current implementation is sufficient. The maintainable direction is a shared sensitive-input reveal pattern across the affected UI stacks, not the closed unmerged web-only or Android-only attempts.
