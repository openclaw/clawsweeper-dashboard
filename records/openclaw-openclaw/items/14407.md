## Summary

Keep open. Current main still only has partial web reveal support, while the requested cross-platform password/token/API-key visibility pattern remains missing from Skills API-key editing and several native credential entry surfaces; the linked implementation attempt is closed and unmerged.

## What This Changes

Request to add reveal/hide controls for password, token, and API-key inputs consistently across web, macOS, iOS, and Android.

## Best Possible Solution

Define a reusable sensitive-input reveal pattern for web, SwiftUI, and Android Compose, apply it to the remaining credential/API-key fields, and keep reveal state session-local unless maintainers explicitly choose persistence.

## Reproduction Assessment

Yes, by source inspection for the feature gap rather than a bug reproduction. Current main still has masked or sensitive credential fields without reveal controls on web Skills and SwiftUI surfaces, and Android credential fields without the requested mask-and-toggle pattern.

## Solution Assessment

No current implementation is not enough to solve the issue. The maintainable path is a shared sensitive-input reveal pattern across the affected platforms, not the closed unmerged web-only PR or another isolated toggle.
