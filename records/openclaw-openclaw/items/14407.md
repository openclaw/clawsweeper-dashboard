## Summary

Keep open: current main still has only partial web reveal support, while the Skills API-key editor, macOS/iOS credential fields, and Android credential entry surfaces lack a product-approved cross-platform sensitive-input reveal pattern.

## What This Changes

The issue requests a password/API-key visibility toggle across web, desktop, iOS, and Android, including a persisted visibility preference.

## Best Possible Solution

Define reusable transient sensitive-input reveal components per platform, apply them only to approved credential/API-key fields, avoid persisted reveal state unless explicitly approved, and add focused UI or view tests for the touched surfaces.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection against current main shows partial web reveal controls but no cross-platform reveal pattern across Skills API-key editing and native credential-entry surfaces.

## Solution Assessment

No. A reveal toggle is a reasonable direction, but the proposed persisted cleartext visibility is not the safest default without product/security approval; a transient, reusable platform pattern is the narrower maintainable path.
