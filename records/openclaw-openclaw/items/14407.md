## Summary

Keep open. Current main and v2026.5.7 still provide only partial web reveal support; Skills API-key fields and native credential-entry surfaces still lack a consistent masked-by-default reveal toggle, and the linked implementation attempts are closed unmerged.

## What This Changes

Add consistent password and sensitive-input visibility toggles across web, desktop, iOS, and Android credential fields.

## Best Possible Solution

Define a reusable sensitive-input reveal pattern per UI stack, apply it to the remaining credential fields, and keep reveal state session-local unless maintainers explicitly approve persistence.

Not applicable as a bug reproduction; this is a feature request. The feature gap is high-confidence by source inspection because current main still has concrete Skills, macOS, iOS, and Android credential fields without a consistent reveal-control pattern.

## Solution Assessment

No, the current partial implementation is not sufficient. The maintainable path is a shared sensitive-input pattern for each UI stack rather than another isolated field toggle or the closed unmerged web-only branch.
