## Summary

Keep open. Current main still has partial web reveal support, but the requested cross-platform password/token/API-key visibility pattern is not implemented across Skills, macOS/iOS SwiftUI credential fields, or Android Compose credential fields; the linked implementation attempts are closed and unmerged.

## What This Changes

The issue requests password/token/API-key reveal toggles with consistent behavior across web, desktop, iOS, and Android surfaces.

## Best Possible Solution

Define a reusable sensitive-input reveal pattern for web, SwiftUI, and Android Compose, apply it to remaining credential fields, and keep reveal state session-local unless maintainers approve persistence.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The feature gap is high-confidence by source inspection because current main still has credential fields without reveal controls across web Skills, SwiftUI, and Android Compose surfaces.

## Solution Assessment

No current implementation is not sufficient. The maintainable direction is a shared sensitive-input reveal pattern across the affected UI stacks, not the closed unmerged web-only PR or another isolated toggle.
