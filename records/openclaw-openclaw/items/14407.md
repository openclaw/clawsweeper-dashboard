## Summary

Keep open. Current main still has only partial web reveal support, while the Skills API-key editor and several native credential-entry surfaces lack a consistent show/hide control; the related #7124 implementation attempt is closed and unmerged.

## What This Changes

The issue requests an eye-icon show/hide toggle for password and API-key fields across web, desktop, iOS, and Android, including a proposed persisted visibility preference.

## Best Possible Solution

Add reusable sensitive-input reveal components per UI stack, apply them to the remaining web and native credential fields, keep reveal state non-persistent by default, and cover touched surfaces with focused UI or view tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main gives a high-confidence feature-gap check: Overview/Login and generic config have reveal support, while Skills and native credential fields still do not.

## Solution Assessment

Yes for the reusable reveal-toggle direction, but no for the proposed persisted preference without maintainer approval. The narrow maintainable path is per-stack sensitive-input UI with ephemeral visibility state and focused coverage.
