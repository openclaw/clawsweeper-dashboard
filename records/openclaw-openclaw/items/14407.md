## Summary

Keep open: current main and v2026.6.6 only partially implement reveal controls. Web overview/login/config have reveal patterns, but web Skills, macOS/iOS SecureField, and Android ClawTextField credential entries still lack toggles, and the related implementation PRs are closed unmerged.

## What This Changes

Requests a cross-platform eye-icon reveal toggle for password, token, and API-key fields across web, desktop, iOS, and Android.

## Best Possible Solution

Implement a reusable, masked-by-default sensitive-field reveal pattern per UI stack, with ephemeral reveal state unless maintainers explicitly approve persistence, and cover the touched web/native credential fields with focused UI tests.

## Reproduction Assessment

Not applicable as a failing-bug reproduction because this requests new behavior. Source inspection gives a high-confidence current-main check that several credential fields still lack reveal controls.

## Solution Assessment

No as originally proposed if localStorage persistence is the default. The maintainable path is a reusable, per-platform sensitive-input pattern with masked defaults and an explicit maintainer decision for any persisted reveal preference.
