## Summary

Keep open: current main and v2026.6.6 still only provide partial sensitive-field reveal support. Web Overview, login, and generic config rendering have reveal controls, but the web Skills API-key editor and native macOS, iOS, and Android credential fields still lack a consistent show/hide pattern, and the related implementation PRs are closed unmerged.

## What This Changes

Request a cross-platform eye-icon show/hide control for password, token, and API-key fields, with the issue proposing a persisted visibility preference.

## Best Possible Solution

Implement reusable masked-by-default, ephemeral reveal controls per UI stack, then cover web Skills plus macOS, iOS, and Android credential fields with focused UI/view tests.

## Reproduction Assessment

Not applicable as a failing-bug reproduction because this requests new behavior. Source inspection gives a high-confidence current-main check that several credential fields still lack reveal controls.

## Solution Assessment

No, not as originally proposed if a localStorage-style persisted reveal preference is the default. The safer path is a reusable per-platform sensitive-input pattern with masked defaults and a maintainer-approved decision for any persistence.
