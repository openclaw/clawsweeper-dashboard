## Summary

Keep open: current main has useful partial reveal support in web login/overview/config surfaces, but the requested cross-platform sensitive-input reveal pattern is still missing from Skills API-key editing and native credential-entry surfaces. The linked implementation pull requests are closed and unmerged, and the proposed persisted visibility behavior needs product/security approval before implementation.

## What This Changes

Request adds consistent reveal toggles for password, token, and API-key fields across web, macOS/desktop, iOS, and Android, with a persisted visibility preference proposed.

## Best Possible Solution

Define reusable transient sensitive-input reveal components per platform, apply them only to an approved credential/API-key field inventory, avoid persisted reveal state unless explicitly approved, and add focused UI/view coverage for the touched surfaces.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection against current main shows partial web reveal controls but no cross-platform reveal pattern across Skills API-key editing and native credential-entry surfaces.

## Solution Assessment

No. A reveal toggle is a reasonable direction, but the proposed persisted cleartext visibility is not the safest default without product/security approval; a transient reusable pattern is the narrower maintainable path.
