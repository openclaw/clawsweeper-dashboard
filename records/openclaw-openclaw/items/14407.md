## Summary

Keep open: current main and v2026.6.6 still only partially cover password/secret reveal controls. Web overview/login/config have reveal patterns, but web Skills plus macOS, iOS, and Android credential fields still lack show/hide controls, and the related implementation PRs are closed unmerged.

## What This Changes

Add show/hide controls for password, token, and API-key fields across web, desktop, iOS, and Android.

## Best Possible Solution

Implement reusable masked-by-default sensitive-input reveal controls per UI stack, keep reveal state ephemeral unless maintainers explicitly approve persistence, and cover the touched credential fields with focused UI tests.

## Reproduction Assessment

Not applicable as a failing-bug reproduction because this requests new behavior. Source inspection gives a high-confidence current-main check that several credential fields still lack reveal controls.

## Solution Assessment

No, not as originally proposed if a localStorage-style persisted reveal preference is the default. The best path is a reusable per-platform sensitive-input pattern with masked defaults and an explicit maintainer decision for any persistence.
