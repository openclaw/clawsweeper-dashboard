Keep open: current main has partial web reveal controls, but the requested cross-platform sensitive-input reveal pattern is still missing from Skills API-key editing and multiple native credential-entry surfaces; the linked implementation PRs are closed and unmerged rather than shipped fixes.

## What This Changes

The issue requests password or secret reveal toggles across web, desktop, iOS, and Android credential-entry surfaces.

## Best Possible Solution

Define reusable transient sensitive-input reveal components per platform, apply them only to approved credential/API-key fields, avoid persisted reveal state unless explicitly approved, and add focused UI or view coverage for the touched surfaces.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection against current main shows partial web reveal controls but no cross-platform reveal pattern across Skills API-key editing and native credential-entry surfaces.

## Solution Assessment

No. A reveal toggle is a reasonable direction, but the proposed persisted cleartext visibility is not the safest default without product/security approval; a transient reusable pattern is the narrower maintainable path.
