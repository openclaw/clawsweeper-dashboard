## Summary

Keep open: current main still has only partial web reveal support, while Skills API-key editing and native macOS, iOS, and Android credential-entry surfaces lack a consistent cross-platform reveal pattern, and the proposed persisted reveal preference needs product/security approval.

## What This Changes

The issue requests cross-platform password, token, and API-key visibility toggles with eye-icon reveal behavior and a persisted visibility preference.

## Best Possible Solution

Define reusable transient sensitive-input reveal components per platform, apply them only to approved password/token/API-key fields, avoid persisted reveal state unless explicitly approved, and add focused UI or view coverage.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection against current main shows partial web reveal controls but no cross-platform reveal pattern across Skills API-key editing and native credential-entry surfaces.

## Solution Assessment

No. A reveal toggle is a reasonable direction, but the proposed persisted cleartext visibility preference is not the safest default; a reusable transient reveal pattern with an approved field inventory is the narrower maintainable path.
