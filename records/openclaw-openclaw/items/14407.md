## Summary

Keep open: current main still has partial web reveal support, but the requested cross-platform sensitive-input reveal pattern is missing from Skills/API-key editing and native credential-entry surfaces, and the persisted visibility idea needs product/security approval. The related implementation PRs are closed and unmerged, and the broader sensitive-data umbrella does not fully replace this platform-specific request.

## What This Changes

The issue requests a platform-wide sensitive-input reveal toggle for password, token, and API-key fields across web, desktop, iOS, and Android.

## Best Possible Solution

Define reusable transient sensitive-input reveal components per platform, apply them only to approved credential/API-key fields, avoid persisted reveal state unless explicitly approved, and add focused UI or view coverage.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection against current main shows partial web reveal controls but no cross-platform reveal pattern across Skills API-key editing and native credential-entry surfaces.

## Solution Assessment

No. A reveal toggle is a reasonable direction, but the proposed persisted cleartext visibility is not the safest default without product/security approval; a transient reusable pattern is the narrower maintainable path.
