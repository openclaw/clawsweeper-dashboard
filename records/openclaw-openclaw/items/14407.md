## Summary

Keep open: current main and v2026.6.5 still provide only partial web reveal controls, while the Skills API-key editor and multiple macOS/iOS credential fields remain masked-only; the requested persisted visibility behavior also needs an explicit product/security decision.

## What This Changes

Add consistent show/hide controls for password, token, and API-key inputs across OpenClaw web and companion applications.

## Best Possible Solution

Standardize a transient per-field reveal control on the inventoried web, macOS, and iOS credential-entry surfaces, default every field to masked when its view/session opens, and do not persist visibility unless maintainers separately approve that security tradeoff.

## Solution Assessment

No: persisting the reveal preference is not the safest or narrowest solution; transient per-field state based on the existing web pattern limits exposure and avoids a new settings surface.
