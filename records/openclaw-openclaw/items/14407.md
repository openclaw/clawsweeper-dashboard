## Summary

Keep open. Current main still has partial web reveal support, but the requested cross-platform password visibility behavior is missing from the Skills API-key editor and several macOS, iOS, and Android credential surfaces; the related PR #7124 is closed and unmerged.

## What This Changes

The issue requests show/hide password controls across web, desktop, iOS, and Android credential entry surfaces.

## Best Possible Solution

Define a reusable sensitive-input reveal pattern for web, SwiftUI, and Android Compose, apply it to the remaining credential fields, avoid persisted reveal state unless maintainers explicitly choose it, and add focused UI/view coverage for the touched surfaces.

## Reproduction Assessment

Yes by source inspection, though this is a feature gap rather than a bug reproduction: current main still has masked web/SwiftUI fields without reveal controls and Android fields without the requested masked/plain toggle pattern.

## Solution Assessment

No current implementation is not enough to close this; the best path is a reusable cross-platform sensitive-input control rather than relying on the closed, unmerged PR or adding another one-off web toggle.
