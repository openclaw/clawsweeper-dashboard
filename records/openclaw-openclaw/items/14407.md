## What This Changes

Requests a consistent password, token, and API-key visibility toggle across web, desktop, iOS, and Android credential fields.

## Best Possible Solution

Define reusable sensitive-input reveal components per UI stack, apply them to the remaining credential fields, and keep reveal state session-local unless maintainers explicitly approve persistence.

Not applicable as a bug reproduction; this is a feature request. The remaining gap is high-confidence by source inspection because current main and v2026.5.12 still contain concrete Skills, macOS, iOS, and Android credential fields without reveal controls.

## Solution Assessment

No, closing this as implemented or reviving the stale web-only PR is not the best path. The maintainable solution is a shared sensitive-input reveal pattern per UI stack with an explicit decision on whether reveal state may persist.
