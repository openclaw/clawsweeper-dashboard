## Summary

Keep open: current main and the latest release still only partially implement credential reveal controls. Web Overview and login have toggles, but the Skills API-key editor and native macOS, iOS, and Android credential fields still lack a reusable reveal pattern, and the linked implementation PR is closed unmerged.

## What This Changes

The issue requests a cross-platform show/hide password or secret toggle for masked credential fields.

## Best Possible Solution

Implement reusable masked-by-default sensitive-input reveal controls per UI stack, keep reveal state ephemeral by default, and add focused web/native coverage for the touched credential fields.

Yes for source verification: current main still has specific credential fields without reveal controls, including the Skills API-key editor and native credential entries. This is not a failing-bug reproduction because the issue asks for a new cross-platform feature.

## Solution Assessment

No for the proposed persisted localStorage preference as the default; the safer solution is a reusable masked-by-default reveal control with a separate maintainer decision on persistence.
