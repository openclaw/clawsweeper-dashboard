## Summary

Keep open: current main and v2026.6.10 still only partially implement sensitive-field reveal controls, with concrete gaps in web Skills, macOS, iOS, and Android credential entry surfaces; this remains a product/security decision rather than a strict bug fix.

## What This Changes

The issue requests show/hide password toggles for password, token, and API-key fields across web, desktop, iOS, and Android, including a persisted visibility preference.

## Best Possible Solution

Implement reusable masked-by-default sensitive-input reveal controls per UI stack, reuse the existing Control UI reveal pattern where possible, and treat persisted visibility as a separate security/product decision.

## Reproduction Assessment

Yes, for source-level verification: current main still has credential fields without reveal controls across web Skills, macOS/iOS SwiftUI, and Android Compose. This is not a broken-contract bug repro; it proves the requested feature remains incomplete.

## Solution Assessment

No, not as a blanket persisted clear-text preference. The safer maintainable path is reusable masked-by-default reveal controls, with any persisted visibility behavior handled only after maintainer security/product approval.
