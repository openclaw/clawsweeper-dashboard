## Summary

Keep open. Current main still has only partial web reveal support, while the requested cross-platform password/API-key visibility behavior is missing from Skills API-key editing and several native credential entry surfaces; the linked implementation attempt is closed and unmerged.

## What This Changes

The issue requests password visibility toggles across web, macOS/Windows desktop, iOS, and Android credential fields.

## Best Possible Solution

Define a reusable sensitive-input reveal pattern for web, SwiftUI, and Android Compose, apply it to the remaining credential/API-key fields, and keep reveal state session-local unless maintainers explicitly choose persistence.

## Reproduction Assessment

Yes, by source inspection, but this is a feature-gap verification rather than a bug reproduction. Current main still has masked or secure credential fields without reveal controls on web Skills and native SwiftUI surfaces, plus Android fields without the requested toggle pattern.

## Solution Assessment

No current implementation is not enough to solve the issue. The maintainable path is a shared sensitive-input reveal pattern across platforms, not the closed unmerged web-only PR or another isolated toggle.
