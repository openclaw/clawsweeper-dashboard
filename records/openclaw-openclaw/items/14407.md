## Summary

Keep open: current main still only partially implements password/secret reveal controls. Web Overview and Login have reveal toggles, but web Skills, macOS/iOS Skills, and Android credential entry surfaces still lack equivalent masked-by-default reveal controls, and the linked implementation PR is closed unmerged.

## What This Changes

The issue requests password/secret visibility toggles for sensitive inputs across web, desktop, iOS, and Android surfaces.

## Best Possible Solution

Implement reusable masked-by-default sensitive-input reveal controls per UI stack, keep reveal state ephemeral by default, and add focused coverage for the touched credential fields after maintainer scope approval.

## Reproduction Assessment

Yes for source verification: current main still has specific credential fields without reveal controls. This is a feature request rather than a failing-bug reproduction.

## Solution Assessment

No, not with persisted clear-text visibility as the default. The safer path is reusable masked-by-default reveal controls with a separate maintainer decision for any persistence behavior.
