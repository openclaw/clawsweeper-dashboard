## Summary

Keep open: current main only partially covers credential reveal in the web UI, while Skills API-key editing and native macOS/iOS/Android credential fields still lack an approved cross-platform reveal pattern; the requested persisted visibility behavior also needs product/security approval.

## What This Changes

Request to add password/secret visibility toggles across web, desktop, iOS, and Android, including a persisted visibility preference.

## Best Possible Solution

Define a reusable transient sensitive-input reveal pattern per platform, apply it to approved credential/API-key surfaces, and only persist visibility state if product/security explicitly accepts that tradeoff.

## Reproduction Assessment

Not applicable. This is a feature request rather than a broken existing contract, and source inspection shows current main still lacks reveal controls on several requested credential-entry surfaces.

## Solution Assessment

No. A visibility toggle is a reasonable direction, but persisting cleartext visibility is not clearly the safest default; a transient, reusable, product-approved pattern is the better implementation shape.
