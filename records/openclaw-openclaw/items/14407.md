## Summary

Keep open: current main has reveal controls on some web gateway-auth surfaces, but Skills API keys and multiple macOS, iOS, and Android credential fields still lack a consistent reveal pattern. The two related implementation PRs are closed and unmerged, and the proposed persisted visibility behavior needs explicit product/security approval.

## What This Changes

Add consistent controls for revealing masked passwords, tokens, and API keys across OpenClaw’s web, macOS, iOS, and Android interfaces.

## Best Possible Solution

Approve a sensitive-field inventory, then add reusable masked-by-default, ephemeral reveal controls in the web, SwiftUI, and Compose stacks with accessible labels and focused view or UI proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests new behavior; source inspection nevertheless provides high-confidence evidence that the cross-platform capability remains incomplete.

## Solution Assessment

No as proposed: reusable reveal controls are appropriate, but persisted visibility is not the safest default and should be replaced by ephemeral, masked-by-default state unless maintainers explicitly approve persistence.
