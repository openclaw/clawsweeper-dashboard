## Summary

Keep open. Current main has partial web reveal support, but the Skills API-key editor and multiple native macOS, iOS, and Android credential-entry surfaces still lack a consistent show/hide control; the related #7124 implementation attempt is closed and unmerged in the provided context.

## What This Changes

Request to add a cross-platform show/hide control for password, token, and API-key inputs, with an eye icon and a proposed persisted visibility preference.

## Best Possible Solution

Add reusable sensitive-input reveal components per UI stack, apply them to the remaining web and native credential fields, keep visibility ephemeral by default unless maintainers approve persistence, and cover touched surfaces with focused UI/view tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main gives a high-confidence feature-gap check: Overview/Login and generic config have reveal support, while Skills and native credential fields still do not.

## Solution Assessment

Yes for a reusable per-stack reveal-toggle pattern, but no for making localStorage persistence the default without maintainer approval. The narrow maintainable path is ephemeral sensitive-input reveal UI plus focused coverage.
