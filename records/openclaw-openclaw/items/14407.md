## Summary

Keep open. Current main still has only partial web reveal support, while the requested cross-platform password/API-key visibility behavior is missing from the Skills API-key editor and native credential entry surfaces; the linked implementation PR is closed and unmerged.

## What This Changes

Add a cross-platform password/API-key visibility toggle so users can switch sensitive fields between masked and clear text.

## Best Possible Solution

Define a reusable sensitive-input reveal pattern for web, SwiftUI, and Android Compose, apply it to the remaining credential/API-key fields, and keep reveal state session-local unless maintainers explicitly choose persistence.

## Reproduction Assessment

Yes, by source inspection, but this is a feature-gap verification rather than a bug reproduction. Current main still has masked or secure credential fields without reveal controls on the web Skills editor, macOS/iOS SwiftUI fields, and Android Compose auth fields.

## Solution Assessment

No current implementation is not enough to solve the issue. The maintainable path is a shared sensitive-input reveal pattern across platforms, not the closed unmerged web-only PR or another isolated toggle.
