## Summary

Keep this issue open. Current main still only has partial password/token reveal support in the web Control UI; the Skills API-key editor and multiple macOS, iOS, and Android credential-entry surfaces still lack a show/hide control, and the related PR #7124 is closed and unmerged in the provided GitHub context.

## What This Changes

The issue requests a consistent show/hide password toggle for masked credential and password fields across web, desktop, iOS, and Android surfaces.

## Best Possible Solution

Add a reusable sensitive-input reveal pattern and apply it to the remaining web Skills/API-key field plus native macOS, iOS, and Android credential fields, keeping reveal state local and non-persistent by default and adding focused UI/view coverage for the touched surfaces.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main gives a high-confidence feature-gap check: existing web reveal controls are present in Overview/Login, while Skills and native credential fields still lack equivalent controls.

## Solution Assessment

Yes for the general reveal-toggle direction, but not for the proposed persistent preference. The maintainable path is a reusable, per-surface sensitive-input component/wrapper with ephemeral visibility state and no credential storage/auth-flow changes.
