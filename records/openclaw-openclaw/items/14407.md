## What This Changes

## Best Possible Solution

Define reusable transient sensitive-input reveal components for approved credential fields on Control UI, macOS, iOS, and Android, and add persisted visibility only if product/security explicitly approve it.

## Reproduction Assessment

## Solution Assessment

No, not exactly as proposed. Reveal toggles are reasonable, but persisted cleartext visibility is not the safest default; a reusable transient control plus an approved field inventory is the narrower maintainable direction.
