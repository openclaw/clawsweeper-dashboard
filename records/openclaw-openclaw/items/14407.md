## Summary

Keep open: current main and the latest shipped tag still only partially cover credential reveal in web gateway/config surfaces, while web Skills API-key editing and native macOS/iOS/Android credential entry still lack an approved cross-platform reveal pattern; the requested persisted visibility behavior also needs product/security approval.

## What This Changes

The issue requests password or secret visibility toggles across web, desktop, iOS, and Android credential fields.

## Best Possible Solution

Define a reusable transient sensitive-input reveal pattern per platform, apply it only to approved credential/API-key surfaces, and persist visibility state only if product/security explicitly accepts that tradeoff.

## Reproduction Assessment

Not applicable. This is a feature request rather than a broken existing contract, and source inspection shows current main still lacks reveal controls on several requested credential-entry surfaces.

## Solution Assessment

No. A visibility toggle is a reasonable direction, but the proposed persisted cleartext visibility is not clearly the safest default; a transient, reusable, product-approved pattern is the better implementation shape.
