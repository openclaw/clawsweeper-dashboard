## Repair Work Prompt

Fix the Memory Wiki imported-source transient path-mismatch race reported in https://github.com/openclaw/openclaw/issues/92134 without adding config or weakening fs-safe symlink/hardlink/path-alias protections. Current main has wiki_status call syncMemoryWikiImportedSources before rendering status, bridge sync awaits each writeBridgeSourcePage directly, and writeImportedSourcePage wraps FsSafeError path-mismatch as a fatal imported-source-page error. Add a small caller-side retry for transient FsSafeError code path-mismatch around the existing source-page read/write path, keeping symlink/path-alias/hardlink/not-file failures fail-closed. Add focused regression coverage that simulates a first path-mismatch followed by success and proves wiki_status/source sync does not abort after the transient retry. Do not add workspace/agent scoping config, broad coalescing, RSS backpressure, docs, or changelog changes in this repair; keep https://github.com/openclaw/openclaw/issues/91154 as the broader follow-up for import performance/coalescing decisions.
