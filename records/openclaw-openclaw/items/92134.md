## What This Changes

The issue reports that `wiki_status` can intermittently abort when memory-wiki bridge source pages are atomically re-exported during the tool's imported-source sync.

## Best Possible Solution

Land or replace a focused memory-wiki fix that treats transient fs-safe path swaps during imported-source sync as retryable or per-artifact recoverable while preserving fail-closed behavior for real symlink, hardlink, and outside-root safety violations.

## Reproduction Assessment

Yes for a source-level reproduction path: current `wiki_status` awaits imported-source sync, which serially calls the bridge source writer, and fs-safe can throw `path-mismatch` when a file is swapped during open/verification. I did not run the live concurrent rename timing race in this read-only pass.
