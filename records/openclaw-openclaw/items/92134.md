## Summary

Keep open: current `main` and latest tagged release still have the one-shot memory-wiki source-page write path that can rethrow fs-safe path errors, and the linked fix at https://github.com/openclaw/openclaw/pull/92375 is open but not merged.

## What This Changes

The issue asks OpenClaw to treat transient memory-wiki source-page path-alias/path-mismatch races as retryable or per-page failures so `wiki_status` does not abort during concurrent bridge re-export.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92375 or an equivalent narrow fix that retries transient path-alias writes while preserving hard failures for symlink, hardlink, and non-transient fs-safe validation errors.

## Reproduction Assessment

No live race was reproduced during this read-only review. The source path is clear: `wiki_status` runs bridge source sync, each bridge artifact calls `writeImportedSourcePage`, and a single fs-safe write error currently rejects the whole status call.

## Solution Assessment

Yes, bounded retry in the shared imported-source writer is the narrowest maintainable immediate fix. Per-page skip/warning behavior is a reasonable follow-up if maintainers want `wiki_status` to degrade more gracefully after retries are exhausted.
