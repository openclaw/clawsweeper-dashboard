## What This Changes

The PR changes cron doctor legacy notify migration to delete inert top-level `notify: true` when `cron.webhook` is unset, adds regression tests, and updates cron doctor docs.

## Best Possible Solution

Land the cleanup after maintainers accept that top-level cron `notify` is disposable legacy metadata when delivery is already represented by `delivery`, while preserving configured-invalid webhook warnings.

## Reproduction Assessment

Yes. Source inspection on current main shows doctor counts `notify === true`, then the migration helper warns and preserves that marker when `cron.webhook` is unset, so a later doctor pass can re-detect the same row.

## Solution Assessment

Yes, pending maintainer acceptance of the compatibility tradeoff. Doctor is the right owner for legacy config repair, and this patch removes inert metadata while preserving delivery semantics and configured-invalid webhook warnings.
