## What This Changes

The PR updates cron doctor legacy notify migration to remove inert top-level `notify: true` when `cron.webhook` is unset, adds regression coverage, and aligns cron doctor docs.

## Best Possible Solution

Land the cleanup after maintainers accept that top-level cron `notify` is disposable legacy metadata when delivery is already represented by `delivery`, while preserving the invalid configured-webhook warning.

## Reproduction Assessment

Yes. Source inspection shows current main counts `notify === true` for the doctor preview and then preserves that marker when `cron.webhook` is unset, so a later doctor pass can re-detect the same row.

## Solution Assessment

Yes. Doctor is the right owner for legacy config repair, and this patch removes inert metadata while preserving delivery semantics and keeping configured-but-invalid webhook warnings actionable.
