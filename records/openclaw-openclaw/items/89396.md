## What This Changes

The PR changes cron doctor legacy notify migration to delete top-level `notify` when `cron.webhook` is unset, adds focused regression tests, and updates cron docs.

## Best Possible Solution

Land the doctor cleanup with its tests and docs once maintainers accept deleting the inert legacy marker as the canonical migration behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows doctor counts `notify === true`, then the migration helper warns and preserves that marker when `cron.webhook` is unset, so a later doctor pass can re-detect the same row.

## Solution Assessment

Yes, with a compatibility decision. Doctor migration is the right owner, the patch preserves existing delivery and configured-invalid-webhook warnings, and the only remaining question is whether maintainers accept deleting the inert persisted marker.
