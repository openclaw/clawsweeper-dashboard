## Summary

## What This Changes

The PR changes cron doctor’s legacy notify migration to delete top-level `notify: true` when no usable `cron.webhook` target exists, preserving existing delivery, and adds regression tests for `delivery.mode: none` and announce delivery.

## Best Possible Solution

Land the doctor fix after updating the cron doctor docs to match the no-webhook cleanup semantics and the preserved invalid-config warning path.

## Reproduction Assessment

Yes. Current main’s doctor path can be traced from `notifyCount` in `maybeRepairLegacyCronStore` to `migrateLegacyNotifyFallback` warning without setting `changed` when `cron.webhook` is absent, so the same raw config job is detected again on the next run.

## Solution Assessment

Yes for the reported loop: doctor owns legacy config repair, and removing inert `notify` while preserving current delivery is narrower than changing runtime delivery. The remaining issue is docs alignment for the new no-webhook cleanup behavior.
