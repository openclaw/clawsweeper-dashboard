## What This Changes

The PR changes cron doctor legacy notify migration to delete top-level `notify: true` when `cron.webhook` is unset, preserves existing delivery, and adds regression tests plus docs updates.

## Best Possible Solution

Land the cleanup after maintainers accept that top-level cron `notify` is disposable legacy metadata when delivery is already represented by `delivery`, while preserving the invalid configured-webhook warning.

## Reproduction Assessment

Yes. Source inspection on current main shows doctor counts `notify === true` for the preview, then the migration helper warns and preserves the marker when `cron.webhook` is unset, so a later run can report the same row again.

## Solution Assessment

Yes, with a maintainer compatibility decision. Doctor is the right owner for legacy cron store repair, and deleting inert metadata while preserving delivery semantics is the narrow fix if `notify` no longer has a manual-migration contract.
