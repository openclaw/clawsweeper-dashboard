## Summary

Keep open: the PR targets a real silent-reply leak, but the current patch has a blocking false-positive risk, lacks contributor-supplied real behavior proof, and intentionally changes the prior broad trailing-token policy without maintainer selection.

## What This Changes

The PR changes `normalizeReplyPayload` and tests so any payload shortened by trailing `NO_REPLY` stripping is treated as silent instead of returning the remaining text.

## Best Possible Solution

Land one canonical shared silent-reply classifier that suppresses reasoning-prefaced silent outputs without dropping legitimate non-trailing mentions, with focused regression tests and real dispatcher or channel proof.

## Reproduction Assessment

Yes, by source inspection. Current main strips the trailing token and returns non-empty preamble text, and current tests still assert delivery for trailing-token content.

## Solution Assessment

No. Suppressing reasoning-prefaced silent replies is valid, but this patch uses whitespace-sensitive length inference and broad trailing-token suppression; a safer fix should use an explicit trailing-token predicate or trimmed comparator after maintainers choose the policy.
