## What This Changes

The branch changes `normalizeReplyPayload` and tests so any payload shortened by trailing `NO_REPLY` stripping is treated as silent, including emoji or substantive preambles.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/80930 as the canonical fix while preserving the documented behavior for legitimate substantive replies.

## Reproduction Assessment

Yes. Source inspection shows the current normalizer can return a non-empty preamble after stripping trailing `NO_REPLY`, and current tests still assert that strip-and-deliver behavior.

## Solution Assessment

No. The PR addresses the leak class, but the raw-length heuristic can suppress non-trailing token mentions with trailing whitespace and the broader policy is superseded by the narrower open fix.
