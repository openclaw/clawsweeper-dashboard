## Summary

Keep open: current main still has a source-reproducible reasoning/preamble leak path, but this PR is not merge-ready because its length heuristic can suppress legitimate non-trailing NO_REPLY mentions and the contributor has only supplied unit-test proof.

## What This Changes

The PR changes `normalizeReplyPayload` and its tests to suppress the whole reply whenever trailing `NO_REPLY` stripping shortens the text.

## Best Possible Solution

Land one shared silent-reply classifier that suppresses reasoning-only final `NO_REPLY` outputs without dropping substantive non-trailing mentions, with focused tests and real channel or dispatcher proof.

## Reproduction Assessment

Yes, source-reproducible. Current main strips a trailing silent token and returns non-empty preamble text, and current tests still assert strip-and-deliver behavior for trailing-token content.

## Solution Assessment

No. Suppressing reasoning-prefaced silent replies is a valid direction, but this patch should use an explicit trailing-token predicate or compare with pre-strip trimmed text, and the broader any-trailing-token policy needs maintainer selection.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Detect trailing NO_REPLY without trim-length inference:** `src/auto-reply/reply/normalize-reply.ts:73-74`
