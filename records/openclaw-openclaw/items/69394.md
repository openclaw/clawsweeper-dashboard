## Summary

Keep open: the bug is valid on current main, but this PR still lacks contributor-supplied real behavior proof and the patch introduces a false-positive suppression path for non-trailing `NO_REPLY` mentions with trailing whitespace. A narrower competing implementation with supplied proof exists at https://github.com/openclaw/openclaw/pull/80930, but maintainers still need to choose the canonical suppression policy.

## What This Changes

The PR changes auto-reply normalization so payloads shortened by trailing `NO_REPLY` stripping are treated as silent and updates the corresponding unit tests.

## Best Possible Solution

Land one canonical shared silent-reply classifier that suppresses reasoning-prefaced silent outputs without dropping legitimate non-trailing mentions, with focused regression tests and real dispatcher or channel proof.

## Reproduction Assessment

Yes. Source inspection on current main shows a non-empty preamble before `NO_REPLY` normalizes to deliverable text, and current tests still assert delivery for trailing-token content; I did not run tests because this review is read-only.

## Solution Assessment

No, not as written. Suppressing reasoning-prefaced silent replies is valid, but this patch uses whitespace-sensitive length inference and broad trailing-token suppression; a safer fix should use an explicit trailing-token predicate or the narrower proven path in https://github.com/openclaw/openclaw/pull/80930 after maintainer policy selection.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Use explicit trailing NO_REPLY detection:** `src/auto-reply/reply/normalize-reply.ts:73-74`
  - body: `stripSilentToken()` trims even when it removes no trailing token. With this change, text like `NO_REPLY -- explanation\n` or any message containing `NO_REPLY` plus trailing whitespace makes `stripped.length < text.length` true and gets incorrectly suppressed as silent; compare against the pre-strip trimmed value or use an explicit trailing-token predicate.
  - confidence: 0.91
