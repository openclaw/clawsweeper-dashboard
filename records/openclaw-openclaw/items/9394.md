## Summary

Keep open. Current main still lacks rich outbound iMessage formatting and the named newer effects, while the related draft private-API PR only partially overlaps and explicitly leaves rich text unresolved.

## What This Changes

The issue requests rich text spans, text-size variants, underline/strikethrough, and Shake/Nod/Explode/Ripple/Bloom/Jitter-style effects for outbound iMessage delivery.

## Best Possible Solution

Define a typed outbound formatting/effects capability model for iMessage transports, implement only upstream-backed subsets per transport, preserve plain-text fallback behavior, and document unsupported legacy imsg limits.
