## Summary

Keep open: current main partially implements iMessage rich styling through bold/italic/underline/strikethrough typed runs and `sendWithEffect`, but text-size variants and the requested Shake/Nod/Explode/Ripple/Bloom/Jitter effects are still absent and need product/API direction.

## What This Changes

The issue requests richer outbound iMessage formatting and effects, including style spans, text-size variations, and named animations.

## Best Possible Solution

Define a typed, upstream-backed iMessage outbound formatting/effects capability model, keep plain-text fallback for unsupported transports, and document exactly which styles/effects are supported.

## Reproduction Assessment

Not applicable. This is an enhancement request, and source inspection on current main shows the supported subset plus the missing text-size and requested named effect surfaces.

## Solution Assessment

No complete solution is on main yet. The partial implementation is useful, but finishing this should follow an upstream-backed capability model rather than ad hoc fields or aliases.
