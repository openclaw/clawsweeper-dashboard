## Summary

Keep open. Current main partially addresses the request with markdown-derived bold/italic/underline/strikethrough runs and a private-API `sendWithEffect` action, but text-size support and the requested Shake/Nod/Explode/Ripple/Bloom/Jitter effects are still absent and need maintainer API/product direction.

## What This Changes

Request iMessage outbound support for rich text styles, text-size variants, and named iMessage text or screen effects.

## Best Possible Solution

Define an upstream-backed iMessage outbound formatting/effects capability model, keep the supported subset, and add text-size or newer text-effect support only where `imsg` exposes stable typed fields with graceful fallback behavior.

## Reproduction Assessment

Not applicable. This is an enhancement request, and source inspection on current main verifies that basic style runs and classic effects exist while text-size and the requested named effects remain absent.

## Solution Assessment

No. The current partial implementation is useful, but completing the request needs maintainer product/API direction around upstream `imsg` capabilities and fallback semantics.
