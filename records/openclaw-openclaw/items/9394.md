## Summary

Keep open: current main and v2026.5.12 partially satisfy the request with basic markdown-derived styles and a fixed sendWithEffect set, but text-size attributes and the requested Shake/Nod/Explode/Ripple/Bloom/Jitter effects are still absent and need maintainer product/API direction.

## What This Changes

Request iMessage outbound support for rich text formatting, text-size variations, and additional Messages effects beyond plain text delivery.

## Best Possible Solution

Define an upstream-backed typed iMessage formatting/effects capability model, preserve plain-text fallback behavior, and add text-size or newer effects only when imsg/private-API support is stable.

## Reproduction Assessment

## Solution Assessment
