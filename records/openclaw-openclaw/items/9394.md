## Summary

Keep open. Current main still lacks a rich outbound iMessage formatting model: legacy iMessage sends plain text through imsg, while BlueBubbles strips Markdown for normal sends and only exposes a narrower effectId path; related draft PR #78317 explicitly leaves rich text formatting unresolved.

## What This Changes

Request outbound iMessage support for rich text spans, text-size variations, and named Messages effects.

## Best Possible Solution

Define a typed outbound formatting/effects capability model for iMessage transports, implement only upstream-backed subsets per transport, preserve plain-text fallback behavior, and document unsupported legacy imsg limits.
