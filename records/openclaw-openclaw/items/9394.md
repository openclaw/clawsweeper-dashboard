## Summary

Keep open. Current main still lacks the requested rich iMessage formatting model: legacy iMessage sends plain text through `imsg`, while BlueBubbles strips Markdown on normal sends and only exposes a narrower `effectId` action for existing Apple effects.

## What This Changes

Request support for rich iMessage outbound formatting, text-size attributes, and named text/screen effects in the iMessage/BlueBubbles send surfaces.

## Best Possible Solution

Define a typed outbound capability model for iMessage/BlueBubbles styles and effects, implement only upstream-backed subsets per transport, preserve plain-text fallback, and document unsupported legacy `imsg` limits.
