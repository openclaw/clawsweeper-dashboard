## Summary

Keep open. Current main partially covers the request with basic markdown-derived iMessage style runs and a classic `sendWithEffect` action, but it still lacks text-size support and the requested Shake/Nod/Explode/Ripple/Bloom/Jitter effects.

## What This Changes

The issue requests rich iMessage formatting and effects, including bold, italics, underline, strikethrough, text size, and named message animations.

## Best Possible Solution

Define an upstream-backed iMessage outbound formatting/effects capability model, keep the supported subset, and add text-size or newer text-effect support only where imsg exposes stable typed fields with graceful fallback behavior.

## Reproduction Assessment

Not applicable. This is an enhancement request, and source inspection on current main verifies that basic style runs and classic effects exist while text-size and the named text effects remain absent.

## Solution Assessment

No. The current implementation is a useful partial solution, but completing the request needs maintainer product/API direction around upstream imsg capabilities and fallback semantics.
