## Summary

Keep open. Current main partially implements the request with markdown-derived bold/italic/underline/strikethrough runs and a private-API `sendWithEffect` action, but the requested text-size support and Shake/Nod/Explode/Ripple/Bloom/Jitter effects are still absent and need maintainer API/product direction.

## What This Changes

The issue requests rich iMessage formatting, text size, and newer message effects for the iMessage channel.

## Best Possible Solution

Define an upstream-backed iMessage outbound formatting/effects capability model, keep the current supported subset, and add text-size or newer effects only where `imsg` exposes stable typed fields with graceful fallback behavior.

## Reproduction Assessment

Not applicable. This is an enhancement request, and source inspection on current main verifies that basic style runs and classic effects exist while text-size and the requested named effects remain absent.

## Solution Assessment

No. The current partial implementation is useful, but completing the issue needs a maintainer-owned API/product decision around upstream `imsg` capabilities and fallback semantics.
