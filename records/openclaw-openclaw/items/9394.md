## Summary

Keep open: current main and the latest shipped tag only partially satisfy the request with bold/italic/underline/strikethrough formatting runs and a fixed `sendWithEffect` subset; text-size support and the requested Shake/Nod/Explode/Ripple/Bloom/Jitter effects are still absent and need maintainer product/API direction.

## What This Changes

Request iMessage outbound support for rich text formatting, text-size variations, and named expressive effects beyond plain text.

## Best Possible Solution

Define an upstream-backed typed iMessage formatting/effects capability model, preserve plain-text fallback, and add text size or newer effect names only when `imsg` exposes stable fields for them.

## Reproduction Assessment

Not applicable. This is an enhancement request, and source inspection on current main verifies the partial subset while showing the requested text-size and named effect support are still missing.

## Solution Assessment

No. The current partial implementation is useful, but completing this through ad hoc aliases or guessed rich fields would be less maintainable than an upstream-backed capability contract.
