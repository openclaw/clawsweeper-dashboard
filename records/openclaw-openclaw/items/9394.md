## Summary

Keep open: current main and v2026.5.12 only partially satisfy the request with markdown-derived bold/italic/underline/strikethrough plus a fixed classic effect set; text-size attributes and the requested Shake/Nod/Explode/Ripple/Bloom/Jitter effects remain absent and still need product/API direction.

## What This Changes

The issue asks the iMessage plugin to support rich text styles, text-size variants, and newer iMessage animation effects.

## Best Possible Solution

Define an upstream-backed typed iMessage formatting/effects capability model, preserve plain-text fallback behavior, and add text-size or newer effects only when the imsg/private-API contract exposes stable fields for them.

## Reproduction Assessment

Not applicable. This is an enhancement request, and source/docs inspection on current main plus v2026.5.12 verifies a partial subset while showing text-size and the requested named effects remain absent.

## Solution Assessment

No. Completing this with guessed aliases or ad hoc rich fields would be less maintainable than an upstream-backed capability contract with explicit fallback semantics.
