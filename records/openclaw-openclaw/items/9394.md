## Summary

Keep open: current main partially supports iMessage bold/italic/underline/strikethrough runs and `sendWithEffect`, but text-size support and the requested Shake/Nod/Explode/Ripple/Bloom/Jitter effects are still absent and need product/API direction.

## What This Changes

Request that the iMessage channel send rich formatting, text-size variations, and expressive iMessage animations rather than plain text only.

## Best Possible Solution

Define a typed, upstream-backed iMessage outbound formatting/effects capability model, keep graceful plain-text fallback, and add text-size or newer effects only when `imsg` exposes stable fields.

## Reproduction Assessment

Not applicable. This is an enhancement request, and source inspection on current main shows basic style runs and classic effects exist while text-size and the requested named effects remain absent.

## Solution Assessment

No. The current partial implementation is useful, but completing this should use an upstream-backed capability model instead of ad hoc effect aliases or rich fields that `imsg` may not support.
