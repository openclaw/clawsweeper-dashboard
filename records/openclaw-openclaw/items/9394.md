## Summary

Keep open. Current main implements part of the request through markdown-derived bold/italic/underline/strikethrough handling and a `sendWithEffect` action, but text-size support and the requested Shake/Nod/Explode/Ripple/Bloom/Jitter effects are still absent.

## What This Changes

Add rich iMessage outbound formatting, text-size variants, and named iMessage effects to the bundled iMessage channel.

## Best Possible Solution

Define an upstream-backed iMessage outbound rich-content capability model, keep the supported style/effect subset, and add text-size or newer text effects only where imsg exposes stable typed fields with documented fallback behavior.

## Reproduction Assessment

Not applicable: this is an enhancement request, not a broken documented behavior. Source inspection on current main gives high confidence that only part of the requested capability exists.

## Solution Assessment

No, current main is not yet the complete solution. The partial implementation is useful, but completing the request needs maintainer API/product direction and upstream-backed semantics rather than guessing effect identifiers or formatting fields.
