## Summary

Keep open. Current main partially satisfies the request with Markdown-derived bold/italic/underline/strikethrough runs and classic `sendWithEffect`, but it still lacks text-size support and the requested Shake/Nod/Explode/Ripple/Bloom/Jitter effects, so the remaining work needs maintainer API/product direction.

## What This Changes

The issue requests richer iMessage outbound formatting, text-size variations, and named iMessage text/screen effects.

## Best Possible Solution

Define an upstream-backed iMessage outbound formatting/effects capability model, keep the supported subset, and add text-size or newer text-effect support only where imsg exposes stable typed fields with graceful fallback behavior.

## Reproduction Assessment

Not applicable. This is an enhancement request, and source inspection on current main verifies partial coverage while text-size and the requested named effects remain absent.

## Solution Assessment

No. The current partial implementation is useful, but completing the request needs an explicit capability/fallback model rather than ad hoc effect names or formatting fields.
