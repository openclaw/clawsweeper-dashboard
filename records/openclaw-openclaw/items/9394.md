## Summary

Keep open: current main now partially implements iMessage rich sends with basic style runs and screen effects, but the requested text-size support and Shake/Nod/Explode/Ripple/Bloom/Jitter text effects remain absent and still need product/API direction.

## What This Changes

The issue asks the iMessage plugin to send rich text styles, text-size variations, and animated effects.

## Best Possible Solution

Define typed outbound rich-content capabilities for iMessage backed by imsg feature detection, preserve the existing plain fallback, add text-size and specific animation support only where the bridge contract supports them, and document unsupported cases.

## Reproduction Assessment

Yes, at source level. Current main supports only basic rich style runs and a fixed sendWithEffect alias list, while the requested text-size and Shake/Nod/Explode/Ripple/Bloom/Jitter effects are absent from the formatter and action mapping.

## Solution Assessment

No complete solution is on main yet. The maintainable path is a typed capability model plus upstream-backed imsg implementation, not adding unverified effect names or style fields directly to the current action handler.
