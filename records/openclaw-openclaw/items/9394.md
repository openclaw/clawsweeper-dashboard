## Summary

Keep open. Current main partially implements the request through markdown-derived bold/italic/underline/strikethrough runs and a `sendWithEffect` action, but it still lacks text-size support and the requested Shake/Nod/Explode/Ripple/Bloom/Jitter text effects.

The issue requests rich iMessage outbound formatting, text-size attributes, and expressive message effects beyond plain text.

## Best Possible Solution

Define an upstream-backed iMessage outbound formatting/effects capability model, keep the current supported subset, and add text-size or newer text-effect support only where imsg can expose stable typed fields and graceful fallback behavior.
