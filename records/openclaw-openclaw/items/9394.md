Keep this open. Current main and v2026.5.12 now cover a useful subset of the request, but text-size variations and the specifically requested Shake/Nod/Explode/Ripple/Bloom/Jitter effects are still not implemented or tracked by a narrower canonical issue.

## What This Changes

The issue requests rich iMessage text formatting, text size, and expressive effects for outbound OpenClaw iMessage messages.

## Best Possible Solution

Define a typed iMessage outbound capability model, keep the shipped bold/italic/underline/strikethrough and classic effect support, and add text-size or newer animation support only where imsg exposes a verified transport contract with documented fallback behavior.
