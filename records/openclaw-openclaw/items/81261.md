## Summary

Keep open: current main has since fixed the Xiaomi OpenAI-compatible replay bug through https://github.com/openclaw/openclaw/pull/81589, but the broader Anthropic Messages stream/replay request here is still not implemented. The Anthropic transport still only handles native `thinking_delta`/`signature_delta`, and MiniMax remains on the disable-thinking workaround for OpenAI-style `reasoning_content` streams.

## What This Changes

The issue requests provider-aware support for OpenAI-style `reasoning_content` deltas and replay on Anthropic-compatible endpoints such as MiMo and MiniMax.

## Best Possible Solution

Define a provider-aware Anthropic-compatible stream and replay contract that normalizes OpenAI-style `reasoning_content` only for endpoints that require it, while keeping the existing MiniMax disable-thinking behavior for endpoints that cannot safely replay reasoning.

## Reproduction Assessment

Yes, at source level: an Anthropic Messages stream that emits `delta.reasoning_content` has no current parser branch, and replay still emits native Anthropic thinking rather than `reasoning_content`. I did not validate the full reported 400 cascade against a live provider.

## Solution Assessment

Mostly yes, but the implementation should be provider-aware rather than a blanket Anthropic transport change. The Xiaomi OpenAI-compatible fix is a useful precedent, but it does not solve this Anthropic-compatible stream path.
