## Summary

Keep open: current main still lacks provider-safe support for OpenAI-style `reasoning_content` on Anthropic Messages streams. The existing MiniMax path disables thinking to avoid leaks, while the Anthropic transport still parses and replays native Anthropic thinking blocks only.

## What This Changes

The issue requests normalization and replay support for Anthropic-compatible endpoints that emit OpenAI-style `reasoning_content` instead of native Anthropic thinking deltas.

## Best Possible Solution

Define a provider-aware Anthropic-compatible stream and replay contract that normalizes `reasoning_content` only for endpoints that require it, while keeping existing disable-thinking fallbacks for providers that cannot safely replay reasoning.

## Reproduction Assessment

Yes, at source level: an Anthropic SSE `content_block_delta` carrying `delta.reasoning_content` has no handling branch in current main, while native `thinking_delta` does. I did not validate the full provider 400 cascade live.

## Solution Assessment

Mostly yes: the requested direction is the right end state, but it should be provider-aware rather than adding Xiaomi to the MiniMax disable-thinking workaround or assuming every Anthropic-compatible endpoint accepts `reasoning_content` replay.
