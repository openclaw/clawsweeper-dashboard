## Summary

Keep open: current main has OpenAI-completions MiMo/Kimi replay fixes and a MiniMax Anthropic-messages disabled-thinking workaround, but the fresh `xiaomi-token-plan-ams/mimo-v2.5` Anthropic-messages reproduction is not covered by those paths.

## What This Changes

The issue asks OpenClaw to handle OpenAI-style `reasoning_content` deltas from Anthropic-compatible endpoints and preserve/replay them for MiMo/MiniMax-style providers.

## Best Possible Solution

Add a focused Anthropic-messages compatibility path for affected MiMo/MiniMax-style endpoints that either converts OpenAI-style `reasoning_content` stream deltas into internal thinking blocks with replay support, or explicitly disables thinking only for routes where proper translation is not available, with regression coverage for `xiaomi-token-plan-ams/mimo-v2.5`.

## Reproduction Assessment

Yes, at source level. A `xiaomi-token-plan-ams` Anthropic-messages MiMo model follows paths that are not covered by the Xiaomi provider wrapper, the OpenAI-completions MiMo replay wrappers, or the MiniMax provider-id workaround, and the latest comment provides concrete provider logs.

## Solution Assessment

Unclear. The requested structured translation/replay is likely the right end state, but maintainers should choose whether this belongs in OpenClaw wrappers, the Xiaomi plugin/provider scope, or the upstream pi-ai Anthropic provider rather than landing another provider-id workaround by default.
