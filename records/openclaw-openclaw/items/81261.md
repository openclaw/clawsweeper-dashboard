## Summary

Keep open: current main still lacks a provider-safe Anthropic Messages path for OpenAI-style `reasoning_content` deltas and replay. The active Xiaomi PR covers the related OpenAI-compatible MiMo replay bug, but it does not cover the broader Anthropic-compatible stream parsing and replay contract requested here.

## What This Changes

The issue requests support for Anthropic-compatible endpoints that stream OpenAI-style `reasoning_content`, including converting it into thinking blocks and replaying it safely on later turns.

## Best Possible Solution

Design a provider-aware Anthropic-compatible stream and replay contract that normalizes OpenAI-style `reasoning_content` only for endpoints known to accept that shape, while keeping defensive thinking-disable behavior for providers that cannot safely replay it.

## Reproduction Assessment

Yes at source level: an Anthropic Messages stream that emits `delta.reasoning_content` instead of native `thinking_delta` has no handling path in current main. I did not establish a fresh live 400 cascade against MiMo or MiniMax.

## Solution Assessment

Mostly yes, but the maintainable solution needs provider-aware parsing and replay rather than assuming every Anthropic-compatible endpoint accepts `reasoning_content`. The open Xiaomi PR is the right direction for the OpenAI-compatible sibling bug, not a complete solution for this issue.
