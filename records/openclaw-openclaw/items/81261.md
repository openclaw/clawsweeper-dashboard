## Summary

Keep open: current main still lacks a provider-safe path for OpenAI-style `reasoning_content` on Anthropic Messages streams, and the remaining work needs provider/replay policy design rather than cleanup closure.

## What This Changes

The issue requests support for Anthropic-compatible endpoints that emit OpenAI-style `reasoning_content` deltas, including conversion to thinking blocks and replay on follow-up turns.

## Best Possible Solution

Define a provider-aware stream and replay contract that normalizes OpenAI-style `reasoning_content` on Anthropic-compatible streams only for providers that safely accept that replay shape, while keeping workaround disabling for providers that cannot.

## Reproduction Assessment

Yes, at source level: current main handles native Anthropic thinking blocks and OpenAI-compatible `reasoning_content` on separate paths, but not OpenAI-style `delta.reasoning_content` on Anthropic Messages streams. I did not establish a live MiMo or MiniMax 400 cascade reproduction.

## Solution Assessment

Mostly yes, but the proposed fix needs a provider-aware design rather than a broad assumption that every Anthropic-compatible endpoint accepts `reasoning_content` replay. Adding Xiaomi to the MiniMax disable-thinking workaround would reduce crashes but would not satisfy the requested thinking support.
