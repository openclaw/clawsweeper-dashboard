## Summary

Keep open: current main still lacks a provider-safe path for OpenAI-style `reasoning_content` on Anthropic Messages streams. The existing MiniMax workaround disables thinking to avoid the leak, while the OpenAI transport already has a separate reasoning-content normalization path.

## What This Changes

The issue requests support for Anthropic-compatible providers that stream OpenAI-style `reasoning_content` deltas, including preserving that reasoning across turns.

## Best Possible Solution

Define a provider-aware stream and replay contract that normalizes OpenAI-style `reasoning_content` on Anthropic-compatible streams only for endpoints that accept that replay shape, while keeping thinking disabled for providers that cannot safely replay it.

## Reproduction Assessment

Yes at source level: an Anthropic SSE stream that emits `delta.reasoning_content` instead of native `thinking_delta` has no handling path in current main. I did not validate the full reported 400 cascade against a live provider endpoint.

## Solution Assessment

Mostly yes, but the fix should be provider-aware rather than a broad assumption that every Anthropic-compatible endpoint accepts `reasoning_content` replay. Adding Xiaomi to the MiniMax disable-thinking workaround would reduce crashes but would not satisfy the requested thinking support.
