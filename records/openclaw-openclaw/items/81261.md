## Summary

Keep open: current main has adjacent MiMo reasoning fixes, but the bundled token-plan Anthropic Messages routes still appear to bypass them because their provider ids are `xiaomi-token-plan-*`, not `xiaomi`, and the Anthropic stream parser still has no `reasoning_content` delta path.

## What This Changes

The issue requests support for Anthropic-compatible endpoints that stream OpenAI-style `reasoning_content` deltas, especially Xiaomi MiMo token-plan routes, without disabling thinking.

## Best Possible Solution

Apply a focused provider-runtime fix so Xiaomi token-plan Anthropic Messages routes either receive the existing MiMo reasoning replay wrapper by endpoint class/provider family or get a generic `reasoning_content`-to-thinking-block translation with regression coverage.

## Reproduction Assessment

Yes, source-reproducible. The comment gives an exact failing provider/model/error, the pinned pi-ai catalog exposes that provider as Anthropic Messages, and current main only applies the MiMo replay wrapper to provider `xiaomi`.

## Solution Assessment

Yes, the requested direction is better than disabling thinking. The narrowest maintainable fix should reuse existing provider stream/replay helpers or endpoint-class attribution, with a fallback to generic Anthropic SSE translation only if that is the cleaner shared contract.
