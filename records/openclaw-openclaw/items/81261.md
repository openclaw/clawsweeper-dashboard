## Summary

Keep open: current main still lacks a provider-safe Anthropic Messages path for OpenAI-style `reasoning_content`, and the newer report confirms the bundled `xiaomi-token-plan-ams` Anthropic route can still fail despite adjacent OpenAI-compatible MiMo fixes.

## What This Changes

The issue requests support for OpenAI-style `reasoning_content` on Anthropic-compatible endpoints such as MiMo and MiniMax.

## Best Possible Solution

Add a provider-aware Anthropic Messages stream and replay contract that normalizes OpenAI-style `reasoning_content` only for endpoints that require it, while preserving existing native Anthropic thinking behavior and MiniMax safety defaults where needed.

## Reproduction Assessment

Yes at source level: current main has no Anthropic Messages `delta.reasoning_content` parsing or replay path, and the later live report shows the same 400 on `xiaomi-token-plan-ams/mimo-v2.5`. I did not run a credentialed provider request myself.

## Solution Assessment

Mostly yes: the requested stream normalization and replay direction matches the code gap, but the maintainable fix should be provider-aware rather than a blanket Anthropic-compatible behavior change.
