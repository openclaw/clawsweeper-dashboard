## Summary

Keep open for maintainer product decision: the reported suppression is real on current main, but current source, tests, and docs intentionally define group/channel final replies as message-tool-only unless `messages.groupChat.visibleReplies: "automatic"` is set. Changing the default to automatic would reverse a documented group/channel reply policy rather than fix a broken existing contract.

## What This Changes

Request to make WhatsApp group auto-replies post automatically by default instead of requiring `message(action=send)` or `messages.groupChat.visibleReplies: "automatic"`.

## Best Possible Solution

Maintainers should explicitly decide the group/channel visible-reply default; if it changes, update the resolver, doctor migration behavior, docs, config help, and tests together, otherwise add WhatsApp-specific troubleshooting or diagnostics for this symptom while keeping the tool-only default.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: current main resolves WhatsApp group source turns to `message_tool_only` by default and suppresses automatic source delivery; I did not perform a live WhatsApp group run in this read-only pass.

## Solution Assessment

Is this the best way to solve the issue? No for automated implementation: flipping the default is a product-policy change that conflicts with current docs, doctor behavior, and tests, so the best path is explicit maintainer review before any code change.
