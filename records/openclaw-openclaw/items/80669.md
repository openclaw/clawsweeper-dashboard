## Summary

Keep open: current main still uses the documented tool-only group/channel visible-reply default, so the requested default change is not implemented and needs maintainer policy judgment rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to default group/channel source replies to automatic visible delivery instead of requiring the message tool when `messages.groupChat.visibleReplies` is unset.

## Best Possible Solution

Maintainers should choose the group/channel visible-reply policy, then update the resolver, doctor repair, diagnostics/outcome reporting, docs, and `messages.*` hot reload behavior as one consistent contract.

## Reproduction Assessment

Yes, at source level: current main resolves unset group/channel visible replies to `message_tool_only`, suppresses ordinary automatic final delivery, and the docs/tests encode that behavior. I did not run a live WhatsApp or Discord reproduction.

## Solution Assessment

Unclear: flipping the default to `automatic` would address the reported silent drops, but it conflicts with the currently documented tool-only privacy policy. A safer solution needs maintainer direction on default policy versus diagnostics/fallback behavior.
