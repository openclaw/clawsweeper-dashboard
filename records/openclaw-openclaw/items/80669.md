## Summary

Keep open for maintainer product review: current main intentionally keeps group/channel final replies message-tool-only by default, while the reporter is asking to reverse that documented default for WhatsApp-style group auto-replies.

## What This Changes

Request changing the default group/channel visible reply mode from message-tool-only to automatic so WhatsApp group auto-replies post without explicit config.

## Best Possible Solution

Maintainers should decide whether to keep the global group tool-only default, add a WhatsApp-specific diagnostic or onboarding path, or change the default across all group-capable channels with resolver, doctor, docs, and regression tests updated together.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction of the suppression mechanism: an unset group/channel config resolves to `message_tool_only`, which suppresses automatic delivery and has tests expecting no final send. I did not live-test the reporter's WhatsApp group, and current docs define this as intended behavior rather than a broken contract.

## Solution Assessment

Unclear as an implementation direction: changing the default to `automatic` would reverse a documented, tested group/channel product policy. The safer next step is maintainer product review before deciding between a default change, a WhatsApp-specific diagnostic, or clearer onboarding.
