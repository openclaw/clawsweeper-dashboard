## Summary

Keep open: the proposed default flip conflicts with current documented behavior and needs maintainer product judgment, but the thread also identifies a real current-main gap where `messages.*` reloads are treated as no-op and do not refresh runtime state.

## What This Changes

Request that WhatsApp group auto-replies become visibly automatic by default, or at least stop silently staying private under the group visible-reply policy.

## Best Possible Solution

Preserve the documented default unless maintainers choose a policy change, fix `messages.*` reload so the runtime snapshot updates without restart, and add a clear diagnostic for suppressed non-empty group finals.

## Reproduction Assessment

Yes, at source level: group/channel turns resolve to `message_tool_only`, and pure `messages.*` config changes return before the runtime hot-reload state is refreshed. I did not run a live WhatsApp group, but the issue has multiple 2026.5.7 confirmations and the current code path is clear.

## Solution Assessment

No for the proposed default flip as an automatic fix: current docs and tests intentionally keep group/channel finals private by default. The safer path is to fix the reload bug and add diagnostics, then let maintainers decide whether the group default should change.
