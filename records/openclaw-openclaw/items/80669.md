## Summary

Close as duplicate/superseded: the WhatsApp symptom is the same shared tool-only visible-reply suppression policy tracked by https://github.com/openclaw/openclaw/issues/77320, and the exact default-to-automatic proposal was already reviewed and closed unmerged in https://github.com/openclaw/openclaw/pull/77808 because the current group/channel default is intentional.

## What This Changes

The issue reports that WhatsApp group auto-replies stay private under the default group visible-replies mode and asks to default `messages.groupChat.visibleReplies` to `automatic`.

## Best Possible Solution

Preserve the documented tool-only default unless maintainers explicitly change the privacy policy, and handle diagnostics or fallback behavior under https://github.com/openclaw/openclaw/issues/77320.

## Reproduction Assessment

Yes, at source level: a group WhatsApp turn without an explicit `messages.groupChat.visibleReplies: "automatic"` override resolves to `message_tool_only` and suppresses automatic final delivery. I did not run a live WhatsApp gateway in this read-only review.

## Solution Assessment

No. Making `automatic` the unset default is not the best narrow fix because it reverses the documented private-by-default group/channel contract; the safer path is the canonical diagnostics or fallback decision in https://github.com/openclaw/openclaw/issues/77320.
