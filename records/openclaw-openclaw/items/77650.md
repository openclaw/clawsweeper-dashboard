## Summary

Keep this PR open: the sessions.list performance target is valid, but the current fast path regresses the selected-model contract for explicit overrides and needs a narrow repair before merge.

## What This Changes

The PR adds a sessions.list selected-model cache/fast path plus real-session benchmark utilities and a package script.

## Best Possible Solution

Preserve the sessions.list optimization by caching resolver results or fast-pathing only when persisted metadata is known to match the explicit selected override, then add the required changelog entry.

## Reproduction Assessment

Yes for the PR regression: source inspection maps the new fast path to the existing selected-override test, and the provided CI context reports that assertion failing. The original performance issue is plausible from current main's per-row resolver call, but was not benchmarked here.

## Solution Assessment

No; this is not the best way to solve the issue because it bypasses the resolver contract that explicit overrides win over runtime fallback metadata. The safer fix is to keep a row-context cache while preserving resolveSessionModelRef semantics.
