## Summary

Keep this PR open: it targets a real sessions.list hot path, but the current patch regresses selected-model display by returning persisted runtime metadata ahead of explicit session overrides, and CI already reports the matching failing gateway test.

## What This Changes

The PR adds a selected-model resolution cache/fast path in `src/gateway/session-utils.ts`, plus a `test:sessions:list:bench` package script and benchmark seeding utilities.

## Best Possible Solution

Keep the sessions.list optimization, but preserve the selected-model contract by caching resolver results or fast-pathing only when persisted metadata is known to match the explicit selected override.

## Reproduction Assessment

Yes for the PR regression: head CI reports the existing gateway session-utils test failing, and source inspection maps it to the new runtime-field fast path. The original performance issue is plausible from current main, but I did not run the benchmark in this read-only review.

## Solution Assessment

No; this is not the best way to solve the issue because it bypasses the resolver contract that explicit overrides win over runtime fallback metadata. A safer repair is to keep the row-context cache while preserving `resolveSessionModelRef` semantics.
