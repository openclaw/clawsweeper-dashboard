## Summary

Keep open: this PR targets the canonical Telegram approval cleanup bug, but it is not merge-ready because it redefines the shared `clear-actions` hook to delete delivered content, leaves the Telegram approval deps type incoherent, and still supplies only unit-test output rather than real Telegram proof.

## What This Changes

The PR changes Telegram native approval cleanup to delete the delivered approval message after finalization instead of clearing its inline keyboard.

## Best Possible Solution

Use the approval final-action delete path for resolved Telegram approvals, keep `clearPendingActions` as button-only cleanup for clear-action cases, add focused tests, and require redacted real Telegram proof.

## Reproduction Assessment

Yes, source-level: current main resolves Telegram approvals with `clear-actions`, and the active Telegram runtime edits reply markup instead of deleting the delivered prompt. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No as submitted. Deleting the message is the right outcome for resolved approvals, but it should be modeled as a resolved delete final action with type-safe deps, not by redefining `clear-actions`.
