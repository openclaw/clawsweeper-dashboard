## Summary

Keep this issue open: current main and the latest release still have the reported source path, the closed alternate PR was rejected as not failing closed, and the active draft fix remains under review with a delivery-contract decision and Telegram proof still pending.

## What This Changes

The issue reports that isolated cron delivery can classify unavailable-tool self-debug text as successful assistant output and send it to Telegram.

## Best Possible Solution

Land a structured runner-owned fatal signal for exhausted unavailable-tool loops, with cron outcome tests and live Telegram/Gateway proof matching the maintainer-chosen delivery disposition.

## Reproduction Assessment

Yes. Source inspection shows current main can rewrite repeated unavailable-tool calls into the reported final assistant text and then select final assistant-visible text for cron delivery after non-terminal tool warnings; the issue also includes a real Telegram/Gateway trace, but I did not run a live reproduction in this read-only review.

## Solution Assessment

Unclear as a final contract. The structured runner-owned signal in the active draft appears to be the right layer, but maintainers still need to choose whether the fatal diagnostic is delivered to the user topic or suppressed outside chat.
