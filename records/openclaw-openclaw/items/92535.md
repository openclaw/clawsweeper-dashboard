## Label Justifications

- `P1`: A production isolated-cron workflow can publish internal runtime failure text to chat while recording the run as successful, affecting real channel delivery now.
- `impact:session-state`: The invalid assistant output is appended to the topic transcript and can later enter memory or dreaming state, amplifying the bad result.

## Summary

Keep this issue open: the supplied real-gateway trace matches a current-main gap where repeated unavailable-tool calls can become ordinary assistant text, and isolated cron can classify that text as successful deliverable output. The related open work addresses tool availability, policy preservation, A2A feedback, or downstream memory filtering, but none owns this exact terminal-outcome and delivery leak.

## What This Changes

Prevent isolated cron runs from treating unavailable-tool self-debug text as successful chat output and delivering or persisting it through Telegram or other channels.

## Best Possible Solution

Carry an explicit exhausted-unavailable-tool terminal signal from the agent runner into cron outcome classification, mark isolated cron failed/not-delivered, and retain an operator diagnostic without publishing or mirroring model-authored self-debug text.

## Reproduction Assessment

Yes—current-main source provides a high-confidence path: the unknown-tool guard converts repeated unavailable-tool calls to assistant text, and cron can accept final assistant-visible text as recovered output. The reporter also supplied a concrete real-gateway trajectory and Telegram outbound evidence, although this review did not execute a fresh current-main live run.

## Solution Assessment

No—the locally tested delivery-text guard is useful defense in depth but is not the best primary fix because model phrasing is unstable and legitimate text could match. The canonical fix is a structured runner failure signal consumed by cron before delivery and transcript persistence.

## Repair Work Prompt

Preserve the existing repeated unknown-tool loop protection, but make exhaustion produce a structured terminal failure that reaches cron outcome classification instead of relying on model-authored self-debug text. Isolated cron must record an error/not-delivered outcome and must not announce, mirror, or append that self-debug text to Telegram or other chat transcripts; preserve an operator-facing diagnostic outside the user thread. Add focused regression coverage for the runner signal, cron status, delivery suppression, and transcript/mirror behavior.
