## Summary

Keep this issue open: current main still has the source path that can turn exhausted unavailable-tool calls into ordinary assistant text, and the canonical no-dispatch fix is still the open maintainer-linked PR at https://github.com/openclaw/openclaw/pull/93207 rather than merged code.

## What This Changes

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/93207, or an equivalent structured runner-signal/no-dispatch cron fix, then close this issue through the merged PR.

## Solution Assessment

Yes, the structured runner exhaustion signal consumed by cron before dispatch is the best fix shape. A phrase-match delivery filter is weaker, and the current open PR implements the safer no-dispatch path but has not merged yet.
