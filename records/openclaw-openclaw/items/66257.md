## Summary

Keep open: current main has not implemented the PR's local-first approval behavior, and the proposed patch changes a security-sensitive approval path before preserving Gateway authorization, Gateway side effects, or real behavior proof.

## What This Changes

The PR changes `openclaw devices approve` to try local loopback pairing-file approval before Gateway approval, suppress local fallback for explicit token/password auth, and update CLI fallback tests.

## Best Possible Solution

Preserve the bug-fix goal, but rework the change on current main so approval remains behind Gateway-equivalent authorization and resolution notifications, then add focused tests and redacted real CLI/Gateway proof.

## Reproduction Assessment

Yes for review findings: source inspection shows the PR approves locally before the Gateway path, while current Gateway code and tests enforce non-admin cross-device approval denial. I did not run a live CLI/Gateway repro under the read-only review constraint.

## Solution Assessment

No. Local-first approval is not the narrowest maintainable fix because it changes the approval authority and skips Gateway notification side effects; a safer fix should preserve Gateway authorization or mirror the same checks and broadcasts in a tightly bounded local path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve Gateway authorization before local approval:** `src/cli/devices-cli.ts:168-172`
  - body: The new local-first branch approves pairing files before `device.pair.approve` can enforce caller-device ownership. Because the local helper uses `operator.admin` scopes, an implicit local device-token session can approve a different pending device that the Gateway path would reject.
  - confidence: 0.88
- **[P2] Preserve approval resolution broadcasts:** `src/cli/devices-cli.ts:168-172`
  - body: Returning immediately after local approval skips the Gateway success path that logs approval and broadcasts `device.pair.resolved`. UI or Gateway subscribers that rely on that event can remain stale after CLI local-first approval.
  - confidence: 0.84
