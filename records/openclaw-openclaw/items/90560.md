## What This Changes

The PR changes Matrix reply delivery to pass explicit reasoning replies to `sendMessageMatrix` as `m.notice`, adds a `msgtype` option for text sends, and adds focused Matrix helper tests.

## Best Possible Solution

Add a Matrix-owned dispatch opt-in or channel capability so explicit `isReasoning` payloads reach Matrix as `m.notice` while generic, off, and raw reasoning-looking text remain suppressed, then prove it with a real Matrix turn.

## Reproduction Assessment

Yes, source-reproducible: current main's dispatch path suppresses `isReasoning` before Matrix dispatcher delivery, and the PR leaves that upstream filter in place. I did not run a live Matrix client in this read-only review.

## Solution Assessment

No. Sending `isReasoning` payloads from `deliverMatrixReplies` is plausible but too low in the call path; the best fix needs the Matrix dispatch path to opt into explicit reasoning delivery and tests that cover that path.
