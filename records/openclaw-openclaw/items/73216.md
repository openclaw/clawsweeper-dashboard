## Summary

Keep open. Current main still lacks dynamic GitHub Copilot model discovery, but this PR head needs repairs before merge: it is stale against current main's agent-scoped Copilot auth flow, and the new discovery cache/fallback logic can miss agent auth, leak catalog state across accounts, and drop known offline `xhigh` support.

## What This Changes

The branch adds GitHub Copilot runtime model discovery from the Copilot `/models` API, mapping/caching helpers, thinking-profile mapping, tests, and small adjacent SDK/test cleanups.

## Best Possible Solution

Land a rebased GitHub Copilot plugin implementation that fetches model capabilities through agent-scoped auth, keys cached data by active credential/base URL, preserves known offline fallbacks, and records the user-facing change in the changelog.

## Reproduction Assessment

Yes. Source inspection gives high-confidence paths for the stale auth regression, agentDir omission, module-global cache reuse, and lost no-API `xhigh` fallback without needing live credentials.

## Solution Assessment

No, not as written. Dynamic Copilot `/models` discovery is the right direction, but the implementation needs rebase/auth/cache/fallback repairs before it is the narrow maintainable solution.
