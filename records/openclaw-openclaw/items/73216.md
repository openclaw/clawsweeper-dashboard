## Summary

Keep open. Current main still lacks dynamic GitHub Copilot `/models` catalog discovery, and the related #72824/#72825 context shows a real capability gap, but this PR head is not merge-ready because its auth context, cache scoping, offline thinking fallback, and changelog handling need repair.

## What This Changes

The PR adds GitHub Copilot `/models` API discovery with mapped model capabilities, cache-backed catalog augmentation, dynamic-model preparation, thinking-profile support, tests, and small SDK/test cleanups.

## Best Possible Solution

Land a refined GitHub Copilot plugin implementation that fetches authenticated model capabilities through provider-owned hooks, scopes cached data to the credential/baseUrl/agent context, preserves known offline fallbacks, and records the user-facing change in the changelog.

## Reproduction Assessment

Yes for the PR blockers. Source inspection shows profile-only agent auth misses the shared fetch, module-global cache reuse crosses agents/accounts, and API failure removes current known `xhigh` fallback behavior.

## Solution Assessment

No, not as written. Dynamic Copilot `/models` discovery is the right direction, but the implementation needs request-scoped auth/cache semantics and the existing known-model fallback before it is safe to merge.

  - confidence: 0.9

  - confidence: 0.86

  - body: When live discovery fails, times out, or has not populated the cache, this fallback now returns only the generic levels from `resolveThinkingProfileFromCapabilities(undefined)`. That regresses current `xhigh` support for known Copilot GPT/Codex ids such as `gpt-5.4`, `gpt-5.2`, and Codex variants.
  - confidence: 0.88
