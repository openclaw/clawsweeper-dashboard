## Summary

Keep open. Current main still has the static GitHub Copilot catalog/limit behavior this PR is trying to improve, but the PR head has blocking correctness issues around agent-scoped auth, process-global capability caching, and offline thinking fallbacks, plus the required changelog entry is missing.

## What This Changes

The PR adds GitHub Copilot `/models` API discovery with mapping, caching, catalog augmentation, dynamic-model preparation, thinking-profile support, tests, and small SDK/test cleanups.

## Best Possible Solution

Land a refined GitHub Copilot plugin implementation that fetches authenticated model capabilities through provider-owned hooks, scopes cached data to the credential/baseUrl/agent context, preserves known offline fallbacks, and documents the user-facing change in the changelog.

## Reproduction Assessment

Yes. Source inspection gives high-confidence reproduction paths: an agent-local Copilot profile with no env/default token yields an empty dynamic catalog, two agents or profile changes can reuse the first cached plan catalog, and API failure for known Copilot GPT/Codex models drops `xhigh`.

## Solution Assessment

No, not as written. Dynamic `/models` discovery is the right direction, but the implementation needs request-scoped auth/cache semantics and a fallback that preserves current known-model thinking behavior.

  - confidence: 0.9

  - confidence: 0.86

  - confidence: 0.88
