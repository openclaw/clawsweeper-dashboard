## Summary

Keep open. Current main still lacks authenticated Copilot `/models` discovery, and this PR head still has blocking correctness issues around agent-local auth, cache scoping, and offline thinking-level fallback behavior.

## What This Changes

The PR adds GitHub Copilot runtime `/models` fetching, capability mapping, process caching, catalog augmentation, dynamic-model warm-up, thinking-profile derivation, and related tests plus small SDK/test cleanups.

## Best Possible Solution

Land a provider-owned Copilot dynamic discovery implementation that uses the current agent/auth context for token resolution, scopes or invalidates fetched capabilities by credential and base URL, preserves current fallback thinking behavior when discovery cannot run, and includes focused tests for agent-local profiles, profile switching, cache invalidation, and known-model fallback levels.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: an agent-local Copilot profile can resolve in `catalog.run` but fail inside `fetchAndCacheModels` because `agentDir` is omitted, and API-unavailable paths now lose current-main `xhigh` support for known Copilot models.

## Solution Assessment

No, not as written. Dynamic `/models` discovery is the right product direction, but the implementation needs context-scoped auth/cache handling and a compatible offline fallback before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Pass agentDir into Copilot model discovery:** `extensions/github-copilot/index.ts:54-57`
  - body: `fetchAndCacheModels` resolves the GitHub token without `agentDir`, even though the catalog hook has already used `ctx.agentDir` for preflight auth. Agent-local device-login profiles can pass that preflight and then return an empty dynamic catalog because the actual fetch looks in the default auth store. Accept `agentDir` in the helper and pass it from catalog, augmentation, and dynamic warm-up callers.
  - confidence: 0.89
- **[P2] Scope Copilot model cache by credentials:** `extensions/github-copilot/index.ts:44-45`
  - body: Once any request fills `cachedModelCapabilities`, later requests return the same plan catalog before checking the current agent, auth profile, env token, or Copilot base URL. A gateway serving multiple agents, or a user switching profiles, can show stale capabilities from another Copilot account. Key the cache by the resolved credential/baseUrl/agent context or invalidate it when that context changes.
  - confidence: 0.82
- **[P2] Preserve xhigh fallback when discovery fails:** `extensions/github-copilot/index.ts:463-464`
  - body: Current main exposes `xhigh` for known Copilot GPT/Codex models through a hardcoded allowlist. This fallback now always returns the generic profile without `xhigh` whenever `/models` has not populated the cache, so API outage, timeout, or auth-profile discovery failure regresses `/think xhigh` for models such as `gpt-5.4` and `gpt-5.2`. Keep the known-model fallback when live capabilities are unavailable.
  - confidence: 0.86
