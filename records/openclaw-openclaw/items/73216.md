## Summary

Keep open. Current main still uses the static Copilot catalog/default limits, but this PR needs credential-scoped discovery, opt-out handling, offline thinking fallback preservation, and a changelog entry before merge.

## What This Changes

The PR adds runtime GitHub Copilot `/models` discovery, mapping, caching, catalog augmentation, dynamic-model preparation, thinking-profile support, and related tests.

## Best Possible Solution

Land a refined Copilot plugin implementation that fetches authenticated model capabilities through provider-owned hooks, scopes cache entries by credential/baseUrl/agent context, honors discovery opt-outs, preserves known offline fallbacks, and documents the user-facing change.

## Reproduction Assessment

Yes. Source inspection gives high-confidence paths: use an agent-local Copilot token without env/default token, switch agents or credentials in one gateway, disable Copilot discovery, or let `/models` fail for a known xhigh model.

## Solution Assessment

No, not as written. Dynamic `/models` discovery is the right direction, but the implementation needs request-scoped auth/cache semantics, opt-out consistency, and fallback behavior that preserves current known-model thinking support.

Full review comments:

- **[P2] Pass the agent context into Copilot model discovery:** `extensions/github-copilot/index.ts:54-57`
  - body: `fetchAndCacheModels` re-resolves the GitHub token without `agentDir`, even though the catalog path is agent-scoped. An agent-local Copilot profile with no env/default token can pass the preflight check and then get an empty dynamic catalog because the shared fetch reads the wrong auth store.
  - confidence: 0.91
- **[P2] Scope the model cache to the active Copilot credentials:** `extensions/github-copilot/index.ts:44-45`
  - body: The first successful `/models` fetch fills one module-level cache and later calls return it before checking the current config, env token, auth profile, base URL, or agent. Multi-agent gateways or auth/profile changes can show another account's model plan until restart.
  - confidence: 0.88
- **[P2] Honor disabled Copilot discovery in new fetch hooks:** `extensions/github-copilot/index.ts:421-442`
  - body: `catalog.run` checks `discoveryEnabled`, but `prepareDynamicModel` and `augmentModelCatalog` call `fetchAndCacheModels` unconditionally. A user who disables Copilot discovery can still trigger the authenticated `/models` request from model listing or dynamic model use.
  - confidence: 0.86
- **[P2] Preserve the known xhigh fallback:** `extensions/github-copilot/index.ts:463-464`
  - body: When live discovery fails, times out, or has not populated the cache, this fallback now returns only generic levels. That regresses current `xhigh` support for known Copilot GPT/Codex ids such as `gpt-5.4`, `gpt-5.3-codex`, `gpt-5.2`, and `gpt-5.2-codex`.
  - confidence: 0.88

  - body: This PR changes user-visible GitHub Copilot model listing and thinking capability behavior, but it does not update `CHANGELOG.md`. Repo policy requires a changelog entry for user-facing `feat` changes before merge.
