## Summary

Keep open. Current main still lacks dynamic GitHub Copilot `/models` catalog discovery, and this PR is a useful direction, but the current patch has source-confirmed blockers around agent-scoped auth, cache scoping, offline `xhigh` fallback behavior, and the required changelog entry.

## What This Changes

The PR adds GitHub Copilot `/models` API discovery, model capability mapping, catalog augmentation, dynamic-model preparation, thinking-profile mapping, and related unit tests.

## Best Possible Solution

Land a refined GitHub Copilot plugin implementation that fetches authenticated model capabilities through provider-owned hooks, scopes cached data by the active agent/credential/baseUrl context, preserves existing known fallback levels, and records the user-facing change in the changelog.

## Reproduction Assessment

Yes. Source inspection gives high-confidence paths for the PR blockers: profile-only agent auth misses the shared fetch, module-global cache reuse crosses agents/accounts, and API failure removes current known `xhigh` fallback behavior.

## Solution Assessment

No, not as written. Dynamic Copilot `/models` discovery is the right direction, but the implementation needs request-scoped auth/cache semantics and the existing known-model fallback before it is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Pass the agent context into model discovery:** `extensions/github-copilot/index.ts:54-57`
  - body: `fetchAndCacheModels` resolves the GitHub token without `agentDir`, even though the surrounding catalog and dynamic-model paths are agent-scoped. An auth-profile-only agent with no env/default token can pass the preflight path and still get an empty dynamic catalog because the shared fetch looks in the wrong auth store.
  - confidence: 0.9
- **[P2] Scope the model cache to the active credentials:** `extensions/github-copilot/index.ts:44-45`
  - body: The first successful `/models` fetch fills one module-level cache, and later calls return it before checking the current config, env token, auth profile, base URL, or agent. In a multi-agent gateway or after auth/profile changes, `/models list` and thinking capabilities can show another account's plan until restart.
  - confidence: 0.86
- **[P2] Preserve the known xhigh fallback:** `extensions/github-copilot/index.ts:463-464`
  - body: When live discovery fails, times out, or has not populated the cache, this fallback now returns only the generic levels from `resolveThinkingProfileFromCapabilities(undefined)`. That regresses current `xhigh` support for known Copilot GPT/Codex ids such as `gpt-5.4`, `gpt-5.2`, and Codex variants.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `extensions/github-copilot/index.ts:401-415`
  - body: This PR changes user-visible GitHub Copilot model listing and capability behavior, but it does not update `CHANGELOG.md`. Repo policy requires a changelog entry for user-facing `feat` changes before merge.
  - confidence: 0.82
