## Summary

Keep open. Current main still lacks Copilot `/models` discovery, but the PR head needs changes for agent-scoped auth, credential-scoped caching, the current `xhigh` fallback, and the required changelog entry before it is merge-ready.

## What This Changes

Adds GitHub Copilot runtime `/models` fetching, model mapping/caching hooks, thinking-profile generation, tests, and small SDK/test cleanups.

## Best Possible Solution

Land a refined Copilot plugin implementation that fetches authenticated model capabilities through agent-scoped provider hooks, scopes cached data to credential/baseUrl/agent context, preserves known offline thinking fallbacks, and documents the user-facing feature.

## Reproduction Assessment

Yes. Source inspection gives high-confidence reproduction paths: an agent-local Copilot profile with no env token yields an empty dynamic catalog, multiple agents or profile changes can reuse the first cached plan catalog, and API failure for known Copilot GPT/Codex models drops `xhigh`.

## Solution Assessment

No, not as written. Dynamic `/models` discovery is the right direction, but the implementation needs request-scoped auth/cache semantics and a fallback that preserves current known-model thinking behavior.
