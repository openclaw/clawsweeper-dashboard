The PR adds GitHub Copilot `/models` API discovery with mapping, caching, catalog augmentation, dynamic-model preparation, thinking-profile support, tests, and small SDK/test cleanups.

## Best Possible Solution

Land a refined GitHub Copilot plugin implementation that fetches authenticated model capabilities through provider-owned hooks, scopes cached data to the credential/baseUrl/agent context, preserves known offline fallbacks, and records the user-facing change in the changelog.

Yes. Source inspection gives high-confidence paths: profile-only agent auth misses the shared fetch, module-global cache reuse crosses agents/accounts, and API failure removes current known `xhigh` fallback behavior.

Full review comments:

- **[P2] Pass the agent context into model discovery:** `extensions/github-copilot/index.ts:54-57`
  - body: `fetchAndCacheModels` resolves the GitHub token without `agentDir`, even though the surrounding catalog/dynamic-model paths are agent-scoped. An auth-profile-only agent with no env/default token can pass the preflight path and still get an empty dynamic catalog because the shared fetch looks in the wrong auth store.
  - confidence: 0.9
- **[P2] Scope the model cache to the active credentials:** `extensions/github-copilot/index.ts:44-45`
  - body: The first successful `/models` fetch fills one module-level cache and later calls return it before checking the current config, env token, auth profile, base URL, or agent. In a multi-agent gateway or after auth/profile changes, `/models list` and thinking capabilities can show another account's plan until restart.
  - confidence: 0.86
- **[P2] Preserve the known xhigh fallback:** `extensions/github-copilot/index.ts:463-464`
  - body: When live discovery fails, times out, or has not populated the cache, this fallback now returns only the generic levels from `resolveThinkingProfileFromCapabilities(undefined)`. That regresses current `xhigh` support for known Copilot GPT/Codex ids such as `gpt-5.4`, `gpt-5.2`, and Codex variants.
  - confidence: 0.88

  - body: This PR changes user-visible GitHub Copilot model listing and capability behavior, but it does not update `CHANGELOG.md`. Repo policy requires a changelog entry for user-facing `feat` changes before merge.
