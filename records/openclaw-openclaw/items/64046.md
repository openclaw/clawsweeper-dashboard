## Summary

Keep open. Current main has meaningful partial coverage for SecretRefs, config RPC redaction, Control UI masking, and core log sinks, but the umbrella request is still not complete because normal config writes and backups can still persist plaintext sensitive JSON and some opt-in diagnostic/raw outputs are not uniformly gated.

## What This Changes

Request to support sensitive-data protection across config storage, OpenClaw and Gateway logs, and the built-in UI.

## Best Possible Solution

Keep this as the umbrella until linked work covers storage strategy, backup/service artifact handling, diagnostic/raw sink policy, UI inventory, and regression tests; close only after those outcomes satisfy the original storage, logging, and display requirements.

## Reproduction Assessment

Yes for the persistence portion: current source and tests show config writes and backups preserving plaintext JSON bytes. The broader UI/logging portions are partially reproducible from source inspection, but a full runtime reproduction was not run because this review is read-only.

## Solution Assessment

No, closing this now is not the best way to solve the issue. The maintainable path is to keep it open as a security-sensitive umbrella and split narrow implementation items for persistence, backups, diagnostic/raw sinks, and any remaining UI gaps.
