## Summary

Keep open. Current main covers several log and Control UI redaction paths, but the umbrella request is still active because config files and backups can persist plaintext sensitive values and some diagnostic/raw persistence paths still need security-policy decisions.

## What This Changes

Request to protect sensitive config values at rest, redact secrets consistently across logs/diagnostics, and mask sensitive Control UI fields by default.

## Best Possible Solution

Keep this as the umbrella until storage/key management, backup artifacts, diagnostic/raw sink policy, UI completeness, and regression coverage are either shipped or split into narrower canonical issues.

## Reproduction Assessment

Yes, for the storage and backup portions from current source and tests: config writes serialize plaintext JSON and backup maintenance copies those bytes. The log/UI portions are partially covered on current main, while diagnostic gaps are source-reproducible but were not runtime-exercised in this read-only review.

## Solution Assessment

No, closing this or attempting one broad automated patch is not the best way to solve it. The maintainable path is focused maintainer-reviewed work for storage/key management, backup policy, diagnostic/raw sink policy, and UI completeness.
