## Summary

Keep open. Current main has meaningful log/API/UI redaction coverage, but the umbrella request is still active because config files and managed backups can still persist plaintext secrets, and several diagnostic/raw persistence paths still need explicit security-policy decisions.

## What This Changes

The issue asks for sensitive-data protection across plaintext config storage, OpenClaw/gateway logs, and Control UI masking/reveal behavior.

## Best Possible Solution

Keep this as the umbrella until maintainers split or land focused work for at-rest secret strategy, backup handling, diagnostic/raw sink policy, UI completeness, and regression coverage.

## Solution Assessment

No. Closing this or attempting one broad automated patch is not the best way to solve it; the maintainable path is maintainer-reviewed, focused changes for storage/key management, backup policy, diagnostic/raw sink policy, and UI coverage.

## Security Review

Status: needs_attention

Summary: Current main still has unresolved sensitive-data exposure surfaces in config persistence, backups, and some diagnostic persistence paths.

- **[medium] Plaintext config persistence remains:** `src/config/io.ts:2133`
  - body: The primary config write path serializes the stamped config with JSON.stringify and writes those bytes to disk; file mode 0600 reduces exposure but does not provide at-rest encryption or mandatory SecretRef storage.
  - confidence: 0.96
- **[medium] Backups preserve plaintext config bytes:** `src/config/backup-rotation.ts:120`
  - body: Backup maintenance copies the current config file directly to the .bak ring, and the test suite asserts a token-bearing JSON object is copied unchanged.
  - confidence: 0.96
- **[low] Diagnostic cache trace can persist raw text fields:** `src/agents/cache-trace.ts:200`
  - body: cache-trace writes prompt, note, and error fields without the shared redaction gate, even though other structured payload fields are sanitized.
  - confidence: 0.86
