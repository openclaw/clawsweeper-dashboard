## Summary

Keep open: current main still replaces the stored session entry during `upsertSessionEntry`, and the linked fix PR is open but not merged.

## What This Changes

The issue reports that replacement session upserts can erase existing non-empty `label` and `displayName` fields from persistent session records.

## Best Possible Solution

Current main should preserve existing non-empty human session titles during replacement upserts while keeping explicit rename and clear behavior in the validated session patch path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: store an entry with non-empty `label` and `displayName`, then call `upsertSessionEntry` for the same key with replacement metadata that omits or empties those fields.

## Solution Assessment

Yes. The narrow fix belongs at the session-store upsert boundary, with explicit renames still going through the validated session patch/update paths.
