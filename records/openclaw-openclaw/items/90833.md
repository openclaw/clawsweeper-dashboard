## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use the existing session label limit:** `ui/src/ui/app-render.ts:626`
  - body: The sidebar editor hard-caps and slices labels at 120 characters, but `SessionLabelString` and the existing Sessions-page editor allow the established 512-character contract. Editing a valid longer label through this new path can silently shorten persisted session state, so the sidebar should use the same contract.
  - confidence: 0.94
- **[P2] Keep data-session-key on the clickable link:** `ui/src/ui/app-render.ts:704`
  - body: Moving `data-session-key` onto the wrapper means current browser coverage that clicks `a.sidebar-recent-session[data-session-key=...]` no longer finds the clickable row. Keep the attribute on the `<a>` as well, or update the affected selectors and proof in the same change.
  - confidence: 0.9
- **[P2] Include the session name in the rename button label:** `ui/src/ui/app-render.ts:748-749`
  - body: Every icon-only rename button exposes the same accessible name, `Rename session`, so screen reader users tabbing through multiple recent sessions cannot tell which session will be edited. Include the row label or key in the button's accessible name and add focused coverage.
  - confidence: 0.86
