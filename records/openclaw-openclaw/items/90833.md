- **[P1] Use the existing session label source:** `ui/src/ui/session-display.ts:100-102`
  - body: This makes a browser-local rename win before row.label/displayName even though current main already has sessions.patch label support and a Sessions-page editor.   - confidence: 0.93
- **[P2] Fix the typed session rows in the new test:** `ui/src/ui/session-rename-store.node.test.ts:96`
  - body: `resolveSessionDisplayName` expects a `GatewaySessionRow`, but the new tests pass `{ key }` without the required `kind` and `updatedAt` fields, which is exactly why `check-test-types` fails. Use the same row helper shape as adjacent session display tests or provide the required fields at each call site.
  - confidence: 0.95
- **[P2] Prevent Escape from saving the draft:** `ui/src/ui/app-render.ts:681`
  - body: Escape clears the inline edit state, but the input's blur handler still calls `commitRename`. When Escape causes the input to blur or be removed, the draft can be saved instead of canceled; track the cancel path or skip the blur commit after Escape.
  - confidence: 0.86
- **[P2] Keep the rename button outside the session link:** `ui/src/ui/app-render.ts:729-733`
  - body: The session row remains an anchor while the new rename control is a nested button, putting one interactive control inside another. Split the row into a wrapper with separate link and button targets to avoid invalid markup and ambiguous keyboard/screen-reader behavior.
  - confidence: 0.82
- **[P3] Localize the new rename labels:** `ui/src/ui/app-render.ts:667`
  - body: The new input and button add hardcoded `Rename session` text in the Control UI. The scoped UI guide makes the English locale bundle the source of truth, so add these strings through i18n instead of embedding them directly.
  - confidence: 0.88
