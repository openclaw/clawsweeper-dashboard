Overall confidence: 0.9

Full review comments:

- **[P1] Use the existing session label source:** `ui/src/ui/session-display.ts:100-102`
  - body: This makes a browser-local rename win before `row.label`/`displayName`, but current main already has `sessions.patch` label support plus a Sessions-page editor. That creates a second source of truth: sidebar labels can disagree with the Sessions page, other browsers, and gateway clients. Rework the sidebar action to call the existing label patch path and remove the localStorage override.
  - confidence: 0.92
- **[P2] Prevent Escape from committing the draft:** `ui/src/ui/app-render.ts:681`
  - body: Escape clears the inline edit state, but the same input's blur handler always calls `commitRename`. When the input is removed or loses focus after Escape, the draft can still be saved instead of canceled. Track the cancel path or skip the blur commit after Escape.
  - confidence: 0.86
- **[P2] Keep the rename button out of the link:** `ui/src/ui/app-render.ts:729-733`
  - body: The session row remains an `<a>` while the new rename control is a nested `<button>`, putting one interactive control inside another. That produces invalid markup and ambiguous keyboard/screen-reader behavior; split the row into a wrapper with separate link and button targets.
  - confidence: 0.8
- **[P3] Localize the new rename labels:** `ui/src/ui/app-render.ts:667`
  - body: The new input and button add hardcoded `Rename session` text in the Control UI. The scoped UI guide makes English locale keys the source of truth, so add the string through the i18n bundle instead of embedding it directly.
  - confidence: 0.88
