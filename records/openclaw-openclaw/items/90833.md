Overall correctness: patch is incorrect

Full review comments:

- **[P2] Use the session label contract limit:** `ui/src/ui/app-render.ts:626`
  - body: The new sidebar editor hard-caps and slices labels at 120 characters, but `SessionLabelString` and `parseSessionLabel` allow 512 and the existing Sessions-page editor sends the normalized value without this sidebar-only truncation. A user renaming from the sidebar can silently lose text that the server and existing UI would accept, so the limit needs to match the established session-label contract.
  - confidence: 0.91
- **[P2] Keep the session key on the clickable link:** `ui/src/ui/app-render.ts:704`
  - body: Moving `data-session-key` onto the wrapper means existing browser coverage and automation that click `a.sidebar-recent-session[data-session-key=...]` no longer find the clickable row. Please keep the attribute on the `<a>` as well, or update the affected selectors in the same change, so session switching proof does not break on this DOM reshuffle.
  - confidence: 0.84
