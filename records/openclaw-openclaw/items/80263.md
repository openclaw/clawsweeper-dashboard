## Summary

Keep open. Current main and the latest release still use `.deleted.*` for retained transcripts and still reuse `pruneAfter` for both visible-entry pruning and `.deleted.*` archive cleanup; the subagent docs are now clearer, but the central config/API terminology change remains a product decision.

## What This Changes

The issue asks OpenClaw to clarify session lifecycle terminology by renaming `.deleted.*`, splitting retention settings, and documenting cron/subagent archive semantics.

## Best Possible Solution

Define a compatibility-preserving session lifecycle contract: keep reading legacy suffixes, choose approved non-destructive archive terminology, split visible-entry and retained-transcript retention if desired, and update cron/subagent docs plus tests together.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.12` use `pruneAfterMs` for stale session entry pruning and `.deleted.*` archive cleanup, and tests assert the rename-then-later-remove lifecycle.

## Solution Assessment

Unclear until maintainers choose the retention contract. The likely direction is an additive, compatibility-preserving config/API and docs update rather than a rename-only patch.
