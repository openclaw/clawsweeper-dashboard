## Summary

Keep open: current main still uses `.deleted.*` retained transcript archives and still reuses one retention window for both visible-entry pruning and retained deleted-archive cleanup. The requested fix changes public lifecycle/config terminology and compatibility policy, so it needs maintainer direction rather than conservative cleanup.

## What This Changes

Request to rename retained transcript archive terminology, split retention controls, and document session, cron, and subagent lifecycle transitions.

## Best Possible Solution

Define a compatibility-preserving lifecycle contract: keep recognizing legacy `.deleted.*` and `.reset.*` archives, choose non-destructive terminology and separate visible-entry versus retained-transcript retention knobs if approved, then update schema, help, docs, cleanup behavior, cron behavior, and regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a public config/API semantics request. Source, docs, and tests provide high-confidence proof that current main still uses `.deleted.*` archives and shared retention windows as described.

## Solution Assessment

Unclear until maintainers choose terminology and compatibility policy. The requested split is plausible, but the maintainable path should be additive and continue recognizing existing archive suffixes.
