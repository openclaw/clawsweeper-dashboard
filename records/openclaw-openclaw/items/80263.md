## Summary

Keep open: current main still writes retained transcripts as `.deleted.*` and uses the same retention window for visible-entry pruning and later deleted-archive removal. The related open issues cover neighboring archive naming and cleanup concerns, but this issue’s requested config/API semantics split still needs maintainer direction.

## What This Changes

The issue requests clearer session-retention terminology by renaming retained `.deleted.*` transcripts, splitting retention settings, and documenting session, cron, and subagent lifecycle transitions.

## Best Possible Solution

Define an additive, legacy-aware session retention lifecycle contract, then update schema/types, cleanup behavior, cron behavior, docs, help text, and regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a public config/API semantics request. Source inspection gives high-confidence proof that current main still uses `.deleted.*` archives and one shared retention window as described.

## Solution Assessment

Unclear until maintainers choose the compatibility policy and terminology. The requested direction is plausible, but the safer implementation would be additive and continue recognizing existing archive suffixes.
