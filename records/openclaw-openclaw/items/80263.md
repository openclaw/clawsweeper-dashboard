## Summary

Keep open: current main still uses `.deleted.*` retained transcript archives and still uses the same retention window for visible session entry pruning and later physical cleanup of deleted archives. The requested change affects public config/API terminology, retention policy, and compatibility with existing archive consumers, so it needs maintainer direction rather than cleanup closure.

## What This Changes

Request to rename or clarify retained session transcript archive states and split session retention settings so visible-session pruning and retained-transcript deletion are controlled separately.

## Best Possible Solution

Define an additive, compatibility-preserving session lifecycle contract that keeps reading legacy suffixes, separates visible-entry and retained-transcript retention only after maintainer approval, and updates schema, docs, cron behavior, cleanup behavior, and tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a public config/API semantics request. Source, docs, tests, and live related-item search give high-confidence proof that current main still has the reported `.deleted.*` naming and shared retention-window behavior.

## Solution Assessment

Unclear until maintainers choose the compatibility and terminology policy. The requested direction is plausible, but the safest implementation should be additive and preserve recognition of existing `.deleted.*` archives.
