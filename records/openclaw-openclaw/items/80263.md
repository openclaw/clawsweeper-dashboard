## Summary

Keep open: current main still uses `.deleted.*` retained transcript archives and one retention window for both visible-entry pruning and deleted-archive cleanup. Some docs now describe parts of the behavior, but the requested config/API terminology split remains a product and compatibility decision, and the related open issues do not fully supersede this one.

## What This Changes

The issue requests clearer session-retention terminology and configuration semantics around retained transcript archives, `pruneAfter`, cron session retention, and subagent archive timers.

## Best Possible Solution

Define an additive lifecycle contract that preserves legacy `.deleted.*` recognition, separates visible-session and retained-transcript retention if approved, and updates schema, config help, docs, cron cleanup, session cleanup, and regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a public config/API semantics request. Source and tests on current main confirm the reported `.deleted.*` archive naming and shared retention-window behavior still exist.

## Solution Assessment

Unclear until maintainers choose the terminology and compatibility policy. The requested direction is plausible, but the maintainable solution should be additive and continue recognizing existing archive suffixes.
