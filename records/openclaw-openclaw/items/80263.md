## Summary

Keep open: current main still uses `.deleted.*` as a retained transcript archive suffix and still reuses one retention window for both visible-entry pruning and deleted-archive hard deletion. The requested fix changes public config/API semantics and archive compatibility policy, so it needs maintainer direction rather than conservative cleanup.

## What This Changes

The issue asks OpenClaw to clarify session retention naming and split ambiguous retention settings around `.deleted.*` transcript archives.

## Best Possible Solution

Define an additive retention lifecycle contract that preserves legacy archive recognition, separates visible-session and retained-transcript retention only if maintainers approve, and updates schema, runtime cleanup, cron behavior, docs, and regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a public config and lifecycle semantics request. Current source, docs, and tests provide high-confidence proof that main still behaves as described.

## Solution Assessment

Unclear until maintainers choose the compatibility policy and terminology. The requested direction is plausible, but the maintainable solution should be additive and continue recognizing existing `.deleted.*` archives.
