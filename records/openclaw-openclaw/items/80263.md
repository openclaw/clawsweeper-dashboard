## Summary

Keep open: current main still uses `.deleted.*` retained transcript archives and reuses the same retention window for both session-store pruning and hard cleanup, while the remaining request asks for public config/API terminology that needs maintainer direction.

## What This Changes

Request clearer session-retention lifecycle semantics by renaming or differentiating retained transcript archives, splitting retention knobs, and documenting cron/subagent behavior.

## Best Possible Solution

Define an additive retention lifecycle contract that preserves legacy `.deleted.*` recognition, separates visible-session and retained-transcript retention if approved, and updates schema, help, docs, cleanup behavior, and regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config semantics request. Current source, docs, and tests confirm the reported lifecycle and naming behavior exists on main.

## Solution Assessment

Unclear until maintainers choose the public compatibility policy. The requested direction is plausible, but the maintainable fix should be additive and continue recognizing existing `.deleted.*` archives.
