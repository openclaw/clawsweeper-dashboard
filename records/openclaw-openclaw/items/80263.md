## Summary

Keep open: current main still uses `.deleted.*` archive names and still reuses one retention window for both removing visible session-store entries and deleting retained deleted-archive transcripts. Some docs now clarify cron and subagent behavior, but the requested config/API terminology and compatibility policy still need maintainer direction.

## What This Changes

Request clarifies session retention by renaming `.deleted.*`, splitting `pruneAfter` into explicit visible-entry and retained-transcript windows, and documenting cron/subagent lifecycle semantics.

## Best Possible Solution

Define an additive session-retention lifecycle contract that preserves legacy `.deleted.*` recognition, adds explicit retained-transcript retention only if maintainers approve it, and updates schema, help, docs, cleanup behavior, cron behavior, and regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a public config/API terminology request. Source and docs inspection give high-confidence proof that current main still has the reported `.deleted.*` naming and shared retention-window behavior.

## Solution Assessment

Unclear until maintainers choose the compatibility policy and terminology. The requested direction is plausible, but the maintainable path should be additive and continue recognizing existing archive suffixes.
