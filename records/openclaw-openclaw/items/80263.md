## Summary

Keep open: current main still uses `.deleted.*` retained transcript archives and reuses the same retention window for both removing visible session-store entries and deleting retained deleted-archive files. This is a public retention/config semantics request, and the remaining decision needs maintainer compatibility and naming direction rather than cleanup closure.

## What This Changes

Request to make session retention terminology and knobs clearer by separating visible-session retention from retained-transcript deletion and avoiding `.deleted.*` for retained archives.

## Best Possible Solution

Define an additive retention lifecycle contract that preserves legacy `.deleted.*` recognition, separates visible-session and retained-transcript retention if maintainers approve, and updates schema, help, docs, cron cleanup, session cleanup, and regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a public config and lifecycle semantics request. Current-main source and tests directly confirm the reported `.deleted.*` naming and shared-retention behavior.

## Solution Assessment

Unclear until maintainers choose the compatibility and terminology policy. The requested direction is plausible, but the maintainable path should be additive and keep recognizing existing archive suffixes.
