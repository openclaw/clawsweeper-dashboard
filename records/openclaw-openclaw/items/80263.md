## Summary

Keep open: current main still has the reported retention ambiguity, and resolving it requires a user-facing retention terminology/config contract decision rather than conservative cleanup closure.

## What This Changes

Request clearer session-retention terminology by renaming retained `.deleted.*` archives, splitting retention knobs, and documenting session, cron, and subagent lifecycle transitions.

## Best Possible Solution

Define a backward-compatible retention lifecycle contract, keep reading legacy `.deleted.*` archives, add clearer terminology and explicit retention knobs if maintainers choose that direction, and update config schema/help, docs, CLI behavior, and tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config/docs semantics request. Current-main source and tests do confirm the described behavior: visible entries are pruned, transcripts are retained as `.deleted.*`, and later cleanup reuses the same retention window.

## Solution Assessment

Unclear until maintainers choose the compatibility policy. The requested direction is plausible, but the safest maintainable solution is additive terminology/config work that preserves legacy archive recognition.
