## Summary

Keep open: current main still has the intentional but confusing retention semantics described, and changing persistent archive terminology plus retention config is a user-facing compatibility/product decision rather than an auto-closeable bug fix.

## What This Changes

Request clearer session-retention terminology by renaming retained transcript suffixes, splitting visible-entry and retained-transcript retention settings, and documenting session, cron, and subagent lifecycle transitions.

## Best Possible Solution

Define an additive session-lifecycle contract that keeps reading legacy `.deleted.*` and `.reset.*` archives, introduces clearer retained-transcript terminology and split retention knobs only with backward compatibility, and updates schema, CLI help, docs, cron, subagent, and regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config/docs semantics request. Source inspection and existing tests confirm the described current state transitions on main.

## Solution Assessment

Unclear until maintainers choose the compatibility policy. The requested direction is plausible, but the safest implementation is additive and backward compatible rather than a direct suffix or config replacement.
