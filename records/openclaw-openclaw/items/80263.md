## Summary

Keep open: current `origin/main` still uses `.deleted.*` retained transcript archives and one retention window for both pruning visible session entries and deleting retained transcript files. The remaining work changes public config, lifecycle terminology, and compatibility policy, so it needs maintainer direction rather than cleanup closure.

## What This Changes

Clarify and revise session retention lifecycle semantics by renaming retained `.deleted.*` transcript archives, splitting the double-duty `pruneAfter` retention behavior, and documenting cron/subagent retention states.

## Best Possible Solution

Define an additive session-retention lifecycle contract that keeps recognizing legacy `.deleted.*` archives, separates visible-session and retained-transcript retention only if maintainers approve, and updates schema, help, docs, cron cleanup, session maintenance, and regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a public config and lifecycle-semantics request. Source inspection of `origin/main` gives high-confidence proof that the described `.deleted.*` archive naming and shared retention window still exist.

## Solution Assessment

Unclear until maintainers choose the compatibility and terminology policy. The safest direction is additive: preserve legacy archive recognition while introducing any new suffixes or split retention knobs through schema, docs, and tests together.
