## Summary

Keep open: current main still has the reported `.deleted.*` archive naming and still uses one retention window for both visible session pruning and retained transcript deletion. This is a public lifecycle/config semantics request, not a narrow bug fix, and it overlaps related open retention-policy issues without being fully superseded by them.

## What This Changes

Request to clarify session-retention lifecycle terminology by replacing misleading `.deleted.*` archive naming, splitting `pruneAfter` semantics, and documenting cron/subagent retention behavior.

## Best Possible Solution

Define an additive lifecycle contract that preserves legacy archive recognition, chooses maintainer-approved terminology and retention split knobs, then updates schema, docs, cron/session cleanup behavior, subagent docs, and regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a public config/API semantics request. Source, docs, and tests give a high-confidence path showing current main still uses `.deleted.*` archives and shared retention windows as described.

## Solution Assessment

Unclear until maintainers choose the compatibility and terminology policy. The requested direction is plausible, but the maintainable solution should be additive and continue recognizing existing archive suffixes.
