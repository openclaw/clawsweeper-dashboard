- `impact:session-state`: The issue concerns persisted session entries, transcript archive files, cron run sessions, and subagent session lifecycle state.

## Summary

Keep open: current main and the latest release still use `.jsonl.deleted.*` for retained transcripts and still reuse `pruneAfter` as both the visible-entry prune window and the deleted-archive physical-removal window. Related open issues cover naming/data-loss and broader retention rules, but this issue’s specific config/docs semantics are not fully owned by a safe canonical item yet.

## What This Changes

The issue asks OpenClaw to make session lifecycle terminology and retention knobs clearer for `.deleted.*` archives, `pruneAfter`, cron session retention, and subagent auto-archive behavior.

## Best Possible Solution

Settle one durable session archive/retention contract, then update core config/schema/help/docs/tests so visible-entry retention, retained-transcript retention, and physical deletion are named and migrated explicitly.

## Reproduction Assessment

Yes, at source level: current main still creates `.jsonl.deleted.*` archives and still uses `pruneAfter` for both entry pruning and deleted-archive cleanup. I did not run a live Gateway repro because this review is read-only.

## Solution Assessment

Unclear: the direction is valid, but the exact rename and config split are public compatibility decisions. The best fix should be designed with the broader archive naming and retention-rule issues rather than implemented as a narrow rename-only patch.
