## Summary

Keep open: current main still uses retained `.deleted.*` transcript archives and reuses the same retention window for entry pruning and deleted-archive cleanup, while the requested rename/split is a user-facing config and compatibility decision.

## What This Changes

Clarify session-retention terminology and configuration around retained `.deleted.*` transcripts, cron retention, and subagent archive timers.

## Best Possible Solution

Design an additive retention lifecycle contract that preserves existing `.deleted.*` compatibility, introduces clearer retained-transcript terminology if maintainers choose it, and updates config schema, CLI/docs, cron behavior, and tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/docs/config semantics request. Source and tests confirm the reported current behavior on main: entries are pruned, transcripts are retained as `.deleted.*`, and cleanup reuses the same retention window.

## Solution Assessment

Unclear until maintainers choose the compatibility policy. The requested direction is plausible, but the safest implementation would be additive documentation and new knobs while continuing to recognize existing `.deleted.*` archives.
