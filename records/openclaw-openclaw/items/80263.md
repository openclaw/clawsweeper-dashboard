## Summary

Keep open: current main still has the reported `.deleted.*` archive naming and double-duty retention semantics, while related open issues cover only parts of the suffix and cleanup policy. This needs maintainer API/product direction rather than conservative cleanup closure.

## What This Changes

The issue asks OpenClaw to clarify session-retention terminology, split ambiguous retention settings, and document lifecycle behavior for deleted archives, cron sessions, and subagent archive timers.

## Best Possible Solution

Define an additive lifecycle contract that preserves legacy `.deleted.*` compatibility, introduces clearer retained-transcript terminology and separate retention knobs if maintainers want them, and updates config schema/help, docs, CLI behavior, and regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config/docs semantics request. Source and tests do provide high-confidence confirmation that current main has the described `.deleted.*` archives and shared retention windows.

## Solution Assessment

Unclear until maintainers choose the compatibility policy. The requested direction is plausible, but the maintainable path should be additive and coordinated with existing archive-preservation and cleanup issues.
