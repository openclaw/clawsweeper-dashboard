## Summary

Keep open: current main intentionally has the confusing retention semantics described, and changing on-disk suffixes plus config shape is a user-facing compatibility/product decision rather than an auto-closeable bug fix.

## What This Changes

The issue requests clearer session-retention terminology and separate retention knobs for visible session entries, retained deleted transcripts, cron run sessions, and subagent auto-archive behavior.

## Best Possible Solution

Design an additive retention lifecycle contract that preserves existing `.deleted.*` compatibility, introduces clearer deleted-archive retention terminology if maintainers want it, and updates CLI/config/docs/tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/docs/config semantics request. Source and tests do confirm the described current behavior: visible entries are pruned, transcripts are renamed to `.deleted.*`, and later cleanup reuses the same retention window.

## Solution Assessment

Unclear until maintainers choose the compatibility policy. The requested direction is plausible, but the safest implementation would be additive documentation and new knobs while continuing to recognize existing `.deleted.*` archives.
