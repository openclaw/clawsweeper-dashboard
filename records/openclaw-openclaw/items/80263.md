## Summary

Keep open: current main still uses `.deleted.*` retained transcript archives and still reuses one retention window for both removing visible session-store rows and physically removing retained deleted archives. The requested fix changes public config terminology, persistent filename compatibility, and retention policy, so it needs maintainer product direction rather than cleanup closure or an automated bug-fix PR.

## What This Changes

Request to rename retained session transcript archive semantics, split session retention knobs, and document session, cron, and subagent lifecycle states clearly.

## Best Possible Solution

Define an additive, legacy-compatible session retention lifecycle contract that separates visible-row retention from retained-transcript deletion if approved, then update schema, runtime cleanup, cron behavior, docs, and regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a config/API semantics request. Source, docs, and tests provide high-confidence proof that current main still uses `.deleted.*` archives and shared retention windows as described.

## Solution Assessment

Unclear until maintainers choose compatibility policy and terminology. The requested direction is plausible, but the maintainable path should be additive and continue recognizing existing archive suffixes.
