## Summary

Keep open: current main still uses `.deleted.*` transcript archive names and still reuses one retention window for both removing visible session-store entries and deleting retained transcript archives. The requested fix changes persistent naming, public config semantics, cron retention wording, and compatibility policy, so it needs maintainer product direction rather than conservative cleanup.

## What This Changes

The issue asks OpenClaw to make session-retention lifecycle names and config knobs clearer by renaming retained `.deleted.*` archives, splitting `pruneAfter`-style retention phases, and documenting cron/subagent archive semantics.

## Best Possible Solution

Define an additive session-retention lifecycle contract: keep reading legacy `.deleted.*` archives, choose maintainer-approved non-destructive terminology for newly retained transcripts, split visible-session and retained-transcript retention only if approved, and update schema/help/docs/cron/subagent behavior and regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a public config and lifecycle semantics request. Source, docs, and tests provide a high-confidence current-main proof of the described `.deleted.*` naming and shared retention-window behavior.

## Solution Assessment

Unclear until maintainers choose the compatibility and retention policy. The requested direction is plausible, but the maintainable path should be additive and preserve legacy archive recognition.
