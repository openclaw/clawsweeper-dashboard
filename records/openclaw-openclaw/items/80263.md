## Summary

Keep open: current main still has the reported retention semantics, and changing the archive suffix plus splitting retention knobs is a user-facing config, docs, and compatibility decision rather than a conservative cleanup close.

## What This Changes

The issue asks OpenClaw to clarify session-retention state names, split ambiguous retention settings, and document subagent and cron retention lifecycle behavior.

## Best Possible Solution

Define an additive retention lifecycle contract that preserves existing `.deleted.*` compatibility, adds clearer archive/detach terminology and separate retention controls only after maintainer approval, and updates config schema, CLI help, docs, and regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/docs/config semantics request. Source and tests confirm the described current-main behavior with high confidence.

## Solution Assessment

Unclear until maintainers choose the compatibility policy. The safest direction is additive documentation and new knobs while continuing to recognize existing `.deleted.*` retained transcripts.
