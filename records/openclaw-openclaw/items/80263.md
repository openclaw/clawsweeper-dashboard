## Summary

Keep open: current main still writes retained transcripts as `.deleted.*`, still uses `session.maintenance.pruneAfter` for both visible-entry pruning and deleted-archive cleanup, and the requested rename/split is a public config and compatibility policy decision. Related open issues overlap, especially https://github.com/openclaw/openclaw/issues/56609, but they do not fully supersede this issue’s specific `pruneAfter` double-duty concern.

## What This Changes

Request clarifies session retention lifecycle semantics by renaming retained `.deleted.*` transcripts, splitting `pruneAfter` retention duties, and documenting cron and subagent archive behavior.

## Best Possible Solution

Define an additive retention lifecycle contract that preserves legacy `.deleted.*` recognition, separates visible-session and retained-transcript retention if maintainers approve, and updates schema, help, docs, cron behavior, cleanup behavior, and tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a public config/API semantics request. Source and tests provide high-confidence proof that current main still uses `.deleted.*` archives and a shared retention window for the described transitions.

## Solution Assessment

Unclear until maintainers choose the compatibility policy and terminology. The requested direction is plausible, but the maintainable path should be additive and continue recognizing existing archive suffixes.
