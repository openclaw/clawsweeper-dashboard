## Summary

Keep open. Current main still uses `.deleted.*` for retained transcripts and still uses `session.maintenance.pruneAfter` as both the visible-entry stale cutoff and the deleted-archive hard-delete retention window, while only parts of the requested documentation are already present.

## What This Changes

The issue asks OpenClaw to clarify session retention semantics by renaming retained `.deleted.*` transcript archives, splitting retention knobs, and documenting ordinary, subagent, and cron lifecycle transitions.

## Best Possible Solution

Define a compatibility-preserving session archive lifecycle contract: keep reading legacy suffixes, choose the future archive terminology and retention knobs, then update cleanup behavior, config schema/help, docs, and regression tests together.

## Reproduction Assessment

Yes. Source and tests on current main show `.deleted.<timestamp>` archives are still created and later physically removed using the same `pruneAfter` window that prunes visible session-store rows.

## Solution Assessment

Unclear as an implementation proposal because the requested rename and config split need maintainer decisions on naming, defaults, migration, and legacy compatibility. A docs-only patch would help but would not fully solve the misleading API shape.
