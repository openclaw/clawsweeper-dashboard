## Summary

Keep open: current main still has the retention semantics described, and changing the `.deleted.*` on-disk contract plus splitting retention settings is a compatibility/product decision rather than a stale or already-fixed bug.

## What This Changes

The issue requests clearer session-retention lifecycle terminology, a non-destructive archive suffix, split retention knobs, and docs for session, cron, and subagent retention behavior.

## Best Possible Solution

Design an additive retention lifecycle contract that preserves existing `.deleted.*` compatibility, introduces clearer archive/delete terminology if approved, and updates config types, schema help, docs, and regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config/docs semantics request. Source and tests do confirm the described current behavior on current main.

## Solution Assessment

Unclear until maintainers choose the compatibility policy. The requested direction is plausible, but the narrow maintainable path should be additive and keep recognizing existing `.deleted.*` archives.
