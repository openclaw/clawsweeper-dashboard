## Summary

Keep open: current main still has the session-retention semantics described, and the requested suffix/config/docs changes require a user-facing compatibility and product decision rather than conservative cleanup.

## What This Changes

Request clearer session-retention lifecycle terminology, non-destructive archive naming, split retention knobs, and docs for cron/subagent cleanup.

## Best Possible Solution

Define an additive retention lifecycle contract that keeps reading legacy `.deleted.*` archives, introduces clearer archive terminology and separate retained-transcript retention only if approved, and updates config schema/help, docs, CLI behavior, cron, subagents, and regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config/docs semantics request. Source and tests confirm the described current-main behavior: visible entries are pruned, transcripts are renamed to `.deleted.*`, and later cleanup reuses the same retention window.

## Solution Assessment

Unclear until maintainers choose the compatibility policy. The narrow maintainable direction is additive documentation and new knobs while preserving recognition of existing `.deleted.*` archives.
