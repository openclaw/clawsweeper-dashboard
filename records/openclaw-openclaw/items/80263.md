## Summary

Keep open: current main still has the reported `.deleted.*` archive naming and still reuses one retention window for both removing visible session-store entries and deleting retained transcript archives. The remaining work changes public config/API semantics and compatibility policy, so this should not be swept closed or auto-fixed without maintainer direction.

## What This Changes

The issue requests clearer session-retention lifecycle terminology, separate retention knobs for visible sessions versus retained transcripts, and documentation for subagent and cron retention semantics.

## Best Possible Solution

Define an additive retention lifecycle contract that preserves legacy `.deleted.*` recognition, separates visible-session and retained-transcript retention if maintainers approve, and updates schema, help, docs, cleanup behavior, cron behavior, and regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a public config/API semantics request. Source, docs, and tests provide high-confidence proof that current main still uses `.deleted.*` archives and shared retention windows as described.

## Solution Assessment

Unclear until maintainers choose the compatibility policy and terminology. The requested direction is plausible, but the maintainable path should be additive and continue recognizing existing archive suffixes.
