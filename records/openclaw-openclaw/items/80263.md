## Summary

Keep open: current main and the latest release still use `.deleted.*` archives and reuse one retention window for both visible-session pruning and retained-transcript deletion. Related open issues cover adjacent archive naming and garbage collection, but this distinct public config/API semantics choice still needs maintainer direction.

## What This Changes

Request to rename misleading session archive terminology, split the dual `pruneAfter` retention roles, and document session, cron, and subagent lifecycle semantics.

## Best Possible Solution

Define an additive retention lifecycle contract that preserves legacy archive recognition, separates visible-session and retained-transcript retention only if maintainers approve, and updates schema, help, docs, runtime cleanup, and regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a config/product semantics request. Source, tests, docs, and latest-release inspection confirm the current behavior described by the issue.

## Solution Assessment

Unclear until maintainers choose the public compatibility policy. The requested direction is plausible, but the safest implementation should be additive and reconcile the open archive-preservation and cleanup issues first.
