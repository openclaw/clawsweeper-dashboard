## Summary

Keep open: the Android/Termux shell-wrapper bug still exists on current main, but this PR is not merge-ready because it only changes shell selection for a literal `android` platform while the headless node-host path currently maps Node's Android runtime platform to `unknown`. The branch is also conflicting and has only unit-test proof, not a real Termux node-host run.

## What This Changes

The branch adds a Termux shell path for Android node shell argv construction, focused unit coverage, and two docs notes about Android/Termux `system.run` behavior.

## Best Possible Solution

Update the fix to carry Android node-host platform metadata through the runner, choose the Termux shell in shell argv construction, cover both paths with focused tests, and add redacted real Termux `exec host=node` proof.

## Reproduction Assessment

Yes for a source reproduction, but not a live Termux run. Current main maps Android node-host runtime platform values to `unknown`, then passes that platform into `buildNodeShellCommand`, which falls through to `/bin/sh -lc`.

## Solution Assessment

No, this PR is only a plausible partial fix. The best fix must also update the node-host Android platform/deviceFamily mapping before shell selection and prove the real Termux execution path.
