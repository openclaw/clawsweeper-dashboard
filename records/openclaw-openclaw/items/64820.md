## Summary

Keep this PR open. The linked Feishu TDZ bug still appears valid on current `main`, but the PR branch is stale and not mergeable as-is, and the proposed patch drops current parser fields and bypasses the repo’s runtime-boundary lazy-import pattern.

## What This Changes

This PR extracts Feishu message parsing into `message-parser.ts`, lazy-loads Feishu message/card handlers from the account monitor, updates sequential-key imports, and adds parser/import-graph tests.

## Best Possible Solution

Land a current-main Feishu fix that preserves parser semantics, breaks the eager bot/card/menu handler import graph through real runtime boundaries, and proves the bundle does not emit `[INEFFECTIVE_DYNAMIC_IMPORT]`.

## Reproduction Assessment

Yes. The linked bug report gives concrete multi-bot Feishu group mention steps and logs for `ReferenceError: Cannot access 'utils_1' before initialization`, and current main still has the static import graph implicated by that report.

## Solution Assessment

No. The parser split and lazy loading are the right direction, but this branch needs a current-main rebase or replacement, runtime-boundary lazy imports, preserved reply-target fields, and less brittle source-path tests before it is the best fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve reply-target fields in the extracted parser:** `extensions/feishu/src/message-parser.ts:59-72`
  - body: Current main carries `replyTargetMessageId` and `suppressReplyTarget` into `FeishuMessageContext`, but the extracted parser builds `ctx` without those fields. A rebase that keeps this copy would drop reply threading and suppression metadata for Feishu replies and synthetic messages.
  - confidence: 0.9
- **[P2] Route lazy handlers through runtime boundaries:** `extensions/feishu/src/monitor.account.ts:407-418`
  - body: The new lazy loaders import `./bot.js` and `./card-action.js` directly. This bypasses the repo’s runtime-boundary pattern for lazy production imports, so the bundler can inline the same graph and leave the TDZ crash unresolved; use real `*.runtime.ts` boundaries and build-check for `[INEFFECTIVE_DYNAMIC_IMPORT]`.
  - confidence: 0.86
- **[P2] Resolve source paths with fileURLToPath:** `extensions/feishu/src/message-parser.test.ts:154`
  - body: The source-reading tests derive paths from `new URL(import.meta.url).pathname`, which leaves URL encoding and Windows path quirks in place. Use `fileURLToPath(import.meta.url)` before `path.dirname(...)` so these tests do not fail in supported paths containing spaces or on Windows.
  - confidence: 0.82
