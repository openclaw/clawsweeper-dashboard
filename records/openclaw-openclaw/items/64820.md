Full review comments:

- **[P2] Preserve reply-target fields in the extracted parser:** `extensions/feishu/src/message-parser.ts:61-75`
  - body: Current main carries `replyTargetMessageId` and `suppressReplyTarget` through `FeishuMessageContext`, and reply dispatch uses them when selecting the Feishu reply target. The extracted parser builds `ctx` without those fields, so rebasing this branch as-is would drop existing reply threading and suppression behavior.
  - confidence: 0.9
- **[P2] Route lazy handlers through runtime boundaries:** `extensions/feishu/src/monitor.account.ts:405-422`
  - body: The lazy loaders import `./bot.js` and `./card-action.js` directly. Repo policy requires a narrow `*.runtime.ts` lazy boundary plus build verification for dynamic imports, otherwise bundling can inline the handler graph back into the same chunk and leave the TDZ crash unresolved.
  - confidence: 0.86
- **[P2] Avoid caching bot.js before Feishu handler mocks:** `extensions/feishu/src/message-parser.test.ts:15`
  - body: The new test imports `./bot.js` at top level, while the unit lane is non-isolated by default and adjacent Feishu tests mock `./bot.js`. This can cache the real bot module before handler mocks are installed, making those tests exercise the wrong module instance.
  - confidence: 0.84
- **[P3] Resolve source paths with fileURLToPath:** `extensions/feishu/src/message-parser.test.ts:154-155`
  - body: The source-reading tests derive file paths from `new URL(import.meta.url).pathname`, which keeps URL encoding and Windows drive quirks. Use `fileURLToPath(import.meta.url)` before `path.dirname` so the tests work on supported paths with spaces and on Windows.
  - confidence: 0.82
