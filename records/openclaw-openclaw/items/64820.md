- **[P2] Preserve reply-target fields in the extracted parser:** `extensions/feishu/src/message-parser.ts:61-75`
  - body: Current main carries `replyTargetMessageId` and `suppressReplyTarget` through `FeishuMessageContext`, and reply dispatch uses them when selecting the Feishu reply target.   - body: The lazy loaders import `./bot.js` and `./card-action.js` directly. Repo policy requires a narrow `*.runtime.ts` lazy boundary plus build verification for dynamic imports; otherwise bundling can inline the handler graph back into the same chunk and leave the TDZ crash unresolved.
  - confidence: 0.86
- **[P2] Avoid caching bot.js before Feishu handler mocks:** `extensions/feishu/src/message-parser.test.ts:15`
  - body: The new test imports `./bot.js` at top level, while the Feishu/unit lanes run with non-isolated workers and adjacent tests mock `./bot.js`. CI already shows `monitor.bot-menu.test.ts` failing after this new test runs, consistent with the real bot module being cached before the mock can replace it.
  - confidence: 0.88
- **[P3] Resolve source paths with fileURLToPath:** `extensions/feishu/src/message-parser.test.ts:154-155`
  - body: The source-reading tests derive file paths from `new URL(import.meta.url).pathname`, which keeps URL encoding and Windows drive quirks. Use `fileURLToPath(import.meta.url)` before `path.dirname` so these tests work on supported paths with spaces and on Windows.
  - confidence: 0.82
