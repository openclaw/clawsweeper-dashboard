## Summary

Keep open: current main still has a packaged media-understanding-core image-ops artifact that imports `sharp`, while the root published package does not declare `sharp`, so the reported npm global install gap remains source-reproducible.

## What This Changes

Ensure packaged/global OpenClaw installs provide `sharp` for the bundled media-understanding-core image optimization path instead of relying on its nested manifest.

## Best Possible Solution

Make the packaged media-understanding-core image-ops dependency available to npm installs, preferably without reintroducing startup package-manager repair, and add package-level regression coverage for this manifestless bundled support package.

## Solution Assessment
