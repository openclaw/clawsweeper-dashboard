## Summary

Current main and shipped releases no longer have the reported bundled `media-understanding-core` Sharp dependency hole: the old bundled Sharp image-ops file was removed, media image operations now go through the root-installed Rastermill dependency, and the fix is contained in stable release v2026.5.26 and later.

## What This Changes

The issue reports that packaged/global installs can miss `sharp` because the bundled `media-understanding-core` runtime imported it while only the nested bundled package declared it.

## Best Possible Solution

Keep the current root-owned Rastermill media backend and close this Sharp packaging report; broader user messaging for unavailable image processors can stay tracked separately if still reproducible.

## Reproduction Assessment

Yes for the historical bug: the issue and confirmation comment describe a concrete global npm install path, and history shows the old bundled media image runtime imported Sharp from a nested package dependency. Current main does not reproduce that exact path because the bundled Sharp image ops file is gone.

## Solution Assessment

Yes. The maintainable fix is the current root-owned image backend: media operations now use the root-installed Rastermill package, while the nested bundled Sharp dependency and import path were removed.
