## Summary

Keep open: current main still packages the media-understanding image ops path while the root published package does not declare `sharp`, so npm global installs can miss the module unless another path installs it.

## What This Changes

The issue requests that the published OpenClaw package include `sharp` in the root dependency graph for bundled media image optimization.

## Best Possible Solution

Declare `sharp` in the root published package, preferably as an `optionalDependency`, keep the plugin-local declaration, and update ownership/package tests plus changelog so npm global installs have the native image optimizer available.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: package a global install from current main without another root `sharp`, then trigger a media resize path that loads `media-understanding-core/image-ops.js`. I did not run the install because this review is read-only.

## Solution Assessment

Yes, the narrowest maintainable fix is package metadata and ownership-test coverage, not a runtime workaround; the existing code already treats `sharp` as optional tooling for media image processing.
