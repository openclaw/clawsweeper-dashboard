## Summary

Keep open: current main and the latest release still lack root `sharp` package metadata while the bundled media image runtime imports `sharp`; the issue is already paired with an open fixing PR that should be reviewed or landed before closure.

## What This Changes

The issue requests making `sharp` available from the root published OpenClaw package so npm global installs can run bundled `media-understanding-core` image operations.

## Best Possible Solution

Land or supersede the linked root `optionalDependencies` package fix with aligned lockfile, changelog, and ownership checks, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a package/global install without root `sharp` loads `dist/extensions/media-understanding-core/image-ops.js`, which dynamically imports `sharp`; user comments also include real global-install failures on Linux and macOS.

## Solution Assessment

Yes. Declaring `sharp` as a root optional dependency is the narrow package-boundary fix for a bundled runtime import, while the existing in-limit fallback only mitigates part of the runtime symptom.
