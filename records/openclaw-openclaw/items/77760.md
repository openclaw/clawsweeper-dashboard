## Summary

Close. Current main and shipped v2026.5.26+ no longer import or declare Sharp in media-understanding-core; image operations now use the root-installed Rastermill dependency, so npm global installs no longer miss the reported nested dependency.

## What This Changes

The issue asks OpenClaw to avoid missing Sharp in npm global installs for bundled media-understanding image operations.

## Best Possible Solution

Keep the Rastermill-based root image backend and leave broader image-processor diagnostics in https://github.com/openclaw/openclaw/issues/73148 if still reproducible.

## Reproduction Assessment

Yes for the historical bug: the report and confirmation comment give concrete npm global install failures, and old source imported Sharp from a nested bundled package. Current main does not reproduce that path because the Sharp runtime file and nested dependency are gone.

## Solution Assessment

Yes. The current root-owned Rastermill backend is the narrow maintainable solution because it removes the nested bundled Sharp dependency instead of depending on npm to install a private extension dependency.
