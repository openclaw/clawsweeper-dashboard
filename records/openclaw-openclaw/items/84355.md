## Summary

## What This Changes

The PR updates the basic and full gateway agent list summaries to use `agents.list[].identity.name` when `agents.list[].name` is unset, with regression tests and a changelog entry.

## Best Possible Solution

Land this narrow gateway fallback after the normal automerge and CI gates, preserving explicit `agents.list[].name` precedence and regression coverage for both summary paths.

## Solution Assessment

Yes. Setting the top-level summary `name` from the configured name first and then the already-configured identity name is the narrow backend fix, and it keeps explicit names authoritative for existing configs.
