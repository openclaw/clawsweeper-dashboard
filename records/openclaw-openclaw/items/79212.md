## Summary

Keep open: current main still misclassifies Bedrock inference-profile-prefixed Cohere embedding model IDs, and the only matching fix PR was closed unmerged. The failure remains source-reproducible in the Amazon Bedrock plugin.

## What This Changes

Bedrock memory search should classify inference-profile-prefixed Cohere embedding model IDs for local catalog and family lookup while preserving the full model ID for Bedrock invocation.

## Best Possible Solution

Add a Bedrock plugin-local inference-profile prefix normalizer for `resolveSpec()` and `inferFamily()`, preserve `client.model` for invocation, and add focused regression coverage plus a changelog fix entry.

## Reproduction Assessment

Yes, by source inspection: current main classifies `global.cohere.embed-v4:0` against unprefixed catalog and family strings, misses Cohere v4, and falls back to Titan v1. I did not live-call AWS Bedrock in this review.

## Solution Assessment

Yes, the best fix is narrow and plugin-local: strip AWS system inference-profile prefixes only for local catalog/family lookup while preserving the full model ID for runtime invocation. No core API, config option, or product decision appears necessary.
