## Summary

Keep open: current main still has the raw-model lookup bug, and an open PR already references this issue with closing syntax, so the terminal outcome is to land or close that PR rather than closing the issue early.

## What This Changes

The report asks OpenClaw to classify Bedrock embedding inference-profile IDs like `global.cohere.embed-v4:0` using the bare model ID while preserving the prefixed ID for Bedrock invocation.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79220 if it remains correct and green, then close this issue as implemented by that merge.

## Reproduction Assessment

Yes. By source inspection, configuring Bedrock memory search with `global.cohere.embed-v4:0` makes current `resolveSpec()` and `inferFamily()` miss the Cohere v4 catalog/family path and fall back to Titan v1; I did not live-call AWS Bedrock in this review.

## Solution Assessment

Yes. The narrow maintainable solution is to strip known Bedrock inference-profile prefixes only for catalog/family/dimension lookup while preserving the full configured model ID for `InvokeModelCommand.modelId`.
