## Summary

Keep open: current main still does raw Bedrock embedding catalog/family matching against prefixed model IDs, so the reported Cohere v4 inference profile falls through to the Titan v1 request shape. The only linked fix PR is closed and unmerged.

## What This Changes

The issue requests Bedrock embedding lookup support for inference-profile-prefixed model IDs such as `global.cohere.embed-v4:0` while preserving the full configured ID for AWS invocation.

## Best Possible Solution

Implement plugin-local Bedrock embedding prefix normalization for catalog, family, and dimension lookup; preserve the configured model ID for `InvokeModelCommand.modelId`; add focused regression coverage and a changelog fix entry.

## Reproduction Assessment

Yes. By source inspection and a read-only expression mirroring current main, `global.cohere.embed-v4:0` misses the Cohere v4 catalog/family path and falls back to Titan v1; I did not live-call AWS Bedrock.

## Solution Assessment

Yes. The narrow maintainable fix is to strip AWS system inference-profile prefixes only for local Bedrock embedding lookup/family classification while preserving the full configured model for AWS invocation.
