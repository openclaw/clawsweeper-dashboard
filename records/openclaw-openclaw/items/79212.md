## Summary

Keep open: current main still appears to misclassify Bedrock inference-profile-prefixed embedding model IDs, and the only directly related fix PR was closed unmerged. This is a narrow source-backed Amazon Bedrock plugin bug with a clear replacement-fix path.

## What This Changes

The issue requests Bedrock embedding lookup and family detection to handle AWS inference-profile-prefixed model IDs such as `global.cohere.embed-v4:0` without changing the runtime `InvokeModelCommand.modelId`.

## Best Possible Solution

Apply a plugin-local Bedrock embedding normalization helper for AWS system inference-profile prefixes in catalog, dimension, and family lookup only, preserving the original configured model ID for invocation and runtime identity.

## Reproduction Assessment

Yes, by source inspection: configure Bedrock memory search with `global.cohere.embed-v4:0` and current `resolveSpec()`/`inferFamily()` will miss the Cohere v4 catalog/family path and fall back to Titan v1. I did not live-call AWS Bedrock in this review.

## Solution Assessment

Yes, the narrow maintainable fix is to strip AWS system inference-profile prefixes only for local Bedrock embedding lookup and family classification while preserving `InvokeModelCommand.modelId`. The closed related PR is useful source work but is not available to merge as-is because it was closed unmerged.
