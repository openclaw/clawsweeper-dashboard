## Summary

Keep open: current main still has the Bedrock embedding prefix bug, and the only linked fix PR was closed unmerged. The issue is source-reproducible and narrow enough for a focused replacement/fix PR.

## What This Changes

The issue asks OpenClaw to classify Bedrock inference-profile-prefixed embedding model IDs such as `global.cohere.embed-v4:0` using the bare model ID while preserving the full configured ID for Bedrock invocation.

## Best Possible Solution

Add a Bedrock embedding-local helper that strips AWS system inference-profile prefixes for `resolveSpec()` and `inferFamily()` only, keep `InvokeModelCommand.modelId` unchanged, and add regression coverage for prefixed Cohere v4 IDs.

## Reproduction Assessment

Yes, by source inspection: configuring Bedrock memory search with `global.cohere.embed-v4:0` still makes current main miss the Cohere v4 catalog/family checks and fall back to Titan v1. I did not run a live AWS Bedrock call in this review.

## Solution Assessment

Yes, the narrow maintainable fix is to strip AWS system inference-profile prefixes only for local catalog/family/dimension lookup while preserving the full configured model ID for Bedrock routing.
