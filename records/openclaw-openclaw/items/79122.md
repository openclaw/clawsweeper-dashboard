Overall confidence: 0.93

Full review comments:

- **[P1] Remove the auto-loaded Docker override:** `docker-compose.override.yml:4-8`
  - body: `docker-compose.override.yml` is loaded automatically by Docker Compose from the repo root. This override runs the OpenClaw services as root and mounts both `/var/run/docker.sock` and private host paths, so a normal `docker compose` invocation would silently expose the host Docker daemon and break non-Chris machines.
  - confidence: 0.96
- **[P2] Keep private Prestigio services out of default compose:** `docker-compose.yml:167-181`
  - body: The root compose file is the documented OpenClaw Docker entry point, and `docker compose up` starts all services by default. Adding the Prestigio writer service here makes a normal OpenClaw Docker startup depend on private Supabase credentials and a Chris-specific app mount instead of keeping that integration external.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The diff introduces concrete Docker and supply-chain risks that are unrelated to a narrow pricing-source change.

Concerns:

- **[high] Auto-loaded root Docker socket mount:** `docker-compose.override.yml:6`
  - body: The added compose override runs OpenClaw containers as root and mounts `/var/run/docker.sock`; because the file name is Docker Compose's automatic override name, this would apply without an explicit opt-in and effectively grants host control.
  - confidence: 0.97
- **[medium] Service-role key expanded into private services:** `docker-compose.yml:173`
  - body: The default compose file wires `SUPABASE_SERVICE_KEY` into new Prestigio and marketing services. Even though the key is not committed, merging this would expand the service-role secret's runtime blast radius through private, root-level services outside OpenClaw's plugin/config boundary.
  - confidence: 0.86
- **[medium] Unpinned npm install in new Dockerfiles:** `gmail-reader/Dockerfile:4`
  - body: New service Dockerfiles run `npm install --production` without a lockfile, so builds resolve third-party dependencies at build time instead of using the repo's pinned dependency workflow.
  - confidence: 0.84
