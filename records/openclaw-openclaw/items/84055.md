## Summary

This PR remains useful: current main still omits `compat.supportsTools` when Ollama capabilities are unavailable, while the branch makes the fallback explicit and adds the matching regression assertion. The patch is small, well-scoped, and backed by after-patch CLI/provider output; the remaining merge decision is whether maintainers accept the compatibility tradeoff for truly non-tool-capable Ollama-compatible servers without reported capabilities.

## What This Changes

The PR changes the Ollama fallback model definition to set `compat.supportsTools: true` when capabilities are unavailable and adds a regression assertion for that path.

## Best Possible Solution

Keep the narrow provider metadata fix if maintainers accept the documented explicit-compat opt-out for incompatible local models, and leave the broader Ollama timeout/thinking/keep-alive work to separate review.

## Solution Assessment
