## Summary

Keep open. Current main still lacks the two requested GitHub Copilot model rows in the plugin-owned defaults and static manifest catalog, and both static and dynamic fallback metadata still assign 128k context to Copilot IDs that are not specially modeled. Open PR #72829 references this issue with closing syntax, while #71924 is a broader open live-catalog alternative, so the issue should remain open until one path lands or is explicitly superseded.

## What This Changes

Add `gpt-5.5` and `claude-opus-4.7-1m-internal` to the GitHub Copilot plugin defaults/catalog and give the 1M-context Copilot variant accurate context metadata.

## Best Possible Solution

Update the GitHub Copilot plugin-owned defaults, manifest static catalog, dynamic fallback metadata, and focused tests so the requested models are visible without manual config and 1M-context Copilot IDs do not inherit 128k metadata. If maintainers prefer live Copilot catalog discovery, that path should also cover provider-filtered list visibility and correct context/runtime metadata for these IDs.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main by inspecting the Copilot defaults, static manifest catalog, provider catalog hook, and dynamic fallback path; the requested IDs are absent and 1M-style IDs still resolve to 128k metadata.

## Solution Assessment

Yes in direction. A provider-local fix is the narrow maintainable path for this report, while live `/models` discovery may be the better long-term answer if maintainers want to stop hand-maintaining Copilot catalog rows.
