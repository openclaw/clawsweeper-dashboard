## Summary

Keep open. Current main still lacks the two requested GitHub Copilot model rows in the plugin-owned defaults and static manifest catalog, and the fallback metadata still assigns 128k context to unknown Copilot model IDs. Open PR #72829 references this issue with closing syntax, while #71924 is a broader live-catalog alternative, so the issue should remain open until one path is reviewed and landed or explicitly superseded.

## What This Changes

Request to add `gpt-5.5` and `claude-opus-4.7-1m-internal` to the GitHub Copilot provider model catalog/defaults and represent 1M-context Copilot IDs accurately.

## Best Possible Solution

Update the GitHub Copilot plugin-owned defaults, manifest static catalog, dynamic fallback metadata, and focused tests so the requested models are visible without manual config and 1M-context Copilot IDs do not inherit 128k metadata. If maintainers choose live Copilot catalog discovery instead, it should cover provider-filtered list visibility and correct context/runtime metadata for these IDs.
