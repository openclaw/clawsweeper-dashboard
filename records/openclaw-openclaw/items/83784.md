## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Route typing keepalives through the backoff handler:** `extensions/telegram/src/native-tool-progress-draft.ts:183-189`
  - body: The new status-with-typing path resolves `bot.api.sendChatAction` directly and schedules keepalives from the native draft helper. Existing Telegram typing goes through the per-account `sendChatActionHandler` because it applies 401 backoff and a circuit breaker; bypassing it can reintroduce repeated unauthorized typing calls, so pass/reuse that handler instead of calling the raw API here.
  - confidence: 0.93
- **[P2] Preserve fallback when the first native draft send rejects:** `extensions/telegram/src/native-tool-progress-draft.ts:383-385`
  - body: Dispatch falls back to the edited preview only when `nativeToolProgressDraft.update()` returns `false`, but this branch returns `true` immediately for the first send before Telegram accepts it. If that first `sendMessageDraft` later rejects, native drafts are disabled after the fallback chance has already been skipped; keep the final path non-blocking without reporting success until the first send succeeds or otherwise trigger the fallback.
  - confidence: 0.9
