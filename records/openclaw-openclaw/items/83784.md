The PR adds transient `sessions_spawn` subagent progress items, maps them into channel progress drafts, and reworks Telegram native tool-progress draft cadence, cancellation, and final-path behavior.

## Best Possible Solution

Keep the non-blocking final-answer path, preserve fallback when native draft sends reject, and add redacted Telegram Desktop or equivalent live proof for subagent start/completion drafts and normal final delivery.

## Reproduction Assessment

No current-main bug reproduction is required because this is a feature PR. The patch-introduced fallback regression is source-reproducible from PR head: `update()` returns true before the native draft send settles, while dispatch falls back only on a false result.

## Solution Assessment

No, not yet. The subagent progress surface is reasonable, but the native draft send path should stay non-blocking without losing the first rejection fallback, and the Telegram-visible behavior still needs live proof.
