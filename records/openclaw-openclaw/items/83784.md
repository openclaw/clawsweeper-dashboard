## Summary

Keep this draft PR open: the feature direction is useful, but it still lacks real Telegram behavior proof and the latest native-draft change regresses the first-send fallback when Telegram rejects `sendMessageDraft`.

## What This Changes

The PR adds transient `sessions_spawn` subagent progress items, maps them into channel progress drafts, and changes Telegram native tool-progress draft sending to coalesce, abort, and avoid blocking final-answer delivery.

## Best Possible Solution

Preserve nonblocking final delivery while keeping native-draft failure fallback semantics, then land only with redacted Telegram Desktop/DM proof showing subagent start and completion drafts update, clear, and leave final answer delivery durable.

## Reproduction Assessment

Not applicable as a current-main bug report; this is a feature PR. The patch defect is source-reproducible because dispatch treats `update(true)` as delivered while the PR returns true before the first native draft send settles.

## Solution Assessment

No, not as-is: the nonblocking native draft direction is maintainable, but it should preserve first-send fallback semantics or get an explicit maintainer tradeoff decision before live Telegram proof is accepted.
