## Summary

Keep open. Current main still enforces private 0600/0700-style permissions across config/state writers, security repair/audit, cron/state logs, exec approvals, and fs-safe-backed private stores, while the related open PR only covers one config-helper path and does not resolve the broader ACL-mask breakage.

## What This Changes

Request an opt-in permissions policy or config/env surface for group-readable OpenClaw state/config files and directories in multi-user deployments while keeping private defaults.

## Best Possible Solution

Define a maintainer-approved opt-in permissions policy that preserves private defaults, validates allowed file and directory modes, applies consistently across OpenClaw writers and fs-safe-backed stores, and aligns audit, doctor, docs, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a group/ACL-readable state directory, trigger config/state/cron/auth writes or exec approvals, and current chmod/dirMode paths can restore 0700/0600-style private modes that collapse group or POSIX ACL access.

## Solution Assessment

Unclear. An opt-in permission policy is the right direction, but a global skip flag is too narrow unless it covers the atomic writer, fs-safe-backed stores, audit/repair behavior, docs, and live upgrade proof together.
