## Summary

Keep this PR open: the duplicate-row fix is useful, but the current patch introduces a pre-dispatch SQLite-backed metadata read that can fail the ACP child turn before the existing best-effort task-tracking fallback can run, and the external PR currently has only test proof rather than after-fix real behavior proof.

## What This Changes

The PR suppresses Gateway CLI task-row creation for confirmed ACP manual-spawn child turns and adds focused gateway regression tests.

## Best Possible Solution

Keep the narrow CLI-row suppression, but make ACP confirmation best-effort with fallback to CLI tracking on metadata read failure, then add a throwing-read regression and real runtime smoke proof.

## Reproduction Assessment

Yes, source-reproducible: current main has ACP spawn call the Gateway with `acpTurnSource: "manual_spawn"` while also creating an ACP task row, and Gateway task tracking currently defaults that child run to CLI tracking. I did not run a live ACP smoke in this read-only review.

## Solution Assessment

No. The suppression is in the right area, but the implementation must preserve best-effort task tracking by treating metadata-read failure as unconfirmed and falling back to CLI tracking instead of blocking dispatch.
