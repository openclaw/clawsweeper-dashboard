## Summary

## What This Changes

The PR appends a bounded configured-roots hint to `assertLocalMediaAllowed` `path-not-allowed` errors and adds regression tests for short and overflow allowed-root lists.

## Best Possible Solution

Land the bounded shared formatter and tests after maintainers accept the disclosure boundary, preserve `LocalMediaAccessError.code` and the rejected-path contract, and document the user-facing diagnostic change in the Unreleased changelog with contributor credit.

## Reproduction Assessment

Yes. On current main, calling `assertLocalMediaAllowed` with an out-of-root local path and explicit allowed root would hit the source-verified bare `path-not-allowed` message at both rejection paths.

## Solution Assessment

Yes, subject to maintainer approval of the disclosure tradeoff. The shared helper with a four-root cap is a narrow maintainable fix; if the disclosure is rejected, the safer alternative is a generic or sanitized hint rather than listing real roots.
