## Summary

Keep open. Current main and v2026.6.9 still lack a `sessions_history` or `chat.history` path that traverses prior reset/archive/family transcript fragments for the same visible WebChat conversation, and the current implementation candidate is the open `includeFamily` PR at https://github.com/openclaw/openclaw/pull/90239.

## What This Changes

Request that WebChat `sessions_history` surface prior reset/archive/family transcript fragments for the same visible dashboard conversation, or clearly report that older fragments exist.

## Best Possible Solution

Land one maintainer-approved, bounded, header-checked, opt-in logical-history API that returns ordered fragment/source metadata or an explicit older-history-available indicator.

## Reproduction Assessment

Yes, at source level: current main exposes no family/archive selector, stores lineage separately, and reads only the active session id unless that active transcript is missing. I did not run a live WebChat reset in this read-only review.

## Solution Assessment

Unclear until maintainers accept the contract. The open `includeFamily` implementation is the closest current path, but the archived-history visibility boundary and public selector name need maintainer approval before this can close.
