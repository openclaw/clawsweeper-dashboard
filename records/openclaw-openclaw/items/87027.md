## Summary

Keep this PR open, but it should not merge as-is: the patch fixes the reported Slack DM symptom by bypassing the existing explicit wildcard acknowledgment for public DMs, which turns an invalid `dmPolicy: "open"` account config into public Slack DM access.

## What This Changes

Adds an early Slack direct-message authorization return for `dmPolicy: "open"` and a new unit test file covering open and allowlist DM authorization.

## Best Possible Solution

Keep the explicit `allowFrom: ["*"]` acknowledgment for public Slack DMs; if a valid account config with the wildcard still fails, fix that narrower path with regression coverage, otherwise route affected users through config repair such as `openclaw doctor --fix`.

## Reproduction Assessment

Yes, source inspection reproduces the submitted scenario: current main blocks `dmPolicy: "open"` without a sender or wildcard allowlist match before dispatch. I did not run a live Slack workspace because this review is read-only.

## Solution Assessment

No, the early return is not the best fix as written because it bypasses the explicit wildcard contract for public DMs. A safer path is to preserve the wildcard requirement and only repair a failing valid account configuration if one is demonstrated.
