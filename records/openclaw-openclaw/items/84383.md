## Summary

Keep open: the patch is aimed at the right lifecycle bookkeeping bug, but the latest message-tool mirror fix is still incomplete because generated run-scoped idempotency keys depend on `options.runId`, and the normal embedded runner tool-construction path does not pass that run id into `createOpenClawCodingTools`.

## What This Changes

The PR adds subagent completion cleanup credit for requester delivery mirrors, run-scoped mirror matching, message-tool idempotency propagation, and regression coverage for the Slack-style NO_REPLY path.

## Best Possible Solution

Thread the active embedded run id into the message tool on the normal agent/subagent runner path, keep the mirror matcher run-scoped, and prove final-head Slack behavior with no duplicate output and no retry-limit bookkeeping.

## Reproduction Assessment

Yes, by source inspection: current main finalizes subagent cleanup from the later announce `didAnnounce` result and does not consult requester delivery mirrors, matching the linked Slack retry-limit report. I did not run a live Slack reproduction in this read-only review.

## Solution Assessment

No, not yet. The lifecycle/mirror approach is the right boundary, but the submitted implementation does not actually feed the active embedded run id into the normal message-tool path that the latest mirror-credit fix depends on.
