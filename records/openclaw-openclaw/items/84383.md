## Summary

Keep open: this targets a real subagent delivery bookkeeping bug and the follow-up Slack proof is useful, but the new mirror credit can accept an older identical requester `delivery-mirror` entry and suppress retries for a currently undelivered completion.

## What This Changes

The PR adds a subagent lifecycle cleanup check that credits matching requester-session `openclaw/delivery-mirror` messages as delivered completions, plus a focused regression test.

## Best Possible Solution

Keep the mirror-credit direction, but require evidence tied to the current subagent run before clearing delivery errors and add stale-identical mirror regression coverage before merge.

## Reproduction Assessment

Yes, from source inspection and the linked Slack report: current main trusts the parent announce `didAnnounce` result, so a parent `NO_REPLY` after a committed mirror leaves retry bookkeeping. I did not run a current-main live repro in this read-only pass, so this is source-reproducible rather than locally reproduced.

## Solution Assessment

No, not yet: the proposed direction is maintainable, but exact-text matching across the last 25 history rows is too broad. The safer solution is to bind the mirror to the current run or delivery window before treating it as delivered.
