## Summary

Keep open. Current main has a Slack-scoped `thread.requireExplicitMention` escape hatch and shared implicit-kind allowlists, but it still lacks the requested cross-channel strict mention policy; the remaining work is a public config/API decision rather than conservative cleanup.

## What This Changes

The issue requests an additive strict mention mode that prevents implicit thread replies or bound sessions from satisfying `requireMention`.

## Best Possible Solution

Add a backward-compatible shared strict implicit-mention policy through the SDK/channel activation contract, wire it into affected plugin config and docs, and keep existing implicit behavior as the default unless maintainers choose a migration.

## Reproduction Assessment

Yes, source-level. Current shared mention gating treats implicit mention kinds as effective mentions, and Discord bound sessions can clear `requireMention` before the mention decision; no live transport run was performed.

## Solution Assessment

Unclear until maintainers choose the public API. A strict mode is directionally sound, but the maintainable fix should be additive and coordinated across shared SDK contracts, Slack's existing option, Discord bound-session behavior, and docs.
