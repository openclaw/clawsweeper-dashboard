## Summary

Keep open. Current main and v2026.5.18 still provide only boolean `requireMention`, an implicit-kind allowlist, and Slack’s narrower `thread.requireExplicitMention`; the requested cross-channel strict mention policy remains a real product/config decision for Discord and other reply-aware channel paths.

## What This Changes

The issue requests a strict mention policy that disables implicit reply, thread, or session mention bypasses when mention gating is enabled.

## Best Possible Solution

Add a backward-compatible strict implicit-mention policy to the shared activation contract, wire it through affected channel configs and docs, and preserve existing implicit behavior as the default unless maintainers explicitly choose a migration.

## Reproduction Assessment

Yes, source-level. Current code and tests show implicit mention kinds and Discord bound-thread/session context still satisfy or bypass mention gating without a shared strict policy.

## Solution Assessment

Unclear. A strict mode is directionally sound, but the exact public shape should be a maintainer-approved additive policy that reconciles Slack’s existing option, Discord thread/session behavior, and the shared SDK contract.
