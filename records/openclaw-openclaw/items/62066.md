## Summary

Keep open. Current main has Slack’s account-level `thread.requireExplicitMention` partial workaround and shared implicit-kind filtering, but it still lacks the requested cross-channel strict mention policy; Discord and other channel paths still treat reply/thread/session context as an effective mention under current configuration.

## What This Changes

The issue requests a strict mention mode that requires explicit mentions and disables implicit reply/thread/session wakeups across group-capable channel plugins.

## Best Possible Solution

Define one additive strict mention policy on top of the existing implicit-kind allowlist, wire it through affected bundled channel plugins and config schemas, document the default, and preserve Slack’s existing option as a compatible alias or narrower override.

## Reproduction Assessment

Yes, source-level. Slack has an explicit-thread-only partial option, while Discord bound-thread and bot-owned autoThread paths still bypass `requireMention`, and current tests assert some of that behavior.

## Solution Assessment

Unclear. The requested strict mode is directionally sound, but the best maintainable solution needs maintainer API judgment so Slack’s existing option, Discord thread/session behavior, and shared implicit-kind policy stay coherent.
