## Summary

Keep open. Current main has a Slack-only explicit-thread-mention option and a shared implicit-mention allowlist seam, but it still lacks the requested cross-channel strict mention policy and Discord still contains source-backed bypass paths for bound or bot-owned thread sessions.

## What This Changes

The issue requests a strict mention mode or equivalent config that disables implicit thread/session mention wakeups so only explicit bot mentions or mention patterns pass the gate.

## Best Possible Solution

Define one additive mention policy on top of the existing implicit-kind allowlist, wire it through the affected bundled plugins and config schemas, document the default, and preserve Slack's existing option as a compatible narrow alias or special case.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a bound Discord thread/session or bot-owned autoThread with `requireMention: true`, then send an unmentioned follow-up; current preflight clears the mention requirement before the no-mention drop branch, and tests assert that behavior.

## Solution Assessment

Unclear. The requested strict mode is directionally sound, but the best maintainable fix needs a maintainer API decision so Slack's existing option, Discord thread behavior, and the shared implicit-kind policy do not diverge.
