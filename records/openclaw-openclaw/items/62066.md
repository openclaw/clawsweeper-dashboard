## Summary

Keep open. Current main partially addresses the Slack thread case with `channels.slack.thread.requireExplicitMention`, but it does not implement the requested general strict mention mode and still has Discord thread-bound routing that disables `requireMention` before the shared mention gate can skip unmentioned messages.

## What This Changes

The issue requests an opt-in strict mention mode so implicit reply, thread, or session participation does not satisfy `requireMention` unless the bot is explicitly mentioned or a configured mention pattern matches.

## Best Possible Solution

Define an additive mention policy backed by the existing `allowedImplicitMentionKinds` seam, then wire it through the affected channel plugins starting with Discord bound threads and document how it relates to Slack's existing `thread.requireExplicitMention` option.

## Reproduction Assessment

Yes. A source-level reproduction exists for the remaining Discord path: create a guild thread-bound session with configured `requireMention: true`, then send an unmentioned follow-up; current main sets the bound-thread bypass before the no-mention skip can fire.

## Solution Assessment

No. The current Slack-specific option is useful but incomplete for this issue; the maintainable direction is an explicit opt-in policy that preserves defaults while letting operators disable implicit mention kinds consistently.
