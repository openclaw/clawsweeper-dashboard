## Summary

Keep open. Current main has a shipped Slack-only `thread.requireExplicitMention` opt-in and a generic SDK allowlist for implicit mention kinds, but Discord, Telegram, WhatsApp, and Microsoft Teams still pass implicit reply/quote evidence through the mention gate without a user-facing strict mention mode; the remaining cross-channel config behavior needs maintainer product review rather than cleanup close.

## What This Changes

The issue requests a strict mention mode that disables implicit reply/thread or bound-session wakeups while still allowing explicit bot mentions and configured mention patterns.

## Best Possible Solution

Define one opt-in strict explicit-mention policy for channel ingress, preserve current implicit behavior by default, then wire the affected plugins and docs/tests to that shared contract.

## Reproduction Assessment

Yes, at source level. Current main shows implicit reply/quote kinds still satisfy `requireMention` unless a caller supplies an implicit-kind allowlist, and only Slack currently exposes a user-facing strict thread option.

## Solution Assessment

Unclear. `requireMention: "strict"` is plausible, but the best fix needs a maintainer decision on cross-channel config shape and compatibility because Slack already uses `thread.requireExplicitMention`.
