## Summary

Keep open. Current main has a shared implicit-mention allowlist seam and a Slack-only opt-in, but it does not provide the requested cross-channel `requireMention: "strict"` policy, and Discord/Telegram/WhatsApp/Microsoft Teams paths still preserve implicit or bound-session wake behavior by default.

## What This Changes

Add a stricter mention-gating mode so implicit thread or bound-session wakeups do not satisfy `requireMention` without an explicit mention.

## Best Possible Solution

Define an additive strict implicit-mention policy through the shared mention-decision seam, keep current defaults for compatibility, and wire/docs-test each affected channel that supports implicit reply or bound-session wakeups.

## Reproduction Assessment

Yes, source-level. Core mention gating still treats implicit mention kinds as effective mentions unless a policy allowlist excludes them, Slack only exposes a Slack-thread-specific opt-in, and Discord bound-thread preflight can still bypass the mention requirement.

## Solution Assessment

Unclear until maintainers choose the public API. The requested strict mode is plausible, but the best implementation should be additive and coordinated across shared mention policy plus channel-owned config/docs rather than an unconditional behavior change.
