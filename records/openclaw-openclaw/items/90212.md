## Best Possible Solution

Land the PR if maintainers accept that native slash command feedback should use the same suppression-bypass metadata as normal inline command replies, while keeping separate CardKit rendering work outside this branch.

## Solution Assessment

Yes, subject to maintainer acceptance of the delivery semantic: reusing the existing explicit-command suppression-bypass marker is narrower than adding Feishu-specific behavior or a second dispatch path. A stricter alternative would mark only /compact command-handler replies, but that would leave sibling native slash replies inconsistent with the existing inline-action command path.
