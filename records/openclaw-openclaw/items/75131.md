The issue asks to make Telegram delivery retries idempotent, permanently fail terminal Telegram 400/403 errors, and split long Telegram messages before send.

## Best Possible Solution

Land a focused reliability fix that gives logical direct sends stable queue identity, classifies the reported terminal Telegram errors as permanent, and preserves the current chunk-aware Telegram send path.
