Keep open. Current main already has Telegram long-text chunking, but the delivery queue still creates a fresh UUID for each non-recovery send attempt and the permanent-error classifier still misses several terminal Telegram errors; the linked fix PR is open but not merged.

The issue requests stable delivery-queue identity for logical retries, permanent handling for terminal Telegram 400/403 errors, and pre-splitting Telegram text before the 4096-character limit.

## Best Possible Solution

## Solution Assessment
