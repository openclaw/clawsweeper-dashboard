## Summary

Keep open. Current main still leaves the reported Telegram delivery reliability cluster unresolved: queued live send attempts still get fresh queue IDs, the permanent-error classifier omits several reported Telegram terminal failures, and direct Telegram action sends can still bypass the chunked text-send path.

## What This Changes

The issue requests a Telegram delivery fix so retries preserve queue identity, permanent Telegram 400/403 failures move to failed delivery state, and outbound Telegram text is split before exceeding the 4096-character provider limit.

## Best Possible Solution

Land one focused reliability fix that gives failed logical sends stable queue update semantics, classifies unrecoverable Telegram delivery errors before live/recovery retry, and routes every Telegram text-send surface through a chunk-aware <=4000/4096-character path.
