## Summary

Close as duplicate/superseded. Current main still lacks the requested generic RPM/RPD-aware fallback cooldown contract, but the same remaining request/token interval budget work is already tracked by https://github.com/openclaw/openclaw/issues/30828; adjacent provider-pool and quota-bucket variants remain covered by https://github.com/openclaw/openclaw/issues/41135 and https://github.com/openclaw/openclaw/issues/28847.

## What This Changes

Request configurable provider RPM/RPD quotas so model fallback can preflight quota windows and set precise `cooldownUntil` values instead of relying on reactive cooldowns.

## Best Possible Solution

Track the shared request/token interval budget design in https://github.com/openclaw/openclaw/issues/30828, preserving this issue's fallback cooldown details there and leaving provider-pool or quota-bucket variants on the existing routing trackers.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks `rpm`/`rpd` config, quota counters, and a pre-request quota skip path.

## Solution Assessment

Yes for triage. Consolidating this under the open request/token interval budget tracker is the narrowest maintainable cleanup because the remaining work is a public budget/scheduler contract, not a separate already-defined bug fix.
