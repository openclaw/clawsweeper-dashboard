## Summary

Keep open. Current main still lacks a generic RPM/RPD-aware fallback contract; the related provider-profile routing tracker is relevant, but this provider-level quota cooldown API remains unimplemented and needs maintainer API/product direction.

## What This Changes

Request to add configurable per-provider RPM/RPD quota windows that preflight fallback and set precise auth-profile cooldownUntil values.

## Best Possible Solution

Define a documented generic quota/budget contract, persist lock-backed minute/day window state, feed exact reset deadlines into cooldownUntil, and cover config, schema, docs, fallback, and auth-profile tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks rpm/rpd fields, persisted quota counters, and a pre-request quota skip path.

## Solution Assessment

Unclear pending maintainer design. The requested outcome is plausible, but provider config, auth cooldown config, and plugin-supplied quota metadata imply different public API and concurrency contracts.
