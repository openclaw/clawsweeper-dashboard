## Summary

Keep open. Current main still lacks the requested generic RPM/RPD-aware fallback contract: provider config has no quota fields, usage state has no minute/day counters, and fallback preflight only reacts to existing cooldowns; the current WHAM path is provider-specific for OpenAI Codex rather than a generic quota mechanism.

## What This Changes

The issue requests generic RPM/RPD-aware provider quota metadata and fallback cooldown windows so providers can be skipped until exact quota reset times instead of relying only on reactive cooldowns.

## Best Possible Solution

Define a generic quota metadata and persisted window-state contract, then integrate lock-backed pre-request skipping with cooldownUntil plus config schema/help/docs and focused fallback/auth-profile tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an enhancement request. Source inspection gives a high-confidence absence check for the requested provider quota fields, persisted counters, and pre-request quota skip path.

## Solution Assessment

Unclear pending maintainer review. The requested outcome is plausible, but the durable API shape is a product/compatibility decision because provider config, auth cooldowns, and plugin-owned usage metadata have different contract implications.
