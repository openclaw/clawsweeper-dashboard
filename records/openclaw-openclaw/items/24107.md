## Summary

Keep open. Current main still lacks a generic RPM/RPD-aware fallback contract: provider config has no quota fields, usage state has no minute/day counters, and fallback preflight only reacts to existing cooldowns; the current WHAM path is provider-specific for OpenAI Codex and does not cover configurable per-provider quotas.

## What This Changes

The issue requests generic provider RPM/RPD quota-aware fallback cooldowns that skip providers until exact reset windows instead of relying on reactive backoff.

## Best Possible Solution

Define a documented generic quota metadata and persisted window-state contract, then integrate lock-backed pre-request skipping with cooldownUntil plus config schema/help/docs and focused fallback/auth-profile tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an enhancement request. Source inspection gives a high-confidence absence check for the requested provider quota fields, persisted counters, and pre-request quota skip path.

## Solution Assessment

Unclear pending maintainer review. The requested outcome is plausible, but the durable API shape is a product/compatibility decision because provider config, auth cooldowns, and plugin-owned usage metadata have different contract implications.
