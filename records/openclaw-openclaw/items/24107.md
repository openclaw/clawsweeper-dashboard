## Summary

Keep open. Current main still lacks the requested generic RPM/RPD-aware fallback contract: provider config has no quota fields, auth-profile state has no request/day counters, and fallback preflight only reacts to already-recorded cooldowns rather than configured quota windows.

## What This Changes

Request adding configurable per-provider RPM/RPD quotas that compute precise pre-request cooldown windows and integrate with auth-profile fallback state.

## Best Possible Solution

Define a documented generic request-quota/budget contract, persist exact reset-window state, integrate lock-backed pre-request fallback skipping with cooldownUntil, and update config schema, docs, and focused fallback/auth-profile tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks rpm/rpd fields, persisted quota counters, and a pre-request quota skip path.

## Solution Assessment

Unclear pending maintainer design. The requested outcome is plausible, but the durable shape should be chosen as a public API/product decision because provider config, shared budget state, and plugin-owned quota metadata have different compatibility implications.
