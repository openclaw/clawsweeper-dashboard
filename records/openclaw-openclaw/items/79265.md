## Summary

Keep this PR open: the gateway lock-recovery direction matches a real current-main failure path, but the external PR still needs real behavior proof before merge and is missing the required changelog entry for a user-facing gateway fix.

## What This Changes

This PR changes the embedded PI compaction retry loop to yield once before retrying and changes the gateway run loop to convert EADDRINUSE restart failures into GatewayLockError lock-recovery errors.

## Best Possible Solution

Land a tested gateway run-loop fix that routes post-first-start main-listener EADDRINUSE conflicts through supervised lock recovery, with the compaction-yield change kept only if runtime evidence shows it is needed.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection does show a credible current-main path where a post-first-start EADDRINUSE startup failure is swallowed by `runGatewayLoop` instead of reaching supervised lock recovery.

## Solution Assessment
