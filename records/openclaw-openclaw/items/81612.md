Adds optional `reloadKind` metadata to `config.schema.lookup` results and child summaries, sourced from the existing config reload planner, with protocol docs, generated Swift models, and focused tests updated.

## Best Possible Solution

Land this additive lookup metadata if maintainers accept the API surface, keeping reload behavior single-sourced from the existing planner and protocol/docs/client artifacts aligned.

## Reproduction Assessment

Yes. Current-main source inspection shows lookup responses cannot include reload metadata, and the linked issue includes a concrete OpenClaw 2026.5.7 runtime reproduction for `gateway.handshakeTimeoutMs`.

## Solution Assessment

Yes, subject to maintainer API approval. Reusing the existing reload-rule matcher through an optional lookup resolver is the narrow maintainable path because it avoids duplicating reload policy and keeps the full generated schema unchanged.
