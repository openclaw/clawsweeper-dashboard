## Summary

This PR should stay open: current main still lacks reload metadata on `config.schema.lookup`, and the branch is a focused additive protocol change with sufficient live proof and no blocking functional or security finding from the diff.

## What This Changes

The branch adds optional `reloadKind` metadata to `config.schema.lookup` results and child summaries, wires the Gateway handler to the existing reload planner, and updates tests, protocol docs, and generated Swift models.

## Best Possible Solution

Land the additive lookup metadata after maintainer acceptance of the protocol surface, preserving the existing reload planner as the single source of truth and keeping the full generated schema contract unchanged.

## Reproduction Assessment

Yes. Current main source shows `config.schema.lookup` cannot return reload metadata while the reload planner can classify the same paths, and the linked issue includes release-level proof for `gateway.handshakeTimeoutMs`.

## Solution Assessment

Yes. The optional resolver-backed `reloadKind` field is the narrowest maintainable approach in this PR because it avoids duplicating reload rules and keeps the change additive to the lookup contract.
