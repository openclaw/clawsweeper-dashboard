## Summary

Keep open for normal PR review: the patch is a narrow additive protocol change with sufficient real behavior proof, green relevant checks, and no blocking findings. The linked issue should remain open until this PR lands.

## What This Changes

The branch adds optional `reloadKind` metadata to `config.schema.lookup` results and child summaries through the config schema lookup, Gateway protocol schema, Gateway handler, Swift model, docs, and focused tests, plus a small SDK subpaths docs update.

## Best Possible Solution

Land the additive lookup metadata if maintainers accept the protocol surface, keeping reload behavior single-sourced from the existing reload planner and schema/protocol/client docs aligned.

## Reproduction Assessment

Yes. Current main shows lookup result and child types omit reload metadata and the Gateway handler calls `lookupConfigSchema` without a resolver, while the reload planner already classifies those paths; the linked issue also includes Docker proof on v2026.5.7.

## Solution Assessment

Yes. The optional resolver-backed `reloadKind` is the narrowest maintainable solution because it reuses the existing planner, avoids a second rule table, and keeps the full config schema contract unchanged.
