## What This Changes

This PR adds a Telegram bare grammY network-envelope retry match and tests, while also changing Ollama think request placement and sessions JSON metadata projection.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Fix the actual safe send retry predicate:** `extensions/telegram/src/network-errors.ts:267-269`
  - body: Normal Telegram message and media delivery uses `isSafeToRetrySendError` through the non-idempotent request wrapper, and that helper intentionally does not infer safe retry from a plain grammY envelope. Changing only `isRecoverableTelegramNetworkError` broadens ancillary send-context retries but does not prove that dropped `sendMessage` replies will retry; carry nested pre-connect evidence into the safe-send path instead.
  - confidence: 0.88
- **[P2] Keep native Ollama think on the chat request field:** `extensions/ollama/src/stream.ts:236-239`
  - body: Ollama's current chat request contract exposes `think` as a top-level field, while `options` is for model runner options and does not define `think`. Moving runtime and configured values into `options.think` can make `/think off` and effort controls ineffective for reasoning-capable native Ollama models, so this unrelated fix needs to preserve top-level `think` for supported models or be split for separate review.
  - confidence: 0.9
- **[P2] Do not close the sessions metadata issue with a partial projection:** `src/commands/sessions-table.ts:27-37`
  - body: The sessions change adds some runtime fields but omits requested lineage fields that already exist in `SessionEntry` and gateway rows, including `spawnedWorkspaceDir`, `subagentRole`, and `subagentControlScope`. If this branch lands with the closing commit, the linked sessions issue can close while the supported CLI JSON surface is still incomplete.
  - confidence: 0.78
