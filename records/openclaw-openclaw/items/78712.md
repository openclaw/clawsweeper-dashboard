## Summary

Keep open: current main and v2026.6.8 still grant default operator scopes, including admin, to shared-secret callers on trusted-operator plugin HTTP routes. The remaining work is security/product contract review because current docs and shipped bundled routes encode conflicting expectations.

## What This Changes

The issue reports that shared-secret bearer callers on trusted-operator plugin HTTP routes receive default operator scopes including operator.admin despite narrow or absent scope headers.

## Best Possible Solution

Maintainers should decide and document the shared-secret trusted-operator contract, then align the resolver, route tests, bundled route docs, and real Gateway proof.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: token/password auth sets trustDeclaredOperatorScopes false, the trusted-operator resolver returns default scopes including admin, and current tests assert admin helper access.

## Solution Assessment

Unclear. Narrowing the resolver is plausible, but the best fix needs maintainer/security agreement because shipped tests, admin-route docs, and plugin internals docs disagree on the shared-secret trusted-operator contract.
