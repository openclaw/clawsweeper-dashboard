## Summary

Keep open: current main and v2026.6.8 still grant admin-capable default operator scopes to shared-secret callers on trusted-operator plugin HTTP routes, and the two linked fix PRs closed unmerged. The remaining work needs maintainer/security contract review because current docs and bundled admin-style routes encode conflicting expectations.

## What This Changes

Prevent shared-secret callers from receiving implicit admin-capable runtime scopes on plugin HTTP routes that opt into the trusted-operator surface.

## Best Possible Solution

Maintainers should decide and document the shared-secret trusted-operator contract, then land a security-reviewed change that aligns the resolver, route tests, bundled route docs, and real Gateway proof.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: token/password auth sets `trustDeclaredOperatorScopes: false`, and the trusted-operator resolver maps that to default scopes containing `operator.admin`. I did not run a live Gateway repro in this read-only review.

## Solution Assessment

Unclear. Narrowing the resolver is plausible, but the best fix needs maintainer/security agreement because shipped tests, admin-route docs, and plugin internals docs disagree on whether shared-secret trusted-operator plugin routes should be full-operator or conservative.
