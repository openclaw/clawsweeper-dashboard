## Summary

Keep open. Current `main` and the latest release still restore the full default operator scope set for shared-secret callers on `trusted-operator` plugin routes, and the open closing PR has not merged.

## What This Changes

The issue requests preventing shared-secret bearer callers from receiving implicit admin/default operator scopes on `trusted-operator` plugin HTTP routes.

## Best Possible Solution

Review and land or reject the open security fix path at https://github.com/openclaw/openclaw/pull/78759, with resolver, tests, docs, changelog, and real route proof aligned to the intended `trusted-operator` contract.

## Reproduction Assessment

Yes. Current-main source inspection shows token/password auth sets `trustDeclaredOperatorScopes: false`, and the `trusted-operator` resolver branch converts that into default scopes containing `operator.admin`; focused and dispatch-layer tests assert the same behavior.

## Solution Assessment

Unclear. The central resolver is the right place to fix the reported mapping, but the route contract is security-sensitive and currently used by bundled plugins, so maintainer/security review should settle the exact fallback semantics before merge.
