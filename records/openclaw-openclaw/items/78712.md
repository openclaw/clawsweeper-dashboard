## What This Changes

The issue reports that shared-secret bearer callers on trusted-operator plugin HTTP routes receive default operator scopes including operator.admin despite narrow or absent declared scopes.

## Best Possible Solution

Review and land or reject the linked fix PR after aligning resolver behavior, route-level tests, docs, changelog, and real gateway proof so shared-secret trusted-operator routes cannot receive implicit admin scopes.

## Reproduction Assessment

Yes. Source inspection on current main shows token/password auth sets trustDeclaredOperatorScopes false, and trusted-operator route resolution converts that path into default scopes containing operator.admin; existing tests assert the same path.

## Solution Assessment

Unclear. Removing the implicit admin escalation is the right direction, but the active PR still needs route-contract alignment, real behavior proof, and security review before it is the best mergeable solution.
