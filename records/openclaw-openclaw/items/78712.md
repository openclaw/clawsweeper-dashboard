## Summary

Keep open: current main and v2026.6.6 still grant admin-capable default scopes to shared-secret callers on `trusted-operator` plugin HTTP routes, while the related fix PRs are closed unmerged and the final contract needs maintainer/security review.

## What This Changes

The issue asks OpenClaw to prevent shared-secret callers on `trusted-operator` plugin HTTP routes from receiving implicit admin-capable runtime scopes.

## Best Possible Solution

Maintainers should settle the `trusted-operator` shared-secret contract, then align the resolver, tests, docs, and live Gateway proof with that security decision.

## Reproduction Assessment

Yes. Source inspection shows token/password auth sets `trustDeclaredOperatorScopes: false`, and current `trusted-operator` plugin-route scope resolution converts that path into default scopes containing `operator.admin`.

## Solution Assessment

Unclear. A narrow resolver change is plausible, but the best fix depends on a maintainer/security decision because shipped code, tests, SECURITY.md, and plugin-route docs do not currently express one consistent shared-secret contract.
