## Summary

Keep open: current main and v2026.6.6 still grant default operator scopes, including admin, to shared-secret callers on trusted-operator plugin HTTP routes, and the only linked fix PR closed unmerged.

## What This Changes

The issue asks whether gateway-auth plugin HTTP routes using `trusted-operator` should stop giving shared-secret callers implicit admin-capable runtime scopes.

## Best Possible Solution

A security-reviewed change should align the resolver, route-level tests, docs, and real Gateway proof around one explicit trusted-operator shared-secret contract.

## Reproduction Assessment

Yes. Source inspection shows token/password auth sets `trustDeclaredOperatorScopes: false`, the trusted-operator resolver converts that path to default scopes containing `operator.admin`, and current tests assert the admin-capable result.

## Solution Assessment

Unclear. A narrow resolver change is plausible, but the best fix requires maintainer/security agreement because current shipped code, tests, and admin HTTP RPC docs intentionally preserve full shared-secret operator behavior for opted-in trusted routes.
