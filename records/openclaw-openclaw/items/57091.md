## Summary

Keep open. Current main and v2026.6.6 still only report loaded skills, so malformed or incomplete workspace `SKILL.md` candidates can be dropped before CLI/status/UI surfaces can explain why; https://github.com/openclaw/openclaw/pull/92028 is an active implementation candidate, but it is not merged and still needs maintainer review for whether CLI lint is enough for the requested Web/status visibility.

## What This Changes

The issue requests CLI/Web/Gateway visibility and validation for malformed workspace `SKILL.md` frontmatter so invalid skills are not silently omitted.

## Best Possible Solution

Land a maintainer-approved shared diagnostics/validation path for malformed workspace `SKILL.md` files, with redacted bounded output and a clear decision on whether `skills lint`, `skills list/check`, and the Web console all surface skipped candidates.

## Reproduction Assessment

Yes, at source level. A missing frontmatter block produces empty metadata and missing `description` or parse/read failures return `null` from the loader before status, CLI, or UI formatting can report the skipped candidate.

## Solution Assessment

Unclear as a final product shape. The direction is right, but maintainers still need to decide whether a standalone `skills lint` command is sufficient or whether the normal status/list/Web surfaces must carry the diagnostics contract too.
