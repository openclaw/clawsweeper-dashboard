## Summary

Keep open. Current main still drops malformed or incomplete workspace skill candidates before `skills.status`, CLI formatting, or the Control UI can report them, and the merged `oc-path` work only adds `openclaw path validate`, not `openclaw skills validate` or loader diagnostics.

## What This Changes

The issue requests visible warnings and a validation workflow for malformed workspace `SKILL.md` frontmatter in CLI and Web/Control UI skill surfaces.

## Best Possible Solution

Add a shared skill discovery diagnostics model for skipped `SKILL.md` candidates, surface it through `skills.status`, CLI list/check/validate, and the Control UI, and document the workflow while preserving compatible fallback behavior unless maintainers choose to tighten it.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: malformed, unfenced, unreadable, or description-less `SKILL.md` candidates become `{}` or `null` in the loader and never reach status, CLI, or UI formatting.

## Solution Assessment

Yes, with a design caveat. The maintainable solution is a shared loader/status diagnostic contract with redacted and bounded logging; a standalone CLI-only warning or path-substrate validator would not cover the UI and gateway surfaces.
