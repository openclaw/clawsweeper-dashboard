## Label Justifications

- `P2`: Malformed workspace skills can silently disappear from user-visible lists and agent context, but the affected surface is bounded to skill diagnostics rather than a core outage.
- `impact:session-state`: The issue affects the skill catalog that is injected into agent context by omitting user-created skills without visible status.

## Summary

Keep open. Current main and v2026.5.22 still lack skipped-skill diagnostics or a native `skills validate` command, so invalid workspace skill candidates can still disappear before CLI/status/UI surfaces can report them; the related diagnostics PR was closed unmerged.

## What This Changes

Request adds visible SKILL.md frontmatter parsing and validation diagnostics for workspace skills across CLI, Web UI, and docs, including a `skills validate` command.

## Best Possible Solution

Add a shared, redacted skipped-skill diagnostics model in discovery/status, surface it through CLI `list`/`check`/`validate`, Gateway/UI status, and docs, while preserving compatible loading behavior by default.

## Reproduction Assessment

Yes, at source level. A missing frontmatter block yields `{}`, and a missing description returns `null` in `loadSingleSkillDirectory` before status, CLI, or UI formatting can report the skipped candidate.

## Solution Assessment

Yes directionally, with a design caveat. The maintainable path is a shared diagnostics/status contract with redaction and caps, not CLI-only warnings or raw loader log spam.
