## Label Justifications

- `P2`: Malformed workspace skills can silently disappear from user-visible lists and agent context, but the scope is a bounded diagnostics/product improvement rather than a core outage.
- `impact:session-state`: The failure changes the skill catalog injected into agent context by silently omitting user-created skills.

## Summary

Keep open. Current main still drops malformed or incomplete workspace skill candidates before they can appear in `skills.status`, CLI output, or the Control UI, and the related diagnostics PR was closed in favor of this broader canonical issue.

## What This Changes

Add user-visible diagnostics and validation for malformed workspace `SKILL.md` frontmatter across skill loading, CLI, Web UI, and docs.

## Best Possible Solution

Add a shared, redacted, bounded skipped-skill diagnostics model in skill discovery/status and surface it through CLI list/check/validate, Control UI, and docs while preserving compatible loading behavior.

## Reproduction Assessment

Yes, at source level. A malformed, unfenced, unreadable, or description-less `SKILL.md` becomes `null` or `{}` before status, CLI, or UI formatting can report the skipped candidate.

## Solution Assessment

Yes directionally, with a design caveat. The maintainable path is a shared diagnostics/status contract with redaction and caps, not CLI-only warnings or raw loader log spam.
