## Summary

Keep open. Current main still silently drops malformed or incomplete workspace skill candidates before the status payload, CLI, or Control UI can report them, and the related `oc-path` work does not add `openclaw skills validate` or loader diagnostics.

## What This Changes

The issue requests visible diagnostics and validation for malformed or incomplete workspace `SKILL.md` frontmatter so custom workspace skills do not disappear silently.

## Best Possible Solution

Add a shared skill discovery diagnostics model for skipped candidate paths, surface it through `skills.status`, CLI list/check/validate, and the Control UI, and document the validation workflow while preserving current compatibility where possible.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: malformed or unfenced frontmatter becomes `{}` or a failed load, then `loadSingleSkillDirectory` returns `null` before status/CLI/UI formatting can report the skipped file.

## Solution Assessment

Yes. The maintainable path is a shared loader/status diagnostic contract, not CLI-only logging; `openclaw skills validate` can then reuse that same discovery result.
