## Label Justifications

- `P2`: Malformed workspace skills can be silently absent from user-visible skill lists and agent context, but the issue is a bounded skills diagnostics improvement rather than a core outage.
- `impact:session-state`: The reported failure changes the skill catalog injected into agent context by silently omitting user-created skills.

## Summary

Keep open. Current main still has the silent-drop shape: malformed, unfenced, unreadable, or description-less workspace skill candidates disappear before `skills.status`, CLI formatting, or the Control UI can report them, and the related diagnostics PR was closed as superseded by this broader issue.

## What This Changes

The issue requests visible parsing/validation diagnostics for malformed workspace `SKILL.md` frontmatter in the CLI, Web console, and docs.

## Best Possible Solution

Add a shared, redacted, bounded skill discovery diagnostics model for skipped `SKILL.md` candidates and surface it through `skills.status`, CLI list/check/validate, the Control UI, and docs while preserving compatible fallback behavior unless maintainers choose a stricter contract.

## Reproduction Assessment

Yes, at source level. A malformed, unfenced, unreadable, or description-less `SKILL.md` under workspace skills becomes `{}` or `null` in the loader, and the status/CLI/UI contracts only carry loaded skills.

## Solution Assessment

Yes, with a design caveat. The requested direction is the right product shape, but the maintainable fix should be a shared diagnostics/status contract rather than CLI-only warnings or raw log spam.
