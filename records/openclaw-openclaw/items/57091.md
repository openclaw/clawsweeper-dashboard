## Label Justifications

- `impact:session-state`: The issue affects the skill catalog injected into agent context by silently omitting user-created skills.

## Summary

Keep open. Current main still drops invalid workspace skill candidates before they can appear in status, CLI output, or the Control UI; https://github.com/openclaw/openclaw/pull/48960 was closed unmerged, and https://github.com/openclaw/openclaw/pull/43253 only mitigates one generated-template YAML footgun.

## What This Changes

The request asks for visible SKILL.md frontmatter diagnostics, a new `openclaw skills validate` command, aligned CLI/UI behavior, and documentation for custom workspace skill validation.

## Best Possible Solution

Add a shared, redacted skipped-skill diagnostics model in discovery/status, then surface it through CLI list/check/validate, Control UI, and docs while preserving compatible loading behavior.

## Reproduction Assessment

Yes, at source level. A malformed, unfenced, unreadable, or description-less SKILL.md becomes `null` or `{}` before status, CLI, or UI formatting can report the skipped candidate.

## Solution Assessment

Yes directionally, with a design caveat. The maintainable path is a shared diagnostics/status contract with redaction and caps, not CLI-only warnings or raw loader log spam.
