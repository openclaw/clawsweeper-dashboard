## Summary

Keep open. Current main still documents and implements auto-created fallback session pins as sticky until reset, and the related implementation PR at https://github.com/openclaw/openclaw/pull/79678 remains open and conflicting rather than merged into main.

## What This Changes

Requested automatic reconciliation of auto-created fallback session model pins so recovered configured primary models are retried instead of being bypassed indefinitely.

## Best Possible Solution

Keep explicit user model selections strict, but add a bounded expiry or throttled primary-probe policy for auto-created fallback pins with docs and regression coverage.

## Reproduction Assessment

Yes for the source behavior: current main persists auto fallback overrides and reuses stored session overrides on later turns. I did not run a live Slack/provider outage reproduction in this read-only review.

## Solution Assessment

No. Current main is not the best end state for this report because it intentionally keeps auto fallback pins sticky until reset; the narrower maintainable direction is an auto-only expiry or primary-probe policy that does not weaken explicit user overrides.
