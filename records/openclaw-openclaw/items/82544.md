## Summary

Keep open. Current main still intentionally keeps auto fallback overrides selected across later turns, so the requested automatic reconciliation back to the configured primary is not implemented; a related open PR is already targeting the likely fix path.

## What This Changes

Request automatic reconciliation of auto-persisted session fallback model pins back to the configured primary, with stricter handling for invalid or empty fallback persistence.

## Best Possible Solution

Land a bounded auto-pin expiry or primary-probe policy for `modelOverrideSource: "auto"` that retries the configured primary without changing explicit user model selections.

## Reproduction Assessment

Yes for the central source path: current main persists auto fallback overrides and reuses them on later turns. The exact old `provider: null` persisted-state shape was not live-reproduced on current main.

## Solution Assessment

No, current main is not the best end state for this report because indefinite auto pins prevent recovery after the primary is healthy. A narrow expiry or reconciliation policy for auto-created pins is safer than changing user-selected strict overrides.
