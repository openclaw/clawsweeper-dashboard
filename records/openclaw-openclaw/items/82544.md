## Summary

Keep open: current main and the latest release still preserve auto-created fallback model pins across later turns, so the reported indefinite session pinning is not fixed.

## What This Changes

The issue requests automatic reconciliation of session-level fallback model pins back to the configured primary after transient fallback conditions recover.

## Best Possible Solution

Land one bounded automatic fallback recovery policy that retries the configured primary for auto-created pins while preserving explicit user model selections.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current docs, code, and tests show auto fallback overrides are persisted and reused across later turns until reset. I did not run a live Slack/provider outage repro in this read-only review.

## Solution Assessment

No, current main is not the best end state because it intentionally keeps auto pins sticky. The maintainable fix is a narrow auto-pin expiry or rollback policy, with maintainers choosing between the existing open PR approaches.
