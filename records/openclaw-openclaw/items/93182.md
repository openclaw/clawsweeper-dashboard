## What This Changes

The PR adds `-journal` to memory reindex temp cleanup discovery/deletion and adds two regression cases for orphaned rollback-journal sidecars.

## Best Possible Solution

## Solution Assessment

Yes. Adding `-journal` to both the cleanup discovery and deletion sets is the narrow owner-boundary fix; adding config, doctor migration, or broader atomic-swap changes would be heavier than this specific startup cleanup gap.
