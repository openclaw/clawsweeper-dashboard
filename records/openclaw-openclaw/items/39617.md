## Summary

Keep open. The bug is still source-reproducible for Discord and Slack on current main, but this PR cannot merge as-is because it changes legacy `src/*` channel files that no longer exist on main and lacks required real behavior proof.

## What This Changes

The branch adds fresh `loadConfig()` calls before Discord, Slack, and Telegram slash/native command route resolution in legacy `src/*` channel files.

## Best Possible Solution

Retarget the narrow fix to plugin-owned Discord and Slack native/slash command route resolution, preserve Telegram's current `runtimeCfg` flow, and add focused DM session-key regression coverage plus redacted real channel proof.

## Reproduction Assessment

Yes, by source inspection on current main. Discord native commands and Slack slash commands still route with provider-startup `cfg`, while `resolveAgentRoute` derives DM session keys from `cfg.session.dmScope`.

## Solution Assessment

No. Reloading route-sensitive config is the right direction, but this branch changes removed legacy `src/*` channel files; the maintainable fix belongs in `extensions/discord` and `extensions/slack`, while Telegram's current runtime-config path should stay intact.
