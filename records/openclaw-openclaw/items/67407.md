## Summary

Keep open: current main still lacks the requested first-class `openclaw gateway preflight [--json]` command and shared no-bind startup validation path, and the related open preflight/update/config items are broader adjacent work rather than replacements.

## What This Changes

The issue requests a Gateway CLI preflight command that validates config, context engines, plugins, and port bindability without serving traffic or mutating session state.

## Best Possible Solution

Design one shared Gateway startup-validation routine and expose it as `openclaw gateway preflight [--json]` with explicit no-bind, no-mutation, redacted-output, exit-code, docs, and tests before composing it into safe update flows.

## Risks / Open Questions

- The JSON and human preflight diagnostics could expose local paths, endpoints, SecretRefs, tokens, phone numbers, or plugin error details unless redaction is part of the initial contract.
