## Summary

Keep open: current main has adjacent config validation, doctor lint, safe restart, update hardening, and the lossless-claw fix, but it still lacks the requested first-class no-bind Gateway preflight command and shared dry-run startup validation path. The related open issues are broader or narrower, so they do not supersede this specific Gateway CLI/startup request.

## What This Changes

The issue requests `openclaw gateway preflight [--json]` to validate Gateway config, context-engine/plugin startup paths, and port availability without binding the Gateway port or touching session state.

## Best Possible Solution

Design a shared Gateway startup-validation routine and expose it as `openclaw gateway preflight [--json]` with explicit no-bind, no-mutation, redacted-output, exit-code, docs, and tests before composing it into safe update flows.
