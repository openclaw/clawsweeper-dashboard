## Summary

Keep open: current main still has adjacent startup/config checks, but it does not provide the requested first-class `openclaw gateway preflight [--json]` command or a shared no-bind/no-mutation startup validation path that exercises plugin and context-engine resolution before serving traffic.

## What This Changes

Add a Gateway preflight CLI command that validates config, plugin/context-engine startup, and port bindability without accepting connections or mutating session state.

## Best Possible Solution

Design a shared Gateway startup-validation routine and expose it as `openclaw gateway preflight [--json]` with stable redacted output, exit-code semantics, plugin/context-engine validation in a no-mutation mode, port bindability checks, and docs before composing it into update flows.

Not applicable: this is a new Gateway CLI/startup-validation feature request, not a current-main bug. Source and live GitHub searches give high confidence that the requested no-bind command is absent.

## Solution Assessment

Unclear: the requested command is a plausible operator-facing surface, but the safest implementation needs maintainer decisions on the dry-run startup boundary, JSON schema, redaction policy, and overlap with broader preflight framework work.
