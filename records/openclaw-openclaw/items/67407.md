## Summary

Keep this issue open. Current main still lacks a first-class `openclaw gateway preflight [--json]` command or a shared no-bind/no-mutation Gateway startup validation path; the related lossless-claw regression and update-race mitigations are separate fixes, not the requested primitive.

## What This Changes

Add `openclaw gateway preflight [--json]` to validate config, plugin/context-engine startup, and port bindability without accepting connections or mutating session state.

## Best Possible Solution

Add a Gateway-owned validation phase reused by startup, expose it as `openclaw gateway preflight [--json]`, and document it before layering update rollback behavior on top.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the requested CLI surface: source inspection on current main shows no `gateway preflight` registration or docs, and the normal run path still starts the real server.

## Solution Assessment

Is this the best way to solve the issue? Unclear because there is no patch yet; the requested command is the right surface, but the maintainable fix should split the real startup path into explicit no-bind/no-mutation validation instead of adding a parallel checker.
