Keep this open. Current main has adjacent config/startup safeguards and the cited lossless-claw regression is fixed, but it still lacks the requested first-class `openclaw gateway preflight [--json]` command and shared no-bind/no-mutation Gateway startup validation path.

The issue requests a first-class `openclaw gateway preflight [--json]` command that validates config, context engines, plugins, and port availability without binding the port or mutating session state.

## Best Possible Solution

Define a shared side-effect-aware Gateway startup-validation routine, expose it as `openclaw gateway preflight [--json]`, document the operator contract, and compose safe update flows on top after the primitive is stable.
