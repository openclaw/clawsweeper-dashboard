## Summary

Keep open: current main has adjacent config, safe-restart, and startup checks, but it still lacks the requested first-class no-bind Gateway preflight command and shared dry-run startup validation path.

## What This Changes

The issue requests `openclaw gateway preflight [--json]` to validate config, engine/plugin startup, and port bindability without binding the server port or mutating session state.

## Best Possible Solution

Design a shared Gateway startup-validation routine and expose it as `openclaw gateway preflight [--json]` with explicit no-bind, no-mutation, redacted-output, exit-code, docs, and test contracts before composing it into safe update flows.
