## Summary

Keep open. The PR addresses part of the Control UI live-stream artifact bug, but the proposed renderer fix still leaves committed cumulative stream segments visible in multi-tool turns, and the user-facing fix is missing the required changelog entry.

## What This Changes

The PR updates Control UI chat rendering and chat event handling to trim committed stream prefixes, clear live tool-stream artifacts on terminal chat events, and add unit coverage for stream trimming plus terminal cleanup.

## Best Possible Solution

Land a narrow Control UI fix that delta-normalizes cumulative assistant text across committed stream segments and the live stream tail, keeps terminal cleanup semantics for final/aborted/error events, adds a two-tool regression, and records the user-facing fix in the active changelog.

## Reproduction Assessment

Yes. The linked issue gives concrete multi-tool WebChat steps, and the source path matches it: tool starts snapshot `chatStream` into `chatStreamSegments` while gateway chat deltas contain merged cumulative text. A focused renderer input with `segments=[A, A+B]` and a later live suffix reproduces the remaining duplicate segment.

## Solution Assessment

No. Clearing terminal artifacts is useful, but trimming only the final live stream tail is not the narrow complete fix because already-committed cumulative segment cards still render raw. The safer solution normalizes the segment sequence before rendering it.
