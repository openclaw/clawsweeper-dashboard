## Summary

Keep open. The PR moves in the right Control UI area and now covers terminal cleanup for final/aborted/error events, but it still does not fully fix the linked multi-tool cumulative rendering bug because committed stream segment cards remain unnormalized. The user-facing fix also needs a changelog entry before it is release-ready.

## What This Changes

The PR updates Control UI chat rendering/controller code to trim committed stream prefixes from live stream text, clear live tool messages and stream segments on terminal chat events, and add unit coverage for terminal cleanup plus one committed-segment trim case.

## Best Possible Solution

Land a narrow Control UI fix that delta-normalizes cumulative assistant text across both committed `streamSegments` and the live stream tail at tool boundaries, keeps the terminal cleanup/reload semantics, adds multi-tool and mirrored-bubble regression coverage, and records the user-facing fix in `CHANGELOG.md`.

## Reproduction Assessment

Yes. The linked issue gives concrete multi-tool WebChat steps and hard-refresh proof, and the source path reproduces the state shape: cumulative gateway deltas can be committed into multiple stream segment cards before rendering.

## Solution Assessment

No. Clearing terminal artifacts is useful, but trimming only `props.stream` is not the best complete fix because already-committed cumulative segment cards still render raw; the safer fix normalizes the segment sequence as well.
