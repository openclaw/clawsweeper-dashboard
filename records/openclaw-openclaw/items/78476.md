## Summary

Keep open: current main still lacks a Google Home/Home APIs Android bridge, the existing Home Assistant add-on/ClawHub skill is only an alternative route, and the related node-tool work covers generic model-visible routing rather than Android SDK, OAuth consent, and physical-device safety decisions.

## What This Changes

The issue requests native Google Home API support in the Android app so Talk can control Google Home devices without requiring Home Assistant as the bridge.

## Best Possible Solution

Maintainers should first choose the product boundary: a generic Android/local capability contract plus plugin/ClawHub ownership is safer unless direct Google Home support earns a core Android exception with explicit OAuth, revocation, low-risk trait allowlists, audit, and high-risk-device confirmation policy.
