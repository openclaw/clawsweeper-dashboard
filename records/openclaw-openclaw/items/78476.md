## Summary

Keep open. Current main has Android node and Talk plumbing, but no Google Home/Home APIs dependency, permission flow, capability, or command surface; the request also needs mobile SDK, consent, safety, and routing product decisions.

## What This Changes

Add native Google Home API support to the Android app so Talk can control a user’s Google Home devices without Home Assistant.

## Best Possible Solution

Define a maintainer-approved smart-home capability boundary first, decide first-party Android versus plugin/node ownership, then add Google Home behind explicit consent, low-risk defaults, and sensitive-action confirmations.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior. Source review against current main shows Android node/Talk support exists, but no Google Home bridge exists to reproduce.

## Solution Assessment

Unclear as an implementation path. A native Android bridge is plausible, but the maintainable solution needs product design around Android SDK ownership, normalized smart-home schema, node/Talk routing, and dangerous-action confirmations before a patch is safe.
