## Summary

Keep open. Current main has Android node and Talk plumbing, but no Google Home/Home APIs dependency, permission flow, capability, or command surface; the request also needs mobile SDK, safety, and routing product decisions before implementation.

## What This Changes

Add native Google Home API support to the Android app so Talk can discover and control authorized Google Home devices without using Home Assistant as the bridge.

## Best Possible Solution

Define a maintainer-approved smart-home capability boundary first, then decide whether Google Home belongs in the first-party Android app or a separate node/plugin path with explicit consent, safety confirmation, and node/Talk routing semantics.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior. Source review against current main shows Android node/Talk support exists, but no Google Home bridge exists to reproduce.

## Solution Assessment

Unclear as an implementation path. The direction is plausible, but the best maintainable solution needs product design around Android SDK ownership, normalized smart-home schema, node/Talk routing, and dangerous-action confirmations before a patch is safe.
