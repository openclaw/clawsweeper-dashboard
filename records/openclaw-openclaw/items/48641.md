## Summary

Keep this open. Current main still resolves Discord DM allowlists with account-level precedence, and the linked union-style fix PR was closed unmerged; changing that behavior can widen DM access, so it needs an explicit maintainer decision rather than cleanup closure.

## What This Changes

The issue asks for inbound Discord DMs from an allowlisted user in a multi-account setup to be routed instead of being dropped while outbound DMs and guild messages continue to work.

## Best Possible Solution

Define the intended Discord multi-account DM allowlist contract, then update the Discord plugin code, tests, docs, and changelog to match that contract without accidentally widening access.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the narrowed source path: when a selected Discord account has its own `allowFrom`, the top-level list is ignored and DM preflight can drop the sender before routing. The full live four-account Discord setup was not reproduced.

## Solution Assessment

Is this the best way to solve the issue? Unclear: the linked PR's union approach is narrow, but it changes access-control semantics. The safer path is maintainer-approved documentation of whether global `allowFrom` is inherited, unioned, or overridden per account.
