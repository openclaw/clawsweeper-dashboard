## Summary

Keep this open. Current main still resolves Discord DM allowlists with account-local values ahead of root values, denied DMs return before activity/routing, and the linked union-style fix was closed unmerged because the access semantics need more design work.

## What This Changes

The issue reports that allowlisted inbound Discord direct messages are silently dropped in a multi-account setup while outbound direct messages and guild messages still work.

## Best Possible Solution

Define the intended Discord multi-account DM access contract, then align account/root allowlist resolution, preflight diagnostics, tests, docs, and changelog without unintended access widening.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for the full live four-account Discord setup; yes at source level for the narrowed path where account-local allowFrom overrides root allowFrom and denied DMs return before routing.

## Solution Assessment

Is this the best way to solve the issue? Unclear: unioning root and account allowFrom is plausible, but it changes access-control semantics, so maintainers should choose inherit, override, union, or explicit DM-receiving-account behavior first.
