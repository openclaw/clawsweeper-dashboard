## Summary

Keep this open. Current main has better shared ingress authorization, but Discord still resolves the account-scoped DM allowlist before that gate, the union-style fix was closed unmerged, and the remaining behavior needs an explicit access-control contract decision.

## What This Changes

The issue asks OpenClaw to stop silently dropping inbound Discord DMs from allowlisted users in multi-account Discord configurations where outbound DMs and guild messages work.

## Best Possible Solution

Define the intended Discord multi-account DM access contract, then align account/root allowlist resolution, preflight diagnostics, tests, docs, and changelog without unintended access widening.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for the full reported four-account live Discord setup; yes at source level for the narrowed path where account-level allowFrom overrides root allowFrom and denied DMs return null before routing.

## Solution Assessment

Is this the best way to solve the issue? Unclear: unioning global and account allowFrom is plausible, but it changes access-control semantics, so maintainers should decide inherit, override, union, or explicit DM-receiving-account behavior first.
