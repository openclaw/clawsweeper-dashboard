## Summary

Keep open: current main and the latest release still preserve account-scoped Discord `allowFrom` override precedence, while the only linked fix PR was closed unmerged because the union-versus-override authorization decision needs maintainer design.

## What This Changes

The issue requests reliable inbound Discord DM routing for allowlisted users when global and account-scoped `allowFrom` settings are present in a multi-account Discord setup.

## Best Possible Solution

Make an explicit maintainer/security decision for Discord account allowlist inheritance, then align code, docs, tests, and live DM proof around that chosen behavior.

## Reproduction Assessment

Yes, partially: current source clearly reproduces the account-override path where a root allowlist entry is not effective once the account has its own `allowFrom`. The exact multi-account live Discord setup was not reproduced in this read-only review.

## Solution Assessment

Unclear. The closed union PR is a plausible repair, but not proven best because it widens authorization; the best fix depends on a maintainer decision about whether root allowlists should be inherited, merged, or documented as overridden.
