## Summary

Keep open: current main and the latest release still preserve account-scoped Discord `allowFrom` precedence, the linked fix PR was closed unmerged over the union-vs-override authorization decision, and the adjacent open DM-read PRs do not fix inbound DM admission.

## What This Changes

The issue requests fixing Discord inbound DMs from allowlisted users being silently dropped in multi-account allowlist setups while outbound DMs and guild messages still work.

## Best Possible Solution

Make the Discord DM allowlist inheritance contract explicit, then either keep account override semantics with better diagnostics or implement a tested, security-reviewed root-plus-account inheritance rule inside the Discord plugin.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Medium-confidence source reproduction exists for the account-precedence portion: current code and tests show account `allowFrom` can override top-level `allowFrom` before Discord DM preflight authorization; no live Discord DM repro was run.

## Solution Assessment

Is this the best way to solve the issue? Unclear. The proposed root-plus-account union is plausible for the reported configuration, but it changes an authorization boundary that current tests and docs preserve as account-level precedence.
