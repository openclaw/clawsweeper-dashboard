## Summary

Keep open: current main and v2026.6.8 still preserve Discord account-scoped `allowFrom` override semantics, while the linked union fix was closed unmerged because the authorization model needs maintainer/security direction.

## What This Changes

The issue reports inbound Discord DMs from an allowlisted user being silently dropped in a multi-account setup while outbound DMs and guild messages still work.

## Best Possible Solution

Make an explicit maintainer/security decision for Discord account allowlist inheritance, then align code, docs, tests, and live multi-account DM proof around that chosen behavior.

## Reproduction Assessment

Yes, partially: current source reproduces the account-override path where a root allowlist entry stops being effective once the account has its own `allowFrom`. The exact multi-account live Discord setup was not reproduced in this read-only review.

## Solution Assessment

Unclear. The closed union PR is a plausible repair, but it is not proven best because it widens authorization; the best fix depends on a maintainer decision about inherited, merged, or override-only allowlists.
