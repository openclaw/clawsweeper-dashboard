## Summary

Keep open: current main and v2026.6.10 still use account-scoped Discord DM allowlist precedence, and the only direct fix PR was closed unmerged because union-versus-override authorization semantics need maintainer/security design.

## What This Changes

The issue reports allowlisted Discord inbound DMs being silently dropped in a multi-account setup while outbound DMs and guild messages still work.

## Best Possible Solution

Keep this issue as the canonical design/bug report, have maintainers decide whether root Discord `allowFrom` is a shared baseline or only a fallback default, then implement the chosen contract with docs, focused tests, and live Discord DM proof.

## Reproduction Assessment

Yes, source-reproducible but not live-reproduced here. Current main still lets account-level Discord `allowFrom` override root `allowFrom`, and that effective list is what DM preflight uses before routing inbound DMs.

## Solution Assessment

Unclear as a final behavior choice. Merging root and account allowlists is a plausible fix for the report, but maintainers need to choose the authorization contract because it can widen access for account-scoped configurations.
