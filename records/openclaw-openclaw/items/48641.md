## Summary

Keep this open. Current main and the latest release still preserve account-scoped Discord DM allowlist precedence, the attempted union-style fix was closed unmerged after the author called out unresolved semantics, and the exact multi-account DM drop is not explained by a shipped fix.

## What This Changes

The issue reports that allowlisted Discord direct messages are silently dropped while outbound DMs and guild messages still work.

## Best Possible Solution

Define the intended Discord multi-account DM allowlist contract, including inherit, override, union, or explicit DM-receiving account behavior, then align code, tests, docs, and changelog without unintended access widening.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for the full reported four-account Discord setup; yes at source level for the narrowed path where account-level `allowFrom` overrides root `allowFrom` and denied direct messages return `null` before routing.

## Solution Assessment

Is this the best way to solve the issue? Unclear: unioning global and account `allowFrom` is plausible, but it changes access-control semantics, so the safer path is an explicit maintainer decision on the contract first.
