## Summary

Keep this open. Current main still preserves account-local Discord DM allowlist precedence, denied DMs return before activity/routing, and the linked union-style fix was closed unmerged because the access semantics need maintainer design work.

## What This Changes

The issue reports that inbound Discord direct messages from an allowlisted user are dropped silently in a multi-account Discord setup while outbound direct messages and guild-channel messages still work.

## Best Possible Solution

Define the intended Discord multi-account DM access contract, then align account/root allowlist resolution, preflight diagnostics, tests, docs, and changelog without unintended access widening.

## Reproduction Assessment

No for the full live four-account Discord setup; yes at source level for the narrowed path where account-local allowFrom overrides root allowFrom and denied DMs return before routing.

## Solution Assessment

Unclear. Unioning root and account allowFrom is plausible, but it changes access-control semantics, so maintainers should choose inherit, override, union, or explicit DM-receiving-account behavior first.
