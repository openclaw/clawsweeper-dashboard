- `impact:message-loss`: The reported failure silently drops inbound Discord direct messages before they reach an agent session.
- `impact:security`: The likely repair changes Discord direct-message allowlist authorization semantics and could widen or restrict account-scoped access.

## Summary

Keep open: current main and the latest release still preserve account-local Discord DM allowlist precedence, denied DMs can return before inbound activity is recorded, and the linked union-style fix was closed unmerged because the authorization semantics need maintainer design review.

## What This Changes

Discord direct messages from allowlisted users should be logged, routed to the agent, and answered in multi-account setups instead of being silently dropped.

## Best Possible Solution

Define the intended Discord multi-account DM access contract, then align allowlist resolution, denial diagnostics, tests, docs, and live Discord proof without widening account-scoped authorization unintentionally.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for the full live four-account Discord setup; source inspection does show a narrower current-main path where account-local allowFrom can override root allowFrom and denied DMs return before inbound activity logging.

## Solution Assessment

Is this the best way to solve the issue? Unclear: unioning root and account allowFrom would address one hypothesis, but it changes access-control semantics, so maintainers should choose the contract before implementation.
