## Summary

Keep open: current main and v2026.6.10 still preserve account-scoped Discord allowFrom precedence, and the direct fix PR was closed unmerged because union-versus-override authorization semantics need maintainer/security design.

## What This Changes

The issue requests that allowlisted inbound Discord DMs in multi-account configurations be logged and routed instead of silently dropped while outbound DMs still work.

## Best Possible Solution

Have maintainers define the Discord DM account-inheritance and multi-account ownership contract, then implement it in the Discord plugin with focused precedence tests, docs updates, and live DM proof.

## Reproduction Assessment

Partly. Source inspection shows account-level allowFrom takes precedence over root allowFrom and denied DMs return null after only verbose logging, but I did not establish a live current-main Discord DM repro for the exact multi-account report.

## Solution Assessment

No. The closed union PR was plausible but not clearly the best fix because it would widen account-scoped authorization; maintainers should choose the security contract before a repair branch changes behavior.
