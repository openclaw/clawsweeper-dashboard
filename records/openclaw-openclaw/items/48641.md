## Summary

Keep open: current main and v2026.6.10 still use account-scoped Discord DM allowlist precedence rather than merging global and account lists, and the direct fix PR was closed unmerged because union-versus-override authorization semantics need maintainer/security design.

## What This Changes

The issue reports that allowlisted inbound Discord direct messages are silently dropped while outbound DMs and guild messages still work.

## Best Possible Solution

Have maintainers choose the Discord account allowlist contract first, then implement that contract with focused account-resolution tests, docs updates, and live Discord DM proof if behavior changes.

## Reproduction Assessment

Partly. Source inspection reproduces the proposed root cause that account-scoped allowFrom overrides root allowFrom, but the exact live DM drop from the report has not been reproduced on current main.

## Solution Assessment

No, a simple union of global and account allowFrom is not yet proven to be the best fix because it changes account-scoped authorization semantics; the safer next step is maintainer/security agreement on the intended contract.
