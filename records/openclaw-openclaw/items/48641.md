## Summary

Keep open: current main still preserves Discord account-local DM allowlist precedence, denied DMs return before normal inbound activity/routing, and the linked union-style fix was closed unmerged because the access-control semantics need maintainer review.

## What This Changes

Discord inbound DMs from an allowlisted user are reported as silently dropped while outbound Discord DMs and guild channel messages work.

## Best Possible Solution

Define the intended Discord multi-account DM access contract, then align allowlist resolution, diagnostics, tests, docs, and live proof without unintentionally widening account-scoped access.

## Reproduction Assessment

No for the full live four-account Discord setup; yes at source level for the narrower path where account-local allowFrom overrides root allowFrom and a denied DM returns before routing or normal inbound activity logging.

## Solution Assessment

Unclear: unioning root and account allowFrom would address one narrowed failure mode, but it changes access-control semantics, so maintainers should choose override, union, inheritance, or explicit DM-receiving-account behavior first.
