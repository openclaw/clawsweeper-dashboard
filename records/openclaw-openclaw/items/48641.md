## Summary

Keep open: current main and v2026.6.8 still preserve account-scoped Discord `allowFrom` override behavior, while the direct fix PR was closed unmerged because union-versus-override authorization semantics need maintainer design.

## What This Changes

Discord DMs from an allowlisted user are reported as silently dropped in multi-account Discord setups while outbound DMs and guild messages work.

## Best Possible Solution

Have a maintainer choose the Discord multi-account allowlist contract, then implement that contract with focused source tests, docs, and live multi-account DM proof.

## Reproduction Assessment

No, not for the exact live setup. Source inspection gives a medium-confidence reproduction shape for account-scoped `allowFrom` overriding top-level `allowFrom`, but I did not live-reproduce the Discord DM drop.

## Solution Assessment

No. The previous union-list PR was a plausible code fix, but it broadens an authorization boundary, so maintainer/security direction on the intended account inheritance contract is needed first.
