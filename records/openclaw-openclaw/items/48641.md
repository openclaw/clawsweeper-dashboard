## Summary

Keep open: current main and the latest release still make Discord DM admission depend on the effective account-scoped allowlist, and current tests preserve account override semantics. The linked fix PR was closed unmerged because root-plus-account allowlist merging needs design/security judgment, while the newer Discord DM read issue tracks a different path.

## What This Changes

The issue reports inbound Discord DMs from an allowlisted user being silently dropped in a multi-account allowlist setup while outbound DMs and guild messages work.

## Best Possible Solution

Make a maintainer-owned Discord DM authorization decision for root versus account allowFrom inheritance, then land a focused fix with regression tests and live multi-account DM proof that does not widen account-scoped access unexpectedly.

## Reproduction Assessment

Yes for the account/global allowlist mismatch path: current source and tests show account-scoped allowFrom overrides top-level allowFrom, and DM preflight checks only the effective list. The exact four-account live setup was not reproduced in this read-only review.

## Solution Assessment

No, the closed merge-all allowFrom PR was only a plausible fix because it could widen account-specific allowlists. The best fix needs a maintainer/security decision on account inheritance semantics before code changes.
