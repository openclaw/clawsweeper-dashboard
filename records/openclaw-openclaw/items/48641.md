## Summary

Keep this issue open. Current main still resolves Discord DM allowlists with account-level precedence over the top-level list, the docs and tests preserve that behavior, and the only related implementation PR was closed unmerged. Because the likely fix can broaden DM authorization across accounts, this should stay with maintainers for an explicit access-control decision.

## What This Changes

The issue requests fixing Discord direct-message inbound routing for allowlisted users in multi-account setups where outbound DMs and guild messages work but inbound DMs are silently dropped.

## Best Possible Solution

Define the intended multi-account Discord DM allowlist contract, then update the Discord plugin, regression tests, docs, and changelog to match. If top-level allowFrom is meant to be shared, implement a deduped union or clearly documented inheritance rule; if account overrides are intentional, improve diagnostics so rejected DMs are not silent and users can see the effective account allowlist.

## Reproduction Assessment

Yes: configure channels.discord.allowFrom with the sender, configure the selected account with a different allowFrom, and current main resolves only the account list before DM preflight authorization. The reporter's exact four-bot live account-order path is not independently live-verified.

## Solution Assessment

Unclear: the direct union from the closed PR likely fixes the reported drop, but it may widen access for accounts that currently restrict DMs. The safer maintainable path is to choose and document the security semantics before changing runtime behavior.
