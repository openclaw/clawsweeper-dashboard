## Summary

Keep open: current main and v2026.6.8 still use account-scoped Discord `allowFrom` precedence, and the direct fix PR was closed unmerged because union-versus-override authorization semantics need maintainer/security design.

## What This Changes

Discord allowlisted inbound DMs are reported as silently dropped while outbound DMs and guild messages work.

## Best Possible Solution

Keep this issue as the canonical design/bug report, have maintainers decide whether top-level Discord `allowFrom` is a shared baseline or a default that account-level lists may restrict, then implement the chosen behavior with docs, focused tests, and live Discord proof.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection does show that current main and v2026.6.8 still use account-level `allowFrom` before root `allowFrom`, which reproduces the suspected override failure shape.

## Solution Assessment

Unclear as currently proposed: unioning global and account allowlists is a plausible fix for one failure mode, but it changes authorization semantics. The safer path is a maintainer/security decision plus docs/tests for the chosen inheritance contract.
