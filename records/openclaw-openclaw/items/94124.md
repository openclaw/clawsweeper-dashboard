## Label Justifications

- `P1`: The report describes active subagent runs being killed by the idle watchdog during real tool-heavy workflows before their total run budget expires.
- `impact:session-state`: The affected behavior can terminate an in-progress subagent session and lose the state needed to produce the final report.

## Summary

Keep this issue open: current main still resets the embedded runner idle watchdog only on model/LLM request activity, while the open candidate fixes at https://github.com/openclaw/openclaw/pull/95536 and https://github.com/openclaw/openclaw/pull/94133 are not merged.

## What This Changes

Reset the embedded subagent LLM idle watchdog on successful tool activity so long-running tool calls do not prematurely terminate an active subagent run.

## Best Possible Solution

Review and land the strongest run-scoped tool-activity heartbeat fix with focused timeout tests and real greater-than-120-second subagent proof, then close this issue through the merged PR.

## Reproduction Assessment

Yes, source-reproducible: current main arms the LLM idle watchdog and listens only for LLM request activity while tool completion is emitted separately. I did not run a live greater-than-120-second subagent reproduction in this read-only review.

## Solution Assessment

Yes on direction: a run-scoped heartbeat at the shared tool-completion boundary is the narrow maintainable fix, with no new config surface. This issue should remain open until an open candidate PR is reviewed, proven, and merged.
