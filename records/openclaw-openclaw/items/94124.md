## Label Justifications

- `P1`: The report describes active subagent runs being terminated before producing their final output, which breaks an agent workflow for real research-style use.
- `impact:session-state`: The failure is tied to embedded session ownership and idle timeout behavior that can terminate an active subagent session.
- `impact:message-loss`: A killed subagent can lose the final report it was about to return after collecting data.

## Summary

Keep open: current main and the latest release still lack a tool-activity heartbeat for the embedded-runner LLM idle watchdog, and the open candidate fix is blocked on review findings and real behavior proof.

## What This Changes

The issue asks for successful subagent tool calls to refresh the embedded-runner idle timeout so active research runs are not terminated before their final response.

## Best Possible Solution

Land a narrowed fix that records run-scoped tool activity from the shared tool execution path and keeps the idle watchdog alive only while model or tool progress is active, with focused regression tests and redacted real subagent proof.

## Reproduction Assessment

Yes by source inspection, but not by a live run: current main has a 120s provider idle watchdog with no tool-activity signal, while shared tools execute between model turns. A real long-tool subagent run should still be used as proof for the fix.

## Solution Assessment

No complete fix is on main yet. The proposed direction is plausible, but the open candidate needs to move the heartbeat to shared run-scoped tool activity and add tests plus real behavior proof before it is the best solution.
