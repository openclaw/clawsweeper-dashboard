## Summary

Keep open. Current main and v2026.6.9 still use the LLM idle watchdog without a tool-activity heartbeat, and https://github.com/openclaw/openclaw/pull/94133 is an open, unmerged candidate fix rather than landed behavior.

## What This Changes

The issue asks OpenClaw to refresh subagent LLM idle timeout activity when tool calls complete, so tool-heavy `sessions_spawn` runs are not killed while still within their run budget.

## Best Possible Solution

Land a corrected fix through the linked candidate PR or a replacement PR that records run-scoped tool activity across direct and deferred tool execution, with focused idle-timeout tests and redacted real subagent proof.
