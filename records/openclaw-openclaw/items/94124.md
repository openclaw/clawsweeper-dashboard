## Summary

Keep open: the issue has an explicit open candidate fix PR, and current main plus the latest release still only reset the LLM idle watchdog on provider-stream activity, not on completed tool work. The linked PR should remain the fix path until it is repaired, proven, and merged or closed.

## What This Changes

The issue requests that subagent idle timeout accounting be refreshed when tool calls complete so long-running research subagents are not killed while still within their run budget.

## Best Possible Solution

Keep this issue as the canonical tracker and resolve it through a corrected, proven version of https://github.com/openclaw/openclaw/pull/94133 or a replacement fix that covers the same timeout path.

## Reproduction Assessment

Yes, source-level only: current main has an LLM idle watchdog with no tool-completion reset path, so a native subagent using a cloud-provider idle watchdog can still time out while a long direct tool call prevents model tokens. I did not run a live subagent timeout reproduction in this read-only review.

## Solution Assessment

Yes for the direction, not for the current candidate implementation: the best fix is a run-scoped or shared tool-completion activity signal wired through direct and cataloged tool execution, with focused timeout tests and real behavior proof before merge.
