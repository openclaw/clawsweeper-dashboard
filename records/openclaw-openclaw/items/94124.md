## Label Justifications

- `P1`: The issue describes active subagent runs being terminated by the idle watchdog during tool-heavy workflows, which can break real agent work.
- `impact:session-state`: The failure can terminate an in-progress subagent session before it produces its final result.

## Summary

Keep open: current main and the latest release still lack a tool-activity heartbeat for the embedded-runner LLM idle watchdog, and the issue is paired with open unmerged fix candidates.

## What This Changes

The issue requests resetting the embedded subagent idle watchdog from successful tool activity so active long-running tool work is not terminated as model idleness.

## Best Possible Solution

Land one run-scoped heartbeat fix that preserves tool metadata, covers direct and cataloged tool completion paths, includes focused regression tests plus real subagent proof, and then close this issue through the merged PR.

## Reproduction Assessment

No live current-main reproduction was run in this review. Source inspection does show the relevant path: current main has a 120s LLM idle watchdog and no tool-activity heartbeat tying completed tool work back to that watchdog.

## Solution Assessment

Yes on direction, but the issue should close only after a merged PR. The best implementation is a run-scoped heartbeat at the shared tool-completion boundary, with metadata preservation and proof, rather than adding a new config knob.
