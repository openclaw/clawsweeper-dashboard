## Label Justifications

- `P1`: Prematurely terminating active subagent runs can break real multi-tool agent workflows before they return a final result.
- `impact:session-state`: The reported failure kills a live subagent session while it is still within its configured run budget.
- `impact:message-loss`: The user-visible failure mode is losing the subagent’s final report after tool work has already completed or nearly completed.

## Summary

Keep open: current main still has a model idle watchdog with no main-branch tool-completion heartbeat, and the same author has an open PR that directly claims to fix this issue. The issue should remain open until that PR lands or maintainers choose a different focused implementation.

## What This Changes

The issue asks for subagent session idle timeout activity to be refreshed when long-running tool calls complete so research subagents are not killed mid-run.

## Best Possible Solution

Keep this issue open and resolve it by reviewing and landing https://github.com/openclaw/openclaw/pull/94133 or an equivalent focused fix that wires tool-completion activity into the idle watchdog with regression coverage.

## Reproduction Assessment

Yes from source at medium confidence: the current idle watchdog resets on model chunks or `notifyLlmRequestActivity`, while tool lifecycle completion has no matching idle-watchdog notification on main. I did not run a live 120-second subagent scenario.

## Solution Assessment

Unclear for the exact linked PR: a no-config activity hook is the right kind of fix for this narrow bug, but the PR still needs review of the signal boundary and behavior proof before it can be called the best implementation.
