## Label Justifications

- `P1`: The report describes active subagent runs being terminated by the idle watchdog during tool-heavy workflows.
- `impact:session-state`: The affected behavior can kill an in-progress subagent session before it returns the final state/report.

## Summary

This issue should stay open: current main and the latest release still have the LLM idle watchdog wired only to stream/LLM request activity, while the requested tool-activity heartbeat is being handled by open, unmerged fix PRs.

## What This Changes

The issue requests a tool-activity heartbeat so successful subagent tool calls reset the embedded-agent LLM idle timeout instead of letting active subagents die after 120 seconds.

## Best Possible Solution

Land a run-scoped heartbeat at the shared tool-completion boundary, with focused idle-timeout tests and real subagent proof, then close this issue through the merged fix PR.

## Reproduction Assessment

Yes from source inspection at medium confidence: current main has a provider-stream idle watchdog and separate LLM request heartbeat, but no current-main tool-completion heartbeat was found. I did not run a live 120-second subagent repro in this read-only review.

## Solution Assessment

Yes in direction: the maintainable fix is a run-scoped heartbeat at the shared tool-completion boundary. The newer open PR appears to be the better candidate, but this issue should remain open until a fix is proven and merged.
