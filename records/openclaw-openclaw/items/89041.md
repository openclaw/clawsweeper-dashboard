## Summary

Keep open: the PR is still necessary because current main and the latest release still construct Discord gateway sockets with the ws 8.21.0 retained-part defaults, and the branch applies a narrow Discord-only mitigation with no blocking line-level findings. ## Best Possible Solution

## Solution Assessment

Yes, this is the narrowest mitigation I found: it scopes the opt-out to Discord gateway sockets instead of downgrading `ws` or changing all WebSocket users. It still needs maintainer acceptance because disabling a security-motivated limit is an intentional availability/memory tradeoff.
