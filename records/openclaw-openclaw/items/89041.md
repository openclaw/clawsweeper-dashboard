## Summary

Keep open: this PR is still the focused candidate fix for the open Discord `ws` 8.21 receiver-limit gateway regression, and current `main` plus the latest release still use the new `ws` defaults on both Discord gateway socket paths. ## What This Changes

The PR sets `maxBufferedChunks: 0` and `maxFragments: 0` on both Discord gateway WebSocket constructors and adds focused tests for the base, monitored, and proxy-aware paths.

## Best Possible Solution

Maintainers should either land this Discord-only receiver-limit exception as the linked fix, or ask for explicit finite caps if they want to keep a receiver-side resource ceiling.

## Solution Assessment
