## Summary

Keep open. Current main and v2026.6.10 partially cover usage visibility with a context indicator, provider usage pill, Usage page, and per-message token/cost metadata, but they still do not implement the requested OpenRouter/LLMAPI/Ollama quota semantics, richer chat widget, or low-credit alert policy.

## What This Changes

Request to show persistent token usage, remaining quota, estimated cost, per-message usage, and provider-specific quota semantics in the Control UI chat interface.

## Best Possible Solution

Define provider-owned quota/cost and alert semantics first, then reuse the existing chat context notice, provider usage pill, per-message metadata, and Usage page surfaces instead of adding a parallel chat-only accounting path.

## Reproduction Assessment

Not applicable: this is a feature request, not a report of broken documented behavior. Source inspection gives high-confidence gap proof because current main and the latest release partially show usage but do not implement the requested provider-specific quota and alert behavior.

## Solution Assessment

Unclear as a single implementation path: the UX is reasonable, but the maintainable fix needs provider-owned quota semantics and UI alert policy first. Reusing the existing provider usage hooks and chat/session usage surfaces is safer than duplicating usage polling in chat.
