## Summary

Keep this PR open, but it is not merge-ready: the bridge export direction is useful and the live terminal proof shows the intended local behavior, but the patch adds an unconditional filesystem export path that breaks documented URI-backed LanceDB storage and can write private memory artifacts to an unintended local path.

## What This Changes

The PR moves shared memory workspace artifact listing into the core memory host SDK, registers a LanceDB Markdown artifact plus workspace artifacts from memory-lancedb, preserves discovered memory capabilities for activate:false loads, and makes native wiki links extensionless.

## Best Possible Solution

Keep the shared artifact seam, but make the LanceDB table export URI-safe by using a real local export/cache path or by clearly skipping table export for non-file dbPath while still preserving workspace memory artifacts.

Yes from source inspection, though I did not execute a live repro in this read-only review. Current main has memory-wiki reading active memory public artifacts while memory-lancedb lacks a provider, and the PR's own test/proof exercises the intended local repaired path.

## Solution Assessment

No, not yet. The ownership move and artifact seam are maintainable, but the LanceDB export must handle URI-backed dbPath safely before this is the best fix.
