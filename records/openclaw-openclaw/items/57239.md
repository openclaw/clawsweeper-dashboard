## What This Changes

The PR adds a fast-lane path for short active-session follow-ups by creating transient run/lane state and a queue-policy override, with queue-policy unit coverage.

## Best Possible Solution

Land a maintainer-approved quick-reply or concurrency design that isolates reply operations and session state, preserves ordering, enforces runtime tool controls, and includes real active-run proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an eligible fast-lane message still passes the parent sessionKey to runReplyAgent, and current main rejects a second active ReplyOperation for that key.

## Solution Assessment

No. A fast lane may be useful, but this implementation is not the best mergeable solution until reply-operation identity, ordering/state isolation, hard tool controls, and real behavior proof are addressed.
