## Summary

Keep open: the PR targets a real latency feature, but it is not mergeable against current main because the proposed fast-lane path still collides with current reply-run identity checks and relies on prompt-only tool suppression for a parallel run.

## What This Changes

The branch adds a generic auto-reply fast-lane for short messages during active runs, with a force-run-now queue-policy flag, transient session/lane handling, cleanup, and queue-policy tests.

## Best Possible Solution

Land a maintainer-approved design that either extends the proven /btw side-result model or adds a documented fast-lane concurrency seam with isolated reply identity, hard tool controls, ordering guarantees, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: during an active normal run, an eligible fast-lane message keeps the parent sessionKey when calling runReplyAgent, and current main rejects a second ReplyOperation for that active key.

## Solution Assessment

No. The PR is a plausible direction, but it is not the best mergeable solution until reply-operation identity, tool disabling, ordering, and session state isolation are hard guarantees rather than prompt or lane-only conventions.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Use an isolated reply operation key:** `src/auto-reply/reply/get-reply-run.ts:672`
  - body: The fast-lane path creates an isolated queue/lane, but it still passes the parent sessionKey into runReplyAgent. On current main, runReplyAgent creates a ReplyOperation for that key and rejects it while the parent run is active, so the fast-lane message returns the existing still-shutting-down response instead of running immediately.
  - confidence: 0.88
- **[P2] Disable tools for fast-lane runs:** `src/auto-reply/reply/get-reply-run.ts:647-649`
  - body: The parallel fast-lane turn only adds a system prompt asking the model not to call tools. Since the embedded execution path has a hard disableTools flag and this path does not set it, the fast-lane run can still invoke privileged tools concurrently with the parent session.
  - confidence: 0.84
