## Summary

Keep open: current main and the latest release still gate behavior-override carryover on explicit reset commands, so an implicit daily/idle rollover can drop `/think`, `/verbose`, `/reasoning`, `/trace`, or auto-TTS session state. The open fix path is https://github.com/openclaw/openclaw/pull/92575, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Preserve user-set behavior overrides across implicit daily/idle session rollover the same way explicit `/new` already does.

## Best Possible Solution

Land the narrow `initSessionState` carryover fix and regression coverage from https://github.com/openclaw/openclaw/pull/92575 or an equivalent patch, then close this issue as implemented.

## Reproduction Assessment

Yes, source-reproducible: seed a stale session entry with `thinkingLevel: "medium"`, send a normal message through `initSessionState`, and current main takes the new-session branch with `resetTriggered === false` while skipping the behavior carryover block. I did not run tests because this cleanup review is read-only.

## Solution Assessment

Yes: the best fix is to carry existing behavior override fields across any new-session mint from an existing entry, matching the already-merged model/auth rollover pattern while keeping CLI/spawn lineage gated to explicit resets.
