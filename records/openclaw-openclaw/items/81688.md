## What This Changes

## Best Possible Solution

Keep the narrow Slack diagnostic, fix the doctor collector to mirror account-level name-matching inheritance, and require real behavior proof without changing ID-first routing semantics.

## Solution Assessment

No, not yet: the runtime warning location is reasonable, but the doctor warning must respect account-level `dangerouslyAllowNameMatching: false` overrides and the PR needs accepted real behavior proof before merge.
