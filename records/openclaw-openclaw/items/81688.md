## Summary

Keep this PR open: it is a focused fix for the linked Slack silent allowlist diagnostic and the supplied terminal proof is sufficient, but the startup warning is skipped when Slack channel resolution throws and runtime falls back to the original name-keyed entries.

## What This Changes

The PR adds Slack doctor and startup diagnostics for name-keyed channel allowlist routes, plus focused Slack doctor and provider tests.

## Best Possible Solution

Land Slack-owned doctor and startup diagnostics that warn for non-ID allowlist route keys whenever they will be ignored, including Slack API resolution failure, without enabling name matching or rejecting config.

## Reproduction Assessment

Yes, source-reproducible: current routing ignores name-keyed routes by default, and the PR's new startup warning only runs after successful channel resolution. A thrown resolver path still falls back to raw entries without the actionable route-key warning.

## Solution Assessment

Not quite: doctor plus startup warnings are the right narrow solution, but the startup path should also emit the warning when channel resolution fails and OpenClaw continues with the original configured entries.
