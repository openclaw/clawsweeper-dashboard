## What This Changes

The issue reports that llama.cpp/OpenAI-compatible SSE streaming records zero token usage because a final usage-only chunk is missed after stream finalization.

## Best Possible Solution

Fix the OpenAI-completions streaming path to preserve llama.cpp usage by consuming the final usage/timings data or adding a robust timings fallback, with focused regression coverage and redacted real-run proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-level yes, live no: current main depends on SDK-yielded chunks, the OpenAI SDK skips SSE after `[DONE]`, and the reporter supplied llama.cpp logs plus direct curl evidence, but I did not run a live llama.cpp turn.

## Solution Assessment

Is this the best way to solve the issue? No final fix is on main yet, and the linked PR should be reviewed or revised because suppressing `stream_options` does not preserve the usage data needed by `/status` and compaction.
