## Summary

Keep open: current main still has a source-reproducible path where llama.cpp-style usage that arrives after `[DONE]` cannot reach saved OpenAI-completions assistant/session usage, and the only linked PR was closed unmerged.

## What This Changes

The issue asks OpenClaw to preserve token usage for OpenAI-compatible llama.cpp streaming when usage is emitted in a final SSE chunk after the stop chunk.

## Best Possible Solution

Fix the OpenAI-completions transport or ingestion boundary so late llama.cpp usage/timings reach normalized assistant and session usage while strict custom providers can still omit unsupported request fields.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: current main still relies on SDK-yielded chunks, while OpenAI Node v6.37.0 skips SSE after `[DONE]`; the issue discussion supplies live llama.cpp evidence that useful usage exists but is not persisted.

## Solution Assessment

Is this the best way to solve the issue? Yes to keeping this open for a transport/parser or fallback fix; the closed request-flag-only PR was not sufficient because the reported backend needs final usage to survive into saved session accounting.
