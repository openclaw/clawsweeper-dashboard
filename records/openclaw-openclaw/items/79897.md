## Summary

Keep open: current main still has a source-reproducible path where llama.cpp usage can be missed, and the open target-repo fix PR has not merged and already has review concerns about preserving usage visibility.

## What This Changes

The issue reports that OpenAI-compatible streaming against llama.cpp records zero token usage because OpenClaw misses a final usage-only SSE chunk.

## Best Possible Solution

Fix the OpenAI-completions streaming path to consume or recover llama.cpp usage metadata while preserving token visibility for known local backends, then close this issue after the canonical PR merges.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-level yes: current main requests streaming usage, the OpenAI SDK skips SSE data after `[DONE]`, and OpenClaw only records usage chunks yielded by that iterator; I did not run a live llama.cpp server.

## Solution Assessment

Is this the best way to solve the issue? Not yet: the active fix direction should preserve real usage accounting, and the linked open PR needs review or replacement before this issue can close.
