## Summary

Keep open: current main still has a source-plausible llama.cpp/OpenAI-compatible streaming usage gap, and the linked upstream PR is open with a blocking review finding rather than a merged fix.

## What This Changes

The issue reports that llama.cpp/OpenAI-compatible streaming saves zero token usage because the client stops before processing a trailing usage-only SSE chunk.

## Best Possible Solution

Preserve the usage request and fix the OpenAI-completions streaming path so llama.cpp usage or timings reach normalized transcript and `/status` usage, with focused regression coverage and redacted real-run proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-level yes, live no: current main only records SDK-yielded usage chunks while OpenAI Node v6.37.0 skips SSE after `[DONE]`, and the reporter supplied llama.cpp log plus direct curl evidence for the trailing usage chunk.

## Solution Assessment

Is this the best way to solve the issue? No: current main is not fixed, and https://github.com/openclaw/openclaw/pull/79958 currently gates off the usage request instead of preserving the usage data needed by `/status` and compaction.
