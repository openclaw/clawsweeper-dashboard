## Summary

Keep open: current main still has a source-reproducible path where llama.cpp-style usage sent after `[DONE]` is requested but lost before OpenClaw records it, and the related closed PR did not land a correct upstream fix.

## What This Changes

The issue requests nonzero transcript, `/status`, and compaction usage for llama.cpp OpenAI-compatible streams whose final usage chunk arrives after stream termination.

## Best Possible Solution

Keep requesting streamed usage and fix the OpenAI-completions transport or ingestion boundary so trailing llama.cpp usage/timings reach normalized assistant and session usage with focused regression coverage.

## Reproduction Assessment

Yes at source level, but not from a live current-main run: current main depends on SDK-yielded chunks, OpenAI Node 6.37.0 skips SSE after `[DONE]`, and reporters supplied llama.cpp log/curl/live mitigation evidence.

## Solution Assessment

Yes to keeping this open for a transport/parser fix; the closed request-gating PR is not the best fix because it can stop affected local backends from emitting the metadata `/status` and compaction need.
