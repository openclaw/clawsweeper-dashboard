Keep open: current main still has embedded-runner usage fallback paths where an all-zero assistant usage snapshot can suppress the nonzero accumulator fallback, and the referenced fork PR/commit is not on current main.

## What This Changes

The issue requests correct saved token usage for llama.cpp/OpenAI-compatible streaming when a final usage-only SSE chunk follows the stop chunk.

## Best Possible Solution

Keep this as a narrow embedded-runner usage-accounting bug and add regression coverage that treats all-zero assistant usage snapshots as missing when nonzero attempt/accumulator usage exists.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source inspection gives a credible repro shape through mocked embedded-runner usage, but I did not run a live llama.cpp turn or a failing test in this read-only environment.

## Solution Assessment

Is this the best way to solve the issue? The best fix is likely to treat all-zero normalized assistant usage as missing in embedded-runner fallback/metadata assembly, while preserving the existing stream usage chunk handling rather than adding a broad transport workaround.
