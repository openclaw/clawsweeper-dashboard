## Summary

Keep open: current main still has a source-backed llama.cpp/OpenAI-compatible streaming usage gap, and https://github.com/openclaw/openclaw/pull/79958 remains the open closing-reference path rather than a merged fix.

## What This Changes

The issue reports zero saved token usage for llama.cpp OpenAI-compatible streaming because OpenClaw misses the final usage-only SSE chunk.

## Best Possible Solution

Fix the OpenAI-completions streaming path to preserve llama.cpp usage by consuming the final usage/timings data or adding a robust timings fallback, with focused regression coverage and redacted real-run proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-level yes, live no: current main initializes usage to zero, depends on OpenAI SDK-yielded chunks, and OpenAI SDK 6.37.0 skips post-`[DONE]` SSE data while the reporter supplied llama.cpp logs and curl evidence.

## Solution Assessment

Is this the best way to solve the issue? No current fix is merged; the open PR would gate off the usage request for false-compat backends, while the narrower maintainable fix is to preserve or recover the usage data needed by `/status` and compaction.
