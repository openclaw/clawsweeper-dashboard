## Summary

Keep open: current main still only records OpenAI-completions usage that the OpenAI SDK yields, while llama.cpp usage reported after `[DONE]` is skipped before it reaches OpenClaw. The related upstream PR was closed unmerged, and a later live reproduction confirms the useful fix direction is preserving final streamed usage into session persistence.

## What This Changes

The issue requests that OpenAI-compatible llama.cpp streaming preserve final usage or timings so transcripts, `/status`, and compaction see nonzero token counts.

## Best Possible Solution

Preserve llama.cpp final usage by fixing the guarded OpenAI-completions streaming path or a focused timings fallback, with regression coverage for usage after `[DONE]` and redacted real llama.cpp proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-level yes, live current-main no: current main consumes only SDK-yielded chunks, and OpenAI Node skips SSE frames after `[DONE]`; the issue discussion provides live llama.cpp logs and a 2026.5.7 mitigation result.

## Solution Assessment

Is this the best way to solve the issue? Yes in direction: OpenClaw should keep requesting usage and preserve final usage/timings into persisted session usage, not disable `stream_options.include_usage` for local/custom backends.
