The issue requests that OpenAI-compatible llama.cpp streaming preserve final usage metadata so saved transcript usage, `/status`, and compaction accounting do not stay at zero.

## Best Possible Solution

Fix the OpenAI-compatible stream ingestion boundary so final llama.cpp usage or timings reach normalized assistant/session usage, while keeping current request-compatibility policy and transport guards intact.
