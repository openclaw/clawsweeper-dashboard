## Summary

Keep open: current main still has a source-reproducible path where llama.cpp-style usage or timings emitted after `[DONE]` never reaches OpenClaw’s saved assistant/session usage. The merged request-shaping work helps strict providers and preserves known local opt-ins, but it does not add a raw SSE or fallback path for post-DONE metadata.

## What This Changes

The issue requests preserving nonzero OpenAI-compatible llama.cpp streaming usage when the final usage-only chunk arrives after the stop/DONE chunk.

## Best Possible Solution

Add a narrow OpenAI-completions transport or ingestion fix that preserves llama.cpp post-DONE usage/timings into normalized assistant/session usage while keeping compat-gated request shaping for strict providers.

## Reproduction Assessment

Yes, at source level: current main still depends on OpenAI SDK-yielded chunks for usage, and OpenAI Node v6.37.0 skips SSE messages after `[DONE]`. I did not run a live current-main llama.cpp endpoint.

## Solution Assessment

Yes to keeping this open for a transport/parser fix. The prior one-line request gate and the open transcript-estimate fallback work are adjacent mitigations, but they do not preserve the exact final llama.cpp usage chunk described here.
