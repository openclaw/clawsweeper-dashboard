## Summary

Keep open: current main has adjacent streaming-usage improvements, but the OpenAI-completions path still depends on chunks yielded by the OpenAI SDK, and OpenAI Node 6.37.0 skips SSE messages after `[DONE]`, matching the reported llama.cpp late-usage loss path.

## What This Changes

The issue asks OpenClaw to preserve nonzero token usage for llama.cpp/OpenAI-compatible streaming when final usage metadata arrives after the normal stop chunk.

## Best Possible Solution

Fix the OpenAI-completions transport or ingestion boundary so late llama.cpp usage/timings reach normalized assistant and session usage without broadening request parameters for strict providers.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: current main relies on SDK-yielded chunks, while OpenAI Node 6.37.0 skips SSE after `[DONE]`; the issue discussion supplies live llama.cpp evidence that useful usage exists but is not persisted.

## Solution Assessment

Is this the best way to solve the issue? Yes to keeping this open for a focused transport/parser or fallback fix; closing based on the current compat gate would be unsafe because that gate does not recover post-DONE SSE usage.
