## Summary

Keep open because current main still matches the reported OpenAI-compatible streaming usage path and an open linked fix PR has not merged yet.

## What This Changes

The issue reports that llama.cpp OpenAI-compatible streaming records zero usage because OpenClaw misses the final usage/timings data after stream finalization.

## Best Possible Solution

Fix the OpenAI-compatible stream handling so llama.cpp usage is retained, either by consuming/recovering the final usage/timings metadata or by an equivalent narrow fallback with regression coverage.

## Reproduction Assessment

Yes, at source level: current main requests streaming usage, the OpenAI SDK stops yielding after `[DONE]`, and OpenClaw only records usage from yielded chunks. I did not run a live llama.cpp turn in this read-only review.

## Solution Assessment

No, the active linked one-line PR is not the best fix as written because it suppresses the usage request for affected false-compat backends instead of preserving or recovering the usage data that status and compaction need.
