Keep open: current main still builds the Codex workspace bootstrap after the developer-instruction payload is finalized, then sends that bootstrap only through `config.instructions`. ## What This Changes

## Best Possible Solution

Append the existing non-AGENTS workspace bootstrap block to Codex app-server `developerInstructions` on start/resume while preserving the current `config.instructions` path and context-engine additions, then lock it with focused regression coverage.

## Repair Work Prompt

Fix the Codex app-server bootstrap forwarding bug for https://github.com/openclaw/openclaw/issues/77363. On current main, `SOUL.md` and other non-AGENTS workspace bootstrap files are merged into Codex `config.instructions` but are not appended to the explicit `developerInstructions` sent on `thread/start`/`thread/resume`. Implement the narrow fix in `extensions/codex/src/app-server/run-attempt.ts`: build the workspace bootstrap before finalizing the before-prompt hook payload, join it with the base/context-engine developer instructions, preserve existing context-engine additions, preserve the existing `mergeCodexConfigInstructions` behavior, and keep `AGENTS.md` excluded from the explicit bootstrap block because Codex loads it natively. Update `extensions/codex/src/app-server/run-attempt.test.ts` so the bootstrap-file test asserts `SOUL.md` appears in both `developerInstructions` and `config.instructions`, and `AGENTS.md` content appears in neither explicit bootstrap path. Update docs/changelog if the user-facing Codex harness behavior text changes. Do not add app-server personality config, do not remove `config.instructions`, and do not include `AGENTS.md` in the OpenClaw-rendered bootstrap unless a maintainer explicitly changes the product decision. Coordinate with overlapping PR #76739 and note whether the narrow fix supersedes or conflicts with it.
