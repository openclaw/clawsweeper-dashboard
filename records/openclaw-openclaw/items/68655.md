## Summary

Keep open. Current main still routes active streaming assistant text into the grouped chat renderer and renders markdown with `unsafeHTML`, while the open linked fix PR has not merged yet.

## What This Changes

The issue reports that streaming markdown in the web UI flickers because `renderGroupedMessage()` replaces the rendered markdown subtree on each token update.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/72983 or an equivalent focused Control UI fix that avoids `unsafeHTML` subtree replacement during active streaming, with regression coverage for streamed markdown/code-fence updates.

## Reproduction Assessment

Yes, at source level. The issue gives concrete manual steps with long streamed markdown, and current main still shows active streaming text reaching `renderGroupedMessage()` while the markdown branch renders through `unsafeHTML`.

## Solution Assessment

Yes, the linked PR's streaming-only plain-text fallback is a narrow maintainable fix if review confirms the UI tradeoff; an append-only markdown renderer could improve live formatting later but is not required to fix the flicker.
