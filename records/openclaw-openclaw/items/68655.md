## Summary

Keep open. Current main still routes active Control UI streaming text into the grouped markdown renderer, and the assistant/tool markdown branches still render via Lit unsafeHTML; the directly linked implementation candidate #72983 remains open and unmerged, so this issue should wait for that PR or an equivalent focused fix.

## What This Changes

The issue requests a streaming-safe Control UI markdown render path so completed streamed paragraphs do not flicker from full DOM replacement on each token update.

## Best Possible Solution

Land #72983 after review fixes, or land an equivalent focused Control UI patch that avoids unsafeHTML subtree replacement while streaming, preserves sanitized markdown after completion, and adds grouped-render coverage for stable streamed markdown/code-fence updates.

## Reproduction Assessment

Yes. The issue provides concrete manual steps with long streamed markdown, and current-main code inspection confirms active stream text reaches renderGroupedMessage with isStreaming true while the body still renders markdown through unsafeHTML.

## Solution Assessment

Unclear. A paragraph-stable streaming renderer is the strongest UX, but the plain-text streaming fallback proposed in #72983 is a smaller maintainable fix if whitespace and regression coverage are handled.
