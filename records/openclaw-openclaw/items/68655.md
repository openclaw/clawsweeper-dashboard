## Summary

Keep open. Current main and the latest shipped release still route active streaming chat markdown through `unsafeHTML(toSanitizedMarkdownHtml(...))`, and the linked closing PR remains open rather than merged.

## What This Changes

The issue asks the Control UI streaming renderer to keep completed markdown stable instead of replacing the full rendered subtree on each token update.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/72983 or an equivalent focused Control UI fix that avoids unsafeHTML subtree replacement during active streaming while preserving final sanitized markdown rendering and adding targeted proof.

## Reproduction Assessment

Yes, at source level: active streaming text reaches `renderGroupedMessage()` with `isStreaming: true`, but markdown still renders through `unsafeHTML(toSanitizedMarkdownHtml(markdown))`. I did not run a live browser recording in this read-only review.

## Solution Assessment

Yes as a direction: the maintainable fix is a streaming-safe escaped text or incremental markdown path during active streaming, with final sanitized markdown after completion. The linked PR or an amended equivalent should be reviewed and landed before closure.
