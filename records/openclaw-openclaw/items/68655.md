## Summary

Keep open. Current `main` still sends streaming assistant text through the grouped chat renderer and still renders markdown with `unsafeHTML`; the linked closing PR remains open and unmerged, so this issue should close only after that PR or an equivalent fix lands.

## What This Changes

The issue reports that Control UI streaming markdown replaces the rendered chat text DOM subtree on token updates, causing visible flicker for long markdown and code fences.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/72983 or an equivalent focused Control UI fix that keeps active streaming text DOM-stable while preserving final sanitized markdown rendering.

## Reproduction Assessment

Yes, at source level: current `main` passes active streaming assistant text to `renderGroupedMessage()` with `isStreaming: true`, then still renders markdown through `unsafeHTML`. The reporter also supplied human web UI verification, but I did not run a live browser capture.

## Solution Assessment

Yes as a direction: avoid `unsafeHTML` subtree replacement during active streaming and keep final sanitized markdown rendering after completion. The linked PR or an amended equivalent should be reviewed and landed before this issue closes.
