## Summary

Keep open. Current main still sends active Control UI streaming text into `renderGroupedMessage` with `isStreaming: true`, and the markdown render branches still use `unsafeHTML`; the linked implementation PR remains open, so this issue should wait for that PR or an equivalent focused fix to land.

## What This Changes

The issue requests a web UI fix so streaming markdown updates do not flicker by replacing the full rendered DOM subtree on each token update.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/72983, or land an equivalent focused Control UI patch that avoids `unsafeHTML` subtree replacement during active streaming and adds regression coverage for streamed markdown/code-fence updates.

## Reproduction Assessment

Yes. The issue gives concrete manual steps with long streamed markdown, and current-main source shows active stream text reaches `renderGroupedMessage` while markdown still renders through `unsafeHTML`; I did not run a live browser capture.

## Solution Assessment

Unclear. A paragraph-stable streaming renderer would be the best UX, while the linked PR's plain-text streaming fallback is a smaller maintainable fix if review covers whitespace, reasoning output, final markdown rendering, and tests.
