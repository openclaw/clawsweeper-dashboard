## Summary

Keep open. Current main still routes active Control UI streaming text into the grouped markdown renderer, and the assistant/tool markdown branches still render through `unsafeHTML`; the linked implementation PR #72983 is open and unmerged, so this issue should wait for that PR or an equivalent focused fix.

## What This Changes

The report asks the Control UI streaming message renderer to avoid replacing the full sanitized markdown DOM subtree on each token update.

## Best Possible Solution

Review and land #72983, or land an equivalent focused Control UI patch that avoids `unsafeHTML` subtree replacement while streaming and adds regression coverage for streamed markdown/code-fence updates.

## Reproduction Assessment

Yes. The report gives concrete manual steps with long streamed markdown, and current-main code inspection confirms active stream text reaches `renderGroupedMessage` while markdown still renders through `unsafeHTML`.

## Solution Assessment

Unclear. A paragraph-stable streaming markdown renderer would be the strongest UX, while #72983's plain-text streaming fallback is a smaller maintainable fix if whitespace, directionality, and final markdown rendering are covered by review and tests.
