## Summary

Keep open. Current main still routes streaming assistant text through the grouped chat renderer and still renders markdown through `unsafeHTML`; the open linked PR is the right place to finish and verify the fix before this issue closes.

## What This Changes

The issue reports web UI streaming markdown flicker caused by full markdown DOM replacement on token updates.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/72983 or an equivalent focused Control UI fix that keeps active streaming text DOM-stable, preserves final sanitized markdown rendering, and adds targeted regression coverage.

## Reproduction Assessment

Yes, at source level: current main passes active streaming text into `renderGroupedMessage()` with `isStreaming: true` while markdown still renders through `unsafeHTML(toSanitizedMarkdownHtml(markdown))`. I did not run a live browser recording in this read-only review.

## Solution Assessment

Yes as a direction: avoid `unsafeHTML` subtree replacement during active streaming and keep final sanitized markdown rendering after completion. The linked PR or an amended equivalent should be reviewed and landed before closing this issue.
