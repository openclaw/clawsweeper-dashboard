## Summary

Keep open. Current main still sends active streaming chat text through the grouped renderer with `isStreaming: true`, but the markdown branches still use `unsafeHTML(toSanitizedMarkdownHtml(markdown))`; the linked closing PR remains open and should land or be replaced before this issue closes.

## What This Changes

The issue requests a web UI fix so streaming markdown does not flicker from full `unsafeHTML` subtree replacement on each token update.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/72983 or an equivalent focused Control UI fix that keeps active streaming markdown DOM-stable, preserves final sanitized markdown rendering, and adds targeted regression proof.

## Reproduction Assessment

Yes, at source level: current main passes active streaming assistant text into `renderGroupedMessage()` with `isStreaming: true` while markdown still renders through `unsafeHTML(toSanitizedMarkdownHtml(markdown))`. I did not run a live browser recording in this read-only review.

## Solution Assessment

Yes as a direction: avoid `unsafeHTML` subtree replacement during active streaming and keep final sanitized markdown rendering after completion. The linked PR or an amended equivalent should be reviewed and landed before closing this issue.
