## Summary

Keep open: current main and latest release still accept macOS CLI gateway passwords through argv, and the only linked fix PR is closed unmerged. The report remains source-reproducible, but the exact safe-secret input contract needs maintainer security/product approval.

## What This Changes

The issue requests a safer macOS CLI password input path so gateway passwords do not need to be supplied through argv.

## Best Possible Solution

Add a maintainer-approved safe secret input contract for the macOS CLI, aligned with the core Gateway CLI pattern where possible, while preserving or explicitly warning on shipped argv behavior.

## Solution Assessment

Unclear as a final product direction: a non-argv credential path is the right direction, but maintainers should choose the exact stdin/file/env contract and scope before implementation.
