Add an approved additive macOS CLI safe-secret input contract, keep shipped inline flags compatible with explicit warnings, and cover connect, configure-remote, and wizard consistently with focused Swift tests.

Unclear as a final implementation. A non-argv secret input path is the right direction, but maintainers should approve the exact stdin/file/env/token coverage and mirror the existing core Gateway safe-secret pattern where appropriate.
