## Summary

Keep open. Current main has implemented the ElevenLabs realtime STT and provider-generic Voice Call streaming pieces, but the remaining DX asks are still unresolved: config-selected distinct duplicate plugin ids still warn, there is no per-id duplicate warning suppression or override acknowledgement config, and docs only cover plugin config schemas generally rather than focused guidance for extending or forking a bundled plugin with new config keys.

## What This Changes

Request to add ElevenLabs Scribe realtime STT support to Voice Call, add explicit duplicate-plugin override or warning-suppression controls, and document configSchema updates for custom or forked plugins.

## Best Possible Solution

Add a tested per-plugin duplicate override or suppression acknowledgement in the plugin config and manifest-registry diagnostics, document the duplicate-id behavior, and add focused plugin-author docs explaining that new custom config keys require updating the plugin manifest configSchema when extending or forking a bundled plugin. If maintainers prefer smaller tracking, split those two remaining DX asks into canonical follow-ups before closing this umbrella item.

## Reproduction Assessment

Yes for the remaining duplicate-warning behavior: the current manifest-registry unit test explicitly expects one duplicate warning when a config-loaded plugin replaces a bundled duplicate. The schema-docs gap is a documentation coverage check rather than a runtime reproduction, and targeted docs search did not find the focused guidance requested.

## Solution Assessment

No. Current main is the right shape for the ElevenLabs STT portion, but it is not yet the best complete solution for this issue because duplicate override acknowledgement and focused schema-extension docs are still missing.
