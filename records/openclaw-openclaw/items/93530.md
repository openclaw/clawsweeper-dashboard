Keep open: current main and v2026.6.9 still contain the reported cron watchdog phase gap, and multiple open PRs already reference this issue as implementation candidates.

## What This Changes

The issue reports premature aborts in isolated cron agent turns when the pre-execution watchdog does not clear after runner progress.

## Best Possible Solution

Land one canonical cron watchdog fix, likely https://github.com/openclaw/openclaw/pull/93914 or an equivalent patch, then let the merged PR close this issue and retire overlapping fix PRs.

## Reproduction Assessment

Yes. Source inspection shows the no-hook before_agent_reply emission gap and the watchdog still clearing only on execution-stage progress; current regression coverage also expects context_engine progress to end in a pre-execution abort.

## Solution Assessment

Yes in direction, but not complete on main. The best fix belongs in the cron watchdog progress contract, or an equivalent runner change that preserves true-stall detection, rather than adding provider-specific behavior or new configuration.
