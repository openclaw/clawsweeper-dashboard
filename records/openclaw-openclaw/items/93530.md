## Summary

Keep open: current main and v2026.6.9 still contain the cron watchdog phase gap, and multiple open PRs already use closing syntax for this bug, so the issue should wait for one canonical fix to merge.

## What This Changes

Cron isolated-agent turns can abort before the configured cron timeout because the pre-execution watchdog is not cleared when runner phase reporting remains in setup-like phases or skips `before_agent_reply`.

## Best Possible Solution

Land or reject one canonical fix that treats real isolated cron progress as sufficient to clear the pre-execution watchdog while preserving the no-progress runner-start guard.

## Reproduction Assessment

Yes. Current main still has the watchdog clear condition, runner phase guard, and a regression test that expects `context_engine` progress to hit the pre-execution timeout.

## Solution Assessment

Yes in direction: the maintainable fix belongs in the cron watchdog/phase transition, not provider-specific behavior or a new config option. The issue should stay open until one linked PR merges.
