## Summary

Keep open: current main and the latest release still have the cron isolated-agent watchdog behavior described here, and multiple linked closing PRs remain open rather than merged.

## What This Changes

The issue reports premature isolated cron agentTurn aborts when the pre-execution watchdog is not cleared by runner phase progress, especially when `before_agent_reply` is not emitted without hooks.

## Best Possible Solution

Land a reviewed cron watchdog or runner phase fix through the linked implementation path, preserving the no-progress guard while letting real setup progress avoid the 60-second false stall.

## Reproduction Assessment

Yes. Source inspection shows the current runner can skip `before_agent_reply` when no hook is registered, while the current watchdog only clears the pre-execution timer for execution-stage phases or `firstModelCallStarted`.

## Solution Assessment

Yes in direction, but not yet landed. The maintainable fix should stay in the cron watchdog/phase-reporting contract, with regression coverage that preserves the no-progress guard.
