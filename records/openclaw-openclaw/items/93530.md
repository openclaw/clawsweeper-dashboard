## Summary

This issue is still necessary: current main and the latest release both retain the reported watchdog/phase gap, and multiple open PRs are linked as candidate fixes but none has merged yet.

## What This Changes

Cron isolated-agent turns can be aborted by the 60s pre-execution watchdog before the configured job timeout when execution progress is reported only through setup phases or when `before_agent_reply` is not emitted.

## Best Possible Solution

Land one focused watchdog/phase-progress fix with regression coverage, preferably the cron watchdog contract that treats known setup phases as liveness while preserving the no-progress stall guard.

## Reproduction Assessment

Yes, from source inspection. Current main can be modeled by starting an isolated cron run, reporting `runner_entered` plus a setup phase such as `context_engine`, and advancing past the 60s pre-execution watchdog, which matches the existing regression test expectation.

## Solution Assessment

No final fix has landed yet. The maintainable solution should update the watchdog progress contract or otherwise guarantee cron sees real execution progress, then let the linked PR close this issue after merge.
