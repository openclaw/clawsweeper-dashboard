## Summary

Keep open: current main and the latest release still have the reported cron watchdog/phase gap, and multiple open PRs already reference this issue as a fix target. This should not be closed until one canonical watchdog fix lands.

## What This Changes

Cron isolated-agent turns can abort early because the pre-execution watchdog may not clear when the runner reports progress before a `before_agent_reply` phase.

## Best Possible Solution

Land one canonical watchdog fix that treats known isolated runner progress as progress under the configured job timeout while preserving the pre-execution guard for runner-started-with-no-phase stalls.

## Reproduction Assessment

Yes, source-reproducible: current main can report runner start plus a setup phase such as `context_engine` and still remain under the 60s pre-execution watchdog. I did not run a live cron job in this read-only review.

## Solution Assessment

Yes in direction: the clean fix belongs at the watchdog progress contract, with unconditional phase emission as a narrower mitigation. The safer end state is one canonical rule backed by cron timer regression coverage.
