## What This Changes

The issue reports premature aborts in isolated cron `agentTurn` runs when runner progress does not clear the pre-execution watchdog.

## Best Possible Solution

Land one canonical cron watchdog fix that treats real isolated-runner progress consistently without weakening the no-progress stall guard, then close this issue with the merged PR and regression proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main still gates `before_agent_reply` on hook presence and leaves setup phases under the pre-execution watchdog. I did not run a live slow-provider cron job in this read-only review.

## Solution Assessment

Unclear which open PR is the best final fix: unconditional `before_agent_reply` emission addresses this issue's narrow phase gap, while the related setup-progress issue argues for the broader known-phase progress contract. Maintainers should pick the intended watchdog semantics and land one regression-tested patch.
