Keep open: current main and v2026.6.9 still contain the reported isolated-cron watchdog phase gap, and multiple open closing PRs target this issue, so closure should wait for a reviewed merge.

## What This Changes

The issue reports that isolated cron agentTurn runs can false-abort when the 60s pre-execution watchdog is not cleared by runner phase progress.

## Best Possible Solution

Land one canonical cron watchdog fix that treats known setup progress as liveness while preserving the true no-progress stall guard, then let the merged PR close this issue and retire overlapping PRs.

## Reproduction Assessment

Yes. Source inspection shows current main still maps setup progress to pre_execution without clearing the pre-execution watchdog, and regression coverage still expects context_engine progress to end in stalled before execution start.

## Solution Assessment

Yes in direction, but it is not complete on main. The best fix belongs in the cron watchdog progress contract or an equivalent runner phase-emission repair with focused regression coverage, not provider-specific behavior or a new config option.
