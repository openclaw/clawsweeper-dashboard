## Summary

Keep open: current main and v2026.6.9 still have the cron pre-execution watchdog phase gap, and several open PRs still reference this issue with closing syntax, so the issue should wait for one reviewed implementation to merge.

## What This Changes

The issue requests fixing isolated cron agent turns that can be falsely aborted by the 60-second pre-execution watchdog when execution progress is not reported as an execution-stage phase.

## Best Possible Solution

Land one canonical cron watchdog fix, likely https://github.com/openclaw/openclaw/pull/93914 or an equivalent patch that counts real setup progress while preserving true runner-start stall protection.

## Reproduction Assessment

Yes. Source inspection shows the no-hook before_agent_reply emission gap and the watchdog only clears on execution-stage phases; current regression coverage also still expects context_engine progress to hit the pre-execution abort.

## Solution Assessment

Yes in direction, but it is not implemented on main. The broader watchdog-layer fix is the cleanest path because current main also treats setup phases such as model_resolution, auth, and context_engine as non-progress.
