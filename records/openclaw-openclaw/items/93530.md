## Summary

Keep open: current main and the latest stable release still have the reported no-hook cron watchdog phase gap, and several open closing PRs already own candidate fixes, so this should wait for one reviewed implementation to merge.

## What This Changes

The issue reports isolated cron agent-turn runs aborting early because `before_agent_reply` is not emitted when no hook is registered, leaving the 60-second pre-execution watchdog armed.

## Best Possible Solution

Land one canonical cron watchdog/progress fix that prevents the no-hook false abort while preserving true-stall protection, then let the merged PR close this issue and retire overlapping PRs.

## Reproduction Assessment

Yes, by source inspection: current main still gates `before_agent_reply` on hook registration while the watchdog clears only on execution-stage progress or `firstModelCallStarted`. I did not run a live scheduled cron repro in this read-only review.

## Solution Assessment

Yes in direction, but not completed on main. The best fix is a narrow cron watchdog/progress-contract repair or equivalent runner change that preserves the true no-progress guard, not a new config option or provider-specific workaround.
