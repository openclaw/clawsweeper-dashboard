## What This Changes

The issue reports that isolated cron agent-turn runs can abort at the 60s pre-execution watchdog because cron phase progress is not reported consistently when no before_agent_reply hook is registered.

## Best Possible Solution

Land one canonical cron watchdog fix that reports or interprets cron phase progress consistently while preserving true-stall protection, then let the merged PR close this issue and retire overlapping fix PRs.

## Reproduction Assessment
