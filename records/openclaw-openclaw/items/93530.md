## Summary

Keep open: current main and the latest release still have the reported watchdog behavior, and this issue has multiple open closing PRs that should either land or be rejected before the issue closes.

## What This Changes

Cron isolated-agent turns can be aborted by the 60s pre-execution watchdog when progress is reported through setup phases or when `before_agent_reply` is not emitted because no hook is registered.

## Best Possible Solution

Choose and land one canonical watchdog fix that treats known setup progress as liveness while preserving the no-phase stall timeout, then let the merged PR close this issue and the narrowed follow-up.

## Reproduction Assessment

Yes. Source inspection on current main shows the hook-gated phase emission and the watchdog clear condition, and the existing regression test still expects `context_engine` progress to hit the 60-second pre-execution abort.

## Solution Assessment

The broader watchdog progress-contract fix is the best direction. Unconditionally emitting `before_agent_reply` is a plausible targeted mitigation, but it does not by itself settle setup-phase progress and fallback re-arm cases already narrowed in https://github.com/openclaw/openclaw/issues/93912.
