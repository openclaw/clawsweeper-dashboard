## Summary

Keep open. Current main and live repository state have adjacent intake, stale cleanup, Barnacle routing labels, ClawSweeper work-lane labels, duplicate cleanup, proof labels, and broad priorities, but they still do not define the requested repo-owned issue lifecycle, P0-P3 semantics, priority authority, cadence, visible cycle-priority surface, or state-aware stale transitions.

## What This Changes

The issue proposes a lightweight repository triage framework with lifecycle labels, P0-P3 priorities, regular cadence, a visible priority surface, and state-aware stale cleanup.

## Best Possible Solution

Define a maintainer-approved triage playbook plus matching GitHub labels, project board or pinned-priority surface, and stale-policy updates that integrate with existing intake, Barnacle, ClawSweeper, proof, stale, duplicate-cleanup, and roadmap mechanisms.
