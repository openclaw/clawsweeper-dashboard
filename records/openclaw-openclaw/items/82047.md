## Summary

Keep open. Current main already has a collapsed Control UI message-footer Context disclosure and a persistent session context meter, but the requested configurable footer fields plus explicit cache status, cache TTL/hit state, and compaction-waterline display are still a UX/product feature rather than implemented behavior.

## What This Changes

The issue requests configurable Control UI message footer fields for cache, token, cost, and compaction-waterline information.

## Best Possible Solution

If maintainers want this, add a small documented Control UI footer preference backed by existing chat.history usage and session token data; otherwise explicitly settle on the current Context disclosure plus context notice as the supported UX.
