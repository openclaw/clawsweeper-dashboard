## Summary

Keep open: current main and v2026.6.10 still have internal code-mode namespaces, Tool Search, and MCP namespace support, but not the requested public external catalog-provider bridge for provider-owned search/describe/call delegation.

## What This Changes

## Best Possible Solution

Have maintainers define a narrow generic plugin/API and security contract, then implement the bridge only if Tool Search, MCP, and internal namespaces cannot safely express this composition.
