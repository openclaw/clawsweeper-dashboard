## Summary

Keep open: current main and v2026.6.10 already have code mode, Tool Search, MCP namespacing, and an internal namespace registry, but they do not expose the requested public external catalog-provider bridge for provider-owned search/describe/call delegation. ## What This Changes

The issue requests a generic code-mode external catalog provider bridge so provider-owned search, describe, and call APIs can be used without nested executor runtimes.

## Best Possible Solution

Have maintainers decide the public code-mode catalog-provider/plugin API and security contract, then implement a narrow generic bridge only if Tool Search, MCP, and internal namespace surfaces cannot safely express this composition.
