## Summary

Keep open: current main has OpenClaw code mode, Tool Search, MCP code-mode namespaces, and an internal namespace registry, but it still does not expose the requested public external catalog-provider bridge for provider-owned search/describe/call delegation.

## What This Changes

Request to add a generic code-mode external catalog bridge so configured provider plugins can expose search, describe, and call without nesting a second executor runtime.

## Best Possible Solution

Have maintainers approve a generic plugin-owned external catalog API and security contract, then implement the bridge only if Tool Search, MCP, and internal namespaces cannot safely express the composition.
