## Summary

Keep open: current main and v2026.6.10 have OpenClaw code mode, Tool Search, MCP code-mode namespaces, and internal plugin-owned namespaces, but not the requested public external catalog-provider bridge for provider-owned search/describe/call delegation.

## What This Changes

The issue requests a generic code-mode external catalog bridge so configured providers can expose search, describe, and call without nesting another executor runtime.

## Best Possible Solution

Have maintainers approve a narrow generic plugin/API and security boundary first, then implement the bridge only if existing MCP, Tool Search, and internal namespace surfaces cannot safely express the composition.
