## Summary

Keep open: current main and the latest release have OpenClaw code mode, Tool Search, internal plugin-owned namespaces, and MCP namespace support, but not the requested public external catalog-provider bridge for provider-owned search/describe/call delegation. The remaining work is a maintainer product/API/security-boundary decision, not conservative cleanup.

## What This Changes

The issue requests a first-class external catalog-provider namespace for code mode so providers can expose search, describe, and call without nesting another executor runtime.

## Best Possible Solution

Have maintainers decide the plugin/API and security boundary, then implement a narrow generic bridge only if current Tool Search, MCP, and internal namespace surfaces cannot safely express this composition.
