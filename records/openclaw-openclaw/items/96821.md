## Summary

Keep open: current main and v2026.6.10 have OpenClaw code mode, Tool Search, MCP code-mode namespacing, and an internal namespace registry, but they still do not provide the requested public external catalog-provider bridge for provider-owned search/describe/call delegation.

## What This Changes

Request to add a generic code-mode bridge so external catalog providers such as Executor can expose search, describe, and call without nesting another code executor.

## Best Possible Solution

Have maintainers decide the public code-mode catalog-provider/plugin API and security contract, then implement a narrow generic bridge only if Tool Search, MCP, and internal namespaces cannot safely express this composition.
