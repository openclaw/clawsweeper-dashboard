## Summary

Keep open: current main and v2026.6.10 have adjacent code mode, Tool Search, MCP namespacing, and an internal namespace registry, but no public external catalog-provider bridge for provider-owned search/describe/call delegation. The request remains a product, plugin API, and security-boundary design item rather than conservative cleanup.

## What This Changes

The issue requests a generic code-mode external catalog-provider bridge so provider namespaces can expose search, describe, and call without nesting another executor runtime.

## Best Possible Solution

Have maintainers define a narrow generic plugin/catalog API and security contract, then implement it through plugin-owned provider adapters only if existing Tool Search, MCP, and internal namespace surfaces cannot cover the composition.
