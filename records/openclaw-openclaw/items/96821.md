## Summary

Keep open: current main and v2026.6.10 already have OpenClaw code mode, Tool Search, MCP code-mode namespaces, and an internal namespace registry, but they still do not expose the requested public external catalog-provider bridge for provider-owned search/describe/call delegation. This remains a product/API/security-boundary design item, not conservative cleanup.

## What This Changes

The issue requests a generic code-mode external catalog-provider bridge so providers can expose search, describe, and call without routing through a nested executor runtime.

## Best Possible Solution

Have maintainers decide the public code-mode catalog-provider/plugin API and security contract, then implement a narrow generic bridge only if Tool Search, MCP, and internal namespaces cannot safely express this composition.
