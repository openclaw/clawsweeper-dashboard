## Summary

Keep open: current main still performs the broad Windows `schtasks /Query` runtime-status preflight, while this PR contains the narrow code change and regression coverage for the linked Windows status bug. The code direction looks correct, but the external real-behavior-proof gate is still missing live Windows proof before merge.

## What This Changes

This PR removes the broad Windows `schtasks /Query` preflight from `readScheduledTaskRuntime()`, maps task-scoped query throws into fallback handling, updates schtasks fixtures/tests, and adds a changelog entry.

## Best Possible Solution

Land the narrow direct task-scoped runtime query patch after redacted live Windows `openclaw status` or scheduled-task proof is added and the proof gate passes.

## Reproduction Assessment

Yes from source inspection and the proposed regression path: current main calls `assertSchtasksAvailable()` before the named task query, so a failing broad `/Query` with a successful or fallback-capable `/Query /TN` reproduces the reported failure shape. I did not run a live Windows scheduled task in this read-only review.

## Solution Assessment

Yes for the code direction: removing only the runtime-status preflight while preserving install/restart preflights and Startup fallback behavior is the narrow maintainable fix. Merge readiness still depends on live Windows proof, not another code repair.
