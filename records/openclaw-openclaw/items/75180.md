## Summary

Keep open: this PR targets a real mixed text+media reply bug, but the current head still marks media as confirmed from payload contents before route or dispatcher delivery has actually succeeded, and the supplied proof is unit-test output rather than real Telegram or equivalent channel behavior.

## What This Changes

The branch adds callback-returned sent-media tracking to the block reply pipeline, threads the return type through reply delivery and dispatch, updates pipeline tests, and adds a changelog fix entry.

## Best Possible Solution

Keep the callback-result direction, but return sentMediaUrls only from an actual successful channel or dispatcher delivery signal, add failure-path coverage, and attach redacted live Telegram or equivalent channel proof.

## Reproduction Assessment

Yes, as a source-level path. The linked Telegram report gives concrete direct-chat steps, and current main records mixed block media before final media dedupe can strip matching attachments.

## Solution Assessment

No. The callback-return contract is a maintainable direction, but this implementation still derives confirmation from normalized payload contents instead of successful delivery and needs failure-path coverage.
