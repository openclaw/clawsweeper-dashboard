## Summary

Keep open. The underlying mixed text+media reply bug is valid and the delivery-aware callback shape is useful, but this PR still confirms media from payload content before the dispatch path proves delivery, so final dedupe can still strip media that never reached the channel.

## What This Changes

The PR adds a BlockReplyResult sent-media confirmation path for block replies, updates reply delivery types and dispatch-from-config, adds block-pipeline tests, and records a changelog fix for mixed text+media reply dedupe.

## Best Possible Solution

Keep the delivery-aware callback seam, but gate sent-media confirmations on successful direct delivery and avoid or explicitly acknowledge queued dispatcher confirmations before final dedupe consumes them.

## Reproduction Assessment

Yes. #75156 provides concrete Telegram DM steps, and current-main source shows mixed text+media block media is added to the block sent-media set and later consumed by final media dedupe.

## Solution Assessment

No. The callback result is the right direction, but this PR still derives confirmation from payload content instead of a delivery success result, so it is not yet the narrowest safe fix.
