## Summary

Keep open: current main and v2026.6.6 still only expose ACPX permission policy as plugin-global config plus a session-level `permissionProfile` runtime option, not as persistent per-binding or per-agent defaults. The request remains useful and security-relevant, but it needs maintainer product/security direction before implementation.

## What This Changes

The issue requests scoped ACP permission defaults at `bindings[].acp` and `agents.list[].runtime.acp` instead of only the global ACPX plugin config.

## Best Possible Solution

Define a generic scoped ACP permission-default contract, likely based on `permissionProfile`, then map it to ACPX permission controls with explicit precedence, docs, and security coverage.

## Reproduction Assessment

Yes. Current source inspection shows strict config schemas omit scoped permission defaults, and ACPX consumes permission policy only from plugin-global config.

## Solution Assessment

Unclear. The requested precedence is plausible, but the best fix should settle a generic ACP permission-default contract instead of adding only ACPX-specific `permissionMode` keys.
