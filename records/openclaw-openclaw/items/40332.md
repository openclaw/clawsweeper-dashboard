## Summary

Keep open: current main and v2026.6.8 still only support global ACPX permissionMode plus session-level runtime permissionProfile, not per-binding or per-agent scoped ACP permission defaults. The request is valid and security-relevant, but the public config and security contract need maintainer direction before implementation.

## What This Changes

Request per-binding and per-agent ACP permission defaults so different ACP sessions can use different approval policies instead of one global ACPX permissionMode.

## Best Possible Solution

Define one scoped ACP permission-default contract, likely generic permissionProfile at binding and agent-runtime levels, then map it to ACPX permission controls with explicit precedence and security tests.

## Reproduction Assessment

Yes. Source inspection shows current strict schemas omit scoped permission defaults at bindings[].acp and agents.list[].runtime.acp, while ACPX consumes permissionMode only from plugin-global config.

## Solution Assessment

Unclear. The requested precedence is plausible, but the best fix should settle a generic ACP permission-default contract, likely building on permissionProfile, instead of adding only ACPX-specific permissionMode keys.
