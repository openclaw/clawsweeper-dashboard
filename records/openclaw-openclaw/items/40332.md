## Summary

Keep open. Current main still has global ACPX permission settings, while binding and per-agent ACP config schemas do not carry scoped permission policy; the May 12 Gemini ACP comment adds a concrete over-permission use case rather than stale speculation.

## What This Changes

Request per-binding and per-agent ACP permission policy overrides, falling back to the current global ACPX plugin config.

## Best Possible Solution

Define a typed scoped ACP permission contract resolved as binding > agent runtime > plugin global, pass the effective policy through the ACP runtime backend seam, and document/test the security behavior.

## Reproduction Assessment

Yes. Source inspection on current main is enough to reproduce the gap: strict binding and per-agent runtime schemas omit scoped permission fields, and ACPX still receives one plugin-global permission policy.

## Solution Assessment

Yes directionally, but this is not a narrow automatic bug fix. The requested precedence matches existing ACP override layers, while the exact permissionProfile versus ACPX permissionMode contract needs maintainer security/product judgment.
