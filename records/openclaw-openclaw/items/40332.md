## Summary

Keep open: current main still has only global ACPX `permissionMode`, while binding and per-agent ACP config schemas do not accept or pass scoped permission defaults. Related items either closed the global config/docs confusion or discuss broader ACP architecture; none replaces this scoped permission decision.

## What This Changes

The issue requests binding-level and per-agent ACP permission overrides instead of applying one global ACPX `permissionMode` to every ACP session.

## Best Possible Solution

Keep this open for maintainer-sponsored design that adds scoped ACP permission defaults with clear precedence, security semantics, docs, and regression coverage across binding and agent-runtime config.

## Reproduction Assessment

Yes for the config gap from source: current strict schemas omit the requested fields, and the binding lifecycle never forwards a scoped permission policy. I did not run a live ACP session in this read-only review, but the issue comments add a concrete Gemini reproduction/use case.

## Solution Assessment

No as an autonomous fix yet. The narrow implementation should likely build on the generic ACP runtime option/defaulting model rather than hard-coding an ACPX-only field, and that requires maintainer product/security direction.
