## Summary

Keep open: this remains a valid feature gap. Current main supports global ACPX permission settings, but the existing per-binding and per-agent ACP override chain omits `permissionMode` and `nonInteractivePermissions`, and the pinned `acpx` runtime exposes those settings at runtime/client construction rather than per session.

## What This Changes

Request per-binding and per-agent ACPX permission overrides with global plugin config as the fallback.

## Best Possible Solution

Define a documented permission override contract for ACP bound sessions after maintainers choose the runtime seam for safely scoping ACPX permission policy per session or per configured binding.

## Reproduction Assessment

Not applicable as a feature request, but source inspection gives high confidence that the requested fields are not implemented: the current schemas, binding resolver, metadata, and docs only carry existing ACP fields.

## Solution Assessment

Unclear. The requested precedence matches the existing ACP override model, but the current ACPX dependency does not expose `permissionMode` on per-session initialization, so maintainers should decide the API and security model first.
