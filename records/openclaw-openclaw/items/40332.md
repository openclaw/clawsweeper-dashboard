## Summary

Keep open: current main and the latest release still lack scoped ACP permission defaults, while the over-permission problem is valid and security-relevant. The remaining work needs maintainer product/security direction because current main has a generic session-level `permissionProfile` path that may be the right canonical contract instead of adding ACPX-only `permissionMode` fields.

## What This Changes

Request scoped ACP permission policy so bindings or agent runtime config can override the current global ACPX `permissionMode`.

## Best Possible Solution

Extend one canonical ACP permission-default model into documented per-binding and per-agent defaults, then map it to ACPX permission controls with focused precedence and security tests.

## Reproduction Assessment

Yes. Source inspection shows current strict config schemas omit scoped permission defaults at `bindings[].acp` and `agents.list[].runtime.acp`, while ACPX consumes `permissionMode` only from plugin-global config.

## Solution Assessment

Unclear. The requested precedence is plausible, but the best fix should settle a generic ACP permission-default contract, likely building on `permissionProfile`, instead of adding only ACPX-specific `permissionMode` keys.
