## Summary

Keep open: current main and v2026.6.8 still do not provide per-binding or per-agent ACP permission defaults. The request remains valid and security-relevant, but the permanent config contract needs maintainer product/security direction because current main also has a newer per-session `permissionProfile` runtime-option path.

## What This Changes

The issue requests scoped ACP permission policy overrides at binding and agent-runtime levels instead of relying only on the global ACPX plugin `permissionMode` setting.

## Best Possible Solution

Extend a single documented scoped ACP permission model, likely by turning binding/agent defaults into `permissionProfile` runtime options, and map that safely to ACPX permission controls with focused schema, docs, and security tests.

## Reproduction Assessment

Yes. Source inspection shows current strict config schemas omit scoped permission defaults at `bindings[].acp` and `agents.list[].runtime.acp`, while ACPX consumes `permissionMode` only from plugin-global config or per-session runtime options.

## Solution Assessment

Unclear. The requested precedence is plausible, but the best fix should settle a generic ACP permission-default contract, likely building on `permissionProfile`, instead of adding only ACPX-specific `permissionMode` keys.
