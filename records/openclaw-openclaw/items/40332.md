## Summary

Keep open. Current main still has global ACPX permission settings and canonical session runtime permission profiles, but it does not expose or propagate per-binding or per-agent ACP permission defaults; the recent Gemini repro confirms the over-permission tradeoff is still an active security/product gap.

## What This Changes

Request per-binding and per-agent ACP permissionMode overrides for ACP sessions, with binding-level values taking precedence over agent runtime config and global ACPX plugin config.

## Best Possible Solution

Define one typed scoped ACP permission contract with binding > per-agent runtime > plugin fallback precedence, align it with canonical permissionProfile/runtime-control mapping where feasible, and document/test backend support plus rejection behavior.

## Reproduction Assessment

Yes, source-level. Current main still rejects the requested scoped config fields and does not carry a scoped permission policy from binding or per-agent config into ACP session startup.

## Solution Assessment

Unclear pending maintainer product/security decision. The requested precedence is directionally sound, but the durable shape should line up plugin permissionMode/nonInteractivePermissions with OpenClaw's canonical permissionProfile runtime control instead of adding a one-off field only.
