## Summary

Keep open. Current main still has only plugin-global ACPX permission config; binding, agent runtime, persistent binding, and runtime-backend surfaces do not carry scoped permission policy, and the latest OpenClaw 2026.5.7 Gemini use case confirms real over-permission pressure.

## What This Changes

Request to allow ACPX permissionMode and nonInteractivePermissions to be scoped per ACP binding and per agent runtime, falling back to plugin-global config.

## Best Possible Solution

Adopt a typed scoped ACP permission contract resolved as binding > agent runtime > plugin global, pass it through the ACP runtime backend seam, and document/test the security behavior including bounded artifact-write semantics.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: the strict binding and per-agent runtime schemas omit scoped permission keys, and ACPX still receives one plugin-global permission policy.

## Solution Assessment

Yes directionally, but not as an automatic narrow bug fix. The requested precedence matches existing ACP override layers, while public permission and non-interactive prompt semantics need maintainer security/product review before becoming a config contract.
