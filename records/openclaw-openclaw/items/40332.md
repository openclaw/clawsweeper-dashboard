## Summary

Keep open. Current main still has only plugin-global ACPX startup permission policy; binding config, per-agent ACP runtime defaults, configured binding persistence, and ACP runtime startup do not carry scoped `permissionMode` or `nonInteractivePermissions`.

## What This Changes

The issue requests per-binding and per-agent ACPX permission policy overrides, with existing plugin-global ACPX config as the fallback.

## Best Possible Solution

Define a typed scoped ACP permission contract for binding and agent runtime config, compute effective policy as binding > agent runtime > plugin global, persist and rehydrate it for configured bindings, pass it through ACP runtime startup, and document/test the precedence.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the configuration gap: strict schemas reject the proposed keys, and the binding/runtime paths do not carry a scoped permission policy.

## Solution Assessment

Yes in direction, but not as an automatic narrow fix. The requested precedence matches existing ACP override shape, while permission and non-interactive prompt semantics need maintainer security/product review before implementation.
