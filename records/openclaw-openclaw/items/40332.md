## Summary

Keep open: current main and v2026.6.6 still do not provide per-binding or per-agent ACP permission defaults. The request is valid and security-relevant, but the permanent config contract needs maintainer product/security direction because current main also has a newer generic session-level `permissionProfile` path that should likely be the canonical shape.

## What This Changes

Request per-binding and per-agent ACP permission overrides so ACP sessions with different trust levels do not share one global ACPX `permissionMode`.

## Best Possible Solution

Extend the existing generic ACP permission profile model into documented per-binding and per-agent defaults, then map it to ACPX permission controls with focused precedence and security tests.

## Reproduction Assessment

Yes. Source inspection shows current strict config schemas omit scoped permission defaults at `bindings[].acp` and `agents.list[].runtime.acp`, while ACPX consumes `permissionMode` only from plugin-global config.

## Solution Assessment

Unclear. The requested precedence is plausible, but the best fix should settle a generic ACP permission-default contract, likely building on `permissionProfile`, instead of adding only ACPX-specific `permissionMode` keys.
