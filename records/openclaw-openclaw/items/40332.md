## Summary

Keep open. Current main still has ACPX permissionMode/nonInteractivePermissions as plugin-global startup policy only; the requested per-binding and per-agent public config contract is not implemented, and the remaining work is security/product-sensitive rather than a stale cleanup close.

## What This Changes

Request per-binding and per-agent ACP permission policy overrides instead of relying only on the ACPX plugin-global permissionMode.

## Best Possible Solution

Define a typed scoped ACP permission contract, compute effective policy as binding > agent runtime > plugin global, persist it for configured bindings, pass it through an explicit ACP runtime startup/control seam, and document/test the security semantics.

## Reproduction Assessment

Yes. Source inspection on current main is enough: strict binding and per-agent ACP runtime schemas reject the proposed keys, and the binding/runtime startup paths do not carry scoped permissionMode or nonInteractivePermissions.

## Solution Assessment

Yes in direction, but not as an automatic narrow fix. The requested precedence matches the existing ACP override shape, while permission and non-interactive prompt semantics need maintainer security/product review before adding public config.
