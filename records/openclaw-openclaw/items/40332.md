## Summary

Keep open. Current main and the latest shipped release still expose ACPX permissions as plugin-global config plus per-session runtime controls, while the requested per-binding and per-agent defaults remain absent and security-sensitive.

## What This Changes

Request scoped ACP permission defaults for ACP bindings and agent runtime config instead of only `plugins.entries.acpx.config.permissionMode`.

## Best Possible Solution

Define one typed scoped ACP permission contract with binding > agent runtime > plugin/backend fallback precedence, reuse the generic `permissionProfile` path where possible, and update docs plus security-focused tests.

## Reproduction Assessment

Yes, source-level. Current main and `v2026.5.12` omit scoped permission fields from strict binding/per-agent schemas, and configured binding initialization does not pass any scoped permission option into ACP session startup.

## Solution Assessment

Unclear pending maintainer product/security decision. The requested precedence is directionally sound, but the implementation should likely model generic ACP `permissionProfile` plus ACPX-specific non-interactive behavior instead of hard-coding only one plugin field name.
