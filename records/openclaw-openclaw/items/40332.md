## Summary

Keep open: current main and v2026.6.8 still lack per-binding or per-agent ACP permission defaults. The request is valid and security-relevant, but the config/security contract needs maintainer direction because current main also has a generic session-level permissionProfile path that should likely inform the permanent shape.

## What This Changes

Request scoped ACP permission defaults at bindings[].acp and agents.list[].runtime.acp so different ACP sessions can use different permission policies instead of sharing one global ACPX permissionMode.

## Best Possible Solution

Define a generic, documented scoped ACP permission profile default for per-binding and per-agent configuration, map it to ACPX permission controls, and cover precedence plus security behavior in focused tests and docs.

## Reproduction Assessment

Yes. Source inspection shows current strict config schemas omit scoped permission defaults at bindings[].acp and agents.list[].runtime.acp, while ACPX consumes permissionMode only from plugin-global config.

## Solution Assessment

Unclear. The requested precedence is plausible, but the best fix should settle a generic ACP permission-default contract, likely building on permissionProfile, rather than adding only ACPX-specific permissionMode keys.
