## Summary

Keep open: current `main` and v2026.6.6 already keep `/btw` ephemeral and out of chat history, but they still do not expose a deliberate session-scoped system-prompt overlay for WebSocket/chat clients. The remaining request is a new prompt-authority/session-state API that needs maintainer product and security review before implementation.

## What This Changes

The issue requests a session-scoped `/sticky` command or `/btw --system` flag that lets WebSocket clients inject persistent system-prompt context without adding visible chat history.

## Best Possible Solution

Design an explicit authenticated session prompt-overlay/API or plugin hook surface with visibility, reset, audit, transcript/export, cache, and precedence semantics, while keeping `/btw` ephemeral.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs show `/btw` is already ephemeral, while `chat.send` lacks the requested persistent system-prompt overlay field.

## Solution Assessment

No for `/btw --system` as the main shape: `/btw` is an ephemeral side-question contract. A separate session-scoped prompt-overlay API or plugin/SDK capability is the cleaner design if maintainers approve the trust boundary.
