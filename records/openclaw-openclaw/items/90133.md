## Label Justifications

- `P3`: This is a speculative but concrete feature/API request rather than a broken existing contract.

## Summary

Keep open: current `main` and v2026.6.8 already keep `/btw` ephemeral and out of chat history, but they still do not expose the requested session-persistent system-prompt overlay for WebSocket/chat clients. The remaining work is a new prompt-authority/session-state API, so it needs maintainer product and security review rather than conservative cleanup closure.

## What This Changes

The issue requests a session-scoped `/sticky` command or `/btw --system` mode that injects client-provided persistent context into the system prompt instead of chat/message history.

## Best Possible Solution

Design a deliberate authenticated session prompt-overlay or plugin/API surface with visibility, reset, audit, transcript/export, cache, and precedence semantics, while preserving `/btw` as ephemeral.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request for a new prompt-authority/session-state API. Source, docs, and latest-release tag inspection show `/btw` is ephemeral and `chat.send` lacks the requested persistent prompt-overlay field.

## Solution Assessment

No for `/btw --system` as the main shape because `/btw` is now documented and implemented as an ephemeral side-question contract. A separate session-scoped prompt-overlay API or plugin/SDK capability is the cleaner design if maintainers approve the trust boundary.
