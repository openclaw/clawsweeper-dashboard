## Summary

Keep open. Current main only partially mitigates the bloat by stripping `skillsSnapshot.resolvedSkills`; it still persists the lightweight `skillsSnapshot` shell and `systemPromptReport`, stale-session handling still forces snapshot rebuild behavior, and open PR #48425 uses closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to stop `sessions.json` bloat by avoiding redundant `skillsSnapshot` and `systemPromptReport` persistence and stale-session snapshot rebuilds.

## Best Possible Solution

Land #48425 or a scoped replacement that defines the durable session-entry serialization shape, aligns disk-budget measurement with actual writes, and stores only the prompt-report state needed for warning dedupe.

## Reproduction Assessment

Yes. Source inspection shows the session-store write path still serializes fields left on `SessionEntry`, while only `skillsSnapshot.resolvedSkills` is stripped and `systemPromptReport` is still copied after each run.

## Solution Assessment

Unclear. Current main is only a partial mitigation; the narrow maintainable fix is a shared serialization boundary or equivalent that removes transient fields while preserving required dedupe metadata.
