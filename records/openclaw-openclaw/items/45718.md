## Summary

Keep open: current main and v2026.6.6 mitigate the largest `skillsSnapshot.prompt` duplication with content-addressed prompt blobs, but the linked closing PR is still open and current code still preserves `systemPromptReport` plus stale-session rollover behavior that needs maintainer judgment.

## What This Changes

The issue asks OpenClaw to stop unbounded session-store growth from repeated `skillsSnapshot` and `systemPromptReport` persistence across agent runs.

## Best Possible Solution

Keep the shipped prompt-blob mitigation, define the minimal durable `systemPromptReport` state needed for warning dedupe, and resolve https://github.com/BingqingLyu/openclaw/pull/869 or a successor before closing this issue.

## Reproduction Assessment

Yes at source level: current main still writes `systemPromptReport` after runs and still treats stale unrequested sessions as new. I did not run a live long-uptime size-growth reproduction, and the largest prompt duplication path is already mitigated by prompt blobs.

## Solution Assessment

Unclear as a final fix: the shipped prompt-blob design is a maintainable mitigation for duplicated skill prompts, but the remaining `systemPromptReport` and stale rollover behavior need maintainer agreement before another persistence change lands.
