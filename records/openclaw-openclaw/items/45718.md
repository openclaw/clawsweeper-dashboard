## Summary

Keep open. Current main has a partial mitigation for `skillsSnapshot.resolvedSkills`, but the session-store write path still serializes top-level `skillsSnapshot` metadata and `systemPromptReport`, and the open implementation PR with closing syntax for this issue has not merged.

## What This Changes

The issue asks for session persistence to stop repeatedly accumulating `skillsSnapshot` and `systemPromptReport` data that causes `sessions.json` growth and context/UI failures.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/48425 or a scoped replacement that reduces transient session JSON at the shared serialization boundary while preserving only the prompt-report state needed for warning dedupe.

## Reproduction Assessment

Yes. Source inspection on current main shows the session write, disk-budget, and run-completion paths still retain the reported fields, so the bloat path is reproducible from code without running a live gateway workload.

## Solution Assessment

Yes. A shared session-entry serialization boundary with matching disk-budget measurement is the narrowest maintainable fix, provided it preserves the lightweight snapshot and warning-dedupe contracts already on main.
