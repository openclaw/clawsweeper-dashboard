## Summary

Keep open: current main and v2026.6.6 mitigate the largest `skillsSnapshot.prompt` duplication with content-addressed prompt blobs, but current source and tests still intentionally persist `systemPromptReport`. The prior dsantoreis and cncoder comments add useful root-cause and real-world dedupe evidence, and the linked PR is still open/conflicting while preserving the remaining report field.

## What This Changes

The issue asks OpenClaw to stop session-store bloat from persisted `skillsSnapshot` and `systemPromptReport` metadata across repeated agent runs.

## Best Possible Solution

Define a narrow retained report shape that preserves bootstrap-warning dedupe and any intended `contextWeight` API data without persisting the full per-run report payload, then revise or close the linked PR around the shipped promptRef/blob design.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a live current-main repro in this read-only pass; source shows the remaining `systemPromptReport` persistence path and tests require it, while the largest `skillsSnapshot.prompt` duplication is already mitigated.

## Solution Assessment

Is this the best way to solve the issue? No for the linked whole-field stripping approach: current main already uses promptRef blobs, and the remaining report behavior needs a narrower retention design that preserves warning dedupe.
