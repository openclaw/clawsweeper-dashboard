## Summary

Keep open: current main and v2026.6.6 mitigate the largest persisted skill-prompt duplication with content-addressed prompt blobs, but the remaining `systemPromptReport` persistence and stale-session rebuild behavior are still present and need a maintainer product/compatibility decision.

## What This Changes

The issue asks OpenClaw to stop session-store growth caused by repeated persisted skill snapshot and system prompt report metadata.

## Best Possible Solution

Keep the promptRef/blob design, define a smaller persisted report shape that preserves required warning dedupe state, and close or rework the conflicting linked PR around that policy.

## Reproduction Assessment

No high-confidence full live reproduction was established for current main. Source review does reproduce the remaining disputed behavior: `systemPromptReport` is still persisted and stale sessions can still be treated as new, while large skill prompts are already mitigated.

## Solution Assessment

No, the linked whole-field stripping approach is not the best current fix because main now relies on promptRef/blob persistence and intentionally keeps report state for warning dedupe. The better path is a narrow persisted-report policy with regression coverage.
