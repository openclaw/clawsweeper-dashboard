## Summary

Keep open. Current main only strips the runtime-only `skillsSnapshot.resolvedSkills` payload; it still persists `skillsSnapshot.prompt` and `systemPromptReport`, and the open prompt-stripping fix at https://github.com/openclaw/openclaw/pull/78259 is not merged and currently reports conflicts.

## What This Changes

The issue reports unbounded session-store growth from repeated persistence of `skillsSnapshot` and `systemPromptReport` data across agent runs.

## Best Possible Solution

Review and land a corrected prompt-stripping fix that preserves resume and warning-dedupe semantics, then decide whether any narrower `systemPromptReport` compaction is still needed before closing this as implemented.

## Reproduction Assessment

Yes, at source level: current main saves retained `skillsSnapshot.prompt` and assigns `systemPromptReport` after runs, and the issue/PR discussion includes real deployment size measurements. I did not run a live gateway workload in this read-only review.

## Solution Assessment

Unclear as a complete fix: stripping `skillsSnapshot.prompt` appears to be the right direction, but the open PR needs conflict and resume-semantics review, while `systemPromptReport` needs a narrower durability decision because current tests preserve it for warning dedupe.
