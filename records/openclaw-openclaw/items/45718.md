## Summary

Keep open. Current main still serializes full session entries, only strips `skillsSnapshot.resolvedSkills`, still persists `systemPromptReport`, and the open implementation PR that uses closing syntax for this issue has not merged.

## What This Changes

The issue reports unbounded session-store growth because `skillsSnapshot` and `systemPromptReport` are repeatedly persisted across agent runs.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/48425 or a scoped replacement that omits only top-level `SessionEntry.skillsSnapshot`, aligns disk-budget sizing with actual writes, and preserves the prompt-report state needed for warning dedupe.

## Reproduction Assessment

Yes. Source inspection on current main shows agent runs can persist `skillsSnapshot`, the store and disk-budget paths stringify full entries, and run completion still copies `systemPromptReport`.

## Solution Assessment

Yes, a shared session-entry serialization boundary is the narrowest maintainable direction. The current open PR is close but still needs its serializer scoped to top-level session entries before it can safely solve this issue.
