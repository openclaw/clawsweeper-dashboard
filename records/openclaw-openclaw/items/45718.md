## Summary

Keep open: current main and v2026.6.6 address the largest `skillsSnapshot.prompt` duplication with prompt blobs, but the issue is not fully cleanup-closeable because `systemPromptReport` still persists on every reported run path and the open linked closing PR is conflicting and preserves that field.

## What This Changes

The issue reports unbounded session-store growth from repeated `skillsSnapshot` and `systemPromptReport` persistence across agent runs.

## Best Possible Solution

Keep the shipped promptRef/blob storage design, decide the minimal durable `systemPromptReport` state needed for warning dedupe, and then update or close the linked fork PR around that narrower contract.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a live current-main repro in this read-only review; source inspection does reproduce the remaining `systemPromptReport` persistence and stale-session snapshot rebuild condition, while current main mitigates the dominant prompt duplication.

## Solution Assessment

Is this the best way to solve the issue? No for a whole-field stripping PR as-is; the best fix is narrower durable report state plus the already-shipped promptRef/blob design, with explicit maintainer agreement on warning-dedupe semantics.
