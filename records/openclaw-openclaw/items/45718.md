## Summary

Keep open. Current main only removed the largest `resolvedSkills` payload; it still persists `skillsSnapshot.prompt` and `systemPromptReport`, and an open prompt-stripping PR plus fresh production data show the remaining storage issue is still active.

## What This Changes

The issue asks OpenClaw to stop repeatedly persisting static `skillsSnapshot` and `systemPromptReport` data into session state so `sessions.json` does not grow unbounded.

## Best Possible Solution

Land a narrow, backward-compatible storage reduction after the open prompt-stripping work preserves resume semantics, then explicitly decide how much `systemPromptReport` state must remain durable for warning dedupe and diagnostics.

## Reproduction Assessment

Yes, at source level: current main serializes retained `skillsSnapshot.prompt` and `systemPromptReport` fields into the full session store, so many retained sessions reproduce the storage growth path. I did not run a live gateway growth workload in this read-only review.

## Solution Assessment

Unclear for the complete issue. Stripping and rebuilding `skillsSnapshot.prompt` is the best-supported direction from the open PR and latest production data, but the durable `systemPromptReport` contract still needs maintainer judgment before this can be closed.
