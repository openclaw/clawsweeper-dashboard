## Summary

Keep open: current main and the latest release mitigate the largest `skillsSnapshot.prompt` duplication with content-addressed prompt blobs, but the issue is not fully obsolete because `systemPromptReport` is still intentionally persisted and stale unrequested sessions still take the `isNewSession` path. ## What This Changes

The issue requests bounded session persistence so `skillsSnapshot` and `systemPromptReport` do not grow `sessions.json` across repeated agent runs.

## Best Possible Solution

Keep the shipped prompt-blob mitigation, then have maintainers adapt or close the open PR while narrowing the remaining durable `systemPromptReport` and stale-session snapshot policy into one tested session-store change.
