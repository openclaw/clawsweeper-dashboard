## Summary

Keep open: current main and v2026.6.6 mitigate the largest skillsSnapshot.prompt duplication with content-addressed prompt blobs, but the issue is not fully resolved because systemPromptReport is still deliberately persisted and the open linked PR is conflicting and only partial.

## What This Changes

The issue requests bounded session persistence so skillsSnapshot and systemPromptReport data do not repeatedly bloat sessions.json across agent runs.

## Best Possible Solution

Define a narrow retention policy that keeps the needed bootstrap-warning dedupe state without persisting the full report payload, then close, replace, or substantially revise https://github.com/BingqingLyu/openclaw/pull/869 around the shipped promptRef design.

## Reproduction Assessment

Yes, source-level only: current main still writes systemPromptReport into session entries and has a test requiring that persistence. I did not run a live size-growth reproduction, and the skillsSnapshot.prompt part is already partially mitigated by promptRef blobs.

## Solution Assessment

No, the linked whole-field stripping approach is not the best current fix because main now relies on promptRef/blob persistence and the remaining systemPromptReport behavior is intentional for warning dedupe. The best fix is a narrower report-retention design that preserves the required dedupe state.
