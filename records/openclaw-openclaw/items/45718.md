## Summary

Keep open. Current main still persists `skillsSnapshot.prompt` and durable `systemPromptReport`; the prompt-stripping implementation at https://github.com/openclaw/openclaw/pull/78259 remains open/conflicting, and the prompt-report portion still needs a retention decision because warning dedupe currently depends on persisted report metadata.

## What This Changes

The issue asks OpenClaw to stop session-store growth by avoiding repeated persistence and rebuilds of `skillsSnapshot` and `systemPromptReport` across agent runs.

## Best Possible Solution

Land a corrected prompt-stripping fix with resume coverage, then define a narrow `systemPromptReport` retention or compaction contract that preserves warning dedupe.

## Reproduction Assessment

Yes, at source level: current main preserves `skillsSnapshot.prompt`, assigns `systemPromptReport` after runs, and rebuilds snapshots for stale sessions. I did not run a live gateway workload in this read-only review.

## Solution Assessment

Unclear as a complete fix: stripping `skillsSnapshot.prompt` through the linked PR is the right direction, but `systemPromptReport` persistence needs a narrower durability decision because current tests preserve it for warning dedupe.
