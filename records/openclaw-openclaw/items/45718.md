## What This Changes

The issue reports unbounded session-store growth from repeated `skillsSnapshot` and `systemPromptReport` persistence across agent runs.

## Best Possible Solution

Define a narrow retained report shape that preserves bootstrap-warning dedupe and any intended context-weight API data without persisting the full per-run report payload, then revise or close the linked PR around the shipped promptRef/blob design.

## Reproduction Assessment

No live current-main reproduction was run in this read-only review. Source inspection reproduces the remaining path: run metadata still writes `systemPromptReport` into session state and tests require that field for warning dedupe, while the dominant `skillsSnapshot.prompt` duplication is already mitigated.

## Solution Assessment

No for the linked whole-field stripping approach as-is. The best fix is narrower: keep the shipped promptRef/blob design and decide what minimal durable report state is needed for warning dedupe instead of preserving the full report payload.
