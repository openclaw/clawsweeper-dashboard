## Summary

Keep open. Current main still has a source-level path that persists the reported session fields, with only `skillsSnapshot.resolvedSkills` mitigated, and the open implementation PR with closing syntax has not merged.

## What This Changes

The issue asks OpenClaw to stop repeated `skillsSnapshot` and `systemPromptReport` persistence from bloating `sessions.json` across agent runs and stale sessions.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/48425 or a scoped replacement that removes bulky recomputable session payloads at the shared persistence boundary while preserving warning dedupe and cold-resume snapshot semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows the save, disk-budget, run-completion, and stale-session snapshot paths still retain the reported fields; I did not run a live gateway workload.

## Solution Assessment

Yes directionally. A shared persistence/serialization boundary with matching disk-budget measurement is the narrow maintainable fix, provided it preserves the existing warning-dedupe and snapshot-metadata contracts.
