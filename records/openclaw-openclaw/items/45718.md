## Summary

The central session-bloat defect is fixed: OpenClaw now stores repeated large skill prompts as content-addressed blobs instead of duplicating them inline in every session entry. The separate claim that systemPromptReport accumulates on every run is not supported by current code, which replaces the latest report and intentionally preserves restart warning-dedupe state.

## What This Changes

Prevent repeated skillsSnapshot and systemPromptReport data from bloating persisted session state across agent runs.

## Best Possible Solution

Keep the shipped content-addressed skill-prompt storage and the latest durable warning-dedupe state; open a narrower new issue only if current-release measurements show material residual systemPromptReport bloat.

## Reproduction Assessment

Yes for the historical inline skillsSnapshot.prompt duplication: source and merged history establish the old repeated payload and the current deduplicated representation. No for the claimed per-run systemPromptReport accumulation, because current code overwrites that field.

## Solution Assessment

Yes for the central user problem: content-addressed prompt blobs deduplicate repeated prompts while retaining hydration, recovery, disk accounting, and cleanup. Removing systemPromptReport wholesale would regress restart warning dedupe.
