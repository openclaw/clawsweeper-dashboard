## Summary

Keep open: current main and v2026.6.10 mitigate the largest duplicated `skillsSnapshot.prompt` payload with content-addressed prompt blobs, but `systemPromptReport` is still persisted into session entries and the linked closing PR has not landed.

## What This Changes

The issue asks OpenClaw to stop `sessions.json` from growing unbounded by repeatedly retaining `skillsSnapshot` and `systemPromptReport` metadata across agent runs.

## Best Possible Solution

Keep this issue as the canonical tracker and land a maintainer-approved session metadata policy that preserves minimal warning-dedupe state while preventing stale or inactive entries from retaining bulky prompt-report payloads.

## Reproduction Assessment

Yes at source level: current main still writes `systemPromptReport` into durable session entries after runs. No live long-running size-growth reproduction was run in this read-only review.

## Solution Assessment

No, not yet: stripping all prompt reports and preserving them unchanged are both incomplete because warning dedupe is load-bearing. The best fix should persist only the minimal durable state needed for dedupe and compact or omit bulky report details from stale rows.
