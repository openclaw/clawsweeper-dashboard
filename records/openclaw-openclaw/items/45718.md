## Summary

Keep open. Current main fixes the largest `resolvedSkills` payload, but it still intentionally persists `systemPromptReport` and lightweight `skillsSnapshot` metadata, so the remaining storage contract needs maintainer/product review rather than a cleanup close.

## What This Changes

The issue asks OpenClaw to stop `sessions.json` from growing by avoiding repeated `skillsSnapshot`/`systemPromptReport` persistence and stale-session snapshot rebuilds.

## Best Possible Solution

Define and test a narrow session-persistence policy that strips recomputable large payloads while preserving required warning-dedupe and snapshot-resume metadata.

## Reproduction Assessment

Yes, at source level. Current main still persists `systemPromptReport` and lightweight `skillsSnapshot` metadata, and the stale-session agent-command path can still rebuild snapshots when a session is not fresh; I did not run a live gateway workload.

## Solution Assessment

Unclear. Deleting whole fields at serialization is too broad; the maintainable direction is a narrower persistence/hash policy that keeps dedupe and resume metadata while avoiding large recomputable payloads.
