The branch makes CLI config mutation choose missing object versus array containers from the runtime config schema, with regression tests for numeric Discord/Telegram record keys and schema-backed `agents.list` indexes.

Land the schema-aware config writer fix after normal maintainer review so https://github.com/openclaw/openclaw/issues/83331 can close through this PR’s closing reference.

## Reproduction Assessment

Yes, at source level. Current main still uses an all-digit next-segment heuristic for path-based `config set`, while the schema and docs require numeric-looking channel IDs under record/object paths to remain object keys.

## Solution Assessment

Yes. The schema-aware approach is the narrow maintainable fix because it preserves numeric record keys without repeating the global object-key change that broke documented array-index paths in https://github.com/openclaw/openclaw/pull/83341.
