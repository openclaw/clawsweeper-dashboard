## Summary

Keep this PR open, but it is not merge-ready: the branch changes the shipped `mentionPatterns.mode: "allow"` contract, leaves generated metadata/public docs on the old semantics, and still lacks inspectable real behavior proof.

## What This Changes

The PR changes explicit provider `mentionPatterns.mode: "allow"` into an allowlist scoped by `allowIn`, updates source UI hints for five channels, and adds resolver-focused tests.

## Best Possible Solution

Preserve broad `allow` semantics and use `mode: "deny"` plus `allowIn` for scoped allowlists, or land a maintainer-approved breaking contract change with docs, generated metadata, upgrade guidance, and real transport proof.

## Reproduction Assessment

Yes. Source inspection shows current main enables explicit `allow` mode unless `denyIn` matches, and the PR changes that path; deciding it is a bug requires maintainer compatibility judgment because docs and the latest release define the current behavior as the contract.

## Solution Assessment

No. The existing `mode: "deny"` plus `allowIn` path already provides scoped allowlists without changing the shipped `allow` contract, so this is not the narrowest maintainable fix as written.
