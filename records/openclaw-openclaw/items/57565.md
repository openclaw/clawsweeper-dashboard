## Summary

Keep open: current main still routes Mattermost channel/group thread replies through the suppressed `effectiveReplyToId` when `replyToMode` is off, while this PR is useful but not ready to merge as-is because it is entangled with open related work and lacks inspectable after-fix real behavior proof.

## What This Changes

The branch separates Mattermost delivery/thread context from session routing for `replyToMode: "off"`, adds unknown-channel-type handling, updates typing/tool context, and adjusts session reset classification.

## Best Possible Solution

Land a fresh current-main fix that keeps `effectiveReplyToId` authoritative for session forking while carrying a separate non-DM Mattermost delivery/tool thread id for channel and group thread replies, coordinated with the unknown-channel-type and message-tool follow-up PRs.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: a channel/group post with `post.root_id` and `replyToMode: "off"` makes `effectiveReplyToId` undefined, and the outbound reply roots still use that undefined value.

## Solution Assessment

Yes conceptually, but not as a direct merge yet. Separating session routing from delivery routing is the narrow maintainable fix; the branch should be rebased or replaced on current main while sequencing the related unknown-channel-type and message-tool context work.
