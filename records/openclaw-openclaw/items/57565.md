## Summary

Keep this PR open: the Mattermost `replyToMode: "off"` thread-delivery bug is still present on current `main`, but this stale branch should be replaced or rebased rather than merged directly because it is stacked on #57970 and predates current Mattermost routing drift.

## What This Changes

This PR separates Mattermost delivery/tool thread context from session routing for `replyToMode: "off"`, adds unknown channel-type handling, and adjusts non-forked session classification.

## Best Possible Solution

Land a fresh current-main fix that keeps `effectiveReplyToId` authoritative for session forking while carrying the actual non-DM Mattermost thread root through delivery, typing, draft preview, and tool context, with focused tests and a changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction exists on current main: channel/group posts with `post.root_id` and `replyToMode: "off"` produce `effectiveReplyToId: undefined`, and the affected delivery paths still use that undefined value for thread routing.

## Solution Assessment

Yes conceptually, but not as a direct merge of this branch. Separating session routing from delivery routing is the narrow maintainable fix; the safer path is a current-main replacement or rebase that also handles #57970 and newer Mattermost paths.
