## Summary

Keep open: the Mattermost `replyToMode: "off"` delivery bug is still present on current `main`, but this external PR is not ready to merge because it lacks inspectable real behavior proof, the author has abandoned it, and the branch broadens the shared `ChatType` contract for a Mattermost-local lookup failure.

## What This Changes

The PR separates Mattermost delivery thread identity from session routing for `replyToMode: "off"`, updates reply/typing/context call sites and tests, and also adds `unknown` to shared chat-type routing surfaces.

## Best Possible Solution

Land a fresh or rebased Mattermost-local fix that carries the actual inbound thread root separately from session forking, keeps shared `ChatType` as `direct | group | channel`, adds focused delivery/session tests, and includes redacted real Mattermost proof.

## Reproduction Assessment

Yes, at source level. Current docs require off-mode inbound thread replies to stay in the thread, while current main suppresses the thread root and then uses that suppressed value for `MessageThreadId`, typing, and final delivery.

## Solution Assessment

No, not as a direct merge. Separating session routing from delivery routing is the right fix shape, but the shared `ChatType` expansion should be replaced with a Mattermost-local handling path.
