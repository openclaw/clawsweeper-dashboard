## Summary

Keep open: current main still has a source-provable Mattermost `replyToMode: "off"` thread-delivery bug, but this branch is not ready because it broadens shared `ChatType`, overlaps safer Mattermost-local follow-up work, is conflicting, and lacks inspectable real behavior proof.

## What This Changes

The PR separates Mattermost delivery thread state from session-routing state, routes replies, typing, and tool context through that thread id, changes missing Mattermost channel type handling to `unknown`, and adjusts session reset classification plus tests.

## Best Possible Solution

Land a fresh or rebased Mattermost-local fix that carries the inbound thread root separately from session forking, keeps missing channel metadata handling out of shared core `ChatType`, coordinates with https://github.com/openclaw/openclaw/pull/78530 and https://github.com/openclaw/openclaw/pull/52120, and includes focused tests plus redacted real Mattermost proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: docs promise off-mode replies stay in existing Mattermost threads, while current main suppresses the thread root and then uses that suppressed value for reply delivery.

## Solution Assessment

No, not as a direct merge. Separating session routing from delivery routing is the right fix shape, but missing channel-type handling should stay Mattermost-local and this branch needs a rebase plus inspectable proof.
