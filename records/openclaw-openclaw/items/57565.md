## Summary

Keep open: the Mattermost bug is still present on current main, but this external branch should not merge as-is because it lacks inspectable after-fix Mattermost proof, is abandoned/conflicting, and still carries a shared ChatType approach that should stay Mattermost-local.

## What This Changes

The PR separates Mattermost session routing from delivery/thread context for `replyToMode: "off"`, updates reply/typing/tool-context paths, adjusts session reset classification, and also widens shared ChatType/Slack/SDK typing for missing Mattermost channel types.

## Best Possible Solution

Land a fresh current-main Mattermost fix that keeps `effectiveReplyToId` for session forking, carries a separate actual thread root for delivery/tool context, coordinates with https://github.com/openclaw/openclaw/pull/78530, and includes focused tests plus redacted real Mattermost proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: docs promise off-mode replies stay in an existing Mattermost thread, while current main suppresses the only thread id and then uses that suppressed value for reply delivery; I did not run a live Mattermost server.

## Solution Assessment

No, not as a direct merge. Separating session routing from delivery routing is the right fix shape, but this branch should be rebased or replaced to avoid the shared `unknown` ChatType approach and to add inspectable real behavior proof.
