## Summary

Keep open: the Mattermost bug is still source-reproducible on current main, but this PR should not merge as-is because it carries a superseded unknown-channel-type approach, has blocking review findings, and lacks inspectable real behavior proof.

## What This Changes

The branch separates Mattermost delivery/tool thread context from session routing for `replyToMode: "off"`, updates reply, typing, context, session-reset, and tests, and also carries unknown-channel-type/shared `ChatType` changes.

## Best Possible Solution

Land a fresh current-main Mattermost fix that keeps `effectiveReplyToId` authoritative for session forking while carrying a separate non-DM delivery/tool thread id, coordinated with the Mattermost-local channel-type replacement and message-tool context work.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: docs promise `replyToMode: "off"` replies stay in an existing thread, but threaded channel/group posts set `effectiveReplyToId` to `undefined` and delivery uses that value as the reply root.

## Solution Assessment

No, not as a direct merge. Separating session routing from delivery routing is the right fix shape, but this branch should be rebased or replaced after removing the superseded shared `unknown` chat type approach and adding inspectable real behavior proof.
