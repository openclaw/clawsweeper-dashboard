## Summary

Keep open: current main still has the documented Mattermost `replyToMode: "off"` thread-delivery bug, but this external PR is not merge-ready because real behavior proof is missing and the included unknown-channel-type dependency is superseded by a narrower replacement with a concrete functional issue still present here.

## What This Changes

This PR separates Mattermost delivery/tool thread context from session routing, routes replies and typing through that thread id, adjusts session reset classification, and pulls in unknown-channel-type handling.

## Best Possible Solution

Land a fresh current-main Mattermost fix that keeps `effectiveReplyToId` authoritative for session forking while carrying a separate non-DM delivery/tool thread id, based on or coordinated with the Mattermost-local missing-channel-type replacement and the message-tool context work.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: a channel/group post with `post.root_id` and `replyToMode: "off"` makes `effectiveReplyToId` undefined, and the outbound reply root still uses that undefined value.

## Solution Assessment

No, not as a direct merge. Separating session routing from delivery routing is the right fix shape, but this branch should be rebased or replaced after the superseded unknown-channel-type dependency is removed and inspectable real behavior proof is supplied.
