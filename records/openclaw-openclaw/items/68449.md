## Summary

Keep this open: current main has several useful fixes for source stopwords, cron transcript classification, silent replies, heartbeat noise, and archive handling, but the issue discussion's remaining legacy/read-path and light-narrative hygiene gaps are still visible in source.

## What This Changes

The issue asks memory-core Dreaming to filter broader concept noise and exclude cron/silent automation transcripts from session corpus ingestion.

## Best Possible Solution

Memory-core should converge on one shared Dreaming concept-tag sanitizer for derive, REM, and light narrative inputs, plus a metadata-backed and documented session exclusion path coordinated with #72611.

## Reproduction Assessment

Yes, source-level. Persisted tags such as `theme`, `kept`, `surfacing`, or `1.00` can still flow through REM/light Dreaming paths, and direct cron prompts in ordinary sessions can still leave paired assistant narration by design.

## Solution Assessment

No. Current main is a partial fix; the narrow maintainable repair is shared tag sanitation, while the broader cron/session exclusion needs a security-aware metadata or config design rather than ad hoc narration regexes.
