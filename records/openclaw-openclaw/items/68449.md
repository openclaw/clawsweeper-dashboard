## Summary

Keep this issue open: current main has partial source-side stopword and cron transcript filtering, but active discussion and source inspection still show remaining Dreaming hygiene gaps in persisted/read-path tags, light narrative themes, and REM candidate truth selection from session-corpus entries.

## What This Changes

Dreaming should filter stopword, transcript, NO_REPLY, and cron-delivery noise so light/REM/deep phases promote meaningful user memory rather than automation artifacts.

## Best Possible Solution

Converge on one Dreaming hygiene path: shared concept-tag sanitization for derive, REM, and light narrative inputs, plus metadata/config-backed session exclusion for cron and rotated transcripts coordinated with https://github.com/openclaw/openclaw/issues/72611 and a replacement for https://github.com/openclaw/openclaw/pull/72913.

## Reproduction Assessment

Yes. Current-source inspection gives a high-confidence reproduction path: persisted conceptTags can flow into REM/light outputs without shared sanitization, and session-corpus entries can become REM Possible Lasting Truths without a source-type guard; I did not replay a live production corpus.

## Solution Assessment

No. Current main is only a partial fix; the maintainable solution is shared sanitization plus metadata/config-backed exclusion, not more compiled-dist patches or broad ad hoc narration regexes.
