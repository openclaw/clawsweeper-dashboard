## Summary

Keep open: current main has partial stopword and cron-transcript filtering, but the remaining Dreaming hygiene gaps are still visible in source and active in recent discussion.

## What This Changes

The issue asks memory-core Dreaming to filter stopword/transcript artifacts and keep cron, NO_REPLY, and automation noise out of corpus ingestion and promotion surfaces.

## Best Possible Solution

Unify Dreaming hygiene across concept derivation, stored-tag read paths, light/REM narrator inputs, and REM candidate-truth source selection, while coordinating the session/cron exclusion contract with https://github.com/openclaw/openclaw/issues/72611 and the denoise work in https://github.com/openclaw/openclaw/pull/71837.

## Reproduction Assessment

Yes, at source level. Current main still routes raw stored conceptTags into light narrator themes, uses only a five-token REM tag blacklist, and selects REM candidate truths without a source-type guard; I did not run a live deployment replay.

## Solution Assessment

No, current main is only a partial fix. The maintainable solution is a shared sanitizer/source-exclusion contract plus maintainer-approved cron/session filtering policy, not more isolated dist-level filters.
