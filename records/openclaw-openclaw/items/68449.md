## Summary

Keep open: current main has partial stopword, cron-transcript, heartbeat, and silent-reply filtering, but the remaining Dreaming hygiene and REM promotion gaps are still source-visible and active in related discussion.

## What This Changes

The issue reports that Dreaming concept extraction and corpus ingestion allow stopword/transcript noise plus cron or NO_REPLY session artifacts to dominate themes and REM candidates.

## Best Possible Solution

Unify Dreaming hygiene across concept derivation, REM reflections, light narrator input, REM candidate truths, and session-corpus promotion, while coordinating the cron/source exclusion contract with https://github.com/openclaw/openclaw/issues/72611 and the denoise work in https://github.com/openclaw/openclaw/pull/71837.

## Reproduction Assessment

Yes, at source level. Current main still lets raw stored `conceptTags` feed light narrator themes, uses only a five-token REM tag blacklist, and selects REM candidate truths without a source-type guard; I did not replay the reporter's live deployment corpus.

## Solution Assessment

No, current main is not the complete solution. The maintainable fix is shared tag/source sanitization plus a maintainer-approved cron/session-corpus boundary, not more isolated dist-level or narration-prefix filters.
