Keep open: current main partially fixes the original derive-time stopwords and cron transcript classification, but source inspection and recent issue discussion still show remaining Dreaming noise paths in session-corpus ingestion, Light/REM theme inputs, REM candidate truths, and promotion boundaries.

## What This Changes

The issue asks memory-core Dreaming to broaden concept stopword filtering and exclude cron, NO_REPLY, and automation noise from session-corpus ingestion and downstream Dreaming outputs.

## Best Possible Solution
