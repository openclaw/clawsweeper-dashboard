## Summary

Keep open: current main has partial stopword, cron-run, heartbeat, and silent-reply filtering, but the remaining Dreaming hygiene bugs still have clear source paths and recent production confirmation. The related configurable-exclusion issue and denoise PR are still open/unmerged, so there is no terminal cleanup reason yet.

## What This Changes

Dreaming should filter stopword/transcript artifacts and cron or NO_REPLY session-corpus noise so light, REM, and deep phases promote meaningful user interaction signals.

## Best Possible Solution

Memory-core should use one shared Dreaming hygiene path for derive, REM, and light narrator inputs, then add source-type/session exclusion for cron and Dreaming artifacts in coordination with https://github.com/openclaw/openclaw/issues/72611.

## Reproduction Assessment

Yes, at source level. Current main still lets persisted noisy tags reach REM/light paths and lets REM Possible Lasting Truths select snippets without a cron/session-corpus source guard; I did not replay the reporter's live deployment corpus.

## Solution Assessment

No. Current main is only a partial fix; the maintainable solution is shared tag/source sanitization plus a reviewed session-exclusion contract, not more dist patches or broad narration-prefix regexes.
