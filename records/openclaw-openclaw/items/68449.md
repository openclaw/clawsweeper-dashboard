## Summary

Keep this issue open. Current main has partial Dreaming cleanup for source-side stopwords, deleted transcript artifacts, classified cron transcripts, direct cron prompts, heartbeat prompts, and silent replies, but the remaining report is still real: persisted conceptTags are only narrowly filtered on the REM read path, light Dream Diary themes still use raw conceptTags, decimal/template legacy tags can still surface, and unclassified direct cron prompts still intentionally do not suppress paired assistant narration.

## What This Changes

The issue asks memory-core Dreaming to filter common transcript/score/template noise from concept themes and exclude cron/NO_REPLY session noise from session-corpus ingestion.

## Best Possible Solution

Keep the issue open until memory-core has one coherent Dreaming hygiene path: shared concept-tag sanitization for derive, REM, and light narrative inputs; read-path handling for legacy decimal/template tags; and cron/session exclusion via reliable metadata plus documented operator config coordinated with #72611.

## Reproduction Assessment

Yes. Static source and tests provide a high-confidence reproduction path: feed persisted conceptTags such as `theme`, `kept`, `surfacing`, or `1.00` into REM/light Dreaming, or inspect an unclassified session with a direct `[cron:...]` user prompt followed by assistant narration; current code still admits those remaining classes.

## Solution Assessment

No, current main is not yet the complete best solution. The narrow maintainable direction is a shared tag sanitizer plus metadata-backed/configurable session exclusion, not more ad hoc dist-level patches or a cross-message regex that can be spoofed.
