## Summary

Keep open: current main partially addresses the original derive-time stopwords and cron transcript classification, but source inspection and recent live reports still show unhandled Dreaming noise paths in light themes, REM reflections, REM candidate truths, and session-corpus promotion boundaries.

## What This Changes

The issue asks memory-core Dreaming to stop surfacing stopword/transcript artifacts and to exclude cron, NO_REPLY, silent-mode, and cron-delivery noise from corpus ingestion and promotion surfaces.

## Best Possible Solution

Unify Dreaming hygiene across concept derivation, stored-tag read paths, session-corpus ingestion, light/REM narrator inputs, and REM candidate-truth source selection, with a maintainer-approved session/cron exclusion contract and focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main still has raw stored-tag light themes, a narrow REM tag blacklist, session-corpus promotability, and REM candidate-truth selection without a source-type guard; I did not run a live multi-day cron deployment replay.

## Solution Assessment

No, current main is only a partial fix. The narrow durable solution is a shared Dreaming sanitizer/source-exclusion contract plus maintainer-approved cron/session filtering policy, not more isolated dist-level patches.
