## Summary

Keep open: current main partially addresses the original stopword and cron-transcript problems, but source inspection and recent issue evidence still show unresolved Dreaming noise paths in session-corpus recall, Light theme input, REM reflections/truth selection, and cron/process-output policy.

## What This Changes

The issue asks for Dreaming to filter transcript stopwords and exclude cron/NO_REPLY/session noise so top themes and memory promotion reflect real user interactions.

## Best Possible Solution

Define one maintainer-approved Dreaming hygiene boundary and apply it consistently across session ingestion, Light narrator themes, REM reflections/truths, and promotion eligibility, with docs and configuration only where the boundary is meant to be operator-controlled.

## Reproduction Assessment

Yes, at source level. Current main still accepts session-corpus entries as short-term memory paths and REM candidate truths without a cron/source guard, and the ingestion loop has no content hygiene gate after snippet normalization; I did not run a live multi-day cron replay.

## Solution Assessment

No, current main and the linked PRs are only partial fixes. The maintainable solution is a coordinated boundary for generated/session-corpus/cron evidence rather than another isolated stopword or snippet filter.
