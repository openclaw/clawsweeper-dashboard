## Summary

Keep this open: current main has partial hygiene fixes, but source inspection still shows unfiltered legacy/read-path concept tags, raw light-dream narrative themes, and a cron/session exclusion gap that overlaps #72611.

## What This Changes

The issue asks memory-core Dreaming to suppress stopword/template concept noise and keep cron/NO_REPLY automation transcripts out of the session corpus.

## Best Possible Solution

Memory-core should share one concept-tag sanitizer across derive, REM, and light narrative inputs, then handle remaining cron/session exclusion through metadata-backed filtering or the configurable path tracked in #72611.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: persisted conceptTags such as theme/kept/surfacing/1.00 can still feed REM and light themes, and direct cron prompt filtering intentionally does not suppress paired assistant narration in ordinary transcripts.

## Solution Assessment

No. Current main is a partial fix; the narrow maintainable repair is shared tag sanitization, while the cron/session exclusion policy should be resolved through metadata/config rather than broader ad hoc narration regexes.
