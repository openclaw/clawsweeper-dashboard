## Summary

Keep open: current main has partial stopword and cron-ingestion filtering, but Dreaming still has source-visible gaps for legacy/noisy concept tags, Light/REM narrator input, and REM candidate-truth selection from session-corpus artifacts. The related denoise PR and configurable session-filter issue are still open, so this should stay available as the active bug thread.

## What This Changes

The issue reports Dreaming memory-core noise from narrow concept stopwords and cron/NO_REPLY session-corpus ingestion that causes meaningless themes and REM candidates.

## Best Possible Solution

Unify Dreaming hygiene across derive-time tags, stored-tag read paths, Light/REM narrator inputs, and REM candidate-truth source selection, while coordinating the session/cron exclusion contract with https://github.com/openclaw/openclaw/issues/72611 and the denoise work in https://github.com/openclaw/openclaw/pull/71837.

## Reproduction Assessment

Yes, at source level. Current main still has clear paths where raw stored conceptTags feed Light/REM output and session-corpus entries can become REM candidate truths; I did not run a live deployment replay.

## Solution Assessment

No, current main is only a partial solution. The maintainable fix is a shared sanitizer/source-exclusion contract plus maintainer-approved cron/session filtering policy, not more isolated dist-level filters.
