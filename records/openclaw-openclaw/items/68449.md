## Summary

Keep this issue open: current main contains useful partial fixes for source-side stopwords, cron-run transcript classification, deleted/checkpoint artifacts, direct cron prompts, silent replies, and heartbeat noise, but the remaining Dreaming hygiene problem is still real on the legacy/read paths and light narrative input path.

## What This Changes

The issue requests that memory-core Dreaming stop surfacing transcript/stopword artifacts as themes and exclude cron or NO_REPLY noise from session corpus ingestion.

## Best Possible Solution

Memory-core should converge on one Dreaming hygiene path: shared concept-tag sanitization for derive, REM, and light narrative inputs, plus metadata-backed session exclusion and documented operator filters coordinated with #72611.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path for the remaining bug: persisted tags such as `theme`, `kept`, `surfacing`, or `1.00` can still flow through REM/light Dreaming paths, and unclassified cron-like prompts can still leave paired assistant narration because current code intentionally avoids cross-message suppression.

## Solution Assessment

No. Current main is a partial fix; the maintainable path is a shared sanitizer plus metadata/config-backed session exclusion, not more compiled-dist patches or increasingly broad ad hoc narration regexes.
