## Summary

Keep open: current main still has the Feishu streaming architecture that can mix reasoning, answer snapshots, and public tool-progress status in one mutable CardKit reply, and no merged fix or shipped release resolves this reporter's noisy/repeated stream behavior. The related Feishu streaming-card cluster has useful open PRs and issues, but none is merged and safe enough to close this issue as implemented or superseded.

## What This Changes

The issue reports that Feishu streaming replies in v2026.6.5 repeatedly show thinking/reasoning content and interleave tool execution status instead of presenting a clean final answer.

## Best Possible Solution

Land one maintainer-chosen Feishu/shared progress fix that keeps final replies readable, prevents repeated reasoning and status pollution, preserves only intentional public progress previews, and coordinates with the open Feishu streaming-card cluster.

## Reproduction Assessment

Yes at source level for visible tool-status interleaving: Feishu's streaming dispatcher appends work-tool status into the same streaming card updates and tests assert that behavior. The exact repeated-reasoning sequence for xiaomi-tp/mimo-v2.5 still needs a normalized event trace or live current-main reproduction.

## Solution Assessment

Unclear as a final implementation plan. The Feishu dispatcher is the right owner for card mutation and status cleanup, but the visibility of progress/reasoning is partly intentional today and overlaps open PRs, so maintainers should pick one canonical behavior before automation changes it.
