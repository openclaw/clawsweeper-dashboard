## Summary

Keep open. Current main still has no devcontainer surface, and this PR is the active implementation candidate, but the proposed patch should not merge yet because host credentials are injected container-wide before dependency installation, the Copilot wording is stale, and the user-facing feature is missing the required changelog entry.

## What This Changes

The PR adds a VS Code devcontainer Dockerfile/config/bootstrap script, devcontainer docs and nav entry, labeler coverage, a zh-CN glossary update, and file-content tests for a Docker/Podman contributor environment.

## Best Possible Solution

Keep the devcontainer direction, but revise the bootstrap so dependency installation runs without forwarded host secrets, keep auth-only checks separate, align the Copilot wording in docs/comments, add the required changelog entry, rebase, and then validate the targeted devcontainer docs/tests plus one Docker and one Podman create path.

## Reproduction Assessment

Yes for the blocking review finding: on the PR head, setting GH_TOKEN or ANTHROPIC_API_KEY on the host makes those variables available through containerEnv while postCreateCommand runs pnpm install first. For the feature itself, reproduction is not a bug flow; current main simply lacks the requested devcontainer files.

## Solution Assessment

No. Adding a devcontainer is the right product surface, but this implementation should not expose host secrets to install-time lifecycle scripts and should finish the docs/changelog cleanup before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep install scripts away from forwarded tokens:** `.devcontainer/devcontainer.json:11-14`
  - body: `containerEnv` makes `GH_TOKEN` and `ANTHROPIC_API_KEY` available to every process while `postCreateCommand` runs, and `on-create.sh` starts with `pnpm install`. Dependency and package lifecycle scripts therefore inherit contributor tokens during container creation; run the install with these variables unset or move token-dependent checks after an install step that cannot see them.
  - confidence: 0.88

  - body: The final script no longer installs or uses the retired `github/gh-copilot` extension, but the docs still describe GitHub Copilot as a `gh extension`. Update the wording so contributors are not pointed back at the removed path.
  - confidence: 0.9

  - body: This is a user-facing `feat`/DX addition, but the PR does not touch `CHANGELOG.md`. Add a single-line entry under the active changelog section with the contributor credit required by repo policy.
  - confidence: 0.82
