## Summary

Keep this PR open: current main does not have the devcontainer surface, but the current head still has blocking token-exposure and docs/i18n regressions, plus insufficient latest-head real environment proof for an external contributor PR.

## What This Changes

Adds a VS Code devcontainer with Dockerfile/config/bootstrap script, install docs/nav/labeler updates, zh-CN glossary changes, and file-content tests.

## Best Possible Solution

Keep the devcontainer direction, but install dependencies without forwarded host secrets, preserve the existing glossary while adding the new term, align Copilot wording with the built-in gh command, and require current-head Docker plus Podman proof before merge.

## Reproduction Assessment

Yes for the blocking review findings: source inspection shows containerEnv token forwarding, postCreateCommand running on-create.sh, immediate pnpm install, and current-main install lifecycle scripts. The devcontainer feature itself is not a bug reproduction path because current main lacks that surface.

## Solution Assessment

No. A devcontainer is a reasonable contributor-DX feature, but this implementation should not expose host secrets to install-time scripts or replace unrelated glossary entries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Keep forwarded tokens out of install scripts:** `.devcontainer/devcontainer.json:11-14`
  - body: `containerEnv` forwards `GH_TOKEN` and `ANTHROPIC_API_KEY` to all container processes, and `postCreateCommand` immediately runs `pnpm install` before auth checks. Since current main has install lifecycle scripts, contributors opening the container can expose host tokens to project or dependency install hooks; run install without those variables and only surface secrets to later auth checks.
  - confidence: 0.91
- **[P2] Preserve the existing zh-CN glossary entries:** `docs/.i18n/glossary.zh-CN.json:1-3`
  - body: The PR replaces the current zh-CN glossary with a much smaller file while only needing to add `Dev Container`, dropping existing terms such as `OpenAI` and `Plugin manifest`. Preserve the current glossary and add the new term so translation guidance does not regress.
  - confidence: 0.9
- **[P3] Align Copilot wording with the built-in gh command:** `docs/install/devcontainer.md:14`
  - body: The bootstrap now checks the supported built-in `gh copilot` command, but this docs line still calls GitHub Copilot a `gh extension`. Update the wording so contributors are not pointed back at the retired extension model.
  - confidence: 0.88
