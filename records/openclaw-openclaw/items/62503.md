## Summary

Keep open: current main still has no devcontainer implementation, and this active PR has merge-blocking security/docs findings plus insufficient current-head real behavior proof after the latest force-push.

## What This Changes

The PR adds a VS Code devcontainer setup with Dockerfile/config/bootstrap script, install docs/nav/labeler updates, zh-CN glossary changes, and devcontainer file-content tests.

## Best Possible Solution

Keep the devcontainer direction, but make dependency installation secret-free, restore the zh-CN glossary by adding only the new term, align Copilot wording with `gh copilot`, and require Docker plus Podman proof for the latest head.

## Reproduction Assessment

Yes for the blocking review findings: the PR head shows container-wide credential forwarding, `postCreateCommand`, and immediate `pnpm install`, and current main has install lifecycle scripts. For the feature itself, reproduction is not a bug path because current main lacks the requested devcontainer files.

## Solution Assessment

No. A devcontainer is a reasonable contributor-DX feature, but this implementation should not expose host secrets to install-time scripts or replace unrelated i18n glossary entries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep forwarded tokens out of install scripts:** `.devcontainer/devcontainer.json:11-14`
  - body: `containerEnv` forwards `GH_TOKEN` and `ANTHROPIC_API_KEY` to all container processes, and `postCreateCommand` immediately runs `pnpm install` before auth checks. Because pnpm install scripts are enabled unless `--ignore-scripts` is used, contributors opening the devcontainer can expose host tokens to project or dependency lifecycle scripts; run install with those variables unset or split auth into a later secret-aware step. ([containers.dev](https://containers.dev/implementors/json_reference/))
  - confidence: 0.9
- **[P2] Restore the full zh-CN glossary:** `docs/.i18n/glossary.zh-CN.json:1-3`
  - body: The PR replaces the existing zh-CN glossary with a much smaller file while only needing to add `Dev Container`, dropping existing terms such as `OpenAI` and `Plugin manifest`. Preserve the current glossary and add the new term instead so docs translation prompt guidance does not regress.
  - confidence: 0.88
- **[P3] Fix stale Copilot extension wording:** `docs/install/devcontainer.md:14`
  - body: The bootstrap now checks the supported `gh copilot` command, but this docs line still calls GitHub Copilot a `gh extension`. Update the wording so contributors are not pointed back at the retired extension model. ([cli.github.com](https://cli.github.com/manual/gh_copilot))
  - confidence: 0.9
