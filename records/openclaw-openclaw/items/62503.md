## Summary

Keep open. Current main still lacks devcontainer support, and the PR is useful, but the current head is not ready because host auth tokens are injected container-wide before `pnpm install` runs, with docs and changelog cleanup still pending.

## What This Changes

The PR adds a VS Code devcontainer Dockerfile/config/bootstrap script, devcontainer docs and nav entry, labeler and zh-CN glossary updates, and file-content tests for a Docker/Podman contributor environment.

## Best Possible Solution

Keep the devcontainer direction, but split dependency installation from auth-secret access, update the Copilot wording, add the changelog entry, and then validate one Docker and one Podman create path.

## Reproduction Assessment

Yes for the blocking review finding: the PR head shows `containerEnv` forwarding `GH_TOKEN` and `ANTHROPIC_API_KEY`, and `postCreateCommand` immediately runs a script whose first action is `pnpm install`. For the feature itself, this is not a bug reproduction path; current main simply lacks the requested devcontainer files.

## Solution Assessment

No. Adding a devcontainer is an appropriate DX surface, but this implementation should not expose host secrets to install-time lifecycle scripts; install should run with those variables unset or before auth-only checks receive them.
