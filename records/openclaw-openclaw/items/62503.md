## Summary

## What This Changes

Adds a VS Code devcontainer setup with Dockerfile/config/bootstrap script, install docs/nav/labeler updates, zh-CN glossary changes, and devcontainer file-content tests.

## Best Possible Solution

Keep the devcontainer direction, but run dependency installation without forwarded secrets, preserve existing glossary entries while adding the new term, align Copilot wording with `gh copilot`, and require latest-head Docker plus Podman proof.

## Reproduction Assessment

Yes for the blocking review findings: the PR head shows `containerEnv` token forwarding and `postCreateCommand`, the bootstrap immediately runs `pnpm install`, and current main has install lifecycle scripts. For the feature itself, reproduction is not a bug path because current main lacks the requested devcontainer surface.

## Solution Assessment

No. A devcontainer is a reasonable contributor-DX feature, but this implementation should not expose host secrets to install-time scripts or replace unrelated glossary entries.
