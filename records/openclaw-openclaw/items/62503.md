## What This Changes

## Best Possible Solution

Keep the devcontainer direction, but isolate dependency installation from forwarded secrets, preserve the current glossary while adding the new term, align Copilot wording with the current CLI, resolve conflicts, and attach latest-head Docker plus Podman proof.

## Reproduction Assessment

Yes for the review blockers: source inspection of the PR head shows token forwarding, automatic install, glossary replacement, and stale wording. The feature itself is not a bug reproduction path because current main lacks the requested devcontainer surface.

No. A devcontainer is a reasonable contributor-DX direction, but this implementation should not expose host secrets to install-time scripts or replace unrelated glossary entries.
