## Summary

Keep open. Current main still lacks a devcontainer, and this PR addresses a real contributor-DX gap, but the final diff forwards host tokens into the automatic install/bootstrap path and the branch is currently dirty against main.

## What This Changes

The PR adds a VS Code devcontainer, bootstrap script, devcontainer docs/nav entries, labeler coverage, and file-content tests for a portable Docker/Podman contributor environment.

## Best Possible Solution

Keep the devcontainer direction, but revise the branch so dependency installation runs without forwarded host secrets, update the stale Copilot wording, rebase onto current main, and then validate the devcontainer plus the targeted docs/tests.

## Reproduction Assessment

Yes for the review finding: with host `GH_TOKEN` or `ANTHROPIC_API_KEY` set, `containerEnv` exposes them to `.devcontainer/on-create.sh`, and that script runs `pnpm install` before any auth-only work. For the feature request itself, reproduction is not applicable; current main simply lacks the devcontainer files.

## Solution Assessment

No. Adding a devcontainer is the right surface, but the current implementation should isolate install-time scripts from host secrets and align the docs with the final Copilot CLI behavior before it is the best maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep install scripts away from forwarded tokens:** `.devcontainer/devcontainer.json:11-14`
  - body: `containerEnv` makes `GH_TOKEN` and `ANTHROPIC_API_KEY` available while `postCreateCommand` runs, and `on-create.sh` starts with `pnpm install`. That means dependency lifecycle scripts inherit developer tokens during container creation; run the install with these variables unset or move the auth checks into a later step that does not wrap package installation.
  - confidence: 0.86
- **[P3] Fix stale Copilot extension wording:** `docs/install/devcontainer.md:14`
  - body: The final script deliberately avoids `github/gh-copilot`, but the docs still describe GitHub Copilot as a `gh extension`. Update the wording so users are not pointed back at the retired extension path.
  - confidence: 0.9
