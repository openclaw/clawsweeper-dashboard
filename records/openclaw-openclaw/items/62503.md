## Summary

Keep open: the devcontainer feature is not on current main, but this PR still has blocking security, docs, and i18n regressions plus insufficient current-head real behavior proof.

## What This Changes

Adds a VS Code devcontainer setup with Dockerfile/config/bootstrap script, install docs/nav/labeler updates, zh-CN glossary changes, and devcontainer file-content tests.

## Best Possible Solution

Keep the devcontainer direction, but isolate dependency installation from forwarded secrets, preserve the existing glossary while adding the new term, align Copilot wording with current GitHub CLI behavior, and require latest-head Docker plus Podman proof.

## Reproduction Assessment

Yes for the blocking review findings: the PR head shows container-wide token forwarding, an automatic bootstrap that runs `pnpm install`, current main install lifecycle scripts, the glossary replacement, and the stale Copilot wording. The feature itself is not a bug reproduction path because current main lacks the requested devcontainer surface.

## Solution Assessment

No. A devcontainer is a reasonable contributor-DX feature, but this implementation should not expose host secrets to install-time scripts or replace unrelated glossary entries.

## Review Findings

Overall correctness: patch is incorrect
