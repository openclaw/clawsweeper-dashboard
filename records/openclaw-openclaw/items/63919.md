## Best Possible Solution

Have the gateway/security owner decide whether this HTTP coding-tool surface is acceptable, then merge this guarded implementation if CI and proof gates are green.

Mostly yes. The raw factory, HTTP-only guard, default deny-list expansion, provider-gated `apply_patch` exclusion, and owner-only `exec`/`process` tagging are a maintainable shape; the open question is whether maintainers want this surface at all.
