## Summary

Keep open: this is recent, active feature work and current main has not implemented the broad HTTP coding-tool surface. The PR still needs a security/authz fix, contributor real behavior proof, a rebase onto current main, and maintainer approval for the broader surface versus the narrower exec-only alternative.

## What This Changes

The PR adds a raw coding-tool factory and wires `read` plus opt-in `write`/`edit`/`exec`/`process` coding primitives into Gateway HTTP `/tools/invoke`, with deny-list, docs, changelog, and tests.

## Best Possible Solution

Have Gateway/security owners choose the approved surface, then rebase the chosen approach onto current main with owner/admin gating for exec-capable tools, aligned docs/tests, and live `/tools/invoke` proof.

## Reproduction Assessment

Yes for the source-level gap: current main's Gateway resolver only materializes `createOpenClawTools`, so coding primitives are absent from `/tools/invoke`. This remains feature work rather than a broken existing behavior with a runtime bug repro.

## Solution Assessment

No, not as currently proposed: the broad factory approach exposes exec-capable tools without owner-only HTTP gating and still needs maintainer approval. A safer path is a dedicated owner-only HTTP exec/process registration or a maintainer-approved broad surface with equivalent admin gating.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep HTTP exec-capable coding tools owner-only:** `src/gateway/tool-resolution.ts:149-156`
  - body: This appends the normal agent coding tool set to `/tools/invoke`. After `gateway.tools.allow` removes `exec` or `process` from the HTTP deny list, those returned tools are not marked `ownerOnly`, while the HTTP owner-only filter only protects explicit owner-only tools plus `cron`/`gateway`/`nodes` by name. That lets trusted-proxy callers with `operator.write` but without `operator.admin` reach command execution; register or wrap the HTTP `exec`/`process` entries as owner-only before exposing them.
  - confidence: 0.86
