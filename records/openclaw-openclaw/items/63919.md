## Summary

Keep open. Current main still does not wire coding primitives into Gateway HTTP `/tools/invoke`, but this external PR is a broad, security-sensitive feature expansion that is currently conflicting, lacks contributor real behavior proof, and overlaps with a narrower active exec-only alternative.

## What This Changes

The PR adds an unwrapped coding-tool factory and wires `read` plus opt-in `write`/`edit`/`exec`/`process` coding primitives into Gateway HTTP `/tools/invoke`, with deny-list, docs, changelog, and regression-test updates.

## Best Possible Solution

Have Gateway/security owners choose broad coding-tool exposure versus the exec-only path, then rebase the chosen approach onto current main and merge only with live `/tools/invoke` proof and aligned policy/docs/tests.

## Reproduction Assessment

Yes for the source-level implementation gap: current main's Gateway resolver only materializes `createOpenClawTools`, so coding primitives are absent from `/tools/invoke`. This is feature work rather than a broken existing behavior with a runtime bug repro.

## Solution Assessment
