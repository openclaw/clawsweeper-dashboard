## Summary

The branch adds `gateway.tools.directInvoke.hostFsRead`/`hostFsWrite`, wires built-in `read`/`write`/`edit` coding tools into Gateway `/tools/invoke` and SDK `tools.invoke`, updates audits/docs/tests, and bumps plugin-SDK surface budgets.

## Best Possible Solution

Preserve existing plugin tool availability by scoping the new default deny behavior to built-in coding tools, then rebase and land only if maintainers accept the owner-gated host-FS direct-invoke contract.

## Solution Assessment

No, not yet: the triple-key owner gate is a plausible shape for built-in host tools, but the best fix must preserve same-named plugin tools by default and then get maintainer acceptance for host-FS write over direct invoke.
