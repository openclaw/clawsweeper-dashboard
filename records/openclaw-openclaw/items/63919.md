## Summary

Keep open: current main still lacks the coding-tool wiring, and this PR looks like a valid implementation with no blocking review findings; it still needs maintainer approval because it expands the Gateway HTTP tool execution/data surface.

## What This Changes

This PR adds a raw coding-tool factory, wires core coding tools into Gateway HTTP `/tools/invoke`, extends the default HTTP deny list/docs/changelog, and adds regression tests.

## Best Possible Solution

Land after maintainer approval of the HTTP coding-tool exposure and keep #37131 for any broader direct-invocation surfaces outside this narrow Gateway HTTP path.

## Reproduction Assessment

Yes. On current main, a `/tools/invoke` request for an allowlisted coding primitive such as `read` cannot resolve because the Gateway resolver only creates Gateway/OpenClaw tools; the PR adds focused coverage for the intended success and deny-list paths.

## Solution Assessment

Yes for the code shape. The branch reuses the existing resolver, policy, owner, hook, and deny-list pipeline while keeping MCP loopback unchanged and avoiding double hook execution through the raw factory.
