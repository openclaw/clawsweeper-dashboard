## Summary

Keep open. Current main still does not inherit Codex trusted project config into the ACPX isolated Codex home, and this PR is directionally useful, but the proposed parser misses supported Codex TOML project shapes and the user-facing ACPX fix needs a changelog entry before merge.

## What This Changes

The PR copies Codex trusted project declarations and the ACPX workspace cwd into the isolated ACPX Codex home while adding bridge tests that source auth and notify hooks are not copied.

## Best Possible Solution

Keep the fix in the ACPX plugin, inherit only trusted Codex project declarations plus the ACPX workspace cwd into the isolated Codex home, parse Codex project trust with TOML-aware handling for explicit tables and inline maps, and keep auth, notify hooks, and unrelated Codex config excluded.

## Reproduction Assessment

Yes. Current main can be verified by inspection: prepareIsolatedCodexHome writes only a generated config header, so a trusted project in CODEX_HOME/config.toml is not propagated into the isolated ACPX Codex home.

## Solution Assessment

No, not as written. The owner-local fix direction is maintainable, but the line-based parser is incomplete for Codex-supported TOML shapes; a TOML-aware extractor or equivalent focused parser with inline-project coverage is the safer solution.
