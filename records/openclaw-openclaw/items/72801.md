## Summary

Keep open. The current-main bug is still source-reproducible, but this draft PR is not merge-ready because its parser misses Codex-supported TOML project shapes and the required changelog entry is absent.

## What This Changes

The PR branch updates ACPX's Codex auth bridge to inherit trusted project declarations into its isolated CODEX_HOME config and adds tests for inherited trust while keeping auth/notify config excluded.

## Best Possible Solution

Keep the fix owner-local in extensions/acpx, use TOML-aware trust extraction for all Codex-supported project shapes, render only trusted project declarations plus the active workspace cwd, preserve auth/notify exclusion, and add the required changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is to put a trusted project entry in CODEX_HOME/config.toml and call prepareAcpxCodexAuthConfig; current main writes an isolated config containing only the generated header, so the trust entry is lost.

## Solution Assessment

No, not as written. The direction is right, but the parser only handles one TOML table shape; the maintainable fix should follow Codex-supported project trust forms and add the missing changelog entry.
