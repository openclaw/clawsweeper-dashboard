## Summary

Keep open. The PR targets a real ACPX Codex isolation bug that still exists on current main, but the proposed parser misses supported Codex TOML project-trust shapes and the user-facing fix still needs a changelog entry.

## What This Changes

The PR changes the ACPX Codex auth bridge to copy trusted Codex project declarations and the workspace cwd into the isolated Codex home while keeping auth and notify config excluded.

## Best Possible Solution

Keep the fix owner-local in `extensions/acpx`, use a TOML-aware Codex trust extractor that handles all supported project shapes, copy only trusted project declarations plus the active workspace cwd, and add the required changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is to put a trusted project entry in `CODEX_HOME/config.toml` and call `prepareAcpxCodexAuthConfig`; current main writes an isolated config containing only the generated header, so the trust entry is lost.

## Solution Assessment

No, not as written. The direction is right, but a line-based parser that only recognizes one table form is not the narrowest maintainable fix for Codex-supported TOML config.
