## Best Possible Solution

## Reproduction Assessment

Yes. Current `main` keeps C0/C1/DEL characters because both relevant normalizers strip line separators and then accept every code point <= 0xff; a focused Node Headers probe also shows NUL breaks Authorization header construction.

## Solution Assessment

No, not quite. Changing the core helper is the right layer for provider/setup/auth-profile paths, but the best fix should keep the web-content runtime's duplicate normalizer and tests in sync before merge.
