## Summary

Keep open: the PR is a useful direction for the linked long-context max-token regression, but it appears incomplete because one current outbound max-token source can still bypass the new clamp, and the external PR has no after-fix real behavior proof yet.

## What This Changes

The PR adds an embedded-runner stream wrapper that estimates used input tokens and clamps configured, runtime, and model max token values against the remaining context window, with focused unit coverage.

## Best Possible Solution

Clamp the final outbound max-token budget across every source the request builder can choose, including `model.params` aliases, while preserving smaller explicit runtime caps and adding regression coverage for the option, model, and model.params paths.

## Reproduction Assessment

Yes. Source inspection shows current main can forward a full OpenAI-compatible model/config output cap without subtracting input tokens, and the linked issue provides a concrete StepFun failure pattern.

## Solution Assessment

No, not yet. The stream-wrapper approach is plausible, but the proposed fix does not cover `model.params` max-token aliases that the current request builder prefers over `model.maxTokens`.
