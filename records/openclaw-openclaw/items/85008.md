## Summary

Keep open: the fix is aimed at a real source-reproducible max-token regression, but this PR still misses a supported configured-token path and has no after-fix real provider proof.

## What This Changes

The PR adds an agent stream wrapper/helper and unit coverage to clamp runtime/configured/model `maxTokens` against the estimated remaining context window before provider calls.

## Best Possible Solution

Clamp the final effective generation-token cap in the provider request path, or normalize every cap source before transport, so runtime options, model params, and model maxTokens share one remaining-context calculation with focused regression tests and redacted provider proof.

## Reproduction Assessment

Yes. Source inspection shows current main can forward an OpenAI-compatible output cap directly to `max_tokens` / `max_completion_tokens` without subtracting input tokens, matching the linked StepFun failure.

## Solution Assessment

No. Dynamic per-call clamping is the right direction, but this implementation should cover the transport's final effective cap source, including `model.params`, before it is the best fix.
