Keep open. Current main still lacks Telegram-specific persisted-state diagnostics or a safe reset/quarantine path in channel status, and the related generic session-store recovery PR does not cover the Telegram artifact/status UX requested here.

The issue requests detection and recovery guidance for oversized, malformed, or poisoned persisted Telegram session/state JSON that can wedge the Telegram channel.

## Best Possible Solution

Add a non-destructive Telegram state diagnostic that reports relevant artifact paths, size, parse/schema health, and backup/reset guidance; reserve automatic quarantine or reset for an explicit maintainer policy decision.

## Solution Assessment
