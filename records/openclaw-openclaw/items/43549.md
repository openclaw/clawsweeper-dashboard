## Summary

Keep open. Current main has Telegram polling status warnings and hardened offset parsing, but it still does not inspect the Telegram persisted state artifact for size, parse/schema health, or recovery guidance in status; the related generic session-store recovery pull request does not close this Telegram status UX gap. The linked Keep My Claw skill is useful backup mitigation, not the requested detection/recovery path.

## What This Changes

Add Telegram persisted-state diagnostics and a safe recovery path so bad local session JSON is surfaced by status instead of wedging the channel silently.

## Best Possible Solution

Add a non-destructive Telegram state diagnostic that reports artifact path, size, parse/schema health, and explicit backup/reset guidance; handle automatic quarantine or reset only after an explicit maintainer policy decision.

## Reproduction Assessment

No high-confidence reproduction of the original wedge exists because the bad file path and contents were not supplied. Source inspection does confirm current main lacks Telegram persisted-state checks in status and still treats invalid update-offset JSON as a fallback-to-null path.

## Solution Assessment

Partly. The diagnostic and recovery direction is reasonable, but automatic self-heal or reset is not the safest first step without maintainer policy; a non-destructive status or doctor diagnostic with backup guidance is the narrowest maintainable starting point.
