## Summary

Keep open. Current main and v2026.6.8 still collapse malformed session-store JSON into an empty store without surfacing Telegram-facing session-state health, and the open related PR only covers generic stale temp-artifact recovery, not status guidance or a safe reset/quarantine UX.

## What This Changes

Request status, doctor, and gateway handling so malformed or oversized Telegram session JSON is diagnosed and safely recoverable.

## Best Possible Solution

Add non-destructive session-store health diagnostics in status and doctor, land safe generic temp recovery if accepted, and separately design Telegram reset/quarantine UX with backups.

## Reproduction Assessment

Source inspection gives a high-confidence malformed-store path to an empty returned store; the live Telegram wedge and oversized-file threshold were not reproduced.

## Solution Assessment

Not ready for automated repair because diagnostics are clear enough, but reset/quarantine/self-heal behavior needs a maintainer decision on data preservation and operator UX.
