## Summary

Keep open. Current main still lacks Telegram-specific persisted-state diagnostics or a safe reset/quarantine path in channel status; the linked Keep My Claw skill is useful backup context, but not a fix for the requested status/recovery UX, and the open generic sessions-store recovery PR does not cover this Telegram path.

## What This Changes

The issue asks channel status to detect malformed or oversized Telegram persisted JSON and guide safe recovery instead of leaving the channel wedged.

## Best Possible Solution

Add a non-destructive Telegram state diagnostic that reports relevant artifact paths, size, parse/schema health, and backup/reset guidance; reserve automatic quarantine/reset behavior for an explicit maintainer product decision.

## Reproduction Assessment

No high-confidence reproduction exists for the original wedge because the bad file path and contents were not provided. Source inspection does confirm that current main lacks the requested persisted-state health checks in Telegram status.

## Solution Assessment

Yes, the diagnostic/recovery direction is reasonable, but automatic self-heal is not the safest first step without a policy decision. A non-destructive status or doctor diagnostic with explicit backup/reset guidance is the narrower maintainable path.
