## Summary

Keep open. Current main still lacks Telegram-specific persisted-state diagnostics or a safe reset/quarantine path in channel status, and the open generic sessions-store recovery PR does not cover this Telegram status/recovery UX.

## What This Changes

Request `openclaw channels status`/Telegram troubleshooting to detect malformed, oversized, or poisoned Telegram persisted session JSON and guide safe recovery.

## Best Possible Solution

Add a non-destructive Telegram state diagnostic that reports relevant artifact paths, size, parse/schema health, and backup/reset guidance, then let maintainers decide whether any automatic quarantine behavior is safe.

## Reproduction Assessment

No high-confidence reproduction exists for the original wedge because the corrupted file path and contents were not provided. Source inspection does confirm that current main lacks the requested persisted-state health checks in Telegram status.

## Solution Assessment

Yes, the diagnostic/recovery direction is reasonable, but automatic self-heal is not yet the safest first step. Maintainers should choose the artifact boundary and prefer explicit backup/reset guidance before destructive recovery.
