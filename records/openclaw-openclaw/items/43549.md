## Summary

Keep open. Current main has Telegram liveness warnings and persisted-offset rotation handling, but it still does not diagnose malformed or oversized Telegram-owned state in status output or provide a safe reset path; the Keep My Claw ClawHub skill is useful backup context, and https://github.com/openclaw/openclaw/pull/79910 addresses a related generic session-store failure, not this Telegram status/recovery UX.

## What This Changes

Detect malformed or oversized Telegram persisted session/state JSON in status tooling and provide a safe backup/reset or quarantine recovery path.

## Best Possible Solution

Add a non-destructive Telegram/channel-state diagnostic that reports the exact artifact path, size, parse/schema health, and recommended backup/reset action before considering automatic quarantine.

## Reproduction Assessment

No high-confidence reproduction exists for the original wedge because the corrupted file path and contents were not provided. Source inspection does confirm the current status path lacks the requested persisted-state health checks.

## Solution Assessment

Yes, the diagnostic/recovery direction is reasonable, but automatic self-heal is not yet the safest first step. Maintainers should choose the artifact boundary and prefer explicit backup/reset guidance before destructive recovery.
