## Summary

Keep open. Current main and the latest release still collapse malformed session-store JSON into an empty store for status/health/doctor paths, and the open recovery PR only covers generic stale temp-artifact recovery rather than the Telegram-facing diagnosis and safe reset UX requested here.

## What This Changes

The issue requests Telegram session-state diagnostics and a safe recovery path when persisted Telegram session JSON is oversized, malformed, or otherwise wedges the channel.

## Best Possible Solution

Add non-destructive session-store health diagnostics first, keep generic temp-artifact recovery in the linked store PR, and make Telegram reset/quarantine behavior a maintainer-approved recovery UX.

## Reproduction Assessment

Current source gives a medium-confidence reproduction path for malformed `sessions.json`: `loadSessionStore` catches parse/read failure and status/doctor callers then see an empty store. The live Telegram wedge and oversized-artifact threshold are still not reproduced.

## Solution Assessment

No, not as an automated narrow repair yet. Non-destructive diagnostics are clearly useful, but reset/quarantine behavior needs maintainer decisions about backup, continuity loss, and whether recovery belongs in core doctor/status or Telegram-owned command UX.
