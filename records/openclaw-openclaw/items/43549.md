## Summary

Keep this open: current main still lacks the requested bad-session-state diagnosis. The existing Keep My Claw ClawHub plugin mentioned in the discussion is a useful backup safety net, but it does not replace first-party status/doctor guidance for a poisoned Telegram session store.

## What This Changes

The issue requests Telegram status/recovery diagnostics for oversized, malformed, or otherwise poisoned persisted session JSON.

## Best Possible Solution

Add a read-only session-store health diagnostic that reports path, size, parse/schema status, and a backup-then-reset recovery hint, while leaving destructive reset or auto-quarantine behavior for a separate maintainer decision.

## Reproduction Assessment

No high-confidence live reproduction path is available from the issue. Source inspection does show the diagnostic gap: malformed session-store JSON is swallowed into an empty store and Telegram status collection has no session-state warning path.

## Solution Assessment

Partly. The best first fix is narrower than the full suggested recovery UX: surface non-destructive status/doctor diagnostics and backup guidance first, without auto-deleting or quarantining session files.
