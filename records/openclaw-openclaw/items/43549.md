## Summary

Keep open: current main and the latest shipped release still lack Telegram persisted-state artifact diagnostics or reset guidance in channel status. The cross-referenced generic session-store recovery PR at https://github.com/openclaw/openclaw/pull/79910 is useful adjacent work, and the Keep My Claw backup skill linked in discussion is a mitigation, but neither closes the Telegram status/recovery UX gap.

## What This Changes

Add Telegram status diagnostics for malformed, oversized, or poisoned persisted session JSON and guide operators through safe recovery.

## Best Possible Solution

Add a non-destructive Telegram state diagnostic that reports artifact path, size, parse/schema health, and explicit backup/reset guidance; gate automatic quarantine or reset on a maintainer policy decision.

## Reproduction Assessment

No high-confidence reproduction of the original wedge exists because the bad file path, size, and contents were not supplied. Source inspection does show current status lacks persisted-state diagnostics and invalid update-offset JSON falls back without operator guidance.

## Solution Assessment

Partly. The diagnostic direction is maintainable, but the safest first step is non-destructive status or doctor reporting with backup guidance; automatic self-heal or reset needs a maintainer policy decision first.
