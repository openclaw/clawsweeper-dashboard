## Summary

Keep open. Current main and v2026.6.10 still collapse malformed or empty session-store JSON into an empty store without a Telegram-facing session-state diagnosis, and the open related recovery PR only covers generic stale temp-artifact recovery, not status guidance or safe reset/quarantine UX.

## What This Changes

Request Telegram/status troubleshooting to detect oversized or malformed persisted session JSON and guide safe reset, backup, or quarantine recovery.

## Best Possible Solution

Add non-destructive session-store health diagnostics in status and doctor, land safe generic store recovery if accepted, and make any reset/quarantine path explicit, backed up, and tested for Telegram-visible recovery.

## Reproduction Assessment

Partly. Source inspection gives a clear malformed or empty sessions.json path that returns an empty store, but the exact live Telegram wedge and oversized-file threshold have not been reproduced against current main.

## Solution Assessment

Unclear as a full fix. Non-destructive diagnostics are the right first step, but reset/quarantine/self-heal semantics need a maintainer decision because they can trade recovery speed against session continuity.
