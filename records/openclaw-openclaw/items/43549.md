## Summary

Keep open: current main and the latest release still expose Telegram runtime/config/audit status, but not persisted Telegram JSON artifact health or safe reset guidance. ## What This Changes

The issue asks for Telegram status or doctor tooling to detect malformed, oversized, or poisoned persisted Telegram JSON state and guide safe recovery.

## Best Possible Solution

Add a non-destructive Telegram state diagnostic that reports artifact path, size, parse/schema health, and explicit backup/reset guidance; define reset or quarantine policy before any automatic mutation.

## Reproduction Assessment

No high-confidence reproduction of the original wedge exists because the bad file path, size, and contents were not supplied. Source inspection does show current status lacks persisted-state diagnostics and invalid update-offset JSON falls back without operator guidance.

## Solution Assessment

Partly. A non-destructive status or doctor diagnostic with backup guidance is the safest first step; automatic self-heal, reset, or quarantine needs a maintainer policy decision first.
