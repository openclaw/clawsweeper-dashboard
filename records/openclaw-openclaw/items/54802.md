## Summary

Keep open: current main already fixed the active conversation-label path, but the TTS summarization path still uses the old Codex Responses request shape, and this external PR lacks required after-fix real behavior proof.

## What This Changes

The PR moves simple-completion instructions into `systemPrompt` for auto-topic labels and TTS summaries, omits `temperature` for Codex Responses, handles error stop reasons, and adds regression tests.

## Best Possible Solution

Keep the merged conversation-label implementation on main, apply the same Codex request-shape and error handling to `summarizeText`, cover it in the current TTS contract suite, and require redacted real behavior proof before merge.

## Reproduction Assessment

Yes. Current `summarizeText` still sends its instructions as user-message text and always passes `temperature: 0.3`, while the Codex Responses transport derives top-level `instructions` from `systemPrompt` and forwards any defined temperature.

## Solution Assessment

No, not as this PR currently stands. The TTS request-shape fix is the right narrow direction, but the label half is obsolete after https://github.com/openclaw/openclaw/pull/78450 and the branch still lacks required real behavior proof.
