## Summary

Keep open: the conversation-label half was superseded by a later merged fix, but current main still leaves the TTS summarization simple-completion request without `systemPrompt`, so the remaining bug fix is still useful. ## What This Changes

The PR changes old auto-topic-label and TTS summarization simple-completion call sites to pass instructions through `systemPrompt`, omit `temperature` for Codex Responses, handle completion errors, and add regression tests.

## Best Possible Solution

Keep the merged conversation-label implementation, narrow this PR or a replacement to the current TTS summarization request shape and contract suite, use a distinct provider-error fallback, and require redacted real Codex Responses TTS proof.

## Reproduction Assessment

Yes, source-reproducible: current `summarizeText` embeds the summary instructions in the user message while the Codex Responses transport derives request instructions from `context.systemPrompt`. I did not run a live Codex TTS call.

## Solution Assessment

No, not as this PR currently stands. The TTS `systemPrompt` direction is the narrow fix, but the branch should drop the superseded label-helper changes, move coverage to the current contract suite, and use a distinct provider-error fallback.
