## Summary

Keep this PR open because the remaining TTS Codex Responses bug is still present on current main, but the branch is not mergeable as-is: the Telegram label half has been superseded, real behavior proof is missing, and the TTS error fallback still needs a small diagnostic fix.

## What This Changes

The branch moves simple-completion instructions to `systemPrompt`, omits `temperature` for Codex Responses, handles completion errors, and adds tests for the old auto-topic-label helper plus TTS summarization.

## Best Possible Solution

Carry forward only the current TTS request-shape fix into `src/tts/tts-core.ts` and the TTS contract suite, keep the merged conversation-label implementation from https://github.com/openclaw/openclaw/pull/78450, use a distinct provider-error fallback, and require redacted real Codex Responses TTS proof.

## Reproduction Assessment

Yes, source-reproducible: current `summarizeText` embeds instructions in user content and passes `temperature`, while the Codex Responses transport derives `instructions` from `context.systemPrompt`. I did not run a live Codex TTS call.

## Solution Assessment

No, not as this PR currently stands. The TTS `systemPrompt` direction is the narrow fix, but the branch should drop the superseded label-helper changes, move coverage to the current contract suite, and use a distinct provider-error fallback.
