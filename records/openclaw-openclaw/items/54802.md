## Best Possible Solution

## Solution Assessment

No, not as this PR currently stands. The TTS `systemPrompt` direction is the narrow maintainable fix, but the branch should drop the superseded label-helper changes, move coverage to the current contract suite, and use a distinct provider-error fallback.
