## Summary

This PR should stay open: it targets a real, source-reproducible session path bug, but the current patch should not merge because the new remap ignores an active custom sessionsDir/session.store and redirects transcripts into the default state tree.

## What This Changes

The branch adds same-agent cross-root sessionFile remapping in src/config/sessions/paths.ts and a focused regression test in src/config/sessions.test.ts.

## Best Possible Solution

Remap validated same-agent stale filenames under the active resolved sessionsDir/realBase, preserve cross-agent fallback behavior, and add custom-store regression coverage.

## Reproduction Assessment

Yes. The linked issue and PR body provide concrete stale session metadata and CLI ENOENT logs, and current-main source shows the resolver can return the old absolute path before acquireSessionWriteLock mkdirs that directory.

## Solution Assessment

No. The fix belongs in session path resolution, but this implementation should target the active sessionsDir/realBase instead of recomputing the default state-derived agent sessions directory.
