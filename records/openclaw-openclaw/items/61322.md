## Summary

Keep open: the bug fix is still useful and current main does not implement the same-agent stale-root remap, but this branch has a P1 compatibility defect because the new remap can bypass the active custom session store and write into the default state tree.

## What This Changes

## Best Possible Solution

Keep the narrow same-agent/direct-file guard, but remap the validated filename onto the active resolved `sessionsDir`, add custom `session.store` coverage, and require redacted real behavior proof before merge.

## Reproduction Assessment

Yes. Source inspection shows current main preserves structurally valid same-agent stale absolute paths, and the PR's added remap can be traced through custom-store callers that pass `sessionsDir` from `storePath`.

## Solution Assessment

No. The same-agent remap is the right narrow direction, but the implementation should target the active `sessionsDir` contract rather than recomputing the default agent sessions directory.
