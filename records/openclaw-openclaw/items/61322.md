## Summary

Keep this PR open: it fixes a real session-path bug, but the current patch can redirect repaired same-agent transcripts outside a configured custom session store.

## What This Changes

The PR adds a same-agent cross-root `sessionFile` remap in session path resolution and a focused regression test.

## Best Possible Solution

Remap validated same-agent stale filenames under the active resolved `sessionsDir`/`realBase`, preserve cross-agent fallback behavior, and add custom-store regression coverage.

## Reproduction Assessment

Yes. The linked issue gives concrete stale metadata and CLI `ENOENT` logs, and current-main source shows the resolver can still return the old absolute path before later session writes try to create that directory.

## Solution Assessment

No. The fix direction is right, but the helper should target the active sessions directory instead of recomputing the default state-derived agent sessions directory.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

That splits transcript writes outside the user's selected session store.
