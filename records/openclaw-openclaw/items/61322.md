## Summary

Keep open: the PR targets a valid session-path bug, but the current patch would still misplace transcripts for custom `session.store` users by remapping into the default state tree instead of the active sessions directory.

## What This Changes

The PR adds same-agent stale absolute `sessionFile` remapping in session path resolution plus a focused regression test.

## Best Possible Solution

Retarget the same-agent stale-root remap to the active resolved sessions directory, preserve existing cross-agent fallback behavior, and add default-store plus custom-store regression coverage.

## Reproduction Assessment

Yes. The paired issue gives concrete stale metadata and CLI steps, and current-main source shows the resolver can still return an old absolute structural path before the write lock creates that path's directory.

## Solution Assessment

No as written. Session path resolution is the right boundary, but the helper must rebuild under the active `sessionsDir`/`realBase`, not `resolveAgentSessionsDir(...)` from the default state root.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92
