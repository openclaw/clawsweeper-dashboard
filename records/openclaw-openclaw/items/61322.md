## Summary

Keep this PR open, but it still needs changes before merge. The fix direction is right for the stale same-agent session path bug, but the current patch remaps through the default state-derived agent sessions directory and can bypass a configured custom session store.

## What This Changes

The PR adds same-agent cross-root absolute `sessionFile` remapping in `src/config/sessions/paths.ts` and a focused regression test in `src/config/sessions.test.ts`.

## Best Possible Solution

Remap validated same-agent stale filenames beneath the active resolved `sessionsDir`/`realBase`, preserve cross-agent fallback behavior, and add custom `session.store` regression coverage.

## Reproduction Assessment

Yes. The linked issue provides concrete stale `sessions.json` metadata and ENOENT mkdir evidence, and current-main source shows the structural fallback can return the old absolute path before persistence and writes.

## Solution Assessment

No as-is. Session path resolution is the right fix location, but the remap must target the active `sessionsDir`/`realBase` instead of recomputing the default state-derived agent sessions directory.
