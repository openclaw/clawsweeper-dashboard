## Summary

Keep this PR open, but it still needs changes before merge. The bug and fix location are valid, yet the current patch can redirect custom `session.store` users back into the default state tree instead of the active sessions directory.

## What This Changes

The PR adds same-agent stale absolute `sessionFile` remapping in `src/config/sessions/paths.ts` and a focused regression test in `src/config/sessions.test.ts`.

## Best Possible Solution

Retarget the same-agent remap to the active resolved sessions directory, preserve cross-agent structural fallback behavior, and add default plus custom `session.store` regression coverage before landing.

## Reproduction Assessment

Yes, by source inspection. The paired issue gives concrete stale metadata and CLI steps, and current-main path resolution can preserve canonical old-root absolute paths that the write lock then tries to create.

## Solution Assessment

No as-is. Session path resolution is the right fix location, but the helper must rebuild under the active `sessionsDir` or `realBase` instead of recomputing the default state-derived agent sessions directory.

## Review Findings
