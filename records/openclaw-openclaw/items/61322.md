## Summary

Keep open: this is a useful session-state bug fix, but the current patch rebuilds same-agent stale paths under the default state directory and can violate custom `session.store` roots.

## What This Changes

The PR adds resolver logic and a regression test to remap same-agent stale absolute `sessionFile` paths from an old state root into the current agent sessions directory.

## Best Possible Solution

Retarget the same-agent remap to the active resolved `sessionsDir` or `realBase`, preserve cross-agent fallback behavior, and cover both default-store and custom `session.store` cases.

## Reproduction Assessment

Yes, at source level. The linked bug gives concrete stale metadata and CLI steps, and current main can still return an old-root structural path before the write lock creates that path's parent directory.

## Solution Assessment

No as written. Session path resolution is the right boundary, but the helper must rebuild under the active `sessionsDir` or `realBase`, not the default state-derived agent directory.
