## Summary

Keep this PR open: the underlying stale same-agent session path bug is valid and this is the focused fix path, but the current patch is not merge-ready because it redirects custom `session.store` users into the default state tree and the external proof gate is still insufficient.

## What This Changes

The PR adds a same-agent cross-root absolute `sessionFile` remap in `src/config/sessions/paths.ts` and a regression test in `src/config/sessions.test.ts`.

## Best Possible Solution

Keep the narrow resolver fix, but remap validated same-agent stale filenames into the active `sessionsDir`/`realBase`, add custom-store regression coverage, and provide redacted after-fix CLI proof before merge.

## Reproduction Assessment

Yes. ## Solution Assessment
