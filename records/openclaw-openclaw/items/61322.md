## Summary

Keep open: the PR targets a real session-path bug, but the current patch has a blocking regression for custom `session.store` users because the new remap rebuilds paths under the default state-derived sessions tree instead of the active `sessionsDir`.

## What This Changes

The branch adds same-agent stale absolute `sessionFile` remapping in session path resolution plus a focused regression test for old-root metadata.

## Best Possible Solution

Retarget the same-agent stale-path remap to the active resolved `sessionsDir` or `realBase`, preserve cross-agent fallback behavior, and add custom `session.store` regression coverage.

## Reproduction Assessment

Yes. The paired issue gives concrete stale metadata and CLI steps, and current-main source inspection shows the resolver can preserve the old-root absolute path before the write lock creates that path's directory.

## Solution Assessment

No as-is. Session path resolution is the right fix location, but the new helper must rebuild under the active `sessionsDir` instead of calling `resolveAgentSessionsDir()`.
