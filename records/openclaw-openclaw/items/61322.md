## Summary

Keep open. The bug is valid and the PR is a useful implementation candidate, but the patch currently remaps stale same-agent paths through the default state-derived agent directory instead of the active `sessionsDir`, which can split transcript writes for custom `session.store` users.

## What This Changes

The PR adds same-agent stale absolute `sessionFile` remapping in session path resolution plus a focused regression test for state-root moves.

## Best Possible Solution

Remap only validated same-agent stale filenames under the active resolved `sessionsDir`/`realBase`, while preserving cross-agent structural fallback behavior and adding custom-store regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main still lets canonical old-root same-agent absolute `sessionFile` paths fall through to a structural fallback that returns the original path before write-locking creates that directory.

## Solution Assessment

No. The fix direction is right, but this implementation should target the active sessions directory rather than recomputing `resolveAgentSessionsDir` from the default state root.
