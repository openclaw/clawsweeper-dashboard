## Summary

Keep this PR open: it targets a real source-reproducible session path bug, but the current patch can redirect custom `session.store` transcript writes into the default state tree.

## What This Changes

Adds same-agent old-root `sessionFile` remapping in `src/config/sessions/paths.ts` plus a regression test in `src/config/sessions.test.ts`.

## Best Possible Solution

Retarget the same-agent stale-root remap to the active resolved sessions directory, preserve cross-agent fallback behavior, and add default-store plus custom-store regression coverage.

## Reproduction Assessment

Yes. The paired issue gives concrete stale metadata and CLI steps, and current-main source shows the resolver can still return an old absolute structural path before the write lock creates that path's directory.

## Solution Assessment

No as written. Session path resolution is the right boundary, but the helper must rebuild under the active `sessionsDir`/`realBase`, not `resolveAgentSessionsDir(...)` from the default state root.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Remap stale paths under the active sessionsDir:** `src/config/sessions/paths.ts:188`
