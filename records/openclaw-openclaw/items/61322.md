## Summary

Keep this PR open. The bug and fix direction are valid, but the proposed remap currently bypasses the active custom `session.store` directory and can split transcript writes into the default state tree.

## What This Changes

The branch adds same-agent cross-root `sessionFile` remapping in session path resolution plus a focused regression test.

## Best Possible Solution

Remap only validated same-agent stale filenames under the active resolved `sessionsDir`/`realBase`, preserve cross-agent fallback behavior, and add custom-store regression coverage.

## Reproduction Assessment

Yes. The linked issue gives concrete stale metadata and CLI `ENOENT` logs, and current-main source shows the resolver can return the old absolute path before the write lock creates that directory.

## Solution Assessment

No. The fix direction is right, but the current helper should target the active sessions directory instead of recomputing the default state-derived agent sessions directory.
