## Summary

Keep this PR open, but do not merge it as-is. It targets a real session-path bug, yet the proposed remap rebuilds stale same-agent paths under the default state root instead of the active `sessionsDir`, which can break users with custom `session.store` roots.

## What This Changes

The PR adds same-agent stale absolute `sessionFile` remapping in session path resolution and a focused regression test for state-root moves.

## Best Possible Solution

Retarget the same-agent stale-path remap to the active resolved `sessionsDir`/`realBase`, preserve the existing cross-agent fallback behavior, and add custom `session.store` regression coverage.

## Reproduction Assessment

Yes. The paired issue gives concrete stale metadata and ENOENT output, and current-main source shows the resolver can return the old absolute path before the write path creates that directory.

## Solution Assessment

No as-is. Session path resolution is the right fix location, but the new helper must rebuild under the active `sessionsDir`/`realBase` rather than `resolveAgentSessionsDir()`.
