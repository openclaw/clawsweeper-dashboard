## Summary

Keep open. The PR addresses a valid session-path bug, but the current patch would remap stale same-agent paths into the default state-derived sessions directory instead of the active custom `session.store` directory.

## What This Changes

The PR adds a same-agent old-root `sessionFile` remap in `src/config/sessions/paths.ts` and a focused regression test in `src/config/sessions.test.ts`.

## Best Possible Solution

Retarget the same-agent remap to the active resolved sessions directory, preserve cross-agent fallback behavior, and add default plus custom `session.store` regression coverage.

## Reproduction Assessment

Yes, source-reproducible. The paired issue gives concrete stale metadata and CLI steps, and current-main source inspection shows the resolver can still return the old absolute path before write locking creates that path's directory.

## Solution Assessment

No as written. Session path resolution is the right fix boundary, but the remap must rebuild under the active `sessionsDir` or `realBase`, not `resolveAgentSessionsDir`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Remap stale paths under the active sessionsDir:** `src/config/sessions/paths.ts:188`
