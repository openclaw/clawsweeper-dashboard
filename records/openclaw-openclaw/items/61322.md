## Summary

Keep this PR open, but it still needs changes before merge. The reported stale same-agent session path bug is real and the fix direction is right, but the branch currently remaps custom `session.store` users back into the default state-derived sessions directory instead of the active store directory.

## What This Changes

The PR adds same-agent cross-root absolute `sessionFile` remapping in the session path resolver and a focused regression test.

## Best Possible Solution

Remap validated same-agent stale filenames under the active resolved `sessionsDir`/`realBase`, preserve cross-agent fallback behavior, and add custom-store regression coverage.

## Reproduction Assessment

Yes. The linked issue and PR body provide concrete stale metadata plus CLI ENOENT evidence, and current-main source shows the resolver can preserve the old absolute path before the write lock creates that directory.

## Solution Assessment

No. Session path resolution is the right fix location, but this implementation should target the active `sessionsDir`/`realBase` instead of recomputing the default state-derived agent sessions directory.
