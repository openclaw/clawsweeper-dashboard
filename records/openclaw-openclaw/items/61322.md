## Summary

Keep open: the PR targets a real session-state bug with useful proof, but the current patch has a blocking custom `session.store` path defect that can split recovered transcript files from the active store.

## What This Changes

The PR adds same-agent cross-root `sessionFile` remapping in session path resolution and a regression test for stale absolute transcript paths.

## Best Possible Solution

Retarget same-agent stale-root recovery to the active resolved `sessionsDir` or `realBase`, preserve cross-agent fallback behavior, and cover both default-store and custom `session.store` cases.

## Reproduction Assessment

Yes, at source level. The linked bug gives concrete stale metadata and CLI steps, and current `main` can still return an old-root structural path before the write lock creates that path's parent directory.

## Solution Assessment

No as written. Session path resolution is the right boundary, but the helper must rebuild under the active `sessionsDir` or `realBase`, not the default state-derived agent directory.
