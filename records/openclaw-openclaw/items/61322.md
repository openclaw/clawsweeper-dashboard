## Summary

Keep this PR open, but it still needs changes before merge. The bug is real on current main, and the PR targets the right resolver, but the current diff rebuilds remapped paths under the default state directory instead of the active custom session-store directory.

## What This Changes

The PR adds same-agent cross-root `sessionFile` remapping in `src/config/sessions/paths.ts` plus a focused regression test in `src/config/sessions.test.ts`.

## Best Possible Solution

Rebuild validated same-agent stale filenames under the active resolved `sessionsDir`/`realBase`, while preserving cross-agent fallback behavior and adding custom-store regression coverage.

## Reproduction Assessment

Yes. The paired issue gives concrete persisted metadata and CLI steps, and current-main source shows the resolver can return the old absolute path before the write lock creates that path's directory.

## Solution Assessment

No as-is. Session path resolution is the right fix location, but the proposed remap should target the active `sessionsDir` rather than recomputing the default state-derived sessions directory.

## Review Findings
