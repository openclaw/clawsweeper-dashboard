## Summary

Keep this PR open because it targets a real session-state bug, but the current patch is not merge-ready: its same-agent remap rebuilds through the default state root instead of the active `sessionsDir`, which can redirect users with custom `session.store` layouts, and the external PR still lacks inspectable after-fix real behavior proof.

## What This Changes

The PR adds a same-agent cross-root absolute `sessionFile` remap in `src/config/sessions/paths.ts` and a focused regression test in `src/config/sessions.test.ts`.

## Best Possible Solution

Keep the resolver-level repair, but remap stale same-agent paths relative to the active `sessionsDir`/`realBase`, preserve cross-agent fallback behavior, add custom `session.store` regression coverage, and require redacted after-fix proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `resolveSessionFilePath` with an explicit `agentId`, a non-default `sessionsDir` from `storePath`, and a stale same-agent absolute `sessionFile`; the PR remap can produce the default state-root path instead of the active store path.

## Solution Assessment

No. The resolver is the right layer, but the implementation should remap against the active `sessionsDir` or `realBase` instead of unconditionally rebuilding through `resolveAgentSessionsDir`.
