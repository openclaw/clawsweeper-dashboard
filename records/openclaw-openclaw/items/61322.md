## Summary

Keep this PR open. The reported bug is real and the PR heads in the right direction, but the current diff can redirect users with a custom `session.store` into the default state tree instead of the active store root.

## What This Changes

The PR adds same-agent stale absolute `sessionFile` remapping in session path resolution plus a focused regression test for state-root moves.

## Best Possible Solution

Remap only validated same-agent stale filenames under the active resolved `sessionsDir`/`realBase`, while preserving cross-agent structural fallback behavior and adding custom-store regression coverage.

## Reproduction Assessment

Yes. The linked issue gives concrete CLI/state steps, and source inspection on current main shows canonical old-root absolute `sessionFile` paths can still fall through to a structural fallback that returns the original path.

## Solution Assessment

No. The fix direction is right, but this implementation should target the active sessions directory instead of recomputing `resolveAgentSessionsDir` from the default state root.

## Review Findings
