## Summary

Keep this PR open, but it still needs changes before merge because the proposed remap fixes the default state-root case while bypassing a configured custom session-store root.

## What This Changes

The PR adds a same-agent old-root `sessionFile` remap in `src/config/sessions/paths.ts` and a focused regression test in `src/config/sessions.test.ts`.

## Best Possible Solution

Retarget the same-agent remap to the active resolved sessions directory, preserve cross-agent fallback behavior, and add default plus custom `session.store` regression coverage.

## Reproduction Assessment

Yes, source-reproducible. The paired issue gives concrete stale metadata and CLI steps, and current-main source inspection shows the resolver can still return the old absolute path before write locking creates that path's directory.

## Solution Assessment

No as written. Session path resolution is the right fix boundary, but the remap must rebuild under the active `sessionsDir` or `realBase`, not the default state-derived `resolveAgentSessionsDir`.
