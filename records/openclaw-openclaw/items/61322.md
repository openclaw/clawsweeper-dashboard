## Summary

Keep this PR open: the bug is valid and the fix is in the right area, but the current patch is not merge-ready because the new remap path bypasses the active custom session-store directory.

## What This Changes

The PR adds same-agent cross-root absolute `sessionFile` remapping in session path resolution plus a focused regression test.

## Best Possible Solution

Update the remap to rebuild validated same-agent stale filenames under the active resolved `sessionsDir`/`realBase`, while preserving cross-agent fallback behavior and adding custom-store regression coverage.

## Reproduction Assessment

Yes. The paired issue gives concrete persisted metadata and ENOENT write-lock output, and current-main source shows a canonical old-root absolute path can reach the structural fallback and be returned unchanged.

## Solution Assessment

No as-is. Session path resolution is the right fix location, but the proposed remap must target the active `sessionsDir` rather than recomputing the default state-derived agent sessions directory.

## Review Findings
