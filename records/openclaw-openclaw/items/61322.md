## Summary

Keep this open: the reported bug is real, but this patch still redirects same-agent stale paths to the default state tree when a custom `session.store` is active. The remaining repair is narrow and should update the resolver plus custom-store regression coverage.

## What This Changes

The PR adds read-time remapping for same-agent old-root absolute `sessionFile` values and a focused regression test.

## Best Possible Solution

Retarget the same-agent remap to the active resolved sessions directory, preserve cross-agent structural fallback behavior, and add default plus custom `session.store` regression coverage.

## Reproduction Assessment

Yes. The paired issue gives concrete stale metadata and CLI steps, and current-main source inspection shows the resolver can return the old absolute path before persistence and write-locking use it.

## Solution Assessment

No as-is. Session path resolution is the right fix boundary, but the remap must rebuild under the active `sessionsDir` or `realBase`, not the default state-derived directory.
