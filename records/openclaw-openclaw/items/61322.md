## Summary

Keep open: the bug is source-reproducible on current main, but this PR’s remap can ignore a custom session store and send transcript writes outside the active store root.

## What This Changes

The PR adds same-agent stale absolute session transcript path remapping and a focused regression test for state-root moves.

## Best Possible Solution

Retarget the same-agent stale-root remap to the active resolved sessions directory, preserve cross-agent fallback behavior, and cover both default-store and custom-store cases while keeping https://github.com/openclaw/openclaw/pull/66174 as complementary save-time work.

## Reproduction Assessment

Yes. The paired issue gives concrete stale metadata and CLI steps, and current-main source shows the resolver can still return an old absolute structural path before the write lock creates that path’s directory.

## Solution Assessment

No as written. Session path resolution is the right boundary, but the helper must rebuild under the active `sessionsDir`/`realBase`, not the default state-derived session directory.
