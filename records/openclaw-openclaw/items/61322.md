## Summary

Keep open: the underlying stale `sessionFile` bug is real, but this PR is not merge-ready because the new remap can redirect custom `session.store` users into the default state dir and still lacks inspectable after-fix real behavior proof.

## What This Changes

The branch adds a same-agent cross-root absolute `sessionFile` remap in the session path resolver plus a focused regression test.

## Best Possible Solution

Keep the resolver-level fix, but derive same-agent remaps from the active `sessionsDir` or `realBase`, add custom `session.store` coverage, and require inspectable after-fix proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `resolveSessionFilePath` with an explicit `agentId`, a non-default `sessionsDir` from `storePath`, and a stale same-agent absolute `sessionFile`; the PR remap can produce the default state-root path instead of the active store path.

## Solution Assessment

No. The resolver is the right layer, but the implementation should remap against the active `sessionsDir` or `realBase` instead of unconditionally rebuilding through `resolveAgentSessionsDir`.
