## Summary

Keep this PR open, but do not merge it as written. The bug is valid and the PR has useful proof, but the new remap rebuilds stale paths under the default state-derived sessions directory instead of the active custom store directory, which can split transcript data for custom `session.store` users.

## What This Changes

The PR adds same-agent old-root absolute `sessionFile` remapping in session path resolution plus a focused regression test.

## Best Possible Solution

Retarget same-agent stale-root remapping to the active resolved sessions directory, preserve cross-agent fallback behavior, and add default-store plus custom-store regression coverage.

## Reproduction Assessment

Yes. The paired issue gives concrete stale metadata and CLI steps, and current main source shows the resolver can still return an old-root structural path before the write lock creates that path's directory.

## Solution Assessment

No as written. Session path resolution is the right boundary, but the helper must rebuild under the active `sessionsDir` or `realBase`, not the default state-derived agent directory.
