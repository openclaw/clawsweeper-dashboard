## Summary

Keep open. Current main still has the stale same-agent sessionFile bug, and this PR is a useful implementation candidate, but the proposed remap is not merge-ready because it can redirect custom session.store transcripts into the default state tree and it lacks the required changelog entry.

## What This Changes

The PR adds same-agent stale absolute sessionFile remapping in session path resolution and a regression test for state-root moves.

## Best Possible Solution

Revise the remap to validate the stale same-agent filename but write under the active resolved sessionsDir/realBase, preserve cross-agent structural fallback behavior, add custom-store regression coverage, and add the required changelog entry.

## Reproduction Assessment

Yes. Source inspection shows current main preserves a stale same-agent absolute agents/<agent>/sessions/<file> path through the structural fallback, and the PR-specific blocker is source-reproducible by using a custom session.store-derived sessionsDir.

## Solution Assessment

No. The fix direction is right, but this implementation is not the narrowest maintainable solution because the remap should target the active sessionsDir, not resolveAgentSessionsDir from the default state root.
