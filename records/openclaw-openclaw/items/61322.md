## Summary

Keep this PR open, but it needs changes before merge. The bug is valid and the resolver is the right area, but the current diff remaps same-agent stale paths through the default state-derived agent sessions directory instead of the active sessionsDir, which can break custom session.store users.

## What This Changes

The PR adds same-agent cross-root absolute sessionFile remapping in src/config/sessions/paths.ts and a focused regression test in src/config/sessions.test.ts.

## Best Possible Solution

Retarget same-agent stale-root remapping to the active resolved sessionsDir/realBase, preserve cross-agent fallback behavior, and add both default-store and custom-store regression coverage.

## Reproduction Assessment

Yes. The paired issue gives concrete stale metadata and CLI steps, and current-main source shows the resolver can still return the old-root structural path before the write lock creates that path's directory.

## Solution Assessment

No as written. Session path resolution is the right boundary, but the helper must rebuild under the active sessionsDir/realBase rather than the default state-derived agent directory.
