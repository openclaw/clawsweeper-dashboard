## Summary

Keep open: the bug is valid and this PR is the active implementation path, but the current branch still misroutes stale same-agent paths for custom session-store users by ignoring the active `sessionsDir`.

## What This Changes

The PR adds same-agent cross-root `sessionFile` remapping in `src/config/sessions/paths.ts` plus a focused regression test in `src/config/sessions.test.ts`.

## Best Possible Solution

Remap validated same-agent stale filenames beneath the active resolved `sessionsDir`/`realBase`, preserve cross-agent fallback behavior, and add default-root plus custom-store regression coverage.

## Reproduction Assessment

Yes. The paired issue provides concrete stale `sessions.json` metadata and ENOENT write-lock evidence, and current-main source shows the resolver can preserve that old absolute path before persistence and writes.

## Solution Assessment

No as-is. Session path resolution is the right fix location, but the remap must target the active `sessionsDir`/`realBase` instead of recomputing the default state-derived agent sessions directory.
