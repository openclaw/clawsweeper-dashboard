## Summary

Keep this PR open for repair: the reported current-main bug is valid, but the proposed helper rebuilds stale same-agent paths under the default state directory instead of the active `session.store` sessions directory.

## What This Changes

The PR adds a same-agent cross-root `sessionFile` remap helper in `src/config/sessions/paths.ts` and a focused regression test in `src/config/sessions.test.ts`.

## Best Possible Solution

Retarget the same-agent stale-path remap to the active resolved `sessionsDir`/`realBase`, preserve existing cross-agent fallback behavior, and add custom-store regression coverage.

## Reproduction Assessment

Yes. The paired issue gives concrete stale metadata and ENOENT output, and source inspection shows current main can return that old absolute path before the write lock creates its directory.

## Solution Assessment

No as-is. Session path resolution is the right fix location, but the helper must rebuild under the active sessions directory instead of calling the default state-root resolver.

## Review Findings
