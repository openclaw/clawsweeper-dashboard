## Summary

Keep open. This PR targets a real source-reproducible session-path bug, but the proposed remap currently bypasses the active `sessionsDir` used by custom `session.store` roots.

## What This Changes

The PR adds same-agent stale absolute `sessionFile` remapping in session path resolution plus a focused regression test for state-root moves.

## Best Possible Solution

Retarget the same-agent stale-path remap to the active resolved `sessionsDir`/`realBase`, preserve cross-agent fallback behavior, and add custom `session.store` regression coverage.

## Reproduction Assessment

Yes. The paired issue gives concrete stale metadata and ENOENT output, and current-main source shows the resolver can return the old absolute path before the write lock creates that directory.

## Solution Assessment

No as-is. Session path resolution is the right fix location, but the new helper must rebuild under the active `sessionsDir`/`realBase` rather than `resolveAgentSessionsDir()`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91
