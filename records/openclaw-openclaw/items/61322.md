## Summary

This PR should stay open because it targets a real session-resume bug, but the current diff redirects same-agent stale paths through the default state tree instead of the active custom session store, so it needs a small code/test repair before merge.

## What This Changes

The PR adds same-agent cross-root absolute sessionFile remapping in session path resolution plus a default-root regression test.

## Best Possible Solution

Keep the resolver fix in this path, but make the same-agent stale filename remap land under the active resolved sessionsDir/realBase and add regression coverage for both default-root and custom-store configs.

## Reproduction Assessment

Yes. The paired issue and PR body provide concrete stale sessions.json metadata plus ENOENT output, and source inspection shows current main can preserve that old absolute path before session writes.

## Solution Assessment

No as-is. Session path resolution is the right layer, but the remap target must use the active sessionsDir/realBase rather than recomputing the default state-derived agent directory.
