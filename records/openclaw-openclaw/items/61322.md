## Summary

Keep this PR open: it targets a valid session resume bug and is paired with the still-open linked bug report, but the current patch redirects same-agent stale paths through the default state tree instead of the active custom session store root.

## What This Changes

The branch adds same-agent stale absolute sessionFile remapping in session path resolution plus a focused regression test.

## Best Possible Solution

Target the remap at the active resolved sessionsDir/realBase after validating the stale same-agent filename, while preserving cross-agent structural fallback behavior and adding custom-store regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main preserves same-agent old-root canonical sessionFile paths through the structural fallback, and the PR-specific regression is source-reproducible with a custom session.store-derived sessionsDir.

## Solution Assessment

No. The fix direction is right, but this implementation is not the narrowest maintainable solution because the remap should write under the active sessionsDir rather than resolveAgentSessionsDir from the default state root.

## Review Findings

Overall correctness: patch is incorrect
