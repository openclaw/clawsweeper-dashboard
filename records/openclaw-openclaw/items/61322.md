## Summary

Keep this PR open but not merge-ready: current main still preserves the stale same-agent old-root path, while the PR's proposed remap can misroute custom session-store users into the default state directory.

## What This Changes

The PR adds same-agent cross-root `sessionFile` remapping in `src/config/sessions/paths.ts` plus a focused regression test in `src/config/sessions.test.ts`.

## Best Possible Solution

Keep the focused resolver repair, but remap stale same-agent paths relative to the active `sessionsDir` or `realBase`, preserve sibling-agent fallback behavior, add custom-store coverage, and require redacted after-fix proof.

## Reproduction Assessment

Yes for the PR defect: from source inspection, calling `resolveSessionFilePath` with an explicit `agentId`, a custom `sessionsDir` from `storePath`, and a stale same-agent absolute `sessionFile` returns the default state-dir path instead of the active store path. I did not run a mutating runtime reproduction in this read-only review.

## Solution Assessment

No. The resolver is the right layer, but the implementation should remap against the active `sessionsDir` or `realBase` instead of unconditionally rebuilding through `resolveAgentSessionsDir`.
