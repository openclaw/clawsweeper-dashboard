## Summary

Keep open: the focused session-state bug fix is still useful, but the patch currently remaps stale same-agent paths into the default state tree instead of the active session store for custom `session.store` callers, and the external contributor proof remains insufficient.

## What This Changes

The PR adds a same-agent cross-root `sessionFile` remap helper in `src/config/sessions/paths.ts` and a focused regression test in `src/config/sessions.test.ts`.

## Best Possible Solution

Keep the narrow resolver fix, but remap same-agent stale paths relative to the active `sessionsDir` or `realBase`, preserve sibling-agent fallback behavior, and add a custom `session.store` regression before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `resolveSessionFilePath` with an explicit `agentId`, a custom `sessionsDir` derived from `storePath`, and a stale same-agent absolute `sessionFile`; the PR helper returns the default agent sessions dir instead of the active store dir.

## Solution Assessment

No. The narrow resolver location is appropriate, but the implementation should remap into the active `sessionsDir` rather than unconditionally using `resolveAgentSessionsDir`.
