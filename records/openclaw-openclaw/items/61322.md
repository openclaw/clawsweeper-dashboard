## Summary

Keep open: the PR targets a real, source-reproducible session path bug, but the current diff still has a blocking custom-store defect because the new remap rebuilds paths under the default state directory instead of the active session store directory.

## What This Changes

The PR adds same-agent stale absolute session transcript path remapping and a focused regression test for state-root moves.

## Best Possible Solution

Retarget the same-agent stale-root remap to the active resolved sessions directory, preserve cross-agent fallback behavior, and add default-store plus custom-store regression coverage.

## Reproduction Assessment

Yes. The paired issue gives concrete stale metadata and CLI steps, and current-main source shows the resolver can still return an old absolute structural path before the write lock creates that path's directory.

## Solution Assessment

No as written. Session path resolution is the right boundary, but the helper must rebuild under the active `sessionsDir`/`realBase`, not `resolveAgentSessionsDir(...)` from the default state root.
