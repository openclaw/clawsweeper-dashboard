## Summary

Keep this PR open, but do not merge it as-is: the fix targets the right session resolver bug, but the proposed remap can bypass configured custom session-store roots.

## What This Changes

The PR adds same-agent stale absolute `sessionFile` remapping in session path resolution plus a focused regression test.

## Best Possible Solution

Retarget the same-agent stale-path remap to the active resolved `sessionsDir`/`realBase`, preserve cross-agent fallback behavior, and add custom-store regression coverage.

## Reproduction Assessment

Yes. The paired issue gives concrete stale `sessionFile` metadata and ENOENT write-lock output, and current-main source shows the same canonical old-root path can be returned unchanged.

## Solution Assessment

No as-is. Session path resolution is the right fix location, but the proposed helper must rebuild the filename under the active sessions directory instead of the default state-derived directory.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Remap stale paths under the active sessionsDir:** `src/config/sessions/paths.ts:188`
  - body: When `resolveSessionFilePath` is called with a custom `sessionsDir` from `session.store`, this helper ignores it and rebuilds stale same-agent paths under `resolveAgentSessionsDir()`.
