## Summary

Keep open: the reported bug is valid and the fix direction is right, but this branch would route same-agent stale paths into the default state tree instead of the active custom `session.store` directory.

## What This Changes

The branch adds same-agent cross-root `sessionFile` remapping in session path resolution and a focused regression test.

## Best Possible Solution

Remap only validated same-agent stale filenames under the active resolved `sessionsDir`/`realBase`, preserve existing cross-agent fallback behavior, and add custom-store regression coverage.

## Reproduction Assessment

Yes. The linked issue gives concrete stale metadata and CLI steps, and current-main source shows old-root absolute paths can reach the structural fallback that preserves the stale path.

## Solution Assessment

No. The fix direction is correct, but the helper should target the active sessions directory instead of recomputing the default state-derived agent sessions directory.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91
