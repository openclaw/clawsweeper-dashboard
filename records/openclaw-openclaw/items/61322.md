## Summary

Keep this PR open, but do not merge it as-is. The reported bug is real and the patch is in the right resolver area, but the new remap ignores the active custom session store directory and can redirect transcript writes back into the default state tree.

## What This Changes

The PR changes session path resolution to remap same-agent stale absolute sessionFile paths into the current state directory and adds a focused regression test.

## Best Possible Solution

Retarget the same-agent stale-path remap to the active resolved sessionsDir/realBase, preserve the existing cross-agent fallback behavior, and add custom-store regression coverage.

## Reproduction Assessment

Yes. The paired issue gives concrete stale sessionFile metadata and ENOENT write-lock output, and current-main source shows the same old-root path can be returned unchanged before the write lock creates its directory.

## Solution Assessment

No as-is. Session path resolution is the right fix location, but the helper must rebuild under the active sessionsDir/realBase rather than resolveAgentSessionsDir from the default state root.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Remap stale paths under the active sessionsDir:** `src/config/sessions/paths.ts:188`
