## Summary

Keep this PR open: the bug is valid and the fix direction is narrow, but the current patch redirects same-agent stale paths into the default state-derived sessions directory instead of the active custom session store directory.

## What This Changes

The PR adds same-agent cross-root sessionFile remapping in session path resolution and a focused regression test.

## Best Possible Solution

Remap validated same-agent stale filenames under the active resolved sessionsDir/realBase, preserve cross-agent fallback behavior, and add custom-store regression coverage.

## Reproduction Assessment

Yes. The linked issue gives concrete stale metadata and CLI ENOENT logs, and current-main source inspection shows that the resolver can return the old absolute path before the write lock creates that directory.

## Solution Assessment

No. The fix belongs in session path resolution, but the proposed helper should target the active sessionsDir/realBase instead of recomputing the default state-derived agent sessions directory.
