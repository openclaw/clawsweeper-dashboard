## Summary

Keep this PR open, but it needs changes before merge. The resolver is the right boundary for the stale same-agent session path bug, but the proposed remap rebuilds paths under the default state-derived agent directory and can bypass a configured custom session store.

## What This Changes

The PR adds same-agent old-root absolute `sessionFile` remapping in session path resolution plus a focused regression test.

## Best Possible Solution

Retarget same-agent stale-root remapping to the active resolved sessions directory, preserve cross-agent fallback behavior, and add both default-store and custom-store regression coverage.

## Reproduction Assessment

Yes. The paired issue gives concrete stale metadata and CLI steps, and current-main source shows the resolver can still return the old-root structural path before the write lock creates that path's directory.

## Solution Assessment

No as written. Session path resolution is the right boundary, but the helper must rebuild under the active `sessionsDir` or `realBase`, not the default state-derived agent directory.
