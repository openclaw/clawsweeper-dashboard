## Summary

Keep open. The bug is valid and this PR is the right narrow area, but the current branch still bypasses configured custom session-store roots when remapping same-agent stale paths.

## What This Changes

The PR adds a same-agent old-root sessionFile remap in `src/config/sessions/paths.ts` and a focused regression test in `src/config/sessions.test.ts`.

## Best Possible Solution

Retarget the same-agent remap to the active resolved sessions directory, preserve cross-agent structural fallback behavior, and add default plus custom `session.store` regression coverage.

## Reproduction Assessment

Yes. The paired issue gives concrete stale metadata and CLI steps, and current-main source inspection shows the resolver can still return the old absolute path before persistence and write locking use it.

## Solution Assessment

No as-is. Session path resolution is the right fix boundary, but the remap must rebuild under the active `sessionsDir` or `realBase`, not the default state-derived directory.
