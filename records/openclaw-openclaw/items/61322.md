## Summary

Keep open. The linked bug is still present on current main, and the PR's fix direction is useful, but the proposed remap ignores configured custom session.store directories and can write transcripts under the default state tree instead of the active store; it also lacks the required changelog entry for a user-facing fix.

## What This Changes

The PR adds same-agent stale absolute sessionFile remapping in session path resolution and a unit regression test for state-root moves.

## Best Possible Solution

Revise the remap so it validates the stale same-agent filename but writes under the already-resolved active sessionsDir/realBase, while preserving cross-agent structural fallback behavior and adding regression coverage for both default state-root and custom session.store paths.

## Reproduction Assessment

Yes. Current main reaches the structural fallback for a stale absolute path like /home/rai/.openclaw/agents/shikamaru/sessions/sess-1.jsonl and preserves that old path; the PR-specific blocker is reproducible by passing a custom sessionsDir/store path and observing the proposed helper remap to resolveAgentSessionsDir instead of the active directory.

## Solution Assessment

No. The fix direction is right, but this implementation is not the narrowest maintainable solution because it remaps through the default state-derived agent sessions directory rather than the active sessionsDir used by the resolver.
