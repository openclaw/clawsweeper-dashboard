## Summary

Keep open: current main still exposes browser file upload through standard file chooser / file-input paths, while custom drop/paste/native drag upload behavior for components such as ByteUploader remains a product and security design decision rather than implemented core behavior.

## What This Changes

Request browser upload support for non-standard custom uploader components by adding an explicit DataTransfer, clipboard, or native-drag style file injection path.

## Best Possible Solution

Define an explicit browser file-drop or paste upload action with clear local-file safety checks, target semantics, and live proof on a representative custom uploader before wiring it into core.

## Reproduction Assessment

## Solution Assessment
