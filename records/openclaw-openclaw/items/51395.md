## Summary

Keep open: current main and v2026.5.22 still implement browser upload as a file chooser or direct file-input path, with no DataTransfer, clipboard paste, native drag, or ByteUploader-style custom uploader path. The external FSB reference is useful because it supports an explicit drop-file API shape, but OpenClaw still needs a maintainer product/API and file-injection security decision before implementation.

## What This Changes

Request browser upload support for non-standard custom uploaders such as ByteUploader, potentially through DataTransfer drop, clipboard paste, or native drag simulation.

## Best Possible Solution

Add an explicit custom-upload or drop-file browser action in the browser plugin, with target semantics, upload-root validation, synthetic custom-uploader tests, docs, and a security-reviewed boundary instead of a silent fallback behind ordinary upload.
