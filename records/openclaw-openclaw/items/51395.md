## Summary

Keep open. Current main still routes browser uploads through Playwright/Chrome MCP file-chooser or standard file-input APIs only; it has no generic DataTransfer, paste, or native drag fallback for custom uploader widgets, and the adjacent #38844 tracks stale refs and timeout messaging rather than non-standard upload components.

## What This Changes

Request a browser upload fallback for custom uploader components that bypass standard `<input type="file">` handling.

## Best Possible Solution

Add a generic, opt-in browser-plugin upload fallback with explicit target/mode semantics, preserve the existing upload-root validation, add synthetic uploader tests, and document when agents should use it.

## Reproduction Assessment

No high-confidence live reproduction is available because the report names service-specific ByteDance surfaces without a public fixture. Source inspection does show that current main lacks the requested fallback path.

## Solution Assessment

Unclear. The requested direction is valid, but the maintainable fix needs a maintainer decision on whether this should be automatic fallback or an explicit upload mode/target contract.
