## Summary

Keep this open: current main and v2026.6.8 still only expose standard file-input/file-chooser upload paths, while the requested ByteUploader/drop/paste/native-drag behavior remains a new Browser plugin capability with product and security decisions still needed.

## What This Changes

The issue requests a browser upload fallback for custom non-file-input uploaders such as ByteUploader.

## Best Possible Solution

Add an explicit Browser plugin drop-file or paste-file upload mode only after maintainers settle the API shape and security semantics; keep standard file-input upload behavior unchanged.
