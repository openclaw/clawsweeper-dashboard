## Summary

Keep this open: current main and v2026.6.10 still expose standard file chooser, direct file-input, and Chrome MCP upload paths only. The requested ByteUploader-style drop, paste, or native-drag behavior remains a new Browser plugin capability that needs explicit product, security, and live-repro review.

## What This Changes

The issue requests a browser upload capability for non-standard ByteUploader-style components that do not populate a standard file input through the existing upload path.

## Best Possible Solution

Add an explicit Browser plugin drop-file, paste-file, or native-drag upload mode only after maintainers settle the API and security contract; keep standard file-input upload behavior unchanged.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows current upload support is limited to file chooser, direct file-input, and Chrome MCP upload_file paths, so the requested custom-uploader mode remains absent.

## Solution Assessment

No for a silent fallback. The maintainable path is an explicit Browser plugin upload mode with approved security semantics, path validation, docs, tests, and live custom-uploader proof.
