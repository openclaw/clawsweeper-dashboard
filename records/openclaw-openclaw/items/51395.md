## Summary

Keep open: current main and v2026.6.10 still expose standard file chooser, direct file-input, and Chrome MCP upload paths only. The requested ByteUploader-style drop, paste, or native-drag upload behavior remains a new Browser plugin capability that needs maintainer API and security direction.

## What This Changes

The issue requests Browser upload support for non-standard ByteUploader/custom uploaders via DataTransfer, clipboard paste, or native drag fallback.

## Best Possible Solution

Add an explicit Browser plugin drop-file, paste-file, or native-drag upload mode only after maintainers settle the API and security contract; keep standard file-input upload unchanged and require live custom-uploader proof.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows current upload support is limited to file chooser, file input, and Chrome MCP upload paths, but no ByteUploader page was exercised.

## Solution Assessment

No for a silent fallback. The maintainable path is an explicit Browser plugin upload mode with approved security semantics, managed-path validation, docs, tests, and live custom-uploader proof.
