## Summary

Keep open. Current main and v2026.5.18 still only support file chooser or direct file-input uploads; the FSB reference provides a useful explicit drop-file model, but this is new browser-tool/API work with product and safety choices.

## What This Changes

Request a browser upload fallback or explicit custom-uploader path for ByteDance-style non-standard upload widgets that do not populate standard file inputs.

## Best Possible Solution

Add an explicit opt-in drop/custom-upload browser action that preserves upload-root validation, declares target and strategy semantics, and has synthetic dropzone tests plus docs.

## Reproduction Assessment

No. There is no high-confidence live ByteUploader reproduction because the report names service-specific ByteDance surfaces without a public fixture; source inspection does show current main lacks DataTransfer, paste, or native drag upload support.

## Solution Assessment

Unclear. The requested mechanisms are plausible, but the safer maintainable direction is an explicit drop/custom-upload action rather than silently guessing after ordinary upload calls.
