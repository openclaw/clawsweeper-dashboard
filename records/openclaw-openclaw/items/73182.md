## Summary

Keep open. The issue discussion's reproduction still matches current main: API-level thinking can still resolve non-off by default for Claude 4.6 and reasoning-capable catalog models, cron jobs without payload thinking still inherit runtime defaults, and Teams rendering still lacks the shared reasoning-payload suppression guard.

## What This Changes

Request API thinking to be opt-in by default, propagate explicit thinking defaults to cron jobs, and keep reasoning payloads out of chat delivery unless explicitly visible.

## Best Possible Solution

Define a single cost-safe policy: API thinking stays off unless explicitly configured or truly provider-required, cron and normal turns resolve agent/global defaults consistently, reasoning visibility remains separate, and all channel renderers reuse the shared reasoning-payload suppression helper.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main by resolving an unconfigured Claude 4.6 or reasoning-capable catalog model, running a cron `agentTurn` without payload `thinking`, or passing an `isReasoning` payload through the Teams renderer.

## Solution Assessment

No. Current main is still a partial mitigation: it separates some reasoning visibility from API thinking and special-cases Opus 4.7, but it keeps non-off implicit API thinking for other paths and leaves Teams delivery unsuppressed.
