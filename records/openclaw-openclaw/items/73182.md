## Summary

Keep open. Current main contains useful partial mitigations, including global reasoning visibility config and Opus 4.7 defaulting API thinking off, but the core cost complaint is not fully resolved: Claude 4.6 still defaults to adaptive thinking, generic reasoning-capable catalog models still resolve to a non-off medium-equivalent, cron jobs still fall through runtime thinking defaults when payload thinking is absent, and Teams final rendering still lacks the shared reasoning-payload suppression guard.

## What This Changes

The issue requests making Claude/Anthropic API thinking opt-in by default, adding global/default controls and warnings, honoring those defaults in cron jobs, and preventing reasoning content from appearing in chat surfaces.

## Best Possible Solution

Define one explicit policy: API thinking defaults off unless configured or truly required by the provider/model, global and per-agent thinking defaults apply consistently to chat and cron, reasoning visibility stays separate and off by default, channel delivery uses the shared reasoning-payload guard before rendering, and docs/changelog/doctor diagnostics call out billing-impacting defaults clearly.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by resolving an unconfigured Claude 4.6 or reasoning-capable catalog model, running a cron agentTurn without payload thinking, or passing a reasoning-marked payload through the Teams renderer.

## Solution Assessment

No. Current main is not the final solution because it only partially mitigates Opus 4.7 and visibility defaults while preserving non-off implicit API thinking for other reasoning-capable models and leaving Teams without a reasoning-payload suppression guard.
