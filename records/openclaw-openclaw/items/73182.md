## Summary

Keep open. Current main and v2026.6.8 have partially addressed the report by adding global defaults, keeping Opus 4.7/4.8 thinking off by default, and landing a Telegram/Feishu preview fix, but the central billing/default policy remains: Claude 4.6 can still resolve to adaptive thinking and generic reasoning-capable models still fall back to non-off thinking without an explicit user setting.

## What This Changes

The issue asks OpenClaw to make Claude/Anthropic reasoning and thinking opt-in by default, add global/default and warning ergonomics, and separate billed API thinking from rendered reasoning visibility.

## Best Possible Solution

Choose a maintainer-approved cost-safe thinking policy for Anthropic and generic reasoning-capable defaults, preserve explicit opt-in configs, and cover chat plus cron paths with focused tests and docs.
