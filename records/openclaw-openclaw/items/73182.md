## Summary

Keep open. Current main has useful partial fixes, including global `agents.defaults.reasoningDefault` and Telegram/Feishu preview handling, but the billing default remains source-reproducible for Claude 4.6 and generic reasoning-capable models, cron still inherits that non-off thinking resolver when payload thinking is omitted, and the Microsoft Teams renderer still lacks a local `isReasoning` suppression guard.

## What This Changes

The issue asks OpenClaw to make Claude/API thinking opt-in, provide a global reasoning visibility default, warn on enabled reasoning, and prevent reasoning blocks from leaking into chat surfaces.

## Best Possible Solution

Adopt an explicit policy that API thinking is off unless configured or provider-required, ensure cron and normal turns share that policy, and suppress hidden reasoning payloads in every channel renderer unless visibility is explicitly enabled.

## Reproduction Assessment

Yes, source-level. Current main still has non-off implicit thinking defaults for Claude 4.6 and generic reasoning-capable models, cron jobs without `payload.thinking` inherit that resolver, and the Teams renderer can render reasoning-marked payloads if they reach that path.

## Solution Assessment

No, the current implementation is not enough to close this. It solves the global visibility-default and Telegram/Feishu preview pieces, but the remaining fix should split explicit thinking-default policy from channel-level hidden-reasoning suppression.
