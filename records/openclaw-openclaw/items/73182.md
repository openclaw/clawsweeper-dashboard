## Summary

Keep open. Current main still has source-, docs-, and test-backed non-off implicit API thinking defaults for reasoning-capable models, including Claude 4.6/adaptive paths; isolated cron jobs without payload thinking still fall through the same default resolver; and the Microsoft Teams renderer still lacks the reasoning-payload suppression guard used by generic channel/WebChat paths. The existing global `agents.defaults.reasoningDefault`/`thinkingDefault` config support helps, but it does not close the billing-default, cron-inheritance, warning, or Teams-delivery parts of this report.

## What This Changes

The issue asks OpenClaw to stop silently defaulting Anthropic Claude reasoning/thinking on, add a global/default control, warn about billing-impacting enabled reasoning, and prevent reasoning blocks from being rendered into chat channels.

## Best Possible Solution

Define one explicit policy across runtime, cron, docs, and channel delivery: API thinking stays off unless explicitly configured or truly required by the provider/model; cron `agentTurn` jobs inherit the same resolved agent/global thinking default as normal turns; startup or doctor surfaces warn when configured agents or cron jobs will request billable thinking; and chat channels suppress `isReasoning` payloads unless they intentionally implement a separate reasoning lane.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: resolve an unconfigured reasoning-capable/Claude 4.6 model or run an isolated `agentTurn` cron payload with no `thinking`, then observe the resolver selecting non-off API thinking; pass an `isReasoning` payload into the Teams renderer and it is converted to a sendable text message.

## Solution Assessment

No. Current main is only a partial mitigation because it has global config fields and an Opus 4.7 `off` exception, but still keeps non-off implicit defaults, incomplete default-agent cron inheritance, and no Teams reasoning-payload guard.

- Changing cron inheritance for default-agent jobs may affect existing scheduled jobs that rely on current implicit thinking defaults.
