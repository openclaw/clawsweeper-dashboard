## Summary

Keep open. Current main has partial fixes for global reasoning visibility and Telegram/Feishu previews, but the API-thinking cost default is still source-reproducible for reasoning-capable models, cron jobs without explicit `thinking` still inherit that resolver, and Microsoft Teams still has a rendering path without reasoning-payload suppression.

## What This Changes

The issue requests cost-safe Claude/reasoning defaults, global/default ergonomics, startup/audit warnings, and separation of API thinking from visible reasoning output.

## Best Possible Solution

Adopt an explicit policy that API thinking is off unless configured or provider-required, make cron and normal turns share it, and suppress hidden reasoning payloads in every channel renderer unless visibility is explicitly enabled.

## Reproduction Assessment

Yes, source-level. Current main still has non-off implicit thinking defaults for reasoning-capable models, omitted cron thinking still inherits that resolver, and the Teams renderer can render reasoning-marked payloads if they reach that plugin-local path.

## Solution Assessment

No, the current partial implementation is not enough to close this. The remaining fix needs a maintainer decision on cost-safe thinking defaults plus a channel-level hidden-reasoning suppression pass.
