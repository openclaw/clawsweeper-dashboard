## Summary

Keep open. Current main still has the cost-affecting implicit API thinking behavior for Claude 4.6 and other reasoning-capable models, cron jobs without explicit payload thinking still resolve through that default path, and the Teams renderer still lacks a reasoning-payload suppression guard. The issue discussion includes recent reproduction and deployment cost data, so this is neither stale nor already implemented.

## What This Changes

Request to make Claude/API thinking opt-in by default, apply reasoning/thinking defaults consistently across agents and cron jobs, warn when costly reasoning is enabled, and prevent reasoning blocks from rendering into chat surfaces.

## Best Possible Solution

Define one explicit policy: API thinking is off unless configured or truly required by the provider/model, cron agentTurn jobs inherit the same resolved agent/global thinking default as normal turns, channel delivery suppresses reasoning-marked payloads unless a dedicated reasoning lane is implemented, and billing-impacting default changes are surfaced in diagnostics and release notes.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: resolve an unconfigured Claude 4.6 or reasoning-capable catalog model, run an isolated cron agentTurn with no payload thinking, or pass a reasoning-marked payload through the Teams renderer.

## Solution Assessment

No. Current main is only a partial mitigation because it keeps non-off implicit API thinking defaults and leaves cron inheritance and Teams reasoning delivery behavior unresolved.
