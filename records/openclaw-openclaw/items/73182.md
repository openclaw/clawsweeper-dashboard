## Summary

Keep open. Current main still leaves API-level thinking enabled by default for some Claude/provider-advertised and reasoning-capable catalog paths, cron jobs without explicit `thinking` still resolve through that default path, and the Microsoft Teams renderer can still deliver reasoning payload text.

## What This Changes

The issue asks OpenClaw to make API thinking opt-in, add global/default warnings, make cron inheritance cost-safe, and prevent hidden reasoning blocks from rendering in chat surfaces.

## Best Possible Solution

Adopt a single cost-safe policy: API thinking stays off unless explicitly configured or provider-required, cron inherits agent/global defaults consistently, reasoning visibility remains separate, and Teams uses the shared reasoning-payload suppression path with docs or doctor/startup warnings.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: unconfigured reasoning-capable defaults still resolve non-off in the thinking resolver, cron jobs without payload `thinking` call that resolver, and Teams renders an `isReasoning` payload without suppression.

## Solution Assessment

No. Current main is only a partial mitigation: global defaults and Opus 4.7 `off` exist, but Claude 4.6/provider-advertised defaults, catalog medium fallback, cron inheritance, and Teams delivery still leave the reported billing and visibility problem open.
