## Summary

Keep open. Current main has partial mitigations, but it still intentionally defaults API thinking to non-off for reasoning-capable and Claude 4.6 paths, cron jobs can inherit that fallback when no thinking is set, and Teams rendering still lacks an `isReasoning` suppression guard.

## What This Changes

The issue requests opt-in API thinking/reasoning defaults, a global default, warnings, and channel suppression so Claude thinking does not add billing or visible chat output without explicit configuration.

## Best Possible Solution

Adopt one explicit policy: API thinking stays off unless configured or provider-required, cron and normal turns share agent/global defaults, hidden reasoning visibility is independent, and channel renderers suppress hidden reasoning payloads by default.

## Reproduction Assessment

Yes. Source-level reproduction is available by resolving an unset Claude 4.6 or reasoning-capable model, running a cron `agentTurn` without `payload.thinking`, or passing an `{ isReasoning: true }` payload through the Teams renderer.

## Solution Assessment

No. Current main is only a partial mitigation because global defaults exist and reasoning visibility is guarded when thinking is active, but unset API thinking still resolves non-off and Teams can still render hidden reasoning payloads.
