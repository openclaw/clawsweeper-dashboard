## Summary

Keep open. Current main has partial fixes for global reasoning visibility defaults and Telegram/Feishu previews, but unset agent/cron thinking still falls through to provider/catalog non-off defaults and the Teams plugin-local render path still lacks a local reasoning-payload suppression guard.

## What This Changes

The issue asks OpenClaw to make reasoning/thinking defaults explicit and cost-safe for Claude and cron/agent turns, while preventing reasoning blocks from leaking to chat.

## Best Possible Solution

Adopt a maintainer-approved policy that API thinking is off unless explicitly configured or provider-required, keep cron and normal turns on that policy, and suppress reasoning-marked payloads in every channel renderer unless reasoning visibility is explicitly enabled.

## Reproduction Assessment

Yes, source-level: current main still routes omitted thinking through provider/catalog defaults, documents and tests non-off fallback for reasoning-capable models, and lacks a Teams-local `isReasoning` suppression check in the renderer path.

## Solution Assessment

No, current main is only a partial solution; the remaining default policy and channel suppression work require maintainer product/security review rather than closing the issue as fixed.
