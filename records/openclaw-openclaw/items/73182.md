## Summary

Keep open. Current main has partial fixes, but it still preserves non-off implicit API thinking defaults for some Claude/reasoning-capable paths, cron jobs without explicit `thinking` still inherit that resolver, and the Teams renderer still has a source-visible path that can render reasoning-marked payloads.

## What This Changes

The issue asks for Claude API thinking to be opt-in, for global/per-job defaults or warnings to prevent billing surprises, and for hidden reasoning blocks not to leak into chat surfaces.

## Best Possible Solution

Adopt an explicit default-thinking policy, make cron and normal turns share that policy predictably, and ensure every channel renderer suppresses hidden reasoning payloads unless visibility is explicitly enabled.

## Reproduction Assessment

Yes, source-level. Current main still has non-off implicit thinking defaults for some Claude/reasoning-capable paths, cron jobs without `payload.thinking` inherit that resolver, and Teams can render reasoning-marked payloads if they reach the plugin renderer.

## Solution Assessment

No, current main is only a partial mitigation. The maintainable fix should split the remaining work into an explicit thinking-default policy, cron/default audit behavior, and channel-level hidden-reasoning suppression rather than treating the issue as already implemented.
