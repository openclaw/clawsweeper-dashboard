## Summary

Keep open. Current main still has source-level paths that enable implicit API thinking for reasoning-capable/Claude models, cron jobs without explicit `thinking` can inherit that policy, Teams can render reasoning payloads, and the open linked PR is only a partial channel-specific fix.

## What This Changes

Requests restoring billing-safe opt-in thinking/reasoning defaults for Claude and reasoning-capable models, adding default/audit controls, and suppressing hidden reasoning blocks in messaging channels.

## Best Possible Solution

Adopt one explicit policy: API thinking stays off unless configured or provider-required, cron and normal turns share agent/global defaults, and channel renderers suppress hidden reasoning payloads unless visibility is explicitly enabled.

## Reproduction Assessment

Yes, source-level. An unconfigured reasoning-capable model reaches the implicit `medium`/provider-default thinking resolver, cron `agentTurn` jobs without `payload.thinking` use the same resolver, and Teams renders reply payloads without an `isReasoning` guard.

## Solution Assessment

No. Current main is only a partial mitigation, and the open linked PR addresses Telegram/Feishu `reasoningDefault` fallback rather than the core billing default, cron inheritance, or Teams hidden-reasoning delivery path.
