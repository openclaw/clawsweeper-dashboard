## Summary

Keep open. Current main still has only partial mitigation: global reasoning visibility exists and Opus 4.7 defaults off, but Claude 4.6 and generic reasoning-capable fallbacks can still resolve to non-off API thinking, cron jobs without payload thinking still fall through that policy, and Teams rendering still lacks the shared reasoning-payload suppression guard.

## What This Changes

The issue asks OpenClaw to make API thinking opt-in by default, add global/default controls and diagnostics, carry those defaults into cron agent turns, and prevent reasoning payloads from leaking into chat surfaces.

## Best Possible Solution

Define one explicit policy: API thinking is off unless explicitly configured or truly required by the provider/model, normal turns and cron agentTurn jobs resolve agent/global thinking defaults consistently, reasoning visibility remains separate and off by default, channel delivery uses a shared reasoning-payload guard before rendering, and docs/changelog/diagnostics call out billing-impacting defaults clearly.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: resolve an unconfigured Claude 4.6 or reasoning-capable catalog model, run an isolated cron agentTurn without payload `thinking`, or pass a reasoning-marked payload through the Teams renderer and observe the non-off or unsuppressed path.

## Solution Assessment

No. Current main is not the best final solution because it only partially mitigates visibility defaults and Opus 4.7 while preserving non-off implicit API thinking for other reasoning-capable models and leaving Teams delivery without a reasoning-payload suppression guard.
