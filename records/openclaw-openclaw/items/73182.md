## Summary

Keep open. Current main still has non-off implicit API thinking defaults for reasoning-capable models and Claude 4.6, cron `agentTurn` jobs without explicit `thinking` still fall through to that resolver, and the Microsoft Teams block-delivery renderer still lacks a reasoning-payload suppression guard.

## What This Changes

Request defaulting API thinking/reasoning to opt-in behavior, adding global/default audits and warnings, and preventing hidden thinking from rendering in chat channels.

## Best Possible Solution

Adopt one explicit policy: API thinking stays off unless configured or provider-required, cron and normal turns share agent/global defaults, reasoning visibility remains independent, and every channel renderer suppresses hidden reasoning payloads.

## Reproduction Assessment

Yes. A source-level reproduction exists by resolving an unconfigured reasoning-capable or Claude 4.6 model, running a cron `agentTurn` without `payload.thinking`, or passing `{ isReasoning: true }` through the Teams renderer.

## Solution Assessment

No. Current main is only a partial mitigation: global `reasoningDefault` exists, but API thinking still defaults on for capable models and Teams still has an unsuppressed reasoning delivery path.
