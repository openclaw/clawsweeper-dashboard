## Summary

Keep open. Current main still preserves implicit non-off API thinking defaults for reasoning-capable models and Claude 4.6, cron jobs without explicit `thinking` can still inherit that resolver, and the Teams renderer still has a path that can render reasoning payloads.

## What This Changes

The issue asks OpenClaw to make Claude/API thinking opt-in, provide safer global defaults and startup/doctor warnings, and prevent hidden reasoning from reaching chat channels.

## Best Possible Solution

Adopt one central policy: API thinking stays off unless explicitly configured or provider-required, cron and normal turns share agent/global defaults, reasoning visibility is independent, and every channel renderer suppresses hidden reasoning payloads by default.

## Reproduction Assessment

Yes. Current main gives a source-level reproduction by resolving a reasoning-capable catalog model with no explicit thinking default, running an `agentTurn` cron without `payload.thinking`, or passing a reasoning payload into the Teams renderer.

## Solution Assessment

No. The current main behavior and the linked open PR are only partial because they do not jointly fix API thinking defaults, default-agent cron inheritance, and Teams reasoning suppression.
