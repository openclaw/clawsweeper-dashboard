## Summary

Keep open. Current main has partial mitigations, including global thinking/reasoning defaults and an Opus 4.7 off default, but the cost-affecting implicit API thinking path still resolves non-off for Claude 4.6 and generic reasoning-capable catalog models, cron still applies that implicit default when payload thinking is absent, and Teams still renders reasoning-marked payloads without the shared suppression guard.

## What This Changes

The issue requests opt-in/off reasoning API defaults for Claude and reasoning-capable models, safer cron inheritance and diagnostics, clearer upgrade notes, and suppression of reasoning blocks in chat delivery.

## Best Possible Solution

Define one explicit policy across runtime and automation: API thinking stays off unless the user configured it or the selected model truly requires it; cron agentTurn jobs inherit the same resolved agent/global thinking default as normal turns; channel delivery drops reasoning-marked payloads unless a channel deliberately implements a reasoning lane; docs, doctor/startup diagnostics, and release notes call out any billing-affecting default change.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: resolve a Claude 4.6 or reasoning-capable catalog model with no explicit thinking setting, run a cron agentTurn without payload.thinking, or pass a payload marked isReasoning through the Teams renderer.

## Solution Assessment

No. Current main is only a partial mitigation because it keeps non-off implicit API thinking defaults for relevant models and leaves Teams reasoning delivery unsuppressed; the safer path is a coordinated runtime, cron, channel, and docs policy fix.
