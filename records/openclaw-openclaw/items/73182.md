## Summary

Keep open. Current main still preserves non-off implicit API thinking defaults for reasoning-capable models, including Claude 4.6 adaptive defaults and catalog-backed `medium` fallback, cron still resolves omitted payload thinking through that same runtime default path, and Microsoft Teams delivery still lacks the reasoning-payload suppression guard used by generic/Slack/WhatsApp delivery paths.

## What This Changes

The issue asks OpenClaw to make Claude/API thinking opt-in by default, add global/default diagnostics, make cron inherit those defaults safely, and prevent reasoning blocks from rendering into messaging channels.

## Best Possible Solution

Define one explicit runtime policy: API thinking stays `off` unless explicitly configured or required by the selected provider/model, cron `agentTurn` jobs inherit the same resolved agent/global thinking policy as normal turns, and channel delivery suppresses reasoning-marked payloads unless that channel deliberately implements a separate reasoning lane. Pair that with docs, changelog, and startup/doctor diagnostics for billing-impacting defaults.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: resolve an unconfigured reasoning-capable catalog model or Claude 4.6 model, or run a cron `agentTurn` without payload `thinking`, and the code path resolves non-off thinking; Teams can render any reasoning-marked payload passed into its renderer because no guard is present.

## Solution Assessment

Unclear as a concrete patch because this issue spans billing policy, cron inheritance, docs/diagnostics, and channel delivery safety. The requested direction is maintainable if split into explicit API-thinking policy, cron-default propagation, and channel-level suppression with targeted tests.
