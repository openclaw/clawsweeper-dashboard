## Summary

Keep open: current main and the latest release only partially address the report. Claude Opus 4.7/4.8 now default thinking off and Telegram/Feishu previews honor configured reasoning defaults, but shipped source/docs still make Claude 4.6 default to adaptive thinking, generic reasoning-capable models can still fall back to a non-off thinking level, and cron agentTurn jobs without an explicit thinking field still inherit provider/model defaults.

## What This Changes

The issue reports a silent Claude reasoning/thinking default change that increased Anthropic cost and exposed thinking text, and requests opt-in defaults, global defaults, startup/doctor warnings, and separation of API thinking from rendered reasoning.

## Best Possible Solution

Maintainers should make an explicit default-policy call: either keep the documented provider-owned thinking defaults and close/narrow the complaint as intentional, or change implicit thinking to opt-in with focused auto-reply, cron, docs, and upgrade-warning coverage.

## Reproduction Assessment

Yes, at source level for the remaining behavior: current main and v2026.6.8 still resolve configured Anthropic Claude 4.6 to adaptive thinking and cron jobs with no payload.thinking still inherit provider/model defaults. The specific Opus 4.7 example is now fixed to off, so not every example in the original report still reproduces.

## Solution Assessment

No, this issue is not fully solved by the existing partial fixes. The best fix depends on a maintainer decision about whether implicit provider/model thinking defaults should remain documented behavior or become opt-in with warning/doctor coverage.
