## Summary

Keep open. Current main has partial mitigations, including a global reasoning visibility default and an Opus 4.7 off default, but the cost-affecting API thinking default is still non-off for Claude 4.6/reasoning-capable models, cron can still fall through to that resolver when no payload thinking is set, and the Teams renderer still lacks the reasoning-payload suppression guard used elsewhere.

## What This Changes

The issue requests cost-safe opt-in thinking defaults for Claude/Anthropic models, global default ergonomics, cron inheritance, startup/doctor warnings, and separation of API thinking from user-visible reasoning output.

## Best Possible Solution

Define one explicit runtime policy: API thinking should be off unless explicitly configured or truly required by the selected provider/model; cron agentTurn jobs should inherit the same resolved agent/global thinking default as normal turns; channel delivery should suppress reasoning-marked payloads unless a surface deliberately implements a reasoning lane; and billing-impacting default changes should be called out in release notes and diagnostics.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: resolve an unconfigured Claude 4.6/reasoning-capable model, run a cron agentTurn with no payload thinking, or pass a reasoning-marked payload through the Teams renderer.

## Solution Assessment

No. Current main is only a partial mitigation because it preserves non-off implicit API thinking defaults and does not fully close the cron inheritance or Teams delivery safety gaps.
