## Summary

Keep open. Current main still preserves non-off implicit API thinking defaults for reasoning-capable models and Claude 4.6, cron `agentTurn` jobs without explicit `thinking` can still inherit that resolver, and the Microsoft Teams renderer still lacks a hidden-reasoning suppression guard.

## What This Changes

Request opt-in API thinking/reasoning defaults for Claude and other reasoning-capable models, global/default audits, startup warnings, and hidden-reasoning suppression in messaging channels.

## Best Possible Solution

Adopt one explicit policy: API thinking stays off unless configured or provider-required, cron and normal turns share agent/global defaults, reasoning visibility stays independent, and every channel renderer suppresses hidden reasoning payloads.

## Reproduction Assessment

Yes. Current main gives a source-level reproduction by resolving an unconfigured reasoning-capable or Claude 4.6 model, running a cron `agentTurn` without `payload.thinking`, or passing `{ isReasoning: true }` through the Teams renderer.

## Solution Assessment

No. Current main is only a partial mitigation because global/default fields exist, but API thinking still defaults non-off for important model paths and Teams still has an unsuppressed reasoning delivery path.
