## Summary

Keep open. Current main still resolves non-explicit API thinking to a non-off level for Claude 4.6 and reasoning-capable fallbacks, cron agent-turn jobs without payload thinking still flow through that resolver, and the Microsoft Teams renderer still lacks the reasoning-payload suppression guard used by shared outbound paths.

## What This Changes

The issue asks OpenClaw to make Claude/API thinking opt-in by default, add global/default diagnostics, carry reasoning/thinking defaults into cron jobs, and prevent reasoning-marked payloads from appearing in chat surfaces.

## Best Possible Solution

Set one explicit runtime contract: API thinking is off unless a user/config explicitly enables it or a provider truly requires it; isolated cron jobs inherit the same resolved agent/global thinking default as normal turns; channel delivery suppresses reasoning-marked payloads unless that channel has a deliberate reasoning lane; and docs, changelog, status, and doctor/startup diagnostics warn about billing-impacting thinking defaults.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: resolve an unconfigured Claude 4.6 or reasoning-capable catalog model, run an isolated cron agentTurn without payload thinking, or pass a reasoning-marked payload through the Teams renderer.

## Solution Assessment

No. Current main is only a partial mitigation because it adds some defaults and an Opus 4.7 off special case, but leaves non-off implicit API thinking for Claude 4.6/reasoning-capable fallbacks and leaves the Teams rendering path unguarded.
