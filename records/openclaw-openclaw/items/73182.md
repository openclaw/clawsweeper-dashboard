## Summary

Keep open. Current main has partial mitigation, but it still preserves non-off implicit API thinking for Claude 4.6 and generic reasoning-capable models, cron jobs without explicit thinking still inherit that policy, Teams lacks hidden-reasoning suppression, and the open linked PR is only a channel-specific partial fix.

## What This Changes

The issue asks OpenClaw to make Claude/API thinking opt-in, add safer global/audit controls, and prevent hidden reasoning blocks from leaking into messaging channels.

## Best Possible Solution

Adopt one explicit policy: API thinking stays off unless configured or provider-required, cron follows the same policy or warns loudly, and all channel renderers suppress hidden reasoning unless visibility is explicitly enabled.

## Reproduction Assessment

Yes, source-level. An unconfigured reasoning-capable model can still reach a non-off thinking default, cron `agentTurn` jobs without `payload.thinking` use that resolver, and Teams renders sendable payloads without a reasoning guard.

## Solution Assessment

No. Current main is only a partial mitigation: global reasoning visibility defaults exist and Opus 4.7 defaults off, but the billing-impacting default policy, cron inheritance/audit behavior, and Teams hidden-reasoning delivery path remain unresolved.
