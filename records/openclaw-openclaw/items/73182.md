## Summary

Keep open. Current main includes partial mitigations, but the remaining billing and visibility concern is still source-reproducible for non-off implicit thinking defaults and the Teams rendering path still lacks hidden-reasoning suppression.

## What This Changes

The issue requests opt-in API thinking/reasoning defaults for Claude models, a global default/warning surface, and prevention of hidden reasoning blocks reaching chat channels.

## Best Possible Solution

Adopt an explicit policy: API thinking is off unless configured or provider-required, cron and normal turns share the same thinking defaults, and every channel renderer suppresses hidden reasoning payloads unless visibility is explicitly enabled.

## Reproduction Assessment

Yes, source-level. Current main still has non-off implicit thinking defaults for some Claude/reasoning-capable paths, cron jobs without payload.thinking inherit that resolver, and Teams can render reasoning-marked payloads.

## Solution Assessment

No. The merged fixes are useful partial mitigations, but the remaining issue should be split into an explicit default-thinking policy decision plus channel-level hidden-reasoning suppression rather than treated as already solved.
