## Summary

Keep open. Current main and latest release include partial fixes, but omitted thinking still falls through to provider/default non-off behavior and the Microsoft Teams renderer still lacks a local reasoning-payload suppression guard.

## What This Changes

The issue asks OpenClaw to make Claude/API thinking opt-in, avoid showing reasoning content in chat, and add safer defaults or warnings for agent and cron turns.

## Best Possible Solution

Adopt a maintainer-approved safe-default policy for API thinking, apply it consistently to normal and cron turns, and suppress reasoning-marked payloads in every channel renderer unless visibility is explicitly enabled.

## Reproduction Assessment

Yes, source-level. Current main and v2026.6.6 still route omitted thinking through provider/default non-off paths, and the Microsoft Teams renderer can render reasoning-marked payloads without a local suppression check.

## Solution Assessment

No, current main is only a partial solution. The best fix needs maintainer policy review for cost-safe defaults plus a channel-suppression repair, not closure as implemented.
