## Summary

Keep open. Current main partially documents and exposes thinking/reasoning defaults, but it still intentionally gives reasoning-capable models a non-off implicit thinking fallback, Anthropic Claude 4.6 still inherits adaptive thinking through the shared Claude profile, and Microsoft Teams block delivery still lacks the shared reasoning-payload suppression check.

## What This Changes

The issue asks OpenClaw to make API-level reasoning opt-in for Claude/reasoning-capable models, add safer global defaults and diagnostics, and prevent reasoning blocks from rendering into chat channels.

## Best Possible Solution

Adopt one explicit policy: API thinking stays off unless configured or provider-required, cron and normal turns share agent/global defaults, reasoning visibility stays independent from API thinking, and all channel renderers share hidden-reasoning suppression.

## Reproduction Assessment

Yes. A source-level reproduction exists on current main by resolving an unconfigured Anthropic Claude 4.6 or reasoning-capable catalog model, running a cron `agentTurn` without payload `thinking`, or passing an `isReasoning` payload through the Teams block renderer.

## Solution Assessment

No. Current main is a partial mitigation because it exposes defaults and special-cases some models, but it preserves non-off implicit API thinking and leaves the Teams block-rendering path unsuppressed.
