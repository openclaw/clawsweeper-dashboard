## Summary

Keep open for maintainer review: the central display fix is well scoped, but the patch adds an all-ineligible explicit-order fallback that can still show an expired credential as `active now`, which is the same kind of misleading auth status this PR is meant to repair.

## What This Changes

The PR changes Codex `/codex account` to prefer explicit OpenAI/Codex auth order over stale `lastGood`, with regression tests and a changelog entry.

## Best Possible Solution

Land the explicit-order display alignment after changing the no-eligible-candidate path to avoid marking unusable credentials active and keeping regression coverage for fresh, alias, cooldown, and all-ineligible cases.

## Reproduction Assessment

Yes. Source inspection shows current main checks `lastGood` before the runtime resolver fallback, and the PR’s tests model the reported store shape with explicit order plus stale `lastGood`.

## Solution Assessment

No, not quite. The explicit-order-first display fix is the right narrow direction, but the all-ineligible fallback should not mark a rejected credential as active.
