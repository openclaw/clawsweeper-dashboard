## Summary

Keep this PR open for maintainer review. Current main does not include the proposed force-IPv4 startup override, the patch is focused and has real Windows log proof, but it adds a new Telegram environment knob that needs maintainer product acceptance and focused validation rather than cleanup closure.

## What This Changes

The PR adds `OPENCLAW_TELEGRAM_FORCE_IPV4` to force Telegram dispatchers onto IPv4 from startup and documents the override in Telegram docs and the changelog.

## Best Possible Solution

If maintainers accept this operator escape hatch, land it with focused `extensions/telegram/src/fetch.test.ts` coverage and keep the broader Windows gateway degradation tracker open.

Source inspection does show current `main` starts Telegram transport with `forceIpv4: false` and only reaches IPv4-only transport through the sticky fallback path after eligible failures.

## Solution Assessment
