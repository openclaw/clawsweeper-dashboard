## Summary

Keep open because current main still has the same source-level lane collision described in the report, and the focused tests still encode foreground-turn priority over deferred maintenance rather than a transcript-correctness checkpoint before the next same-session inference.

## What This Changes

The issue reports that deferred context-engine turn maintenance can livelock under steady same-session chat traffic because it waits for the same session lane that foreground inference uses.

## Best Possible Solution

Define the transcript-correctness invariant for background turn maintenance, then move or gate only the correctness-critical commit before the next same-session inference while keeping heavier bookkeeping deferred.

## Reproduction Assessment

Yes, at source level with high confidence: current main waits for the same session lane used by foreground turns, and the focused test suite asserts that foreground turns can pass deferred maintenance. I did not run a live Telegram reproduction.

## Solution Assessment

Unclear. The reporter's inverted-gating proposal is plausible, but the existing no-stall behavior and tests were intentional, so maintainers should choose the transcript-correctness checkpoint before implementation.
