## Summary

Keep open. Current main still has the reported same-lane idle wait: deferred turn maintenance waits for the session lane to empty while user turns and background transcript rewrites use that same resolved lane, and existing tests intentionally let foreground turns outrun maintenance. The report is credible, but the fix needs maintainer direction because it may reverse the responsiveness guarantee added for #65220.

## What This Changes

The issue asks OpenClaw to prevent deferred context-engine turn maintenance from livelocking under rapid same-session chat traffic and leaving transcript-correctness work uncommitted.

## Best Possible Solution

Split correctness-critical transcript commits from deferrable bookkeeping, or gate each new same-session inference on the prior turn's maintenance checkpoint while leaving expensive indexing/compaction work in the background.

## Reproduction Assessment

Yes, source-reproducible. A continuous stream of same-session turns can keep `getQueueSize(resolveSessionLane(sessionKey))` above zero, and current tests already prove foreground turns and rewrites are allowed to run ahead of deferred maintenance.

## Solution Assessment

Unclear. The issue's Option B is a plausible short-term fix, but it conflicts with current tests and the #65220 design goal, so maintainers should first choose whether to invert gating or split canonical transcript commits from background maintenance.
