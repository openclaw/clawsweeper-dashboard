## Summary

Keep open. Current main still has the source-level same-session lane collision described in the report, and the existing tests intentionally preserve foreground-turn priority over deferred maintenance, so this is not cleanup-closeable without a maintainer decision on the transcript-correctness invariant.

## What This Changes

The issue asks OpenClaw to prevent deferred context-engine turn maintenance from livelocking behind steady same-session chat traffic and letting later inference read pre-maintenance transcript state.

## Best Possible Solution

Define the transcript-correctness invariant for background context-engine maintenance, then update the worker and tests so a same-session user turn cannot observe pre-maintenance transcript state while heavy bookkeeping can still remain deferred.

## Reproduction Assessment

Yes at source level with high confidence: the current worker waits on the same session lane used by embedded foreground turns, and focused tests assert foreground turns can pass maintenance. I did not establish an independent live Telegram reproduction.

## Solution Assessment

Unclear. The reporter’s inverted-gating option is a plausible near-term mitigation, but current tests and the original no-stall PR intentionally prefer foreground turns, so maintainers should choose the invariant before implementation.
