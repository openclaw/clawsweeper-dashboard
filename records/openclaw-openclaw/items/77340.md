## Summary

Keep open. Current main still has the same source-level idle-wait on the foreground session lane, and the current tests intentionally preserve foreground-turn priority over deferred maintenance; recent same-session coalescing reduces the pile-up-only failure mode from https://github.com/openclaw/openclaw/issues/82779 but does not prove the transcript-correctness livelock is fixed.

## What This Changes

The issue reports that background context-engine turn maintenance waits for the same session lane used by rapid foreground turns, letting same-session inference read pre-maintenance transcript state and accumulate stale trailing assistant messages.

## Best Possible Solution

Define the transcript-correctness invariant first, then change deferred turn maintenance so the canonical transcript commit cannot be observed as pending by the next same-session inference while heavier indexing or compaction can remain background work.

## Reproduction Assessment

Yes, at source level with high confidence: current main waits for the same resolved session lane used by foreground turns, and focused tests assert foreground turns and rewrites can run before deferred maintenance. I did not run the live Telegram reproduction in this read-only pass.

## Solution Assessment

Unclear. The reporter's inverted-gating or inline-commit direction is plausible, but current main intentionally preserves foreground-turn no-stall behavior and now has per-session coalescing, so maintainers should choose the transcript ordering invariant before implementation.
