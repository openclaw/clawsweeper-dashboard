## Summary

Keep open: current main still has the same-lane deferred-maintenance wait and tests explicitly allow foreground turns to pass before maintenance, so this is a plausible core lifecycle bug rather than a closeable stale, duplicate, or plugin-scope item.

## What This Changes

The issue requests fixing deferred context-engine turn maintenance so same-session chat traffic cannot indefinitely postpone transcript rewrites and accumulate trailing assistant turns.

## Best Possible Solution

Change context-engine turn maintenance so transcript-correctness rewrites cannot be indefinitely postponed by same-session inbound turns, with regression coverage for a continuously busy stable session key.

## Reproduction Assessment

No, not as an end-to-end live reproduction in this read-only pass. The source path is clear: current main waits for the same resolved session lane used by inference, and tests show later foreground work can run before maintenance.

## Solution Assessment

Unclear. The report's Option B is a plausible narrow mitigation, but changing the priority between foreground turns, maintenance commits, and deferred rewrites is core lifecycle behavior that maintainers should choose before a repair PR.
