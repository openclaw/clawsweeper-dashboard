## Summary

Keep open: current main still preserves fresh live-PID session locks during doctor repair, and the open closing PR #76854 already tracks the narrow false-live owner-check fix for this report.

## What This Changes

The issue asks for `openclaw doctor --fix` to clean session locks left behind when the gateway is offline even if the recorded PID is alive.

## Best Possible Solution

Review and land or replace #76854 with a conservative owner-check fix that removes fresh live-PID locks only when the owner is readable and clearly not OpenClaw, while preserving active or unknown OpenClaw locks.

## Reproduction Assessment

Yes. Source inspection shows a recent `.jsonl.lock` with an alive PID gets no stale reason and is not removed by `cleanStaleLockFiles({ removeStale: true })`; the current tests also assert a fresh live lock remains.

## Solution Assessment

No for the literal suggested gateway-down force cleanup, because it is too broad. The best path is the narrower owner-verification approach already proposed by #76854.
