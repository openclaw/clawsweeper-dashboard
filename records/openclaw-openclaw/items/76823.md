## Summary

Keep open: current main still leaves fresh live-PID session locks untouched during doctor repair, and the existing shipped fixes cover dead, old, recycled, or self-owned locks rather than the gateway-down false-live PID case reported here.

## What This Changes

Request `openclaw doctor --fix` to clean session `.jsonl.lock` files automatically when the gateway is down, even when the recorded PID is alive.

## Best Possible Solution

Add a conservative session-lock ownership check that can clear false-live orphan locks when the gateway is offline, while preserving locks held by active embedded or local OpenClaw session writers.

## Reproduction Assessment

Yes, from source inspection: create a recent `.jsonl.lock` payload with an alive PID and `doctor --fix` will report it but not remove it because `stale` stays false. I did not run a mutating live doctor command in this read-only review.

## Solution Assessment

No for the literal suggested fix: removing all live-PID locks whenever the gateway is down is too broad. The maintainable fix should prove the lock owner is not an active OpenClaw writer, using gateway status plus process or lock-owner metadata where available.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/76823. Current `openclaw doctor --fix` only removes locks classified stale by `cleanStaleLockFiles`; a fresh `.jsonl.lock` whose PID is alive but not the gateway/session writer remains and can block recovery when the gateway is down. Implement a conservative repair path in the doctor/session-lock cleanup area that can identify and remove false-live orphan locks when the gateway is offline or the recorded owner is provably not an active OpenClaw session writer. Do not add a config option, do not blindly delete all live-PID locks just because the gateway is unreachable, and preserve active embedded CLI fallback/local agent locks. Add focused regression coverage for the gateway-down false-live case and active-writer preservation, update user-facing doctor docs and CHANGELOG if behavior changes.
