## Summary

Keep open: current main still preserves fresh live-PID session locks during doctor repair, and the open closing PR #76854 already tracks the narrow false-live owner-check fix for this report.

## What This Changes

The issue asks for `openclaw doctor --fix` to clean false-live `.jsonl.lock` files that block sessions after Gateway exits.

## Best Possible Solution

Review and land or replace #76854 with a conservative owner-check fix that removes false-live non-OpenClaw locks while preserving active or unreadable OpenClaw locks.

## Reproduction Assessment

Yes. Source inspection of current main shows a recent `.jsonl.lock` with an alive PID gets no stale reason and is not removed by `cleanStaleLockFiles({ removeStale: true })`; the existing test also asserts a fresh live lock remains.

## Solution Assessment

No for the literal gateway-down delete-all suggestion, because that is broader than the safety contract. The narrow owner-verification approach in #76854 is the more maintainable fix path.
