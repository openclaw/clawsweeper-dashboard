## Summary

Keep open: current main still preserves fresh live-PID session locks during doctor repair, and open PR #76854 already tracks a narrow owner-check fix for the reported false-live lock case.

## What This Changes

The issue requests `openclaw doctor --fix` to clean blocking session lock files when the gateway is down even if the recorded PID is still alive.

## Best Possible Solution

Review and land or replace #76854 with a conservative owner-check fix that removes fresh live-PID locks only when the owner is readable and clearly not OpenClaw.

## Reproduction Assessment

Yes. Source inspection shows a recent `.jsonl.lock` with an alive PID gets no stale reason, `cleanStaleLockFiles({ removeStale: true })` removes only stale locks, and current tests assert a fresh live lock remains.

## Solution Assessment

No for the literal gateway-down delete-all rule because it is too broad. The best path is the narrower false-owner verification approach already proposed by #76854.
