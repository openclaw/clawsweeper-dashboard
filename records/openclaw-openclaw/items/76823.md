## Summary

Keep open: current main still removes only stale session locks, and the open closing PR #76854 already tracks the narrow false-live PID owner-check fix for this report.

## What This Changes

The issue asks `openclaw doctor --fix` to clean blocking session lock files when the gateway is down even if the recorded PID is alive.

## Best Possible Solution

Review and land or replace #76854 with a conservative owner-check fix that removes fresh live-PID locks only when the owner is readable and clearly not OpenClaw.

## Reproduction Assessment

Yes. Source inspection shows a recent `.jsonl.lock` with a live PID gets no stale reason and is not removed by `cleanStaleLockFiles({ removeStale: true })`; the current tests also assert a fresh live lock remains.

## Solution Assessment

No for the literal gateway-down delete-all request because it is too broad. The narrower owner-verification approach in #76854 is the safer maintainable fix.
