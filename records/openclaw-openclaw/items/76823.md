## Summary

Current main still keeps fresh live-PID session locks during `doctor --fix`, and the open closing PR #76854 is already the right implementation vehicle for this report.

## What This Changes

The issue asks for `openclaw doctor --fix` to remove session lock files when the gateway is down even if the recorded PID is still alive.

## Best Possible Solution

Review and land or replace #76854 with a conservative owner-check fix that removes fresh live-PID locks only when the owner is readable and clearly not OpenClaw.

## Reproduction Assessment

Yes. Source inspection shows a recent `.jsonl.lock` with an alive PID gets no stale reason and is not removed by `cleanStaleLockFiles({ removeStale: true })`, and current tests assert fresh live locks remain.

## Solution Assessment

No for the literal gateway-down force cleanup, because it is too broad for active embedded/local writers. The narrower owner-verification direction in #76854 is the safer maintainable fix.
