## Summary

Keep open: current main still preserves fresh live-PID session locks during doctor repair, and the linked open PR #76854 already tracks the narrow false-live owner-check fix.

## What This Changes

The issue requests that `openclaw doctor --fix` clean session lock files when the gateway is down even if the recorded PID is alive.

## Best Possible Solution

Review and land or replace #76854 with a conservative owner-check fix that removes fresh live-PID locks only when the owner is readable and clearly not OpenClaw.

## Reproduction Assessment

Yes. Source inspection shows a recent `.jsonl.lock` with an alive PID gets no stale reason and is not removed by `cleanStaleLockFiles({ removeStale: true })`; current tests also assert a fresh live lock remains.

## Solution Assessment

No for the literal gateway-down delete-all proposal because it is too broad. The narrow owner-verification approach in #76854 is the safer maintainable fix boundary.
