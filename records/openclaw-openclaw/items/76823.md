## Summary

Keep open: current main still preserves fresh live-PID session locks during doctor repair, and the open closing PR #76854 already tracks the narrow false-live owner-check fix for this report.

## What This Changes

Make `openclaw doctor --fix` clean blocking session lock files when the gateway is down even if the lock records a still-alive PID.

## Best Possible Solution

Review and land or replace #76854 with a conservative owner-check fix that removes proven non-OpenClaw false-live locks while preserving active or unreadable OpenClaw owners.

## Reproduction Assessment

Yes. Source inspection shows a recent `.jsonl.lock` with an alive PID gets no stale reason and is not removed by `cleanStaleLockFiles({ removeStale: true })`; checked-in tests also assert fresh live locks remain.

## Solution Assessment

No for the literal gateway-down delete-all suggestion, because that is broader than the lock safety model. The narrower owner-verification approach in #76854 is the maintainable fix direction.
