## Summary

Keep open: current main still preserves fresh live-PID session locks during doctor repair, and the open closing PR #76854 already tracks a narrower false-live owner-check fix for this report.

## What This Changes

The issue asks for `openclaw doctor --fix` to clean gateway-left session lock files even when the recorded PID is alive but belongs to another process.

## Best Possible Solution

Review and land or replace #76854 with a conservative owner-check repair that removes fresh live-PID locks only when ownership is readable and clearly non-OpenClaw.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a fresh `.jsonl.lock` with a live non-OpenClaw PID, then `doctor --fix` will preserve it because current cleanup only removes locks already classified as stale.

## Solution Assessment

No for the literal gateway-down delete-all rule; it is too broad. The narrower owner-verification approach in #76854 is the maintainable fix because it preserves active or unknown OpenClaw owners.
