## Summary

Keep open: current main still preserves fresh live-PID session locks during doctor repair, and open closing PR #76854 already tracks the narrow false-live owner-check fix.

## What This Changes

The issue asks `openclaw doctor --fix` to remove blocking `.jsonl.lock` session locks when the Gateway is down even if the recorded PID is alive.

## Best Possible Solution

Review and land or replace #76854 with conservative owner verification that removes fresh live-PID locks only when the owner is readable and clearly not OpenClaw.

## Reproduction Assessment

Yes. Source inspection shows a recent `.jsonl.lock` with an alive PID gets no stale reason and is not removed by `cleanStaleLockFiles({ removeStale: true })`; current tests also assert a fresh live lock remains.

## Solution Assessment

No for the literal Gateway-down force cleanup, because that is broader than necessary and can race active non-Gateway writers. The owner-verification approach in #76854 is the narrower maintainable direction.
