## Summary

Keep open: current main still leaves fresh live-PID session locks untouched during doctor repair, and the linked open PR #76854 is the right implementation candidate rather than closing this issue early.

## What This Changes

The issue asks `openclaw doctor --fix` to remove blocking session lock files when the gateway is down even if the recorded PID is alive.

## Best Possible Solution

Land or replace #76854 with a conservative false-live owner-check that removes locks owned by readable non-OpenClaw processes while preserving active or unknown OpenClaw owners.

## Reproduction Assessment

Yes. Source inspection of current main shows a fresh live-PID lock gets no stale reason and is not removed; current tests also assert that a fresh live lock remains during repair cleanup.

## Solution Assessment

No for the literal gateway-down delete-all suggestion, because that is broader than the safety boundary. The narrower owner-verification approach in #76854 is the maintainable fix path.
