## Summary

Keep open. Current main still has adjacent sessions, agents, workspaces, skill allowlists, sandbox/tool policy, and workspace skill precedence, but it does not implement the requested first-class Project object, Projects dashboard surface, or Gateway project protocol/API.

## What This Changes

Request a first-class Project concept in the Gateway dashboard with project/session/workspace binding, scoped skills/tools, and runtime constraint metadata.

## Best Possible Solution

Keep this as the canonical core/product request until OpenClaw has an explicit Project model, additive Gateway projects.* protocol, Control UI Projects list/detail/switching surface, project-to-home-session and workstream bindings, workspace/doc links, and clearly labeled project-scoped constraint display or enforcement.
