## Summary

Keep open. Current main still lacks a first-class Project model, Projects dashboard surface, Gateway `projects.*` API, and project-to-session/workspace/constraint bindings; the related closed issues point to this issue as the canonical tracker for the remaining work.

## What This Changes

The issue requests first-class Projects in the Gateway Dashboard with durable project-session/workspace bindings and project-scoped skill, tool, and runtime constraints.

## Best Possible Solution

Keep this as the canonical core/product request until OpenClaw has an explicit Project domain model, additive Gateway `projects.*` protocol, Control UI project list/detail/switching surface, project-session/workstream bindings, workspace/doc links, and clearly labeled project constraint semantics that reuse existing agents, sessions, workspaces, skills, and sandbox policy where appropriate.
