## Summary

Keep open. Current main still has only adjacent agents, sessions, workspaces, skill allowlists, and sandbox/tool policy; it does not implement the requested first-class Project model, Projects dashboard surface, or Gateway project API.

## What This Changes

The issue requests a first-class Project concept in the Gateway dashboard with project lists, home-session and workstream bindings, workspace/doc links, skill/tool scope, and runtime constraint metadata.

## Best Possible Solution

Keep this as the canonical core/product request until OpenClaw has an explicit Project domain model, additive Gateway project protocol, Control UI project list/detail/switching, project-to-session/workstream bindings, workspace/doc links, and clearly labeled constraint semantics that reuse existing agents, sessions, workspaces, skills, and sandbox policy where appropriate.
