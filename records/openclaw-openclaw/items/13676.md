Keep open. Current main still lacks the requested first-class Project model, Projects dashboard surface, and Gateway project API; the issue discussion and related closures treat this as the canonical core/product tracker rather than a duplicate of narrower session/project requests.

## What This Changes

The issue requests first-class Projects in the Gateway Dashboard with project/session/workspace bindings, scoped tools/skills, and runtime constraints.

## Best Possible Solution

Add an explicit Project domain model with additive Gateway project protocol/storage and a Control UI project list/detail/switching surface, reusing existing agents, sessions, workspaces, skill filters, and sandbox policies where appropriate.
