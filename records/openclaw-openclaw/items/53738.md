## What This Changes

The PR exports the unknown-model helper, removes the registry-only media resolver, and switches image and PDF media model lookup to the config-aware resolver with focused tests.

## Best Possible Solution

Choose one canonical PDF resolver path that honors configured provider overrides for registry-backed models and config-defined custom providers, preserves text-extraction fallback for resolved text-only models, and leaves the already-fixed image path on main alone.

## Reproduction Assessment

Yes, source-reproducible. Current main’s PDF execution path still uses a cfg-less registry helper before passing `model.baseUrl` into native PDF providers or `complete()`, while the existing resolver contract supports `models.providers` overrides elsewhere.

## Solution Assessment

No, not as-is. The config-aware resolver direction is valid, but this branch needs a rebase/reduction to the remaining PDF behavior and must fix the plugin dynamic text-only fallback regression before merge.
