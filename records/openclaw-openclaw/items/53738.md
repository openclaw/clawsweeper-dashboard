## Summary

Keep open: this is a plausible bug-fix PR and current main still leaves the PDF execution path on a cfg-less registry resolver, but the branch is conflicting, the image half is already covered on main, the PDF patch has a correctness issue, and after-fix real behavior proof is missing.

## What This Changes

The branch exports the unknown-model error helper, removes the registry-only media resolver, switches image and PDF model lookup toward `resolveModelWithRegistry`, and adds image/PDF regression tests around configured providers and text-only fallback behavior.

## Best Possible Solution

Land a refreshed PDF-only resolver path that applies configured provider overrides to registry-backed/native PDF models and configured custom models, preserves the current main image behavior, keeps text-only extraction fallback for resolved text models, and includes focused PDF tests plus redacted real behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main shows the PDF tool still resolves through a cfg-less registry helper before using `model.baseUrl`, and the linked runtime report demonstrates the same family of PDF custom-provider failure.

## Solution Assessment

No, not as-is. The direction is valid, but the branch needs a rebase, should drop the already-implemented image churn, and should avoid rejecting resolved plugin/dynamic text-only PDF models before the text-extraction fallback.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Allow resolved PDF text models through fallback:** `src/agents/tools/pdf-tool.ts:239-240`
  - body: `resolveModelWithRegistry` can return a plugin dynamic text-only model that is neither declared in `models.providers` nor found by `modelRegistry.find`. This guard then throws before the existing extracted-text fallback, so PDFs with images plus extractable text can fail instead of returning a text-only summary. Treat the resolved model's actual input/source as sufficient while still rejecting synthesized fallback models.
  - confidence: 0.84
