## Summary

Keep open: the PR still points at a valid PDF model-resolution gap on current main, but it is not mergeable as-is because the image half is already covered, the remaining PDF work overlaps https://github.com/openclaw/openclaw/pull/65746, the branch is conflicting, real behavior proof is missing, and the current diff can regress plugin dynamic text-only PDF fallback.

## What This Changes

The branch exports `buildUnknownModelError`, removes the registry-only media resolver, routes image and PDF model resolution through `resolveModelWithRegistry`, and adds image/PDF regression tests for configured provider handling.

## Best Possible Solution

Keep the current image implementation, choose one PDF execution resolver boundary that honors configured provider metadata, preserve text-extraction fallback for real text-only models, and land focused PDF regression coverage plus real behavior proof.

## Reproduction Assessment

Yes, by source inspection rather than a live run. ## Solution Assessment

No, not as-is. The resolver direction is valid for the remaining PDF bug, but this branch is conflicting, partly superseded by main, overlaps a narrower open PDF fix, lacks real proof, and needs the plugin dynamic text-only fallback regression fixed before merge.
