## Summary

Keep open: the remaining PDF behavior is still valid on current main, but this branch should not merge as-is because it is dirty against main and still has a PDF text-only fallback regression.

## What This Changes

The PR exports the unknown-model error helper, removes the registry-only media resolver, switches image and PDF model lookup to `resolveModelWithRegistry`, and adds image/PDF regression tests.

## Best Possible Solution

Land a refreshed PDF-focused resolver that applies provider overrides for registry-backed and configured PDF models, preserves text-only extraction fallback for real resolved models, and includes focused tests plus the required changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows the PDF tool still calls the cfg-less registry helper, and the PR guard regression is source-reproducible with a plugin-dynamic text-only PDF model that has extractable text and images.

## Solution Assessment

No. The direction is valid, but this branch is not the best landing shape because current main already implements the image side, the PR is dirty against main, and the PDF guard rejects a valid resolved-model fallback case.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Allow resolved PDF text models through fallback:** `src/agents/tools/pdf-tool.ts:239-240`

Treat the resolved model's actual input/source as sufficient while still rejecting synthesized fallback models.
