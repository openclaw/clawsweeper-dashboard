## Summary

Keep open. Current main still has a PDF-side provider-override gap, but this PR cannot merge as-is because the image side has moved on main, the PDF change has a blocking text-only fallback regression, the branch is conflicting, and contributor real behavior proof is still missing.

## What This Changes

The PR exports the unknown-model error helper, removes the shared registry-only media resolver, switches image/PDF model resolution toward `resolveModelWithRegistry`, and adds PDF/image regression tests.

## Best Possible Solution

Keep current main's image resolver, land one canonical PDF resolver that honors `models.providers` overrides for registry-backed and config-defined models, preserves PDF text extraction fallback, and has focused PDF tests plus real behavior proof.

## Reproduction Assessment

Yes, source-reproducible. Current main's PDF path calls the cfg-less registry helper before using `model.baseUrl`, while the config contract defines `models.providers.*.baseUrl`, `api`, and headers as provider configuration.

## Solution Assessment

No, not as-is. The resolver direction is valid, but this branch should be rebased/reduced or replaced by a canonical PDF fix that avoids the plugin dynamic text-only regression and includes contributor real behavior proof.
