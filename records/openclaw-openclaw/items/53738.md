## Summary

Keep open: this PR targets a real media-tool provider-override bug, but it is not mergeable as-is because current main already covers the image side, the PDF side still needs a canonical resolver boundary, the branch is conflicting, real behavior proof is missing for this PR, and the diff still has a PDF fallback regression.

## What This Changes

The PR exports the unknown-model helper, removes the registry-only media resolver, switches image and PDF model resolution to `resolveModelWithRegistry`, and adds image/PDF regression tests.

## Best Possible Solution

Keep the current image implementation and land one refreshed PDF resolver path that honors `models.providers` overrides for registry-backed, configured, and plugin-resolved models while preserving PDF text-extraction fallback behavior.

## Reproduction Assessment

Yes, by source inspection rather than a live run. Current main's PDF execution path still uses a cfg-less registry resolver while the config contract defines provider `baseUrl` and related overrides; the image side no longer reproduces because main is already config-aware.

## Solution Assessment

No, not as-is. The resolver direction is valid for the remaining PDF bug, but this branch is partly superseded by main, conflicts with current main, lacks real proof, overlaps a narrower open PDF fix, and needs the plugin dynamic text-only fallback regression fixed.
