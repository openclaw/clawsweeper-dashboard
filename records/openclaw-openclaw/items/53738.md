## Summary

Keep open: current main still has the PDF-side provider override gap, and the PR is not merge-ready because its PDF guard introduces a fallback regression while the branch is currently dirty against main. The image-side part is already covered on main, but the remaining PDF behavior still needs one canonical maintainer-approved fix.

## What This Changes

The PR exports the unknown-model error helper, removes the registry-only media resolver, switches image and PDF model lookup toward `resolveModelWithRegistry`, and adds PDF/image regression tests around configured provider behavior.

## Best Possible Solution

Land one canonical PDF resolver path that honors configured provider overrides for registry-backed built-ins, config-defined custom providers, and plugin-dynamic models while preserving native PDF handling and the documented extracted-text fallback.

## Reproduction Assessment

Yes. The PDF bug is reproducible from the #65742 config steps and current source inspection: `pdf` still uses the registry-only resolver, while the main resolver path applies `models.providers` overrides. The PR finding is reproducible with a plugin-dynamic text-only model plus a PDF containing both rendered images and extractable text.

## Solution Assessment

No. The config-aware resolver direction is valid, but this branch is dirty and the new PDF guard is not the narrowest maintainable fix because it rejects plugin-dynamic resolved models instead of using the resolved model capability/source to preserve text fallback.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Allow plugin-resolved PDF models through text fallback:** `src/agents/tools/pdf-tool.ts:239-240`
  - body: `resolveModelWithRegistry` can return a plugin-dynamic text-only model that is neither declared in `models.providers` nor found by `modelRegistry.find`. This guard then throws before the existing extracted-text fallback, so PDFs with extractable text plus page images can fail instead of returning a text-only summary. Use the resolved model capability/source rather than only config or registry membership.
