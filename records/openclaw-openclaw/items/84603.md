## Summary

Keep open: current main and v2026.6.8 still contain the unsafe routed-peer cron transcript mirror path, so the PR is still relevant to https://github.com/openclaw/openclaw/issues/84583, but this branch is conflicted and not merge-ready because it drops destination-session context without target-session awareness or live proof.

## What This Changes

The PR changes cron direct delivery so resolved routed non-main sessions skip transcript mirroring while preserving direct outbound send and main-session awareness behavior.

## Best Possible Solution

Land a refreshed combined fix that skips unsafe routed-peer transcript writes, preserves target-session awareness for the delivered content, and includes redacted live Telegram or equivalent cron-plus-active-session proof.

## Reproduction Assessment

Yes at source level. Current main and v2026.6.8 still resolve routed delivery sessions and append the cron transcript mirror, and the linked issue contains live reports of the same takeover signature, although I did not run a live Telegram repro in this review.

## Solution Assessment

No. Skipping unsafe routed peer transcript writes is the right direction, but the best fix must also preserve the target session's next-turn awareness or it trades the lock race for a context-loss regression.
