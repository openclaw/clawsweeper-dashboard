## Summary

## What This Changes

This branch adds a `sessionKey`/`session_key` field to Claude permission requests, stores it with pending permission entries, gates `yes/no` replies by matching session, and adds MCP channel tests.

## Best Possible Solution

Preserve the existing four-field permission-request compatibility surface, bind pending approvals to trusted OpenClaw session context or an explicitly approved protocol extension, and prove the fresh-install and upgrade paths with tests plus redacted real MCP evidence.

No. Binding permission replies to the originating session is the right security direction, but making `session_key` a required client field is not the safest or most compatible implementation path.
