## Summary

Close because the failure is caused by the external `chrome-openclaw-sider` npm package being published without the compiled runtime artifacts OpenClaw requires; current and shipped OpenClaw behavior correctly rejects that package shape.

## What This Changes

The issue requests that installing the third-party `chrome-openclaw-sider` plugin succeed instead of failing on missing compiled runtime files and hook-pack metadata.

## Best Possible Solution

The `chrome-openclaw-sider` publisher should republish the package with built JavaScript runtime files and matching runtime metadata; OpenClaw should keep enforcing the installed-plugin package contract.

## Reproduction Assessment

Yes. The published npm package metadata and tarball show TypeScript-only entries with no built runtime output, and OpenClaw’s installer source shows that this package shape is rejected by design.

## Solution Assessment

Yes. The narrow maintainable solution is outside this repository: publish compiled JS and runtime entry metadata for the Sider plugin package, rather than weakening OpenClaw’s installed-package checks.
