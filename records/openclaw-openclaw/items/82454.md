## Summary

This is an external package publication problem, not an OpenClaw source bug: the latest `chrome-openclaw-sider` npm package still publishes TypeScript entries without the built runtime files or runtime metadata that OpenClaw requires, and current/shipped OpenClaw correctly rejects that package shape.

## What This Changes

The issue reports that installing the external `chrome-openclaw-sider` plugin fails because its npm package lacks compiled JavaScript runtime files and hook-pack metadata.

## Best Possible Solution

The `chrome-openclaw-sider` publisher should republish the package with built JavaScript runtime files and matching `openclaw.runtimeExtensions` / `openclaw.runtimeSetupEntry` metadata; OpenClaw should keep enforcing the installed-package contract.

## Reproduction Assessment

Yes. The current npm metadata and tarball for the latest `chrome-openclaw-sider` release show TypeScript-only entries with no built runtime artifacts, and current OpenClaw source rejects that package shape by design.

## Solution Assessment

Yes. The narrow maintainable fix belongs outside this repository: publish the Sider package with compiled runtime output and runtime metadata rather than weakening OpenClaw's package validation.
