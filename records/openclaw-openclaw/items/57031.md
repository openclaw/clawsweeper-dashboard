## Summary

Keep open. Current `main` and the latest release still swallow queued writer mkdir/append failures, and an open cross-referenced PR already proposes the warning path; because the impact is now framed as low diagnostic visibility and the queue is intentionally lossy for optional logs, maintainers should decide the warning policy before closing this issue.

## What This Changes

The issue requests an operator-visible warning when `QueuedFileWriter` has repeated diagnostic JSONL write failures.

## Best Possible Solution

Land a narrow target-repo fix that preserves non-fatal diagnostic writes while adding bounded, non-secret warning visibility for repeated failures, or explicitly close the candidate PR if maintainers decide silent lossy diagnostics should remain intentional.
