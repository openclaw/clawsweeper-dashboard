## Summary

Keep open. Current main and the latest shipped tag still stage the backup manifest under ambient `os.tmpdir()`, while macOS service startup can set `TMPDIR` inside the archived state directory; the focused repair PR #75031 was closed without merging.

## What This Changes

Change backup creation so a state-local `TMPDIR` cannot cause `manifest.json` to be archived twice.

## Best Possible Solution

Backup creation should stage `manifest.json` in a temp root proven outside every selected backup source, keep the verifier’s exactly-one-root-manifest invariant, and add regression coverage for state-local `TMPDIR`.

## Reproduction Assessment

Yes, source-level. With `TMPDIR=<stateDir>/tmp`, `createBackupArchive()` stages `manifest.json` under the state source and then passes both that manifest and the state source to tar; I did not run a live archive because this sweep is read-only.

## Solution Assessment

Yes. Reusing `resolvePreferredOpenClawTmpDir()` or an equivalent source-outside temp-root helper is the narrow maintainable fix; the verifier should stay strict rather than accepting duplicate root manifests.

## Repair Work Prompt

Fix issue 75007: `openclaw backup create` can archive two root `manifest.json` entries when `TMPDIR` points inside the OpenClaw state directory, such as the macOS LaunchAgent `~/.openclaw/tmp` setup. In `src/infra/backup-create.ts`, stage the manifest/temp working directory under `resolvePreferredOpenClawTmpDir()` or an equivalent helper that proves the temp root is outside every selected backup source. Preserve the existing output temp archive and no-overwrite publish behavior. Do not add CLI flags, config options, or relax `backup verify` duplicate-manifest checks. Add a focused regression in `src/infra/backup-create.test.ts` that sets `TMPDIR` to `<stateDir>/tmp`, creates a normal state backup, asserts exactly one `<archiveRoot>/manifest.json` entry, and verifies the archive. Add a user-facing `CHANGELOG.md` fix entry.
