## Summary

Keep open. Current main still stages the backup manifest under ambient `os.tmpdir()`, while the normal backup plan includes the state directory and the macOS service environment intentionally sets `TMPDIR` under that state tree. The focused related fix PR #75031 was closed unmerged, so this remains a valid narrow repair.

## What This Changes

The issue requests that `openclaw backup create` avoid writing its temporary manifest inside the backed-up state tree when the macOS LaunchAgent sets `TMPDIR` to `~/.openclaw/tmp`.

## Best Possible Solution

Backup creation should stage transient manifest files in a trusted OpenClaw temp root outside every planned backup source, preserve the verifier invariant of exactly one root manifest, and add regression coverage for the LaunchAgent `TMPDIR`-under-state case.

## Reproduction Assessment

Yes. The high-confidence reproduction path is source-level: run backup creation with `TMPDIR` resolving under the state directory, then the staged manifest can be included both as the explicit manifest tar entry and through the recursive state source.

## Solution Assessment

Yes. Reusing `resolvePreferredOpenClawTmpDir()` or an equivalent containment-checked OpenClaw temp root is the narrowest maintainable fix; changing the macOS service `TMPDIR` policy or weakening verifier behavior would be broader and would not address backup self-inclusion.
