## Summary

Keep open. Current `main` still stages the backup manifest under ambient `os.tmpdir()`, the macOS service environment still points `TMPDIR` inside the OpenClaw state directory, and the focused repair PR was closed without merging.

## What This Changes

The issue asks `openclaw backup create` to avoid packaging `manifest.json` twice when service `TMPDIR` resolves inside `~/.openclaw/tmp`.

## Best Possible Solution

Backup creation should stage its manifest in an OpenClaw-managed temp root outside every selected backup source while preserving the verifier’s exactly-one-root-manifest invariant.

## Reproduction Assessment

Yes. The high-confidence reproduction path is source-level: run state/full backup creation with `TMPDIR=<stateDir>/tmp`, where the staged manifest is under the recursive state source and also passed explicitly to tar.

## Solution Assessment

Yes. Reusing `resolvePreferredOpenClawTmpDir()` or an equivalent source-containment-checked temp root in backup creation is the narrowest maintainable fix; service-wide `TMPDIR` and backup verification should not be loosened.
