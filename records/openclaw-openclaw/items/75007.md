## Summary

Keep open. Current main still stages the backup manifest under ambient `os.tmpdir()`, while macOS service environments set `TMPDIR` under the state directory; the focused repair PR #75031 was closed without merging, so this remains a narrow actionable bug.

## What This Changes

Fix `openclaw backup create` so its transient manifest staging cannot live inside the backed-up state tree when macOS LaunchAgent `TMPDIR` points at `~/.openclaw/tmp`.

## Best Possible Solution

Stage backup transient files under `resolvePreferredOpenClawTmpDir()` or an equivalent trusted temp root outside all planned backup sources, and add a regression asserting one root manifest when `TMPDIR` is inside the state directory.

## Reproduction Assessment

Yes. The source-level path trace is high-confidence: set `TMPDIR` to `<stateDir>/tmp`, run backup creation, and the staged manifest can be archived once explicitly and once through the recursive state source.

## Solution Assessment

Yes. Reusing the existing preferred OpenClaw temp-root helper in `createBackupArchive()` is the narrowest maintainable fix; changing launchd TMPDIR policy or relaxing verification would be broader and would not address backup self-inclusion.
