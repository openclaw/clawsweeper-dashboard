## Summary

Keep open. Current main still stages the backup manifest under ambient `os.tmpdir()`, so when `TMPDIR` resolves inside the OpenClaw state directory the tar input path can include the same manifest both explicitly and through the recursive state payload. The focused linked fix PR #75031 is closed without merging, so the issue remains actionable.

## What This Changes

`openclaw backup create` should stage its temporary manifest outside the backed-up state payload when `TMPDIR` points under `~/.openclaw/tmp`, avoiding duplicate root `manifest.json` entries.

## Best Possible Solution

Stage the backup manifest under `resolvePreferredOpenClawTmpDir()` or an equivalent trusted temp root outside every included backup source, keep archive verification strict, and add regression coverage for `TMPDIR` under the state directory.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: set `TMPDIR` to `<stateDir>/tmp`, include the state directory in backup planning, and `createBackupArchive()` can tar the same staged manifest as both the explicit manifest entry and a recursive state payload entry.

## Solution Assessment

Yes. Reusing the existing preferred OpenClaw temp resolver is the narrowest maintainable fix; changing the macOS service `TMPDIR` or relaxing verifier behavior would be broader and would not address self-inclusion at the backup boundary.
