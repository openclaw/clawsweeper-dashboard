## Summary

Keep open. Current main still stages the backup manifest under ambient `os.tmpdir()`, while the macOS service environment deliberately sets `TMPDIR` under the OpenClaw state directory; the focused fix PR #75031 was closed without merging, so this remains a narrow valid repair.

## What This Changes

Ensure `openclaw backup create` stages its manifest outside the backed-up state tree when macOS LaunchAgent `TMPDIR` points under `~/.openclaw/tmp`.

## Best Possible Solution

Backup creation should use an OpenClaw-managed temp root outside every planned source, preserve the one-root-manifest verifier invariant, and cover the LaunchAgent `TMPDIR`-under-state regression.

## Reproduction Assessment

Yes. The high-confidence reproduction path is source-level: run backup creation with `TMPDIR` resolving to `<stateDir>/tmp`, then the staged manifest can be archived once explicitly and once through recursive state traversal.

## Solution Assessment

Yes. Reusing `resolvePreferredOpenClawTmpDir()` or an equivalent containment-checked temp root is the narrowest maintainable fix; changing service-wide `TMPDIR` or loosening verification would be broader and less targeted.
