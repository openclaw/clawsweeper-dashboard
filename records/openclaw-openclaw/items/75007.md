## Summary

Keep open. Current main still creates the backup staging directory from ambient `os.tmpdir()`, while macOS service environments set `TMPDIR` under the OpenClaw state directory that the backup plan recursively archives; the provided context also shows PR #75031 is the active open fix candidate with closing syntax for this issue.

## What This Changes

The issue requests that `openclaw backup create` stage its temporary manifest outside the backed-up state payload when service `TMPDIR` points at `~/.openclaw/tmp`.

## Best Possible Solution

Land PR #75031 or an equivalent focused change so backup staging uses `resolvePreferredOpenClawTmpDir()` or another trusted temp root outside every included backup source, with a regression test proving exactly one root `manifest.json` when service `TMPDIR` is inside the state directory.

## Reproduction Assessment

Yes. Static current-main reproduction is high-confidence: macOS service env sets `TMPDIR` under state, backup creation writes `manifest.json` under `os.tmpdir()`, and tar archives both that explicit manifest and the recursive state source.

## Solution Assessment

Yes in direction. Replacing ambient `os.tmpdir()` with the existing trusted OpenClaw temp resolver, plus focused manifest-count regression coverage, is the narrow maintainable fix.
