## Summary

Keep open; current main still stages the backup manifest under ambient `os.tmpdir()`, and the focused repair PR #75031 was closed without merging, so the reported macOS LaunchAgent duplicate-manifest path remains valid and narrowly fixable.

## What This Changes

Fix `openclaw backup create` so backup staging temp files cannot be placed under the state directory when `TMPDIR` points inside `~/.openclaw`.

## Best Possible Solution

Backup creation should stage transient files in an OpenClaw-controlled temp root outside every archive source and add regression coverage for the LaunchAgent `TMPDIR`-under-state case.

## Reproduction Assessment

Yes. Set process `TMPDIR` to `<stateDir>/tmp`, include the state directory in the backup plan, and create a backup; the current source path puts `manifestPath` inside the recursively archived state tree while also passing it explicitly to tar.

## Solution Assessment

Yes. A backup-owned temp root outside planned sources is the narrowest maintainable fix; changing the macOS service `TMPDIR` policy or weakening backup verification would be broader and would not address backup self-inclusion.
