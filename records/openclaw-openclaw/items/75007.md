## Summary

Keep open. Current main still stages the backup manifest under ambient `os.tmpdir()`, so the macOS LaunchAgent `TMPDIR` path inside the state directory can cause the same `manifest.json` to be archived both explicitly and recursively. The focused related fix PR #75031 was closed without merging, so this remains a narrow actionable repair.

## What This Changes

Fix `openclaw backup create` so manifest staging stays outside the backed-up state directory when `TMPDIR` points at `~/.openclaw/tmp`, preventing duplicate `manifest.json` entries.

## Best Possible Solution

Backup creation should stage transient manifest files in a trusted OpenClaw temp root outside every planned backup source, keep the one-root-manifest verifier invariant, and add regression coverage for the LaunchAgent `TMPDIR`-under-state case.

## Reproduction Assessment

Yes. The source-level path trace is high-confidence: set `TMPDIR` to `<stateDir>/tmp`, create the staged manifest there, and the tar input list can include that file both as the explicit manifest and through the recursive state source.

## Solution Assessment

Yes. Reusing `resolvePreferredOpenClawTmpDir()` or an equivalent containment-checked OpenClaw temp root is the narrowest maintainable fix; changing the macOS service `TMPDIR` policy or relaxing backup verification would be broader and would not address backup self-inclusion.
