## Summary

Keep open. Current main still stages the backup manifest under ambient `os.tmpdir()`, while the macOS service environment points `TMPDIR` inside the OpenClaw state directory; the closed unmerged #75031 was a focused fix, so the bug remains actionable.

## What This Changes

`openclaw backup create` should avoid staging its temporary manifest inside the backed-up state tree when `TMPDIR` points to `~/.openclaw/tmp`.

## Best Possible Solution

Make backup staging use `resolvePreferredOpenClawTmpDir()` or an equivalent trusted temp root outside every included backup source, then cover the macOS service `TMPDIR=<state>/tmp` case with a regression that proves the archive has exactly one root manifest.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: set `TMPDIR` to the state directory's `tmp` child, run backup creation with the state directory included, and the staged manifest can be archived both explicitly and via the recursive state source.

## Solution Assessment

Yes. Reusing the existing preferred OpenClaw temp resolver is the narrowest maintainable fix; changing the macOS service `TMPDIR` or relaxing verification would be broader and would not address backup self-inclusion.
