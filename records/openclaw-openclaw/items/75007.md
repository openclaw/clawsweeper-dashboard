## Summary

Keep open. Current main still stages the backup manifest under ambient `os.tmpdir()`, while the macOS service environment can set `TMPDIR` inside the backed-up state directory; the linked focused fix PR is closed without merging, so the bug remains actionable.

## What This Changes

Prevent `openclaw backup create` from archiving duplicate root `manifest.json` entries when `TMPDIR` points inside the OpenClaw state directory.

## Best Possible Solution

Stage backup metadata under `resolvePreferredOpenClawTmpDir()` or an equivalent trusted temp root outside every included backup source, keep the macOS service `TMPDIR` behavior intact, and add regression coverage that proves a state-contained `TMPDIR` still produces exactly one root manifest.

## Reproduction Assessment

Yes. The source-level path is high confidence: set `TMPDIR` to `<state>/tmp`, create a backup with the state directory included, then inspect tar entries for root `manifest.json`; current main still has the staging and tar-input chain that permits duplication.

## Solution Assessment

Yes. Reusing the existing preferred OpenClaw temp resolver is the narrowest maintainable fix; changing LaunchAgent `TMPDIR` would be broader and could regress the SQLite temp-file workaround.
