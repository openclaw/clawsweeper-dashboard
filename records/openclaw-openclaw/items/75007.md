## Summary

Keep open. Current main still creates the backup manifest staging directory from ambient `os.tmpdir()`, while the macOS service environment sets `TMPDIR` under the OpenClaw state directory; the focused repair PR #75031 was closed without merging.

## What This Changes

Ensure `openclaw backup create` stages its temporary `manifest.json` outside `~/.openclaw/tmp` when the macOS LaunchAgent points `TMPDIR` there.

## Best Possible Solution

Backup creation should use an OpenClaw-managed temp root outside every planned archive source and add regression coverage for the LaunchAgent `TMPDIR`-under-state case.

## Reproduction Assessment

Yes. The high-confidence reproduction path is source-level: set `TMPDIR` to `<stateDir>/tmp`, create a state/full backup, and the staged manifest can be archived once explicitly and once through recursive state traversal.

## Solution Assessment

Yes. Reusing `resolvePreferredOpenClawTmpDir()` or an equivalent containment-checked temp root is the narrowest maintainable fix; changing service-wide `TMPDIR` or loosening verification would be broader.

## Repair Work Prompt

Fix the backup manifest self-inclusion bug described by the current issue. `src/infra/backup-create.ts` stages `manifest.json` under ambient `os.tmpdir()`, which is inside the OpenClaw state directory when the macOS LaunchAgent sets `TMPDIR=<stateDir>/tmp`; tar can then archive that manifest both explicitly and through recursive state traversal. Keep the fix scoped to backup creation: use `resolvePreferredOpenClawTmpDir()` or an equivalent OpenClaw-managed temp root outside planned backup sources, remove the unused `node:os` import if applicable, and do not change the macOS service TMPDIR policy. Add a regression test in `src/infra/backup-create.test.ts` that sets `TMPDIR` under the test state dir, creates a real backup, and asserts exactly one root `manifest.json`. Preserve `backupVerifyCommand()`'s one-root-manifest invariant. Add a single-line user-facing `CHANGELOG.md` fix entry.
