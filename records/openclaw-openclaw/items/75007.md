## Summary

Keep open. Current main still stages the backup manifest under ambient `os.tmpdir()` while the macOS service environment points `TMPDIR` inside the OpenClaw state directory, and the previously linked focused fix PR is closed without merging.

## What This Changes

Fix `openclaw backup create` so its temporary manifest staging directory is outside the backed-up OpenClaw state tree when service `TMPDIR` points at `~/.openclaw/tmp`.

## Best Possible Solution

Change backup creation to stage its manifest under `resolvePreferredOpenClawTmpDir()` or an equivalent trusted temp root outside every included backup source, remove the now-unused `os` import if applicable, and add regression coverage proving exactly one root `manifest.json` when `TMPDIR` is under the state directory.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: macOS service env sets `TMPDIR` to `<state>/tmp`, backup stages `manifest.json` under `os.tmpdir()`, and tar archives both the explicit manifest and the recursive state source.

## Solution Assessment

Yes for the issue's proposed direction. Reusing the existing preferred OpenClaw temp resolver is the narrow maintainable fix; changing service `TMPDIR` or loosening verifier behavior would be broader and less targeted.
