## Summary

Keep open. Current main still stages the backup manifest under ambient `os.tmpdir()`, which can be inside the backed-up state tree under the macOS service environment, and the focused fix PR was closed without merging.

## What This Changes

Fix `openclaw backup create` so its manifest staging directory is outside the backed-up state tree when service `TMPDIR` points at `~/.openclaw/tmp`.

## Best Possible Solution

Backup creation should stage its manifest in an OpenClaw-managed temp root outside every selected backup source while preserving the verifier’s exactly-one-root-manifest invariant.

## Reproduction Assessment

Yes. The high-confidence reproduction path is source-level: create a normal state backup with `TMPDIR=<stateDir>/tmp`, where the staged manifest is under the recursive state source and also passed explicitly to tar.

## Solution Assessment

Yes. Reusing `resolvePreferredOpenClawTmpDir()` or an equivalent source-containment-checked temp root in backup creation is the narrowest maintainable fix; service-wide `TMPDIR` and backup verification should stay strict.
