## Summary

Keep open. Current main has adjacent status, usage, context-warning, per-message metadata, and provider-auth overview surfaces, but it still lacks the requested persistent in-interface status area for active agent/runtime model plus token, quota, subscription, or cost context during operations.

## What This Changes

Request a persistent UI status area showing active agent/runtime model plus token, quota, subscription, or cost context during operations.

## Best Possible Solution

Define one Control UI status surface that composes active session/agent, actual runtime model, fresh context usage, response usage/cost, provider quota/subscription windows, and clear stale/unavailable/redacted states.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The gap is high-confidence by source and docs inspection because current main exposes controls, warnings, message metadata, reports, and command workarounds rather than the requested persistent active status area.

## Solution Assessment

Unclear for implementation. The direction is valid, but maintainers should scope placement and data semantics so topbar, chat context, provider quota, cost, and active-agent work do not fragment into conflicting surfaces.
