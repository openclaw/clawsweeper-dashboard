## Summary

Keep open. Current main has adjacent model pickers, context warnings, per-message metadata, usage reports, and provider-auth overview cards, but it still lacks the requested persistent in-interface active agent/model plus token, quota, subscription, or cost status surface during operations.

## What This Changes

Request a persistent interface status area showing the active agent/model and token, quota, subscription, or cost context during operations.

## Best Possible Solution

Define one Control UI status surface that composes active session/agent, actual runtime model, fresh context usage, response usage/cost when permitted, provider quota/subscription windows, and clear stale/unavailable/redacted states while coordinating the narrower related items.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main exposes adjacent controls, warnings, reports, and metadata rather than the requested persistent active-operation status area.

## Solution Assessment

Unclear for implementation. The direction is valid, but maintainers should first scope placement and data semantics so topbar, chat context, provider quota, cost, and active-agent work do not split into conflicting surfaces.
