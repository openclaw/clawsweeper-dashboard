## Summary

Keep open. Current main still has adjacent controls, warnings, reports, and provider-auth overview data, but it does not provide the requested persistent in-interface status area for active agent/runtime model plus token, quota, subscription, or cost context during operations; the related open items cover narrower pieces and do not fully supersede this umbrella request.

## What This Changes

The issue requests a persistent in-interface status area showing the active agent/model and token, quota, subscription, or cost context during operations.

## Best Possible Solution

Define one Control UI status surface that composes active session agent/runtime, actual runtime model, fresh context-token usage, permitted response usage/cost, provider quota/subscription windows, and clear unavailable or redacted states.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The gap is high-confidence by source and docs inspection because current main provides pickers, threshold warnings, overview/report surfaces, and command workarounds rather than the requested persistent active status area.

## Solution Assessment

Unclear for implementation. The requested direction is valid, but the best path needs maintainer UX/API scoping first so the narrower topbar, context, cost, and provider-quota requests do not create duplicate or contradictory surfaces.
