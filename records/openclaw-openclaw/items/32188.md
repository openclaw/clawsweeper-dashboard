## Summary

Keep open. Current main still has adjacent controls, warnings, reports, and provider-auth overview data, but it does not provide the requested persistent in-interface status area for the active agent/model plus token, quota, subscription, or cost context during operations. The related open items cover narrower pieces, especially topbar token/model display and persistent context usage, but they do not fully supersede this broader UX request.

## What This Changes

The issue requests a persistent interface status area showing the active agent/model and token, quota, subscription, or cost context during operations.

## Best Possible Solution

Keep this as a product/UX umbrella until maintainers define one Control UI status surface that composes active session agent/runtime, actual runtime model, fresh context-token usage, permitted response usage/cost, provider quota/subscription windows, and clear stale/unavailable/redacted states without relying on manual status commands.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Current source, tests, docs, and related-item context provide a high-confidence gap check because existing surfaces are controls, threshold warnings, reports, or overview cards rather than the requested persistent active status area.

## Solution Assessment

Unclear for implementation. The requested direction is valid, but the best path needs maintainer-scoped UX/API design first so narrower topbar, persistent context, active-agent, cost, and provider-quota work do not create duplicate or contradictory surfaces.
