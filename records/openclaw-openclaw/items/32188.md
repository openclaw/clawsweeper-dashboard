## Summary

Keep open. Current main has adjacent status commands, session metadata, model controls, context warnings, per-message usage metadata, and a model-auth overview card, but it still does not provide the requested persistent in-interface active agent/model plus token, quota, subscription, or cost status area during operations.

## What This Changes

Request a persistent Control UI status area showing the active agent/model and token, quota, subscription, or cost information during operations.

## Best Possible Solution

Define a single Control UI status surface that composes active session agent, actual runtime model, fresh context-token usage, permitted response usage/cost, provider quota/subscription windows, and clear stale/unavailable/redacted states.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main exposes adjacent controls, warnings, reports, and metadata rather than the requested persistent active-operation status area.

## Solution Assessment

Unclear for implementation. The direction is valid, but maintainers should first scope placement and data semantics so topbar, chat context, provider quota, cost, and active-agent work do not split into conflicting surfaces.
