The PR adds `--disallowedTools ScheduleWakeup,CronCreate` to the bundled Anthropic `claude-cli` backend defaults for fresh and resumed `-p` runs.

## Best Possible Solution

Land the argv-level denial only after maintainer acceptance of the compatibility tradeoff and redacted live CLI proof showing both fresh and resume paths reject the two tools and recover through a supported follow-up mechanism.

## Solution Assessment
