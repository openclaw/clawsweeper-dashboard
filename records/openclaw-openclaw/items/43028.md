## Summary

Keep open: current main has adjacent cron backoff, failure alerts, retention pruning, run logs, and `cron_changed` hooks, but it still lacks the requested recurring-job circuit breaker and delete-time cleanup contract. The changelog request also needs a product decision because current audit data lives in JSONL run history and hook events, not `.learnings/changelog.md`.

## What This Changes

The issue requests cron circuit-breaker auto-pause behavior, delete-time session cleanup, and automatic changelog/audit recording for cron runs.

## Best Possible Solution

Define a backward-compatible cron failure policy, explicit safe cleanup semantics, and the intended audit surface, then update scheduler, protocol/CLI, docs, and focused tests together.

## Reproduction Assessment

Yes by source inspection for the current gaps: recurring errors reschedule with backoff while enabled, `cron.remove` has no cleanup parameter, and cleanup is retention-based. No live cron run was executed because this review is read-only.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable solution needs explicit policy for thresholds, delete/archive safety, protocol/CLI shape, and whether `.learnings` should exist alongside run logs and hooks.
