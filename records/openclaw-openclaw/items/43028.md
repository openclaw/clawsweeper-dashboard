## Summary

Keep open: current main has cron timeout cleanup, error backoff, failure alerts, run-session retention, JSONL run history, and cron hooks, but it still lacks the requested recurring-job auto-pause/circuit breaker and delete-time `cron rm --cleanup` session cleanup. The requested `.learnings/changelog.md` writer also needs product direction because core already exposes JSONL run history plus `cron_changed` hooks.

## What This Changes

Request adds recurring cron circuit breaker/auto-pause behavior, delete-time cron session cleanup, and automatic changelog/audit records for cron executions.

## Best Possible Solution

Define a backward-compatible cron failure policy plus explicit cleanup contract, then either document the existing JSONL/hook audit trail or add a deliberate core changelog writer.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: repeated recurring errors increment counters and schedule backoff while the job remains enabled, `cron.remove` accepts no cleanup flag, and finished runs write JSONL/hook events rather than `.learnings/changelog.md`.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable fix should be designed and split explicitly; a core changelog writer may be less appropriate than extending or documenting JSONL run history and `cron_changed` hooks.
