## Summary

Keep open: current main has cron error backoff, failure alerts, retention-based run-session pruning, JSONL run history, and `cron_changed` hooks, but it still lacks the requested recurring-job auto-pause/circuit breaker and delete-time cleanup contract. The changelog request should be decided against the existing run-log and hook audit surfaces, and related open issues track parts of the work without replacing the whole bundle.

## What This Changes

Request adds cron auto-pause/circuit breaking for repeated timeouts, delete-time session cleanup, and automatic cron execution changelog or audit records.

## Best Possible Solution

Define a backward-compatible cron failure policy, explicit safe cleanup semantics, and the intended audit surface, then split the work across the existing focused cron guardrail and cleanup threads rather than landing a one-off timer-only change.

## Reproduction Assessment

Yes by source inspection for the current gaps: recurring failures reschedule with backoff while staying enabled, `cron.remove` accepts only id/jobId, and cleanup is retention-based. No live cron run was executed because this review is read-only.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable solution needs explicit policy for thresholds, delete/archive safety, protocol/CLI shape, and whether `.learnings` should exist alongside run logs and hooks.
