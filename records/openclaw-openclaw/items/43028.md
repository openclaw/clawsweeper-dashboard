## Summary

Keep open: current main has cron retry backoff, failure alerts, retention-based run-session pruning, JSONL run history, and cron change hooks, but it still does not define the requested recurring-job circuit breaker, delete-time session cleanup contract, or `.learnings/changelog.md` audit behavior.

## What This Changes

Request automatic cron circuit-breaking after repeated timeouts, optional cleanup when removing cron jobs, and automatic changelog/audit entries after cron executions.

## Best Possible Solution

Define a backward-compatible cron lifecycle policy that reuses existing run logs, hooks, failure alerts, and retention cleanup where possible, then split auto-pause and safe cleanup into reviewed protocol/CLI/config changes.

## Reproduction Assessment

Yes by source inspection for the current gaps: recurring failures reschedule with backoff while enabled, cron.remove accepts no cleanup parameter, and cleanup is retention-based. No live cron run was executed because this review is read-only.

## Solution Assessment

Unclear. The requested direction is reasonable, but the best fix needs maintainer decisions on failure thresholds, reset behavior, transcript deletion safety, and whether .learnings should be added or replaced by run logs and hooks.
