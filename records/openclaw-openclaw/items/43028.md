## Summary

Keep open. Current main has cron timeouts, error backoff, failure alerts, run-session retention, JSONL run history, and plugin hooks, but it still lacks the requested recurring-job circuit breaker/auto-pause, delete-time cron session cleanup option, and automatic `.learnings/changelog.md` writer.

## What This Changes

The issue requests cron automation improvements: recurring-job circuit breaking after repeated timeouts, cleanup of related cron sessions when removing jobs, and automatic changelog/audit recording after cron runs.

## Best Possible Solution

Keep this open and define the product/API split: add a backward-compatible recurring cron circuit-breaker config/schema/docs/test surface in the cron service, add an explicit safe cleanup option across Gateway protocol/CLI/UI/session ownership, and decide whether JSONL run history plus plugin hooks are sufficient or core should also write `.learnings/changelog.md`.

## Reproduction Assessment

Yes. The source gives a high-confidence reproduction path: a recurring cron job that times out becomes an error, increments `consecutiveErrors`, and schedules backoff while staying enabled; `cron.remove` accepts no cleanup flag and only removes the job record.

## Solution Assessment

Unclear. The requested direction is reasonable, but the best maintainable fix should be split and designed explicitly; a core `.learnings/changelog.md` writer may be less appropriate than documenting or extending the existing JSONL run history and `cron_changed` hook surfaces.
