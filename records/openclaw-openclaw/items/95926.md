Keep open: the PR is aimed at the right Groq/media-provider recovery problem, but the current helper still accepts official provider catalog entries that do not own the media-understanding contract, so it can emit misleading install guidance for text-only provider IDs.

## Best Possible Solution

Gate the hint on `contracts.mediaUnderstandingProviders`, preserve legacy output for non-media IDs, and add real CLI or Gateway output proving the changed error path.

Yes at source level: current main throws the generic missing-provider error when a configured media provider is absent from the registry, and the linked issue supplies Groq CLI/Gateway logs. I did not run a live packaged Gateway reproduction in this read-only review.

## Solution Assessment

No: appending the hint at these throw sites is the right layer, but the ownership check is still too broad because it accepts provider catalog entries that lack the media-understanding contract.
