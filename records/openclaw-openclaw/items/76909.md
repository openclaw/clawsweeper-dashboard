  - body: `discoverInDirectory` requires `env` and uses it for hardlink rejection and plugin API compatibility checks. The new system-tier call omits that field, so the patch does not satisfy the existing discovery contract and would scan with an undefined env if type checking were bypassed.
  - confidence: 0.92
- **[P1] Require admin-owned system plugin roots:** `src/plugins/discovery.ts:982-992`
  - body: This new scan delegates to the generic plugin directory safety checks, which skip permission validation on Windows and accept current-user-owned paths on POSIX.   - confidence: 0.9
- **[P1] Make system origin win duplicate ids:** `src/plugins/manifest-registry.ts:98`

That breaks the stated guarantee that system policy plugins have highest precedence and cannot be replaced by user-controlled sources.
  - confidence: 0.88
