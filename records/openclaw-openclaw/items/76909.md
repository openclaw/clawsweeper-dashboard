## Summary

Keep open; this draft should not merge until the privileged system-plugin trust boundary, ACL enforcement, precedence, and hook ordering semantics are approved and hardened.

## What This Changes

The PR adds a `system` plugin origin discovered from machine-wide directories, bypasses user activation controls, grants implicit hook permissions, floors hook priority, and adds related tests/display handling.

## Best Possible Solution

Define the operator-managed system-plugin trust model first, then land a smaller implementation with enforced ACL/ownership checks, reserved precedence/order semantics, docs, changelog, and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection of the PR diff and current-main hook/discovery code reproduces the blockers: Windows permission checks return early, duplicate precedence lacks system handling, and hook execution sorts by raw priority.

## Solution Assessment

No. The branch starts by bypassing existing user controls for privileged in-process plugins; the safer path is an approved admin trust boundary with explicit enforcement across discovery, activation, precedence, hook ordering, docs, and tests.

  - body: This branch makes system plugins ignore the global plugin toggle, allowlist, denylist, and per-entry disables while later granting privileged prompt/conversation hook access.
