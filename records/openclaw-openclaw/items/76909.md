## Summary

Keep open: the draft PR introduces a privileged, un-disableable plugin trust tier and still has concrete security and precedence blockers that need maintainer product/security review before merge.

## What This Changes

The PR adds a `system` plugin origin discovered from machine-wide plugin directories, bypasses normal activation controls, grants implicit hook permissions, floors hook priority, and adds related tests.

## Best Possible Solution

Define and approve the operator-managed system-plugin trust boundary first, then land a smaller implementation with enforced ownership/ACL checks, reserved ordering/precedence, docs, changelog, and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the blockers: current discovery skips Windows permission checks, hook execution sorts by raw numeric priority, and duplicate precedence does not handle the new origin.

## Solution Assessment

No. A privileged machine-wide plugin tier should not start by bypassing existing plugin controls; the safer path is an approved admin trust model with explicit enforcement across discovery, activation, precedence, docs, and tests.

Full review comments:

- **[P1] Define admin trust before bypassing plugin controls:** `src/plugins/config-activation-shared.ts:141-150`
  - body: This branch makes system plugins ignore the global plugin toggle, allowlist, denylist, and per-entry disables while later granting privileged prompt/conversation hook access.   - body: The new `%ProgramData%\OpenClaw\plugins` root is treated as privileged, but the existing discovery permission gate returns without checking ownership or ACLs on Windows. Add platform-specific admin ACL validation or keep the system tier disabled on Windows until that exists.
  - confidence: 0.86
- **[P2] Rank system origins before lower-trust duplicates:** `src/plugins/manifest-registry.ts:101`
  - body: The PR says system plugins cannot be overridden, but adding `system` as the lowest origin rank and leaving duplicate precedence to the default path lets config/global/workspace/bundled duplicates win registry or catalog metadata. System origin needs explicit top precedence wherever duplicate/plugin-origin ordering is resolved.
  - confidence: 0.9
- **[P2] Reserve the system hook tier in ordering:** `src/plugins/registry.ts:2075-2080`
  - body: Clamping system hooks to `>= 10000` does not guarantee they run first because non-system hooks can still register a larger priority and the runner sorts only by raw numeric priority. Add an origin tier to hook ordering or cap non-system priorities.
  - confidence: 0.91
