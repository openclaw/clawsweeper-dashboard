## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Suppress split-only text sends:** `src/infra/outbound/deliver.ts:969`
  - body: When text contains only `[[SPLIT]]` markers, `splitOnSplitTags()` returns an empty array and this fallback sends the original marker text to users. Empty split-only output should be suppressed instead of delivered as a visible control token.
  - confidence: 0.94
- **[P2] Suppress split-only media captions:** `src/infra/outbound/deliver.ts:1237`
  - body: The media path uses the original payload text as the caption when every segment is removed by split parsing, so a caption like `[[SPLIT]]` is sent literally with the attachment. Use an empty caption/no follow-up when the split result is empty.
  - confidence: 0.92
- **[P2] Store the full opening fence marker:** `src/utils/split-tag.ts:87-88`
  - body: The parser saves only the first fence character in `openFence.marker`, so a three-backtick line can close a four-backtick fence. That makes `[[SPLIT]]` inside still-open code blocks eligible for delivery splitting.
  - confidence: 0.96
- **[P2] Buffer split tags across streamed chunks:** `src/agents/pi-embedded-subscribe.ts:714`
  - body: Streaming calls `splitOnSplitTags()` on each emitted chunk independently. If a marker arrives as `[[SP` and then `LIT]]`, the raw directive leaks and no split happens; this needs a small boundary-aware accumulator.
  - confidence: 0.88
- **[P2] Reject relative media local roots:** `src/media/local-roots.ts:73`
  - body: `media.localRoots` expands local file access, but relative entries are accepted and resolved against the process working directory. Validate configured roots as absolute paths before adding them to the allowlist.
  - confidence: 0.9
- **[P1] Serialize shoarchive registry writes:** `src/shoarchive/pdf-shoarchive.ts:213`
  - body: Each registration reads `INDEX.md`, computes the next ID, and later writes the registry/index without a lock or compare-and-swap. Concurrent PDF registrations can choose the same ID and overwrite each other's index updates.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The diff introduces security-sensitive surfaces and has a concrete local-file-access validation issue in `media.localRoots`.

Concerns:

- **[medium] Relative media roots broaden local file access:** `src/media/local-roots.ts:73`
  - body: `media.localRoots` accepts arbitrary strings and resolves them with `path.resolve`, so relative or empty entries can expand attachment-read permissions based on the gateway working directory. Because this is a local file allowlist, config validation should require absolute, normalized paths.
  - confidence: 0.9
- **[low] Broad security-sensitive surfaces need owner review:**

Those changes are not inherently malicious, but they are outside the stated feature and need focused owner/security review before merge.
  - confidence: 0.78
