## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Suppress split-only text sends:** `src/infra/outbound/deliver.ts:969`
  - body: When the output is only `[[SPLIT]]` markers, this fallback sends the control token as a normal text message. Split-only output should produce no text unit instead of leaking the directive to users.
  - confidence: 0.94
- **[P2] Suppress split-only media captions:** `src/infra/outbound/deliver.ts:1237`
  - body: For media payloads whose caption is only split markers, this fallback uses the original marker text as the media caption. That leaks `[[SPLIT]]` with the attachment instead of sending an empty caption and no follow-up.
  - confidence: 0.92
- **[P2] Store the full opening fence marker:** `src/utils/split-tag.ts:87-88`
  - body: The parser stores only the first fence character, so a three-backtick line can close a four-backtick fence. That lets `[[SPLIT]]` inside still-open code blocks split outbound content.
  - confidence: 0.96
- **[P2] Buffer split tags across streamed chunks:** `src/agents/pi-embedded-subscribe.ts:714`
  - body: Streaming calls `splitOnSplitTags()` on each emitted chunk independently. If the model emits `[[SP` and then `LIT]]`, the raw directive leaks and no split occurs; this needs a boundary-aware accumulator.
  - confidence: 0.88
- **[P2] Reject relative media local roots:** `src/media/local-roots.ts:73`
  - body: `media.localRoots` expands host file access, but relative entries are resolved against the process working directory and added to the allowlist. Config validation should require absolute, normalized paths.
  - confidence: 0.9
- **[P1] Serialize shoarchive registry writes:** `src/shoarchive/pdf-shoarchive.ts:213-266`
  - body: Each registration reads the index, computes the next ID, and writes the registry/index without locking or compare-and-swap. Concurrent PDF registrations can choose the same ID or drop each other's index rows.
  - confidence: 0.84
