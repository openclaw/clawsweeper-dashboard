## Summary

This issue is valid and still necessary. Current main breaks the documented Windows absolute-path command contract, while the linked implementation PR remains open and has not yet demonstrated the complete Windows QMD workflow.

## What This Changes

Fix Windows command resolution for the QMD memory backend so absolute executable paths retain separators and QMD can index and search successfully.

## Best Possible Solution

Land one focused cross-platform executable-resolution fix that preserves the documented command-path contract and proves a real Windows QMD index or search operation; track the default-entrypoint failure separately only if it remains reproducible afterward.

## Reproduction Assessment

Yes at source level: current main sends the documented unquoted Windows absolute executable path through POSIX parsing that consumes its backslashes before the Windows spawn adapter. A full live Windows QMD invocation was not independently run during this read-only review.

## Solution Assessment

Unclear for the linked implementation as the complete fix: preserving Windows path separators is the correct narrow repair for the confirmed parser defect, but the reported default-entrypoint failure and actual Windows QMD round trip remain unproven.
