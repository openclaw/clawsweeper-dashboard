## Summary

Current main still contains the reported Windows path-corruption bug, and the issue is correctly paired with an open, clean fix PR. Keep it open until that PR is reviewed, proven on Windows, and merged.

## What This Changes

The issue reports that Windows absolute paths configured for the QMD memory command are corrupted before process launch, preventing QMD indexing and search.

## Best Possible Solution

Preserve Windows absolute command strings before shell parsing, retain the shared no-shell Windows executable resolver, and prove both an explicit QMD path and the default bare `qmd` command through real Windows indexing and search.

## Reproduction Assessment

Yes for the configured-path failure: current main deterministically sends a backslash-delimited Windows path through POSIX shell parsing before launch. The complete default-command failure still needs a live Windows QMD run.

## Solution Assessment

Yes in direction: the linked PR fixes the narrow boundary where absolute Windows paths are corrupted. It should not be considered complete until Windows proof covers the actual QMD executable contract and both reported command modes.
