## Summary

Keep open: current main still has a source-visible path where blocked subagent terminal metadata can be normalized to success, and the open fix PR https://github.com/openclaw/openclaw/pull/80886 specifically closes this issue but has not merged.

## What This Changes

The issue reports that context-overflow subagent runs with terminal `livenessState: "blocked"` are reported to the parent as successful empty completions instead of errors.

## Best Possible Solution

Land or explicitly supersede https://github.com/openclaw/openclaw/pull/80886 so blocked terminal metadata is normalized before parent completion synthesis and covered by regression tests.

## Reproduction Assessment

Yes at source level: current main can emit `livenessState: "blocked"` for context overflow, then later map terminal `end` or `ok` wait snapshots to successful subagent outcomes. I did not run a live Atlas/provider overflow in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to normalize blocked liveness before gateway wait, registry completion, and announcement success synthesis. The open PR https://github.com/openclaw/openclaw/pull/80886 is the right current path; https://github.com/openclaw/openclaw/pull/75462 is adjacent swallowed-error work, not a replacement for the blocked-terminal fix.
