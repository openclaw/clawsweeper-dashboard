## Summary

Keep open: current main still has a source-visible false-success path for blocked subagent terminal states, and an open PR already references this issue with closing syntax, so the issue should remain open until that fix or an equivalent one merges.

## What This Changes

The issue asks OpenClaw to report subagent context_overflow or livenessState "blocked" terminal states as blocked/error outcomes with useful messages instead of successful completions.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80886 or an equivalent shared-boundary normalization, then close this issue after merge and verification.

## Reproduction Assessment

Yes at source level: current main can carry livenessState "blocked" from embedded context-overflow metadata into lifecycle/wait snapshots, then map the terminal state to ok before the parent announcement. I did not run a live Atlas/GPT-5.5 overflow reproduction.

## Solution Assessment

Yes, the narrow maintainable fix is to normalize blocked liveness/error metadata at shared agent.wait and dedupe boundaries, with defensive registry and announcement handling. https://github.com/openclaw/openclaw/pull/80886 is the current candidate for that path.
