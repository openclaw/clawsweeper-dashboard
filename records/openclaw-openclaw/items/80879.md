## Summary

Current main still has a source-visible false-success path for blocked subagent terminal metadata, and the linked open PR already uses closing syntax for this issue. Keep this open until that PR, or an equivalent fix, is reviewed and merged.

## What This Changes

Report that subagent runs ending with `livenessState: "blocked"` or `context_overflow` can be announced to the parent as successful with no useful output or error.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80886, or an equivalent shared-boundary fix that normalizes blocked liveness through gateway wait/dedupe, run-wait, registry completion, and parent announcement rendering.

## Reproduction Assessment

## Solution Assessment

Yes; the narrow maintainable solution is to classify blocked/error terminal metadata before parent completion synthesis and cover gateway wait, dedupe, registry, and announce paths. The existing open PR is the right active implementation candidate to review.
