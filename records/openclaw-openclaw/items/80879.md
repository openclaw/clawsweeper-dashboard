## Summary

Keep open. Current main still has a source-visible false-success path: blocked/context-overflow metadata reaches lifecycle and wait snapshots, but subagent completion synthesis can still normalize that terminal state to an ok outcome and announce success.

## What This Changes

Report subagent runs with blocked/context_overflow terminal metadata as blocked or failed to the parent instead of successful no-output completions.

## Best Possible Solution

Normalize blocked/error terminal metadata before parent completion synthesis, surface the error text to the requester, and add regression coverage for lifecycle end and agent.wait snapshots carrying livenessState "blocked" or context_overflow metadata.

## Reproduction Assessment

Yes, at source level. A terminal lifecycle end or agent.wait ok snapshot carrying livenessState "blocked" still flows into ok completion synthesis; no live Atlas overflow run was executed in this read-only review.

## Solution Assessment

Yes. A narrow bridge/status-normalization fix is the best direction, and it should not change provider compaction behavior or add configuration.
