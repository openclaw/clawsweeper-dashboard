## What This Changes

The PR adds optional realtime consult thinking-level and fast-mode overrides for the Voice Call plugin and Control UI Talk realtime consults, wiring them through config/schema/docs, Gateway protocol/session results, chat.send one-shot options, runtime handoff, UI code, and focused tests.

## Best Possible Solution

Land or adapt the additive override implementation after correcting the Talk docs examples and getting exact-head validation green, while keeping the separate fast memory/context design handled by its already-closed follow-up path rather than expanding this PR.

## Reproduction Assessment

Yes. Source inspection on current main is enough to reproduce the missing capability: the strict Voice Call and Talk config paths, Talk session result, chat.send schema, shared consult runtime, and UI handoff do not carry consultThinkingLevel or consultFastMode through the realtime consult run.

## Solution Assessment

Yes for this narrower override feature after the docs correction. Additive config plus one-shot runtime handoff preserves unset behavior and is narrower than redesigning the realtime fast-context path.
