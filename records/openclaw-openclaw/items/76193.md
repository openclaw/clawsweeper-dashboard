## Summary

Keep open. Current main still has the Telegram fallback path that can synthesize generic visible fallback text after message-tool-only source delivery; this PR is a narrow implementation with no blocking review finding found.

## What This Changes

The PR adds a resolved source-reply delivery-mode callback from shared dispatch, uses it in Telegram to suppress fallback delivery for message-tool-only turns, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land this PR or an equivalent narrow patch after maintainer review, keeping mode resolution in shared dispatch and Telegram fallback policy in the Telegram adapter.

## Reproduction Assessment

Yes by source inspection. Current main resolves message_tool_only and suppresses automatic source delivery, while Telegram still reaches the generic error/NO_REPLY fallback path when no visible delivery occurs.

## Solution Assessment

Yes. The callback seam is narrow because shared dispatch owns the resolved source-delivery mode and Telegram owns post-dispatch fallback behavior; it does not change non-Telegram fallback policy.
