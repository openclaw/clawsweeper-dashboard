## Summary

Keep open. Current main still lacks any way for Telegram’s post-dispatch fallback path to know that shared source delivery resolved to `message_tool_only`, while this PR adds that seam, Telegram fallback suppression, focused regression tests, and the required changelog entry; related #75857 covers a different callback-query fallback path and does not supersede this PR.

## What This Changes

The PR adds a resolved source-reply delivery-mode callback to shared dispatch and uses it in Telegram to suppress generic fallback replies for message-tool-only turns, with regression coverage and a changelog entry.

## Best Possible Solution

Land this PR or an equivalent narrow change after normal CI and mergeability checks, keeping delivery-mode resolution in shared dispatch and the fallback policy in the Telegram adapter.

## Solution Assessment
