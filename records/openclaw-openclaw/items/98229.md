## What This Changes

The issue requests that group-chat history render user messages with sender metadata instead of always labeling every prior user turn as `User:`.

## Best Possible Solution

Update the source renderer and focused session-history tests so trusted sender metadata is used for user-history labels, while preserving a sanitized `User` fallback for unattributed direct or legacy turns.
