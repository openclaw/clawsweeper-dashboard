## What This Changes

The issue reports that upgrading to v2026.6.1 broke DeepSeek prompt-cache hit rates for `deepseek/deepseek-v4-flash` on the OpenAI-compatible completions path and asks for upgrade-safe provider/cache handling.

## Best Possible Solution

Run a DeepSeek-live comparison with repeated stable-prefix prompts on v2026.6.1 and current main, then update the DeepSeek provider catalog/policy, doctor migration, and docs only around the contract that the live API proves.

## Reproduction Assessment

Partially yes: source inspection shows current main still omits DeepSeek `prompt_cache_key` unless the model config explicitly opts in. No high-confidence live reproduction exists here for the full sustained zero-cache-hit/cost spike claim.

## Solution Assessment

Unclear: adding `supportsPromptCacheKey` is a plausible narrow repair for OpenClaw's gate, but the issue says manual opt-in did not restore hits and DeepSeek's public contract points to automatic prefix caching. The best fix needs provider-live proof and a maintainer decision at the DeepSeek plugin boundary.
