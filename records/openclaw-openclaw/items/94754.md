- **[P2] Carry marker-derived silent state across media fan-out:** `extensions/telegram/src/send.ts:650`
  - body: This computes `effectiveSilent` inside one low-level send call, but `sendTelegramPayloadMessages` sends only the first media item with the payload text and sends later media items with `text: ""`. For a multi-media Telegram payload ending in `notify=false`, the first media send becomes silent while later media sends have no marker and remain loud. Normalize the marker before fan-out or propagate the derived silent flag to every media send in the batch.
  - confidence: 0.9
- **[P2] Cover the native reply delivery path:** `extensions/telegram/src/send.ts:650`

A reply text ending in `notify=false` on that path will still leak the marker and notify normally unless the same normalization is applied there or the claim is narrowed with proof that path cannot receive this marker.
