## Summary

Keep open: this protected draft maintainer PR contains a useful provider-factory isolation idea, but the current diff changes the all-failing selected provider path from an observable initialization failure into silent provider unavailability.

The PR changes bundled web-provider public-artifact loading to skip throwing factories, return an empty provider list when all matching factories fail, and updates fast-path tests for all-fail and mixed selected-provider cases.

## Best Possible Solution

Keep healthy sibling-provider isolation, but preserve an observable all-failing provider path through a thrown error, diagnostic-backed failure result, or explicit maintainer-approved policy applied consistently across public-artifact loaders.

## Solution Assessment

No, not as written. Sibling-provider isolation is a plausible bug fix, but the all-failing selected provider path should stay observable or receive explicit maintainer approval before becoming fail-open.
