## Summary

Keep open: this maintainer-labeled draft still contains a meaningful provider-routing behavior change that current main and the latest release do not implement, and the all-failing-provider fail-open semantics need maintainer compatibility approval before merge.

## What This Changes

The PR changes bundled web-provider public-artifact discovery so a plugin whose matching provider factories all throw contributes no providers instead of throwing, while adding focused fast-path coverage for that behavior.

## Best Possible Solution

Keep the merged per-factory isolation from https://github.com/openclaw/openclaw/pull/88807, and land this all-failing-provider fail-open behavior only if maintainers explicitly accept the compatibility change and the setup/contract diagnostics story is clear.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main throws when every matching bundled web-provider factory for a selected plugin fails, and the PR changes that path to return an empty provider list. I did not run tests because this review is read-only.

## Solution Assessment

Unclear: the patch is a narrow way to keep healthy sibling providers active, but making an all-failing selected provider silently unavailable is a compatibility and product-policy decision. A safer merge state would pair the fail-open behavior with explicit diagnostics or maintainer-approved documentation of the new contract.
