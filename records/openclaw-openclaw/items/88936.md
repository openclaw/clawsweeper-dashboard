## Summary

Keep this draft maintainer PR open for review. The patch is focused and source-reproducible, but it deliberately changes the all-factories-fail path from a loud initialization error into provider unavailability, so provider-routing/fallback semantics need maintainer acceptance before merge.

## What This Changes

The PR changes bundled web provider public-artifact discovery so throwing provider factories are skipped and all-failing plugin artifacts contribute no provider entries, with focused tests for the new behavior.

## Best Possible Solution

Keep the per-factory isolation, but merge only after maintainers either explicitly accept fail-open all-failing web provider artifacts or narrow the change so configured/single-provider failures remain visible.

## Reproduction Assessment

Yes. Current main source shows an all-failing matching factory set throws before later selected providers can be resolved, and the PR test changes that expected result to an empty provider list.

## Solution Assessment

Unclear. Skipping a broken plugin so healthy siblings can load is a good mitigation, but the best fix may need to preserve loud behavior for configured or single-plugin failures rather than making every all-failing artifact unavailable.
