## Summary

Keep open: current main and the latest release still use the loud all-failing factory behavior, and this draft maintainer PR is the active proposal to change that compatibility-sensitive provider-routing contract.

## What This Changes

The PR changes bundled web-provider public-artifact discovery so throwing provider factories are skipped and healthy selected sibling providers can still resolve.

## Best Possible Solution

Maintainers should explicitly choose the provider-routing contract, ideally preserving sibling-provider isolation while making all-failing plugin unavailability visible through diagnostics or documented fail-open behavior.

## Solution Assessment

Unclear. The implementation is narrow and plausible for isolating healthy sibling providers, but silently treating an all-failing selected provider as unavailable is a maintainer compatibility decision.
