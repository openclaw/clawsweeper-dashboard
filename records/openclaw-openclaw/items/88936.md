## Summary

Keep this PR open for explicit maintainer review. The branch is focused and well-proven, but it intentionally changes shipped selected-provider failure behavior from a visible initialization error into silent provider unavailability, and the PR has a protected maintainer label.

## What This Changes

The PR changes bundled web-provider public-artifact loading so a selected plugin whose matching provider factories all throw is treated as unavailable while healthy selected sibling providers still load.

## Best Possible Solution

Keep the already-merged sibling-provider isolation, and merge this branch only if maintainers intentionally approve all-failing selected bundled web providers becoming unavailable with an observable diagnostic or explicit fail-open contract.

Yes. Source inspection and adjacent tests show current main throws for an all-failing selected bundled web-provider factory, while this PR changes that same path to return an empty provider list.

Unclear. The already-merged sibling-provider isolation is the narrow bug fix; making an all-failing selected provider disappear is a policy choice unless maintainers add diagnostics or approve that fail-open contract.
