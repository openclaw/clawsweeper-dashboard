## Summary

Keep open: the PR addresses a plausible startup-isolation bug, but as written it changes shipped all-failing selected web-provider artifacts from a loud initialization error into silent empty provider availability. Because this is a protected draft maintainer PR and provider-routing semantics are compatibility-sensitive, it needs maintainer/author resolution rather than cleanup closure.

## What This Changes

The PR changes bundled web-provider public-artifact loading to skip throwing provider factories, return an empty provider list when every matching factory fails, and add fast-path coverage for keeping a healthy sibling provider.

## Best Possible Solution

Keep the healthy sibling-provider isolation, but preserve an observable all-failing provider path through an error, diagnostic-backed failure result, or explicit maintainer-approved policy applied consistently across public-artifact loaders.

## Reproduction Assessment

Yes. Source inspection and adjacent tests show current main throws for an all-failing selected bundled web-provider factory, while this PR changes that same path to return an empty provider list.

## Solution Assessment

No, not as written. The sibling-provider isolation is a reasonable bug fix, but the all-failing selected provider path should stay observable or receive explicit maintainer approval before becoming fail-open.
