## Summary

Keep open for maintainer review: the patch is narrow and covered by focused tests, but it deliberately changes bundled web provider factory failures from loud initialization failures into silent provider unavailability, which is a provider-routing compatibility decision.

## What This Changes

The PR changes bundled web provider public-artifact discovery so throwing provider factories are skipped and all-failing plugins contribute no provider entries.

## Best Possible Solution

Maintainers should explicitly choose the desired provider-routing contract, ideally preserving sibling-provider isolation while making all-failing plugin unavailability visible through diagnostics or documented fail-open behavior.
