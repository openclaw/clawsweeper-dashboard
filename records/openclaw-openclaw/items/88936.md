## Summary

Keep open: this maintainer PR contains focused provider hardening work, but it changes current and shipped all-failing web provider factory behavior from a visible initialization failure into provider unavailability, so it needs maintainer contract review rather than cleanup closure.

## What This Changes

The PR changes bundled web provider public-artifact discovery so throwing provider factories are skipped while healthy sibling providers can still load.

## Best Possible Solution

Maintainers should explicitly choose the provider failure contract, ideally keeping sibling-provider isolation while preserving observable diagnostics or documented fail-open behavior for all-failing selected providers.

## Solution Assessment

Unclear: the implementation is a narrow mitigation for sibling-provider poisoning, but silently treating an all-failing selected provider as unavailable is a maintainer product and compatibility decision.
