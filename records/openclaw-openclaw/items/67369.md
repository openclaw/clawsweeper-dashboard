## Summary

Keep open. Current main still covers agent scoping, JSON output, limits, search/label, and recent-session filtering, but it does not provide duration-style older/newer filters, size-threshold filtering, or a public session-size metric across the CLI, Gateway protocol, Gateway rows, docs, and agent tool surface.

## What This Changes

Add session-list filtering by agent, age, and size, with structured output for automation.

## Best Possible Solution

Define the public size metric first, then add additive older/newer and size filters consistently across the sessions CLI, Gateway `sessions.list` protocol and rows, `sessions_list`, JSON output, docs, and focused regression coverage.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. Source inspection gives a high-confidence verification path: current CLI, Gateway protocol, row, docs, and tool surfaces expose only existing agent, JSON, limit, search/label, and active-recency support.

## Solution Assessment

Unclear until maintainers define the public size metric and field names. Duration filters are a narrow additive extension, but size filtering should be designed once and mirrored through all public session-list surfaces.
