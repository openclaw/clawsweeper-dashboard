## Summary

Keep open. Current main still only partially satisfies the request: agent scoping, JSON output, and recent-session filtering exist, but duration-style older/newer filters, size-threshold filtering, and byte-size session metadata are still absent from the CLI, gateway protocol, gateway rows, docs, and `sessions_list` tool surface.

## What This Changes

Extend `openclaw sessions` and related automation surfaces to filter sessions by agent, age, and size with structured JSON output.

## Best Possible Solution

Define one public session size metric, then add additive older/newer and size query fields through the CLI, gateway `sessions.list` protocol, gateway row metadata/filtering, `sessions_list`, docs, and focused tests for combined filters and JSON output.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. There is a high-confidence source verification path: inspect the CLI registration, command implementation, gateway schema/types/filtering, agent tool schema, docs, and tests, which show only existing agent scoping, JSON output, and `activeMinutes` recency filtering.

## Solution Assessment

Unclear until maintainers define the size metric. Older/newer duration filters are a narrow extension of existing recency filtering, but size filtering should wait for an explicit public field and threshold semantics.
