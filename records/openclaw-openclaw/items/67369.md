## Summary

Keep open. Current main still only partially satisfies the request: agent scoping, JSON output, limit, search, label, and recent-session filtering exist, but duration-style older/newer filters, size-threshold filtering, and a public session-size metric are still missing.

## What This Changes

The issue requests additional `openclaw sessions` filtering by agent, age, and size, plus structured JSON output for automation.

## Best Possible Solution

Define the public session-size metric first, then add additive older/newer and size filters consistently across the sessions CLI, Gateway protocol and rows, `sessions_list`, JSON output, docs, and focused regression coverage.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. Source inspection on current main gives a high-confidence verification path: the CLI, Gateway protocol, row, docs, and tool surfaces still expose only the existing recency, agent, search, label, limit, and JSON support.

## Solution Assessment

Unclear until maintainers define the public size metric and field names. Duration filters are a narrow additive extension, but size filtering should be designed once and mirrored through all public session-list surfaces.
