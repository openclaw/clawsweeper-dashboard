## Summary

Keep open. Current main still only partially satisfies the request: agent scoping, JSON output, and recent-session filtering exist, but duration-style older/newer filters, size-threshold filtering, and a public byte-size session metric are still missing across the CLI, Gateway protocol, Gateway rows, docs, and `sessions_list` tool surface.

## What This Changes

Add session query filters to `openclaw sessions` for agent, age, size, and structured automation output.

## Best Possible Solution

Define one public session size metric, then add additive older/newer and size query fields consistently through the CLI, Gateway `sessions.list` protocol, Gateway row metadata/filtering, `sessions_list`, docs, and focused tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. There is a high-confidence source verification path: current main shows only existing `--agent`, `--json`, `--active`/`activeMinutes`, search, label, and limit support across the CLI, Gateway, docs, and tool surfaces.

## Solution Assessment

Unclear until maintainers define the size metric and public field names. Older/newer duration filters are a narrow extension of existing recency filtering, but size filtering should wait for an explicit CLI/Gateway/tool contract.
