## Summary

Keep open. Current main partially satisfies the request with agent scoping, JSON output, limits, search/label, and recent-session filtering, but it still lacks duration-style older/newer filters, a size-threshold filter, and a public size metric across the CLI, Gateway protocol, Gateway rows, docs, and agent tool surface; the related close-API and lifecycle-hook issues do not supersede this remaining query work.

## What This Changes

The issue requests `openclaw sessions` filtering by agent, older/newer age, and size threshold, plus structured JSON output for automation.

## Best Possible Solution

Define the public session-size metric, then add additive older/newer and size filters consistently across the sessions CLI, Gateway `sessions.list`, `sessions_list`, JSON rows, docs, and focused tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. Source inspection on current main gives a high-confidence verification path: the relevant CLI, Gateway protocol, Gateway row, docs, and agent tool surfaces still expose only existing recency, agent, search/label, limit, and JSON support.

## Solution Assessment

Unclear until maintainers define the public size metric and field names. Duration filters can be an additive extension, but size filtering should be designed once and mirrored through all public session-list surfaces.
