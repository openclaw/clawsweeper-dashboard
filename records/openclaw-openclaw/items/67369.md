## Summary

Keep open. Current main still only partially satisfies the request: `--agent`, `--json`, `--active`, `activeMinutes`, search, label, and limit exist, but duration-style older/newer filters, size-threshold filtering, and a public byte-size session metric are still missing from the CLI, Gateway protocol, Gateway row shape, and `sessions_list` tool surface.

## What This Changes

The issue requests extending `openclaw sessions` with agent, age, size, and JSON filtering for fleet automation.

## Best Possible Solution

Define one documented session-size metric, then add additive older/newer and size filters consistently through the CLI, Gateway `sessions.list`, `sessions_list`, JSON/docs, and focused tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. Source inspection on current main gives a high-confidence verification path: the relevant CLI, Gateway, row, docs, and tool surfaces only expose the existing recency, agent, search, label, limit, and JSON support.

## Solution Assessment

Unclear until maintainers define the public size metric and field names. Older/newer duration filters are a narrow additive extension, but size filtering should wait for an explicit CLI/Gateway/tool contract.
