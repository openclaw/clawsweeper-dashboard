## Summary

Keep open. Current main still only partially covers the request: agent scoping, JSON output, and `activeMinutes`/`--active` recency filtering exist, but duration-style older/newer filters, size-threshold filtering, and byte-size metadata are still absent from the CLI, gateway protocol, gateway rows, and `sessions_list` tool surface.

## What This Changes

Request to extend `openclaw sessions` with agent, age, and size filters plus structured output for fleet automation.

## Best Possible Solution

Define one public size metric first, then add additive age and size query fields through the CLI, gateway schema/implementation, row metadata, and agent tool, with docs and focused tests that cover combined filters and JSON output.

## Reproduction Assessment

Not applicable as a runtime bug reproduction, but there is a high-confidence source verification path: inspect the current CLI registration, command implementation, gateway schema/implementation, row types, tool schema, docs, and tests for the requested options and metadata.

## Solution Assessment

Unclear for an automated patch right now. Older/newer duration filters are a narrow extension of existing recency filtering, but the size filter is not safely defined until maintainers choose the metric and public field name.
