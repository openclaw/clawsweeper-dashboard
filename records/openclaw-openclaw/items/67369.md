## Summary

Keep open. Current main still only partially covers the request: agent scoping, JSON output, limits, search/label, and recent-session filtering exist, but duration-style older/newer filters, size-threshold filtering, and a public session-size metric are still missing from the CLI, Gateway protocol, Gateway rows, docs, and the `sessions_list` tool.

## What This Changes

The issue requests expanding session listing with agent, age, size, and structured-output filters for automation.

## Best Possible Solution

Define the public size metric first, then add additive older/newer and size filters consistently across the sessions CLI, Gateway `sessions.list` protocol and rows, `sessions_list`, JSON output, docs, and focused regression coverage.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. Source inspection on current main gives a high-confidence verification path: the relevant CLI, Gateway protocol, row, docs, and tool surfaces still expose only existing recency, agent, search/label, limit, and JSON support.

## Solution Assessment

Unclear until maintainers define the public size metric and field names. Duration filters are a narrow additive extension, but size filtering should be designed once and mirrored through all public session-list surfaces.
