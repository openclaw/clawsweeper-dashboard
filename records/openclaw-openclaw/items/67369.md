## Summary

Keep open. Current main still partially satisfies the request with agent scoping, JSON output, limits, search/label filters, and recent-session filtering, but the requested older/newer duration filters, size-threshold filter, and public session-size metadata are still absent.

## What This Changes

The issue requests `openclaw sessions` filters by agent, age, and size, plus structured JSON output for automation.

## Best Possible Solution

Define the public session-size metric first, then add additive older/newer and size filters consistently across the sessions CLI, Gateway protocol and rows, `sessions_list`, JSON output, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. Source inspection gives a high-confidence verification path: current main exposes recency, agent, search, label, limit, and JSON support, but not duration-style stale filters or size queries.

## Solution Assessment

Unclear until maintainers define the public size metric and cross-surface field names. Duration filters are a narrow additive extension, but size filtering should be designed once and mirrored through every public session-list surface.
