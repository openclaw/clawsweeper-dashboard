## Summary

Keep open. Current main still only partially satisfies the request: agent scoping, JSON output, and recent-session filtering exist, but duration-style older/newer filters, a size-threshold query, and byte-size metadata are absent from the CLI, gateway protocol/rows, gateway implementation, docs, and `sessions_list` tool surface. The related lifecycle issues are adjacent but do not supersede this query work.

## What This Changes

Request to extend session listing with agent, age, and size filters plus structured output for automation.

## Best Possible Solution

Define one public session size metric, then add additive older/newer and size query fields through the CLI, `sessions.list` protocol, gateway row metadata/filtering, `sessions_list`, docs, and focused tests for combined filters and JSON output.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. There is a high-confidence source verification path: inspect the current CLI registration, command implementation, gateway schema/row types/list implementation, agent tool schema, docs, and tests, all of which show only the existing `--active`/`activeMinutes` recency path plus agent scoping and JSON.

## Solution Assessment

Unclear until maintainers define the size metric. Older/newer duration filters are a narrow extension of existing recency filtering, but size filtering should not be implemented until the public field and threshold semantics are chosen.
