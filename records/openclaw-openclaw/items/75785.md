## Summary

Keep open. Current main still lacks persisted child-session reconciliation in the parent announce timeout path, and open PR #75786 is the linked implementation candidate for the same bug.

## What This Changes

The issue asks for subagent completion handoff to treat a stale timed-out wait as success when the persisted child session already finished successfully.

## Best Possible Solution

Land PR #75786, or an equivalent narrow change, that passes child session identity into timeout reconciliation and covers `agent.wait` timeout plus persisted `done` child state.

## Reproduction Assessment

Yes. A high-confidence reproduction can mock `agent.wait` returning `timeout` while `loadSessionEntryByKey` returns a child entry with `status: "done"`, then assert the announce event reports `ok` and delivers the child result.

## Solution Assessment

Yes. Reconciling only timeout outcomes against persisted terminal child state is the narrow maintainable fix; broad retry behavior or normal success/failure path changes would be riskier.
