## Summary

Keep open. Current main still lacks session-store reconciliation in the parent announce wait path, and the issue is paired with open PR #75786, so the right path is to review or land that PR before closing this issue.

## What This Changes

The report asks for subagent parent completion handoff to reconcile a timed-out wait with persisted child success before announcing timeout.

## Best Possible Solution

Land PR #75786, or an equivalent narrow fix, so timeout waits are upgraded only when the persisted child session is already terminal success and the behavior has regression coverage.

## Reproduction Assessment

Yes. A high-confidence reproduction can mock agent.wait returning timeout while the child session store entry is already done, then assert the announce event reports success and delivers the child result.

## Solution Assessment

Yes. Reconciling only timeout waits against persisted terminal child state is the narrow maintainable fix; a second agent.wait alone does not cover the stale wait result described here.
