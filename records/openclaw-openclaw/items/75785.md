## Summary

Keep open. Current main still trusts a timeout from the parent-side `agent.wait` path before announcing subagent completion, and #75786 is an open, unmerged closing PR for the same race.

## What This Changes

Reconcile subagent completion announce timeouts with persisted terminal child session state so successful child runs are announced as success.

## Best Possible Solution

Review and land #75786, or an equivalent narrow fix, with regression coverage for timeout reconciliation against persisted terminal child state.

## Reproduction Assessment

Yes, at source level: mock `agent.wait` returning `timeout` while the child session store entry is already `done`, then run the announce path and assert the parent event reports ok. Current main lacks that session-store reconciliation in the immediate wait path.

## Solution Assessment

Yes. The narrow maintainable fix is to recheck persisted terminal child state only after a timeout wait, leaving normal ok/error wait handling unchanged.
