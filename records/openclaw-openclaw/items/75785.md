## Summary

Keep open. Current main still lacks the session-store reconciliation described by the report, and open PR #75786 is the paired implementation candidate that should be reviewed or landed before this issue closes.

## What This Changes

The issue reports that subagent completion handoff can treat a successful child run as timed out and send only partial progress.

## Best Possible Solution

Land PR #75786, or an equivalent narrow fix, so timeout waits in parent announce reconcile against persisted terminal child state and have regression coverage.

## Reproduction Assessment

Yes. Mock agent.wait returning timeout while the child session store entry is already done, then assert the parent announce reports success and delivers the child result instead of partial progress.

## Solution Assessment

Yes. Reconciling only timeout waits against persisted terminal child state is the narrow maintainable fix; another raw agent.wait recheck does not cover the stale wait result described here.
