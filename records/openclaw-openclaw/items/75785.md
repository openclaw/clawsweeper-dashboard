## Summary

Keep open. Current main still has a source-level path where the parent announce flow accepts a raw `agent.wait` timeout without reconciling the persisted child session state, and the open unmerged closing PR is the right implementation candidate to review rather than closing this issue early.

## What This Changes

The issue asks for subagent completion announce to treat already-finished child runs as successful instead of replacing their result with a timeout partial-progress stub.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75786, or an equivalent narrow fix, so timeout wait results are reconciled against persisted terminal child session state with regression coverage.

## Reproduction Assessment

Yes, at source level: mock `agent.wait` returning `timeout` while the child session store entry is already `done`, then assert the parent announce resolves to success and includes the child result. I did not run the Vitest harness because this review is read-only.

## Solution Assessment

Yes. Reconciling only timeout wait outcomes against persisted terminal child session state is the narrow maintainable fix; normal ok/error wait handling should remain unchanged.
