## Summary

Keep open. Current main still applies a raw parent-side `agent.wait` timeout without reconciling it against the persisted child session state, and the issue is paired with open PR #75786 as the active implementation candidate.

## What This Changes

The issue reports that a successfully completed subagent can be announced to the parent as timed out when the parent-side wait result is stale.

## Best Possible Solution

Review and land PR #75786, or an equivalent narrow fix, so timeout waits are upgraded only from persisted terminal child state and covered by regression tests.

## Reproduction Assessment

Yes. A high-confidence reproduction can mock `agent.wait` returning `timeout` while the child session store already has the child entry marked `status: "done"`, then assert the parent announce reports success and includes the child result.

## Solution Assessment

Yes. Reconciling only timeout waits against persisted terminal child state is the narrow maintainable fix for the reported race; changing normal successful or failed wait handling would be broader than needed.
