## Summary

Keep open. Current main still lacks the reported timeout-to-session-store reconciliation, and the issue is already paired with open PR #75786 using closing syntax, so the safe maintainer outcome is to review and land, revise, or close that PR rather than close this issue early.

## What This Changes

The issue requests that subagent completion announces re-check persisted child session state after an `agent.wait` timeout so successful child runs are not reported to the parent as timed out.

## Best Possible Solution

Land PR #75786, or an equivalent narrow change, that only reconciles timeout wait results against a persisted terminal child session state before building the parent completion announce.

## Reproduction Assessment

Yes. A high-confidence regression can mock `agent.wait` returning `timeout` while the child session store already has a terminal successful entry, then assert the parent announce internal event reports `ok` and includes the child result.

## Solution Assessment

Yes. Reconciling timeout waits against the persisted terminal child session state is narrower than retrying general waits and directly addresses the stale status source described in the report.
