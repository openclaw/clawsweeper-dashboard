## Summary

Keep open. Current main still lacks the reported session-store reconciliation in the parent announce wait path, and the issue is already paired with open PR #75786, so the right cleanup path is to review or land that PR rather than close this issue early.

## What This Changes

The issue requests that subagent completion announces reconcile timed-out `agent.wait` results with terminal child session state before reporting a successful child run as timed out.

## Best Possible Solution

Land PR #75786, or an equivalent narrow change, that passes the child session key into timeout reconciliation and adds a regression test for `agent.wait` timeout plus persisted terminal child success.

## Reproduction Assessment

Yes. A high-confidence reproduction can mock `agent.wait` returning `timeout` while the child session store entry is already `done`, then assert the announce event reports success and delivers the child result.

## Solution Assessment

Yes. Reconciling only timeout waits against persisted terminal child state is the narrow maintainable fix; retrying `agent.wait` alone does not cover the stale wait result described here.
