## Summary

Keep open. Current main still trusts the parent-side `agent.wait` timeout in the announce path without reconciling against a persisted terminal child session, and this issue is already paired with open PR #75786, so it should be handled by reviewing or landing that PR rather than closing early.

## What This Changes

The issue asks OpenClaw to prevent subagent completion handoff from reporting a timeout when the child session has already finished successfully.

## Best Possible Solution

Land PR #75786, or an equivalent narrow fix, that reconciles timeout waits against a terminal child session entry and adds regression coverage for `agent.wait` timeout plus persisted `done` state.

## Reproduction Assessment

Yes. A focused test can mock `agent.wait` returning `timeout` while the child session store already contains a terminal successful entry, then assert the parent announce reports success and includes the child result.

## Solution Assessment

Yes. Reconciling only timeout outcomes against persisted terminal child state is the narrow maintainable fix; the current second `agent.wait` recheck does not cover the stale wait result described here.
