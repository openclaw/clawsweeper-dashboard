## Summary

Keep open. Current main still trusts the parent-side `agent.wait` timeout in the subagent announce path, and the open paired PR #75786 is the closing implementation candidate rather than shipped behavior.

## What This Changes

The issue asks OpenClaw to reconcile subagent announce timeouts with persisted child session state so a successfully finished child run is delivered as success instead of timeout.

## Best Possible Solution

Land PR #75786 or an equivalent focused fix that upgrades only stale timeout waits when persisted child session state is already terminal success, with regression coverage.

## Reproduction Assessment

Yes. A focused regression can mock `agent.wait` returning `timeout` while the child session store entry is already `done`, then assert the parent announce reports success and delivers the child result.

## Solution Assessment

Yes. Rechecking persisted child session state only after a timeout is the narrow maintainable direction; relying on a second `agent.wait` alone does not cover the stale wait result described here.
