## Summary

Keep open. Current main still trusts the parent-side `agent.wait` timeout in the subagent announce path, and #75786 is an open, unmerged closing PR for this exact race.

## What This Changes

The issue reports that parent-side subagent completion handoff can announce a timeout and emit partial progress even when the child session has already completed successfully.

## Best Possible Solution

Review and land #75786, or an equivalent narrow fix, so timeout waits are reconciled against terminal persisted child session state with regression coverage.

## Reproduction Assessment

Yes, at source level: mock `agent.wait` returning `timeout` while the child session store entry is already `done`, then assert the announce path should deliver success instead of timeout. I did not run the Vitest harness in this read-only review.

## Solution Assessment

Yes. Rechecking persisted terminal child state only after a timeout is the narrow maintainable direction; changing normal ok/error wait handling is not needed for the reported race.
