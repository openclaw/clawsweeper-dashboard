## Summary

Keep open. Current main still trusts the raw parent-side `agent.wait` timeout in the subagent announce path, and the linked open PR #75786 is the implementation candidate rather than a reason to close before merge.

## What This Changes

The issue requests that subagent completion announce success when a timed-out parent wait is stale but the persisted child session has already finished successfully.

## Best Possible Solution

Review and land PR #75786, or an equivalent narrow fix with regression coverage, then close this issue after the implementation reaches main.

## Reproduction Assessment

Yes. A high-confidence reproduction can mock `agent.wait` returning `timeout` while the child session store entry is already `done`, then assert the parent announce reports success and delivers the child result.

## Solution Assessment

Yes. Timeout-only reconciliation against persisted terminal child state is the narrow maintainable fix; repeating `agent.wait` alone does not cover the stale wait result described here.
