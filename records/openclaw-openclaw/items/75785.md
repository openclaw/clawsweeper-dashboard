## Summary

Keep open. Current main still applies the raw parent-side `agent.wait` timeout in the subagent announce path, and the open paired PR #75786 is the right implementation candidate to review before this issue is closed.

## What This Changes

The issue requests that subagent completion announcements reconcile stale timeout waits against a child session that already finished successfully.

## Best Possible Solution

Land PR #75786 or an equivalent narrow fix that upgrades only stale timeout waits when persisted child session state is already terminal success, with regression coverage.

## Reproduction Assessment

Yes. Mock `agent.wait` returning `timeout` while the child session store entry is already `done`, then assert the parent announce reports success and delivers the child result.

## Solution Assessment

Yes. Rechecking persisted terminal child state only after a timeout is the narrow maintainable direction; relying on a second `agent.wait` alone does not cover the stale wait result described here.
