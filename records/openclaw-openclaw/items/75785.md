## Summary

Keep open. Current main still trusts a raw `agent.wait` timeout in the parent-side subagent announce path, and the linked open PR is the correct implementation candidate to review before closing this issue.

## What This Changes

The issue requests that subagent completion announces reconcile a timed-out parent wait with persisted child session success before reporting a timeout.

## Best Possible Solution

Land PR #75786 or an equivalent narrow fix that upgrades only stale timeout waits when the persisted child session is already terminal success, with announce-path regression coverage.

## Reproduction Assessment

Yes. The high-confidence unit path is to mock `agent.wait` returning `timeout` while the child session store entry is already `done`, then assert the parent announce reports success and delivers the child result.

## Solution Assessment

Yes. Rechecking persisted terminal child state only after a timeout is the narrow maintainable direction; relying on another `agent.wait` call alone does not cover the stale wait result described here.
