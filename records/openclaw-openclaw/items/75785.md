## Summary

Keep open. Current main still lacks the reported session-store reconciliation path, and the issue is paired with open, unmerged PR #75786 using closing syntax, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that subagent completion handoff reconcile stale `agent.wait` timeouts against persisted child session state so successful child runs announce success and deliver their result.

## Best Possible Solution

Land PR #75786, or an equivalent narrow change, that only upgrades a timeout wait result when the persisted child session is already terminal success, with focused regression coverage for the stale-timeout path.

## Reproduction Assessment

Yes. A deterministic test can mock `agent.wait` returning `timeout` while the child session store entry is already terminal `done`, then assert the parent announce internal event reports `ok` and includes the child result.

## Solution Assessment

Yes. Reconciling timeout waits against persisted terminal child session state is the narrowest durable fix because the stale value is specifically out of sync with the stored child completion state; a blind retry would still miss the reported race when the wait result remains stale.
