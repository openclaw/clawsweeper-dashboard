## Summary

Keep open. Current main still applies the raw parent-side `agent.wait` timeout in the subagent announce path without reconciling against the persisted child session state, and this issue is paired with open unmerged PR #75786.

## What This Changes

Ensure subagent completion announcements reconcile stale `agent.wait` timeouts with persisted successful child-session completion before reporting a timeout to the parent.

## Best Possible Solution

Land PR #75786, or an equivalent narrow fix, so timeout waits are upgraded only when the persisted child session is already terminal success and the behavior has regression coverage.

## Reproduction Assessment

Yes. A high-confidence reproduction can mock `agent.wait` returning `timeout` while the child session store entry is already `status: "done"`, then assert the parent announce reports success and delivers the child result.

## Solution Assessment

Yes. Reconciling only timeout waits against persisted terminal child state is the narrow maintainable fix; blanket retries or treating every timeout as success would be less precise.
