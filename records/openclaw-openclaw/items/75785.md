## Summary

Keep open. Current main still trusts a raw `agent.wait` timeout in the parent announce path without reconciling against the persisted terminal child session state, and the issue is already paired with open PR #75786 using closing syntax.

## What This Changes

Ensure subagent completion handoff reports success and delivers child output when the child session has already finished successfully, even if `agent.wait` returns a stale timeout.

## Best Possible Solution

Land PR #75786, or an equivalent narrow fix, so timeout waits are upgraded only when the persisted child session is already terminal success and the behavior has regression coverage.

## Reproduction Assessment

Yes. A focused test can mock `agent.wait` returning `timeout` while the child session store entry is already `done`, then assert the parent announce event reports `ok` and includes the child result.

## Solution Assessment

Yes. Reconciling only timeout waits against persisted terminal child state is the narrow maintainable fix; repeating `agent.wait` alone does not cover the stale wait result described here.
