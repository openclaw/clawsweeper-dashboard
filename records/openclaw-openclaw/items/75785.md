## Summary

Keep open. Current main still lets a timeout from `agent.wait` become the parent announce outcome without consulting the persisted child session state, and #75786 is already the open closing PR for this race.

## What This Changes

The issue asks OpenClaw to reconcile parent-side subagent wait timeouts with persisted child success so completed child results are not replaced by partial-progress timeout stubs.

## Best Possible Solution

Review and land #75786, or an equivalent narrow fix, with regression coverage for a stale `agent.wait` timeout when the persisted child session is already terminal success.

## Reproduction Assessment

Yes, at source level: mock `agent.wait` returning `timeout` while the child session store entry is already `done`, then assert the announce path reports success and delivers the child result. I did not run tests because this review was read-only.

## Solution Assessment

Yes. Rechecking persisted terminal child state only after a timeout is the narrow maintainable fix; changing normal ok/error wait handling is not needed for this race.
