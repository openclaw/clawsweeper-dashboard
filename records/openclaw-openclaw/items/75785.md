## Summary

Keep open. Current main still lacks the specific session-store reconciliation described in the report, and the issue is already paired with open PR #75786 using closing syntax, so the right cleanup outcome is to review and land or close that PR rather than close this issue early.

## What This Changes

Fix the subagent announce path so a parent does not report a timeout when the child session has already completed successfully.

## Best Possible Solution

Land PR #75786, or an equivalent narrow change, that reconciles only timeout wait results against a persisted terminal child session state before building the parent announce status and result.

## Reproduction Assessment

Yes. A deterministic reproduction can mock `agent.wait` returning `timeout` while the child session store entry is already terminal success, then assert the announce internal event reports `ok` instead of `timed out`.

## Solution Assessment

Yes. Reconciling timeout waits against persisted terminal child session state is narrower and more durable than another `agent.wait` retry, because the stale value being reported is specifically out of sync with stored child completion state.
