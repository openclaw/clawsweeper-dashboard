## Summary

Keep open. Current main still does not reconcile a timed-out parent announce wait against persisted child session success, and this issue is already paired with open PR #75786 rather than shipped behavior.

## What This Changes

The issue requests that subagent completion announce logic treat a stale `agent.wait` timeout as success when the persisted child session has already completed successfully.

## Best Possible Solution

Land PR #75786, or an equivalent narrow change, that rechecks persisted terminal child session state only after an `agent.wait` timeout and adds focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence reproduction can mock `agent.wait` returning `timeout` while the child session store entry is already `done`, then assert the parent announce reports success and includes the child result.

## Solution Assessment

Yes. Timeout-only reconciliation against persisted child state is the narrow maintainable fix; retrying `agent.wait` alone does not cover the stale wait result described here.
