## Label Justifications

- `P1`: The report describes a worker memory leak that can restart gateway sessions and break long-running agent workflows.
- `impact:session-state`: The retained per-session/per-turn context can pin agent session state and kill long-running subagent sessions on restart.
- `impact:crash-loop`: The issue reports RSS growth high enough to force worker recycle and service restarts.

## Summary

Keep open: current main still has a credible retention path for full per-run prompt reports in session entries, and the newer retainer comment matches source-level closures and fields rather than a stale guess. Existing cache TTLs and maintenance lower some risk, but they do not prove the reported long-session worker growth is fixed.

## What This Changes

The issue reports gateway worker RSS growth from retained per-turn bootstrap/system prompt context, especially for cron and long-lived agent sessions.

## Best Possible Solution

Bound or shrink retained session prompt reports for completed and ephemeral cron/turn paths while preserving explicit lightweight context diagnostics for `/context` and usage APIs.

## Reproduction Assessment

No, not as a live reproduction in this read-only review. Source inspection gives a medium-confidence reproduction path because per-run prompt reports are built, persisted onto session entries, and reachable through session-store/transcript-recorder paths that match the heap retainer comment.

## Solution Assessment

Unclear. Evicting or summarizing the report is likely the right direction, but maintainers should first decide the durable diagnostic contract for `/context` and `includeContextWeight`.
