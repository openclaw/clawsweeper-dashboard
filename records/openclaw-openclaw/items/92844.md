## Summary

Keep open: the report is credible and current main still retains completed agent results, including per-turn systemPromptReport data, in the gateway dedupe cache; existing TTL/max eviction lowers the risk but does not prove the reported worker growth is fixed.

## What This Changes

The issue reports gateway worker RSS growth from retained per-turn bootstrap/systemPromptReport objects in an apparent unbounded cache.

## Best Possible Solution

Keep this open until gateway terminal dedupe no longer retains heavy per-turn bootstrap/systemPromptReport payloads longer than needed while preserving documented agent and agent.wait replay semantics.

## Reproduction Assessment

No high-confidence live reproduction was run here. The issue has strong heap-snapshot evidence and current source shows a plausible retention path through gateway dedupe, but the exact current-main unbounded behavior still needs a long-session memory repro.

## Solution Assessment

Unclear. Evicting or sharing bootstrap snapshots is plausible, but current main already has TTL/max dedupe cleanup, so the best fix should first identify the retaining map and then shrink the retained terminal payload without breaking idempotent replay.
