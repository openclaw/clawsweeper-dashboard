## Summary

Keep open: this maintainer-labeled PR is an active fix path for #76957, and source review found the current-main regression while the proposed explicit hook opt-in looks narrow with sufficient live Gateway proof and no blocking findings.

## What This Changes

Adds an optional `emitCommandHooks` field to `sessions.create`, sends it from Control UI `/new` when there is a parent session, emits the command/reset/session lifecycle hooks on that path, and updates tests, Swift protocol mirrors, and the changelog.

## Best Possible Solution

Land this PR after maintainer approval and exact-head merge checks if maintainers accept the additive protocol flag, leaving SDK parent-session creates hook-free by default.

## Reproduction Assessment

Yes. Current main documents `command:new` for `/new`, but typed Control UI `/new` reaches `sessions.create`, whose current-main implementation has no hook emission path.

## Solution Assessment

Yes. The explicit `emitCommandHooks` opt-in is the narrowest maintainable fix because it restores Control UI `/new` hooks without making every SDK `parentSessionKey` create impersonate a webchat command.
