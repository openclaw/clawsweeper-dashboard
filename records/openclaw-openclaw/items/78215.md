## Summary

Keep open: this maintainer-labeled PR addresses a source-reproducible Control UI `/new` hook regression with no blocking review findings, and the live Gateway proof matches the intended opt-in behavior.

## What This Changes

Adds an optional `emitCommandHooks` flag to `sessions.create`, wires Control UI new-session creation to pass it for real parent sessions, emits the command/reset lifecycle hooks on that path, and updates tests, Swift protocol mirrors, and the changelog.

## Best Possible Solution

Land this PR after normal maintainer review and merge checks if maintainers accept the additive protocol flag; it should close #76957 while preserving programmatic `sessions.create` defaults.

## Reproduction Assessment

Yes. Source inspection on current main shows typed Control UI `/new` reaches `sessions.create` with `parentSessionKey`, while `sessions.create` has no hook-emission path despite the docs requiring `command:new` for `/new`.

## Solution Assessment

Yes. The explicit `emitCommandHooks` opt-in is the narrowest maintainable fix because it restores Control UI `/new` hooks without repeating the over-broad `parentSessionKey` behavior called out in earlier attempts.
