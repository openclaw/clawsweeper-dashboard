## Summary

Keep open. Current main still leaves the reported Playwright `CRSession._onMessage` assertion class unclassified: browser CDP attachment remains active, Playwright 1.60.0 still asserts on unexpected response ids, and the Gateway still exits for unknown unhandled rejections.

## What This Changes

Prevent Playwright CDP `CRSession._onMessage` assertion rejections from crashing the Gateway and losing unrelated session work.

## Best Possible Solution

Add a browser-plugin-owned, narrowly matched classifier or containment path for the reported Playwright `CRSession._onMessage` unmatched-response assertion, with positive and negative regression coverage while preserving the fatal fallback for unknown rejections.

## Reproduction Assessment

No high-confidence live reproduction path has been established. The source path is clear: current main still attaches via `connectOverCDP`, upstream Playwright 1.60.0 still asserts in `CRSession._onMessage` for unexpected response ids, and OpenClaw still exits for unclassified unhandled rejections.

## Solution Assessment

Yes, a narrow browser-owned rejection classifier is the best first repair for the concrete crash signature. Broadly swallowing Playwright errors or moving browser/CDP into a subprocess would be a larger architecture decision and should not be bundled into this bug fix.
