## Summary

Keep open: current main and the latest release still route unclassified Playwright CDP assertion rejections to the fatal process handler, and I did not find a merged fix that covers the reported `CRSession._onMessage` crash.

## What This Changes

The issue asks OpenClaw to prevent Playwright CDP assertion failures from crashing the Gateway and losing unrelated in-flight agent/session work.

## Best Possible Solution

Preserve the fatal policy for unknown process errors, but isolate or narrowly classify recoverable Playwright CDP assertion failures so browser/CDP faults degrade browser capability without killing unrelated Gateway sessions.

## Reproduction Assessment

No, not as a high-confidence live repro. The stack maps cleanly to current source and Playwright 1.60.0 still has the asserted `CRSession._onMessage` path, but the exact CDP message trigger was not reproduced on current main.

## Solution Assessment

Unclear. The issue’s suggested global catch is too broad; a scoped browser/CDP isolation boundary or tightly matched recoverable-error classifier looks safer but needs maintainer agreement and live proof.
