Keep open. ## What This Changes

Prevent Playwright CDP assertion failures from escaping as fatal unhandled rejections and crashing the Gateway process.

## Best Possible Solution

Add a browser-plugin-owned, narrowly matched handler for the reported Playwright CDP assertion class, with positive and negative regression coverage, rather than globally swallowing all Playwright assertions or requiring browser subprocess isolation first.

## Reproduction Assessment

No high-confidence live reproduction is available from this review. Source inspection does show the reported dependency assertion path still exists in Playwright 1.59.1 and would fall through OpenClaw's current fatal unhandled-rejection handler unless a registered browser handler consumes it.

## Solution Assessment

Yes, a narrow browser-owned rejection classifier is the best first fix. A broad global Playwright swallow or child-process isolation would be riskier and larger than needed for this concrete crash signature.
