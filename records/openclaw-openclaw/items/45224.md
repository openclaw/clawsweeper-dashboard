## Summary

Keep open. Current main and the latest shipped tag still leave the reported Playwright `CRSession._onMessage` assertion unclassified: browser CDP connections still use `connectOverCDP`, the browser plugin only suppresses a dialog-race Playwright rejection, and the Gateway still exits for unknown unhandled rejections.

## What This Changes

The issue asks OpenClaw to prevent a Playwright CDP `CRSession._onMessage` assertion from crashing the Gateway and losing in-flight session state.

## Best Possible Solution

Add a browser-plugin-owned, narrowly matched containment/classifier for the reported `CRSession._onMessage` unmatched-response assertion, with positive and negative regression coverage while preserving fatal handling for unknown rejections.

## Reproduction Assessment

No, there is not a high-confidence live current-main reproduction for the exact CDP message that triggers `CRSession._onMessage`. The source path is reproducible by inspection: current main still uses `connectOverCDP`, upstream Playwright still asserts on unexpected response ids, and OpenClaw still exits for unclassified unhandled rejections.

## Solution Assessment

Yes, a narrow browser-owned rejection classifier or containment path is the best first fix for this concrete crash signature. A browser subprocess redesign or broad global Playwright swallow would be a larger product/architecture change and should not be bundled into the focused bug fix.
