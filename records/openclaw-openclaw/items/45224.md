## Summary

Keep open. Current main and latest v2026.6.6 still route Playwright-backed CDP profiles through `connectOverCDP`, and unclassified unhandled rejections still exit the Gateway; the newer Chrome MCP existing-session path is a useful mitigation for signed-in browser sessions, but it does not prove this crash class is fixed for local-managed or remote CDP profiles.

## What This Changes

Prevent Playwright CDP assertion errors from crashing the Gateway and killing unrelated active sessions.

## Best Possible Solution

Keep Chrome MCP/existing-session as the real-browser path, then add targeted browser-owned isolation for Playwright-backed CDP failures after a live repro, with tests proving the Gateway degrades browser control instead of exiting.

## Reproduction Assessment

No high-confidence live reproduction was established. Source and dependency inspection show the risky path remains for Playwright-backed CDP profiles, but the exact current-main runtime trigger still needs live CDP proof.

## Solution Assessment

Unclear. A broad global Playwright swallow is not the best default; maintainers should choose a targeted browser-boundary fix or process isolation shape backed by live repro and regression tests.
