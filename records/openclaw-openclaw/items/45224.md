## Summary

Keep open. Current main still uses Playwright CDP for browser profile attachment, only suppresses the narrow dialog-race rejection class, and the upstream Playwright 1.60.0 `CRSession._onMessage` path still asserts on unexpected response ids that OpenClaw will treat as a fatal unhandled rejection if it is not classified.

## What This Changes

The issue reports that an unhandled Playwright CDP `CRSession._onMessage` assertion can crash the Gateway process and lose unrelated in-flight session work.

## Best Possible Solution

Add a browser-plugin-owned, narrowly matched classifier for the reported Playwright `CRSession._onMessage` unmatched-response assertion and cover positive and negative cases while leaving broader browser subprocess isolation as a separate architecture discussion.

## Reproduction Assessment

No high-confidence live reproduction path was established here. The source path is clear: Playwright CDP attachment remains active, upstream Playwright 1.60.0 still has the `CRSession._onMessage` assertion, and OpenClaw still exits for unclassified unhandled rejections.

## Solution Assessment

Yes, a narrow browser-owned rejection classifier is the best first repair for the concrete crash signature. A gateway-wide Playwright swallow or browser subprocess redesign would be broader and riskier than needed for this specific assertion path.
