## Summary

Keep open. Current main still leaves the reported Playwright `CRSession._onMessage` / `CRConnection._onMessage` assertion outside the browser plugin's narrow dialog-race suppression, so it reaches the Gateway fatal unhandled-rejection exit path.

## What This Changes

The issue asks OpenClaw to prevent Playwright CDP `CRSession._onMessage` assertion rejections from crashing the Gateway when browser control is connected.

## Best Possible Solution

Add a browser-owned CDP assertion guard keyed to Playwright `CRSession`/`CRConnection` stack markers that logs or degrades browser control while preserving fatal handling for unrelated rejections.

## Reproduction Assessment

Yes, by source inspection. Registering the browser runtime handler and passing an `Assertion error` with `CRSession._onMessage` or `CRConnection._onMessage` stack text would currently leave it unhandled, after which the Gateway fatal path exits.

## Solution Assessment

Yes for the near-term repair. Extending the existing browser plugin `registerUnhandledRejectionHandler` seam is narrower than adding a process-wide catch-all; subprocess isolation can remain a larger architecture follow-up.
