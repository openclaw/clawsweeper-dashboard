## Summary

Keep open. Current main still defines only operator/node Gateway roles and has no federation RPC or peer-routing surface; related #43605 and #43656 are open and narrower, so they do not supersede this broader federation/HA request.

## What This Changes

Request support for Gateway-to-Gateway federation, including authenticated peer Gateways and routed cross-Gateway messages.

## Best Possible Solution

Keep this as a core Gateway/protocol design item until maintainers define the federation trust model, peer identity, routing semantics, configuration surface, and compatibility plan.

## Reproduction Assessment

Not applicable as a runtime bug. The missing capability is high-confidence from source and docs: current main rejects `role: "gateway"`, lacks a federation RPC, and documents isolated multi-Gateway operation.

## Solution Assessment

Unclear. The requested `role: "gateway"` plus `gw.send` direction is plausible, but the maintainable solution needs a reviewed core protocol/security design; #43656 is only a narrower open candidate for session-tool routing.
