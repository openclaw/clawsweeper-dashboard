## Security Review

Status: needs_attention

Summary: Gateway federation is security-sensitive because it creates a new cross-Gateway control-plane trust boundary.

Concerns:

- **[medium] Define peer trust before federation:** `docs/gateway/protocol.md:10`
  - body: Cross-Gateway routing could let one Gateway send session messages, trigger agent work, or expose control-plane state on another, so the design needs fail-closed peer identity, mutual auth, URL allowlisting, scoped authorization, transcript ownership, and visibility semantics.
  - confidence: 0.9
- **[medium] Protect remote session orchestration surfaces:** `docs/gateway/tools-invoke-http-api.md:116`
  - body: The documented tool policy treats sessions_spawn as remote orchestration and sessions_send as message injection, so peer routing must preserve or replace those safety boundaries explicitly.
  - confidence: 0.88
