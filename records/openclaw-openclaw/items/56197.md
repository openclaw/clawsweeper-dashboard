The issue reports that Feishu embedded-agent replies can expose a raw provider HTTP 401 invalid-token error instead of retrying with refreshed auth or showing safe re-authentication guidance.

## Best Possible Solution

Rebase, review, and land the linked safe-401 masking PR if maintainers accept its scope, then handle automatic Z.AI/AutoGLM refresh-and-retry as a separate provider-owned runtime-auth design.

## Solution Assessment
