## Summary

Keep open. Current main still has the source-level path for surfacing plain HTTP 401 invalid-token text, and this issue is paired with open PR #77394 using closing syntax, so cleanup closure would be premature.

## What This Changes

When Feishu embedded agents hit an expired Z.AI/AutoGLM token, retry with refreshed auth or at least avoid returning the raw HTTP 401 text to the user.

## Best Possible Solution

Review and either land or revise PR #77394 for the safe user-facing error text, then separately decide whether Z.AI/AutoGLM can own a `prepareRuntimeAuth` refresh path.

## Reproduction Assessment

No, not as a live high-confidence reproduction. The issue gives concrete logs and current source shows the raw HTTP 401 fallthrough path, but I did not establish a live Feishu/AutoGLM expiry run on current main.

## Solution Assessment

Unclear for the full retry request. The narrow friendly-error fix in the linked PR is maintainable, while automatic refresh should stay provider-owned through `prepareRuntimeAuth` only if Z.AI/AutoGLM has a deterministic token exchange.
