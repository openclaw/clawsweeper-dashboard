## Summary

Keep open: current main and the latest release still have the colliding cwd-to-session-directory encoding and cwd-blind recent-session selection; the linked candidate fix PR is open and should be reviewed or replaced before this issue closes.

## What This Changes

The issue reports that distinct project directories such as `/home/alice/dev/client/app` and `/home/alice/dev/client-app` can share one encoded session directory, causing `continueRecent` to resume the wrong project's session state.

## Best Possible Solution

Keep this issue open until an upgrade-safe session-manager fix lands, preferably preserving shipped session discovery while making resume/list selection cwd-aware or explicitly migrating old directories.

## Reproduction Assessment

Yes. Source inspection shows the exact encoder collision and the resume path's lack of a stored-cwd filter; I did not run a mutating on-disk repro in this read-only review.

## Solution Assessment

Yes, the right fix boundary is the shared session-manager cwd lookup and recent-session selection path. The current linked PR is a plausible candidate, but it needs compatibility-safe repair before it becomes the best landed solution.
