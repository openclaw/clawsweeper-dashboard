## Label Justifications

- `P2`: This is a deterministic session-state correctness bug with limited blast radius to projects whose cwd paths collide under the current encoder.
- `impact:session-state`: The failure can silently associate one project's transcript and context with a different requested cwd during resume.

## Summary

Keep open: current main and the latest release still use the non-injective cwd session-directory encoder, and the open candidate fix is not merged yet.

## What This Changes

The issue requests that default project sessions keep distinct cwd paths separate and avoid resuming a transcript whose recorded cwd does not match the requested project.

## Best Possible Solution

Review and repair or replace https://github.com/openclaw/openclaw/pull/96547 so session lookup becomes cwd-safe without orphaning existing project transcripts, then close this issue after the fix merges.

## Reproduction Assessment

Yes, at source level: current main still maps the example cwd pair to the same default session directory and `continueRecent` still resumes the newest valid file without a header-cwd check. I did not run an artifact-writing repro because this review is read-only.

## Solution Assessment

Yes, the right boundary is the shared session manager lookup path. The best fix should make default resume/list/fork behavior cwd-safe while preserving or migrating existing shipped session directories.
