## Summary

Current main already addresses the reported Control UI prompt/heartbeat race: active-run history reloads are deferred, busy prompts are visibly queued, and queued prompts have steer/remove controls. The same report was also handled as implemented in a duplicate thread, so keeping this open would track solved work.

## What This Changes

The issue requests visible, cancelable prompt queuing when a user prompt races with a heartbeat in the same channel.

## Best Possible Solution

Keep the shipped Control UI implementation and close this duplicate request; any narrower remaining queue polish should be tracked in a fresh issue with current-main reproduction evidence.

## Reproduction Assessment

No current-main failure was reproduced in this read-only pass. The old race has high-confidence source and PR-body reproduction evidence, and current tests now cover the deferred reload and visible queue paths.

## Solution Assessment

Yes. The current UI-side solution is the narrow maintainable fix: preserve optimistic user messages, defer racing history reloads, and surface queued prompts with steering/removal controls.
