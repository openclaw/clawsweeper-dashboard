## Summary

Keep open: the PR is useful directionally, and the Greptile edge-case comments appear addressed, but the current Telegram wiring still cannot satisfy the linked multi-bot repro because skipped @otherbot history is detected against only the current bot and the added byte-stability test is stale against current main.

## What This Changes

The PR adds optional bot-targeting metadata to auto-reply history/context, renders it in inbound prompt JSON, and adds Telegram parsing/session plumbing plus tests.

## Best Possible Solution

Keep the additive metadata contract, but finish Telegram with an approved sibling-bot username source, rebase the prompt baseline test onto current main, and add the required changelog entry before merge.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: current main stores mention-skipped Telegram group history without target metadata, and this PR still cannot classify a skipped @bot-b message for bot-a without a sibling-bot source.

## Solution Assessment

No. The core field shape is reasonable, but the proposed Telegram wiring is incomplete for the linked repro until it can identify sibling-bot mentions or represent that state explicitly.
