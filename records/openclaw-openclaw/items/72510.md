## Summary

## What This Changes

This PR adds optional bot-targeting fields to auto-reply history/template context, renders them in inbound prompt metadata, and wires Telegram parsing/session plumbing plus focused tests.

## Best Possible Solution

Keep the additive metadata contract, but finish Telegram with an approved sibling-bot username source, update the rebased prompt baseline, and add the required changelog entry before merge.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: current main records mention-skipped Telegram group history without target metadata, and this PR still cannot mark a skipped '@bot-b' history entry for bot-a without a sibling-bot source.

## Solution Assessment

No. The core field shape is reasonable, but the Telegram implementation is incomplete for the linked reproduction until it can identify sibling-bot mentions or explicitly model that unresolved state.
