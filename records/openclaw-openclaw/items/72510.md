## Summary

Keep this PR open. The underlying Telegram group-chat bug is real from source inspection, but the latest branch still does not solve the linked sibling-bot repro because the skipped-history detector only knows the current bot, and the external PR has no real Telegram proof.

## What This Changes

The PR adds optional bot-targeting metadata to auto-reply history/context, renders it in prompt metadata, and wires Telegram parsing/session plumbing plus tests.

## Best Possible Solution

Keep the additive metadata direction, but finish Telegram with an approved sibling-bot source, update the prompt tests against current main, and require live Telegram proof before merge.

## Reproduction Assessment

Yes for a source-level reproduction: current main stores skipped Telegram group history as flat entries, and the PR still cannot classify a skipped @bot-b message for bot-a because no sibling-bot usernames are supplied. No live Telegram run was performed in this read-only review.

## Solution Assessment

No. The metadata contract is plausible, but the proposed Telegram wiring leaves the required sibling-bot source undefined, so it does not fully solve the linked multi-bot repro.
