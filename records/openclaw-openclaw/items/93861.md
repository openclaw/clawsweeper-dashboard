## Best Possible Solution

Preserve newer-visible stale suppression while fixing truncation, likely with delivery-phase or lazy foreground-fence state that prevents both data loss and out-of-order replay.

## Reproduction Assessment

## Solution Assessment

No. Guarding only on `activeGenerations` tracks whole-dispatch lifetime rather than visible-delivery freshness, so a corrected delivery-phase or lazy-fence fix should avoid truncation without replaying stale older output.
