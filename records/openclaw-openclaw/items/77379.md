## Summary

Keep open: current main still performs the installed-plugin-record fallback read before loader-cache reuse when callers do not supply `installRecords`, and this PR is a narrow complement to the merged snapshot-threading work in https://github.com/openclaw/openclaw/pull/85026 with sufficient real behavior proof and no blocking findings.

## What This Changes

Adds a process-local memo for plugin install records in `src/plugins/loader.ts` and clears it when plugin loader or registry caches are invalidated.

## Best Possible Solution

Land the memoized fallback after focused loader-cache validation, keeping the merged snapshot-threaded `installRecords` path as preferred and using the memo only when callers have no snapshot records.

## Reproduction Assessment

Yes, at source level. Current main still calls the synchronous installed-record fallback before loader-cache reuse when `options.installRecords` is absent, and the PR discussion includes after-patch timing data from a real Linux VPS Discord setup.

## Solution Assessment

Yes. Memoizing only the fallback read is narrower than duplicating the merged snapshot-threading path, and extending the existing loader/registry invalidation hooks is the right freshness boundary for normal plugin install mutations.
