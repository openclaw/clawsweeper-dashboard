## Summary

Keep open for maintainer merge review: current main still has the fallback synchronous installed-record read before plugin-loader cache reuse, while this PR narrowly memoizes that remaining fallback and complements https://github.com/openclaw/openclaw/pull/85026.

## What This Changes

The PR adds a process-local memo for fallback installed-plugin install records in `src/plugins/loader.ts`, uses it when `PluginLoadOptions.installRecords` is absent, and clears it with loader/registry cache invalidation.

## Best Possible Solution

Land this narrow fallback memoization, or an equivalent invalidation-aware version, while keeping the merged snapshot-threaded `installRecords` path as the preferred path.

## Reproduction Assessment

Yes. Source inspection on current main shows the sync fallback read happens before reusable loader-cache lookup when no `installRecords` are passed, and the linked discussion includes live Linux VPS timing that matches that path.

## Solution Assessment

Yes. The patch is narrow because it memoizes only the no-snapshot fallback left after https://github.com/openclaw/openclaw/pull/85026, with existing install/config mutation paths clearing the registry load cache.
