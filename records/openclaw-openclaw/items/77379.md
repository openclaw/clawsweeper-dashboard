## What This Changes

The PR memoizes the plugin install-records fallback read in `src/plugins/loader.ts` and clears that memo when loader or registry caches are invalidated.

## Best Possible Solution

Land the memoized fallback after focused loader-cache validation, keeping the snapshot-threaded `installRecords` path as preferred and using the memo only when callers have no snapshot records.

## Reproduction Assessment

## Solution Assessment

Yes. Memoizing only the fallback read is narrower than duplicating the merged snapshot-threading path, and normal plugin install/uninstall flows already route through registry-cache invalidation.
