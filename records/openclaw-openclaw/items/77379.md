## Summary

Keep open: current main still has a fallback synchronous install-record read when callers do not supply `options.installRecords`, and this PR remains a narrow complement to the merged snapshot-threading work in https://github.com/openclaw/openclaw/pull/85026.

## What This Changes

The PR memoizes the plugin loader fallback install-record read and clears that memo when loader or registry caches are invalidated.

## Best Possible Solution

Land the memoized fallback after focused loader-cache validation, keeping the snapshot-threaded `installRecords` path from https://github.com/openclaw/openclaw/pull/85026 as the preferred path.

## Reproduction Assessment

Yes, at source level. Current main still performs the fallback sync install-record read before the loader cache lookup when `options.installRecords` is absent, and the discussion includes matching runtime timing evidence.

## Solution Assessment

Yes. Memoizing only the fallback read is narrower than duplicating the merged snapshot-threading path, and existing plugin registry invalidation is the right place to refresh the memo.
