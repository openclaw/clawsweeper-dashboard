## Summary

Keep open: current main still has the fallback synchronous install-records read when callers do not supply install records, and this PR remains a small unique fix that complements the merged snapshot-threading work. I found no blocking patch defect.

## What This Changes

The PR memoizes the fallback installed-plugin install-records read in `src/plugins/loader.ts` and clears that memo when loader or registry caches are invalidated.

## Best Possible Solution

Land the memoized fallback after focused plugin-loader cache validation, preserving the snapshot-threaded `installRecords` path as preferred and using the memo only when callers have no supplied records.

## Reproduction Assessment

Yes. Current main shows the synchronous fallback read is evaluated while building the loader cache context before registry-cache reuse, and the PR discussion includes Linux Discord timing evidence for the same path.

## Solution Assessment

Yes. Memoizing only the fallback read is narrower than duplicating the merged snapshot-threading work, and normal plugin install/uninstall flows already route through registry-cache invalidation.
