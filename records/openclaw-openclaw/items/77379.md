## Summary

Keep open: current main still has the synchronous fallback read when `installRecords` is absent, while the merged snapshot-threading PR only covers callers that already pass records. This PR is a narrow fallback memoization with sufficient real-behavior proof and no blocking review findings.

## What This Changes

The PR adds an env-scoped in-process memo for fallback install records in `src/plugins/loader.ts` and clears it when plugin loader or registry caches are invalidated.

## Best Possible Solution

Land the fallback memoization after focused loader-cache validation, keeping snapshot-threaded records as the preferred path and using the memo only for callers without supplied records.

## Reproduction Assessment

Yes. Source inspection on current main shows the sync fallback runs before cache reuse when `options.installRecords` is absent, and the PR discussion includes matching before/after runtime timing evidence.

## Solution Assessment

Yes. Memoizing only the fallback read is narrower than duplicating the merged snapshot-threading path, and the patch resets the memo through the existing loader and registry cache invalidation hooks.
