## Summary

Keep open: current main and the latest shipped tag still use a workspace-keyed skills watcher that re-watches shared skills roots per agent workspace, and the linked closing PR is open but not merged yet.

## What This Changes

The issue requests fixing multi-agent skills watcher fan-out so shared skills directories are watched once per gateway/directory instead of once per agent workspace.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/85130 or an equivalent path-keyed watcher registry with subscriber fan-out and regression tests for single shared watcher ownership plus multi-workspace invalidation.

## Reproduction Assessment

Yes at source level: current main creates one Chokidar watcher per workspace while including the same shared skills roots in every workspace target set. I did not run a live 25-plus-agent EMFILE reproduction in this read-only review.

## Solution Assessment

Yes: deduplicating watchers by normalized watched directory with workspace subscriber fan-out is the narrow maintainable fix. Raising `LimitNOFILE` or disabling `skills.load.watch` remains a workaround, not the best product behavior.
