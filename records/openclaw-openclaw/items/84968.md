## Summary

Keep open: current main still creates workspace-scoped skills watchers that include shared managed and extra skill roots, so the reported multi-agent EMFILE cascade is still source-supported and not covered by the existing traversal-bound fixes.

## What This Changes

The issue asks OpenClaw to prevent multiple agents from creating duplicate watchers for the same shared skills bundle directory, which can exhaust file descriptors and leave sessions stuck after bundle-MCP disposal.

## Best Possible Solution

Implement a gateway-level skills watcher registry keyed by normalized watched root or target set, with subscriber fan-out that bumps the affected workspace snapshot versions without changing the default live-reload contract.

## Reproduction Assessment

Yes for the core watcher fan-out: current source shows each distinct workspace creates its own watcher while including the same shared skill roots. I did not run a live 28-agent EMFILE load reproduction in this read-only review.

## Solution Assessment

Yes: shared watcher dedupe with workspace subscriber fan-out is the narrow maintainable fix. Raising `LimitNOFILE` or disabling `skills.load.watch` is useful operational mitigation, not the best product behavior.
