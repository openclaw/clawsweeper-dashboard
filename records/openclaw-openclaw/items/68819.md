## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Match Node's Windows PATH de-duplication:** `src/process/windows-command.ts:26-28`
  - body: `resolveWindowsPathEnv` picks the first insertion-order `PATH`/`Path` variant, but Node sorts env keys and keeps the first case-insensitive duplicate before spawning. With duplicate PATH keys this helper can inspect a different PATH than the child receives and rewrite to the wrong binary; mirror Node's env-key normalization before resolving shims.
  - confidence: 0.88
- **[P2] Preserve libuv's direct-spawn precedence:** `src/process/windows-command.ts:47-52`
  - body: `hasExeOnPath` only checks PATH for `<name>.exe`, while libuv checks the current directory and then each PATH entry for `.com` and `.exe` before spawn. If a cwd executable or `.com` would win, this code can still unwrap a later `.cmd` shim and run a different target than direct spawn would have selected.
  - confidence: 0.82
