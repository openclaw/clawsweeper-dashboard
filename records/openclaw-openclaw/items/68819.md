  - body: This branch now throws before spawn whenever a bare Windows command cannot be resolved and the explicit child env has no PATH.   - body: `getEnvironmentValue` returns the first case-insensitive key match even when that value is `undefined`, so an env object like `{ PATH: undefined, Path: "C:\\bin" }` is treated as having no PATH. Since `toStringEnv` and Windows process envs can preserve a later valid mixed-case key, continue scanning on undefined values before deciding the PATH is empty.
