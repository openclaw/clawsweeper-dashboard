## Summary

Keep open. Current main still enforces an any-blocked-address DNS policy before IPv4-first pinning, so the reported mixed public IPv4 plus special-use IPv6 failure remains source-reproducible and the closed fix PRs did not land.

## What This Changes

The issue asks for `web_fetch` SSRF DNS pinning to allow a hostname when at least one resolved address is allowed while excluding blocked special-use IPv6 addresses from the pinned pool.

## Best Possible Solution

Implement a security-reviewed filter-then-pin change in `src/infra/net/ssrf.ts`: keep strict host policy, discard blocked resolved addresses, throw only when none remain, and cover both normal pinned DNS and dispatcher override paths with focused regression tests.

## Reproduction Assessment

Yes, at source level. A lookup that returns a public IPv4 plus `::`, `::1`, Teredo, or another blocked special-use IPv6 reaches `assertAllowedResolvedAddressesOrThrow()` before IPv4-first pinning can select the safe public address.

## Solution Assessment

Yes in direction, but it needs security review. The best fix is not a broad SSRF relaxation or a config workaround; it is filtering the resolved address pool before pinning and preserving all-blocked denial.
