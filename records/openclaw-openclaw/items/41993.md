## Summary

Keep open. Current main and the latest release still reject mixed DNS answer sets as soon as any resolved address is blocked, before IPv4-first pinning can select a safe public address; the remaining work is real but security-sensitive.

## What This Changes

Issue asks OpenClaw to let `web_fetch` proceed when DNS returns at least one allowed public address alongside blocked IPv6 special-use answers, while still denying all-blocked results.

## Best Possible Solution

Approve and implement a narrow filter-then-pin SSRF change: keep strict hostname checks, discard blocked DNS answers from normal and override pools, deny all-blocked sets, and add focused regression coverage.

## Reproduction Assessment

Yes, at source level. A lookup result containing a public IPv4 plus `::`, `::1`, Teredo, or another blocked IPv6 answer reaches `assertAllowedResolvedAddressesOrThrow()` before IPv4-first pinning can select the safe address.

## Solution Assessment

Yes in direction, but not as a broad allow-any-mixed-result change. The maintainable fix is to filter blocked resolved addresses before pinning while preserving all-blocked denial and dispatcher override safety.
