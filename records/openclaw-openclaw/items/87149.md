The PR adds a QQBot `MediaApi.uploadMedia()` direct URL validator that parses HTTP(S) URLs, runs the SDK SSRF hostname/IP preflight before token lookup or QQ API upload, and adds focused media API regression tests.

## Best Possible Solution

Keep the sink-level SSRF guard, preserve documented public HTTP(S) direct uploads as the latest head does, update the PR body to match the final policy, and require redacted real behavior proof or maintainer-provided QQBot validation before merge.
