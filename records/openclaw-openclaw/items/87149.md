## Summary

This PR is still useful because current main forwards QQBot direct media URLs without validation, but the latest patch still has a blocking security bug: it allows a literal RFC 2544 benchmark-range URL to be forwarded to the QQ upload API instead of limiting the fake-IP allowance to DNS answers for public hostnames.

## What This Changes

The PR adds QQBot direct media URL validation in `MediaApi.uploadMedia()` and new focused tests for public HTTP(S), invalid schemes, SSRF-blocked targets, and fake-IP behavior.

## Best Possible Solution

Keep the new direct-upload sink guard, but reject literal private/internal/special-use URL hosts before applying the RFC 2544 allowance only to DNS answers for public hostnames.

## Solution Assessment
