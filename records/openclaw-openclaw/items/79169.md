## Summary

Keep open: current main still models safe-bin executable trust by trusted directories, and adding exact path or SHA-256 trust would be a new security-sensitive config surface that overlaps documented exec allowlist behavior.

## What This Changes

The issue requests a `tools.exec.safeBinTrustedBinaries`-style config to trust specific safe-bin executable paths and optionally pin SHA-256 hashes.

## Best Possible Solution

Decide whether the existing host exec allowlist is the intended per-path trust answer; if not, design a safe-bin-scoped binary/hash contract with schema, docs, audit/doctor support, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main supports directory-level safe-bin trust plus separate host allowlist path trust, but no per-binary safe-bin path or hash config.

## Solution Assessment

Unclear. The proposed field could reduce coarse directory trust, but the safer maintainer path is product/security design first because documented exec allowlists already provide explicit per-path executable trust.
