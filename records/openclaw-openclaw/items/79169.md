## Summary

Keep open: current main still lacks the requested path/hash safe-bin trust, and the remaining work is a security-sensitive product contract decision rather than a narrow bug fix. The reporter’s latest production comment confirms the need and explicitly defers implementation until maintainers choose the canonical trust surface.

## What This Changes

The issue requests per-binary safe-bin exec trust by absolute path and optionally SHA-256 hash, instead of trusting whole directories.

## Best Possible Solution

Define the canonical exec trust contract first; if safe-bin scoped binary/hash trust is accepted, add schema, docs, audit/doctor guidance, and focused tests around the new semantics.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main only supports directory-level safe-bin trust plus the separate host allowlist path.

## Solution Assessment

Unclear. A per-binary/hash field could solve the wrapper maintenance problem, but the issue changes exec trust semantics and overlaps with documented allowlist behavior, so product/security design should come before implementation.
