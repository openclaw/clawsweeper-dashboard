## Summary

Keep open: current main still models safe-bin executable trust by trusted directory only, and the requested per-binary path/hash trust would add a new security-sensitive config surface that overlaps with existing host allowlists.

## What This Changes

Add safe-bin scoped per-binary path and optional SHA-256 trust config instead of trusting whole `safeBinTrustedDirs` directories.

## Best Possible Solution

Decide whether the existing host allowlist is the intended per-path trust answer; if safe-bin scoped binary/hash trust is accepted, add a typed schema, docs, audit, doctor guidance, and regression tests as one reviewed contract.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main only supports directory-level safe-bin trust plus the separate host allowlist path.

## Solution Assessment

Unclear. Per-binary or hash trust could reduce wrapper maintenance, but it changes exec trust semantics and overlaps with documented allowlist behavior, so product/security design should come before implementation.
