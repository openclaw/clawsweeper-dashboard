## Summary

Keep open: current main still only has directory-based safe-bin trust, the prior duplicate was closed into a meta issue, and no open PR supersedes the request; adding per-path or hash trust needs maintainer exec-security design because docs already route specific executable trust through the host allowlist.

## What This Changes

Request adds per-binary safe-bin trust by absolute path and optionally SHA-256 hash instead of trusting whole directories with `safeBinTrustedDirs`.

## Best Possible Solution

Define the exec trust model first; if accepted, add a typed config/schema/docs/audit/doctor/test contract covering path normalization, symlinks, binary updates, hash timing, and interaction with host allowlists.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main still supports safe-bin trust by trusted directory only, with per-path executable trust handled separately by host allowlists.

## Solution Assessment

Unclear. The proposed config would reduce wrapper maintenance, but it changes safe-bin trust semantics and overlaps with existing allowlist behavior, so product/security design should precede implementation.
