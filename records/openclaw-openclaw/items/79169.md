## Summary

Keep open: current main still has directory-only safe-bin executable trust, and the requested per-binary path/hash trust would add a new security-sensitive config surface that overlaps with the existing exec approvals allowlist.

## What This Changes

Request per-binary safe-bin trust by absolute path and optional SHA-256 hash instead of trusting whole executable directories.

## Best Possible Solution

Decide the canonical exec trust surface first; if safe-bin scoped binary/hash trust is accepted, add a typed schema, docs, audit/doctor guidance, and regression tests around that contract.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main supports directory-level safe-bin trust and a separate host allowlist, but no per-binary or hash safe-bin trust field.

## Solution Assessment

Unclear. The requested path/hash trust could solve the wrapper maintenance problem, but it changes exec approval boundaries and overlaps the existing host allowlist, so maintainers should settle the product/security contract before implementation.
