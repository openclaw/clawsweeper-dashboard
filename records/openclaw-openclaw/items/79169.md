## Summary

Keep open: current main and the latest tagged release still provide directory-level safe-bin executable trust only, while the requested path/hash trust would add a new security-sensitive exec configuration surface that overlaps the existing host allowlist.

## What This Changes

The issue requests a per-binary safe-bin trust option by resolved path, with optional SHA-256 pinning, instead of trusting whole directories via `safeBinTrustedDirs`.

## Best Possible Solution

Decide the canonical exec trust contract first; if accepted, add a typed config/schema/docs/audit contract with focused regression coverage around path, symlink, update, hash, and allowlist interactions.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence current behavior: safe-bin trust is directory-based and there is no per-binary or hash trust field on current main.

## Solution Assessment

Unclear. The requested path/hash trust could address the production wrapper workaround, but it changes executable trust semantics and overlaps the existing host allowlist, so the maintainable fix needs a product/security contract decision first.
