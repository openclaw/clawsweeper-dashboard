## Summary

Keep open: current main and v2026.5.7 still implement safe-bin trust at directory granularity, and the requested path/hash trust is a security-sensitive config/product decision rather than a cleanup-close candidate.

## What This Changes

Add a safe-bin trust mechanism for specific executable paths or SHA-256-pinned binaries instead of trusting whole directories via `safeBinTrustedDirs`.

## Best Possible Solution

Define the exec trust contract first, then implement the chosen path with aligned schema, docs, audit/doctor guidance, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main supports directory-level safe-bin trust and separate host allowlist path matching, but not safe-bin scoped path/hash trust.

## Solution Assessment

Unclear. Per-binary or hash trust would solve the wrapper workaround, but the maintainable fix depends on a maintainer product/security decision about its relationship to host allowlists.
