## Summary

Keep open: current main still models safe-bin executable trust by trusted directory only, while the requested per-binary path/hash trust would add a new security-sensitive config surface that overlaps with the existing host allowlist.

## What This Changes

Add per-binary safe-bin trust by absolute path, optionally pinned by SHA-256 hash, instead of trusting whole binary directories.

## Best Possible Solution

Decide the exec-trust model first; if accepted, add a typed `safeBinTrustedBinaries` path/hash contract with schema, docs, audit/doctor coverage, and focused tests while keeping safeBins narrowly scoped.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main only supports directory-level safe-bin trust plus the separate host allowlist path.

## Solution Assessment

Unclear. A per-binary/hash config could solve the stated maintenance problem, but it changes exec trust semantics and overlaps with the documented allowlist, so product/security design should come before implementation.
