## Summary

Keep open: current main still has only directory-level safe-bin trust, and the requested path/hash trust would add a security-sensitive exec trust contract that overlaps with the existing host allowlist.

## What This Changes

Add per-binary safe-bin exec trust by resolved path, with an optional SHA-256 pin, instead of trusting whole directories through `safeBinTrustedDirs`.

## Best Possible Solution

Decide whether per-binary trust belongs in the existing host allowlist or as a new typed safe-bin config surface, then implement schema, docs, audit/doctor guidance, and regression coverage around the chosen contract.

## Reproduction Assessment

Not applicable as a feature request. Source inspection provides high-confidence current behavior: safe-bin trust is directory-only and no per-binary/hash config is accepted.

## Solution Assessment

Unclear. The requested config could solve the wrapper maintenance problem, but it changes exec trust semantics and overlaps with documented allowlist behavior, so the best fix depends on maintainer product/security design first.
