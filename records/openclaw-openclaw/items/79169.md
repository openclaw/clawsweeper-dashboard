## Summary

Keep open: current main still models safe-bin executable trust by trusted directory only, while the requested per-binary path or SHA-256 trust is a security-sensitive new config surface with fresh production context from the reporter.

## What This Changes

Add per-binary path or SHA-256 trust for safe-bin exec so operators can trust named binaries without trusting their containing directories.

## Best Possible Solution

Define whether per-binary or hash trust belongs in safe-bin config or the existing host allowlist, then implement a typed schema/runtime/audit/doctor/docs/test contract if accepted.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection against current main confirms the current behavior is directory-level safe-bin trust plus separate host allowlist path trust.

## Solution Assessment

Unclear. A per-binary or hash config could solve the reported wrapper maintenance problem, but maintainers should settle the security and product contract before implementation because it overlaps existing exec allowlists.
