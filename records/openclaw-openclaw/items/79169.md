## Summary

Keep open: current main still trusts safe-bin executables by directory only, and the requested per-path or SHA-256 trust would add a security-sensitive exec config surface that needs maintainer product/security design.

## What This Changes

Request per-binary safe-bin exec trust by absolute path, optionally pinned by SHA-256, so operators can avoid trusting whole package-manager directories.

## Best Possible Solution

Define the exec trust contract first, then implement the accepted surface with typed config, docs, audit/doctor guidance, and focused runtime tests if maintainers want safe-bin-scoped path or hash trust.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms the missing capability: safe-bin trust is driven by `safeBinTrustedDirs`, and no per-binary or hash-pinned trust field exists.

## Solution Assessment

Unclear. Path or hash trust could solve the wrapper maintenance problem, but the maintainable fix depends on a product/security decision because it overlaps with the existing host allowlist and changes exec approval boundaries.
