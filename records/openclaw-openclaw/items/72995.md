## Summary

Keep open. Current main still matches the reported failure class: the managed web_search path does not pass the agent tool AbortSignal into the provider contract, Gemini receives only a timeout-derived fetch signal, and AbortError still exits the tool adapter before the normal failed-tool logging branch.

## What This Changes

The issue asks OpenClaw to stop Gemini-backed web_search from intermittently returning an immediate abort without gateway.err.log visibility.

## Best Possible Solution

Add a backwards-compatible web-search execution context that carries AbortSignal and abort diagnostics through AnyAgentTool, runWebSearch, the provider SDK/runtime types, and the Gemini provider; then log and classify unexpected provider aborts while preserving true user/session cancellation as cancellation.

## Reproduction Assessment

Unclear for the exact intermittent live Gemini failure, because this read-only review did not run a live Gemini call. The source-level reproduction path is clear: a selected web-search provider can throw AbortError, and current main rethrows it before normal tool failure logging while the managed web_search path lacks caller signal context.

## Solution Assessment

Not applicable as a proposed patch; there is no PR to evaluate. The requested direction should not be solved by only increasing Gemini timeout, because the current source path needs a generic cancellation/diagnostic seam plus logging tests.
