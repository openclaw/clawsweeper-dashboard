## Summary

Keep open. Current main still matches the reported failure class: the managed `web_search` tool does not forward the agent tool `AbortSignal` into the provider contract or Gemini provider, and provider-shaped `AbortError`s still bypass the normal tool failure logging branch, so the no-`gateway.err.log` symptom remains credible.

## What This Changes

The issue reports intermittent instant `AbortError` failures from Gemini-backed `web_search`, with no `gateway.err.log` entry, and asks for cancellation and logging behavior to be fixed.

## Best Possible Solution

Add a backwards-compatible web-search execution context carrying `AbortSignal` and abort diagnostics from the agent tool adapter through `runWebSearch` into provider definitions, merge that signal with provider timeouts in the Gemini path, and classify/log unexpected provider aborts while preserving true session cancellation behavior.

## Reproduction Assessment

Unclear for the original live intermittent Gemini abort because no live Gemini run was performed and the report is timing-dependent. The supporting failure path is source-reproducible by having a selected provider throw an `AbortError`, which current main rethrows before normal tool failure logging.

## Solution Assessment

Not applicable as a proposed patch. The current implementation is not the best end state because timeout configuration is the only context reaching Gemini, while caller cancellation and provider-abort diagnostics are lost.
