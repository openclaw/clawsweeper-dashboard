## Best Possible Solution

Keep this issue open until an ACP maintainer runs a current-release rapid two-spawn smoke; if it still fails, fix the ACPX launch/readiness path or add a narrowly scoped OpenClaw startup gate without disabling intentional different-session parallelism.
