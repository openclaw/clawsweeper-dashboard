The PR adds a local ISO timestamp formatter, appends local ISO and UTC text to `session_status`, and adds a pinned-clock regression test.

## Best Possible Solution

Thread a machine-readable current-time line through the shared status renderer so `/status` and `session_status` stay consistent, then prove the real tool output after refresh.

## Reproduction Assessment

Yes. Source inspection is enough: current main routes `session_status` through `buildStatusText()` without any current-time line, while docs tell agents to use `session_status` when they need the current time.

## Solution Assessment

No, not as-is. The direction is useful, but the PR should refresh onto current main, use the shared `timeLine` renderer path, update the stale readable-time assertion, and add real behavior proof.
