## Summary

Keep open. Current main still does not expose a machine-readable current-time line through `session_status`, and this PR is aimed at a real documented gap, but the branch is conflicting, lacks after-fix real behavior proof, and has a stale readable-time assertion that will fail against current main.

## What This Changes

This PR adds local ISO/UTC current-time text to `session_status`, adds a local ISO formatter, and adds pinned-clock regression coverage.

## Best Possible Solution

Thread one machine-readable current-time line through the shared status renderer so `/status` and `session_status` keep a single ordering/format contract, refresh the test expectation, and include redacted real output proof.

## Reproduction Assessment

Yes. Source inspection is enough: docs direct agents to `session_status` for current time, but current main calls `buildStatusText()` without a `timeLine` and returns status text without a timestamp line.

## Solution Assessment

No, not as-is. The direction is narrow, but the branch pins obsolete readable-time formatting and should use the shared status renderer boundary instead of appending after render.
