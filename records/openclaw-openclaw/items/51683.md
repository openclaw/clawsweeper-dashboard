## Summary

Keep open: this is a focused fix for a real `session_status` time gap, but the current branch is conflicting, the diff bypasses the shared status renderer, one new assertion is stale against current main, and the external PR still lacks required real behavior proof.

## What This Changes

The branch adds a local ISO timestamp formatter, appends local ISO and UTC time to `session_status` output, and adds a pinned-clock regression test.

## Best Possible Solution

Add a machine-readable current-time line by exposing `timeLine` on `BuildStatusTextParams`, passing it through `buildStatusText()` into `buildStatusMessage()`, updating the stale assertion, and proving real `session_status` output after rebase.

## Reproduction Assessment

Yes. Source inspection is enough: docs direct agents to `session_status` for current time, while current main calls `buildStatusText()` without any time-line parameter and the status text params type has no `timeLine` field.

## Solution Assessment

No, not as-is. The direction is useful, but the implementation should refresh onto current main and pass the new time line through the shared renderer instead of appending a session_status-only footer.
