## Summary

Keep open. Current main still routes Z.AI `reasoning_content` through hidden thinking output, the latest release has the same routing, and the directly linked fix PR #76078 is closed unmerged with no open replacement found.

## What This Changes

Z.AI GLM-5 reasoning responses with empty `content` and user-facing `reasoning_content` should render visible assistant text instead of an empty reply.

## Best Possible Solution

Land a narrow transport fix that uses the existing Z.AI compat signal to surface GLM-5 `reasoning_content` as visible text, with regression coverage and a non-Z.AI control case.

## Reproduction Assessment

Yes, at source level: feed `processOpenAICompletionsStream` a Z.AI completions chunk with empty `delta.content` and non-empty `delta.reasoning_content`. Current main classifies it as thinking and routes it to hidden output rather than visible text.

## Solution Assessment

Yes, consuming the existing `compat.thinkingFormat === "zai"` signal in the stream reasoning extractor/routing is the narrowest maintainable fix. It avoids hardcoded GLM model IDs and keeps other providers' reasoning hidden unless already marked visible.
