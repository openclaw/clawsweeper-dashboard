## Summary

Keep open. Current main still classifies OpenAI-compatible `reasoning_content` as hidden thinking for Z.AI GLM-5 streams, and the open linked PR #76078 is the active implementation path rather than a reason to close this issue early.

## What This Changes

The report asks OpenClaw to route Z.AI GLM-5 `reasoning_content` as visible text instead of hidden thinking when streamed `content` is empty.

## Best Possible Solution

Land a provider-aware Z.AI stream normalization that converts GLM-5 answer-only `reasoning_content` into visible text while preserving opt-in hidden thinking replay semantics.

## Reproduction Assessment

Yes. A Z.AI `openai-completions` stream chunk with empty `delta.content` and non-empty `delta.reasoning_content` flows through current main as a thinking block and produces no visible text block.

## Solution Assessment

No, not as a blanket routing rule. Consuming the existing Z.AI compat signal is the right seam, but the implementation should distinguish final-answer fallback from preserved-thinking replay.
