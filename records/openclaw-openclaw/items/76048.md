## Summary

Keep open. Current main still routes OpenAI-compatible `reasoning_content` into hidden thinking blocks for Z.AI models, and open PR #76078 already references this issue with closing syntax, so the issue should stay open until that PR is reviewed, merged, or closed.

## What This Changes

The issue reports that Z.AI GLM-5 models place user-visible output in `reasoning_content` while OpenClaw routes that field into hidden thinking blocks, producing empty visible replies.

## Best Possible Solution

Review and either land or revise PR #76078 so Z.AI GLM-5 `reasoning_content` reaches visible assistant text without regressing intentional preserved-thinking replay.

## Reproduction Assessment

Yes. A source-level stream fixture with provider `zai`, empty `delta.content`, and non-empty `delta.reasoning_content` should currently produce a hidden thinking block and no visible text block.

## Solution Assessment

Unclear. The linked PR is the right place to solve this, but maintainers should confirm the final routing distinguishes Z.AI visible-answer fallback from preserved-thinking behavior before merging.
