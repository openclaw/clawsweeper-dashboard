## Summary

Keep open. Current main and v2026.5.2 still route Z.AI GLM-5 `reasoning_content` into hidden thinking output, and the directly related fix PR #76078 is closed unmerged.

## What This Changes

The issue asks OpenClaw to surface Z.AI GLM-5 visible output from `reasoning_content` instead of hiding it as thinking when `content` is empty.

## Best Possible Solution

Land a narrow transport fix that uses the existing Z.AI compat signal to surface GLM-5 `reasoning_content` as visible text, with regression coverage for Z.AI and a control case for other providers.

## Reproduction Assessment

Yes, at source level: feed `processOpenAICompletionsStream` a Z.AI OpenAI-completions chunk with empty `delta.content` and non-empty `delta.reasoning_content`. Current main classifies that value as `thinking`, routes it through `appendThinkingDelta`, and visible extraction later keeps only `text` blocks.

## Solution Assessment

Yes, consuming the existing `compat.thinkingFormat === "zai"` signal in the stream extractor is the narrowest maintainable direction. The fix should avoid hardcoded GLM model IDs and keep non-Z.AI reasoning output hidden unless already classified as visible text.
