## Summary

Keep open. Current main still classifies OpenAI-compatible `reasoning_content` as hidden thinking for Z.AI completions streams, and open PR #76078 explicitly references this issue with closing syntax, so the issue should remain open until that PR is reviewed and merged or rejected.

## What This Changes

The issue reports that Z.AI GLM-5 reasoning models put user-visible assistant output in `reasoning_content` while OpenClaw routes that field to hidden thinking output, producing empty replies.

## Best Possible Solution

Review and either land or revise PR #76078 so Z.AI GLM-5 final-answer `reasoning_content` reaches visible assistant text while preserved thinking remains hidden where intended.

## Reproduction Assessment

Yes. A source-level stream fixture with provider `zai`, empty `delta.content`, and non-empty `delta.reasoning_content` follows current main into a thinking block and produces no visible text block.

## Solution Assessment

Unclear. The linked PR is aimed at the right narrow stream-normalization surface, but review should confirm the compat rule is scoped tightly enough around Z.AI final-answer output versus preserved thinking replay.
