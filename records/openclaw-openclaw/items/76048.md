## Summary

Keep open. Current main and the latest release still classify OpenAI-compatible `reasoning_content` as hidden thinking for Z.AI completions streams, and open PR #76078 already references this issue with closing syntax, so the issue should wait for that PR to be reviewed and resolved.

## What This Changes

Route Z.AI GLM-5 stream chunks with empty `content` and non-empty `reasoning_content` to visible assistant text instead of hidden thinking.

## Best Possible Solution

Land or revise PR #76078 so Z.AI GLM-5 `reasoning_content` becomes visible assistant text while explicit preserved-thinking replay remains intact.

## Reproduction Assessment

Yes, by source inspection: a Z.AI completions stream chunk with empty `delta.content` and non-empty `delta.reasoning_content` will be emitted as a thinking block, not a text block. I did not run a live Z.AI request because this was a read-only review.

## Solution Assessment

Yes at the fix direction level. Consuming the existing `thinkingFormat: "zai"` compat signal in the stream reasoning classifier is the narrow maintainable path, with regression coverage for `glm-5-turbo`/`glm-5.1`.
