## Summary

Keep open. Current main and v2026.5.3 still route bare Z.AI `reasoning_content` into hidden thinking even though compat detects `thinkingFormat: "zai"`, and the directly related fix PR is closed unmerged.

## What This Changes

Z.AI GLM-5 reasoning models should show assistant text when upstream streams it in `reasoning_content` with empty `content`, instead of hiding it as thinking.

## Best Possible Solution

Land a narrow transport regression fix that uses the existing Z.AI compat signal to surface GLM-5 `reasoning_content` as visible text, with regression coverage and no hardcoded GLM model IDs.

## Reproduction Assessment

Yes, at source level: feed the completions stream parser a Z.AI chunk with empty `delta.content` and non-empty `delta.reasoning_content`. Current main classifies it as `thinking`, routes it to `appendThinkingDelta`, and later channel extraction ignores non-text blocks.

## Solution Assessment

Yes. Consuming `compat.thinkingFormat === "zai"` in the stream extraction/routing path is the narrowest maintainable fix because the detection already exists and non-Z.AI reasoning output should remain hidden unless explicitly visible.
