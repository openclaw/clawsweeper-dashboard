## Summary

Keep open. Current main still classifies Z.AI `reasoning_content` as hidden thinking, the latest release tag has the same routing, and the directly related fix PR #76078 is closed unmerged.

## What This Changes

Z.AI GLM-5 reasoning models return empty visible replies when upstream sends generated text in `reasoning_content` with empty `content`.

## Best Possible Solution

Land a narrow transport fix that uses the existing Z.AI compat signal to surface GLM-5 `reasoning_content` as assistant text, with regression coverage and a user-facing changelog entry.

## Reproduction Assessment

Yes, at source level: a Z.AI OpenAI-completions stream chunk with empty `delta.content` and non-empty `delta.reasoning_content` becomes a `thinking` block on current main, so no visible text is extracted. I did not run a live Z.AI API call in this read-only review.

## Solution Assessment

Yes. Consuming the existing `compat.thinkingFormat === "zai"` signal in the stream extractor is the narrowest maintainable fix; avoid hardcoded GLM model IDs and keep other providers' reasoning hidden unless already configured as visible text.
