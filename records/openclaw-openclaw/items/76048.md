## Summary

Keep open. Current main still routes OpenAI-compatible `reasoning_content` into hidden thinking blocks for Z.AI completions streams, and open PR #76078 directly references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix is resolved.

## What This Changes

Z.AI GLM-5 reasoning responses with empty `content` and non-empty `reasoning_content` should surface as visible assistant text instead of hidden thinking.

## Best Possible Solution

Land or revise the existing focused fix so Z.AI GLM-5 `reasoning_content` reaches visible text while preserving hidden-thinking behavior for other providers and intentional preserved-thinking replay.

## Reproduction Assessment

Yes. A current-main source fixture using provider `zai` with empty `delta.content` and non-empty `delta.reasoning_content` will be classified as a thinking block and filtered out of visible assistant text.

## Solution Assessment

Yes. The narrow maintainable solution is to consume existing compat metadata in the OpenAI-completions stream routing, rather than adding channel workarounds or hardcoding this in message delivery.
