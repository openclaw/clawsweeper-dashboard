## Summary

Keep open. Current main still classifies streamed `reasoning_content` as hidden thinking for Z.AI completions, and the linked implementation PR is open and unmerged, so this issue should remain open until that work lands or is replaced.

## What This Changes

Report a Z.AI GLM-5 regression where OpenAI-compatible responses put user-visible text in `reasoning_content` while `content` is empty, producing empty OpenClaw replies.

## Best Possible Solution

Keep the issue open while #76078, or an equivalent narrow compat-aware stream fix, is reviewed and landed; then close it as implemented after merge.

## Reproduction Assessment

Yes. A source-level completions stream chunk with empty `delta.content` and non-empty `delta.reasoning_content` follows the current path into a thinking block and never reaches visible text extraction.

## Solution Assessment

Unclear for the reporter's exact one-line route. The maintainable fix should be a narrow compat-aware stream change with regression coverage, while preserving hidden reasoning behavior for other providers and Z.AI preserved-thinking replay.
