## Summary

Keep open. Current main and v2026.5.2 still classify Z.AI GLM-5 `reasoning_content` as hidden thinking while the Z.AI compat signal exists but is not used by the stream classifier; the directly related fix PR #76078 is closed unmerged.

## What This Changes

The issue asks OpenClaw to surface Z.AI GLM-5 `reasoning_content` as visible assistant text instead of hidden thinking when `content` is empty.

## Best Possible Solution

Land a narrow transport fix that uses an explicit compat signal to surface the Z.AI GLM-5 visible-answer `reasoning_content` path while preserving hidden reasoning behavior for other providers and for private thinking cases.

## Reproduction Assessment

Yes, at source level: a Z.AI OpenAI-completions stream chunk with empty `delta.content` and non-empty `delta.reasoning_content` is classified as `thinking` and routed to `appendThinkingDelta`. Visible assistant extraction later keeps only `text` blocks, so the user-visible response is empty.

## Solution Assessment

Yes, the narrow maintainable direction is to consume the existing compat information in the stream classifier rather than hardcoding model IDs. The patch should include a non-Z.AI control and a Z.AI thinking/preserved-thinking control to avoid leaking real reasoning traces.
