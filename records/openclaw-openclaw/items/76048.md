## Summary

Keep open. Current main still has the reported source-level routing path, and #76078 is the open closing PR for it; the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Z.AI GLM-5 reasoning models should surface final answer text returned in `reasoning_content` instead of hiding it as thinking when `content` is empty.

## Best Possible Solution

Land a narrow provider-aware stream fix, preferably via #76078 after review, that surfaces GLM-5 empty-content `reasoning_content` as visible answer text without breaking `preserveThinking` replay.

## Reproduction Assessment

Yes. A source-level completions stream chunk for provider `zai` with empty `delta.content` and non-empty `delta.reasoning_content` is classified as `kind: "thinking"` and appended as a thinking block, while visible assistant extraction reads only text blocks.

## Solution Assessment

Unclear for the exact proposed snippet. Consuming `compat.thinkingFormat` is the right seam, but the final patch should distinguish final-answer fallback from preserved hidden reasoning so Z.AI replay behavior stays intact.
