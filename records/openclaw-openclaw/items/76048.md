## Summary

Keep open. Current main still matches the reported Z.AI stream-routing failure, and the later OpenAI-outage workaround comment does not supersede the source-level evidence; the open linked PR should be reviewed rather than closing this issue early.

## What This Changes

The issue requests that Z.AI GLM-5 `reasoning_content` be delivered as visible assistant text when `content` is empty.

## Best Possible Solution

Review and land #76078 or an equivalent narrow provider-stream fix that uses the existing Z.AI compat seam while preserving opt-in hidden thinking replay.

## Reproduction Assessment

Yes. A source-level stream chunk with `delta.reasoning_content` and empty or absent `delta.content` will produce a `thinking` block on current main, and normal visible replies are extracted from `text` blocks only.

## Solution Assessment

Unclear. The existing `thinkingFormat: "zai"` compat seam is the right direction, but the final patch should prove that disabled-thinking final answers become visible without breaking Z.AI `preserveThinking` replay semantics.
