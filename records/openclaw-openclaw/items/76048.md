## Summary

Keep open. Current main still routes OpenAI-compatible `reasoning_content` into hidden thinking blocks for Z.AI models, so the reported empty visible reply remains plausible and source-reproducible against the current stream pipeline.

## What This Changes

Z.AI GLM-5 models should deliver user-visible text when their OpenAI-compatible response puts the final answer in `reasoning_content` with empty `content`.

## Best Possible Solution

Implement Z.AI-owned stream/output normalization that converts GLM-5 `reasoning_content` into visible text when thinking is disabled and no visible content is present, while keeping preserved-thinking replay hidden when thinking is enabled.

## Reproduction Assessment

Yes. A source-level fixture with a Z.AI `openai-completions` stream chunk containing `delta.reasoning_content` and empty `delta.content` will produce a `thinking` block and no visible text block on current main.

## Solution Assessment

Not exactly. The proposed always-route-Z.AI check is too broad because Z.AI preserved thinking intentionally uses `reasoning_content`; a narrower provider-owned wrapper or explicit compat path should distinguish final-answer fallback from hidden reasoning.
