## Summary

Keep open. Current main and the latest release still contain the reported Z.AI routing path, and the directly related fix PR is closed unmerged.

## What This Changes

The issue asks OpenClaw to surface Z.AI GLM-5 `reasoning_content` as visible assistant text when `content` is empty instead of hiding it as thinking.

## Best Possible Solution

Land a narrow transport fix that consumes the existing Z.AI compat signal when classifying OpenAI-compatible reasoning deltas, with a regression test and a non-Z.AI control case.

## Reproduction Assessment

Yes, at source level. A Z.AI streaming chunk with empty `delta.content` and non-empty `delta.reasoning_content` traces through current main into a `thinking` block, and channel-visible extraction keeps only `text` blocks.

## Solution Assessment

Yes, the proposed direction is the narrowest maintainable fix. Use `compat.thinkingFormat === "zai"` or an equivalent existing compat signal in the stream extractor, and avoid hardcoded GLM model IDs or broad reasoning-policy changes.
