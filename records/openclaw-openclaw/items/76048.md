## Summary

Keep open. Current main still routes OpenAI-compatible `reasoning_content` from Z.AI GLM streams into hidden thinking blocks while visible delivery reads only text blocks, so the reported empty user reply remains source-reproducible and is not superseded by the related DeepSeek/OpenRouter fix.

## What This Changes

The issue asks OpenClaw to make Z.AI GLM-5 responses whose meaningful output is in `reasoning_content` appear as visible assistant text instead of hidden thinking.

## Best Possible Solution

Add a narrow Z.AI/OpenAI-completions normalization path that surfaces `reasoning_content` as visible text only for the empty-content disabled-thinking fallback, while preserving opt-in hidden reasoning replay.

## Reproduction Assessment

Yes. A source-level fixture with a Z.AI `openai-completions` stream chunk containing `delta.reasoning_content` and no visible `delta.content` will produce a thinking block and no text block on current main.

## Solution Assessment

No, the proposed unconditional `compat.thinkingFormat === "zai"` routing is too broad. The maintainable fix should be aware of Z.AI thinking state and `preserveThinking` so hidden reasoning remains hidden when intentionally enabled.

## Repair Work Prompt

Fix the Z.AI GLM-5 OpenAI-completions stream regression reported in https://github.com/openclaw/openclaw/issues/76048. Current main turns `delta.reasoning_content` into a hidden thinking block, so Z.AI responses with empty `content` produce no visible channel text. Implement the smallest maintainable fix, preferably in the Z.AI/provider-stream compatibility path or the OpenAI-completions reasoning routing, so `reasoning_content` is surfaced as visible text only when it is the disabled-thinking empty-content fallback. Do not unconditionally route all Z.AI reasoning to text: preserve opt-in Z.AI preserved thinking (`params.preserveThinking` / `preserve_thinking`, `thinking: { type: "enabled", clear_thinking: false }`) and do not change DeepSeek/OpenRouter reasoning replay behavior. Add focused tests covering a Z.AI `glm-5.1` or `glm-5-turbo` stream chunk with empty content plus `reasoning_content`, and a preserved-thinking case that stays hidden/replay-safe. Add a user-facing changelog fix entry. Keep changes scoped to stream/provider compatibility and tests.
