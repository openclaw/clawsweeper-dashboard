## Summary

Keep open. Current main still routes Z.AI `reasoning_content` chunks into hidden thinking blocks, and the focused fix PR was closed unmerged rather than landed, so the reported empty-response bug still appears actionable.

## What This Changes

The issue asks OpenClaw to render GLM-5 Z.AI `reasoning_content` as visible assistant text when thinking is disabled instead of swallowing it as hidden thinking.

## Best Possible Solution

Land a scoped stream parser fix that treats Z.AI `reasoning_content` as visible text only when the final request payload has thinking disabled, while preserving hidden/replayable thinking for opt-in preserved-thinking mode.

## Reproduction Assessment

Yes, at source level. A current-main OpenAI-compatible Z.AI stream chunk with empty `delta.content` and non-empty `delta.reasoning_content` flows into a `thinking` block, and visible extraction only includes `text` blocks.

## Solution Assessment

Yes. The narrowest maintainable fix is the final PR #76078 shape: pass a disabled-thinking signal into `getCompletionsReasoningDeltas` instead of routing all Z.AI reasoning content to visible text.

## Repair Work Prompt

Fix #76048 on current main. Z.AI GLM-5 models can emit the user-visible answer in `delta.reasoning_content` with empty `delta.content` even when OpenClaw sends `thinking: { type: "disabled" }`; current `src/agents/openai-transport-stream.ts` turns that into a hidden thinking block, producing an empty visible reply. Implement the scoped repair from closed PR #76078: after OpenAI-completions params are built and provider wrappers have patched `params.thinking`, pass a narrow disabled-thinking signal into `processOpenAICompletionsStream`/`getCompletionsReasoningDeltas` so Z.AI `reasoning_content` becomes `kind: "text"` only when Z.AI compat is active and thinking is explicitly disabled. Do not route all Z.AI `reasoning_content` to visible text; preserved thinking (`preserveThinking` / enabled thinking) must remain hidden and replayable. Add regression coverage for both disabled-thinking visible text and enabled/preserved-thinking hidden reasoning. Add a single-line user-facing CHANGELOG.md fix entry crediting the human contributor from PR #76078 if appropriate, and avoid unrelated provider/core refactors.
