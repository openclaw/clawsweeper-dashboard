## Summary

Keep open. Current main and v2026.5.3 still classify Z.AI `reasoning_content` as hidden thinking, and the directly related fix PR #76078 is closed unmerged.

## What This Changes

The issue requests that Z.AI GLM-5 OpenAI-compatible streams with empty `content` and non-empty `reasoning_content` deliver that content as visible assistant text.

## Best Possible Solution

Land a narrow transport fix that uses the existing Z.AI compat signal to surface GLM-5 `reasoning_content` as visible text, with regression coverage and a user-facing changelog entry.

## Reproduction Assessment

Yes, at source level: a Z.AI OpenAI-completions stream chunk with empty `delta.content` and non-empty `delta.reasoning_content` is classified as `thinking`, routed to `appendThinkingDelta`, and later filtered out of visible assistant text. I did not run Vitest in this read-only sweep.

## Solution Assessment

Yes. Consuming `compat.thinkingFormat === "zai"` in the stream extractor is the narrowest maintainable fix because the compat signal already exists and avoids hardcoded GLM model IDs.
