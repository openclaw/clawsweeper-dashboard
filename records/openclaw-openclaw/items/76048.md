## Summary

Keep open. Current main still has the source-level Z.AI GLM-5 failure path: OpenAI-compatible `reasoning_content` is classified as hidden thinking, the Z.AI compat marker is not passed into the stream extractor, and the visible reply extractor later ignores thinking-only content; the direct fix PR #76078 was closed unmerged.

## What This Changes

The issue asks OpenClaw to show Z.AI GLM-5 `reasoning_content` as user-visible assistant text instead of hidden thinking when `content` is empty.

## Best Possible Solution

Land a narrow transport fix that uses the existing Z.AI compat signal to surface GLM-5 `reasoning_content` as visible assistant text, with regression coverage for Z.AI and a control case proving other providers still keep reasoning hidden unless explicitly visible.

## Reproduction Assessment

Yes, at source level: feed `processOpenAICompletionsStream` a Z.AI OpenAI-completions stream chunk with empty `delta.content` and non-empty `delta.reasoning_content`. Current main classifies that value as `thinking`, routes it through `appendThinkingDelta`, and visible extraction later keeps only `text` blocks.

## Solution Assessment

Yes, consuming the existing `compat.thinkingFormat === "zai"` signal in the stream reasoning extractor or routing path is the narrowest maintainable fix. The repair should avoid hardcoded GLM model IDs and preserve hidden reasoning behavior for non-Z.AI providers.
