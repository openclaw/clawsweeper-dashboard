## Summary

Keep open. Current main still routes OpenAI-compatible `reasoning_content` to hidden thinking for Z.AI models, and the supplied GitHub context shows an open unmerged closing PR (#76078), so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that Z.AI GLM-5 reasoning model `reasoning_content` be surfaced as visible assistant text instead of hidden thinking when `content` is empty.

## Best Possible Solution

Review and land #76078 or an equivalent Z.AI-specific stream fix that turns GLM-5 `reasoning_content` into visible text only for the intended fallback path while preserving replay semantics.

## Reproduction Assessment

Yes. A source-level stream chunk for `provider: "zai"` with empty `delta.content` and non-empty `delta.reasoning_content` will be classified as a thinking block on current main, and visible extraction only emits text blocks.

Yes for the issue lifecycle: keeping this open is the right state because #76078 uses closing syntax and remains unmerged. The implementation should be reviewed on that PR rather than closing the issue early.
