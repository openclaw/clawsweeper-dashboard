## Summary

Keep open. Current main still routes Z.AI GLM-5 `reasoning_content` into hidden thinking, and the directly related fix PR #76078 is closed unmerged.

## What This Changes

The issue requests restoring visible Z.AI GLM-5 responses when the OpenAI-compatible API returns user-facing text in `reasoning_content` with empty `content`.

## Best Possible Solution

Land a narrow transport fix that uses the existing Z.AI compat signal to surface GLM-5 `reasoning_content` as assistant text, with regression coverage and a non-Z.AI control.

## Reproduction Assessment

Yes, at source level: feed `processOpenAICompletionsStream` a Z.AI OpenAI-completions chunk with empty `delta.content` and non-empty `delta.reasoning_content`. Current main classifies that value as `thinking`, routes it through `appendThinkingDelta`, and visible extraction later keeps only `text` blocks.

## Solution Assessment

Yes, consuming the existing `compat.thinkingFormat === "zai"` signal in the stream extractor or routing decision is the narrowest maintainable fix. The change should avoid hardcoded GLM IDs and keep non-Z.AI reasoning output hidden unless already classified as visible text.
