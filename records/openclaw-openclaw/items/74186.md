## Summary

Keep open. The PR targets a valid sandboxed media URI bug that current main still has, but the latest head still rewrites unvalidated `media://` inputs through a staged inbound fallback before applying the existing media URI contract, so it should not merge as-is.

## What This Changes

The PR adds early `media://` handling to `resolveSandboxedBridgeMediaPath`, staged inbound fallback resolution with workspace-boundary checks, and sandbox helper tests.

## Best Possible Solution

Land a narrow sandbox helper fix that validates `media://inbound/<id>` with the same authority and decoded single-segment ID rules as `resolveInboundMediaReference`, uses the validated ID for any staged fallback probe, returns the raw URI for downstream handling when staging is unavailable, preserves workspace-boundary enforcement, and adds regression coverage for unsupported hosts, nested or encoded separator IDs, and malformed media URLs.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: the sandboxed image tool calls `resolveSandboxedBridgeMediaPath`, and Node path resolution turns `media://inbound/<id>` into `/workspace/media:/inbound/<id>` before downstream media URI handling can run.

## Solution Assessment

No. Early detection is the right seam, but the proposed fallback must validate the media URI contract before rewriting to `media/inbound/<basename>`; otherwise it broadens accepted inputs in a sandbox/media trust boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Validate media URIs before probing the fallback:** `src/agents/sandbox-media-paths.ts:45-46`
  - body: The new branch accepts every `media://` authority and collapses the URL path to `path.basename` before probing `media/inbound`. If `media://outbound/foo.png` or `media://inbound/a/b.png` is supplied and that basename exists in the staged inbound directory, this loads a file that `resolveInboundMediaReference` would reject. Validate host `inbound` and the decoded single-segment ID before probing; otherwise return the raw URI for downstream validation.
  - confidence: 0.91
- **[P3] Guard malformed media URLs before parsing:** `src/agents/sandbox-media-paths.ts:45`
  - body: `new URL(filePath)` runs before any defensive parse/catch in the new media branch. Malformed media references such as an invalid host now throw a raw `TypeError` from the sandbox helper instead of flowing through the existing media-reference error path, so parse defensively and return the raw URI or convert to the existing validation error.
  - confidence: 0.78
