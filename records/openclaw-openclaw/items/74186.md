## Summary

Keep open. The sandboxed media URI bug is still real on current main, but the PR head is not safe to merge as-is because it rewrites unvalidated `media://` references to staged inbound files before applying the stricter media-reference contract. The linked bug and overlapping fix PRs remain open, so this should stay in review rather than be closed or superseded yet.

## What This Changes

The PR adds early `media://` handling to `resolveSandboxedBridgeMediaPath`, tries `media/inbound/<basename>` fallback resolution through the sandbox bridge, preserves raw URI fallback, and adds sandbox helper tests.

## Best Possible Solution

Land a narrow sandbox helper fix that detects `media://` before bridge path resolution, validates only `media://inbound/<single decoded id>` with the same rules as `resolveInboundMediaReference`, probes the staged fallback only for that validated ID, returns raw URIs for downstream handling when staging is unavailable, and covers unsupported-host, nested-ID, malformed-URI, fallback-miss, and workspace-boundary cases.

## Reproduction Assessment

Yes. Current main has a code-level reproduction path: sandboxed image execution calls the helper before `loadWebMedia`, and POSIX path resolution mangles `media://inbound/<id>` into a `media:/inbound` workspace path. The PR-introduced validation bug is also reproducible by staging `media/inbound/foo.png` and passing `media://outbound/foo.png` or a nested inbound URI.

## Solution Assessment

No. Detecting media URIs before bridge resolution is the right direction, but rewriting arbitrary `media://` inputs by basename is not the best fix because it bypasses the existing authority and single-segment ID validation; reuse or mirror the media-reference parser before attempting fallback resolution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Validate media URIs before probing inbound fallback:** `src/agents/sandbox-media-paths.ts:46-47`
  - body: The new branch rewrites any `media://` URL to `media/inbound/<basename>` when that staged file exists. That bypasses the media-reference contract: `media://outbound/foo.png`, `media://inbound/a/b.png`, or encoded nested IDs can load a staged inbound file instead of being rejected. Parse and validate the authority plus decoded single-segment ID before probing; otherwise return the raw URI for downstream validation.
  - confidence: 0.91
- **[P3] Handle malformed media URLs before the fallback probe:** `src/agents/sandbox-media-paths.ts:46`
  - body: `new URL(filePath)` runs outside the fallback `try`, so malformed `media://` inputs throw a raw URL parse error from the sandbox helper whenever an inbound fallback dir is configured. Guard the parse and fall through to the raw URI or the existing media-reference validation path.
  - confidence: 0.78
