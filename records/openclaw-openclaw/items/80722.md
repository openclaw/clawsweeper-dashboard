## Summary

Keep open: current main and the latest release still emit the generic restart hint from the CLI config mutation paths, while the canonical reload plan treats agent/model config as hot-applying. The linked pull request https://github.com/openclaw/openclaw/pull/80823 is the active fix candidate, so this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue asks for `openclaw config set` and related config mutation success text to stop saying a gateway restart is required for hot-loadable agent/model config edits.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/80823 so CLI config write output derives from the canonical reload planner, while leaving the broader machine-readable apply-status contract to https://github.com/openclaw/openclaw/issues/83851.

## Reproduction Assessment

Yes. Source inspection shows current `config set`, `config patch`, and direct `config unset` success paths still print the generic restart hint, while the reload planner and docs classify agent/model config as hot-applying.

## Solution Assessment

Yes. The narrow maintainable fix is to derive CLI success hints from `buildGatewayReloadPlan()` instead of adding a suppression flag or duplicating per-agent heuristics; the open linked PR attempts that path.
