## Summary

The report is valid on current main, but it should stay open because https://github.com/openclaw/openclaw/pull/80823 is an open closing PR for the same bug and has already supplied terminal proof for the proposed behavior.

## What This Changes

`openclaw config set` on hot-loadable active-agent model paths should avoid the misleading gateway restart hint while restart-required paths keep it.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80823, or an equivalent narrow CLI hint selector that preserves the conservative restart hint for non-agent, dormant-runtime, `agentRuntime`-touching, and mixed-batch mutations.

## Reproduction Assessment

Yes. Source inspection shows current main always emits the restart hint on config set/patch/unset success paths, while the issue and linked PR body provide terminal proof that an active-agent model change is picked up on the next invocation without restarting.

## Solution Assessment

Yes. The linked PR's post-mutation classification approach is the narrowest maintainable fix because it changes only the success hint, keeps write semantics unchanged, and defaults to the existing restart warning unless the operation is positively hot-loadable.
