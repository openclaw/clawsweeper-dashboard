## Summary

Keep this issue open: current main still emits the same restart hint for config set/unset success paths, and the open linked fix at https://github.com/openclaw/openclaw/pull/80823 uses closing syntax for this issue, so the issue should wait for that PR to land or be replaced.

## What This Changes

The issue asks for `openclaw config set`/`unset` to avoid the gateway-restart hint for hot-loadable `agents.list[N]` changes when the agent has no `agentRuntime` override.

## Best Possible Solution

Land the linked PR or an equivalent bounded CLI hint change that keeps the restart hint for non-agent, `agentRuntime`-touching, dormant-runtime, and mixed restart-required mutations.

## Reproduction Assessment

Yes. Source inspection shows current main always emits the restart hint from the config mutation success paths, while `agents.list` is reload-planned as hot and the issue includes terminal proof of a model swap taking effect without restart.

## Solution Assessment

Yes. The best fix is a narrow success-message classifier based on the post-mutation agent config shape, with conservative fallback to the existing restart hint when the operation is not positively hot-loadable.
