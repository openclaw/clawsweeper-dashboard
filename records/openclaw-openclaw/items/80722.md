The issue asks the config CLI to show a next-invocation hint for hot-loadable active-agent config mutations while keeping the gateway restart hint for runtime-override and other restart-required paths.

## Best Possible Solution

## Solution Assessment

Yes. The open PR is the narrow maintainable fix path: choose the success hint from the post-mutation config, and preserve the restart hint for non-agent, `agentRuntime`-touching, dormant-runtime, and mixed cases.
