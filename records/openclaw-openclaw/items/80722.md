Keep this issue open: current `main` still emits the gateway-restart hint unconditionally for the relevant config write paths, and https://github.com/openclaw/openclaw/pull/80823 is an open closing PR for the same behavior. Per issue/PR pairing policy, the issue should stay open until that PR lands or is closed.

## What This Changes

The issue asks `openclaw config set` to avoid telling users to restart the gateway for active agent model changes that hot-load on the next invocation.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80823, or an equivalent narrow CLI hint change, while preserving restart hints for non-agent, `agentRuntime`-touching, runtime-pinned, and mixed-batch mutations.

## Reproduction Assessment

Yes. Source inspection shows current `main` always emits the restart hint after successful config set/patch/unset writes, while docs and reload rules classify `agents.list` model config as hot-loadable.

## Solution Assessment

Yes. The linked PR is the narrowest maintainable path because it changes only CLI success-message hint selection after the post-mutation config is known and adds focused tests for the hot-loadable versus restart-required split.
