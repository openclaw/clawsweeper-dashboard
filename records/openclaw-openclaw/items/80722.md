## What This Changes

Request differentiated config CLI success hints so hot-loadable `agents.list[N].model.*` writes for active agents do not tell operators to restart the gateway.

## Best Possible Solution

Review and land the linked narrow CLI hint change, then close this issue as implemented with the merge commit proof.

Yes. The source path is clear: current main appends the same restart hint after every successful config set/patch/unset write, while `agents.list` changes are already treated as hot reload by the gateway reload plan.

## Solution Assessment

Yes. The best fix is the narrow approach in https://github.com/openclaw/openclaw/pull/80823 or equivalent: choose the success hint from the post-mutation config and default to the existing restart hint for unknown, non-agent, runtime-touching, or mixed operations.
