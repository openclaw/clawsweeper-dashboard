## Summary

Keep open: current main and the latest release still emit the generic restart hint from the CLI config write paths. The merged reload-metadata work is useful adjacent infrastructure, but it does not consume that metadata in `config set`/`config unset`, and the linked issue-specific PR remains open.

## What This Changes

The issue asks for `openclaw config set` to stop printing a generic gateway restart warning for hot-loadable active-agent model changes while preserving restart guidance for fields that really require a gateway restart.

## Best Possible Solution

Review, land, or intentionally replace the linked CLI fix so config write output is derived from the canonical reload planner; keep the broader machine-readable post-apply contract in the separate follow-up issue.

## Reproduction Assessment

Yes. Source inspection shows current `config set`, `config patch`, and direct `config unset` success paths still print the generic restart hint, while the reload planner and docs classify agent/model config as hot-applying.

## Solution Assessment

Yes. The best fix is a narrow CLI success-text change that reuses the existing gateway reload planner, rather than adding a suppression flag or duplicating per-agent heuristics.
