## Summary

Keep open: current `main` and the latest release still emit the generic restart hint from the config CLI write paths. The merged reload-metadata work in https://github.com/openclaw/openclaw/pull/81612 is useful adjacent infrastructure, but it does not consume that metadata in `config set`/`unset`, and the issue-specific fix remains open in https://github.com/openclaw/openclaw/pull/80823.

## What This Changes

The issue asks for `openclaw config set` to avoid or refine the gateway restart warning for active agents whose config hot-loads without an `agentRuntime` override.

## Best Possible Solution

Review, land, or replace the linked CLI fix so config writes report restart guidance from the actual reload/apply truth; broader machine-readable post-apply status can remain with https://github.com/openclaw/openclaw/issues/83851 if maintainers keep that separate.

## Reproduction Assessment

Yes. Source inspection of current `main` shows `config set`, `config patch`, and direct `config unset` success paths still print the generic restart hint, while docs and reload rules classify agent/model config as hot-applying; I did not run the CLI because this review is read-only.

## Solution Assessment

Yes, a narrow CLI wording fix is the right shape. The safer implementation should reuse existing reload metadata plus post-write config state for `agentRuntime` overrides rather than adding a broad suppression flag.
