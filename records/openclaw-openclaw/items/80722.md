## Summary

Keep this open: current main still emits the restart hint unconditionally on the relevant config mutation success paths, and an open closing PR already targets the narrow fix. The issue should remain open until https://github.com/openclaw/openclaw/pull/80823 lands or is replaced.

## What This Changes

The issue asks `openclaw config set` to avoid the gateway-restart hint for hot-loadable active-agent `agents.list[N].model.*` changes while keeping it for runtime-overridden agents.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80823 or an equivalent narrow CLI hint fix, preserving the restart hint as the conservative fallback for non-agent, `agentRuntime`-touching, runtime-overridden, and mixed-batch mutations.

## Reproduction Assessment

Yes, source-reproducible. Current main always appends the restart hint on the config set/patch/unset success paths, while `agents.list` is already a hot reload prefix and agent invocation reads the selected agent model from config.

## Solution Assessment

Yes. The linked PR is the narrow maintainable path because it changes only success-message hint selection after the post-mutation config is known and keeps the old restart hint when the operation is not positively hot-loadable.
