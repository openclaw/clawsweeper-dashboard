## Summary

Keep open: current main still emits the undifferentiated restart hint, and there is an open closing PR that appears to be the active fix path, so this issue should not be closed before that PR merges or is explicitly rejected.

## What This Changes

The issue asks for `openclaw config set` to use a next-invocation hint for hot-loadable active agent config changes while preserving the gateway-restart warning for `agentRuntime`-pinned or otherwise restart-required changes.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80823, or an equivalent narrow CLI hint-selection fix, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, at source level. Current main always appends the restart hint on the config mutation success paths, while the reload plan and tests already classify `agents.list` changes as hot reloads; I did not run the CLI because this review is read-only.

## Solution Assessment

Yes. The linked PR's direction is the narrow maintainable fix: inspect the post-mutation config, soften the hint only for positively classified active-agent paths, and keep the restart hint for non-agent, `agentRuntime`-touching, runtime-pinned, and mixed cases.
