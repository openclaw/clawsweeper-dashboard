## Summary

## What This Changes

Add a built-in inbound user directive that strips suffixes like `reply in thread` or `rit` and forces the outbound reply into the current thread.

## Best Possible Solution

Define accepted trigger syntax and cross-channel override policy first, then implement a shared inbound directive or plugin/API seam with regression coverage for BodyForAgent, reply modes, and supported threaded channels.

Not applicable as a bug reproduction: this is a feature request. Source inspection confirms current main and v2026.6.8 lack the requested inbound suffix parser while existing reply tags and `replyToMode` cover different behavior.
