## Summary

Keep open: current main and the latest release still do not implement an inbound natural-language reply-in-thread suffix directive. The only direct implementation PR is closed unmerged, and the remaining decision is product shape rather than a conservative cleanup close.

## What This Changes

Requested a built-in inbound directive that strips suffixes such as `reply in thread` or `rit` from user text and forces the outgoing reply to target the current message thread.

## Best Possible Solution

Define a maintainer-approved per-message reply override contract first, then implement a narrow shared inbound directive or plugin/API seam with regression coverage for prompt stripping and supported threaded channels.

## Reproduction Assessment
