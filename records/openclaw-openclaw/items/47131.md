## Summary

Keep open. Current main and the latest release still support outbound reply tags and existing reply-threading policy, but not the requested shared inbound suffix directive; accepting global magic trigger words and optional config needs maintainer product judgment first.

## What This Changes

Add a built-in inbound message directive that strips suffixes like `reply in thread` or `rit` and forces the outbound reply to target the inbound message thread.

## Best Possible Solution

If accepted, add a typed suffix-only inbound directive after maintainers choose the trigger policy, strip it from agent and command bodies, feed it into existing reply-threading, and cover docs plus focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests a new user-facing directive. Source inspection gives high confidence that current main lacks the inbound suffix parser and carrier field.

## Solution Assessment

Unclear until maintainers approve the trigger vocabulary and config policy. The maintainable implementation would be a shared inbound directive feeding the existing reply-threading path, not channel-specific prompt instructions.
