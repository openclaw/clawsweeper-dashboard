## Summary

Keep open. Current main still supports outbound reply tags and reply-threading policy, but it does not implement a shared inbound suffix directive; approving global trigger words such as `thread` or a new trigger config surface needs maintainer product judgment.

## What This Changes

The request is to add a global inbound message suffix directive such as `reply in thread` or `rit` that strips the trigger before the agent sees it and forces the response into the inbound message thread.

## Best Possible Solution

If accepted, add one typed suffix-only inbound directive in shared inbound finalization, feed the existing reply-threading payload path, document the approved trigger policy, and cover both agent/command stripping and payload-routing tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests new user-facing directive behavior rather than broken documented behavior. Source inspection on current main gives high confidence that the inbound suffix parser and per-turn carrier field do not exist.

## Solution Assessment

Unclear until maintainers approve the trigger vocabulary and config policy. The maintainable path would be a shared inbound directive feeding the existing reply-threading path, not channel-specific prompt instructions.
