## Summary

Keep open. Current main still supports assistant-output reply tags and reply policies, but it does not implement the requested shared inbound suffix directive, and the exact global trigger vocabulary needs maintainer product approval before a fix lane should implement it.

## What This Changes

Add a built-in inbound suffix directive such as `reply in thread` or `rit` that strips the suffix from user text and forces the outbound reply to target the inbound message thread.

## Best Possible Solution

If maintainers accept the feature, add a typed suffix-only directive in shared inbound finalization, approve a narrow trigger set, carry the result into existing reply payload threading, and document/test the behavior across the shared path.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests a new user-facing directive. Source inspection gives high confidence that current main lacks the suffix parser, context carrier, and payload plumbing for the requested behavior.

## Solution Assessment

Unclear until maintainers approve whether global magic suffixes, especially bare `thread`, belong in core or need a stronger plugin/API seam first. If approved, the shared inbound finalization path plus existing reply-threading payload path is the maintainable implementation shape.
