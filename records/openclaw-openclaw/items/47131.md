## Summary

Keep open. Current main still supports outbound reply tags and reply-threading policy, but it does not implement a shared inbound suffix directive; choosing global magic trigger words such as `thread` needs maintainer product approval before code work is safe.

## What This Changes

Add an inbound user directive that strips suffixes such as `reply in thread` or `rit` from incoming text and forces the outbound response to reply to the current message.

## Best Possible Solution

If accepted, add a typed suffix-only inbound directive in shared inbound finalization, route it into the existing reply-threading payload path, document the approved trigger policy, and cover `BodyForAgent`/command stripping tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this requests a new user-facing directive. Source inspection on current main gives high confidence that the inbound suffix parser and carrier field do not exist.

## Solution Assessment

Unclear until maintainers approve the trigger vocabulary and whether any config surface should exist. The maintainable implementation would be shared inbound finalization plus existing reply-threading payload logic, not channel-specific prompt instructions.
