## Summary

## What This Changes

Request adds a cross-channel inbound user directive that strips trailing phrases like `reply in thread` or `rit` and forces the next response to reply to the inbound message.

## Best Possible Solution

Decide the product contract first; if accepted, implement a conservative shared inbound directive with tests for `BodyForAgent` stripping, replyToMode interaction, and false-positive trigger phrases.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a new inbound directive request. Source inspection confirms the requested built-in phrase parsing is absent from current main and latest release.

## Solution Assessment

Unclear until maintainers decide the product shape. The direction is plausible, but the safe solution needs a narrower trigger set, `BodyForAgent` handling, and an explicit interaction with channel replyToMode/config before implementation.
