## Summary

Keep open: current main does not implement `allowedExternalPaths`, and this PR still changes a security-sensitive workspace file trust boundary with source-backed blockers and no real setup proof.

## What This Changes

The PR adds a default/per-agent `workspaceConfig.allowedExternalPaths` allowlist for trusted external symlink targets across agent file APIs, bootstrap context loading, docs, logging, schemas, and tests.

## Best Possible Solution

If maintainers want this config, implement it as an opt-in external-path authority on top of the current fs-safe `WorkspaceRoot` and refreshed bootstrap loader, with generated config metadata, docs/changelog, focused symlink tests, and real setup proof.

## Reproduction Assessment

Yes for the PR blockers: the PR head source shows the lowercase alias-write guard, session-key cache reuse before reload, and missing schema help/label metadata. The requested `allowedExternalPaths` capability itself is a new feature, not a current-main bug reproduction.

## Solution Assessment

No, not in this branch shape. The allowlist direction may be useful, but the maintainable path is to rebase onto current fs-safe and bootstrap-refresh code, fix the remaining safety/metadata gaps, and require real setup proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject case-only symlink aliases on writes:** `src/gateway/server-methods/agents.ts:882`
  - body: The guard lowercases both the resolved write path and requested name, so on a case-sensitive workspace a symlink such as `MEMORY.md -> memory.md` can pass as a casing-only match and write a different file. That bypasses the PR's documented rule that in-workspace symlink aliases are read-only.
  - confidence: 0.88
- **[P2] Preserve bootstrap refresh when allowlists change:** `src/agents/bootstrap-cache.ts:10-12`
  - body: This returns cached bootstrap files for a session before reloading, but the loaded content now depends on `allowedExternalPaths`. Removing an allowlisted directory can therefore leave previously cached external context injected into long-lived sessions.
  - confidence: 0.9
- **[P2] Add config help and label metadata for the new key:** `src/config/zod-schema.agent-defaults.ts:39-44`
  - body: The PR adds `workspaceConfig` to the runtime schema, but it does not add matching `FIELD_HELP`/`FIELD_LABELS` metadata. The new public config key will appear in `config.schema` and Control UI/help surfaces without the field documentation expected for config additions.
  - confidence: 0.86
