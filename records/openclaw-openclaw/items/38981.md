Decide the workspace symlink trust model explicitly, then either land a rebased allowlist implementation that preserves current cache/fs-safe protections and negative tests, or supersede it with the narrower scoped fix in https://github.com/openclaw/openclaw/pull/73401.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve dangling external symlink targets for writes:** `src/gateway/server-methods/agents.ts:238-243`
  - body: When an allowlisted symlink points to an external file that does not exist yet, `realpath()` throws and this code returns a missing result with `ioPath` set to the symlink itself. `agents.files.set` then treats it as an in-workspace write and the safe writer rejects the symlink, so `set` still cannot create the trusted external target even though normal missing workspace files are creatable and the docs promise read/write support. Resolve the link target with `readlink` for the missing-target case, validate it against `allowedExternalPaths`, and pass that external path into the write branch.
  - confidence: 0.88
- **[P2] Preserve bootstrap cache refresh:** `src/agents/bootstrap-cache.ts:10-15`
  - body: This branch returns the first cached `sessionKey` snapshot without reloading. Current main refreshes bootstrap files each turn so long-lived sessions pick up edits; with this PR, changing `allowedExternalPaths` or updating the shared symlink target can leave prompt context stale, or keep previously allowed external content after config changes, until session rollover. Rebase this helper onto current `BootstrapSnapshot` refresh logic and pass the allowlist into the fresh load.
  - confidence: 0.86
