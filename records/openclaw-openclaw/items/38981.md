## Summary

Keep open: current main still lacks the requested `workspaceConfig.allowedExternalPaths` contract, while this PR remains a broad external-read/write trust change with source-visible correctness and security blockers plus missing real behavior proof.

## What This Changes

The PR adds default and per-agent `workspaceConfig.allowedExternalPaths` support for trusted external symlinked agent workspace files across config schemas, gateway file APIs, bootstrap loading, docs, and tests.

## Best Possible Solution

Decide between the scoped fixed-bootstrap approach in https://github.com/openclaw/openclaw/pull/73401 and this broader writable allowlist; if the allowlist wins, rebase it onto current root-file APIs, fix the cache and dangling-target bugs, add negative security coverage, and require real setup proof.

## Reproduction Assessment

Yes. Source inspection shows current main still rejects external symlinked bootstrap files through the workspace-root reader, and the PR head's stale import, missing-target write path, and cache-refresh regression are visible in the changed source.

## Solution Assessment

No, not as-is. A broad allowlist may be viable, but this branch needs concrete fixes and maintainer security/product review against the narrower fixed-bootstrap-file alternative.

  - confidence: 0.86
