## Summary

Keep open for maintainer review: current main does not implement the allowlist, the PR still adds a broad external read/write trust boundary, and the latest head has source-visible blockers plus no real behavior proof.

## What This Changes

The PR adds default and per-agent `workspaceConfig.allowedExternalPaths` support for trusted external symlinked workspace files across agent file APIs, bootstrap loading, docs, and tests.

## Best Possible Solution

Decide the workspace-bootstrap trust model first; if this allowlist direction wins, rebase it onto current fs-safe APIs, preserve bootstrap cache refresh semantics, handle dangling external targets, add negative security coverage, and require real setup proof.

## Reproduction Assessment

Yes, by source inspection. Current main still rejects outside-workspace bootstrap symlinks through the workspace-root reader, and the PR-head compile/rebase, missing-target write, and cache-refresh defects are visible in the changed source.

## Solution Assessment

No, not as-is. The allowlist may be viable after maintainer trust-model approval, but the scoped bootstrap-reader approach is safer unless broad external read/write support is explicitly desired.
