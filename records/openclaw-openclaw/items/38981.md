  - body: Current main now opens bootstrap files through `openRootFile` with root-file guard semantics, but this branch still layers the allowlist onto the older `openBoundaryFile`/`openVerifiedFileSync` path.   - confidence: 0.88

  - body: This early return restores the old session-key cache behavior, so long-lived sessions keep stale AGENTS/SOUL/USER content after workspace files change. Current main reloads first and only reuses the existing snapshot when the refreshed files are equal.
  - confidence: 0.93
- **[P2] Preserve dangling external symlink targets for writes:** `src/gateway/server-methods/agents.ts:239-243`
  - body: When an allowlisted symlink points to a missing external file, `realpath` fails and this path returns `ioPath: candidatePath`, so `agents.files.set` later writes through the in-workspace symlink path and is rejected. The promised external write workflow needs to resolve or preserve the link target before returning the missing result.
  - confidence: 0.84
- **[P2] Do not lowercase the alias-write guard unconditionally:** `src/gateway/server-methods/agents.ts:882`
  - body: On case-sensitive filesystems, `AGENTS.md` and `agents.md` can be different paths. Lowercasing both sides allows a symlink alias that differs only by case to pass the write guard, contradicting the documented rule that in-workspace symlink aliases are read-only.
  - confidence: 0.87
