## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject case-only symlink aliases on writes:** `src/gateway/server-methods/agents.ts:882`
  - body: The write guard lowercases both paths after resolving the symlink target, so on a case-sensitive workspace a symlink such as `MEMORY.md -> memory.md` can pass as a casing-only match and write a different file. That bypasses the documented rule that in-workspace symlink aliases are read-only.
  - confidence: 0.88
- **[P2] Preserve bootstrap refresh when allowlists change:** `src/agents/bootstrap-cache.ts:10-12`
  - body: This returns cached bootstrap files for a session before reloading, while the loaded content now depends on `allowedExternalPaths`. Removing an allowlisted directory can therefore keep previously cached external context injected into long-lived sessions.
  - confidence: 0.9
- **[P2] Regenerate the public config metadata:** `src/config/zod-schema.agent-defaults.ts:39-44`
  - body: The PR adds `workspaceConfig` to the runtime schema but does not update the generated config schema/help/label metadata. The new public key will be missing from gateway config metadata and Control UI/help surfaces until those files are refreshed.
  - confidence: 0.86

## Security Review

Status: needs_attention

  - body: The write guard lowercases the resolved and requested paths, so a case-only symlink alias inside a case-sensitive workspace can bypass the intended read-only alias policy and write another workspace file.
