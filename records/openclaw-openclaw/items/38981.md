## Summary

Keep open: current main does not implement the requested external workspace symlink allowlist, but this PR still needs changes before merge because the head branch has source-reproducible security/config blockers and no real behavior proof from a symlinked workspace setup.

## What This Changes

The PR adds an opt-in `workspaceConfig.allowedExternalPaths` allowlist for symlinked agent workspace files and wires it through agent file APIs, bootstrap loading, schemas, docs, logging, and tests.

## Best Possible Solution

If maintainers want this capability, implement it as a narrow opt-in config on top of the current fs-safe and bootstrap-refresh contracts, with generated config metadata, focused tests, docs/changelog, and real setup proof.

## Reproduction Assessment

Yes for the PR blockers: the head code shows the case-folded alias-write guard, the session-key-only bootstrap cache, and missing generated config metadata. The requested allowlist itself is a new feature rather than a current-main bug reproduction.

## Solution Assessment

No, not in this branch shape. The allowlist direction may be useful, but the maintainable path is to rebase onto current fs-safe helpers, preserve bootstrap refresh semantics, regenerate metadata, and require real symlink setup proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject case-only symlink aliases on writes:** `src/gateway/server-methods/agents.ts:882`
  - body: The guard lowercases both the resolved write path and requested file name, so on a case-sensitive workspace a symlink such as `MEMORY.md -> memory.md` can pass as a casing-only match and write a different file. That bypasses the documented rule that in-workspace symlink aliases are read-only.
  - confidence: 0.88
- **[P2] Preserve bootstrap refresh when allowlists change:** `src/agents/bootstrap-cache.ts:10-12`
  - body: This returns cached bootstrap files for a session before reloading, while the loaded content now depends on `allowedExternalPaths`. Removing an allowlisted directory can therefore keep previously cached external context injected into long-lived sessions.
  - confidence: 0.9
- **[P2] Regenerate the public config metadata:** `src/config/zod-schema.agent-defaults.ts:39-44`
  - body: The PR adds `workspaceConfig` to the runtime schema but does not update the generated config help/label metadata. The new public key will be missing from gateway config metadata and UI/help surfaces until those generated files are refreshed.
  - confidence: 0.86

## Security Review
