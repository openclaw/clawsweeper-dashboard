## What This Changes

The PR adds default and per-agent `workspaceConfig.allowedExternalPaths` support for external symlinked agent workspace files across file APIs, bootstrap loading, config schemas, docs, logging, and tests.

## Best Possible Solution

Rebase the feature onto the current fs-safe-backed workspace file and refreshed bootstrap-cache contracts, keep symlink alias safety and config metadata aligned, and require focused tests plus redacted real symlink setup proof before merge.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject case-only symlink aliases on writes:** `src/gateway/server-methods/agents.ts:882`
  - body: The write guard lowercases both paths after resolving the symlink target, so on a case-sensitive workspace a symlink such as `MEMORY.md -> memory.md` can pass as a casing-only match and write a different file. That bypasses the PR's documented rule that in-workspace symlink aliases are read-only.
  - confidence: 0.88
- **[P2] Include allowlist state in bootstrap cache refresh:** `src/agents/bootstrap-cache.ts:10-12`
  - body: This returns cached bootstrap files for a session before reloading, while the loaded content now depends on `allowedExternalPaths`. Removing an allowlisted directory can therefore keep previously cached external context injected into long-lived sessions.
  - confidence: 0.9
- **[P2] Add metadata for the new config key:** `src/config/zod-schema.agent-defaults.ts:39-44`
  - body: Adding `workspaceConfig` to the defaults schema makes it a public config key, but the PR does not add matching labels/help metadata for `agents.defaults.workspaceConfig.allowedExternalPaths`. Gateway config metadata and Control UI/help surfaces will miss or poorly display the new setting until that contract is updated.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The patch broadens workspace file access and still has concrete symlink-alias and allowlist-revocation safety gaps.

Concerns:

- **[medium] Case-only symlink aliases can be written:** `src/gateway/server-methods/agents.ts:882`
  - body: The write guard lowercases the resolved and requested paths, so a case-only symlink alias inside a case-sensitive workspace can bypass the intended read-only alias policy and write another workspace file.
  - confidence: 0.88
- **[medium] Allowlist revocation can stay cached:** `src/agents/bootstrap-cache.ts:10`
  - body: Bootstrap content depends on `allowedExternalPaths`, but the PR returns a cached snapshot by session key before reloading or comparing allowlist state, so removed external trust can continue affecting an existing session.
  - confidence: 0.9
