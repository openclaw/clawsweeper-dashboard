## Summary

Keep open. Current main still does not implement `workspaceConfig.allowedExternalPaths`, so this PR is not obsolete, but the branch is not merge-ready: it weakens the in-workspace symlink write guard, can keep stale bootstrap allowlist decisions cached, and omits generated config metadata for the new public key.

## What This Changes

The PR adds an opt-in `workspaceConfig.allowedExternalPaths` allowlist for trusted external symlink targets across agent file APIs, bootstrap context loading, extra bootstrap files, config schemas, docs, and regression tests.

## Best Possible Solution

Rebase the PR onto current main, preserve current bootstrap refresh semantics while keying or validating cached entries against allowlist state, reject symlink aliases based on real alias status rather than lowercased path text, regenerate all config schema/help/label outputs, add the required changelog entry, and land only after focused agent file/bootstrap/config checks plus changed-gate validation pass.

## Reproduction Assessment

Yes. The blockers are reproducible by static inspection: a case-only symlink alias reaches the lowercased write guard, the PR cache returns before reloading with the new allowlist input, and generated config metadata lacks the new key.

## Solution Assessment

No. The opt-in allowlist direction may be acceptable, but this branch is not the best merge shape until it preserves the existing cache and file-write safety properties and updates the generated public config surfaces.
