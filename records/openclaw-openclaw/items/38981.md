## Summary

Keep open. Current main still lacks this external-symlink workspace allowlist, so the PR is not obsolete, but the branch is not merge-ready because it regresses existing `agents.files.set` create behavior, leaves generated config schema/help surfaces stale, and adds security-sensitive allowlist state without preserving the current bootstrap snapshot refresh/revocation behavior.

## What This Changes

The PR adds `agents.defaults.workspaceConfig.allowedExternalPaths` and per-agent allowlists for trusted external symlink targets across agent file APIs, bootstrap context loading, extra bootstrap hooks, config schemas, docs, and regression tests.

## Best Possible Solution

Rebase the PR onto current main, keep the opt-in allowlist narrow and absolute-path based, preserve current bootstrap refresh semantics, update generated config schema/help/labels, and land only after focused agent file/bootstrap tests plus config schema checks pass.

## Reproduction Assessment

Yes. The main absence is reproducible by source search, and the PR regressions are reproducible statically from the diff: `agents.files.set` now returns on the missing result before writing, and bootstrap cache output depends on `allowedExternalPaths` while the cache still short-circuits by session key.

## Solution Assessment

No. The requested direction may be acceptable, but this branch is not the best merge shape until it preserves existing file creation, updates generated config surfaces, and handles allowlist changes/revocation in cached bootstrap context.
