## Summary

Keep open for maintainer review: this PR is not implemented on current main and carries a real feature proposal, but the branch is not merge-ready because it broadens the workspace filesystem trust boundary, lacks real behavior proof, is conflicting with current root-file APIs, and still has source-reproducible correctness/security defects. ## What This Changes

The PR adds an opt-in `workspaceConfig.allowedExternalPaths` allowlist for trusted external symlink targets across agent workspace file reads, writes, bootstrap context loading, config schemas, docs, and tests.

## Best Possible Solution

Decide the external workspace trust model first; if the broad allowlist is accepted, rebase onto current root-file APIs, preserve per-turn bootstrap refresh semantics, fix dangling external symlink writes, add focused security regression coverage, and require redacted real setup proof.
