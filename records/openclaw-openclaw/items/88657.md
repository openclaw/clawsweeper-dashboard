## Summary

Keep open: current main and the latest release still block non-visible retries after replay-unsafe tool activity and return the same verify-before-retrying incomplete-turn warning. Recent merged work helps replay-safe empty-final and stop-with-tool-call cases, but it does not resolve the central `tools>0 replaySafe=no` recovery policy.

## What This Changes

The issue asks OpenClaw to recover provider turns that end with no visible payload after tool activity instead of surfacing the incomplete-turn error when replay is unsafe.

## Best Possible Solution

Keep this issue open for a maintainer-approved replay-safety policy and a narrow embedded-runner recovery path that retries only post-tool turns proven safe to reissue.

## Reproduction Assessment

Yes, at source level: current main still suppresses non-visible retries when replay metadata has potential side effects, then returns the reported incomplete-turn warning. I did not run a live DeepSeek/OpenRouter turn in this read-only review.

## Solution Assessment

No for the requested blanket retry when `replaySafe=no`; that could duplicate tool side effects. The maintainable solution is a narrow replay policy that proves which post-tool attempts are safe before retrying.
