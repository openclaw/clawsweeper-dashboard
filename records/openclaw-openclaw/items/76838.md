## Summary

Keep open: current main still has a plausible prompt-policy conflict between injected MEMORY.md guidance and the TTS system hint, and the latest human follow-up is still asking for truncation logs/config rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to consistently honor MEMORY.md guidance that forbids model-emitted TTS directives in a Telegram voice/TTS workflow.

## Best Possible Solution

Define TTS directive precedence explicitly, then align the TTS prompt and docs so the existing config-level directive-disable path suppresses [[tts:...]] guidance without making arbitrary MEMORY.md text a hidden runtime-policy override.

## Reproduction Assessment

No. Source inspection shows MEMORY.md is injected and the TTS hint can advertise [[tts:...]] after Project Context, but the issue lacks current-main gateway logs, /context output, messages.tts config, and a transcript proving the switch.

## Solution Assessment

Unclear. The safer maintainable path is config-driven TTS precedence, likely making buildTtsSystemPromptHint respect modelOverrides.enabled=false if maintainers confirm that policy, rather than treating MEMORY.md as a runtime-policy override.
