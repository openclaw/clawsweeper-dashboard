## Summary

Keep open: current main loads and refreshes `MEMORY.md`, but it also injects later TTS guidance that explicitly advertises `[[tts:...]]`, so the reported conflict is plausible and needs logs/config plus a maintainer precedence decision.

## What This Changes

The issue asks OpenClaw to consistently honor `MEMORY.md` instructions that forbid model-emitted TTS directives and require local Piper voice-file delivery.

## Best Possible Solution

Define TTS directive precedence, then either suppress directive guidance when model overrides are disabled or document the supported Piper/local-audio setup through `messages.tts` config.

## Reproduction Assessment

No. Source inspection gives a plausible current-main prompt conflict, but there is no attached Telegram/Kimi transcript, gateway log, `/context` output, or active TTS config proving the reported fallback path.

## Solution Assessment

Unclear. Making `MEMORY.md` override runtime policy would be broader than the existing contract; the narrower maintainable path is to settle precedence and use the existing `messages.tts.modelOverrides.enabled=false` behavior or adjust the TTS prompt around it.
