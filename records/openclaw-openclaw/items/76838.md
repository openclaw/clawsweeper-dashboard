## Summary

Keep open: current main still has a plausible prompt-policy conflict between injected `MEMORY.md` guidance and the enabled TTS system hint, and the newest human follow-up asks for gateway truncation logs rather than cleanup closure.

## What This Changes

Ensure agents consistently obey `MEMORY.md` instructions that forbid `[[tts:...]]` and require local Piper-generated Telegram voice messages.

## Best Possible Solution

Define the intended TTS precedence explicitly, then either make the TTS hint/config path honor a no-model-directives policy for this workflow or document the local Piper setup through `messages.tts` config instead of relying on `MEMORY.md`.

## Reproduction Assessment

No. Source inspection shows a plausible prompt conflict and a possible truncation diagnostic path, but there is no current-main Telegram/Kimi transcript or gateway log proving where the directive fallback starts.

## Solution Assessment

Unclear. Using `MEMORY.md` to suppress `[[tts:...]]` is fragile while the TTS system hint advertises directives; the narrower maintainable path is to define TTS directive precedence and use or adjust the existing `messages.tts.modelOverrides.enabled=false`/provider configuration behavior.
