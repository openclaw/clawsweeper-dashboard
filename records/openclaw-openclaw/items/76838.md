## Summary

Keep open: current main does inject MEMORY.md, but the TTS prompt also appends directive guidance whenever TTS is enabled, which can plausibly conflict with a stored rule that forbids `[[tts:...]]`; the exact Telegram/Kimi/Piper drift still needs live or focused prompt reproduction.

## What This Changes

The issue asks OpenClaw to keep following MEMORY.md instructions that forbid `[[tts:...]]` and require local Piper/Thorsten voice-message generation throughout Telegram sessions.

## Best Possible Solution

Make the TTS system prompt honor resolved TTS config and avoid encouraging model-authored directives when the configured provider path should synthesize plain replies, with regression coverage for the prompt text.

## Reproduction Assessment

No high-confidence live reproduction yet. Source inspection shows a plausible current-main prompt conflict, but I did not reproduce a Telegram voice-message session with kimi/k2p6 and Piper or observe the drift after several turns.

## Solution Assessment

Unclear but there is a narrow likely fix. The maintainable direction is not to make MEMORY.md hard-authoritative; it is to make `buildTtsSystemPromptHint` reflect `modelOverrides` and auto mode so configured local TTS can work without nudging the model toward `[[tts:...]]`.

## Repair Work Prompt

Investigate and fix the TTS prompt conflict behind openclaw/openclaw#76838. On current main, `MEMORY.md` is injected, but `buildTtsSystemPromptHint` always tells the model to use `[[tts:...]]` when TTS is enabled. Update the resolved TTS prompt text so configured auto-TTS/local CLI providers can synthesize plain replies without nudging the model toward directives. Honor `messages.tts.modelOverrides.enabled=false`; mention `[[tts:...]]`/`[[tts:text]]` only when model-provided directives are actually allowed or when `messages.tts.auto` is `tagged`. Preserve existing Telegram voice-note delivery, directive parsing, and streaming directive stripping. Add focused regression tests for the generated TTS prompt and update `docs/tools/tts.md` plus `CHANGELOG.md` if user-facing wording/behavior changes. Do not add a new config option and do not make MEMORY.md a hard policy layer.
