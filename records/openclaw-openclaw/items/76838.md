## Summary

Keep open: current source shows `MEMORY.md` is loaded for normal sessions, but TTS also has a higher-priority system hint that encourages `[[tts:...]]` when TTS is enabled, so this needs a maintainer decision rather than cleanup closure.

## What This Changes

The issue reports that session-loaded `MEMORY.md` instructions are not treated as durable guidance, allowing Telegram voice responses to fall back to `[[tts:...]]` after some turns.

## Best Possible Solution

Define the intended precedence explicitly: either make the TTS prompt/config path honor a no-model-directives policy, or document that local Piper should be configured through TTS settings with model overrides disabled instead of relying on `MEMORY.md`.

## Reproduction Assessment

No. Source inspection shows a plausible prompt conflict between injected `MEMORY.md` and the TTS system hint, but the report lacks logs or a deterministic current-main Telegram/Kimi transcript proving the raw-tag fallback.

## Solution Assessment

Unclear. Relying on `MEMORY.md` to suppress `[[tts:...]]` is not the cleanest fix while the TTS system prompt explicitly advertises directives; a safer solution is an explicit TTS policy/config or docs path.
