## What This Changes

The issue asks OpenClaw to make workspace `MEMORY.md` instructions against model-authored TTS directives hold consistently during Telegram voice sessions.

## Best Possible Solution

Define TTS directive precedence explicitly, then align the prompt with that policy, preferably by making config-disabled model overrides stop advertising `[[tts:...]]` while documenting the local Piper setup path.

No high-confidence live reproduction is available. Source inspection shows a plausible current-main conflict between injected `MEMORY.md` guidance and later TTS runtime guidance, but the report still needs logs/config/transcript proof for the affected Telegram/Kimi session.

## Solution Assessment
