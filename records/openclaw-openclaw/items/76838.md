## Summary

Keep open: current main appears to load `MEMORY.md`, but it also appends a later TTS system hint that tells the model to use `[[tts:...]]` directives when TTS is active. That is a plausible prompt-precedence conflict, not a cleanup-close candidate.

## What This Changes

The issue asks OpenClaw to consistently honor `MEMORY.md` instructions that forbid `[[tts:...]]` and require local Piper audio replies during Telegram voice sessions.

## Best Possible Solution

Define TTS directive precedence explicitly: either make the TTS prompt/config path authoritative for disabling model directives, or document that local Piper/no-directive behavior must be configured through TTS settings instead of relying on `MEMORY.md`.

## Reproduction Assessment

No high-confidence runtime reproduction is attached. Source inspection shows a plausible current-main prompt conflict when TTS is enabled, but it does not prove the Telegram/Kimi raw-tag fallback without a transcript or focused harness case.

## Solution Assessment

Unclear. Making `MEMORY.md` suppress a later TTS system hint is a product-policy decision; the narrower maintainable path is to make the existing TTS config/directive policy explicit and test-covered.
