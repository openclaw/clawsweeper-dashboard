## Summary

Keep open. Current main still leaves several focusable Control UI chat, mobile form, config, and usage controls below the 16px iOS Safari focus threshold, and open PR #64673 already references this issue with closing syntax.

## What This Changes

The issue asks for Control UI text-entry fields to compute to at least 16px on iOS so Safari does not auto-zoom when fields receive focus.

## Best Possible Solution

Land a narrow Control UI CSS patch that raises focusable text inputs, textareas, and selects to at least 16px on iOS or touch-primary devices while preserving desktop sizing, with a focused CSS check and iOS Safari verification.
