## Summary

Keep open: this focused bugfix is useful, but it only suppresses the Homebrew recommendation note; unsupported platforms can still select brew-backed skill installers and then receive Homebrew install guidance, so the linked FreeBSD behavior is not fully fixed.

## What This Changes

The PR changes skills onboarding to gate the Homebrew recommendation prompt to darwin/linux and adds a FreeBSD-stubbed unit test.

## Best Possible Solution

Use one supported-Homebrew-platform predicate across the prompt, brew-only install choices, and missing-brew guidance, then prove the unsupported-platform onboarding flow.

## Reproduction Assessment

Yes. The linked screenshot and current source show that selecting a brew-backed missing skill on a non-win32 platform with no brew reaches the Homebrew prompt; I did not run a live FreeBSD onboarding session in this read-only review.

## Solution Assessment

No. Gating only the prompt is a plausible partial fix, but the maintainable fix should apply the same platform contract before offering or executing brew-only installers and before printing missing-brew guidance.
