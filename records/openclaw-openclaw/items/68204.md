## Summary

Keep open. Current main still has separate low-level agent events, cache trace records, diagnostic trace context/events, trajectory bundles, ACP relay JSONL records, task ledger records, and Task Flow state rather than one canonical run trace schema adopted across agent, ACP, subagent, and task-flow paths. The related run-trace implementation PR #43145 is still open and unmerged, so this remains valid architecture work rather than cleanup-close material.

## What This Changes

Request to define and adopt one canonical run trace schema across core agent runs, ACP/subagent relays, task execution, and Task Flow state.

## Best Possible Solution

Keep this as the architecture/observability tracker until maintainers settle a versioned run-trace contract and map existing producers into it with compatibility adapters, focused contract tests, and docs that explain how it relates to trajectory bundles and diagnostics traces.

## Reproduction Assessment

Not applicable as a feature/architecture request. The high-confidence verification path is static inspection of current main, and that inspection shows the canonical schema and producer mappings are still absent.

## Solution Assessment

Unclear because no solution is merged on main. The likely maintainable path is a maintainer-reviewed, versioned trace contract, using #43145 as prior art where appropriate, rather than an automated broad rewrite across several runtime surfaces.
