## Summary

Keep open. Current main still has separate agent event, trajectory, diagnostics timeline, ACP stream-log/progress, task ledger, subagent registry, and Task Flow record shapes rather than one canonical run trace schema adopted across the requested flows; the related run-trace PR remains open and unmerged.

## What This Changes

The issue requests one canonical run trace schema shared by main agent runs, ACP sessions, subagents, and task flows, with parent-child linkage and step/tool-level events.

## Best Possible Solution

Keep this as an architecture/observability tracker until main has a small versioned run-trace contract plus adapters from agent events, trajectory/runtime events, ACP relay, subagent lifecycle, task records, and Task Flow state, with focused contract tests and docs.

## Reproduction Assessment

Not applicable as a feature/architecture request. The verification path is static inspection of current main: targeted searches and source/docs inspection show the unified schema is still missing.

## Solution Assessment

Unclear because no merged solution exists on main. The maintainable direction is a versioned contract that maps existing producers and clarifies its relationship to trajectory and diagnostics rather than adding another unowned parallel trace surface; #43145 should be reviewed as prior art.
