## Summary

Keep open. Current main has useful diagnostics, trajectory, agent-event, ACP relay, and task records, but not one canonical run-trace schema adopted across agent, ACP, subagent, and task flows; the directly related implementation attempt at https://github.com/openclaw/openclaw/pull/43145 is still open and unresolved, while https://github.com/openclaw/openclaw/pull/70980 only added trace-context propagation for diagnostics hooks.

## What This Changes

Request a canonical run trace schema with parent-child, step, tool, task, status, and timing fields adopted by agent, ACP, subagent, and task execution flows.

## Best Possible Solution

Keep this as the architecture tracker until maintainers define a versioned, redacted, authorized run-trace contract and then land focused adapters/tests for the existing producers, using https://github.com/openclaw/openclaw/pull/43145 as prior art rather than merging it wholesale.

## Solution Assessment
