## Summary

Keep open. Current main still has separate agent event, diagnostic trace context, diagnostics timeline, trajectory bundle, ACP relay log, task ledger, subagent registry, and Task Flow state shapes rather than one canonical run trace schema adopted across agent, ACP, subagent, and task-flow paths; the directly related run-trace PR is still open and unmerged.

## What This Changes

Define and adopt a canonical run trace schema across agent runs, ACP parent-child streams, subagent/task execution, and Task Flow lifecycle/progress events.

## Best Possible Solution

Keep this as an architecture/observability tracker until maintainers agree on a small versioned run-trace contract with adapters from agent events, diagnostic trace context, trajectory runtime events, ACP relay, subagent lifecycle, task records, and Task Flow state, plus focused contract tests and docs explaining the relationship to trajectory bundles.

## Reproduction Assessment

Not applicable as a feature/architecture request. The high-confidence verification path is static inspection of current main: targeted searches plus source/docs review show the unified schema and producer mappings are still missing.

## Solution Assessment

Unclear because no solution is merged on main. The maintainable path is a maintainer-reviewed versioned trace contract, likely using #43145 as prior art, rather than an automated broad rewrite across several runtime and observability surfaces.
