## Summary

Keep open. Current main still has separate agent event, trajectory, ACP stream-log, task ledger, and Task Flow state shapes rather than one canonical run trace schema adopted across agent, ACP, subagent, and task-flow paths; the directly related run-trace PR is still open and unmerged.

## What This Changes

Define and adopt a unified run trace schema for main agent runs, ACP sessions, subagents, and task/task-flow execution events, including parent-child, step, tool, status, and timing fields.

## Best Possible Solution

Keep this as an architecture/observability tracker until maintainers merge a small versioned run-trace contract with adapters from agent events, trajectory/runtime events, ACP relay, subagent lifecycle, task records, and Task Flow state, plus focused contract tests and docs that explain the relationship to trajectory bundles.

## Reproduction Assessment

Not applicable as a feature/architecture request. The high-confidence verification path is static inspection of current main: targeted searches plus source/docs review show the unified schema and adopted producer mappings are still missing.

## Solution Assessment

Unclear because there is no merged solution on main. The maintainable path is a maintainer-reviewed versioned trace contract, using #43145 as prior art, rather than an automated broad rewrite of several observability and runtime surfaces.
