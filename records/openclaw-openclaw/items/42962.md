## Summary

Keep open. The PR targets a real user-facing warning leak, but its guard only suppresses errors that are already marked read-only, while current `main` still classifies `gateway` calls with `action: "config.schema.lookup"` as mutating.

## What This Changes

The PR adds a targeted payload warning suppression for non-mutating `gateway` `config schema path not found` errors and one regression test for that synthetic non-mutating state.

## Best Possible Solution

Land one coordinated fix for #42856 that marks gateway `config.schema.lookup` as read-only in production and suppresses only the exact benign path-not-found lookup miss.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: `gateway` with `action: "config.schema.lookup"` is classified as mutating, the gateway returns `config schema path not found` for a missing schema path, and the payload warning policy still surfaces mutating tool failures.

## Solution Assessment

No. The proposed payload exception is a useful half of the fix, but it is not the best complete solution unless paired with the mutation-classification change or moved before the mutating branch for this exact benign lookup miss.
