## Summary

Keep open. Current main still has the user-facing warning path, and the PR is useful but incomplete because the production runtime still records `gateway` `config.schema.lookup` calls as mutating before payload rendering.

## What This Changes

The PR adds a targeted payload warning suppression for non-mutating `gateway` `config schema path not found` errors plus one regression test for that synthetic non-mutating state.

## Best Possible Solution

Coordinate this PR with #42898 or fold both changes together: classify compound gateway read-only actions by leaf verb, suppress only the exact benign `config.schema.lookup` path-not-found miss, and cover the production mutation-to-payload path plus inverse cases for unrelated or mutating gateway failures.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: `gateway` with `action: "config.schema.lookup"` is currently classified as mutating, the gateway returns `config schema path not found` for a missing schema path, and payload rendering warns on unresolved mutating tool errors.

## Solution Assessment

No. The PR is a reasonable payload half, but by itself it is not the best complete fix because its strict `mutatingAction === false` condition does not match the current production state for this gateway action.
