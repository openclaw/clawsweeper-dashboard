## Summary

Keep open: this targets a real warning-noise bug, but the patch is not correct on current main because the reported `gateway` `config.schema.lookup` call is still marked mutating, so the new `mutatingAction === false` suppression will not fire. The PR also lacks after-fix real behavior proof and is currently conflicting with main.

## What This Changes

The PR adds a payload warning-policy guard that suppresses `gateway` `config.schema.lookup` path-not-found warnings when `lastToolError.mutatingAction` is explicitly false, plus a regression test for that synthetic read-only error.

## Best Possible Solution

Land one coherent fix that classifies `gateway` `config.schema.lookup` as read-only and suppresses benign path-not-found chat notifications with regression coverage and redacted runtime or Telegram proof, reconciling the overlap with https://github.com/openclaw/openclaw/pull/42898.

## Reproduction Assessment

Yes, source-reproducible. Current main shows `config.schema.lookup` is a read-scoped gateway method that can return `config schema path not found`, while the agent classifier still marks that dotted gateway action as mutating and feeds that bit into warning rendering.

## Solution Assessment

No. This PR suppresses only errors already marked non-mutating, but the reported runtime path is marked mutating on current main; the durable fix needs classifier correction and, if desired, a targeted payload suppression for the benign miss.
