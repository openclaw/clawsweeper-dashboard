## Summary

Keep open: this PR is useful source work, but it does not yet fix the live path because the new suppression requires `mutatingAction === false` while current main still classifies `gateway` `config.schema.lookup` as mutating.

## What This Changes

The PR adds targeted payload-warning suppression for `gateway` `config schema path not found` errors when `mutatingAction === false`, plus one regression test for that state.

## Best Possible Solution

Land one coordinated fix for #42856 that classifies compound read-only gateway lookups correctly and suppresses only the exact benign path-not-found lookup warning.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: `gateway` with `action: "config.schema.lookup"` and a missing path returns `config schema path not found`, but current mutation classification marks the compound action as mutating and the payload policy emits a warning for mutating failures.

## Solution Assessment

No. The proposed payload exception is a useful part of the fix, but it is not sufficient unless paired with the classifier change from #42898 or moved so the exact benign lookup miss bypasses the mutating warning branch.
