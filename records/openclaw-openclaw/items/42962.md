## Summary

Keep open. The PR targets a real user-visible gateway warning bug, but the proposed suppression depends on `mutatingAction === false` while the current runtime still classifies `gateway` `config.schema.lookup` as mutating, so the reported path can continue to warn; the external PR also lacks after-fix real behavior proof.

## What This Changes

The PR adds a payload warning-policy exception for gateway `config.schema.lookup` path-not-found errors when `lastToolError.mutatingAction` is false, plus one regression test.

## Best Possible Solution

Land one coordinated fix that classifies compound gateway lookup as read-only, suppresses the exact benign path-not-found warning, preserves warnings for real mutating gateway failures, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `config.schema.lookup` misses return `config schema path not found`, current main still marks the dotted gateway action as mutating, and payload rendering turns mutating tool errors into user-facing warnings.

## Solution Assessment

No. The payload exception is useful, but this PR is incomplete unless it is paired with the classifier fix from https://github.com/openclaw/openclaw/pull/42898 or otherwise moved so the exact benign lookup miss bypasses the mutating-warning branch in the real runtime path.
