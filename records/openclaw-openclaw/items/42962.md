## Summary

Keep open: the PR targets a real bug, but the proposed suppression depends on `mutatingAction === false`, and current main still classifies `gateway` `config.schema.lookup` as mutating, so the runtime warning path is not fixed. It also needs the required user-facing changelog entry before merge.

## What This Changes

The PR adds targeted payload-warning suppression for `gateway` `config.schema.lookup` path-not-found errors when `lastToolError.mutatingAction` is false, plus one regression test.

## Best Possible Solution

Land one coordinated fix that classifies compound read-only gateway lookups correctly, suppresses only the exact benign path-not-found lookup warning, adds focused regression coverage, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `gateway` with `action: "config.schema.lookup"` and a missing path returns `config schema path not found`, but current mutation classification treats the compound action as mutating and payload rendering emits a mutating-tool warning.

## Solution Assessment

No. The payload exception is useful but incomplete unless paired with the compound-action classifier fix or moved so the exact benign lookup miss bypasses the mutating warning branch in the real runtime path.
