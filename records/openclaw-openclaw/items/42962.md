## Summary

Keep open. The PR targets a real user-facing warning leak, but the proposed guard only suppresses already-read-only gateway errors while current main still classifies `gateway` `config.schema.lookup` as mutating, so the patch does not fix the production path by itself.

## What This Changes

The PR adds a targeted payload warning suppression for non-mutating `gateway` `config schema path not found` errors and one regression test for that synthetic non-mutating state.

## Best Possible Solution

Land one coordinated fix for #42856: classify compound gateway read-only actions by leaf verb, suppress only the exact benign `config.schema.lookup` path-not-found miss, and add tests for the production mutation-to-payload path plus inverse cases for unrelated or mutating gateway failures.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: `gateway` with `action: "config.schema.lookup"` is currently classified as mutating, the gateway returns `config schema path not found` for a missing schema path, and the payload warning policy still surfaces mutating tool failures.

## Solution Assessment

No. The proposed payload exception is a useful half of the fix, but it is not the best complete solution unless paired with the mutation-classification change or moved before the mutating branch for this exact benign lookup miss.
