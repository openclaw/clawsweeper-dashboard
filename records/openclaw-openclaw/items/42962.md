## Summary

Close as duplicate/superseded: the earlier open classifier-level PR tracks the same bug and fixes the root cause, while this payload-only guard would not fire in the reported runtime path because `config.schema.lookup` failures currently carry `mutatingAction: true`.

## What This Changes

The PR adds a payload-rendering guard that suppresses `gateway` `config schema path not found` warnings when `lastToolError.mutatingAction` is explicitly false, plus a focused regression test.

## Best Possible Solution

Keep the root-cause work on https://github.com/openclaw/openclaw/pull/42898 and https://github.com/openclaw/openclaw/issues/42856: fix gateway compound read-only action classification in `src/agents/tool-mutation.ts`, then validate with real gateway or Telegram warning proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `gateway` `config.schema.lookup` is read-only at the gateway layer but is currently classified as mutating before failed-tool warning rendering.

## Solution Assessment

No. The payload-only special-case is not the best fix because it depends on `mutatingAction === false`, while the reported runtime path sets that flag to `true`; the classifier-level PR is the narrower durable fix.

## Review Findings
