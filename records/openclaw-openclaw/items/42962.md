## Summary

Keep open. The reported bug is real, but this PR's payload guard only fires when `mutatingAction` is already `false`; current main marks `gateway` `config.schema.lookup` as mutating before payload rendering, so the reported warning path remains. The PR also lacks after-fix real behavior proof and overlaps the classifier-level fix in https://github.com/openclaw/openclaw/pull/42898.

## What This Changes

The PR adds a payload warning suppression for `gateway` `config.schema.lookup` path-not-found errors when the tool error is marked non-mutating, plus one regression test.

## Best Possible Solution

Fix the gateway action classifier so dotted read-only actions like `config.schema.lookup` become non-mutating before payload rendering, then decide whether an exact benign-error payload suppression is still needed and prove the user-visible channel behavior.

## Reproduction Assessment

Yes. Source inspection shows the gateway tool exposes `config.schema.lookup`, the method is read-scoped and can return `config schema path not found`, and current mutation classification still marks the dotted action as mutating before warning rendering.

## Solution Assessment

No. The proposed payload guard tests a state that the current runtime does not produce for this action; the durable fix belongs in the mutation classifier, with this PR at most serving as a later complement for no-reply benign lookup misses.
