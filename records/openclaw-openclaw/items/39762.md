## Summary

Keep open. Current main does not implement this Feishu read-only group mode, and the PR still has two correctness blockers plus missing real Feishu runtime proof before it can be merged.

## What This Changes

The PR adds a Feishu `channels.feishu.disableGroupReplies` config/docs/tests path to ingest group chats while suppressing group replies and explicit group send targets.

## Best Possible Solution

If maintainers want this guardrail, keep it Feishu-local, use a contract-complete shared no-op dispatcher, reject every resolved group `chat_id` target when disabled, update focused docs/tests, and require redacted Feishu runtime proof.

## Reproduction Assessment

Yes for the PR defects via source inspection: the PR head no-op dispatcher omits `getFailedCounts`, and the outbound guard misses bare `oc_*` targets that resolve as `chat_id`. Live Feishu behavior proof is still missing.

## Solution Assessment

No. The Feishu-local direction may be viable, but this patch is not mergeable until it satisfies the dispatcher contract, blocks resolved group chat targets, supplies real Feishu proof, and gets maintainer acceptance for the new config surface.
