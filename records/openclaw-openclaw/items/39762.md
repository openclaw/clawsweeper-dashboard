## Best Possible Solution

If maintainers want this guardrail, keep it Feishu-local, use a complete shared no-op dispatcher, reject every resolved group `chat_id` target when disabled, update focused docs/tests, and require real Feishu proof before merge.

## Solution Assessment

No. The Feishu-local direction may be viable, but this patch is not mergeable until it satisfies the dispatcher contract, blocks resolved group chat targets, supplies real Feishu proof, and gets maintainer acceptance for the new config surface.
