## Summary

Keep open: current main still lacks the Feishu read-only group mode, and the PR remains a meaningful feature proposal, but the current head is not mergeable because it has two source-proven blockers, no real Feishu behavior proof, and a new config/policy surface that needs maintainer acceptance.

## What This Changes

Adds `channels.feishu.disableGroupReplies` to Feishu config/docs/tests and routes disabled group chats through no-op reply dispatchers while blocking explicit group send targets.

## Best Possible Solution

If maintainers want read-only Feishu groups, land a rebased Feishu-local implementation that uses a shared contract-complete no-op dispatcher, blocks all resolved group chat sends when disabled, documents/tests the config, and includes real Feishu proof.

## Solution Assessment

No. The Feishu-local direction may be viable, but this patch is not the best mergeable solution until the dispatcher contract, resolved group-target blocking, rebase, real behavior proof, and config-policy acceptance are handled.
