Keep this PR open: current main still lacks the Feishu read-only group mode, but the PR is not merge-ready because its no-op dispatcher is type-incomplete, raw Feishu chat IDs can still bypass the outbound guard, and the user-facing feature lacks a changelog entry.

## What This Changes

The PR adds a Feishu `disableGroupReplies` config/docs switch, no-op group reply dispatch paths, explicit group-target rejection, Feishu tests, and a secrets-baseline line/timestamp refresh.

## Best Possible Solution

Keep the feature in the Feishu plugin and finish it as a narrow channel guardrail: add the config/schema/docs/changelog key, use a complete shared no-op dispatcher for every accepted group inbound path, and reject outbound targets that resolve to `chat_id` while preserving DM/user sends.

Yes. Current-main source inspection shows the feature is absent, and PR diff inspection shows the no-op dispatcher misses required `getFailedCounts` while raw `oc_*` targets resolve to `chat_id` but are not rejected.

## Solution Assessment

No. The plugin-local direction is maintainable, but this patch should use a complete shared no-op dispatcher, block all resolved `chat_id` outbound targets, and add the required changelog entry before merge.
