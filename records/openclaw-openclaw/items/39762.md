## Summary

Keep open: the Feishu read-only group mode is not implemented on current main, and this PR is useful, but the head still has merge-blocking issues around an incomplete no-op dispatcher and an outbound raw `chat_id` bypass.

## What This Changes

The PR branch adds a Feishu `disableGroupReplies` config/docs switch, no-op group reply dispatch paths, explicit group target blocking, Feishu tests, and a secrets-baseline refresh.

## Best Possible Solution

Keep the feature Feishu-plugin-local, but finish it with one complete shared no-op dispatcher, reject every resolved `chat_id` outbound target when the flag is enabled, preserve DM/user sends, and add the active-version changelog entry.

Yes. Source and PR-diff inspection show current main lacks the feature, while the PR diff omits required `getFailedCounts` on no-op dispatchers and still lets bare `oc_...` targets resolve to `chat_id`.

## Solution Assessment

No. The plugin-local direction is the right shape, but this patch should share a complete typed no-op dispatcher, block all resolved group chat targets, and add the required changelog entry before merge.
