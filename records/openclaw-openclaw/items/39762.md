## Best Possible Solution

Keep the feature in the Feishu plugin and finish it as a narrow channel guardrail: schema/docs/changelog support, a complete shared no-op dispatcher for all accepted group inbound paths, and outbound rejection for every target that resolves to `chat_id` while preserving DM/user sends.

## Solution Assessment

No. The Feishu-local direction is maintainable, but this patch should use a complete typed no-op dispatcher, block all resolved `chat_id` outbound targets, adapt to current main, and add the required changelog entry before merge.
