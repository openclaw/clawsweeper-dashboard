## Summary

Keep open: current main does not implement the Feishu read-only group mode, and this PR is on the right plugin-local surface but still has blocking correctness gaps in the no-op dispatcher shape and outbound chat-id blocking.

## What This Changes

The PR adds a Feishu `disableGroupReplies` config/docs switch, routes group replies through no-op dispatchers when enabled, blocks explicit group send targets, and adds Feishu tests plus a secrets-baseline refresh.

## Best Possible Solution

Keep the feature in the Feishu plugin and finish it as a narrow channel guardrail: add the config/schema/docs/changelog key, use a shared complete no-op dispatcher for every accepted group inbound path, and reject outbound targets that resolve to `chat_id` while preserving DM/user sends.

## Reproduction Assessment

Yes. Current-main source inspection shows the feature is absent, and PR diff inspection shows the no-op dispatcher misses required `getFailedCounts` while raw `oc_*` targets resolve to `chat_id` but are not rejected.

## Solution Assessment

No. The plugin-local direction is maintainable, but this patch should use a complete shared no-op dispatcher, block all resolved `chat_id` outbound targets, and add the required changelog entry before merge.
