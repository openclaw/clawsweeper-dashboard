## Summary

This PR should stay open because the requested Feishu read-only group mode is not on current main, and the branch still has correctness blockers plus missing real Feishu proof.

## What This Changes

The PR adds a Feishu `disableGroupReplies` config/docs/tests path intended to keep group messages inbound-only while preserving DM replies and blocking group send targets.

## Best Possible Solution

Land a rebased Feishu-local implementation only if maintainers want this policy: use a complete no-op dispatcher, block all resolved group chat sends while preserving DMs, keep schema/docs/tests aligned, and provide redacted real Feishu proof.

## Reproduction Assessment

Yes for the PR defects via source inspection: the PR no-op dispatchers omit a required dispatcher method, and raw `oc_*` targets resolve as group `chat_id` while the new guard only rejects explicit prefixes. No live Feishu runtime reproduction or proof is present.

## Solution Assessment

No. The Feishu-local guardrail direction is plausible, but this patch is not mergeable until the dispatcher contract, resolved group target guard, real behavior proof, and config-policy acceptance are handled.
