## Summary

Keep open: the protected `maintainer` label blocks cleanup-close, and current main still has no explicit scheduled-preview mode for manual cron runs. Source inspection shows manual runs now go through the Gateway cron service and shared executor, but live macOS LaunchAgent/Windows parity for env and context loading is still unproven and needs maintainer product/API direction.

## What This Changes

The issue requests that manual `cron run` executions match scheduled cron execution for working directory, environment, and context loading.

## Best Possible Solution

Maintainers should define the manual cron preview contract, then either document the current Gateway-queued parity as sufficient or add a narrow scheduled-preview mode that applies workspace, env, and context through run-scoped runner seams.
