## Summary

Keep this open: current main and latest `v2026.6.6` still route the Control UI/Dashboard paperclip through `.click()` on a hidden file input, while the macOS Dashboard surface is still a `WKWebView`; the earlier WebChat attachment fix was merged before this report and does not prove the macOS picker failure is fixed.

## What This Changes

Reports that macOS desktop WebChat in v2026.5.28 does not open a file picker when the paperclip attachment button is clicked.

## Best Possible Solution

Make the Dashboard/Control UI attachment trigger open the macOS file picker from a user-activation-preserving path or native bridge, while keeping browser Control UI and native SwiftUI Chat attachment behavior intact.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection matches the reported path: current main and `v2026.6.6` still click a hidden Control UI file input inside a WKWebView-backed Dashboard.

## Solution Assessment

Unclear until macOS Dashboard proof is captured. Closing based on the native SwiftUI Chat `NSOpenPanel` path or the earlier browser-focused attachment PR would be too broad; the best fix should target the reported Dashboard/Control UI picker path.
