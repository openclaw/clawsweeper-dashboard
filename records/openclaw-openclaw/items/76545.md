## What This Changes

Yes. Source inspection shows current main and `v2026.6.6` filter the reaper with `isCronRunSessionKey`, while isolated cron base keys are `agent:<id>:cron:<jobId>` and do not match that predicate.

## Solution Assessment

No. The `isCronSessionKey` swap is a useful clue but too broad as a final fix; the safer solution must protect durable base-session metadata or explicitly approve its deletion.
