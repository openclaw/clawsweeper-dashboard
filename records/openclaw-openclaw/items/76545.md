## What This Changes

The PR replaces the cron reaper’s run-only key predicate with `isCronSessionKey` and adds an Unreleased changelog fix entry for pruning base cron sessions.

## Best Possible Solution

Land the predicate change after rebasing to a clean diff, with regression coverage for expired base cron keys and comments/docs/log wording aligned to pruning cron sessions, not only run sessions.

## Reproduction Assessment

Yes. Source inspection is high-confidence: current main’s sweep loop gates on `isCronRunSessionKey`, and that regex rejects `agent:main:cron:<jobId>` while accepting only `cron:<jobId>:run:<runId>`.

## Solution Assessment

Mostly yes. Switching to `isCronSessionKey` is the narrow code fix, but the PR should add a focused regression test, clean the stale changelog diff, and update run-only wording before merge.
