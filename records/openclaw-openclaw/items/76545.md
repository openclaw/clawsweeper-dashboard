## Summary

Keep open. Current main still has the cron base-session pruning bug, and this PR is the right narrow code direction, but it lacks after-fix real behavior proof and needs the public docs/config help aligned with the broadened retention behavior.

## What This Changes

The PR switches cron retention cleanup from `isCronRunSessionKey` to `isCronSessionKey`, updates reaper wording/log output, adds expired base-session coverage, and edits the changelog.

## Best Possible Solution

Land a clean version that changes the cron-session predicate, preserves focused expired-base coverage, updates retention docs/help, and closes the linked issue only after merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed an expired `agent:main:cron:<jobId>` entry and call `sweepCronRunSessions`; current main rejects that key through `isCronRunSessionKey` before pruning.

## Solution Assessment

Mostly yes. Switching the sweep predicate to `isCronSessionKey` with an expired-base regression test is the narrow maintainable fix, but the PR also needs docs/help alignment and real behavior proof before merge.
