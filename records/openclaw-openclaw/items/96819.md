## What This Changes

The branch widens configured-model Qwen thinking compatibility checks and the vLLM thinking-policy provider gate to treat `bailian` like `vllm`.

## Best Possible Solution

Keep any supported custom `bailian` inference in the generic configured-model paths, remove the vLLM policy change, add focused regression coverage, and require redacted proof for a Bailian Qwen config where this patch actually changes behavior.
