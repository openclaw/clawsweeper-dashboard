## Summary

The PR remains necessary: current main only supports thread-bound persistent ACP sessions, while this branch adds a distinct parent-owned delegate mode for non-thread hub chats. The exact-head implementation is coherent, owner-scoped, and backed by strong real behavior proof; no discrete blocking correctness or security defect was found, but maintainers should explicitly accept the new session-state, delivery, protocol, and lifecycle compatibility contract before merge.

## What This Changes

Adds hub-owned persistent ACP delegate sessions with label-based follow-up, operator list/status/close commands, idle/max-age cleanup policy, gateway session metadata, documentation, and regression coverage.

## Best Possible Solution

Keep the explicit `delegate: true` mode as a separate parent-owned background contract, land it with the owner-scoped routing and fail-closed lifecycle checks intact, and leave generalized threadless `mode: "session"` behavior to the broader product discussion at https://github.com/openclaw/openclaw/issues/53548.
