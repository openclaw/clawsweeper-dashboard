## Summary

Keep this PR open: it targets a real duplicate progress-draft bug and has sufficient live proof, but the patch currently gates typing keepalive refreshes together with the progress callback.

## What This Changes

The PR narrows tool-stream `onToolStart` callbacks to `phase === "start"` and adds a focused unit test for a start/update/update/end tool event sequence.

## Best Possible Solution

Split tool progress from typing: emit `onToolStart` only for start phases while still refreshing typing on start and update phases.

## Reproduction Assessment

Yes. Source inspection shows args-less tool update events reach shared progress formatting on current main, and the PR supplies live Telegram/Discord before-and-after output showing the duplicate-line symptom and intended result.

## Solution Assessment

No, not as written. The start-only progress callback is the right direction, but the implementation should not remove update-phase typing TTL refreshes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

- **[P2] Keep typing refreshes on tool updates:** `src/auto-reply/reply/agent-runner-execution.ts:1791`
