## Best Possible Solution

Rebase and repair the shared progress bridge behavior so visible draft rows merge or filter duplicate args-less updates while update-phase lifecycle and typing signals remain available in both direct and queued follow-up paths.

Yes, source-reproducible: current `main` forwards args-less update tool events to `onToolStart`, and the renderer formats missing args as a lower-detail tool line. I did not run a live Telegram or Discord reproduction in this read-only review.

## Solution Assessment
