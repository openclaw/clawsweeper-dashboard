## Summary

Keep open. ## What This Changes

The PR narrows `runAgentTurnWithFallback` tool-event handling so `onToolStart` only fires for `phase: "start"` and adds a unit test for a start/update/update/end tool sequence.

## Best Possible Solution

Land a rebased repair that separates typing keepalive from visible progress rendering: update phases should still refresh typing, visible progress should avoid duplicate bare update rows, and both main and queued follow-up runners should have focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main forwards args-rich `start` and args-less `update` tool events into the same renderer, and the inspected live screenshots show the before/after duplicate-line behavior.

## Solution Assessment

No, not as-is. The maintainable fix should separate typing keepalive from visible progress rendering and apply the same phase policy to the queued follow-up runner path.
