## Summary

Keep this issue open. Current main still has the unguarded memory-wiki shared-search call, and open PR #75435 already references this issue as the focused fix candidate.

## What This Changes

The issue requests that `wiki_search` tolerate unavailable or malformed shared memory search when `memory-wiki` uses `search.backend="shared"` and `corpus="memory"` or `"all"`.

## Best Possible Solution

Review and land #75435, or an equivalent narrow memory-wiki guard, so optional shared memory failures cannot discard already-computed wiki results.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure `memory-wiki` with shared search, call `searchMemoryWiki` with corpus `all` or `memory`, and have `getActiveMemorySearchManager` return a truthy object without callable `search`; current main reaches `extensions/memory-wiki/src/query.ts:1217`.

## Solution Assessment

Yes. The best fix is a narrow memory-wiki consumer-boundary guard and fallback because shared memory search is optional augmentation and can cross third-party plugin/runtime boundaries.
