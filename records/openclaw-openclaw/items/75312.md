## Summary

Keep this issue open. Current main still has the unguarded `memory-wiki` shared-search call, and open PR #75435 is already a focused closing implementation candidate for the same bug.

## What This Changes

`wiki_search` should preserve wiki results when shared memory search is unavailable, malformed, or throws for `search.backend="shared"` with `corpus="memory"` or `"all"`.

## Best Possible Solution

Review and land #75435, or an equivalent narrow `memory-wiki` consumer-boundary guard, so optional shared memory failures cannot discard already-computed wiki results.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure `memory-wiki` with `search.backend="shared"`, call `searchMemoryWiki` with `corpus="all"` or `"memory"`, and have `getActiveMemorySearchManager` return a truthy object without callable `search`; current main reaches the unguarded call at `extensions/memory-wiki/src/query.ts:1217`.

## Solution Assessment

Yes. A narrow guard/fallback in `memory-wiki` is the best maintainable fix because shared memory search is optional augmentation across a plugin/runtime boundary.
