## Summary

Keep this issue open. Current main still has the unguarded `wiki_search` shared-memory call, and open PR #75435 already references this issue as the focused fix candidate.

## What This Changes

The issue requests that memory-wiki tolerate unavailable or malformed shared memory search managers when `search.backend="shared"` and the selected corpus includes memory.

## Best Possible Solution

Review and land PR #75435, or an equivalent narrow memory-wiki patch, so optional shared memory failures cannot discard already-computed wiki results.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure memory-wiki with shared backend, call `searchMemoryWiki` with corpus `all` or `memory`, and have `getActiveMemorySearchManager` return a truthy value without callable `search`; current main reaches the unguarded call at `extensions/memory-wiki/src/query.ts:1217`.

## Solution Assessment

Yes. A narrow guard and fallback at the memory-wiki consumer boundary is the best maintainable fix because shared memory search is optional augmentation supplied through a runtime/plugin contract.
