## Summary

Keep this issue open. Current main still has the unguarded `memory-wiki` shared-search call, and open PR #75435 already links this issue with closing syntax as the focused fix candidate.

## What This Changes

The issue asks for `wiki_search` to tolerate unavailable or malformed shared memory search and preserve wiki results when a shared corpus is selected.

## Best Possible Solution

Land #75435 or an equivalent narrow `memory-wiki` guard so shared search is optional and wiki results survive unavailable, malformed, or throwing active memory managers.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure shared search, request `corpus="memory"` or `"all"`, and have `getActiveMemorySearchManager` return a truthy object without callable `search`; current main reaches `extensions/memory-wiki/src/query.ts:1217`.

## Solution Assessment

Yes. A defensive fallback at the `memory-wiki` consumer boundary is the narrowest maintainable fix because shared memory search is optional augmentation supplied by runtime/plugin contracts.
