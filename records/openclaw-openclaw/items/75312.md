## Summary

Keep this issue open. Current main still has the unguarded `memory-wiki` shared-search call, and open PR #75435 already references this issue as the focused fix candidate.

## What This Changes

Make `wiki_search` tolerate malformed or throwing shared memory managers when `search.backend="shared"` and the requested corpus includes memory.

## Best Possible Solution

Review and land #75435, or an equivalent narrow `memory-wiki` patch, so optional shared memory failures preserve already-computed wiki results.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure `memory-wiki` with shared search, request `corpus="all"` or `"memory"`, and have `getActiveMemorySearchManager` return a truthy object without callable `search`; current main reaches `extensions/memory-wiki/src/query.ts:1217`.

## Solution Assessment

Yes. A defensive guard/fallback at the `memory-wiki` consumer boundary is the narrowest maintainable fix because shared memory search is optional augmentation supplied through runtime/plugin contracts.
