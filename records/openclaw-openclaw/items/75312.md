## Summary

Keep this issue open. Current main still contains the reported unguarded shared-memory search path, and open PR #75435 already references this issue with closing syntax, so cleanup should wait for that PR or an equivalent fix to merge.

## What This Changes

The issue requests that memory-wiki tolerate unavailable or malformed shared memory search when `wiki_search` uses `search.backend="shared"` with `corpus="all"` or `corpus="memory"`.

## Best Possible Solution

Review and land #75435, or an equivalent narrow patch, so memory-wiki validates that shared search is callable, catches shared-search failures, preserves wiki results for `corpus="all"`, and adds query/tool regression coverage without changing the public config contract.

## Reproduction Assessment

Yes. ## Solution Assessment

Yes. The narrowest maintainable fix is at the memory-wiki consumer boundary, because both built-in and third-party memory providers flow through that runtime object and shared search is optional augmentation.
