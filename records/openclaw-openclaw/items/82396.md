## Summary

Keep open: this draft PR has concrete correctness blockers in restart drain behavior and full-process trace coverage, and the PR body does not provide after-fix real gateway proof.

## What This Changes

Adds a gateway restart benchmark script/package alias plus restart trace logging around restart drain, close steps, and readiness, with tests for benchmark JSON/trace parsing.

## Best Possible Solution

Rework this as a smaller gateway-owned change that preserves current restart-drain semantics, either carries trace state through respawn or explicitly scopes tracing to in-process restarts, and includes redacted real gateway restart benchmark proof.

## Reproduction Assessment

Yes for the review findings from source inspection: current main has a 30s embedded-run cap and matching test, while PR head passes the full restart budget. The full-process trace gap is also source-reproducible because normal respawn starts a fresh process without the PR's module-local trace state.

## Solution Assessment

No. The benchmark and trace direction may be useful, but this implementation regresses restart-drain behavior and does not cover the default full-process restart path it claims to trace.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve the short embedded-run drain grace:** `src/cli/gateway-cli/run-loop.ts:444-450`
  - body: This rewrites the drain back to a single `Promise.all` and passes the full restart drain budget to `waitForActiveEmbeddedRuns`. Current main intentionally caps embedded runs at 30s and aborts them before waiting for the full task-drain window; the existing run-loop test asserts `waitForActiveEmbeddedRuns(30_000)` plus the grace warning. With this change, a default restart can wait the whole task/restart budget while compaction holds session locks, delaying restart beyond current behavior.
  - confidence: 0.95
- **[P2] Carry restart trace state across full-process restarts:** `src/gateway/server.impl.ts:1466`
  - body: `finishGatewayRestartTrace` reads module-local state set by `startGatewayRestartTrace` in the process that received the signal. The normal fresh-PID/supervised restart path exits that process, while the benchmark forces `OPENCLAW_NO_RESPAWN=1`, so default full-process restarts will emit parent close spans but no child `restart.ready` span. Persist or hand off the trace start state, or scope the trace feature to in-process restarts.
  - confidence: 0.86
