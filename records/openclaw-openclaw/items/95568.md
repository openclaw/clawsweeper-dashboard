## Summary

Keep open: current main and the latest release still have the `sessions export-trajectory` store-resolution mismatch, and https://github.com/openclaw/openclaw/pull/95570 is the open candidate fix that should land or be closed before this issue is resolved.

## What This Changes

The issue requests that `openclaw sessions export-trajectory` resolve implicit and explicit session stores through the same shared resolver used by the rest of the sessions CLI.

## Best Possible Solution

Review and land or explicitly reject https://github.com/openclaw/openclaw/pull/95570, then close this issue once the resolver alignment is merged.

## Reproduction Assessment

Yes. Source inspection on current main shows `sessions export-trajectory` bypasses configured `session.store` and the shared resolver while sibling sessions commands route through `resolveSessionStoreTargetsOrExit` and `resolveStorePath`.

## Solution Assessment

Yes. The narrow shared-resolver alignment in https://github.com/openclaw/openclaw/pull/95570 is the best fix shape because it preserves explicit `--store` precedence while making implicit and templated paths match the existing sessions CLI contract.
