## What This Changes

The issue requests that `openclaw sessions export-trajectory` use the configured `session.store` default, including agent templating, when `--store` is omitted.

## Best Possible Solution

Land a narrow fix that makes implicit export-trajectory store resolution use the existing configured session-store path while preserving explicit `--store` override behavior.
