## What This Changes

The issue requests aligning `openclaw sessions export-trajectory` implicit and explicit store resolution with the shared sessions resolver.

## Solution Assessment

Yes. The best fix is the narrow shared-resolver alignment in the export command, preserving explicit `--store` precedence while making implicit and templated paths match the existing sessions CLI contract.
