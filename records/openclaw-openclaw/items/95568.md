Keep open: current main and the latest release still resolve `sessions export-trajectory` differently from the rest of the sessions CLI surface, and the open linked PR is the candidate fix path.

## What This Changes

The issue requests that `openclaw sessions export-trajectory` use the shared session-store resolver for both implicit configured stores and explicit `--store` paths.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/95570, then close this issue once the resolver alignment is merged.

## Reproduction Assessment

Yes. Source inspection on current main shows `sessions export-trajectory` bypasses configured `session.store`, while sibling sessions commands route through the shared resolver.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse `getRuntimeConfig` and `resolveStorePath` in the export command while preserving explicit `--store` precedence.
