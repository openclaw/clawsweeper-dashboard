## Label Justifications

- `P2`: This is a normal-priority CLI/session-store bug with a clear workaround but real impact for custom session-store users.
- `impact:session-state`: The command can miss stored session trajectory data when the session row lives in the configured session store.

## Summary

Keep open: current main and v2026.6.9 still use the old `sessions export-trajectory` store resolution, while the linked open PR is the active candidate fix.

## What This Changes

The issue asks for `openclaw sessions export-trajectory` to use the shared session-store resolver for both implicit and explicit store paths.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95570, or an equivalent narrow fix, so export reuses the existing configured-store resolver while preserving explicit `--store` precedence.

## Reproduction Assessment

Yes. Source inspection on current main shows export without `--store` uses the default agent store while sibling session commands resolve configured `session.store`.

## Solution Assessment

Yes. The best fix is to reuse the existing runtime config plus `resolveStorePath` contract in the export command instead of adding a parallel resolver.
