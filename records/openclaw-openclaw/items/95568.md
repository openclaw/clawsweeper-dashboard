## Summary

Keep open: current main and v2026.6.9 still resolve `sessions export-trajectory`'s implicit store from the default agent sessions directory, and https://github.com/openclaw/openclaw/pull/95570 is the open unmerged candidate fix.

## What This Changes

The issue asks `openclaw sessions export-trajectory` to honor configured `session.store`, including `{agentId}` templating, when `--store` is omitted.

## Best Possible Solution

The desired end state is a narrow CLI fix that reuses the existing configured-store resolver for implicit export paths while preserving explicit `--store` precedence and regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows export without `--store` uses the default session store while session listing uses configured `session.store`; the reporter also described a temporary configured-store repro.

## Solution Assessment

Yes. The narrow maintainable fix is to make the export command resolve its implicit store through the existing runtime config and `resolveStorePath` contract, with explicit `--store` continuing to override config.
