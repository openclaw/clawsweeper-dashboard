## Summary

Keep open: current main and the latest release still have the `sessions export-trajectory` store-resolution mismatch, and the open linked PR is the candidate fix that should land or be rejected before this issue closes.

## What This Changes

The issue asks for `openclaw sessions export-trajectory` to resolve implicit and explicit session stores through the same config-aware resolver used by the rest of the sessions CLI.

## Best Possible Solution

Review and land or explicitly reject https://github.com/openclaw/openclaw/pull/95570, then close this issue once the resolver alignment is merged.

## Reproduction Assessment

Yes. Source inspection on current main shows export bypasses configured `session.store` and shared path expansion while sibling sessions commands use the config-aware resolver; no live CLI command was run because this review is read-only.

## Solution Assessment

Yes. The narrowest maintainable solution is to reuse `getRuntimeConfig` and `resolveStorePath` in the export command while preserving explicit `--store` precedence, which is exactly the linked PR's approach.
