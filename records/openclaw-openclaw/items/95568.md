- `impact:session-state`: The command can look in the wrong persisted session store when trajectory export should use configured session state.

## Summary

Keep open: current main and the latest release still have the `sessions export-trajectory` store-resolution mismatch, and https://github.com/openclaw/openclaw/pull/95570 is the open candidate fix that should land or be explicitly rejected before this issue closes.

## What This Changes

Request that `sessions export-trajectory` resolve implicit and explicit session stores through the same config-aware resolver used by the rest of the sessions CLI surface.

## Best Possible Solution

Review and land or explicitly reject https://github.com/openclaw/openclaw/pull/95570, then close this issue once resolver alignment is merged.

## Reproduction Assessment

Yes. Source inspection on current main shows `sessions export-trajectory` bypasses configured `session.store` and the shared resolver, while sibling sessions commands route through the shared store target path.

## Solution Assessment

Yes. The narrow fix is to reuse `getRuntimeConfig` and `resolveStorePath` in the export command while preserving explicit `--store` precedence; broader auth or session architecture changes are unnecessary for this mismatch.
