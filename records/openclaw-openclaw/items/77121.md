## Summary

Keep open: current main still has no exec-level guard for unisolated heavy validation commands, and the issue is already paired with open draft PR #77140 via closing syntax, so it should remain open until that PR is reviewed, landed, or closed.

## What This Changes

The issue requests a Gateway exec resource policy that rejects or isolates known-heavy validation commands before they run in the live Gateway service context.

## Best Possible Solution

Review and either land or close #77140; the end state should be a maintainer-approved exec resource policy with tests for direct, chained, shell-wrapped, light, and already-isolated commands.

## Reproduction Assessment

No, not for the full production OOM incident. Source inspection does show the relevant current-main path: gateway exec can still pass an unisolated heavy command through to runExecProcess without a heavy-command preflight.

## Solution Assessment

Unclear whether the proposed blacklist, wrapper name, and OPENCLAW_EXEC_HEAVY_ENFORCEMENT escape hatch are the best long-term interface. A maintainer-approved exec/resource-policy seam is the narrow maintainable direction, and #77140 is the active implementation candidate.
