## Summary

Keep open: current main still lacks an exec-level heavy-command guard, and open draft PR #77140 is the active implementation candidate that explicitly closes this issue, so the issue should remain open until that PR or an equivalent patch lands.

## What This Changes

The issue requests a Gateway exec preflight policy that blocks or routes known-heavy validation commands before they run unisolated inside the live Gateway service resource domain.

## Best Possible Solution

Review and land PR #77140, or an equivalent maintainer-approved exec resource-policy patch, then close this issue after the guard is on main.

## Reproduction Assessment

No high-confidence live reproduction of the full systemd OOM incident was established here. Source inspection does show the relevant current-main acceptance path: unisolated Gateway exec commands reach the shell/supervisor path without a heavy-command preflight.

## Solution Assessment

Unclear whether the exact blacklist, wrapper name, and OPENCLAW_EXEC_HEAVY_ENFORCEMENT bypass are the final interface. The maintainable direction is an explicit exec resource-policy guard with maintainer-approved defaults and tests for direct, chained, shell-wrapped, light, and already-isolated commands.
