## Summary

Keep open: current main and v2026.6.9 still lack an exec-level guard or finite isolation requirement for known-heavy validation commands, and a fresh production incident on June 21, 2026 supports the same Gateway resource-domain failure mode.

## What This Changes

The issue requests a Gateway exec preflight or isolation policy that blocks known-heavy validation commands unless they run under an approved resource-limited wrapper or scope.

## Best Possible Solution

Keep this issue as the canonical maintainer/security decision point, then land a policy-approved exec guard or finite per-command isolation path with docs, focused tests, and live Linux Gateway proof for both refusal and approved allow cases.
