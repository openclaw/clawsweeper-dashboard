## Summary

Keep open: current main still has no exec-level heavy-command enforcement, and open draft PR #77140 is the active implementation candidate for this issue.

## What This Changes

Request an exec-tool resource policy that blocks or requires isolation for known-heavy validation commands launched on the local Gateway host.

## Best Possible Solution

Review PR #77140's exec-level heavy-command guard and either land an accepted policy with tests/docs/changelog or keep this issue as the policy tracker for a replacement design.

## Reproduction Assessment

No high-confidence live reproduction was established for the full systemd OOM or hard-hang incident. Source inspection does confirm current main accepts local Gateway exec commands without a heavy-command preflight before runExecProcess.

## Solution Assessment

Unclear whether the exact proposed blacklist, wrapper name, and OPENCLAW_EXEC_HEAVY_ENFORCEMENT bypass are the best long-term interface. The narrow maintainable direction is an exec/resource-policy seam with maintainer-approved defaults and bypass semantics.
