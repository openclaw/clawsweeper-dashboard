## Summary

Keep open: current main still has a clear source path for unisolated local Gateway exec to spawn heavyweight validation commands, and the linked draft PR is the active implementation candidate rather than a reason to close this issue before merge.

## What This Changes

Add an exec-level resource policy that blocks or isolates known-heavy validation commands before local Gateway execution.

## Best Possible Solution

Review the linked draft PR or an equivalent design, then ship an exec resource guard that rejects or reroutes known-heavy Gateway commands with maintainer-approved bypass semantics and regression tests.

## Reproduction Assessment

No high-confidence live reproduction was established for the full systemd OOM or host-hang incident. Source inspection does give a clear current-main path: local gateway exec in full/off mode can reach runExecProcess with the original heavyweight command and no heavy-command preflight.

## Solution Assessment

Unclear whether the exact proposed blacklist, wrapper allowance, and OPENCLAW_EXEC_HEAVY_ENFORCEMENT bypass are the best long-term interface. The maintainable direction is an explicit exec resource-policy seam with tests for direct, chained, shell-wrapped, light, and already-isolated commands.
