## Summary

Keep open: current main still has no Gateway exec heavy-command preflight, and the only exact implementation PR is closed unmerged. The adjacent OOM-score, local script-pressure, heavy-check lock, and broad-find work are useful but do not enforce an exec-entry resource policy for arbitrary agent-generated validation commands.

## What This Changes

The issue requests a Gateway exec resource policy that blocks or isolates known-heavy validation commands before launching them in the live Gateway process domain.

## Best Possible Solution

Define and document a maintained Gateway exec resource policy that rejects known-heavy unisolated validation commands or routes them through a verified finite resource-limited scope, with regression coverage and live Gateway proof.
