## Summary

Keep open: current main and the latest release still lack the requested heavy-command preflight for local Gateway exec, while adjacent OOM-score and local-check safeguards only cover related surfaces. ## What This Changes

This issue requests an exec-level policy that detects known-heavy validation commands before local Gateway host execution and requires resource isolation or an explicit bypass.

## Best Possible Solution

Keep this as the canonical maintainer/security decision for Gateway exec resource isolation, then land a focused guard or scoped-run design with docs and live Gateway proof for both refusal and approved allow paths.

## Reproduction Assessment

## Solution Assessment

Unclear as a final design. An exec-entry guard is a plausible narrow fix, but maintainers still need to choose the resource-isolation contract, wrapper/bypass semantics, and required live proof.
