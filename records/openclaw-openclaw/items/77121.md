## Summary

Keep open: current main still lacks an exec-level heavy-command guard or resource-scope requirement, and the merged OOM-score plus systemd OOMPolicy work only mitigates adjacent failure modes after a heavy child process has already started.

## What This Changes

This issue asks OpenClaw to reject or resource-isolate known-heavy local Gateway exec commands such as package-manager installs, broad tests, lint, typecheck, and build commands before they run in the live Gateway resource domain.

## Best Possible Solution

Land a maintainer-approved core execution boundary that resource-scopes known-heavy local Gateway exec commands or rejects them with documented bypass semantics and regression coverage.

## Reproduction Assessment

No high-confidence safe live repro was run; intentionally triggering OOM in the live Gateway resource domain can disrupt the host. Source inspection and the June 21 production incident show the current exec path still lacks the requested guard.

## Solution Assessment

Unclear: an exec-level preflight is a plausible mitigation, but maintainers need to choose between fail-closed heuristics, resource-scoped execution, and documented bypass/config behavior.
