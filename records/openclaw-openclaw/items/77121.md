## Summary

Keep open: current main still allows local `host=gateway` exec to run arbitrary heavy validation commands through the Gateway process supervisor, while the existing OOM and heavy-check mitigations are adjacent rather than exec-tool enforcement. The remaining fix changes resource/security policy, so it needs maintainer direction before automation.

## What This Changes

Add exec-tool enforcement that detects known-heavy validation commands before local Gateway host execution and refuses or routes them through resource isolation.

## Best Possible Solution

Adopt a maintainer-approved Gateway host-exec resource policy, then implement the guard or isolation wrapper at the exec/runtime boundary with focused regression tests and docs.

## Reproduction Assessment

Yes for the missing enforcement path: current source shows a `host=gateway` command such as `pnpm lint:core` can reach `runExecProcess` without a heavy-command classifier. I did not reproduce the actual OOM or hard-hang on this host.

## Solution Assessment

Unclear as proposed: an exec-level fail-closed guard is plausible, but maintainers should first choose between command rejection, resource-scoped execution, or both because the change alters documented full host-exec behavior.
