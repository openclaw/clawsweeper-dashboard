  - body: The PR queues every transcript file after `stat`, so bounded ranges like `/total_usage 5h` scan full history before record-level filtering. Keep the current `mtimeMs < sinceTime` prefilter for bounded ranges while allowing the all-time path to scan everything.

  - confidence: 0.9

  - confidence: 0.83

  - body: The quick-select buttons use raw `/total_usage ...` callback data. When text commands are disabled on a native Telegram command surface, those callbacks are treated as text instead of native commands and the handler returns null; use the native command callback path or a registry arg menu.
  - confidence: 0.82

  - body: Making `models` required changes the `CostUsageSummary` contract, but current-main typed fixtures and cache summary builders still return summaries without that field. Backfill every producer/fixture on rebase or make the field backward-compatible.
  - confidence: 0.78
