- **[P2] Populate cache-backed model summaries:** `src/infra/session-cost-usage.types.ts:68`
  - body: This makes `models` required on `CostUsageSummary`, but the durable cache summary builder on current main still returns no model aggregate list.   - confidence: 0.88
- **[P2] Use native callback payloads for quick selects:** `src/auto-reply/reply/commands-session.ts:425-434`
  - body: These buttons send raw `/total_usage ...` callback text. Telegram only preserves native command routing for `tgcmd:` payloads, so with `commands.text=false` the callback can bypass this handler and fail to return the selected usage summary.
  - confidence: 0.9
- **[P2] Preserve bounded transcript filtering:** `src/infra/session-cost-usage.ts:399`
  - body: Removing the `mtime` prefilter makes bounded requests like `/total_usage 5h` enumerate and scan every transcript before discarding out-of-range entries. Keep the fast prefilter for bounded ranges and reserve full-history scans for an explicit all-time request.
  - confidence: 0.84
- **[P2] Key model buckets by provider and model:** `src/infra/session-cost-usage.ts:426-431`
  - body: Aggregating by `entry.model` alone merges same-named model IDs from different providers and keeps whichever provider was seen first on the combined row, producing incorrect model totals and provider attribution.
  - confidence: 0.88
- **[P2] Keep discovery filtering before session limits:** `src/infra/session-cost-usage.ts:497`
  - body: Removing the `startMs` filter from discovered sessions lets out-of-range sessions survive until `sessions.usage` sorts and slices by `limit`, so older sessions can push in-range sessions out of the response.
  - confidence: 0.8
