## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Populate model totals in cache summaries:** `src/infra/session-cost-usage.types.ts:68`
  - body: The PR makes `CostUsageSummary.models` required, but current main's cache-backed summary path still returns no model aggregates. Rebase this onto the durable cache path and populate provider-aware model totals there, or cache-served summaries will violate the new contract.
  - confidence: 0.9
- **[P2] Keep bounded usage scans efficient:** `src/infra/session-cost-usage.ts:397-402`
  - body: The direct summary path now enumerates every transcript file and filters records only after scanning. Narrow ranges like `/total_usage 5h` should use the cache-backed summary path or keep a safe bounded prefilter to avoid full-history scans.
  - confidence: 0.88
- **[P2] Filter discovered sessions before limiting results:** `src/infra/session-cost-usage.ts:494-500`
  - body: Removing the `startMs` discovery filter lets out-of-range sessions reach downstream sorting and limits before per-session filtering. Large histories can then hide in-range sessions from the returned usage totals.
  - confidence: 0.86
- **[P2] Partition model buckets by provider:** `src/infra/session-cost-usage.ts:426-431`
  - body: The model aggregate key is only `entry.model`, while the bucket stores one provider. Same-named models from different providers can be merged into one mislabeled row, so key by provider and model.
  - confidence: 0.91
- **[P2] Use structured Telegram callback payloads:** `src/auto-reply/reply/commands-session.ts:425-434`
  - body: The quick-select buttons send raw `/total_usage ...` callback text. Telegram native command callbacks use the structured `tgcmd:` path, so these buttons can lose native routing when text commands are disabled.
  - confidence: 0.86
