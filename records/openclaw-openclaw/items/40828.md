## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Update the cache-backed summary producer:** `src/infra/session-cost-usage.types.ts:62`
  - body: The PR makes `models` required on `CostUsageSummary`, but current main also returns summaries from `buildCostUsageSummaryFromCache()` via `usage.cost`. That producer still returns no `models`, so the rebased patch breaks the current cache-backed summary contract; add model aggregates to the cache path or keep the field explicitly backward-compatible.
  - confidence: 0.9
- **[P2] Restore bounded transcript prefiltering:** `src/infra/session-cost-usage.ts:402`
  - body: This removes the `mtimeMs < sinceTime` skip before scanning aggregate usage files. Bounded ranges such as `/total_usage 5h` will scan the full transcript history and discard entries only after reading them, which regresses current-main behavior on long-lived installations.
  - confidence: 0.88
- **[P2] Partition model totals by provider:** `src/infra/session-cost-usage.ts:426-440`
  - body: The new model bucket key is only `entry.model`, while the bucket keeps a single provider from the first matching entry. Same-named models from different providers will be merged and mislabeled in `summary.models` and `/total_usage by-model`; key by provider plus model or preserve provider separation another way.
  - confidence: 0.9
- **[P2] Reapply startMs before limiting discovered sessions:** `src/infra/session-cost-usage.ts:499`
  - body: The PR removes the discovery-time `startMs` filter. `sessions.usage` still sorts discovered sessions and applies `slice(0, limit)` before per-session usage filtering, so out-of-range sessions can consume the limit and push in-range sessions out of the result.
  - confidence: 0.85
- **[P2] Use structured Telegram command callbacks:** `src/auto-reply/reply/commands-session.ts:424-434`
  - body: The quick-select buttons use raw `/total_usage ...` callback text. Telegram native command callbacks preserve native routing only for the `tgcmd:` payload path, so these buttons can fall through as ordinary callback text when text commands are disabled; build the callback data through the native command callback helper or registry-backed argument menu.
  - confidence: 0.84
- **[P2] Mock the cache-backed usage loader:** `src/gateway/server-methods/usage.test.ts:10`
  - body: This test mock changes the stubbed export to `loadCostUsageSummary`, but `src/gateway/server-methods/usage.ts` still calls `loadCostUsageSummaryFromCache()`. The cache helper test will no longer stub the function under test, and `vi.mocked(loadCostUsageSummaryFromCache).mock.calls` is not backed by a mock.
  - confidence: 0.82
