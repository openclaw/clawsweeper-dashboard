## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Restore bounded transcript prefiltering:** `src/infra/session-cost-usage.ts:402`
  - body: The PR queues every transcript file after `stat`, so bounded ranges like `/total_usage 5h` scan full history before record-level filtering. Keep the current `mtimeMs < sinceTime` prefilter for bounded ranges while allowing the all-time path to scan everything.
  - confidence: 0.87
- **[P2] Partition model totals by provider and model:** `src/infra/session-cost-usage.ts:426-440`
  - body: The aggregate key is only `entry.model`, but the bucket stores one provider value from the first entry. Same-named models from different providers will be merged and mislabeled in `summary.models` and `/total_usage by-model`.
  - confidence: 0.9
- **[P2] Keep discovery filtering bounded by requested start:** `src/infra/session-cost-usage.ts:500`
  - body: `discoverAllSessions` no longer applies `params.startMs` before session discovery. Downstream session usage lists sort and limit discovered sessions before per-session range filtering, so old sessions can push in-range sessions out of the response.
  - confidence: 0.83
- **[P2] Preserve native routing for Telegram quick buttons:** `src/auto-reply/reply/commands-session.ts:425-434`
  - body: The quick-select buttons use raw `/total_usage ...` callback data. Native Telegram command callbacks are prefixed so callback handling can preserve `commandSource: native`; without that, these buttons fall back to text-command routing and can fail when text commands are disabled.
  - confidence: 0.82
- **[P2] Update all CostUsageSummary producers and fixtures:** `src/infra/session-cost-usage.types.ts:68`
  - body: Making `models` required changes the `CostUsageSummary` contract, but current-main cache summary builders and typed test fixtures still return summaries without that field. Backfill every producer/fixture on rebase or make the field backward-compatible.
  - confidence: 0.78
