## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Restore bounded transcript prefiltering:** `src/infra/session-cost-usage.ts:402`
  - body: The PR queues every transcript file for aggregate usage scans and only filters records after reading them. Bounded ranges such as `/total_usage 5h` can regress to scanning full history; preserve the current prefilter for non-all-time ranges or use the cache-backed path while keeping all-time unbounded.
  - confidence: 0.88
- **[P2] Partition model totals by provider:** `src/infra/session-cost-usage.ts:426-440`
  - body: The aggregate key is only `entry.model`, while the bucket stores one provider from the first matching entry. Same-named models from different providers will be merged and mislabeled in `summary.models` and `/total_usage by-model`; key by provider and model or otherwise preserve provider separation.
  - confidence: 0.9
- **[P2] Reapply startMs during session discovery:** `src/infra/session-cost-usage.ts:500`
  - body: The PR removes the discovery-time `startMs` filter. `sessions.usage` sorts and limits discovered sessions before per-session usage filtering, so old out-of-range sessions can consume the limit and push in-range sessions out of the response.
  - confidence: 0.84
- **[P2] Use structured Telegram command callbacks:** `src/auto-reply/reply/commands-session.ts:425-434`
  - body: The quick-select buttons use raw `/total_usage ...` callback text. Telegram native command callbacks preserve native routing only for structured `tgcmd:` payloads, so these buttons can fall through as ordinary callback text when text commands are disabled; use the native command callback helper or a registry-backed argument menu.
  - confidence: 0.83
- **[P2] Update every CostUsageSummary producer:** `src/infra/session-cost-usage.types.ts:68`
  - body: The PR makes `models` required on `CostUsageSummary`, but current main has cache-backed summary producers that do not return model aggregates. Backfill the cache path and tests on rebase, or make the field explicitly backward-compatible.
  - confidence: 0.78
