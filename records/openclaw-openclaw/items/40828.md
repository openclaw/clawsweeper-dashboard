- **[P2] Update cache-backed usage summaries:** `src/infra/session-cost-usage.types.ts:68`
  - body: The PR makes `CostUsageSummary.models` required, but current main has cache-backed summary producers that still return only daily totals, aggregate totals, and cache status.   - confidence: 0.86
- **[P2] Restore bounded transcript prefiltering:** `src/infra/session-cost-usage.ts:402`
  - body: `loadCostUsageSummary` now queues every transcript file and only filters records after reading them. Bounded ranges such as `/total_usage 5h` can regress to full-history scans; preserve the current prefilter for non-all-time ranges or use the cache-backed path.
  - confidence: 0.88
- **[P2] Partition model totals by provider:** `src/infra/session-cost-usage.ts:426-431`
  - body: The aggregate key is only `entry.model`, while the bucket stores a single provider from the first matching entry. Same-named models from different providers will be merged and mislabeled in `summary.models` and `/total_usage by-model`; key by provider and model.
  - confidence: 0.9
- **[P2] Reapply startMs during session discovery:** `src/infra/session-cost-usage.ts:500`
  - body: The PR removes discovery-time `startMs` filtering. `sessions.usage` sorts and limits discovered sessions before per-session usage filtering, so old out-of-range sessions can consume the limit and push in-range sessions out of the response.
  - confidence: 0.84
- **[P2] Use structured Telegram command callbacks:** `src/auto-reply/reply/commands-session.ts:425-434`
  - body: The quick-select buttons use raw `/total_usage ...` callback text. Telegram native command callbacks preserve native routing through `tgcmd:` payloads, so these buttons can fall through as ordinary callback text when text commands are disabled.
  - confidence: 0.85
