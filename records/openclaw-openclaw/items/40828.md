## Best Possible Solution

Rebase or replace the implementation so it preserves current usage-cache and range contracts, uses provider-aware model buckets, routes Telegram quick buttons through structured native command callbacks, updates every summary producer/test/doc, and includes redacted real behavior proof.

Yes. Source inspection of PR head `9979f2de59cecfea938c493ec7536b4bf5ed1127` against current main shows the removed range prefilters, provider-insensitive model key, raw Telegram callback command data, and summary-contract drift.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Restore bounded transcript prefiltering:** `src/infra/session-cost-usage.ts:399`
  - body: The PR queues every transcript file after `stat`, so bounded ranges like `/total_usage 5h` scan full history before record-level filtering. Preserve the current bounded prefilter for non-all-time ranges while still allowing the all-time path to scan everything.
  - confidence: 0.87
- **[P2] Partition model totals by provider and model:** `src/infra/session-cost-usage.ts:423-438`
  - body: The aggregate key is only `entry.model`, but the bucket stores one provider value from the first entry. Same-named models from different providers will be merged and mislabeled in `summary.models` and `/total_usage by-model`.
  - confidence: 0.9
- **[P2] Keep discovery filtering bounded by requested start:** `src/infra/session-cost-usage.ts:497-502`
  - body: `discoverAllSessions` no longer applies `params.startMs` before session discovery. Downstream session usage lists sort and limit discovered sessions before per-session range filtering, so old sessions can push in-range sessions out of the response.
  - confidence: 0.83
- **[P2] Preserve native routing for Telegram quick buttons:** `src/auto-reply/reply/commands-session.ts:419-432`
  - body: The quick-select buttons use raw `/total_usage ...` callback data. When text commands are disabled on a native Telegram command surface, those callbacks can fall through as text instead of structured native commands; use the native command callback path or a registry arg menu.
  - confidence: 0.82
- **[P2] Update all CostUsageSummary producers and fixtures:** `src/infra/session-cost-usage.types.ts:63`
  - body: Making `models` required changes the `CostUsageSummary` contract, but current-main cache summary builders and typed fixtures still return summaries without that field. Backfill every producer and fixture on rebase, or make the field backward-compatible.
  - confidence: 0.78
