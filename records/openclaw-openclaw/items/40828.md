Full review comments:

- **[P2] Partition model totals by provider and model:** `src/infra/session-cost-usage.ts:426-440`
  - body: The new aggregate key is only `entry.model`, while each bucket stores one provider from the first matching entry. Same-named models from different providers will merge tokens and costs and be labeled with whichever provider was seen first; use a provider-plus-model key like the existing session usage aggregate.
  - confidence: 0.9
- **[P2] Restore bounded transcript prefiltering:** `src/infra/session-cost-usage.ts:398-402`
  - body: After `stat`, every transcript file is returned for scanning, so bounded queries such as `/total_usage 5h` enumerate and scan full history before record-level filtering drops old entries. Long-lived installs can turn a small range command into a full-history I/O scan.
  - confidence: 0.84
- **[P2] Keep discovery filtering bounded by requested start:** `src/infra/session-cost-usage.ts:494-500`
  - body: `discoverAllSessions()` no longer honors `params.startMs`, but `sessions.usage` still sorts and slices discovered sessions before loading per-session summaries. Out-of-range transcript files can consume the limit and push in-range sessions out of the response or totals.
  - confidence: 0.78
- **[P2] Update CostUsageSummary fixtures:** `src/infra/session-cost-usage.types.ts:68`
  - body: Adding `models` as a required `CostUsageSummary` field changes existing typed fixtures. `commands-session-usage.test.ts` still returns summary objects without `models`, so the test-type lane needs those fixtures updated or the field made backward-compatible.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `docs/tools/slash-commands.md:106`
  - body: This is a user-facing feature that adds a new command and documentation, but the PR file list does not include `CHANGELOG.md`. Add an Unreleased Changes entry crediting @cclank.
  - confidence: 0.86
