## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Keep user delimiter blocks out of hidden runtime context:** `src/agents/embedded-agent-runner/run/runtime-context-prompt.ts:60`
  - body: When this helper appends generated runtime self-context for a prompt-only run, it sets `transcriptPrompt` to the full original prompt. `resolveRuntimeContextPromptParts` then extracts every protected delimiter block from the combined prompt, so a user-authored `<<<BEGIN_OPENCLAW_INTERNAL_CONTEXT>>>` block in the original prompt is copied into the hidden `openclaw.runtime-context` custom message under the runtime-generated notice. Keep extraction scoped to the generated block, or escape/strip user-authored delimiters before synthesizing the transcript split.
  - confidence: 0.87
- **[P1] Sync policy groups before adding the runtime tool:** `src/agents/tool-catalog.ts:104-109`
  - body: This adds `runtime` to the core `group:runtime`/OpenClaw tool surface, but the policy plugin still expands `group:runtime` to only `exec`, `process`, and `code_execution`, and the security docs say the same. Policy conformance checks that require runtime tools to be denied can therefore pass while the new `runtime` metadata tool is not covered by the policy plugin's group definition. Update the policy group map and its tests/docs, or keep `runtime` out of that group until the policy contract is approved.
  - confidence: 0.82
