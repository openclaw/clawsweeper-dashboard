Full review comments:

- **[P1] Keep user delimiters out of hidden runtime context:** `src/agents/embedded-agent-runner/run/runtime-context-prompt.ts:60`
  - body: When this helper appends generated runtime self-context for a prompt-only run, it sets `transcriptPrompt` to the original prompt. Since `resolveRuntimeContextPromptParts` extracts all protected delimiter blocks whenever a transcript prompt exists, a user-authored `<<<BEGIN_OPENCLAW_INTERNAL_CONTEXT>>>` block in that prompt is copied into the hidden runtime-context custom message. Escape or strip user-authored delimiters before synthesizing the split, or extract only the generated block.
  - confidence: 0.87
- **[P1] Sync policy groups before adding runtime:** `src/agents/tool-catalog.ts:104-109`
  - body: This registers `runtime` in the core `group:runtime`/OpenClaw surfaces, but the policy plugin and security/tool-policy docs still define `group:runtime` as only `exec`, `process`, and `code_execution`. Policy conformance checks that require runtime tools to be denied can therefore pass while the new model-facing metadata tool is not covered. Update the policy group map and tests/docs, or keep `runtime` out of that group until the policy contract is approved.
  - confidence: 0.82
