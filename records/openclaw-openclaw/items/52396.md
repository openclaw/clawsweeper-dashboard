## Repair Work Prompt

Fix openclaw/openclaw#52396 with a narrow Discord plugin change. Current main retries direct Discord outbound requests only for Carbon `RateLimitError` / HTTP 429, while `withDiscordDeliveryRetry` only covers status-coded 429 and >=500 errors. Add a Discord-local transient classifier for statusless transport failures such as timeout, DNS/connect/reset/closed/unavailable/temporarily unavailable and `fetch failed`; preserve `RateLimitError.retry_after` handling; account for the existing status-coded delivery wrapper so 5xx responses do not get excessive nested retries. Use public plugin SDK exports only, do not move Discord-specific policy into core unless a tiny generic seam is already present, do not change Carbon dependency pins, and do not implement durable outbox work here. Add focused tests for direct send/runner behavior and outbound adapter behavior, update `docs/concepts/retry.md`, and add a user-facing `CHANGELOG.md` fix entry crediting `Thanks @sunshineo` (and other non-bot source authors if their code/context is used).
