## Summary

Keep open: current main has session/run abort primitives and some channel edit/delete event observation, but no generic authorized recall/edit event contract or per-channel wiring that cancels active channel runs.

## What This Changes

Request support for messaging-platform recall/delete/edit events to stop or update the active agent task instead of continuing from the original message.

## Best Possible Solution

Add a typed, authorized channel recall/delete/edit event capability, then wire capable channel plugins to existing `sessions.abort`/fast-abort or steering behavior with focused per-channel tests.

## Reproduction Assessment

Not applicable. This is a new cross-channel feature request; current-main source inspection shows abort primitives and event observation, but no recall/edit event path that cancels an active run.
