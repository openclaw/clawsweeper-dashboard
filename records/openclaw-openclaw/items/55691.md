## Summary

Keep open: current main still has abort and queue-interrupt primitives, plus some channel-specific edit/delete observations, but it still lacks the generic source-message recall/delete/edit contract needed to cancel or reprocess the active run that came from a recalled or edited message. The related Discord and Feishu items are narrower open slices, while the closed Telegram/Feishu reports explicitly point back to this broader tracker.

## What This Changes

The issue requests cross-channel handling for message recall/delete/edit events so active agent work can be canceled or reprocessed from the corrected source message.

## Best Possible Solution

Define an authorized generic source-message lifecycle contract, then implement capable channel mappings for recall/delete cancellation and edit replacement or re-dispatch with focused core and per-channel tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a new cross-channel feature/API request rather than a broken documented contract. Source inspection does confirm the requested recall/edit/delete-to-abort semantics are missing on current main.

## Solution Assessment

Unclear as a concrete automatic patch. The direction is useful, but the maintainable fix should settle source-message identity, authorization, compatibility defaults, and per-channel capability semantics before adding isolated platform hooks.
