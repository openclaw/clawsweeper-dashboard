## Summary

Keep open. Current main has abort primitives and some channel edit/delete awareness, but it still lacks the requested cross-channel source-message recall/edit contract; the active Discord PR covers only one platform slice and is still open.

## What This Changes

The issue requests message recall/delete events to cancel the run started by that source message, with message edit handling as corrected-context or cancel-and-rerun behavior.

## Best Possible Solution

Confirm the generic source-message lifecycle contract, then land channel-owned implementations starting with the active Discord PR and separate per-platform follow-ups for Telegram, Signal, Feishu, WhatsApp, and similar channels.

## Reproduction Assessment

Yes. Source inspection on current main shows Discord lacks message update/delete listeners, Telegram edits only update reply-chain metadata, and the generic channel ingress contract has no recalled/deleted source-message event kind.

## Solution Assessment

Yes as a broad tracker, not as an automated one-shot fix. The maintainable path is a maintainer-confirmed source-message lifecycle contract plus channel-owned implementations, rather than closing this while only a Discord slice has an open PR.
