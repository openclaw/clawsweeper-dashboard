## Summary

Keep open. Current main still has run/session abort primitives and source message ids, but it does not have a shared source-message edit/delete/recall lifecycle contract or a native-message-id to active-run cancellation map; related work is channel-specific, open, closed-unmerged, or adjacent.

## What This Changes

Request support for channel-native message recall/delete/edit events so recalled source messages can cancel active tasks and edited messages can update or supersede pending context.

## Best Possible Solution

Define a minimal maintainer-approved source-message lifecycle contract, then implement channel-specific edit/delete/recall adapters and tests while keeping outbound message actions separate.

## Reproduction Assessment

Yes, at source level. Current main exposes source message ids in inbound context and supports run/session abort, but the inspected contracts do not expose source-message edit/delete/recall lifecycle events or a native message-id to active-run cancellation map.

## Solution Assessment

Unclear as a final implementation path. The maintainable direction is a shared lifecycle contract where needed plus channel-specific handlers and tests; the open Discord PR is a useful slice but not a merged replacement for the cross-channel issue.
