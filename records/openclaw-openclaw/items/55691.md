## Summary

Keep open: current main has session/run abort primitives and some channel-specific edit observation, but it still lacks a generic channel recall/delete/edit event contract that can cancel or reprocess active work. The open Discord reports are narrower platform slices, and the closed Feishu-oriented report points back here as the canonical cross-channel request.

## What This Changes

The issue requests cross-channel support for message recall/delete/edit events so recalled messages cancel active tasks and edited messages can update or rerun the request.

## Best Possible Solution

Define an authorized generic channel-event contract for source-message recall/delete/edit, then let capable channel plugins map those events to precise session/run cancellation or edit reprocessing with focused per-channel tests.

## Reproduction Assessment

Not applicable. This is a new cross-channel feature request, and current source inspection shows abort primitives but no existing recall/edit-to-abort contract to reproduce as broken behavior.

## Solution Assessment

Unclear as an implementation plan. The requested direction is useful, but the maintainable solution should first define source-message mapping, authorization, and per-channel capability semantics instead of adding isolated platform hooks.
