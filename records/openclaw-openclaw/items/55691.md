## Summary

Keep open: current main has abort and queue-interrupt primitives, but it still lacks the generic source-message recall/delete/edit contract, source-message-to-run mapping, and authorization policy needed to cancel or reprocess work from the recalled or edited message.

## What This Changes

The issue requests cross-channel handling for recalled, deleted, or edited source messages so active agent tasks can be cancelled or reprocessed.

## Best Possible Solution

Define an authorized generic source-message lifecycle contract, then implement capable channel mappings for recall/delete cancellation and edit replacement or re-dispatch with focused core and per-channel tests.
