## Summary

Keep open: current main has explicit abort primitives and some edit/delete event observation, but it still does not connect channel recall/edit events to cancelling or replacing the active session run.

## What This Changes

Support channel message recall/delete and edit events so capable messaging plugins can cancel, update, or rerun active agent work.

## Best Possible Solution

Define a small authorized channel-event contract for message edit/delete/recall, then implement per-channel mappings that can cancel or steer active runs with focused tests.

## Reproduction Assessment

Not applicable: this is a new cross-channel behavior request, not a broken documented behavior. Source inspection shows the existing `/stop` and `sessions.abort` paths plus edit/delete observation, but no recall/edit-to-abort contract.

## Solution Assessment

Unclear. The requested behavior is useful, but the maintainable solution likely needs a generic plugin/channel event seam plus per-channel authorization and capability rules rather than one-off channel branches.
