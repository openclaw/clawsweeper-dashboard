## Summary

Keep open: current main still has abort and queue-interrupt primitives, but it does not have a generic source-message recall/delete/edit event contract, source-message-to-run mapping, or authorization policy for cancelling or reprocessing work from the original platform message.

## What This Changes

Request support for channel message recall/delete/edit events so recalled messages cancel active agent work and edited messages can update or re-dispatch the turn.

## Best Possible Solution

Define an authorized generic source-message lifecycle contract in the channel/plugin SDK, map platform message ids to active run/session ownership, then add per-channel recall/delete cancellation and edit re-dispatch semantics with compatibility defaults.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a new cross-channel feature request. Source inspection gives high confidence that current main lacks the requested recall/edit/delete-to-abort semantics.

## Solution Assessment

Unclear as a ready implementation path. The direction is useful, but the maintainable fix needs a generic SDK/core event contract, source-message identity mapping, authorization rules, compatibility defaults, and channel-specific policy choices.
