## Summary

Keep open. Current main has session/run abort primitives and carries source message ids in inbound context, but it still lacks a shared source-message edit/delete/recall lifecycle event or native-message-id to active-run mapping; related items are channel-specific slices or adjacent outbound message-management work, not a merged replacement.

## What This Changes

Request support for messaging-platform message recall/delete/edit events so OpenClaw can cancel, update, or reprocess active agent work triggered by the original message.

## Best Possible Solution

Define a minimal source-message lifecycle contract between channel plugins and core/SDK, then implement channel-specific edit/delete/recall adapters with focused tests while keeping outbound message actions separate.

## Reproduction Assessment

Yes, source-level. Current main has source message ids in inbound context and active run cancellation by run/session, but the inspected contracts do not expose source-message edit/delete/recall lifecycle events or a native message-id to active-run cancellation map.

## Solution Assessment

No current implementation fully solves the issue. The maintainable path is a shared lifecycle contract where needed plus channel-specific handlers and tests, rather than closing the umbrella because one channel has partial or adjacent behavior.
