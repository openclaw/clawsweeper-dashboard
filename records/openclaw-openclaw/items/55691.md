## Summary

Keep open: current main has working session abort and queue interruption primitives, plus a few channel-specific observations of edits/deletes, but it still does not define a generic source-message recall/delete/edit event contract that can cancel or reprocess the active run that came from that source message.

## What This Changes

The issue requests cross-channel support for source message recall/delete/edit events so active agent work can be canceled or reprocessed when the user corrects or withdraws a message.

## Best Possible Solution

Define an authorized generic source-message event contract for recall/delete/edit, then let capable channel plugins map native events to active-run cancellation, edit replacement, or re-dispatch with focused per-channel tests.

## Reproduction Assessment

Not applicable. This is a new cross-channel feature request, and current source inspection shows missing recall/edit/delete-to-abort semantics rather than a broken existing documented contract.

## Solution Assessment

Unclear as a concrete implementation plan. The product direction is useful, but the maintainable fix should define source-message identity, authorization, and capability semantics before adding isolated platform hooks.
