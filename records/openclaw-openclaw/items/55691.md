## Summary

Keep open. Current main has run/session abort primitives and some channel-specific edit handling, but it does not define or wire a cross-channel source-message recall/edit/delete lifecycle event to abort or reprocess active runs; the related items are channel-specific slices or adjacent outbound message-management work.

## What This Changes

The issue requests cross-channel support for source message recall/delete/edit events so an in-flight agent task can be canceled or corrected.

## Best Possible Solution

Keep this as the broad design tracker, define the minimal source-message lifecycle seam where plugin-local handlers cannot safely map native edit/delete/recall events to exact run cancellation or reprocessing, then implement channel adapters with focused tests.

## Reproduction Assessment

Yes, source-level. Current main has source message ids in inbound context and active run cancellation by run/session, but the inspected contracts do not expose source-message edit/delete/recall events or a native message-id to active-run cancellation map.

## Solution Assessment

No current implementation fully solves the issue. The maintainable solution is a shared lifecycle contract where needed plus channel-specific handlers and tests, not closing the umbrella because one channel has partial or adjacent behavior.
