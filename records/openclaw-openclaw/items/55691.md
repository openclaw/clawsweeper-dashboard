## Summary

Keep open: this is a real cross-channel feature request, and current main has run-abort primitives plus some edit/delete event observation, but it does not yet connect message recall/edit events to cancelling the active session run.

## What This Changes

Support platform message recall/delete events, and optionally edit events, as a way to stop or update active channel-based agent work.

## Best Possible Solution

Add an authorized generic channel-event contract for recall/delete/edit events, then let capable channel plugins map those events to session/run cancellation or steering with focused per-channel coverage.

## Reproduction Assessment

Not applicable. This is a new cross-channel behavior request, and source inspection shows existing abort paths plus event observation but no current recall/edit-to-abort contract.

## Solution Assessment

Unclear. The requested behavior is useful, but the maintainable path likely needs a small generic channel-event API plus per-channel authorization and capability rules before implementation.
