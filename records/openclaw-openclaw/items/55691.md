## Summary

Keep open: current main has active-run abort primitives and some channel edit/delete observation, but it still does not define or wire a recall/edit event contract that cancels the active session run. The prior ClawSweeper comment reached the same conclusion, and the current source still matches that review.

## What This Changes

The issue requests channel support for message recall/delete and edit events so recall cancels the active LLM task and edits update or restart the turn.

## Best Possible Solution

Add an authorized generic channel event contract for recall/delete/edit events, then let capable channel plugins map those events to session cancellation or steering with focused per-channel coverage.

## Reproduction Assessment

Not applicable. This is a new cross-channel behavior request, and source inspection shows abort paths plus some edit/delete event observation but no current recall/edit-to-abort contract.

## Solution Assessment

Unclear. The requested behavior is useful, but the maintainable solution likely needs a small generic channel-event API plus per-channel authorization and capability rules before implementation.
