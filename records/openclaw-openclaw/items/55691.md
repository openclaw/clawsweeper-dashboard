- `impact:session-state`: The requested behavior changes how an edited or deleted source message owns, supersedes, or cancels an active agent/session turn.
- `impact:message-loss`: Without recall/edit handling, corrected or withdrawn user intent can be ignored while the original message continues through the channel pipeline.

## Summary

Keep open. Current main and the latest release still lack a cross-channel source-message recall/edit-to-cancel contract; the active Discord work is narrower and still open, while Feishu inbound recall remains separate from the open Feishu message-management PR.

## What This Changes

Request support for channel message recall/delete and edit events so recalled messages cancel in-flight agent work and edits can update or rerun context.

## Best Possible Solution

Keep this as the canonical cross-channel design tracker, let the active Discord PR own Discord-specific implementation, and add a narrow plugin SDK/channelRuntime seam only if plugin-local handlers cannot safely map platform recall/edit events to exact source-message cancellation.

## Reproduction Assessment

Yes for the source-level feature gap: current main lacks shared edited/deleted/recalled ingress event kinds, Discord is create-only, and Feishu has no recall event registration. This is not a regression against a documented existing contract.

## Solution Assessment

Unclear as a final design. Per-channel plugin work is the likely first step, but maintainers need to decide whether a generic SDK/channelRuntime source-message cancellation contract is required before broad rollout.
