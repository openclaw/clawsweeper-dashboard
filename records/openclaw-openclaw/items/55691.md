## Summary

Keep open. Current main and the latest release have run/session abort primitives and carry source message ids into inbound turn context, but they still do not define a shared source-message edit/delete/recall lifecycle event or native-message-id to active-run cancellation map; the related items are channel-specific slices, closed-unmerged attempts, or adjacent outbound message-management work rather than a merged replacement.

## What This Changes

Request support for channel-native message recall/delete and edit events so OpenClaw can cancel or update running agent work instead of processing stale user input.

## Best Possible Solution

Define a minimal maintainer-approved source-message lifecycle contract, then implement channel-specific edit/delete/recall adapters and tests while keeping outbound message actions separate.

## Reproduction Assessment

Yes, at source level. Current main exposes source message ids in inbound context and supports run/session abort, but the inspected contracts do not expose source-message edit/delete/recall lifecycle events or a native message-id to active-run cancellation map.

## Solution Assessment

No current implementation fully solves it. The maintainable path is a shared lifecycle contract where needed plus channel-specific handlers and tests; the open Discord PR is a useful slice but not a merged replacement for the cross-channel issue.
