The branch adds `cron.webhookAllowPrivateNetwork` config/docs/schema plumbing, forwards `{ allowPrivateNetwork: true }` to cron webhook POST paths, adds gateway cron tests, and includes unrelated extension test edits.

## Best Possible Solution

Maintainers should choose the cron webhook network policy explicitly: land a tightly documented opt-in with final naming, warnings, and proof, or supersede it with the stricter reject-at-save direction in https://github.com/openclaw/openclaw/pull/49120.

Unclear. The branch uses the existing `SsrFPolicy` seam narrowly, but choosing an operator opt-in over rejecting private webhook URLs before persistence is a maintainer/security policy decision.
