## Summary

Keep open. Current main still contains the Telegram group text-command path that can drop `/status` with no reply while allowing normal group text, and the linked closing PR is still the implementation candidate rather than a merged fix.

## What This Changes

Fix Telegram group slash-command handling so `/status` replies in configured group sessions where normal group messages already route successfully.

## Best Possible Solution

Review and land PR #74894, or replace it with an equivalent focused Telegram change, so configured group and topic sessions deliver `/status` replies while command authorization remains strict for senders outside the allowed group policy.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a configured Telegram group where normal group policy access passes but the effective command allowlist has no entries; normal text reaches the agent, while `/status` hits the text-command gate and returns `null` before any reply is sent.

## Solution Assessment

Yes. The narrow maintainable fix belongs in Telegram's pre-dispatch text-command authorization path; the shared command gate should stay fail-closed as a generic primitive.
