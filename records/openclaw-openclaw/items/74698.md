## Summary

Keep open. Current main still has the Telegram group text-command gate that can silently drop `/status` in configured open groups without an effective sender command allowlist, and PR #74894 is already the open, unmerged implementation candidate with closing syntax.

## What This Changes

The issue reports that Telegram group `/status` commands no longer send a reply while ordinary Telegram group messages still route to the agent.

## Best Possible Solution

Review and land PR #74894, or replace it with an equivalent focused Telegram change, so configured group and topic sessions can deliver `/status` while command authorization still blocks unauthorized senders.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a configured Telegram group whose normal group access passes via per-group `groupPolicy: "open"` but whose `effectiveGroupAllow` has no entries; `/status` trips the control-command gate and returns `null` before any reply.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Telegram pre-dispatch text-command authorization path, because the shared gate is a generic fail-closed primitive and the bug is Telegram-specific treatment of configured open group sessions without a separate sender command allowlist.
