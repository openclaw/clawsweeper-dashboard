## Summary

Keep open. Current main still contains the Telegram group text-command gate that can classify a whole-message group `/status` as unauthorized when the effective group command allowlist is empty, and the linked implementation PR #74894 is still the active unmerged fix path rather than a reason to close the issue early.

## What This Changes

The issue requests restoring Telegram group `/status` replies when normal Telegram group messages still route successfully.

## Best Possible Solution

Review and land PR #74894, or replace it with an equivalent focused Telegram fix, so configured group `/status` commands reply in the originating group or topic while sender allowlists, owner-only commands, and disabled group/topic policies continue to fail closed.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a Telegram group that passes normal group access through explicit group configuration or open policy but has no effective group command allowlist entries; `/status` then hits the current control-command gate and is dropped before any visible reply.

## Solution Assessment

Yes. The narrowest maintainable solution is in the Telegram inbound text-command gate, not in core command semantics, because the failure happens before the normal command handler can decide authorization and produce a reply.
