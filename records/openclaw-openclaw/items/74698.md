## Summary

Keep open. Current main still has the source/test-backed split where per-group open policy can route ordinary Telegram group text, while the text-command gate can drop `/status` with no reply; PR #74894 is open with closing syntax and should be reviewed, landed, or superseded before this issue closes.

## What This Changes

Telegram group `/status` should produce a status reply in configured group sessions where normal Telegram group messages already route successfully.

## Best Possible Solution

Review and land PR #74894 or an equivalent focused Telegram text-command authorization fix so `/status` follows the same open group policy as ordinary group messages.

## Reproduction Assessment

Yes. Source inspection on current main shows a per-group `groupPolicy: "open"` setup lets normal group text reach the reply path, while `/status` hits `resolveControlCommandGate` with no configured sender authorizer and returns `null` before delivery; existing tests encode both halves.

## Solution Assessment

Yes at the issue level. The narrow maintainable path is a Telegram text-command authorization change that aligns open group policy with normal message routing while preserving disabled and allowlist boundaries.
