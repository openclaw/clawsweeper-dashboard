## Summary

Keep open. Current main still has the reported Telegram group split: ordinary text can route in a per-group `groupPolicy: "open"` group, while `/status` is treated as an unauthorized control command and dropped before any reply is delivered; the linked open PR is an implementation candidate, not a reason to close before merge.

## What This Changes

The issue requests restoring Telegram group slash command replies, especially `/status`, when normal group messages already route successfully.

## Best Possible Solution

Review and land PR #74894 or an equivalent focused Telegram text-command authorization fix that makes open group sender policy apply consistently to control commands while keeping restricted group modes fail-closed.

## Reproduction Assessment

Yes. Source inspection and current tests provide a high-confidence reproduction shape: configure a Telegram group with per-group `groupPolicy: "open"`, observe normal text route, then send `/status` and the text-command gate returns `null` before any reply.

## Solution Assessment

Yes for the issue direction. The narrow maintainable fix belongs in Telegram text-command authorization so open group policy is honored consistently without broadening explicit allowlists or disabled-group behavior.
