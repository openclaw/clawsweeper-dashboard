## Summary

Keep open. Current main still matches the reported Telegram group split: ordinary text can pass through a per-group `groupPolicy: "open"` configuration, while `/status` is treated as an unauthorized control command and dropped before reply delivery when no effective command sender allowlist matches. PR #74894 is open with closing syntax for this issue, so the issue should stay open until that PR lands or is superseded.

## What This Changes

The issue reports that Telegram group `/status` commands produce no visible reply on OpenClaw 2026.4.27 while normal messages in the same groups still reach the agent.

## Best Possible Solution

Review and land PR #74894, or replace it with an equivalent focused Telegram text-command authorization change, so configured open group and topic sessions deliver `/status` replies while disabled groups and explicit sender allowlists continue to fail closed.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: configure a Telegram group with a per-group `groupPolicy: "open"` override and no effective command sender allowlist, then ordinary text reaches the reply path while `/status` is dropped by the group command gate.

## Solution Assessment

Yes for the direction, pending PR review. The narrow maintainable fix belongs in the Telegram command-authorization path rather than weakening the shared fail-closed command gate for all channels.
