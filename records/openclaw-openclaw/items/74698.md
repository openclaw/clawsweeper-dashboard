## What This Changes

Telegram group `/status` should produce a visible status reply in group sessions where normal Telegram messages still route successfully.

## Best Possible Solution

Resolve https://github.com/openclaw/openclaw/pull/74894 or replace it with a focused Telegram command-authorization fix that preserves disabled groups, explicit allowlists, and the documented command-auth boundary while adding regression coverage for the group text-versus-status split.

Yes, source-level. Configure a Telegram group with a per-group open policy and no effective command sender allowlist: current tests show normal group text routes, while `/status` from the same unallowlisted sender does not call the reply path.
