## Summary

Keep open. Current main still contains the Telegram pre-dispatch command gate that can silently drop a group `/status` when normal group messages pass but the effective command allowlist is empty, and the linked closing PR is still open and unmerged.

## What This Changes

The issue requests restoring visible Telegram group `/status` replies in configured group sessions where ordinary Telegram group messages still reach the agent.

## Best Possible Solution

Review and land PR #74894, or replace it with an equivalent focused Telegram change, so group `/status` replies are delivered in configured group/topic sessions while the existing sender and owner-command authorization boundaries stay intact.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Telegram group where normal group access passes but `effectiveGroupAllow` has no entries; sending `/status` reaches `resolveControlCommandGate`, evaluates unauthorized, and returns `null` before any visible reply is sent.

## Solution Assessment

Yes. The narrowest maintainable fix belongs in the Telegram pre-dispatch text-command gate because the shared command-gating helper is a generic fail-closed primitive and the failure is Telegram-specific handling of configured group sessions with no separate command allowlist.
