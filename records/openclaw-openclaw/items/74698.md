## Summary

Keep open. Current main still contains the Telegram pre-dispatch text-command gate that can silently drop a group `/status` before any status reply is sent, and the linked closing PR is open and unmerged.

## What This Changes

Telegram group `/status` should send a Gateway status reply in configured group sessions instead of silently doing nothing while normal group messages still work.

## Best Possible Solution

Review and land PR #74894, or replace it with an equivalent focused Telegram change, so group `/status` replies are delivered in configured group/topic sessions without weakening existing authorization boundaries.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Telegram group where normal group access passes but the effective command sender allowlist has no matching entry; `/status` reaches `resolveControlCommandGate`, sets `shouldBlock`, and returns `null` before any reply is sent.

## Solution Assessment

Yes. The narrowest maintainable fix belongs in the Telegram pre-dispatch text-command gate or its Telegram-owned authorization inputs, because the shared command-gating helper is a generic fail-closed primitive and the reported mismatch is Telegram group access versus command allowlist handling.
