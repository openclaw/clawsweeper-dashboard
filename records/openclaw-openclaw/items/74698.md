## Summary

Keep open. Current main still has the reported Telegram group split: per-group open policy can let ordinary group text route, but `/status` is treated as an unauthorized control command and dropped before any reply; the linked closing PR is still an implementation candidate rather than grounds to close this issue.

## What This Changes

Telegram group slash commands such as `/status` should reply in configured group sessions where normal Telegram group messages already route successfully.

## Best Possible Solution
