## Summary

Keep open. Current main still has a Telegram group control-command gate that can drop `/status` before the normal reply path when no effective group command allowlist is configured, and the linked closing PR is still open and unmerged in the provided GitHub context.

## What This Changes

Telegram group `/status` commands should reply in configured group sessions where normal text routes successfully, instead of being silently swallowed.

## Best Possible Solution

Review and land PR #74894, or replace it with an equivalent focused Telegram fix, so configured group `/status` commands reach the normal command reply path while explicit sender restrictions and owner-only command checks remain enforced.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a Telegram group that allows normal group messages but has no effective group command allowlist; current tests and source show normal text can reply while `/status` is dropped before dispatch.

## Solution Assessment

Yes. The narrowest maintainable fix belongs in the Telegram message-level pre-dispatch command gate, because the failure happens before core `/status` handling can decide authorization or send a reply.
