## Summary

Keep open: current main still has a split where normal Telegram group messages can use scoped group policy while native `/status` falls through to command authorization that remains allowlist-based. ## What This Changes

The issue reports that Telegram group `/status` commands produce no visible reply in OpenClaw 2026.4.27 even though normal group messages still reach the agent.

## Best Possible Solution

Make an explicit maintainer decision on the Telegram native-command authorization boundary, then land a narrow Telegram plugin fix that preserves `commands.allowFrom`, disabled groups, group/topic allowlists, focused regression coverage, and live Telegram proof.

## Reproduction Assessment

Yes. Source inspection shows current main can allow normal Telegram group messages through scoped group-policy overrides while native `/status` still falls through to allowlist-only command authorization; I did not run live Telegram in this read-only review.

## Solution Assessment

Unclear until maintainer policy acceptance. The narrow Telegram-owned fix attempted in https://github.com/openclaw/openclaw/pull/90644 is the best technical shape I found, but it depends on accepting open group policy as a native-command authorization source when command allowlists are unset.
