## Summary

Keep open. Current main still has a Telegram text-command path where a configured open group can route normal messages but drop `/status` before any reply when the effective command sender allowlist has no entries, and PR #74894 is already open as the linked implementation candidate.

## What This Changes

The issue asks for Telegram group `/status` slash commands to reply in configured group sessions on 2026.4.27 when normal text in the same groups still routes successfully.

## Best Possible Solution

Review and land PR #74894, or replace it with an equivalent focused Telegram command-authorization change, so configured group and topic sessions deliver `/status` replies while fail-closed behavior remains intact for disabled or unauthorized group command use.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: configure a Telegram group with a per-group `groupPolicy: "open"` override and no effective command sender allowlist, then normal text replies while `/status` is dropped before reply delivery.

## Solution Assessment

Yes for the direction, pending PR review. The maintainable fix belongs in the Telegram command-authorization path rather than weakening the shared command gate, because the shared helper is a generic fail-closed primitive and the mismatch is Telegram-specific open-group treatment.
