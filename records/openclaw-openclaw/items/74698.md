## Summary

Keep open. Current main still has the Telegram text-command gate that can silently drop group `/status` when normal group messages pass through a configured open group but the effective command allowlist has no entries, and PR #74894 is still the open/unmerged implementation candidate with closing syntax.

## What This Changes

Telegram group slash commands such as `/status` should produce a Gateway reply in configured group sessions where normal messages already route successfully.

## Best Possible Solution

Review and land PR #74894, or replace it with an equivalent focused Telegram change, so configured group and topic sessions deliver `/status` replies while preserving fail-closed behavior for genuinely unauthorized command senders.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a configured Telegram group where normal group policy access passes but `effectiveGroupAllow` has no entries; sending `/status` makes the control-command gate unauthorized and returns `null` before any reply is sent.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Telegram pre-dispatch text-command authorization path, because the shared gate is a generic fail-closed primitive and the bug is Telegram-specific treatment of configured open group sessions without a separate sender command allowlist.
