## Summary

Keep open. Current main still contains the Telegram group text-command path that can route normal group text but drop `/status` before any reply when the effective group command allowlist has no matching sender, and PR #74894 is already open with closing syntax for this issue.

## What This Changes

The issue reports that Telegram group slash commands such as `/status` silently produce no reply while normal group messages still work.

## Best Possible Solution

Review and land PR #74894, or replace it with an equivalent focused Telegram fix, so configured open group and topic sessions deliver `/status` replies while disabled groups and explicit sender allowlists continue to fail closed.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: configure a Telegram group with a per-group `groupPolicy: "open"` override and no matching effective group command allowlist, then ordinary text reaches the reply path while `/status` is dropped before reply delivery.

## Solution Assessment

Yes for the direction, pending PR review. The narrow maintainable fix belongs in the Telegram text-command authorization path rather than weakening the shared fail-closed command gate.
