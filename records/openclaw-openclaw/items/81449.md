## What This Changes

The issue asks for the Control UI header model picker to immediately patch the active session through `sessions.patch` when a different model is selected.

## Best Possible Solution

Keep the current main implementation and ship it in the next release; users on v2026.5.7 can continue using `/model` as the workaround until then.
