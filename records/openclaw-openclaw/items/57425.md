## Summary

Keep open: current main has meaningful partial restart-drain, safe-restart, restart-continuation, main-session, subagent, and cron catch-up work, but the full graceful restart recovery contract requested here remains incomplete and a member comment explicitly left the broader scope open.

## What This Changes

The issue requests native graceful gateway restart recovery with a pre-restart manifest, post-restart interruption events, cron retry, readiness gating, drain-window replay, and gateway restart configuration.

## Best Possible Solution

Define a restart recovery contract and split implementation across manifest/persistence, channel delivery replay, cron retry semantics, and session/subagent interruption notifications, landing focused reviewed changes against that contract.

## Reproduction Assessment

Yes. Source and existing tests provide a high-confidence path for the remaining gaps: drain enqueues reject, proposed gateway.restart keys are absent, interrupted cron runs are marked failed/disabled rather than retried, and restart recovery has narrow skip/fail cases.

## Solution Assessment

No. Closing this now is not the best solution; current main is a partial implementation and the remaining restart recovery behavior needs maintainer scoping across lifecycle, persistence, cron, and delivery semantics.
