## Summary

Keep open. Current main has meaningful restart-drain, safe-restart preflight, restart-continuation, subagent, cron, and main-session recovery pieces, but it still does not implement the requested graceful restart contract for durable manifests, drain replay, cron retry-on-interrupt, or full interruption propagation; a repository member also explicitly left this issue open after the active-run drain fix.

## What This Changes

Request graceful gateway restarts that preserve or recover active sessions, subagents, cron runs, and messages received during drain.

## Best Possible Solution

Keep this as the umbrella or split it into canonical smaller tracks for restart manifests, readiness gating, durable drain replay, cron retry policy, and interruption propagation.

## Reproduction Assessment

Yes. Source and tests provide a high-confidence path for the remaining gaps: enqueue during drain still rejects with GatewayDrainingError, interrupted cron runs are failed or disabled rather than retried, and the requested gateway.restart config keys are absent.

## Solution Assessment

No. Closing this now is not the best way to solve the issue; current main is a useful partial implementation, but the remaining behavior needs maintainer scoping across lifecycle, persistence, cron, and channel delivery before a narrow fix is safe.
