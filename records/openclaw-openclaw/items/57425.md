## Summary

Keep open: current main has partial safe-restart, restart-drain, main-session, subagent, and cron recovery pieces, but it does not implement the requested manifest, broad interruption propagation, drain-window replay, cron retry, or gateway.restart config contract. A member comment also explicitly left this broader scope open after the active-run drain fix landed.

## What This Changes

Request graceful gateway restart recovery with a pre-restart manifest, post-restart session/subagent/cron recovery, readiness gate, drain-aware message replay, and new gateway.restart config.

## Best Possible Solution

Define a restart recovery contract and split it into focused reviewed changes for manifest/persistence, channel delivery replay, cron retry semantics, and session/subagent interruption notifications.

## Reproduction Assessment

Yes. Source inspection on current main shows the remaining gaps: drain-window enqueues reject, the requested gateway.restart keys are absent, interrupted one-shot cron runs are failed/disabled, and recovery paths are partial/internal rather than a full manifest/replay contract.

## Solution Assessment

No. Closing this now is not the best solution because current main is only a partial implementation; the remaining behavior needs maintainer-scoped lifecycle, persistence, cron, and delivery semantics.
