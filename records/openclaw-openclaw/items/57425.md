## Summary

Keep open. Current main has substantial partial recovery and drain work, but a maintainer already called PR #71465 only the active-run kill fix, and source/tests still show the broader graceful-restart surface remains incomplete.

## What This Changes

The request is for graceful gateway restart behavior covering active-session manifests, post-restart recovery notifications, readiness/force gating, drain-window message replay, and interrupted cron retry semantics.

## Best Possible Solution

Keep this as the umbrella unless maintainers split it into canonical smaller issues for readiness gating, durable drain replay, restart manifest semantics, and cron retry policy.

Yes. Source and tests provide a high-confidence reproduction path for remaining gaps: new enqueues reject during drain, proposed restart config keys are absent, and cron startup tests assert interrupted jobs are failed/disabled rather than requeued.

## Solution Assessment

No, closing this now is not the best solution. Current main is a useful partial implementation, but the remaining behavior needs maintainer scoping across lifecycle, persistence, cron, and channel delivery.
