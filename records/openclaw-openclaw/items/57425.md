## Summary

Keep open: current main has partial safe-restart, restart-drain, main-session, subagent, and cron recovery pieces, but the broader manifest, replay, cron retry, and interruption-propagation contract requested here is still missing; a member comment explicitly left that remaining scope open.

## What This Changes

The issue requests graceful gateway restarts that recover active sessions, subagents, cron runs, and messages received during drain.

## Best Possible Solution

Define a restart recovery contract and split it into focused reviewed changes for manifest/persistence, channel delivery replay, cron retry semantics, and session/subagent interruption notifications.

## Reproduction Assessment

Yes. Source inspection on current main shows the remaining gaps: drain-window enqueues reject, the requested gateway.restart config keys are absent, interrupted cron runs are marked failed rather than retried, and existing recovery paths are partial/internal rather than a full manifest/replay contract.

## Solution Assessment

No. Closing this now is not the best solution because current main is only a partial implementation; the remaining behavior needs maintainer-scoped lifecycle, persistence, cron, and delivery semantics.
