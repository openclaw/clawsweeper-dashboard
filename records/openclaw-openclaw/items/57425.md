## Summary

Keep open. Current main has meaningful partial restart protection, and the maintainer comment explicitly says PR #71465 covered only the active-run kill/drain portion, while broader session recovery, interruption propagation, readiness gating, durable drain-window replay, and cron retry semantics remain unresolved.

## What This Changes

Request graceful gateway restarts that preserve or recover in-flight sessions, cron runs, subagent interruption context, drain-window messages, readiness checks, and related restart recovery configuration.

## Best Possible Solution

Keep this as an umbrella until maintainers either split the remaining restart-recovery work into narrower implementation issues or approve a single design that extends the existing drain, restart sentinel, main-session recovery, subagent orphan recovery, cron state, and delivery queue mechanisms without adding a competing lifecycle path.

## Reproduction Assessment

Yes. The current-main source and tests give a high-confidence reproduction path for the remaining gaps: new enqueues reject during drain, cron interrupted runs are marked failed/disabled rather than retried, and the requested restart config/readiness surface is absent.

## Solution Assessment

No. Closing this now is not the best way to solve it because current main only partially addresses the report; the maintainable path is maintainer-scoped design or split follow-up issues for the still-missing restart recovery pieces.
