## Summary

Keep open: current main has several partial restart-recovery pieces, but not the requested coherent gateway restart manifest, drain-window replay, cron retry, interruption propagation, readiness policy, or `gateway.restart` config contract.

## What This Changes

The issue asks for graceful gateway restarts that capture interrupted sessions and cron runs, recover or notify them after startup, gate unsafe restarts, queue drain-window messages, and expose restart recovery config.

## Best Possible Solution

Define a versioned restart recovery contract that composes the existing partial pieces with manifest/persistence, drain-window replay, cron retry semantics, session/subagent interruption notices, and documented config policy.

## Reproduction Assessment

Yes. Source inspection on current main shows the remaining gap: new lane enqueues still reject during drain, cron startup marks interrupted jobs as errors, and no requested `gateway.restart` config block exists; I did not run a live restart because this review is read-only.

## Solution Assessment

Unclear. The manifest approach is plausible, but the best solution needs a maintainer-owned contract across restart lifecycle, durable state, cron retry, delivery replay, subagent reporting, and user-visible policy rather than a single broad automatic patch.
