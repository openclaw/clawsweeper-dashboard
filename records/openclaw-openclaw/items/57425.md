## Summary

Keep open. Current main has substantial partial fixes for restart drain and main-session recovery, but the issue still tracks broader product decisions around cron retry semantics, subagent interruption propagation, inbound replay, readiness-gate defaults, and new restart configuration surfaces.

## What This Changes

The issue requests graceful gateway restarts that preserve or recover interrupted sessions, cron runs, subagents, and queued messages after restart.

## Best Possible Solution

Keep this as the maintainer-owned graceful-restart umbrella and split remaining work into narrower designs for cron retry, subagent notification, inbound replay, and restart policy once the desired semantics are confirmed.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection does show the remaining behavior boundaries: drain rejects new enqueues with GatewayDrainingError, cron runs are aborted/reset rather than durably retried, and main-session recovery skips cron/subagent session keys.

## Solution Assessment

Unclear. Current main's drain plus main-session recovery is a good partial implementation, but the requested manifest, cron retry, readiness-gate defaults, inbound replay, and config knobs need maintainer product direction before a best single fix can be named.
