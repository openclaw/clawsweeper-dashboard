## Summary

## What This Changes

The issue requests graceful gateway restart handling that preserves or recovers active sessions, cron runs, subagents, and drain-window messages across restart.

## Best Possible Solution

Define one maintainer-owned restart recovery contract that composes the existing drain, restart-sentinel, main-session, subagent, task, and cron pieces, then split implementation into narrower PRs for durable manifest/checkpoint storage, replay/idempotency, cron retry, and user-visible interruption notices.

## Reproduction Assessment

Yes, at source level. Current main still rejects drain-window enqueues, marks interrupted cron runs failed instead of retrying them, lacks the requested `gateway.restart` config block, and only partially recovers main sessions after forced or timed-out drains.

## Solution Assessment

Unclear. The manifest approach is plausible, but the best maintainable solution needs a maintainer-owned contract across restart lifecycle, durable state, cron retry, replay idempotency, subagent reporting, and channel UX rather than a single broad patch.
