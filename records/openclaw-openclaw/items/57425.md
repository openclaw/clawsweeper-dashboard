## Summary

Keep open. Current main has meaningful partial restart protection and recovery, but it still does not cover the full requested graceful-restart scope: cron interruption retry semantics, durable drain-window/inbound replay, restart readiness policy, and the proposed config surface remain unresolved product/design work.

## What This Changes

The issue requests graceful gateway restart handling with session recovery, cron retry, readiness gating, drain-aware message queuing, and restart configuration.

## Best Possible Solution

Keep this as the maintainer-owned graceful-restart umbrella, and land or split narrower designs for cron retry, durable inbound/followup replay, restart readiness policy, and recovery-through-delivery once maintainers choose the semantics.

## Reproduction Assessment

No high-confidence live reproduction covers the full umbrella. Source inspection does reproduce central remaining slices: drain-window enqueues still fail with GatewayDrainingError, and interrupted cron runs are marked failed rather than durably retried.

## Solution Assessment

Unclear as one implementation. Current main's drain and main/subagent recovery are good partial fixes, but the proposed manifest, cron retry, readiness gate, inbound replay, and config knobs need maintainer product direction before a best single fix can be named.
