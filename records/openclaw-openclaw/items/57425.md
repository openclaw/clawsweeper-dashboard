## Summary

Keep open. Current main has useful restart-drain, safe-restart preflight, restart-continuation, and partial main/subagent recovery work, but the broader graceful session recovery contract requested here is still incomplete and a member comment explicitly left this issue open after the active-run drain fix.

## What This Changes

Request graceful gateway restarts that capture interrupted sessions and cron runs, recover or notify them after restart, gate unsafe restarts, and durably replay messages received during drain.

## Best Possible Solution

Keep this as the umbrella or split it into canonical smaller tracks for restart manifest semantics, readiness gating, durable drain replay, cron retry policy, and interruption propagation.

## Reproduction Assessment

Yes. Source and tests provide a high-confidence reproduction path for the remaining gaps: current main still rejects new enqueues during drain, lacks several proposed restart config keys, and marks interrupted cron runs failed or disabled instead of requeuing them.

## Solution Assessment

No, closing this now is not the best way to solve the issue. Current main is a useful partial implementation, but the remaining behavior needs maintainer scoping across lifecycle, persistence, cron, and channel delivery before a narrow fix is safe.
