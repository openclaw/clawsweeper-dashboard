## What This Changes

This PR expands the `model.usage` diagnostic event catalog entry and adds a logging note that diagnostic events use the event bus rather than JSONL file logs.

## Best Possible Solution

Keep the expanded field catalog, but qualify the logging note so it says diagnostic events go through the event bus and names only consumers that actually receive the cited event examples.

## Reproduction Assessment

Yes, for the remaining docs defect. Source inspection shows production `model.usage` is emitted as trusted while the stability recorder subscribes through `onDiagnosticEvent`, which filters trusted events.

## Solution Assessment

No, not as-is. Updating the diagnostic catalog and logging page is the right layer, but the logging note should qualify or remove the stability-recorder consumer claim before merge.
