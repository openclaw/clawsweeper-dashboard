## Summary

Keep this PR open but do not merge it as-is. The linked bug is real on current main, but this branch still allows a setup-created profiled workspace to be treated as a shared base when evaluated under a different profile, preserving the nested workspace layout it is trying to prevent.

## What This Changes

This PR changes agent workspace resolution for setup-managed defaults, adds resolver regression tests, documents peer-level agent workspaces, and adds a changelog entry for #71889.

## Best Possible Solution

Land one canonical fix for the linked workspace bug that keeps new `agents add` defaults peer-level, preserves explicit workspaces and intentional #59789 shared bases, and covers profile transitions in both directions with focused resolver or wizard tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: persist `agents.defaults.workspace` as `<stateDir>/workspace-dev` from setup under `OPENCLAW_PROFILE=dev`, then evaluate a non-default agent without that profile; PR head falls through to `<stateDir>/workspace-dev/<agentId>`.

## Solution Assessment

No. This branch improves the default setup case, but its runtime managed-workspace classifier is incomplete across profile transitions; a complete classifier or the narrower wizard-prefill approach from #71941 is safer.
