## Summary

Keep this PR open. The branch targets a real current-main bug and includes useful regression coverage, but its managed-workspace guard still misses a setup-created workspace from a different profile, leaving a nested workspace path in the same class of case the PR is trying to prevent; the open overlapping fix PR also needs maintainer choice before this one can be superseded or merged.

## What This Changes

The PR changes agent workspace resolution so setup-created main workspace defaults fall back to peer-level `workspace-<agentId>` paths, adds resolver regression tests, documents peer-level workspace placement, and adds a changelog entry for the linked bug.

## Best Possible Solution

Land one canonical fix for the linked workspace-prefill bug that keeps brand-new `agents add` defaults peer-level, preserves explicit workspaces and the #59789 custom shared-base contract, covers both profile-transition directions, and updates the changelog/docs in the same PR.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: setup writes `agents.defaults.workspace`, the wizard uses `resolveAgentWorkspaceDir` for `initialValue`, and the current resolver appends non-default ids under that fallback; the PR head still reproduces this when a profile-created `workspace-<profile>` fallback is later evaluated without that profile.

## Solution Assessment

No. This PR improves the default setup case, but the runtime guard is incomplete for profile transitions and broadens the fix beyond the wizard prefill; a wizard-only suggestion helper or a complete managed-main-workspace classifier would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Recognize profiled setup workspaces before nesting:** `src/agents/agent-scope-config.ts:72-73`
  - body: The guard only treats `<stateDir>/workspace` and the current profile's default workspace as setup-managed. If setup ran with `OPENCLAW_PROFILE=dev` and persisted `<stateDir>/workspace-dev`, running the wizard later without that profile leaves the fallback unrecognized, so this branch still returns `<stateDir>/workspace-dev/<agentId>`, preserving the nested layout the patch is meant to prevent.
  - confidence: 0.84
