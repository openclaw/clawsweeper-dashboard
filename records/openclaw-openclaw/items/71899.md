## Summary

Keep this PR open. Current main still has the reported `agents add` prefill bug, but this patch is not ready to merge because its managed-workspace guard still misses a profile-created main workspace when the wizard later runs outside that profile; the related open PR #71941 is an alternate implementation that maintainers should compare rather than closing this work now.

## What This Changes

The PR changes agent workspace resolution so setup-created main workspace defaults fall back to peer-level `workspace-<agentId>` paths, adds resolver regression tests, documents peer-level workspace placement, and adds a changelog entry for #71889.

## Best Possible Solution

Land one canonical fix for #71889 that keeps brand-new `agents add` defaults peer-level, preserves explicit workspaces and the #59789 custom shared-base contract, covers both profile-transition directions, and updates focused wizard/resolver tests plus the changelog/docs in the same chosen PR.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: setup writes `agents.defaults.workspace`, the wizard uses `resolveAgentWorkspaceDir` for `initialValue`, and current main appends non-default ids under that fallback; PR head still reproduces the same nesting when a profile-created `workspace-<profile>` fallback is later evaluated without that profile.

## Solution Assessment

No. This PR improves the default setup case but the runtime guard is incomplete for profile transitions and changes a shared resolver; a wizard-only suggestion helper or a complete managed-main-workspace classifier would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Recognize profiled setup workspaces before nesting:** `src/agents/agent-scope-config.ts:72-73`
  - body: The guard only treats `<stateDir>/workspace` and the current profile's default workspace as setup-managed. If setup ran with `OPENCLAW_PROFILE=dev` and persisted `<stateDir>/workspace-dev`, running the wizard later without that profile leaves the fallback unrecognized, so this branch still returns `<stateDir>/workspace-dev/<agentId>` and preserves the nested layout the patch is meant to prevent.
  - confidence: 0.84

## Security Review
