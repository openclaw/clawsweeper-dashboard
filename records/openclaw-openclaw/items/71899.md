## Summary

Close as superseded by #71941. The reported bug is real on current main, but this branch uses a runtime resolver change that still leaves a profile-transition nesting case, while the linked open PR tracks the same #71889 fix with a narrower wizard-prefill approach.

## What This Changes

This PR changes agent workspace resolution to avoid nesting added agents under a setup-created main workspace, and adds resolver tests, a multi-agent docs note, and a changelog entry for #71889.

## Best Possible Solution

Close this superseded PR and keep #71889/#71941 as the canonical path: land one narrow fix that changes the `agents add` default for brand-new agents to a peer-level workspace while preserving explicit workspaces, intentional shared bases, and existing runtime resolution compatibility.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path on current main: setup persists `agents.defaults.workspace`, the `agents add` wizard uses `resolveAgentWorkspaceDir` for the initial value, and the resolver appends the new id under that fallback.

## Solution Assessment

No. This PR improves one default setup path, but changing runtime resolution is broader than necessary and still misses a setup-created workspace from a different profile; the safer solution is the wizard-only suggestion tracked in #71941 or an equally complete managed-workspace classifier.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Recognize profiled setup workspaces before nesting:** `src/agents/agent-scope-config.ts:72-73`
  - body: The guard only treats `<stateDir>/workspace` and the current profile's default workspace as setup-managed. If setup ran with `OPENCLAW_PROFILE=dev` and later `agents add work` runs without that profile, the persisted `<stateDir>/workspace-dev` is treated as a shared base and this branch returns `<stateDir>/workspace-dev/work`, preserving the nested layout this PR is meant to avoid.
  - confidence: 0.84
