## Summary

Keep this PR open. The linked bug is real on current main, but this branch still has a blocking profile-transition gap in the runtime workspace guard, and an overlapping wizard-scoped alternative (#71941) is already tracking the same user-facing fix path.

## What This Changes

The PR changes agent workspace resolution so setup-created main workspace defaults fall back to peer-level `workspace-<agentId>` paths, adds resolver regression tests, documents peer-level workspace placement, and adds a changelog entry for the linked bug.

## Best Possible Solution

Land one canonical fix for the workspace-prefill bug that keeps brand-new interactive `agents add` defaults peer-level, preserves explicit workspaces and intentional shared-base configs, covers profile transitions in both directions, and keeps the docs/changelog aligned with the chosen behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: setup can persist `agents.defaults.workspace`, the wizard feeds `resolveAgentWorkspaceDir` into the prompt, and at PR head a setup run under `OPENCLAW_PROFILE=dev` followed by `agents add` without that profile still resolves under `workspace-dev/<agentId>`.

## Solution Assessment

No. This PR fixes the default-profile scenario but broadens the change into runtime resolution and leaves a cross-profile managed-workspace case unrecognized; a wizard-scoped helper like #71941 or a complete managed-main classifier would be safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Recognize profiled setup workspaces before nesting:** `src/agents/agent-scope-config.ts:72-73`
  - body: The guard only treats `<stateDir>/workspace` and the current profile's default workspace as setup-managed. If setup ran with `OPENCLAW_PROFILE=dev` and persisted `<stateDir>/workspace-dev`, running the wizard later without that profile leaves the fallback unrecognized, so this branch still returns `<stateDir>/workspace-dev/<agentId>` and preserves the nested layout the patch is meant to prevent.
  - confidence: 0.84
