## Summary

Keep this PR open for maintainer handling: current main still has the nested workspace default bug, but this branch still leaves a profile-transition path that preserves the same nesting footgun and overlaps with the wizard-scoped fix being tracked in #71941.

## What This Changes

The PR changes agent workspace resolution so setup-created main workspace defaults fall back to peer-level `workspace-<agentId>` paths, adds resolver regression tests, documents peer-level workspace placement, and adds a changelog entry for #71889.

## Best Possible Solution

Land one canonical fix for #71889 that keeps brand-new `agents add` defaults peer-level, preserves explicit workspaces and intentional #59789 shared bases, covers profile transitions in both directions, and carries focused resolver/wizard tests plus docs and changelog updates.

## Reproduction Assessment

Yes. Configure `agents.defaults.workspace` as `<stateDir>/workspace-dev` the way setup would under `OPENCLAW_PROFILE=dev`, then evaluate `resolveAgentWorkspaceDir` for a non-default agent without that profile; PR head still returns `<stateDir>/workspace-dev/<agentId>`.

## Solution Assessment

No. This branch improves the default setup case, but the managed-workspace classifier is incomplete across profile transitions and broadens the fix into runtime resolution; a complete classifier or a narrower wizard-prefill helper is safer.
