## Summary

This PR is a focused and useful fix for the stale `running` session-row starvation report, but it should not merge yet because the new heal path can run after `chat.abort` rejects the abort request.

## What This Changes

The branch clears stale `running` session rows on `sessions.abort` no-active-run, broadens restart recovery to markerless running rows, and adds regression coverage for both paths.

## Best Possible Solution

Keep the stale-row recovery direction, but gate healing on an authorized successful no-active-run abort result and make the restart recovery topology assumption explicit or proven before merge.

## Reproduction Assessment

Yes, at source level. Current main has a no-active-run abort path with no stale-row mutation and restart recovery still requires `abortedLastRun === true`; I did not run tests because this review is read-only.

## Solution Assessment

No, not yet. The recovery approach is the right narrow code path, but the PR must separate an authorized no-active-run result from a rejected `chat.abort` before it mutates session state.
