## Summary

Keep open: current main still has the transcript cwd bug, and this PR is directionally useful but incomplete because it preserves process.cwd() fallback paths and lacks contributor real-behavior proof.

## What This Changes

The PR replaces process.cwd() with agent workspace resolution in sessions.create, mirrored transcript, and chat-created transcript header paths, plus a focused sessions helper regression test.

## Best Possible Solution

Land one current-main-compatible fix that derives a single effective agent id and workspace from existing runtime config for every create-if-missing transcript header, with regression coverage for mirrored, sessions.create, chat, and abort-created paths.

## Reproduction Assessment

Yes. Source inspection on current main shows the three create-if-missing transcript header paths still write process.cwd(), and the abort persistence path can create a missing transcript without an agent id.

## Solution Assessment

No, not as submitted. The PR moves in the right direction, but it leaves process.cwd() fallback paths reachable instead of deriving one explicit workspace for every transcript header creation path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Derive mirrored transcript cwd without a fallback:** `src/config/sessions/transcript.ts:78-80`
  - body: ensureSessionHeader still falls back to process.cwd() when agentId is absent. appendAssistantMessageToSessionTranscript accepts optional agentId, so mirrored transcript creation can still write the runtime cwd instead of deriving the workspace from the effective session agent.
  - confidence: 0.78

  - confidence: 0.86
