## Summary

Keep open. Current main still has the transcript-header cwd bug, but this PR is not merge-ready because it leaves reachable `process.cwd()` fallbacks, reloads config inside post-send mirror bookkeeping, and lacks contributor real-behavior proof.

## What This Changes

The branch replaces `process.cwd()` with agent workspace resolution in sessions.create, mirrored transcript, and chat-created transcript header paths, and adds a focused sessions helper regression test.

## Best Possible Solution

Land one current-main-compatible explicit-workspace fix that derives a single effective agent id and workspace from existing runtime config for every create-if-missing transcript header, with regression coverage across mirrored, sessions.create, chat.send, and abort-created paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a brand-new transcript header while the process cwd differs from the target agent workspace, and current main writes the process-global cwd into the header.

## Solution Assessment

No, not as submitted. The PR moves in the right direction for sessions.create, but the maintainable fix should derive one effective agent id and workspace from existing runtime config without fallback paths or post-send config reloads.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Pass existing config into mirrored cwd resolution:** `src/config/sessions/transcript.ts:79`
  - body: `appendAssistantMessageToSessionTranscript()` already accepts runtime config and outbound callers pass it. Calling `loadConfig()` inside header creation happens after delivery; if the on-disk config is temporarily invalid or unreadable, mirror bookkeeping can make a successful send fail or retry. Resolve the workspace from the existing config before writing the header.
  - confidence: 0.82
- **[P2] Derive mirrored transcript cwd without a fallback:** `src/config/sessions/transcript.ts:78-80`
  - body: `ensureSessionHeader()` still falls back to `process.cwd()` when `agentId` is absent. Since mirrored transcript appends allow optional `agentId`, create-if-missing transcript headers can still record `/app` or another agent workspace instead of the target workspace.
  - confidence: 0.78
- **[P2] Derive abort-created transcript cwd without a fallback:** `src/gateway/server-methods/chat.ts:760-762`
  - body: `ensureTranscriptFile()` still falls back to `process.cwd()` when no `agentId` is passed. `persistAbortedPartials()` creates missing transcripts through `appendAssistantTranscriptMessage()` without an agent id, so aborted runs can still write Docker `/app` or another process-global cwd into new transcript headers.
  - confidence: 0.86
