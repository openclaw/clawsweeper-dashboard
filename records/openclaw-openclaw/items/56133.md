## Summary

Keep open. The bug still exists on current main, and this PR is a useful candidate direction, but the branch still leaves reachable process-global cwd paths, lacks contributor real behavior proof, and overlaps the open canonical issue and another open candidate fix.

## What This Changes

The PR changes three session transcript header creation paths to resolve cwd from the agent workspace and adds a focused gateway regression test.

## Best Possible Solution

Land one current-main-compatible fix that derives a single effective agent id and workspace from runtime config for every create-if-missing transcript header path, with focused regression coverage and redacted Docker/runtime proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a brand-new mirrored, sessions.create, or chat-created transcript while process cwd differs from the target agent workspace, and current main writes the process-global cwd into the header.

## Solution Assessment

No. The direction is right, but this patch still reloads config inside helpers, preserves process.cwd fallbacks when agentId is absent, and leaves abort-created chat transcripts able to reach the fallback path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Use runtime config for mirrored transcript cwd:** `src/config/sessions/transcript.ts:79`
  - body: Calling loadConfig() inside header creation makes transcript mirroring depend on disk config after delivery already succeeded, even though callers already pass runtime config. Thread that config into cwd resolution so bookkeeping does not fail or retry because on-disk config changed.
  - confidence: 0.86
- **[P2] Derive mirrored cwd without process cwd fallback:** `src/config/sessions/transcript.ts:80`
  - body: When agentId is absent, this helper still writes process.cwd() into a newly created header, preserving the /app or cross-agent workspace case the PR is meant to remove. Resolve one effective agent id/workspace before header creation and pass that through.
  - confidence: 0.78
- **[P2] Remove the chat transcript process cwd fallback:** `src/gateway/server-methods/chat.ts:762`
  - body: persistAbortedPartials() still calls appendAssistantTranscriptMessage() without an agent id, so this fallback remains reachable and abort-created transcripts can still record the process-global cwd. Resolve the target agent/workspace before calling ensureTranscriptFile().
  - confidence: 0.84
