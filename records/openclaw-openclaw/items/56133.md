## Summary

Keep open. Current main still has the reported transcript `cwd` bug, and this PR moves in the right direction, but it is not mergeable as submitted because changed create-if-missing paths still have process-global `cwd` fallbacks, the branch reloads config inside transcript helpers, contributor real behavior proof is absent, and the work overlaps an open canonical issue and candidate PR.

## What This Changes

The branch replaces transcript header `cwd` writes with `resolveAgentWorkspaceDir` across sessions.create, mirrored assistant transcript, and chat-created transcript paths, and adds a focused sessions transcript regression test.

## Best Possible Solution

Land one current-main-compatible explicit-workspace fix that derives a single effective agent id and workspace from existing runtime config for every create-if-missing transcript header, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a brand-new mirrored, sessions.create, or chat-created transcript while the process cwd differs from the target agent workspace, and current main writes that process-global cwd into the header.

## Solution Assessment

No. The branch is directionally useful, but the submitted fix still reloads config inside transcript helpers and leaves `process.cwd()` fallbacks reachable instead of deriving one effective agent id and workspace from existing runtime config.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Use runtime config for mirrored transcript cwd:** `src/config/sessions/transcript.ts:79`
  - body: Calling `loadConfig()` inside header creation makes transcript mirroring depend on disk config after delivery already succeeded; a transient invalid config can turn bookkeeping into a send failure or retry even though callers already pass runtime config. Thread that config into cwd resolution instead.
  - confidence: 0.86
- **[P2] Derive mirrored cwd without process cwd fallback:** `src/config/sessions/transcript.ts:80`
  - body: When `agentId` is absent, this helper still writes `process.cwd()` into a newly created header, preserving the `/app` or cross-agent workspace case the PR is meant to remove. Resolve one effective agent id/workspace before header creation and pass that through.
  - confidence: 0.78
- **[P2] Remove the chat transcript process cwd fallback:** `src/gateway/server-methods/chat.ts:762`
  - body: `persistAbortedPartials()` still calls `appendAssistantTranscriptMessage()` without an agent id, so this fallback remains reachable and abort-created transcripts can still record the process-global cwd. Resolve the target agent/workspace before calling `ensureTranscriptFile()`.
  - confidence: 0.84
