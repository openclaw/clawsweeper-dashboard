## Summary

Keep open. Current main still has the transcript `cwd` regression, and the PR targets the right area, but it is not merge-ready because the chat-created transcript helper still falls back to `process.cwd()` for create-if-missing callers that do not pass an agent id; the related #49523/#50454 cluster should converge on one current-main-compatible fix. https://github.com/openclaw/openclaw/pull/56133

## What This Changes

The PR changes session transcript header creation to derive `cwd` from the agent workspace instead of `process.cwd()` and adds focused regression coverage.

## Best Possible Solution

Land one current-main-compatible fix that removes `process.cwd()` from all transcript header creation paths, derives the target workspace from explicit config/session context, and covers sessions.create, mirrored delivery transcripts, chat-created transcripts, and abort-created missing transcripts with regression tests.

## Reproduction Assessment

Yes. Current-main source shows three transcript header creation helpers writing `cwd: process.cwd()`, and the PR body gives Docker evidence where that becomes `/app` instead of the agent workspace.

## Solution Assessment

No, not as submitted. The direction is correct, but the chat helper must derive the agent from the session key/config or require an agent id before writing a new transcript header instead of retaining a process-cwd fallback.
