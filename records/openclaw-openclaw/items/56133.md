## Summary

Keep open. Current main still has the reported transcript-header `process.cwd()` behavior, and this PR is directionally useful but incomplete because one create-if-missing path can still write the Docker process cwd instead of the agent workspace.

## What This Changes

The PR changes session transcript header creation to derive `cwd` from the agent workspace in sessions.create, mirrored transcript appends, and chat-created transcripts, with focused regression coverage.

## Best Possible Solution

Land one current-main-compatible fix that removes `process.cwd()` from all transcript header creation paths, derives the workspace from explicit config/session context, and covers sessions.create, mirrored delivery transcripts, chat-created transcripts, and abort-created missing transcripts with regression tests.

## Reproduction Assessment

Yes. Current-main source shows three transcript header creation helpers writing `cwd: process.cwd()`, and the PR body provides Docker evidence where that becomes `/app` instead of the agent workspace.

## Solution Assessment

No, not as submitted. The direction is correct, but the chat helper must derive the agent from session/config context or require a resolved agent id before writing a new transcript header.
