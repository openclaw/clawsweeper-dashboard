## Summary

Keep open. Current main still has the source-level transcript `cwd` regression, but this PR is not merge-ready because it is conflicting, lacks external real behavior proof, and still leaves fallback/config-resolution gaps in the session transcript paths.

## What This Changes

The PR changes session, mirrored outbound, and chat transcript header creation to use an agent workspace-derived `cwd` instead of `process.cwd()`, with a focused gateway regression test.

## Best Possible Solution

Land one consolidated current-main-compatible fix that computes one effective agent id/workspace from runtime config for every create-if-missing transcript header path, removes process-global cwd fallbacks, adds focused regression coverage, and includes redacted runtime transcript proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a brand-new mirrored, `sessions.create`, or chat-created transcript while `process.cwd()` differs from the target agent workspace, and current main writes the process-global cwd into the header.

## Solution Assessment

No. The direction is right, but this branch should use runtime config and one effective agent id/workspace consistently, remove the `process.cwd()` fallbacks, and be rebased against current main before it is the best fix path.
