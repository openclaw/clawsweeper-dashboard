## Summary

Keep open. The regression is valid on current main and the PR moves in the right direction, but it still leaves a create-if-missing chat abort path able to write `process.cwd()` into a new transcript header.

## What This Changes

The PR changes session transcript header creation to use agent workspace resolution instead of `process.cwd()` across sessions, mirrored transcripts, and chat-created transcripts, with focused helper coverage.

## Best Possible Solution

Land one current-main-compatible fix that removes `process.cwd()` from all agent session transcript header creation paths, resolves the target workspace from explicit config/session context, and covers the cluster with regression tests.

## Reproduction Assessment

Yes. Current-main source shows three transcript header creation helpers writing `process.cwd()`, and the PR body gives Docker evidence where that becomes `/app` instead of the agent workspace.

## Solution Assessment

No, not as submitted. The resolver direction is right, but the chat helper must derive or require the agent id before create-if-missing writes instead of keeping a process-cwd fallback.
