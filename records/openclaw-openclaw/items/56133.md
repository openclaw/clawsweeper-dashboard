The PR changes mirrored, chat, and sessions.create transcript header creation to resolve cwd from the agent workspace instead of `process.cwd()`, and adds a focused sessions.create regression test.

## Best Possible Solution

Land one current-main-compatible fix that computes one effective agent id and workspace from runtime config for every create-if-missing transcript header path, with focused regression coverage and redacted runtime proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a brand-new mirrored, sessions.create, or chat-created transcript while `process.cwd()` differs from the target agent workspace, and current main writes the process-global cwd into the header.

## Solution Assessment

No. The direction is right, but this branch should use runtime config and one effective agent id/workspace consistently, remove the `process.cwd()` fallbacks, and be rebased against current main before it is the best fix path.
