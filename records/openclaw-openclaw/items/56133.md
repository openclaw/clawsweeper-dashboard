## Summary

Keep open. Current main and the latest release still contain the transcript `cwd` bug this PR targets, but the branch is conflicting, lacks real behavior proof, and keeps config/fallback gaps that make it unsafe as-is.

## What This Changes

The branch changes session, mirrored outbound, and chat-created transcript headers to derive `cwd` from an agent workspace instead of `process.cwd()`, and adds a focused gateway regression test.

## Best Possible Solution

Land one consolidated fix that uses caller runtime config plus one normalized effective agent id/workspace for every create-if-missing transcript header path, removes ambient cwd fallbacks, adds focused regression coverage, and includes redacted runtime proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a brand-new mirrored, `sessions.create`, or chat-created transcript while `process.cwd()` differs from the target agent workspace, and current main writes the process-global cwd into the header.

## Solution Assessment

No. The explicit-workspace direction is right, but this branch should use runtime config consistently, resolve one effective agent/workspace, remove `process.cwd()` fallbacks, and be rebased before it is the best fix path.
