## Summary

Keep open. Current main still has the transcript `cwd` bug and this PR is directionally useful, but the patch is not merge-ready because reachable `process.cwd()` fallbacks remain, runtime config is reloaded inside transcript helpers, real behavior proof is missing, and the work overlaps the open canonical bug and candidate fix at https://github.com/openclaw/openclaw/issues/49523 and https://github.com/openclaw/openclaw/pull/50454.

## What This Changes

The branch replaces transcript header `cwd` writes with `resolveAgentWorkspaceDir` in sessions.create, mirrored assistant transcript, and chat-created transcript paths, plus a focused sessions transcript test.

## Best Possible Solution

Land one current-main-compatible fix that derives a single effective agent id and workspace from existing runtime config for every create-if-missing transcript header, with focused regression coverage and redacted runtime proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a brand-new mirrored, sessions.create, or chat-created transcript while the process cwd differs from the target agent workspace, and current main writes that process-global cwd into the header.

## Solution Assessment

No. The direction is right, but the submitted patch still reloads config inside helpers and leaves `process.cwd()` fallbacks reachable instead of resolving one effective agent id and workspace before header creation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Use runtime config for mirrored transcript cwd:** `src/config/sessions/transcript.ts:79`
  - body: Calling `loadConfig()` inside header creation makes transcript mirroring depend on disk config after delivery already succeeded, even though callers already pass runtime config. Thread that config into cwd resolution instead so bookkeeping does not fail or retry because on-disk config changed.
  - confidence: 0.86
- **[P2] Derive mirrored cwd without process cwd fallback:** `src/config/sessions/transcript.ts:80`
  - body: When `agentId` is absent, this helper still writes `process.cwd()` into a newly created header, preserving the `/app` or cross-agent workspace case the PR is meant to remove. Resolve one effective agent id/workspace before header creation and pass that through.
  - confidence: 0.78
- **[P2] Remove the chat transcript process cwd fallback:** `src/gateway/server-methods/chat.ts:762`
  - body: `persistAbortedPartials()` still calls `appendAssistantTranscriptMessage()` without an agent id, so this fallback remains reachable and abort-created transcripts can still record the process-global cwd. Resolve the target agent/workspace before calling `ensureTranscriptFile()`.
  - confidence: 0.84
