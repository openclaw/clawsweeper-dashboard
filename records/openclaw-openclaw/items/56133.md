## Summary

Keep open. The regression is still present on current main, but this external PR is not merge-ready because it is proof-gated, merge-conflicting, overlaps another open fix lane, and still leaves session/workspace correctness gaps in the patched paths.

## What This Changes

The branch changes session, mirrored outbound, and chat transcript header creation to derive `cwd` from an agent workspace instead of `process.cwd()`, plus a focused gateway regression test.

## Best Possible Solution

Land one consolidated current-main-compatible fix that computes one effective agent id/workspace from runtime config for every create-if-missing transcript header path, removes process-global cwd fallbacks, adds focused regression coverage, and includes redacted runtime transcript proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a brand-new mirrored, `sessions.create`, or chat-created transcript while `process.cwd()` differs from the target agent workspace, and current main writes the process-global cwd into the header.

## Solution Assessment

No. The explicit-workspace direction is right, but this branch should use runtime config and one effective agent id/workspace consistently, remove the `process.cwd()` fallbacks, and be rebased against current main before it is the best fix path.
