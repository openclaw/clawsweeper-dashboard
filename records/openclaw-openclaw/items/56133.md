## Summary

Keep open: the bug is valid on current main and the PR is directionally useful, but it still leaves a create-if-missing transcript path that can write `process.cwd()` when abort persistence lacks an agent id.

## What This Changes

The PR changes gateway/session transcript header creation to derive `cwd` from an agent workspace resolver in sessions.create, mirrored transcript appends, and chat-created transcripts, plus adds focused regression coverage.

## Best Possible Solution

Land one current-main-compatible fix that removes `process.cwd()` from transcript header create-if-missing paths, derives cwd from explicit config plus one effective agent id, and covers sessions.create, mirrored delivery, chat-created, and abort-created missing transcripts with regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows the three transcript header helpers still write `cwd: process.cwd()`, and the PR body provides Docker evidence that this becomes `/app` instead of the agent workspace.

## Solution Assessment

No, not as submitted. The direction is right, but every create-if-missing header path must derive an explicit workspace before writing; the chat helper's `process.cwd()` fallback keeps one reported failure mode alive.
