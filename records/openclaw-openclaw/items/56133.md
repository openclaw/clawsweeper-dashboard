## Summary

Keep open. The underlying transcript-cwd bug is still valid on current main, but this PR is not merge-ready because it preserves process-global cwd fallbacks, lacks inspectable after-fix real behavior proof, conflicts with current main, and overlaps with the open canonical-looking path in https://github.com/openclaw/openclaw/pull/50454 and https://github.com/openclaw/openclaw/issues/49523.

## What This Changes

The branch replaces some transcript header `cwd` writes with `resolveAgentWorkspaceDir`, threads config into `sessions.create`, and adds a focused sessions transcript regression test.

## Best Possible Solution

Land one current-main-compatible explicit-workspace fix that derives one effective agent id and workspace from existing runtime config for every create-if-missing transcript header, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows create-if-missing transcript headers still read `process.cwd()` in mirrored, sessions.create, and chat-created paths while the documented multi-agent contract expects per-agent workspaces/session history.

## Solution Assessment

No, not as submitted. The branch moves in the right direction for `sessions.create`, but it preserves `process.cwd()` fallbacks and reloads config inside other create-if-missing paths instead of using one effective agent id/workspace from existing runtime config.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Pass runtime config into mirrored cwd resolution:** `src/config/sessions/transcript.ts:79`
  - body: The PR calls `loadConfig()` inside header creation after delivery has already succeeded. If on-disk config is temporarily invalid or unreadable, mirroring can fail or retry even though callers already pass runtime config into `appendAssistantMessageToSessionTranscript()`; resolve from that existing config instead.
  - confidence: 0.82
- **[P2] Remove mirrored transcript process cwd fallback:** `src/config/sessions/transcript.ts:78-80`
  - body: When `agentId` is absent, this helper still writes `process.cwd()` into a newly created header, preserving the `/app` or cross-agent workspace case the PR is meant to remove. Derive one effective agent id/workspace before header creation and pass that value through.
  - confidence: 0.78
- **[P2] Remove chat-created process cwd fallback:** `src/gateway/server-methods/chat.ts:760-762`
  - body: `ensureTranscriptFile()` still falls back to `process.cwd()`, and `persistAbortedPartials()` calls `appendAssistantTranscriptMessage()` without an `agentId`. Abort-created transcripts can therefore still record the process-global cwd instead of the target workspace.
  - confidence: 0.86
