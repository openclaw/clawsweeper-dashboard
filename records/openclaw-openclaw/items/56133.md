## Summary

Keep open. Current main still writes process-global cwd into new transcript headers, while this PR remains incomplete, conflicting, and missing required real behavior proof.

## What This Changes

The PR changes three session transcript header creation paths to use agent workspace resolution instead of `process.cwd()` and adds a focused gateway regression test.

## Best Possible Solution

Land one current-main-compatible fix that derives one effective agent id and workspace from runtime config for every create-if-missing transcript header path, with focused regression coverage and redacted Docker/runtime proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a brand-new mirrored, sessions.create, or chat-created transcript while `process.cwd()` differs from the target agent workspace, and current main writes the process-global cwd into the header.

## Solution Assessment

No. The direction is right, but this branch still reloads config inside helpers, preserves `process.cwd()` fallbacks, and is conflicting against current main; the safer path is one consolidated current-main patch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Use runtime config for mirrored transcript cwd:** `src/config/sessions/transcript.ts:79`
  - body: Calling `loadConfig()` inside header creation makes transcript mirroring depend on disk config after delivery already succeeded, even though current callers pass runtime config into the append path. Thread that config or a resolved cwd into header creation so bookkeeping does not fail or resolve a different workspace because on-disk config changed.
  - confidence: 0.86
- **[P2] Remove the mirrored process.cwd fallback:** `src/config/sessions/transcript.ts:80`
  - body: When `agentId` is absent, this helper still writes `process.cwd()` into a newly created header, preserving the `/app` or cross-agent workspace case the PR is meant to remove. Resolve one effective agent id/workspace before header creation and pass that through.
  - confidence: 0.78
- **[P2] Pass an agent id into abort transcript creation:** `src/gateway/server-methods/chat.ts:762`
  - body: `persistAbortedPartials()` still calls `appendAssistantTranscriptMessage()` without an agent id, so abort-created transcripts can hit this fallback and record the process-global cwd. Resolve the target agent/workspace before calling `ensureTranscriptFile()`.
  - confidence: 0.84
