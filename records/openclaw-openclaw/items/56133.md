## Summary

Keep this PR open, but not merge-ready. Current main still has the reported transcript-header bug, and this PR is useful source work, but the patch still allows at least one create-if-missing transcript path to fall back to `process.cwd()` and it overlaps the open canonical bug/fix thread in #49523/#50454.

## What This Changes

The PR changes gateway and session transcript header creation to use agent workspace resolution instead of `process.cwd()` and adds a focused regression test for `sessions.create` transcript cwd handling.

## Best Possible Solution

Land one current-main-compatible fix that removes `process.cwd()` from all session transcript header creation paths, derives the target agent workspace from explicit config/session context, and covers `sessions.create`, mirrored delivery transcripts, chat-created transcripts, and abort-created missing transcripts with focused regression tests.

## Reproduction Assessment

Yes. Current-main source shows three transcript header creation helpers writing `cwd: process.cwd()`, and the PR body provides production Docker evidence where that becomes `/app` instead of the agent workspace.

## Solution Assessment

No, not as submitted. The direction is correct, but the chat helper should not keep a process-cwd fallback for create-if-missing paths that can derive the agent from the session key/config or require callers to pass it.

## Review Findings
