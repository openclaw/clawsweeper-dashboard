## Summary

Current main and the shipped v2026.4.29 release now implement the reported Discord explicit-target fix. `parseDiscordExplicitTarget` preserves `target.normalized`, regression coverage verifies `user:<id>` and mentions stay direct-message targets, and the shared cron/heartbeat delivery resolver uses that parsed value before outbound Discord normalization.

## What This Changes

Fix Discord cron and heartbeat delivery to explicit DM targets by preserving `user:<id>` prefixes instead of stripping them to bare numeric IDs.

## Best Possible Solution

Keep the shipped Discord plugin-level normalized-target implementation and treat any remaining open PR carrying the same one-field fix as superseded or ready for maintainer cleanup.

## Reproduction Assessment

Yes. The reported failure chain depended on `parseExplicitTarget({ raw: "user:<id>" })` returning a bare ID; current main has a focused regression test proving it now returns `user:<id>` with `chatType: "direct"`.

## Solution Assessment

Yes. Preserving `target.normalized` in the Discord plugin parser is the narrowest maintainable fix because it keeps explicit DM intent while leaving bare numeric IDs as channel targets.
