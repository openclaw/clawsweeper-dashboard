## What This Changes

The PR updates `scripts/docs-list.js` to parse quoted inline `read_when` strings, adds a docs-list regression test, and routes docs-list script edits to that test.

## Best Possible Solution

Land the focused parser and test-routing fix after normal maintainer review and CI, keeping `docs:list` as the lightweight source-doc metadata discovery path.

## Reproduction Assessment

Yes. On current main, `node scripts/docs-list.js | rg ... -A1` shows the affected pages without a `Read when:` line even though their frontmatter has quoted inline `read_when` values.

## Solution Assessment

Yes. The bug is localized to `extractMetadata`, and adding quoted-scalar parsing beside the existing inline-array path is narrower than replacing this lightweight script with a full YAML parser.
