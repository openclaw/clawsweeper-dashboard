Full review comments:

- **[P1] Restore the Nostr gateway bus contract:** `extensions/nostr/src/nostr-bus.ts:64`
  - body: This replacement exports `createNostrBus` instead of the `startNostrBus`/`NostrBusHandle` contract that `extensions/nostr/src/gateway.ts` still imports and uses for `sendDm`, `publishProfile`, and `getProfileState`. The Nostr plugin will fail to typecheck or load once this diff is built.
  - confidence: 0.96
- **[P2] Honor NO_PROXY for Discord env proxy fallback:** `extensions/discord/src/monitor/provider.ts:440-441`
  - body: Deriving an explicit Discord REST proxy from `HTTPS_PROXY` here bypasses target-aware `NO_PROXY` handling. In environments with `HTTPS_PROXY` plus `NO_PROXY=discord.com`, startup REST calls are forced through the proxy instead of direct routing; use target-aware proxy selection or keep this path explicit-config-only.
  - confidence: 0.88
- **[P2] Write LINE allowlist updates to the selected account:** `extensions/line/src/setup-surface.ts:159-163`
  - body: The new onboarding handler hardcodes `DEFAULT_ACCOUNT_ID`, so configuring a non-default LINE account writes the allowlist to the wrong account. The previous setup path used the active `accountId`; this should preserve that account boundary.
  - confidence: 0.86
- **[P2] Reject malformed Nostr pubkeys:** `extensions/nostr/src/nostr-bus.ts:23-35`
  - body: After failed `npub` decode, `normalizePubkey` now returns any trimmed lowercase string instead of requiring a 64-byte hex pubkey. Bus callers can pass malformed keys into profile lookup, encryption, and event signing paths, turning validation failures into later runtime failures.
  - confidence: 0.82
