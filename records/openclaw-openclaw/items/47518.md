- **[P1] Honor NO_PROXY before applying Discord env proxy fallback:** `extensions/discord/src/monitor/provider.ts:438`
  - body: The env fallback converts HTTPS_PROXY/HTTP_PROXY into an explicit Discord proxy without checking the target host against NO_PROXY.   - body: hasProxyEnvConfigured() is broader than the env-proxy dispatcher path and does not account for NO_PROXY on the current target. Gate the trusted_env_proxy opt-in with the target URL helper so HTTP(S), ALL_PROXY, and bypass cases match the dispatcher behavior.
  - confidence: 0.86

  - body: normalizePubkey now accepts malformed non-npub values by returning the lowercased input. Callers rely on this helper to reject invalid sender and allowlist IDs early, so malformed keys can now reach encryption, profile, and allowlist paths before failing unpredictably.
  - confidence: 0.92
