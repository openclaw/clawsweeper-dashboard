Full review comments:

- **[P1] Restore the Nostr bus gateway contract:** `extensions/nostr/src/nostr-bus.ts:64`
  - body: This replacement exports `createNostrBus`/`NostrBus`, but the PR head still imports `startNostrBus` and `NostrBusHandle` from this module in the Nostr channel path. The plugin will fail to typecheck or load until the existing bus contract is preserved or all callers are migrated in the same patch.
  - confidence: 0.96
- **[P2] Honor NO_PROXY for Discord env proxy fallback:** `extensions/discord/src/monitor/provider.ts:440-441`
  - body: Deriving `resolvedProxy` from `HTTPS_PROXY`/`HTTP_PROXY` here turns ambient env proxy settings into an explicit `ProxyAgent`, bypassing `NO_PROXY`/`no_proxy` exclusions. Environments that set a global proxy but exempt `discord.com` will now force startup REST calls through the proxy instead of direct routing.
  - confidence: 0.88
- **[P2] Write LINE allowlist updates to the selected account:** `extensions/line/src/setup-surface.ts:159-162`
  - body: The new onboarding handler hardcodes `DEFAULT_ACCOUNT_ID`, so configuring a non-default LINE account writes the initial allowlist to the wrong account. Multi-account setups can appear configured while enforcing allowlist rules on a different account than the one being edited.
  - confidence: 0.86
- **[P2] Reject malformed Nostr pubkeys:** `extensions/nostr/src/nostr-bus.ts:23-35`
  - body: After failed `npub` decoding, `normalizePubkey` now returns any trimmed lowercase string instead of requiring a valid 32-byte hex public key. Callers that rely on this function throwing for invalid IDs will let malformed keys reach profile, encryption, and relay paths as later runtime failures.
  - confidence: 0.84
- **[P2] Gate trusted env-proxy mode on HTTP(S) proxy vars:** `extensions/google/oauth.http.ts:14`
  - body: `hasProxyEnvConfigured()` is true for `ALL_PROXY`, but this PR's trusted mode path constructs `EnvHttpProxyAgent` without explicit ALL_PROXY-derived options. With only `ALL_PROXY` set, the claimed proxy fix still sends OAuth requests direct instead of through the proxy.
  - confidence: 0.8
