Land a narrow current-main fix for #46184 in the Gemini OAuth/SSRF dispatcher path using target-aware env-proxy behavior and verified dispatcher teardown, with focused tests and a changelog entry; split Discord, LINE, and Nostr changes into separate reviewed PRs.

  - body: This rewrite removes `startNostrBus` and `NostrBusHandle`, but the PR's `extensions/nostr/src/channel.ts` still imports and calls them. The Nostr plugin will fail to typecheck/load until the existing bus API is preserved or all callers are updated in the same diff.

  - confidence: 0.9
- **[P2] Gate Gemini OAuth proxy mode by target URL:** `extensions/google/oauth.http.ts:14`
  - body: `hasProxyEnvConfigured()` is broader than the EnvHttpProxyAgent path and does not account for `NO_PROXY` on the current target. Gate the trusted proxy opt-in with the target URL helper so HTTP(S), `ALL_PROXY`, and bypass behavior matches the dispatcher contract.
  - confidence: 0.88
- **[P1] Reinstate strict Nostr pubkey validation:** `extensions/nostr/src/nostr-bus.ts:35`
  - body: `normalizePubkey` now returns the lowercased input for malformed non-`npub` values. Callers rely on this helper to reject sender and allowlist IDs before crypto, so invalid keys can now propagate into send/decrypt/profile paths and fail later.
  - confidence: 0.92
- **[P2] Write LINE allowlist updates to the selected account:** `extensions/line/src/setup-surface.ts:159-162`

  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The PR changes SSRF-adjacent proxy routing and Nostr key validation, and the current diff has concrete security-sensitive regressions.

Concerns:

- **[medium] Discord env proxy fallback ignores NO_PROXY:** `extensions/discord/src/monitor/provider.ts:440`
  - body: The PR converts ambient proxy env vars into explicit Discord ProxyAgent usage, bypassing target-aware `NO_PROXY` exclusions and changing operator egress policy for Discord REST and gateway traffic.
  - confidence: 0.9
- **[medium] Gemini OAuth proxy opt-in is not target-aware:** `extensions/google/oauth.http.ts:14`
  - body: The PR gates trusted env-proxy mode on any proxy env var instead of the current URL, which can diverge from HTTP(S) proxy and `NO_PROXY` behavior in an SSRF-sensitive fetch path.
  - confidence: 0.88
- **[medium] Nostr key validation is weakened:** `extensions/nostr/src/nostr-bus.ts:35`

  - confidence: 0.9
