Full review comments:

- **[P1] Honor NO_PROXY before deriving Discord env proxies:** `extensions/discord/src/monitor/provider.ts:440-441`
  - body: This turns HTTPS_PROXY/HTTP_PROXY into an explicit Discord REST proxy before the target host is known. That explicit ProxyAgent path bypasses NO_PROXY, so deployments that exempt discord.com can still be forced through the proxy and fail or violate operator routing policy.
  - confidence: 0.9
- **[P2] Gate Gemini trusted proxy mode on the target URL:** `extensions/google/oauth.http.ts:14`
  - body: The OAuth helper enables trusted_env_proxy whenever any proxy env var exists, including ALL_PROXY-only or NO_PROXY-excluded targets. Use the target-aware helper so the OAuth fetch follows the guarded-fetch proxy contract for the URL being fetched.
  - confidence: 0.86
- **[P1] Reject malformed Nostr pubkeys before routing:** `extensions/nostr/src/nostr-bus.ts:35`
  - body: normalizePubkey now returns lowercased input for malformed non-npub values instead of throwing. Callers rely on this helper rejecting invalid sender, target, and allowlist IDs early, so bad keys can reach cryptographic and routing paths before failing unpredictably.
  - confidence: 0.92
- **[P2] Write LINE allowlist updates to the selected account:** `extensions/line/src/setup-surface.ts:161-162`
  - body: The setup allowlist handler hardcodes DEFAULT_ACCOUNT_ID, so configuring a named LINE account updates the root/default account instead of the account being edited. Preserve the active accountId for multi-account setup.
  - confidence: 0.84
