## Summary

Keep open: current main still searches external channel secret-contract sidecars only at the plugin root, while the Discord docs support the reported env-backed SecretRef token shape and an open linked PR targets this exact remaining gap.

## What This Changes

The issue reports that @openclaw/discord skips startup with an env-backed SecretRef token because the external channel secret-contract loader misses published dist/ sidecars.

## Best Possible Solution

Review and land or close the linked dist-sidecar PR, keeping the fix limited to external channel contract discovery and a dist-layout regression test.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a plugin record whose contract file exists only at rootDir/dist/secret-contract-api.js will make loadExternalChannelSecretContractFromRecord return undefined on current main. I did not run the test fixture because this sweep is read-only.

## Solution Assessment

Yes, the narrow maintainable fix is to make resolvePluginContractApiPath recognize compiled dist/ contract sidecars while preserving the existing boundary-file check and flat-layout compatibility.
