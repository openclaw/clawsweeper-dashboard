## Summary

Keep open: current main still misses the reported published-plugin layout, and the issue has an open closing PR plus two independent confirmations in the discussion.

## What This Changes

The issue reports that env-backed SecretRef Discord tokens in npm-installed `@openclaw/discord` remain unresolved because external channel secret-contract sidecars under `dist/` are not discovered.

## Best Possible Solution

Land a narrow resolver and regression-test change that discovers `dist/secret-contract-api.*` and `dist/contract-api.*` under the same plugin-root boundary, then close this issue through the linked PR.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: an external channel plugin whose `rootDir` contains only `dist/secret-contract-api.js` makes current `resolvePluginContractApiPath` return null, so the Discord token SecretRef is not collected; I did not live-run the gateway because this sweep is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to extend external contract discovery to the package-local `dist/` sidecar layout while preserving the existing plugin-root boundary checks and flat-layout compatibility.
