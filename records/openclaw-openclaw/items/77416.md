## Summary

Keep open: current main still only searches root-level external channel contract sidecars, while the Discord package build path emits top-level public surfaces under `dist/` and the docs support SecretRef token config. The issue is also paired with open closing PR #77421, so it should remain open until that PR lands or is superseded.

## What This Changes

Restore Discord startup for env-backed SecretRef tokens in npm-installed external channel plugins by loading `dist/` secret contract sidecars.

## Best Possible Solution

Land or supersede #77421 with generic `dist/` sidecar lookup and regression coverage, then close this issue after the fix is on main or shipped.

## Reproduction Assessment

Yes: on current main, a plugin record with only `<rootDir>/dist/secret-contract-api.js` is source-reproducible because `resolvePluginContractApiPath()` never checks `dist/`, so the external contract returns `undefined` and Discord SecretRef assignment collection is skipped. I did not execute tests because this was a read-only review.

## Solution Assessment

Yes: adding `dist/` sidecar discovery to the generic external channel contract loader, with a dist-layout regression test, is the narrow maintainable fix. The issue should stay open while #77421 is reviewed and landed or replaced.
