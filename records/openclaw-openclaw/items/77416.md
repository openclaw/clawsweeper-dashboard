## Summary

Keep open: current main still misses compiled `dist/` external channel secret-contract sidecars, and open PR #77421 already targets the exact documented Discord SecretRef regression with closing syntax.

## What This Changes

Discord env-backed SecretRef tokens can remain unresolved for npm-installed `@openclaw/discord` because external channel contract discovery checks only plugin-root sidecars instead of the compiled `dist/` layout.

## Best Possible Solution

Review and land or close #77421, keeping the fix limited to external channel contract discovery and a dist-layout regression test.

## Reproduction Assessment

Yes. A source-level reproduction is clear: an external plugin record whose only contract file is `rootDir/dist/secret-contract-api.js` will make current-main `loadExternalChannelSecretContractFromRecord` return `undefined`.

## Solution Assessment

Yes. The narrow maintainable fix is to teach `resolvePluginContractApiPath` about compiled `dist/` sidecars while preserving root-layout compatibility and boundary-file checks.
