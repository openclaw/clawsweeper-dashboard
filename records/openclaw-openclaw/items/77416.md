## Summary

Keep open: current main still only searches external channel secret-contract sidecars at the plugin root, while the documented Discord SecretRef config and the package runtime build path support the reported dist-sidecar layout. The linked PR #77421 is open and unmerged, and two additional commenters confirmed the Discord channel failure.

## What This Changes

The issue requests fixing Discord channel startup when an npm-published external channel plugin exposes its SecretRef contract only as a compiled dist sidecar.

## Best Possible Solution

Land #77421 or an equivalent narrow fix that searches dist/ contract sidecars inside the existing plugin-root boundary and adds a dist-layout regression test.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: an external Discord plugin record whose contract exists only at rootDir/dist/secret-contract-api.js will not load a contract, so the SecretRef assignment is skipped.

## Solution Assessment

Yes. Extending resolvePluginContractApiPath to include dist/ candidates while preserving root-boundary checks and flat-layout compatibility is the narrow maintainable fix.
