## What This Changes

Env-backed SecretRef tokens should resolve for npm-installed Discord channel plugins whose compiled secret-contract sidecar lives under `dist/`.

## Best Possible Solution

Land a narrow fix that discovers compiled `dist/` secret-contract sidecars for external channel plugins and adds a dist-layout regression test, with any adjacent entrypoint metadata handled in the same review if needed.

Yes. Source inspection gives a high-confidence path: a plugin record with only `<rootDir>/dist/secret-contract-api.js` makes the current resolver return null and skip runtime SecretRef assignments; I did not run live Discord or tests because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to extend external contract discovery to compiled `dist/` sidecars while preserving root-layout compatibility and boundary-file checks.
