## Summary

Keep open: this is a real installer bug fix, but the PR is not ready because moving Homebrew only behind the Node check can break macOS installs that still need Git, and the supplied proof is mock-only.

## What This Changes

The branch moves Homebrew setup inside the failed Node check in `scripts/install.sh` and adds installer-flow regression tests for supported-Node and missing/outdated-Node macOS paths.

## Best Possible Solution

Ensure Homebrew is skipped when Node and Git are already present, but still install or validate Homebrew before every macOS path that calls `brew install`, then update installer docs and provide redacted real macOS proof.

## Reproduction Assessment

Yes at source level: current `main` calls `install_homebrew` before `check_node`, and `install_homebrew` can exit before a supported Node on PATH is considered. I did not run a live macOS non-admin environment in this read-only review.

## Solution Assessment

No as written: moving Homebrew only behind a failed Node check is too narrow because macOS Git installation also depends on brew. The safer solution is to make Homebrew conditional on any brew-backed install path, then update docs and proof.
