## Summary

Keep open: the latest branch fixes the earlier macOS Git/Homebrew gap and updates the docs, but it still removes the Homebrew bootstrap that current admin macOS installs rely on when npm native build-tool repair needs `brew install cmake`. The supplied proof is still stubbed installer test output rather than a real macOS installer run.

## What This Changes

The PR moves macOS Homebrew setup behind dependency-specific Node/Git paths in `scripts/install.sh`, updates installer docs, and adds install.sh regression tests for supported Node, missing/outdated Node, and missing Git flows.

## Best Possible Solution

Keep the dependency-local Homebrew direction, but ensure every macOS path that actually uses brew, including native build-tool/cmake repair, bootstraps Homebrew before invoking brew and add real macOS installer proof before merge.

## Reproduction Assessment

Yes at source level: current `main()` invokes Homebrew before checking Node, and `install_homebrew()` can exit before a supported Node is accepted. I did not run a live non-admin macOS installer session in this read-only review.

## Solution Assessment

No as implemented: moving Homebrew behind actual dependency needs is the right shape, but the PR only covers Node and Git while leaving the macOS cmake/build-tool repair dependent on preexisting `brew`. The maintainable fix is to cover all brew-backed dependency paths and prove the real installer flow.
