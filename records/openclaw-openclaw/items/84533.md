## Best Possible Solution

Land the narrow installer ordering fix after the contributor adds redacted real macOS behavior proof and updates the installer flow docs to describe conditional Homebrew setup.

## Solution Assessment

Mostly yes: moving Homebrew behind a failed Node check matches the dependency boundary because macOS Homebrew is only needed for the `install_node` branch. The PR should also update the installer flow docs and add real behavior proof before merge.
