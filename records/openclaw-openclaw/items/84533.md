## Summary

Keep open: this PR is a focused fix for a real installer bug, but it is not merge-ready because it moves Homebrew only into the missing-Node branch while macOS Git installation still depends on `brew`. The latest proof update is still mock/test output rather than a real no-admin macOS installer run.

## What This Changes

The PR moves macOS Homebrew setup behind the Node.js version check in `scripts/install.sh` and adds installer call-order regression tests for supported/current Node and missing/outdated Node flows.

## Best Possible Solution

Adopt the broader conditional Homebrew shape: check Node first, call Homebrew only before macOS Node or Git installs when those checks fail, update installer docs, and add redacted macOS no-admin proof for the Node-and-Git-present path.

## Reproduction Assessment

Yes at source level: current main calls `install_homebrew` before `check_node`, and `install_homebrew` can exit for a no-admin macOS user without brew before a supported Node is considered. I did not run a live no-admin macOS environment.

## Solution Assessment

No for this PR as written: making Homebrew conditional is the right direction, but it must remain immediately before every macOS brew-backed consumer, including Git, and the docs/proof still need to match that behavior.
