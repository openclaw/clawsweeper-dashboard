## Summary

Keep open: the Windows performance goal is plausible, but this patch weakens path-boundary freshness and still needs inspectable Windows proof before merge.

## What This Changes

The PR adds process-wide realpath and existing-ancestor caches to infra path helpers, routes path-guard exports through them, and adds explicit-invalidation tests.

## Best Possible Solution

Keep the performance goal, but use per-operation/request-scoped or fs-safe-owned invalidation-aware caching, with stale symlink/ancestor regression tests and inspectable Windows profile proof.

## Reproduction Assessment

Yes for the PR-introduced defect: source inspection shows process-wide caches returning path answers before live filesystem checks, while runtime security callers do not invalidate them. The original Windows slowdown was not independently reproduced beyond the PR body's summarized profile numbers.

## Solution Assessment

No. A global cache at the OpenClaw re-export layer is not the safest fix for a security-sensitive path surface; scoped caching or an upstream fs-safe design with explicit invalidation is the maintainable path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep realpath checks tied to current filesystem state:** `src/infra/path-safety.ts:7`
  - body: `globalRealpathCache` is process-wide, but runtime plugin hook, skill, and installed-plugin boundary checks do not clear it when symlinks or plugin roots change. A path that was safe once can keep being treated as safe after the filesystem changes; the new tests only prove manual invalidation works.
  - confidence: 0.9
- **[P1] Do not cache ancestor resolution without runtime invalidation:** `src/infra/boundary-path.ts:6`
  - body: The new ancestor cache stores `targetPath -> resolved ancestor` indefinitely, while sandbox host path normalization depends on this helper reflecting the current existing ancestor. Creating directories or swapping a symlinked ancestor later can leave sandbox bind decisions using stale canonical paths.
  - confidence: 0.88
