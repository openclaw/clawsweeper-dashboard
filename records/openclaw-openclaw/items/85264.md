## Summary

Keep open: the Windows lstat bottleneck is plausible and the linked issue remains open, but this branch is not merge-ready because the optimization puts security-sensitive path checks behind process-wide staleable caches.

## What This Changes

This PR wraps OpenClaw's fs-safe realpath and existing-ancestor helpers with module-level caches, reroutes path guards through the wrapper, and adds invalidation tests.

## Best Possible Solution

Move the optimization into a scoped or invalidation-aware path boundary, preserve live fs-safe freshness for security checks, and add stale symlink/ancestor regression coverage plus inspectable Windows proof.

## Reproduction Assessment

Yes for the PR-introduced cache defect: source inspection shows module-level Maps returning before fresh fs-safe realpath and existing-ancestor reads. The original Windows slowdown was not independently reproduced because the PR does not attach inspectable profile artifacts.

## Solution Assessment

No. The current approach caches above fs-safe with process-wide lifetime; the safer solution is scoped or invalidation-aware caching with stale symlink and ancestor tests plus inspectable Windows proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Keep realpath checks tied to live filesystem:** `src/infra/path-safety.ts:13-17`
  - body: This process-wide cache lets `safeRealpathSync` return a stale symlink target before fs-safe rechecks the filesystem. The same helper feeds `isPathInsideWithRealpath` call sites for plugin hooks, skills, installed plugin files, and hook installs, and the added invalidator is only exercised by tests, so path-boundary decisions can stay stale until restart.
  - confidence: 0.93
- **[P1] Do not cache ancestor resolution across filesystem mutations:** `src/infra/boundary-path.ts:9-14`
  - body: Caching `resolvePathViaExistingAncestorSync(targetPath)` by input path makes missing-leaf resolution ignore later changes to the deepest existing ancestor or parent symlink. `resolveSandboxHostPathViaExistingAncestor` depends on live ancestor resolution for sandbox host paths, and no production caller clears this cache.
  - confidence: 0.89
