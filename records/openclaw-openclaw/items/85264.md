## Summary

Keep open. The Windows performance problem is plausible and linked to https://github.com/openclaw/openclaw/issues/85262, but this patch should not merge as written because it turns caller-scoped realpath/root canonicalization into process-global caches used by security-sensitive path boundary checks.

## What This Changes

The PR adds module-level realpath and ancestor-path result caches in `src/infra/path-safety.ts`, `src/infra/path-guards.ts`, and `src/infra/boundary-path.ts` to reduce repeated Windows `lstat` work.

## Best Possible Solution

Keep the performance goal, but implement it with bounded operation-scoped or explicitly invalidated caching, stale-symlink and ancestor-change regression tests, and redacted Windows profile proof before merge.

## Reproduction Assessment

Yes for the review blocker: source inspection shows the PR returns process-global cached realpaths and ancestor canonical paths before checking the filesystem again. I did not reproduce the Windows performance profile itself in this read-only review.

## Solution Assessment

No. The performance goal is reasonable, but using process-global caches in security-sensitive boundary checks is not the narrowest maintainable fix; cache scope or invalidation needs to be part of the design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep realpath caching scoped to a single boundary check:** `src/infra/path-safety.ts:7-14`
  - body: This module cache is reused by security checks such as plugin hooks and skills. If a symlinked path is first resolved inside the root and later retargeted outside it, `isPathInsideWithRealpath` will return the stale inside realpath without touching the filesystem, so an escape can pass. Keep this cache operation-scoped or add a real invalidation contract before using it for authorization.
  - confidence: 0.9
- **[P1] Do not memoize ancestor canonicalization globally:** `src/infra/boundary-path.ts:5-12`
  - body: This result depends on which ancestor currently exists and where it resolves. A deleted/recreated directory or retargeted parent symlink can leave `resolvePathViaExistingAncestorSync` returning the old canonical path for sandbox/root boundary code, so it has the same stale-boundary failure mode.
  - confidence: 0.86
- **[P3] Add braces to the new inline if returns:** `src/infra/boundary-path.ts:9`
  - body: CI `check-lint` fails with three `Expected { after 'if' condition` errors from the new one-line `if` statements, so the branch is not mergeable until those statements follow the repo lint style.
  - confidence: 0.83

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete security-boundary regression risk by caching realpath and root canonicalization results process-wide.

Concerns:

- **[high] Global realpath cache can stale containment checks:** `src/infra/path-safety.ts:7`
  - body: The new `globalRealpathCache` can return an old inside-root canonical path after a symlink or directory is replaced, bypassing the fresh realpath check that plugin hook and skill boundary checks depend on.
  - confidence: 0.9
