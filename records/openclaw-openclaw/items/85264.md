## Summary

Keep open, but not merge-ready: latest head still adds process-wide caches to security-sensitive path-boundary helpers without runtime invalidation, and the Windows performance proof is summarized rather than inspectable.

## What This Changes

This PR adds global realpath and existing-ancestor caches in infra path helpers, reroutes path-guard exports through them, and adds cache-invalidation tests.

## Best Possible Solution

Keep the performance goal, but make caching per-operation or explicitly invalidation-aware at the fs-safe boundary, then add stale symlink/ancestor regression coverage and inspectable Windows profile proof.

## Reproduction Assessment

Yes for the PR-introduced defect: source inspection shows process-wide caches returning before fresh fs-safe checks, while current runtime callers do not call the invalidators. The original Windows slowdown was not independently reproduced from an inspectable profile artifact.

## Solution Assessment

No. The narrowest maintainable fix is scoped or invalidation-aware caching that preserves fs-safe freshness, not unconditional process-global path-boundary caches.

Overall confidence: 0.9

Full review comments:

- **[P1] Keep realpath checks tied to the live filesystem:** `src/infra/path-safety.ts:6`
  - body: The new module-level `globalRealpathCache` is shared by plugin hook/skill and installed-plugin boundary checks, but those runtime callers never clear it. If a symlink or plugin root is first resolved inside the boundary and later swapped outside, `isPathInsideWithRealpath` can reuse the stale inside realpath and allow a path the fresh fs-safe call would reject.
  - confidence: 0.91
- **[P1] Do not cache ancestor resolution without runtime invalidation:** `src/infra/boundary-path.ts:5`
  - body: This cache keys only by the requested path and is also used by sandbox host path canonicalization for missing leaves. After an existing ancestor or symlink changes in a long-running process, callers that do not call `invalidateAncestorPathCache` can keep using the old canonical target.
  - confidence: 0.87
