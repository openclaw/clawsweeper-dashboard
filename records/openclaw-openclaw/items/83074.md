## Summary

Keep open. This is a fresh feature/API design request: current main has SKILL.md/frontmatter skills, plugin-shipped skills, and compatible bundle mapping for some Claude/Codex/Cursor components, but it does not define the requested native skill-level `manifest.yaml` multi-component contract.

## What This Changes

Request to extend OpenClaw skills with an optional `manifest.yaml` and multi-component skill folders for templates, agents, references, hooks, output styles, and slash commands.

## Best Possible Solution

Design a versioned skill/bundle component contract that reuses current plugin and compatible-bundle surfaces where possible, and add core APIs only for component kinds that cannot be expressed externally.

## Reproduction Assessment

Not applicable: this is a feature/API design request, not a broken existing behavior report. Static inspection of current main confirms the current surface is SKILL.md/frontmatter plus plugin and bundle compatibility, not the proposed native `manifest.yaml` skill component model.

## Solution Assessment

Unclear whether the exact proposed `manifest.yaml` shape is the best final contract. The maintainable path is explicit product/API design that decides what belongs in skill metadata, compatible bundles, native plugins, and ClawHub publishing.
