## Summary

Keep this PR open, but it is not merge-ready. Current main still lacks a sanitizer for leading `to=<tool> ... {json}` leaks, while the branch patches the old inline `errors.ts` implementation and its broad matcher can truncate legitimate user-facing prose.

## What This Changes

The PR adds a leading `to=<tool> ... {json}` stripping pass for user-facing assistant text and two sanitizer regression tests.

## Best Possible Solution

Port a stricter prefix sanitizer into `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts`, preserve prose and code examples, add focused regression tests, and add the required single-line changelog entry.

## Reproduction Assessment

Yes, by source inspection. A final assistant string like `to=cron ... {"action":"create"}\nSet.` flows through `sanitizeAssistantVisibleText` and `sanitizeUserFacingText`, and current main has no stage that strips that leading prefix shape.

## Solution Assessment

No, not as submitted. The direction is useful, but the patch should move to the active split sanitizer module and use a stricter leaked-transcript shape before removing user-facing text.
