## Summary

Keep this PR open. Current main still does not implement the proposed leading `to=<tool> ... {json}` user-facing strip pass or its regression coverage, while the submitted branch is stale against the split sanitizer module and has an over-broad matcher that should be tightened before merge.

## What This Changes

The PR adds a `sanitizeUserFacingText` prefix-strip helper and tests intended to remove leaked leading `to=... {json}` tool-call text from user-facing agent replies.

## Best Possible Solution

Add a conservative defensive sanitizer in the current `sanitize-user-facing-text.ts` path, with tests for the Discord `to=cron ... {json}` leak and leading-prose false positives, while leaving the already-fixed Discord self-message ingestion behavior and cron execution path separate.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence static reproduction: a final assistant string like `to=cron ... {"action":"create"}\nSet.` flows through `sanitizeAssistantVisibleText` and `sanitizeUserFacingText` on current main without a matching strip stage.

## Solution Assessment

No, not as submitted. The direction is useful, but the patch should be ported to the split sanitizer module and the prefix detection should be constrained to a stricter leaked-transcript shape before removing user-facing text.
