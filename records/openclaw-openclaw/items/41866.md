## Summary

Keep open: this is a plausible focused bug-fix PR, but current main still lacks the exact bare leading `to=cron ... {json}` sanitizer and the submitted branch is not correct against the current split sanitizer boundary.

## What This Changes

The PR adds a `sanitizeUserFacingText` prefix-strip helper and regression tests for leading `to=<tool> ... {json}` text in user-facing agent replies.

## Best Possible Solution

Port a narrow transcript-shaped prefix sanitizer into `src/agents/pi-embedded-helpers/sanitize-user-facing-text.ts`, add regressions for leaked prefixes and literal prose examples, and leave broader outbound envelope stripping coordinated with https://github.com/openclaw/openclaw/issues/44905.

## Reproduction Assessment

Yes, source-reproducible. Current main routes visible assistant text through `sanitizeUserFacingText`, and the active sanitizer has no handler for a leading bare `to=cron ... {"action":"create"}` prefix shape.

## Solution Assessment

No, not as submitted. The fix needs to move to the active split sanitizer module and use a stricter leaked-transcript detector before deleting user-facing text.
