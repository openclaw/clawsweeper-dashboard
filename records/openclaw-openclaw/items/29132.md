## Summary

Current main still lacks a public same-turn tool-result transform, and an open PR already proposes the central API; keep the issue open while maintainers decide the security-sensitive public contract and review that implementation.

## What This Changes

Add a synchronous plugin hook that can sanitize tool results before same-turn model consumption.

## Best Possible Solution

Land one narrow typed hook at the canonical model-visible tool-result boundary after agreeing on trust policy, supported content shapes, composition, failure behavior, and persistence semantics.

## Reproduction Assessment

Yes. Source inspection shows current main emits the result before the observational post-call hook, while the mutable plugin hook is attached to transcript persistence, leaving no public transform at the requested same-turn boundary.

## Solution Assessment

Yes directionally: a public hook at the existing tool-result emission seam is more maintainable than per-tool wrappers, but the exact scope and security/failure contract require maintainer judgment.
