## Summary

Keep open. Current main still lacks a public pre-model tool-result sanitization hook for external plugins; the existing same-turn result middleware has the requested timing but is intentionally bundled-only, and the related open PRs still need maintainer API/security review before this can be considered implemented.

## What This Changes

The issue requests a synchronous plugin hook that runs after tool execution and before model visibility so plugin authors can deterministically sanitize tool results before persistence.

## Best Possible Solution

Keep this open until maintainers either land a documented, trust-gated public post-execution/pre-model tool-result mutation hook or explicitly decide that same-turn model-visible rewriting must remain bundled-only. The design should coordinate ordering with `before_tool_call`, internal middleware, `after_tool_call`, `tool_result_persist`, and `before_message_write`.

## Reproduction Assessment

Not applicable as a runtime bug report, but the API gap is high-confidence from current-main source inspection: the requested hook names are absent and external plugins are explicitly rejected from the only existing pre-model tool-result middleware seam.

## Solution Assessment

Unclear. A public hook is a plausible solution, but the best maintainable version needs maintainer review of trust policy and whether the first public contract should be synchronous/text-only, async, or a different guarded middleware API.
