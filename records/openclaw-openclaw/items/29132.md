## Summary

Close as implemented on current main: the exact hook name was not added, but the central need is covered by the supported `api.registerAgentToolResultMiddleware(...)` path for bundled and explicitly enabled installed plugins, and that middleware runs after tool execution before the runtime feeds the result back to the model.

## What This Changes

The issue requests a synchronous Plugin SDK hook that can sanitize or replace tool results after tool execution and before any model step consumes them.

## Best Possible Solution

Keep the current trusted middleware contract as the supported pre-model tool-result rewrite path and close the narrower hook request as implemented on main.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection shows current main now has the requested pre-model rewrite capability through trusted tool-result middleware, while v2026.6.5 still shows the older gap.

## Solution Assessment

Yes. The current middleware contract is a narrower maintainable solution than adding another hook name because it is runtime-neutral, manifest-gated, explicitly enabled for installed plugins, and already wired before model continuation.
