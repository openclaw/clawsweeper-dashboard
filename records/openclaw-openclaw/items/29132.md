## Summary

Keep open: current main and the latest release still lack a public pre-model tool-result sanitization hook, and the existing awaited pre-model rewrite seam remains bundled-only. The issue discussion, including the production workaround comment, supports this as a real Plugin SDK gap, but exposing model-visible tool-result rewriting is an API and security-boundary decision rather than conservative cleanup.

## What This Changes

The issue requests a public synchronous Plugin SDK hook that can sanitize or replace tool results after tool execution and before model visibility, with documented ordering and tests.

## Best Possible Solution

Keep this issue as the canonical Plugin SDK/security design thread until maintainers either land an approved public pre-model tool-result contract or explicitly document that model-visible rewriting remains bundled-only.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new Plugin SDK feature request. Source inspection gives a high-confidence feature-gap check: current main and v2026.5.12 lack the public hook names while bundled middleware owns the trusted pre-model path.

## Solution Assessment

Unclear. A synchronous pre-model transform hook fits the deterministic same-turn use case, but the public API shape needs maintainer decisions on trust, ordering, failure behavior, scope, and audit semantics before implementation.
