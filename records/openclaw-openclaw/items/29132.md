## Summary

Keep open: current main still lacks a public pre-model tool-result sanitization hook, and the related implementation work is unmerged plus security-sensitive rather than shipped.

## What This Changes

The issue requests a synchronous `tool_result_before_model` plugin hook that transforms tool results after execution and before model consumption.

## Best Possible Solution

Use this issue as the canonical design thread until maintainers either land a reviewed Plugin SDK contract with explicit ordering, trust, failure, scope, audit/provenance, docs, and tests, or document that pre-model result rewriting remains bundled-only.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that the feature gap exists on current main because the public hook names are absent and the only pre-model rewrite seam remains bundled-only.

## Solution Assessment

Unclear. A synchronous pre-model hook matches the reported use case, but maintainers still need to choose the safe public API shape and trust boundary, especially alongside https://github.com/openclaw/openclaw/pull/64907 and https://github.com/openclaw/openclaw/issues/62123.
