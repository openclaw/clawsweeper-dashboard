## Summary

Keep open: current main has a pre-model tool-result middleware path, but it is explicitly limited to bundled plugins, while the request is for a supported plugin hook/API that external plugin authors can use for deterministic policy enforcement. The related open issue #62123 and open PR #72733 overlap with the security/sanitization space but do not supersede this narrower external pre-model tool-result contract.

## What This Changes

The issue requests a synchronous plugin hook that transforms tool results after execution and before any model step consumes them.

## Best Possible Solution

Design and document a supported pre-model tool-result transformation contract, or explicitly keep the capability bundled-only, with ordering/security tests across Pi and Codex runtime paths.

## Reproduction Assessment

Not applicable. This is a feature/API request, and source inspection shows current main has only a bundled-only pre-model middleware, not an external plugin hook that can be reproduced as a broken existing behavior.

## Solution Assessment

Unclear. A pre-model result transform matches the reported need, but maintainers should decide whether the safest solution is a public hook, a trusted-plugin-only extension of the existing middleware, or a consolidated sync/async contract with #62123.
