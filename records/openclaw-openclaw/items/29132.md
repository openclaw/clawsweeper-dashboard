## Summary

Keep open: current main still lacks the requested public `tool_result_before_model` hook, and the only existing pre-model tool-result rewrite seam remains bundled-only because it crosses a high-trust model-input boundary. The related implementation at https://github.com/openclaw/openclaw/pull/64907 is open, conflicting, and text-only, so it does not close the broader policy-enforcement request yet.

## What This Changes

Request a synchronous public plugin hook named `tool_result_before_model` that rewrites tool results after execution and before model-visible consumption, with docs and ordering tests.

## Best Possible Solution

Consolidate the related hook proposals into a reviewed plugin SDK contract, or land the narrow text-only hook as an explicit first step with docs, ordering tests, Pi/Codex coverage, and a clear trust policy.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence about the feature gap: current public hook names omit `tool_result_before_model`, while `api.registerAgentToolResultMiddleware(...)` is restricted to bundled plugins.

## Solution Assessment

Unclear. A pre-model transform hook matches the reported need, but the safest public API shape should be decided with https://github.com/openclaw/openclaw/pull/64907 and adjacent requests at https://github.com/openclaw/openclaw/issues/52849 and https://github.com/openclaw/openclaw/issues/62123 before exposing model-visible result mutation to external plugins.
