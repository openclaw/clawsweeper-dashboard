## Summary

Keep open. Current main still lacks a public pre-model tool-result sanitization hook for external plugins; the existing same-turn rewrite seam has the requested timing but remains bundled-only, and the related open work still needs maintainer API/security review.

## What This Changes

The issue requests a synchronous public plugin hook that can sanitize tool results after execution and before the model can consume them.

## Best Possible Solution

Keep this open until maintainers land a documented, trust-gated public post-execution/pre-model tool-result mutation contract, or explicitly decide that same-turn model-visible result rewriting must remain bundled-only.

## Reproduction Assessment

Not applicable as a runtime bug report. The API gap is high-confidence from source inspection: the requested hook names are absent, while the only same-turn pre-model result rewrite seam is blocked for external plugins.

## Solution Assessment

Unclear. A public hook is plausible, but the best maintainable fix needs maintainer API/security review of trust policy, ordering, sync/async behavior, failure handling, and whether the first contract should be text-only or full-result.
