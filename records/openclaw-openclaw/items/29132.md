## Summary

Keep open: current main still lacks a public pre-model tool-result sanitization hook, while the only awaited rewrite seam is intentionally bundled-only. The related production workaround and duplicate threads support a real Plugin SDK gap, but exposing model-visible result rewriting needs maintainer security and API design judgment.

## What This Changes

The issue requests a synchronous Plugin SDK hook that transforms tool results after execution and before any model step can consume them.

## Best Possible Solution

Use this issue as the canonical design thread, then ship a reviewed versioned Plugin SDK contract with explicit trust, ordering, bounds, failure, audit, Pi/Codex coverage, and docs, or document that pre-model result rewriting remains bundled-only.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence about the feature gap: current main lacks the requested public hook names, and the only awaited pre-model result rewrite seam remains bundled-only.

## Solution Assessment

Unclear. A synchronous pre-model transform hook matches the reported use case, but the safe public API shape needs maintainer security and Plugin SDK alignment with the related async and text-only proposals.
