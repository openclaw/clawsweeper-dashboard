## Summary

Keep open: current main and the latest release still do not expose a public pre-model tool-result transform hook, and the comparable pre-model middleware remains a bundled-plugin-only trust boundary. The related implementation path is still open at https://github.com/openclaw/openclaw/pull/64907 and needs Plugin SDK/security-boundary approval rather than cleanup closure.

## What This Changes

The issue requests a synchronous public `tool_result_before_model` plugin hook that can transform tool results after tool execution and before model visibility.

## Best Possible Solution

Keep this as the canonical synchronous pre-model tool-result hook design thread until maintainers either land an approved public contract through https://github.com/openclaw/openclaw/pull/64907 or decide to keep pre-model tool-result rewriting bundled-only, with async behavior tracked separately in https://github.com/openclaw/openclaw/issues/62123.

## Reproduction Assessment

Yes, for a source-level feature-gap check rather than a bug reproduction. Current main and v2026.5.20 lack the requested public hook names while docs, registry code, and tests keep the comparable pre-model tool-result middleware bundled-only.

## Solution Assessment

Unclear. A synchronous pre-model transform hook fits the deterministic same-turn use case, but maintainers still need to choose the public trust boundary, data shape, ordering, failure behavior, bounds, audit/provenance, and policy gating.
