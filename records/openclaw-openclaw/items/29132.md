## Summary

Keep open: current main still lacks a public pre-model tool-result sanitization hook, while the existing pre-model rewrite seam is intentionally restricted to bundled middleware. Related comments, duplicate threads, and the open text-only implementation PR show a real Plugin SDK gap, but exposing model-visible tool-output rewriting needs maintainer security and API design judgment.

## What This Changes

Request a new synchronous `tool_result_before_model` Plugin SDK hook that transforms tool results after execution and before model consumption, distinct from `tool_result_persist`.

## Best Possible Solution

Use this as the canonical design thread, then land a maintainer-approved versioned Plugin SDK contract with explicit trust, ordering, bounds, failure, audit, Pi/Codex coverage, and docs, or document that pre-model result rewriting remains bundled-only.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence about the feature gap: current main lacks the requested public hook names, and the only awaited pre-model result rewrite seam remains bundled-only.

## Solution Assessment

Unclear. A synchronous pre-model transform hook matches the reported use case, but the safe public API shape needs maintainer security and Plugin SDK alignment with the related async, text-only, and guardrail proposals.
