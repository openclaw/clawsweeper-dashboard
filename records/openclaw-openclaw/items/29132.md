## Summary

Keep open: current main still lacks the requested public pre-model tool-result hook, and the only implemented pre-model rewrite seam remains bundled-plugin-only. The related implementation at https://github.com/openclaw/openclaw/pull/64907 is open, dirty, and intentionally text-only, so it does not close this broader security-policy request yet.

## What This Changes

The issue requests a synchronous public plugin hook that can sanitize tool results after execution and before the model can consume them.

## Best Possible Solution

Consolidate the overlapping hook proposals into a reviewed plugin SDK contract with explicit scope, ordering, failure behavior, trust policy, docs, and Pi/Codex ordering tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a missing public feature seam. Source inspection gives high confidence that current main has only bundled-only middleware for this pre-model timing point.

## Solution Assessment

Unclear. A pre-model transform hook matches the requested capability, but the safest public contract should be decided alongside https://github.com/openclaw/openclaw/pull/64907 and the adjacent async/blocking proposals before exposing model-visible result mutation to external plugins.
