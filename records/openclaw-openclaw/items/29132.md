## Summary

Close as implemented: current main covers the requested deterministic pre-model tool-result rewrite boundary through manifest-gated `api.registerAgentToolResultMiddleware(...)`, even though the exact proposed hook name was not added.

## What This Changes

Add a plugin extension point that can transform tool results after execution and before same-turn model consumption.

## Best Possible Solution

Keep the manifest-gated agent tool-result middleware as the supported pre-model rewrite boundary; any broader async or separate hook semantics should stay in the separate design thread at https://github.com/openclaw/openclaw/issues/62123.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high-confidence current-main behavior: the supported middleware path can rewrite tool results before they are returned to the model loop.

## Solution Assessment

Yes. The current manifest-gated middleware is a narrower security-conscious solution than adding an unrestricted parallel hook name, because it reuses the trusted runtime contract and explicit plugin enablement gates.
