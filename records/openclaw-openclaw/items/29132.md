## Summary

Keep open: current main and the latest release still lack a public pre-model tool-result sanitization hook, while the existing awaited rewrite seam is explicitly bundled-only. The related open implementation candidate is unmerged, conflicting, and narrower than the security-policy boundary requested here, so this still needs Plugin SDK and security-boundary judgment.

## What This Changes

Add a synchronous public plugin hook that can transform tool results after execution and before same-turn model visibility, with docs and ordering tests.

## Best Possible Solution

Keep this as the canonical synchronous pre-model design thread, then land a maintainer-approved Plugin SDK contract or document that model-visible result rewriting remains bundled-only.

Not applicable as a bug reproduction; this is a new Plugin SDK feature request. Source inspection gives a high-confidence feature-gap check: current main and `v2026.5.12` lack the public hook names, while bundled middleware already owns the trusted pre-model path.

## Solution Assessment

Unclear. A synchronous pre-model transform hook fits the deterministic same-turn use case, but the safe public API shape needs maintainer decisions on trust, ordering, failure behavior, bounds, and audit semantics.
