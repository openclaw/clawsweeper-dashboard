## Summary

Keep open: current main has a trusted pre-model tool-result middleware path, but it is explicitly bundled-only and does not provide the requested public plugin hook for external deterministic sanitization.

## What This Changes

Add a synchronous plugin hook that lets plugins transform tool results after execution but before the model can consume them.

## Best Possible Solution

Design and document a supported pre-model tool-result transformation contract, or explicitly keep the capability bundled-only, with ordering and security tests across Pi and Codex runtimes.

## Reproduction Assessment

Not applicable. This is a feature/API request, and source inspection shows current main has only bundled-only pre-model middleware rather than a broken public hook with a failing runtime path.

## Solution Assessment

Unclear. A pre-model result transform matches the stated need, but maintainers should decide whether the safest solution is a public hook, a trusted-plugin-only extension, or a contract aligned with #62123.
