## Summary

Keep open: current main still lacks the requested public `tool_result_before_model` hook, and the existing awaited pre-model tool-result middleware remains a bundled-plugin-only trust boundary. The open related implementation at https://github.com/openclaw/openclaw/pull/64907 is promising but unmerged and narrower than the full security-policy use case, so it should not close this issue yet.

## What This Changes

The issue requests a synchronous public plugin hook that sanitizes tool results after execution and before any model-visible consumption in the same run.

## Best Possible Solution

Consolidate the related hook proposals into a reviewed plugin SDK contract, or land the narrow text-only hook from https://github.com/openclaw/openclaw/pull/64907 as an explicit first step with docs, ordering tests, Pi/Codex coverage, and a clear trust policy.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence about the feature gap: current public hook names omit `tool_result_before_model`, while `api.registerAgentToolResultMiddleware` is restricted to bundled plugins.

## Solution Assessment

Unclear. A pre-model transform hook matches the need, but the safest API shape should be decided with the open text-only PR at https://github.com/openclaw/openclaw/pull/64907 and adjacent async/blocking proposals before exposing model-visible result mutation to external plugins.
