## Summary

Keep open: current main and the latest release still lack a public pre-model tool-result sanitization hook, while the only awaited pre-model rewrite seam remains bundled-only. This is a real Plugin SDK feature gap, but it needs maintainer API and security-boundary judgment rather than an automatic cleanup close.

## What This Changes

The issue requests a synchronous public Plugin SDK hook that can sanitize or replace tool results after execution and before the model sees them.

## Best Possible Solution

Keep this as the canonical synchronous pre-model design thread, then land a maintainer-approved Plugin SDK contract or document that model-visible tool-result rewriting remains bundled-only.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new Plugin SDK feature request. Source inspection gives a high-confidence feature-gap check: current main and v2026.5.12 lack the public hook names while bundled middleware already owns the trusted pre-model path.

## Solution Assessment

Unclear. A synchronous pre-model transform hook fits the deterministic same-turn use case, but the safe public API shape needs maintainer decisions on trust, ordering, failure behavior, bounds, and audit semantics.
