Keep open: current main and the latest release still lack a public pre-model tool-result sanitization hook, and the remaining work is a Plugin SDK/security-boundary decision rather than stale cleanup.

## What This Changes

The issue requests an additive synchronous `tool_result_before_model` hook that transforms tool results after execution but before same-turn model visibility, with `tool_result_persist` remaining transcript-specific.

## Best Possible Solution

Keep this as the canonical Plugin SDK/security design thread until maintainers either land an approved public pre-model tool-result contract or document that pre-model rewriting remains bundled-only, with async transcript/write behavior tracked separately at https://github.com/openclaw/openclaw/issues/62123.
