## Summary

Keep open. Current main still lacks a public post-execution, pre-model tool-result mutation hook; the closest implementation is the agent tool-result middleware path, and that remains intentionally bundled-only. This is real plugin SDK and security-boundary design work with active related PRs/discussion, not a conservative cleanup close.

## What This Changes

The issue requests a new synchronous `tool_result_before_model` plugin hook that rewrites tool results after tool execution and before any model step can consume them, with documented ordering and tests.

## Best Possible Solution

Land a documented, trust-gated public post-execution/pre-model tool-result mutation contract, or explicitly decide that same-turn model-visible result rewriting must remain bundled-only. The design should specify ordering relative to `before_tool_call`, internal middleware, `after_tool_call`, `tool_result_persist`, and `before_message_write`, plus failure/timeout behavior, SDK types, docs, and runtime tests.
