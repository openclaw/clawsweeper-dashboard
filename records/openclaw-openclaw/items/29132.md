## Label Justifications

- `P2`: The request is a meaningful plugin API/security improvement with limited blast radius and is already implemented.
- `impact:security`: The issue is about preventing untrusted tool output from reaching the model before a policy layer can sanitize it.
- `impact:session-state`: The requested boundary controls which tool-result content enters same-turn and later model/session context.

## Summary

Current main and the latest release already provide a supported pre-model tool-result rewrite seam through runtime tool-result middleware, so the central plugin/security boundary requested here is implemented even though it did not land under the proposed hook name.

## What This Changes

This issue requests a synchronous plugin hook that can sanitize tool results after execution and before the model can consume them in the same run.

## Best Possible Solution

Close this issue and point plugin authors to the shipped `api.registerAgentToolResultMiddleware(...)` contract; any stricter synchronous-only or differently named API should start as a new design discussion only if the shipped middleware is insufficient.

## Reproduction Assessment

Not applicable: this is a feature/API request rather than a broken existing behavior report. Source inspection confirms the requested class of pre-model tool-result transformation is now available through the shipped middleware API.

## Solution Assessment

Yes: the shipped runtime-neutral middleware seam is a better long-term solution than adding another hook name because it covers OpenClaw and Codex runtimes, supports async reducers, and is manifest/trust gated.
