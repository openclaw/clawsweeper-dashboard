## Summary

Keep open: current main still lacks a public pre-model tool-result sanitization hook, the only awaited pre-model result rewrite seam remains bundled-only, and this issue is already the canonical thread used to close a duplicate. The production workaround and validation offer in the discussion support the feature gap, while the open implementation candidate is still conflicting and narrower than the full policy boundary described here.

## What This Changes

Add a synchronous Plugin SDK hook that transforms tool results after tool execution and before any model-visible consumption.

## Best Possible Solution

Use this issue as the canonical design thread, then either ship a reviewed versioned Plugin SDK contract with explicit ordering, trust, failure, bounds, docs, and Pi/Codex coverage, or document that pre-model result rewriting remains bundled-only.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence about the feature gap: the public hook list omits the requested hook, while the only awaited pre-model result middleware is bundled-only.

## Solution Assessment

Unclear. A synchronous pre-model transform hook matches the reported need, but the safe public API shape needs maintainer security and SDK design alignment with the related open hook proposals.
