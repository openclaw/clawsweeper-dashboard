## Label Justifications

- `P2`: This was a meaningful security-oriented plugin API gap with bounded blast radius, now resolved on current main.
- `impact:security`: The requested boundary prevents untrusted tool output from reaching the model before deterministic policy enforcement.
- `impact:session-state`: Tool-result replacement controls the content entering same-turn and future agent context.

## Summary

Current main now provides the requested deterministic pre-model boundary through the broader, runtime-neutral `registerAgentToolResultMiddleware(...)` contract, including manifest-gated access for explicitly enabled installed plugins; the narrower open hook PR is no longer needed to solve this issue.

## What This Changes

Add a plugin-accessible hook that transforms tool results before same-turn model consumption for deterministic security policy enforcement.

## Best Possible Solution

Keep the manifest-declared, explicitly enabled agent tool-result middleware as the single canonical pre-model transformation API across OpenClaw and Codex, and close or redirect overlapping hook proposals to that contract.

## Reproduction Assessment

Yes. Source and release comparison show that v2026.6.5 retained the bundled-only restriction, while current main exposes the complete pre-model middleware to manifest-declared, explicitly enabled installed plugins.

## Solution Assessment

No. A separate `tool_result_before_model` hook would duplicate the broader current middleware, which already supports complete result replacement, async or sync handlers, and both OpenClaw and Codex runtimes.
