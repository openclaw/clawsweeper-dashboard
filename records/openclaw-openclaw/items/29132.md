## Summary

Keep open: current main has a pre-model tool-result middleware, but it is explicitly bundled-plugin-only, while this issue asks for a deterministic hook available to plugin authors for policy enforcement. The related open issue #62123 and PR #72733 do not supersede this because neither is a merged implementation of the requested external plugin contract.

## What This Changes

The issue requests a synchronous plugin hook that transforms tool results after execution and before any model step can consume them.

## Best Possible Solution

Design a supported, documented pre-model tool-result transformation contract, or explicitly keep the capability bundled-only, with ordering tests across Pi/Codex dynamic tools and docs distinguishing it from `tool_result_persist`.

## Reproduction Assessment

Not applicable. This is a feature/API request, and the source check confirms the missing part is external plugin access to a deterministic pre-model result transform rather than a failing existing behavior.

## Solution Assessment

Unclear. A pre-model transform seam matches the reported need, but maintainers should decide whether to generalize the current trusted bundled middleware, add a narrower external hook, or keep this boundary bundled-only.
