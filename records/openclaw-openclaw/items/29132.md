## Summary

Keep open: current main still lacks a public pre-model tool-result sanitization hook, and the existing awaited rewrite seam remains bundled-only because it can rewrite high-trust tool output before model visibility. The discussion includes a real production workaround and validation offer, and the related implementation at https://github.com/openclaw/openclaw/pull/64907 is still open, conflicting, and narrower than the browser-snapshot policy boundary described here.

## What This Changes

The issue requests a synchronous plugin hook that transforms tool results after execution and before same-turn model consumption.

## Best Possible Solution

Consolidate the related hook proposals into a reviewed Plugin SDK contract, or explicitly document that pre-model result rewriting remains bundled-only; if exposed, ship it with ordering, trust, failure-mode, bounds, docs, and Pi/Codex coverage.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence about the feature gap: current public hook names omit the requested hook, while the only awaited pre-model tool-result middleware is restricted to bundled plugins.

## Solution Assessment

Unclear. A pre-model transform hook matches the reported need, but the safe public API shape needs maintainer security and SDK design alignment with https://github.com/openclaw/openclaw/pull/64907, https://github.com/openclaw/openclaw/issues/52849, and https://github.com/openclaw/openclaw/issues/62123.
