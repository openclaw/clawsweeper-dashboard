## Summary

Keep open: current main and v2026.5.12 still lack a public `tool_result_before_model` or equivalent pre-model tool-result hook. The adjacent awaited rewrite path exists only as bundled tool-result middleware, and the active related implementation PR still needs maintainer Plugin SDK and security-boundary review before this issue can be closed.

## What This Changes

The issue requests a synchronous public Plugin SDK hook that sanitizes tool results after tool execution and before same-turn model consumption.

## Best Possible Solution

Keep this as the canonical Plugin SDK/security design thread until maintainers land an approved public pre-model tool-result contract, or document that model-visible rewriting remains bundled-only while tracking async follow-up separately at https://github.com/openclaw/openclaw/issues/62123.

Not applicable as a bug reproduction; this is a Plugin SDK feature request. Yes for a source-level feature-gap check: current main and v2026.5.12 lack the requested public hook names while bundled middleware owns the trusted pre-model path.

## Solution Assessment

Unclear. A synchronous pre-model transform hook fits the deterministic same-turn use case, but the public API shape needs maintainer decisions on trust, ordering, failure behavior, scope, bounds, and audit semantics before implementation.
