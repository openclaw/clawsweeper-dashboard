## Summary

Keep open: this is a valid, focused bug-fix PR. Current main still gates `stream: "thinking"` emission on `onReasoningStream`, while the Gateway `chat.send` path does not supply that callback; the PR fixes that path and adds targeted regression coverage, but public docs still need alignment before merge.

## What This Changes

The PR removes the `onReasoningStream` callback requirement for embedded Pi reasoning streaming, optional-chains the callback, and adds no-callback plus `thinkingLevel: "off"` regression tests.

## Best Possible Solution

Land the narrow callback-decoupling fix with its regression tests after updating the reasoning/WebChat docs; keep capability-gated delivery and Control UI rendering work in the related follow-up PRs.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction path: current main requires `onReasoningStream` for `streamReasoning`, while Gateway `chat.send` does not provide that callback. I did not run a live current-main Gateway failure in this read-only review.

## Solution Assessment

Yes for the runtime path: removing the callback dependency while preserving `canShowReasoning` is the narrow maintainable fix. The docs update is the remaining cleanup; broader capability-gated thinking-event routing belongs in the related API/product follow-up.
