## Summary

Keep open: the runtime patch is a narrow, source-backed fix with supplied real behavior proof, but the public reasoning/WebChat docs still describe `stream` as Telegram-only and would be stale after merge.

## What This Changes

The PR decouples embedded Pi reasoning-stream event emission from the optional channel callback and adds no-callback plus `thinkingLevel: "off"` regression tests.

## Best Possible Solution

Land the narrow callback-decoupling fix after updating the public reasoning and WebChat docs; leave capability-gated thinking subscriptions and UI rendering to their separate follow-up PRs.

## Reproduction Assessment

Yes. Source inspection shows current main requires `onReasoningStream`, while Gateway `chat.send` does not provide that callback, so callback-less WebSocket runs cannot emit `stream: "thinking"` events.

## Solution Assessment

Yes for the runtime path. Decoupling global event emission from the optional channel callback while preserving `canShowReasoning` is the narrow fix; the remaining safer step is docs alignment.
