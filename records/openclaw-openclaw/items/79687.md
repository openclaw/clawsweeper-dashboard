## What This Changes

The PR decouples embedded Pi reasoning streaming from the optional `onReasoningStream` callback and adds no-callback plus `thinkingLevel: "off"` regression coverage.

## Best Possible Solution

Land the narrow callback-decoupling fix with docs that describe Gateway/WebSocket live thinking events, while leaving UI rendering and capability-subscription changes to separate follow-up work.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: current main requires `onReasoningStream`, while Gateway `chat.send` does not provide that callback, so `stream: "thinking"` events cannot be emitted for that path.

## Solution Assessment

Yes for the runtime path. Decoupling event emission from the optional channel callback while preserving `canShowReasoning` is the narrow maintainable fix; the remaining safer step is docs alignment.
