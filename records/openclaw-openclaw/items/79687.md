## Summary

Keep this PR open: current main still has the callback-dependent reasoning stream guard, and the branch is a focused fix with relevant regression coverage and sufficient real behavior proof. The only clear remaining cleanup is stale public docs for `/reasoning stream`.

## What This Changes

The PR decouples `agent stream=thinking` emission from the optional `onReasoningStream` channel callback and adds no-callback plus `thinkingLevel: "off"` regression tests.

## Best Possible Solution

Land the narrow callback-decoupling fix with docs aligned, while leaving capability-gated thinking-event subscription and WebChat rendering to the broader follow-up PRs.

## Reproduction Assessment

Yes, at source level: current main requires `onReasoningStream`, while gateway `chat.send` does not provide that callback, so `emitReasoningStream` returns before broadcasting `stream: "thinking"`. I did not run a live current-main gateway failure in this read-only review.

## Solution Assessment

Yes for the runtime fix: keeping `canShowReasoning` while making the callback optional is the narrowest maintainable code change. The safer remaining follow-up is documentation alignment, not a different runtime design in this PR.
