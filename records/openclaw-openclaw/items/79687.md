## What This Changes

The PR decouples live reasoning event emission from the optional `onReasoningStream` callback and adds no-callback regression coverage for gateway/WebSocket-style clients.

## Best Possible Solution

Land one current-file implementation that preserves `canShowReasoning`, `silentExpected`, and optional channel callbacks, updates reasoning docs, and then close this older conflicting PR as superseded after the replacement merges or this branch is refreshed.

## Reproduction Assessment

Yes. Source inspection shows current main and `v2026.6.8` require `onReasoningStream` before broadcasting thinking events, while the gateway `chat.send` replyOptions path does not provide that callback.

## Solution Assessment

Yes for the central runtime shape, but no for this branch as the current landing vehicle. The maintainable fix is to make the channel callback optional while preserving visibility and silence gates, then land it on current embedded-agent files with docs and tests.
