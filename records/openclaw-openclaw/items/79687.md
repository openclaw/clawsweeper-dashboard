## Summary

Keep this PR open: the runtime fix is narrowly targeted and source-backed, but the public reasoning/WebChat docs still contradict the behavior this would ship.

## What This Changes

The PR decouples embedded Pi reasoning-stream emission from the optional `onReasoningStream` channel callback, makes the callback invocation null-safe, and adds no-callback plus `thinkingLevel: "off"` regression tests.

## Best Possible Solution

Land the narrow callback-decoupling fix after aligning the public reasoning/WebChat docs; keep capability-gated thinking subscriptions and UI rendering as separate follow-up work.

## Reproduction Assessment

Yes. Source inspection on current main shows the thinking stream producer is still callback-coupled, while the Gateway `chat.send` path does not provide `onReasoningStream`.

## Solution Assessment

Yes for the runtime path. Decoupling event emission from the optional channel callback while preserving `canShowReasoning` is the narrow fix; the remaining safer step is docs alignment.
