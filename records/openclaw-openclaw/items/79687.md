## What This Changes

The PR decouples live reasoning event emission from the optional `onReasoningStream` callback and adds no-callback plus visibility-gate regression tests in the old `pi` subscribe files.

## Best Possible Solution

Land one current-file, conflict-free implementation that changes `src/agents/embedded-agent-subscribe.ts`, preserves `canShowReasoning` and `silentExpected`, keeps the callback optional, and includes focused no-callback plus callback-preservation tests.

## Reproduction Assessment

Yes, source-reproducible: current main and `v2026.6.6` require `onReasoningStream` before `emitAgentEvent({ stream: "thinking" })`, while the `chat.send` path does not provide that callback. I did not run a live gateway repro in this read-only review.

## Solution Assessment

Yes for the runtime shape, but not as this branch's current landing vehicle. The maintainable fix is to make the existing callback optional while preserving `canShowReasoning` and `silentExpected`, then land it on the current `embedded-agent` files after conflict resolution.
