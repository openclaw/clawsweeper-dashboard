## Summary

Keep open: the runtime fix is source-backed and proof-supplied, but the public reasoning docs still say `stream` is Telegram-only, so the PR needs a narrow docs alignment before merge.

## What This Changes

The PR removes the `onReasoningStream` callback requirement from embedded Pi reasoning streaming, optional-chains the channel callback, and adds no-callback plus visibility-gate regression tests.

## Best Possible Solution

Land the narrow callback-decoupling fix with a matching public docs update, leaving capability-gated subscriptions and Control UI rendering to separate follow-up work.

## Reproduction Assessment

Yes at source level. Current `main` requires `onReasoningStream` for `streamReasoning`, while the Gateway `chat.send` path does not pass that callback, so callback-less WebSocket runs cannot emit live thinking events.

## Solution Assessment

Yes for the runtime path. Decoupling global thinking event emission from the optional channel callback while retaining `canShowReasoning` is the narrow fix; the remaining safer step is docs alignment.
