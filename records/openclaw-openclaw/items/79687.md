## Summary

Keep open: the runtime fix is valid and source-reproducible on current main, but the branch still needs docs alignment because public docs still describe `/reasoning stream` as Telegram-only.

## What This Changes

The PR decouples embedded Pi reasoning streaming from the optional `onReasoningStream` callback and adds no-callback plus `thinkingLevel: "off"` regression tests.

## Best Possible Solution

Land the narrow callback-decoupling fix with public reasoning/WebChat docs updated; keep capability-gated thinking subscriptions and Control UI rendering in separate work such as https://github.com/openclaw/openclaw/pull/54821 and https://github.com/openclaw/openclaw/pull/59086.

## Reproduction Assessment

Yes, source-level: current main requires `onReasoningStream` for `streamReasoning`, while gateway `chat.send` does not pass that callback. I did not run a live baseline gateway failure in this read-only review.

## Solution Assessment

Yes for the runtime path: removing the callback dependency while preserving `canShowReasoning` is the narrowest maintainable fix. The branch still needs docs alignment before it is ready to ship.
