## Summary

Keep open. Current main still registers and implements the bundled session-memory hook only for `command:new` and `command:reset`, while subagent completion goes through the typed `subagent_ended` plugin lifecycle path and never invokes the bundled memory writer. Related duplicate reports point back here, the direct fix PR #55974 is closed unmerged, and open PR #56672 is adjacent `session_end` plugin-hook work rather than a direct bundled session-memory fix.

## What This Changes

The issue requests that completed subagent sessions trigger the bundled session-memory hook so their transcripts are saved as dated Markdown files under the subagent workspace memory directory.

## Best Possible Solution

Add a narrow lifecycle bridge so run-mode subagent completion resolves the completed child session key, session entry, workspace, and transcript, then reuses the bundled session-memory writer to create exactly one dated Markdown artifact without changing existing `/new` or `/reset` behavior. Update the bundled hook metadata/docs and add regression coverage for the completed-child-session path.
