## Summary

Keep open. Current `main` and the latest release still register bundled session-memory only for `/new` and `/reset`, while subagent completion still emits `subagent_ended`; the related lifecycle PR at https://github.com/openclaw/openclaw/pull/56672 is still open and has not landed.

## What This Changes

The issue asks for completed subagent sessions to produce session-memory files like main-session `/new` and `/reset` do.

## Best Possible Solution

Review and either land or replace the lifecycle approach in https://github.com/openclaw/openclaw/pull/56672 so completed subagent transcripts can be captured without duplicating lifecycle policy inside session-memory.

## Reproduction Assessment

Yes from source inspection: the bundled hook is only registered for command events and the handler rejects non-command events, while subagent completion emits `subagent_ended`. I did not run a live subagent session in this read-only review.

## Solution Assessment

Unclear as a final implementation: the requested behavior is useful, but current docs and hook types make the fix a lifecycle/API direction decision rather than a documented broken contract.
