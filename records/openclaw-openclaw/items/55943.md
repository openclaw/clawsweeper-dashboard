## Summary

Keep open. Current main and v2026.5.20 still scope bundled session-memory to `/new` and `/reset`, while subagent completion emits only the typed plugin `subagent_ended` path; the related lifecycle PR is still open and not a landed canonical fix.

## What This Changes

The issue requests memory-file creation for completed subagent sessions, matching the existing main-session memory capture behavior after `/new` or `/reset`.

## Best Possible Solution

Choose one supported session-completion lifecycle contract, then persist the completed child transcript to the correct workspace memory directory exactly once while preserving existing `/new` and `/reset` behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: enable the bundled session-memory hook, complete a native `sessions_spawn` child, and the completion path emits `subagent_ended` while session-memory accepts only `command:new` and `command:reset`.

## Solution Assessment

Unclear. The requested outcome is reasonable, but maintainers should choose the durable lifecycle boundary before adding a memory-writer branch for child sessions.
