## Label Justifications

- `P2`: This is a normal-priority session-memory lifecycle gap with clear source evidence and limited blast radius.
- `impact:session-state`: The issue is about missing transcript-derived memory artifacts when child agent sessions finish.

## Summary

Keep open. Current main and the latest release still keep bundled session-memory scoped to `/new` and `/reset`, while native subagent completion emits only the typed plugin `subagent_ended` hook; related lifecycle PRs are open but have not landed a canonical fix.

## What This Changes

The issue requests that bundled session-memory save memory artifacts when subagent sessions finish, not only when the main session is reset or replaced.

## Best Possible Solution

Choose one lifecycle contract, then persist the child session transcript to the correct workspace exactly once on subagent completion while preserving existing `/new` and `/reset` behavior.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: enable the bundled session-memory hook, complete a native `sessions_spawn` child, and the completion path emits `subagent_ended` while session-memory accepts only `command:new` and `command:reset`. I did not run a live subagent session in this read-only review.

## Solution Assessment

Unclear. The requested outcome is reasonable, but maintainers should choose the supported lifecycle boundary before adding a one-off memory writer path.
