## Summary

Keep open. Current main and v2026.6.6 still limit bundled session-memory to /new and /reset command events, while completed subagents emit subagent_ended; the related lifecycle PR was closed unmerged and the broader session:end issue does not fully cover writing subagent memory files.

## What This Changes

Request session-memory capture when subagent sessions complete so their transcripts create memory/YYYY-MM-DD.md files.

## Best Possible Solution

Define the intended lifecycle boundary first, then wire session-memory to capture completed subagent transcripts through that canonical hook path with focused regression coverage.

## Reproduction Assessment

Yes, at source level. The bundled hook subscribes to and accepts only /new and /reset command events, while subagent completion emits subagent_ended, so the reported path cannot currently produce a session-memory file.

## Solution Assessment

Unclear as a final implementation. Directly teaching session-memory about subagent_ended is plausible, but the safer product direction is to settle the session lifecycle contract first and avoid one-off duplicate completion semantics.
