## Summary

Keep open. Current main and the latest release still keep bundled session-memory scoped to `/new` and `/reset`, while completed subagents emit separate plugin lifecycle hooks; related implementation work is either closed unmerged, adjacent, or still needs maintainer API direction.

## What This Changes

The issue requests that bundled session-memory also save memory files when subagent sessions finish.

## Best Possible Solution

Define one canonical completed-child-session lifecycle contract and wire session-memory through that path with focused coverage for child transcript lookup, workspace selection, and duplicate-write avoidance.

## Reproduction Assessment

Yes, at source level: session-memory subscribes to and accepts only `command:new`/`command:reset`, while subagent completion uses `subagent_ended` and plugin session lifecycle surfaces. I did not run a live subagent filesystem repro in this read-only review.

## Solution Assessment

Unclear. Directly teaching session-memory about `subagent_ended` is plausible, but the safer maintainable solution is to settle the lifecycle contract first and avoid parallel completion semantics.
