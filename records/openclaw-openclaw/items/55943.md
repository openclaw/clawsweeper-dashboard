## Summary

Keep open. Current main and v2026.5.12 still keep the bundled session-memory writer registered and filtered to `command:new` / `command:reset`, while native subagent completion only emits the plugin `subagent_ended` lifecycle hook; the related lifecycle PR is still open and does not supersede this specific session-memory gap.

## What This Changes

Request that the bundled session-memory hook save memory files when spawned subagent sessions finish, not only after main-session `/new` or `/reset` commands.

## Best Possible Solution

Land a maintainer-approved lifecycle bridge that persists a child session's transcript to the correct workspace exactly once on subagent completion, while preserving existing `/new` and `/reset` behavior.

## Reproduction Assessment

Yes. By source inspection, enable the bundled session-memory hook, complete a native `sessions_spawn` child, and the completion path only emits `subagent_ended` while the writer only accepts `command:new` and `command:reset`.

## Solution Assessment

Unclear. The requested direction is reasonable, but the best implementation should be chosen with the open lifecycle work rather than adding a competing one-off writer path.
