- `impact:session-state`: The missing hook leaves completed child-session transcript state out of memory persistence.

## Summary

Keep open. Current main and v2026.5.22 still keep bundled session-memory limited to `command:new` and `command:reset`, while native subagent completion emits `subagent_ended`; the related lifecycle PR is still open and not a landed fix.

## What This Changes

The issue asks for completed subagent sessions to generate session-memory files under `memory/` the way main-session `/new` and `/reset` currently do.

## Best Possible Solution

Establish one idempotent session-completion lifecycle contract, then have session-memory consume completed child-session transcripts exactly once while preserving existing `/new` and `/reset` behavior.

## Reproduction Assessment

Yes by source inspection: enable session-memory, complete a native `sessions_spawn` child, and the completion path emits `subagent_ended` while session-memory only handles `command:new` and `command:reset`. I did not run a live subagent scenario because this review is read-only.

## Solution Assessment

Unclear. The requested outcome is reasonable, but maintainers should choose whether the durable fix is typed `session_end`, an internal `session:end` event, or direct session-memory lifecycle wiring before implementation.
