## Summary

Keep open. Current main and latest release v2026.6.8 still keep bundled session-memory scoped to `/new` and `/reset`, while completed subagents emit the separate `subagent_ended` plugin hook; the related lifecycle PR is closed unmerged and the broader session-end work does not implement this specific memory-file behavior.

## What This Changes

The issue requests that bundled session-memory create `memory/YYYY-MM-DD.md` files when subagent sessions finish, not only when the main session is reset or replaced.

## Best Possible Solution

Define one canonical completed-child-session lifecycle contract, then wire session-memory through that path with focused coverage for transcript lookup, workspace selection, and deduplication.

## Reproduction Assessment

Yes, at source level: session-memory subscribes to and accepts only `command:new` and `command:reset`, while completed subagents emit `subagent_ended`. This is a source-visible feature gap rather than a failure of the current documented command-only hook contract.

## Solution Assessment

Unclear. Directly teaching session-memory about `subagent_ended` is plausible, but the safer solution is to settle the lifecycle contract first and avoid one-off duplicate completion semantics.
