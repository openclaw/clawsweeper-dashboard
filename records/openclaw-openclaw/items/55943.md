## Summary

Keep open. Current main still command-gates the bundled session-memory writer, and subagent completion only emits the plugin `subagent_ended` lifecycle path; extending documented command-only memory capture to subagent completion remains unresolved and overlaps the open #56672 lifecycle proposal.

## What This Changes

The issue requests that the bundled session-memory hook write memory artifacts when subagent sessions complete, not only when `/new` or `/reset` runs.

## Best Possible Solution

Settle the subagent/session-end lifecycle contract, then extend the bundled session-memory path to consume that lifecycle with focused tests and docs for child-session memory artifacts.

## Reproduction Assessment

Yes, at source level: enable session-memory, complete a native `sessions_spawn` subagent, and the current completion path reaches `subagent_ended` while the memory writer only accepts `/new` and `/reset` command events. I did not run the live scenario in this read-only review.

## Solution Assessment

Unclear as a patch because there is no merged implementation here. The safer direction is to decide whether #56672's `session_end` lifecycle is the intended contract, then bridge session-memory through that path rather than adding an unrelated one-off trigger.
