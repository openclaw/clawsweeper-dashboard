## Summary

Keep open. Current main still registers and implements the bundled session-memory hook only for `command:new` and `command:reset`, while subagent completion only emits the typed `subagent_ended` plugin lifecycle hook. The related UTC/local-date bug was fixed, but this subagent memory persistence gap remains; open PR #56672 is related lifecycle-hook work, not a merged fix for the bundled memory writer.

## What This Changes

The issue requests that completed subagent sessions write session-memory Markdown artifacts the same way main-session `/reset` and `/new` transitions do.

## Best Possible Solution

Add a narrow lifecycle bridge so a completed child session resolves its child session key, workspace, session entry, and transcript, then reuses the existing session-memory writer to create one dated Markdown artifact without changing existing `/new` or `/reset` behavior. Update hook metadata/docs and add focused regression coverage for subagent completion persistence.

## Reproduction Assessment

Yes. The static reproduction path is high-confidence: enable the bundled session-memory hook, complete a `sessions_spawn` subagent, and the current completion path emits only `subagent_ended` while the memory handler is registered for and gated to command events.

## Solution Assessment

Not applicable as a patch, but the requested direction is valid. The narrow maintainable fix is to bridge the existing subagent lifecycle to the bundled memory writer or an equivalent session-end event after #56672 lands, rather than redesigning the broader hook/plugin system.
