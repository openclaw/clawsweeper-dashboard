## Summary

Keep open. Current main still registers and implements the bundled session-memory hook only for `/new` and `/reset`, while subagent completion emits `subagent_ended` plugin lifecycle hooks and does not reach the memory writer.

## What This Changes

The issue requests that bundled session-memory persistence also run when subagent sessions finish.

## Best Possible Solution

Add a narrow lifecycle bridge so completed child sessions resolve their workspace, session entry, transcript, and session key, then reuse the existing session-memory writer to create one dated Markdown artifact without changing existing `/new` or `/reset` behavior.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes: enable the bundled session-memory hook, complete a `sessions_spawn` subagent, and inspect the child workspace memory directory; source inspection shows that path only emits `subagent_ended` while the memory writer is command-gated.

## Solution Assessment

Is this the best way to solve the issue? Not applicable as a patch, but the requested direction is valid. The narrow maintainable fix is to bridge subagent/session-end lifecycle data into the existing memory writer, ideally reusing any merged `session_end` plumbing from #56672.
