## Summary

Keep open. Current main and the latest release still keep bundled session-memory scoped to `/new` and `/reset`, while completed subagents emit `subagent_ended`; the related lifecycle PR is closed unmerged and the broader open session:end issue does not itself implement memory-file creation for subagent sessions.

## What This Changes

The issue asks for bundled session-memory capture to run when subagent sessions finish, not only when the main session is reset or replaced.

## Best Possible Solution

Define the lifecycle contract first, then wire session-memory through that canonical hook with focused coverage for child transcript lookup, workspace selection, and deduplication against reset/delete paths.

## Reproduction Assessment

Yes, at source level. The bundled hook subscribes to and accepts only `command:new` and `command:reset`, while completed subagents emit `subagent_ended`, so the reported path cannot currently produce a session-memory file.

## Solution Assessment

Unclear as a final implementation. Directly teaching session-memory about `subagent_ended` is plausible, but the safer solution is to settle the session lifecycle contract first and avoid one-off duplicate completion semantics.
