## Summary

Keep open. Current main and v2026.5.18 still keep bundled session-memory limited to command:new/command:reset, while native subagent completion only emits the plugin subagent_ended hook; the open lifecycle PRs are related but do not yet resolve this specific memory gap.

## What This Changes

Request session-memory persistence when a subagent session completes, so child-session transcripts create memory files like main sessions do on /new or /reset.

## Best Possible Solution

Choose one lifecycle contract, then persist the child session transcript to the correct workspace exactly once on subagent completion while preserving existing /new and /reset behavior.

## Reproduction Assessment

Yes. By source inspection, enable the bundled session-memory hook, complete a native sessions_spawn child, and the completion path only emits subagent_ended while session-memory accepts only command:new and command:reset.

## Solution Assessment

Unclear. The requested outcome is reasonable, but maintainers should choose the supported lifecycle boundary with the open session_end/internal-hook work rather than adding a competing one-off writer path.
