## Summary

Keep open. Current main still has a source-backed diagnostics gap: exec/supervisor internals retain signal, timeout, no-output-timeout, and termination-reason metadata, but foreground `exec` details and `process.poll` details do not expose enough of it for this Linux SIGKILL report.

## What This Changes

The issue asks OpenClaw to stop surfacing intermittent Linux broad-search exec SIGKILLs as abrupt, low-context failures and to provide a clear resource, timeout, or supervisor-cause explanation.

## Best Possible Solution

Expose existing `RunExit`, `ExecProcessOutcome`, and finished-session termination metadata in backward-compatible foreground `exec` and `process.poll` details, while leaving OOM attribution, signal escalation, and broad recursive-scan guardrails to separate focused work.

## Reproduction Assessment

Yes for the remaining diagnostics gap at source level: current main records termination metadata internally and then omits key fields from foreground exec and `process.poll` details. No deterministic live reproduction was established for the intermittent Linux broad-search SIGKILL trigger itself.

## Solution Assessment

Yes for the narrow first repair. Additive metadata propagation is the maintainable fix here; changing kill semantics, adding recursive-scan policy, or claiming OS-level OOM attribution should stay separate.
