## Summary

This issue should stay open: current main and the latest release still format signal exits as a bare `Command aborted by signal ...`, while the direct closing PR remains open and unmerged. The merged Linux OOM-score work explains a plausible cause and protects the gateway, but it explicitly leaves this user-facing SIGKILL diagnostic gap as separate work.

## What This Changes

The issue requests clear diagnostics or handling when Linux exec-tool broad `find`/`grep` discovery commands are intermittently killed with SIGKILL without matching host OOM log evidence.

## Best Possible Solution

Land or replace the focused Linux SIGKILL diagnostic fix so wrapper-active Linux signal exits surface actionable OOM-score guidance without changing process kill behavior.

## Solution Assessment
