  - body: This new source-wide call can pass a cross-file chunk list with cache hits and misses into `embedChunksWithBatch()`. ## Repair Work Prompt

For openclaw/openclaw PR https://github.com/openclaw/openclaw/pull/89138, repair the source-wide memory embedding batching fallback bug. In `extensions/memory-core/src/memory/manager-embedding-ops.ts`, `embedChunksWithBatch()` batches only missing chunks but its fallback embeds the full chunk list, so source-wide batching can map fallback embeddings to the wrong missing chunk when cached chunks precede misses. Fix the mapping by making fallback operate on `missingChunks` or by mapping full fallback results back through original indexes. Keep the source-wide opt-in behavior and avoid adding config/env flags. Add focused regression coverage in `extensions/memory-core/src/memory/index.test.ts` for a source-wide provider with cache hits before misses and a forced provider-batch fallback, asserting stored/searchable embeddings remain aligned. Do not edit `CHANGELOG.md`.
