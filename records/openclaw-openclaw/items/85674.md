## What This Changes

Requests a new `paths:` field in `SKILL.md` frontmatter so skills are withheld from `<available_skills>` until matching file paths are touched in the session.

## Best Possible Solution

Define a first-class conditional-skill contract in the skill metadata and prompt snapshot layer, including parsing, activation events, CLI/status output, docs, and tests, while preserving existing allowlist and `disable-model-invocation` behavior.

## Reproduction Assessment

Not applicable: this is a new skill-loading mode request, not a broken current behavior. Source inspection shows current prompt visibility is controlled by eligibility, allowlists, and `disable-model-invocation`, not file-path-triggered activation.

## Solution Assessment
