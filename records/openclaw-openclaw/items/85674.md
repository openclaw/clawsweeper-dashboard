## What This Changes

The issue requests an optional `paths` field in `SKILL.md` frontmatter so matching skills stay out of `<available_skills>` until a touched file path activates them for the session.

## Best Possible Solution

Define a first-class conditional-skill metadata contract covering parsing, activation events, prompt snapshot refresh, CLI/status output, docs, and tests while preserving existing allowlist and `disable-model-invocation` behavior.
