## Summary

Keep open. Current main and the latest release still contain the legacy `gh-issues` `/data/.clawdbot` paths, but this PR is not mergeable as-is because it duplicates a token fallback, drops env-aware config guidance, is conflicting, and lacks contributor real behavior proof.

## What This Changes

This PR replaces `/data/.clawdbot` references in the bundled `gh-issues` skill instructions with `~/.openclaw` or `$HOME/.openclaw` paths.

## Best Possible Solution

Land a corrected, rebased skill-docs patch that removes the legacy hardcoded paths, preserves env-aware config/state lookups, deletes the duplicate fallback block, and includes concise real behavior proof.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: current main and `v2026.5.12` still show the legacy `gh-issues` paths, and the PR head shows the duplicate fallback plus env-guidance regression.

## Solution Assessment

No, not as currently proposed. The cleanup direction is correct, but the maintainable fix should preserve `OPENCLAW_CONFIG_PATH` / `OPENCLAW_STATE_DIR` behavior and remove the duplicate fallback instead of adding more literal default-path reads.
