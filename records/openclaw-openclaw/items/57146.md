## Summary

Keep open. The `gh-issues` path cleanup is still needed on current main, but this PR currently leaves a duplicate token fallback, regresses env-aware config lookup guidance, and lacks contributor-supplied real behavior proof.

## What This Changes

The PR edits `skills/gh-issues/SKILL.md` to replace legacy `/data/.clawdbot` examples with `~/.openclaw` or `$HOME/.openclaw` paths in token, claim, cursor, and sub-agent snippets.

## Best Possible Solution

Land a corrected, rebased skill-docs patch that removes the legacy hardcoded paths, preserves env-aware config/state lookups, deletes the duplicate fallback block, and includes concise real behavior proof.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.5.12` finds the legacy `gh-issues` paths, and source inspection of the PR head shows the duplicate fallback plus env-guidance regression.

## Solution Assessment

No, not as currently proposed. The cleanup direction is right, but the maintainable fix should preserve `OPENCLAW_CONFIG_PATH` and `OPENCLAW_STATE_DIR` behavior while removing the duplicate fallback.
