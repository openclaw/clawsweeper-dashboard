## Summary

## What This Changes

## Best Possible Solution

Land a corrected, rebased skill-docs patch that removes the legacy hardcoded paths, preserves env-aware config/state lookups, deletes the duplicate fallback block, and includes concise real behavior proof.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence. `rg` on current main finds the legacy paths, and the live PR diff shows the duplicate fallback plus env-aware guidance regression on the proposed head.

## Solution Assessment

No, not as currently proposed. The cleanup direction is correct, but the branch should preserve `OPENCLAW_CONFIG_PATH` / `OPENCLAW_STATE_DIR` guidance and remove the duplicate fallback before it is the narrow maintainable fix.
