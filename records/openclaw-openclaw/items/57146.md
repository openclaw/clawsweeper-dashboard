## Summary

Keep open. Current main and the latest release still contain the reported legacy `gh-issues` `/data/.clawdbot` examples, but this PR head needs changes because it leaves a duplicate token fallback and regresses the env-aware config-path guidance now present on main.

## What This Changes

The PR changes the bundled `gh-issues` skill docs to replace legacy `/data/.clawdbot` examples with `~/.openclaw` and `$HOME/.openclaw` paths.

## Best Possible Solution

Land a rebased or replacement docs-only change that removes every `gh-issues` `/data/.clawdbot` example, preserves the existing env-aware config/state path pattern, and deletes the duplicate fallback instead of replacing it with another literal default-path read.

## Reproduction Assessment

Yes. Source inspection on current main reproduces the issue with `rg -n '/data/\.clawdbot' skills/gh-issues/SKILL.md`, and PR-head inspection shows the duplicate fallback at lines 101-104.

## Solution Assessment

No, not as currently proposed. The direction is right, but the branch should be rebased to preserve `OPENCLAW_CONFIG_PATH` / `OPENCLAW_STATE_DIR` behavior, remove the duplicate fallback, and include real after-fix proof before merge.
