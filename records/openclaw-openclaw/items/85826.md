## What This Changes

The issue reports long local vLLM model calls being classified as stalled around the diagnostics threshold, leading to takeover/session failure and lost channel replies, and asks for tunable thresholds plus graceful recovery.

## Best Possible Solution

Keep this as the canonical local-model stall/recovery policy item while maintainers review the narrow silent-model diagnostics fix, preserve the shipped stream-progress behavior, and route false-takeover cleanup through the owned-write/session-transcript migration path.

## Reproduction Assessment

## Solution Assessment

Unclear. Adding another timeout knob is not the best first fix because global diagnostics and provider timeouts already exist; the maintainable path is an explicit recovery policy for silent model calls plus narrow session-fence ownership fixes where takeover is the root cause.
