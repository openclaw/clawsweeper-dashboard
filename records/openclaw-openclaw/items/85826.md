## Summary

Keep open: the hard-coded/no-config part is contradicted by current source and shipped docs, but current main still allows silent long model calls to become stalled active-abort candidates; the remaining question is session-recovery policy for long local-provider turns, not cleanup closure.

## What This Changes

Request to prevent legitimate long local vLLM model calls from being killed by stalled-agent recovery, with configurable thresholds and graceful user-visible recovery.

## Best Possible Solution

Keep this as a session-recovery policy follow-up: document/use the existing diagnostics knobs today, then decide whether long silent local model calls need provider-aware progress, active-abort rules, or channel-visible still-working/fallback semantics.

## Reproduction Assessment

Yes at source level, but not as a live vLLM run. Current code and tests show active model calls can be classified as stalled and become active-abort candidates after the configured abort threshold; the exact takeover timing from the report was not reproduced.

## Solution Assessment

Unclear. Adding another timeout knob is not the best first fix because global diagnostics and provider timeout controls already exist; the maintainable path is a deliberate recovery policy for long silent local model calls.
