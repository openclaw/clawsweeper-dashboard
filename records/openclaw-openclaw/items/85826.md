## Summary

Keep open: the “no configuration knob” premise is contradicted by current and shipped source, but the remaining long silent model-call path can still become active-abort recovery and user-visible message loss, so this needs maintainer recovery-policy review rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to avoid killing legitimate long local vLLM model calls after stall detection and to provide configurable, graceful recovery behavior.

## Best Possible Solution

Keep this as a session-recovery policy follow-up: use the existing diagnostics and provider timeout knobs today, then decide whether long silent local model calls should be observed, surfaced as still working, or abort-drained under clearer provider-aware rules.

## Reproduction Assessment

Yes at source level, but not as a live vLLM run. Current source and tests show active model calls can be classified as stalled and become active-abort recovery candidates after diagnostics.stuckSessionAbortMs; the exact takeover timing from the report was not reproduced.

## Solution Assessment

Unclear. A new hard-coded or duplicate timeout knob is not the best first fix because diagnostics and provider timeout controls already exist; the maintainable path is a deliberate recovery policy for long silent local model calls.
