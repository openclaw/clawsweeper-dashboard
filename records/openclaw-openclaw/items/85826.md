## Summary

Keep open: the no-config premise is contradicted by current source and the reported release, but the source still shows long silent model calls can become stalled/abort-eligible and the reported takeover/message-loss path needs session-recovery policy review rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to stop failing long local vLLM turns by making stall handling tunable and recovering gracefully instead of losing replies.

## Best Possible Solution

Keep this as a session-recovery policy follow-up: use the existing global diagnostics and vLLM timeout knobs today, then decide whether long local model calls need provider-aware progress, abort, and still-working semantics.

## Reproduction Assessment

Yes at source level, but not as a live vLLM run. Current code and tests show active model calls can be classified as stalled after the diagnostics threshold and can become active-abort candidates after the abort threshold; the exact takeover timing from the report was not reproduced here.

## Solution Assessment

Unclear. A new hard-coded or duplicate timeout knob is not the best first fix because global diagnostics and provider timeout controls already exist; the maintainable path is a deliberate recovery policy for long silent local model calls.
