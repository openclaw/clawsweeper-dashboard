## Summary

Keep open: the no-config premise is already contradicted by current source and docs, but current main still allows long silent model calls to become stalled and active-abort eligible, so the remaining vLLM/session-recovery behavior needs maintainer policy review rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to avoid killing legitimate long local vLLM model calls and to provide configurable or graceful recovery behavior when a session is quiet for more than the default diagnostic window.

## Best Possible Solution

Keep this as a session-recovery policy issue: use existing diagnostics and provider timeout knobs today, then decide whether long local model calls need provider-aware progress, active-abort changes, or user-visible still-working behavior.

## Reproduction Assessment

Yes at source level, but not as a live vLLM run. Current code and tests show quiet model calls can classify as stalled and can become active-abort candidates after diagnostics.stuckSessionAbortMs; the exact takeover timing from the report was not reproduced.

## Solution Assessment

Unclear. A new timeout knob is not the narrow first fix because global diagnostics and provider timeout controls already exist; the maintainable path is a deliberate recovery policy for long silent local model calls.
