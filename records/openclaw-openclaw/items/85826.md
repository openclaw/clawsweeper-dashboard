## Summary

Keep open: current main and v2026.6.9 already have diagnostic timeout knobs and streamed model progress tracking, but silent/non-streaming local model calls are still intentionally classified as stalled and active-aborted after `diagnostics.stuckSessionAbortMs`.

## What This Changes

The issue asks OpenClaw to stop the diagnostics stall detector from killing legitimate long local vLLM model calls and to provide configurable, graceful recovery behavior.

## Best Possible Solution

Keep the issue open for a maintainer decision on silent/non-streaming model-call recovery semantics, preserving the existing threshold knobs and streamed-progress fix while separating transcript-fence work into its canonical tracker.

## Reproduction Assessment

Yes, at source level: current tests and classifier code show a silent vLLM-shaped local model call becomes `session.stalled` and requests active abort at `diagnostics.stuckSessionAbortMs`. I did not run a live current-main vLLM reproduction in this read-only review.

## Solution Assessment

Unclear. The current threshold configuration and streamed-progress tracking are good partial fixes, but the remaining silent/non-streaming model-call behavior needs a maintainer policy decision rather than a mechanical cleanup close.
