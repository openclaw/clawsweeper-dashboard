## Summary

Keep open: current source and the reported release contradict the no-config premise, but source-level evidence still shows long silent model calls can become stalled and active-abort eligible; the reported takeover/message-loss path needs maintainer session-recovery policy review and live local-provider proof.

## What This Changes

The issue asks OpenClaw to stop legitimate long local vLLM model calls from being treated as failed stalled sessions and to provide configurable or graceful recovery for slow model calls.

## Best Possible Solution

Treat this as a session-recovery policy follow-up: document and use the existing global diagnostics and provider timeout knobs today, then decide whether long silent local model calls need provider-aware progress, safer active-abort semantics, or a user-visible still-working path.

## Reproduction Assessment

Yes at source level, but not as a live vLLM run. Current code and tests show active model calls can be classified as stalled after the diagnostics threshold and can become active-abort candidates after the abort threshold; the exact takeover timing from the report was not reproduced.

## Solution Assessment

Unclear. A duplicate timeout knob is not the best first fix because global diagnostics and provider timeout controls already exist; the maintainable path is a deliberate recovery policy for long silent local model calls.
