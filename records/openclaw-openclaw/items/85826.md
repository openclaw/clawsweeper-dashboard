## Summary

Keep open: current source and the reported release already expose diagnostics knobs, but current main still classifies silent model calls as stalled and can active-abort them after the abort threshold; the exact vLLM timing needs maintainer recovery-policy review rather than cleanup closure.

## What This Changes

The issue requests configurable and graceful handling for long local vLLM model calls that are classified as stalled and can end with failed sessions and undelivered channel replies.

## Best Possible Solution

Define a deliberate recovery policy for long silent local model calls: operator-tuned budgets should prevent false aborts, genuinely orphaned runs should still release lanes, and channels should receive visible fallback or status when a turn is aborted.

## Reproduction Assessment

Yes at source level, but not as a live vLLM run. Current code and tests show active model calls can be classified as stalled after the diagnostics threshold and can become active-abort candidates after the abort threshold; the exact takeover timing from the report was not reproduced here.

## Solution Assessment

Unclear. A duplicate timeout knob is not the best first fix because diagnostics and provider timeout controls already exist; the maintainable path is a deliberate recovery policy for long silent local model calls.
