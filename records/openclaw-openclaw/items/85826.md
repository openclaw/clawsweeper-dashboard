## Summary

Keep open: the hard-coded/no-config premise is contradicted by current source and the reported release, but the remaining long silent model-call abort/recovery behavior is real enough to need maintainer session-recovery policy review rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to avoid failing long local vLLM model calls by making stall thresholds tunable and recovering or reporting progress gracefully.

## Best Possible Solution

Keep this as a session-recovery policy follow-up: document/use the existing diagnostics and provider timeout knobs now, then decide provider-aware progress, abort, and still-working semantics for long silent local model calls.

## Reproduction Assessment

Yes at source level, but not as a live vLLM run. Current code and tests show stale model calls with active embedded-run ownership can be classified as stalled and sent through active-abort recovery after the diagnostics abort threshold.

## Solution Assessment

Unclear. Adding a duplicate timeout knob is not the best first fix because global diagnostics and provider timeout controls already exist; the maintainable path is a deliberate recovery policy for long silent local model calls.
