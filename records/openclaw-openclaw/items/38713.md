## Summary

Keep open: current main still has exact-only `skills info` lookup, while this PR supplies the focused formatter fix, covers the previously flagged ambiguous fallback case, and has no blocking review findings.

## What This Changes

The PR changes the skills CLI formatter so `openclaw skills info <name>` resolves exact, unique case-insensitive, and unique separator-normalized skill names, sanitizes not-found output, and adds tests plus a changelog entry.

## Best Possible Solution

Land this focused CLI formatter fix after normal maintainer review and merge gating, then track any remaining agent skill-availability problem separately if it still reproduces.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main forwards `<name>` into exact `name`/`skillKey` equality, so a list-visible case or separator variant such as `Excel-XLSX` or `excel_xlsx` can miss when requested as `excel-xlsx`.

## Solution Assessment

Yes. The PR is the narrow maintainable fix because exact matches keep current behavior, fallback matches are uniqueness-gated, and ambiguous candidates return not-found instead of showing details for the wrong skill.
