## Summary

Keep open. Current main still has exact-only `skills info` lookup, while this PR implements the missing unique case-insensitive and separator-normalized lookup behavior, addresses the prior ambiguity finding, has sufficient terminal proof, and has no blocking review findings.

## What This Changes

This PR updates the skills CLI formatter so `openclaw skills info <name>` trims and sanitizes requests, resolves exact then unique case-insensitive/separator-normalized name or key variants, adds regression tests, and records a changelog fix.

## Best Possible Solution

Land this focused PR after maintainer review confirms the exact head remains clean and mergeable; the lookup fix belongs in the CLI formatter without changing the broader skill loading contract.

## Reproduction Assessment

Yes, by source inspection. Current main passes the raw `skills info` argument into an exact-only formatter lookup, so a listed skill whose displayed name or `skillKey` differs only by case or common separators can return not-found.

## Solution Assessment

Yes. The PR keeps exact-match precedence, confines the change to the CLI formatter, requires non-exact fallbacks to be unique, and adds focused coverage for the fixed and ambiguous paths.
