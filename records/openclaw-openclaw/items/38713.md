## What This Changes

The branch updates the skills CLI formatter, tests, and changelog so `openclaw skills info <name>` resolves exact, unique case-insensitive, and unique separator-normalized skill name/key matches with sanitized not-found output.

## Best Possible Solution

Land the focused CLI formatter fix after maintainer merge gating, then track any remaining agent tool-list symptom as a narrower follow-up only if it still reproduces.

## Reproduction Assessment

Yes, by source inspection. Current main passes the raw requested name into exact `name`/`skillKey` equality, so a report containing `skillKey: "Excel-XLSX"` can be visible in `skills list` but miss `openclaw skills info excel-xlsx`.

## Solution Assessment

Yes. The PR preserves exact lookup behavior, adds uniqueness-gated fallback matching for the reported variants, and avoids returning arbitrary setup guidance for ambiguous case or separator collisions.
