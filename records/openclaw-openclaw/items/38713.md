## What This Changes

The PR updates the skills CLI formatter so `skills info` resolves exact, unique case-insensitive, and unique separator-normalized skill name/key variants, sanitizes not-found output, adds tests, and adds a changelog entry.

## Best Possible Solution

## Reproduction Assessment

Yes. Current main’s `formatSkillInfo` does exact `name`/`skillKey` equality, so a report containing `skillKey: "Excel-XLSX"` can miss `skills info excel-xlsx`; I did not run the CLI because this review is read-only.

## Solution Assessment

Yes. The PR keeps exact matches unchanged, adds uniqueness-gated fallback matching for the reported name variants, and returns not-found for ambiguous candidates instead of choosing by insertion order.
