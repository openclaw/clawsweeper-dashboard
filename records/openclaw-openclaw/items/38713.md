## Summary

Keep open: current main still has the exact-only `skills info` lookup, while this PR’s final head implements the focused unique fallback behavior with regression coverage, sufficient terminal proof, and no blocking review findings.

## What This Changes

The PR changes the skills CLI formatter, tests, and changelog so `openclaw skills info <name>` resolves exact, unique case-insensitive, and unique separator-normalized skill name/key variants while treating ambiguous fallbacks as not found.

## Best Possible Solution

Land this focused formatter, test, and changelog change, or an equivalent implementation, so list-visible skills can be inspected by stable key/name variants without displaying the wrong skill guidance on ambiguous fallbacks.

## Reproduction Assessment

Yes. Source inspection shows `openclaw skills info` forwards the requested name to an exact-only `formatSkillInfo` lookup on current main, so a list-visible skill keyed `Excel-XLSX` requested as `excel-xlsx` would miss; I did not run a local repro because this was a read-only review.

## Solution Assessment

Yes. The PR is the narrow maintainable fix: exact matches keep existing behavior, non-exact fallbacks are uniqueness-gated, and ambiguous candidates return not-found instead of showing the wrong skill guidance.
