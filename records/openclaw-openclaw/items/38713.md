## Summary

Keep open: current main still performs exact-only `skills info` lookup, while this PR supplies a focused uniqueness-gated resolver, regression coverage, changelog, sufficient terminal proof, and no blocking review findings.

## What This Changes

This PR updates the skills CLI formatter, tests, and changelog so `openclaw skills info <name>` resolves exact, unique case-insensitive, and unique separator-normalized skill name/key matches.

## Best Possible Solution

Land this focused formatter, test, and changelog change, or an equivalent implementation, so list-visible skills can be inspected by stable key/name variants without ambiguous fallbacks showing the wrong guidance.

## Reproduction Assessment

Yes, from source inspection: `skills info` forwards the requested name to `formatSkillInfo`, and current main only checks exact `name`/`skillKey`, so a skill keyed `Excel-XLSX` requested as `excel-xlsx` would miss. I did not run a local command because this was a read-only review.

## Solution Assessment

Yes. The PR is the narrow maintainable fix: exact matches keep existing behavior, non-exact fallback modes are uniqueness-gated, and not-found output is sanitized.
