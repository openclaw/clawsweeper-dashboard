## What This Changes

This PR updates the skills CLI formatter so `openclaw skills info <name>` resolves exact, unique case-insensitive, and unique separator-normalized skill name/key matches, sanitizes not-found output, and adds tests plus a changelog entry.

## Best Possible Solution

Land the focused CLI formatter fix after maintainer review and merge gating, then track any remaining agent-availability symptom as a separate narrower report if needed.

## Reproduction Assessment

Yes. Source inspection on current main shows `skills info` uses exact `name`/`skillKey` equality, so a report with `skillKey: "Excel-XLSX"` can miss `openclaw skills info excel-xlsx`; I did not run the CLI because this was a read-only review.

## Solution Assessment

Yes. The PR keeps exact matches first, adds uniqueness-gated fallback matching for the reported variants, and returns not-found for ambiguous candidates instead of choosing by insertion order.
