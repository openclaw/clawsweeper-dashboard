## Summary

Keep open: current main still has exact-only `skills info` lookup, so the PR targets a real remaining CLI bug, but the PR head still needs a uniqueness guard for case-insensitive fallback resolution and a changelog entry before merge.

## What This Changes

The PR adds trimmed, case-insensitive, and separator-normalized `skills info` lookup with formatter tests and sanitized not-found output.

## Best Possible Solution

Keep exact matches first, require exactly one candidate for every fallback resolver path, add collision regression coverage, and add a single-line `CHANGELOG.md` fix entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: construct a `SkillStatusReport` with two entries whose `name` or `skillKey` differ only by case, then call `formatSkillInfo` with a lower-cased token; the PR head's case-insensitive `.find()` branch can select the first match.

## Solution Assessment

No. The PR addresses the right surface, but it is not the narrow maintainable fix until the case-insensitive fallback uses the same uniqueness rule as the normalized fallback and the changelog is added.
