## Summary

## What This Changes

The PR updates `openclaw skills info` lookup formatting to support exact, case-insensitive, and separator-normalized skill-name matches only when fallback matches are unique, sanitizes not-found output, adds tests, and adds a changelog entry.

## Best Possible Solution

Land the focused formatter fix after maintainer review, then track any separate agent skill availability symptom only if it still reproduces against the fixed lookup behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: the CLI passes `<name>` directly to `formatSkillInfo`, and current main only compares exact `name` or `skillKey`, so `skillKey: "Excel-XLSX"` queried as `excel-xlsx` misses.

## Solution Assessment

Yes. Exact-first lookup with unique case-insensitive and separator-normalized fallbacks is the narrowest maintainable fix for the CLI mismatch; broader agent availability behavior should not be folded into this formatter PR without a separate reproduction.
