## Summary

Keep open: current main still uses exact-only `skills info` lookup, while this PR adds a focused unique fallback resolver with regression coverage, passing checks, and sufficient terminal proof. I found no blocking correctness issue in the current PR head.

## What This Changes

The PR updates `openclaw skills info` lookup to keep exact matches first, add unique case-insensitive and separator-normalized fallbacks, sanitize not-found output, add regression tests, and add a changelog entry.

## Best Possible Solution

Land this focused formatter/test/changelog change, or an equivalent implementation, so list-visible skills can be inspected by stable key/name variants while ambiguous fallbacks remain not-found.

## Reproduction Assessment

Yes, from source inspection: `openclaw skills info` reaches `formatSkillInfo`, and current main only checks exact `name` or `skillKey`. A status report with `Excel XLSX` / `Excel-XLSX` and a request for `excel-xlsx` would miss on current main.

## Solution Assessment

Yes, this is the narrow maintainable fix: exact matches keep existing behavior, non-exact fallbacks are uniqueness-gated, and ambiguous candidates do not display the wrong skill guidance.
