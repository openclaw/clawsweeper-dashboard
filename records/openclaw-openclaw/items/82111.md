The branch adds a lazy cron runtime-plugin facade, preloads runtime plugins during cron context preparation with gateway-bindable mode, and adds mocks plus regression coverage for preload options and ordering.

## Best Possible Solution

Land this PR or an equivalent main change after normal maintainer gates so cron context setup loads runtime plugins once, in gateway-bindable mode, before model and delivery resolution.

## Reproduction Assessment

Yes, at source level: current main resolves cron model and delivery before loading runtime plugins, and the PR body supplies concrete Windows cron steps plus after-fix console output. I did not run a live current-main cron repro in this read-only review.

## Solution Assessment

Yes; the lazy facade preserves cron import laziness while moving the existing runtime-plugin helper to the point where model and delivery resolution need the registry. Passing gateway-bindable mode matches the later embedded cron run and avoids duplicate plugin initialization.
