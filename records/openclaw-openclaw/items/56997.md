## Summary

Keep this open: current main and v2026.6.8 still expose channel metadata but do not provide separated channel tabs, channel filters, or a merged all-channel timeline toggle in the Control UI. The related closed reports route their remaining Control UI channel/session-view work here, while the open sidebar grouping PR is narrower and does not satisfy the requested view mode.

## What This Changes

Request to add a Control UI session view toggle between separated per-channel conversations and a merged all-channel timeline.

## Best Possible Solution

Keep this as the canonical Control UI channel-view product thread, define separated-view and merged-timeline semantics, reuse existing session row channel metadata where possible, and add Gateway/protocol support only for a real channel filter or merged transcript contract.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and release inspection give high confidence that the requested separated/merged channel view does not currently exist.

## Solution Assessment

Unclear until maintainers choose the UX and data boundary. A UI-only separated view over existing channel metadata is narrower, while a true merged timeline needs transcript aggregation and privacy/session semantics.
