## Summary

Keep this issue open: current main still lacks the requested separated channel tabs and merged all-channel timeline toggle. The closed related reports point here as the canonical product thread, and the only open related PR handles a narrower sidebar recent-session grouping, not the full session view mode.

## What This Changes

Request to add a Control UI mode that can show channel conversations separately and also merge all channel conversations into one timeline.

## Best Possible Solution

Keep this as the canonical product thread, define the separated and merged Control UI semantics, then implement the UI mode over existing session channel metadata with a Gateway filter or timeline contract only if UI-side grouping is insufficient.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main does not provide separated channel tabs or a merged all-channel timeline toggle.

## Solution Assessment

Unclear until maintainers choose the UX and contract. A UI-first grouped/toggle view over existing channel and lastChannel metadata is plausible, but a true merged conversation timeline may need an explicit Gateway or chat-history contract.
