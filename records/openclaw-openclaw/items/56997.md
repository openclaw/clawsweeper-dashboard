## Summary

Keep this issue open: current main exposes channel metadata on session rows, but the Control UI still lacks the requested separated channel view and merged all-channel timeline toggle. The related closed issues already point here as the canonical product thread, and an open related PR only handles sidebar recent-session grouping, not the full request.

## What This Changes

The issue requests a Control UI mode that can switch between per-channel conversation views and one merged timeline across channel conversations.

## Best Possible Solution

Keep this canonical issue open, settle the Control UI UX for separated versus merged channel views, then implement it on existing session row channel metadata with a Gateway filter/mode only if UI-side grouping is insufficient.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main does not provide the requested separated channel tabs or merged timeline toggle.

## Solution Assessment

Unclear until maintainers choose the UX. A UI-first grouped/toggle view over existing `channel` and `lastChannel` metadata looks plausible, but a true merged conversation timeline may need a more explicit Gateway or chat-history contract.
