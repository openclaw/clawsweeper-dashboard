## Summary

Keep this issue open. Current main still supports selecting one session key at a time, including channel-derived keys, but it does not provide separated channel tabs or a merged all-channel timeline in Control UI.

## What This Changes

Add a Control UI mode toggle that can show channel conversations separately or merge them into one unified timeline.

## Best Possible Solution

Add an explicit core Control UI separated/merged mode backed by an additive Gateway contract that aggregates authorized session transcripts with timestamped channel/account/source metadata.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-state check: the UI and Gateway still operate on one active sessionKey and have no merged timeline contract.

## Solution Assessment

Yes, the requested direction belongs in core Control UI/Gateway rather than ClawHub because it changes built-in chat rendering and cross-session transcript access. The implementation should be additive and preserve the existing session selector as the separated-session foundation.
