## Summary

Related closed channel-filter reports route their remaining work here, while the open sidebar grouping PR is narrower and not a replacement.

## What This Changes

The issue requests Control UI modes to view channel conversations either separated by channel or merged into one timeline.

## Best Possible Solution

Keep this as the canonical Control UI channel-view thread, define separated-view and merged-timeline semantics first, then reuse existing session row channel metadata and add Gateway/protocol support only where the chosen UX requires it.

## Solution Assessment

Unclear until maintainers choose the UX and data boundary. A separated view over existing session rows is narrower, while a true merged timeline needs explicit cross-session transcript aggregation, privacy, and protocol semantics.
