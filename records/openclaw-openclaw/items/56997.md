The issue requests a Control UI toggle between per-channel separated sessions and a merged all-channel conversation timeline.

## Best Possible Solution

Keep this as the canonical product thread, define the Control UI UX and session aggregation contract, then reuse existing session channel metadata with a Gateway mode only if UI-side grouping is insufficient.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers choose the UX. A UI-first grouped or filtered view over existing session metadata looks plausible for separated navigation, but a true merged conversation timeline likely needs an explicit Gateway or chat-history contract.
