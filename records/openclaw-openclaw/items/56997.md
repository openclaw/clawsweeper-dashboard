## Summary

Keep this open: current main and v2026.6.6 still expose channel metadata on session rows but do not provide the requested separated channel tabs or merged all-channel timeline toggle. The closed related issues point here as the canonical product thread, and the open sidebar-grouping PR is useful but partial.

## What This Changes

Request a Control UI session view mode that toggles between separated per-channel conversations and a merged all-channel timeline.

## Best Possible Solution

Keep this as the canonical product thread, decide the separated-versus-merged UX and data contract, then implement channel-aware Control UI views over existing session metadata with Gateway additions only if UI-side grouping is insufficient.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main does not provide separated channel tabs or a merged timeline toggle.

## Solution Assessment

Unclear until maintainers choose the UX and data boundary. A UI-first view over existing channel metadata looks plausible, but a true merged conversation timeline may need a new Gateway or chat-history contract.
