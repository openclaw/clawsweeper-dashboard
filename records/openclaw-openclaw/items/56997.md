## Summary

Keep this open: current main and v2026.6.6 still do not provide separated channel tabs or a merged all-channel timeline toggle in the Control UI. Current code exposes enough channel/session metadata to make the request plausible, and the related closed issues point back here as the canonical product thread.

## What This Changes

Request a Control UI mode that can show channel conversations separately by channel or merge them into one unified timeline.

## Best Possible Solution

Keep this as the canonical product thread, define separated/merged UX and privacy/session semantics, then implement channel-aware Control UI views over existing metadata with Gateway/API additions only if a real merged transcript is required.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection of current main and v2026.6.6 gives high confidence that the requested Control UI separated/merged channel view does not exist.

## Solution Assessment

Unclear until maintainers choose the UX and data boundary. A UI-only separated/grouped view is likely narrow, but a true merged timeline needs a decision on transcript aggregation versus a new server-side session mode.
