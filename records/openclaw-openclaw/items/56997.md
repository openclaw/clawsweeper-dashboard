## Summary

Keep this open: current main and v2026.6.8 still provide session listing, session switching, and some channel-derived metadata, but they do not provide separated channel tabs, a channel view-mode toggle, or a merged all-channel chat timeline in the Control UI. Related closed reports route their remaining channel/filtering work here, while the open sidebar grouping PR is narrower than the requested feature.

## What This Changes

Request a Control UI mode that can show channel conversations separately by channel or merged into one unified timeline.

## Best Possible Solution

Keep this as the canonical Control UI channel-view product thread, define separated-view and merged-timeline semantics first, reuse existing session row channel metadata where possible, and add Gateway/protocol support only for a real channel filter or merged transcript contract.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and release inspection give high confidence that the requested separated/merged channel view does not currently exist.

## Solution Assessment

Unclear until maintainers choose the UX and data boundary. A separated view over existing session rows is narrower, while a true merged timeline needs explicit transcript aggregation, privacy, and protocol semantics.
