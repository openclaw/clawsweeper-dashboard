## Summary

Keep this issue open. Current main has useful session-selection groundwork, including channel-derived session labels, but Control UI and the Gateway still operate on one active session key and do not expose separated channel tabs or a merged all-channel timeline.

## What This Changes

Request a Control UI mode toggle between separated per-channel conversation views and a merged unified timeline across channel conversations.

## Best Possible Solution

Add an additive Control UI/Gateway design for separated and merged session views that preserves existing per-session navigation while exposing authorized aggregate transcripts with stable channel, account, source, and timestamp metadata.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-state check: Control UI and chat.history still load one active sessionKey, with no merged timeline contract.

## Solution Assessment

Yes, the requested direction belongs in core Control UI/Gateway rather than a plugin because it changes built-in chat rendering and cross-session transcript access. The implementation should be additive and preserve the existing session selector as the separated-session foundation.
