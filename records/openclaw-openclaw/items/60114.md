- **[P2] Parse alias arguments from the raw command text:** `src/tui/tui-command-handlers.ts:528`
  - body: `args` has already been normalized by `parseCommand`, which splits on whitespace and rejoins the rest. That means `/alias review "line1  line2"` is saved with collapsed spacing before `parseTuiAliasArgs` can preserve the user's prompt, so the alias may send different text than authored. Pass the raw command tail into the alias parser or have `parseCommand` expose raw args.
  - confidence: 0.9

Those paths are unrelated to TUI aliases and should stay covered or move in a separate Matrix-focused change.
  - confidence: 0.84
- **[P3] Show aliases with the actual invocation form:** `src/tui/tui-command-handlers.ts:263`
  - body: The list output formats entries as `/${name} -> ${prompt}`, which implies users can type the alias as its own slash command. The implemented invocation is `/alias <name>`, so the list should show that form or otherwise avoid suggesting a non-existent command.
  - confidence: 0.78
