## What This Changes

## Best Possible Solution

Review and prove the linked fix PR, then either land it as the narrow dropdown reconciliation or keep the broader selected/effective model contract in the related resolved-backend issue.

## Reproduction Assessment

Yes, source-reproducible: current main prefers cached dropdown state before the server row and the dropdown switch does not consume the existing `sessions.patch.resolved` model payload. I did not run a live provider fallback/browser repro in this read-only review.

## Solution Assessment

The requested outcome is correct, but the best implementation needs maintainer direction between a narrow dropdown reconciliation using `sessions.patch.resolved` and a broader selected-versus-effective Gateway row contract.
