## Summary

Keep open: current main still has a cache-first chat model picker and a Gateway session-row projection that can collapse selected and runtime model identity, so the reported misleading model state is not safely resolved.

## What This Changes

The issue asks the chat model picker to reflect or clarify the effective runtime model when selected/preferred model state drifts because of fallback or default behavior.

## Best Possible Solution

Expose selected/preferred and effective/runtime model identity separately in Gateway session metadata, then update the chat picker/status UI and regression tests around the chosen maintainer contract.

## Reproduction Assessment

Yes, source-reproducible: current UI resolves the picker from chatModelOverrides before sessions.list, and sessions.list collapses selected/runtime identity into row.model. I did not run a live Gateway repro because this was a read-only sweep.

## Solution Assessment

Unclear: the requested direction is right, but a cache-only UI fix would not cover the Gateway selected-over-runtime projection. The maintainable fix needs an explicit selected/effective model contract before implementation.
