## Summary

Keep open: current main still has a cache-first chat model picker and a selected-first Gateway session-row projection, so the reported misleading selected-versus-runtime model state is not resolved.

## What This Changes

The issue asks for the web chat model dropdown to reflect the effective runtime model, or clearly distinguish selected/preferred and active runtime model when they differ.

## Best Possible Solution

Expose selected/preferred and effective/runtime model identity separately in Gateway session metadata, then update the chat picker/status UI and focused regression tests around that contract.

## Reproduction Assessment

Yes, source-reproducible: the UI resolves the dropdown from chatModelOverrides before sessions.list, and sessions.list collapses selected/runtime identity into row.model. I did not run a live Gateway repro in this read-only review.

## Solution Assessment

Unclear: the requested selected/effective distinction is the right direction, but a cache-only UI fix would not cover the Gateway selected-over-runtime projection. The maintainable fix needs an explicit session metadata contract before implementation.
