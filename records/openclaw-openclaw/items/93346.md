## Summary

Keep open: current main and v2026.6.9 still have the cache-first WebChat model selector and selected-over-runtime Gateway row projection, and the open linked fix PR has not merged yet.

## What This Changes

The issue reports that WebChat's model dropdown can show a cached selected model while the effective runtime/default model differs after fallback or session drift.

## Best Possible Solution

Land a narrow fix that preserves Default selection semantics while exposing or using authoritative selected-versus-effective model metadata for the dropdown, with real WebChat/Gateway proof.

## Reproduction Assessment

Yes, source-level: the current selector returns the local cached model before reading the active server row, and Gateway row fields still choose selected model identity over runtime identity. I did not run a live Gateway/provider fallback repro in this read-only pass.

## Solution Assessment

Yes on the desired outcome, but the implementation should preserve Default semantics and use an authoritative selected-versus-effective model contract instead of only recaching the requested dropdown value.
