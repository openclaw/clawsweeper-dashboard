## Summary

Keep open: current main and the latest v2026.6.8 release still have the cache-first chat model picker and selected-over-runtime session row projection, while the linked fix PR is open and unmerged.

## What This Changes

The issue reports that the web chat model dropdown can show a cached selected model while the session is actually using a fallback or default runtime model.

## Best Possible Solution

Expose selected/preferred and effective/runtime model identity separately in Gateway session metadata, then update the chat dropdown or status UI and regression tests around that contract.

## Reproduction Assessment

Yes, source-reproducible: the UI returns a cached override before consulting sessions.list, and Gateway rows prefer selected model identity over runtime identity. I did not run a live browser/Gateway fallback scenario in this read-only review.

## Solution Assessment

Unclear: the issue is valid, but a cache-only UI fix is not the best durable solution. The maintainable path needs a maintainer-approved selected/effective model contract before implementation lands.
