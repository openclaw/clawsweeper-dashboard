## Summary

Keep open. Current main and the latest shipped release still lack a documented config option or public `sessions_spawn` parameter for deterministic sub-agent announce suppression; the closed implementation attempts were not merged, and the open parent-routing/per-channel requests are adjacent API-shape work rather than replacements for this narrower skip control.

## What This Changes

The issue requests a framework-level config option or `sessions_spawn` parameter to suppress sub-agent completion announcements without relying on `ANNOUNCE_SKIP` or `NO_REPLY`.

## Best Possible Solution

Add a backward-compatible public subagent completion policy that defaults to current behavior, supports deterministic skip, and is coordinated with parent-routing and per-channel policy before updating schema, docs, generated help, delivery paths, and regression tests.

## Reproduction Assessment
