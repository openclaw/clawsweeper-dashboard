## Summary

Keep open. Current main still lacks a documented framework-level config option or native `sessions_spawn` parameter that deterministically suppresses sub-agent completion announces; prior implementation attempts were closed unmerged and the remaining work is a public API/config feature.

## What This Changes

Requests a framework-level config option and/or `sessions_spawn` parameter to deterministically suppress sub-agent completion announces.

## Best Possible Solution

Add a documented public suppression policy under `agents.defaults.subagents` and/or native `sessions_spawn`, thread it through registry, cleanup, and delivery paths, and coordinate naming with #13911 and #27445.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection gives high-confidence current-main evidence that no supported config or native `sessions_spawn` suppression option exists and public behavior still relies on token/internal lifecycle handling.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is a framework-level policy that avoids model-token enforcement, treats intentional suppression as successful cleanup, and does not conflict with #13911 or #27445.
