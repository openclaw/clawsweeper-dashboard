## Summary

Keep open because current main still lacks a documented framework-level config option or public `sessions_spawn` parameter for deterministic sub-agent announce suppression. The only supported suppression remains token/internal-state driven, while prior implementation PRs are closed unmerged and the open related items cover adjacent per-channel or parent-routing semantics rather than replacing this narrower request.

## What This Changes

Request a framework-level config option or per-spawn `sessions_spawn` parameter to suppress sub-agent completion announcements deterministically.

## Best Possible Solution

Add a backward-compatible native subagent announce policy that defaults to current behavior, supports deterministic skip without token reliance, treats intentional suppression as successful cleanup, and updates schema/help/docs at https://docs.openclaw.ai/tools/subagents while coordinating with https://github.com/openclaw/openclaw/issues/13911 and https://github.com/openclaw/openclaw/issues/27445.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection of current main gives high-confidence evidence that no supported config or native `sessions_spawn` suppression option exists.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is a framework-level policy that avoids model-token enforcement and is designed alongside the per-channel and parent-routing requests.
