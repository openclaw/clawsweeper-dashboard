## Summary

Keep open. Current main and the latest stable release still lack a documented public `sessions_spawn` or `agents.defaults.subagents` control that deterministically suppresses native sub-agent completion announces; suppression remains token/output driven, with related parent-routing and per-channel policy work still open.

## What This Changes

Request a framework-level config default and/or per-spawn option to suppress sub-agent completion announcements without relying on `ANNOUNCE_SKIP`.

## Best Possible Solution

Define one backward-compatible native completion-delivery policy that keeps current announce behavior as the default, supports deterministic per-spawn suppression, and adds config-level defaults only if maintainers approve the extra surface and precedence with parent/per-channel routing.

## Reproduction Assessment

Yes, at source level: current main exposes no public `sessions_spawn` announce/silent parameter and the strict subagent config schema omits `announcePolicy`, while docs still describe exact-token suppression. I did not run a live Telegram transport repro in this read-only review.

## Solution Assessment

Unclear as a final API shape. The narrow goal is valid, but the best implementation should coordinate per-spawn suppression with the open parent-routing and per-channel suppression trackers instead of adding a standalone knob that conflicts later.
