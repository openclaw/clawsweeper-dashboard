## Summary

Keep open. Current main and the latest shipped tag still do not expose a documented config option or public `sessions_spawn` parameter for deterministic sub-agent announce suppression; the remaining work is an API/product decision because skip-only, parent routing, and per-channel policy requests overlap.

## What This Changes

The issue requests a framework-level way to suppress sub-agent completion announcements, either by config or a public per-spawn option.

## Best Possible Solution

Add a backward-compatible public subagent announce policy that defaults to current behavior, supports deterministic skip, and is coordinated with parent-routing and per-channel policy before updating schema, docs, generated help, lifecycle delivery, and regression tests.

## Reproduction Assessment

No, not as a bug reproduction; this is a feature/config request. Source and shipped-tag inspection give high confidence that the requested public config or per-spawn suppression policy is still absent.

## Solution Assessment

Yes at the direction level: structural code/config enforcement is the maintainable way to avoid relying on model-token compliance. The exact option names and composition with parent routing and per-channel policy still need maintainer product/API judgment.
