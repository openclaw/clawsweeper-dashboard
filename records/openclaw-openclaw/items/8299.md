## Summary

Keep open: current main still has no documented config option or public `sessions_spawn` parameter for deterministic sub-agent announce suppression, and the prior implementation PRs were closed unmerged rather than shipped.

## What This Changes

The issue requests a framework-level config or per-spawn `sessions_spawn` option to suppress sub-agent completion announcements without relying on `ANNOUNCE_SKIP` or `NO_REPLY`.

## Best Possible Solution

Add a backward-compatible native subagent announce policy that defaults to current behavior, supports deterministic skip without token reliance, treats intentional suppression as successful cleanup, and updates schema/types, generated config docs, https://docs.openclaw.ai/tools/subagents, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection on current main gives high-confidence evidence that no supported config option or public `sessions_spawn` parameter exists for deterministic suppression.

## Solution Assessment

Unclear until maintainers choose the public API shape. A framework-level policy is the right class of solution, but it should be designed alongside https://github.com/openclaw/openclaw/issues/13911 and https://github.com/openclaw/openclaw/issues/27445 so skip, per-channel, and parent-routing semantics do not diverge.
