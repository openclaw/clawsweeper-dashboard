## Summary

Keep open: current main still lacks a documented framework-level config option or public native `sessions_spawn` parameter that deterministically suppresses sub-agent completion announcements. Related per-channel and parent-routing issues remain open and complementary rather than replacing this narrower suppression request.

## What This Changes

The issue requests a framework-level config or per-spawn option to suppress sub-agent completion announcements without relying on `ANNOUNCE_SKIP` or `NO_REPLY`.

## Best Possible Solution

Add a backward-compatible native subagent announce policy that defaults to current behavior, supports deterministic skip without token reliance, treats intentional suppression as successful cleanup, and coordinates naming with https://github.com/openclaw/openclaw/issues/13911 and https://github.com/openclaw/openclaw/issues/27445.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection of current main gives high-confidence evidence that no supported config or native `sessions_spawn` suppression option exists.

## Solution Assessment

Unclear until maintainers choose the public API shape. A framework-level policy is the right class of solution, but it should be designed alongside the per-channel suppression and parent-routing requests.
