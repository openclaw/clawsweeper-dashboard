## Summary

Keep open. Current main still lacks a documented config key or public `sessions_spawn` parameter that deterministically suppresses sub-agent completion announces; suppression remains token-driven, and related routing/per-channel work is still open rather than a replacement for this narrower control.

## What This Changes

Request a config or per-spawn option to suppress sub-agent completion announces without relying on the model to emit `ANNOUNCE_SKIP`.

## Best Possible Solution

Add a backward-compatible announce-delivery policy that defaults to current behavior, supports deterministic skip through public config and/or `sessions_spawn`, coordinates with routing work, and updates schema, generated config docs, subagent docs, lifecycle state, delivery tests, and cleanup semantics.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high confidence that current main has no public config key or `sessions_spawn` parameter for deterministic sub-agent announce suppression.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is to thread one policy through tool schema, config schema/types, registry/lifecycle state, announce delivery, docs, and regression tests rather than relying on model output tokens.
