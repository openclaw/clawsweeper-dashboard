## Summary

Keep open. Current main still lacks a documented public config option or `sessions_spawn` parameter for deterministic sub-agent announce suppression; suppression remains token-driven, and the related routing/per-channel issues are adjacent rather than complete replacements.

## What This Changes

Feature request for a config or spawn-level option to deterministically suppress sub-agent completion announcements.

## Best Possible Solution

Add a backward-compatible public subagent completion policy that defaults to current behavior, supports deterministic skip, and is coordinated with parent-routing and per-channel policy before updating schema, docs, generated help, delivery paths, and regression tests.

## Reproduction Assessment

No, not as a runtime bug reproduction because this is a feature/config request. Source and latest-tag inspection give high confidence that no public suppression config or `sessions_spawn` parameter exists.

## Solution Assessment

Yes at the direction level: structural code/config enforcement is the maintainable way to avoid relying on model-token compliance. The exact option names and composition with https://github.com/openclaw/openclaw/issues/27445 and https://github.com/openclaw/openclaw/issues/13911 still need maintainer product/API judgment.
