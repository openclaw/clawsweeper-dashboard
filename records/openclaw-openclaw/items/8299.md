## Summary

Keep open: current main still lacks a documented config option or public native `sessions_spawn` parameter for deterministic sub-agent announce suppression, and newer overlapping reports point back to this issue as the canonical suppression tracker.

## What This Changes

Feature request to add deterministic framework-level suppression for sub-agent completion announcements, either through config or a public `sessions_spawn` parameter.

## Best Possible Solution

Add a backward-compatible native subagent announce policy that defaults to current behavior, supports deterministic skip without token reliance, treats intentional suppression as successful cleanup, and updates schema/types, generated config docs, public subagent docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source and docs inspection on current main gives high-confidence evidence that no supported config option or public `sessions_spawn` parameter exists for deterministic suppression.

## Solution Assessment

Unclear until maintainers choose the public API shape. A framework-level policy is the right class of solution, but it should be designed alongside https://github.com/openclaw/openclaw/issues/13911 and https://github.com/openclaw/openclaw/issues/27445 so skip, per-channel, and parent-routing semantics do not diverge.
