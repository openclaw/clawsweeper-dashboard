## Summary

Keep open: current main and v2026.6.5 still lack a documented public `sessions_spawn` parameter or `agents.defaults.subagents` config policy for deterministic sub-agent announce suppression. The remaining work is a product/API decision because related open issues cover broader parent routing, per-channel policy, and spawn-side suppression without establishing one canonical public contract yet.

## What This Changes

The issue requests a framework-level config option or `sessions_spawn` parameter to suppress sub-agent completion announcements without relying on model output tokens.

## Best Possible Solution

Define one typed completion-audience or announce policy with current channel delivery as the default and explicit skip/parent modes, then thread it through spawn registration, delivery, successful cleanup bookkeeping, config/schema/help, docs, and focused regression tests.

## Reproduction Assessment

Yes, at source level. Current main's public tool schema, strict config surface, docs, and announce delivery code show deterministic suppression is unavailable without relying on exact model output tokens.

## Solution Assessment

No, not as the final API shape. Deterministic suppression is needed, but the best solution should coordinate skip, parent routing, and per-channel policy instead of adding a competing boolean/config surface.
