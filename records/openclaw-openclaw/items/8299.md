## Summary

Keep open. Current main still lacks a documented public config option or `sessions_spawn` parameter for deterministic sub-agent announce suppression, and the related prior PRs were closed unmerged while broader routing/per-channel requests remain separate follow-ups.

## What This Changes

Request a framework-level config or per-spawn option to suppress sub-agent completion announces instead of relying on `ANNOUNCE_SKIP` or `NO_REPLY`.

## Best Possible Solution

Add a documented public suppression policy under `agents.defaults.subagents` and/or `sessions_spawn`, thread it through spawn registration, lifecycle cleanup, and announce delivery, and align it with the per-channel and routing follow-ups.

## Reproduction Assessment

Not applicable as a feature/config request. Source and docs inspection on current main gives high-confidence evidence that no supported public config or tool parameter provides deterministic announce suppression.

## Solution Assessment

Unclear until maintainers choose the public API shape. The requested direction is maintainable if implemented as a documented policy that avoids model-token enforcement and preserves cleanup semantics while staying compatible with #13911 and #27445.
