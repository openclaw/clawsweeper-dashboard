## Summary

Current main still lacks a documented framework-level control to suppress sub-agent completion announcements; general suppression remains model-output-token driven, while the open related issues cover broader parent routing or per-channel policy rather than replacing this narrower global/per-spawn decision.

## What This Changes

Requests deterministic framework-level suppression of sub-agent completion announcements through configuration or a `sessions_spawn` parameter.

## Best Possible Solution

Define one typed completion-delivery policy with current channel delivery as the backward-compatible default, deterministic skip as an explicit value, and future routing extensibility only if maintainers choose it; thread the result through spawn registration, lifecycle delivery, cleanup state, schema/help/docs, and focused Telegram or channel proof.

## Reproduction Assessment

Yes. Current source, strict config schema, tests, and docs consistently show no public deterministic suppression control; only model-output tokens and an unrelated inline ACP lifecycle case suppress completion delivery.

## Solution Assessment

Unclear as proposed: deterministic framework control is appropriate, but maintainers should choose one typed policy and precedence model instead of independently adding a boolean config, a differently named tool flag, and channel-specific variants.
