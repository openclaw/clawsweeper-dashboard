## Summary

Keep open. Current main still lacks a documented framework-level config or public `sessions_spawn` parameter for deterministic sub-agent announce suppression, and the related open routing/channel issues are complementary rather than replacements for this canonical suppression request.

## What This Changes

The issue requests a config and/or per-spawn option to suppress sub-agent completion announcements so the parent agent can review output before anything reaches the user.

## Best Possible Solution

Add a backward-compatible public subagent completion policy under `agents.defaults.subagents` and/or `sessions_spawn`, default to current behavior, support deterministic skip, and thread it through registry, lifecycle, delivery, docs, generated config schema/help, and regression tests while coordinating with the open routing and per-channel issues.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source and release inspection give high confidence that the requested public config/per-spawn suppression policy is still absent.

## Solution Assessment

Yes at the direction level: structural code/config enforcement is the maintainable way to avoid model-token reliance. The exact public option names and composition with parent routing and per-channel policy still need maintainer product/API judgment.
