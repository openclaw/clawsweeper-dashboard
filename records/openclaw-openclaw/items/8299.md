## Summary

Keep open. Current main still lacks a documented config option or public `sessions_spawn` parameter for deterministic sub-agent announce suppression; the implemented suppression paths are still token/internal-flag driven, while prior direct PR attempts are closed unmerged and adjacent routing/per-channel requests remain separate product choices.

## What This Changes

The issue requests a framework-level config option or per-spawn parameter to suppress sub-agent completion announcements without relying on `ANNOUNCE_SKIP`.

## Best Possible Solution

Add a documented public suppression policy under `agents.defaults.subagents` and/or `sessions_spawn`, thread it through spawn registration, lifecycle cleanup, and announce delivery, and coordinate the shape with the per-channel and routing follow-ups.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection gives high-confidence evidence that current main lacks the requested supported config/tool parameter and still relies on token-driven suppression for public behavior.

## Solution Assessment

Unclear until maintainers choose the public API shape. A narrow implementation should avoid magic model tokens, treat intentional suppression as successful cleanup, and avoid conflicting with #13911 and #27445.
