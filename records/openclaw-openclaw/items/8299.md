## Summary

Keep open. Current `main` still lacks a documented public config key or `sessions_spawn` parameter that deterministically suppresses native sub-agent completion announcements; suppression is still token-driven or internal lifecycle-only, and the related routing/per-channel issues do not replace this narrower request.

## What This Changes

Request a framework-level config and/or `sessions_spawn` option to suppress sub-agent completion announcements without relying on `ANNOUNCE_SKIP`.

## Best Possible Solution

Add a backward-compatible, documented framework-level suppression policy for native sub-agent completion announces, preserve current behavior by default, and prove deterministic skip plus cleanup success in focused tests.

## Solution Assessment
