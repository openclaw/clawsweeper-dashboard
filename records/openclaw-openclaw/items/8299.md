## Summary

Keep open: current main and the latest shipped tag still lack a documented config option or public native `sessions_spawn` parameter for deterministic sub-agent announce suppression, while prior implementation PRs were closed unmerged and related routing/per-channel requests remain open.

## What This Changes

Request a framework-level config and/or per-spawn option to suppress sub-agent completion announces without relying on model-emitted `ANNOUNCE_SKIP` or `NO_REPLY` tokens.

## Best Possible Solution

Add a backward-compatible native subagent announce policy that defaults to current behavior, supports deterministic skip through config and/or `sessions_spawn`, treats intentional suppression as successful cleanup, and updates schema, generated config docs, public subagent docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source, docs, shipped-tag, and test inspection give high-confidence evidence that no supported config option or public `sessions_spawn` parameter exists for deterministic suppression.

## Solution Assessment

Unclear until maintainers choose the public API shape. A framework-level policy is the right class of solution, but it should be designed alongside https://github.com/openclaw/openclaw/issues/13911 and https://github.com/openclaw/openclaw/issues/27445 so skip, per-channel, and parent-routing semantics do not diverge.
