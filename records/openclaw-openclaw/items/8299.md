## Summary

Keep open. Current main still lacks a documented config option or public `sessions_spawn` parameter for deterministic sub-agent announce suppression; suppression is still token-driven, and the related suppress/routing proposals remain unmerged or separate follow-ups.

## What This Changes

Request a framework-level config or per-spawn option to suppress sub-agent completion announcements without relying on model output.

## Best Possible Solution

Add a documented public suppression policy that threads through spawn registration, lifecycle cleanup, and announce delivery while coordinating with the related per-channel and routing issues.

## Reproduction Assessment

Not applicable as a feature request. Source and docs on current main give high-confidence evidence that no public suppress config or `sessions_spawn` parameter exists and suppression remains token-driven.

## Solution Assessment

Unclear until maintainers choose the public API shape. A narrow implementation should avoid relying on magic model tokens, treat intentional suppression as successful cleanup, and not conflict with #13911 or #27445.
