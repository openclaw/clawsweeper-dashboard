## Summary

Keep open. Current main and the latest shipped tag still lack a public config key or `sessions_spawn` parameter that deterministically suppresses sub-agent completion announces, and the related attempts were closed unmerged while broader routing/per-channel follow-ups remain open.

## What This Changes

Request a framework-level config option and/or `sessions_spawn` parameter to skip sub-agent completion announcements without relying on `ANNOUNCE_SKIP` or `NO_REPLY`.

## Best Possible Solution

Add a backward-compatible announce-delivery policy that defaults to current behavior, supports deterministic skip via public config and/or `sessions_spawn`, coordinates with parent-routing work, and updates schema, generated config docs, https://docs.openclaw.ai/tools/subagents, and focused lifecycle/delivery tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high confidence that current main has no public config key or `sessions_spawn` parameter for deterministic sub-agent announce suppression.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is to thread one policy through tool schema, config schema/types, subagent registry/lifecycle, announce delivery, docs, and regression tests rather than relying on model output tokens.
