## Summary

Keep open: current main still lacks a documented config option or public `sessions_spawn` parameter for deterministic sub-agent announce suppression. The internal `expectsCompletionMessage` path can skip announce cleanup in selected runtime-owned cases, but it is not exposed as the requested user-facing policy, and the newer duplicate https://github.com/openclaw/openclaw/issues/91960 was closed back to this issue as the canonical thread.

## What This Changes

Request a framework-level way, via config and/or `sessions_spawn`, to suppress sub-agent completion announcements so parent agents can gate output.

## Best Possible Solution

Add a documented deterministic suppress mode in the core subagent completion lifecycle, preferably as a narrow public per-spawn control first, with schema, docs, and regression coverage; add config defaults only after maintainer approval of the extra surface.

## Reproduction Assessment

Yes. Source inspection against current main shows no public `sessions_spawn` announce/silent parameter and no `agents.defaults.subagents.announcePolicy`, while suppression remains token-driven or internal-state-driven.

## Solution Assessment

Unclear. A deterministic skip control belongs in the core spawn/announce lifecycle, but maintainers still need to choose whether the permanent API is per-spawn, config-backed, per-channel, or part of broader routing.
