## Summary

Keep open because current main still lacks a documented framework-level config option or public native `sessions_spawn` parameter that deterministically suppresses native sub-agent completion announcements. The supported suppression path remains output-token driven, while prior implementation PRs were closed unmerged and adjacent per-channel/parent-routing requests remain open.

## What This Changes

Request a config and/or per-spawn control to suppress sub-agent completion announcements without relying on `ANNOUNCE_SKIP` model output.

## Best Possible Solution

Add a backward-compatible native subagent announce policy that defaults to current behavior, supports deterministic skip without token reliance, treats intentional suppression as successful cleanup, and coordinates naming with https://github.com/openclaw/openclaw/issues/13911 and https://github.com/openclaw/openclaw/issues/27445.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection of current main gives high-confidence evidence that no supported config or native `sessions_spawn` suppression option exists.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is a framework-level policy that avoids model-token enforcement and is designed alongside the per-channel and parent-routing requests.
