## Summary

Keep open. Current main still lacks a documented public config option or `sessions_spawn` parameter for deterministic sub-agent announce suppression; suppression remains token-driven or internal lifecycle-driven, and adjacent routing/per-channel issues do not replace this narrower request.

## What This Changes

Request a framework-level way to suppress sub-agent completion announcements via `agents.defaults.subagents` config and/or a public `sessions_spawn` parameter.

## Best Possible Solution

Add a documented public suppression policy under `agents.defaults.subagents` and/or `sessions_spawn`, thread it through spawn registration, lifecycle cleanup, and announce delivery, and align the shape with the open per-channel and routing follow-ups.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection gives high-confidence evidence that current main lacks the requested supported config/tool parameter and still relies on token or internal lifecycle suppression for public behavior.

## Solution Assessment

Unclear until maintainers choose the public API shape. A narrow implementation should avoid magic model tokens, treat intentional suppression as successful cleanup, and avoid conflicting with #13911 and #27445.
