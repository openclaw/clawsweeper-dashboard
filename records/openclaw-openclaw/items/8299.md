## Summary

Keep open. Current main still lacks a documented public config or `sessions_spawn` parameter that deterministically suppresses sub-agent completion announcements; suppression remains token-driven or internal-lifecycle driven. Prior direct implementation attempts were closed unmerged, and the open routing/per-channel follow-ups are adjacent rather than replacements for this narrower skip control.

## What This Changes

The issue requests a framework-level config option and/or per-spawn `sessions_spawn` parameter to suppress sub-agent completion announcements without relying on the sub-agent model replying with an exact skip token.

## Best Possible Solution

Add a backward-compatible, documented framework-level suppression policy for native sub-agent completion announces. Keep current behavior as the default, support deterministic skip through a public `sessions_spawn` option and/or `agents.defaults.subagents`, thread the choice through spawn registration and announce cleanup so intentional suppression counts as successful cleanup, and add focused config/docs/tests; leave parent routing and per-channel policy to their dedicated follow-ups unless the implementation naturally shares a small seam.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the public tool schema, strict config schema, generated config schema, docs, and focused tests expose only token-driven suppression, while top-level completion delivery still routes externally.

## Solution Assessment

Yes. A narrow framework-level skip policy is the best way to solve this specific issue, provided it preserves default behavior, treats intentional suppression as successful cleanup, and does not absorb the broader parent-routing or per-channel feature requests.
