## Summary

Keep open: current main still lacks a documented config option or public `sessions_spawn` parameter for deterministic sub-agent announce suppression, and suppression remains token/internal-lifecycle driven. Prior implementation attempts #8306 and #8393 were closed unmerged, while #27445 and #13911 cover broader routing/per-channel variants rather than replacing this narrower request.

## What This Changes

Request to add framework-level control, via config and/or `sessions_spawn`, to suppress sub-agent completion announcements without relying on `ANNOUNCE_SKIP`.

## Best Possible Solution

Add a documented deterministic sub-agent announce policy and/or `sessions_spawn` parameter, thread it through spawn registry, lifecycle, cleanup, and delivery paths, and keep broader parent-routing or per-channel behavior in separate linked follow-ups if out of scope.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection on current main shows the requested public config/per-spawn control is absent and current suppression is still token/internal-flag based.

## Solution Assessment

Unclear until maintainers choose the public contract. A framework-level deterministic policy is the right boundary, but it should be designed alongside the adjacent routing and per-channel requests so cleanup and delivery semantics stay consistent.
