## Summary

Keep open. Current main and the latest v2026.6.6 release still lack a documented config option or public `sessions_spawn` parameter for deterministic sub-agent announce suppression; the only supported suppression remains token/model-output driven or an internal lifecycle flag. Related closed items route back to this older issue, while the open parent-routing and per-channel items overlap but do not replace this narrower skip/suppress request.

## What This Changes

Request a framework-level config option or `sessions_spawn` parameter that deterministically suppresses sub-agent completion announces instead of relying on model-emitted skip tokens.

## Best Possible Solution

Define one backward-compatible public announce policy, likely a per-spawn skip control with an optional config default only if maintainers approve that extra surface; apply it before announce delivery, treat intentional suppression as successful cleanup, and update schema, docs, and delivery tests with the broader routing items kept consistent.

## Reproduction Assessment

Yes at source level: current main exposes no public `sessions_spawn` announce/silent parameter and no `agents.defaults.subagents.announcePolicy`, while docs still describe token-based suppression. I did not run a live Telegram reproduction because the missing public surface is directly visible in source and docs.

## Solution Assessment

Unclear. A per-spawn skip control is the narrowest likely useful repair, but config defaults, parent routing, and per-channel policy overlap enough that maintainers should settle the public contract before implementation.
