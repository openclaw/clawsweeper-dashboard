## Summary

Keep open. Current main still lacks a documented framework-level config option or public native `sessions_spawn` parameter for deterministic sub-agent announce suppression; the remaining work is a product/API feature, not an already-implemented cleanup close.

## What This Changes

Request adds a deterministic config or per-spawn option to suppress native sub-agent completion announces instead of relying on `ANNOUNCE_SKIP` or `NO_REPLY`.

## Best Possible Solution

Add a documented public suppression policy under `agents.defaults.subagents` and/or native `sessions_spawn`, thread it through spawn registration, lifecycle cleanup, and announce delivery, and keep it compatible with the open per-channel and parent-routing follow-ups.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection gives high-confidence evidence that current main lacks the requested supported config/tool parameter and still relies on token or internal lifecycle suppression for public behavior.

## Solution Assessment

Unclear until maintainers choose the public API shape. A maintainable fix should avoid model-token enforcement, treat intentional suppression as successful cleanup, and align with #13911 and #27445 rather than creating a conflicting one-off option.
