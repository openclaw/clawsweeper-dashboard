## Summary

Keep open. Current main and the latest v2026.6.9 release still lack a documented config key or public `sessions_spawn` parameter for deterministic sub-agent announce suppression; suppression remains token/internal-state driven, and related open issues cover broader routing or per-channel policy rather than replacing this narrower request.

## What This Changes

The issue requests a deterministic framework-level way to suppress sub-agent completion announcements, either via sub-agent defaults config or a public `sessions_spawn` option.

## Best Possible Solution

Maintainers should choose one backward-compatible public announce suppression contract, likely a per-spawn skip control plus optional config default only if approved, then thread it through spawn registration, announce delivery, schema/help/docs, and cleanup bookkeeping as an intentional successful skip.

## Reproduction Assessment

Yes. Current main can be checked at source level: the public spawn schema and strict config schemas lack the requested suppression control, while announce suppression remains token-driven.

## Solution Assessment

Unclear. A per-spawn skip flag is the narrowest apparent repair, but config defaults, parent routing, and per-channel policy overlap enough that maintainers should settle the public contract before implementation.
