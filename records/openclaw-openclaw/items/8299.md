## Summary

Keep open. Current main still lacks a documented public config key or `sessions_spawn` parameter for deterministic sub-agent announce suppression, and the remaining work is a feature/API decision rather than cleanup-close material.

## What This Changes

Request a framework-level way to suppress sub-agent completion announcements, either through `agents.defaults.subagents` config or a per-spawn `sessions_spawn` option.

## Best Possible Solution

Add a backward-compatible announce-delivery policy that defaults to current behavior, supports deterministic skip through public config and/or `sessions_spawn`, and coordinates with the open routing and per-channel designs.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high confidence that current main has no public config key or `sessions_spawn` parameter for deterministic native sub-agent announce suppression.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is to thread one policy through tool schema, config schema/types, registry/lifecycle state, announce delivery, docs, and regression tests rather than relying on model output tokens.
