Requests a framework-level config or per-spawn control to deterministically suppress sub-agent completion announcements.

## Best Possible Solution

Define one closed completion-audience contract with current channel delivery as the default and explicit parent and none modes, then thread that result through spawn registration, delivery, successful cleanup bookkeeping, config/schema/help, documentation, and regression coverage.

No, the proposed boolean and policy sketches establish the requirement but are not yet the best permanent API. A unified completion-audience mode would cover deterministic skip while avoiding conflict with parent-routing and per-channel work.
