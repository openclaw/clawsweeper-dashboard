## Summary

Keep open because current main still lacks a documented framework-level config option or native `sessions_spawn` parameter that deterministically suppresses native sub-agent completion announces; the remaining work is a public API/config feature, not stale cleanup.

## What This Changes

Request adding a framework-level config option and/or `sessions_spawn` parameter to deterministically suppress sub-agent completion announcements.

## Best Possible Solution

Add a backward-compatible native subagent announce policy that defaults to current behavior, supports deterministic skip without token reliance, treats intentional suppression as successful cleanup, and updates schema, docs, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection of current main gives high-confidence evidence that no supported config or native `sessions_spawn` suppression option exists.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is a framework-level policy that avoids model-token enforcement and coordinates with the per-channel and parent-routing requests.
