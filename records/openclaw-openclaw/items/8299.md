## Summary

Keep open. Current main and the latest shipped tag still lack a documented framework-level config or public `sessions_spawn` parameter for deterministic sub-agent announce suppression; the remaining work needs a maintainer API decision because the narrow skip control overlaps open parent-routing and per-channel policy requests.

## What This Changes

Add a public config option and/or `sessions_spawn` parameter that deterministically suppresses sub-agent completion announces without relying on model replies like `ANNOUNCE_SKIP` or `NO_REPLY`.

## Best Possible Solution

Add a backward-compatible public subagent completion policy under `agents.defaults.subagents` and/or `sessions_spawn`, default to current behavior, support deterministic skip, and thread it through registry, lifecycle, delivery, docs, generated config schema/help, and regression tests while coordinating with the open routing and per-channel issues.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source and shipped-tag inspection give high confidence that the requested public config/per-spawn suppression policy is still absent.

## Solution Assessment

Yes at the direction level: structural code/config enforcement is the maintainable way to avoid model-token reliance. The exact option names and composition with parent routing and per-channel policy still need maintainer product/API judgment.
