## Summary

Keep open. Current main still lacks a documented config or public `sessions_spawn` parameter for deterministic sub-agent announce suppression; existing suppression is token/internal-state driven, and related issues track broader routing/per-channel variants rather than replacing this canonical skip request.

## What This Changes

Request a framework-level `agents.defaults.subagents` policy or per-spawn `sessions_spawn` argument to skip sub-agent completion announces without relying on `ANNOUNCE_SKIP`.

## Best Possible Solution

Define one backward-compatible suppression contract that defaults to current announce behavior, treats intentional skip as successful cleanup, and coordinates precedence with #13911/#27445.

## Reproduction Assessment

Yes. Current main exposes no public `sessions_spawn` announce/silent parameter and no `agents.defaults.subagents` suppression policy, while docs still describe exact-token suppression only.

## Solution Assessment

Unclear until maintainers choose the public contract; per-spawn suppression is the narrowest fit, while global/per-channel config and parent routing overlap broader product direction.
