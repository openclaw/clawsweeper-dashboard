## Summary

Keep open because current main still has no documented public config option or `sessions_spawn` parameter for deterministic sub-agent announce suppression; recent code improves parent-mediated completion handling, but suppression remains model-token or internal-delivery behavior rather than a user-facing policy.

## What This Changes

Request a framework-level config option or `sessions_spawn` parameter to suppress sub-agent completion announces without relying on `ANNOUNCE_SKIP` or `NO_REPLY`.

## Best Possible Solution

Add a backward-compatible public subagent completion policy that defaults to current behavior, supports deterministic skip, and is coordinated with parent-routing and per-channel policy before updating schema, docs, generated help, delivery paths, and regression tests.

## Reproduction Assessment

No, not as a runtime bug reproduction because this is a feature/config request. Source inspection on current main gives high confidence that no public suppression config or `sessions_spawn` parameter exists.

## Solution Assessment

Yes at the direction level: structural code/config enforcement is the maintainable way to avoid relying on model-token compliance. The exact option names and precedence should be coordinated with https://github.com/openclaw/openclaw/issues/27445 and https://github.com/openclaw/openclaw/issues/13911 before implementation.
