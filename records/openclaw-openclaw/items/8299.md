## Summary

Keep open. Current main still lacks a documented config key or public `sessions_spawn` parameter for deterministic native sub-agent announce suppression, and the related parent-routing/per-channel items are adjacent rather than a replacement for this narrower control.

## What This Changes

The issue requests a framework-level config option or `sessions_spawn` parameter to suppress sub-agent completion announcements without relying on model output tokens.

## Best Possible Solution

Add a backward-compatible announce-delivery policy that defaults to current behavior, supports deterministic skip through public config and/or `sessions_spawn`, and coordinates with the open routing and per-channel suppression designs.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high confidence that current main has no public config key or `sessions_spawn` parameter for deterministic native sub-agent announce suppression.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is to thread one policy through tool schema, config schema/types, registry/lifecycle state, announce delivery, docs, and regression tests rather than relying on model output tokens.
