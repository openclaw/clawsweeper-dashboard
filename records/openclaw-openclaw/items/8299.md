## Summary

Keep open: current main and the latest release still lack a documented config option or public `sessions_spawn` parameter for deterministic sub-agent announce suppression. The implemented suppression path remains token-driven, and the related open routing/per-channel requests do not replace this narrower skip-control request.

## What This Changes

Request a framework-level config option or per-spawn `sessions_spawn` parameter to suppress sub-agent completion announces without relying on `ANNOUNCE_SKIP`.

## Best Possible Solution

Add a backward-compatible public subagent completion policy that defaults to current behavior, supports deterministic skip, and is coordinated with parent-routing and per-channel policy before updating schemas, docs, delivery paths, registry bookkeeping, and regression tests.

## Reproduction Assessment

Yes, as a source-level feature gap rather than a failing runtime bug: current main's public `sessions_spawn` schema and strict subagent config do not expose a deterministic announce-suppression option.

## Solution Assessment

Yes at the direction level: structural code/config enforcement is the maintainable alternative to model-token compliance. The exact option names and precedence should be decided with the open parent-routing and per-channel policy requests before implementation.
