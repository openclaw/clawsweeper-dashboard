## Summary

Keep open. Current main and latest release still lack a documented deterministic sub-agent announce suppression control in either `sessions_spawn` or `agents.defaults.subagents`; the remaining work is a product/API decision, with this issue still the canonical narrow suppression tracker rather than a duplicate of the broader routing threads.

## What This Changes

Request a framework-level config option and/or per-spawn parameter to suppress sub-agent completion announces without relying on `ANNOUNCE_SKIP` model output.

## Best Possible Solution

Define one backward-compatible completion audience/suppression contract for native subagents, defaulting to current behavior, with explicit maintainer-approved precedence across global config, per-spawn overrides, and channel-specific policy.

## Reproduction Assessment

Yes, at source level: current main exposes no public `sessions_spawn` announce/silent parameter and no `agents.defaults.subagents.announcePolicy`, while docs still describe exact-token suppression. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Unclear. The requested direction is valid, but the best fix should be a maintainer-approved completion audience/suppression contract rather than adding one isolated config key that could conflict with parent-routing and per-channel suppression work.
