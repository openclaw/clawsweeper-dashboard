## Summary

Keep open. Current main still lacks a documented public config option or `sessions_spawn` parameter that deterministically suppresses native sub-agent completion announcements, and the related open routing/per-channel issues are complementary rather than a replacement for this narrower suppression request.

## What This Changes

Request adds a framework-level config and/or `sessions_spawn` parameter to suppress sub-agent completion announcements instead of relying on `ANNOUNCE_SKIP` or `NO_REPLY`.

## Best Possible Solution

Add a backward-compatible framework-level sub-agent completion policy that supports deterministic `skip`/suppression through `sessions_spawn` and/or `agents.defaults.subagents`, coordinates naming with parent-routing and per-channel requests, and updates config schema/types, docs, and targeted lifecycle/delivery tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source, docs, and live issue context give high-confidence evidence that the requested public suppression control is absent on current main.

## Solution Assessment

Unclear until maintainers choose the public API shape. A per-spawn override plus `agents.defaults.subagents` default is the narrow maintainable direction, but it should be coordinated with https://github.com/openclaw/openclaw/issues/27445 and https://github.com/openclaw/openclaw/issues/13911 before implementation.
