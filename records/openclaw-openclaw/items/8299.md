## Summary

Keep open. Current main and v2026.6.10 still lack a public config key or `sessions_spawn` parameter for deterministic sub-agent announce suppression; suppression remains token-driven or internal-state-driven, while related open issues cover broader routing or per-channel policy rather than replacing this narrower request.

## What This Changes

The issue requests a framework-level config option or per-spawn parameter to suppress sub-agent completion announces without relying on `ANNOUNCE_SKIP`.

## Best Possible Solution

Add a backward-compatible public announce suppression contract, preferably a narrow per-spawn skip control plus an optional config default only if maintainers approve it, then thread it through spawn registration, lifecycle cleanup, announce delivery, schema/docs, and regression tests as an intentional successful skip.

## Reproduction Assessment

Yes. Source inspection is enough here: current main exposes no public `sessions_spawn` announce/silent parameter and no `agents.defaults.subagents.announcePolicy`, while announce suppression is still tied to sentinel tokens or an internal boolean.

## Solution Assessment

Unclear. A per-spawn skip control is the narrowest apparent fix, but global config, per-channel policy, and parent-routing semantics overlap enough that maintainers should settle the durable public contract first.
