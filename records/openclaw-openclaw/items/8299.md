## Summary

Keep open. Current main and v2026.6.6 still lack a public `sessions_spawn` parameter or `agents.defaults.subagents` policy for deterministic sub-agent announce suppression; the only supported suppression remains token/model-output driven, and prior implementation PRs were closed unmerged.

## What This Changes

The issue requests a framework-level config option or `sessions_spawn` parameter to suppress sub-agent completion announcements without relying on model-emitted skip tokens.

## Best Possible Solution

Define one backward-compatible sub-agent completion-announcement policy that defaults to current delivery, exposes deterministic per-spawn suppression, and adds config only if maintainers approve the extra public surface and its precedence with parent/per-channel routing.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.6 expose no public `sessions_spawn` announce/silent parameter and no `agents.defaults.subagents.announcePolicy`, while docs still describe exact-token suppression only.

## Solution Assessment

Unclear pending maintainer API direction. Deterministic framework suppression is the right product problem to solve, but the maintainable shape should be coordinated with parent routing and per-channel policy rather than adding a one-off boolean/config key.
