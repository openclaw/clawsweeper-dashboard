## Summary

Keep open. Current main and the latest v2026.6.8 release still do not expose a documented config key or public sessions_spawn parameter for deterministic sub-agent announce suppression; suppression remains internal or token/model-output driven, and related open items cover broader routing/per-channel policy rather than replacing this narrower request.

## What This Changes

The issue requests a framework-level config option or sessions_spawn parameter to suppress sub-agent completion announces without relying on ANNOUNCE_SKIP.

## Best Possible Solution

Maintainers should choose one backward-compatible public announce policy, likely a per-spawn skip control plus an optional config default only if approved, then apply it before delivery and treat intentional suppression as successful cleanup.

## Reproduction Assessment

Yes. Current main exposes no documented sessions_spawn suppress parameter and no strict config key for announce suppression, so the missing deterministic control is source-reproducible without a live Telegram run.

## Solution Assessment

Unclear. A per-spawn skip control is the narrowest likely repair, but config defaults, parent routing, and per-channel policy overlap enough that maintainers should settle the public contract before implementation.
