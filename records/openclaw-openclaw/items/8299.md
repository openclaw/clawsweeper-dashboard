## Summary

Keep open because current main still lacks a documented framework-level config option or native `sessions_spawn` parameter that deterministically suppresses native sub-agent completion announces; the remaining work is a public API/config feature, not stale cleanup.

## What This Changes

Request a framework-level config option and/or `sessions_spawn` parameter to suppress sub-agent completion announcements without relying on `ANNOUNCE_SKIP` or `NO_REPLY`.

## Best Possible Solution

Add a backward-compatible public announce policy for native subagents that defaults to current behavior, supports deterministic skip, treats intentional suppression as successful cleanup, updates schema/docs/tests, and coordinates naming with the open channel and parent-routing requests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection of current main gives high-confidence evidence that no supported config or native `sessions_spawn` suppression option exists.

## Solution Assessment

Unclear until maintainers choose the public API shape. The maintainable direction is a framework-level policy that avoids model-token enforcement and stays compatible with the per-channel and parent-routing requests.
