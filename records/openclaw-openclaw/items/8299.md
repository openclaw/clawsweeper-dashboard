## Summary

Keep open: current main still has no supported deterministic sub-agent announce suppression setting or public spawn parameter, and the related open requests are complementary rather than replacements.

## What This Changes

The issue requests a framework-level config option or per-spawn parameter to suppress sub-agent completion announcements without relying on model output tokens.

## Best Possible Solution

Add a backward-compatible native subagent announce policy that defaults to current behavior, supports deterministic skip without token reliance, treats intentional suppression as successful cleanup, and coordinates naming with https://github.com/openclaw/openclaw/issues/13911 and https://github.com/openclaw/openclaw/issues/27445.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection on current main gives high-confidence evidence that no supported config option or public sessions_spawn parameter exists for deterministic suppression.

## Solution Assessment

Unclear until maintainers choose the public API shape. A framework-level policy is the right class of solution, but it should be designed alongside the per-channel suppression and parent-routing requests.
