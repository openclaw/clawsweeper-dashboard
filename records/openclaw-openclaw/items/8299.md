## Summary

Keep open. Current main and the latest shipped tag still lack a documented config option or public `sessions_spawn` parameter for deterministic sub-agent announce suppression; the closed implementation attempts were not merged, and the open parent-routing/per-channel requests are adjacent API-shape work rather than replacements for this narrower skip control.

## What This Changes

The issue requests a framework-level config option or `sessions_spawn` parameter to suppress sub-agent completion announcements without relying on `ANNOUNCE_SKIP` model compliance.

## Best Possible Solution

Add a backward-compatible public subagent completion policy that defaults to current behavior, supports deterministic skip, and is coordinated with parent-routing and per-channel policy before updating schema, docs, generated help, lifecycle delivery, and regression tests.

## Reproduction Assessment

No, not as a bug reproduction; this is a feature/config request. Source inspection on current main and `v2026.5.20` gives high confidence that no public suppression config or `sessions_spawn` parameter exists.

## Solution Assessment

Yes at the direction level: structural code/config enforcement is the maintainable way to avoid relying on model-token compliance. The exact option names and composition with https://github.com/openclaw/openclaw/issues/27445 and https://github.com/openclaw/openclaw/issues/13911 still need maintainer product/API judgment.
