## Summary

Keep open. Current main and the latest shipped tag still lack a documented config option or public `sessions_spawn` parameter for deterministic sub-agent announce suppression; suppression remains model-token driven, and related routing/per-channel requests are still open API/product decisions.

## What This Changes

The issue requests a framework-level config or per-spawn control to suppress sub-agent completion announcements instead of relying on model output tokens.

## Best Possible Solution

Add a backward-compatible public subagent completion policy under `agents.defaults.subagents` and/or `sessions_spawn`, preserve current behavior by default, support deterministic skip, and coordinate the API with parent-routing and per-channel policy trackers.

No, not as a runtime bug reproduction; this is a feature/config request. Source and shipped-tag inspection give high confidence that the requested public config or per-spawn suppression policy is still absent.

## Solution Assessment

Yes at the direction level: structural code/config enforcement is the maintainable way to avoid relying on model-token compliance. The exact option names and composition with parent routing and per-channel policy still need maintainer product/API judgment.
