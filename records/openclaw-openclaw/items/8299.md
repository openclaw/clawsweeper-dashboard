## Summary

Keep open. Current main still lacks a documented config option or public `sessions_spawn` parameter to deterministically suppress sub-agent completion announces; prior implementation PRs were closed unmerged and adjacent routing requests remain open, so this needs maintainer API/product direction rather than cleanup closure.

## What This Changes

Request a framework-level sub-agent announce suppression control, either through `agents.defaults.subagents` config or a `sessions_spawn` parameter.

## Best Possible Solution

Add a backward-compatible announce delivery policy that defaults to current behavior, supports deterministic skip through public config and/or `sessions_spawn`, coordinates with parent-routing work, and updates schema, generated config docs, user docs, and focused lifecycle/delivery tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high confidence that current main has no public config key or `sessions_spawn` parameter for deterministic sub-agent announce suppression.

## Solution Assessment

Unclear until maintainers choose the public primitive. The maintainable direction is a tool/config policy threaded through registry, lifecycle, and delivery, coordinated with https://github.com/openclaw/openclaw/issues/13911 and https://github.com/openclaw/openclaw/issues/27445.
