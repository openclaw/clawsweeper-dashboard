## Summary

Keep open: current main and the latest release still lack a documented public config option or `sessions_spawn` parameter for deterministic sub-agent announce suppression. The only supported suppression path remains token-driven, while the open routing and per-channel issues are adjacent rather than a replacement for this narrower skip control.

## What This Changes

The issue requests a framework-level config option or per-spawn control to suppress sub-agent completion announces without relying on `ANNOUNCE_SKIP`.

## Best Possible Solution

Add a backward-compatible public subagent completion policy that defaults to current behavior, supports deterministic skip, and is coordinated with parent-routing and per-channel policy before updating schemas, docs, delivery paths, registry bookkeeping, and regression tests.

## Reproduction Assessment

Yes. This is source-reproducible as a feature gap: current main's public `sessions_spawn` schema, strict subagent config, and docs do not expose a deterministic announce-suppression option.

## Solution Assessment

Yes at the direction level: structural code/config enforcement is the maintainable alternative to model-token compliance. The exact public API and precedence are still a maintainer product decision because adjacent parent-routing and per-channel requests remain open.
