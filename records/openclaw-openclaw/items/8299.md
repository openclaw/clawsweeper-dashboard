## Summary

Keep open: current main still lacks a documented config option or public native `sessions_spawn` parameter that deterministically suppresses sub-agent completion announcements. The existing supported suppression path remains token/model-output driven, and the related open items cover complementary per-channel and parent-routing designs rather than replacing this narrower request.

## What This Changes

Request a framework-level sub-agent announce suppression control, either in `agents.defaults.subagents` or as a `sessions_spawn` parameter.

## Best Possible Solution

Add a backward-compatible native subagent announce policy that defaults to current behavior, supports deterministic skip without token reliance, treats intentional suppression as successful cleanup, and coordinates naming with https://github.com/openclaw/openclaw/issues/13911 and https://github.com/openclaw/openclaw/issues/27445.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection of current main gives high-confidence evidence that no supported config option or native `sessions_spawn` suppression parameter exists.

## Solution Assessment

Unclear until maintainers choose the public API shape. A framework-level policy is the right class of solution, but it should be designed alongside the per-channel suppression and parent-routing requests.
