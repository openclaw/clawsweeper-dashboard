## What This Changes

The issue requests a framework-level config option or `sessions_spawn` parameter to deterministically suppress sub-agent completion announcements.

## Best Possible Solution

Add a backward-compatible public subagent completion policy under `agents.defaults.subagents` and/or `sessions_spawn`, default to current behavior, support deterministic skip, and thread it through registry, lifecycle, delivery, docs, generated config schema/help, and regression tests while coordinating with open routing and per-channel requests.
