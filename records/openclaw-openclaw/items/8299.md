## Summary

Keep open: current main and v2026.6.10 still lack a public config key or `sessions_spawn` parameter for deterministic sub-agent announce suppression; the only supported suppression remains sentinel-token based or internal-only, and related open issues cover broader routing/per-channel policy rather than replacing this narrower request.

## What This Changes

Request a framework-level config option and/or per-spawn parameter to suppress sub-agent completion announces without relying on `ANNOUNCE_SKIP`.

## Best Possible Solution

Add a backward-compatible public announce suppression contract, preferably a narrow per-spawn skip control plus an optional config default only if maintainers approve it, then thread intentional suppression through lifecycle cleanup and delivery as a successful skip.

## Reproduction Assessment

Yes. Source inspection is enough here: current main exposes no public `sessions_spawn` announce/silent parameter and no `agents.defaults.subagents.announcePolicy`, while announce suppression is still tied to sentinel tokens or an internal boolean.

## Solution Assessment

Unclear. A per-spawn skip control looks like the narrowest implementation, but maintainers should settle how it composes with global defaults, per-channel policy, and parent-routing proposals before code is queued.
