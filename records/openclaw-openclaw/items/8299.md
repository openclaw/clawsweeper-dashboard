## Summary

Keep open: current main and latest release v2026.5.22 still lack a documented public config option or `sessions_spawn` parameter for deterministic sub-agent announce suppression. The only implemented suppression paths are token-driven or internal lifecycle flags, and the adjacent parent-routing and per-channel requests do not replace this narrower skip control.

## What This Changes

The issue requests a framework-level config and/or `sessions_spawn` option to suppress sub-agent completion announcements instead of relying on `ANNOUNCE_SKIP`.

## Best Possible Solution

Add a backward-compatible public subagent completion policy that defaults to current behavior, supports deterministic skip, and is coordinated with parent-routing and per-channel policy before updating schemas, docs, delivery paths, registry bookkeeping, and regression tests.

## Reproduction Assessment

Yes, as a source-level feature gap: current main's public `sessions_spawn` schema and strict subagent config do not expose a deterministic announce-suppression option. The runtime suppression path still depends on `ANNOUNCE_SKIP` or `NO_REPLY` output rather than a public policy.

## Solution Assessment

Yes at the direction level: structural code/config enforcement is the maintainable alternative to model-token compliance. The exact option names, precedence, and relationship to parent/per-channel routing still need maintainer product/API choice before implementation.
