## Summary

Keep open. Current main and the latest shipped tag still lack a documented config option or public `sessions_spawn` parameter for deterministic sub-agent announce suppression; the only supported suppression remains model/token driven, and the API shape overlaps open routing and per-channel policy requests.

## What This Changes

Request for a framework-level config option or per-spawn parameter to deterministically suppress sub-agent completion announcements.

## Best Possible Solution

Add a backward-compatible public subagent completion policy under `agents.defaults.subagents` and/or `sessions_spawn`, default to current behavior, support deterministic skip, and thread it through registry, lifecycle, delivery, docs, generated config schema/help, and regression tests while coordinating with the open routing and per-channel requests.

## Solution Assessment

Yes at the direction level: structural code/config enforcement is the maintainable way to avoid relying on model-token compliance. The exact option names and composition with parent routing and per-channel policy still need maintainer product/API judgment.
