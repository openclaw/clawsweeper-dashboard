## Summary

Keep open; Codex review found the active memory-core keyword path still filters FTS rows by the current embedding model on main and in the latest release, and the linked implementation pull request remains open and conflicting.

## What This Changes

The issue asks for `memory_search` hybrid mode to return lexical FTS matches instead of empty results when matching keyword rows already exist in the SQLite FTS index.

## Best Possible Solution

Land or replace the linked PR so FTS keyword search ignores embedding model while vector search stays model-scoped, with regression coverage for mismatched models and redacted real `memory_search` proof.

## Reproduction Assessment

Yes for source-level reproduction: seed `chunks_fts` with a row whose stored `model` differs from the active provider model, then call the keyword helper with `providerModel` set. Current main appends `AND model = ?`, filtering the lexical row before hybrid merge.

## Solution Assessment

Yes in direction: lexical FTS should drop the embedding-model predicate while vector search remains model-scoped. The current linked PR is an implementation candidate, but it is conflicting and targets obsolete `src/memory` paths, so it likely needs retargeting or replacement.
