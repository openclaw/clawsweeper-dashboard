## What This Changes

The issue reports that `memory_search` hybrid mode drops lexical FTS keyword hits despite matching rows in the SQLite FTS index.

## Best Possible Solution

Land or replace the linked PR against `extensions/memory-core` so lexical FTS ignores embedding model while vector search remains model-scoped, with regression coverage and redacted real `memory_search` proof.

## Reproduction Assessment

Yes for source-level reproduction: seed `chunks_fts` with a row whose `model` differs from the active provider and call the active keyword helper with `providerModel` set. Current main appends `AND model = ?`, filtering that lexical row before hybrid merge.

## Solution Assessment

Yes in direction: lexical FTS should drop the embedding-model predicate while vector search stays model-scoped. The current linked PR is not ready as the fix path because it is open, conflicting, and targets obsolete `src/memory` files.
