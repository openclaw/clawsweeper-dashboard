## Summary

Keep open; current main and v2026.5.22 still apply the active embedding model as a predicate on lexical FTS rows before hybrid merge, so exact FTS hits indexed under another embedding model can still be dropped. The reporter's stale-sync/default concern is now mitigated by current sync defaults, but the remaining model-filtered FTS bug is still source-reproducible and the target-repo fix PR was closed unmerged.

## What This Changes

The issue reports `memory_search` hybrid mode returning empty results for exact FTS matches even though `chunks_fts` contains matching rows.

## Best Possible Solution

Patch active memory-core keyword search so lexical FTS MATCH and LIKE fallback paths ignore the embedding model, while vector search remains model-scoped and regression coverage proves cross-model FTS hits survive hybrid merge.

## Reproduction Assessment

Yes at source level: seed `chunks_fts` with a row whose stored `model` differs from the active provider model, then call the active keyword helper with `providerModel` set. Current main appends `AND model = ?`, filtering the lexical row before hybrid merge.

## Solution Assessment

Yes in direction: lexical FTS should drop the embedding-model predicate while vector search stays model-scoped. The closed implementation PR is not the best path now because it targeted obsolete `src/memory` files instead of active `extensions/memory-core` files.
