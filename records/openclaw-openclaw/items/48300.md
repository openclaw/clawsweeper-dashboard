## Summary

Keep open: current main still has a source-visible cross-model FTS drop path, and the open linked fix PR is unmerged and conflicting. The reporter's follow-up makes the original local setup partly about stale gateway state, but the narrower provider/model FTS bug remains plausible and already has an implementation candidate to review.

## What This Changes

The issue reports that `memory_search` hybrid mode can omit FTS keyword hits even when `chunks_fts MATCH` returns the expected memory rows.

## Best Possible Solution

Port or replace the linked fix in `extensions/memory-core`, remove the model predicate only from FTS keyword search, keep vector search model-scoped, add current-path regression coverage, and land it with real `memory_search` proof.

## Reproduction Assessment

Yes for the narrower cross-model bug from source inspection: seed an FTS row with a different `model` than the active provider and `searchKeyword` will append `AND model = ?` and drop it. I did not live-reproduce the reporter's stale gateway state.

## Solution Assessment

Yes, the narrowest maintainable fix is to remove model filtering from lexical FTS search while keeping it for vector comparisons. Changing sync defaults is not the core fix here, and those defaults are already `true` on current main.
