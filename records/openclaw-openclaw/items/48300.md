## Summary

Keep open: current main still has a source-level path where hybrid FTS keyword search is filtered by the active provider model, and an open linked PR is already targeting that exact issue. This should wait for review, landing, or explicit closure of that PR rather than being swept now.

## What This Changes

The issue asks for hybrid `memory_search` to return SQLite FTS keyword matches instead of `[]` when matching FTS rows exist.

## Best Possible Solution

Review or replace the linked fix against the current `extensions/memory-core` paths, letting FTS keyword search span indexed models while keeping model filtering for vector search and adding cross-model regression coverage.

## Reproduction Assessment

Yes for the provider/model mismatch variant: seed an FTS row under one model and call `searchKeyword` with a different active provider model, and the current SQL appends `AND model = ?`. I did not run a full Ollama gateway reproduction.

## Solution Assessment

Yes. Removing the model predicate only from FTS keyword search is the narrow maintainable fix; vector search should keep model filtering because embedding spaces are model-specific.
