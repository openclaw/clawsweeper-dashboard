## Summary

Keep open: current main still model-scopes provider-backed FTS keyword search, so exact FTS matches indexed under a different embedding model can be hidden before the hybrid merge; open PR #48328 remains the unmerged linked fix candidate.

## What This Changes

The issue reports that `memory_search` hybrid mode can return no exact keyword/FTS results even when the SQLite FTS index contains matching chunks.

## Best Possible Solution

Review and land #48328 or a current-path replacement that makes lexical FTS recall independent of embedding provider/model while keeping vector search model-scoped, with active regression coverage for hybrid exact-keyword recall across model changes.

## Reproduction Assessment

Yes. The report includes concrete steps and direct SQLite MATCH proof, and current source shows a high-confidence path where provider-backed hybrid search can drop matching FTS rows by applying `AND model = ?` before the merge.

## Solution Assessment

Yes. The narrow maintainable fix is to adjust the memory-core keyword search contract and add regression tests, rather than changing plugin APIs or adding a new config knob.
