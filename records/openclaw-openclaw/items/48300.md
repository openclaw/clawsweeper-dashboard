## Summary

Keep this open. Current main improves stale-index behavior, but the provider-backed hybrid FTS path still filters keyword rows by the active embedding model, and open PR #48328 remains the linked implementation candidate rather than a merged fix.

## What This Changes

The issue reports that `memory_search` in hybrid mode fails to return SQLite FTS keyword matches even though matching rows exist in the FTS index.

## Best Possible Solution

Keep this issue open until #48328 or a current-path replacement is reviewed. The fix should make lexical FTS recall independent of embedding provider/model where appropriate, keep vector search model-scoped, and add active regression coverage for hybrid exact-keyword recall across provider/model changes.

## Reproduction Assessment

Yes. The issue includes concrete steps and direct SQLite proof, and current source shows a high-confidence code path where provider-backed hybrid search can drop otherwise matching FTS rows by applying `AND model = ?` before the hybrid merge.

## Solution Assessment

Yes, the narrow maintainable direction is to fix the memory-core keyword search contract and add regression tests, not to change core plugin APIs. Current main is not the final solution because it still couples lexical FTS lookup to the active embedding model.
