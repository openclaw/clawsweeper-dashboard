## Summary

Keep open: current main still filters active FTS keyword SQL by the current provider model, so exact FTS rows indexed under another model can be dropped before hybrid merge. The linked fix PR already references this issue, but it still targets obsolete `src/memory` files rather than active `extensions/memory-core` code.

## What This Changes

The issue reports that hybrid `memory_search` drops SQLite FTS keyword matches even when the FTS index contains matching chunks.

## Best Possible Solution

Review, retarget, or replace https://github.com/openclaw/openclaw/pull/48328 so active `extensions/memory-core` FTS keyword search drops provider/model filtering for MATCH and LIKE fallback paths while vector search remains model-specific.

## Reproduction Assessment

Yes for a source-level path: seed an FTS row under one model and call active keyword search with a different `providerModel`; current main appends `AND model = ?` before returning keyword rows. I did not run the full Ollama gateway reproduction.

## Solution Assessment

Yes for the fix direction: remove the provider-model predicate from lexical FTS only, while keeping vector search model-bound. The current linked PR must be retargeted from old `src/memory` paths to active `extensions/memory-core` before it can solve the issue.
