- `impact:session-state`: The bug affects memory recall state by returning empty search results despite indexed lexical content existing.

## Summary

Keep open; current main and v2026.5.20 still apply the active embedding model as a predicate on lexical FTS rows before hybrid merge, so exact keyword matches indexed under a different model can still disappear. The linked implementation PR is open but conflicting and currently targets obsolete memory paths, so this issue should stay open until that PR is retargeted, replaced, or closed.

## What This Changes

The issue asks for hybrid `memory_search` to return FTS keyword matches when indexed lexical data exists, instead of returning an empty result set.

## Best Possible Solution

Land or replace the linked PR so active memory-core lexical MATCH and LIKE fallback paths ignore the embedding model while vector search remains model-scoped, with regression coverage and redacted real `memory_search` proof.

## Reproduction Assessment

Yes for source-level reproduction: seed `chunks_fts` with a row whose stored `model` differs from the active provider model, then call the active keyword helper with `providerModel` set. Current main appends `AND model = ?`, filtering the lexical row before hybrid merge.

## Solution Assessment

Yes in direction: lexical FTS should drop the embedding-model predicate while vector search stays model-scoped. The current linked PR is only an implementation candidate because it is conflicting and targets obsolete `src/memory` paths.
