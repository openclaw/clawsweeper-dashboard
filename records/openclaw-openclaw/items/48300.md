## Summary

Keep open. Current main still filters FTS keyword results by the current embedding model, so exact FTS hits indexed under another model can be hidden; the open closing PR at https://github.com/openclaw/openclaw/pull/48328 is the right implementation path to review or rebase rather than closing this issue early.

## What This Changes

The issue asks for hybrid `memory_search` to return lexical FTS matches that exist in the SQLite FTS index instead of dropping them from results.

## Best Possible Solution

Land a current-layout equivalent of https://github.com/openclaw/openclaw/pull/48328 that removes model filtering from FTS keyword search while preserving model filtering for vector search and adds the cross-model regression test.

## Reproduction Assessment

Yes from source inspection. Seed an FTS row with a model different from the active provider model, then call `searchKeyword` with a provider model; the current query appends `AND model = ?` and drops the lexical hit.

## Solution Assessment

Yes. The narrow maintainable fix is to remove the provider/model predicate from FTS keyword search only, while leaving vector search model filtering intact because vector embeddings remain model-specific.
