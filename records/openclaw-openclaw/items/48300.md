Fix `memory_search` hybrid keyword recall so FTS/BM25 matches remain visible when chunks were indexed under a different embedding provider or model.

## Best Possible Solution

Land or replace the linked PR against active `extensions/memory-core` so lexical FTS ignores embedding model while vector search stays model-scoped, with regression coverage and real `memory_search` proof.
