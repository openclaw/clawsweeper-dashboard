## Summary

Keep open: current main still has a source-level path where active hybrid keyword search filters FTS rows by the current embedding model, and an open linked PR already references this issue as its fix path.

## What This Changes

The issue asks for hybrid `memory_search` to return FTS keyword matches when `chunks_fts` contains matching rows.

## Best Possible Solution

Review, retarget, or replace https://github.com/openclaw/openclaw/pull/48328 so the FTS model-filter removal lands in `extensions/memory-core`, covers MATCH and LIKE fallback paths, and preserves vector-search model filtering.

## Reproduction Assessment

Yes for a source-level reproduction: seed an FTS row under one model and call the active keyword helper with a different `providerModel`; current main appends `AND model = ?` to the keyword query. I did not run the full Ollama gateway reproduction.

## Solution Assessment

Yes for the fix direction: remove the provider-model predicate from FTS keyword search only, while keeping vector search model-specific. The currently linked PR needs retargeting from old `src/memory` paths to active `extensions/memory-core` before it can solve the issue.
