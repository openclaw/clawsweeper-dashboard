## Summary

Keep open: current main still applies the active provider model as a lexical FTS filter, and the open closing PR that references this issue is conflicting and edits obsolete `src/memory` files instead of the active `extensions/memory-core` implementation.

## What This Changes

The issue reports that hybrid `memory_search` returns empty results for exact FTS keyword matches even though matching rows exist in the SQLite FTS index.

## Best Possible Solution

Review, retarget, or replace https://github.com/openclaw/openclaw/pull/48328 so active `extensions/memory-core` lexical FTS search drops provider/model filtering for MATCH, LIKE fallback, and substring paths while vector search remains model-specific.

## Reproduction Assessment

Yes for a source-level path: seed `chunks_fts` with a row under one model and call the active keyword helper with a different `providerModel`; current main appends `AND model = ?` and excludes that lexical hit. I did not run the full Ollama gateway reproduction.

## Solution Assessment

Yes for the fix direction: remove the provider-model predicate from lexical FTS while keeping vector search model-bound. The current open PR is not sufficient as written because it edits obsolete `src/memory` files instead of active `extensions/memory-core` paths.
