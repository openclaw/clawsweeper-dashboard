## What This Changes

The issue reports that running `/pair qr` from Control UI webchat can persist `gateway.bind=tailnet`, restart the Gateway, and leave an existing Tailscale Serve Control UI route proxying loopback with HTTP 502 failures.

## Best Possible Solution

Keep pairing URL generation non-mutating, isolate any current path that persists `gateway.bind=tailnet`, and require explicit operator confirmation or a supported managed Serve/public URL path before any disruptive Gateway bind change.

## Reproduction Assessment

No high-confidence current-main reproduction for the full `/pair qr` trigger yet. The broken route state is source-reproducible once `gateway.bind=tailnet` is persisted because loopback is no longer served, but the silent writer path remains unclear.
