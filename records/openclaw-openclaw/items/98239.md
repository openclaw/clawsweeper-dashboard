## What This Changes

The issue reports that `/pair qr` from Control UI webchat can change `gateway.bind`, restart the Gateway, and break an existing Tailscale Serve loopback proxy for webchat.

## Best Possible Solution

Keep pairing URL generation non-mutating, block or require explicit confirmation for `gateway.bind` changes that would break an existing loopback/Tailscale Serve route, and return actionable Serve/publicUrl guidance instead.

## Reproduction Assessment

No high-confidence current-main repro was established; the issue gives strong v2026.6.10 field logs, but current main’s direct `/pair qr` path appears to stop as a non-mutating plugin command.
