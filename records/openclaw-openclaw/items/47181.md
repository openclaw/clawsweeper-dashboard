Full review comments:

- **[P2] Reject unsupported Azure endpoint paths:** `src/commands/anthropic-azure-utils.ts:60-62`
  - body: When a full URL path does not already end with `/anthropic`, this code appends the suffix instead of rejecting the unexpected path. For example, `https://fabric-hub.services.ai.azure.com/anthropic/v1` is stored as `.../anthropic/v1/anthropic`, so onboarding succeeds with a malformed runtime base URL.
  - confidence: 0.92
- **[P2] Validate Azure resource labels in URL input:** `src/commands/anthropic-azure-utils.ts:58`
  - body: The full-URL branch only checks that the host ends with `.services.ai.azure.com`. Hosts such as `fabric_hub.services.ai.azure.com` bypass the bare-resource validation and persist an invalid Azure resource host that will fail later at runtime.
  - confidence: 0.9
- **[P2] Honor generic token input for API-key routing:** `src/commands/auth-choice.apply.api-providers.ts:176-178`
  - body: `--auth-choice apiKey --token-provider anthropic-azure --token ...` normalizes into this Azure branch, but the branch reads only `anthropicAzureApiKey`. Existing scripted API-key flows using the generic `--token` value are ignored and fall back to env or prompts.
  - confidence: 0.87

  - body: This PR adds a user-facing onboarding feature plus CLI and docs surface, but it does not update `CHANGELOG.md`. Repo policy requires a changelog entry for user-facing feature changes before merge.
