## What This Changes

The PR adds an Azure Claude AI Foundry onboarding choice with CLI flags/env vars, URL and model helpers, auth profile/config wiring, tests, and Anthropic docs.

## Best Possible Solution

Move the Azure Claude path into the Microsoft Foundry provider or a dedicated provider plugin, preserve generic token semantics, validate Foundry endpoints strictly, and require redacted real onboarding proof before merge.

## Reproduction Assessment

Yes for the review findings: source inspection of the PR head shows the generic --token value is ignored after apiKey/tokenProvider normalization, and malformed Foundry URL forms can be accepted. I did not run a live Azure onboarding flow in this read-only review.

## Solution Assessment

No. The feature direction is reasonable, but provider-specific Azure Claude onboarding should be owned by the Microsoft Foundry provider or a dedicated provider plugin rather than adding a new core auth choice with duplicate URL/token handling.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Honor generic token input for Azure API-key auth:** `src/commands/auth-choice.apply.api-providers.ts:177`
  - body: This branch is also reached when `--auth-choice apiKey --token-provider anthropic-azure` normalizes to `anthropic-azure-api-key`, but it passes only `anthropicAzureApiKey` into `ensureApiKeyFromOptionEnvOrPrompt`. Scripted setup that supplies the generic `--token` will be ignored and fall back to env or prompt, unlike the plugin API-key path on current main.
  - confidence: 0.9
- **[P2] Reject unsupported Azure endpoint paths:** `src/commands/anthropic-azure-utils.ts:60-62`
  - body: For full URL input, any path that does not end with `/anthropic` gets `/anthropic` appended. A pasted endpoint like `https://fabric-hub.services.ai.azure.com/anthropic/v1` becomes `.../anthropic/v1/anthropic`, storing a malformed base URL and deferring failure to runtime instead of failing during onboarding.
  - confidence: 0.88
- **[P2] Validate the resource label in URL hosts:** `src/commands/anthropic-azure-utils.ts:53-56`
  - body: The URL branch only checks that the host ends with `.services.ai.azure.com`, so `https://fabric_hub.services.ai.azure.com/anthropic` can pass even though the bare resource path rejects underscores. Apply the same resource-label validation before accepting full URLs.
  - confidence: 0.86
