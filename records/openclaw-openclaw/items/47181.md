## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Reject invalid Azure resource labels in URL input:** `src/commands/anthropic-azure-utils.ts:50-52`
  - body: The URL branch only checks that the hostname ends with `.services.ai.azure.com`, so `https://fabric_hub.services.ai.azure.com/anthropic` is accepted even though the bare-resource branch rejects underscores. That stores an Azure endpoint that cannot resolve and pushes a clear onboarding validation error to runtime instead.
  - confidence: 0.9
- **[P2] Reject unsupported Azure endpoint paths:** `src/commands/anthropic-azure-utils.ts:55-58`
  - body: For full URL input, any path that does not already end with `/anthropic` gets `/anthropic` appended. A copied endpoint like `/anthropic/v1/messages` becomes `/anthropic/v1/messages/anthropic`, and the Anthropic transport will append `/v1/messages` again, so onboarding persists a broken base URL instead of failing fast.
  - confidence: 0.9
