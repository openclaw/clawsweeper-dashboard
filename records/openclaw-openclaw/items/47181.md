## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Honor generic token input for Azure API-key auth:** `src/commands/auth-choice.apply.api-providers.ts:173`
  - body: This branch is reached when `--auth-choice apiKey --token-provider anthropic-azure` normalizes to `anthropic-azure-api-key`, but it passes only `anthropicAzureApiKey` into `ensureApiKeyFromOptionEnvOrPrompt`. Scripted setup that supplies the generic `--token` will be ignored and fall back to env/prompt, unlike the shared plugin API-key path on main.
  - confidence: 0.9
- **[P2] Reject unsupported Azure endpoint paths:** `src/commands/anthropic-azure-utils.ts:60-62`
  - body: For full URL input, any path that does not end with `/anthropic` gets `/anthropic` appended. A pasted endpoint like `https://fabric-hub.services.ai.azure.com/anthropic/v1` becomes `.../anthropic/v1/anthropic`, storing a malformed base URL and deferring failure to runtime instead of failing during onboarding.
  - confidence: 0.88
- **[P2] Validate the resource label in URL hosts:** `src/commands/anthropic-azure-utils.ts:53-56`
  - body: The URL branch only checks that the host ends with `.services.ai.azure.com`, so `https://fabric_hub.services.ai.azure.com/anthropic` can pass even though the bare resource path rejects underscores. Apply the same resource-label validation before accepting full URLs.
  - confidence: 0.86
