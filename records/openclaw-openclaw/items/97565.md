## What This Changes

The branch parses Claude stream-json `thinking_delta` records, threads an optional reasoning callback through CLI/live-session execution, emits those deltas as `thinking` agent events, and switches the CLI reasoning bridge from assistant text to thinking events.

## Best Possible Solution

Keep the Claude `thinking_delta` parser callback, but deliver it only through the existing gated reasoning-preview path and prove both `/reasoning stream` and `/reasoning off` in an edit-capable channel.

## Reproduction Assessment

Yes at source level. PR head emits `thinking` agent events in the CLI runner with no reasoning-mode gate, while current embedded reasoning emission is gated by `reasoningMode === "stream"`, thinking support, and `onReasoningStream`.

## Solution Assessment

No. Parsing `thinking_delta` is the right boundary, but publishing raw thinking from `execute.ts` is too early; the event should be emitted only after the existing reasoning visibility decision or via an explicit gated signal.
