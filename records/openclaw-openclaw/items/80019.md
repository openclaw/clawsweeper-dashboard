## Summary

Keep open: current main still blocks scoped `cron runs` and only exposes self-filtered `list`, while an open closing PR is already the implementation candidate for the remaining scoped-runs gap.

## What This Changes

Request a read-only cron inspection scope for isolated scheduled-job turns covering scheduler status, job listing, and recent run inspection while preserving mutation denials.

## Best Possible Solution

Define the safe isolated read boundary, then land one canonical fix, likely https://github.com/openclaw/openclaw/pull/80022 or a successor, with focused tests and docs for the chosen `status`/`list`/`runs` behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows scoped isolated cron runs can read `status` and self-filtered `list`, but `runs` is still rejected by the guard and locked in by tests.

## Solution Assessment

Unclear. A current-job-only `runs` allowance is a plausible narrow fix, but broader inventory and run-history inspection needs maintainers to choose the safe isolated read scope first.
