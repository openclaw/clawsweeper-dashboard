## What This Changes

The issue reports that the published OpenClaw v2026.4.29 package could not load bundled Discord and Telegram channel modules because extension-owned runtime dependencies were absent until bundled-plugin runtime-deps repair was run.

## Best Possible Solution

A shipped package should stage configured Discord and Telegram runtime dependencies through the extension-owned managed runtime-deps path before any built channel module import, and release validation should fail when built staged manifests or package artifacts miss those dependencies.

## Reproduction Assessment

Yes. The issue gives a high-confidence reproduction against the shipped v2026.4.29 global package: resolve discord-api-types/v10 and grammy from the built Discord/Telegram channel files before repair, then confirm resolution succeeds after running bundled-plugin runtime-deps repair.

## Solution Assessment

Unclear. Root-mirroring all Discord/Telegram dependencies is not the maintainable direction; the safer path is to preserve extension-owned staged deps and add or land package/release proof that configured channel deps are staged before import.
