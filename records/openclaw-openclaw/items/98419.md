## What This Changes

The issue asks OpenClaw to update persisted plugin install records during install-record-only plugin updates without rewriting user-authored `openclaw.json`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/98422 if its config-boundary behavior is correct, then close this issue through the merged PR.

## Reproduction Assessment

Yes at source level: current main commits changed plugin install records through `commitPluginInstallRecordsWithConfig`, whose commit path calls `replaceConfigFile`. I did not run the Windows upgrade reproduction in this read-only review.

## Solution Assessment

Yes: the narrow best fix is to persist install records and refresh plugin metadata without replacing `openclaw.json` when the authored source config did not change, while keeping config writes for real legacy-record cleanup or hook/config mutations.
