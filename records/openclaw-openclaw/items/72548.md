## Summary

Keep open; the feature is useful, but the branch is not mergeable because the advertised third-party SecretRef provider seam still validates sources that runtime cannot load, CI has concrete branch failures, and external real behavior proof is still missing.

## What This Changes

The PR opens SecretRef source/provider schemas, adds a secret-provider SDK and manifest contract, loads bundled secret-provider artifacts, and adds bundled GCP and OS keyring plugins with docs and tests.

## Best Possible Solution

Complete enabled bundled, installed, and workspace provider resolution through the plugin registry/public-artifact boundary, or deliberately narrow the docs/types/changelog to bundled-only, then fix CI and add inspectable real behavior proof before merge.

## Reproduction Assessment

Yes for the review findings. Source inspection of the PR head shows non-bundled plugin sources can pass the new open schema while runtime lookup only consults bundled loaders, and CI logs show concrete branch failures.

## Solution Assessment

No. The manifest-first direction fits the plugin architecture, but the current branch is incomplete unless it either supports non-bundled providers at runtime or narrows the advertised scope to bundled-only with a tracked follow-up.
