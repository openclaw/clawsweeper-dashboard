## Summary

Keep this PR open, but it is not merge-ready. The branch is active and useful, yet the latest head still has blocking correctness findings, failing CI, missing inspectable real behavior proof for the claimed live secret-backend smokes, and an unresolved product/API scope question around bundled-only versus third-party plugin provider loading.

## What This Changes

The PR adds a manifest-declared SecretRef provider seam, a public plugin SDK subpath, bundled GCP and OS keyring secret-provider plugins, schema/protocol changes, docs, generated baselines, and related tests.

## Best Possible Solution

Land a security-reviewed SecretRef provider seam that either supports enabled bundled, installed, and workspace plugin providers through the public-artifact boundary, or explicitly scopes this first step to bundled GCP/keyring providers with docs/types narrowed and follow-up issues for third-party loading and broader SecretRef field coverage.

## Reproduction Assessment

Yes for the PR review blockers: CI reproduces the dependency and cron diagnostic failures, and source inspection shows the docs/schema accept third-party plugin-owned sources while runtime lookup is bundled-only. Reproduction is not a current-main bug path because this is an additive feature PR.

## Solution Assessment

No. The manifest-first direction fits OpenClaw's plugin architecture, but the current patch over-advertises third-party support, changes an existing validation diagnostic, fails dependency checks, and lacks inspectable real behavior proof.
