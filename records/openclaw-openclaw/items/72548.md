## Summary

Keep open. This is useful feature work, but the latest head still has blocking correctness, validation-contract, dependency-check, proof, and maintainer-scope issues; current main has not already implemented the requested SecretRef provider seam.

## What This Changes

This PR opens SecretRef source/config/wire contracts to plugin-owned sources, adds a secret-provider SDK subpath and bundled GCP/keyring plugins, and updates docs, generated baselines, and adjacent SecretRef call sites.

## Best Possible Solution

Land a security-reviewed SecretRef provider seam that either truly supports enabled bundled and installed/workspace plugin providers, or explicitly narrows the first step to bundled GCP/keyring providers with separate follow-up issues for third-party loading and broader SecretRef field coverage.

## Reproduction Assessment

Yes for the PR review blockers: CI reproduces the dependency-check and cron diagnostic failures on the latest head, and source inspection shows third-party sources are advertised while runtime lookup is bundled-only. This is not a current-main bug reproduction because the PR is additive feature work.

## Solution Assessment

No. The manifest-first direction fits OpenClaw's plugin architecture, but this diff over-advertises third-party support, changes an existing validation diagnostic contract, fails checks, and lacks inspectable real behavior proof.
