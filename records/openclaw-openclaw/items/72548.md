## Summary

Keep open. The branch is useful but not mergeable yet because the advertised third-party SecretRef provider seam remains bundled-only at runtime, and the external contributor still has not attached inspectable real behavior proof.

## What This Changes

The PR adds a manifest-declared SecretRef provider seam, a plugin SDK subpath, schema and gateway protocol support for plugin-owned SecretRef sources, plus bundled GCP Secret Manager and OS keyring provider plugins.

## Best Possible Solution

Land only after the seam either supports enabled bundled, installed, and workspace secret-provider artifacts through registry/public-artifact APIs, or the public docs/types/changelog are narrowed to bundled-only with third-party support tracked separately.

## Reproduction Assessment

Yes, for the review finding. Source inspection of PR head shows zod/docs accept arbitrary plugin-owned sources while runtime lookup only calls bundled owner and bundled artifact loaders, so an installed/workspace provider can validate and then fail as unknown.

## Solution Assessment

No. The manifest-first direction fits the plugin architecture, but this is not complete while advertised third-party provider sources cannot load; either add installed/workspace loading or narrow the public contract.
