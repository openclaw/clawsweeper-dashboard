## Summary

Keep open: current main still has an env/file/exec-only SecretRef model, and this active PR is the implementation candidate rather than merged evidence. The branch still has blocking correctness findings, failed checks, merge conflicts, and no inspectable real behavior proof.

## What This Changes

The PR opens SecretRef source/provider contracts for plugin-owned sources, adds a secret-provider SDK/public-artifact seam, adds bundled GCP and keyring plugins, and updates schemas, docs, generated baselines, labels, and tests.

## Best Possible Solution

Land a security-reviewed SecretRef provider seam that either fully supports enabled bundled, installed, and workspace plugin providers through the public-artifact boundary, or explicitly ships a bundled-only first step with docs/types narrowed and a tracked follow-up for third-party loading and broader field coverage.

## Reproduction Assessment

Yes for the review findings. Source inspection shows PR schemas/docs accept arbitrary plugin-owned sources while runtime lookup is bundled-only, and live CI reproduces the dependency and cron diagnostic failures.

## Solution Assessment

No. The manifest-first direction fits OpenClaw's plugin architecture, but the current branch over-advertises third-party support, changes an existing validation diagnostic, fails dependency checks, and lacks inspectable real behavior proof.
