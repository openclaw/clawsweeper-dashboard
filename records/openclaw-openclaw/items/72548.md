## Summary

Keep open: the feature direction is useful, but this PR is not merge-ready because the advertised plugin-owned SecretRef seam is bundled-only at runtime, latest CI has concrete failures, and the contributor still needs inspectable real behavior proof.

## What This Changes

The PR adds a manifest-declared secret-provider SDK contract, opens SecretRef config and wire schemas for plugin-owned sources, and adds bundled GCP and OS keyring secret provider plugins with docs and tests.

## Best Possible Solution

Complete enabled bundled, installed, and workspace secret-provider resolution through the plugin public-artifact boundary, or deliberately narrow the docs/types/changelog to bundled-only, then preserve existing diagnostics, satisfy CI, and add redacted real behavior proof.

## Reproduction Assessment

Yes for the PR blockers. Source inspection shows non-bundled plugin sources can validate while runtime lookup only consults bundled loaders, and latest CI reproduces the dependency and cron diagnostic failures.

## Solution Assessment

No. The manifest-first direction fits the plugin architecture, but the current branch is incomplete unless it supports installed/workspace providers at runtime or intentionally narrows the public contract to bundled-only.
