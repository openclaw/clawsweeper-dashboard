## Summary

Keep open: current main and the latest release still use a 30-minute Discord component callback fallback with a 24-hour `agentComponents.ttlMs` cap, and https://github.com/openclaw/openclaw/pull/90394 is the open candidate implementation pending maintainer policy/security acceptance.

## What This Changes

Request to support asynchronous Discord review workflows by raising the component callback default from 30 minutes to 7 days and the existing TTL cap to 30 days.

## Best Possible Solution

Resolve the TTL default and cap policy in https://github.com/openclaw/openclaw/pull/90394 or a narrow equivalent, update the matching docs/metadata/tests, and close this issue only after the accepted change lands.

## Reproduction Assessment

## Solution Assessment

Unclear: changing the Discord plugin fallback and schema cap is the narrow implementation boundary if maintainers accept the policy. The exact 7-day default and 30-day maximum still need maintainer acceptance or revised values.
