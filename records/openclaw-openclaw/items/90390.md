## Summary

Current main still has the reported 30-minute fallback, and the open linked implementation PR has not merged. The issue remains the appropriate place for maintainers to decide the intended default lifetime before resolving the paired work.

## What This Changes

Request changing Discord’s default component callback lifetime from 30 minutes to seven days and increasing the configurable maximum from 24 hours to 30 days.

## Best Possible Solution

Use the existing Discord plugin-owned TTL seam, explicitly choose a finite safe default and maximum, then refresh and review the linked PR with authorization, bounded storage, documentation, and boundary tests kept aligned.

## Reproduction Assessment

Yes. Current source deterministically applies a 30-minute fallback when `ttlMs` is unset, and the public documentation confirms callbacks expire at that boundary.

## Solution Assessment

Unclear pending maintainer judgment. Changing the existing plugin-owned fallback is the narrowest implementation path, but seven days and a 30-day ceiling are product-policy choices with upgrade and stale-action consequences.
