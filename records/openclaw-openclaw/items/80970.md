## Summary

Keep open: current `main` still has a cleanup path that can validate a session-store key as a transcript session id, and the documented `sessions cleanup --fix-missing` behavior should handle stale or missing transcript entries without crashing.

## What This Changes

The issue reports that `openclaw sessions cleanup` rejects session-store keys such as `agent:main:main` when cleanup tries to validate them as session IDs.

## Best Possible Solution

Keep transcript filename validation strict, but make cleanup distinguish store keys from transcript IDs and handle legacy or malformed rows by pruning or skipping them safely, with regression coverage for `agent:main:main`.

## Reproduction Assessment

Yes, at source level: create a store row keyed `agent:main:main` whose `sessionId` is also `agent:main:main`, then run cleanup with `--fix-missing`; `pruneMissingTranscriptEntries` calls `resolveSessionFilePath`, which falls into `validateSessionId` and throws. I did not establish a live CLI reproduction in this read-only checkout.

## Solution Assessment

Yes, the narrow maintainable direction is to fix cleanup-specific handling of legacy or malformed store entries, not to broadly allow colons in transcript session IDs without a separate path-safety decision.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/80970. Current `sessions cleanup --fix-missing` can call `resolveSessionFilePath(entry.sessionId, entry, ...)` with legacy or malformed store entries where `entry.sessionId` is actually a session key such as `agent:main:main`, causing `validateSessionId` to throw before cleanup can prune or report the row. Keep canonical transcript ID/path validation strict; do not broadly widen `SAFE_SESSION_ID_RE` unless you prove the path-safety and compaction-checkpoint implications. Prefer cleanup-specific handling: resolve a valid persisted `sessionFile` when present, otherwise treat invalid/missing transcript references as `prune-missing` candidates for `--fix-missing`, and ensure ordinary cleanup/disk-budget paths skip invalid transcript references without crashing. Add regression coverage for a store containing key `agent:main:main` with `sessionId: "agent:main:main"`, including dry-run and applied `--fix-missing`; also verify non-fix cleanup does not throw. Preserve path traversal protections and compaction checkpoint filename rejection. Add a single user-facing `CHANGELOG.md` fix entry if the repair changes shipped CLI behavior. Stop and ask for maintainer review if the fix would require a new cleanup policy, config option, or broad session ID contract change.
