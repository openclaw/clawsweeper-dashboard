## Summary

Keep open: current main still feeds malformed key-shaped stored session IDs into strict transcript ID validation during `sessions cleanup --fix-missing`, and the same-scope fix PRs were closed without merging.

## What This Changes

The issue requests that `openclaw sessions cleanup` handle legacy rows whose stored `sessionId` is a session key like `agent:main:main` without crashing.

## Best Possible Solution

Keep transcript filename validation strict, but make `--fix-missing` treat cleanup-time resolver failures as malformed missing rows that can be reported and pruned with focused dry-run and applied regression coverage.

## Reproduction Assessment

Yes, at source level. A store row with `sessionId: "agent:main:main"` and no usable `sessionFile` reaches `resolveSessionFilePath`, then `validateSessionId` rejects the colon-containing value before cleanup can continue.

## Solution Assessment

Yes. The narrow maintainable fix is cleanup-specific recovery under `--fix-missing`; relaxing `SAFE_SESSION_ID_RE` globally or skipping bad rows would either broaden transcript path acceptance or leave malformed rows unpruned.
