## What This Changes

The issue reports that `openclaw sessions cleanup --fix-missing` crashes when a legacy store row persists a session key such as `agent:main:main` as `sessionId`.

## Best Possible Solution

Add cleanup-specific recovery for malformed stored session IDs under `--fix-missing`, keep global transcript-id validation strict, and cover dry-run plus applied cleanup with focused regression tests.

## Reproduction Assessment

Yes, at source level. A store row with `sessionId: "agent:main:main"` and no usable `sessionFile` reaches `resolveSessionFilePath(...)` from `pruneMissingTranscriptEntries`, then falls through to the strict validator that rejects `:`.

## Solution Assessment

Yes. The narrow maintainable fix is cleanup-specific handling for malformed legacy rows during missing-transcript pruning, not relaxing the global transcript filename validator.
