## Summary

Keep open: current main still has the source-level `--fix-missing` crash path, and an open linked PR specifically closes this issue, so the issue should remain open until that PR or an equivalent fix merges.

## What This Changes

The issue reports that `openclaw sessions cleanup --fix-missing` crashes when a malformed session-store row stores a key-shaped `sessionId` such as `agent:main:main`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81012, or an equivalent narrow cleanup recovery, then close this issue after merge.

## Reproduction Assessment

Yes, at source level. A store row whose stored `sessionId` is `agent:main:main` and lacks a usable `sessionFile` reaches `resolveSessionFilePath`, then `validateSessionId` rejects the colon before `--fix-missing` can prune it.

## Solution Assessment

Yes. The narrow maintainable direction is cleanup-specific handling for invalid stored session IDs while keeping shared transcript filename validation strict.
