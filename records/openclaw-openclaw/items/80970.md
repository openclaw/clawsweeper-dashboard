## Summary

Keep open: current main still has the same cleanup crash path for malformed legacy rows, and the known same-scope fix PRs were closed without merging.

## What This Changes

The issue asks for `openclaw sessions cleanup --fix-missing` to handle a persisted session key such as `agent:main:main` in `sessionId` without throwing.

## Best Possible Solution

Keep transcript filename validation strict, but make the cleanup missing-entry path treat invalid stored `sessionId` values as malformed store rows that can be reported and pruned under `--fix-missing`, with regression coverage.

## Reproduction Assessment

Yes, at source level. A store row with `sessionId: "agent:main:main"` reaches `resolveSessionFilePath` during `--fix-missing`, then falls through to `validateSessionId`, which rejects the colon-containing value.

## Solution Assessment

Yes. The narrow maintainable fix is cleanup-specific handling for malformed legacy rows, not changing the global transcript ID validator to accept session keys.
