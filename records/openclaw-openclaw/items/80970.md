## Summary

Keep open: current main still lets `sessions cleanup --fix-missing` resolve a stored `sessionId` through strict transcript-id validation, so a malformed/key-shaped row like `agent:main:main` can abort cleanup before the row can be reported or pruned.

## What This Changes

The issue reports that `openclaw sessions cleanup` crashes when cleanup treats a key-shaped session-store value such as `agent:main:main` as a transcript session id.

## Best Possible Solution

Keep transcript filename validation strict, but make cleanup treat invalid or key-shaped stored `sessionId` values as per-entry cleanup data errors: recover from a safe `sessionFile` when possible, otherwise report or prune the malformed row under `--fix-missing` without aborting the whole command.

## Reproduction Assessment

Yes, at source level. A store entry whose stored `sessionId` is `agent:main:main` and lacks a usable `sessionFile` reaches `resolveSessionFilePath(entry.sessionId, ...)`, then `validateSessionId` rejects the colon-containing value before cleanup can continue.

## Solution Assessment

Yes. The narrow maintainable fix is cleanup-specific handling for malformed legacy store rows, not broadening `SAFE_SESSION_ID_RE` to accept session keys globally.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/80970 on current main. `sessions cleanup --fix-missing` currently calls `resolveSessionFilePath(entry.sessionId, entry, opts)` for each store row, so a malformed stored `sessionId` such as `agent:main:main` can throw `Invalid session ID` and abort cleanup. Keep `SAFE_SESSION_ID_RE` and transcript filename validation strict. Change cleanup to handle invalid/key-shaped stored `sessionId` values per entry: if a safe persisted `sessionFile` exists, use it; otherwise under `--fix-missing` mark/prune/report the malformed row without throwing. Preserve true-missing behavior and, if you also address stale `sessionFile` fallback from https://github.com/openclaw/openclaw/issues/50248 / https://github.com/openclaw/openclaw/pull/69954, track repair-only mutations correctly. Add regression tests for malformed `sessionId` rows, valid `sessionFile` recovery, and true-missing pruning. Do not relax session-id validation globally, do not add a config option, and do not implement startup/load-time migrations. Add a changelog fix entry because this is user-visible CLI behavior.
