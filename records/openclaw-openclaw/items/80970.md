## Summary

Keep open: current main still has the `--fix-missing` crash path, and the same-scope fix PRs for this report were closed unmerged, so no shipped or current-main fix exists yet.

## What This Changes

The issue reports that `openclaw sessions cleanup --fix-missing` crashes when a legacy store row stores a key-shaped `sessionId` such as `agent:main:main`.

## Best Possible Solution

Keep transcript filename validation strict, but make `--fix-missing` treat invalid or key-shaped stored `sessionId` values as malformed cleanup rows: use a safe persisted `sessionFile` when available, otherwise report and prune the row without throwing, with regression coverage.

## Reproduction Assessment

Yes, at source level. A row with `sessionId: "agent:main:main"` and no usable `sessionFile` reaches `resolveSessionFilePath` during `--fix-missing`, then strict validation rejects the colon-containing value before cleanup can continue.

## Solution Assessment

Yes. The narrow maintainable fix is cleanup-specific malformed-row handling; relaxing `validateSessionId` globally would weaken the transcript filename boundary and is not needed for this CLI cleanup bug.
