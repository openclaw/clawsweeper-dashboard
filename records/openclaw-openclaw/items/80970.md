## Summary

Keep open: current main still has the source-level crash path where `sessions cleanup --fix-missing` resolves a stored key-shaped `sessionId` as a transcript id and the strict validator rejects it. The same-scope fix PRs are closed unmerged, while the remaining open adjacent PR covers stale `sessionFile` fallback rather than this invalid-id crash.

## What This Changes

The issue asks for session cleanup to handle legacy store rows whose `sessionId` contains a session key such as `agent:main:main`, instead of throwing during `--fix-missing`.

## Best Possible Solution

Add cleanup-specific recovery that treats unresolvable stored session IDs as corrupt or missing metadata under `--fix-missing`, keeps transcript filename validation strict everywhere else, and covers dry-run plus enforced cleanup with regression tests.

## Reproduction Assessment

Yes, at source level. A store row with `sessionId: "agent:main:main"` and no usable `sessionFile` reaches `resolveSessionFilePath(...)` from `pruneMissingTranscriptEntries`, then `validateSessionId` rejects the colon-containing value.

## Solution Assessment

Yes. The narrow maintainable fix is cleanup-specific recovery for malformed legacy rows, not relaxing the global transcript-id validator to accept session keys.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/80970. Current main lets `pruneMissingTranscriptEntries` pass `entry.sessionId` directly into `resolveSessionFilePath`; when a legacy row stores a key-shaped value such as `agent:main:main` and has no usable `sessionFile`, `validateSessionId` throws before `--fix-missing` can report or prune the row. Add cleanup-specific handling in `src/config/sessions/cleanup-service.ts` so unresolvable or invalid stored session IDs are treated as missing/corrupt cleanup metadata only in the `--fix-missing` path. Keep `SAFE_SESSION_ID_RE`, `validateSessionId`, and normal transcript path resolution strict; do not broaden global session-id validation or change unrelated session-key semantics. Add regression coverage for dry-run and applied cleanup with a store row keyed `agent:main:main` whose stored `sessionId` is also `agent:main:main`, plus a healthy row to prove the rest of cleanup continues. Coordinate with open PR https://github.com/openclaw/openclaw/pull/69954 if still open; do not fold in its stale canonical transcript fallback scope unless maintainers decide to combine them. Add a user-facing changelog entry for the CLI cleanup fix.
