## Summary

Keep open. Current main has the shared Discord/Telegram table-conversion path, but code-mode table width and padding still use code-unit `.length`, so CJK cells remain under-padded; the open draft fix at https://github.com/openclaw/openclaw/pull/55596 references this issue with closing syntax and should be reviewed or replaced before closing.

## What This Changes

The issue asks for CJK display-width-aware Markdown table fallback rendering for Telegram and Discord.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/55596, or a narrow replacement, after it fixes shared display-width measurement in `renderTableAsCode` and keeps focused CJK and emoji regression coverage.

## Reproduction Assessment

Yes, by source inspection. A CJK Markdown table rendered through `markdownToIR(..., { tableMode: "code" })` reaches `renderTableAsCode`, where current main and `v2026.5.18` measure and pad cells with `.length`.

## Solution Assessment

Yes. The narrow maintainable fix is to use display-width measurement in the shared `renderTableAsCode` path instead of duplicating Telegram and Discord-specific table alignment logic.
