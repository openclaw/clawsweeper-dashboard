The issue requests display-width-aware markdown table rendering for CJK text in Telegram and Discord.

## Best Possible Solution

Port display-width-aware code-table padding to `packages/markdown-core`, keep the existing Telegram/Discord table-mode routing, add CJK and emoji regression coverage, then land or replace the linked PR.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main still measures code-table cells with `cell?.text.length`, and Telegram plus Discord route markdown table conversion through that shared renderer.

## Solution Assessment

Yes. The narrow maintainable solution is shared markdown-core display-width measurement with focused Telegram/Discord regression coverage; separate channel-specific table padding would duplicate the existing table-mode contract.
