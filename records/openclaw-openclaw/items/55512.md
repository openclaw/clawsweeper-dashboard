## Summary

Keep open. Current main still measures code-mode Markdown table columns with UTF-16 string length, so CJK cells remain under-padded; the open draft fix at https://github.com/openclaw/openclaw/pull/55596 references this issue and should be reviewed or replaced before closing.

## What This Changes

Fix CJK-aware Markdown table alignment for Telegram and Discord code-block table rendering.

## Best Possible Solution

Fix display-width measurement in the shared `renderTableAsCode` path and keep focused CJK/emoji regression coverage so Telegram and Discord inherit the same aligned output.

## Reproduction Assessment

Yes, by source inspection. A CJK Markdown table rendered with `tableMode: "code"` reaches `renderTableAsCode`, where current main and `v2026.5.18` still measure and pad cells with `.length`.

## Solution Assessment

Yes. The maintainable fix is a narrow shared renderer change for display-width measurement, rather than duplicating Telegram- and Discord-specific alignment logic.
