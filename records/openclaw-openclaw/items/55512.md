## Summary

Keep open. Current main and the latest release still use code-unit length for shared code-mode Markdown table padding, and the open linked implementation PR remains unmerged and conflicting rather than a fix on main.

## What This Changes

Markdown table rendering should align CJK columns in Telegram and Discord by using display-width-aware code-table padding and converting Discord tables into aligned code blocks.

## Best Possible Solution

Port display-width-aware code-table padding into `packages/markdown-core`, align with existing visible-width semantics where practical, add CJK and emoji regression coverage, then land or replace the linked PR.

## Reproduction Assessment

Yes, source-level. Current main measures code-table cells with `cell?.text.length`, and both Telegram and Discord route Markdown table conversion through that shared renderer.

## Solution Assessment

Yes. The narrow maintainable solution is a shared markdown-core display-width fix with focused channel regression coverage, not separate Telegram and Discord padding logic.
