## Summary

Keep open. Current main and v2026.5.18 still measure code-mode Markdown table cells with code-unit `.length`, so CJK cells remain under-padded; the open draft implementation candidate at https://github.com/openclaw/openclaw/pull/55596 references this issue with closing syntax and should be reviewed or replaced before closing this issue.

## What This Changes

The issue asks OpenClaw to render Telegram and Discord Markdown table fallbacks as CJK-aware aligned code blocks instead of under-padded or raw-pipe text.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/55596, or a narrow replacement, after it fixes shared display-width measurement in `renderTableAsCode` and keeps focused CJK and emoji regression coverage.

## Reproduction Assessment

Yes, by source inspection. A CJK Markdown table rendered through `markdownToIR(..., { tableMode: "code" })` reaches `renderTableAsCode`, where current main and v2026.5.18 measure and pad cells with `.length`.

## Solution Assessment

Yes. The narrow maintainable fix is to use display-width measurement in the shared `renderTableAsCode` path, which avoids duplicating Telegram and Discord-specific table alignment logic.
