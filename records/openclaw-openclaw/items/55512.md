## Summary

Keep open. Current main still measures code-table column width with UTF-16 string length on the shared markdown-core path used by Telegram and Discord, and the open linked pull request is an implementation candidate that needs porting rather than a cleanup-close reason.

## What This Changes

The issue requests CJK display-width-aware Markdown table alignment for Telegram and Discord output.

## Best Possible Solution

Port display-width/grapheme-aware code-table padding to `packages/markdown-core`, keep the existing Telegram/Discord table-mode routing, add CJK/emoji regression tests, then land or replace the linked pull request.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current `packages/markdown-core/src/ir.ts` still measures code-table cells with `cell?.text.length`, and Telegram plus Discord route table conversion through that shared code; I did not run tests because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is shared markdown-core display-width measurement with focused channel regression coverage; adding separate Telegram or Discord table logic would duplicate the existing table-mode contract.
