## Summary

Keep open. Current main and the latest release still use code-unit length when rendering code-mode Markdown tables, so CJK cells remain under-padded, and https://github.com/openclaw/openclaw/pull/55596 is still the open draft implementation candidate that references this issue with closing syntax.

## What This Changes

The issue requests CJK display-width-aware Markdown table alignment for Telegram code-block rendering and Discord table fallback.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/55596, or a narrow replacement, after it fixes shared display-width measurement in `renderTableAsCode` and keeps focused CJK and emoji regression coverage.

## Reproduction Assessment

Yes, by source inspection. A CJK Markdown table rendered through `markdownToIR(..., { tableMode: "code" })` reaches `renderTableAsCode`, where current main measures and pads cells with `.length`.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared `renderTableAsCode` path used by Telegram and Discord table fallback, and the open draft PR already targets that path.
