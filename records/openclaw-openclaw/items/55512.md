## Summary

Keep open. Current main still sizes code-mode Markdown table columns and padding with JavaScript string length, so CJK cells remain under-padded; the issue is also paired with open draft PR #55596 using closing syntax, so it should remain open until that PR or an equivalent replacement lands.

## What This Changes

Request to make Telegram and Discord Markdown table fallback align CJK text by using display-width-aware table rendering and converting Discord tables to aligned code blocks.

## Best Possible Solution

Land PR #55596 or a narrow replacement that fixes `renderTableAsCode` in the shared Markdown renderer to use display-width and grapheme-aware measurement, preferably by reusing or extracting the existing width helper, then add regression tests for CJK, emoji, and mixed-width cells across the shared converter paths.

## Reproduction Assessment

Yes. By inspection on current main, a cell like `类型` has `.length === 2` but occupies four monospace columns, and `renderTableAsCode` uses `.length` for both width and padding, which reproduces the under-padding without needing a live Telegram or Discord send.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared code-mode table renderer because Discord already routes tables through `convertMarkdownTables` and Telegram passes table mode into the same Markdown IR renderer; channel-specific fixes would duplicate the wrong layer.
