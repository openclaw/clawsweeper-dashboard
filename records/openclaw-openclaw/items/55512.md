## Summary

Keep open. Current main still measures code-mode Markdown table widths and padding with JavaScript `.length`, so CJK cells remain under-padded, and the issue is already paired with an open draft fix PR using closing syntax.

## What This Changes

The issue requests CJK-aware Markdown table alignment for Telegram and Discord table fallback rendering.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/55596 or a narrow replacement that fixes display-width measurement in `renderTableAsCode` and adds CJK/emoji regression coverage after real behavior proof is provided.

## Reproduction Assessment

Yes, by source inspection. Passing a CJK table through `markdownToIR(..., { tableMode: "code" })` reaches `renderTableAsCode`, where cells such as `类型` are measured as length 2 even though they occupy four monospace columns.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared `renderTableAsCode` path used by Telegram and Discord table fallback; channel-specific padding fixes would duplicate renderer logic.
