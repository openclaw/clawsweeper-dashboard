Keep open. Current main and v2026.6.8 still use code-unit length for shared code-mode Markdown table padding that Telegram and Discord both reach, and https://github.com/openclaw/openclaw/pull/55596 remains an open conflicting implementation candidate rather than a fix on main.

## What This Changes

The issue requests CJK display-width-aware Markdown table rendering for Telegram and Discord code/plaintext table fallbacks.

## Best Possible Solution

Port display-width-aware padding into `packages/markdown-core/src/ir.ts`, align with the existing terminal-core visible-width behavior where practical, add CJK/emoji regression coverage, and land or replace the linked PR.

## Reproduction Assessment

Yes, source-level. Current main measures code-table cells with `cell?.text.length`, and both Telegram and Discord route Markdown table conversion through that shared renderer.

## Solution Assessment

Yes. The narrow maintainable fix is shared markdown-core display-width padding with focused channel coverage, not separate Telegram and Discord padding logic.
