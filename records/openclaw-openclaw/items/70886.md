## Summary

Keep open. Current main still routes ZAI image understanding through the configured provider-level Coding Plan base URL, while ZAI vision catalog entries and the ZAI media provider have no current-main override to the standard multimodal endpoint; open PR #70915 already uses closing syntax for this issue, so the issue should remain open until that PR lands or is replaced.

## What This Changes

The issue requests fixing the built-in image tool so `agents.defaults.imageModel` values like `zai/glm-4.6v` work with ZAI Coding Plan configurations instead of returning 404.

## Best Possible Solution

Review and land or replace #70915 with a narrow ZAI-owned routing fix that sends known ZAI vision models from Coding Plan base URLs to the matching standard multimodal endpoint while leaving text/coding models on the configured Coding Plan route, with regression coverage for global and CN endpoints.

## Reproduction Assessment

Yes. Configure ZAI with `zai-coding-global` or `zai-coding-cn`, set `agents.defaults.imageModel` to `zai/glm-4.6v`, and call the `image` tool; current source still leaves the coding base URL on the resolved image model.

## Solution Assessment

Yes for triage. Keeping this issue open while #70915 remains open is the right path; the implementation should stay in the ZAI provider/media-understanding boundary or a narrow provider-routing seam, not a broad core migration.
