## Summary

Keep open. Current main still leaves ZAI vision requests on the provider-level Coding Plan base URL, and open PR #70915 already uses closing syntax for this issue, so closing before that PR lands would be premature.

## What This Changes

The issue requests fixing the built-in image tool so `agents.defaults.imageModel` can use ZAI vision models such as `zai/glm-4.6v` when ZAI is configured with Coding Plan endpoints.

## Best Possible Solution

Keep this issue open until #70915 or a replacement lands; the fix should stay in the ZAI provider or media-understanding boundary with regression coverage for global and CN Coding Plan routes.

## Reproduction Assessment

Yes. Configure ZAI with `zai-coding-global` or `zai-coding-cn`, set `agents.defaults.imageModel` to `zai/glm-4.6v`, and call the `image` tool; current source still preserves the Coding Plan provider `baseUrl` for that vision model.

## Solution Assessment

Yes for triage. Keeping this open while #70915 remains open is the narrowest maintainer-safe path, and the implementation should avoid broad core model-specific routing.
