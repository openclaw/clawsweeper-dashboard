## Summary

Keep open: current main and v2026.6.5 still skip plugin bootstrap for CLI installs, while the linked fix PR remains open and the assignee has requested an explicit decision about the future `before_install` contract.

## What This Changes

Make CLI `openclaw plugins install` honor the documented install-time `before_install` plugin hook consistently with Gateway-driven installs.

## Best Possible Solution

Keep `security.installPolicy` as the primary operator-owned security gate, then either make `before_install` consistently available on supported install paths with recovery-safe bootstrap or explicitly narrow and deprecate its documented purpose, tests, and compatibility contract.

## Reproduction Assessment

Yes, at source level with high confidence: current main and v2026.6.5 do not preload plugins for `plugins install`, so the process-local hook runner is unavailable and the scan returns without invoking the hook. This read-only review did not execute the failing baseline.

## Solution Assessment

Unclear pending maintainer judgment. The linked bootstrap fix coherently preserves the existing hook contract, but maintainers must decide whether broad runtime preload is acceptable or whether `before_install` should be narrowed in favor of `security.installPolicy`.
