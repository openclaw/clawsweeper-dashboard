## Summary

Current main and the shipped v2026.5.3+ updater already block package updates launched from the Gateway service/process tree before stopping the managed service, which addresses the Linux systemd self-termination failure described here.

## What This Changes

The issue asks OpenClaw to prevent Linux user-systemd Gateway auto-update from killing the service and failing to restart when the update command runs inside the Gateway service context.

## Best Possible Solution

Keep the shipped v2026.5.3+ guard behavior and direct users still on 2026.5.2 to recover with an interactive `openclaw update` or install of the latest release.

## Reproduction Assessment

No current-main failing reproduction remains: source and regression tests show the Gateway-owned package-update path now exits before package replacement or service restart. The old-version failure path is still credible from the report and is covered by the shipped #75729/#75819 fixes.

## Solution Assessment

Yes. Blocking service-owned package updates before stopping the Gateway is the narrowest maintainable fix because it avoids adding new systemd policy while preserving the documented shell-update path for supervised installs.
