## Summary

Close as implemented: the report matches a 2026.5.2 in-service package-update path, and current main plus shipped v2026.5.3+ now block service-owned package updates before service stop, package mutation, or restart-helper execution.

## What This Changes

Request that Linux systemd-managed gateway auto-update avoid stopping itself and failing to restart when launched from the service cgroup.

## Best Possible Solution

Keep the shipped guard that leaves the old gateway running when package update is invoked from the managed gateway service; track any future self-applying Linux systemd handoff as a separate enhancement.

## Reproduction Assessment

No. The original 2026.5.2 incident is plausible, but current main has source-level guards and regression coverage that prevent the service stop/package install path from a service-owned auto-update process.

## Solution Assessment

Yes. Blocking package updates from the gateway service/process tree is the narrow safe fix for this outage path because it leaves the existing gateway running instead of relying on a child restart helper inside the same systemd cgroup.
