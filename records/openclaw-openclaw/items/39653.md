## Summary

Keep open: current main still accepts opted-in plaintext WebSocket hostnames, and this PR addresses the right security bug, but the branch is incomplete because caller tests/shared mocks still preserve the old contract and the external PR lacks real behavior proof.

## What This Changes

The PR removes the `allowPrivateWs` hostname fallback from `isSecureWebSocketUrl` and updates gateway net unit coverage to reject opted-in plaintext WebSocket hostnames while allowing private IP literals.

## Best Possible Solution

Land a complete fail-closed gateway WebSocket validator change that updates the validator, caller tests, shared mocks, and maintainer-owned release notes so the flag consistently allows only localhost plus private/loopback IP literals.

## Reproduction Assessment

Yes. Current main’s source path shows the `allowPrivateWs` fallback returns true for non-IP hostnames, and live Node `net.isIP()` checks confirm those hostnames return `0`.

## Solution Assessment

No, not as submitted. Removing the fallback is the narrow maintainable fix, but the PR must also update caller contracts/shared mocks and provide redacted real behavior proof before merge.
