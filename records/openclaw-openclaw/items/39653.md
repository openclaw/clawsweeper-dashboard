## Summary

Keep open. The patch addresses a real source-level hostname fallback, but as submitted it reverses a current gateway caller/onboarding contract added for Docker/Kubernetes private-DNS use and still lacks real behavior proof, so it needs maintainer security/product review.

## What This Changes

The PR removes the `allowPrivateWs` hostname fallback in `src/gateway/net.ts` and updates gateway net tests so `ws://` hostnames are rejected while private IP literals remain accepted.

## Best Possible Solution

Make one explicit gateway WebSocket policy decision, then update the validator, clients, onboarding, docs, mocks, and tests together.

## Reproduction Assessment

Yes, for source-level reproduction. Current main accepts ordinary hostnames under `allowPrivateWs`, and caller tests lock in that behavior while a Node `net.isIP` check explains why the fallback returns true.

## Solution Assessment

No, not as submitted. Removing the fallback may be the right hardening direction, but this isolated validator/test change leaves the client, connection-details, onboarding, docs, and test-mock policy surface inconsistent.
