## Summary

Keep open: current main still has the hostname pass-through bug and the PR’s fail-closed direction is appropriate, but the branch is not merge-ready because caller tests, the shared mock, and the changelog still encode the old hostname-allowance contract.

## What This Changes

The PR removes the `allowPrivateWs` hostname fallback in `isSecureWebSocketUrl()` and adds gateway net tests expecting hostnames to be rejected while private IP literals remain allowed.

## Best Possible Solution

Land a complete fail-closed validator change that accepts localhost plus private/loopback IP literals only, updates all caller tests and shared mocks to that contract, and documents the behavior reversal in the active changelog.

## Reproduction Assessment

Yes. Current main’s `allowPrivateWs` branch returns true for hostnames because Node `net.isIP()` returns `0` for non-IP strings such as `openclaw-gateway.ai`, and the existing tests still assert that behavior.

## Solution Assessment

No, not as submitted. Removing the hostname fallback is the narrow maintainable direction, but the PR must also align caller contract tests, the shared mock, and the changelog before merge.
