## Summary

Keep this PR open: it addresses a real current-main security gap, but the submitted patch is not merge-ready because downstream gateway caller tests/shared mocks still encode hostname acceptance and the user-facing behavior change lacks a changelog entry.

## What This Changes

The PR removes the `allowPrivateWs` hostname fallback in `isSecureWebSocketUrl()` and adds gateway net tests that reject hostname `ws://` URLs while preserving private IP literal allowance.

## Best Possible Solution

Land a complete fail-closed validator change that accepts localhost and private/loopback IP literals only, updates caller tests/shared mocks and the changelog, and leaves private-DNS hostname support to a separately designed allowlist or resolution feature.

## Reproduction Assessment

Yes. Current main’s `allowPrivateWs` branch returns true for hostnames because Node `net.isIP()` returns `0` for non-IP strings such as `openclaw-gateway.ai`, so the issue is reproducible from source inspection and a focused Node check.

## Solution Assessment

No, not as submitted. Removing the hostname fallback is the narrow maintainable direction, but the PR must also align caller contract tests, the shared mock, and the changelog before merge.
