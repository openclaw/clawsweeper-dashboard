Keep this PR open: it targets a real security-sensitive validator path on current main, but the branch is incomplete because caller tests/shared mocks still encode the hostname-allowed contract and the external PR lacks after-fix real behavior proof.

## What This Changes

The PR removes the allowPrivateWs hostname fallback from the gateway WebSocket URL validator and adds focused gateway net tests for rejecting opted-in ws:// hostnames.

## Best Possible Solution

Choose the gateway security policy explicitly, then land a complete validator/caller/test/docs/changelog update; if hostname support must remain, replace the synchronous hostname pass-through with safe DNS verification or an explicit allowlist design.
