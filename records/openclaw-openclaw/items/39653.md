## Summary

Keep open. The patch addresses a real source-reproducible security concern, but current main intentionally documents and tests private-DNS hostname support under the break-glass flag, so a validator-only deletion conflicts with shipped gateway caller and docs policy; the external PR also lacks real behavior proof.

## What This Changes

The PR removes the `allowPrivateWs` hostname fallback from `isSecureWebSocketUrl()` and updates gateway net tests to expect hostname rejection while retaining private IP literal acceptance.

## Best Possible Solution

Make one explicit gateway plaintext WebSocket policy decision, then update the validator, runtime callers, onboarding, mocks, docs, and regression tests together.

## Reproduction Assessment

Yes, source-level: current main accepts non-IP hostnames under `allowPrivateWs`, and caller tests/docs lock in `ws://openclaw-gateway.ai` or trusted private-DNS support. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as submitted. Removing the fallback may be the right hardening direction, but a validator/net-test-only change conflicts with current caller contracts and docs instead of making one coordinated gateway WebSocket policy change.
