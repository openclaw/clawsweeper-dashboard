## Summary

Keep open: the validator change is the right narrow security direction, but this PR is not merge-ready because downstream caller tests, the shared gateway test mock, and the active changelog still reflect hostname support.

## What This Changes

The PR removes the `allowPrivateWs` hostname fallback in `isSecureWebSocketUrl()` and updates gateway net tests to reject hostname `ws://` URLs while still allowing private IP literals.

## Best Possible Solution

Land a complete fail-closed validator change that accepts localhost plus private/loopback IP literals only, aligns all caller tests and shared mocks to that contract, and documents the reversal in the active changelog.

## Reproduction Assessment

Yes. Current main’s source path and tests show that `allowPrivateWs` accepts hostnames because Node `net.isIP()` returns `0` for non-IP strings.

## Solution Assessment

No, not as submitted. Removing the hostname fallback is the narrow maintainable fix, but the PR must also update caller contract tests, the shared mock, and changelog before merge.
