## Summary

Keep this PR open: the core validator change is a plausible security fix, but the branch is incomplete against current main because caller tests and the shared mock still encode hostname allowance, and the external PR lacks real behavior proof.

## What This Changes

The branch removes the hostname fallback from `isSecureWebSocketUrl(..., { allowPrivateWs: true })` and updates the gateway net unit test to reject opted-in `ws://` hostnames while keeping private IP literals allowed.

## Best Possible Solution

Land a complete fail-closed gateway WebSocket validator change that updates all caller tests and shared mocks to accept localhost plus private/loopback IP literals only, with maintainer-owned changelog at landing and redacted real behavior proof.

## Reproduction Assessment

Yes. Current main’s source path shows the `allowPrivateWs` fallback returns true for non-IP hostnames, and Node’s `net.isIP()` contract confirms those hostnames return `0`.

## Solution Assessment

No, not as submitted. Removing the fallback is the narrow maintainable fix, but the PR must also update caller contracts/shared mocks and provide real behavior proof before merge.
