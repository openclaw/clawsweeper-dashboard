## What This Changes

The PR removes the `allowPrivateWs` hostname fallback in `isSecureWebSocketUrl` and updates `src/gateway/net.test.ts` to reject hostname `ws://` URLs while keeping private IP literals allowed.

## Best Possible Solution

Land a complete fail-closed gateway WebSocket validator change that rejects opted-in hostname `ws://` consistently across the validator, caller tests, and shared mocks, with redacted real behavior proof before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows the `allowPrivateWs` fallback returns true for non-IP hostnames, and Node’s `net.isIP()` contract confirms those hostnames return `0`.

## Solution Assessment
