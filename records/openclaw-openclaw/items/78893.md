## Summary

Keep open: current main still has the source-reproducible QQBot cron prefix bug, and the open fixing PR that references this issue has not merged yet.

## What This Changes

Normalize QQBot cron delivery targets so explicit-channel deliveries do not duplicate `qqbot:` prefixes and repeated provider prefixes are rejected.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/78916 if it remains correct, then close this issue from the merge.

## Reproduction Assessment

Yes. At source level, current main prepends the channel to an already provider-prefixed QQBot target and lacks a QQBot `parseExplicitTarget` hook to make `qqbot:c2c:...` channel-local before preview/routing.

## Solution Assessment

Yes. The best fix is QQBot-owned explicit target parsing plus prefix-aware cron preview formatting, preserving the documented shared provider-prefix guard instead of adding QQBot special cases to generic cron storage.
