## Summary

Keep open: the Discord account-routing bug is still relevant on current main, but this PR is not merge-ready because the strict token guard also catches setup/introspection paths, the branch is conflicting, and the external PR still has only test proof rather than real Discord behavior proof.

## What This Changes

The PR changes Discord token/account resolution so explicit non-default account lookups can fail closed instead of falling back to the channel-level token, with focused token/account regression tests.

## Best Possible Solution

Keep the focused bug fix, but move or thread strict explicit-routing semantics so outbound explicit sends fail closed while setup, inspection, enumeration, and implicit default-account paths preserve documented inherited-token behavior; then rebase and prove the live Discord multi-account send path.

## Solution Assessment

No. The bug is real, but putting strict behavior in the generic token resolver is too broad because setup and introspection callers still use named account IDs for inherited-token configs.
