## Summary

Keep open: the Discord account-routing bug is real, but this PR still makes the generic token resolver fail closed for non-delivery setup paths that currently inherit `channels.discord.token`, and it lacks after-fix real Discord behavior proof.

## What This Changes

The PR changes Discord token/account resolution so explicitly supplied non-default account IDs without per-account tokens return `source: "none"`, with regression tests and enumeration exceptions.

## Best Possible Solution

Scope strict missing-token behavior to explicit outbound delivery/account-routing calls while preserving inherited channel-token fallback for setup, status, account scans, and other non-delivery configuration paths.

## Reproduction Assessment

Yes, at source level: current main and its token tests show non-default Discord accounts without per-account tokens inherit `channels.discord.token`, while the PR changes that branch to return no token. I did not establish a live Discord cron repro in this read-only review.

## Solution Assessment

No. Strict explicit-send behavior is the right direction, but defaulting the generic resolver to strict is broader than the cron/outbound boundary and leaves setup token-resolution paths broken.
