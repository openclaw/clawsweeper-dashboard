## Summary

Keep open. The PR targets a real Discord identity-routing bug that current main still has, but the latest diff still treats internal account-resolution calls as explicit outbound targeting, which can break named-account configs that currently inherit `channels.discord.token`.

## What This Changes

The PR makes explicit non-default Discord account token resolution fail closed when no per-account token exists, threads an explicitness flag through account resolution, and adds token/account regression tests.

## Best Possible Solution

Land a Discord-owned fix that makes externally targeted cron/message sends strict while preserving channel-token inheritance for implicit/default account resolution, runtime scans, setup, and status/discovery paths.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: current main returns the channel token for `resolveDiscordToken(cfg, { accountId: "work" })` when `accounts.work` has no token, while the PR makes generic `resolveDiscordAccount({ accountId: "work" })` calls strict even when they come from internal scans.

## Solution Assessment

No. The strict token guard is the right direction for explicit outbound delivery, but caller intent should be passed from the cron/message delivery boundary or a separated strict resolver path, not inferred from every account ID passed to the generic account resolver.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep internal account resolution implicit:** `extensions/discord/src/accounts.ts:67-68`
  - body: `callerProvidedAccountId` is true for any non-empty `params.accountId`, but the plugin framework also passes account IDs while enumerating, starting, and setting up configured accounts. With `channels.discord.token` plus `accounts.work` that has no token, those internal calls now resolve `work` with `tokenSource: "none"` instead of inheriting the channel token, so a valid named-account config can lose runtime/setup access. Thread strict intent only from outbound/cron sends or split the resolver path.
