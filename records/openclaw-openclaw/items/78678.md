## Summary

Keep open: latest head still has three source-proven read/write asymmetries where negative numeric map/object keys can resolve but cannot be written, so the patch is not ready to merge.

## What This Changes

The PR adds an `oc://` workspace addressing substrate, `openclaw path` CLI commands, CLI docs, tests, and a changelog entry for markdown, JSONC, JSONL, and YAML workspace files.

## Best Possible Solution

Fix the negative numeric key read/write symmetry in the three edit paths, then let maintainers decide whether this broad substrate should land as one exceptional PR or be split around the canonical workspace-addressing discussion.

## Reproduction Assessment

Yes for the PR findings: source inspection of latest head shows resolver paths that allow negative numeric map/object keys to fall through as literal keys while the edit paths reject them. This is not a current-main bug because the feature surface is introduced by this PR.

## Solution Assessment

No, not as-is. The narrow repair is to let `-N` fall through to literal map/object-key lookup on keyed containers while still resolving `$first` and `$last` positionally.

## Review Findings

Full review comments:

- **[P2] Preserve negative JSONC object keys during writes:** `src/oc-path/jsonc/edit.ts:98-105`
  - body: `resolveJsoncOcPath` lets `-N` fall through to literal object-key lookup when positional resolution does not apply, but this edit path returns `unresolved` immediately.   - confidence: 0.93

A line object containing a negative numeric key is readable but not writable; match the resolver by falling through to literal lookup on objects.
  - confidence: 0.91
- **[P2] Preserve negative YAML map keys during writes:** `src/oc-path/yaml/edit.ts:160-163`
  - body: `resolveYamlOcPath` allows `-N` on maps to fall through to literal-key lookup, but `resolvePositionalSegments` aborts before `doc.hasIn` can try that key. YAML paths with negative map keys are readable but not writable; let `-N` fall through for maps while still resolving `$first` and `$last`.
  - confidence: 0.9
