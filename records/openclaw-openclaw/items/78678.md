## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Reject field-only paths before formatting:** `src/oc-path/oc-path.ts:250-258`
  - body: `formatOcPath` still accepts a struct like `{ file: "X", field: "name" }` because this guard only runs when `section` is also present. It formats as `oc://X/name`, which parses back as a section rather than a field, breaking the advertised nesting and formatter/parser round-trip contract for programmatic callers.
  - confidence: 0.94
- **[P2] Expand or reject file-slot patterns before reading:** `src/commands/path.ts:321-323`
  - body: The docs describe wildcards across all four slots, but `find` resolves `pattern.file` to one literal filesystem path and reads it before calling `findOcPaths`. A pattern such as `oc://*.lobster/steps/*/command` will try to read a literal `*.lobster` file instead of enumerating matching files or rejecting the unsupported form cleanly.
  - confidence: 0.9
- **[P2] Keep pattern detection quote-aware:** `src/oc-path/oc-path.ts:425-431`
  - body: `isPattern` splits slots with plain `slot.split('.')`, unlike the parser and walkers. A quoted literal key such as `"a.*.b"` is split into pieces and treated as a wildcard pattern, so concrete quoted paths can be rejected by `resolveOcPath` and `setOcPath` before exact resolution.
  - confidence: 0.88
