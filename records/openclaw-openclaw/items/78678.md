  - body: `formatOcPath` says field-without-item is invalid, but this guard only throws when a section is also present. A programmatic `{ file: "X", field: "name" }` formats as `oc://X/name`, which parses back as a section and breaks the round-trip contract; reject any `field` without `item`.
  - confidence: 0.9
- **[P2] Handle file-slot wildcards before reading:** `src/commands/path.ts:321-323`
  - body: The PR describes wildcards across all slots and uses file patterns like `oc://*.lobster/...`, but the CLI resolves `pattern.file` to one literal filesystem path before finding matches. Either expand supported file globs or reject/undocument file-slot wildcards so `find` does not try to open a literal `*.lobster` file.
  - confidence: 0.86
