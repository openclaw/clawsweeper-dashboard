## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve negative JSONC object keys during writes:** `src/oc-path/jsonc/edit.ts:98-105`
  - body: `resolveJsoncOcPath` intentionally lets `-N` fall through to literal object-key lookup so keys like negative Telegram group IDs can resolve. This write path now returns `unresolved` as soon as `resolvePositionalSeg` returns null on keyed containers, so a path can resolve but cannot be set. Fall through to the literal key for negative object segments while still resolving `$first` and `$last`.
  - confidence: 0.92
- **[P2] Preserve negative JSONL object keys during writes:** `src/oc-path/jsonl/edit.ts:97-104`
  - body: The JSONL resolver has the same negative-key fallback inside line values, but the edit path treats any keyed `-N` segment as a failed positional token. A line body containing an object key like `"-5028303500"` will resolve and then fail to update. Match the resolver by falling through to literal lookup when positional resolution is not applicable on objects.
  - confidence: 0.9
- **[P2] Preserve negative YAML map keys during writes:** `src/oc-path/yaml/edit.ts:160-163`
  - body: `resolveYamlOcPath` documents that `-N` on a map falls through to literal-key lookup for negative numeric IDs, but `resolvePositionalSegments` aborts before `doc.hasIn` can try that key. This makes YAML paths with negative map keys readable but not writable. Keep `$first`/`$last` resolution, but let negative segments fall through for maps.
  - confidence: 0.88
- **[P2] Resolve positional parent segments before JSONC insertions:** `src/oc-path/universal.ts:761-764`
  - body: JSONC insertion validates the parent with `resolveJsoncInsertion`, which honors `$first`, `$last`, and negative array indexes, but `mutateJsoncContainer` re-walks the original parent path. A path such as inserting into the last object of an array can report a valid insertion point and then fail during mutation. Normalize positional parent segments in the mutation walker too.
  - confidence: 0.86
- **[P2] Resolve YAML positional tokens when reporting lines:** `src/oc-path/universal.ts:222-231`
  - body: The universal YAML resolver returns line numbers by re-walking the path, but this helper only unquotes segments and then does raw map/sequence lookup. Positional paths like `steps/$last/command` can resolve correctly while reporting line 1 because `$last` is never converted to the concrete index. Reuse the YAML positional logic here or surface the concrete path from the resolver.
  - confidence: 0.84
