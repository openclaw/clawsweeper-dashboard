## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Reject field-only paths before formatting:** `src/oc-path/oc-path.ts:250-258`
  - body: `formatOcPath` still accepts a struct like `{ file: "X", field: "name" }` because this guard only fires when `section` is also present. It formats as `oc://X/name`, which parses back as a section, not a field, so the advertised formatter/parser round-trip contract is broken for programmatic callers.
  - confidence: 0.94
- **[P2] Handle file-slot wildcards before reading:** `src/commands/path.ts:321-323`
  - body: The PR and docs describe file-slot wildcards such as `oc://*.lobster/...`, but `find` resolves `pattern.file` to one literal filesystem path before calling `findOcPaths`. Either expand supported file globs or reject/undocument file-slot wildcards at this CLI boundary.
  - confidence: 0.9
- **[P2] Keep pattern detection quote-aware:** `src/oc-path/oc-path.ts:425-431`
  - body: `isPattern` splits slots with plain `slot.split('.')`, unlike the parser and resolvers. A quoted literal key such as `"a.*.b"` is split into pieces and treated as a wildcard pattern, so concrete quoted paths can be rejected before resolution.
  - confidence: 0.88
