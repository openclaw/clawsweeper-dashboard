## Summary

Keep open. Current main still does not implement the PR's helper-level metadata-entry path or validated-snapshot reuse, so this is not cleanup-closeable, but the PR should not merge as written because the snapshot reuse bypasses runtime config finalization before plugin CLI loading.

## What This Changes

The PR reuses the validated plugin CLI config snapshot, adds metadata-entry loading helpers, routes lazy known-core CLI startup through the metadata-only plugin registry path, and adds focused plugin CLI regression tests.

## Best Possible Solution

Keep the PR open, rebase it onto current main, and preserve the metadata-only lazy CLI optimization only after the config snapshot reuse is redesigned to keep the same runtime finalization semantics as the current `getRuntimeConfig()` path.

## Reproduction Assessment

Yes. A focused test can mock or exercise `registerPluginCliCommandsFromValidatedConfig()` with config finalization effects such as `env.shellEnv` or runtime overrides and verify the plugin loader context sees the finalized runtime config/env; the static call chain already shows the PR would pass `snapshot.config` and skip `getRuntimeConfig()`.

## Solution Assessment

No, not as written. The metadata-entry optimization is a plausible narrow fix, but the config-load optimization must preserve `loadConfig`/`getRuntimeConfig` finalization before it is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.83

Full review comments:

- **[P2] Preserve config finalization before plugin loading:** `src/plugins/cli.ts:38`
  - body: `readConfigFileSnapshot()` returns a runtime-shaped config, but it does not run the `loadConfig()` finalization path that imports `env.shellEnv`, persists generated owner display secrets, and applies runtime overrides. Returning `snapshot.config` here means `registerPluginCliCommandsFromValidatedConfig()` passes a pre-finalized cfg into the plugin loader context, so plugin auto-enable/activation can run without shell-imported credentials or overrides that the current path applies. Keep the snapshot reuse only if those finalization effects are preserved.
  - confidence: 0.83
