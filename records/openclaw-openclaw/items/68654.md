## Summary

Keep open. The branch still has a blocking config-finalization regression: it would pass a pre-finalized snapshot config into plugin CLI loading, skipping the runtime finalization current main preserves. The earlier eager-mode review feedback appears addressed, and the metadata-loading direction remains narrow enough for an automated repair.

## What This Changes

The PR reuses the validated plugin CLI config snapshot, adds metadata-entry loading helpers, routes lazy core-command startup through metadata-only plugin CLI registration, and adds focused plugin CLI tests.

## Best Possible Solution

Keep the PR open and repair the branch so the lazy metadata-only optimization preserves the same finalized runtime config semantics as current main. The safest path is to keep `getRuntimeConfig()` after snapshot validation, or introduce an explicitly finalized snapshot helper with regression coverage, then rebase and add the required changelog entry if the user-facing perf change remains.
