## Summary

Keep open. This draft MEMBER PR has the protected `maintainer` label, and the proposed workflow change still needs human security review before merge because it adds Docker provisioning to a self-hosted hydrate path that later handles provider-secret environment.

## What This Changes

The PR adds Docker setup to the Crabbox hydrate workflow, refreshes kitchen-sink ClawHub E2E fixture expectations, and adjusts package/plugin metadata gates.

## Best Possible Solution

Keep the PR open, replace the Docker setup with a pinned or image-baked Crabbox path and runner-scoped socket access, then rebase over the parts already on `main` before landing the diagnostic coverage updates.

## Reproduction Assessment

Yes. The source path is clear: the kitchen-sink E2E runner invokes Docker, current `main` Crabbox hydration does not install/start Docker before readiness, and current registry diagnostics include messages missing from the current assertion allowlist.

## Solution Assessment

No. The fixture and diagnostic updates match current behavior, but the Docker provisioning is not the narrowest maintainable fix because it adds mutable root code execution and world-writable Docker socket access.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Avoid executing Docker's mutable installer in the workflow:** `.github/workflows/crabbox-hydrate.yml:71`
  - body: The hydrate job would run `curl https://get.docker.com | sudo sh` before the same job hydrates provider-secret environment. That gives a mutable third-party install script root execution on a self-hosted runner; use a pre-baked Crabbox image or a pinned package/repository installation path instead.
  - confidence: 0.84
- **[P1] Avoid making the Docker socket world-writable:** `.github/workflows/crabbox-hydrate.yml:82`
  - body: Changing `/var/run/docker.sock` to mode `666` gives every local process root-equivalent Docker control for the lifetime of the hydrated box. Keep access scoped to the runner user/group or use a narrower wrapper so later commands do not inherit a host-control socket opened to all users.
  - confidence: 0.9
