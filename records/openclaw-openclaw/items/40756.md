## Summary

Keep open: current main and the latest release still append the generated workspace-path hint to configured heartbeat prompts despite the documented verbatim contract. This PR targets the right narrow fix for https://github.com/openclaw/openclaw/issues/40255, but it is currently conflicting and lacks required after-fix real behavior proof from the contributor.

## What This Changes

The branch changes heartbeat prompt composition so user-configured heartbeat prompts bypass workspace-path hint injection and adds a focused unit test for that helper.

## Best Possible Solution

Refresh or replace this branch so user-authored heartbeat prompts remain verbatim while default and internally generated heartbeat prompts retain workspace-path guidance, with regression coverage and redacted real behavior proof.

Yes. ## Solution Assessment

Yes in direction, but not merge-ready as submitted. Tracking whether the base heartbeat prompt came from user config and skipping only the generated workspace-path hint is the narrow maintainable fix; the branch needs conflict refresh and real behavior proof before merge.
