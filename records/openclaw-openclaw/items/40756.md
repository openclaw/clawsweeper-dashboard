## What This Changes

The PR adds a custom-prompt guard so user-configured heartbeat prompts are not appended with the workspace HEARTBEAT.md path hint.

## Best Possible Solution

Refresh or replace the PR with a focused guard that preserves user-authored heartbeat prompts verbatim while keeping workspace-path guidance for default and internally generated heartbeat prompts.

## Reproduction Assessment

Yes, by source inspection. Configure agents.defaults.heartbeat.prompt or agents.list[].heartbeat.prompt with text mentioning HEARTBEAT.md; current main resolves that text and still passes it through appendHeartbeatWorkspacePathHint.

## Solution Assessment

Yes, with a refresh. Tracking whether the base heartbeat prompt came from user config and skipping only that case is the narrow maintainable fix, but it must be applied to current main's normal and commitment append paths.
