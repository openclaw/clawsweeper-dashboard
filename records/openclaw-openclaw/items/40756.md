## Summary

Keep open: current main and the latest release still unconditionally append the workspace-path hint to configured heartbeat prompts, and this PR targets the right regression but is conflicting and lacks real behavior proof.

## What This Changes

The PR adds an `isUserCustomPrompt` guard to `appendHeartbeatWorkspacePathHint`, derives it from configured heartbeat prompts when no exec/cron event is active, and adds helper-level regression tests.

## Best Possible Solution

Preserve user-authored heartbeat prompts verbatim while retaining workspace-path guidance for default/internal heartbeat prompts, with conflict-safe regression coverage and redacted real-run proof.

## Reproduction Assessment

## Solution Assessment

Yes in direction, but not merge-ready as submitted. Tracking whether the base heartbeat prompt came from user config and skipping only the generated workspace-path hint is the narrow fix; the branch still needs a current-main refresh and real behavior proof.
