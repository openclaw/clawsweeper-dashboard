## Summary

Keep open: this PR targets a valid current-main regression in heartbeat prompt composition, but it is not ready to merge because the branch is stale against current prompt assembly and still lacks contributor real behavior proof.

## What This Changes

The branch adds an `isUserCustomPrompt` guard to heartbeat workspace-path hint injection and a focused unit test for custom, default, unrelated, and already-hinted prompts.

## Best Possible Solution

Preserve user-authored heartbeat prompts verbatim while retaining workspace-path guidance for default and internally generated heartbeat prompts, then refresh the branch and add redacted real behavior proof before merge.

## Reproduction Assessment

Yes. By source inspection, configure `agents.defaults.heartbeat.prompt` or `agents.list[].heartbeat.prompt` with text mentioning `HEARTBEAT.md`; current main resolves that text as the base prompt and still passes it through the unguarded workspace-path hint helper.

## Solution Assessment

Yes in direction, but not merge-ready as submitted. Tracking whether the base heartbeat prompt came from user config and skipping only the generated workspace-path hint is the narrow maintainable fix; the branch needs a current-main refresh and real behavior proof.
