## Summary

Keep open: current main still appends the workspace `HEARTBEAT.md` hint to configured heartbeat prompts, while the linked regression remains open and this PR directly targets that gap. The implementation direction is sound, but the PR needs a required changelog entry and a current-main landing pass because prompt composition has changed since its base.

## What This Changes

The PR adds an `isUserCustomPrompt` guard to heartbeat workspace-path hint injection, exports the helper for focused testing, and adds a unit test for custom, default, unrelated, and duplicate-hint prompt cases.

## Best Possible Solution

Preserve user-authored heartbeat prompts verbatim while retaining workspace-path guidance for default, exec, cron, due-task, and due-commitment heartbeat prompts generated internally, with focused regression coverage and release-note credit.

## Reproduction Assessment

Yes. Configure a non-empty `agents.defaults.heartbeat.prompt` or per-agent `heartbeat.prompt` that mentions `HEARTBEAT.md`; current main resolves that prompt and still sends it through unconditional workspace-path hint injection.

## Solution Assessment

Yes. The PR's guard is the narrow maintainable fix for the regression, but the landed version should be rebased onto current main, cover the newer due-commitment append branch, and add the required changelog entry.
