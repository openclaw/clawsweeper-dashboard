## Summary

Keep open: current main and the latest release still mutate configured heartbeat prompts that mention `HEARTBEAT.md`; this PR targets the right narrow fix, but it is currently conflicting and still lacks contributor real behavior proof.

## What This Changes

The PR adds an `isUserCustomPrompt` guard to heartbeat prompt hinting and a focused unit test so user-configured heartbeat prompts are not modified by the workspace-path hint.

## Best Possible Solution

Refresh or replace this PR so user-authored heartbeat prompts remain verbatim while default and internally generated heartbeat prompts retain workspace-path guidance, with regression coverage and redacted real behavior proof.

## Reproduction Assessment

Yes, by source inspection. Configure `agents.defaults.heartbeat.prompt` or `agents.list[].heartbeat.prompt` with text mentioning `HEARTBEAT.md`; current main resolves that text as the base prompt and still passes it through the unguarded workspace-path hint helper.

## Solution Assessment

Yes in direction, but not merge-ready as submitted. Tracking whether the base heartbeat prompt came from user config and skipping only the generated workspace-path hint is the narrow maintainable fix; the branch needs a current-main refresh and real behavior proof before merge.
