## Summary

Keep this PR open: current main and the latest stable release still append the workspace-path hint to any heartbeat prompt mentioning HEARTBEAT.md, while the docs promise configured heartbeat prompts are sent verbatim. The PR is a focused implementation for the linked open regression, but it needs maintainer review and a current-main rebase before merge.

## What This Changes

The PR exports the heartbeat workspace-path hint helper, adds an `isUserCustomPrompt` guard so user-configured heartbeat prompts are not mutated, and adds focused unit coverage for custom, default, unrelated, and no-duplicate prompt cases.

## Best Possible Solution

Land a rebased version that preserves user-authored heartbeat prompts verbatim while retaining workspace-path guidance for built-in, exec, cron, and other internally generated heartbeat prompts, with regression coverage through the heartbeat runner path.

## Reproduction Assessment

Yes. Configure a non-empty `agents.defaults.heartbeat.prompt` or per-agent `heartbeat.prompt` that mentions `HEARTBEAT.md`; current main resolves that prompt and still appends the workspace path hint in `resolveHeartbeatRunPrompt()`.

## Solution Assessment

Yes, the PR direction is the narrow maintainable fix for the regression. The landed version should be rebased onto current main and apply the guard to the newer due-commitment append path as well as the ordinary default prompt path.
