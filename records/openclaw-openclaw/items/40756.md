## Summary

Keep open. Current main still mutates configured heartbeat prompts even though the docs say heartbeat prompts are sent verbatim, and this PR is a useful closing candidate for https://github.com/openclaw/openclaw/issues/40255, but it needs a refresh for current main's two append paths plus real behavior proof before merge.

## What This Changes

The PR exports the heartbeat hint helper for testing, adds an `isUserCustomPrompt` guard so configured heartbeat prompts are not mutated, and adds focused helper coverage.

## Best Possible Solution

Refresh or replace the PR with a focused guard that preserves user-authored heartbeat prompts verbatim while keeping workspace-path guidance for default and internally generated heartbeat prompt flows.

## Reproduction Assessment

Yes, by source inspection: configure `agents.defaults.heartbeat.prompt` or `agents.list[].heartbeat.prompt` with text mentioning `HEARTBEAT.md`, and current main still passes that configured text through `appendHeartbeatWorkspacePathHint()`. I did not run tests because this was a read-only review.

## Solution Assessment

Yes, with a refresh. Tracking whether the base heartbeat prompt came from user config and skipping only that case is the narrow maintainable fix, but it must be applied to current main's normal and due-commitment append paths.
