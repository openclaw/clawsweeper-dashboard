## What This Changes

The PR adds an `isUserCustomPrompt` guard so user-configured heartbeat prompts are not appended with the workspace `HEARTBEAT.md` path hint.

## Best Possible Solution

Refresh or replace the PR with a focused guard that preserves user-authored heartbeat prompts verbatim while retaining workspace-path guidance for default and internally generated heartbeat prompts.

## Reproduction Assessment

Yes, by source inspection. Configure `agents.defaults.heartbeat.prompt` or `agents.list[].heartbeat.prompt` with text mentioning `HEARTBEAT.md`; current main resolves that text and still sends it through `appendHeartbeatWorkspacePathHint`.

## Solution Assessment

Yes in direction, but not as-is. Tracking whether the base heartbeat prompt came from user config and skipping only that case is the narrow maintainable fix, but the branch must be refreshed to guard both current append paths.
