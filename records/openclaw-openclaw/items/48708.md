## What This Changes

The issue requests an optional timestamp display for user and assistant messages in the OpenClaw TUI chat log.

## Best Possible Solution

If maintainers want the feature, add a deliberate TUI-owned opt-in that preserves real message timestamps across live, pending, streaming, and history rendering, with config/docs/tests aligned.

Not applicable as a bug reproduction: this is a feature request. Source inspection does confirm the current TUI has message timestamps in transcript/history data but does not render them in chat rows.

## Solution Assessment
