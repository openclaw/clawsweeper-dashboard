## What This Changes

Request to add optional user-visible timestamps to messages in the TUI chat view.

## Best Possible Solution

Define one opt-in TUI timestamp display contract that uses stored message timestamps consistently, then land or replace one reviewed implementation path.

## Reproduction Assessment

Yes for the missing capability from source inspection: current TUI rows render text only, while timestamps are used for reconciliation and model-facing context rather than user-visible labels. This is a feature request, not a broken existing behavior reproduction.

## Solution Assessment

Unclear until maintainers choose the product shape. The narrow maintainable direction is an opt-in TUI-owned display contract using persisted message timestamps across live and replayed rows, not divergent local-clock prefixes.
