## What This Changes

Request adding an optional user-facing timestamp display for TUI chat messages, likely controlled by a config option or slash-command toggle.

## Best Possible Solution

If maintainers want the feature, add a deliberate TUI-owned opt-in that preserves real message timestamps across live, streaming, pending, and history rendering, with config, docs, and focused TUI tests aligned.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection confirms current TUI code has timestamp data in history paths but does not render per-message timestamps in chat rows.

## Solution Assessment

Unclear until maintainers choose the product shape. Optional timestamps are reasonable, but the safest implementation needs a TUI-owned timestamp display contract rather than local text prefixing that can be overwritten by streaming and history rebuild paths.
