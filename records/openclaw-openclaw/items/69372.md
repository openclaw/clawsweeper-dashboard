## Summary

## What This Changes

The PR changes Control UI canvas/a2ui iframe URL resolution to fail closed without a scoped capability URL, adds a timed websocket reconnect to refresh canvas capability URLs, and updates related UI tests.

## Best Possible Solution

Keep the fail-closed protected iframe hardening, but refresh scoped canvas surfaces through the current plugin-surface contract without tearing down active Control UI sessions, then prove it in a real browser/session.

Yes, at source level: current main can return raw protected canvas/a2ui paths without a scoped surface URL, and the PR’s forced close path follows directly to pending-request rejection and chat run interruption. I did not run a live browser repro in this read-only review.

## Solution Assessment

No. The resolver hardening is a good direction, but closing the full Control UI websocket and using deprecated `hello.canvasHostUrl` is not the narrowest maintainable fix on current main.
