## Summary

Keep open: the resolver hardening is useful, but the refresh path adds a timed full websocket close that can drop active runs, still targets deprecated `hello.canvasHostUrl` instead of current `pluginSurfaceUrls.canvas`, and lacks real behavior proof.

## What This Changes

The branch makes Control UI canvas/a2ui iframe URL resolution fail closed without a scoped host and adds a timed Gateway websocket reconnect plus tests to refresh canvas capability URLs.

## Best Possible Solution

Keep the fail-closed protected iframe behavior, add positive scoped rewrite coverage for canvas and a2ui, and refresh `pluginSurfaceUrls.canvas` through a non-teardown or explicitly idle-safe operator-reviewed path.

## Reproduction Assessment

Yes, by source inspection rather than a live browser run: current main can return raw protected canvas/a2ui paths without a scoped surface URL, and the PR regression follows from its timer close plus current close/reconnect state handling.

## Solution Assessment

No. The resolver hardening is a good direction, but closing the full Control UI websocket and keying refresh off deprecated `hello.canvasHostUrl` is not the narrowest maintainable solution on current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Avoid refreshing canvas capability by closing the socket:** `ui/src/ui/gateway.ts:406`
  - body: The new timer closes every scoped Control UI websocket after about nine minutes. Current close handling rejects pending browser requests, and the reconnect hello clears active chat run/stream state, so long-running chats or RPCs crossing the refresh boundary can be dropped even when no canvas is active. Refresh the plugin surface without transport teardown, or gate it to an idle-safe path.
  - confidence: 0.92
- **[P2] Target the current plugin surface field:** `ui/src/ui/gateway.ts:503-504`
  - body: Current main exposes scoped canvas URLs through `hello.pluginSurfaceUrls.canvas` and documents `canvasHostUrl` as unsupported. This refresh scheduling path still keys off `hello.canvasHostUrl`, so after rebase it will either fail type checking or miss the canvas surface entirely.
  - confidence: 0.87
