## Summary

Keep open: the fail-closed iframe resolver change addresses a real source-visible Control UI bug, but the PR refreshes canvas capability by tearing down the websocket, still targets deprecated `hello.canvasHostUrl` instead of current `pluginSurfaceUrls.canvas`, and has no real after-fix behavior proof.

## What This Changes

The PR makes Control UI canvas/a2ui iframe URL resolution fail closed without a scoped capability URL and adds a timer-driven websocket reconnect plus tests to refresh canvas capabilities.

## Best Possible Solution

Keep the fail-closed protected iframe behavior, add positive scoped rewrite coverage for both canvas and a2ui, and refresh `pluginSurfaceUrls.canvas` through a non-teardown or explicitly idle-safe operator-reviewed path.

## Reproduction Assessment

Yes, by source inspection rather than a live browser run: current main returns raw protected canvas/a2ui paths without a scoped surface URL, and the PR regression follows from the added timer close plus current close/reconnect handlers that reject pending work and reset chat state.

## Solution Assessment

No. The resolver hardening is a good part of the fix, but closing the full Control UI websocket and keying refresh off deprecated `hello.canvasHostUrl` is not the narrowest maintainable solution on current main.
