## Summary

Keep this PR open but blocked. The fail-closed canvas URL hardening is useful, but the branch refreshes the capability by closing the whole Control UI websocket, which can drop pending RPCs and active chat state; it also lacks real after-fix behavior proof.

## What This Changes

The PR makes protected Control UI canvas/a2ui iframe URLs fail closed without a scoped capability host, adds a timed browser websocket reconnect to refresh canvas capability URLs, and updates UI/gateway tests.

## Best Possible Solution

Keep the fail-closed protected iframe URL behavior, add positive scoped rewrite coverage for both canvas and a2ui, and refresh the canvas plugin surface through an operator-safe non-teardown path or a clearly idle-safe flow.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main returns raw protected canvas URLs without a scoped host, and the PR regression follows from the added timer close through existing close and hello handlers that drop pending work/state.

## Solution Assessment

No. The resolver hardening is a good part of the fix, but closing the full Control UI websocket to refresh an iframe/plugin-surface token is not the narrowest maintainable solution.
