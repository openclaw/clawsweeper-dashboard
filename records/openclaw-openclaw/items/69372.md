## Summary

Keep open: the PR addresses a real current-main canvas iframe problem, but the refresh approach can interrupt active Control UI sessions and is based on the deprecated `canvasHostUrl` field instead of the current plugin-surface contract.

## What This Changes

The PR changes Control UI canvas/a2ui iframe URL resolution to fail closed without scoped capability URLs, adds a timed websocket reconnect for canvas capability refresh, and updates related UI/browser-client tests.

## Best Possible Solution

Keep the fail-closed protected iframe hardening, but refresh the scoped canvas surface through the current plugin-surface contract without tearing down active Control UI sessions, then prove the behavior in a real browser/session.

## Reproduction Assessment

Yes, at source level: current main can return raw protected canvas/a2ui paths without a scoped surface URL, and the PR regression follows from its timer close plus current close/reconnect handling. I did not run a live browser repro in this read-only sweep.

## Solution Assessment

No. The resolver hardening is a good direction, but closing the full Control UI websocket and keying refresh off deprecated `hello.canvasHostUrl` is not the narrowest maintainable solution on current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Avoid refreshing canvas by closing the websocket:** `ui/src/ui/gateway.ts:406`
  - body: The new timer closes every scoped Control UI websocket after about nine minutes. Current close handling rejects pending browser requests, and the reconnect path clears in-flight chat run state, so long-running chats or RPCs crossing the refresh boundary can be dropped even when no canvas is active. Refresh the plugin surface without transport teardown, or gate it to an idle-safe path.
  - confidence: 0.92
- **[P2] Use pluginSurfaceUrls for canvas refresh:** `ui/src/ui/gateway.ts:503-504`
  - body: Current main exposes scoped canvas URLs through `hello.pluginSurfaceUrls.canvas` and documents `canvasHostUrl` as unsupported. This scheduling path still keys off `hello.canvasHostUrl`, so after rebase it will either fail type checking or miss the canvas surface entirely.
  - confidence: 0.87
