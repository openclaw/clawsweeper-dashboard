## Summary

Keep open, but this PR should not merge as-is. The fail-closed canvas URL hardening addresses a real current-main path, but the timed websocket close can drop active Control UI work, the branch is stale against the current plugin-surface protocol, and the external contributor has not supplied real browser/session proof.

## What This Changes

The PR makes protected Control UI canvas/a2ui iframe URLs fail closed without a scoped capability URL, adds a timed websocket reconnect to refresh canvas capability URLs, and updates related UI/gateway tests.

## Best Possible Solution

Keep the fail-closed protected iframe URL behavior, add positive scoped rewrite coverage for both canvas and a2ui, and refresh the current `pluginSurfaceUrls.canvas` surface through a non-teardown or explicitly idle-safe operator-reviewed path.

## Reproduction Assessment

Yes, by source inspection rather than a live browser run: current main returns raw protected canvas/a2ui paths without a scoped surface URL, and the PR regression follows from the added timer close plus current close/reconnect handlers that reject pending work and reset chat state.

## Solution Assessment

No. The resolver hardening is a good part of the fix, but closing the full Control UI websocket and keying refresh off deprecated `hello.canvasHostUrl` is not the narrowest maintainable solution on current main.
