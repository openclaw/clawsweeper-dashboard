## Summary

Current main already implements WebChat/Control UI non-image file uploads, including PDF and Office file extensions, and routes uploaded files through Gateway managed media so agents receive readable media paths. The related WebChat attachment PR was merged and is included in the latest shipped release.

## What This Changes

Add WebChat/Control UI file upload support for PDF and Office documents so agents can analyze uploaded files.

## Best Possible Solution

Keep the shipped WebChat attachment path: Control UI accepts non-video files, Gateway stores them as managed media, and the agent receives typed media paths. Future provider-native Office parsing can be tracked separately if maintainers want richer automatic extraction beyond the current agent-readable file path model.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The current-main verification path is high confidence: inspect the Control UI picker/send serialization plus Gateway attachment routing tests for PDF and Office-shaped files.

## Solution Assessment

Yes. The current implementation is the narrow maintainable solution because it keeps UI upload handling generic, preserves filenames/MIME metadata, and uses the existing managed-media/MediaPaths pipeline instead of adding provider-specific document parsing to WebChat.
