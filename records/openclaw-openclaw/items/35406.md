## Summary

Keep open. Current main still routes non-image PDF/Office reads through the upstream text reader, and an open closing PR already targets the narrow fallback while overlapping DOCX extraction work still needs maintainer/security judgment.

## What This Changes

Request that `read` stop returning raw decoded binary for PDF/Office files, either by returning a clear fallback message or parsing supported formats.

## Best Possible Solution

Land or replace a focused, bounded `read` fallback for known binary document formats, while keeping richer Office/PDF extraction behind the document-extract plugin seam with explicit limits, docs, and tests.

## Reproduction Assessment

Yes. Source inspection shows non-image `read` calls still delegate to the upstream reader, and the upstream reader UTF-8-decodes non-image buffers.

## Solution Assessment

Yes for the safe-fallback direction, but current main does not implement it. The narrower maintainable path is a bounded MIME/extension guard in `read`, with parser-backed extraction handled through the existing document-extract surface.
