## Summary

Keep open. Current main still delegates generic `read` calls to the wrapped upstream reader before any PDF or Office-document guard, and the sandbox MIME hook still only returns image MIME types. The active implementation path is open draft PR #57600, which uses closing syntax for this issue, so this issue should remain open until that PR or a maintainer equivalent lands.

## What This Changes

The issue asks for the `read` tool to avoid raw binary output for PDF and Office documents by returning a concise unsupported-file fallback or extracted document text.

## Best Possible Solution

Review and land #57600 or a maintainer-owned equivalent that adds a small preflight in `src/agents/pi-tools.read.ts` before adaptive paging, reuses shared MIME detection for PDF and Office formats, covers host and sandbox read paths, returns a concise fallback pointing PDFs to the `pdf` tool and Office files to parser/conversion tooling, and adds regression tests for `.pdf`, `.docx`, `.xlsx`, and `.pptx`.
