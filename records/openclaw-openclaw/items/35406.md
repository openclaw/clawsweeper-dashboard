## Label Justifications

- `impact:session-state`: Raw binary document bytes can pollute agent-visible context and degrade the transcript/state used for reasoning.

## Summary

Keep open: current main and the latest release still let filesystem `read` fall through to the upstream non-image text path for PDF/Office documents, and open linked PRs already cover the fallback and DOCX-parser directions that need maintainer review.

## What This Changes

Request to make the `read` tool avoid raw binary output for PDF/Office documents by returning a warning fallback or extracting readable text.

## Best Possible Solution

Keep this as the canonical tracker, land one bounded read-path guard to stop raw binary token waste, and route parser-backed extraction through the document-extractor boundary after maintainer approval.

## Reproduction Assessment

Yes. Current main delegates non-image reads to the upstream read tool, and the locked upstream implementation converts non-image buffers to UTF-8 text.

## Solution Assessment

Unclear as a final direction. A bounded warning fallback is the safest first step, while automatic parsing should go through the shared document-extractor boundary with limits.
