## Label Justifications

- `P2`: The issue affects a common agent tool path and can waste context, but it is not an outage or urgent regression.
- `impact:session-state`: Raw binary document bytes can pollute agent-visible context and degrade the session transcript/state used for reasoning.

## Summary

Keep open: current main and the latest release still route non-image document reads through the generic read path, and there is an open closing PR for this issue that has not landed. The remaining decision is the product boundary between a bounded safe fallback and parser-backed extraction.

## What This Changes

The issue asks OpenClaw to prevent the `read` tool from returning raw binary document bytes for formats like `.docx`, `.pdf`, `.pptx`, and `.xlsx`, either by warning or extracting text.

## Best Possible Solution

Choose one bounded document-read path: keep `read` from emitting raw binary bytes by default, route parser-backed extraction through the shared document-extractor boundary when supported, and land or close the linked PRs accordingly.

## Reproduction Assessment

Yes, source-reproducible. Current main's wrapper only adds adaptive paging and image normalization/sanitization, while the locked upstream read implementation converts non-image buffers to UTF-8 text.

## Solution Assessment

Unclear as a single implementation direction. The safe fallback is a good first step, but parser-backed extraction should use the shared document-extractor boundary with bounded resource handling rather than a parallel one-off core parser.
