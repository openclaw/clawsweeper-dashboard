## Summary

Keep open. Current main still lacks a `read`-path binary document fallback, and the open PR that uses closing syntax for this issue has not merged.

## What This Changes

The issue asks OpenClaw to stop the `read` tool from returning raw binary document contents for formats such as DOCX, PDF, PPTX, and XLSX.

## Best Possible Solution

Land or replace the focused read-wrapper guard with bounded MIME/file-type handling and regression coverage, while keeping richer Office parsing as separate document-extraction work.

## Reproduction Assessment

Yes. Source inspection shows current main delegates non-image reads to `@mariozechner/pi-coding-agent@0.73.1`, whose non-image branch UTF-8-decodes file bytes, so OOXML ZIP payloads can still surface as `PK...` text.

## Solution Assessment

Yes for the safe-fallback direction, but current main does not implement it. The narrowest maintainable fix is a bounded read-wrapper guard for known binary document formats, with dedicated parser tools handled separately.
