## Summary

Keep open: current main and latest release still decode non-image `read` targets as text, and the only open linked fix PR is a conflicting fork PR that needs maintainer disposition.

## What This Changes

Requests that the `read` tool avoid returning raw binary Office/PDF bytes by warning on binary document formats or routing them to document extraction.

## Best Possible Solution

Land or sponsor a narrow read-tool safe fallback that prevents raw binary document bytes from entering context, while tracking native extraction and provider forwarding in the broader document-input issues.

## Reproduction Assessment

Yes, at source level. Current main only detects images before non-image files are decoded as text, so a DOCX ZIP/PDF buffer can become `read` tool text output; I did not run a live binary read in this read-only review.

## Solution Assessment

Unclear pending maintainer direction. A safe fallback is likely the narrowest first fix, while automatic document parsing and provider-native forwarding need explicit ownership and security decisions.
