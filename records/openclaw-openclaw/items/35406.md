## Summary

Keep open: current main and v2026.6.5 still decode non-image reads as UTF-8, and the open linked implementation PR has not merged and is currently draft/conflicting.

## What This Changes

Request that `read` avoid returning raw Office/PDF binary bytes by either giving actionable guidance or extracting document text.

## Best Possible Solution

Add bounded MIME or prefix detection in the canonical read backend, route PDFs toward the existing `pdf` tool, and return clear guidance for unsupported Office formats until owned extraction exists.

## Reproduction Assessment

Yes. Source inspection shows non-image reads reach `buffer.toString("utf-8")`, so Office/PDF bytes can become model-visible text; this read-only review did not run a live binary-file repro.

## Solution Assessment

Unclear. The safe-fallback direction is maintainable, but maintainers still need to choose rejection versus extraction and ensure any linked PR uses bounded detection in the canonical read backend.
