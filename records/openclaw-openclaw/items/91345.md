The PR adds a CLI command-suggestion helper, wires closest-command hints into Commander unknown-command output and the early unowned-root guard, and adds focused formatter and runCli regression tests.

## Best Possible Solution

Keep the shared root suggestion helper, but only show root-command suggestions for root unknown commands or make Commander formatting use the current command's candidate list and command prefix.

## Reproduction Assessment

Yes. Source inspection shows Commander subcommands inherit the root output configuration and OpenClaw has nested commands like openclaw plugins update, while the added formatter call only scores root-command suggestions.

## Solution Assessment

No, not as-is. The root-command helper and early-guard wiring are the right shape for the central issue, but the Commander formatter needs root-only gating or nested command context before merge.
