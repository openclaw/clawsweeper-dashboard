## Summary

This PR still addresses a real prompt-starvation bug, but it is not a current merge candidate: it conflicts with main, edits removed Pi settings files, lacks external real behavior proof, and changes upgrade-visible reserve behavior without a maintainer compatibility decision. Current main has the related floor cap and warning infrastructure, but it does not fully implement this PR's requested behavior of lowering an already-too-high configured/current reserve.

## What This Changes

The branch clamps Pi compaction reserve settings to half of the known context window and changes context-overflow recovery text when the configured reserve consumes too much prompt budget.

## Best Possible Solution

Move the fix to the active agent-settings and Codex mapping surfaces, settle the compatibility semantics for lowering oversized reserves, and merge only after redacted overflow-path proof or after the maintainer candidate lands cleanly.
