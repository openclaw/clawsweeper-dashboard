## Summary

Keep open: this PR targets a real Slack attachment failure, but the current patch strips the guard-created pinned dispatcher from the final media request and still needs maintainer security approval, a current-main refresh, and inspectable after-fix proof.

## What This Changes

The branch strips the SSRF guard dispatcher from Slack media fetch init before delegating to runtime fetch, updates the focused Slack media test, and rewrites Slack raw-fetch allowlist line numbers.

## Best Possible Solution

Land a refreshed Slack media fix that removes the Undici conflict while preserving Slack auth, redirect behavior, and the intended SSRF transport contract, either with equivalent dispatcher enforcement or an explicitly approved Slack preflight-only exception.

## Reproduction Assessment

Yes, source-reproducible with high confidence. The linked issue gives Slack Socket Mode/container steps and invalid onRequestStart logs, and current main still contains the dispatcher handoff path; I did not run a live Slack workspace.

## Solution Assessment

No, not as currently proposed. Stripping the dispatcher matches the observed failure mode, but it removes pinned dispatcher enforcement from the final media request unless maintainers approve that Slack-specific SSRF contract or choose an enforcement-preserving replacement.
