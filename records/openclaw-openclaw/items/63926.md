## Summary

Keep open. Current main still lacks the requested Control UI browser-login handoff panel, the live discussion treats this as the canonical surface, and the remaining work needs UX plus security scoping rather than conservative cleanup.

## What This Changes

## Best Possible Solution

Ship a generic admin-scoped Browser area in Control UI, backed by browser.request or additive typed gateway methods, that shows readiness, tabs, manual-action node/profile/tab context, diagnostics, safe tab actions, and a clear resume path after login or permission prompts.

## Solution Assessment

Unclear until the UX and security boundaries are scoped. The requested direction is useful, but the maintainable solution should be a generic Control UI browser handoff surface rather than a Google Meet-only panel or CLI-only workaround.
