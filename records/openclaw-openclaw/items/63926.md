Keep open: current main and v2026.6.10 still have browser-control primitives and CLI/docs handoff guidance, but no visible Control UI browser-login handoff panel; the issue discussion also explicitly treats this as the canonical handoff tracker.

## What This Changes

Add a visible Control UI browser handoff panel for managed browser status, active tabs, tab actions, manual login, and same-state agent resume.

## Best Possible Solution

Design and ship an operator-scoped browser handoff surface with typed profile/tab context, read-only status by default, explicit write controls, redaction, and same-state resume semantics coordinated with https://github.com/openclaw/openclaw/issues/27574 and https://github.com/openclaw/openclaw/pull/80388.

## Solution Assessment
