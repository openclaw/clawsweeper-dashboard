## Summary

Keep open. Current main still lacks the requested Control UI browser login-handoff panel, and the live discussion explicitly treats this as the canonical browser-login handoff surface rather than stale cleanup or a duplicate.

## What This Changes

Request to add a visible Control UI browser section for managed-browser status, active tabs, user login handoff, tab actions, and agent resume after manual login.

## Best Possible Solution

Ship a generic admin-scoped Browser area in Control UI that uses browser.request or additive typed gateway methods to show profile/readiness, tabs, manual-action node/profile/tab context, diagnostics, safe tab actions, and a clear resume path after login or permission prompts.

## Solution Assessment
