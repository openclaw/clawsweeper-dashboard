## Summary

Keep open: current main and the latest stable release still lack a visible Control UI browser-login handoff panel, and the live discussion explicitly treats this as canonical feature work needing product and security scoping.

## What This Changes

Request a visible Control UI Browser panel for managed browser status, tabs, safe tab actions, manual login handoff, and agent resume against the same browser state.

## Best Possible Solution

Ship a generic admin-scoped Browser area in Control UI that reuses browser.request or additive typed gateway methods to show profile/readiness, active tabs, manual-action node/profile/tab context, diagnostics, safe tab actions, and a clear resume path after login or permission prompts.

## Solution Assessment

Unclear until maintainers scope the UX and security model. The maintainable direction is a generic secured Control UI browser handoff surface, not a Google Meet-only panel or a broad terminal/browser co-working PR with new execution surfaces.
