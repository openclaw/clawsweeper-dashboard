## What This Changes

Request a visible Control UI browser panel for managed-browser status, tabs, user login handoff, and resume against the same browser session state.

## Best Possible Solution

Ship a secured, generic Control UI browser handoff surface with typed browser/session context, read-only status and tab visibility by default, explicit write/login handoff actions, safe same-profile resume semantics, and matching docs/security guidance.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers scope the UX, security boundary, and runtime handoff contract. The requested direction is useful, but the best solution should be a generic secured Control UI browser handoff surface rather than a service-specific workaround or raw browser-control tab bundle.
