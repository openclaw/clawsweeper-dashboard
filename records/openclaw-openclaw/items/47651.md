## Summary

Keep open. Current main implements the narrower Gemini Embedding 2 path for opt-in image/audio files discovered through `memorySearch.extraPaths`, but PDF/video modalities and a user-sent-media remember workflow are still outside the public memory contract.

## What This Changes

Request support for embedding non-Markdown media files and remembered inbound attachments with Gemini Embedding 2 so `memory_search` can retrieve images, video, audio, and PDFs alongside text notes.

## Best Possible Solution

Keep the shipped image/audio `extraPaths` foundation, then split the remaining work into an explicit PDF/video/document modality contract and a separate opt-in retained-media ingestion path for attachments users ask the agent to remember.

## Reproduction Assessment

Not applicable as a bug reproduction. There is a high-confidence current-main verification path by inspecting the modality classifier, schema, scanner, docs, tests, and attachment staging code, which show image/audio `extraPaths` support only.

## Solution Assessment

No, the current shipped subset is not the complete solution for this request. The maintainable path is an explicit public contract for new media modalities and retained attachments rather than silently indexing every inbound file.
