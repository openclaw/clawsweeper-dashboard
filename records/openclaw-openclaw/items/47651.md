## Summary

Keep open. Current main covers an opt-in Gemini image/audio indexing subset under `memorySearch.extraPaths`, but PDF/video modalities and a user-sent-media remember flow remain outside the public memory contract and need product/security review.

## What This Changes

Request support for indexing images, video, audio, and PDFs through Gemini Embedding 2 in memory search, including media sent to an agent and marked worth remembering.

## Best Possible Solution

Keep the shipped image/audio `extraPaths` foundation, then design PDF/video support as an explicit memory modality contract and handle user-sent attachments through a separate opt-in retained-media ingestion path with clear privacy and provider-upload rules.

## Reproduction Assessment

Not applicable as a bug reproduction. There is a high-confidence current-main verification path by inspecting the classifier, scanner, schema, docs, tests, and attachment staging code, which show image/audio `extraPaths` support only.

## Solution Assessment

No, the current shipped subset is not a complete solution for this request. The narrow maintainable path is an explicit public contract for new media modalities and retained attachments, not silently indexing every inbound file.
