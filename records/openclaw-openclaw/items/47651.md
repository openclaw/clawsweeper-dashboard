## Summary

Keep open. Current main implements an opt-in Gemini image/audio file-indexing subset, but PDF/video indexing and user-sent media marked for remembering remain outside the public memory contract and need product/security decisions.

## What This Changes

Request multimodal memory indexing for images, video, audio, and PDFs via Gemini Embedding 2, including user-sent media marked worth remembering.

## Best Possible Solution

Keep the image/audio foundation, then design PDF/video modalities and remembered inbound attachments as explicit memory contracts with size, MIME, retention, deletion, search-result, and provider-upload rules.

## Reproduction Assessment

Not applicable as a bug reproduction. Source, docs, and tests on current main give high-confidence verification that only image/audio file indexing is public, with no PDF/video or retained inbound-media indexing path.

## Solution Assessment

No. The current implementation is a useful subset, but the narrow maintainable solution is an explicit public modality and retained-attachment contract rather than silently indexing every inbound file.
