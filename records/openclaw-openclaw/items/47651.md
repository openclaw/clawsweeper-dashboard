## Summary

Keep open. Current main ships a useful Gemini image/audio indexing subset for `memorySearch.extraPaths`, but the requested PDF/video modalities and user-sent-media remember flow are still outside the public memory contract.

## What This Changes

Request support for indexing non-Markdown media, including images, video, audio, PDFs, and user-sent remembered media, with Gemini Embedding 2 in the memory search index.

## Best Possible Solution

Keep the image/audio `extraPaths` foundation and design PDF/video plus remembered inbound attachments as explicit memory contracts with size, MIME, result-display, retention, and provider-upload rules.

## Reproduction Assessment

Not applicable as a bug reproduction. Current-main source verification is high-confidence: implementation, schema, docs, and tests show image/audio `extraPaths` support only, with no PDF/video or retained inbound-media indexing path.

## Solution Assessment

No. The shipped subset is useful but not a complete solution for this request; the narrow maintainable path is an explicit public memory modality and retained-attachment contract rather than silently indexing every inbound file.
