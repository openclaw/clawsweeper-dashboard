## Summary

Keep open: current main and latest release still have timer-only Matrix crypto IDB persistence, and the synchronous shared-client stop helpers still return without an awaitable final persist. The open stop-helper PR is partial and does not address the event-driven crypto persistence half of the reported failure.

## What This Changes

Fix Matrix E2EE crypto-store durability so newly acquired crypto state is flushed before restart and shared-client shutdown paths can await final persistence.

## Best Possible Solution

Keep this issue as the canonical tracker, land or reshape the partial stop-helper PR only with owner-approved API direction, and add debounced Matrix crypto-event persistence with listener cleanup and focused Matrix SDK tests.

## Reproduction Assessment

Yes, source-level: current main still persists Matrix crypto IDB once at init and then on a 60s timer, while shared stop helpers call stop without awaiting final persistence. A live Matrix restart repro was not run in this read-only review.

## Solution Assessment

Unclear as an implementation plan: the requested two-part direction is coherent, but the exact crypto-event trigger set, helper API/deprecation shape, and live proof standard need maintainer acceptance before implementation is safe to automate.
