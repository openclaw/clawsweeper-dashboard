Keep open: current main still has both Matrix crypto persistence gaps, and https://github.com/openclaw/openclaw/pull/76709 only covers the awaitable shared-client shutdown half while the event-driven crypto-store flush remains unimplemented. ## What This Changes

The issue requests Matrix E2EE crypto-store persistence on crypto state changes and awaitable shared-client shutdown paths so Olm/Megolm state survives restarts.

Keep the issue open, land or replace the async stop-helper PR for the shutdown half, and add Matrix-owned debounced crypto-event snapshot flushing with listener cleanup and regression tests.
