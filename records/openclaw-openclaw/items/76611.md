## Summary

Keep open: current main and the latest release still have the timer-only Matrix crypto IDB flush and non-awaitable shared-client stop helpers, while https://github.com/openclaw/openclaw/pull/76709 only addresses the shutdown-helper half and has not merged.

## What This Changes

The issue asks OpenClaw to persist Matrix rust-crypto IDB state sooner than the 60s timer and expose or use shutdown helpers that await final persistence so E2EE state survives restarts.

## Best Possible Solution

Keep this as the canonical Matrix crypto-store persistence task: land or replace https://github.com/openclaw/openclaw/pull/76709 for the async shutdown-helper half, then add debounced crypto-event IDB flushing with listener cleanup and focused regression coverage.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only pass. Source inspection does reproduce the reported lifecycle gaps: Matrix crypto state is persisted only on init, final shutdown, and a 60s interval, while the shared synchronous stop helpers return after `client.stop()` without awaiting the final persist promise.

## Solution Assessment

Yes, the right fix direction is inside the Matrix plugin's crypto/client lifecycle: keep the periodic fallback, add debounced persistence on typed rust-crypto state-change events, and use additive awaitable stop helpers rather than breaking existing synchronous exports without a compatibility decision.
