## Label Justifications

## Summary

Keep open: current main and v2026.6.6 still show timer-only Matrix crypto snapshot persistence plus void shared-client stop helpers, and the only related open PR covers only the shutdown-helper half while explicitly leaving event-driven persistence unresolved.

## What This Changes

The issue asks for Matrix E2EE crypto state to persist promptly on meaningful crypto events and for shared-client shutdown helpers to await final persistence so encrypted sessions survive restarts.

## Best Possible Solution

Land a bounded Matrix plugin fix that debounces crypto-event persistence with listener cleanup and resolves the shared shutdown-helper path, either by finishing https://github.com/openclaw/openclaw/pull/76709 plus a focused follow-up or by replacing it with one coordinated fix.

## Reproduction Assessment

Yes for source-level reproduction: current main still shows the 60s IDB persistence interval and void shared stop helpers, and a user reports the same Matrix/Element failure on OpenClaw 2026.5.28. I did not run a live homeserver restart reproduction in this read-only pass.

## Solution Assessment

Yes, broadly: event-driven persistence plus awaited shutdown targets the actual durability gap. The exact event list and shared-helper API shape still need maintainer review and dependency-backed tests.
