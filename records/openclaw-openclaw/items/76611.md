## Summary

Keep open. Current main still persists the Matrix rust-crypto IndexedDB snapshot only after init, on the 60s interval, and during shutdown; the shared stop helpers still call `stop()` without exposing or awaiting the final persistence promise.

## What This Changes

The issue asks for Matrix E2EE crypto-store persistence to flush on crypto state changes and for shared-client shutdown paths to await final persistence where clean shutdown matters.

## Best Possible Solution

Keep the fix in the Matrix plugin: add debounced crypto-event snapshot flushing, clean listener teardown, and an awaited persist shutdown path for callers that need clean Matrix E2EE shutdown.

## Reproduction Assessment

Yes for the source-level defect: current main has a deterministic path where crypto-state changes after init are not flushed until the 60s interval or shutdown, and several stop helpers return before the final persist promise resolves. I did not live-reproduce the downstream `InvalidMAC` loop against a homeserver.

## Solution Assessment

Yes, the requested direction is the narrowest maintainable fix: add debounced Matrix crypto-event persistence and route clean shutdown callers through an awaited persist API. This should remain plugin-owned and should not require a core API change.
