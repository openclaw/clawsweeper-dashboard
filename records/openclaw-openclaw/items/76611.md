## Summary

Keep open. Current main still has the Matrix crypto persistence gap, and the linked open PR only addresses the shared-client stop-helper half, so the issue should not close before the event-driven flush path is resolved or split into a canonical follow-up.

## What This Changes

The issue asks for Matrix rust-crypto IndexedDB state to be persisted promptly after crypto state changes and for shared-client shutdown helpers to expose an awaitable final persist path.

## Best Possible Solution

Keep the fix in the Matrix plugin: add debounced crypto-event snapshot flushing with listener cleanup, and make clean shutdown paths await final IDB/sync persistence; decide whether #76709 should be expanded or split.

## Reproduction Assessment

Yes for the source-level defect: current main persists rust-crypto IDB state after init, on the 60s timer, and on shutdown, while the shared stop helpers still return before final persistence resolves. I did not live-reproduce the downstream InvalidMAC loop against a homeserver.

## Solution Assessment

Yes. Debounced Matrix crypto-event persistence plus an awaitable shared-client persist shutdown path is the narrow maintainable fix; #76709 alone is not complete because it leaves initializeCryptoIfNeeded unchanged.
