## Summary

Keep open. Current main still has a source-level Matrix E2EE persistence gap: crypto IndexedDB state is flushed after init, on the 60s interval, and at shutdown, while several shared stop helpers still expose only fire-and-forget `stop()` behavior.

## What This Changes

The issue asks for Matrix rust-crypto IndexedDB state to flush on crypto state changes and for shared-client clean shutdown paths to await final persistence.

## Best Possible Solution

Keep the repair in the Matrix plugin: add debounced crypto-event snapshot flushing with listener teardown, preserve the 60s fallback, and route clean shutdown callers through an awaited persist path.

## Reproduction Assessment

Yes for the source-level defect: current main deterministically leaves post-init crypto changes unflushed until the 60s interval or shutdown, and some shared stop helpers return before the shutdown persist promise resolves. I did not live-reproduce the reported Matrix `InvalidMAC` loop.

## Solution Assessment

Yes. A Matrix-plugin-owned fix that debounces verified crypto events and awaits clean-shutdown persistence is the narrowest maintainable path; it does not require a core API or config change.
