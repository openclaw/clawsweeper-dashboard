## Summary

Keep open. Current main still has the reported Matrix crypto persistence gap: new rust-crypto IndexedDB state is flushed after init, on the 60s timer, and during shutdown, while several shared-client stop helpers still return without awaiting the final persistence promise.

## What This Changes

The issue requests Matrix plugin changes so crypto-store updates flush promptly and clean shutdown paths await final persistence before returning.

## Best Possible Solution

Keep the repair in the Matrix plugin by adding debounced crypto-event snapshot flushing with listener cleanup, then add or route callers through an awaitable shared-client persist shutdown path.

## Reproduction Assessment

Yes for the source-level defect: current main has a deterministic path where crypto-state changes after initialization are not flushed until the 60s interval or shutdown, and several stop helpers return before final persistence resolves. I did not live-reproduce the downstream InvalidMAC loop against a Matrix homeserver.

## Solution Assessment

Yes. The requested direction is the narrowest maintainable fix: add debounced Matrix crypto-event persistence and make clean shutdown callers use an awaited persistence API, without changing core plugin APIs or adding configuration.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/76611 in the Matrix plugin only. Add debounced event-driven crypto IndexedDB snapshot flushing in MatrixClient.initializeCryptoIfNeeded for verified matrix-js-sdk CryptoEvent signals such as KeyBackupDecryptionKeyCached, RehydrationCompleted, DevicesUpdated, and KeysChanged; keep the 60s interval as a fallback and clean up any registered listeners/timers on shutdown. Add an awaitable shared-client stop-and-persist API or route clean shutdown callers through releaseSharedClientInstance(..., "persist")/stopAndPersist so callers that need durable E2EE state can await final persist. Do not add core plugin APIs, config options, dependency changes, or broad Matrix product behavior. Add focused regressions in sdk.test.ts and shared.test.ts, and add a CHANGELOG.md fix entry because this is user-facing Matrix E2EE reliability work.
