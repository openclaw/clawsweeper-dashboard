## Summary

Keep open: current main still has the Matrix crypto persistence gap, and the open related PR only covers the awaitable shared-client shutdown half while event-driven crypto-store flushing remains unimplemented.

## What This Changes

The issue asks for Matrix E2EE crypto-store state to be persisted on relevant crypto events and for shared-client shutdown paths to await final persistence so decryption state survives restarts.

## Best Possible Solution

Keep this issue open, land or replace the open async stop-helper PR for the shutdown half, and add a Matrix-owned debounced crypto-event snapshot flush with listener cleanup and regression tests for the remaining persistence gap.

Yes for a source-level reproduction: current main has a deterministic path where post-init crypto state waits for the 60s timer or shutdown, and several shared-client stop helpers return before the final persist promise resolves. I did not establish the downstream InvalidMAC loop against a live homeserver in this review.

## Solution Assessment

Yes: the narrow maintainable fix is to keep the repair inside the Matrix plugin, add debounced persistence on dependency-backed crypto events with cleanup, and route clean shutdown paths through an awaitable persist API without changing config or core plugin contracts.

## Repair Work Prompt

Fix the remaining Matrix E2EE crypto-store persistence gap from https://github.com/openclaw/openclaw/issues/76611. In `extensions/matrix/src/matrix/sdk.ts`, after rust crypto initializes, register debounced persistence handlers on dependency-backed `CryptoEvent` signals already used by `sdk/decrypt-bridge.ts` (`KeyBackupDecryptionKeyCached`, `RehydrationCompleted`, `DevicesUpdated`, `KeysChanged`) so `persistIdbToDisk({ snapshotPath: this.idbSnapshotPath, databasePrefix: this.cryptoDatabasePrefix })` runs promptly without waiting for the 60s fallback timer. Track listener registrations and remove them on shutdown/stop paths to avoid leaking handlers; keep the existing periodic timer as a safety net.
