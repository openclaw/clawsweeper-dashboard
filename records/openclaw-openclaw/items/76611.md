## Summary

Keep open. Current main still has the Matrix crypto persistence gap: rust-crypto IndexedDB state is flushed after init, on the 60s timer, and during shutdown, while synchronous shared-client stop helpers still return before final persistence resolves; #76709 is open but only covers the awaited-helper half.

## What This Changes

The issue requests Matrix plugin crypto-store flushing after crypto state changes plus awaitable shared-client shutdown helpers so E2EE keys survive restarts.

## Best Possible Solution

Keep the fix in the Matrix plugin: land or extend #76709 for awaited shutdown, then add debounced crypto-event IDB snapshot flushing with listener cleanup and regression tests.

## Reproduction Assessment

Yes for the source-level defect: current main has a deterministic path where crypto-state changes after initialization are not persisted until the 60s timer or shutdown, and several shared stop helpers return before final persistence resolves. I did not live-reproduce the Matrix InvalidMAC loop.

## Solution Assessment

Yes. The narrowest maintainable solution is Matrix-plugin-local persistence repair: add debounced crypto-event flushing and route clean shutdown callers through an awaited persistence API; #76709 covers only part of that path so far.
