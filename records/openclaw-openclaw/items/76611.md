## Summary

Keep open: current main and latest release v2026.6.10 still have timer-based Matrix crypto IDB persistence, and the shared stop helpers still return without awaiting final persistence. The open linked PR only addresses the stop-helper half, while the event-driven crypto-state flush remains unresolved.

## What This Changes

The issue asks OpenClaw to make Matrix E2EE crypto-state persistence durable across restarts by adding event-driven IDB flushes and awaitable shared-client shutdown paths.

## Best Possible Solution

Keep this issue as the canonical Matrix crypto persistence lifecycle tracker, review the partial stop-helper PR separately, and land an owner-approved debounced crypto-event persistence path with listener cleanup and focused Matrix SDK tests.

## Reproduction Assessment

Yes, source-level: current main still persists Matrix crypto IDB once at init and then on a 60s timer, while shared stop helpers call stop without awaiting final persistence. A live self-hosted Matrix restart repro was not run in this read-only review.

## Solution Assessment

No, current main is not a complete solution. The maintainable fix needs both awaited shutdown behavior and debounced event-driven crypto persistence wired to verified matrix-js-sdk/Rust crypto signals.
