## Summary

Keep this open. Current main still receives Telegram reactions as notification/context events only; it does not wake an agent turn, normalize Telegram custom emoji reactions, or provide configurable reaction semantics, and the issue is already paired with open implementation PR #47687 plus the narrower open bug #64752.

## What This Changes

Request first-class Telegram reaction triggers that can wake authorized agent execution, normalize standard and custom emoji, support per-emoji operator semantics, and expose useful debugging signals.

## Best Possible Solution

Review #47687 against #64752 and either land it or replace it with a focused Telegram-plugin change that normalizes standard and custom emoji into stable keys, maps configured reaction actions, wakes only authorized mapped reactions, logs ignored/rejected/executed decisions, and updates targeted tests plus the public Telegram docs.
