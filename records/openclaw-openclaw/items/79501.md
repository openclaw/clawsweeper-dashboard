## Best Possible Solution

Rebase the PR and make ambient heartbeat a single runner-level contract: ambient mode should reliably select the ambient `heartbeat_respond` and message-budget prompt, or be explicitly documented and tested as supported only in chosen delivery modes.

## Reproduction Assessment

## Solution Assessment

No. Adding the helper is plausible, but the best fix is to select ambient semantics at the runner decision point and test schema plus `runHeartbeatOnce` behavior across default automatic and response-tool modes before adding public config docs.
