## Summary

Keep open: the duplicate-param display fix is useful, but the shared redaction output also feeds git plugin managed-repo cache keys, so the patch can change update behavior for existing authenticated git installs with repeated secret query params. The external PR also lacks contributor after-fix real behavior proof beyond tests/CI commands.

## What This Changes

The branch updates `redactSensitiveUrl` to rebuild query params entry-by-entry when masking sensitive keys and adds a regression test for duplicate token parameters.

## Best Possible Solution

Preserve duplicate redacted query params for display/log surfaces while keeping git install cache identity stable through a separate secret-free key or migration-backed compatibility test, then add real after-fix proof.
