## Summary

Keep open: current main still targets Android minSdk 31, and this branch is conflicting with an outstanding changes-request. The PR has useful compatibility work, but lowering the support floor to API 26 still needs a current-main rebase, a broader API 26-30 audit, runtime motion gating, and stronger device proof before merge.

## What This Changes

The PR lowers the Android app minSdk from 31 to 26 and adds older-API fallbacks for theme colors, foreground service startup, DNS discovery, permissions, notification access, location, thermal status, and motion UI state.

## Best Possible Solution

Treat API 26 support as a deliberate Android support-floor project: rebase, audit every reachable API 29/30/31+ path, share capability gates from UI through runtime command handling, and require API 26-28 proof before lowering minSdk.
