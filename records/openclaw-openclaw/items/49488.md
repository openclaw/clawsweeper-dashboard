## Summary

Keep open: current main still targets Android API 31, and this PR remains a product-level support-floor change with unresolved API 26-30 compatibility gaps and insufficient runtime proof.

## What This Changes

The branch lowers the Android app minSdk from 31 to 26 and adds compatibility fallbacks across theme, foreground service, DNS discovery, location, notification, thermal, and motion UI code.

## Best Possible Solution

Decide the Android support floor explicitly; if API 26 is accepted, land a rebased compatibility pass that guards all API 29/30/31+ surfaces and includes real emulator or device proof for the affected flows.

## Reproduction Assessment

Yes for the review blockers by source inspection: lowering minSdk to 26 makes API 29 RoleManager and ACTIVITY_RECOGNITION-dependent motion paths reachable on API 26-28. I did not run an emulator in this read-only review.

## Solution Assessment

No. The PR has useful compatibility work, but it is not a mergeable solution until maintainers accept the support floor, the branch is rebased, the remaining API gates are fixed, and runtime proof is added.
