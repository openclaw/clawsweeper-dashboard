## Summary

Keep open: current main still targets Android API 31, and this PR is a broader Android support-floor feature with active requested changes around API 26-30 compatibility rather than a cleanup item that is already implemented or safe to close.

## What This Changes

The PR lowers the Android app minSdk from 31 to 26 and adds compatibility guards/fallbacks for theme colors, foreground service startup, DNS discovery, location capture, notification listener access, thermal status, and motion-related UI/device reporting.

## Best Possible Solution

Decide the Android support floor explicitly; if API 26 is accepted, rebase this work and land a comprehensive compatibility pass with guarded Q/R/S+ APIs, runtime capability alignment, and API 26, 29, 30, and current-device validation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: apply the PR's minSdk 26 floor to current main, then open Settings on API 26-28 for the unguarded RoleManager path or expose motion capability on a sensor-equipped API 26-28 device while motion invocation still requires ACTIVITY_RECOGNITION.

## Solution Assessment

No. The PR is a useful start and includes real compatibility fixes, but it is not the best mergeable solution until the Android support-floor decision, remaining API-level guards, runtime capability alignment, and rebase are resolved.
