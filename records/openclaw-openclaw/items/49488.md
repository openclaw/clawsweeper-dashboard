## Summary

Keep open: the direction has useful user validation for API 26/Android 11, but current main still ships minSdk 31 and this PR remains a broad Android support-floor change with unresolved API compatibility blockers and an explicit maintainer concern that API 26 support needs a dedicated decision.

## What This Changes

The PR lowers the Android app minSdk to 26 and adds pre-Android 12 color fallback plus API-level guards for foreground service, DNS, location, notifications, thermal, motion, and permission UI paths.

## Best Possible Solution

Decide the Android support floor explicitly; if API 26 is accepted, rebase this work and land a comprehensive compatibility pass with guarded API 29/30/31+ surfaces, runtime capability alignment, changelog/docs updates, and API 26, 29, 30, and current-device validation.

## Solution Assessment
