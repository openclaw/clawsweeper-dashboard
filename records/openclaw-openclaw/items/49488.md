## Summary

Keep open: current main still targets Android minSdk 31, and this PR remains a broader API 26 support-floor change with a human changes-requested review, unresolved compatibility findings, and insufficient real-device behavior proof.

## What This Changes

The PR lowers the Android app minSdk from 31 to 26 and adds older-API fallbacks or guards for theme colors, foreground service startup, DNS discovery, location, notification access, thermal state, and motion UI paths.

## Best Possible Solution

Decide the official Android support floor; if API 26 is accepted, rebase and land a complete API 26-30 compatibility pass with guards for every reachable newer-API surface and device or emulator proof for affected flows.

## Reproduction Assessment

Yes for the review blockers by source inspection: lowering minSdk to 26 makes current-main assistant-role and motion-permission/runtime paths reachable on API 26-28 without complete guards. I did not run an emulator in this read-only review.

## Solution Assessment

No. Lowering minSdk may be worthwhile, but the mergeable path is a maintainer-approved support-floor decision plus a rebased compatibility audit, not a partial fallback patch with limited proof.
