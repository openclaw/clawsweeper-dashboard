## Summary

Keep this PR open, but it is not merge-ready: the Android API 26 support direction is useful, yet current review evidence still shows pre-29 compatibility gaps, a conflicting/stale branch, and insufficient real behavior proof for the affected runtime paths.

## What This Changes

The PR lowers the Android app minimum SDK from 31 to 26 and adds compatibility fallbacks for theme colors, foreground service startup, DNS discovery, location, notification access, thermal state, and motion UI permission reporting.

## Best Possible Solution

Decide the Android support floor explicitly; if API 26 is accepted, land a rebased compatibility pass that guards all API 29/30/31+ surfaces and includes real emulator or device proof for the affected flows.

## Reproduction Assessment

Yes for the review blockers by source inspection: lowering minSdk to 26 makes current Settings RoleManager calls reachable on API 26-28, and PR-head motion capability reporting can still advertise an ungrantable ACTIVITY_RECOGNITION path. I did not run an emulator in this read-only review.

## Solution Assessment

No. The PR contains useful compatibility work, but it is not the best mergeable solution until the support-floor decision is made, the branch is rebased, the remaining API gates are fixed, and real runtime proof is added.
