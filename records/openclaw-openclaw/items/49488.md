## Summary

Keep open: current main and the latest shipped release still keep Android `minSdk` at 31, and this PR remains a product-level support-floor expansion with changes requested, source-reproducible lower-API gaps, and insufficient after-fix proof.

## What This Changes

The PR lowers the Android app minimum SDK from 31 to 26 and adds compatibility guards or fallbacks for Android 8-11 runtime paths.

## Best Possible Solution

Decide the Android support floor explicitly; if API 26 is accepted, rebase this work and land a comprehensive API 26-30 compatibility pass with guarded API 29/30/31+ surfaces and real device or emulator proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path once the PR lowers `minSdk` to 26: Settings reaches `RoleManager` on API 26-28, and runtime motion commands can still be advertised from sensor availability while invocation requires an API-29 permission.

## Solution Assessment

No. The PR contains useful compatibility work, but it is not the best mergeable solution until the support-floor decision is made and the remaining API 26-30 gates are fixed and validated.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Guard assistant-role APIs below Android Q:** `apps/android/app/build.gradle.kts:66`
  - body: Lowering `minSdk` to 26 makes API 26-28 devices supported, but Settings still initializes and launches `RoleManager` without an SDK guard. `RoleManager` is API 29+, so opening Settings or tapping the assistant-role control can crash on pre-Q devices; hide or guard this block below Q.
  - confidence: 0.88
- **[P2] Gate runtime motion availability below Android Q:** `apps/android/app/src/main/java/ai/openclaw/app/tools/Features.kt:9`
  - body: `Features.Motion` hides some UI on pre-Q devices, but runtime command advertisement still comes from `MotionHandler` sensor availability while invocation requires `ACTIVITY_RECOGNITION`, an API-29 permission. API 26-28 devices can still advertise motion commands that cannot succeed; include the SDK gate in runtime availability or handler behavior.
  - confidence: 0.87
