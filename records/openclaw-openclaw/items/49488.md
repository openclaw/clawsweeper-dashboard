## Summary

Keep open. The PR has useful Android compatibility work, but lowering the support floor to API 26 is still a product decision, current main remains on minSdk 31, maintainer review has requested changes, and source review still shows lower-API gaps that need a rebased audit and stronger runtime proof.

## What This Changes

The PR lowers the Android app minSdk from 31 to 26 and adds compatibility fallbacks across theme, foreground service, DNS discovery, location, notification access, thermal state, and motion UI/device permission paths.

## Best Possible Solution

Decide the Android support floor explicitly; if API 26 is accepted, rebase and land a complete API 26-30 compatibility pass with guards for every reachable API 29+ surface and device/emulator proof for the affected flows.

## Reproduction Assessment

Yes for the review blockers by source inspection: lowering minSdk to 26 makes API 29+ Android APIs reachable on API 26-28 unless every current-main path is guarded. I did not run an emulator in this read-only review.

## Solution Assessment

No. The direction may be valuable, but this PR is not the best mergeable solution until maintainers accept the support floor, the branch is rebased, remaining API gates are fixed, and runtime proof is added.
