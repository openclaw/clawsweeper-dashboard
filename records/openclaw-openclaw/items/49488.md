## Summary

Keep open, but not merge-ready: current main still enforces and documents minSdk 31, the live PR has changes requested plus a failing real-behavior-proof gate, and source inspection still leaves API 26-28 compatibility work around assistant-role and motion runtime paths.

## What This Changes

The PR lowers the Android app minSdk from 31 to 26 and adds compatibility fallbacks for theme colors, foreground service startup, DNS discovery, location, notification access, thermal state, and motion UI/device permission paths.

## Best Possible Solution

Decide the Android support floor explicitly; if API 26 is accepted, rebase and land a complete API 26-30 compatibility pass with guards for every reachable newer-API surface and device or emulator proof for the affected flows.

## Reproduction Assessment

Yes for the review blockers by source inspection: lowering minSdk to 26 makes newer Android APIs reachable on API 26-28 unless every current-main path is guarded. I did not run an emulator in this read-only review.

## Solution Assessment

No. The direction may be valuable, but this PR is not the best mergeable solution until maintainers accept the support floor, the branch is rebased, remaining API gates are fixed, and runtime proof is added.
