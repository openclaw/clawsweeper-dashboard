## Summary

Keep open. The branch contains useful Android compatibility work, but lowering the support floor is still a support-contract change: the PR is merge-conflicting, has an unresolved changes-requested review, leaves at least the motion command runtime availability inconsistent with the API 29 permission contract, and still needs stronger real API 26-28 proof before merge.

## What This Changes

The PR lowers the Android app minSdk from 31 to 26 and adds older-Android fallbacks or guards across theme colors, foreground service startup, DNS discovery, location, notifications, thermal status, and motion UI paths.

## Best Possible Solution

Treat API 26 support as a deliberate Android support-floor project: rebase the PR, audit every reachable API 29/30/31+ path, share platform capability gates across UI and runtime advertisement, and require real API 26-28 plus current-device proof before changing minSdk.
