## Summary

Keep open: current main still targets Android minSdk 31, and this useful support-floor PR is not merge-ready because it conflicts with main, has changes requested, and still leaves older-API runtime command/proof gaps.

## What This Changes

The branch lowers the Android app minSdk to 26 and adds older-API guards or fallbacks for theme colors, foreground service startup, DNS discovery, location, notification access, thermal status, and motion UI/permission surfaces.

## Best Possible Solution

Treat API 26 support as a deliberate Android support-floor project: rebase, audit every reachable API 29/30/31+ path, share platform capability gates from UI through runtime command advertisement/invocation, and require API 26-28 proof before lowering minSdk.

## Solution Assessment

No as-is. Lowering the support floor can be the right direction, but the maintainable solution needs a current-main rebase, a full API 26-30 audit, runtime capability gating, and real older-device proof before merge.
