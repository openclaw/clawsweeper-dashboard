## Summary

Keep open: current main and v2026.6.9 still preserve normal `chat_type: "group"` root-id precedence, so the reported `om_*` versus `omt_*` Feishu topic-message-format session split remains unresolved; overlapping open PRs exist, but none is merged as the canonical fix.

## What This Changes

The issue asks OpenClaw to keep Feishu `groupSessionScope: "group_topic"` sessions stable when reply-context events in topic-message-format groups expose an `om_*` reply root while the topic id is `omt_*`.

## Best Possible Solution

Land one Feishu-plugin fix that reliably identifies topic-message-format normal groups or provides a maintainer-approved session migration, preserves existing reply-created normal-group continuity, and includes focused tests plus redacted live Feishu proof.

## Reproduction Assessment

Yes, at source level: current main can still choose `rootId` before `threadId` for normal `chat_type: "group"` topic-scoped events, and it does not parse the reporter's `topic_id` metadata. I did not run a live Feishu tenant repro in this read-only review.

## Solution Assessment

No, current main is not a complete solution for this reported shape. The best fix is Feishu-plugin-owned routing or hydration normalization with maintainer review for session-key compatibility and real Feishu topic-message-format proof.
