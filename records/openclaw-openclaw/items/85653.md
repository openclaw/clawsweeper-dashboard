## Summary

Keep open: current main and the latest release still leave the reported Feishu normal-group topic-message-format path able to choose an `om_*` root id instead of the `omt_*` topic id, and the overlapping PRs have not merged as a safe canonical fix.

## What This Changes

The issue asks for Feishu `group_topic` sessions to stay stable when reply-context messages expose an `om_*` reply/root id while the actual topic is identified by `omt_*`.

## Best Possible Solution

Land one Feishu-plugin fix that reliably identifies topic-message-format normal groups or provides a maintainer-approved session migration, preserves existing reply-created normal-group continuity, and includes focused tests plus redacted live Feishu proof.

## Reproduction Assessment

## Solution Assessment
