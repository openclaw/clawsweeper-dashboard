## What This Changes

The issue reports that Feishu/Lark `group_topic` sessions still split when normal group topic replies carry reply-context `om_*` ids instead of the canonical `omt_*` topic id.

## Best Possible Solution

Land one Feishu-plugin fix that reliably identifies topic-message-format normal groups or provides a maintainer-approved session migration, preserves existing reply-created normal-group continuity, and includes focused tests plus redacted live Feishu proof.
