## Summary

Keep open. Current main and latest shipped Feishu code support adjacent session scoping, DM-only dynamic agent creation, and explicit thread-bound subagent hooks, but they still do not implement automatic Feishu group per-sender or per-role subagents with isolated skills, config, memory, lifecycle, quotas, and admin controls; the KinthAI comments are useful design input rather than an OpenClaw implementation.

## What This Changes

Request Feishu group chat support for dynamically creating and routing per-sender or per-role subagents with isolated memory, skills, configuration, lifecycle, and admin controls.

## Best Possible Solution

Design an opt-in Feishu group subagent mode that maps verified sender or role identity to bounded agent profiles with explicit private/shared memory namespaces, lifecycle cleanup, quotas, admin authorization, focused regression tests, docs, and redacted live Feishu group proof.

Source and shipped-release inspection give high confidence that OpenClaw lacks automatic Feishu group per-sender or per-role subagent provisioning despite adjacent session-scoping, DM dynamic-agent support, and explicit thread-bound subagent hooks.

## Solution Assessment

Unclear. Per-sender subagents are plausible, but the maintainable solution needs an opt-in Feishu identity, memory, lifecycle, quota, and admin-authority design rather than a narrow routing patch.
