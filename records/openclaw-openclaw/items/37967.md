## Summary

Keep open. Current main has adjacent Feishu sender/topic-scoped group sessions, DM-only dynamic agent creation, and thread-bound subagent hooks, but not automatic per-sender or per-role Feishu group subagents with isolated skills, config, memory, lifecycle, quotas, and admin controls; the KinthAI comments add useful prior art for the design rather than making the issue obsolete.

## What This Changes

Request dynamic Feishu group-chat per-sender or per-role subagents with isolated workspace, skills, configuration, memory, routing, and admin controls.

## Best Possible Solution

Design an opt-in Feishu plugin mode that maps sender or role identity to bounded isolated agent profiles with explicit memory namespaces, lifecycle cleanup, quotas, admin authorization, and regression plus live Feishu proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection shows adjacent Feishu group session routing, DM dynamic agents, and explicit thread subagent binding, but no automatic group-sender or role subagent provisioning to reproduce as broken existing behavior.

## Solution Assessment

Unclear. Per-sender subagents may be a sound direction, but the maintainable solution needs an opt-in design around Feishu identity, memory boundaries, lifecycle, quotas, and admin authority rather than a routing-only patch.
