## Summary

Keep open. Current main has Feishu group session scoping, DM-only dynamic agent creation, and manual thread-bound subagent hooks, but it still lacks the requested automatic Feishu group per-sender or per-role subagents with isolated skills, config, memory, lifecycle, quotas, and admin controls.

## What This Changes

Request automatic Feishu group-chat routing to user- or role-specific subagents with isolated skills, configuration, memory, and reply delivery.

## Best Possible Solution

Design an opt-in Feishu group subagent mode in the Feishu plugin that maps verified sender or role identity to bounded agent profiles with explicit memory namespaces, lifecycle cleanup, quotas, admin authorization, tests, docs, and redacted live Feishu group proof.

Unclear. Per-sender or per-role subagents are plausible, but the maintainable solution needs an opt-in Feishu identity, memory, lifecycle, quota, and admin-authority design rather than a narrow routing patch.
