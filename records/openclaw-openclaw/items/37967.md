## Summary

Keep open. Current main and v2026.5.22 have adjacent Feishu group session scoping, DM-only dynamic agent creation, and thread-bound subagent hooks, but they still do not provide the requested automatic Feishu group per-sender or per-role subagents with isolated skills, config, memory, lifecycle, quotas, routing, and admin controls.

## What This Changes

The issue requests automatic Feishu group-chat subagents per sender or role with isolated skills, configuration, memory, routing, lifecycle, and admin controls.

## Best Possible Solution

Design an opt-in Feishu plugin mode that maps verified sender or role identity to bounded agent profiles with explicit memory namespaces, lifecycle cleanup, quotas, admin authorization, tests, docs, and redacted live Feishu group proof.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main lacks automatic Feishu group per-sender or per-role subagent provisioning despite adjacent session-scoping, DM dynamic-agent, and manual thread-bound subagent support.

## Solution Assessment

Unclear. Per-sender or per-role subagents are plausible, but the maintainable solution needs an opt-in Feishu identity, memory, lifecycle, quota, and admin-authority design rather than a narrow routing patch.
