## Summary

Keep open. Current main has Feishu sender/topic-scoped group sessions, DM/comment dynamic-agent creation, and explicit Feishu subagent/ACP conversation binding, but it still does not automatically provision each group sender or role as an isolated subagent with separate skills, config, memory, lifecycle, quotas, and admin controls; the KinthAI comments add useful design lessons, not a merged implementation.

## What This Changes

Request Feishu group chat per-sender or role-specific subagents with isolated skills, configuration, memory, lifecycle, and admin controls.

## Best Possible Solution

Design an opt-in Feishu plugin mode that maps sender or role identity to bounded isolated agent profiles with explicit memory namespaces, cleanup/quotas, admin controls, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection shows current main routes Feishu groups by sender/topic and supports DM dynamic agents plus explicit subagent thread binding, but not automatic group-sender or role subagents.

## Solution Assessment

Unclear. Per-sender subagents may be a reasonable direction, but the maintainable solution should be opt-in and designed around Feishu identity, memory boundaries, lifecycle, quotas, and admin authority rather than a routing-only patch.
