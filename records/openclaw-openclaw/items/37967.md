## Summary

Keep open. Current main has Feishu sender/topic-scoped group sessions, DM-only dynamic agent creation, and explicit Feishu thread subagent binding, but it still does not automatically provision each group sender or role as an isolated subagent with separate skills, config, memory, lifecycle, quotas, and admin controls; the KinthAI comments are useful design input rather than a merged implementation.

## What This Changes

The issue requests automatic Feishu group-chat subagents per sender or role with isolated memory, skills, config, lifecycle, and admin controls.

## Best Possible Solution

Design an opt-in Feishu plugin mode that maps sender or role identity to bounded isolated agent profiles with explicit memory namespaces, cleanup/quotas, admin controls, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection shows adjacent Feishu group session routing, DM dynamic agents, and explicit thread subagent binding, but no automatic group-sender or role subagent provisioning to reproduce as broken existing behavior.

## Solution Assessment

Unclear. Per-sender subagents may be a reasonable direction, but the maintainable solution should be opt-in and designed around Feishu identity, memory boundaries, lifecycle, quotas, and admin authority rather than a routing-only patch.
