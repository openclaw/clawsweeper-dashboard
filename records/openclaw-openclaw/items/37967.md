## Summary

Keep open. Current main has Feishu group session scoping, DM-only dynamic agent creation, and explicit thread-bound subagent hooks, but it still does not implement automatic Feishu group per-sender or per-role subagents with isolated skills, config, memory, lifecycle, quotas, and admin controls.

## What This Changes

Request automatic Feishu group-chat routing that creates or reuses user- or role-specific subagents with isolated skills, configuration, memory, and replies relayed back to the group.

## Best Possible Solution

Design an opt-in Feishu group subagent mode that maps verified sender or role identity to bounded agent profiles with explicit memory namespaces, lifecycle cleanup, quotas, admin authorization, regression tests, docs, and redacted live Feishu group proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks automatic Feishu group per-sender or per-role subagent provisioning despite adjacent session-scoping and DM dynamic-agent support.

## Solution Assessment

Unclear. Per-sender subagents are plausible, but the maintainable solution needs an opt-in Feishu identity, memory, lifecycle, quota, and admin-authority design rather than a narrow routing patch.
