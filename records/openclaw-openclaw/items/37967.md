## Summary

Keep open. Current main has Feishu sender/topic-scoped group sessions, DM-only dynamic agent creation, and thread-bound subagent hooks, but not automatic per-sender or per-role Feishu group subagents with isolated skills, config, memory, lifecycle, quotas, and admin controls; the KinthAI comments add useful design prior art rather than making the request obsolete.

## What This Changes

Request automatic Feishu group-chat routing to per-user or per-role subagents with isolated skills, configuration, memory, lifecycle, and admin controls.

## Best Possible Solution

Design an opt-in Feishu plugin mode that maps sender or role identity to bounded isolated agent profiles with explicit memory namespaces, lifecycle cleanup, quotas, admin authorization, and regression plus live Feishu proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main has adjacent group session scoping, DM dynamic agents, and thread-bound subagent hooks, but no automatic group-sender or role subagent provisioning to reproduce as broken existing behavior.

## Solution Assessment

Unclear. Per-sender subagents may be a sound direction, but the maintainable solution needs an opt-in design around Feishu identity, memory boundaries, lifecycle, quotas, and admin authority rather than a routing-only patch.
