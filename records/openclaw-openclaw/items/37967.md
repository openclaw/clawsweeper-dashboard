## Summary

Keep open: current main and v2026.6.6 cover adjacent Feishu sender-scoped sessions, DM dynamic agents, and explicit thread-bound subagent delivery, but they do not implement automatic persistent group-sender agents with isolated workspaces, skills, config, and memory. The remaining work needs maintainer product and security direction, especially because the KinthAI comments add useful real-world lifecycle, routing, and memory-isolation constraints.

## What This Changes

Request automatic Feishu group-chat routing that provisions and targets one isolated subagent per sender, with per-user skills, config, memory, lifecycle, and reply delivery.

## Best Possible Solution

Design a generic, administrator-authorized identity-to-agent binding contract first, then implement optional Feishu group-sender provisioning with lifecycle quotas, SQLite-backed state, memory isolation, deterministic routing, and security/audit coverage.

## Reproduction Assessment

Yes, at source level. Current main shows sender-scoped Feishu group sessions and explicit subagent conversation binding, while automatic persistent group-sender agent provisioning is absent; this is a missing capability rather than a broken existing contract.

## Solution Assessment

No as proposed. A Feishu-only implicit subagent factory is too broad for the first step; the safer path is a generic authorized identity-to-agent binding and lifecycle contract, then an optional Feishu implementation.
