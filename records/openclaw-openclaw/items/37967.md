## Summary

Keep open. Current main has Feishu sender/topic-scoped group sessions, direct-message dynamic agent creation, and explicit thread-bound subagent binding, but it still does not automatically provision each group sender or role as an isolated subagent with separate skills, config, memory, lifecycle, quotas, and admin controls.

## What This Changes

The issue requests Feishu group chat routing that automatically creates per-user or per-role subagents with isolated skills, config, memory, lifecycle, and admin controls.

## Best Possible Solution

Design an opt-in Feishu plugin mode that maps sender or role identity to bounded isolated agent profiles with explicit memory namespaces, cleanup/quotas, admin controls, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection shows current main has sender/topic group session routing, DM dynamic-agent creation, and explicit thread-bound subagent binding, but not automatic per-user or per-role group subagents.

## Solution Assessment

Unclear. Per-sender subagents may be a reasonable direction, but the maintainable solution should be opt-in and designed around Feishu identity, memory boundaries, lifecycle, quotas, admin authority, and tool/config permissions rather than a routing-only patch.
