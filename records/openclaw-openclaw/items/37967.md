## Summary

Keep open. Current main and v2026.5.22 still provide Feishu group session scoping, DM-only dynamic agent creation, and manual thread-bound subagent hooks, but not automatic Feishu group per-sender or per-role subagents with isolated skills, config, memory, lifecycle, quotas, and admin controls; the KinthAI comments are useful prior art, not an OpenClaw implementation.

## What This Changes

The issue requests Feishu group-chat user or role-specific subagents with isolated memory, skills, configuration, lifecycle, quotas, and admin controls.

## Best Possible Solution

Design an opt-in Feishu plugin mode that maps verified sender or role identity to bounded agent profiles with explicit memory namespaces, lifecycle cleanup, quotas, admin authorization, tests, docs, and redacted live Feishu group proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks automatic Feishu group per-sender or per-role subagent provisioning despite adjacent session-scoping, DM dynamic-agent, and manual thread-bound subagent support.

## Solution Assessment

Unclear. Per-sender or per-role subagents are plausible, but the maintainable solution needs an opt-in Feishu identity, memory, lifecycle, quota, and admin-authority design rather than a narrow routing patch.
