## Summary

Keep open. Current main has Feishu sender/topic-scoped group sessions, DM-only dynamic-agent creation, and explicit Feishu ACP/thread subagent binding, but it still does not automatically provision each group sender or role as an isolated subagent with separate skills, config, memory, lifecycle, quotas, and admin controls; the KinthAI comments are useful prior-art/design input, not a merged implementation.

## What This Changes

The issue requests automatic per-sender or per-role Feishu group-chat subagents with isolated skills, configuration, memory, lifecycle, and admin controls.

## Best Possible Solution

Design an opt-in Feishu plugin mode that maps sender or role identity to bounded isolated agent profiles with explicit memory namespaces, cleanup/quotas, admin controls, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main shows Feishu currently provides sender/topic session routing, DM dynamic agents, and explicit thread binding, but not automatic group-sender or role subagents.

## Solution Assessment

Unclear. Per-sender subagents may be a reasonable direction, but the maintainable solution should be opt-in and designed around Feishu identity, memory boundaries, lifecycle, quotas, and admin authority rather than a routing-only patch.
