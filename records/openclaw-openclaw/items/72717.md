## Summary

Keep open: current main still has digest-assisted Markdown scanning for `wiki_search`, not a SQLite FTS-backed wiki index. The follow-up comment correctly calls out stale-index detection and rebuild behavior as the main design risk, and #75312 is adjacent shared-memory runtime breakage rather than a duplicate of this performance/indexing request.

## What This Changes

The issue requests a SQLite FTS5-backed index for `memory-wiki` search that updates on wiki writes and falls back to file scanning when missing or stale.

## Best Possible Solution

Implement a read-through FTS cache inside the bundled `memory-wiki` plugin with Markdown as source of truth, stale detection, a rebuild path, and file-scan fallback; keep the broader durable decisions/exchange database idea separate.

## Reproduction Assessment

Yes. Source inspection on current main shows `wiki_search` still reads Markdown after an optional compiled-digest prefilter and has no SQLite FTS path; a performance benchmark would be needed only to quantify the scale impact.

## Solution Assessment

Yes, if scoped narrowly. A local SQLite FTS cache is the maintainable fix for the search-performance gap, while the companion DB's durable decision/exchange schema would be a larger product decision.

## Repair Work Prompt

Implement #72717 as a narrow `memory-wiki` plugin enhancement. Add a SQLite FTS5-backed cache for wiki pages that `wiki_search` queries before falling back to the existing Markdown scan. Markdown remains the source of truth. Update the cache from `wiki compile` and `wiki_apply`, add stale detection for manual edits using mtime or content hashes, and provide a rebuild path rather than silently trusting stale rankings. If SQLite/node:sqlite/FTS is unavailable, missing, corrupt, or stale, fall back to current file-backed search without failing. Do not implement the companion durable decisions/exchange database schema in this PR, and do not treat #75312's shared-memory `.search` runtime bug as solved unless a directly required defensive seam is touched. Update focused tests, docs, and CHANGELOG for the user-facing search behavior.
