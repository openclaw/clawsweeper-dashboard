## Summary

Keep open: current main still lacks a SQLite FTS-backed wiki index; `wiki_search` remains digest/file-backed, and `wiki compile`/`wiki_apply` refresh Markdown and digest artifacts only. The drift concern in the discussion is real acceptance criteria, not a reason to close.

## What This Changes

Add a SQLite FTS5 cache for memory-wiki `wiki_search`, refreshed by compile/apply flows with stale-index fallback to file scanning.

## Best Possible Solution

Implement a memory-wiki-owned read-through FTS cache with Markdown as source of truth, stale-index detection/rebuild, file-scan fallback, focused tests, docs, and a changelog entry.

## Reproduction Assessment

Yes. Source inspection of current main shows `searchMemoryWiki` calls `searchWikiCorpus`, which reads digest candidates and Markdown files with no SQLite/FTS path; a large-vault benchmark would quantify latency but is not needed to reproduce the missing capability.

## Solution Assessment

Yes, if scoped narrowly. A plugin-local FTS cache over existing wiki pages is the maintainable fix; making SQLite the durable source of truth would be a broader product change.

## Repair Work Prompt

Implement this issue as a narrow memory-wiki plugin enhancement. Add a SQLite FTS5-backed read-through cache for wiki pages used by `wiki_search`, with Markdown remaining the source of truth. Refresh or rebuild the index from `wiki compile` and after `wiki_apply`; add stale/missing detection using mtime/hash or equivalent metadata, and fall back to the existing file-scan path whenever SQLite/FTS is unavailable, the index is stale, or rebuild fails. Keep the companion durable decisions/exchange schema out of scope. Respect extension boundaries: do not import core `src/**` from plugin code; use documented plugin SDK/package exports or add a generic documented seam only if truly necessary. Update CLI/gateway behavior only as needed for compile/rebuild/status, update docs and CHANGELOG for the user-facing feature, and add focused regression coverage for indexed search, stale index fallback/rebuild, apply refresh, compile rebuild, and missing SQLite fallback. Related adjacent issue #75312 is not the same bug and should not be folded into this PR unless the touched path needs a tiny defensive guard.
