## What This Changes

Request to add a SQLite FTS-backed index for memory-wiki `wiki_search`, refreshed by compile/apply, to avoid scanning Markdown and improve durable wiki decision retrieval.

## Best Possible Solution

Design a memory-wiki-owned SQLite-backed search/cache layer with explicit source-of-truth, freshness validation, rebuild/doctor behavior, and then implement it through compile/apply/search with focused upgrade and regression tests.

Unclear. SQLite FTS is a plausible direction, but the best fix needs maintainer decisions on cache freshness, rebuild/doctor policy, and whether preserved decision exchanges belong in wiki Markdown, plugin state, or a broader memory model.
