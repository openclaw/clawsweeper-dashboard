## Summary

Keep open: current main still lacks a SQLite FTS-backed wiki index; `wiki_search` uses compiled digest candidate selection plus Markdown file reads, and `wiki_apply`/`wiki compile` only refresh Markdown/digest artifacts. The linked #75312 report is adjacent shared-memory runtime breakage, not a duplicate of this indexing/performance request.

## What This Changes

Request adding a SQLite FTS5 cache for memory-wiki `wiki_search`, updated by compile/apply paths and falling back to Markdown scans when stale or missing.

## Best Possible Solution

Implement the cache inside the bundled `memory-wiki` plugin with Markdown as source of truth, stale-index detection, an explicit rebuild path, and file-scan fallback whenever SQLite is unavailable or the index is missing/stale.

## Reproduction Assessment

Yes for the missing implementation: source inspection on current main shows `wiki_search` still reads Markdown after an optional compiled-digest prefilter and has no SQLite/FTS path. The exact performance impact needs a benchmark vault with roughly 50 or more pages.

## Solution Assessment

Yes if scoped narrowly. A local read-through FTS cache is the maintainable fix for this search-performance gap; the companion DB's durable decision/exchange schema should remain a separate product discussion.

## Repair Work Prompt

Implement a narrow SQLite FTS-backed cache for the bundled `memory-wiki` plugin to satisfy https://github.com/openclaw/openclaw/issues/72717. Keep Markdown pages as the source of truth. Add a plugin-local SQLite/FTS implementation for `wiki_search` that stores page path/title/body plus enough mtime/hash metadata to detect stale rows, queries FTS first for ranked wiki results, and falls back to the existing Markdown scan when SQLite is unavailable, the index is missing, or any row is stale. Refresh or rebuild the index from `wiki compile`, `wiki_apply`, imports/ingest paths that call compile, and expose an explicit rebuild/repair path if needed. Do not implement the issue body's broader companion DB durable decisions/exchange schema in this PR, do not conflate this with #75312, and do not import core `src/**` internals from plugin production code. Update memory-wiki docs, CLI docs if a command/flag is added, and CHANGELOG.md for the user-facing behavior.
