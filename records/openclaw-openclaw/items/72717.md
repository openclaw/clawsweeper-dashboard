## Summary

Keep open: current main still lacks a SQLite FTS-backed wiki index, and the remaining work is a valid, bounded memory-wiki feature rather than an implemented, duplicate, or ClawHub-only request. The linked shared-memory runtime bug is related context, but it does not cover the same indexed wiki-search capability.

## What This Changes

The issue requests a SQLite FTS-backed cache for `wiki_search`, refreshed by wiki compile/apply operations with fallback to Markdown file scanning when missing or stale.

## Best Possible Solution

Add a memory-wiki-owned read-through FTS cache with hash or mtime drift detection, rebuild through compile/apply, file-scan fallback, tests, docs, and a changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows `wiki_search` is still digest/file-backed and no memory-wiki SQLite/FTS implementation is present; a large-vault benchmark would quantify latency but is not needed to reproduce the missing capability.

## Solution Assessment

Yes, if scoped narrowly. A plugin-local FTS cache over existing wiki pages is the maintainable path; making SQLite the durable wiki source of truth would be a broader product decision.

## Repair Work Prompt

Implement the feature requested in https://github.com/openclaw/openclaw/issues/72717 as a narrow memory-wiki plugin change. Add an internal SQLite FTS5 cache for wiki pages that keeps Markdown as the source of truth, updates or rebuilds during `wiki compile` and `wiki_apply`, checks page hash/mtime on read to detect stale or missing index rows, and falls back to the existing file/digest search when SQLite/FTS is unavailable or stale. Use existing plugin-safe SDK surfaces such as `openclaw/plugin-sdk/memory-core-host-engine-storage` where appropriate; do not move policy into core or change the durable source-of-truth model. Include focused regression tests for indexed search, stale/missing-index fallback, apply/compile refresh behavior, and manual-edit drift. Update `docs/plugins/memory-wiki.md`, `docs/cli/wiki.md` if CLI behavior changes, and `CHANGELOG.md` because this is user-facing. Keep #75312 separate unless the implementation naturally touches the shared-memory fallback path.
