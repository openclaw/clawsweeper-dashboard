## Summary

Keep open: current main and the latest release still lack a wake-inbox drain/replay path, and the requested durability layer depends on the still-open live user-turn injection API and security contract.

## What This Changes

The issue requests a durable wake-inbox drain that replays unconsumed wake records into matching live sessions on activation and marks them consumed only after stdin write succeeds.

## Best Possible Solution

First settle the supported opt-in live user-turn injection contract in https://github.com/openclaw/openclaw/issues/83809; then add wake-inbox replay with ordering, session matching, dedupe, consume-after-write semantics, partial-failure retry behavior, security review, and focused tests.

## Reproduction Assessment

No as a strict bug repro; this is a new durability/API feature request. Source inspection does confirm that current main has no wake-inbox drain and no supported live-session injection API for such a drain to use.

## Solution Assessment

Unclear: the replay rules are plausible, but the best solution depends on the maintainer-approved injection contract, opt-in behavior, and security model. A narrow drain should come after that contract, not before it.
