## Summary

## What This Changes

The issue requests draining unconsumed durable wake-inbox records into a matching live session when that session activates, resumes, or begins a turn.

## Best Possible Solution

First settle a supported, opt-in live user-turn injection contract; then add wake-inbox drain/replay with ordered matching, dedupe, consume-after-stdin-write, partial-failure retry behavior, docs, and focused tests.
