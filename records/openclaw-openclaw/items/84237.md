## Summary

## What This Changes

The issue requests a durability layer that drains unconsumed wake-inbox records into a matching live Claude session when that session activates, resumes, or begins a turn.

## Best Possible Solution

First settle a supported, authorized, opt-in live user-turn injection contract; then implement wake-inbox drain/replay with ordering, session matching, dedupe, consume-after-stdin-write, partial-failure behavior, audit events, and focused tests.

## Reproduction Assessment

No; this is a feature/API durability request rather than a broken current contract. Source inspection does confirm current main lacks both a wake-inbox drain and the supported live injection API that the drain would need.

## Solution Assessment

Unclear as requested because the replay rules are plausible, but the best implementation depends on the maintainer-approved injection/security contract. The safer direction is an explicit opt-in runtime contract before adding file-backed replay into live session stdin.
