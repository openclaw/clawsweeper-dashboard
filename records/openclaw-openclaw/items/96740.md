## What This Changes

The PR adds `attach.adopt` and `resumeSessionId`, stores a cwd-scoped Claude CLI binding, and changes `openclaw attach` to start or resume Claude through that binding.

## Best Possible Solution

Persist Claude CLI bindings only after a real session exists, prove `chat.history` import plus resume/cwd behavior, then land after maintainer/security acceptance and stack order are settled.

## Reproduction Assessment

Yes for the PR defect from source inspection: `attach.adopt` runs before `--print-config` returns and before child spawn can prove a Claude session exists. The intended feature has partial live-output proof, but not direct `chat.history` import proof.

## Solution Assessment

No: the owner boundary is plausible, but the current implementation should not persist a resume binding until there is a real Claude session to resume. A safer shape is to adopt only after confirmed session creation or to avoid setup-only adoption until Claude actually runs.
