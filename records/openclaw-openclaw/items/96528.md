## Summary

## What This Changes

## Best Possible Solution

Preserve explicit config/env/caller `maxHoldMs` values while changing only the default fallback, or land a maintainer-approved hard-ceiling policy with docs/help, tests, upgrade proof, and real contention proof.

## Reproduction Assessment

Yes, at source level: current main resolves the default acquire timeout to 60000 ms and `maxHoldMs` to 300000 ms, so a legal holder can outlive waiters. I did not run a live contending gateway reproduction in this read-only review.

## Solution Assessment

No. The acquire layer is a plausible fix location, but the branch turns acquire timeout into an unconditional ceiling for explicit `maxHoldMs`; the safer fix preserves configured high values unless maintainers approve the stricter contract.
