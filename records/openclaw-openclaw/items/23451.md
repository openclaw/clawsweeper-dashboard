## Summary

Keep open. Current main has exec approvals and plugin-driven `before_tool_call.requireApproval`, but it still lacks the requested built-in `tools.confirmation` config, risk-tier classifier, and same-response assistant-text-aware gate before the first tool executes.

## What This Changes

The issue requests a configurable core confirmation gate that can pause tool execution based on tool risk and same-response assistant text before any tool runs.

## Best Possible Solution

Design a backward-compatible core `tools.confirmation` policy that composes with plugin and exec approvals, flushes visible assistant text before holding tools, and defines deterministic approve, deny, timeout, and transcript behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high-confidence evidence that the requested config/schema surface and response-level confirmation gate are absent.

## Solution Assessment

Unclear until maintainers settle the safety and product contract. The requested core gate is plausible, but the maintainable implementation should compose with plugin `requireApproval` and exec approvals rather than replacing those primitives.
