## Summary

Keep open. Current main has plugin-driven `before_tool_call.requireApproval` and exec approval primitives, but it still lacks the requested built-in `tools.confirmation` config, risk-tier classifier, and same-response assistant-text-aware gate before the first tool executes.

## What This Changes

Request a configurable core tool confirmation gate that can pause tool execution based on tool risk and same-response assistant confirmation text.

## Best Possible Solution

Add a backward-compatible core executor gate with documented config, deterministic classification, user-visible flush before approval, and fail-closed transcript semantics.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence evidence that the requested config/schema surface and response-level confirmation gate are absent on current main.

## Solution Assessment

Unclear until maintainers settle the product and safety contract. A core gate is plausible, but it should compose with plugin `requireApproval`, exec approvals, and channel approval routing rather than replacing those primitives.
