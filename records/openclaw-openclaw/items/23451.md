## Summary

Keep open. Current main has exec approvals and plugin-driven `before_tool_call.requireApproval`, but it still lacks the requested built-in `tools.confirmation` config/schema surface, deterministic risk-tier classifier, and same-response assistant-text-aware gate before the first tool executes.

## What This Changes

Request a configurable core tool confirmation gate that can pause risky tool calls for human approval before execution.

## Best Possible Solution

Design a backward-compatible core `tools.confirmation` surface that composes with plugin and exec approvals, flushes user-visible assistant text before holding tools, and defines deterministic approve, deny, timeout, and transcript behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence evidence that the requested config/schema surface and response-level confirmation gate are absent on current main.

## Solution Assessment

Unclear until maintainers settle the safety and product contract. A core executor gate is the right boundary, but the exact config shape, risk classifier, and same-response text heuristic need design review.
