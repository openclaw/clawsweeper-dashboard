## Summary

Keep open. Current main has plugin-controlled `before_tool_call.requireApproval` and exec approvals, but it still lacks the requested built-in `tools.confirmation` config surface, risk-tier classifier, and same-response assistant-text-aware gate before the first tool executes.

## What This Changes

Request a configurable tool-level confirmation gate that can pause risky tool execution for human approval based on risk level and assistant text.

## Best Possible Solution

Design a backward-compatible core `tools.confirmation` surface that composes with plugin approvals and exec approvals, flushes user-visible assistant text first, and defines deterministic approve, deny, timeout, and transcript behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence evidence that current main lacks the requested config/schema surface and response-level assistant-text-aware gate.

## Solution Assessment

Unclear until maintainers settle the safety and product contract. The requested core gate is plausible, but the maintainable implementation should compose with plugin `requireApproval` and exec approvals rather than replacing those primitives.
