## Summary

Keep open. Current main has exec approvals and plugin-controlled `before_tool_call.requireApproval`, including the merged work from #55339, but it still lacks the requested built-in configurable tool-level confirmation gate that can classify tool risk and inspect same-response assistant text before the first tool executes.

## What This Changes

Request a configurable core tool confirmation gate that can pause model-issued tool calls by risk level and same-response confirmation text before execution.

## Best Possible Solution

Design a backward-compatible core `tools.confirmation` surface that composes with plugin `before_tool_call.requireApproval`, classifies tools deterministically, flushes assistant text before holding execution, and defines approve/deny/timeout transcript behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence evidence that current main lacks the requested config/schema surface and response-level confirmation gate.

## Solution Assessment

Unclear until maintainers settle the product and safety policy. The direction is plausible, but the maintainable solution should be a core gate that composes with existing plugin approvals rather than replacing the lower-level hook primitive.
