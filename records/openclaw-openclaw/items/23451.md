## Summary

Keep open. Current main still has host exec approvals and plugin-controlled `before_tool_call.requireApproval`, but it lacks the requested built-in `tools.confirmation` risk-tier config and same-response assistant-text gate before the first tool executes.

## What This Changes

Request a configurable tool confirmation gate that can pause or skip tool execution based on tool risk tier and same-response assistant confirmation text.

## Best Possible Solution

Design a backward-compatible core `tools.confirmation` config/schema/docs surface that classifies tool risk, flushes assistant text before holding execution, and defines approve, deny, timeout, and transcript behavior while preserving plugin `before_tool_call.requireApproval` as the lower-level custom policy hook.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence evidence that current main lacks the requested config/schema surface and response-level confirmation gate.

## Solution Assessment

Unclear until maintainers settle the product and safety policy. The proposed direction is plausible, but the maintainable implementation should be a core gate that composes with existing plugin and exec approvals rather than replacing those primitives.
