## Summary

Keep open. Current main has exec approvals and plugin-controlled `before_tool_call.requireApproval`, but it still lacks a built-in `tools.confirmation` config surface, risk-tier classifier, and same-response assistant-text-aware gate before the first tool executes.

## What This Changes

Add a configurable tool confirmation layer that can pause tool execution based on risk and assistant text before same-response tool calls run.

## Best Possible Solution

Design a backward-compatible core `tools.confirmation` surface that composes with plugin approvals and exec approvals, flushes user-visible assistant text first, and defines deterministic approve, deny, timeout, and transcript behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence evidence that the requested config/schema surface and response-level confirmation gate are absent on current main.

## Solution Assessment

Unclear until maintainers settle the safety and product contract. The requested core gate is plausible, but the maintainable implementation should compose with plugin `requireApproval` and exec approvals rather than replacing those primitives.
