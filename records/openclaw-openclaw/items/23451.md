## Summary

Keep open: current main and the latest release have exec approvals and plugin `before_tool_call.requireApproval`, but they still lack the requested built-in `tools.confirmation` config, risk-tier classifier, and same-response assistant-text gate before the first tool executes. The discussion correctly distinguishes text-level consent from executor-enforced consent, so this needs maintainer product and security review rather than cleanup closure.

## What This Changes

Add a configurable runtime confirmation gate that can pause risky tool calls before execution when model text asks for confirmation in the same response.

## Best Possible Solution

Design a backward-compatible core `tools.confirmation` contract with deterministic risk tiers, same-response assistant-text awareness, visible text flush before holding tools, explicit approve/deny/timeout semantics, and regression coverage across agent, config, plugin, and channel approval paths.

## Reproduction Assessment

Not applicable as a feature/design request. Source and release inspection give a high-confidence state check that the requested config/schema surface and response-level gate are absent on current main and v2026.5.12.

## Solution Assessment

Yes, keeping this open for core design is the best current path. A plugin-only or exec-only solution would not cover the requested built-in gate across risky tools, same-response assistant text, and chat channels.
