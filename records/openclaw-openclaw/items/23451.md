## Summary

Keep open: current main still has useful plugin and exec approval primitives, but not the requested built-in, configurable, response-level confirmation gate that can inspect same-response assistant text before the first risky tool executes.

## What This Changes

Requested a configurable tool-level confirmation gate that pauses risky tool calls for user approval before execution.

## Best Possible Solution

Design a core, backward-compatible `tools.confirmation` contract with deterministic risk tiers, same-response text awareness, user-visible text flush, explicit approve/deny/timeout semantics, and regression coverage across agent, config, plugin, and channel approval paths.

## Reproduction Assessment

Not applicable as a feature/design request. Current-main source inspection gives a high-confidence state check that the requested config/schema surface and response-level gate are absent.

## Solution Assessment

Yes, keeping this open for core design is the best current path. A plugin-only or narrow exec-only fix would not address the requested runtime-enforced gate across all risky tools and chat channels.
