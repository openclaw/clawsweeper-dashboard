## Summary

Keep open: current main and the latest release have host exec approvals and plugin `before_tool_call.requireApproval`, but they still do not provide the requested built-in `tools.confirmation` config, risk-tier classifier, or same-response assistant-text gate before the first tool executes.

## What This Changes

Requests a configurable tool confirmation gate that can pause risky tool calls for human approval based on risk level and same-response assistant text.

## Best Possible Solution

Design a backward-compatible core `tools.confirmation` contract with deterministic risk tiers, same-response assistant-text awareness, visible text flush before holding tools, explicit approve/deny/timeout semantics, and regression coverage across agent, config, plugin, and channel approval paths.

## Reproduction Assessment

Not applicable as a feature/design request. Current-main source inspection gives a high-confidence state check that the requested config/schema surface and response-level gate are absent.

## Solution Assessment

Yes, keeping this open for core design is the best current path. A plugin-only or exec-only solution would not cover the requested built-in gate across risky tools and chat channels.
