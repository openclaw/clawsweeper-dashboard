## Summary

Keep open: current main and the latest release still have plugin/exec approval primitives, but not the requested built-in `tools.confirmation` config, risk-tier classifier, or same-response assistant-text gate before the first tool executes. The issue is correctly scoped as a core tool-execution consent design that needs product and security review.

## What This Changes

Request a configurable, risk-tiered core tool confirmation gate that can pause same-response tool calls for user approval before execution.

## Best Possible Solution

Design a backward-compatible core `tools.confirmation` contract with deterministic risk tiers, same-response assistant-text awareness, visible text flush before holding tools, explicit approve/deny/timeout semantics, and regression coverage across agent, config, plugin, and channel approval paths.

## Reproduction Assessment

Not applicable as a feature/design request. Source inspection gives a high-confidence state check that the requested config/schema surface and response-level gate are absent on current main and the latest release.

## Solution Assessment

Yes, keeping this open for core design is the best current path. A plugin-only, MCP-only, or exec-only solution would not cover the requested built-in gate across risky tools and chat channels.
