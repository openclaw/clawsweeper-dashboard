## Summary

Keep open: the wrapper-only fix is the right boundary for the memory-wiki `wiki_lint` tool, but the current PR still has a cross-platform separator bug and an explicit tool-details compatibility risk before merge.

## What This Changes

The PR changes the memory-wiki `wiki_lint` agent tool to emit a vault-relative report path and bounded details object, with a focused regression test.

## Best Possible Solution

Keep the wrapper-only fix, slash-normalize the returned tool `reportPath`, preserve lint core/CLI/gateway absolute-path behavior, and land with focused memory-wiki tests plus the supplied direct-tool proof.

## Reproduction Assessment

Yes for the merge-blocking review finding: direct Node probing shows `path.win32.relative` returns a backslash-separated path. For the linked runtime failure, current source and related issue proof confirm the raw absolute-path output shape, but the full bridge-mode failure was not reproduced here.

## Solution Assessment

Mostly yes: changing only `createWikiLintTool` is the narrow maintainable fix because lint core, CLI, and gateway callers keep their current file-oriented result. The implementation still needs slash normalization before it is merge-ready.
