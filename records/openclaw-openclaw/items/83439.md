## Summary

Keep open: the wrapper-only fix matches the linked memory-wiki bug and the supplied proof is useful, but the patch needs one cross-platform path normalization fix before merge.

## What This Changes

The PR changes the memory-wiki `wiki_lint` tool to return a vault-relative report path in tool text/details while leaving lint core and CLI behavior unchanged.

## Best Possible Solution

Keep the wrapper-only design, normalize the relative report path to slash separators before returning it, leave lint core/CLI/gateway absolute result contracts unchanged, and then land with focused memory-wiki tests.

## Reproduction Assessment

Yes for the review finding: source inspection is enough because `node:path.relative` is platform-specific and returns backslash separators on Windows, while the PR's expected tool output is `reports/lint.md`. I did not run the full bridge-mode failure path in this read-only review.

## Solution Assessment

Mostly yes: changing only `createWikiLintTool` is the narrow maintainable fix because lint core and CLI callers keep the absolute report path. The implementation should also normalize separators after `path.relative` so the tool contract is stable across platforms.
