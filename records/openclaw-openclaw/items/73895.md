## Summary

Keep open. Current main still lacks the requested routing/config/timing/token metadata on the Gateway chat, OpenAI-compatible, OpenResponses, and agent-ack surfaces, and exposing config lineage or decision traces needs Gateway owner-approved schema and redaction before implementation.

## What This Changes

The issue requests a promotion-grade Gateway v2 metadata envelope containing routing/config lineage, policy/version, timing, decision trace, and token usage fields for live benchmark smoke tests.

## Best Possible Solution

Define a typed, additive, redacted Gateway metadata envelope at the protocol and HTTP response boundaries, document versioning and omitted-field semantics, then wire it through the live benchmark-facing surfaces with compatibility tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is an instrumentation feature request. Source inspection gives a high-confidence verification path because the relevant schemas and response builders do not contain the requested fields.

## Solution Assessment

Unclear because there is no patch yet. Emitting from the routing layer is directionally maintainable, but the safe first step is owner approval of schema shape, redaction policy, target surfaces, versioning, and reasoning-token semantics.
