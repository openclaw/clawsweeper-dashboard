## Summary

Keep open: current main still lacks the requested Gateway v2 routing/config lineage, timing, and reasoning-token metadata, and this is a public Gateway schema expansion that needs owner design before implementation.

## What This Changes

Request Gateway v2 response metadata for routing/config lineage, latency, and token usage so benchmark smoke checks can run against live Gateway output.

## Best Possible Solution

Define an additive, typed, redacted, versioned Gateway metadata envelope at the protocol and HTTP response boundaries, then wire it through live response paths with docs and compatibility tests.

Not applicable as a runtime bug reproduction; this is an instrumentation feature request. We do have a high-confidence verification path by searching for the requested fields and inspecting the current response builders and schemas.

## Solution Assessment

Unclear because there is no patch yet. Emitting metadata from the routing layer is plausible, but the maintainable solution needs approved schema, redaction, versioning, and reasoning-token semantics first.
