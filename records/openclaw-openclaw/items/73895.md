## Summary

Keep open. Current main still does not emit the requested Gateway v2 routing/config lineage, policy, timing, decision-trace, gateway-version, or reasoning-token metadata on the inspected Gateway protocol, OpenAI-compatible Chat Completions, OpenResponses, or agent acknowledgement surfaces; exposing those fields also needs Gateway owner approval for schema, versioning, and redaction.

## What This Changes

Request Gateway v2 responses/events to emit a full routing/config lineage metadata envelope with policy, timing, decision trace, gateway version, and reasoning-token usage for benchmark smoke validation.

## Best Possible Solution

Define a typed, additive, redacted Gateway v2 metadata envelope at the protocol and HTTP response boundaries, document omitted-field semantics and versioning, then wire it through the benchmark-facing live surfaces with compatibility tests and provider-specific usage normalization.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is an instrumentation feature request. We do have high-confidence source verification because exact field search plus the relevant closed schemas and response builders omit the requested fields.

## Solution Assessment

Unclear because there is no patch yet. The suggested routing-layer envelope is directionally maintainable, but the safest next step is maintainer approval of schema shape, redaction policy, target surfaces, versioning, and reasoning-token semantics.
