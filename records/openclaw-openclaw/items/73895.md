## Summary

Keep open. Current main still exposes only narrow chat/OpenAI/OpenResponses/agent acknowledgement metadata, and the requested promotion surface crosses public Gateway schemas plus sensitive config/trace data that needs owner-approved shape and redaction before implementation.

## What This Changes

Emit an additive Gateway v2 metadata envelope with routing/config lineage, policy, timing, gateway version, decision trace, and reasoning-token usage so live benchmark smoke can validate gateway output.

## Best Possible Solution

Define a typed, additive, redacted Gateway metadata envelope at the protocol and HTTP response boundaries, document versioning and omitted-field semantics, then wire it through the approved live benchmark-facing surfaces with compatibility tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is an instrumentation feature request. Source inspection gives a high-confidence verification path because the relevant schemas and response builders do not contain the requested fields.

## Solution Assessment

Unclear because there is no patch yet. Emitting from the routing layer is directionally maintainable, but the safe first step is owner approval of schema shape, redaction policy, target surfaces, versioning, and reasoning-token semantics.
