## Summary

Keep open. Current main still lacks the requested Gateway v2 metadata envelope on the inspected Gateway protocol, OpenAI-compatible Chat Completions, OpenResponses, and agent acknowledgement surfaces; exposing config lineage and decision traces also needs maintainer-approved schema, redaction, and versioning before implementation.

## What This Changes

Gateway should emit promotion-grade v2 routing/config lineage, policy, timing, decision trace, gateway version, and token usage metadata for benchmark smoke validation.

## Best Possible Solution

Define a typed, additive, redacted Gateway v2 metadata envelope at the protocol and HTTP response boundaries, document omitted-field and versioning semantics, then wire it through the benchmark-facing live surfaces with compatibility tests and provider-specific usage normalization.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is an instrumentation feature request. We do have a high-confidence verification path for the missing behavior from source inspection of the relevant schemas and response builders.

## Solution Assessment

Unclear because there is no patch yet. Emitting from the routing layer is directionally reasonable, but the safest path is maintainer approval of schema shape, target surfaces, redaction policy, versioning, and reasoning-token semantics first.
