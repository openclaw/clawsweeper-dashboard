## Summary

Keep open: current main still lacks the requested routing/config lineage, policy, timing, decision-trace, and reasoning-token metadata on the Gateway response surfaces. This is a public Gateway wire-schema expansion with redaction and versioning implications, so it needs Gateway/protocol owner review rather than cleanup closure.

## What This Changes

Request Gateway v2 responses to include routing/config lineage, policy identifiers, decision trace, timing, and token-usage metadata for benchmark smoke promotion.

## Best Possible Solution

Define an additive, typed, redacted, versioned Gateway metadata envelope at the protocol and HTTP response boundaries, then wire it through routing, usage, and timing with docs and compatibility tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is an instrumentation feature request. Source inspection gives a high-confidence verification path because the requested fields are absent from current schemas/builders and tests only cover standard usage shapes.

## Solution Assessment

Unclear because there is no patch yet. Emitting from the routing layer is directionally plausible, but schema approval, redaction policy, versioning, and reasoning-token semantics should come before wiring responses.
