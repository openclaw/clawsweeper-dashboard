Keep open. The existing ClawSweeper comment already identified this as real remaining Gateway metadata work, and a fresh current-main review at 4cbd1b53 confirms the requested routing/config lineage, timing, decision trace, gateway version, and reasoning-token fields are still absent from the Gateway WebSocket chat event, agent acknowledgement, OpenAI-compatible Chat Completions, OpenResponses, and shared usage surfaces. ## What This Changes

Requests Gateway v2 live responses to emit promotion-grade routing/config lineage, policy, timing, gateway version, decision trace, and token-usage metadata so benchmark smoke tests can run without stub-only mode.

## Best Possible Solution

Define a typed, additive, redacted Gateway v2 metadata envelope at the protocol and HTTP response boundary, then wire it consistently through the approved benchmark-facing live surfaces with compatibility tests, docs, omitted-field semantics, and provider-specific usage normalization.

## Reproduction Assessment

## Solution Assessment
