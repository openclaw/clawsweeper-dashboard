## Summary

Keep open: current main still emits only standard response and usage shapes across Gateway Chat Completions, OpenResponses, WebSocket chat events, and agent ack paths. Adding the requested surface is a public, security-sensitive Gateway schema change that needs owner review before implementation.

## What This Changes

Request that Gateway v2 responses include routing/config lineage, policy, timing, gateway-version, decision-trace, and reasoning-token usage metadata for benchmark smoke validation.

## Best Possible Solution

Define an additive, typed, redacted, versioned Gateway metadata envelope at the protocol and HTTP response boundaries, then wire it through the live response surfaces with docs and compatibility tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is an instrumentation feature request. Source inspection gives a high-confidence verification path because the current schemas and response builders do not contain the requested fields.

## Solution Assessment

Unclear because there is no patch yet. Emitting from the routing layer is directionally plausible, but the maintainable path is to approve the schema, redaction policy, versioning, and reasoning-token semantics before wiring responses.
