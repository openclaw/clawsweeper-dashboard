## Summary

Keep open. Current main still lacks the requested promotion-grade Gateway v2 metadata envelope on the chat, OpenAI-compatible, OpenResponses, and agent-ack response surfaces, and the config/trace fields need Gateway owner-approved schema and redaction before implementation.

## What This Changes

The issue requests Gateway responses to emit routing/config lineage, policy/version, timing, decision trace, and token-usage metadata so live benchmark smoke tests can run without stubs.

## Best Possible Solution

Define an additive, typed, redacted Gateway metadata envelope at the protocol and HTTP response boundaries, document versioning and omitted-field semantics, then wire it through the benchmark-facing live surfaces with compatibility tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is an instrumentation feature request. Source inspection gives a high-confidence verification path because the relevant schemas and response builders do not contain the requested fields.

## Solution Assessment

Unclear because there is no patch yet. Emitting from the routing layer is directionally maintainable, but schema shape, redaction policy, target surfaces, versioning, and reasoning-token semantics need owner approval first.
