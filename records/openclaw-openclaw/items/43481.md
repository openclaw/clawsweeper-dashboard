## Summary

Keep open: current main and v2026.6.10 still lack a provider media trust policy, current fallback handling only drops explicit retry images, prompt-referenced images can still be hydrated for fallback attempts, and the linked open PR at https://github.com/openclaw/openclaw/pull/57959 is an unresolved implementation candidate rather than a merged fix.

## What This Changes

The issue requests provider-level media trust controls so cross-provider model fallback does not send user images or attachments to providers the user has not trusted for that data class.

## Best Possible Solution

Keep this issue as the canonical tracker until maintainers land a privacy-preserving fallback media policy or explicitly decide that configured model fallback is sufficient consent for media transfer.

## Reproduction Assessment

Yes, source-level. Current fallback retries drop only explicit `opts.images`, while the embedded runner can still call `detectAndLoadPromptImages` for prompt-referenced images without a fallback/provider trust gate.

## Solution Assessment

Unclear until maintainer security/product review. A trust-policy direction fits the problem, but the permanent default and upgrade behavior need a maintainer decision before implementation is considered settled.
