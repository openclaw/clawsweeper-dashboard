## Summary

Keep open: current main still has no provider-level media trust policy, prompt-detected images can still be loaded during fallback, and the issue is paired with an open conflicting implementation PR.

## What This Changes

Request provider trust controls so images and attachments are not sent across model-provider fallback boundaries unless that media sharing is explicitly trusted.

## Best Possible Solution

Keep this issue tied to https://github.com/openclaw/openclaw/pull/57959 or a replacement that makes one retry-media decision cover explicit images, prompt-detected images, attachments, config, and docs.

## Reproduction Assessment

Yes. Source inspection shows fallback retries strip explicit images while prompt-local detection still runs, and current config/docs have no provider media-trust control; I did not run a live provider fallback.

## Solution Assessment

Unclear. The implementation boundary is likely runAgentAttempt plus the embedded runner, but maintainers still need to decide whether the right policy is configurable trust or fail-closed cross-provider media stripping.
