## Summary

Keep open. Current main still has no provider-level fallback media trust policy, prompt-detected image and attachment refs are still loaded for the selected fallback model without cross-provider trust context, and open PR #57959 is already the paired implementation candidate using closing syntax.

## What This Changes

Request provider-level fallback media trust controls so user images and attachments are not unexpectedly forwarded to a different provider during model fallback.

## Best Possible Solution

Keep this issue open until PR #57959 lands or maintainers replace it with a schema-backed fallback media trust policy covering explicit images, prompt-detected image refs, attachment-style refs, same-provider retries, and cross-provider retries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a cross-provider model fallback and submit a prompt containing a local image path or attachment-style image marker; current main can still load prompt-detected media for the selected fallback model without target-provider trust context.

## Solution Assessment

Unclear. The paired PR uses fail-closed cross-provider media stripping, but maintainers still need to decide whether that is the desired policy or whether the configurable provider trust design requested here is required.
