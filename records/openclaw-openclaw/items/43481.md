## Summary

Keep open: current main still lacks a provider media-trust policy, prompt-detected image and attachment refs can still be loaded for fallback-capable models, and the linked closing PR remains open and conflicting rather than merged.

## What This Changes

Request provider-level media trust controls so cross-provider model fallback can distinguish text fallback trust from image and attachment forwarding trust.

## Best Possible Solution

Resolve https://github.com/openclaw/openclaw/pull/57959 or a narrower replacement with one provider-media trust policy covering explicit images, prompt-detected refs, attachment refs, config/docs, tests, and upgrade behavior.

## Reproduction Assessment

Yes for source-level reproduction. Current main lacks a provider-media trust policy, and fallback-capable embedded runs can still load prompt-detected image and attachment refs through detectAndLoadPromptImages without a fallback trust gate.

## Solution Assessment

Unclear. A configurable provider trust policy is plausible, but maintainers still need to choose the compatibility default, fail-closed behavior, docs/config surface, and upgrade path before this is a settled fix.
