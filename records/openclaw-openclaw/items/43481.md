## Summary

Keep open. Current main still lacks a provider-level fallback media trust policy, prompt-detected image and attachment references are still loaded without fallback-provider trust context, and open PR #57959 is already the paired implementation candidate using closing syntax for this issue.

## What This Changes

The issue asks for provider-level fallback media trust controls so images and attachments are not sent across provider boundaries unless the target provider is trusted for that data.

## Best Possible Solution

Keep this issue open until PR #57959 lands or is superseded by a centralized, schema/docs-backed fallback media trust policy that covers explicit images, prompt-detected image references, attachment-style refs, same-provider retries, and cross-provider retries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure a cross-provider model fallback and submit a prompt containing a local image path or attachment-style image marker; current main can still load prompt-detected media for the fallback model without target-provider trust context.

## Solution Assessment

No current-main solution exists. PR #57959 is the paired candidate, but maintainers should decide whether its fail-closed cross-provider media stripping is the desired policy or whether the configurable trust-policy design from this issue is required first.
