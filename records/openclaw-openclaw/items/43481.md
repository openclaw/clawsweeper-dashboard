## Summary

Current main and v2026.6.5 still lack a provider trust boundary for fallback images or attachments, while the open linked PR remains the active implementation path and requires maintainer product/security review.

## What This Changes

Add provider-level trust controls for forwarding images and attachments during cross-provider model fallback.

## Best Possible Solution

Keep this issue as the canonical decision and land one policy that computes a closed non-text sharing mode at fallback selection, carries it into every image or attachment ingestion path, and documents an explicitly approved upgrade-safe default.

## Reproduction Assessment

Yes at source level: current main can switch provider candidates and then resolve explicit plus prompt-detected images for the selected attempt without a separate trust policy. A live provider-side request capture has not been run.

## Solution Assessment

Unclear on the exact policy shape: a canonical fallback data-sharing decision is the right layer, but maintainers must choose between compatibility-preserving opt-in restriction and fail-closed cross-provider stripping.
