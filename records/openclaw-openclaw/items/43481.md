## Summary

Keep open: current main still lacks a provider media-trust policy, source inspection shows prompt-detected image and attachment refs can be loaded for fallback-capable models without a cross-provider trust gate, and the open closing PR remains unmerged and conflicting.

## What This Changes

The issue requests provider-level media trust controls so cross-provider model fallback can avoid forwarding user images and attachments unless that provider is trusted for media.

## Best Possible Solution

Resolve https://github.com/openclaw/openclaw/pull/57959 or a narrower replacement with one provider-media trust policy covering explicit images, prompt-detected refs, attachment refs, skipped fallback candidates, config/docs, tests, and upgrade behavior.

Yes for source-level reproduction. Current main has cross-provider fallback paths and still calls the prompt image/attachment loading path without a provider media-trust gate.

## Solution Assessment

Unclear. A configurable provider trust policy is plausible, but maintainers still need to choose the compatibility default, fail-closed behavior, docs/config surface, and upgrade path before this is a settled fix.
