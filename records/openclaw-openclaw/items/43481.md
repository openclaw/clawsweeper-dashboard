## Label Justifications

- `P1`: Cross-provider fallback can affect sensitive user media in an active agent workflow and needs timely maintainer security and product review.
- `impact:security`: The issue concerns whether user-supplied images and attachments cross an intended provider trust boundary.
- `impact:auth-provider`: The affected behavior is model-provider fallback routing and the data sent to the selected provider.

## Summary

Keep open: current main still has no provider-aware media trust policy, and the open linked implementation remains the canonical path pending security and product decisions about defaults, compatibility, and complete coverage of explicit and prompt-detected images.

## What This Changes

Add a provider-aware trust boundary for images and attachments forwarded during cross-provider model fallback.

## Best Possible Solution

Define one generic media-sharing decision at the model-fallback boundary, carry the primary and target provider facts into the attempt, apply the result to explicit and prompt-detected media, and document and test the approved default for fresh and existing configurations.

## Reproduction Assessment

Yes at source level: current main has separate explicit-image and prompt-detected-image paths during fallback and no provider trust policy. This read-only review did not make a live provider request.

## Solution Assessment

The requested trust-policy direction is reasonable, but the exact configuration shape and default are not yet established; the best implementation is a centralized, comprehensive fallback-boundary decision rather than scattered provider checks or a partial image-strip helper.
