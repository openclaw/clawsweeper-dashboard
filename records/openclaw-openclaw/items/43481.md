## Summary

Keep open. Current main still lacks provider-level fallback media trust, prompt-detected media can be loaded for the selected fallback model without target-provider trust context, and open PR #57959 is already the linked implementation candidate using closing syntax.

## What This Changes

Request provider-level fallback media trust controls so cross-provider retries do not send user images or attachments to untrusted fallback providers.

## Best Possible Solution

Keep this issue open until #57959 or a replacement lands with schema-backed fallback media policy covering explicit images, prompt-detected image refs, attachment-style refs, same-provider retries, and cross-provider retries.

## Reproduction Assessment

Yes, by source inspection. Configure a cross-provider fallback and submit a prompt containing a local image path or `[Image: source: ...]`; current main can still load prompt-detected media for the selected fallback model without target-provider trust context.

## Solution Assessment

Unclear. The linked PR’s fail-closed stripping/suppression may be a good security hardening path, but the issue asks for configurable provider trust, so maintainers should decide the product policy before closing this as solved.
