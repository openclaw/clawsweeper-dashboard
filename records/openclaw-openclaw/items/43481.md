## Summary

Keep open. Current main still has no provider-level fallback media trust policy and the remaining prompt-detected media path can load image or attachment refs for the selected fallback model without cross-provider trust context; open PR #57959 is already the paired implementation candidate using closing syntax, so this issue should not be closed before that PR lands or is superseded. https://github.com/openclaw/openclaw/issues/43481

## What This Changes

Request provider-level fallback trust controls so images and attachments do not cross provider boundaries unless the target provider is trusted for that media.

## Best Possible Solution

Keep this issue open until PR #57959 lands or is replaced by a schema-backed fallback media trust policy covering explicit images, prompt-detected image refs, attachment-style refs, same-provider retries, and cross-provider retries.

Unclear. PR #57959 is the paired candidate, but maintainers should decide whether fail-closed cross-provider media stripping is the desired policy or whether the configurable provider trust design proposed here is required.
