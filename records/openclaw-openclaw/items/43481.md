## Summary

Keep open. Current main still lacks a provider-level fallback media trust policy, prompt-detected image and attachment refs can still be loaded for the selected fallback model without target-provider trust context, and open PR #57959 is already the paired implementation candidate.

## What This Changes

Request provider-level trust controls so cross-provider model fallbacks do not forward images or attachment-derived media unless the target provider is trusted.

## Best Possible Solution

Keep this issue open until PR #57959 lands or maintainers replace it with a schema-backed fallback media trust policy covering explicit images, prompt-detected refs, attachment/media URIs, same-provider retries, and cross-provider retries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a cross-provider fallback and send a prompt containing a local image path, [Image: source: ...], or media attachment marker; current main can still load prompt-detected media for the selected fallback model without target-provider trust context.

## Solution Assessment

Unclear. The paired PR's fail-closed stripping is a narrower privacy hardening, but maintainers still need to decide whether that is the desired policy or whether the configurable provider-trust design requested here is required.
