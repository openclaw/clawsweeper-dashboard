## Summary

Keep open. Current main still lacks provider-level fallback media trust configuration, and prompt-detected local image/media refs can still be loaded for the selected fallback provider without target-provider trust context; open PR #57959 is the linked implementation candidate and should be reviewed rather than closing this early.

## What This Changes

Requests a provider-level media trust policy so cross-provider model fallback does not forward user images or attachments without explicit trust.

## Best Possible Solution

Keep this issue open until PR #57959 lands or maintainers replace it with a schema-backed fallback media trust policy covering explicit images, prompt-detected paths, media:// attachment refs, same-provider retries, and cross-provider retries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a cross-provider model fallback and send a prompt containing a local image path, [Image: source: ...], or [media attached: media://inbound/...] marker; current main can load those prompt/media refs for the selected fallback model without target-provider trust context.

## Solution Assessment

Unclear. The current-main behavior is not a complete provider-trust solution, and maintainers still need to decide whether the best fix is configurable trust as requested or the fail-closed cross-provider stripping approach in PR #57959.
