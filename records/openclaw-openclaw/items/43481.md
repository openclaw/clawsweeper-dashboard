## Summary

Keep open: current main still lacks a provider-level media trust policy, prompt-detected image/attachment refs can still be loaded during fallback, and an open closing PR already owns the implementation path. This is security-sensitive product behavior, not stale cleanup.

## What This Changes

Add provider-level trust controls so cross-provider model fallback can distinguish text fallback trust from image and attachment data trust.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/57959 with one shared retry-media decision covering explicit images, prompt-detected images, attachment refs, config, and docs.

## Reproduction Assessment

Yes. Source inspection of current main shows fallback retries strip explicit images while prompt-local image and attachment detection still runs, and no config/docs media-trust policy exists; I did not run live providers.

## Solution Assessment

Unclear. The durable implementation likely belongs at the runAgentAttempt/runWithModelFallback boundary, but maintainers need to decide whether the policy is configurable trust or fail-closed cross-provider media stripping.
