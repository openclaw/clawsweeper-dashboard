Keep open: current main still lacks the requested provider media-trust boundary, and the issue is paired with an open closing PR, so it should stay open until that PR is reviewed, landed, or replaced.

## What This Changes

The issue requests provider-level control over whether images and attachments may cross provider boundaries during model fallback retries.

## Best Possible Solution

Resolve the linked PR path with one shared fallback-media decision covering explicit images, prompt-detected images, attachment refs, config, docs, and upgrade behavior.

## Reproduction Assessment

Yes, by source inspection rather than live provider execution. Current main has no provider media-trust setting, strips only explicit retry images by blanket fallback state, and still allows prompt/attachment image detection in the embedded runner path.

## Solution Assessment

Unclear. A configurable trust policy matches the requested privacy control, but maintainers need to decide whether the durable boundary should be configurable trust or fail-closed cross-provider media suppression at the runAgentAttempt/runWithModelFallback boundary.
