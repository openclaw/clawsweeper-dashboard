## Summary

Keep open: current main still lacks a provider-level media trust policy, prompt-detected image and attachment refs can still be loaded during fallback, and an open closing PR already owns the implementation path.

## What This Changes

The issue requests provider-level media trust handling so cross-provider fallback does not unintentionally forward user images or attachments.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/57959 with one shared retry-media decision covering explicit images, prompt-detected images, attachment refs, config, and docs.

## Reproduction Assessment

Yes. Source inspection of current main shows fallback retries strip explicit images while prompt-local image and attachment detection still runs, and no config/docs media-trust policy exists; I did not run live providers.

## Solution Assessment

Unclear. A configurable trust policy may be the right product shape, but maintainers need to decide whether the durable fix is explicit trust configuration or fail-closed cross-provider media stripping at the runAgentAttempt/runWithModelFallback boundary.
