## Summary

Keep open: the latest PR head still lets a read-only direct status lookup clear and persist an auto fallback override because model selection probes by default while the status runtime passes a mutable session store without opting out. The `maintainer` label also prevents cleanup close, but the remaining blocker is narrow and repairable.

## What This Changes

The PR adds cooldown-gated primary probing for auto-persisted fallback model overrides, persists a last-probe timestamp, and updates focused tests, docs, and the changelog.

## Best Possible Solution

Land a small follow-up on this PR that opts direct status lookups out of auto-fallback probing while keeping cooldown probing for real reply and direct agent runs.

## Reproduction Assessment

Yes, from source: direct status resolution supplies a mutable `sessionStore` and `storePath`, while the PR probes and persists unless `allowAutoFallbackPrimaryProbe` is explicitly false. I did not run tests in this read-only review.

## Solution Assessment

No, not as patched. Cooldown probing is a maintainable fix for real model runs, but observational status paths need an explicit read-only opt-out or equivalent non-mutating selection path.
