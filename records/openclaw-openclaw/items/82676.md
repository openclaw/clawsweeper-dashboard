  - body: `resolveDirectStatusReplyForSession` still calls this helper with a mutable `sessionStore` and `storePath` but does not pass `allowAutoFallbackPrimaryProbe: false`. Since the origin remains the primary, this refreshes the cooldown and can postpone recovery probes indefinitely under unstable fallback chains; only refresh the timestamp when the primary was actually attempted.
  - confidence: 0.86
