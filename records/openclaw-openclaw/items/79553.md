## Summary

Keep open: current main and v2026.6.10 still leave the guided wizard path able to call external setup callbacks without external-plugin credential promotion or selected-account scoping, and the focused fix PR is closed unmerged.

## What This Changes

The issue reports that adding a new account through the channel wizard can overwrite existing multi-account credentials for plugins such as WeCom.

## Best Possible Solution

Land a focused setup fix that promotes external account-scoped root credentials before non-default wizard writes and scopes legacy/default-account plugin callbacks to the selected account, without coupling it to DM-policy flag work.

## Reproduction Assessment

Yes. Source inspection plus the npm-published WeCom 2026.5.7 setup package show selected-account wizard setup can still run callbacks that read and write the default account.

## Solution Assessment

Yes for the central bug: the maintainable fix belongs in the shared setup promotion and wizard callback scoping boundary. The DM-policy flag proposal is separate and should not be required to stop credential overwrites.
