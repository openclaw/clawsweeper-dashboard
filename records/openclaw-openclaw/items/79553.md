## Summary

Keep open: current main and the latest v2026.6.6 tag still show the guided setup path writing through plugin setup callbacks without the same promotion/scoping needed for the reported WeCom-style multi-account overwrite, while the linked proof PR was closed unmerged and the open DM-policy PR is a separate enhancement.

## What This Changes

The issue asks for the channel setup wizard to preserve existing account credentials and write new credentials only under the selected new account when adding a multi-account plugin account.

## Best Possible Solution

Land a focused setup-path fix that promotes legacy account-scoped root fields before named-account wizard writes and scopes default-account-only plugin callbacks to the selected account, while leaving the DM-policy flag feature to separate review.

## Reproduction Assessment

Yes, from source inspection: the guided wizard selected-account path reaches plugin setup.applyAccountConfig without the non-default promotion that the docs require, and the published WeCom 2026.5.7 artifact confirms default-account callbacks can ignore accountId. I did not execute the wizard because this review is read-only.

## Solution Assessment

Yes for the central bug: the narrow maintainable fix belongs in the shared setup adapter/promotion boundary, with WeCom-style callback scoping covered by contract tests. The separate DM-policy flag proposal is not required to stop credential overwrites.
