## Summary

Keep open: the reported overwrite remains source-reproducible on current main, and an open linked fix PR explicitly resolves this issue but has not merged yet.

## What This Changes

The issue reports that the channel setup wizard can overwrite existing multi-account credentials when adding a new account for external account-schema plugins such as WeCom.

## Best Possible Solution

Land a shared setup-helper fix that promotes account-scoped root credentials through the plugin setup contract before writing the selected account, with regression coverage for an external WeCom-style setup flow.

## Reproduction Assessment

Yes, source-backed: current main still writes selected-account patches without contract-aware promotion, while the docs define promotion before non-default account writes. I did not run a live WeCom network setup in this read-only pass.

## Solution Assessment

Yes, the narrow maintainable fix is in the shared setup promotion/write path, not a WeCom-only config workaround. The linked open PR is the active implementation path and should be reviewed or closed before this issue is resolved.
