## Summary

Keep open: current main and v2026.6.8 still leave the guided wizard path without the external-plugin credential promotion and callback scoping needed for the reported WeCom-style overwrite; the focused proof PR is closed unmerged and the open DM-policy PR is a separate enhancement.

## What This Changes

The issue reports that adding a new account through the setup wizard can reuse or overwrite an existing multi-account channel credential instead of writing the selected new account.

## Best Possible Solution

Land a focused shared setup fix that promotes external account-scoped root fields before non-default wizard writes and scopes legacy/default-account plugin callbacks to the selected account.

## Reproduction Assessment

Yes, from source inspection and the published WeCom package: selected-account wizard setup still reaches plugin callbacks that can read/write the default account unless the host scopes them. I did not run the interactive wizard in this read-only checkout.

## Solution Assessment

Yes for the central bug: the narrow maintainable fix belongs in the shared setup adapter and promotion boundary, with WeCom-style callback scoping covered by regression tests. The DM-policy flag proposal is not required to stop credential overwrites.
