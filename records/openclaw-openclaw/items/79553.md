## What This Changes

The issue reports that adding a new non-default account in the setup wizard can reuse or overwrite existing channel credentials for account-schema plugins such as WeCom.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79574 if it remains correct, then close this issue from the merged fix.

## Reproduction Assessment

No live current-main reproduction was run in this read-only review. Source inspection gives a clear reproduction path: the documented non-default-account promotion contract is not applied by the shared setup adapter before account credential writes.

## Solution Assessment

Yes, the linked PR’s direction is the narrow maintainable fix: reuse the contract-backed promotion path in shared setup code and add regression coverage, rather than adding plugin-specific WeCom logic or a new config option.
