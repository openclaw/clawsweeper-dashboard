## Summary

Keep open: current main still lacks the requested conformance harness and pre-abort handling, but this PR is not merge-ready because it imports an undeclared TypeBox package, can reject after notifying a client tool call, and needs a rebase that preserves newer client-tool behavior on main.

## What This Changes

The PR adds a Pi tool adapter conformance matrix and parameterized tests, documents the harness, and adds AbortSignal checks around agent and client tool adapter execution.

## Best Possible Solution

Land a rebased version that keeps the conformance harness and abort fast-path, imports from `typebox`, checks abort before hook/tool/client delegation work, returns pending once client delegation is notified, and preserves current client-tool recorder and parameter coercion semantics.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: pass an already-aborted `AbortSignal` into the agent or client adapter execute function and it can still reach hook, tool, reservation, or client delegation because the agent side lacks a pre-check and the client side ignores the signal.

## Solution Assessment

No, not as currently proposed. The harness direction is sound, but the patch should use the declared package import, avoid rejecting after client notification, add the required changelog entry for the runtime fix, and rebase without losing current client-tool recorder/coercion behavior.
