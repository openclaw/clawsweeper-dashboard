## Summary

Keep open. Current main still lacks the proposed Pi tool-adapter conformance harness and the already-aborted signal pre-checks, but this PR should not merge as-is because the branch adds a test import for an undeclared TypeBox package and keeps an abort check after client delegation notification that can produce a notified-but-rejected state.

## What This Changes

The PR adds a Pi tool adapter conformance matrix and parameterized tests, documents the harness in the Pi workflow guide, and adds abort pre-checks around agent and client tool adapter execution.

## Best Possible Solution

Revise this PR rather than close it: rebase onto current main, keep the conformance harness, use the repo-declared `typebox` package, implement already-aborted handling without rejecting after client delegation has been signaled, and preserve current client parameter coercion behavior before normal changed-gate validation.

## Reproduction Assessment

Yes. The missing current-main behavior is reproducible by source inspection and a focused unit test: pass an already-aborted signal into the agent/client adapter execute functions and assert hooks, tool execution, or client delegation are not called.

## Solution Assessment

No, not as currently proposed. The harness direction is maintainable, but the implementation should use the declared TypeBox dependency and avoid the post-`onClientToolCall` abort check; a rebase also needs to retain current JSON-string client param coercion.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use the declared TypeBox package:** `src/agents/pi-tool-definition-adapter.conformance.test.ts:2`
  - body: The new conformance test imports `Type` from `@sinclair/typebox`, but this repo declares and imports `typebox` 1.1.34. Under pnpm's normal strict resolution this added test can fail to resolve unless the package metadata is intentionally changed, so import from `typebox` here.
  - confidence: 0.92
- **[P2] Return pending after notifying the client:** `src/agents/pi-tool-definition-adapter.ts:244`
  - body: This abort check runs after `onClientToolCall` has already been invoked. If that synchronous callback aborts the controller, the client has been notified of a delegated tool call but the adapter rejects instead of returning the pending result, leaving the two sides inconsistent. Move the check before the callback or remove this final check.
  - confidence: 0.84
