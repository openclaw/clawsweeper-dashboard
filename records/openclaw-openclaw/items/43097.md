## Summary

Keep this PR open: the conformance harness and abort fast-path are useful, but the current patch still has two blocking correctness issues and needs a rebase over current Pi client-tool behavior before merge.

## What This Changes

The PR adds a Pi tool-definition adapter conformance matrix and parameterized tests, documents the harness in the Pi workflow guide, and adds abort-signal checks around agent and client tool adapter execution.

## Best Possible Solution

Land a rebased version that keeps the harness and abort fast-path, imports from `typebox`, checks abort before hook/tool/client delegation work, returns pending once client delegation is notified, and preserves current client-tool slot and parameter coercion semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: pass an already-aborted `AbortSignal` into current main's agent or client adapter execute functions and the code can still reach hook, tool, or client delegation because the agent side has no pre-check and the client side ignores the signal.

## Solution Assessment

No, not as currently proposed. The conformance harness direction is sound, but the patch should use the declared package import, avoid rejecting after client notification, and rebase without losing current client-tool recorder/coercion behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Import Type from the declared TypeBox package:** `src/agents/pi-tool-definition-adapter.conformance.test.ts:2`
  - body: The new conformance test imports `Type` from `@sinclair/typebox`, but this workspace declares and imports `typebox`. With pnpm's strict package resolution this can fail unless package metadata is intentionally changed, so import from `typebox` here.
  - confidence: 0.92
- **[P2] Return pending after notifying the client:** `src/agents/pi-tool-definition-adapter.ts:244`
  - body: This abort check runs after `onClientToolCall` has already been invoked. If that synchronous callback aborts the controller, the client has been notified but the adapter rejects instead of returning the pending result; move the check before notification or drop it.
  - confidence: 0.84
