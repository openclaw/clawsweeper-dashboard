## Summary

Keep this PR open because current main still lacks the conformance harness and still has the source-visible already-aborted signal gap, but the branch needs fixes, a careful rebase, and real behavior proof before merge.

## What This Changes

The PR adds a Pi tool adapter conformance matrix/tests, documents the harness, and adds AbortSignal fast-fail checks around Pi agent/client tool adapter execution.

## Best Possible Solution

Land a rebased version that keeps the conformance harness and abort fast-path, imports from `typebox`, checks abort before hook/tool/client delegation work, preserves current client recorder/coercion semantics, and returns pending after client notification.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: pass an already-aborted `AbortSignal` into the agent or client adapter execute function and it can still reach hook, tool, reservation, or client delegation.

## Solution Assessment

No, not as currently proposed. The harness direction is sound, but the patch should use the declared package import, avoid rejecting after client notification, and rebase without losing current client-tool recorder/coercion behavior.
