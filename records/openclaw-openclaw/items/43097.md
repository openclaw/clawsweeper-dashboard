## Summary

Keep open. The Pi adapter conformance harness and abort fast-path still cover behavior missing from current main, but this PR is dirty against main and has two merge-blocking issues: an undeclared TypeBox import and a client-delegation abort check that can reject after notifying the client.

## What This Changes

The PR adds a Pi tool adapter conformance matrix and parameterized tests, documents the harness in the Pi workflow guide, and adds abort-signal checks around agent and client tool adapter execution.

## Best Possible Solution

Land a rebased version that keeps the harness and abort fast-path, imports Type from typebox, checks abort before hook/tool/client delegation work, returns pending once client delegation is notified, and preserves current client parameter coercion plus recorder semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: pass an already-aborted AbortSignal into current main's agent or client adapter execute functions and the code can still reach hook/tool/client delegation because the agent side has no pre-check and the client side ignores the signal.

## Solution Assessment

No, not as currently proposed. The conformance harness direction is sound, but the PR should fix the undeclared package import, remove or move the post-notification abort check, and rebase without losing current client-tool coercion/recorder behavior.
