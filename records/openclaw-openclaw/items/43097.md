## Summary

Keep this PR open: the conformance harness and already-aborted signal coverage are not fully present on current main, but the branch needs a refresh before it is mergeable. The current patch has blocking review findings and only test/CI proof, not real Pi/client-tool behavior proof.

## What This Changes

Adds a Pi tool-definition adapter conformance matrix/test suite, a docs note, and AbortSignal fast-fail checks in the agent and client tool adapters.

## Best Possible Solution

Land a refreshed version of the conformance coverage on current main, using the current typebox import, preserving recorder/coercion behavior, and checking abort state only before hooks/tool execution/client notification or before returning a result.

## Reproduction Assessment

Yes for the review findings: source inspection shows the stale import no longer matches current dependencies, and the client-tool abort check can reject after delegation has been emitted. I did not run the suite because this was a read-only review.

## Solution Assessment

No. The harness direction is useful, but this implementation should be rebased onto current main, import Type from typebox, and remove or move the post-notification abort check.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Import Type from the current typebox package:** `src/agents/pi-tool-definition-adapter.conformance.test.ts:2`
  - body: Current main has migrated agent code to `typebox` and `package.json` no longer declares `@sinclair/typebox`. Adding this new test with `@sinclair/typebox` will fail module resolution on a rebased branch, so import `Type` from `typebox` instead.
  - confidence: 0.92
- **[P2] Do not abort after client delegation:** `src/agents/pi-tool-definition-adapter.ts:244`
  - body: This new check runs after `onClientToolCall` has already notified the client. If that synchronous callback aborts the controller as a side effect, the adapter rejects after delegation was recorded instead of returning the pending result, leaving the agent/client state inconsistent. Drop this final check or perform the abort check before notifying the client.
  - confidence: 0.84
