## Summary

Keep open: the conformance harness is still useful, but this PR is not merge-ready against current main because the new test imports the old TypeBox package name, the client-tool abort check can reject after delegation, and contributor real behavior proof is still missing.

## What This Changes

Adds a Pi tool-adapter conformance matrix/test suite, documents it in the Pi workflow guide, and adds abort-signal fast-fail checks in the agent and client tool adapters.

## Best Possible Solution

Rebase onto current main, preserve the current client-tool recorder and parameter-coercion behavior, switch the conformance tests to `typebox`, avoid throwing after client delegation, and add redacted runtime proof plus focused Pi test/check output.

## Reproduction Assessment

Yes for the blocking findings. Source inspection shows the new test imports an undeclared package on current main, and the added client-tool abort check runs after the delegation callback.

## Solution Assessment

No. The direction is useful, but the current patch needs a current-main rebase, the repo-standard TypeBox import, and safer client-tool abort ordering before it is the narrow maintainable solution.
