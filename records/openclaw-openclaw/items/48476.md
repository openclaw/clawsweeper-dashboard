## Summary

Close as superseded: https://github.com/openclaw/openclaw/pull/81728 now tracks the safe retry-only launchd fix, while this branch’s unique legacy `launchctl load -w` fallback has blocking correctness and security concerns.

## What This Changes

The PR changes macOS LaunchAgent bootstrap to retry after already-registered failures, fall back to legacy `launchctl load -w` for GUI-domain failures, and adds focused launchd tests.

## Best Possible Solution

Review and land the retry-only launchd recovery in https://github.com/openclaw/openclaw/pull/81728 after macOS smoke proof, while keeping headless/sudo behavior on the existing GUI-session guidance unless a separate LaunchDaemon product decision is made.

## Reproduction Assessment

Yes for source-level PR review: PR head can return success from `launchctl load -w` solely on exit code 0 without proving the intended LaunchAgent registered. I did not establish a live macOS launchctl reproduction in this read-only review.

## Solution Assessment

No. The duplicate-bootstrap retry is useful, but the unqualified legacy `launchctl load -w` fallback is not the narrowest maintainable solution; https://github.com/openclaw/openclaw/pull/81728 keeps the safe retry-only slice.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Validate legacy load before returning success:** `src/daemon/launchd.ts:229-231`
  - body: This branch returns as soon as `launchctl load -w` exits 0, but legacy `load`/`unload` can return zero except for improper usage. In the fallback paths this can report install success even when the intended LaunchAgent was not registered.
  - confidence: 0.87
- **[P1] Avoid root-scoped load fallback for LaunchAgents:** `src/daemon/launchd.ts:229`
  - body: The fallback is reachable for sudo/wrong-user runs, but legacy `launchctl load` run as root targets the system domain rather than the target desktop user's `gui/<uid>` LaunchAgent domain. Returning success there can leave the intended user service uninstalled.
  - confidence: 0.83
- **[P2] Do not claim headless success with Aqua-only load:** `src/daemon/launchd.ts:229-231`
  - body: The fallback runs `launchctl load -w` without selecting a Background session while OpenClaw writes an Interactive per-user LaunchAgent. In SSH/headless contexts this can report success for a job that still depends on GUI-login behavior.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff adds a root-sensitive legacy launchd load success path for a per-user LaunchAgent.

Concerns:

- **[medium] Root fallback can target the wrong launchd domain:** `src/daemon/launchd.ts:229`
  - body: Legacy `launchctl load` infers its target from execution context; for sudo/root paths this can return success outside the intended per-user LaunchAgent domain, which matters because the PR explicitly makes sudo-triggered GUI-domain failures enter this fallback.
  - confidence: 0.82
