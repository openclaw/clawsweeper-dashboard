## Summary

Keep open. The PR has useful Windows scheduled-task and path-test cleanup, but the diff also changes production helper semantics and still has blocking regressions in home-prefix expansion and Windows executable resolution that prior review comments correctly called out.

## What This Changes

The PR changes Windows scheduled-task cleanup assertions and path/executable tests, and updates home-prefix expansion, executable resolution, and stable Node path matching helpers.

## Best Possible Solution

Preserve the Windows schtasks assertion normalization, but revise or split the production helper changes so tilde expansion remains replacement-only, PATHEXT is used for suffix search rather than explicit-file rejection, and stable-node Windows/casing behavior is covered with focused regression tests before merge.

## Reproduction Assessment

Yes. The diff gives high-confidence helper-level reproduction paths: `expandHomePrefix("~alice/bin/tool", { home: "/home/me" })` would become cwd-absolute, and on Windows a real `node.exe` can be rejected when caller PATHEXT is narrowed to `.CMD`.

## Solution Assessment

No. The narrow test-normalization goal is valid, but this branch is broader than the safest fix because it changes executable and approval path semantics in production helpers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve unmatched tilde tokens:** `src/infra/home-dir.ts:73`
  - body: Wrapping the replacement in `path.resolve(...)` also resolves inputs the regex intentionally leaves untouched. A value like `~alice/bin/tool` becomes cwd-anchored, and callers use this helper for executable and allowlist paths, so an unresolved token can silently target a different filesystem path.
  - confidence: 0.92
- **[P2] Keep PATHEXT out of explicit file checks:** `src/infra/executable-path.ts:41`
  - body: Passing the caller env into `isExecutableFile` makes Windows reject an already-found file when its extension is not in PATHEXT. With `PATHEXT=.CMD`, a concrete `node.exe` path can return `undefined` even though the executable exists; PATHEXT should drive suffix search, not invalidate qualified paths.
  - confidence: 0.88
- **[P3] Normalize formula casing before fallback:** `src/infra/stable-node-path.ts:32`
  - body: The Cellar matcher is case-insensitive, but the default formula fallback still checks `formula === "node"`. A path that only matched because of the `i` flag, such as `Cellar/Node/.../node.exe`, skips the prefix bin fallback.
  - confidence: 0.62

## Security Review
