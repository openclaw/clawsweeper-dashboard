## Summary

Keep open: the useful Windows assertion cleanup is mixed with runtime helper changes that still introduce command/path-resolution regressions, and the external PR still lacks real Windows behavior proof.

## What This Changes

The branch updates Windows scheduled-task and path/executable tests while changing home-prefix expansion, executable validation, and stable Node path fallback helpers.

## Best Possible Solution

Keep the useful Windows scheduled-task assertion normalization, preserve current home-prefix and executable-validation contracts, add focused regressions for any intended runtime changes, rebase, and provide redacted Windows proof.

## Reproduction Assessment

Yes for the PR-introduced blockers by source inspection: unsupported tilde-user inputs become cwd-absolute and caller PATHEXT is applied during explicit file validation. I did not run Windows tests in this read-only review.

## Solution Assessment

No. The schtasks assertion cleanup is reasonable, but changing approval-path and executable-resolution helper semantics as a side effect is not the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve unmatched tilde tokens:** `src/infra/home-dir.ts:73`
  - body: expandHomePrefix should not resolve the whole string after a failed replacement. Unsupported inputs like ~alice/bin/tool are currently left literal, but this change turns them into cwd-absolute paths that can change allowlist and executable matching.
  - confidence: 0.92
- **[P2] Keep caller PATHEXT out of file validation:** `src/infra/executable-path.ts:41`
  - body: Forwarding caller env into isExecutableFile makes an already-found Windows executable depend on that caller's PATHEXT. A sanitized env such as PATHEXT=.CMD can reject a concrete .exe path even though suffix lookup, not validation, should use caller PATHEXT.
  - confidence: 0.88
- **[P3] Normalize formula casing before fallback:** `src/infra/stable-node-path.ts:13`
  - body: The Cellar matcher becomes case-insensitive, but the fallback still checks formula === "node". A matched Cellar/Node/.../node.exe path would skip the default-formula bin fallback introduced by the same change.
  - confidence: 0.62
