Keep open: the schtasks assertion updates match current Windows runtime behavior, but the branch still introduces blocking path/executable regressions and lacks contributor-supplied real Windows behavior proof.

## What This Changes

The PR updates Windows path and schtasks tests and changes home-prefix expansion, Windows executable detection, and stable Node path fallback handling.

## Best Possible Solution

Keep the useful Windows test normalization, preserve current home-prefix and executable-validation contracts, add focused regressions, rebase, and provide redacted Windows proof.

## Reproduction Assessment

Yes for the PR-introduced blockers by source inspection: unmatched ~user inputs become cwd-absolute and caller PATHEXT is applied during file validation. I did not run Windows tests in this read-only review.

## Solution Assessment

No. The schtasks assertion update is directionally reasonable, but changing approval-path and executable-resolution semantics as a side effect is not the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve unmatched tilde tokens:** `src/infra/home-dir.ts:73`
  - body: `expandHomePrefix` should not resolve the whole string after a failed replacement. Unsupported inputs like `~alice/bin/tool` are currently left literal, but the PR turns them into cwd-absolute paths, which can change allowlist and executable matching.
  - confidence: 0.92
- **[P2] Keep caller PATHEXT out of file validation:** `src/infra/executable-path.ts:41`
  - body: Forwarding caller env into `isExecutableFile` makes an already-found Windows executable depend on that caller's PATHEXT. A sanitized env such as `PATHEXT=.CMD` can reject a concrete `.exe` path even though suffix lookup, not validation, should use caller PATHEXT.
  - confidence: 0.88
- **[P3] Normalize formula casing before fallback:** `src/infra/stable-node-path.ts:13`
  - body: The Cellar matcher becomes case-insensitive, but the fallback still checks `formula === "node"`. A matched `Cellar/Node/.../node.exe` path would skip the default-formula bin fallback introduced by the same change.
  - confidence: 0.62
