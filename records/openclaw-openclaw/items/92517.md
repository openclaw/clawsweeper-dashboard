## Summary

This PR should not merge as written because it disables an existing Discord fallback that warns users about otherwise-unacknowledged mutating-tool failures. The failed-progress filtering is useful, but the warning suppression is a blocking delivery regression and the PR also lacks after-fix real Discord proof.

## What This Changes

The PR filters failed item and command-output progress from Discord drafts and unconditionally suppresses synthetic tool-error warning payloads for Discord-originated runs.

## Best Possible Solution

Retain the failed draft-progress filters, preserve Discord’s existing fallback-only tool-error warning, and prove both behaviors with focused regression coverage plus a redacted live Discord run.

## Reproduction Assessment

Yes at source level: failed callbacks currently reach the Discord draft renderer, and the added mock tests exercise that path. There is no after-fix live Discord reproduction of the complete user-visible flow.

## Solution Assessment

No. Filtering failed draft events is appropriately local, but disabling the independent fallback warning violates the existing failure-delivery contract; preserving that warning is the narrower solution.
