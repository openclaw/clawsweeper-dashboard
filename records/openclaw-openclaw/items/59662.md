## Summary

Keep open. Current main still treats Anthropic Max usage-alert text as ordinary assistant-visible text through extraction, message finalization, and final reply payload construction; no merged fix or canonical duplicate now owns the remaining work.

## What This Changes

The issue requests stripping Anthropic Max usage/billing alert text blocks before assistant transcript persistence and channel delivery.

## Best Possible Solution

Add one shared assistant-output sanitizer before history persistence and channel delivery, with tests for standalone and embedded Anthropic Max alerts and a deliberate capture path if maintainers want to preserve quota metadata internally.

## Reproduction Assessment

Yes, from source inspection. A synthetic assistant content block with type text and the reported alert text would pass through visible-text extraction, message-end finalization, and reply payload construction as non-empty assistant output.

## Solution Assessment

Unclear until maintainer/security review chooses strip-only versus strip-and-capture. The best technical boundary is a shared assistant-output sanitizer before transcript and delivery, not a Discord-only filter or a generic regex-only user setting.
