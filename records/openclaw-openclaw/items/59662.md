## Summary

Keep open. Current main and the latest release still lack an Anthropic Max usage-alert sanitizer, and the affected assistant text extraction, finalization, transcript-adjacent, and reply payload paths still treat matching provider text as ordinary assistant output.

## What This Changes

The issue asks OpenClaw to strip Anthropic Max usage and billing alert text blocks before session transcript persistence and channel delivery.

## Best Possible Solution

Add a shared assistant-output sanitizer or capture step for Anthropic Max usage alerts before both history persistence and channel delivery, with focused tests for standalone and embedded alerts that preserve normal answer text.

## Reproduction Assessment

Yes, from source inspection. A synthetic assistant text block containing the reported alert would pass through the current extraction and delivery paths as non-empty assistant text; I did not run a live Anthropic Max OAuth replay.

## Solution Assessment

Unclear pending maintainer direction on strip-only versus strip-and-capture/configurable handling. The narrow maintainable boundary is shared assistant-output sanitization before transcript and channel delivery.
