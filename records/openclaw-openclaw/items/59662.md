## What This Changes

The issue reports Anthropic Claude Max OAuth usage alert text being stored and delivered as assistant output instead of treated as provider billing metadata.

## Best Possible Solution

Add a provider-owned private strip-or-capture path before transcript persistence and channel delivery after maintainers decide whether the quota details should be discarded or stored as non-user-visible metadata.

## Reproduction Assessment

No high-confidence end-to-end repro yet: the source path is clear if Anthropic emits the alert as ordinary assistant text, but live Anthropic Max OAuth payload proof is still missing. Current main would stream, persist, and deliver such text as assistant output.

## Solution Assessment

A new generic outputStripPatterns config surface should not be added without explicit product approval.
