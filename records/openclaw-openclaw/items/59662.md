## Summary

Keep open. Current main still has no Claude Max usage-alert sanitizer or configured output-strip surface, and the assistant text extraction, message-end handling, and reply payload paths still treat a matching alert block as ordinary non-empty assistant output; the new comments add useful fix-shape caution but do not supersede the bug.

## What This Changes

Detect and remove Anthropic Claude Max usage and billing alert text blocks from assistant transcripts and outbound channel replies.

## Best Possible Solution

Add a shared assistant-output filter for Anthropic Max usage alert blocks before transcript persistence and channel delivery, with regression coverage for standalone and embedded alerts; defer generic output patterns or quota capture unless maintainers choose that broader product path.

## Reproduction Assessment

Yes, from source inspection. A synthetic assistant text block containing the reported alert would pass through the current sanitizer/extraction/payload path as ordinary non-empty assistant text; I did not run a live Anthropic OAuth replay.

## Solution Assessment

Yes for the narrow bug. A shared sanitizer in the assistant-visible extraction/delivery path is the maintainable fix; `messages.outputStripPatterns` or quota capture would be broader config/product work, not required to stop the leak.
