## Summary

Keep open. Current main still lacks a Claude Max usage-alert sanitizer, and the shared assistant text extraction, stream/message-end delivery, and final reply payload paths continue to treat non-empty ordinary assistant text as user-visible output.

## What This Changes

The issue requests stripping Anthropic Max usage/billing alert text blocks before transcript persistence and channel delivery.

## Best Possible Solution

Add a narrow provider-output sanitizer in the shared assistant-visible extraction path, with regression coverage for standalone and embedded alerts that preserves real answer text.

## Reproduction Assessment

## Solution Assessment

Yes. A targeted sanitizer for this provider-injected metadata is the narrowest maintainable fix; a broad configurable output-regex setting would be a separate product/config decision.
