## Summary

Keep open: current main and v2026.6.8 still lack a Claude Max usage-alert sanitizer, and the current Anthropic text stream, assistant finalization, persistence, and generic channel delivery paths would treat matching provider text as ordinary assistant output if Anthropic emits it as reported. This remains maintainer-reviewed because the unresolved fix choice is strip-only versus private quota capture, plus live Anthropic Max OAuth proof of the exact payload.

## What This Changes

The issue requests filtering Anthropic Claude Max usage-alert text blocks before session transcript persistence and channel delivery.

## Best Possible Solution

Add a provider-owned private strip-or-capture path before transcript persistence and channel delivery after maintainers choose whether the quota details should be discarded or captured as non-user-visible metadata.

## Reproduction Assessment

No, not end-to-end: the source path is clear if Anthropic emits the alert as ordinary assistant text content, but live Anthropic Max OAuth payload proof is still missing. The focused source check shows such text would be streamed, persisted, and delivered as assistant output today.

## Solution Assessment

Unclear: the suggested regex strip is a plausible mitigation, but the best maintainable fix depends on whether maintainers want strip-only behavior or private quota capture. A new generic outputStripPatterns config surface should not be added unless maintainers explicitly choose that product direction.
