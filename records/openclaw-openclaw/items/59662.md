## Summary

Keep open: current main still has no clear Anthropic Max usage-alert sanitation path, and ordinary Anthropic text blocks still flow through transcript persistence and channel delivery as assistant content. The report is concrete, but the implementation needs maintainer product/security direction on strip-only versus private quota capture.

## What This Changes

The issue requests that Anthropic Max usage/billing alert text blocks be treated as provider metadata, not assistant content, before transcript persistence and channel delivery.

## Best Possible Solution

Add provider-output sanitation before transcript persistence and channel delivery; if quota values are useful, store them in a private provider/session metadata surface instead of public assistant content.

## Reproduction Assessment

No live high-confidence reproduction was established in this read-only review. Source inspection does show a plausible current-main path from Anthropic `text` blocks into transcript persistence and channel delivery without a provider-metadata filter.

## Solution Assessment

The user-visible strip is directionally correct, but a generic `messages.outputStripPatterns` config is not clearly the best fix under the repo's high config-surface bar. A provider-aware strip-and-private-capture path is the cleaner maintainable option if maintainers want to preserve quota data.
