## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/59662. Anthropic Claude Max subscription/OAuth responses can inject a text block beginning `⚠️ **Claude Max Usage Alert**` with `Session:`, `Weekly:`, and `Extra: $.../$...` lines. Current OpenClaw treats that block as assistant output, so it can be stored in session JSONL and delivered to channels. Add a targeted sanitizer that removes only this provider-injected usage alert from assistant text before persistence and channel delivery. Cover both standalone alert blocks, which should produce no assistant reply payload, and embedded alert text, where the real assistant text before/after the alert should remain with sane whitespace. Prefer a shared helper used by assistant visible-text extraction/stream or provider output transformation so `handleTextStream`, `handleMessageEnd`, `assistantTexts`, and `buildEmbeddedRunPayloads` stay consistent. Verify whether stream output transforms run early enough to affect persisted session messages; if not, add the filter at the persistence-facing message normalization point too. Do not add a broad `messages.outputStripPatterns` product surface unless maintainers explicitly ask, and do not strip normal prose that mentions billing, usage, Session, Weekly, or Extra outside the exact alert shape. Add focused Vitest coverage and a `CHANGELOG.md` fix entry crediting `Thanks @tuncer-deniz`.
