- `impact:message-loss`: Provider metadata is being delivered as bot output to channels, which is a message-delivery correctness problem.

## Summary

Keep open. Current main still lacks an Anthropic Max usage-alert sanitizer, and the shared assistant text extraction, message-end finalization, and reply payload paths still treat non-empty provider text blocks as user-visible assistant output.

## What This Changes

Strip Anthropic Max Usage Alert billing/quota text blocks from assistant-visible transcripts and channel replies.

## Best Possible Solution

Add a shared assistant-output sanitizer that removes Anthropic Max usage alert blocks before transcript persistence and channel delivery, with tests for standalone and embedded alerts while preserving real answer text.

## Reproduction Assessment

Yes, from source inspection. A synthetic assistant text block containing the reported alert would pass through extractAssistantVisibleText, message_end finalization, and buildReplyPayload as ordinary non-empty assistant text on current main; I did not run a live Anthropic Max OAuth replay.

## Solution Assessment

Yes for the narrow bug. A targeted shared sanitizer in the assistant-visible extraction/delivery path is the maintainable fix; generic outputStripPatterns or quota capture would be separate product/config work.
