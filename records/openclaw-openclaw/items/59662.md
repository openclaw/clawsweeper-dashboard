## Summary

Keep open: current main still has no Anthropic Max usage-alert strip or private-capture path, and ordinary assistant text can still flow into transcript/final channel delivery. The issue is concrete, but the fix needs maintainer product/security direction on discard versus private quota capture.

## What This Changes

The issue requests suppression of Anthropic Claude Max usage/billing alert text blocks before session persistence and external channel delivery.

## Best Possible Solution

Add a central provider-notice classifier at the assistant text/transcript boundary, suppress matched provider billing notices from user-visible output, and retain quota data only as private metadata if maintainers choose that path.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No live high-confidence reproduction was run, but source inspection shows ordinary assistant text currently has no provider usage-alert classifier before stream emission, block reply delivery, or assistant text finalization.

## Solution Assessment

Is this the best way to solve the issue? No for the regex-only suggestion; the maintainable fix is a central provider notice classifier with tested suppression and a maintainer decision on private quota capture.
