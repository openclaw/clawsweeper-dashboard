## Label Justifications

- `P2`: This is a real user-visible sanitation bug with limited blast radius to Anthropic Max OAuth streams and channel delivery.
- `impact:security`: The reported alert contains provider quota and spend metadata that can leak into shared channel surfaces.
- `impact:message-loss`: Provider metadata is being delivered as bot output, which is a channel message-delivery correctness problem.
- `impact:session-state`: The same metadata is reported as persisted into session JSONL as assistant transcript content.

## Summary

Keep open. Current main still has no Anthropic Max usage-alert sanitizer, and the assistant text extraction, message-end finalization, and reply payload paths still treat matching provider text blocks as ordinary user-visible assistant output.

## What This Changes

The issue requests stripping Anthropic Claude Max usage/billing alert text blocks before transcript persistence and channel delivery.

## Best Possible Solution

Add a shared assistant-output sanitizer for Anthropic Max usage alerts before history persistence and channel delivery, with tests for standalone and embedded alerts while preserving normal answer text.

## Reproduction Assessment

Yes, from source inspection. A synthetic assistant text block containing the reported alert would pass through extraction, message-end finalization, and payload delivery as non-empty assistant text; I did not run a live Anthropic Max OAuth replay.

## Solution Assessment

Yes for the narrow bug. A targeted shared sanitizer in the assistant-visible extraction/delivery path is the maintainable fix; configurable output patterns or quota capture are separate product/config work.
