## Summary

Keep open: current main still lacks a provider usage-alert strip/capture path, and the assistant text pipeline can treat ordinary provider text as deliverable and persistable output. The remaining fix needs maintainer product/security direction on discard vs private quota capture.

## What This Changes

The issue requests suppressing Anthropic Max usage/billing alert text blocks before session transcript storage and external channel delivery.

## Best Possible Solution

Add a central provider notice classifier at the assistant text/message-write boundary, suppress it from user-visible output, and store any retained quota data as private metadata rather than assistant transcript text.

## Reproduction Assessment

No live high-confidence reproduction was run. Source-level reproduction is reasonably clear: current generic assistant sanitation and delivery paths do not contain a usage-alert filter and would preserve ordinary non-error assistant text.

## Solution Assessment

No for the regex-only proposal. The best fix is a central provider notice classifier with tested suppression and optional private quota capture, not a brittle channel-only or hardcoded discard path.
