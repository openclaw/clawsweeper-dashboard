## Summary

Keep open: current main still lacks an Anthropic Max usage-alert sanitation path, and the source path still treats ordinary Anthropic text blocks as assistant text that can reach transcript and channel delivery. The remaining blocker is maintainer product/security direction on strip-only versus private quota capture, plus live Anthropic Max OAuth proof of the exact payload.

## What This Changes

The issue asks OpenClaw to remove Anthropic Max billing/quota alert text from assistant transcripts and channel replies.

## Best Possible Solution

Add provider-owned Anthropic Max usage-alert sanitation before transcript persistence and channel delivery; if quota values are retained, store them in private provider/session usage metadata rather than assistant text.

## Reproduction Assessment

No: there is not a high-confidence live reproduction against current main. Source inspection does show a plausible path from Anthropic text blocks into assistant transcript and channel delivery without provider metadata filtering.

## Solution Assessment

Unclear: stripping the text from public output is the right direction, but the suggested generic outputStripPatterns config is not clearly the best fit under the repo's high config-surface bar. A provider-aware strip-and-private-capture path is the more maintainable option if maintainers want quota data.
