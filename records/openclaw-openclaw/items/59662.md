## Summary

Keep open: current main and the latest release still lack a Claude Max usage-alert sanitation path, and the current source path continues to treat Anthropic text deltas as normal assistant text that can reach channel delivery. The remaining blocker is maintainer product/security direction on strip-only versus private quota capture, plus live Anthropic Max OAuth proof of the exact payload.

## What This Changes

The issue asks OpenClaw to remove Anthropic Claude Max usage/billing alert text blocks from assistant transcript state and channel delivery.

## Best Possible Solution

Add provider-owned sanitation before assistant text becomes transcript or channel output, and only capture quota metadata privately through the existing Anthropic usage path if maintainers approve that product shape.

## Reproduction Assessment

No high-confidence live reproduction is available yet: the exact Anthropic Max OAuth payload was not exercised in this review. The source path is reproducible with synthetic Anthropic text deltas because current main routes ordinary text deltas into assistant text and block replies unless another phase/reasoning marker suppresses them.

## Solution Assessment

No: a hardcoded regex alone is only a plausible mitigation, not clearly the best fix. The maintainable path is a provider-owned metadata filter, with an explicit maintainer decision on whether quota values should be captured privately rather than discarded.
