## Summary

Keep open. Current main still lacks an Anthropic Max usage-alert sanitizer, and the Anthropic stream, assistant text extraction, message-end delivery, transcript mirror, and final payload paths still treat matching non-empty text as assistant-visible output.

## What This Changes

The issue requests stripping Anthropic Claude Max usage/billing alert text blocks from assistant transcripts and channel replies.

## Best Possible Solution

Strip the Claude Max alert at the Anthropic OAuth transport or shared assistant-output normalization boundary before it becomes persisted or delivered assistant text, with regression tests for standalone and embedded alerts.

## Reproduction Assessment

Yes, from source inspection. A synthetic assistant text block containing the reported alert would pass through the current Anthropic/text extraction/message-end/final-payload path as non-empty assistant text; I did not run a live Anthropic OAuth replay.

## Solution Assessment

Yes. A narrow provider-alert sanitizer before persistence and delivery is the maintainable fix; adding a general messages.outputStripPatterns config would be a broader product/config decision.
