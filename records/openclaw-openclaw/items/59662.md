Keep open. Current main still has no Claude Max usage-alert sanitizer, and the assistant text extraction, message-end delivery, transcript assistantTexts path, and final reply payload path continue to treat ordinary non-empty text blocks as user-visible assistant output.

## What This Changes

The issue asks OpenClaw to strip Anthropic Claude Max usage/billing alert text blocks from assistant output before transcript persistence and channel delivery.

## Best Possible Solution

Add a narrow shared assistant-output sanitizer for the Claude Max usage alert block, with regression tests for standalone and embedded alerts while preserving real answer text.

## Solution Assessment

Yes. A targeted sanitizer in the shared assistant-visible extraction and delivery path is the narrowest maintainable fix; configurable output regex stripping would be a broader product/config decision.
