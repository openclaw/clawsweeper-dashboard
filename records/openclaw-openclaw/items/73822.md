## Summary

Keep open. This PR addresses a real SecretRef phone-number gap that still exists on current main, but the current head is not merge-ready because it lets any SecretRef satisfy the public-open DM wildcard guard.

## What This Changes

The PR widens messaging and voice-call phone-number/handle config schemas, secret contracts, manifests, UI hints, docs, changelog, and tests so SecretRef values can be used in allowlists and default send targets.

## Best Possible Solution

Land a corrected SecretRef implementation that keeps dmPolicy="open" gated by a literal "*" while allowing SecretRefs for ordinary allowlists and resolved sender/target fields.

## Reproduction Assessment

Yes. PR head source inspection shows dmPolicy="open" with allowFrom containing any SecretRef returns without an issue, and the new test explicitly expects that SecretRef-only open policy to pass.

## Solution Assessment

No. The feature direction is useful, but the safest implementation preserves the auditable literal wildcard requirement for public-open configs and only lets SecretRefs satisfy non-open allowlist entries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require a literal wildcard for open DM policy:** `src/config/zod-schema.core.ts:640`
  - body: This makes dmPolicy="open" pass when allowFrom contains any SecretRef object, so a ref can resolve to "*" and widen DM access without a visible wildcard in reviewed config. Keep SecretRefs valid for allowlist entries, but require a literal "*" for the open-policy opt-in.
