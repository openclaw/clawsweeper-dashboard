## Summary

Keep open. The SecretRef feature is useful and current main still lacks the requested phone-number coverage, but the PR is not safe to merge because it lets any SecretRef satisfy the visible wildcard guard for public-open DM policy.

## What This Changes

The branch adds SecretRef-aware schemas, resolution contracts, UI hints, docs, changelog, and tests for phone-number and sender fields across WhatsApp, Signal, iMessage, BlueBubbles, and voice-call.

## Best Possible Solution

Land the SecretRef feature after preserving the visible wildcard guard: SecretRefs may count as configured allowlist entries, but dmPolicy="open" should still require a literal "*".

## Reproduction Assessment

Yes. A schema-level check of requireOpenAllowFrom with policy="open" and allowFrom=[SecretRef] reproduces the regression; the PR's added test currently expects that unsafe acceptance.

## Solution Assessment

No. The feature direction is maintainable, but the open-policy validation should be narrower so only allowlist presence can be SecretRef-backed while public-open remains an explicit reviewed opt-in.
