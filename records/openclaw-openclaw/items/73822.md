## Summary

Keep open. Current main still lacks the requested SecretRef phone-number support, and this PR is the active implementation for #56502, but latest head remains blocked because it lets any SecretRef in `allowFrom` satisfy the public-open DM wildcard guard.

## What This Changes

The branch adds SecretRef support for phone-number and sender fields across WhatsApp, Signal, iMessage, BlueBubbles, and voice-call, with schema, secret-resolution, metadata, docs, UI hint, changelog, and test updates.

## Best Possible Solution

Land the SecretRef feature with the public-open contract preserved: SecretRefs may satisfy non-empty allowlists, but `dmPolicy="open"` should still require a literal `"*"` before the linked feature is merged.

## Reproduction Assessment

Yes. The PR diff and added test path show a SecretRef-only `allowFrom` is accepted for `dmPolicy="open"`; current main code and docs provide the contrasting expected behavior that only a literal `"*"` should pass.

## Solution Assessment

No. The feature direction is maintainable, but the open-policy validation should be narrower so only allowlist presence can be SecretRef-backed while public-open remains an explicit visible opt-in.
