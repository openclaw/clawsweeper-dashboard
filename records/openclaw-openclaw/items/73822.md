## Summary

Keep open: the feature addresses a real privacy gap, but the PR head still weakens the visible public-open access-control guard and regresses the voice-call CLI gateway delegation path.

## What This Changes

The branch adds SecretRef support for phone-number and sender fields across WhatsApp, Signal, iMessage, BlueBubbles, and the voice-call plugin, with schemas, secret contracts, UI hints, docs, changelog, and tests.

## Best Possible Solution

Land the SecretRef feature after preserving public-open's visible wildcard guard and retaining gateway-first voice-call CLI behavior, while still allowing SecretRefs for resolved allowlist and sender values.

## Reproduction Assessment

Yes. A schema-level call with policy="open" and allowFrom=[SecretRef] passes on PR head and is locked by the added test; the voicecall call regression is visible in the PR head source by comparing it with current main's gateway helper path.

## Solution Assessment

No. The feature direction is maintainable, but public-open should require a literal "*" in config and the CLI should normalize SecretRef-backed fallback values without bypassing gateway delegation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require a literal wildcard for open DM policy:** `src/config/zod-schema.core.ts:636`
  - body: This treats any SecretRef in `allowFrom` as satisfying `dmPolicy="open"`. That moves the public-open opt-in out of reviewed config and into external secret material, contradicting the current validation/docs contract that open requires a visible `"*"`.
  - confidence: 0.92
- **[P2] Keep voicecall call on the gateway delegation path:** `extensions/voice-call/src/cli.ts:518-528`
  - body: The `call` command used to invoke `initiateCallViaGatewayOrRuntime`, so a running gateway owned call initiation and the CLI only fell back locally when unavailable. This block now always creates a local runtime while `start` still delegates, bypassing the active gateway/secrets snapshot for the main command.
  - confidence: 0.84
