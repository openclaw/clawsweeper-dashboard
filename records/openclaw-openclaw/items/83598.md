- `impact:auth-provider`: The affected path is provider auth profile resolution, OAuth expiry handling, and `anthropic:claude-cli` credential routing.

## Summary

Keep open: current main still preserves a file-only no-prompt Claude CLI credential path, while this report is specifically about Keychain-only `anthropic:claude-cli` credentials failing in the main runtime lane after the persisted profile expires.

## What This Changes

The issue reports that expired `anthropic:claude-cli` OAuth profiles can dead-end main-lane Anthropic requests on macOS even when Claude CLI Keychain credentials are fresh.

## Best Possible Solution

Define the supported no-prompt Keychain policy, then make expired managed Claude CLI profiles adopt or persist fresh already-approved Keychain credentials in the main runtime lane with clear skip diagnostics and regression coverage.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: an expired managed `anthropic:claude-cli` profile plus Keychain-only fresh credentials enters provider auth with no-prompt file-only discovery and can reach the explicit-profile throw. I did not run the live macOS Keychain flow here.

## Solution Assessment

Unclear until maintainers choose the no-prompt Keychain policy. Reusing or persisting already-approved fresh Keychain credentials for expired managed profiles looks narrower than loosening all background Keychain prompting.
