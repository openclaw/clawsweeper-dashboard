## Summary

Keep this open: current main and v2026.6.8 improved the fresh external-credential overlay path for `anthropic:claude-cli`, but the Linux/container follow-up still appears source-plausible because OpenClaw only reads Claude CLI stored credentials and does not invoke Claude CLI refresh when the external file access token is already expired.

## What This Changes

The issue reports that expired `anthropic:claude-cli` OAuth can dead-end main agent traffic instead of adopting or refreshing fresh Claude CLI credentials.

## Best Possible Solution

Keep the shipped scoped-overlay behavior, then have maintainers decide and prove the Claude CLI refresh policy for expired external file/keychain credentials before closing this report.

## Reproduction Assessment

Yes, at source level: the fresh external overlay path is covered, but the commented Linux/file path remains plausible because imported expired credentials are not bootstrapped. No live expired Claude CLI run was performed in this read-only review.

## Solution Assessment

No, not enough to close as implemented. Current main appears to solve the fresh-external-credential subset, but the best remaining solution needs maintainer policy and live proof for triggering or handling Claude CLI refresh when the external source itself has expired access.
