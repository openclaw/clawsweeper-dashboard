## Label Justifications

- `P1`: The report describes a current-release auth regression that can break all main-lane Anthropic traffic for Claude CLI OAuth users after token expiry.
- `impact:auth-provider`: The failure is in provider auth profile resolution, OAuth expiry handling, and `anthropic:claude-cli` credential routing.

## Summary

Keep open: current `main` and the latest release still preserve the reported no-prompt Claude CLI Keychain gap, so an expired `anthropic:claude-cli` profile can plausibly reach the same explicit-profile failure path without adopting the fresh Keychain credential.

## What This Changes

The issue asks OpenClaw to fix and diagnose a macOS Claude CLI OAuth refresh path where expired `anthropic:claude-cli` profiles can block main-lane Anthropic traffic despite fresh Keychain credentials.

## Best Possible Solution

Define the no-prompt Keychain reuse policy, then make expired managed Claude CLI profiles adopt or persist fresh external CLI credentials without prompting, with info-level diagnostics when bootstrap is skipped before an explicit-profile failure.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: a Keychain-only macOS setup with an expired `anthropic:claude-cli` profile enters provider auth with `allowKeychainPrompt: false`, cannot use Keychain-backed credentials, and can reach the explicit-profile throw path. I did not run the live macOS Keychain flow in this Linux checkout.

## Solution Assessment

Unclear until maintainers choose the no-prompt Keychain policy; reusing already-read fresh Keychain credentials looks narrower than allowing background prompts. The fix should add regression coverage for expired local profile adoption and the main-lane explicit-profile failure.
