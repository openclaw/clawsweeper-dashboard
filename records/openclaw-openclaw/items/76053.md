## Summary

Keep open. The PR addresses a real exec/session-context gap on current main, but it should not merge as-is because the new runtime markers can preserve stale inherited or caller-supplied session values when the current value is unavailable.

## What This Changes

The branch adds exec defaults for session id/key propagation, injects `OPENCLAW_SESSION_KEY` and `OPENCLAW_SESSION_ID` into local exec environments, documents them, adds a focused gateway env test, and updates the changelog.

## Best Possible Solution

Treat `OPENCLAW_SESSION_KEY` and `OPENCLAW_SESSION_ID` as reserved runtime markers: strip inherited/requested values before injection, then set only the current normalized values, with regression tests for missing `sessionId` and stale inherited markers.

## Reproduction Assessment

Yes. On the proposed patch, an exec run with `opts.env.OPENCLAW_SESSION_ID` set and no current `defaults.sessionId` will preserve that stale value because `shellRuntimeEnv` spreads `opts.env` before conditionally adding the runtime marker.

## Solution Assessment

No. The feature direction is narrow and useful, but the safest implementation must scrub these reserved marker names before adding current session context so spawned commands cannot see stale or spoofed context.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Scrub stale session markers before spawning exec:** `src/agents/bash-tools.exec-runtime.ts:570-571`
  - body: Because `shellRuntimeEnv` starts from `...opts.env` and only conditionally overwrites `OPENCLAW_SESSION_KEY` / `OPENCLAW_SESSION_ID`, any inherited process env or tool `env` value survives when the current key or id is absent. That lets commands observe a stale or spoofed session even though the docs describe these as runtime-injected values; remove those keys before adding the normalized current values and cover the missing-`sessionId` case.
  - confidence: 0.89

## Security Review

Status: needs_attention

Summary: The diff introduces a low-severity session-context integrity/privacy risk until reserved session env markers are scrubbed before re-injection.

Concerns:

- **[low] Stale session context can leak through env inheritance:** `src/agents/bash-tools.exec-runtime.ts:570`
  - body: The new env markers may include channel/thread routing identifiers. Since the patch conditionally adds them over the inherited exec env, a stale parent or user-provided `OPENCLAW_SESSION_KEY` / `OPENCLAW_SESSION_ID` can reach spawned commands when the current value is absent, misrepresenting the active session.
