## Summary

Keep open. The PR addresses a real gap on current main, but the proposed env injection is not merge-ready because reserved session markers can survive from inherited or requested env when the current session value is absent.

## What This Changes

The PR propagates agent session key/id defaults into local exec child environments and updates exec docs, environment docs, changelog, and focused gateway env coverage.

## Best Possible Solution

Treat `OPENCLAW_SESSION_KEY` and `OPENCLAW_SESSION_ID` as reserved runtime markers: remove inherited/requested values first, then add only normalized current values with regression coverage for absent and overriding cases.

## Reproduction Assessment

Yes. On the proposed patch, run exec with `opts.env.OPENCLAW_SESSION_ID` set and no current `defaults.sessionId`; the spread-before-conditional-add logic preserves that stale value.

## Solution Assessment

No. The feature direction is narrow and useful, but the implementation should scrub reserved marker names before setting current session context so the documented runtime-injected values cannot be spoofed or stale.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Scrub stale session markers before spawning exec:** `src/agents/bash-tools.exec-runtime.ts:570-571`
  - body: Because `shellRuntimeEnv` starts from `...opts.env` and only conditionally overwrites `OPENCLAW_SESSION_KEY` / `OPENCLAW_SESSION_ID`, any inherited process env or tool `env` value survives when the current key or id is absent. That lets commands observe a stale or spoofed session even though the docs describe these as runtime-injected values; remove those keys before adding the normalized current values and cover the missing-`sessionId` case.
  - confidence: 0.89

## Security Review

Status: needs_attention

Summary: The diff introduces a low-severity session-context integrity/privacy risk until reserved session env markers are scrubbed before re-injection.

Concerns:

Since the patch conditionally adds them over the inherited exec env, a stale parent or user-provided `OPENCLAW_SESSION_KEY` / `OPENCLAW_SESSION_ID` can reach spawned commands when the current value is absent, misrepresenting the active session.
