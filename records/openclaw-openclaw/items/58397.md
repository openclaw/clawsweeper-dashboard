## Summary

Keep open: the zero-timeout `sessions_send` announce fix is still relevant on current main, but this PR head is not mergeable because it makes the A2A helper require `announcePlan` while existing direct calls still omit it. The branch also has only unit-test output, so the real behavior proof gate remains unmet.

## What This Changes

The PR changes `sessions_send` announce planning so zero-timeout sends make a synchronous fail-closed routing decision, normalizes announce-target metadata, and splits/restores sessions and subagents test coverage.

## Best Possible Solution

Port the zero-timeout fail-closed announce-planning fix onto current main with helper compatibility, focused regressions, preserved current A2A safeguards, and redacted real OpenClaw or Telegram proof.

## Reproduction Assessment

Yes for source-level reproduction: current main's `timeoutSeconds: 0` path still starts A2A, and A2A still reaches the `sessions.list`-capable announce resolver despite docs saying fire-and-forget should return immediately. I did not run live E2E in this read-only review.

## Solution Assessment

No, not as currently proposed. The direction is narrow and relevant, but the patch must keep `runSessionsSendA2AFlow()` compatible with existing direct calls or update all callers, then add real behavior proof before merge.
