## Summary

Current main now covers the central isolated-cron fan-out/fan-in failure: it tracks run-scoped descendants, waits for all active child runs to drain, allows a post-completion synthesis turn, and falls back to combined child results instead of delivering the stale interim acknowledgement. This implementation landed after the reporter’s tested releases and is not in v2026.6.5.

## What This Changes

The issue requests a reliable fan-out/fan-in mechanism for isolated cron sessions to spawn subagents, await all results, and continue aggregation.

## Best Possible Solution

Retain the current run-scoped descendant drain and synthesis path, and document the supported isolated-cron orchestration behavior when it ships rather than adding a separate blocking sessions_await API.

## Reproduction Assessment

No on current main: source and focused tests show isolated cron delivery now waits for all run-scoped descendants and captures either the parent synthesis or combined child output. The report remains credible for versions 2026.3.13 through v2026.6.5 because the fix is newer and unreleased.

## Solution Assessment

Yes; integrating fan-in with the existing cron delivery and subagent registry lifecycle is narrower and avoids a parallel blocking tool or new configuration surface.
