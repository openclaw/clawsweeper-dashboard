## Summary

Current main has descendant-aware delivery and yielded-run lifecycle support, but no deterministic wait-for-all result or guaranteed parent synthesis turn. The issue remains necessary as a core product/design item rather than an implemented bug fix.

## What This Changes

Add a reliable way for isolated cron orchestrators to spawn parallel subagents, await their completion, and synthesize their results before delivery.

## Best Possible Solution

Design one core cohort primitive that returns structured child outcomes and owns wait-all, partial, timeout, and cancellation semantics, then connect parent continuation, cron terminal status, and final delivery to that same lifecycle outcome.

## Reproduction Assessment

Yes at the source-contract level: current main drains descendant runs and may discover a later synthesis, but it has no deterministic cohort barrier, structured combined result, or guaranteed parent aggregation turn. This read-only review did not run a live cron scenario.

## Solution Assessment

The requested capability is valid, but the three suggested implementations are not interchangeable; a lifecycle-owned cohort primitive is safer than extending transcript polling or making `sessions_yield` implicitly own cron orchestration.
