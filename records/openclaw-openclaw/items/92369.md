## Summary

Keep open: current main has an isolated-cron descendant drain and fallback delivery path, but it still does not provide the model-visible barrier/await primitive or documented unsupported stance requested here. The remaining decision is product/API shape, not a safe cleanup close.

## What This Changes

Requests reliable fan-out/fan-in for cron isolated subagent workflows so an orchestrator can spawn children, wait for all results, and aggregate them or be told the pattern is unsupported.

## Best Possible Solution

Choose one explicit maintainer-approved contract for cron isolated fan-in: a model-visible barrier/await surface, a durable cron yield continuation, or clear docs that this orchestration pattern is unsupported, while preserving the current descendant-drain fallback.

## Reproduction Assessment

Yes, as a source-level gap: current main still has no sessions_await/barrier/cohort tool surface, and sessions_yield does not return child results or await an explicit set of runs. I did not run a live cron repro because the remaining gap is visible in source and docs.

## Solution Assessment

Unclear. The current cron descendant drain is useful partial coverage, but the issue lists three different product shapes; maintainers should choose the supported contract before implementation.
