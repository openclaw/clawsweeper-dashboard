## Summary

Keep open: current main has a delivery-side descendant-drain mitigation for isolated cron runs, but it still does not provide the agent-visible spawn/wait/aggregate primitive or documented unsupported boundary requested here.

## What This Changes

The issue requests reliable cron-isolated subagent fan-out/fan-in orchestration so a parent can spawn workers, wait for all results, and aggregate them before delivery.

## Best Possible Solution

Define a narrow core-owned cron-safe fan-out/fan-in contract, or explicitly document unsupported orchestration, and evaluate https://github.com/openclaw/openclaw/pull/64281 against that contract before adding another fix PR.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-level, yes for the missing primitive: current main has non-blocking spawn, yield-as-end-turn behavior, and delivery fallback, but no agent-visible await/barrier tool; I did not run a live cron repro.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the product contract; delivery fallback is useful for simple announce cases, but the requested reliable fan-out/fan-in workflow needs a deliberate API or an explicit docs boundary.
