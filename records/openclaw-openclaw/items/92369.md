## Summary

Keep open: current main has cron-side descendant delivery mitigations, but it still lacks a model-facing fan-out/fan-in primitive or a supported cron-isolated lifecycle that lets an orchestrator spawn multiple subagents, wait for all required completions, and aggregate within the original workflow.

## What This Changes

Request a reliable cron-isolated subagent fan-out/fan-in path that can spawn multiple children, wait for all required completions, and deliver an aggregate result.

## Best Possible Solution

Keep the issue open until maintainers choose and document the supported outcome: a first-class fan-in primitive, a changed cron-yield lifecycle, or an explicit unsupported limitation with safer guidance.

## Reproduction Assessment

Yes, source-reproducible: current main has no sessions_await/barrier implementation, docs still describe non-blocking spawn plus sessions_yield, and tests assert a yielded parent run is no longer active; I did not run a live cron E2E.

## Solution Assessment

Unclear: the repository needs a maintainer product choice between adding a model-facing barrier, changing cron yield lifecycle semantics, or documenting cron-isolated fan-in as unsupported.
