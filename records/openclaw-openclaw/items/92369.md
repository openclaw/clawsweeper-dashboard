## Summary

This should remain open as the canonical report: current main marks `sessions_yield` attempts as paused, but isolated cron still completes and disposes its owned run context without consuming that paused state, so reliable multi-child aggregation is not implemented.

## What This Changes

Provide reliable fan-out/fan-in subagent orchestration in isolated cron sessions so the parent can await multiple children and synthesize one final result.

## Best Possible Solution

Keep this as the canonical lifecycle issue and make isolated cron retain a durable parent continuation until every declared child reaches a terminal outcome, then resume that parent for one aggregation stage and finalize cron delivery only afterward.

## Reproduction Assessment

Yes, at the source level with high confidence: `sessions_yield` marks the embedded attempt paused, but isolated cron does not consume that state before processing the outcome and disposing its owned run context.

## Solution Assessment

The issue is the right canonical report, but the final implementation needs a maintainer decision; repairing lifecycle-owned continuation is preferable to polling, and a new public barrier should be added only if the existing lifecycle cannot express reliable cohort completion.
