## Summary

Keep open for maintainer review: the ReDoS hardening is still needed because current main and the latest release still compile plugin manifest `modelPatterns` with raw `RegExp`; the patch is narrow and appears correct, with a compatibility decision remaining before merge.

## What This Changes

This PR routes plugin manifest `modelPatterns` through `compileSafeRegex`, adds a ReDoS regression test, and documents that unsafe patterns are skipped.

## Best Possible Solution

Merge the hardening with the docs and regression coverage after maintainers accept unsafe `modelPatterns` as invalid manifest patterns; add a diagnostic first if silent skipping is too opaque for upgrades.

## Reproduction Assessment

Yes. Current main source compiles manifest `modelPatterns` with raw `new RegExp`, and the PR body supplies terminal before/after proof showing `(a+)+$` timing out with raw regex and being rejected by `compileSafeRegex`.

## Solution Assessment

Yes, with a maintainer compatibility decision. Reusing the existing safe-regex helper at the resolver boundary is the narrowest maintainable fix; a diagnostic can be added first if maintainers do not want silent skipping.
