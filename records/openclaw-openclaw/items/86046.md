## Summary

Keep this PR open for maintainer merge review: the patch is a focused security hardening for a source-reproducible ReDoS path and is not obsolete on current main. The remaining decision is compatibility, because existing plugin `modelPatterns` with nested repetition will silently stop matching instead of executing.

## What This Changes

The PR changes plugin provider model pattern matching to compile manifest regex sources through the existing safe-regex guard, adds regression coverage, and documents the manifest behavior.

## Best Possible Solution

Land the safe-regex guard if maintainers accept the documented fail-closed manifest contract; add a manifest diagnostic first only if silent skip is too hard to support.

## Reproduction Assessment

Yes: current main clearly constructs a raw `RegExp` from plugin `modelPatterns`, and the PR proof shows `(a+)+$` can hang while `compileSafeRegex` rejects it. I did not run tests because this review was read-only.

## Solution Assessment

Yes: reusing the existing safe-regex helper is the narrowest maintainable fix for this regex surface. The only open policy choice is whether silent skip is acceptable or should be paired with a diagnostic.
