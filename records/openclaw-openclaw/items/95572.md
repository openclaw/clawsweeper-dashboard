The PR updates agent binding parsing to reject extra colon-delimited account segments and adds Matrix regression coverage for the malformed and valid bind-spec paths.

## Best Possible Solution

Let the exact-head automerge path or maintainer review merge this shared parser rejection after required checks are green; current main and `v2026.6.9` should not be treated as already fixed.

## Solution Assessment

Yes. Rejecting extra segments in shared `parseBindingSpecs` is the narrowest maintainable fix because `agents bind`, `agents unbind`, and `agents add --bind` all consume that parser before config writes.
