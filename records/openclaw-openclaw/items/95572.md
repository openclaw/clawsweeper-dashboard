## Summary

## What This Changes

The PR changes agent binding parsing to reject extra colon segments in `--bind` specs and adds Matrix parser regression coverage.

## Best Possible Solution

Land the shared parser rejection and regression tests once maintainers accept the malformed-spec fail-closed behavior and exact-head gates pass.

## Reproduction Assessment

Yes. Current main and `v2026.6.9` both show `parseBindingSpecs` using `split(":", 2)`, which truncates `matrix:work:extra` to account `work`; the PR body also includes before/after terminal output from a harness importing the real parser.

## Solution Assessment

Yes. The shared `parseBindingSpecs` function is the right owner because `agents bind`, `agents unbind`, and non-interactive `agents add --bind` all consume it before config writes; validating later would duplicate command logic or rely on account-id normalization that canonicalizes invalid characters.
