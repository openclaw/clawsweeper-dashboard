## What This Changes

The PR changes `parseBindingSpecs` to reject `<channel>:<account>:...` bind specs and adds Matrix parser tests for the malformed and valid cases.

## Best Possible Solution

Land the parser-level rejection and regression tests after required checks pass; keep release-note context in the PR body or squash message rather than editing `CHANGELOG.md`.

## Reproduction Assessment

## Solution Assessment

Yes. Rejecting extra colon segments in the shared `parseBindingSpecs` function is the narrowest maintainable fix because `agents bind`, `agents unbind`, and `agents add --bind` all pass through that parser before config writes.
