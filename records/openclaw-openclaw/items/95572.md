## What This Changes

The PR changes agent binding parsing to reject bind specs with extra colon-delimited account segments and adds Matrix parser regression coverage.

## Best Possible Solution

Merge the shared parser rejection and regression tests once maintainers accept the malformed-spec fail-closed behavior and exact-head gates are satisfied.

## Reproduction Assessment

## Solution Assessment

Yes. The shared `parseBindingSpecs` function is the right owner because `agents bind`, `agents unbind`, and `agents add --bind` all consume it before config writes; validating later would duplicate command logic or rely on account-id normalization that intentionally canonicalizes invalid characters.
