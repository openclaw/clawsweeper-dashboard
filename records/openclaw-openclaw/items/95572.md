## Summary

Keep open for normal PR review/merge: current main still truncates extra colon segments in agent binding specs, while the documented bind format and account-id contract make a third segment malformed. The PR is a narrow parser-boundary fix with targeted regression coverage and no blocking findings from this read-only review.

## What This Changes

The PR changes agent bind spec parsing to reject `<channel>:<account>:...` values and adds Matrix parser regression coverage for the malformed and valid cases.

## Best Possible Solution

Land the parse-time rejection with the regression test once required CI passes.

## Reproduction Assessment

Yes. Current main source shows the truncating `split(":", 2)` path, and the PR body provides before/after parser output for `matrix:work:extra`.

## Solution Assessment

Yes. Rejecting extra segments in `parseBindingSpecs` is the narrowest maintainable fix because both `agents bind` and `agents add --bind` share that parser before config writes.
