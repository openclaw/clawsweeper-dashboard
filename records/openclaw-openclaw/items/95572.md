## What This Changes

The PR changes agent binding spec parsing to reject extra colon-delimited account segments and adds Matrix parser regression coverage.

## Best Possible Solution

Merge this narrow parser rejection after exact-head CI/automerge gates pass, preserving the documented one-colon bind-spec contract.

## Solution Assessment

Yes. Rejecting extra segments in parseBindingSpecs is the narrowest maintainable fix because agents bind, agents unbind, and agents add --bind all use that parser before route-binding writes.
