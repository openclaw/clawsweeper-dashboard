## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Track channel context registrations in plugin cleanup:** `src/plugins/registry.ts:2790-2812`
  - body: These handlers write into new process-global maps, but the loader only clears/restores the existing global plugin registries during activation, snapshot loads, cache restore, and rollback. A reload or non-activating load can leave stale builders/observers live after the plugin is disabled, so Discord may still inject or record channel text from an inactive plugin. Add clear/list/restore lifecycle for this registry or resolve registrations from the active plugin registry instead.
  - confidence: 0.88
- **[P2] Gate recovered context to fresh Discord sessions:** `extensions/discord/src/monitor/message-handler.context.ts:196-220`
  - body: The new hook resolves supplements for every guild-channel message with a guild id, even when the turn is part of an already-active session. Once enabled, the previous turn stored by the observer can be prepended into the next active-session turn even though the model transcript already carries that conversation. Gate this to the fresh-session/reset path the docs describe, and cover the active-session no-duplicate case.
  - confidence: 0.86
