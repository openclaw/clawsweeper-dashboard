## Summary

Keep open: current main partially addressed the small fallback by moving custom-provider defaults to 128K, but it still does not surface or collect context-window/max-token metadata during custom-provider setup, and the dedicated implementation candidate remains open.

## What This Changes

The issue asks OpenClaw to make custom and OpenAI-compatible provider context-window behavior visible, configurable, or auto-detected instead of silently relying on a fallback.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/43493 with a guarded custom-provider metadata prompt/visibility path, keep the 128K fallback, and split auto-detection or runtime overflow wording into follow-ups if needed.

## Reproduction Assessment

Yes. Source inspection on current main shows the old small fallback has been raised to 128K, but custom-provider setup still saves defaults without collecting or displaying contextWindow/maxTokens metadata.

## Solution Assessment

Yes for the near-term direction: setup-time metadata visibility is the narrowest maintainable fix. Raising the fallback alone has helped but does not tell users whether the effective value was inferred, defaulted, or manually configured.
