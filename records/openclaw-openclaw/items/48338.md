## Summary

Keep this issue open. Current main still gives non-Azure custom/OpenAI-compatible provider models a small implicit context window, now 4,000 tokens via the hard-min constant rather than the 16K value named in the report, and setup still does not collect or display contextWindow/maxTokens metadata or fallback warnings. The open #43493 PR remains the clearest implementation candidate, but it is not on main.

## What This Changes

Request custom and OpenAI-compatible provider setup to expose, configure, or detect effective context-window metadata instead of silently using small fallback defaults.

## Best Possible Solution

Review and land #43493 or an equivalent custom-provider setup change that lets users see and edit effective contextWindow/maxTokens during setup, warns when fallback metadata is used, updates non-interactive/docs/tests, and scopes auto-detection or runtime overflow copy as explicit follow-up if not included.

## Reproduction Assessment

Yes. Current-main source inspection gives a high-confidence reproduction path: configure a non-Azure custom provider and the code path writes the hard-min context window and 4096 maxTokens without prompting for those values; the colocated test asserts that behavior.

## Solution Assessment

Yes for the near-term direction: the narrowest maintainable fix belongs in the custom-provider setup/configuration surface, not in a generic core fallback change alone. Raising defaults without visibility would still leave users unable to tell whether metadata was detected, defaulted, or manually set.
