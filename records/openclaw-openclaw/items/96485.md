## Summary

Keep open: current main and the latest release still contain the Anthropic OAuth callback bind-host versus redirect-URI mismatch, and the protected security label requires maintainer security handling rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to harden Anthropic OAuth callback handling so the callback bind host and OAuth redirect URI cannot diverge or accept non-loopback hosts.

## Best Possible Solution

Align Anthropic OAuth with the existing OpenAI loopback-host pattern: allow only localhost, 127.0.0.1, or ::1, derive the redirect URI from that host, and add focused regression coverage after maintainer security triage.

## Reproduction Assessment

Yes. Source inspection on current main shows CALLBACK_HOST can diverge from the fixed localhost REDIRECT_URI; I did not run a live Anthropic OAuth flow because it requires an external provider login.

## Solution Assessment

Yes, if narrowed to callback consistency and loopback hardening. The existing OpenAI OAuth helper is the best in-repo pattern, while arbitrary remote callback hosts should not become a new supported surface.
