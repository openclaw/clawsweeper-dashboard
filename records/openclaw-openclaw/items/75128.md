## Summary

Keep open: this PR addresses a real source-reproducible BOOT.md disclosure path and now includes credible terminal proof, but the changed sanitizer still misses internal-runtime-context blocks encoded with literal `\n` before the send path normalizes them.

## What This Changes

The PR wraps BOOT.md content in internal-runtime-context delimiters, adds message-tool runtime-context and boot-echo sanitization, updates regression tests, and adds a changelog entry for https://github.com/openclaw/openclaw/issues/53732.

## Best Possible Solution

Normalize escaped line breaks before runtime-context stripping, or strip again after send-path normalization, while keeping the BOOT.md wrapper and boot-echo guard with focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible. Current main embeds BOOT.md directly, fallback retries preserve the same body, and the message tool is a user-visible delivery path; the PR body now supplies after-fix terminal proof for the patched path.

## Solution Assessment

No. The PR targets the right layers, but sanitizer ordering still leaves an escaped-newline bypass, so the maintainable fix is to normalize or re-strip at the same boundary that sends outbound text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Normalize escaped newlines before stripping context:** `src/agents/tools/message-tool.ts:793`
  - body: The new sanitizer runs before `handleSendAction` turns literal `\n` sequences in `message` into real newlines. Normalize escaped newlines before stripping, or run the same strip again after that conversion.
