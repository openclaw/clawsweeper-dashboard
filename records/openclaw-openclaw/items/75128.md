## Summary

Keep open. The PR is a useful, focused fix path for the fallback BOOT.md disclosure bug, but the latest head still leaves a prompt-disclosure bypass because stringified `presentation` payloads skip the new sanitizer and are parsed later by the outbound runner.

## What This Changes

The PR wraps BOOT.md in internal-runtime-context delimiters, adds a session boot-echo guard, sanitizes message-tool visible send fields, suppresses empty sanitized sends, and adds tests plus a changelog entry.

## Best Possible Solution

Keep the layered BOOT.md/internal-runtime-context approach, but sanitize parsed presentation payloads before delivery or parse presentation strings before this sanitizer, with focused regression coverage for stringified presentation input.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a `message(action="send")` call can pass `presentation` as a JSON string containing internal-runtime-context text, the new sanitizer returns it unchanged, and `runMessageAction` parses it before dispatch.

## Solution Assessment

No. The proposed direction is narrow and maintainable, but the current patch is incomplete until stringified presentation payloads are sanitized or rejected before they can become outbound presentation content.
