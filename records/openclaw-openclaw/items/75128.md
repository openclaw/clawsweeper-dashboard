## Summary

Keep open. The PR addresses a real prompt-disclosure path and the wrapper/message-tool sanitization layers fit the existing runtime-context pattern, but the newly added BOOT.md echo guard still misses exact markerless echoes at unaligned offsets, so the patch is not merge-ready.

## What This Changes

The PR wraps BOOT.md in internal-runtime-context delimiters, strips internal runtime context from message-tool text fields, adds a per-session boot echo guard with tests, and updates the changelog for #53732.

## Best Possible Solution

Keep the wrapper and message-tool runtime-context strip, but revise the substantial-echo guard to cover every outbound substring at or above the threshold, or replace it with an equivalent rolling/fingerprint approach. Add an unaligned 80-character slice regression test before merge.

## Reproduction Assessment

Yes. Against the PR head, use an outbound message equal to bootPrompt.slice(1, 81); it is an exact 80-character BOOT prompt substring, but the current 20-character step scan returns false.

## Solution Assessment

No. The two delimiter-based layers are a good fit, but the sampled echo detector is not reliable enough for a prompt-disclosure guard; complete window coverage or a rolling fingerprint is the safer narrow solution.
