## Summary

Keep open. The PR addresses the right prompt-disclosure surface, but its new substantial-echo guard still samples boot-prompt windows every 20 characters, so an exact unaligned 80-character BOOT.md echo can pass through user-visible message delivery.

## What This Changes

The PR wraps BOOT.md prompt content in internal-runtime-context markers, strips those blocks from message-tool text fields, adds a per-session boot echo guard with tests, and adds a changelog entry for #53732.

## Best Possible Solution

Keep the wrapper and message-tool strip layers, but change the substantial-echo guard to cover every outbound substring at or above the threshold, or use an equivalent rolling/fingerprint check, with an unaligned 80-character regression test before merge.

## Reproduction Assessment

Yes. A deterministic source-level check against the PR algorithm shows bootPrompt.slice(1, 81) is an exact 80-character prompt substring while containsSubstantialBootEcho returns false.

## Solution Assessment

No. The delimiter-based layers are a good fit, but the sampled echo detector is not dependable enough for the stated prompt-disclosure guard; complete window coverage or an equivalent rolling approach is the safer narrow fix.
