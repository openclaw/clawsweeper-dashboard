## Summary

Keep open. The PR addresses a valid BOOT.md prompt-disclosure path, but its new substantial-echo guard still misses copied boot-prompt substrings that do not align with the 20-character sampling grid, leaving a concrete bypass before merge.

## What This Changes

The PR wraps BOOT.md content in internal-runtime-context markers, strips that context from message-tool text fields, adds a per-session boot echo guard, and adds changelog plus unit coverage.

## Best Possible Solution

Keep the wrapper and message-tool sanitizer layers, but revise the substantial-echo guard to cover every outbound substring/window at or above the threshold, or replace it with an equivalent rolling-hash/fingerprint approach, with regression coverage for unaligned 80-character slices.

## Reproduction Assessment

Yes. A high-confidence deterministic reproduction is to run the PR guard logic against the PR's long boot prompt fixture and an 80-character slice starting at offset 1; that copied prompt slice is not detected.

## Solution Assessment

No. The direction is right, but sampled-window detection is not complete enough for security-sensitive prompt suppression; a full sliding-window or fingerprint implementation is the safer narrow fix.
