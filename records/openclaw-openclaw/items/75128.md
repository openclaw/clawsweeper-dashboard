## Summary

Keep open. The PR targets a valid prompt-disclosure bug, but the latest head still lets an unaligned exact 80-character BOOT.md prompt substring bypass the new echo guard because the guard samples prompt windows every 20 characters instead of checking all eligible windows.

## What This Changes

The PR wraps BOOT.md content in internal-runtime-context delimiters, strips internal runtime context from message-tool text fields, adds a per-session boot echo guard with tests, and updates the changelog for #53732.

## Best Possible Solution

Keep the delimiter wrapping and message-tool runtime-context strip, but replace the sampled echo scan with complete window coverage or an equivalent rolling/fingerprint implementation and add an unaligned 80-character slice regression test before merge.

## Reproduction Assessment

Yes. At PR head, an outbound message equal to bootPrompt.slice(1, 81) is an exact 80-character substring of the registered boot prompt, but containsSubstantialBootEcho returns false because offset 1 is never scanned.

## Solution Assessment

No. The delimiter-based layers are a good fit, but the sampled echo detector is not reliable enough for prompt-disclosure hardening; scanning every eligible window or using an equivalent rolling match is the safer narrow fix.
