## Summary

Keep open. The PR is aimed at a real prompt-disclosure path, but the new boot echo guard still misses exact unaligned 80-character BOOT.md substrings, so the security hardening is not reliable enough to merge as-is.

## What This Changes

The PR wraps BOOT.md prompt content in internal-runtime-context delimiters, sanitizes message-tool outbound text fields, adds a per-session boot echo guard with tests, and adds a changelog entry for #53732.

## Best Possible Solution

Keep the delimiter wrapper and message-tool runtime-context strip, but revise the substantial-echo guard to cover every outbound substring at or above the threshold, or replace it with an equivalent rolling/fingerprint approach, then add an unaligned 80-character slice regression test before merge.

## Reproduction Assessment

Yes. Against the PR algorithm, an outbound message equal to bootPrompt.slice(1, 81) is an exact 80-character BOOT prompt substring, but the sampled 20-character step scan returns false.

## Solution Assessment

No. The delimiter-based layers are appropriate, but the sampled echo detector is not a dependable prompt-disclosure guard; complete window coverage or an equivalent rolling/fingerprint check is the safer narrow fix.
