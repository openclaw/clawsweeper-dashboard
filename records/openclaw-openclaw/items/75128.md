## Summary

Keep open. The PR addresses a real prompt-disclosure path from #53732, and current main still embeds BOOT.md as ordinary prompt text while message-tool sanitization only strips reasoning tags. However, the PR head's new substantial-echo guard still misses unaligned 80-character BOOT.md excerpts, so the security hardening is not reliable enough to merge as-is.

## What This Changes

The PR wraps BOOT.md boot content in internal-runtime-context delimiters, extends message-tool text-field sanitization, adds a per-session BOOT.md echo guard with tests, and updates the changelog for #53732.

## Best Possible Solution

Keep the wrapper and message-tool runtime-context strip, but make the boot-echo detector cover every outbound substring at or above the threshold, or replace it with an equivalent rolling-hash/fingerprint approach, then add regression coverage for unaligned min-length slices.

## Reproduction Assessment

Yes. Against the PR head, use a boot prompt well over 100 characters and an outbound message equal to bootPrompt.slice(1, 81); the guard only checks offsets 0, 20, 40, etc. plus the tail, so that exact 80-character echo is not detected.

## Solution Assessment

No. The layered direction is sound, but the sampled scan is not the narrowest reliable solution for a prompt-disclosure guard; complete window coverage or an equivalent fingerprint detector is safer.
