## Summary

Keep open: this is a useful, focused security fix with sufficient terminal proof for the covered BOOT.md echo path, but the current patch still leaves a user-visible disclosure bypass through JSON-string `presentation` payloads that are parsed later by the outbound runner.

## What This Changes

The PR wraps BOOT.md content as internal runtime context, adds a session-scoped boot echo guard and message-tool sanitization, and adds focused tests plus a changelog entry.

## Best Possible Solution

Keep the layered BOOT.md wrapper and echo guard, but also sanitize or reject JSON-string presentation payloads before delivery, then land this PR as the canonical fix for https://github.com/openclaw/openclaw/issues/53732.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path on current main, and the PR head still has a source-reproducible bypass where a string `presentation` skips sanitization before `runMessageAction` parses it into visible presentation content.

## Solution Assessment

No. The overall approach is the right narrow fix, but it is incomplete until JSON-string presentations are parsed and sanitized, or rejected, before they can become channel-visible payloads.
