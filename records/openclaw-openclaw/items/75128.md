## Summary

Keep open. Current main still has the reported BOOT.md prompt-disclosure path, and this PR is aimed at the right surface, but the latest head is not merge-ready because the new substantial-echo guard still misses unaligned 80-character verbatim BOOT prompt slices.

## What This Changes

The PR wraps BOOT.md content in internal runtime-context delimiters, applies internal-runtime-context stripping to message-tool text fields, adds a per-session boot-echo guard, expands boot/message-tool tests, and adds a changelog entry for #53732.

## Best Possible Solution

Keep the BOOT.md wrapper and message-tool runtime-context strip, but change the substantial-echo guard to cover every outbound substring at or above the threshold, or replace it with an equivalent rolling/fingerprint implementation. Add an unaligned 80-character slice regression test before maintainer security review and merge.

## Reproduction Assessment

Yes. The PR algorithm returns false for an outbound message equal to bootPrompt.slice(1, 81), even though that is an exact 80-character substring of the registered boot prompt.

## Solution Assessment

No. The delimiter wrapper and message-tool strip are the right narrow primitives, but the sampled echo detector is not reliable enough for a prompt-disclosure guard; complete window coverage or a rolling fingerprint is the safer fix.
