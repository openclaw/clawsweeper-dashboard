## Summary

Keep open. The PR targets a valid BOOT.md prompt-disclosure bug, but the newly added substantial-echo guard samples the boot prompt every 20 characters and can miss an unaligned exact 80-character echo.

## What This Changes

The PR wraps BOOT.md in internal-runtime-context markers, extends message-tool text sanitization, adds a per-session boot echo guard with tests, and adds a changelog entry for #53732.

## Best Possible Solution

Keep the wrapper and message-tool sanitizer layers, but make containsSubstantialBootEcho cover every 80+ character window or an equivalent rolling/fingerprint match, with an unaligned-window regression test before merge.

## Reproduction Assessment

Yes. A deterministic source-level reproduction using the PR algorithm shows bootPrompt.slice(1, 81) is an exact 80-character boot-prompt substring while containsSubstantialBootEcho returns false.

## Solution Assessment

No. The delimiter wrapping and message-tool stripping are the right layers, but the sampled echo detector is not complete enough for the prompt-disclosure guard it claims to provide.
