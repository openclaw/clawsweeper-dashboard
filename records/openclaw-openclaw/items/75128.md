## Summary

Keep open. The PR addresses a real current-main BOOT.md prompt-disclosure path and has useful proof, but source inspection still shows a JSON-string presentation payload can bypass the new sanitizer and be parsed later for delivery.

## What This Changes

The PR wraps BOOT.md prompts as internal runtime context, adds a per-session boot-echo guard, sanitizes message-tool visible fields, suppresses empty sanitized sends, and adds focused tests plus a changelog entry.

## Best Possible Solution

Keep the layered BOOT.md/internal-runtime-context approach, but parse and sanitize JSON-string presentation payloads before delivery, or move presentation sanitization after the existing parsing path, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path for the underlying current-main bug, and the remaining PR blocker is source-reproducible because presentation strings skip the sanitizer and are parsed later by runMessageAction.

## Solution Assessment

No. The proposed direction is narrow and maintainable, but it is incomplete until stringified presentation payloads are sanitized or rejected before the outbound runner can deliver them.
