## Summary

Keep open. The PR fixes the right prompt-disclosure surface, but a remaining P2 issue still lets a suppressed BOOT.md echo appear in generic tool-error logs via unsanitized raw params.

## What This Changes

The PR wraps BOOT.md as internal runtime context, adds a per-session boot echo guard, sanitizes message-tool visible fields and presentation text, and adds regression tests plus a changelog entry.

## Best Possible Solution

Keep the wrapper, message-tool sanitizer, and echo guard, but make intentional boot-echo suppression bypass or redact the generic raw-params failure path and cover that behavior with a regression test.

## Reproduction Assessment

Yes. Source inspection shows the BOOT.md prompt reaches fallback attempts and the message tool, and the PR's own terminal proof exercises that fallback message-tool path; the remaining log leak follows when suppression leaves an empty send that throws.

## Solution Assessment

No. The PR touches the correct layers, but intentional suppression should not fall through to a validation error that logs the original unsanitized tool call.
