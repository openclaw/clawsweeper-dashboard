## Summary

Keep open: the PR addresses the right BOOT.md prompt-disclosure path and fixes the earlier sampled-window guard, but the message-tool sanitization remains incomplete for user-visible presentation payloads.

## What This Changes

The PR wraps BOOT.md prompt content in internal-runtime-context delimiters, registers a per-session boot echo guard, extends message-tool top-level text sanitization, adds focused tests, and updates CHANGELOG.md.

## Best Possible Solution

Extend the same internal-runtime-context and boot-echo sanitization to every user-visible message-tool presentation string, add regression coverage, then review and land the PR with the existing changelog entry.

## Reproduction Assessment

Yes. Source inspection shows an action=send payload can put the wrapped BOOT.md block in presentation.title or presentation.blocks[].text; the PR sanitizes only top-level text/content/message/caption while the runner treats presentation as user-visible content.

## Solution Assessment

No. The wrapper and per-session echo guard are a good narrow direction, but the message-tool fix is not complete until presentation strings go through the same sanitizer path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Sanitize presentation text before sending:** `src/agents/tools/message-tool.ts:778-786`
