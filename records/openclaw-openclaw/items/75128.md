## Summary

Keep open: this is a valid prompt-disclosure bug fix, but the latest PR head still has a blocking bypass because JSON-string `presentation` payloads skip the new sanitizer and are parsed later before delivery.

## What This Changes

The PR wraps BOOT.md in internal-runtime-context delimiters, adds message-tool visible-payload sanitization, registers a per-session boot echo guard, and adds regression tests plus a changelog entry.

## Best Possible Solution

Keep the layered BOOT.md wrapper and echo guard, but cover parsed JSON-string presentations before delivery and add focused regression coverage for that path.

## Reproduction Assessment

Yes, from source inspection. Current main exposes BOOT.md to fallback candidates and the PR head still has a source-reproducible stringified-presentation bypass because sanitization returns non-object presentations unchanged before the runner parses them.

## Solution Assessment

No. The overall direction is narrow and maintainable, but this PR is incomplete until JSON-string presentation payloads are parsed and sanitized, or rejected, before outbound delivery.
