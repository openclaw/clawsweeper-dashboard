## What This Changes

The PR wraps BOOT.md prompts in internal-runtime-context markers, adds a per-session boot echo guard, sanitizes message-tool text and presentation fields, and adds regression tests plus a changelog entry.

## Best Possible Solution

Normalize escaped newlines before stripping runtime context, or re-run the strip after message normalization, then add redacted real fallback-run proof before merge.

Yes, source-reproducible. Current main embeds BOOT.md directly, fallback retries preserve that body, and the message tool is a user-visible delivery path; I did not perform a live Gemini fallback run.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Normalize escaped newlines before stripping runtime context:** `src/agents/tools/message-tool.ts:793`
  - body: The new sanitizer runs before `handleSendAction` turns literal `\\n` sequences in `message` into real newlines. A model can send `<<<BEGIN_OPENCLAW_INTERNAL_CONTEXT>>>\\n...\\n<<<END_OPENCLAW_INTERNAL_CONTEXT>>>`; this pass misses it, then the runner normalizes it and gateway/plugin sends can forward the marker-delimited block. Normalize escaped newlines before stripping, or run the same strip again after that conversion.
