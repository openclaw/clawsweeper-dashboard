## Summary

Keep this PR open: the boot wrapping and top-level message-tool sanitization are still useful, but the latest head still leaves a user-visible presentation payload bypass that can leak the same BOOT.md/runtime-context content.

## What This Changes

The PR wraps BOOT.md in internal-runtime-context markers, adds a per-session boot echo guard, sanitizes top-level message-tool text fields, adds focused tests, and updates the changelog.

## Best Possible Solution

Extend the same runtime-context and boot-echo sanitizer to every user-visible presentation string, add regression coverage for presentation payloads, then land the PR with the existing changelog entry.

## Reproduction Assessment

Yes. Source inspection of the PR head shows an action=send payload can put a wrapped BOOT.md block or 80+ character boot echo in presentation.title or presentation.blocks[].text, while the PR only applies the new sanitizer to top-level text/content/message/caption.

## Solution Assessment

No. The BOOT.md wrapper and per-session echo guard are a good narrow direction, but this is not complete until presentation title, block text, placeholders, button labels, and option labels use the same sanitizer path.
