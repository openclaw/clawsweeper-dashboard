## Summary

Keep open. The PR is a focused fix for a real BOOT.md prompt-disclosure path, but the latest head still lets JSON-string `presentation` payloads bypass the new sanitizer before `runMessageAction` parses and delivers them.

## Best Possible Solution

Keep the layered BOOT.md/internal-runtime-context approach, but parse and sanitize JSON-string presentation payloads before delivery, or move presentation sanitization after the existing parsing path, with focused regression coverage.
