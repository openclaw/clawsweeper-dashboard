## Summary

Keep open: the latest force-push adds useful boot wrapping and an echo guard, but the message-tool hardening still misses rich presentation payloads, so a user-visible prompt-disclosure path remains.

## What This Changes

The PR wraps BOOT.md prompt content in internal-runtime-context delimiters, adds a per-session boot echo guard, sanitizes top-level message-tool text fields, and adds tests plus a changelog entry.

## Best Possible Solution

Extend one shared outbound user-visible text sanitizer to both top-level message fields and presentation title/block/button/option strings, keep the boot wrapper and guard, then land with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection is enough: presentation-only sends are valid, and the PR's new sanitizer does not run on presentation.title or presentation.blocks text before delivery.

## Solution Assessment

No. The boot wrapper and echo guard are a good narrow direction, but the current patch is incomplete until every user-visible presentation string goes through the same sanitizer path.
