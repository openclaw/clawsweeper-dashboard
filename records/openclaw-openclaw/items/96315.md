## Summary

## What This Changes

The branch updates Feishu WebSocket handoff, inbound queue/session keying, topic thread-id hydration, docs, and tests so configured topic-group sessions can process per topic in parallel while unset topic groups remain chat-scoped.

## Best Possible Solution

Land the Feishu plugin fix after maintainers accept the early-ack and per-topic ordering tradeoffs, with the latest CI green and the unset topic-group default preserved.

## Reproduction Assessment

Yes, at source level: current main queues Feishu messages by account and chat id, while configured topic sessions use more specific conversation ids. The PR body also provides live Feishu proof that two topics completed independently after the fix.

## Solution Assessment

Yes, with maintainer acceptance of the delivery tradeoff: the fix stays in the Feishu plugin boundary, preserves the shipped default, and aligns queue keys with configured session keys. A narrower alternative would avoid the transport-level async wrapper if maintainers do not want early SDK acknowledgement.
