## Summary

Keep open. Current main still enqueues non-ok and thrown `/hooks/agent` results into prompt-visible session system events, and the related result-return/announcement-suppression work is still open at https://github.com/openclaw/openclaw/issues/49019 and https://github.com/openclaw/openclaw/pull/67433.

## What This Changes

The issue requests a way to keep hook delivery errors and artifacts out of agent sessions so agents do not relay them as noisy channel messages.

## Best Possible Solution

Define an explicit Gateway hook event-audience policy or API option that keeps machine-only hook artifacts out of model-visible prompts while preserving operator diagnostics, trust metadata, idempotency, timeout/error semantics, and docs.

## Reproduction Assessment

Yes, by source inspection: a `/hooks/agent` run that returns `status: "error"` or `status: "skipped"`, or throws before completion, still reaches `enqueueSystemEvent`, and those queued entries are drained into the next prompt. I did not run a live Discord reproduction in this read-only review.

## Solution Assessment

Unclear. Suppressing prompt injection for machine-only hook artifacts is plausible, but current tests intentionally preserve non-ok announcements, so the maintainable fix needs a Gateway hook event-audience/API decision rather than a Discord-only outbound text filter.
