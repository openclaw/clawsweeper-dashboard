- `impact:message-loss`: The issue is about hook artifacts causing unintended outbound channel messages and dedup noise.
- `impact:session-state`: The reported failure path injects hook artifacts into agent sessions where they become prompt-visible context.

## Summary

Keep open. Current main and latest release still enqueue non-ok and thrown hook-agent outcomes as prompt-visible system events, and the related result-return/suppression work is still open rather than shipped.

## What This Changes

The issue asks for hook delivery errors/artifacts to stop being injected into agent-visible sessions where agents may narrate them back into channels.

## Best Possible Solution

Add a maintainer-approved hook relay/event-audience policy that keeps machine-only delivery artifacts out of model-visible prompts while preserving operator-visible diagnostics and documented opt-in failure visibility.

## Reproduction Assessment

Yes, by source inspection. A hook agent run that returns `status: "error"` or `status: "skipped"`, or throws before completion, still reaches `enqueueSystemEvent`, and queued events are drained into the next prompt as `System:` lines.

## Solution Assessment

Unclear. Suppressing prompt injection for hook artifacts is plausible, but the maintainable fix belongs in hook dispatcher/event-audience policy with explicit failure-visibility semantics rather than a Discord-only outbound text filter.
