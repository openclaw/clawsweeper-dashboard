## Summary

Keep open. Current main has narrowed the original noise path for successful `deliver:false` hooks and attempted delivery fallback, but failed/skipped hook runs and thrown hook dispatcher errors are still intentionally enqueued as untrusted `Hook ...` system events into an agent main session, with no hook-level suppression or audience policy.

## What This Changes

Suppress or make configurable the gateway relay of hook delivery artifacts and errors into agent sessions so agents do not produce noisy channel replies.

## Best Possible Solution

Add a core hook relay/audience policy that keeps hook delivery artifacts and failures out of agent prompts when appropriate, while preserving structured logs and targeted tests for direct and mapped hook paths.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by mocking an isolated hook run to return `status: "error"` or throw; current tests assert that this enqueues `Hook ...` system events, and prompt-building code then prepends drained events into agent input.

## Solution Assessment

No. Current main's partial mitigation is not the best final solution because it still uses model-visible system events as the failure relay; an explicit hook delivery relay policy or event audience is narrower than relying on agent instructions or outbound message filters.

## Repair Work Prompt

Fix the hook delivery relay noise reported in https://github.com/openclaw/openclaw/issues/57700. Current main still enqueues failed/skipped/thrown hook agent run summaries as `Hook ...` system events in `src/gateway/server/hooks.ts`, making them model-visible through `session-system-events.ts` and `prompt-prelude.ts`. Add a conservative hook delivery relay policy/config, using a clear name such as `hooks.suppressDeliveryRelay` if that fits local config conventions, that can suppress hook delivery artifacts/errors from agent sessions while preserving structured logs/audit visibility. Keep the existing successful `deliver:false` and attempted-delivery suppression behavior. Do not implement a channel outbound text filter as the primary fix, because that leaves the prompt polluted. Update hook config types/schema labels/help/generated docs as needed, public docs, and `CHANGELOG.md`. Add or adjust tests for direct `/hooks/agent` and mapped agent failure/non-delivery paths, including explicit-agent routing, and keep the behavior compatible with third-party hook callers.
