## Summary

Keep open. Current main and v2026.5.18 still route non-ok and thrown `/hooks/agent` outcomes into prompt-visible session system events, and the related Gateway API work at https://github.com/openclaw/openclaw/issues/49019 and https://github.com/openclaw/openclaw/pull/67433 remains open rather than shipped.

## What This Changes

The issue requests preventing hook delivery error/artifact system events from entering agent sessions, or adding a suppression/filter policy for those artifacts.

## Best Possible Solution

Ship a maintainer-approved `/hooks/agent` event-audience/result policy that keeps machine-only hook artifacts out of model-visible prompts while preserving explicit operator diagnostics and documented failure visibility.

## Reproduction Assessment

Yes, by source inspection. A hook agent run that returns `status: "error"` or `status: "skipped"`, or throws before completion, still reaches `enqueueSystemEvent`, and those queued events are drained into the next prompt.

## Solution Assessment

Unclear. Suppressing prompt injection for hook artifacts is plausible, but the maintainable fix belongs in hook dispatcher/event-audience policy with explicit failure-visibility semantics rather than a Discord-only outbound text filter.
