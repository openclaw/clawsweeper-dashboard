## Summary

Keep open. Current main still sends non-ok and thrown hook-agent outcomes into model-visible session system events, and the landed narrow fix only covered successful silent or already-attempted delivery paths, so the remaining hook failure/artifact relay policy still needs maintainer direction.

## What This Changes

Request to suppress or configure hook delivery error/artifact relay so `Hook Hook` system events stop causing channel noise.

## Best Possible Solution

Add a hook relay/event-audience policy that keeps delivery artifacts and optionally failed hook diagnostics out of model-visible prompts while preserving operator-visible failure diagnostics and focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. A hook agent run that returns `status:"error"` or `status:"skipped"`, or throws, still follows the current dispatcher path into `enqueueSystemEvent`, and prompt construction prepends those events to the next agent input.

## Solution Assessment

Unclear. The requested suppression is valid, but the maintainable fix should be a hook relay/audience policy in the dispatcher or system-event path rather than an outbound Discord text filter, and the default failure-visibility behavior needs maintainer choice.
