## Summary

Keep open. Current main still relays returned non-ok hook outcomes and thrown hook-agent failures into model-visible session system events; the merged narrow fix only silenced successful no-delivery/already-attempted paths, so the remaining hook failure/artifact relay policy still needs maintainer judgment.

## What This Changes

The issue requests a hook delivery relay policy or config that prevents hook delivery errors/artifacts from being injected into target agent sessions where they can trigger noisy channel replies.

## Best Possible Solution

Add a hook relay/event-audience policy that keeps delivery artifacts and optionally failed hook diagnostics out of model-visible prompts while preserving operator-visible failure diagnostics and focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. A hook agent run that returns `status:"error"` or `status:"skipped"`, or throws, still reaches `enqueueSystemEvent`, and queued events are drained into the next agent prompt.

## Solution Assessment

Unclear. The requested suppression is valid, but an outbound Discord text filter would be the wrong layer; the maintainable fix belongs in the hook dispatcher/system-event audience policy with explicit failure-visibility semantics.
