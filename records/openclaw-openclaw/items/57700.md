## Summary

Keep open. Current main still routes returned non-ok or thrown hook-agent outcomes into model-visible untrusted system events; the merged narrow fix only silenced successful no-delivery or already-attempted paths, and the remaining failure/artifact visibility needs a maintainer policy/API decision.

## What This Changes

Requests a hook delivery relay suppression or outbound filter so hook error/artifact system events do not prompt agents to post noisy Discord messages.

## Best Possible Solution

Add a hook relay/event-audience policy that keeps delivery artifacts and optionally failed hook diagnostics out of model-visible prompts while preserving operator-visible diagnostics and focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. A hook agent run that returns `status:"error"` or `status:"skipped"`, or throws, still reaches `enqueueSystemEvent`, and those queued events are drained into the next prompt.

## Solution Assessment

Unclear. Suppressing prompt injection of hook artifacts is reasonable, but an outbound Discord text filter is the wrong layer; the maintainable fix belongs in hook dispatcher/event audience policy with explicit failure-visibility semantics.
