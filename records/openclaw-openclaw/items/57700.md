## Summary

Keep open. Current main still relays failed, skipped, and thrown hook-agent results into model-visible session system events, while only successful silent or already-delivered hook paths have been narrowed; the remaining fix needs a maintainer decision on hook failure visibility and relay policy.

## What This Changes

The issue requests suppressing or policy-controlling hook delivery/error artifacts so `Hook ...` system events do not enter agent sessions and create channel noise.

## Best Possible Solution

Add a core hook relay or event-audience policy that keeps hook delivery artifacts and failures out of model-visible prompts when appropriate, while preserving structured operator-visible failure diagnostics and focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Mocking an isolated hook run to return `status:"error"` or `status:"skipped"`, or to throw, follows current tested paths that enqueue `Hook ...` events which prompt construction then prepends to the agent input.

## Solution Assessment

Unclear. The requested direction is valid, but the best maintainable fix should be a hook relay/audience policy in the dispatcher/system-event path rather than an outbound Discord text filter; the default and operator-visibility behavior need maintainer choice.
