## Summary

Keep open. Current main and v2026.5.18 still route non-ok or thrown `/hooks/agent` outcomes into prompt-visible system events, and the related result-return/announcement PR remains open rather than shipped.

## What This Changes

Request suppressing hook delivery error/artifact system events from model-visible agent sessions, or adding a hook config/API policy to prevent noisy channel replies.

## Best Possible Solution

Add a maintainer-approved hook relay/event-audience policy that keeps machine-only delivery artifacts out of model-visible prompts while preserving operator-visible diagnostics and documented opt-in failure visibility.

## Reproduction Assessment

Yes, by source inspection. A hook agent run that returns `status: "error"` or `status: "skipped"`, or throws before completion, still reaches `enqueueSystemEvent`, and those queued events are drained into the next prompt.

## Solution Assessment

Unclear. Suppressing prompt injection for hook artifacts is plausible, but the maintainable fix belongs in hook dispatcher/event-audience policy with explicit failure-visibility semantics rather than a Discord-only outbound text filter.
