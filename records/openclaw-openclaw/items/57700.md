## What This Changes

The issue requests preventing hook delivery error or artifact summaries from being injected into agent sessions where agents can relay noisy follow-up messages to Discord.

## Best Possible Solution

Define a documented hook announcement policy that preserves default failure observability while offering an explicit machine-callback suppression mode, likely by reviewing and refining the linked `announceToMain:false` work or adding an equivalent config-level control with tests.

## Reproduction Assessment

Yes for the source-level injection path, but not for the full Discord narration symptom. Current main code and tests show non-ok hook results enqueue `Hook ... (error)` system events into an agent session.

## Solution Assessment

Unclear. The requested suppression is plausible, but maintainers need to choose between per-request `announceToMain:false`, a config-level suppression policy, or preserving failure announcements by default with a narrower opt-in.
