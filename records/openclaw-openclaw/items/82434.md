## Summary

Keep open: the requested Gateway/SDK surface is not on current main and the PR is active, but the patch still has merge blockers around public admin-scope SDK exposure, missing protocol documentation, and real behavior proof.

## What This Changes

The branch adds `plugin.approval.resolveVerified` with plugin ownership checks, admin-scoped Gateway client helpers, protocol/Swift/schema updates, SDK docs, and focused tests.

## Best Possible Solution

Ship a narrow plugin-owned verified resolver with owner checks, expose only the smallest approved SDK helper for that use case, document the protocol contract, and keep AgentKit itself as a ClawHub package.

## Reproduction Assessment

Not applicable as a current-main bug; this is additive Gateway/SDK API work. The review blockers are source-reproducible from the PR diff and current docs: the public admin helper is exported and the new RPC is not documented in protocol docs.

## Solution Assessment

No, not as-is. The resolver direction is focused, but exporting a generic `operator.admin` client helper through the public SDK broadens the authority boundary, and the new RPC needs protocol documentation before merge.
