## Summary

## What This Changes

The PR adds `externalResolution`, `plugin.approval.resolveVerified`, protocol/SDK docs and types, and channel/UI rendering for external plugin approval verification commands.

## Best Possible Solution

Land the narrow external verification seam only after `resolveVerified` is bound to authenticated caller/plugin identity, or after maintainers explicitly choose and document an operator-scoped trust model with matching tests and docs.

## Reproduction Assessment

Yes. ## Solution Assessment

No. The `externalResolution` command-template shape is a plausible narrow API, but the best fix is to bind verified resolution to a real caller/plugin identity or explicitly document an operator-scoped resolver rather than trusting `params.pluginId`.
