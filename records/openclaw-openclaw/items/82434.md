## Best Possible Solution

Keep AgentKit itself outside core, but land this generic approval seam only after `resolveVerified` is bound to authenticated caller/plugin identity or explicitly documented and tested as an operator-scoped trust model.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create an external-resolution approval for plugin A, connect a generic approval-runtime client, and call `plugin.approval.resolveVerified` with plugin A's id because the handler trusts `params.pluginId`.

## Solution Assessment

No. The narrow `externalResolution` shape is plausible, but the resolver is not the best implementation until ownership is bound to authenticated caller/plugin identity or explicitly changed to an operator-scoped contract.
