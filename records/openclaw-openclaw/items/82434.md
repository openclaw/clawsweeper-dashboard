## Summary

Keep open: current main still lacks this additive Gateway/SDK API, and the PR is useful but the latest head still has a plugin ownership gap on idempotent verified-resolution replays. The supplied proof is also CI/test oriented and the external AgentKit HITL script mocks the approval resolver rather than showing a real Gateway flow through the new method.

## What This Changes

Adds an admin-scoped `plugin.approval.resolveVerified` Gateway method, a narrow plugin SDK helper, protocol/Swift/docs/API baseline updates, and focused tests.

## Best Possible Solution

Land the narrow admin-scoped resolver and SDK helper only after plugin ownership validation covers both first resolution and idempotent repeat paths, with a real Gateway/plugin proof and aligned companion host APIs.

## Reproduction Assessment

Not applicable as a current-main bug, but the PR defect is source-reproducible: resolve an approval once with the owning pluginId, then replay the same id and decision through `plugin.approval.resolveVerified` with another pluginId. The shared resolver returns success before running the new ownership validator.

## Solution Assessment

No, not yet. The narrow helper is the right direction, but the ownership check needs to apply on resolved-repeat paths and the proof should exercise the real Gateway method before merge.
