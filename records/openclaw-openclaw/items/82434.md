## Summary

Keep open: this PR adds useful Gateway/SDK API surface, but it is not merge-ready because it exposes a generic public operator-admin SDK helper, leaves the new RPC out of the protocol docs, and provides checks-only proof rather than a real Gateway/plugin approval run.

## What This Changes

The PR adds an admin-scoped `plugin.approval.resolveVerified` RPC with `pluginId` ownership checks, SDK operator-admin Gateway helpers, generated protocol artifacts, docs snippets, and focused tests.

## Best Possible Solution

Ship a narrow verified-approval resolver for plugin-owned pending requests, expose only a purpose-built SDK helper unless maintainers explicitly approve the admin trust model, document the protocol contract, and require redacted real-flow proof.

## Reproduction Assessment

Not applicable for a current-main bug reproduction; this is additive API work. The PR review blockers are source-visible in the diff: the public SDK exports admin helpers and the protocol docs still list only the existing plugin approval methods.

## Solution Assessment

No, not as-is. The verified resolver itself is focused, but the public operator-admin helper is broader than the stated use case; a purpose-built verified-resolution helper plus protocol docs and real-flow proof would be safer.
