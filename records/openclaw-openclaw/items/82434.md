## Summary

Keep open: the API gap is real and the related generic approval work remains active, but the current patch should not merge as-is because it exposes a generic admin Gateway client helper through the public plugin SDK and still lacks real after-fix behavior proof.

## What This Changes

The PR adds an admin-scoped `plugin.approval.resolveVerified` Gateway method, public operator-admin Gateway SDK helpers, protocol/schema/generated model updates, docs, and focused approval tests.

## Best Possible Solution

Ship a narrow verified-approval resolver with plugin ownership checks, document the protocol method, and expose only a purpose-built SDK helper unless maintainers explicitly approve a generic public admin Gateway client.

## Reproduction Assessment

Not applicable as a current-main bug; this is an additive API PR. The blocking review concern is source-reproducible from the PR diff because `openclaw/plugin-sdk/gateway-runtime` exports helpers that request `operator.admin`.

## Solution Assessment

No, not as-is. The verified resolver itself is narrow, but publishing a generic admin client helper broadens the plugin SDK control-plane boundary beyond the stated proof-backed approval use case.
