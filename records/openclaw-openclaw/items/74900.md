## Summary

Keep open. Current main still lacks the requested plugin-neutral `openclaw/plugin-sdk/embeddings` subpath, the only exported/documented facade is still memory-core-named, and the linked implementation PR #67242 was closed unmerged, so this remains an SDK contract decision for owners.

## What This Changes

Expose embedding provider factory, registry, type, and batch helpers through a stable plugin-neutral SDK subpath independent of memory-core activation.

## Best Possible Solution

Ship an owner-approved additive `openclaw/plugin-sdk/embeddings` facade, or explicitly bless and document the current memory-core-named path as stable, with package exports, SDK metadata, API baseline, docs, and import-topology checks kept aligned.

## Solution Assessment
