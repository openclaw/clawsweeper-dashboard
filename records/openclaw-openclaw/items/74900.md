## Best Possible Solution

Add an owner-approved additive `openclaw/plugin-sdk/embeddings` facade, keep the current memory-core-named path compatible, document the preferred import/deprecation status, and update exports, entrypoint metadata, API checks, focused contract tests, and changelog together.

## Solution Assessment

Unclear pending SDK/memory owner input. A narrow additive generic subpath matches the SDK boundary guidance, but the public name and exact export set should be approved before implementation.
