## What This Changes

Expose a stable public Plugin SDK embeddings subpath so third-party memory-slot plugins can use embedding helpers and provider registry APIs without importing the memory-core-named facade.

## Best Possible Solution

Ship an owner-approved additive `openclaw/plugin-sdk/embeddings` facade, keep the current memory-core-named path compatible or documented as legacy, and update exports, SDK metadata, docs, API checks, focused coverage, and changelog in one narrow change.

## Solution Assessment
