Full review comments:

- **[P1] Integrate per-agent policy with the current write contract:** `src/config/sessions/store.ts:399-400`
  - body: Current main already lets callers pass a pre-resolved `maintenanceConfig`, and normal reply/session writes resolve that from global `session.maintenance`. Reintroducing an `agentId` option here without moving per-agent resolution into that current contract can leave runtime saves enforcing global retention while cleanup uses per-agent settings.
  - confidence: 0.89
- **[P1] Keep shared cleanup stores on an owner-safe policy:** `src/commands/sessions-cleanup.ts:374`
  - body: `--all-agents` target resolution can dedupe multiple agents to one store path, but passing the surviving `target.agentId` into destructive maintenance applies one agent's retention policy to the whole file. A stricter override can prune or cap sessions owned by another agent sharing that store.
  - confidence: 0.9
- **[P2] Detach manual store cleanup from default-agent overrides:** `src/commands/sessions-cleanup.ts:374`
  - body: For `--store`, target resolution uses the default agent id as a placeholder for an arbitrary operator-supplied path. Feeding that id into maintenance makes manual cleanup inherit the default agent's override instead of a safe global policy for the selected store.
  - confidence: 0.86
- **[P2] Normalize agent ids before matching overrides:** `src/config/sessions/store-maintenance.ts:141`
  - body: This lookup compares raw ids, while normal agent resolution compares normalized ids. A configured id such as `Ops` can resolve elsewhere but miss its maintenance override here and silently fall back to global policy.
  - confidence: 0.86
- **[P2] Resolve legacy pruneDays before merging overrides:** `src/config/sessions/store-maintenance.ts:162`
  - body: A raw spread can keep both global `pruneAfter` and agent `pruneDays` on the merged object. Since the resolver prefers `pruneAfter`, a supported per-agent `pruneDays` override will not win over a global `pruneAfter`.
  - confidence: 0.83
