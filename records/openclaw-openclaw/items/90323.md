  - body: `ExecDb` only declares `exec()`, but the rollback-journal branch still calls `db.prepare(...).get()`. The widened type therefore does not actually cover the helper’s current API use.
  - confidence: 0.94
- **[P1] Preserve the reindex swap read lock:** `extensions/memory-core/src/memory/manager-db.ts:178-184`
  - body: Current main acquires a swap read lock for each live memory DB handle and releases it from `closeMemoryDatabase()`. This version returns opened DB handles without holding that lock, so a safe reindex can publish while another process still has the database open.
  - confidence: 0.9
- **[P1] Declare better-sqlite3 in the shipped package graph:** `packages/memory-host-sdk/package.json:21-23`
  - body: Adding `better-sqlite3` only to the private `@openclaw/memory-host-sdk` package does not make the module available to published `openclaw` installs; root `package.json` and `npm-shrinkwrap.json` still lack it. The runtime `require('better-sqlite3')` can fail after install even if workspace tests pass.
  - confidence: 0.9
- **[P2] Keep rollback-journal temp sidecars in cleanup:** `extensions/memory-core/src/memory/manager-db.ts:22-23`
  - body: Current main includes `-journal` in both temp discovery and deletion because network-backed stores can run in rollback-journal mode. Dropping that suffix reintroduces leaked `.tmp-<uuid>-journal` files after hard-killed reindexes on those stores.
  - confidence: 0.86
