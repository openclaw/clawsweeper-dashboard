  - body: After this change removes the `node:sqlite` type import from `sqlite-vec.ts`, `assertSqliteVecAvailable` and `loadExtensionAndVerify` still refer to `DatabaseSync`. Those unresolved identifiers will break TypeScript compilation; type the helpers against `MemoryDb` or a narrower local load/prepare contract.
  - confidence: 0.94
- **[P1] Include prepare in the WAL DB contract:** `src/infra/sqlite-wal.ts:7`
  - body: `ExecDb` only includes `exec()`, but the rollback-journal branch still calls `db.prepare("PRAGMA journal_mode = DELETE;").get()`. The widened type is therefore internally inconsistent and will fail typechecking for the same helper this PR is trying to generalize.
  - confidence: 0.93
- **[P1] Declare better-sqlite3 in the shipped package graph:** `packages/memory-host-sdk/package.json:21-22`
  - body: The runtime now calls `require("better-sqlite3")`, but this adds the dependency only to the private `@openclaw/memory-host-sdk` workspace package. The published root package dependencies and npm-shrinkwrap do not include it, so installed OpenClaw packages can hit a missing-module failure even if the workspace tests pass.
  - confidence: 0.88
- **[P1] Preserve the reindex swap read lock:** `extensions/memory-core/src/memory/manager-db.ts:178-184`
  - body: Current main attaches a swap read lock to each opened memory DB so safe reindex publication cannot race a live peer. This PR removes the `acquireMemoryReindexSwapReadLock`/`liveDatabaseSwapLocks` lifecycle and returns opened handles without the guard, which can regress peer-write and index swap safety.
  - confidence: 0.85
- **[P2] Keep rollback-journal temp sidecars in cleanup:** `extensions/memory-core/src/memory/manager-db.ts:22-23`
  - body: Current main cleans `-journal` sidecars for rollback-journal reindex temp DBs on network stores. Dropping `-journal` from both suffix lists means hard-crash temp journals can be rediscovered neither for grouping nor deletion, reintroducing the leak this path recently fixed.
  - confidence: 0.86
