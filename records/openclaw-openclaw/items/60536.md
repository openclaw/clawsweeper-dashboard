The PR changes gateway WebSocket pre-handshake close logging so loopback health-probe normal closes with a matching origin log at debug instead of warn.

## Best Possible Solution

Use a deterministic server-visible internal-probe signal that the real probe client actually sends, preserve WARN for unrelated pre-connect closes, and add focused regression coverage plus redacted runtime log proof.

## Solution Assessment
