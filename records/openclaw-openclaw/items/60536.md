## What This Changes

The PR adds an `isInternalProbeClose()` helper in `src/gateway/server/ws-connection.ts` to downgrade loopback code-1000 pre-handshake closes with a matching Origin from warn to debug.

## Best Possible Solution

Use a deterministic internal-probe signal that the server can actually observe, downgrade only clean internal probe closes, and add focused regression coverage plus redacted real gateway proof.

## Reproduction Assessment

Yes, at source level. Current main still warns for non-SwiftPM pre-client closes, the probe path stops a probe-mode `GatewayClient`, and the PR predicate depends on a request Origin that the current client does not set.

## Solution Assessment

No. The PR's server-only Origin check is not the best fix because the current probe client does not provide that signal; a deterministic probe marker or other reliable server-visible cause would be narrower and more maintainable.
