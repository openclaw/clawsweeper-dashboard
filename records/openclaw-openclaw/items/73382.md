## Summary

Keep this PR open. It targets the right Bedrock discovery bug, but the proposed family fallback is still incorrect for the Claude 4 Bedrock ID shapes it claims to cover, and a related configurable-override alternative remains open for maintainer comparison.

## What This Changes

The branch adds missing Amazon Bedrock Claude context-window entries, introduces a Claude family fallback, adds discovery regression tests, and updates the changelog for the Bedrock context-window bug.

## Best Possible Solution

If the built-in fallback direction is kept, update the matcher and tests so Claude 3.x and actual Claude 4 Bedrock naming shapes resolve correctly while explicit 1M rows keep precedence. If maintainers prefer user-configurable overrides, use #73959 as the canonical path and close or supersede this PR after that decision.

## Reproduction Assessment

Yes. Current main has a static reproduction for the original bug: an unlisted ID such as `anthropic.claude-3-5-sonnet-20241022-v2:0` misses `KNOWN_CONTEXT_WINDOWS` and falls through to the 32000 discovery default; the PR finding is also reproducible by testing the proposed regex against `anthropic.claude-sonnet-4-20250514-v1:0`.

## Solution Assessment

No, not as-is. The explicit Claude 3.x backfill is narrow and appropriate, but the documented Claude 4 fallback must either match real Claude 4 Bedrock IDs or be scoped only to Claude 3.x; #73959 is the broader alternative if maintainers want operator-managed overrides.
