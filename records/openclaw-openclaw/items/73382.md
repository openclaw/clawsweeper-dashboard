## Summary

Keep this PR open. The Bedrock context-window bug is real and current main still leaves common Claude 3.x IDs on the 32K discovery default, but the proposed family fallback is not correct as written: it claims Claude 4 coverage while missing actual Bedrock Claude 4 ID shapes such as `anthropic.claude-sonnet-4-...` and `anthropic.claude-opus-4-...`.

## What This Changes

The branch adds Amazon Bedrock Claude context-window entries, a regex-based Claude family fallback, discovery regression tests, and a changelog entry for the Bedrock 32K fallback bug.

## Best Possible Solution

Keep the narrow built-in table/fallback approach only if the matcher covers real Bedrock Claude 3.x and Claude 4 naming shapes while preserving explicit 1M entries. If maintainers prefer operator-managed overrides, use #73959 as the canonical direction and close or supersede this PR after that decision.

## Reproduction Assessment

Yes. Current main statically reproduces the original bug because an unlisted ID such as `anthropic.claude-3-5-sonnet-20241022-v2:0` misses the known table and falls through to the 32K discovery default; the PR-specific bug is reproducible by testing its regex against AWS-documented Claude 4 IDs. citeturn1view0turn1view2

## Solution Assessment

No, not as-is. The explicit Claude 3.x backfill is narrow and appropriate, but the family fallback must either match actual Claude 4 Bedrock IDs or be scoped and documented as Claude 3.x-only.
