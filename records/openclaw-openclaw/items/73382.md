## Summary

Keep this PR open, but it needs changes before merge. Current main still has the reported Bedrock discovery bug, while this patch's advertised Claude 4 family fallback misses the real Claude 4 Bedrock ID shape and would still fall through to the 32K default for unlisted Claude 4 variants.

## What This Changes

The PR adds missing Amazon Bedrock Claude context-window entries, a Claude 3/4 family fallback, discovery regression tests, and a changelog entry for the linked Bedrock context-window bug.

## Best Possible Solution

Repair the fallback so it either correctly covers real Claude 4 Bedrock ID forms with tests or explicitly scopes the new family fallback to Claude 3.x, then let maintainers choose between this narrow table/fallback fix and #73959's broader override design.

## Reproduction Assessment

Yes. Current main has a static reproduction through missing known-table rows falling to the 32000 default, and the PR blocker is reproducible by testing the proposed regex against current-main Claude 4 ID shapes.

## Solution Assessment

No, not as-is. The explicit Claude 3.x backfill is narrow and maintainable, but the advertised Claude 4 fallback must match `claude-sonnet-4`, `claude-opus-4`, and `claude-haiku-4` forms or be scoped out.

## Review Findings
