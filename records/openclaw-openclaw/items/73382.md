## Best Possible Solution

Repair the fallback so it accurately covers real Claude 4 Bedrock ID forms with regression tests, or narrow the fallback/comment/changelog claim to Claude 3.x while maintainers decide whether #73959's broader override design should land separately.

## Reproduction Assessment

Yes. Current main has a static reproduction because the reported Claude 3.x IDs are absent from `KNOWN_CONTEXT_WINDOWS` and fall through to the 32000 discovery default; the PR blocker is reproducible with the proposed regex against Claude 4 Bedrock IDs.

## Solution Assessment

No, not as-is. The explicit Claude 3.x entries are a narrow maintainable fix, but the Claude 4 family fallback must match `claude-sonnet-4`, `claude-opus-4`, and `claude-haiku-4` forms or be scoped out.
