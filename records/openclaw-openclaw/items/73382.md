## Summary

Keep open: current main still has the Bedrock context-window bug, but this PR is not merge-ready because its Claude 4 fallback misses real Bedrock ID shapes and it lacks after-fix real behavior proof.

## What This Changes

The PR backfills Amazon Bedrock Claude context-window table entries, adds a Claude 3/4 regex family fallback, adds discovery tests, and updates the changelog.

## Best Possible Solution

Repair the Bedrock plugin fallback to cover real Claude 4 ID shapes with regression tests, or scope the fallback to Claude 3.x while keeping the explicit table backfill for reported IDs.

## Reproduction Assessment

Yes. Source inspection shows current main falls through to the 32000 default for reported unlisted Claude IDs, and a read-only regex check reproduces the PR's Claude 4 matcher miss.

## Solution Assessment

No, not as-is. The explicit table backfill is narrow, but the advertised Claude 4 family fallback must either match `claude-sonnet-4`, `claude-opus-4`, and `claude-haiku-4` forms or be removed from the claimed scope.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Match real Claude 4 Bedrock IDs in the fallback:** `extensions/amazon-bedrock/discovery.ts:134`
  - body: The new fallback claims to cover Claude 4 variants, but `^anthropic\.claude-(?:3|4)` only matches IDs beginning `anthropic.claude-4`. Widen the matcher and add a Claude 4 fallback test, or scope the fallback to Claude 3.x only.
