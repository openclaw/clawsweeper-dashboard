## Summary

Keep open: the Bedrock context-window bug is real and current main still lacks the requested Claude 3.x rows, but this PR has a blocking matcher issue for the Claude 4 fallback it advertises.

## What This Changes

This PR adds Amazon Bedrock Claude context-window rows, a Claude family fallback in discovery, regression tests, and a changelog entry for the linked context-window bug.

## Best Possible Solution

Update the fallback to correctly cover real Claude 4 Bedrock ID forms with regression tests, or explicitly narrow the new family fallback and changelog wording to Claude 3.x while preserving exact-table precedence for 1M rows.

## Reproduction Assessment

Yes. Current main can be checked statically: missing known-table rows fall through to the 32,000-token default, and the PR blocker is reproducible by testing the proposed regex against current Claude 4 ID shapes.

## Solution Assessment

No, not as-is. The explicit Claude 3.x backfill is narrow and maintainable, but the family fallback must either match real Claude 4 forms or stop claiming Claude 4 coverage.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Match Claude 4 Bedrock IDs in the fallback:** `extensions/amazon-bedrock/discovery.ts:134`
  - body: The new fallback claims to cover Claude 4 variants, but `^anthropic\.claude-(?:3|4)` only matches IDs starting with `anthropic.claude-4`. Widen the matcher and add a Claude 4 fallback test, or scope the fallback/comment/changelog to Claude 3.x only.
