## Summary

Keep this PR open. It addresses a real Bedrock discovery bug, but the proposed Claude 4 family fallback does not match the Claude 4 ID shapes already used by current `main`, so it should not merge as-is.

## What This Changes

The PR adds missing Amazon Bedrock Claude context-window entries, a Claude family fallback, discovery regression tests, and a changelog entry for the linked Bedrock context-window bug.

## Best Possible Solution

Repair the fallback so it correctly covers real Claude 4 Bedrock ID forms with tests, or scope the fallback/comment/changelog to Claude 3.x while leaving the broader override design to #73959.

## Reproduction Assessment

Yes. Current main has a static reproduction through missing known-table rows falling to the 32000 default, and the PR blocker is reproducible by testing the proposed regex against current-main Claude 4 ID shapes.

## Solution Assessment

No, not as-is. The explicit Claude 3.x backfill is narrow and maintainable, but the advertised Claude 4 fallback must match `claude-sonnet-4`, `claude-opus-4`, and `claude-haiku-4` forms or be scoped out.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Match real Claude 4 Bedrock IDs in the fallback:** `extensions/amazon-bedrock/discovery.ts:134`
  - body: The new fallback claims to cover Claude 4 variants, but `^anthropic\.claude-(?:3|4)` only matches IDs starting `anthropic.claude-4...`. Widen the matcher and add a Claude 4 fallback test, or scope the fallback/comment/changelog to Claude 3.x only.
