## Summary

Keep this PR open. Current main still has the reported Bedrock discovery gap, and the PR is useful, but its new family fallback is inconsistent with the stated Claude 4 coverage because it does not match real Claude 4 Bedrock ID shapes.

## What This Changes

The PR adds missing Amazon Bedrock Claude context-window entries, a Claude-family fallback, discovery regression tests, and a changelog entry for the Bedrock context-window bug.

## Best Possible Solution

Repair the PR so Bedrock discovery resolves the missing Claude 3.x IDs and either correctly covers real Claude 4 Bedrock ID shapes with tests or explicitly scopes the new fallback to Claude 3.x. If maintainers prefer configurable operator overrides, #73959 should become the canonical path instead.

## Reproduction Assessment

Yes. Current main has a static reproduction: an unlisted discovered ID such as `anthropic.claude-3-5-sonnet-20241022-v2:0` misses the known table and falls through to the 32000 discovery default; the PR blocker reproduces by testing the proposed regex against `anthropic.claude-sonnet-4-20250514-v1:0`.

## Solution Assessment

No, not as-is. The explicit Claude 3.x backfill is narrow and maintainable, but the advertised Claude 4 fallback must match real Bedrock IDs or be scoped out; #73959 is the broader alternative if maintainers want user-managed overrides.
