## Summary

Keep open: current main does not show `directLike` as an existing supported Discord option, and adding it would be a new end-to-end Discord config/runtime behavior rather than a closeable implemented bug.

## What This Changes

The issue asks OpenClaw to accept a `directLike` Discord channel override consistently in config schema, bundled plugin manifest metadata, and gateway startup validation.

## Best Possible Solution

If maintainers want this behavior, implement `directLike` as a Discord-owned config feature end to end across types, Zod schema, generated bundled metadata, runtime prompt/typing/session behavior, docs, changelog, and regression tests.

## Reproduction Assessment

No high-confidence runtime reproduction was established. Source inspection shows current main would reject `directLike` through the strict Discord channel schema, but it also shows no existing runtime or docs contract for that key.

## Solution Assessment

No, the suggested manifest-only schema patch is not the best complete fix. The maintainable path is either to document that this key is unsupported or add the new Discord-owned behavior consistently through schema, runtime, generated metadata, docs, and tests.
