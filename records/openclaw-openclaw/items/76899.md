## Summary

Keep open: current main does not already implement `directLike`, and the evidence points to a requested new Discord channel mode rather than a schema-only drift fix. The schema rejection is real from source inspection, but runtime and docs do not define `directLike` as existing supported behavior.

## What This Changes

The issue asks for Discord channel config and bundled plugin metadata to accept a `directLike` boolean for guild-channel overrides.

## Best Possible Solution

Decide whether Discord should have a direct-like guild-channel mode; if yes, implement it through the Discord schema, generated metadata, docs, and all runtime direct-vs-channel decision points together.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established. The rejection portion is source-reproducible because both the Zod schema and generated metadata are strict and omit `directLike`, but the claimed existing runtime support is not present in current main.

## Solution Assessment

No. A manifest/schema-only addition is not the best fix unless maintainers confirm the behavior already exists elsewhere; current main would also need runtime semantics for prompt, session, mention, and delivery behavior.
