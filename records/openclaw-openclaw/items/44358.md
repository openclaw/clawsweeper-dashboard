## Summary

Keep open: current main still leaves canonical OpenClaw `provider/model` refs outside the bundled OpenProse agent/session `model:` grammar, while the core `sessions_spawn` path accepts and forwards arbitrary model strings. The prior issue comment correctly identified the same OpenProse boundary, so the remaining work belongs in the bundled OpenProse plugin/spec/docs surface.

## What This Changes

The issue requests that OpenProse `agent.model` values such as `openai/gpt-5.2` are consistently honored instead of falling back to the default subagent model.

## Best Possible Solution

Update the bundled OpenProse VM/compiler specs and user docs so `model:` accepts canonical OpenClaw `provider/model` refs everywhere existing aliases are accepted, preserve the aliases, and add focused proof that the resolved model string reaches `sessions_spawn.model`.

## Reproduction Assessment

Yes for the actionable mismatch: `model: openai/gpt-5.2` is outside the current OpenProse alias-only grammar, while current core spawn code accepts such strings if forwarded. The reported nondeterministic fallback itself was not live-reproduced from logs or a runnable fixture.

## Solution Assessment

Yes: the narrow maintainable fix is in the bundled OpenProse plugin/spec/docs boundary, not in core subagent model selection. Core already preserves qualified model overrides, so changing fallback behavior broadly would be riskier than fixing the OpenProse model syntax/forwarding contract.
