## Summary

Keep open: current bundled OpenProse still only defines alias models and treats other values as invalid, while the issue lacks deterministic logs or a runnable `.prose` snippet proving the silent fallback path on current `main`. This needs a maintainer decision on whether OpenProse should accept OpenClaw provider/model refs or fail fast more clearly.

## What This Changes

The issue asks OpenProse to consistently honor per-agent `model:` values such as `openai/gpt-5.2` instead of falling back to the default subagent model.

## Best Possible Solution

Set one explicit OpenProse model contract and implement it consistently in `prose.md`, `compiler.md`, examples/docs, and the Task or `sessions_spawn` handoff.

No for the nondeterministic fallback: the issue has no runnable `.prose` file, config, logs, or compile output. Source inspection does confirm that `openai/gpt-5.2` is outside the current bundled OpenProse model grammar.
