## Summary

Keep open: current main still leaves fully-qualified OpenClaw model refs outside the bundled OpenProse grammar and compiler docs, while the core subagent spawn path already accepts arbitrary model strings. The remaining bug is in the OpenProse plugin/spec surface, not core model selection.

## What This Changes

OpenProse should honor per-agent and per-session `model:` values such as `openai/gpt-5.2` instead of falling back to the configured default subagent model.

## Best Possible Solution

Update the bundled OpenProse VM/compiler specs and user docs so `model:` accepts canonical OpenClaw `provider/model` refs everywhere existing aliases are accepted, preserve the aliases, and add a small example or regression artifact showing the resolved model string reaches `sessions_spawn.model`.

## Reproduction Assessment

Partially: the nondeterministic runtime fallback is not directly reproducible from the report, but the central mismatch is statically reproducible because `model: openai/gpt-5.2` is outside the current OpenProse grammar while core `sessions_spawn` would accept that string if forwarded.

## Solution Assessment

Yes: the narrow maintainable solution is to fix the bundled OpenProse plugin/spec/docs boundary and add focused coverage or an example, leaving core subagent model selection unchanged unless a separate runtime gap is found.
