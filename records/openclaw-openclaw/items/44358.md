## Summary

Current main still leaves the reporter's `model: openai/gpt-5.2` example outside the bundled OpenProse grammar/compiler surface, while the core `sessions_spawn` path already accepts and forwards model strings. ## What This Changes

The issue asks for OpenProse agent/session `model:` declarations to reliably honor exact per-agent provider/model selections instead of falling back to the default subagent model.
