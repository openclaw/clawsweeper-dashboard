## Summary

Keep open for maintainer decision. Current main still defines OpenProse `model:` values as the aliases `sonnet`, `opus`, and `haiku`, so the reported fully-qualified provider/model syntax is not supported today; no canonical duplicate or merged fix for this model-contract gap was found.

## What This Changes

The issue asks for OpenProse agent-level `model:` values such as `openai/gpt-5.2` to be honored instead of falling back to the default subagent model.

## Best Possible Solution

Decide the OpenProse contract explicitly: either accept validated OpenClaw model refs in `model:` and document/test that path, or keep aliases only and make unsupported model values fail clearly.

## Reproduction Assessment

No high-confidence deterministic reproduction of the silent fallback is available from the report. Source inspection does show the reported fully-qualified model syntax is outside the current OpenProse grammar.

## Solution Assessment

No. A blind bug fix that accepts any string is not clearly the best path until maintainers decide whether OpenProse should support OpenClaw provider/model refs or remain alias-only with clearer validation.
