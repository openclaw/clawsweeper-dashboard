## Summary

Keep open: current main still leaves fully qualified OpenClaw `provider/model` refs outside the bundled OpenProse `model:` grammar, even though the core `sessions_spawn` path accepts and forwards arbitrary model strings when it receives them. The prior discussion correctly identified the same OpenProse boundary, so this remains a narrow source-reproducible plugin/spec/docs issue.

## What This Changes

The issue asks OpenProse to reliably honor per-agent `model:` values such as `openai/gpt-5.2` instead of falling back to the default subagent model.

## Best Possible Solution

Update the bundled OpenProse VM/compiler specs and user docs so `model:` accepts canonical OpenClaw `provider/model` refs wherever aliases are accepted, preserves existing aliases, and proves the resolved string reaches `sessions_spawn.model`.

## Reproduction Assessment

Yes for the actionable source mismatch: `model: openai/gpt-5.2` is outside the current OpenProse alias-only grammar, while current core spawn/model-selection code accepts and preserves qualified model refs if forwarded. The exact nondeterministic fallback was not live-reproduced from logs or a fixture.

## Solution Assessment

Yes: the narrow maintainable fix is in the bundled OpenProse plugin/spec/docs boundary, not in core subagent model fallback. Core already preserves qualified overrides, so broad fallback changes would be riskier than aligning OpenProse model syntax and forwarding instructions.
