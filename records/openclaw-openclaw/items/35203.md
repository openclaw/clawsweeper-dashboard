## Summary

Keep open for maintainer product/API design. Current main has adjacent isolated subagents, bounded session visibility, agent-to-agent tools, memory plugins, and usage reporting, but it does not implement the RFC's core capability profiling/routing, shared blackboard, layered memory visibility, or pre-call token/cost budget enforcement; related Octo/teleconference work is partial, open, or closed-unmerged rather than a complete replacement.

## What This Changes

Request to add a four-layer multi-agent collaboration stack covering capability profiling, shared blackboard state, layered memory boundaries, and token cost governance.

## Best Possible Solution

Keep this as an umbrella RFC, then split it into focused design issues for capability metrics/routing, artifact or blackboard search/query, memory visibility and promotion gates, token/cost aggregation with pre-call enforcement, and delegation privilege boundaries.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC/enhancement request. Source and docs inspection give a high-confidence verification path that the requested four-layer stack is not implemented on current main.

## Solution Assessment

No, implementing the whole RFC as one change is not the best immediate path. The maintainable direction is staged maintainer design with narrow core/SDK/plugin follow-ups and a security review before shared memory, learned routing, or hard budget enforcement land.
