## Summary

Keep open: current main still has isolated/nested sub-agents, bounded session messaging, memory plugins, context/token visibility, and usage-cost reporting, but it does not implement the RFC’s learned routing, shared blackboard, layered memory visibility, or enforceable multi-agent budget governance. The recent comments add useful production design constraints, and the remaining work is a product/security architecture decision rather than cleanup closure.

## What This Changes

The issue proposes adding capability profiling, shared blackboard coordination, layered memory visibility, and token-cost governance to OpenClaw multi-agent workflows.

## Best Possible Solution

Keep this as the umbrella architecture RFC and split accepted work into scoped design/implementation phases for telemetry, routing, shared coordination storage, memory visibility, and budget admission.
