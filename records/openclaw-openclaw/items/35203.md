## Summary

Keep open for maintainer architecture review: current main has hierarchical sub-agent spawning, isolated/forked context, plugin-owned memory, and usage telemetry, but it does not implement learned routing, a shared blackboard, layered cross-agent memory visibility, or enforceable token/cost budgets. Related open work covers pieces of the RFC, while the full four-layer core stack needs product and security decomposition before any implementation lane is safe.

## What This Changes

The issue requests a broad multi-agent architecture stack covering capability profiling, shared blackboard state, layered memory boundaries, and token cost governance.

## Best Possible Solution

Keep this as an umbrella design discussion, but split implementation into separately reviewed phases for a generic budget admission API, minimal handoff/teleconference primitives, plugin-owned shared knowledge with ACLs, and optional capability routing only after missing core or SDK seams are proven.
