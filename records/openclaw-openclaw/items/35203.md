## Summary

Keep open: this is a broad, active architecture RFC with recent discussion and explicit product/security labels. Current main has adjacent isolated sub-agent runs, usage reporting, memory search, and tool/sandbox policy, but it does not implement learned capability profiles, a shared blackboard/query layer, layered memory visibility, or enforceable multi-agent token/cost governance.

## What This Changes

This RFC requests capability profiling, shared blackboard state, layered memory boundaries, and token-cost governance for multi-agent collaboration.

## Best Possible Solution

Keep this as the umbrella architecture RFC, then split accepted work into scoped phases for telemetry, routing, shared coordination storage/search, memory visibility policy, and budget enforcement.
