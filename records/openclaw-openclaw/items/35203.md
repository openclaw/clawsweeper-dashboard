## Summary

Keep open: current main has isolated sub-agents, bounded agent-to-agent session exchange, memory/search plugins, and usage reporting, but it still lacks the RFC's learned capability profiles, shared blackboard, layered private/team/global memory, and enforceable token/cost budgets. Recent comments and related threads show this is still an active roadmap/design discussion, not stale cleanup.

## What This Changes

Request to add capability profiling, a shared blackboard, layered memory visibility, and token/cost governance to OpenClaw's multi-agent system.

## Best Possible Solution

Keep this as the broad roadmap RFC and split it into maintainer-scoped phases for capability telemetry/routing, a shared artifact or blackboard query API, memory visibility policy, and token/cost enforcement.

## Reproduction Assessment

Not applicable; this is a feature RFC rather than a broken existing behavior report. Source and docs inspection show related partial surfaces, not a current-main failure to reproduce.

## Solution Assessment

No, not as one all-layer implementation. The maintainable path is phased design with explicit product, persistence, privacy, and security boundaries, reusing existing subagent, sessions_send, memory, usage, and agent-to-agent surfaces where they fit.
