## What This Changes

The issue proposes capability-aware agent routing, a shared collaboration blackboard, visibility-layered memory, and enforceable token/cost budgets for multi-agent workflows.

## Best Possible Solution

Retain this as an umbrella design discussion, but require separately scoped proposals for generic usage-budget enforcement, plugin-owned shared knowledge with explicit visibility ACLs, and optional capability routing, reusing current session/plugin seams and canonical SQLite stores rather than adding one core orchestration stack.
