Full review comments:

- **[P1] Preserve systemd exit-78 lock recovery:** `src/cli/gateway-cli/run.ts:568`
  - body: Current main and the latest release use exit 78 for healthy systemd-supervised duplicate gateway starts so `RestartPreventExitStatus=78` stops `Restart=always` loops. This branch exits 0 after a successful probe, which would conflict with that shipped supervisor contract and can reintroduce the loop this path was changed to avoid.
  - confidence: 0.91
- **[P2] Route trusted-proxy probes through a trusted source:** `src/cli/gateway-cli/run.ts:562`
  - body: Passing synthetic trusted-proxy headers is not enough: gateway auth also rejects loopback sources unless `allowLoopback` is explicitly enabled and rejects host-local non-loopback interface sources. In default trusted-proxy deployments this probe will still return unhealthy for a healthy gateway, so the EADDRINUSE restart loop remains for that auth mode.
  - confidence: 0.88
- **[P2] Format IPv6 bind hosts before probing:** `src/cli/gateway-cli/run.ts:169`
  - body: `resolveGatewayBindUrl` formats custom hosts as `scheme://host:port`, so a custom IPv6 host such as `::1` becomes an invalid WebSocket URL and gets filtered out before probing. Bracket IPv6 literals or use URL construction before feeding candidates to `isSecureWebSocketUrl`.
  - confidence: 0.83
