  - body: The patch sends fabricated trusted-proxy headers from the local CLI probe. Current trusted-proxy auth still requires the request source to be a configured trusted proxy and rejects loopback sources unless `allowLoopback` is explicitly enabled, while the docs tell internal Gateway clients to use password auth instead.   - confidence: 0.88
