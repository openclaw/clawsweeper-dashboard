Full review comments:

- **[P2] Do not forge trusted-proxy identity for the probe:** `src/cli/gateway-cli/run.ts:562`
  - body: This sends synthetic trusted-proxy headers from a direct GatewayClient probe, but trusted-proxy auth also requires the socket peer to be a configured trusted source and rejects loopback by default. In common trusted-proxy deployments this still reports a healthy gateway as unhealthy, so the EADDRINUSE restart-loop behavior remains for that mode; use a probe path that matches the server auth contract or cover the rejected cases explicitly.
  - confidence: 0.86
- **[P2] Bracket IPv6 bind hosts before probing:** `src/cli/gateway-cli/run.ts:169`
  - body: `resolveGatewayBindUrl` can return values like `wss://::1:18789` for a custom IPv6 bind host, and this new probe path passes that string directly into the URL filter/client. That invalid URL is dropped and the code falls back to IPv4 loopback, so an IPv6-only existing gateway is still misclassified as unhealthy on EADDRINUSE.
  - confidence: 0.78

  - body: Normal PRs should not edit `CHANGELOG.md`; release generation owns that file. Keep the release-note context in the PR body or squash message instead so the release process can derive the final entry.
