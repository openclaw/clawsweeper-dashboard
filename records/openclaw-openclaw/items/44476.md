## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve trusted-proxy healthy-gateway recovery:** `src/cli/gateway-cli/run.ts:562`
  - body: In trusted-proxy mode this self-probe only injects identity headers, but the server also requires the socket source to match `gateway.trustedProxies` and rejects loopback unless `trustedProxy.allowLoopback` is true. A healthy gateway behind a normal reverse proxy will fail this probe and the new catch path exits 1, reintroducing the supervised restart loop this PR is trying to prevent.
  - confidence: 0.86
- **[P1] Keep LAN/custom plaintext binds from falling back to failure:** `src/cli/gateway-cli/run.ts:155-156`
  - body: Filtering the configured non-loopback `ws://` URL means a healthy plaintext LAN/custom gateway is only probed through `127.0.0.1`. When that listener is not bound on loopback, the probe reports unhealthy and startup exits 1 even though the existing OpenClaw gateway is healthy.
  - confidence: 0.82
- **[P2] Bracket IPv6 bind hosts before probing:** `src/cli/gateway-cli/run.ts:168-170`
  - body: The bind-aware probe passes `resolveGatewayBindUrl()` output through unchanged, but custom IPv6 hosts are emitted like `ws://::1:<port>`, which is not a valid URL and gets filtered out. An already-healthy IPv6-bound gateway can therefore be misclassified as unhealthy on EADDRINUSE.
  - confidence: 0.78
