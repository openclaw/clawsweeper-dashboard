  - body: Current main documents and tests healthy systemd duplicate gateway conflicts as exit 78 so `RestartPreventExitStatus=78` stops `Restart=always` loops.   - body: In trusted-proxy mode this self-probe only injects identity headers, but the server also requires the socket source to match `gateway.trustedProxies`. A healthy gateway behind a normal reverse proxy can fail this probe and the catch path exits 1, reintroducing the restart loop this PR is trying to prevent.
  - confidence: 0.86
- **[P1] Keep plaintext LAN/custom binds reachable:** `src/cli/gateway-cli/run.ts:155-156`
  - body: Filtering configured non-loopback `ws://` bind URLs on this PR head means a healthy gateway bound only to `gateway.bind=lan` or `customBindHost` may not be probed unless the break-glass env var is set; the loopback fallback will not reach that listener, so startup can still exit 1 on EADDRINUSE.
  - confidence: 0.8

  - confidence: 0.76
