## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve the systemd exit-78 path:** `src/cli/gateway-cli/run.ts:568`
  - body: This exits 0 whenever the new probe finds a healthy listener, but current main and the generated systemd unit use `Restart=always` with `RestartPreventExitStatus=78`. Returning 0 for the systemd duplicate-start case can restart again instead of hitting the prevent-exit status, so the rebase needs to keep systemd on exit 78 while only changing any intentionally supported non-systemd path.
  - confidence: 0.9
- **[P2] Do not authenticate trusted-proxy probes with local identity headers:** `src/cli/gateway-cli/run.ts:562`
  - body: The probe dials the gateway directly and only injects trusted-proxy headers, but current trusted-proxy auth rejects loopback sources by default and rejects sources outside `gateway.trustedProxies`. For internal same-host callers the documented path is the local token/password fallback, so this still misclassifies many trusted-proxy deployments as unhealthy.
  - confidence: 0.86
- **[P2] Bracket IPv6 bind hosts before probing:** `src/cli/gateway-cli/run.ts:169`
  - body: `resolveGatewayBindUrl` formats a custom IPv6 host like `::1` as `ws://::1:<port>`, and this helper passes that string straight into the URL security filter. That invalid URL is dropped and the probe falls back to IPv4 loopback, so an already-running gateway bound only to IPv6 loopback can still be treated as unhealthy.
  - confidence: 0.8
