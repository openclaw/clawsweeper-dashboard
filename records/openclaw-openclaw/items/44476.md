Full review comments:

- **[P1] Preserve systemd exit-78 semantics:** `src/cli/gateway-cli/run.ts:568`
  - body: Current main documents and tests healthy systemd duplicate-gateway lock conflicts as exit 78 so `RestartPreventExitStatus=78` stops `Restart=always` loops. This PR exits 0 after a successful probe, so the rebased fix needs to keep systemd-specific exit behavior or carry an explicit maintainer-approved policy change.
  - confidence: 0.88
- **[P1] Probe trusted-proxy listeners through a trusted source:** `src/cli/gateway-cli/run.ts:562`
  - body: In trusted-proxy mode this path injects identity headers, but the server also requires the socket source to match `gateway.trustedProxies`. A healthy gateway behind a normal reverse proxy can fail this direct self-probe and still exit 1 on EADDRINUSE.
  - confidence: 0.86
- **[P1] Keep LAN and custom plaintext binds reachable:** `src/cli/gateway-cli/run.ts:155-156`
  - body: Filtering non-loopback `ws://` bind URLs on this PR head means a healthy gateway bound only to a LAN or custom host may not be probed unless the break-glass env var is set; the loopback fallback often cannot reach that listener, so the restart-loop symptom remains.
  - confidence: 0.8
- **[P2] Bracket IPv6 custom bind URLs before probing:** `src/cli/gateway-cli/run.ts:168-169`
  - body: The PR consumes `resolveGatewayBindUrl` output directly, but a custom IPv6 host like `::1` becomes `ws://::1:<port>`, which is not a valid URL and is filtered out before probing. An IPv6-only healthy listener would still be treated as unhealthy.
  - confidence: 0.76
