- **[P1] Preserve systemd exit-78 semantics:** `src/cli/gateway-cli/run.ts:568`
  - body: Current main documents and tests healthy systemd lock conflicts as exit 78 so RestartPreventExitStatus=78 stops Restart=always loops. This new successful-probe path exits 0 for every supervisor, so a rebased version needs to keep the systemd-specific exit code or document an explicit maintainer-approved policy change.
  - confidence: 0.86
- **[P1] Preserve trusted-proxy healthy-gateway recovery:** `src/cli/gateway-cli/run.ts:562`
  - body: In trusted-proxy mode this self-probe only injects identity headers, but the server also requires the socket source to match gateway.trustedProxies and rejects loopback unless trustedProxy.allowLoopback is true. A healthy gateway behind a normal reverse proxy will fail this probe and the new catch path exits 1, reintroducing the supervised restart loop this PR is trying to prevent.
  - confidence: 0.86
- **[P1] Keep LAN/custom plaintext binds from falling back to failure:** `src/cli/gateway-cli/run.ts:155-156`
  - body: Filtering the configured non-loopback ws:// bind URL means a healthy gateway bound only to gateway.bind=lan or customBindHost is not probed on this PR head unless the break-glass env var is set; the remaining loopback fallback will not reach that listener. That changes existing port-conflict behavior into exit 1 for a valid running gateway.
  - confidence: 0.8
- **[P2] Bracket IPv6 bind hosts before probing:** `src/cli/gateway-cli/run.ts:168-170`
  - body: resolveGatewayBindUrl emits custom hosts directly, so customBindHost: "::1" becomes an invalid ws://::1:<port> candidate and is filtered out before the probe runs. For an IPv6-only loopback bind, the fallback to 127.0.0.1 cannot confirm the healthy gateway and startup exits non-zero on EADDRINUSE.
  - confidence: 0.76
