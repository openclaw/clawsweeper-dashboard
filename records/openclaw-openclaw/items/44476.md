Overall correctness: patch is incorrect

Full review comments:

- **[P2] Probe trusted-proxy listeners through a trusted source:** `src/cli/gateway-cli/run.ts:562`
  - body: Passing synthetic trusted-proxy headers is not enough when the direct probe's socket source is not listed in `gateway.trustedProxies`; `authorizeTrustedProxy` rejects on `remoteAddress` before those headers can authenticate, so non-loopback proxy deployments still exit non-zero on `EADDRINUSE`.
  - confidence: 0.87
- **[P2] Bracket IPv6 custom bind hosts before probing:** `src/cli/gateway-cli/run.ts:169`
  - body: For `gateway.bind=custom` with an IPv6 host like `::1`, this uses the advertised bind URL as `wss://::1:<port>`, which is not parseable as a WebSocket URL and gets skipped, so a healthy IPv6-bound gateway is treated as unhealthy.
  - confidence: 0.84
