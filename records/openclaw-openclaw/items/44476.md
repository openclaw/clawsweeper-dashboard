Full review comments:

- **[P1] Preserve systemd exit-78 semantics:** `src/cli/gateway-cli/run.ts:568`
  - body: Current main documents and tests healthy systemd duplicate starters as exit 78 so `RestartPreventExitStatus=78` stops `Restart=always` loops. This successful-probe path exits 0 for every supervisor, so a rebased version needs to keep the systemd-specific exit code or carry an explicit maintainer-approved policy change with tests.
  - confidence: 0.87
- **[P1] Probe trusted-proxy listeners through a trusted source:** `src/cli/gateway-cli/run.ts:562`
  - body: In trusted-proxy mode this self-probe injects identity headers, but the server also requires the socket source to match `gateway.trustedProxies`. A healthy gateway behind a normal reverse proxy can fail this direct probe and the catch path exits 1, reintroducing the restart loop this PR is trying to prevent.
  - confidence: 0.86
- **[P2] Keep plaintext LAN/custom binds reachable after rebase:** `src/cli/gateway-cli/run.ts:155-156`
  - body: On this PR head, filtering non-loopback `ws://` bind URLs through the older strict policy means a healthy gateway bound only to LAN/custom plaintext may not be probed unless the break-glass env var is set. The rebase should preserve current main's trusted private-IP behavior or otherwise prove those listeners remain reachable.
  - confidence: 0.78
- **[P2] Bracket IPv6 bind hosts before probing:** `src/cli/gateway-cli/run.ts:168-169`
  - body: `resolveGatewayBindUrl` emits custom hosts directly, so `customBindHost: "::1"` becomes an invalid `ws://::1:<port>` candidate and is filtered out before the probe runs. For an IPv6-only loopback bind, the fallback to `127.0.0.1` cannot confirm the healthy gateway.
  - confidence: 0.76
