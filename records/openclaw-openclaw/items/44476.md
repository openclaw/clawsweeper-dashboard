Overall confidence: 0.84

Full review comments:

- **[P2] Keep trusted-proxy probes on a valid auth path:** `src/cli/gateway-cli/run.ts:562`
  - body: Passing synthetic headers is not enough for trusted-proxy auth: the server also requires the socket source to be in `gateway.trustedProxies`, and loopback sources require `allowLoopback`. In common deployments that trust only the reverse proxy IP, this self-probe still reports a healthy listener as unhealthy, so the EADDRINUSE loop remains for an auth mode the PR claims to support.
  - confidence: 0.86
- **[P2] Bracket IPv6 custom bind hosts before probing:** `src/cli/gateway-cli/run.ts:169`
  - body: This adds the custom bind URL returned by `resolveGatewayBindUrl()` directly, but that helper emits hosts as `scheme://host:port`. For an IPv6 custom bind such as `::1`, the result is not a valid URL and gets filtered out, leaving only the IPv4 loopback fallback and misclassifying an IPv6-only healthy gateway as unhealthy.
  - confidence: 0.82
