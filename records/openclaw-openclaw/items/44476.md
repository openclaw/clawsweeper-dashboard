Full review comments:

- **[P2] Do not treat forged proxy headers as a valid probe:** `src/cli/gateway-cli/run.ts:562`
  - body: This passes trusted-proxy headers into the probe, but trusted-proxy auth also checks that `req.socket.remoteAddress` is in `gateway.trustedProxies`. For the common case where the trusted proxy is a different host, a direct local probe still returns `trusted_proxy_untrusted_source`, so a healthy gateway is treated as unhealthy and the EADDRINUSE restart-loop behavior remains.
  - confidence: 0.87
- **[P2] Bracket IPv6 bind hosts before probing:** `src/cli/gateway-cli/run.ts:169`
  - body: The bind URL helper can return values like `wss://::1:18789` for `gateway.bind=custom`, and this path passes that string directly into the URL security filter. That is not a valid URL, so the actual IPv6 listener is skipped and the code falls back to IPv4 loopback, misclassifying a healthy IPv6-only gateway as unhealthy.
  - confidence: 0.78
- **[P3] Remove the direct changelog edit:** `CHANGELOG.md:64`
  - body: Normal PRs should not edit `CHANGELOG.md`; release generation owns that file. Please keep the release-note context in the PR body or eventual squash/merge message instead of adding this entry directly.
