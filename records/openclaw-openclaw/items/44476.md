- **[P1] Preserve systemd exit-78 semantics:** `src/cli/gateway-cli/run.ts:568`
  - body: Current main documents and tests systemd duplicate-start conflicts as exit 78 so `RestartPreventExitStatus=78` stops `Restart=always` loops. This PR exits 0 after any successful probe, so a rebased version would undo that supervisor contract unless it keeps systemd-specific exit handling.

- **[P1] Probe trusted-proxy listeners through a trusted source:** `src/cli/gateway-cli/run.ts:562`
  - body: In trusted-proxy mode this self-probe only injects identity headers, but the server also requires the socket source to match `gateway.trustedProxies`. A healthy gateway behind a normal reverse proxy can fail this probe and the catch path exits 1, reintroducing the restart-loop behavior this PR is trying to prevent.
  - confidence: 0.86
- **[P1] Keep plaintext LAN and custom binds reachable:** `src/cli/gateway-cli/run.ts:155-156`
  - body: On this stale head, filtering configured non-loopback `ws://` bind URLs through the loopback-only WebSocket policy means a healthy gateway bound only to LAN/custom can be skipped unless the break-glass env var is set; the loopback fallback will not reach that listener.
  - confidence: 0.8
- **[P2] Bracket IPv6 bind hosts before probing:** `src/cli/gateway-cli/run.ts:168-170`
  - body: `resolveGatewayBindUrl` emits custom hosts directly, so `customBindHost: "::1"` becomes an invalid `ws://::1:<port>` candidate and is filtered out before probing. For an IPv6-only loopback bind, the fallback to `127.0.0.1` cannot confirm the healthy gateway.
  - confidence: 0.76
