## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve systemd exit-78 recovery:** `src/cli/gateway-cli/run.ts:568`
  - body: This branch exits 0 whenever the probe succeeds, but current main and v2026.6.6 use exit 78 for healthy systemd lock conflicts so `RestartPreventExitStatus=78` stops `Restart=always`. A rebase must preserve that shipped systemd path or the EADDRINUSE race can loop again under systemd.
  - confidence: 0.88
- **[P2] Probe trusted-proxy through a trusted source:** `src/cli/gateway-cli/run.ts:562`
  - body: Passing synthetic trusted-proxy headers does not make this direct local connection trusted when `gateway.trustedProxies` names a separate reverse proxy host. `authorizeTrustedProxy` still checks the socket source address, so those deployments will keep reporting the existing gateway as unhealthy and continue the restart-loop behavior.
  - confidence: 0.84
- **[P2] Bracket IPv6 bind hosts before probing:** `src/cli/gateway-cli/run.ts:169`
  - body: `resolveGatewayBindUrl` can return an unbracketed IPv6 custom host such as `ws://::1:18789`, and this code feeds that string directly into the URL/security filter. An IPv6-only healthy listener then gets skipped and the loopback fallback cannot prove it healthy.
  - confidence: 0.78
