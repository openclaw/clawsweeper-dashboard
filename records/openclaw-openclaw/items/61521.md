## Summary

Keep this PR open, but it still needs changes before merge: current main does not include the requested complete/async channel setup validation seam, and the proposed branch has blocking deferred-candidate/account-id bugs plus a current-main CLI setup-option mismatch. The external real behavior proof gate is also not satisfied yet.

## What This Changes

The PR adds complete/async channel setup validation hooks, defers wizard and non-interactive channel config writes until candidate validation, adds VK-oriented setup input/CLI fields, and adds a focused wizard seam test.

## Best Possible Solution

Keep the deferred validation seam only if maintainers want this SDK contract, but rebase it onto current main, build and validate one full candidate before write, propagate the resolved account id consistently, keep plugin-specific flags in plugin metadata, and add focused regression coverage plus real behavior proof.

## Solution Assessment
