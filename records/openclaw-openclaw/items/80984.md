## What This Changes

Add a typed, actionable cron delivery error for `delivery.channel: "last"` or omitted channel when the selected session has no prior chat route.

## Best Possible Solution

Add a cron-side typed no-prior-chat route error and preview/list wording for `last` without session history, while preserving provider-prefixed target selection, session `lastChannel`, and single-configured-channel fallback.

## Reproduction Assessment

Yes, source-level. A cron announce target with omitted/`last` channel, no selected session `lastChannel`, and no resolvable fallback still flows through generic channel selection before returning a wrapped generic error.

## Solution Assessment

Yes, with a small seam adjustment. The maintainable fix is to detect the session-state-specific no-prior-chat condition in cron delivery resolution and emit a typed actionable error there, rather than mixing session history semantics into generic config/plugin channel selection.
