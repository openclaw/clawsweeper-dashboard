## What This Changes

The issue requests a typed, actionable cron delivery error for `delivery.channel="last"` or omitted channel when the selected session has no prior chat route and no resolvable fallback channel.

## Best Possible Solution

Add a narrow cron-side typed no-prior-chat diagnostic for unresolved implicit `last` delivery, and surface that actionable message in preview/list output while preserving provider-prefixed targets, explicit channels, and single-channel fallback behavior.

## Reproduction Assessment

Yes, source-level: with `delivery.channel` omitted or set to `last`, no session `lastChannel`, and no resolvable fallback, `resolveDeliveryTarget` still calls generic channel selection and exposes the wrapped selector error through cron preview/list output. ## Solution Assessment

Yes, the requested direction is reasonable if implemented at the cron delivery seam. The maintainable fix is to distinguish the session-history failure before returning generic config/channel-selection errors, not to overload the generic selector with session-store semantics.
