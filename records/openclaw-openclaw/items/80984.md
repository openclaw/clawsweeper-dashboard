## Summary

Keep open: current main and the latest shipped release still lack the requested typed no-prior-chat cron delivery error, and the `last`/empty-session path still reaches generic channel selection before surfacing a wrapped generic error in cron preview/run output.

## What This Changes

Request to add a typed, actionable cron delivery error for `delivery.channel: "last"` or omitted channel when the selected session has no prior chat route and no fallback channel can resolve.

## Best Possible Solution

Add a narrow cron-side typed no-prior-chat route error and preview/list message for `last` without session history, while preserving configured-channel fallback, provider-prefixed targets, and explicit target resolution.

## Reproduction Assessment

Yes, source-level. A cron announce target with omitted or `last` channel, no selected session `lastChannel`, no explicit target, and no resolvable fallback still reaches generic channel selection and returns the wrapped selector error.

## Solution Assessment

Yes, with a seam adjustment. The maintainable fix is to detect the session-state-specific no-prior-chat condition in `resolveDeliveryTarget`; putting it only in generic channel selection would mix config/plugin selection with session history semantics.
