## Summary

Keep open. Current main still has adjacent `openclaw agent --to --deliver` and `sessions_send` A2A behavior, but it does not implement the requested meeting lifecycle, teleconference/meet command surface, durable meeting store, or transcript verb; the related handoff PRs remain open and one explicitly excludes the meeting lifecycle.

## What This Changes

The issue requests a new agent teleconference feature with dial/pickup/send/wrap/list/show/transcript verbs, durable pending/active/closed meeting state, visible channel frames, and optional agent icons.

## Best Possible Solution

Keep this RFC open and converge on a minimal lifecycle layer above the handoff/session primitives: stable meeting ids, atomic pending/active/closed transitions, visible frames, durable transcript output, docs, and focused tests, with broader orchestration deferred.

## Reproduction Assessment

Not applicable; this is a feature RFC rather than a bug report. The current-main gap is source-verifiable because adjacent session/send primitives exist, but no scoped meeting lifecycle or teleconference command surface is present.

## Solution Assessment

Yes for the maintainer-narrowed Phase 1 direction, but not as the full multi-phase RFC in one change. The maintainable path is a minimal lifecycle layer that composes with the handoff work or clearly documents a temporary fallback until that contract lands.
