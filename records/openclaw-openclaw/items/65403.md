## Summary

Keep open. Current main has adjacent agent delivery, subagent spawning, and bounded agent-to-agent `sessions_send` exchange, but it still lacks the requested teleconference command surface, meeting lifecycle records, and durable meeting transcript primitive; the maintainer comment also narrows this toward native OpenClaw lifecycle work above #63789 rather than a cleanup close.

## What This Changes

This issue proposes a scoped agent teleconference primitive with dial/pickup/send/wrap/list/show/transcript CLI verbs, visible channel frames, and pending/active/closed meeting persistence.

## Best Possible Solution

Land a small native lifecycle layer over the existing handoff/session primitives: stable meeting ids, dial/pickup/send/wrap/list/show/transcript verbs, atomic state transitions, visible channel frames, durable transcript output, and focused docs/tests, while leaving broader orchestration and discovery to separate trackers.

## Reproduction Assessment

Not applicable as a feature/RFC rather than a bug report. The current-main gap is source-verifiable: code and docs expose adjacent agent/session primitives but no scoped meeting lifecycle surface.

## Solution Assessment

Yes for the maintainer-narrowed direction, but not as a broad all-phases implementation. The maintainable path is a minimal lifecycle layer that composes with #63789 or clearly documents a temporary fallback.
