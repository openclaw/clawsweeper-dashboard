## Summary

Keep open. Current main still has adjacent one-turn agent delivery, subagent spawning, bounded sessions_send ping-pong, and allowlisted agent-to-agent access, but it does not implement the requested meeting lifecycle, teleconference CLI verbs, pending/active/closed records, or durable meeting transcript surface. The maintainer comment narrows this toward a native OpenClaw lifecycle layer above #63789 rather than closing it as already covered.

## What This Changes

Request a minimum-viable agent teleconference primitive with dial/pickup/send/wrap/list/show/transcript verbs, explicit meeting lifecycle, visible channel frames, and durable transcript artifacts.

## Best Possible Solution

Land a small native lifecycle layer over the existing handoff/session primitives: stable meeting ids, dial/pickup/send/wrap/list/show/transcript verbs, atomic state transitions, durable transcript output, visible channel frames, and focused tests/docs, with broader orchestration and discovery phases left to separate trackers.

## Reproduction Assessment

Not applicable as a feature/RFC rather than a bug report. The current-main gap is source-verifiable: code and docs expose adjacent agent/session primitives but no scoped meeting lifecycle surface.

## Solution Assessment

Yes for the maintainer-narrowed direction, not for the full broad RFC at once. The maintainable path is a minimal lifecycle layer that composes with #63789 or clearly documents a temporary fallback.
