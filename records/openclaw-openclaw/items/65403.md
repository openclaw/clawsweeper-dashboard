## Summary

Keep this open: current main and v2026.6.8 still provide lower-level `openclaw agent --deliver`, `sessions.send`, and `sessions_send` A2A reply-loop primitives, but not the requested meeting lifecycle, dial/pickup/send/wrap command surface, durable meeting artifact, visible meeting frames, or transcript verb. The issue remains the scoped RFC that related threads point back to, and implementation needs maintainer product/API decisions rather than stale cleanup.

## What This Changes

Request a new agent teleconference primitive with dial/pickup/send/wrap/list/show/transcript verbs, durable meeting lifecycle state, visible channel frames, and transcript output.

## Best Possible Solution

Keep this as the canonical scoped meeting-lifecycle RFC, then accept a narrow implementation only after maintainers choose the storage model, handoff boundary, transcript privacy rules, and whether the surface belongs in core or a plugin/API extension.

## Reproduction Assessment

Not applicable: this is an RFC for a new meeting primitive, not a report of broken existing behavior. Source and release inspection confirm the requested lifecycle is absent rather than failing at runtime.

## Solution Assessment

Unclear: a minimal lifecycle layer over current delivery/session primitives is plausible, but the proposed sidecar storage, config field, transcript semantics, and handoff boundary are not yet confirmed as the best OpenClaw shape.
