## Summary

Keep open. Current main still has adjacent one-turn agent delivery and `sessions_send` A2A behavior, but it does not implement the requested scoped meeting lifecycle, durable meeting store, transcript command, or teleconference command surface; live maintainer discussion also keeps this as a distinct lifecycle layer above the handoff proposal.

## What This Changes

The issue requests a scoped agent teleconference lifecycle with dial, pickup, send, wrap, list, show, transcript, durable meeting artifacts, visible frames, and optional agent icon configuration.

## Best Possible Solution

Land a small native lifecycle layer above the handoff/session primitives with stable meeting ids, atomic state transitions, visible frames, durable transcript output, docs, and focused tests, while leaving external A2A interop and broad orchestration out of the first change.

## Reproduction Assessment

Not applicable; this is a feature RFC rather than a bug report. The current-main gap is source-verifiable: adjacent agent/session primitives exist, but no scoped meeting lifecycle or teleconference command surface is present.

## Solution Assessment

Yes for the maintainer-narrowed direction, but not as the full multi-phase RFC in one change. The maintainable path is a minimal lifecycle layer that composes with https://github.com/openclaw/openclaw/pull/63789 or clearly documents a temporary fallback until that contract lands.
