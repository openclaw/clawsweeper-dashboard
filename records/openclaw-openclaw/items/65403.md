## Summary

Keep open. Current main still has adjacent `openclaw agent --deliver`, `sessions_send`, and bounded A2A reply-back behavior, but not the requested meeting lifecycle, command surface, persistent meeting records, or transcript primitive; maintainer discussion also keeps this as the lifecycle layer above the still-open handoff proposal.

## What This Changes

The issue proposes a native agent teleconference primitive with dial/pickup/send/wrap/list/show/transcript verbs, meeting lifecycle storage, visible channel frames, and durable transcripts.

## Best Possible Solution

Land a small native lifecycle layer above the handoff/session primitives with stable meeting ids, atomic state transitions, visible frames, durable transcript output, focused docs, and tests, while keeping broad orchestration, discovery, and cost governance in separate tracks.

## Reproduction Assessment

Not applicable; this is a feature RFC rather than a bug report. The current-main gap is source-verifiable: adjacent agent/session primitives exist, but no scoped meeting lifecycle or teleconference command surface is present.

## Solution Assessment

Yes for the maintainer-narrowed direction, but not as the all-phases RFC in one change. The maintainable solution is a minimal lifecycle layer that composes with the handoff contract or documents a temporary fallback until that contract lands.
