## Summary

Keep open: current main and the latest release still provide lower-level `agent --to --deliver`, `sessions_send`, and bounded agent-to-agent reply loops, but not the requested meeting lifecycle, pickup queue, durable meeting artifact, visible meeting frames, or transcript command. The RFC is concrete, but it needs maintainer product/security decisions before implementation.

## What This Changes

This issue requests a scoped agent teleconference primitive with dial/pickup/send/wrap/list/show/transcript verbs, durable meeting state, visible channel frames, and transcript artifacts.

## Best Possible Solution

Keep this as the scoped meeting-lifecycle RFC; any implementation should settle API shape, storage ownership, transcript privacy, authorization, and handoff layering before adding core verbs.
