## Summary

Keep open. Current main has related overflow recovery, memory flush, transcript rotation, and session inspection pieces, but it still lacks the requested one-time promote-then-rotate lifecycle suggestion with dry-run promotion candidates and operator confirmation.

## What This Changes

Request to detect repeated session context overflow and surface an opt-in promote-then-rotate workflow using dry-run memory promotion candidates before archiving and rotating the session.

## Best Possible Solution

Define a lifecycle suggestion contract that tracks repeated overflow pressure per session, emits one deduped structured suggestion with a stable id and candidate count, exposes it in CLI/UI surfaces, and requires explicit operator confirmation before promotion or rotation.

## Reproduction Assessment

Yes. The existing overflow loop test is a high-confidence simulation path for repeated context overflow, and focused source search confirms the requested suggestion/action surface is still absent. I did not run tests because this review was required to stay read-only.

## Solution Assessment

Unclear pending maintainer design approval. The suggestion-and-confirmation direction is safer than automatic promotion or rotation, but it should be designed around current memory flush behavior and the open memory review-quality concerns.
