## Summary

Keep open. Current main still has overflow diagnostics, capped auto-compaction, memory promotion preview/apply, and transcript archive/delete primitives, but it does not wire repeated overflow pressure into a deduped promote-then-rotate suggestion with dry-run candidates and operator confirmation.

## What This Changes

Request a lifecycle suggestion that detects repeated session context overflow, dry-runs memory promotion candidates, and offers operator-confirmed promote/rotate actions.

## Best Possible Solution

Approve a lifecycle design that tracks repeated per-session overflow pressure, emits one deduped structured suggestion with a stable id and candidate count, and requires explicit confirmation before promotion or rotation.

## Reproduction Assessment

Yes. `src/agents/pi-embedded-runner/run.overflow-compaction.loop.test.ts` provides a high-confidence simulation of repeated context overflow, and source inspection confirms current main lacks the requested promote/rotate suggestion branch.

## Solution Assessment

Unclear pending maintainer design. A suggestion-and-confirmation flow is safer than unconditional promotion or rotation, but it should be reconciled with existing memory flush behavior and related memory review-quality issues.
