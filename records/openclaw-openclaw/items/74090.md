## Summary

Keep open. Current main still has overflow diagnostics, automatic compaction, optional transcript rotation, memory promotion preview/apply, and reset/session primitives, but it does not connect repeated overflow pressure to a deduped promote-then-rotate lifecycle suggestion or operator confirmation flow.

## What This Changes

Add an operator-visible promote-then-rotate suggestion with dry-run memory promotion candidates when a long-lived session repeatedly trips context overflow.

## Best Possible Solution

Approve a lifecycle design first, then add a generic per-session overflow-pressure tracker that emits one deduped structured suggestion with a stable id and dry-run candidate count, exposes it in operator-visible CLI/UI surfaces, and requires explicit confirmation before promotion or session rotation.

## Reproduction Assessment

Yes. The current overflow loop test provides a high-confidence simulation path: repeated context overflows compact and retry until a terminal `context_overflow` result, and targeted search confirms no promote-and-rotate suggestion surface exists. I did not run tests because this review was required to stay read-only.

## Solution Assessment

Unclear pending maintainer design approval. The requested suggestion-and-confirmation model is safer than automatic promotion/rotation, but the maintainable fix should define a generic lifecycle suggestion contract coordinated with memory review safeguards.
