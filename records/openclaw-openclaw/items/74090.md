## Summary

Keep open. Current main has adjacent overflow recovery, pre-compaction memory flush, transcript rotation, and manual memory promotion primitives, but it still does not implement the requested one-time promote-then-rotate lifecycle suggestion with dry-run candidates and operator confirmation.

## What This Changes

Request a one-time operator suggestion to preview memory promotion candidates and optionally promote or rotate when repeated session context overflow occurs.

## Best Possible Solution

Approve a lifecycle design that tracks repeated per-session overflow pressure, emits one deduped structured suggestion with a stable id and candidate count, exposes it in CLI/UI surfaces, and requires explicit confirmation before promotion or rotation.

## Reproduction Assessment

Yes. The current overflow loop test provides a high-confidence simulation path for repeated context overflow, and source inspection confirms the requested lifecycle suggestion and promote/rotate action surface is absent; no tests were run because this review was read-only.

## Solution Assessment

Unclear pending maintainer design approval. A suggestion-and-confirmation flow is safer than automatic promote/rotate, but it should be designed around the existing memory flush behavior and the open memory promotion review-quality concerns.
