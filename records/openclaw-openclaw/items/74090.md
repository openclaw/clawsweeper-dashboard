## Summary

Keep open. Current main still has overflow diagnostics, bounded auto-compaction, memory-promotion CLI preview/apply, and transcript rotation primitives, but it does not connect repeated overflow pressure to a deduped promote-then-rotate suggestion with dry-run candidates and explicit operator confirmation.

## What This Changes

The issue requests a lifecycle suggestion that detects repeated session context overflow and prompts operators to dry-run memory promotion, then optionally promote and rotate the session.

## Best Possible Solution

Approve a lifecycle design that tracks repeated per-session overflow pressure, emits one deduped structured suggestion with a stable id and candidate count, exposes it in operator surfaces, and requires explicit confirmation before promotion or rotation.

## Reproduction Assessment

Yes. The existing overflow-loop test gives a high-confidence source simulation for repeated context overflow, and source inspection shows the requested promote/rotate suggestion is absent from that recovery path.

## Solution Assessment

Unclear pending maintainer design approval. A suggestion-and-confirmation flow is safer than unconditional promotion or rotation, but it should be reconciled with existing memory-flush behavior and the open memory review-quality issues.
