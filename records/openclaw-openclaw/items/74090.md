## Summary

Keep open. Current main still has overflow diagnostics, capped auto-compaction, memory promotion preview/apply, memory flush, and transcript rotation primitives, but it does not connect repeated overflow pressure to a deduped promote-then-rotate suggestion with dry-run candidates and operator confirmation.

## What This Changes

Request that repeated session context overflows surface a one-time promote-then-rotate lifecycle suggestion with dry-run memory candidates and opt-in execution.

## Best Possible Solution

Define a maintainer-approved lifecycle design that records repeated per-session overflow pressure, emits one deduped structured suggestion with a dry-run candidate count, exposes it in CLI/UI surfaces, and keeps promotion/rotation operator-confirmed by default.

## Reproduction Assessment

Yes. The existing overflow loop test gives a high-confidence simulation path for repeated context overflow, and source/search inspection confirms the requested promote/rotate lifecycle suggestion is absent on current main.

## Solution Assessment

Unclear pending maintainer design approval. A suggestion-and-confirmation flow is safer than unconditional promotion or rotation, but it should be reconciled with existing memory flush behavior and the related memory review-quality issues.
