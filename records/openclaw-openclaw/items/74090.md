## Summary

Keep open. Current main has the overflow diagnostic, capped compaction retries, memory-promotion preview/apply, pre-compaction memory flush, and transcript-rotation primitives, but it still lacks a deduped promote-then-rotate lifecycle suggestion with dry-run candidates and operator confirmation.

## What This Changes

Request to surface a one-time promote-then-rotate suggestion after repeated session context overflow, with dry-run memory candidates and opt-in promote/rotate actions.

## Best Possible Solution

Approve a lifecycle design that tracks repeated per-session overflow pressure, precomputes promotion candidates without writing, surfaces one deduped suggestion in logs and operator surfaces, and requires explicit confirmation before any promotion or rotation.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection and the existing overflow loop test do give high-confidence proof of the current gap: repeated overflow still ends in capped compaction and context_overflow guidance only.

## Solution Assessment

Unclear pending maintainer design approval. A suggestion-and-confirmation flow is safer than unconditional promotion or rotation, but it should be reconciled with existing memoryFlush, truncateAfterCompaction, and the open memory review-quality issues.
