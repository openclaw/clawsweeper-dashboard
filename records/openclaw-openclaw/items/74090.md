## Summary

Keep open. Current main has the overflow diagnostic, bounded compaction retries, memory promotion preview/apply primitives, and transcript-rotation primitives, but it still does not wire repeated overflow pressure into a deduped promote-then-rotate lifecycle suggestion or operator confirmation flow.

## What This Changes

The issue requests an operator-visible lifecycle suggestion that previews memory-promotion candidates and offers promote/rotate actions after repeated session context-overflow pressure.

## Best Possible Solution

Approve a lifecycle design first: persist per-session overflow pressure, emit a deduped structured suggestion with a stable id and dry-run candidate count, surface it through CLI/UI operator flows, and require explicit confirmation before either memory promotion or session rotation.

## Reproduction Assessment

Yes. The existing overflow loop test provides a high-confidence simulated path for repeated context overflow, and current source inspection shows that path compacts/retries or returns `context_overflow` without a promote-and-rotate suggestion; tests were not executed because this review was read-only.

## Solution Assessment

Unclear pending maintainer design approval. The issue’s opt-in suggestion and confirmation model is safer than automatic promotion/rotation, but the persistence window, candidate source, action semantics, and main-session rotation behavior need to be settled first.
