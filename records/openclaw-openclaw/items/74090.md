## Summary

Keep open. Current main still has overflow diagnostics, capped auto-compaction, memory promotion preview/apply, and session reset/archive primitives, but it does not wire repeated overflow pressure into a deduped promote-then-rotate lifecycle suggestion with dry-run candidates and operator confirmation.

## What This Changes

Add an opt-in lifecycle suggestion that detects repeated session context overflows, previews memory promotion candidates, and lets operators confirm promotion plus session rotation.

## Best Possible Solution

Approve a lifecycle design that tracks repeated per-session overflow pressure, emits one deduped structured suggestion with a stable id and candidate count, exposes it in CLI/UI surfaces, and requires explicit confirmation before promotion or rotation.

## Reproduction Assessment

Yes. The current overflow loop test provides a high-confidence simulation path for repeated context overflow, and source inspection confirms the requested promote/rotate lifecycle suggestion is absent; no tests were run because this review was read-only.

## Solution Assessment

Unclear pending maintainer design approval. A suggestion-and-confirmation flow is safer than automatic promote/rotate, but it should be aligned with existing memory flush behavior and the open memory quality concerns in #66287, #67363, and #72887.
