## Summary

Keep open. Current main has the overflow diagnostics, bounded auto-compaction retry loop, manual memory promotion preview/apply, and session reset/archive primitives, but it still does not connect repeated overflow pressure to a deduped promote-then-rotate lifecycle suggestion or operator confirmation flow.

## What This Changes

Request to surface a one-time promote-then-rotate operator suggestion, with dry-run memory promotion candidates and confirmation actions, when the same session repeatedly trips context overflow pressure.

## Best Possible Solution

Approve a lifecycle design first, then add a generic per-session overflow-pressure tracker that emits one deduped structured suggestion with a stable id and dry-run candidate count, exposes it in operator-visible CLI/UI surfaces, and requires explicit confirmation before promotion or session rotation.

## Reproduction Assessment

Yes. Current source plus the overflow loop test provide a high-confidence simulation path: repeated context overflows compact and retry until a terminal `context_overflow` error, with no promote-and-rotate suggestion. I did not execute tests because this review was required to stay read-only.

## Solution Assessment

Unclear pending maintainer design approval. The issue's suggestion-and-confirmation model is safer than automatic promotion/rotation, but the maintainable implementation should be a generic lifecycle suggestion path coordinated with memory review safeguards.
