## Summary

Keep open. Current main still has the overflow diagnostic, capped auto-compaction, memory-promotion preview/apply, pre-compaction memory flush, and transcript reset/rotation primitives, but it does not wire repeated overflow pressure into a deduped promote-then-rotate suggestion with dry-run candidates and operator confirmation.

## What This Changes

Request a lifecycle suggestion that detects repeated session context overflow, previews memory-promotion candidates, and offers operator-confirmed promotion and session rotation.

## Best Possible Solution

Approve a lifecycle design that tracks repeated per-session overflow pressure, emits one deduped structured suggestion with a stable id and candidate count, exposes it in CLI/UI surfaces, and requires explicit confirmation before promotion or rotation.

## Reproduction Assessment

Yes. The existing overflow loop test gives a high-confidence simulation path for repeated context overflow, and source/search inspection confirms the requested promote/rotate lifecycle suggestion is absent on current main.

## Solution Assessment

Unclear pending maintainer design approval. A suggestion-and-confirmation flow is safer than unconditional promotion or rotation, but it should be reconciled with existing memory flush behavior and the related memory review-quality issues.
