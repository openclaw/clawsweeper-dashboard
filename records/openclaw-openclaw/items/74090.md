## Summary

Keep open. Current main has adjacent memory-flush, memory-promotion, overflow-compaction, and transcript-rotation primitives, but it still does not surface a deduped promote-then-rotate lifecycle suggestion with dry-run candidates and operator confirmation.

## What This Changes

The issue requests a lifecycle suggestion that detects repeated session context overflow and offers an operator-confirmed memory promote plus session rotation flow.

## Best Possible Solution

Define a lifecycle policy that tracks repeated per-session overflow pressure, emits one deduped operator suggestion with a stable id and candidate count, and keeps promotion or rotation behind explicit confirmation.

## Reproduction Assessment

Yes for the product gap: current-main source and the overflow-loop regression show repeated overflow still flows through auto-compaction and terminal overflow handling without a promote-then-rotate suggestion. I did not run tests because this was a read-only review.

## Solution Assessment

Unclear pending maintainer design approval. A suggestion-and-confirmation flow is safer than automatic promotion or rotation, but it should be reconciled with existing memoryFlush/truncateAfterCompaction behavior and the open memory review-quality issues.
