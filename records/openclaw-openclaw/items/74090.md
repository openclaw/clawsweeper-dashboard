## Summary

Keep open. Current main still has adjacent overflow recovery, memory promotion preview/apply, memory-flush housekeeping, and session archive/delete primitives, but it does not implement the requested one-time promote-then-rotate lifecycle suggestion with dry-run candidates and operator confirmation.

## What This Changes

Request a lifecycle suggestion that detects repeated per-session context overflow, previews memory promotion candidates, and lets an operator promote and/or rotate the bloated session.

## Best Possible Solution

Approve a lifecycle design that tracks repeated overflow pressure per session, emits one deduped suggestion with a stable id and candidate count, previews promotion candidates, and requires explicit confirmation before any memory write or session rotation.

## Reproduction Assessment

Yes. The simulated repeated-overflow test path gives a high-confidence reproduction of the current behavior, and source inspection on current main confirms the requested suggestion and promote/rotate action surface is absent.

## Solution Assessment

Unclear pending maintainer design approval. A suggestion-and-confirmation flow is safer than automatic promote/rotate, but it should be designed around existing silent memory flush behavior and the open promotion review-quality concerns.
