## Summary

Keep open. Current main has overflow diagnostics, capped auto-compaction, silent pre-compaction memory flush, memory promotion preview/apply, and session reset/archive primitives, but it still does not connect repeated overflow pressure to a deduped promote-then-rotate lifecycle suggestion with dry-run candidates and operator confirmation.

## What This Changes

Request a lifecycle suggestion that detects repeated session context overflow, previews memory promotion candidates, and lets an operator confirm promotion plus session rotation.

## Best Possible Solution

Approve a lifecycle design that tracks repeated per-session overflow pressure, emits one deduped structured suggestion with a stable id and candidate count, exposes it in operator surfaces, and requires explicit confirmation before promotion or rotation.

## Reproduction Assessment

Yes. The current overflow loop test provides a high-confidence simulation path for repeated context overflow, and source inspection confirms the requested promote/rotate lifecycle suggestion is absent; no tests were run because this review was read-only.

## Solution Assessment

Unclear pending maintainer design approval. A suggestion-and-confirmation flow is safer than automatic promote/rotate, but it should be aligned with existing memory flush behavior, session rotation semantics, and the open memory promotion quality issues.
