## Summary

Keep open. Current main still auto-compacts and can rotate compacted transcripts, and `memory promote` can preview/apply candidates, but there is no one-time promote-then-rotate lifecycle suggestion with dry-run candidates and operator confirmation.

## What This Changes

Request a lifecycle suggestion that detects repeated session context overflow, previews memory-promotion candidates, and lets an operator promote and rotate the session.

## Best Possible Solution

Define a core lifecycle suggestion that dedupes repeated per-session overflow pressure, previews promotion candidates without writing, and requires explicit operator confirmation before promotion or rotation.

## Reproduction Assessment

Yes. The existing overflow loop test is a high-confidence simulation path for repeated context overflow, and source inspection confirms the requested suggestion and promote/rotate action surface are absent; no tests were run because this was read-only.

## Solution Assessment

Unclear pending maintainer design approval. A suggestion-and-confirmation workflow is safer than automatic promote/rotate, but it should align with the open memory promotion quality and review-queue concerns in #66287, #67363, and #72887.
