## Summary

Keep open. Current main still has overflow diagnostics, capped auto-compaction, manual memory promotion preview/apply, session reset/archive primitives, and a separate pre-compaction memory-flush mitigation, but it does not connect repeated overflow pressure to a deduped promote-then-rotate lifecycle suggestion or operator confirmation flow.

## What This Changes

Request a lifecycle suggestion that detects repeated session context overflow, dry-runs memory promotion candidates, and offers an opt-in promote-and-rotate workflow for bloated sessions.

## Best Possible Solution

Approve a generic lifecycle-suggestion design, then add per-session overflow-pressure tracking that emits one stable, deduped suggestion with dry-run promotion candidate metadata and exposes explicit operator actions without auto-writing memory or rotating the active session by default.

## Reproduction Assessment

Yes. Current source and the overflow-loop test provide a high-confidence reproduction path: repeated context overflows compact and retry until a terminal `context_overflow` result, with no promote-and-rotate suggestion emitted. I did not execute tests because this review was required to stay read-only.

## Solution Assessment

Unclear pending maintainer design approval. The issue's suggestion-and-confirmation model is safer than automatic promotion/rotation, but the maintainable implementation should be a generic lifecycle surface coordinated with memory review safeguards rather than a one-off overflow hook.
