## Summary

Keep open. Current main has the underlying overflow diagnostics, bounded compaction retry loop, memory promotion preview/apply command, session usage display, and reset/archive primitive, but it still does not wire repeated overflow pressure into a deduped promote-then-rotate lifecycle suggestion or operator confirmation flow.

## What This Changes

Add an operator-visible lifecycle suggestion when a session repeatedly trips context overflow, previewing memory-promotion candidates and letting the operator confirm promotion and/or session rotation.

## Best Possible Solution

Define a generic lifecycle-suggestion contract first, then add a per-session overflow-pressure tracker that emits one deduped structured suggestion with dry-run candidate count, surfaces it in operator-visible CLI/UI paths, and gates promotion plus session rotation behind explicit confirmation.

## Reproduction Assessment

Yes. Current source and the overflow loop test provide a high-confidence simulation path: repeated context overflows compact and retry until a terminal `context_overflow` payload, with no promote-and-rotate suggestion. I did not execute tests because this review was required to stay read-only.

## Solution Assessment

Unclear pending maintainer design approval. The issue's suggestion-and-confirmation direction is safer than automatic promotion/rotation, but the maintainable fix should coordinate lifecycle state, memory review safeguards, config schema, and reset behavior deliberately.
