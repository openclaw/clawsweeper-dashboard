## Summary

Keep open. Current main still has the adjacent pieces, including overflow diagnostics, repeated overflow compaction retries, memory promotion dry-run/apply, and session reset/archive primitives, but it does not wire them into the requested one-time promote-then-rotate lifecycle suggestion or operator confirmation flow.

## What This Changes

Request a lifecycle suggestion that detects repeated session context overflow, previews memory promotion candidates, and offers operator-confirmed promotion plus session rotation.

## Best Possible Solution

Approve a lifecycle design that tracks repeated per-session overflow pressure, emits one deduped structured suggestion with a stable id and candidate count, exposes it in CLI/UI surfaces, and requires explicit confirmation before promotion or rotation.

## Reproduction Assessment

Yes. The current overflow loop test provides a high-confidence simulation path for repeated context overflow, and source inspection confirms the requested suggestion/action surface is absent; no tests were run because this review was read-only.

## Solution Assessment

Unclear pending maintainer design approval. A suggestion-and-confirmation flow is safer than automatic promote/rotate, but it needs to fit the existing memory flush behavior and the open memory promotion review-quality concerns.
