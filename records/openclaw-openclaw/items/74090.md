## Summary

Keep open. Current main still logs and retries overflow compaction and has separate memory-promotion/session primitives, but it does not connect repeated overflow pressure to a deduped promote-then-rotate suggestion or confirmation flow.

## What This Changes

Request a lifecycle suggestion that detects repeated session context overflow, previews memory-promotion candidates, and offers opt-in promote/rotate actions through logs, CLI, and UI/notification surfaces.

## Best Possible Solution

Approve a lifecycle design that tracks per-session overflow pressure, emits one deduped structured suggestion with dry-run candidate counts, and requires explicit confirmation before promotion or rotation.

## Reproduction Assessment

Not applicable as a feature request; the source path is nevertheless clear. The existing overflow loop test exercises repeated overflow while current code only compacts/reports `context_overflow` and no lifecycle-suggestion/promote-rotate surface exists.

## Solution Assessment

Unclear pending maintainer design. The opt-in confirmation flow is safer than automatic promotion or rotation, but it needs to be reconciled with memory-promotion quality, config shape, and CLI/UI notification ownership.
