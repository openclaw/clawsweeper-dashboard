## Summary

Keep open. Current main has adjacent mitigations, including automatic memory flush, overflow compaction retries, transcript rotation after compaction, and UI context-pressure compaction affordances, but it still does not implement the requested one-time promote-then-rotate lifecycle suggestion with dry-run promotion candidates and operator confirmation.

## What This Changes

Request to detect repeated session context overflow and surface an opt-in promote-then-rotate lifecycle suggestion with dry-run memory promotion candidates and operator confirmation.

## Best Possible Solution

Approve a lifecycle design that builds on the existing overflow diagnostics, memory flush, compaction rotation, and `memory promote` dry-run primitives: track per-session repeated overflow pressure, emit one deduped structured suggestion with a stable id and candidate count, expose it in CLI/UI surfaces, and require explicit confirmation before promotion or rotation.

## Reproduction Assessment

Yes. The current overflow loop test provides a high-confidence simulation path for repeated context overflow, and targeted source search confirms the requested lifecycle suggestion and promote/rotate action surface is absent. I did not run tests because this review was required to stay read-only.

## Solution Assessment

Unclear pending maintainer design approval. The requested suggestion-and-confirmation flow is safer than automatic rotate/promote, but it should be designed around the newly present memory flush behavior and the open memory review-quality concerns before an implementation PR is queued.
