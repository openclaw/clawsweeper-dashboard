## Summary

Keep open. Current main still has overflow diagnostics, capped auto-compaction, memory promotion preview/apply, and session archive/delete primitives, but it does not connect repeated overflow pressure to a deduped promote-then-rotate suggestion with dry-run candidates and operator confirmation.

## What This Changes

Add a lifecycle suggestion that detects repeated session context overflow and offers a dry-run-backed promote-then-rotate operator workflow.

## Best Possible Solution

Approve a lifecycle design that tracks repeated per-session overflow pressure, emits one deduped structured suggestion with a stable id and candidate count, exposes it in CLI/UI surfaces, and requires explicit confirmation before promotion or rotation.

## Reproduction Assessment

Yes. The existing overflow loop test gives a high-confidence simulation path for repeated context overflow, and source search confirms current main does not produce the requested promote/rotate lifecycle suggestion.

## Solution Assessment

Unclear pending maintainer design approval. A suggestion-and-confirmation flow is safer than unconditional promotion or rotation, but it should be reconciled with existing memory flush behavior and the related memory review-quality issues.
