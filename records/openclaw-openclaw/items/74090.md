## Summary

Keep open. Current main has the overflow diagnostic, bounded auto-compaction retries, memory-promotion preview/apply, and transcript/session maintenance primitives, but it still does not connect repeated overflow pressure to a deduped promote-then-rotate lifecycle suggestion with dry-run candidates and operator confirmation.

## What This Changes

Request to add a repeated context-overflow lifecycle suggestion that previews memory-promotion candidates and lets operators promote and/or rotate a bloated session.

## Best Possible Solution

Approve a lifecycle design that tracks repeated per-session overflow pressure, emits one deduped structured suggestion with a stable id and candidate count, exposes it in CLI/UI surfaces, and requires explicit confirmation before promotion or rotation.

## Reproduction Assessment

Yes. The overflow-loop test gives a high-confidence source simulation for repeated context overflow, and current-main source inspection shows the requested promote/rotate suggestion is absent from the overflow recovery path.

## Solution Assessment

Unclear pending maintainer design approval. A suggestion-and-confirmation flow is safer than unconditional promotion or rotation, but it should be reconciled with existing memory-flush behavior and the open memory review-quality issues.
