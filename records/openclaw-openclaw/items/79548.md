## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use routed binding targets for fallback replies:** `src/auto-reply/reply/bind-aware-dispatch.ts:97-105`
  - body: The fallback resolves a binding but then calls routeReply with the raw binding conversation id and omits the projected thread route. For Telegram forum bindings whose conversation is a bare topic id with a parent chat id, this can send to the topic id as the chat id or drop the reply instead of using the existing routeFromBindingRecord/routeToDeliveryFields path.
  - confidence: 0.92
- **[P2] Do not replay bind-routed blocks at finalization:** `src/auto-reply/reply/dispatch-acp-delivery.ts:492-496`
  - body: After fallback delivery succeeds, this code still leaves visible delivery accounting tied to dispatcher failure state. If the first post-teardown visible block increments failedCounts.block, later bind-routed blocks can be delivered live and then repeated as a final accumulated block because finalization still sees failed visible delivery.
  - confidence: 0.86
- **[P1] Update the new fallback to current normalization imports:** `src/auto-reply/reply/bind-aware-dispatch.ts:25-28`
  - body: The added fallback imports ../../shared/string-coerce.js, but current main has moved these helpers to @openclaw/normalization-core/string-coerce and no longer has src/shared/string-coerce.ts. As merged against current main, the new module will fail to resolve its imports.
  - confidence: 0.94
