## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Use routed binding targets for fallback replies:** `src/auto-reply/reply/bind-aware-dispatch.ts:97-105`
  - body: The new fallback resolves a binding but then calls `routeReply` with the raw `binding.conversation.conversationId` and omits required route context such as `replyKind` and thread fields. Existing bound-delivery callers project the binding through the route helpers first; without that, Telegram forum bindings with `parentConversationId` can be sent to the topic id as the chat id or fail type checking.
  - confidence: 0.92
- **[P2] Do not replay bind-routed blocks at finalization:** `src/auto-reply/reply/dispatch-acp-delivery.ts:492-496`
  - body: After fallback delivery succeeds, this code still marks visible delivery using the dispatcher-backed counters. If the first post-teardown block increments `failedCounts.block`, later bind-routed live blocks can be delivered and then replayed as one final accumulated block because `hasFailedVisibleTextDelivery()` remains true.
  - confidence: 0.86
- **[P1] Update the new fallback to current normalization imports:** `src/auto-reply/reply/bind-aware-dispatch.ts:25-28`
  - body: This added file imports `../../shared/string-coerce.js`, but current main no longer has that module after normalization helpers moved to `@openclaw/normalization-core/string-coerce`. Merging the added file as-is leaves the fallback unable to resolve its imports on the current base.
  - confidence: 0.94
