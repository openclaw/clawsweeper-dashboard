## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Route bound replies through route projection:** `src/auto-reply/reply/bind-aware-dispatch.ts:97-105`
  - body: The fallback sends `binding.conversation.conversationId` directly as `to` and drops `parentConversationId`. Telegram topic bindings need the existing route projection so delivery targets the parent chat with the topic `threadId`; otherwise the reported supergroup topic can still fail or misroute.
  - confidence: 0.92
- **[P2] Do not replay bind-routed visible blocks:** `src/auto-reply/reply/dispatch-acp-delivery.ts:458-461`
  - body: Fallback is triggered by any prior failed count, so a prior block/final failure can make later successful bind-routed live blocks still look failed to finalization, which then resends the accumulated visible text at end of turn. Track bind-routed visible delivery separately or exclude the pre-fallback dispatcher failure from final replay accounting.
  - confidence: 0.86
