## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Resolve bound targets before routeReply:** `src/auto-reply/reply/bind-aware-dispatch.ts:97-100`
  - body: This fallback sends `binding.conversation.conversationId` as `to` and drops `parentConversationId`/`threadId`. Telegram forum bindings can store the topic as the conversation id and the supergroup as the parent conversation, so this routes to the topic id instead of `{ to: chatId, threadId: topicId }`; use the existing delivery-target resolver and cover that binding shape.
  - confidence: 0.92
- **[P2] Avoid duplicating bind-routed block text:** `src/auto-reply/reply/dispatch-acp-delivery.ts:458-461`
  - body: When a prior dispatcher failure is `block` or `final`, later visible blocks can be delivered through the binding fallback, but `settleVisibleText()` still sees the old failed count and finalization can send the accumulated text again. Cover `failedCounts.block/final` and mark recovered bind-routed visible text so it is not repeated.
  - confidence: 0.84
