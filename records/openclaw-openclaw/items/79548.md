## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use route projection for bound fallback replies:** `src/auto-reply/reply/bind-aware-dispatch.ts:97-105`
  - body: The fallback resolves a binding but sends `binding.conversation.conversationId` directly as `to`, omitting the existing parent-conversation/thread projection. Telegram forum bindings can store a topic id plus parent chat id, so this can target the topic id as the chat or drop/misroute the reply instead of delivering to the original topic.
  - confidence: 0.92
- **[P2] Prevent fallback-delivered blocks from replaying:** `src/auto-reply/reply/dispatch-acp-delivery.ts:492-496`
  - body: After a prior dispatcher block/final failure triggers the fallback, later bind-routed visible blocks can still leave finalization seeing failed visible delivery from dispatcher counts and send the accumulated text again as a final block. Track successful bind-routed visible delivery separately or clear the replay condition for those blocks.
  - confidence: 0.86
- **[P1] Update the fallback imports for current main:** `src/auto-reply/reply/bind-aware-dispatch.ts:25-28`
  - body: The new fallback path imports normalization helpers from `../../shared/string-coerce.js`, but current main uses `@openclaw/normalization-core/string-coerce` and no longer has that shared module. The branch needs to be rebased and the imports updated before it can build.
  - confidence: 0.94
