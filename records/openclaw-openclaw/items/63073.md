## Summary

Current main and `v2026.5.7` already implement the MiniMax usage endpoint routing in a safer form, so this branch is obsolete and should not be merged over the shipped code.

## What This Changes

The PR changes MiniMax usage polling to derive the remains endpoint origin from MiniMax config or `MINIMAX_API_HOST` and adds URL/method assertions to the MiniMax usage fetcher tests.

## Best Possible Solution

Keep the shipped `v2026.5.7` implementation and close this overlapping branch instead of merging an older endpoint-routing variant.

## Reproduction Assessment

Yes. The PR branch can be source-reproduced with a `minimax-portal` OAuth credential configured under `models.providers.minimax-portal.baseUrl` and no `models.providers.minimax.baseUrl`; the branch reads `ctx.provider` after usage normalization and can route the token to the wrong host.

## Solution Assessment

No for the PR branch; yes for current main. The shipped implementation resolves the usage host from the actual MiniMax portal/API provider config and has broader regression coverage, so it is the narrower maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86
