## Summary

Keep open: the PR addresses the hardcoded MiniMax usage host, but the current diff can still route a `minimax-portal` regional OAuth token using the normalized `minimax` provider config or env fallback instead of the credential source, and related #65059/#65054 overlap the same endpoint semantics.

## What This Changes

The PR changes MiniMax usage polling to derive the remains endpoint origin from provider `baseUrl` or `MINIMAX_API_HOST` and adds URL/method assertions to the MiniMax usage tests.

## Best Possible Solution

Land one MiniMax-owned usage endpoint resolver that keeps `GET /v1/api/openplatform/coding_plan/remains`, routes by the actual credential/config source for `minimax`, `minimax-portal`, and env overrides, documents the behavior, and covers global API, CN API, and OAuth setups.

## Reproduction Assessment

Yes. A high-confidence unit reproduction is a `minimax-portal` OAuth credential with `models.providers.minimax-portal.baseUrl` set to the CN host and no `models.providers.minimax.baseUrl`; the PR still looks under the normalized `minimax` provider before falling back.

## Solution Assessment

No. The PR moves in the right direction by making the host configurable, but deriving it from `ctx.provider` after usage auth normalization is not the narrowest safe fix; the safer path is to carry or re-derive the credential source host.
