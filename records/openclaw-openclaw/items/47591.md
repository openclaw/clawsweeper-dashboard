## Summary

Keep open. Current main has adjacent typing/presence and durable outbound delivery work, but it still lacks the requested runtime-enforced silence watchdog plus response-delivery confirmation/no-response alert contract; the related closed watchdog-only issue points back here as the canonical thread.

## What This Changes

Request adds gateway-level silence watchdog beacons plus response delivery confirmation, retry, and alerting for operator-visible progress and final response delivery.

## Best Possible Solution

Scope a shared runtime/channel operator-visibility contract that builds on durable delivery, adds capability-aware IDs/ACKs/retry/dedupe/no-response policy, and defines watchdog beacon defaults before changing channel behavior.

## Reproduction Assessment

Not applicable as a single bug reproduction; this is a feature/design request. The implementation gap is source-verifiable because the proposed config, hooks, state names, and generic no-response alert path are absent on current main.

## Solution Assessment

Unclear without maintainer scoping. The current durable delivery lifecycle is a useful substrate, but the requested end state needs a shared contract that respects channel capabilities, duplicate-message safety, intentional silence, and final-only delivery modes.
