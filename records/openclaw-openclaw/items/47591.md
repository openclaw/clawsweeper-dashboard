## Summary

Keep open. Current main has partial visibility and delivery mitigations, but it still lacks the requested gateway-level silence watchdog and turn-level response-delivery confirmation state machine; the related open issues only cover parts of the remaining design.

## What This Changes

Request a gateway-native silence watchdog plus response-delivery confirmation subsystem with turn/response IDs, delivery ACK tracking, retries, no-response alerts, and per-channel configuration.

## Best Possible Solution

Keep this as a core/runtime design item. The end state should be a shared operator-visible-output and delivery-result contract across the reply dispatcher, routeReply, outbound delivery, and message-tool paths, with stable turn/response IDs, channel capability/config gates, retry dedupe, no-response/final-failure alerts, and regression coverage for final replies, message sends, chunks, preview/final transitions, and final-only channels.

## Reproduction Assessment

Not applicable as a single bug reproduction; this is a feature/design request. We do have a high-confidence implementation-gap check: the proposed config/hooks/states are absent on current main, and adjacent code only provides partial progress, presence, recovery, and diagnostics behavior.

## Solution Assessment

Unclear as proposed. The direction is sound, but the best maintainable solution is not a small default-on timer; it needs a shared runtime contract that respects channel capabilities, final-only settings, ambiguous send failures, and existing queue recovery semantics.
