## Summary

Keep open. Current main has adjacent typing/presence and durable outbound delivery pieces, but it still lacks the requested runtime-enforced silence watchdog plus response-delivery confirmation, retry, dedupe, and no-response alert contract.

## What This Changes

Requests gateway-level silence beacons and delivery-confirmation/retry/no-response alerts with per-channel configuration.

## Best Possible Solution

Define a shared channel/runtime operator-visibility contract that extends durable delivery with capability-aware watchdog reset events, idempotent retry/receipts, and no-response alert policy before enabling per-channel defaults.

## Reproduction Assessment

Not applicable as a single bug reproduction; this is a feature/design request. The implementation gap is source-verifiable because the proposed config, hooks, state names, and generic no-response alert path are absent on current main.

## Solution Assessment

Unclear until maintainers scope the runtime and channel policy. The durable delivery lifecycle is a useful substrate, but the requested default-enabled watchdog/retry/alert behavior needs channel capability, duplicate-safety, and silent-reply decisions first.
