## Summary

Keep open. Current main has partial progress, typing, and outbound-recovery safeguards, and #47382 tracks only the watchdog half; this issue still requests a new gateway-level delivery-confirmation and no-response contract that needs maintainer runtime/product scoping.

## What This Changes

Request gateway-level silence watchdog beacons plus response delivery confirmation for operator-visible progress and failed or missing outbound replies.

## Best Possible Solution

Scope a shared runtime/channel delivery contract, use #47382 for the narrower watchdog work, and add channel-capability-aware response IDs, delivery states, retry/dedupe, no-response handling, and regression coverage before changing defaults.

## Reproduction Assessment

Not applicable as a single bug reproduction; this is a feature/design request. The implementation-gap check is high-confidence because the requested config, hooks, and state names are absent and current code only provides partial typing/progress/retry mitigations.

## Solution Assessment

Unclear without maintainer design scoping. The broad direction is reasonable, but the maintainable fix is a shared runtime contract that respects channel capabilities, final-only settings, intentional silence, and duplicate-message safety.
