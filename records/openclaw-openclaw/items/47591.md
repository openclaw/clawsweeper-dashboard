## Summary

Keep open. Current main has partial presence/progress and outbound recovery safeguards, but it does not implement the requested runtime-enforced silence watchdog plus response-delivery/no-response confirmation contract; the open #47382 only covers the narrower watchdog half.

## What This Changes

Request to add configurable gateway-level silence beacons and response-delivery confirmation with delivery IDs, retries, duplicate suppression, alerts, and per-channel settings.

## Best Possible Solution

Scope a shared runtime/channel delivery contract, keep #47382 as the watchdog tracker, and add capability-aware response IDs, delivery states, retry/dedupe, no-response policy, and regression coverage before changing defaults.

## Reproduction Assessment

Not applicable as a single bug reproduction; this is a feature/design request. The implementation-gap check is high-confidence because the requested config keys, hook names, and delivery state model are absent while current code only provides partial progress, typing, diagnostics, and queue recovery.

## Solution Assessment

Unclear without maintainer design scoping. The maintainable path is a channel-capability-aware runtime contract rather than a small automatic fix, because default beacons, duplicate-safe retries, and intentional silence semantics all affect product behavior.
