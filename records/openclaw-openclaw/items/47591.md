## Summary

Keep open. Current main has adjacent typing/presence keepalive, message:sent hooks, pending-final-delivery preservation, and durable outbound recovery, but it still does not implement the requested channel-agnostic silence watchdog, per-channel watchdog config, delivery-confirmation state machine, no-response alerting, or duplicate-suppressed retry policy.

## What This Changes

The issue requests a gateway-level silence watchdog plus response delivery confirmation so operators see progress during long turns and get alerts or retry handling when final delivery fails.

## Best Possible Solution

Define a shared runtime/channel operator-visibility contract first, then implement capability-aware watchdog and delivery-confirmation policy with tests and live channel proof before enabling defaults.
