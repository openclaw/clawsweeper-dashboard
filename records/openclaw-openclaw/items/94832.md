## Best Possible Solution

Keep the shared invoke parser and stream-normalizer expansion, but narrow the sanitizer exception so true prose examples are preserved without bypassing full-line XML-ish tool-call stripping after ordinary prose.

## Reproduction Assessment

Yes, source-reproducible. The PR-head sanitizer helpers return before the shared stripper whenever prose precedes `<function=` or `<invoke` without ending in `call`, which preserves internal XML-ish blocks that current main strips.

## Solution Assessment

No. The shared repair package is the right layer for invoke parsing, but bypassing the whole user-visible stripper is too broad; the safer fix is a context-specific example guard plus regression coverage.
