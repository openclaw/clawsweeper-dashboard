Keep open. Current main still has a mostly flat `scripts/` tree, no root `scripts/README.md`, and many package, CI, docs, and release references that depend on root-level script paths. The cleanup is valid, but the requested all-at-once move needs maintainer-approved taxonomy and compatibility planning before implementation.

Define the taxonomy first, then land the cleanup in phases: add a root `scripts/README.md` index, move agreed categories with compatibility wrappers where needed, update package/workflow/docs/test references together, and validate the touched CI/package/release paths.
