## What This Changes

The PR adds tool-use/tool-result pairing repair at context-engine harness assembly and OpenAI Chat Completions request building, expands OpenAI completions transport-transform policy, and adds regression tests.

## Best Possible Solution

Either close the branch as covered by the merged attempt-level repair, or land only a narrow final-boundary hardening that matches the existing errored-turn drop policy and has a regression for that edge case.

## Reproduction Assessment

Yes. Source inspection shows the PR-added pre-conversion sanitizer can synthesize a tool result for an errored or aborted assistant turn, and the existing converter then drops that assistant while still serializing the result.

## Solution Assessment

No. A final Chat Completions guard is a plausible safety net, but this implementation does not match the existing attempt repair policy; the safer path is to align the policy or omit the duplicate guard now that the reported path is fixed on main.
