## Summary

Current main addresses the reported external WhatsApp TypeScript-load path: the WhatsApp plugin package is now versioned for 2026.5.3, the plugin npm publish path builds package-local JavaScript runtime files and overlays runtimeExtensions/runtimeSetupEntry before pack/publish, and the loader validates and prefers those runtime entries for installed packages. The fix is on main only; no release tag contains the proof commit yet.

## What This Changes

Prevent the external WhatsApp plugin from loading uncompiled TypeScript and starving cron model calls on Windows.

## Best Possible Solution

Keep the compiled-runtime packaging path on main and ship it in the next OpenClaw and @openclaw/whatsapp release; reopen with 2026.5.3+ logs if cron stalls persist after the package no longer loads index.ts.

## Reproduction Assessment

Yes for the affected release path: the report includes concrete Windows logs showing @openclaw/whatsapp loading index.ts plus event-loop delay and cron stall symptoms. I did not run a live Windows cron reproduction against current main.

## Solution Assessment

Yes. The narrow maintainable fix is to make the official npm plugin publish and advertise compiled JavaScript runtime entries, which directly removes the reported runtime TypeScript compilation path without adding a new config or cron policy.
