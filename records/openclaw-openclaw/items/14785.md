## Summary

Keep open. Current main has partial pressure valves, but the central fixed tool-schema/context overhead remains: model-facing definitions still carry descriptions and schemas, the normal system prompt still renders a Tooling section, and the message tool only shrinks to a small schema in send-only cases.

## What This Changes

The issue asks OpenClaw to reduce fixed per-session tool schema/context overhead through channel-aware pruning, lazy or slim schemas, shorter tool descriptions, and related prompt reductions.

## Best Possible Solution

Keep this as the canonical tracker and sequence measured, compatibility-aware reductions: prompt deduplication, empty-tool-list suppression, safe message schema pruning, description trimming, and a deliberate slim or lazy schema mode.

## Reproduction Assessment

Yes. Current source gives a high-confidence source reproduction: selected tools still become model-facing definitions with descriptions and parameter schemas, and normal prompts still render Tooling text; I did not run a live token measurement in this read-only review.

## Solution Assessment

Yes for keeping this as an incremental tracker, no for a single automated fix. The requested reductions cross provider payloads, channel/plugin contracts, prompt policy, and optional config/default behavior, so maintainers should choose the next slice before implementation.
