## What This Changes

The issue asks OpenClaw to reduce fixed per-session tool schema overhead through channel-aware pruning, lazy or slim tool loading, shorter descriptions, and splitting or compressing high-footprint tools.

## Best Possible Solution

Keep this as the canonical tracker and sequence incremental, compatibility-aware reductions: action/capability-aware message schema pruning, optional Tooling-text reduction, empty-tool-list handling, cron description slimming, and a compatible slim or lazy schema design for plugins and declarative tools.
