Keep open. Current main and the latest shipped tag still carry the core overhead paths: broad message schemas, full model-facing tool descriptions/parameters, prompt-side Tooling text, and long cron guidance; related issues are narrower follow-ups rather than a completed replacement.

## What This Changes

The issue asks OpenClaw to reduce fixed model context overhead from always-sent tool schemas, especially broad message schemas, long cron guidance, duplicated tool listings, and non-lazy tool definitions.

## Best Possible Solution

Keep this as the canonical performance tracker and sequence incremental core changes for duplicated Tooling text, empty effective tool lists, channel/capability-aware message schema trimming, concise descriptions, and compatible slim or lazy schema modes.

Yes for the umbrella direction, but not as a single automated fix. The maintainable path needs incremental design across provider schema contracts, prompt policy, channel/plugin compatibility, cron and isolated-agent behavior, and default versus opt-in schema compaction.
