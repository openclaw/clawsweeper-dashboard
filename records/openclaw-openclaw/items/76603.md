## Summary

Keep open. Current main still has the reported control-plane/runtime mismatch: manifest tool discovery can require disabled bundled plugin IDs, and registry reuse then rejects an otherwise usable runtime registry before enabled plugin tools are appended.

## What This Changes

Reported that disabled bundled tool plugins can cause enabled plugin-backed tools such as browser and memory tools to disappear from the effective agent tool surface.

## Best Possible Solution

Filter runtime tool-discovery required plugin IDs to the actually enabled/loadable plugin owners while preserving bundled manifest visibility for catalog, setup, and doctor flows.

## Reproduction Assessment

Yes, at source level: current main can include disabled bundled tool-contract plugins in required runtime IDs, and the active registry lookup rejects registries unless all required IDs are loaded. I did not run a live effective-inventory trace, so this is source-reproducible rather than end-to-end reproduced.

## Solution Assessment

Yes. Excluding disabled plugins from the runtime tool-discovery required-ID set is the narrow maintainable fix; changing global control-plane availability would risk catalog, setup, and doctor flows that intentionally inspect bundled metadata without loading runtime code.
