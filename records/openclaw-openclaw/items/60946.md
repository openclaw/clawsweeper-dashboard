## Summary

Keep open: current main still only partially satisfies the request, with edit mismatch errors showing bounded current contents while similar-match, duplicate-location, apply_patch mismatch, and exec command-not-found recovery guidance remain missing.

## What This Changes

The issue requests richer tool-failure diagnostics with contextual hints, similar matches, duplicate-location details, and actionable retry guidance.

## Best Possible Solution

Core tool runtime should produce concise, bounded, model-facing diagnostics for edit, apply_patch, and exec failures while preserving existing tool semantics and plugin trust boundaries.

## Reproduction Assessment

Yes. Source inspection of current main shows edit mismatch only appends `Current file contents`, apply_patch still throws plain missing-context or missing-expected-lines errors, and exec exit 127 still formats as `Command not found`; no runtime tests were run because this review is read-only.

## Solution Assessment

Yes. A bounded core runtime helper wired into the existing failure paths is the narrowest maintainable direction because external plugins cannot register the pre-model tool-result middleware needed for this behavior.
