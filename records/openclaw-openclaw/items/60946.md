## Summary

Keep open. Current main still only partially addresses the request: edit mismatch failures include bounded current file contents, but similar-match candidates, duplicate-location guidance, apply_patch mismatch help, and exec command-not-found recovery hints are still missing. The related #46548 is adjacent but not a full superseding tracker because it focuses on user-visible chat failure/retry surfacing rather than model-facing tool-result diagnostics.

## What This Changes

Enhance tool failure responses with contextual troubleshooting hints, similar text suggestions, duplicate match locations, and actionable retry steps.

## Best Possible Solution

Add a shared bounded diagnostic helper in the core agent tool runtime, then wire it into edit mismatch, apply_patch mismatch, and exec exit-127 formatting so failures include concise line-numbered candidates and retry guidance without changing tool semantics or plugin trust boundaries.

## Reproduction Assessment

Yes. Current-main source and tests provide a high-confidence reproduction path: edit mismatch only appends current contents, apply_patch mismatch errors remain terse, and the exec formatter/test still expect `Command not found`. Tests were not run because this review is read-only.

## Solution Assessment

Yes. A bounded core runtime diagnostic helper is the narrow maintainable path; ClawHub or external plugins are not sufficient today because external tool-result middleware cannot rewrite high-trust results before the model sees them.
