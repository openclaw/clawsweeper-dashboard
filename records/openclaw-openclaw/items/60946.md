Keep open: current main only partially satisfies the request, with edit mismatches showing bounded current file contents while similar-match, duplicate-location, apply_patch mismatch, and exec command-not-found guidance remain missing from the core tool-result paths.

## What This Changes

Request richer tool failure responses with contextual troubleshooting hints, similar or duplicate match details, and actionable fix suggestions across edit, apply_patch, and exec failures.

## Best Possible Solution

Add a bounded shared diagnostic helper in the core agent tool runtime and wire it into edit mismatch, apply_patch mismatch, and exec exit-127 formatting without changing tool semantics or plugin trust boundaries.

## Solution Assessment

Yes. A bounded core runtime helper is the narrowest maintainable direction because the public plugin path cannot currently rewrite high-trust tool results before the model sees them.

## Repair Work Prompt

Implement https://github.com/openclaw/openclaw/issues/60946 as a bounded core runtime diagnostic improvement. Add or reuse a small shared helper under `src/agents` for concise failed-tool diagnostics, then wire it into edit exact-match mismatches, apply_patch missing context/expected-lines failures, and exec exit-127 formatting. Include line-numbered similar candidates or duplicate-match locations where the file content is already available, common-cause/retry guidance, and conservative command-not-found hints based only on local evidence such as the attempted command or PATH-visible alternatives. Keep snippets tightly bounded, do not add speculative package-manager install advice, do not change tool semantics, and do not open external tool-result middleware to third-party plugins as part of this fix. Add or update focused regression tests and a user-facing CHANGELOG.md entry.
