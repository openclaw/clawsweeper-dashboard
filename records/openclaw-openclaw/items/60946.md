## Summary

Keep open: current main still only partially satisfies the request, with edit mismatches showing bounded current file contents while similar-match, duplicate-location, apply_patch mismatch, and exec command-not-found guidance remain missing. ## Repair Work Prompt

Implement bounded rich tool error diagnostics for the model-facing tool result paths in current main. Add or reuse a small shared helper under `src/agents/` for line-numbered, size-bounded diagnostic snippets. Wire it into `src/agents/pi-tools.host-edit.ts` so exact edit mismatch errors can include conservative common causes, similar candidate lines, and duplicate-match locations when available; into `src/agents/apply-patch-update.ts`/`src/agents/apply-patch.ts` so missing context or expected lines include nearby line-numbered candidates and retry guidance; and into `src/agents/bash-tools.exec-runtime.ts` so exit 127 has useful command-not-found guidance based only on conservative local evidence such as the attempted command and PATH-visible alternatives. Do not add package-manager install speculation, new config options, plugin API changes, or changes to tool semantics. Keep all snippets tightly bounded and add focused regression coverage. Add a single changelog entry if the final change is user-visible.
