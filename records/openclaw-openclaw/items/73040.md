## Summary

Keep open: the branch is a useful queue-level cleanup, but it does not fix the linked approval cascade because it only deduplicates exact `(text, contextKey)` pairs while the real path generates fresh approval IDs and bypasses the system-event queue for followups.

## What This Changes

The PR changes `src/infra/system-events.ts` to scan the queued ring for exact `(text, contextKey)` duplicates, delay `lastContextKey` updates until accepted enqueues, remove `lastText`, and add four regression tests.

## Best Possible Solution

Carry a stable exec intent key through approval registration, forwarding, followup, and any system-event fallback so retries of the same command/session collapse even when approval IDs rotate; keep the queue-wide dedupe only as a local cleanup if maintainers still want it.

## Reproduction Assessment

Yes, at source level: current main generates fresh approval IDs/context keys and renders the ID into the approval prompt, while reporter and production comments show the live cascade as repeated distinct IDs. I did not run a live Telegram failover reproduction in this read-only review.

## Solution Assessment

No. Exact `(text, contextKey)` queue dedupe is narrower than the reported failure; the maintainable fix needs stable approval intent identity or normalization at the exec approval and forwarder boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Deduplicate retries by stable exec intent:** `src/infra/system-events.ts:117-118`
  - body: The new queue scan only drops exact text/contextKey matches. The reported cascade creates a fresh approval ID, and current main renders that ID into both the approval prompt and context identity while the followup path goes through the approval forwarder, so the same command retried with a new ID can still produce another prompt. Please key or normalize this around the stable command/session intent, with a regression that uses different approval IDs for the same command.
  - confidence: 0.9
