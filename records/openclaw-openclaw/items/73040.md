## Summary

Keep open. The patch is focused and improves exact non-consecutive system-event dedupe, but it does not fully fix the linked heartbeat-failover cascade because current approval and node-exec paths mint fresh approval/run IDs and include those IDs in the prompt text and context key. The PR also lacks the required changelog entry for a user-facing fix.

## What This Changes

The branch changes `src/infra/system-events.ts` to scan the in-memory queue for exact duplicate `(text, contextKey)` events, preserve `lastContextKey` on skipped or contextless enqueues, remove `lastText`, and add four regression tests.

## Best Possible Solution

Thread or derive a stable exec intent dedupe key from approval creation through node/gateway execution and approval forwarding, while preserving the useful system-event queue cleanup for exact duplicates. Add a regression where the same command/session intent retries under different approval or run IDs, plus the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to enqueue or forward the same exec command twice while minting different approval/run IDs; current main puts those IDs in the prompt text and `contextKey`, and this PR compares those exact values.

## Solution Assessment

No. Queue-wide exact `(text, contextKey)` dedupe is a useful partial cleanup, but the reported cascade needs stable intent-level identity or normalization of ID-bearing fields to coalesce fresh approval retries safely.
