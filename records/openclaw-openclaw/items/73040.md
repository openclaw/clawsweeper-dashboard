## Summary

Keep open. The PR is aimed at a real regression and improves exact duplicate suppression, but it does not solve the linked heartbeat-failover cascade because current approval and exec-event paths generate fresh ID-bearing text and context values for the same command intent.

## What This Changes

The branch changes `src/infra/system-events.ts` to scan the session queue for duplicate `(text, contextKey)` events, preserve `lastContextKey` on skipped or contextless enqueues, remove `lastText`, and add focused regression tests.

## Best Possible Solution

Carry or derive a stable exec intent dedupe key from the approval or heartbeat-failover layer into the system-event and approval-forwarding path, while preserving distinct context-specific events. The final patch should include a regression where the same command/session intent retries under different approval or run IDs, plus the required changelog entry.

## Reproduction Assessment

Yes. A focused code-level reproduction is to enqueue or forward the same exec command twice with different approval/run IDs; current main embeds those IDs in the text and context, and the PR compares those exact values.

## Solution Assessment

No. Queue-wide exact `(text, contextKey)` dedupe is useful for non-consecutive exact duplicates, but the reported cascade needs a stable intent-level key or normalization of ID-bearing fields to be a complete fix.
