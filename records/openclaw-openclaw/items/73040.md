## Summary

Keep open. The PR is focused, but exact `(text, contextKey)` queue dedupe does not address the linked #69478 cascade because current approval and node-event paths embed fresh approval/run IDs in both text and context; it also needs a changelog entry.

## What This Changes

The branch changes system-event enqueueing to scan the 20-event queue for exact `(text, contextKey)` duplicates, adjusts `lastContextKey` writes, removes `lastText`, and adds regression tests.

## Best Possible Solution

Carry or derive a stable exec intent dedupe key through the approval/failover path into system events and approval forwarding, preserve context separation for genuinely distinct prompts, then add mixed-ID retry coverage plus an Unreleased changelog fix entry.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is the same command/session retrying under fresh approval or run IDs; current main includes those IDs in approval prompt text and exec event text/context, while this PR only compares exact text and context.

## Solution Assessment

No. Full-queue exact-pair dedupe is a useful local improvement, but the linked cascade needs a stable intent identity or ID normalization at the approval/failover boundary.
