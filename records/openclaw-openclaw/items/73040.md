## Summary

Keep open. Current main still has the old consecutive-only system-event dedupe, and this PR is not merge-ready because exact `(text, contextKey)` matching does not cover the linked approval cascade when retries mint fresh approval/run IDs.

## What This Changes

The PR changes system-event queuing to drop duplicate exact `(text, contextKey)` entries anywhere in a session queue, preserves `lastContextKey` on skipped/contextless enqueues, removes `lastText`, and adds regression tests.

## Best Possible Solution

Carry or derive a stable exec intent dedupe key from the approval/failover layer into system-event and approval-forwarding paths, keep context separation, then add a mixed-ID retry regression and a changelog fix entry.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is the same command/session retrying with fresh approval or run IDs; current main embeds those IDs in approval text and exec event text/context, while this PR compares exact text and context.

## Solution Assessment

No. Queue-wide exact-pair dedupe is useful for interleaved identical entries, but the reported cascade needs stable intent identity or normalization at the approval/failover boundary.
