## Summary

Keep open: the queue-wide dedupe is a focused cleanup, but as written it does not fix the linked fresh-approval-ID cascade because the exec approval forwarder path uses new IDs/context keys and can deliver directly outside `enqueueSystemEvent`.

## What This Changes

The branch replaces adjacent `lastText` suppression with queue-wide `(text, contextKey)` dedupe in `src/infra/system-events.ts`, changes `lastContextKey` updates, removes `lastText`, and adds four regression tests.

## Best Possible Solution

Thread a stable exec intent key through approval registration, forwarding, failover, and followup delivery; keep this queue dedupe only as a narrowed infra cleanup if maintainers still want it.

## Reproduction Assessment

Yes, source-reproducible with high confidence: current main creates fresh approval IDs/context keys and includes the ID in the rendered approval prompt, while the PR comments provide live cascade logs with distinct IDs. I did not run a live Telegram cascade in this read-only review.

## Solution Assessment

No. Exact `(text, contextKey)` queue dedupe is useful local cleanup, but the reported cascade needs stable exec intent identity before the approval forwarder/followup path emits fresh prompts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
