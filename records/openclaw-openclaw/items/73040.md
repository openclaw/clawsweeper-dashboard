## Summary

Keep open: the PR head is currently not mergeable because `system-events.ts` is broken after the `lastText` removal, and the exact `(text, contextKey)` dedupe still does not cover the reported fresh-ID approval-forwarder cascade. It also needs after-fix real behavior proof before merge.

## What This Changes

The PR changes system-event queueing to scan for duplicate `(text, contextKey)` entries, preserve `lastContextKey` only after accepted enqueues, remove `lastText`, and add four regression tests.

## Best Possible Solution

Fix the broken PR head, then carry a stable exec intent dedupe key through approval registration, failover, forwarding, and system-event paths with mixed-ID regression coverage and real deployment proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main generates fresh approval/run IDs and includes them in text/context, while this PR only dedupes exact `(text, contextKey)` pairs; the PR head also has a source-visible compile break.

## Solution Assessment

No. Queue-wide exact-pair dedupe is a useful helper, but the safer fix is stable intent identity or normalization at the approval/failover/forwarder boundary, after repairing the malformed queue-state code.
