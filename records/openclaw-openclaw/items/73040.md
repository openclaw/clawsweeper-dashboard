## Summary

Keep open: this is a focused bug-fix PR, but the current head is source-broken, lacks after-fix real behavior proof, and exact `(text, contextKey)` queue dedupe does not close the fresh-ID approval-forwarder cascade described by reporters.

## What This Changes

This PR changes system-event queueing to scan for duplicate `(text, contextKey)` entries, preserves `lastContextKey` only after accepted enqueues, removes `lastText`, and adds four regression tests.

## Best Possible Solution

Fix the branch parse/type errors, then coalesce exec approval retries by stable command/session intent across approval registration, failover, forwarding, and system-event paths with mixed-ID regression coverage and live proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current main only suppresses adjacent identical text, while approval and node exec paths include fresh approval/run IDs in both rendered text and context identity.

## Solution Assessment

No. Queue-wide exact-pair dedupe is useful cleanup, but it is not the narrowest complete fix for the reported cascade; the fix needs stable approval-intent identity or normalization at the approval/forwarder boundary after repairing the broken PR head.
