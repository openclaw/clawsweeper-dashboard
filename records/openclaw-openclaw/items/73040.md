## Summary

Keep open: this PR is not mergeable because the current head is source-broken, lacks current-head real behavior proof, and its exact `(text, contextKey)` queue dedupe does not address the fresh-ID approval-forwarder cascade confirmed in later discussion.

## What This Changes

The PR replaces single-slot system-event dedupe with a queue-wide `(text, contextKey)` scan, changes `lastContextKey` update policy, removes `lastText`, and adds four system-event regression tests.

## Best Possible Solution

Replace or substantially revise this PR so the fix uses a stable exec approval intent across approval registration, failover, forwarding, and queueing, with mixed-ID regression coverage and proof against the linked production scenario.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main generates fresh approval/run IDs and includes them in prompt text/context identity, and reporter comments add live logs for the same cascade shape.

## Solution Assessment

No. Queue-wide exact-pair dedupe is a useful local cleanup, but the maintainable fix needs stable intent identity or normalization at the approval/failover/forwarder boundary after repairing the malformed queue-state code.
