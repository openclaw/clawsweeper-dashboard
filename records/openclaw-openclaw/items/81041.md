## Summary

Keep open: current main and the latest release still hash the full assembled system prompt in the Claude live-session fingerprint, and fingerprint drift still restarts the live process. The linked closing PR is open and already has a needs-changes review, so this issue should remain paired with that implementation path until a stable prompt-fingerprint fix lands.

## What This Changes

The issue reports that Claude CLI live-session fingerprinting hashes the full assembled system prompt, causing chat-channel turns with volatile metadata to restart live sessions and lose context.

## Best Possible Solution

Land a reviewed Claude live-session fingerprint fix that excludes per-turn chat metadata while preserving durable invalidation for real prompt, auth, model, workspace, MCP, skills, argv, and environment changes.

## Reproduction Assessment

Yes, at source level. Two chat turns can vary `context.systemPrompt` through inbound metadata, and current main closes/recreates the Claude live session when that fingerprint changes.

## Solution Assessment

The diagnosis is valid, but the one-line removal proposed in the issue is not the best final shape because it can ignore stable prompt changes. The maintainable fix is a stable prompt fingerprint that strips only transient chat metadata.
