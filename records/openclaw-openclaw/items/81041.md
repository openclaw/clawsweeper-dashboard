## Summary

Keep open: current main still hashes the full assembled system prompt in the Claude live-session fingerprint and restarts the live process on fingerprint drift. The linked closing PR is still open and has a needs-changes review, so this issue should remain open until a stable prompt-fingerprint fix lands.

## What This Changes

The issue requests preventing per-turn chat metadata in `context.systemPrompt` from forcing Claude CLI live-session restarts.

## Best Possible Solution

Add or carry a stable Claude live-session prompt fingerprint that excludes per-turn chat metadata while still rotating the process for durable prompt, auth, MCP, model, workspace, skills, argv, and environment changes.

## Reproduction Assessment

Yes. By source inspection, two chat turns with different inbound metadata can produce different `context.systemPrompt` values, and current main restarts the Claude live process when that fingerprint changes.

## Solution Assessment

No. Removing the entire `systemPromptHash` fixes volatile metadata restarts but can ignore real stable prompt changes; the maintainable fix is a stable prompt fingerprint that strips only transient chat metadata.
