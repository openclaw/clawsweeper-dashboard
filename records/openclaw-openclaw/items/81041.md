Keep this issue open until https://github.com/openclaw/openclaw/pull/81047 or a replacement lands a narrower fingerprint boundary that excludes volatile prompt text while preserving static session invalidation.

Yes, from source inspection rather than a live rerun: current main hashes the full `systemPrompt` and closes the live Claude process when that fingerprint changes. The linked PR body also supplies live Telegram and journal evidence for the reported symptom.

## Solution Assessment

No for the one-line removal as currently proposed; the better fix should remove volatile per-turn prompt text from the live-session identity while preserving invalidation for stable prompt, config, bootstrap, hook, skill, and tool changes.
