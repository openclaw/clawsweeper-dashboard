## Summary

Keep open. Current main still hashes the full assembled system prompt in the Claude live-session fingerprint, so volatile chat metadata can still rotate the live claude-cli process; the linked open fix PR closes this issue but has unresolved review feedback.

## What This Changes

The issue asks OpenClaw to stop per-turn chat prompt metadata from changing the Claude live-session fingerprint and restarting claude-cli chat sessions.

## Best Possible Solution

Land a stable Claude live-session fingerprint that excludes per-turn chat metadata while preserving invalidation for real stable prompt, auth, MCP, model, workspace, skills, argv, and env changes.

## Reproduction Assessment

Yes. By source inspection, two chat turns with different inbound metadata can change `context.systemPrompt`; current main hashes that full prompt and restarts the Claude live process when the fingerprint changes.

## Solution Assessment

Yes for the direction, but the current linked PR needs the safer stable-prompt boundary before merge. The issue should remain open until that PR or a replacement fix lands.
