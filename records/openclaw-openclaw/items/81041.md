## Summary

Keep open: current `main` and latest release `v2026.6.8` still include `systemPromptHash` in the Claude live-session fingerprint, and the open closing PR is the active fix path rather than a merged fix.

## What This Changes

Stop Claude CLI live-session fingerprints from hashing volatile per-turn chat-channel system prompt content so chat turns can reuse the existing subprocess and retain context.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81047, or an equivalent maintainer-approved patch that removes volatile full-system-prompt data from Claude live-session fingerprinting while preserving stable auth, model, workspace, MCP, and skills guards.

## Reproduction Assessment

Yes from source inspection: current `main` still hashes the full system prompt, chat-channel prompt context flows into that prompt, and the live-session path restarts when the fingerprint differs. I did not run a live Telegram or Discord repro in this read-only review.

## Solution Assessment

Yes at the issue level: removing volatile full-system-prompt bytes from the Claude live fingerprint is the narrow source-aligned fix. The exact mergeable implementation still needs maintainer review on the linked PR because it changes session reuse boundaries.
