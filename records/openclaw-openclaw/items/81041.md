The report is source-reproducible and should stay open until the live-session prompt fingerprint is fixed without losing stable prompt invalidation.

## What This Changes

The issue requests that Claude CLI live-session reuse ignore per-turn chat metadata in the prompt fingerprint so chat sessions do not restart and lose context on every turn.

## Best Possible Solution

Land a stable Claude live-session prompt identity that excludes per-turn chat metadata while still rotating for durable prompt, auth, MCP, model, workspace, skills, argv, and environment changes.

## Reproduction Assessment

Yes. Source inspection shows chat turns can change `context.systemPrompt` through inbound metadata, and current main restarts the Claude live process whenever that full-prompt fingerprint changes.

## Solution Assessment

No for the one-line proposed removal in the linked PR, because it can ignore real stable prompt changes. The maintainable fix is to fingerprint a stable prompt representation or otherwise strip only transient chat metadata.
