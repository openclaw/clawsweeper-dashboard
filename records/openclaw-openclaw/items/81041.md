## Summary

Keep open. Current main still hashes the full prepared system prompt in the Claude live-session fingerprint and tears down the warm process whenever that fingerprint changes; the chat runner already carries a separate static extra-prompt hash, so the reported boundary is credible and narrow. I did not find a paired implementation PR or merged fix.

## What This Changes

The issue requests excluding per-turn system-prompt content from Claude live-session fingerprinting so chat-channel turns keep the same claude-cli subprocess and context.

## Best Possible Solution

Make the Claude live-session fingerprint ignore dynamic per-turn system-prompt material while retaining static prompt, auth, model, MCP, skills, argv, and env invalidation, then add a focused live-session regression test.

## Reproduction Assessment

Yes, as a source-level repro: two Claude live-session turns with the same key but a changed full `systemPrompt` will compute different fingerprints and hit the restart branch. I did not execute a real Telegram/Discord plus claude-cli session in this read-only pass.

## Solution Assessment

Partly. Excluding dynamic prompt text from the live fingerprint is the right direction, but the implementation should preserve static system-prompt/config invalidation instead of blindly dropping all prompt sensitivity.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/81041. In current main, Claude live sessions include `sha256(context.systemPrompt)` in `buildClaudeLiveFingerprint`, so dynamic-only prompt changes can close and recreate the warm claude-cli subprocess. Adjust the fingerprint contract so per-turn/dynamic chat prompt material does not force live-session restart, while preserving intentional invalidation for provider/model/auth/auth epoch/MCP/skills/argv/env and static prompt/config changes. Prefer carrying or deriving a stable prompt fingerprint from `prepareCliRunContext` rather than blindly removing all prompt sensitivity if that would let real static prompt changes reuse an old live process. Add focused regression coverage in `src/agents/cli-runner.spawn.test.ts` for two Claude live-session turns with the same key and same static prompt hash but changed dynamic system-prompt material reusing one process, plus coverage that a real static prompt hash change still restarts when appropriate. Do not broaden into the update-drain, delivery-mirror, gateway restart, or channel-routing issues from related reports. Add a user-facing `CHANGELOG.md` fix entry if this becomes a shipped bug fix.
