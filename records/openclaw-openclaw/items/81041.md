## Summary

Keep open: current main and the latest release still hash the full assembled system prompt in the Claude live-session fingerprint, and the linked closing PR remains open with a needs-changes review.

## What This Changes

The issue reports claude-cli chat sessions losing continuity because the live fingerprint changes when per-turn system prompt metadata changes.

## Best Possible Solution

Land a stable Claude live-session prompt fingerprint that excludes per-turn chat metadata while preserving invalidation for durable prompt, auth, MCP, model, workspace, skills, argv, and environment changes.

## Reproduction Assessment

Yes, source-level. Two chat turns with different inbound metadata can produce different `context.systemPrompt` values, and current main restarts the Claude live process when that fingerprint changes.

## Solution Assessment

No, the suggested one-line removal is not the best complete fix because it can ignore real stable prompt changes. The safer solution is a stable prompt hash that strips transient chat metadata only.
