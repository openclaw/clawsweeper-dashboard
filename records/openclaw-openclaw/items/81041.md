## Summary

Keep open: current main still hashes the full assembled system prompt in the Claude live-session fingerprint, and fingerprint drift still restarts the live process. The linked closing PR is open and has a needs-changes review, so this issue should remain open until a stable prompt-fingerprint fix lands.

## What This Changes

The issue requests that Claude CLI live-session reuse stop treating per-turn chat metadata in the system prompt as a reason to restart the live subprocess.

## Best Possible Solution

Land a stable Claude live-session prompt fingerprint that excludes per-turn channel metadata while still rotating on durable prompt, auth, MCP, model, workspace, skills, argv, and environment changes.

## Reproduction Assessment

Yes, source-level: two chat turns can differ only in volatile inbound metadata, yet current main hashes the full `context.systemPrompt` and restarts the Claude live process on fingerprint drift. I did not rerun the live Telegram/Discord path in this read-only review.

## Solution Assessment

No, the issue's one-line removal is not the best final shape because it can skip real stable prompt invalidation. The maintainable fix is to fingerprint a stable prompt representation and exclude only transient chat metadata.
