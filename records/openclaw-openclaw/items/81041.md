Keep open: current main and the latest release still hash the full assembled Claude live-session system prompt, and the linked closing PR remains open with a needs-changes review.

## What This Changes

The issue requests preventing per-turn chat-channel metadata in Claude CLI live-session prompts from forcing subprocess restarts and losing cross-turn context.

## Best Possible Solution

Land a focused fix that fingerprints stable session-changing prompt inputs while excluding per-turn chat metadata, then close this issue through the linked implementation PR.

## Reproduction Assessment

Yes, by source inspection: chat turns can change `context.systemPrompt` through inbound metadata, and current main restarts Claude live sessions when that prompt hash changes. I did not execute a live Telegram or Discord repro in this read-only review.

## Solution Assessment

Mostly yes: the bug belongs in the Claude live-session fingerprint boundary, but the best fix should exclude volatile chat metadata without ignoring stable system-prompt changes.
