## What This Changes

The issue requests stopping volatile chat-channel system prompt content from invalidating Claude CLI live sessions on every turn.

## Best Possible Solution

Add a stable Claude live-session prompt fingerprint that excludes per-turn chat metadata while still rotating the process for durable prompt, auth, MCP, model, workspace, skills, argv, and environment changes.

## Solution Assessment

No for the one-line suggested removal. The maintainable fix should preserve invalidation for stable system-prompt changes while excluding only transient chat metadata from the Claude live-session identity.
