## Summary

Keep open: current main still includes the volatile full-system-prompt hash in the Claude live-session fingerprint, and the issue is already paired with an open closing PR that needs review/repair before this can close.

## What This Changes

The issue reports claude-cli live sessions restarting because `buildClaudeLiveFingerprint` includes per-turn chat-channel metadata from the assembled system prompt.

## Best Possible Solution

Land a narrow fix, via the linked PR or a replacement, that fingerprints stable prompt-changing inputs while excluding per-turn inbound metadata from Claude live-session reuse.

## Reproduction Assessment

Yes, by source inspection: current main hashes the full `context.systemPrompt`, chat reply construction injects volatile inbound metadata, and fingerprint mismatch restarts the Claude live process. I did not execute the live Telegram/Discord repro in this read-only review.

## Solution Assessment

No, the open PR’s simple removal is not the best final shape because it also drops stable prompt invalidation. The maintainable solution should replace the full prompt hash with a stable prompt fingerprint or equivalent stable inputs.
