## Summary

Keep open: current `main` still includes the volatile full-system-prompt hash in the Claude live-session fingerprint, and the linked closing PR is open rather than merged.

## What This Changes

Stop Claude live-session reuse from hashing per-turn chat metadata so `claude-cli` chat sessions retain context across adjacent turns.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81047, or an equivalent narrow patch that removes volatile full-prompt hashing while preserving stable invalidation inputs.

## Reproduction Assessment

Yes. By source inspection, two chat turns with different inbound metadata can produce different `context.systemPrompt` values, and current `main` restarts the Claude live process when that fingerprint changes.

## Solution Assessment

Yes. The linked PR is the right narrow direction because it removes the volatile `systemPromptHash` from the live fingerprint while keeping model, workspace, auth, MCP, env, skills, argv, and `extraSystemPromptHash` invalidation.
