## Summary

Keep open: current main still hashes the full assembled Claude live-session system prompt, and the linked closing PR remains open with a needs-changes review rather than a merged fix.

## What This Changes

The issue reports Claude CLI chat sessions losing continuity because volatile per-turn prompt metadata changes the live-session fingerprint and forces fresh CLI subprocesses.

## Best Possible Solution

Carry a stable Claude live-session prompt fingerprint that excludes transient chat metadata while preserving restarts for durable prompt, auth, MCP, model, workspace, skills, argv, and environment changes.

## Reproduction Assessment

Yes. Source inspection shows chat turns can change `context.systemPrompt` through volatile inbound metadata, and current main restarts the Claude live process when the resulting fingerprint changes.

## Solution Assessment

No for the open PR's current one-line removal; it fixes volatile metadata resets but can ignore real stable prompt changes. The safer path is to hash a stable prompt representation or exclude only the transient chat sections.
