Keep open: current main still hashes the full assembled system prompt in the Claude live-session fingerprint, and the open closing PR at https://github.com/openclaw/openclaw/pull/81047 is the active fix path.

## What This Changes

Stop Claude CLI live-session fingerprints from changing on per-turn chat metadata so chat turns can reuse the same live process.

## Best Possible Solution

Land or replace the linked PR with a stable-prompt fingerprint fix that excludes per-turn inbound metadata while preserving invalidation for real prompt changes.

## Reproduction Assessment

Yes. By source inspection, chat metadata can change `context.systemPrompt`, and current main hashes that full prompt before restarting the live process on fingerprint mismatch.

## Solution Assessment

Yes for keeping this issue open. The maintainable fix should go through the linked PR path and narrow the fingerprint to stable session-changing prompt inputs, not close the issue before the PR lands.
