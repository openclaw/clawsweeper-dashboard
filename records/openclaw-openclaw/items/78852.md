## Summary

Keep open: the performance direction and terminal proof are useful, but the current head still bypasses the generation factories' existing auth preflight without preserving current main's config-key provider auth path, so existing custom image/video/music generation providers can disappear.

## What This Changes

The branch centralizes media tool availability planning and lets image, video, and music generation factories skip their own registration-time availability checks when `createOpenClawTools` has already planned them.

## Best Possible Solution

Make the central media plan reuse or exactly mirror the generation factories' `hasGenerationToolAvailability`/`hasProviderAuthForTool` contract, with focused `createOpenClawTools` regression coverage for config-key-only image, video, and music generation providers.

## Reproduction Assessment

Yes by source inspection. A focused repro would use a manifest generation provider plus only `models.providers.<id>.apiKey` auth, then call `createOpenClawTools`; current main's factory preflight accepts that path, while the PR's central plan can omit it before the factory runs.

## Solution Assessment

No as-is. The best fix is still the central fast path, but only if it delegates to the same provider-auth contract as the factories it bypasses instead of maintaining a narrower duplicate decision.
