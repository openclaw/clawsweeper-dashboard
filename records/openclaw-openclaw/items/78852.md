## Summary

Keep open: the performance direction is sound, but the centralized media availability plan drops an existing config-backed custom-provider auth path, so merging as-is can hide media generation tools for existing setups.

## What This Changes

This PR centralizes media tool availability planning in `createOpenClawTools`, adds `availabilityResolved` fast paths to image/video/music generation factories, and expands workspace/auth regression coverage.

## Best Possible Solution

Make the central media plan preserve the full `hasGenerationToolAvailability` contract, including manifest provider IDs configured through `models.providers.<id>.apiKey`, then keep the benchmark and focused regression coverage.

## Reproduction Assessment

Yes: from source, a plugin-advertised generation provider with `models.providers.<id>.apiKey` but no env/profile auth is accepted by current `hasGenerationToolAvailability` and dropped by the PR's central plan. I did not run tests because this review is read-only.

## Solution Assessment

No: reusing a resolved availability plan is the right layer for the performance fix, but this implementation is not the best fix until the plan preserves the existing custom-provider auth fallback.
