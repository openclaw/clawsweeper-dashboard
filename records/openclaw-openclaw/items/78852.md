## Summary

Needs changes before merge. The performance direction and real-behavior proof are solid, but latest head still loses explicit-workspace image availability when an unrelated default current snapshot exists, so a workspace manifest fallback provider can disappear from the image tool plan.

## What This Changes

The PR adds image availability to the optional media tool plan, passes an availability-resolved fast path to image/video/music generation factories, updates focused tests, and adds a changelog entry.

## Best Possible Solution

Keep the shared media availability plan and fast path, but make explicit-workspace image planning use the matching workspace current snapshot or the already loaded workspace manifest fallback without borrowing a default snapshot.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence repro path: install a default current snapshot, use an explicit workspace with only a manifest fallback media-understanding provider, and the PR head drops `imageSnapshot` to `undefined` where current main would load the workspace manifest.

## Solution Assessment

No. Reusing the availability plan is the right maintainable direction, but the latest snapshot selection still needs to preserve explicit-workspace manifest fallback semantics.
