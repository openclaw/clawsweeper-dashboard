## Summary

Keep open: the performance direction and supplied proof are useful, but the latest head still changes explicit-workspace image planning in one mixed snapshot case, so this needs a narrow fix before merge.

## What This Changes

The PR moves image availability into the optional media factory plan, passes an `availabilityResolved` fast path to image/video/music factories, updates focused tests, and adds a changelog entry.

## Best Possible Solution

Keep the shared media availability plan, but compute image availability from the matching workspace current snapshot or the already loaded workspace manifest fallback without borrowing an unrelated default snapshot.

## Reproduction Assessment

Yes. Source inspection shows the PR head sets `imageSnapshot` to `undefined` when an explicit workspace has no matching current snapshot but a default current snapshot exists, while current main would load the workspace manifest fallback.

## Solution Assessment

No. Reusing the availability plan is the right shape, but the image snapshot selection must preserve explicit-workspace fallback semantics in the mixed default-current/explicit-workspace case.
