## Summary

This PR should stay open because the availability-plan reuse is useful, but the latest diff still has a workspace-scoping regression in image tool planning that can select the active registry workspace instead of the explicit workspace passed by the caller.

## What This Changes

The PR reuses media tool availability planning during agent tool preparation and adds an `availabilityResolved` fast path for image, video, and music generation tool factories.

## Best Possible Solution

Keep the availability-plan reuse, but make image planning use the workspace-scoped snapshot and add focused regression coverage for explicit `workspaceDir` image availability.

## Reproduction Assessment

Yes. Source inspection on the PR head shows the omitted `workspaceDir` in the new image snapshot lookup, and the PR body provides terminal timing proof for the performance behavior being optimized.

## Solution Assessment

No. Reusing the media availability plan is the right direction, but image planning should pass `params.workspaceDir` to `getCurrentCapabilityMetadataSnapshot` or reuse the already loaded workspace-scoped `snapshot`.
