## Summary

Needs changes before merge. The latest branch is now scoped to the media availability work and the terminal benchmark proof is sufficient, but current main now has a merged config-backed media generation preflight fix from https://github.com/openclaw/openclaw/pull/85570; this PR makes the new plan authoritative without preserving that fallback, so existing `models.providers.*.apiKey` setups can lose image/video/music generation tools.

## What This Changes

The PR reuses the optional media tool availability plan during `createOpenClawTools`, adds `availabilityResolved` fast paths for media generation factories, updates focused tests, and adds a changelog entry.

## Best Possible Solution

Keep the performance fast path, but make the media factory plan mirror the current factory availability contract, including config-backed provider auth, before setting `availabilityResolved: true`.

## Reproduction Assessment

Yes, by source inspection: current main accepts config-backed provider auth in `hasGenerationToolAvailability`, while the PR's new plan does not and then skips the factory check with `availabilityResolved: true`. I did not run tests because this was a read-only review.

## Solution Assessment

Partly yes: carrying prepared availability facts forward is the right performance direction, but the plan must preserve every existing availability signal before it replaces the factory check.
