## Summary

This PR should stay open: the workspace fallback issue appears addressed, but the latest head still drops config/auth-backed image tool availability when plugins are globally disabled. Current main keeps image-capable `models.providers` entries with an auth profile available even without plugin availability, so the fast path needs one more regression fix before merge.

## What This Changes

The PR moves image/media generation availability checks into the shared media factory plan and passes an `availabilityResolved` fast path to image, video, and music generation tool factories.

## Best Possible Solution

Keep the shared availability plan and `availabilityResolved` fast path, but let image planning evaluate config-driven vision-provider auth signals without re-enabling manifest-backed plugin availability when `plugins.enabled` is false.

## Reproduction Assessment

Yes. A focused regression should use `plugins.enabled: false`, an image-capable `models.providers` entry, an auth profile for that provider, and an `agentDir`; current main's helper permits `image`, while PR head's plan returns false because the snapshot is suppressed.

## Solution Assessment

No. Reusing the availability plan is the right maintainable direction, but this implementation must preserve current configured-provider image availability separately from plugin-manifest availability.
