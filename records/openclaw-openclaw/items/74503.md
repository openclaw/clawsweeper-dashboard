## Summary

Keep open: this is useful source work for the real #73781 replay-loop regression, but the current diff still has a false-positive loop detector and lacks the required changelog entry.

## What This Changes

The PR disables embedded Pi SDK auto-retry via an in-memory retry-disabled settings snapshot and adds repeated-thinking loop-hint injection for transport replay and OpenAI-completions payloads.

## Best Possible Solution

Land after the detector is scoped to an active tool-result loop or resets on user turns, focused tests cover both retry disabling and Qwen/OpenAI-completions replay, and the changelog records the user-facing fix.

## Reproduction Assessment

Yes. The linked bug report gives a concrete Qwen/openai-completions replay pattern, and source inspection confirms current main leaves Pi retry enabled while the Qwen completions builder bypasses OpenClaw transport-message transformation.

## Solution Assessment

No. The in-memory retry override and completions wiring are the right direction, but the repeated-thinking guard should not scan across unrelated user prompts before appending a synthetic user hint.
