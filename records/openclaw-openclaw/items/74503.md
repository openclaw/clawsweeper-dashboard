## Summary

Keep open: the underlying replay-loop fix is still relevant on current main, but this PR is not merge-ready because it still has a stale test invariant, an over-broad repeated-thinking detector, and no after-fix real behavior proof.

## What This Changes

This PR disables embedded Pi SDK retry in prepared runtime settings and adds a repeated-thinking/tool-call loop hint to agent transport message serialization.

## Best Possible Solution

Land or consolidate a narrow retry-ownership fix with updated regression coverage and redacted real behavior proof; keep broader repeated-thinking/circuit-breaker policy scoped to the separate tracker unless maintainers choose otherwise.

## Reproduction Assessment

Yes at source level, but not with live after-fix proof: current main leaves Pi SDK retry enabled while the pinned SDK removes the failed assistant turn and calls continue(), and OpenClaw has an outer retry loop. I did not reproduce the full Feishu/Qwen replay loop in a real channel setup during this read-only review.

## Solution Assessment

No, not yet: disabling Pi SDK auto-retry is the right narrow direction, but this branch still needs the prepared-settings test update, detector scoping on real user turns, and real behavior proof before merge.
