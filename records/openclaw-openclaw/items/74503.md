## Summary

Keep open: this PR targets a valid source-level agent replay bug, but the current patch still has merge-blocking issues in the new loop detector and the required test/changelog follow-through.

## What This Changes

The PR disables embedded Pi SDK retry through prepared settings and injects a repeated-thinking loop hint into OpenAI responses/completions transport message preparation.

## Best Possible Solution

Land a narrow runtime-scoped fix that disables embedded Pi SDK retry without persisting settings, scopes repeated-thinking detection to an active replay cycle, adds focused regression coverage, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes, at source level. Current main leaves Pi SDK retry enabled, upstream retry removes the failed assistant turn before `continue()`, and Qwen uses the completions path that currently bypasses transport-message replay repair.

## Solution Assessment

No. The retry override and completions wiring are the right direction, but the detector should reset at real user turns or be scoped to the current assistant/tool-result replay cycle, and the patch needs aligned tests plus the required changelog entry.
