## Summary

Keep open: this PR targets a real agents-runtime replay-loop bug, but it still needs after-fix real behavior proof, a rebase/test update for current main, and a fix to the repeated-thinking detector so unrelated user turns are not counted as one loop.

## What This Changes

This PR disables embedded Pi SDK auto-retry via runtime settings and adds repeated-thinking/tool-call loop hint injection to agent transport message serialization.

## Best Possible Solution

Land or consolidate a rebased retry-ownership fix with focused regression coverage and redacted real behavior proof; keep broader circuit-breaker policy aligned with https://github.com/openclaw/openclaw/issues/78865 unless maintainers choose to include it here.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: current main leaves Pi SDK retry enabled, the pinned SDK defaults retry on and strips the failed assistant turn before continue(), and OpenClaw has its own outer retry loop. I did not establish a full live Feishu/Qwen replay in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? No, not yet: disabling Pi SDK auto-retry is the right narrow direction, but this branch still needs real behavior proof, a current-main rebase/test update, and tighter detector scoping before it is mergeable.
