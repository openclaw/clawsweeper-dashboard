## Summary

Keep open. Current main has shipped fixes for the websocket fallback and Discord `InteractionEventListener` timeout symptoms, and the plugin loader still applies manifest config defaults before passing `api.pluginConfig`; however, there is still no high-confidence proof that the reported `memory-lancedb-pro` `sessionStrategy` default regression is fixed on current main without the explicit workaround.

## What This Changes

The issue asks OpenClaw maintainers to investigate a 2026.3.28 regression where an external memory plugin resolved `sessionStrategy` as `none`, OpenAI websocket streaming logged HTTP 500 fallback, and Discord interaction handling slowed down.

## Best Possible Solution

Verify the original `memory-lancedb-pro@1.1.0-beta.10` setup against current OpenClaw without the explicit `sessionStrategy` override. If it now registers memory-reflection hooks, close this as resolved by the shipped websocket, Discord, and related memory-config fixes; if it still resolves to `none`, split a focused plugin-config compatibility issue with a minimal config, plugin manifest/schema evidence, and current-main logs.
