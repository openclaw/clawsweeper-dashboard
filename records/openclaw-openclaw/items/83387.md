## Summary

Keep open: current main still has a source-level path where the model-visible OpenClaw code-mode `exec` tool reaches `before_tool_call` without hook context, while nested catalog calls are wrapped with context that includes `agentId`.

## What This Changes

Request to propagate the active agent id into `before_tool_call` for the outer OpenClaw generic code-mode `exec` dispatch.

## Best Possible Solution

Carry the existing run hook context into the outer code-mode `exec` and `wait` control tools before SDK tool-definition conversion, and add focused regression coverage for outer control `ctx.agentId` plus nested calls.

## Reproduction Assessment

Yes. The source path is high-confidence: raw code-mode controls reach `toToolDefinitions`, which invokes `runBeforeToolCallHook` without a `HookContext`, while nested catalog tools are wrapped with one.

## Solution Assessment

Yes. Passing the already-built run hook context into the outer code-mode control dispatch is the narrowest maintainable fix; plugins should not have to recover the agent id from `sessionKey`.
