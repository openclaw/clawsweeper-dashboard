## Summary

Keep open: current main still has a source-level path where the model-visible OpenClaw code-mode `exec` tool reaches `before_tool_call` without hook context, while nested catalog calls use a separate wrapped context that does preserve `agentId`.

## What This Changes

Report requests that OpenClaw generic code-mode's outer model-visible `exec` dispatch populate `ctx.agentId` for `before_tool_call` hooks.

## Best Possible Solution

Carry the active run hook context into the outer code-mode `exec` and `wait` control tools before Pi adapter dispatch, then add regression coverage for `ctx.agentId` on outer `exec` and for nested calls remaining intact.

## Reproduction Assessment

Yes. The reporter's minimal hook path is source-reproducible on current main: unwrapped code-mode controls enter `toToolDefinitions`, which calls `runBeforeToolCallHook` without a `HookContext`.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix: pass the already-resolved run context into the outer code-mode control dispatch rather than asking plugins to parse `sessionKey` or widen policies.
