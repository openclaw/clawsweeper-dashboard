## Summary

Keep open: current main still has a source-level path where OpenClaw code-mode visible control dispatch can reach `before_tool_call` without the per-run hook context, so the reported `ctx.agentId` gap remains valid and security-sensitive.

## What This Changes

The issue asks OpenClaw code-mode outer `exec` dispatch to pass the active agent id into `before_tool_call`, with a possible follow-up to clarify or align the `exec` parameter shape.

## Best Possible Solution

Forward the existing per-run `HookContext` into visible OpenClaw code-mode control dispatch, add focused regression coverage for outer `exec`/`wait` and nested calls, and split any `exec` input-shape alignment into a separate API decision.

## Reproduction Assessment

Yes, source-reproducible on current main. The adapter call for unwrapped visible tools omits `ctx`, while `runBeforeToolCallHook` only exposes `agentId` when `args.ctx` is present.

## Solution Assessment

Yes for the minimum bug: carry the already-known run hook context into the outer code-mode control hook call. The broader input-shape request should be split because it changes plugin-facing contract semantics.
