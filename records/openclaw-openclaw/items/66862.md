## Summary

Keep open. Current main still does not implement the PR's intended no-flag 1M behavior for Anthropic Opus/Sonnet 4.6, and the PR still has a blocking capability mismatch because it changes context planning without aligning the Anthropic beta-header and OAuth fallback path.

## What This Changes

The PR changes `src/agents/context.ts` and `src/agents/context.test.ts` so Anthropic or Claude CLI Opus/Sonnet 4 models resolve to 1,048,576 context tokens by default unless `params.context1m` is explicitly false.

## Best Possible Solution

Land a coordinated Anthropic context-window change that keeps resolver output, beta-header injection, OAuth/legacy-token fallback, discovered/configured context windows, session/status limits, compaction thresholds, provider docs, and tests in agreement.

## Reproduction Assessment

Yes. The original current-main bug is statically reproducible through `src/agents/context.test.ts`, where no-flag `anthropic/claude-opus-4-6` still resolves to `200_000`; the PR regression is reproducible by following the no-flag path through `resolveAnthropicBetas()`, which sends no 1M beta header.

## Solution Assessment

No. A resolver-only default with a manual OAuth opt-out is not the narrowest maintainable fix because context planning can diverge from the real Anthropic request capability; the safer path is to align the resolver and provider beta/auth path in the same change.
