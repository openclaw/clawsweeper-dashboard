## Summary

Keep open: current main still has the Ollama request-construction failure path, but this draft is not the best landing path yet because it keeps unreadable tools advertised with empty schemas and misses direct tool-field accessor failures while a broader skip-bad-tool draft is open.

## What This Changes

The branch wraps Ollama tool-schema traversal in defensive readers, falls back to empty object schemas for unreadable containers, skips unreadable union arrays, and adds a regression test around request construction.

## Best Possible Solution

Use one canonical Ollama hardening path that skips unreadable tool schemas while preserving healthy siblings and guards all tool field reads before request construction.

## Reproduction Assessment

Yes, source-reproducible: current main walks Ollama tool schemas with direct entries and array traversal inside request construction, and the PR regression uses throwing proxy containers to exercise that path. I did not execute tests because this review is read-only.

## Solution Assessment

No. This is a plausible mitigation, but the best fix is to skip unreadable tools and guard tool field/accessor reads, as shown by the newer draft or equivalent changes folded here.
