Full review comments:

- **[P2] Skip unreadable schemas instead of sending empty tools:** `extensions/ollama/src/stream.ts:849-852`
  - body: When the root schema cannot be enumerated, this returns `{ type: "object", properties: {} }` and still advertises the tool to Ollama. That preserves a malformed or unreadable tool as callable with no required arguments, so the model can choose a tool whose real schema could not be read; omit the bad tool and keep healthy siblings instead.
  - confidence: 0.82
- **[P2] Guard tool field access before normalization:** `extensions/ollama/src/stream.ts:1050`
  - body: `extractOllamaTools` still reads `tool.parameters` directly before the new schema-entry guards run. A throwing `parameters` accessor or revoked proxy can still escape here and emit the same request-construction error this PR is trying to prevent, so read the tool fields defensively before normalizing.
  - confidence: 0.88
