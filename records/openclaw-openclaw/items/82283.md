## Summary

Keep this PR open, but do not merge it as-is: it fixes the stale Codex/Pi first-choice readback, has sufficient contributor proof, and is blocked by a narrow correctness issue where official OpenAI loses Pi as an available alternate runtime choice. The protected `maintainer` label and overlapping https://github.com/openclaw/openclaw/pull/82286 mean the final landing path should be consolidated by a maintainer.

## What This Changes

The PR changes `/models` provider data so the OpenAI runtime choice is derived from `resolveAgentHarnessPolicy`, showing `OpenAI Codex` first for official OpenAI defaults while keeping custom OpenAI-compatible endpoints on Pi.

## Best Possible Solution

Land one consolidated fix that derives the first runtime choice from the effective harness policy, preserves Pi as a de-duplicated alternate for official OpenAI, keeps custom OpenAI-compatible endpoints Pi-first, and keeps focused regression coverage in `commands-models.test.ts`.

## Solution Assessment

No. The direction is right, but this branch is not the best complete fix because it drops Pi from official OpenAI runtime choices; the safer implementation should put the effective runtime first while preserving valid alternates.
