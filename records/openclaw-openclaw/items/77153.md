## Summary

Keep this draft maintainer PR open, but do not merge as-is: the Control UI suppresses the risk text whenever highlights are present, and the changelog hunk corrupts the Unreleased section.

## What This Changes

The branch adds parser-backed command risk lines/highlights to exec approval payloads, forwards them through gateway/view-model/UI paths, extends carrier parsing, adds tests, updates the changelog, and pins the npm registry in `.npmrc`.

## Best Possible Solution

Keep the additive approval metadata path, render textual risk lines alongside highlights when risks exist, refresh the changelog against current main, then rerun focused approval UI, explainer, and gateway checks before undrafting.

## Reproduction Assessment

Yes. Source inspection reproduces the introduced UI defect: the PR creates both explanation lines and highlights, then the Control UI drops the lines whenever highlights exist.

## Solution Assessment

No. The feature direction is maintainable, but the proposed UI rendering should show the risk copy alongside highlights and the changelog hunk must be rebased/refreshed before merge.
