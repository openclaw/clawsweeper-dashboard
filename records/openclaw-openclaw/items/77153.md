## Summary

Keep open: this draft maintainer PR is useful, but current head still has blocking approval-surface gaps because generated risk explanation lines are stubbed out and native chat approval clients do not render the new explanation field.

## What This Changes

The PR adds parser-derived exec approval risk highlights and explanation metadata through host registration, gateway payloads, forwarded messages, and the Control UI, with command-explainer tests, UI tests, a changelog entry, and an npm registry pin.

## Best Possible Solution

Generate concise non-empty risk lines and render them consistently across Web, forwarded, and native approval clients before undrafting and landing.

## Reproduction Assessment

Yes. Source inspection of PR head 69adad0fccbd826e53744b03067b27d81650b614 shows formatCommandExplanationLines always returns [], and native approval presenters build pending prompts without commandExplanationLines.

## Solution Assessment

No. The additive metadata and highlight plumbing are maintainable, but the implementation should generate the promised risk lines and consume them in native approval surfaces, or explicitly narrow the feature scope before merge.

  - confidence: 0.88

Users approving from chat will not see the new Risks section even after it is generated.
