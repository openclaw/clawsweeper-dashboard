## Summary

Keep open: this draft, maintainer-labeled PR has a concrete P2 import-boundary regression before merge. The Web highlighting direction is reasonable, but the command explainer must stay lazy-loaded on host exec approval paths.

## What This Changes

The PR adds parser-backed command span metadata to exec approval requests, gateway payloads, and Web approval rendering so command fragments can be highlighted in the approval modal.

## Best Possible Solution

Keep the Web span highlighting feature, but preserve the lazy-loading boundary with a dynamic import inside `resolveCommandExplanation` and focused guard coverage.

## Reproduction Assessment

Yes, for the review finding by source inspection: importing the PR head helper loads the command explainer barrel, then `extract`, then `tree-sitter-runtime`/`web-tree-sitter` before `resolveCommandExplanation` runs. Runtime proof was not run in this read-only review.

## Solution Assessment

No, not as-is. The payload and Web rendering approach is maintainable, but the parser-backed explainer should be dynamically imported inside `resolveCommandExplanation` so lightweight host exec approval imports stay cheap.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Lazy-load the command explainer:** `src/agents/bash-tools.exec-approval-request.ts:1`
  - body: This static import makes `bash-tools.exec-approval-request` load the command explainer barrel whenever host exec approval helpers are imported. Move the explainer import inside `resolveCommandExplanation`.
