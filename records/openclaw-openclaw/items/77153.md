## Summary

Keep open: this is a protected maintainer-labeled draft PR, and current head still has a blocking lazy-loading regression in the host exec approval path; the earlier workspace registry-pin concern appears resolved because `.npmrc` is no longer in the PR file list.

## What This Changes

The PR adds parser-backed exec command highlight metadata, forwards it through exec approval payloads, renders command/risk highlights in the Control UI approval modal, and adds related tests plus a changelog entry.

## Best Possible Solution

Keep the focused Control UI highlight feature, but lazy-load the parser only during approval-highlight generation and coordinate it with the approval lazy-loading work before merge.

## Reproduction Assessment

Not applicable. This is a feature PR rather than a reported bug; current main has a plain Control UI exec approval command block, while the PR adds new commandExplanation metadata and rendering.

## Solution Assessment

No, not as-is. The UI and payload shape are reasonable, but the command explainer should be dynamically imported inside the highlight-generation path so host exec imports do not eagerly load tree-sitter dependencies.

## Review Findings

Overall confidence: 0.84

Full review comments:

- **[P2] Lazy-load command explainer imports:** `src/agents/bash-tools.exec-approval-request.ts:1-5`
  - body: These static imports make `bash-tools.exec-approval-request` load the tree-sitter command explainer as soon as host exec modules import approval helpers, even for exec paths that never need approval highlights.
