## Summary

Keep open: the Web highlighting feature is useful, but this PR still has a blocking import-boundary regression because host approval registration statically imports the parser-backed command explainer. The protected `maintainer` label also means this should stay in normal maintainer handling, not cleanup closure.

## What This Changes

This PR adds parser-derived exec command span metadata through host approval registration, gateway protocol/broadcasts, Control UI rendering, tests, and a changelog entry.

## Best Possible Solution

Keep the Web highlight feature, but compute spans behind a dynamic import or lazy helper and preserve existing command validity checks before maintainer merge.

## Reproduction Assessment

Yes, for the review findings by source inspection: the PR head has a direct static import chain from the host approval helper to `web-tree-sitter`, and the UI parser now accepts raw whitespace-only command strings. I did not run a live build or browser repro in this read-only review.

## Solution Assessment

No, not as-is. The metadata and Control UI rendering direction is reasonable, but parser-backed span computation should be lazy-loaded and raw command preservation should still reject whitespace-only commands.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Lazy-load the command explainer:** `src/agents/bash-tools.exec-approval-request.ts:1`
  - body: This static import makes the host exec approval helper load the command-explainer barrel whenever gateway/node host approval modules import it. That barrel reaches `extract.ts` and `tree-sitter-runtime.ts`, whose top-level `web-tree-sitter` import breaks the lightweight approval setup boundary; move the explainer imports inside `resolveCommandSpans` or a small dynamic-import helper.
  - confidence: 0.89
- **[P3] Keep whitespace-only commands out of the queue:** `ui/src/ui/controllers/exec-approval.ts:82-83`
  - body: Changing this from `normalizeOptionalString` preserves leading spaces for span offsets, but it also accepts commands like `"   "` and can enqueue a blank approval modal. Check `command.trim().length > 0` while keeping the original command string in the returned payload.
  - confidence: 0.78
