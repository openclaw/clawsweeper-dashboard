## Summary

Keep open: this MEMBER-authored draft PR has the protected `maintainer` label and still introduces an eager parser-backed command explainer import across the host exec approval helper boundary.

## What This Changes

The PR adds parser-backed command-span metadata for host exec approvals, carries it through Gateway and Control UI payloads, renders highlighted command fragments in the Web approval modal, and adds related tests and changelog coverage.

## Best Possible Solution

Keep the Web highlight feature, but preserve the lazy-loading boundary by importing the parser-backed explainer only inside the span-computation path and adding a focused guard test.

## Reproduction Assessment

Yes for the review finding by source inspection: importing the PR head helper loads the command explainer barrel, then `extract`, then `tree-sitter-runtime`/`web-tree-sitter` before `resolveCommandExplanation` runs. Runtime proof was not run in this read-only review.

## Solution Assessment

No, not as-is. The payload and Web rendering direction are reasonable, but the parser-backed explainer should be dynamically imported inside `resolveCommandExplanation` so lightweight host exec approval imports stay cheap.
