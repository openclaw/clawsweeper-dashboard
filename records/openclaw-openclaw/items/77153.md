## Summary

Keep open. This draft, maintainer-labeled PR is dirty against current main, and the latest head still needs normal PR review because source inspection shows current-main `commandAnalysis` behavior is not preserved and raw highlight offsets are forwarded after command sanitization.

## What This Changes

The PR adds parser-backed exec command risk lines and command highlights across host approval registration, Gateway payloads, approval forwarding, view models, Control UI rendering, command-carrier parsing, tests, changelog, and `.npmrc`.

## Best Possible Solution

Rebase onto current main, keep `commandAnalysis` alongside the new explanation metadata, remap highlights to the sanitized display command, and then rerun the focused exec-approval, Control UI, lint, and changed-gate checks.

## Reproduction Assessment

Yes. Source inspection of current main plus the latest PR patch shows the `commandAnalysis` preservation gap and the raw-offset highlight path without needing to execute tests.

## Solution Assessment

No. The feature direction is reasonable, but this head is not the best mergeable solution until it preserves current-main analysis behavior and aligns highlight offsets with the rendered sanitized command.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve command analysis in approval requests:** `src/gateway/server-methods/exec-approval.ts:256-257`
  - body: Current main attaches `commandAnalysis` to every `exec.approval.request` payload and tests that inline-eval warnings survive on the request. This head adds the new explanation fields without carrying that existing field, so consumers of the current analysis payload would lose those warnings after rebase unless the request object preserves both metadata paths.
  - confidence: 0.91
- **[P2] Remap highlights onto sanitized command text:** `src/gateway/server-methods/exec-approval.ts:257`
  - body: The highlights are computed from raw command text, but the Gateway stores the sanitized display command and forwards the original offsets. When sanitization escapes invisibles, redacts secrets, or truncates text before a risky fragment, the UI will slice the wrong part of the displayed command or drop the intended risk highlight.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: Approval prompt highlight integrity needs attention because raw offsets can misrepresent sanitized commands; the `.npmrc` registry pin is not a concrete supply-chain concern here.

Concerns:

- **[medium] Raw highlight offsets can misrepresent sanitized commands:** `src/gateway/server-methods/exec-approval.ts:257`
  - body: Approval prompts sanitize command text before display, but this PR forwards highlight offsets computed against the raw command. In redaction, invisible-character, or truncation cases, a security-sensitive prompt can highlight the wrong fragment or omit the intended risky fragment.
  - confidence: 0.87
