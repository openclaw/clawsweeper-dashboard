## Summary

Keep open for maintainer review: this Web UI feature is useful, but the current head still adds an eager parser import on the host exec approval path and regresses blank-command rejection in the Control UI parser.

## What This Changes

The PR adds parser-derived command span metadata through exec approval registration, gateway payloads, Control UI rendering, tests, and a changelog entry.

## Best Possible Solution

Keep the Web highlight feature, but make parser-backed span extraction lazy and preserve current trim-empty command validation before merge.

## Reproduction Assessment

Yes. Source inspection of PR head 4f53d9b1b329 shows the static command-explainer import and raw-string command acceptance, and current main shows the previous trim-empty validation.

## Solution Assessment

No, not as-is. The metadata and Control UI rendering direction is maintainable, but span computation should stay behind a lazy import and the parser should reject whitespace-only commands while preserving raw offsets.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Lazy-load the command explainer:** `src/agents/bash-tools.exec-approval-request.ts:1`
  - body: This imports the command-explainer barrel whenever the host exec approval helper is loaded. That barrel reaches `extract.ts` and then `tree-sitter-runtime.ts`, whose top-level `web-tree-sitter` import pulls the rich parser into host approval setup before spans are needed; move this behind a dynamic import inside `resolveCommandSpans` or a small lazy helper.
  - confidence: 0.89
- **[P3] Keep whitespace-only commands out of the queue:** `ui/src/ui/controllers/exec-approval.ts:82-83`
  - body: Preserving raw spacing is needed for span offsets, but replacing `normalizeOptionalString` means a command like `"   "` no longer gets rejected and can enqueue a blank approval modal. Check `command.trim().length > 0` while keeping the original command string in the returned payload.
  - confidence: 0.8
