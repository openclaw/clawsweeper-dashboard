Full review comments:

- **[P2] Avoid exposing xargs placeholder risk copy:** `src/infra/command-explainer/format.ts:87-91`
  - body: For positional xargs execution, the detector uses the synthetic flag `<command>`. This formatter interpolates that flag directly, so common commands like `xargs -n 1 sh -c ...` can show `xargs <command> can run arbitrary code...` alongside the normal xargs carrier warning instead of user-facing risk text.
  - confidence: 0.87
- **[P3] Preserve generated Risks formatting when forwarding:** `src/infra/exec-approval-forwarder.ts:262-263`
  - body: `formatCommandExplanationLines` already returns a `Risks:` heading and bullet-prefixed risk rows, but forwarded approval messages wrap every line in another markdown bullet. Real forwarded approvals will render as `- Risks:` and `- • python -c...`; render the generated lines as-is or format the heading/list separately.
  - confidence: 0.84
