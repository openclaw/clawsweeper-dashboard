## Summary

Keep open: this is a useful maintainer-owned draft feature PR, but the current patch still introduces an eager command-explainer import on the host exec approval helper path.

## What This Changes

The branch adds parser-backed command span metadata for host exec approvals, forwards it through Gateway and Control UI payloads, renders highlighted spans in the Web approval modal, and adds tests plus a changelog entry.

## Best Possible Solution

Keep the Web highlight feature, but load the command explainer dynamically only inside the explanation path before taking the PR out of draft.

## Reproduction Assessment

Not applicable. This is a feature PR rather than a bug report; current main has a plain Control UI exec approval command block, while the PR adds command span metadata and rendering.

## Solution Assessment

No, not as-is. The payload and UI direction are reasonable, but the command explainer should be dynamically imported inside `resolveCommandExplanation` so host exec modules do not eagerly load tree-sitter dependencies.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Lazy-load command explainer imports:** `src/agents/bash-tools.exec-approval-request.ts:1`
  - body: This static import makes `bash-tools.exec-approval-request` load the Tree-sitter command explainer as soon as host exec modules import approval helpers, even when no approval highlight is generated. That pulls parser runtime code into lightweight host exec startup and conflicts with the lazy-loading boundary being tracked in https://github.com/openclaw/openclaw/pull/76943; import the explainer inside `resolveCommandExplanation` instead.
  - confidence: 0.86
