Full review comments:

- **[P2] Preserve the existing changelog section:** `CHANGELOG.md:25`
  - body: The added `### Fixes` heading splits the current Unreleased `### Changes` list while the original Unreleased `### Fixes` heading remains later in the same section. That reclassifies unrelated existing changes as fixes, so this entry needs to be moved into the existing structure without adding a duplicate heading.
  - confidence: 0.91
- **[P2] Avoid highlighting placeholder xargs risk text:** `src/infra/command-explainer/format.ts:55-60`
  - body: For xargs positional execution, the detector reports the synthetic flag `<command>`, but this formatter searches for the literal text `xargs <command>`. That string is not present in real approval commands, so the danger mark falls back to the start of the xargs segment and can highlight options instead of the carried command.
  - confidence: 0.87
- **[P2] Skip xargs option values before parsing carried wrappers:** `src/infra/command-explainer/extract.ts:923-928`
  - body: `xargsCommandArgv` skips values for only a few options. Common value-taking forms such as `xargs -n 1 sh -c ...` leave `1` as the first carried token, so the recursive wrapper parser misses the actual `sh -c` payload for common xargs invocations.
  - confidence: 0.83
- **[P3] Anchor command highlights to the executable token:** `src/infra/command-explainer/format.ts:27-28`
  - body: Command steps keep the span for the whole shell command, while `argvFromCommand` skips leading variable assignments. With a command like `FOO=1 python -c ...`, using `command.span.startIndex` highlights the assignment text instead of the executable name; the formatter should use an executable span or locate the executable within the command text.
  - confidence: 0.76
