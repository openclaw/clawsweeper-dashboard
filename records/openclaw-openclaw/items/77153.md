## Summary

Keep open: this is an active draft, maintainer-labeled PR, and source inspection of the current head still finds approval-copy and highlight-offset correctness issues that can mislead exec approval prompts.

## What This Changes

The PR adds parser-backed exec command risk lines and command/risk highlights through host approval registration, gateway payloads, forwarded approval messages, approval view models, and the Control UI, with tests, changelog, and an npm registry pin.

## Best Possible Solution

Keep the PR open, fix the formatter and offset handling on this branch, then rerun focused explainer, forwarding, gateway, and Control UI checks before undrafting or merging.

## Reproduction Assessment

Yes. Source inspection of PR head 77a27cfab5814778e09838246865002d87fbaba1 shows the xargs placeholder risk-copy path, the UI command-normalization offset mismatch, and forwarded-message bullet wrapping deterministically.

## Solution Assessment

No. The additive metadata path is maintainable, but the formatter should avoid synthetic placeholder copy, the UI should preserve or translate offsets against the displayed command, and the forwarder should render generated Risks lines without extra bullets.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Avoid exposing xargs placeholder risk copy:** `src/infra/command-explainer/format.ts:87-91`
  - body: For positional xargs execution, the detector uses the synthetic flag `<command>`. This formatter interpolates that flag directly, so commands like `xargs -n 1 sh -c ...` can show `xargs <command> can run arbitrary code...` instead of user-facing risk text.
  - confidence: 0.9
- **[P2] Keep highlights aligned with the displayed command:** `ui/src/ui/controllers/exec-approval.ts:127-129`
  - body: `parseExecApprovalRequested` still trims the command string, but the new highlights are generated against the exact command text. A command with leading whitespace shifts the offsets, so the approval modal can mark the wrong substring as risky; preserve the exact command or translate/drop offsets after normalization.
  - confidence: 0.84
- **[P3] Preserve generated Risks formatting when forwarding:** `src/infra/exec-approval-forwarder.ts:263`
  - body: `formatCommandExplanationLines` returns a `Risks:` heading and bullet-prefixed rows, but forwarded approval messages wrap every line in another markdown bullet. Real forwarded approvals render as `- Risks:` and `- • python -c...`; render generated lines as-is or format the heading/list separately.
  - confidence: 0.86
