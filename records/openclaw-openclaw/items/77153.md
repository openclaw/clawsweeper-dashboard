## Summary

Keep open: this maintainer-labeled draft PR is active and useful, but the current head still introduces a parser eager-load regression in the host exec approval helper, plus a small xargs option parsing edge case.

## What This Changes

The PR adds parser-backed exec approval command span metadata through agent, gateway, protocol, and Control UI paths, with tests and a changelog entry.

## Best Possible Solution

Keep the Web span-highlighting feature, but preserve the lazy-loading boundary for rich command explanation and tighten xargs option parsing before merge.

## Reproduction Assessment

Yes, for the review findings by source inspection: the PR head imports the command explainer at module load, and xargs help shows bare --replace/--eof do not require consuming the next token. Runtime proof was not run in this read-only review.

## Solution Assessment

No, not as-is. The metadata and Control UI rendering direction is maintainable, but the parser-backed explainer should be dynamically imported inside the explanation path and xargs optional-value handling should not skip the command executable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Lazy-load the command explainer:** `src/agents/bash-tools.exec-approval-request.ts:1`
  - body: This static import makes the host exec approval helper load the command explainer barrel at module import time. That path reaches the tree-sitter runtime, so node/gateway approval setup can pull parser code before highlight generation is requested; move the explainer import into `resolveCommandExplanation` or a small lazy helper.
  - confidence: 0.88
- **[P3] Treat bare xargs --replace as no-value:** `src/infra/command-explainer/extract.ts:915-917`
  - body: GNU xargs documents `--replace[=R]` and `--eof[=END]` as optional-value long options. Putting bare `--replace`/`--eof` in the skip-next set makes `xargsCommandArgv` skip the actual command in forms like `xargs --replace sh -c ...`, so nested shell/eval spans are missed.
  - confidence: 0.79
