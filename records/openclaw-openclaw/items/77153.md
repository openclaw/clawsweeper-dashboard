## Summary

Keep open: this is a draft MEMBER-authored PR with the protected `maintainer` label, and source review found a blocking import-boundary regression in the latest head.

## What This Changes

The PR adds parser-derived command span metadata to exec approval requests and renders highlighted command fragments in the Control UI approval modal, with tests and a changelog entry.

## Best Possible Solution

Keep the Web highlighting feature, but compute command spans through a lazy helper or dynamic import so host approval setup remains lightweight before maintainer review and validation.

## Reproduction Assessment

Yes, for the review finding by source inspection: the PR head statically imports the command explainer barrel from the host approval helper, and the dependency chain reaches the top-level `web-tree-sitter` import. I did not run a live runtime repro in this read-only review.

## Solution Assessment

No, not as-is. The metadata and Control UI rendering direction is reasonable, but the parser-backed explainer should be dynamically imported inside span resolution rather than loaded with the host approval helper.

## Review Findings
