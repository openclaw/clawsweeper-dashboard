## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Reject direct-API Spark configs during validation:** `extensions/openai/openclaw.plugin.json:225-230`
  - body: This new conditional suppression is what keeps Spark off direct OpenAI API routes, but `validateConfigObjectWithPlugins` still skips every suppression with a `when` clause. A config using `models.providers.openai.api: "openai-responses"` and `openai/gpt-5.3-codex-spark` will now validate successfully even though runtime suppression rejects that exact route, so validation needs to evaluate this condition or add an equivalent direct-API rejection path.
  - confidence: 0.9
- **[P2] Set Spark contextTokens with the 128k window:** `extensions/openai/openai-chatgpt-provider.ts:286-290`
  - body: When Spark is cloned from a Codex template, `cloneFirstTemplateModel` preserves template fields and then applies this patch. The gpt-5.4/gpt-5.3-codex templates carry `contextTokens: 272_000`, and `resolveContextWindowInfo` prefers `contextTokens` over `contextWindow`, so Spark can be budgeted as 272k despite the 128k window. Set `contextTokens: 128_000` here too.
  - confidence: 0.88
- **[P2] Give fallback Spark a 128k context budget:** `extensions/codex/provider-catalog.ts:33-40`
  - body: Adding Spark to the fallback Codex catalog routes it through `buildCodexModelDefinition`, which hard-codes `contextWindow: 272_000` for every fallback model. If discovery is disabled or times out, OpenClaw will advertise and budget Spark at 272k even though the PR's own runtime path treats Spark as 128k; add per-model context metadata or a Spark override in the fallback builder.
  - confidence: 0.86
