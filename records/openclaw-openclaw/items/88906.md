## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Set Spark contextTokens to the 128k limit:** `extensions/openai/openai-chatgpt-provider.ts:287-290`
  - body: Spark's patch lowers only `contextWindow`. When `cloneFirstTemplateModel` clones a `gpt-5.3-codex` or `gpt-5.4` template it preserves the template's `contextTokens: 272_000`, and the context guard uses `contextTokens` before `contextWindow`, so Spark runs can be budgeted as 272k instead of 128k. Set `contextTokens` to the Spark limit here and cover it in the expectation.
  - confidence: 0.93
- **[P2] Validate direct-API Spark configs before runtime:** `extensions/openai/openclaw.plugin.json:229`
  - body: These new `when`-gated suppressions are the only manifest block for direct API Spark, but config validation currently skips every suppression with `when`. A config that selects `openai/gpt-5.3-codex-spark` with `models.providers.openai.api: "openai-responses"` can validate and then fail at runtime. Teach validation to evaluate this direct-API condition, or add an explicit validation-only guard.
  - confidence: 0.9
- **[P2] Carry Spark context metadata into the Codex catalog:** `extensions/codex/provider-catalog.ts:33-39`
  - body: Adding Spark to `FALLBACK_CODEX_MODELS` makes the standalone Codex provider expose it through `buildCodexModelDefinition`, which hard-codes `DEFAULT_CONTEXT_WINDOW` to 272k for every row. Since this PR treats Spark as a 128k model on the OpenAI harness path, `codex/gpt-5.3-codex-spark` can still be over-budgeted when discovery uses the Codex catalog path. Add per-model context metadata or special-case Spark in the catalog builder.
  - confidence: 0.82
