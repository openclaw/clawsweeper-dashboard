## Summary

Keep open. Current main still pools shared-workspace agents through the memory-core dreaming pipeline, and the active draft fix PR is open rather than merged, so this is not cleanup-closeable.

## What This Changes

The issue reports that memory-core dreaming pools session snippets from agents sharing a workspace, causing first-person Dream Diary output to synthesize another agent's experiences as local memory.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/76140 with one coherent agent-scoped dreaming contract across workspace resolution, session corpus storage, recall keys, ranking/promotion, Dream Diary generation, UI/CLI surfaces, and shared-workspace regression tests.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: configure two agents with the same workspace and session transcripts, then dreaming resolves both agent IDs, appends one workspace session corpus, and records __dreaming_sessions__:<day> without an agent discriminator.

## Solution Assessment

No. The current workspace-pooled implementation is not the narrow maintainable solution for the documented isolation contract; the fix should make dreaming artifacts and promotion paths agent-scoped or explicitly reject ambiguous shared-workspace runs.
