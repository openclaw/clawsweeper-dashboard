## Summary

Keep open. Current main and the latest release still resolve shared-workspace agents into one dreaming workspace, ingest those agents' transcripts into a shared corpus, and rank/persist dreaming output without an active-agent discriminator; the linked draft fix PR is still open and unmerged: https://github.com/openclaw/openclaw/pull/76140.

## What This Changes

The issue requests agent-scoped dreaming so one agent's session snippets are not ingested, recalled, or narrated as another agent's memories in shared-workspace multi-agent setups.

## Best Possible Solution

Land or replace the linked fix with a consistent agent-boundary contract across workspace resolution, session corpus storage, __dreaming_sessions__ recall keys, ranking/promotion, Dream Diary generation, and two-agent shared-workspace regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure two agents with the same workspace and separate session transcripts, then the current resolver returns both agent ids, ingestion reads both session stores, and recall records one unscoped dreaming query.

## Solution Assessment

No. The current workspace-pooled implementation is not the best solution for the documented multi-agent isolation expectation; the right path is to finish the linked agent-scoped dreaming fix or a narrower replacement before closing this issue.
