## Summary

Keep open. Current main still groups all agents that share a normalized dreaming workspace and ingests their session transcripts into one workspace-level corpus/recall namespace; the linked draft fix PR remains open and should be reviewed or replaced before this issue closes.

## What This Changes

The issue asks memory-core dreaming to keep session corpus ingestion, recall, and synthesized dream output isolated per agent in multi-agent setups.

## Best Possible Solution

Land or replace the draft fix with one consistent agent-boundary contract across workspace resolution, session corpus storage, recall keys, ranking/promotion, Dream Diary generation, CLI/UI surfaces, and shared-workspace regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure two agents with the same workspace and separate session transcripts; current main resolves both IDs, reads both session stores, writes one corpus file, and records __dreaming_sessions__:<day> without an agent discriminator.

## Solution Assessment

No. The current workspace-pooled implementation is not the narrow maintainable solution for the documented isolation contract; the correct direction is to finish an agent-scoped dreaming fix and verify ranking/promotion paths too.
