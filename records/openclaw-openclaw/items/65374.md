## Summary

Keep this open. Current main still builds a workspace-level dreaming corpus from every agent assigned to the same workspace, and a draft fix PR already targets this issue but has not landed.

## What This Changes

The issue requests agent-level isolation for memory-core dreaming so one agent's session snippets are not ingested, recalled, or synthesized into another agent's dream narrative.

## Best Possible Solution

Finish and land or replace https://github.com/openclaw/openclaw/pull/76140 with one consistent agent-boundary contract across dreaming ingestion, corpus storage, recall/ranking, promotions, and manual/REM paths.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: configure multiple active agents with the same workspace, run dreaming ingestion, and the current code resolves all agent IDs for that workspace into one corpus and ranking path.

## Solution Assessment

Yes for keeping this issue open. The narrow maintainable path is to finish the linked fix PR or an equivalent agent-isolation patch; closing before that merges would drop the tracker for a still-present current-main bug.
