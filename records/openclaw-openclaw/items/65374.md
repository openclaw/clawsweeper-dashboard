## Label Justifications

- `P1`: The issue describes a real multi-agent memory workflow where one agent's session history can be synthesized into another agent's dreaming output.
- `impact:data-loss`: The derived dreaming corpus, recall store, and promoted memory artifacts can be silently polluted with another agent's session material.
- `impact:session-state`: The bug can associate transcript snippets and first-person dream narratives with the wrong agent identity.
- `impact:security`: Shared-workspace dreaming crosses an agent isolation boundary and can expose one agent's sensitive session content to another agent's memory pipeline.

## Summary

Keep open: current main and the latest release still group all agents that share a workspace into one memory-core dreaming workspace, ingest their session transcripts into a single workspace-level corpus, and rank promotions from workspace-level recall state. The open fix candidate still exists, but it is draft/conflicting and has unresolved proof/review concerns, so it is not a reason to close this issue yet.

## What This Changes

The issue requests agent-level isolation for memory-core dreaming so shared-gateway or shared-workspace agents cannot ingest, recall, or synthesize each other's session snippets as first-person memories.

## Best Possible Solution

Land a bounded memory-core fix that scopes dreaming ingestion, recall, ranking, and promotion by agent for shared workspaces, then close this issue from merged current-main proof and real shared-workspace validation.

## Reproduction Assessment

Yes. Source inspection on current main shows a shared workspace resolves all attached agent IDs, ingests each agent's transcript files, and writes them into one workspace-level dreaming corpus; I did not run a live gateway repro in this read-only pass.

## Solution Assessment

Yes for the requested end state, but not yet for a landed patch. The narrow maintainable fix is an agent-level boundary in memory-core dreaming ingestion, recall, and promotion; the open draft PR is the current candidate to review, refresh, or replace.
