## Summary

Keep open. Current main still resolves all agents attached to a shared dreaming workspace into one ingestion path and one workspace-level corpus, and the linked fix PR remains open, draft/dirty, and still needs real shared-workspace proof.

## What This Changes

The issue asks memory-core dreaming to isolate session corpus ingestion, ranking, and first-person dream synthesis by agent in shared-workspace multi-agent setups.

## Best Possible Solution

Land a corrected version of https://github.com/openclaw/openclaw/pull/76140, or a replacement, that threads current-agent scope through dreaming ingestion, corpus storage, recall/ranking, and promotion with redacted real shared-workspace proof.

Yes, source-reproducible: current main groups shared-workspace agent IDs, ingests all returned agents into one workspace corpus, and ranks promotions without a current-agent discriminator. I did not run a live gateway reproduction because this review is read-only.

## Solution Assessment

Yes on direction, not yet on merged implementation: the maintainable fix belongs in memory-core's existing dreaming pipeline with agent-scoped ingestion/ranking and safe storage. The current linked PR is the right review path but still needs proof and merge readiness.
