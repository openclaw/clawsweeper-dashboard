## Summary

Keep open. Current main still has the shared-workspace dreaming isolation bug, and the active fix path is the open draft PR at https://github.com/openclaw/openclaw/pull/76140 rather than a merged implementation.

## What This Changes

The issue requests agent-scoped memory-core dreaming so one agent cannot ingest or synthesize another agent's session snippets in shared-workspace setups.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/76140 with one coherent agent-scoped dreaming contract across workspace resolution, corpus storage, recall keys, ranking/promotion, diary generation, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure two agents with the same workspace and session transcripts, then dreaming resolves both agent IDs, appends one workspace session corpus, and records __dreaming_sessions__:<day> without an agent discriminator.

## Solution Assessment

No. The current workspace-pooled implementation is not the right fix for the documented isolation contract; the maintainable solution should scope dreaming artifacts and promotion paths by the active agent or fail closed when the agent boundary is ambiguous.
