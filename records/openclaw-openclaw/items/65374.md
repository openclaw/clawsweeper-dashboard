## Summary

Keep open. Current main still implements dreaming as workspace-scoped: agents sharing a workspace are grouped together, their session transcripts are ingested into one corpus, and recall/promotion/diary generation proceeds without an active-agent discriminator.

## What This Changes

The issue asks memory-core dreaming to prevent cross-agent session corpus pooling and first-person identity contamination in multi-agent gateway setups.

## Best Possible Solution

Define the intended isolation contract; if agent isolation is expected, thread agent scope through workspace resolution, corpus storage, __dreaming_sessions__ recall keys, ranking/promotion, and Dream Diary generation, with a two-agent shared-workspace regression.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is two configured agents sharing one workspace with separate session transcripts; current main groups them into one dreaming workspace, iterates both agents, appends both streams into one corpus file, and records one unscoped recall query.

## Solution Assessment

Unclear until maintainers choose the product boundary. The current workspace-pooled implementation is not the best solution for the reported identity-isolation expectation, but a durable fix should either make dreaming agent-scoped or explicitly document workspace pooling with controls.
