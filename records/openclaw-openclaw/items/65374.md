## Summary

Keep open: current main and the latest release still resolve all agent IDs for a shared memory-core dreaming workspace, ingest those agents' session transcript snippets, and write them into one workspace-level dreaming corpus before recall/promotion. The direct fix candidate is open, draft, and conflicting, so this should close only after a reviewed fix lands.

## What This Changes

The issue reports that memory-core dreaming can pool session snippets from multiple agents in a shared workspace and synthesize first-person memories for the wrong agent.

## Best Possible Solution

Land or replace the direct memory-core fix so shared-workspace dreaming carries a current-agent boundary through ingestion, recall, ranking, promotion, and cleanup of existing pooled artifacts.

## Reproduction Assessment

Yes. Source inspection on current main shows shared-workspace agent IDs are resolved together, all resolved agents' sessions are ingested, and the corpus/recall path remains workspace-scoped; I did not run a live multi-agent dreaming cycle in this read-only review.
