## Summary

Keep open. Current main still pools shared-workspace agents into one dreaming session corpus and recall namespace, and the active fix path is an open draft PR rather than merged behavior.

## What This Changes

The issue asks for memory-core dreaming to preserve agent boundaries so one agent's dream cycle cannot ingest or synthesize another agent's session history.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/76140 with one consistent agent-boundary contract across workspace resolution, session corpus storage, recall keys, ranking/promotion, Dream Diary generation, and regression coverage.

## Reproduction Assessment

Yes, at source level. Configure two agents with the same workspace and separate session transcripts; current main resolves both agent IDs, reads both session stores, writes one corpus file, and records __dreaming_sessions__:<day> without an active-agent discriminator.

## Solution Assessment

No for current main. The maintainable direction is an agent-scoped dreaming pipeline; workspace-level pooling contradicts the documented multi-agent isolation contract.
