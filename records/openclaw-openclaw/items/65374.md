## Summary

Keep open. Current main still resolves all agents that share a normalized dreaming workspace, ingests all of their session files into one workspace-level corpus, and records daily dreaming recall without an active-agent discriminator; the linked draft fix PR remains open and conflicting, so this issue is not cleanup-closeable.

## What This Changes

Fix memory-core dreaming so shared-workspace multi-agent setups do not pool session snippets or synthesize another agent's history as first-person dream memory.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/76140 with one consistent agent-boundary contract across workspace resolution, session corpus storage, recall keys, ranking/promotion, Dream Diary generation, CLI/UI surfaces, and shared-workspace regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure two agents with the same workspace and separate session transcripts, then current main resolves both IDs, reads both session stores, writes one corpus file, and records __dreaming_sessions__:<day> without an agent discriminator.

## Solution Assessment

No. The current workspace-pooled implementation is not the narrow maintainable solution for the documented isolation contract; the right direction is to finish an agent-scoped dreaming fix and verify ranking/promotion paths too.
