## Summary

Keep open. Current main and the latest release still group shared-workspace agents into one dreaming workspace and ingest their sessions into a shared corpus/recall namespace, while the open draft fix PR has not merged.

## What This Changes

The issue requests agent-level isolation for memory-core dreaming so one agent cannot ingest or synthesize another agent's session history in shared-workspace multi-agent setups.

## Best Possible Solution

Land or replace the linked draft fix with one consistent agent-boundary contract across workspace resolution, session corpus storage, recall keys, ranking/promotion, Dream Diary generation, and shared-workspace regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure two agents with the same workspace and separate session transcripts, then current main resolves both IDs, ingests both session stores, and records one unscoped dreaming recall query.

## Solution Assessment

No. The current workspace-pooled implementation is not the narrow maintainable solution for the documented isolation contract; the right path is to finish or replace the linked agent-scoped dreaming fix before closing this issue.
