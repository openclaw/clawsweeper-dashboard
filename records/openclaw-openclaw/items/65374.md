## Summary

Keep open. Current main still has a source-backed shared-workspace dreaming path that resolves all agents for one workspace, ingests their transcripts into one workspace-level session corpus, and records recall under a day-only query; the active fix path is an open draft PR.

## What This Changes

The issue asks for memory-core dreaming to isolate session corpus ingestion and dream synthesis by agent so one agent cannot absorb another agent's session history.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/76140 with one coherent agent-scoped dreaming contract across workspace resolution, session corpus storage, recall keys, ranking/promotion, Dream Diary generation, and shared-workspace regression tests.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: configure two active agents with the same workspace and session transcripts, then dreaming resolves both agent IDs, appends one workspace-level session corpus, and records __dreaming_sessions__:<day> without an active-agent discriminator.

## Solution Assessment

No. The current workspace-pooled implementation is not the narrow maintainable solution for the documented isolation contract; dreaming artifacts and promotion paths should be agent-scoped or fail closed for ambiguous shared-workspace runs.
