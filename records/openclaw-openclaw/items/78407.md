## Summary

Close: the 2026.5.5 doctor route regression was fixed in the shipped v2026.5.6 hotfix, and current main now avoids the reported OAuth lockout by routing repaired OpenAI agent refs through Codex auth instead of direct OpenAI API-key PI routing.

## What This Changes

The issue asks OpenClaw to stop `doctor --fix` and update-time doctor repair from migrating Codex-OAuth PI users into `openai/*` routes that require direct OpenAI API-key auth.

## Best Possible Solution

Keep the shipped/current route policy that prevents `doctor --fix` from leaving Codex-OAuth users on direct API-key PI routing, and track any native-Codex sandbox concern as a separate security report.

## Reproduction Assessment

Yes. The 2026.5.5 failure path is high-confidence from the source history and multiple live reports: doctor rewrote Codex OAuth routes to `openai/*` plus PI, which then selected direct OpenAI API-key auth.

## Solution Assessment

Yes. The shipped hotfix removed the unsafe automatic rewrite, and current main’s newer route policy keeps `openai/*` on Codex auth/runtime or explicit Codex-auth PI routing, which addresses the lockout without adding a new config flag.
