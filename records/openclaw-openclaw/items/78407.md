## What This Changes

The issue reports that `openclaw doctor --fix` rewrites working `openai-codex/*` Codex OAuth PI model routes to direct `openai/*` routes, breaking OAuth-only users after the 2026.5.5 update.

## Best Possible Solution

Review and land the canonical fix that preserves or recovers supported `openai-codex/*` PI OAuth routes during doctor repair, then close this issue after the merged fix is on `main`.

## Reproduction Assessment

Yes. From current source inspection, `doctor --fix` reaches `maybeRepairCodexRoutes`, rewrites `openai-codex/gpt-5.5` to `openai/gpt-5.5`, and can select PI; the issue comments add multiple live macOS/Linux confirmations of the same failure.

## Solution Assessment

Yes, the linked fix direction is the narrow maintainable solution: stop doctor from migrating a working Codex OAuth PI setup into a direct OpenAI API route, and update the regression coverage and docs to match.
