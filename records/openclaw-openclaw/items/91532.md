## Summary

Keep open: current main and v2026.6.6 still have source-reproducible cron paths where recovered tool-error payloads or run-level diagnostics can persist an isolated cron run as `error`, and the related open PR only covers part of this broader report.

## What This Changes

Investigate isolated cron agent-turn runs being marked failed after the agent recovers from tool errors and completes successfully.

## Best Possible Solution

Keep this issue as the broader cron false-positive tracking item until maintainers decide the recovered-tool-error status contract and either update the related PR or split a narrower follow-up for the diagnostic/meta-error latching path.
