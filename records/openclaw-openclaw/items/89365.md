Full review comments:

- **[P1] Preserve wildcard allowlists as unrestricted:** `src/agents/turn-surface-policy.ts:130-138`
  - body: When `toolsAllow` is `["*"]`, this compiler still adds `DEFAULT_SCOPE_DENY_TOOL_NAMES` to `excludeToolNames`, so the later policy pass removes tools such as `message`, `web_search`, and `exec`. Current main treats wildcard allowlists as unrestricted in both embedded and Codex dynamic filtering, so this breaks existing compatibility unless wildcard skips the default-deny list.
  - confidence: 0.91
- **[P1] Reuse the existing allowlist matcher for turn checks:** `src/agents/turn-surface-policy.ts:64`
  - body: This exact-set check does not expand `group:*`, plugin ids, or globs the way the existing tool policy matcher does. A turn allowed by `group:messaging`, `bundle-mcp`, a plugin id, or an MCP glob can pass the first filter and then be dropped here, so the new gate needs to use the same matcher/expanded policy semantics before filtering.
  - confidence: 0.89
- **[P1] Keep message-tool-only delivery from losing its send tool:** `src/agents/embedded-agent-runner/run/attempt-tool-construction-plan.ts:129-132`
  - body: Returning the scoped allowlist unchanged means `toolsAllow: []` or a narrow list no longer materializes `message` even when `forceMessageTool` is true. Since message-tool-only turns suppress automatic source delivery, this can leave channel replies with no visible send path unless strict blocking is made explicit and compatibility-preserving behavior remains the default.
  - confidence: 0.86
- **[P2] Wire Gateway turn allowlists through real callers:** `src/gateway/tool-resolution.ts:59`
  - body: The new `toolsAllow` parameter is only exercised by direct helper tests; current production callers of `resolveGatewayScopedTools` do not pass it. That means the claimed Gateway scoped-surface filtering is not proven on real Gateway turns, so either wire the request/protocol path into this resolver or keep the change scoped to the surfaces that actually pass a turn allowlist.
  - confidence: 0.78
