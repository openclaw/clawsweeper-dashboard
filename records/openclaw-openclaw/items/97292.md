## Summary

## What This Changes

The issue reports GitHub Copilot Claude replay failures when OpenClaw sends an Anthropic error `tool_result` whose sanitized content is empty.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/97293, then close this issue only after the chosen provider-boundary fix is merged.

Yes, source-level reproduction is high-confidence: current main can send an empty sanitized `tool_result.content` with `is_error: true`, and the issue provides matching redacted provider 400 logs. I did not run the live Copilot replay because the required credentials and session state are private.

## Solution Assessment

Yes, the narrow provider-boundary fix in the linked PR appears to target the right layer: the invalid outbound payload is constructed in `src/llm/providers/anthropic.ts`, and substituting a non-empty error placeholder avoids changing persisted session data or provider config.
