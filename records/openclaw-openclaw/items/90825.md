## Summary

Keep open. Current main still silently returns zero cost for all-zero model pricing, and this PR addresses the linked runtime visibility issue at the shared helper without blocking findings; the refreshed terminal proof is sufficient for this helper-level change.

## What This Changes

The PR adds a once-per-provider/model warning in `src/llm/model-utils.ts` when token usage is costed against all-zero pricing, plus focused `calculateCost` tests.

## Best Possible Solution

Land a shared-runtime warning after maintainer review, preserving zero numeric `usage.cost` fields while making missing pricing visible.

## Reproduction Assessment

Yes, from source inspection: current main multiplies all-zero `model.cost` rates by nonzero usage and returns zero cost fields without logging. I did not run a live provider because this was a read-only review.

## Solution Assessment

Yes. Warning at the shared `calculateCost` boundary is the narrowest maintainable mitigation for the silent-zero behavior while preserving the existing `usage.cost` shape; configurable fallback pricing would be larger product work.
