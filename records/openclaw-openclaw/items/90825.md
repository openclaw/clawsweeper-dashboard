## Summary

Keep open for maintainer review: the shared-boundary warning addresses silent zero costs with supplied terminal proof, but the all-zero heuristic has a compatibility/logging edge for deliberately free zero-cost models exposed through the SDK.

## What This Changes

The branch adds a once-per-provider/model warning in `calculateCost` when nonzero token usage is evaluated against an all-zero model cost table, plus focused model-utils tests.

## Best Possible Solution

Keep the warning at the shared cost boundary, but land only after maintainers accept or refine the zero-pricing semantics so explicit free models are not mislabeled as unknown pricing.

## Reproduction Assessment

Yes from source inspection and the PR's runtime proof: current main silently returns all-zero `usage.cost`, while the PR body shows a post-patch command emitting the warning. I did not run commands in this read-only review.

## Solution Assessment

Mostly yes: the shared `calculateCost` boundary is the right place to cover the provider and agent transports. The remaining question is whether the all-zero heuristic or warning text is the best long-term contract for explicitly free models.
