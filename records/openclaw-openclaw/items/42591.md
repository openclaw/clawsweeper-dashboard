## Summary

Keep open. Current main and v2026.6.8 still ship the Unix installer as one large public script, but a safe split needs maintainer direction on whether modules are source-only inputs to a generated single `curl | bash` artifact.

## What This Changes

The issue requests splitting `scripts/install.sh` into smaller shell modules for UI, download, platform, npm, git, and build-tool responsibilities.

## Best Possible Solution

Adopt a maintainer-approved modular source layout that deterministically generates and verifies the single public `install.sh` artifact, or explicitly keep the monolith as intended.

## Reproduction Assessment

Yes for the source observation: current main still has one large `scripts/install.sh` and no `scripts/install/` module tree. It is not a runtime bug reproduction.

## Solution Assessment

No for a direct served multi-file split. The safer maintainable direction is modular source that generates and verifies the single public installer artifact.
