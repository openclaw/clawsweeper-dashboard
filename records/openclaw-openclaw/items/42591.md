Keep open: current main and the latest release still ship a single large public installer script, and the requested module split is not present. The cleanup is reasonable, but it needs maintainer direction because the hosted `curl | bash` installer contract and sync workflow must stay intact.

## What This Changes

The issue requests splitting the large public `scripts/install.sh` installer into smaller shell modules for UI, download, platform, npm, git, and build-tool responsibilities.

## Best Possible Solution

Decide the public installer artifact contract first, then split internal modules behind a preserved self-contained or generated `install.sh` entrypoint with updated website-sync, smoke, and unit coverage.

## Solution Assessment

Unclear. Modularization is plausible, but maintainers need to choose whether the hosted installer stays checked-in standalone, is generated from modules, or uses another public boundary before implementation.
