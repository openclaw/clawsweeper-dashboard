## Summary

Keep open: the PR is a plausible security bug fix, but the current patch still leaves permission-only changes out of live gateway pairing paths and the external real behavior proof gate is unmet under a protected label.

## What This Changes

The PR changes node pairing reconciliation to replace pending requests when caps, commands, or permissions change, returns superseded request ids, broadcasts resolved events, and adds infra regression coverage.

## Best Possible Solution

Keep the replacement-request behavior, but thread permissions through the WebSocket and RPC pairing inputs, align schema/docs/tests, and land only after maintainer review plus real behavior proof.

## Reproduction Assessment

Yes from source inspection: current main refreshes any single pending node request because `canRefreshSingle` always returns true. I did not run tests because this review is read-only.

## Solution Assessment

No: replacing changed approval surfaces is the right direction, but this patch does not make permission changes reach the pairing comparison through the actual WebSocket or RPC gateway paths.
