## Summary

Keep open. Current main still has the reported CI maintainability problem: the main workflow is large, preflight still exports many coupled routing and matrix outputs, matrices remain inline, and the requested JSON/reusable-workflow split is absent.

## What This Changes

Refactor the main CI workflow by extracting matrix definitions into JSON, moving repeated CI patterns into reusable workflows, and documenting the new structure.

## Best Possible Solution

Land a staged CI decomposition that preserves the current job graph, required check names, smart-scope semantics, checkout trust model, artifact/cache flow, and docs, with workflow sanity and real PR CI proof after each step.

## Reproduction Assessment

Not applicable as a runtime bug. The maintenance problem is directly verifiable from current main by inspecting .github/workflows/ci.yml, the absent requested files, and the CI docs.

## Solution Assessment

Unclear whether the exact JSON-config plus reusable-workflow split is the best final design. The safer path is a staged extraction with tests around generated shard contracts and required check names before shrinking the main workflow.
