## Summary

Keep open. Current main still has the reported CI maintainability problem: the main workflow is large, preflight still exports many coupled routing and matrix outputs, matrices are still built inline, and the requested extracted matrix/reusable-workflow structure is absent.

## What This Changes

Request to split the large CI workflow by extracting matrix definitions, reusable workflows, and workflow documentation without changing CI behavior.

## Best Possible Solution

Land a staged CI decomposition that preserves the current job graph, required check names, smart-scope semantics, checkout trust model, artifact/cache flow, and docs, with workflow sanity and real PR CI proof after each step.

## Reproduction Assessment

Not applicable as a runtime bug. The maintenance request is directly verifiable from current main by inspecting `ci.yml`, the absent requested files, and the CI docs.

## Solution Assessment

Unclear whether the exact JSON-config plus reusable-workflow split is the best final design. The safest path is a staged extraction with tests around generated shard contracts and required check names before shrinking the main workflow.
