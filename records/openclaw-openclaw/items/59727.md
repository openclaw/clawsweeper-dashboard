## Summary

Keep open. Current main still has the reported CI maintainability problem: the main workflow is far above the requested size, routing and matrices are still produced inside `ci.yml`, downstream jobs still consume many preflight outputs directly, and the requested JSON matrix/reusable-workflow/doc split is absent.

## What This Changes

The issue requests refactoring the large CI workflow into extracted matrix configs, reusable workflows, and workflow documentation while preserving CI behavior.

## Best Possible Solution

Land a staged CI decomposition that preserves the current job graph, required check names, smart-scope semantics, checkout trust model, artifact/cache flow, and docs, with workflow sanity and real PR CI proof after each step.

## Reproduction Assessment

Not applicable as a runtime bug. The maintainability issue is source-verifiable on current main by inspecting `.github/workflows/ci.yml`, the missing requested files, and the CI docs.

## Solution Assessment

Unclear whether the exact JSON-config plus reusable-workflow split is the best final design. A safer solution is a staged extraction with tests around generated shard contracts and required check names before shrinking the main workflow.
