## Summary

Keep open. Current main still has the CI maintainability problem: the main workflow is large, preflight still exports many coupled outputs, matrices are still generated inline, and the requested extracted matrix/reusable-workflow/documentation structure is absent. This is real work, but it is broad CI control-plane work that needs maintainer-approved design rather than conservative cleanup automation.

## What This Changes

Refactor the main CI workflow by extracting matrix definitions to JSON configs, moving repeated job patterns into reusable workflows, and documenting the workflow structure.

## Best Possible Solution

Land a staged, maintainer-approved CI decomposition that preserves the current job graph, required check names, smart-scope semantics, checkout trust model, and artifact/cache flow, with workflow docs updated once the new structure is proven in PR CI.

## Reproduction Assessment

Not applicable as a bug reproduction. The maintenance request is directly verifiable from current main by inspecting `ci.yml`, the absent requested files, and the existing CI docs.

## Solution Assessment

Unclear whether the exact static JSON and reusable-workflow split is the best final shape. A safer approach is to extract repeated checkout/setup/build patterns in small steps while keeping dynamically generated shard contracts covered by tests until an equivalent config contract exists.
