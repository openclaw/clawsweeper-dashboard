## Summary

Keep open. Current main still has the reported CI maintainability problem: the main CI workflow remains large, preflight still exports many routing and matrix outputs, matrices are still built inside the workflow, and there is no canonical open PR owning the requested JSON/reusable-workflow split. The related merged preflight refactor improved routing but did not implement this issue’s decomposition.

## What This Changes

Request to decompose the main CI workflow by extracting matrix definitions, reusable workflow patterns, and workflow documentation without changing CI behavior.

## Best Possible Solution

Land a staged CI decomposition that preserves the current job graph, required check names, smart-scope semantics, checkout trust model, artifact/cache flow, and docs, with workflow sanity and real PR CI proof after each step.

## Reproduction Assessment

Not applicable as a runtime bug. The maintainability issue is source-verifiable on current main by inspecting .github/workflows/ci.yml, the absence of .github/matrices and .github/workflows/README.md, and the current CI docs.

## Solution Assessment

Unclear whether the exact JSON-config plus reusable-workflow split is the best final design. A safer solution is a staged extraction with guardrails around generated shard contracts, required check names, checkout trust, and live PR CI behavior.
