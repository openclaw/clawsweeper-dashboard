## Summary

Keep open: current main still plans per-agent models.json from the full configured provider map and merge mode preserves existing provider entries, so the reported credential-boundary issue remains source-reproducible and unsuperseded.

## What This Changes

The issue requests scoping each agent's generated models.json so it contains only that agent's credential-bearing provider entry, while keeping bridge cross-validation as defense-in-depth.

## Best Possible Solution

Define an owner-reviewed per-agent credential materialization rule that removes sibling credential-bearing providers from generated models.json while preserving safe shared catalog metadata and existing SecretRef marker semantics.

## Reproduction Assessment

Yes, at source level: a config with multiple credential-bearing models.providers entries flows wholesale into the per-agent models.json planner, and merge mode retains existing provider entries. I did not run a live saqi bridge setup.

## Solution Assessment

Unclear: the requested credential isolation direction is sound, but the exact fix should be owner-reviewed so it does not remove legitimate shared catalog or fallback metadata.
