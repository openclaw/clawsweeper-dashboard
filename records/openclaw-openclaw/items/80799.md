## Summary

Keep open: current `main` still has a source-provable credential-boundary gap in generated per-agent `models.json`, and the related open PR only addresses plaintext auth-profile key persistence, not sibling provider scoping.

## What This Changes

Scope each agent's generated `models.json` to that agent's own provider entry while keeping the existing cross-validation defense.

## Best Possible Solution

Define an owner-reviewed per-agent credential materialization rule that removes sibling credential-bearing providers from generated `models.json` while preserving safe shared catalog metadata and SecretRef marker behavior.

## Reproduction Assessment

Yes, at source level: current planning merges all explicit `models.providers`, and merge mode preserves existing provider rows before applying new entries. I did not run a live saqi bridge setup.

## Solution Assessment

Unclear: scoping credential-bearing providers is the right direction, but the exact rule needs owner review so shared non-secret catalog and fallback behavior are not accidentally removed.
