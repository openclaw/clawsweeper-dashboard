## Summary

Keep open: current main still has a source-provable per-agent models.json credential-boundary problem, and no merged fix or canonical duplicate now owns the remaining work.

## What This Changes

Request that generated per-agent models.json files include only that agent's own credential-bearing provider entry instead of sibling providers and bearer keys.

## Best Possible Solution

Define an owner-reviewed per-agent credential materialization contract that filters credential-bearing providers per agent while preserving safe shared catalog metadata and SecretRef marker behavior.

## Reproduction Assessment

Yes, at source level: a multi-provider cfg.models.providers object is planned wholesale, and merge mode retains existing providers before applying new entries. I did not run a live saqi bridge setup.

## Solution Assessment

Unclear: scoping credential-bearing providers is the right security direction, but the exact rule needs owner review so shared non-secret catalog and fallback behavior are not accidentally removed.
