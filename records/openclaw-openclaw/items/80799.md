## Summary

Keep open: current main still has a source-provable per-agent models.json credential-boundary problem, and the remaining fix is security-sensitive rather than safe maintainer cleanup.

## What This Changes

The issue requests scoping each agent's generated models.json so credential-bearing provider entries are limited to that agent's own provider.

## Best Possible Solution

Define an owner-reviewed per-agent credential materialization rule that removes sibling credential-bearing providers from generated models.json while preserving safe catalog metadata and SecretRef marker behavior.

## Reproduction Assessment

Yes. Source inspection of current main shows a multi-provider cfg.models.providers object is planned wholesale, and merge mode retains existing provider rows before applying new entries.

## Solution Assessment

Unclear. Scoping credential-bearing providers per agent is the right security direction, but the exact rule needs maintainer/security review so safe shared catalog and fallback behavior are not accidentally removed.
