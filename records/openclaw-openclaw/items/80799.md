## Summary

Keep open: current main still has a source-level path for cross-agent provider credential disclosure through generated or merged models.json data, and I did not find a merged fix or canonical superseding item for the per-agent scoping request.

## What This Changes

The issue asks OpenClaw to scope each generated agent models.json to that agent's own provider entry instead of exposing sibling provider credentials.

## Best Possible Solution

Define an owner-reviewed per-agent credential materialization rule that removes sibling credential-bearing providers from generated models.json while preserving legitimate shared catalog metadata, source-managed SecretRef markers, and bridge cross-validation.

## Reproduction Assessment

Yes, at source level. A config with multiple credential-bearing models.providers entries, or an existing models.json with sibling providers, will be written or retained because current planning is not agent-scoped and merge mode preserves existing provider entries.

## Solution Assessment

Unclear: per-agent scoping is the right security direction, but the safest implementation needs an explicit owner decision on credential-bearing entries versus shared provider catalog metadata.
