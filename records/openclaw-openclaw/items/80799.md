## Summary

Keep open: current main still has a source-provable per-agent models.json credential-boundary problem, and I did not find a merged fix or canonical duplicate that owns the remaining work.

## What This Changes

The issue asks OpenClaw to scope each agent's generated models.json so it contains only that agent's provider entry instead of sibling provider credentials.

## Best Possible Solution

Define an owner-reviewed per-agent credential materialization rule that removes sibling credential-bearing providers from generated models.json while preserving safe shared catalog metadata, SecretRef markers, and documented merge behavior.

## Reproduction Assessment

Yes, at source level: a multi-provider cfg.models.providers object is planned wholesale for the target agent, and merge mode retains existing provider entries before applying new entries. I did not run a live saqi bridge setup.

## Solution Assessment

Unclear: scoping credential-bearing providers is the right security direction, but the exact owner/provider boundary needs maintainer review so shared non-secret catalog and fallback behavior are not accidentally removed.
