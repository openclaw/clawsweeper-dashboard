## Summary

Keep open: current main and latest release still materialize and retain configured provider entries in per-agent models.json without an agent/provider boundary, so the reported credential disclosure needs owner/security follow-up.

## What This Changes

The issue asks for each per-agent models.json to contain only that agent's own credential-bearing provider instead of sibling saqi-max provider entries.

## Best Possible Solution

Define and implement an owner-reviewed per-agent credential materialization rule that excludes sibling credential-bearing providers while preserving safe shared catalog metadata and SecretRef marker behavior.

## Reproduction Assessment

Yes, at source level: a config with multiple models.providers entries is planned wholesale into the per-agent models.json, and merge mode keeps existing providers before applying new entries. I did not run the live saqi bridge scenario.

## Solution Assessment

Unclear: scoping credential-bearing providers is the right direction, but the exact rule needs owner/security review so safe shared catalog and fallback behavior are not accidentally removed.
