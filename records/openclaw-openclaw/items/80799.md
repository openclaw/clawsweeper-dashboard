## Summary

Keep open: current main and the latest release still generate an agent-local models.json from the full configured provider map and merge mode can preserve already-present sibling provider entries, so the requested per-agent provider scoping is still unresolved.

## What This Changes

The issue asks OpenClaw to scope each generated agent models.json to only that agent's own model-provider entry so sibling provider credentials are not readable by another agent.

## Best Possible Solution

Define a maintainer-approved per-agent provider ownership rule, update models.json generation and merge cleanup to emit only authorized provider entries per agent, and preserve SecretRef marker handling plus bridge cross-validation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: generation starts from the full configured provider map and merge mode preserves self-contained existing provider entries; I did not run a live command because this read-only review cannot write agent state.

## Solution Assessment

Yes on the security direction, but the implementation shape is not settled. The best fix needs a maintainer-owned provider scoping rule plus cleanup semantics, not an ad hoc filter by provider name.
