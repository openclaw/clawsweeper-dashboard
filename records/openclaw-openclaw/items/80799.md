## Summary

Keep open: current main still has a plausible credential-boundary issue in models.json materialization, and there is no source/history evidence that the requested per-agent scoping has already been fixed or superseded.

## What This Changes

Scope each agent's generated models.json to that agent's own provider credentials instead of exposing sibling agent provider entries.

## Best Possible Solution

Define an owner-reviewed per-agent credential materialization rule that removes sibling credential-bearing providers from generated models.json while preserving legitimate shared catalog metadata and existing SecretRef marker behavior.

## Reproduction Assessment

Yes, at source level: a config with multiple credential-bearing models.providers entries or an existing models.json with sibling providers will be written or retained because current merge logic is not agent-scoped. I did not run a live saqi setup, so this is not a live reproduction.

## Solution Assessment

Unclear: the requested direction is right for credential isolation, but the safest fix should be narrower than dropping every non-own provider if shared catalog entries or fallbacks are intentionally supported.
