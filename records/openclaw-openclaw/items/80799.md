## Summary

Keep open: current main still has a source-provable per-agent models.json credential-boundary risk, and I did not find a merged fix or narrower canonical duplicate that owns the sibling-provider scoping work.

## What This Changes

The issue requests scoping each generated per-agent models.json so it does not include sibling agents' credential-bearing provider entries.

## Best Possible Solution

Define an owner-reviewed per-agent provider materialization rule that removes sibling credential-bearing providers from generated models.json while preserving safe shared catalog metadata and existing SecretRef marker behavior.

## Reproduction Assessment

Yes, at source level: cfg.models.providers is merged wholesale into each generated plan, and merge mode preserves existing provider entries. I did not run a live saqi bridge setup.

## Solution Assessment

Unclear: scoping credential-bearing providers is the right security direction, but the exact rule needs owner review so shared non-secret catalog and fallback behavior are not accidentally removed.
