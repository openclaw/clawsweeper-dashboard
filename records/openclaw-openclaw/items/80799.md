## Summary

Keep open: current main and v2026.6.8 still generate agent `models.json` from the full configured provider map and merge mode can preserve already-present sibling provider entries, so the requested per-agent provider scoping is not implemented.

## What This Changes

The issue requests that each agent's generated `models.json` include only that agent's own provider entry so sibling provider credentials are not readable from another agent's config.

## Best Possible Solution

Define a maintainer-approved per-agent provider ownership rule, update generation and merge cleanup to emit only authorized provider entries per agent, and preserve SecretRef marker handling plus bridge cross-validation.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: generation starts from all configured providers and merge mode preserves self-contained existing provider entries. I did not run a live command because the path can rewrite agent state.

## Solution Assessment

No, current main is not the best end state because it does not enforce per-agent provider scoping. The requested security direction is sound, but ownership and cleanup semantics need maintainer decision before implementation.
