## Summary

Keep open: current main still has a source-reproducible credential-boundary gap in per-agent `models.json` materialization, and no merged or canonical fix supersedes this issue.

## What This Changes

The issue asks OpenClaw to scope each generated agent `models.json` so it contains only that agent's own credential-bearing provider entry.

## Best Possible Solution

Define an owner-reviewed per-agent credential materialization rule that removes sibling credential-bearing providers from generated `models.json` while preserving safe shared catalog metadata, legitimate fallbacks, and existing SecretRef marker behavior.

## Reproduction Assessment

Yes, at source level: a config with multiple credential-bearing `models.providers` entries is merged and serialized into the selected agent's `models.json`, and merge mode can retain existing sibling rows. I did not run a live saqi setup.

## Solution Assessment

Unclear: the requested direction is right for credential isolation, but the safest implementation needs a precise security/owner contract rather than dropping every non-own provider row unconditionally.
