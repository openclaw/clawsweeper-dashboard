## Summary

Keep open: current `main` still has a source-visible per-agent `models.json` scoping gap. The related plaintext SecretRef persistence bug is fixed, but the planner still starts from the full provider map and merge mode can preserve already-present sibling provider entries, so the central credential-disclosure hardening remains unresolved.

## What This Changes

Restrict each generated per-agent `models.json` to that agent's own provider entry so sibling provider bearer keys are not readable from another agent profile.

## Best Possible Solution

Define a maintainer-approved per-agent provider ownership rule, update generation and merge cleanup to emit only authorized provider entries, and add migration/proof that preserves SecretRef markers and bridge validation.

## Reproduction Assessment

No live high-confidence reproduction was run because this review is read-only and the path writes agent state. Source inspection supports the reported shape with medium confidence: current generation uses the full provider map and merge mode preserves self-contained existing provider entries.

## Solution Assessment

No, the current implementation is not the best end state because it does not implement per-agent provider scoping. The requested direction is valid security hardening, but the exact ownership signal and cleanup behavior need maintainer/security design first.
