## Summary

Keep open: current main still has a source-provable credential-boundary issue in per-agent `models.json` generation, and I did not find a merged fix or canonical duplicate that owns the remaining work.

## What This Changes

The issue asks for each agent's generated `models.json` to exclude sibling providers and bearer keys while preserving bridge cross-validation defense-in-depth.

## Best Possible Solution

Add an owner-reviewed per-agent materialization rule that writes only allowed credential-bearing providers for the target agent, prunes stale sibling entries on regeneration, and preserves safe catalog metadata and SecretRef marker semantics.

## Reproduction Assessment

Yes, at source level: explicit `cfg.models.providers` is planned wholesale and merge mode preserves existing providers before applying new entries. I did not run a live saqi bridge setup.

## Solution Assessment

Unclear: per-agent scoping is the right security direction, but the exact rule needs owner/security review so shared non-secret catalog metadata and fallback behavior are not accidentally removed.
