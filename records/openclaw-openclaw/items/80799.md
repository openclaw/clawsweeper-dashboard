## Summary

Keep open: current main still has a source-provable credential-boundary problem in per-agent models.json materialization, and no merged fix or canonical duplicate was found.

## What This Changes

The issue asks for each agent's generated models.json to contain only that agent's own credential-bearing provider while keeping bridge cross-validation as defense in depth.

## Best Possible Solution

Define an owner-reviewed per-agent credential materialization rule that removes sibling credential-bearing providers from generated models.json while preserving legitimate shared catalog metadata and SecretRef marker behavior.

## Reproduction Assessment

Yes, at source level: a multi-provider cfg.models.providers object is planned wholesale, and merge mode retains existing providers before applying new entries. I did not run a live saqi bridge setup.

## Solution Assessment

Unclear: scoping credential-bearing providers is the right security direction, but the exact rule needs owner review so shared non-secret catalog and fallback behavior are not accidentally removed.
