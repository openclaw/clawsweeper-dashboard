## Summary

Keep open: current main still has a source-provable per-agent credential-boundary gap in models.json materialization, and the related search found broader or adjacent security/model-catalog items but no merged fix or canonical duplicate for this exact provider-scoping problem.

## What This Changes

The issue requests that each agent's generated models.json include only its own credential-bearing provider entry instead of sibling agents' provider entries.

## Best Possible Solution

Define and implement an owner-reviewed per-agent credential materialization contract that excludes sibling credential-bearing providers while preserving safe shared catalog metadata and SecretRef marker behavior.

## Reproduction Assessment

Yes, at source level: the planner uses cfg.models.providers wholesale and merge mode retains existing providers before applying new entries. I did not run a live saqi bridge setup, so runtime bridge-log acceptance remains unverified.

## Solution Assessment

Unclear: scoping credential-bearing providers is the right security direction, but the exact rule needs owner/security review. A safer fix is credential-aware scoping in the planner/merge path rather than deleting every non-primary catalog row indiscriminately.
