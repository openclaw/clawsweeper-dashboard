## Summary

Keep open: current main still plans each agent's models.json from the full configured provider map and merge mode still preserves existing provider entries, so the requested credential-boundary scoping is not implemented or superseded.

## What This Changes

The issue requests that each generated per-agent models.json include only that agent's own credential-bearing provider entry while preserving bridge cross-validation.

## Best Possible Solution

Define an owner-reviewed per-agent credential materialization rule that removes sibling credential-bearing providers from generated models.json while preserving safe shared catalog metadata, SecretRef markers, and documented merge semantics.

## Reproduction Assessment

Yes, at source level: current main writes per-agent models.json from cfg.models.providers as a whole and merge mode preserves existing provider entries. I did not run a live saqi setup, so this is not a live reproduction.

## Solution Assessment

Unclear. The requested isolation direction is right for credential boundaries, but the maintainable fix should be narrower than deleting every non-own provider entry if shared catalog metadata or fallbacks are intentional.
