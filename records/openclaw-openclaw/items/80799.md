## Summary

Keep open: current main still has a source-provable credential-boundary problem in per-agent models.json materialization, and I did not find a merged fix or canonical duplicate. The existing ClawSweeper comment reached the same conclusion; the open related model-auth PR is adjacent credential work but does not scope generated models.json provider entries.

## What This Changes

The issue requests scoping each agent's generated models.json so it contains only that agent's own credential-bearing provider entry.

## Best Possible Solution

Define and implement an owner-reviewed per-agent credential materialization rule that filters credential-bearing provider entries and purges retained stale siblings while preserving allowed non-secret catalog metadata and SecretRef marker behavior.

## Reproduction Assessment

Yes. Current main is source-reproducible: a multi-provider cfg.models.providers object is planned wholesale for an agent, and merge mode preserves existing provider entries before applying new ones; I did not run a live saqi bridge setup.

## Solution Assessment

Unclear. Scoping credential-bearing providers is the right security direction, but the exact rule needs owner/security review so shared non-secret catalog and fallback behavior are not accidentally removed.
