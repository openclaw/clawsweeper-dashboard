## What This Changes

The issue requests removing OpenClaw's client-side iMessage split-send coalescing after upstream `imsg` coalescing is the supported floor.

## Best Possible Solution

After maintainer confirmation of the `imsg >= 0.11.1` floor, remove the local coalescing config/runtime/docs/tests with a doctor migration and focused iMessage plus live validation.

## Solution Assessment

Yes as the intended end state, but not as an immediate autonomous fix while the protected maintainer/product decision remains. The narrow maintainable solution is a timed removal after declaring the `imsg >= 0.11.1` floor, not another compatibility-preserving fallback PR.
