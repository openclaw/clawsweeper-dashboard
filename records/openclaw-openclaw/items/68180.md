## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Include the XDG fnm default path:** `src/daemon/service-env.ts:166-167`
  - body: The patch adds `~/.fnm/aliases/default/bin`, but modern Linux fnm defaults to `~/.local/share/fnm/aliases/default/bin` unless `FNM_DIR` is present. Without that default path, the generated service PATH can still miss the setup reported in #68169.
  - confidence: 0.88
- **[P2] Redact Telegram input before truncating it:** `src/telegram/send.ts:385-389`
  - body: The new preview truncates `params.input` before calling `redactSensitiveText`. If a token starts near the 128-character boundary, the shortened token may no longer match redaction patterns and can leak a secret prefix in the error text.
  - confidence: 0.8

## Security Review

Status: needs_attention

Summary: The PR includes hidden security-sensitive ACP and Telegram changes, and the Telegram redaction change can still leak partial secrets because it truncates before redacting.

Concerns:

- **[medium] Redact before truncating Telegram previews:** `src/telegram/send.ts:385`
  - body: The Telegram error preview clamps the raw input first and only then applies redaction, so a token cut at the preview boundary may fall below the recognizer shape and leak a partial secret in the emitted error.
