## Summary

Keep open. Current main already shipped the stated daemon fnm fix for #68169 and covers the Feishu/Slack equivalents differently, but this PR still carries unrelated ACP and Telegram security-sensitive changes, and the proposed daemon hunk is less complete than main.

## What This Changes

The PR branch adds Linux fnm `aliases/default/bin` handling to daemon service PATH resolution and also changes ACP session-list limits, cron Feishu/Lark delivery matching, Slack channel-type coercion, and Telegram chat-not-found input previews.

## Best Possible Solution

Replace this with focused outcomes: keep the shipped daemon, cron, and Slack behavior on main, and split any desired ACP limit clamp or Telegram error redaction into separate security-reviewed PRs with regression tests.

## Reproduction Assessment

Yes. The linked daemon bug has concrete Linux fnm steps, and source inspection on current main verifies the fixed paths; the PR findings are also reproducible by reading the proposed daemon and Telegram hunks.

## Solution Assessment

No. Current main is the narrower and more complete daemon fix, while the ACP and Telegram behavior should be handled as separate, reviewed patches rather than merged as side effects of this PR.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Include the XDG fnm default path:** `src/daemon/service-env.ts:166-167`
  - body: The daemon hunk adds `${home}/.fnm/aliases/default/bin` but omits `${home}/.local/share/fnm/aliases/default/bin`, which is the default Linux fnm layout described in #68169. A service generated from this patch can still miss `node` for modern fnm installs unless `FNM_DIR` is set.
  - confidence: 0.88
- **[P2] Redact Telegram input before truncating it:** `src/telegram/send.ts:385-389`
  - body: The new preview slices raw `params.input` before redaction. A token crossing the 128-character boundary can be shortened below the recognizer shape, leaving a secret prefix in the error message; redact first, then clamp.
  - confidence: 0.8

## Security Review
