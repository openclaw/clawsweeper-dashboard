## Summary

Keep this open: current main and the latest release still hardcode the public npm registry for update metadata, and the linked fixing PR is still open with closing syntax, so the issue should remain the canonical tracker until a fix lands.

## What This Changes

The issue asks OpenClaw update metadata checks to honor npm registry mirror configuration instead of always fetching from `registry.npmjs.org`.

## Best Possible Solution

Land the linked PR or a replacement that centralizes registry-base resolution in the npm metadata helper, covers env and configured registry inputs, and preserves the existing timeout/error-swallowing behavior.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: on current main, every `fetchNpmPackageTargetStatus` path still builds a `registry.npmjs.org` URL regardless of `npm_config_registry`; I did not run a full gateway update check in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix belongs in the central npm metadata fetch path so startup, status, and update preflight callers inherit the same resolver and tests.
