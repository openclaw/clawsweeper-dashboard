## What This Changes

The issue asks for update metadata checks to respect npm registry mirror configuration instead of always fetching from registry.npmjs.org.

## Best Possible Solution

Add a central update metadata registry resolver that honors npm registry env and npmrc/userconfig/home config, wire `fetchNpmPackageTargetStatus` through it, and prove the mirror path with focused tests plus real behavior output.

## Reproduction Assessment

## Solution Assessment
