## Summary

This should stay open: current main still hardcodes the public npm registry for update metadata, while the package-manager install path otherwise preserves npm/pnpm registry configuration.

## What This Changes

The issue requests that update checks use the user's configured npm registry or mirror instead of always querying the public npm registry.

## Best Possible Solution

Centralize registry-base resolution for update metadata fetches, honor existing npm/pnpm registry configuration or environment, and fall back to the public npm registry only when no mirror is configured.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: any update metadata path with `npm_config_registry` set still reaches a helper that builds `https://registry.npmjs.org/openclaw/<target>`. I did not run a full gateway update check because the read-only review can prove the path directly from source.

## Solution Assessment

Yes, the narrowest maintainable fix is in the central npm metadata fetch path, not in `update-channels.ts`; all callers should inherit the same registry resolver and tests.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/79140. Current main hardcodes `https://registry.npmjs.org/openclaw/<target>` in `src/infra/update-check.ts`, so update status, gateway startup update checks, beta fallback resolution, target-version resolution, and Node-engine preflight ignore npm/pnpm registry mirror configuration. Add a small central registry resolver for npm metadata fetches that honors existing package-manager registry configuration, at minimum `npm_config_registry` and `NPM_CONFIG_REGISTRY`, with safe URL normalization and fallback to `https://registry.npmjs.org` only when no usable registry is configured. If reading npm/pnpm config beyond env is implemented, keep it bounded and do not add unbounded subprocess work to the Gateway startup path. Preserve beta fallback semantics, Node-engine preflight behavior, timeout/error handling, and package install command behavior. Add regression tests in the update-check/update-startup/update-cli surface proving mirror URLs are used and trailing slashes are handled. Add a user-facing changelog fix entry crediting @sixerLiu unless maintainer policy says otherwise.
