## Summary

Keep open: current main still hardcodes the public npm registry for update metadata, and an open PR already references this issue with closing syntax, so the issue should close only after that fix or an equivalent replacement merges.

## What This Changes

The issue requests that OpenClaw update metadata checks honor npm registry mirror configuration instead of always querying the public npm registry.

## Best Possible Solution

Centralize registry-base resolution for update metadata, cover the existing npm registry configuration paths without adding an unbounded startup subprocess, and close this issue only after the linked fix merges.

## Reproduction Assessment

Yes. Source inspection on current main shows every update metadata lookup still reaches `fetchNpmPackageTargetStatus`, which builds a `registry.npmjs.org` URL directly even when npm registry env/config would point elsewhere.

## Solution Assessment

Yes, the right fix boundary is the central npm metadata fetch helper, with focused tests for env/config precedence and URL normalization; the current issue should remain open while the linked implementation PR is reviewed.
