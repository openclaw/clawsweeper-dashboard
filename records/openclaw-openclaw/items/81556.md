## Summary

Keep open. Current main still rewrites the stable Flash-Lite model id to the preview id in Google provider normalization, and the current Gemini API docs treat the stable and preview endpoints as distinct with the preview scheduled to migrate to the stable model.

## What This Changes

The issue requests that OpenClaw stop remapping `gemini-3.1-flash-lite` to `gemini-3.1-flash-lite-preview` before Google provider calls.

## Best Possible Solution

Remove only the stable Flash-Lite-to-preview alias from Google model normalization and manifest metadata, keep explicit preview ids accepted, and add regression coverage that the stable id survives to the Google request model id.

## Reproduction Assessment

Yes, at source level: resolving `google/gemini-3.1-flash-lite` on current main passes through Google/static normalization that returns `gemini-3.1-flash-lite-preview`. I did not run a live provider request in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to remove the stale stable-to-preview alias in the Google-owned normalization surfaces and cover it with focused model-id tests, without changing unrelated Gemini Pro/Flash compatibility aliases.
