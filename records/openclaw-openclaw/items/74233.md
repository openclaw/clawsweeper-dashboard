## Summary

Keep this open. Current main still has the reported host-side behavior: context-engine assembly updates the active messages, but the embedded runner still passes the pre-assembly snapshot into the hard overflow precheck, and the precheck helper still lets the larger unwindowed estimate win. The issue is also paired with open closing PR #74255, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to make context-engine assembled messages authoritative for prompt admission instead of hard-gating the turn on larger pre-assembly history.

## Best Possible Solution

Land or adapt #74255 so successful context-engine assembly is prompt-authoritative by default, raw pre-assembly history is explicit opt-in or separate maintenance/debt telemetry, and the precheck tests/docs reflect that final contract.

## Reproduction Assessment

Yes. Source and test inspection give a high-confidence reproduction shape: pass a small assembled `messages` array and a much larger `unwindowedMessages` array into the current precheck helper, then observe that the larger unwindowed estimate drives compaction.

## Solution Assessment

Yes. The narrow maintainable direction is a generic context-engine prompt-authority contract or default, not a lossless-claw-specific bypass and not removal of overflow protection entirely.
