## Summary

Keep open: the PR targets a real current-main cron self-scope regression, but the patch still filters a single paged `cron.list` response after dispatch, so a current job outside the first 200 returned jobs is reported as missing; the changed tool description also leaves prompt snapshots failing.

## What This Changes

This PR lets self-remove-scoped cron runs call `cron.status` and a session-agent-filtered `cron.list`, filters list results to the current job, and updates cron tests, docs, and the changelog.

## Best Possible Solution

Land this PR or an equivalent patch after self-scoped list lookup reliably finds the current job across pagination or by id lookup, prompt snapshots are updated, and head checks are green.

## Reproduction Assessment

Yes. Source inspection on current `main` shows self-remove-scoped `status` and `list` are rejected before gateway dispatch, and the linked regression report has matching v2026.5.4 gateway logs; this read-only review did not run tests.

## Solution Assessment

No, not yet. The permission shape is narrow, but the self-scoped `list` implementation must find the current job across pagination before returning a current-job-only response, and the prompt snapshot check must be made green.
