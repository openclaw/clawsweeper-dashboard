## Summary

Keep open: current main still lacks the Active Time card, but this PR still has a blocking metric semantics issue and no inspectable real behavior proof from the contributor.

## What This Changes

This PR adds a localized Active Time card to the Control UI Usage Overview, passes selected day/hour filters into duration stats, and updates focused usage tests plus generated locale metadata.

## Best Possible Solution

Land a narrow localized Active Time card after aligning or hiding duration-based throughput when totals are not sliced to the same time window, then require redacted real UI proof.

## Reproduction Assessment

Yes for the blocking review finding: PR-head source shows selected-hour totals still use whole-session usage while the new duration calculation is clipped to the selected window. The PR's own test asserts whole-session tokens over a clipped one-hour duration.

## Solution Assessment

No. The Active Time card direction is reasonable, but the patch should either align the throughput numerator with the clipped active-time window or suppress throughput when totals are not sliced to the same basis.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87
