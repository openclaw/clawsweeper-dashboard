## Summary

Keep this PR open for normal review: current main still lacks the Active Time card, but the latest branch still has a metric correctness issue and the external contributor has not attached inspectable real behavior proof.

## What This Changes

The PR adds a localized Active Time card to the Control UI Usage Overview, clips duration stats by selected day/hour filters, and updates focused usage metrics/rendering tests plus generated i18n metadata.

## Best Possible Solution

Land a narrow Active Time overview card after keeping filtered duration and throughput semantics on the same basis, with focused UI/metrics coverage and real after-fix UI proof.

## Reproduction Assessment

Yes for the blocking review finding: source inspection shows selected-hour totals still use whole matching sessions while the PR clips duration before computing throughput. I did not run the UI because this review was read-only.

## Solution Assessment

No. The Active Time card direction is reasonable, but the current patch should either align the throughput numerator with clipped active time or suppress throughput when totals are not sliced to the same filter window.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Align filtered throughput with active time:** `ui/src/ui/views/usage.ts:325-329`

A partially overlapping session can therefore show all tokens and cost over only the clipped active minutes, inflating the Throughput card as users narrow the hour filter.
