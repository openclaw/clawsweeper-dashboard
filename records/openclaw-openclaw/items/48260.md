## Summary

Keep open: this PR adds a useful Usage Overview feature that is not on current main, but PR head still has a filtered-throughput consistency bug and lacks inspectable after-fix real behavior proof.

## What This Changes

The PR adds an Active Time card to the Control UI Usage Overview, updates duration-filtering metrics/tests, and refreshes generated i18n locale metadata.

## Best Possible Solution

Land a narrow Active Time card after aligning filtered throughput with the same totals/duration window and attaching redacted real UI proof.

## Reproduction Assessment

Yes for the blocking PR finding: PR-head source shows selected-hour totals still use whole-session totals while active duration is clipped. The added test also asserts 120 tokens over one clipped hour, preserving the mismatch.

## Solution Assessment

No. The Active Time card direction is reasonable, but the patch should align filtered throughput totals with the clipped active-time window or suppress throughput when those values are not on the same basis.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Align filtered throughput with active time:** `ui/src/ui/views/usage.ts:325-329`
  - body: Passing selected day/hour filters into `buildUsageInsightStats` clips active duration, but selected-hour `displayTotals` still sums each matching session's full usage totals. A partially overlapping session can therefore show all tokens and cost over only the clipped active minutes, inflating Throughput as users narrow the hour filter.
