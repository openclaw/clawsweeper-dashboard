## Summary

Keep this PR open. Current main still lacks the Active Time overview card, and the PR is a useful narrow UI feature, but the latest patch still lets hour-filtered throughput divide full-session token/cost totals by a clipped active-time denominator; it also needs the required user-facing changelog entry before merge.

## What This Changes

The PR adds a localized Active Time card to the Control UI Usage Overview, clips active-duration stats to selected day/hour filters, updates focused usage tests, and regenerates Control UI locale metadata/bundles.

## Best Possible Solution

Land the Active Time card as a focused Control UI improvement after making throughput use the same filtered scope as the active-time denominator, or suppressing throughput for hour-filter slices where token/cost totals cannot be sliced accurately, and add the required Unreleased changelog entry with contributor credit.

## Reproduction Assessment

Yes. A session from 09:30 to 11:30 with 120 tokens and hour 10 selected exercises the bug: the PR clips active duration to 60 minutes while displayTotals still carries the full-session 120 tokens, so throughput reports 2 tok/min instead of an aligned value or an empty throughput card.

## Solution Assessment

No. The UI direction is reasonable, but the proposed implementation is not the best merge path until throughput numerator and denominator scopes are aligned; the narrower safe fix is to suppress throughput for unsupported hour slices or compute a sliced token/cost numerator from precise buckets if available.
