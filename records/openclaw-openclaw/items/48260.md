## Summary

Keep this PR open. Current main still lacks the Active Time overview card, but the latest PR diff still makes the Throughput card divide full selected-session totals by a clipped hour/day duration, so selected-hour filters can overstate throughput; it also needs the required changelog entry.

## What This Changes

The PR adds a localized Active Time summary card to the Control UI Usage Overview, clips active-duration stats to selected day/hour filters, and updates focused usage tests plus generated locale metadata.

## Best Possible Solution

Land the Active Time card as a narrow Control UI improvement after suppressing or correctly slicing throughput for selected-hour filters and adding one Unreleased changelog entry.

## Reproduction Assessment

Yes. A 120-token session from 23:30 to 01:30 UTC with hour 0 selected exercises the issue: the PR clips active duration to 60 minutes while displayTotals still carries the full 120 tokens, so throughput reports 2 tok/min from mismatched scopes.

## Solution Assessment

No. The UI direction is reasonable, but the current implementation is not the best merge path until the throughput numerator and denominator use the same scope; the safer fix is to suppress throughput for unsupported hour slices or compute sliced token/cost totals from precise buckets.
