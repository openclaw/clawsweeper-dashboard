## Summary

Keep open: current main still lacks a separate Active Time overview card, but this PR should not merge yet because the hour-filter path can inflate Throughput and the external contributor has not attached real behavior proof.

## What This Changes

This PR adds an Active Time summary card with localized strings and tests, clips usage insight duration by selected day/hour filters, and moves Error Rate subtext to message count.

## Best Possible Solution

Land a narrow Active Time card that keeps active-time and throughput math aligned for day/hour filters, then attach a real UI proof artifact before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path for the review finding: apply an hour filter to a session spanning multiple hours, and the PR clips duration while keeping full-session token/cost totals for Throughput.

## Solution Assessment

No. The Active Time card direction is reasonable, but the implementation should either align the Throughput numerator with the clipped duration or suppress Throughput when hour-filtered totals are not precise.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep throughput aligned when clipping active time:** `ui/src/ui/views/usage.ts:325-329`

Throughput then divides full-session totals by partial active time, so a session spanning multiple hours can report an inflated tok/min and cost/min after users narrow the hour filter.
