## Summary

Keep this PR open: current main still does not have an Active Time card, but the latest branch is not mergeable as-is because filtered throughput can be computed from whole-session totals over clipped active time, and the external PR still lacks inspectable real behavior proof.

## What This Changes

The PR adds a localized Active Time card to Control UI Usage Overview, clips duration stats by selected day/hour filters, updates focused usage metrics/rendering tests, and regenerates UI locale metadata.

## Best Possible Solution

Land a narrow Active Time overview card after aligning duration and throughput semantics, adding focused regression coverage, and attaching real after-fix UI proof.

## Reproduction Assessment

Yes for the blocking review finding: source inspection of PR head shows selected-hour totals still use whole-session usage while the new duration calculation is clipped to the selected window. The PR's own test also asserts whole-session tokens over a clipped one-hour duration.

## Solution Assessment

No. The Active Time card direction is reasonable, but the patch should either align the throughput numerator with the clipped active time window or suppress throughput when totals are not sliced to the same basis.
