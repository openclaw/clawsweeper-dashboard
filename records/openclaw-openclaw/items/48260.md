This PR adds an Active Time card to the Control UI Usage Overview, clips usage insight duration by selected day/hour filters, adds focused usage metrics/render tests, and regenerates Control UI locale metadata.

## Best Possible Solution

Land a narrow Active Time card after aligning filtered throughput with the same totals/duration basis, or hiding throughput when those values are incomparable, with focused tests and redacted UI proof.

## Reproduction Assessment

Yes for the blocking PR finding: PR-head source shows selected-hour totals still use whole overlapping sessions while duration is clipped to selected hours. The added test also preserves that mismatch by expecting 120 tokens over one clipped hour.

## Solution Assessment

No. The Active Time card direction is reasonable, but the patch should align filtered throughput totals with the clipped active-time window or suppress throughput when the numerator and denominator are not comparable.
