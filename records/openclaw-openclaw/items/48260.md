## Summary

Keep open: current main still does not implement the Active Time overview card, and the latest external PR still needs real behavior proof plus a P2 throughput correctness fix before merge.

## What This Changes

The branch adds localized Active Time labels and a Usage Overview card showing total active time and average session duration, plus focused usage metrics/rendering tests.

## Best Possible Solution

Keep the narrow Active Time card direction, but align filtered active-time and throughput semantics or hide throughput when totals are not sliced, then attach real after-fix UI proof before merge.

## Reproduction Assessment

Yes for the blocking review finding: the latest PR diff shows selected-hour totals still come from whole matching sessions while active duration is clipped by selectedHours before throughput is computed. I did not run the UI because this review was read-only.

## Solution Assessment

No. The Active Time card is a reasonable narrow UI feature, but the current implementation should keep throughput numerator and denominator on the same filtered basis or suppress throughput for that case.
