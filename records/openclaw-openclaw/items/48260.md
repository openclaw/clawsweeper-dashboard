## Summary

Keep open: current main still lacks the Active Time overview card, and the latest PR head still has a blocking metric-semantics issue because clipped active time is combined with whole-session selected-hour totals.

## What This Changes

This PR adds a localized Active Time card to the Control UI Usage Overview, passes selected day/hour filters into usage insight stats, updates focused usage rendering/metrics tests, and refreshes generated locale metadata.

## Best Possible Solution

Land a narrow Active Time overview card after aligning duration and throughput semantics, adding focused regression coverage, and attaching redacted real UI proof.

## Reproduction Assessment

Yes for the blocking review finding: PR-head source shows selected-hour totals still use whole-session usage while the new duration calculation is clipped to the selected window. The PR's own test asserts whole-session tokens over a clipped one-hour duration.

## Solution Assessment

No. The Active Time card direction is reasonable, but the patch should either align the throughput numerator with the clipped active-time window or suppress throughput when totals are not sliced to the same basis.
