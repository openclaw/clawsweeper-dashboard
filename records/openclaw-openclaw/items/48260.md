## Summary

Keep open. Current main still lacks the Active Time card, while the PR remains useful but still has a blocking throughput semantics bug for hour-filtered sessions and no inspectable real behavior proof attached.

## What This Changes

The PR adds a localized Active Time summary card to the Control UI Usage Overview, adds focused metrics/render tests, and refreshes generated locale metadata.

## Best Possible Solution

Land a narrow Active Time card after aligning throughput math with filtered active-time semantics and attaching inspectable real UI proof, with generated i18n kept in sync.

## Reproduction Assessment

Yes, from source inspection rather than a runtime test. A multi-hour session filtered to one selected hour now clips duration while retaining full-session totals, and the new test expectation confirms the inflated throughput path.

## Solution Assessment

No. The Active Time card direction is reasonable, but the implementation should either compute matching in-range token/cost numerators or suppress throughput when the denominator is clipped and the numerator is not.
