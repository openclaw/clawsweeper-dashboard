## What This Changes

The branch adds localized Active Time usage overview rendering, generated i18n metadata, and focused usage insight/rendering tests that clip active duration by selected day/hour filters.

## Best Possible Solution

Land a narrow Active Time card only after aligning duration-based throughput with the same filtered usage window, resolving conflicts, and attaching redacted real UI proof.

## Reproduction Assessment

Yes for the blocking review finding: PR-head source shows selected-hour totals still use whole-session usage while active duration is clipped to the selected window. The PR test also asserts all 120 tokens over one clipped hour, which preserves the mismatch.

## Solution Assessment

No. The Active Time card direction is reasonable, but the patch should align the throughput numerator with the clipped active-time denominator or suppress throughput when totals are not sliced to the same basis.
