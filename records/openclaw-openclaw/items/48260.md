## Summary

Keep open: current main and the latest release still lack the requested Active Time overview card, but the PR head still has a blocking throughput mismatch for selected hour/day filters and lacks inspectable real behavior proof before merge.

## What This Changes

This PR adds a localized Active Time card to the Control UI Usage Overview, clips active-time duration by selected day/hour filters, updates focused usage overview tests, and refreshes generated i18n metadata.

## Best Possible Solution

Land the localized Active Time card after making duration-based throughput use a consistent numerator and denominator for selected day/hour filters, then validate focused UI metrics/rendering coverage and attach redacted real UI proof.

## Reproduction Assessment

Yes for the blocking review finding: PR-head source shows selected-hour totals still use whole-session usage while active duration is clipped to the selected window. The PR test also asserts all 120 tokens over one clipped hour, which preserves the mismatch.

## Solution Assessment

No. The Active Time card direction is reasonable, but the patch should align the throughput numerator with the clipped active-time window or suppress throughput when totals are not sliced to the same basis.
