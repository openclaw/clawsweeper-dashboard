## What This Changes

This PR adds a localized Active Time card to the Control UI Usage Overview, clips duration stats by selected day/hour filters, updates focused usage tests, and refreshes generated Control UI locale metadata.

## Best Possible Solution

Land a narrow Active Time overview card after aligning duration and throughput semantics, adding focused regression coverage, rebasing cleanly, and attaching redacted real UI proof.

## Reproduction Assessment

Yes for the blocking review finding: PR-head source clips active duration by selected hour/day while displayTotals still uses whole matching sessions. The PR's own test expects all 120 tokens over a clipped one-hour duration, which demonstrates the mismatch from source.

## Solution Assessment

No. The Active Time card direction is reasonable, but the patch should align the throughput numerator with the clipped active-time window or suppress throughput when totals are not sliced to the same basis.
