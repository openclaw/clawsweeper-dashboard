Keep open. Current main still does not have a separate Active Time overview card, but this branch should not merge as-is because the clipped active duration can make Throughput overstate hour-filtered usage, and the PR lacks inspectable real behavior proof.

## What This Changes

This PR adds an Active Time card to Control UI Usage Overview, clips duration stats by selected day/hour filters, updates usage overview tests, and regenerates UI locale strings and metadata.

## Best Possible Solution

Keep a narrow Control UI-only Active Time card, either slice the throughput numerator to the same window or suppress throughput when selected hours make totals incomparable, then rebase and verify the rendered Usage page.
