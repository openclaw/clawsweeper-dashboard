## Summary

Keep open: current main still lacks an Active Time overview card, and this PR is useful but remains blocked by a source-reproducible throughput mismatch plus missing real behavior proof.

## What This Changes

The PR adds a localized Active Time card to the Control UI Usage Overview, updates usage insight duration filtering/tests, and refreshes generated locale metadata.

## Best Possible Solution

Land a narrow localized Active Time card after aligning throughput math for time filters and attaching inspectable real UI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: filter a multi-hour session to a selected hour, and the PR clips active duration while keeping full-session token/cost totals for Throughput.

## Solution Assessment

No. The Active Time card direction is reasonable, but throughput should use a matching in-range numerator or be hidden when hour-filtered totals are not precise.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep throughput aligned when clipping active time:** `ui/src/ui/views/usage.ts:325-329`
  - body: When an hour filter is active, `displayTotals` still comes from each matching session's full totals, but this call clips only duration to the selected day/hour window. Throughput then divides full-session tokens/cost by partial active time, so narrowing a multi-hour session to one hour can inflate tok/min and cost/min.
  - confidence: 0.87
