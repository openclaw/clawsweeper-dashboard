## Summary

Keep open: current main still lacks a separate Active Time overview card, but the current PR head has a blocking metric mismatch and no accessible after-fix behavior proof.

## What This Changes

The branch adds a localized Active Time summary card to Usage Overview, clips duration stats for selected day/hour filters, updates the Error Rate sublabel, and adds focused UI tests.

## Best Possible Solution

Keep the localized Active Time card, but align Active Time and Throughput semantics before merge and require accessible redacted proof of the real UI behavior.

## Reproduction Assessment

Yes for the blocking review finding: source inspection shows selected hour filters pass clipped duration into the stats while totals remain whole-session totals. I did not run the UI because this was a read-only review.

## Solution Assessment

No. The added card is a reasonable direction, but the current implementation should either align the throughput numerator to the same filtered slice as Active Time or suppress/relabel throughput when only whole-session totals are available.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:
