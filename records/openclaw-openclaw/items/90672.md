The PR adds a Telegram `/status` warning for authorized native commands whose normal group-message ingress would be blocked, and threads channel status notes through the shared status renderer.

## Best Possible Solution

Rebase onto current main, fix or remove the misleading `sender allowed` detail for empty allowlists, then land the Telegram-owned diagnostic with focused status and Telegram command coverage.

## Reproduction Assessment

Yes. The source path is a Telegram group/topic where `commands.allowFrom.telegram` authorizes `/status` while normal `groupPolicy=allowlist` ingress rejects ordinary messages; I did not run a live bot, but the inspected Telegram screenshot shows the after-fix warning.

## Solution Assessment

Mostly yes. Keeping the diagnosis in the Telegram plugin and reusing the normal ingress evaluator is the right shape, but the sender-allowed display value should use strict allowlist semantics or be omitted for empty allowlists.

- **[P3] Use strict sender matching for empty allowlists:** `extensions/telegram/src/bot-native-commands.ts:326-330`
