## Summary

Keep open: current main still has the reported crash path because status reads persisted session JSON as raw unknown values and passes model/provider fields to helpers that call `.trim()` without runtime string narrowing. This issue is already paired with open PR #76216, which uses closing syntax for this issue, so the maintainer path is to review and land or close that PR rather than closing this issue early.

## What This Changes

Harden `openclaw status` so malformed session model/provider fields cannot crash the status command.

## Best Possible Solution

Normalize unknown persisted session model/provider fields at the status boundary and add regression coverage for corrupted `sessions.json` entries while preserving the read-only status scan.

## Reproduction Assessment

Yes. A non-string `model`, `modelProvider`, `providerOverride`, or `modelOverride` in a parsed session entry can reach direct `.trim()` calls on current main; this review did not mutate local session state to run the CLI repro.

## Solution Assessment

Yes. A small hardening patch using the existing string normalization helper at the boundary is the narrowest maintainable solution, and the linked PR is the current implementation candidate.
