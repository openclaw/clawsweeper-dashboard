## Summary

Keep open: current main still lets raw read-only session-store fields reach direct `.trim()` calls in the status model-resolution path, and the linked fix PR is closed unmerged.

## What This Changes

The issue requests hardening `openclaw status` so non-string persisted session model/provider fields do not crash the CLI.

## Best Possible Solution

Normalize persisted session model/provider fields with the existing string-narrowing helper before any `.trim()` call, keep status read-only, and add regression coverage for corrupted `sessions.json` rows.

## Reproduction Assessment

Yes. Source inspection on current main shows a non-string persisted `model`, `modelProvider`, `providerOverride`, or `modelOverride` can flow from `sessions.json` through `openclaw status` into `.trim()`; I did not mutate local session state to run the CLI crash path because this review is read-only.

## Solution Assessment

Yes. A narrow hardening patch using `normalizeOptionalString` at the persisted model resolver/display boundary, or at the status boundary where raw JSON enters, is the maintainable fix; a startup migration or mutable status repair would be broader than needed.
