## Summary

Keep open: current main still has the reported crash path, and an open closing PR (#76216) is already attached to this issue, so the right cleanup is to review and land or replace that PR rather than close the issue early.

## What This Changes

Harden `openclaw status` so corrupted non-string persisted session model/provider fields cannot crash model selection.

## Best Possible Solution

Land or replace #76216 with a narrow string-narrowing fix at the status/model-selection boundary plus regression coverage for corrupted `sessions.json` entries.

## Reproduction Assessment

Yes. A non-string `model`, `modelProvider`, `providerOverride`, or `modelOverride` parsed from `sessions.json` reaches direct `.trim()` calls on current main; I verified the source path read-only rather than mutating local state to run the CLI crash.

## Solution Assessment

Yes. Treating persisted session model/provider fields as unknown and normalizing them before display/resolution is the narrowest maintainable fix, while preserving the read-only behavior of `openclaw status`.
