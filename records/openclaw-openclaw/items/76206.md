## Summary

Keep open: current `main` still has the unsafe read-only status path, where raw persisted session fields reach direct `.trim()` calls; the linked fix PR is closed unmerged, so it does not resolve this issue.

## What This Changes

The issue asks to stop `openclaw status` from crashing when persisted session model/provider fields contain non-string JSON values.

## Best Possible Solution

Normalize persisted session model/provider fields as unknown before any `.trim()` call, preserve read-only status behavior, and add regression coverage for corrupted `sessions.json` entries.

## Reproduction Assessment

Yes. Source inspection on current main shows a non-string `model`, `modelProvider`, `providerOverride`, or `modelOverride` from `sessions.json` reaches `.trim()` through the read-only status loader; I did not mutate local session state to run the CLI crash path because this review is read-only.

## Solution Assessment

Yes. A narrow hardening patch using the existing `normalizeOptionalString` pattern at the status/model-selection boundary is the maintainable fix; changing session loading or adding a migration would be broader than needed.
