## Summary

Keep open: current main still has the reported crash path, and an open closing PR (#76216) is already attached, so the issue should stay open until that implementation is reviewed and merged.

## What This Changes

Harden `openclaw status` so malformed session entries with non-string model/provider fields do not crash the CLI.

## Best Possible Solution

Review #76216 against this call chain, land it if it fully covers the malformed persisted-session fields, and keep the status path read-only rather than switching to the mutable store loader.

## Reproduction Assessment

Yes. A non-string `model`, `modelProvider`, `providerOverride`, or `modelOverride` in parsed `sessions.json` reaches direct `.trim()` calls through `openclaw status`; this review verified the static call chain without mutating local session state.

## Solution Assessment

Yes. String-narrowing the persisted session fields at the status/model-selection boundary, or making the shared helpers unknown-safe, is the narrow maintainable fix while preserving read-only status behavior.
