## Summary

Keep open: current main still lacks the requested Slack per-channel thread override, and the PR contains a useful narrowed channel schema plus tests, but the patch is not correct as submitted because it only applies the channel value at ingress activation while implicit thread mention collection is still gated by the account-level setting.

## What This Changes

The PR adds Slack per-channel `thread.requireExplicitMention` config fallback across schema/types/resolver, Slack monitor call sites, docs, generated metadata, and tests, plus unrelated provider/Telegram/contract-test changes.

## Best Possible Solution

Rebase and scope the branch to Slack config/docs/tests, compute one effective thread mention policy before both implicit mention collection and ingress activation, preserve current Slack bot controls, and add redacted live Slack proof.

## Reproduction Assessment

Not applicable as a user bug report; this PR adds a new Slack config surface. The patch defect is source-reproducible because account-level `true` still prevents implicit mention collection before the channel-level `false` override reaches ingress activation.

## Solution Assessment

No as submitted. The Slack-owned schema/resolver direction is reasonable, but the effective override must be applied before implicit mention collection and the unrelated/conflicting changes need to be removed or separately reviewed.
