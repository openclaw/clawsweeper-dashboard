## Summary

Keep this PR open. Current main still lacks the Slack per-channel `thread.requireExplicitMention` config surface and still gates implicit thread mentions only from the account-level setting; the PR is a focused implementation candidate that appears correct and now addresses the earlier schema-narrowing review concern.

## What This Changes

The PR adds Slack channel-level `thread.requireExplicitMention` schema/types/docs/generated metadata, resolves exact/wildcard/account fallback through Slack channel config, wires the resolved value into Slack mention gating, adds Slack resolver and prepare tests, and removes one unused Telegram test helper.

## Best Possible Solution

Land the focused Slack-owned implementation after maintainer review, keeping the channel schema narrow, the exact/wildcard/account fallback order intact, and docs plus generated config metadata aligned with the new public config surface.

## Solution Assessment
