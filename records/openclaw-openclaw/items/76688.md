## Summary

Keep open. Current main still has the exact Feishu bot-identity timeout path from the report, and without a recovered bot open_id the default mention-gated group flow can stay nonresponsive even though the WebSocket client starts.

## What This Changes

The issue reports Feishu WebSocket startup timing out while probing bot identity on Fedora 44 with OpenClaw 2026.5.2.

## Best Possible Solution

Make the Feishu startup identity probe failure diagnosable and recoverable: document the existing timeout knobs now, then add a narrow code fix only if live traces show OpenClaw's SDK/token request path differs from the successful manual API path.

## Reproduction Assessment

Yes from source inspection, but not as a live Fedora/Feishu reproduction. Forcing `probeFeishu` to return a timeout follows the report's logged path into unknown bot identity and default mention-gated group skips.

## Solution Assessment

Unclear. The report's network/app checks are useful, but the best repository-side path is first to document and surface the existing Feishu HTTP/startup timeout controls, then only change runtime behavior if live evidence shows a code-level timeout bug.
