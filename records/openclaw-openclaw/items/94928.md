## Best Possible Solution

Land one canonical lifecycle-owner fix that preserves generated reply delivery while keeping no-reply compaction failures fatal, then close or supersede the duplicate candidate PRs after merge.

## Solution Assessment

Yes, this is the best current fix shape if maintainers accept warning-only post-turn maintenance failures. It keeps `runCliTurnCompactionLifecycle` fail-closed and puts the reply-preserving exception in `agentCommandInternal`, gated by the shared outbound projection rather than weakening the helper globally.
