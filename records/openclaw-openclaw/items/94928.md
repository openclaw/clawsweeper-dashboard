The PR wraps post-turn CLI compaction in `agentCommandInternal` so deliverable reply payloads continue to pending final delivery and channel delivery after compaction failure, with regression tests for text, media-only, and no-reply turns.

## Reproduction Assessment

## Solution Assessment

Yes, this is the best current fix shape if maintainers accept the failure-semantics change. It keeps the compaction helper fail-closed and puts the reply-preserving exception in the turn lifecycle owner, gated by the shared outbound payload projection.
