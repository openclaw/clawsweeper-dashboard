## What This Changes

The branch wraps post-turn CLI compaction in `agentCommandInternal`, preserves delivery for sendable replies after compaction failure, and adds regression coverage for reply, media-only, and no-reply paths.

## Best Possible Solution

Land one canonical lifecycle-owner fix that preserves generated reply delivery while keeping empty/no-reply compaction failures fatal, then close or supersede the duplicate candidates after that fix merges.

## Reproduction Assessment

Yes. Source inspection shows current main persists the CLI transcript, awaits post-turn compaction, and only then reaches pending final delivery and `deliverAgentCommandResult`, so a compaction throw can skip delivery after content exists.

## Solution Assessment

Yes, this is the best current fix shape if maintainers accept the behavior change. It keeps `runCliTurnCompactionLifecycle` fail-closed and puts the reply-preserving exception in `agentCommandInternal`, gated by the shared outbound payload projection; broader helper-level PRs are more likely to swallow no-reply failures.
