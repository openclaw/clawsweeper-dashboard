## Summary

## What This Changes

The PR wraps post-turn CLI compaction in `agentCommandInternal` so deliverable replies continue to final delivery after compaction failure, with regression coverage for reply-present, media-only, and no-reply paths.

## Best Possible Solution

Land one canonical lifecycle-owner fix that preserves generated reply delivery while keeping empty/no-reply compaction failures fatal, then close or supersede the duplicate candidate PRs after that fix merges.

## Reproduction Assessment

Yes. Source inspection shows current main persists the CLI transcript, awaits post-turn compaction, and only then reaches pending final delivery and `deliverAgentCommandResult`, so a compaction throw can skip delivery after content exists; this read-only review did not run tests.

## Solution Assessment

Yes, this is the best current fix shape if maintainers accept the behavior change. It keeps `runCliTurnCompactionLifecycle` fail-closed and puts the reply-preserving exception in `agentCommandInternal`, while broader helper-level fixes risk swallowing no-reply failures.
