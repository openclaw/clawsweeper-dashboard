## Summary

Keep open for maintainer review: current main and v2026.6.9 still have the compaction-before-delivery failure path, and this member-authored PR is a strong candidate fix, but it intentionally changes post-turn compaction failure semantics and overlaps with several sibling candidate PRs.

## What This Changes

The PR wraps post-turn CLI compaction in `agentCommandInternal` with a deliverable-reply gate and adds compaction-rotation regression tests for reply-present, media-only, and no-reply failure paths.

## Best Possible Solution

Land one canonical lifecycle-owner fix that preserves generated reply delivery while keeping empty/no-reply compaction failures fatal, then close or supersede the sibling candidates after merge.

## Reproduction Assessment

Yes. Source inspection of current main shows transcript persistence succeeds, compaction is awaited, and final delivery is only reached afterwards; forcing `runCliTurnCompactionLifecycle` to throw reproduces the skipped-delivery path without needing live network failure.

## Solution Assessment

Yes, contingent on maintainer acceptance of the behavior change. Keeping `runCliTurnCompactionLifecycle` fail-closed and putting the reply-preserving exception in `agentCommandInternal` is narrower than helper-level sibling fixes that could swallow no-reply failures.
