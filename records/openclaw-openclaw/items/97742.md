## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve text-first replay before structured fallback:** `src/llm/providers/tool-result-text.ts:105-110`
  - body: Current provider replay already sends explicit text blocks when they exist, and the merged visible-output helper keeps that text-first invariant before falling back to structured serialization. This loop now appends serialized structured blocks after existing text, so mixed text-plus-structured tool results will send extra metadata to the model instead of preserving the prior text-only payload; collect non-empty text first and only serialize structured blocks when no text is available.
  - confidence: 0.91
- **[P1] Omit MIME-tagged data before provider replay:** `src/llm/providers/tool-result-text.ts:36-37`
  - body: The replacer omits `blob`, `bytes`, and `buffer`, but structured tool results also carry binary payloads as `data` under records with `mimeType`, `mime_type`, or `media_type`, such as PDF/image/audio document sources. The existing visible-output sanitizer treats those as binary and has regression coverage; without the same guard this PR can serialize base64 document/media data into provider-bound model context.
  - confidence: 0.88
- **[P2] Only redact actual data URI values:** `src/llm/providers/tool-result-text.ts:43-45`
  - body: The `/data:/` replacement runs anywhere inside a string, so ordinary values like `metadata:ready` are rewritten even though they are not inline data URIs. The sibling visible-output tests intentionally preserve ordinary `data:` substrings while redacting whole data URI values; keep that narrower behavior so tool output text is not corrupted during replay.
  - confidence: 0.82
