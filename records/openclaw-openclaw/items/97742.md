Overall correctness: patch is incorrect

Full review comments:

- **[P1] Skip opaque encrypted tool-output blocks:** `src/llm/providers/tool-result-text.ts:69`
  - body: `readToolResultBlockText` serializes every non-image, non-text record. Codex's current function-call output contract includes `EncryptedContent`, and its own text helper deliberately omits that payload; if such a block reaches this recovery path, this PR would place the opaque encrypted value into model-visible text. Narrow this to known safe text-like shapes and explicitly skip encrypted/private/binary blocks.
  - confidence: 0.88
- **[P1] Preserve existing text result length:** `src/llm/providers/tool-result-text.ts:3`
  - body: All four call sites invoke `extractToolResultText` without options, so the new default caps even existing `type: "text"` tool results at 8,000 characters. Current main forwards the joined text and relies on the session/tool-result truncation layer for prompt budgets, so long `exec` or `read` outputs can be shortened by the provider converter. Keep text blocks unbounded here or use the existing truncation policy only where it already applied.
  - confidence: 0.92
