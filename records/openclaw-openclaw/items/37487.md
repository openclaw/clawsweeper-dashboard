## Summary

Keep open. Current main partially supports named browser profiles, sandbox browsers, global Chrome proxy flags, and process-level HTTP proxying, but it still lacks the requested first-class per-profile proxy config, agent/default browser-profile binding, sub-agent inheritance, and reviewed concurrency/resource policy; the browser-lock and FSB links in the discussion are useful prior-art references, not an OpenClaw implementation.

## What This Changes

The issue requests concurrent per-agent browser isolation with per-profile outbound proxy settings, agent-to-browser-profile defaults, inheritance for sub-agents, diagnostics, docs, and examples.

## Best Possible Solution

Design a staged browser/proxy contract covering secret-safe per-profile proxy config, agent/default profile binding with sub-agent inheritance, explicit tab ownership, SSRF behavior, launch serialization, resource limits, diagnostics, docs, and regression coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature/config request; source inspection gives a high-confidence missing-surface check for per-profile proxy config, agent browser-profile binding, sub-agent inheritance, and concurrency policy on current main.

## Solution Assessment

Is this the best way to solve the issue? Unclear as a direct implementation path; the requested direction is plausible, but the maintainable fix needs product and security design around credentials, SSRF, profile-selection authority, tab ownership, and resource limits before code is added.
