## Summary

Keep open: current main has named browser profiles, per-profile CDP/userDataDir settings, explicit per-call profile selection, tab cleanup, and a global `browser.extraArgs` proxy workaround, but it still lacks the combined first-class contract for agent/subagent browser binding plus per-profile proxy configuration. The prior implementation PR was closed unmerged after security concerns, so this needs maintainer product and security direction rather than cleanup closure.

## What This Changes

The issue requests first-class per-agent browser profile isolation, per-profile proxy configuration, concurrent browser execution, and docs/examples for multi-agent browser routing.

## Best Possible Solution

Keep this open until maintainers approve a staged browser ownership and proxy design that reuses current profiles, preserves SSRF guardrails, protects proxy credentials, and defines explicit agent/subagent binding rather than implicit sharing.

## Reproduction Assessment

Yes for the capability gap: current source and docs show manual profiles and explicit per-call profile selection, but no per-profile proxy field or agent/subagent browser binding config. This is not a strict broken-contract bug reproduction.

## Solution Assessment

No implementation is ready to land yet; the abandoned PR was a plausible direction but not the best complete solution because it left credential exposure and SSRF boundary questions unresolved. The best path is a maintainer-approved staged design over the current browser plugin and proxy policy surfaces.
