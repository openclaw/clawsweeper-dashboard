## Summary

Keep open. Current main and v2026.5.12 still support named browser profiles, profile-specific storage for managed profiles, and a global Chrome proxy flag workaround, but they do not provide first-class per-profile proxy config, per-agent browser-profile binding, subagent inheritance, or the security/resource policy needed for concurrent profile ownership.

## What This Changes

Request first-class concurrent per-agent browser isolation with per-profile HTTP/SOCKS proxy configuration, agent/default profile binding, and docs/examples.

## Best Possible Solution

Design a staged browser/profile contract covering secret-safe per-profile proxy config, agent/default profile binding with subagent inheritance, tab ownership, launch serialization, resource limits, diagnostics, docs, and regression coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature/config request, but source inspection gives a high-confidence missing-surface check for per-profile proxy config, agent browser-profile binding, subagent inheritance, and concurrency policy on current main and v2026.5.12.

## Solution Assessment

Is this the best way to solve the issue? Unclear as a direct patch; the requested direction is plausible, but the maintainable solution needs product and security design for proxy credentials, SSRF semantics, profile-selection authority, tab ownership, and resource limits.
