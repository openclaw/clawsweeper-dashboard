## Summary

Keep open. Current main has named browser profiles, profile-specific storage/CDP ports, explicit `profile` routing, and a process-level HTTP proxy feature, but it still does not provide first-class per-profile browser proxy config, agent/default browser-profile binding, sub-agent inheritance, or a reviewed concurrent resource/isolation policy.

## What This Changes

The issue requests per-agent isolated browser instances with per-profile proxy support, agent-to-profile defaults, sub-agent inheritance, concurrency guarantees, diagnostics, and docs.

## Best Possible Solution

Design this as a staged browser/proxy contract covering secret-safe per-profile proxy config, agent/default profile binding with sub-agent inheritance, explicit tab ownership, SSRF behavior, launch serialization, resource limits, diagnostics, docs, and regression tests coordinated with the broader proxy umbrella.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature/config request; source inspection gives a high-confidence missing-surface check for per-profile proxy config, agent browser-profile binding, sub-agent inheritance, and concurrency policy on current main.

## Solution Assessment

Is this the best way to solve the issue? Unclear as a direct implementation path; the requested direction is plausible, but the maintainable fix needs product and security design around credentials, SSRF, profile-selection authority, tab ownership, and resource limits before code is added.
