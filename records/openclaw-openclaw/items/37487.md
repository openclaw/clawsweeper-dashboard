## Summary

Keep open. Current main still lacks per-profile browser proxy config, agent/default browser-profile binding, sub-agent inheritance, and a reviewed concurrency/resource policy; the linked browser-lock and FSB comments are useful design context but do not make this implemented in OpenClaw.

## What This Changes

The issue requests first-class per-agent browser isolation with per-profile proxy configuration for concurrent browser automation.

## Best Possible Solution

Design this in the bundled browser plugin and agent config contracts with secret-safe per-profile proxy handling, agent-scoped profile defaults, sub-agent inheritance rules, explicit SSRF behavior, concurrency/resource limits, diagnostics, docs, and focused regression tests.
