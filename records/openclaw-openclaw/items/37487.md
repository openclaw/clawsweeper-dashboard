## Summary

Keep open. Current main has named browser profiles, sandbox/browser-node paths, tab cleanup, and process-level proxy support, but it still lacks the requested first-class per-profile browser proxy config, agent/default browser-profile binding, sub-agent inheritance, and reviewed concurrency/resource policy.

## What This Changes

The issue requests first-class per-agent browser isolation, per-profile browser proxy configuration, agent/default browser-profile binding, and reliable concurrent multi-agent browser execution.

## Best Possible Solution

Design this as a staged browser/proxy contract: secret-safe per-profile proxy config, agent/default profile binding with sub-agent inheritance, explicit tab ownership, SSRF behavior, launch serialization, resource limits, diagnostics, docs, and regression tests coordinated with the broader proxy umbrella.
