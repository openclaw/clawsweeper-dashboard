## Summary

Keep open: current main and v2026.6.6 still lack the requested combined contract for agent/subagent-owned browser profiles plus per-profile proxy configuration, even though named managed profiles already isolate user-data directories and CDP ports when explicitly selected.

## What This Changes

The issue requests first-class concurrent per-agent browser profile isolation, per-profile outbound proxy support, inheritance/default-profile binding for subagents, diagnostics, and docs.

## Best Possible Solution

Design a maintainer-approved browser ownership contract that keeps named profiles as the isolation primitive, adds secure per-profile proxy configuration, and explicitly defines agent/subagent defaulting, inheritance, visibility, cleanup, and concurrency limits before implementation.

## Reproduction Assessment

Yes, for the missing capability: source and docs show named profiles and global proxy flags, but no per-profile proxy field or agent/subagent browser-profile binding. This is a source-verifiable feature gap rather than a failing established contract.

## Solution Assessment

Unclear as an implementation shape: the requested direction is valid, but the all-in-one config/API change needs product and security scoping first. A safer solution is staged design around named profile ownership, proxy credential handling, SSRF behavior, and subagent inheritance semantics.
