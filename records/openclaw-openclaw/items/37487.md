## Summary

Keep open. Current main has managed browser profiles, per-profile ports/user data dirs, tab cleanup, and global/process proxy support, but it still lacks the requested first-class per-profile proxy config and agent/default browser-profile binding; the linked browser-lock and FSB comments are useful prior art, not a reason to close.

## What This Changes

The issue requests first-class per-agent isolated browser profiles with per-profile proxy support for concurrent browser automation.

## Best Possible Solution

Design a staged browser/profile contract covering SecretRef-safe per-profile proxy config, agent/default profile binding with subagent inheritance, launch serialization, resource limits, tab ownership, diagnostics, docs, and regression coverage.

Unclear as a direct patch. The requested direction is plausible, but the maintainable solution needs product and security design for proxy credentials, SSRF semantics, profile-selection authority, tab ownership, and resource limits.
