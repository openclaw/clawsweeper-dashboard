## Summary

Keep open. Current main and v2026.6.5 support named browser profiles, separate managed CDP ports/directories, explicit tool profile selection, global Chrome proxy flags, and session-owned tab cleanup, but still lack first-class per-profile proxy configuration and agent/subagent profile binding; the narrower open reports do not cover this combined concurrency and isolation contract.

## What This Changes

Add concurrent agent-isolated browser instances with explicit agent/profile routing and per-profile authenticated HTTP or SOCKS proxy configuration.

## Best Possible Solution

Define a bundled browser-plugin contract for explicit agent/subagent profile assignment or inheritance, SecretRef-backed per-profile proxy settings, session-owned process/tab lifecycle, and bounded concurrency, then implement it with live authenticated-proxy and parallel-isolation proof.

## Reproduction Assessment

Yes at the source-contract level: current main has no per-profile proxy field or agent browser-profile configuration, and managed userDataDir selection is not available per agent. This is feature absence rather than a failing existing behavior test.

## Solution Assessment

Unclear until maintainers choose the authority and inheritance model. Adding fields alone is not the best fix unless the same design also covers secrets, SSRF, lifecycle ownership, resource limits, and failed-run cleanup.

## Risks / Open Questions

- Proxy credentials and endpoints introduce secret-redaction, SecretRef resolution, SSRF, logging, and outbound-network policy decisions.
- Implicit browser inheritance could expose a parent's authenticated cookies or internal-site access to an unintended child agent.
- True concurrency requires lifecycle ownership, launch serialization, resource limits, failure cleanup, and tab/process isolation rather than only additional config keys.
- The scope overlaps two narrower open issues, so maintainers should define which item owns agent/session semantics and which owns general proxy infrastructure before implementation.
