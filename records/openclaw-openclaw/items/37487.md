## Summary

Keep open. Current main still has isolated named browser profiles and a global `browser.extraArgs` proxy workaround, but it does not provide first-class `browser.profiles.<name>.proxy` configuration or agent/subagent browser-profile defaults. The closed related PRs did not merge, and the linked `m13v/browser-lock` project is useful prior art rather than shipped OpenClaw behavior.

## What This Changes

Request first-class support for concurrently running isolated per-agent browser profiles with profile-specific proxy configuration and agent/subagent browser-profile defaults.

## Best Possible Solution

Define and implement a maintainer-reviewed browser profile proxy contract, including secret-aware config shape, redacted status/errors, agent and subagent profile inheritance semantics, docs/examples, and focused tests that preserve current remote CDP and existing-session behavior.
