## Summary

Keep open. Current main has named isolated browser profiles and explicit per-call profile selection, but it still lacks first-class per-profile proxy config and agent/default browser-profile binding; the external browser-lock project and the closed implementation PR are useful context, not shipped OpenClaw support.

## What This Changes

The issue requests first-class per-agent isolated browser/profile binding, per-profile proxy configuration, and concurrent browser isolation guarantees.

## Best Possible Solution

Design this in the bundled browser plugin and agent config contracts with secret-safe proxy config, clear SSRF semantics, agent-scoped profile defaults, diagnostics, docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main has named profile routing and global `browser.extraArgs` proxy flags, but not per-profile proxy config or agent/default browser-profile binding.

## Solution Assessment

Unclear. The requested direction is plausible, but the best solution needs maintainer product/security design because it adds browser egress, credential, config, and concurrency semantics.
