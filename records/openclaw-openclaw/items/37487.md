## Summary

Keep this open: current main has named isolated browser profiles and a global Chrome proxy-flag workaround, but it still lacks first-class per-profile proxy config and agent-bound default browser profile routing. The linked implementation PR #42806 was closed unmerged, and #44635 only covers a narrower isolation subset.

## What This Changes

The issue requests concurrent per-agent browser isolation with agent/subagent default profile binding and per-profile outbound proxy configuration.

## Best Possible Solution

Keep this open until maintainers decide the browser egress and agent-routing contract, then land a scoped browser-plugin/config change with SecretInput-safe per-profile proxy launch wiring, agent/subagent profile inheritance, docs, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction, but the missing feature is source-reproducible: current main has strict schemas without browser.profiles.*.proxy or agents.*.browser.profile. The docs only describe global browser.extraArgs for managed-browser proxy flags.

## Solution Assessment

Unclear pending maintainer and security judgment. The requested browser-plugin/config boundary is plausible if accepted, but it should be designed around proxy secret handling, SSRF policy, and agent/subagent inheritance rather than copied blindly from the closed unmerged PR.
