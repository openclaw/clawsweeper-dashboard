## Summary

Keep open. Current main partially covers browser profile isolation with named managed profiles and global Chrome proxy flags, but it still lacks first-class per-profile proxy config and agent/default browser-profile binding; the related prior-art comments and closed implementation PR show this is useful but still needs product and security design.

## What This Changes

The issue requests per-agent isolated browser instances with per-profile proxy support for concurrent browser automation.

## Best Possible Solution

Design a staged browser/profile contract covering SecretRef-safe per-profile proxy config, agent/default profile binding with subagent inheritance, launch serialization, resource limits, tab ownership, diagnostics, docs, and regression coverage, aligned with https://github.com/openclaw/openclaw/issues/60035 and https://github.com/openclaw/openclaw/issues/53763 where their broader proxy/browser scope overlaps.

## Reproduction Assessment

Yes for the source-level missing feature surface: current strict config schemas lack browser.profiles.<name>.proxy and agents.*.browser.profile, and browser tool profile selection is not agent-scoped. This is not a runtime bug reproduction; it is a feature gap visible from current main.

## Solution Assessment

Unclear as a direct implementation because the requested direction is plausible but security-sensitive. The maintainable solution should be a designed contract for proxy credentials, SSRF semantics, profile-selection authority, tab ownership, and concurrency limits rather than a narrow one-off config field.
