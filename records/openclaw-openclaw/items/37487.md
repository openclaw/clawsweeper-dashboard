## Summary

Keep open. Current main has named browser profiles, separate managed profile storage/CDP ports, tab cleanup for tracked sessions, and a documented global Chrome proxy escape hatch through `browser.extraArgs`, but it still lacks first-class per-profile proxy config, agent/default browser-profile binding, subagent inheritance, and a reviewed concurrency/resource policy; the linked prior implementation was closed unmerged and the external `browser-lock`/FSB comments are useful prior art rather than an OpenClaw-core implementation.

## What This Changes

The issue requests first-class per-agent isolated browser instances, per-profile proxy configuration, and reliable concurrent browser execution with docs and diagnostics.

## Best Possible Solution

Design a staged browser/proxy contract covering secret-safe per-profile proxy config, agent/default profile binding with subagent inheritance, profile-selection authority, tab ownership/cleanup, launch serialization, resource limits, diagnostics, docs, and regression coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature/config request; source inspection gives a high-confidence missing-surface check for per-profile proxy config, agent browser-profile binding, subagent inheritance, and concurrency policy on current main.

## Solution Assessment

Is this the best way to solve the issue? Unclear as a direct implementation path; the requested direction is plausible, but the maintainable fix needs product and security design around credentials, SSRF, profile-selection authority, tab ownership, and resource limits before code is added.
