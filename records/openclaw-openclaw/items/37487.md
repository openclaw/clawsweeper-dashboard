## Summary

Keep open. Current main supports named browser profiles and explicit per-call profile selection, but it still lacks first-class per-profile proxy config, agent/default browser-profile binding, and concurrency policy; the related implementation PR was closed unmerged, and the linked browser-lock project is useful external context rather than an OpenClaw implementation.

## What This Changes

Request first-class per-agent browser isolation, per-profile proxy configuration, and concurrent browser execution guarantees for agents and sub-agents.

## Best Possible Solution

Design this in the bundled browser plugin and agent config contracts with secret-safe proxy handling, clear SSRF/proxy semantics, agent-scoped profile defaults, diagnostics, docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main has explicit profile selection plus global browser.extraArgs proxy flags, not per-agent defaults or per-profile proxy config.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer product and security design because it changes browser egress, proxy credentials, SSRF semantics, config schema, and concurrency policy.
