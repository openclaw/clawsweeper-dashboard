## Summary

Keep open. Current main has named browser profiles, isolated managed profile directories, existing-session userDataDir support, global proxy routing/extraArgs workarounds, and tracked-tab cleanup, but it still lacks the requested first-class per-profile proxy config, per-agent/default browser-profile binding, subagent inheritance rules, and reviewed concurrency/resource policy.

## What This Changes

Request first-class per-agent isolated browser instances with per-profile proxy support for concurrent agent execution.

## Best Possible Solution

Design a staged browser/profile contract covering secret-safe per-profile proxy config, agent/default profile binding with subagent inheritance, tab ownership, launch serialization, resource limits, diagnostics, docs, and regression coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a runtime bug, but yes for a source-level missing-surface check: current main does not accept browser.profiles.<name>.proxy, agents.defaults.browser.profile, or agents.list[].browser.profile.

## Solution Assessment

Is this the best way to solve the issue? Unclear as a direct patch; the requested direction is plausible, but the maintainable solution needs product and security design for proxy credentials, SSRF semantics, profile-selection authority, tab ownership, and resource limits.
