## Security Review

Status: needs_attention

Summary: The requested feature is security-sensitive because it changes browser outbound egress, proxy credential handling, SSRF assumptions, and cross-agent browser-state boundaries.

Concerns:

- **[medium] Design proxy credential handling before adding profile proxy:**
  - body: A future implementation should avoid leaking proxy credentials through config snapshots, logs, status payloads, or Chromium process arguments, and should define whether authenticated proxies are supported directly or via a safer sidecar/secret mechanism.
  - confidence: 0.9
- **[medium] Preserve SSRF policy when browser egress is proxy-routed:** `extensions/browser/src/browser/navigation-guard.ts:117`
  - body: Current navigation guard blocks known proxy-routed browser navigation in strict SSRF mode; per-profile proxy support must thread that fact reliably so local DNS/hostname checks are not treated as final-destination guarantees.
  - confidence: 0.88
- **[medium] Define agent isolation as a trust boundary:**
  - body: The feature is partly motivated by preventing tab, cookie, localStorage, and session cross-talk between concurrent agents, so the design should treat profile selection and shared-profile opt-in as privacy/security policy, not only convenience routing.
  - confidence: 0.84
