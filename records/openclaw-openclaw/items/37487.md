## Summary

Keep open. Current main and the latest inspected release still provide named browser profiles plus global `browser.extraArgs` proxy flags, but they do not implement first-class per-profile proxy config or agent/default browser-profile binding; the related implementation PRs were closed unmerged, and the separate open isolation bug only covers part of this broader request.

## What This Changes

The issue requests first-class per-profile browser proxy configuration and agent-bound default browser profiles for concurrent isolated browser automation.

## Best Possible Solution

Design this in the bundled browser plugin and agent config contracts with secret-safe per-profile proxy config, agent-scoped profile defaults, explicit SSRF semantics, concurrency/resource policy, docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms the requested per-profile proxy and agent/default browser-profile binding config surfaces are absent while named profiles and global `extraArgs` workarounds are present.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer product/security design around credential handling, SSRF behavior, profile-selection authority, concurrency limits, and coordination with the narrower open browser-isolation bug.
