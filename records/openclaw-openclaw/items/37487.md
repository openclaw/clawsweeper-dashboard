## Summary

Keep open. Current main and v2026.5.12 still have named browser profiles, separate managed user-data/CDP ports, session-tab cleanup, and a global Chrome proxy flag workaround, but not first-class per-profile proxy config or agent/default profile binding; the linked implementation attempt closed unmerged after concrete proxy credential and SSRF review concerns.

## What This Changes

The issue requests first-class per-agent browser instance/profile isolation with per-profile proxy configuration for concurrent browser automation.

## Best Possible Solution

Design a staged browser/profile contract covering secret-safe per-profile proxy config, agent/default profile binding with subagent inheritance, tab ownership, launch serialization, resource limits, diagnostics, docs, and regression coverage.

## Reproduction Assessment

Not applicable as a feature/config request, but there is a high-confidence source-level missing-surface check: current main lacks browser.profiles.<name>.proxy, agents.defaults.browser.profile, and agents.list[].browser.profile.

## Solution Assessment

Unclear as a direct patch. The requested direction is plausible, but the maintainable solution needs product and security design for proxy credentials, SSRF semantics, profile-selection authority, tab ownership, and resource limits.
