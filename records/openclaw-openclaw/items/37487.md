## Summary

Keep open. Current main still provides named browser profiles and a global `browser.extraArgs` proxy workaround, but it does not implement per-profile proxy config, agent/default browser-profile binding, sub-agent inheritance, or a concurrency/resource policy; the related implementation at https://github.com/openclaw/openclaw/pull/42806 was closed unmerged after concrete security concerns.

## What This Changes

The issue requests first-class per-profile browser proxy configuration, per-agent browser profile defaults, and reliable concurrent isolated browser usage.

## Best Possible Solution

Design this in the bundled browser plugin and agent config contracts with secret-safe per-profile proxy config, agent-scoped profile defaults, sub-agent inheritance rules, explicit SSRF behavior, concurrency/resource limits, diagnostics, docs, and focused regression tests.

Status: needs_attention

Summary: This feature request changes browser egress, proxy credentials, and cross-agent state isolation, so security design is required before implementation.
