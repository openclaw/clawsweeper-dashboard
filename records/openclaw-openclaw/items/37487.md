## Summary

Keep open. Current main supports multiple browser profiles and explicit per-call profile selection, but it does not implement per-profile proxy configuration or agent/default browser-profile binding; the related implementation PR was closed unmerged, while the linked browser-lock project is useful external context rather than an OpenClaw implementation.

## What This Changes

Request first-class per-agent browser/profile binding with per-profile proxy support for concurrent isolated browser automation.

## Best Possible Solution

Design the feature in the bundled browser plugin and agent config contracts, with secret-safe proxy config, clear SSRF semantics, agent-scoped profile defaults, diagnostics, docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main has explicit profile selection plus global `browser.extraArgs` proxy flags, not per-agent defaults or per-profile proxy config.

## Solution Assessment

Unclear. The requested direction is plausible, but the best solution needs maintainer product/security design because it adds browser egress, credential, config, and concurrency semantics.
