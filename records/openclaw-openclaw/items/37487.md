## Summary

Keep open: current main supports named browser profiles with isolated managed profile directories and CDP ports, but it still lacks the requested per-profile proxy contract and per-agent default browser-profile binding.

## What This Changes

The issue requests first-class per-agent browser profile isolation, per-profile outbound proxy configuration, and reliable concurrent browser use for agent and sub-agent workflows.

## Best Possible Solution

Define a scoped browser-plugin and config design that adds managed-profile proxy settings, resolves a default browser profile from agent context when no tool profile is supplied, and documents concurrency/resource behavior with regression coverage.
