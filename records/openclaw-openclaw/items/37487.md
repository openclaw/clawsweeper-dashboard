## Summary

Keep open. Current main already has named isolated browser profiles and explicit per-call profile selection, but it still lacks first-class per-profile proxy config and agent/default browser-profile binding; the related implementation PR was closed unmerged, and the linked browser-lock project is useful external context rather than an OpenClaw implementation.

## What This Changes

The issue requests first-class per-agent isolated browser profiles, per-profile proxy configuration, and concurrent browser execution support.

## Best Possible Solution

Design this in the bundled browser plugin and agent config contracts with secret-safe proxy config, clear SSRF semantics, agent-scoped profile defaults, diagnostics, docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main has explicit browser profile selection plus global `browser.extraArgs` proxy flags, not per-agent defaults or per-profile proxy config.

## Solution Assessment

Unclear. The requested direction is plausible, but the best solution needs maintainer product/security design because it adds browser egress, credential, config, and concurrency semantics.
