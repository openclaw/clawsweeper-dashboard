## Summary

Current main already provides isolated named browser profiles and a global proxy-flag workaround, but it still does not provide the requested first-class agent/subagent profile ownership or per-profile proxy contract. This issue remains the canonical combined tracker: the related isolation reports were closed into it, while the live proxy and subagent-session issues cover narrower adjacent requirements rather than the whole request.

## What This Changes

Add concurrent per-agent browser isolation, per-profile proxy configuration, and predictable agent-to-profile binding.

## Best Possible Solution

Keep named browser profiles as the single isolation primitive, then design secure per-profile proxy inputs and an explicit agent/subagent browser-session ownership contract with inheritance, sharing, cleanup, observability, and concurrency limits before implementation.

## Reproduction Assessment

Yes for the reported capability gap: current source and docs show isolated named profiles and global proxy flags, but no first-class per-profile proxy field or agent/subagent browser-profile binding. This is a source-verifiable feature gap rather than a failing existing contract.

## Solution Assessment

No, not as the original all-in-one implementation shape. The maintainable path is to reuse named profiles and separately settle secure proxy configuration and explicit browser session ownership semantics, while coordinating the narrower open proxy and subagent-session reports.
