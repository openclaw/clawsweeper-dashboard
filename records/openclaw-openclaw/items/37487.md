## Summary

Keep open. Current main and the latest release still have named browser profiles, isolated managed profile directories, global browser extraArgs proxy workaround, and tracked-tab cleanup, but they do not provide first-class per-profile proxy config, agent/default browser-profile binding, subagent inheritance, SOCKS support, or the security/resource policy needed for concurrent profile ownership.

## What This Changes

Request first-class per-agent isolated browser profile binding with per-profile HTTP/SOCKS proxy support for concurrent agents and subagents.

## Best Possible Solution

Design a staged browser/profile contract covering secret-safe per-profile proxy config, agent/default profile binding with subagent inheritance, tab ownership, launch serialization, resource limits, diagnostics, docs, and regression coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature/config request; source inspection gives a high-confidence missing-surface check for per-profile proxy config, agent browser-profile binding, subagent inheritance, and concurrency policy on current main.

## Solution Assessment

Is this the best way to solve the issue? Unclear as a direct patch; the requested direction is plausible, but the maintainable solution needs product and security design around proxy credentials, SSRF semantics, profile-selection authority, tab ownership, and resource limits before implementation.
