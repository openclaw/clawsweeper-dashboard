## Summary

Keep open. Current main and v2026.6.5 still have named browser profiles, per-profile storage/CDP for managed profiles, and global Chrome proxy flags, but not first-class per-profile proxy config, agent-bound default profile routing, or a concurrency/ownership contract; the linked prior art and closed unmerged PR are useful design input rather than an implemented fix.

## What This Changes

Request first-class per-agent/subagent isolated browser profiles with per-profile outbound proxy configuration and reliable concurrent ownership semantics.

## Best Possible Solution

Define a maintainer-approved browser-session ownership and proxy policy, then implement per-profile proxy config and agent/profile binding in the browser plugin with SecretRef, redaction, SSRF, concurrency, diagnostics, docs, and upgrade tests.

## Reproduction Assessment

Yes for a source-level missing-surface check: current main omits browser.profiles.<name>.proxy, agent-level browser profile binding, and subagent browser ownership inputs. I did not run a live concurrent-browser scenario because the item is a feature/config design request rather than a strict broken-contract bug.

## Solution Assessment

Unclear as a single implementation: the direct PR shape already surfaced credential and SSRF concerns. The safer path is staged design across browser proxy config, agent ownership, SecretRef/redaction, SSRF policy, concurrency limits, docs, and tests.

## Risks / Open Questions

- Per-profile browser proxy support changes browser egress and can weaken SSRF assumptions unless proxy-routed profiles are explicitly modeled and tested.
- Proxy credentials need a SecretRef and redaction design that avoids leaking through config snapshots, logs, status payloads, Chrome argv, process listings, and diagnostics.
- Agent-bound browser defaults and subagent inheritance can expose cookies, localStorage, SSO sessions, and tabs across agents if ownership semantics are implicit.
- Concurrent browser lanes need resource limits, launch serialization, tab ownership, and cleanup semantics; prior comments call out multi-Chromium memory and tab cleanup failures.
