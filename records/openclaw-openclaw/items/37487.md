## Summary

Keep open. Current main covers named managed browser profiles, separate CDP ports, separate managed user-data dirs, explicit tool-level profile selection, node browser proxying, and process-level proxy support, but it still lacks first-class per-profile browser proxy config and agent/default browser-profile binding; the related prior-art comments make the workflow concrete, while the remaining work needs product and security design.

## What This Changes

The issue requests per-agent isolated browser instances with per-profile proxy support, agent/subagent browser-profile binding, concurrency guarantees, diagnostics, and docs.

## Best Possible Solution

Design a staged browser ownership contract covering SecretRef-safe per-profile proxy config, agent/default profile binding with subagent inheritance, tab ownership, launch serialization, resource limits, SSRF semantics, diagnostics, docs, and regression coverage while linking the narrower proxy and subagent-session follow-ups.

## Reproduction Assessment

Yes for the source-level missing feature surface: current strict schemas lack browser.profiles.<name>.proxy and agents.defaults/list browser-profile binding, and the browser tool only reads an explicit profile argument. This is not a runtime bug reproduction; it is a feature gap visible from current main.

## Solution Assessment

Unclear as a direct implementation because the requested direction is plausible but security-sensitive. The maintainable solution should define proxy credentials, SSRF semantics, profile-selection authority, tab ownership, launch concurrency, and diagnostics rather than adding a narrow config field alone.

- A one-off browser.profiles.*.proxy field could leak proxy credentials through config snapshots, status payloads, logs, process argv, crash diagnostics, or docs if the SecretRef and redaction contract is not designed first.

- Concurrency support needs resource limits, launch serialization, tab ownership, and cleanup semantics under load; external comments show these are real operational concerns, not just config shape.
