## Summary

Keep this open. Current main still uses static OpenRouter app-attribution headers and the relevant wrapper/provider-hook/config surfaces do not expose an agent-aware OpenRouter title path; related #50206 is still open and adds a generic `X-OpenClaw-Agent` header rather than OpenRouter app-title attribution.

## What This Changes

Add per-agent OpenRouter request attribution so multiple OpenClaw agents can be distinguished in OpenRouter activity logs and cost tracking.

## Best Possible Solution

Implement OpenRouter-specific app-title attribution in the LLM request-header path while preserving the existing `OpenClaw` default unless maintainers intentionally opt users into per-agent disclosure. The durable end state should thread a sanitized resolved agent identity into the OpenRouter wrapper/provider context, document the behavior in the OpenRouter provider docs, and coordinate with #50206 so generic proxy tracking and OpenRouter dashboard attribution are clearly separate or intentionally unified.

## Reproduction Assessment

Yes for the current behavior. The static title is reproducible by inspecting `createOpenRouterWrapper` and the provider-attribution tests, which still expect `X-OpenRouter-Title: OpenClaw` for every OpenRouter request.

## Solution Assessment

Unclear because there is no accepted patch for this issue yet. The implementation should be narrow, but maintainers should first decide whether per-agent OpenRouter titles are automatic, opt-in, or configurable because the choice affects privacy and OpenRouter analytics semantics.
