## Summary

This should stay open: current `main` still applies OpenRouter app attribution as a fixed OpenClaw identity, and the related open provider-header work does not clearly cover overriding protected OpenRouter attribution headers or the product policy for exposing agent names to OpenRouter analytics.

## What This Changes

Request per-agent OpenRouter attribution so multi-agent users can distinguish agent usage and costs in OpenRouter logs.

## Best Possible Solution

Make a maintainer-level decision on whether OpenRouter app attribution may include agent identity; if yes, implement it through the provider request-header/attribution contract with explicit disclosure and precedence tests.

## Reproduction Assessment

Yes for the missing capability from source inspection: current `main` still derives OpenRouter title attribution from the fixed OpenClaw product identity and protects those headers from override. This is a feature/config request, not a broken documented behavior report.

## Solution Assessment

Unclear. A one-off `X-OpenRouter-Title` mutation would be narrow, but the maintainable solution should align with the broader provider-header work and decide explicitly whether protected app-attribution headers can carry per-agent identity.
