## Summary

Keep open: current main still has static OpenRouter app attribution, and adjacent #50206 does not implement the OpenRouter app-title path. The remaining choice is a product/privacy decision before changing provider-visible agent identity.

## What This Changes

The issue requests per-agent OpenRouter `X-Title` attribution so OpenRouter logs can distinguish requests from different OpenClaw agents.

## Best Possible Solution

Add a narrow, documented OpenRouter-specific title-attribution path that preserves the current `OpenClaw` default unless maintainers explicitly choose opt-in or automatic per-agent disclosure.

## Reproduction Assessment

Yes for the current behavior: source inspection and checked-in tests show every verified OpenRouter request still uses `X-OpenRouter-Title: OpenClaw`. I did not run tests because this was a read-only review and `pnpm` is unavailable in this environment.

## Solution Assessment

Unclear because there is no accepted patch yet. The maintainable direction is likely a sanitized opt-in OpenRouter title or explicit coordination with #50206, but maintainers need to choose the privacy and UX contract first.
