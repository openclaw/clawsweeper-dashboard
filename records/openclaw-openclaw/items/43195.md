## Summary

Keep open: current main still has a duplicate-listener path, but this PR would move plugin suppressors off the existing shared registry key, so it needs a current-main-compatible repair before merge.

## What This Changes

The PR moves unhandled-rejection handlers and install state into a Symbol.for-backed global object and skips later installUnhandledRejectionHandler() listener installs.

## Best Possible Solution

Repair the PR to add a shared installed flag while preserving or bridging the existing handler registry keys, then add a focused repeated-install regression test.

## Reproduction Assessment

Yes. Current main has two installer call sites and an unconditional process.on path; a read-only Node probe confirmed duplicate process.on registrations add duplicate listeners.

## Solution Assessment

No as written. The installed flag is the right narrow fix, but the branch should preserve or bridge Symbol.for("openclaw.unhandledRejection.handlers") so staged plugin runtime copies keep registering suppressors into the Set observed by the listener.
