## Summary

The branch makes `installUnhandledRejectionHandler()` idempotent and moves unhandled-rejection handler state into a shared `globalThis` object.

## Best Possible Solution

Keep the shipped handler Set key and current fatal/exception behavior, then add a separate process-global install guard with focused coverage for duplicate installer calls and plugin runtime handler sharing.

Yes at source level: current main can call the installer from both the direct entrypoint and `runCli()`, while the installer unconditionally attaches a process listener. I did not run a listener-count repro because this review was read-only.

## Solution Assessment

No, not as currently branched. The install-once guard is the right narrow direction, but the safe implementation should preserve the shipped handler registry key and current fatal/exception behavior.
