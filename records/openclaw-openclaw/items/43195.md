## What This Changes

The PR moves unhandled-rejection handler state to a globalThis-backed state object and makes installUnhandledRejectionHandler skip repeated process listener installation.

## Best Possible Solution

Rebase the small install-idempotency guard onto current main, preserve the existing global handler and uncaught-exception registry behavior, and add focused proof that repeated installs do not duplicate the process listener.

## Reproduction Assessment

Yes, by source inspection: current main's installUnhandledRejectionHandler() unconditionally calls process.on("unhandledRejection", ...), so repeated calls register repeated listeners. I did not run a runtime repro because this review is read-only.

## Solution Assessment

Yes conceptually: an install-once guard is the narrow maintainable fix for duplicated process listeners. The current PR needs rebasing onto main's existing global handler state rather than replacing newer adjacent behavior.
