## What This Changes

The branch updates `src/infra/unhandled-rejections.ts` to store the unhandled-rejection install flag and handler Set in shared `globalThis` state and to return early on repeated installer calls.

## Best Possible Solution

Rebase a minimal install-once guard onto current main, preserve or safely migrate `openclaw.unhandledRejection.handlers`, add listener-count coverage, and attach redacted runtime output proving repeated installs add only one listener.

## Reproduction Assessment

Yes at source level: current main's legacy direct entrypoint installs the handler before delegating to `runCli()`, and `runCli()` installs it again. I did not run a listener-count repro because this review is read-only.

## Solution Assessment

No, not as currently branched. The install-once guard is the right small direction, but the branch must preserve or migrate the shipped handler Set key instead of replacing it with a competing registry.
