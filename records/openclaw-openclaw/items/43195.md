## Summary

Keep open: current main still lacks an idempotent install guard, so the PR targets a real infra cleanup, but the branch is not mergeable as-is because it replaces the shipped global handler registry key and still lacks real runtime proof.

## What This Changes

This PR makes unhandled-rejection handling share install state and handler registration through a global symbol so repeated calls skip additional `process.on("unhandledRejection")` listeners.

## Best Possible Solution

Rebase a minimal install-once guard onto current main, preserve or safely migrate the shipped handler Set key, add focused listener-count coverage, and provide redacted runtime proof.

## Reproduction Assessment

Yes. At source level, current main reaches `process.on("unhandledRejection", ...)` on every `installUnhandledRejectionHandler()` call, and both `src/index.ts` and `src/cli/run-main.ts` call that installer.

## Solution Assessment

No, not as currently branched. The install-once guard is a reasonable narrow direction, but the patch must preserve the shipped handler registry key and include real runtime proof before merge.
