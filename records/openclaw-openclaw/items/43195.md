## What This Changes

The PR moves unhandled-rejection install state and the handler registry into a Symbol.for-backed global object and skips later listener installs.

## Best Possible Solution

Keep the installed-guard behavior, preserve or bridge the existing handler registry key, and add focused listener-count regression coverage against current main.

## Solution Assessment

No as written. The installed guard is the right narrow direction, but the branch should preserve or bridge Symbol.for("openclaw.unhandledRejection.handlers") so mixed plugin runtime copies still share suppressors.
