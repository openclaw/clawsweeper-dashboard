## Summary

Keep open: current main still installs duplicate `unhandledRejection` listeners on repeated calls, so the idempotence work is still useful, but this PR is stale/conflicting and its state-key refactor would break the shared handler registry now present on main.

## What This Changes

The PR changes `src/infra/unhandled-rejections.ts` to store rejection install state and handlers in a `globalThis` `Symbol.for` object and skip repeated listener installation.

## Best Possible Solution

Land a rebased version that adds idempotent process-listener installation while preserving the existing `Symbol.for("openclaw.unhandledRejection.handlers")` registry and current fatal/transient classifications.

## Reproduction Assessment

Yes. Current main’s source has an unconditional `process.on("unhandledRejection", ...)` install path, and a read-only Node probe confirmed duplicate `unhandledRejection` registrations are additive.

## Solution Assessment

No as written. The installed guard is the right direction, but the branch should be rebased to preserve or bridge the existing shared handler key and add focused regression coverage.
