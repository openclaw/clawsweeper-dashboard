## What This Changes

This PR moves unhandled-rejection state into a `globalThis` slot and adds an installed flag so repeated installs do not register duplicate process listeners.

## Best Possible Solution

Rebase a minimal install-once guard onto the current global handler registries, add focused listener-idempotency coverage, and include redacted runtime proof before merge.

## Solution Assessment

No, not as-is. The install-once direction is narrow and maintainable, but the branch should be rebased onto current main's global registry shape and backed by focused coverage plus real behavior proof.
