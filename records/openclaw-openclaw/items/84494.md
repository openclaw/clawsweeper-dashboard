## Summary

Keep open for maintainer merge review: the patch appears correct and well-proven, but it intentionally changes Codex interrupted-turn delivery semantics and broad CI failures still need rerun or unrelated-failure classification before merge.

## What This Changes

The PR stops the Codex app-server projector from treating `turn/completed` status `interrupted` as OpenClaw `aborted` by itself, and adds projector/runner regressions for interrupted tool-only no-visible-answer turns plus explicit cancellation.

## Best Possible Solution

Merge after maintainers accept that only explicit OpenClaw abort, timeout, or abort-marker signals suppress Codex no-visible-answer recovery, then let https://github.com/openclaw/openclaw/issues/84492 close through this PR.

## Reproduction Assessment

Yes. Current main and the latest release map app-server `interrupted` to OpenClaw `aborted`, and the PR supplies a focused before-fix failing assertion plus after-fix proof; I did not run tests because this review is read-only.

## Solution Assessment

Yes, with the contract caveat. Removing only the blanket status-to-aborted mapping while preserving explicit abort, timeout, abort-marker, and abort-controller paths is the narrowest maintainable fix I found; if upstream `interrupted` is overloaded, the safer alternative is a stronger cancellation signal.
