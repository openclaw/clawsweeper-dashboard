## Summary

The PR looks like a focused fix for a source-reproducible Codex message-delivery bug, with sufficient live proof and no blocking review findings; it should stay open for maintainer acceptance of the interrupted-turn contract and CI status handling.

## What This Changes

The PR stops treating Codex app-server `interrupted` terminal status as OpenClaw `aborted` by itself and adds projector, run-attempt, and incomplete-turn regressions.

## Best Possible Solution

Keep abort classification tied to explicit OpenClaw cancellation, timeout, or abort-marker evidence, then merge this focused fix once required checks are green or classified unrelated.

## Solution Assessment

Yes. Removing the blanket status-to-aborted mapping while preserving explicit cancellation through `markAborted()`, `markTimedOut()`, and the run abort controller is the narrowest maintainable fix I found.
