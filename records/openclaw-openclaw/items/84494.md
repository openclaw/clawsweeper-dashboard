## Summary

Keep this PR open: the patch is small, source-backed, and appears correct, but merging changes Codex interrupted-turn delivery semantics and still needs maintainer acceptance plus required-check handling.

## What This Changes

The PR stops mapping Codex app-server `turn.status: "interrupted"` to OpenClaw `aborted` by itself and adds focused projector, run-attempt, and incomplete-turn regressions.

## Best Possible Solution

Keep abort classification tied to explicit OpenClaw cancellation, timeout, or abort-marker evidence, then merge after maintainers accept that contract and required checks are green or classified unrelated.

## Reproduction Assessment

Yes by source inspection and supplied failing regression: current main maps app-server `interrupted` to `aborted`, and the incomplete-turn guard skips recovery when `aborted` is true. I did not run tests because this was a read-only review.

## Solution Assessment

Yes, if maintainers accept the interrupted-turn contract. Removing the blanket status-to-aborted mapping while preserving explicit cancellation through `markAborted()`, `markTimedOut()`, abort markers, and the run abort controller is the narrowest maintainable fix I found.
