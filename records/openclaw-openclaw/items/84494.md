## Summary

## What This Changes

The PR stops Codex app-server `interrupted` turn completions from setting projector `aborted` unless an explicit abort/timeout path does so, and adds focused projector/runner regressions for interrupted tool-only/no-visible-answer cases.

## Best Possible Solution

Merge after maintainers accept that only explicit OpenClaw abort, timeout, or abort-marker signals should suppress Codex no-visible-answer recovery, then close https://github.com/openclaw/openclaw/issues/84492 when the fix lands.
