## What This Changes

The PR stops Codex app-server `turn.status: "interrupted"` from automatically becoming OpenClaw `aborted` and adds projector, run-attempt, and incomplete-turn regressions for interrupted tool-only turns.

## Best Possible Solution

Merge after maintainers accept that only explicit OpenClaw abort, timeout, or abort-marker signals suppress Codex no-visible-answer recovery, then let the linked bug issue close through this PR.

## Reproduction Assessment

Yes. Current-main source maps app-server `interrupted` to `aborted`, the incomplete-turn guard skips aborts, and the PR supplies a before-fix failing assertion plus focused after-fix local and Crabbox proof.

## Solution Assessment

Yes, with the maintainer contract caveat. Removing only the blanket status-to-aborted mapping while preserving explicit abort, timeout, and abort-marker paths is the narrowest maintainable fix I found; if upstream `interrupted` is overloaded, the safer alternative is a stronger cancellation signal.
