## What This Changes

The PR changes Codex app-server interrupted-turn projection so interrupted terminal status alone no longer marks the OpenClaw attempt aborted, with focused projector and incomplete-turn regression tests.

## Best Possible Solution

Merge after maintainers accept that only explicit OpenClaw abort, timeout, abort-marker, or abort-controller signals suppress Codex no-visible-answer recovery, then let https://github.com/openclaw/openclaw/issues/84492 close through this PR.

## Reproduction Assessment

Yes. Current main clearly maps app-server `interrupted` terminal status to OpenClaw `aborted`, and the incomplete-turn helper skips no-visible-answer recovery when `aborted` is true; I did not run tests because this review is read-only.

## Solution Assessment

Yes, with the contract caveat. Removing only the blanket status-to-aborted mapping while preserving explicit abort, timeout, abort-marker, and abort-controller paths is the narrowest maintainable fix I found; if upstream `interrupted` is overloaded, the safer alternative is a stronger cancellation signal.
