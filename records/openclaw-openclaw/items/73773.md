## Summary

Keep open. Current main still lacks the `/memory` Control UI route and the proposed `memory.*` Gateway RPCs, so this is a useful implementation candidate, but the current patch is not merge-ready: it removes existing Gateway plugin/session protocol surfaces, can crash on pathless equal-score search hits, retains index jobs without a cap, violates the memory-host-sdk package boundary, and drops the chat new-session confirmation.

## What This Changes

The PR adds a Control UI `/memory` page backed by scoped memory Gateway RPCs, protocol/Swift model updates, UI state/controllers/views/i18n strings, and memory-core/package changes for provider registration, agent-scoped memory search, indexing, and source excerpts.

## Best Possible Solution

Land or collapse the prerequisite memory stack first, then make this memory page diff strictly additive: preserve existing Gateway/protocol methods, keep package imports inside supported SDK seams, bound RPC-retained state, keep chat reset safety unchanged, add the required changelog/docs, and validate the UI against the scoped memory RPCs.

## Reproduction Assessment

Yes for the review defects. Source inspection on current main plus the PR patch shows the removed existing Gateway methods, the pathless sort crash path, the unbounded `indexJobs` registry, the package-boundary violation, and the dropped `/new` confirmation path.

## Solution Assessment

No, not as written. The product direction is plausible, but the current patch should not merge until it is rebased after the prerequisite stack and repaired into an additive, bounded, package-safe Gateway/UI change.

- **[P1] Restore the existing Gateway method registrations:** `src/gateway/server-methods.ts:27`
  - body: The patch adds `memoryHandlers` while removing existing plugin/session surfaces such as `pluginHostHookHandlers`, `plugins.uiDescriptors`, and `sessions.pluginPatch` from handler, method-list, scope, and protocol surfaces.   - confidence: 0.92
- **[P1] Guard pathless search hits before sorting:** `src/gateway/server-methods/memory.ts:442-444`
  - body: `decorateSearchHit` can leave `path` undefined, but the equal-score sort fallback calls `left.path.localeCompare(right.path)`. A pathless hit with the same score will throw and fail `memory.search.debug`; normalize missing paths before comparing.
  - confidence: 0.93
- **[P2] Bound the index job registry:** `src/gateway/server-methods/memory.ts:50-52`
  - body: `indexJobs` is process-global and entries are added by `memory.index.run` without TTL, pruning, or a size cap. A long-running Gateway receiving repeated index requests will retain completed or failed jobs indefinitely; evict old jobs or cap the map.
  - confidence: 0.89
- **[P2] Keep the memory host SDK package self-contained:** `packages/memory-host-sdk/src/host/memory-schema.ts:2`
  - body: The package-local memory schema changes from `./error-utils.js` to a deep `../../../../src/infra/errors.js` import. The package-boundary test only permits explicit runtime bridge files to reference core `src`, so this breaks the package boundary and should stay on the local helper or a supported package export.
  - confidence: 0.88
- **[P2] Keep the new-session confirmation:** `ui/src/ui/app-render.ts:2366`
  - body: The patch removes `confirmReset` from the Control UI New Session button, bypassing `confirmChatResetCommand` for `/new`. That makes a one-click toolbar action reset the current chat without the existing confirmation, which is unrelated to the memory page.
  - confidence: 0.86
