## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks backup content checksums, so the feature is not obsolete, but the PR head bypasses newer backup safeguards, has checksum edge-case regressions, and lacks after-fix real behavior proof.

## What This Changes

Adds schema-v2 backup manifests with per-asset SHA-256 checksums, checksum verification in `openclaw backup verify`, focused backup tests, and backup CLI docs updates.

## Best Possible Solution

Land a rebased checksum implementation that computes digests from archived payload bytes while preserving current backup exclusions, retry behavior, temp-root handling, and schema-v2 fail-closed semantics, with redacted real CLI proof.

## Reproduction Assessment

Yes for the PR review findings: the regressions are source-reproducible by comparing PR head's initial archive pass and checksum gate with current main's backup writer and verifier contracts. This is not a current-main bug reproduction because checksum verification is a new feature proposed by the PR.

## Solution Assessment

No. Per-asset checksum verification is a reasonable direction, but this branch is not the best implementation until it is rebuilt on current backup safeguards, handles empty directories and links consistently, and fails closed for schema-v2 digest metadata.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Preserve backup filters during the initial archive pass:** `src/infra/backup-create.ts:413-424`
  - body: The initial archive is extracted and repacked as the final backup, but this direct `tar.c` call does not use current main's filter for volatile state files or installed plugin dependency trees. Those entries can reappear in checksummed backups even though current main intentionally excludes them.
  - confidence: 0.95
- **[P2] Keep the initial archive pass on the retry wrapper:** `src/infra/backup-create.ts:413-424`
  - body: The first pack calls `tar.c` directly instead of current main's `writeTarArchiveWithRetry`, so live-file EOF races that current main retries can fail `backup create` before checksums are computed.
  - confidence: 0.91
- **[P2] Keep the temp manifest outside backup sources:** `src/infra/backup-create.ts:383-384`
  - body: This branch stages the manifest under `os.tmpdir()` instead of current main's `chooseBackupTempRoot` path. When the system temp directory is inside the state tree, the manifest can be walked as payload and included twice, recreating the duplicate root-manifest failure current main fixed.
  - confidence: 0.9
- **[P1] Fail schema-v2 archives when every digest is missing:** `src/commands/backup-verify.ts:468-470`
  - body: For schema-v2 archives with all `sha256` fields removed, `hasChecksums` is false and the verifier skips checksum validation entirely, returning success without checking content integrity.
  - confidence: 0.94
- **[P2] Accept empty directory assets during checksum verify:** `src/commands/backup-verify.ts:340-343`
  - body: Create can produce a digest for an empty directory asset, but verify excludes directory entries before collecting asset entries and then throws `no entries found`. An empty workspace asset can fail verification immediately after creation.
  - confidence: 0.86
- **[P2] Hash link entries consistently:** `src/infra/backup-create.ts:287-290`
  - body: The create-side directory hash only includes extracted regular files, while verify hashes every non-directory tar entry. Assets containing symlinks or other link entries can mismatch on a fresh archive.
  - confidence: 0.8
