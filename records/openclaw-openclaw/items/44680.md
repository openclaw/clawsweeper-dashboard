## Summary

Keep this PR open: checksum verification is a useful feature direction, but the current branch still regresses current backup creation protections, can skip schema-v2 integrity checks when digests are stripped, and lacks required real CLI proof.

## What This Changes

The PR adds schema v2 backup manifests with per-asset SHA-256 checksums, checksum verification in `backup verify`, regression tests, and backup documentation updates.

## Best Possible Solution

Rebase or rebuild the checksum implementation on current main, hash archived bytes while preserving the filtered retry-backed backup path, fail closed for schema-v2 digest metadata, and add focused coverage for empty directories and link-like archive entries.

## Reproduction Assessment

Yes. The blocking regressions are source-reproducible by comparing the PR-head direct `tar.c` and `hasChecksums` paths with current main's filtered retry-backed backup creation contract and tests.

## Solution Assessment

No. Per-asset archive checksums are a reasonable direction, but this branch is not the narrowest maintainable solution until it preserves current backup behavior and fails closed for schema-v2 integrity metadata.

Full review comments:

- **[P1] Preserve current backup filters during the first pack:** `src/infra/backup-create.ts:413-424`
  - body: The first archive is now the source for the final archive, but this direct `tar.c` call bypasses current main's filters for volatile state files and installed plugin dependency trees. Those files can be reintroduced into the backup even though current main intentionally excludes them.
  - confidence: 0.95
- **[P2] Keep source packing on the retry wrapper:** `src/infra/backup-create.ts:413-424`
  - body: The first pack calls `tar.c` directly instead of `writeTarArchiveWithRetry`, so live-file EOF races that current main retries can fail `backup create` before checksums are computed.
  - confidence: 0.9
- **[P1] Fail schema v2 when every digest is missing:** `src/commands/backup-verify.ts:468-470`
  - body: For schema-v2 archives with all `sha256` fields removed, `hasChecksums` is false and the verifier skips checksum validation entirely, returning success without checking content integrity.
  - confidence: 0.94
- **[P2] Accept empty directory assets during checksum verify:** `src/commands/backup-verify.ts:336-343`
  - body: Create can produce a digest for an empty directory asset, but verify excludes directory entries before collecting asset entries and then throws `no entries found`. An empty workspace asset can fail verification immediately after creation.
  - confidence: 0.86
- **[P2] Hash link entries consistently:** `src/infra/backup-create.ts:287-290`
  - body: The create-side directory hash only includes extracted regular files, while verify hashes every non-directory tar entry. Assets containing symlinks or other link entries can mismatch on a fresh archive.
  - confidence: 0.8
