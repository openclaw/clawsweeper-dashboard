## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Preserve the current backup writer safeguards:** `src/infra/backup-create.ts:413-424`
  - body: This direct `tar.c` path is based on the old writer and does not use current main's temp-root selection, retry wrapper, volatile-file filter, plugin `node_modules` filter, or `skippedVolatileCount` reporting. Merging it would reintroduce duplicate-manifest, live-write, and archive-bloat regressions that current main already guards against; port the checksum pass through the current writer instead of replacing it.
  - confidence: 0.95
- **[P1] Reject schema-v2 manifests with no digests:** `src/commands/backup-verify.ts:468-475`
  - body: `hasChecksums` is false when every schema-v2 asset has its `sha256` stripped or blank, so verification returns `ok` without checking content. Since schema v2 is the integrity format, any missing digest on a v2 asset should fail before reporting success.
  - confidence: 0.94
- **[P2] Hash symlink entries consistently:** `src/infra/backup-create.ts:287-293`
  - body: The create-side directory hash walks extracted files with `Dirent.isFile()` only, while verify hashes every non-directory tar entry. A directory asset containing symlinks can therefore produce a fresh archive whose checksum does not match on verify; either include symlinks in both hashes or exclude them in both paths.
  - confidence: 0.86
