## Summary

Keep open: current main still lacks content checksum verification, and this PR is useful but not merge-ready because the latest head bypasses current backup safeguards, has a fail-open schema-v2 checksum path, and lacks external real behavior proof.

## What This Changes

The PR adds schema-v2 backup manifests with per-asset SHA-256 checksums, checksum verification output/result fields, backup tests, and CLI backup documentation updates.

## Best Possible Solution

Land a rebased checksum implementation that hashes archived payload bytes while preserving current backup exclusions, retry behavior, temp-root handling, and fail-closed schema-v2 digest validation.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the PR defects: they are source-reproducible by comparing the PR head's direct initial tar.c pass and hasChecksums logic with current main's backup writer/verifier contracts and tests. This is not a current-main bug reproduction because checksum integrity is a new feature absent from main.

## Solution Assessment

Is this the best way to solve the issue? No. Per-asset checksums are a reasonable direction, but this branch should be rebuilt on the current backup writer and should fail closed for schema-v2 checksum metadata before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Preserve current backup filters in the initial pack:** `src/infra/backup-create.ts:413-424`
  - body: The initial archive is the payload later hashed and repacked, but this direct tar.c call does not use current main's volatile-file or installed-plugin dependency filters. That can put files current backups intentionally exclude into checksum archives.
  - confidence: 0.95
- **[P1] Fail schema-v2 archives when all digests are stripped:** `src/commands/backup-verify.ts:465-467`
  - body: hasChecksums is false when a schema-v2 manifest has no sha256 fields, so verification skips checksum validation and returns ok. A v2 checksum manifest with missing digest metadata should fail closed.
  - confidence: 0.94
- **[P2] Run the initial pack through the retry wrapper:** `src/infra/backup-create.ts:413-424`
  - body: The first pack reads live state and workspace files but bypasses writeTarArchiveWithRetry, so EOF races current main retries can still fail backup create before checksums are computed.
  - confidence: 0.91
- **[P2] Accept empty checksummed directory assets:** `src/commands/backup-verify.ts:337-340`
  - body: Create can hash an empty directory asset, but verify removes directory entries before collecting asset entries and then throws no entries found. An empty workspace asset can fail verification immediately after creation.
  - confidence: 0.86
- **[P2] Hash link entries consistently during create and verify:** `src/commands/backup-verify.ts:304-310`
  - body: Create's directory hash walks only regular files from the extracted archive, while verify hashes every non-directory tar entry. Assets containing symlinks or link entries can mismatch on a fresh archive.
  - confidence: 0.8
