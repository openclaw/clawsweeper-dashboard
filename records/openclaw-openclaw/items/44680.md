## Summary

Keep open. Current main still has structural-only backup verification, so the central checksum-integrity feature is not implemented or superseded, but this PR is stale against current backup safeguards and should not merge as-is.

## What This Changes

This PR adds schema-v2 backup manifests with per-asset SHA-256 checksums, checksum verification in `openclaw backup verify`, tests, and backup CLI docs.

## Best Possible Solution

Rebuild the checksum feature on top of current main's filtered, retry-backed, temp-root-safe backup writer; fail closed for schema-v2 digest metadata; add empty-directory and link-entry coverage; then provide redacted real CLI proof.

## Reproduction Assessment

Yes for the PR review findings: the regressions are source-reproducible by comparing PR head's direct first `tar.c` pass and checksum gate with current main's backup packing contract. This is not a current-main bug reproduction because checksum verification is a new feature proposed by the PR.

## Solution Assessment

No. Per-asset checksum verification is a reasonable direction, but this branch is not the best implementation until it is rebased around current backup safeguards and fails closed for schema-v2 integrity metadata.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Preserve backup filters during the first archive pass:** `src/infra/backup-create.ts:413-424`
  - body: The initial archive is later extracted and repacked as the final backup, but this direct `tar.c` call does not use current main's filter for volatile state files or installed plugin dependency trees. Those entries can reappear in checksummed backups even though current main intentionally excludes them.
  - confidence: 0.95
- **[P2] Keep the first archive pass on the retry wrapper:** `src/infra/backup-create.ts:413-424`
  - body: The first pack calls `tar.c` directly instead of current main's `writeTarArchiveWithRetry`, so live-file EOF races that current main retries can fail `backup create` before checksums are computed.
  - confidence: 0.91
- **[P2] Keep the temp manifest outside backup sources:** `src/infra/backup-create.ts:383-384`
  - body: This branch stages the manifest under `os.tmpdir()` instead of current main's `chooseBackupTempRoot` path. When the system temp directory is inside the state tree, the manifest can be walked as payload and explicitly included, recreating the duplicate root-manifest failure current main fixed.
  - confidence: 0.9
- **[P1] Fail schema v2 when every digest is missing:** `src/commands/backup-verify.ts:468-470`
  - body: For schema-v2 archives with all `sha256` fields removed, `hasChecksums` is false and the verifier skips checksum validation entirely, returning success without checking content integrity.
  - confidence: 0.94
- **[P2] Accept empty directory assets during checksum verify:** `src/commands/backup-verify.ts:340-343`
  - body: Create can produce a digest for an empty directory asset, but verify excludes directory entries before collecting asset entries and then throws `no entries found`. An empty workspace asset can fail verification immediately after creation.
  - confidence: 0.86
- **[P2] Hash link entries consistently:** `src/infra/backup-create.ts:287-290`
  - body: The create-side directory hash only includes extracted regular files, while verify hashes every non-directory tar entry. Assets containing symlinks or other link entries can mismatch on a fresh archive.
  - confidence: 0.8
