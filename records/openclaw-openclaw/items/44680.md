## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve the current backup writer safeguards:** `src/infra/backup-create.ts:413-423`
  - body: This direct `tar.c` path is based on an older writer and drops current main's temp-root selection, volatile-file filter, plugin `node_modules` filter, retry wrapper, and skipped-count reporting. Rebuild the checksum pass around the current writer so existing users do not get larger or different archives and do not lose live-write retry behavior.
  - confidence: 0.94
- **[P1] Reject schema-v2 archives with no digest fields:** `src/commands/backup-verify.ts:468-470`
  - body: `hasChecksums` is false when every schema-v2 asset digest is stripped, so verification skips `verifyAssetChecksums` entirely and returns OK after structural checks. Schema v2 should fail closed when any required asset checksum is missing, including the all-missing case.
  - confidence: 0.93
- **[P1] Handle empty directory assets during checksum verify:** `src/commands/backup-verify.ts:340-344`
  - body: Create computes a valid empty-directory hash, but verify removes directory entries from `normalizedToRaw` and throws when no non-directory entries remain. A fresh backup containing an empty workspace or other empty asset directory can fail `backup create --verify` against its own archive.
  - confidence: 0.9
- **[P1] Hash symlink entries consistently:** `src/infra/backup-create.ts:287-290`
  - body: The create-side directory hash only includes `item.isFile()` entries, while verify later hashes every non-directory archive entry. Directories containing symlinks can therefore fail checksum verification immediately after creation unless symlinks are either excluded from both sides or included with a stable representation.
  - confidence: 0.88
