## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve the current backup writer safeguards:** `src/infra/backup-create.ts:413-423`
  - body: This direct `tar.c` path is based on the older writer and skips current main's temp-root selection, volatile-file/plugin-dependency filters, `skippedVolatileCount` reporting, and EOF retry wrapper. Merging it would reintroduce duplicate-root-manifest failures when `TMPDIR` is inside a source path and include files current backups intentionally omit; rebuild the checksum pass around the current writer instead of replacing it.
  - confidence: 0.94
- **[P1] Fail closed when every schema v2 digest is missing:** `src/commands/backup-verify.ts:468`
  - body: `hasChecksums` becomes false when a v2 manifest has all `sha256` fields stripped or blank, so the verifier never calls the v2 missing-checksum guard and returns ok with `checksumsVerified: false`. A schema-v2 manifest should reject any missing digest before deciding whether checksums were verified.
  - confidence: 0.93
- **[P1] Hash the same archive entry types on create and verify:** `src/infra/backup-create.ts:289-292`
  - body: Create-side directory hashes only include `Dirent.isFile()` entries from the extracted staging tree, but verify hashes every non-directory tar entry. A symlink inside a backed-up directory is omitted from the manifest hash and then included during verification, so a freshly created archive can fail checksum verification.
  - confidence: 0.87
- **[P2] Allow valid empty directory assets to verify:** `src/commands/backup-verify.ts:340-343`
  - body: The create path emits a digest for directory assets even if they contain no file entries, but verify throws when `assetEntries.length === 0`. An empty workspace or empty top-level asset directory can make `backup create --verify` reject its own archive; compute and compare the empty directory digest instead.
  - confidence: 0.82
