## Best Possible Solution

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Preserve backup filters in the initial archive pass:** `src/infra/backup-create.ts:413-424`
  - body: The initial archive is extracted and repacked as the final backup, but this direct `tar.c` call does not use current main's filters for volatile state files or installed plugin dependency trees. Those entries can reappear in checksummed backups even though current main intentionally excludes them.
  - confidence: 0.95
- **[P1] Fail schema-v2 archives when every digest is missing:** `src/commands/backup-verify.ts:468-470`
  - body: `hasChecksums` is false when all schema-v2 `sha256` fields are removed, so the verifier skips checksum validation and returns success without checking content integrity. A v2 archive with stripped digest metadata should fail closed.
  - confidence: 0.94
- **[P2] Run the initial pack through the retry wrapper:** `src/infra/backup-create.ts:413-424`
  - body: The first pack calls `tar.c` directly instead of current main's `writeTarArchiveWithRetry`, so live-file EOF races that current main retries can fail `backup create` before checksums are computed.
  - confidence: 0.91
- **[P2] Stage temporary backup files outside backup sources:** `src/infra/backup-create.ts:383-384`
  - body: This branch creates its temp manifest under `os.tmpdir()` instead of current main's `chooseBackupTempRoot` path. When the system temp directory overlaps a backed-up state tree, the temporary backup directory can be walked as payload during the first pass, reintroducing the self-inclusion class current main avoids.
  - confidence: 0.86
- **[P2] Accept empty checksummed directory assets:** `src/commands/backup-verify.ts:340-343`
  - body: Create can produce a digest for an empty directory asset, but verify removes directory entries before collecting asset entries and then throws `no entries found`. An empty workspace asset can fail verification immediately after creation.
  - confidence: 0.86
- **[P2] Hash link entries consistently between create and verify:** `src/infra/backup-create.ts:287-292`
  - body: The create-side directory hash only includes extracted regular files, while verify hashes every non-directory tar entry. Assets containing symlinks or other link entries can mismatch on a fresh archive.
  - confidence: 0.8
