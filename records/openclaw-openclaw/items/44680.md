## Summary

Keep open for contributor and maintainer follow-up: checksum integrity is a useful feature and is not already on current main, but this branch is not merge-ready because source review still shows blocking backup-integrity regressions and missing real behavior proof.

## What This Changes

Adds schema-v2 backup manifests with SHA-256 per-asset checksums, checksum verification/result fields, backup tests, and CLI docs.

## Best Possible Solution

Land a rebased checksum implementation that hashes archived payload bytes while preserving current backup exclusions, retry behavior, temp-root handling, and fail-closed schema-v2 digest validation.

## Reproduction Assessment

Yes for the PR defects: the regressions are source-reproducible by comparing the PR head with current main's backup writer and verifier contracts. This is not a current-main bug reproduction because checksum integrity is a new feature absent from main.

## Solution Assessment

No. Per-asset checksums are a reasonable direction, but this branch should be rebuilt on the current backup writer and should fail closed for schema-v2 checksum metadata before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Preserve backup filters in the initial archive pass:** `src/infra/backup-create.ts:413-424`
  - body: The first pass is the payload later hashed and repacked, but this direct tar.c call does not use current main's volatile-file or installed-plugin dependency filters. That can put files current backups intentionally exclude into checksum archives.
  - confidence: 0.95
- **[P1] Fail schema-v2 archives when all digests are stripped:** `src/commands/backup-verify.ts:468-470`
  - body: hasChecksums is false when a schema-v2 manifest has no sha256 fields, so verification skips checksum validation and returns ok. A v2 checksum manifest with missing digest metadata should fail closed.
  - confidence: 0.94
- **[P2] Run the initial pack through the retry wrapper:** `src/infra/backup-create.ts:413-424`
  - body: The first pack reads live state and workspace files but bypasses writeTarArchiveWithRetry, so EOF races current main retries can still fail backup create before checksums are computed.
  - confidence: 0.91
- **[P2] Accept empty checksummed directory assets:** `src/commands/backup-verify.ts:340-343`
  - body: Create can hash an empty directory asset, but verify removes directory entries before collecting asset entries and then throws no entries found. An empty workspace asset can fail verification immediately after creation.
  - confidence: 0.86
- **[P2] Hash link entries consistently during create and verify:** `src/commands/backup-verify.ts:306-313`
  - body: Create's directory hash walks only regular files from the extracted archive, while verify hashes every non-directory tar entry. Assets containing symlinks or link entries can mismatch on a fresh archive.
  - confidence: 0.8
