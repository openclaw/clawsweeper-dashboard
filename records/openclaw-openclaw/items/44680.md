## Summary

Keep open: this PR adds useful backup integrity hardening, but the current branch is merge-conflicting, still has source-reproducible backup writer/verifier defects, and lacks required real behavior proof from the contributor.

## What This Changes

The PR adds schema-v2 backup manifests with per-asset SHA-256 checksums, checksum verification result fields/output, backup tests, and backup CLI docs.

## Best Possible Solution

Rebase the checksum implementation onto the current backup writer, hash archived payload bytes while preserving filters/temp-root/retry behavior, and fail closed for schema-v2 digest metadata before adding redacted real-run proof.

## Reproduction Assessment

Yes for the PR blockers via source inspection: the PR head calls tar.c directly without current main's filter/retry/temp-root path, and hasChecksums skips all-stripped schema-v2 manifests. This is not a current-main bug reproduction because checksum integrity is a new feature absent from main.

## Solution Assessment

No. Per-asset checksums are a reasonable direction, but this branch should be rebuilt on the current backup writer and should fail closed for schema-v2 checksum metadata before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Preserve backup filters in the initial pack:** `src/infra/backup-create.ts:413-424`
  - body: The initial archive is the payload later hashed and repacked, but this direct tar.c call does not use current main's volatile-file or installed-plugin dependency filters. That can put files current backups intentionally exclude into checksum archives.
  - confidence: 0.95
- **[P1] Fail schema-v2 archives with no digests:** `src/commands/backup-verify.ts:468-470`
  - body: hasChecksums is false when a schema-v2 manifest has all sha256 fields stripped, so verification skips checksum validation and returns ok. A v2 checksum manifest with missing digest metadata should fail closed.
  - confidence: 0.94
- **[P2] Keep backup temp roots outside source paths:** `src/infra/backup-create.ts:383-386`
  - body: This uses os.tmpdir() directly for the temp manifest path, dropping current main's fallback when TMPDIR is inside a backed-up asset. In that setup the first pack can re-include the temp manifest and produce duplicate root manifests.
  - confidence: 0.9
- **[P2] Run the initial pack through the retry wrapper:** `src/infra/backup-create.ts:413-424`
  - body: The first pack reads live state and workspace files but bypasses writeTarArchiveWithRetry, so EOF races current main retries can still fail backup create before checksums are computed.
  - confidence: 0.91
