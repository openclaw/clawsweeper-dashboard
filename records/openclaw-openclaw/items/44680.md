## Summary

Keep open because checksum integrity is a useful backup feature and is not on current main, but this PR is not merge-ready: the branch replaces current backup-writer safeguards and leaves schema-v2 checksum verification fail-open.

## What This Changes

Adds schema-v2 per-asset SHA-256 checksums to backup manifests, verifies them in `openclaw backup verify`, updates backup docs, and adds checksum tests.

## Best Possible Solution

Rebuild checksum integrity on top of current backup create/verify code, require schema-v2 digests for every asset, preserve existing exclusions/temp-root/retry behavior, and prove it with a real CLI backup/tamper run.

## Reproduction Assessment

Yes. The merge blockers are source-reproducible: the PR head uses a bare backup writer path that omits current safeguards, and its verifier skips checksum enforcement when schema-v2 assets all lack digests.

## Solution Assessment

No. The maintainable path is to layer checksum integrity onto current main's backup writer and verifier contracts, not replace them with an older writer shape.
