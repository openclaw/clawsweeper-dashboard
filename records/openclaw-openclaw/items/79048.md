## Repair Work Prompt

Fix the installer diagnostics gap reported in https://github.com/openclaw/openclaw/issues/79048. The issue shows macOS install.sh running `npm --loglevel error --silent --no-fund --no-audit install -g openclaw@latest`; both failure attempts then print no npm code/debug/first-error/tail lines, so the real install failure is not diagnosable. In `scripts/install.sh`, stop suppressing default npm failure output in a way that leaves the captured log empty, while keeping successful installs quiet by redirecting normal npm output to the log. Prefer removing or narrowly gating `NPM_SILENT_FLAG` rather than changing Node version policy or package dependencies. Preserve `--verbose` behavior, keep `OPENCLAW_NPM_LOGLEVEL` useful, and add focused regression coverage in the installer shell tests that verifies the default npm command no longer hides errors needed by `print_npm_failure_diagnostics`. Add a user-facing changelog entry because installer failure output changes. Do not attempt a dependency/toolchain fix unless a real npm error identifies it.
