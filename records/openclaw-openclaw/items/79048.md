## Summary

Keep this open: the reporter’s follow-up identifies a real npm EACCES failure, while current and shipped installer code still applies the user-local npm prefix repair only on Linux, not macOS.

## What This Changes

The issue reports that the macOS curl installer fails during the default global npm install with an EACCES mkdir error under `/usr/local/lib/node_modules/openclaw`.

## Best Possible Solution

Extend the existing user-local npm prefix repair to macOS for unwritable npm prefixes, update the installer docs/tests, and verify with a macOS installer smoke before closing.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: with `OS=macos`, current `fix_npm_permissions` returns before checking an unwritable npm prefix, while the reporter’s direct npm output shows the expected EACCES failure. I did not run a full macOS installer smoke in this review.

## Solution Assessment

Yes, the narrow maintainable fix is to reuse the existing user-local npm prefix repair for macOS instead of adding sudo guidance or a new config flag. The fix should include a macOS-specific regression test and docs update for EACCES troubleshooting.
