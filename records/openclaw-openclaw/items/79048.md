## Summary

Keep open: the reporter's follow-up identifies an npm EACCES failure, and current main plus the latest released installer still skip npm-prefix repair on macOS while running a global npm install.

## What This Changes

The issue asks for the macOS curl installer to complete successfully instead of failing `npm install -g openclaw` with EACCES under `/usr/local/lib/node_modules`.

## Best Possible Solution

Handle unwritable macOS npm global prefixes before the global install, matching the existing Linux user-prefix repair or giving clear remediation, and keep npm failure output actionable.

## Reproduction Assessment

Yes at source level: with `OS=macos`, current `fix_npm_permissions` returns before checking npm prefix writability, then the installer runs `npm install -g`. The reporter's direct npm output supplies the real EACCES path, but I did not run a live macOS install.

## Solution Assessment

Yes: extending or refactoring the existing npm-prefix preflight for macOS is the narrow maintainable fix. A docs-only workaround or advice to use sudo would leave the supported curl installer failing for this layout.
