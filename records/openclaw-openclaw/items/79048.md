Fix https://github.com/openclaw/openclaw/issues/79048. The observable bug is that `curl -fsSL https://openclaw.ai/install.sh | bash` on macOS can fail at `npm install -g openclaw` with `EACCES: permission denied, mkdir '/usr/local/lib/node_modules/openclaw'`; current `scripts/install.sh` only redirects unwritable npm prefixes on Linux. Keep the fix scoped to installer npm-prefix/EACCES handling: either preflight macOS unwritable global prefixes using the existing user-local prefix pattern and warnings, or add an EACCES-specific recovery/retry path that avoids recommending `sudo npm i -g`. Add focused coverage in `test/scripts/install-sh.test.ts` for the macOS case, preserve Node 22/24 and Homebrew behavior, and do not change package-manager defaults or unrelated install methods. Add a single `CHANGELOG.md` fix entry because this is user-visible.
