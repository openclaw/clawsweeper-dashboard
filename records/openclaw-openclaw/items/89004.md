## Best Possible Solution

Land the focused hook/model-input split after maintainer acceptance, preserving cloned loaded session history for hook observation while keeping Codex native model input and `llm_input` history empty.

## Reproduction Assessment

## Solution Assessment

Yes. Separating hook-observation messages from Codex native turn/model input is the narrow owner-boundary fix; changing docs to bless empty hook messages or sending mirrored history into `turn/start` would be less maintainable.
