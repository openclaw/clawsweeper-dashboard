## Summary

Keep open. Current main still routes bare Slack native IM `D...` outbound targets through the Slack plugin's channel branch, while docs and inbound routing treat Slack DMs as direct sessions keyed by the Slack user ID.

## What This Changes

The issue asks OpenClaw to canonicalize Slack native IM `D...` targets in outbound delivery-mirror routing so they use the same direct-session bucket as inbound Slack DMs.

## Best Possible Solution

Fix Slack-owned outbound route reconstruction so native IM channel IDs resolve to canonical direct peers and cannot create `slack:channel:d...` mirror sessions, while preserving current channel and MPIM behavior.

## Reproduction Assessment

Yes, from source inspection. A `message.send` with `target=D...` reaches Slack outbound session resolution, where the target is parsed as a channel and only `G...` IDs are reclassified before the mirror session key is written; I did not run a live Slack workspace repro in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is in the Slack plugin outbound route resolver, likely by returning richer IM info from the Slack channel lookup or adding a focused IM-channel resolver there, with regression tests around bare and prefixed `D...` targets.
