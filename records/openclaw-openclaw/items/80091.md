## Summary

Keep open: current main still has the reported Slack outbound routing split. The Slack plugin treats bare `D...` IM channel IDs as channel targets in outbound mirror route reconstruction, while docs and inbound routing canonicalize the same Slack DM as a direct user session.

## What This Changes

The issue asks OpenClaw to canonicalize Slack outbound delivery-mirror routes for bare IM channel IDs so they land in the same direct-message session as inbound Slack DMs.

## Best Possible Solution

Fix the Slack plugin’s outbound session-route canonicalization so native IM channel IDs resolve to direct Slack user peers, preserve existing MPIM/channel behavior, and add focused regression coverage plus a safe path for old split session entries.

## Reproduction Assessment

Yes, at source level: current main deterministically routes bare `D...` Slack targets through the channel path in `resolveSlackOutboundSessionRoute`, while inbound Slack DMs route through `direct:<user>`. I did not run a live Slack workspace in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Slack plugin’s outbound route resolver and tests, with any legacy session cleanup handled as a repair/doctor path rather than by changing core session-key semantics.
