## Summary

Keep open. Current main still has the source-reproducible Slack outbound routing bug, and the open fixing PR explicitly references this issue, so the issue should remain open until that PR lands or is closed.

## What This Changes

Slack outbound delivery-mirror routing should canonicalize native IM `D...` Slack targets to direct user sessions instead of channel sessions.

## Best Possible Solution

Review and land the linked Slack plugin fix once maintainer and real-behavior-proof gates are satisfied, then close this issue through the merged PR; handle legacy session repair separately if needed.

## Reproduction Assessment

Yes, from source inspection. A `message.send` targeting a bare Slack `D...` ID reaches the Slack outbound session resolver, which parses it as a channel target and only reclassifies `G...` IDs before the mirror route is written.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Slack plugin outbound route resolver and conversation metadata helper; the open PR at https://github.com/openclaw/openclaw/pull/80111 is the right path to review rather than opening a duplicate repair lane.
