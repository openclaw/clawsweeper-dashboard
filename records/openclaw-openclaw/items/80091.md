## Summary

Keep open. Current main and the latest tagged release still have the Slack outbound route path that can classify bare native IM `D...` targets as channel peers, and an open closing PR already owns the proposed fix path.

## What This Changes

The issue requests Slack outbound delivery-mirror routing to canonicalize native Slack IM `D...` targets to direct user sessions instead of creating `slack:channel:d...` session keys.

## Best Possible Solution

Review and land or revise the linked Slack PR so outbound mirror routing resolves native IM `D...` IDs to canonical direct `U...` peers and prevents new `slack:channel:d...` mirror sessions; handle legacy session repair separately if it grows beyond the hot path.

## Reproduction Assessment

Yes, source-reproducible. A `message.send` target of `D...` reaches Slack outbound route reconstruction, where current main parses it as a channel and only reclassifies `G...` IDs before building the mirror session key.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Slack plugin outbound route resolver and Slack conversation-info helper, while the already-open PR should be reviewed and proven rather than closing the issue early.
