## Summary

Keep open. Current main still has a source-reproducible Slack outbound DM mirror split, and the linked fix PR is open and unmerged, so this issue should remain open until that candidate lands or is replaced.

## What This Changes

Fix Slack outbound delivery-mirror routing so native Slack IM `D...` targets use the canonical direct user session instead of a channel session.

## Best Possible Solution

Land a Slack-owned fix that resolves Slack IM `D...` targets to canonical `U...` direct peers, or skips mirror creation when IM metadata cannot be resolved, with focused regression coverage.

## Reproduction Assessment

Yes, from source inspection. A `message.send` target of `D...` reaches the Slack outbound route resolver, is parsed as a channel, and only `G...` IDs are reclassified before the mirror session key is built.

## Solution Assessment

Yes. The narrow maintainable solution belongs in the Slack plugin outbound route resolver and channel metadata helper; the linked open PR follows that direction but should be reviewed and proven before this issue closes.
