## Summary

Keep open. Current main still has the Slack outbound route split described in the report, and a linked open implementation PR already uses closing syntax for this issue, so the issue should stay open until that PR lands or is replaced.

## What This Changes

Fix Slack outbound delivery-mirror routing so native Slack IM `D...` targets resolve to canonical direct user sessions instead of channel sessions.

## Best Possible Solution

Slack outbound mirror routing should canonicalize native IM channel IDs to direct user peers and avoid writing known-wrong channel sessions when the IM-to-user mapping is unavailable.

## Reproduction Assessment

Yes, from source inspection. On current main, a `message.send` target like `D0...` is parsed as a channel target and only `G...` IDs are reclassified before mirror session key construction; I did not run a live Slack workspace repro in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix belongs in the Slack plugin outbound route resolver and conversation metadata helper, with regression coverage for bare and `channel:`-prefixed `D...` targets. Existing bad-session migration is a separable cleanup if it grows beyond preventing new splits.
