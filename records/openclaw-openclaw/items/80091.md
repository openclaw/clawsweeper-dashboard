## Summary

Keep open. Current main still has the Slack outbound routing bug, and an open closing PR already targets this exact issue, so the issue should remain open until that PR lands or is rejected.

## What This Changes

The issue requests canonical Slack outbound delivery-mirror routing for native IM `D...` targets so Slack DMs are stored as direct user sessions instead of channel sessions.

## Best Possible Solution

Review and land the Slack-owned fix in https://github.com/openclaw/openclaw/pull/80111 after maintainer handling and real-behavior proof; track any legacy session repair separately if needed.

## Reproduction Assessment

Yes, source-reproducible. A `message.send` target of `D...` reaches Slack outbound session resolution, where current main parses it as a channel and only reclassifies `G...` IDs, while inbound routing and docs classify Slack DMs as direct user sessions.

## Solution Assessment

Yes. The narrow maintainable solution is to fix the Slack plugin's outbound route resolver to resolve native IM channel IDs to direct peer-user session routes and avoid writing known-wrong `slack:channel:d...` mirror keys.
