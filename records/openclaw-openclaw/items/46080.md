## Summary

Keep open. Current main has useful adjacent fixes for empty Anthropic turns, including seeded `content_block_start` text preservation, but the specific successful custom `anthropic-messages` post-tool empty final turn is still not fully recovered: nonzero empty `stop` turns do not enter the Anthropic retry path, and terminal payload building does not synthesize a safe success reply from successful tool results.

## What This Changes

The issue asks OpenClaw to preserve or recover a user-visible final assistant reply after a successful custom `anthropic-messages` tool_use/tool_result round trip that currently ends with empty assistant content.

## Best Possible Solution

The runtime should guarantee a visible terminal outcome for successful Anthropic tool rounds: preserve final provider text when it exists, and when a successful tool result is followed by an empty final assistant turn, recover with one replay-safe continuation or a sanitized completion acknowledgement that does not rerun side effects or expose raw tool details.

## Reproduction Assessment

Yes. A high-confidence mocked reproduction can create a successful tool result, then return a custom `anthropic-messages` assistant `stop` with `content: []`, `payloadCount: 0`, and nonzero usage; current source flow skips the retry gate and reaches the incomplete-turn error path.

## Solution Assessment

No. The current seeded-text fix is useful but incomplete for this issue; the narrow maintainable solution is a post-tool empty-turn recovery path that distinguishes replay-safe read-like tool results from replay-unsafe or mutating successful tools.
