## Summary

Keep open. Current main has generic empty-turn safeguards that avoid a totally silent direct terminal path, but the reported successful custom `anthropic-messages` post-tool empty `stop` turn still falls through to a generic failure reply rather than recovering final assistant text or acknowledging the completed tool action.

## What This Changes

The issue asks OpenClaw to avoid `No reply from agent` after a successful Anthropic-compatible tool round produces an empty final assistant turn.

## Best Possible Solution

Add a provider-family-aware post-tool empty-turn recovery path that preserves real final text, retries only when replay-safe, and otherwise emits a sanitized completed-action acknowledgement.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction can mock a custom `api: "anthropic-messages"` run where a successful `toolResult` is followed by an assistant `stop` with `content: []`, nonzero usage, and no payloads; current main reaches the generic incomplete-turn error path instead of recovered success output.

## Solution Assessment

No. The current generic error fallback is safer than a silent terminal path, but it is not the best fix for a successful post-tool round because it discards the distinction between a failed answer and a completed tool action.
