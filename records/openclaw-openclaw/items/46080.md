## Summary

Keep open. Current main has generic empty-turn safeguards that prevent a fully silent terminal path in the embedded runner, but the specific successful custom `anthropic-messages` post-tool empty `stop` turn still does not recover a final assistant reply or safe success acknowledgement.

## What This Changes

The issue asks OpenClaw to avoid surfacing `No reply from agent` after a successful Anthropic-compatible tool round returns an empty final assistant turn.

## Best Possible Solution

Add a post-tool empty-turn recovery path for Anthropic-compatible providers that preserves real final text when present, retries only when replay-safe, and otherwise emits a sanitized success acknowledgement without rerunning mutating tools or exposing raw tool payloads.

## Reproduction Assessment

Yes. A high-confidence mocked reproduction can return a successful tool result followed by an `anthropic-messages` assistant `stop` with `content: []`, nonzero usage, and no payloads; current main then reaches the generic incomplete-turn error path instead of a recovered success response.

## Solution Assessment

No. The current generic error fallback is safer than silence, but it is not the narrowest maintainable solution for a successful post-tool round because it loses the distinction between a failed answer and a completed tool action.
