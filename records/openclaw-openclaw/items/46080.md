## Summary

Keep open. Current main still does not apply the replay-safe non-visible retry guard to generic custom/proxied `anthropic-messages` providers with nonzero output usage, so the core reported path remains a valid narrow agent-runner bug even though nearby empty-turn handling has improved.

## What This Changes

The issue requests fixing custom `anthropic-messages` tool-result and empty-final-turn handling so users receive visible assistant output instead of an empty payload.

## Best Possible Solution

Extend the existing replay-safe non-visible retry guard to generic `modelApi === "anthropic-messages"`, add focused non-Kimi regression coverage, and keep broad synthesized fallback policy in the separate no-reply issue.

Yes. The narrow maintainable fix is a model-API-based guard plus regression tests, not a provider-name allowlist or a global synthesized tool-result reply.
