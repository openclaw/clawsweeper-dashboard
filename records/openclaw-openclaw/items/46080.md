## Summary

Keep open. Current main still excludes `anthropic-messages` from the non-visible-turn retry guard that handles empty successful assistant turns, while the report and follow-up comments describe exactly that post-tool-result empty-turn path.

## What This Changes

The issue asks OpenClaw to avoid returning no user-visible reply when an Anthropic-compatible `anthropic-messages` tool round-trip succeeds but the final assistant turn is empty.

## Best Possible Solution

Extend the existing replay-safe non-visible-turn retry logic to cover `modelApi: "anthropic-messages"`, with focused regression coverage for a custom provider after a successful tool result and no broad global fallback from tool output.

## Reproduction Assessment

Yes, at source level. A custom `anthropic-messages` run whose final assistant stop has no visible content reaches the current empty-response resolver with `modelApi: effectiveModel.api`, but the guard only allows `openai-completions` and Ollama-like providers, matching the reported empty `payloads` path.

## Solution Assessment

Yes. Adding `anthropic-messages` to the existing non-visible-turn retry guard, with a regression test, is narrower than synthesizing tool-result replies or adding a new dispatch policy.
