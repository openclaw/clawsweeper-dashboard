## Summary

Keep open. Current main has a generic empty-turn error fallback, but the specific generic custom/proxied `anthropic-messages` recovery path remains incomplete: non-Kimi Anthropic-compatible providers still miss the non-visible retry guard that neighboring provider paths use.

## What This Changes

The issue requests that successful Anthropic `tool_result` turns stop ending in empty assistant content and no user-visible reply.

## Best Possible Solution

Extend the replay-safe non-visible retry guard to generic `modelApi === "anthropic-messages"` with focused non-Kimi regression coverage, while leaving broader tool-result synthesis policy to a separate product decision if maintainers want it.

## Reproduction Assessment

Yes. From source inspection, a non-Kimi custom/proxied `anthropic-messages` empty assistant turn with nonzero output usage bypasses the non-visible retry guard and reaches fallback/error handling instead of the continuation retry used by neighboring providers.

## Solution Assessment

Yes. The narrow maintainable fix is a model-API-based guard plus regression tests; a broad synthesized tool-result reply would be a larger policy change and should not be bundled into this bug fix.
