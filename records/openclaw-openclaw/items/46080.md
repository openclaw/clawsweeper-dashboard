## Summary

Keep open. Current main still supports custom `api: "anthropic-messages"` providers, but the empty non-visible-turn retry guard still omits `anthropic-messages`, so a successful post-tool empty `stop` turn can fall through to the incomplete-turn/error path instead of producing a recovered final answer or a completed-action acknowledgement.

## What This Changes

The issue asks OpenClaw to recover or produce a visible fallback when a custom `anthropic-messages` provider returns an empty final assistant turn after a successful `tool_result`.

## Best Possible Solution

Extend the existing non-visible-turn recovery for `anthropic-messages` runs, with regression coverage for replay-safe retry and a safe visible acknowledgement when replay is unsafe after successful tools.

## Reproduction Assessment

Yes, source-level. A mocked custom `api: "anthropic-messages"` run with a successful `toolResult`, output usage, `stopReason: "stop"`, empty assistant content, and zero payloads reaches the current guard omission without needing a live provider.

## Solution Assessment

Yes. The narrow maintainable fix is provider-API-aware recovery in the embedded runner, not a global no-reply policy change or unsafe replay of mutating tool actions.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/46080. On current main, custom providers using `api: "anthropic-messages"` can complete a tool_result round trip and then return a final assistant `stop` with empty content; `shouldApplyNonVisibleTurnRetryGuard` does not include `anthropic-messages`, and terminal payload building does not inline successful tool results. Keep the fix in the embedded-runner/incomplete-turn path: add provider-API-aware recovery for `anthropic-messages`, retry only replay-safe empty post-tool turns, and for replay-unsafe successful tool turns emit a sanitized visible completed-action acknowledgement rather than duplicating side effects or exposing raw structured details. Do not implement the broader always-reply/config-policy request from https://github.com/openclaw/openclaw/issues/60607. Add focused regression tests in the incomplete-turn runner suite and update the changelog if the user-visible behavior changes.
