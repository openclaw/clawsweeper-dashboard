## Summary

Keep open. Current main has added a generic visible incomplete-turn error for empty final turns, so the exact silent `No reply from agent` outcome is narrower now, but the reported Anthropic-compatible tool-result path still lacks the same non-visible retry coverage that exists for `openai-completions` and zero-usage Claude turns.

## What This Changes

The issue reports Anthropic-compatible `anthropic-messages` tool-result turns ending with empty final assistant content and a user-visible `No reply from agent` result.

## Best Possible Solution

Extend the existing non-visible-turn recovery contract to `anthropic-messages` for replay-safe empty/thinking-only final turns after successful tool results, while preserving side-effect safety and adding regression coverage.

## Reproduction Assessment

Yes, at source level. A non-zero-output empty final assistant turn from an `anthropic-messages` model API skips the current non-visible retry guard and falls through to a generic incomplete-turn error instead of recovering a visible post-tool answer.

## Solution Assessment

Yes, for the replay-safe path. The narrow fix is to extend the existing retry guard and tests to `anthropic-messages`, while keeping the current side-effect guard or an explicit visible fallback for mutating tools.
