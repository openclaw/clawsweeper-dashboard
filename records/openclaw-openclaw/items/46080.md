## Summary

Keep open: current main still does not route `anthropic-messages` empty or reasoning-only terminal turns through the existing non-visible retry guard, and the normal terminal payload path does not promote successful tool results as user-visible replies.

## What This Changes

The issue requests a visible final reply or safe fallback when an Anthropic-compatible `tool_result` succeeds but the final assistant message has empty content.

## Best Possible Solution

Extend the existing non-visible retry guard to `anthropic-messages` for replay-safe attempts, add focused regression coverage, and preserve the current side-effect guards.

## Reproduction Assessment

No, not for the full live provider symptom. Source inspection does give a clear current-main path: `anthropic-messages` skips the non-visible retry guard while terminal payloads can be empty when assistant content is empty and tool results are not inlined.

## Solution Assessment

Yes for a first fix. The narrowest maintainable path is a modelApi-based `anthropic-messages` retry guard plus tests, not a provider regex or broad tool-result fallback unless the narrow fix proves insufficient.
