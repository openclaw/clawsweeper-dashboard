## Summary

Keep open. The current code still has the Anthropic-specific gap called out in the discussion: non-visible final-turn recovery is model-API aware for `openai-completions` but not `anthropic-messages`, so custom Anthropic-compatible tool-result turns can miss the established visible-answer retry path.

## What This Changes

Custom Anthropic-compatible providers using `anthropic-messages` can complete a successful `tool_result` turn with empty final assistant content and no useful final user reply.

## Best Possible Solution

Add `anthropic-messages` to the replay-safe non-visible final-turn retry path with focused regression coverage, while preserving existing side-effect guards and clear user-visible fallback/error behavior for replay-unsafe tool turns.

## Reproduction Assessment

Yes, at source level. The current guard excludes `anthropic-messages`, and the issue comments describe the matching tool_result success followed by empty final assistant content on current-main-era builds; I did not perform a live custom-provider run.

## Solution Assessment

No. Current main is missing the narrow contract-driven fix; adding `anthropic-messages` to the existing non-visible retry guard is safer than a broad unconditional tool-result fallback.
