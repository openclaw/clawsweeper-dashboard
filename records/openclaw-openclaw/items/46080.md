## Summary

Keep open. Current main still has a narrow source-level gap for non-Kimi `anthropic-messages` providers that return an empty final assistant turn with nonzero usage/output after a replay-safe attempt, so the reported user-visible silence is not fully fixed.

## What This Changes

The issue reports that custom Anthropic-compatible `anthropic-messages` providers can complete tool execution but persist an empty final assistant message, leaving the user with no reply.

## Best Possible Solution

Add a generic replay-safe `anthropic-messages` non-visible retry path with focused regression coverage, while leaving side-effecting tool-result fallback policy to the broader no-reply issue.

## Reproduction Assessment

Yes, from source inspection rather than live execution. A non-Kimi `anthropic-messages` empty assistant turn with nonzero output usage reaches `resolveEmptyResponseRetryInstruction`, but the current guard does not return the retry instruction for that model API unless the provider id is Kimi.

## Solution Assessment

Yes for the narrow replay-safe fix: extend the existing model-API-based guard shape and add non-Kimi regression tests. Do not use provider-name regexes or retry side-effecting tool turns without a stronger replay-safety signal.
