The PR adds embedded-runner attempt heartbeat/timeout callbacks and command-queue abort/release timeout controls so long native tool execution can keep its lane alive while ignored timeout or cancellation paths release the lane after a bounded grace.

## Best Possible Solution

Land this branch only after maintainers intentionally accept the generic lane abort/release semantics and final CI for the current head is green; otherwise narrow the fix back to a runner-owned heartbeat plus bounded cleanup path.
