## Summary

Keep this PR open for maintainer handling: the compaction fixation may be real, but this patch promotes ordinary user text into the system prompt and also weakens unrelated plugin contract parity checks.

## What This Changes

The PR appends a truncated copy of normal user prompts to `extraSystemPrompt` in the embedded runner and relaxes two bundled plugin contract parity assertions.

## Best Possible Solution

Preserve the latest user turn through transcript or context-engine assembly at user authority, add a focused post-compaction regression test, and fix any bundled provider snapshot drift by updating exact contract metadata or the registry contract directly.

## Reproduction Assessment

Unclear: the production fixation is described and related reports add context, but there is no minimal current-main fixture or failing test. The introduced security regression is reproducible by code inspection because `params.prompt` becomes part of `extraSystemPrompt`, which current main submits as system prompt content.

## Solution Assessment

No: labeling user text as untrusted does not lower its system-role placement. A safer fix should keep the latest user input in transcript/user-level context, and the plugin contract drift should be handled directly rather than by loosening parity checks.
