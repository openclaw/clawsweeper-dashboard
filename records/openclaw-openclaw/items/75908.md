## Summary

Keep this PR open, but the proposed patch should not merge as-is because it promotes untrusted user text into a trusted system-prompt path.

## What This Changes

The PR appends a truncated copy of each non-silent user prompt to `extraSystemPrompt` in `runEmbeddedPiAgent` so it survives compaction.

## Best Possible Solution

Preserve the latest user turn through compaction as untrusted user/transcript context, or add a context-engine seam that rehydrates it without raising its authority above normal user messages.

## Reproduction Assessment

Unclear: the reported production compaction fixation is described in the PR body but is not reproduced by a test or minimal fixture. The introduced regression is reproducible by code inspection because `params.prompt` would be rendered into the submitted system prompt.

## Solution Assessment

No: `extraSystemPrompt` is a trusted system-prompt path, so this is not the safest way to solve compaction amnesia. A lower-priority quoted runtime/transcript anchor would preserve context without changing instruction authority.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep user-authored text out of the system prompt:** `src/agents/pi-embedded-runner/run.ts:328-333`
  - body: This copies `params.prompt` into `extraSystemPrompt`, which current prompt assembly renders directly into the system prompt. A normal chat user can therefore place instructions such as ignoring tool or safety rules into a higher-priority prompt section, instead of leaving them as user-level transcript content. The anchor needs to remain explicitly untrusted or be carried through the transcript/context-engine path.
  - confidence: 0.91
