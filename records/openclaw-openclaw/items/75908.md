## Summary

Keep open, but do not merge this patch as-is. The production compaction problem is plausible, but the PR promotes untrusted user text into system-prompt context and also weakens unrelated plugin registry contract tests.

## What This Changes

This PR copies each normal user prompt into `runEmbeddedPiAgent`'s `extraSystemPrompt` as an untrusted last-message anchor and relaxes two plugin contract assertions from exact equality to array containment.

## Best Possible Solution

Preserve the latest user turn as user/body prompt context or a typed noncompactable user-turn field, keep it last, and add focused compaction regression coverage without weakening plugin contract tests.

## Reproduction Assessment

No. The PR body reports production trace evidence and current source shows the relevant compaction/system-prompt paths, but I did not establish a focused failing current-main reproduction where a compacted summary outranks the latest user prompt.

## Solution Assessment

No. Copying `params.prompt` into `extraSystemPrompt` is not the best fix because it changes user text into system-prompt content; the safer direction is to keep the current prompt last in user/body prompt projection or add an explicit noncompactable user-turn anchor.
