## Summary

Keep open; the reported compaction fixation may be real, but this PR’s current implementation creates a prompt-authority security regression and weakens unrelated plugin contract tests.

## What This Changes

The branch appends a truncated copy of normal user prompts to `extraSystemPrompt` in the embedded runner and relaxes bundled provider parity assertions in plugin contract tests.

## Best Possible Solution

Preserve the latest user turn through transcript or context-engine assembly at user authority, add a focused post-compaction regression test, and handle any bundled provider snapshot drift with exact contract metadata or registry updates.

## Reproduction Assessment

Unclear for the original production fixation because the PR body provides production observations but no minimal fixture or failing current-main test. The introduced security regression is reproducible by code inspection: `params.prompt` becomes `extraSystemPrompt`, and current main renders that field into the submitted system prompt.

## Solution Assessment

No: marking copied user text as “untrusted” does not change its system-role placement. The safer direction is to keep the latest user input in transcript/user-level prompt context and fix plugin contract drift directly instead of weakening parity assertions.
