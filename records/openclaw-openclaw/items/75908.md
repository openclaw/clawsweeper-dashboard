Keep this PR open for maintainer/security review, but the current patch is not mergeable: it promotes user-authored text into system-prompt placement and weakens plugin contract parity tests. The PR body also does not include inspectable after-fix runtime proof for the production compaction scenario.

## Best Possible Solution

Preserve the latest user turn through transcript or context-engine assembly at user authority, add a focused post-compaction regression test, and keep provider contract parity exact unless that contract is deliberately redesigned.

Unclear: the PR describes a production compaction fixation, but there is no minimal current-main fixture or live proof that reproduces the original behavior. The patch defects are source-verifiable from the diff and current prompt/test code.

## Solution Assessment
