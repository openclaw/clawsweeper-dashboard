## Summary

Keep this PR open, but it should not merge as written: the force-pushed “untrusted” wording still puts user-authored text in the trusted system-prompt channel, which is a prompt-authority regression.

## What This Changes

The PR appends a truncated copy of each normal user prompt to `extraSystemPrompt` as a last-message anchor intended to survive context compaction.

## Best Possible Solution

Preserve the latest user turn through transcript or context-engine assembly at user-level authority, with a focused regression fixture for post-compaction latest-message handling.

## Reproduction Assessment

Unclear: the production fixation is described and related reports #75841 and #75852 give context, but there is no minimal fixture or failing test. The introduced regression is reproducible by code inspection because `params.prompt` becomes part of the submitted system prompt.

## Solution Assessment

No: labeling the text as untrusted inside `extraSystemPrompt` does not lower its role. A safer fix should keep the latest user input in transcript/user-level context or a dedicated lower-authority context block.
