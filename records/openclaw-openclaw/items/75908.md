## Summary

Keep this PR open for maintainer handling: the reported compaction fixation is plausible and related reports add useful context, but this patch is not mergeable because it promotes ordinary user text into the system prompt and weakens bundled plugin contract tests.

## What This Changes

The PR appends a truncated latest-user-message anchor to `extraSystemPrompt` in the embedded runner and changes two plugin contract tests from exact equality to subset checks.

## Best Possible Solution

Preserve the latest user turn through transcript or context-engine assembly at user authority, add a focused post-compaction regression test, and keep bundled provider contract parity exact unless the registry contract itself is deliberately changed.

## Reproduction Assessment

Unclear for the original production compaction bug because there is no minimal current-main fixture here. The PR's security regression is source-reproducible: `params.prompt` is copied into `extraSystemPrompt`, which current main renders and submits as system-prompt content.

## Solution Assessment

No: marking user text as untrusted inside `extraSystemPrompt` does not change its system-role placement. A safer fix should keep the latest input at user authority and handle provider/contract snapshot drift directly instead of loosening parity checks.
