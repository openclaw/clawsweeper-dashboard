## Summary

Keep this PR open, but the current patch is not mergeable as-is because it promotes user text into system-prompt placement and weakens unrelated plugin contract guards.

## What This Changes

The PR appends a truncated latest-user-message anchor to `extraSystemPrompt` in the embedded runner and changes two plugin contract tests from exact equality to subset assertions.

## Best Possible Solution

Preserve the latest user turn at user authority in transcript or context-engine assembly, add a focused post-compaction regression test, and handle provider snapshot drift by updating the contract data rather than relaxing parity checks.

## Reproduction Assessment

Unclear for the original production compaction bug because this PR does not include a minimal current-main fixture. The PR's introduced security regression is source-reproducible: `params.prompt` is copied into `extraSystemPrompt`, which current main submits as system prompt content.

## Solution Assessment

No. Labeling user text as untrusted inside `extraSystemPrompt` does not preserve its lower authority, and the unrelated contract-test loosening should be handled separately.
