## Summary

Keep open: the reported post-compaction latest-turn loss looks plausible, but this branch promotes arbitrary user text into a trusted system-prompt sink and also weakens unrelated plugin contract assertions.

## What This Changes

The PR appends a truncated copy of each normal user prompt to `extraSystemPrompt` and relaxes two bundled plugin contract test assertions.

## Best Possible Solution

Preserve the latest user turn through transcript or context-engine assembly at user authority, add a focused post-compaction regression test, and fix any bundled provider snapshot drift by updating the exact contract metadata or registry contract.

## Reproduction Assessment

Unclear: the production fixation is described and related reports add context, but there is no minimal fixture or failing current-main test. The introduced security regression is reproducible by code inspection because `params.prompt` becomes part of `extraSystemPrompt`, which current main submits as system prompt content.

## Solution Assessment

No: labeling user text as untrusted inside `extraSystemPrompt` does not lower its system-role placement. A safer fix should keep the latest user input in transcript/user-level context, and the plugin contract drift should be handled directly rather than by loosening parity checks.
