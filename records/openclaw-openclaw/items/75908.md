## Summary

Keep this PR open, but do not merge it as-is: current main does not already implement the requested compaction survivability behavior, and this patch introduces a prompt-authority/security regression by moving user-authored text into the trusted system prompt.

## What This Changes

The PR appends a truncated copy of each non-silent, non-model user prompt to `extraSystemPrompt` in `runEmbeddedPiAgent` so the latest ask survives compaction.

## Best Possible Solution

Preserve the latest user turn through context-engine or transcript assembly as explicitly untrusted context, with a focused regression test for post-compaction prompt selection.

## Reproduction Assessment

Unclear: the reported compaction fixation is production-observed but not reduced to a test or local fixture. The introduced regression is reproducible by source inspection because `params.prompt` flows into `extraSystemPrompt` and then the submitted `systemPrompt`.

## Solution Assessment

No: this solves survivability by changing instruction authority. A safer fix would keep the latest user turn in the user/transcript/context-engine layer or mark it as untrusted runtime context.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep user text out of the system prompt:** `src/agents/pi-embedded-runner/run.ts:322-333`
  - body: This copies `params.prompt` into `extraSystemPrompt`, which current prompt assembly renders as part of the system prompt. A normal channel user can therefore include instructions that are promoted above user-role transcript content, so this anchor needs to stay in an untrusted transcript or context-engine path.
  - confidence: 0.91
