## Summary

Keep open; the production compaction issue may be real, but this patch should not merge because it promotes untrusted user text into system-prompt authority and weakens plugin contract tests.

## What This Changes

The PR appends a truncated copy of normal user prompts to `extraSystemPrompt` in the embedded runner and relaxes bundled plugin registry/loader contract test equality checks.

## Best Possible Solution

Preserve the latest user turn through transcript or context-engine assembly at user authority, add a focused post-compaction regression test, and keep bundled plugin registry parity checks exact or fix the underlying metadata drift directly.

## Reproduction Assessment

Unclear: the production reports and PR body describe the compaction fixation pattern, but there is no minimal current-main fixture for the reported behavior. The introduced security regression is high-confidence by inspection because `params.prompt` is copied into `extraSystemPrompt`, which current main submits as system prompt content.

## Solution Assessment

No: placing user-authored text in `extraSystemPrompt` is not the narrowest maintainable fix because the authority level remains system-role even when labeled untrusted. A safer fix should preserve the latest user input in transcript/user-level context and handle plugin contract drift separately.
