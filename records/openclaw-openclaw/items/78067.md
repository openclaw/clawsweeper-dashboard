## Summary

Keep this PR open: it addresses a real source-level bug, but the patch is too broad for normal config refreshes, lacks a changelog entry, and does not include after-fix real behavior proof.

## What This Changes

The PR preserves web search/fetch source metadata during secrets runtime refreshes and adds a fast-path regression test for a stripped refresh source.

## Best Possible Solution

Preserve web metadata only for proven partial-source refreshes or fix the writer/projection path that loses the source surface, while covering both partial-refresh and explicit-removal cases.

## Reproduction Assessment

Yes, source-level: activate a snapshot with `tools.web.search.provider`, then refresh with a source config that lacks `tools.web`; current main's fast path can return empty web metadata and overwrite the active metadata. I did not run a live gateway repro in this read-only review.

## Solution Assessment

No: the current PR preserves omitted web surfaces for every refresh, but normal config writes publish canonical next source configs where omission may be an intentional removal. A safer fix needs a partial-source signal or a writer-side repair plus explicit-removal regression coverage.

## Review Findings

Overall correctness: patch is incorrect
