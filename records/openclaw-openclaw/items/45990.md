## Summary

Keep open, but the patch is not merge-ready: current main still has the per-call observability gap, while this branch drops pre-stream failure telemetry, changes existing llm_output semantics without compatibility markers, bypasses usage normalization, and predates current hook metadata. The PR body also lacks inspectable real behavior proof.

## What This Changes

The PR moves raw llm_input and llm_output plugin hook emission from attempt start/end to per-assistant-round stream events and updates hook comments plus tests.

## Best Possible Solution

Ship an additive, documented per-call observability contract that preserves attempt-level aggregate compatibility, pre-stream input failure telemetry, normalized usage, and current context/tool metadata.

## Reproduction Assessment

Yes. Source inspection shows current main records and commits assistant usage per message_end, but raw llm_output is still dispatched once at the attempt boundary with accumulated getUsageTotals data; I did not run a live multi-round agent in this read-only review.

## Solution Assessment

No. The PR targets the right gap, but replacing the existing aggregate hooks solely with stream-event hooks is not the narrowest maintainable solution until aggregate compatibility, failure telemetry, usage normalization, and current metadata are preserved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Emit llm_input before pre-stream failures:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:74`
  - body: Moving llm_input emission to assistant message_start means auth, rate-limit, network, timeout, or precheck failures during prompt submission no longer emit input telemetry. Current main emits before activeSession.prompt, so tracing plugins would lose the failing round.
  - confidence: 0.9
- **[P2] Normalize usage before emitting per-round output:** `src/agents/pi-embedded-subscribe.ts:407-418`
  - body: The new per-round output path forwards message.usage through a narrow cast instead of the existing normalizeUsage path. Provider shapes using totalTokens, prompt_tokens, completion_tokens, cache-token fields, or timing fields can be dropped, so plugins may undercount or miss usage.
  - confidence: 0.88
- **[P2] Keep aggregate llm_output distinguishable:** `src/agents/pi-embedded-subscribe.ts:405-418`
  - body: The patch reuses llm_output for per-round payloads and removes the final aggregate event without a round index, aggregate marker, or separate hook. Existing plugins cannot tell whether assistantTexts and usage describe one call or the whole turn.
  - confidence: 0.86
- **[P2] Preserve current hook metadata in per-round events:** `src/agents/pi-embedded-subscribe.ts:400-420`
  - body: Current main's hook contract includes fields such as tools on llm_input and contextTokenBudget, contextWindowSource, resolvedRef, harnessId, and prompt on llm_output. The new subscription-level emitter only sends the older core fields, so a rebase that keeps this shape would silently drop documented metadata.
  - confidence: 0.84
