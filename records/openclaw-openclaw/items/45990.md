## Summary

Keep open: the observability gap is real, but the branch replaces aggregate hooks with per-round stream events in a way that drops failure telemetry, usage normalization, current metadata, and compatibility, and it still lacks inspectable real behavior proof.

## What This Changes

The PR moves raw `llm_input`/`llm_output` plugin hook dispatch from embedded attempt boundaries into `message_start`/`message_end` stream handlers and updates hook comments plus tests.

## Best Possible Solution

Ship an additive, documented per-call observability contract that preserves aggregate `llm_output` compatibility, pre-stream input telemetry, normalized usage, current metadata, and raw conversation access gating.

## Reproduction Assessment

Yes. Source inspection on current main shows each assistant `message_end` can record usage, while raw `llm_output` still dispatches once later with aggregate `getUsageTotals()` data; I did not run a live multi-round agent in this read-only review.

## Solution Assessment

No. The PR targets the right gap, but replacing the existing aggregate hook semantics solely with stream-event hooks is not the narrowest maintainable solution until compatibility, failure telemetry, normalized usage, and current metadata are preserved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Emit llm_input before prompt submission:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:74`
  - body: Moving `llm_input` emission to `message_start` means rounds that fail before the first stream event, such as auth, rate-limit, network, timeout, or prompt precheck failures, no longer emit input telemetry. Current main emits before `activeSession.prompt(...)`, so tracing plugins would lose the failing round.
  - confidence: 0.9
- **[P2] Normalize usage before emitting per-round output:** `src/agents/pi-embedded-subscribe.ts:407-418`
  - body: The new per-round output path forwards `message.usage` through a narrow cast instead of the existing `normalizeUsage` path. Provider shapes using `totalTokens`, `prompt_tokens`, `completion_tokens`, cache-token fields, or timing fields can be dropped, so plugins may undercount or miss usage.
  - confidence: 0.88
- **[P2] Keep aggregate llm_output distinguishable:** `src/agents/pi-embedded-subscribe.ts:405-418`
  - body: The patch reuses `llm_output` for per-round payloads and removes the final aggregate event without a round index, aggregate marker, or separate hook. Existing plugins cannot tell whether `assistantTexts` and `usage` describe one call or the whole turn, and aggregate consumers lose their current event.
  - confidence: 0.86
- **[P2] Preserve current llm_output metadata:** `src/agents/pi-embedded-subscribe.ts:400-420`
  - body: Current main's hook contract includes context-window metadata, `resolvedRef`, `harnessId`, and `prompt` on `llm_output`. This new subscription-level emitter sends only the older core fields, so a rebase that keeps this shape would silently drop metadata for plugins.
  - confidence: 0.84
