## Summary

Keep this PR open for maintainer review. Current main still emits raw `llm_input` before prompt submission and raw `llm_output` once after the embedded attempt with accumulated usage, so the requested per-round raw hook behavior is not implemented; the PR contains useful source work but has blocking contract/correctness issues before merge.

## What This Changes

The PR moves embedded-runner `llm_input` and `llm_output` hook emission from attempt-level start/end into subscription `message_start`/`message_end`, threads provider/model/context data into subscription params, and updates hook tests and comments for per-round semantics.

## Best Possible Solution

Define and document a stable per-LLM-call observability contract alongside #64268, preserve failed-submission input telemetry, normalize per-call usage through the existing usage helper, and clearly version or label per-call versus aggregate output semantics for plugin authors.

## Reproduction Assessment

Yes. The blocking issues reproduce by inspection and focused tests: make `activeSession.prompt(...)` fail before any assistant `message_start`, and feed assistant usage shaped like `{ totalTokens: 42 }` through the proposed per-round output path.

## Solution Assessment

No. The direction is useful, but emitting raw input only at `message_start` is later than the provider-call boundary, and the output helper bypasses the normalized usage contract; the safer path keeps failed-submission observability and explicitly designs the per-call versus aggregate contract.
