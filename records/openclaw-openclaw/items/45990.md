## Summary

Keep open: current main still lacks the requested per-call raw hook contract, and the linked issue remains open, but this PR needs API-contract fixes, a rebase, and real behavior proof before merge.

## What This Changes

The PR moves `llm_input` and `llm_output` hook emission into embedded Pi message start/end handling so plugins can observe each agentic LLM round.

## Best Possible Solution

Ship an additive, documented per-call observability contract that keeps existing aggregate compatibility, preserves current hook metadata, normalizes usage, and records input telemetry for failed prompt submissions.

## Reproduction Assessment

Yes. Source inspection on current main shows per-message usage is available at `message_end`, while raw `llm_output` is still dispatched once later with aggregate `getUsageTotals()` data.

## Solution Assessment

No. The direction is useful, but replacing the existing hook semantics in place is not the safest shape; an additive hook or explicit round/aggregate markers with preserved metadata is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep aggregate hook compatibility:** `src/agents/pi-embedded-subscribe.ts:405-418`
  - body: The new per-round path replaces the single aggregate `llm_output` event with one event per assistant message, but it keeps the same hook name and does not add a round index, aggregate marker, or final aggregate event. Existing plugins that compute run cost from `usage` or build a transcript from `assistantTexts` will silently change behavior after this patch.
  - confidence: 0.87
- **[P2] Emit input hooks before prompt submission:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:74`
  - body: Moving `llm_input` to `message_start` means auth, rate-limit, network, or other failures thrown by `activeSession.prompt(...)` before the first stream event no longer emit input telemetry. Current main emits the hook before prompt submission, so this regresses observability for the failure cases these hooks are meant to debug.
  - confidence: 0.9
- **[P2] Normalize per-round usage before emitting:** `src/agents/pi-embedded-subscribe.ts:407-418`
  - body: This casts `message.usage` to the hook shape instead of passing it through the existing usage normalizer. Providers that report `totalTokens`, `total_tokens`, prompt/completion aliases, cache aliases, or timing usage will produce missing or partial `llm_output.usage` values on the new per-round path.
  - confidence: 0.89
- **[P2] Preserve current output metadata:** `src/agents/pi-embedded-subscribe.ts:400-420`
  - body: The moved `llm_output` payload omits current-main fields such as `contextTokenBudget`, `contextWindowSource`, `contextWindowReferenceTokens`, `resolvedRef`, `harnessId`, and trace/run context currently emitted from `attempt.ts`. Rebasing this change without carrying those fields forward would silently remove public hook metadata.
  - confidence: 0.84
