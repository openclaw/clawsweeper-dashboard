## Summary

This PR should stay open because it changes a public plugin hook contract and still has concrete correctness issues: input hooks move too late to observe prompt-submission failures, output usage bypasses current normalization, and the related per-call observability design remains open in https://github.com/openclaw/openclaw/issues/64268.

## What This Changes

The PR moves `llm_input` and `llm_output` plugin hook emission from attempt-level run boundaries into embedded stream message start/end handling so hooks fire once per assistant LLM round, with updated hook comments and tests.

## Best Possible Solution

Resolve the per-LLM-call observability contract in the canonical design thread, preserving prompt-submission input telemetry and normalized usage while either adding a distinct per-call hook or clearly versioning/documenting changed hook semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: make `activeSession.prompt(...)` fail before any assistant `message_start` and the PR will not emit `llm_input`; feed usage shaped like `{ totalTokens: 42 }` through the PR output path and `usage.total` will not be normalized.

## Solution Assessment

No. Per-call observability is useful, but this PR is not yet the safest contract because it moves input telemetry after prompt submission and changes output payload semantics without a settled API decision.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Emit input hooks before prompt submission:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:74`
  - body: Triggering `llm_input` only from `message_start` drops telemetry when `activeSession.prompt(...)` fails before the first stream event, such as auth, rate-limit, or network failures. Current main emits before prompt submission, so tracing plugins would lose exactly the failed round they need to diagnose.
  - confidence: 0.92
- **[P2] Normalize assistant usage before emitting output:** `src/agents/pi-embedded-subscribe.ts:413-418`
  - body: `emitLlmOutputHook` forwards `message.usage` through a narrow cast instead of `normalizeUsage`. Provider aliases such as `totalTokens`, `total_tokens`, `prompt_tokens`, and cache-token fields can be dropped, so per-round hook consumers may undercount or miss usage.
  - confidence: 0.9
