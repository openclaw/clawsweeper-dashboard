## Summary

Keep open: the underlying per-call observability gap still exists on current main, but this branch is not merge-ready because it regresses failure telemetry and changes the public hook payload contract without a compatibility path or inspectable real behavior proof.

## What This Changes

The branch moves `llm_input` and `llm_output` emission from the embedded attempt boundary into assistant `message_start`/`message_end` handling and updates hook comments/tests for per-round behavior.

## Best Possible Solution

Ship an additive, documented per-call observability contract that preserves pre-stream failure telemetry and keeps aggregate turn summaries distinguishable from per-call payloads.

## Reproduction Assessment

Yes. Source inspection shows current main records per-assistant-message usage in `handleMessageEnd()` but emits a single aggregate `llm_output` later from `runEmbeddedAttempt`; I did not run a live multi-round agent in this read-only review.

## Solution Assessment

No. The PR targets the right observability gap, but replacing attempt-level hooks solely with stream-event hooks is not the safest solution until failure telemetry, usage normalization, context metadata, and aggregate-versus-per-call compatibility are settled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Emit llm_input before pre-stream failures:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:74`
  - body: Moving `llm_input` emission to assistant `message_start` means auth, rate-limit, network, or timeout-before-first-byte failures during prompt submission no longer emit input telemetry. Current main emits before prompt execution, so tracing plugins would lose the failing round.
  - confidence: 0.9
- **[P2] Normalize usage before emitting per-round output:** `src/agents/pi-embedded-subscribe.ts:407-418`
  - body: The new per-round output path forwards `message.usage` through a narrow cast instead of the existing normalization path. Provider shapes using `totalTokens`, `prompt_tokens`, `completion_tokens`, cache-token fields, or timing fields can be dropped, causing plugins to miss or undercount usage.
  - confidence: 0.88
- **[P2] Keep aggregate llm_output distinguishable:** `src/agents/pi-embedded-subscribe.ts:405-418`
  - body: The patch reuses the existing `llm_output` hook name for per-round events and removes the final aggregate event without adding a round index, aggregate marker, or separate hook. Existing plugins cannot tell whether `assistantTexts` and `usage` are one call or the whole turn.
  - confidence: 0.84
- **[P2] Preserve llm_output context metadata:** `src/agents/pi-embedded-subscribe.ts:393-418`
  - body: Current main includes `contextTokenBudget`, context-window source fields, `resolvedRef`, `harnessId`, and the prompt on `llm_output`. The new subscription-level event only sends run/session/provider/model/text/usage fields, so merging it would silently drop documented metadata.
  - confidence: 0.82
