## Summary

Keep open: the feature direction is useful, but the PR should not merge as-is because it drops pre-stream llm_input telemetry, bypasses usage normalization, lacks release-note coverage, and overlaps the still-open #64268 hook-contract design.

## What This Changes

This PR moves embedded llm_input/llm_output plugin hooks from one attempt-level emission to per-assistant-round message_start/message_end emissions and updates hook comments/tests.

## Best Possible Solution

Define the per-LLM-call observability contract alongside #64268, keep llm_input at prompt submission, normalize per-call usage, and update hook types, docs, changelog, and focused tests together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: make activeSession.prompt fail before any assistant message_start and llm_input will not fire; feed usage shaped like { totalTokens: 42 } through the PR output path and usage.total will not be normalized.

## Solution Assessment

No. Per-call observability is a reasonable feature, but this implementation is not the narrowest maintainable path because it moves input telemetry too late and bypasses the existing usage-normalization contract.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Emit input hooks before prompt submission:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:74`
  - body: Moving llm_input to assistant message_start drops telemetry when activeSession.prompt fails before the first stream event, such as auth, rate-limit, or network failures. Current main emits before prompt submission, so tracing plugins would lose exactly the failed round they need to diagnose.
  - confidence: 0.92
- **[P2] Normalize assistant usage before emitting output:** `src/agents/pi-embedded-subscribe.ts:407-418`
  - body: emitLlmOutputHook forwards message.usage through a narrow cast instead of normalizeUsage. Provider aliases such as totalTokens, total_tokens, prompt_tokens, and cache-token fields can be dropped, so per-round hook consumers may undercount or miss usage.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `src/plugins/types.ts:594`
  - body: This changes public plugin hook frequency and payload semantics but leaves CHANGELOG.md untouched. Plugin authors need release-note migration guidance when llm_input/llm_output stop behaving like once-per-run payloads.
  - confidence: 0.86
