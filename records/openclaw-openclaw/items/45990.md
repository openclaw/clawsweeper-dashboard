## Summary

Keep open: current main still has the observability gap this PR targets, but the branch replaces attempt-level hook semantics with per-round stream events in a way that regresses failure telemetry and needs plugin API contract review. The PR also lacks inspectable contributor real behavior proof, so it is not ready to merge.

## What This Changes

The PR moves `llm_input` and `llm_output` dispatch into embedded PI assistant `message_start`/`message_end` handling, passes provider/model context into subscription params, updates hook comments, and adds per-round hook tests.

## Best Possible Solution

Ship an additive, documented per-call observability contract that preserves pre-stream failure telemetry and clearly distinguishes per-call payloads from aggregate turn summaries.

## Reproduction Assessment

Yes for the underlying behavior gap: source inspection shows per-assistant-message usage is recorded in `handleMessageEnd`, while current main emits one aggregate `llm_output` later from `runEmbeddedAttempt`. I did not run a live multi-round agent in this read-only review.

## Solution Assessment

No. The PR addresses the right observability gap, but replacing attempt-level hooks solely with stream-event hooks is not the safest solution until failure telemetry, normalized usage, and aggregate-versus-per-call API compatibility are settled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Emit llm_input before pre-stream failures:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:74`
  - body: Moving `llm_input` emission to `message_start` means prompts that fail before the provider emits a stream event, such as auth, rate-limit, network, or timeout-before-first-byte failures, no longer emit input telemetry. Current main emits before prompt execution, so this regresses failure observability for tracing plugins.
  - confidence: 0.9
- **[P2] Normalize usage before emitting per-round output:** `src/agents/pi-embedded-subscribe.ts:407-418`
  - body: The new per-round output path forwards `message.usage` through a narrow cast instead of the existing normalization path. Provider shapes using fields such as `totalTokens`, `prompt_tokens`, `completion_tokens`, cache tokens, or timing fields can be dropped, causing plugins to undercount or miss usage data.
  - confidence: 0.88
- **[P2] Keep aggregate llm_output distinguishable:** `src/agents/pi-embedded-subscribe.ts:405-418`
  - body: The patch reuses the existing `llm_output` hook name for per-round events and removes the final aggregate event without adding a round index, aggregate marker, or separate hook. Existing plugins that treat `assistantTexts` and `usage` as the full-run payload cannot tell whether they are seeing one call or the final run summary.
  - confidence: 0.82
