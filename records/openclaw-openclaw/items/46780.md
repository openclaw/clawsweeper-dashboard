## Summary

Keep open. Current main has generic plugin tool hooks and runtime agent-event subscriptions, but it still does not expose a first-class skill invocation/success/failure lifecycle hook; the related diagnostics PR is open and would not by itself settle the product/security contract for intercepting skill use.

## What This Changes

Request to add before/after skill execution hooks or plugin events so skill usage can be monitored, analyzed, and permission-checked.

## Best Possible Solution

Define a documented skill lifecycle contract, or explicitly document the supported combination of runtime events and tool hooks if maintainers decide not to add first-class skill execution events.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source and docs inspection give a high-confidence current-main check that generic tool/run events exist but first-class skill lifecycle hooks do not.

## Solution Assessment

Unclear until maintainer review. A skill hook may be useful, but the narrowest maintainable solution depends on defining skill execution semantics and how they compose with existing onAgentEvent, before_tool_call, after_tool_call, and diagnostics surfaces.
