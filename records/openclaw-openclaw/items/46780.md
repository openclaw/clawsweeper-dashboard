## Summary

Keep open. Current main still lacks a skill execution lifecycle hook, and the enterprise permission-check use case makes this a core plugin API/security design gap rather than stale or ClawHub-only work.

## What This Changes

The issue requests before/after skill execution hooks so plugins or scripts can log, monitor, analyze, or gate skill invocations.

## Best Possible Solution

Add an additive, documented typed plugin hook surface around explicit skill invocation, with a pre-invocation decision hook and a post-invocation observation hook, then define whether and how prompt-based skill use is represented.

## Reproduction Assessment

Not applicable as a runtime bug; static inspection of the current hook catalogs, hook runner, internal hook event types, skill metadata, docs, and slash-skill dispatch path gives high confidence that the requested lifecycle boundary is absent.

## Solution Assessment

Unclear for the proposed JSON script-hook shape; a typed plugin API with explicit blocking, approval, timeout, ordering, and redaction semantics is the safer maintainable direction.
