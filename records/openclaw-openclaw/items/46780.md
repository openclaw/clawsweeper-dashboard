## Summary

Keep open. Current main still has plugin and internal hooks for agent, message, tool, session, gateway, install, and dispatch surfaces, but no skill execution lifecycle hook; the enterprise permission-check comment makes this a core plugin API/security design question rather than stale cleanup or ClawHub-only work.

## What This Changes

Add before/after skill execution lifecycle hooks for logging, analytics, debugging, monitoring, and permission checks.

## Best Possible Solution

Add an additive, documented typed plugin hook surface around skill invocation, with a pre-invocation decision hook and post-invocation observation hook that cover both direct skill commands and model-selected skill paths.

## Reproduction Assessment

Not applicable as a runtime bug. Static inspection of the current hook catalogs, hook runner, internal hook event types, skill metadata, docs, and slash-skill dispatch path gives a high-confidence way to verify the requested lifecycle boundary is absent.

## Solution Assessment

Unclear for the proposed JSON script-hook shape. A typed plugin API with explicit blocking, approval, timeout, ordering, and redaction semantics is the safer maintainable direction for the permission-gating use case.
