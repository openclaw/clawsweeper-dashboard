## Summary

Keep open. Current main still has no skill-specific before/after execution lifecycle hook, while existing tool, agent-run, install, and runtime event surfaces only cover adjacent parts of the request. The related OTel PR is still open and only addresses observability, not the blocking/interception semantics requested in the issue discussion.

## What This Changes

Request to add skill execution lifecycle hooks or events so plugins or scripts can log, analyze, monitor, or block skill use.

## Best Possible Solution

Define whether OpenClaw should expose skill usage as a first-class lifecycle contract; if yes, add a narrow documented plugin API with privacy, permission, and command-vs-prompt semantics instead of ad hoc script hooks.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection give a high-confidence current-main check that there is no skill-specific lifecycle hook; existing hook surfaces cover tools, agent runs, install policy, messages, and runtime event subscription.

## Solution Assessment

Unclear until maintainer product review. A narrow plugin lifecycle or diagnostics contract may be maintainable, but the proposed script-style beforeExecute/afterExecute hooks overlap with existing before_tool_call, before_agent_run, runtime events, and the fact that skills are currently instructional artifacts.
