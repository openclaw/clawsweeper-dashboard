## Summary

Keep open. Current main still has tool-, install-, run-, session-, and gateway-scoped hooks, but no per-skill before/after execution hook, success/failure event, or blocking skill authorization boundary with reliable skill attribution.

## What This Changes

Request to add skill lifecycle hooks or events around skill invocation so integrations can log, analyze, monitor, or authorize skill execution.

## Best Possible Solution

Define a first-class, versioned skill execution lifecycle contract with reliable skill identity, invocation source, blocking semantics, success/failure phases, timeout/error behavior, privacy boundaries, and a documented relationship to diagnostics, tool hooks, setup hooks, and permission manifests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection give a high-confidence current-main absence check for the requested skill execution hook names, skill identity payload, and before/after outcome contract.

## Solution Assessment

Unclear. A first-class plugin lifecycle event may be the right direction, but maintainers need to decide attribution, blocking behavior, failure semantics, privacy, and overlap with diagnostics before implementation.
