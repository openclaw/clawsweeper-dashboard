## Summary

Keep open. Current main still lacks a skill execution lifecycle hook, and the enterprise permission-check comment makes this a core plugin API and security-design gap rather than stale cleanup; related PR #72645 is still open telemetry work, not a merged authorization hook.

## What This Changes

The issue requests before/after skill execution lifecycle hooks so plugins or scripts can log, analyze, monitor, or gate skill runs.

## Best Possible Solution

Add an additive, documented typed plugin hook surface around explicit skill invocation, with a pre-invocation decision hook and post-invocation observation hook, then explicitly document any limits for prompt-selected skills.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a runtime bug; static inspection of current hook catalogs, hook runners, internal hooks, skill metadata, docs, diagnostics events, and explicit skill dispatch gives high confidence that the requested lifecycle boundary is absent.

## Solution Assessment

Is this the best way to solve the issue? Unclear for the proposed JSON script-hook shape; a typed plugin API with explicit blocking, approval, timeout, ordering, and redaction semantics is the safer maintainable direction.
