## Summary

Keep open. Current main still lacks per-skill before/after invocation or success/failure lifecycle hooks with skill attribution; existing hooks are tool, run, install, session, and gateway scoped, while the related OTel, setup, and install-policy threads cover different lifecycle moments.

## What This Changes

Add skill execution lifecycle hooks or execution logging so plugins or scripts can observe, analyze, and potentially gate skill invocation outcomes.

## Best Possible Solution

Define a versioned skill execution lifecycle contract with skill identity, invocation source, blocking semantics, success/failure phases, privacy boundaries, and a documented relationship to tool hooks and diagnostics events.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection gives a high-confidence current-main check that the requested skill execution hook names and payload contract are absent.

## Solution Assessment

Unclear. A first-class plugin lifecycle hook may be the right direction, but maintainers need to decide execution boundaries, skill attribution, blocking behavior, failure semantics, and overlap with diagnostics before implementation.
