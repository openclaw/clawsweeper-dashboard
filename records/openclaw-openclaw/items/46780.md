## Summary

Keep open. Current main still has tool-call and install-policy hooks, but no per-skill before/after invocation or success/failure lifecycle hook with skill attribution; the related OTel PR is still open and observability-only, while setup/pre-install issues cover different lifecycle moments.

## What This Changes

The issue requests skill lifecycle hooks around invocation, completion, and failure so plugins or scripts can observe or gate skill execution.

## Best Possible Solution

Add a versioned plugin/skill lifecycle contract with explicit skill identity, blocking semantics, success/failure phases, privacy boundaries, and a documented relationship to existing tool hooks and diagnostics events.

## Reproduction Assessment

Not applicable. This is a feature/API request; current-main source and public docs give a high-confidence check that the requested skill lifecycle hook names and payload contract are absent.

## Solution Assessment

Unclear. A first-class plugin lifecycle hook may be the right direction, but maintainers need to decide event timing, skill attribution, failure semantics, permission behavior, and overlap with diagnostics telemetry before implementation.
