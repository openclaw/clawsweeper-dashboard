## Summary

Keep open. Current main still has tool hooks and observational agent events, but no per-skill before/after or success/failure lifecycle hook with reliable skill attribution or blocking semantics; the request needs maintainer product and security review.

## What This Changes

Add skill lifecycle hooks such as before/after execution or invoke/success/fail events for logging, analytics, debugging, monitoring, and permission checks.

## Best Possible Solution

Define a versioned skill execution lifecycle contract with skill identity, invocation source, blocking semantics, success/failure phases, privacy boundaries, and a documented relationship to tool hooks, agent events, and diagnostics telemetry.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection on current main gives a high-confidence absence check for the requested skill execution hook names, payload contract, and lifecycle phases.

## Solution Assessment

Unclear. First-class plugin lifecycle hooks may be the right direction, but maintainers need to decide skill attribution, blocking behavior, failure semantics, privacy/redaction, and overlap with diagnostics before implementation.
