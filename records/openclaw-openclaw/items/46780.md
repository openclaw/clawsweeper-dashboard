## Summary

Keep open. Current main has tool-, install-, session-, run-, and agent lifecycle hooks plus skill prompt/command surfaces, but it still lacks a skill-attributed before/after execute lifecycle event or blocking skill authorization hook; the related telemetry/setup items do not cover that contract.

## What This Changes

Request to add before/after skill execution lifecycle hooks or plugin events for logging, analytics, monitoring, and pre-execution permission checks.

## Best Possible Solution

Define a first-class, versioned skill execution lifecycle contract with reliable skill identity, invocation source, blocking semantics, success/failure phases, timeout/error behavior, privacy boundaries, and documented overlap with tool hooks, diagnostics, setup hooks, runtime events, and permission manifests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection on current main verify the requested hook names, skill identity payload, and before/after outcome contract are absent.

## Solution Assessment

Unclear. A first-class plugin lifecycle event may be the right direction, but maintainers need to decide attribution, blocking behavior, failure semantics, privacy, and overlap with diagnostics/runtime events before implementation.

## Risks / Open Questions

- A pre-execution skill hook used for permissions would become a security boundary, so fail-closed behavior, timeout handling, redaction, and exposed prompt/env context need explicit design.

- Related observability work may help analytics, but telemetry alone would not satisfy blocking authorization or reliable before/after execution semantics.
