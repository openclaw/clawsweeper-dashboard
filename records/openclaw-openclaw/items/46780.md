## Summary

Keep open. Current main exposes agent/tool/message/session/install plugin hooks, but not per-skill before/after execution or success/failure events; the linked diagnostics PR is observability-only and the linked setup-hook issue covers install/update, so this still needs maintainer API and security design.

## What This Changes

Request adds before/after skill execution lifecycle hooks or events so plugins or scripts can log, analyze, monitor, or block skill usage.

## Best Possible Solution

Add a versioned plugin/API lifecycle contract for skill invocation and outcomes, with explicit skill attribution, blocking semantics, error phases, privacy boundaries, and a documented relationship to existing tool hooks and diagnostics events.

## Reproduction Assessment

Not applicable. This is a feature/API request, and current-main source inspection shows the requested skill lifecycle hook names are absent while existing hooks cover tool calls and install vetting.

## Solution Assessment

Unclear. A first-class plugin hook may be the right direction, but maintainers need to decide event timing, skill attribution, failure semantics, and how this overlaps with existing before_tool_call/after_tool_call and diagnostics telemetry.
