## What This Changes

This PR adds a config-audit scrub helper, doctor health contribution, and tests to redact historical `argv`/`execArgv` values in `config-audit.jsonl` via `openclaw doctor --fix`.

## Best Possible Solution

Land a doctor `--fix` scrub that reuses the forward redactor, preserves permissions, detects or merges concurrent audit-log appends, and includes redacted real CLI proof plus focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible: current main redacts only new process snapshots and appends audit records without any historical rewrite path. I did not execute the CLI in this read-only review, but https://github.com/openclaw/openclaw/issues/80777 includes real before-state grep evidence from an affected log.

## Solution Assessment

No as-is: doctor-side scrubbing is the right narrow direction, but the implementation must not replace a live append-only audit log in a way that can drop concurrent records and it still needs real after-fix CLI proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Avoid overwriting concurrent audit appends:** `src/config/io.audit.ts:557`
  - body: The scrub reads `config-audit.jsonl` once and later renames that snapshot over the live file.
