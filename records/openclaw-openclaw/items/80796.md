## Summary

Keep open: the PR addresses a real config-audit credential-at-rest bug and now has sufficient live doctor-log proof, but the proposed scrub still has a blocking race because it checks the audit-log size before writing the temp file and can overwrite appends that happen before the final rename.

## What This Changes

This PR adds a doctor `Config audit` health contribution plus `scrubConfigAuditLog` tests to rewrite historical config-audit `argv`/`execArgv` entries through the existing redactor.

## Best Possible Solution

Keep the doctor-side scrub, but make the replace step detect appends after temp-file creation or otherwise coordinate with audit writers, then land it as the fix for https://github.com/openclaw/openclaw/issues/80777.

## Reproduction Assessment

Yes. Current main redacts only new audit snapshots and still appends existing `config-audit.jsonl` records without a historical rewrite path; the linked issue and attached logs provide live before/after evidence from an affected log.

## Solution Assessment

No as-is. Doctor-side scrubbing through the existing redactor is the right narrow direction, but the file replacement guard must cover appends that happen after the current size check and before the rename.
