Full review comments:

- **[P1] Do not fetch returned URLs with provider auth:** `extensions/custom-image-providers/index.ts:161`
  - body: When the provider response contains `url` or `image_url`, this path sends the same request headers to that response-controlled URL. That can leak the user's provider `Authorization` header and bypass OpenClaw's SSRF controls.
  - confidence: 0.94
- **[P2] Route provider requests through the guarded HTTP helper:** `extensions/custom-image-providers/index.ts:317-321`
  - body: The image request is sent with bare `fetch`, so configured provider request settings, timeouts, dispatcher policy, and SSRF/private-network handling used by the shared OpenAI-compatible image provider are skipped.
  - confidence: 0.88
- **[P2] Avoid reading the state-dir config during plugin registration:** `extensions/custom-image-providers/index.ts:55-60`
  - body: The plugin decides which providers to register by reading `OPENCLAW_STATE_DIR/openclaw.json` itself, which bypasses the resolved runtime config path and can miss workspace/config-path snapshots that OpenClaw already loaded.
  - confidence: 0.78

## Security Review
