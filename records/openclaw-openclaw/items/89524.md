## Summary

The report is understandable and likely real for the WeChat integration, but the actionable send-media implementation is owned by the external openclaw-weixin package; current OpenClaw core normalizes the reply into one media payload and calls the channel adapter once per media URL.

## What This Changes

The issue reports that openclaw-weixin duplicates text and image delivery for MEDIA replies and asks for one combined WeChat message.

## Best Possible Solution

Keep OpenClaw core's generic one-send-per-media contract and route the duplicated WeChat delivery fix to the external openclaw-weixin plugin unless its maintainers identify a missing SDK seam.

## Reproduction Assessment

No. The reporter gives clear steps, but I did not have a live WeChat/plugin runtime repro; source inspection in this repo only proves OpenClaw core hands a single media send to the external plugin.

No for this repository. The maintainable fix belongs in @tencent-weixin/openclaw-weixin, unless that plugin can show OpenClaw's channel SDK is missing or breaking a needed contract.
