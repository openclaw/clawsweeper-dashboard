## Summary

Keep open: current main and v2026.6.6 still allow an oversized `contextTokens` override to win before the known model window is applied, and https://github.com/openclaw/openclaw/pull/90889 is the open closing-reference repair path.

## What This Changes

The issue asks OpenClaw to cap effective session `contextTokens` overrides by the selected model's known context window before status or session metadata reports them.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/90889 or an equivalent shared resolver/session-store fix, then close this issue from merged implementation evidence.

## Reproduction Assessment

Yes. Source inspection on current main shows the shared resolver returns the oversized override first, and the session-store path can persist the same value before applying a known smaller model window.

## Solution Assessment

Yes. The linked repair path is the narrow maintainable solution because it updates the shared resolver invariant and routes session-store persistence through that same resolver.
