## Label Justifications

- `impact:session-state`: The reported over-reporting affects resolved and persisted session context metadata consumed by status and downstream session logic.

## Summary

Keep open: current `main` and latest `v2026.6.6` still return/persist a positive context-token override before applying a smaller known model window in the shared resolver/session-store path. https://github.com/openclaw/openclaw/pull/90889 is the open closing-reference repair for the central oversized-override bug, while https://github.com/openclaw/openclaw/pull/92709 is useful complementary slash-model context work but does not make the original issue implemented yet.

## What This Changes

The issue requests that effective session context-window resolution use `min(override, knownModelWindow)` when both values are known, so `/status` and persisted session metadata cannot over-report the selected model's actual window.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/90889 or an equivalent shared resolver/session-store fix, then decide whether the slash-model work in https://github.com/openclaw/openclaw/pull/92709 remains separate before closing this issue from merged evidence.

## Reproduction Assessment

Yes. A source-level reproduction is clear: call `resolveContextTokensForModel` with a known provider/model window smaller than a positive override and current main returns the override first; the session-store path can then persist that value before applying the smaller known model window.

## Solution Assessment

Yes. The linked repair direction is the narrow maintainable fix because it makes the shared resolver the authority and routes session persistence through that capped result; a duplicate new fix PR should not be queued while the closing-reference PR remains open.
