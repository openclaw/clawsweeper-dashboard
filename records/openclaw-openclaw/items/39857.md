## What This Changes

The issue asks for effective session context tokens to be capped by the selected model's known context window when an override is larger.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/90889 or an equivalent shared resolver/session-store fix, with any sibling status-summary parity intentionally handled, then close this issue from merged evidence.

## Reproduction Assessment

Yes. Source inspection on current main shows the shared resolver returns the oversized override first, and the session-store path can persist the same value before applying a known smaller model window.

## Solution Assessment

Yes. The linked repair path is the narrow maintainable direction because it updates the shared resolver invariant and routes session-store persistence through that same resolver; PR review should also decide whether the CLI status-summary sibling needs parity.
