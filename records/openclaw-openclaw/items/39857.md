## Summary

Keep open: current main still has source-reproducible override-first context-token paths, and open PR https://github.com/openclaw/openclaw/pull/39859 already references this issue as the implementation candidate.

## What This Changes

The issue requests that effective session context window resolution cap configured overrides by a known selected-model context window before status reporting or session metadata persistence.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/39859 or an equivalent narrow fix that caps resolved, persisted, and reported effective context windows by known selected-model limits while preserving override behavior when the model window is unknown.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists by passing a positive contextTokensOverride larger than a known configured provider/model window through the shared resolver, session-store persistence, or status-summary path; current main can return or persist the override first.

## Solution Assessment

Yes. The maintainable fix is shared model-aware resolution plus updates to raw persistence and reporting callers; a status-only change would leave session metadata and other resolved-context consumers able to over-report.
