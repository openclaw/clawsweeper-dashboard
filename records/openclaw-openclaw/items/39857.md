- `impact:session-state`: The affected value is effective session context-window state stored in session metadata and surfaced through status paths.

## Summary

Keep open: current main and the latest release still have override-first context-window resolution in the shared resolver, session persistence, and CLI status summary, while the linked fix PR remains open and conflicting.

## What This Changes

The issue requests that effective session context-window resolution and reporting clamp an override to the selected model's known smaller context window when both values are available.

## Best Possible Solution

Review, rebase, or replace https://github.com/openclaw/openclaw/pull/39859 with a current-main fix that clamps known-window overrides in shared resolution and affected persistence/status callers while leaving broader context/max-token policy to https://github.com/openclaw/openclaw/issues/71136.

## Reproduction Assessment

Yes. Source inspection on current main shows a larger configured or persisted `contextTokens` value can flow as `contextTokensOverride` through resolver, session-store, and CLI status-summary paths before the known model window is consulted.

## Solution Assessment

Yes for the narrow bug. The maintainable fix is to clamp only when both an override and a known model window are available, and the best path is the existing linked PR rebased or replaced rather than a parallel fix.
