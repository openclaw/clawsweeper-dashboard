## Summary

Keep open: current main still has override-first context-token resolution in the shared resolver, session persistence, and CLI status summary, and the linked closing PR remains open and conflicting rather than merged.

## What This Changes

The issue asks OpenClaw to cap effective session context tokens to the smaller known selected-model window when a configured override is larger.

## Best Possible Solution

Review, rebase, or replace https://github.com/openclaw/openclaw/pull/39859 with a current-main fix that clamps known-window overrides in the shared resolver and downstream status/session paths while leaving broader precedence policy to https://github.com/openclaw/openclaw/issues/71136.

## Reproduction Assessment

Yes. Source inspection shows that setting agents.defaults.contextTokens above a selected model's known smaller window can still flow through override-first resolver, session-store, and CLI status-summary paths on current main.

## Solution Assessment

Yes for the narrow behavior: min(override, knownWindow) is the right maintainable fix when both values are known. The implementation should be the existing linked PR rebased or replaced against current main, not a new parallel repair lane.
