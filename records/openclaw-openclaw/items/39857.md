## Summary

Keep open: current main still has source-reproducible override-first paths that can over-report a known smaller model window, and open PR https://github.com/openclaw/openclaw/pull/39859 directly references this issue as the implementation candidate.

## What This Changes

The issue asks OpenClaw to cap effective session context tokens to the smaller of a configured override and the selected model's known window.

## Best Possible Solution

Land PR https://github.com/openclaw/openclaw/pull/39859 or an equivalent narrow fix that caps configured and persisted effective context windows by known selected-model limits while preserving override behavior when the model window is unknown.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: pass a positive contextTokensOverride larger than a known configured model window to the shared resolver or session-store/status-summary path, and current main returns or persists the override before capping it.

## Solution Assessment

Yes. The best fix is shared model-aware resolution plus updating raw persistence/reporting callers; a status-only change would leave session metadata and status summary able to over-report.
