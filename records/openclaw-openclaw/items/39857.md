## Best Possible Solution

Review, rebase, or replace https://github.com/openclaw/openclaw/pull/39859 with a narrow current-main fix that applies min(override, knownWindow) in the shared resolver, session persistence, and status summary while preserving override behavior when the model window is unknown.

## Solution Assessment

Yes. The maintainable fix is a model-aware cap in the shared resolver plus the named persistence/status bypasses; a display-only change would leave stored metadata wrong.
