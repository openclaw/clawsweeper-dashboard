## Summary

Keep open: current main still has override-first context-token paths that can report or persist a larger override before checking the selected model's known window, and the linked fixing PR is still open and merge-conflicting.

## What This Changes

The issue asks for effective session/status context tokens to be capped to `min(contextTokens override, known selected-model window)` when both values are known.

## Best Possible Solution

Review, rebase, or replace https://github.com/openclaw/openclaw/pull/39859 with a current-main fix that applies the cap consistently in the shared resolver, session persistence, and status summary paths while preserving override behavior when the model window is unknown.

## Reproduction Assessment

Yes, source-level. Configure `agents.defaults.contextTokens` above a selected model's known window; the shared resolver, session store, and status summary paths still choose the override before the known smaller window.

## Solution Assessment

Yes. The requested `min(override, knownWindow)` behavior is the narrow maintainable fix, but it needs to cover all resolver and persistence/status bypasses rather than only display wording.
