## Summary

Keep open: current main and the latest release still have override-first context-token paths that can over-report a smaller known model window, and the direct same-author fixing PR is still open rather than merged.

## What This Changes

The issue requests capping effective session/status context tokens to the smaller of a configured override and the selected model's known context window.

## Best Possible Solution

Review, rebase, or replace https://github.com/openclaw/openclaw/pull/39859 with a narrow current-main fix that applies min(override, knownWindow) consistently while preserving override behavior when the model window is unknown.

## Reproduction Assessment

Yes for source-level reproduction. Current main still returns positive overrides before known-window resolution in the shared resolver, session persistence, and status-summary paths.

## Solution Assessment

Yes, the requested direction is the right narrow fix. The implementation should cap in the shared resolver and remove the session/status bypasses rather than making a display-only change.
