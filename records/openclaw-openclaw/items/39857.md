## What This Changes

The issue requests that effective context-window resolution and status/session persistence use min(configured override, known selected-model window) when both values are known.

## Best Possible Solution

Review, rebase, or replace https://github.com/openclaw/openclaw/pull/39859 with a current-main fix that clamps known-window overrides in the shared resolver, session persistence, and status-summary paths while leaving broader precedence work to https://github.com/openclaw/openclaw/issues/71136.

## Reproduction Assessment

Yes, source-level. Configure agents.defaults.contextTokens above a selected model's known smaller window; current main still has resolver, session-store, and status-summary paths that choose the override before the smaller known model window.

## Solution Assessment

Yes for the narrow behavior: min(override, knownWindow) is the maintainable fix when both values are known. The implementation path should be the existing linked PR rebased or replaced against current main, not a new parallel repair lane.
