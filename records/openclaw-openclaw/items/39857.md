## Summary

Keep open: current main still has override-first context-token paths that can over-report a known smaller model window, and open PR #39859 directly links itself as the implementation candidate.

## What This Changes

The issue asks OpenClaw to cap effective session/status contextTokens to the smaller of configured or persisted overrides and the selected model's known context window.

## Best Possible Solution

Land #39859 or an equivalent narrow fix that caps configured and persisted effective context windows by known selected-model limits while preserving override behavior when the model window is unknown.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: set agents.defaults.contextTokens above a known provider/model window and follow the shared resolver, session-store, reply setup, or status summary paths where the override is returned before the smaller known window can cap it.

## Solution Assessment

Yes. The best fix is shared model-aware resolution that treats overrides as caps when a model window is known, plus updating raw persistence/reporting callers; a status-only change would leave session metadata able to over-report.
