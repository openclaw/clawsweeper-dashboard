## What This Changes

The issue requests that effective session context be capped to the smaller of an explicit override and a known selected-model window so status and session metadata do not over-report the model's real limit.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/90889 or an equivalent shared resolver/session-store fix, decide the slash-model sibling scope, then close this issue from merged evidence.

## Reproduction Assessment

Yes for a source-level reproduction: call `resolveContextTokensForModel` with a known provider/model window smaller than a positive override and current main returns the override first. The session-store path can then persist that value before applying the known smaller model window.

## Solution Assessment

Yes: the linked repair direction is the narrow maintainable solution because it fixes the shared resolver invariant and routes session persistence through that same result. A duplicate new fix PR should not be queued while that closing-reference PR remains open.
