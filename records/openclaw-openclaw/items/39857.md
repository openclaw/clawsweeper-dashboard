## Summary

Keep open: current main still has source-reproducible override-first context-window paths, and open PR https://github.com/openclaw/openclaw/pull/39859 directly references this issue as the implementation candidate.

## What This Changes

The issue asks OpenClaw to cap resolved session/status context windows to the selected model's known smaller context window when an override is larger.

## Best Possible Solution

Land PR https://github.com/openclaw/openclaw/pull/39859 or an equivalent narrow fix that caps overrides by known selected-model limits while preserving override behavior when the model window is unknown.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: pass a positive contextTokensOverride larger than a known configured provider/model window to the shared resolver, session-store, or status-summary path, and current main returns or persists the override before capping it.

## Solution Assessment

Yes. The best fix direction is shared model-aware resolution plus updating raw persistence/reporting callers; a status-only patch would leave session metadata and status summary able to over-report.
