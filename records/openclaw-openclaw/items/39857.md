## Summary

Keep open: current main and v2026.6.6 still have the override-first resolver/session-store path, while the open repair PR with closing syntax already owns the fix.

## What This Changes

The issue reports that effective session context tokens can be resolved from an oversized override instead of being capped by a known selected-model window.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/90889 or an equivalent shared resolver/session-store fix, preserving unknown-model override behavior, then close this issue from merged implementation evidence.

## Reproduction Assessment

Yes. Source inspection on current main shows `resolveContextTokensForModel` returns an oversized positive override before known model-window lookup, and the session-store path can persist that same value before applying a known smaller model limit.

## Solution Assessment

Yes. The narrow maintainable fix is to make the shared resolver treat overrides as caps when a known model window exists and to route session-store persistence through that resolver; the open linked PR follows that shape.
