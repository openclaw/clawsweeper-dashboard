## Summary

Keep open: current main still has source-reproducible override-first context-token paths, and https://github.com/openclaw/openclaw/pull/39859 remains the direct open implementation candidate.

## What This Changes

Cap effective session context tokens by the known selected-model window when an override is larger, including persisted session metadata and `/status` reporting.

## Best Possible Solution

Review, rebase or replace, and land https://github.com/openclaw/openclaw/pull/39859 or an equivalent narrow fix that caps resolver, session persistence, and status-summary outputs by known selected-model limits while preserving override behavior when the model window is unknown.

## Reproduction Assessment

Yes. Source inspection shows a high-confidence path: pass a positive contextTokensOverride above a configured lower provider/model window through the shared resolver, session-store persistence, or status-summary path, and the override can still win.

## Solution Assessment

Yes. The maintainable solution is model-aware min(override, knownWindow) resolution across the shared resolver plus the session and status bypasses; a display-only change would leave persisted metadata and downstream consumers wrong.
