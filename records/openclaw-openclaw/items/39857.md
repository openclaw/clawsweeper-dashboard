## Summary

Keep open. Current main still has override-first context-token resolution and raw configured/persisted override paths that can report a larger effective window than a known smaller model limit, and open PR #39859 directly links itself as the implementation candidate for this issue.

## What This Changes

The issue requests capping effective session context tokens to `min(configuredOverride, knownModelWindow)` when both values are known.

## Best Possible Solution

Land #39859 or an equivalent narrow fix that caps configured and persisted effective context windows by known selected-model limits while preserving override behavior when the model window is unknown.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: configure an agent/default context token override above a known provider/model window, then follow the resolver, reply setup, session-store, or non-fallback `/status` path where the override is returned before the smaller known model window can cap it.

## Solution Assessment

Yes. The best fix is in shared model-aware context resolution plus the raw persistence/reporting callers; a status-only or docs-only change would leave stored session metadata and reply setup behavior able to over-report.
