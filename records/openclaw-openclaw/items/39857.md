## Summary

Keep open: current main and the latest release still have override-first context-token resolution paths that can report a larger override instead of a smaller known model window, and the direct same-author fixing PR is still open with closing syntax.

## What This Changes

The issue requests capping effective session and status context-token reporting to the smaller of an override and the selected model's known window.

## Best Possible Solution

Review, rebase, or replace https://github.com/openclaw/openclaw/pull/39859 with a narrow fix that applies min(override, knownWindow) consistently in the shared resolver and status/session persistence paths while preserving override behavior when no model window is known.

## Reproduction Assessment

Yes for source-level reproduction. Current main still returns positive overrides before known-window resolution in the shared resolver, session-store persistence, and status-summary resolver paths.

## Solution Assessment

Yes, the requested direction is the right narrow fix. The implementation should cap in the shared resolver and remove the session/status bypasses, not make this a display-only change.
