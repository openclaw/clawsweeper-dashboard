## Summary

Keep open: current main still has source-reproducible override-first context-token paths in the shared resolver, session persistence, and status summary, while the direct closing PR remains open and conflicting.

## What This Changes

Cap effective session context tokens to the smaller of an override and the selected model's known context window in resolver, persisted session metadata, and status reporting.

## Best Possible Solution

Review, rebase or replace https://github.com/openclaw/openclaw/pull/39859 with a narrow fix that caps overrides by known model windows in the shared resolver, session persistence, and status summary while preserving override behavior when the model window is unknown.

## Reproduction Assessment

Yes. Source inspection on current main shows a positive contextTokensOverride can still bypass known lower provider/model limits in the shared resolver, session-store path, and status-summary path.

## Solution Assessment

Yes. The maintainable solution is model-aware min(override, knownWindow) resolution across the shared resolver plus the named persistence/status bypasses; a display-only change would leave stored metadata wrong.
