## Summary

Keep open. Current main still returns configured or persisted context-token overrides before checking known selected-model windows, and open PR #39859 directly links itself as the implementation candidate.

## What This Changes

The issue asks OpenClaw to cap effective session context-token overrides by a selected model's known context window before persisting or reporting them.

## Best Possible Solution

Land #39859 or an equivalent narrow fix that caps configured and persisted effective context windows by known selected-model limits while preserving override behavior when the model window is unknown.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: configure agents.defaults.contextTokens above a known provider/model window, then follow the resolver, session-store, reply setup, or non-fallback /status path where the override is returned before the smaller known window can cap it.

## Solution Assessment

Yes. The maintainable fix belongs in shared model-aware context resolution plus the raw persistence/reporting callers; a status-only change would leave other stored effective-window paths able to over-report.
