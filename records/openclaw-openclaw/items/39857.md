## Summary

Keep open: current main still has source-reproducible override-first context-token paths, and https://github.com/openclaw/openclaw/pull/39859 remains the direct open implementation candidate.

## What This Changes

The issue requests capping effective session/status context tokens to the smaller of a user override and the selected model's known context window.

## Best Possible Solution

Review, rebase or replace, and land https://github.com/openclaw/openclaw/pull/39859 or an equivalent narrow fix that caps resolver, session persistence, and status-summary outputs by known selected-model limits while preserving override behavior when the model window is unknown.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists by passing a positive contextTokensOverride larger than a known configured provider/model window through the shared resolver, session-store persistence, or status-summary path; current main can still return or persist the override first.

## Solution Assessment

Yes. The maintainable solution is shared model-aware min(override, knownWindow) resolution plus session-store and status-summary coverage; a display-only change would leave persisted metadata and other consumers able to over-report.
