## Summary

Keep open: current main and latest v2026.5.22 still have override-first context resolution in the shared resolver, session store, and CLI status summary, while the linked closing PR is open and merge-conflicting.

## What This Changes

The issue asks OpenClaw to clamp an explicit session context-token override to the selected model's known smaller context window before reporting or persisting it.

## Best Possible Solution

Review, rebase, or replace https://github.com/openclaw/openclaw/pull/39859 with a current-main fix that clamps known-window overrides in the shared resolver, session persistence, and CLI status summary while leaving broader context/max-token precedence policy to https://github.com/openclaw/openclaw/issues/71136.

## Reproduction Assessment

Yes. Source inspection on current main shows larger configured or persisted contextTokens values can flow as contextTokensOverride through resolver, session-store, and CLI status-summary paths before the known model window is consulted.

## Solution Assessment

Yes for the narrow bug. The maintainable fix is to clamp only when both an override and a known model window are available, and the best path is the existing linked PR rebased or replaced rather than a new parallel fix.
