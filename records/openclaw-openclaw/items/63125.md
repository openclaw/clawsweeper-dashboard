## What This Changes

The issue reports multiple regressions after upgrading from OpenClaw 2026.4.1 to 2026.4.8: gateway CPU spikes, `openclaw agent` timeouts, Discord voice join aborts, and Messenger bridge fallback replies caused by CLI timeout behavior.

## Best Possible Solution

Keep this umbrella open until maintainers either land and ship the Discord voice repair tracked by #63098 and resolve or split the gateway CPU/RPC-stall family tracked by #72338, then close this with release provenance or canonical follow-up links after affected-host retesting.

## Reproduction Assessment

The report gives concrete steps, and source inspection gives high-confidence simulated paths for the CLI fallback and Discord voice timeout behavior, but full verification needs an affected Linux host plus live Discord voice and Messenger bridge coverage.

## Solution Assessment

Not applicable as a patch, and current main is not enough to call the umbrella solved. The narrowest maintainable path is to fix the Discord plugin issue separately, continue gateway CPU/RPC retesting in #72338, and then close or split this issue once packaged-release evidence exists.
