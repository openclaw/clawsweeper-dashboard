## Summary

Keep open: current main and the latest release still short-circuit bare `/new` and `/reset` to static acknowledgements, and the open linked PR is the active candidate for changing that behavior.

## What This Changes

The issue asks to restore or make configurable bare `/new` and `/reset` so they can trigger the persona startup greeting instead of only returning a canned acknowledgement.

## Best Possible Solution

Keep this issue open while maintainers review https://github.com/openclaw/openclaw/pull/81939 or choose an explicit product path: restore startup-turn behavior for bare resets, or keep ack-only behavior with a documented opt-in for persona startup turns.

## Reproduction Assessment

Yes. Source inspection shows the empty-tail reset branch returns a terminal acknowledgement before the model path can evaluate `isBareSessionReset`, and current tests explicitly assert this no-model behavior.

## Solution Assessment

Unclear. Restoring fallthrough is a narrow fix, but the current tests and reporter discussion indicate ack-only behavior may have been intentional, so maintainers need to confirm the product default or choose an opt-in shape.
