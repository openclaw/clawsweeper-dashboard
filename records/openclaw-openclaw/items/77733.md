## Summary

Keep open. Current `main` and the latest release still short-circuit bare `/new` and `/reset` before the startup/model path, and an open PR with closing syntax now owns the narrow fallthrough fix, so this issue should stay open until that PR is reviewed and merged or rejected.

## What This Changes

The issue asks to restore persona/startup model execution for bare `/new` and `/reset`, or provide an opt-in configuration for that behavior.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/81939, then align reset handling, docs, tests, and live-channel proof around the chosen bare-reset contract.

## Reproduction Assessment

Yes. Source inspection on current `main` shows bare `/new` and `/reset` return a terminal command result before `get-reply-run.ts` can build the bare reset startup prompt; issue comments also confirm the same behavior in real WebChat and related channel installs.

## Solution Assessment

Unclear. Restoring fallthrough as in https://github.com/openclaw/openclaw/pull/81939 is the narrow code direction for the reported regression, but current docs/tests and https://github.com/openclaw/openclaw/pull/78968 encode acknowledgement-only behavior that maintainers need to reconcile.
