## Summary

Keep open. ## What This Changes

Requests that bare `/new` and `/reset` again run a startup/persona greeting, or expose a supported opt-in mode for that behavior instead of only returning static acknowledgements.

## Best Possible Solution

Choose the canonical bare-reset contract, then align auto-reply, Gateway `agent`, docs, tests, and real-channel proof around that decision, using a typed opt-in policy if acknowledgement-only remains the default.

## Solution Assessment

Unclear. Restoring fallthrough is the narrow repair for the reported regression, but current docs/tests and the broader reset-greeting policy discussion mean maintainers need to choose the permanent default or opt-in contract first.
