## Summary

Keep open. Current main and the latest release still short-circuit bare `/new` and `/reset` into static acknowledgements before the startup/model path, and the open closing PR is not merged, so the reported regression is not resolved yet.

## What This Changes

The issue asks for bare `/new` and `/reset` to run the persona startup/model greeting again, or expose an opt-in mode, instead of returning only static acknowledgements.

## Best Possible Solution

Choose the canonical bare-reset contract, then align auto-reply, Gateway agent reset handling, docs, tests, and real-channel proof around that decision before closing this issue.

## Reproduction Assessment

Yes. Source inspection shows bare `/new` and `/reset` return a terminal acknowledgement before `get-reply-run.ts` can build the bare-reset startup prompt, and focused tests assert no model call.

## Solution Assessment

Unclear. Restoring fallthrough is the narrow code path for the reported regression, but current acknowledgement-only tests and related PRs mean maintainers need to decide the permanent bare-reset contract first.
