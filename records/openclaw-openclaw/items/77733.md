## Summary

Keep open: current `main` and latest release still implement the reported bare `/new` and `/reset` no-model acknowledgement behavior, and an open linked PR is the candidate fix but needs maintainer reset-contract direction before this issue can close.

## What This Changes

The issue asks OpenClaw to restore, or make configurable, persona/startup model turns after bare `/new` and `/reset` instead of returning only a static acknowledgement.

## Best Possible Solution

Choose one maintainer-approved bare-reset contract across auto-reply, Gateway, docs, and tests; if acknowledgement-only stays the default, expose persona/startup turns through an explicit reset policy rather than an implicit fallback.

## Reproduction Assessment

Yes, at source level. Current `main` returns a terminal command result for bare `/new` and `/reset`, and the caller returns that result before the downstream bare-session startup prompt/model path can run.

## Solution Assessment

Unclear. Restoring fallthrough would address the reported regression, but current docs/tests now encode acknowledgement-only behavior, so the best fix needs a maintainer-approved reset policy instead of a single-surface revert.
