Keep open: current main and the latest release still implement the reported no-model bare `/new` and `/reset` acknowledgement path, and an open closing PR already owns the candidate fix while maintainers decide the reset contract.

## What This Changes

The issue asks OpenClaw to restore or make configurable persona/startup model turns for bare `/new` and `/reset` instead of returning only canned acknowledgements.

## Best Possible Solution

Choose one maintainer-approved bare-reset contract, then align auto-reply, Gateway `agent`, docs, tests, and proof around it; if acknowledgement remains the default, expose startup-turn behavior only through an approved explicit policy.

## Reproduction Assessment

Yes, at source level. Current `maybeHandleResetCommand` returns a terminal acknowledgement for bare `/new` and `/reset`, and current tests explicitly expect no model fallthrough.

## Solution Assessment

Unclear until maintainers choose the product contract. A straight fallthrough restore is mechanically narrow, but current docs and tests treat acknowledgement-only behavior as intentional, so the best fix may be a broader alignment or an approved opt-in startup-turn mode.
