## Summary

Keep open. Current main and v2026.5.7 still short-circuit bare `/new` and `/reset` to canned acknowledgements, and an open PR with closing syntax already targets this issue, so the issue should remain open until that PR or an equivalent maintainer-approved fix lands or is rejected.

## What This Changes

The issue asks to restore or configure the old behavior where bare `/new` and `/reset` fall through to an agent startup/persona turn instead of returning only a canned acknowledgement.

## Best Possible Solution

Keep this as the canonical tracker, review https://github.com/openclaw/openclaw/pull/77758 or an equivalent fix, and settle the reset contract with updated docs, e2e coverage, and proof that prior empty-provider/leaky-output regressions do not return.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.5.7 shows bare `/new` and `/reset` return `shouldContinue: false` before `runPreparedReply`, while the downstream bare-session startup prompt path still exists if command handling falls through.

## Solution Assessment

Unclear. Restoring fallthrough is a plausible narrow fix for the reported regression, but current docs/tests and the prior empty-provider-call fixes intentionally enforce no-model acknowledgements, so the linked PR needs maintainer review, real behavior proof, and aligned guardrails before it is the best solution.
