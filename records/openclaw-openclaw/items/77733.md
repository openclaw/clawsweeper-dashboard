Keep open. Current `main` and the latest release still short-circuit bare `/new` and `/reset` into static acknowledgements before the startup/model path, and the linked closing PR is open and conflicting rather than merged.

## What This Changes

The issue requests restoring or making configurable the bare `/new` and `/reset` persona/startup greeting behavior that changed to a static acknowledgement.

## Best Possible Solution

Choose one canonical bare reset contract, then align auto-reply, Gateway `agent` reset handling, docs, tests, and real-channel proof around that decision in the linked PR.

## Reproduction Assessment

Yes. Source inspection shows bare `/new` and `/reset` return a terminal acknowledgement before `get-reply-run.ts` can build the bare-reset startup prompt, and focused tests assert no model call.

## Solution Assessment

Unclear until maintainers choose the reset UX. Restoring fallthrough is the narrow code path for the reported regression, but the best fix needs entry-point parity with the Gateway `agent` path and tests/docs for the chosen behavior.
