## Summary

Keep open: current main still has the sessions_yield abort/write-lock lifecycle described by the report, and I did not find a merged or shipped fix for the live-PID parent transcript lock timeout. Related work exists, especially https://github.com/openclaw/openclaw/issues/81490 and https://github.com/openclaw/openclaw/pull/85955, but neither currently closes this specific held-lock completion-callback failure.

## What This Changes

The issue asks for sessions_yield to reliably release the yielded parent session transcript lock so subagent completion callbacks can write back without timing out.

## Best Possible Solution

Add regression coverage for the yielded-parent held-lock exit path, then ensure the embedded attempt session-lock controller releases any parent transcript lock before yielded-session completion delivery can contend on it; keep the route-resume work in https://github.com/openclaw/openclaw/issues/81490 separate unless maintainers consolidate the cluster.
