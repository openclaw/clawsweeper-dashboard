## Summary

Keep open: the reported sessions_yield lock timeout is still plausible on current main and in v2026.5.22, and the closest repair path is the unmerged https://github.com/openclaw/openclaw/pull/85716 rather than an already-landed fix.

## What This Changes

The issue reports that `sessions_yield` can leave a yielded parent session transcript lock owned by the live gateway, causing subagent completion writes to time out and miss callback delivery.

## Best Possible Solution

Land a bounded, regression-tested fix through https://github.com/openclaw/openclaw/pull/85716 or a narrower replacement that releases the parent transcript lock before subagent completion writes can contend.

## Reproduction Assessment

No, not as a live current-main reproduction from this review. The issue supplies concrete v2026.5.22 runtime lock evidence, and source inspection shows the relevant `sessions_yield` abort/lock lifecycle still present on current main.

## Solution Assessment

Unclear. A defensive release in the yield/abort cleanup path is plausible, but the best fix should keep abort settling bounded and prove the exact held-lock completion-write path with regression coverage.
