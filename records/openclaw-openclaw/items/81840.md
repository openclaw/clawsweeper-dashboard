## Summary

This matches the current, documented group/channel visible-reply policy rather than a Feishu dispatcher failure: group turns are processed, but normal final answers stay private unless the agent uses the message tool or the operator opts back into automatic group replies.

## What This Changes

The issue asks for Feishu group-chat turns routed to a sub-agent to auto-post final replies back to the group like direct-message turns.

## Best Possible Solution

Keep the current group/channel privacy default and direct operators who want legacy group auto-posting to set `messages.groupChat.visibleReplies: "automatic"` or rely on `message(action=send)`.

## Reproduction Assessment

Yes, from source: a default group/channel turn with the message tool available resolves to `message_tool_only`, so the agent processes the message but automatic final delivery is suppressed. I did not run a live Feishu room because the current code, docs, and tests already describe the logged behavior.

## Solution Assessment

Yes for the current product policy: using the existing visible-reply config or explicit message tool is the narrowest maintainable solution. Changing Feishu group defaults to match DMs would be a product-policy change, not a Feishu routing bug fix.
