## Summary

Keep open: current main and the latest release still lack a shared outbound mention parameter and the QQ Bot send path only emits ordinary text/markdown payloads. The feature fits messaging-channel direction, but it needs maintainer product/API direction because the proposed DingTalk-shaped fields would expand a shared plugin contract while DingTalk is not bundled and WeCom is an external plugin.

## What This Changes

The issue asks for outbound @mention support for DingTalk, QQ Bot, and WeCom group messages through the OpenClaw message tool and channel send paths.

## Best Possible Solution

Define a portable outbound mention contract or plugin-owned message-tool schema first, then implement focused mappings in the owning channel plugins and coordinate external plugin owners where needed.

## Reproduction Assessment

No bug reproduction is applicable because this is a feature request. Source inspection gives high confidence that current main does not expose or carry the requested outbound mention metadata.

## Solution Assessment

Unclear: outbound mention support is useful, but the proposed short-term platform-specific fields are not yet proven to be the best API. A portable mentions object or plugin-owned schema/payload mapping is the safer direction to decide first.
