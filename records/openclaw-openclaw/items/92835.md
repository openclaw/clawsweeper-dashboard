## Summary

Keep open: current main and v2026.6.9 still lack an outbound native mention contract for the message tool. QQBot is the only requested runtime present in this repo, while DingTalk is absent and WeCom is cataloged as an external plugin, so this needs maintainer direction on a portable versus plugin-owned schema before implementation.

## What This Changes

Request native @mention and @all support for DingTalk, QQBot, and WeCom group-message sends through the OpenClaw message tool.

## Best Possible Solution

Choose a portable mention contract or plugin-owned message-tool schema pattern, then implement QQBot mapping in-tree and coordinate DingTalk/WeCom mappings in their owning plugins with focused tests and platform proof.

## Reproduction Assessment

Yes for source-level verification: current main has no shared outbound mention fields and QQBot’s send path builds payloads without native mention metadata. No live DingTalk, QQBot, or WeCom transport reproduction was attempted.

## Solution Assessment

Unclear: the requested behavior is valid, but the proposed platform-specific fields are not clearly the best contract because plugin-owned schema contributions already exist. Maintainers should choose the API ownership shape before implementation.
