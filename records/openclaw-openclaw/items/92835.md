## Summary

Keep open: current main and v2026.6.6 still do not expose outbound mention metadata through the shared message tool or QQ Bot text payload path, and the remaining work needs a maintainer API-boundary choice because the request spans shared core contracts, a bundled QQ Bot plugin, an external WeCom plugin, and no bundled DingTalk implementation.

## What This Changes

Request outbound @mention/@all support for DingTalk, QQ Bot, and WeCom group messages from OpenClaw's message tool.

## Best Possible Solution

Define a portable outbound mentions contract or endorse plugin-local message-tool schema plus channelData, then implement per-channel payload mapping and tests in the owning plugins.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection confirms current main lacks the requested outbound mention metadata in the shared message tool and QQ Bot send path.

## Solution Assessment

Unclear: outbound mention support is a reasonable goal, but the proposed DingTalk-shaped shared fields are not proven to be the best API. A portable mentions object or plugin-owned schema/payload mapping should be chosen first.
