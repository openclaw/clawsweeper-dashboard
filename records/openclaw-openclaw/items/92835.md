## Summary

Keep open: current main and v2026.6.6 still lack outbound mention/@all parameters in the shared message tool and QQ Bot send path. The request is a valid additive channel feature, but the API boundary needs maintainer direction because platform-specific shared fields would cut across plugin-owned message-tool schema, QQ Bot bundled code, WeCom external plugin ownership, and absent DingTalk bundled source.

## What This Changes

The issue requests outbound @mention/@all support through the shared message tool for DingTalk, QQ Bot, and WeCom group sends.

## Best Possible Solution

Maintainers should choose the outbound mention contract first, then implement it in the owning channel plugins with focused schema/payload mapping and tests rather than baking one platform's field names into the shared context by default.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection shows current main and v2026.6.6 do not expose the requested mention metadata in the shared message tool or QQ Bot outbound payload path.

## Solution Assessment

Unclear: outbound mentions are useful, but the proposed platform-specific fields are not proven to be the best shared API. A portable mentions shape or plugin-owned message-tool schema plus prepareSendPayload mapping should be decided before implementation.
