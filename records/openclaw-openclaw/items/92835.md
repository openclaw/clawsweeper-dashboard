## Summary

Keep open: current main still lacks the requested outbound mention metadata path, and the remaining decision is whether this should be a portable message-tool contract or channel-owned schema/action behavior.

## What This Changes

Request native outbound @mention and @all support for DingTalk, QQBot, and WeCom group messages through the OpenClaw message tool or channel plugins.

## Best Possible Solution

Define the outbound mention contract first, preferably through channel/plugin-owned message-tool schema and send-path mapping, and add a shared SDK primitive only if maintainers decide mentions are portable across channels.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection confirms current main lacks the shared outbound mention metadata and the bundled QQBot sender does not pass native mention payloads.

## Solution Assessment

Unclear. The requested behavior is useful, but the proposed shared atMobiles/atUserIds/isAtAll fields are probably not the best final shape until maintainers decide whether outbound mentions are portable or plugin-owned.
