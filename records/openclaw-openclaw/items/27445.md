## Summary

Keep open. Current main still lacks a native `announceTarget` option for `sessions_spawn` or `agents.defaults.subagents`, and the related open issues cover complementary suppression or structured handoff behavior rather than superseding this parent-routing request.

## What This Changes

Request a per-spawn and config-level subagent completion announce routing option that can send completion payloads to the parent session instead of directly to the channel.

## Best Possible Solution

Add a backward-compatible native parent-routing option that defaults to current channel behavior, supports both per-spawn and `agents.defaults.subagents`, injects parent-targeted completions with `deliver: false`, and updates docs, generated config/help, changelog, and targeted tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source and docs inspection give high-confidence evidence that the requested public tool parameter, config key, and routing switch are absent on current main.

## Solution Assessment

Yes at the direction level. A per-spawn override plus config default is the narrow maintainable shape, but maintainers should settle naming and interaction with related suppression and structured handoff work before implementation.
