## Summary

Keep open. Current main and the latest release still lack the requested native `announceTarget` `sessions_spawn` and `agents.defaults.subagents` surface, and current thread-bound native session spawns still suppress the completion wake that this request is trying to make explicit.

## What This Changes

Request native sub-agent completion announce routing so callers can opt into parent-session handoff instead of direct channel delivery.

## Best Possible Solution

Define one narrow native completion-routing contract that preserves current channel delivery by default, keeps ACP `streamTo` separate, and covers thread-bound parent wake behavior without a broad orchestration framework.

## Reproduction Assessment

Yes for source-level reproduction. Current main lacks the requested tool/config surface, and the thread-bound native session path still records `expectsCompletionMessage:false` with tests preserving that behavior.

## Solution Assessment

Unclear. Parent-targeted completion routing appears useful, but the best shape needs maintainer narrowing because current main already has partial parent-handoff behavior and the requested config/tool API adds public surface.
