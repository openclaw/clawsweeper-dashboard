## Summary

Keep open. Current main already covers much of the original parent-mediated/review-first completion handoff, but it still lacks the requested native `announceTarget` tool/config contract and still suppresses terminal parent completion wake for directly delivered thread-bound session spawns.

## What This Changes

The issue requests an `announceTarget` option for `sessions_spawn` and `agents.defaults.subagents` so sub-agent completion announces can be routed into the parent session instead of directly to the channel.

## Best Possible Solution

Narrow the remaining contract to the thread-bound parent completion wake first, and add a public `announceTarget` mode only if maintainers explicitly want multiple supported completion routing modes.

## Reproduction Assessment

Yes. Current source has no native `announceTarget` parameter/default, and the thread-bound direct-delivery path sets `expectsCompletionMessage` false, so the remaining gap is source-reproducible without a live channel run.

## Solution Assessment

No for the original broad shape. Current main already parent-mediates ordinary completions, so maintainers should narrow whether the remaining thread-bound wake gap needs a new routing option or a canonical completion event.
