## Summary

Keep open. Current main and the latest release still lack a native `sessions_spawn`/`agents.defaults.subagents` `announceTarget` option; existing code has parent handoff mechanics, but the routing policy is still depth/thread/delivery based rather than an explicit backward-compatible parent-routing contract.

## What This Changes

Request to add `announceTarget: "channel" | "parent"` for sub-agent completion announces at both config-default and per-`sessions_spawn` levels.

## Best Possible Solution

Add a backward-compatible native parent-routing option for subagent completion, default it to current channel behavior, and cover direct, active/yielded, nested, thread-bound, and channel fallback paths in docs and tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source and release-tag inspection give high confidence that the requested native tool parameter, config default, and explicit force-parent routing override are absent.

## Solution Assessment

Yes at the direction level: a per-spawn override plus config default is the narrow maintainable shape for native orchestration. Maintainers still need to settle the public API and how it composes with suppression, structured handoff actions, `sessions_yield`, and thread inheritance.
