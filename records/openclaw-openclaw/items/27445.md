## Summary

Keep open. Current main still lacks native `announceTarget` support for `sessions_spawn` or `agents.defaults.subagents`, and the existing implementation preserves external channel delivery for top-level requester sessions while only nested subagent requesters get internal `deliver=false` injection.

## What This Changes

Add a config default and per-spawn `announceTarget` option so native subagent completion announces can route to the parent session instead of the channel.

## Best Possible Solution

Implement a backward-compatible native subagent announce-routing option with `channel` as the default and `parent` as an explicit internal handoff mode, while keeping ACP `streamTo` and broader callback/thread work separate.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence evidence that current main has no native `announceTarget` parameter, config key, or delivery switch.

## Solution Assessment

Yes. The requested config default plus per-spawn override is the narrow maintainable shape already outlined by maintainer context, provided it preserves channel delivery by default and keeps ACP `streamTo` separate.

## Repair Work Prompt

Implement #27445 as a focused backward-compatible native subagent announce-routing feature. Add `announceTarget: "channel" | "parent"` to native `sessions_spawn` and `agents.defaults.subagents`, defaulting to current channel behavior. For `announceTarget: "parent"`, route the completion announce into the requester session as an internal parent-session handoff with `deliver: false` and no direct channel announce, so the parent agent can decide what reaches the user. Keep ACP `streamTo` behavior separate, preserve all existing defaults, and do not fold in the broader announce suppression or structured callback work from related issues. Update config schema/types/generated config docs, `docs/tools/subagents.md`, changelog if user-facing, and targeted tests covering schema, config default resolution, per-spawn override, top-level parent routing, default channel routing, and ACP separation.
