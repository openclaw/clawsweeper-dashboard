## Summary

Keep open. Current main still lacks native parent-targeted announce routing on both the native `sessions_spawn` tool and `agents.defaults.subagents`, and related open items cover complementary suppression or structured-handoff behavior rather than replacing this request.

## What This Changes

Request adds a backward-compatible `announceTarget: "channel" | "parent"` option for sub-agent completion announces, with both a per-spawn override and a config default.

## Best Possible Solution

Add a focused native parent-routing option that defaults to current channel behavior, supports both per-spawn and `agents.defaults.subagents`, injects parent-targeted completions with `deliver: false`, and updates docs, generated config/help, changelog, and targeted tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence evidence that the requested public tool parameter, config key, and routing switch are absent on current main.

## Solution Assessment

Yes at the direction level. The per-spawn override plus config default is the narrow maintainable shape, but maintainers should settle naming and interaction with related suppression and structured handoff work before implementation.
