## Summary

Keep open. Current main still lacks native `announceTarget` support for native `sessions_spawn` and `agents.defaults.subagents`, while completion delivery remains depth-based: top-level requesters still get external channel delivery and only nested requester subagents get session-internal `deliver: false` injection. The issue has cross-channel corroboration and workable-but-fragile `sessions_send` callback workarounds, plus an explicit maintainer comment already keeping the issue open for a focused implementation.

## What This Changes

Request a config default and per-spawn override that can route native subagent completion announces to the parent session instead of directly to the requester chat channel.

## Best Possible Solution

Add a backward-compatible native `announceTarget: "channel" | "parent"` enum to `sessions_spawn` and `agents.defaults.subagents`, default it to `channel`, and have `parent` inject the completion handoff into the requester session with no direct channel announce while retaining existing access checks, idempotency, routing metadata, docs, config schema, changelog, and focused regression coverage.

## Reproduction Assessment

Yes for the missing current-main behavior: schema/config/source inspection shows there is no `announceTarget` surface, and the delivery code still routes top-level requester completions to external delivery while reserving `deliver: false` for nested subagent requesters. I did not run runtime tests because this review is read-only.

## Solution Assessment

Yes. The requested explicit enum is the narrowest maintainable direction because it preserves today’s default, avoids overloading ACP-only `streamTo`, and gives orchestrators a native session-trigger path without requiring prompt-level `sessions_send` boilerplate.
