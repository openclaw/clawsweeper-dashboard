## Summary

Keep open. Current main still lacks native `announceTarget` support on native `sessions_spawn` and `agents.defaults.subagents`, and completion delivery remains depth-based rather than an explicit per-spawn/config routing choice.

## What This Changes

Request adds a native `announceTarget` default and per-spawn override so subagent completion handoffs can route to the parent session instead of directly to the channel.

## Best Possible Solution

Add a backward-compatible `announceTarget: "channel" | "parent"` enum to native spawn and subagent defaults, default to `channel`, and make `parent` inject completion into the requester session without direct channel delivery.

## Reproduction Assessment

Yes for the missing current-main feature surface: source inspection shows no `announceTarget` in the native tool schema or config defaults, and delivery still selects top-level external channel delivery by requester depth. This is a feature request rather than a broken existing contract.

## Solution Assessment

Yes. The explicit enum is the narrowest maintainable direction because it preserves existing channel behavior by default, keeps ACP-only `streamTo` separate, and removes prompt-level `sessions_send` boilerplate for orchestrators.
