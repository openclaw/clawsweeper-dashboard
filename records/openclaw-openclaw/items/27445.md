## Summary

Current main now solves the central orchestration problem: native child completions are injected into the requester agent with delivery disabled, active requesters are woken or steered, and the parent owns any visible reply. The exact `announceTarget` option was not added because the implementation adopted the cleaner parent-owned behavior as the canonical path.

## What This Changes

The issue requests config-level and per-spawn routing of sub-agent completion announcements to the parent session instead of directly to the channel.

## Best Possible Solution

Keep the shipped requester-owned completion handoff as the single canonical behavior and avoid adding a redundant routing config or per-spawn option.

## Reproduction Assessment

## Solution Assessment

No; the shipped canonical parent-handoff model solves the observable problem with less public configuration and fewer competing routing paths than the proposed option.
