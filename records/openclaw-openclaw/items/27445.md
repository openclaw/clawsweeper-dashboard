## Summary

Keep open. Current main still lacks native `announceTarget` routing for `sessions_spawn` or `agents.defaults.subagents`, and recent maintainer plus user comments confirm the parent-session wakeup gap is still a valid feature request rather than stale cleanup.

## What This Changes

Request adds a backward-compatible `announceTarget: "channel" | "parent"` default and per-spawn override so sub-agent completion announces can wake the parent session instead of posting directly to the channel.

## Best Possible Solution

Implement an explicit parent-routing option for native sub-agent completion announces, preserving channel delivery as the default and keeping ACP `streamTo` separate.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, and tests give high-confidence evidence that current main lacks the requested native parameter, config key, and delivery switch.

## Solution Assessment

Likely yes for the parent-routing slice: a per-spawn override plus config default is narrow and backward-compatible. Maintainers should still settle naming and interaction with suppression-related issues before implementation.
