## Summary

Keep open. Current main still has no native `announceTarget` parameter or config default, and top-level requester delivery is still controlled by depth-based delivery logic rather than an explicit parent-routing option.

## What This Changes

Add a config default and per-spawn `announceTarget` option so native sub-agent completions can wake the parent session without directly posting to the channel.

## Best Possible Solution

Implement a backward-compatible native `announceTarget: "channel" | "parent"` for `sessions_spawn` and `agents.defaults.subagents`, defaulting to `channel` and using `deliver: false` parent-session injection only when explicitly requested.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence evidence that current main lacks the requested native parameter, config key, and delivery switch.

## Solution Assessment

Yes. The requested per-spawn override plus config default is the narrow maintainable shape because it preserves current channel delivery by default and avoids overloading ACP-only `streamTo`.
