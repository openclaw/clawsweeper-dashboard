## Summary

Keep open. Current main still lacks native parent-target announce routing for native `sessions_spawn`, and the live discussion plus related issues show this remains a distinct feature request rather than an implemented or superseded cleanup item.

## What This Changes

Add `announceTarget: "channel" | "parent"` as a per-spawn and default config option for sub-agent completion announce routing.

## Best Possible Solution

Add a backward-compatible parent-routing option that defaults to current channel behavior, supports both `sessions_spawn` and `agents.defaults.subagents`, injects parent-targeted completions with `deliver: false`, and updates schema, docs, generated config/help, changelog, and targeted tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source, docs, and tests give high-confidence evidence that the requested public tool parameter, config key, and routing switch are absent on current main.

## Solution Assessment

Yes at the direction level: a per-spawn override plus config default is the narrow maintainable shape for native orchestration. Maintainers should settle the public API name and its interaction with related suppression and structured handoff work before implementation.
