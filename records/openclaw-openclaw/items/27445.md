## Summary

Current main still lacks native `announceTarget` routing for native `sessions_spawn` completions. The tool schema and tests keep `streamTo` ACP-only, the native spawn path has no completion-routing parameter, config defaults reject the proposed key, docs still describe top-level requester delivery as external channel delivery, and delivery logic remains depth-based rather than option-based.

## What This Changes

Allow native sub-agent completion announcements to route into the parent/requester session instead of directly to the channel, with a config default and per-spawn override.

## Best Possible Solution

Implement a focused, backward-compatible native subagent announce-routing feature: add `announceTarget: "channel" | "parent"` to `sessions_spawn` and `agents.defaults.subagents`, default to current channel delivery, route `parent` completions into the requester session with `deliver: false` and no direct channel announce, keep ACP `streamTo` separate, and update schema, generated config artifacts, docs, changelog, and targeted tests.
