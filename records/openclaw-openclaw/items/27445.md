## Summary

Keep open: current main and v2026.6.5 have parent-session completion-event improvements, but they still do not expose the requested native `announceTarget` tool/config contract for native `sessions_spawn`. The remaining work is a public API/config decision, not a stale or already-implemented cleanup item.

## What This Changes

The issue requests a native `announceTarget` default and per-spawn option so sub-agent completion announces can wake the parent session instead of posting directly to the channel.

## Best Possible Solution

Define one canonical, backward-compatible native subagent completion-routing contract that coexists with `sessions_yield`; if maintainers approve the config/tool surface, update schema, types, docs, and targeted spawn/announce tests together.

## Reproduction Assessment

Yes. Source inspection can reproduce the missing API surface: native `sessions_spawn` and `agents.defaults.subagents` still lack `announceTarget`, while related docs/tests show only adjacent completion-event behavior.

## Solution Assessment

Unclear. The proposed `announceTarget` shape is plausible, but current main already has `sessions_yield` and parent-agent completion events, so maintainers should choose whether to add a config knob, a delivery-mode option, or fold this into the broader handoff/suppression design.
