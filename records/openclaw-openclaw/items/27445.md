## Summary

Keep this open: current main has added parent-review completion handoff paths, but the requested native `announceTarget` config/tool contract is still absent and thread-bound session spawns can still disable the terminal parent completion wake.

## What This Changes

The issue requests native sub-agent completion announce routing so callers can choose channel delivery or parent-session wakeup.

## Best Possible Solution

Define one backward-compatible native completion-routing contract, then align `sessions_spawn`, `agents.defaults.subagents`, thread-bound session behavior, docs, and focused tests to that contract.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction for the missing requested surface: current main has no native `announceTarget` parameter or subagent config default, and thread-bound session spawns can register with completion announce disabled. I did not run a live channel scenario in this read-only review.

## Solution Assessment

No, this should not be closed or auto-fixed as-is; the best path is maintainer narrowing of the public completion-routing contract before changing config and tool semantics. Current main is a partial implementation, not the final requested API shape.
