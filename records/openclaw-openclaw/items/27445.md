## Summary

Keep open. Current main still lacks a native `announceTarget` parameter or `agents.defaults.subagents` default, and completion routing still depends on existing depth/thread delivery behavior rather than an explicit parent-routing option.

## What This Changes

Add a per-spawn and config-level option for native sub-agent completion announcements to route into the parent session instead of directly to the external channel.

## Best Possible Solution

Implement a backward-compatible `announceTarget: "channel" | "parent"` for `sessions_spawn` and `agents.defaults.subagents`, defaulting to current channel behavior and using parent-session `deliver: false` injection only when explicitly requested.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence evidence that current main lacks the requested native parameter, config key, and delivery switch.

## Solution Assessment

Yes. The requested per-spawn override plus config default is the narrow maintainable shape because it preserves current channel delivery by default and keeps ACP-only `streamTo` separate.

Add `announceTarget: "channel" | "parent"` to native `sessions_spawn` and `agents.defaults.subagents`; default to `channel`; when set to `parent`, inject the completion payload into the requester/parent session with `deliver: false`, trigger exactly one parent turn, and suppress direct final channel announce.
