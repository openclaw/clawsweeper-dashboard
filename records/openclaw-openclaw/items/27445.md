## Summary

Keep open. Current main still lacks the requested native `announceTarget` tool/config contract; existing code only has ACP-only `streamTo` parent streaming and depth-based native announce delivery, so the central user problem remains.

## What This Changes

The issue requests a `sessions_spawn` override and `agents.defaults.subagents` default that route native sub-agent completion announces into the parent session instead of directly to the channel.

## Best Possible Solution

Keep this open for a maintainer-sponsored, backward-compatible native announce-routing design that preserves current channel delivery by default and clearly separates ACP `streamTo` from native `announceTarget`.

## Reproduction Assessment

Yes for the missing contract: current main has no `announceTarget` field in `sessions_spawn` or `agents.defaults.subagents`, and docs still describe depth-based delivery. I did not run a live Discord, Feishu, or Telegram orchestration scenario in this read-only review.

## Solution Assessment

Unclear as an implementation direction because the requested API is plausible but adds tool/config surface that maintainers have already marked for product review. A safe solution should be default-compatible and explicitly test native parent injection, thread-bound sessions, and ACP separation.
