## Summary

Keep open. Current main still lacks a native `announceTarget` parameter/default for subagent completion routing, and the recent issue discussion plus labels show this remains a public tool/config feature needing maintainer product review.

## What This Changes

Request a backward-compatible `announceTarget: "channel" | "parent"` option for `sessions_spawn` and `agents.defaults.subagents` so subagent completion announces can wake the parent session instead of posting directly to the channel.

## Best Possible Solution

Add a backward-compatible native parent-routing option for subagents, default it to current channel delivery, keep ACP `streamTo` separate, and update schema, generated config docs, user docs, changelog, and targeted announce/thread tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection gives high-confidence evidence that the requested native tool parameter, config default, and parent-wake routing override are still absent on current main.

## Solution Assessment

Yes at the direction level: a per-spawn override plus config default is the narrow maintainable shape for native orchestration. Maintainers still need to settle how it composes with suppression, structured handoff actions, and thread inheritance.
