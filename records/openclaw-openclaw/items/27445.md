## Summary

Keep open. Current main has moved much of subagent completion delivery toward requester/parent handoff, but the requested native `announceTarget` tool/config surface is still absent and the thread-bound `mode="session"` path still disables completion wake when the child run is delivered directly into its bound thread.

## What This Changes

Add configurable and per-spawn subagent completion announce routing so completion output can wake the parent session instead of going straight through channel delivery.

## Best Possible Solution

Define one narrow native completion-routing contract that preserves current defaults, gives thread-bound/session orchestrators a parent wake when requested, and avoids adding a broad config knob unless maintainers decide the existing handoff behavior cannot cover it.

## Reproduction Assessment

For a feature request, there is no failing bug repro required; source inspection does confirm the requested API/config surface is absent and the thread-bound session no-wake case is preserved by tests.

## Solution Assessment

Unclear. The desired parent-review behavior is partly present on current main, so the best fix is likely a narrowed completion-routing contract rather than blindly adding the originally proposed `announceTarget` shape everywhere.
