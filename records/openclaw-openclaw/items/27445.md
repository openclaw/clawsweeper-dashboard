## Label Justifications

- `P2`: This is a normal-priority public tool/config enhancement for multi-agent orchestration with real but bounded workflow impact.
- `impact:message-loss`: The requested routing controls whether subagent completion messages are delivered to the channel, withheld from the channel, or routed into the parent session.
- `impact:session-state`: The core problem is that parent sessions do not receive completion state as an explicit wake event for orchestration.

## Summary

Keep open. Current main still lacks a native `sessions_spawn` `announceTarget` parameter, an `agents.defaults.subagents` default, and an option-based parent-session completion route; related open issues cover suppression, structured handoff, and parent-send variants rather than superseding this request.

## What This Changes

Add a backward-compatible `announceTarget: "channel" | "parent"` option for sub-agent completion announces, with a config default and per-spawn override.

## Best Possible Solution

Add a backward-compatible native parent-routing option for subagents, default it to current channel delivery, keep ACP `streamTo` separate, and cover normal, yielded, nested, dormant, and thread-bound completion paths in docs and tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection gives high confidence that the requested native tool parameter, config default, and parent-routing override are absent on current main.

## Solution Assessment

Yes at the direction level: a per-spawn override plus config default is the narrow maintainable shape for native orchestration. Maintainers still need to settle the exact public API and how it composes with suppression, structured handoff actions, `sessions_yield`, and thread inheritance.
