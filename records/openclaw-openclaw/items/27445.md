## Summary

Keep open. Current main and v2026.5.22 still lack a native `sessions_spawn` or `agents.defaults.subagents` `announceTarget` option; existing parent handoff mechanics remain delivery-policy driven rather than an explicit parent-routing contract, and the related open issues cover complementary suppression, structured callback, or parent-send behavior rather than superseding this request.

## What This Changes

The issue requests a new native `announceTarget` option for `sessions_spawn` and `agents.defaults.subagents` so sub-agent completion announces can route to the parent session instead of directly to the channel.

## Best Possible Solution

Add a backward-compatible native parent-routing option for subagent completion, default it to current channel behavior, and cover direct, active/yielded, nested, thread-bound, and fallback delivery paths in docs and tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source and release-tag inspection give high confidence that the requested native tool parameter, config default, and explicit force-parent routing override are absent.

## Solution Assessment

Yes at the direction level: a per-spawn override plus config default is the narrow maintainable shape for native orchestration. Maintainers still need to settle the public API and how it composes with suppression, structured handoff actions, `sessions_yield`, and thread inheritance.
