## Summary

Keep open. Current main has improved sub-agent completion handoff, but the requested public `announceTarget` tool/config contract is still absent, and thread-bound native session spawns still disable completion announce registration.

## What This Changes

Request adds a config default and per-spawn option to route native sub-agent completion announces to the parent session instead of the channel.

## Best Possible Solution

Keep this as a maintainer-sponsored core sub-agent routing design: preserve current defaults, decide the public contract, and include explicit thread-bound session completion coverage.

## Reproduction Assessment

Yes. Source inspection is a high-confidence reproduction path: current main has no native `sessions_spawn.announceTarget` or `agents.defaults.subagents.announceTarget`, and thread-bound session spawns still register without completion announce.

## Solution Assessment

Unclear as a final product choice. The requested option is plausible, but maintainers should decide whether `announceTarget` or a narrower completion-delivery policy is the durable contract.
