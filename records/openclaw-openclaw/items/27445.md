## Summary

Keep open. Current main has improved default sub-agent completion handoff into the requester agent, but the requested public `announceTarget` tool/config contract is still absent and thread-bound `mode: "session"` spawns still suppress completion wakes when the child run is delivered directly.

## What This Changes

Request native `announceTarget` routing for sub-agent completion announcements so completions can wake the parent session instead of always announcing directly to the channel.

## Best Possible Solution

Have maintainers choose one backward-compatible subagent completion policy contract that covers parent routing, channel delivery/suppression, and thread-bound session wakes, then implement it across tool schema, config schema/types, docs, and targeted tests.

## Reproduction Assessment

Yes for the missing contract: current main exposes no `announceTarget` in `sessions_spawn` or `agents.defaults.subagents`, and thread-bound session spawns still record no completion announce when child direct delivery is bound. This is source-proven rather than a failing runtime bug repro.

## Solution Assessment

Unclear. A native option is plausible, but the best fix needs maintainer product direction because current main already changed the default handoff behavior and related open issues ask for suppression and structured callback semantics too.
