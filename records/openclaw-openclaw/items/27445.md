## Summary

Keep open. Current main improved ordinary sub-agent completion handoff, but the requested public `announceTarget` tool/config contract is still absent and thread-bound session spawns still suppress completion announce registration, so the remaining product request is not implemented.

## What This Changes

The issue requests native `announceTarget` routing for sub-agent completion announces so parent sessions can receive completion events instead of direct channel announcements.

## Best Possible Solution

Keep this as a maintainer-sponsored core sub-agent routing design: preserve current defaults, decide whether the public contract should be `announceTarget` or a narrower completion-delivery policy, and include explicit thread-bound session completion coverage.

## Reproduction Assessment

Yes, source inspection is a high-confidence reproduction path for the remaining gap: current main has no `sessions_spawn.announceTarget` parameter or `agents.defaults.subagents.announceTarget`, and thread-bound session spawns still register `expectsCompletionMessage: false`. I did not run a live transport scenario because this review is read-only and the source/test contract is explicit.

## Solution Assessment

Unclear as a final product choice. The requested option is plausible, but the best maintainable fix may be a narrower completion-delivery policy that makes current parent handoff fully cover thread-bound sessions without adding avoidable config surface.
