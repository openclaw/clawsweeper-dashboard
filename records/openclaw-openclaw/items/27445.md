## Summary

Keep open. Current main still lacks native `announceTarget` routing for native `sessions_spawn` and `agents.defaults.subagents`, and the related open issues cover complementary announce suppression, thread inheritance, and routing bugs rather than superseding this feature request.

## What This Changes

Request to add a backward-compatible `announceTarget` option so sub-agent completion announces can route to the parent session instead of directly to the channel.

## Best Possible Solution

Add a focused, backward-compatible native parent-routing option that defaults to current channel behavior, supports both per-spawn and `agents.defaults.subagents`, injects parent-targeted completions with `deliver: false`, and updates docs, generated config/help, changelog, and targeted tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection of current main gives high-confidence evidence that the requested public tool parameter, config key, and routing switch are absent.

## Solution Assessment

Yes at the direction level. The per-spawn override plus config default is the narrow maintainable shape, but maintainers still need to settle naming and interaction with related suppression and structured handoff work.
