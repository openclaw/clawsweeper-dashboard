## Label Justifications

- `P2`: This is a normal-priority public tool/config enhancement for multi-agent orchestration with limited but real workflow impact.
- `impact:message-loss`: The requested routing affects whether subagent completion messages are delivered to the channel, suppressed from the channel, or routed to the parent session.
- `impact:session-state`: The core problem is that parent sessions do not receive completion state as a wake event for orchestration.

## Summary

Keep open. Current main still lacks the requested native `announceTarget` tool parameter and `agents.defaults.subagents` default, and recent issue discussion shows this remains an active public routing/product decision rather than stale cleanup.

## What This Changes

Request to add a backward-compatible `announceTarget: "channel" | "parent"` option for native subagent completion routing on `sessions_spawn` and `agents.defaults.subagents`.

## Best Possible Solution

Add a backward-compatible native parent-routing option for subagents, default it to current channel delivery, keep ACP `streamTo` separate, and cover normal, yielded, and thread-bound completion paths in tests and docs.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection gives high-confidence evidence that the requested native tool parameter, config default, and parent-wake routing override are still absent on current main.

## Solution Assessment

Yes at the direction level: a per-spawn override plus config default is the narrow maintainable shape for native orchestration. Maintainers still need to settle the exact public API and how it composes with suppression, structured handoff actions, and thread inheritance.
