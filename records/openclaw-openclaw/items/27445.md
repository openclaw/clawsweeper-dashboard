## Summary

Keep open: current main and latest release still lack native `sessions_spawn` or `agents.defaults.subagents` `announceTarget` routing, and the current delivery path remains requester-kind/thread-policy driven rather than an explicit parent-routing contract. Related open issues cover complementary suppression, structured callback, and broader handoff work, but none supersedes this specific feature request.

## What This Changes

Request native sub-agent completion announce routing via a config default and per-spawn `announceTarget` override so completions can wake the parent session instead of posting directly to the channel.

## Best Possible Solution

Add a backward-compatible native parent-routing option for subagent completion, default it to current channel behavior, and cover schema/types, generated config docs, user docs, direct/nested/thread-bound delivery paths, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction, but yes for a source-level feature gap: current main lacks the requested tool parameter, config default, and explicit force-parent routing branch. I did not run a live Discord/Feishu/Telegram orchestration scenario in this read-only review.

## Solution Assessment

Yes at the direction level: a per-spawn override plus config default is the narrow maintainable shape for native orchestration. The exact API, precedence, and interaction with suppression/structured handoff work still need maintainer product review.
