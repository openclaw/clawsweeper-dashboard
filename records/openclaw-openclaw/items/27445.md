## Repair Work Prompt

Implement a focused native subagent announce-routing feature for https://github.com/openclaw/openclaw/issues/27445. Add a documented `announceTarget: "channel" | "parent"` option to `sessions_spawn` and `agents.defaults.subagents`, preserving current default behavior unless the config/spawn override says otherwise. For `parent`, route completion into the requester session as internal/user-message-style completion context with `deliver: false` and no direct channel announce; let the parent agent decide whether and what to send. Keep ACP `streamTo` separate and ACP-only. Update strict config schema/types, generated config docs, `docs/tools/subagents.md`, changelog, and focused tests. Do not fold in broader suppression policy from #8299 or structured callback/thread inheritance from #33478 beyond what is necessary for this routing option.
