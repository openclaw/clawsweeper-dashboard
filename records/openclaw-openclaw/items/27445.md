## Summary

Keep open. Current main still lacks the requested native `announceTarget` parameter and `agents.defaults.subagents` default, and thread-bound native `mode: "session"` spawns still suppress completion announces rather than offering a parent-wake override.

## What This Changes

The issue requests a backward-compatible `announceTarget: "channel" | "parent"` routing option for native sub-agent completion announces, with both a config default and per-spawn override.

## Best Possible Solution

Keep this issue open as the canonical native completion-routing contract decision, then implement one backward-compatible design that preserves current channel behavior by default and covers parent routing plus thread-bound parent wakes.

## Reproduction Assessment

Yes for source reproduction: current main lacks `announceTarget` in both `sessions_spawn` and `agents.defaults.subagents`, and thread-bound session spawns still register completion announces as not required. I did not run a live channel reproduction because the requested public contract is directly absent in source.

## Solution Assessment

Unclear. `announceTarget: "parent"` is a plausible narrow contract, but maintainers should first decide how it coexists with current requester-agent handoff, announce suppression policy, and ACP-only `streamTo`.
