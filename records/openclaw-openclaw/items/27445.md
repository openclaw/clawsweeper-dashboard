## Label Justifications

- `P2`: The request is a normal-priority multi-agent orchestration improvement with real workflow impact but it needs product/API judgment before implementation.
- `impact:session-state`: The requested behavior changes whether sub-agent completion state wakes and enters the parent session for orchestration.

## Summary

Keep open. Current main has improved the default sub-agent completion flow so completions are handed to the requester agent, but it still does not expose the requested native `announceTarget` tool/config contract and still suppresses completion announces for thread-bound `mode: "session"` spawns with direct child delivery.

## What This Changes

The issue requests a native `announceTarget` option for sub-agent completion announcements so callers can route completion output into the parent session instead of directly to the channel.

## Best Possible Solution

Keep this issue open for a maintainer decision on one native completion-routing contract that covers explicit parent routing, thread-bound session completion wakes, and overlap with announce suppression without adding competing knobs.

## Reproduction Assessment

Yes for source reproduction: current main lacks `announceTarget` in the `sessions_spawn` schema and `agents.defaults.subagents`, and thread-bound session spawns still register completion announces as not required. I did not run a live channel reproduction because this is a read-only issue review and the source gap is direct.

## Solution Assessment

Unclear. `announceTarget: "parent"` is a plausible contract, but current main already has parent/requester-agent completion handoff by default, so maintainers should decide whether the remaining fix is an explicit routing option, a broader announce policy, or a narrower thread-bound wake repair.
