## Label Justifications

- `P2`: This is a normal-priority public tool/config enhancement for multi-agent orchestration with limited but real workflow impact.
- `impact:message-loss`: The requested routing affects whether subagent completion messages are delivered to the channel, suppressed from the channel, or routed to the parent session.
- `impact:session-state`: The core problem is that parent sessions do not receive completion state as an explicit wake event for orchestration.

## Summary

Keep open. Current main still lacks a native `announceTarget` parameter, an `agents.defaults.subagents` default, or an option-based parent-routing path for native subagent completion announces; related open issues cover adjacent suppression and handoff hardening but do not supersede this request.

## What This Changes

Request adding `announceTarget` to `sessions_spawn` and `agents.defaults.subagents` so subagent completion can route to the parent session instead of directly announcing to the channel.

## Best Possible Solution

Add a backward-compatible native parent-routing option for subagents, default it to current channel delivery, keep ACP `streamTo` separate, and cover normal, yielded, and thread-bound completion paths in tests and docs.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection gives high-confidence evidence that the requested native tool parameter, config default, and parent-routing override are absent on current main.

## Solution Assessment

Yes at the direction level: a per-spawn override plus config default is the narrow maintainable shape for native orchestration. Maintainers still need to settle the exact public API and how it composes with suppression, structured handoff actions, and thread inheritance.
