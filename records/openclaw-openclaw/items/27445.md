## Summary

Keep open. Current main still lacks a native `announceTarget` parameter/default for subagent completion routing, and the requested behavior is a public tool/config feature with maintainer/product-review labels rather than an already-fixed bug.

## What This Changes

Add a `sessions_spawn` and `agents.defaults.subagents` option to route subagent completion announces to the parent session instead of directly to the channel.

## Best Possible Solution

Add a backward-compatible native parent-routing option for subagents, default it to current channel delivery, keep ACP `streamTo` separate, and update schema, generated config docs, user docs, changelog, and targeted announce/thread tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection gives high-confidence evidence that the requested native tool parameter, config default, and thread-bound parent-wake override are still absent on current main.

## Solution Assessment

Yes at the direction level: a per-spawn override plus config default is the narrow maintainable shape for native orchestration. Maintainers still need to settle the public API and how it composes with related suppression and structured handoff requests.
