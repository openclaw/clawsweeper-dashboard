## Summary

Keep open. Current main has improved sub-agent completion handoff, but source still lacks the requested public `announceTarget` tool/config contract and thread-bound native session spawns can still skip completion announce registration, so this remains a maintainer product/config decision.

## What This Changes

The issue requests native `announceTarget` routing for sub-agent completion announces so completions can wake the parent session instead of posting directly to the channel.

## Best Possible Solution

Keep this as a maintainer-sponsored sub-agent routing design: decide the public contract, preserve current default behavior, and require explicit coverage for thread-bound session completion wakeups before implementation.

## Reproduction Assessment

Yes. Source inspection is a high-confidence path: current main has no native `sessions_spawn.announceTarget` or `agents.defaults.subagents.announceTarget`, and thread-bound session spawns can still register without a completion announce wake.

## Solution Assessment

Unclear as a final product choice. The requested option is plausible, but current main has already moved toward requester-session completion handoff, so maintainers should decide whether `announceTarget` or a narrower completion-delivery policy is the durable contract.
