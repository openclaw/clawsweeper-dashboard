Keep open. Current main still lacks a native `announceTarget` option for `sessions_spawn` or `agents.defaults.subagents`, and the completion delivery path still chooses channel delivery versus session-only injection from existing requester-depth/thread behavior rather than an explicit parent-routing option.

Request add `announceTarget: "channel" | "parent"` as a sub-agent completion routing default and per-spawn override.

Implement a backward-compatible native parent-routing option that defaults to current channel behavior and keeps ACP `streamTo` and announce-suppression policy separate.

## Repair Work Prompt

Implement a focused, backward-compatible native subagent announce-routing feature for https://github.com/openclaw/openclaw/issues/27445. Add `announceTarget: "channel" | "parent"` to native `sessions_spawn` and `agents.defaults.subagents`, defaulting to current `channel` behavior. A per-spawn value should override the config default. For `parent`, route the completion payload into the requester/parent session as an internal user-triggering message with `deliver: false` and no direct completion announce to the external channel; preserve thread-bound child active-run delivery and wake the parent exactly once on completion. Keep ACP `streamTo` separate and still ACP-only. Do not implement general suppression, per-channel deny lists, or structured `sessions_send` callback actions in this PR. Update schema/types/generated config docs/user docs/changelog and add focused regression coverage for default channel behavior, parent routing, and thread-bound `mode: "session"` parent wake behavior.
