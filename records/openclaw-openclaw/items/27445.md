## Summary

Keep open. Current main still lacks native `announceTarget` support in `sessions_spawn` or `agents.defaults.subagents`, and completion delivery is still selected from requester depth rather than a per-spawn/config routing option.

## What This Changes

Request to add a backward-compatible `announceTarget: "channel" | "parent"` default and per-spawn override for native sub-agent completion announces.

## Best Possible Solution

Add the native option through tool schema, config schema/types, spawn metadata, and announce delivery so `channel` remains the default and `parent` injects the completion into the requester session with `deliver: false`, with docs, generated config docs, changelog, and regression tests updated.

## Solution Assessment

Yes. The requested config default plus per-spawn override is the narrow maintainable shape because it reuses the existing announce delivery decision point, preserves channel delivery by default, and keeps ACP `streamTo` separate.
