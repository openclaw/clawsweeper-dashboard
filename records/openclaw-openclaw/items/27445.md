## Label Justifications

- `P2`: This is a normal-priority public tool/config enhancement for multi-agent orchestration with real but bounded workflow impact.
- `impact:message-loss`: The requested routing controls whether subagent completion messages go directly to the channel, are withheld, or are routed into the parent session.
- `impact:session-state`: The central problem is that parent sessions do not receive subagent completion state as an explicit orchestration wake event in the requested mode.

## Summary

Keep open. Current main still lacks a native `announceTarget` tool parameter, an `agents.defaults.subagents` default, and an option-based way to force subagent completion into the parent session; related open issues cover suppression, structured callbacks, or streaming rather than superseding this request.

## What This Changes

Request adds a native `announceTarget` per-spawn option and `agents.defaults.subagents` default to route subagent completion announces to the parent session instead of the channel.

## Best Possible Solution

Add a backward-compatible native parent-routing option for subagent completion, default it to current channel behavior, and cover direct, group/channel, nested, active, dormant, yielded, and thread-bound completion paths in docs and tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection gives high confidence that the requested native tool parameter, config default, and force-parent routing override are absent on current main and the latest release.

## Solution Assessment

Yes at the direction level: a per-spawn override plus config default is the narrow maintainable shape for native orchestration. Maintainers still need to settle the public API and how it composes with suppression, structured handoff actions, `sessions_yield`, and thread inheritance.
