## Summary

Keep open. Current main has newer parent-review plumbing for some completion paths, but it still does not expose the requested native `announceTarget` tool/config API, and thread-bound native session spawns can still suppress the terminal parent wake entirely.

## What This Changes

The issue requests a backward-compatible `announceTarget: "channel" | "parent"` option for native sub-agent completion announce routing at both config-default and per-spawn levels.

## Best Possible Solution

Add a focused parent-routing option that defaults to current channel behavior, supports native `sessions_spawn` and `agents.defaults.subagents`, injects parent-targeted completions with `deliver: false`, keeps ACP `streamTo` separate, and updates schema, generated config/help, docs, changelog, and targeted tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection on current main gives high-confidence evidence that the requested public tool parameter, config default, and thread-bound parent-wake override are absent.

## Solution Assessment

Yes at the direction level: a per-spawn override plus config default is the narrow maintainable shape for native orchestration. Maintainers should settle the public API and how it composes with related suppression and structured handoff requests before implementation.
