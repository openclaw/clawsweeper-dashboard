## Summary

Keep open. Current main still lacks the requested native `announceTarget` tool/config surface, and the remaining behavior overlaps public API and routing policy decisions already marked for maintainer review.

## What This Changes

Add a native `announceTarget` option so sub-agent completion announcements can route to the parent session instead of directly to the channel.

## Best Possible Solution

Add a focused backward-compatible parent-routing option for native subagents, default it to current channel behavior, keep ACP `streamTo` separate, and update schema, config docs, user docs, changelog, and targeted announce/thread tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection gives high-confidence evidence that the requested public tool parameter, config default, and thread-bound parent-wake override are still absent on current main.

## Solution Assessment

Yes at the direction level: a per-spawn override plus config default is the narrow maintainable shape for native orchestration. Maintainers still need to settle the public API and how it composes with related suppression and structured handoff requests.
