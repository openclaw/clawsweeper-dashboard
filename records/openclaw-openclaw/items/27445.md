## Summary

Keep open. Current main still lacks native `announceTarget` routing for `sessions_spawn` or `agents.defaults.subagents`, and the remaining work is a public tool/config feature rather than a stale or implemented cleanup item.

## What This Changes

The issue requests a new `announceTarget` option on `sessions_spawn` and `agents.defaults.subagents` so sub-agent completions can wake the parent session instead of posting directly to the requester channel.

## Best Possible Solution

Add a backward-compatible native parent-routing option that defaults to current channel behavior, supports both per-spawn and `agents.defaults.subagents` configuration, injects parent-targeted completions with `deliver: false`, and updates docs, generated config/help, and targeted tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection of current main gives high-confidence evidence that the requested public tool parameter, config key, and routing switch are absent.

## Solution Assessment

Yes at the direction level. The per-spawn override plus config default is the narrow maintainable shape, but maintainers still need to settle naming and interaction with related suppression and structured handoff work.
