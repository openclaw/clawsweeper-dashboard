## Summary

Keep open. Current main still lacks native parent-target announce routing for `sessions_spawn`, and the recent issue discussion plus adjacent open requests show this remains a distinct feature/config decision rather than implemented or superseded cleanup.

## What This Changes

Requests a configurable `announceTarget` for native `sessions_spawn` and sub-agent defaults so completion announcements can wake the parent session instead of posting directly to the channel.

## Best Possible Solution

Add a backward-compatible parent-routing option that defaults to current channel behavior, supports both `sessions_spawn` and `agents.defaults.subagents`, injects parent-targeted completions with `deliver: false`, and updates schema, docs, generated config/help, changelog, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source, docs, tests, and live issue context give high-confidence evidence that the requested public tool parameter, config key, and routing switch are absent on current main.

## Solution Assessment

Yes at the direction level: a per-spawn override plus config default is the narrow maintainable shape for native orchestration. Maintainers still need to settle the public API name and its interaction with adjacent suppression, ACP, and structured handoff work.
