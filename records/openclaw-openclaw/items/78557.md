## Summary

Keep open: the PR targets a real current-main doctor false positive, but the new suppression is too broad and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds a doctor-memory escape hatch, regression tests, and a changelog entry so `noteMemorySearchHealth` suppresses the missing-memory-plugin note when a non-default memory slot owner is selected.

## Best Possible Solution

Keep the suppression, but narrow it to a verified alternate memory slot owner and add coverage for unconfigured or unavailable alternate slots before merging.

## Reproduction Assessment

Yes, from source inspection: current main documents `memory-lancedb` as the active memory slot owner, but the doctor warning path still treats a null memory-host runtime probe as no active memory plugin unless the gateway probe is ready. I did not run live doctor output because this review was read-only.

## Solution Assessment

No. The direction is right, but the proposed guard is broader than the reported configured-plugin case because it treats an absent plugin entry as active; the safer fix should require explicit configured ownership or manifest/registry proof before suppressing the note.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Require a configured alternate slot before suppressing doctor warnings:** `src/commands/doctor-memory-search.ts:320`
  - body: This treats a slot-only alternate memory selection as healthy because `plugins.entries[memorySlot]?.enabled !== false` is true when the entry is missing. For `plugins.slots.memory = "memory-lancedb"` without valid plugin config, the plugin parser throws `memory config required` / `embedding config required`, `resolveActiveMemoryBackendConfig` still returns null, and doctor would now suppress the only direct missing-runtime note. Please require an explicit configured/enabled entry or registry-backed active owner before returning here, and keep coverage for the unconfigured alternate-slot case.
