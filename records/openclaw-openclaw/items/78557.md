Keep open: the bug is source-reproducible and the fix direction is plausible, but the guard is broader than the configured-plugin case and the external PR still lacks after-fix terminal proof.

## What This Changes

The PR adds an alternate-memory-slot escape hatch to doctor memory diagnostics, regression tests, and an Unreleased changelog entry.

## Best Possible Solution

Keep the alternate-plugin suppression, but require a verified active/configured alternate memory owner or registry-backed installed memory plugin, add coverage for configured LanceDB and unconfigured slot cases, then merge after terminal proof.

## Reproduction Assessment

Yes. Source inspection shows current main treats a null memory-host runtime probe as no active memory plugin, while the docs and LanceDB plugin code show `memory-lancedb` is an active memory slot owner that uses tools/hooks instead of that host-runtime contract.

## Solution Assessment

No. The suppression direction matches the reported bug, but this implementation should narrow the guard to an explicitly configured or registry-confirmed alternate memory owner before suppressing the diagnostic.
