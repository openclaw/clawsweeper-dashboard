## Summary

Keep open: the PR targets a valid doctor false-positive, but the current guard can suppress the warning for a slot-only or unconfigured alternate memory plugin, and the real behavior proof still contains placeholders instead of concrete before/fix/negative terminal evidence.

## What This Changes

Adds a doctor memory diagnostic guard that skips the missing-memory-plugin note when a non-default memory slot is selected, plus two regression tests and a changelog entry.

## Best Possible Solution

Require explicit configured/enabled or registry-confirmed alternate memory ownership before suppressing the note, keep the default memory-core missing-runtime warning, add a slot-only or invalid-config regression test, and merge only after concrete redacted doctor output proof.

## Reproduction Assessment

Yes, source-level: current main warns when `resolveActiveMemoryBackendConfig` returns null, while `memory-lancedb` is documented as an active slot plugin but does not register the memory-host runtime that this check expects. I did not run a live doctor session in this read-only review.

## Solution Assessment

No, not yet. Suppressing the warning for alternate memory slots is the right direction, but slot text alone is not enough to prove that an alternate memory plugin is configured and active.

## Review Findings
