## What This Changes

The PR adds an alternate-memory-slot guard to `noteMemorySearchHealth`, regression tests for alternate/default memory slot behavior, and an Unreleased changelog entry.

## Best Possible Solution

Keep the alternate-plugin suppression, but require an explicitly configured/enabled or registry-confirmed active alternate memory owner, preserve the warning for missing/invalid entries, and merge only after terminal proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main warns on a null memory-host runtime, while documented `memory-lancedb` slot ownership uses tools/hooks instead of that runtime. I did not run a live `openclaw doctor` setup in this read-only review.

## Solution Assessment

No. The suppression direction matches the reported bug, but the proposed guard is too broad because slot selection alone does not prove an alternate memory plugin is configured and active.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Require a configured alternate slot before suppressing warnings:** `src/commands/doctor-memory-search.ts:323`
  - body: This treats a slot-only alternate memory selection as healthy because `plugins.entries[memorySlot]?.enabled !== false` is true when the entry is missing. With `plugins.slots.memory = "memory-lancedb"` but no valid configured entry, the LanceDB plugin rejects the missing config and only registers a disabled warning service, so doctor can now hide the missing-runtime note for an unavailable memory plugin. Require explicit configured or registry-backed active ownership before returning here, and cover the unconfigured alternate-slot case.
  - confidence: 0.84
