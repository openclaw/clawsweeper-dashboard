## Summary

Keep open: the reported false doctor warning is valid, but this PR suppresses the note from slot text alone and can hide an unavailable or unconfigured alternate memory plugin; the real behavior proof is also still too placeholder-heavy for merge.

## What This Changes

The PR adds a doctor memory-search guard that suppresses the missing active memory plugin note for non-default memory slot owners, adds two tests, and adds a changelog entry.

## Best Possible Solution

Require explicit configured/enabled or registry-confirmed alternate memory ownership before suppressing the note, or land a plugin-owned `memory-lancedb` runtime adapter; cover both the documented configured entry and the slot-only invalid case.

## Reproduction Assessment

Yes, at source level: current main warns when `resolveActiveMemoryBackendConfig` returns null, and `memory-lancedb` is an active slot plugin that registers tools/hooks rather than the memory-host runtime. I did not run a live `openclaw doctor` session in this read-only review.

## Solution Assessment

No. The suppression target is right, but the current PR line treats a missing alternate plugin entry as healthy; the safer fix must prove configured active ownership or expose the plugin through the memory runtime contract.
