## Summary

This PR targets a real, source-reproducible doctor false warning, but the current patch is still not mergeable because the new guard treats a slot-only alternate memory selection as healthy and the real behavior proof still contains template placeholders.

## What This Changes

The PR adds an alternate-memory-slot guard to doctor memory diagnostics, regression tests for alternate/default slots, and a changelog entry.

## Best Possible Solution

Require explicit enabled/configured or registry-confirmed alternate memory ownership before suppressing the note, add a slot-only or missing-config regression test, and merge after concrete redacted doctor output proves the real setup.

## Reproduction Assessment

Yes, at source level: current main warns when resolveActiveMemoryBackendConfig returns null, while memory-lancedb is documented as an active slot plugin but does not register the memory-host runtime this path expects. I did not run a live macOS/npm-global doctor session in this read-only review.

## Solution Assessment

No, not yet. The suppression target is right, but the current guard should require an explicitly configured/enabled entry or registry-backed active plugin instead of treating missing entry state as healthy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Require configured alternate memory ownership:** `src/commands/doctor-memory-search.ts:323`
  - body: This returns true when the selected alternate slot has no entry because `plugins.entries[memorySlot]?.enabled !== false` is also true for a missing entry. In that state memory-lancedb registers only its disabled-until-configured warning service and no memory tools/hooks, so doctor can now hide the missing-runtime note for an unavailable memory plugin. Require an explicit configured/enabled entry or registry-backed active plugin before suppressing the warning.
