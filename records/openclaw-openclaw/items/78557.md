Keep open: the reported doctor warning is source-reproducible and the suppression direction fits, but this PR can hide the warning for slot-only or invalid alternate memory configs, and the real-behavior proof still contains placeholders instead of concrete doctor output.

## What This Changes

The PR adds an alternate-memory-slot guard to doctor’s memory-search warning, two doctor-memory-search tests, and an Unreleased changelog entry.

## Best Possible Solution

Require explicit configured/enabled or registry-confirmed alternate memory ownership before suppressing the note, add a slot-only or invalid-config regression test, and merge only after concrete real doctor proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main warns on null memory-host backend config, while documented memory-lancedb slot ownership uses tools and hooks instead of the memory-host runtime.

## Solution Assessment

No, not yet. The suppression target is right, but slot text alone is not enough to prove an alternate memory plugin is configured and active.

## Review Findings

- **[P2] Require a configured alternate slot before suppressing warnings:** `src/commands/doctor-memory-search.ts:323`

Require explicit configured or registry-backed active ownership before returning here.
