## Summary

Keep open: the PR targets a valid source-reproducible false doctor warning, but the proposed guard is too broad and can suppress the warning for an unconfigured alternate memory slot; the external real-behavior proof is also still incomplete.

## What This Changes

The PR adds a doctor memory-slot suppression helper, two doctor-memory-search unit tests, and a changelog entry for the memory-lancedb false warning.

## Best Possible Solution

Narrow the suppression to a verified active alternate memory owner, add a regression for slot-only or unconfigured alternate slots, and require concrete live openclaw doctor before/after output.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main warns when the memory-host backend config is null, while memory-lancedb is documented as a slot-owned memory plugin that does not register the host runtime this check expects.

## Solution Assessment

No. The suppression target is right, but the current PR treats slot text alone as proof of an active plugin; the safer fix is to require a configured/enabled entry or registry-confirmed active memory owner before suppressing the note.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Require active alternate memory ownership before suppressing:** `src/commands/doctor-memory-search.ts:323`
  - body: This returns true when `plugins.entries[memorySlot]` is missing, because `undefined?.enabled !== false` is true. With `plugins.slots.memory = "memory-lancedb"` but no configured entry, memory-lancedb registers only its disabled-until-configured warning service and no tools/hooks, so doctor would now hide the missing-memory warning for an inactive plugin. Require a configured/enabled entry or registry-confirmed active owner before returning here.
