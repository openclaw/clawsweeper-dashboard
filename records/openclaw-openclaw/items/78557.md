## Summary

Keep open: the reported false doctor warning is valid, but this PR still suppresses the warning from slot text alone and can hide an invalid or unconfigured alternate memory plugin. A newer open runtime-registration PR tracks a likely better direction, but it is not merged and also still needs fixes/proof, so this PR should stay in review rather than be closed as implemented or superseded.

## What This Changes

The PR suppresses the doctor missing-memory-plugin note for non-default memory slot owners, adds two doctor-memory-search tests, and adds a changelog entry.

## Best Possible Solution

Recognize a valid configured memory-lancedb slot without masking invalid config, preferably through a generic memory runtime/capability or explicit active-ownership check, with focused regression coverage and redacted real doctor output.

## Reproduction Assessment

Yes, at source level. Current main warns when resolveActiveMemoryBackendConfig returns null, and memory-lancedb is documented as an active slot plugin while not registering the host memory runtime this doctor path expects.

## Solution Assessment

No. The suppression target is valid, but this implementation treats slot text as enough proof of an active alternate memory plugin; require configured or registry-confirmed active ownership, or finish the runtime-registration approach in https://github.com/openclaw/openclaw/pull/80211.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Require active ownership before suppressing:** `src/commands/doctor-memory-search.ts:323`
  - body: This treats a slot-only alternate memory selection as healthy because `plugins.entries[memorySlot]?.enabled` is `undefined` when the entry is missing, so `undefined !== false` returns true. With `plugins.slots.memory = "memory-lancedb"` but no valid configured entry, memory-lancedb registers only its warning service and no memory tools/hooks, and doctor would now hide the missing-runtime note for an unavailable memory plugin. Require an explicit configured entry or registry-confirmed active ownership before returning here.
  - confidence: 0.87
