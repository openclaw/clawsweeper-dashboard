## Best Possible Solution

Resolve this through the open linked PR or a narrow replacement that retries the status JSON read without adding config surface, preserves fast missing-binary fallback, and suppresses non-final retry noise.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. Source inspection shows the reported timing window remains: Serve setup is followed immediately by one `status --json` read whose verbose failure can be logged.

## Solution Assessment
