## What This Changes

The issue asks for retry/backoff around the Tailscale hostname status probe after enabling Serve to avoid transient verbose startup errors.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82727, or an equivalent narrow retry that suppresses only intermediate transient status failures while preserving final failure visibility, then close this issue after merge.

## Reproduction Assessment

Yes at source level: current main enables Serve, immediately queries `tailscale status --json`, and verbose `runExec` logs failed commands before rethrowing. I did not establish a live macOS Tailscale daemon reproduction in this read-only review.

## Solution Assessment

Yes: retrying the optional post-Serve status probe and quieting only intermediate retry failures is the narrow maintainable fix. Adding `gateway.tailscale.binaryPath` would be broader config surface and would not directly address daemon settling after Serve setup.
