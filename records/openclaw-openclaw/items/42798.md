The issue requests a narrow fix for transient Tailscale Serve startup failures when `tailscale status --json` runs immediately after enabling Serve mode.

Keep this issue as the canonical report and resolve it through the linked PR after that branch is rebased, reviewed, and proven.

## Solution Assessment

Yes in direction: a narrow retry around the status JSON read is the maintainable fix boundary. The alternative binary-path config option would expand config surface and would not directly solve the post-Serve startup race.
