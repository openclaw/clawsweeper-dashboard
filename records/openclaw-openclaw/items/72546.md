## Summary

Keep open. Current main still allows managed memory-core dreaming to log workspace write failures while returning the generic processed result, so an `ok` cron run can still hide missing phase reports, Dream Diary entries, or completion events.

## What This Changes

The issue reports that managed memory-core dreaming cron runs on 2026.4.24 return ok with eligible REM/deep material but stop writing Dream Diary, phase report, and `memory.dream.completed` artifacts.

## Best Possible Solution

Make managed dreaming success match observable outputs: attempted artifact/event writes should either complete or surface a clear failed cron diagnostic, while preserving legitimate disabled, missing-workspace, limit-zero, and no-material skips.

## Reproduction Assessment

Yes. The issue provides operational reproduction with ok cron history, non-empty harness/status output, and missing artifact/event checks; current source also exposes a deterministic unit path by forcing phase/report writes to fail.

## Solution Assessment

## Repair Work Prompt

Fix the memory-core managed dreaming silent-success path reported in https://github.com/openclaw/openclaw/issues/72546. In current main, cron-triggered `runShortTermDreamingPromotionIfTriggered` can catch workspace write failures, log them, and still return `memory-core: short-term dreaming processed`, leaving no `DREAMS.md`, `memory/dreaming/*`, or `memory.dream.completed` artifacts. Implement a narrow success invariant in `extensions/memory-core`: when a managed dreaming sweep attempts artifact/event writes and any workspace write path fails, surface a clear failure/diagnostic to the cron-visible result instead of the generic processed success. Preserve legitimate skips for disabled dreaming, no workspace, `limit=0`, and real no-material cases. Add deterministic regression coverage by forcing phase/report/event writes to fail; do not broaden scoring, backfill recovery, core plugin APIs, or unrelated dreaming behavior. Add a user-facing `CHANGELOG.md` fix entry. Related refs: #72546; #74639 is related but includes additional DREAMS.md/backfill structure symptoms and should not be over-scoped unless the same invariant directly covers it.
