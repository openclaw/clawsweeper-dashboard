## What This Changes

The issue reports that OpenClaw 2026.4.24 memory-core dreaming cron runs report success with eligible REM/deep material but stop writing DREAMS.md, per-phase reports, and memory.dream.completed events after April 18, 2026.

## Best Possible Solution

Make the memory-core managed dreaming sweep enforce an observable success invariant: when dreaming is enabled and a workspace has eligible material, the cron-triggered sweep should write the expected phase report/diary/event artifacts or return a clear failure/diagnostic instead of a generic processed success.
