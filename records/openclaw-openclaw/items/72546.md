## Summary

Keep open. Current main still has a silent-success path where cron-triggered memory-core dreaming can log per-workspace write failures and return the generic processed result, so successful cron history does not prove that Dream Diary, phase reports, or completion events were written.

## What This Changes

The issue reports that managed memory-core dreaming cron runs can report success with eligible REM/deep material while producing no new dreaming artifacts or completion events.

## Best Possible Solution

Enforce a managed-dreaming success invariant: an enabled sweep that attempts artifact or event writes should either produce the expected outputs or return a clear diagnostic failure, while preserving legitimate disabled, limit-zero, missing-workspace, and no-material skips.

## Reproduction Assessment

Yes. The issue gives a high-confidence operational reproduction with ok cron runs, non-empty `rem-harness`/status output, and missing artifact/event checks; current source also exposes a deterministic unit path by forcing phase/report writes to fail.

## Solution Assessment
