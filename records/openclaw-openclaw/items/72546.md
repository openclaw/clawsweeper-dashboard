## Summary

Keep open. Current main still has a memory-core dreaming path where workspace/report write failures are logged but the cron handler returns the generic processed result, so successful cron history can still hide missing `DREAMS.md`, phase reports, or `memory.dream.completed` events.

## What This Changes

The issue asks memory-core dreaming cron runs with eligible REM/deep material to produce visible dreaming artifacts and completion events instead of reporting `ok` with no writes.

## Best Possible Solution

Make managed dreaming success reflect the artifact/event contract: artifact write failures should either produce visible outputs and completion events or return a clear diagnostic failure, while preserving legitimate disabled, limit-zero, missing-workspace, and no-material skips.

## Reproduction Assessment
