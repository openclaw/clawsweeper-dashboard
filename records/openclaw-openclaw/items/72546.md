## Summary

Keep open. Current main still has a memory-core dreaming path where cron-triggered workspace/report write failures are logged but the handler returns the generic processed result, so successful cron history can still hide missing `DREAMS.md`, phase report, or `memory.dream.completed` artifacts. Related open reports do not supersede this no-artifacts path.

## What This Changes

The issue asks for memory-core dreaming cron runs with eligible REM/deep material to either write documented dreaming artifacts/events or stop reporting success.

## Best Possible Solution

Make managed dreaming success reflect the documented artifact/event contract: attempted phase/report writes should either produce visible outputs and completion events or return a clear diagnostic failure, while preserving legitimate disabled, limit-zero, no-workspace, and no-material skips.
