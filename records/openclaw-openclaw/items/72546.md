## Summary

Keep open. Current main has an adjacent primary-workspace fix, but the reported success-with-no-artifacts invariant is still not enforced because cron-triggered dreaming can log per-workspace write failures and still return the generic processed result.

## What This Changes

The issue reports that managed memory-core dreaming cron runs return ok with eligible REM/deep material but write no new DREAMS.md, phase report, or memory.dream.completed artifacts.

## Best Possible Solution

Make managed memory-core dreaming enforce an observable success invariant: with dreaming enabled and eligible material present, cron should either write the expected artifacts/events or return a clear diagnostic failure while preserving legitimate disabled/no-material skips.
