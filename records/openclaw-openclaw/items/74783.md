## Summary

Keep this PR open. The feature is valid and current main does not have per-client health connection telemetry, but the PR should not merge until delayed pong handling is fixed and final ping-loop ownership is coordinated with the overlapping keepalive PR.

## What This Changes

This PR adds optional per-client `health.connection` telemetry to authenticated WebSocket health responses, records WebSocket ping/pong RTT in gateway maintenance, updates docs/types, and adds focused tests plus a changelog entry.

## Best Possible Solution

Keep the per-response `health.connection` overlay design, but track pending or bounded recent ping payloads so delayed valid pongs still refresh connection health, then land with a single protocol ping implementation path.

## Reproduction Assessment

Yes for the review finding: with the PR code, send pings at 0ms and 5000ms, then receive the valid 0ms pong at 6000ms; the latest-timestamp equality rejects it, so heartbeat state never updates. For the original feature request, reproduction is not applicable because current-main source inspection proves the fields are absent.

## Solution Assessment

No, not as currently written. The overlay approach is the right boundary, but the pong matching should tolerate delayed valid pong samples within the stale window rather than accepting only the most recent ping timestamp.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Accept delayed pong samples within the stale window:** `src/gateway/server/connection-health.ts:32`
  - body: The maintenance loop sends a new ping every 5s, but this check only accepts a pong whose payload equals the latest ping timestamp. If a valid pong arrives after the next ping was sent, every sample can be rejected, `lastHeartbeatAt` never advances, and `health.connection.connected` flips false after 20s even though pongs are arriving. Track pending/recent ping timestamps or otherwise accept bounded valid samples instead of only the latest one.
  - confidence: 0.84
