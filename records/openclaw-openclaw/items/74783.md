## What This Changes

The PR adds optional per-client `health.connection` fields to Gateway health responses, tracks WebSocket ping/pong RTT and heartbeat state, and updates docs, changelog, types, and gateway tests.

## Best Possible Solution

Land the `health.connection` response overlay on top of a single authenticated Gateway WebSocket ping/pong owner, reusing or replacing #76111's ping timer rather than adding a second loop.

## Solution Assessment
