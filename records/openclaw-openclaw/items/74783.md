## Summary

## What This Changes

The PR adds an optional per-client `health.connection` block to Gateway health responses, tracks RTT from authenticated WebSocket ping/pong state, and updates health docs, changelog, types, and gateway tests.

## Best Possible Solution

Land or revise this PR after maintainer API review, preserving a single authenticated WebSocket ping owner and keeping `connection` as a per-response overlay outside the shared cached health snapshot.

## Solution Assessment
