## Summary

Keep open: this PR adds a new Telegram streaming transport, but it conflicts with the current Telegram streaming direction and current main behavior, and the supplied proof does not show the patched OpenClaw runtime path.

## What This Changes

The PR adds an opt-in Telegram `streaming.nativeTransport` path using `sendMessageDraft`, with tests and docs for fallback and final normal delivery.

## Best Possible Solution

Keep Telegram preview streaming on the existing editable preview-message path; reconsider native drafts only through an owner-approved design that updates finalization semantics and proves the full OpenClaw Telegram runtime.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. The changed behavior is source-traceable: enabling `streaming.nativeTransport` routes Telegram draft lanes to `sendMessageDraft`.

## Solution Assessment

No. The proposed opt-in is not the best current solution because it reintroduces a Telegram transport that current maintainer notes and docs intentionally avoid in favor of one editable persistent preview message.

## Review Findings
