## Summary

Keep open. Current main still does not implement `channels.signal.linkPreview`, and a maintainer comment already identified the remaining Signal design/security questions as needing follow-up rather than cleanup closure.

## What This Changes

The issue requests a first-class `channels.signal.linkPreview` setting, default-safe behavior, and Signal JSON-RPC pass-through for link-preview configuration and preview fields.

## Best Possible Solution

Keep this open for maintainer-reviewed Signal plugin work: add the config type, strict schema, generated metadata, docs, and tests; decide account/default semantics and `updateConfiguration` timing; and pass only explicit safe preview fields through Signal JSON-RPC unless #22060 separately approves metadata fetching.

## Reproduction Assessment

Not applicable as a feature request, but current-main verification is high-confidence: the config/schema/docs/send surfaces omit `channels.signal.linkPreview`, and the strict schema would not accept the requested key.

## Solution Assessment

No, the proposal is not ready as a blind automated fix. The maintainable path is the same narrow Signal plugin implementation plus an explicit security/primary-device design decision before enabling preview configuration or metadata handling.
