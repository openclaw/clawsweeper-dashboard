## Risks / Open Questions

- Top-level versus per-account semantics for multi-account Signal setups remain undecided.
- `signal-cli updateConfiguration --link-previews` can fail on linked devices, so OpenClaw needs an explicit policy for failure handling, retries, and operator visibility.
- Preview metadata fetching or injection could overlap the open prompt-injection issue at https://github.com/openclaw/openclaw/issues/22060; a fix should pass only explicit safe fields unless a separate reviewed metadata design exists.
- Native signal-cli JSON-RPC and the bbernhard container adapter may not have the same preview/configuration support, so a default-on setting needs compatibility proof.
