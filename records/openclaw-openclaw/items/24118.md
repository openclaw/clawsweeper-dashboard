## Summary

Keep open. Current main still does not implement `channels.signal.linkPreview`: the Signal type, strict schema, generated metadata, docs, and JSON-RPC send path all omit the requested setting and preview/updateConfiguration wiring. Upstream `signal-cli` does support link previews and a `link-previews` configuration flag, but `updateConfiguration` is primary-device-only and the open #22060 link-preview metadata security concern makes this maintainer-review work rather than an automatic close or repair lane.

## What This Changes

Add a configurable `channels.signal.linkPreview` option and pass Signal link-preview configuration/send fields through the Signal JSON-RPC send path.

## Best Possible Solution

Keep this as maintainer-reviewed Signal plugin work: add `channels.signal.linkPreview` to the public type, strict schema, generated metadata, docs, and tests; decide top-level versus per-account semantics and default behavior; define `updateConfiguration` timing/error handling for primary versus linked devices; and wire safe explicit preview fields through JSON-RPC send in coordination with #22060.
