## Summary

Keep open. Current main and the latest release still omit Signal link-preview configuration and outbound preview mapping; upstream supports the capability, but primary-device restrictions, two Signal backends, and untrusted metadata handling require maintainer product and security decisions before implementation.

## What This Changes

Add a `channels.signal.linkPreview` setting and map Signal preview configuration and metadata into outbound native JSON-RPC and container sends.

## Best Possible Solution

Define one Signal-plugin-owned preview policy with explicit account scoping, safe default semantics, graceful linked-device behavior, equivalent native/container payload mappings, and no OpenClaw metadata fetching or agent-context ingestion.

Not applicable: this issue requests a new Signal configuration and outbound capability rather than reporting broken behavior against an existing documented contract.

No, not exactly as proposed. A first-class Signal option is plausible, but default-on configuration mutation and native-only pass-through do not safely cover linked devices or the supported container backend.
