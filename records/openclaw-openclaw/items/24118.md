## Summary

Keep open: current main still lacks `channels.signal.linkPreview` in Signal config/schema/docs/metadata and does not pass Signal link-preview fields through outbound send. Upstream support exists, but primary-device configuration behavior, native/container payload mapping, and related link-preview metadata safety need maintainer design before implementation.

## What This Changes

Request adds a configurable `channels.signal.linkPreview` option and forwards explicit Signal link-preview fields through native/container send APIs.

## Best Possible Solution

Implement this inside the Signal plugin with typed config/schema/generated metadata/docs/tests, explicit account/default semantics, primary-device-safe configuration behavior, and explicit preview-field pass-through without untrusted metadata fetching unless the security design in https://github.com/openclaw/openclaw/issues/22060 is resolved.
