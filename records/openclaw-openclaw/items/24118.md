## Summary

Keep open: current main still lacks the Signal link-preview config, docs/metadata, and JSON-RPC preview pass-through, and the requested behavior needs maintainer design because it adds a new config option and overlaps the open link-preview metadata security concern.

## What This Changes

Add a first-class `channels.signal.linkPreview` config option and pass Signal link-preview settings and fields through the signal-cli JSON-RPC send path.

## Best Possible Solution

Implement this as maintainer-reviewed Signal plugin work with typed config/schema/metadata/docs/tests, explicit account-scope semantics, cautious primary-device configuration handling, and safe preview-field pass-through that does not fetch or inject metadata until the related security design is settled.
