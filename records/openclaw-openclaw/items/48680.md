## Summary

Keep open. Current main and v2026.5.18 still have the source-reproducible classifier gap, and the open closing PR should be reviewed or landed before this issue closes.

## What This Changes

The issue reports that provider business-error payloads marked `isError=true` can still be treated as successful model fallback candidates, stopping the remaining fallback chain.

## Best Possible Solution

Ship a narrow fix in the embedded fallback classifier and shared failover matcher so provider denial payloads are classified before a fallback candidate is marked successful.

## Reproduction Assessment

Yes. Source inspection on current main is high-confidence: a non-GPT embedded result whose only signal is unclassified `isError` payload text can return `null` from the classifier, letting fallback treat the attempt as success.

## Solution Assessment

Yes. The narrow maintainable solution is to classify embedded provider-error payload text through the existing failover matcher before the non-GPT success path, which is what the linked PR attempts.
