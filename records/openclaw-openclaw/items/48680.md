## Summary

Keep open because current main and v2026.5.20 still have the classifier gap, while the open closing PR is an implementation candidate that has not merged.

## What This Changes

The issue reports that embedded provider business-denial errors can be treated as successful model-fallback candidates, preventing later fallback models from running.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/84814 or an equivalent focused classifier/matcher fix, then close this issue through the merged commit.

## Reproduction Assessment

Yes. Current main is source-reproducible: a non-GPT embedded result whose only signal is unclassified `isError` payload text can return `null` from the classifier, allowing fallback to treat the attempt as success.

## Solution Assessment

Yes. The narrow maintainable fix is to classify embedded provider-error payload text through the existing failover matcher before the non-GPT success path, with the open PR or equivalent repair preserving existing local-block semantics.
