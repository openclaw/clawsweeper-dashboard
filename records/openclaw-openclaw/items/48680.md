## Summary

Keep open. Current main still has a credible source-reproducible fallback gap for the reported CE-011 business-denial shape: related 404 and finish_reason cases were fixed, but this issue's CJK denial markers are still unclassified and no canonical PR appears to own the remaining fix.

## What This Changes

The issue reports that semantic provider error payloads can be treated as successful fallback candidates, preventing later fallback models from being tried.

## Best Possible Solution

Teach the embedded/failover classification boundary to treat the reported `success:false` CE-011 denial as failover-worthy, then verify the chain advances without weakening side-effect and visible-reply guards.

## Reproduction Assessment

Yes, source-reproducible with high confidence from current main and the issue logs. A normal embedded result whose only signal is the reported unclassified `isError` payload can still return null from the classifier, allowing the fallback attempt to be treated as success.

## Solution Assessment

Yes. The narrow maintainable fix is in semantic provider-error classification for embedded results or shared failover patterns, not a broad retry of every surfaced error after fallback success is already committed.
