## What This Changes

The issue reports provider business-denial responses being treated as successful model fallback candidates instead of failed candidates.

## Best Possible Solution

Land the focused classifier/matcher repair in https://github.com/openclaw/openclaw/pull/84814 or an equivalent fix, then close this issue through the merged commit.

## Reproduction Assessment

Yes. Current main is source-reproducible: a non-GPT embedded result whose only signal is unclassified `isError` payload text can return `null` from the classifier, allowing fallback to treat the attempt as success.

## Solution Assessment

Yes. The narrow maintainable path is to classify embedded provider-error payload text through the existing failover matcher before the non-GPT success path, which is what the linked replacement PR targets.
