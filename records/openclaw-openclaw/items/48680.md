## Summary

Keep open. Current main and the latest release still have the source-reproducible classifier gap, and the issue has an open closing PR that should be reviewed or landed before this issue closes.

## What This Changes

The issue reports that embedded provider business-error payloads marked as `isError` can still be treated as successful fallback attempts, stopping later model candidates.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83042 if maintainer validation agrees, then close this issue through that merged fix.

## Reproduction Assessment

Yes. Current main is source-reproducible: a non-GPT embedded result whose only signal is an unclassified `isError` payload can return `null` from the classifier, allowing fallback to treat the attempt as success.

## Solution Assessment

Yes. The narrow maintainable solution is to classify embedded provider-error payload text through the existing failover matcher before the non-GPT success path, which is what the linked PR attempts.
