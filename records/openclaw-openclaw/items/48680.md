- `impact:auth-provider`: The affected behavior is provider rejection classification and model fallback routing across configured provider/model candidates.

## Summary

Keep open. Current `main` and the latest release still have the source-level classifier gap described in the issue discussion, and the open closing PR at https://github.com/openclaw/openclaw/pull/84814 has not merged.

## What This Changes

Treat embedded provider business-rejection error payloads as fallback failures so the remaining configured model candidates are attempted.

## Best Possible Solution

Land the focused classifier/matcher repair in https://github.com/openclaw/openclaw/pull/84814 or an equivalent fix, then close this issue through the merged commit.

## Reproduction Assessment

Yes. Current main is source-reproducible: a non-GPT embedded result whose only signal is unclassified `isError` payload text can return `null` from the classifier, allowing fallback to treat the attempt as success.

## Solution Assessment

Yes. The narrow maintainable solution is to classify embedded provider-error payload text through the existing failover matcher before the non-GPT success path, which is what the linked replacement PR targets.
