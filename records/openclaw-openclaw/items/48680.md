## Summary

Keep open. Current main and the latest release still have the source-reproducible classifier gap, and the issue is now paired with an open closing PR that should be reviewed or landed before the issue closes.

## What This Changes

The issue asks model fallback to treat embedded provider business-denial payloads such as CE-011 as failed candidates so later fallback models are attempted.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83042 if maintainer validation agrees, then close this issue through that merged fix.

## Reproduction Assessment

Yes. Current main is source-reproducible: a non-GPT embedded result whose only signal is the reported unclassified `isError` payload can return `null` from the classifier, allowing fallback to treat the attempt as success.

## Solution Assessment

Yes. The narrow maintainable solution is to classify embedded provider-error payload text through the existing failover matcher before the non-GPT success path, which is exactly what the linked PR attempts.
