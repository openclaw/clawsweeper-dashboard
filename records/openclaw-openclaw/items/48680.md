## Summary

## What This Changes

The issue asks for provider business-denial responses surfaced as embedded `isError` payloads to be treated as failed fallback candidates so remaining configured models are tried.

## Best Possible Solution

Rebase and land https://github.com/openclaw/openclaw/pull/84814, or an equivalent focused classifier/matcher fix, so embedded provider-denial error payloads classify through the existing failover matcher before fallback success is committed.

## Reproduction Assessment

Yes. Current main is source-reproducible: a non-GPT embedded result whose only signal is unclassified `isError` CE-011/CJK denial text returns `null` from the classifier, and `runFallbackAttempt` treats that as candidate success.

## Solution Assessment

Yes. The narrow maintainable fix is the linked PR's approach: classify embedded provider-error payload text through the existing failover matcher before the non-GPT success path, while preserving the unclassified-error behavior.
