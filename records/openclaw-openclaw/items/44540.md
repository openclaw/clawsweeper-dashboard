## Summary

Keep open. Current main still drops media modality after file classification, so the builtin hybrid ranker scores image/audio vector hits with textScore 0 under default weights; no linked PR has replaced that narrow fix.

## What This Changes

The issue asks for multimodal memory-search results to avoid BM25 dilution when image/audio chunks are only meaningfully comparable by vector score.

## Best Possible Solution

Persist image/audio modality through the builtin memory schema and search result pipeline, refresh affected media rows safely, and score media vector hits by vector relevance while preserving BM25-plus-vector weighting for text chunks.

## Reproduction Assessment

Yes, at source level. Current main classifies image/audio entries before indexing, drops modality from persisted/search rows, and then gives vector-only media hits textScore 0 in the default weighted hybrid formula.

## Solution Assessment

Yes. Carrying persisted modality into the ranker is the narrow maintainable fix; changing global weights would weaken text search, and inferring modality from file paths would be brittle.
