## Summary

Current main still applies the configured text weight to every hybrid result, including vector-only media chunks, so the multimodal ranking defect remains. Keep this issue open because the active closing PR already owns the focused repair and should be reviewed or closed before any duplicate work is queued.

## What This Changes

The issue asks memory hybrid search to stop discounting image and audio results with a near-zero BM25/text component when multimodal embeddings are enabled.

## Best Possible Solution

Review and land or reject the active closing PR; if it lands, GitHub should close this issue through that merged fix rather than a duplicate repair branch.

## Reproduction Assessment

Yes. Current main deterministically gives vector-only media candidates `textScore: 0` and still applies the configured BM25/text weight, so the source path reproduces the reported discount; I did not run a live Gemini corpus in this read-only review.

## Solution Assessment

Yes. The linked PR's modality-aware normalization at the existing hybrid fusion boundary is narrower than changing global weights or adding a new config knob, provided the PR preserves keyword-only and zero-vector-weight edge cases.
