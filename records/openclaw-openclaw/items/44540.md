## Summary

Keep open. Current main still documents Gemini multimodal memory indexing and default hybrid BM25+vector search, but the built-in memory index drops the image/audio modality before search and the hybrid ranker still applies the same weighted BM25 formula to vector-only media hits.

## What This Changes

The issue requests that built-in hybrid memory search stop downweighting image/audio chunks whose BM25 text score is not meaningful under default multimodal search.

## Best Possible Solution

Persist chunk modality through memory schema/search rows and use vector-only scoring for image/audio chunks while preserving normal BM25+vector weighting for text chunks.

## Reproduction Assessment

Yes, source-level. Current main classifies image/audio files before indexing, drops modality before persistence/search, and then scores vector-only candidates as vectorWeight * vectorScore when textScore is zero.

## Solution Assessment

Yes. Carrying modality into the ranker is the narrow maintainable fix; changing global weights or guessing media type from paths would weaken normal text hybrid scoring or be brittle.
