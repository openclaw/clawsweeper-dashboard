## Summary

Keep open. Current main still drops the image/audio modality before persistence and uses a modality-blind hybrid score, so the reported ranking penalty remains a valid memory-core bug rather than a stale or duplicate item.

## What This Changes

The issue asks for hybrid memory search to avoid applying a BM25 penalty to image/audio multimodal chunks whose relevance comes from vector embeddings.

## Best Possible Solution

Persist explicit chunk modality through indexing, schema, search rows, and result merging, then score image/audio chunks by vector relevance without weakening BM25 for text chunks.

## Reproduction Assessment

Yes, source-level. Current main carries modality on MemoryFileEntry but drops it before chunk persistence and search, while vector-only hybrid hits get textScore 0 and are multiplied by the default 0.7 vector weight.

## Solution Assessment

Yes. The narrow maintainable fix is to carry a typed modality signal through the memory schema/result path and branch scoring on that signal, rather than inferring from filenames or changing global hybrid weights.
