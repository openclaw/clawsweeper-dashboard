## Summary

Keep open. Current main still carries image/audio modality only during file-entry construction, then drops it from storage/search rows and applies the same vector-plus-BM25 weighted score to every hybrid candidate, so the reported penalty remains source-reproducible.

## What This Changes

The issue requests fair hybrid ranking for multimodal memory-search chunks so image and audio results are not penalized by near-zero BM25 text scores.

## Best Possible Solution

Carry explicit chunk modality through memory schema/write/search result contracts and score non-text multimodal vector candidates without changing global default weights for normal text hybrid search.

## Reproduction Assessment

Yes, source-level. Current main creates multimodal entries with modality, persists/searches them without modality, and ranks vector-only candidates as vectorWeight * vectorScore when textScore is zero.

## Solution Assessment

Yes. The maintainable fix is to persist or otherwise carry explicit modality into the ranker; changing default weights or inferring media type from filenames at merge time would be a weaker workaround.
