## Summary

Keep open. Current main still preserves image/audio modality only before indexing, drops it from the persisted/search result shape, and applies the same weighted BM25-plus-vector formula to vector-only media hits.

## What This Changes

The issue asks for default hybrid memory search to rank multimodal image/audio chunks fairly instead of suppressing them when their BM25 text score is near zero.

## Best Possible Solution

Persist chunk modality through the built-in memory schema/search rows, reindex or refresh media rows safely, and score image/audio chunks by vector relevance while preserving BM25-plus-vector weighting for text chunks.

## Reproduction Assessment

Yes, at source level. Current main can build multimodal entries with modality, but persisted chunks/search results omit modality and mergeHybridResults gives vector-only hits textScore 0 under the default weighted formula.

## Solution Assessment

Yes. Carrying persisted modality into the ranker is the narrow maintainable fix; changing global weights would weaken text search, and inferring modality from file paths would be brittle.
