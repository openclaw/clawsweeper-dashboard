## Summary

Keep open. Current main still applies one weighted vector/BM25 merge to every builtin memory result, while multimodal image/audio chunks are stored and returned without persisted modality metadata. That means a vector-only image/audio candidate still gets multiplied by the default 0.7 vector weight and receives textScore 0, matching the reported ranking penalty. The existing tests prove multimodal indexing and literal image/audio searches, but not natural-language multimodal recall under default hybrid ranking.

## Best Possible Solution

Fix this in the builtin memory engine by carrying reliable image/audio modality metadata from multimodal indexing into stored chunks/search rows, then make hybrid merging avoid BM25 dampening for non-text vector candidates while preserving the current weighted merge for text chunks. Add regression coverage with mixed Markdown and image/audio candidates showing a natural-language visual/audio query surfaces multimodal results under default hybrid weights.
