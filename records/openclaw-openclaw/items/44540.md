## Summary

Keep open. Current main still matches the reported source-level behavior: multimodal image/audio files carry modality before indexing, but that signal is not persisted into search rows, and hybrid ranking still scores vector-only hits as `vectorWeight * vectorScore` when BM25 has no useful text match.

## What This Changes

Fix hybrid memory search so image/audio chunks indexed through multimodal embeddings are not unfairly downweighted by the BM25 component.

## Best Possible Solution

Persist chunk modality through the built-in memory schema/search rows, refresh existing media rows safely, and score image/audio chunks with vector-only ranking while preserving normal BM25+vector weighting for text chunks.

## Reproduction Assessment

Yes, at source level. Current main classifies image/audio before indexing, drops modality before persistence/search, and scores vector-only candidates as vectorWeight * vectorScore when textScore is zero.

## Solution Assessment

Yes. Carrying modality into the ranker is the narrow maintainable fix; changing global hybrid weights would weaken text search, and guessing media type from paths would be brittle.
