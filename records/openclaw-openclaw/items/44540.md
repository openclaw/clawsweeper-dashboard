## Summary

Keep open. Current main still persists multimodal files as label-text chunks without carrying image/audio modality into stored search rows, and the hybrid merge still applies the same weighted vector+BM25 formula to every result.

## What This Changes

Make default hybrid memory search rank image/audio multimodal chunks fairly instead of damping vector-only matches with the BM25 text weight.

## Best Possible Solution

Persist explicit chunk modality through the memory schema/result path and score non-text multimodal vector candidates without weakening BM25 for normal text chunks.

## Reproduction Assessment

Yes, source-level. Current main creates multimodal label-text chunks, drops modality before persistence, and gives vector-only hybrid hits a weighted score of vectorWeight * vectorScore when textScore is zero.

## Solution Assessment

Yes. The maintainable fix is to carry modality through indexing/storage/search rows and branch the ranker on that explicit signal, not to change global default weights or infer media type from filenames at merge time.
