## Summary

Keep open: current main and v2026.6.10 still apply uniform hybrid scoring that discounts vector-only image/audio memory results, and the exact fix path is already represented by the open closing PR at https://github.com/openclaw/openclaw/pull/92196.

## What This Changes

The issue reports that hybrid memory search downranks image/audio results because BM25 text scoring is applied to media chunks with only synthetic labels.

## Best Possible Solution

Review and land, revise, or reject https://github.com/openclaw/openclaw/pull/92196; close this issue only after a fix reaches main or another canonical solution replaces it.

## Reproduction Assessment

Yes. Source inspection shows vector-only candidates enter hybrid fusion with textScore 0 and are scored with the default 0.7 vector plus 0.3 text formula; I did not run a live Gemini-keyed scenario in this read-only review.

## Solution Assessment

Yes. The best fix belongs at the memory-core hybrid fusion boundary using the existing multimodal classifier, and the open linked PR follows that shape without adding config or schema surface.
