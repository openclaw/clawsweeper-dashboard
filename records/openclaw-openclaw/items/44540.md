## Summary

Keep open. Current main still preserves image/audio modality only before indexing, drops it before persistence/search, and applies the normal hybrid weighted score to every vector-only result.

## What This Changes

The issue requests that hybrid memory search stop down-weighting indexed image/audio chunks that have valid vector matches but no meaningful BM25 text score.

## Best Possible Solution

Persist chunk modality through the built-in memory schema/search rows, refresh media rows safely, and score image/audio chunks by vector relevance while preserving BM25+vector weighting for text chunks.

## Reproduction Assessment

Yes, at source level. Current main classifies image/audio before indexing, drops modality before persistence/search, and scores vector-only candidates as vectorWeight * vectorScore when textScore is zero.

## Solution Assessment

Yes. Carrying modality into the persisted search row and ranker is the narrow maintainable fix; changing global hybrid weights would weaken text search, and inferring media type from paths would be brittle.
