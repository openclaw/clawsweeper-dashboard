## Summary

Keep open. Current main still has the documented multimodal indexing path, but the builtin hybrid ranker has no persisted modality signal and still applies the same vectorWeight plus textWeight formula to every candidate, so image/audio vector hits with no lexical match remain downweighted under the default hybrid configuration.

## What This Changes

The issue requests that builtin hybrid memory search avoid penalizing indexed image/audio chunks with a near-zero BM25 component when multimodal memory search is enabled.

## Best Possible Solution

Carry explicit chunk modality through schema, persistence, search rows/results, and hybrid ranking, then score image/audio vector candidates without weakening normal text hybrid scoring.

## Reproduction Assessment

Yes, source-level. Current main classifies multimodal files before indexing, drops modality before persistence/search, and ranks vector-only candidates as vectorWeight * vectorScore when textScore is zero.

## Solution Assessment

Yes. Persisting modality into the ranker is the narrow maintainable fix; changing global weights or guessing media type from paths at merge time would be weaker.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/44540. The builtin memory hybrid search path currently classifies multimodal files before indexing but drops modality before persistence/search and then computes vectorWeight * vectorScore + textWeight * textScore for all candidates, so image/audio vector-only hits are downweighted under default hybrid settings. Add a persisted/searchable chunk modality signal, thread it through search result types and mergeHybridResults, and score non-text image/audio vector candidates without applying the BM25 text-weight penalty when there is no lexical hit. Preserve normal text hybrid scoring, keyword-only fallback behavior, and existing public contracts except for any necessary additive result/schema typing. Do not solve this by changing default hybrid weights or by ranking-time filename extension guessing when the index can carry modality. Add focused regression coverage for the hybrid helper and a builtin memory integration/search path; add a CHANGELOG.md Fixes entry for the user-visible bug.
