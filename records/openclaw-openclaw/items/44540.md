## Summary

Keep open. Current main still carries multimodal image/audio classification only through indexing, then drops modality before persistence and applies the same hybrid BM25/vector weighting to all vector hits, so the reported media-ranking bug remains source-reproducible.

## What This Changes

The issue asks for hybrid memory search to avoid penalizing image/audio vector results with near-zero BM25 text scores under default weights.

## Best Possible Solution

Persist image/audio modality through the builtin memory schema and search result pipeline, refresh affected media rows safely, and score non-text media vector hits by vector relevance while preserving BM25-plus-vector weighting for text chunks.

## Reproduction Assessment

Yes, at source level. Current main classifies image/audio entries before indexing, drops modality from persisted/search rows, and then gives vector-only media hits textScore 0 in the default weighted hybrid formula.

## Solution Assessment

Yes. Carrying persisted modality into the ranker is the narrow maintainable fix; changing global weights would weaken text search, and inferring modality from file paths would be brittle.
