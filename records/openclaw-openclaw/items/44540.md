## Label Justifications

- `P2`: This is a bounded retrieval-quality bug in opt-in multimodal memory with a documented configuration workaround and no core availability failure.
- `impact:session-state`: Artificially discounted image and audio matches can be omitted from retrieved memory context despite relevant embeddings.

## Summary

Current main still applies the configured text weight to every vector result even when a media chunk has no meaningful text match, so the reported retrieval defect remains. Keep this issue open because the clean, proof-backed closing PR is still open and specifically owns the repair.

## What This Changes

Make hybrid memory fusion score vector-backed image and audio chunks without an artificial BM25 discount while preserving existing text and keyword-only behavior.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92196 if its final head continues to preserve text-only scoring, keyword-only media matches, and zero-vector-weight configurations; GitHub should then close this issue.

## Reproduction Assessment

Yes. Current main initializes unmatched media candidates with `textScore: 0` and still applies the default 0.3 text weight, providing a high-confidence source reproduction of the discount; the linked PR also reports a real before/after manager-pipeline scenario.

## Solution Assessment

Yes. Modality-aware renormalization at the existing memory-core fusion boundary is narrower and more maintainable than changing global hybrid defaults, provided keyword-only and zero-vector-weight cases remain covered.
