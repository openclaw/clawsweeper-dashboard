## Summary

Keep open. Current main still drops image/audio modality before ranking and applies the same global vector-plus-BM25 weighted score to every hybrid result, so the reported multimodal penalty remains source-reproducible.

## What This Changes

The issue asks for hybrid memory search to stop penalizing image/audio chunks with BM25 text scores when multimodal memory is enabled.

## Best Possible Solution

Persist explicit chunk modality through the schema/write/search-row contracts and score non-text multimodal vector candidates without weakening BM25 for normal text chunks.

## Reproduction Assessment

Yes, source-level. Current main creates multimodal chunks as label text, persists rows without modality, and gives vector-only hybrid hits a weighted score of vectorWeight * vectorScore when textScore is zero.

## Solution Assessment

Yes. The narrow maintainable fix is to carry modality through indexing/storage/search and branch the ranker on that explicit signal, not to change global default weights or infer media type from filenames at merge time.
