## What This Changes

The issue requests fixing hybrid memory search so vector-only image/audio matches are not penalized by near-zero BM25 text scores under the default 0.7/0.3 hybrid weights.

## Best Possible Solution

Review and land, revise, or reject https://github.com/openclaw/openclaw/pull/92196; close this issue only after that fix or another canonical solution reaches main.

## Reproduction Assessment

## Solution Assessment

Yes. The best fix belongs at the memory-core hybrid fusion boundary using the existing multimodal classifier, and the linked PR follows that shape without adding config or persisted schema surface.
