## Summary

Keep open. Current main still drops image/audio modality before builtin memory search ranking and applies the same weighted vector-plus-BM25 formula to every hybrid candidate, so multimodal vector hits with no lexical match remain penalized under the documented defaults.

## What This Changes

The issue requests that builtin hybrid memory search rank image/audio multimodal chunks fairly instead of discounting them for missing BM25 text matches.

## Best Possible Solution

Carry explicit chunk modality through schema, persistence, and search result contracts, then rank image/audio vector candidates without weakening normal text hybrid scoring.

## Reproduction Assessment

Yes, source-level. Current main creates multimodal entries with modality, stores and searches them without modality, then ranks vector-only candidates as vectorWeight * vectorScore when textScore is zero.

## Solution Assessment

Yes. The narrow maintainable fix is to carry modality into the ranker; changing global default weights or inferring media type from filenames at merge time would be weaker.
