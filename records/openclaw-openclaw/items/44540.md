## Summary

Keep open. Current main still uses a modality-blind hybrid merge, so vector-only image/audio hits are dampened by the default vector weight when their BM25 score is zero.

## What This Changes

The issue asks for hybrid memory search to avoid BM25 dampening for image/audio chunks indexed through multimodal Gemini embeddings.

## Best Possible Solution

Persist reliable image/audio modality metadata from indexing through stored chunks and search rows, then score non-text vector candidates without BM25 dampening while preserving the existing weighted BM25 plus vector behavior for text chunks.

## Reproduction Assessment

Yes. Source inspection on current main shows image/audio chunks are stored as short label text, modality is not persisted, and vector-only media hits are scored as vectorWeight * vectorScore when textScore is 0.

## Solution Assessment

Yes. The narrow maintainable fix is to carry an explicit modality signal through the memory schema/result path and use vector-only scoring for non-text chunks, rather than weakening BM25 globally or inferring from filenames at the end.

Fix https://github.com/openclaw/openclaw/issues/44540. Current hybrid memory search penalizes Gemini multimodal image/audio chunks because their BM25 text score is usually 0 and mergeHybridResults applies the default 0.7 vector weight to every vector-only hit. Implement a narrow bug fix: preserve an explicit modality signal for multimodal chunks from indexing through persisted chunks/search rows/results, then have the builtin hybrid merge score image/audio candidates using their vector score without BM25 dampening. Preserve existing weighted BM25 + vector behavior for text chunks, do not add a new config option, and do not solve this by changing global default weights or filename-only heuristics. Add focused regression tests for hybrid ranking of a non-text vector hit and for modality persistence/search plumbing. Add a CHANGELOG.md fix entry because this is user-facing. Stop and report if the implementation requires a product-policy change or a broad memory schema redesign.
