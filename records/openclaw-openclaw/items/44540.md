## Summary

Keep open. Current main and v2026.5.12 still match the reported source-level behavior: image/audio modality is known during indexing, but it is not persisted into search rows, and hybrid scoring still downweights vector-only media hits by the default vector weight.

## What This Changes

Make default hybrid memory search rank Gemini-indexed image/audio chunks fairly by avoiding a meaningless BM25 penalty for non-text media.

## Best Possible Solution

Persist chunk modality through the built-in memory schema/search rows, refresh existing media rows safely, and score image/audio chunks with vector-only ranking while preserving normal BM25+vector weighting for text chunks.

## Reproduction Assessment

Yes, at source level. Current main classifies image/audio before indexing, drops modality before persistence/search, and scores vector-only candidates as vectorWeight * vectorScore when textScore is zero.

## Solution Assessment

Yes. Carrying modality into the ranker is the narrow maintainable fix; changing global hybrid weights or guessing media type from paths would either weaken text search or be brittle.
