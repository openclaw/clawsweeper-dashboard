## Summary

Current main still exhibits the reported ranking defect: hybrid fusion discounts vector-only multimodal matches by the configured text weight, while existing coverage only searches the synthetic media-type labels and does not exercise cross-modal semantic retrieval. This remains a focused, actionable memory-core bug rather than work owned by the broader multimodal feature issue.

## What This Changes

Make hybrid memory scoring modality-aware so semantically relevant image and audio chunks are not penalized for lacking meaningful BM25 text matches.

## Best Possible Solution

Carry the existing file-modality fact into the canonical hybrid candidate model and renormalize configured weights over signals meaningful for that modality, with mixed text/image/audio coverage proving semantic media retrieval and unchanged text-only ordering.

## Reproduction Assessment

Yes, at source level with high confidence. Current fusion structurally discounts semantic-only media candidates by the text weight, while the existing integration test avoids the reported case by querying the synthetic `image` and `audio` labels.

## Solution Assessment

No, the suggested direct `finalScore = vectorScore` branch is only a starting sketch because it may put media and text results on inconsistent scales. Modality-aware weight renormalization in the existing fusion path is the narrower and safer fix.
