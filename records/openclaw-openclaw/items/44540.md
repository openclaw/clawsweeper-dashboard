## Summary

Keep open. Current main still knows image/audio modality before indexing, but the persisted chunks/search rows do not carry it into hybrid ranking; vector-only media hits get `textScore: 0` and are still weighted by the default hybrid formula, so the reported ranking bug remains source-reproducible.

## What This Changes

The issue asks for multimodal image/audio memory chunks to rank fairly under default hybrid search instead of being penalized for near-zero BM25 text matches.

## Best Possible Solution

Persist media modality through the builtin memory schema and search-result pipeline, then score image/audio vector hits without BM25 dilution while preserving current hybrid weighting for text chunks.

## Reproduction Assessment

Yes, at source level. Current main classifies image/audio entries before indexing, drops modality from persisted/search rows, and then gives vector-only media hits `textScore: 0` in the default weighted hybrid formula.

## Solution Assessment

Yes. Carrying persisted modality into the ranker is the narrow maintainable fix; changing global hybrid weights would weaken text search, and inferring modality from file paths would be brittle.
