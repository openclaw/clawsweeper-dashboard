## Summary

Keep open: current main and v2026.6.10 still use uniform hybrid scoring that discounts vector-only image/audio candidates, while the focused closing fix remains open at https://github.com/openclaw/openclaw/pull/92196.

## What This Changes

The issue asks for hybrid memory search to stop penalizing vector-only image/audio results with the BM25 text-weight component when multimodal memory indexing is enabled.

## Best Possible Solution

Review and land, revise, or reject https://github.com/openclaw/openclaw/pull/92196; close this issue only after a fix reaches main or another canonical solution replaces it.

## Reproduction Assessment

Yes. Source inspection shows vector-only candidates enter hybrid fusion with textScore 0 and are scored with the default 0.7 vector plus 0.3 text formula; I did not run a live Gemini-keyed scenario.

## Solution Assessment

Yes. The active PR targets the fusion boundary using existing multimodal classification without adding config or schema surface; the issue should remain open until that PR merges or maintainers choose another fix.
