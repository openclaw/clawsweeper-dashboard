## Summary

Keep open. Current main still documents default hybrid ranking and multimodal indexing, preserves modality during file-entry construction, but drops that modality before persistence/search and applies the same weighted BM25+vector score to vector-only media hits.

## What This Changes

The issue requests modality-aware hybrid memory-search ranking so indexed image/audio chunks can surface under default BM25+vector weights.

## Best Possible Solution

Persist chunk modality through the built-in memory schema/search rows, refresh media rows safely, and score image/audio chunks by vector relevance while preserving BM25+vector weighting for text chunks.

## Reproduction Assessment

Yes, at source level. Current main can build multimodal entries with modality, but persisted chunks/FTS/search result shapes omit modality and mergeHybridResults gives vector-only hits textScore 0 under the default weighted formula.

## Solution Assessment

Yes. Carrying modality into persisted search rows and using it in the ranker is the narrow maintainable fix; changing global weights would weaken text search, and inferring modality from paths would be brittle.
