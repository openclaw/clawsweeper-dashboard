## Summary

Keep open. Current main and the latest release still match the reported behavior: media modality is known while indexing, but it is not persisted or returned to the hybrid ranker, which still scores vector-only media hits as `vectorWeight * vectorScore` under the default hybrid weights.

## What This Changes

The issue asks for multimodal image/audio memory chunks to avoid being penalized by the BM25 component of default hybrid memory search.

## Best Possible Solution

Persist chunk modality through the built-in memory schema/search rows and score image/audio chunks with vector-only ranking while preserving normal BM25+vector weighting for text chunks.

## Reproduction Assessment

Yes, at source level. Current main classifies image/audio files before indexing, drops modality before persistence/search, and then scores vector-only candidates as `vectorWeight * vectorScore` when `textScore` is zero.

## Solution Assessment

Yes. Carrying modality into the ranker is the narrow maintainable fix; changing global weights or guessing media type from paths would weaken text search or be brittle.
