## Summary

Keep open. Current main still loses image/audio modality before hybrid ranking, so vector-only media hits are merged with `textScore: 0` and diluted by the default weighted BM25 component.

## What This Changes

Fix hybrid memory search so indexed image/audio chunks can rank fairly under default hybrid BM25 + vector scoring.

## Best Possible Solution

Persist media modality through the memory schema and search-result pipeline, then score image/audio vector hits without BM25 dilution while preserving current hybrid weighting for text chunks.

## Reproduction Assessment

Yes, at source level. Current main classifies image/audio entries before indexing, drops modality from persisted/search rows, and gives vector-only hits `textScore: 0` in the default weighted hybrid formula.

## Solution Assessment

Yes. Carrying persisted modality into the ranker is the narrow maintainable fix; changing global hybrid weights would weaken text search, and inferring modality from file paths would be brittle.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/44540. Current main classifies image/audio memory files before indexing, but persisted chunks/search rows do not retain modality, so hybrid ranking scores vector-only media hits as vectorWeight * vectorScore + textWeight * 0 under default BM25+vector weights. Add a narrow modality contract to the memory index so image/audio modality survives persistence and search result construction, then update hybrid merge/ranking so non-text media vector hits are not diluted by missing BM25 text score while text chunks keep current weighting. Do not change global default weights, do not infer modality from path extensions at rank time, and preserve existing fallback lexical ranking behavior from manager-search.
