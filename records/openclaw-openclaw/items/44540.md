## Summary

Keep open. Current main still indexes Gemini multimodal files as short label text, drops the image/audio modality before persistence/search, and applies the same weighted vector/BM25 merge to every result, so the reported non-text ranking penalty remains. The nearby open keyword-fallback PR is adjacent memory-search work, but it does not address modality-aware hybrid scoring.

## What This Changes

The issue asks for multimodal memory search results to avoid BM25 dampening under default hybrid scoring so image/audio vector hits can surface.

## Best Possible Solution

Persist reliable image/audio modality metadata from indexing through stored chunks and search rows, then have the builtin hybrid merge score non-text vector candidates without BM25 dampening while preserving the existing weighted BM25 + vector behavior for text chunks.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current main stores image/audio chunks as short label text, does not persist modality, and lowers vector-only media hits by multiplying their vectorScore by the default 0.7 weight when textScore is 0.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix, but it should carry an explicit modality signal through the memory schema/result path instead of inferring media type from filenames only at final ranking.
