## Summary

Keep open. Current main and the latest stable release still use a modality-blind weighted hybrid merge, while persisted memory rows do not carry image/audio modality, so the reported ranking penalty remains a valid memory-core bug.

## What This Changes

The issue asks `memory_search` hybrid ranking to stop BM25/text-score dampening for image/audio chunks indexed via multimodal memory.

## Best Possible Solution

Persist explicit chunk modality through the memory schema/result path and make hybrid scoring use vector-only ranking for non-text multimodal chunks while leaving text hybrid behavior unchanged.

## Reproduction Assessment

Yes, source-level. On current main, multimodal entries become label-text chunks with modality dropped before persistence, and vector-only hits are scored as `vectorWeight * vectorScore` when `textScore` is 0.

## Solution Assessment

Yes. The narrow maintainable fix is to carry an explicit modality signal through indexing, storage, and search rows, then score non-text vector candidates without weakening BM25 globally or inferring from filenames at merge time.
