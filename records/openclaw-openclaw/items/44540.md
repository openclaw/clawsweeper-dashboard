## Summary

Keep open. Current main still classifies multimodal files before indexing but drops the modality before persistence/search results, and the builtin hybrid ranker still applies the same vector-plus-BM25 weighted formula to all candidates, so image/audio vector hits with no lexical match remain downweighted under the documented defaults.

## What This Changes

The issue requests fair hybrid memory ranking for image/audio multimodal chunks by avoiding the BM25 penalty that applies to text-only chunks.

## Best Possible Solution

Carry explicit chunk modality through schema, persistence, search results, and hybrid ranking, then score image/audio vector candidates without weakening normal text hybrid scoring.

## Reproduction Assessment

Yes, source-level. Current main creates multimodal entries with modality, stores/searches them without modality, and ranks vector-only candidates as vectorWeight * vectorScore when textScore is zero.

## Solution Assessment

Yes. Carrying modality into the ranker is the narrow maintainable fix; changing global default weights or guessing media type from paths at merge time would be weaker.
