## Summary

Keep open: current main and v2026.6.10 still implement `models status --probe` as a live embedded model run with `maxTokens` and only a per-target run timeout, not a whole-probe wall-clock cap or refresh-only auth check. The reporter's timing evidence matches the current source shape, while the open related PR only covers adjacent external CLI auth discovery.

## What This Changes

The issue asks to make `openclaw models status --probe` cheaper and timeout-bounded for auth-health use, document its real API-call cost, investigate auth-command startup overhead, and consider a refresh-only auth command.

## Best Possible Solution

Split the work into a narrow probe-timeout/cost-doc fix first, then separately decide whether OpenClaw should add a refresh-only auth command and a deeper startup-performance investigation.

## Reproduction Assessment

Yes, at source level: current main performs a live embedded model run with `maxTokens` for each probe target and has no outer wall-clock cap around the full probe. I did not run the live Anthropic/Windows repro in this read-only review.

## Solution Assessment

Unclear for the full issue: honoring a wall-clock cap and documenting the real completion cost is a narrow fix, but adding `models auth refresh` and changing probe cost semantics need maintainer product direction.
