Keep open: current main and the latest release still run `models status --probe` through a live embedded model completion with the timeout passed into each target, not as a whole-probe wall-clock cap. ## What This Changes

The issue requests cheaper model auth/status checks, whole-probe `--probe-timeout` enforcement, a refresh-only auth path, and documentation that probes perform real provider calls.

## Best Possible Solution
