## Summary

Keep open: the self-approval bypass is source-reproducible on current main and the PR targets the right gateway seam, but it is not merge-ready because it lacks real behavior proof and its tests/guard are stale against current approval visibility and trusted-backend replay behavior.

## What This Changes

The PR adds a same-connId rejection guard to `exec.approval.resolve`, adds focused self-approval tests, and updates node.invoke approval-bypass tests to use a separate resolver socket.

## Best Possible Solution

Refresh the gateway approval hardening so requester self-allow is rejected while current device/client/internal approval visibility and trusted-backend replay contracts remain intentional and covered.

## Reproduction Assessment

Yes, by source inspection. Current main stores `requestedByConnId` on exec approval requests and resolves approvals without comparing it to the resolving `client.connId`; I did not run tests because this was a read-only review.

## Solution Assessment

No as submitted. The resolve validation seam is appropriate, but the branch needs to account for current approval visibility, preserve or deliberately change trusted-backend replay, clean pending timers in tests, and provide real behavior proof.

## Review Findings

Overall confidence: 0.9

- **[P2] Align allow-path tests with current approval visibility:** `src/gateway/server-methods/exec-approval-self-approval.test.ts:72-73`
  - body: `createPending()` only sets `requestedByConnId`, then the test expects a different bare `connId` to resolve the record. On current main that record is visible only to the requester connId, so this path returns unknown/expired instead of proving cross-connection approval; add current device/client/admin/internal approval metadata before expecting success.
  - confidence: 0.9
- **[P2] Preserve trusted-backend chat replay approvals:** `src/gateway/server-methods/exec-approval.ts:362-368`
  - body: This rejects every non-deny resolve from the same `connId`, but current chat approval replay requests and resolves on one trusted-backend socket before replaying through another backend socket. Update that flow or explicitly preserve/revise the trusted-backend policy before blocking all same-connection resolves.
  - confidence: 0.88
- **[P2] Clean up rejected approval timers:** `src/gateway/server-methods/exec-approval-self-approval.test.ts:40-41`
  - body: The same-connection rejection tests register approvals with real 30-second timers and return before `manager.resolve()` or `expire()` clears them. Add fake timers, explicit expiry, or per-test cleanup so rejected records do not leave pending handles until timeout.
  - confidence: 0.86
