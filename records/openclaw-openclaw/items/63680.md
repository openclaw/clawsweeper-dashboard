## Summary

Keep open: current main still lacks the requester self-approval guard, but this external security PR is not ready to merge because it lacks real behavior proof and still has two current-main correctness issues in the patch.

## What This Changes

The branch adds a same-WebSocket self-approval guard to `exec.approval.resolve`, adds focused unit tests, and updates node-invoke approval-bypass tests to use separate requester and resolver sockets.

## Best Possible Solution

Land a refreshed gateway approval hardening that rejects requester self-allow decisions, preserves legitimate trusted-backend chat replay, cleans up rejected-path test state, and includes redacted real runtime proof.

## Reproduction Assessment

Yes, by source inspection. Current main records `requestedByConnId` on request, resolves approvals without comparing it to the resolving `client.connId`, and leaves identity enforcement to later node-invoke replay checks.

## Solution Assessment

Unclear as submitted. The guard is the narrow production seam for the same-WebSocket bypass, but the branch needs a current-main chat replay refresh, rejected-path timer cleanup, and real behavior proof before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Handle current chat replay approvals:** `src/gateway/server-methods/exec-approval.ts:358-366`
  - body: This guard rejects every non-deny resolve from the same `connId`, but current main's `requestChatAllowOnceApproval()` still requests and resolves through one trusted-backend WebSocket before replaying through another backend connection. Gateway connections always have a `connId`, so merging this as-is will fail that current-main chat replay path before `node.invoke` is exercised; update the chat replay setup to resolve from a distinct approver connection or explicitly preserve the trusted-backend policy if that is intended.
  - confidence: 0.86
- **[P2] Clean up rejected approval timers:** `src/gateway/server-methods/exec-approval-self-approval.test.ts:39-41`
  - body: `createPending()` registers each fake approval with a real 30-second timer. In the same-connection rejection cases, validation returns before `manager.resolve()` clears that timer, leaving pending records and open timer handles until timeout; expire the records in cleanup, use controlled timers, or avoid registering long-lived timers for rejected-path tests.
  - confidence: 0.87
