## What This Changes

The PR adds a same-WebSocket self-approval guard to `exec.approval.resolve`, adds focused unit coverage, and updates node-invoke approval bypass tests to resolve approvals from a separate socket.

## Best Possible Solution

Land a narrow gateway approval hardening that rejects requester self-allow decisions, preserves legitimate different-connection and self-deny flows, cleans up rejected-path test timers, and includes redacted real runtime proof.

## Reproduction Assessment

Yes. By source inspection, current main records `requestedByConnId` on request, then resolves approvals without comparing it to the resolving `client.connId`; the bypass helper also currently requests and resolves on the same socket.

## Solution Assessment

Unclear as submitted. The production guard is the narrowest fix for the same-WebSocket bypass, but the rejected-path timer leak, missing real behavior proof, and same-connection versus same-client/device boundary need resolution before merge.
