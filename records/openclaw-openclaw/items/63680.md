## Summary

Keep open: current main still lacks the requester self-approval check, so the PR is relevant, but it should not merge as-is because it lacks real behavior proof and has current-main test compatibility and timer-cleanup blockers.

## What This Changes

The branch adds a same-WebSocket requester self-approval guard to `exec.approval.resolve`, adds focused unit tests, and updates node-invoke approval-bypass tests to use separate requester and resolver sockets.

## Best Possible Solution

Land a refreshed gateway approval hardening that rejects requester self-allow decisions, preserves legitimate different-connection and trusted-backend replay flows, cleans up rejected-path test state, and includes redacted real runtime proof.

## Reproduction Assessment

Yes, by source inspection. Current main records `requestedByConnId` on request, resolves approvals without comparing it to the resolving `client.connId`, and has same-socket approval helpers that exercise the bypass shape; I did not run tests because this review is read-only.

## Solution Assessment

Unclear as submitted. The production guard is the narrow core fix for the same-WebSocket bypass, but the PR needs current-main chat replay updates, rejected-path timer cleanup, real behavior proof, and maintainer acceptance of the exact security boundary.
