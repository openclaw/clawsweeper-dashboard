## Summary

Keep this PR open: current main still lacks the same-connection self-approval guard, but the submitted branch is not merge-ready because it is missing real behavior proof and still conflicts with current-main trusted-backend chat approval replay behavior.

## What This Changes

The PR adds a same-WebSocket self-approval guard to `exec.approval.resolve`, adds focused unit tests, and updates node-invoke approval-bypass tests to use separate requester and resolver sockets.

## Best Possible Solution

Land a refreshed gateway approval hardening that rejects requester self-allow decisions, preserves legitimate trusted-backend chat replay or explicitly revises that policy, cleans rejected-path test timers, and includes redacted runtime proof.

## Reproduction Assessment

Yes, by source inspection. Current main records `requestedByConnId` on request, then resolves approvals without comparing it to the resolving `client.connId`.

## Solution Assessment

No as submitted. The guard is the right narrow production seam for the same-WebSocket bypass, but the PR needs a current-main chat replay refresh, rejected-path timer cleanup, and real behavior proof before it is the maintainable fix.
