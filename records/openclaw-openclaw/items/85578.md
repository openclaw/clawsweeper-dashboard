## Summary

Keep this PR open. The hardening is valuable, but it is not merge-ready because the new IPv6 embedded-IPv4 detector still misses legal dotted IPv4-mapped forms such as `::ffff:127.0.0.1`, leaving a concrete SSRF-bypass gap in the security boundary.

## What This Changes

The PR hardens Package Acceptance URL tarball downloads, adds policy-backed `source=trusted-url`, updates workflow/docs, and adds resolver regression tests.

## Best Possible Solution

Fix the dotted embedded-IPv4 parsing gap and add regressions, then land only after maintainers approve the strict `source=url` migration and trusted-source credential policy.

## Reproduction Assessment

Yes. Source inspection plus a focused Node probe of the PR helper logic shows dotted IPv4-mapped loopback text is treated as IPv6 by Node but returns `unsafe=false` in the PR parser.

## Solution Assessment

No, not yet. The strict public `source=url` plus explicit `source=trusted-url` policy is a maintainable direction, but the patch must close the dotted embedded-IPv4 gap and get maintainer approval for the compatibility and credential boundaries.
