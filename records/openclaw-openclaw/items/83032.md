## Summary

Keep open: this protected maintainer-labeled PR is not obsolete, but the current patch still has a blocking budget-accounting bug where selects reserve one action while Telegram renders every option as an inline button. The capability-driven adapter direction looks aligned with the SDK/channel boundary, but it needs revision and real channel proof before merge.

## What This Changes

The PR adds a public presentation limits contract, a central presentation adapter wired into outbound delivery, per-channel limit declarations for Slack, Discord, Telegram, Feishu, and Matrix, SDK exports, docs deprecations, and focused tests.

## Best Possible Solution

Keep the centralized adapter, but make shared budget accounting match each renderer's native output, add a Telegram regression for a max-option select plus an extra button, and land after maintainer review with real channel proof.

## Reproduction Assessment

Yes, source-reproducible: under the PR head, a Telegram presentation with a 100-option select plus one button reserves one action for the select, then Telegram maps the 100 options and surviving button to 101 inline buttons. I did not run live Telegram proof in this read-only review.

## Solution Assessment

No as currently patched: central adaptation is the right architectural place, but the adapter must account for renderer-native output counts before this public SDK/channel contract lands.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Count select options against Telegram action budgets:** `src/channels/plugins/outbound/presentation-limits.ts:337`
  - body: The adapter reserves only one shared action slot for each select, but Telegram renders every select option into an inline button and this PR declares Telegram `maxActions: 100`. A 100-option select plus one button can pass adaptation and still render 101 native buttons, so over-budget Telegram markup can be sent. Count rendered options against the shared budget for Telegram or keep Telegram selects on fallback when they would exceed inline-button capacity.
  - confidence: 0.88
