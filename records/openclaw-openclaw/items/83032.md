## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Treat Telegram button overrides as non-null only:** `extensions/telegram/src/outbound-adapter.ts:215-218`
  - body: `renderPresentation` treats any `telegram.buttons` key as an explicit override, even when the value is `null` or `undefined`. That skips generating buttons from `presentation`, and delivery later strips `presentation`, so optional nullish button fields lose all Telegram controls.
  - confidence: 0.9
- **[P2] Preserve button priority under row-limited layouts:** `src/channels/plugins/outbound/presentation-limits.ts:126-127`
  - body: The global button selection uses `remainingRows * maxActionsPerRow`, but rows are consumed later per authored button block. With `maxRows=1`, an earlier low-priority block can consume the only row before a higher-priority selected button in a later block renders, violating the documented priority behavior.
  - confidence: 0.86
