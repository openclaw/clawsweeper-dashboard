## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Treat Telegram button overrides as non-null only:** `extensions/telegram/src/outbound-adapter.ts:215-218`
  - body: `renderPresentation` treats any `telegram.buttons` key as an explicit override, even when the value is `null` or `undefined`. That skips generating buttons from `presentation`, and delivery later strips `presentation`, so optional nullish button fields lose all Telegram controls.
  - confidence: 0.9
- **[P2] Preserve button priority under row-limited layouts:** `src/channels/plugins/outbound/presentation-limits.ts:386`
  - body: The global button selection uses aggregate row capacity, but rows are consumed later per authored button block. With limited rows, an earlier low-priority block can consume the final row before a selected higher-priority later block renders, violating the documented priority behavior.
  - confidence: 0.86
- **[P2] Route adaptive-card payload sends through injected deps:** `extensions/msteams/src/outbound.ts:76-80`
  - body: When `sendPayload` sees a `presentationCard`, it calls `sendAdaptiveCardMSTeams` directly instead of resolving the injected outbound send dep used by the text and media paths. Environments that inject Teams send shims can therefore fall back to real network delivery for presentation cards.
  - confidence: 0.88
