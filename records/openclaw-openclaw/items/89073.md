## Summary

Keep open for maintainer review: the patch cleanly addresses the linked exec/process fan-out bug, but it intentionally changes same-scope duplicate background exec semantics and maintainers should accept that compatibility tradeoff before merge.

## What This Changes

The PR adds scoped execution-identity matching so duplicate in-flight background-capable `exec` calls can reuse an existing running process session instead of launching another child process.

## Best Possible Solution

Merge after maintainers accept exact duplicate coalescing as the default behavior, or add an explicit duplicate-launch escape hatch if identical concurrent background exec launches must remain supported.

## Reproduction Assessment

Yes, as a source-level reproduction: current main sends repeated background-capable `exec` calls through `runExecProcess`, which creates and registers a new session before the running-result guidance can be reused. I did not run a live Xiaomi/MiMo replay in this read-only review.

## Solution Assessment

Yes, this is the best code layer for the bug: it fixes the generic exec/process runtime boundary after approval/preflight and uses exact resolved execution identity rather than provider-specific Xiaomi/MiMo policy or fuzzy command matching. The remaining question is whether maintainers accept the compatibility tradeoff.
