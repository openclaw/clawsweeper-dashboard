## Summary

Keep open. The requested capability is still unresolved, but this branch is not ready to land because current main has moved web_fetch network exceptions under `ssrfPolicy`, and the PR still has a security-relevant metadata endpoint gap.

## What This Changes

The branch adds a top-level `tools.web.fetch.allowPrivateNetwork` boolean, threads it through `web_fetch` and SSRF resolution/cache keys, hard-blocks selected metadata endpoints, and updates config metadata, tests, and changelog.

## Best Possible Solution

Implement the private-network opt-in, if approved, under `tools.web.fetch.ssrfPolicy` with generated config metadata, docs, cache partitioning, targeted SSRF tests, and a maintainer-approved metadata endpoint policy.

## Reproduction Assessment

Yes. Source inspection shows current main rejects the requested top-level key and only exposes nested narrow `ssrfPolicy` flags; the supplied PR diff shows the branch’s broad private-network mode skips generic link-local blocking while hard-blocking only selected metadata endpoints.

## Solution Assessment

No. The branch is useful source work, but the safer current direction is a nested `ssrfPolicy` opt-in plus an explicit maintained metadata-endpoint policy before exposing broad private-network fetch access.
