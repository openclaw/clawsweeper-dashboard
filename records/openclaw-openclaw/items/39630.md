## Summary

Keep open for maintainer review: current main still lacks the requested broad web_fetch private-network opt-in, but this PR should not land as-is because the API shape and metadata/link-local security policy remain unsettled and the patch leaves an ECS credential endpoint reachable under the new opt-in.

## What This Changes

The PR adds a default-off top-level `tools.web.fetch.allowPrivateNetwork` config option and threads it through web_fetch SSRF checks, cache partitioning, config metadata, tests, and changelog.

## Best Possible Solution

Land or replace this with one canonical default-off `tools.web.fetch.ssrfPolicy` private-network opt-in that preserves existing fake-IP flags, partitions cache entries by the full effective policy, documents the operator trust boundary, and has maintainer-approved metadata/link-local behavior.

No. The requested capability is valid, but this top-level config branch is not the best merge path; maintainers should choose the nested SSRF policy shape and metadata/LAN/redirect threat model first.
