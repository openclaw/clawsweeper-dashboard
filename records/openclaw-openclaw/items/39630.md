## Summary

Keep open: this PR addresses a real web_fetch private-network opt-in gap, but current main has moved to a nested ssrfPolicy shape, maintainer guidance says this branch should not land as-is, and the branch still has a security-relevant metadata endpoint gap.

## What This Changes

The branch adds a top-level tools.web.fetch.allowPrivateNetwork boolean, threads it into web_fetch SSRF resolution and cache keys, hard-blocks selected metadata endpoints, and updates config metadata, tests, and changelog.

## Best Possible Solution

Implement the opt-in under tools.web.fetch.ssrfPolicy with generated config/docs, cache partitioning, targeted SSRF tests, changelog, and a documented metadata-endpoint policy approved by maintainers.

Yes. Source inspection shows current main rejects the requested top-level key and only exposes narrow ssrfPolicy flags; the PR defect is source-reproducible from the branch diff because broad private mode skips generic link-local blocking while only hard-blocking selected metadata addresses.

## Solution Assessment
