## Summary

Keep open: current main still lacks the requested broad web_fetch private-network opt-in, and this PR remains a useful implementation attempt but is not merge-ready because its API shape, security policy, and live proof are unresolved.

## What This Changes

This PR adds top-level `tools.web.fetch.allowPrivateNetwork`, threads it through web_fetch SSRF and cache handling, hard-blocks selected metadata endpoints, and updates config metadata, tests, and changelog.

## Best Possible Solution

Land one canonical default-off web_fetch private-network opt-in under the accepted `ssrfPolicy` surface, with approved metadata endpoint policy, cache partitioning, public docs, generated config metadata, changelog, and targeted SSRF tests.

## Reproduction Assessment

Yes. Source inspection shows current main lacks the broad web_fetch opt-in, and the PR defect is source-reproducible because its broad private mode bypasses generic link-local blocking while hard-blocking only selected metadata addresses.

## Solution Assessment

No. The branch proves the direction is implementable, but the maintainable path is a nested `ssrfPolicy` API plus an explicit maintainer-approved metadata endpoint policy rather than this stale top-level config branch.
