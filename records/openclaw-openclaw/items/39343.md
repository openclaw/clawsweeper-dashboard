## Label Justifications

- `P2`: Fragmented multi-modal turns are a normal-priority workflow problem with meaningful but channel-bounded impact.
- `impact:session-state`: The issue concerns whether related media and text are associated with one coherent agent turn or split across conversation state.

## Summary

This remains the canonical tracker for media-aware multi-part input aggregation: current main now handles Telegram albums, but LINE and shared inbound debounce still dispatch media-bearing events separately, and the safe heuristic semantics require a maintainer product decision.

## What This Changes

Add optional batching of rapid media messages into one agent turn, using native Telegram album identifiers and sender/chat timing heuristics where grouping metadata is unavailable.

## Best Possible Solution

Keep Telegram’s native album grouping, then define an opt-in typed media-aware aggregation policy—shared where semantics are portable and channel-owned where they are not—with sender/chat isolation, attachment preservation, ordering, text-triggered flush, timeout behavior, and false-merge coverage.

## Reproduction Assessment

Yes, at source level. LINE immediately dispatches every media event independently, while the shared debounce policy excludes media-bearing input; no live transport reproduction was performed.

## Solution Assessment

No, a gateway-only media buffer is not the best boundary. Native grouping should remain channel-owned, while heuristic aggregation should use an explicitly approved typed shared or channel contract rather than duplicate buffering policy in the gateway.
