## Summary

Keep open: current main and v2026.6.10 implement the Telegram album/config half, but LINE still processes each downloadable media webhook as a separate turn with no same-sender media buffer or LINE batching config.

## What This Changes

The issue requests gateway/channel-layer media batching so rapid image messages, especially Telegram albums and LINE photo bursts, reach the agent as one coherent turn.

## Best Possible Solution

Implement the remaining behavior inside the LINE plugin boundary with maintainer-approved timing, keying, reply-token, and config semantics; leave the existing Telegram album path unchanged.

## Reproduction Assessment

Yes, at source level. LINE currently creates and dispatches one inbound context per downloadable media webhook, while the shared inbound debounce policy intentionally excludes media-bearing events.

## Solution Assessment

No, not as a generic gateway buffer. The safer maintainable direction is a LINE-plugin-owned media aggregation path with explicit config/default and lifecycle semantics because Telegram album buffering is already implemented in the Telegram plugin.
