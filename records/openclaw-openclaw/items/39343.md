## Label Justifications

- `impact:session-state`: The issue is about whether related media and text stay associated with one coherent agent turn or split across conversation/session context.

## Summary

Keep open: current main and the latest release cover Telegram album buffering, but LINE still dispatches media messages one at a time and the remaining LINE/shared aggregation semantics need a maintainer product decision.

## What This Changes

Request to batch rapid inbound image/media updates, especially Telegram albums and LINE image bursts, into one agent turn instead of separate agent replies.

## Best Possible Solution

Keep this as the canonical tracker and implement media aggregation in the owning channel/plugin boundary: use LINE `imageSet.id` when available, add any heuristic delay only as an approved opt-in, and leave Telegram native album buffering plus shared text debounce semantics intact.

## Reproduction Assessment

Yes, at source level. LINE immediately processes each media event as its own turn, and shared inbound debounce excludes media-bearing events; no live LINE transport run was performed in this read-only review.

## Solution Assessment

No, the proposed gateway-layer buffer is not the best boundary. Native grouping should stay channel-owned, with LINE using `imageSet.id` where available and any time-window fallback gated by a maintainer-approved channel/shared contract.
