## Summary

Keep open. Current main covers the Telegram album-buffering/config half, but LINE still handles each downloadable media webhook event as its own agent turn and exposes no LINE batching option or docs surface.

## What This Changes

The issue requests optional buffering for rapid media messages so Telegram albums and LINE image bursts are forwarded as one agent turn with multiple attachments.

## Best Possible Solution

Implement this in the LINE plugin boundary: optional account-aware media batching, sender/conversation-scoped timers, text/non-media flush behavior, aggregated `allMedia`, schema/types/docs/tests, and no generic gateway change.

## Reproduction Assessment

Yes, at source level. Current main shows LINE creates and processes one `allMedia` array per `MessageEvent`, while Telegram albums already use a configured media-group buffer.

## Solution Assessment

No for the requested gateway-layer placement. The narrower maintainable path is LINE-plugin-local buffering because access checks, replay dedupe, media download, reply-token handling, and context construction already meet there.
