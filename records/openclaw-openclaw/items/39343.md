## Summary

Keep open. Current main and the latest release cover the Telegram album/config half, but LINE still downloads and dispatches each rapid media webhook event as its own agent turn with no LINE batching config or docs surface.

## What This Changes

Request optional media batching so rapid image/media sends are buffered and dispatched as one agent turn, using Telegram album metadata and LINE sender/chat/time heuristics.

## Best Possible Solution

Keep Telegram's existing media-group handling, then add optional LINE-plugin-owned burst coalescing keyed by account, conversation, and sender only after maintainers accept the latency/config semantics.

## Reproduction Assessment

Yes, at source level: multiple downloadable LINE media webhook events from the same sender/conversation each build a fresh allMedia array and call processMessage separately. I did not run a live LINE webhook burst because this was a read-only review.

## Solution Assessment

Unclear. The requested gateway-layer buffer is not necessarily the best boundary; Telegram shows plugin-owned batching works, while LINE heuristic buffering needs a maintainer decision on config/defaults and latency semantics.
