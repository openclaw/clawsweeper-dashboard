## Summary

Keep open. Current main implements the Telegram album-buffering/config portion, but the LINE portion still handles each image webhook event as its own agent turn; the remaining change is a public LINE timing/config behavior that needs maintainer acceptance before implementation.

## What This Changes

Request optional media batching so rapid LINE images or Telegram albums can reach the agent as one turn instead of separate per-image turns.

## Best Possible Solution

Implement an opt-in LINE-plugin-local media batcher with account/conversation/sender-scoped timers, text/non-media flush behavior, aggregated allMedia, schema/types/docs/tests, and no generic gateway policy change.

## Reproduction Assessment

Yes, at source level for the remaining behavior gap. Current main shows LINE builds and dispatches one media context per MessageEvent, while Telegram albums are already buffered and configurable.

## Solution Assessment

No for the requested gateway-layer placement. The narrower maintainable direction is a LINE-plugin-local opt-in buffer because LINE access checks, replay dedupe, media download, reply behavior, and context construction already meet in that handler.
