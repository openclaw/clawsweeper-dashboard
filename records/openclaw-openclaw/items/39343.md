## Summary

Keep open. Current main covers Telegram album buffering and its config key, but the LINE plugin still processes each downloadable media webhook event as its own agent turn and has no LINE batching config or docs surface.

## What This Changes

Request to add optional batching of rapid inbound image/media messages so Telegram albums and LINE image bursts reach the agent as one combined turn.

## Best Possible Solution

Implement optional LINE media batching inside the LINE plugin: account/conversation/sender-scoped buffering, timer and same-key text flushes, aggregation into one LineInboundContext, and no generic gateway or Telegram behavior changes.

## Reproduction Assessment

Yes, at source level. Current main creates a fresh LINE allMedia array per MessageEvent and immediately calls processMessage while handleLineWebhookEvents iterates events one by one.

## Solution Assessment

No for the proposed generic gateway-layer placement. The maintainable path is plugin-local LINE buffering because access gating, replay dedupe, media downloads, reply-token handling, and context construction already meet there.
