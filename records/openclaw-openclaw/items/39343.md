## Summary

Keep open. Current main covers the Telegram album-buffering/config part, but LINE still downloads and forwards each image MessageEvent as its own agent turn with no LINE batching option or docs surface; the remaining work adds timing/config behavior, so it needs maintainer product acceptance before implementation.

## What This Changes

Request optional media batching so rapid image/media messages can be forwarded to the agent as one turn instead of separate turns.

## Best Possible Solution

Implement the remaining behavior inside the LINE plugin boundary with optional account-aware media batching, sender/conversation-scoped timers, text/non-media flush behavior, aggregated allMedia, schema/types/docs/tests, and no generic gateway policy change.

## Reproduction Assessment

Yes, at source level. Current main shows LINE constructs a new allMedia array per MessageEvent and calls processMessage immediately, while Telegram media groups already use a configured buffer.

## Solution Assessment

No for the requested gateway-layer placement. The narrower maintainable solution is LINE-plugin-local buffering because LINE access checks, replay dedupe, media download, reply behavior, and context construction already meet in that handler.
