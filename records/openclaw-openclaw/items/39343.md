## Label Justifications

- `impact:session-state`: The issue concerns whether related media inputs stay associated with one coherent agent turn or split across conversation/session context.

## Summary

Keep open: current main and v2026.6.6 cover the Telegram album/config pieces, but LINE still downloads and dispatches each media webhook as its own agent turn with no LINE-owned media batching surface; the remaining behavior needs maintainer product/config decisions before implementation.

## What This Changes

Request an optional media batching buffer so rapid image/media inputs, especially LINE image bursts and Telegram albums, can be delivered to the agent as one turn.

## Best Possible Solution

Keep this as the canonical remaining tracker and design a LINE-plugin media aggregation path, metadata-first with any heuristic delay/config approved as opt-in and covered by reply-token, redelivery, and multi-account tests.

## Reproduction Assessment

Yes, at source level. LINE currently creates and dispatches one inbound context per downloadable media webhook, while shared inbound debounce intentionally excludes media-bearing messages.

## Solution Assessment

No, the proposed generic gateway-layer buffer is not the best boundary as written. The safer path is a LINE-plugin design that uses LINE image-set metadata where valid and treats fallback heuristics/config as a maintainer-approved product choice.
