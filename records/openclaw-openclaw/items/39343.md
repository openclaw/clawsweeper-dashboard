## Summary

Keep open. Current main solves the Telegram album/config side, but LINE still processes each downloadable image webhook event as its own agent turn and has no LINE media batching setting or docs surface.

## What This Changes

The issue requests optional batching of rapid image/media updates so one user burst reaches the agent as a single turn instead of separate turns.

## Best Possible Solution

Keep Telegram's existing album handling, then add optional LINE-plugin-owned media burst coalescing keyed by account, conversation, and sender after maintainers settle latency and reply-token semantics.

## Reproduction Assessment

Yes, at source level. Multiple downloadable LINE media webhook events from the same sender/conversation each create a fresh allMedia array and call processMessage separately; I did not run a live LINE burst in this read-only review.

## Solution Assessment

Unclear. The requested gateway-layer buffer is not the best-proven boundary; the maintainable direction appears to be LINE-plugin-owned optional batching, with any shared core contract decided separately.
