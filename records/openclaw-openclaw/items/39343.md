## Summary

Keep open. Current main and the latest shipped tag still handle LINE downloadable media one webhook event at a time, while the Telegram album/config portion is already implemented; the remaining LINE or shared media-batching behavior is a user-facing feature/config decision rather than a cleanup close.

## What This Changes

Request optional media batching so rapid image/media sends, especially Telegram albums and LINE bursts, are delivered to the agent as one turn instead of separate replies.

## Best Possible Solution

Make an explicit product call, then either add LINE-plugin media burst coalescing keyed by account/conversation/sender or extend the shared messages.inbound contract for media-aware batching without weakening existing Telegram album behavior.

## Reproduction Assessment

Yes, at source level: send multiple LINE downloadable media webhook events from the same sender/conversation and current code constructs a fresh allMedia array and calls processMessage for each event. I did not run a live LINE webhook burst.

## Solution Assessment

Unclear. Telegram’s plugin-owned album buffering supports a plugin-boundary solution, but the requested gateway-layer and LINE heuristic behavior changes user-visible latency/config semantics and needs maintainer direction before implementation.
