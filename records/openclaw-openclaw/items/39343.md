## Summary

Keep open. Current main and v2026.5.20 cover the Telegram album/config half of the request, but LINE still handles each downloadable media webhook event as its own agent turn with no LINE batching config or docs surface.

## What This Changes

The issue requests optional media batching so rapid image sends, especially Telegram albums and LINE image bursts, reach the agent as one turn.

## Best Possible Solution

Keep Telegram as implemented, then add optional LINE-plugin-owned media burst coalescing keyed by account, conversation, and sender, using shared SDK/core changes only if the LINE plugin cannot express the behavior cleanly.

## Reproduction Assessment

Yes, at source level. Two downloadable LINE media webhook events from the same sender/conversation each create a fresh allMedia array and call processMessage separately; I did not run a live LINE burst in this read-only review.

## Solution Assessment

Unclear as proposed. A generic gateway-layer buffer is not the best-proven boundary; the safer direction is an optional LINE-plugin-owned buffer after maintainers settle latency, reply-token, and batching-scope semantics.
