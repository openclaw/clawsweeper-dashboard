## Summary

Keep open. Current main covers the Telegram album/config half, but the LINE half is still unresolved: LINE media webhook events are still processed one event at a time, and there is no LINE media batching config or docs surface.

## What This Changes

Request optional media batching for rapid image/media messages so related Telegram album or LINE image-burst updates reach the agent as one turn.

## Best Possible Solution

Implement the remaining behavior inside the LINE plugin with an optional media batching delay that aggregates same-account, same-conversation, same-sender media before building one inbound context, while leaving Telegram and core gateway contracts unchanged.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is two allowed LINE image events from the same account, conversation, and sender: each event enters handleMessageEvent with a fresh allMedia array and calls processMessage separately.

## Solution Assessment

No for the proposed generic gateway-layer placement. The narrower maintainable solution belongs in the LINE plugin, where access gating, replay dedupe, media downloads, reply-token fallback, and context construction already meet.
