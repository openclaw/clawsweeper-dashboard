## Summary

Keep open: current main and v2026.6.8 cover Telegram album buffering/config, but LINE still handles each downloadable media webhook as its own agent turn with no LINE batching config or buffer.

## What This Changes

The issue requests optional gateway/channel media batching so rapid LINE images and Telegram albums arrive as one agent turn instead of separate replies.

## Best Possible Solution

Keep this as the canonical remaining tracker and design a LINE-plugin media aggregation path with maintainer-approved timing, keying, reply-token, and config semantics; leave the implemented Telegram album path unchanged.

## Reproduction Assessment

Yes, at source level: LINE currently creates and dispatches one inbound context per downloadable media webhook, while shared inbound debounce intentionally excludes media-bearing messages.

## Solution Assessment

Unclear until maintainers choose the product contract. A generic gateway buffer is not the best boundary as written; the safer shape is LINE-plugin-owned batching with explicit config/default and lifecycle semantics.
