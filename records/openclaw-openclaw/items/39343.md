## Summary

Keep open. Current main already buffers Telegram albums by `media_group_id`, but LINE still handles each downloadable media webhook as its own fresh agent turn with no same-sender media buffer or LINE config/docs surface.

## What This Changes

Request media batching so rapid LINE images and Telegram albums can be aggregated into one agent turn instead of producing separate agent replies.

## Best Possible Solution

Implement the remaining behavior inside the LINE plugin boundary, with config/docs/changelog updates, while leaving Telegram follow-ups to the narrower #55216 and #76149 threads.

## Reproduction Assessment

Yes. A source-level reproduction is two LINE image `message` webhook events from the same account/source: current `handleLineWebhookEvents` sends both through `handleMessageEvent`, and each call builds and processes a separate one-item `allMedia` context.

## Solution Assessment

No for the proposed generic gateway-layer placement. The narrow maintainable solution is LINE-plugin-scoped buffering using existing media/context seams and LINE image-set metadata when available.
