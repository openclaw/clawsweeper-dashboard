- `impact:session-state`: The issue is about whether related media inputs stay associated with one coherent agent turn or split across conversation/session context.

## Summary

Keep open: current main and v2026.6.6 cover the Telegram album/config half, but LINE still processes each downloadable media webhook as its own agent turn and the remaining batching behavior needs a maintainer product/config decision.

## What This Changes

The issue requests gateway-layer media batching so rapid image/media messages are delivered to the agent as one turn, using Telegram media-group IDs and LINE same-sender/time-window grouping.

## Best Possible Solution

Keep this as the canonical remaining tracker and implement LINE-owned media aggregation, preferably metadata-first with any heuristic delay/config approved as opt-in and covered by reply-token, redelivery, and multi-account tests.

## Reproduction Assessment

Yes, at source level. LINE currently creates and dispatches one inbound context per downloadable media webhook, while shared inbound debounce intentionally excludes media-bearing messages.

## Solution Assessment

No, the proposed generic gateway-layer buffer is not the best boundary as written. The safer path is a LINE-plugin design that uses LINE image-set metadata where valid and treats fallback heuristics/config as a maintainer-approved product choice.
