## Summary

Keep open. Current main handles the Telegram album/config half, but the LINE half remains unresolved: each LINE media webhook event still builds its own media context and starts its own turn.

## What This Changes

The issue requests optional batching of rapid image/media updates so one user media burst is delivered to the agent as a single turn.

## Best Possible Solution

Implement LINE-plugin-scoped media batching using the existing inbound debounce/media-context seams, leaving Telegram behavior and core gateway contracts unchanged.

## Reproduction Assessment

Yes. Source-level reproduction is two allowed LINE image message events from the same account, conversation, and sender: each event reaches handleMessageEvent, gets a fresh allMedia array, and calls processMessage separately.

## Solution Assessment

No for the proposed generic gateway-layer placement. The narrow maintainable fix belongs in the LINE plugin, where access gating, replay dedupe, media downloads, reply tokens, and context construction already meet.
