## Summary

Keep open. Current main covers Telegram album buffering and configuration, but LINE still dispatches each image MessageEvent immediately and lacks an opt-in batching config/docs surface; the remaining behavior adds public timing/config semantics, so it needs maintainer product acceptance before implementation.

## What This Changes

Request optional media batching so rapid Telegram albums or LINE image bursts reach the agent as one turn with multiple attachments.

## Best Possible Solution

Implement the remaining behavior inside the LINE plugin boundary as an opt-in media batcher keyed by account, conversation, and sender, flushing on timer or before same-key text/non-media work while leaving gateway core generic and Telegram follow-ups separate.

## Reproduction Assessment

Yes, at source level for the remaining behavior gap: LINE currently builds and dispatches one context per MessageEvent, while Telegram albums are already buffered and configurable. This is not a strict bug repro because the remaining LINE behavior is a new opt-in feature/config surface.

## Solution Assessment

No for the requested gateway-layer placement. The narrower maintainable solution is LINE-plugin-local batching because LINE access checks, replay dedupe, media download, reply behavior, and context construction already meet in the LINE handler.
