## Summary

Keep open: current main and the latest release cover the Telegram album-buffering/config half, but LINE still handles each downloadable media webhook event as an immediate separate agent turn with no LINE batching option or docs surface.

## What This Changes

The issue requests optional media batching for rapid image/media messages, using Telegram album metadata and LINE same-sender time proximity to send one agent turn with all media attached.

## Best Possible Solution

Implement the remaining behavior inside the LINE plugin boundary with optional account/conversation/sender-scoped media batching, timer and same-key text flushes, aggregated allMedia, schema/types/docs/tests, and no generic gateway or Telegram behavior change.

## Reproduction Assessment

Yes, at source level. Current main creates one LINE allMedia array per MessageEvent and immediately calls processMessage while webhook events are iterated one by one; no live LINE webhook run was performed.

## Solution Assessment

No for the requested gateway-layer placement. The narrower maintainable solution is LINE-plugin-local buffering because access gating, replay dedupe, media download, reply-token handling, and context construction already meet there.
