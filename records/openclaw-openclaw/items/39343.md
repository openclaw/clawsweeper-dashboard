## Summary

Keep open. Current main implements the Telegram album buffering/config half, but LINE still handles each downloadable media webhook event as its own agent turn and has no LINE media-batching config or docs surface.

## What This Changes

Request optional media batching for rapid image/media bursts, using Telegram media_group_id and LINE sender/chat proximity, preferably at the gateway layer.

## Best Possible Solution

Implement the remaining behavior inside the LINE plugin boundary with optional account-aware media batching, sender/conversation-scoped timers, text/non-media flush behavior, aggregated allMedia, schema/types/docs/tests, and no generic gateway policy change.

## Reproduction Assessment

Yes, at source level. Current main shows LINE creates and processes one allMedia array per MessageEvent, while Telegram albums already use a configured media-group buffer.

## Solution Assessment

No for the requested gateway-layer placement. The narrower maintainable solution is LINE-plugin-local buffering because access checks, replay dedupe, media download, reply-token handling, and context construction already meet there.
