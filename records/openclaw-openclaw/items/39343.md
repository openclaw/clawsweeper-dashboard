- `impact:session-state`: The issue concerns whether related media inputs stay associated with one coherent agent turn or split across conversation context.

## Summary

Keep open: Telegram album buffering/config is implemented, but current main and v2026.6.10 still process each LINE media webhook independently with no LINE media batching config, so the remaining feature needs maintainer product direction.

## What This Changes

Request optional media/image batching so rapid Telegram album or LINE image bursts reach the agent as one coherent inbound turn.

## Best Possible Solution

Design the remaining behavior inside the LINE plugin: optional same-sender media batching by account and conversation, explicit flush timing/defaults, and preservation of the existing Telegram album path and shared text-only debounce semantics.

## Reproduction Assessment

Yes, at source level: LINE creates and dispatches one inbound context per downloadable media webhook, while shared inbound debounce intentionally excludes media-bearing messages.

## Solution Assessment

No for the generic gateway-layer design as written. The safer solution is LINE-plugin-owned batching with maintainer-approved timing, keying, reply-token, and config semantics.
