## Summary

Keep open. Current main handles Telegram albums, but the LINE side still builds and dispatches one agent turn per webhook media event and has no media-batch delay config/docs surface.

## What This Changes

Add optional media batching so rapid image/media messages can be forwarded to the agent as one media-bearing turn instead of separate turns.

## Best Possible Solution

Implement the remaining behavior inside the LINE plugin boundary, using the existing multi-media context shape and leaving the separate Telegram follow-ups to their own issues.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: two LINE image webhook events enter `handleMessageEvent` separately, each with a fresh `allMedia` array and its own `processMessage` call. I did not live-run LINE webhooks in this read-only review.

## Solution Assessment

Yes for a LINE-plugin-scoped fix, not a core gateway buffer. The narrow maintainable path is an optional LINE media batching delay that aggregates same-account/conversation/sender media before `buildLineMessageContext`, while preserving existing Telegram media-group behavior.
