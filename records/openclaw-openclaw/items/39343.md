## Summary

Keep open. Current main and v2026.5.19 cover the Telegram album/config half, but LINE still handles each downloadable media webhook event as its own agent turn and has no LINE media batching config or docs surface.

## What This Changes

Request optional media batching so rapid image/media messages, especially Telegram albums and LINE image bursts, reach the agent as one combined turn.

## Best Possible Solution

Keep Telegram's existing media-group handling, then add optional LINE-plugin-owned burst coalescing keyed by account, conversation, and sender after maintainers accept the latency and reply-token semantics.

## Reproduction Assessment

Yes, at source level. Multiple downloadable LINE media webhook events from the same sender/conversation each create a fresh allMedia array and call processMessage separately; I did not run a live LINE webhook burst in this read-only review.

## Solution Assessment

Unclear. The requested gateway-layer buffer is not necessarily the best boundary; current Telegram behavior is plugin-owned, and the remaining LINE heuristic needs maintainer decisions on config, defaults, latency, and reply-token behavior.
