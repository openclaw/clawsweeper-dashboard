## What This Changes

The issue reports that `openclaw gateway install` writes a macOS service PATH that omits the active OpenClaw CLI bin directory for npm-global installs.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84207, or an equivalent narrow fix, that adds the active OpenClaw CLI bin directory without merging the full ambient PATH.

## Reproduction Assessment

Yes. Current main excludes darwin user bin directories and only passes the selected Node runtime bin as an extra service PATH dir, while the LaunchAgent wrapper sources that generated PATH.

## Solution Assessment

Yes. The narrow maintainable direction is to carry the installer-visible OpenClaw CLI bin directory into the service PATH explicitly while preserving the existing minimal macOS PATH policy.
