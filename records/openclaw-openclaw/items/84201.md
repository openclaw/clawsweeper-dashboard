## What This Changes

The issue reports that `openclaw gateway install` writes a managed service PATH that omits the active OpenClaw CLI bin directory for npm-global and other prefix installs.

## Best Possible Solution

Land one focused fix that carries the active OpenClaw CLI bin directory into the managed service PATH while preserving the minimal PATH policy, then close this issue and the superseded PR after merge.

## Reproduction Assessment

Yes. Current main is source-reproducible because macOS service PATH generation omits user bin dirs by default and gateway install supplies only the selected Node bin directory, matching the reporter's npm-global failure mode.

## Solution Assessment

Yes. The narrow maintainable solution is to add the active OpenClaw CLI bin directory as an explicit managed-service PATH input, not to copy the full ambient shell PATH into launchd.
