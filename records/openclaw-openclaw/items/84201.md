## Summary

Keep open: current main and the latest release still generate the managed gateway PATH from the minimal service policy plus the selected Node runtime bin only, and the linked fix PRs are still open rather than merged.

## What This Changes

Ensure `openclaw gateway install` includes the active `openclaw` binary directory in the generated service environment PATH for npm-global and other prefix-based installs.

## Best Possible Solution

Review and land the focused PATH fix, likely https://github.com/openclaw/openclaw/pull/84475 if maintainers accept that implementation, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Current main source shows macOS service PATH generation excludes user bin defaults and only prepends the selected Node runtime bin, while the launchd wrapper sources that generated PATH before exec.

## Solution Assessment

Yes. The maintainable fix is to add only the active OpenClaw CLI bin directory through the install-plan service PATH input, not to merge the full ambient shell PATH; the open PR at https://github.com/openclaw/openclaw/pull/84475 appears to follow that direction.
