## What This Changes

The issue reports that macOS gateway service env generation omits the active OpenClaw CLI bin directory from PATH for npm-global and other prefix installs.

## Best Possible Solution

Land a focused fix that adds the active OpenClaw CLI bin directory as an explicit service PATH input while preserving the minimal macOS PATH policy and aligning service audit expectations.

## Reproduction Assessment

Yes, source-reproducible. Current main omits macOS user bin defaults and only prepends the selected Node runtime directory, while the launchd wrapper sources the generated PATH over the inherited environment.

## Solution Assessment

Yes. The narrow maintainable solution is to carry the installer OpenClaw CLI bin directory into `extraPathDirs` or an equivalent explicit service PATH input, not to merge the full ambient shell PATH.
