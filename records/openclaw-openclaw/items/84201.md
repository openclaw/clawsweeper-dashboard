## What This Changes

The issue requests that gateway service environment generation include the active `openclaw` CLI binary directory so npm-global and other prefix installs can call `openclaw` from gateway-spawned shells.

## Best Possible Solution

Review and land the linked fix PR, or replace it with an equivalent narrow change that carries the active OpenClaw CLI bin directory into the explicit service PATH without broad ambient PATH merging.

## Reproduction Assessment

Yes. Source inspection on current main shows the generated service PATH only receives the selected Node bin directory, while the wrapper sources that PATH before launching the gateway; I did not run a live macOS npm-global install in this read-only review.

## Solution Assessment

Yes. The narrow fix is to pass the active `openclaw` command's bin directory through the existing explicit `extraPathDirs` path; broad ambient PATH preservation would weaken the current minimal service-env policy.
