## What This Changes

The issue asks for `openclaw gateway install` to include the active OpenClaw CLI binary directory in the generated service PATH for npm-global and other prefix installs.

## Best Possible Solution

Land one focused fix, preferably the active PR that adds only the resolved OpenClaw CLI bin directory and aligns doctor/audit behavior, then close this issue as implemented after merge.

## Reproduction Assessment

Yes, source-reproducible: current main excludes macOS user bin dirs by default and the gateway install plan passes only the selected Node bin into service PATH generation. The reporter also supplied concrete macOS npm-global release evidence, though I did not run a live LaunchAgent install in this read-only review.

## Solution Assessment

Yes, the narrow maintainable solution is to carry the active OpenClaw CLI bin directory as an explicit service PATH input while preserving the minimal macOS PATH policy. Broadly copying the full interactive shell PATH would be a larger product/security decision and is already tracked by the broader launchd PATH issue.
