## Summary

Keep open. Current main still seeds the compaction identifier audit from a sampled, text-only, post-prune view, and the linked closing PR remains open and unmerged, so closing this issue would preempt the active fix path.

## What This Changes

The issue requests that compaction verify opaque identifiers from original messages, tool-call arguments, and tool results survive into summaries, with strict-policy retry and logging when identifiers are lost.

## Best Possible Solution

Land or replace the linked compaction-safeguard fix so the guard scans the complete pre-compaction source set, excludes stripped `toolResult.details`, enforces strict identifier survival, and adds focused regressions.

## Reproduction Assessment

Yes. Source inspection shows an opaque identifier that appears only in `toolCall.arguments` or a tool-result payload will not enter `identifierSeedText`, because current main only extracts string content and `text` blocks before auditing the summary.

## Solution Assessment

Yes. The maintainable fix belongs in the existing compaction safeguard extraction and retry path; this issue should stay open while https://github.com/openclaw/openclaw/pull/75336 is reviewed, landed, replaced, or closed.
