## What This Changes

The issue asks the compaction quality guard to extract opaque identifiers from the original transcript, including tool-call arguments and tool-result content, verify survival in summaries, retry strict losses, and log losses.

## Best Possible Solution

Land or replace the linked compaction-safeguard fix so identifier survival is seeded from the full pre-compaction input, excludes stripped `toolResult.details`, validates strict survival, and includes focused regressions.

## Reproduction Assessment

Yes. Source inspection shows an opaque identifier that appears only in assistant tool-call arguments will not enter `identifierSeedText`, because current main extracts only string content and text blocks before auditing the summary.
