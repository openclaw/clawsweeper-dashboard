## Summary

Keep open. Current main still samples identifier auditing from text-only message content, and the linked implementation PR https://github.com/openclaw/openclaw/pull/75336 is open and unmerged.

## What This Changes

The issue requests compaction quality checks that verify opaque identifiers from original messages, tool-call arguments, and tool results survive in post-compaction summaries.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/75336 with a reviewed fix that extracts identifiers from the full pre-compaction input, excludes stripped toolResult.details, and adds regression coverage for text, tool-call arguments, tool results, and pruning paths.

## Reproduction Assessment

Yes, source-reproducible. An opaque identifier that appears only in a tool-call argument will not enter identifierSeedText because current main only reads string content and text blocks before auditing the summary.

## Solution Assessment

Yes. Keeping this issue open while the linked implementation PR is reviewed is the narrow maintainable path; the fix belongs in the existing compaction safeguard extraction and quality-retry path, not in a new config or product-policy layer.
