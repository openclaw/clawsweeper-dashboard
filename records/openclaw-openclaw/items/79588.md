## What This Changes

The issue asks the compaction quality guard to verify that opaque identifiers from messages, tool-call arguments, and tool results survive compaction summaries, with retry/logging on loss.

## Best Possible Solution

Land or replace a reviewed safeguard fix that validates the final returned summary against the full pre-compaction source set, excluding stripped toolResult details, with regression coverage for tool-call arguments, tool results, dropped/pruned messages, retry behavior, and logging.
