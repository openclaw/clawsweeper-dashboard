## Summary

Keep open. Current main still seeds the compaction quality guard from a sampled text-only view, while the documented strict policy promises opaque identifier preservation and the linked closing PR remains open and unmerged.

## What This Changes

The issue asks the compaction quality guard to verify opaque identifiers from full transcript inputs, including tool-call arguments and tool-result content, survive into post-compaction summaries.

## Best Possible Solution

Land or replace the linked PR with a focused compaction-safeguard fix that extracts identifiers from the full pre-compaction source set, excludes stripped `toolResult.details`, enforces strict identifier survival, logs/retries losses, and adds regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows an identifier that appears only in assistant tool-call arguments will not enter `identifierSeedText`, because the guard samples the last ten messages through a text-only extractor before auditing the summary.

## Solution Assessment

Yes. The narrow maintainable solution is to fix the existing compaction quality guard extraction and retry path; the open linked PR is the active implementation candidate, but it should be corrected and proven before merge.
