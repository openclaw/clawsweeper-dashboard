Keep this PR open: current main still has the undocumented POSIX uid ownership behavior, and the docs-only patch matches that source behavior. The remaining blocker is mergeability rather than patch correctness, because GitHub reports the branch as conflicting/dirty and a maintainer asked for an update against current main.

## What This Changes

This PR updates the CLI secrets guide, gateway secrets guide, and gateway configuration reference to document that SecretRef file/exec provider paths on POSIX normally must be owned by the gateway user's uid and may require allowInsecurePath for trusted root-owned binaries.

## Best Possible Solution

Rebase this focused docs patch onto current main, resolve the docs conflict, and land or replace it so the public docs consistently describe the existing POSIX uid ownership check.

## Reproduction Assessment

## Solution Assessment
