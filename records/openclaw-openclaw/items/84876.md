## Summary

Keep this PR open: the copied live-output proof now covers the intended dry-send guard path, but the patch still adds a transport-wide Telegram content filter that can block unrelated technical messages before delivery.

## What This Changes

The PR adds a Telegram outbound plain-language assertion in `deliverOutboundPayloads` plus tests for blocking a technical product update and allowing a plain customer-language update.

## Best Possible Solution

Keep outbound delivery generic and enforce this copy policy only through explicit product-update metadata or a `message_sending` hook, with coverage proving ordinary technical Telegram messages still send.

## Reproduction Assessment

## Solution Assessment

No. The central transport layer does not know whether a message is a product/customer update, and the existing `message_sending` hook is the narrower supported surface for optional outbound copy policy.
