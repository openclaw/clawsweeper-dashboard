## Summary

Keep open: current main and the latest release still use exact inline-style predicates for web_fetch visibility sanitization, so the reported near-white, same-color, and tiny-font prompt-injection gap remains. The fix needs maintainer/security policy on color and size thresholds before it is a safe cleanup close.

## What This Changes

The issue requests expanding web_fetch HTML sanitization to reject near-invisible CSS such as near-white text, tiny nonzero fonts, and matching foreground/background colors.

## Best Possible Solution

Add conservative CSS declaration handling in the shared sanitizer: normalize common explicit opaque colors, compare explicit foreground/background colors, and apply a maintainer-approved tiny-font or contrast policy with coverage for both basic and Readability extraction paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: HTML with style="color:white", style="font-size:1px", or matching color/background-color does not match the current removal predicates and would survive sanitizeHtml.

## Solution Assessment

No, not as the issue's narrow regex sketch. The safer maintainable direction is the broader normalizer and explicit foreground/background comparison described in discussion, plus a maintainer-approved threshold for tiny text.
