## Summary

Keep open: current main and the latest release still use exact hidden-style predicates, so the reported near-invisible CSS cases remain a valid security-hardening gap that needs maintainer policy for color and tiny-font thresholds.

## What This Changes

Request to extend web_fetch HTML sanitization to strip near-invisible CSS patterns such as near-white text, tiny nonzero font sizes, and same foreground/background colors.

## Best Possible Solution

Add conservative CSS declaration handling in the shared sanitizer: normalize common explicit opaque colors, remove explicit same foreground/background color pairs, and apply a maintainer-approved tiny-font policy with targeted tests for both basic and Readability paths.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to pass sanitizeHtml HTML containing style="color:white", style="font-size:1px", or matching color/background-color; current main’s predicates do not match those declarations, so the payload remains available to extraction.

## Solution Assessment

Unclear as proposed. The issue’s named-regex additions are useful examples, but the maintainable fix is a declaration/color normalizer plus an explicit tiny-font policy chosen by maintainers.
