## Summary

Keep open: the feature is still potentially useful, but the current PR is not mergeable because it exposes raw final reply payloads without the existing conversation-access guard, is stale against current final payload assembly, and lacks contributor real behavior proof.

## What This Changes

This PR adds a public `final_reply_payloads` hook, exports its types, adds a hook-runner method, and calls it from `runReplyAgent` before appending the usage line.

## Best Possible Solution

Define one documented final outbound provenance hook with conversation-access protection or public-payload filtering, place it after all non-usage payload assembly, and cover it with hook/security/reply-runner tests plus real plugin proof.

## Reproduction Assessment

Not applicable: this is a new public plugin hook/API proposal rather than a current-main regression. Source inspection confirms current main still has no `final_reply_payloads` hook.

## Solution Assessment

No, not as submitted. The hook direction may be useful, but it needs the conversation-access boundary fixed, placement updated for current final assembly, documentation/tests, and maintainer agreement on the provenance contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate final payload hooks as conversation access:** `src/plugins/hook-types.ts:282`
  - body: The new hook receives raw `ReplyPayload[]`, which can include `isReasoning` payloads that generic delivery paths suppress. Because the PR adds `final_reply_payloads` as a public hook without adding it to `CONVERSATION_HOOK_NAMES`, non-bundled plugins can register it without `allowConversationAccess`; either guard it like raw conversation hooks or pass only already-public payloads.
  - confidence: 0.9
- **[P2] Run the hook after final status payload assembly:** `src/auto-reply/reply/agent-runner.ts:820`
  - body: Current main appends `trailingPluginStatusPayload` before the usage footer, but this insertion runs immediately after prefix payloads. A plugin using the advertised finalized-payloads-before-usage hook could not inspect or rewrite the trace/status payload; move the hook after trailing status assembly and before `appendUsageLine`.
  - confidence: 0.86
