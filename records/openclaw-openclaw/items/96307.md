## Summary

Keep open: the linked bug is valid and the PR targets the right restart-recovery surface, but the patch is not merge-ready because the new persisted retry counter is never cleared after successful recovery or later real user progress, and the added tests currently fail type checking.

## What This Changes

The PR adds persisted main-session restart-recovery attempt/quarantine fields, increments the counter at restart mark sites, and skips automatic recovery after the retry ceiling.

## Best Possible Solution

Keep the durable retry budget, but make the session-store owner reset recovery attempts and quarantine fields on successful automatic recovery and on real user-driven progress, with focused tests for both paths.
