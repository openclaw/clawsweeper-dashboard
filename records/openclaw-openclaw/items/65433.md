## Summary

Keep open. The branch contains useful NUL hardening, but the Sonnet 4.6 context fallback would break the existing model-only guard pattern, and the Gemini template change overlaps the still-open #67758 catalog/model-version work.

## What This Changes

The PR strips NUL bytes from untrusted inbound JSON string values, changes Gemini CLI forward-compatible template IDs to prefer 3.1 preview IDs with 3.0 fallbacks, and adds default 1M context handling for anthropic/claude-sonnet-4-6.

## Best Possible Solution

Keep the useful NUL hardening, but rework before merge: gate any native Sonnet 4.6 1M fallback on an explicit or otherwise unambiguous Anthropic provider source, coordinate the Gemini template change with #67758, add focused regression tests, and add the required changelog coverage for the user-facing model/status fixes.

## Reproduction Assessment

Yes. The review finding is reproducible by a static unit path: with this diff, resolveContextTokensForModel({ model: "anthropic/claude-sonnet-4-6", fallbackContextTokens: 200000 }) would return the new 1M provider-specific fallback despite the current model-only guard pattern.

## Solution Assessment

No. The NUL sanitizer is narrow, but the Sonnet fallback should follow the existing explicit-provider guard, and the Gemini portion needs to be reconciled with the current catalog/version work in #67758 before this is the best landing shape.
