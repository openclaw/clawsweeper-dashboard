## Summary

Keep open: the NUL hardening is still useful on current main, but this branch is not merge-ready because it bundles that fix with stale/conflicting model-window and Gemini Flash template changes, and it still lacks contributor real behavior proof.

## What This Changes

The PR strips NUL bytes from untrusted inbound metadata JSON strings, changes Gemini CLI preview template lookup IDs, and adds Sonnet 4.6 1M context fallback/status tests.

## Best Possible Solution

Split or rebase to a focused NUL sanitizer change with regression coverage, while leaving Sonnet display behavior on current main and handling Gemini Flash through the narrower open follow-up.

## Reproduction Assessment

Yes, by source inspection. A buildInboundUserContextPrefix call with UntrustedStructuredContext payload strings containing NUL bytes still reaches JSON.stringify through sanitizeUntrustedJsonValue on current main; I did not run a live spawn repro in this read-only review.

## Solution Assessment

No. The sanitizer boundary is the right fix, but the mixed PR is not the best merge shape because the Sonnet fallback bypasses current explicit-provider safeguards and the Gemini Flash template change conflicts with the upstream-backed current mapping.
