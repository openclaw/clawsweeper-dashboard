## Summary

Keep open. The structured inbound NUL hardening is still useful on current main, but this PR should not merge as-is because the Sonnet and Gemini hunks conflict with newer main behavior and the branch still lacks real behavior proof.

## What This Changes

The PR strips NUL bytes in untrusted inbound JSON metadata strings, changes Google Gemini CLI forward-compat template IDs, and adds Sonnet 4.6 1M context defaults and status coverage.

## Best Possible Solution

Land a narrow rebased change that keeps the arbitrary structured-metadata NUL sanitization with focused regression coverage, while dropping or reconciling the Sonnet and Gemini hunks against current main and the related Gemini PR.

## Reproduction Assessment

Yes, by source inspection. Arbitrary UntrustedStructuredContext string payloads still flow through sanitizeUntrustedJsonValue to JSON.stringify without stripNullBytes on current main, while the existing NUL test only covers known metadata fields.

## Solution Assessment

No. The sanitizer boundary is the right fix, but the PR should be narrowed because current main already has the safer explicit-provider Sonnet behavior and the Gemini Flash direction should remain gemini-3-flash-preview.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate the Sonnet fallback on an explicit provider:** `src/agents/context.ts:432-434`
  - body: The added fallback runs after provider/model parsing even when the provider was inferred from a slash-containing model string. Model-only IDs can come from routers, so this can overstate the context window as 1M; keep the fallback behind an explicit Anthropic or Claude CLI provider like current main does.
  - confidence: 0.9
- **[P2] Keep Gemini Flash templates on the working Flash id:** `extensions/google/provider-models.ts:27`
  - body: This prefers gemini-3.1-flash-preview for Gemini CLI forward-compat templates, but current main normalizes that legacy Flash form to gemini-3-flash-preview and the related Gemini PR tracks the same direction. Reintroducing the 3.1 Flash preference can pick the wrong template for Flash clones.
  - confidence: 0.86
