## Summary

Keep this PR open, but it should not merge as-is: current main already has safer Sonnet 4.6 handling and a different Gemini Flash direction, while the remaining NUL hardening is still useful and should be split or rebased.

## What This Changes

The branch strips NUL bytes from inbound untrusted JSON strings, changes Google Gemini CLI template fallback IDs, and adds Sonnet 4.6 1M context defaults/status coverage.

## Best Possible Solution

Land a narrow rebased branch that keeps the structured inbound NUL sanitization with focused regression coverage, while dropping or reconciling the Sonnet and Gemini hunks against current main and https://github.com/openclaw/openclaw/pull/67758.

## Reproduction Assessment

Yes, by source inspection. The PR's Sonnet regression follows from resolveContextTokensForModel accepting model-only slash IDs, and the remaining NUL gap follows from UntrustedStructuredContext payload strings reaching sanitizeUntrustedJsonValue without stripNullBytes.

## Solution Assessment

No. The NUL sanitizer is the right boundary, but the Sonnet fallback should keep main's explicit-provider guard and the Gemini Flash template should follow the current working gemini-3-flash-preview direction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate the Sonnet fallback on an explicit provider:** `src/agents/context.ts:432-434`
  - body: The new Sonnet fallback runs after provider/model parsing even when the provider was inferred from a slash-containing model string.   - confidence: 0.9

  - confidence: 0.84
