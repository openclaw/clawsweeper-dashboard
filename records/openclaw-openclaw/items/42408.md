## Summary

Keep open: current main and the latest release still expose inclusion-only memory extra paths, no public indexing exclusion surface, and only missing/inaccessible-path diagnostics, so the path-hygiene and benchmark-contamination request is not implemented. The closed exclusion PR did not merge, and the open narrower exclusion and QMD traversal issues do not fully supersede this issue's docs and likely-workspace-drift warning request.

## What This Changes

Request docs, optional memory indexing exclusions, and stronger diagnostics for memorySearch.extraPaths path drift and benchmark-file contamination.

## Best Possible Solution

Define one shared memory indexing exclusion and path-hygiene diagnostics contract, then document and test it across builtin indexing, QMD collections, status output, watcher/reindex state, and memory search docs.

## Reproduction Assessment

No high-confidence end-to-end reproduction of the local/hybrid ranking degradation was established. Source inspection does confirm current main recursively indexes configured extraPaths without a public exclusion surface or likely-workspace-drift diagnostic, so the contamination path is source-reproducible.

## Solution Assessment

Unclear until maintainers choose the public contract. Exclusion/glob support plus docs are plausible, but the maintainable fix needs agreed naming, matcher semantics, defaults, QMD parity, and reindex behavior before implementation.
