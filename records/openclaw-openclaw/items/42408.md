## Summary

Keep open. Current main and the latest release still only cover part of the report by surfacing missing extraPaths in deep status; they do not provide a memory-search exclusion contract or docs guidance for benchmark/queryset contamination, and the related exclusion PR was closed unmerged while narrower follow-up issues remain open.

## What This Changes

The issue requests memory-search path-hygiene docs, optional exclusion patterns for indexing, and stronger warnings when extraPaths drift away from the active workspace.

## Best Possible Solution

Define one shared memory-search path-hygiene contract across builtin indexing, QMD collections, watcher/update behavior, status/docs, and reindex cleanup, while coordinating the narrower exclusion work with https://github.com/openclaw/openclaw/issues/71285.

## Reproduction Assessment

Partly. Source inspection confirms recursive indexing of memory roots and extraPaths, existing missing-path diagnostics, and no exclusion contract, but this read-only review did not live-reproduce the local+hybrid ranking instability.

## Solution Assessment

Unclear. Exclusion support is plausible, but the best fix needs maintainer agreement on config semantics and QMD/builtin parity; docs-only guidance would not remove contaminated files from indexes.
