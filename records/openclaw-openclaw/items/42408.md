## Summary

Keep open: current main and v2026.6.10 still lack a complete memory-search exclusion and path-hygiene contract, and the active `excludePaths` PR covers only the exclusion-config portion rather than the broader `extraPaths` drift, benchmark-contamination, diagnostics, and backend-parity request.

## What This Changes

The issue asks for memory-search path-hygiene documentation, optional indexed-file exclusion support, and stronger diagnostics for misaligned `extraPaths` in local/hybrid memory search.

## Best Possible Solution

Define one documented memory-search path-hygiene contract across schema/types, builtin scanning, QMD behavior or explicit scope, status diagnostics, docs, focused tests, and reindex cleanup.

## Reproduction Assessment

Partly yes at source level: current main has `extraPaths` and recursively indexes eligible Markdown without a user exclusion config, so benchmark/queryset files can enter the corpus if included. I did not reproduce the local+hybrid ranking degradation live with the reported embedding model and corpus.

## Solution Assessment

Unclear. A configurable exclusion plus docs is plausible, but the best solution needs maintainer agreement on public config naming, matcher semantics, QMD/builtin parity, status behavior, and stale-index cleanup.
