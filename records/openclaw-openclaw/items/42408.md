## Summary

Keep open: current main still has inclusion-only memory indexing paths, no public exclusion/glob contract, and only limited diagnostics/docs for the path-drift plus benchmark-contamination workflow. This is useful memory-search product/docs work, but it needs maintainer direction before a repair lane should add a new public config option or warning policy.

## What This Changes

The issue requests memory-search path-hygiene docs, optional indexing exclusions, and stronger diagnostics for local/hybrid retrieval quality problems caused by extraPaths drift and benchmark-file contamination.

## Best Possible Solution

Define a public memory indexing exclusion/glob contract and outside-workspace warning policy, then apply it consistently across config types, schema/help, builtin indexing/read/sync/status, QMD collection mapping, and memory docs.

## Reproduction Assessment

No. Source inspection gives a high-confidence path for the missing exclusion/docs/warning surfaces, but the reported unstable ranking requires a real local/hybrid corpus with contaminated benchmark files to reproduce end to end.

## Solution Assessment

Unclear. excludePaths or glob support is plausible, but the best fix needs a maintainer-approved public config shape and cross-backend warning policy rather than an ad hoc backend-only patch.
