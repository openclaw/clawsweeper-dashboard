## Summary

Keep open. Current main still has include-only `memorySearch.extraPaths` behavior, builtin indexing still recursively admits eligible files from those paths, and the public memory docs still do not provide an exclusion contract or enough benchmark-contamination/path-hygiene guidance to resolve the reported failure mode.

## What This Changes

The issue requests memory-search docs for path hygiene and benchmark contamination, optional indexing exclusions, and stronger diagnostics for misaligned `extraPaths` in local hybrid memory search.

## Best Possible Solution

Add a documented builtin memory-search exclusion mechanism that is part of the config contract, applies consistently to file discovery, memory file reads, watcher filtering, and reindex scope invalidation, then expand the public memory docs with path hygiene, `memory status --deep`, reindex-after-path-change, broad-path noise, and benchmark/evaluation-contamination guidance.

## Reproduction Assessment

Yes for the source-level indexing failure mode: current main recursively includes eligible `.md` files under `memorySearch.extraPaths` and has no exclusion knob, so a benchmark Markdown file inside an extra path will be indexed. A full relevance-score reproduction would require the reporter's corpus and local embedding model, but that is not needed to verify the remaining config/docs gap.

## Solution Assessment

No current-main solution exists yet. The narrow maintainable fix is to add an explicit exclusion contract for the builtin memory indexer plus docs and focused tests, while keeping QMD-specific collection patterns separate unless equivalent semantics are intentionally implemented.
