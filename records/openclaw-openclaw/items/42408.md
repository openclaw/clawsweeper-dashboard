## Summary

Keep open: current main and the latest release still lack a public memory indexing exclusion surface, outside-workspace intent diagnostics, and docs for `extraPaths` hygiene or benchmark/queryset contamination. The open related issues cover important pieces of the exclusion and QMD traversal problem, but they do not fully replace this issue's combined path-drift warning and documentation request.

## What This Changes

The issue requests memory-search docs, an optional indexing exclusion surface, and stronger `extraPaths` diagnostics to reduce local/hybrid retrieval contamination.

## Best Possible Solution

Define one memory indexing exclusion and path-hygiene diagnostics contract, then apply it consistently across schema/types/help, builtin scanning, QMD collections/search/update, watcher/reindex state, status output, docs, and regression coverage.

## Reproduction Assessment

No high-confidence end-to-end reproduction of the local/hybrid ranking drop was established. Source inspection does confirm current main recursively indexes configured `extraPaths` without a public exclusion surface or outside-workspace intent diagnostic, so the contamination/path-hygiene failure mode is source-reproducible.

## Solution Assessment

Unclear until maintainers choose the public contract. Exclusion/glob support plus docs are plausible, but the maintainable fix needs agreed naming, matcher semantics, defaults, and builtin/QMD parity before implementation.
