## Summary

Keep open. Current main and v2026.6.6 only cover the missing-`extraPaths` status diagnostic; they still lack a memory-search exclusion contract and docs guidance for benchmark/queryset contamination, and the related exclusion PR closed unmerged while narrower follow-up issues remain open.

## What This Changes

The issue requests memory-search docs and diagnostics for `extraPaths` drift and benchmark-file contamination, plus an optional exclusion mechanism for memory indexing.

## Best Possible Solution

Define a shared memory-search path-hygiene contract across builtin indexing, QMD collections, watcher/update behavior, status/docs, and reindex cleanup, coordinating the narrower exclusion work with https://github.com/openclaw/openclaw/issues/71285.

## Solution Assessment

Unclear. Exclusion support is plausible, but the best fix needs maintainer agreement on config semantics and QMD/builtin parity; docs-only guidance would not remove contaminated files from indexes.
