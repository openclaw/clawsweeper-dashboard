## Summary

Keep open: current main and the latest release still expose inclusion-only memory extra paths, recursively index configured directories without a public exclusion contract, and only diagnose missing/inaccessible extra paths rather than likely workspace drift or benchmark contamination. The related exclusion PR was closed unmerged, while the narrower open dreaming-exclusion issue does not cover this issue's path-hygiene and benchmark-contamination docs request.

## What This Changes

The issue requests memory-search docs for path hygiene and benchmark contamination, an optional exclusion surface for indexing, and stronger warnings for likely misaligned extraPaths.

## Best Possible Solution

Define one memory indexing exclusion and path-hygiene diagnostics contract, then document and test it across builtin and QMD indexing paths.

## Reproduction Assessment

No high-confidence end-to-end reproduction of the local/hybrid ranking degradation was established. Source inspection does confirm the reported contamination path: current main recursively indexes configured extraPaths without a public exclusion surface or likely-workspace-drift diagnostic.

## Solution Assessment

Unclear until maintainers choose the public contract. Exclusion/glob support plus docs are plausible, but the maintainable fix needs agreed naming, matcher semantics, defaults, upgrade/reindex behavior, and builtin/QMD parity before implementation.
