## Summary

Keep open: current main still indexes Markdown under `extraPaths` without a public exclusion/glob config, and the docs do not cover path-hygiene or benchmark-contamination guidance while status diagnostics only catch missing paths.

## What This Changes

Request docs and config/runtime support to avoid `memory_search` quality drift from misaligned `extraPaths` and benchmark/queryset files entering the memory index.

## Best Possible Solution

Add a versioned memory indexing exclusion contract and consistent docs/diagnostics for `extraPaths` path hygiene and benchmark contamination across builtin and QMD-backed indexing.

## Reproduction Assessment

No. Source inspection confirms configured `extraPaths` are resolved and recursively indexed for Markdown, and status reports missing additional memory paths, but no live local+hybrid ranking benchmark was established on current main.

## Solution Assessment

Unclear. The requested `excludePaths`/glob support and stronger outside-workspace warnings are plausible, but the narrowest maintainable fix needs a decided public config/API shape before implementation.
