## Summary

Keep open. Current main and latest release still provide only the missing-path status diagnostic; they do not provide memory-search exclusion semantics or docs guidance for benchmark/queryset contamination, and the closed exclusion PR was superseded while narrower related issues remain open.

## What This Changes

Requests memory-search path-hygiene docs, optional indexing exclusions, and stronger warnings for misaligned `memorySearch.extraPaths`.

## Best Possible Solution

Define one documented memory-search path-hygiene and exclusion contract across schema/types, builtin indexing/watch, QMD collections/update behavior, status/docs, and reindex cleanup, coordinating narrower exclusion work with https://github.com/openclaw/openclaw/issues/71285 and QMD traversal work with https://github.com/openclaw/openclaw/issues/54463.

## Reproduction Assessment

Unclear for the full reported ranking degradation because no live corpus was run. Source inspection confirms current main still recursively indexes configured memory paths, reports missing paths only as a status diagnostic, and has no exclusion contract.

## Solution Assessment

Unclear. Exclusion support plus docs is plausible, but the best fix needs maintainer agreement on config semantics, QMD/builtin parity, watcher behavior, and reindex cleanup; docs-only guidance would not remove contaminated files from indexes.
