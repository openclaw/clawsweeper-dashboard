## Summary

Keep open: current `main` still has inclusion-only memory extra paths, no backend-consistent exclusion surface, and no likely-workspace-drift warning beyond missing or inaccessible paths. The related exclusion PR is closed unmerged, and the narrower open exclusion/QMD issues do not fully replace this issue's path-hygiene and benchmark-contamination docs request.

## What This Changes

The issue requests memory-search docs for path hygiene and benchmark contamination, an optional exclusion surface for memory indexing, and stronger diagnostics for `extraPaths` workspace drift.

## Best Possible Solution

Define one memory indexing exclusion and path-hygiene diagnostics contract, then document and test it across builtin and QMD indexing paths instead of adding a backend-specific filter.

## Reproduction Assessment

No high-confidence end-to-end reproduction of the local/hybrid ranking degradation was established. Source inspection does confirm current `main` recursively indexes configured `extraPaths` without a public exclusion surface or likely-workspace-drift diagnostic, so the contamination path is source-reproducible.

## Solution Assessment

Unclear until maintainers choose the public contract. Exclusion/glob support plus docs are plausible, but the maintainable fix needs agreed naming, matcher semantics, defaults, and builtin/QMD parity before implementation.
