## Summary

Keep open: current main and v2026.5.12 still have inclusion-only memory indexing, limited extraPath diagnostics, and docs that do not cover exclusion/path-hygiene or benchmark-contamination pitfalls. This remains useful memory-search product/docs work, but the public exclusion config and outside-workspace warning policy need maintainer direction before implementation.

## What This Changes

The issue requests memory-search docs, indexing exclusions, and stronger diagnostics for local/hybrid retrieval quality problems caused by path drift and benchmark-file contamination.

## Best Possible Solution

Define a public memory indexing exclusion/glob contract and outside-workspace warning policy, then apply it consistently across config types, schema/help, builtin indexing/read/sync/status, QMD mapping, and memory docs.

## Reproduction Assessment

No high-confidence end-to-end reproduction is available from this review. Source inspection confirms the missing exclusion/docs/warning surfaces, but the unstable ranking needs a real local/hybrid corpus with contaminated benchmark files.

## Solution Assessment

Unclear. excludePaths or glob support is plausible, but the best fix needs a maintainer-approved public config shape and cross-backend warning policy rather than an ad hoc backend-only patch.
