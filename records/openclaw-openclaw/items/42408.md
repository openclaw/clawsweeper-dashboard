## What This Changes

Requests docs, configurable exclusions, and stronger diagnostics for memory `extraPaths` path drift and benchmark/queryset files polluting local/hybrid `memory_search` results.

## Best Possible Solution

Define one memory indexing exclusion and path-hygiene diagnostics contract, then apply it consistently across schema/types/help, builtin scanning, watcher/reindex state, QMD collections, status output, docs, and regression coverage.

## Reproduction Assessment

No high-confidence end-to-end reproduction of the local/hybrid ranking drop was established. Source inspection does confirm current main recursively indexes configured `extraPaths` without a public exclusion or outside-workspace intent diagnostic, so the contamination failure mode is source-reproducible.

## Solution Assessment

Unclear until maintainers choose the public contract. Exclusion/glob support and docs are plausible, but the durable fix needs agreed names, matcher semantics, defaults, and builtin/QMD parity before implementation.
