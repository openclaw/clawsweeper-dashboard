## Summary

Keep open: current main and v2026.6.8 still lack a memory-search exclusion/path-hygiene contract, the linked exclusion PR is closed unmerged, and https://github.com/openclaw/openclaw/issues/71285 only covers the narrower dreaming-artifact slice.

## What This Changes

The issue asks for memory-search path-hygiene docs, optional index exclusion patterns, and stronger diagnostics for misaligned `extraPaths` that can contaminate local/hybrid retrieval quality.

## Best Possible Solution

Define one documented memory-search path-hygiene and exclusion contract across schema/types, builtin scanning, QMD collections/update behavior, watcher filtering, status output, docs, tests, and reindex cleanup.

## Reproduction Assessment

Partly yes at source level: current main shows no exclusion config and recursive `extraPaths` indexing across builtin and QMD paths. No live reproduction was run for the full local+hybrid quality symptom with the reported model and corpus.

## Solution Assessment

Unclear. Exclusion support plus docs is plausible, but the best fix needs maintainer agreement on config naming, matcher semantics, QMD/builtin parity, default behavior, and stale-index cleanup.
