## Summary

This PR should stay open but is not merge-ready: the CJK false-overflow problem is real, but the branch relaxes tool-result pressure for all payloads and leaves the sibling guard/proof gaps unresolved.

## What This Changes

This PR changes the embedded-agent pre-prompt tool-result token divisor from 2 to 4 and adds CJK/mixed-language estimator tests while adjusting existing overflow test budgets.

## Best Possible Solution

Use a shared script-aware tool-result estimator that fixes CJK text without weakening Latin/JSON pressure, align the pre-prompt and mid-turn guard paths, and add redacted real CJK-heavy agent-run proof.

## Reproduction Assessment

Yes, by source inspection and related prior evidence: current main weights CJK text and then divides tool-result content by 2, which overestimates CJK tool text at about two tokens per character. I did not run a live CJK workflow in this read-only review.

## Solution Assessment

No. The bug is real, but this global divisor change is too broad because it also halves Latin and JSON/object tool-result pressure and does not align the sibling guard path.
