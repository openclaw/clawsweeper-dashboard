## Summary

Keep open: current main still has no canonical workspace-bounded, CWD-relative AGENTS.md/CLAUDE.md bootstrap path for normal OpenClaw agent turns; the related session helper is separate and still security-sensitive, and the remaining work needs maintainer product/security direction.

## What This Changes

The issue requests automatic loading of directory-relative AGENTS.md/CLAUDE.md files from the workspace or project root down to the active CWD, with bootstrap budgets and optional config controls.

## Best Possible Solution

Design one canonical workspace-bounded directory-context path for the normal bootstrap prompt, reuse existing bootstrap budgets, and coordinate it with the open security-boundary and extra-file context work before adding config/default surface.

## Reproduction Assessment

Yes, source-level: the main bootstrap path loads recognized files from workspaceDir and configured hook extras only, while the separate CWD ancestor helper is not wired into the normal bootstrap path and is currently unbounded.

## Solution Assessment

Unclear as proposed: Codex-style root-to-CWD discovery is a reasonable target, but the best OpenClaw solution needs a maintainer decision on defaults/config, workspace bounds, filename precedence, and whether to extend the canonical bootstrap path rather than reuse the session helper as-is.
