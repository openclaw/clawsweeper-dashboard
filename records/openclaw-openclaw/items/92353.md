## Summary

Current main only loads the fixed bootstrap filenames from the workspace root, so the directory-scoped instruction capability is genuinely missing and no merged or viable canonical item supersedes it. Keep this open for a maintainer product decision because OpenClaw separately models workspace and execution CWD, while Codex resolves hierarchy at session initialization and currently does not refresh instructions after a CWD change; the broader first-read, refresh, filename, mode, and configuration choices therefore need a narrower OpenClaw contract before implementation.

## What This Changes

Add deterministic root-to-session-directory discovery of project instruction files such as `AGENTS.md` and `CLAUDE.md` to bootstrap prompt context.

## Best Possible Solution

Define one canonical session project directory at run initialization, resolve a single root-to-directory instruction hierarchy through the existing guarded and budgeted bootstrap pipeline, and explicitly specify filename precedence and whether CWD changes require a new session rather than implicit context refresh.

## Reproduction Assessment

Yes at high confidence for the capability gap: current main directly loads only fixed workspace-root bootstrap files, so nested `AGENTS.md` or `CLAUDE.md` files are not automatically injected. This is not a regression against an existing documented OpenClaw behavior.

## Solution Assessment

No, not in the full proposed shape. Session-initial root-to-directory loading is the most maintainable direction, but first-file inference, mid-session refresh, configurable modes, and default `CLAUDE.md` precedence require explicit product decisions and should not become parallel runtime paths.
