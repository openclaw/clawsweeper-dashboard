## Summary

Keep this open: current main has partial and conflicting context-loading behavior, but the requested safe, workspace-bounded, budgeted directory-relative bootstrap behavior is not settled. The existing sessions helper already scans `AGENTS.md` / `CLAUDE.md` ancestors, but it currently walks to filesystem root and is the subject of an open security-boundary report, while the documented workspace bootstrap path still only guarantees root workspace files.

## What This Changes

The issue requests cwd-aware auto-loading of nested `AGENTS.md` and `CLAUDE.md` files from workspace/project root toward the session working directory.

## Best Possible Solution

Define one canonical, workspace-bounded directory-context contract that shares the existing bootstrap budgets, avoids duplicating native Codex `AGENTS.md` handling, and documents how it relates to the security-boundary and context-elevation follow-ups.

## Reproduction Assessment

Yes, source-reproducible: the workspace bootstrap path only loads fixed root workspace files, and the docs explicitly say arbitrary subdirectory files are not injected. A separate sessions helper does scan ancestors, but it is not the safe, budgeted workspace-root-to-cwd behavior requested here.

## Solution Assessment

Unclear as an implementation path: the direction is plausible, but the best fix needs maintainer choices on defaults, filenames, workspace boundary, and Codex-native duplication. A narrow automated patch should wait until those product/security decisions are settled.
