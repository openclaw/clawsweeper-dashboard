## Summary

Keep open: current main still does not provide the requested CWD-aware directory context in the main workspace bootstrap path, and the separate session resource-loader path is not a safe or complete substitute.

## What This Changes

The issue requests deterministic root-to-CWD discovery and prompt injection of directory-relative `AGENTS.md` / `CLAUDE.md` project instructions.

## Best Possible Solution

Make a maintainer product/security decision, then implement bounded project-root-to-CWD discovery in the existing bootstrap context pipeline with budget guards, docs, and focused tests, coordinating with https://github.com/openclaw/openclaw/issues/39120 and https://github.com/openclaw/openclaw/issues/17810.

## Reproduction Assessment

Yes, source-level: the primary bootstrap resolver takes `workspaceDir` only and the workspace loader enumerates fixed root-level bootstrap files, while the CWD-walk loader is disabled for embedded runs.

## Solution Assessment

Unclear as a final design: root-to-CWD context loading is a good direction, but the best fix needs maintainer choices around defaults, file names, security boundary, and whether to reuse or replace the existing unsafe session loader.
