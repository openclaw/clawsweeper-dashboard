## Summary

Keep open: current main and the latest stable release still lack a workspace-bounded, budgeted CWD-relative AGENTS.md/CLAUDE.md bootstrap policy; the existing session helper is unbounded and the embedded-agent path still loads only fixed workspace-root bootstrap files.

## What This Changes

The issue requests automatic directory-relative AGENTS.md/CLAUDE.md loading from project root to task CWD with deterministic ordering, bootstrap budgets, and configurable filenames/defaults.

## Best Possible Solution

Keep this as the canonical feature discussion, land or replace the workspace-boundary security fix first, then implement one workspace-bounded and budgeted directory-context path with clear defaults, docs, and tests for ordering, bounds, and cache behavior.

## Reproduction Assessment

Yes at source level: embedded runs separate task cwd from workspace bootstrap, current bootstrap only loads fixed workspace-root files, and the standalone CWD-walk helper is unbounded rather than the requested workspace-bounded bootstrap behavior.

## Solution Assessment

Unclear as a final design: CWD-relative loading matches Codex semantics, but OpenClaw still needs maintainer decisions on trusted root boundaries, defaults, filename precedence, budget handling, and coordination with #93368.
