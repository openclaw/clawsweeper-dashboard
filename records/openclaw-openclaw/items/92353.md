## Summary

Keep open: current main still does not implement the requested OpenClaw runtime behavior. The normal bootstrap path remains workspace-root based, while the only CWD-walking helper is a separate session resource-loader path with an open security-boundary concern and no maintainer-approved product decision for defaults or scope.

## What This Changes

Request opt-in CWD-aware discovery and injection of nested `AGENTS.md` / `CLAUDE.md` files from the workspace or project root down to the session CWD.

## Best Possible Solution

Add a bounded directory-context resolver to the shared bootstrap path, walking only from the approved workspace/project root to the session CWD, applying existing bootstrap budgets and deterministic ordering, and coordinate it with the open security boundary report.

## Reproduction Assessment

Yes, source-level. Current main's normal bootstrap path only resolves workspace-root bootstrap files, and the separate CWD-walking helper is not a safe or complete implementation of the requested bounded runtime behavior.

## Solution Assessment

Unclear pending maintainer product direction. The maintainable path is not the existing unbounded session helper; it should be a shared, bounded bootstrap resolver with explicit defaults and security tests.
