## Summary

Current main does not implement directory-relative instruction discovery, and no merged or viable exact replacement owns the work. Keep this open for a product decision because OpenClaw has a workspace-root bootstrap contract but no consistent project/CWD identity, containment boundary, precedence rule, or refresh lifecycle across its agent runtimes.

## What This Changes

Automatically discover and inject project instruction files such as AGENTS.md and CLAUDE.md from the project root through the active working directory.

## Best Possible Solution

Define a canonical project-directory field and lifecycle for applicable sessions, then extend the shared bootstrap resolver to build one deterministic, contained, budgeted root-to-directory instruction chain consumed consistently by embedded, realtime, ACP, spawned, and custom context-engine paths; avoid adding configuration until default semantics prove insufficient.

## Reproduction Assessment

Yes, at source level. Current main resolves bootstrap from the agent workspace, documents that subdirectory files are not automatically injected, and lacks a universal session CWD input to the bootstrap resolver; this is a missing feature rather than a regression in an existing contract.

## Solution Assessment

No, not exactly as proposed. Automatic scoped loading is a reasonable direction, but adding a default-off config block or inferring scope from tool activity before defining the canonical project directory and refresh lifecycle would create competing context behavior.

## Risks / Open Questions

- A project directory inferred from command CWD, spawned CWD, first file read, or workspace root would produce different instruction sets; OpenClaw needs one explicit contract.
- Walking above or outside the agent workspace can admit untrusted prompt instructions unless root anchoring, realpath containment, symlink handling, and allowed filenames are defined.
- Reloading after tool-level directory changes could stale or churn prompt context and caches; resolving only once could surprise users expecting lazy subdirectory behavior.
- The adjacent open PR https://github.com/openclaw/openclaw/pull/74735 may change the shared bootstrap seam and should be considered before implementation.
