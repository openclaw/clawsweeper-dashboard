## Summary

Keep open: current main still lacks an OpenClaw-owned, workspace-bounded directory-context path for this request. The main bootstrap pipeline loads configured workspace-root files, the embedded resource loader disables the session-level ambient context walker, and the existing walker has a separately tracked security-boundary problem rather than being a complete answer.

## What This Changes

The issue requests automatic CWD-relative loading of nested AGENTS.md and CLAUDE.md files into agent bootstrap context.

## Best Possible Solution

Add a workspace-bounded directory-context design to the main bootstrap/context pipeline, coordinate it with https://github.com/openclaw/openclaw/issues/92561, and cover AGENTS.md/CLAUDE.md precedence, limits, docs, and tests without unbounded ancestor traversal.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: the main bootstrap resolver only loads workspace-root bootstrap files, and embedded sessions disable the session resource loader's ambient context files. I did not run a live agent because the gap is clear from the current code and docs.

## Solution Assessment

Unclear as proposed; the feature direction is aligned, but the best implementation needs a maintainer decision on default behavior, project-root anchoring, CLAUDE-family filename support, and the security boundary tracked in https://github.com/openclaw/openclaw/issues/92561.
