## Summary

This is a well-supported feature gap on current main, not a duplicate or already-implemented cleanup target; it should remain open for a focused product decision about the authoritative working-directory and instruction trust contract.

## What This Changes

Add deterministic project-root-to-CWD discovery and bootstrap injection of nested AGENTS.md and CLAUDE.md instructions.

## Best Possible Solution

Define an explicit trusted session project root and working directory, then build one bounded root-to-CWD instruction snapshot during bootstrap with deterministic precedence, existing character budgets, and no mid-turn filesystem rediscovery.

## Reproduction Assessment

Yes, at source level: current main loads fixed bootstrap basenames directly from `workspaceDir` and has no project-root-to-CWD instruction traversal. A live agent run was unnecessary because the limitation is explicit in the canonical loader and callers.

## Solution Assessment

No, not exactly as proposed. The maintainable path is an explicit session-CWD snapshot; a new default-off config tree and dynamic inference from later file reads or shell `cd` operations would add competing policy and unstable prompt behavior.

## Risks / Open Questions

- OpenClaw does not currently expose one clearly authoritative project CWD across every channel, CLI, spawned-agent, and resumed-session entry point.
- Walking parent directories changes the system-prompt trust boundary; the root anchor and behavior outside the canonical workspace must be explicit.
- Refreshing instructions after a tool-level `cd` or first file read would make prompt context nondeterministic and conflict with session snapshot caching.
- Supporting CLAUDE.md and local override filenames requires a product decision about compatibility names and precedence rather than an open-ended configurable filename list.
