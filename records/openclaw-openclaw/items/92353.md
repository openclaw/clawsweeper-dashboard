## Summary

Keep open: current main does not implement the requested automatic cwd-to-project-root bootstrap discovery for normal OpenClaw agent context. The main bootstrap path still loads fixed files from `workspaceDir`, while the separate sessions loader that scans `AGENTS.md`/`CLAUDE.md` ancestors is not a complete solution and is linked to an open security-boundary issue.

## What This Changes

The issue requests cwd-aware automatic loading of nested `AGENTS.md`/`CLAUDE.md` files so project-specific instructions enter agent context without per-project manual configuration.

## Best Possible Solution

Define a safe workspace/project-root-bounded directory-context contract for normal bootstrap injection, reuse existing bootstrap budgets and diagnostics, document the config/default choice, and coordinate the boundary with the open security follow-up.

## Reproduction Assessment

Yes, at source level: the normal bootstrap path loads fixed filenames from `workspaceDir`, and the only nested-file support found is explicit hook configuration or a separate sessions loader. I did not run a live session because this review was read-only.

## Solution Assessment

No, not as already solved by current main. The maintainable fix should be bounded to a workspace/project root, integrated into the normal bootstrap path with existing budgets and docs, and reconciled with the open security issue before landing.
