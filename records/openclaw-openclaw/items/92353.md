## Summary

Current main still anchors bootstrap context to the configured workspace even when tools use a different CWD, so automatic nested project-instruction loading remains absent. The request is coherent and aligned with agent-harness ergonomics, but root selection, trust boundaries, cache lifecycle, supported filenames, and mid-session behavior need a maintainer product decision before implementation.

## What This Changes

Add deterministic root-to-CWD discovery and system-prompt injection for nested AGENTS.md and CLAUDE.md project instructions.

## Best Possible Solution

Define one immutable run-level project root and initial task CWD, resolve a narrow supported instruction convention once through the existing bootstrap pipeline with current boundary and budget guards, and leave later tool-level cd operations context-neutral unless a separate session-transition design is approved.

## Reproduction Assessment

Yes at source level: current main explicitly resolves bootstrap files from workspaceDir while a distinct task CWD is supplied only to tools and resource loading. The focused CWD-split test makes the missing automatic injection path high-confidence without requiring a live run.

## Solution Assessment

No, not exactly as proposed: initial directory-relative context is a plausible core capability, but configurable filenames and modes plus mid-session cd reloading would introduce unnecessary configuration and cache complexity. A single run-level project-context contract is the narrower maintainable direction.
