## Summary

Current main has a real gap: OpenClaw resolves a CLI session CWD separately from the workspace, but bootstrap instruction loading remains workspace-root based, and the explicit extra-file hook does not provide automatic directory-chain semantics. The feature is worth keeping, but the proposed configuration and refresh behavior require a maintainer product decision before implementation.

## What This Changes

Add deterministic root-to-CWD discovery and prompt injection for directory-scoped AGENTS.md and CLAUDE.md instructions.

## Best Possible Solution

Add a narrow core session-instruction resolver that takes an explicit, workspace-contained session CWD, loads one canonical instruction file per directory in root-to-CWD order with existing budgets, and remains session-stable; defer configurable filenames, modes, local overrides, and tool-driven refresh until separately justified.

## Reproduction Assessment

Yes at source level: a CLI run can have `cwd` distinct from `workspaceDir`, while the bootstrap call receives only `workspaceDir`; this read-only review did not execute a runtime reproduction.

## Solution Assessment

No, not exactly as proposed: automatic directory scope is a maintainable core capability, but configurable filename lists, traversal modes, inferred CWD from file reads, and mid-session reloads should not be bundled into the first implementation.

## Risks / Open Questions

- Non-CLI channel sessions may not have a meaningful project CWD, so using the first file read or tool-local `cd` could make instruction scope nondeterministic.
- Automatically accepting `CLAUDE.md` or local override files changes which trusted text enters the system prompt and needs explicit precedence and workspace-boundary rules.
- A new `directoryContext` config block would add filenames and modes before the canonical behavior is settled, contrary to the repository's high bar for new config surfaces.
- Mid-session refresh could invalidate prompt-cache and session-reuse assumptions; a session-stable scope is safer unless a concrete reload owner is designed.
