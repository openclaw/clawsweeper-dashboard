## What This Changes

The PR changes memory-core embedding-cache writes from JSON TEXT to packed Float32 BLOBs, keeps legacy JSON TEXT reads, exports/reuses the vector BLOB helper, and adds cache round-trip tests.

## Best Possible Solution

Rebase onto current main, use the current `vector-blob.ts` helper, and then land the cache-only encoding change if maintainers accept the in-place mixed TEXT/BLOB cache boundary.

## Solution Assessment

Yes for the cache-only slice: it reuses the existing vector encoder, preserves legacy JSON reads, and avoids the durable index. The branch still needs a rebase, and maintainers need to decide whether the in-place cache encoding change is acceptable or should use an explicit new storage shape.
