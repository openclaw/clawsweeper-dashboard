## Summary

Keep open for maintainer review: current main and the latest release still use JSON text for this cache, while the PR is a focused cache-only storage optimization with sufficient real behavior proof. The remaining blocker is maintainer acceptance of storing BLOB values in an existing TEXT-affinity persisted cache column, not a concrete line-level defect.

## What This Changes

The PR changes memory-core embedding cache writes from JSON text to packed Float32 BLOBs, preserves legacy JSON reads, exports the existing vector BLOB helper, and adds cache round-trip tests.

## Best Possible Solution

Land the cache-only BLOB encoding if maintainers accept the internal cache compatibility tradeoff, while leaving durable index storage, cache-size tuning, and hybrid recall to https://github.com/openclaw/openclaw/issues/77301.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a performance/storage cleanup PR. Current-main and release source confirm the existing JSON cache write path, and the PR screenshot shows after-fix mixed JSON/BLOB cache behavior.

## Solution Assessment

Yes, this appears to be the narrowest maintainable implementation for the cache-only slice because it reuses the existing vector BLOB encoder and preserves legacy JSON reads. The safer alternative is an additive BLOB column if maintainers want stricter storage-shape compatibility.
