## Summary

Keep open for maintainer review: current main still uses JSON text cache rows and this PR appears narrow with sufficient proof, but changing stored cache values in the existing cache column is a compatibility tradeoff maintainers should explicitly accept before merge.

## What This Changes

The PR changes memory-core embedding cache writes from JSON text to packed Float32 BLOBs, adds mixed-format cache decoding, exports `vectorToBlob`, and adds cache round-trip tests.

## Best Possible Solution

Land the cache-only BLOB encoding if maintainers accept the internal cache compatibility tradeoff, while leaving durable index storage, cache-size tuning, and hybrid recall to https://github.com/openclaw/openclaw/issues/77301.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a performance/storage cleanup PR. Current-main source confirms the existing JSON cache write path, and the PR body supplies after-fix benchmark output for the proposed cache path.

## Solution Assessment

Yes, this appears to be the narrowest maintainable implementation for the cache-only slice because it reuses the existing vector BLOB encoder and preserves legacy JSON reads. The safer alternative is an additive BLOB column if maintainers want stricter storage-shape compatibility.
