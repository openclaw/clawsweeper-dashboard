## Summary

## What This Changes

This PR changes memory-core to write embedding cache rows as packed Float32 BLOBs via the existing vector encoder, decode both new BLOB and legacy JSON TEXT rows, and add cache round-trip tests.

## Best Possible Solution

Land the cache-only BLOB encoding if maintainers accept the internal cache compatibility tradeoff, while leaving durable index storage, cache-size tuning, and hybrid recall sequencing to https://github.com/openclaw/openclaw/issues/77301.

## Reproduction Assessment

## Solution Assessment

Yes, if maintainers accept the cache encoding tradeoff. The patch is a narrow cache-only change that reuses the existing vector BLOB encoder; an additive BLOB column is the safer alternative if maintainers want an explicit schema boundary.
