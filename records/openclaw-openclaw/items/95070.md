## What This Changes

The PR changes memory-core embedding cache writes from JSON text to packed Float32 BLOBs, adds mixed-format cache decoding, exports the existing vector BLOB helper, and adds cache tests.

## Best Possible Solution

Land the cache-only BLOB encoding if maintainers accept the internal cache compatibility tradeoff, while leaving durable index storage, cache-size tuning, and hybrid recall to https://github.com/openclaw/openclaw/issues/77301.

## Reproduction Assessment

## Solution Assessment

Yes, this appears to be the narrowest maintainable implementation for the cache-only slice because it reuses the existing vector BLOB encoder and preserves legacy JSON reads. The safer alternative is an additive BLOB column if maintainers want stricter storage-shape compatibility.
