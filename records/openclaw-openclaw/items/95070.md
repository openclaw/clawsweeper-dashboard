## What This Changes

This PR changes memory-core embedding cache writes from JSON text to packed Float32 BLOBs, adds mixed-format cache decoding, exports the existing vector BLOB helper, and expands cache tests.

## Best Possible Solution

Land the cache-only BLOB encoding if maintainers accept the internal cache compatibility tradeoff and exact-head checks pass, while leaving durable index storage, cache-size tuning, and hybrid recall sequencing in https://github.com/openclaw/openclaw/issues/77301.

## Reproduction Assessment

## Solution Assessment

Yes, this appears to be the narrowest maintainable fix for the cache-only slice: it reuses the existing vector BLOB encoder and leaves durable index and hybrid retrieval changes out of scope. The safer alternative is an additive BLOB column if maintainers want stricter storage-shape compatibility.
