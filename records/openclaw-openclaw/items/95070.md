## Summary

Keep open: current main still writes the memory embedding cache as JSON text, and this PR is a concrete, well-proven cache-only slice of the broader memory SQLite work. ## What This Changes

The PR changes memory-core embedding-cache writes from JSON text to packed Float32 BLOBs, keeps legacy JSON-text reads, and adds cache round-trip coverage.

## Best Possible Solution

Land the cache-only encoding change if maintainers accept the in-place mixed JSON/BLOB cache boundary; otherwise require an explicit BLOB storage shape or keep the decision with https://github.com/openclaw/openclaw/issues/77301.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a performance/storage-format cleanup PR. Current main source confirms the prior JSON cache behavior, and the supplied proof demonstrates the after-fix mixed JSON/BLOB path.

## Solution Assessment

Yes for the cache-only slice if maintainers accept the storage boundary: it reuses the existing Float32 vector encoding, preserves legacy JSON reads, and avoids touching the durable index. The safer alternative is an explicit BLOB column or broader sequencing if maintainers do not want BLOB values in the existing TEXT-affinity column.
