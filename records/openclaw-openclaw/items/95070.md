## Summary

Keep open for maintainer review: current main and v2026.6.8 still use JSON TEXT for the memory embedding cache, while this PR is a focused, well-proven cache encoding change that still needs maintainer acceptance of the in-place TEXT-column/BLOB-value compatibility tradeoff.

## What This Changes

The PR changes memory-core embedding-cache writes from JSON TEXT to packed Float32 BLOBs, preserves legacy JSON reads, exports the existing vector BLOB helper, and adds cache round-trip tests.

## Best Possible Solution

Land the cache-only BLOB encoding if maintainers accept mixed JSON/BLOB cache rows as the upgrade boundary for this rebuildable cache, while leaving cache_size, durable-storage, and retrieval-ranking work to the broader memory issue.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a performance/storage-format cleanup PR. Current-main and latest-release source confirm the prior JSON cache behavior, and the PR proof demonstrates the after-fix mixed JSON/BLOB path.

## Solution Assessment

Yes for the cache-only slice: it reuses the existing vector encoder, preserves legacy JSON reads, and avoids touching the durable index. The remaining question is whether maintainers prefer this in-place encoding change or an explicit new storage shape.
