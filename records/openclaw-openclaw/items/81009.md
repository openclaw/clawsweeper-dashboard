## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Make `/progress` affect every exposed surface:** `src/auto-reply/commands-registry.shared.ts:831-834`
  - body: Registering `/progress` as a shared built-in exposes it wherever core commands are published, and the shared directive handler will acknowledge and persist the override for any provider. Only Telegram dispatch reads that session override, while Discord, Slack, and Matrix still use config-only progress checks, so `/progress off` can report success on those surfaces while progress lines continue. Either scope this command to Telegram or thread the session override through every surface that exposes it.
  - confidence: 0.91
- **[P2] Report the effective configured progress state:** `src/auto-reply/reply/directive-handling.impl.ts:236`
  - body: When no session override is set, this status path hard-codes the state as `on (config)`. A user with `streaming.preview.toolProgress: false` will be told progress is on even though the resolver suppresses it, so `/progress` needs to compute the same effective session/config value that dispatch uses.
  - confidence: 0.86
