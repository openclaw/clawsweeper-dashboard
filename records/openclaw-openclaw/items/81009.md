## Summary

Keep open. The feature direction is plausible, but this PR currently has source-proven correctness issues in the directive path and lacks the required real Telegram behavior proof for an external Telegram streaming change.

## What This Changes

The PR adds a Telegram `/progress` command and session-level `previewToolProgress` override to control streaming tool-progress lines without changing global config.

## Best Possible Solution

Wire progress into the full directive application path, compute status from the effective session/config resolver, add coverage through the real resolve/apply path, and attach redacted Telegram proof before maintainer acceptance.

## Reproduction Assessment

Yes for the PR-introduced bug: source inspection shows `/progress` is parsed but omitted from `applyInlineDirectiveOverrides`' directive gate, so progress-only commands can return before persistence or a status reply. I did not run tests because this was a read-only review.

## Solution Assessment

No. The requested direction is reasonable, but this patch is not the best mergeable solution until progress is routed through every apply gate, status reflects the effective config/session value, and real Telegram behavior is proven.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Route progress directives through the apply gate:** `src/auto-reply/reply/get-reply-directives.ts:309`
  - body: Adding `hasProgressDirective` here makes `/progress` visible to the earlier parser gate, but `applyInlineDirectiveOverrides` still computes `hasAnyDirective` without that flag. A message containing only `/progress off`, `/progress on`, or `/progress default` returns before the new persistence/status logic runs, so the advertised command does not actually toggle the session unless another directive is also present.
  - confidence: 0.93
- **[P2] Report the effective configured progress state:** `src/auto-reply/reply/directive-handling.impl.ts:235-236`
  - body: The status path treats an unset session override as `on (config)` without reading the channel streaming config. If `streaming.preview.toolProgress` is configured `false`, `/progress` would report progress as on even though Telegram suppresses the tool-progress lines, so this should use the effective session/config resolver value.
  - confidence: 0.86
