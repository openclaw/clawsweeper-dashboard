## Summary

Keep open: current main and the latest release still expose only `backup create` and `backup verify`, while the requested native restore command is not implemented. Ryce's keepmyclaw ClawHub plugin is useful prior art and a workaround, and open PR #59280 is the direct implementation candidate, but neither makes the first-party CLI gap resolved yet.

## What This Changes

Requested addition of a native `openclaw backup restore <archive>` command.

## Best Possible Solution

Review, revise, and either land or close PR #59280; if maintainers want one umbrella instead, explicitly consolidate this into #13616 while preserving keepmyclaw as a workaround and prior-art reference.

## Reproduction Assessment

Yes. A high-confidence source-level check is to inspect current main's backup command registration, registration tests, backup docs, and generated CLI index; all show create/verify only and no native restore command.

## Solution Assessment

Yes, a first-party restore command is the right direction if it includes conservative safety behavior. The keepmyclaw plugin is useful prior art and a workaround, but it does not complete the built-in backup lifecycle that core already started with create and verify.
