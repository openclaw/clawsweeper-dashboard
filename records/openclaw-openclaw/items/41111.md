## Summary

Keep open: current main and the latest release still expose only backup create and verify, while the requested native restore command remains unmerged. The keepmyclaw ClawHub plugin is useful prior art and a workaround, and open PR #59280 is the direct implementation candidate, but the first-party CLI gap still exists.

## What This Changes

Request to add a native `openclaw backup restore <archive>` command alongside the existing backup create and verify commands.

## Best Possible Solution

Keep the issue open until OpenClaw ships a safe native `openclaw backup restore <archive>` workflow, or maintainers explicitly consolidate it into a canonical backup/restore tracker such as #13616.

## Reproduction Assessment

Yes. A high-confidence source-level check is to inspect current main's backup command registration, registration tests, backup docs, and generated CLI index; all show create/verify only and no native restore command.

## Solution Assessment

Yes, a first-party restore command is the right direction if it includes conservative safety behavior. The keepmyclaw plugin is useful prior art and a workaround, but it does not complete the built-in backup lifecycle that core already started with create and verify.
