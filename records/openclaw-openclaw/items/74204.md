## Summary

Keep open. Current main still resolves unset QMD embed timeout to 120000ms, passes that value directly to semantic `qmd embed` runs, and logs the raw timeout/backoff message without naming `memory.qmd.update.embedTimeoutMs`; the provided prior ClawSweeper comment remains accurate after this review.

## What This Changes

Request to make QMD semantic embedding usable on local GGUF hosts by raising or contextualizing the default embed timeout and surfacing the config fix in diagnostics/docs.

## Best Possible Solution

Implement a QMD-owned fix that preserves explicit overrides, gives unset semantic `qmd embed` runs a local-GGUF-friendly timeout, and updates timeout warnings, schema help, QMD docs, tests, and changelog together.

## Reproduction Assessment

Yes. Configure QMD with `searchMode: "query"` or `"vsearch"`, leave `memory.qmd.update.embedTimeoutMs` unset, and current main resolves `120000ms`, passes it to `qmd embed`, then logs only the raw timeout/backoff text on failure.

## Solution Assessment

Yes. The narrow maintainable solution belongs in QMD config resolution and QMD embed diagnostics/docs, not in the builtin memory engine or lexical `searchMode: "search"` behavior.
