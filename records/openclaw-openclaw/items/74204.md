## Summary

Keep open. Current main still resolves an unset QMD embed timeout to 120000ms, passes that budget to semantic `qmd embed` runs, and logs the raw timeout/backoff warning without naming the config key that fixes it.

## What This Changes

The issue asks OpenClaw to make QMD semantic embedding timeouts usable for local GGUF workloads by raising or adapting the default and surfacing `memory.qmd.update.embedTimeoutMs` in errors and docs.

## Best Possible Solution

Implement a QMD-owned repair that preserves explicit overrides, uses a local-GGUF-friendly unset semantic embed timeout or equivalent default, and makes timeout warnings plus docs name `memory.qmd.update.embedTimeoutMs`.

## Solution Assessment
