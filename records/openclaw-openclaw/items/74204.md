## Summary

Keep open. The report is current-main valid: the target SHA still resolves an unset QMD embed timeout to 120000ms, semantic QMD modes pass that value directly to `qmd embed`, and the warning path logs the raw timeout/backoff text without naming `memory.qmd.update.embedTimeoutMs`. The docs expose the knob but still lack a troubleshooting entry for the reported `qmd embed timed out after 120000ms` symptom.

## What This Changes

The issue requests a higher or context-aware default for `memory.qmd.update.embedTimeoutMs`, clearer QMD embed timeout diagnostics, and QMD troubleshooting docs for local GGUF workloads.

## Best Possible Solution

Implement a focused QMD-owned repair: make the unset embed timeout local-GGUF-friendly for semantic `vsearch`/`query` modes, keep explicit user overrides respected, add a timeout warning that names `memory.qmd.update.embedTimeoutMs` and the effective value, and document the symptom and recommended values in the QMD troubleshooting/config reference pages.
