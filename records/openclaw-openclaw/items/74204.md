## Repair Work Prompt

Fix the QMD semantic embed timeout gap reported in https://github.com/openclaw/openclaw/issues/74204. Current main defaults unset `memory.qmd.update.embedTimeoutMs` to 120000ms, passes it to `qmd embed` for `query`/`vsearch`, and logs timeout/backoff text without naming the config key. Keep explicit user overrides working. Make the unset semantic QMD embed timeout local-GGUF-friendly, preferably aligning with the existing 600000ms local inline embedding precedent or an equally narrow contextual resolver. Add a timeout diagnostic hint that mentions `memory.qmd.update.embedTimeoutMs`, the effective value, and a practical example such as `600000`. Add QMD docs troubleshooting for `qmd embed timed out after 120000ms`, align schema/config help as needed, and add a single-line user-facing changelog fix entry. Do not change the built-in memory engine, QMD lexical `search` skip behavior, or unrelated QMD scheduling semantics.
