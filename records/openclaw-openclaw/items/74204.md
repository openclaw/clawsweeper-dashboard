The issue requests a local-GGUF-friendly QMD embed timeout default and clearer timeout diagnostics/docs for `memory.qmd.update.embedTimeoutMs`.

## Best Possible Solution

Implement a QMD-owned repair that preserves explicit overrides, uses a local-GGUF-friendly semantic embed timeout default or equivalent unset behavior, and makes timeout warnings plus docs name `memory.qmd.update.embedTimeoutMs`.
