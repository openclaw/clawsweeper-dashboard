## Summary

Keep open. Current main still matches the reported QMD embed timeout problem: an unset QMD embed timeout resolves to 120000ms, semantic QMD modes pass that value to `qmd embed`, and timeout warnings surface only the raw subprocess timeout/backoff text without naming `memory.qmd.update.embedTimeoutMs`. Related older QMD timeout work added configurability and fixed other scheduler/lexical-mode cases, but it does not close this local-GGUF default and diagnostic gap, and no open PR was found for this issue.

## What This Changes

Request to make QMD semantic embed timeouts more suitable for local GGUF workloads and surface `memory.qmd.update.embedTimeoutMs` in timeout diagnostics and docs.

## Best Possible Solution

Implement a focused QMD-owned repair: make the unset semantic embed timeout local-GGUF-friendly, preserve explicit `embedTimeoutMs` overrides, enrich QMD embed timeout warnings with the config key and effective value, add regression coverage, and document the reported timeout symptom with recommended values.

Yes. ## Solution Assessment
