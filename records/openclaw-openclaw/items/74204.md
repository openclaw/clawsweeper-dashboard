## Summary

Keep open. Current main still has the reported QMD semantic-embed timeout gap: the unset embed timeout resolves to 120000ms, semantic QMD modes pass that directly to `qmd embed`, and timeout warnings still do not name the config key that fixes the failure.

## What This Changes

Request to make QMD semantic embed timeouts more local-GGUF-friendly and surface `memory.qmd.update.embedTimeoutMs` in diagnostics/docs.

## Best Possible Solution

Implement a QMD-owned repair that preserves explicit overrides, uses a local-GGUF-friendly unset semantic embed timeout, and makes timeout warnings plus docs point directly to `memory.qmd.update.embedTimeoutMs`.

## Reproduction Assessment

Yes. Configure QMD with `searchMode: "query"` or `"vsearch"`, leave `memory.qmd.update.embedTimeoutMs` unset, and current main resolves 120000ms, passes it to `qmd embed`, then logs only the raw timeout/backoff message on failure.

## Solution Assessment

Yes. The narrow maintainable fix belongs in QMD config resolution, QMD embed diagnostics, focused QMD tests, docs, and changelog; it should not change the builtin memory engine or lexical `searchMode: "search"` behavior.
