## Summary

Keep open. Current main still resolves an unset QMD embed timeout to 120000ms, passes that timeout directly to semantic `qmd embed` runs, and logs the raw timeout/backoff message without naming `memory.qmd.update.embedTimeoutMs`.

## What This Changes

Raise or clarify QMD embed timeout handling so semantic QMD modes with local GGUF embeddings do not fail repeatedly at the 120s default without a clear remediation.

## Best Possible Solution

Keep this issue open and make a QMD-owned fix that preserves explicit overrides, uses a local-GGUF-friendly unset embed timeout for semantic modes, and names `memory.qmd.update.embedTimeoutMs` in timeout diagnostics and docs.
