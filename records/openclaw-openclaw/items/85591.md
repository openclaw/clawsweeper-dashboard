## Summary

Keep open: the PR is a useful sandbox hardening direction, but it still needs real Docker proof and the new chmod path can make copied non-rw sandbox skills stale after the first sync.

## What This Changes

The PR adds Docker read-only overlays for workspace skill directories, includes overlay state in container hashing, chmods copied sandbox skills read-only for non-rw workspaces, and adds mount tests.

## Best Possible Solution

Keep the Docker rw read-only overlay and hash-state approach, but make non-rw copied skill protection refresh-safe before treating this as the canonical fix for sandbox skill immutability.

## Reproduction Assessment

Yes at source level: current main mounts the whole workspace writable in Docker rw mode, and the PR's chmod path can be traced against the current skill-sync delete-and-copy implementation. I did not run a live Docker container in this read-only review.

## Solution Assessment

No; the rw overlay direction is a good fit, but the non-rw chmod solution needs a refresh-safe design before it should close the linked security issue.
