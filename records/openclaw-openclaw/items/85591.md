## Best Possible Solution

Land the focused Docker hardening after maintainers accept the rw skill immutability contract and the existing container recreate behavior for upgrades.

## Solution Assessment

Yes, with maintainer signoff. Read-only bind overlays in rw mode plus the existing read-only `/workspace` mount in ro/none mode is a cleaner sandbox boundary than chmod or copied-file permission tricks; I found no discrete patch defect.
