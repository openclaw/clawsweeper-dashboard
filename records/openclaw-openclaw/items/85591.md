## Summary

Keep open. The PR is a useful partial Docker hardening step, but it is not ready to merge as-is because it leaves the linked sandbox skill immutability issue unresolved for non-rw modes, misses a container-reuse case for newly added skill directories, and has no after-fix real sandbox proof.

## What This Changes

The branch bumps the sandbox mount format version and adds Docker read-only bind overlays for existing workspace `skills/` and `.agents/skills/` directories when `workspaceAccess` is `rw`, with mount-argument tests.

## Best Possible Solution

Land an owner-reviewed sandbox skill-mount contract that protects eligible skill sources across workspace access modes, participates in container hashing and fs-path authority, documents the compatibility change, and includes real Docker proof.

## Reproduction Assessment

Yes, at source level. Current main still copies skills into writable sandbox workspaces for non-rw modes, and the PR diff only overlays existing workspace skill directories for Docker `rw` mounts; I did not run a live Docker container in this read-only review.

## Solution Assessment

No, not as submitted. Read-only Docker overlays are the right direction, but the branch needs either a narrower non-closing partial scope or a fuller security design covering non-rw copies, recreation hashing, fs-path authority, docs, and real behavior proof.
