## Risks / Open Questions

- Merging intentionally makes existing workspace skill roots read-only inside `workspaceAccess: "rw"` sandboxes, so workflows that edit those paths from inside the sandbox will start failing with read-only errors.
- Existing hot sandbox containers with stale mount state may log a recreate hint rather than being killed immediately, so the hardening may not apply until the operator recreates or the container cools down.
- Custom Docker binds are still appended after managed workspace mounts, so operators can intentionally shadow this protection with their own bind configuration; maintainers should accept that as part of the security contract.
