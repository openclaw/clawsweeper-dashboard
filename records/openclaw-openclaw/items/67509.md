## Summary

## What This Changes

This PR adds a CLI root guard, tests, and entrypoint calls that block uid/euid 0 unless `OPENCLAW_ALLOW_ROOT=1` or the forwarded-container env pair is present.

## Best Possible Solution

Land the root-execution guard after the install docs describe the non-root/wrapper path, release notes mention the behavior change, and the PR includes redacted runtime proof from a real root Linux setup.

## Reproduction Assessment

Yes, source-reproducible. Current main has no uid/euid guard before CLI startup and `runCli` can resolve root-scoped state before command dispatch; I did not run the destructive root state path during this read-only review.

## Solution Assessment

No, not as submitted. A global guard is a plausible fix, but the docs/changelog/proof gaps mean this is not yet the narrowest maintainable merge-ready solution.
