## Summary

Keep open. Current main still does not implement the root-execution guard that the linked bug needs, but this PR should not merge as-is because it changes documented DigitalOcean root-shell setup behavior and still lacks the required user-facing changelog entry.

## What This Changes

The PR adds a CLI root guard with UID/EUID checks and explicit/container-forwarding bypasses, wires it into the modern and legacy CLI entrypoints, and adds root-guard unit tests.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off: guard both CLI entrypoints before mutable state/config side effects, check both real and effective UID, preserve intentional container forwarding, update DigitalOcean/1-Click guidance to the supported non-root or wrapper workflow, and add the required changelog entry.

## Reproduction Assessment

Yes. The linked bug has deterministic root-SSH reproduction steps, and current main still reaches CLI startup without a UID/EUID guard while default mutable state resolves under the effective home directory.

## Solution Assessment

No, not as submitted. A core root guard is a narrow plausible fix for #67478, but the branch also needs DigitalOcean setup guidance and changelog coverage before it is the maintainable merge shape.
