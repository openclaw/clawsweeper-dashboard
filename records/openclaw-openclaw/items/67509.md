## Summary

Keep open. The PR addresses a real current-main bug, but it is not merge-ready because the global root refusal conflicts with published DigitalOcean root-session commands and the branch lacks required changelog coverage.

## What This Changes

Adds a CLI root guard with UID/EUID checks and root/container bypasses, calls it from modern and legacy entrypoints, and adds guard tests.

## Best Possible Solution

Decide whether root refusal is global CLI policy or DigitalOcean/managed-image scoped, then land the guard with matching install docs, recovery guidance, and changelog coverage.

## Reproduction Assessment

Yes, at source level. Current main enters both CLI entrypoints without a UID/EUID guard and defaults mutable state to `~/.openclaw`; the linked DigitalOcean report gives the destructive root-shell path, which I did not run.

## Solution Assessment

No, not as-is. The guard implementation covers the important UID/EUID and entrypoint paths, but merging it without policy-aligned docs and changelog would break published DigitalOcean setup commands.
