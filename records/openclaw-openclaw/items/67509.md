## Summary

Keep open. Current main still lacks root protection, and the PR is useful, but the latest diff remains incorrect because it would break the documented root-shell DigitalOcean setup and still needs changelog/test hardening.

## What This Changes

The PR adds a CLI root guard module, wires it into modern and legacy entrypoints, and adds tests for real/effective UID handling plus explicit and container-forwarded bypasses.

## Best Possible Solution

Land this PR or an equivalent fix only after the root-shell install flow is made consistent, with docs/changelog and non-root/euid test coverage updated while preserving the container-forwarded bypass.

## Reproduction Assessment

Yes. The linked bug provides deterministic DigitalOcean root SSH plus `openclaw doctor --fix` steps, and current main still enters both CLI entrypoints without root refusal while mutable state defaults to `~/.openclaw` under the invoking home.

## Solution Assessment

No, not as-is. Early root refusal in both entrypoints is a plausible narrow code fix, but the public root-shell setup path and required changelog/test coverage must be reconciled before merge.
