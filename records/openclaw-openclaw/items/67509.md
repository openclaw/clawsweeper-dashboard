## Summary

Keep open: the root guard targets a valid source-reproducible bug, but this branch is not merge-ready because it still dead-ends the current DigitalOcean root setup docs, lacks required changelog coverage, and has no after-fix real behavior proof.

## What This Changes

Adds a CLI root guard wired into modern and legacy entrypoints with uid/euid checks, an explicit root override, a container-forwarded bypass, and unit tests.

## Best Possible Solution

Land this PR or an equivalent early CLI guard only after aligning the root install/onboarding docs, documenting the override in release notes, preserving intentional container forwarding, and adding redacted real Linux root-run proof.

## Reproduction Assessment

Yes, source-reproducible. The linked report at https://github.com/openclaw/openclaw/issues/67478 gives deterministic DigitalOcean root steps, and current main still has HOME-derived state resolution with no entrypoint root guard; I did not run the destructive droplet scenario.

## Solution Assessment

No, this is not the best merge-ready form yet. A core root guard is a plausible narrow fix, but the documented DigitalOcean root setup path, changelog coverage, override policy, and real root-run proof need to be resolved first.
