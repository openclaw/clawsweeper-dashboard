## Summary

Keep open. The branch is a focused fix for a real root-run corruption bug, but the current diff still changes root CLI behavior without reconciling documented DigitalOcean root-shell commands or adding the required user-facing changelog entry.

## What This Changes

The PR adds an `assertNotRoot` CLI guard, tests real/effective UID handling and bypasses, and wires the guard into the modern and legacy CLI entrypoints.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off, with both entrypoints guarded before mutable state/config side effects, real and effective UID covered, intentional container forwarding preserved, and DigitalOcean guidance plus changelog aligned.

## Reproduction Assessment

Yes. #67478 gives deterministic root SSH and `doctor --fix` steps, and current main still resolves mutable state from the invoking user's home while both CLI entrypoints lack a root refusal.

## Solution Assessment

No, not yet. The core guard is a plausible narrow fix, but the branch should also reconcile root-based DigitalOcean setup instructions, add the required changelog entry, and harden the non-root test against root-based runners.
