## Summary

Keep open: current main still has the root-run corruption path, and this PR is a plausible direction, but it would currently break the published DigitalOcean root-shell setup and still needs changelog/test cleanup before merge.

## What This Changes

The PR adds a CLI root guard with real/effective UID checks, an explicit and container-forwarding bypass, entrypoint wiring for modern and legacy CLI paths, and unit tests.

## Best Possible Solution

End state should be a guarded CLI plus a documented non-root or wrapper-based DigitalOcean flow, a deliberate override story, a changelog entry, and passing targeted CLI tests.

## Reproduction Assessment

Yes. The linked bug gives deterministic DigitalOcean root SSH plus openclaw doctor --fix steps, and current main still has unguarded entrypoints plus ~/.openclaw state resolution under the invoking home; I did not live-run the destructive scenario.

## Solution Assessment

No, not as-is. Early guarding in both entrypoints is a reasonable narrow fix, but the published DigitalOcean root-shell path, required changelog, and euid-sensitive test need to be reconciled first.
