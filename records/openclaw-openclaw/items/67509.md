## Summary

Keep open: current main still lacks the requested root guard, and this PR is useful but not ready because it conflicts with published DigitalOcean root-shell setup guidance and still needs changelog/test cleanup.

## What This Changes

The PR adds an early CLI real/effective-root guard with explicit allow-root and container-forwarding bypasses, wires it into modern and legacy entrypoints, and adds focused unit coverage.

## Best Possible Solution

Land early root/effective-root protection in both entrypoints after reconciling DigitalOcean setup guidance and override semantics, then add the required changelog entry and stable UID/eUID coverage.

## Reproduction Assessment

Yes. The linked issue gives deterministic root SSH plus openclaw doctor --fix steps, and source inspection shows current main still uses HOME-derived ~/.openclaw state with unguarded entrypoints; I did not run the destructive Droplet scenario.

## Solution Assessment

No, not as-is. The early-entrypoint guard is the right implementation direction, but this PR should first reconcile published root-shell setup guidance and complete the changelog and eUID test fixes.
