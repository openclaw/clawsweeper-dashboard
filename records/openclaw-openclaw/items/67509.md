## Summary

Keep open: current main still lacks a UID/EUID root guard and this PR addresses the valid root-state corruption path from #67478, but the branch is not merge-ready because it conflicts with published root-session DigitalOcean setup docs and lacks the required user-facing changelog entry.

## What This Changes

The PR adds an `assertNotRoot` CLI guard with UID/EUID checks, an explicit root override, a container-forwarding bypass, tests, and calls from both modern and legacy CLI entrypoints.

## Best Possible Solution

Align the root policy with setup guidance, then land the early UID/EUID guard with matching DigitalOcean/VPS docs and changelog coverage or narrow it to the affected image wrapper.

## Reproduction Assessment

Yes, at source level. Current main has no UID/EUID guard in either CLI entrypoint, state defaults to `~/.openclaw`, and #67478 gives a deterministic root-shell DigitalOcean path; I did not run the destructive droplet scenario.

## Solution Assessment

No, not as a complete merge candidate. The guard direction is plausible for the corruption bug, but the branch must first reconcile published root-session setup docs and add the required changelog entry.
