## Summary

Keep open: the PR addresses a valid root-state corruption path, but the current branch still conflicts with published DigitalOcean/root-shell setup docs and lacks required changelog coverage.

## What This Changes

The PR adds a CLI root-execution guard with UID/EUID checks, an explicit override, a container-forwarding bypass, tests, and calls from both modern and legacy entrypoints.

## Best Possible Solution

Land an early UID/EUID guard only after root-based install docs describe the supported non-root or wrapper path, with changelog coverage and the container-forwarded bypass preserved.

## Reproduction Assessment

Yes, at source level. Current main has no UID/EUID guard in either CLI entrypoint and mutable state resolves under the invoking user's home; #67478 gives deterministic root-shell DigitalOcean steps, but I did not run the destructive droplet scenario.

## Solution Assessment

No, not as a complete merge candidate. The guard implementation is plausible for the corruption bug, but the branch must first reconcile root-based install guidance and add the required changelog entry.
