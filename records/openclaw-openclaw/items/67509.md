## Summary

Keep open. Current main does not implement the root-execution guard, and this live PR addresses a real linked bug, but it still needs maintainer policy review and docs/changelog alignment before merge.

## What This Changes

The PR adds a CLI root guard with tests and wires modern and legacy entrypoints to refuse real or effective UID 0 unless explicitly overridden or running through the container-forwarded path.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off, with docs and changelog aligned to the chosen root-command behavior.

## Reproduction Assessment

Yes. The linked bug has deterministic DigitalOcean root-SSH steps, and current main still lets both CLI entrypoints reach startup while mutable state defaults through the invoking user's home directory.

## Solution Assessment

No, not as submitted. The guard shape is a plausible narrow fix, but it must be reconciled with DigitalOcean/VPS install guidance and the required changelog before it is the best maintainable solution.
