## What This Changes

## Best Possible Solution

The linked report at https://github.com/openclaw/openclaw/issues/67478 gives deterministic DigitalOcean root-run steps and logs, but I did not run the destructive Droplet corruption path.

## Solution Assessment

Unclear as submitted: guarding both entrypoints before mutable startup is a narrow and plausible fix, but the PR must first reconcile the documented root setup flow and get maintainer approval for the new override/bypass policy.
