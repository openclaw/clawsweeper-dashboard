## Summary

Keep open. Current main still lacks the root guard and the linked DigitalOcean state-corruption bug remains plausible, but this PR is not merge-ready because it changes root CLI behavior while DigitalOcean setup docs still run OpenClaw commands from a root SSH session and the user-facing fix lacks a changelog entry.

## What This Changes

The PR adds a CLI root guard module with tests, wires it into the modern and legacy CLI entrypoints, and exempts explicit root override or container-forwarded invocations.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off, with docs and changelog aligned: guard both CLI entrypoints before mutable state/config side effects, check both real and effective UID, preserve intentional container forwarding, and document the supported non-root or explicit-override workflow for DigitalOcean users.

## Reproduction Assessment

Yes. The linked bug has deterministic root-SSH reproduction steps, and current main still routes CLI startup through the invoking user's home-based state directory without a UID/EUID guard; I did not run the destructive live repro.

## Solution Assessment
