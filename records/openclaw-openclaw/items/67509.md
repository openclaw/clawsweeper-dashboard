## Summary

Keep open: current main still lacks the root guard and the linked root-session bug remains valid, but this branch needs maintainer sign-off plus docs, changelog, and test cleanup before merge.

## What This Changes

The branch adds `assertNotRoot`, tests, and entrypoint wiring to block real or effective UID 0 unless explicit root or container-forwarding overrides are present.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off, with both CLI entrypoints guarded before mutable state side effects, container forwarding preserved, and DigitalOcean setup guidance aligned to the chosen non-root workflow.

## Reproduction Assessment

Yes. The linked bug gives deterministic DigitalOcean root SSH plus `openclaw doctor --fix` steps, and current main shows both entrypoints can reach startup while state defaults under the invoking home directory.

## Solution Assessment

No, not yet. Blocking root early in both entrypoints is a plausible narrow fix, but the branch still needs DigitalOcean setup alignment, a changelog entry, and a runner-independent eUID test before it is the maintainable solution.
