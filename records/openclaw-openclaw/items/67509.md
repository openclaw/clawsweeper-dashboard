## Summary

Keep open: the PR is a plausible fix for the linked root-execution bug, but the latest branch still needs maintainer policy review because it would reject commands that current DigitalOcean setup docs run from a root SSH shell, and it is missing required changelog/test cleanup.

## What This Changes

The PR adds a CLI root/effective-root guard with explicit allow-root and container-forwarding bypasses, wires it into modern and legacy entrypoints, and adds focused unit tests.

## Best Possible Solution

Land this guard or an equivalent early-entrypoint protection after the DigitalOcean root-shell path is reconciled, the user-facing changelog entry is added, UID/eUID tests are stable on root runners, and the container child bypass remains narrow.

## Solution Assessment

No, not as-is. Early guarding in both entrypoints is a maintainable direction, but the PR must first reconcile published root-shell setup docs and complete the changelog/test cleanup.
