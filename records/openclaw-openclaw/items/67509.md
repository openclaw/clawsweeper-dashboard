## Summary

Keep open. The linked root-execution bug is real and not implemented on current main, but this branch is not merge-ready because it changes root-session CLI behavior without reconciling the documented DigitalOcean setup path and still needs small policy/test hygiene fixes.

## What This Changes

The PR adds a CLI root guard module and tests, then wires it into the modern and legacy CLI entrypoints to block real or effective UID 0 unless explicit root or container-forwarding overrides are present.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off, with both CLI entrypoints guarded before mutable state side effects, container forwarding preserved, DigitalOcean guidance aligned, and the changelog/test gaps closed.

## Reproduction Assessment

Yes. The linked bug gives deterministic DigitalOcean root SSH plus `openclaw doctor --fix` steps, and current main shows unguarded entrypoints with mutable state defaulting under the invoking home directory.

## Solution Assessment

No, not yet. Early root refusal is a plausible narrow fix, but the branch must first reconcile the root-session DigitalOcean setup path, add the required changelog entry, and harden the non-root eUID test.
