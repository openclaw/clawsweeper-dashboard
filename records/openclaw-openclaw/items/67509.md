## Summary

Keep open: current main still has unguarded root CLI startup paths, and this PR appears to address the linked state-corruption bug without a blocking correctness finding. The patch covers both primary and legacy entrypoints, preserves container-forwarded execution, and handles effective-root wrappers after prior review feedback.

## What This Changes

The PR adds a root-execution guard module, wires it into the main and legacy CLI entrypoints, and adds Vitest coverage for root, euid, override, and container-bypass behavior.

## Best Possible Solution

Land a guarded CLI startup path that blocks accidental root execution before state/config work, while preserving explicit root opt-in and forwarded container execution.

## Reproduction Assessment

Yes. Current main source shows both entrypoints can reach CLI startup without a root check, and the linked DigitalOcean report gives concrete root-user steps that explain the state and systemd corruption path.

## Solution Assessment

Yes. The proposed guard is narrow for the reported failure: it runs before normal state/config startup, covers the legacy path, checks effective UID, and keeps explicit/container bypasses scoped; no safer smaller code fix is apparent from this review.
