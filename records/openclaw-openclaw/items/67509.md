## Summary

Keep open. Current main still has no CLI-wide root guard, and this PR remains the direct implementation candidate for the linked DigitalOcean root-run crash/state-corruption bug, but the latest diff is not merge-ready because it would reject commands from the documented root-shell DigitalOcean setup path and still lacks the required user-facing changelog entry.

## What This Changes

The PR adds a CLI root guard module with explicit override/container-forwarding bypass, wires it into modern and legacy CLI entrypoints, and adds unit tests for real/effective UID handling.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off, with the DigitalOcean setup path reconciled, a changelog entry added, both entrypoints guarded before mutable state side effects, effective UID handling retained, and container forwarding preserved.

## Reproduction Assessment

Yes, source-reproducible with high confidence. The linked bug gives deterministic DigitalOcean root SSH plus `openclaw doctor --fix` steps, and current main shows unguarded entrypoints plus state defaulting under the invoking home directory.

## Solution Assessment

No, not as-is. Early guarding in both entrypoints is a plausible narrow fix, but the documented root-shell DigitalOcean workflow and required changelog/test hardening need to be resolved before merge.
