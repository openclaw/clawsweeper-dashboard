## Summary

Keep open. Current main still lacks the root guard this PR targets, and the author has addressed several earlier review defects, but the patch is not merge-ready because the DigitalOcean root-session docs and required changelog are still not aligned with the new CLI behavior.

## What This Changes

The PR adds a CLI root guard and tests, then wires modern and legacy entrypoints to reject real or effective UID 0 unless explicit root or container-forwarding bypasses are present.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off, with both CLI entrypoints guarded early, real/effective UID covered, intentional container forwarding preserved, and DigitalOcean plus changelog text updated.

## Reproduction Assessment

Yes. #67478 gives deterministic root-SSH reproduction steps, and current main still lets root invocations reach CLI startup while mutable state defaults through the invoking user's home directory.

## Solution Assessment

No, not yet. The code direction is a plausible narrow fix and prior review defects appear addressed, but the docs/changelog mismatch and root-policy choice must be resolved before this is the best maintainable solution.
