The PR adds a CLI root guard module and tests, wires it into the modern and legacy CLI entrypoints, and allows explicit or container-forwarded root bypasses.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off, with docs and changelog aligned: guard both entrypoints before mutable state/config side effects, check real and effective UID, preserve intentional container forwarding, and document the supported non-root or explicit-override workflow for DigitalOcean/1-Click users.
