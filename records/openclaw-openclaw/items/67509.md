## Summary

Keep open. The PR addresses a source-reproducible root-run corruption bug and current main still lacks an equivalent guard, but it is not merge-ready without real behavior proof plus docs/changelog reconciliation for root-shell install flows.

## What This Changes

The PR adds an `assertNotRoot` CLI guard, wires it into modern and legacy entrypoints, and tests uid/euid blocking, root override, Windows, and container-bypass cases.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off, with both entrypoints guarded before mutable startup work, euid-aware root detection, preserved container forwarding, clear non-root service-user guidance, and a changelog entry.

Yes, at source level: run a mutating `openclaw` command as root on current main and the entrypoints do not block while state resolves through the root home. I did not run a live DigitalOcean droplet reproduction in this read-only review.

## Solution Assessment

No, not as-is. The guard shape is a plausible narrow core fix, but merge should wait for docs/changelog updates, real behavior proof, and maintainer acceptance of the root override and container bypass policy.
