## Summary

Keep open. Current main still lacks the root guard, and the PR is a focused implementation for the linked root-state corruption bug, but it still introduces a documented root-session setup break that should block merge until the DigitalOcean guidance or install flow is reconciled.

## What This Changes

The PR adds a CLI root guard with tests and wires it into both modern and legacy CLI entrypoints to refuse real or effective UID 0 unless an explicit root or container-forwarding bypass is present.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off, with both CLI entrypoints guarded before mutable state side effects and the DigitalOcean/1-Click setup guidance aligned with the chosen non-root or explicit-override workflow.

## Reproduction Assessment

Yes. The linked bug gives deterministic DigitalOcean root SSH plus doctor --fix steps, and current main shows both CLI entrypoints can still reach startup while mutable state resolves under the invoking user's home directory.

## Solution Assessment

No, not yet. The core guard is a plausible narrow fix, but the branch should first reconcile root-session setup docs or installer behavior, add the required changelog entry, and make the non-root geteuid test independent of host runner UID.
