## Summary

Keep this PR open: it targets a valid root-state corruption bug from #67478 and has addressed several earlier code-review issues, but it is not merge-ready because the repository still publishes root-session DigitalOcean setup commands that the new guard would reject and the required user-facing changelog entry is missing.

## What This Changes

The PR adds an `assertNotRoot` CLI guard with UID/EUID checks, explicit and container-forwarding bypasses, modern and legacy entrypoint integration, and unit coverage.

## Best Possible Solution

Align the root policy and setup surfaces: land the early UID/EUID guard with non-root VPS/DigitalOcean instructions and changelog coverage, or scope protection to the image wrapper if root-based installs remain supported.

## Reproduction Assessment

Yes, at source level. Current main has no UID/EUID guard in either CLI entrypoint, state defaults to `~/.openclaw`, and #67478 gives a deterministic root-shell DigitalOcean path; I did not run the destructive droplet scenario.

## Solution Assessment

No, not as a complete merge candidate. The guard is a plausible fix direction, but the branch must first reconcile published root-session setup docs and add the required changelog entry.
