## Summary

Keep open. Current main still lacks the root guard and the PR is a focused implementation for the linked DigitalOcean root-session bug, but the branch still conflicts with documented DigitalOcean root-shell setup and needs maintainer policy/docs alignment before merge.

## What This Changes

The PR adds a CLI root guard, tests, and entrypoint wiring to block real or effective UID 0 unless an explicit root override or container-forwarding hint is present.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off, with DigitalOcean setup guidance reconciled, a changelog entry added, both entrypoints guarded before mutable state side effects, effective UID handling retained, and container forwarding preserved.

## Reproduction Assessment

Yes. The linked bug gives deterministic DigitalOcean root SSH plus `openclaw doctor --fix` steps, and current-main source shows both entrypoints can reach startup while state defaults under the invoking home directory.

## Solution Assessment

No, not as-is. Early guarding in both entrypoints is a plausible narrow fix, but the documented root-shell DigitalOcean workflow and user-facing release notes must be aligned first.
