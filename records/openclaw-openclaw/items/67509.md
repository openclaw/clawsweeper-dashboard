## Best Possible Solution

Land this PR or an equivalent early-entrypoint guard after aligning the root/VPS docs, adding a user-facing changelog entry, and confirming the global root-refusal policy.

## Solution Assessment

Mostly yes for the code direction, but not as a complete merge candidate. Guarding both entrypoints before mutable CLI startup is the narrow fix, while docs, changelog, and root-policy alignment still need to be finished.
