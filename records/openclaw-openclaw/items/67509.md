## Summary

Keep open. The linked root-execution bug is still valid on current main, and this PR is a focused implementation, but it still conflicts with DigitalOcean root-session docs and lacks the required changelog entry.

## What This Changes

Adds a CLI root guard module with tests and wires modern and legacy entrypoints to reject real/effective UID 0 unless explicit root or container-forwarding bypasses are present.

## Best Possible Solution

Land this PR or an equivalent core fix after docs and changelog are aligned: guard both CLI entrypoints early, check real and effective UID, and preserve intentional container forwarding.

## Reproduction Assessment

Yes. The linked bug has deterministic root-SSH steps, and current main still lets both CLI entrypoints reach startup while mutable state defaults through the invoking user's home directory.

## Solution Assessment

No, not yet. The guard is a plausible narrow fix and the prior review defects appear addressed, but DigitalOcean setup guidance and the changelog must be reconciled before this is the best maintainable change.
