## Summary

Keep open. Current main still lacks the root guard this PR adds, and the linked DigitalOcean failure remains source-reproducible; however the patch still conflicts with the published root-shell DigitalOcean setup path and needs that policy/docs work before merge.

## What This Changes

This PR adds an early CLI root guard with an explicit env override, real/effective UID checks, a container-forwarding bypass, modern and legacy entrypoint wiring, and focused unit tests.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off, with the DigitalOcean setup path updated, a changelog entry added, effective-UID handling retained, both entrypoints guarded before mutable state side effects, and container forwarding preserved.

## Reproduction Assessment

Yes. The linked bug gives deterministic DigitalOcean root SSH plus `openclaw doctor --fix` steps, and current main has unguarded entrypoints plus `~/.openclaw` state resolution under the invoking home directory.

## Solution Assessment

No, not as-is. Early guarding in both entrypoints is a plausible narrow fix, but the published DigitalOcean root-shell setup path and required changelog/test polish need to be handled before merge.
