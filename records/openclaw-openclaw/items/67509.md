## Summary

Keep open: the root guard addresses a real current-main bug, but the proposed patch still conflicts with published root-shell DigitalOcean setup docs, lacks the required changelog entry, and has one root-runner-sensitive unit test.

## What This Changes

The PR adds a CLI root/effective-root guard with explicit allow-root and container-forwarding bypasses, wires it into modern and legacy entrypoints, and adds focused unit tests.

## Best Possible Solution

Land early-entrypoint root protection after maintainer-approved DigitalOcean/root-shell guidance, a changelog entry, and stable UID/eUID tests are in place.

## Reproduction Assessment

Yes. The linked issue gives deterministic root SSH plus openclaw doctor --fix steps, and current main still has unguarded entrypoints with HOME-derived ~/.openclaw state; I did not live-run the destructive Droplet scenario.

## Solution Assessment

No, not as-is. Early guarding in both entrypoints is a maintainable direction, but the PR needs to reconcile published root-shell setup guidance and complete the changelog/test cleanup first.
