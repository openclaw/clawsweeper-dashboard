## Summary

Keep open: current main does not implement the root guard and the PR addresses a real #67478 failure mode, but it is not merge-ready because the current DigitalOcean docs still run protected commands from a root shell and the required changelog/test cleanup is missing.

## What This Changes

The PR adds a CLI root guard and tests, then wires modern and legacy entrypoints to reject real or effective UID 0 unless explicit root or container-forwarding bypasses are present.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off, with both entrypoints guarded before mutable state/config side effects, real and effective UID covered, container forwarding preserved, and DigitalOcean guidance plus changelog aligned.

## Reproduction Assessment

Yes. #67478 gives deterministic root SSH and doctor steps, and current main still lets both entrypoints reach startup while resolveStateDir defaults mutable state to the invoking user's home directory.

## Solution Assessment

No, not yet. The root guard is a plausible narrow fix, but the branch should not merge until DigitalOcean setup guidance, the required changelog entry, and the small geteuid test robustness issue are resolved.
