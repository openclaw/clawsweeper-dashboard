## Summary

Keep open. Current main still lacks the root-execution guard this PR adds, and the linked DigitalOcean failure remains source-reproducible, but the patch still conflicts with the published root-shell setup path and needs maintainer policy/docs work before merge.

## What This Changes

This PR adds an early CLI root guard with an explicit env override, real/effective UID checks, a container-forwarding bypass, modern and legacy entrypoint wiring, and focused unit tests.

## Best Possible Solution

Land this PR or an equivalent core fix after maintainer sign-off, with the DigitalOcean setup path updated, a changelog entry added, effective-UID handling retained, both entrypoints guarded before mutable state side effects, and container forwarding preserved.

## Reproduction Assessment

Yes. The linked bug gives deterministic DigitalOcean root SSH plus openclaw doctor --fix steps, and current main still has unguarded entrypoints plus ~/.openclaw state resolution under the invoking home; I did not live-run the destructive scenario.

## Solution Assessment

No, not as-is. Early guarding in both entrypoints is a plausible narrow fix, but the public root-shell setup path and required changelog/test coverage need to be reconciled before merge.
