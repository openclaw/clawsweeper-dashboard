## Summary

Keep open. The PR targets a real root-state corruption bug and has already incorporated prior review fixes for container forwarding, the legacy entrypoint, and effective UID handling, but it still conflicts with the documented DigitalOcean root-shell setup path and needs release/test polish before merge.

## What This Changes

The PR adds a CLI root guard module, tests, and entrypoint wiring to block real or effective UID 0 unless an explicit root or container-forwarding bypass applies.

## Best Possible Solution

Land this root protection or an equivalent fix after aligning the DigitalOcean install path, preserving container forwarding, adding the changelog entry, and hardening the UID tests.

## Reproduction Assessment

Yes, from source and the linked bug report. #67478 gives deterministic root SSH plus `openclaw doctor --fix` steps, while current main has unguarded entrypoints and defaults mutable state to the invoking user's `~/.openclaw`.

## Solution Assessment

No, not as-is. Early guarding in both entrypoints is a plausible narrow fix, but the DigitalOcean root-shell workflow and user-facing release/test requirements must be reconciled first.
