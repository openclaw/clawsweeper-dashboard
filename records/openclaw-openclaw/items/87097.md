## Summary

Keep this PR open, but not merge-ready: the CLI hang it targets is a real availability problem, while the draft patch can override the existing agent timeout contract and still lacks current-main source-level command proof.

## What This Changes

Adds a pre-entry `agent --local` wall-clock timeout and forces the modern and legacy CLI entrypoints to exit after the command promise resolves.

## Best Possible Solution

Keep the availability fix direction, but derive the outer guard from the same CLI/config timeout contract or require an explicit `--timeout`, then land only with maintainer approval and current-main real command proof.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. The source path and related reports support the hang class, but the PR itself says current-main command proof is still missing.

## Solution Assessment

No, not as submitted. Bounding the local CLI lifetime may be the right direction, but the hard timeout must honor the existing timeout contract and the forced-exit policy needs maintainer acceptance.
