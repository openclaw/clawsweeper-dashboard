## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep strict recovery from treating whitespace as zero capture:** `src/shared/text/reasoning-tags.ts:88`
  - body: The strict-mode guard uses trimmed output, so an input like `"\n<think>secret"` can recover `secret` even though strict mode already captured a raw prefix. Telegram and message-tool strict stripping use this as a reasoning boundary, so whitespace-formatted output can expose an unclosed reasoning tail instead of failing closed.
  - confidence: 0.9
- **[P2] Emit sessions.changed after successful reset:** `src/gateway/server-methods/sessions.ts:996`
  - body: `sessions.reset` returns success without broadcasting `sessions.changed`, so other subscribed clients do not learn that the session ID and metadata changed after a reset. This can leave other tabs stale until a manual refresh or another mutation emits an event.
  - confidence: 0.87
- **[P2] Put a timeout around awaited transcribed hooks:** `src/auto-reply/reply/message-preprocess-hooks.ts:28-34`
  - body: `message:transcribed` is now awaited before the agent reply starts, but the await has no timeout. A slow or stuck hook handler can block the inbound reply path indefinitely for that message.
  - confidence: 0.84
- **[P2] Use execFile for workspace memory extraction:** `src/hooks/bundled/session-memory/handler.ts:185-187`
  - body: The new session-memory extraction path builds a shell command with interpolated script and session values. If either contains shell metacharacters, the shell can execute more than the intended Python script; use `execFile("python3", [extractScript, "--session", sessionId], ...)` instead.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: Security review needs attention because the PR branch introduces unrelated shell execution and a large generated backup tree.

Concerns:

- **[medium] Shell interpolation in bundled memory hook:** `src/hooks/bundled/session-memory/handler.ts:185`
  - body: `execAsync` is called with a shell command assembled from `extractScript` and `sessionId`, which can execute unintended shell syntax if path or session values contain metacharacters. Use `execFile` with an argv array if this feature remains.
  - confidence: 0.86
- **[low] Generated backup tree obscures supply-chain review:** `dist.bak/acp-cli-BpCQk37E.js:1`
  - body: The PR adds a large `dist.bak/*` generated JavaScript tree unrelated to the Telegram fix. Even if not imported, committing generated backup code makes the review surface too large and can hide executable changes from normal source review.
  - confidence: 0.78
