## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve strict mode when only whitespace was captured:** `src/shared/text/reasoning-tags.ts:91`
  - body: The strict-mode guard still uses `!result.trim()`, so input such as `"\n<think>secret"` appends `secret` even though strict mode already captured a raw prefix. Telegram and message-tool callers use strict stripping as a reasoning-safety boundary, so whitespace-formatted model output can expose an unclosed reasoning tail.
  - confidence: 0.92
- **[P2] Restore reset broadcasts:** `src/gateway/server-methods/sessions.ts:998`
  - body: `sessions.reset` now responds successfully without emitting `sessions.changed`. Other subscribed clients rely on that event to reload the new session id and metadata, so resets from one tab can leave other clients stale until another mutation or refresh occurs.
  - confidence: 0.87
- **[P2] Bound awaited transcribed hooks:** `src/auto-reply/reply/message-preprocess-hooks.ts:31-38`
  - body: `message:transcribed` is now awaited before the agent reply starts, but the await has no timeout. A slow or stuck internal hook can block the inbound reply path indefinitely for that message, unlike the previous fire-and-forget behavior.
  - confidence: 0.84
- **[P2] Use argv execution for memory extraction:** `src/hooks/bundled/session-memory/handler.ts:186`
  - body: The new memory extraction path builds a shell command with interpolated `extractScript` and `sessionId`. If either value contains shell metacharacters, the shell can execute more than the intended Python script; use `execFile("python3", [extractScript, "--session", sessionId], ...)` instead.
  - confidence: 0.86

## Security Review
