## What This Changes

## Best Possible Solution

Land a reviewed additive SDK/runtime seam that keeps auth, model/profile policy, timeouts, and structured parsing host-owned while preserving `mediaUnderstanding.extractStructuredWithModel` as the narrower provider-owned media lane.

Yes, pending maintainer approval: putting generic structured inference under `runtime.llm` fits the existing host-owned completion path and keeps provider-explicit media extraction separate. The profile hardening now covers explicit and embedded profile selection before host auth preparation.
