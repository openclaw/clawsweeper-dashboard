## Summary

Keep this PR open. Current main still does not render Created or Last Used in the paired-device table, while the underlying pairing/token data already carries the timestamps needed for the feature. The PR is narrow CLI display work and is not superseded, but it should be rebased onto current main and covered before merge.

## What This Changes

This PR adds Created and Last Used columns to the human `openclaw devices list` paired-device table, deriving Last Used from the latest non-revoked token `lastUsedAtMs` while leaving JSON output unchanged.

## Best Possible Solution

Rebase the PR onto current main, keep the timestamp display as a CLI-only addition using existing relative-time formatting and token summaries, preserve the current sanitization/access-upgrade behavior, add focused `devices-cli` coverage, then land if maintainers accept active-token-only Last Used semantics.

## Reproduction Assessment

Yes for the missing feature: inspect or run `openclaw devices list` on current main with a paired device, and the paired table is defined with only Device/Roles/Scopes/Tokens/IP. The source and tests provide a high-confidence check without needing a live gateway.

## Solution Assessment

Yes, the proposed direction is a narrow maintainable fix because it uses existing timestamp formatting and already-summarized token metadata without changing the JSON contract. The safer merge path is to rebase first and add coverage for both human table output and unchanged JSON.
