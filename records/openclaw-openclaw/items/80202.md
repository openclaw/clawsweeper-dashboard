The issue reports that OpenClaw 2026.5.9-beta.1 fails to register the bundled HuggingFace, OpenRouter, and xAI plugins because their register-time API lacks `registerModelCatalogProvider`.

## Best Possible Solution

Reproduce the packaged gateway startup with the affected stock plugins enabled, fix the shared plugin API injection or package-resolution path centrally, and add a packaged startup regression before the stable 5.9 line proceeds.

## Reproduction Assessment

No. The reporter provided concrete Windows logs and the beta tarball confirms the relevant plugin call surface, but I did not establish a high-confidence current-main packaged gateway reproduction path.

## Solution Assessment

Unclear until the packaged startup path is isolated. The best fix should be in the shared plugin API builder, registry, loader, or package-resolution seam, not by removing catalog registration from individual provider plugins.
