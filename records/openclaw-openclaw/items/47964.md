## Summary

Keep open. Current main still has the two source-reproducible auth-profile compatibility gaps, and https://github.com/openclaw/openclaw/pull/75030 is an open closing PR for this issue; https://github.com/openclaw/openclaw/pull/77588 was closed as superseded by that contributor PR.

## What This Changes

The issue asks OpenClaw to handle present-but-unusable OpenAI Codex OAuth auth profiles caused by legacy OAuth field aliases or stale api_key mode overrides.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/75030 or an equivalent narrow patch that normalizes OpenAI Codex OAuth aliases on read and emits a redacted mode-mismatch diagnostic, then close this issue.

## Reproduction Assessment

Yes. By source inspection, an OpenAI Codex OAuth profile stored only with legacy OAuth aliases will not become a usable PI OAuth credential, and a stale api_key config-mode override filters an existing OAuth profile before the generic missing-auth error.

## Solution Assessment

Yes. The best fix is the narrow linked-PR direction: load-time normalization for known OpenAI Codex OAuth aliases plus a metadata-only diagnostic for config/store mode mismatches.
