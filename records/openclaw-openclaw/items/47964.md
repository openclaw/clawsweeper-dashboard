## What This Changes

The issue asks whether OpenClaw should normalize or diagnose older OpenAI Codex OAuth auth-profile formats that appear to log in successfully but are unusable at runtime.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/75030 or an equivalent narrow patch that normalizes known OpenAI Codex OAuth aliases on persisted-profile load and emits secret-free diagnostics for config/store mode conflicts.

## Reproduction Assessment

Yes, from source inspection rather than a live OAuth run. Current main still lacks persisted OAuth alias normalization, and auth-profile ordering can filter out a stored OAuth credential when config forces api_key mode.

## Solution Assessment

Yes. The open linked PR is the narrow maintainable path because it fixes compatibility at persisted-profile load time and adds metadata-only diagnostics without changing provider policy or exposing token values.
