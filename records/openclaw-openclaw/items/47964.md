## Summary

Keep open. Current main still leaves the reported OpenAI Codex OAuth compatibility gap in place, and the linked open PR #75030 is the active implementation path for both legacy alias normalization and stale mode-mismatch diagnostics.

## What This Changes

The issue asks whether OpenAI Codex OAuth auth profiles with legacy field names or stale mode overrides should be migrated or diagnosed instead of silently falling back.

## Best Possible Solution

Land PR #75030, or an equivalent narrow patch, with regression coverage for legacy OAuth aliases, stale mode masking, and token-redacted diagnostics.

## Reproduction Assessment

Yes. A source-traced reproduction is an openai-codex OAuth profile containing only alias fields such as access_token/refreshToken/expires_at, which current main leaves unnormalized and PI conversion rejects; the stale api_key mode override path is also visible in current eligibility code.

## Solution Assessment

Yes. Normalizing known aliases at auth-profile load and adding a secret-free mode-mismatch diagnostic is the narrow maintainable fix for the two reported present-but-unusable credential paths.
