## Summary

Keep open: current main still probes only the plugin root for external channel contract artifacts, while two open closing PRs already target the dist/ fallback fix.

## What This Changes

The issue asks OpenClaw to load external channel secret contract APIs from npm plugin dist/ output so Discord SecretRef tokens resolve at gateway startup.

## Best Possible Solution

Review and land one dist-aware resolver fix with regression coverage and changelog, then close this issue after the merge and release provenance are clear.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: a plugin record whose contract exists only at rootDir/dist/secret-contract-api.js will not be found by current main. The issue discussion adds live Discord confirmations and a root-level shim workaround that restores startup.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve root-level precedence while adding rootDir/dist probes for the same contract artifact names, which is the direction of the linked open PRs.
