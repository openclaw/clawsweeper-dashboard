## Summary

Keep open: current main still has the reported root-only external channel contract lookup, and open closing PRs #77246 and #77247 already target the bug, so this issue should not close before one fix merges.

## What This Changes

The issue reports that npm-installed channel plugin secret contracts under `dist/` are not discovered, leaving Discord SecretRefs unresolved at gateway startup.

## Best Possible Solution

Land one canonical dist-aware resolver patch with regression coverage, then close this issue as implemented after the merge and release provenance are clear.

## Reproduction Assessment

Yes, source-level: current main checks only root-level `secret-contract-api` and `contract-api` filenames and returns undefined before loading any dist-only external contract. The issue discussion also includes multiple live confirmations and a root-level shim workaround that restores Discord startup.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve root precedence while adding `rootDir/dist` as a fallback for the same contract artifact names with regression coverage; because open PRs already implement that direction, this issue should remain open until one merges.
