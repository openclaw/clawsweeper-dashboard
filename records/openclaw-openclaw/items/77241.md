## Summary

## What This Changes

External npm channel plugin secret-contract sidecars should be resolved from dist/ so @openclaw/discord SecretRef tokens load at gateway startup.

## Best Possible Solution

Land one canonical dist-aware resolver/test PR, then close this issue as implemented once the merge and release provenance are clear.

## Reproduction Assessment

Yes, source-level. A current-main external plugin record with only dist/secret-contract-api.* makes the resolver return null before assignment collection, and the issue discussion includes multiple live confirmations plus a working root shim.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve root precedence and boundary checks while probing rootDir/dist for the same contract artifact names, with regression coverage.
