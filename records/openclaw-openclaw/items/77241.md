## Summary

Keep open: current main and the latest shipped v2026.5.3 release still use a root-only external channel secret-contract lookup, and two open closing PRs already target the dist fallback, so this should remain open until one lands.

## What This Changes

Resolve external channel secret contract files when npm-installed channel plugins place compiled contract artifacts under dist/.

## Best Possible Solution

Land one dist-aware resolver with regression coverage, preserving root precedence and boundary checks; track any remaining Discord readiness problem separately after the contract loader fix ships.

## Reproduction Assessment

Yes, source-level. Current main and v2026.5.3 only probe the plugin root for external channel contract files, and the issue discussion includes multiple live Discord confirmations plus a root re-export shim workaround.

## Solution Assessment

Yes. The narrow maintainable fix is to search rootDir/dist after the existing root candidates for the same contract artifact names, while keeping current boundary checks and root precedence.
