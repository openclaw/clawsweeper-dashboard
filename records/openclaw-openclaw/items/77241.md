## Summary

Keep open: current main still has the reported root-only external channel contract lookup, and open PR #77247 already targets this bug with closing syntax, so this issue should stay open until a fix merges.

## What This Changes

The issue asks OpenClaw to discover external channel secret contract artifacts from npm package `dist/` output so Discord SecretRefs are loaded for runtime startup.

## Best Possible Solution

Land one canonical dist-aware resolver patch with focused regression coverage, then close this issue once the merge and release provenance are clear.

## Reproduction Assessment

Yes, source-level. Current main cannot discover a dist-only external channel contract because the resolver only probes package-root filenames and the loader returns `undefined` before assignments can be collected.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve root precedence while adding `rootDir/dist` as a fallback for the same contract artifact names; PR #77247 already targets that direction.
