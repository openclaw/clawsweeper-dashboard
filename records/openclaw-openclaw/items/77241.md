## Summary

Keep open: current main still has the reported root-only external channel contract lookup, and this issue is already paired with open closing PRs #77246 and #77247, so it should not close before one fix merges.

## What This Changes

The issue reports that external channel secret contract lookup only probes the plugin package root, so npm-installed Discord packages with dist/secret-contract-api.js never load SecretRef assignments.

## Best Possible Solution

Review and land one dist-aware resolver PR with regression coverage, then close this issue as implemented after the merge and release provenance are clear.

## Reproduction Assessment

Yes, source-level: current main checks only rootDir for secret-contract-api/contract-api and returns undefined before loading any dist-only external contract. The issue discussion also includes live Discord confirmations and a root-level shim workaround that restores startup.

## Solution Assessment

Yes. The narrow maintainable fix is to keep root precedence and boundary checks while probing rootDir/dist for the same contract artifact names; because open PRs already implement that direction, the issue should remain open until one merges.
