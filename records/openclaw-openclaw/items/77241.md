## Summary

Keep open: current main still has the reported root-only external channel contract lookup, the supported npm/Discord SecretRef path is documented, and open closing PRs #77246 and #77247 already target this bug. This issue should not close until one fix merges.

## What This Changes

Resolve external channel plugin secret contract artifacts from a `dist/` subdirectory so npm-installed Discord SecretRefs are loaded.

## Best Possible Solution

Land one dist-aware resolver change with regression coverage, keep root-level precedence and boundary checks, then follow up separately on any remaining Discord readiness symptom.

Yes, source-level: current main only probes the plugin root for contract artifacts and returns before loading any dist-only external contract. The issue discussion adds live Discord confirmations and a root shim workaround, but I did not run a live Discord gateway in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to probe `rootDir/dist` as a fallback while preserving existing root precedence and boundary validation; the open closing PRs already implement that direction.
