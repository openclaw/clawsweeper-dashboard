## Summary

Keep open: current main still has the reported root-only external channel contract lookup, and the shipped Discord npm package places the secret contract under dist, so the failure path remains valid and narrow.

## What This Changes

The issue requests that external channel secret contract loading find npm plugin contract artifacts emitted under dist so Discord SecretRefs resolve at gateway startup.

## Best Possible Solution

Extend external channel contract resolution to probe root and dist contract artifacts with root precedence, then add an installed npm-style regression test for a dist-only secret contract.

## Reproduction Assessment

Yes, source-level. Current main has no dist candidate, and the 2026.5.3 Discord tarball contains package/dist/secret-contract-api.js but no package/secret-contract-api.js; I did not run a live gateway in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is to have the external contract resolver probe dist/secret-contract-api.* and dist/contract-api.* after root-level candidates, while retaining boundary-file checks and adding regression coverage.
