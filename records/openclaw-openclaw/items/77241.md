## Summary

Keep open: current main still probes only package-root contract sidecars for external channel plugins, so a dist-only npm package contract can be skipped before Discord SecretRefs are assigned.

## What This Changes

The issue requests that external channel secret contract loading find dist-only npm plugin artifacts so Discord SecretRef-backed tokens load during gateway startup.

## Best Possible Solution

Extend external channel contract resolution to probe root and dist contract artifacts with root precedence, then add an installed npm-style dist-only regression test and a changelog entry.

## Reproduction Assessment

Yes, source-level. A plugin record whose rootDir contains only dist/secret-contract-api.js currently makes resolvePluginContractApiPath return null, and the external loader then skips the contract before runtime assignments are collected.

## Solution Assessment

Yes. The narrowest maintainable repair is to add dist/secret-contract-api.* and dist/contract-api.* probes after root-level probes, preserving root precedence and boundary-file validation.
