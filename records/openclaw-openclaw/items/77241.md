## Summary

Keep open: current main still has the root-only external channel contract lookup, and an open closing PR already targets the same bug, so this issue should remain open until the canonical fix merges.

## What This Changes

The issue asks OpenClaw to discover npm-installed channel plugin secret contract artifacts when they are compiled under `dist/`, so Discord SecretRef tokens resolve at gateway startup.

## Best Possible Solution

Land one canonical dist-aware resolver change that preserves root precedence, adds a dist-only regression test, and records the user-facing Discord SecretRef fix in `CHANGELOG.md`.

## Reproduction Assessment

Yes, source-level: current main resolves external channel secret contracts only from the plugin package root, then skips assignment collection when no contract file is found. The discussion adds multiple operator confirmations that a root-level shim restores Discord startup.

## Solution Assessment

Yes. The narrow maintainable fix is to add `rootDir/dist` as a fallback for the same contract artifact names while keeping package-root precedence and covering the dist-only npm package layout in tests.
