## Summary

Keep open: current main still misses dist-only external channel secret contracts, and open closing PR #77247 already tracks the fix. This should close only after a canonical PR lands.

## What This Changes

The issue reports that external npm channel plugins with dist-only secret contract files are not discovered, leaving Discord SecretRefs unresolved at gateway startup.

## Best Possible Solution

Land one canonical dist-aware resolver fix with regression coverage and a changelog entry, then let the linked closing PR close this issue after merge.

## Reproduction Assessment

Yes, source-level: current main only probes root-level contract filenames and returns undefined before loading a dist-only external channel contract. The issue comments also include multiple live confirmations and a root-level shim workaround that restores Discord startup.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve root precedence while adding a `rootDir/dist` fallback for the same contract artifact names with regression coverage; because an open closing PR already targets that direction, this issue should remain open until one fix merges.
