## Summary

Keep open: current main still has the reported root-only external channel contract lookup, and two open PRs explicitly reference this issue as their fix, so the issue should remain open until one lands.

## What This Changes

Resolve external channel secret contract artifacts from npm plugin dist/ subdirectories so Discord SecretRefs load for installed plugins.

## Best Possible Solution

Land one narrow fix that preserves root-level precedence, adds dist/secret-contract-api.* and dist/contract-api.* fallback probing inside the plugin root boundary, and includes a dist-only external plugin regression test.

## Reproduction Assessment

Yes, source-level: a plugin record whose rootDir contains only dist/secret-contract-api.* will return null from the current resolver and skip external secret assignments. I did not run a live gateway with Discord credentials in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable solution is the linked PR direction: search root first, then dist, for the existing contract artifact names while keeping boundary checks and adding regression coverage.
