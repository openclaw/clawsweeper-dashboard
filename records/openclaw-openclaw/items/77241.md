## Summary

Keep open: current main still has the root-only external channel contract lookup, and open closing PR #77247 already targets this bug, so this issue should remain open until a canonical fix merges.

## What This Changes

Make external channel secret contract loading discover npm plugin contract sidecars under `dist/` so Discord SecretRef tokens resolve at gateway startup.

## Best Possible Solution

Land one canonical dist-aware resolver change that preserves root precedence, adds dist-only regression coverage, and records the user-facing fix in `CHANGELOG.md`.

## Reproduction Assessment

Yes, source-level: current main resolves only root-level contract files, so a dist-only npm channel contract returns undefined before SecretRef assignments are collected. The issue discussion adds multiple live confirmations and a root-level shim workaround that restores Discord startup.

## Solution Assessment

Yes. The narrow maintainable fix is to add `rootDir/dist` as a fallback for the same contract artifact names while keeping package-root precedence, with a regression test for a dist-only external channel plugin.
