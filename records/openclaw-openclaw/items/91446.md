## Summary

Keep open: this targets a real TUI/subagent tool-exposure gap, but the patch is not merge-ready because it can expose `sessions_spawn` where local TUI still lacks a supported spawn execution path, misses per-agent allowlists, and has insufficient real behavior proof.

## What This Changes

The branch changes `createOpenClawTools` to include `sessions_spawn` in embedded sessions when `agents.defaults.subagents.allowAgents` is a non-empty array.

## Best Possible Solution

Make TUI spawn availability mirror a supported execution path: gateway-bound embedded runs can expose `sessions_spawn` when the effective requester allowlist permits it, while local mode either gains real local/in-process dispatch or remains documented as Gateway-only, with focused tests and real TUI/tool-list proof.

## Reproduction Assessment

Yes, by source inspection for the local embedded path: current main sets local TUI into embedded mode and the current gate excludes `sessions_spawn` unless gateway binding is explicitly allowed. I did not run a live Windows/npm TUI reproduction.

## Solution Assessment

No. `createOpenClawTools` is a plausible layer, but the condition should mirror both the effective requester allowlist and a supported spawn execution context rather than exposing a Gateway-dependent tool from config alone.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Do not expose spawn where local TUI cannot execute it:** `src/agents/openclaw-tools.ts:397-398`
  - body: This makes `hasSubagentConfig` a third way to materialize `sessions_spawn` in embedded mode. Local TUI sets embedded mode but does not install a gateway request context; `spawnSubagentDirect` only uses in-process dispatch when that context exists, otherwise it falls back to `callGateway` for the child `agent` run. With `openclaw chat`/`--local`, this can advertise a tool that still fails at execution instead of providing working subagent spawn. Keep the gate tied to a supported gateway-bound/local dispatch path, or add that path with coverage.
  - confidence: 0.87
- **[P2] Mirror the effective subagent allowlist:** `src/agents/openclaw-tools.ts:394-396`
  - body: The new gate only treats `agents.defaults.subagents.allowAgents` as configured. Runtime target validation and `agents_list` both resolve per-agent `subagents.allowAgents` first and fall back to defaults, and the docs advertise the per-agent path, so an embedded TUI agent with only a per-agent allowlist still will not get `sessions_spawn`. Please derive the effective requester allowlist here before deciding to materialize the tool.
  - confidence: 0.9
