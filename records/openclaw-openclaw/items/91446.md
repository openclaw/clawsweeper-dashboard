## Summary

Keep open: the PR addresses a real embedded TUI tool-exposure bug, but the proposed gate is too narrow because it only checks global defaults while the runtime and docs resolve per-agent `subagents.allowAgents` first; contributor-side real behavior proof is also still insufficient.

## What This Changes

The PR changes `src/agents/openclaw-tools.ts` so embedded runs include `sessions_spawn` when `agents.defaults.subagents.allowAgents` is a non-empty array.

## Best Possible Solution

Use the requester agent's effective subagent allowlist for embedded `sessions_spawn` materialization, cover default and per-agent configurations, and land only with redacted TUI or tool-inventory proof.

## Reproduction Assessment

Yes, by source inspection. Current main sets local TUI into embedded mode and the OpenClaw tool gate omits `sessions_spawn` there unless `allowGatewaySubagentBinding` is true, while the linked config/docs path expects allowlisted subagents to be usable.

## Solution Assessment

No. `createOpenClawTools` is the right layer, but this PR should mirror the effective requester allowlist used by `agents_list` and `sessions_spawn`, then add focused coverage and real behavior proof.
