## Summary

Keep open: the patch is not merge-ready because it exposes a Gateway-dependent `sessions_spawn` tool in embedded TUI based only on default config, while local execution is unsupported or unproven and per-agent allowlists remain on the old missing-tool path.

## What This Changes

The PR changes `src/agents/openclaw-tools.ts` so embedded tool construction includes `sessions_spawn` when `agents.defaults.subagents.allowAgents` is a non-empty array.

## Best Possible Solution

Gate embedded `sessions_spawn` on the effective per-agent/default allowlist and a proven execution path: gateway-bound embedded runs can expose it, while local mode needs real dispatch support or explicit exclusion.

## Reproduction Assessment

Yes. Source inspection shows local TUI enters embedded mode and current tool assembly excludes `sessions_spawn` unless `allowGatewaySubagentBinding` is true; I did not run a live TUI flow.

## Solution Assessment

No. `createOpenClawTools` is a plausible materialization layer, but this patch should mirror effective requester policy and avoid exposing a Gateway-dependent tool where local embedded execution is unsupported or unproven.
