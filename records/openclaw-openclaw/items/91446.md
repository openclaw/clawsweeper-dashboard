## Summary

Keep open: the linked TUI tool-exposure bug is still real, but this PR is not merge-ready because it exposes `sessions_spawn` in embedded local sessions without proving execution and only checks the defaults allowlist instead of the effective requester allowlist.

## What This Changes

The PR adds a defaults-based `agents.defaults.subagents.allowAgents` check to `createOpenClawTools` so embedded sessions can materialize `sessions_spawn` without `allowGatewaySubagentBinding`.

## Best Possible Solution

Gate embedded `sessions_spawn` on the same effective allowlist used by runtime policy, then either prove local embedded spawn execution or keep local mode hidden/documented while enabling only executable gateway-bound contexts.

## Reproduction Assessment

Yes, by source inspection: local TUI sets embedded mode and current main hides `sessions_spawn` unless `allowGatewaySubagentBinding` is true. I did not run a live TUI spawn because this review was read-only.

## Solution Assessment

No. `createOpenClawTools` is the right materialization layer, but the fix should share effective allowlist policy and resolve or explicitly exclude local embedded execution.
