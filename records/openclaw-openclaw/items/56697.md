## Summary

Keep this PR open. Current main still does not add tool-specific examples to empty/missing parameter validation errors, but the submitted patch is not merge-ready because its edit example contradicts the current `edits[]` contract and it lacks the required user-facing changelog/test follow-through.

## What This Changes

The PR modifies `src/agents/pi-tools.params.ts` to add hard-coded read/write/edit parameter examples to validation errors, thread `toolName` into the validation helper, and accumulate unused accepted-key text.

## Best Possible Solution

Keep the narrow idea, but revise it so examples are generated or hard-coded against the current canonical schemas, especially `edit({"path":"...","edits":[{"oldText":"...","newText":"..."}]})`, with regression coverage for empty-object validation failures and a changelog entry. Leave retry-budget/non-retryable handling to the separate retry-loop work.

## Solution Assessment

No, not as currently written. Adding examples is a plausible small fix, but the PR must match the current runtime edit schema, remove or use dead accepted-key code, and add tests/changelog before merge.
