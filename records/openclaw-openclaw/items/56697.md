## Summary

Keep open. The requested validation-message improvement is still not on current main, but this PR is not merge-ready because its edit example no longer matches the current canonical `edits[]` contract, it leaves dead code behind, and it lacks required tests/changelog coverage.

## What This Changes

The PR adds hard-coded read/write/edit examples to tool parameter validation errors in `src/agents/pi-tools.params.ts` and threads `toolName` through the validation helper.

## Best Possible Solution

Keep the narrow example-guidance idea, but implement it against the current canonical schemas with focused regression tests and a single-line changelog entry; leave retry-budget behavior to #68070 or its successor.

## Reproduction Assessment

Yes for the validation-message surface: current main can be checked through `assertRequiredParams({}, REQUIRED_PARAM_GROUPS.read, "read")`, which still emits the generic retry suffix without an example. The live provider self-correction claim is not reproducible from the static PR alone.

## Solution Assessment

No, not as currently written. Adding examples is a plausible small fix, but the patch must use the current `edit({"path":"...","edits":[{"oldText":"...","newText":"..."}]})` shape, remove or use dead code, and add tests/changelog before merge.
