Keep open: the current-main problem still exists for truly malformed local skills, but this PR is not mergeable as-is because it misses two CLI readiness views, lacks real behavior proof, and is now behind the current loader shape.

## What This Changes

The PR adds parse-error plumbing for malformed skill frontmatter and displays that state in skills list text and JSON output.

## Best Possible Solution

Rebase onto the current local-loader/status contract and surface one parse-error state consistently through skills list, info, check, and JSON outputs with focused regression coverage and real CLI proof.

## Reproduction Assessment

Yes for the PR review findings: source inspection of the PR head and prior review comments show parseError is only surfaced in skills list while skill info and skills check remain misleading. The original user problem is also source-reproducible on current main for truly malformed SKILL.md files because local-loader.ts catches parse errors and returns null.

## Solution Assessment

No. The patch moves in the right direction, but it needs to retarget local-loader.ts and carry parse errors through all readiness views, not only skills list.
