## Summary

Keep open: the PR is the active and useful fix candidate for status session cost visibility, but it is not merge-ready because slash `/status` can still replace the full transcript-backed cost with a bounded-tail subtotal for long sessions.

## What This Changes

The branch adds transcript-backed session cost display to slash `/status`, a conditional `Cost` column for `openclaw status`, docs, and focused regression tests.

## Best Possible Solution

Make the full transcript-backed cost authoritative for slash status, while keeping the bounded tail reader for token/cache/model fallback and for cost only when no full-session cost is available.

## Reproduction Assessment

Yes from source: `status-text.ts` passes a full transcript cost into `buildStatusMessage`, and `status-message.ts` can then replace it with the bounded-tail reader's cost. A focused test can construct an older cost outside the 256 KB tail plus a newer tail cost to reproduce the underreporting.

## Solution Assessment

No. The direction is the narrow maintainable fix, but the current implementation must make full transcript-backed cost win over bounded-tail fallback before this is the best solution.
