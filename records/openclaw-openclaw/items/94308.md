## Summary

Keep open: the feature is useful and current main still lacks it, but current PR head treats `estimatedCostUsd` as a session total even though current session persistence snapshots it per run, so multi-turn session cost can be underreported.

## What This Changes

The PR adds session cost display to slash `/status` and `openclaw status`, threads a cost field through status summaries, and updates related docs and tests.

## Best Possible Solution

Use a true cumulative session-cost source, or clearly label the display as last-run cost, while keeping slash and CLI status bounded and adding current-head proof for both surfaces.

## Reproduction Assessment

Yes for the PR defect by source inspection: current head maps `estimatedCostUsd` into status output, while current session persistence treats that field as a run snapshot rather than a cumulative session total. I did not run live proof because this review is read-only.

## Solution Assessment

No. The direction is useful, but the current implementation conflates last-run and cumulative session cost; the safer fix is a cumulative source or an explicitly labeled last-run value plus current-head proof.
