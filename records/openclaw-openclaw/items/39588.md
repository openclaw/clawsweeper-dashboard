## Summary

Keep open. Current main still appears to expose the reported Launch at login drift: the UI-bound setting is derived from live LaunchAgent health, and the latest shipped release has not separated durable user intent from launchd registration state.

## What This Changes

The issue asks for the macOS Launch at login setting to persist across app restarts and Sparkle app updates instead of appearing to turn itself off.

## Best Possible Solution

Persist the user's Launch at login intent separately from launchd health, repair or re-register ai.openclaw.mac when that intent is enabled, and preserve the current manual-quit/no-KeepAlive behavior.

## Reproduction Assessment

No, we do not have a high-confidence end-to-end reproduction of the signed Sparkle update and macOS login path. Source inspection does show a current-main drift path where LaunchAgentManager.status() can return false and AppState copies that into the UI-bound setting.

## Solution Assessment

Yes, the narrow maintainable fix is to separate persisted user intent from launchd health and reconcile the existing app LaunchAgent. The related gateway LaunchAgent repair does not by itself solve this app toggle state drift.
