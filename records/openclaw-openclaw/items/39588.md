## Summary

Keep open. Current main and the latest shipped release still derive the macOS Launch at login toggle from live LaunchAgent health rather than durable user intent, so the reported post-update drift remains source-backed and unsolved.

## What This Changes

The issue asks for macOS Launch at login to remain enabled and auto-launch reliably across app restarts and Sparkle app updates.

## Best Possible Solution

Persist the user's Launch at login intent separately from live LaunchAgent health, reconcile or repair ai.openclaw.mac when that intent is enabled, and preserve the current no-KeepAlive manual-quit behavior.

## Reproduction Assessment

No, there is not a high-confidence live reproduction of the full signed Sparkle update and logout/login path. Source inspection still shows a clear drift path where LaunchAgentManager.status() returns false for missing/unhealthy ai.openclaw.mac state and AppState copies that false value into the UI-bound setting.

## Solution Assessment

Yes, the narrow maintainable fix is to separate persisted user intent from launchd health and reconcile the existing app LaunchAgent. The fix should avoid reintroducing KeepAlive for the app LaunchAgent because that would regress manual quit behavior fixed by the adjacent LaunchAgent change.
