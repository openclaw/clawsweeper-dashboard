## Summary

Keep open. Current main and the latest release still derive the visible macOS Launch at login toggle from live launchd status instead of durable user intent, so the reported post-update toggle drift is still plausible from source inspection.

## What This Changes

The issue asks for the macOS app's Launch at login setting to stay enabled across app restarts and updates and continue auto-launching after login.

## Best Possible Solution

Persist the user's Launch at login intent separately from launchd health, reconcile or re-register the current app LaunchAgent when that intent is enabled, and keep the no-`KeepAlive` quit behavior.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established because this review did not run a signed Sparkle update and macOS login cycle. Source inspection does show a current-main path where live launchd health is copied into the UI-bound `launchAtLogin` setting.

## Solution Assessment

Yes. The narrow maintainable fix is to separate persisted user intent from live launchd health and repair or re-register the existing app LaunchAgent when intent is enabled, without reintroducing `KeepAlive`.
