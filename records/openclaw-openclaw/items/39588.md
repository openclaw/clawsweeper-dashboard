## Summary

Keep open. Current main still derives the macOS Launch at login toggle from live launchd status, has no durable user-intent key, and the adjacent KeepAlive/manual-quit fix does not address update-time or launchd-health drift.

## What This Changes

The issue asks the macOS app to keep Launch at login enabled across app restarts and Sparkle updates.

## Best Possible Solution

Persist the user's Launch at login intent separately from launchd health, repair or reconcile the ai.openclaw.mac LaunchAgent when enabled intent is unhealthy, and preserve the existing no-KeepAlive manual-quit behavior.

## Reproduction Assessment

No high-confidence live reproduction of the full signed Sparkle update and logout/login path was performed. Current main still provides a source-reproducible drift path: a missing or unloaded launchd registration makes LaunchAgentManager.status() return false, and AppState copies that false value into the UI toggle.

## Solution Assessment

Yes as a direction. The narrow maintainable fix is to separate persisted user intent from live launchd health and reconcile the existing desktop app LaunchAgent; the merged KeepAlive-only fix does not cover this symptom.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/39588. Current main derives the macOS Launch at login UI state from LaunchAgentManager.status(), which returns false for missing/unloaded launchd state, and AppState writes that false value into the UI-bound launchAtLogin property. Add a durable macOS defaults key for the user's Launch at login intent, read that intent for the UI, update it when the user toggles the setting, and reconcile/repair the ai.openclaw.mac LaunchAgent when intent is enabled but live launchd state is missing or unhealthy. Keep the change scoped to the macOS app LaunchAgent; do not add a new user-facing config option, do not re-add KeepAlive, and do not solve the separate CLI gateway duplicate-service issue tracked by https://github.com/openclaw/openclaw/issues/67631 and https://github.com/openclaw/openclaw/pull/73974. Add focused Swift regression coverage for intent persistence/reconciliation plus preserve the existing RunAtLoad/no-KeepAlive contract. Add a changelog fix entry because this changes user-visible macOS behavior.
