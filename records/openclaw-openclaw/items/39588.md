## Summary

Keep open. Current main still has a source-backed path where the macOS Launch at login setting mirrors live LaunchAgent health instead of durable user intent, and the related gateway LaunchAgent work does not close this app-toggle drift.

## What This Changes

The issue reports that the macOS Launch at login setting stops persisting across app restarts or Sparkle updates and no longer reliably auto-launches after macOS login.

## Best Possible Solution

Persist the user's Launch at login intent separately from live launchd health, reconcile or repair ai.openclaw.mac when that intent is enabled, and keep the current no-KeepAlive manual-quit behavior.

## Reproduction Assessment

No, there is not a high-confidence end-to-end reproduction of the signed Sparkle update and macOS login path. Source inspection does show a current-main drift path where LaunchAgentManager.status() can return false and AppState copies that into the UI-bound setting.

## Solution Assessment

Yes, the narrow maintainable direction is to separate persisted user intent from launchd health and reconcile the existing app LaunchAgent. A broader gateway LaunchAgent repair would not by itself fix the app toggle state drift.
