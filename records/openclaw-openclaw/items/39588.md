## Summary

Keep open. Current main and v2026.5.19 still initialize the UI-bound `launchAtLogin` state from live LaunchAgent status, with no durable user-intent key, so launchd/update drift can still make the setting appear off after the user enabled it.

## What This Changes

The issue asks OpenClaw's macOS app to keep `Launch at login` enabled across restarts and app updates.

## Best Possible Solution

Persist the user's Launch at login intent separately from launchd health, reconcile or re-register `ai.openclaw.mac` when intent is enabled, and keep the desktop app LaunchAgent free of `KeepAlive`.

## Reproduction Assessment

No, not end to end: there is no high-confidence signed Sparkle update plus macOS login reproduction. Source inspection does show a current-main drift path where `LaunchAgentManager.status()` can return false and `AppState` copies that value into the UI-bound setting.

## Solution Assessment

Yes. The narrow maintainable fix is to separate persisted user intent from launchd health, then repair or re-register the existing app LaunchAgent when intent is enabled.
