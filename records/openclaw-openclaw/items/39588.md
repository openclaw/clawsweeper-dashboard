## Summary

Keep open. Current main and the latest shipped tag still derive the visible macOS Launch at login setting from live launchd status rather than durable user intent, and no merged/canonical follow-up owns this exact app-toggle drift.

## What This Changes

The issue reports that the macOS app's Launch at login setting appears to turn off and stops reliably launching after app updates.

## Best Possible Solution

Persist the user's Launch at login intent separately from launchd health, reconcile or re-register the app LaunchAgent when intent is enabled, and preserve the no-`KeepAlive` desktop app behavior.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established: this review did not run a signed Sparkle update and macOS login cycle. Source inspection does show a current-main drift path where launchd health can be copied into the UI-bound setting.

## Solution Assessment

Yes, the narrow maintainable direction is to separate persisted user intent from live launchd health and repair/re-register the existing app LaunchAgent when intent is enabled. Reintroducing `KeepAlive` would conflict with the adjacent merged fix for manual quit behavior.
