## Summary

Keep open. Current main and the latest shipped tag still derive the visible macOS Launch at login setting from live launchd status rather than durable user intent, so the reported app-toggle drift remains plausible and is not owned by the adjacent CLI gateway LaunchAgent reports.

## What This Changes

Investigate the macOS app Launch at login toggle losing persistence or showing disabled after app updates.

## Best Possible Solution

Persist Launch at login user intent separately from live launchd health, then reconcile or re-register `ai.openclaw.mac` when intent is enabled while preserving the no-`KeepAlive` manual quit behavior.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established because this review did not run a signed Sparkle update and login cycle. Source inspection does show a current-main path where transient launchd status can be copied into the UI-bound `launchAtLogin` setting.

## Solution Assessment

Yes, the narrow maintainable fix is to separate persisted user intent from live launchd health and repair or re-register the app LaunchAgent when intent is enabled. Reintroducing `KeepAlive` is not the best fix because it would regress the manual quit behavior fixed by https://github.com/openclaw/openclaw/pull/40213.
