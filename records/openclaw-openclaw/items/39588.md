## Summary

Keep open. Current main and the latest shipped tag still derive the visible macOS Launch at login toggle from live launchd registration state rather than durable user intent, and the related open LaunchAgent ownership work does not own this exact app-toggle drift.

## What This Changes

The issue reports that the macOS app Launch at login setting appears to turn itself off after app updates and does not reliably auto-launch afterward.

## Best Possible Solution

Persist the user's Launch at login intent separately from launchd health, reconcile or re-register the app LaunchAgent when intent is enabled, and preserve the no-`KeepAlive` desktop app behavior.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established because this review did not run a signed Sparkle update and macOS login cycle. Source inspection does show a current-main drift path where launchd health can be copied into the UI-bound setting.

## Solution Assessment

Yes. The narrow maintainable direction is to separate persisted user intent from live launchd health and repair or re-register the existing app LaunchAgent when intent is enabled, without reintroducing `KeepAlive`.
