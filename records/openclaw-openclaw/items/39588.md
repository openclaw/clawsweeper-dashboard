## Summary

Keep open. Current main and v2026.5.18 still derive the macOS Launch at login toggle from live LaunchAgent health, with no durable user-intent key, so the reported update-related state drift remains plausible and unresolved.

## What This Changes

The issue reports that macOS Launch at login can stop persisting across app updates and the settings toggle can later appear disabled.

## Best Possible Solution

Persist the user’s Launch at login intent separately from launchd registration health, reconcile or re-register ai.openclaw.mac when intent is enabled, and preserve the current manual-quit/no-KeepAlive behavior.

## Reproduction Assessment

No, there is not a high-confidence end-to-end reproduction of the signed Sparkle update and macOS login path. Source inspection does show a current-main drift path where LaunchAgentManager.status() can return false and AppState copies that into the UI-bound setting.

## Solution Assessment

Yes, the narrow maintainable fix is to separate persisted user intent from launchd health, then repair or re-register the existing app LaunchAgent when intent is enabled. The related gateway LaunchAgent repair does not solve this app toggle state drift.
