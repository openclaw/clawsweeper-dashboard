## Summary

Keep open. Current main and v2026.5.12 still have the macOS Launch at login UI coupled to live LaunchAgent health rather than durable user intent, and no merged or open closing PR owns this specific app-toggle drift.

## What This Changes

The issue reports that the macOS app Launch at login setting can stop appearing enabled and fail to survive app updates.

## Best Possible Solution

Persist the user's Launch at login intent separately from launchd health, reconcile ai.openclaw.mac when that intent is enabled, and preserve the current no-KeepAlive manual-quit behavior.

## Reproduction Assessment

No, there is not a high-confidence live reproduction of the full signed Sparkle update and logout/login path. Source inspection does show a current-main path where LaunchAgentManager.status() can return false and AppState copies that value into the UI-bound setting.

## Solution Assessment

Yes, the narrow maintainable direction is to separate persisted user intent from launchd health and reconcile the existing app LaunchAgent. A broader Gateway launchd repair would not by itself fix the app toggle state drift.
