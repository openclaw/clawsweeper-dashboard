## Summary

Keep open. Current `main` and latest `v2026.6.1` still derive the visible macOS Launch at login toggle from live LaunchAgent status, and the related shipped fixes only addressed startup deletion and manual-quit relaunch behavior, not Sparkle update reconciliation or durable user intent.

## What This Changes

The issue reports that the macOS app’s Launch at login preference appears to stop persisting after Sparkle app updates, causing unreliable auto-launch on login.

## Best Possible Solution

Persist the user’s desired launch-at-login setting independently from live launchd status, reconcile the LaunchAgent after Sparkle replacement when desired, and cover stale or missing launchd state in Swift tests plus signed macOS update/login proof.

## Reproduction Assessment

No high-confidence live reproduction path is available from this review. Source inspection does show a plausible current-main path: startup copies live LaunchAgent status into the visible toggle instead of preserving a durable desired launch-at-login state.

## Solution Assessment

No, the current implementation is not the best end state for this report. A maintainable fix should separate user intent from launchd health and repair/re-register the LaunchAgent after updates without turning the preference off.
