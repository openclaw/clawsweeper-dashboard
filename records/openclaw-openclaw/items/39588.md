## Summary

Keep open: current main and the latest release still derive the macOS Launch at login toggle from live launchd registration, and I did not find durable desired-state persistence or Sparkle post-update reconciliation that would make the report obsolete.

## What This Changes

The issue reports that the macOS app Launch at login setting can stop persisting across app updates.

## Best Possible Solution

Persist the user’s desired Launch at login setting separately from transient launchd status, then reconcile or repair the app LaunchAgent after app updates without misleading the UI.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection does show a plausible current-main path where the UI is derived from live launchd status rather than a durable desired preference, matching the reported drift shape.

## Solution Assessment

Unclear until maintainers choose the product contract for this toggle. The narrow maintainable direction is to persist desired state and reconcile launchd after updates, rather than treating one transient `launchctl print` result as the user preference.
