## Summary

Keep open. Current main still has a source-backed drift path where the macOS Launch at login UI-bound state is derived from live LaunchAgent health, and the latest release still contains the same behavior; the related gateway LaunchAgent reports are adjacent but not a canonical fix for this app toggle persistence bug.

## What This Changes

The issue asks for macOS Launch at login to stay enabled across app restarts, app updates, and the next macOS login once the user enables it.

## Best Possible Solution

Persist the user's Launch at login intent separately from launchd registration health, reconcile or re-register `ai.openclaw.mac` when intent is enabled, and preserve the current manual-quit/no-KeepAlive behavior.

## Reproduction Assessment

No, not as an end-to-end signed Sparkle update or logout/login flow. Source inspection does show a current-main path where `LaunchAgentManager.status()` can return false and `AppState` copies that health result into the UI-bound setting.

## Solution Assessment

Yes, the narrow maintainable fix is to separate persisted user intent from launchd health and repair or re-register the existing app LaunchAgent when intent is enabled. The related gateway LaunchAgent repair work does not solve this app toggle state drift.
