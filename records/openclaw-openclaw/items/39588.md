## Summary

Keep open. Current main still derives the macOS Launch at login toggle from live launchd state, has no durable user-intent key, and the merged KeepAlive fix is adjacent but does not address update-time or launchd-health drift.

## What This Changes

The issue asks the macOS app to keep the existing Launch at login setting and auto-launch behavior stable across app restarts and Sparkle updates.

## Best Possible Solution

Persist the user's Launch at login intent separately from launchd health, show or repair launchd state without flipping intent off, and reconcile `ai.openclaw.mac` after startup or update when intent is enabled.

## Reproduction Assessment

No for the full signed Sparkle update and logout/login path in this read-only review. Current source does provide a source-reproducible drift path: missing or unloaded launchd state makes `LaunchAgentManager.status()` return false, and `AppState` copies that false value into the UI toggle.

## Solution Assessment

Yes as a direction. The narrow maintainable fix is to separate persisted user intent from live launchd health and reconcile the existing desktop app LaunchAgent; the merged KeepAlive-only fix does not cover this symptom.
