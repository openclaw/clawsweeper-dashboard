## Summary

Keep open. Current main still lets the macOS Launch at login UI mirror live launchd health instead of durable user intent, and the adjacent merged KeepAlive fix does not address update-time or unloaded-LaunchAgent drift.

## What This Changes

The issue reports that the macOS app Launch at login toggle stops persisting or appears disabled after app updates.

## Best Possible Solution

Persist the user's Launch at login intent separately from launchd health, then repair or reconcile the ai.openclaw.mac LaunchAgent when intent is enabled.

## Reproduction Assessment

No high-confidence live reproduction of the full signed Sparkle update and logout/login path was performed. Source inspection still shows a reproducible drift path: missing or unloaded launchd state makes status false, and AppState copies that false value into the UI-bound toggle.

## Solution Assessment

Yes as a direction. The narrow maintainable fix is to separate persisted user intent from live launchd health and reconcile the existing desktop app LaunchAgent, while leaving the gateway LaunchAgent update path alone.
