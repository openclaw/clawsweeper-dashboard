## Summary

Keep open. Current main still has a source-reproducible Launch at login drift path because the macOS settings UI is driven by live LaunchAgent health rather than durable user intent, and the adjacent shipped manual-quit fix does not address update-time reconciliation or toggle persistence.

## What This Changes

The issue asks OpenClaw to keep the macOS app `Launch at login` setting enabled across app restarts, Sparkle updates, and subsequent macOS logins.

## Best Possible Solution

Persist the user's Launch at login intent separately from LaunchAgent health, reconcile or repair `ai.openclaw.mac` when that intent is enabled, and preserve the current no-`KeepAlive` manual-quit behavior.

## Reproduction Assessment

No high-confidence live reproduction of the full signed Sparkle update and logout/login path was performed. The source still shows a reproducible drift path: a missing or unhealthy `ai.openclaw.mac` LaunchAgent makes `LaunchAgentManager.status()` return false, and `AppState` copies that false value into the UI-bound setting.

## Solution Assessment

Yes, the suggested direction is the best maintainable fix. The narrow repair is to separate persisted user intent from launchd health and reconcile the existing desktop app LaunchAgent; the shipped `KeepAlive` fix does not cover this symptom.
