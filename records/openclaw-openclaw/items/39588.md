## Summary

Keep open. Current main and the latest shipped release still bind the macOS Launch at login toggle to live LaunchAgent health rather than durable user intent, so stale launchd/update state can still make the setting appear off after the user enabled autostart.

## What This Changes

The issue asks for the macOS app Launch at login setting to remain enabled and continue autostarting across app restarts and updates.

## Best Possible Solution

Persist the user's Launch at login intent separately from launchd health, reconcile or re-register ai.openclaw.mac when intent is enabled, and preserve the current no-KeepAlive manual-quit behavior.

## Reproduction Assessment

No, not end to end: there is no high-confidence signed Sparkle update plus macOS login reproduction. Source inspection does show a current-main drift path where LaunchAgentManager.status() can return false and AppState copies that value into the UI-bound Launch at login setting.

## Solution Assessment

Yes, the narrow maintainable fix is to separate persisted user intent from launchd health, then repair or re-register the existing app LaunchAgent when intent is enabled. The related gateway LaunchAgent repair work does not solve this app toggle state drift.
