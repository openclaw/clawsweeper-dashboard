## Summary

Keep open. Current main and the latest shipped tag still drive the macOS Launch at login toggle from live LaunchAgent health instead of durable user intent, so the reported post-update drift remains source-backed and unsolved.

## What This Changes

The issue asks for the macOS app Launch at login setting to persist and keep auto-launch working across app restarts and Sparkle updates.

## Best Possible Solution

Persist the user's Launch at login intent separately from live LaunchAgent health, reconcile or repair ai.openclaw.mac when that intent is enabled, and preserve the current no-KeepAlive manual-quit behavior.

## Reproduction Assessment

No high-confidence live reproduction of the full signed Sparkle update and logout/login path was performed. Source inspection still shows a clear drift path: missing or unhealthy ai.openclaw.mac LaunchAgent state makes LaunchAgentManager.status() return false, and AppState copies that false value into the UI-bound setting.

## Solution Assessment

Yes, the narrow maintainable fix is to separate persisted user intent from launchd health and reconcile the existing app LaunchAgent. The fix should avoid reintroducing KeepAlive for the app LaunchAgent because that would regress manual quit behavior fixed by the adjacent LaunchAgent change.
