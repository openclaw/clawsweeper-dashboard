## Summary

Keep open. Current main still lets the macOS Launch at login toggle mirror live LaunchAgent health instead of durable user intent, and the latest shipped release contains the same drift path.

## What This Changes

The issue reports that the macOS app Launch at login preference can stop sticking or stop launching reliably after app updates.

## Best Possible Solution

Persist the user’s Launch at login intent separately from launchd health, reconcile or re-register ai.openclaw.mac when intent is enabled, and preserve the current manual-quit/no-KeepAlive behavior.

## Reproduction Assessment

No, we do not have a high-confidence end-to-end reproduction of the signed Sparkle update and macOS login path. Source inspection does show a current-main drift path where LaunchAgentManager.status() can return false and AppState copies that into the UI-bound setting.

## Solution Assessment

Yes, the narrow maintainable direction is to separate persisted user intent from launchd registration health and then repair/re-register the existing app LaunchAgent when needed. The related gateway LaunchAgent repair does not solve the app toggle state drift.
