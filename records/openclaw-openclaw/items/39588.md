## Summary

Keep open. Current main still derives the macOS Launch at login toggle from live launchd status, has no durable user-intent key, and the adjacent merged KeepAlive fix does not address update-time or launchd-health drift.

## What This Changes

The issue reports that the macOS app Launch at login setting can stop appearing enabled and fail to auto-launch after app updates.

## Best Possible Solution

Persist the user's Launch at login intent separately from launchd health, repair or reconcile the `ai.openclaw.mac` LaunchAgent when intent is enabled, and preserve the existing no-`KeepAlive` manual-quit behavior.

## Reproduction Assessment

No high-confidence live reproduction of the full signed Sparkle update and logout/login path was performed. The current source still provides a source-reproducible drift path: a missing or unloaded launchd registration makes `LaunchAgentManager.status()` return false, and `AppState` copies that false value into the UI toggle.

## Solution Assessment

Yes as a direction. The narrow maintainable fix is to separate persisted user intent from live launchd health and reconcile the existing desktop app LaunchAgent; the merged KeepAlive-only fix does not cover this symptom.
