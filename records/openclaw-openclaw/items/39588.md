## Summary

Keep open. Current main and the latest release still copy live LaunchAgent status into the visible macOS Launch at login setting, so the reported update/login drift remains plausible and not fully solved.

## What This Changes

The issue reports that the macOS app Launch at login setting can become unchecked and stop auto-launching after app updates.

## Best Possible Solution

Persist the user's Launch at login intent separately from LaunchAgent health, reconcile or re-register the current app LaunchAgent when that intent is enabled, and preserve the existing no-KeepAlive quit behavior.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established because this review did not run a signed Sparkle update and macOS login cycle. Source inspection does show a current-main path where live LaunchAgent health is copied into the UI-bound Launch at login setting.

## Solution Assessment

Yes. The narrow maintainable fix is to separate persisted user intent from live LaunchAgent health and repair or re-register the app LaunchAgent when intent is enabled, without reintroducing KeepAlive.
