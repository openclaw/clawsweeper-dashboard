## Summary

Keep open. Current main and latest v2026.5.19 still derive the visible Launch at login setting from live launchd health rather than durable user intent, and duplicate search did not find a merged or canonical follow-up that owns this exact app-toggle drift.

## What This Changes

The issue reports that the macOS app Launch at login setting appears to stop sticking after app updates and the app may not auto-launch at the next login.

## Best Possible Solution

Persist the user's Launch at login intent separately from launchd health, reconcile or re-register `ai.openclaw.mac` when intent is enabled, and keep the desktop app LaunchAgent free of `KeepAlive`.

## Reproduction Assessment

No, not end to end: this review did not run a signed Sparkle update and macOS login cycle. Source inspection does show a current-main drift path where `LaunchAgentManager.status()` can return false and `AppState` copies that health value into the UI-bound setting.

## Solution Assessment

Yes, the narrow maintainable direction is to separate persisted user intent from launchd health, then repair or re-register the existing app LaunchAgent when intent is enabled. Reintroducing `KeepAlive` would be the wrong fix because the adjacent merged regression test intentionally removed it.
