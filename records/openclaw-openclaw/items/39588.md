## Summary

Keep open. Current main still derives the visible macOS Launch at login toggle from live LaunchAgent status instead of a separate durable user intent, so the reported post-update toggle drift remains plausible and is not fixed in the latest release.

## What This Changes

The issue reports that the macOS app's Launch at login setting appears to turn itself off or stop working after app updates.

## Best Possible Solution

Persist the user's Launch at login intent separately from LaunchAgent health, reconcile or re-register the current app LaunchAgent when that intent is enabled, and preserve the existing no-KeepAlive quit behavior.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established because this review did not run a signed Sparkle update and macOS login cycle. Source inspection does show a current-main path where live LaunchAgent health is copied into the UI-bound Launch at login setting.

## Solution Assessment

Yes. The narrow maintainable fix is to separate persisted user intent from live LaunchAgent health and repair or re-register the existing app LaunchAgent when intent is enabled, without reintroducing KeepAlive.
