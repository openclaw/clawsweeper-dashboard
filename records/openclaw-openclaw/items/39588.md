## Summary

Keep open. Current main still derives the macOS Launch at login setting from live launchd state and has no durable user-intent key or Sparkle/update reconciliation, so the reported setting drift remains actionable; the merged KeepAlive fix is adjacent but narrower.

## What This Changes

The issue requests that the macOS app keep Launch at login enabled across app restarts and app updates after the user turns it on.

## Best Possible Solution

Persist the user's Launch at login intent in the macOS app, report launchd health separately, and reconcile or re-bootstrap `ai.openclaw.mac` on startup and update completion when intent is enabled.

## Reproduction Assessment

No for the full end-to-end signed Sparkle update path. Current source does give a source-reproducible drift path: missing or unloaded launchd state makes `LaunchAgentManager.status()` return false, and `AppState` copies that false value into the UI setting.

## Solution Assessment

Yes. The narrow maintainable fix is to separate persisted user intent from live launchd health in the macOS app and reconcile the desktop app LaunchAgent after startup or update; the #40213 KeepAlive-only fix does not cover this symptom.
