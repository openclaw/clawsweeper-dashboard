## Summary

Keep open. Current main still binds the macOS Launch at login toggle to live LaunchAgent health instead of durable user intent, and the latest shipped release still has the same drift path; the linked closed gateway LaunchAgent issue is adjacent, not a canonical fix for this app toggle behavior.

## What This Changes

The issue reports that the macOS app Launch at login setting stops staying enabled across app updates and later fails to auto-launch.

## Best Possible Solution

Persist the user’s Launch at login intent separately from launchd registration health, reconcile or re-register ai.openclaw.mac when intent is enabled, and preserve the current manual-quit/no-KeepAlive behavior.

## Reproduction Assessment

No, there is not a high-confidence end-to-end reproduction of the signed Sparkle update and macOS login path. Source inspection does show a current-main drift path where LaunchAgentManager.status() can return false and AppState copies that into the UI-bound setting.

## Solution Assessment
