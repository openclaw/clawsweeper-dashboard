## Summary

Keep open. Current main still has a source-backed path where the macOS Launch at login UI reflects live LaunchAgent health rather than durable user intent, and the latest shipped release still contains that behavior; related LaunchAgent reports are adjacent but do not close this app-toggle drift.

## What This Changes

The issue asks for the macOS app Launch at login setting to remain enabled across restarts and app updates and to keep auto-launch working after login.

## Best Possible Solution

Persist the user's Launch at login intent separately from launchd registration health, reconcile or re-register ai.openclaw.mac when intent is enabled, and preserve the current no-KeepAlive manual-quit behavior.

## Reproduction Assessment

No, not as an end-to-end signed Sparkle update and macOS login reproduction. Source inspection does show a current-main path where LaunchAgentManager.status() can return false and AppState copies that health result into the UI-bound setting.

## Solution Assessment

Yes. The narrow maintainable fix is to separate durable user intent from launchd health, then repair or re-register the existing app LaunchAgent when intent is enabled rather than treating a transient status failure as the user's preference.
