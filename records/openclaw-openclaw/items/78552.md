## Summary

Keep this PR open for maintainer review: current main still lacks the proposed configurable memory-lancedb capture/category API, and the PR includes credible live proof, but it adds a new regex-based config surface that overlaps with an open narrower trigger proposal and has a short-CJK capture gap before it can safely close the linked i18n issue.

## What This Changes

The PR adds optional memory-lancedb `triggers` regexes and `categoryRules` regex mappings, wires them into auto-capture/category detection, and updates the manifest, docs, changelog, and parser tests.

## Best Possible Solution

Adopt one memory-lancedb-owned multilingual capture contract, either repairing this regex/category API with short-CJK behavior and tighter schema alignment or choosing the safer literal-trigger direction from the related PR.

## Reproduction Assessment

Not applicable as a bug reproduction: this is an explicit feature/config addition. Source inspection confirms current main has fixed built-in capture/category rules and no accepted `triggers` or `categoryRules` config keys.

## Solution Assessment

Unclear pending maintainer product approval. The PR stays inside the memory-lancedb plugin boundary and has useful proof, but the regex/category API should be reconciled with https://github.com/openclaw/openclaw/pull/76121 and repaired for short CJK trigger behavior before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Honor short CJK custom-trigger matches:** `extensions/memory-lancedb/index.ts:555`
  - body: The new `extraTriggers` branch is only reached after `shouldCapture` has already returned false for `text.length < 10`, so configured patterns like `记住` or `重要` still cannot capture the short CJK phrases called out by the linked i18n issue, despite the new docs saying a message is captured when it matches any trigger. Move the length gate after trigger matching or add a CJK/custom-trigger exception before using this PR to close that issue.
  - confidence: 0.86
- **[P3] Align categoryRules manifest validation with runtime parsing:** `extensions/memory-lancedb/openclaw.plugin.json:145-149`
  - body: The manifest schema accepts any `categoryRules` property name, but the runtime parser rejects keys outside the five memory categories. That lets config/manifest validation accept values that later disable the plugin at parse time, so the manifest should enumerate the valid category properties or otherwise reject unknown keys.
  - confidence: 0.78
