## Summary

## What This Changes

## Best Possible Solution

Rebase and redesign the privacy path to redact all current user-context metadata, keep trusted system metadata cache-stable, align the full config contract, and settle keyed pseudonym semantics before merge.

Yes, source-reproducible: current main's `buildInboundUserContextPrefix` still serializes raw identifying inbound metadata into prompt-visible user context. I did not run a live configured OpenClaw prompt in this read-only review.

## Solution Assessment

No, not as-is. The direction is useful, but the patch needs a current-main rebase, full config contract follow-through, broader metadata coverage, real behavior proof, and a maintainer/security decision on keyed versus unkeyed pseudonyms.

  - body: This branch strips older group labels, but current main also emits `group_members`, `topic_name`, chat-window sender labels, and reply-chain sender fields in prompt-visible user context.
