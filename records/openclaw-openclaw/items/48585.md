## What This Changes

The PR adds a `privacy.redactPII` config option and threads it into inbound prompt metadata builders to hash, strip, or pseudonymize selected OpenClaw-injected metadata, with focused tests.

## Best Possible Solution

Land a rebased privacy implementation that updates the typed/generated/documented config contract, covers every current LLM-visible system-injected metadata field, and either uses keyed pseudonyms or explicitly narrows the documented privacy guarantee.

## Solution Assessment

No, not as-is. The direction is useful, but the patch needs a current-main rebase, complete config-contract updates, full coverage of newer metadata fields, and a maintainer/security decision on keyed versus unkeyed pseudonyms.

## Security Review

Status: needs_attention

Summary: The diff has no CI, dependency, or package supply-chain changes, but the privacy-sensitive redaction feature uses reversible unkeyed hashes for low-entropy identifiers.

Concerns:

- **[medium] Reversible low-entropy pseudonyms:** `src/auto-reply/reply/inbound-meta.ts:7`
