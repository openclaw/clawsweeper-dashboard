## Repair Work Prompt

Repair the daily session reset scheduler from https://github.com/openclaw/openclaw/pull/84009. The scheduler scans concrete targets from resolveAllAgentSessionStoreTargetsSync, but stale rows currently call performGatewaySessionReset with only the session key; change the reset path so scheduled resets mutate the exact scanned target.storePath, target.agentId, and matched key. Either extend performGatewaySessionReset with an explicit resolved target/store option or add a narrow helper that preserves reset hooks/archive/cleanup while mutating the supplied store. Add a regression where a stale discovered/custom store and a fresh configured store share the same session key; assert the stale scanned row is the one reset and the fresh row is not rotated. Keep the proactive daily rollover behavior and provider-owned CLI exemption; do not add a new config option unless maintainers request it.
