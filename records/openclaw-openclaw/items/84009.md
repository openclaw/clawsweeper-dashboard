## Summary

Keep open: the PR addresses the source-backed daily reset gap, but the scheduler currently treats idle expiry under a daily policy as a background reset trigger too, which is broader than the reported 4am wall-clock bug and current docs.

## What This Changes

The PR adds a gateway maintenance scheduler for daily session resets, wires it into gateway timer lifecycle, extends reset reason plumbing, and updates session docs/tests.

## Best Possible Solution

Keep the scheduler approach, but gate the background sweep on daily-boundary staleness only and add regression coverage for daily mode with idleMinutes before merge.

## Reproduction Assessment

Yes. Source inspection shows current main only evaluates daily freshness during session use, and the PR side effect is also source-reproducible because evaluateSessionFreshness returns false for idle expiry as well as daily expiry.

## Solution Assessment

No, not as written. The gateway-maintenance scheduler is the right shape, but it should distinguish daily-boundary staleness from idle expiry before calling the reset service.
