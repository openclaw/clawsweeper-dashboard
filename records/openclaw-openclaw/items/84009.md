## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Pass agentId into scheduled daily resets:** `src/gateway/session-daily-reset-scheduler.ts:50-55`
  - body: For rows that resolve to the canonical `global` key, the scheduler keeps `sessionTarget.agentId` only for the later notification but calls `performGatewaySessionReset` without it. The reset service therefore resolves bare `global` through the default-agent path, so non-default global stores can stay stale or the wrong global row can be reset. Thread the selected agent id through the default reset call and cover a non-default/global store case.
  - confidence: 0.9
- **[P1] Send scheduled session changes only to subscribers:** `src/gateway/server-maintenance.ts:83-84`
  - body: The new scheduled reset emitter still calls the broad `broadcast("sessions.changed", ...)` path, which reaches every read-scoped WebSocket client, while existing session mutations first get the session-event subscriber ids and broadcast only to that set. Adding `nodeSendToAllSubscribed` does not remove the broad delivery, so scheduled resets can notify clients that never subscribed to session events. Reuse or share the subscriber-scoped emitter instead.
  - confidence: 0.88
