## Best Possible Solution

Keep the proactive gateway-maintenance reset path, but make active-run protection compare canonical plus raw alias keys and add regression coverage for `main` before maintainers accept the timing change.

Yes. The original bug is source-reproducible because current main only evaluates daily freshness when a requested session is loaded, and the new review finding is source-reproducible from raw `chat.send` active keys versus canonical scheduler store keys.
