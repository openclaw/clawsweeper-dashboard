## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve agent id when resetting global session rows:** `src/gateway/session-daily-reset-scheduler.ts:190-193`
  - body: The scheduler scans every agent store, and `resolveGatewaySessionStoreTarget` preserves an explicit agent id for `global` keys. Here the selected row is reset with only `resetSessionKey`, so a stale `global` row in a non-default agent store is handed to `performGatewaySessionReset` without `agentId`; that resolver falls back to the default agent's global store, leaving the selected row stale or potentially targeting the wrong row. Pass `sessionTarget.agentId` through the scheduler/default reset path and cover a non-default `global` store.
  - confidence: 0.88
- **[P2] Send scheduled session changes only to session subscribers:** `src/gateway/server-maintenance.ts:83-84`
  - body: Current `sessions.changed` emitters first read `getSessionEventSubscriberConnIds()` and then call `broadcastToConnIds`, so clients explicitly opt into this session-row feed. This new maintenance path calls the broad gateway `broadcast`, which sends session ids/model/status metadata to every read-scoped WebSocket client even when it never subscribed. Route scheduled reset notifications through the same subscriber-scoped helper or pass the subscriber set into maintenance.
  - confidence: 0.9
