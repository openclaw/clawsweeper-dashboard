Full review comments:

- **[P1] Pass agentId into scheduled resets:** `src/gateway/session-daily-reset-scheduler.ts:50-55`
  - body: When the scheduler scans a non-default agent store whose canonical session key is `global`, `sessionTarget.agentId` identifies the selected store, but the default reset callback only passes `key`. `performGatewaySessionReset` only selects that non-default global store when `agentId` is supplied, so this can skip the stale selected store or target the default agent instead. Thread `agentId` through `performReset` and cover a non-default global-store daily reset.
  - confidence: 0.92
- **[P1] Target scheduled session changes at subscribers:** `src/gateway/server-maintenance.ts:83-84`
  - body: `broadcastScheduledDailyReset` sends the full refreshed `sessions.changed` row through `params.broadcast`, which reaches every read-scoped WS client. Existing session mutations use `getSessionEventSubscriberConnIds()` and `broadcastToConnIds()` so only clients that subscribed to session events receive this stream; this path bypasses that boundary and can deliver unsolicited or duplicate session updates. Pass the subscriber registry into maintenance and reuse the scoped emission shape.
  - confidence: 0.9
