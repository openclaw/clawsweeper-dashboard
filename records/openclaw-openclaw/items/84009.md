## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve the implicit CLI-session reset exemption:** `src/gateway/session-daily-reset-scheduler.ts:50-57`
  - body: The scheduler applies the default daily policy to every stale row and then calls `performGatewaySessionReset`. Current behavior and docs explicitly keep provider-owned CLI sessions through the implicit daily default unless `session.reset` is configured, so this would unexpectedly drop provider CLI continuity on upgrade. Mirror the existing `hasProviderOwnedSession(entry) && resetPolicy.configured !== true` guard before treating the row as stale.
  - confidence: 0.9
- **[P1] Avoid resetting sessions with active runs:** `src/gateway/session-daily-reset-scheduler.ts:66-68`
  - body: This maintenance loop can rotate a stale session at the reset boundary even if that session currently has an in-flight agent/chat run. Existing lazy reset happens before a new request starts; resetting from the timer while a run is still writing can archive the old transcript and replace the session row underneath the active run. Pass enough active-run state into the scheduler, or otherwise skip active sessions until they are idle.
  - confidence: 0.78
