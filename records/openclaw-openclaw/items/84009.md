Full review comments:

- **[P1] Reset the store row selected by the scheduler:** `src/gateway/session-daily-reset-scheduler.ts:84`
  - body: The scheduler scans a concrete `target.storePath`, but then calls `performReset(sessionKey)`, and the reset service re-runs store resolution from only that key. If configured/discovered stores contain the same canonical session key, the sweep can find a stale row in one store and reset a different fresher row, leaving the stale session untouched or corrupting the wrong session state. Carry the scanned target and matched key into the mutation or add a target-bound reset helper.
