## Repair Work Prompt

Fix the scheduled daily reset race in https://github.com/openclaw/openclaw/pull/84009. The observable bug is that `performGatewaySessionReset({ reason: "daily", expectedDailySession })` can run command/internal hooks, plugin cleanup, ACP cleanup, lifecycle hooks, or transcript archival after the selected stale row has been replaced, because the final expected-entry check happens inside `updateSessionStore` after those side effects. Keep the gateway-owned proactive daily scheduler, sessionStartedAt daily-boundary contract, hot config reload behavior, active-run guards, alias/store guards, and implicit provider-owned CLI exemption. Add focused regression coverage that refreshes the row during an awaited pre-mutation path and asserts the daily reset is skipped without cleanup side effects. Do not add config knobs or change manual `/new`/`/reset` semantics. No contributor changelog request is needed.
