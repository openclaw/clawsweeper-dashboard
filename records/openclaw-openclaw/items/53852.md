## What This Changes

The issue reports duplicated streamed `partialArgs` JSON for isolated cron `agentTurn` jobs, causing tool-call argument parsing to fail.

## Best Possible Solution

Keep this issue as the canonical bug and land or replace the related PR with a constrained repair that accepts exact or semantically identical duplicate argument objects while keeping conflicting concatenations fail-closed.

## Reproduction Assessment

Yes, at source level. Feeding two complete `delta.tool_calls[].function.arguments` chunks for one tool call through the current OpenAI-compatible stream path still appends them into `partialArgs` as concatenated JSON; I did not run a live cron job.

## Solution Assessment

Yes on the fix boundary, but not by closing this issue yet. The maintainable repair belongs in stream assembly or the malformed-argument repair helper and must preserve fail-closed behavior for conflicting concatenated payloads.
