Keep open: current main and the latest release still preserve the source-level duplicate `partialArgs` failure path, while the related open PR is only a proposed fix and has unresolved proof and safety concerns.

## What This Changes

The issue reports isolated `agentTurn` cron jobs producing malformed concatenated JSON in streamed tool-call `partialArgs`, causing scheduled tool execution to fail.

## Best Possible Solution

Keep this issue as the canonical bug and land or replace the related PR with a constrained repair that accepts exact or semantically identical duplicate argument objects while keeping conflicting concatenations fail-closed.

## Reproduction Assessment

Yes, at source level. Feeding two complete argument chunks for one OpenAI-compatible tool call through the current stream path still accumulates malformed concatenated JSON before final parsing; I did not run a live cron job.

## Solution Assessment

Yes on the issue boundary, but not by closing this issue yet. The maintainable repair belongs in stream assembly or the malformed-argument helper and must preserve fail-closed behavior for conflicting concatenated payloads.
