## Summary

Keep open. The prior ClawSweeper read still matches current main: isolated cron agent turns still enter the embedded OpenAI-compatible stream path, that path still appends `function.arguments` chunks into `partialArgs`, and completed tool-call blocks still retain the scratch `partialArgs` field. No open PR currently references this issue with closing syntax.

## What This Changes

The issue asks OpenClaw to stop producing duplicated, malformed tool-call `partialArgs` JSON for isolated cron `agentTurn` jobs.

## Best Possible Solution

Normalize repeated or cumulative complete `function.arguments` payloads at the OpenAI-compatible stream boundary, strip internal `partialArgs` from completed tool-call blocks, and add focused regressions for duplicated complete JSON chunks in the isolated cron/OpenRouter-Kimi-shaped path.
