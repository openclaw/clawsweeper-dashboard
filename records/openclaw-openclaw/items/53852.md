## Summary

Keep open. Current main still has the reported source-level failure mode: isolated agentTurn cron jobs enter the embedded OpenAI-compatible stream path, duplicate complete tool-call argument chunks are appended into `partialArgs`, completed tool-call blocks retain that scratch field, and the existing repair wrapper rejects a repeated complete JSON object as unsafe trailing content.

## What This Changes

Fix malformed duplicated tool-call `partialArgs` JSON for isolated `agentTurn` cron jobs.

## Best Possible Solution

Normalize repeated or cumulative complete `function.arguments` payloads at the OpenAI-compatible stream boundary, strip internal `partialArgs` from completed tool-call blocks, and add focused regressions for duplicated complete JSON chunks in the isolated cron/OpenRouter-Kimi-shaped path.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to feed `processOpenAICompletionsStream` two identical complete `tool_calls[].function.arguments` chunks for the same call; current main appends them into one invalid JSON string and the existing repair path rejects that duplicate object suffix.

## Solution Assessment

Yes. The narrow maintainable fix is at the OpenAI-compatible stream or exact duplicate-argument repair boundary; changing cron session routing or broadly accepting long trailing suffixes would be a less safe solution.
