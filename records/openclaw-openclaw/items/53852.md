Keep open. Current main and the latest release still have the source-level path that can concatenate repeated OpenAI-compatible tool-call argument chunks into `partialArgs`, and the related open repair PRs have not merged.

## What This Changes

The issue asks OpenClaw to stop isolated cron `agentTurn` jobs from producing duplicated malformed JSON in tool-call `partialArgs`.

## Best Possible Solution

Add focused duplicate or cumulative tool-call argument normalization at the OpenAI-compatible stream or repair boundary, preserve fail-closed behavior for conflicting payloads, and cover isolated cron/OpenRouter-shaped streams with regression tests.

## Reproduction Assessment

Yes, at source level. Feeding the current OpenAI-compatible stream path two identical complete `delta.tool_calls[].function.arguments` chunks for one active tool call follows the append path and leaves `{...}{...}` in `partialArgs`; I did not run a live cron job.

## Solution Assessment

Yes, the narrow maintainable fix belongs at the OpenAI-compatible stream or malformed-argument repair boundary. It should only repair proven duplicate/cumulative argument shapes and keep conflicting concatenated payloads fail-closed.
