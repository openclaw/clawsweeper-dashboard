## Summary

Keep open. Current main still has the source-level duplicate-argument path in OpenAI-compatible streamed tool calls, and the related malformed-Kimi PR is closed/superseded rather than a fix for this issue.

## What This Changes

The issue reports duplicated tool-call `partialArgs` JSON when isolated cron `agentTurn` jobs stream OpenAI-compatible tool arguments.

## Best Possible Solution

Normalize exact duplicate or cumulative complete `function.arguments` payloads at the OpenAI-compatible stream/repair boundary, strip internal `partialArgs` from completed tool-call blocks if it is not part of the public contract, and add focused regressions for the isolated cron/OpenRouter-shaped path.

## Reproduction Assessment

Yes, at source level. Feeding `processOpenAICompletionsStream` two identical complete `delta.tool_calls[].function.arguments` chunks for one tool-call id follows the current append path and produces `{...}{...}` in `partialArgs`; I did not run a live cron job in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix belongs at the OpenAI-compatible stream or malformed-argument repair boundary; changing cron session routing would not address the duplicated argument assembly and broad trailing-junk acceptance would weaken safety.
