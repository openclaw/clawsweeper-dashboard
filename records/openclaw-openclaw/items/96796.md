The issue reports that QMD-backed memory_search conflates real zero-hit searches with guard/setup outcomes and can trigger a blocking forced sync.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/96799 or an equivalent narrow fix that distinguishes QMD non-result outcomes and gates forced sync to real dirty zero-hit searches.

## Reproduction Assessment

Yes from source inspection. Current main still force-syncs after any empty memory_search result, while QMD search returns [] for scope-denied, empty-query, and no-collection paths.
