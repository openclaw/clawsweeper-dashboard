Keep open: current main and v2026.6.10 still have the reported source shape, and https://github.com/openclaw/openclaw/pull/96799 is an open closing PR that should be reviewed rather than duplicating the fix.

## What This Changes

Prevent QMD memory_search from triggering forced syncs after scope-denied, empty-query, no-collection, or clean-index zero-hit outcomes by distinguishing search outcomes.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/96799 or an equivalent narrow fix that preserves search() compatibility while using outcome-aware internals to sync only for ok, dirty zero-hit searches.

## Reproduction Assessment

Yes from source inspection. Current main still force-syncs after any empty memory_search result while QMD returns [] for scope-denied, empty-query, and no-collection paths; no live QMD run was performed in this read-only review.

## Solution Assessment

Yes in direction: discriminated internal outcomes are the narrow maintainable fix because they preserve search() compatibility while letting memory_search avoid sync for guard and clean-index outcomes. The linked PR still needs normal compatibility and behavior review before merge.
