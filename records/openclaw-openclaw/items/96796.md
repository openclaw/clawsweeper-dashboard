## What This Changes

The issue asks OpenClaw to distinguish QMD memory_search non-result outcomes so scope-denied, empty-query, no-collection, and clean-index zero-hit searches do not all trigger blocking forced sync.

## Best Possible Solution

Land a reviewed fix through https://github.com/openclaw/openclaw/pull/96799 or an equivalent narrow PR, then close this issue once main no longer force-syncs for QMD guard/non-result outcomes.

## Reproduction Assessment

Yes from source inspection, though not from a live run in this review. Current main shows QMD search returning [] for scope-denied, empty-query, and no-collection paths, and memory_search force-syncing after any empty result.

## Solution Assessment

Yes in direction: a discriminated internal outcome is the narrow maintainable fix because it preserves search() compatibility while letting memory_search avoid sync for guard and clean-index outcomes. The linked PR still needs normal compatibility and behavior review before merge.
