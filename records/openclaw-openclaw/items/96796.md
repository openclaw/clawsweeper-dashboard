## What This Changes

The issue reports that QMD-backed memory_search can force a long blocking sync after ambiguous empty QMD search results such as scope-denied or empty-query outcomes.

## Best Possible Solution

Land a reviewed fix through https://github.com/openclaw/openclaw/pull/96799 or an equivalent focused PR, then close this issue once main can distinguish QMD non-result outcomes from true zero-hit searches.

## Reproduction Assessment

Yes from source inspection and reporter runtime logs. Current main force-syncs after any empty memory_search result while QMD returns [] for scope-denied, empty-query, and no-collection outcomes.

## Solution Assessment

Yes in direction: discriminated internal QMD search outcomes are a narrow maintainable fix because they preserve search() compatibility while avoiding sync on guard/setup outcomes; the linked PR still needs normal maintainer review before merge.
