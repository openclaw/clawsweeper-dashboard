## Summary

## What This Changes

The issue asks `memory_search` to distinguish QMD empty-result outcomes so the forced sync retry only runs for real zero-hit, dirty-index searches.

## Best Possible Solution

Consolidate this under https://github.com/openclaw/openclaw/issues/90023 and make the chosen fix cover denied, empty-query, no-collection, clean-index, and dirty-index cases with focused memory-core tests.
