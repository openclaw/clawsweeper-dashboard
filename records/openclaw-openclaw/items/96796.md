## Summary

Keep open: current main and v2026.6.10 still have the reported QMD empty-result forced-sync path, and the linked closing PR at https://github.com/openclaw/openclaw/pull/96799 is still open rather than merged.

## What This Changes

The issue reports that QMD memory_search conflates empty search results with policy or setup outcomes and can trigger a forced QMD sync after non-result outcomes.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/96799 or an equivalent focused memory-core fix, then close this issue after main no longer blind-syncs on QMD guard/non-result outcomes.

## Reproduction Assessment

Yes, from source inspection: current main still force-syncs after any empty memory_search result, while QMD returns [] for scope-denied, empty-query, and no-collection paths. I did not run a live QMD runtime reproduction in this read-only review.

## Solution Assessment

Yes in direction: discriminated internal outcomes are the maintainable fix because they preserve search() compatibility while allowing memory_search to avoid sync for guard and clean-index outcomes. The linked PR still needs maintainer review before merge.
