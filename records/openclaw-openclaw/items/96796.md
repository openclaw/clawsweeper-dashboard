## Summary

Keep open: current main and the latest release still have the source-reproducible QMD empty-outcome forced-sync path, and the open closing PR should be reviewed before this issue closes.

## What This Changes

The issue asks for QMD memory_search to distinguish non-result outcomes and only force a sync for legitimate zero-hit searches on a dirty index.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/96799 or a narrower equivalent that keeps QMD guard/non-result outcomes out of blind forced syncs, with regression coverage for scope-denied, empty-query, no-collections, and clean-index zero-hit cases.

## Reproduction Assessment

Yes from source inspection. Current main still force-syncs after any empty memory_search result while QMD returns [] for scope-denied, empty-query, and no-collection paths; I did not run a live QMD timing repro.

## Solution Assessment

Yes in direction, but the exact API boundary needs maintainer review. A discriminated internal outcome is the maintainable fix shape because it preserves search() compatibility while letting memory_search avoid sync for guard and clean-index outcomes.
