## What This Changes

The issue requests that QMD memory_search distinguish guard/setup outcomes from real zero-hit searches so forced sync only runs for legitimate dirty zero-hit results.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/96799 after review and conflict resolution, or land an equivalent narrow fix that distinguishes QMD non-result outcomes and gates forced sync only on legitimate dirty zero-hit searches.

## Solution Assessment

Yes in direction: discriminating QMD search outcomes is the narrow maintainable fix because it preserves the public search() shape while letting memory_search avoid sync for guard and clean-index outcomes. The open PR still needs maintainer review and conflict resolution before this issue can close.
