## What This Changes

The issue asks for QMD-backed `memory_search` to distinguish empty-result outcomes so denied, empty, or unconfigured searches do not trigger synchronous forced QMD sync.

## Best Possible Solution

Land a reviewed memory/QMD fix that preserves `search()` compatibility while giving `memory_search` a discriminated outcome so it only force-syncs legitimate dirty zero-hit searches.

## Reproduction Assessment

Yes. Source inspection shows current main force-syncs after any empty `memory_search` result, while QMD returns `[]` for scope-denied, empty-query, and no-collection paths.

## Solution Assessment

Yes in direction. A discriminated internal outcome is the narrow maintainable fix because it preserves `search()` compatibility while letting `memory_search` avoid sync for guard and clean-index outcomes.
