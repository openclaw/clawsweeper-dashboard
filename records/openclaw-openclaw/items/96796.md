## Summary

Keep open: current main and v2026.6.10 still have the reported blind empty-result retry, and the open closing PR has not merged yet.

## What This Changes

The issue asks for QMD `memory_search` to distinguish guard/setup empty outcomes from legitimate zero hits so the interactive tool only force-syncs after an `ok` zero-hit search with a dirty index.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/96799 or an equivalent narrow fix that carries QMD search outcomes into the `memory_search` retry decision, then close this issue after main no longer force-syncs for QMD guard/setup outcomes.

## Reproduction Assessment

Yes from source inspection, though not from a live run in this review. Current main shows `memory_search` force-syncing after any empty result while QMD returns `[]` for scope-denied, empty-query, and no-collection paths.

## Solution Assessment

Yes in direction. Discriminated internal outcomes preserve the public `search()` behavior while letting `memory_search` avoid sync for guard/setup outcomes; the linked PR still needs merge-conflict and API-boundary review before this issue can close.
