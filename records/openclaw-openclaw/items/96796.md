## What This Changes

The issue reports that QMD-backed memory_search treats several non-result outcomes as empty hits and can force a blocking sync in the interactive search path.

## Best Possible Solution

Land a reviewed fix through https://github.com/openclaw/openclaw/pull/96799 or an equivalent narrow PR, then close once main distinguishes QMD outcomes and only force-syncs legitimate dirty zero-hit searches.

## Reproduction Assessment

Yes. Current main shows memory_search force-syncing after any empty result while QMD returns [] for scope-denied, empty-query, and no-collection paths; I did not run a live QMD timing repro.

## Solution Assessment

Yes in direction. A discriminated internal outcome is the narrow maintainable fix because it preserves search() compatibility while letting memory_search skip sync for guard and clean-index outcomes; the linked PR still needs normal compatibility review.
