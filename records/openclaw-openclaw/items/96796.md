## What This Changes

The issue requests that QMD memory_search distinguish legitimate zero-hit searches from guard/setup outcomes before triggering forced sync.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/96799 or an equivalent narrow fix that preserves search() compatibility while exposing enough internal outcome detail to skip forced sync for QMD guard/setup outcomes.

## Reproduction Assessment

Yes, source-reproducible but not live-reproduced here. Current main still force-syncs after any empty memory_search result while QMD returns [] for scope-denied, empty-query, and no-collection paths.

## Solution Assessment

Yes in direction. A discriminated outcome is the narrow maintainable fix shape because it preserves search() compatibility while letting memory_search avoid sync for QMD guard/setup outcomes; the linked PR still needs normal maintainer review before closure.
