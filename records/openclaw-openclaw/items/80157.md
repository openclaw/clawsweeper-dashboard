## Summary

Keep open: the diagnostic direction is useful and the real behavior proof is adequate, but the latest PR head still looks up agent tools by the raw binding ID while runtime normalizes binding IDs, so a routed mixed-case/sanitized agent can still avoid the new doctor warning.

## What This Changes

The PR adds a doctor preview warning and tests for channel-bound agents whose resolved tool policy excludes the `message` tool.

## Best Possible Solution

Keep the doctor warning, resolve binding agent IDs with the same normalized lookup runtime uses, add a mixed-case regression test, and avoid closing the broader linked issue unless the remaining runtime-introspection work is separately tracked or intentionally deferred.

## Reproduction Assessment

Yes. Source inspection of the latest PR head shows the raw `agentById.get(agentId)` lookup, while current main routes bindings through `pickFirstExistingAgentId`; an agent configured as `Commander` and bound as `commander` reproduces the missed-warning path by inspection.

## Solution Assessment

No. The doctor diagnostic is the right narrow surface, but the implementation should use runtime-equivalent normalized agent resolution before reading per-agent tools, and the broader linked issue should not be closed by a doctor-only branch without an explicit follow-up.
