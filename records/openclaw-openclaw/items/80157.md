## Summary

Keep open: this PR addresses a real doctor diagnostic gap and supplies enough live-output proof, but the latest head still misses a runtime-valid binding path because it reads agent tools with the raw binding id instead of the normalized agent id runtime uses.

## What This Changes

The PR adds a doctor preview warning and tests for channel-bound agents whose resolved tool policy excludes the `message` tool.

## Best Possible Solution

Keep the doctor warning, resolve binding agent IDs with the same normalized lookup runtime uses, add a mixed-case regression test, and keep the runtime self-introspection request in https://github.com/openclaw/openclaw/issues/80128 separate.

## Reproduction Assessment

Yes. Source inspection of PR head shows a raw `agentById.get(agentId)` lookup, while current main routes bindings through `pickFirstExistingAgentId`; a mixed-case configured agent and binding reproduces the missed-warning path by inspection.

## Solution Assessment

No. The doctor diagnostic is the right narrow surface, but the PR needs runtime-equivalent normalized agent resolution before reading per-agent tools.
