## Summary

Keep open: the doctor warning is useful, but the latest PR head still looks up bound agents by raw ID while runtime routing resolves agent IDs through normalized lookup, so mixed-case or hand-edited bindings can miss the warning.

## What This Changes

The PR adds a doctor preview warning and tests for channel-bound agents whose resolved tool policy excludes the `message` tool.

## Best Possible Solution

Keep the doctor warning, resolve binding agent IDs with the same normalized lookup runtime uses, add a mixed-case regression test, and avoid treating the broader linked issue as fully fixed until its runtime-introspection half is handled or explicitly deferred.

## Reproduction Assessment

Yes. Source inspection of the latest PR head shows a raw `agentById.get(agentId)` lookup, while current main routes bindings through normalized `pickFirstExistingAgentId`; a configured `Commander` agent bound as `commander` reproduces the missed-warning path by inspection.

## Solution Assessment

No. The doctor diagnostic is the right narrow surface, but it needs runtime-equivalent normalized agent resolution before reading per-agent tools.
