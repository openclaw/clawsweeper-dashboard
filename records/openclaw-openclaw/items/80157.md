## Summary

Keep open: the doctor-warning direction is useful and source-backed, but the latest PR head still misses runtime-equivalent agent ID resolution, so a routed mixed-case agent can remain silent in doctor.

## What This Changes

The PR adds a doctor preview warning and focused tests for channel-bound agents whose resolved tool policy excludes the `message` tool.

## Best Possible Solution

Keep the doctor warning, but resolve binding agent IDs the same way runtime does and add a mixed-case regression test.

## Reproduction Assessment

Yes. Source inspection shows the PR uses raw binding IDs for the new warning while runtime normalizes binding IDs through `pickFirstExistingAgentId`, so a mixed-case configured agent can route correctly but avoid the new warning.

## Solution Assessment

No. The diagnostic surface is the right narrow fix, but the implementation should use runtime-equivalent normalized agent resolution before reading the agent tool policy.
