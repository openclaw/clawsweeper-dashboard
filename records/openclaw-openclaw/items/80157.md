## What This Changes

## Best Possible Solution

Keep the doctor warning, resolve binding agent ids with the same normalized/fallback semantics as runtime, add the missing regression coverage, and leave the runtime self-introspection half to the linked issue or a follow-up.

Yes. Source inspection gives a concrete path: `agents.list[].id: "Commander"` with `tools.allow: ["read"]` and a binding to `commander` resolves at runtime, but this PR reads no per-agent tools and can suppress the warning.
