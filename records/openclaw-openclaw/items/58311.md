## Summary

Keep this PR open. Current main still has the targeted check/read patterns, and the patch is useful source work, but the security-sensitive error-semantics change needs maintainer review plus focused tests and a changelog entry before merge.

## What This Changes

The PR changes exec approvals, device identity, and device auth storage reads from `existsSync()` plus `readFileSync()` to direct reads with missing-file handling.

## Best Possible Solution

Land a maintained hardening patch that covers the intended local-state reads, uses explicit missing-file handling, has owner-approved fail-closed semantics, and includes focused regression tests plus a changelog entry.

## Reproduction Assessment

Yes. Current main has immediate `existsSync()` before `readFileSync()` in the affected paths, and the race/error behavior can be reproduced with mocked filesystem state where the existence check succeeds but the read throws or reads changed content.

## Solution Assessment

Mostly yes. Direct read plus explicit `ENOENT` handling is the narrow maintainable direction for this pattern, but this PR should not merge until the non-ENOENT behavior, identity-helper scope, tests, and changelog are handled.
