## What This Changes

The branch updates `src/infra/device-auth-store.ts`, `src/infra/device-identity.ts`, and `src/infra/exec-approvals.ts` to replace check-then-read file reads with direct reads and `ENOENT` handling.

## Best Possible Solution

Preserve current private-store auth and identity handling, then apply direct-read plus `ENOENT` hardening only to exec approvals with focused tests and an explicit decision on non-ENOENT read failures.

## Reproduction Assessment

Yes, source-level: current main still has `fs.existsSync(filePath)` immediately before `fs.readFileSync(filePath, "utf8")` in both exec approvals read paths. I did not run a race harness, so the runtime exploitability is not fully reproduced.

## Solution Assessment

No as-is. Direct read plus `ENOENT` handling is the right narrow shape for exec approvals, but the auth and identity hunks are stale and the fail-closed behavior needs explicit policy and upgrade tests.
