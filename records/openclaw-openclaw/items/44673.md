## Summary

Keep open: the English macOS VM guidance is still missing on current main, but this PR cannot merge as-is because it adds broad account auto-login guidance to a generic LaunchAgent error path, edits generated zh-CN docs, and has no real after-fix VM proof.

## What This Changes

The PR adds macOS VM auto-login guidance to English and zh-CN docs and appends an auto-login hint to the LaunchAgent bootstrap failure message.

## Best Possible Solution

Add narrowly scoped auto-login guidance to the English macOS VM source doc after Setup Assistant or in the headless section, keep runtime errors security-safe, and let the i18n pipeline regenerate localized docs.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection confirms current main's LaunchAgent GUI-domain failure path and macOS VM docs still omit auto-login guidance, but the PR lacks after-fix VM proof.

## Solution Assessment

No, not as proposed. The English docs direction is useful, but runtime auto-login advice needs isolated-VM scoping/security review and generated localized docs should come from the i18n pipeline.
