## Summary

Keep open. Current main still lacks auto-login guidance in the English macOS VM guide, so the central docs improvement is not implemented, but this PR remains blocked by missing real VM proof, a broad runtime auto-login recommendation, and an edit to generated localized docs.

## What This Changes

The PR adds macOS VM auto-login guidance to English and zh-CN docs and appends an auto-login hint to the LaunchAgent bootstrap failure message.

## Best Possible Solution

Add isolated-headless-VM auto-login guidance to the English macOS VM guide after Setup Assistant, let localization regenerate outside this repo, and keep runtime LaunchAgent errors scoped unless maintainers approve VM-only wording.

## Reproduction Assessment

No. Source inspection verifies the shared LaunchAgent GUI-domain failure path and the current docs gap, but I did not establish a real headless macOS VM reproduction or after-fix proof.
