## Summary

Keep open: the English macOS VM docs change is plausibly useful, but this PR should not merge as-is because it lacks real headless-VM proof, adds security-sensitive auto-login advice to a generic LaunchAgent error path, and edits generated localized docs that current policy keeps outside this repo.

## What This Changes

The branch updates English and zh-CN macOS VM docs with post-Setup Assistant auto-login guidance and adds a headless/VM auto-login hint to the LaunchAgent bootstrap error.

## Best Possible Solution

Add tightly scoped auto-login guidance to the English macOS VM guide after Setup Assistant, let the i18n pipeline regenerate locales, and keep runtime LaunchAgent wording scoped to sign-in/LaunchDaemon guidance unless maintainers approve VM-specific auto-login copy.

## Reproduction Assessment

No high-confidence real reproduction is present. Source inspection shows the current docs omit auto-login guidance and the LaunchAgent diagnostic path is generic, but the PR does not include a real headless macOS VM failure or after-fix run.

## Solution Assessment

No. The English doc direction is useful, but the PR mixes a broad account auto-login recommendation into a shared runtime error and edits localized generated docs instead of the English source/i18n pipeline boundary.
