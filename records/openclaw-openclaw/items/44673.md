## Summary

Keep open. The PR addresses a real documentation/runtime-guidance gap, but it should not merge as-is because the LaunchAgent error would recommend auto-login from a generic macOS bootstrap failure path and the branch still edits a localized docs file that current main intentionally no longer maintains.

## What This Changes

The PR adds macOS VM auto-login guidance to the English and Chinese macOS VM setup docs and adds an auto-login hint to the macOS LaunchAgent bootstrap failure message.

## Best Possible Solution

Refresh this into an English-source-docs-only patch that scopes auto-login to isolated headless macOS VMs, removes the localized docs edit, adds the required changelog entry, and updates the launchd guidance test if the generic runtime error text changes.

## Reproduction Assessment

Unclear for the live headless macOS VM bootstrap failure because I did not run a macOS VM. Static verification is high confidence: current main lacks the proposed auto-login guidance, and the PR diff shows exactly where it would be added.

## Solution Assessment

No, not as-is. The narrow maintainable solution is VM-scoped English docs guidance plus deliberate, tested runtime wording if maintainers want the generic LaunchAgent error to mention that workaround.
