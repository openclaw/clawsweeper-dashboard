## Summary

Keep open: the English macOS VM guidance is plausible, but the PR still lacks real headless VM proof and has merge-blocking issues around broad auto-login advice in a shared LaunchAgent diagnostic plus editing localized docs that this repo no longer maintains.

## What This Changes

The PR adds macOS auto-login guidance to the macOS VM docs and zh-CN docs, and adds an auto-login hint to the macOS LaunchAgent bootstrap error.

## Best Possible Solution

Add tightly scoped auto-login guidance to the English macOS VM guide after Setup Assistant with an isolated-VM caveat, leave localization to the i18n pipeline, and keep runtime LaunchAgent diagnostics focused on logged-in session or custom LaunchDaemon guidance unless maintainers approve broader auto-login copy.

## Reproduction Assessment

No high-confidence real reproduction is present. Source inspection shows the docs gap and generic LaunchAgent diagnostic path, but the PR body/checks do not include a real headless macOS VM failure or after-fix run.

## Solution Assessment

No. The VM guide direction is useful, but the patch broadens auto-login advice into a shared runtime error and edits generated localized docs; a scoped English docs update plus proof is safer.
