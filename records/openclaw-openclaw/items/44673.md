## Summary

Keep open. The PR targets a real macOS VM setup gap and Greptile's original placement concern is addressed in the current patch, but current main still lacks the requested auto-login guidance and the PR should not merge as-is because it adds a broad auto-login hint to a generic LaunchAgent failure path and edits generated localized docs that current policy says not to maintain in this repo.

## What This Changes

The PR adds macOS VM auto-login guidance to the English and Chinese VM setup docs and adds an auto-login hint to the macOS LaunchAgent bootstrap failure message.

## Best Possible Solution

Refresh the PR into an English-source-docs-only change: add narrowly scoped post-setup auto-login guidance for isolated headless macOS VMs, decide whether the generic LaunchAgent bootstrap error should mention that VM-specific workaround at all, update launchd guidance test coverage if runtime text changes, and leave localized docs to the i18n publish pipeline.

## Reproduction Assessment

Unclear for the live headless macOS VM bootstrap failure because I did not run a macOS VM reproduction. Static verification is high confidence: current main lacks the proposed guidance, and the PR diff shows the broad runtime auto-login hint plus the localized-doc edit.

## Solution Assessment

No, not as-is. The useful part is the VM setup guidance, but the narrower maintainable solution is to scope auto-login to isolated VM documentation, avoid broad host-level security advice in a generic LaunchAgent error, and drop the generated localized docs edit.

## Review Findings

Overall correctness: patch is incorrect
