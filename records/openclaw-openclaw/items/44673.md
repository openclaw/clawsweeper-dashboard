## Summary

This external PR should stay open because the English macOS VM docs gap is real, but the current patch still has blocking issues: the runtime error broadly recommends account auto-login, the PR edits generated zh-CN docs, and the contributor has not provided after-fix VM proof.

## What This Changes

The PR adds auto-login guidance to the macOS VM install docs in English and zh-CN and appends a headless/VM auto-login hint to the LaunchAgent bootstrap failure message.

## Best Possible Solution

Keep the useful guidance in the English macOS VM guide after Setup Assistant or in a headless-VM note, avoid broad runtime auto-login advice unless it is explicitly isolated-VM scoped, and let localization regenerate from source docs.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection confirms the current LaunchAgent GUI-session failure path and the macOS VM doc gap, but the PR lacks after-fix proof from a headless macOS VM.

## Solution Assessment
