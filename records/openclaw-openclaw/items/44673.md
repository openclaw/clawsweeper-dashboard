## Summary

Keep open: current main still lacks the useful macOS VM auto-login guidance, but this PR is not mergeable as proposed because it adds broad auto-login advice to a generic LaunchAgent failure path and edits generated localized docs.

## What This Changes

The PR adds auto-login guidance to the macOS VM docs in English and zh-CN and appends an auto-login hint to the LaunchAgent bootstrap error message.

## Best Possible Solution

Add narrowly scoped auto-login guidance to the English macOS VM source doc, avoid generic runtime auto-login advice unless it is explicitly isolated-VM scoped, and let the i18n pipeline regenerate localized docs.

## Reproduction Assessment

No high-confidence live VM reproduction or after-fix proof is available. Source inspection shows current main’s LaunchAgent GUI-domain error path and current macOS VM docs still omit auto-login guidance.

## Solution Assessment

No, not as proposed. The English doc direction is useful, but runtime auto-login advice needs isolated-VM/security scoping and generated locale files should not be edited in this source repo.
