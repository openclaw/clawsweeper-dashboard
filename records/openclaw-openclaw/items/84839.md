## Summary

Keep open: the report matches a plausible current-main race in the subagent completion/yield path, and I did not find a shipped or current-main fix that explicitly prioritizes a completed child event over the saved `sessions_yield` continuation.

## What This Changes

The issue reports that a parent session using `sessions_spawn` plus `sessions_yield` can resume from the generic yield continuation after its child has already completed, missing the child completion event.

## Best Possible Solution

Add a regression-tested routing change so completed child events are delivered or replayed ahead of the generic `sessions_yield` continuation when they race during parent yield teardown.

## Reproduction Assessment

No runnable high-confidence reproduction was established in this read-only pass. Source inspection does show a credible race: completion delivery can queue into a still-active requester while yield teardown later persists the generic `openclaw.sessions_yield` context.

## Solution Assessment

Yes: this should be fixed in the session yield/completion routing path, not by telling models to poll `sessions_list` or `subagents`. The narrow maintainable fix is to make child completion delivery win over or replay after the yield continuation when the two race.
