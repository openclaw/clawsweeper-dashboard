## Summary

Keep open: the report matches a documented subagent/yield contract, and current main still has a plausible race/visibility gap where the saved `sessions_yield` continuation can be the only transcript-visible context even though completion delivery is supposed to wake or hand off to the parent. The merged related fix for delivery mirrors is useful context, but it does not prove this yielded-parent completion path is fixed.

## What This Changes

The issue asks for `sessions_yield` to wake a yielded parent with the child completion event instead of a generic yield continuation.

## Best Possible Solution

Add a regression-tested session orchestration change so a child completion delivered during parent yield teardown is delivered or replayed ahead of the generic `sessions_yield` continuation.

## Reproduction Assessment

No high-confidence runnable reproduction was established. Source inspection shows a credible current-main race/visibility path between hidden `sessions_yield` continuation persistence and separate child completion handoff.

## Solution Assessment

Yes, the best fix belongs in the session yield/completion routing path. Models should not be told to poll `sessions_list`; the runtime should make child completion delivery win or replay when it races with yield continuation.
