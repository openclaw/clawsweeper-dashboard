## Summary

The related open work is useful context, but it does not prove that child completions now beat or replace the hidden yield continuation described in this report.

## What This Changes

The issue reports that a parent session using `sessions_spawn` followed by `sessions_yield` can resume from the yield continuation without receiving the completed child session event.

## Best Possible Solution

Choose one canonical fix path that preserves yielded parent session identity and delivers or replays child completion before the generic yield continuation can be model-visible.

## Reproduction Assessment

No high-confidence runnable current-main reproduction was executed in this read-only pass. The reporter supplied detailed v2026.5.19 logs, and current source shows the yielded parent becomes inactive while completion delivery only wakes active requesters before falling back to a separate handoff.

## Solution Assessment

Yes, the best fix belongs in session yield/completion routing rather than model polling. The implementation should coordinate with the adjacent paused-session work so dormant completed sessions still use direct fallback, while yielded parents keep a replayable completion target.
