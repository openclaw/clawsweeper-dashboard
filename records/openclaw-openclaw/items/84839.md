## Summary

Keep open: current main still shows a credible source-level gap between the documented `sessions_yield` completion contract and the yielded-parent wake path, and I did not find a merged fix proving the reporter's v2026.5.19 child-completion race is resolved.

## What This Changes

The issue reports that a parent using `sessions_spawn` then `sessions_yield` is resumed with the old hidden yield context instead of a model-visible child completion event.

## Best Possible Solution

Add regression-tested yielded-parent completion routing so child completion is delivered or replayed ahead of the generic yield context, coordinated with the paused-session lifecycle work already open.

## Reproduction Assessment

No high-confidence runnable current-main reproduction was established in this read-only pass. Source inspection and existing tests show that a yielded parent is idle, active-run wake can fail with `no_active_run`, and completion delivery then depends on a separate handoff path.

## Solution Assessment

Yes, the fix belongs in session orchestration and completion routing rather than model polling. The durable end state should make child completion win or replay when it races with yield continuation.
