## Summary

Keep open. Current main has a hardened low-level macOS `screen.snapshot` command, but the agent-facing `nodes` tool still exposes only `screen_record` plus generic invoke, so the reported assistant-facing screenshot and failure UX is not resolved.

## What This Changes

The issue asks OpenClaw to avoid telling users to perform the same operation the agent failed to perform, with screenshots as the concrete example.

## Best Possible Solution

Add a dedicated agent-facing `screen_snapshot` media action backed by `screen.snapshot`, with stable capture/permission errors and assistant guidance that offers OpenClaw-owned alternatives instead of reassigning the same failed operation to the user.

## Reproduction Assessment

No, not for the exact assistant transcript. Source inspection shows the agent-facing screenshot path is still missing, but this review did not run a live macOS denied-screen-recording scenario or capture current model wording.

## Solution Assessment

No, current main is not a complete solution. The low-level snapshot primitive is useful, but generic node invoke lacks media handling and does not define the assistant failure UX reported here.
