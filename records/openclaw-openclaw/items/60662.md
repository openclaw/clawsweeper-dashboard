## Summary

Keep open. Current main has the low-level macOS `screen.snapshot` primitive, but the agent-facing `nodes` tool still exposes `screen_record` and generic `invoke` without dedicated screenshot media handling or a defined failure-guidance contract.

## What This Changes

The issue asks OpenClaw to avoid handing a failed agent action back to the user and to provide an OpenClaw-owned path for screenshot-style requests.

## Best Possible Solution

Define a first-class agent-facing `screen_snapshot` media action backed by `screen.snapshot`, with tests for media result handling and failure guidance that offers OpenClaw-owned alternatives instead of redirecting the same operation back to the user.

## Reproduction Assessment

No high-confidence live reproduction exists for the exact assistant transcript on current main. Source inspection does show the remaining agent-facing gap: `screen.snapshot` exists below the tool layer, but `nodes` still lacks dedicated screenshot media handling.

## Solution Assessment

No, the current low-level primitive plus generic invoke path is not the best complete solution. The maintainable direction is an explicit agent media action and a defined failure-guidance contract.
