## Summary

Keep open: current main still lacks MCP OAuth refresh-token serialization, and this PR is the active narrow fix, but the patch still has a stale-store overwrite when another OAuth metadata write lands during a slow refresh.

## What This Changes

The PR adds MCP OAuth refresh-token serialization, atomic JSON store writes, streamable HTTP/SSE transport fetch wrapping, and focused tests.

## Best Possible Solution

Land the narrow serialization fix after refreshed-token persistence re-reads and merges the latest MCP OAuth store state, with regression coverage for concurrent non-token OAuth metadata writes.

## Reproduction Assessment

Yes, by source inspection. Current main sends SDK refresh-token POSTs through an unwrapped fetch and stores MCP OAuth credentials through direct JSON read/modify/write, so concurrent rotating-token refreshes can replay stale refresh tokens.

## Solution Assessment

No, not as written. Wrapping at the provider/transport boundary is the right fix shape, but refreshed-token persistence must merge into the latest store state before the patch is safe to merge.
