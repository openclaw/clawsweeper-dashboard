## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Merge the latest store before writing refreshed tokens:** `src/agents/mcp-oauth.ts:328`
  - body: The wrapper reads `store` before calling the token endpoint, then writes `{ ...store, tokens }` after the network response. If login or discovery code saves `clientInformation`, `codeVerifier`, `discoveryState`, redirect metadata, or `state` while that request is in flight, this write drops the newer metadata; re-read the store and merge the refreshed tokens into that latest state before persisting.
  - confidence: 0.9
- **[P2] Validate the token response before persisting it:** `src/agents/mcp-oauth.ts:327-328`
  - body: `parseRefreshResponseTokens` casts the cloned response JSON to `OAuthTokens` and stores it before the SDK runs `OAuthTokensSchema.parse`. A 200 response with JSON missing `access_token` or `token_type` would be written to disk and then rejected by SDK auth, leaving invalid credentials for later calls; validate the payload before `writeStore`.
  - confidence: 0.84
