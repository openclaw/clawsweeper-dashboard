## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Merge latest store before writing refreshed tokens:** `src/agents/mcp-oauth.ts:328`
  - body: The wrapper reads `store` before calling the token endpoint, then writes `{ ...store, tokens }` after the response. If login or discovery code saves client information, verifier, redirect, state, or discovery metadata while that request is in flight, this write drops the newer metadata; re-read the store after the response and merge tokens into that latest state.
  - confidence: 0.91
- **[P2] Bound refresh calls before stale lock reclaim:** `src/agents/mcp-oauth.ts:140-142`
  - body: The lock can be removed as stale after 120 seconds, but the token fetch inside the critical section has no timeout or abort. A slow live refresh can still be running when a later caller removes the lock and sends the same rotating refresh token, so keep the refresh deadline below the stale window or use the existing bounded lock pattern.
  - confidence: 0.78
