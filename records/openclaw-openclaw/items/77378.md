## Summary

Keep open: current main still leaves Mattermost non-direct inbound sessions without a persisted routable last route when `updateLastRoute` is omitted, and open PR #77442 already targets this report with closing syntax.

## What This Changes

The report asks OpenClaw to avoid duplicate Mattermost sessions after reload by completing delivery context for group/channel sessions and preventing stale queued replies from flushing later.

## Best Possible Solution

Land or revise PR #77442 so non-direct Mattermost inbound turns persist routable delivery context and legacy channel/group duplicates do not replay stale queued replies.

## Reproduction Assessment

No, not for the full duplicate-flush sequence; I did not live-run Mattermost reload plus LLM interruption. Source inspection on current main does show the missing non-direct route-persistence path that can leave Mattermost group/channel sessions without routable delivery context.

## Solution Assessment

Unclear until PR #77442 is reviewed. Persisting a fallback route from inbound context is the narrow first fix, but the legacy `channel:`/`group:` duplicate-session and stale queue behavior should also be covered before this closes.
