## Summary

Keep open: current main still has a source-visible Mattermost path where non-DM turns build a routable target but do not persist `updateLastRoute`, and the legacy `channel:`/`group:` duplicate-session cleanup is not fully covered by the related open PRs.

## What This Changes

Fix Mattermost reload/session rotation so one group channel cannot split across duplicate `channel:` and `group:` sessions with incomplete delivery context and replayed queued replies.

## Best Possible Solution

Add a Mattermost-owned fix and regression coverage that persists routable delivery context for group/channel sessions and safely handles legacy `channel:` versus `group:` entries for the same channel id without replaying stale queued replies.

## Reproduction Assessment

No for the full live incident, because I did not run Mattermost reload plus LLM interruption. Yes by source inspection for the core route-persistence failure: non-DM Mattermost turns still skip `updateLastRoute`, leaving no routable persisted delivery context.

## Solution Assessment

Unclear as a complete fix until the related PR is coordinated. The narrow maintainable direction is to fix this in the Mattermost plugin path, with only generic core cleanup if needed for legacy duplicate-session reconciliation.
