## Summary

Keep open: current main still has the Mattermost non-direct delivery-context persistence gap, and open PR #77442 already targets this report with closing syntax.

## What This Changes

The issue reports Mattermost reload/session rotation creating duplicate group/channel sessions whose old delivery context lacks a routable `to` target, causing queued replies and duplicate flushes.

## Best Possible Solution

Review and land or replace #77442 with a regression fix that persists routable delivery context for non-direct Mattermost sessions and confirms legacy duplicate-session/stale-queue behavior is handled.

## Reproduction Assessment

No high-confidence live reproduction for the full duplicate-flush sequence; I did not run Mattermost reload plus interrupted LLM delivery. Source inspection on current main does reproduce the missing non-direct route-persistence path that matches the reported incomplete delivery context.

## Solution Assessment

Unclear as a complete fix until #77442 is reviewed. Persisting non-direct inbound delivery context is the narrow first repair, but the legacy `channel:`/`group:` split and stale queued-response handling should be covered before this closes.
