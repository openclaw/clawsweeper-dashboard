Start in `extensions/mattermost/src/mattermost/monitor.ts`, `extensions/mattermost/src/mattermost/monitor-resources.ts`, `extensions/mattermost/src/session-route.ts`, and the session-store/update helpers only if needed. Check related PRs #52236 and #69010 first and reuse their useful coverage/approach without blindly stacking duplicate behavior. Expected fix boundary: persist `{channel,to,accountId,threadId}` for non-direct Mattermost turns, handle transient channel-kind lookup fallback or legacy `channel:` vs `group:` aliases for Mattermost safely, and prevent stale old-alias queued replies from being delivered as duplicate posts after rotation. Add focused regression tests and a user-facing changelog entry; do not change dependencies or broad provider routing policy.
