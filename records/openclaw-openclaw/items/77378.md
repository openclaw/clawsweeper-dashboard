Keep open: current main still leaves the Mattermost non-DM last-route persistence gap visible in source, and the related open PRs do not fully resolve the reported duplicate `channel:`/`group:` session rotation case.

## What This Changes

The issue requests that Mattermost group/channel reloads keep one routable active session, persist complete delivery context, and avoid replaying stale queued responses.

## Best Possible Solution

Add a Mattermost-owned fix and regression coverage that persist routable delivery context for group/channel sessions and reconcile legacy `channel:` versus `group:` sessions for the same Mattermost channel id without replaying stale queued responses.
