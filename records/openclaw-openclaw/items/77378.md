## Summary

Keep open: current main still has a source-visible Mattermost non-DM delivery-context gap, and open PR #77442 explicitly closes this issue but is not merged yet.

## What This Changes

The issue requests fixing Mattermost session rotation so group/channel sessions keep complete delivery context, legacy `channel:`/`group:` duplicates do not coexist for the same channel id, and stale queued replies are not flushed as duplicate posts.

## Best Possible Solution

Review and land PR #77442 or a successor that persists routable delivery context for non-direct inbound sessions and safely reconciles legacy Mattermost `channel:`/`group:` duplicates without replaying stale queues.

## Reproduction Assessment

No for a high-confidence end-to-end duplicate flush, because I did not live-run Mattermost reload plus LLM interruption. Yes for the source-level core of the bug: current main builds `OriginatingTo` for group/channel inbound messages but omits the non-DM `updateLastRoute` path that persists a routable `deliveryContext.to`.

## Solution Assessment

Unclear as a complete fix until PR #77442 is reviewed. Its direction matches the narrow missing-context repair, but the duplicate legacy session and stale-queue behavior should be checked before this issue closes.
