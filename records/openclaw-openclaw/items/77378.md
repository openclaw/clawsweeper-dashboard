## Summary

Keep open: current main still has the Mattermost non-DM delivery-context persistence gap, and open PR #77442 already targets this report with closing syntax.

## What This Changes

Mattermost session rotation can leave an old channel-scoped group session with incomplete delivery context, causing undelivered queued responses and duplicate posts after rotation.

## Best Possible Solution

Land a fix that persists routable delivery context for non-direct Mattermost inbound sessions and handles legacy channel/group duplicates without replaying stale queued responses.

## Reproduction Assessment

No, not for the full end-to-end duplicate-flush sequence. Source inspection on current main does show the missing route-persistence path for Mattermost non-DM sessions.

## Solution Assessment

Unclear as a complete fix until PR #77442 is reviewed. Persisting non-direct delivery context is the narrow first repair, but stale legacy duplicate-session behavior should be covered before this is closed.
