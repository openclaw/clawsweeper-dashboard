## Summary

Keep open: current main still has the Mattermost non-direct delivery-context gap, and open PR #77442 already references this issue with closing syntax, so the issue should stay open until that PR lands or is rejected.

## What This Changes

Report that Mattermost reload/session rotation can leave duplicate channel/group sessions with incomplete deliveryContext, causing queued responses to flush as duplicate posts.

## Best Possible Solution

Use PR #77442 as the active fix candidate, ensuring it persists routable non-direct inbound delivery context and covers legacy channel/group duplicate-session behavior before this issue closes.

## Reproduction Assessment

No for the complete live duplicate-flush sequence; I did not run Mattermost reload plus interrupted LLM delivery. Yes at source level for the core missing route persistence path: non-direct Mattermost turns omit `updateLastRoute`, and core returns without persisting `deliveryContext.to`.

## Solution Assessment

Unclear as a complete fix until the linked PR is reviewed. Persisting a fallback route from inbound context is the narrow first repair, but the legacy `channel:`/`group:` duplicate-session case needs explicit regression coverage or a documented follow-up.
