## Summary

Keep open: current main still has a source-visible Mattermost non-DM delivery-context gap, and open PR #77442 explicitly uses closing syntax for this issue, so the issue should remain open until that implementation is reviewed and merged or rejected.

## What This Changes

The issue reports Mattermost reloads creating duplicate `channel:`/`group:` sessions with incomplete delivery context, causing undelivered queued replies to flush later as duplicate posts.

## Best Possible Solution

Land a narrow session-routing fix that persists routable delivery context for Mattermost group/channel turns and safely handles legacy `channel:` versus `group:` entries for the same Mattermost channel id without replaying stale queued replies.

## Reproduction Assessment

Yes for the core missing-route path by source inspection: current main computes `OriginatingTo` for Mattermost group/channel turns but does not persist `updateLastRoute` for them. No for the full duplicate flush, because I did not live-run Mattermost reload plus interrupted LLM delivery.

## Solution Assessment

Unclear as complete issue coverage: persisting non-direct inbound delivery context is the narrow first fix, but the final solution should confirm legacy duplicate `channel:`/`group:` sessions do not continue processing and replay queued responses.
