## Summary

Keep open. Current main has useful partial coverage for configured provider models in catalog, list, runtime resolution, and thinking metadata, but the RFC asks for a product-wide first-class resolution invariant and shared descriptor; `models status`, list rows, runtime, and capability policy still consume separate resolution paths.

## What This Changes

Request to treat `models.providers.<provider>.models` entries as first-class forward-compat model definitions across status, list, runtime, availability, thinking gates, docs, and validation.

## Best Possible Solution

Keep the RFC open until maintainers decide the canonical model-resolution contract, then land incremental PRs that move status, list, runtime, availability, thinking policy, and docs onto that shared result.

## Reproduction Assessment

Not applicable. This is an RFC for a product-wide model-resolution invariant, and the review used current source, docs, related discussion, and history instead of trying to reproduce one concrete failure.

## Solution Assessment

Unclear for implementation. The requested direction is coherent, but the best fix needs maintainer design agreement because the current code has partial support spread across separate list, status, runtime, catalog, and thinking paths.
