## Summary

Keep open. Current main still has lower-level `promptMode` support, but `sessions_spawn` does not expose or forward a public `promptMode` parameter; the linked task-visibility bug was fixed separately by https://github.com/openclaw/openclaw/pull/78356 and does not supersede this API request.

## What This Changes

Expose an optional `promptMode: "full" | "minimal" | "none"` parameter on `sessions_spawn` so native sub-agents can run with deterministic blank prompts.

## Best Possible Solution

If maintainers approve the public API, add a native `sessions_spawn.promptMode` enum with current defaults preserved, forward it through `SpawnSubagentParams` into the child run, preserve task and attachment visibility under `none`, and document native/ACP semantics.

## Reproduction Assessment

Yes for the current implementation state. Source inspection shows lower-level `agent` runs accept `promptMode`, while the public `sessions_spawn` schema, native params, forwarding path, and docs still omit it.

## Solution Assessment

Unclear until maintainers approve the public API semantics. Exposing the existing enum is the narrow technical path, but raw-mode task/attachment delivery and ACP behavior need to be defined first.
