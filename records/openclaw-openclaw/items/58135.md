## Summary

Keep open. Current main still exposes `promptMode` only on lower-level agent runs and system-prompt rendering, not on the model-callable `sessions_spawn` schema or native subagent spawn handoff, so this remains a real but product-sensitive API request.

## What This Changes

Request to add an optional `promptMode` parameter to `sessions_spawn` so spawned sub-agents can use `none` mode for deterministic blank prompt contexts.

## Best Possible Solution

If maintainers approve the API, add a validated optional promptMode enum to `sessions_spawn`, preserve the current minimal default, forward it into the child agent run, document the semantics, and cover schema plus forwarding behavior in focused tests.

## Reproduction Assessment

Yes for source-level verification of the missing feature: current `sessions_spawn` does not expose or forward `promptMode`, while lower-level agent runs and system-prompt rendering already support it. This is not a strict bug reproduction because the issue requests new public API surface.

## Solution Assessment

Unclear until maintainers approve the public prompt-mode contract. Exposing the existing enum is the narrow technical path if `sessions_spawn` should own this control; otherwise prompt mode should remain runtime-owned and documented as not model-callable.
