## Summary

Keep open. Current main still has lower-level `promptMode` support, but the public `sessions_spawn` schema, forwarding path, and docs do not expose it; a direct passthrough also needs product/API judgment because `promptMode: "none"` is raw-run behavior while native sub-agent task delivery currently depends on system-prompt context.

## What This Changes

Expose an optional `sessions_spawn.promptMode` parameter for deterministic blank native sub-agents while preserving current defaults.

## Best Possible Solution

Define a public native `sessions_spawn.promptMode` contract, preserve current defaults, document unsupported combinations, and add tests proving task and attachment visibility under `none` or deliberately rejecting unsafe combinations.

## Reproduction Assessment

Yes for the current source state: the public schema and native forwarding path omit `promptMode` while the lower-level Gateway `agent` RPC accepts it. This is source-verifiable missing feature surface, not a broken existing documented contract.

## Solution Assessment

Unclear until maintainers approve the public contract. If approved, extending native `sessions_spawn` is the narrow code path, but `promptMode: "none"` needs defined task, attachment, and ACP behavior before implementation.
