## Summary

Keep open. Current main and the latest stable release still do not expose `promptMode` on `sessions_spawn`; the lower-level `agent` request path supports it, but making it model-callable through spawn remains a public API and prompt-context product decision.

## What This Changes

The issue requests an optional `promptMode` parameter on `sessions_spawn` so callers can spawn deterministic blank sub-agents with `promptMode: "none"`.

## Best Possible Solution

If maintainers approve the API, add a native `sessions_spawn.promptMode` enum with current defaults preserved, define `none` semantics for task and attachment visibility, document the contract, and add focused regression coverage.

## Reproduction Assessment

Yes for source-level verification, not as a strict runtime bug. Current main and `v2026.5.19` clearly omit `promptMode` from the public `sessions_spawn` schema, native params, and forwarding path while lower-level agent runs accept it.

## Solution Assessment

Unclear until maintainers settle the public prompt-mode contract. Exposing the existing enum is the narrow technical path if approved, but `none` mode, attachments, task visibility, and ACP/trusted-caller semantics need to be specified first.
