## Summary

Keep open. Current main and v2026.6.8 still expose promptMode on the lower-level agent run API, but not on the public sessions_spawn schema or native spawn handoff; adding it remains a useful but product-sensitive public API decision.

## What This Changes

Request to add an optional promptMode parameter to sessions_spawn so spawned child agents can choose full, minimal, or none prompt construction while preserving the current default.

## Best Possible Solution

If maintainers approve the API, add a bounded optional promptMode field that preserves the current default, defines none-mode task and attachment semantics, updates docs, and adds focused schema plus child-run forwarding coverage.

## Reproduction Assessment

Yes for source-level verification, not as a broken existing-behavior repro. Current main and v2026.6.8 omit promptMode from sessions_spawn while the lower-level agent run API accepts it.

## Solution Assessment

Unclear until maintainers settle the public prompt-mode contract. Exposing the existing enum is the narrow technical path if sessions_spawn should own this control, but none-mode raw-run semantics need approval first.
