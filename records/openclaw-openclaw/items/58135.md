## Summary

Keep open. Current main still has lower-level `promptMode` support, but `sessions_spawn` does not expose or forward it; because docs currently describe prompt modes as runtime-only, this remains a public API/product decision rather than an implemented feature.

## What This Changes

The issue requests an optional `sessions_spawn.promptMode` enum so callers can spawn deterministic blank sub-agents with `promptMode: "none"`.

## Best Possible Solution

If maintainers approve the public API, add a native `sessions_spawn.promptMode` enum with current defaults preserved, forward it through `SpawnSubagentParams` into the child run, preserve task and attachment visibility under `none`, and document native/ACP semantics.

## Reproduction Assessment

Yes for the current implementation state. Source inspection shows lower-level `agent` runs accept `promptMode`, while the public `sessions_spawn` schema, executor forwarding, native params, tests, and docs omit it.

## Solution Assessment

Yes with a maintainer API caveat. Exposing the existing enum on native `sessions_spawn` is the narrow technical path if this should become public, but raw-mode task delivery and ACP behavior must be settled first.
