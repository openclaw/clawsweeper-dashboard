## Summary

Keep open. Current main still supports `promptMode` only in lower-level agent/model-run paths; the public `sessions_spawn` tool does not expose, parse, forward, test, or document a prompt-mode parameter, and the related task-delivery fix at https://github.com/openclaw/openclaw/issues/77950 did not add this API.

## What This Changes

Expose an optional `promptMode: "full" | "minimal" | "none"` parameter on `sessions_spawn` so native sub-agents can be spawned with a deterministic blank prompt mode.

## Best Possible Solution

If maintainers approve the public API, add a native `sessions_spawn.promptMode` enum with existing defaults preserved, forward it into the child run safely, keep task and attachment delivery visible in `none` mode, and document the native/ACP behavior at https://docs.openclaw.ai/tools/subagents.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: lower-level `agent` runs accept `promptMode`, while the current `sessions_spawn` schema, executor forwarding, native params, tests, and docs omit it.

## Solution Assessment

Yes with a maintainer API caveat. Exposing the existing enum on native `sessions_spawn` is the narrowest technical path if maintainers want this public surface, but task delivery and ACP semantics need to be settled first.
