## Summary

Keep open. Current main and the latest stable release still do not expose `promptMode` on `sessions_spawn`, while lower-level agent runs already accept it; the remaining work is an additive public API/product decision rather than cleanup-close material.

## What This Changes

Expose an optional `promptMode` enum on `sessions_spawn` so spawned sub-agents can request full, minimal, or no injected prompt context.

## Best Possible Solution

If maintainers approve the API, add a native `sessions_spawn.promptMode` enum with current defaults preserved, forward it into child agent runs, document native/ACP behavior, and test task plus attachment visibility under `none`.

## Reproduction Assessment

Yes for source-level verification, not as a strict runtime bug. Current main clearly omits `promptMode` from the public `sessions_spawn` schema, native params, and forwarding path while lower-level agent runs accept it.

## Solution Assessment

Unclear until maintainers settle the public prompt-mode contract. Exposing the existing enum is the narrow technical path if approved, but `none` mode, attachments, and ACP/trusted-caller semantics need to be specified first.
