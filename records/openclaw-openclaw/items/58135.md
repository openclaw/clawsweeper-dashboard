## Summary

Keep open. Current main and the latest release still do not expose `promptMode` on the model-callable `sessions_spawn` path, and the remaining work is a product-sensitive public API decision rather than a cleanup-close candidate.

## What This Changes

Request to add an optional `promptMode: "full" | "minimal" | "none"` parameter to `sessions_spawn` so callers can create deterministic blank sub-agents.

## Best Possible Solution

Define the public blank-subagent contract first; if approved, add a validated optional enum to `sessions_spawn`, preserve the current default, forward it through native spawn intentionally, document the `none` semantics, and cover schema plus runtime behavior in focused tests.

## Reproduction Assessment

Yes for source-level verification: the current `sessions_spawn` schema and native spawn params do not include `promptMode`, while lower-level agent runs do. This is not a failing existing-behavior repro because the issue asks for new public API surface.

## Solution Assessment

Unclear until maintainers approve the contract. Exposing the existing enum is the narrow technical path, but current `promptMode="none"` raw-run behavior means the best implementation must define blank-subagent semantics rather than blindly forwarding the lower-level flag.
