## Summary

Keep open. Current main has lower-level `promptMode` support for direct agent runs and the system-prompt builder, but the public `sessions_spawn` tool does not expose, parse, or forward a prompt-mode parameter, and the subagent docs do not document one. The request is valid and narrow, with one important implementation caveat: native subagents currently put the assigned task in `extraSystemPrompt`, which `promptMode: "none"` would skip unless the fix preserves task delivery another way.

## What This Changes

Expose an optional `promptMode: "full" | "minimal" | "none"` parameter on `sessions_spawn` so native subagent runs can request deterministic blank prompt mode while preserving current defaults.

## Best Possible Solution

Add a native `sessions_spawn.promptMode` API that preserves the existing omitted-parameter default, forwards valid modes into the child agent run, and explicitly preserves task and attachment delivery when `none` is requested without reintroducing AGENTS/TOOLS/skills/workspace context. Document the parameter at https://docs.openclaw.ai/tools/subagents and cover the schema, forwarding, default, and `none` task-delivery behavior with focused tests.

## Reproduction Assessment

Yes. The current-main source path is enough to reproduce the missing feature: `promptMode` exists on the lower-level agent run but is absent from `sessions_spawn` schema, parsing, native params, forwarding, and docs.

## Solution Assessment

Yes, with one caveat. Exposing the existing prompt-mode enum on native `sessions_spawn` is the narrowest maintainable direction, but the implementation must not rely on `extraSystemPrompt` for the assigned task when `promptMode: "none"` is selected.
