## Summary

Keep open. Current main still supports `promptMode` below the public tool layer, but `sessions_spawn` does not expose, parse, forward, or document a prompt-mode parameter; the request is narrow but needs care because `promptMode: "none"` currently takes the raw-run path that can discard the native subagent task prompt.

## What This Changes

Expose an optional `promptMode: "full" | "minimal" | "none"` parameter on `sessions_spawn` for deterministic blank native subagents while preserving current defaults.

## Best Possible Solution

Add a native `sessions_spawn.promptMode` API that preserves omitted-parameter defaults, forwards valid modes into the child run, preserves task and attachment delivery in `none` mode, and documents the behavior at `https://docs.openclaw.ai/tools/subagents`.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: `promptMode` exists on lower-level agent runs, but current `sessions_spawn` schema, parsing, native spawn params, gateway forwarding, and docs omit it.

## Solution Assessment

Yes, with a caveat. Exposing the existing prompt-mode enum on native `sessions_spawn` is the narrowest maintainable direction, but the implementation must not rely on `extraSystemPrompt` for the assigned task when `promptMode: "none"` is selected.
