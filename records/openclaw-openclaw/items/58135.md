## Summary

Keep open. Current main still supports `promptMode` at the lower-level agent run boundary, but the public `sessions_spawn` schema, forwarding path, native spawn params, tests, and docs do not expose it; making it public needs maintainer API/product review because `promptMode: "none"` changes raw-run, task-delivery, attachment, and ACP semantics.

## What This Changes

Expose an optional `sessions_spawn.promptMode` enum so callers can request deterministic blank sub-agent runs.

## Best Possible Solution

If maintainers approve the public API, add a native `sessions_spawn.promptMode` enum with current defaults preserved, forward it into the child run, define native/ACP behavior, and cover task plus attachment visibility under `none`.

## Reproduction Assessment

Yes for the current implementation state: source inspection shows lower-level `agent` runs accept `promptMode`, while the public `sessions_spawn` schema, forwarding path, native params, and docs omit it. This is a source-verifiable missing feature rather than a broken existing contract.

## Solution Assessment

Yes with a maintainer API caveat. Exposing the existing enum on native `sessions_spawn` is the narrow technical path if this should become public, but raw-mode task delivery, attachments, and ACP behavior need explicit semantics first.
