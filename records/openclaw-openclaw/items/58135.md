## Summary

Keep open. Current main and v2026.6.8 still lack `promptMode` on the public `sessions_spawn` schema and native spawn handoff, while lower-level agent runs already accept it; docs currently frame prompt mode as runtime-set, so this remains a useful but product-sensitive public API decision.

## What This Changes

Request to add an optional `promptMode: "full" | "minimal" | "none"` parameter to `sessions_spawn` for deterministic blank sub-agent runs.

## Best Possible Solution

If maintainers approve the API, add a bounded optional `promptMode` field that preserves current defaults, defines none-mode semantics, updates docs, and adds focused schema plus child-run forwarding coverage.

## Reproduction Assessment

Yes for source-level verification, not as a broken existing-behavior repro. Current main and v2026.6.8 omit `promptMode` from `sessions_spawn`, while the lower-level agent request and embedded runner accept it.

## Solution Assessment

Unclear until maintainers settle the public prompt-mode contract. Exposing the existing enum is the narrow technical path if `sessions_spawn` should own this control, but none-mode raw-run semantics need approval first.
