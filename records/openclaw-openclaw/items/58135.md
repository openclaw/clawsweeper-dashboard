## Summary

Keep open. Current main and v2026.6.8 still expose promptMode only on the lower-level agent run API, not on the public sessions_spawn schema or native spawn forwarding path; adding it remains a public API/product decision because promptMode="none" is also treated as raw-run behavior.

## What This Changes

Request to add an optional promptMode enum to sessions_spawn so callers can create deterministic blank sub-agent runs.

## Best Possible Solution

If maintainers approve the API, add a bounded optional promptMode field that preserves the current default, defines none-mode child task and attachment semantics, updates docs, and adds focused regression coverage.

## Reproduction Assessment

Yes for source-level verification, not as a broken existing-behavior repro. Current main and v2026.6.8 omit promptMode from sessions_spawn while the lower-level agent run API accepts it.

## Solution Assessment

Unclear until maintainers settle the public prompt-mode contract. Exposing the existing enum is the narrow technical path if sessions_spawn should own this control, but none-mode raw-run semantics need approval first.
