## Summary

Keep open. Current main and the latest release still omit `promptMode` from `sessions_spawn` even though lower-level agent runs support it; this remains a public API/product decision rather than a cleanup-close candidate.

## What This Changes

The issue requests adding an optional `promptMode: "full" | "minimal" | "none"` parameter to `sessions_spawn` for deterministic blank sub-agent runs.

## Best Possible Solution

If maintainers approve the API, add a native `sessions_spawn.promptMode` enum with current defaults preserved, forward it through the child run, document native/ACP behavior, and test task plus attachment visibility under `none`.

## Reproduction Assessment

Yes. Source inspection on current main shows lower-level `agent` runs accept `promptMode`, while the public `sessions_spawn` schema, params, forwarding path, tests, and docs do not expose it.

## Solution Assessment

Yes with a maintainer API caveat. Exposing the existing enum on native `sessions_spawn` is the narrow technical path if this should become public, but raw-mode, attachment, and ACP semantics need to be specified first.
