The issue reports duplicate Control UI/WebChat user messages entering the same agent session through different sender identities and bypassing message-id-based inbound dedupe.

## Best Possible Solution

Keep the shipped gateway active-send dedupe and regression coverage; the broader idless inbound fallback remains separate in https://github.com/openclaw/openclaw/pull/72314.
