## Summary

Keep open: the branch targets a real scheduler-loop class, but the proposed timer floor leaves stale schedule state in place and the updated re-review comments still provide only fake-timer proof rather than real runtime behavior proof.

## What This Changes

This PR adds a 2 s minimum heartbeat scheduler refire gap for past-due timers, fake-timer scheduler coverage, a Discord voice test mock signature tweak, and a changelog entry.

## Best Possible Solution

Repair stale heartbeat nextDueMs after terminal skipped or deferred scheduled outcomes so the runner returns to the configured cadence, with focused regression coverage and redacted runtime proof.

## Reproduction Assessment

Yes for the scheduler defect by source inspection: current main can leave nextDueMs in the past after a skipped/disabled scheduled outcome and then re-arm immediately. No live Raspberry Pi or Docker crash-loop reproduction was established here.

## Solution Assessment

No. The timer floor breaks the setTimeout(0) loop, but it still leaves stale nextDueMs intact and converts the failure into recurring 2 s scheduler wakes instead of restoring normal cadence.
