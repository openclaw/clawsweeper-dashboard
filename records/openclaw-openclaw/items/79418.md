## Summary

Keep this PR open, but the current patch should not merge as-is: it converts a zero-delay heartbeat hot loop into a recurring 2 s wake loop instead of repairing the stale schedule state, and the external PR still has only fake-timer proof for a reported real Raspberry Pi Docker crash loop.

## What This Changes

The PR adds a 2 s heartbeat scheduler refire floor for past-due timers, two fake-timer scheduler tests, a Discord voice test mock signature tweak, and a changelog entry.

## Best Possible Solution

Repair stale heartbeat nextDueMs after terminal skipped or deferred scheduled outcomes so the runner returns to the configured cadence, with focused regression coverage and redacted real runtime proof.

## Reproduction Assessment

Yes for the scheduler defect by source inspection: current main can leave nextDueMs in the past after skipped/disabled outcomes and then re-arm with delay 0. No live Raspberry Pi or Docker crash-loop reproduction was established here.

## Solution Assessment

No. The timer floor breaks the tight setTimeout(0) loop, but the maintainable fix is to repair or clear the stale schedule state so the runner settles back to its configured heartbeat cadence.
