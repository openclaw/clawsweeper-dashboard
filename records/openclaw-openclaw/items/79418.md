## What This Changes

Adds a 2 second minimum heartbeat scheduler refire delay for past-due timers, fake-timer scheduler tests, a Discord test mock signature tweak, and a changelog entry.

## Best Possible Solution

Repair stale heartbeat schedule state after terminal skipped or deferred scheduled outcomes so the runner returns to configured cadence, then add focused regression coverage and real gateway proof.

## Reproduction Assessment

## Solution Assessment

No. A 2 second floor avoids the tightest setTimeout(0) loop, but the maintainable fix should advance, clear, or recompute stale schedule state instead of polling the same past-due agents every 2 seconds.
