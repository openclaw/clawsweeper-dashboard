## Summary

Keep open: the reported scheduler bug is source-reproducible on current main, but this PR floors stale timers instead of repairing stale heartbeat schedule state, so it can still wake every 2 seconds indefinitely; it also lacks after-fix real runtime proof.

## What This Changes

This PR adds a 2 second minimum delay for past-due heartbeat scheduler timers, fake-timer scheduler tests, a Discord voice test mock signature tweak, and a changelog entry.

## Best Possible Solution

Repair stale heartbeat nextDueMs after terminal skipped or deferred scheduled outcomes so timer re-arms return to the configured cadence, with focused regression coverage and redacted real runtime proof.

## Reproduction Assessment

Yes for the scheduler defect by source inspection: current main can leave nextDueMs in the past after a skipped/disabled scheduled outcome and then re-arm with delay 0. The full Raspberry Pi Docker crash-loop symptom was not live-reproduced in this review.

## Solution Assessment

No. A timer floor is a mitigation, but the narrower maintainable fix is to advance, clear, or recompute stale schedule state so the runner returns to its configured heartbeat cadence.
