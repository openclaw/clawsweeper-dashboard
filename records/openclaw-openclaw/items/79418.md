## What This Changes

The branch adds a 2 s heartbeat scheduler refire floor for past-due timers, fake-timer scheduler tests, a Discord voice mock signature tweak, and a changelog entry.

## Best Possible Solution

Repair stale heartbeat schedule state after terminal skipped or deferred scheduled outcomes so agents return to configured cadence, with focused regression coverage and redacted real gateway proof.

## Reproduction Assessment

Yes for the scheduler hot-loop path: current main can keep `nextDueMs` stale after disabled scheduled skips, then re-arm with a 0 ms timeout. No live Raspberry Pi/Docker crash-loop reproduction was established in this review.

## Solution Assessment

No. A 2 s floor avoids the tightest `setTimeout(0)` loop, but the narrower maintainable fix is to advance, clear, or recompute the stale schedule state instead of polling stale agents every 2 seconds.
