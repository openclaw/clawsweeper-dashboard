## Summary

Keep open. The reported scheduler failure is credible from current-main source inspection, but this PR leaves stale `nextDueMs` state in place and only changes the hot loop into repeated 2-second polling; it also still lacks after-fix proof from a real OpenClaw runtime.

## What This Changes

The PR adds a 2-second heartbeat scheduler refire floor, scheduler fake-timer tests, a changelog entry, and a Discord voice test mock signature adjustment.

## Best Possible Solution

Advance or clear stale heartbeat `nextDueMs` for terminal skipped/deferred scheduled outcomes so the runner returns to the configured interval, with focused regression tests and redacted real runtime proof.

## Reproduction Assessment

Yes for the scheduler failure by source inspection: current main can leave `nextDueMs` in the past and re-arm a zero-delay timer. The full Raspberry Pi Docker CPU symptom was not live-reproduced in this read-only review.

## Solution Assessment

No. The PR reduces immediate event-loop pressure, but the narrower maintainable fix is to repair the stale schedule state instead of polling the same past-due entry every 2 seconds.
