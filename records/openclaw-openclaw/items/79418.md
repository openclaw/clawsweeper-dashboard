## Summary

Keep open: the reported scheduler defect is plausible from current source, but this PR mitigates the hot loop by polling every 2 seconds instead of repairing stale schedule state, and the external PR still has only mock/test proof.

## What This Changes

The PR adds a 2 s minimum heartbeat scheduler refire gap for past-due timers, fake-timer scheduler coverage, a Discord voice test mock signature tweak, and a changelog entry.

## Best Possible Solution

Repair stale heartbeat nextDueMs after terminal skipped or deferred scheduled outcomes so the runner returns to the configured cadence, with focused regression coverage and redacted real runtime proof.

## Reproduction Assessment

Yes for the scheduler defect by source inspection: current main can leave nextDueMs in the past after a skipped/disabled scheduled outcome and then re-arm with delay 0. No live Raspberry Pi or Docker crash-loop reproduction was established in this review.

## Solution Assessment

No. A timer floor prevents the tight setTimeout(0) loop, but it still wakes repeatedly every 2 seconds for stale schedules instead of restoring the agent to its configured heartbeat cadence.
