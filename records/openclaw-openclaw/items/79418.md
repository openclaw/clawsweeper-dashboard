## Summary

Keep open. The scheduler defect is source-reproducible and the PR is focused, but the patch changes the zero-delay hot loop into a recurring 2 s poll while leaving the stale schedule state in place, and the branch still lacks real runtime proof for the reported Raspberry Pi/Docker gateway symptom.

## What This Changes

The PR adds a 2 s floor to heartbeat scheduler refires when the next due time is in the past, adds fake-timer scheduler tests, adjusts a Discord voice test mock signature, and adds a changelog entry.

## Best Possible Solution

Repair stale heartbeat nextDueMs after terminal skipped or deferred scheduled outcomes so the runner returns to the configured cadence, with focused regression coverage and redacted real runtime proof.

## Reproduction Assessment

Yes for a source-level scheduler reproduction: current main can leave nextDueMs in the past after a scheduled skipped/disabled result and then re-arm scheduleNext with rawDelay 0. No live Raspberry Pi or Docker crash-loop reproduction was established in this review.

## Solution Assessment

No. A 2 s timer floor prevents setTimeout(0), but it still wakes repeatedly for stale schedules instead of restoring the agent to its configured heartbeat cadence; the safer fix is to repair or clear the stale due state.
