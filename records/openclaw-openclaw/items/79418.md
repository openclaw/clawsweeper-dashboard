## Summary

Keep this PR open: it targets a real, source-reproducible heartbeat scheduler failure, but the proposed 2 s timer floor is only a mitigation and the PR still lacks after-fix proof from a real OpenClaw setup.

## What This Changes

The PR adds a 2 s heartbeat refire floor for past-due scheduler timers, scheduler fake-timer tests, a changelog entry, and a Discord voice test mock signature update.

## Best Possible Solution

Advance or clear stale heartbeat `nextDueMs` for terminal skipped/deferred outcomes so scheduled wakes return to the configured interval, keep focused regression coverage, and collect redacted real runtime proof from a Raspberry Pi or comparable OpenClaw setup.

## Reproduction Assessment

Yes for the scheduler failure by source inspection: current main can leave `nextDueMs` in the past after a disabled scheduled skip and immediately re-arm the scheduler. The full Raspberry Pi Docker CPU symptom was not live-reproduced in this read-only review.

## Solution Assessment

No. The PR's timer floor reduces `setTimeout(0)` pressure, but the narrower maintainable fix is to update or clear the stale heartbeat schedule state instead of polling the same stale due entry every 2 seconds.
