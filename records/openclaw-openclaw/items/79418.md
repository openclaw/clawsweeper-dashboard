## What This Changes

The PR adds a 2 s minimum past-due heartbeat scheduler delay, fake-timer scheduler tests, a Discord voice test mock parameter change, and a changelog entry.

## Best Possible Solution

Repair stale heartbeat `nextDueMs` after terminal skipped or deferred scheduled outcomes so timer re-arms return to the configured cadence, keep focused regression tests, and require redacted real runtime proof of recovery.

## Reproduction Assessment

Yes for the scheduler defect by source inspection: current main can leave `nextDueMs` in the past after a disabled scheduled outcome and then re-arm with delay 0. The full Raspberry Pi Docker symptom was not live-reproduced.

## Solution Assessment

No. A timer floor reduces immediate event-loop pressure, but the narrower maintainable fix is to advance, clear, or recompute stale schedule state instead of polling the same past-due entry every 2 seconds.
