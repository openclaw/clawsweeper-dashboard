## Summary

Keep open. Source inspection supports the scheduler defect, but this PR converts the hot loop into repeated 2 second polling instead of repairing stale heartbeat schedule state, and the external PR still has mock-only proof plus a failing type-check CI shard.

## What This Changes

The PR adds a 2 second minimum heartbeat refire delay for past-due schedules, fake-timer scheduler tests, a Discord voice test mock signature change, and a changelog entry.

## Best Possible Solution

Repair stale heartbeat `nextDueMs` after terminal skipped or deferred scheduled outcomes so timer re-arms return to the configured cadence, keep focused regression tests, and require redacted real runtime proof of recovery.

## Reproduction Assessment

Yes for the scheduler defect by source inspection: current main can leave `nextDueMs` in the past after a disabled scheduled outcome and then re-arm with delay 0. The full Raspberry Pi Docker symptom was not live-reproduced in this read-only review.

## Solution Assessment

No. A timer floor reduces immediate event-loop pressure, but the narrower maintainable fix is to advance, clear, or recompute stale schedule state instead of polling the same past-due entry every 2 seconds.
