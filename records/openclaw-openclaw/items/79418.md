## Summary

Keep this PR open: it targets a plausible heartbeat scheduler regression, but the branch is not mergeable because real behavior proof is mock-only, CI lint is failing, and the timer-floor fix appears to throttle rather than clear the stale due schedule.

## What This Changes

The PR adds a 2 s minimum heartbeat scheduler refire gap when the next due time is in the past, adds fake-timer scheduler tests, and adds a changelog entry for the Raspberry Pi CPU spin regression.

## Best Possible Solution

Fix the stale heartbeat schedule at the source by advancing or otherwise clearing past-due nextDueMs for terminal skipped/deferred states, keep focused regression coverage, and require real after-fix runtime proof for the linked Raspberry Pi/Docker symptom before merge.

## Reproduction Assessment

Source inspection gives a high-confidence scheduler reproduction path: current main can leave nextDueMs in the past after a scheduled disabled skip and re-arm scheduleNext with rawDelay 0. The full Raspberry Pi Docker CPU symptom was not live-reproduced in this review.

## Solution Assessment

No. The PR's timer floor reduces setTimeout(0) pressure, but it leaves the stale schedule intact and can keep polling every 2 s; the narrower maintainable fix should update the heartbeat scheduling state that became stale.
