## Summary

Keep open: the linked scheduler bug is source-reproducible on current main, but this PR is not ready because it throttles stale refires to 2 seconds instead of repairing stale schedule state and it still lacks real runtime proof.

## What This Changes

The PR adds a 2 s minimum delay for past-due heartbeat scheduler timers, fake-timer scheduler coverage, a Discord voice test mock signature adjustment, and a changelog entry.

## Best Possible Solution

Repair stale heartbeat `nextDueMs` after terminal skipped or deferred scheduled outcomes so scheduler wakes return to the configured cadence, keep focused regression tests, and require redacted real runtime proof of recovery.

## Reproduction Assessment

Yes for the scheduler failure by source inspection: current main can leave `nextDueMs` in the past after a disabled scheduled outcome and then re-arm with delay 0. The full Raspberry Pi Docker CPU symptom was not live-reproduced in this read-only review.

## Solution Assessment

No. A timer floor reduces immediate event-loop pressure, but the narrower maintainable fix is to update or clear the stale schedule state instead of polling the same past-due entry every 2 seconds.
