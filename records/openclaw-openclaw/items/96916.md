Keep open: current main still lacks schema-aware parsing for MCP array/object parameters that arrive as JSON strings, and the focused prior fix PR was closed unmerged rather than becoming a canonical implementation.

## What This Changes

The issue requests that OpenClaw coerce stringified MCP array/object tool parameters back to their schema-expected JSON values before validation/execution.

## Best Possible Solution

Add bounded, schema-aware JSON parsing in shared tool-argument validation for string values only when an object or array schema branch is expected, with regression tests for nullable/union MCP schemas.

## Repair Work Prompt

Fix the MCP/tool-argument validation bug where providers emit schema-expected object or array parameters as JSON strings. Keep the change narrow: add bounded schema-aware coercion in the shared validation path so strings are parsed only when the active JSON schema branch expects `object` or `array`, including nullable/anyOf/oneOf cases, then validate the parsed value before accepting it. Cover MCP-style parameters such as `tags: "[\"test\",\"debug\"]"` and `params: "{}"`, preserve already-correct objects/arrays, reject malformed or wrong-shape strings, and avoid provider-specific or server-specific special cases. Do not edit CHANGELOG.md; put release-note context in the PR body if needed.
