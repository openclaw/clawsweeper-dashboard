## Summary

Keep open: current main and v2026.6.10 still leave schema-expected object/array arguments as strings during shared tool validation, while the related older fix attempts are closed unmerged or solve adjacent provider-wire-shape issues.

## What This Changes

The issue asks OpenClaw to coerce model-emitted JSON-stringified array/object MCP tool arguments back to their schema-expected values before validation.

## Best Possible Solution

Add narrow schema-aware JSON parsing in the shared tool-argument validator for string values when an object or array branch is expected, with regression coverage for nested, nullable, and union MCP schemas.

## Reproduction Assessment

Yes from source inspection, but not from an executed live repro: a string under an array/object schema is never JSON-parsed before validator failure.

## Solution Assessment

Yes. A shared validator fix is the best fit because the issue is not specific to one MCP server or one provider, and provider-only fixes would leave sibling tool paths drifting.
