## What This Changes

The issue reports MCP tool validation failures when providers emit array or object tool parameters as JSON strings instead of structured JSON values.

## Best Possible Solution

Review and land the focused shared llm-core validator fix, preferably the open mergeable candidate with bounded schema-gated parsing, then close the issue as implemented on main.

## Reproduction Assessment

Yes, by source inspection at medium confidence: current `validateToolArguments` only coerces scalar strings, so a JSON string for an array/object property remains a string and fails array/object validation. I did not run the reporter's live MiMo/Hindsight MCP setup.

## Solution Assessment
