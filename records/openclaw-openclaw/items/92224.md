## Summary

Keep open: current main and the latest release still compute model-list `available` without honoring the already-detected local loopback flag, and the active fix path is the open linked PR at https://github.com/openclaw/openclaw/pull/92363.

## What This Changes

The issue requests `openclaw models list --json` to report local Ollama loopback models as available when the service/model is running instead of showing `available: false`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92363 or an equivalent narrow fix that makes local loopback model rows available without changing remote auth-gated availability semantics.

## Reproduction Assessment

Yes, source-level: current main can produce `local: true` while `available` remains false because `toModelRow` ignores `local` when computing availability. I did not run a live Ollama daemon in this read-only review.

## Solution Assessment

Yes, the narrow maintainable fix is to make the existing local-base-url classifier participate in the row availability decision and add focused regression coverage; the open linked PR targets that spot.
