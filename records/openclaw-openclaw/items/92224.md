Keep open: current main and latest release still compute model-list availability from registry/auth state without treating a local Ollama base URL as available, and the related fix PR is open but unmerged. The issue should stay open until that PR lands or is replaced by an equivalent focused fix.

## What This Changes

The issue asks for `openclaw models list --json` to report reachable local Ollama model rows as available instead of marking them unavailable because API-key auth is absent.

## Best Possible Solution

Land or replace the focused open fix PR so local loopback Ollama rows are marked available, with regression coverage and ideally a redacted live `openclaw models list --json` proof against Ollama.

## Reproduction Assessment

Yes at source level: construct a loopback Ollama row with no matching `availableKeys` entry and no provider auth, and current main returns `local: true` while keeping `available: false`. I did not run a live Ollama daemon in this read-only pass.

## Solution Assessment

Yes: fixing the model-list row availability decision is the narrowest maintainable path because the command already computes `local` there and docs treat local Ollama hosts as not requiring a real bearer token. The linked PR still needs normal review/proof before this issue closes.
