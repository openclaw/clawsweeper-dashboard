## What This Changes

The issue reports that `openclaw models list --json` marks local Ollama models unavailable even though the local service and sessions work.

## Best Possible Solution

Keep the issue open until https://github.com/openclaw/openclaw/pull/92363 or a replacement lands with the row-renderer fix, regression coverage, and real Ollama CLI proof.

## Reproduction Assessment

Yes, by source inspection: a configured local row can have `local: true` while `available` stays false when registry availability or auth fallback does not include the model. I did not execute a live Ollama reproduction in this read-only pass.

## Solution Assessment

Yes, the best fix is a narrow row-availability change with regression tests and live CLI proof. The open PR targets the central expression, so the issue should stay open for that review path rather than spawning a competing fix.
