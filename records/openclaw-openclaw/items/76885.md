## Summary

This PR is useful but should stay open: the merge path fixes the non-empty state-dir `.env` case, but the early return still drops an operator-only existing `gateway.systemd.env` from the regenerated unit, and the user-facing fix needs a changelog entry.

## What This Changes

The PR changes systemd env-file staging to merge existing `gateway.systemd.env` entries with state-dir `.env` entries and adds a regression test for preserving provider keys during re-stage.

## Best Possible Solution

Keep the merge-based preservation, but make the env-file writer keep and reference an existing operator env file even when there are no incoming state-dir `.env` entries, then document the shipped fix in the changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: pre-create `gateway.systemd.env`, leave state-dir `.env` absent or empty, then re-stage; the proposed early return returns no environment file before reading the existing secrets.

## Solution Assessment

No. The proposed merge is the right direction, but the narrow maintainable fix also needs to handle the empty-incoming case and add the required changelog entry.
