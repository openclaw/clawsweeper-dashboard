## Summary

Keep open: the PR targets a real source-reproducible daemon bug, but the current merge preserves too many existing env-file keys and can keep stale secrets active over inline service environment values; it also needs the required changelog entry.

## What This Changes

The PR changes systemd service staging to read an existing `gateway.systemd.env`, merge it with state-dir `.env` variables, and adds a regression test for preserving operator API keys.

## Best Possible Solution

Preserve only operator-owned env-file keys while excluding keys managed by the current state-dir `.env` or inline service environment, add regression coverage for stale inline override keys, and include a changelog entry.

## Reproduction Assessment

Yes. Source inspection shows current main rewrites `gateway.systemd.env` from state-dir `.env`, and the linked report gives concrete upgrade/doctor steps; I did not run tests during this read-only review.

## Solution Assessment

No. Merging every existing key is too broad because existing tests model EnvironmentFile values overriding inline environment values; the narrower fix should filter preserved entries to operator-owned keys.
