## Summary

The PR is still useful but should not merge yet: the follow-up fixes the stale inline-token case, but the current diff can still fail to preserve file-only operator secrets through the normal doctor/install planning path and misses the empty state-dir `.env` case.

## What This Changes

The PR reads an existing `gateway.systemd.env`, merges it with state-dir `.env` values, filters keys passed as inline-managed, adds systemd regression tests, and adds a changelog entry for #76860.

## Best Possible Solution

Preserve file-owned operator secrets in `gateway.systemd.env` across re-stage, clear stale inline-managed or tracked managed keys, and cover the doctor/install planning path plus the empty incoming `.env` edge case.

## Reproduction Assessment

Yes. Source inspection shows current main rewrites `gateway.systemd.env` from state-dir `.env`, and the PR’s remaining edge cases are traceable through `readSystemdServiceExecStart` -> `buildGatewayInstallPlan` -> `writeSystemdUnit`; I did not run live upgrade steps in this read-only pass.

## Solution Assessment

No. The follow-up is directionally right, but using all planned environment keys as inline-managed is too broad and the merge still returns before reading an existing env file when incoming `.env` entries are empty.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve file-only secrets through the install plan:** `src/daemon/systemd.ts:537`
  - body: Passing every planned `environment` key as `inlineManagedKeys` treats operator secrets read from the existing EnvironmentFile as inline-managed. In the normal doctor/install path those safe file values are preserved into the plan, then this line causes `writeSystemdGatewayEnvironmentFile` to remove them from `gateway.systemd.env` and `environmentSansDotEnvEntries` can emit them inline in the unit, so the reported env-file preservation still fails for that path.
  - confidence: 0.9
- **[P2] Handle existing env files when incoming .env is empty:** `src/daemon/systemd.ts:568-570`
  - body: The new preservation logic is still behind `entries.length === 0`. If an existing `gateway.systemd.env` contains only operator-added keys and the state-dir `.env` is empty or fully filtered, re-stage returns no EnvironmentFile and the existing secrets are no longer loaded. Read and merge the existing file before deciding whether there are entries to write/return.
  - confidence: 0.84

## Security Review

Status: needs_attention
