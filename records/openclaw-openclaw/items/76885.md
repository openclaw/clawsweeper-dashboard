## Summary

Keep open: the PR fixes the destructive overwrite direction, but it still allows operator-owned env-file secrets to be serialized into inline systemd `Environment=` entries through the doctor/install re-stage path.

## What This Changes

The PR merges existing `gateway.systemd.env` entries with state-dir `.env` variables during systemd staging, clears marked OpenClaw-managed stale file keys, adds regression tests, and records a changelog fix for #76860.

## Best Possible Solution

Preserve operator-owned env-file keys across re-stage while carrying file ownership through the install/doctor path so those keys are excluded from inline `Environment=` output, with regression coverage for the full planning path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: read a unit whose `EnvironmentFile` contains `OPENROUTER_API_KEY`, feed the resulting `command.environment` through `buildGatewayInstallPlan`, then stage the PR-head unit; the key is both preserved in the env file and kept inline because filtering only checks state-dir `.env` keys.

## Solution Assessment

No. Merging the env file is the right direction, but the maintainable fix must also preserve source ownership or otherwise filter all retained env-file keys out of inline systemd rendering.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep preserved env-file secrets out of Environment=:** `src/daemon/systemd.ts:535-539`
  - body: When doctor re-stages from an existing systemd unit, `readSystemdServiceExecStart` merges `EnvironmentFile` values into `command.environment`, and `buildGatewayInstallPlan` preserves non-OpenClaw keys such as provider API keys. This PR writes those keys back to `gateway.systemd.env`, but `environmentSansDotEnvEntries` still only removes state-dir `.env` keys, so a provider secret retained from the env file can also be emitted as an inline `Environment=` value in the unit. Please carry file-source ownership into staging, or filter inline env against the keys retained in `gateway.systemd.env`.
  - confidence: 0.9
