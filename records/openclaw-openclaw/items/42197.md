## What This Changes

The PR adds native title tooltips to Usage-panel skill/tool/file names and ellipsis styling for long context breakdown labels.

## Best Possible Solution

Port the intended behavior onto current main by adding the title attributes in `usage-render-details.ts` and a scoped `.context-breakdown-item .mono` ellipsis rule in `ui/src/styles/usage.css`.

## Reproduction Assessment

Yes. Source inspection shows current main still renders long context names as unbounded `.mono` spans without `title` attributes or a scoped truncation rule, and the PR screenshots show the before overflow and after truncation behavior.

## Solution Assessment

Yes in direction, but not as the branch currently stands. Native title attributes plus a scoped ellipsis rule are the narrow maintainable fix, but the CSS hunk needs to move from the deleted `usage-styles` path into `ui/src/styles/usage.css`.
