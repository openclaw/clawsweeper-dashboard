The PR adds native title tooltips to Usage context breakdown skill/tool/file names and attempts scoped ellipsis styling for long labels.

## Best Possible Solution

Land a rebased UI-only patch that keeps the title attributes and puts scoped `.context-breakdown-item .mono` truncation in `ui/src/styles/usage.css`, with focused render/style coverage and a visual check.

Yes. Current main source renders long Usage breakdown names in flex rows without truncation or full-name title attributes, and the PR screenshots show the before overflow and after truncation path; I did not run a browser, so this is source-reproducible rather than reproduced.

## Solution Assessment

No, not as currently patched. The title attributes are appropriate, but the CSS must move to the active `ui/src/styles/usage.css` surface instead of the removed `usage-styles` module.
