## Summary

Keep open: this is a useful, narrow Usage-panel UI fix, but the branch is not correct against current main because the truncation CSS is still on the removed `usage-styles` path while current main uses `ui/src/styles/usage.css`.

## What This Changes

Adds native title attributes to Usage context breakdown skill/tool/file labels and CSS ellipsis styling for long names.

## Best Possible Solution

Port the title attributes to current `usage-render-details.ts`, add the scoped `.context-breakdown-item .mono` ellipsis rule in `ui/src/styles/usage.css`, and verify the Usage panel visually.

## Reproduction Assessment

Yes. Source inspection shows current main renders long context names as unbounded `.mono` spans, and the PR screenshots show the overflow before and truncation/tooltip after; I did not run a live UI session in this read-only review.

## Solution Assessment

Yes in direction, but not in the branch’s current form. Native `title` attributes plus a scoped context-breakdown ellipsis selector are the narrow fix; the CSS needs to move to the consolidated stylesheet.
