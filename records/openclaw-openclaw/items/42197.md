## Summary

Keep open: the Usage-panel overflow fix is still useful, but this branch is stale because current main moved Usage styling into `ui/src/styles/usage.css` while the PR still edits the removed `usage-styles` file.

## What This Changes

Adds native title tooltips and scoped ellipsis styling for long skill, tool, and file names in the Usage panel context breakdown.

## Best Possible Solution

Port the title attributes to `usage-render-details.ts` and add a scoped `.context-breakdown-item .mono` ellipsis rule in `ui/src/styles/usage.css`, then validate the Usage panel visually.

## Reproduction Assessment

Yes. Current main still renders long context names as unbounded `.mono` spans with no title attributes or scoped truncation rule, and the PR screenshots show the before overflow and after truncation behavior.

## Solution Assessment

Yes in direction, but not as this branch currently stands. Native `title` attributes plus a scoped ellipsis rule are the narrow fix, but the CSS needs to move from the removed `usage-styles` path into `ui/src/styles/usage.css`.
