## Summary

Keep open: the Usage panel overflow fix is useful and still missing on current main, but this stale branch puts the CSS in a deleted style module, so it needs a narrow retarget before merge.

## What This Changes

Adds native title tooltips to Usage context breakdown skill/tool/file labels and scoped ellipsis styling for those `.mono` names.

## Best Possible Solution

Retarget the valid UI behavior to current main by keeping the title attributes in the active renderer and adding the scoped `.context-breakdown-item .mono` truncation rule to `ui/src/styles/usage.css`.

## Reproduction Assessment

Yes. Source inspection shows current main renders long Usage context labels as unbounded bare `.mono` spans with no scoped ellipsis rule, and the PR screenshots show the before/after overflow behavior.

## Solution Assessment

No. The desired behavior is correct, but this exact patch is no longer the best implementation because the CSS belongs in `ui/src/styles/usage.css`, not the deleted `usage-styles` module.
