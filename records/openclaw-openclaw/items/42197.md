Adds native title tooltips and ellipsis styling for long skill, tool, and file names in the Usage panel context breakdown.

## Best Possible Solution

Port the same behavior onto current main: add the name tooltips in `usage-render-details.ts`, add scoped context-breakdown ellipsis styling in `usage.css`, and validate with a UI build plus visual Usage-panel proof.

## Reproduction Assessment

Yes. Current main renders long context names as unbounded `.mono` spans and has no scoped truncation rule, while the PR screenshots show the overflow before and truncation/tooltip after; I did not run a live UI session.

## Solution Assessment

Yes in direction, but not in this branch's current form. Native `title` attributes plus a scoped `.context-breakdown-item .mono` ellipsis selector are the narrow maintainable fix, and the CSS needs to move to `ui/src/styles/usage.css`.
