## Best Possible Solution

Land a rebased UI-only patch that keeps the title attributes and puts scoped `.context-breakdown-item .mono` truncation in `ui/src/styles/usage.css`, with focused render/style coverage and a visual check.

Yes. A Usage context breakdown with a long skill, tool, or file name reproduces the visible overflow; the PR screenshots show it and current main still lacks the tooltip attributes and truncation CSS.

No, not as currently patched. The tooltip attributes are appropriate, but the CSS must target the active `ui/src/styles/usage.css` surface instead of the removed `usage-styles` module.
