## Summary

Keep open: the visible Usage overflow is real and the tooltip/truncation behavior is useful, but this branch is stale against the current Usage stylesheet consolidation and would put the CSS in a deleted style module.

## What This Changes

The PR adds native title tooltips to Usage context breakdown skill/tool/file labels and CSS ellipsis truncation for those labels.

## Best Possible Solution

Retarget the same behavior to the current Usage renderer and `ui/src/styles/usage.css`, preserving the tooltips and adding a scoped ellipsis rule without restoring deleted style modules.

## Reproduction Assessment

Yes. Source inspection on current main shows the long labels are unbounded bare `.mono` spans with no active scoped truncation rule, and the PR screenshots show the before/after overflow behavior.

## Solution Assessment

No. The intended behavior is the right fix, but this patch is no longer the best implementation because the CSS hunk belongs in `ui/src/styles/usage.css`, not the deleted `usage-styles` module.
