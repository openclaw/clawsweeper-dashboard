## Summary

Keep open: the UI overflow fix is still useful, but this PR is not mergeable as-is because its CSS change targets a deleted Usage style module while current main loads the consolidated Usage stylesheet.

## What This Changes

The PR adds native title tooltips and ellipsis styling for long skill/tool/file names in the Usage context breakdown.

## Best Possible Solution

Keep the tooltip direction, add the scoped `.context-breakdown-item .mono` truncation rule to `ui/src/styles/usage.css`, and validate the Usage context breakdown visually with a long name.

## Reproduction Assessment

Yes, source-reproducible: current main renders the breakdown names as bare `.mono` spans and the active stylesheet lacks a scoped truncation rule. The PR screenshots also show the visible before/after overflow behavior.

## Solution Assessment

No, not as-is. The tooltip and truncation approach is appropriate, but the CSS must move to the active `ui/src/styles/usage.css` file instead of the deleted `usage-styles` module.
