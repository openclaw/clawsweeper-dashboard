## Summary

Keep this PR open: the visible Usage panel overflow fix is still useful on current main, but the branch is stale because its CSS lands in the removed `usage-styles` module instead of the active consolidated Usage stylesheet.

## What This Changes

The PR adds `title` tooltips to Usage context breakdown skill/tool/file labels and an ellipsis rule for long monospace names.

## Best Possible Solution

Keep the tooltip behavior, move the scoped truncation rule into `ui/src/styles/usage.css`, and preserve the consolidated Usage stylesheet introduced by the merged refactor.

## Reproduction Assessment

Yes. A Usage context breakdown containing a long skill, tool, or file name demonstrates the overflow, and current-main source still lacks both the label tooltips and scoped truncation rule while the PR screenshots show the before/after behavior.

## Solution Assessment

No, not as-is. The tooltip and scoped truncation approach is appropriate, but the CSS must be applied to the active `ui/src/styles/usage.css` file instead of the removed `usage-styles` module.
