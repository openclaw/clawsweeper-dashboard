## Summary

Keep open: the PR addresses a real Usage UI overflow, but it is not mergeable as-is because the CSS change targets a deleted Usage style module while current main uses `ui/src/styles/usage.css`.

## What This Changes

The PR adds title tooltips to Usage context breakdown skill/tool/file labels and truncates long labels with ellipsis styling.

## Best Possible Solution

Keep the UI improvement, but rebase it onto current main by adding title attributes in the renderer and the scoped ellipsis rule in `ui/src/styles/usage.css`, without restoring the old `usage-styles` modules.

## Reproduction Assessment

Yes, source-reproducible: current main renders long skill/tool/file labels as bare `.mono` spans and the active Usage stylesheet lacks the scoped ellipsis rule. The PR screenshots also show the before overflow and after truncation behavior.

## Solution Assessment

No, not as-is. The tooltip and scoped truncation direction is appropriate, but the CSS must move to the active `ui/src/styles/usage.css` file rather than the deleted `usage-styles` module.
