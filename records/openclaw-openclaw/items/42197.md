## Summary

Keep open. The Usage panel overflow problem is still present on current main and the PR body gives direct visual proof, but the branch needs a narrow repair because its CSS change targets an old Usage style module that current main removed.

## What This Changes

The PR adds native title tooltips to Usage panel context-breakdown names and ellipsis styling for long skill, tool, and file labels.

## Best Possible Solution

Apply the tooltip attributes in the current Usage detail renderer, move the scoped ellipsis rule into `ui/src/styles/usage.css`, and avoid restoring the deleted usage style modules.

## Reproduction Assessment

## Solution Assessment

No. The PR intent is the right narrow UI fix, but the CSS target is stale; the maintainable solution is to put the scoped rule in the active `ui/src/styles/usage.css` stylesheet.
