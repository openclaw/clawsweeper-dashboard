## Summary

Keep open: the Usage UI overflow is real and the tooltip/truncation approach is useful, but this branch still puts the CSS in a stylesheet tree that current main deleted, so the visual fix needs a small repair before merge.

## What This Changes

The PR adds title tooltips to skill/tool/file names in the Usage context breakdown and adds scoped ellipsis styling for long names.

## Best Possible Solution

Port the tooltip attributes to the current Usage renderer and add the scoped ellipsis rule to `ui/src/styles/usage.css`, with focused render/style validation and one visual check using a long skill, tool, or file name.

## Reproduction Assessment

Yes. The PR screenshots show the long-name overflow and after-fix truncation/tooltip, and current main still lacks the renderer attributes and active stylesheet rule needed to prevent it.

## Solution Assessment

No, not as currently patched. The tooltip attributes are appropriate, but the CSS must move from the deleted `ui/src/ui/views/usage-styles/usageStyles-part3.ts` path into the active `ui/src/styles/usage.css` stylesheet.
