## Summary

Keep open: the visible Usage-panel overflow fix is valid and still missing on current main, but this branch cannot land as-is because its CSS hunk targets a removed stylesheet. A narrow repair should keep the label tooltips and move the scoped truncation rule into the active Usage stylesheet.

## What This Changes

This PR adds native `title` tooltips to Usage-panel skill/tool/file context labels and adds scoped ellipsis styling for long context breakdown names.

## Best Possible Solution

Port the intended behavior onto current main by adding the three `title` attributes in `usage-render-details.ts` and a scoped `.context-breakdown-item .mono` ellipsis rule in `ui/src/styles/usage.css`.

## Reproduction Assessment

Yes. Source inspection shows current main still renders long context names as unbounded `.mono` spans without `title` attributes or a scoped truncation rule, and the PR screenshots show the before overflow and after truncation/tooltip behavior.

## Solution Assessment

Yes in direction, but not as the branch currently stands. Native title attributes plus a scoped ellipsis rule are the narrow maintainable fix, but the CSS hunk must move from the deleted `usage-styles` path into `ui/src/styles/usage.css`.
