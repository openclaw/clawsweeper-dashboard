## What This Changes

Adds native title tooltips to usage context breakdown skill/tool/file labels and scoped ellipsis styling for those labels in the old split usage stylesheet.

## Best Possible Solution

Refresh the PR to keep the title attributes, add scoped `.context-breakdown-item .mono` truncation in `ui/src/styles/usage.css`, and cover the render/style behavior with focused tests plus UI proof.

## Reproduction Assessment

Yes. Current main renders long skill/tool/file names as unbounded `.mono` spans in flex rows, and the PR's before screenshot shows the resulting usage-panel overflow.

## Solution Assessment

Mostly yes. Scoped ellipsis plus native `title` is a narrow maintainable fix, but the CSS must move to the current stylesheet before the patch actually solves the current-main UI.
