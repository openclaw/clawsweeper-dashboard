## Summary

Keep open: current main still has the strict-agentic planning-only retry gap and this PR is the focused canonical fix path, but the current PR head still has a short-answer false positive in active-narration matching that should block merge.

## What This Changes

The branch broadens strict-agentic actionable-prompt and active-narration planning-only retry detection in `incomplete-turn` and adds focused regression tests.

## Best Possible Solution

Keep the helper-based fix, but make bare acknowledgement phrases standalone or require explicit ongoing-action wording, then add negative tests for short completed answers before landing.

## Reproduction Assessment

Yes, source-level. Current main rejects the reported imperative prompt and lacks an active-narration branch, and the PR head false-positive is shown directly by the changed regex and completion guard behavior.

## Solution Assessment

No, not quite. The central helper is the right place to fix the bug, but the active-narration regex must distinguish standalone acknowledgements or ongoing-action claims from short completed answers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Constrain acknowledgement-only active narration:** `src/agents/pi-embedded-runner/run/incomplete-turn.ts:118`
  - body: `PLANNING_ONLY_ACTIVE_NARRATION_RE` still accepts `Got it` or `On it` when they are only the prefix of a short completed answer, and the resolver then lets `isActiveNarration` bypass the action-verb gate. In a strict-agentic turn after `go ahead`, a valid answer like `Got it. The answer is 42.` would be retried and could exhaust into the blocked state instead of being delivered. Please make bare acknowledgements standalone or pair them with explicit ongoing-action wording, with a negative test for short completed answers.
  - confidence: 0.88
