## Summary

Keep open: this PR targets a real strict-agentic retry bug that current main still has, but the latest diff introduces a false-positive path where negated prompts such as `do not run anything` are treated as actionable and can force a retry against explicit user intent.

## What This Changes

The PR broadens strict-agentic planning-only retry detection for common imperative prompts and active narration, updates retry wording, and adds focused regression tests.

## Best Possible Solution

Keep the focused strict-agentic fix, but add a negation guard and regression coverage so broadened imperative detection does not retry when the user explicitly says not to act.

## Reproduction Assessment

Yes. Current main source still misses the reported imperative and active-narration forms, and the PR body includes after-fix terminal proof from a built bundle; the new negated-`do` false positive is also source-reproducible from the PR-head regex.

## Solution Assessment

No, not quite. Updating the central incomplete-turn helper with colocated tests is the right shape, but the directive regex needs to exclude negated `do not` / `please do not` prompts before this is the best maintainable fix.
