## Summary

Keep this PR open: the strict-agentic bug is real and the branch is aimed at the right helper, but the current active-narration regex still creates a short-answer false positive that should block merge.

## What This Changes

The branch broadens strict-agentic actionable-prompt and planning-only narration detection in `src/agents/pi-embedded-runner/run/incomplete-turn.ts` and adds focused regression tests in `src/agents/pi-embedded-runner/run.incomplete-turn.test.ts`.

## Best Possible Solution

Keep the helper-based fix, but make bare `Got it` and `On it` retry only when they are standalone acknowledgements or paired with explicit ongoing-action wording, then add short-answer negative tests before landing.

## Reproduction Assessment

Yes, source-level. Current main still misses the reported imperative and active-narration gates, and a read-only regex smoke shows the current PR head's short-answer false positive.

## Solution Assessment

No, not quite. The central helper is the right place to solve the reported bug, but the active-narration matcher needs one more narrowing pass so completed short answers are not retried as planning-only turns.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Constrain acknowledgement-only active narration:** `src/agents/pi-embedded-runner/run/incomplete-turn.ts:118`
  - body: `PLANNING_ONLY_ACTIVE_NARRATION_RE` still treats `Got it` and `On it` as a valid prefix, and the resolver lets `isActiveNarration` bypass the action-verb gate. In a strict-agentic turn after `go ahead`, a valid short answer like `Got it. The answer is 42.` would be retried and could exhaust into the blocked state instead of being delivered. Please require those acknowledgement forms to be standalone or pair them with explicit ongoing-action wording, with a short-answer negative test.
  - confidence: 0.88
