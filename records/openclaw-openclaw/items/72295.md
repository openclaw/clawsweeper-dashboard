## Summary

Keep open. The PR addresses a real strict-agentic planning-only retry gap that is still present on current main, but the current head still introduces a false-positive active-narration path for short direct answers beginning with acknowledgements like "Got it" or "On it", so it needs a narrow regex/test repair before merge.

## What This Changes

The branch broadens strict-agentic actionable-prompt and planning-only narration detection in `src/agents/pi-embedded-runner/run/incomplete-turn.ts` and adds focused helper regression tests.

## Best Possible Solution

Keep the helper-based fix, but require bare acknowledgement phrases to be standalone or pair them with explicit ongoing-action wording, then add negative tests for short completed answers before landing.

## Reproduction Assessment

Yes, source-level. Current main rejects the reported imperative prompt and lacks an active-narration branch, and the PR head false-positive can be shown directly from the changed regex and resolver gates.

## Solution Assessment

No, not quite. The central helper is the right place to fix this, but the active-narration regex still needs to distinguish standalone acknowledgements or ongoing-action claims from short completed answers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Constrain acknowledgement-only active narration:** `src/agents/pi-embedded-runner/run/incomplete-turn.ts:118`
  - body: `PLANNING_ONLY_ACTIVE_NARRATION_RE` still matches short completed answers that start with `Got it` or `On it`, and the resolver then lets `isActiveNarration` bypass the action-verb gate. In a strict-agentic turn after `go ahead`, a valid answer like `Got it. The answer is 42.` would be retried and could exhaust into the blocked state instead of being delivered. Please make bare acknowledgements standalone or require an explicit ongoing-action phrase, with a negative test for short completed answers.
