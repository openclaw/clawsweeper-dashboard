## Summary

This PR should stay open because it addresses a real strict-agentic planning-only retry bug, but the current patch still introduces a retry false positive for short valid answers that begin with bare acknowledgements like `Got it` or `On it`.

## What This Changes

The PR broadens strict-agentic planning-only retry heuristics in `incomplete-turn.ts` and adds helper regression tests for actionable prompts and active narration.

## Best Possible Solution

Land the helper-level strict-agentic fix after constraining bare acknowledgement matches to standalone narration or explicit ongoing-action wording, with negative regression coverage for short completed answers.

## Reproduction Assessment

Yes, source-level. Current main still lacks the reported actionable verbs and active-narration detector, and a PR-head regex smoke reproduces the remaining false-positive path for short valid answers.

## Solution Assessment

No, not quite. The central helper is the right place to solve the bug, but the active-narration matcher should not treat `Got it` or `On it` followed by concise answer prose as planning-only work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

- **[P2] Constrain bare acknowledgements to standalone narration:** `src/agents/pi-embedded-runner/run/incomplete-turn.ts:118`
