## Summary

Keep open: current main still has the reported strict-agentic retry gaps, but the PR head introduces a false positive where a short completed text answer beginning with an acknowledgment can be forced into another retry.

## What This Changes

The PR broadens strict-agentic planning-only retry detection for more actionable user prompts and active narration replies, and adds regression coverage.

## Best Possible Solution

Keep the central helper/test approach, but narrow active-narration detection so bare acknowledgments only retry when they stand alone or are followed by ongoing-action wording, with regression coverage for short completed answers.

## Reproduction Assessment

Yes. Current main source shows the reported imperative prompt and present-continuous narration paths fall through, and PR-head source inspection plus a regex smoke gives a high-confidence reproduction of the new false-positive path.

## Solution Assessment

No, not as-is. Updating the central incomplete-turn helper is the right fix shape, but the current bare `got it` active-narration match needs tightening before merge.

## Review Findings

- **[P2] Require bare acknowledgments to stand alone:** `src/agents/pi-embedded-runner/run/incomplete-turn.ts:118`
