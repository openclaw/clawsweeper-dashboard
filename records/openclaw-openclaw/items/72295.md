## Summary

Keep open. The linked bug is still present on current main and this PR is the canonical focused fix path, but the latest head still introduces a false-positive active-narration path that can retry legitimate short answers beginning with bare acknowledgement text.

## What This Changes

The PR broadens strict-agentic planning-only retry heuristics in the embedded runner and adds regression tests for broader actionable prompts, active narration, and negative prompt cases.

## Best Possible Solution

Land the helper-level strict-agentic fix after narrowing active narration so bare acknowledgements only retry when they are standalone or clearly assert ongoing action.

## Reproduction Assessment

Yes, source-level. Current main's actionable prompt regexes reject the reported imperative prompt, and the planning-only resolver lacks an active-narration path for short replies like "Great. I'm running it now.".

## Solution Assessment

No, not yet. The helper and test surface are the right place to fix this, but the active-narration matcher should be narrowed so short completed answers are not retried as planning-only turns.

- **[P2] Require standalone acknowledgements for active narration:** `src/agents/pi-embedded-runner/run/incomplete-turn.ts:118`
