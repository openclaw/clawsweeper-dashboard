## Summary

Keep open: this is a focused fix for a real current-main strict-agentic retry gap, but the latest PR head still overmatches bare acknowledgement text and can retry valid short answers.

## What This Changes

The PR broadens strict-agentic planning-only retry heuristics in `incomplete-turn.ts` and adds helper regression tests for actionable prompts and active narration.

## Best Possible Solution

Land the helper-based fix after constraining bare acknowledgement matches to standalone acknowledgements or explicit ongoing-action wording, with negative regression coverage for short completed answers.

## Reproduction Assessment

Yes, source-level. Current main's allowlists omit the reported imperative verbs and lacks active narration, and a read-only regex smoke against the PR head shows the remaining false-positive path.

## Solution Assessment

No, not quite. The central helper is the right place to fix this, but the active-narration regex should distinguish standalone acknowledgements or ongoing-action claims from short completed answers.
