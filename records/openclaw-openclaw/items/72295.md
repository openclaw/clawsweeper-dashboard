## Summary

Keep open: the underlying strict-agentic retry gap is valid, and the PR is focused, but the current patch still retries concise valid answers that start with bare acknowledgements such as "Got it." or "On it.".

## What This Changes

The branch broadens strict-agentic planning-only retry detection for missing actionable verbs and present-continuous/ack-style active narration, with helper regression tests.

## Best Possible Solution

Land the focused strict-agentic helper fix after narrowing bare `Got it`/`On it` matches to standalone acknowledgements or explicit ongoing-action wording, with negative regression coverage for short answers.

## Reproduction Assessment

Yes, source-level. Current main still lacks the reported verb and active-narration retry paths, and a PR-head regex smoke reproduces the remaining false-positive path for short valid answers.

## Solution Assessment

No, not quite. The helper-level approach is the right place to fix the bug, but bare acknowledgement matching needs to be constrained so valid concise answers are not retried.
