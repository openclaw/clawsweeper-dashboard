## Summary

Keep open: the reported strict-agentic gap is valid and the PR fixes the central path, but the new active-narration matcher is still too broad and can retry short completed answers that start with acknowledgements like "Got it."

## What This Changes

The PR broadens strict-agentic planning-only retry detection in `src/agents/pi-embedded-runner/run/incomplete-turn.ts`, tightens the retry instruction, and adds focused helper regression tests.

## Best Possible Solution

Keep the helper-based fix, but narrow active narration to standalone acknowledgements or explicit ongoing-action phrases and add negative tests for short completed answers before landing.

## Reproduction Assessment

Yes, source-level. Current main's prompt regexes reject the reported imperative prompt, and the planning-only resolver lacks an active-narration path for short replies like "Great. I'm running it now.".

## Solution Assessment

No, not quite. The centralized helper is the right place to fix this, but the current PR also treats short completed answers beginning with "Got it" or some "I'm <verb>ing" forms as active narration and should be narrowed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Constrain active narration acknowledgements:** `src/agents/pi-embedded-runner/run/incomplete-turn.ts:118`
  - body: `PLANNING_ONLY_ACTIVE_NARRATION_RE` matches any short reply starting with `Got it`, and the resolver then bypasses the action-verb gate. In a strict-agentic GPT-5 turn, a valid short answer like `Got it. The answer is 42.` would be retried and could exhaust into the blocked state instead of being delivered. Please make bare acknowledgements standalone or add a stronger ongoing-action requirement, with a negative regression test for short completed answers.
