If maintainers accept the direction, implement a method-aware WhatsApp login contract owned by the WhatsApp plugin, with phone validation, Baileys readiness timing, QR backward compatibility, gateway/CLI/agent/setup/docs updates, regression tests, and live proof.

## Solution Assessment

Unclear until maintainers decide the product shape. The Baileys primitive is real, but the maintainable solution should be method-aware across plugin auth, CLI, gateway protocol, setup, agent tool, UI/status copy, and docs rather than a narrow local shim.
