## Best Possible Solution

If maintainers accept the product direction, implement a method-aware WhatsApp login contract in the WhatsApp plugin-owned CLI, gateway/UI, agent tool, setup, status-copy, docs, and tests while keeping QR linking backward-compatible.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers decide the product shape. The requested Baileys primitive is real, but the maintainable solution should be a method-aware login contract across plugin, CLI, gateway/UI, agent tool, setup, status copy, and docs rather than a narrow local shim.
