Keep open. Current main and the latest shipped tag are still QR-only for WhatsApp device linking, while the prior pairing-code implementation PRs are closed and unmerged; this is a concrete feature request that changes auth, CLI, gateway, setup, and agent-tool surface, so it needs maintainer product/security review before implementation.

## What This Changes

Request WhatsApp login via Baileys phone-pairing code as an alternative to the existing QR linking flow.

## Best Possible Solution

If maintainers accept the product direction, add a method-aware WhatsApp device-link login contract owned by the WhatsApp plugin, with phone validation, Baileys timing guards, method-aware CLI/gateway/agent/setup/status copy, regression tests, and live pairing proof.

## Solution Assessment
