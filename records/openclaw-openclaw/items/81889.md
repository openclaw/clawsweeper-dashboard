If maintainers accept the direction, implement method-aware WhatsApp device-link login in the official WhatsApp plugin surfaces with phone validation, Baileys event timing, QR backward compatibility, tests, docs, and live proof.

## Solution Assessment

Unclear until maintainers decide the product shape. The Baileys primitive is real, but the maintainable fix should be a cohesive method-aware login contract across plugin, CLI, gateway/UI, agent tool, setup, and status copy rather than a narrow local shim.
