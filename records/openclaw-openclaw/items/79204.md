## Summary

Keep open. This is a valid security fix area, but the current PR still accepts implicit default-admin semantics through the helper and only supplies mocked unit-test proof; the protected maintainer label also requires human handling.

## What This Changes

The PR replaces managed outgoing image requester-session ownership checks with OpenAI-compatible sender-owner auth, updates managed-image tests, and adds a Gateway/media changelog entry.

## Best Possible Solution

Use a media-specific owner/admin predicate that rejects requester-session self-declaration and implicit default-admin fallback, then prove the route with a real gateway media fetch.

## Reproduction Assessment

Yes. Source inspection on current main shows a managed image GET with trusted-scope or device-token auth reaches byte serving without requester-session ownership because privilegedAccess skips that branch.

## Solution Assessment

No. The PR removes the client-declared requester-session check, but the replacement helper still grants owner semantics through the documented no-scope default-admin fallback.
