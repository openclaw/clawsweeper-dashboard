## Summary

Keep open: this external PR is still useful to review as a proposed diagnostics/API change, but the current head is syntactically broken, lacks after-fix real behavior proof, and raises a security-sensitive contract question by exposing differentiated revoke failures.

## What This Changes

The PR changes device-pairing token helpers, gateway device-token handlers, websocket connect token issuance, and related tests to return structured error objects instead of existing null or ok/reason denial shapes.

## Best Possible Solution

Keep public denial contracts stable for now; if richer diagnostics are desired, add non-breaking internal log context or design a versioned device-pairing error API with explicit security review.
