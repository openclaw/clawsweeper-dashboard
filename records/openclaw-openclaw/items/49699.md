## Summary

Keep open: the PR is not merge-ready because the head diff still contains malformed TypeScript, changes stable device-token result contracts, and exposes differentiated revoke failures where current main keeps public denial messages generic.

## What This Changes

This PR changes device-pairing token helpers, gateway handlers, and related tests to return and handle structured error objects for scope-escalation and token errors.

## Best Possible Solution

Preserve the stable ok/reason and token-or-null helper contracts unless maintainers approve a versioned API change; add structured diagnostic context compatibly in server-side logs and keep public gateway denials generic.

## Reproduction Assessment

Yes. The PR diff itself is a high-confidence source reproduction path for the parse blockers, and current main source/tests show the contract baseline the PR changes.

## Solution Assessment

No. Structured diagnostics may be useful, but replacing the helper discriminators and exposing differentiated public revoke errors is not the narrowest maintainable or safest solution.
