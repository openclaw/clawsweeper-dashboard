## Summary

Keep this PR open, but do not merge it as-is. Current main still uses the existing device-token contracts, and the PR branch has parse-breaking TypeScript plus gateway authz contract and information-disclosure regressions that need maintainer review.

## What This Changes

The PR changes device-pairing token helpers, gateway handlers, WebSocket handshake code, and tests to return and handle structured error objects instead of null or ok/reason-only denial results.

## Best Possible Solution

Preserve the existing ok/reason discriminators or add structured fields compatibly, keep public rotate/revoke denial responses generic, log structured scope details server-side, and update focused infra/gateway tests for scope escalation and token denial paths.

## Reproduction Assessment

Yes. The PR diff itself is a high-confidence reproduction path because both changed gateway files contain invalid TypeScript syntax; current main source and tests provide the legacy contract baseline.

## Solution Assessment

No. Structured server-side context is useful, but replacing stable discriminators and exposing differentiated revoke messages is not the safest path; a compatible result shape plus generic public errors is narrower and more maintainable.
