## Summary

Keep open: current main does not implement this PR’s structured-error contract, and the proposed patch is not mergeable as-is because it introduces syntax errors, breaks existing token result discriminators, and exposes more detailed gateway revoke denial state to callers.

## What This Changes

The PR changes device-pairing token helpers, gateway RPC/WebSocket handlers, and tests to return and handle structured error objects instead of null or ok/reason-only denial results.

## Best Possible Solution

Keep the existing public denial semantics unless maintainers approve a versioned contract change; if structured context is desired, add it compatibly alongside ok/reason results, keep public RPC errors generic, log detail server-side, and update focused gateway/device-pairing tests plus changelog.

## Reproduction Assessment

Yes. The PR diff itself is a high-confidence source reproduction path because the changed gateway files contain malformed TypeScript, and current main plus tests show the legacy contract baseline it changes.

## Solution Assessment

No. Structured server-side context may be useful, but replacing stable ok/reason/null contracts and exposing differentiated revoke errors is not the narrowest safe path; a compatible result shape with generic public denials is safer.

  - confidence: 0.99
