## Summary

Keep open: this external PR still has source-reproducible parse blockers, changes stable device-token helper contracts, exposes more specific revoke failure messages to callers, and lacks after-fix real behavior proof.

## What This Changes

This PR changes device-pairing token helpers, gateway token handlers, websocket handshake token issuance, and related tests to return structured error objects instead of existing null or ok/reason denial results.

## Best Possible Solution

Keep the current public denial contracts; if richer diagnostics are desired, add non-breaking internal log context or design a versioned device-pairing API with explicit maintainer and security approval.

## Reproduction Assessment

Yes. Source inspection of the PR diff at head `1ce8933e7a00eb08fe5df74167c6359551adae6f` is enough to reproduce the parse blockers, and current main source/tests show the stable contracts the PR changes.

## Solution Assessment

No. Replacing nullable and ok/reason contracts plus exposing differentiated revoke failures is not the narrowest maintainable solution without a versioned API and security decision; internal logging would be the safer narrow path for diagnostics.
