## Summary

Keep open: the PR is a focused feature proposal, but the current head is not mergeable because it is syntactically malformed, lacks contributor real-behavior proof, and changes token-denial contracts in a security-sensitive way.

## What This Changes

The PR changes device-pairing token helpers, gateway device-token RPC handlers, websocket token issuance, and tests to return structured error objects instead of existing null or ok/reason denial shapes.

## Best Possible Solution

Keep the current generic public denial contract unless maintainers approve a versioned structured-error API; add richer internal diagnostics only where they do not expose token state.

Yes for the PR defects: source inspection of PR head 1ce8933e7a00eb08fe5df74167c6359551adae6f shows malformed TypeScript in two gateway files. There is no current-main bug reproduction because this PR proposes a new token-denial error contract.

## Solution Assessment
