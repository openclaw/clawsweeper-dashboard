This PR changes device-pairing token helpers, gateway RPC handlers, websocket token issuance, and tests to return structured error objects for token-denial paths.

## Best Possible Solution

Keep public token-management denials generic unless maintainers approve a versioned structured-error contract; add richer server-side diagnostics while preserving existing discriminated helper contracts.

## Reproduction Assessment

Yes for the PR defects: the live PR head has malformed TypeScript at the device-method logger, revoke handler, and websocket token-result branch. There is no current-main bug reproduction because this PR proposes a new token-denial error contract.

## Solution Assessment

No. The current patch is not the best way to solve the diagnostics problem because it breaks parsing and exposes differentiated token state; the safer path is internal diagnostics or an owner-approved versioned API.
