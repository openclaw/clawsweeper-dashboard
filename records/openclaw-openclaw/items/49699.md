## Summary

Keep open: this PR still proposes a breaking gateway/device-token contract change, but the current head has parse blockers, missing real behavior proof, and a concrete authz information-disclosure concern. Current main separately fixed the related scope-containment security class while preserving stable ok/reason and generic public denial contracts.

## What This Changes

The PR changes device-pairing token helpers, gateway handlers, and tests to return structured error objects instead of null or ok/reason denial results for scope-escalation and token error cases.

## Best Possible Solution

Preserve the stable ok/reason and nullable helper contracts unless maintainers approve a versioned API change; add richer diagnostic context compatibly in server-side logs and keep public gateway denials generic.

## Reproduction Assessment

Yes. The PR diff itself is a high-confidence source reproduction path for the parse blockers, and current main source/tests show the stable contracts the PR changes.

## Solution Assessment

No. Structured diagnostics may be useful, but replacing helper discriminators and exposing differentiated revoke failures is not the narrowest maintainable or safest path without an explicit versioned API decision.
