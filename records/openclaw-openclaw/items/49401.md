## Summary

Keep open: the bug direction is valid, but this branch still has a blocking privacy issue, lacks inspectable after-fix real behavior proof, and is now conflicting with current main’s shared formatter layout.

## What This Changes

The PR adds detection for plain-text OpenAI-style provider internal errors in assistant error formatting/sanitization and failover classification, with formatter and failover tests.

## Best Possible Solution

Add shared formatter/sanitizer coverage for generic provider internal errors that returns generic chat-safe copy without raw request IDs, while preserving request IDs only in logs or hashed diagnostics and keeping failover classification behavior.

## Reproduction Assessment

Yes. By source inspection, a reported provider `server_error` payload can flow through `formatAssistantErrorText`, into `buildEmbeddedRunPayloads`, and then into the Feishu post sender without a channel-safe generic fallback on current main.

## Solution Assessment

No. The shared formatter/sanitizer seam is the right narrow direction, but this PR should not put raw upstream request IDs into chat-visible copy and should be rebased or replaced against the current shared formatter layout.
