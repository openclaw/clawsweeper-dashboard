## Summary

Keep open: this PR targets a real source-reproducible provider-error leak, but the proposed chat-facing fallback exposes upstream request IDs and the external PR still lacks inspectable after-fix real behavior proof.

## What This Changes

The PR adds generic OpenAI-style internal-error detection, fallback formatting, failover classification, and formatter/classifier tests.

## Best Possible Solution

Update the shared formatter/sanitizer path to return generic chat-safe internal-error copy without raw request IDs, keep request IDs in hashed diagnostics/logs, and add focused regression tests.

## Reproduction Assessment

Yes, by source inspection. A Codex/server_error payload can flow through the shared formatter into formatted error text, then through `buildEmbeddedRunPayloads` and Feishu send without a channel-safe generic fallback.

## Solution Assessment

No. The shared formatter direction is right, but preserving raw request IDs in chat-facing copy conflicts with the related issue and diagnostics privacy boundary, and the branch needs a current-main rebase.
