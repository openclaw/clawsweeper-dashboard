## Summary

Keep open: the bug is source-reproducible and the PR is directionally useful, but the branch is stale against the current shared formatter layout and would expose raw provider request IDs in chat-visible copy.

## What This Changes

The PR adds generic plain-text provider internal-error detection, rewrites those errors with a support request ID, and adds formatter/failover tests under the agent helper surface.

## Best Possible Solution

Implement the rewrite in `src/shared/assistant-error-format.ts` and the sanitizer path, keep existing timeout classification, and keep raw request IDs in logs/hashed diagnostics rather than chat copy.

## Reproduction Assessment

Yes, by source inspection. Current main already classifies the phrase as timeout, but formatter and sanitizer paths still allow the plain provider message or embedded provider message text to reach user-facing copy.

## Solution Assessment

No. The useful detection should be rebased into the current shared formatter/sanitizer layout, and the user-facing message should omit or hash raw upstream request IDs.
