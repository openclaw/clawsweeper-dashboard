## Summary

Keep open: the reported formatter gap is still source-reproducible on current main, but this PR is not mergeable as-is because it adds raw upstream request IDs to chat-facing error copy and still lacks inspectable real behavior proof.

## What This Changes

The branch adds generic OpenAI-style internal-error detection in assistant error helpers, formatter/failover tests, and friendly fallback copy that preserves the support request ID.

## Best Possible Solution

Add shared formatter/sanitizer coverage for generic provider internal errors that returns generic chat-safe copy without raw request IDs, while preserving request IDs only in hashed diagnostics/logs and keeping the existing failover classification behavior.

## Reproduction Assessment

Yes, by source inspection. A Codex `server_error` payload can flow through the shared formatter into `formatAssistantErrorText`, then through `buildEmbeddedRunPayloads` and Feishu send without a channel-safe generic fallback.

## Solution Assessment

No. The shared formatter direction is right, but user-facing fallback copy should not include raw request IDs and the branch needs to be rebased or replaced against the current shared formatter layout.
