## Summary

Keep open: the PR targets a real source-reproducible provider-error leak, but the current diff puts raw upstream request IDs into chat-visible fallback text and still lacks inspectable after-fix real behavior proof.

## What This Changes

The PR adds generic OpenAI-style internal-error detection in assistant error helpers, rewrites affected messages while preserving request IDs, and adds formatter/failover tests.

## Best Possible Solution

Add shared formatter/sanitizer coverage for generic provider internal errors that returns generic chat-safe copy without raw request IDs, while preserving IDs only in hashed diagnostics/logs and keeping transient failover classification.

## Reproduction Assessment

Yes, by source inspection. A reported Codex `server_error` payload can flow through the shared formatter into `formatAssistantErrorText`, then through `buildEmbeddedRunPayloads` and Feishu send without a channel-safe generic fallback on current main.

## Solution Assessment

No. The shared formatter direction is the right narrow seam, but the PR should not include raw upstream request IDs in user-facing copy and needs to be adapted to the current shared formatter layout.
