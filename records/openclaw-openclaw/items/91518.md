## Summary

Keep open: current main and v2026.6.8 still have no Slack pre-dispatch pending receive journal or startup replay path, but the exact fix needs maintainer direction because a local journal may not cover events Slack never delivered while Socket Mode was disconnected.

## What This Changes

The issue requests a Slack-owned durable inbound receive journal, replay path, and documented configurable spool behavior comparable to existing durable ingress patterns in other channels.

## Best Possible Solution

Add a Slack-owned durable receive/replay path using the existing SQLite-backed channel ingress queue or approved plugin state, and fold complete downtime recovery into the broader Slack catch-up design if a receive journal alone is insufficient.

## Reproduction Assessment

No live current-main Slack restart repro was run. Source inspection shows Slack lacks a pre-dispatch pending journal and replay path, so the missing durability is source-reproducible while the exact Socket Mode loss window still needs live proof.

## Solution Assessment

Unclear as written. Reusing the existing SQLite-backed durable receive/ingress queue looks safer than adding a new configurable filesystem spool, and complete restart-window recovery may require Slack history catch-up or an isolated receiver rather than only local journaling.
