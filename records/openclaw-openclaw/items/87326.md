## Summary

Keep open: current main still has the reported Telegram preview behavior, and the fix direction needs maintainer product judgment because preserving inter-tool text can conflict with the existing suppression work for transient tool narration.

## What This Changes

The issue requests that Telegram partial streaming preserve non-final `block` text produced between tool calls instead of overwriting it with later preview or final text.

## Best Possible Solution

Define which inter-tool text is durable user-facing `block` content, then preserve only that content in Telegram without reopening transient tool/progress narration leaks.

## Reproduction Assessment

Yes for source-level reproduction: current Telegram lane delivery tests and code show a non-final block preview update followed by a final update on the same message. I did not establish a live Telegram run in this read-only review.

## Solution Assessment

Unclear: preserving all interleaved text would not be the safest fix by itself because prior work intentionally suppresses transient tool narration. The maintainable path needs a narrower durable-block contract before code changes.
