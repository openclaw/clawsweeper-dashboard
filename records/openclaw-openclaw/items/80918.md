## Summary

Keep open. Current main has fixed the text-only deliverability filter, but the embedded-runner incomplete-turn branch still throws away already collected assistant text when a terminal turn is classified incomplete, so the reported data-loss path still needs a targeted fix and regression coverage.

## What This Changes

The issue reports that an update_plan-finalized run can be misclassified as incomplete, discard collected assistant text, and miss outbound delivery while reporting success.

## Best Possible Solution

Resolve terminal assistant freshness for embedded runs and ensure verified post-tool assistant text is not discarded when the incomplete-turn guard fires incorrectly; add a focused update_plan regression while preserving the existing terminal-toolUse protections.

## Reproduction Assessment

Unclear for a full live current-main WhatsApp reproduction, but source inspection gives a clear failing path if attempt.lastAssistant remains toolUse while assistantTexts already contains the final answer. The issue body also includes concrete production trace data for the released package.

## Solution Assessment

No current implementation fully solves it. The maintainable fix should use the freshest terminal assistant state for classification and avoid dropping collected post-tool assistant text, rather than only relying on the generic incomplete-turn error fallback.
