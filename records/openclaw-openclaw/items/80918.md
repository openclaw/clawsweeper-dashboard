## Summary

Keep open. Current main still keys the incomplete-turn guard and payload construction off stale `lastAssistant` in the reported path, while the linked focused fix PR was closed unmerged; the text-only deliverability part is already fixed, but the final-answer discard path remains.

## What This Changes

Prevent the embedded runner from treating a post-`update_plan` final `stop` turn as incomplete and silently dropping the user's final reply.

## Best Possible Solution

Land a narrow terminal-assistant selection fix using `currentAttemptAssistant ?? lastAssistant` for incomplete-turn classification, payload construction, and final metadata, with regression coverage for stale `lastAssistant` after `update_plan`; keep broader recovery and observability ideas separate.
