## Summary

Keep open. Current main has the text-only deliverability mitigation, but the embedded-runner incomplete-turn path still keys the terminal `toolUse` guard off stale `lastAssistant` and still returns only the generic incomplete-turn error when that branch fires; the focused fix PR is closed unmerged.

## What This Changes

The issue reports an embedded-runner failure where a final assistant `stop` turn after `update_plan` is misclassified as incomplete, dropping the real final text and historically causing a silent channel send miss.

## Best Possible Solution

Use `currentAttemptAssistant ?? lastAssistant` as the terminal assistant at classifier, payload construction, and final metadata boundaries, while preserving the terminal-toolUse guard from https://github.com/openclaw/openclaw/issues/76477.
