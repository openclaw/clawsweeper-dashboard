## Summary

Keep open. Current main and the latest release still classify the stale `lastAssistant.stopReason = "toolUse"` path from stale state, so the final text can still be replaced by the generic incomplete-turn error; the linked focused fix was closed unmerged.

## What This Changes

The issue reports that a stale incomplete-turn classifier can discard a final text answer after an `update_plan` tool call and silently skip channel delivery.

## Best Possible Solution

Use the freshest terminal assistant state for incomplete-turn classification and final assistant metadata, preserve real final text when present, and keep broader recovery or observability proposals separate.

## Reproduction Assessment

Yes for source-level reproduction: an attempt with stale `lastAssistant.stopReason = "toolUse"`, fresh `currentAttemptAssistant.stopReason = "stop"`, and non-empty final assistant text still reaches the stale incomplete-turn branch on current main. I did not establish the full live WhatsApp path in this read-only review.

## Solution Assessment

No, current main is only a partial mitigation because text-only payloads are deliverable but the real final answer can still be replaced by the generic incomplete-turn error. The narrow maintainable fix is to select `currentAttemptAssistant ?? lastAssistant` at the classifier and final metadata boundaries while preserving the true terminal-toolUse guard.
