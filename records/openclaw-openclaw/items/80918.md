## Summary

Keep open. Current main has fixed text-only deliverability, but the core stale-terminal-assistant path remains: incomplete-turn detection and payload/final metadata still use `attempt.lastAssistant`, and the incomplete branch still returns only the generic error payload instead of preserving collected assistant text.

## What This Changes

The issue reports that stale embedded-runner terminal assistant state can classify a clean post-update_plan final answer as incomplete, discard assistant text, and skip channel delivery.

## Best Possible Solution

Use the freshest terminal assistant (`currentAttemptAssistant ?? lastAssistant`) at the incomplete-turn, payload, and final-metadata boundaries, add the stale post-update_plan regression, and keep broader observability/recovery ideas separate.

## Reproduction Assessment

Yes at source level, but not as a live channel run in this review. Current main still reads stale `attempt.lastAssistant` on the implicated branch while the issue body provides concrete production trace evidence for the post-update_plan final-answer shape.

## Solution Assessment

No, current main is not the complete solution because it only resolves text-only deliverability. The narrow maintainable fix is to select the freshest terminal assistant for classification and payload/final metadata while preserving the existing true terminal-toolUse protection.
