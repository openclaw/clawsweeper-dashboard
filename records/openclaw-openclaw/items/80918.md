## Summary

Keep open. Current main still keys the embedded incomplete-turn classifier and incomplete-error branch off stale `lastAssistant`, so the reported post-`update_plan` final text can still be replaced by the generic incomplete-turn error; only the text-only deliverability part is already fixed.

## What This Changes

The issue asks OpenClaw to preserve and deliver a completed `stopReason="stop"` final answer after an `update_plan` tool turn instead of treating the stale prior `toolUse` snapshot as incomplete.

## Best Possible Solution

Use the freshest terminal assistant state for incomplete-turn classification, payload construction, and final assistant metadata, with a focused regression for stale `lastAssistant=toolUse` after `update_plan`; keep broader observability and recovery helpers as separate follow-up work.
