## Summary

Keep open: the patch fixes the main mid-chain compaction loss path and the supplied terminal proof is sufficient, but the current retry-partial branch still drops oversized-message omission notes when the oversized retry partially succeeds and then fails.

## What This Changes

The PR teaches compaction to attach and reuse partial summaries after non-abort mid-chain summarization failures and adds focused Vitest coverage for full, fallback, abort, timeout, and oversized-retry paths.

## Best Possible Solution

Preserve the newest available partial summary while carrying forward oversized omission notes in every path, with regression coverage for retry partial failure.

## Reproduction Assessment

Yes. Current main lets a later non-abort `generateSummary` failure escape `summarizeChunks` without exposing the rolling summary, and the PR's injected-failure proof plus tests demonstrate the reproduction shape; I did not run tests because this review is read-only.

## Solution Assessment

No, not as-is. The partial-summary approach is maintainable, but the oversized-message retry must also keep its omission notes when it falls back to a retry partial.
