## Summary

Keep open: the PR targets a real compaction session-state loss, but the current head still bypasses the existing fallback when no generated chunk succeeded, the supplied proof remains mock-only, and CI lint is failing.

## What This Changes

The branch changes compaction summarization to remember the last successful chunk summary, return it on later non-abort failures, and adds regression tests plus a changelog entry.

## Best Possible Solution

Return a partial summary only after an actual generateSummary call succeeds; first-chunk or no-generated-partial failures should rethrow into summarizeWithFallback, with lint fixed and redacted real or injected runtime proof added before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main loses a later failed chunk's prior generated summary, and the PR regression is visible because the first-chunk catch returns before summarizeWithFallback can run its fallback; I did not run tests in this read-only sweep.

## Solution Assessment

No. The direction is right, but the implementation should only return a generated partial after a chunk actually succeeds, should rethrow otherwise, and should include runtime proof before merge.
