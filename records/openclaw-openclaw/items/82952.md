## Summary

Keep open: the fix direction is useful, but the PR head still bypasses the existing failure fallback when the first chunk fails, and the supplied proof is code/typecheck/test proof rather than real multi-chunk compaction behavior.

## What This Changes

The PR changes agent compaction to return the last successful chunk summary on non-abort, non-timeout chunk failure, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Return a partial only after at least one generated chunk summary succeeds; otherwise rethrow into the existing fallback path, add focused first-chunk and later-chunk regression coverage, and supply redacted real multi-chunk failure proof before merge.

## Reproduction Assessment

Yes, from source inspection: current main updates `summary` chunk-by-chunk and loses that local value if a later chunk throws. The PR regression is also source-reproducible because its catch returns the default fallback before any generated chunk succeeds.

## Solution Assessment

No. The direction is right, but the implementation needs to distinguish a real generated partial from the initial/default state and needs real behavior proof before merge.
