## Summary

Keep open: the branch addresses a real compaction loss case, but the current patch returns from the chunk loop too early and can bypass existing oversized-message fallback behavior; the supplied proof also still does not demonstrate the changed runtime path.

## What This Changes

The PR adds guarded per-chunk compaction recovery that returns the last generated summary after later non-abort failures and adds a new partial-summary Vitest suite.

## Best Possible Solution

Keep the guarded partial-summary recovery, but preserve the existing oversized-message fallback path or append equivalent omitted-message notes, then add redacted real or injected runtime proof for the changed failure mode.

## Reproduction Assessment

Yes. Current main’s summarizeChunks loop has no per-chunk catch, so a later retry-exhausted error leaves summarizeWithFallback to produce generic fallback text; the PR’s mocked test sequence is a high-confidence reproduction path, though I did not run tests in this read-only review.

## Solution Assessment

No. The hasGeneratedChunk guard is the right shape for mid-chain quota/network failures, but returning directly from summarizeChunks is too broad because it can skip the existing oversized-message fallback contract.
