## Summary

Keep open: the bug being addressed is real and the proof for the intended mid-chain recovery path is useful, but the patch returns from `summarizeChunks` early enough to bypass the existing oversized-message fallback, which can lose later recoverable transcript content.

## What This Changes

The PR changes compaction chunk summarization to return a marked partial summary after a non-abort mid-chain chunk failure and adds focused Vitest coverage for the new error branches.

## Best Possible Solution

Carry the successful rolling summary as a fallback candidate, but let `summarizeWithFallback` choose the richest recovery path, including the existing non-oversized retry, with regression coverage for mixed small, oversized, and later-small transcripts.

## Reproduction Assessment

Yes, by source inspection. Current main lets a later non-abort `generateSummary` failure escape `summarizeChunks`, and the blocking regression can be reproduced with a mocked first successful chunk, a failing oversized middle chunk, and a later small message that the existing fallback should still summarize.

## Solution Assessment

No, not as written. Returning directly inside `summarizeChunks` is too early because it bypasses an existing recovery path; the safer fix is to surface the partial summary to the caller and use it only when the fallback cannot recover more context.
