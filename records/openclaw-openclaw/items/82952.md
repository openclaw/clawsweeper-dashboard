## Summary

Keep this PR open: it addresses a real compaction failure path on current main, but the proposed early return can bypass the existing oversized-message fallback and the changed runtime behavior is still proven only through mocked tests.

## What This Changes

The PR changes agents compaction to return the last successful chunk summary after a later non-abort summarization failure and adds a focused Vitest file for partial-summary behavior.

## Best Possible Solution

Preserve partial-summary progress at the `summarizeWithFallback` layer, or carry a structured partial-summary result upward, so no-oversized failures can return the partial summary while oversized failures still retry the small transcript and append omitted-message notes.

## Reproduction Assessment

Yes by source inspection. Current main exits `summarizeChunks` on a later retry-exhausted `generateSummary` failure, and PR head source shows the new early return that bypasses the oversized fallback path.

## Solution Assessment

No, not as currently written. The guard is close to the right behavior for ordinary chunk failures, but it lives too low in the stack and skips an existing fallback contract that should still run when oversized messages are involved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve oversized fallback before returning partial summaries:** `src/agents/compaction.ts:380`
  - body: `summarizeWithFallback` currently handles full-summary failures by detecting oversized messages, retrying the small transcript, and appending `[Large ... omitted]` notes. Returning `summary!` here means that if an earlier chunk succeeds and a later oversized chunk fails, the caller never reaches that fallback and the persisted summary silently omits the oversized-message marker. Carry the partial summary up to the fallback layer, or otherwise preserve the oversized retry/annotation path before returning.
  - confidence: 0.88
