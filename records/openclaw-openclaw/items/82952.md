## Summary

Keep this PR open: the partial-summary recovery is useful and well evidenced, but the current patch still drops oversized-message omission notes when the oversized retry produces its own partial summary.

## What This Changes

This PR adds partial-summary recovery for non-abort mid-chain compaction failures and adds focused Vitest coverage for the new fallback paths.

## Best Possible Solution

Preserve the best available partial summary in every fallback path while carrying forward oversized omission notes whenever oversized messages were filtered out.

## Reproduction Assessment

Yes. Source inspection shows current main loses the rolling summary when a later non-abort chunk fails, and the PR body includes injected-failure proof of the fixed behavior.

## Solution Assessment

No, not as-is. The partial-summary recovery is the right narrow direction, but the retry-partial path also needs to preserve oversized omission notes.
