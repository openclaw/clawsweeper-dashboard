## What This Changes

The PR changes agent compaction to return the last successful chunk summary after non-abort, non-timeout chunk generation failure, with regression tests and a changelog entry.

## Best Possible Solution

Return a partial only after at least one generated chunk summary succeeds; otherwise rethrow into the existing fallback path, keep focused regression coverage for both paths, and supply redacted runtime proof before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main loses a generated chunk summary when a later non-timeout chunk throws, and the PR regression is visible when the first chunk throws before any generated summary exists. This read-only review did not run tests or live compaction.

## Solution Assessment

No. The direction is right, but the implementation should only return a generated partial after a chunk succeeds and should preserve the existing fallback for first-chunk failures.
