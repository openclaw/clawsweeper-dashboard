## What This Changes

## Best Possible Solution

Return a partial only after at least one generated chunk summary succeeds; otherwise rethrow into the existing fallback path, cover both first-chunk and later-chunk failures, and add redacted runtime or injected proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main loses a generated chunk summary when a later non-timeout chunk throws, and the PR regression is visible when the first chunk throws before any generated summary exists; this read-only review did not run tests or live compaction.

## Solution Assessment

No. The direction is right, but the implementation should only return a generated partial after a chunk succeeds and should preserve the current fallback path when none did.
