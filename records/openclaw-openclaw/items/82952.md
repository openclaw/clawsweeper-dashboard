## Summary

Keep open: the fix targets a real compaction state-loss path, but the current diff introduces a first-chunk failure regression and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR updates agent compaction to return the last successful chunk summary after non-abort, non-timeout retry exhaustion and adds a changelog entry.

## Best Possible Solution

Return a generated partial only after at least one chunk succeeds; otherwise preserve abort/timeout propagation and rethrow non-timeout first-chunk failures so the existing fallback path can report that summary generation failed.

## Reproduction Assessment

Yes from source inspection: current main updates `summary` chunk-by-chunk, then loses that local value if a later chunk throws into `summarizeWithFallback`. I did not run a live provider-failure injection because the review constraints prohibit test or artifact generation in the checkout.

## Solution Assessment

No: the direction is right, but the implementation must distinguish a real generated partial from the initial/default state and needs focused regression coverage plus real behavior proof.
