## Summary

Keep open: the intent and proof are useful, but the PR is not merge-ready because the early partial-summary return bypasses an existing compaction fallback that can preserve later recoverable transcript content.

## What This Changes

The PR changes compaction chunk summarization to return a marked partial summary after non-abort mid-chain failures and adds a new Vitest file for those error branches.

## Best Possible Solution

Carry the successful rolling summary as a fallback candidate, but let `summarizeWithFallback` choose the richest recovery path, including the existing non-oversized retry, with regression coverage for mixed small, oversized, and later-small transcripts.

## Reproduction Assessment

Yes, by source inspection. Current main lets later non-abort `generateSummary` failures reach `summarizeWithFallback`, while the PR regression can be exercised with a successful first chunk, a failing oversized middle chunk, and later small messages that should still be summarized.

## Solution Assessment

No, not as written. Returning directly inside `summarizeChunks` is too early; the safer fix is to surface the partial summary as a candidate and use it only after the existing fallback cannot recover more context.
