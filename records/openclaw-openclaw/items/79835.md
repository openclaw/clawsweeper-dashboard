## Summary

Needs changes before merge: the PR addresses the documented imageModel auto-switch path and fixes the earlier modelOverride spoofing hole, but the attachment parser still trusts a .png filename after ZIP/DOCX content sniffing, so disguised non-image bytes can still be routed as image media.

## What This Changes

The PR adds Gateway image-attachment detection and reply modelOverride plumbing so Dashboard image sends can use agents.defaults.imageModel when the active model lacks vision, with fallback filtering and regression tests.

## Best Possible Solution

Keep the Gateway-owned imageModel switch, but move the generic-sniff MIME trust rule into the shared attachment parser so spoofed image filenames cannot become image media, then land with focused Gateway/reply tests and green CI.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a text-only primary plus a vision-capable agents.defaults.imageModel, send an image through Dashboard chat.send, and current main dispatches the reply without a modelOverride despite the docs contract.

## Solution Assessment

No. The Gateway/reply boundary is the right place for the turn-scoped override, but the current patch leaves the final attachment parser MIME decision inconsistent with the new preflight detector.
