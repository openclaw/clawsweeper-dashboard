## What This Changes

The PR adds Gateway attachment sniffing plus auto-reply modelOverride routing so image attachments can temporarily use a configured vision-capable imageModel when the selected session model lacks image support.

## Best Possible Solution

Land this Gateway-owned implementation after maintainer approval and required gates, keeping image-model selection temporary per turn and preserving provider-scoped auth/profile behavior.

## Reproduction Assessment

Yes, by source inspection: current main's chat.send path only checks the selected session model's image capability before parsing attachments and dispatches replyOptions without modelOverride, while docs promise imageModel when the primary model cannot accept images. I did not run a live failing pre-fix turn in this read-only review.

## Solution Assessment

Yes. Moving the decision into Gateway before attachment parsing matches where MIME/content classification happens, and the auto-reply changes carry the temporary model choice without persisting or leaking auth profiles across providers.
