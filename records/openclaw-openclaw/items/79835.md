## Summary

Keep open: this is a useful bug-fix PR with sufficient live proof, but the latest head still drops the prepared image fallback list before the runner failover path, so image override turns can fall back to normal text-model fallbacks.

## What This Changes

The PR adds Gateway image attachment content sniffing, conditional per-turn image-model overrides/fallback preparation, auth-profile handling for temporary model switches, and focused auto-reply/Gateway tests.

## Best Possible Solution

Land the Gateway image detection and temporary override after the prepared image fallback list is threaded into the actual agent runner failover path.

## Reproduction Assessment

Yes. Current main is source-reproducible: the docs define `imageModel` for primary-model image incompatibility, while current `chat.send` only changes attachment routing and never dispatches an image-model override for the main agent run.

## Solution Assessment

No, not yet. Gateway sniffing plus a per-turn image override is the right boundary, but the accepted override path must also carry `imageModel.fallbacks` into runner `fallbacksOverride` so failover remains vision-capable.
