## Summary

Needs changes before merge: the central Gateway imageModel override fix is plausible and the earlier MIME-spoofing parser blocker appears addressed, but the current PR still treats provider wildcard allowlists as exact keys when applying the image override and fallbacks.

## What This Changes

The PR adds Dashboard/Gateway image attachment detection, imageModel modelOverride plumbing, fallback filtering, MIME sniffing safeguards, and regression tests for image and non-image attachments.

## Best Possible Solution

Keep the Gateway-owned imageModel switch, but route all new image override allowlist decisions through the same wildcard-aware model visibility helper used by current main.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a text-only primary plus a vision-capable `agents.defaults.imageModel`, send an image via Dashboard `chat.send`, and current main dispatches the turn without a model override.

## Solution Assessment

No. The Gateway/reply boundary is the right place for the fix, but the PR must honor current-main provider wildcard allowlists before it is the narrow maintainable solution.
