## Summary

Keep this PR open, but it still needs a narrow correctness fix before merge: the final head computes image-model fallbacks but does not carry them into the actual agent run failover path after the primary image override is accepted.

## What This Changes

The PR adds Gateway image attachment sniffing and per-turn auto-reply image model overrides when the active session model cannot accept images.

## Best Possible Solution

Keep the Gateway-owned image detection and temporary override path, but thread the filtered imageModel fallback list into the actual agent run fallback override so image turns fail over only through image-capable configured fallbacks.

## Reproduction Assessment

Yes. Current main source shows chat.send decides inline image routing from the session model capability without passing a model override, while the docs say imageModel is used when the primary cannot accept images.

## Solution Assessment

No. Gateway content sniffing plus a temporary auto-reply override is the right boundary, but the patch must carry imageModel fallbacks into runtime failover before merge.
