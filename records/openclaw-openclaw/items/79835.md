## Summary

Keep open: this is the active canonical fix for the documented imageModel routing bug, but the latest head still has a P2 fallback plumbing blocker before merge.

## What This Changes

The PR adds Gateway image-attachment sniffing and temporary auto-reply model override plumbing so Dashboard image turns can use a configured vision model when the session model is text-only.

## Best Possible Solution

Wire the prepared imageModel fallback list into the actual agent run fallback override for temporary image-model turns, while keeping normal text-model fallback, session override, heartbeat, auth-profile, and media-understanding behavior unchanged.

## Reproduction Assessment

Yes by source inspection: current main routes Dashboard image attachments through the selected session/default model unless that model already supports images, despite docs saying imageModel is used when the primary model cannot accept images. I did not run a live pre-fix repro in this read-only review.

## Solution Assessment

No, not as-is. The Gateway sniffing and temporary override direction is maintainable, but the PR must thread imageModel fallbacks into the runner's modelFallbacksOverride path before it fully satisfies the documented fallback behavior.
