## Summary

Keep open: this PR addresses a real, doc-backed imageModel routing bug and the supplied proof covers the main image/PDF behavior, but the latest head still has a merge-blocking auth-profile persistence edge case for temporary image overrides.

## What This Changes

The PR adds Gateway image-attachment MIME/content sniffing, temporary imageModel override and fallback routing through auto-reply, auth-profile handling for image overrides, and focused tests.

## Best Possible Solution

Land a version that keeps the per-turn image override and MIME hardening while carrying the pre-override session provider into auth-profile cleanup and run-prep so temporary provider changes never rewrite stored session credentials.

## Reproduction Assessment

Yes by source inspection. Current main lacks the documented imageModel override path for text-only session models, and the PR-head auth-profile edge follows directly from the defaultProvider comparison plus the existing profile-clearing helper.

## Solution Assessment

No as-is. The Gateway placement and attachment detection are the right direction, but auth routing must compare against the selected session provider being temporarily replaced, not the configured default provider.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve profiles when image overrides replace a session provider:** `src/auto-reply/reply/model-selection.ts:309-310`
  - body: The cross-provider guard compares the image override provider to `defaultProvider`, but the provider being temporarily replaced can be a session-selected model. If the configured default is OpenAI, a user has an Anthropic session/auth profile, and `imageModel` is OpenAI, this falls through to auth-profile cleanup/resolution and can clear or rewrite the stored Anthropic profile even though the image override is only for this turn. Carry the pre-override session provider, or an explicit provider-changed flag, and use it here and in `runPreparedReply`.
  - confidence: 0.86
