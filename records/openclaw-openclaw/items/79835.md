## What This Changes

The PR adds Gateway image-attachment sniffing and temporary auto-reply model override plumbing so Dashboard image turns can use a configured vision-capable imageModel while non-image attachments stay on the selected model path.

## Best Possible Solution

Land this Gateway-owned implementation after normal maintainer approval and required gates, keeping image-model selection temporary per turn and preserving provider-scoped auth behavior.

## Reproduction Assessment

Yes by source inspection: current main routes Dashboard chat attachments through the selected session model only, while the docs say imageModel is used when the primary model cannot accept images. I did not run a live failing pre-fix turn because this review is read-only.

## Solution Assessment

Yes. Gateway is the right boundary because it has attachment MIME/content context before dispatch, and the auto-reply changes carry a temporary, allowlist-checked model choice without persisting a session model change.
