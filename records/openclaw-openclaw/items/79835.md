## Repair Work Prompt

Repair the image model override auth-profile provider comparison for https://github.com/openclaw/openclaw/pull/79835. The current PR compares the selected image provider to the configured defaultProvider in src/auto-reply/reply/get-reply-run.ts, but Gateway decides the override from the effective session model, which may come from a stored session override. Preserve same-provider auth profiles when the image override provider matches the effective pre-override session provider, and avoid clearing stored profiles when an automatic image override switches away from that provider. Prefer passing a small explicit pre-override/effective provider fact through the existing GetReplyOptions/runPreparedReply path or deriving it once from the stored override; do not broaden model-selection semantics, add config, or change unrelated auth-profile behavior. Add focused regression coverage for: configured default provider A + stored session model/profile provider B + imageModel provider B preserves profile; configured default provider B + stored session model/profile provider A + imageModel provider B does not clear the stored A profile during the automatic image turn.
