## Summary

Keep open. The PR addresses a documented Dashboard imageModel gap and the supplied live proof is enough, but latest head still has two merge-blocking correctness gaps around allowlisted vision selection and retry fallbacks.

## What This Changes

The branch adds Gateway image attachment sniffing, a temporary image-model override path with fallback/auth handling, and regression tests for Dashboard image attachments.

## Best Possible Solution

Land a Gateway-owned image override that only inlines images after selecting an allowed vision model or allowed vision fallback, and make retry fallback resolution stay within the imageModel fallback chain for that turn.

## Reproduction Assessment

Yes. Source inspection shows current main documents imageModel fallback behavior but chat.send attachment routing only checks the session model's image support and has no modelOverride path.

## Solution Assessment

No, not yet. The Gateway override direction is maintainable, but the patch must prove an allowlisted vision selection before keeping images inline and must carry imageModel fallbacks into runtime retry resolution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep blocked overrides on the media path:** `src/gateway/server-methods/chat.ts:2351-2352`
  - body: Here `supportsImages` can become true from `imageModelSupportsImages` before `getReplyFromConfig` checks `agents.defaults.models`. If the configured imageModel is not allowlisted, the reply falls back to the text model while the Gateway has already kept images inline, bypassing the safer MediaPaths/media-understanding path.
  - confidence: 0.86
- **[P2] Carry image fallbacks into runtime retries:** `src/auto-reply/reply/get-reply.ts:976`
  - body: The PR prepares `modelOverrideFallbacks` only for the initial override selection. `runPreparedReply` receives only `hasAppliedImageModelOverride`, so runtime retries still derive fallbacks from the normal model chain; a failed vision model can retry text fallbacks with inline images.
  - confidence: 0.87
