## Summary

Keep this PR open, but it still needs changes before merge: the main image-model override path is useful and has live proof, but prepared image override fallbacks are not carried into runtime model retries.

## What This Changes

The PR adds Gateway image attachment sniffing, temporary `imageModel` model overrides for Dashboard image turns, auth/profile handling for temporary overrides, and regression tests around image/non-image attachment routing.

## Best Possible Solution

Keep the Gateway-owned temporary image override, but carry the prepared image fallback list into `FollowupRun` and `resolveModelFallbackOptions` so image turns retry only configured vision fallbacks.

## Reproduction Assessment

Yes. Source inspection of current main shows Dashboard attachments are parsed and dispatched using the session model path without applying the documented `imageModel` override when the session model is text-only.

## Solution Assessment

Not yet. Selecting the temporary override in Gateway is the right boundary, but the proposed patch is incomplete until runtime fallback retries use `agents.defaults.imageModel.fallbacks` for that temporary image run.
