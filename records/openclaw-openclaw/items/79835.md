## Summary

Keep open: this PR addresses a real documented imageModel routing bug and includes sufficient real behavior proof, but the patch still has one P2 correctness blocker around allowlist-blocked imageModel overrides preserving inline images before the reply layer can reject the override.

## What This Changes

The PR adds Gateway MIME/content sniffing and auto-reply modelOverride plumbing so Dashboard image turns can use a configured vision-capable imageModel with fallback and auth-profile handling.

## Best Possible Solution

Update Gateway so inline image preservation is based on an allowlist-validated imageModel selection, with a regression covering a blocked imageModel falling back to the existing MediaPaths offload path.

## Reproduction Assessment

Yes by source inspection: current main routes Dashboard attachments through the selected session model and does not pass an imageModel override, while the docs promise imageModel use when the primary cannot accept images. I did not run a live pre-fix Dashboard turn in this read-only review.

## Solution Assessment

No as-is: the PR chooses the right owner boundary, but Gateway must not keep images inline until it knows the imageModel override or an allowed fallback will actually be used. The safer fix is to share or duplicate the allowlist check before setting supportsImages, and otherwise preserve the MediaPaths offload behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Gate inline image handling on an allowed override:** `src/gateway/server-methods/chat.ts:2323-2324`
  - body: This sets `supportsImages` from `imageModelSupportsImages` before the downstream allowlist check in `getReplyFromConfig`. If `agents.defaults.imageModel.primary` is vision-capable but absent from `agents.defaults.models`, Gateway keeps the attachment inline; `getReplyFromConfig` later rejects `modelOverride`, so the text-only session model receives `images` instead of the existing `MediaPaths` offload path. Check the override/fallback against the same allowlist before setting `supportsImages`, or fall back to media offload when the override is rejected.
  - confidence: 0.86
