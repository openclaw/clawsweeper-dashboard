## Summary

Keep open: the PR addresses a documented imageModel routing gap and includes useful live proof, but it is not merge-ready because the new imageModel fallback list is not wired into the actual model failover path for accepted image overrides.

## What This Changes

The PR adds Gateway content-sniffed image attachment detection, passes image model overrides into auto-reply/model selection, adjusts auth-profile handling, and adds focused gateway/model-selection tests.

## Best Possible Solution

Land the Gateway-owned image detection and temporary override path after it preserves agents.defaults.imageModel fallback semantics through the actual run fallback engine.

## Reproduction Assessment

Yes. Current main source shows chat.send decides image routing from the session model capability without passing a model override, while the docs say imageModel should be used when the primary cannot accept images.

## Solution Assessment

No. Gateway classification plus a temporary auto-reply override is the right boundary, but the current patch needs to carry imageModel fallbacks into runtime failover before merge.
