## Summary

Keep open. The PR is a plausible fix with supplied live proof, but the accepted image-model override path still drops `agents.defaults.imageModel.fallbacks` before the actual run failover engine, so a failed vision primary can fall through the normal model fallback chain.

## What This Changes

The PR adds Gateway image attachment sniffing, per-turn image model overrides and fallback options, auth-profile handling for temporary provider switches, and focused auto-reply/Gateway tests.

## Best Possible Solution

Land the Gateway-owned image detection and per-turn override after the filtered image-model fallback list is threaded into the run fallback override used by the agent runner.

## Reproduction Assessment

Yes. Current main source shows `chat.send` routes image attachments based on the session model capability without passing an image model override, while the docs state `agents.defaults.imageModel` should be used when the primary cannot accept images.

## Solution Assessment

No, not yet. Gateway sniffing plus a temporary auto-reply override is the right boundary, but the proposed patch needs to preserve `agents.defaults.imageModel.fallbacks` through the actual runtime failover path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Thread image fallbacks into run failover:** `src/auto-reply/reply/get-reply.ts:976`

Pass the filtered image fallback list into the run fallback override whenever the image override is applied.
