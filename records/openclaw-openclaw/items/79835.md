## Summary

Keep open: current main still lacks the documented imageModel auto-switch for image turns, and this active PR is the canonical implementation path with sufficient real behavior proof, green relevant CI, and no blocking findings from this review.

## What This Changes

The PR adds Gateway attachment sniffing plus auto-reply model override plumbing so real image attachments temporarily use a configured vision-capable image model when the selected session model cannot accept images.

## Best Possible Solution

Land this Gateway-owned implementation after maintainer approval and required gates, keeping image-model selection temporary per turn and preserving provider-scoped auth/profile behavior.

## Reproduction Assessment

Yes by source inspection: current main routes Dashboard chat attachments through the selected session model only, while the docs say imageModel should be used when the primary model cannot accept images. I did not run a live failing pre-fix turn in this read-only review.

## Solution Assessment

Yes. Moving the decision into Gateway before attachment parsing matches where MIME/content classification happens, and the auto-reply changes carry the temporary model choice without persisting or leaking auth profiles across providers.
