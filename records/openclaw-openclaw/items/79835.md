## Summary

Keep open: the latest head addresses the main Dashboard image-switch path and includes sufficient live proof, but runtime retry/failover still uses the regular model fallback policy instead of the prepared `imageModel` fallback list.

## What This Changes

The PR adds Gateway image attachment content sniffing and threads a temporary vision-capable `imageModel` override plus filtered override fallbacks through auto-reply for Dashboard image turns.

## Best Possible Solution

Keep the Gateway-owned temporary image override, but carry its prepared image fallback list into the run fallback override path so vision-model failures retry only configured vision fallbacks.

## Reproduction Assessment

Yes. Source inspection of current main shows Dashboard attachments are parsed and dispatched using the session model path without applying the documented `imageModel` override when the session model is text-only.

## Solution Assessment

Mostly yes. Selecting the temporary override in Gateway before attachment parsing is the right boundary, but the PR is incomplete until runtime fallback uses `agents.defaults.imageModel.fallbacks` for that temporary image run.
