## Summary

Keep open: this is a valid fix for the documented imageModel auto-switch gap and the live proof is sufficient, but the branch still leaves image turns using the normal text-model fallback chain after the primary image override is accepted.

## What This Changes

The PR adds Gateway image attachment sniffing, per-turn imageModel overrides and fallbacks, auth-profile handling for temporary model switches, and focused Gateway/auto-reply tests.

## Best Possible Solution

Land the Gateway-owned image detection and temporary override after the prepared imageModel fallback list is threaded into the actual runner fallback options for image override turns.

## Reproduction Assessment

Yes. Current main is source-reproducible: the docs define imageModel as the fallback when the primary model cannot accept images, while current `chat.send` only offloads/routes images and does not dispatch an image-model override for the main agent run.

## Solution Assessment

No, not yet. Gateway sniffing plus a per-turn auto-reply override is the right boundary, but the PR must preserve `agents.defaults.imageModel.fallbacks` through the runtime failover path before it is mergeable.
