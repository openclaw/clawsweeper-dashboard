## Summary

Keep open: the PR targets a real imageModel routing gap, but the current diff introduces two source-reproducible routing regressions and the supplied live-proof links are not inspectable.

## What This Changes

The PR adds Gateway image-attachment detection, threads image-model overrides and fallbacks into auto-reply, and adds tests for image versus non-image chat.send routing.

## Best Possible Solution

Keep the imageModel path as fallback routing only: preserve selected vision-capable models, use a resolved and allowlisted imageModel/fallback when needed, and otherwise continue through MediaPaths/media-understanding.

## Reproduction Assessment

Yes for the PR regressions via source inspection: image turns with a vision-capable selected model and turns with an unresolved imageModel override both exercise the bad branches. I did not establish a live failing run in this read-only review.

## Solution Assessment

No. The narrow maintainable fix should preserve current vision-capable model routing and resolve/allowlist image overrides before treating attachments as safe inline vision input.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve vision-capable session models:** `src/gateway/server-methods/chat.ts:2216-2219`
  - body: This assigns `imageModel` as `modelOverride` whenever image attachments are present, before checking whether the selected session/channel model already supports images. The documented contract is fallback routing only when the primary model cannot accept images, so users already on a vision-capable model would be silently switched for every image turn.
  - confidence: 0.78
- **[P2] Resolve image overrides before keeping attachments inline:** `src/gateway/server-methods/chat.ts:2233-2235`
  - body: When override resolution fails, this branch still uses the raw override string for vision detection, which can mark `supportsImages` true. `getReplyFromConfig` later drops the unresolved override and uses the default model, so a text-only session can receive inline images instead of the existing MediaPaths/media-understanding path.
  - confidence: 0.88
