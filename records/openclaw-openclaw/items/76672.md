## Summary

Keep open. The PR is protected by the `maintainer` label and current head `3d2939b` still has a security-relevant sudo carrier parsing bypass in the exec approval path, even though the grouped `-EH` case was fixed after the earlier review.

## What This Changes

The PR introduces shared command-analysis helpers for exec approval parsing, display summaries, carrier handling, and strict inline-eval detection, then routes gateway/node approval surfaces and tests through that boundary.

## Best Possible Solution

Keep the shared command-analysis boundary, but make sudo carrier parsing match real command-running forms and add regression coverage for `sudo -i`, `sudo -s`/`--shell`, and `sudo -k <command>` before merge.

## Reproduction Assessment

Yes, source-reproducible for the PR finding. Head `3d2939b` routes approval checks through `resolveSudoLikeCarriedArgv`, and `sudo -h` confirms the omitted or misclassified flags can still carry a command.

## Solution Assessment

No. The shared boundary is maintainable, but the proposed parser is not yet safe for the exec approval security path because it rejects valid sudo command-running forms before inspecting the carried argv.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Recognize sudo modes before unwrapping carried commands:** `src/infra/command-analysis/risks.ts:60-78`
  - body: `sudo -i`, `sudo -s`/`--shell`, and `sudo -k <command>` are valid command-running forms, but this resolver either treats them as unknown or non-executing and returns `null`. That prevents both `detectPolicyInlineEval` and `buildCommandPayloadCandidates` from seeing carried commands like `python3 -c ...` or `/approve ...`, bypassing the strict inline-eval and approval-command guards.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff changes the exec approval security boundary and can miss inner commands behind valid sudo carrier forms.

Concerns:

- **[medium] Strict inline-eval carrier bypass:** `src/infra/command-analysis/risks.ts:422`
  - body: When `resolveCarrierCommandArgv` returns `null` for valid sudo modes, `detectPolicyInlineEval` does not inspect the inner interpreter argv, so approval-only inline-eval commands can be treated as ordinary allowlisted carrier commands.
  - confidence: 0.9
- **[medium] Approval command guard carrier bypass:** `src/infra/command-analysis/risks.ts:370`
  - body: `buildCommandPayloadCandidates` depends on the same carrier resolver; if sudo mode parsing returns `null`, the `/approve` shell-execution guard sees only the outer sudo command and misses the carried approval command.
  - confidence: 0.86
