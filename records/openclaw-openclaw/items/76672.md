## Summary

Keep open. This PR has the protected `maintainer` label and the current diff still has approval-boundary parser gaps: valid `sudo` carrier forms can stop unwrapping before the inner command, which can hide strict inline-eval or guarded approval text from the shared analysis.

## What This Changes

The PR introduces a shared `src/infra/command-analysis` boundary and routes exec approval policy checks, display summaries, carrier parsing, and inline-eval detection through it.

## Best Possible Solution

Keep the shared command-analysis direction, but make carrier parsing preserve valid command-running `sudo`/`doas` forms, add regression coverage for grouped and login/shell sudo options, then rerun the changed gate and security review.

## Reproduction Assessment

Yes. Source inspection of the PR parser plus `sudo -h` gives a high-confidence path: valid `sudo -EH python3 -c 'print(1)'` and `sudo -i python3 -c 'print(1)'` forms return null before the inner interpreter is checked.

## Solution Assessment

No. The shared boundary is maintainable, but the proposed carrier parser must match command-executing sudo forms before it is the safest solution for this approval boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Handle clustered sudo flags before unwrapping:** `src/infra/command-analysis/risks.ts:207-220`
  - body: `sudo` accepts grouped standalone flags such as `-EH`, but this parser only compares the whole token against exact option names and returns null for the cluster. A valid command like `sudo -EH python3 -c 'print(1)'` is no longer unwrapped, so strict inline-eval detection can miss the inner interpreter command.
  - confidence: 0.9
- **[P1] Cover command-running sudo modes:** `src/infra/command-analysis/risks.ts:59-89`
  - body: The sudo option tables omit valid command-executing modes such as `-i`, `-s`, and `--shell`, and classify `-k` as non-executing even though `sudo -k <command>` runs a command after resetting credentials. These forms return null before `buildCommandPayloadCandidates` or `detectInlineEvalArgv` can inspect the carried command.
  - confidence: 0.88
