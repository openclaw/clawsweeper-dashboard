## Summary

## What This Changes

The PR changes the Anthropic-family OpenAI compatibility wrapper and its tests so Anthropic custom tools, custom tool_choice, and custom allowed_tools entries are projected as OpenAI function payloads.

## Best Possible Solution

Keep the bridge-layer projection, but make custom handling terminal so every custom definition either converts to a function tool or is quarantined, then add focused malformed-custom coverage and redacted real provider proof.

Yes, at source level. Current main preserves custom tool payloads in the reported wrapper, and PR-head source still preserves an unprojectable custom snapshot through the fallback branch; I did not run a live OpenAI request in this read-only review.

## Solution Assessment
