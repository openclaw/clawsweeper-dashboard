## Summary

Keep open. Current main still has the approval-completion session-resume failure path that can direct-send unmatched approval result text, so this remains a distinct unresolved leak path from the already closed restart delivery-recovery report.

## What This Changes

The issue requests fail-closed handling for `/approve` post-approval follow-up fallback when session resume fails and `resultText` contains raw internal handler or source-like text.

## Best Possible Solution

Fail closed at the direct fallback boundary for unverified or source-like `other` result text, then fix the producer so only verified agent replies reach that path.

## Reproduction Assessment

Yes, source-level. A deliverable Telegram route plus session key, a thrown `callGatewayTool("agent", ...)`, and JS-like unmatched `resultText` follows the current-main path that prefixes and sends sanitized raw text.

## Solution Assessment

Yes for immediate containment. A fail-closed guard in `formatDirectExecApprovalFollowupText` or `sendDirectFollowupFallback` is the narrowest maintainable boundary fix, with a producer-side root-cause fix afterward.
