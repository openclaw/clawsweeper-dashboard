## Summary

Keep open. Current main still has the approval-completion to session-resume to direct-fallback path described in the report, and unrecognized approval result text can still be delivered after generic sanitization rather than being fail-closed.

## What This Changes

Prevent exec approval follow-up recovery from sending raw internal handler/source-like result text directly to Telegram when session resume fails.

## Best Possible Solution

Add a fail-closed guard at the exec approval direct fallback so unrecognized/source-like text is dropped or replaced with a generic unavailable message, then repair the producer and add regression coverage for source-like unmatched payloads.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to call `sendExecApprovalFollowup` with a deliverable Telegram route, a session key, rejected `callGatewayTool("agent", ...)`, and JavaScript-like unmatched `resultText`; current main parses it as `other` and sends sanitized raw text through direct fallback.

## Solution Assessment

Yes, the two-layer direction in the report is the right shape: contain the delivery boundary first, then fix the producer once maintainers identify where the source-like `resultText` is generated.
