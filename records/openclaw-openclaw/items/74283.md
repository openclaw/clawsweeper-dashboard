## Summary

Keep open. Current main still has the approval-completion to session-resume-failure to direct-fallback path, and unmatched approval result text can still be delivered after generic sanitization; the issue remains distinct from #72184.

## What This Changes

The issue reports that `/approve` post-approval follow-up recovery can send raw internal handler source or local state to Telegram after automatic session resume fails.

## Best Possible Solution

Add a fail-closed guard for unmatched approval follow-up text at the direct delivery boundary, then trace and fix the producer that creates source-like `resultText`, with regression coverage for the reported snippets.

## Reproduction Assessment

Yes, source-level. Calling `sendExecApprovalFollowup` with a deliverable Telegram route, a session key, a rejected `callGatewayTool("agent", ...)`, and JS-like unmatched `resultText` follows the current-main path that parses as `other` and sends prefix plus sanitized raw text.

## Solution Assessment

Yes for immediate containment: fail closed in `formatDirectExecApprovalFollowupText` or `sendDirectFollowupFallback` is the narrowest maintainable boundary fix. The producer-side repair should follow once maintainers trace where the source-like result text is created.
