## Summary

Keep open. Current main still passes unmatched approval result text through the direct follow-up fallback after session-resume failure, so this remains a live security-sensitive leak path distinct from #72184.

## What This Changes

Prevent approval follow-up recovery from delivering raw internal handler/source-like result text to Telegram when session resume fails.

## Best Possible Solution

Fail closed at the approval direct-delivery boundary for unmatched/source-like result text, add regression coverage for the reported snippets, then trace and fix the producer that creates invalid `resultText`.

## Reproduction Assessment

Yes, source-level. A deliverable Telegram route plus a session key, rejected `callGatewayTool("agent", ...)`, and JS-like unmatched `resultText` follows current main into `sendDirectFollowupFallback`, which sends the fallback prefix plus sanitized raw text.

## Solution Assessment

Yes for immediate containment: a fail-closed guard in `formatDirectExecApprovalFollowupText` or `sendDirectFollowupFallback` is the narrowest boundary fix. The producer-side repair should follow once maintainers trace where source-like `resultText` is created.
