## Summary

Keep open. Current main still has the approval-completion to session-resume to direct-fallback path described in the report, and unmatched approval result text can still be delivered after only generic sanitization; this remains distinct from #72184's restart recovery path.

## What This Changes

Prevent exec approval follow-up recovery from sending raw internal handler/source-like `resultText` to Telegram when session resume fails.

## Best Possible Solution

Add a fail-closed guard at the exec approval direct fallback so only recognized approval result shapes or verified safe reply text can be delivered, then repair the producer and add regression coverage for source-like snippets.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to call `sendExecApprovalFollowup` with a deliverable Telegram route, a session key, rejected `callGatewayTool("agent", ...)`, and JavaScript-like unmatched `resultText`; current main parses it as `other` and sends the sanitized raw text through direct fallback.

## Solution Assessment

Yes for immediate containment: a fail-closed boundary guard is the narrowest maintainable fix for the leak. The producer repair should follow once maintainers identify where source-like `resultText` is created.
