## What This Changes

Adds `openclaw cron run --wait` with timeout/poll flags, exact `cron.runs` `runId` filtering, protocol/Swift model wiring, docs, and focused cron tests.

## Best Possible Solution

Land the opt-in CLI wait mode if maintainers accept the API surface, keeping `cron.run` non-blocking while using exact `runId` history polling for completion.

## Reproduction Assessment

Not applicable: this is an additive feature PR, not a broken existing behavior report. Source inspection of current main confirms the wait/run-id filter capability is absent, and the PR body supplies after-fix live Gateway output.

## Solution Assessment

Yes, subject to maintainer approval: polling `cron.runs` by the returned `runId` is the narrow maintainable approach because it preserves the non-blocking Gateway `cron.run` RPC. The earlier zero poll interval concern is now addressed by pre-enqueue validation.
