## Best Possible Solution

## Solution Assessment

Yes, subject to maintainer approval: polling `cron.runs` by the returned `runId` is a narrow solution that avoids making the Gateway `cron.run` RPC block. The earlier zero-poll-interval issue is now guarded before enqueueing.
