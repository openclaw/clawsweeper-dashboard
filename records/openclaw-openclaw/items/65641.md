## Summary

Keep open: the PR addresses real gateway diagnostic/null-preservation gaps that are not fully on current main, but the branch is not mergeable as-is because it broadens device-less loopback auth and lacks contributor real behavior proof.

## What This Changes

The PR preserves explicit `deviceIdentity: null`, adds early connect-error text to gateway close errors, omits identity for no-token loopback calls, and updates gateway/TUI tests.

## Best Possible Solution

Revise the PR to preserve explicit constructor null and add connect-error detail to the current `GatewayTransportError` path while keeping identity omission limited to documented backend shared-auth loopback calls.

Yes, by source inspection: the branch passes null for any loopback call without token/password, and `GatewayClient` skips stored device auth when `deviceIdentity` is falsy. I did not run live gateway commands in this read-only review.

## Solution Assessment

No. The PR addresses real gaps, but broad no-auth loopback identity omission is not the narrowest maintainable fix; the auth contract should stay on the current backend shared-token/password carveout.
