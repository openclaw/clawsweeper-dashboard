## Summary

Keep this PR open. Current main still does not fully implement the two intended fixes, but the PR as written is not merge-ready because it broadens device-identity omission for loopback RPCs in a way that conflicts with current gateway auth semantics.

## What This Changes

The PR preserves explicit `deviceIdentity: null`, threads early `onConnectError` details into gateway RPC close errors, and adds gateway/TUI regression tests while changing loopback device-identity selection.

## Best Possible Solution

Revise or replace the PR with a narrow current-main patch: preserve explicit constructor `deviceIdentity: null`, capture and include early connect errors in `callGateway` transport failures, and keep current main's backend/shared-token loopback identity omission contract with focused regression coverage for paired stored-token sessions.

## Reproduction Assessment

Yes. The current-main source shows the constructor null coercion and missing `onConnectError` threading, and the PR diff shows the broader loopback omission; the stored-token auth path requires a device identity, which gives a high-confidence static reproduction path for the PR regression.

## Solution Assessment

No. The PR targets real gaps, but broad loopback/no-token identity omission is not the narrowest maintainable fix; the safer approach is to preserve explicit null only when requested and retain the documented backend shared-auth carveout.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve device identity for paired loopback sessions:** `src/gateway/call.ts:192`
  - body: This branch treats every loopback call without `token` or `password` as device-less. `GatewayClient` only loads cached per-device auth when a device identity exists, so paired local sessions that rely on stored device-token auth can no longer present credentials. Please keep the omission limited to the documented backend `gateway-client` shared-token/password path, or only honor an explicit caller-provided `deviceIdentity: null`.
