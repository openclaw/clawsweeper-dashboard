## Summary

Keep this PR open. The intended diagnostics/null-preservation fixes are still relevant on current main, but the patch is not merge-ready because it broadens loopback device-identity omission beyond the documented backend shared-auth exception and can break paired local sessions that rely on cached per-device auth.

## What This Changes

The PR preserves explicit `deviceIdentity: null`, threads early `onConnectError` details into gateway RPC close errors, adds gateway/TUI regression tests, and changes loopback device-identity selection.

## Best Possible Solution

Revise or replace the PR with a narrow current-main patch: preserve explicit constructor `deviceIdentity: null`, capture and include early connect errors in `callGateway` transport failures, and keep the backend/shared-token loopback identity omission contract with regression coverage for paired stored-token sessions.

## Reproduction Assessment

Yes. Current-main source shows the remaining null-coercion and missing `onConnectError` threading, while the PR diff and current auth code show that forcing `deviceIdentity: null` prevents stored device-token auth from being selected.

## Solution Assessment

No. The PR targets real gaps, but broad loopback/no-token identity omission is not the narrowest maintainable fix; the safer path is to honor explicit null only when requested and retain the documented backend shared-auth exception.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve device identity for paired loopback sessions:** `src/gateway/call.ts:192`
  - body: This branch treats every loopback call without `token` or `password` as device-less. `GatewayClient` only loads cached per-device auth when a device identity exists, so paired local sessions that rely on stored device-token auth can no longer present credentials. Please keep omission limited to the documented backend `gateway-client` shared-token/password path, or only honor an explicit caller-provided `deviceIdentity: null`.
  - confidence: 0.91
