## Label Justifications

- `P2`: The issue affects third-party plugin hook dispatch and has a bounded plugin API surface, but it is not a core outage and needs API-compatibility judgment.
- `impact:message-loss`: Plugin handlers can miss inbound message events when typed hook names are registered through the wrong public-looking API surface.
- `impact:session-state`: The broken hook path blocks plugin context capture and next-turn context injection workflows that depend on message and run state.

## Summary

Keep open: current main still separates `registerHook(...)` internal-hook registration from typed plugin-hook dispatch, so `registerHook("message_received")` can register without feeding the `message_received` runner. The original `api.on("before_prompt_build")` part is fixed, while the newer `allowConversationAccess` and outbound `message_sending` layers are better tracked by the linked open issues.

## What This Changes

The issue asks OpenClaw to stop silently accepting typed hook names through `registerHook(...)` when those handlers will not be dispatched, especially for `message_received` and prompt/context injection workflows.

## Best Possible Solution

Keep this as the canonical tracker for the `registerHook(...)` typed-name ambiguity, choose either a tested compatibility bridge or a clear warning/rejection path, and leave `allowConversationAccess` visibility to https://github.com/openclaw/openclaw/issues/79421 and broader outbound hook coverage to https://github.com/openclaw/openclaw/issues/50126.

## Reproduction Assessment

Yes, source-based. On current main, `registerHook("message_received")` stores `message_received` as an internal event key, while inbound runtime dispatch reads typed `registry.typedHooks` and internal `message:received` keys instead.

## Solution Assessment

Unclear. `api.on(...)` is now the clean typed-hook path, but maintainers still need to choose whether `registerHook(...)` should bridge typed names for compatibility or emit explicit diagnostics so silent registrations stop.
