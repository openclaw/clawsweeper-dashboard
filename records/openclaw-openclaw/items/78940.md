## Summary

Keep open: the direction is useful and the supplied live proof is sufficient, but the PR needs a small repair because mixed-case `infer model run` inputs now call the full model catalog loader before the lean model-run path, and the user-facing fix lacks the required changelog entry.

## What This Changes

The PR adds shared catalog-backed case-only model-ref canonicalization for `infer model run --model`, applies it before local and Gateway dispatch, and adds regression tests for plain and auth-profile refs.

## Best Possible Solution

Keep the shared canonicalization approach, but make model-run lookup side-effect-free/read-only and add the required changelog before merge.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source: current main forwards raw mixed-case `params.model` on both local and Gateway model-run paths, and the PR body supplies after-fix live local/Gateway output showing canonical `openrouter/auto`.

## Solution Assessment

Is this the best way to solve the issue? No. The shared helper is the right shape, but it should use a side-effect-free catalog lookup so `infer model run` does not regain global catalog generation before lean dispatch.
