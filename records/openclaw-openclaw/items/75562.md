## Summary

Keep open. Current main still has global/env proxy routing and provider `baseUrl` overrides, but no provider-scoped web-search forward-proxy config or `BRAVE_PROXY_URL`/`PERPLEXITY_PROXY_URL`; the remaining work needs maintainer/security design because it changes SSRF-guarded egress.

## What This Changes

The issue requests provider-scoped web-search proxy configuration, such as a Brave-only proxy URL, instead of the existing global proxy routing.

## Best Possible Solution

Add an approved web-search transport config surface that reuses guarded explicit-proxy routing, keeps public API targets on strict SSRF policy, redacts proxy secrets, and updates provider manifests, docs, generated config metadata, and Brave/Perplexity coverage together.

## Reproduction Assessment

Not applicable as a classic runtime bug reproduction; this is a feature request. Source inspection gives high confidence the capability is missing because strict provider schemas reject `webSearch.proxy`, no provider-specific proxy env path was found, and the shared web-search helper has no explicit proxy parameter.

## Solution Assessment

Unclear until maintainer/security design is approved. The requested direction is reasonable, but the maintainable fix is a generic guarded web-search proxy seam aligned with existing provider request proxy concepts, not a Brave-only raw dispatcher patch or direct reuse of Telegram's channel transport helper.
