## Summary

Keep open. Current main still lacks a provider-scoped web-search forward-proxy config or provider-specific proxy env vars, and adding one would change public config plus SSRF-guarded egress behavior that needs maintainer/security design.

## What This Changes

Request adds per-web-search-provider proxy settings and provider-specific proxy env vars so Brave or Perplexity API calls can use a local forward proxy without routing all web tools through the global proxy.

## Best Possible Solution

Add an approved web-search request transport config surface that uses guarded explicit-proxy routing, keeps public provider targets strict, redacts proxy credentials, and updates manifests, docs, generated config metadata, and provider coverage together.

## Reproduction Assessment

Not applicable as a classic runtime bug; this is a feature/config request. Source inspection is high-confidence because strict shared and provider schemas expose no proxy field, provider-specific proxy env vars are absent, and the shared web-search helper has no provider-scoped proxy parameter.

## Solution Assessment

Unclear until maintainer/security design is approved. The requested direction is reasonable, but the maintainable solution should be a generic guarded web-search transport override aligned with existing explicit-proxy contracts, not a Brave-only raw dispatcher patch.
