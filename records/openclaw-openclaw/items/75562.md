## Summary

Keep open. Current main still supports only global/env-managed proxy routing for web-search calls; there is no provider-scoped Brave or Perplexity proxy config, and adding one needs maintainer/security design because it changes SSRF-protected egress.

## What This Changes

Issue requests provider-scoped proxy configuration for web search providers so Brave or Perplexity can use a proxy without routing every web tool through the global proxy.

## Best Possible Solution

Add an approved provider-owned web-search proxy transport surface, route it through a shared guarded explicit-proxy helper, and update the plugin manifests, config docs/generated surfaces, provider tests, and SSRF policy docs together.

## Reproduction Assessment

Not applicable as a classic runtime bug reproduction. The missing capability is high-confidence by source inspection: strict schemas reject provider proxy fields, and Brave/Perplexity runtimes never resolve or pass provider-specific proxy URLs.

## Solution Assessment

Unclear pending maintainer/security design. The requested direction is valid, but the safest implementation is a guarded shared provider-proxy path rather than a Brave-only raw dispatcher patch or channel-specific makeProxyFetch reuse.
