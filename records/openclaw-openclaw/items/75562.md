## Summary

Keep open. Current main has global env/managed proxy routing for web tools, but no provider-scoped web-search proxy config or runtime plumbing; the related global proxy and Brave baseUrl items are adjacent rather than terminal, and the SSRF/private-network implications make this a maintainer/security design item.

## What This Changes

Request per-provider proxy configuration for web search providers so Brave, Perplexity, and other provider API calls can use a scoped proxy without globally proxying all web tools or bypassing SSRF guards.

## Best Possible Solution

Add an approved provider-owned web-search proxy transport surface, likely under each provider plugin's `webSearch` config and backed by a shared guarded explicit-proxy helper. The implementation should update manifests, SDK exports, config redaction/help/docs, Brave/Perplexity and shared provider tests, and coordinate the private-network policy question with #74357.

## Reproduction Assessment

Not applicable as a classic runtime bug reproduction. The missing capability is reproducible with high confidence by config and source inspection: current schemas reject proxy fields, and provider runtimes never resolve or pass a provider-specific proxy URL.

## Solution Assessment

Unclear pending maintainer/security design. The provider-scoped direction is reasonable, but the exact config and env alias shape should use plugin-owned config plus guarded explicit-proxy routing, not the local workaround that bypasses SSRF checks or a Brave-only `baseUrl` workaround.
