## Summary

Keep open. Current main still lacks provider-scoped web-search proxy configuration or provider-specific proxy environment variables; the related Brave baseUrl and global proxy fixes only cover narrower workarounds, and the remaining request needs maintainer/security design.

## What This Changes

Add per-provider HTTP proxy configuration and provider-specific proxy environment variables for web search providers such as Brave and Perplexity.

## Best Possible Solution

Define an approved generic web-search transport proxy contract that reuses guarded explicit-proxy routing, keeps public provider API targets under strict SSRF policy, redacts proxy credentials, and updates provider manifests, docs, generated config metadata, and Brave/Perplexity coverage together.

## Reproduction Assessment

Not applicable as a classic bug reproduction because this is a feature request. Source inspection gives high confidence the capability is missing from strict schemas, provider manifests, provider env lookup, and the shared web-search transport helper.

## Solution Assessment

Unclear until maintainer/security design is approved. The requested direction is reasonable, but the maintainable path is a generic guarded web-search proxy seam rather than a Brave-only raw dispatcher patch or direct reuse of Telegram transport internals.
