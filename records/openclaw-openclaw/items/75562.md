## Summary

Keep open. Current main still has global/env proxy routing and Brave/Perplexity `baseUrl` overrides, but no provider-scoped web-search forward-proxy config or provider-specific proxy environment variables; the remaining work needs maintainer/security design because it changes SSRF-guarded egress.

## What This Changes

Add per-provider web-search proxy configuration and environment overrides so Brave/Perplexity API calls can use a dedicated forward proxy without proxying all web tools.

## Best Possible Solution

Add an approved web-search transport config surface that reuses guarded explicit-proxy routing, redacts proxy secrets, keeps public provider targets on strict SSRF policy, and updates provider manifests, docs, generated metadata, and Brave/Perplexity coverage together.

## Reproduction Assessment

Not applicable as a classic runtime bug reproduction; this is a feature request. Source inspection gives high confidence the capability is absent because strict web-search schemas lack proxy keys and the shared helper accepts no explicit proxy parameter.

## Solution Assessment

Unclear until maintainer/security design is approved. The requested direction is reasonable, but the maintainable path is a generic guarded web-search proxy seam, not a Brave-only raw dispatcher patch or direct reuse of Telegram channel transport code.

## Security Review

Status: needs_attention

  - confidence: 0.9

  - body: Current main separates strict public-provider endpoints from self-hosted endpoints; provider proxy support should change transport routing only and must not allow private final targets for Brave, Perplexity, or other public API providers.
