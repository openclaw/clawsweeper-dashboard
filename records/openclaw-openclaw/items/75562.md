## Summary

Keep open. Current main supports global/env managed proxy routing and Brave/Perplexity baseUrl overrides, but it still has no provider-scoped web-search proxy config or BRAVE_PROXY_URL/PERPLEXITY_PROXY_URL path; the remaining work is a security-sensitive public config and egress-policy decision.

## What This Changes

Request provider-scoped HTTP proxy configuration for web_search providers such as Brave and Perplexity instead of relying only on global proxy routing.

## Best Possible Solution

Add an approved web-search transport/config seam that reuses guarded explicit-proxy routing, preserves strict public-provider target policy, and updates provider manifests, docs, generated metadata, and Brave/Perplexity regression tests together.

## Reproduction Assessment

Not applicable as a classic runtime bug reproduction; this is a new configuration-surface request. The missing capability is high-confidence by source inspection: strict provider schemas reject webSearch.proxy and the shared web-search helper has no provider proxy parameter.

## Solution Assessment

Unclear until maintainer/security design is approved. The requested direction is reasonable, but the safest implementation is a generic guarded provider-proxy seam rather than a Brave-only raw dispatcher patch or direct reuse of Telegram's channel-specific fetch wrapper.

## Security Review

Status: needs_attention

Summary: Provider-scoped web-search proxying is security-sensitive because it changes guarded outbound egress behavior and may carry proxy credentials.

Concerns:

- **[medium] Preserve SSRF checks for provider proxies:** `src/infra/net/fetch-guard.ts:359`
  - body: A future implementation should route provider proxy URLs through the guarded explicit-proxy path, not raw fetchWithRuntimeDispatcher, so proxy validation, target validation, redirects, and cleanup remain enforced.
  - confidence: 0.9
- **[medium] Keep public provider targets on strict policy:** `src/agents/tools/web-guarded-fetch.ts:10`
  - body: Current main separates strict public-provider endpoints from self-hosted endpoints; proxy support should change only transport routing and must not allow private final targets for Brave, Perplexity, or other public API providers.
  - confidence: 0.88
- **[medium] Redact and SecretRef proxy credentials:** `docs/reference/secretref-credential-surface.md:23`
  - body: If provider proxy URLs are added, embedded proxy credentials need the same SecretRef/redaction treatment as other user-supplied transport secrets before diagnostics, generated docs, or config metadata expose them.
  - confidence: 0.82
