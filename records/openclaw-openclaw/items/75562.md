## Summary

Keep open. Current main has global/env-managed proxy routing and Brave/Perplexity baseUrl overrides, but it still lacks provider-scoped forward-proxy config for web-search providers; the remaining design is security-sensitive because it changes SSRF-guarded outbound egress.

## What This Changes

Add provider-scoped proxy configuration or env vars for web-search providers so Brave or Perplexity can use a forward proxy without routing all web-tool egress.

## Best Possible Solution

Add an approved plugin-facing web-search proxy transport seam that reuses guarded trusted explicit-proxy routing, preserves strict public-provider target policy, and updates provider manifests, docs, generated config metadata, and tests together.

## Reproduction Assessment

Not applicable as a classic runtime bug. The missing capability is high-confidence by source inspection: strict web-search schemas reject `webSearch.proxy`, no `BRAVE_PROXY_URL` or `PERPLEXITY_PROXY_URL` path was found, and the shared web-search helper has no explicit proxy parameter.

## Solution Assessment

Unclear until maintainer/security design is approved. The requested direction is reasonable, but the safer path is a generic guarded provider-proxy helper rather than a Brave-only raw dispatcher patch or copying Telegram's channel-specific transport directly.
