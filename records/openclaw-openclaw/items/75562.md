## Summary

Keep open. Current main still supports global/env proxy routing and provider baseUrl overrides, but not provider-scoped forward-proxy config for web_search; the remaining design affects SSRF-guarded egress and should not be closed as stale or implemented.

## What This Changes

Add provider-scoped HTTP proxy configuration and env overrides for web_search providers so Brave or Perplexity can use a local proxy without proxying all web tools.

## Best Possible Solution

Add a generic, guarded provider-scoped web-search proxy transport seam that preserves strict public-provider target policy and updates plugin schemas, generated config metadata, docs, and provider tests together.

## Reproduction Assessment

Not applicable as a classic runtime bug. The missing capability is high-confidence by source inspection: strict schemas reject provider proxy fields, no requested env names exist, and web-search helpers expose no explicit proxy parameter.

## Solution Assessment

Unclear until maintainer/security design is approved. The requested direction is reasonable, but the maintainable fix should be a generic guarded transport seam rather than a Brave-only raw dispatcher patch or direct channel-transport copy.
