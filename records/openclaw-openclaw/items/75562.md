## Summary

Keep open. Current main supports global/env proxy routing and provider `baseUrl` overrides, but it still lacks a provider-scoped web-search proxy config; the requested change is a new security-sensitive config surface, not a stale or implemented request.

## What This Changes

Add per-provider web-search proxy configuration, including provider-specific config and environment-variable paths such as Brave or Perplexity proxy URLs.

## Best Possible Solution

Add an approved provider-scoped web-search transport override that reuses guarded explicit-proxy routing, preserves strict target policy for public APIs, redacts proxy secrets, and updates provider schemas, docs, and regression tests together.

## Reproduction Assessment

Not applicable as a classic runtime bug reproduction; this is a feature request. Source inspection gives high confidence the capability is missing because strict provider schemas reject `webSearch.proxy`, no provider-specific proxy env path was found, and the shared web-search helper has no explicit proxy parameter.

## Solution Assessment

Unclear until maintainer/security design is approved. The requested direction is reasonable, but the safest maintainable fix is a generic guarded web-search proxy seam, not a Brave-only raw dispatcher patch or direct reuse of Telegram's channel helper.

## Security Review

Status: needs_attention

Summary: Provider-scoped web-search proxying is security-sensitive because it changes guarded outbound egress and may carry proxy credentials.

Concerns:

- **[medium] Preserve SSRF checks for provider proxies:** `src/infra/net/fetch-guard.ts:181`
  - body: A future implementation should route provider proxy URLs through the guarded explicit-proxy path so proxy validation, target validation, redirects, and dispatcher cleanup remain enforced.
  - confidence: 0.9
- **[medium] Keep public provider targets on strict policy:** `src/agents/tools/web-guarded-fetch.ts:10`
  - body: Current main separates strict public-provider endpoints from self-hosted endpoints; provider proxy support should change only transport routing and must not allow private final targets for Brave, Perplexity, or other hosted API providers.
  - confidence: 0.86
- **[medium] Redact provider proxy credentials:** `src/config/zod-schema.agent-runtime.ts:320`
  - body: Proxy URLs commonly embed credentials, so any new config field needs sensitive metadata, redaction coverage, and careful diagnostic output before it becomes public config.
  - confidence: 0.82
