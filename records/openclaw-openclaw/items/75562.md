## Summary

Keep open. Current main still lacks provider-scoped web-search proxy config or provider-specific proxy environment variables; the merged Brave baseUrl and global proxy fixes cover related workarounds but not this requested capability, and the remaining design is security-sensitive.

## What This Changes

Request adding per-provider web-search proxy settings and environment variables so one search provider can use a forward proxy without routing all web tools through it.

## Best Possible Solution

Define an approved generic web-search provider proxy contract that reuses guarded explicit-proxy routing, preserves strict final-target SSRF checks, redacts proxy credentials, and updates provider manifests, docs, generated config metadata, and Brave/Perplexity coverage together.

## Reproduction Assessment

Not applicable as a classic bug reproduction because this is a feature request. Source inspection gives high confidence that the requested capability is missing from strict schemas, provider manifests, provider env lookup, and the shared web-search transport helper.

## Solution Assessment

Unclear until maintainer/security design is approved. The requested direction is reasonable, but the maintainable path is a generic guarded web-search proxy seam rather than a Brave-only raw dispatcher patch or direct reuse of Telegram transport internals.
