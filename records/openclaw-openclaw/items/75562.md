## Summary

Keep open. Current main still lacks a provider-scoped web-search proxy config or provider-specific proxy environment surface; global/env proxy support, managed proxy routing, and Brave baseUrl cover related workarounds but not this requested capability, and the remaining contract is security-sensitive.

## What This Changes

Request provider-scoped proxy configuration for web-search providers such as Brave and Perplexity without routing all web tools through the global proxy.

## Best Possible Solution

Define a generic web-search provider proxy contract, likely aligned with the existing request.proxy env/explicit modes, that uses guarded explicit-proxy routing, redacts secrets, settles precedence, and updates manifests, docs, generated metadata, and Brave/Perplexity tests together.

## Reproduction Assessment

Not applicable as a classic bug reproduction because this is a feature request. Source inspection on current main gives high confidence that provider-scoped proxy config/env support is missing from strict schemas, provider manifests, and the shared web-search transport helper.

## Solution Assessment

Unclear until maintainer/security design is approved. The requested direction is reasonable, but the maintainable path is a generic guarded web-search proxy seam rather than a Brave-only raw dispatcher patch or a direct copy of Telegram transport internals.
