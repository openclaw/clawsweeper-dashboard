## Summary

Keep open. Current main has global managed proxy routing and a new Brave `baseUrl` reverse-proxy path, but it still lacks a provider-scoped web-search proxy config or `BRAVE_PROXY_URL`/`PERPLEXITY_PROXY_URL` handling; the remaining work changes SSRF-protected egress and needs maintainer/security design.

## What This Changes

Add provider-scoped web-search proxy configuration so Brave, Perplexity, and other search providers can route only their own API calls through a proxy.

## Best Possible Solution

Add an approved generic web-search transport proxy contract, thread it through provider manifests/config docs/tests, and implement it via guarded explicit-proxy fetch without relaxing strict public-provider SSRF policy.

## Reproduction Assessment

Yes, by source inspection rather than a live regional network test. The proposed config/env fields are rejected or ignored by current schemas/manifests, and Brave/Perplexity runtimes never resolve provider-specific proxy URLs.

## Solution Assessment

Unclear pending maintainer/security design. The requested provider-scoped proxy direction is valid, but the safest implementation is a shared guarded explicit-proxy path rather than a Brave-only raw dispatcher patch; Brave `baseUrl` is only a partial workaround.
