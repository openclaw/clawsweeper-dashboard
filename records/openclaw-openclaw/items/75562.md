## Summary

Keep open. Current main has global env/managed proxy support for web tools, but no per-provider web-search proxy config, and the relevant provider schemas/runtimes do not accept or pass a proxy URL. The related items are adjacent rather than terminal: #27405 fixed global HTTP_PROXY behavior, #19075 tracks a Brave-only baseUrl workaround, and #74357 tracks SSRF policy scope.

## What This Changes

Request per-provider proxy configuration and env vars for web search providers so one provider's API calls can route through a proxy without changing all web tools.

## Best Possible Solution

Add a provider-owned web-search proxy config surface, likely under `plugins.entries.<provider>.config.webSearch.proxy`, plus any approved provider-specific env aliases, and route it through a guarded explicit-proxy helper that preserves SSRF checks. Update plugin manifests, config docs, provider tests, and coordinate the private-network policy question with #74357.

## Reproduction Assessment

Not applicable as a classic bug reproduction. The missing capability is reproducible by config/source inspection: current schemas reject proxy fields and Brave/Perplexity runtimes never resolve or pass a provider-specific proxy URL.

## Solution Assessment

Unclear pending maintainer/security design. The narrowest maintainable direction is provider-owned config plus a shared guarded explicit-proxy helper, not a local runtime patch that bypasses SSRF protection or a Brave-only `baseUrl` workaround.
