## Summary

Keep this PR open for maintainer review: current main and the latest release still forward configured `dimensions` with no fallback, while the PR adds a focused retry-without-dimensions path with tests and live terminal screenshot proof. The remaining blocker is compatibility policy, not a mechanical code defect: maintainers should explicitly accept the extra failed request plus local prefix truncation for rejecting OpenAI-compatible providers.

## What This Changes

The PR changes the memory-lancedb direct OpenAI-compatible embeddings client to retry without `dimensions` on dimensions-field rejection, truncate fallback vectors locally, and update regression tests plus dimension wording in docs.

## Best Possible Solution

Land the narrow direct-client fallback only after maintainers explicitly accept retry-and-local-prefix-truncation as the intended compatibility behavior for raw memory-lancedb OpenAI-compatible endpoints.

## Reproduction Assessment

Yes. Current main and v2026.6.8 source forward configured `dimensions` with no fallback, and inspected terminal screenshots show NVIDIA rejecting `body.dimensions` on current main while the branch succeeds.

## Solution Assessment

Yes with maintainer acceptance. The patch is narrow for memory-lancedb's direct client and preserves server-side dimensions for accepting providers; the broader shared-provider alternative needs an owner policy decision.
