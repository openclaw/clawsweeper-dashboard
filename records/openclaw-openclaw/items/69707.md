## Summary

This PR is still necessary because current main still forwards configured `dimensions` on the direct memory-lancedb OpenAI-compatible client with no retry path; the branch is a plausible, well-proven plugin-local fix, but it should stay open for maintainer acceptance of the retry-without-dimensions and local truncation policy.

## What This Changes

The PR adds a memory-lancedb direct-client retry that first sends `dimensions`, retries without it only on clear dimensions-field rejection, locally truncates fallback vectors, and adds focused docs and regression coverage.

## Best Possible Solution

Land this plugin-local fallback if maintainers accept retry-and-local-prefix-truncation for raw memory-lancedb endpoints, while keeping the shared provider policy in its own review path.

## Reproduction Assessment

Yes. Current main source still forwards configured `dimensions`, and the inspected terminal screenshots show a NVIDIA-compatible endpoint rejecting `body.dimensions` on current main and succeeding on the branch.

## Solution Assessment

Yes with maintainer acceptance. The patch is narrow for memory-lancedb's direct client and preserves server-side dimensions for accepting providers; the broader shared-provider alternative remains a distinct policy path, not a safe superseder today.
