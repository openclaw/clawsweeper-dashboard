## Summary

Keep open: current main and the latest release still forward configured embedding dimensions on the direct memory-lancedb OpenAI-compatible path, while this PR adds a focused retry-without-dimensions fallback with sufficient real terminal screenshot proof. The remaining blocker is maintainer acceptance of the compatibility policy for one failed request, one retry, and local prefix truncation.

## What This Changes

The branch changes memory-lancedb's direct OpenAI-compatible embeddings client to retry without `dimensions` after clear field-rejection errors, locally truncate fallback vectors, and add focused tests plus docs wording.

## Best Possible Solution

Land the plugin-local fallback only if maintainers accept retry-and-local-prefix-truncation as the intended compatibility behavior for raw memory-lancedb OpenAI-compatible endpoints.

## Reproduction Assessment

Yes. Source inspection shows current main always forwards configured `dimensions`, and inspected terminal screenshots show NVIDIA rejecting `body.dimensions` on current main while the branch succeeds.

## Solution Assessment

Yes with maintainer acceptance. The patch is narrow for memory-lancedb's direct client and preserves server-side dimensions for accepting providers; the broader shared-provider alternative remains a separate policy path, not a safe superseder today.
