## Summary

Keep open: the live proof shows the NVIDIA-compatible endpoint rejecting `dimensions` and the branch succeeding, but the patch unconditionally removes shipped server-side dimension requests that current main, the latest release, and prior DashScope regression coverage rely on.

## What This Changes

The PR changes `memory-lancedb` direct OpenAI-compatible embeddings to omit upstream `dimensions`, locally truncate returned vectors, add dimension validation coverage, and update docs.

## Best Possible Solution

Keep server-side dimensions as the default for existing direct OpenAI-compatible configs, add a documented opt-out or retry/local-truncation path for endpoints that reject the field, and prove both provider classes.

## Reproduction Assessment

Yes, there is a high-confidence reproduction path from the contributor's live NVIDIA screenshots plus current source: configure direct `memory-lancedb` embeddings with `baseUrl`, `apiKey`, and `dimensions`, then run `openclaw ltm search`. I did not rerun the live provider locally.

## Solution Assessment

No, this is not the best fix as written. It solves rejecting endpoints but removes the shipped server-side dimensions path for compatible providers instead of preserving it by default and adding a narrower compatibility escape hatch.
