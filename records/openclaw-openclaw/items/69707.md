## Summary

Keep this PR open: current main and the latest release still forward `dimensions` without a retry on the direct memory-lancedb OpenAI-compatible path, while the PR is a focused, proof-backed fix with no discrete blocking code finding. The remaining blocker is maintainer acceptance of the compatibility policy: retrying the same configured endpoint without `dimensions` and locally truncating the returned vector.

## What This Changes

The PR changes memory-lancedb direct OpenAI-compatible embeddings to retry once without `dimensions` after clear field-rejection errors, locally truncate fallback vectors, and add focused tests plus docs wording for positive integer dimensions.

## Best Possible Solution

Land the plugin-local fallback only if maintainers accept retry-and-local-prefix-truncation as the intended compatibility behavior for raw memory-lancedb OpenAI-compatible endpoints.

## Reproduction Assessment

Yes. Source inspection shows current main always forwards configured `dimensions`, and inspected terminal screenshots show NVIDIA rejecting `body.dimensions` on current main while the branch succeeds.

## Solution Assessment

Yes with maintainer acceptance. The patch is narrow for memory-lancedb's direct client and preserves server-side dimensions for accepting providers; the broader shared-provider alternative is a separate policy path, not a safe superseder today.
