## Summary

Keep this PR open, but not merge-ready: it fixes the NVIDIA/vLLM-style rejection case, yet it unconditionally removes a shipped `embedding.dimensions` pass-through contract that current docs, tests, and sibling OpenAI embedding paths still rely on.

## What This Changes

The PR changes `memory-lancedb` direct OpenAI-compatible embeddings to omit upstream `dimensions`, locally truncate returned vectors, add dimension validation coverage, and update docs wording.

## Best Possible Solution

Preserve server-side `dimensions` as the default shipped behavior, and add a narrow fallback or explicit mode for providers that reject that field with tests and proof for both provider classes.

## Reproduction Assessment

Yes, source inspection shows current main forwards `dimensions`, and the PR screenshots show a live NVIDIA endpoint rejecting that field. I did not rerun the live provider path locally.

## Solution Assessment

No, this is a plausible fix for reject-field providers but not the best merge shape because it changes the default for providers that accept or require server-side dimensions. The safer solution preserves the shipped default and narrows local truncation to the incompatible-provider path.
