## Summary

Keep open: the patch appears technically correct with sufficient real behavior proof, but it changes compatibility behavior for direct OpenAI-compatible memory-lancedb configs by adding a retry and local truncation path that needs maintainer policy approval before merge.

## What This Changes

The PR changes memory-lancedb's direct OpenAI-compatible embeddings client to retry without dimensions after clear dimensions-field rejection, trim fallback vectors locally, and adds focused tests plus docs wording.

## Best Possible Solution

Land the direct-client fallback only if maintainers accept retry-and-local-prefix-truncation as the intended compatibility behavior; otherwise move the policy to a shared provider or adapter design before merging.

## Reproduction Assessment

Yes. Current main and v2026.6.8 source forward configured dimensions with no fallback, and inspected terminal screenshots show NVIDIA rejecting body.dimensions on the same ltm search flow while the branch succeeds.

## Solution Assessment

Yes with maintainer policy acceptance. The patch is narrow for memory-lancedb's direct API-key/baseUrl client and preserves dimensions for accepting providers; the remaining question is whether failure-first retry and local truncation should be supported policy.
