## Summary

Keep open: current main and the latest release still lack the dimensions-rejection retry, and this is the canonical open direct memory-lancedb fix; the remaining blocker is maintainer acceptance of the compatibility policy around retrying and locally truncating embeddings.

## What This Changes

The PR changes memory-lancedb's direct OpenAI-compatible embedding client to send dimensions first, retry without dimensions on clear field rejection, locally truncate fallback vectors, add regression coverage, and tighten docs wording.

## Best Possible Solution

Land or decline the plugin-local fallback after maintainers choose the dimension-rejection compatibility rule, and decide separately whether the shared OpenAI-compatible provider should be aligned.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.11 post dimensions once with no retry, and the inspected terminal screenshot shows NVIDIA rejecting body.dimensions on main; I did not rerun the live NVIDIA call.

## Solution Assessment

Yes if maintainers want direct memory-lancedb recovery from dimensions-field rejection. If the intended rule is provider-wide consistency, the safer path is to align the shared OpenAI-compatible embedding provider before or alongside this change.
