## Summary

Keep open. The PR addresses a real memory-lancedb failure for NVIDIA/vLLM-style OpenAI-compatible endpoints and now includes live terminal proof, but the current patch is too broad because it removes upstream `dimensions` for every direct OpenAI-compatible request, including existing providers that rely on server-side dimensions.

## What This Changes

The PR changes memory-lancedb direct OpenAI-compatible embeddings to omit `dimensions`, locally trim returned vectors, validate dimensions as positive integers, and update docs/tests.

## Best Possible Solution

Preserve server-side dimensions for providers that support them, add a narrow provider-specific opt-out or capability path for providers that reject the field, and cover both behaviors with regression tests and live proof.

## Reproduction Assessment

Yes. Current main and the latest release still forward `dimensions`, and the inspected live terminal screenshot shows NVIDIA rejecting that field while the branch succeeds after local truncation.

## Solution Assessment

No, not yet. Local truncation is a plausible fix for rejecting providers, but making it unconditional is not the best fix because it replaces existing server-side dimension behavior for providers that support or require it.

  - body: This now omits `dimensions` for every direct OpenAI-compatible embedding request.
