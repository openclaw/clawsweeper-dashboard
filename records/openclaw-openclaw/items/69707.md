## Summary

Keep open: current main still sends `dimensions` directly with no fallback, while this PR now preserves server-side dimensions first and retries without the field only after a dimensions rejection. The remaining merge question is compatibility policy for local truncation fallback, not a cleanup close.

## What This Changes

The PR changes `memory-lancedb` direct OpenAI-compatible embeddings to retry without `dimensions` on clear provider rejection, locally truncate fallback vectors, and add focused docs/tests around dimension handling.

## Best Possible Solution

Land the direct-path fallback only after maintainers accept retry-without-dimensions plus local truncation as the compatibility contract, and track any broader generic-provider fallback separately if desired.

## Reproduction Assessment

Yes. Current main clearly sends `dimensions` with no retry, and the inspected live screenshot shows NVIDIA rejecting `body.dimensions` for the documented memory-lancedb direct endpoint setup.

## Solution Assessment

Yes, this is an acceptable narrow fix for the direct path because it preserves server-side dimensions first and only falls back on dimensions rejection. The main product judgment is whether local truncation is the desired compatibility fallback and whether a shared-provider follow-up is needed.
