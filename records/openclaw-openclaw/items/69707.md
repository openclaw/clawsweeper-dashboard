## Summary

Keep open: current main still forwards configured dimensions in the direct memory-lancedb OpenAI-compatible client, and the related open PRs do not safely supersede this plugin-local fix. ## What This Changes

The PR changes memory-lancedb's direct OpenAI-compatible embedding client to retry without `dimensions` after clear dimensions-field rejection, locally truncate fallback vectors, and update focused tests plus docs wording.

## Best Possible Solution

Land this plugin-local fallback only if maintainers accept retry-and-local-prefix-truncation as intended compatibility behavior for raw memory-lancedb OpenAI-compatible endpoints.

## Solution Assessment
