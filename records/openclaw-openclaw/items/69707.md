## Summary

Keep open: current main and the latest release still forward `dimensions` once with no direct-client retry, while this PR has focused code, regression coverage, and inspected live NVIDIA proof; the remaining blocker is maintainer acceptance of the compatibility policy.

## What This Changes

The PR changes `memory-lancedb` direct OpenAI-compatible embeddings to retry without `dimensions` after recognized field rejection, locally truncate fallback vectors, and update dimensions tests/docs.

## Best Possible Solution

Land the direct-client fallback only if maintainers accept that compatibility policy, and separately decide whether the shared OpenAI-compatible provider should adopt the same retry/truncation contract.

## Reproduction Assessment

Yes. Current main and `v2026.6.10` still forward `dimensions` once with no retry, and inspected terminal screenshots show a live NVIDIA-compatible endpoint rejecting `body.dimensions` on main and succeeding on the branch.

## Solution Assessment

Yes, if maintainers want the direct memory-lancedb path to recover from dimensions-field rejection. A broader shared-provider policy would be cleaner if the same behavior should apply to every OpenAI-compatible embedding route.
