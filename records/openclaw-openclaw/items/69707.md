## Summary

Keep this PR open. The memory-lancedb code change is a narrow, valid fix for OpenAI-compatible embedding providers that reject `dimensions`, but the branch is missing the required user-facing changelog entry for a bug fix before it is ready to land.

## What This Changes

This PR changes memory-lancedb to omit `dimensions` from direct OpenAI-compatible embedding requests, truncate returned vectors locally, validate positive integer dimensions in config/manifest metadata, update docs, and add regression tests.

## Best Possible Solution

Add the missing changelog entry, then land the narrow memory-lancedb fix after exact-head validation while preserving the current raw `/embeddings` transport, proxy setup, timeout/retry behavior, response normalization, and provider-adapter path.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: with configured dimensions on the direct OpenAI-compatible path, the outbound `/embeddings` body includes `dimensions`, which providers such as vLLM-compatible servers can reject. The PR's focused tests cover omitted upstream dimensions, local truncation, and too-short vectors.

## Solution Assessment

Yes for the code path, with one packaging gap. Omitting the non-portable upstream field and truncating normalized vectors locally is the narrow maintainable fix; provider-specific server-side dimensionality controls should stay in provider adapters or a separate explicit contract.
