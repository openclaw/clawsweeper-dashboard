## Summary

Keep this PR open: current main and the latest release still send configured memory-lancedb embedding dimensions once with no retry, while the PR provides a focused implementation and real terminal proof for the NVIDIA-style rejection case. I did not find a concrete patch defect, but the fallback/truncation behavior is compatibility-sensitive and needs maintainer policy approval before merge.

## What This Changes

The PR changes memory-lancedb's direct OpenAI-compatible embedding client to send dimensions first, retry without dimensions on clear field rejection, locally truncate fallback vectors, and add focused docs and tests.

## Best Possible Solution

Land or decline the plugin-local fallback after maintainers choose the dimensions-rejection compatibility rule, and decide separately whether the shared OpenAI-compatible provider should be aligned.

## Reproduction Assessment

Yes, source-reproducible with supporting real proof: current main and v2026.6.11 post dimensions once with no retry, and the inspected terminal screenshot shows NVIDIA rejecting body.dimensions on main.

## Solution Assessment

Yes if maintainers want direct memory-lancedb recovery from dimensions-field rejection. If the intended rule is provider-wide consistency, the safer path is to align the shared OpenAI-compatible embedding provider before or alongside this change.
