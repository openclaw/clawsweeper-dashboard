## Summary

Keep open: the NVIDIA/vLLM-style failure is real and the supplied screenshots show the branch can succeed for a provider that rejects `dimensions`, but the patch is not merge-ready because it unconditionally removes a shipped direct-provider `dimensions` contract and one new regression test asserts the wrong `memory_recall` error behavior.

## What This Changes

The PR changes `memory-lancedb` direct OpenAI-compatible embeddings to omit `dimensions` upstream, locally truncate returned vectors, tighten dimension validation/tests, and adjust docs/manifest wording.

## Best Possible Solution

Preserve upstream `dimensions` by default, retry without it only for a clear provider rejection, locally truncate only on that fallback path, and add proof/tests for both accepting and rejecting providers plus the expected unavailable-result handling.

## Reproduction Assessment

Yes. Current main source forwards `embedding.dimensions`, and the inspected main screenshot shows NVIDIA rejecting `body.dimensions`; the branch screenshots show the same user path succeeding after omitting it.

## Solution Assessment

No. Omitting `dimensions` for every direct provider is too broad; the safer fix is to preserve the current server-side path and fall back to omit-and-truncate only for providers that reject the field.
