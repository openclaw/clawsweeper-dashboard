## Summary

Keep this PR open. Current main and the latest release still forward configured `dimensions` in the direct memory-lancedb OpenAI-compatible client without a retry, while the PR adds focused retry/truncation behavior with sufficient real CLI proof; the remaining blocker is maintainer acceptance of that compatibility policy, not a concrete patch defect.

## What This Changes

The PR changes memory-lancedb's direct OpenAI-compatible embedding client to retry without `dimensions` after clear field rejection, locally truncate fallback vectors, validate dimensions, update docs, and add regression tests.

## Best Possible Solution

Keep the direct memory-lancedb fix narrow if maintainers accept the policy: server-side dimensions first, one retry only for clear dimensions-field rejection, local truncation only for long fallback vectors, and separate shared-provider follow-up.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.10 still forward configured `dimensions` with no fallback, and the inspected terminal screenshots show the live NVIDIA-compatible endpoint failing on main and succeeding on the branch.

## Solution Assessment

Yes with maintainer acceptance. The patch is narrow for memory-lancedb's direct client and preserves server-side dimensions for accepting providers; the broader shared-provider path is adjacent work, not a safe superseder for this PR.
