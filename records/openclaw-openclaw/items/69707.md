## Summary

Keep open: current main and v2026.6.10 still forward configured direct memory-lancedb embedding dimensions with no retry, while this PR contains a narrow, proof-backed fix that still needs maintainer acceptance of the compatibility policy before merge.

## What This Changes

The PR changes memory-lancedb direct OpenAI-compatible embedding calls to retry without `dimensions` on clear field rejection, locally truncate fallback vectors, tighten dimension wording/coverage, and add regression tests.

## Best Possible Solution

Land the direct-client fallback only after maintainers accept retry-without-dimensions plus local truncation as the memory-lancedb compatibility policy, or move the policy to the shared provider layer if consistency is required first.
