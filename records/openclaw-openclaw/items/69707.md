## Summary

Keep this PR open: current main and the latest release still forward `dimensions` on the direct memory-lancedb OpenAI-compatible path, while the PR adds a focused retry-without-dimensions fallback with real terminal screenshot proof. The remaining blocker is maintainer acceptance of the compatibility policy, not a concrete patch defect.

## What This Changes

The PR changes memory-lancedb's direct OpenAI-compatible embedding client to retry once without `dimensions` on clear dimensions-field rejection, locally truncate long fallback vectors, validate dimensions, update docs, and add regression tests.

## Best Possible Solution

Keep the direct memory-lancedb behavior explicit: server-side dimensions first, one retry only for clear dimensions-field rejection, local truncation only for long fallback vectors, and separate review for the shared provider path.

## Reproduction Assessment

Yes. Current main and v2026.6.10 still forward configured `dimensions` without fallback, and the inspected terminal screenshots show a NVIDIA-compatible endpoint rejecting `body.dimensions` on main and succeeding on the branch.

## Solution Assessment

Yes with maintainer acceptance. The patch is narrow for memory-lancedb's direct client and preserves server-side dimensions for accepting providers; the broader shared-provider path is adjacent work, not a safe superseder for this PR.
