## Summary

Keep open: current main and v2026.6.8 still forward configured embedding `dimensions` with no rejection fallback, while this PR is the active canonical fix candidate with sufficient real behavior proof. ## What This Changes

The PR updates memory-lancedb's direct OpenAI-compatible embeddings client to retry once without `dimensions` on field-rejection errors, locally truncate fallback vectors, add regression/config tests, and tighten docs wording.

## Best Possible Solution

Land the narrow direct-client fallback after maintainers explicitly accept retry-and-local-prefix-truncation as the intended compatibility behavior for raw memory-lancedb OpenAI-compatible endpoints.

## Reproduction Assessment

## Solution Assessment

Yes with maintainer acceptance. The patch is narrow for memory-lancedb's direct client and preserves server-side dimensions for accepting providers; the broader shared-provider alternative needs an owner policy decision.
