## Summary

Close as superseded: current main already has the explicit doctor repair path from #18544, later Telegram/open-DM policy work requires fail-closed wildcard semantics, and the PR author agreed schema-time wildcard insertion is the wrong direction.

## What This Changes

The PR adds a Telegram config schema normalizer that inserts `"*"` into top-level or account `allowFrom` when `dmPolicy` is `"open"`, plus a config validation test expecting that auto-heal.

## Best Possible Solution

Close this PR and keep the shipped fail-closed schema/runtime contract, with recovery handled only through explicit doctor/audited repair or a future maintainer-approved security design.

## Reproduction Assessment

Yes. Source inspection of the PR diff shows the new helper appends `"*"` to restrictive Telegram `allowFrom` lists before validation, while current main tests and schema reject that same configuration.

## Solution Assessment

No. Schema-time mutation is not the best solution because it silently broadens access; the maintainable direction is the shipped explicit doctor repair plus fail-closed validation/runtime behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Do not widen restrictive Telegram allowlists:** `src/config/zod-schema.providers-core.ts:153`
  - body: Current main treats a non-wildcard Telegram `allowFrom` as an authorization boundary and rejects `dmPolicy="open"` unless public access is explicit. This helper appends `"*"` before validation, so a sender-scoped list becomes public DM access instead of failing closed.
  - confidence: 0.92

## Security Review

Status: needs_attention

Summary: The diff introduces a Telegram access-control regression by silently converting restrictive allowlists into wildcard public DM access.

Concerns:

- **[high] Auto-heal can make Telegram DMs public:** `src/config/zod-schema.providers-core.ts:153`
  - body: Appending `"*"` to a non-wildcard Telegram `allowFrom` changes an operator-scoped allowlist into public bot access during schema validation, conflicting with the current fail-closed policy and documented wildcard semantics.
  - confidence: 0.92
