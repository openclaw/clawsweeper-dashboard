## What This Changes

The PR adds a `lastAssistant` guard around embedded-runner `llm_output` hook emission, normalizes hook usage to `undefined`, and adds JSON-stringified chat-content parsing with tests.

## Best Possible Solution

Land a rebased, focused hook fix that gates `llm_output` on current-attempt assistant output with regression coverage and real timeout proof, while keeping JSON-stringified content parsing in its dedicated PR.

## Reproduction Assessment

Yes, at source level: current main can select `lastAssistant` from the full session while `currentAttemptAssistant` is absent for the current attempt, and the hook path does not require current-attempt output. I did not run a live timeout scenario in this read-only review.

## Solution Assessment

No, as proposed: the hook fix should use a current-attempt output signal, and the unrelated parser work should be removed from this PR and handled through https://github.com/openclaw/openclaw/pull/39583.
