## Summary

Keep open: the underlying `llm_output` timeout bug is still source-reproducible on current main, but this PR guards on transcript-wide `lastAssistant`, so it can still fire for a no-output attempt after an earlier assistant turn; it also bundles unrelated chat-content parser work and lacks real behavior proof.

## What This Changes

The PR adds a `lastAssistant` guard and usage coercion to embedded `llm_output` hook emission, plus JSON-stringified chat-content parsing and tests.

## Best Possible Solution

Guard embedded `llm_output` emission on current-attempt assistant output or non-empty current assistant text, add focused regression coverage, and leave JSON-stringified chat-content parsing to `https://github.com/openclaw/openclaw/pull/39583`.

## Reproduction Assessment

Yes, source-reproducible. Current main computes `lastAssistant` from the whole transcript and emits `llm_output` without a current-attempt output guard, so a later timeout after a prior assistant can still emit an output hook.

## Solution Assessment

No. The PR targets a real bug, but the `lastAssistant` guard only handles empty-transcript timeouts and the bundled parser change belongs in `https://github.com/openclaw/openclaw/pull/39583`.
