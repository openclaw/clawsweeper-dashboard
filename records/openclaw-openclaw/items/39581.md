## Summary

Keep open: the embedded-runner hook issue is still source-reproducible on current main, but this PR is not ready as-is because it lacks real behavior proof and includes an unrelated chat-content parser change with concrete correctness problems.

## What This Changes

The PR adds a `lastAssistant` guard around embedded-runner `llm_output` hook emission, coerces usage to `undefined`, and adds JSON-stringified chat-content parsing/tests.

## Best Possible Solution

Land a focused embedded-runner fix that skips `llm_output` when the current attempt has no assistant output, with regression coverage; handle stringified chat-content support separately with provider-shape tests.

## Reproduction Assessment

Yes, source-reproducible: current main can reach `runLlmOutput` after a prompt timeout/no-output path because `promptErrorSource: "prompt"` is allowed and `lastAssistant` is optional. I did not run a live timeout scenario in this read-only review.

## Solution Assessment

No. The embedded-runner guard is the right direction, but it should key off current-attempt output rather than any prior `lastAssistant`, and the unrelated chat-content parser change should be split out or fixed in its own PR.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Guard on current attempt output:** `src/agents/pi-embedded-runner/run/attempt.ts:2006`
  - body: `lastAssistant` is the last assistant in the whole session snapshot, so a timeout on a later turn can still satisfy this guard with an older assistant and emit `llm_output` even though the current attempt produced no assistant output. Please guard on current-attempt output, such as `currentAttemptAssistant` or non-empty current assistant text, before firing this hook.
  - confidence: 0.88
- **[P2] Keep stringified block parsing consistent with arrays:** `src/shared/chat-content.ts:18-24`
  - body: The JSON-string path only recurses when every parsed item has both `type` and `text`, but the direct-array path skips non-text blocks and still extracts later text. A stringified mixed array such as image plus text therefore returns raw JSON here instead of the text the array path would return.
  - confidence: 0.9
