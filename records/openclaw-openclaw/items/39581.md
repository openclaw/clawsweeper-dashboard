## Summary

## What This Changes

This PR guards embedded-runner `llm_output` emission with `lastAssistant`, coerces usage to `undefined`, and adds JSON-stringified chat-content parsing plus tests.

## Best Possible Solution

Land a focused hook fix that gates embedded-runner `llm_output` on current-attempt assistant output with regression coverage, and leave JSON-stringified chat-content parsing to the separate parser PR.

## Reproduction Assessment

Yes at source level: current main records session-level `lastAssistant` separately from `currentAttemptAssistant` and emits `llm_output` without requiring current-attempt assistant output. I did not run a live timeout scenario in this read-only review.

## Solution Assessment

No. The hook fix should gate on `currentAttemptAssistant` or current-attempt assistant text, and the unrelated JSON parser work should stay in https://github.com/openclaw/openclaw/pull/39583 or a corrected parser PR.
