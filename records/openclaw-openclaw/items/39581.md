This PR should stay open because the embedded-runner bug is still present on current main, but the branch is not merge-ready: it is conflicting, lacks real timeout proof, and the unrelated chat-content change introduces a content-loss edge case.

## What This Changes

The PR adds a `lastAssistant` guard around embedded-runner `llm_output` hook calls and also adds JSON-stringified content-block parsing/tests for chat-content extraction.

## Best Possible Solution

Rebase a narrow fix onto current main that suppresses embedded-runner `llm_output` when no assistant output exists, add focused regression coverage, and drop or separately fix the chat-content parser change.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: register an `llm_output` hook, trigger an embedded-runner timeout before any assistant message exists, and the current main hook call can pass `lastAssistant: undefined`. I did not run the timeout scenario in this read-only review.

## Solution Assessment

No, not as submitted. The central guard is the right direction, but the best fix is a narrow current-main patch with regression coverage; the unrelated chat-content JSON parsing needs to be removed or corrected separately.
