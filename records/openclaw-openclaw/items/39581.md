## Summary

Keep open: the timeout-hook bug is still source-reproducible on current main, but this PR is not mergeable as-is because the guard uses session-level `lastAssistant`, includes a separate parser change that overlaps https://github.com/openclaw/openclaw/pull/39583, and still lacks real timeout proof.

## What This Changes

The branch adds a `lastAssistant` guard and usage nullish coercion to embedded-runner `llm_output` emission, plus JSON-stringified chat-content parsing and tests.

## Best Possible Solution

Land a focused current-main fix that gates embedded-runner `llm_output` on current-attempt assistant output with regression coverage, and keep JSON parsing in https://github.com/openclaw/openclaw/pull/39583.

## Reproduction Assessment

Yes by source inspection: current main computes `currentAttemptAssistant` separately from session-level `lastAssistant`, yet emits `llm_output` without requiring current-attempt output. I did not run a live timeout scenario in this read-only review.

## Solution Assessment

No. The hook fix should gate on `currentAttemptAssistant` or current-attempt assistant text, and the unrelated JSON parser work should stay in the separate parser PR.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate llm_output on current-attempt output:** `src/agents/pi-embedded-runner/run/attempt.ts:2007`
  - body: A timeout after an earlier assistant message can leave session-level `lastAssistant` truthy, so this condition can still emit `llm_output` even though the current model attempt produced no output. Gate on `currentAttemptAssistant` or current-attempt assistant text instead.
  - confidence: 0.87
- **[P2] Preserve unsupported serialized blocks as text:** `src/shared/chat-content.ts:18-24`
  - body: The stringified-array path recurses for items with `type` and `text`, but the array extractor only accepts `type === "text"`. Payloads like `[{"type":"output_text","text":"hello"}]` now collapse to `null` instead of preserving the original string, so only parse supported text blocks or leave unsupported serialized arrays unchanged.
  - confidence: 0.88
