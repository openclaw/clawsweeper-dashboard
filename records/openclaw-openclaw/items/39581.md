## Best Possible Solution

Land a focused hook fix that gates `llm_output` on current-attempt assistant output with regression coverage and real timeout proof, while leaving JSON-stringified chat-content parsing to https://github.com/openclaw/openclaw/pull/39583 or a dedicated parser PR.

## Solution Assessment

No. The hook fix should gate on `currentAttemptAssistant` or non-empty current-attempt assistant text, and the unrelated JSON parser work should be removed from this PR and handled through https://github.com/openclaw/openclaw/pull/39583.
