## Summary

Keep open: current main still has the reported `llm_output` hook path without a `lastAssistant` guard, but this PR is conflicting, lacks real timeout proof, and includes an unrelated serialized chat-content parser regression.

## What This Changes

The PR gates embedded-runner `llm_output` hook emission on `lastAssistant`, coerces missing usage to `undefined`, and adds JSON-stringified chat-content parsing with tests.

## Best Possible Solution

Land a narrow current-main patch that gates `runLlmOutput` on `lastAssistant` after the existing prompt-error hook gate, adds focused regression coverage if feasible, and handles serialized chat-content parsing separately.

## Reproduction Assessment

Yes. Source inspection on current main shows `lastAssistant` is optional and still passed to `runLlmOutput` without an output-presence guard when the hook gate passes; I did not run a live timeout scenario in this read-only review.

## Solution Assessment

No, not as submitted. The hook guard is the right direction, but the branch must be rebased onto current main and the unrelated serialized chat-content parser change should be removed or fixed separately.
