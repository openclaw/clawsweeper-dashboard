## Summary

Keep open. Current main and latest release still cap the Control UI Logs scroll container at fixed pixel heights, and https://github.com/openclaw/openclaw/pull/53916 is already the open unmerged closing implementation candidate.

## What This Changes

The issue requests that the Control UI Logs viewer use responsive vertical space instead of fixed-height `.log-stream` caps.

## Best Possible Solution

Review and finish https://github.com/openclaw/openclaw/pull/53916, or land an equivalent narrow Control UI stylesheet change, then close this issue after the responsive Logs height is on main.

## Reproduction Assessment

Yes. Source inspection on current main shows the Logs tab renders `.log-stream`, and the active desktop stylesheet caps that container at 500px, so a tall viewport with enough log rows will leave unused vertical space.

## Solution Assessment

Yes. A narrow CSS change to make `.log-stream` viewport-responsive, with real visual proof before merge, is the smallest maintainable solution for the reported layout bug.
