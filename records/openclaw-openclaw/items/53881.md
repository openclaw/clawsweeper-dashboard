## Summary

Keep open. Current main and the latest release still cap the Control UI Logs scroll container at fixed pixel heights, and open PR #53916 is already the implementation candidate that closes this issue, so the issue should stay open until that PR or an equivalent CSS fix lands.

## What This Changes

Make the Control UI Logs viewer scroll area responsive to the browser viewport instead of capping it at a fixed 500px height.

## Best Possible Solution

Add the required changelog entry to #53916, refresh or rerun the relevant checks, and land that PR or an equivalent narrow Control UI CSS change that gives `.log-stream` a viewport-responsive maximum height with a usable minimum height.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: on a tall window the rendered Logs view uses `.log-stream`, and current main applies `max-height: 500px` to that container.

## Solution Assessment

Yes. A narrow stylesheet change on `.log-stream` is the most maintainable fix because the data loading, routing, and scroll handlers already target the right container; the remaining question is the exact responsive height formula and breakpoint behavior.
