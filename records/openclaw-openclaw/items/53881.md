## Summary

Keep open. Current main still caps the Control UI Logs stream at a fixed 500px desktop height, and the linked open PR is the active closing implementation candidate rather than a reason to close before merge.

## What This Changes

The issue requests a responsive Control UI Logs scroll area that uses available browser height instead of a fixed pixel cap.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/53916, or an equivalent narrow Control UI CSS fix, then close this issue after the responsive Logs height is on main.

## Reproduction Assessment

Yes. Source inspection on current main shows the Logs tab renders `.log-stream` and the stylesheet caps that container at 500px, so a tall viewport with enough log rows will leave unused vertical space.

## Solution Assessment

Yes. A narrow CSS change that makes `.log-stream` viewport-responsive with a small-height floor is the smallest maintainable fix; the open PR follows that direction but still needs real visual proof.
