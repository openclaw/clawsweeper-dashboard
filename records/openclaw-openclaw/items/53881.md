## Summary

Keep open. Current main still applies fixed pixel height caps to the Control UI Logs scroll container, and open PR #53916 is already the implementation candidate linked with closing syntax, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that the Control UI Logs viewer use available viewport height instead of capping `.log-stream` at fixed pixel heights.

## Best Possible Solution

Review and land PR #53916, or replace it with an equivalent Control UI CSS change that makes `.log-stream` respond to available viewport height while preserving a usable minimum height and mobile behavior.
