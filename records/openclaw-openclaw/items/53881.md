## Summary

Keep open. Current main and the latest release still cap the Control UI Logs scroll container at fixed pixel heights, and open PR #53916 is already the closing implementation candidate, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that the Control UI Logs viewer use the available viewport height instead of capping `.log-stream` at a fixed 500px height.

## Best Possible Solution

Review and finish PR #53916, or land an equivalent narrow Control UI stylesheet change, so `.log-stream` uses a viewport-responsive maximum height with a usable minimum height; after merge, close this issue as implemented.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: on current main, the Logs tab renders `.log-stream`, and that container is capped at 500px on desktop, leaving unused space on tall browser windows with enough log rows.

## Solution Assessment

Yes. A narrow CSS fix on `.log-stream` is the maintainable path because the Logs data flow, scroll handler, and route already target the right container; the remaining work is release-note coverage and visual validation of the responsive height formula.
