## Summary

Keep open: current main and v2026.6.10 still have adjacent macOS HUD, WebChat, Canvas, activity, and agent-selector surfaces, but not configurable native floating per-agent desktop bubbles with persisted placement, drag/anchor or mouse-follow behavior, and click-to-agent chat routing.

## What This Changes

Request a native macOS floating bubble UI, one bubble per running agent, with configurable names, persisted positions, drag/anchor or mouse-follow behavior, and inline chat access.

## Best Possible Solution

Keep this as the native macOS bubble product/API tracker: either implement a maintainer-approved macOS bubble coordinator using canonical activity and sender-identity state, or first design a native window/plugin API that makes an external ClawHub implementation feasible.

## Reproduction Assessment

Not applicable as a bug: this is a new macOS capability request, not a failing existing behavior. Source and release inspection give high confidence that current main and v2026.6.10 do not implement the requested floating per-agent bubbles.

## Solution Assessment

Unclear until maintainers choose the product direction. Native SwiftUI/AppKit is the likely bundled owner, but persistence, click-through behavior, accessibility, activity state, and chat routing need a maintainer-designed contract rather than an autonomous patch.
