## Summary

Keep open: current main and the latest release still have adjacent macOS HoverHUD, WebChat, Canvas, Talk overlay, and session-menu surfaces, but not configurable per-agent floating bubbles with persisted positions, mouse-follow or anchor behavior, and explicit click-to-agent chat routing.

## What This Changes

Feature request for a native macOS floating per-agent bubble UI with chat routing, persisted positions, configurable names, and drag/anchor or mouse-follow controls.

## Best Possible Solution

If bundled, design a native macOS bubble coordinator under the app that uses canonical agent/session activity, app-owned persistence, and explicit session routing; otherwise first define a native macOS UI/window plugin API for an external implementation.

## Reproduction Assessment

Not applicable as a bug: this is a new macOS capability request, not a failing existing behavior. Source inspection gives a high-confidence current-main comparison showing the requested floating-bubble surface is absent.

## Solution Assessment

Unclear until maintainers choose the product direction. Native SwiftUI/AppKit is the likely owner if bundled, but persistence, click-through behavior, accessibility, and session routing need a maintainer-designed contract rather than a narrow autonomous patch.
