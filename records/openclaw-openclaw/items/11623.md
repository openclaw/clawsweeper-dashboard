## Summary

Keep open. Current main still has adjacent macOS HoverHUD, WebChat, Canvas, Talk overlay, and activity-store surfaces, but it does not implement configurable per-agent floating bubbles with persisted positions, drag/anchor or mouse-follow behavior, and per-agent chat routing; existing review comments also kept this open for the same product/API gap.

## What This Changes

The issue requests configurable floating macOS agent bubbles with drag or mouse-follow behavior, persisted position/state, and click-to-chat routing for each running agent.

## Best Possible Solution

Define and implement a native macOS companion surface, or first add a native UI/window plugin API, that maps agent/session activity to persistent per-agent bubbles and routes each bubble to the selected chat.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug. Source/docs inspection on current main shows adjacent macOS UI surfaces but no per-agent floating bubble implementation.

## Solution Assessment

Unclear. A native macOS implementation is plausible, but maintainers need to choose the built-in UX, persistence model, click-through behavior, and plugin/API boundary before this becomes a narrow implementation task.
