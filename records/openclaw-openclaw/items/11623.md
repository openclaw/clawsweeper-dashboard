## Summary

Keep open. Current main still has adjacent macOS menu-bar HUD, WebChat, Talk overlay, and activity surfaces, but none implements configurable floating per-agent bubbles with persisted placement and per-agent chat routing; the related WebUI/activity trackers do not supersede the native macOS companion UX request.

## What This Changes

Request to add floating macOS per-agent bubbles with configurable labels, persisted placement, drag/anchor or mouse-follow behavior, and click-to-chat routing.

## Best Possible Solution

Define the native macOS companion UX and API boundary, then implement persistent per-agent bubble state, placement, and selected-agent chat routing in the macOS app or expose a supported native window plugin API first.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug. Source inspection on current main gives high confidence that the requested per-agent floating bubble implementation is absent.

## Solution Assessment

Unclear until maintainers decide the built-in UX versus plugin/API boundary. A native macOS implementation is plausible, but the maintainable path needs an explicit state, persistence, input, and chat-routing design first.
