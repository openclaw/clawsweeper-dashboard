## Summary

Keep open. Current main still has adjacent macOS HoverHUD, WebChat, Canvas, Talk overlay, and activity-store surfaces, but it does not implement persistent configurable per-agent floating bubbles with drag/anchor or mouse-follow behavior and per-agent chat routing; prior issue discussion already reached the same keep-open conclusion.

## What This Changes

Request to add native macOS floating per-agent bubbles with configurable names, persisted positions, drag/anchor or mouse-follow behavior, and click-to-chat routing.

## Best Possible Solution

Define and implement a native macOS companion surface, or first add a native UI/window plugin API, that maps agent/session activity to persistent per-agent bubbles and routes each bubble to its selected chat.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug. Source/docs inspection on current main shows adjacent macOS UI surfaces but no per-agent floating bubble implementation.

## Solution Assessment

Unclear. A native macOS implementation is plausible, but maintainers need to choose the built-in UX, persistence model, click-through behavior, and plugin/API boundary before this becomes a narrow implementation task.
