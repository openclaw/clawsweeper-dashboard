## Summary

Keep open: current main and the latest release still have adjacent macOS HoverHUD, WebChat, Canvas, Talk, and session surfaces, but they do not implement configurable per-agent floating bubbles with persisted positions, drag/anchor or mouse-follow behavior, and explicit click-to-agent chat routing.

## What This Changes

The issue asks for configurable, always-on-top macOS floating bubbles per running agent with persisted position, mouse-follow or anchor controls, and click-to-agent chat routing.

## Best Possible Solution

If bundled, design a native macOS bubble/window coordinator that derives state from canonical agent/session activity, persists positions through an approved app-state owner, supports drag/anchor or mouse-follow behavior, and routes chat to the selected agent; otherwise define a native macOS UI/window plugin API for an external implementation.

## Reproduction Assessment

Not applicable as a bug: this is a new macOS capability request, not a failing existing behavior. Source inspection gives a high-confidence current-main comparison showing the requested floating-bubble surface is absent.

## Solution Assessment

Unclear until maintainers choose the product direction. Native SwiftUI/AppKit is the likely owner if bundled, but persistence, click-through behavior, accessibility, and session routing need a maintainer-designed contract rather than a narrow autonomous patch.
