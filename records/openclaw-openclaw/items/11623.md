## Summary

Keep open: current main and the latest release have adjacent native macOS HoverHUD, WebChat, Canvas, and session-menu surfaces, but they still do not implement configurable per-agent floating bubbles with persisted positions, drag/anchor or mouse-follow behavior, and explicit click-to-agent chat routing.

## What This Changes

The issue requests a native macOS floating bubble per running agent, with configurable labels, drag/anchor or mouse-follow behavior, persisted position/state, and click-to-chat routing.

## Best Possible Solution

If maintainers want this bundled, design a native macOS bubble/window coordinator that uses canonical agent/session activity, approved app-state persistence, and explicit session routing; otherwise first define a native macOS UI/window plugin API for an external implementation.

## Reproduction Assessment

Not applicable as a bug: this is a new macOS capability request, not a failing existing behavior. Source inspection gives a high-confidence current-main comparison showing the requested floating-bubble surface is absent.

## Solution Assessment

Unclear until maintainers choose the product direction. Native SwiftUI/AppKit is the likely owner if bundled, but persistence, click-through behavior, accessibility, and session routing need a maintainer-designed contract rather than a narrow autonomous patch.
