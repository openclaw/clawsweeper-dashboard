## Summary

Keep open: current main still has adjacent macOS HoverHUD, WebChat, Canvas, and plugin UI descriptor surfaces, but not configurable per-agent floating bubbles with persisted placement, drag/anchor or mouse-follow behavior, and click-to-agent chat routing.

## What This Changes

This issue requests configurable floating macOS agent bubbles that show each running agent, persist position, support mouse-follow/anchor behavior, and open chat to the selected agent.

## Best Possible Solution

Define a maintainer-approved native macOS bubble/window coordinator or a native macOS window plugin API that derives from canonical agent/session activity, persists placement through an approved state owner, and routes chat to the selected agent.

## Reproduction Assessment

Not applicable as a bug: this is a new macOS capability request, not a failing existing behavior. Source inspection gives high confidence that the requested floating-bubble surface is absent on current main.

## Solution Assessment

Unclear until maintainers choose the product direction. Native SwiftUI/AppKit is a plausible owner, but persistence, click-through behavior, accessibility, activity state, and chat routing need a maintainer-designed contract rather than an autonomous patch.
