## Summary

Current main still lacks the requested per-agent floating-bubble UX, and the related native-macOS umbrella explicitly leaves floating bubbles as a narrower follow-up. The request fits the companion-app direction, but persistent always-on-top interaction, agent-to-session routing, accessibility, and state ownership require a maintainer product decision before implementation.

## What This Changes

Add configurable, draggable, always-on-top per-agent bubbles to the native macOS app with agent-specific chat access.

## Best Possible Solution

If maintainers approve the UX, add a native macOS bubble/window coordinator that derives state from canonical agent and session activity, persists user preferences through the appropriate SQLite-backed owner, and routes each bubble to an explicit agent session.

## Reproduction Assessment

Not applicable: this issue requests a new macOS capability rather than reporting broken existing behavior; source inspection confirms the requested surface is absent.

## Solution Assessment

Partly: native SwiftUI/AppKit is the correct owner, but JSON sidecar state is not the best persistence design; the implementation should reuse canonical activity, session-routing, window, and SQLite-backed state seams.
