## Summary

Keep open. Current main still has adjacent macOS HoverHUD, WebChat, Canvas, Talk overlay, and activity-store surfaces, but it does not implement persistent configurable per-agent floating bubbles with drag/anchor or mouse-follow behavior and per-agent chat launch/routing; prior issue discussion reached the same keep-open conclusion.

## What This Changes

The issue requests native macOS floating per-agent bubbles with configurable labels, drag/anchor or mouse-follow behavior, persisted state, and click-to-chat routing.

## Best Possible Solution

Keep this open for a maintainer-designed native macOS feature, or first define a native UI/window plugin API that maps agent/session state to persistent per-agent bubbles and routes each bubble to the selected chat.

## Reproduction Assessment

Not applicable: this is a feature request. The high-confidence verification path is source/docs inspection on current main, and the checked macOS surfaces show no per-agent floating bubble implementation.

## Solution Assessment

Unclear. The native macOS direction is plausible, but maintainers need to choose the built-in UX, persistence model, click-through behavior, and plugin/API boundary before this becomes a narrow implementation task.
