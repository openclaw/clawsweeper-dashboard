## Summary

Keep this open: current main still has adjacent macOS status, HoverHUD, WebChat, Canvas, and work-activity surfaces, but it does not implement configurable per-agent floating bubbles with persisted positions, drag/anchor or mouse-follow behavior, and explicit click-to-agent chat routing.

## What This Changes

The issue requests a native macOS floating per-agent bubble UI with configurable names, drag/anchor or mouse-follow behavior, persisted positions, and click-to-chat routing.

## Best Possible Solution

If maintainers want this bundled, design a native macOS bubble/window coordinator that uses canonical agent/session activity, approved app-state persistence, and explicit session routing; otherwise first add a native macOS UI/window plugin API so it can live outside core.

## Reproduction Assessment

Not applicable as a bug: this is a new macOS capability request, not a failing existing behavior. Source inspection does confirm the requested floating-bubble surface is absent on current main.

## Solution Assessment

Partly: native SwiftUI/AppKit under the macOS app is the likely owner if this is bundled, but JSON sidecar state is not the best persistence design under OpenClaw's storage policy.
