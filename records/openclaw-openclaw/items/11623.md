## Summary

Current `main` still lacks the requested per-agent floating macOS bubbles. Existing native surfaces are single-session, menu-bar-anchored overlays, and the related native-app issue was closed because the app already exists—not because this narrower feature was implemented—so this remains a valid low-priority product-design item.

## What This Changes

Add configurable always-on-top macOS bubbles for individual agents, including activity state, positioning, anchoring or mouse-follow behavior, and per-agent chat access.

## Best Possible Solution

Design a canonical native macOS overlay around existing agent/session activity and WebChat routing, with app-owned persisted layout state and explicit accessibility, focus, Spaces, and multi-display behavior; consider a plugin API only if maintainers want native UI ownership outside the app.

## Reproduction Assessment

Not applicable: this is a new-capability proposal, not a broken-behavior report. Current-main source inspection does provide high-confidence confirmation that the requested multi-agent bubble surface is absent.

## Solution Assessment

Unclear. A native macOS implementation is the likely maintainable direction, but the prototype's JSON state and interaction ideas are not yet a complete production design for session identity, persistence, routing, accessibility, and window behavior.
