Current main still does not implement the requested macOS per-agent floating bubbles; the native macOS app, HoverHUD, WebChat, Canvas, and session surfaces are adjacent but do not provide configurable always-on-top bubbles with persisted positions and per-agent chat routing.

## Best Possible Solution

If maintainers approve the UX, add a native macOS bubble/window coordinator that derives state from canonical agent/session activity, persists positions through an approved app-state owner, and routes each bubble to an explicit agent session.
