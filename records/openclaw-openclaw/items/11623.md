## Summary

Current main still lacks the requested floating per-agent macOS interface. The native app now has useful adjacent pieces—multi-session work activity, HoverHUD, and session-keyed WebChat—but they remain menu-bar/single-controller presentations and do not provide personalized always-on-top bubbles, persisted placement, follow/anchor interaction, or one-click routing for multiple agents. The open gateway activity proposal could supply part of the state contract, but it does not supersede this native UX request.

## What This Changes

Add configurable native macOS bubbles for multiple agents with visible activity, persistent placement, follow/anchor interactions, and direct chat routing.

## Best Possible Solution

Define bubbles as a deliberate native macOS product surface with an explicit agent-versus-session identity model, consume canonical gateway activity events, persist only UI preferences through established app settings or database state, and reuse session-keyed WebChat routing without replacing its existing presentations.

## Reproduction Assessment

Not applicable; this issue requests a new macOS interaction mode rather than reporting broken existing behavior. Current-main source inspection does provide high-confidence confirmation that the requested UI is absent.

## Solution Assessment

No, not exactly as proposed. Native SwiftUI/AppKit is the correct ownership boundary, but JSON sidecar state and an unspecified agent/session activity model should be replaced with canonical persistence and a shared gateway contract.
