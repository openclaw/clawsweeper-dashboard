## Summary

Keep open. Current main still has adjacent macOS menu-bar HUD, single-session WebChat panel/window, Canvas, Talk overlay, and activity/session surfaces, but it does not implement configurable persistent per-agent floating bubbles with drag/anchor or mouse-follow behavior and per-agent chat routing; the prior discussion reached the same keep-open conclusion.

## What This Changes

Request configurable always-on-top macOS floating bubbles for running agents, with names, persisted positions, drag/anchor or mouse-follow behavior, and click-to-chat routing.

## Best Possible Solution

Keep this open for a maintainer-designed native macOS feature or a generic native UI/window plugin API that maps agent/session state to persistent configurable per-agent bubbles and routes each bubble to the selected agent chat.

## Reproduction Assessment

Not applicable: this is a feature request. The high-confidence verification path is source/docs inspection against current main, which shows adjacent overlays and session menus but no per-agent floating bubble implementation.

## Solution Assessment

Unclear. A native macOS app implementation is plausible, but maintainers need to choose the built-in UX versus a new native UI/window plugin API before code work is the best path.
