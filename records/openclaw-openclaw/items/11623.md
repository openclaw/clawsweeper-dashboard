## Summary

Keep open: current main and v2026.6.8 have adjacent macOS HoverHUD, WebChat, Canvas, and activity surfaces, but they do not implement configurable per-agent floating bubbles with persisted positions, mouse-follow or anchor behavior, and click-to-agent chat routing.

## What This Changes

Request a native macOS floating bubble surface where each running agent has a configurable on-screen orb with drag/anchor behavior and direct chat access.

## Best Possible Solution

If bundled, design a native macOS bubble/window coordinator that derives state from canonical agent/session activity, persists positions through an approved app-state owner, and routes chat to the selected agent; otherwise define a native macOS UI/window plugin API for an external implementation.

## Reproduction Assessment

Not applicable as a bug: this requests a new macOS capability, not a failing existing behavior. Source inspection gives a high-confidence current-main and latest-release comparison showing the requested floating-bubble surface is absent.

## Solution Assessment

Unclear until maintainers choose the product direction. Native SwiftUI/AppKit is the likely bundled owner, but persistence, click-through behavior, accessibility, and session routing need a maintainer-designed contract rather than a narrow autonomous patch.
