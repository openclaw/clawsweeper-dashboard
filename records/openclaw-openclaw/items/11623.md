## Summary

Keep open: current main and v2026.6.10 have adjacent macOS HoverHUD, WebChat, activity, and plugin-surface pieces, but they do not implement configurable per-agent floating bubbles with persisted placement, drag/anchor or mouse-follow behavior, and click-to-agent chat routing.

## What This Changes

The issue requests native macOS floating per-agent bubbles with configurable names, active state, drag/anchor or mouse-follow behavior, persisted position/state, and click-to-agent chat entry.

## Best Possible Solution

Define a maintainer-approved native macOS bubble/window coordinator or native macOS window plugin API that derives from canonical agent/session activity, persists placement through an approved state owner, and routes chat to the selected agent.

## Reproduction Assessment

Not applicable as a bug: this is a new macOS capability request, not a failing existing behavior. Source inspection gives high confidence that the requested floating-bubble surface is absent on current main and the latest release.

## Solution Assessment

Unclear until maintainers choose the product direction. Native SwiftUI/AppKit is a plausible owner, but persistence, click-through behavior, accessibility, activity state, and chat routing need a maintainer-designed contract rather than an autonomous patch.
