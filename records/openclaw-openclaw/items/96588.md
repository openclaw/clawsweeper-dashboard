## Summary

Keep open: current main and the latest release still sort matching model ids with plain lexicographic `localeCompare`, and several open PRs are candidate fixes for this issue rather than merged provenance for closure.

## What This Changes

The issue asks OpenClaw to resolve partial model aliases to the numerically newest matching version instead of the lexicographically highest model id.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/96609 or an equivalent tested numeric-aware comparator fix, then close this issue after the fix merges.

## Reproduction Assessment

Yes, with a high-confidence source reproduction: `parseModelPattern("opus", [claude-opus-4-9, claude-opus-4-10])` reaches the alias bucket and current main sorts those ids lexicographically. I did not run a repro command because this review is read-only and avoided local artifact creation.

## Solution Assessment

Yes, numeric-aware collation at the existing `tryMatchModel` sort points plus focused regression coverage is the narrowest maintainable fix. The issue should stay open until one linked fix PR merges.
