## Summary

This issue should stay open: current main still sorts matching model ids with plain lexicographic `localeCompare`, so the reported double-digit minor-version rollover can select the older model. The fix is narrow and belongs in the existing agent session model resolver with focused regression coverage.

## What This Changes

The issue requests numeric-aware model alias resolution so partial aliases like `opus` choose the newest numeric version instead of the lexicographically highest model id.

## Best Possible Solution

Update the resolver comparator to use numeric-aware ordering for version-like ids, keep the dated-version sibling path consistent, and add focused regression coverage without adding config or provider policy.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to resolve `parseModelPattern("opus", [claude-opus-4-9, claude-opus-4-10])`; the current comparator orders the older `4-9` ahead of `4-10`.

## Solution Assessment

Yes. Numeric-aware collation at the existing `tryMatchModel` ordering point is the narrowest maintainable fix because exact provider/model matching and configured alias lookup can remain unchanged.
