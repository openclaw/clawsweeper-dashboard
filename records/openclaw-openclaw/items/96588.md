## Summary

This issue should stay open: current main still sorts matching model IDs with plain lexicographic `localeCompare`, so the reported double-digit minor-version rollover would select the older model. The fix is narrow and belongs in the existing agent session model resolver with focused regression coverage.

## What This Changes

Request numeric-aware ordering when alias or partial model resolution chooses among multiple versioned model IDs.

## Best Possible Solution

Update the existing resolver comparator to use numeric-aware ordering for version-like IDs and add focused regression coverage without adding config, fallback, or migration surface.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to resolve `parseModelPattern("opus", [claude-opus-4-9, claude-opus-4-10])`; the current comparator orders the older `4-9` ahead of `4-10`.

## Solution Assessment

Yes. Numeric-aware collation at the existing `tryMatchModel` ordering point is the narrowest maintainable fix because exact provider/model matching and configured alias lookup can remain unchanged.
