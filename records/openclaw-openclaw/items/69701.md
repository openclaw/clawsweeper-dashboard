## Summary

Keep open. Current main and the latest release still lack the Windows guard, while this PR directly targets the linked gateway hang and has no blocking functional or security finding. The remaining repo-policy cleanup is a changelog entry before normal maintainer review and landing.

## What This Changes

The PR adds a Windows-only skip of the `::1` loopback bind in gateway listen-host resolution and expands `src/gateway/net.test.ts` coverage for win32 and non-Windows behavior.

## Best Possible Solution

Land this narrow win32 guard, focused regression coverage, and a changelog entry, or land an equivalent fix that prevents the Windows gateway from binding both `127.0.0.1` and `::1` on the same port while preserving non-Windows dual-loopback behavior.

## Reproduction Assessment

Yes. The linked bug report gives concrete Windows 11 gateway and `curl` steps, and current main statically maps that path to dual loopback binding when `::1` is bindable. This review did not run a live Windows repro because it is read-only.

## Solution Assessment

Yes, with one process cleanup. The PR is the narrowest maintainable code fix for the reported default-loopback hang; it should add the required changelog entry before merge.
