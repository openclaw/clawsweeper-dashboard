## Summary

This PR should stay open: current main still has newest-first tool-card result matching, and the branch is a focused fix for that UI pairing bug. I did not find a blocking code/security finding, but contributor-supplied real Control UI proof is still missing before merge.

## What This Changes

The branch changes Control UI tool-card result matching to prefer exact tool-call IDs and otherwise fall back to the earliest unmatched same-name card, plus adds a changelog entry.

## Best Possible Solution

Land the matcher change after redacted real Control UI proof is added, with focused sequential same-name regression coverage if maintainers want a stronger guard.

## Reproduction Assessment

Yes, by source inspection against current main. A message content array with two same-name tool calls followed by their same-name results hits the reverse scan and can attach the first result to the later card before the earlier exact or oldest match is considered.

## Solution Assessment

Yes. Exact-ID-first matching with earliest-unmatched name fallback is the narrowest maintainable fix because it preserves explicit ID pairing while correcting sequential same-name fallback ordering.
