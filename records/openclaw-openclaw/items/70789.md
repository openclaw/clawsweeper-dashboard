## What This Changes

The PR changes Control UI tool-card result lookup to prefer exact tool-call IDs before falling back to the first unmatched same-name card, and adds a changelog entry.

## Best Possible Solution

Land the exact-ID-first and earliest-unmatched-name matcher after redacted real Control UI proof is added; a focused sequential same-name regression test would strengthen the guard.

## Reproduction Assessment

Yes, by source inspection against current main. With same-name call A, call B, result A, result B, the reverse scan can attach result A to the later card before reaching the intended earlier or exact match, leaving the first expanded card without output.

## Solution Assessment

Yes for the code change. Exact-ID-first matching with an earliest-unmatched same-name fallback is the narrowest maintainable fix because it preserves explicit ID pairing while correcting fallback ordering.
