## Summary

Keep open. Current main still has the reverse-scan pairing behavior, and this PR is the right narrow code fix, but it still needs the required changelog entry before it is ready to merge cleanly.

## What This Changes

The PR changes Control UI tool-card extraction so tool results first match exact card IDs and otherwise attach to the first unmatched same-name tool call.

## Best Possible Solution

Add the required changelog entry, then land this PR or an equivalent narrow patch in `ui/src/ui/chat/tool-cards.ts`; a focused sequential same-name regression test would make the proof stronger without broadening the UI renderer change.

## Reproduction Assessment

Yes. A message content array with two same-name `tool_use` blocks followed by two same-name `tool_result` blocks reproduces the pairing error by source inspection because current main scans backward for the first unmatched same-name card.

## Solution Assessment

Yes. Exact-ID-first matching with first-unmatched same-name fallback is the narrow maintainable fix for this renderer helper; the remaining merge cleanup is changelog policy and optional focused regression coverage, not a broader rewrite.
