## What This Changes

The PR changes Control UI tool-card result matching to prefer exact IDs and otherwise pair same-name results with the earliest unmatched card, plus a changelog entry.

## Best Possible Solution

Land the exact-ID-first matcher after the branch is refreshed and redacted Control UI proof shows expanded outputs for sequential same-name calls; add a focused regression test if maintainers want stronger coverage.

## Reproduction Assessment

Yes, by source inspection against current main. Two same-name tool calls followed by their results can make the reverse scan attach a result to the newest unmatched card while expanded rendering depends on that matched card's outputText.

## Solution Assessment

Yes for the code direction: exact-ID-first matching with earliest-unmatched name fallback is the narrowest maintainable fix for this matcher. The merge path still needs real behavior proof, conflict resolution, and optionally a stronger regression test.
