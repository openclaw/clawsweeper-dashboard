## Summary

Keep open. Current main still uses the reverse same-name matcher that can mispair sequential same-name tool results, while this PR’s exact-ID-first and earliest-unmatched fallback is a narrow Control UI fix with no blocking review finding.

## What This Changes

This PR updates Control UI tool-card result matching to prefer exact card IDs before falling back to the earliest unmatched same-name card, and adds a changelog fix entry.

## Best Possible Solution

Land the targeted matcher and changelog change after current-head CI is green, with a focused sequential same-name regression test if maintainers want stronger coverage before merge.

## Reproduction Assessment

Yes. A content array with two same-name `tool_use` blocks followed by two same-name `tool_result` blocks exercises current main’s reverse scan and pairs the first result with the later card first by source inspection.

## Solution Assessment

Yes. Exact-ID-first matching with earliest-unmatched same-name fallback is the narrow maintainable fix for this UI pairing bug; the missing exact-shape test is a coverage gap, not a demonstrated correctness blocker.
