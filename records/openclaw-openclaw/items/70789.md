## Summary

Keep open. Current main still has the reverse same-name fallback that can mispair sequential tool results, and the PR’s matcher change is a narrow, correct fix, but this user-facing Control UI fix still needs the required changelog entry before it is ready to merge.

## What This Changes

The PR updates Control UI tool-card result matching so exact card IDs are preferred and otherwise same-name results attach to the first unmatched tool call instead of scanning from newest to oldest.

## Best Possible Solution

Add the required changelog entry, then land this PR or an equivalent narrow patch in `ui/src/ui/chat/tool-cards.ts`; adding a focused sequential same-name regression test would make the fix easier to preserve without changing the product boundary.

## Reproduction Assessment

Yes. A message content array with two same-name `tool_use` blocks followed by two same-name `tool_result` blocks reproduces the pairing problem by source inspection because current main scans pending cards backward for the name-only fallback.

## Solution Assessment

Yes. Exact-ID-first matching with a first-unmatched same-name fallback is the narrow maintainable fix for this renderer helper; the remaining safer follow-through is changelog and focused regression coverage, not a broader UI rewrite.
