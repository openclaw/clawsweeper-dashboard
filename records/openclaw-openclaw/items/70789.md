## Summary

Keep open. Current main still uses newest-first same-name result matching, so the reported Control UI pairing bug is not implemented on main; the PR is a narrow, plausible fix that should receive maintainer review rather than be auto-closed or replaced.

## What This Changes

The PR changes `ui/src/ui/chat/tool-cards.ts` so tool results prefer exact card-id matches and otherwise attach to the first unmatched same-name tool card.

## Best Possible Solution

Land this PR or an equivalent narrow patch in `ui/src/ui/chat/tool-cards.ts`, ideally with a focused node test covering sequential same-name result pairing plus exact-id priority.

## Reproduction Assessment

Yes. A content array with two same-name `tool_use` blocks followed by two same-name `tool_result` blocks reproduces the source-level bug on current main because the matcher scans backward and chooses the newest unmatched card first.

## Solution Assessment

Yes. The PR's exact-id-first, first-unmatched-name fallback is the narrow maintainable fix for this renderer helper; the safer enhancement is stronger regression coverage, not a different product boundary.
