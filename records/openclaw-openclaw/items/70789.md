## Summary

Keep open: current main still has the reverse-scan matcher that can mispair sequential same-name tool results, and the PR’s two-pass matcher appears to be the narrow code fix. Merge should still wait for contributor-provided real Control UI proof.

## What This Changes

This PR changes Control UI tool-result matching to prefer exact tool-call ID matches before falling back to the earliest unmatched same-name tool card, and adds a changelog entry.

## Best Possible Solution

Land the matcher change after redacted real Control UI proof is added; add focused sequential same-name regression coverage if maintainers want a stronger guard.

## Reproduction Assessment

Yes, by source inspection against current main. Two same-name tool calls followed by their results can make the reverse scan attach the first result to the newest unmatched card, while expanded rendering only shows output from the matched card.

## Solution Assessment

Yes for the code fix: exact-ID-first matching with earliest-unmatched name fallback is the narrowest maintainable change because it preserves explicit ID pairing while correcting sequential same-name fallback ordering. The merge path still needs real behavior proof from the contributor.
