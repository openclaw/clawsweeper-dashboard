## Summary

Keep this open: current main still shows the reported safeguard compaction model bug, and https://github.com/openclaw/openclaw/pull/73704 is an open closing PR for the same issue. The later user comments also confirm the issue is still affecting a Homebrew/macOS plugin-provider setup and are waiting on that PR rather than asking for a separate cleanup close.

## What This Changes

Safeguard auto-compaction should honor `agents.defaults.compaction.model` and configured compaction providers instead of using the active session model first.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73704 if its PR gates pass, then let the merge close this issue.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: safeguard runtime registration stores the session model and the hook prefers `ctx.model`, so `agents.defaults.compaction.model` cannot win in initialized safeguard sessions.

## Solution Assessment

Yes. The linked PR's direction is the narrow maintainable fix: resolve the configured compaction target before registering the safeguard runtime, then prefer the resolved runtime model in the hook while preserving session-model fallback.
