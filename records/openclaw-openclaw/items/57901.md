## What This Changes

The issue asks for safeguard compaction to honor `agents.defaults.compaction.model` instead of always using the session model.

Yes, as a source-level reproduction: docs promise an exact `agents.defaults.compaction.model` override, while current main registers the session model into safeguard runtime and the hook prefers `ctx.model`. I did not run a live provider compaction session.

## Solution Assessment

Yes. The best current fix is the linked focused PR or equivalent code that resolves the configured compaction target before safeguard runtime registration and prefers that runtime model in the hook; the one-line precedence swap alone is too narrow.
