## Summary

Keep open. Current main still appears to populate safeguard compaction runtime with the active session model and the hook still prefers the session model, while the linked closing PR remains open as the active fix path.

## What This Changes

The issue reports that safeguard auto-compaction ignores `agents.defaults.compaction.model` and uses the active session model instead.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73704, or an equivalent focused fix, then close this issue after current main resolves both safeguard runtime registration and hook model precedence.

## Reproduction Assessment

Yes, at source level. Current main still registers the active session model in the safeguard runtime and then prefers `ctx.model`, so a different configured compaction model cannot win in initialized safeguard sessions.

## Solution Assessment

Yes. The narrow maintainable fix is to resolve the configured compaction target before safeguard runtime registration and prefer the resolved runtime model in the hook; the linked open PR follows that direction.
