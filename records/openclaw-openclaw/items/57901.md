## Summary

Keep open: current main and the latest v2026.6.10 release still route safeguard compaction through the active session model, while the linked fix PR remains open and unmerged.

## What This Changes

The issue reports that safeguard compaction ignores `agents.defaults.compaction.model` and uses the active session model instead.

## Best Possible Solution

Land or replace the linked fix so safeguard registration resolves the configured compaction target before storing runtime state, and the hook prefers that runtime model while preserving no-override fallback behavior.

## Reproduction Assessment

Yes, at source level: docs define `agents.defaults.compaction.model` as the override, but current safeguard registration stores the session model and the hook chooses `ctx.model` first. I did not run a live provider compaction session.

## Solution Assessment

Yes, the linked PR direction is the right shape: resolve the configured target before runtime registration and then prefer `runtime.model` in the hook. The earlier one-line precedence swap alone is too narrow because the registered runtime model can already be the session model.
