## Summary

Keep open. Current main and `v2026.5.20` still register safeguard compaction with the active session model and then prefer `ctx.model`, so the documented `agents.defaults.compaction.model` override remains ineffective in this path; https://github.com/openclaw/openclaw/pull/73704 is the active linked fix path rather than a reason to close early. Later discussion found a separate stale `installs.json` cause for one third-party plugin symptom, but that does not resolve the core safeguard model-selection bug.

## What This Changes

The issue asks for safeguard auto-compaction to honor `agents.defaults.compaction.model` instead of always using the active session model.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73704, or an equivalent focused fix, then close this issue once main resolves both safeguard runtime registration and hook model precedence.

## Reproduction Assessment

Yes, at source level. Current main registers the active session model in safeguard runtime and then prefers `ctx.model`, so a different configured compaction model cannot win in initialized safeguard sessions.

## Solution Assessment

Yes. The narrow maintainable fix is to resolve the configured compaction target before safeguard runtime registration and prefer the resolved runtime model in the hook; the linked open PR follows that direction.
