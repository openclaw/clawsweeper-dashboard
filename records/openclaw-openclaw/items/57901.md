## Summary

Keep open. Current main still has the reported safeguard compaction path, and the open closing-fix path is https://github.com/openclaw/openclaw/pull/73704 rather than a merged implementation.

## What This Changes

Safeguard compaction should honor `agents.defaults.compaction.model` instead of using the active session model when the two differ.

## Best Possible Solution

Land or replace the focused fix that resolves the configured compaction target before safeguard runtime registration, prefers the resolved runtime model in the hook, and proves the behavior with regression coverage plus observed safeguard auto-compaction evidence.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main stores the session model in safeguard runtime registration and then prefers `ctx.model`, so a configured different compaction model cannot win in initialized safeguard sessions.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing compaction target resolver before safeguard runtime registration and then prefer the resolved runtime model in the hook; https://github.com/openclaw/openclaw/pull/73704 is already the open implementation path.
