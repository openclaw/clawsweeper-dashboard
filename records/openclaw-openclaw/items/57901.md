## Summary

Keep open. Current main and latest release still route safeguard compaction through the session model, and the open linked fix PR is the active implementation path rather than a reason to close early.

## What This Changes

The issue reports that safeguard compaction should honor `agents.defaults.compaction.model` instead of using the active session model.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73704, or an equivalent focused fix, then close this issue once main resolves both safeguard runtime registration and hook model precedence.

## Reproduction Assessment

Yes, at source level. Current main registers the active session model in safeguard runtime and then prefers `ctx.model`, so a different configured compaction model cannot win in initialized safeguard sessions.

## Solution Assessment

Yes. The narrow maintainable fix is to resolve the configured compaction target before safeguard runtime registration and prefer the resolved runtime model in the hook; the linked open PR follows that direction.
