## Summary

Keep open. Current main still has the safeguard runtime registration and hook precedence that make the documented `agents.defaults.compaction.model` override lose to the session model, and https://github.com/openclaw/openclaw/pull/73704 is an open closing PR for this exact issue. The `compaction-weighted` comments add useful third-party provider evidence, but they reinforce the same fix path rather than making this closeable now.

## What This Changes

The issue reports safeguard compaction ignoring `agents.defaults.compaction.model` and using the active session model/provider instead.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73704 or an equivalent focused fix that resolves the compaction target before safeguard runtime registration and prefers that runtime model in the hook, then close this issue after merge.

## Reproduction Assessment

Yes, as a source-level reproduction: current main documents `agents.defaults.compaction.model` as an override, but safeguard registration stores the active session model and the hook prefers `ctx.model`. I did not execute a live compaction run in this read-only checkout.

## Solution Assessment

Yes, the open PR direction is the narrow maintainable fix because it reuses the existing compaction target resolver before safeguard runtime registration and then lets the resolved runtime model win. Merge should wait for normal PR review and conflict resolution rather than closing this issue early.
