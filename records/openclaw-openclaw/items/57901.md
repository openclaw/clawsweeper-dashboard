## Summary

Keep open. Current main still registers the safeguard runtime with the active session model and the hook still prefers `ctx.model`, so the documented `agents.defaults.compaction.model` override cannot win in initialized safeguard sessions; https://github.com/openclaw/openclaw/pull/73704 is the open closing-fix path.

## What This Changes

The issue reports that safeguard-mode compaction ignores `agents.defaults.compaction.model` and uses the active session model instead.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/73704 after adequate proof, resolving the configured compaction target before safeguard runtime registration and giving that runtime model precedence in the hook.

## Reproduction Assessment

Yes. Source inspection on current main shows safeguard runtime registration stores the session model and the hook then prefers `ctx.model`, so a configured different compaction model cannot win in initialized safeguard sessions; I did not run a live provider compaction.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing compaction target resolver before safeguard runtime registration and then prefer the resolved runtime model in the hook; the linked open PR is already the implementation path.
