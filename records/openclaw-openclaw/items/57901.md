## Summary

Keep open. Current main still has the reported safeguard compaction path: runtime registration stores the active session model and the hook prefers `ctx.model`, while the open closing-fix PR remains the active implementation path.

## What This Changes

The issue asks safeguard-mode compaction to honor `agents.defaults.compaction.model` instead of using the active session model.

## Best Possible Solution

Land or replace the focused fix that resolves the configured compaction target before safeguard runtime registration and gives the resolved runtime model precedence in the hook.

## Reproduction Assessment

Yes, as a source-level reproduction. Current main stores the session model in safeguard runtime registration and then prefers `ctx.model`, so a configured different compaction model cannot win in initialized safeguard sessions.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing compaction target resolver before safeguard runtime registration and then prefer the resolved runtime model in the hook; https://github.com/openclaw/openclaw/pull/73704 is already the open implementation path.
