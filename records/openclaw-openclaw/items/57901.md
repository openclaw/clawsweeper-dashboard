## Summary

Keep open. Current main and the latest release still show the reported safeguard compaction model-precedence bug, and the active closing path is the open fix PR at https://github.com/openclaw/openclaw/pull/73704.

## What This Changes

The issue asks safeguard auto-compaction to honor `agents.defaults.compaction.model` instead of using the active session model.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73704, or an equivalent focused fix, then close this issue after the safeguard runtime and hook paths are fixed on main.

## Reproduction Assessment

Yes, at source level. Current main registers the active session model in safeguard runtime and then prefers `ctx.model`, so a configured different compaction model cannot win in initialized safeguard sessions.

## Solution Assessment

Yes. The narrow maintainable fix is to resolve the configured compaction target before safeguard runtime registration and prefer the resolved runtime model in the hook; the linked open PR follows that direction.
