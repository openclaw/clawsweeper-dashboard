## Summary

Keep open. Current main and the latest release still show the safeguard compaction path preferring the active session model, while the open linked fix PR remains the active owner for the exact bug.

## What This Changes

The issue asks for safeguard compaction to honor `agents.defaults.compaction.model` instead of the active session model.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73704, or an equivalent narrow fix that resolves the safeguard runtime target from the configured compaction model and then prefers that runtime model in the hook.

## Reproduction Assessment

Yes, at source level: docs and schema define `agents.defaults.compaction.model` as the compaction override, while current main registers the session model and the safeguard hook prefers `ctx.model`. I did not run a live provider-backed compaction session.

## Solution Assessment

Yes, the linked fix direction is the narrow maintainable path: resolve the safeguard runtime target where the embedded extension runtime is registered, then prefer `runtime.model` in the hook.
