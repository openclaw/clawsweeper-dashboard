## Label Justifications

- `P2`: This is a documented compaction override bug affecting users with custom compaction models, but it is not a core outage or security emergency.
- `impact:auth-provider`: The issue is about model/provider selection for compaction ignoring the configured model.
- `impact:session-state`: The bug sits in compaction context/transcript handling and can affect session compaction outcomes.

## Summary

Keep open. Current main and release v2026.5.18 still show the safeguard compaction path registering and preferring the active session model, while the open focused fix path is https://github.com/openclaw/openclaw/pull/73704. Later plugin-install comments add useful context but do not remove the core model-precedence bug in source.

## What This Changes

Safeguard compaction should honor `agents.defaults.compaction.model` instead of using the active session model.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73704, or an equivalent focused fix, then close this issue once main resolves both safeguard runtime registration and hook model precedence.

## Reproduction Assessment

Yes, at source level: current main registers the active session model in safeguard runtime and then prefers `ctx.model`, so a different configured compaction model cannot win in initialized safeguard sessions. I did not run a live provider compaction session in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to resolve the configured compaction target before safeguard runtime registration and prefer the resolved runtime model in the hook; the linked open PR follows that direction.
