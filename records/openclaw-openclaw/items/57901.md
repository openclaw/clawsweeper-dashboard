## Summary

Keep open. Current main still has the safeguard compaction model-precedence bug, and the active resolution path is the open closing PR at https://github.com/openclaw/openclaw/pull/73704 rather than a merged fix.

## What This Changes

The issue requests that safeguard compaction honor `agents.defaults.compaction.model` instead of using the active session model.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73704 or an equivalent focused fix, then close this issue after the fix is on main.

## Reproduction Assessment

Yes, at source level. With safeguard mode and a configured compaction model different from the session model, current main registers the session model in safeguard runtime and then the hook prefers `ctx.model`, so the override cannot win when `ctx.model` is present.

## Solution Assessment

Yes. The narrow maintainable fix is to resolve the configured compaction target before safeguard runtime registration and prefer the resolved runtime model in the hook; the open closing PR follows that path.
