## Summary

Keep open: current main and the latest release still carry the safeguard path that can ignore the configured compaction model, and the open linked fix PR has not merged yet.

## What This Changes

The issue requests that safeguard auto-compaction honor `agents.defaults.compaction.model` instead of using the active session model.

## Best Possible Solution

Land the linked fix PR or an equivalent narrow change that resolves the safeguard runtime target before registration and makes `runtime.model` win over the session model in the hook.

## Reproduction Assessment

Yes. Current source gives a high-confidence reproduction path: with safeguard mode and a configured compaction model different from the session model, the runtime is registered with the session model and the hook prefers `ctx.model`.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing compaction target resolver before safeguard runtime registration and prefer `runtime.model` in the hook, without adding another config surface.
