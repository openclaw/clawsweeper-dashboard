## Summary

Keep open. Current main still has the reported safeguard compaction behavior, and the active closing path is the open PR at https://github.com/openclaw/openclaw/pull/73704 rather than a merged fix.

## What This Changes

Safeguard auto-compaction should honor `agents.defaults.compaction.model` instead of preferring the active session model.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73704 or an equivalent focused fix, then close this issue after the safeguard runtime and hook paths are fixed on main.

## Reproduction Assessment

Yes at source level. Current main stores the session model in safeguard runtime registration and then prefers `ctx.model`, so a configured different compaction model cannot win in initialized safeguard sessions; I did not run a live provider compaction session.

## Solution Assessment

Yes. The narrow maintainable direction is to resolve the configured compaction target before safeguard runtime registration and prefer the resolved runtime model in the hook; the linked open PR follows that path but should finish normal review/proof before merge.
