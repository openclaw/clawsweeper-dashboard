## Summary

Keep open. Current main and the latest release still register safeguard compaction from the active session model and the hook still prefers `ctx.model`, while the open closing PR https://github.com/openclaw/openclaw/pull/73704 remains the active fix path.

## What This Changes

Fix safeguard compaction so `agents.defaults.compaction.model` is honored instead of using the active session model.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73704, or an equivalent focused fix, then close this issue after main resolves both safeguard runtime registration and hook model precedence.

## Reproduction Assessment

Yes, at source level. Current main still registers safeguard runtime with the active session model and the hook prefers `ctx.model`, so a different configured compaction model cannot win in initialized safeguard sessions.

## Solution Assessment

Yes. The maintainable fix is to resolve the configured compaction target before safeguard runtime registration and then prefer that resolved runtime model in the hook; the linked open PR follows that shape.
