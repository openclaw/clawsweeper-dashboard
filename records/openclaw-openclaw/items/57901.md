## Summary

Keep open: current main still has the reported safeguard model-precedence path, and PR #73704 is an open implementation candidate paired with this issue, so this should not be closed before that PR is reviewed, merged, or rejected.

## What This Changes

The issue asks for safeguard compaction to honor `agents.defaults.compaction.model` instead of selecting the active session model when both are available.

## Best Possible Solution

Review PR #73704 as the canonical implementation candidate: the desired end state is for safeguard runtime registration to receive the resolved compaction provider/model/auth context, for the hook to prefer the configured runtime model when present, and for regression tests to cover both configured-override and no-override fallback behavior.

## Reproduction Assessment

Yes. A focused test can set `ctx.model` to the session model and `runtime.model` to a different configured compaction model, then assert safeguard auth/summarization uses the runtime model; current main’s `ctx.model ?? runtime?.model` line explains the failure path.

## Solution Assessment

Yes for the proposed direction, pending PR review. Resolving the compaction target before safeguard runtime registration and preferring that runtime model in the hook is narrower than changing the public config contract or adding a second model-selection path.
