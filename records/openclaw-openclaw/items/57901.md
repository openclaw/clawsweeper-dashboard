## Summary

Keep open. Current main still has the reported safeguard compaction path: runtime registration uses the active session model and the hook prefers `ctx.model`, while https://github.com/openclaw/openclaw/pull/73704 is the open closing-fix path and newer user comments confirm the same behavior on shipped versions.

## What This Changes

Make safeguard auto-compaction honor `agents.defaults.compaction.model` and configured compaction providers instead of using the active session model.

## Best Possible Solution

Land or replace the focused fix that resolves the configured compaction target before safeguard runtime registration, prefers the resolved runtime model in the hook, and proves the behavior with regression coverage plus real safeguard auto-compaction evidence.

## Reproduction Assessment

Yes. Source inspection on current main shows safeguard runtime registration stores the session model and the hook then prefers `ctx.model`, so a configured different compaction model cannot win in initialized safeguard sessions; I did not run a live provider compaction.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing compaction target resolver before safeguard runtime registration and then prefer the resolved runtime model in the hook; https://github.com/openclaw/openclaw/pull/73704 is already the open implementation path.
