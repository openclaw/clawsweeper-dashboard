## Summary

This issue is still necessary because current main and the latest release both keep the reported safeguard model-selection path: the hook prefers `ctx.model`, and safeguard runtime registration stores the active session model instead of resolving `agents.defaults.compaction.model`.

## What This Changes

The issue asks for safeguard auto-compaction to honor `agents.defaults.compaction.model` instead of always using the active session model.

## Best Possible Solution

Use one canonical compaction-target resolver for safeguard registration and explicit compaction paths, then prove the configured override, no-override fallback, provider registration, and auth-profile behavior together.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: enable safeguard compaction with a different `agents.defaults.compaction.model`, then the hook still prefers `ctx.model` and the runtime is registered from `params.model`. I did not run a live compaction session because this is a read-only cleanup review.

## Solution Assessment

Yes, the best fix direction is to reuse the existing compaction target resolver for safeguard runtime registration and make the hook prefer the runtime target over `ctx.model`. The open linked PR appears to follow that shape, but it still needs maintainer review and a clean merge state.
