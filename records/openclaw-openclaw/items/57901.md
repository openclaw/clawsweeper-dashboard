## Summary

Keep this issue open. Current main still lets safeguard compaction use the active session model before any configured compaction model, while the docs promise that `agents.defaults.compaction.model` delegates summarization to a separate exact model; https://github.com/openclaw/openclaw/pull/73704 is an open candidate fix but has not merged.

## What This Changes

The issue requests that safeguard-mode compaction honor the configured `agents.defaults.compaction.model` instead of silently using the session model.

## Best Possible Solution

Land a clean focused fix that resolves the configured compaction target before safeguard runtime registration, prefers that runtime target during summarization, preserves no-override fallback, and tests the sibling compaction paths plus auth-profile dropping.

## Reproduction Assessment

Yes, at source level: current main registers the safeguard runtime with the session model and then the hook prefers `ctx.model` over `runtime.model`. I did not run a live provider compaction session.

## Solution Assessment

Yes, the two-layer fix shape is the best direction: resolve the configured compaction target before safeguard runtime registration and then let the runtime compaction target win in the hook. The earlier one-line precedence-only PR was too narrow because registration still supplied the session model.
