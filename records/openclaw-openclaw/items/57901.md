## Summary

Keep open. Current main and the latest release still register safeguard compaction with the active session model and then prefer `ctx.model`, while https://github.com/openclaw/openclaw/pull/73704 is an open, unmerged closing PR for the same root cause. The later `installs.json` workaround comment is useful plugin setup context, but it does not remove the core safeguard model-precedence bug.

## What This Changes

The issue reports that safeguard compaction ignores `agents.defaults.compaction.model` and uses the active session model instead.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73704, or an equivalent focused fix, then close this issue once main resolves both safeguard runtime registration and hook model precedence.

## Reproduction Assessment

Yes, at source level: current docs define `agents.defaults.compaction.model` as the compaction override, while current main registers the active session model and the hook prefers `ctx.model`. I did not run a live provider-backed compaction session.

## Solution Assessment

Yes. The narrow maintainable fix is to resolve the configured compaction target before safeguard runtime registration and prefer the resolved runtime model in the hook; the linked open PR follows that direction.
