## Label Justifications

- `P2`: This is a normal-priority compaction/model-routing bug with real cost and session-quality impact but a bounded agent-runtime surface.
- `impact:session-state`: Safeguard compaction rewrites session context, and routing it to the wrong model can degrade or mis-shape preserved session state.
- `impact:auth-provider`: The bug is about configured provider/model selection and associated auth-profile routing for compaction summarization.

## Summary

Keep open: current main still wires the safeguard runtime to the active session model and the hook still prefers `ctx.model`, so the documented `agents.defaults.compaction.model` override remains ineffective in safeguard mode. The open linked PR is the right fix path, but it is not merged and is currently conflicting.

## What This Changes

The issue asks for safeguard compaction to honor `agents.defaults.compaction.model` instead of using the active session model.

## Best Possible Solution

Land a refreshed version of https://github.com/openclaw/openclaw/pull/73704, or an equivalent narrow fix, that resolves the configured compaction target before safeguard runtime registration and lets that runtime model win in the hook while preserving no-override fallback and auth-profile safety.

## Reproduction Assessment

Yes. Source inspection on current main shows the safeguard runtime stores the active session model and the hook prefers `ctx.model`, while docs define `agents.defaults.compaction.model` as a different-model override.

## Solution Assessment

Yes, the linked PR's approach is the best fix shape once refreshed: reuse the existing compaction target resolver before runtime registration, then prefer the resolved runtime model in the safeguard hook.
