Keep this issue open because current `main` still has the reported safeguard compaction routing problem, and the linked fixing PR is open and conflicting rather than merged.

## What This Changes

The issue reports that safeguard compaction ignores `agents.defaults.compaction.model` and uses the active session model instead.

## Best Possible Solution

Land a cleaned, mergeable version of https://github.com/openclaw/openclaw/pull/73704 or an equivalent focused fix that resolves the compaction target before safeguard runtime registration and lets the configured runtime model win in the safeguard hook.

## Reproduction Assessment

Yes. Source inspection shows the active run passes the session model into safeguard runtime registration and the hook then prefers `ctx.model` over `runtime.model`, while docs require `agents.defaults.compaction.model` to be honored.

## Solution Assessment

Yes. The best fix is the linked PR's two-layer shape: resolve the configured compaction target before safeguard runtime registration, then prefer the runtime model inside the safeguard hook with fallback coverage for unset overrides.
