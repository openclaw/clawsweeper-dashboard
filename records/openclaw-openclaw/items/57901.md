## Summary

Keep open: current main and the latest release still show the safeguard compaction path using the session model/runtime wiring that makes the configured compaction model ineffective, and the linked fix PR is still open rather than merged.

## What This Changes

The issue reports that safeguard-mode compaction ignores `agents.defaults.compaction.model` and routes summarization through the active session model instead.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73704 after conflict repair, or merge an equivalent focused fix that resolves the configured compaction provider/model before safeguard runtime registration and prefers that runtime model in the hook.

## Reproduction Assessment

Yes. Source inspection shows the current safeguard hook prefers `ctx.model` and the embedded extension runtime is registered with the session model, while docs promise `agents.defaults.compaction.model` should override compaction summarization.

## Solution Assessment

Yes. The best fix is the linked PR's two-layer repair: resolve the configured compaction target before safeguard runtime registration and make the safeguard hook prefer the runtime model, while preserving fallback behavior when no override is set.
