## Summary

Keep open. Current `main` and the latest release still show the safeguard compaction model-precedence bug, and https://github.com/openclaw/openclaw/pull/73704 is an open closing PR rather than a merged fix; the latest `compaction-weighted` plugin comment explains one reporter's separate stale local install record, but it does not resolve the original configured-model precedence path.

## What This Changes

Fix safeguard auto-compaction so `agents.defaults.compaction.model` is honored instead of defaulting to the active session model.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73704, or an equivalent focused fix that resolves the compaction target before safeguard runtime registration and lets the resolved runtime model win in the hook.

## Reproduction Assessment

Yes, at source level: current `main` registers the session model into safeguard runtime and then the hook prefers `ctx.model`, so a different configured compaction model cannot win in initialized safeguard sessions. I did not run a live provider compaction session.

## Solution Assessment

Yes. The narrow maintainable solution is the linked PR's approach: reuse the compaction target resolver before safeguard runtime registration, then prefer the resolved runtime model in the safeguard hook while preserving session-model fallback when no override exists.
