Keep open: current main and the latest release still contain the reported safeguard compaction model-selection path, and the linked fix is an open pull request rather than merged implementation. The issue should remain open until the fix PR lands or maintainers choose a different repair path.

## What This Changes

The issue reports that safeguard auto-compaction ignores `agents.defaults.compaction.model` and uses the active session model instead.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/73704 or replace it with an equivalent focused fix that resolves the configured compaction target before safeguard runtime registration and lets the runtime model take precedence in the hook.

## Reproduction Assessment

Yes from source inspection. Current main still registers the safeguard runtime with the active run model and then resolves the hook model as `ctx.model ?? runtime?.model`, while docs say `agents.defaults.compaction.model` should override the session model; I did not run a live provider compaction in this read-only review.

## Solution Assessment

Yes, the linked fix direction appears to be the right repair shape: reuse the existing compaction target resolver for safeguard runtime registration and then prefer `runtime.model` in the hook. A simple hook-only swap is too narrow because current registration can already put the session model into `runtime.model`.
