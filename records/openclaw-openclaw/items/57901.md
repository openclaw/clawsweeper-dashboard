## Summary

Keep open. Current main and the latest release still use the active session model in the safeguard compaction path, while https://github.com/openclaw/openclaw/pull/73704 is an open closing PR for the same bug and should remain the active fix path.

## What This Changes

The issue reports that safeguard compaction ignores `agents.defaults.compaction.model` and should use the configured compaction model/provider instead of the active session model.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73704, or an equivalent focused fix that resolves the configured compaction target before safeguard runtime registration and then prefers that runtime model in the hook.

## Reproduction Assessment

Yes, at source level. Current main still registers the active session model in the safeguard runtime and the hook still prefers `ctx.model`, while docs say an explicit compaction model override should be honored.

## Solution Assessment

Yes. The maintainable fix is to reuse the existing compaction target resolver before safeguard registration and then prefer the resolved runtime model in the hook; the linked open PR follows that shape.
