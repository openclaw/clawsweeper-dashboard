## What This Changes

The issue reports that safeguard-mode compaction ignores `agents.defaults.compaction.model` and routes compaction through the active session model instead.

## Best Possible Solution

Keep this issue open and resolve it by reviewing, updating, and landing the linked safeguard compaction fix or an equivalent narrow maintainer-approved repair.

## Reproduction Assessment

Yes from source inspection: current `main` still prefers `ctx.model` over `runtime.model` in the safeguard hook, and safeguard registration stores the session model instead of resolving the compaction override. I did not run a live compaction session in this read-only review.

## Solution Assessment

Yes, the issue identifies the right fix boundary: safeguard registration should resolve the compaction target and the hook should prefer the runtime compaction model when configured. The linked PR appears to target that boundary, but it must be reconciled because GitHub reports it as conflicting.
