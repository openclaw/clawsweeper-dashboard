## Summary

Keep open. Current main still has the reported safeguard compaction path, and the active closing-fix path is https://github.com/openclaw/openclaw/pull/73704 rather than a merged implementation.

## What This Changes

Safeguard auto-compaction should honor `agents.defaults.compaction.model` instead of selecting the active session model.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/73704 with regression coverage and real behavior proof, then close this issue through the merged fix.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: with safeguard mode and a different configured compaction model, current main registers the session model into safeguard runtime and the hook then prefers `ctx.model`. I did not run a live provider compaction session.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing compaction target resolver before safeguard runtime registration and then prefer the resolved runtime model in the hook; the open PR is already the active implementation path.
