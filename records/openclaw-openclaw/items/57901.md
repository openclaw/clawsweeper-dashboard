## Summary

Keep open: current main and the latest release still wire safeguard compaction to the active session model, and the open linked fix PR at https://github.com/openclaw/openclaw/pull/73704 has not merged yet.

## What This Changes

The issue reports that safeguard auto-compaction ignores the documented `agents.defaults.compaction.model` override and uses the active session model instead.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/73704 with a focused fix that resolves the compaction target before registering safeguard runtime and makes the hook prefer the configured runtime model when present.

## Reproduction Assessment

Yes. Source inspection on current main shows safeguard runtime registration stores the active session model, and the hook then prefers `ctx.model` over `runtime.model` despite docs promising a compaction-model override.

## Solution Assessment

Yes. The maintainable fix shape is to reuse the existing compaction target resolver before safeguard runtime registration and then let the hook use that runtime model ahead of the session model; a one-line hook precedence swap alone is too narrow.
