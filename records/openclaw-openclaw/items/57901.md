## Summary

Keep open. Current main still registers the safeguard runtime with the active session model and the hook still prefers `ctx.model`; the open linked fix at https://github.com/openclaw/openclaw/pull/73704 has closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that safeguard compaction ignores `agents.defaults.compaction.model` and uses the active session model instead.

## Best Possible Solution

Land or replace the focused fix that resolves the configured compaction target before safeguard runtime registration and prefers that runtime model in the hook, then close this issue after merge.

## Reproduction Assessment

Yes, from source inspection: current main registers the active session model in safeguard runtime and then the hook prefers `ctx.model`, so a different configured compaction model cannot win in initialized safeguard sessions. I did not run a live compaction against a real provider in this read-only review.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable fix: reuse the existing compaction target resolver before safeguard runtime registration and then prefer the resolved runtime model in the hook. The issue should stay open while maintainers review or land that implementation path.
