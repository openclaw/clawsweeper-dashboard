## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Fail closed when no scoped Tool Search catalog matches:** `src/agents/tool-search.ts:604-609`
  - body: `resolveCatalog` returns the only process-global catalog when the current control tool has no matching session, session key, or agent scope. A stale or under-scoped Tool Search control can therefore search or call another run's catalog; remove the singleton fallback and require a scoped match.
  - confidence: 0.83
- **[P2] Reject ambiguous name-based catalog calls:** `src/agents/tool-search.ts:671-673`
  - body: `findEntry` accepts an id or name and returns the first match, while client tools may share names with plugin tools. If the model passes a name instead of the catalog id, Tool Search can silently call the wrong implementation; require ids or fail when a name maps to more than one entry.
  - confidence: 0.84
- **[P2] Preserve direct-tool error results for bridged calls:** `src/agents/pi-embedded-runner/run/attempt.ts:2558-2565`
  - body: Cataloged OpenClaw/plugin tools are stored before `toToolDefinitions` wraps them, and this executor calls the raw `tool.execute`. Throws that direct tools convert into structured `status: "error"` results will reject `tool_search_code`/`tool_call` instead, so route through the same adapter or normalize errors here.
  - confidence: 0.8
