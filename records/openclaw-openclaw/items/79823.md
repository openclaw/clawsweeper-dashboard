## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Do not pass host bridge functions into VM code:** `src/agents/tool-search-code-mode.ts:161-163`
  - body: The sandbox installs outer-realm functions as `openclaw.tools.*`, so model code can call `openclaw.tools.search.constructor("return process")()` and recover the child Node process despite `codeGeneration.strings=false`. The bridge methods need to be exposed without host-function constructor access.
  - confidence: 0.97
- **[P1] Cancel bridged tool executions on timeout:** `src/agents/tool-search-code-mode.ts:656`
  - body: The timeout path kills only the child process, but `runtime.call` may already be awaiting `entry.tool.execute(...)` in the parent with no cancellation signal. A timed-out `tool_search_code` can therefore leave side-effecting tools running after the turn failed.
  - confidence: 0.94
- **[P2] Generate unique IDs for bridged tool calls:** `src/agents/tool-search-code-mode.ts:656`
  - body: Every call to the same catalog entry uses `tool_search_code:<entry.name>` as the toolCallId. Client-tool completion is keyed by toolCallId, so repeated calls to one client tool in a code-mode run can overwrite or collapse earlier invocations.
  - confidence: 0.93
- **[P2] Fail closed when the session catalog is missing:** `src/agents/tool-search-code-mode.ts:485-487`
  - body: If lookup by sessionId, sessionKey, or agentId misses, `resolveCatalog` falls back to the sole global catalog in memory. A call with missing context can bind to another run's tool catalog instead of failing closed.
  - confidence: 0.89
- **[P2] Return structured errors from bridged tool calls:** `src/agents/tool-search-code-mode.ts:656`
  - body: `ToolSearchRuntime.call` lets catalog tool exceptions reject `tool_search_code` or `tool_call`. The normal direct tool path converts non-abort failures into structured `status: "error"` tool results so the model can recover from validation or runtime failures.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The PR introduces model-authored JavaScript execution and still has concrete sandbox, cancellation, and session-boundary issues.

Concerns:

- **[high] Model code can reach the child Node process:** `src/agents/tool-search-code-mode.ts:161`
  - body: The VM receives host-created bridge functions; their constructor can create code in the child process realm and expose `process`, bypassing the intended bridge-only capability boundary.
  - confidence: 0.97
- **[high] Timed-out calls can continue parent-side tool execution:** `src/agents/tool-search-code-mode.ts:656`
  - body: Killing the child process does not abort an in-flight `entry.tool.execute(...)` promise in the parent, so side-effecting tools may continue after `tool_search_code` reports timeout.
  - confidence: 0.94
- **[medium] Catalog fallback can cross session boundaries:** `src/agents/tool-search-code-mode.ts:485`
  - body: The fallback to the only in-memory catalog can expose or execute a different active run's tools when the tool context lacks session identifiers.
  - confidence: 0.89
