## Summary

Keep open: current main does not contain core Tool Search, and this recent maintainer-labeled core feature PR still has concrete catalog-boundary issues that should block merge.

## What This Changes

The PR adds a core Tool Search capability with `tools.toolSearch` config, code/structured bridge tools, catalog compaction for OpenClaw/MCP/client tools, docs, tests, and a gateway E2E harness.

## Best Possible Solution

Keep Tool Search as a per-run scoped core capability only if the bridge fails closed on missing catalog scope and requires unique catalog ids, or explicitly rejects ambiguous names.

## Reproduction Assessment

Yes for the review findings: PR-head source shows the singleton catalog fallback and first-match name lookup, and the attempt path allows client/plugin name coexistence. The feature itself is new capability work, not a current-main bug reproduction.

## Solution Assessment

No. Core Tool Search may be a reasonable direction, but this implementation should not merge until catalog resolution is strictly scoped and bridged calls cannot resolve ambiguous tool names.
