## Summary

Keep open: this protected-label core feature PR has useful real behavior proof, but the latest head still has merge-blocking security and runtime-contract issues around model-authored code execution and bridged tool calls.

## What This Changes

The PR adds PI Tool Search with `tools.toolSearch`, code and structured bridge tools, catalog compaction for OpenClaw/MCP/client tools, config/docs/changelog updates, tests, and a gateway E2E harness.

## Best Possible Solution

Revise the branch so Tool Search remains PI-scoped but uses a real network-denied execution boundary or disables code mode, aborts or drains bridge work on settlement, fails closed on catalog scope, rejects ambiguous name calls, preserves normal tool error results, and reruns targeted tests plus gateway E2E proof.

## Reproduction Assessment

Yes for the review findings: PR-head source inspection and local Node help show the network permission gap, thenable bridge lifecycle, catalog fallback, ambiguous lookup, and direct-tool error-contract mismatch. The feature itself is not a bug reproduction case.

## Solution Assessment

No. Core catalog compaction is a plausible direction, but this implementation is not the narrowest maintainable merge shape until the sandbox boundary and bridge execution semantics match OpenClaw's existing PI tool contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Block network from the code-mode subprocess:** `src/agents/tool-search.ts:840`
  - body: `--permission` does not deny outbound network access, so model-authored `tool_search_code` can still use sockets or fetch directly from the child process and bypass OpenClaw tool policy. Add a real network-deny boundary or keep code mode disabled until that boundary exists.
  - confidence: 0.91
- **[P1] Abort bridge work when code mode settles:** `src/agents/tool-search.ts:973-980`
  - body: Bridge calls are thenable, so `Promise.resolve(openclaw.tools.call(...))` starts host-side execution before the model returns. Normal result settlement kills the child but does not abort the already-started bridge request, leaving tool side effects after `tool_search_code` has reported success.
  - confidence: 0.88
- **[P2] Reject ambiguous catalog name calls:** `src/agents/tool-search.ts:635-639`
  - body: `findEntry` accepts ids or names and returns the first match. Client and plugin tools can share a function name, so a name-based call can silently invoke whichever catalog entry sorts first instead of requiring the unambiguous id.
  - confidence: 0.84
- **[P2] Fail closed without a scoped catalog match:** `src/agents/tool-search.ts:570-575`
  - body: When no session, key, or agent scope matches, `resolveCatalog` returns the only in-memory catalog. A control tool with missing or stale context can therefore search or execute another run's catalog instead of failing closed.
  - confidence: 0.82
- **[P2] Preserve normal bridged tool error results:** `src/agents/pi-embedded-runner/run/attempt.ts:2526-2533`
  - body: The bridge executes the raw tool through `runToolLifecycle`, but the normal PI adapter converts non-abort tool throws into structured error results. A validation/runtime throw from a cataloged tool will reject the bridge call instead of giving the model the recoverable tool output it gets from direct tools.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces model-authored JavaScript execution and still has concrete network, post-settlement side-effect, and cross-session catalog risks.

Concerns:

- **[high] Code-mode child can use direct network:** `src/agents/tool-search.ts:840`
  - body: The child process is launched with Node `--permission`, but the available permission flags do not deny outbound networking; model-authored code can bypass OpenClaw tool policy by opening sockets or fetches directly.
  - confidence: 0.91
- **[high] Bridge calls can outlive the code-tool result:** `src/agents/tool-search.ts:991`
  - body: Thenable-started bridge calls can be dispatched before the final result, and normal settlement does not abort already-started host execution, allowing tool side effects after the reported lifecycle has ended.
  - confidence: 0.88
- **[medium] Catalog fallback can cross run boundaries:** `src/agents/tool-search.ts:570`
  - body: Returning the only in-memory catalog when no scoped key matches can expose or execute tools from the wrong active or leftover run if a control tool is created with missing or stale scope.
  - confidence: 0.82
