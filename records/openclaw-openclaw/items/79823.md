## Summary

Keep open: this protected, security-sensitive core feature still has current-head correctness issues in the Tool Search bridge lifecycle, session catalog isolation, and bridged error semantics, despite sufficient real behavior proof for the happy path.

## What This Changes

The PR adds core PI Tool Search with `tool_search_code`, structured search/describe/call tools, `tools.toolSearch` config, catalog compaction for OpenClaw/MCP/client tools, docs, tests, and a gateway E2E script.

## Best Possible Solution

Revise the branch so Tool Search is PI-scoped, fail-closed on catalog lookup, cancels or drains all in-flight bridged host calls when code mode settles, preserves normal PI tool error results, and then rerun the targeted tests plus the gateway E2E proof.

## Reproduction Assessment

Yes for the review findings: source inspection of PR head `fb4d50e097aa6228a2c5272bae9d67b7285ed380` shows the bridge lifecycle, catalog fallback, and raw execution paths. The feature itself is not a bug reproduction case.

## Solution Assessment

No. Tool Search is plausibly a core PI capability, but this implementation should not merge until the bridge lifecycle, catalog isolation, and error semantics match the existing runtime contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Cancel bridged calls when code mode settles:** `src/agents/tool-search.ts:898-900`
  - body: The settled guard only stops later message handling; a model can start `openclaw.tools.call(...).then(...)`, return, and leave the host tool running because the bridge passes only the outer run signal. Add a per-code-call abort controller and abort or drain in-flight bridge requests on result settlement and timeout.
  - confidence: 0.86
- **[P2] Require a session-scoped catalog match:** `src/agents/tool-search.ts:511-517`
  - body: When lookup by `sessionId`, `sessionKey`, and `agentId` misses, this fallback returns the only catalog in memory. A control tool created with missing or stale context can therefore search or execute another run's tools instead of failing closed.
  - confidence: 0.9
- **[P2] Return structured errors from bridged tool calls:** `src/agents/pi-embedded-runner/run/attempt.ts:2503-2510`
  - body: This bridge path calls the raw cataloged tool directly, while normal PI tool calls go through `toToolDefinitions`, which converts non-abort throws into structured tool error results. A validation/runtime throw from a bridged tool will currently reject `tool_search_code` or `tool_call` instead of giving the model recoverable tool output.
  - confidence: 0.82

## Security Review
