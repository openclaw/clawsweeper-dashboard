## Summary

Keep open. Current main still lacks the lower-level expected-byte contract in the host pinned-write helper and sandbox mutation helper, and the PR addresses a real write-integrity gap, but the proposed rollback copy fallback needs one more fix before merge because an incomplete backup copy can be treated as restorable.

## What This Changes

The PR adds expected byte-size verification and rollback handling to host pinned writes, threads expected sizes through local and remote sandbox writes, and adds regression coverage for encoded payload sizes and mismatch failures.

## Best Possible Solution

Keep the lower-level byte-count integrity guard in the host pinned-write helper and sandbox mutation helper, but make backup-copy rollback use a full-write loop or verify the completed backup size before it can be restored; then add the changelog entry and run the focused tests plus changed-gate validation before maintainer merge.

## Reproduction Assessment

Yes. Current-main inspection reproduces the implementation gap by showing no expected-size parameter or final-size verification in the host and sandbox helper paths, and the PR adds targeted mismatch regressions; the new review finding can be covered by forcing the backup-copy fallback to short-write before a size-mismatch rollback.

## Solution Assessment

No, not merge-ready as-is. The lower-level helper guard is the right layer, but the fallback backup copy must be made complete or size-verified before rollback is trusted.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Verify backup copies before using them for rollback:** `src/infra/fs-pinned-write-helper.ts:104`
  - body: In the hardlink-unsupported fallback, `copy_backup_file()` treats a backup as ready after one `os.write()` per chunk. Since `os.write()` may write fewer bytes than requested, a short backup write can leave an incomplete backup that is later restored over the destination on size-verification failure, corrupting the original file during the failure path. Loop until each chunk is fully written, or verify the backup size before setting `backup_ready`.
  - confidence: 0.82
