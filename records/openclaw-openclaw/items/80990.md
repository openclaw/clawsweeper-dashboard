## Summary

Current source and docs show this as documented Telegram group visible-reply policy, not a group-to-DM routing bug: group turns default to message-tool-only visible output, and the Telegram source routing still uses the originating group chat id when visible group output is enabled or when the message tool defaults to the current source conversation.

## What This Changes

The issue asks for Telegram group mention replies to post back to the originating group instead of going to the sender's private DM.

## Best Possible Solution

Keep the existing policy and use messages.groupChat.visibleReplies="automatic" when operators want legacy automatic group posting; open a new actionable bug only if that setting or a same-chat message tool send still targets the user id.

## Reproduction Assessment

No, there is not a high-confidence source-level bug reproduction from the provided report. The configuration omits messages.groupChat.visibleReplies, and current docs/source explain the observed private/default behavior as policy rather than a Telegram target mismatch.

## Solution Assessment

Not applicable as a code fix for this issue. The narrow solution is configuration: set messages.groupChat.visibleReplies="automatic" for automatic group replies, or rely on message(action=send), whose current default target is the originating Telegram chat.
