- `P2`: This is a normal-priority agent runtime bug fix with limited but real impact around compaction timeout teardown.
- `merge-risk: 🚨 compatibility`: The PR changes `unsubscribe` from synchronous cleanup to async cleanup while current callers still use it synchronously.

## Summary

Keep open: the underlying event-loss problem is plausible in current main, but this PR is not merge-ready because it changes the unsubscribe contract, can still unsubscribe before the final retry completion, is stale/conflicting, and lacks real behavior proof.

## What This Changes

The PR makes embedded Pi session unsubscribe wait for compaction cleanup before removing the session listener and adds unsubscribe-during-compaction tests.

## Best Possible Solution

Land a rebased teardown fix that preserves the unsubscribe contract or updates all callers, waits for the final `compaction_end`, uses current event names, and includes deterministic tests plus redacted real behavior proof.

## Reproduction Assessment

Yes, at source level: current main aborts in-flight compaction and immediately removes the session listener, while the dependency contract can still emit a later `compaction_end`. I did not run a live 48-hour timeout scenario in this read-only review.

## Solution Assessment

No, not as written. Waiting before listener removal is the right direction, but this patch changes the synchronous cleanup contract and resolves cleanup on retry-end events instead of only the final end event.
