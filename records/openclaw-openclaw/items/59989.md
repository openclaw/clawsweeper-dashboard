Full review comments:

- **[P1] Keep unsubscribe synchronous or await every caller:** `src/agents/pi-embedded-subscribe.ts:702`
  - body: Changing `unsubscribe` to `async` changes the cleanup contract, but teardown callers still invoke it as a synchronous cleanup step. That lets cleanup continue before `sessionUnsubscribe()` actually runs and moves failures outside the existing try/catch path, making listener removal nondeterministic during abort/timeout cleanup.
  - confidence: 0.9
- **[P1] Wait only for the final compaction end:** `src/agents/pi-embedded-subscribe.handlers.compaction.ts:75-79`
  - body: This resolver fires for every `auto_compaction_end`, including `willRetry=true`. A retrying compaction is not final, so `unsubscribe()` can remove the listener before the later final end event and still lose the notification the PR is trying to preserve.
  - confidence: 0.91
- **[P3] Avoid a real five-second timeout in the test:** `src/agents/pi-embedded-subscribe.unsubscribe-compaction-cleanup.test.ts:51-52`
  - body: The timeout-path test awaits the production 5 second timer, which makes the focused suite slower and more timing-sensitive. Use fake timers so the regression proof is deterministic and cheap.
  - confidence: 0.83
