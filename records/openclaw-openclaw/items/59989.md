Full review comments:

- **[P1] Preserve the unsubscribe completion contract:** `src/agents/pi-embedded-subscribe.ts:699`
  - body: Changing `unsubscribe` to `async` breaks the current teardown contract: the runner still calls `unsubscribe()` without `await` inside a synchronous try/catch, so cleanup continues before `sessionUnsubscribe()` runs and async failures bypass that catch. Keep this API synchronous or update the shared teardown path so all callers await the same cleanup promise.
  - confidence: 0.94
- **[P1] Resolve cleanup only after final compaction end:** `src/agents/pi-embedded-subscribe.handlers.compaction.ts:75-79`
  - body: This resolver runs for every compaction end event, including `willRetry=true`. A retrying compaction still has a later final end event, so this can remove the listener before the final compaction-end notification is delivered, preserving the race the PR is meant to fix.
  - confidence: 0.95
- **[P2] Register the cleanup waiter without a missed-end window:** `src/agents/pi-embedded-subscribe.ts:680`
  - body: The cleanup promise is registered after the state check, leaving a window where compaction can finish after the check but before `compactionCleanupResolve` is stored. In that case unsubscribe waits for the full timeout even though the end event already happened.
  - confidence: 0.82
- **[P2] Use current compaction event names in tests:** `src/agents/pi-embedded-subscribe.unsubscribe-compaction-cleanup.test.ts:19-31`
  - body: The new tests emit `auto_compaction_start` and `auto_compaction_end`, but current main's embedded-agent dispatcher uses `compaction_start` and `compaction_end`. These tests need to target the current event contract after the branch is rebased.
  - confidence: 0.87
- **[P3] Avoid the real five-second timeout in the test:** `src/agents/pi-embedded-subscribe.unsubscribe-compaction-cleanup.test.ts:53-59`
  - body: The timeout test awaits the real cleanup timeout, adding five seconds and CI timing variance for a deterministic unit path. Use fake timers so the timeout behavior is covered without slowing the suite.
  - confidence: 0.9
