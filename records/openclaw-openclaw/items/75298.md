## What This Changes

The issue reports repeated webhooks route registration during gateway event-loop saturation from cron catchup, Control UI polling, and memory-core dreaming.

## Best Possible Solution

Trace the plugin activation/reload caller during saturation windows, then add regression coverage proving webhooks route registration runs once per gateway lifecycle except explicit plugin config reload or gateway restart.

## Reproduction Assessment

Unclear: the issue has credible timestamped logs and current source maps the repeated log line to the webhooks registration path, but I did not establish a high-confidence current-main reproduction for event-loop saturation causing repeated registration.

## Solution Assessment
