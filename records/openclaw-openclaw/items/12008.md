## Summary

Keep open. Current main and the latest release still lack a typed Gemini safety-settings config surface, and the direct Google Generative AI request path still returns payloads without `safetySettings`; the previous implementation candidate at https://github.com/openclaw/openclaw/pull/49529 is now closed unmerged, so no active linked PR owns the remaining work.

## What This Changes

The issue requests configurable Gemini safety thresholds in OpenClaw provider or model configuration so Gemini requests can include Google API safety settings.

## Best Possible Solution

Add a narrow, typed, schema-validated Gemini safety-settings contract, document it as Google Generative AI only, and inject it into the direct Google request body without overriding any explicit per-request payload value.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the missing capability: current config validation rejects a first-class `safetySettings` provider key, and the Google request builder emits no `safetySettings` field.

## Solution Assessment

Yes in direction, but the final API shape needs maintainer approval. A typed Google-scoped config plus Google transport injection is safer than a generic untyped params pass-through or the now-closed PR branch as-is.
