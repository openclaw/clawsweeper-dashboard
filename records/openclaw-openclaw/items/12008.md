## Summary

Keep open: current main and v2026.6.10 still lack a first-class Gemini `safetySettings` config contract and Google GenerateContent request injection, while the prior implementation PR is closed unmerged and no longer a viable landing path.

## What This Changes

The issue requests configurable Gemini safety thresholds in OpenClaw provider/model config and mapping those values into Google Gemini request payloads.

## Best Possible Solution

Define a narrow Google-owned `safetySettings` contract, validate it in config/schema/help/docs, inject it only on the direct Google GenerateContent request path without overriding explicit request payload values, and require focused tests plus redacted live Gemini proof.

## Reproduction Assessment

Not applicable as a broken-behavior reproduction, but yes at source level: current main has no accepted `safetySettings` config key and the Google GenerateContent builder does not emit that request field.

## Solution Assessment

Unclear as proposed: provider/model config is plausible, but maintainers still need to choose the public schema, category set, threshold policy, Google-only scope, and safety posture before implementation.
