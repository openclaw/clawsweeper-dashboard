## What This Changes

Requests provider/model-level Gemini safety-settings configuration that OpenClaw maps into Google Gemini requests to reduce blocked or empty Gemini responses.

## Best Possible Solution

Define a narrow Google-owned safety-settings contract, validate it in config/schema/help/docs, inject it only on the direct Google GenerateContent request path without overriding explicit request payload values, and require focused tests plus redacted live Gemini proof.

## Reproduction Assessment

Yes, at source level: current main has no typed safetySettings config key and the Google GenerateContent request builder does not emit that field. This is a feature/config gap rather than a failing existing-behavior bug reproduction.

## Solution Assessment

Unclear as proposed: provider/model config is plausible, but the exact public schema, Google-only scope, category set, threshold policy, and safety posture need maintainer selection before implementation.
