## Summary

Keep open. Current main and latest release still lack a typed, documented Gemini safety-settings config surface, the direct Google Gemini request builder still omits `safetySettings`, and the prior implementation PR is closed unmerged, so this remains the canonical maintainer decision item.

## What This Changes

The issue requests configurable Gemini safety settings for Gemini-based providers/models so operators can tune upstream safety thresholds.

## Best Possible Solution

Define a narrow Google-owned safety-settings contract, validate it in config/schema/help/docs, apply it only in the Google GenerateContent transport without overriding explicit request-level values, and prove it with focused tests plus redacted live Gemini proof.

## Reproduction Assessment

Yes, at source level for the missing capability: current-main and v2026.6.8 OpenClaw have no typed safety-settings config field and the direct Google request builder does not emit `safetySettings`. This is a feature/config request rather than a failing existing behavior test.

## Solution Assessment

Unclear as proposed. Provider/model config is plausible and vision-aligned, but the exact public schema, category set, threshold policy, scope, and safety posture need maintainer selection before implementation.
