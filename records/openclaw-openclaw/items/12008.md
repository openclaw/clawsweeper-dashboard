## Summary

Keep open. Current main still lacks a typed, schema-validated Gemini safety-settings config surface and the direct Google Generative AI request builder does not emit `safetySettings`; open PR #49529 remains the active implementation candidate.

## What This Changes

Request to add configurable Gemini safety thresholds at provider and/or model level and map them into Gemini API requests.

## Best Possible Solution

Review and land PR #49529 or an equivalent focused change that adds a strict `models.providers.*.safetySettings` contract, generated schema/help/docs, Google-only request injection, explicit payload precedence, and focused regression tests.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature request. The current-main verification path is high-confidence source inspection: strict provider config has no `safetySettings` field, and the direct Gemini request builder has no payload field for it.

## Solution Assessment

Yes. The narrow maintainable direction is a Google-scoped, schema-validated config field wired into the direct Gemini request path while preserving explicit request-level payload values; reviewing PR #49529 is preferable to duplicate repair work.
