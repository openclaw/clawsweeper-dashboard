## Summary

Keep open. Current main and v2026.6.8 still lack a typed, documented Gemini safety-settings config surface, and the prior implementation PR is closed unmerged, so this remains the canonical tracker for a maintainer-chosen provider contract.

## What This Changes

The issue requests configurable Gemini safety thresholds for Google/Gemini provider requests so operators can tune upstream content filtering.

## Best Possible Solution

Define a narrow Google-owned safety-settings contract, validate it in config/schema/help/docs, apply it only in the Google GenerateContent transport without overriding explicit request-level values, and prove it with focused tests plus redacted live Gemini proof.

## Reproduction Assessment

Yes at source level for the missing capability. Current main's strict provider schema and Google request builders have no safetySettings path, though this is a feature request rather than a failing existing behavior test.

## Solution Assessment

Unclear as proposed. Provider/model config is plausible and vision-aligned, but the exact public schema, category set, threshold policy, scope, and safety posture need maintainer selection before implementation.
