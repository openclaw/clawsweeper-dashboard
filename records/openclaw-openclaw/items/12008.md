## Summary

Keep open. Current main still lacks typed, schema-validated, documented Gemini safety-settings config and the direct Google Generative AI request builder still emits payloads without `safetySettings`; open PR #49529 remains the active implementation candidate.

## What This Changes

The issue asks OpenClaw to expose Gemini safety thresholds in provider/model config and map them into Gemini API requests.

## Best Possible Solution

Review and land PR #49529 or an equivalent focused change that adds a strict Google-scoped `models.providers.*.safetySettings` contract, generated schema/help/docs, runtime payload injection, precedence rules, and regression tests.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature request. The current-main verification path is high-confidence source inspection: strict provider config has no `safetySettings` field, and the direct Gemini request builder has no payload field for it.

## Solution Assessment

Yes. The maintainable direction is a schema-validated, Google-scoped config field wired into direct Gemini requests while preserving explicit request-level payload values; reviewing the existing PR is preferable to duplicate repair work.
