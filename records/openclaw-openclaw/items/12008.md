## Summary

Keep open. Current main still has no typed/schema-validated Gemini safety-settings configuration and the direct Google Generative AI request builder does not emit `safetySettings`; https://github.com/openclaw/openclaw/pull/49529 remains the active implementation candidate rather than a reason to close this issue.

## What This Changes

Allow users to configure Gemini safety thresholds for Google/Gemini providers and pass them to the Gemini API as `safetySettings`.

## Best Possible Solution

Review, repair, or replace https://github.com/openclaw/openclaw/pull/49529 with a strict Google-provider safety-settings contract, request injection that preserves explicit payload precedence, docs/schema/help updates, and focused config/runtime regression coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a normal bug reproduction because this is a feature/config request; source inspection on current main shows the requested config key is not typed/schema-accepted and Gemini payloads are serialized without `safetySettings`.

## Solution Assessment

Is this the best way to solve the issue? Yes in direction, but it needs maintainer-approved product shape: strict provider-scoped validation plus Google-transport-owned top-level request injection is the maintainable path.
