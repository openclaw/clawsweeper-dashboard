## Summary

Keep open. Current main still lacks a typed, schema-validated Gemini safety-settings configuration surface, and the direct Google Generative AI request builder still emits payloads without `safetySettings`; open PR #49529 remains the active implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests configurable Gemini safety thresholds at provider and/or model level, mapped into outgoing Gemini API requests.

## Best Possible Solution

Keep this issue open while maintainers review and land PR #49529 or an equivalent focused implementation that adds a strict `models.providers.*.safetySettings` contract, generated schema/help coverage, Google-only request injection, user-facing docs, and focused config/runtime regression tests.

## Reproduction Assessment

Not applicable as a normal bug reproduction; this is a feature request. The current-main verification path is high-confidence source inspection: provider-level `safetySettings` is absent from the strict config schema, and the Google request builder has no payload field for it.

## Solution Assessment

Yes. The narrow maintainable direction is a Google-scoped, schema-validated config field wired into the direct Gemini request path while preserving explicit payload-level values; reviewing PR #49529 is preferable to starting duplicate work.
