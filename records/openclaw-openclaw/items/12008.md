## Summary

Current main and v2026.6.5 still lack a validated Gemini safety-settings configuration and outbound request mapping. The previous implementation closed unmerged because its request shape and generic agent-layer mutation were not acceptable, so the capability remains necessary but needs maintainer approval of the public config and safety policy before replacement work proceeds.

## What This Changes

Add configurable Gemini harm-category thresholds and forward them on Google Generative AI requests.

## Best Possible Solution

Define a narrow Google-provider safety contract with explicit supported categories and thresholds, validate it through config/schema/help, apply it in the owning Google GenerateContent transport without overriding explicit request values, document the safety implications, and prove the serialized request with focused tests plus a redacted live Gemini run.

## Reproduction Assessment

Yes at source level: current main's strict provider schema has no safety-settings field, and the Google GenerateContent payload has no safetySettings member or injection path. A live blocked-response scenario was not run, but the missing capability itself is directly reproducible from source.

## Solution Assessment

No, the closed pull request is not the best implementation because it mutated requests through a generic agent wrapper and used an unsuitable request shape. The maintainable solution is a Google-provider-owned transport change after maintainers approve the public config and safety policy.
