## Summary

Keep open. Current main still lacks a typed, schema-validated Gemini safety-settings configuration surface and the direct Google Gemini request builder does not emit `safetySettings`; open PR #49529 remains the active implementation candidate.

## What This Changes

Add configurable Gemini safety settings for Gemini-based providers/models and map them into Google Gemini API requests.

## Best Possible Solution

Review and land PR #49529 or an equivalent focused change that adds a strict `models.providers.*.safetySettings` contract, generated schema/help/docs, Google-only request injection, explicit payload precedence, and focused regression tests.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature request. The current-main verification path is high-confidence source inspection: the strict provider config schema lacks `safetySettings`, and the Google request builder has no payload field for it.

## Solution Assessment

Yes. The narrow maintainable direction is a Google-scoped, schema-validated config field wired into the direct Gemini request path while preserving explicit request-level payload values; reviewing PR #49529 is preferable to duplicate repair work.
