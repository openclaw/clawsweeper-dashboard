## Summary

Keep open. Current main and the latest release still lack a typed Gemini safety-settings config surface and direct Google GenerateContent payload injection, while the prior implementation PR is closed unmerged and not a viable canonical fix.

## What This Changes

The issue requests configurable Gemini safety thresholds in OpenClaw provider/model configuration so Gemini requests can tune API safety filtering.

## Best Possible Solution

Define a narrow Google-owned safety-settings contract, validate it in config/schema/help/docs, apply it only in the Google GenerateContent transport without overriding explicit request-level values, and prove it with focused tests plus redacted live Gemini proof.

## Reproduction Assessment

Yes at source level for the missing capability, though this is a feature request rather than a bug reproduction. Current main's strict provider schema lacks safetySettings and the Google request builder returns Gemini payloads without that field.

## Solution Assessment

Unclear as proposed. Provider/model config is plausible and vision-aligned, but the exact schema, category set, threshold policy, and safety posture need maintainer selection before implementation.
