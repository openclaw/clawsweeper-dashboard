## Summary

Keep open. Current main still has no typed, schema-validated Gemini safety-settings config surface, direct Gemini requests still omit `safetySettings`, and https://github.com/openclaw/openclaw/pull/49529 remains the active unmerged implementation candidate with review blockers.

## What This Changes

The issue requests configurable Gemini safety thresholds for Gemini providers/models so users can reduce blocked or empty responses.

## Best Possible Solution

Review, repair, or replace https://github.com/openclaw/openclaw/pull/49529 with a focused implementation that adds strict config validation, schema/help/docs/changelog coverage, top-level Google `safetySettings` injection, precedence rules, and regression tests.

## Reproduction Assessment

Not applicable as a normal bug reproduction because this is a feature/config request. Current-main source inspection gives high confidence that the config schema lacks `safetySettings` and direct Gemini payload construction never emits it.

## Solution Assessment

Yes, the provider-scoped config direction is the right maintainable path, but it should be completed through the existing PR or an equivalent replacement. The fix should stay Google-scoped and put `safetySettings` on the Gemini request root while preserving explicit request-level values.
