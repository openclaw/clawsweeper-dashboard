## Summary

Keep open. Current main and latest release still lack a typed Gemini safety-settings config surface, and the matching implementation PR is closed unmerged, leaving this issue as the canonical tracker.

## What This Changes

The issue requests provider/model-level configuration for Gemini safety settings so OpenClaw can pass safety thresholds to Google Gemini requests.

## Best Possible Solution

Define a narrow Google-owned safety-settings contract, validate it in config/schema/help, apply it only in the Google GenerateContent transport without overriding explicit payload values, document the safety implications, and prove the serialized request with focused tests plus redacted live Gemini proof.

## Reproduction Assessment

Yes, at source level: current main's strict provider schema lacks a safetySettings key and Google request builders return Gemini payloads without safetySettings. I did not run a live Gemini request in this read-only review.

## Solution Assessment

Unclear as proposed: the provider/config direction is plausible and aligned, but the exact public schema, category set, threshold policy, and safety posture need maintainer selection before implementation.
