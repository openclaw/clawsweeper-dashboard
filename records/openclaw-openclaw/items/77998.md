## Summary

Keep open. Current main still has the source-reproducible skills watcher lifecycle leak, but this PR still has a correctness blocker because deleting a workspace version can fall back to an older global version and skip a needed snapshot refresh.

## What This Changes

The PR clears per-workspace skills snapshot versions during watcher teardown, adds idle watcher eviction, updates tests, and adds changelog entries.

## Best Possible Solution

Keep the lifecycle cleanup, but make teardown and idle eviction leave a monotonic or explicit invalidation state so the next session refreshes safely.

## Reproduction Assessment

Yes. Source inspection shows watcher events write workspace-scoped versions while current-main teardown closes the watcher without clearing that map entry; the linked issue and PR body also include terminal proof for the Map lifecycle shape.

## Solution Assessment

No, not yet. The cleanup boundary is right, but deleting the workspace version can lower the observable version to an older global fallback and let a stale cached skills snapshot survive after idle eviction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve refresh invalidation when clearing workspace versions:** `src/agents/skills/refresh-state.ts:75-76`
  - body: Deleting the workspace entry can make `getSkillsSnapshotVersion` fall back to an older nonzero `globalVersion`. Make clear/eviction force a refresh or preserve a monotonic invalidation value, and cover this case in tests.
