## Summary

Keep this PR open: it addresses a valid skills watcher/version-map leak on current main, but the deletion-only version clear can miss a required refresh after idle eviction when a nonzero global version is older than the cached workspace-local version, and the changelog entry still needs contributor credit.

## What This Changes

The PR adds a per-workspace skills snapshot version clear helper, idle-evicts skills watchers after a one-hour inactivity TTL, adds regression tests, and adds a changelog fix entry.

## Best Possible Solution

Keep the lifecycle cleanup, but make cleared workspace versions force a next-turn snapshot refresh across nonzero global-version states, add regression coverage for that case, and add the required changelog credit before landing.

## Reproduction Assessment

Yes, source-reproducible: current main bumps `workspaceVersions` for workspace-scoped watcher events, the watch-disabled teardown only removes the watcher/timer, and the session update path calls `ensureSkillsWatcher` per turn without a session-end disposal path.

## Solution Assessment

No, not yet: the cleanup direction is right, but deleting the workspace version can make the observable version fall back to an older nonzero `globalVersion`, so a reattached watcher may reuse a stale snapshot instead of forcing refresh.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Force refresh after clearing workspace versions:** `src/agents/skills/refresh-state.ts:75`
  - body: Deleting the workspace entry can make the observable version go backwards when `globalVersion` is nonzero but older than the workspace-local cached version. After idle eviction, a skills file can change while no watcher is attached; on the next turn the watcher is reattached, but `shouldRefreshSnapshotForVersion(cachedLocal, olderGlobal)` returns false, so the stale skills snapshot can persist until another watched event. Make the clear path or comparison force a refresh for cleared workspaces and cover the nonzero-global case.
  - confidence: 0.86
