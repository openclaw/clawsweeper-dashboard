## Summary

Keep open: the linked Telegram deadlock is a valid P1 reliability bug, but this PR should not merge as-is because the new error classifier import targets a non-exported internal package subpath and would fail before dead-lettering the update. The external PR also lacks after-fix real Telegram or runtime-log proof.

## What This Changes

The PR changes Telegram polling-session retry handling to dead-letter MissingAgentHarnessError instead of requeueing the same spooled update.

## Best Possible Solution

Dead-letter only nonrecoverable missing-harness spool errors through a supported SDK or core boundary, keep transient Telegram failures retryable, and prove the queue drains after the poison update.

## Reproduction Assessment

Yes. Source inspection shows current main requeues every bot.handleUpdate failure from the Telegram ingress spool, and the related report gives live MissingAgentHarnessError logs that follow that path.

## Solution Assessment

No. The dead-letter direction is appropriate, but this implementation uses a non-exported core import and needs a supported boundary plus regression and live/log proof.
