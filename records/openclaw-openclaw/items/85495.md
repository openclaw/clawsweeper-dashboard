## Summary

Keep open: this targets a real Telegram spool deadlock, but the PR is not merge-ready because it imports an unexported OpenClaw subpath and lacks after-fix real Telegram or runtime proof.

## What This Changes

The PR changes Telegram polling-session failure handling to dead-letter MissingAgentHarnessError spooled updates instead of requeueing them for retry.

## Best Possible Solution

Classify missing-harness spool failures through a supported core or SDK boundary, dead-letter only that nonrecoverable update, preserve retries for transient failures, and prove the Telegram queue drains afterward.

## Reproduction Assessment

Yes, source-reproducible. On current main, any bot.handleUpdate error enters #releaseFailedSpooledUpdate, releases the claim, and requeues the update, and the linked issue provides live MissingAgentHarnessError retry logs.

## Solution Assessment

No. Dead-lettering this nonrecoverable failure is the right direction, but this implementation uses a non-exported package subpath and needs a supported boundary, regression coverage, and real behavior proof.
