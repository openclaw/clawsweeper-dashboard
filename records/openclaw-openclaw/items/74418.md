## What This Changes

The PR normalizes per-model `params.thinking` values `false`, `"disabled"`, and `"none"` to `"off"` in `resolveThinkingDefault` and adds three model-selection regression tests.

## Best Possible Solution

Merge the resolver-level normalization with focused tests after adding a single-line `Unreleased` / `Fixes` changelog entry, while leaving the broader `extra_body` collision work to #75040.

## Reproduction Assessment

Yes. On current main, configuring `agents.defaults.models["deepseek/deepseek-v4-pro"].params.thinking` as `false`, `"disabled"`, or `"none"` leaves the resolver on the provider fallback path, which is DeepSeek V4 `high` thinking.

## Solution Assessment

Yes, with the changelog addition. Normalizing disabled aliases before provider fallback is the narrow maintainable fix because the DeepSeek wrapper already handles stripped replay correctly once it receives `off` or `none`.
