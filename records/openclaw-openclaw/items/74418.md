This PR normalizes per-model `params.thinking` values `false`, `"disabled"`, and `"none"` to `off` in `resolveThinkingDefault` and adds focused DeepSeek V4 Pro model-selection tests.

## Best Possible Solution

Merge this resolver-level normalization with its focused tests after adding a single-line `CHANGELOG.md` entry, while leaving the broader `extra_body` collision work to #75040.

## Reproduction Assessment

Yes. On current main, configure `agents.defaults.models["deepseek/deepseek-v4-pro"].params.thinking` as `false`, `"disabled"`, or `"none"`; the resolver misses those values and falls through to DeepSeek's `high` provider default.

## Solution Assessment

Yes, with the changelog addition. Normalizing disabled aliases before provider fallback is the narrow maintainable fix because the DeepSeek wrapper already handles disabled replay correctly once it receives `off` or `none`.
