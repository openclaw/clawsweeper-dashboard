## Summary

Keep open. Current main still ignores `params.thinking` values `false`, `disabled`, and `none`, while DeepSeek V4 declares a `high` default and the disabled wrapper only activates after the resolved level is `off`/`none`; this PR is a narrow fix with sufficient resolver-level proof and no blocking findings.

## What This Changes

The PR changes the agent thinking default resolver to treat `false`, `disabled`, and `none` as `off`, adds DeepSeek V4 resolver tests, and adds a changelog note.

## Best Possible Solution

Merge the resolver alias normalization after exact-head validation, while leaving the broader `extra_body` payload-collision behavior to the linked follow-up issue.

## Reproduction Assessment

Yes, at source level. Current main accepts arbitrary per-model params but `resolveThinkingDefault` ignores `false`, `disabled`, and `none`, which can fall through to DeepSeek V4's `high` default.

## Solution Assessment

Yes. Normalizing explicit disabled aliases in `resolveThinkingDefault` is the narrowest maintainable fix for this resolver bug; the broader payload collision belongs in https://github.com/openclaw/openclaw/issues/75040.
