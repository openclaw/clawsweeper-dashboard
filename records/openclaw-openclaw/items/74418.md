## What This Changes

The PR normalizes per-model `params.thinking` values `false`, `"disabled"`, and `"none"` to `"off"` in the thinking resolver and adds three model-selection tests.

## Best Possible Solution

Merge the resolver-level normalization with focused tests after adding a single-line `Unreleased` / `Fixes` changelog entry, while leaving the broader `extra_body` collision discussion to #75040.

## Reproduction Assessment

Yes. On current main, configure `agents.defaults.models["deepseek/deepseek-v4-pro"].params.thinking` as `false`, `"disabled"`, or `"none"`; the resolver does not recognize those values and falls through to DeepSeek's `high` thinking default.

## Solution Assessment

Yes, with the changelog addition. Normalizing disabled aliases before provider fallback is the narrow maintainable fix because the DeepSeek wrapper already handles disabled replay correctly once it receives `off` or `none`.
