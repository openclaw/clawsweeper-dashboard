## Summary

Keep open. Current main already hardens the package-manager update path and Claude CLI missing-transcript recovery, but the report also covers cron/systemEvent session-store rows without transcripts, and the current source/docs do not prove that path is fixed or intentionally excluded.

## What This Changes

The issue asks OpenClaw to make gateway package updates atomic, prevent shell-only cron/systemEvent jobs from becoming resumable transcript sessions, and recover chat sessions when a stored CLI transcript is missing.

## Best Possible Solution

Keep the issue open until the cron/systemEvent session-store behavior is reproduced or split, then ship a targeted fix/test for any remaining shell-only cron transcript rows alongside the existing update and resume hardening.

## Reproduction Assessment

No high-confidence current-main reproduction was run. Source inspection confirms fixes for the update and Claude resume paths, but the cron/systemEvent missing-transcript creation path remains unclear.
