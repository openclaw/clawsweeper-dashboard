## Summary

Close as implemented on current main: the reported v2026.5.2 failure modes are covered by staged verified package swaps, restart gating on successful updates, stable cron-session persistence, and Claude CLI missing-transcript recovery; no containing release tag was found yet.

## What This Changes

Gateway package updates should avoid exposing half-installed packages, cron/system-event sessions should not create broken resumable rows, and missing Claude CLI transcripts should recover by starting fresh.

## Best Possible Solution

Keep the current-main staged update, cron persistence, and CLI resume-recovery behavior, then ship it in the next stable release for users still on v2026.5.2.

## Reproduction Assessment

Yes for the affected v2026.5.2 behavior by source/log path; I did not live-run the macOS/npm/Telegram scenario. Current main is expected not to reproduce because the relevant update, cron-session, and CLI resume paths now have direct source and regression-test coverage.

## Solution Assessment

Yes. Staging and verifying npm installs before the live swap, restarting only after success, keeping cron state under stable keys, and clearing invalid Claude CLI resume bindings are the narrow maintainable fixes for the observed crash and resume loop.
