## Repair Work Prompt

Fix the remaining session-store cleanup side of https://github.com/openclaw/openclaw/issues/77011. Current main already hardens package update verify/swap and Claude CLI missing-transcript resume recovery; do not rework those paths except to preserve tests. Add regression coverage and code so `openclaw sessions cleanup --dry-run/--fix-missing` and `openclaw doctor` do not abort when a session store entry has an invalid `sessionId` such as `agent:main:main`; treat malformed IDs with missing/unresolvable transcripts as reportable/prunable entries with clear output. Then investigate the cron/session rows described in the issue and canary comment: if current main can still persist non-resumable cron rows with no primary transcript, either prevent registering them as resumable sessions or exclude clearly metadata-only rows from missing-transcript diagnostics while preserving cleanup for real broken sessions. Stop and report if the cron behavior requires a new config option or product policy. Add a single-line changelog fix for the user-facing cleanup/doctor behavior.
