## What This Changes

## Best Possible Solution

Keep the new timeout notice for non-compaction zero-payload run/tool timeouts, but reuse the existing liveness and outbound-delivery gates so compaction pauses and out-of-band prompts remain correctly classified, then add real behavior proof.

## Solution Assessment

No: the terminal notice is a reasonable layer for the silent timeout, but this implementation is too broad. The safer fix is to thread the existing compaction and out-of-band delivery evidence into the decision before emitting the notice.
