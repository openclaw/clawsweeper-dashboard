## Summary

Keep open. Current main still has the reported hard-coded Claude live-session turn caps, the config/schema only expose watchdog tuning, and the v2026.4.29 release tag contains the same abort path, so this remains a valid latest-release CLI backend reliability bug.

## What This Changes

Request to make Claude CLI live-session per-turn output limits configurable or raise the default so tool-heavy Claude CLI turns do not abort without a useful reply.

## Best Possible Solution

Add bounded, documented live-session output-limit tuning to the CLI backend reliability config, apply it to the Claude stdio parser, and cover both the default guard and an operator-raised limit with focused tests and a changelog entry crediting the reporter.

## Reproduction Assessment

Yes. Current main has a high-confidence source-level reproduction: feed a Claude live-session turn enough parsed stream-json lines to exceed 2 MiB raw output or 5,000 lines, and the parser aborts with the reported FailoverError path.

## Solution Assessment

Yes. The requested direction is best if implemented as bounded configuration rather than removing the guard; raising the default alone would help some workloads but would not give operators a maintainable tuning surface.
