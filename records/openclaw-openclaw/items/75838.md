## Summary

Keep open. Current main still enforces the reported hard-coded Claude live-session turn limits and the CLI backend config/schema only exposes watchdog reliability tuning, not per-turn raw output or line caps; v2026.4.29 contains the same guard, so the release-regression context is credible.

## What This Changes

Request to make Claude CLI live-session turn output limits configurable, or raise the defaults, so tool-heavy `claude-cli` stream-json turns do not abort with `FailoverError`.

## Best Possible Solution

Add bounded, documented live-session output-limit tuning to the CLI backend reliability config, apply it to the Claude stdio parser, and cover both the default guard and an operator-raised limit with focused tests. If maintainers choose a higher bundled Claude default, keep the upper bound explicit and add a changelog entry crediting the reporter.

## Reproduction Assessment

Yes. Current main has a high-confidence source-level reproduction: feed a Claude live-session turn enough parsed stream-json lines to push `rawChars` over 2 MiB or `rawLines` to 5,000, and the parser aborts with the same `FailoverError` reason path shown in the report.

## Solution Assessment

Yes, the requested direction is maintainable if implemented as bounded configuration rather than removing the guard. Raising the default alone may reduce failures, but configurability is the narrower long-term fix for different host capacities and Claude tool-output profiles.
