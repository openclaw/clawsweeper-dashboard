## Summary

Current main still enforces the reported hard-coded Claude live-session turn limits and has no `cliBackends` config, schema, or docs surface for tuning raw turn character or line caps, so the v2026.4.29 regression report remains valid and actionable.

## What This Changes

The issue requests a configurable or larger Claude CLI live-session output limit so tool-heavy turns do not abort silently when no fallback model is configured.

## Best Possible Solution

Add bounded, documented Claude live-session output-limit tuning to the CLI backend reliability configuration, wire the resolved values into the Claude JSONL parser, and cover both default-limit and raised-limit behavior with focused tests plus a user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: feed a Claude live-session turn enough parsed JSONL output to push `rawChars` over 2 MiB or `rawLines` over 5,000, and current main returns the same `FailoverError` reason path described in the issue.

## Solution Assessment

Yes. The requested direction is the maintainable fix if implemented as bounded configuration rather than removing the guard; raising only the default would reduce one workload's failures but would not address different host capacities and tool-output profiles.
