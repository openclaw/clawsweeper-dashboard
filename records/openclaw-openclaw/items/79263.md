## Summary

Keep open: current main and the latest release still do not load a home-level service dotenv for bare local CLI inference, and deciding whether that should be supported is a credential-source security/product choice.

## What This Changes

The issue asks OpenClaw to clarify or change bare local `openclaw infer model run` credential sourcing so home-level service dotenv credentials are either used or diagnosed after upgrade.

## Best Possible Solution

Keep the documented state-dir dotenv behavior as the default unless maintainers explicitly approve a security-reviewed compatibility path; otherwise improve diagnostics or docs for home-level service dotenv users.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.10 only load CWD `.env`, state-dir `.env`, and the scoped gateway compatibility file, not home-level `~/.env`. I did not run a live OpenCode request with real credentials.

## Solution Assessment

Unclear: restoring home-level dotenv loading would match the reporter's smoke-test setup, but docs, diagnostics, or doctor-guided migration to state-dir `.env` may be the safer maintainable solution.
