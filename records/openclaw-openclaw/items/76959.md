## Summary

Current main already covers the reported sandbox ping-pong: managed npm installs preserve existing sandbox dependencies, and doctor/update repair enumerates configured plugin and channel needs before reinstalling missing official plugins.

## What This Changes

The issue asks OpenClaw to preserve all configured official external plugins in the shared `~/.openclaw/npm` sandbox so repairing Brave does not remove Feishu and repairing Feishu does not break Brave.

## Best Possible Solution

Keep the current managed npm root and configured-plugin repair behavior, and let the next release carry this main-only fix to users on `2026.5.2`.

## Reproduction Assessment

No, not against current main: source inspection shows the sandbox dependency set is merged before npm install, and doctor/update repair now considers configured providers and channels together. I did not run a live macOS npm sandbox mutation because this review was read-only.

## Solution Assessment

Yes. Preserving managed npm dependencies and repairing the configured plugin set through doctor/update is the narrow maintainable fix; a Brave- or Feishu-specific workaround would be too owner-specific.
