## Summary

Keep open. The PR appears to fix a real Telegram transient-recovery gap, but it changes a shared outbound message-gateway timeout default, so a maintainer should explicitly accept or narrow that tradeoff before merge.

## What This Changes

The PR makes Telegram Bot API 5xx errors recoverable outside non-idempotent sends, raises the outbound message gateway default timeout from 10s to 75s, adds focused tests, and updates Telegram troubleshooting docs.

## Best Possible Solution

Land after normal CI and maintainer acceptance of the 75s outbound message-gateway default; if that default is too broad, narrow the longer wait to the outbound send path.

## Reproduction Assessment

Yes, at source level. Current main preserves Telegram Bot API error_code across getUpdates paths, rejects a 502 as non-recoverable in the classifier, and uses a 10s outbound gateway default while Telegram text sends are bounded at 60s.

## Solution Assessment

Yes, with one maintainer tradeoff. The classifier change is the right Telegram-owned retry boundary, and the timeout change is a narrow outbound-message helper fix, but maintainers should explicitly accept that the shared message gateway default becomes 75s.
