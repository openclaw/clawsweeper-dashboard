## Summary

## What This Changes

The PR classifies Telegram Bot API 5xx responses as recoverable outside non-idempotent sends, raises the outbound message gateway default timeout from 10s to 75s, and updates Telegram troubleshooting docs.

## Best Possible Solution

Land after normal CI and maintainer acceptance of the 75s outbound message-gateway default; if that default is too broad, narrow the longer wait to the send path.

## Reproduction Assessment

Yes at source level. Current main preserves Telegram Bot API error_code across the isolated getUpdates worker, then exits when the classifier rejects a 502, and the outbound gateway path still has a 10s default before this PR.

## Solution Assessment

Yes, with one maintainer tradeoff. The classifier change is the right Telegram-owned retry boundary, and the timeout change is a narrow outbound-message helper fix, but maintainers should explicitly accept that the shared message gateway default becomes 75s.
