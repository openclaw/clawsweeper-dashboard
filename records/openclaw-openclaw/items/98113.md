## Summary

Keep open for normal maintainer review: current main and the latest release still have unbounded OpenAI OAuth token-body reads, while this PR moves the token response, JSON parse, and diagnostic text paths onto existing bounded readers with focused proof and no blocking patch defect found.

## What This Changes

The PR bounds OpenAI Codex OAuth token response, success JSON, and error-text reads and adds regression tests for oversized and malformed token responses.

## Best Possible Solution

Land the bounded OpenAI OAuth response-read change after ordinary maintainer and CI gates, keeping the cap at the token fetch boundary and retaining the focused regression coverage.

## Reproduction Assessment

Yes. Current main and v2026.6.10 both show postTokenForm using raw response.arrayBuffer() before release(), so an oversized token endpoint body can be buffered before downstream parsing sees it.

## Solution Assessment

Yes. Capping the guarded token response before release() is the right owner-boundary fix; capping only response.json() or diagnostic text would leave the original body-buffering path exposed.
