The PR bounds OpenAI OAuth token response reads in the OpenAI plugin runtime and adds oversized and malformed response tests.

## Best Possible Solution

Keep the bounded read inside postTokenForm before release(), use a small diagnostic error-body cap for exchange and refresh failures, add a regression test for an oversized failed token response, and require redacted path-level terminal/log proof before merge.
