## Summary

Keep this PR open, but it is not merge-ready: the patch applies bounded JSON/text readers only after `postTokenForm` has already fully buffered the token response with `arrayBuffer()`, so the OpenAI Codex OAuth OOM path remains.

## What This Changes

The branch replaces raw OpenAI Codex OAuth token response `text`/`json` reads with shared bounded readers and adds runtime tests for oversized and malformed token responses.

## Best Possible Solution

Move the bounded success and error reads into the guarded-response lifetime, remove the raw `arrayBuffer()` clone, and keep a regression that fails if the token response is consumed before the cap.

## Reproduction Assessment

Yes from source inspection: `postTokenForm` consumes the guarded token response with `response.arrayBuffer()` before the new bounded readers can run. I did not run tests because this review is read-only.

## Solution Assessment

No. The best fix is to move bounded JSON/text handling to the original guarded response before `release()`, not to parse a cloned in-memory response after unbounded buffering has already occurred.
