## Summary

Keep open. The PR targets a real avatar bug, but this branch should not merge as-is because it replaces current shared Windows-aware path containment, drops resolver-side avatar size validation, and lacks after-fix real behavior proof.

## What This Changes

The PR changes IDENTITY.md avatar parsing and local avatar resolution, adds avatar parser/resolver tests plus a changelog entry, and includes unrelated onPayload type-cast updates.

## Best Possible Solution

Refresh or replace this with a narrow current-main patch that unwraps single-backtick IDENTITY.md avatar values while preserving shared path containment, max-size validation, public metadata/source behavior, and route coverage.

## Reproduction Assessment

Yes, at source level for the backtick-wrapped avatar value: current main preserves the backticks in parseIdentityMarkdown, so local avatar resolution receives a bad path/extension. The original Windows UI 404 has credible issue evidence, but I did not establish a live Windows reproduction on current main.

## Solution Assessment

No. Backtick unwrapping is useful, but this branch replaces the shared Windows-aware avatar path policy and drops resolver-side max-size validation; the safer solution is a current-main patch using the existing avatar-policy helpers.
