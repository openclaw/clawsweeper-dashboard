## Summary

Keep open. The PR contains a useful backtick-parsing fix and the author did address the early Greptile review, but the current branch regresses avatar path/size policy and still lacks real gateway/UI behavior proof, so it should not merge as-is.

## What This Changes

The PR changes IDENTITY.md avatar parsing and local avatar resolution, adds avatar parser/resolver tests plus a changelog entry, and includes unrelated onPayload type-cast updates.

## Best Possible Solution

Refresh or replace this with a narrow current-main avatar fix that unwraps single-backtick IDENTITY.md avatar values while preserving shared path containment, max-size validation, public metadata/source behavior, and focused route coverage.

## Reproduction Assessment

Yes, at source level for the backtick-wrapped avatar value: current main preserves the backticks in parseIdentityMarkdown, so the local resolver receives a bad file extension/path. The original Windows UI 404 has credible issue evidence, but I did not establish a live Windows reproduction on current main.

## Solution Assessment

No. Backtick unwrapping is useful, but this branch replaces the shared Windows-aware avatar path policy and drops resolver-side max-size validation; the safer path is a current-main patch using the existing avatar-policy helpers.
