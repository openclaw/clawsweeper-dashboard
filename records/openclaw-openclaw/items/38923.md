## Summary

Keep open: current main still has a source-level backtick-wrapped `IDENTITY.md` avatar failure, and this PR is the open implementation candidate, but the branch has merge-blocking correctness issues and lacks real behavior proof.

## What This Changes

The branch unwraps backtick avatar values from `IDENTITY.md`, rewrites local avatar path resolution, adds parser/resolver tests and a changelog entry, and includes unrelated `onPayload` type-cast edits.

## Best Possible Solution

Refresh or replace this branch with a narrow current-main fix that unwraps single-backtick `IDENTITY.md` avatar values while preserving shared avatar policy, max-size handling, metadata behavior, and focused route/resolver tests.

## Reproduction Assessment

Yes, at source level: current main preserves backticks in `parseIdentityMarkdown`, and the resolver plus `/avatar` route mapping explains how a backtick-wrapped local path becomes a 404. The original Windows packaged runtime path has credible issue evidence but was not live-reproduced here.

## Solution Assessment

No. The backtick parser change is useful, but this PR also replaces shared avatar-policy helpers and drops resolver-side max-size validation; the safer fix is a smaller current-main patch that keeps those contracts.
