## Summary

Keep this PR open, but not mergeable as-is: it contains a useful `IDENTITY.md` backtick parser fix, while the branch is conflicting, lacks real behavior proof, and regresses current avatar safety/metadata contracts.

## What This Changes

This PR changes WebChat/Control UI avatar parsing and resolution for workspace `IDENTITY.md` avatars, adds parser/resolver tests, and includes unrelated `onPayload` type-cast updates.

## Best Possible Solution

Refresh or replace the branch with a narrow current-main patch that unwraps single-backtick `IDENTITY.md` avatar values while preserving shared avatar policy helpers, source metadata, UI override behavior, `AVATAR_MAX_BYTES`, and focused parser/resolver/route coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, at source level: current main preserves backticks in parsed `IDENTITY.md` avatar values, so a backtick-wrapped local avatar can be resolved literally and fail extension/path lookup; I did not live-run the Windows gateway flow.

## Solution Assessment

Is this the best way to solve the issue? No. The parser fix is useful, but this branch replaces shared avatar policy helpers and drops resolver-side max-size rejection; a smaller current-main refresh is safer.
