Keep this PR open, but it should not merge as-is. The linked avatar bug is still real on current main because `IDENTITY.md` avatar values wrapped in single backticks are preserved, while this PR's useful parser direction is bundled with a resolver regression that drops the existing avatar max-size rejection and with unrelated wrapper churn.

## What This Changes

The PR updates agent identity parsing and avatar resolution tests for workspace `IDENTITY.md` avatars and backtick-wrapped values, adds a changelog entry, and changes several `onPayload` wrapper call sites.

## Best Possible Solution

Land a focused current-main avatar fix that unwraps single-backtick `IDENTITY.md` avatar values, preserves shared avatar-policy helpers and `AVATAR_MAX_BYTES` behavior, keeps Control UI avatar metadata contracts intact, and adds parser/resolver/HTTP regression coverage for the reported workspace-only setup.

## Reproduction Assessment

Yes. A backtick-wrapped `Avatar:` value is preserved by the current parser, then fed into local avatar resolution where the trailing backtick makes the path fail extension/file validation and ultimately produce a 404; the current tests cover plain `IDENTITY.md` fallback but not the backtick case.

## Solution Assessment

No. The parser unwrapping is the right narrow fix, but this PR is not the best implementation because it removes existing max-size validation and broadens the diff beyond the avatar bug.
