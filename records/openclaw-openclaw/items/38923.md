## Summary

Keep this PR open, but do not merge the current head. It addresses part of the avatar parsing problem, yet the branch replaces shared avatar policy code, loses current Windows path-normalization behavior, removes max-size validation, and lacks external real behavior proof.

## What This Changes

The PR strips backticks from `IDENTITY.md` avatar values, normalizes relative avatar paths, adds avatar parser/resolver tests and a changelog entry, and includes unrelated `onPayload` typing casts.

## Best Possible Solution

Land or replace this with a focused current-main avatar fix that preserves shared fs-safe/avatar-policy helpers, `AVATAR_MAX_BYTES`, source metadata, and route behavior while adding only the needed `IDENTITY.md` parsing and Windows/custom-workspace regression coverage.

## Reproduction Assessment

Yes, at source level. The PR regressions are visible from the branch source: mixed Windows namespace and drive-letter paths are rejected by the local helper, and oversized local avatars resolve as local because the max-size check was removed; no live Windows UI run was performed.

## Solution Assessment

No. Unwrapping `IDENTITY.md` avatar values is a reasonable narrow fix, but this implementation is not the best solution because it reimplements shared path policy, drops existing size validation, and carries unrelated wrapper churn.
