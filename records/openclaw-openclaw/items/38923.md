## Summary

Keep open: this targets a real WebChat/Control UI avatar bug, but the current PR head is not merge-ready because it drops existing avatar path and size-limit policy and lacks contributor-supplied real behavior proof.

## What This Changes

The PR strips backtick-wrapped avatar values from IDENTITY.md, rewrites local workspace avatar resolution for `/avatar/{agentId}`, adds parser/resolver tests and a changelog entry, and updates unrelated onPayload casts.

## Best Possible Solution

Land a narrow current-main fix that unwraps single-backtick IDENTITY.md avatar values while preserving shared avatar-policy helpers, max-size rejection, public source metadata, UI override behavior, and focused parser/resolver/route coverage.

## Reproduction Assessment

Yes, at source level: current main preserves backticks in `parseIdentityMarkdown`, and the resolver/route mapping explains how that can become a 404. The original Windows packaged-runtime symptom has credible issue evidence but was not live-reproduced here.

## Solution Assessment

No. Backtick unwrapping is useful, but this branch replaces current shared avatar policy and drops resolver-side max-size validation; the safer fix is a smaller current-main patch that keeps those contracts.
