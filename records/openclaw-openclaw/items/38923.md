## What This Changes

The PR updates workspace `IDENTITY.md` avatar parsing/resolution for `/avatar/{agentId}`, adds avatar regression tests and a changelog entry, and changes several `onPayload` wrapper call sites.

## Best Possible Solution

Land a focused current-main avatar fix that unwraps single-backtick `IDENTITY.md` avatar values, preserves shared avatar-policy helpers, source metadata, and `AVATAR_MAX_BYTES` behavior, and avoids unrelated wrapper churn.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a backtick-wrapped `Avatar:` value is preserved by the parser, then the resolver treats the backticks as part of the path and fails local avatar resolution.

## Solution Assessment

No. Unwrapping the parsed value is the right narrow fix, but this PR is not the best implementation because it drops the existing max-size guard and changes unrelated wrapper code.
