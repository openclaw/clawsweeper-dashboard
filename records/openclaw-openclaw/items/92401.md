## Label Justifications

- `P2`: The fixed audit thresholds can make healthy long-running work look unhealthy and mislead operator automation, but the confirmed defect is limited to diagnostics rather than a core runtime outage.
- `impact:session-state`: The report concerns active subagent, task, and TaskFlow lifecycle state being misclassified as stale or failed.

## Summary

Current main still has a real fixed-threshold task-health problem: both task and TaskFlow audits use 30-minute stale limits without the configured subagent run deadline. The report should remain open, but its four proposals need separation because historical filtering is already tracked by https://github.com/openclaw/openclaw/issues/83253, the human CLI already prints an error/warning summary header, and `delivery_failed` is emitted from delivery state rather than from the stale timer.

## What This Changes

Make task auditing distinguish healthy long-running subagents from genuinely stale work while reducing historical and timestamp-noise findings and improving operator triage.

## Best Possible Solution

Store an authoritative expected deadline or equivalent lifecycle fact in canonical task/flow state, use it only for active stale classification, keep delivery failure independently visible, and resolve historical suppression or acknowledgement through the canonical related issue.

## Reproduction Assessment

Yes for the fixed-threshold portion: current main deterministically flags active task and TaskFlow records after fixed 30-minute windows without consulting the configured subagent deadline. The exact 154-finding dataset and the asserted timer-to-`delivery_failed` link were not independently reproduced.

## Solution Assessment

No, not as one bundled implementation. Deadline-aware stale classification is the narrow correctness fix, while new flags, default suppression, aggregation semantics, and historical acknowledgement require separate product decisions, with historical separation already tracked by https://github.com/openclaw/openclaw/issues/83253.
