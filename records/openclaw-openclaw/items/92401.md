- `impact:session-state`: The report concerns active and retained task/subagent lifecycle state being incorrectly classified in health output.

## Summary

Current main still has a real deadline-blind liveness defect: task and TaskFlow audits classify unchanged running work as stale after fixed 30-minute windows, even when a subagent is configured to run longer. The issue should remain open, but it combines that bug with historical-record filtering already tracked by https://github.com/openclaw/openclaw/issues/83253, a summary-header request already present on main, and a `delivery_failed` explanation that does not match current source.

## What This Changes

Make task auditing avoid false failure signals for healthy long-running subagents and reduce noise from retained or repetitive findings.

## Best Possible Solution

Persist an authoritative expected deadline or equivalent prepared liveness fact in canonical task state, use it consistently for task and linked-flow auditing, and resolve retained-history classification through the existing canonical issue without changing delivery-failure semantics or duplicating CLI policy.

## Reproduction Assessment

Yes for the fixed-threshold defect: current-main source deterministically marks unchanged running tasks and TaskFlows stale after 30 minutes without consulting configured run duration. The exact 154 findings and the asserted stale-timer cause of `delivery_failed` were not reproduced.

## Solution Assessment

No as one combined implementation: deadline-aware liveness is the narrow root repair, while new filtering flags, default suppression, aggregation, and retained-history UX require separate product and compatibility decisions.
