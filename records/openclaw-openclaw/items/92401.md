## Summary

Keep open: current main and v2026.6.8 still use fixed 30-minute task and TaskFlow audit thresholds, only partially discount retained-lost history, and do not provide the requested freshness/default-suppression or grouping contract. The broader retained-history work in https://github.com/openclaw/openclaw/issues/83253 overlaps, but this issue adds distinct timeout-aware long-running subagent and audit-output requirements.

## What This Changes

Request changes `openclaw tasks audit` to respect long-running subagent timeouts, reduce timestamp and historical TaskFlow noise, add freshness filtering, and improve severity summary output.

## Best Possible Solution

Define one maintainer-approved task/TaskFlow health contract covering timeout-aware active-run audits, current-vs-historical filtering, grouped timestamp anomalies, JSON/text compatibility, docs, and regression coverage while linking the retained-history split to https://github.com/openclaw/openclaw/issues/83253.
