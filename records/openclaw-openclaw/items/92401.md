## Summary

Keep open: current main and v2026.6.9 still use fixed 30-minute task and TaskFlow audit thresholds, subagent task records do not carry `runTimeoutSeconds`, and the requested fresh/history/grouping CLI contract is still missing. https://github.com/openclaw/openclaw/issues/83253 overlaps on historical task noise but does not own this issue's timeout-aware audit behavior.

## What This Changes

The issue requests timeout-aware task audit health, grouped timestamp noise, fresh/history filtering, and a severity-aware CLI summary for `openclaw tasks audit`.

## Best Possible Solution

Define one maintainer-approved task-audit health model that carries timeout/deadline facts into audit decisions, separates current from historical findings, groups repeated timestamp noise, and preserves an explicit all/history mode for operators.

## Reproduction Assessment

Yes at source level: current main uses fixed 30-minute task and TaskFlow audit thresholds while subagent `runTimeoutSeconds` is stored on subagent run records and not on `TaskRecord`. I did not run the reporter's live multi-subagent workload in this read-only review.

## Solution Assessment

Unclear: the timeout-aware bug fix is well grounded, but the requested `--fresh`/history suppression, grouping, and human/JSON summary semantics need maintainer product direction before implementation.
