## Summary

Keep open. Current main and v2026.6.6 still use fixed 30-minute task/task-flow audit thresholds and a combined flat audit output, while the related historical-noise issue only covers part of the requested behavior.

## What This Changes

The issue asks for `openclaw tasks audit` to stop reporting healthy long-running subagents and old task history as current failures, with timeout-aware liveness, grouped timestamp noise, fresh/historical filtering, and clearer severity summaries.

## Best Possible Solution

Define one maintainer-approved task audit model that respects runtime-owned liveness budgets, separates current from historical findings by default, groups repeated timestamp anomalies, and keeps full raw history available through an explicit option.

## Reproduction Assessment

Yes at source level: current main uses fixed 30-minute audit thresholds, lacks run-timeout data on TaskRecord, and prints combined findings without fresh/historical filtering. I did not replay the reporter's live four-subagent workload in this read-only review.

## Solution Assessment

Unclear as a complete fix: respecting subagent run timeouts is the right bug direction, but the proposed fresh/default filtering and JSON/text output contract need maintainer product direction before implementation.
