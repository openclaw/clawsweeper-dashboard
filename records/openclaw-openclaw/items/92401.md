## Summary

Keep open: current main and v2026.6.8 still use fixed task/TaskFlow audit thresholds, row-level timestamp findings, and only partial retained-lost discounting; the linked retained-history issue overlaps but does not cover the distinct timeout-aware and audit-output contract requested here.

## What This Changes

The issue requests a quieter `openclaw tasks audit` contract that respects long-running subagent timeouts, groups timestamp noise, separates fresh from historical findings, and gives severity-aware CLI/JSON triage output.

## Best Possible Solution

Define one maintainer-approved task-audit health model that keeps live failures visible, suppresses or groups historical noise by default, respects configured subagent run deadlines where data is available, and preserves an explicit full-output mode for scripts.

## Reproduction Assessment

Yes at source level: current main still has fixed audit thresholds, no timeout field in task records, and no freshness or historical-output option. I did not reproduce the reporter's live 154-finding system state in this read-only review.

## Solution Assessment

Unclear as an implementation path: the requested direction is sensible, but changing default audit suppression, JSON shape, and timeout semantics needs maintainer approval before a narrow fix is safe.
