## Summary

Keep open: current main and v2026.6.9 still have fixed task/TaskFlow audit thresholds, no timeout data in task audit records, and no freshness/history/grouping contract, so the reported false-positive audit noise is not implemented away.

## What This Changes

The issue requests a quieter, timeout-aware `openclaw tasks audit` that respects subagent run timeouts, groups timestamp anomalies, separates current findings from historical records, and improves CLI/JSON triage output.

## Best Possible Solution

Define one maintainer-approved task audit contract covering timeout-aware active-run checks, current-vs-historical filtering, grouped anomalies, compatible JSON/text output, docs, and regression coverage.
