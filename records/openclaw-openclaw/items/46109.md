## Summary

Keep this issue open. Current main has the SecretRef/runtime/docs foundation, but the requested source-aware Control UI config UX is still missing: structured SecretRefs render as empty read-only generic placeholders and SecretInput schemas still use the plain string editor path.

## What This Changes

The issue requests first-class `.env` and environment-backed secret UX in CLI and Control UI config surfaces, including masked source-aware display and config/env/unset editing.

## Best Possible Solution

Define a redacted SecretRef source/status metadata contract, then implement a source-aware SecretInput renderer with aligned CLI/docs/tests and no resolved secret values sent to the browser.

## Reproduction Assessment

Yes. Static inspection plus the existing browser test gives a high-confidence reproduction path: an env SecretRef at channels.discord.token renders as a read-only empty input with only a generic Structured SecretRef placeholder.

## Solution Assessment

Unclear until maintainers set the disclosure boundary. The narrow maintainable path is a redacted metadata contract plus Control UI/docs/tests, not reusing resolved-secret payloads or adding placeholder-only labels.
