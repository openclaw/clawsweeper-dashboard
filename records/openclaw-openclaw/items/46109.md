## Summary

Keep this issue open. Current main has SecretRef runtime, CLI, redaction, and docs foundations, but the requested source-aware Control UI/config UX is still missing: structured SecretRefs render as read-only generic placeholders and SecretInput unions still use the plain string editor path.

## What This Changes

Request first-class environment-backed secret UX that shows masked config/env/unset source state and preserves env references in CLI and Control UI config editing.

## Best Possible Solution

Define a redacted SecretRef source/status metadata contract, then implement a source-aware SecretInput renderer with aligned CLI/docs/tests and no resolved secret values sent to browser clients.

## Reproduction Assessment

Yes. The current Control UI rendering path and existing browser test for channels.discord.token with an env SecretRef show a read-only empty input with a generic Structured SecretRef placeholder instead of source-aware metadata.

## Solution Assessment

Unclear until maintainers set the disclosure boundary. The narrow maintainable solution is a redacted metadata contract plus UI/docs/tests, not reusing value-bearing secrets.resolve payloads or adding placeholder-only labels.
