## Summary

Keep this open. Current main has SecretRef runtime, redaction/restore, CLI, and docs groundwork, including the merged hardening from https://github.com/openclaw/openclaw/pull/58044, but the requested source-aware Control UI/config UX is still missing.

## What This Changes

The issue requests first-class environment-backed secret configuration UX with masked source-aware config/env/unset display and editing in Control UI and CLI helpers.

## Best Possible Solution

Add a source-aware SecretInput renderer and metadata flow on top of existing SecretRef contracts, with masked display, safe source labels, env identity when approved, resolved/unresolved status, explicit config/env/unset editing, and aligned docs/tests.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source, docs, and test inspection are enough to show current main still uses generic read-only SecretRef placeholders rather than source-aware secret UX.

## Solution Assessment

Yes as a product direction, but not as an automated bug fix. The maintainable path is to extend the existing SecretRef/secret-target model into Control UI instead of adding ad hoc field-specific badges.
