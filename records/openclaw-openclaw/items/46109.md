## Summary

Keep this open. Current main has solid SecretRef runtime, audit/configure docs, and config write hardening, including the merged work in https://github.com/openclaw/openclaw/pull/58044 that superseded https://github.com/openclaw/openclaw/pull/55034, but the requested source-aware Control UI/config UX is still missing: structured SecretRefs render as empty read-only placeholders rather than config/env/unset state.

## What This Changes

The issue requests first-class environment-backed secret configuration UX so secret fields can show masked, source-aware config/env/unset state instead of looking blank.

## Best Possible Solution

Extend the existing SecretRef and secret-target contracts into a source-aware SecretInput renderer with masked display, safe source labels, approved env identity, resolved/unresolved status, explicit config/env/unset editing, and aligned docs/tests.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source and test inspection still show the current Control UI renders env SecretRefs as generic read-only empty inputs rather than source-aware fields.

## Solution Assessment

Yes as a product direction, but not as an automated bug fix. The maintainable path is to build on the existing SecretRef registry and config UI schema analysis rather than adding field-specific badges.
