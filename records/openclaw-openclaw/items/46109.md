## Summary

Keep open. Current main has strong SecretRef runtime, CLI, and round-trip hardening, but the requested source-aware Control UI/config UX is still missing: structured refs render as generic empty read-only placeholders and the docs document that limitation.

## What This Changes

The issue requests first-class env-backed secret configuration UX in CLI and Control UI, including source-aware masked fields, env-reference preservation, and clearer migration guidance.

## Best Possible Solution

Add a source-aware SecretInput renderer and metadata flow on top of existing SecretRef contracts: masked source labels, safe env identity/resolution status, config/env/unset editing, and aligned docs/tests.

## Reproduction Assessment

Yes. Source inspection shows the current Control UI form path renders structured SecretRefs as read-only empty placeholders, and the browser test asserts that behavior; I did not run a browser session because this review is read-only.

## Solution Assessment

Yes directionally, but not as an automatic fix. The maintainable solution is to extend the existing SecretRef contracts with deliberate UI metadata and disclosure rules rather than patching placeholders in isolation.
