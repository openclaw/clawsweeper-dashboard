## Summary

Keep open: this is a focused and useful memory-wiki bug fix, but the patch moves the reserved-stem rule into the shared slug helper, which also feeds persisted bridge, unsafe-local, and OKF identities. That broadens the fix into compatibility-sensitive path/id churn that should be narrowed before merge.

## What This Changes

The PR changes `memory-wiki` slug generation to suffix reserved `index`/`log` stems and adds repro coverage for `create_synthesis` and `ingest` titled `Index`.

## Best Possible Solution

Keep the reserved-stem protection at the direct wiki page-stem allocation boundary for synthesis and ingest, while preserving `slugifyWikiSegment` output for namespaced or hash-composed callers and covering both paths with focused regressions.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main maps `Index` to `index.md`, while compile/query/status treat `index.md` as generated or non-queryable. The PR body also includes before/after focused test output, though this read-only review did not run tests.

## Solution Assessment

No, this is not the best fix as written. The direct reserved-page bug is real, but changing the shared slug helper is broader than necessary and should be replaced with a narrower page-stem guard or wrapper.
