## What This Changes

The PR adds a memory-wiki page-stem helper for the reserved `index` stem and routes synthesis creation plus source ingest through it so they no longer write user content to compiler-owned `<dir>/index.md` files.

## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection shows current main writes `Index` synthesis/source content to `index.md`, and compile/query/status exclude or rewrite that filename; I did not run tests because this review is read-only, but the PR body includes before/after on-disk memory-wiki output.

## Solution Assessment

Yes. The dedicated direct page-stem helper is the best boundary: changing shared `slugifyWikiSegment` would churn bridge/unsafe-local/OKF composed identities, while treating `index.md` as user content would conflict with compiler-owned directory indexes.
