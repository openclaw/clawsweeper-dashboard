## Summary

Keep this PR open, but do not merge it as-is: the `Index` data-loss fix belongs in the slug layer, while the added global `log` reservation is not backed by current compiler behavior and can fork existing pages titled `Log` after upgrade.

## What This Changes

This PR changes memory-wiki slug generation to suffix titles whose slug is `index` or `log`, and adds a regression test for `Index` synthesis/source retrieval.

## Best Possible Solution

Keep the slug-layer fix for compiler-generated `index.md`, but remove `log` from the global reserved stem set unless maintainers also add an explicit `log.md` ownership contract, migration, and upgrade tests.

## Reproduction Assessment

Yes. Source inspection shows `Index` titles currently write to `syntheses/index.md` or `sources/index.md`, while compile/query/status exclude or overwrite `index.md`; I did not run the repro because this review is read-only.

## Solution Assessment

No. The canonical slug layer is the right boundary for `index`, but adding `log` there is too broad unless OpenClaw first makes `log.md` a regular compiler-owned page with upgrade handling.
