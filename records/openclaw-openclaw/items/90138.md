## Best Possible Solution

Land the narrow wrapper/test fix after updating the MiniMax thinking documentation to describe the legacy M2.x shim and the MiniMax-M3 exception.

## Solution Assessment

Yes, with a docs follow-up: the wrapper is the narrow owner of the automatic payload injection, the M3 guard preserves M2.x behavior, and explicit thinking payloads remain under the underlying Anthropic stream path. The remaining fix is to align provider/thinking docs with the new model-specific behavior.
