## Summary

Keep this open. Current main still defaults `commands.nativeSkills` to `"auto"`, still resolves that default on for Discord and Telegram, and the remaining default-change request requires maintainer product/migration judgment rather than conservative cleanup.

## What This Changes

Request to change the default `commands.nativeSkills` behavior from `"auto"` to opt-in/disabled so Discord and Telegram do not register every skill as a native command by default.

## Best Possible Solution

Make an explicit maintainer choice: either flip the default with coordinated schema/docs/tests/changelog migration, or keep `auto` and land a narrower diagnostics path that points users to `commands.nativeSkills` and provider overrides.

## Reproduction Assessment

Yes for the remaining default behavior: current source, generated schema, docs, and resolver tests all show `commands.nativeSkills: "auto"` still enables native skill commands for Discord and Telegram. The exact 2026.2.21 startup-failure mode is partly mitigated now by Discord dropping per-skill commands and Telegram capping/retrying registration, so I did not treat that crash as reproduced on current main.

## Solution Assessment

Unclear. The requested default flip is coherent, but it is a backward-affecting product/migration decision; the narrower #50191 warning approach helps diagnostics but does not implement the requested opt-in default.
