## Summary

Keep open. Current main still defaults native skill commands to `"auto"`, but source inspection shows Discord and Telegram now have overflow-degradation paths, and the exact gateway-startup failure claim has not been proven against current main. The remaining decision is a product/config default tradeoff, with a narrower open diagnostics PR already tracking the likely safer path.

## What This Changes

Request to change the default `commands.nativeSkills` behavior from automatic native command registration to disabled so large skill sets do not exhaust Telegram or Discord command quotas.

## Best Possible Solution

Maintainers should decide the default policy explicitly; if `auto` remains the default, land a focused diagnostics fix that points overflow users at `commands.nativeSkills: false`, and if the default changes, update schema, docs, generated baselines, tests, and migration notes together.

## Reproduction Assessment

No. Source inspection confirms the default is still `"auto"` and that overflow branches are reachable, but current main also caps or degrades command registration, so the reported full gateway-startup failure is not established without live 100-command Telegram/Discord proof.

## Solution Assessment

Unclear. A default flip may be valid product policy, but the current source and the closed default-change PR point toward improved overflow diagnostics as the narrower maintainable fix unless maintainers intentionally choose the compatibility break.
