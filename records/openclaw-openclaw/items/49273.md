## Summary

Current main and the latest release still leave this as an unresolved policy decision: `commands.nativeSkills` defaults to `"auto"`, Discord and Telegram still opt into native skill auto defaults, and docs/tests describe that as expected. The old Telegram hard-failure path has been partly mitigated by capping and retry behavior, but the requested default flip has not been implemented.

## What This Changes

Request to change the default for `commands.nativeSkills` from `"auto"` to disabled so skills are not registered as Discord/Telegram native platform commands unless users opt in.

## Best Possible Solution

Make an explicit maintainer policy decision for native skill command registration, then align schema defaults, generated config metadata, help text, docs, tests, overflow diagnostics, and changelog with that decision.

## Reproduction Assessment

Yes for the remaining default behavior: current source, tests, release content, manifests, and docs all show `commands.nativeSkills: "auto"` still enables Discord and Telegram native skill commands. The exact old startup-crash path is only source-plausible now because current overflow handling partly mitigates platform limits.

## Solution Assessment

Unclear. The requested default flip is coherent and scoped, but it changes established command-menu behavior; a maintainer should choose between changing the default and keeping `auto` with better diagnostics such as https://github.com/openclaw/openclaw/pull/50191.
