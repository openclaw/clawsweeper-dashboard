## Summary

Keep this open. Current main still defaults `commands.nativeSkills` to `"auto"`, still resolves that default on for Discord and Telegram, and still documents the same behavior; the newer overflow handling reduces the startup-failure blast radius but does not implement the requested opt-in default.

## What This Changes

The issue requests changing the default native skill command registration policy from `commands.nativeSkills: "auto"` to disabled/opt-in to avoid Discord and Telegram command quota exhaustion.

## Best Possible Solution

Keep this issue open until maintainers decide the product policy. If the default should flip, update the config schema default, generated schema/help, docs, resolver tests, Discord/Telegram expectations, and changelog migration note together; if `auto` should stay, land or close the narrower overflow-guidance work and make the warnings/docs point clearly at `commands.nativeSkills` and per-provider overrides.

## Reproduction Assessment

Yes for the remaining default behavior: source, generated schema, docs, and resolver tests deterministically show `commands.nativeSkills: "auto"` still enables native skill commands for Discord and Telegram on current main. The exact 2026.2.21 hard startup failure was not treated as fully current because Telegram and Discord now have overflow mitigations.

## Solution Assessment

Unclear. The requested default flip is a coherent way to avoid command quota exhaustion, but it is also a product/migration decision; the safer narrower alternative is to keep `auto` and improve overflow diagnostics and docs around `commands.nativeSkills`.
