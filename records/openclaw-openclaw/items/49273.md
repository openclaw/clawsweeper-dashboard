## Summary

Keep this open. Current main still defaults `commands.nativeSkills` to `"auto"` and still enables that auto path for Discord and Telegram; changing the default remains a maintainer product/migration decision, while the open warning-copy PR addresses only diagnostics.

## What This Changes

Request to change the global `commands.nativeSkills` default from auto-registration to opt-in so Discord and Telegram do not spend native command quota on every skill by default.

## Best Possible Solution

Choose the command-registration policy explicitly, then keep schema defaults, generated schema/help, docs, tests, changelog, and channel overflow guidance aligned with that decision.

## Reproduction Assessment

Yes for the remaining default behavior: source, docs, and tests all show `commands.nativeSkills: "auto"` still enables Discord and Telegram native skill commands. The exact old startup-failure mode is not established on current main because Telegram now caps menu registration and Discord drops per-skill commands over the limit.

## Solution Assessment

Unclear. The requested default flip is coherent, but it changes existing command-menu behavior; the narrower warning approach in #50191 improves diagnosis without settling the default policy.
