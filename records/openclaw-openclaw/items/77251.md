## Summary

Keep open: current main still has the direct generic ACP terminal delivery path, and #77365 is an open unmerged PR with closing syntax for this issue, so the issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

The issue requests that group-channel background ACP/subagent completions wake the parent session for a real user-facing reply instead of stopping at direct `Background task ...` lifecycle text.

## Best Possible Solution

Review #77365 and land it if it correctly routes parent-owned group-channel ACP completions through the parent announce flow with direct lifecycle text only as fallback telemetry.

## Reproduction Assessment

Yes, at source level: current main allows ACP terminal tasks through `shouldAutoDeliverTaskTerminalUpdate`, then `maybeDeliverTaskTerminalUpdate` formats `Background task ...` text and sends it directly to the requester origin. Existing tests assert the same direct-delivery shape, though I did not run a live group-channel harness in this read-only review.

## Solution Assessment

Yes, the narrow maintainable direction is to reuse the parent-agent announce/completion path for parent-owned ACP runs and keep raw lifecycle messages as fallback/status telemetry. The linked open PR is the current implementation candidate, so closing this issue before merge would be premature.
