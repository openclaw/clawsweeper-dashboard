## Summary

Keep open: current main still has a source-visible ACP terminal delivery path that can send generic background-task text directly to a group/channel, and open PR #77365 is already the linked implementation candidate. https://github.com/openclaw/openclaw/issues/77251

## What This Changes

The issue requests parent-agent completion replies for background ACP/subagent work in group channels instead of stopping at generic lifecycle status messages.

## Best Possible Solution

Review and land PR #77365, or an equivalent narrow change, so parent-owned ACP completions wake the parent announce path with rich child-result context while direct lifecycle text remains fallback telemetry.

## Reproduction Assessment

Yes, at source level: create a pending session-scoped ACP task with a deliverable `requesterOrigin`, let it reach terminal state, and `maybeDeliverTaskTerminalUpdate` sends formatted `Background task ...` content directly while current tests assert the parent queue remains empty.

## Solution Assessment

Yes, the maintainable direction is to route parent-owned ACP completion through the existing parent announce machinery; the open linked PR is the right review target before this issue closes.
