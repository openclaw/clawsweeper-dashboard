## Summary

Keep open: current main still has the source-visible direct terminal delivery path, and open PR #77365 already references this issue as the implementation candidate.

## What This Changes

The issue asks for background ACP/subagent completions in group channels to wake the parent-agent reply path instead of stopping at generic `Background task ...` lifecycle messages.

## Best Possible Solution

Review and land PR #77365, or an equivalent narrow change, so parent-owned group/channel ACP completions wake the parent announce path with rich child-result context and keep generic lifecycle text only as fallback telemetry.

## Reproduction Assessment

Yes, at source level: create a pending ACP task with a deliverable `requesterOrigin`, let it reach a terminal state, and `maybeDeliverTaskTerminalUpdate` sends the formatted `Background task ...` message directly while existing tests assert the parent queue remains empty.

## Solution Assessment

Yes for the linked direction: routing parent-owned ACP completion through the existing parent announce/session handoff is narrower and more maintainable than adding a user toggle or changing generic channel delivery globally.
