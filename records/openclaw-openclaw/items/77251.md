## Summary

Keep open: current main still has a source-level ACP task completion path that can direct-deliver generic `Background task ...` text to a requester channel, and open PR #77365 already references this issue with closing syntax.

## What This Changes

The issue asks for group-channel background ACP/subagent completions to route through the parent agent so users receive a real completion summary instead of only generic lifecycle status text.

## Best Possible Solution

Review and land or revise #77365 so ACP group-channel terminal completions wake the parent-agent announce path with child-result context, then close this issue after merge.

## Reproduction Assessment

Yes. At source level, create a pending ACP task with a deliverable requester origin and let it reach a terminal state; current `maybeDeliverTaskTerminalUpdate` formats `Background task ...` text and sends it directly, with current tests asserting that path.

## Solution Assessment

Yes at the direction level: the narrow maintainable fix is to route parent-owned ACP completions through the existing parent-agent announce model and keep generic lifecycle text only as fallback telemetry. The open PR is the right item to review for that implementation.
