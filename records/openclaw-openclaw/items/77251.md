## Summary

Keep open: current main still has a source-reproducible ACP task delivery path that can direct-post generic `Background task ...` terminal text, and an open linked PR claims the fix with closing syntax, so the issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

The issue asks for group-channel ACP/subagent background completions to be routed through the parent agent for a real user-facing summary instead of stopping at generic lifecycle text.

## Best Possible Solution

Review and land the linked parent-routing PR, or replace it with an equivalent fix that sends ACP parent-owned completion context through the parent announce/session path while keeping generic lifecycle text only as fallback telemetry.
