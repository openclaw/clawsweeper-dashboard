## Summary

Keep open: current main still has a source-visible direct-delivery path for ACP terminal task updates, and open PR #77365 is already linked as the implementation candidate.

## What This Changes

Route background ACP/group-channel terminal completions through the parent-agent handoff instead of leaving users with only generic task-status messages.

## Best Possible Solution

Land PR #77365, or an equivalent narrow fix, so parent-owned ACP completions wake the parent announce flow and keep generic lifecycle text only as fallback telemetry.

## Reproduction Assessment

Yes, at source level: create a pending ACP task with a deliverable `requesterOrigin`, let it reach a terminal state, and `maybeDeliverTaskTerminalUpdate` sends formatted `Background task ...` text directly via `sendMessage`. Existing tests assert that direct send and an empty parent queue.

## Solution Assessment

Yes for the linked direction: routing parent-owned ACP completions through the existing parent announce machinery is the narrow maintainable fix. Direct lifecycle delivery should remain fallback/status telemetry rather than the primary user-facing answer.
