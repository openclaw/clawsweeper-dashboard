## Summary

Keep open: current main still has a source-backed ACP terminal-delivery path that can directly post generic lifecycle text, and open PR #77365 already references this issue with closing syntax, so cleanup should wait for that PR to land or be replaced.

## What This Changes

The issue asks for parent-owned background ACP/subagent completions in group channels to hand completion context back to the parent agent instead of leaving users with only generic task lifecycle text.

## Best Possible Solution

Land a focused fix, such as #77365 if review passes, that routes group-channel ACP terminal updates through the parent-agent announce path and keeps generic lifecycle text as fallback telemetry only.

## Reproduction Assessment

Yes. Current-main source inspection shows a pending ACP task with a deliverable requester origin reaches `maybeDeliverTaskTerminalUpdate`, formats `Background task ...`, and sends it directly; the existing test asserts that exact path and an empty parent event queue.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing parent-agent announce/completion machinery for parent-owned ACP completions, with direct generic lifecycle delivery retained only as fallback telemetry.
