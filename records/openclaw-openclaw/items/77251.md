## Summary

Keep open: current main still has the direct ACP terminal delivery path that can emit only generic lifecycle text, and the already-linked #77365 PR is open and unmerged as the implementation candidate.

## What This Changes

The issue asks for parent-owned background ACP/subagent completions in group channels to wake the parent-agent reply path instead of stopping at generic `Background task ...` status text.

## Best Possible Solution

Review and land or close #77365; the desired end state is parent-owned group ACP completions using the parent announce path with direct generic lifecycle text only as fallback telemetry.

## Reproduction Assessment

Yes, at source level: create a pending ACP task with a deliverable `requesterOrigin`, let it reach terminal state, and current `maybeDeliverTaskTerminalUpdate` sends `formatTaskTerminalMessage` directly through `sendMessage`. Existing tests assert the same `Background task done` direct-delivery path and empty parent event queue.

## Solution Assessment

Yes, the narrow maintainable direction is to route parent-owned ACP terminal updates through the existing parent-agent announce model and keep direct lifecycle sends as fallback/status telemetry. Because #77365 is already open with closing syntax, the issue should remain open until that PR is resolved.
