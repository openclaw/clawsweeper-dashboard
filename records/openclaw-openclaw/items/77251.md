## Summary

Keep open: current main still has the direct ACP terminal task delivery path described by the report, and open PR #77365 references this issue with closing syntax, so the issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

The issue reports that group/channel ACP or subagent background completions can surface only generic `Background task ...` lifecycle text instead of triggering a parent-agent follow-up reply.

## Best Possible Solution

Review and land or revise PR #77365 so group-channel ACP completions wake the parent announce path with enough child-result context, while generic lifecycle delivery remains only fallback telemetry.

## Reproduction Assessment

Yes, at source level: a pending ACP task with a deliverable requester origin reaches terminal state, remains eligible for auto-delivery, and sends `Background task ...` directly through `maybeDeliverTaskTerminalUpdate`. I did not run a live group-channel harness in this read-only review.

## Solution Assessment

Yes, the narrow maintainable direction is to route group-channel ACP completions through the existing parent-agent announce path and preserve direct generic task messages only when no parent handoff is possible.
