## Summary

Keep open. Current main still has the split-state exec approvals behavior, and this PR is the focused implementation candidate rather than obsolete shipped work. The remaining blocker is maintainer security/design review for host exec approval state-root semantics, especially fresh override directories and socket placement.

## What This Changes

This PR updates exec approvals file, default socket, and effective-policy host-source resolution, plus docs, changelog, and regression tests, so they honor `OPENCLAW_STATE_DIR` while preserving the `~/.openclaw` default when the variable is unset.

## Best Possible Solution

Land this PR or a maintainer-approved replacement that derives the approvals JSON, default socket, and effective-policy host-source from the active state-dir contract, keeps the unset default stable, updates user-facing docs/diagnostics, and explicitly records the accepted security semantics for migration and socket safety.

Yes. A high-confidence source-level reproduction is to set `OPENCLAW_STATE_DIR` and inspect current main: `resolveExecApprovalsPath()` and `resolveExecApprovalsSocketPath()` still expand `~/.openclaw` constants instead of the configured state root.

Unclear as merge-ready. The resolver direction is narrow and matches the reported bug, but the best final fix needs maintainer acceptance or adjustment of migration/warning and socket-directory safety behavior because this store gates host exec approvals.

## Security Review

Status: needs_attention

Summary: The diff has no dependency, workflow, lockfile, or publishing risk, but it moves host exec approval state and the default approval socket under an operator-controlled state root.

Concerns:

- **[medium] Review fresh state-dir approval policy semantics:** `src/infra/exec-approvals.ts:185`
  - body: When `OPENCLAW_STATE_DIR` points to a new directory, the PR would create/use a new approvals file instead of warning about or migrating an existing stricter `~/.openclaw/exec-approvals.json`. That may be intended state-dir behavior, but it should be explicit because this policy controls host exec prompts and durable allow rules.
  - confidence: 0.72
- **[medium] Review socket safety for overridden state roots:** `src/infra/exec-approvals.ts:210`
  - body: The default exec-approvals socket would move under `OPENCLAW_STATE_DIR` without a new ownership or mode check for shared/writable directories. If an operator points the state root at an unsafe location, a local process may be able to pre-bind or spoof the approval socket.
  - confidence: 0.64
