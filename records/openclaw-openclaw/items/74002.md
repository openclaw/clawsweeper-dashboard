## Summary

Keep open. Current main still hardcodes exec approval file, socket, and effective-policy host-source paths under `~/.openclaw`, while this PR is a narrow tested fix that still needs maintainer/security acceptance because it moves host exec approval state and the default approval socket under an env-selected state root.

## What This Changes

The PR makes exec approval file, socket, and display-path resolution honor `OPENCLAW_STATE_DIR`, then updates effective-policy reporting, docs, changelog, and focused regression tests.

## Best Possible Solution

Land this PR or a maintainer-approved replacement that derives approvals JSON, default socket, effective-policy host-source text, and docs from the active state-dir contract while explicitly deciding the migration and socket-directory safety semantics.

## Reproduction Assessment

Yes. A source-level reproduction is to set `OPENCLAW_STATE_DIR` and inspect current main: `resolveExecApprovalsPath()` and `resolveExecApprovalsSocketPath()` still expand `~/.openclaw` constants, and persistence flows call those helpers.

## Solution Assessment

Unclear as merge-ready. The resolver direction is the narrow maintainable fix for the reported split-state bug, but the final merge should include maintainer/security acceptance or adjustment of legacy-policy migration and socket-root safety behavior.

## Security Review

Status: needs_attention

Summary: The diff has no dependency, workflow, lockfile, or publishing risk, but it moves host exec approval policy and the default IPC socket under an env-selected state root.

Concerns:

- **[medium] Review fresh state-dir approval policy semantics:** `src/infra/exec-approvals.ts:189`
  - body: When `OPENCLAW_STATE_DIR` points at a new directory, this PR creates and uses a new approvals file instead of using or warning about an existing stricter `~/.openclaw/exec-approvals.json`. That may be intended isolation, but it should be explicit because this file gates host exec approvals.
  - confidence: 0.72
- **[medium] Review socket safety for overridden state roots:** `src/infra/exec-approvals.ts:212`
  - body: The default exec-approvals socket would move under `OPENCLAW_STATE_DIR` without ownership or mode checks for shared/writable directories. If an operator points the state root at an unsafe location, a local process may be able to pre-bind or spoof the socket.
  - confidence: 0.64
