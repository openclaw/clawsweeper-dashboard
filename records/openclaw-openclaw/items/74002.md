## Summary

Keep open. Current main still has the reported exec-approvals state-root bug, and this PR is useful source work, but the branch is reported unmergeable and the approval-state migration/socket-root security semantics still need maintainer review before landing.

## What This Changes

The PR makes exec-approvals JSON, default socket, effective-policy host-source text, docs, tests, and changelog honor `OPENCLAW_STATE_DIR` while preserving the unset default at `~/.openclaw`.

## Best Possible Solution

Land this PR or a maintainer-approved replacement that routes approvals JSON, default socket, effective host-source text, and all user-facing docs/diagnostics through the active state-dir contract while explicitly accepting or hardening migration and socket-root safety semantics.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: set `OPENCLAW_STATE_DIR` on current main and the resolver helpers still expand `~/.openclaw/exec-approvals.*` instead of using the configured state root. I did not run tests because this review was constrained to read-only inspection.

## Solution Assessment

Unclear as merge-ready. The resolver direction is narrow and matches the bug, but the best final fix needs maintainer/security judgment on legacy approval migration and unsafe socket-root handling.

## Security Review

Status: needs_attention

Summary: The diff has no dependency, workflow, lockfile, or publishing risk, but it moves host exec approval policy and the default approval socket under an env-selected state root.

Concerns:

- **[medium] Review fresh state-dir approval semantics:** `src/infra/exec-approvals.ts:189`
  - body: When `OPENCLAW_STATE_DIR` points to a new directory, the PR creates and uses a new approvals file rather than warning about or migrating an existing stricter default approvals file. That may be intended state-root isolation, but it should be explicit because omitted approvals defaults fall back to permissive host behavior.
  - confidence: 0.72
- **[medium] Review overridden socket-root safety:** `src/infra/exec-approvals.ts:214`
  - body: The default exec-approvals socket would follow `OPENCLAW_STATE_DIR`; if operators point it at a shared or unsafe directory, the core client sends the approval token in the first request and does not validate socket ownership before connecting. The macOS server hardens its parent directory and validates peer UID, but the cross-component safety contract still needs maintainer review.
  - confidence: 0.66
