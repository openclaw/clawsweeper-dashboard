Keep open: current main and the latest release still hardcode exec approvals under `~/.openclaw`, while this PR is the active implementation candidate and is currently conflicting with main with security-sensitive state-root semantics still unresolved.

## What This Changes

The PR routes exec approvals JSON, default socket, effective-policy host-source text, docs, tests, and changelog through `OPENCLAW_STATE_DIR` while keeping the unset default at `~/.openclaw`.

## Best Possible Solution

Land a rebased maintainer-approved fix that derives approvals JSON, default socket, and effective-policy host-source text from the active state-dir/profile contract, updates public docs and focused tests, and explicitly handles migration and socket-root safety semantics.
