## Summary

Keep open. Current main still has the exec-approvals split-state bug, and this PR is a useful narrow implementation candidate, but it changes host exec approval policy/socket placement under an environment-selected state root and needs maintainer security acceptance before merge.

## What This Changes

The PR makes exec approval JSON, default socket, effective-policy host-source text, docs, tests, and changelog honor `OPENCLAW_STATE_DIR` while preserving the unset default at `~/.openclaw`.

## Best Possible Solution

Land this PR or a maintainer-approved replacement that derives approvals JSON, default socket, effective host-source text, and user-facing docs from the active state-dir contract while explicitly accepting or hardening migration and socket-root safety semantics.

## Reproduction Assessment

Yes. On current main, set `OPENCLAW_STATE_DIR` and inspect `resolveExecApprovalsPath()` / `resolveExecApprovalsSocketPath()`; the helpers still expand `~/.openclaw/exec-approvals.*` instead of the configured state root.

## Solution Assessment

Unclear as merge-ready. The resolver direction is narrow and matches the reported bug, but the best final fix needs maintainer/security judgment on legacy approval migration and socket-directory safety because this store gates host exec.

Concerns:
