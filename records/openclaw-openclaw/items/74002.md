## Summary

Keep open. Current main still hardcodes exec approvals under `~/.openclaw`, and this PR is a focused fix with tests, docs, changelog, and source-PR credit, but merge should wait for maintainer/security judgment on fresh state-root policy and overridden socket-root safety; one public docs cleanup also remains.

## What This Changes

This PR makes exec approvals JSON/socket resolution, effective-policy host-source reporting, docs, tests, and changelog honor `OPENCLAW_STATE_DIR` while preserving the unset `~/.openclaw` default.

## Best Possible Solution

Land this PR or a maintainer-approved replacement that derives approvals JSON, default socket, effective host-source text, and user-facing docs from the active state-dir contract while explicitly accepting or hardening migration and socket-root safety semantics.

## Reproduction Assessment

Yes from source inspection. On current main, setting `OPENCLAW_STATE_DIR` would not affect `resolveExecApprovalsPath()` or `resolveExecApprovalsSocketPath()` because both still expand `DEFAULT_FILE`/`DEFAULT_SOCKET` under `~/.openclaw`.

## Solution Assessment

Unclear as merge-ready. The resolver direction is narrow and matches the bug, but the best final fix needs maintainer/security agreement on legacy approval migration and socket directory safety.
