## Summary

Keep open. Current main still hardcodes exec approval JSON, socket, and default effective-policy host-source paths under `~/.openclaw`, so this PR addresses a real split-state bug already tracked by #29736 and carried forward from source PR #65736. The implementation direction is narrow, but the PR touches host exec approval state and socket semantics, and the existing security review raises migration and socket-root questions that need maintainer security/design review before merge.

## What This Changes

The PR updates exec approval file, socket, and default host-source path resolution, docs, changelog, and focused tests so `OPENCLAW_STATE_DIR` controls the approvals state root while the unset default remains `~/.openclaw`.

## Best Possible Solution

Land this PR or a maintainer-approved replacement that derives the approvals JSON, default socket, and effective-policy host-source path from the active state-dir contract, preserves the unset default, updates all affected user-facing approvals docs, and explicitly records the accepted migration and socket-safety semantics.

## Reproduction Assessment

Yes. Set `OPENCLAW_STATE_DIR` and inspect current main: the approvals file and socket resolvers still expand `~/.openclaw/exec-approvals.*`, and the persistence helpers use those resolvers for real reads and writes.

## Solution Assessment

Unclear as merge-ready. The resolver direction is the narrow fix for the reported split state, but the best final solution needs maintainer confirmation on legacy fallback, warning/migration behavior, and socket-directory trust for overridden state roots.
