## Summary

Keep open: current main still has the reported exec approvals state-dir bug, and this PR is the active focused implementation candidate, but it needs maintainer/security review before merge because it changes command-execution approval storage and the default approval socket root.

## What This Changes

The PR routes exec approval JSON, default socket, effective-policy host-source text, docs, tests, and changelog through `OPENCLAW_STATE_DIR` while keeping the unset default at `~/.openclaw`.

## Best Possible Solution

Land this PR or a maintainer-approved equivalent after aligning remaining diagnostics/docs and explicitly accepting the migration and socket-root safety semantics for exec approval state.

## Reproduction Assessment

Yes. Source inspection on current main shows `resolveStateDir()` honors `OPENCLAW_STATE_DIR`, while exec approval file/socket/default host-source helpers still use hardcoded `~/.openclaw/exec-approvals.*` defaults.

## Solution Assessment

Unclear as merge-ready. The resolver direction is the narrow fix for the reported bug, but final landing needs maintainer/security judgment on fresh state-root migration and overridden socket-root safety, plus cleanup of remaining user-facing path text.
