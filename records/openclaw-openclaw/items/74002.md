## Summary

Keep open: current main still ignores `OPENCLAW_STATE_DIR` for exec approval JSON, socket, and effective-policy source text. This PR is a plausible fix, but it needs maintainer/security review for state-root migration and socket-root semantics, plus a rebase and small docs/diagnostic cleanup before merge.

## What This Changes

The PR routes exec approvals file, socket, effective-policy host-source text, docs, tests, and changelog through `OPENCLAW_STATE_DIR` while preserving the unset `~/.openclaw` fallback.

## Best Possible Solution

Land this PR or a maintainer-approved replacement after rebasing, aligning remaining diagnostics/docs, and explicitly accepting the migration and socket-root safety semantics.

## Reproduction Assessment

Yes, source-reproducible. On current main, `resolveStateDir()` honors `OPENCLAW_STATE_DIR`, while `resolveExecApprovalsPath()` and `resolveExecApprovalsSocketPath()` still expand hardcoded `~/.openclaw/exec-approvals.*` defaults.

## Solution Assessment

Unclear as merge-ready. The resolver direction is the narrow fix for the bug, but landing needs maintainer/security judgment on migration and socket-root behavior plus cleanup of remaining path text.
