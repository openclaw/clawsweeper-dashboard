## Best Possible Solution

Split and rebase the branch so only the config-validation fix lands on the current plugin metadata snapshot/catalog boundary, while preserving inline TTS directive behavior and dropping the already-covered Slack change.

## Solution Assessment

No. Centralizing known plugin IDs is reasonable, but this branch should be split and rebased onto the current metadata boundary instead of carrying a stale Slack hunk and an unrelated TTS contract change.
