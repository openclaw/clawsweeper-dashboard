## Summary

Keep open: the config-validation problem is still plausible and not implemented on current main, but this branch is not mergeable as-is because it changes TTS directive behavior, imports stale metadata, and lacks real behavior proof.

## What This Changes

The PR adds a config-validation helper for known plugin IDs from registry and bundled metadata, plus Slack socket-mode `ignoreSelf` assertions/config and TTS directive parser/test changes.

## Best Possible Solution

Ship a focused config-validation fix using the current plugin metadata/snapshot seam, preserve inline TTS directive behavior, and leave Slack `ignoreSelf` to the implementation already on main.

## Reproduction Assessment

Yes, by source inspection. Current main still builds known config plugin IDs only from the manifest registry, while the blocking PR regression is visible because current main accepts inline TTS directives and PR head only matches standalone-line directives.

## Solution Assessment

No. Centralizing known plugin ID resolution is reasonable, but this PR is not the best vehicle until it is rebased or split, preserves inline TTS behavior, and uses the current metadata boundary.
