## Best Possible Solution

Ship a focused config-validation patch on current main that centralizes known plugin-id resolution through current manifest/metadata contracts, preserves inline TTS parsing, and leaves the already-shipped Slack behavior alone.

Yes. Source inspection gives a high-confidence reproduction for the PR regression: current main parses `parseTtsDirectives("Hello [[tts:provider=edge]] world", policyWithAllowProvider)`, while this PR's newline-bounded directive regex would ignore it.
