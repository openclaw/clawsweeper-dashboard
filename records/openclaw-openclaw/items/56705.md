## Summary

Keep this PR open, but it should not merge as-is: the config-validation direction is useful, while the branch still carries an unrelated inline TTS directive regression and stale bundled-metadata imports relative to current main.

## What This Changes

The PR centralizes config validation known-plugin-ID collection, adds plugin-validation tests, adds Slack Bolt `ignoreSelf: false` coverage, and changes TTS directive parsing/tests around standalone directive lines.

## Best Possible Solution

Ship a focused config-validation patch on current main that centralizes known plugin-id resolution through current manifest/metadata contracts, preserves inline TTS parsing, and leaves the already-shipped Slack behavior alone.

## Reproduction Assessment

Yes. A source-level reproduction is `parseTtsDirectives("Hello [[tts:provider=edge]] world", policyWithAllowProvider)`: current main parses the provider override, while this PR's newline-bounded directive regex would ignore it.

## Solution Assessment

No. Centralizing known plugin-id validation is a reasonable narrow fix, but this branch is not the best vehicle because it mixes unrelated behavior changes and is stale against the current bundled metadata boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve inline TTS directives:** `src/tts/directives.ts:75`
  - body: The new `directiveRegex` only matches `[[tts:...]]` markers on standalone lines. Current main, tests, and docs support inline directives such as `Hello [[tts:provider=edge]] world`; with this patch those overrides are left in the reply and ignored, so tagged TTS can silently use the wrong provider or voice. Keep inline parsing unless this ships as an explicit, documented migration.
  - confidence: 0.9

  - body: The added config helper imports `../plugins/bundled-capability-metadata.js` and `BUNDLED_PROVIDER_PLUGIN_ID_ALIASES`, but current main has no top-level module or provider-alias export there. Rebase this onto the current manifest/metadata snapshot boundary before merging, otherwise the focused config fix cannot build cleanly on main.
  - confidence: 0.82
