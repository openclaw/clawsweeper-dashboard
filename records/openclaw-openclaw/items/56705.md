## Summary

Keep open: the central config-validation fix is not already on current main, but this branch is not mergeable as-is because it still changes the TTS directive contract and imports a stale plugin metadata boundary; the Slack part is already covered on main.

## What This Changes

The PR adds known-plugin-ID resolution for config validation, Slack socket-mode `ignoreSelf` handling/tests, and TTS directive parser/test changes.

## Best Possible Solution

Land a focused current-main config-validation fix through the current plugin metadata boundary, preserve inline TTS behavior unless maintainers deliberately migrate that contract, and leave the already-implemented Slack socket handling on main.

## Solution Assessment

No. Centralizing known plugin IDs is reasonable, but this branch should be split and rebased onto current metadata contracts instead of carrying unrelated TTS behavior changes and stale Slack edits.
