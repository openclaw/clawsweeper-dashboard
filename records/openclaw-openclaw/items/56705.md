## Summary

Keep this PR open, but it is not merge-ready. The config-validation refactor may still be useful, but the branch mixes in unrelated Slack and TTS changes, and the TTS parser change would silently stop parsing inline directives that current main, tests, and docs still support. Current main also already contains the Slack `ignoreSelf: false` behavior in a newer helper, so that part of the branch is redundant.

## What This Changes

The PR branch centralizes config plugin ID validation, adds config validation tests, sets Slack Bolt `ignoreSelf: false` coverage, and tightens TTS directive parsing/tests to standalone directive lines except compact text blocks.

## Best Possible Solution

Split or rebase the useful config-validation work into a narrow current-main patch that uses the current manifest/control-plane contracts, preserves inline TTS directive parsing, and omits the already-shipped Slack option change unless maintainers want separate coverage.

## Reproduction Assessment

Yes. A source-level reproduction is `Hello [[tts:provider=edge]] world` with provider overrides enabled: current main parses and strips that inline directive, while the PR's newline-bounded directive regex would leave it in the reply text and ignore the override.

## Solution Assessment

No. Centralizing known-plugin-ID resolution may be a reasonable direction, but this mixed branch is not the best fix because it changes unrelated TTS behavior and carries a Slack hunk already implemented on main.
