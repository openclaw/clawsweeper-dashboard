## Summary

Keep open: the useful config-validation cleanup is not fully present on current main, but this branch is not correct as-is because it silently drops inline TTS directives that current main, tests, and docs still support. The Slack socket-mode hunk is already covered on main through the shared Bolt app factory, so the safest path is a narrow config-only rebase or replacement.

## What This Changes

The PR centralizes config validation known-plugin-ID collection, adds plugin-validation tests, adds Slack Bolt `ignoreSelf: false` test coverage, and changes TTS directive parsing/tests to require standalone directive lines except compact text blocks.

## Best Possible Solution

Ship a focused config-validation patch on current main that resolves known plugin ids through the current manifest and plugin-id normalization contracts, keeps inline TTS directive parsing unchanged, and omits the already-shipped Slack behavior unless maintainers want separate test-only coverage.

## Reproduction Assessment

Yes. A source-level reproduction is `parseTtsDirectives("Hello [[tts:provider=edge]] world", policyWithAllowProvider)`: current main parses the provider override, while the PR's newline-bounded directive regex would leave it in text and ignore it.

## Solution Assessment

No. Centralizing known plugin-id validation is a reasonable direction, but this mixed branch is not the best fix because it bundles unrelated Slack and TTS behavior changes and needs to be rebased onto the current plugin metadata boundary.
