## Summary

Keep open: the config-validation idea is useful, but this PR is not mergeable as-is because it lacks real behavior proof, changes TTS directive semantics outside the stated scope, and imports a stale plugin metadata surface. The Slack `ignoreSelf` part is already covered on current main, while the related TTS-only PR at https://github.com/openclaw/openclaw/pull/56706 has a maintainer note asking for an API/docs decision before merging standalone-only semantics.

## What This Changes

The PR adds a known-plugin-ID helper for config validation, Slack socket-mode `ignoreSelf` assertions/config, and TTS directive parser/test changes.

## Best Possible Solution

Land a focused current-main config-validation fix using plugin metadata/snapshot contracts, preserve inline TTS behavior unless the API/docs/changelog deliberately migrate it, and rely on the Slack `ignoreSelf` implementation already on main.

## Reproduction Assessment

Yes, by source inspection. Current main still builds known config plugin IDs only from registry records, and the PR-introduced TTS regression is clear because current tests/docs accept inline directives while the PR regex only matches standalone lines.

## Solution Assessment

No. Centralizing known plugin-ID resolution is reasonable, but this branch should be split and rebased so it uses the current metadata boundary and does not change TTS directive semantics without a maintainer API decision.
