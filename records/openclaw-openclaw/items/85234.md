## Summary

## What This Changes

The issue reports that `openclaw update` fails during the staged npm global install swap when npm/fnm creates hardlinked dependency binaries such as `esbuild`.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/85559 as the canonical tracker and land a scoped package-manager hardlink fix through https://github.com/openclaw/openclaw/pull/85569 after the security-boundary review is satisfied.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: put any regular file with `nlink > 1` under the staged npm package root and the global install swap passes that tree to `movePathWithCopyFallback` with `sourceHardlinks: "reject"`, which throws the reported error.

## Solution Assessment

Yes as triage: closing this duplicate in favor of the broader canonical issue and active PR is the cleanest maintainer path. The code fix should remain scoped to package-manager-owned swap paths rather than weakening hardlink rejection globally.
