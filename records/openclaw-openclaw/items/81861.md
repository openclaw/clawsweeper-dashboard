## Summary

Keep open. Current main still drops the live per-agent `authProfileStore` before the `image_generate` runtime call, so the OpenAI Codex image route can fall back to a different auth-store load and surface the reported generic missing-key error; an open closing PR already targets the narrow fix.

## What This Changes

The issue asks OpenClaw to refresh expired `openai-codex` OAuth tokens for `image_generate` from the current agent auth store, or show a more specific expired-token error.

## Best Possible Solution

Land a focused fix that forwards the existing `authProfileStore` to media generation runtimes and adds regression coverage, keeping cross-agent token sync UX as a separate follow-up only if the narrow fix is insufficient.

## Reproduction Assessment

Yes at source level: current main receives the live auth profile store during tool construction but drops it before `generateImage`, while the provider and runtime contract already depend on `authStore`. I did not run a live expired-token OAuth request.

## Solution Assessment

Yes. Forwarding the existing live store through the existing runtime `authStore` contract is the narrowest maintainable fix; broader OAuth refresh or cross-agent sync changes are not needed for the central bug.
