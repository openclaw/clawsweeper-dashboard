The issue requests fixing embedded-agent web_search provider resolution so agent runs observe the same loaded Brave provider as CLI/plugin inspection over long gateway uptime.

## Reproduction Assessment

Yes at source level, but not as a full live uptime repro: current main can pass preferRuntimeProviders false for a manifest-owned configured search provider before runWebSearch reaches the runtime registry. The 3-6 hour Brave gateway failure still needs live validation.

## Solution Assessment

Likely yes, but the fix is still proposed rather than merged: keeping agent web_search on runtime provider discovery while leaving web_fetch bundled-owner gating intact is the narrow owner-boundary repair. The best path is to review and land or supersede https://github.com/openclaw/openclaw/pull/88684.
