## What This Changes

The issue reports agent-side web_search losing the configured Brave runtime provider while CLI inference and plugin inspection still see Brave loaded.

## Best Possible Solution

Review, prove, and land or supersede https://github.com/openclaw/openclaw/pull/88684, then close this issue from the merged fix.

## Reproduction Assessment

Yes, at source level: current main can set preferRuntimeProviders to false for a manifest-owned configured search provider and then forward that into runWebSearch. I did not live-reproduce the 3-6 hour gateway uptime failure in this read-only review.

## Solution Assessment

Likely yes, but the fix is still open. Keeping agent web_search runtime-first while leaving web_fetch’s bundled-owner gate intact is narrower than duplicating provider ownership policy at the agent boundary.
