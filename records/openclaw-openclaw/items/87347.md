## Summary

Keep open: current main and the latest release still contain the agent-side web_search provider-preference gate that can bypass live runtime provider discovery, and https://github.com/openclaw/openclaw/pull/88684 is still the open closing fix candidate.

## What This Changes

Agent-side web_search should keep resolving the configured Brave/runtime web-search provider in long-lived gateway and embedded-agent runs instead of diverging from CLI and plugin inspection.

## Best Possible Solution

Review, prove, and land or supersede https://github.com/openclaw/openclaw/pull/88684, then close this issue from the merged fix.

## Reproduction Assessment

Yes, at source level: current main can set preferRuntimeProviders to false for a manifest-owned configured search provider and forward that into runWebSearch. I did not live-reproduce the 3-6 hour gateway uptime failure in this read-only review.

## Solution Assessment

Yes for the linked narrow direction, but it is still open. Keeping agent web_search runtime-first while leaving web_fetch's bundled-owner gate intact is the maintainable fix shape shown by the current source evidence.
