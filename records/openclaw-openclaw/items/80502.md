## Summary

Keep open: current main intentionally keeps successful auto fallback selections durable across later turns, while this asks for a new opt-in per-turn fallback persistence policy and config/API decision.

## What This Changes

Request an optional agent model fallback policy that uses a fallback for one turn without leaving that fallback as the session model override.

## Best Possible Solution

Add a documented, schema-validated fallback persistence policy that keeps session durability as the default and implements per-turn rollback only for an explicit opt-in after maintainers choose the config contract.

## Reproduction Assessment

Not applicable; this is a feature/config request, not a broken existing behavior report. Current source, tests, docs, and the latest release schema show durable auto fallback persistence is the defined behavior today.

## Solution Assessment

Unclear until maintainers choose the public policy surface. The requested direction is plausible, but the maintainable solution is an explicit config contract plus focused rollback semantics, not an unconditional runner rollback.
