## Summary

Keep open: current main still intentionally persists auto-selected fallback overrides across turns, while this asks for a new compatible per-turn/session policy and config contract that needs maintainer product/API approval. The related open PR https://github.com/openclaw/openclaw/pull/79678 explores expiry of automatic fallback pins, but it does not replace the requested explicit `turn`/`session` policy decision.

## What This Changes

Request to add a configurable agent model fallback persistence policy that can use a fallback for one turn without persisting it as the active session override.

## Best Possible Solution

Define a documented, schema-validated fallback persistence policy, keep durable `session` behavior as the compatibility default, and implement `turn` rollback with focused runner, model-selection, schema, and docs coverage if maintainers approve.

## Reproduction Assessment

Not applicable; this is a feature/config request rather than a broken existing behavior report. Source, docs, and tests show current main deliberately keeps auto fallback overrides selected across later turns.

## Solution Assessment

Unclear until maintainers choose the config contract. An explicit agent model policy is safer than an unconditional rollback, and the related TTL proposal should be weighed against a deterministic `turn`/`session` mode.
