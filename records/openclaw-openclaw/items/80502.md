## Summary

Keep open: current main and the latest release still intentionally keep auto-selected fallback model state across later turns, while this request asks for a new compatible per-turn persistence policy.

## What This Changes

Add a configurable model-fallback mode that uses a fallback for one turn without persisting it as the active session model override.

## Best Possible Solution

Define a documented, schema-validated fallback persistence policy, keep durable `session` behavior as the default, and implement `turn` rollback only after maintainers approve the config scope and session-safety rules.

## Reproduction Assessment

Not applicable; this is a feature/config request rather than a broken existing behavior report. Current source, docs, tests, and the latest release contract show durable auto fallback state is intentional today.

## Solution Assessment

Unclear until maintainers choose the model-state contract. The requested direction is plausible, but the safer solution is an explicit fallback persistence policy rather than an unconditional runner rollback.
