## Summary

Keep open: current main intentionally keeps auto-selected fallback models across later turns, while this issue asks for a new compatible fallback-persistence policy. The request is not implemented, is not a strict bug against the current contract, and is distinct from broader recovery-policy threads because it specifically proposes a per-turn rollback mode.

## What This Changes

The issue requests an optional fallback persistence mode that uses a fallback model for one turn without leaving that fallback as the session override.

## Best Possible Solution

Add a documented, schema-validated fallback persistence policy, preserve durable `session` behavior as the default, and implement `turn` rollback only when that explicit policy is configured.

## Reproduction Assessment

Not applicable; this is a feature/config request rather than a broken current contract. Source, docs, and tests give high confidence that current main intentionally persists auto fallback overrides across turns.

## Solution Assessment

Unclear until maintainers choose the public config contract. The requested direction is plausible, but the maintainable path is an explicit policy with compatibility default rather than an unconditional runner rollback.
