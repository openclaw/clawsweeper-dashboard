## Summary

Keep open: current main intentionally keeps auto-selected fallback models across later turns, while this issue asks for a new compatible mode that rolls that state back after one successful turn.

## What This Changes

The issue requests a configurable `turn` fallback persistence policy so successful model fallbacks do not remain as the active session model override.

## Best Possible Solution

Add a documented, schema-validated fallback persistence policy on agent model config, preserve `session` as the default, and implement `turn` rollback with focused runner and model-selection regression coverage if maintainers approve.

## Reproduction Assessment

Not applicable; this is a feature/config request rather than a broken existing behavior report. Source, docs, tests, and changelog evidence confirm current main intentionally persists auto fallback overrides across turns.

## Solution Assessment

Unclear until maintainers choose the config contract. The requested direction is plausible, but it should be designed as an explicit agent model policy instead of an unconditional rollback in the runner.
