## Summary

Keep open: current main intentionally keeps auto-selected fallback models across turns, and this issue asks for a new compatible policy/config mode rather than a fix to broken existing behavior.

## What This Changes

Request to add a configurable per-turn model fallback mode that restores prior session model state after a successful fallback turn.

## Best Possible Solution

Add a documented, schema-validated fallback persistence policy, keep durable `session` behavior as the compatibility default, and implement `turn` rollback only after the public config contract is approved.

## Reproduction Assessment

Not applicable; this is a feature/config request rather than a broken existing behavior report. Current source, tests, and docs show the durable fallback behavior and the absence of a `fallbackPersistence` option.

## Solution Assessment

Unclear until maintainers choose the config contract. The requested policy knob is a plausible direction and is safer than changing global fallback persistence unconditionally.
