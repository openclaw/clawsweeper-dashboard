## Summary

Keep open. Current main still implements sticky auto fallback plus periodic primary probing, there is no fallbackPersistence/persist_auto_fallback config surface, and the related open PRs cover either a global default change or a narrower polluted-origin bug rather than the default-preserving product policy requested here.

## What This Changes

The issue requests an optional per-turn model fallback policy so a successful fallback can answer one turn without persisting that fallback as the session model override.

## Best Possible Solution

Keep this as the canonical product tracker, preserve sticky fallback as the default unless maintainers approve a default change, and implement any accepted per-turn policy through aligned schema/help/docs plus reply, follow-up, and direct-agent coverage.
