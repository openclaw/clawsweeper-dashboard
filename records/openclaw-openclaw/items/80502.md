## Summary

Keep open: current main added a throttled recovery probe for auto-pinned fallback sessions, but it still preserves successful fallback overrides across later turns and has no configurable per-turn fallback persistence policy.

## What This Changes

Request adds a configurable agent model fallback persistence policy that can scope a successful fallback selection to one turn while preserving current session persistence by default.

## Best Possible Solution

Define a documented and schema-validated fallback persistence policy, keep durable session persistence as the compatibility default, and add focused runner/model-selection/config/docs coverage for an approved `turn` mode.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source, docs, and tests show current main intentionally uses durable auto fallback state with throttled primary probing rather than per-turn rollback.

## Solution Assessment

Unclear until maintainers choose the public config contract. The requested direction is plausible, but the maintainable path is an explicit policy layered onto the current auto-fallback recovery behavior rather than an unconditional runner rollback.
