## Summary

Keep open: current main still intentionally persists auto-selected fallback models across later turns, while this request asks for a new compatible per-turn policy. This is a feature/config/API decision rather than conservative cleanup.

## What This Changes

Add an optional fallback persistence policy so successful model fallback can be scoped to one turn without leaving the fallback as the session override.

## Best Possible Solution

Define a documented, schema-validated fallback persistence policy, keep durable `session` behavior as the compatibility default, and add focused runner/config/docs coverage for an approved `turn` mode.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source, tests, and docs do give high confidence that current main intentionally persists auto fallback overrides across turns.

## Solution Assessment

Unclear until maintainers choose the config contract. The requested direction is plausible, but it should be implemented as an explicit policy rather than an unconditional rollback in the runner.
