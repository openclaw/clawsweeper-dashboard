## Summary

Keep open. The PR targets two real current-main session repair gaps, but the proposed trailing-assistant trim is too broad because it would also delete trailing assistant tool-call turns that existing replay repair intentionally preserves for tool-result pairing recovery. The protected `maintainer` label and overlapping open implementations mean this needs maintainer consolidation before merge.

## What This Changes

The PR updates session-file repair to rewrite blank-only user messages to `(continue)`, trim trailing assistant entries, report trim counts, add regression tests, and add a changelog entry.

## Best Possible Solution

Land one canonical session-file repair that preserves blank user turns with a neutral non-empty placeholder, trims only trailing non-tool-call assistant replay tails, keeps interrupted assistant tool-call turns available for existing pairing repair, adds regression coverage for that exclusion, and closes the two linked bug reports after merge.

## Reproduction Assessment

Yes. Current main has a source-level reproduction for both underlying bugs, and the PR regression can be reproduced with a session JSONL whose final assistant message contains a `toolCall`, `toolUse`, or `functionCall` content block: the new loop would pop it from disk.

## Solution Assessment

No as written. The repair seam and blank-user rewrite are appropriate, but the trailing trim should mirror the existing outbound guard by skipping assistant entries that contain replay tool calls.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Skip tool-call assistant entries while trimming:** `src/agents/session-file-repair.ts:283-284`
  - body: The new trim loop drops every trailing assistant entry. Current outbound replay intentionally stops before assistant messages that contain replay tool calls, and transcript repair can synthesize missing tool results; deleting those tails on disk would permanently lose interrupted tool-call turns. Please skip assistant entries with tool-call content here and add coverage for that case.
