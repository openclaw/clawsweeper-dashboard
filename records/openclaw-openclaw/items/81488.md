## Summary

Keep this PR open for maintainer review: the patch appears to address a real node-host approval precheck mismatch and I found no blocking correctness finding, but it intentionally tightens compatibility for existing bare-command allowlists.

## What This Changes

The PR changes node-host exec approval analysis to use the forwarded node env with PATH fallback cleared, preserves node allow-always reuse with command markers, and adds regression tests plus a changelog entry.

## Best Possible Solution

Keep the narrow forwarded-env precheck hardening and node allow-always marker preservation, then land only after maintainers accept the compatibility change and normal exact-head checks are green.

## Reproduction Assessment

Yes, from source inspection. Current main prechecks node approval with params.request.env while system.run forwards params.target.env, and bare command resolution falls back to process.env.PATH when PATH is absent.

## Solution Assessment

Yes. Building precheck input from the forwarded node env with PATH/Path cleared is the narrow fix, and the added node command marker path preserves allow-always reuse without reintroducing gateway PATH trust.
