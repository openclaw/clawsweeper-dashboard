## What This Changes

The PR changes node-host exec approval prechecks to analyze commands with the forwarded node env while blanking PATH/Path fallback, and adds regression coverage for gateway PATH not satisfying node allowlists.

## Best Possible Solution

Keep the narrow env-precheck hardening direction, then require real paired-node proof plus green targeted and required checks before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: host=node passes gateway env as request.env while forwarding target.env to system.run, and bare executable resolution can use gateway PATH. I did not run the paired-node scenario in this read-only review.

## Solution Assessment

Yes, the proposed direction is the narrowest maintainable fix: build precheck input from the forwarded node env and blank PATH/Path to avoid gateway fallback. It still needs real behavior proof and green checks before merge.
