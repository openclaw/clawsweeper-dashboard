## What This Changes

The PR changes node-host exec approval analysis to use the forwarded node env with PATH/Path cleared, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Merge the narrow precheck-env hardening after maintainer landing/check handling, keeping the forwarded system.run payload and approval binding unchanged.

## Reproduction Assessment

Yes, by source inspection. Current main analyzes node approval with params.request.env while system.run forwards params.target.env, and bare executable resolution can fall back to process PATH when analysis env lacks PATH/Path.

## Solution Assessment

Yes. Building approval analysis from params.target.env and explicitly blanking PATH/Path is the narrow maintainable fix for the mismatch without changing node execution or approval binding.
