## Summary

Keep open. Current main still has the reported active-requester dead end, and the issue is already paired with open PR #75669, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that Telegram users do not receive push-delivered subagent completion announcements when the parent session is idle between turns.

## Best Possible Solution

Land #75669 or an equivalent narrow patch that preserves successful steering/direct fallback behavior while allowing fallback miss or recoverable fallback failure to continue into the existing gateway agent delivery path, with regression tests for both fallback-false and fallback-throw cases.

## Reproduction Assessment

Yes. A focused unit reproduction can set expectsCompletionMessage true, an active requester session, queueEmbeddedPiMessage returning false, and missing or failing completion fallback data, then assert current main returns undelivered before callGateway is invoked.

## Solution Assessment

Yes. The proposed direction is the narrowest maintainable repair: keep successful steer/fallback returns, but fall through to the existing gateway delivery path when those do not deliver.
