## Summary

Keep open: current main still has the active-requester fallback dead end, and open PR #75669 is the linked implementation candidate with closing syntax.

## What This Changes

The issue requests reliable Telegram delivery for subagent completion announcements when embedded-run steering and direct completion fallback cannot deliver while the parent session is active but idle.

## Best Possible Solution

Land PR #75669 or an equivalent narrow patch with regression coverage that lets recoverable active-requester fallback miss or throw cases continue into the existing gateway agent delivery path.

## Reproduction Assessment

Yes. A focused unit reproduction can set expectsCompletionMessage true, an active requester session id, queueEmbeddedPiMessage returning false, and fallback data that is missing or throws, then assert current main returns undelivered before callGateway is invoked.

## Solution Assessment

Yes. Falling through to the existing gateway agent delivery path is the narrowest maintainable fix because it preserves successful steering and direct fallback behavior without adding new delivery infrastructure.
