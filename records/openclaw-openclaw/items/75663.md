## Summary

Keep open. Current main still has the reported active-requester dead end: if embedded steering fails and the completion fallback throws or returns false, `sendSubagentAnnounceDirectly` returns undelivered before reaching the existing gateway `agent` delivery path. The issue is also paired with open PR #75669 using closing syntax, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that subagent completion announcements fall through to the existing gateway delivery path when Telegram steering and completion fallback do not deliver while the parent session is idle.

## Best Possible Solution

Land #75669 or an equivalent narrow patch that preserves successful steering and direct fallback behavior while allowing fallback miss or recoverable fallback failure to continue into the existing gateway `agent` delivery path, with regression coverage for both fallback-false and fallback-throw cases.

## Reproduction Assessment

Yes. A focused unit reproduction can set completion mode, an active requester session id, `queueEmbeddedPiMessage` returning false, and missing or failing fallback delivery data, then assert current main returns undelivered before `callGateway` is invoked.

## Solution Assessment

Yes. The proposed direction is the narrowest maintainable fix because it reuses the existing gateway delivery path and only removes the dead-end behavior after steering and fallback fail to deliver.
