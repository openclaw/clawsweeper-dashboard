## Summary

Keep open: current main still has the active-requester fallback dead end, and PR #75669 is the open, unmerged implementation candidate with closing syntax.

## What This Changes

The issue requests that Telegram subagent completion announcements fall through to gateway delivery when embedded-run steering and direct fallback do not deliver.

## Best Possible Solution

Land PR #75669 or an equivalent narrow patch that preserves successful steering and direct fallback while letting fallback miss or recoverable fallback failure continue into existing gateway delivery with regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction can set expectsCompletionMessage true, an active requester session id, queueEmbeddedPiMessage returning false, and missing or throwing fallback delivery; current main returns undelivered before callGateway is invoked.

## Solution Assessment

Yes. Letting the active-requester fallback miss fall through to the existing gateway agent delivery path is the narrowest maintainable fix because the delivery mechanism already exists and the expected push completion behavior is documented.
