## Summary

Keep open. Current main still contains the reported active-requester dead end, and the linked open PR is the right implementation candidate to resolve it before this issue is closed.

## What This Changes

The issue reports that Telegram subagent completion announcements can be queued but not pushed when the parent session is idle and the active-requester fallback path exits before gateway delivery.

## Best Possible Solution

Land #75669 or an equivalent narrow patch that preserves successful steering/direct-fallback behavior but falls through to the existing gateway `agent` delivery path when active-requester fallback misses or fails, with regression coverage for both fallback-false and fallback-throw cases.

## Reproduction Assessment

Yes. A focused unit reproduction can set `expectsCompletionMessage: true`, an active requester session id, `queueEmbeddedPiMessage` returning false, and missing or failing fallback data, then assert current main returns undelivered before `callGateway` is invoked.

## Solution Assessment

Yes. The narrow maintainable fix is to keep successful steer and fallback exits, but let recoverable active-requester fallback failure continue into the existing gateway delivery path instead of returning undelivered.
