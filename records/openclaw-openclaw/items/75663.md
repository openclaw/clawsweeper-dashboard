## Summary

Keep open: current main still has the active-requester fallback dead end, and PR #75669 remains the linked implementation candidate with closing syntax.

## What This Changes

The issue asks for subagent completion announcements to fall through to the existing gateway delivery path when steering and direct fallback fail for idle Telegram parent sessions.

## Best Possible Solution

Land PR #75669 or an equivalent narrow patch that preserves successful steering and direct fallback behavior while allowing recoverable fallback misses to continue into gateway agent delivery with regression coverage.

## Reproduction Assessment

Yes. A focused unit reproduction can set expectsCompletionMessage true, an active requester session id, queueEmbeddedPiMessage returning false, and missing or throwing completion fallback data, then assert current main returns undelivered before callGateway is invoked.

## Solution Assessment

Yes. Falling through to the existing gateway delivery path is the narrowest maintainable fix because the delivery infrastructure already exists and only the premature active-requester returns block it.
