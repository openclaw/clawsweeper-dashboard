## Summary

Keep open. Current main still contains the active-requester completion path that can return `delivered: false` before the existing gateway delivery call, and an open linked PR already targets this issue with closing syntax, so the issue should remain open until that PR is reviewed and landed or closed.

## What This Changes

The issue asks for subagent completion announcements on Telegram to fall through to gateway delivery when active-requester steering and direct fallback cannot deliver immediately.

## Best Possible Solution

Review and land PR #75669 or an equivalent narrow fix that lets active-requester fallback miss/failure continue into the existing gateway `agent` delivery path, with regression coverage for active non-streaming Telegram completion delivery.

## Reproduction Assessment

Yes. A focused unit reproduction can set completion delivery with an active requester session, `queueEmbeddedPiMessage` returning false, and an empty/unroutable or throwing fallback, then assert current main returns false before `callGateway` is invoked.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve the successful direct fallback return but let fallback miss or recoverable failure fall through to the existing gateway delivery path, with a regression test to prevent duplicate-send behavior.
