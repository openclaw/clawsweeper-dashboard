## Summary

Keep open: current main still has the active-requester fallback dead end, and the issue is already paired with open PR #75669 using closing syntax.

## What This Changes

The issue reports that Telegram does not receive completed subagent announcements when the requester session is active but not consuming messages.

## Best Possible Solution

Land PR #75669 or an equivalent narrow patch that preserves successful steering and direct fallback behavior while letting fallback miss or recoverable fallback failure continue into the existing gateway delivery path with regression coverage.

## Reproduction Assessment

Yes. A focused source-level reproduction can set `expectsCompletionMessage` true, an active requester session id, `queueEmbeddedPiMessage` returning false, and missing or throwing fallback delivery; current main returns undelivered before `callGateway` is invoked.

## Solution Assessment

Yes. Letting the failure path fall through to the existing gateway delivery call is the narrowest maintainable fix because the delivery mechanism already exists and no new channel, config, or product behavior is needed.
