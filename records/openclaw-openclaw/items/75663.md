## Summary

Keep open: current main still has the active-requester fallback dead end, and PR #75669 is the open, unmerged implementation candidate with closing syntax.

## What This Changes

The issue reports that Telegram subagent completion announcements can be dropped when active-requester steering fails and fallback delivery misses before gateway delivery is attempted.

## Best Possible Solution

Land PR #75669 or an equivalent narrow patch that preserves successful steering and direct fallback behavior while allowing fallback miss or recoverable fallback failure to continue into the existing gateway delivery path with regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `expectsCompletionMessage` true, an active requester session id, `queueEmbeddedPiMessage` returning false for a non-streaming embedded run, and missing or throwing fallback delivery returns undelivered before `callGateway` is invoked.

## Solution Assessment

Yes. Letting the failure path fall through to the existing gateway delivery call is the narrowest maintainable fix because the delivery mechanism already exists in the same function and no new channel, config, or product behavior is needed.
