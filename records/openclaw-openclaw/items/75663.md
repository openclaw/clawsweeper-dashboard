## Summary

Keep open: current main still has the active-requester fallback dead end, and PR #75669 is the open linked implementation candidate that should resolve this after review/merge.

## What This Changes

The report asks to prevent Telegram subagent completion announcements from being dropped when an active parent requester cannot be steered and fallback delivery misses.

## Best Possible Solution

Land PR #75669 or an equivalent narrow patch with regression coverage for fallback false and fallback throw, while preserving successful steering and direct fallback behavior.

## Reproduction Assessment

Yes. A focused unit reproduction can set `expectsCompletionMessage` true, provide an active requester session id, make `queueEmbeddedPiMessage` return false, and omit task-completion fallback text; current main returns undelivered before invoking `callGateway`.

## Solution Assessment

Yes. Letting fallback miss or recoverable fallback failure fall through to the existing gateway delivery path is the narrowest maintainable fix for the observed dead end.
