## Summary

Keep open. Current main still contains the active-requester completion path where a failed or unavailable completion fallback returns `{ delivered: false }` before the existing gateway `agent` delivery path can run, and open PR #75669 uses closing syntax for this issue, so the issue should remain open until that PR is resolved.

## What This Changes

Telegram subagent completion announcements should be delivered immediately when the parent session is active but not consuming embedded messages between turns.

## Best Possible Solution

Revise or land the linked PR so `sendSubagentAnnounceDirectly` only stops the active-requester path on confirmed steering or confirmed direct fallback success, and otherwise falls through to the existing gateway `agent` delivery path with regression tests for fallback miss and fallback throw.

## Reproduction Assessment

Yes. A focused unit reproduction can set `expectsCompletionMessage: true`, an active requester session, `queueEmbeddedPiMessage` returning false, and an empty/unroutable or throwing completion fallback, then assert current main returns false before `callGateway` is invoked.

## Solution Assessment

Yes. Falling through to the already-wired gateway `agent` delivery path after fallback miss is the narrowest maintainable fix, as long as successful direct fallback remains terminal and duplicate-send risk is covered by tests.
