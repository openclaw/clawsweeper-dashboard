## Summary

Keep open: current main still has the active-requester fallback dead end, and #75669 is the open unmerged implementation candidate with closing syntax.

## What This Changes

The issue asks for Telegram subagent completion announcements to be delivered promptly when the parent session is active but not consuming embedded-run messages.

## Best Possible Solution

Land #75669 or an equivalent narrow patch that preserves successful steering and direct fallback behavior while letting fallback miss or recoverable fallback failure continue into the existing gateway delivery path with regression coverage and a changelog entry.

## Reproduction Assessment

Yes, at source level. A focused unit reproduction can call completion delivery with expectsCompletionMessage true, an active requester session id, queueEmbeddedPiMessage returning false, and missing or throwing fallback delivery, then assert current main returns delivered:false before callGateway is invoked.

## Solution Assessment

Yes. Reusing the existing gateway agent delivery path after steering and direct fallback fail is the narrowest maintainable fix, while preserving early returns when steering or fallback succeeds.
