## Summary

Keep open. Current main still has the reported active-requester dead end: when embedded steering fails and the completion fallback either throws or returns false, `sendSubagentAnnounceDirectly` returns undelivered before reaching the existing gateway `agent` delivery path. The discussion already links open PR #75669 as the narrow fix candidate, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Fix subagent completion delivery so an active but non-consuming requester session can fall through to existing gateway `agent` delivery when steering and direct fallback do not deliver.

## Best Possible Solution

Land #75669 or an equivalent narrow patch that preserves successful steering/direct fallback behavior but lets fallback miss or recoverable fallback failure continue into the existing gateway `agent` delivery path, with regression tests for both fallback-false and fallback-throw cases.

## Reproduction Assessment

Yes. A focused unit reproduction can set `expectsCompletionMessage: true`, an active requester session id, `queueEmbeddedPiMessage` returning false, and missing or failing completion fallback data, then assert current main returns undelivered before `callGateway` is invoked.

## Solution Assessment

Yes. The proposed direction is the narrowest maintainable repair: keep successful steer and fallback returns, but fall through to the already-wired gateway delivery path when those do not deliver.
