## Summary

Keep open. Current main still has the reported active-requester dead-end, and an open PR already references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix is merged.

## What This Changes

The issue requests that Telegram-visible subagent completion announcements fall through to the existing gateway `agent` delivery path when steering and direct fallback fail for an active but non-consuming parent session.

## Best Possible Solution

Land a narrow tested fix, likely through open PR #75669 if review passes, so active requester fallback miss or recoverable fallback failure continues into the existing gateway `agent` delivery path while successful direct fallback still stops immediately.

## Reproduction Assessment

Yes. A focused unit reproduction can set completion mode, `requesterActivity.isActive: true`, a requester session id, `queueEmbeddedPiMessage` returning false, and an empty or throwing fallback, then assert current main returns false before `callGateway` is invoked.

## Solution Assessment

Yes. The proposed direction is the narrowest maintainable fix: only confirmed successful fallback should return from the active-requester block; fallback miss/failure should fall through to the already-wired gateway delivery path with regression coverage.
