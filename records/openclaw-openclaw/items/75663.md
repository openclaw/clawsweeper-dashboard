## Summary

Keep open. Current main still has the active-requester early-return path that can stop completion delivery before the existing gateway delivery call runs, and open PR #75669 already references this issue with closing syntax, so the issue should stay open until that fix or an equivalent one lands.

## What This Changes

Fix subagent completion delivery so Telegram users receive push completion announcements when the parent session is idle between turns and the active embedded run cannot be steered.

## Best Possible Solution

Land #75669 or an equivalent narrow fix that preserves successful steer and fallback delivery while allowing fallback miss or recoverable fallback failure to continue into the existing gateway `agent` delivery path, with regression tests for the active Telegram-style completion path.

## Reproduction Assessment

Yes. A focused unit reproduction can set `expectsCompletionMessage: true`, an active requester session id, `queueEmbeddedPiMessage` returning false, and an empty or throwing completion fallback, then assert current main returns false before the gateway `agent` call is invoked.

## Solution Assessment

Yes. The narrowest maintainable solution is to keep the successful fallback early return but let fallback miss or recoverable fallback failure fall through to the already-wired gateway delivery path, covered by targeted regression tests.
