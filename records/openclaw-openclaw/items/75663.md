## Summary

Keep open: current main still has the active-requester fallback dead end, and open PR #75669 is the linked implementation candidate with closing syntax.

## What This Changes

The issue requests that subagent completion announcements on Telegram fall through to gateway delivery when steering and fallback delivery cannot wake an active but idle parent session.

## Best Possible Solution

Land PR #75669 or an equivalent narrow patch that preserves successful steering and direct fallback behavior while letting fallback miss or recoverable fallback failure continue into the existing gateway delivery path with regression coverage.

## Reproduction Assessment

Yes. A focused unit reproduction can set `expectsCompletionMessage` true, an active requester session id, `queueEmbeddedPiMessage` returning false, and missing or throwing completion fallback data, then assert current main returns undelivered before `callGateway` is invoked.

## Solution Assessment

Yes. The proposed fallthrough is the narrowest maintainable fix because the gateway delivery path already exists and only the premature active-requester returns block it.
