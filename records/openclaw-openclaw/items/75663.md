## Summary

Keep open: current main still has the active-requester fallback dead end, and PR #75669 is the open, unmerged implementation candidate with closing syntax.

## What This Changes

Subagent completion announcements should reach Telegram even when the parent requester session is active but not consuming queued messages.

## Best Possible Solution

Land PR #75669 or an equivalent narrow patch that preserves successful steering and direct fallback behavior while allowing fallback miss or recoverable fallback failure to continue into existing gateway delivery with regression coverage.

## Reproduction Assessment

Yes for a source-level reproduction: set `expectsCompletionMessage` true, provide an active requester session id, make `queueEmbeddedPiMessage` return false, and make fallback delivery miss or throw. Current main returns undelivered before invoking `callGateway`.

## Solution Assessment

Yes. Falling through to the existing gateway `agent` delivery path on fallback miss or recoverable fallback failure is the narrowest maintainable fix because the delivery mechanism already exists.
