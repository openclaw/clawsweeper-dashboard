The issue reports WhatsApp Web pairing succeeds but the subsequent WebSocket startup fails with ETIMEDOUT and the channel exits.

## Best Possible Solution

Route retryable opening-phase WhatsApp timeout closes through the existing bounded reconnect policy without changing terminal auth, conflict, QR/login, or relink behavior.

## Solution Assessment
