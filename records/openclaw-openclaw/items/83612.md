## Summary

Keep this PR open. The latest head still has a blocking Codex session compatibility bug: SOUL workspaces can clear an existing `/codex resume` or pre-upgrade binding because the stored binding has no `personality` field while the new computed override is `"none"`.

## What This Changes

The PR adds Codex SOUL-aware native personality routing, context-engine prompt projection changes, session binding of personality state, Codex harness docs, and related tests.

## Best Possible Solution

Keep the SOUL-aware default only if maintainers accept the policy, but make legacy/manual bindings resume with the current personality override and persist the computed personality after resume.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: run `/codex resume <thread-id>` or use an existing pre-upgrade binding in a workspace where SOUL context makes the PR compute `personality: "none"`; the PR compares undefined stored personality with `"none"` and clears the binding before resume. I did not execute tests because this review is read-only.

## Solution Assessment

No, not yet. The SOUL routing approach may be acceptable, but the session-binding invalidation should treat missing personality as legacy/manual state that can be resumed and upgraded rather than as a reason to discard the requested thread.
