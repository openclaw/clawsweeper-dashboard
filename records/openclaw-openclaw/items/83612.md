## Summary

Keep open: the SOUL routing is directionally useful and has terminal proof, but the PR currently breaks manually attached Codex threads because existing `/codex resume` bindings have no stored personality and are discarded in SOUL workspaces.

## What This Changes

This PR adds Codex `personalityMode` config, SOUL-aware app-server personality routing, bridge/context-engine prompt projection, docs, focused Codex harness tests, and a narrow outbound test mock.

## Best Possible Solution

Keep the SOUL-aware default, but preserve existing and manually attached Codex thread bindings by applying the current personality override through `thread/resume` and persisting it with regression coverage.

## Reproduction Assessment

Yes, source-level: run `/codex resume` in a session whose workspace contains `SOUL.md`, then send a Codex turn; the PR's new mismatch check clears that binding before resume. I did not execute it because this was a read-only review, but both code paths are explicit.

## Solution Assessment

No, not as written. The SOUL routing direction is maintainable, but the binding compatibility check should treat missing personality as legacy/manual resume state and update it through `thread/resume` instead of starting over.
