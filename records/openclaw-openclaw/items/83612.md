## Summary

Keep this PR open: the selectable Codex prompt-surface direction is useful, but the current patch can disable Codex's native personality on lightweight cron turns without actually supplying SOUL.md, and the latest head still needs real behavior proof beyond tests.

## What This Changes

The PR adds Codex `workspacePromptSurface` and `personalityMode` config, changes app-server prompt routing and binding fingerprints, and updates docs/tests for SOUL, AGENTS, and lightweight project-doc behavior.

## Best Possible Solution

Keep the selectable prompt-surface design, fix lightweight cron so `personality: "none"` is only used when SOUL is actually supplied, and have maintainers explicitly accept and document the one-time binding rotation behavior.

## Reproduction Assessment

Yes for the review finding by source inspection: a lightweight cron run with SOUL.md can set `personality: "none"` while `shouldInjectCodexOpenClawPromptContext` keeps SOUL out of the prompt. I did not execute tests because this review is read-only.

## Solution Assessment

No, not yet. The selectable routing design is maintainable, but the personality override should be tied to the prompt surface that is actually delivered, and the session-rotation compatibility choice needs maintainer ownership.
