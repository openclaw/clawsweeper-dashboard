## Summary

Keep open. ## What This Changes

The issue reports that WhatsApp outbound sends work but inbound messages never reach the gateway after linking or restarting in containerized deployments.

## Best Possible Solution

Capture a redacted live WhatsApp diagnostic run in Docker or k3s, then fix the WhatsApp plugin so a linked account either restores eligible inbound delivery, backfills or warns on unrecoverable gaps, and reports stale ingress clearly.
