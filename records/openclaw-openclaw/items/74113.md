## Summary

Keep open. The PR addresses a real subagent announce race, but the proposed refresh can reintroduce unnormalized silent-token text and does not affect the `childCompletionFindings` path that still wins when final findings are built.

## What This Changes

The PR imports `readLatestAssistantReply` and adds a best-effort one-shot latest-assistant refresh in `runSubagentAnnounceFlow` before building the announce status.

## Best Possible Solution

Land one canonical subagent completion-output fix that uses a bounded, normalized post-wait refresh or removes successful raw-tool fallback, preserves `ANNOUNCE_SKIP` and `NO_REPLY` semantics, and adds focused regression coverage.

## Reproduction Assessment

Yes. The source-level path is clear on current main: a successful run with no visible assistant text can select `latestRawText`, and #74073 provides the Grok/DeepSeek plus large `web_search` scenario that triggers the race.

## Solution Assessment

No. The PR targets the right layer, but the proposed assignment skips the existing normalization helper and does not update the value that wins when `childCompletionFindings` is present.
