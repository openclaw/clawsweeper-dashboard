## Summary

Keep open. Current main still has the stale subagent output path this PR is trying to address, but the proposed patch is not safe to merge because the refreshed reply bypasses existing silent-token normalization and can be ignored when child completion findings are present. The open overlapping #74340 means maintainers should pick one canonical fix rather than landing both independently.

## What This Changes

The PR imports `readLatestAssistantReply` into `src/agents/subagent-announce.ts` and adds a best-effort one-shot latest assistant reply refresh before announce status construction.

## Best Possible Solution

Land one canonical subagent announce/output fix that performs a bounded, normalized post-wait refresh before constructing completion events, preserves `ANNOUNCE_SKIP` and `NO_REPLY` semantics, handles both child-findings and no-findings paths, and adds regression coverage for delayed post-compaction assistant output replacing stale tool JSON.

## Reproduction Assessment

Yes. The source-level path is clear: current main can select `latestRawText` before finalizing `findings`, and #74073 supplies the live Grok/DeepSeek plus large Tavily `web_search` scenario that triggers the race.

## Solution Assessment

No. The PR targets the right layer, but the one-shot raw latest-assistant read skips existing normalization and does not affect the `childCompletionFindings` value that feeds the final announce event.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Normalize refreshed assistant replies before assigning:** `src/agents/subagent-announce.ts:464-471`
  - body: At this point `reply` has already gone through `stripAndClassifyReply`, but `latestReply` is raw assistant history. If the refreshed reply contains `NO_REPLY` plus real text, this block writes the raw token back into the announcement because the guards only catch a pure silent token. Run refreshed text through the same normalization before assigning it.
  - confidence: 0.86
- **[P2] Refresh the value that feeds findings:** `src/agents/subagent-announce.ts:459-475`
  - body: The retry mutates `reply` even when `childCompletionFindings` is set, but the final `findings` value is still computed as `childCompletionFindings || reply || "(no output)"`. In that path the refreshed reply is discarded, so this does not actually unify the child-findings branch. Refresh the findings value too, or scope the retry to the no-findings path.
  - confidence: 0.78
