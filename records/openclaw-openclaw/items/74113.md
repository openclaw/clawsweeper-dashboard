## Summary

Keep open. The PR targets a real subagent announce race, but it still has blocking semantic issues: the refreshed assistant text bypasses the existing silent-token normalization, and changes to `reply` are ignored whenever `childCompletionFindings` supplies the final announce payload. Current `main` also still has the underlying raw-tool fallback path, with overlapping open fixes that need maintainer consolidation.

## What This Changes

The PR imports `readLatestAssistantReply` into `runSubagentAnnounceFlow` and adds a best-effort post-branch latest-assistant refresh before building the subagent announce status.

## Best Possible Solution

Land one canonical subagent announce/output fix that prevents stale raw tool output from becoming a successful completion result, preserves `ANNOUNCE_SKIP` and `NO_REPLY` semantics through the same normalization path, covers both direct announce and frozen-result capture paths, and adds focused regression tests for post-compaction delayed assistant output.

## Reproduction Assessment

Yes. The source-level path is high confidence: current `main` can still return `snapshot.latestRawText` as selected subagent output, and #74073 gives a live Grok/DeepSeek plus large Tavily `web_search` scenario that triggers the stale-tool-output race.

## Solution Assessment

No. The PR changes the right layer, but the proposed one-shot raw latest-assistant refresh skips existing normalization and does not update the value used when `childCompletionFindings` wins the final `findings` expression; the safer direction is a consolidated normalized output-selection fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Normalize refreshed assistant replies before assigning:** `src/agents/subagent-announce.ts:464-471`
  - body: `reply` has already gone through `stripAndClassifyReply`, but `latestReply` is raw assistant history. If the refreshed text contains `NO_REPLY` plus real content, this block writes the raw token back into the announcement because the guards only catch pure silent-token replies. Run the refreshed text through the same normalization before assigning it.
  - confidence: 0.86
- **[P2] Refresh the value that feeds findings:** `src/agents/subagent-announce.ts:459-475`
  - body: This retry mutates `reply`, but the final announce text is still computed as `childCompletionFindings || reply || "(no output)"`. When `childCompletionFindings` is set, the refreshed reply is discarded, so the new retry does not actually cover both branches. Refresh the findings value too, or scope the retry to the no-findings path.
  - confidence: 0.78
