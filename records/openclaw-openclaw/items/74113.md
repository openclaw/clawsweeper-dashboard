## Summary

Keep open. The underlying bug remains valid on current main, but this PR is not safe to merge as-is because its refreshed assistant reply path bypasses existing silent-token normalization and does not update the value used when child completion findings are present; it also overlaps with open alternative PR #74340.

## What This Changes

The PR imports `readLatestAssistantReply` into `src/agents/subagent-announce.ts` and adds a best-effort one-shot latest-assistant refresh before building the announce status.

## Best Possible Solution

Land one canonical subagent announce/output fix that performs a bounded, normalized post-wait refresh or removes successful raw-tool fallback, preserves `ANNOUNCE_SKIP` and `NO_REPLY` semantics, handles child-findings and no-findings paths, and adds focused regression coverage for delayed post-compaction assistant output.

## Reproduction Assessment

Yes. The source-level path is clear on current main: a successful run with no visible assistant text can select `latestRawText`, and #74073 provides the live Grok/DeepSeek plus large Tavily `web_search` scenario that triggers the race.

## Solution Assessment

No. The PR targets the right layer, but the proposed one-shot raw latest-assistant assignment skips the existing normalization helper and does not change the `childCompletionFindings` value that wins when final findings are built.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Normalize refreshed assistant replies before assigning:** `src/agents/subagent-announce.ts:464-471`
  - body: At this point `reply` has already gone through `stripAndClassifyReply`, but `latestReply` is raw assistant history. If the refreshed reply contains visible text plus a `NO_REPLY` fragment that the existing helper would strip, this block writes the raw token back into the announcement because the guards only catch a pure silent token. Run the refreshed text through the same normalization before assigning it.
  - confidence: 0.86
- **[P2] Refresh the value that feeds findings:** `src/agents/subagent-announce.ts:459-475`
  - body: The retry mutates `reply` even when `childCompletionFindings` is set, but final `findings` is still computed as `childCompletionFindings || reply || "(no output)"`. In that path the refreshed reply is discarded, so this does not actually unify the child-findings branch. Refresh the findings value too, or scope the retry to the no-findings path.
  - confidence: 0.78
