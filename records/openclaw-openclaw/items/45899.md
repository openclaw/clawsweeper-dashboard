## What This Changes

The PR adds Feishu blocked local `MEDIA:` path detection, a one-time final fallback notice, dispatcher tests, and MEDIA-path documentation updates.

## Best Possible Solution

Port the Feishu blocked-local-media hint and one-time final fallback onto current main while preserving streaming closeout, thread fallback, and current MEDIA policy docs, then require redacted Feishu runtime proof.

## Reproduction Assessment

Yes, at source level: blocked local paths still throw `LocalMediaAccessError` with `path-not-allowed`, and current Feishu `onError` only logs and closes streaming. I did not run a live Feishu account reproduction in this read-only review.

## Solution Assessment

No, not as-is. The Feishu dispatcher is the right boundary, but the branch must be ported onto current streaming closeout and top-level reply fallback behavior before it is the maintainable fix.
