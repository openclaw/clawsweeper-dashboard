## Summary

Keep open. Current main still lacks the Feishu blocked-local-media user fallback, so this PR is not obsolete, but the proposed patch is not ready to merge because the fallback guard is set after an awaited cleanup path that can race under the shared dispatcher.

## What This Changes

The PR adds Feishu blocked-local-media error detection, a one-time final-reply plain-text fallback, dispatcher tests, and outbound `MEDIA:` local-path documentation updates.

## Best Possible Solution

Rebase the fallback onto current main, set the one-time fallback guard before any awaited cleanup, preserve the current Feishu streaming error closeout semantics, add overlapping-final-failure coverage, align docs with the `tools.fs.workspaceOnly` trust model, and coordinate with #51375 without relaxing local media access policy.

## Reproduction Assessment

Yes. A focused code-level reproduction is a Feishu final reply containing a local `MEDIA:` path outside allowed roots: `assertLocalMediaAllowed` throws `LocalMediaAccessError` with `path-not-allowed`, and current main's Feishu `onError` only logs and closes streaming.

## Solution Assessment

No, not as written. The direction is narrow and maintainable, but the guard must be set before the first await because the shared dispatcher does not await `onError`; the rebase also needs to preserve current streaming error closeout and docs wording.
