## Summary

Keep open. Current main already has stronger MEDIA local-path documentation, but the Feishu runtime fallback is not implemented, and this PR is not mergeable as-is because it drops current dispatcher error-closeout behavior, misses the quoted-thread top-level fallback path, and lacks real Feishu proof.

## What This Changes

The PR adds Feishu-specific blocked local MEDIA-path hinting, a one-time final fallback message, dispatcher tests, and MEDIA local-path docs.

## Best Possible Solution

Port the Feishu blocked-local-media hint and one-time final fallback onto current main while preserving streaming error closeout, quoted-thread top-level fallback, and the newer MEDIA policy docs.

## Reproduction Assessment

Yes, at source level: blocked local MEDIA paths still throw LocalMediaAccessError with path-not-allowed, and current Feishu onError only logs and closes streaming. I did not run a live Feishu account reproduction in this read-only review.

## Solution Assessment

No, not as-is. The Feishu dispatcher is the right boundary, but the branch must be ported onto current streaming closeout and top-level reply fallback behavior before it is the maintainable fix.

Overall correctness: patch is incorrect
