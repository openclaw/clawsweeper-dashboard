## What This Changes

The PR adds Feishu blocked-local-media detection, a one-time final fallback message, dispatcher tests, and MEDIA path documentation.

## Best Possible Solution

## Reproduction Assessment

Yes, at source level: blocked local media paths still throw LocalMediaAccessError/path-not-allowed, and current Feishu onError only logs and closes streaming. I did not run a live Feishu account reproduction in this read-only review.

## Solution Assessment

No, not as-is. The Feishu dispatcher is the right boundary, but the branch must be ported to current main's streaming closeout and top-level fallback contracts before it is the maintainable fix.
