## What This Changes

The PR adds Feishu detection for blocked local `MEDIA:` paths, sends a one-time final fallback notice, adds dispatcher tests, and updates MEDIA path docs.

## Best Possible Solution

## Reproduction Assessment

Yes, source-level: blocked local paths throw `LocalMediaAccessError` with `path-not-allowed`, and current Feishu media delivery propagates those failures into `onError`, which only logs and closes streaming. I did not run a live Feishu account reproduction in this read-only review.

## Solution Assessment

No, not as-is. The Feishu dispatcher is the right boundary, but the branch must be ported onto current streaming closeout and thread-fallback behavior before it is the maintainable fix.
