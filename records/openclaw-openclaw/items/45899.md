## Summary

Keep open. Current main still lacks the Feishu-specific blocked-local-media fallback, so the PR is not obsolete; however, the branch should not merge as written because its one-time fallback guard is set after awaited cleanup and can race on overlapping final failures.

## What This Changes

The PR adds Feishu detection and logging for disallowed local `MEDIA:` path errors, sends a one-time plain-text fallback for final reply failures, adds dispatcher tests, and updates setup/FAQ docs.

## Best Possible Solution

Rebase the runtime fallback onto current main, set the fallback guard before any awaited cleanup, preserve current streaming error closeout semantics, add coverage for overlapping final failures, and keep the docs aligned with the current `tools.fs.workspaceOnly` media trust model.

## Reproduction Assessment

Yes. A focused reproduction is a Feishu final reply containing a local `MEDIA:` path outside allowed roots; `loadWebMedia`/`assertLocalMediaAllowed` throws `LocalMediaAccessError` with `path-not-allowed`, and current main’s Feishu `onError` only logs and closes out.

## Solution Assessment

No, not as written. The direction is narrow and maintainable, but the fallback guard must be set before the first await because the shared dispatcher does not await `onError`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

- **[P2] Set the fallback guard before cleanup:** `extensions/feishu/src/reply-dispatcher.ts:402-403`

Set the guard before the first await and cover overlapping final failures.
