## Best Possible Solution

Rebase onto current main, keep the Feishu-specific fallback, narrow detection to true allowed-root denials, reconcile docs with the current `tools.fs.workspaceOnly` policy text, and add a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main's Feishu `onError` only logs and closes, while `LocalMediaAccessError` allowed-root denials are thrown by the shared media guard and the PR's unit test directly exercises that callback.

## Solution Assessment

No, not as written. The Feishu-only fallback is the right narrow direction, but the predicate must distinguish allowed-root denials from other `path-not-allowed` media safety failures before merge.
