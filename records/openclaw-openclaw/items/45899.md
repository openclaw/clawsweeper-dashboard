## Summary

Keep open. Current main still only logs and closes the Feishu reply path for blocked local media, while this PR is useful but needs a small repair before merge: guard ordering, current-main closeout semantics, docs alignment, and changelog coverage.

## What This Changes

The PR adds Feishu detection for disallowed local MEDIA paths, a one-time final-reply plain-text fallback, focused dispatcher tests, and MEDIA local-path docs.

## Best Possible Solution

Land a rebased Feishu-owned fallback that keeps local media policy unchanged, sends at most one final user hint per reply cycle, and documents the current workspaceOnly trust model.

## Reproduction Assessment

Yes. A final Feishu reply containing a local MEDIA path outside allowed roots reaches LocalMediaAccessError/path-not-allowed, and current main's Feishu onError path only logs and closes streaming instead of notifying the user.

## Solution Assessment

No, not as written. The direction is narrow and maintainable, but the patch should set the guard before the first await, preserve current-main closeStreaming({ markClosedForReply: false }) semantics, align docs, and add the required changelog entry.
