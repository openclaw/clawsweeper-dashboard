## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep the new abort guard out of existing defaults:** `src/agents/text-repetition-guard.ts:28`
  - body: `enabled: true` makes the new stream abort behavior active for every existing agent with no config change. Because legitimate structured output can repeat stems or lines, this should be opt-in or explicitly maintainer-approved before it changes upgrade behavior for all users.
  - confidence: 0.9
- **[P1] Clear buffered replies before aborting repetition:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:301`
  - body: The guard fires after the chunk has already been appended to the block buffer, then calls `session.abort()`. If the run ends before `message_end`, the lifecycle fallback flush can still send that buffered repetitive text, so the buffer should be cleared or the flush path should honor the abort state before aborting.
  - confidence: 0.86
- **[P2] Wire the new config key into the strict schemas:** `src/config/types.tools.ts:331`
  - body: Adding `textRepetitionGuard` to the TypeScript config types is not enough because the runtime zod schemas are strict and do not accept this key. Users who configure the advertised guard will hit validation failures instead of getting the configured thresholds.
  - confidence: 0.88
- **[P2] Reset guard state when a new message starts:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:187-188`
  - body: The new cursor and abort latch are not reset at `message_start`, and the abort latch is not reset in the shown finalizer either. If abort or error skips `message_end`, the next assistant message can inherit stale guard state and skip checks or drop valid output.
  - confidence: 0.82
- **[P2] Size the suffix tail from the configured cycle range:** `src/agents/text-repetition-guard.ts:229-230`
  - body: The detector hardcodes the suffix scan window to 600 characters, which silently prevents larger configured `maxCyclePatternLength` values from being checked. Either bound the configurable max to that window or size the tail from `maxCyclePatternLength * minCycleRepeats`.
  - confidence: 0.8
