## Summary

Keep open: current main still lacks a semantic assistant-stream repetition guard, and this PR is the active implementation candidate, but the patch is not safe to merge because source inspection shows abort cleanup, state reset, schema coverage, and real behavior proof gaps.

## What This Changes

The PR adds a text-repetition guard module, assistant streaming abort integration, config types/resolution, state fields, and focused unit tests for repeated generated text.

## Best Possible Solution

Land a repaired or replacement stream guard that is schema-backed, documented, conservative by default, clears buffered output on abort, resets state at message boundaries, and leaves confirmation/context-sanitization semantics to explicit maintainer-approved follow-up.

## Reproduction Assessment

Yes, for the PR defects from source inspection: the abort path leaves buffered repeated text available for lifecycle flushing, the new state is not reset at the assistant-message boundary, and the advertised config key is absent from strict runtime schemas. The original user-facing loop still needs contributor-provided real-run proof in this PR.

## Solution Assessment

No, not as currently patched. The assistant-stream boundary is plausible, but the implementation needs abort cleanup, message-boundary state reset, schema/docs coverage, conservative default policy, and real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Clear buffered replies when the guard aborts:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:300-302`
  - body: When this branch aborts and returns, the repetitive chunk has already been appended to `blockBuffer` or `blockChunker`. If the run ends before `message_end`, `handleAgentEnd` still flushes those buffers, so users can receive the output the guard meant to suppress; clear/reset the buffers here or make the lifecycle flush honor the abort flag.
  - confidence: 0.88
- **[P2] Reset repetition guard state at message start:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:187-191`
  - body: `handleMessageStart` resets the normal assistant buffers but not the new checked length or abort flag, and `message_end` is not guaranteed after abort/error paths. A stale cursor can delay later checks, and a sticky abort flag can suppress valid future replies, so reset both fields at the assistant-message boundary.
  - confidence: 0.87
- **[P2] Accept textRepetitionGuard in runtime schemas:** `src/config/types.tools.ts:335`
  - body: The PR exposes `tools.textRepetitionGuard` and agent-level overrides in TypeScript only, while the strict Zod schemas still accept only `loopDetection`. Users who configure the advertised key will get validation errors instead of applying the guard settings; add schema, help/docs, and generated-baseline coverage together.
  - confidence: 0.9
- **[P2] Size suffix scan from the configured cycle range:** `src/agents/text-repetition-guard.ts:229-230`
  - body: The suffix detector always slices the last 600 characters, which caps the effective pattern length to `600 / minCycleRepeats` even when `maxCyclePatternLength` is configured higher. Configs that intentionally allow longer repeated blocks can miss the loops they asked the guard to detect; derive the tail size from `maxCyclePatternLength` and `minCycleRepeats`.
  - confidence: 0.78
