## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Suppress every buffered flush after repetition abort:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:298`
  - body: The guard sets `abortedByTextRepetitionGuard` and aborts the session, but abort/error paths can skip `message_end`; current `handleAgentEnd` still calls `flushBlockReplyBuffer()`, which drains `blockChunker` or emits `state.blockBuffer`. That means the repeated text the guard detected can still be delivered during terminal cleanup, so the abort path needs to clear/suppress the shared buffer before calling abort or teach the central flush path to honor the guard.
  - confidence: 0.9
- **[P1] Do not turn on heuristic stream aborts by default:** `src/agents/text-repetition-guard.ts:27`
  - body: This new guard defaults to `enabled: true`, so existing users can have legitimate repetitive structured output cut off without changing config. The closest existing loop detector is opt-in by default, so this should either default off with explicit config or land with an explicit maintainer-approved policy, docs, and upgrade proof for default-on abort behavior.
  - confidence: 0.86
- **[P2] Register textRepetitionGuard in strict config schemas:** `src/config/types.tools.ts:331`
  - body: The PR adds `tools.textRepetitionGuard` to TypeScript config types, but current runtime schemas are strict and only accept `loopDetection` under both global and agent tools config. Users who add the documented key through normal config validation will get an unknown-key failure instead of the override being applied.
  - confidence: 0.92
- **[P2] Reset guard state at message start:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:184-188`
  - body: The new cursor and abort flag are reset only from `handleMessageEnd`, but the code already treats message_start as the safer assistant-message boundary because end events can be skipped or arrive late. If the guard aborts before message_end, the next assistant message can inherit a stale cursor or abort flag and skip checks or drop valid buffered output.
  - confidence: 0.84
