## Summary

Keep open. Current main does not implement this guard, and the PR still has blocking correctness/config gaps: abort can still flush already-buffered repetitive text through the agent-end path, the new config key is not accepted by the strict runtime schema, and the user-facing feature is missing a changelog entry.

## What This Changes

The PR adds a stream-time text repetition detector, config types, per-message config resolution, assistant text update abort integration, and detector unit tests.

## Best Possible Solution

Land a guarded implementation after suppressing buffered output across both message_end and agent_end abort paths, registering the config schema/docs/generated surfaces, adding integration coverage, and adding the required changelog entry.

## Reproduction Assessment

Yes for the blocking review findings by source inspection: the PR appends text before aborting, while current handleAgentEnd still flushes block buffers, and the strict config schemas do not include the new key. I did not establish a live LLM-loop reproduction in this read-only review.

## Solution Assessment

No. The feature direction is useful, but this patch is not yet the best mergeable solution because it does not suppress all buffered-output leak paths and exposes config types without the runtime schema needed to tune or disable the guard.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Clear buffered replies on repetition abort:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:300-301`
  - body: When the guard fires, this branch sets `abortedByTextRepetitionGuard` and returns after `session.abort()`, but the current chunk has already been appended to `blockBuffer`/`blockChunker`. If abort terminates the run before `message_end`, `handleAgentEnd` still calls `flushBlockReplyBuffer()`, so the repetitive output can be emitted through the lifecycle fallback. Clear/suppress the block buffer when the guard fires, or make the agent-end flush honor the abort flag.
  - confidence: 0.88
- **[P2] Register textRepetitionGuard in the runtime schema:** `src/config/types.tools.ts:631`
  - body: The PR exposes `tools.textRepetitionGuard` in TypeScript types and the resolver reads it, but `AgentToolsSchema` and `ToolsSchema` are strict and still do not include this key. Normal config validation will reject user attempts to disable or tune the new default-on guard, so the Zod schemas and schema coverage need to be updated with the new fields.
  - confidence: 0.91
- **[P3] Add the changelog entry for the new guard:** `src/agents/text-repetition-guard.ts:27-28`
  - body: This is a user-facing `feat` that changes streaming behavior and adds configuration, but the PR file list does not include `CHANGELOG.md`. OpenClaw policy requires a changelog entry for user-facing feature changes before merge.
  - confidence: 0.8
