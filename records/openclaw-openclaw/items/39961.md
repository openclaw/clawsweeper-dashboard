## Summary

This PR should stay open, but it is not merge-ready: current main still lacks a semantic streaming text guard, and the proposed patch leaves concrete buffered-flush, state-reset, and strict config-schema gaps around the new abort path.

## What This Changes

The PR adds a core streaming text-repetition detector, config types and resolver state, embedded Pi message-handler abort wiring, and unit tests for repeated assistant text output.

## Best Possible Solution

Land a maintained core circuit breaker only after schema-backed config, docs/changelog, abort-time buffer suppression, per-message state reset, configured suffix bounds, focused lifecycle tests, and maintainer-approved default behavior are in place.

## Reproduction Assessment

Yes. Source inspection on current main shows no semantic text repetition guard, and the PR defects are source-reproducible from its abort branch interacting with the existing message_end and agent_end flush paths.

## Solution Assessment

No, not yet. A stream-level guard is a plausible direction, but this patch is not the narrowest maintainable solution until it fixes buffered emission, runtime schemas/docs, lifecycle reset, configured threshold behavior, and the default-on policy question.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Suppress abort buffers immediately:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:300-301`
  - body: When repetition is detected, the chunk has already been appended to deltaBuffer and block reply buffering before this branch sets the abort flag. If the run terminates before message_end, handleAgentEnd can still flush the buffered repetitive output, so clear or suppress those buffers when the abort state is set.
  - confidence: 0.91
- **[P2] Reset guard state at message start:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:186-188`
  - body: The new cursor and abort flag are not reset at the same start-of-message boundary as the rest of assistant state. An abort/error path that skips message_end can leave a sticky abort flag or stale cursor and cause the next assistant response to be dropped or under-checked.
  - confidence: 0.9
- **[P2] Register the config in runtime schemas:** `src/config/types.tools.ts:331-335`
  - body: This adds tools.textRepetitionGuard to the TypeScript config surface, but the strict AgentToolsSchema and ToolsSchema still only accept loopDetection. Normal config validation will reject the new key before the resolver can read it.
  - confidence: 0.94
- **[P2] Size suffix scanning from configured limits:** `src/agents/text-repetition-guard.ts:229-230`
  - body: The suffix detector still uses a fixed tail size even though maxCyclePatternLength is configurable. Operators who raise the configured maximum can create advertised loop ranges that the suffix detector never examines.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/agents/text-repetition-guard.ts:27-28`
  - body: This is a user-facing feature that changes streaming behavior and exposes a new config key, but the branch does not update CHANGELOG.md. Repo policy requires changelog coverage before merge.
  - confidence: 0.87
