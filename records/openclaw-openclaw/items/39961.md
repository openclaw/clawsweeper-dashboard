## What This Changes

The PR adds a configurable text-repetition guard module, wires it into assistant streaming to abort repetitive output, and adds unit tests plus config types.

## Best Possible Solution

Land a maintained core streaming circuit breaker only after schema/docs, abort-time buffer suppression, configured threshold semantics, lifecycle tests, and default behavior are approved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Clear buffered output when repetition aborts:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:300-301`
  - body: When detection fires, the triggering chunk has already been appended to `deltaBuffer` and the block-reply buffer. If the abort skips `message_end`, terminal cleanup can still flush that buffered repetitive text, so clear or suppress block buffers as soon as the abort flag is set.
  - confidence: 0.89
- **[P2] Reset repetition abort state at message start:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:186-188`
  - body: `abortedByTextRepetitionGuard` is initialized once and cleared only from `handleMessageEnd`. If an abort ends the stream before message-end cleanup, the sticky flag can make the next assistant message drop valid block output, so reset it at the same message-start boundary as the other assistant state.
  - confidence: 0.9
- **[P2] Register textRepetitionGuard in runtime schemas:** `src/config/types.tools.ts:335`
  - body: This exposes `tools.textRepetitionGuard` in exported TypeScript config types, but the strict runtime tool schemas still accept only `loopDetection` in this area. Normal config validation will reject the new key before the resolver can read it, so add the matching zod schema, help, labels, generated config docs, and tests.
  - confidence: 0.94
- **[P2] Size suffix scanning from configured cycle limits:** `src/agents/text-repetition-guard.ts:229-230`
  - body: The suffix-cycle detector hardcodes the tail window to 600 characters even though `maxCyclePatternLength` is configurable. If an operator raises the configured max pattern length and relaxes other detectors, suffix loops in the advertised range are never checked; size the tail from `maxCyclePatternLength * minCycleRepeats` or cap the config accordingly.
  - confidence: 0.82
