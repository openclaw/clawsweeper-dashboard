Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Clear buffered replies before aborting repetition:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:300-301`
  - body: The guard runs after the chunk has already been appended to blockBuffer/blockChunker, but `session.abort()` can end the run before `message_end`. In that path `flushBlockReplyBuffer` still emits the buffered repetitive text, so clear/suppress the block reply buffer as soon as the guard fires.
  - confidence: 0.91
- **[P2] Latch off later text handling after repetition abort:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:292-302`
  - body: After this branch returns, later queued assistant events can still be processed with `abortedByTextRepetitionGuard` set. Add an early guard for subsequent text/text_end/message_end handling so the same aborted turn cannot emit or mutate visible output after the repetition abort.
  - confidence: 0.86
- **[P2] Reset repetition guard state at message start:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:187-191`
  - body: The new checked-length cursor and abort flag are reset only on message_end cleanup. If abort/error skips message_end, the next assistant message can inherit stale state, skipping checks or dropping valid buffered output, so reset these fields at the message_start boundary.
  - confidence: 0.89
- **[P2] Register the new config in strict schemas:** `src/config/types.tools.ts:335`
  - body: The PR exposes `tools.textRepetitionGuard` in TypeScript config types, but AgentToolsSchema and ToolsSchema do not accept that key. Normal strict config validation will reject the advertised operator control before the resolver can read it.
  - confidence: 0.93
- **[P3] Size suffix scanning from configured cycle limits:** `src/agents/text-repetition-guard.ts:229-230`
  - body: The suffix detector always inspects only the last 600 characters, so configurations that raise `maxCyclePatternLength` above the default can never scan the full advertised range. Derive the tail size from `maxCyclePatternLength * minCycleRepeats` or clamp the public config accordingly.
  - confidence: 0.79
