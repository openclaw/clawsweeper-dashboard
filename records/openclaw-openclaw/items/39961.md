Full review comments:

- **[P1] Suppress buffered flush when repetition abort skips message_end:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:301`
  - body: session.abort() can terminate before message_end, but this patch only checks abortedByTextRepetitionGuard in handleMessageEnd. Current handleAgentEnd still calls ctx.flushBlockReplyBuffer(), which drains state.blockBuffer/blockChunker and can send the repetitive text the guard was meant to stop.
  - confidence: 0.91
- **[P1] Keep the new abort guard out of existing defaults:** `src/agents/text-repetition-guard.ts:28`
  - body: The new default is enabled: true, so upgrading would activate heuristic stream aborts for every agent. Existing users can produce legitimate repetitive structured output, so this needs maintainer-approved default-on policy or an opt-in default with tests/docs for both modes.
  - confidence: 0.87
- **[P2] Wire the new config key into strict schemas:** `src/config/types.tools.ts:331`
  - body: The PR adds tools.textRepetitionGuard only to TypeScript config types, but current ToolsSchema and AgentToolsSchema are strict and only include loopDetection; users who add the advertised key will hit validation errors instead of configuring the guard.
  - confidence: 0.9
- **[P2] Size suffix tail from configured cycle limits:** `src/agents/text-repetition-guard.ts:230`
  - body: The suffix-cycle detector hardcodes a 600-character tail, so raising maxCyclePatternLength above 600 / minCycleRepeats is accepted but never honored. That makes the new config surface misleading and can miss longer configured repetition cycles.
  - confidence: 0.82
