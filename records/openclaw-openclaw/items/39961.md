## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Suppress buffered flush after repetition abort:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:297-298`
  - body: When this branch sets `abortedByTextRepetitionGuard` and calls `session.abort()`, the run can terminate before any `message_end` event. In that path current `handleAgentEnd` still calls `flushBlockReplyBuffer()`, which drains the chunker or `state.blockBuffer`, so the repetitive text the guard meant to suppress can still be emitted. Clear or latch-suppress the buffer as soon as the guard fires, and cover the agent_end fallback path.
  - confidence: 0.9
- **[P1] Do not enable heuristic stream aborts by default:** `src/agents/text-repetition-guard.ts:28`
  - body: `enabled: true` makes the new detector active for every upgraded user with no config change. Since this is a heuristic that aborts valid output when it thinks a stream is repetitive, existing agents that intentionally generate repetitive tables, logs, or structured lists can be cut off unless maintainers explicitly approve that compatibility break or the guard starts opt-in.
  - confidence: 0.84
- **[P2] Register the new config in strict schemas:** `src/config/types.tools.ts:331`
  - body: The PR adds `tools.textRepetitionGuard` and per-agent `textRepetitionGuard` to the TypeScript config types, but current `AgentToolsSchema` and `ToolsSchema` are strict and do not include that key. Users following the new type surface will get validation failures instead of tunable thresholds, so the schema/help/docs/tests need to be added with the type change.
  - confidence: 0.93
- **[P2] Reset guard state at message start:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:184-188`
  - body: The new cursor and abort flag are only initialized once and the cursor is reset in message_end cleanup, but abort/error paths can skip message_end. A later assistant message can inherit a stale cursor or abort flag, disabling checks or causing valid buffered output to be dropped; reset both fields at the reliable message_start boundary with the existing assistant-message state reset.
  - confidence: 0.86
- **[P2] Size suffix-cycle tail from the configured range:** `src/agents/text-repetition-guard.ts:230`
  - body: The suffix-cycle detector hardcodes `tail` to 600 characters even though `maxCyclePatternLength` is configurable. Raising `maxCyclePatternLength` above the effective `600 / minCycleRepeats` limit silently fails to check the advertised range, so long repeating blocks can be missed under valid custom settings.
  - confidence: 0.78
