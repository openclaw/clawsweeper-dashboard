## Summary

Keep open. Current main still lacks a text-output repetition guard, so this PR is not obsolete, but the branch has blocking source-level defects, no real behavior proof, and default-on stream abort policy needs maintainer judgment before merge.

## What This Changes

The PR adds a streaming text-repetition detector, global/agent config types and resolver, assistant stream abort wiring, state fields, and unit tests.

## Best Possible Solution

Land a repaired or replacement schema-backed stream guard that suppresses buffered repeated output before any flush, resets state at message boundaries, documents conservative config/defaults, and leaves confirmation/context sanitization to explicit follow-up design.

## Reproduction Assessment

Yes for the PR defects via source inspection: the abort path runs after buffering, lifecycle flushing ignores the new abort flag, message_start does not reset the new guard state, and strict runtime schemas omit the advertised key. The original live model-loop behavior still lacks after-fix real-run proof from this contributor setup.

## Solution Assessment

No, not as currently patched. The assistant stream hook is plausible, but the branch needs abort cleanup across lifecycle flushes, start-boundary state reset, schema/docs coverage, configured suffix sizing, maintainer-approved default policy, and real behavior proof.
