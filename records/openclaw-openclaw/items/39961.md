  - confidence: 0.93

  - body: Setting `abortedByTextRepetitionGuard` only returns from the current update; later queued `text_end` or `message_end` events can still process and emit the full repeated text when no assistant update was marked emitted.
  - confidence: 0.88

  - body: The cursor and abort flag are not reset on the only reliable new-message boundary. Abort/error paths can skip `finalizeMessageEnd`, leaving stale guard state that either disables checks or drops a valid follow-up message.
  - confidence: 0.9

  - body: The PR exposes `tools.textRepetitionGuard` in TypeScript types and reads it in the resolver, but `AgentToolsSchema` and `ToolsSchema` are strict and do not accept the key. Normal config loading will reject the advertised setting.
  - confidence: 0.95
