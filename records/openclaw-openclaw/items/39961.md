## Summary

Keep open. Current main still lacks a semantic repeated-text stream guard, and this PR remains a plausible implementation candidate, but the current patch has correctness blockers, missing config-schema support, missing real behavior proof, and a default-on abort policy that needs maintainer acceptance.

## What This Changes

The PR adds a TypeScript text-repetition guard, config types/resolver, assistant stream abort wiring, state fields, and unit tests for repeated streaming text.

## Best Possible Solution

Land a repaired or replacement stream guard that is schema-backed, documented, conservative by default, clears or suppresses buffered output before any lifecycle flush, resets state at message boundaries, and leaves confirmation/context sanitization to explicit follow-up design.

## Reproduction Assessment

Yes, for the PR defects from source inspection: the abort branch leaves buffered text available to lifecycle flushing, state is not reset at the message boundary, and the advertised config key is absent from strict schemas. The original model-loop behavior still lacks contributor-provided after-fix real-run proof in this PR.

## Solution Assessment

No, not as currently patched. The assistant stream boundary is plausible, but this needs abort cleanup, state reset, schema/docs coverage, a maintainer-approved default policy, and real behavior proof before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Clear buffered replies when the guard aborts:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:300-302`
  - body: When this branch aborts, the repetitive chunk has already been appended to blockBuffer or blockChunker. If the run ends before message_end, the normal lifecycle flush can still emit that buffered text, so clear/reset those buffers or make every flush path honor the abort flag before returning.
  - confidence: 0.88
- **[P2] Reset repetition guard state at message start:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:187-191`
  - body: handleMessageStart resets the normal assistant buffers but not the new checked length or abort flag, and abort/error paths may skip message_end. A stale cursor can delay later checks and a sticky abort flag can suppress valid future replies, so reset both fields at the assistant-message boundary.
  - confidence: 0.87
- **[P2] Accept textRepetitionGuard in runtime schemas:** `src/config/types.tools.ts:335`
  - body: The PR exposes tools.textRepetitionGuard and agent-level overrides in TypeScript only, while the strict zod schemas still accept only loopDetection. Users who configure the advertised key will get validation errors instead of applying guard settings.
  - confidence: 0.9
- **[P2] Size suffix scans from the configured cycle range:** `src/agents/text-repetition-guard.ts:229-230`
  - body: The suffix detector always slices the last 600 characters, which caps the effective pattern length to 600 / minCycleRepeats even when maxCyclePatternLength is configured higher. Configs that intentionally allow longer repeated blocks can miss the loops they asked the guard to detect.
  - confidence: 0.78
