## Summary

Keep open. Current main still provides opt-in SecretRefs and a narrower OAuth-profile encrypted payload path, but it does not implement native encryption at rest for general OpenClaw config/auth credentials, and the related `.env` encryption request was closed back to this issue as the canonical tracker.

## What This Changes

Request native encryption-at-rest for sensitive OpenClaw config and auth credentials using keychain or master-password support while preserving compatibility.

## Best Possible Solution

Make a security-owned design decision, then implement a documented key-management, migration, recovery, rotation, redaction, and runtime plaintext-exposure model or explicitly define SecretRefs/external vaults as the supported boundary.

## Reproduction Assessment

Not applicable; this is a security feature/design request rather than a runtime bug. Static source and docs checks provide high-confidence evidence that general encrypted-at-rest config/static credential storage is still absent.

## Solution Assessment

Unclear. Built-in encryption may be the right end state, but the maintainable fix requires key-management, migration, recovery, rotation, and plaintext runtime-boundary decisions before code should land.
