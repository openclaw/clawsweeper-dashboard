## What This Changes

Request encrypted storage or keychain-backed migration for sensitive OpenClaw credentials that are currently allowed in plaintext config and auth-profile files.

## Best Possible Solution

Keep this as the canonical security-owned tracker and settle a single credential-at-rest contract covering config, auth profiles, SecretRefs, keychain/master-password backends, migration, recovery, rotation, backups, and plaintext runtime boundaries.

## Reproduction Assessment

Not applicable as a failing-runtime reproduction: this is a security feature/design request. Source and docs give a high-confidence negative check that plaintext credentials remain supported and no default native keychain/master-password encryption contract exists on current main.

## Solution Assessment

Unclear for implementation: the goal is valid, but the best solution needs explicit security/product decisions on key management, migration, recovery, rotation, backups, compatibility, and where plaintext may exist after decryption.
