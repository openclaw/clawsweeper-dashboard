Keep open. Current main has opt-in SecretRefs and cleanup tooling, but the requested native encrypted-at-rest credential storage remains unresolved security/product design work with plaintext credentials still permitted.

## What This Changes

## Best Possible Solution

Make a security/product decision, then either implement a documented key-management, migration, recovery, rotation, and runtime-exposure model, or explicitly document SecretRefs/external vaults/OS-level protection as the supported boundary.

## Solution Assessment

Unclear for implementation. Native encryption may be the right end state, but the maintainable path needs security-owned decisions around key management, migration, recovery, startup, rotation, and plaintext runtime exposure before code lands.
