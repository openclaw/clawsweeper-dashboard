## Summary

Keep open: current main and v2026.6.6 still document plaintext credential-at-rest as a high residual risk, current SecretRefs are opt-in and incomplete for mutable/OAuth credentials, and the linked keychain resolver pull request is open rather than merged.

## What This Changes

Request to encrypt sensitive OpenClaw credentials at rest, preferably through system keychains or a master-password model with backward-compatible migration.

## Best Possible Solution

Keep this as the canonical credential-at-rest tracker until maintainers choose the key-management model and define tested migration/doctor behavior for config, auth profiles, OAuth/runtime-mutated credentials, and plaintext residue cleanup.

## Reproduction Assessment

## Solution Assessment

No current solution is complete: opt-in SecretRefs and the open keychain resolver PR are useful partial paths, but the best fix requires a maintainer-owned key-management and migration design.
