Land a maintainer/security-approved credential-at-rest design that covers config and auth-profile storage, key management, migration, recovery, and rotation while preserving the existing SecretRef workflow as a supported mitigation.

## Reproduction Assessment

Yes, for the missing capability by source inspection. Current source and docs show plaintext credential storage remains supported, SecretRefs are opt-in, and auth-profile payloads are stored as JSON text rather than encrypted payloads.

Unclear. The requested direction addresses a real security gap, but maintainers still need to choose keychain versus passphrase behavior, migration commands, recovery semantics, and rotation guarantees.
