## Summary

Keep open: current main has opt-in SecretRefs and plaintext scrubbing, but it does not implement built-in encrypted-at-rest credential storage with keychain/master-password handling; the related #65239 was closed as a duplicate in favor of this canonical tracker.

## What This Changes

Request built-in encrypted-at-rest storage for sensitive OpenClaw credentials, with keychain or master-password decryption, backward compatibility, and rotation.

## Best Possible Solution

Keep this as the canonical secops-owned design tracker, define the key-management and plaintext-exposure contract, then implement a staged migration that preserves existing SecretRef and plaintext compatibility.

## Reproduction Assessment

Not applicable as a security feature request rather than a runtime bug. Source and docs provide a high-confidence negative check that current main has opt-in env/file/exec SecretRefs and 0600 JSON persistence, not built-in encrypted-at-rest credential storage.

## Solution Assessment

Unclear for implementation: the direction is valid, but the exact keychain/master-password, migration, recovery, and rotation design needs secops maintainer decisions before a narrow code change would be safe.
