Keep open: current main has useful opt-in SecretRef/audit machinery, but it still documents plaintext credential storage as valid and leaves full at-rest encryption, key management, OAuth/runtime-mutated credentials, and existing-install migration unresolved. ## What This Changes

The issue requests encryption or keychain-backed protection for sensitive OpenClaw credentials stored at rest in config and auth files.

Keep this as the canonical credential-at-rest tracker until maintainers choose the key-management model and define tested migration/doctor behavior for config, auth profiles, and OAuth/runtime-mutated credentials.

## Solution Assessment

No current solution is complete: opt-in SecretRefs and the open keychain resolver PR are useful partial paths, but the best fix requires a maintainer-owned key-management and migration design.
