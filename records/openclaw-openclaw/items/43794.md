## Summary

Request encrypted-at-rest storage for sensitive OpenClaw credentials, with system keychain or master-password decryption and backward compatibility.

## Best Possible Solution

Keep this as the canonical security-owned tracker for one credential-at-rest contract covering config, auth profiles, SecretRefs, keychain/master-password backends, migration, recovery, rotation, backups, and plaintext runtime boundaries.

Unclear for implementation: the goal is valid, but the best solution needs explicit security/product decisions on key management, migration, recovery, rotation, backups, compatibility, and where plaintext may exist after decryption.
