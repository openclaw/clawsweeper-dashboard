## Summary

Keep open: current main still treats SecretRefs as opt-in hardening and leaves several credential-at-rest surfaces, including auth profile payloads and OAuth/runtime-mutated credentials, outside transparent encryption or keychain-backed migration. The open linked keychain resolver PR is useful related work, but it is not merged and does not yet cover the full existing-config/auth-store migration requested here.

## What This Changes

Request cross-platform encryption or keychain-backed storage for OpenClaw credentials at rest, including migration away from plaintext config/auth files.

## Best Possible Solution

Keep this as the canonical credential-at-rest tracker until maintainers choose the key-management model and define tested migration/doctor behavior for config, auth profiles, and OAuth/runtime-mutated credentials.

## Reproduction Assessment

Yes at source level: current docs and code explicitly allow plaintext credentials, persist auth profile payloads as JSON, and exclude OAuth/runtime-mutated credentials from SecretRef migration.

## Solution Assessment

No current solution is complete: opt-in SecretRefs and the open keychain resolver PR are useful partial paths, but the best fix requires a maintainer-owned key-management and migration design.
