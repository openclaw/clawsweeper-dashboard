## Label Justifications

- `P1`: This PR targets a reported session-lock regression that can break agent/channel replies, but the patch currently has a merge-blocking trust-boundary issue.
- `merge-risk: 🚨 session-state`: The patch changes when released embedded attempts accept session-file changes made while another controller held the lock.
- `merge-risk: 🚨 security-boundary`: The patch can publish a final fingerprint as OpenClaw-owned even when an unowned edit was interleaved during the locked callback.

## Summary

Keep open: the reported false takeover is a real session-lock problem and the PR has useful current-head proof, but the branch still broadens the trusted session-file boundary in a way that can fold an interleaved external edit into an owned fingerprint.

## What This Changes

The PR adds process-wide session-file fingerprint ownership tracking so prompt-released embedded attempts can accept later same-process locked writes while still intending to reject external edits.

## Best Possible Solution

Keep the narrower owned-transcript behavior unless maintainers explicitly approve broader same-process write trust, and require a regression that covers an external edit interleaved during a locked callback before merging this approach.

## Reproduction Assessment

Yes at source level: current main compares the saved prompt-release fingerprint to the later filesystem fingerprint and throws on mismatch, while the PR body supplies a direct real-filesystem proof for the same-process locked-write case.

## Solution Assessment

No. The PR fixes a plausible false-positive path, but broad process-wide fingerprint trust is not the best solution until it preserves fail-closed behavior for interleaved external edits or maintainers choose that weaker boundary intentionally.

## Review Findings

Full review comments:

- **[P1] Keep interleaved external edits out of owned fingerprints:** `src/agents/pi-embedded-runner/run/attempt.session-lock.ts:384-387`
  - body: `publishOwnedSessionFileWriteIfChanged()` only checks that the pre-write fingerprint was trusted before recording the final fingerprint as owned. If an unowned process appends after `beforeWrite` is sampled but before this same-process locked callback finishes, the final fingerprint can include that external edit and a previously released controller will accept it as OpenClaw-owned. The added tests cover external edits before the locked write starts, but not an external edit interleaved during the locked callback.
  - confidence: 0.88
