## Summary

Keep this PR open: the real-filesystem proof is useful, but the latest head still has a P1 session trust-boundary defect where an explicit owned-write callback can publish a final fingerprint that includes an interleaved unowned file edit.

## What This Changes

This PR adds process-wide owned session-file fingerprint tracking and a `publishOwnedWrite` path so released embedded attempts can accept explicit same-process transcript writes, with focused session-lock regression tests.

## Best Possible Solution

Keep the ownership-aware fence only if explicit owned-write publication cannot launder unowned file state; otherwise narrow and document the takeover threat model before merge.

## Reproduction Assessment

Yes. Current main's strict fingerprint fence and the PR body's real-filesystem before/after proof establish the original race, and the remaining PR defect is source-reproducible by adding an external append inside the explicit owned transcript-write callback.

## Solution Assessment

No. The ownership-aware direction is plausible, but the current publication rule trusts the final fingerprint without proving it contains only OpenClaw-owned changes.
