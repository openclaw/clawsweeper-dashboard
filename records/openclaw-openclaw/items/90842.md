## Summary

Keep open: current main still matches the corrected source-level failure mode where binding/routing-only config edits can be treated as no-op reloads before the runtime snapshot advances, and the linked docs PR was closed unmerged after the reporter corrected the fix surface.

## What This Changes

Request to make per-inbound channel routing see binding/agent/routing config edits without requiring a gateway restart, and then document the resulting plugin contract.

## Best Possible Solution

Advance the runtime snapshot for binding/agent/routing-only config changes without restarting channels or the gateway, then document the resulting channel-plugin routing contract.

## Reproduction Assessment

Yes, at source level: current main classifies bindings/routing edits as no-op reloads and returns before the runtime snapshot activation path, while routing caches by config object identity.

## Solution Assessment

Unclear: a source fix is needed, but maintainers should choose between unconditional snapshot advancement for changed no-op paths and an explicit snapshot-only reload-plan kind before a repair PR.
