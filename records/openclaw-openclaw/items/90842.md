## Summary

Keep open: current main still leaves external binding-only config edits on the no-op reload path, so the runtime snapshot can stay pinned and channel routing can keep using stale bindings. The docs-only PR was closed after the correction, and no merged or open canonical fix now owns the source-level reload contract.

## What This Changes

The issue asks OpenClaw to make channel routing see updated bindings without a gateway restart, after correcting the original docs-only framing to a gateway runtime-snapshot reload problem.

## Best Possible Solution

Advance the gateway runtime snapshot for changed no-op routing/bindings config without restarting channels or the gateway, add a regression test for external binding-only edits, then document the plugin inbound routing expectation.

## Reproduction Assessment

Yes, at source level: an external binding-only config diff is classified as a no-op and returns before hot reload can activate a new runtime snapshot. I did not run a live channel repro in this read-only review.

## Solution Assessment

Unclear: a source fix is needed, but the best permanent shape needs maintainer direction between explicit snapshot-only reload classification and unconditional snapshot advancement for changed no-op paths.
