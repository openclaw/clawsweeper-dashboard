- `impact:message-loss`: Stale route bindings can send inbound channel messages to the wrong agent until the gateway is restarted.

## Summary

Keep open: current main still has the corrected gateway reload failure mode, and no merged related item owns the core fix. Binding/routing-only edits remain no-op reload plans that return before the hot-reload path activates a fresh runtime config snapshot.

## What This Changes

The issue asks OpenClaw to define and fix the runtime config refresh contract so channel routing sees binding/routing-only config edits without requiring a gateway restart.

## Best Possible Solution

Make runtime-dynamic no-op config changes advance the active runtime snapshot without per-event disk reads, then add focused gateway reload coverage for binding/routing-only edits and document the channel author contract only if it remains necessary.

## Reproduction Assessment

Yes. Source inspection shows a high-confidence reproduction path: change only `bindings` or `routing`, the reload plan is a no-op, and `applySnapshot` returns before the active runtime snapshot is swapped.

## Solution Assessment

Unclear. The best layer is the gateway reload/snapshot path rather than channel plugins, but maintainers still need to choose between an unconditional no-op snapshot swap and an explicit `swap-snapshot-only` plan kind.
