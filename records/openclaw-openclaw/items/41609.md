## Summary

Keep open. Current main still does not parse ordinary inbound Feishu `interactive` message events, while the merged parser repair only covers fetched/quoted messages and explicitly left the broader inbound-card work open.

## What This Changes

The issue asks OpenClaw to parse Feishu interactive-card message and callback content instead of exposing only placeholders or type identifiers.

## Best Possible Solution

Keep this as the broad Feishu interactive-card tracker, then land or replace a narrow plugin-owned fix that parses inbound card content and splits card-action field passthrough into the existing narrower callback contract issues if needed.

## Reproduction Assessment

Yes, source-reproducible: current main routes inbound Feishu message events through `parseMessageContent()` with no `interactive` branch, while the existing card parser is only used by fetched-message paths. I did not run a live Feishu tenant reproduction.

## Solution Assessment

Unclear as one patch: the Feishu plugin is the right boundary, but maintainers should scope inbound message parsing, fetched raw-card behavior, merge-forward card handling, and card-action field passthrough before selecting a canonical implementation.
