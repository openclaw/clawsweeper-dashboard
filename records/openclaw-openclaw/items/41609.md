## What This Changes

The issue requests Feishu interactive-card message parsing for inbound card text plus button, dropdown, and form callback content.

## Best Possible Solution

Keep this as the canonical Feishu plugin tracker, reuse or extract the existing interactive-card parser for inbound events, and land callback field passthrough through a deliberate agent-facing payload shape.

## Reproduction Assessment

Yes at source level, but not with live tenant proof. An inbound Feishu event with `message_type: "interactive"` reaches `parseMessageContent`, which has no interactive-card branch, and callback normalization still drops sibling action fields.

## Solution Assessment

Unclear as one broad change. The Feishu plugin is the right boundary, but maintainers should choose whether inbound card text parsing and callback option/form payload passthrough are one repair or separate focused fixes.
