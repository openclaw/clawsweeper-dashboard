## Summary

Keep open: current main still has a Feishu interactive-card message-loss gap in direct inbound events and merge-forward formatting, while readback parsing remains partial; overlapping open Feishu PRs should be consolidated by a maintainer rather than closed as fixed.

## What This Changes

The issue requests complete Feishu interactive-card parsing so inbound cards expose text, button/action labels, dropdown options, and form values instead of placeholders or raw JSON.

## Best Possible Solution

Consolidate one Feishu plugin fix that shares card parsing across direct inbound events, message read/thread/merge-forward readback, and card action extraction, with focused tests plus redacted live Feishu proof when credentials are available.

## Reproduction Assessment

Yes, at source level. A direct inbound `message_type: "interactive"` event still reaches `parseMessageContent()` without an interactive branch, and mocked readback/merge-forward shapes can reach partial parsers that drop card text or actions.

## Solution Assessment

No final solution is on main yet. The Feishu plugin is the right boundary, but the best path is maintainer consolidation of the overlapping parser/readback PRs rather than another independent automated fix lane.
