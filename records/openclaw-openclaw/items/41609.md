## Summary

Keep open. Current main still leaves Feishu interactive-card content incomplete in direct inbound events, message/thread readback, and merge-forward formatting, while several active Feishu parser PRs need maintainer consolidation rather than another automated fix lane.

## What This Changes

Feishu interactive-card messages should expose card text, button values, dropdown options, and form input content to agents instead of placeholder or raw type-only payloads.

## Best Possible Solution

Consolidate a Feishu plugin fix that covers direct inbound cards, message read/thread/merge-forward readback, and card action text/value extraction with focused tests and redacted live Feishu proof.

## Reproduction Assessment

Yes, at source level. Current main sends direct inbound interactive messages through parseMessageContent without an interactive branch, and readback still has limited card parsing plus no full-card request parameter.

## Solution Assessment

No final solution is on main yet. The Feishu plugin is the right boundary, but maintainers should consolidate the overlapping open parser/readback/action work and verify the Feishu API request shape before closing this issue.
