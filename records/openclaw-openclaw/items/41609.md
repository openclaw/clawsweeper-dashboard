## Summary

Keep open. Current main has partial Feishu interactive-card parsing for message reads, but the broader reported event/readback problem is still not fully handled: direct inbound interactive events are not parsed through an interactive-card branch, card readback does not request richer card payloads, and the active fixes are still split across open Feishu parser/readback PRs.

## What This Changes

The issue requests Feishu inbound interactive-card parsing so card text, buttons, options, and form values are surfaced instead of a placeholder or raw type marker.

## Best Possible Solution

Consolidate the active Feishu plugin work into one bounded parser/readback implementation that handles direct inbound events and message reads, preserves existing message-body semantics, adds focused regression tests, and includes redacted real Feishu proof.

## Reproduction Assessment

Yes, at source level. Current main routes direct inbound interactive events through a parser with no interactive branch, and message readback only has partial flat-card extraction coverage.

## Solution Assessment

No final solution is on main yet. The Feishu plugin is the right boundary, but the open PRs need maintainer consolidation around full-card retrieval, bounded parsing, and RawBody/user-text semantics before this issue can close.
