## Summary

Keep open: current main and v2026.5.22 still leave Feishu direct inbound interactive-card parsing and card-action sibling payload fields incomplete, and several open PRs overlap the remaining parser/callback contract rather than resolving it yet.

## What This Changes

Feishu interactive card messages and card-action callbacks should expose readable card text, button/select/form values, and relevant payload data to agents instead of placeholders or dropped fields.

## Best Possible Solution

Choose one canonical Feishu plugin implementation that handles direct inbound cards, readback/quoted and merge-forward card variants, and card action/form callbacks through a documented agent-visible payload contract with focused tests and redacted live Feishu proof.

## Solution Assessment

No final solution is on main yet. The Feishu plugin is the right boundary, but maintainers should consolidate the overlapping parser/readback/action PRs and decide the callback payload contract before merge.
