Full review comments:

- **[P1] Let configured Gateway auth resolve the remote target:** `src/cli/skills-cli.ts:114`
  - body: Passing `gateway.remote.url` as `opts.url` makes this a CLI URL override. The Gateway client deliberately requires explicit credentials for URL overrides and does not reuse configured remote credentials for non-env overrides, so authenticated remote setups can throw and silently fall back to local-only status.
  - confidence: 0.94
- **[P1] Use the unwrapped full skills.status report:** `src/cli/skills-cli.ts:118-124`
  - body: `GatewayClient.request()` resolves the response payload directly, so a successful `skills.status` call returns `SkillStatusReport`, not an `{ ok: true }` envelope. This check skips valid results, and the fallback return object would also drop required metadata like `managedSkillsDir`.
  - confidence: 0.93
