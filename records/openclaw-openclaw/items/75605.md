## Summary

Keep open. Current main does not contain the derived-path seam, so this PR is not obsolete, but the PR head still has security-relevant target mismatches between host-derived `derivedPaths` and the actual `apply_patch` executor semantics.

## What This Changes

The PR adds host-derived `apply_patch` target-path metadata to trusted policy and `before_tool_call` hook events, with SDK docs/API baseline updates, changelog text, matrix/onboarding fixes, and focused tests.

## Best Possible Solution

Keep the additive `derivedPaths` seam, but derive targets from the same semantics as `apply_patch` execution: preserve marker payload bytes after slicing the marker, and either use the sandbox bridge/resolver for sandboxed calls or omit derived paths when the host cannot derive them authoritatively enough for policy use. Add regressions for extra marker payload spaces and container or bind-mount sandbox paths.

## Reproduction Assessment

Yes. The marker mismatch is reproducible by comparing the executor's trim-then-slice behavior with the PR extractor's payload trim for `*** Add File:  src/new.ts`; the sandbox mismatch follows from executor source using `sandbox.bridge.resolvePath` while the PR deriver receives only a cwd.

## Solution Assessment

No. The proposed seam is narrow and useful, but this implementation is not the best fix until derived metadata mirrors the executor target set or is withheld when the host cannot compute that target set safely.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve apply_patch marker payload spaces:** `src/agents/apply-patch-paths.ts:56`
  - body: `normalizePatchPath` trims the path payload after the hunk marker. The executor trims the header and then slices after the marker, so `*** Add File:  src/new.ts` applies to a leading-space filename while `derivedPaths` reports `src/new.ts`. Trusted policies can therefore evaluate a different path from the one `apply_patch` writes.
  - confidence: 0.94
- **[P2] Resolve sandbox derived paths through the bridge:** `src/agents/pi-tools.ts:737`
  - body: Passing only `cwd: sandboxRoot ?? workspaceRoot` means derived paths are resolved with host cwd rules, while sandboxed `apply_patch` execution uses `sandbox.bridge.resolvePath`. Container or bind-mount inputs such as `/workspace/src/new.ts` can be reported as the container path even though execution touches the mapped host target, so policy hooks see the wrong path.
  - confidence: 0.88
