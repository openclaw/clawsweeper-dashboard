## Summary

Keep open: current main does not implement the requested `derivedPaths` hook metadata, and the PR is the active implementation. The final diff is well-scoped and exact-head checks are green, but review found a concrete path-fact correctness gap for native Codex hook relay calls before merge.

## What This Changes

The branch adds a host-owned `apply_patch` target-path extractor, a host tool-param parser registry, `derivedPaths` metadata for trusted-policy and `before_tool_call` events, docs/changelog/API baseline updates, and focused tests.

## Best Possible Solution

Land the additive `derivedPaths` seam after threading tool execution cwd into every `before_tool_call` call path that can expose `apply_patch` path facts, especially native Codex hook relay, with regression coverage that keeps the metadata best-effort and non-authoritative.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is a Codex native `pre_tool_use` payload for `apply_patch` with `cwd` different from `process.cwd()`; current relay metadata parses that cwd but the PR’s derivation path will not see it, so relative `derivedPaths` are rooted incorrectly.

## Solution Assessment

No, not as currently wired. The host-owned parser and additive event field are the right narrow design, but the implementation should pass existing native-hook cwd metadata into `runBeforeToolCallHook` before the path facts are safe to expose consistently.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Thread native hook cwd into derived path facts:** `src/agents/pi-tools.before-tool-call.ts:497`
  - body: `runBeforeToolCallHook` now derives `apply_patch` paths from `ctx.cwd` and otherwise falls back to `process.cwd()`, but the Codex native hook relay already parses `invocation.cwd` and calls this hook without passing it. Native `apply_patch` pre-tool events can therefore report relative `derivedPaths` under the gateway process cwd instead of the tool workspace, causing trusted or `before_tool_call` policies to decide on the wrong target path. Please pass the invocation cwd through this path and add a relay regression test.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The diff is mostly additive, but the new path-fact metadata can mis-root native Codex `apply_patch` targets when cwd is not threaded into derivation.

Concerns:

- **[medium] Native hook derived paths can use the wrong cwd:** `src/agents/pi-tools.before-tool-call.ts:497`
  - body: Because path facts may be consumed by trusted policy hooks, deriving relative `apply_patch` targets from `process.cwd()` on the native Codex relay path can misclassify the file that the native tool will touch. This is a security-adjacent policy correctness issue even though the docs correctly describe `derivedPaths` as best-effort hints.
  - confidence: 0.86
