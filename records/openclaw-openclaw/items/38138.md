## Summary

Keep this issue open. Current main still chooses the Discord agent from static routing inputs before mention detection, so a thread with a parentPeer binding can still resolve to the parent-channel agent while explicit mentions and per-agent mentionPatterns only gate the already selected route. Related #35821 attempted the requested behavior but was closed unmerged, and broader #13304 is not a canonical replacement for this Discord-specific routing override.

## What This Changes

Requests that explicit Discord @agent mentions, or an agent's configured mentionPatterns, override normal binding tiers so a thread can route to the mentioned agent instead of the parent-channel binding.

## Best Possible Solution

Add a documented, cache-safe explicit target-agent mention route step before Discord mention gating, with deterministic precedence rules and regression coverage for a thread inheriting one bound agent while mentioning another configured agent. Update channel-routing and Discord mention docs once the contract is settled.
