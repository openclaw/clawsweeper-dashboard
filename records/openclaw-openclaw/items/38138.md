## Summary

Keep open. Current main still chooses the Discord agent route from configured binding tiers before mention gating, and the related #13304 tracks broader content-based bindings rather than fully covering native/per-agent mention override semantics.

## What This Changes

Request that explicit agent mentions or per-agent mentionPatterns override inherited Discord thread/channel bindings when selecting the target agent.

## Best Possible Solution

Define an explicit mention-to-agent routing policy, likely coordinated with #13304, then add Discord preflight and core routing coverage for parentPeer threads, per-agent mentionPatterns, native mentions, and configured binding precedence.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for routing semantics that current docs and types do not define. Source inspection does show the reported current behavior path: routing resolves before mention gating and has no mention/content input.

## Solution Assessment

Unclear. The requested direction is plausible, but the best implementation needs maintainer judgment on precedence and whether this should be a dedicated mention override or part of the broader content-binding design in #13304.
