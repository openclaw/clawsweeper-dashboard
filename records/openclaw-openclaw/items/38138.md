## Summary

Keep open. Current main still resolves the Discord route before mention detection, and mentionPatterns are evaluated for the already-selected agent, so the requested mention-based override is not implemented; related #13304 is relevant but broader and does not by itself settle Discord/native mention precedence.

## What This Changes

The issue requests explicit mention-based Discord agent routing to override inherited channel/thread bindings.

## Best Possible Solution

Define the routing precedence contract, then implement either a generic content/mention routing seam or a Discord-aware explicit-mention override with regression coverage for parentPeer-bound threads.

## Reproduction Assessment

Yes from source inspection: a Discord thread with a parent channel binding resolves the route before mention detection, and resolveAgentRoute has no message-content or mention input. I did not run a live Discord repro.

## Solution Assessment

No current implementation solves this request. The proposed tier-0 or preflight override is plausible, but it should be reconciled with the broader content-routing design in #13304 before implementation.
