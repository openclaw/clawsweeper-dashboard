## Summary

Keep open. Current main still routes Discord messages from static binding/runtime route state before mention gating, and the open related content-routing and role-mention issues do not fully cover native per-agent mention override precedence for inherited Discord threads.

## What This Changes

Request mention-based Discord agent routing override so an explicit bot mention or per-agent mention pattern can route a thread message to that agent ahead of inherited parent bindings.

## Best Possible Solution

Define an explicit mention-to-agent routing policy, coordinated with content-routing work, then add focused Discord preflight/core routing coverage for parentPeer threads, native bot mentions, per-agent mentionPatterns, and binding precedence.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a new routing capability, not a documented contract failure. Source inspection gives high-confidence current behavior evidence: routing resolves from static metadata before mention gating, and no current input carries an alternate mentioned agent target.

## Solution Assessment

Unclear. The requested direction is plausible, but maintainers need to choose whether this is a dedicated mention override, part of binding content matching, or a plugin/preflight policy before implementation.
