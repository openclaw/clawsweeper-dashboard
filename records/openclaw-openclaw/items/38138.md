## Summary

Keep open. Current main still resolves Discord routes from static binding inputs before per-agent mention detection, so a parentPeer-bound thread can still route to the parent-channel agent even when the message explicitly targets another agent; the closed unmerged PR and broader content-routing issue do not supersede this Discord-specific request.

## What This Changes

The issue requests an explicit Discord mention-based routing override so a message that mentions another configured agent routes to that agent instead of the inherited parent-channel binding.

## Best Possible Solution

Add a documented, cache-safe explicit target-agent mention route step with deterministic precedence rules and regression coverage for parent-bound Discord threads.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Discord thread with parentConversationId bound to one agent and message content or mentions targeting another still resolves the route before mention detection, so effectiveRoute remains binding-derived.

## Solution Assessment

Unclear. The requested behavior is plausible, but the best implementation depends on maintainer decisions about whether this belongs as a resolver tier, a Discord preflight override, or an additive generic routing seam with explicit precedence rules.
