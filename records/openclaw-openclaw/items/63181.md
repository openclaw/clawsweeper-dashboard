## Summary

Keep this open. Current main still has the exact prompt-contract mismatch: the system prompt section is titled `Current Date & Time` while the implementation and cache-stability tests intentionally include only `Time zone: ...`; the separate `session_status` hint does not make that section truthful, and no open PR appears to address this issue directly.

## What This Changes

The issue asks OpenClaw to rename or clarify the timezone-only system prompt section so it no longer implies live current date/time is present.

## Best Possible Solution

Make the model-visible section truthful while preserving timezone-only cache stability: rename it to `Time Zone` or add an adjacent limitation that says current date/time must come from `session_status`, then align the prompt tests and date/time docs.

## Reproduction Assessment

Yes. The source-level reproduction is deterministic: build a system prompt with `userTimezone` set and current main emits `## Current Date & Time` with only `Time zone: ...`; the reported model misreasoning is probabilistic, but the misleading prompt shape is directly reproducible.

## Solution Assessment

Yes. The narrow maintainable fix is prompt-copy and docs/test alignment, not reintroducing dynamic current date/time into the stable system prompt without a broader cache-boundary design change.
