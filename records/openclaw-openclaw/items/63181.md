## What This Changes

The issue asks OpenClaw to rename or clarify the timezone-only system prompt section so it no longer implies that live current date/time is included.

## Best Possible Solution

## Reproduction Assessment

Yes for the prompt-shape part: current main emits `## Current Date & Time` with only a timezone line, and the adjacent test asserts that shape. The downstream April 8 model reasoning mistake was not live-replayed because it is probabilistic model behavior.

## Solution Assessment

Yes. Renaming the section to `Time Zone` is the narrowest maintainable fix because it removes the misleading heading without reintroducing dynamic date/time into the cache-stable system prompt.
