## Summary

Keep open. Current main and the latest release still expose a model-visible `## Current Date & Time` section that contains only `Time zone: ...`, and related time-source/cache items do not fix this narrower heading mismatch.

## What This Changes

The issue requests correcting the timezone-only system-prompt section so it is not labeled as live current date/time context.

## Best Possible Solution

Rename the model-visible heading to `Time Zone` and align docs/tests/changelog while preserving the cache-stable timezone-only prompt and leaving exact wall-clock behavior to `session_status` or late-bound runtime context.

## Reproduction Assessment

Yes for the prompt mismatch: current main source deterministically builds `## Current Date & Time` with only `Time zone: ...`, and tests assert that shape. The reported downstream date wording is probabilistic model behavior, so it should not be the regression oracle.

## Solution Assessment

Yes. A narrow prompt-heading, docs, and test update is the best fix; adding live date/time back into the cached system prompt would undermine the cache-stability contract established by prior work.
