## Summary

Keep open. Origin main still builds a model-visible `## Current Date & Time` section whose only content is `Time zone: ...`, while separate prompt guidance tells the model to call `session_status` for live date/time; the related closed cache and reset-greeting issues do not cover this label mismatch.

## What This Changes

The issue asks to rename or clarify the timezone-only system prompt section so it does not imply live current date/time is present.

## Best Possible Solution

Rename the model-visible section to `Time Zone` and update docs/tests/changelog while preserving the timezone-only cache-stable prompt contract and `session_status` as the live clock source.

## Reproduction Assessment

Yes for the prompt mismatch: source on origin/main deterministically emits `## Current Date & Time` with only `Time zone: ...` when `userTimezone` is set. The exact downstream April 8 wording is probabilistic model behavior, not a deterministic test oracle.

## Solution Assessment

Yes. A narrow prompt-copy, docs, and test update is the best fix; adding live date/time back into the cached system prompt would reopen the cache-stability problem addressed by the existing history.
