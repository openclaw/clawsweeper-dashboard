## What This Changes

The issue asks OpenClaw to align the timezone-only system prompt section name with its actual content so models are not cued to infer that live date/time is present.

## Best Possible Solution

Land or replace the narrow runtime/test/docs rename from the linked PR, keeping timezone-only prompt behavior and `session_status` as the live-time source.

## Reproduction Assessment

Yes for the prompt-shape bug: current source still emits a `Current Date & Time` heading with only a timezone line. The downstream model date mistake was not replayed live in this read-only review.

## Solution Assessment

Yes. The narrow `Time Zone` rename is the best current direction because it preserves cache-stable timezone-only behavior and removes the misleading affordance without reintroducing dynamic timestamps.
