## Summary

Keep open. Current main and the latest release still expose a model-visible `## Current Date & Time` section that contains only `Time zone: ...`, while the same prompt/docs point live date/time needs at `session_status`; adjacent issues cover broader clock authority, volatile timestamp placement, or heartbeat formatting rather than this narrow misleading heading.

## What This Changes

Rename or clarify the timezone-only system-prompt section so it no longer implies live date/time is included.

## Best Possible Solution

Rename the model-visible heading to `Time Zone` and align docs/tests/changelog while preserving the cache-stable timezone-only prompt and leaving exact wall-clock behavior to `session_status` or late-bound event context.

## Reproduction Assessment

Yes for the prompt mismatch: source inspection deterministically shows `## Current Date & Time` with only `Time zone: ...` whenever a user timezone is present. The exact April 8 `due yesterday` wording is probabilistic model behavior, so it is not a stable test oracle.

## Solution Assessment

Yes. A narrow prompt-heading, docs, and test update is the best fix; adding live date/time back into the cached system prompt would undo the cache-stability behavior established by earlier work.
