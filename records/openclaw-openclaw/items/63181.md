## Summary

Keep open. Current main still has the reported mismatch: `buildTimeSection` renders a `Current Date & Time` heading with only a `Time zone:` line, while tests and docs preserve the timezone-only cache-stable behavior instead of making the heading truthful.

## What This Changes

The issue requests making the timezone-only system-prompt time section truthful so it does not imply live date/time is present.

## Best Possible Solution

Rename the model-visible heading to `Time Zone` or explicitly state that live date/time is absent, then align the focused tests, date/time docs, timezone docs, system-prompt docs, and changelog while preserving prompt cache stability.

## Reproduction Assessment

Yes. Building or inspecting `buildAgentSystemPrompt` with `agents.defaults.userTimezone` set deterministically produces `## Current Date & Time` with only a `Time zone:` line; the downstream wrong date inference is model-dependent.

## Solution Assessment

Yes. The narrowest maintainable solution is prompt copy plus docs/test alignment; reintroducing live date/time before the cache boundary would solve a broader problem but would regress the cache-stability contract.
