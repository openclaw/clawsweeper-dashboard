## Summary

Keep open: current main and the latest release still emit and document the misleading timezone-only `Current Date & Time` section, and an open linked PR already owns the narrow rename fix.

## What This Changes

The issue asks OpenClaw to stop labeling a timezone-only system prompt block as `Current Date & Time`, preferably by renaming it to `Time Zone` while preserving cache-stable timezone-only behavior.

## Best Possible Solution

Review and land the existing narrow fix PR if its proof and CI gates clear; the desired end state is a cache-stable timezone-only prompt section named for timezone, with docs and the adjacent prompt test aligned.

## Reproduction Assessment

Yes from source: current main builds a prompt section titled `Current Date & Time` while only injecting `Time zone: ...`, and docs/tests still encode that shape. I did not reproduce the reported model output live.

## Solution Assessment

Yes, the linked narrow rename is the best maintainable fix for this issue because it aligns prompt wording, tests, and docs without reintroducing dynamic timestamps before the cache boundary.
