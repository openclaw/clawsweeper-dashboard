## Summary

Keep open. Current main still emits the reported mismatch: the model-visible section is titled `Current Date & Time` while the section body contains only `Time zone: ...`; related temporal-grounding issues do not directly fix this prompt-label problem.

## What This Changes

The issue asks to make the prompt's timezone-only time section label truthful or explicitly direct agents to fetch current time separately.

## Best Possible Solution

Rename the model-visible section to `Time Zone` or otherwise make the timezone-only contract explicit, while preserving prompt cache stability and `session_status` as the dynamic clock source.

## Reproduction Assessment

Yes. Build or inspect the system prompt with `agents.defaults.userTimezone` set; current main deterministically emits `## Current Date & Time` with only a `Time zone:` line, though the exact wrong date inference is probabilistic.

## Solution Assessment

Yes. A narrow prompt-copy plus docs/test/changelog update is the best path; reintroducing live date/time into the cached system prompt would reopen the cache-stability problem this code intentionally avoided.
