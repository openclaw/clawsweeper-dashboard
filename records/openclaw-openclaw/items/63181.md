## Summary

Keep open: current `main` and the latest release still emit the timezone-only prompt block under `Current Date & Time`, and the open linked PR is the active fix path rather than a reason to close this issue before merge.

## What This Changes

The issue asks OpenClaw to stop labeling a timezone-only system-prompt block as current date/time context.

## Best Possible Solution

Land or reject the narrow linked PR that renames the section while preserving cache-stable timezone-only prompt behavior, then close this issue after that path is resolved.

## Reproduction Assessment

Yes for the prompt-shape part: current source still emits `## Current Date & Time` with only a timezone line, and the adjacent test asserts that shape. I did not reproduce the model's downstream April 8 reasoning error in a live session.

## Solution Assessment

Yes. Renaming the section to `Time Zone` is the narrowest maintainable fix for the mismatch because it removes the misleading heading without reintroducing dynamic date/time into the cache-stable system prompt.
