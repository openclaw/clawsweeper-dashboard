## Summary

Keep open: current main and v2026.6.8 still emit the timezone-only section under `Current Date & Time`, and the open linked fix PR is the active implementation path rather than a reason to close this issue before merge.

## What This Changes

The issue asks OpenClaw to stop labeling a timezone-only system prompt section as `Current Date & Time`, either by renaming it or by making the live-time limitation explicit.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/91250 once its proof gate is satisfied or explicitly waived, keeping timezone-only prompt caching and `session_status` as the live-time path.

## Reproduction Assessment

Yes for the source-level bug: current main builds a `Current Date & Time` section containing only `Time zone: ...`, and the adjacent test asserts that heading. I did not live-replay the probabilistic downstream model date mistake.

## Solution Assessment

Yes. Renaming the section to `Time Zone` is the narrowest maintainable fix because it removes the misleading heading without reintroducing dynamic date/time into the cache-stable system prompt.
