## Summary

Keep open. Current main still exposes the reported mismatch: the model-visible section is titled `Current Date & Time` while `buildTimeSection` emits only `Time zone: ...`, and the focused tests/docs still preserve or describe that timezone-only contract rather than making the heading truthful.

## What This Changes

The issue asks OpenClaw to stop labeling a timezone-only system prompt block as `Current Date & Time` because that prompt shape can mislead date reasoning.

## Best Possible Solution

Rename the model-visible section to `Time Zone` or explicitly state that current date/time is not included, then align prompt tests, date/time docs, timezone docs, and changelog while preserving prompt cache stability.

## Reproduction Assessment

Yes. A high-confidence reproduction is to build or inspect the system prompt with `agents.defaults.userTimezone` set: current main deterministically emits `## Current Date & Time` with only a `Time zone:` line. The exact wrong date inference is probabilistic, but the misleading prompt shape is reproducible from source and tests.

## Solution Assessment

Yes. The narrowest maintainable fix is prompt copy plus docs/test alignment; adding live date/time back into the cached system prompt would be a broader cache-boundary change and is not needed to fix this report.
