## Summary

Keep open. Current main still emits the reported misleading prompt shape: the section is titled `Current Date & Time` while its body is timezone-only, and the tests/docs intentionally preserve that cache-stable contract rather than making the heading truthful.

## What This Changes

The issue requests renaming or clarifying the timezone-only system prompt section so it no longer implies that live current date/time is present.

## Best Possible Solution

Make the model-visible section truthful while preserving the timezone-only cache-stability contract: rename it to `Time Zone` or add an explicit limitation beside the section, then align the prompt tests and date/time docs.

## Reproduction Assessment

Yes. Build a system prompt with `userTimezone` set and current main deterministically emits `## Current Date & Time` with only `Time zone: ...`; the exact model misreasoning is not deterministic, but the misleading prompt shape is.

## Solution Assessment

Yes. The narrow maintainable solution is prompt copy plus docs/test alignment; adding dynamic current date/time back into the stable system prompt is a riskier design change and not needed for this bug.
