## Summary

Keep open. Current main still emits a model-visible `## Current Date & Time` section containing only `Time zone: ...`, while the same prompt and docs tell agents to use `session_status` for live date/time; related issues cover cache stability, reset greetings, heartbeat formatting, or volatile event prompts, not this misleading stable-heading mismatch.

## What This Changes

The issue requests renaming or clarifying the timezone-only system-prompt section so it does not imply that current date/time is present.

## Best Possible Solution

Rename the model-visible section to `Time Zone` and update docs/tests/changelog while keeping live date/time available through `session_status` or existing late-bound message/event timestamps.

## Reproduction Assessment

Yes for the prompt mismatch: current source deterministically emits `## Current Date & Time` with only `Time zone: ...` whenever `userTimezone` is set. The exact downstream April 8 response is probabilistic model behavior, so it is not a stable test oracle.

## Solution Assessment

Yes. A narrow prompt-heading, docs, and test update is the best fix; adding live date/time back into the cached system prompt would undo the cache-stability behavior established by prior work.
