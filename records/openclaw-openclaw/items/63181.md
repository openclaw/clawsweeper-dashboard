## Summary

Keep open. Current main still has the deterministic mismatch reported here: the model-visible section is titled `Current Date & Time`, but implementation, tests, and docs all preserve a timezone-only body; the related temporal-grounding items do not directly cover this normal system-prompt label problem.

## What This Changes

Request to rename or clarify the timezone-only system-prompt section so it no longer implies that current date/time is included.

## Best Possible Solution

Rename the section to `Time Zone` or explicitly state that current date/time is not included, while keeping dynamic current time in `session_status` and message timestamps for cache stability.

## Reproduction Assessment

Yes. Inspecting or building the system prompt with `agents.defaults.userTimezone` set deterministically produces `## Current Date & Time` with only a `Time zone:` line; only the downstream wrong-date wording is probabilistic.

## Solution Assessment

Yes. A narrow prompt-copy, docs, test, and changelog update is the maintainable fix; reintroducing live date/time into the cached system prompt would conflict with the cache-stability behavior added for #63071.
