## What This Changes

The issue asks for structured warning-level diagnostics when an inbound channel route is dropped because the resolved channel config denies it.

## Best Possible Solution

Add warning-level, structured, rate-limited policy-denial logging at the maintainer-chosen Slack or shared channel ingress boundary while preserving fail-closed channel semantics.
