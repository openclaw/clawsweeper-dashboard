## Summary

Keep this open: current main still has the help/version bypass for unknown CLI roots, and https://github.com/openclaw/openclaw/pull/81083 is an open closing PR for the same bug.

## What This Changes

Reject unknown `openclaw <command>` roots consistently when `--help` or `--version` is appended instead of falling through to generic top-level help with exit 0.

## Best Possible Solution

Review and land the focused CLI dispatch PR, or close that PR with a maintainer rationale and keep this issue as the canonical bug until the behavior is fixed.

## Reproduction Assessment

Yes. Source inspection on current main shows `openclaw heartbeat --help` skips the unknown-primary error path because both the helper and caller are gated off for help/version invocations; the reporter output and linked PR proof match that path.

## Solution Assessment

Yes. The narrow fix is to validate unowned command roots before generic help/version fallthrough while preserving legitimate root and plugin help paths; the open linked PR is already aimed at that boundary.
