## Summary

## What This Changes

The issue asks for `openclaw <unknown-command> --help` and `--version` to report the same unknown-command error and nonzero exit status as the no-help form.

## Best Possible Solution

## Solution Assessment

Yes. The narrow maintainable solution is to validate unowned command roots before generic help/version fallthrough while preserving legitimate root, built-in, and plugin help paths; the linked PR targets that boundary.
