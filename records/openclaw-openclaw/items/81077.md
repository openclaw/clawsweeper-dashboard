## What This Changes

The issue asks OpenClaw to reject unknown top-level CLI commands consistently when `--help` is present instead of showing generic root help and exiting 0.

## Best Possible Solution

Fix the CLI dispatch guard so unknown roots are validated before generic help fallthrough, while preserving normal help for real built-in and plugin-owned command roots.

## Solution Assessment
