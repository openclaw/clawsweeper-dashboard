## Summary

Keep this open: current main still skips unknown-command validation for help/version invocations, and https://github.com/openclaw/openclaw/pull/81083 is an open closing PR for the same bug.

## What This Changes

The issue requests that `openclaw <unknown-command> --help` and `--version` report an unknown command with a non-zero exit instead of showing generic top-level help.

## Best Possible Solution

Review and land the focused CLI dispatch PR, or close that PR with a maintainer rationale while keeping this issue as the canonical bug until the behavior is fixed on main.

## Reproduction Assessment

Yes. Source inspection on current main shows `openclaw heartbeat --help` skips the unknown-primary error path because both the helper and caller are gated off for help/version invocations; the reporter output and linked PR proof match that path.

## Solution Assessment

Yes. The narrow maintainable fix is to validate unowned command roots before generic help/version fallthrough while preserving legitimate root, built-in, and plugin help paths; https://github.com/openclaw/openclaw/pull/81083 targets that boundary.
