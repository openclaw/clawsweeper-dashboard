## Summary

Keep open: current main and the latest shipped release still bypass unknown-command rejection when help/version is present, and an open closing PR already carries the focused fix.

## What This Changes

The issue requests consistent nonzero unknown-command handling for `openclaw <unknown-command> --help` instead of silently showing generic root help.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81083, or reject it with rationale and keep this issue as the canonical bug until CLI dispatch validates unknown roots before generic help/version routing.

## Reproduction Assessment

Yes. Source inspection on current main shows `openclaw heartbeat --help` skips unknown-root validation because both the helper and caller are gated off for help/version invocations, matching the reporter's commands and the linked PR proof.

## Solution Assessment

Yes. The narrow maintainable fix is to validate unowned roots before generic help/version fallthrough while preserving legitimate root, built-in, and plugin help paths; the open linked PR targets that boundary.
