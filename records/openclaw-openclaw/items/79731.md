## Summary

Keep open: current main still has the Minimax portal catalog manually detecting profiles and returning the OAuth marker, and both linked fix PRs were closed without merging. The issue has concrete release/version steps and a narrow likely fix path through the existing provider auth resolver seam.

## What This Changes

MiniMax portal OAuth profiles should resolve to usable runtime credentials for model requests instead of failing with missing provider auth after login.

## Best Possible Solution

Land a focused Minimax plugin fix that uses the existing provider catalog auth resolver for portal OAuth, adds regression coverage, and leaves refresh or agent-scope problems to their separate reports.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. Source inspection on current main shows the Minimax portal catalog still bypasses the OAuth-aware resolver, and the linked closed PRs trace the same narrow marker-resolution path.

## Solution Assessment

Yes, `ctx.resolveProviderAuth(PORTAL_PROVIDER_ID, { oauthMarker: MINIMAX_OAUTH_MARKER })` appears to be the narrowest maintainable fix because it uses the existing plugin SDK catalog auth seam already used by Chutes. The previous PR form was not ideal because it mixed in an unrelated Google onboarding change.
