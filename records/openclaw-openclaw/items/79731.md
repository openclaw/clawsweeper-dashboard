## Summary

Keep open: current main still has the MiniMax portal OAuth catalog path that treats an OAuth profile as a marker instead of resolving it through the provider-auth resolver, and an open PR with closing syntax is already tracking the narrow fix.

## What This Changes

The issue reports that MiniMax OAuth login writes a `minimax-portal` profile, but chat/model requests still fail instead of using the stored OAuth token.

## Best Possible Solution

Land a clean MiniMax-only fix that resolves portal OAuth through provider-auth with regression coverage, then close this issue after the fixing PR merges.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection on current main confirms a credible failing path: `minimax-portal` OAuth profiles lead to a raw MiniMax OAuth marker in the provider catalog instead of the resolved OAuth credential.

## Solution Assessment

Yes, the linked PR direction of using `ctx.resolveProviderAuth` is the narrow owner-module fix and matches the existing Chutes OAuth-marker pattern. Maintainers should keep the final patch scoped to MiniMax and include regression coverage for the profile-backed catalog path.
