## What This Changes

The branch updates uninstall state cleanup, tests, docs, and changelog so `openclaw uninstall --state` preserves configured workspace directories unless workspace removal is selected.

## Best Possible Solution

Land this PR or an equivalent narrow fix that preserves configured nested workspaces during state-only cleanup while keeping explicit `--workspace` and `--all` deletion behavior destructive.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the default workspace is under `$HOME/.openclaw/workspace`, state-only cleanup recursively removes `$HOME/.openclaw`, and workspace cleanup is skipped unless the workspace scope is selected.

## Solution Assessment

Yes. Passing configured workspace directories as preservation paths and pruning state contents around them is the narrow maintainable fix; it avoids adding a new option or weakening explicit workspace deletion.
