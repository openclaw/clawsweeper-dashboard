## Summary

Keep open: current main still has the state-only uninstall data-loss path, and this PR is a focused fix with regression coverage, docs, changelog, and no blocking review findings.

## What This Changes

The branch preserves configured workspace paths during state cleanup unless workspace removal is selected, and updates uninstall tests, docs, and changelog.

## Best Possible Solution

Land the focused preservation change after maintainer review, keeping explicit workspace removal destructive and documenting the manual rm -rf caveat.

## Reproduction Assessment

Yes. Source inspection on current main shows the default workspace under .openclaw/workspace, state-only uninstall calls state cleanup without preservePaths, and state cleanup recursively removes the state directory.

## Solution Assessment

Yes. Selectively pruning state contents around configured workspace paths is narrower than skipping state cleanup entirely, and it preserves --workspace and --all as intentionally destructive.
