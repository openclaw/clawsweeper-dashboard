## What This Changes

The PR preserves configured workspace directories during state-only uninstall, adds recursive state cleanup that skips preserved paths, covers the nested-workspace regression, and updates uninstall docs.

## Best Possible Solution

Land this PR after adding a single-line Unreleased > Fixes changelog entry and letting exact-head CI gate the branch, with #75052 linked and closed unmerged #75062 treated as superseded context.

## Reproduction Assessment

Yes. Current main's default workspace is inside the default state directory, and state-only uninstall recursively removes that parent state directory without any workspace preservation hook.

## Solution Assessment

Yes, after the changelog entry. Selectively pruning state contents while skipping configured workspace paths is narrower than skipping all state cleanup, and the PR keeps --workspace and --all intentionally destructive.
