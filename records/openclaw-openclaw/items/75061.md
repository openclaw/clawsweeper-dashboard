## Summary

Keep open: the PR is pointed at a real data-loss path, but the preservation list still misses implicit agent workspaces under the state directory when agents rely on runtime workspace defaults rather than explicit workspace strings.

## What This Changes

The PR adds preservation-aware state cleanup for state-only uninstall, wires uninstall to pass workspace paths when workspace removal is not selected, and updates regression tests, docs, and the changelog.

## Best Possible Solution

Update the cleanup plan or uninstall caller to preserve every runtime-resolved agent workspace under state, then land this PR with regression coverage, docs, and changelog intact.

## Reproduction Assessment

Yes. Source inspection shows current main recursively removes the state dir while the default workspace lives under that tree, and the remaining PR gap is source-reproducible for implicit per-agent workspace paths resolved under state.

## Solution Assessment

No. The selective-pruning direction is maintainable, but it should use the runtime agent workspace resolver or equivalent complete workspace list before pruning state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve implicit agent workspaces as well:** `src/commands/uninstall.ts:179`
  - body: This passes the cleanup plan's `workspaceDirs` as the entire preserve list, but that list only includes explicit workspace strings and falls back to `.openclaw/workspace` only when no explicit workspaces exist. Agents that rely on implicit runtime paths such as `<stateDir>/workspace-<id>` are therefore still absent from `preservePaths`, so `openclaw uninstall --state` can continue deleting those workspace directories. Build the preserve list from the same agent workspace resolver used at runtime, or extend the cleanup plan to include those implicit paths.
