## Summary

Keep open: current main and v2026.6.8 partially fix the first `+ New Session` display by copying the parent model onto the child, but the Default-selection path still has a source-level parent override fallback through the auto-reply model-selection path.

## What This Changes

The issue requests that WebChat child sessions not silently route through a parent session's model override when the UI shows the configured Default model.

## Best Possible Solution

Define a single child-session model contract: either show inherited parent selection explicitly everywhere, or add a narrow child-level default/no-inherit state that prevents parent fallback without breaking thread/channel inheritance.

## Reproduction Assessment

Yes, at source level. A parent-linked child can clear its direct override via the Default selection, after which `resolveStoredModelOverride` still falls back to the parent override in the auto-reply dispatch path.

## Solution Assessment

No, current main is not the complete fix. It fixes the initial child creation display by making inherited selection explicit, but the best fix still needs a maintainer-chosen contract for Default/no-inherit versus intentional parent fallback.
