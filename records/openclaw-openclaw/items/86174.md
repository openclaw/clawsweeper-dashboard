## Summary

Keep open: current main still has the source path where WebChat creates a parent-linked dashboard child, the picker shows the configured default, and model resolution can inherit the parent session override. The related open issues are adjacent, but they do not own this specific child-session routing mismatch.

## What This Changes

The issue reports that WebChat `+ New Session` can display the configured default model while the first run inherits the selected parent session's model override.

## Best Possible Solution

Preserve intentional parent override inheritance for thread-like sessions, but make dashboard `+ New Session` model state explicit so the picker and first run agree.

## Reproduction Assessment

Yes. The source path is high-confidence: WebChat creates a parent-linked child, the picker reads child/default state, and runtime model selection can inherit the parent override; I did not execute a live browser run in this read-only review.

## Solution Assessment

Unclear. The report identifies a real bug, but maintainers need to choose whether dashboard new sessions should be independent, display inherited state, or support an explicit child-level configured-default sentinel.
