Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Fix the app-server request helper types:** `extensions/codex/src/app-server/plugin-tool-invoker.ts:111-112`
  - body: Latest-head CI reports package-boundary type failures because `makeRequest` returns a method-generic helper that no longer matches `CodexPluginBridgeRequest` or the `CodexAppServerClient.request` overloads. As a result, typed calls like `request<{ thread: ... }>("thread/start")` are inferred as generic JSON and the branch does not compile.
  - confidence: 0.95
- **[P1] Remove the CLI runtime from migration context:** `src/commands/migrate/context.ts:44`
  - body: `MigrationProviderContext.runtime` is a `PluginRuntime`, but this assigns the CLI `RuntimeEnv` into it. Current prod/test typecheck annotations fail on this line, and config migration should receive a real config runtime handle instead of the CLI logging runtime.
  - confidence: 0.94
- **[P1] Accept nullable plugin elicitation turn IDs:** `extensions/codex/src/app-server/plugin-tool-invoker.ts:366`
  - body: Codex app-server allows `mcpServer/elicitation/request` to carry `turnId: null`, and the existing elicitation bridge accepts that when the thread matches. This matcher requires a string turn id, so valid plugin approval prompts are ignored and the bridged tool can hang or fail instead of applying the configured approval policy.
  - confidence: 0.91
- **[P2] Refresh the plugin idle timer on progress:** `extensions/codex/src/app-server/plugin-tool-invoker.ts:220-224`
  - body: The plugin turn starts a fixed 30s timer after `turn/start` and never refreshes it when deltas, completed items, elicitations, or other progress arrive. Active plugin turns that take more than 30 seconds before `turn/completed` can reject despite the surrounding 5-minute tool timeout.
  - confidence: 0.88
- **[P3] Add contributor credit to the changelog entry:** `CHANGELOG.md:11`
  - body: This user-facing feature changelog entry is for an external human contributor but does not include the required `Thanks @kevinslin` attribution, so it does not follow the repo's changelog policy.
  - confidence: 0.82
