## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Handle auth-required plugin installs during migration:** `extensions/codex/src/migration/apply.ts:151-156`
  - body: `plugin/install` can return `appsNeedingAuth`, but this path discards the response and marks the item migrated. OAuth-required plugins then get enabled in OpenClaw config even though the resulting tool immediately fails until the user fixes auth out of band.
  - confidence: 0.9
- **[P2] Page through app inventory before enabling apps:** `extensions/codex/src/app-server/plugin-tool-invoker.ts:120-122`
  - body: This reads only the first `app/list` page before deciding which Codex apps to enable. If the selected plugin's app is behind `nextCursor`, `appIds` stays empty and the bridge starts the plugin turn without enabling the required app.
  - confidence: 0.9
- **[P2] Accept nullable plugin elicitation turn IDs:** `extensions/codex/src/app-server/plugin-tool-invoker.ts:367`
  - body: Codex app-server allows `mcpServer/elicitation/request` to carry `turnId: null`, and the existing elicitation bridge accepts that shape when the thread matches. This matcher rejects nullable turn IDs, so valid plugin approval prompts can be ignored and the bridged tool can hang or fail.
  - confidence: 0.92
- **[P2] Refresh the plugin idle timer on progress:** `extensions/codex/src/app-server/plugin-tool-invoker.ts:221-225`
  - body: The plugin turn starts a fixed 30s idle timer after `turn/start` and never refreshes it when deltas, completed items, elicitations, or other progress arrives. Active plugin turns that take longer than 30 seconds before `turn/completed` can reject despite the surrounding 5-minute timeout.
  - confidence: 0.86
- **[P2] Use a valid Codex model provider for plugin threads:** `extensions/codex/src/app-server/plugin-tool-invoker.ts:76-78`
  - body: This sends only the stripped model id from the OpenClaw default model. In mixed-provider configs such as `anthropic/claude-sonnet-4.6`, app-server receives `claude-sonnet-4.6` without a `modelProvider`, so `thread/start` can fail before the plugin runs.
  - confidence: 0.88
- **[P2] Preserve cached plugin review items after discovery:** `extensions/codex/src/migration/source.ts:310`
  - body: When app-server plugin discovery succeeds, this sets cached plugin bundles to `[]`, so non-curated or local cached bundles disappear from the migration plan instead of being reported as manual-review items as the docs promise.
  - confidence: 0.87
- **[P3] Credit the external contributor in the changelog:** `CHANGELOG.md:12`
  - body: This user-facing feature entry is for an external human contributor, but the changelog line does not include the required `Thanks @kevinslin` attribution.
  - confidence: 0.82
