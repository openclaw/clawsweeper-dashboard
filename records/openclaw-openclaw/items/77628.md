## Summary

Keep open: current main fixes the older raw-key label problem and has subagent lineage metadata available, but the Control UI session picker still renders a flat per-agent option list rather than a parent-child subagent view.

## What This Changes

The issue requests clearer Control UI handling for subagent sessions spawned with `sessions_spawn`, either by grouping them under parent sessions, hiding them from the main dropdown, or making switching reliable.

## Best Possible Solution

Choose a Control UI subagent presentation model and implement it in the existing session selector using the gateway lineage fields, with regression coverage for labeled parent and child sessions.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows the current selector still renders subagent rows as flat peers under the agent optgroup, while existing tests cover label display rather than parent-child grouping.

## Solution Assessment

Unclear until maintainers choose the desired UX. Current main is a good base because labels and switch-by-key are already handled and lineage metadata is present, but it does not implement the requested hierarchy or alternate subagent surface.
