## Summary

Keep open: current main has subagent lineage metadata and the older raw-label dropdown problem is covered, but the Control UI session selector still presents sessions as a flat agent-scoped list. The remaining request is a UX/product decision about whether subagents should be indented, hidden, or moved to a separate control.

## What This Changes

Request to make Control UI present spawned subagents as children of their parent session, or otherwise avoid showing them as peer sessions while preserving reliable switching.

## Best Possible Solution

Choose one Control UI product model for subagent visibility, then implement it in the session picker or a dedicated subagent panel using existing lineage metadata without changing core subagent session isolation.
