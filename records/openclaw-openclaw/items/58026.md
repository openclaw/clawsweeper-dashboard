## Summary

Keep open: current main still treats `~/...` builtin memory `extraPaths` as workspace-relative, and the latest release still has the same normalizer behavior. The duplicate issue is closed, and both focused fixing PRs were closed unmerged, so this remains the canonical implementation bug.

## What This Changes

The issue requests home-prefix expansion for `memorySearch.extraPaths` so `~/...` resolves under the user home directory instead of the agent workspace.

## Best Possible Solution

Patch the package-owned builtin memory normalizer to expand a leading home prefix before the absolute/workspace-relative branch, then add focused regression coverage for normalization and builtin list/read behavior through a home-relative extra path.

## Reproduction Assessment

Yes. Current main’s normalizer branch plus a direct Node path check reproduce the reported `/workspace/~/...` path shape, and both builtin list and read call sites still use that normalizer.

## Solution Assessment

Yes. The narrow maintainable fix is in `packages/memory-host-sdk/src/host/internal.ts`; package wrappers should keep re-exporting the package implementation, and QMD’s existing resolver should be left alone.
