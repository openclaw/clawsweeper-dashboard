## Best Possible Solution

Expand a leading home prefix in the canonical builtin `normalizeExtraMemoryPaths()` path before the absolute/workspace-relative branch, add focused regression coverage, and keep doctor diagnostics aligned with the shared runtime behavior.

Yes. The narrow maintainable fix is to expand a leading home prefix in `normalizeExtraMemoryPaths()` before checking `path.isAbsolute()`, while preserving absolute paths, workspace-relative paths, dedupe, and symlink-safety behavior.
