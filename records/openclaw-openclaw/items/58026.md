## Summary

Keep open. Current main still sends `~/...` through the builtin memory extra-path normalizer as a workspace-relative path, while the related fix PR was closed unmerged and the latest release tag still contains the same affected branch.

## What This Changes

The issue asks for `memorySearch.extraPaths` entries beginning with `~/` to expand to the user's home directory before builtin memory path normalization resolves them.

## Best Possible Solution

Fix the package-level builtin memory extra-path normalizer so `~/...` is expanded before the absolute-versus-workspace-relative branch, keep QMD behavior unchanged, and add regression coverage that proves home-relative extra paths are normalized and scanned or read correctly.

## Reproduction Assessment

Yes. A read-only Node check of the same branch logic reproduces the workspace-relative resolution for `~/OneDrive/MyVault`, and current main still uses that branch in `normalizeExtraMemoryPaths()`.

## Solution Assessment

Yes. A narrow normalizer change plus focused tests is the best path; the existing QMD resolver already handles tilde, so this should stay in the builtin memory SDK path rather than broad config migration or backend rewiring.
