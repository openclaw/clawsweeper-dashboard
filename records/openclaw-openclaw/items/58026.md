## Summary

Keep open: current main and the latest release still route `~/...` builtin memory `extraPaths` through the workspace-relative branch, and the duplicate issue plus focused fix PRs are already closed without landing a fix.

## What This Changes

The issue requests that builtin memory `extraPaths` expand a leading `~/` to the user's home directory instead of resolving it under each agent workspace.

## Best Possible Solution

Builtin memory should normalize home-prefixed `extraPaths` through the same home expansion contract that QMD already uses, with focused regression coverage for normalization plus list/read behavior.

## Reproduction Assessment

Yes. Source inspection on current main plus the direct Node path check shows the exact `/workspace/~/...` resolution branch, and both builtin list and read call sites still use that normalizer.

## Solution Assessment

Yes. The narrow maintainable fix is to expand a leading home prefix in `normalizeExtraMemoryPaths()` before the absolute/workspace-relative branch, while leaving QMD collection behavior and read-path symlink protections untouched.
