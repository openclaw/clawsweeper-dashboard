## Summary

Keep open: current main still resolves builtin memory `extraPaths` with `path.isAbsolute()` before any home-prefix expansion, so `~/...` remains workspace-relative. The QMD backend has a separate resolver that expands `~`, and the related fixing PR was closed unmerged.

## What This Changes

Request expansion of `~/...` entries in `agents.defaults.memorySearch.extraPaths` before builtin memory path normalization resolves them against each agent workspace.

## Best Possible Solution

Patch the package normalizer to expand a leading home prefix before the absolute/workspace-relative branch, then add focused builtin memory tests for normalization and file listing or reading through a home-relative extra path.

## Reproduction Assessment

Yes. Current main's branch logic reproduces the reported path shape by resolving `~/OneDrive/MyVault` under the workspace, and both builtin list/read call sites still use that normalizer.

## Solution Assessment

Yes. The narrow maintainable fix is in `packages/memory-host-sdk/src/host/internal.ts`; the current `src/memory-host-sdk/host/internal.ts` wrapper re-exports the package implementation, so duplicating the source edit there is no longer needed.
