## What This Changes

The issue requests that direct TUI/WebChat or Telegram sessions keep their explicit/origin display label instead of being formatted through the group display-name helper after Telegram interaction.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/79040 or an equivalent shared guard with regression coverage, then let the closing reference or a post-merge maintainer cleanup close this issue with proof.

## Reproduction Assessment

Yes. Source inspection on current main shows a direct session with `chatType: "direct"`, sticky `channel: "telegram"`, and an origin label will reach `buildGroupDisplayName()` before the direct-label fallback in both affected display paths.

## Solution Assessment

Yes. The linked PR's shared group/channel guard is the narrow maintainable fix because it preserves group/channel labels while letting direct sessions fall through to explicit or origin labels.
