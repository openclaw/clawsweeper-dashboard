## Summary

Keep open. Current main still has the reported display-name path: a direct session with a sticky `entry.channel` can be formatted by the group display-name builder before falling back to the TUI/WebChat origin label, and no open PR was found that specifically closes this issue.

## What This Changes

The issue reports that the TUI footer shows a Telegram group-style display name for a direct main session after Telegram interaction instead of preserving the TUI/WebChat label.

## Best Possible Solution

Add a generic group-display guard shared by `buildGatewaySessionRow()` and `resolveSessionListSearchDisplayName()` so `buildGroupDisplayName()` runs only for parsed group/channel keys or entries whose `chatType` is `group` or `channel`, while direct sessions continue to fall back to explicit labels and origin labels.

## Reproduction Assessment

Yes. A focused unit reproduction can build a row for a direct main session with `chatType: "direct"`, `channel: "telegram"`, and `origin.label: "openclaw-tui"`; current main will choose the group display-name path before the origin label.

## Solution Assessment

Yes, the proposed guard is the narrowest maintainable fix. Hard-coding main sessions as `Main Session`, as in adjacent #44561, would not preserve the reported `openclaw-tui` label and would change broader display semantics.
