## What This Changes

The issue asks for direct TUI/Telegram sessions to keep their direct session display name instead of being formatted as a Telegram group label after Telegram activity.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/79040 or an equivalent shared group/channel guard with regression coverage, then close this issue from merged fix proof.

## Reproduction Assessment

Yes, source-level. A direct entry with `chatType: "direct"`, sticky `channel: "telegram"`, no explicit `displayName`, and `origin.label` still reaches `buildGroupDisplayName()` before the direct-label fallback on current main.

## Solution Assessment

Yes. The maintainable fix is a shared group/channel guard at both gateway display-name call sites so group labels stay intact while direct sessions fall through to explicit or origin labels.
