## What This Changes

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/79040 or an equivalent shared group/channel guard at both display-name call sites with regression coverage, then close this issue from merged fix proof.

## Reproduction Assessment

Yes, source-level. A direct entry with chatType direct, sticky channel telegram, no explicit displayName, and origin.label still reaches buildGroupDisplayName before the direct-label fallback on current main.

## Solution Assessment

Yes. The narrow maintainable fix is a shared group/channel guard at both gateway display-name call sites, preserving group labels while letting direct sessions fall through to explicit or origin labels.
