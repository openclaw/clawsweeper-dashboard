## Summary

Keep open. Current main still has the reported display-name path at both session-row and session-list search call sites, and a second user independently confirmed the same direct Telegram/Control UI symptom on May 4, 2026.

## What This Changes

Direct TUI/WebChat and Telegram sessions should keep their direct-session display label instead of being formatted as group-style `telegram:g-...` labels after Telegram activity.

## Best Possible Solution

Add a shared group-session guard for display-name resolution so `buildGroupDisplayName()` runs only for parsed group/channel keys or entries with `chatType` of `group` or `channel`, then cover both row and search display behavior with regression tests.

## Reproduction Assessment

Yes. A focused unit reproduction can build/list a direct session entry with `chatType: "direct"`, `channel: "telegram"`, and an `origin.label` such as `openclaw-tui`; current source will take the group-display branch before the direct label fallback.

## Solution Assessment

Yes. Guarding the group-display fallback at both current call sites is the narrowest maintainable fix; the closed #44561 approach of hard-coding main session labels would not preserve the direct-surface label requested here.
