## Summary

Keep open: current main and the latest release still have the unguarded group display-name fallback, and the linked fix at https://github.com/openclaw/openclaw/pull/79040 is open rather than merged.

## What This Changes

The issue requests that direct TUI/Telegram sessions keep the TUI-facing display name instead of falling through to group-style Telegram formatting after a Telegram interaction.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79040 or an equivalent narrow fix that guards group-style labels at both gateway display-name call sites while preserving direct-session labels.

## Reproduction Assessment

Yes. A direct session entry with `channel: "telegram"`, no explicit `displayName`, and `origin.label: "openclaw-tui"` reaches `buildGroupDisplayName()` before the label fallbacks in current main.

## Solution Assessment

Yes. The narrow maintainable solution is to guard group-style display-name formatting behind group/channel session classification at both call sites, which the linked open PR targets.
