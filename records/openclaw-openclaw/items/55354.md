## What This Changes

The issue asks OpenClaw to keep direct TUI/Telegram session labels stable by skipping group display-name generation unless the session is actually a group or channel.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/79040 or an equivalent shared group/channel guard at both gateway display-name call sites with regression coverage, then close this issue from merged fix proof.

## Reproduction Assessment

Yes, at source level. A direct entry with `chatType: "direct"`, sticky `channel: "telegram"`, no explicit displayName, and an origin label still reaches `buildGroupDisplayName()` before the direct-label fallback on current main.

## Solution Assessment

Yes. The narrow maintainable fix is a shared group/channel guard at both display-name call sites, preserving group labels while allowing direct sessions to fall through to explicit or origin labels.
