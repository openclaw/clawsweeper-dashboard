## Summary

Keep open. Current main and the latest release still have a source-level path where queued follow-up tool events return before `onToolStart` when verbose progress is off, while Discord's `onToolStart` path is what updates tool-phase status reactions; open fix PRs exist but none has landed.

## What This Changes

Discord users report that intermediate tool/skill status reaction emojis do not appear during multi-step Discord processing with default verbose settings.

## Best Possible Solution

Review and land one canonical narrow fix that lets Discord status reactions receive tool-start lifecycle events with verbose off while keeping verbose text summaries suppressed, then verify with real Discord proof.

## Reproduction Assessment

Yes at source level: current main returns before forwarding queued follow-up tool events to `onToolStart` when verbose progress is off, and existing tests assert that suppression. I did not run a live Discord reproduction in this read-only review.

## Solution Assessment

Yes, the best fix shape is narrow: decouple reaction lifecycle updates from verbose text/tool-summary delivery gates. Current main is not fixed yet, and the open PRs should be reviewed or consolidated before closing the issue.
