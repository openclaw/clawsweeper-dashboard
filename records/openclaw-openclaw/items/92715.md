Keep open. Current main still ties Discord tool-start reaction updates to the verbose/tool-summary progress gate in supported quiet configurations, so the reported missing intermediate Discord status reactions remain actionable.

## What This Changes

Discord should update the visible status reaction during tool and skill execution instead of staying on queued or stall indicators until final response generation.

## Best Possible Solution

Decouple Discord lifecycle status reactions from verbose/tool-summary text delivery so reaction updates still flow in quiet sessions and with Discord preview streaming disabled.

## Reproduction Assessment

Yes, at source level but not from a live Discord run. With status reactions enabled, verbose off, and Discord draft streaming disabled or inactive, onToolStart is filtered before Discord can call statusReactions.setTool.

## Solution Assessment

Yes, the requested behavior is the right target. The maintainable fix is a narrow lifecycle signal or callback option for reaction-only status updates, separate from verbose text progress delivery.
