## Summary

Keep open: current main and v2026.6.6 still record cron-isolated usage only into cron telemetry/session state while the sibling auto-reply path emits `model.usage`; the open linked fix PR should remain the implementation path until it lands or is replaced.

## What This Changes

Cron-isolated agent runs should emit `model.usage` diagnostic events for nonzero model usage so diagnostics exporters include scheduled spend.

## Best Possible Solution

Review, repair, and land https://github.com/openclaw/openclaw/pull/92347 or a replacement that emits the canonical cron `model.usage` event with regression coverage and real subscriber/exporter proof, then close this issue as implemented.
