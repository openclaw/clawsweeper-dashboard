## Summary

Keep open: current main still has a plausible PATH-dependent discovery gap, but this PR is not merge-ready because it lacks real behavior proof and changes an exec-scoped PATH setting into daemon-wide PATH mutation.

## What This Changes

The PR prepends `tools.exec.pathPrepend` entries to `deps.env.PATH` during config loading so PATH-based binary discovery can see configured exec paths.

## Best Possible Solution

Define an explicit effective PATH for binary discovery, or intentionally document and harden daemon-wide PATH semantics with casing-safe helpers, owned-entry cleanup, focused tests, and maintainer security acceptance.

## Reproduction Assessment

Yes, by source inspection: current main's skill and media binary probes read `process.env.PATH`, while `tools.exec.pathPrepend` is applied only to exec environments. I did not run a live macOS app proof in this read-only review.

## Solution Assessment

No. Direct global PATH mutation addresses the symptom, but the safer fix is an explicit discovery PATH contract or a documented daemon-wide PATH contract with reload cleanup, casing-safe helpers, and security acceptance.
