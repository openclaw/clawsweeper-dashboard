## Summary

Keep this PR open: it targets a real PATH discovery mismatch, but the proposed implementation is not safe to merge as-is because it mutates global PATH one-way, bypasses existing PATH-key casing helpers, and changes an exec-scoped config contract without maintainer/security acceptance.

## What This Changes

The PR prepends `tools.exec.pathPrepend` entries into `deps.env.PATH` during config load so `hasBinary()`-style checks see those directories outside exec runs.

## Best Possible Solution

Define a documented effective PATH contract for binary discovery, then implement it through shared helpers with reload cleanup, casing coverage, docs, and changelog.

## Reproduction Assessment

Yes by source inspection: current main loads config without merging `pathPrepend` into `deps.env`, while `hasBinary()` and media-understanding probes read `process.env.PATH`. A binary only in a `pathPrepend` directory remains invisible before this PR.

## Solution Assessment

No. The symptom is real, but this patch bypasses existing PATH helper semantics, leaks removed entries across reloads, and changes security/docs scope without an explicit contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Track owned PATH prepends across reloads:** `src/config/io.ts:1839-1846`
  - body: This mutates `deps.env.PATH` by adding the current config entries but never records which entries it owns. If a config reload removes or changes `tools.exec.pathPrepend`, the old directories remain process-wide until restart, so skill status and other binary checks still see removed paths.
  - confidence: 0.89
- **[P2] Reuse the PATH helper to preserve key casing:** `src/config/io.ts:1840-1845`
  - body: This reads and writes `deps.env.PATH` directly. Existing env objects can use `Path` or another PATH key casing, and the shared helper deliberately preserves that key; creating a separate `PATH` key can miss the active PATH value or duplicate it.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/config/io.ts:1835-1837`
  - body: This is a user-facing config behavior change because `tools.exec.pathPrepend` would affect skill/status and media binary detection. Repo policy requires a `CHANGELOG.md` entry before merging user-facing fix, feat, or perf work.
  - confidence: 0.78
