## Summary

Keep this PR open but do not merge it as-is: the reported mismatch is source-reproducible, but the patch widens daemon-wide PATH behavior, lacks real after-fix proof, and still has concrete reload/casing correctness defects.

## What This Changes

The PR prepends `tools.exec.pathPrepend` entries into the config-load environment PATH so `hasBinary()`-backed checks can see the same configured directories as exec runs.

## Best Possible Solution

Define an explicit binary-discovery contract: either pass a config-derived effective PATH into skill checks, or implement daemon-wide PATH mutation with casing-safe helpers, owned-entry cleanup, docs, tests, and security acceptance.

## Reproduction Assessment

Yes, by source inspection: configure `tools.exec.pathPrepend` with a directory absent from `process.env.PATH`, then run a `hasBinary()`-backed skill/status check. Current main keeps `pathPrepend` scoped to exec environments while `hasBinary()` reads `process.env.PATH`.

## Solution Assessment

No. Direct global PATH mutation addresses the symptom, but this implementation needs reload cleanup, PATH key casing support, tests/docs, real behavior proof, and explicit acceptance of the widened command-resolution surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Track owned PATH entries across reloads:** `src/config/io.ts:1845`
  - body: This only prepends missing directories and never records which entries came from the current config. If `loadConfig()` runs after `tools.exec.pathPrepend` changes or clears, old directories stay in PATH, so `hasBinary()` can keep treating removed tools as available until restart.
  - confidence: 0.89
- **[P2] Preserve the active PATH key casing:** `src/config/io.ts:1841-1845`
  - body: The patch reads and writes `deps.env.PATH` directly. Env objects can carry `Path` or another PATH key casing, so this can create a second key or miss the active PATH value instead of using the existing PATH helper.
  - confidence: 0.84
