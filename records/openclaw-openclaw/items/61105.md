## Summary

Keep open. The reported symptom is source-reproducible on current main, but this PR is not merge-ready because it widens an exec-scoped PATH setting into process-wide command resolution and still has reload and PATH-key handling bugs.

## What This Changes

The PR adds config-load logic in `src/config/io.ts` that prepends trimmed `tools.exec.pathPrepend` entries into `deps.env.PATH` so process-wide binary checks can see them.

## Best Possible Solution

Define the contract deliberately: either pass a config-derived effective PATH into binary discovery, or implement global PATH mutation with owned-entry tracking, casing-safe helper reuse, tests, docs, and changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `tools.exec.pathPrepend` with a directory absent from `process.env.PATH`, then run a `hasBinary()`-backed skill/status check; current main loads the config without merging that setting while `hasBinary()` scans only global PATH.

## Solution Assessment

No. The PR addresses the immediate symptom, but direct global PATH mutation is not the best current solution without owned-entry cleanup on reload, existing PATH-key casing support, explicit security acceptance, and docs/tests for the widened contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Track and remove owned PATH entries on reload:** `src/config/io.ts:1840-1846`
  - body: This prepends configured directories into `deps.env.PATH` but never records which entries came from the current config. When a live config update removes or changes `tools.exec.pathPrepend`, old directories remain visible to `hasBinary()` and later PATH lookups until process restart.
  - confidence: 0.89
- **[P2] Preserve the existing PATH key casing:** `src/config/io.ts:1841-1845`
  - body: The patch reads and writes `deps.env.PATH` directly instead of using the existing PATH helper. Env objects can carry `Path` or another PATH key casing, so this can create a second key or miss the active PATH value instead of updating it.
  - confidence: 0.84
