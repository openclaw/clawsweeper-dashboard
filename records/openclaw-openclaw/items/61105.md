## Summary

Keep open: the current main code still does not apply `tools.exec.pathPrepend` to global binary discovery, but this PR’s daemon-wide PATH mutation has reload/casing bugs, missing real behavior proof, and security-sensitive config semantics that need maintainer acceptance.

## What This Changes

The PR prepends `tools.exec.pathPrepend` entries into `deps.env.PATH` during config loading so `hasBinary()`-based checks see exec-configured paths.

## Best Possible Solution

Define an explicit effective-PATH contract for binary discovery, then implement it with existing PATH helpers, owned-entry cleanup or a non-mutating discovery path, focused tests, docs, and security acceptance.

## Reproduction Assessment

Yes, by source inspection: configure `tools.exec.pathPrepend` with a directory absent from `process.env.PATH`, then run a `hasBinary()`-backed skill/status check. Current main keeps `pathPrepend` scoped to exec environments while `hasBinary()` reads process PATH.

## Solution Assessment

No. Direct global PATH mutation addresses the symptom, but the maintainable solution needs an explicit effective-PATH contract plus casing-safe helpers, reload cleanup, tests/docs, and security acceptance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Track owned PATH entries across reloads:** `src/config/io.ts:1845`
  - body: This only prepends missing directories and never records which entries came from the current config. If `loadConfig()` runs after `tools.exec.pathPrepend` changes or is cleared, old directories remain in `deps.env.PATH`, so `hasBinary()` can keep finding removed tools until restart.
  - confidence: 0.89
- **[P2] Use the active PATH key casing:** `src/config/io.ts:1841-1845`
  - body: The patch reads and writes `deps.env.PATH` directly. Env maps can use `Path` or another casing, and this bypasses the existing PATH helper, creating or updating the wrong key for binary discovery and subprocess environments.
  - confidence: 0.84
