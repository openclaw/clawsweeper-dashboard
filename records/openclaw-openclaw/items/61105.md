## Summary

Keep open: current main still does not make `tools.exec.pathPrepend` visible to process-wide `hasBinary()` checks, but this branch has correctness and security-contract blockers plus missing real behavior proof before it is mergeable.

## What This Changes

The PR prepends configured `tools.exec.pathPrepend` directories into `deps.env.PATH` during config load so `hasBinary()`-backed checks can find the same binaries as exec runs.

## Best Possible Solution

Define a deliberate binary-discovery contract: either pass a config-derived effective PATH into skill checks, or implement process-wide PATH mutation with owned-entry cleanup, casing-safe helpers, docs, tests, and security acceptance.

## Reproduction Assessment

Yes, by source inspection: configure `tools.exec.pathPrepend` with a directory absent from `process.env.PATH`, then run a `hasBinary()`-backed skill/status check. Current main keeps `pathPrepend` scoped to exec environments while `hasBinary()` reads process PATH.

## Solution Assessment

No. Direct global PATH mutation addresses the symptom, but this patch needs owned-entry reload semantics, PATH key casing support, tests/docs, real behavior proof, and explicit acceptance of the widened command-resolution surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Track owned PATH entries across reloads:** `src/config/io.ts:1845`
  - body: This prepends configured directories but never records which entries came from the current config. If an in-process config reload changes or clears `tools.exec.pathPrepend`, old directories remain visible to `hasBinary()` and later PATH-based lookups until restart.
  - confidence: 0.89
- **[P2] Preserve the active PATH key casing:** `src/config/io.ts:1841-1845`
  - body: The patch reads and writes `deps.env.PATH` directly. Env objects can carry `Path` or another PATH key casing, so this can create a second key or miss the active PATH value instead of using the existing PATH helper.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The patch expands an exec-scoped PATH setting into process-wide command resolution and needs explicit security acceptance before merge.

Concerns:

- **[medium] Global PATH mutation broadens command resolution:** `src/config/io.ts:1845`
  - body: After this change, directories configured for exec runs also affect daemon-wide binary discovery and any later PATH-based subprocess resolution. That may be intended, but it widens the binary-hijack surface beyond the current documented contract.
  - confidence: 0.84
