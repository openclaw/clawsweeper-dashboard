## Summary

Keep open: the mismatch is source-plausible, but this patch expands an exec-scoped PATH setting into daemon-wide PATH with missing real proof, a current merge conflict, and compatibility/security risks.

## What This Changes

The branch prepends `tools.exec.pathPrepend` entries into `deps.env.PATH` during config loading so `hasBinary()` and skill readiness checks can see those configured paths.

## Best Possible Solution

Define an explicit effective PATH for binary discovery, or intentionally document and harden daemon-wide PATH semantics with casing-safe helpers, owned-entry cleanup, focused tests, and maintainer security acceptance.

## Reproduction Assessment

Yes, by source inspection: `hasBinary()` reads `process.env.PATH`, while current `tools.exec.pathPrepend` is applied only to exec environments. I did not run a live macOS app proof in this read-only review.

## Solution Assessment

No. Direct global PATH mutation addresses the symptom, but the safer direction is an explicit discovery PATH contract or a documented daemon-wide PATH contract with reload cleanup, casing-safe helpers, tests, and security acceptance.
