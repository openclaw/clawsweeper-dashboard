## What This Changes

The issue asks OpenClaw to make `typingMode: "instant"` start typing for queued/steered follow-up turns and honor explicit typing mode for Telegram room events.

## Best Possible Solution

Fix both gaps in one narrow change: call `signalRunStart()` for queued follow-up runs and let explicit typing configuration override Telegram room-event suppression while preserving quiet ambient defaults when no mode is configured.

## Reproduction Assessment

Yes. Current main is source-reproducible: queued follow-up runs do not call `signalRunStart()`, while Telegram room events still set `suppressTyping` before the message-tool-only typing mode can take effect.

## Solution Assessment

Yes in direction. Reusing the existing typing signaler and configured typing-mode precedence is the narrowest maintainable fix; the open linked PR should be reviewed as the active implementation candidate.
