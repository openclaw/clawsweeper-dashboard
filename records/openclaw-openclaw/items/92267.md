## What This Changes

The issue asks for `typingMode: "instant"` to work for queued/steered follow-up turns and Telegram `room_event` turns using message-tool-only visible replies.

## Best Possible Solution

Land or revise the linked narrow fix so queued follow-up turns start instant typing through the existing signaler and Telegram honors explicit typingMode for room events while preserving quiet ambient defaults when typingMode is unset.

## Reproduction Assessment

Yes, at source level with high confidence: current main still lacks a follow-up run-start typing signal and Telegram still suppresses typing for every room_event before typingMode can apply. I did not run a live Telegram scenario in this read-only review.

## Solution Assessment

Yes in direction: reusing the established typing signal lifecycle for follow-up runs and respecting explicit typingMode before suppressing ambient room-event typing is the narrowest maintainable path. Adding a new room-event typing knob would duplicate the existing typingMode setting unless maintainers deliberately want a separate product control.
