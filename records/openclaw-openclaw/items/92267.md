## Best Possible Solution

Land the linked focused fix or an equivalent maintainer-approved change that starts instant typing at follow-up run start and lets explicit typing-mode config override quiet room-event defaults.

## Reproduction Assessment

Yes, at source level: current main shows the follow-up runner never starts instant typing at run start, while fresh turns do, and Telegram room events pass `suppressTyping` before configured or message-tool typing behavior can win. I did not run a live Telegram scenario in this read-only review.

## Solution Assessment

Yes in direction: reusing the existing run-start typing signal and configured typing-mode precedence is the narrowest maintainable fix. Adding a new room-event typing knob would be a broader product/config choice and is not required to satisfy the documented behavior.
