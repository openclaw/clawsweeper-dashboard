## Summary

Both reported gaps remain on current main: queued follow-up runs omit the instant-typing start signal, and Telegram room events force typing suppression before configured mode resolution. The issue should remain open because the formally linked repair at https://github.com/openclaw/openclaw/pull/92287 is still open and owns the implementation.

## What This Changes

Fix missing instant typing for queued follow-up turns and honor explicitly configured typing for Telegram room events.

## Best Possible Solution

Start the existing typing signaler for every non-aborted follow-up model run, and keep ambient room events quiet only when the operator has not explicitly selected a typing mode, without adding another configuration knob or unnecessary public SDK surface.

## Reproduction Assessment

Yes at source level with high confidence: current main calls signalRunStart for fresh turns but not follow-up turns, and Telegram suppression takes precedence over configured typing resolution. The reporter also provides a concrete configuration and live log signature, though this review did not execute Telegram traffic.

## Solution Assessment

Yes, the linked direction is the narrowest maintainable fix because it reuses the existing typing lifecycle and typingMode contract rather than adding another room-event option. The implementation should avoid expanding the public Plugin SDK unless a real package boundary requires it.
