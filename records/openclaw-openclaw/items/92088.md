## Summary

Keep this open: current main still exits the detached managed-update helper after failed child update outcomes without starting the gateway service, and the linked closing PR is open rather than merged.

## What This Changes

The issue reports that a managed update handoff can stop the gateway service and fail before restarting it, leaving OpenClaw unresponsive.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/92111 or an equivalent maintainer-approved fix that restarts the managed gateway after post-stop update handoff failures.

## Reproduction Assessment

Yes. Current main's detached helper marks failed handoff outcomes but does not restart the managed service, and the issue includes a concrete systemd exit-127 trace where the gateway was left inactive.

## Solution Assessment

Yes. The active fix should stay in the detached handoff owner because that is the process still alive after the gateway exits; the linked PR is the current implementation candidate and should be resolved before this issue closes.
