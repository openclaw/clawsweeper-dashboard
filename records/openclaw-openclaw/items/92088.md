## Summary

Current main still has the reported outage path: the detached updater records command failure but does not restore the managed gateway. The issue should remain open because the viable, proof-backed fix is still under review in https://github.com/openclaw/openclaw/pull/92111.

## What This Changes

Ensure a managed update handoff failure cannot leave the gateway service stopped after the updater has taken it down.

## Best Possible Solution

Handle failed detached updates in the surviving handoff helper using canonical platform service identities, while preserving in-process CLI recovery and proving idempotent no-bounce behavior.

## Reproduction Assessment

Yes at source level. Current main handles detached update spawn and nonzero-exit failures without restoring the managed gateway, and the linked PR provides a real systemd before-and-after reproduction.

## Solution Assessment

Yes in principle. The detached helper is the only process guaranteed to survive this failure boundary, so best-effort service recovery belongs there with canonical identities and platform-focused tests.
