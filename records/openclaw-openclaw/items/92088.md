## What This Changes

Ensure failed managed update handoffs restore the gateway service instead of leaving it stopped.

## Best Possible Solution

The surviving handoff helper should restore the same canonical managed service only when this update displaced it, with focused systemd, launchd, and Windows regression coverage and consistent CLI recovery paths.

## Reproduction Assessment

Yes. Current `main` has deterministic detached-handoff failure exits with no recovery, and the linked PR documents a live before-fix systemd run ending with an inactive gateway unit and unreachable HTTP endpoint.

## Solution Assessment

Yes in owner placement: the detached helper is the process that survives the failed update and can restore service. The implementation should additionally prove it preserves a previously inactive service and matches canonical launchd and Windows restart behavior.
