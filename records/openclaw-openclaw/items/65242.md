- **[P1] Preserve delegated ACP parent-review handoff:** `src/tasks/task-registry.ts:1009-1026`
  - body: Current main keeps delegated ACP completions pending and queues a parent-session review event, but this branch intercepts successful ACP child-session tasks before that policy path and marks them `session_queued` after only a heartbeat wake.   - confidence: 0.9

  - body: The announce flow claims the completion before it waits for child output or sends anything. Later branches can still return `false`, skip due to empty or silent output, or fail direct delivery, but the retained claim can make fallback paths treat the run as already delivered and leave the requester with no completion message.
  - confidence: 0.9
- **[P1] Keep the gate short-circuit inside cleanup:** `src/agents/subagent-announce.ts:236-237`
  - body: When another path already claimed the completion, this early `return true` exits before the function enters its `try`/`finally`. That bypasses cleanup such as child session deletion while callers interpret the result as a successful announce with no retry path.
  - confidence: 0.88
- **[P2] Keep shadow mode observational for farewells:** `src/agents/subagent-registry-completion.ts:78-83`
  - body: `isCompletionClaimed` can return true in shadow mode because claims are stored, so this branch flips `sendFarewell` to false even though shadow rollout is supposed to log would-be blocks without changing user-visible behavior.
  - confidence: 0.86
- **[P1] Preserve parent-channel lookup for child bindings:** `extensions/discord/src/monitor/thread-bindings.manager.ts:641-642`
