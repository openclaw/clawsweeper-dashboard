## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Claim announce delivery only after it can finish safely:** `src/agents/subagent-announce.ts:231-237`
  - body: This claims the completion before the announce flow enters its `try/finally`. The same function can still return `false` while a child is active, descendants are pending, requester fallback is missing, or delivery finishes with `didAnnounce=false`; once the claim is retained, task/farewell fallback paths can treat the run as already delivered and leave the user with no completion message or cleanup retry.
  - confidence: 0.93
- **[P2] Keep shadow mode observational for farewells:** `src/agents/subagent-registry-completion.ts:78-83`
  - body: `OPENCLAW_COMPLETION_GATE=shadow` records the first claim and only allows later callers after logging, but `isCompletionClaimed()` still returns true for that shadow claim. This branch therefore flips `sendFarewell` to `false` in shadow mode, changing user-visible delivery during a mode that is supposed to observe only.
  - confidence: 0.91
- **[P1] Preserve Discord thread parent resolution:** `extensions/discord/src/monitor/thread-bindings.manager.ts:641-642`
  - body: For child placement with no `parentConversationId`, `conversationId` can be an existing thread id. The previous `resolveChannelIdForBinding()` path looked up that thread and returned its parent channel, but this now passes the thread id as `channelId`, so Discord can reject creating a new thread under a thread.
  - confidence: 0.84
