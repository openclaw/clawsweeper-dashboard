## Summary

Keep open. The PR targets a real ACP/subagent completion-delivery problem that is not already solved on current main, but the branch is not mergeable as-is because it claims ownership before durable delivery and cleanup, makes shadow mode user-visible, weakens Discord parent-channel resolution, and lacks after-fix real behavior proof.

## What This Changes

The PR adds an env-flagged CompletionDeliveryGate across ACP task delivery, subagent announce and farewell paths, silent-wake handling, requester-origin routing, Discord child-thread binding, and related tests.

## Best Possible Solution

Consolidate on one durable completion ownership contract that claims only after successful delivery or releases on failure, keeps shadow mode observational, preserves channel parent resolution, and is proven with redacted live ACP/channel evidence.

## Reproduction Assessment

Yes for the PR defects by source inspection: the announce flow claims before delivery and cleanup, shadow-mode claims can suppress farewells, and Discord child binding drops parent-channel resolution. I did not run live ACP/channel proof in this read-only review.

## Solution Assessment

No. A shared gate may be a reasonable direction, but this branch claims too early, changes shadow behavior, removes an important Discord fallback, and introduces rollout policy that needs maintainer agreement.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Claim announce completion after delivery succeeds:** `src/agents/subagent-announce.ts:231-235`
  - body: The gate claim is taken before the announce flow waits for child output or delivers anything. Later branches can still return false or fail delivery, but the retained claim can make fallback paths treat the run as already delivered and leave the requester with no completion message.
  - confidence: 0.9
- **[P1] Keep the gate short-circuit inside cleanup:** `src/agents/subagent-announce.ts:237`
  - body: When another path already claimed the completion, this early return exits before the function enters its try/finally. That skips cleanup such as child session deletion while callers interpret the result as a successful announce with no retry path.
  - confidence: 0.88
- **[P2] Keep shadow mode observational for farewells:** `src/agents/subagent-registry-completion.ts:78-83`
  - body: isCompletionClaimed can return true in shadow mode because claims are stored, so this branch flips sendFarewell to false even though shadow rollout is supposed to log would-be blocks without changing user-visible behavior.
  - confidence: 0.86
- **[P1] Preserve parent-channel lookup for child bindings:** `extensions/discord/src/monitor/thread-bindings.manager.ts:641-642`
  - body: For child placement, conversationId can still be an existing thread target when parentConversationId is absent. Assigning it directly to channelId makes the binder create a child thread under the wrong channel id; keep the thread-to-parent resolution fallback.
  - confidence: 0.84
