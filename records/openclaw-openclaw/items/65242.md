## Summary

Keep open: the central gate behavior is not on current main, but this branch is not merge-ready because source inspection shows it can claim completion delivery before a successful delivery, skip announce cleanup, change shadow-mode behavior, and remove a Discord parent-channel fallback. It also still lacks real ACP/channel proof and overlaps the open completion-delivery work in https://github.com/openclaw/openclaw/pull/64416.

## What This Changes

The PR adds an env-flagged CompletionDeliveryGate, ACP child-session silent wake behavior, requester-origin routing changes, Discord child-thread binding changes, and related task/agent tests.

## Best Possible Solution

Consolidate with https://github.com/openclaw/openclaw/pull/64416 on one durable completion-delivery path that claims only after successful delivery, preserves cleanup and retry behavior, keeps shadow mode observational, retains Discord parent-channel resolution, and includes live ACP/channel proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the announce flow claims before delivery and before cleanup, the gate-conflict return exits before the try/finally, and farewell suppression can observe shadow-mode claims. I did not run live ACP/channel proof in this read-only review.

## Solution Assessment

No. A shared completion gate may be a reasonable direction, but this branch places ownership too early, changes shadow behavior, mixes in stale Discord routing changes, and needs consolidation with the overlapping completion-delivery PR.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Claim announce completion after delivery succeeds:** `src/agents/subagent-announce.ts:228-232`
  - body: The gate claim is taken before the announce flow waits for output or delivers anything. Later branches can still return false or fail delivery, but the retained claim can make fallback paths treat the run as already delivered and leave the requester with no completion message.
  - confidence: 0.9
- **[P1] Keep the gate short-circuit inside cleanup:** `src/agents/subagent-announce.ts:234`
  - body: When another path already claimed the completion, this early return exits before the function enters its try/finally. That skips cleanup such as child session deletion while callers interpret the result as a successful announce with no retry path.
  - confidence: 0.88
- **[P2] Keep shadow mode observational for farewells:** `src/agents/subagent-registry-completion.ts:80`
  - body: isCompletionClaimed can return true in shadow mode because claims are stored, so this branch flips sendFarewell to false even though shadow rollout is supposed to log would-be blocks without changing user-visible behavior.
  - confidence: 0.86
- **[P1] Preserve parent-channel lookup for child threads:** `extensions/discord/src/monitor/thread-bindings.manager.ts:641`
  - body: For child placement, conversationId can still be an existing thread target when parentConversationId is absent. Assigning it directly to channelId makes the binder create a child thread under a thread channel, which Discord rejects; keep the thread-to-parent resolution fallback.
  - confidence: 0.84
