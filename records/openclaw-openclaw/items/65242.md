## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Claim announce completion after delivery succeeds:** `src/agents/subagent-announce.ts:231-235`
  - body: The gate claim is taken before the announce flow waits for output or delivers anything. Later branches in this function can still return false or fail direct delivery, but the retained claim can make fallback paths treat the run as already delivered and leave the requester with no completion message.
  - confidence: 0.9
- **[P1] Keep gate short-circuit inside cleanup path:** `src/agents/subagent-announce.ts:237`
  - body: When another path already claimed the completion, this early return exits before the function enters its try/finally. That skips label patching and session deletion while callers interpret the result as a successful announce with no retry path.
  - confidence: 0.88
- **[P2] Keep shadow mode observational for farewells:** `src/agents/subagent-registry-completion.ts:82-83`
  - body: isCompletionClaimed can return true in shadow mode because claims are stored, so this branch flips sendFarewell to false even though shadow rollout is supposed to log would-be blocks without changing user-visible behavior.
  - confidence: 0.86
- **[P1] Preserve parent-channel lookup for child threads:** `extensions/discord/src/monitor/thread-bindings.manager.ts:641-642`
  - body: For child placement, conversationId can still be an existing thread target when parentConversationId is absent. Assigning it directly to channelId makes the binder create a child thread under a thread channel, which Discord rejects; keep the thread-to-parent resolution fallback.
  - confidence: 0.84
