## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Claim completion only after delivery succeeds:** `src/agents/subagent-announce.ts:231-237`
  - body: This claims announce_flow ownership before the function has waited for output, checked descendant state, or called deliverSubagentAnnouncement. Several later branches return false or fail delivery, but the claim remains and other paths treat the completion as already handled, so users can lose the fallback completion message instead of getting one deduplicated delivery.
  - confidence: 0.92
- **[P1] Keep blocked announces inside the cleanup path:** `src/agents/subagent-announce.ts:237`
  - body: When another path already claimed the completion, this return exits before the try/finally that patches labels and deletes cleanup:"delete" child sessions. Callers see true and finalize bookkeeping, but the child session cleanup path never runs, leaving stale session/transcript state with no retry.
  - confidence: 0.9
- **[P2] Keep shadow mode observational for farewells:** `src/agents/subagent-registry-completion.ts:82-83`
  - body: Shadow claims are still stored in the gate, and this code suppresses sendFarewell whenever a claim exists. Setting OPENCLAW_COMPLETION_GATE=shadow can therefore change user-visible completion delivery instead of only logging would-be blocks, which defeats the safe rollout mode.
  - confidence: 0.88
- **[P1] Preserve parent-channel lookup for child thread bindings:** `extensions/discord/src/monitor/thread-bindings.manager.ts:641-642`
  - body: For child placement, conversationId can be an existing Discord thread id when no parentConversationId is available. Assigning it directly to channelId makes bindTarget try to create a new thread under a thread, while the current lookup path resolves that thread to its parent channel before creation.
  - confidence: 0.86
