## What This Changes

The PR changes builtin memory keyword search to select an FTS5 `snippet()` result, clamp snippet tokens to SQLite's 64-token limit, and render that snippet instead of truncating the full chunk text.

## Best Possible Solution

Port the FTS5 snippet extraction to the current memory-core search module, preserve fallback and hybrid behavior, add a regression test with a match beyond the first 700 displayed characters, and require real CLI or tool-output proof before merge.

## Reproduction Assessment

Yes from source inspection, but not from a live run in this read-only review. Current main can create chunks larger than the 700-character display budget and still renders keyword snippets by truncating `row.text` from the chunk start.

## Solution Assessment

Unclear as submitted. SQLite `snippet()` is the right dependency-backed primitive for exact FTS MATCH snippets, but this branch must be retargeted to current memory-core code and prove fallback/hybrid behavior before it is the best mergeable fix.
