## Summary

Keep open: this is a plausible narrow bug fix, but the branch edits obsolete `src/memory/manager-search.ts` while current `main` runs keyword search from the memory-core plugin. The linked user issue was closed for the broader file-head symptom, but current main still prefix-truncates chunk text in the live MATCH path, so the PR needs retargeting plus real behavior proof before merge review.

## What This Changes

The branch changes keyword memory search to select SQLite FTS5 `snippet()` output and return it as the result snippet instead of prefix-truncating FTS chunk text.

## Best Possible Solution

Move the FTS5 `snippet()` projection into `extensions/memory-core/src/memory/manager-search.ts` for successful MATCH results, preserve LIKE fallback/full-text scoring behavior, and add a focused deep-match regression test.

## Reproduction Assessment

Yes, by source inspection. Current main can index chunks larger than the 700-character snippet budget, and the live keyword MATCH path still returns `truncateUtf16Safe(row.text, params.snippetMaxChars)` from the chunk prefix.

## Solution Assessment

No, not as currently targeted. FTS5 `snippet()` is a reasonable narrow direction for MATCH results, but this branch must be retargeted from the deleted helper to memory-core and must keep fallback rows using full text.
