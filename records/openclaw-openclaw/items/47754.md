## Summary

Keep open: the snippet problem is still source-reproducible on the active memory-core keyword path, but this branch cannot merge as-is because it edits a removed core path instead of the runtime memory plugin path.

## What This Changes

The PR changes keyword-search SQL in the old `src/memory/manager-search.ts` path to return SQLite FTS5 `snippet()` output instead of truncating full row text.

## Best Possible Solution

Retarget the FTS5 snippet projection to `extensions/memory-core/src/memory/manager-search.ts`, keep full `row.text` for scoring and fallback paths, and add focused deep-match regression coverage plus real CLI/search proof.

## Reproduction Assessment

Yes. Source inspection gives a concrete path: insert or index a long FTS row with the query term after the first `snippetMaxChars`, then the active MATCH path can return a snippet made from the row prefix rather than the match context.

## Solution Assessment

No. FTS5 `snippet()` is a reasonable primitive, but this patch applies it to a retired path; the maintainable fix must be ported to active memory-core while preserving full text for ranking.
