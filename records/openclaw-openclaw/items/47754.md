## Summary

Keep open: the FTS5 snippet direction is still a plausible fix for keyword snippets, but current main has moved the live implementation to memory-core and still prefix-truncates `row.text`; this branch targets an obsolete path and is missing real behavior proof.

## What This Changes

The PR changes keyword memory search to select SQLite FTS5 `snippet()` output and return it as the result snippet instead of truncating the matched chunk text from the beginning.

## Best Possible Solution

Move the FTS5 `snippet()` projection into `extensions/memory-core/src/memory/manager-search.ts` for successful MATCH results, preserve LIKE fallback behavior, keep the token clamp and visible ellipsis markers, and add a focused deep-match regression test.

## Reproduction Assessment

Yes, by source inspection. Current memory-core keyword search selects full FTS chunk text and maps the displayed snippet from the prefix, while the default 700-character budget can exclude a later keyword inside a larger chunk.

## Solution Assessment

No, not as currently targeted. FTS5 `snippet()` is the narrow maintainable direction for MATCH results, but this patch must move to the live memory-core helper and keep fallback rows using full text.
