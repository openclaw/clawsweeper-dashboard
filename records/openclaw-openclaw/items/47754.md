## Summary

Keep open: the FTS5 snippet idea is still a plausible bug fix, but this PR edits the retired `src/memory/manager-search.ts` path while current `main` runs the memory-core implementation, which still selects and truncates `row.text`; the PR also lacks after-fix real behavior proof.

## What This Changes

The branch changes keyword memory search to select an FTS5 `snippet()` value with a clamped ellipsis-marked token window and return it as the result snippet.

## Best Possible Solution

Port the snippet projection to the active memory-core keyword search, keep full `row.text` for fallback ranking, and add a focused deep-match regression test.

## Reproduction Assessment

Yes, from source inspection: the active MATCH path selects `text` and returns `truncateUtf16Safe(row.text, params.snippetMaxChars)`, so a late match inside a large indexed row can be hidden. I did not run a live repro because this review is read-only.

## Solution Assessment

No. FTS5 `snippet()` is directionally right, but this PR patches a retired path; the maintainable fix belongs in `extensions/memory-core/src/memory/manager-search.ts` with full text retained for scoring.
