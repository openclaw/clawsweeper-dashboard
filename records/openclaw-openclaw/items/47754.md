## Summary

Keep open: the underlying memory-search snippet problem is still source-reproducible, but this PR cannot merge as written because it changes a retired path and lacks contributor real behavior proof.

## What This Changes

The PR changes keyword memory search to use SQLite FTS5 `snippet()` output instead of returning the beginning of each matched chunk.

## Best Possible Solution

Port the snippet projection to the active memory-core search path, keep full text for ranking and fallback paths, add a late-match regression test, and attach redacted real CLI/search output.

## Reproduction Assessment

Yes, from source inspection: the active MATCH path returns `truncateUtf16Safe(row.text, 700)` while default chunks can be about 400 tokens, so a late term within one chunk can be omitted from the visible snippet. I did not run a live CLI repro in this read-only review.

## Solution Assessment

No. FTS5 `snippet()` is a reasonable direction, but this patch applies it to a retired file; the maintainable fix belongs in `extensions/memory-core/src/memory/manager-search.ts` with full text retained separately.
