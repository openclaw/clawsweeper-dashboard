## Summary

Keep open. The fix direction is plausible, but this branch targets the retired `src/memory` path while current `main` runs memory search from `extensions/memory-core`, so it needs a rebase plus real behavior proof before merge.

## What This Changes

The PR changes keyword memory search to return SQLite FTS5 `snippet()` output centered on the match instead of truncating chunk text from the beginning.

## Best Possible Solution

Port the `snippet()` extraction to `extensions/memory-core/src/memory/manager-search.ts`, keep full row text where current scoring needs it, and add a focused regression test for a deep match within one chunk.

## Reproduction Assessment

Yes, from source inspection. Current keyword search still returns `truncateUtf16Safe(row.text, ...)`, and default chunks can be larger than the 700-character snippet cap, so a deep match inside one FTS row can be hidden without `snippet()`.

## Solution Assessment

No. The approach is directionally right, but the submitted branch is stale against current `main`; the maintainable fix is to rebase the same idea onto the active memory-core search path and cover it with a regression test.
