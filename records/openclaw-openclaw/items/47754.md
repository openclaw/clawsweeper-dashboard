## Summary

Keep open: the bug is still visible in current main, but this branch targets the old memory search path and needs to be ported to the current memory-core plugin before it can fix shipped behavior.

## What This Changes

The branch updates `src/memory/manager-search.ts` to use SQLite FTS5 `snippet()` for keyword results, clamp snippet token count, and display the snippet instead of truncating full chunk text.

## Best Possible Solution

Port the `snippet()` approach into `extensions/memory-core/src/memory/manager-search.ts` with focused regression coverage for deep keyword matches, while preserving the existing source/model filtering and fallback search behavior.

## Reproduction Assessment

Yes. By source inspection, current `searchKeyword` returns `truncateUtf16Safe(row.text, params.snippetMaxChars)` for keyword matches, so a match after the first 700 characters will be found but not shown in the visible snippet.

## Solution Assessment

No, not as submitted. The FTS5 `snippet()` direction is the narrow maintainable fix, but this branch applies it to the obsolete `src/memory` path instead of the current memory-core plugin implementation.
