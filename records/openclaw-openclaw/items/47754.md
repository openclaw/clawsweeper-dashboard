## Summary

This PR should stay open as a useful bug-fix direction, but it is not mergeable as-is because it edits the obsolete `src/memory` implementation while current main’s live memory-core keyword search still truncates chunk text from the beginning. It also needs after-fix real behavior proof before merge.

## What This Changes

The PR changes keyword memory search to return SQLite FTS5 `snippet()` output instead of truncating the full matched chunk text from the beginning.

## Best Possible Solution

Move the FTS5 `snippet()` change to `extensions/memory-core/src/memory/manager-search.ts`, clamp token count to SQLite’s 1..64 range, keep visible ellipsis markers, preserve LIKE fallback behavior, and add a focused deep-match regression test.

## Reproduction Assessment

Yes, by source inspection. With default chunking, a keyword can appear after the first 700 characters of a single chunk, and current keyword search still maps the result snippet from the beginning of `row.text`.

## Solution Assessment

No, not as currently targeted. FTS5 `snippet()` is the narrow maintainable direction for MATCH results, but the patch must be retargeted to the live memory-core helper and preserve current fallback behavior.
