## Summary

Keep open: the underlying snippet bug is still source-reproducible in the active memory-core keyword path, but this PR edits a removed `src/memory` path, conflicts with current `main`, and lacks after-fix real behavior proof.

## What This Changes

The PR changes the old `src/memory/manager-search.ts` keyword query to return SQLite FTS5 `snippet()` output instead of truncating the selected row text.

## Best Possible Solution

Port the snippet projection to the active memory-core MATCH query, keep full row text for scoring and fallbacks, and add a focused deep-match regression test plus real CLI/search proof.

## Reproduction Assessment

Yes, from source inspection rather than a live run: default chunks can exceed the 700-character display cap, the CLI prints `result.snippet`, and the active MATCH path still truncates `row.text` from the beginning.

## Solution Assessment

No. FTS5 `snippet()` is a reasonable direction, but this branch applies it to a removed path; the maintainable fix belongs in `extensions/memory-core/src/memory/manager-search.ts` with full text retained separately.
