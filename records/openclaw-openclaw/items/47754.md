## What This Changes

The PR changes memory keyword search SQL to return SQLite FTS5 `snippet()` output instead of truncating the beginning of each chunk text.

## Best Possible Solution

Port the snippet projection to `extensions/memory-core/src/memory/manager-search.ts`, keep full text for fallback ranking, add a deep keyword-match regression test, and include redacted real CLI/search output as proof.

## Reproduction Assessment

Yes, from source inspection: active keyword search still returns `truncateUtf16Safe(row.text, 700)` while default chunks can be larger than that, so a late match inside one FTS row can be omitted from the displayed snippet. I did not run a live CLI repro in this read-only review.

## Solution Assessment

No. FTS5 `snippet()` is a reasonable fix direction, but this PR applies it to a retired path and would need to be retargeted while preserving full text for active fallback scoring.
