## Summary

Keep open: current main contains a useful partial fix for structured `server_error` cron retries, but it still does not implement the central fast-fail/skip/notification behavior for plain provider `api_error: Internal server error` outages or the adjacent stream-stall timeout budget problem.

The issue asks OpenClaw cron agent jobs to fast-fail sustained LLM provider outages, retry with backoff, and notify configured delivery targets instead of burning the full cron timeout window.

## Best Possible Solution

Keep this open and split the repair: decide the plain provider-500 classification policy for this issue, while tracking stream-stall/no-first-chunk budgeting in the adjacent open reports.

## Reproduction Assessment
