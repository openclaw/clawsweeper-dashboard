## Best Possible Solution

Keep the shipped structured `server_error` cron retry support, then define an owner-reviewed provider-outage fast-fail policy with live proof and regression coverage for Anthropic-style errors, shared cooldown/skipping semantics, recurring cron catch-up, and failure notification.
