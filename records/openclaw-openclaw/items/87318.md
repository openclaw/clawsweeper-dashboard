## Summary

## What This Changes

The issue requests a supported Amazon Bedrock contract for Claude Haiku 4.5 inference profile routing, either through `params.modelId`, full inference-profile ARN model refs, or an alias mapping.

## Best Possible Solution

Define one Bedrock-owned profile identity contract, preferably exact inference profile IDs/ARNs or a deliberate alias map, document it, and leave the generic API-registry miss to the separate canonical issue.

## Reproduction Assessment

Yes for the source-level `params.modelId` non-override path: current code merges params but still sends `model.id` to Bedrock. A live current-main AWS cron reproduction still needs credentials and should not be claimed from this review.

## Solution Assessment

Unclear. Exact inference profile IDs/ARNs align with the AWS and current discovery model better than a generic `params.modelId` identity override, but maintainers need to choose the supported public contract.
