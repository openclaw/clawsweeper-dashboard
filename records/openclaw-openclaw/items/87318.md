## Summary

Keep open: current main and the latest release still treat `agents.defaults.models[*].params` as request-shaping data rather than a Bedrock model identity override, and the Bedrock plugin only consumes `serviceTier` from `extraParams`. Exact inference-profile discovery exists, but it does not cover the reported Haiku 4.5 shorthand/override contract; the later `bedrock-converse-stream` registry miss is separately tracked at https://github.com/openclaw/openclaw/issues/88707.

## What This Changes

The issue requests Amazon Bedrock support for Claude Haiku 4.5 inference-profile routing, either by honoring `params.modelId`, accepting full ARN model refs, or adding a shorthand alias.

## Best Possible Solution

Define and document one provider-owned Bedrock profile contract in the Bedrock plugin, preferably exact inference profile IDs/ARNs or a deliberate alias map, while leaving the generic registry-miss repair to https://github.com/openclaw/openclaw/issues/88707.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection supports the `params.modelId` non-override path, and the reporter supplied cron/runtime logs showing Bedrock selection then failure in their environment.

## Solution Assessment

Unclear. Exact inference-profile IDs are already the documented direction, while making `params.modelId` override request identity would create a new provider config contract that maintainers should choose deliberately.
