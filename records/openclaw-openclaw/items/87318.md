## Summary

Keep open: current main and v2026.6.8 still do not support the reported Bedrock Haiku 4.5 shorthand, full-ARN model ref, or `params.modelId` identity override, while the related Bedrock registry and stream-abort reports track distinct failures.

## What This Changes

The issue asks the Amazon Bedrock provider to route Haiku 4.5 through a supported inference-profile identity instead of ignoring `params.modelId` or falling back away from Bedrock.

## Best Possible Solution

Define one Bedrock-owned profile identity contract in the Amazon Bedrock plugin, then document and test that exact ref shape for cron and normal agent runs.

## Reproduction Assessment

No live reviewer reproduction was run; source inspection confirms `params.modelId` cannot currently override the Bedrock request identity, and reporter-provided cron logs show the affected provider/model selection path.

## Solution Assessment

Unclear: exact discovered profile IDs are the existing documented path, while ARN model refs, shorthand aliases, or `params.modelId` identity overrides each need an explicit maintainer product/API decision.
