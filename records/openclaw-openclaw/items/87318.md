## Summary

Keep open: current main still merges per-model params but does not use `params.modelId` as the Bedrock Converse `modelId`, and discovery exposes exact inference profile IDs rather than the shorthand Haiku 4.5 alias in the report. ## What This Changes

Issue #87318 requests Amazon Bedrock Haiku 4.5 inference-profile routing via a model-id override, full ARN/profile ref, or documented alias for cron-selected `amazon-bedrock/...` models.

## Best Possible Solution

Keep Bedrock ownership in the amazon-bedrock plugin and choose one documented profile contract, then add focused cron/model-routing coverage plus Bedrock docs; leave stream-abort handling to #87876 and registry-miss repair to #88707.

## Reproduction Assessment

Yes for the source-level claim that `params.modelId` is ignored: current main merges the param but never maps it to the Bedrock command model id; no live AWS current-main repro was run here.

## Solution Assessment

Unclear: `params.modelId` would be an escape-hatch config contract, while exact profile refs or documented aliases may be cleaner and need maintainer selection.
