## Summary

## What This Changes

The PR changes the OpenAI Responses payload policy and regression tests so explicit server compaction can emit `context_management` on known native Responses routes even when `store` is false.

## Best Possible Solution

Land this or an equivalent narrow payload-policy fix after redacted native ChatGPT/Codex route proof or explicit maintainer/provider acceptance; if the route is unsupported, retarget to a clear unsupported-config diagnostic.

## Reproduction Assessment

Yes for the source-level payload gap: current main still gates server compaction on `explicitStore === true`, while the ChatGPT/Codex sender builds `store: false` and applies the payload wrapper afterward. No accepted live native ChatGPT/Codex endpoint run was established in this review.

## Solution Assessment

Unclear until maintainers accept the native ChatGPT/Codex Responses contract: the patch is the narrowest local code fix if that route supports `context_management` with `store: false`, but a diagnostic is safer if the backend rejects that field combination.
