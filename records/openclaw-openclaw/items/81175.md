## Summary

Keep open. Current source still corroborates the two high-priority beta failures, and an open draft PR only covers part of the Codex module-resolution problem while the OpenAI tool-schema validation failure remains unresolved.

## What This Changes

The issue reports beta upgrade failures where `openai/gpt-*` through the Codex runtime fails module resolution and the legacy `openai-codex/gpt-*` workaround sends an invalid OpenAI tool schema.

## Best Possible Solution

Land or adapt the focused Codex module-resolution fix, add a separate OpenAI-facing regression that repairs or rejects array schemas without `items`, and split lower-priority cron/doctor diagnostic follow-ups if they remain after the primary route is healthy.

## Reproduction Assessment

No, not as a full live current-main reproduction. The module-resolution failure is source-reproducible from the npm-installed Codex alias gap, while the OpenAI schema failure is only partially source-corroborated because the sanitized memory tool schema is not in the repo.

## Solution Assessment

No single current or proposed change is the best complete solution. The best path is to review the active Codex runtime PR for the module failure, then handle the schema validation and lower-priority diagnostics as narrower follow-ups.
