## Summary

Keep open. Current main still has the reported overflow recovery shape: generic reserve-floor guidance, bounded compaction retry and reset fallback, default bootstrap reinjection after compaction, and an empty-session truncation branch.

## What This Changes

The issue asks OpenClaw to make context-overflow retry recovery budget-aware so high reserve floors do not still lead to repeated hard resets and bootstrap reinjection loops.

## Best Possible Solution

Define a shared overflow recovery contract that distinguishes model context-window misconfiguration from reserve-floor tuning, avoids reinflating bootstrap context during recovery, and uses a minimal-context or clearly diagnosed fallback before hard reset.

## Reproduction Assessment

Yes, at source level: current main still supports the bounded overflow retries, optional truncation, default bootstrap/post-compaction reinjection, and hard-reset fallback. No, for the exact Basecamp/Voltti session key because that local transcript is not available here.

## Solution Assessment

No. Current main includes useful partial mitigations, but it does not fully solve the misleading reserve-floor guidance, prompt/bootstrap reinjection during recovery, empty-session truncation handling, or repeated reset fallback.
