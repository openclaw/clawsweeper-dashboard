## Summary

Keep open. Current main has mitigations for context-window pressure, but the reported recovery shape still exists: generic reserve-floor guidance, hard reset after embedded overflow, three compaction retries, default per-attempt bootstrap resolution, default post-compaction reinjection, and the empty-session truncation diagnostic.

## What This Changes

The issue asks OpenClaw to investigate repeated context-overflow hard resets where retry recovery appears to rebuild bootstrap context and gives reserve-floor guidance even when headroom is already high.

## Best Possible Solution

Define and implement a budget-aware overflow recovery contract that distinguishes model context-window misconfiguration from reserve-floor tuning, suppresses or shrinks reinjected bootstrap context during recovery, and only hard-resets after a minimal-context or clearly diagnosed fallback path is exhausted.

## Reproduction Assessment

Yes for the source-level recovery shape: current main can still take embedded context-overflow meta errors through bounded compaction retries, optional truncation, and hard-reset fallback. The exact Basecamp/Voltti session key cannot be reproduced from the repository alone.

## Solution Assessment

No. Current main has useful partial mitigations, but they do not fully solve the reported reset loop, misleading fallback guidance, per-attempt bootstrap reinjection, or empty-session truncation branch.
