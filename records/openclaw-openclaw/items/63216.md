## Summary

Keep open. Current main has partial mitigations for persisted-message continuation and repeated identical tool outcomes after compaction, but the reported recovery shape still exists across generic reserve-floor guidance, bounded retry/reset fallback, default bootstrap/post-compaction reinjection, and the empty-session truncation branch.

## What This Changes

The issue asks OpenClaw to harden context-overflow recovery when high reserve floors still lead to repeated compaction retries, bootstrap reinjection, empty-session truncation diagnostics, and hard session resets.

## Best Possible Solution

Define a budget-aware overflow recovery contract that distinguishes model context-window misconfiguration from reserve-floor tuning, avoids reinflating bootstrap context during recovery, and only hard-resets after minimal-context or clearly diagnosed fallback paths are exhausted.

## Reproduction Assessment

Yes for the source-level recovery shape: current main still supports the reported bounded overflow retries, optional truncation, bootstrap/post-compaction reinjection, and hard-reset fallback. No for the exact Basecamp/Voltti session key, which is not available in this checkout.

## Solution Assessment

No. Current main’s continuation and post-compaction guard mitigations are useful, but they do not fully solve the misleading fallback guidance, generic retry/reset loop, bootstrap reinjection, or empty-session truncation branch.
