## Summary

Keep open. Current main has useful mitigations for small model context windows, aggregate tool-result overflow, opt-in bootstrap skipping, mid-turn precheck, and bounded reset replay, but the reported combined recovery shape still exists: generic reserve-floor guidance, bounded compaction retry, retry-time prompt/bootstrap rebuilding by default, post-compaction AGENTS reinjection by default, empty-session truncation diagnostics, and hard-reset fallback.

## What This Changes

The issue asks OpenClaw to fix repeated context-overflow hard reset loops where retry recovery rebuilds bootstrap context and gives reserve-floor guidance even when reserve headroom is already high.

## Best Possible Solution

Define a budget-aware overflow recovery contract that distinguishes model context-window misconfiguration from reserve-floor tuning, suppresses or shrinks retry-time bootstrap reinjection during recovery, and only hard-resets after a minimal-context or clearly diagnosed fallback path is exhausted.

## Reproduction Assessment

Yes for the source-level recovery shape: current main can still take embedded context-overflow failures through bounded compaction retries, optional truncation, and hard-reset fallback while rebuilding normal attempt context by default. The exact Basecamp/Voltti session key cannot be reproduced from this repository alone.

## Solution Assessment

No. Current main has partial mitigations, but they do not fully solve the reported repeated-reset loop, misleading reserve-floor guidance, per-attempt bootstrap rebuilding, or empty-session truncation branch.
