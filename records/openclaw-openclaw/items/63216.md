## Summary

Keep open. Current main has useful mitigations for small context-window misconfiguration and bootstrap opt-outs, but the reported recovery shape still exists: generic reserve-floor guidance, bounded compaction retries, default retry/bootstrap reinjection, post-compaction AGENTS reinjection, empty-session truncation diagnostics, and hard reset fallback.

## What This Changes

Investigate repeated context-overflow hard resets where retry recovery re-injects bootstrap context despite high compaction reserve settings.

## Best Possible Solution

Define a budget-aware overflow recovery contract that distinguishes model context-window misconfiguration from reserve-floor tuning, shrinks or suppresses bootstrap during recovery, and uses a minimal-context fallback before hard reset.

## Reproduction Assessment

Yes for the source-level recovery shape: current main still routes embedded context-overflow failures through three compaction attempts, optional truncation, and hard-reset fallback while default bootstrap/post-compaction injection can add context again. No for the exact Voltti/Basecamp session key, which requires the reporter's live transcript/config.

## Solution Assessment

No. Current main has partial mitigations such as reserve-floor capping, `contextInjection` opt-outs, and post-compaction section controls, but these do not fully solve the default retry loop, generic fallback guidance, bootstrap reinjection, or empty-session truncation branch.
