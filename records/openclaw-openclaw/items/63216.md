## Summary

Keep open: current main and the latest release still allow finite overflow-recovery retries to end in a blocked context-overflow path, and bootstrap reinjection after post-compaction retries remains documented default behavior. The issue asks OpenClaw to investigate repeated context-overflow hard resets where retry recovery appears to re-inject bootstrap context and gives reserveTokensFloor guidance that does not resolve the loop.

Keep this issue open as the canonical recovery-policy tracker while maintainers decide whether the open overflow retry guard PR, tiered bootstrap loading, or a broader minimal-context retry/checkpointing design should own each piece.
