## What This Changes

Adds path-scoped filtering for the agent `gateway` tool's `config.get` result and tests full, scoped, and missing-path behavior.

## Best Possible Solution

Land a narrow agent-tool scoped-read mode that returns only a redacted config subtree for a valid path while preserving full-snapshot behavior without `path`.

## Solution Assessment

Yes, the implementation direction is narrow because gateway RPC `config.get` accepts no params, so filtering the already-redacted snapshot inside the agent tool fits the current protocol boundary. The merge blocker is proof, not a definite code defect.
