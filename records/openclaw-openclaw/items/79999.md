## Summary

## What This Changes

The PR makes agent `gateway` tool `config.get` return a scoped `{ path, value, hash }` result when `path` is supplied, while preserving full-snapshot behavior without `path`.

## Best Possible Solution

Land the agent-tool result-slicing fix after maintainer validation, keeping the gateway RPC contract unchanged and preserving full `config.get` output when no path is supplied.
