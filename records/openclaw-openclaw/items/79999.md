The PR makes the agent-facing gateway tool return a path-scoped config subtree for `config.get` when `path` is supplied, while preserving full-snapshot behavior without `path`, and adds regression tests.

## Best Possible Solution

Land the narrow agent-tool result-slicing fix after maintainer validation, keeping the gateway RPC contract unchanged and preserving full-snapshot behavior when no path is supplied.
