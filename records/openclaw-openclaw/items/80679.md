## Summary

Keep open. Current main still validates the full attachment payload with a regex in the gateway parser, and the reporter has an open closing PR for the same failure; this issue should remain open until that PR lands or is closed.

## What This Changes

The issue reports a mobile webchat image attachment failure where `chat.send` returns `INVALID_REQUEST` after `isValidBase64` throws `RangeError: Maximum call stack size exceeded`.

## Best Possible Solution

Review and land a narrow gateway fix that replaces regex payload validation with stack-safe linear validation and focused large-attachment regression coverage, then close this issue after the fix merges.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. The issue has a concrete shipped stack trace, and current main still has the same `chat.send` attachment parser and full-payload regex path.

## Solution Assessment

Yes. The narrow maintainable solution is to remove unbounded regex validation from `src/gateway/chat-attachments.ts` while preserving invalid-base64 and size-limit checks; the open linked PR is the implementation candidate to review.
