## Summary

Keep open. The report has a precise production stack from `v2026.5.7`, and current `main` still runs a whole-payload base64 regex in the `chat.send` attachment parser before image offload; this is a narrow gateway attachment bug worth fixing rather than closing as stale or implemented.

## What This Changes

Issue reports a mobile webchat `chat.send` crash when multi-megabyte base64 image attachments are validated in `isValidBase64`.

## Best Possible Solution

Replace the regex-based base64 validation in the gateway attachment parser with a stack-safe linear validator or decode/size verification path, add a multi-megabyte attachment regression test, and leave the `sharp` packaging fix to the existing canonical issue/PR.

## Reproduction Assessment

No high-confidence live current-main reproduction was established. The issue includes an exact `v2026.5.7` stack naming `isValidBase64`, and current source still has the same full-payload regex path, but a direct Node regex probe on multi-megabyte strings did not reproduce the RangeError locally.

## Solution Assessment

Yes for the primary bug: the narrow fix should remove unbounded regex validation from `src/gateway/chat-attachments.ts` without changing the attachment protocol. The safer alternative to trusting large payloads by prefix alone is a linear character/padding check plus the existing decoded-size verification.
