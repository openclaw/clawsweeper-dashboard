## Summary

Keep open. Current main still runs a whole-payload regex in the gateway attachment parser, and the linked open fix PR has not landed yet; this issue should stay open until that implementation is reviewed and merged or replaced.

## What This Changes

The issue reports `chat.send` failing on large mobile webchat image attachments because `isValidBase64` validates multi-megabyte base64 payloads with a whole-string regex.

## Best Possible Solution

Land or replace the open non-decoding linear validator PR with focused regression coverage, then close this issue after the gateway fix merges; keep the `sharp` packaging/runtime work on its separate media optimizer tracker.

## Reproduction Assessment

No high-confidence live current-main reproduction is available. Source inspection does show the reported `chat.send` path still runs a whole-string base64 regex over attachment payloads, and the issue includes production logs naming `isValidBase64`.

## Solution Assessment

Yes. The narrow maintainable fix is to replace the regex with non-decoding linear validation in `src/gateway/chat-attachments.ts` while preserving size guards and invalid-base64 behavior.
