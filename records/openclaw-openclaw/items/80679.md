## Summary

Keep open. Current main and the latest release still use the whole-payload base64 regex in the gateway attachment parser, and the paired open PR is unmerged and blocked on proof/CI.

## What This Changes

The issue reports mobile webchat image sends failing because gateway attachment parsing can stack-overflow while regex-validating large base64 payloads.

## Best Possible Solution

Land a stack-safe, non-decoding base64 validator in `src/gateway/chat-attachments.ts` with focused regression coverage and redacted real WebChat/gateway proof, while tracking the `sharp` packaging symptom separately.

## Reproduction Assessment

No high-confidence live current-main reproduction was established. Source inspection shows the reported parser path and whole-payload regex are still present, and the issue includes production logs naming `isValidBase64`.

## Solution Assessment

Yes. The narrow maintainable fix is to replace the gateway parser's regex with a non-decoding linear validator that preserves the existing pre-decode size guard; the open PR is directionally right but still needs proof and lint cleanup.
