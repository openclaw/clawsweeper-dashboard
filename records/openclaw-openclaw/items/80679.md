## Summary

Keep open. Current main still validates the full attachment base64 payload with a regex in the gateway parser, the latest stable and beta release refs still show the same validator, and the same reporter has an open paired fix PR that has not merged yet.

## What This Changes

The issue reports mobile WebChat image sends failing in `chat.send` because gateway attachment base64 validation applies a full-payload regex to large base64 strings.

## Best Possible Solution

Review and land or replace the paired gateway validator fix after redacted real WebChat/gateway proof and focused regression coverage, while tracking the optional `sharp` behavior separately if it still reproduces.

## Reproduction Assessment

No high-confidence live current-main reproduction was established. The reporter provided concrete gateway logs naming `isValidBase64`, and current source still runs the full-payload regex, but a direct 10 MiB Node regex probe did not throw.

## Solution Assessment

Yes, the narrow maintainable fix is a non-decoding linear base64 validator in `src/gateway/chat-attachments.ts` that preserves the existing pre-decode size guard. The open paired PR at https://github.com/openclaw/openclaw/pull/81911 follows that direction but still needs real behavior proof and regression coverage before merge.
