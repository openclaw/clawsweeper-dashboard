## Summary

Keep open. Current main and the latest release still contain the whole-payload base64 regex in the gateway attachment parser, and this issue is paired with an open fixing PR that references it with closing syntax, so it should stay open until that PR lands or is closed.

## What This Changes

The issue reports a `chat.send` failure where large mobile WebChat image attachments trigger `RangeError: Maximum call stack size exceeded` in gateway base64 validation.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/81718; the gateway validator should become stack-safe while preserving invalid-base64 and size-limit behavior.

## Reproduction Assessment

No high-confidence live current-main reproduction is established here. The issue includes an exact production stack from `v2026.5.7`, and current main plus `v2026.5.12` still have the same full-payload regex parser path reached by `chat.send`, so this is source-reproducible but not live-reproduced in this review.

## Solution Assessment

Yes, the narrow maintainable fix is to remove unbounded regex validation from the gateway attachment parser while retaining existing validity and decoded-size guards. Because an open linked PR already targets that fix, the issue should remain open until that implementation is resolved.
