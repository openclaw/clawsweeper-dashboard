## Summary

Keep open. Current main still lacks a general doctor diagnostic for active model allowlists hiding provider-owned synthetic or supplemental catalog rows; the existing health contribution checks only `hooks.gmail.model`, while runtime catalog augmentation and allowlist filtering still make the reported gap possible.

## What This Changes

Request to add a warn-only `openclaw doctor` check that reports provider-owned synthetic catalog models missing from a non-empty `agents.defaults.models` allowlist.

## Best Possible Solution

Add a read-only doctor health contribution backed by stable catalog provenance, compare provider-owned synthetic/supplemental rows against non-empty `agents.defaults.models`, and print add guidance without changing config or runtime allowlist enforcement.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: configure a non-empty `agents.defaults.models`, load provider supplemental catalog rows, and current doctor has no general warning before `buildAllowedModelSetWithFallbacks()` filters those rows out of the allowed catalog.

## Solution Assessment

Yes. A warn-only doctor contribution with a stable catalog-source marker is the narrow maintainable solution; auto-adding models or changing allowlist enforcement would cross the operator-control boundary.
