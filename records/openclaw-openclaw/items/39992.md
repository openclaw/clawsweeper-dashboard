## Summary

Keep open. Current main still lacks a general doctor warning for provider-augmented or forward-compatible catalog rows hidden by a non-empty model allowlist; the only doctor model warning I found is the hooks-specific check, while the runtime catalog and allowlist paths still make the reported gap possible.

## What This Changes

Request to add a warn-only `openclaw doctor` check for runtime/synthetic catalog models hidden by an explicit `agents.defaults.models` allowlist.

## Best Possible Solution

Add a warn-only doctor health contribution backed by stable catalog-source metadata, compare provider-augmented runtime rows against a non-empty `agents.defaults.models` allowlist, and print add/remove guidance without changing the allowlist or runtime enforcement.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-empty `agents.defaults.models` allowlist while provider supplemental catalog rows exist, and current doctor has no general allowlist-gap contribution to warn about the hidden rows.

## Solution Assessment

Yes, with one implementation constraint. The narrow maintainable path is a read-only doctor contribution using a stable catalog-source marker; auto-adding models or changing runtime allowlist enforcement would cross the operator control boundary.
