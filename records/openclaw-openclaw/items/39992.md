## Summary

Keep open: current main still has provider-augmented synthetic/forward-compatible catalog rows and explicit allowlist filtering, but `openclaw doctor` has no check that reports augmented catalog rows missing from `agents.defaults.models`. The related implementation PR #46235 was closed unmerged, so it is useful context but not a superseding fix.

## What This Changes

Request adding a warning-only `openclaw doctor` diagnostic for synthetic or provider-augmented runtime catalog models that are available but absent from an explicit `agents.defaults.models` allowlist.

## Best Possible Solution

Add a narrowly scoped, warning-only doctor contribution that detects provider-augmented catalog rows hidden by an explicit model allowlist and suggests additive allowlist commands without modifying config.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a new diagnostic request. Source inspection does confirm the current gap: augmented catalog rows exist, allowlist filtering exists, and doctor has no matching warning check.

## Solution Assessment

Unclear pending maintainer acceptance. The proposed warning-only direction is plausible, but the current catalog lacks a source marker and the later cost-profile warning idea should be kept separate unless maintainers explicitly want that broader policy.
