## Summary

Keep open. Current main still has only the hooks-specific doctor model allowlist warning, while runtime model catalog loading still merges provider-owned supplemental rows that can be filtered out by a non-empty `agents.defaults.models` allowlist without any general doctor warning.

## What This Changes

The issue requests a read-only `openclaw doctor` warning for runtime catalog models that are available but missing from a configured `agents.defaults.models` allowlist.

## Best Possible Solution

Add a warn-only doctor health contribution that marks provider-augmented or forward-compatible catalog rows through a stable catalog contract, compares them against non-empty `agents.defaults.models`, and prints concrete add guidance while skipping absent or empty allowlists.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: with a non-empty `agents.defaults.models` allowlist and provider supplemental catalog rows present, current `openclaw doctor` has only the hooks model check and no general allowlist-gap warning path.

## Solution Assessment

Yes. The narrow maintainable solution is a read-only doctor contribution backed by a stable catalog-source marker; changing runtime allowlist enforcement or auto-adding models would be less appropriate.
