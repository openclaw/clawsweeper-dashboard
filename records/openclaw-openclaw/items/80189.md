## Summary

Keep open: this is an active member-authored, maintainer-labeled bug-fix PR, and the latest head still has a P2 correctness issue where OpenClawKit can replace explicit native scope requests with cached stored-token scopes.

## What This Changes

The branch propagates `AUTH_SCOPE_MISMATCH`, reuses bootstrap/cached device-token scopes on handoff reconnects, preserves full shared-token operator scope defaults, and updates docs, changelog, and focused tests.

## Best Possible Solution

Keep the diagnostic propagation and full defaults, but mirror the Node gateway client by reusing cached stored scopes only for implicit cached-token reconnects while passing explicit native `GatewayConnectOptions.scopes` through to the gateway.

## Reproduction Assessment

Yes, source-reproducible. `GatewayConnectOptions.scopes` and `openclaw-mac connect --scopes` provide an explicit native scope path, while PR head returns cached `storedScopes` for every selected `.deviceToken` connect before using `requestedScopes`.

## Solution Assessment

No, not yet. The `AUTH_SCOPE_MISMATCH` propagation and restored full shared-token defaults are the right direction, but the Swift scope resolver needs the same explicit-scope preservation already present in the Node gateway client.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve explicit native scope requests:** `apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayChannel.swift:618-622`
  - body: When OpenClawKit selects a stored device token, this branch always replaces the caller's `GatewayConnectOptions.scopes` with cached token scopes. That includes explicit native requests such as `openclaw-mac connect --scopes`, so those callers can connect with less authority than requested and later fail RPCs instead of getting the intended scope-mismatch approval path. Gate cached-scope reuse to default/bootstrap retry paths or track whether scopes were explicitly requested.
  - confidence: 0.86
