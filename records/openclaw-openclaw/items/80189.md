## Summary

Keep this PR open for maintainer review: the diagnostic propagation matches the linked bug, but the patch appears to remove pairing/admin runtime privileges from normal Control UI and native operator connections, not only from bootstrap-issued handoff tokens.

## What This Changes

The PR narrows default operator connect scopes in Swift, macOS CLI, and Control UI, propagates `AUTH_SCOPE_MISMATCH` diagnostics through gateway/protocol/client handling, and updates docs, changelog, and focused tests.

## Best Possible Solution

Keep the dedicated scope-mismatch diagnostic work, but split the scope policy so bootstrap/device-token handoff uses only approved handoff scopes while shared-token or admin-capable UI/native flows can still request pairing/admin when they need those RPCs.

## Reproduction Assessment

Yes, source-reproducible. The PR’s narrowed arrays feed directly into connect scopes, and current gateway authorization checks later RPCs against `client.connect.scopes`, which makes pairing/admin requests fail with `missing scope` after the patch.

## Solution Assessment

No, not as proposed. The `AUTH_SCOPE_MISMATCH` propagation is the right direction, but making all default Control UI/native connects bootstrap-compatible is broader than the bug and should be split by auth path or explicit capability need.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve full Control UI scopes outside bootstrap handoff:** `ui/src/ui/gateway.ts:154-158`
  - body: After this change, `buildConnectPlan()` always sends the narrowed `CONTROL_UI_OPERATOR_SCOPES`, including when the user presents the shared gateway token or already has an admin-capable stored device token. The gateway authorizes RPCs against `client.connect.scopes`, so dashboard calls like `device.pair.list`/`device.pair.approve` and admin methods such as `update.status` start returning `missing scope`. Please keep the bootstrap-compatible set specific to the handoff/device-token path, or request the full dashboard scope bundle when that auth path is allowed to have it.
  - confidence: 0.88
- **[P2] Keep native pairing/admin scopes for non-handoff sessions:** `apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayChannel.swift:149-153`
  - body: The shared OpenClawKit default is used when the native app connects without explicit options. Removing `operator.pairing` and `operator.admin` here means native pairing prompts and admin UI calls run through sessions that no longer carry the scopes required by `node.pair.*`, `device.pair.*`, and admin-class gateway methods. Please keep the narrowed scope set limited to bootstrap-issued operator tokens or pass explicit full scopes for native UI/admin flows.
  - confidence: 0.82
