## Summary

Keep open: this is a maintainer-labeled, member-authored auth-scope PR, and the follow-up commit addresses the earlier full-scope regression, but the current patch still appears to swallow explicit native scope requests when a stored device token exists.

## What This Changes

The PR propagates device-token scope mismatch diagnostics through gateway, Control UI, and OpenClawKit, reuses bootstrap/cached-token scopes for handoff reconnects, and updates docs, changelog, and focused tests.

## Best Possible Solution

Keep the diagnostic propagation and full shared-token defaults, but distinguish default bootstrap/cached-token reconnects from callers that explicitly request native scopes so explicit requests still reach the gateway.

## Reproduction Assessment

Yes, source-reproducible. `GatewayConnectOptions.scopes` and `openclaw-mac connect --scopes` provide an explicit scope path, while the PR head returns cached stored-token scopes for all `.deviceToken` connects before sending the handshake.

## Solution Assessment

No, not yet. The `AUTH_SCOPE_MISMATCH` propagation and restored full shared-token defaults are the right direction, but cached-scope reuse needs a narrower condition that does not swallow explicit native scope requests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve explicit native scope requests:** `apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayChannel.swift:618-622`
  - body: When OpenClawKit is using a stored device token, this branch replaces `GatewayConnectOptions.scopes` with cached token scopes for every `.deviceToken` connect. That also covers callers that intentionally pass explicit scopes, such as `openclaw-mac connect --scopes`, even though the docs added here say explicit scopes keep the requested set. Those clients can connect with less authority than requested and later hit missing-scope RPC failures instead of the intended scope-mismatch/approval path. Please limit cached-scope reuse to the default bootstrap handoff/retry path, or carry an explicit-scope signal.
  - confidence: 0.82
