## Summary

Keep open. The PR addresses a real Matrix MAS cross-signing bootstrap gap, and current main still has the old non-stage-aware UIA fallback, but the submitted patch still trims the configured Matrix password before sending it to `m.login.password`, which is a correctness regression for opaque credentials.

## What This Changes

The PR changes Matrix crypto bootstrap to detect MAS with `auth_metadata`, parse UIA 401 stages and sessions, surface typed MAS cross-signing reset errors with URLs, add regression tests, and add a Matrix changelog entry.

## Best Possible Solution

Request changes, then land the MAS/UIA approach after it preserves the raw configured Matrix password, adds a whitespace-password regression test, and updates the Matrix verification docs for MAS reset recovery. Keep the existing stage-aware UIA/session threading and guarded `auth_metadata` probe design.

## Reproduction Assessment

Yes. The original bug is reproducible from the companion MAS-fronted Synapse report and current-main UIA callback behavior; the PR regression is reproducible with a focused unit test where `channels.matrix.password` intentionally contains leading or trailing whitespace.

## Solution Assessment

No, not as currently patched. The stage-aware UIA handling, session threading, typed MAS reset error, and guarded capability probe are the right direction, but trimming an opaque credential before submission is not the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve the submitted Matrix password:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:217-228`
  - body: The patch assigns `params.password?.trim()` to `password` and then sends that value in the `m.login.password` auth dict. Current main submits the configured password unchanged, and Matrix passwords are opaque credentials, so accounts with intentional leading or trailing whitespace would fail UIA after this change. Use trimming only to decide whether a password is configured, but submit the original `params.password`.
