Full review comments:

  - body: Matrix passwords are opaque credentials. This callback trims the configured password and then submits the trimmed value in the m.login.password auth dict, so a valid password with leading or trailing whitespace would be changed before authentication.
  - confidence: 0.92

  - body: Synapse MAS challenges advertise the stable m.oauth stage and URL under params.m.oauth, while this helper only reads params.org.matrix.cross_signing_reset.url. Handle both stable and unstable keys so stable-only challenges still surface the actionable approval link.
  - confidence: 0.9
- **[P2] Remove or use the mismatch helper:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:68-80`
  - body: isCrossSigningKeyMismatchError is declared but never called, and the live PR checks are failing lint. Delete it or wire it into the intended fallback path before merge.
  - confidence: 0.99

  - body: This test defines bootstrapCrossSigning as a zero-argument mock and then reads mock.calls[0]?.[0], which fails test type checking. Give the mock the bootstrap-options signature or reuse the typed helper used by nearby tests.
  - confidence: 0.97

  - confidence: 0.82
