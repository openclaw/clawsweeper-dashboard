## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Preserve the submitted Matrix password:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:217-228`
  - body: The callback trims `params.password` into `password` and then submits that trimmed value for `m.login.password`. Matrix passwords are opaque credentials; current main only trims to decide whether one is configured while sending the original value, so accounts with intentional leading or trailing whitespace would fail UIA after this patch.
  - confidence: 0.92
- **[P2] Handle the stable MAS OAuth stage:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:232-237`
  - body: This branch only recognizes `org.matrix.cross_signing_reset`, but current Matrix spec and Synapse use the stable `m.oauth` UIA stage for OAuth-authenticated `/keys/device_signing/upload`, with the approval URL under `params.m.oauth.url`. Stable-only homeservers would still fall through to unsupported-stage handling instead of surfacing the MAS reset URL.
  - confidence: 0.9
- **[P2] Remove or wire the mismatch helper:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:68-80`
  - body: `isCrossSigningKeyMismatchError` is declared but never called, and PR CI already fails lint with the unused-function error. Delete it or use it in the intended mismatch fallback path before merge.
  - confidence: 0.99
- **[P2] Type the bootstrap mock before reading its first argument:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.test.ts:607-611`
  - body: This test defines `bootstrapCrossSigning` as `vi.fn(async () => {})`, so TypeScript infers a zero-argument mock and `bootstrapCrossSigning.mock.calls[0]?.[0]` fails `check-test-types` with TS2493. Give the mock an explicit bootstrap-options signature or reuse the typed mock helper.
  - confidence: 0.97
- **[P3] Document MAS reset recovery:** `extensions/matrix/src/matrix/sdk/types.ts:220-221`
  - body: The new reset error tells operators to approve a MAS reset URL and rerun bootstrap within the approval window, but the Matrix docs still describe only no-auth, dummy, and password UIA. Add the operator recovery flow so users do not have to infer it from the error text alone.
  - confidence: 0.82
