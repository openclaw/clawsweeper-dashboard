## Summary

Keep open. Current main still has the old Matrix cross-signing UIA fallback that does not inspect advertised stages or pass the UIA session, so the PR is addressing a real gap from the companion MAS report, but the current patch is not mergeable because it trims the configured Matrix password before submitting it to `m.login.password` UIA.

## What This Changes

The PR updates the Matrix plugin's cross-signing bootstrap to detect MAS via `auth_metadata`, route UIA by advertised stages with sessions, expose typed MAS reset errors/URLs, and add changelog/tests.

## Best Possible Solution

Keep the stage-aware MAS UIA approach, but preserve the raw configured password in the submitted auth dict, add a regression test for intentional leading/trailing whitespace, update Matrix verification docs for MAS reset recovery, then run the focused Matrix bootstrap tests and changed gate.

## Reproduction Assessment

Yes. The companion bug has concrete MAS UIA reproduction data, and the PR regression is reproducible with a unit test where `channels.matrix.password` intentionally contains leading or trailing whitespace and the homeserver advertises `m.login.password`.

## Solution Assessment

No, not as currently patched. Parsing advertised UIA stages, threading `session`, surfacing a typed MAS reset URL, and using the guarded `auth_metadata` probe are the right direction, but trimming an opaque credential before submission is not the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve submitted Matrix passwords:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:217-227`
  - body: The patch assigns `params.password?.trim()` to `password` and then sends that value in the `m.login.password` auth dict. Current main trims only for the presence check and submits the configured password unchanged, and Matrix passwords are opaque credentials, so accounts with intentional leading or trailing whitespace would fail UIA after this change. Keep trimming only for the configured-password check and submit the original `params.password`.
  - confidence: 0.88
