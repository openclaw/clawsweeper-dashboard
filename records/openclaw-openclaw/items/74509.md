## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Preserve the submitted Matrix password:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:217-228`
  - body: The patch trims `params.password` into `password` and then sends that trimmed value in the `m.login.password` auth dict. Matrix passwords are opaque credentials and current main submits the configured value unchanged, so accounts with intentional leading or trailing whitespace would fail UIA after this change. Use trimming only to decide whether a password is configured, but submit the original `params.password`.
  - confidence: 0.92
- **[P2] Remove or use the mismatch helper:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:68`
  - body: `isCrossSigningKeyMismatchError` is added but never called, and PR CI already fails `check-lint`/`lint:extensions:bundled` with the unused-function error. Delete the helper or wire it into the intended fallback path before merge.
  - confidence: 0.99
- **[P2] Type the bootstrap mock before reading its first argument:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.test.ts:597-607`
  - body: This test defines `bootstrapCrossSigning` as `vi.fn(async () => {})`, so TypeScript infers a zero-argument mock and `bootstrapCrossSigning.mock.calls[0]?.[0]` fails `check-test-types` with TS2493. Give the mock an explicit bootstrap-options signature or reuse the existing typed mock helper.
  - confidence: 0.97
- **[P3] Document MAS reset recovery:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:232-248`
  - body: The new reset-stage path changes the operator-facing recovery workflow, but the Matrix docs still describe only the old no-auth/dummy/password UIA sequence. Update `docs/channels/matrix.md` to explain opening the MAS reset URL, approving the reset, and rerunning bootstrap inside the approval window.
  - confidence: 0.82
