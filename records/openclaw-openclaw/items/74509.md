## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Preserve the submitted Matrix password:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:217-228`
  - body: The callback stores `params.password?.trim()` in `password` and then sends that trimmed value in the `m.login.password` auth dict. Matrix passwords are opaque credentials; current main only uses trimming to test configuredness while submitting the original value, so accounts with intentional leading or trailing whitespace would fail UIA after this change.
  - confidence: 0.92
- **[P3] Handle the stable MAS reset stage:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:232-237`
  - body: The reset branch only recognizes `org.matrix.cross_signing_reset`, but current Synapse advertises the stable `m.oauth` stage alongside the unstable name and MSC4312 allows that stable stage to carry the reset URL. Treat `m.oauth` as the same reset flow and read `params["m.oauth"].url` so servers that stop sending the unstable stage still surface the operator URL.
  - confidence: 0.78
- **[P2] Remove or use the mismatch helper:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:68`
  - body: `isCrossSigningKeyMismatchError` is declared but never called, and PR CI already fails lint with the unused-function error. Delete it or wire it into the intended fallback path before merge.
  - confidence: 0.99
- **[P2] Type the bootstrap mock before reading its first argument:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.test.ts:597-607`
  - body: This test defines `bootstrapCrossSigning` as `vi.fn(async () => {})`, so TypeScript infers a zero-argument mock and `bootstrapCrossSigning.mock.calls[0]?.[0]` fails `check-test-types` with TS2493. Give the mock an explicit bootstrap-options signature or reuse the typed mock helper.
  - confidence: 0.97
- **[P3] Document MAS reset recovery:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:232-248`
  - body: The new reset-stage path changes the operator-facing recovery workflow, but the Matrix docs still describe only the old no-auth, dummy, and password UIA sequence. Update the Matrix channel docs to explain opening the MAS reset URL, approving the reset, and rerunning bootstrap inside the approval window.
  - confidence: 0.82
