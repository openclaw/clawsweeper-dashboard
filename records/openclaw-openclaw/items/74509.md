## Summary

## What This Changes

The PR updates Matrix crypto bootstrap to probe MAS auth metadata, route cross-signing UIA challenges by advertised stage/session/reset URL, add typed errors and regression tests, and add a Matrix changelog entry.

## Best Possible Solution

Land the Matrix plugin fix after preserving opaque credentials, supporting stable and unstable MAS/OAuth reset challenges, fixing CI/type failures, and documenting the operator reset-and-rerun flow.

## Reproduction Assessment

Yes, at source level: current main can exercise the UIA callback with a 401 UIA body and no configured password, and it throws the old password fallback error without inspecting stages, reset URLs, or sessions. I did not repeat the live MAS homeserver reproduction.

## Solution Assessment

No, not as currently patched. The stage-aware UIA/MAS reset direction is the narrow maintainable solution, but this branch must preserve raw passwords, handle stable m.oauth reset data, fix CI/type failures, and update operator docs before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Submit the raw Matrix password:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:217-228`
  - body: Matrix passwords are opaque credentials. This callback trims the configured password and then submits the trimmed value in the m.login.password auth dict, so a valid password with leading or trailing whitespace would be changed before authentication.
  - confidence: 0.92
- **[P2] Read the stable m.oauth reset URL:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:62-65`
  - body: Synapse MAS challenges advertise the stable m.oauth stage and URL under params.m.oauth, while this helper only reads params.org.matrix.cross_signing_reset.url. Handle both stable and unstable keys so stable-only challenges still surface the approval link.
  - confidence: 0.9
- **[P2] Remove or use the mismatch helper:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:68-80`
  - body: isCrossSigningKeyMismatchError is declared but never called, and the live PR checks are failing lint. Delete it or wire it into the intended fallback path before merge.
  - confidence: 0.99
- **[P2] Type the bootstrap mock before reading its first argument:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.test.ts:607-611`
  - body: This test defines bootstrapCrossSigning as a zero-argument mock and then reads mock.calls[0]?.[0], which fails test type checking. Give the mock the bootstrap-options signature or reuse the typed helper used by nearby tests.
  - confidence: 0.97
- **[P3] Document the MAS reset recovery flow:** `extensions/matrix/src/matrix/sdk/types.ts:220-221`
  - body: The patch adds a new operator-visible MAS reset URL and approval-window retry flow, but the Matrix channel docs still describe only no-auth, dummy, and password UIA. Update the docs so operators know to open the URL and rerun bootstrap within the approval window.
  - confidence: 0.82
