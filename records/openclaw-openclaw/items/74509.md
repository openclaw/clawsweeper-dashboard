## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Preserve the submitted Matrix password:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:217-228`
  - body: The callback stores `params.password?.trim()` and then sends that trimmed value in the `m.login.password` auth dict. Matrix passwords are opaque credentials; keep the original password for submission and use trimming only to decide whether a password was configured.
  - confidence: 0.92
- **[P2] Read the m.oauth reset URL:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:62-65`
  - body: `extractMasResetUrl` only reads `params.org.matrix.cross_signing_reset.url`, but current Synapse also advertises the stable `m.oauth` stage and URL. A stable-only challenge, or one whose usable URL is only under `params.m.oauth`, would lose the actionable approval link.
  - confidence: 0.9
- **[P2] Remove or use the mismatch helper:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:68-80`
  - body: `isCrossSigningKeyMismatchError` is declared but never called, and the PR's lint checks are failing on that unused function. Delete it or wire it into the intended fallback path before merge.
  - confidence: 0.99
- **[P2] Type the bootstrap mock before reading its first argument:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.test.ts:607-611`
  - body: This test defines `bootstrapCrossSigning` as `vi.fn(async () => {})`, so TypeScript infers a zero-argument mock and `bootstrapCrossSigning.mock.calls[0]?.[0]` fails test type checking. Give the mock an explicit bootstrap-options signature or reuse the typed mock helper.
  - confidence: 0.97
- **[P3] Document the MAS reset recovery flow:** `extensions/matrix/src/matrix/sdk/crypto-bootstrap.ts:232-249`
  - body: The patch adds an operator-visible MAS reset URL and approval-window retry flow, but `docs/channels/matrix.md` still describes only no-auth, dummy, and password UIA. Update the Matrix channel docs so operators know to open the MAS URL and rerun bootstrap inside the approval window.
  - confidence: 0.82
