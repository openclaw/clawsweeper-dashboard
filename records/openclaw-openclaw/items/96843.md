## Summary

Keep this PR open: the default-off Google Chat thread-session idea is useful and now has redacted live proof, but the current patch can bypass existing DM peer route bindings when the option is enabled, so it is not merge-ready.

## What This Changes

The PR adds an optional `channels.googlechat.threadSessions` Google Chat config key, routes configured DM thread events to thread-scoped sessions, keeps replies and typing in the originating thread, and updates schema, docs, generated metadata, and focused tests.

## Best Possible Solution

Land a default-off Google Chat DM thread-session option only after preserving existing DM peer route bindings, adding bound-DM coverage, and confirming the public config name and scope.

## Reproduction Assessment

Yes at source-review level for the blocker: current route resolution matches exact peer ids and optional parent peers, while the PR changes the DM peer id without supplying a parent. The contributor's live proof covers new session-key creation, not existing binding compatibility.

## Solution Assessment

No in the current shape; the narrow feature is plausible, but it must preserve existing DM route bindings and settle the public config contract before merge.
