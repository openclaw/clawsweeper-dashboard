## Summary

Keep open: current main still drops registerHook context mutations, and this PR is a focused fix for the linked plugin hook bug, but it still needs real behavior proof before merge and the mutable plugin hook contract is compatibility-sensitive.

## What This Changes

The PR changes registerHook wrapping to pass the original hook context with temporary pluginConfig injection and adds a regression test for agent:bootstrap context mutation.

## Best Possible Solution

Land a narrow registerHook fix that preserves context mutations, keeps pluginConfig scoped per handler, and includes contributor or maintainer real-run proof before merge.

## Reproduction Assessment

Yes. Current main clones event.context in src/plugins/registry.ts while src/agents/bootstrap-hooks.ts reads the original event.context.bootstrapFiles after triggerInternalHook returns, so an agent:bootstrap handler assignment is lost by source inspection.

## Solution Assessment

Yes, with a merge-proof caveat. Preserving the original context while restoring/removing pluginConfig is the narrowest fix for the mutable hook contract; merging known fields back would be more fragile and an immutable return-value API would be a larger API change.
