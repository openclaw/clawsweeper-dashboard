## Summary

Keep open: current main already has real channel-plugin contracts and plugin lifecycle commands, and Discord/WhatsApp are external packages, but Telegram and Signal are still bundled/private and the remaining official-channel distribution policy is not resolved.

## What This Changes

The issue requests separating OpenClaw core from channel integrations so official channels can be installed, updated, disabled, and rolled back independently while keeping a simple default distribution.

## Best Possible Solution

Define the official-channel distribution policy, then track per-channel externalization only where SDK compatibility, release, rollback, docs, install repair, and onboarding behavior are ready.

## Solution Assessment

Is this the best way to solve the issue? No clear implementation path is ready from this issue alone; the maintainable path is a maintainer-owned distribution decision followed by narrower per-channel work.
