## Summary

Keep open: current main already has channel plugin contracts, external plugin install/update/enable/disable flows, and some downloadable official channel packages, but docs and package metadata show the official-channel migration is incomplete and some named channels remain private/bundled.

## What This Changes

Proposal to make channel integrations separately versioned plugins/packages so core and channels such as Telegram, Discord, WhatsApp, and Signal can be updated, disabled, and rolled back independently.

## Best Possible Solution

Define the target distribution policy for official channel plugins, then split or publish remaining private/bundled channels only where SDK compatibility, release, and rollback guarantees are ready.

## Reproduction Assessment

Not applicable: this is a feature and architecture proposal, not a broken existing behavior report. Source and docs show partial current support plus remaining distribution gaps rather than a failing reproduction path.

## Solution Assessment

Unclear/no: closing as implemented would overstate the current state because the core plugin interface exists but the official-channel packaging and release model is still incomplete. A maintainer-owned roadmap or narrower per-channel tracking issue would be safer.
