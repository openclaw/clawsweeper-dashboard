## Summary

Keep open: current main has real channel-plugin contracts and plugin install/update/enable/disable machinery, but the requested official-channel decoupling is only partially complete and still needs a maintainer-owned distribution policy.

## What This Changes

Architecture proposal to decouple OpenClaw core from channel plugins so channel integrations can be versioned, installed, updated, disabled, or rolled back independently.

## Best Possible Solution

Define the target official-channel distribution policy, then externalize or deliberately retain each bundled channel only when SDK compatibility, release, rollback, docs, and upgrade behavior are ready.

## Reproduction Assessment

Not applicable: this is an architecture and packaging proposal, not a broken existing behavior report. Current source and docs show partial support plus remaining distribution gaps rather than a failing reproduction path.

## Solution Assessment

No: treating this as implemented would overstate the current state because not every requested official channel is independently published or rollbackable. The safer path is a maintainer-owned roadmap or narrower per-channel tracking work.
