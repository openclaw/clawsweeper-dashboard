## Summary

Current main and shipped v2026.6.6 already provide the requested channel-plugin boundary: channel plugins have SDK contracts, official external package catalogs, and install/update/disable/uninstall flows separate from the core package.

## What This Changes

The issue requests separating OpenClaw core from channel integrations so channels can be packaged, installed, updated, disabled, and rolled back independently.

## Best Possible Solution

Keep the current plugin architecture, and track any remaining per-channel packaging or external-plugin bugs as narrower channel-specific issues rather than this broad architecture proposal.
