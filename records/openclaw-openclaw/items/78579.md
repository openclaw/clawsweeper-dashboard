## Summary

Keep open: current main has channel plugin contracts, plugin install/update/enable/disable flows, and a partial official channel catalog, but the docs and package metadata still show an unfinished OpenClaw-owned channel distribution migration with Telegram and Signal private/bundled.

## What This Changes

Request to split official channel integrations from OpenClaw core into independently installable/versioned plugins while preserving bundled defaults and stable adapter contracts.

## Best Possible Solution

Define the official-channel distribution policy, then split or publish remaining private bundled channels only where SDK compatibility, release, and rollback guarantees are ready.

## Reproduction Assessment

Not applicable: this is a feature and architecture proposal, not a broken existing behavior report. Source and docs show partial support plus remaining distribution gaps rather than a failing current-main path.

## Solution Assessment

No/unclear: the existing plugin SDK and lifecycle are the right foundation, but closing as implemented would overstate the current official-channel packaging state. A maintainer-owned roadmap or narrower per-channel tracking issues would be safer.
