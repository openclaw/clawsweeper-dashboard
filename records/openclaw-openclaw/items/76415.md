## Summary

Keep open: current main has active-branch `chat.history` plus separate compaction checkpoint APIs and Sessions-page controls, but the chat view still only exposes a generic compaction divider and does not clearly preserve, explain, or recover pre-compaction visible history after refresh.

## What This Changes

Request that Control UI chat preserve or clearly expose pre-compaction visible history/checkpoints after a refresh.

## Best Possible Solution

Choose and implement a chat-level compaction affordance that makes the boundary explicit and links users to checkpoint context/actions while preserving bounded history semantics.

## Reproduction Assessment

Yes, source-reproducible: `chat.history` reloads the active transcript branch only, while checkpoint snapshots are stored separately and the chat renderer shows only a generic compaction divider. I did not run a live long-session compaction because this review was read-only.

## Solution Assessment

Unclear: current main has the building blocks, but the requested end state needs a maintainer choice between history reconstruction, a richer boundary, or checkpoint actions in the chat view.
