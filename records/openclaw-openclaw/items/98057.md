## Summary

Keep open: the updated proof clears the earlier real-behavior gap and the patch appears correct for the narrow `feishu_chat` contextual-account bug, but it still needs maintainer acceptance because it changes Feishu credential selection for multi-account chat/member lookups and only partially overlaps the broader Feishu account-routing work.

## What This Changes

The PR changes `feishu_chat` registration to use a contextual tool factory and the shared Feishu tool-account resolver, with regression tests for contextual and disabled-account chat routing.

## Best Possible Solution

Keep this as a narrow chat-account routing fix, and either merge it as a complement to the broader Feishu routing work or fold it into the broader PR before landing.

## Reproduction Assessment

Yes at source level: current `registerFeishuChatTools` binds `feishu_chat` to `accounts[0]` during registration, so a tool built with `agentAccountId` cannot change the client account. I did not run a live Feishu tenant repro in this read-only review.

## Solution Assessment

Yes for the chat subset: using the existing Feishu tool-account resolver is the narrowest maintainable fix and matches sibling wiki, drive, perm, doc, and bitable tools. It is not a complete solution for the broader non-inbound binding issue tracked at https://github.com/openclaw/openclaw/issues/96657.
