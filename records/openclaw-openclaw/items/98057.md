## Summary

Keep open: the patch is a focused and likely-correct chat-tool fix, but this external draft PR still needs after-fix real behavior proof and maintainer acceptance of the Feishu credential-routing compatibility change before merge.

## What This Changes

The PR changes `feishu_chat` registration and execution to use the shared Feishu tool-account router with contextual account selection, plus regression tests for multi-account chat routing.

## Best Possible Solution

Land the narrow chat-router alignment only after the contributor adds redacted real behavior proof and maintainers accept the contextual-account credential-selection behavior, while keeping the broader non-inbound binding work in https://github.com/openclaw/openclaw/pull/97991.

## Solution Assessment

Yes for the chat-tool subset: reusing the shared Feishu tool-account router is the narrowest maintainable fix and matches sibling Feishu tools. It is not a complete fix for the broader non-inbound binding issue, which remains tracked separately.
