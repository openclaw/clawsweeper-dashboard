## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Honor aliased provider configs before applying auto emotion:** `extensions/speech-core/src/tts.ts:1298`
  - body: `applyAutoEmotionProviderOverride` decides whether explicit config should block inferred emotion by reading only the canonical provider id from raw config. Providers such as Azure Speech also accept aliases like `providers.azure`, so explicit `rate`/`pitch`/`volume` set through that supported alias can be missed and then overwritten by inferred prosody. Resolve aliased raw provider config, or use an explicit-user-config view, before adding auto emotion and add a regression for the alias path.
  - confidence: 0.86
