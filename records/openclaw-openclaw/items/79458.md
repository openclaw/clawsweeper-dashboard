## Summary

Keep open. Current main and latest release still only partially support localized command descriptions: plugin/native transport plumbing exists for Discord and Telegram, but built-in command definitions, Gateway command catalog output, Control UI/TUI command displays, and Telegram custom command config remain single-description surfaces.

## What This Changes

Request to add localized slash-command description fields so command names stay stable while users can see locale-specific descriptions in UI and native channel menus.

## Best Possible Solution

Use one additive locale-map contract, preferably the existing `descriptionLocalizations` shape, across built-in commands, plugin/skill command APIs, Gateway catalog entries, UI/TUI locale selection, and Telegram custom command config.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.10 shows localized description plumbing for some plugin/native command paths, but built-in commands, Gateway `commands.list`, Control UI/TUI command displays, and Telegram custom commands still expose only one description string.

## Solution Assessment

Unclear. The existing `descriptionLocalizations` map looks like the best contract, but maintainers still need to choose translation ownership and locale selection behavior before implementation.
