## Summary

Keep open: current main has partial `descriptionLocalizations` support for native specs, plugins, skills, and Discord, but the requested built-in command, Web UI, TUI, and broader native-menu localization behavior is still missing.

## What This Changes

Add localized slash-command description metadata so Chinese descriptions can be shown without translating command names.

## Best Possible Solution

Define one stable command-description localization contract, likely extending the existing `descriptionLocalizations` shape, then carry it through built-in command definitions, `commands.list`, Web UI/TUI consumers, and channel native serializers where locale-aware display is supported.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior. Source inspection gives high confidence that current main has only partial localization plumbing and still renders or transports fallback English descriptions on key requested surfaces.

## Solution Assessment

Unclear; the existing `descriptionLocalizations` shape looks preferable to adding per-locale fields like `descriptionZhCN`, but maintainers still need to settle the public API and translation source of truth before implementation.
