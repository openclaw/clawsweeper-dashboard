## Summary

Keep open: current main still only partially supports localized slash-command descriptions. Plugins/skills and Discord native command serialization have `descriptionLocalizations`, but built-in command definitions, gateway/UI command payloads, slash menu/help rendering, and TUI commands remain English-only; the issue comments also treat this as a real feature request needing maintainer direction.

## What This Changes

Add localized slash-command description metadata so English command names can show Chinese descriptions in Web UI and native channel menus.

## Best Possible Solution

Define one stable command-description localization contract, then expose it through built-in command registration, gateway protocol, and only the channel/UI surfaces that can actually render localized descriptions.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken documented behavior. Source inspection gives high confidence that current main has partial native metadata support but not the requested built-in/Web UI/TUI localized descriptions.

## Solution Assessment

Unclear; adding per-locale fields like `descriptionZhCN` is likely not the best shape because current code already uses `descriptionLocalizations`. Maintainers should first settle the API and translation ownership, then implement the consuming surfaces coherently.
