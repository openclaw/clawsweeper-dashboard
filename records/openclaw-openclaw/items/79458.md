## Summary

Keep open. Current main and v2026.6.10 still only partially support localized slash-command descriptions: native/plugin plumbing exists for Discord and Telegram, but built-in command definitions, Gateway catalog entries, and Control UI command display still carry one fallback description string.

## What This Changes

Requests an additive localized description metadata path for slash commands so stable English command tokens can display locale-specific descriptions in native menus and clients.

## Best Possible Solution

Extend the existing descriptionLocalizations locale-map contract consistently through built-in commands, plugin/skill command catalog entries, Gateway protocol output, and UI/TUI locale selection, with English fallback preserved.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.10 shows localized native transport plumbing only when metadata already exists, while built-in command definitions, Gateway catalog entries, and Control UI command models still expose one description string.

## Solution Assessment

Unclear. The existing descriptionLocalizations map is the likely best contract, but maintainers still need to decide built-in translation ownership and locale-selection behavior before implementation.
