## Summary

Keep open: current main has partial `descriptionLocalizations` support for native specs, plugins, skills, and Discord, but the requested built-in command, Web UI, and TUI localized-description path is still missing and needs API/product direction.

## What This Changes

Request adding locale-aware slash command descriptions while keeping command names in English across built-in, UI, TUI, plugin, and native channel command surfaces.

## Best Possible Solution

Define one additive `descriptionLocalizations` contract, extend built-in command declarations and `commands.list` only where locale-aware clients can consume it, then document each channel/UI surface's support and fallback behavior.

## Reproduction Assessment

Not applicable; this is a feature request rather than a current bug. Source inspection on current main gives high confidence that localization metadata exists only for part of the command pipeline and is not rendered by the built-in Web UI/TUI paths.

## Solution Assessment

Unclear; the existing `descriptionLocalizations` shape is likely a better base than per-locale fields such as `descriptionZhCN`, but maintainers still need to decide translation ownership and which surfaces should consume localized descriptions.
