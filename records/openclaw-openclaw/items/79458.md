## Summary

Keep open: current main has useful `descriptionLocalizations` plumbing for plugin/skill native specs and Discord, but the requested built-in command declaration, Gateway/Web UI transport, and non-Discord native-menu behavior are still incomplete and need maintainer API/product direction.

## What This Changes

Request localized slash command description metadata for built-in slash commands and channel/UI command menus.

## Best Possible Solution

Define one stable `descriptionLocalizations` contract, extend built-in command definitions and Gateway/Web UI transport where locale-aware rendering is supported, and document/test per-channel behavior instead of adding ad hoc per-locale fields.
