## Summary

Keep open: current main has useful partial `descriptionLocalizations` plumbing for plugin, skill, Discord, and Telegram native command paths, but built-in command declarations, `commands.list`/Web UI rendering, and built-in Chinese descriptions remain English-only.

## What This Changes

Request localized slash-command description metadata for built-in and plugin commands so native menus and the Web UI can show locale-appropriate descriptions without translating command names.

## Best Possible Solution

Define one stable locale-map contract, then wire it through built-in command declarations, the gateway command protocol, Web UI locale selection, and native channel consumers where supported.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken documented behavior. Source inspection gives high confidence that the remaining gap is real because built-in commands and Web UI command surfaces still expose only English `description` text.

## Solution Assessment

Unclear as proposed; `descriptionLocalizations` is the established current shape, but maintainers need to choose translation ownership and which UI/channel surfaces should consume it before a narrow implementation is safe.
