## Summary

Keep open: current main still has only partial slash-command description localization support. `descriptionLocalizations` exists for native command specs, plugin/skill commands, and Discord serialization, but built-in command registration, Gateway/Web UI command entries, and Telegram native menu registration still use only the English `description`; recent discussion also asks for maintainer direction rather than closure.

## What This Changes

Request to add localized slash-command description metadata and render it in Web UI and native command menus while keeping command names in English.

## Best Possible Solution

Standardize one `descriptionLocalizations` contract, then add built-in command translations and plumb only locale-aware Gateway, Web UI, and native-provider surfaces with focused tests and docs.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior. Source inspection gives high confidence that current main has partial localization metadata but not the requested built-in/Web UI/native-menu behavior.

## Solution Assessment

Unclear; the existing `descriptionLocalizations` shape is a better starting point than adding per-locale fields such as `descriptionZhCN`, but maintainers still need to decide locale keys, translation source of truth, and channel-specific rendering policy.
