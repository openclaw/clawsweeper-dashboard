## Summary

Keep open: current main has partial `descriptionLocalizations` support for plugin, Discord, and Telegram native command paths, but the requested built-in command declaration support, gateway/Web command protocol propagation, Web UI rendering, and built-in Chinese descriptions are still missing.

## What This Changes

Add localized slash-command description metadata so built-in, Web UI, and native channel command menus can display Chinese descriptions while keeping command names in English.

## Best Possible Solution

Define one stable `descriptionLocalizations` contract, then wire it through built-in command declarations, gateway `commands.list`, Web UI locale selection, and native channel consumers where supported.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken documented behavior. Source inspection gives high confidence that the remaining gap exists because built-in commands and Web UI command surfaces still expose only English `description` text.

## Solution Assessment

Unclear as proposed; the current code already uses a locale map named `descriptionLocalizations`, so per-locale fields like `descriptionZhCN` would need maintainer API judgment before implementation.
