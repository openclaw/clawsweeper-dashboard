## Summary

Keep open: current main has useful `descriptionLocalizations` plumbing for plugin/skill/native specs and Discord, but the requested built-in command, Web UI, Telegram, and Mattermost localization behavior is still incomplete and needs product/API direction.

## What This Changes

Add a stable localization contract for slash-command descriptions across built-in commands, plugin commands, Web UI help/menu rendering, and native channel command menus while keeping command names English.

## Best Possible Solution

Settle on `descriptionLocalizations` as the stable command-description contract, then wire built-in definitions, gateway/UI rendering, and only the native channel surfaces that can support localized descriptions, with docs and focused tests.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken documented behavior. Source inspection gives high confidence that current main has partial metadata support but still renders/registers English-only descriptions on several requested surfaces.

## Solution Assessment

Unclear; adding per-locale fields such as `descriptionZhCN` is probably not the best API because current main already uses `descriptionLocalizations`. The narrow maintainable path needs maintainer agreement on locale keys, ownership, and per-channel support before implementation.
