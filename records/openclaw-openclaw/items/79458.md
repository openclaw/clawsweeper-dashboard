## Summary

Keep open: current main has partial `descriptionLocalizations` plumbing for plugin, skill, Discord, and Telegram native command surfaces, but built-in command definitions, the Gateway command-list protocol, Web UI/TUI rendering, and locale selection remain English-only for the central request.

## What This Changes

The issue requests localized slash command description metadata and localized display in Web UI, TUI, and native channel menus while keeping command names unchanged.

## Best Possible Solution

Define one additive `descriptionLocalizations` contract for command definitions and command-list payloads, add built-in translations through the approved i18n source, and consume them only on surfaces with a clear locale signal.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior with a failing path. Source inspection gives high confidence that current main has partial native metadata support but not full built-in/Web UI/TUI localization.

## Solution Assessment

Unclear; the requested direction is valid, but per-locale fields like `descriptionZhCN` are not the best fit now that current main already uses the `descriptionLocalizations` map shape. Maintainers should decide the stable API and translation ownership before implementation.
