## Summary

Keep open: current main has partial `descriptionLocalizations` plumbing for plugin/skill command specs and Discord, but built-in commands, Web UI rendering, TUI descriptions, and Telegram native registration still use English-only `description` strings.

## What This Changes

Request to add localized slash-command description metadata so built-in, UI, and native channel command menus can show Chinese descriptions while keeping command names in English.

## Best Possible Solution

Define one stable `descriptionLocalizations`-style contract, then extend built-in command definitions and only the locale-aware Web UI/TUI/native surfaces that can actually render it.

## Reproduction Assessment

Not applicable; this is a feature request, not a bug with a failing runtime path. Source inspection gives high confidence that current main has only partial localization metadata support and still renders English descriptions in the requested built-in/UI surfaces.

## Solution Assessment

Unclear; the existing `descriptionLocalizations` shape is a better fit than adding separate per-locale fields such as `descriptionZhCN`, but maintainers still need to choose the API and ownership model before implementation.
