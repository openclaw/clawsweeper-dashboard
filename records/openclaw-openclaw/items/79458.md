## Summary

Keep open: current main has useful `descriptionLocalizations` plumbing for native command specs, skills, plugins, and Discord, but the broader requested built-in command, Web UI, TUI, and non-Discord native-menu localization behavior is still not implemented. The recent discussion also explicitly asks for maintainer direction, so this should not be swept closed as stale or already fixed.

## What This Changes

The issue requests localized slash-command description metadata for built-in commands, plugin/channel command specs, and user-facing command menus while keeping command names in English.

## Best Possible Solution

Settle one additive command-description localization contract, then wire built-in command definitions and each locale-aware consumer surface to use it while keeping English `description` as the required fallback.

## Reproduction Assessment

Not applicable; this is a feature request, not a bug with a failing current-main reproduction. Source inspection gives high confidence that current main has partial metadata support but not the requested built-in/Web UI/TUI/native-menu localization behavior.

## Solution Assessment

Unclear; `descriptionLocalizations` is a better existing shape than adding per-locale fields like `descriptionZhCN`, but maintainers still need to decide translation ownership and which channel/UI surfaces should consume it.
