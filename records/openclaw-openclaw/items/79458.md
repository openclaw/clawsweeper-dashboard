## Summary

Keep open. Current main and v2026.6.8 have partial `descriptionLocalizations` support for plugin/skill/native command paths and Telegram/Discord transport plumbing, but the built-in helper input, Gateway command catalog, Web UI slash palette, and Telegram custom-command config still expose one fallback description; the remaining API/product choice is still necessary.

## What This Changes

Request to add localized slash-command description metadata so built-in, plugin, UI, and native command menus can show locale-specific descriptions while command names stay unchanged.

## Best Possible Solution

Use one additive locale-map contract, preferably the existing `descriptionLocalizations` shape, then decide built-in translation ownership, client locale behavior, and whether Telegram custom-command config should expose the same map.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.8 shows partial localization plumbing, but built-in command definitions, Gateway catalog output, Web UI command mapping, and Telegram custom command config still expose only one description string.

## Solution Assessment

No, not exactly as proposed. The useful direction is localized command descriptions, but the maintainable fix is to extend the existing `descriptionLocalizations` map consistently instead of adding one-off per-language fields.
