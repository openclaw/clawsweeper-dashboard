## Summary

Keep open. Current main and v2026.6.10 still have only partial command-description localization: plugin/native transport plumbing exists, but built-in command definitions, Gateway command catalog output, Web UI/TUI command displays, and Telegram custom-command config still expose a single fallback description.

## What This Changes

Request to add localized slash-command description metadata so built-in, plugin, TUI, Web UI, and native channel command menus can show non-English descriptions while command names remain stable.

## Best Possible Solution

Use one additive locale-map contract, preferably the existing `descriptionLocalizations` shape, across built-in commands, plugin/skill command APIs, Gateway catalog entries, UI/TUI locale selection, and Telegram custom-command config, with focused docs and regression tests.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current main and v2026.6.10 show only a single description flowing through built-in helper, Gateway catalog, Web UI/TUI, and Telegram custom config despite partial `descriptionLocalizations` support elsewhere.

## Solution Assessment

No, not exactly as proposed. The maintainable path is to extend the existing `descriptionLocalizations` map consistently instead of adding language-specific fields such as `descriptionZhCN`.
