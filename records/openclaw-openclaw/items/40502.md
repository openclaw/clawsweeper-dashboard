## Summary

Keep open. Current main still lacks dynamic tool-run temperature defaults, and this PR is not merge-ready because it carries unrelated secrets, lockfile, and changelog changes while missing current compaction behavior.

## What This Changes

The PR adds low default temperatures for embedded tool-enabled and shell-like runs, new extra-params tests, normal-attempt tool-name plumbing, and unrelated secrets, lockfile, and changelog changes.

## Best Possible Solution

Rebase or replace the branch with a focused agent-only patch that computes dynamic temperature from actual active thinking/provider params, passes tool-name hints through normal and compaction sessions, and drops stale secrets, lockfile, and unrelated changelog changes.

## Reproduction Assessment

Yes. Read-only source review is enough: current main has no dynamic temperature helper, the PR's own test expects no temperature when reasoning visibility is on, and current compaction builds tool allowlists without passing them into applyExtraParamsToAgent.

## Solution Assessment

No. The extra-params layer is the right place, but this implementation needs active-thinking gating, compaction coverage, and removal of unrelated secrets, lockfile, and changelog churn before merge.

## Review Findings

Full review comments:

  - body: The helper treats any non-off reasoningLevel as active thinking, but /reasoning is a visibility setting for whether thinking blocks are shown. Tool-enabled runs with thinking otherwise off will skip the intended low-temperature default whenever reasoning visibility is on or streaming.
  - confidence: 0.91
- **[P2] Remove unrelated secret-provider changes:** `src/secrets/resolve.ts:291`
  - body: This PR is scoped to agent temperature, but it still carries allowInsecurePath schema/configure/resolver changes. That security-sensitive file-secret path has already landed separately on main via #70662, so keeping it here mixes unrelated behavior into the patch.
  - confidence: 0.84
- **[P2] Apply tool-temperature hints to compaction runs:** `src/agents/pi-embedded-runner/run/attempt.ts:1966`
  - body: The new availableToolNames hint is only added in the normal attempt path. Compaction also creates tool-capable sessions, but its applyExtraParamsToAgent call has no tool-name hint, so those tool-enabled turns remain at provider default temperature.
  - confidence: 0.78
