## Summary

Keep open: current main still does not implement the requested sampling default, but this PR is not ready to merge because the branch has correctness blockers, missing real behavior proof, and unrelated security-owned secrets changes.

## What This Changes

The PR branch adds dynamic low default temperatures for embedded-runner tool-enabled and shell-like turns, plus tests, changelog/lockfile updates, and file-secret allowInsecurePath plumbing.

## Best Possible Solution

Land a focused agents-only patch only if maintainers accept this policy; it should use actual active generation/thinking state, cover normal and compaction tool-capable runs, preserve provider cache semantics, and leave secrets changes to the shipped security path.

## Reproduction Assessment

Yes. Source review reproduces the PR blockers: reasoning visibility is used as a generation-state gate, only normal attempts receive tool-temperature hints, and the diff includes unrelated secrets changes.

## Solution Assessment

No. The extra-params layer may be plausible, but this branch is not the best solution until active-thinking gating, compaction coverage, stale security-surface changes, and real behavior proof are addressed.

## Review Findings

  - confidence: 0.82
- **[P2] Drop unrelated file-secret changes:** `src/secrets/resolve.ts:294`
  - body: This agents-temperature branch still changes file-secret `allowInsecurePath` handling on a security-owned surface. That work already landed separately through https://github.com/openclaw/openclaw/pull/70662, so this PR should rebase away the secrets diff instead of reopening that review scope.
  - confidence: 0.85
