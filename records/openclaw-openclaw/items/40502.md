## Summary

Keep open: the requested behavior is still useful because current main has no dynamic tool-temperature helper, but this PR should not merge as-is due to a wrong reasoning-visibility gate, missing compaction coverage, and unrelated security-sensitive secrets changes.

## What This Changes

The PR adds dynamic low default temperatures for embedded tool-enabled runs, wires normal-attempt tool names into extra params, adds focused tests, and also carries unrelated secrets, lockfile, and changelog changes.

## Best Possible Solution

Replace or rebase this into a focused agent-only patch that derives the temperature default from actual active thinking/provider params, passes tool-name hints through both normal and compaction runs, and drops unrelated changes.

## Reproduction Assessment

Yes. Read-only source review is enough: the PR diff suppresses tool-temperature defaults from `reasoningLevel`, while current docs define `/reasoning` as visibility, and compaction currently has allowed tool names without feeding them to the proposed heuristic.

## Solution Assessment

No. The extra-params layer is a reasonable home, but this implementation is not the narrowest maintainable fix until the reasoning signal, compaction path, and unrelated security-sensitive changes are corrected.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Use active thinking signals for temperature gating:** `src/agents/pi-embedded-runner/extra-params.ts:334-335`
  - body: `reasoningLevel` is the `/reasoning` visibility setting, not proof that provider thinking/reasoning generation is active. With this branch, tool-enabled runs where thinking is otherwise off but reasoning visibility is `on` or `stream` skip the new low-temperature default and fall back to provider defaults.
  - confidence: 0.91
- **[P2] Pass tool-temperature hints into compaction runs:** `src/agents/pi-embedded-runner/run/attempt.ts:1966`
  - body: This only adds `availableToolNames` in the normal attempt path. Compaction also computes tool allowlists and prepares a tool-capable session, but it never passes those names into `applyExtraParamsToAgent`, so compacting sessions keep provider-default temperature for tool-capable turns.
  - confidence: 0.8
- **[P2] Remove unrelated secret-provider changes:** `src/secrets/resolve.ts:291`
  - body: This PR is scoped to agent tool temperature, but it carries `allowInsecurePath` schema/configure/resolver changes for file secrets. That security-sensitive surface already landed separately on main through #70662, so keeping it here mixes unrelated behavior into the patch.
  - confidence: 0.85
