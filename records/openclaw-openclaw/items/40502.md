## Summary

Keep open: current main still only forwards explicit configured temperatures, and this PR has blocking review issues around reasoning visibility, compaction coverage, and unrelated secret-provider changes.

## What This Changes

The PR adds dynamic low default temperatures for embedded-runner tool-enabled turns, adds tests/changelog coverage, and also carries file-secret allowInsecurePath plus lockfile/changelog changes.

## Best Possible Solution

Rebase or replace this with a focused agents-only patch that uses actual active-thinking/provider params, covers normal and compaction tool-capable runs, preserves existing cache behavior, and drops unrelated secrets, lockfile, and stale changelog changes.

## Reproduction Assessment

Yes. Source review is enough to reproduce the PR findings: reasoningLevel is documented as visibility, compaction has a separate applyExtraParamsToAgent path, and the PR diff carries unrelated secret-provider changes.

## Solution Assessment

No. The extra-params layer is a plausible place for the behavior, but this branch is not the best fix until active-thinking gating, compaction coverage, and unrelated security-surface changes are corrected.

  - body: `reasoningLevel` controls whether reasoning text is shown, not whether high-reasoning generation is active. Treating any non-off reasoningLevel as an opt-out means tool runs with reasoning display enabled but thinking otherwise off/unset skip the intended low-temperature default.
  - confidence: 0.91

  - confidence: 0.8

  - body: This agents temperature PR also modifies file-secret allowInsecurePath schema/configure/resolve behavior. That security-sensitive surface has already landed separately on main through #70662, so this branch should drop those changes during rebase instead of reopening that review.
  - confidence: 0.84
