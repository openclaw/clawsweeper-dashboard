## Summary

Keep open: current main does not implement the dynamic tool-temperature default, and this PR should not merge as-is because source review still finds reasoning/compaction correctness gaps, unrelated security-sensitive secrets changes, and missing real behavior proof.

## What This Changes

The PR adds dynamic low default temperatures for embedded-runner tool-enabled turns, focused tests, and changelog coverage, while also modifying file-secret allowInsecurePath handling and lockfile entries.

## Best Possible Solution

If maintainers want this policy, land a focused agents-only patch that uses actual active-thinking/provider state, applies consistently to normal and compaction tool-capable runs, preserves cache semantics, and leaves file-secret changes to the already-merged secrets path.

## Reproduction Assessment

Yes. Source review reproduces the PR blockers: the diff gates on reasoningLevel despite docs defining /reasoning as visibility, only wires tool names through the normal attempt call, and carries unrelated secrets changes.

## Solution Assessment

No. The extra-params layer is plausible, but this branch is not the best solution until active-thinking gating, compaction coverage, unrelated security-surface changes, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use active thinking state for temperature gating:** `src/agents/pi-embedded-runner/extra-params.ts:337-338`
  - body: The new gate treats any non-off `reasoningLevel` as active thinking, but `/reasoning` controls visibility rather than generation effort. Tool runs with reasoning display enabled and thinking otherwise off or unset will skip the intended low default and stay on provider-default temperature.
  - confidence: 0.91
- **[P2] Pass tool-temperature hints through compaction:** `src/agents/pi-embedded-runner/run/attempt.ts:1966`
  - body: This only adds `availableToolNames` on the normal attempt call. Compaction builds a tool-capable session separately and applies extra params without a tool hint, so long sessions that compact mid-run fall back to provider-default temperature.
  - confidence: 0.8
- **[P2] Drop unrelated file-secret changes:** `src/secrets/resolve.ts:294`
  - body: This agents-temperature branch also changes file-secret `allowInsecurePath` handling. That security-owned behavior already landed separately through https://github.com/openclaw/openclaw/pull/70662, so the branch should rebase away these changes instead of reopening a secrets review.
  - confidence: 0.84
