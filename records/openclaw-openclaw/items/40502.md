## Summary

This PR should stay open: current main still does not implement the proposed dynamic low-temperature default for tool-capable embedded runs, and the PR branch has correctness, policy, proof, and security-surface blockers.

## What This Changes

The PR adds dynamic default temperatures for embedded-runner tool-capable turns, updates tests, changes cache-retention plumbing, and includes unrelated file-secret allowInsecurePath config changes.

## Best Possible Solution

If maintainers accept the sampling policy, land a focused agents-only patch that keys thinking detection off real provider params, covers normal and compaction tool-capable runs, preserves cache semantics, drops secrets/lockfile churn, and includes redacted real-run proof.

## Reproduction Assessment

Yes for the PR blockers by source inspection: current main lacks the dynamic-temperature path, while the PR gates on reasoning visibility, misses compaction wiring, and carries unrelated secrets changes. The requested low-temperature default itself is a new policy behavior rather than a broken current-main contract.

## Solution Assessment

No. The extra-params layer is a plausible implementation point, but this branch is not the best solution until maintainers accept the default-sampling policy, fix the reasoning and compaction gaps, remove the secrets churn, and add real behavior proof.
