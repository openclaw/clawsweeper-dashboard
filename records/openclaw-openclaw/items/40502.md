## Summary

Keep this PR open, but it is not merge-ready: current main does not already implement the requested dynamic tool temperature policy, and the branch still has source-reproducible blockers plus missing external real behavior proof.

## What This Changes

The PR adds dynamic low default temperatures for tool-enabled embedded-runner turns, focused tests, changelog entries, lockfile updates, and file-secret `allowInsecurePath` changes.

## Best Possible Solution

If maintainers want this policy, land a focused agents-only patch that uses actual active-thinking/provider state, applies consistently to normal and compaction tool-capable runs, preserves cache semantics, and leaves secrets work to the separately merged secrets path.

## Reproduction Assessment

Yes. Source review reproduces the PR blockers: the diff gates on `reasoningLevel` despite `/reasoning` being reply visibility, only wires tool names through the normal attempt path, and carries unrelated secrets changes.

## Solution Assessment

No. The extra-params layer is plausible, but this branch is not the best solution until active-thinking gating, compaction coverage, unrelated security-surface changes, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use active thinking state for temperature gating:** `src/agents/pi-embedded-runner/extra-params.ts:339-340`
  - body: The new gate treats any non-off `reasoningLevel` as active thinking, but `/reasoning` controls visible reasoning output rather than generation effort.   - confidence: 0.82
- **[P2] Drop unrelated file-secret changes:** `src/secrets/resolve.ts:294`
  - body: This agents-temperature branch also changes file-secret `allowInsecurePath` handling. That security-owned behavior already landed separately through https://github.com/openclaw/openclaw/pull/70662, so the branch should rebase away these changes instead of reopening a secrets review.
  - confidence: 0.85
