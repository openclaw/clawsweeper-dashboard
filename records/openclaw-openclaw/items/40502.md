## Summary

Keep open: current main still only forwards explicit temperature, while this PR's proposed default-temperature policy has concrete blockers around reasoning/thinking detection, compaction coverage, and unrelated secrets changes.

## What This Changes

The PR adds dynamic low default temperatures for embedded-runner tool turns, focused tests, and changelog text, while also carrying unrelated secrets/config and lockfile changes.

## Best Possible Solution

Use a focused agents-only rebase or replacement after maintainer approval of the sampling policy, with active-thinking gates, normal and compaction coverage, and no unrelated secrets or lockfile churn.

## Reproduction Assessment

Yes. Source review of the PR context and current main is enough to reproduce the review findings: the proposed gate uses reasoning visibility, normal attempts alone receive tool-name hints, and compaction has a separate extra-params path.

## Solution Assessment

No. The extra-params layer is plausible, but this branch is not the best merge shape until it uses active thinking signals, covers compaction, and drops unrelated security-sensitive secrets changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use active thinking signals for temperature gating:** `src/agents/pi-embedded-runner/extra-params.ts:338-340`
  - body: `reasoningLevel` controls whether thinking blocks are shown, not whether generation thinking is active. With this gate, tool runs with reasoning visibility on but thinking off skip the intended low default, while active Anthropic thinking needs gating through actual thinking params.
  - confidence: 0.91
- **[P2] Pass tool-temperature hints through compaction:** `src/agents/pi-embedded-runner/run/attempt.ts:1966`
  - body: The new `availableToolNames` hint is only wired into the normal attempt path. Compaction builds a separate tool-capable session and still applies extra params without that hint, so long sessions can switch back to provider-default temperature during compaction.
  - confidence: 0.82
- **[P2] Remove unrelated secret-provider changes:** `src/secrets/resolve.ts:294`
  - body: This temperature PR carries file-secret `allowInsecurePath` schema/configure/resolver changes. Current main already contains that separately reviewed secrets surface, so keeping it here broadens the PR and reopens review of an unrelated bypass knob.
  - confidence: 0.84
