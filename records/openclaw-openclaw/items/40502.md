## What This Changes

The PR adds dynamic low default temperatures for tool-enabled embedded-runner turns, focused tests, changelog coverage, and unrelated file-secret allowInsecurePath/lockfile changes.

## Best Possible Solution

If maintainers want this policy, land a focused agents-only patch that uses actual active-thinking/provider state, applies consistently to normal and compaction tool-capable runs, preserves cache semantics, and leaves secrets work to the already-merged secrets path.

## Reproduction Assessment

Yes. Source review reproduces the PR blockers: the diff gates on reasoningLevel despite docs defining /reasoning as visibility, only wires tool names through the normal attempt call, and carries unrelated secrets changes.

## Solution Assessment

No. The extra-params layer is plausible, but this branch is not the best solution until active-thinking gating, compaction coverage, unrelated security-surface changes, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use active thinking state for temperature gating:** `src/agents/pi-embedded-runner/extra-params.ts:339-340`
  - body: The new gate treats any non-off `reasoningLevel` as active thinking, but `/reasoning` controls visibility rather than generation effort.   - confidence: 0.91
- **[P2] Pass tool-temperature hints through compaction:** `src/agents/pi-embedded-runner/run/attempt.ts:1966`
  - body: This only adds `availableToolNames` on the normal attempt call. Compaction builds a tool-capable session separately and has its own allowed tool list, so long sessions that compact mid-run still fall back to provider-default temperature.
  - confidence: 0.8
- **[P2] Drop unrelated file-secret changes:** `src/secrets/resolve.ts:294`
  - body: This agents-temperature branch also changes file-secret `allowInsecurePath` handling. That security-owned behavior already landed separately through https://github.com/openclaw/openclaw/pull/70662, so the branch should rebase away these changes instead of reopening a secrets review.
  - confidence: 0.84
