## Summary

Keep open. Current main still lacks the proposed dynamic tool-temperature behavior, so the PR is not obsolete; however, the branch is not mergeable as-is because the temperature gate conflates reasoning visibility with active thinking, the normal-run plumbing does not cover compaction tool sessions, and the diff still carries unrelated security-sensitive secrets changes plus lockfile/changelog churn.

## What This Changes

The PR adds dynamic default temperature resolution and tool-name plumbing for embedded agent runs, expands extra-param tests, and also modifies secrets configuration/resolution, the lockfile, and the changelog.

## Best Possible Solution

Rebase onto current main, drop the stale secrets, lockfile, and unrelated changelog entries, then implement the temperature default in the current extra-params/runtime-plan shape for both normal embedded attempts and compaction tool sessions. The fix should use actual thinking/extended-reasoning activation signals, preserve explicit temperature and plain-chat behavior, and include focused regression coverage plus one current changelog entry for the user-visible agent behavior.

## Reproduction Assessment

Yes. The source/diff reproduction is high-confidence: in the PR helper, a tool-enabled run with thinking otherwise off but reasoningLevel set to "on" returns no dynamic temperature even though current docs describe /reasoning as visibility. The compaction gap is also reproducible from source because compact.ts has allowedToolNames but does not pass them into applyExtraParamsToAgent.

## Solution Assessment

No, not as-is. The temperature behavior belongs in the embedded extra-params path, but this implementation should be narrower, should not use reasoning visibility as a thinking signal, should cover compaction if the policy is all tool-enabled turns, and should not carry unrelated secrets or lockfile changes.
