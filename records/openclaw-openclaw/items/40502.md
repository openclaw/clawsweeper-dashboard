## Summary

Keep open: current main still lacks the requested dynamic tool-temperature default, but this PR branch is not merge-ready because source review still finds correctness blockers, missing real behavior proof, and unrelated security-owned secrets changes.

## What This Changes

The branch adds dynamic low default temperatures for embedded-runner tool-enabled and shell-like turns, runner tests, changelog/lockfile edits, and file-secret allowInsecurePath plumbing.

## Best Possible Solution

Land a focused agents-only patch only if maintainers accept the sampling-default policy; it should use active generation/thinking state, cover normal and compaction tool-capable runs, preserve cache semantics, and omit stale secrets/lockfile churn.

## Reproduction Assessment

Yes. Source review reproduces the PR blockers: the final PR head treats non-off reasoning visibility as active thinking, wires availableToolNames only through normal attempts, and includes stale secrets changes.

## Solution Assessment

No. The extra-params layer may be plausible, but this branch is not the best solution until the policy is accepted, active-thinking and compaction handling are fixed, the secrets diff is removed, and real behavior proof is supplied.
