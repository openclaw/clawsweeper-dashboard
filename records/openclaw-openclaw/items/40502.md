## Summary

Keep open. Current main does not implement the requested dynamic tool-run temperature default, but this PR is not merge-ready because the branch is stale, mixes unrelated secrets and lockfile changes into the temperature patch, and still misses current embedded compaction behavior.

## What This Changes

The PR adds dynamic low-temperature defaults for embedded tool-enabled runs, wires tool-name hints into normal attempts, expands extra-param tests, and also changes secrets configuration, the lockfile, and the changelog.

## Best Possible Solution

Rebase onto current main, drop the stale secrets, lockfile, and unrelated changelog changes, then implement the temperature default in the current extra-params/runtime-plan shape for both normal embedded attempts and compaction tool sessions.

## Reproduction Assessment

Yes. The source/diff path is high-confidence: current main has no dynamic temperature helper, the PR test for reasoningLevel 'on' explicitly expects no temperature despite /reasoning being a visibility setting, and current compaction calls applyExtraParamsToAgent without any tool-name hint.

## Solution Assessment

No, not as-is. The temperature behavior belongs in embedded extra-params plumbing, but the patch should use actual thinking/extended-reasoning activation signals, cover compaction, and avoid carrying unrelated secrets and lockfile changes.
