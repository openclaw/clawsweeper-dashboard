## Summary

## What This Changes

The PR adds low dynamic temperature defaults for embedded-runner tool-enabled turns, related tests/changelog text, and file-secret allowInsecurePath plumbing.

## Best Possible Solution

If maintainers accept the sampling policy, land a focused agents-only implementation that uses actual provider thinking state, covers normal and compaction tool-capable runs, preserves cache semantics, drops secrets churn, and includes real provider proof.

Yes for the PR blockers by source inspection: PR head gates on reasoningLevel, only wires availableToolNames through the normal attempt path, and carries unrelated secrets changes. The requested sampling default itself is a new policy behavior, not a broken current-main contract.

## Solution Assessment

No. The extra-params layer is plausible, but this branch is not the best solution until maintainers accept the sampling policy, the reasoning and compaction gaps are fixed, stale secrets changes are removed, and real behavior proof is supplied.
