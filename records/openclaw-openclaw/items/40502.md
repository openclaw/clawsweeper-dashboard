## Summary

Keep open: current main still only forwards explicitly configured temperatures, and this PR should not merge as-is because the branch gates on reasoning visibility, misses compaction runs, and carries unrelated security-sensitive secrets changes.

## What This Changes

The PR adds low default temperatures for embedded runner tool turns, passes allowed tool names through normal attempts, adds focused tests and a changelog entry, and also carries unrelated secrets and lockfile changes.

## Best Possible Solution

Rebase or replace the branch with a focused agents-only patch that uses actual active thinking/provider params, covers normal and compaction tool-capable runs, and drops unrelated secrets, lockfile, and stale changelog churn.

## Reproduction Assessment

Yes. Source review is enough to reproduce the review findings: the PR's own branch suppresses dynamic temperature from reasoningLevel, only wires tool names through the normal attempt path, and current main shows compaction using a separate applyExtraParamsToAgent call.

## Solution Assessment

No. The extra-params layer is a reasonable place for the behavior, but the current patch is not the narrowest maintainable solution until active-thinking gating, compaction coverage, and unrelated security changes are corrected.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Use active thinking signals for temperature gating:** `src/agents/pi-embedded-runner/extra-params.ts:338-340`
  - body: `reasoningLevel` is the visibility setting for whether reasoning text is shown, not proof that high-reasoning generation is active. With this branch, tool-enabled runs where thinking is otherwise off but `/reasoning` is on skip the intended low-temperature default and fall back to provider defaults.
  - confidence: 0.91
- **[P2] Apply tool-temperature hints to compaction runs:** `src/agents/pi-embedded-runner/run/attempt.ts:1966`
  - body: The new `availableToolNames` hint is only passed through the normal attempt setup. Compaction builds a separate tool-capable session and still calls `applyExtraParamsToAgent` without a tool-name hint, so long sessions can switch back to provider-default temperature during compaction.
  - confidence: 0.8
- **[P2] Remove unrelated secret-provider changes:** `src/secrets/resolve.ts:294`
  - body: This temperature PR carries file-secret `allowInsecurePath` schema/configure/resolver changes. That security-sensitive surface has already landed separately on main through #70662, so keeping it here mixes unrelated behavior into the patch and reopens review of an unrelated bypass knob.
  - confidence: 0.84
