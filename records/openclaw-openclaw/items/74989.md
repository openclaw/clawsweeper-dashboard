## Summary

Keep open. Current main still returns raw plugin command handler results and dereferences them as objects in the generic and Telegram native command paths, while this PR makes the expected narrow normalization change with focused regression coverage and no definite blocking findings.

## What This Changes

The PR normalizes nullish or non-object plugin command results at `executePluginCommand`, adds defensive caller-side normalization for generic and Telegram native dispatch, and adds regression tests for undefined command results.

## Best Possible Solution

Land one canonical narrow fix that normalizes plugin command results at the shared executor boundary, keeps direct or mocked dispatch paths resilient, and then close the linked regression issue once merged.

## Reproduction Assessment

Yes. On current main, a plugin command handler that resolves `undefined` can pass through `executePluginCommand` and then crash when `handlePluginCommand` reads `result.continueAgent`; #74800 provides matching Telegram release logs.

## Solution Assessment

Yes. Normalizing at the executor boundary is the narrowest maintainable fix, and the caller-side guards are reasonable defense for direct or mocked runtime paths; the remaining maintainer decision is merge readiness and any SDK contract wording follow-up.
