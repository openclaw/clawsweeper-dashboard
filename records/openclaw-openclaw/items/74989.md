## Summary

Keep open. Current main still returns raw plugin command handler results and dereferences them as objects, while this PR applies the narrow normalization needed for #74800 and adds focused regression coverage; the remaining maintainer-cleanup blocker is the required changelog entry.

## What This Changes

The PR normalizes nullish and non-object plugin command handler results in shared and Telegram command dispatch paths, then adds regression tests for undefined command results.

## Best Possible Solution

Land the normalization and regression tests with a single-line Unreleased Fixes changelog entry, then close #74800 after merge. Any decision to document no-payload command handlers as an explicit SDK guarantee can be handled as a follow-up contract/docs change.

## Reproduction Assessment

Yes. On current main, a plugin command handler that resolves undefined flows through executePluginCommand and then handlePluginCommand reads result.continueAgent; #74800 includes matching Telegram gateway logs for that TypeError.

## Solution Assessment

Yes, with the changelog fix. Normalizing at the shared executor boundary is the narrow maintainable repair, and the caller-side guards are reasonable defense for direct or mocked runtime paths.
