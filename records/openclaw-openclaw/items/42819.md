## Summary

Current main and release v2026.6.5 solve the central failure: Docker keeps Playwright Chromium in its managed cache, while the browser plugin now discovers the changing `chromium-*` directory automatically, so users no longer need a version-pinned executable path. The remaining request to relocate the cache and add a global symlink is broader packaging policy, not necessary to restore browser operation, and the old fork PR is conflicting rather than a viable canonical implementation.

## What This Changes

Make Docker-bundled Playwright Chromium discoverable without a versioned absolute path and consider storing it outside the node cache.

## Best Possible Solution

Keep the shipped automatic Playwright-managed Chromium discovery and existing cache-root contract; evaluate any future `/opt` relocation separately with explicit mount and upgrade compatibility evidence.

## Reproduction Assessment

No on current main and v2026.6.5: the browser plugin automatically discovers versioned Playwright-managed Chromium directories, so the reported need to hardcode a changing executable path no longer reproduces.

## Solution Assessment

Yes, the shipped automatic discovery is the narrowest maintainable fix because it preserves existing Docker cache mounts and environment behavior while removing dependence on the versioned child path.
