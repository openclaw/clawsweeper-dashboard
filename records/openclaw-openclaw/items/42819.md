## Summary

Keep open. Current main still installs Docker Playwright Chromium under `/home/node/.cache/ms-playwright`, does not expose a Dockerfile-created `chrome` or `chromium` command on PATH, and the remaining path/PATH decision is already paired with open implementation PR https://github.com/openclaw/openclaw/pull/42771.

## What This Changes

Request moving Docker Playwright browser files out of the hidden cache path and exposing a stable Chrome/Chromium command on PATH so users do not rely on versioned absolute paths.

## Best Possible Solution

Review, refresh, and either land or close https://github.com/openclaw/openclaw/pull/42771 or a maintainer replacement so the Docker path, PATH command, docs, tests, migration notes, and Docker image proof agree.

## Reproduction Assessment

Yes. Source inspection of current main and `v2026.5.12` shows Docker still exports `/home/node/.cache/ms-playwright` and does not add `chrome` or `chromium` on PATH; I did not run a live Docker build in this read-only review.

## Solution Assessment

Unclear. Current main's autodetection is the narrower fix for brittle versioned Chromium paths, while moving the Docker browser store and adding PATH commands is a broader Docker packaging decision that should be settled in the linked PR or a maintainer replacement.
