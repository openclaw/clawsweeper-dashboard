## Summary

Keep open: current main and v2026.6.10 solve the brittle versioned Playwright path by auto-detecting the cache layout, but they still keep Docker browser assets under `/home/node/.cache/ms-playwright` and do not expose stable `chrome` or `chromium` commands on PATH.

## What This Changes

The issue asks for browser-enabled OpenClaw Docker images to place Playwright browser assets outside the hidden cache path and expose a stable browser command on PATH.

## Best Possible Solution

Keep the shipped Playwright-cache auto-detection, then land a focused Dockerfile/docs/test change only if maintainers approve a new `/opt` browser-store and PATH-command contract with upgrade guidance and Docker proof.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.10 still use `/home/node/.cache/ms-playwright` and do not create stable Docker `chrome` or `chromium` PATH commands; I did not run a live Docker build in this read-only review.

## Solution Assessment

Unclear. Auto-detection is the narrow shipped fix for the brittle versioned executable path, while moving the Docker store to `/opt` and adding PATH commands is a compatibility-sensitive image contract change that needs maintainer approval.
