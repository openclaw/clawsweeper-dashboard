## Summary

Keep open: current main and v2026.6.10 partially solve the brittle versioned executable-path problem by auto-detecting Playwright Chromium, but they still install Docker browser assets under `/home/node/.cache/ms-playwright` and do not expose stable `chrome` or `chromium` commands on PATH. The remaining Docker image contract is already represented by an open cross-repository implementation candidate and needs maintainer approval before a new repair lane starts.

## What This Changes

The issue asks OpenClaw Docker images to store Playwright Chromium outside the hidden node cache and expose a stable browser command on PATH.

## Best Possible Solution

Keep the shipped Playwright cache auto-detection, then land a focused Dockerfile/docs/test change only if maintainers approve the `/opt` browser-store and PATH-command contract with upgrade guidance and Docker proof.

## Reproduction Assessment
