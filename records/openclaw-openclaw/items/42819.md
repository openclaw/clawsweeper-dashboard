## Summary

Keep open. Current main still keeps Docker Playwright browsers under `/home/node/.cache/ms-playwright` and does not create Dockerfile-provided `chrome` or `chromium` commands on PATH; the active same-author implementation PR with closing syntax remains open, so this issue should wait for that PR to be reviewed, landed, refreshed, or explicitly replaced.

## What This Changes

Request Docker browser builds to place Playwright browsers outside the hidden `.cache` path and expose a stable Chrome/Chromium command on PATH.

## Best Possible Solution

Review, refresh, and either land or close https://github.com/openclaw/openclaw/pull/42771 or a maintainer replacement so the Docker path, PATH symlink, docs, tests, and image proof all agree.

## Reproduction Assessment

Yes. Source inspection of current main shows Docker still exports `/home/node/.cache/ms-playwright` and only creates the `openclaw` CLI symlink, not `chrome` or `chromium`; I did not run a live Docker build in this read-only review.

## Solution Assessment

Unclear. Current main's autodetection is a narrower fix for brittle versioned Chromium paths, while moving the Docker browser store to `/opt` and adding PATH symlinks is a broader packaging contract that needs maintainer path-selection and a refreshed implementation.
