## Summary

Keep open. Current main still installs Docker Playwright Chromium under `/home/node/.cache/ms-playwright` and does not expose a Dockerfile-created `chrome` or `chromium` command, while the merged autodetection work only solved the brittle versioned executable discovery part.

## What This Changes

The issue asks Docker browser-enabled images to put Playwright browser files outside the hidden node cache and expose a stable Chrome/Chromium executable on PATH.

## Best Possible Solution

Review, refresh, and either land or close https://github.com/openclaw/openclaw/pull/42771 or a maintainer replacement with an explicit Docker browser path contract, migration docs, Dockerfile coverage, and real Docker proof for browser-enabled and browser-disabled images.

## Reproduction Assessment

Yes. Source inspection of current main shows the Dockerfile still exports `/home/node/.cache/ms-playwright` and lacks Dockerfile-created `chrome` or `chromium` PATH commands; I did not run a live Docker build in this read-only review.

## Solution Assessment

Unclear. Current autodetection is the narrowest shipped fix for brittle versioned executable discovery, while moving the Docker store and adding PATH aliases is a compatibility-affecting packaging decision best settled through https://github.com/openclaw/openclaw/pull/42771 or a maintainer replacement.
