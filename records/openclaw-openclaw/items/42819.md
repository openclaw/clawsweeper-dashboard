## Summary

Keep open: current main and v2026.6.6 solve the brittle versioned Playwright executable path by auto-detecting the cache, but the requested Docker packaging contract still is not implemented: the image still uses the hidden node cache and does not expose stable chrome/chromium commands on PATH.

## What This Changes

The issue requests moving Docker Playwright browser files out of a hidden cache path and exposing a stable browser binary on PATH instead of relying on versioned Playwright directories.

## Best Possible Solution

Keep the shipped autodetection fix, then land a maintainer-approved Dockerfile/docs/test change only if the project wants a new /opt browser-store and PATH-symlink contract with upgrade guidance and Docker proof.

## Reproduction Assessment

Yes, source-reproducible: current main and v2026.6.6 still set Docker's Playwright browser path to /home/node/.cache/ms-playwright and have no Dockerfile-created chrome/chromium PATH symlink. I did not run a live Docker build in this read-only review.

## Solution Assessment

Unclear as a product direction: merged autodetection is the narrow fix for brittle versioned executable paths, while moving the browser store to /opt and exposing PATH commands is a separate compatibility-sensitive Docker packaging choice.
