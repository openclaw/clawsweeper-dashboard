## Summary

Keep open: current main and v2026.6.6 fixed the brittle versioned Playwright executable problem by auto-detecting the Docker cache, but the requested Docker packaging contract still is not implemented because the image still uses `/home/node/.cache/ms-playwright` and does not expose stable `chrome` or `chromium` commands on PATH.

## What This Changes

The issue asks for Docker Playwright browser assets to live outside the hidden node cache and for a stable Chrome/Chromium command on PATH.

## Best Possible Solution

Keep the shipped autodetection fix, then land a maintainer-approved Dockerfile/docs/test/release-workflow change only if the project wants a new `/opt` browser-store and PATH-symlink contract with upgrade guidance and Docker proof.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.6.6` shows Docker still exports `/home/node/.cache/ms-playwright` and there is no Dockerfile-created stable `chrome` or `chromium` PATH command; I did not run a live Docker build in this read-only review.

## Solution Assessment

Unclear as a product direction. The merged autodetection path is the narrow fix for brittle versioned executable paths, while moving the browser store to `/opt` and exposing PATH commands is a compatibility-sensitive Docker packaging choice.
