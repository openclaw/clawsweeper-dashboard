## Summary

Keep open: current main and v2026.6.6 solve the brittle versioned Playwright executable lookup by auto-detecting the Docker cache, but they still keep Docker browsers under `/home/node/.cache/ms-playwright` and do not expose stable `chrome` or `chromium` commands on PATH. The remaining `/opt`/PATH contract is a compatibility-sensitive Docker packaging decision, and the linked implementation attempts are closed or not viable upstream.

## What This Changes

The issue requests moving Docker-baked Playwright browsers out of `/home/node/.cache/ms-playwright` and exposing a stable `chrome`/`chromium` command on PATH.

## Best Possible Solution

Make a maintainer-approved Docker browser packaging decision: either keep the home-cache path with auto-detection, or intentionally move to `/opt` plus PATH symlinks with migration guidance, docs/tests, and Docker proof for browser-enabled and browser-disabled images.
