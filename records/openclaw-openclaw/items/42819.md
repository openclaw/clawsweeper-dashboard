## What This Changes

The issue requests moving Docker Playwright browser files out of `/home/node/.cache/ms-playwright` and exposing a stable browser executable on PATH.

## Best Possible Solution

Maintainers should decide the Docker browser path contract; if changing it, land a focused target-repo fix with migration guidance, stable `chrome`/`chromium` PATH behavior, aligned docs/tests, and Docker proof for browser-enabled, browser-disabled, and upgrade scenarios.
