## Summary

Keep open: current main and the latest release fix the brittle versioned Playwright executable lookup through auto-detection, but the requested Docker image contract still keeps browsers under the hidden home cache and does not expose stable `chrome` or `chromium` commands on PATH.

## What This Changes

The issue asks Docker browser-enabled images to store Playwright browsers outside the hidden home cache and expose a stable Chrome/Chromium command on PATH.

## Best Possible Solution

Keep the shipped Playwright-cache auto-detection, then land a focused Dockerfile/docs/test change only if maintainers approve a new `/opt` browser-store and PATH-command contract with upgrade guidance and Docker proof.

## Reproduction Assessment

Yes, by source inspection: current main still exports `/home/node/.cache/ms-playwright`, lacks a Dockerfile-created `chrome` or `chromium` PATH command, and the test suite expects that current cache path. I did not run a live Docker build in this read-only review.

## Solution Assessment

Unclear until maintainers decide the Docker packaging contract. The shipped auto-detection is the narrow fix for brittle versioned executable paths, while moving the store to `/opt` and adding PATH commands is a compatibility-sensitive image behavior change.
