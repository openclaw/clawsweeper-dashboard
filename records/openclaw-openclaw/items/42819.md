## Summary

Keep open. Current main still installs Playwright Chromium under the hidden `/home/node/.cache/ms-playwright` path, the Docker/browser docs still tell users to persist that path, and there is no Dockerfile-created stable `chrome` or `chromium` command on PATH. The issue is also paired with open implementation PR #42771, so it should remain open until that PR is reviewed, refreshed, landed, replaced, or closed.

## What This Changes

Move Docker Playwright browser files out of the hidden node cache and expose a stable Chrome/Chromium executable path for browser-enabled Docker images.

## Best Possible Solution

Review, refresh, and either land or close #42771 or a maintainer replacement. The desired end state is a Docker browser install path outside `/home/node/.cache`, stable `chrome` or `chromium` commands on PATH, aligned Docker/browser docs, and regression coverage for that Dockerfile contract.
