Keep open. Current main and the latest release still install Docker Playwright Chromium under `/home/node/.cache/ms-playwright` and do not create a stable `chrome` or `chromium` PATH command; the closed implementation PR also shows this still needs an explicit Docker packaging and migration decision.

## What This Changes

The issue requests moving Docker Playwright browser files out of `/home/node/.cache/ms-playwright` and exposing stable Chrome/Chromium commands on PATH.

## Best Possible Solution

Settle a Docker browser path contract, then land a focused replacement with compatibility or migration guidance, aligned docs/tests, stable PATH behavior if desired, and real Docker proof across supported install paths.
