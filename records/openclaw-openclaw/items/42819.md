## Summary

Keep open. Current main and v2026.5.22 still install Docker Playwright Chromium under `/home/node/.cache/ms-playwright` and do not expose Docker-created `chrome` or `chromium` commands on PATH; the merged browser autodetection work only removed OpenClaw's need for a hardcoded revisioned executable path.

## What This Changes

Request to move Docker Playwright browser files out of the hidden home cache and expose a stable Chrome/Chromium executable on PATH.

## Best Possible Solution

Maintainers should decide the Docker browser path contract; if changing it, land a focused Dockerfile/docs/tests update with compatibility guidance and Docker proof for browser-enabled, browser-disabled, upgrade, and PATH alias behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows the Dockerfile still exports the hidden Playwright cache path and lacks Docker-created `chrome` or `chromium` commands on PATH; I did not run a live Docker build in this read-only review.

## Solution Assessment

Unclear. Current main is the narrowest shipped solution for brittle revisioned executable discovery, but moving the Docker store and adding PATH aliases is a compatibility-sensitive packaging decision that needs maintainer approval and migration proof.
