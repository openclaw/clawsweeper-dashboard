## Summary

Keep open. Current main still uses the hidden Playwright cache path in the Docker image and does not expose a Dockerfile-created `chrome` or `chromium` command on PATH; the merged autodetection work reduced the brittle versioned-path problem, but the requested Docker packaging contract remains unresolved and is paired with an open linked PR.

## What This Changes

The issue asks Docker browser builds to place Playwright browsers outside `.cache` and expose a stable browser binary on PATH.

## Best Possible Solution

Review, refresh, and either land or close https://github.com/openclaw/openclaw/pull/42771 or a maintainer replacement so the Docker path, PATH symlink, docs, tests, and image proof agree.

Yes. Source inspection of current main shows Docker still exports `/home/node/.cache/ms-playwright` and only creates the `openclaw` CLI symlink, not `chrome` or `chromium`; I did not run a live Docker build in this read-only review.

## Solution Assessment

Unclear. Current main's autodetection is the narrower fix for brittle versioned Chromium paths, while moving the Docker browser store to `/opt` and adding PATH commands is a broader Docker packaging contract that needs maintainer path-selection and refreshed proof.
