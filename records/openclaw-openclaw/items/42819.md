## Summary

Keep open. Current main and the latest release auto-detect Playwright-managed Chromium from the Docker cache, but they still keep the Docker browser store under `/home/node/.cache/ms-playwright`, document that hidden path, and do not create a stable `chrome` or `chromium` command on PATH; the closed target PRs show the `/opt` migration still needs an explicit compatibility/proof decision.

## What This Changes

Move Docker Playwright browsers out of the hidden node cache and expose a stable Chrome/Chromium command in PATH.

## Best Possible Solution

Make an explicit Docker browser-path contract decision: keep the shipped home-cache autodetection and close as not planned, or approve a migration to `/opt` with compatibility guidance, docs/tests, layout-aware symlinks, and real Docker proof.

## Reproduction Assessment

Yes, for the remaining requested behavior by source inspection: current main still exports the hidden home-cache path and has no Dockerfile-created `chrome`/`chromium` PATH symlink. I did not run a live Docker build in this read-only review.

## Solution Assessment

No, the old `/opt` PR shape is not yet the best proven solution because it was closed with compatibility, ownership, symlink-layout, and proof gaps. A maintainer-approved path contract plus Docker proof is the safer next step.
