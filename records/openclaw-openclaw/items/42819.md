## Summary

Keep open: current main and v2026.6.8 already remove the need to hard-code a versioned Playwright Chromium path through auto-detection, but the requested Docker image contract still keeps browsers under `/home/node/.cache/ms-playwright` and does not expose stable `chrome` or `chromium` commands on PATH.

## What This Changes

The issue asks for Docker Playwright browsers to live outside the hidden home cache and for Chrome/Chromium to be available through a stable PATH command.

## Best Possible Solution

Keep the shipped Playwright-cache auto-detection, then land a focused Dockerfile/docs/test change only if maintainers approve a new `/opt` browser-store and PATH-command contract with upgrade guidance and Docker proof.

## Reproduction Assessment

Yes, by source inspection: current main still exports `/home/node/.cache/ms-playwright`, lacks a Dockerfile-created `chrome` or `chromium` PATH command, and has a test that expects the home-cache path. I did not run a live Docker build in this read-only review.

## Solution Assessment

Unclear until maintainers decide the Docker packaging contract. The shipped auto-detection is the narrow fix for brittle versioned executable paths, while moving the store to `/opt` and adding PATH commands is a compatibility-sensitive image behavior change.
