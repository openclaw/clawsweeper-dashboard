## Summary

Keep open. Current main still uses the Docker Playwright home-cache path and has no Dockerfile-created `chrome` or `chromium` command on PATH; the merged browser autodetection work only addressed the brittle versioned executable discovery part, and the focused target-repo PRs for moving the cache were closed unmerged.

## What This Changes

The issue requests moving Docker's Playwright browser cache out of `/home/node/.cache/ms-playwright` and adding stable `chrome`/`chromium` PATH commands.

## Best Possible Solution

Maintainers should choose a Docker browser path contract explicitly; if moving to `/opt` is desired, land a focused PR with compatibility notes, Dockerfile/docs/test updates, and real Docker proof for browser-enabled, browser-disabled, and persisted-cache upgrade scenarios.

Unclear. Current main's autodetection is the narrow shipped fix for brittle revisioned executable discovery, while moving storage to `/opt` and adding PATH aliases is a compatibility-affecting packaging decision that needs maintainer direction and real container proof.
