## Summary

Keep open. Current main and the latest release still use `/home/node/.cache/ms-playwright` for Docker Playwright browsers and do not expose a Dockerfile-created `chrome` or `chromium` command on PATH; the same-author implementation PR https://github.com/openclaw/openclaw/pull/42771 is still open with closing syntax, so this issue should not be closed before that PR is reviewed, refreshed, landed, or explicitly replaced.

## What This Changes

The issue requests moving Docker Playwright browser assets out of the hidden cache directory and exposing a stable Chrome or Chromium executable on PATH.

## Best Possible Solution

Review, refresh, and either land or close https://github.com/openclaw/openclaw/pull/42771 or a maintainer replacement so the Docker path, PATH symlink, docs, tests, and image proof all agree.

## Reproduction Assessment

Yes. Source inspection of current main and v2026.5.12 shows Docker still exports `/home/node/.cache/ms-playwright` and only creates the `openclaw` CLI symlink, not `chrome` or `chromium`; I did not run a live Docker build in this read-only review.

## Solution Assessment

Unclear. Current main's autodetection is a narrower fix for brittle versioned Chromium paths, while moving the browser store to `/opt` and adding PATH symlinks is a broader Docker packaging contract that needs maintainer path-selection and a refreshed implementation.
