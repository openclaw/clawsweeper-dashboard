## Summary

Keep open: current main still derives Control UI run visibility from message-level origin/provider and uses that flag to suppress WebSocket broadcasts, so a WebChat-bound announce/follow-up run can persist to history without live UI delivery.

## What This Changes

Ensure subagent completion announcements for WebChat-bound parent sessions are pushed live to the Control UI/WebChat session even when the parent run is idle or drained from a queue.

## Best Possible Solution

Centralize Control UI visibility resolution so WebChat-bound sessions stay live-visible for internal announce/follow-up runs while external-channel privacy suppression remains intact, with focused regression coverage.

## Reproduction Assessment

No live reproduction was established in this read-only pass. The current-main source path is reproducible by inspection: an inter-session announce/follow-up run can register isControlUiVisible=false, after which server-chat suppresses live WebSocket events even though history is durable.

## Solution Assessment

Unclear whether the exact pasted patch is the best implementation. A safer repair is a small shared visibility helper used by both registration call sites, with tests proving WebChat-bound session visibility without weakening external-channel suppression.
