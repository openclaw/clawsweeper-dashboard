## Summary

Keep open: current main still has the reported source-level visibility gap, where queued or announce-driven runs can register as not Control UI visible and therefore skip live WebChat broadcasts even though the transcript is later durable.

## What This Changes

The issue asks OpenClaw to push subagent announcement replies to the WebChat/Control UI websocket immediately when the parent orchestrator session is idle.

## Best Possible Solution

Centralize the Control UI visibility decision so both direct and queued announce/followup runs treat WebChat-bound sessions as live-visible without weakening external-channel routing privacy.

## Reproduction Assessment

Yes by source inspection, but not by live execution: an inter-session announce can reach the run-registration path without a webchat origin/provider, register isControlUiVisible=false, and server-chat then suppresses the live websocket events while history remains durable.

## Solution Assessment

Unclear whether the reporter's exact fallback patch is the best shape. A small shared helper used by both visibility registration call sites, with regression coverage for WebChat-bound sessions and hidden external runs, is the safer maintainable fix.

## Repair Work Prompt

Fix the WebChat live-push gap for subagent announcement/followup runs. On current main, queued/direct announce-driven runs can register `isControlUiVisible: false` because visibility is derived only from per-run origin/provider instead of the persisted WebChat-bound session. Add a small shared visibility helper for the relevant registration paths in `src/auto-reply/reply/followup-runner.ts` and `src/auto-reply/reply/agent-runner-execution.ts` or an equivalent narrow seam, using session binding (`channel`, `lastChannel`, `origin.provider`, or existing delivery context helpers) as a fallback only when it is the internal WebChat channel. Preserve suppression for non-Control-UI/external hidden runs. Add focused regression coverage proving WebChat-bound inter-session/subagent announce runs broadcast live and non-WebChat hidden runs stay suppressed. Add a user-facing changelog fix entry. Do not refactor unrelated subagent delivery or external channel routing.
