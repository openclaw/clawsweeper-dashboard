## Summary

Keep open: current main has partial OpenRouter billed-total reconciliation, but provider-reported `usage.cost` and `cost_details` still are not exposed as explicit agent/session runtime metadata.

## What This Changes

Request to capture OpenRouter per-response cost fields and expose per-turn plus cumulative cost to agents or `session_status` so channels can render exact reply costs.

## Best Possible Solution

Add an additive provider-reported usage metadata contract at the OpenAI-compatible/OpenRouter boundary, preserve source and units separately from local estimates, then expose per-turn and cumulative values through agent runtime and session/status tooling.

## Reproduction Assessment

Yes. This is source-reproducible as a feature gap: current main preserves some billed cost on assistant messages, but agent metadata, normalized usage, and session/status output still lack explicit provider-reported OpenRouter cost fields.

## Solution Assessment

Unclear for the exact public API shape. The maintainable direction is additive provider-reported cost metadata with explicit source and units while preserving existing estimated-cost fields.
