## Summary

Keep open: current main still has estimate-based usage/cost surfaces and does not carry OpenRouter provider-reported per-response cost fields into agent runtime metadata, session status, or response footers.

## What This Changes

Expose OpenRouter per-response `usage.cost` and `cost_details.upstream_inference_cost` to agent runtime/session surfaces so replies can show exact provider cost.

## Best Possible Solution

Add a reviewed provider-reported cost metadata contract that preserves OpenRouter credit/upstream cost fields separately from `estimatedCostUsd` and exposes per-turn plus cumulative values through session/runtime APIs.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms OpenRouter usage is normalized to tokens and estimated cost, with no exact provider cost fields reaching agent/session surfaces.

## Solution Assessment

Unclear until maintainers choose the API shape. The narrowest maintainable direction is a distinct provider-reported cost metadata object with explicit source and units, rather than overloading existing estimated-cost fields.
