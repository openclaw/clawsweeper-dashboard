## Summary

Keep open: current main and v2026.6.8 still have local estimated-cost and OpenRouter billed-total reconciliation, but they do not expose OpenRouter provider-reported `usage.cost`, `cost_details.upstream_inference_cost`, source/units metadata, or cumulative per-turn provider cost through agent runtime or `session_status`.

## What This Changes

Request to capture OpenRouter provider-reported per-response usage cost and expose per-turn or cumulative cost metadata to agents and session/status surfaces for user-visible reply footers.

## Best Possible Solution

Define a provider-reported cost metadata contract with explicit source and units, capture it at the OpenAI-compatible/OpenRouter boundary, persist it through transcript/session/runtime metadata, and let `session_status` or a usage tool expose it for channels to render.

## Reproduction Assessment

Yes, as a source-level gap: current main exposes token counters, local estimated cost, and OpenRouter billed-total reconciliation, but the raw OpenRouter `usage.cost` and `cost_details` fields are not carried into agent metadata or `session_status`.

## Solution Assessment

Unclear until maintainers choose the provider-reported cost schema; a Telegram-only footer would be too narrow, while a provider-neutral runtime/session metadata contract is the maintainable direction.
