## Summary

Keep open: current main still captures normalized token usage and locally estimated USD cost, but does not expose OpenRouter provider-reported `usage.cost` or `cost_details` through agent metadata, session status, or response footer data. Related model-routing and billed-cost issues are adjacent rather than a merged replacement for this runtime metadata request.

## What This Changes

The issue requests exposing OpenRouter provider-reported per-response usage cost to agent runtime/session metadata so agents can display exact costs in replies.

## Best Possible Solution

Define an additive provider-reported usage-cost contract at the OpenAI-compatible/OpenRouter boundary, keep it separate from `estimatedCostUsd`, and thread per-turn plus cumulative values through agent metadata, transcripts, session/status tooling, and optional channel footers.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main does not expose OpenRouter `usage.cost` or `cost_details` through agent meta, session state, or response footer metadata.

## Solution Assessment

Yes on the direction, but not as a footer-only patch. The maintainable fix is an additive provider-cost metadata contract that preserves local estimated USD separately from exact OpenRouter charges and leaves display policy to maintainer review.
