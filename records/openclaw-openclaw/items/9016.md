## Summary

Keep open: current main has estimated-cost and transcript usage surfaces, but it still does not expose OpenRouter provider-reported `usage.cost` or `cost_details.upstream_inference_cost` through agent runtime metadata, session rows, or response footers.

## What This Changes

Expose OpenRouter per-response provider cost data to agents and session/status surfaces so replies can include exact OpenRouter cost information.

## Best Possible Solution

Define an additive provider-reported cost contract, capture OpenRouter raw usage at the OpenAI-compatible/OpenRouter boundary, and thread it through transcripts, session metadata, `EmbeddedPiAgentMeta`, `session_status` or a dedicated cost tool, and channel footers.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection on current main gives high-confidence proof that OpenRouter raw cost fields are not exposed on the requested runtime, session, or footer surfaces.

## Solution Assessment

Unclear until maintainers choose the public metadata and cost-authority contract. The narrow maintainable direction is a distinct provider-reported cost object with explicit source, units, and fallback semantics rather than overloading existing `estimatedCostUsd` fields.
