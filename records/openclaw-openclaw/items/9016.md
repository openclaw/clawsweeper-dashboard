## Summary

Keep open: current main still normalizes OpenRouter usage through the shared OpenAI-compatible token/local-estimate path, but it does not preserve provider-reported `usage.cost` or `cost_details.upstream_inference_cost` for agent runtime or session/status metadata.

## What This Changes

Request to expose OpenRouter provider-reported per-message cost data to agents through runtime/session metadata or a usage query surface.

## Best Possible Solution

Define an additive provider-reported usage/cost contract at the OpenAI-compatible/OpenRouter boundary, preserve source and units separately from local estimates, and surface per-turn plus cumulative values through agent runtime and session/status output.

## Reproduction Assessment

Yes. Source inspection shows OpenRouter uses the shared OpenAI-compatible parser, while current runtime/session metadata carries token counters and local estimated cost rather than provider-reported OpenRouter cost fields.

## Solution Assessment

Unclear for the exact public API shape. The maintainable direction is an additive provider-reported cost contract with explicit source and units while preserving existing estimated-cost fields.
