## Summary

Keep open: current main still has estimated/local cost paths, transcript aggregation, and usage surfaces, but it does not expose OpenRouter provider-reported `usage.cost` or `cost_details.upstream_inference_cost` through agent runtime metadata, session rows, or response footer plumbing. Recent discussion correctly distinguishes this request from the closed estimated-cost work and the related open resolved-model and billed-cost follow-ups.

## What This Changes

Expose OpenRouter per-response cost data from provider responses to agents/session metadata so replies and tools can report exact provider cost when available.

## Best Possible Solution

Add a distinct provider-reported cost contract, capture OpenRouter raw usage at the OpenAI-compatible/OpenRouter boundary, and thread per-turn plus cumulative values through transcripts, session metadata, `EmbeddedPiAgentMeta`, `session_status` or a dedicated tool, and optional channel footers.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection on current main gives high-confidence proof that OpenRouter raw cost fields are not exposed on the requested runtime, session, or footer surfaces.

## Solution Assessment

Unclear until maintainers choose the public metadata and cost-authority contract. The narrow maintainable direction is a distinct provider-reported cost object with explicit source, units, and fallback semantics rather than overloading existing `estimatedCostUsd` fields.
