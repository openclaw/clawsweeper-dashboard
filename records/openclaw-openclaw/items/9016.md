## Summary

Keep open: current main supports local estimated-cost reporting and OpenRouter billed-total reconciliation, but it still does not expose provider-reported OpenRouter cost metadata such as usage.cost, cost_details.upstream_inference_cost, source, or units through agent runtime or session_status.

## What This Changes

Expose OpenRouter provider-reported per-response cost data to agent/runtime/session surfaces so agents can render exact per-turn costs.

## Best Possible Solution

Define an additive provider-reported cost metadata contract with explicit source and units, capture OpenRouter streaming and non-streaming cost fields at the provider boundary, persist them through transcript/session/runtime metadata, and render exact costs only when available with estimated-cost fallback.

## Reproduction Assessment

Yes: source inspection shows OpenRouter billed totals can be reconciled internally, while the public agent/session contracts only expose token counts and locally estimated cost fields.

## Solution Assessment

No current implementation fully solves it; the best fix needs a provider-neutral exact-cost metadata contract before Telegram or status rendering consumes it.
