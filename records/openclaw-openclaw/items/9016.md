## Summary

Keep open. Current main has local estimated-cost and usage-summary surfaces, but it still does not expose OpenRouter provider-reported `usage.cost`, `cost_details.upstream_inference_cost`, or cache-write token details through agent runtime or session metadata.

## What This Changes

Expose OpenRouter provider-reported per-response cost fields through agent runtime or session metadata so agents and channels can show exact costs.

## Best Possible Solution

Add an additive provider-reported usage/cost metadata contract at the OpenAI-compatible response boundary, preserve it separately from local estimated USD fields, carry it through transcript/session usage and `EmbeddedPiAgentMeta`, and expose it through `session_status` or a dedicated usage/cost tool.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence way to verify the feature gap: OpenRouter chat responses use the shared OpenAI-compatible path, but current runtime/session metadata carries token counters and estimated cost only.

## Solution Assessment

Unclear for the exact API and UX shape. The maintainable direction is an additive provider-reported cost contract with explicit source and units, while preserving current estimated-cost fields and letting maintainers choose the exposed surfaces.
