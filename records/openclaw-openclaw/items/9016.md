## Summary

Keep open: current `main` and the latest release still expose local estimated cost/session usage surfaces, but they do not carry OpenRouter provider-reported per-response `usage.cost` or `cost_details.upstream_inference_cost` into agent runtime or session metadata.

## What This Changes

Expose OpenRouter per-message provider cost metadata to agents and session status so replies can report exact costs when available.

## Best Possible Solution

Add an additive provider-reported usage/cost metadata contract at the OpenAI-compatible/OpenRouter boundary, preserve it separately from local estimated USD, carry it through transcript/session usage and `EmbeddedPiAgentMeta`, and expose it through `session_status` or a focused usage/cost tool.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence verification path: OpenRouter chat uses the shared OpenAI-compatible parser, but current runtime/session metadata carries token counters and local estimated cost only.

## Solution Assessment

Unclear for the exact API shape. The maintainable direction is an additive provider-reported cost contract with explicit source and units while preserving existing estimated-cost fields.
