## Summary

Keep open: current main still exposes local estimated cost/token metadata, but it does not preserve provider-reported OpenRouter `usage.cost` or `cost_details.upstream_inference_cost` for agent runtime or session/status metadata.

## What This Changes

Expose OpenRouter provider-reported per-response usage cost to agent runtime/session surfaces so agents can report exact reply costs.

## Best Possible Solution

Define an additive provider-reported usage/cost metadata contract at the OpenAI-compatible/OpenRouter boundary, preserve source and units separately from local estimates, and surface per-turn plus cumulative values through agent runtime and session/status output.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence verification path: OpenRouter uses the shared OpenAI-compatible parser, but current runtime/session metadata carries token counters and local estimated cost rather than provider-reported OpenRouter cost fields.

## Solution Assessment

Unclear for the exact public API shape. The maintainable direction is an additive provider-reported cost contract with explicit source and units while preserving existing estimated-cost fields.
