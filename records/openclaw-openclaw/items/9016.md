## Summary

Keep open: current main still has local estimated-cost reporting, but source inspection shows OpenRouter provider-reported `usage.cost` and `cost_details` are not captured into agent metadata, session status, or response footer inputs. The related local-estimate issue is closed as implemented, while the adjacent resolved-model issue remains open and does not replace this provider-cost contract request.

## What This Changes

Expose OpenRouter per-response provider cost data to the agent runtime/session surfaces so agents or channels can report exact reply costs.

## Best Possible Solution

Define an additive provider-reported cost contract at the OpenAI-compatible/OpenRouter boundary, keep exact provider charges separate from `estimatedCostUsd`, and thread per-turn plus cumulative values through agent metadata, transcripts/session rows, `session_status` or a usage tool, and optional channel footers.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main and the latest release do not expose OpenRouter `usage.cost` or `cost_details` through agent metadata, session state, or response footer metadata.

## Solution Assessment

Yes on the requested direction, but not as a Telegram footer-only patch. The maintainable solution is an additive provider-cost metadata contract with explicit source and units, coordinated with related resolved-model and local-cost surfaces.
