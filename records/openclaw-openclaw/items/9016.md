## Summary

Keep open: current main still provides local estimated-cost reporting, but it does not carry OpenRouter provider-reported `usage.cost` or `cost_details.upstream_inference_cost` into agent metadata, session status, or response footer plumbing.

## What This Changes

The issue asks OpenClaw to capture OpenRouter per-response cost fields and expose them to agents/session surfaces so replies, including Telegram replies, can report exact provider cost.

## Best Possible Solution

Define an additive provider-reported cost contract and thread OpenRouter raw usage through the transport, transcript/session metadata, `EmbeddedPiAgentMeta`, `session_status` or a dedicated usage tool, and optional channel footers.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main still lacks OpenRouter raw cost fields on runtime, session, and footer surfaces.

## Solution Assessment

Unclear until maintainers choose the public metadata and cost-authority contract. The narrow maintainable direction is additive provider-reported cost metadata, not overloading existing local estimated-cost fields or applying a Telegram-only footer patch.
