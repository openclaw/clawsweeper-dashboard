## Summary

Keep open: current main and the latest release still expose local token and estimated-cost surfaces, but they do not preserve OpenRouter provider-reported `usage.cost` or `cost_details.upstream_inference_cost` for agent runtime, session status, or reply footer use.

## What This Changes

Expose OpenRouter provider-reported per-response cost data to agent runtime and session/status metadata so agents can report exact reply costs.

## Best Possible Solution

Define an additive provider-reported usage/cost metadata contract at the OpenAI-compatible/OpenRouter boundary, preserve source and units separately from existing estimates, and surface per-turn plus cumulative values through agent runtime and session/status output.

## Reproduction Assessment

Yes. This is a source-level feature-gap check: current main routes OpenRouter chat usage through the shared parser, but runtime/session metadata carries token counters and local estimated cost rather than OpenRouter provider-reported cost fields.

## Solution Assessment

Unclear for the exact public API shape. The maintainable direction is additive provider-reported cost metadata with explicit source and units while preserving existing estimated-cost fields.
