## Summary

Keep open: current main still has estimated/local cost reporting and usage aggregation, but it does not expose OpenRouter provider-reported `usage.cost` or `cost_details.upstream_inference_cost` through agent metadata, session status, or reply footer plumbing. The live discussion and related open items distinguish this from the closed estimated-cost work.

## What This Changes

Expose OpenRouter provider-reported per-response cost fields to agent runtime/session metadata and optional reply footers.

## Best Possible Solution

Define an additive provider-reported cost contract and thread OpenRouter raw usage through transport, transcript/session metadata, `EmbeddedPiAgentMeta`, `session_status` or a dedicated tool, and optional channel footers.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main still lacks the requested OpenRouter raw cost fields on runtime, session, and footer surfaces.

## Solution Assessment

Unclear until maintainers choose the public metadata and cost-authority contract. The narrow maintainable direction is additive provider-reported cost metadata, not overloading existing `estimatedCostUsd` or local `usage.cost` estimates.
