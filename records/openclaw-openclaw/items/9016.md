## Summary

Keep open: current main still computes local estimated cost from normalized token usage, but it does not carry OpenRouter provider-reported `usage.cost` or `cost_details` into agent metadata, session status, or response-footers.

## What This Changes

Expose OpenRouter provider-reported per-response cost to agent runtime/session metadata so agents and channels can show exact per-reply cost.

## Best Possible Solution

Define an additive provider-reported cost contract at the OpenAI-compatible/OpenRouter boundary, keep exact provider charges separate from `estimatedCostUsd`, and thread per-turn plus cumulative values through agent metadata, transcripts/session rows, `session_status` or a usage tool, and optional channel footers.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main does not expose OpenRouter `usage.cost` or `cost_details` through agent metadata, session state, or response footer metadata.

## Solution Assessment

Yes on the requested direction, but not as a Telegram footer-only patch. The maintainable solution is an additive provider-cost metadata contract with explicit source and units, coordinated with related resolved-model and local-cost surfaces.
