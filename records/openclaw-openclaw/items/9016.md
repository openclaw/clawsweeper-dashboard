## Summary

Keep open: current main still has local estimated-cost reporting and transcript cost aggregation, but it does not preserve OpenRouter provider-reported `usage.cost` / `cost_details` into agent runtime metadata, session status, or reply footers.

## What This Changes

Expose OpenRouter provider-reported per-response usage cost to agent runtime/session metadata so agents and channels can display exact reply costs.

## Best Possible Solution

Add a reviewed provider-reported cost metadata contract that preserves OpenRouter exact cost fields separately from `estimatedCostUsd` and exposes per-turn plus cumulative values through runtime/session APIs.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms the requested OpenRouter provider-cost fields are not exposed on the relevant runtime, session, or footer surfaces.

## Solution Assessment

Unclear until maintainers choose the API contract. The narrow maintainable direction is a distinct provider-reported cost object with explicit source and units rather than overloading existing estimated-cost fields.
