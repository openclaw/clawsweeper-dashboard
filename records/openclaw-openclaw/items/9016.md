## Summary

Keep open: current main has local estimated-cost reporting and transcript usage aggregation, but it still does not expose OpenRouter provider-reported per-response `usage.cost` or `cost_details.upstream_inference_cost` through agent runtime metadata, session status, or reply footer plumbing.

## What This Changes

Expose OpenRouter per-message provider cost fields to agent runtime/session metadata so agents and channels can display exact per-reply cost.

## Best Possible Solution

Define an additive provider-reported cost contract, capture OpenRouter `usage.cost`, `cost_details.upstream_inference_cost`, and cache-write/read details at the OpenAI-compatible boundary, and thread them through transcript/session usage, agent metadata, and `session_status` or a dedicated usage tool.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main still lacks OpenRouter raw provider-cost fields on runtime, session, and footer surfaces.

## Solution Assessment

Unclear until maintainers choose the public metadata and cost-authority contract. The maintainable direction is additive provider-reported cost metadata, not overloading existing local estimated-cost fields or applying a Telegram-only footer patch.
