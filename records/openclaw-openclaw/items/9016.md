## Summary

Keep this open. Current main has adjacent estimated-cost/session usage support, but it still does not expose OpenRouter provider-reported per-response `usage.cost` or `cost_details` through agent runtime/session metadata, and the related open issues cover adjacent resolved-model, billed-cost reconciliation, or ledger work rather than replacing this request.

## What This Changes

Expose OpenRouter provider-reported per-response cost data to agent runtime/session surfaces so agents and channels can display exact costs.

## Best Possible Solution

Define an additive provider-reported cost metadata contract, capture OpenRouter raw usage at the OpenAI-compatible response/stream boundary, keep it separate from `estimatedCostUsd`, and expose per-turn/cumulative values through agent runtime and session/status surfaces.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main lacks the requested OpenRouter provider-reported cost carrier in transport, agent metadata, session persistence, and reply footer paths.

## Solution Assessment

Yes on the direction, but it needs maintainer API/product review before implementation. The maintainable path is an additive provider-cost metadata contract rather than reinterpreting `estimatedCostUsd` or patching only Telegram/footer text.
