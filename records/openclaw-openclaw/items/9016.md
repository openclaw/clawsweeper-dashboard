## Summary

Keep open: current main has local estimated-cost and session usage surfaces, but it still does not expose OpenRouter provider-reported per-response `usage.cost` or `cost_details.upstream_inference_cost` through agent runtime or session metadata.

## What This Changes

Request to capture OpenRouter provider-reported per-response cost data and expose it to agents/session status so replies can include exact OpenRouter cost information.

## Best Possible Solution

Define an additive provider-reported usage/cost metadata contract at the OpenAI-compatible/OpenRouter response boundary, preserve it separately from local estimates, and surface it consistently through agent runtime, transcript/session usage, and status/tool output.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence verification path: OpenRouter chat uses the shared OpenAI-compatible parser, but current runtime/session metadata carries token counters and local estimated cost only.

## Solution Assessment

Unclear for the exact public API shape. The maintainable direction is an additive provider-reported cost contract with explicit source and units while preserving existing estimated-cost fields.
