## Summary

Keep open: current main and v2026.6.6 have adjacent OpenRouter cost recording and local estimated-cost display, but they still do not expose provider-reported OpenRouter cost details, source/units, or per-turn exact charges to agent runtime or `session_status`.

## What This Changes

Expose OpenRouter provider-reported per-response cost fields through agent runtime or session metadata so agents can append exact cost information to replies.

## Best Possible Solution

Capture OpenRouter raw usage at the OpenAI-compatible response/stream boundary, preserve provider-reported cost with explicit source and units alongside current estimated USD cost, carry it through transcript/session and `EmbeddedAgentMeta`, then expose per-turn and cumulative values through `session_status` or a dedicated usage tool.

## Reproduction Assessment

Yes, as a source-level gap rather than a failing bug: current main records/reconciles some OpenRouter cost totals, but the runtime metadata and status paths still expose token fields and local estimates instead of provider-reported cost details.

## Solution Assessment

Unclear until maintainers choose the provider-neutral schema; the maintainable direction is to expose source/units-aware runtime usage metadata first, then let Telegram and other channels render it.
