## Summary

Keep open: current main has local estimated-cost and transcript aggregation surfaces, but it still does not preserve OpenRouter provider-reported `usage.cost` or `cost_details` into agent runtime metadata, session rows, or reply footers.

## What This Changes

Expose OpenRouter per-response usage cost to agent runtime/session metadata or a tool so channel replies can show exact per-turn costs.

## Best Possible Solution

Add a reviewed provider-reported cost metadata contract that preserves OpenRouter exact cost fields separately from local estimates and exposes per-turn plus cumulative values through runtime/session APIs.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms the requested OpenRouter provider-cost fields are not exposed on the relevant runtime, session, or footer surfaces.

## Solution Assessment

Unclear until maintainers choose the API contract. The narrow maintainable direction is a distinct provider-reported cost object with explicit source, units, and fallback semantics rather than overloading existing estimated-cost fields.
