## Summary

Keep open. Current main has adjacent estimated-cost reporting, transcript aggregation, `/usage cost`, and session `estimatedCostUsd`, but it still does not preserve OpenRouter provider-reported `usage.cost`, `usage.cost_details.upstream_inference_cost`, or `prompt_tokens_details.cache_write_tokens` for agent/runtime metadata.

## What This Changes

Request that OpenClaw capture OpenRouter response cost fields and make per-turn or cumulative costs available to agents and channels such as Telegram.

## Best Possible Solution

Add an additive provider-reported-cost path that captures OpenRouter raw usage at the OpenAI-compatible response/stream boundary, carries source and units through transcript/session usage and `EmbeddedPiAgentMeta`, and lets channels render exact OpenRouter values when available while retaining local estimates as fallback.

## Reproduction Assessment

Yes at source level. Feed the shared OpenAI-compatible stream parser a final OpenRouter-style usage chunk containing numeric `cost`, `cost_details.upstream_inference_cost`, and `prompt_tokens_details.cache_write_tokens`; current main normalizes token counts and estimated local cost but drops those provider fields.

## Solution Assessment

Yes. The narrow maintainable direction is additive capture and propagation of provider-reported OpenRouter usage metadata, not replacing existing estimated USD fields or relying only on post-hoc dashboards.
