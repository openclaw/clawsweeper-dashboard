## Summary

Keep open: current main has local estimated-cost reporting and aggregated usage surfaces, but it still does not carry OpenRouter provider-reported per-response cost fields into agent runtime metadata, session status, or response footers.

## What This Changes

Expose OpenRouter per-response usage cost metadata to agents and session/status surfaces so replies can include exact provider costs.

## Best Possible Solution

Capture provider-reported OpenRouter cost metadata at the OpenAI-compatible response/stream boundary, keep it distinct from estimated USD pricing, and expose per-turn plus cumulative values through reviewed runtime/session APIs.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence that the requested exact OpenRouter cost metadata is still missing from runtime and session surfaces.

## Solution Assessment

Unclear until maintainers choose the API shape. The narrowest maintainable direction is a distinct provider-reported cost metadata object with explicit source and units, rather than overloading existing estimated-cost fields.
