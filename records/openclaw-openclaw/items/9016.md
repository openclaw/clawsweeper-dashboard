## Summary

Keep open. Current main has local estimated-cost reporting and transcript-backed cost summaries, but it still does not carry OpenRouter provider-reported `usage.cost` or `cost_details` into agent runtime/session metadata; the related issues and PRs cover estimates, resolved-model attribution, or billed-cost reconciliation rather than replacing this exact request.

## What This Changes

Expose OpenRouter provider-reported per-response cost data to agent runtime/session surfaces so agents can show exact per-turn costs, including in Telegram replies.

## Best Possible Solution

Add an additive provider-reported usage/cost metadata contract at the OpenAI-compatible response/stream boundary, preserve it separately from local estimated USD fields, carry it through transcripts/session usage and `EmbeddedPiAgentMeta`, and expose it through `session_status` or a dedicated usage/cost tool with clear fallback to estimates.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence way to verify the feature gap: OpenRouter usage reaches shared OpenAI-compatible parsing, but current runtime/session metadata carries token counters and estimated cost only; no live OpenRouter replay was needed to prove the missing plumbing.

## Solution Assessment

Unclear for the exact API and UX shape. The maintainable direction is an additive provider-reported cost contract with explicit source and units, while preserving current estimated-cost fields and letting maintainers choose the exposed surfaces.
