## Summary

Keep open: current main still exposes local estimated-cost surfaces, but not OpenRouter provider-reported `usage.cost` or `cost_details` in agent runtime metadata, session rows, or response footers.

## What This Changes

Expose OpenRouter provider-reported per-response cost data to agent runtime and session surfaces so replies can show exact OpenRouter costs.

## Best Possible Solution

Add an additive provider-reported cost metadata contract with explicit source and units, then thread per-turn and cumulative OpenRouter cost data through transport, transcript/session metadata, runtime tools, and channel footers.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high-confidence proof that the requested OpenRouter provider-cost fields are not exposed on the runtime, session, or footer surfaces.

## Solution Assessment

Unclear until maintainers choose the public metadata contract. The narrow maintainable direction is a distinct provider-reported cost object with explicit source, units, and fallback semantics rather than overloading existing estimated-cost fields.
