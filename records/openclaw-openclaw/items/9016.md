## Summary

Keep open: current main has partial OpenRouter billed-cost total reconciliation, but it still does not expose provider-reported OpenRouter cost details, source/units, or per-turn exact charges through agent runtime metadata or `session_status`.

## What This Changes

The issue requests exposing OpenRouter per-response `usage.cost` and related cost details to agent runtime or session metadata so agents can report exact costs in replies.

## Best Possible Solution

Define a provider-neutral cost metadata contract with source and units, capture OpenRouter raw usage at the OpenAI-compatible response boundary, persist it through transcript/session/runtime metadata, and expose it through `session_status` or a dedicated usage tool for channels to render.

## Reproduction Assessment

Yes, at source level: current main can reconcile an OpenRouter aggregate billed cost total, but the parser, runtime metadata, session persistence, and `session_status` paths still omit provider-reported cost details and source metadata.

## Solution Assessment

Unclear until maintainers choose the provider-neutral schema; the maintainable direction is to expose source/units-aware runtime usage metadata first, then let Telegram and other channels render it.
