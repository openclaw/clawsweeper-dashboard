## Summary

Keep open: current main has local estimated-cost reporting and usage summaries, but it still does not expose OpenRouter provider-reported `usage.cost` or `cost_details.upstream_inference_cost` through agent runtime, session status, or response footer plumbing. The closed estimated-cost issue and the related open resolved-model/billed-cost issues are adjacent, not replacements for this provider-cost metadata request.

## What This Changes

The issue requests exposing OpenRouter provider-reported per-message usage cost to agent runtime/session surfaces so agents can report costs in replies.

## Best Possible Solution

Define an additive provider-reported cost contract, capture OpenRouter `usage.cost`, `cost_details.upstream_inference_cost`, and cache-write/read details at the OpenAI-compatible boundary, preserve them separately from local `estimatedCostUsd`, and thread them through transcript/session usage, `EmbeddedPiAgentMeta`, and `session_status` or a dedicated usage tool.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main still lacks OpenRouter raw provider-cost fields on runtime, session, and footer surfaces.

## Solution Assessment

Unclear until maintainers choose the public metadata and cost-authority contract. The narrow maintainable direction is additive provider-reported cost metadata, not overloading existing local estimated-cost fields or applying a Telegram-only footer patch.
