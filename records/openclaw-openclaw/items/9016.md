## Summary

Keep open: current main still exposes local estimated-cost surfaces, but not OpenRouter provider-reported `usage.cost` or `cost_details` through agent runtime metadata, session rows, or reply footers. The closed estimate work in https://github.com/openclaw/openclaw/issues/9696 and the related open items at https://github.com/openclaw/openclaw/issues/7006, https://github.com/openclaw/openclaw/issues/51441, and https://github.com/openclaw/openclaw/issues/68066 do not replace this runtime metadata request.

## What This Changes

Expose OpenRouter provider-reported usage cost and cost details to agent runtime/session metadata so agents and channels can show per-reply cost.

## Best Possible Solution

Add an additive provider-reported cost metadata contract with explicit source and units, then capture OpenRouter raw usage at the OpenAI-compatible response boundary and thread it through transcript/session metadata, `EmbeddedPiAgentMeta`, `session_status` or a dedicated cost tool, and channel footers.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high-confidence proof that the requested OpenRouter provider-cost fields are not exposed on runtime, session, or footer surfaces.

## Solution Assessment

Unclear until maintainers choose the public metadata and cost-authority contract. The narrow maintainable direction is a distinct provider-reported cost object with explicit source, units, and fallback semantics rather than overloading existing `estimatedCostUsd` fields.
