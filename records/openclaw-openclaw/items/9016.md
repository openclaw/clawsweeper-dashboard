## Summary

Keep open: current main still has local estimated-cost reporting and usage aggregation, but it does not expose OpenRouter provider-reported `usage.cost` or `cost_details` through agent runtime metadata, session rows, or response footers.

## What This Changes

Expose OpenRouter provider-reported per-message cost data to agent runtime/session metadata and channel reply surfaces.

## Best Possible Solution

Add an additive provider-reported cost metadata contract with explicit source and units, capture OpenRouter raw usage at the OpenAI-compatible response boundary, and thread it through transcripts, session metadata, `EmbeddedPiAgentMeta`, `session_status` or a dedicated cost tool, and channel footers.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high-confidence proof that OpenRouter raw cost fields are not exposed on the requested runtime, session, or footer surfaces.

## Solution Assessment

Unclear until maintainers choose the public metadata and cost-authority contract. The narrow maintainable direction is a distinct provider-reported cost object with explicit source, units, and fallback semantics rather than overloading existing `estimatedCostUsd` fields.
