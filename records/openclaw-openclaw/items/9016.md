## Summary

Keep open: current main and v2026.5.12 still expose local estimated-cost surfaces, but they do not carry OpenRouter provider-reported `usage.cost` or `cost_details` into agent metadata, session status, or reply footer inputs. The related open model-routing and billed-cost issues are adjacent, not replacements for this runtime metadata contract request.

## What This Changes

Expose OpenRouter per-response cost data to agent runtime/session surfaces so agents or channels can report exact provider costs per reply.

## Best Possible Solution

Define an additive provider-reported cost contract at the OpenAI-compatible/OpenRouter boundary, keep exact provider charges separate from `estimatedCostUsd`, and thread per-turn plus cumulative values through agent metadata, transcripts/session rows, `session_status` or a usage tool, and optional channel footers.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request, but source inspection gives high-confidence proof of the current gap. Current main and v2026.5.12 do not expose OpenRouter `usage.cost` or `cost_details` through agent metadata, session state, or response footer metadata.

## Solution Assessment

Yes on the requested direction, but not as a Telegram-footer-only patch. The maintainable solution is an additive provider-cost metadata contract with explicit source and units, coordinated with related resolved-model and local-cost surfaces.
