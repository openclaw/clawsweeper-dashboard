## Summary

Keep open: current main still has local estimated-cost and transcript-summary surfaces, but it does not expose OpenRouter provider-reported per-response cost fields through agent runtime or session metadata.

## What This Changes

Request to capture OpenRouter per-response `usage.cost` data and expose per-turn or cumulative cost to agents and session/status surfaces for display in replies.

## Best Possible Solution

Add an additive provider-reported usage/cost metadata contract at the OpenAI-compatible response boundary, preserve it separately from local estimated USD fields, carry it through transcript/session usage and `EmbeddedPiAgentMeta`, and expose it through `session_status` or a dedicated usage/cost tool.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence verification path: OpenRouter chat uses the shared OpenAI-compatible path, but current runtime/session metadata carries token counters and local estimated cost only.

## Solution Assessment

Unclear for the exact API and UX shape. The maintainable direction is an additive provider-reported cost contract with explicit source and units while preserving current estimated-cost fields.
