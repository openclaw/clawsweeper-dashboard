## Summary

Keep open: current main and the latest release still expose local estimated cost surfaces, but they do not carry OpenRouter's provider-reported per-response `usage.cost` or `cost_details.upstream_inference_cost` into agent runtime or session/status metadata.

## What This Changes

The issue requests exposing OpenRouter provider-reported per-response cost data to agent runtime and session/status surfaces so agents can display exact cost information.

## Best Possible Solution

Define an additive provider-reported usage/cost metadata contract at the OpenAI-compatible/OpenRouter boundary, preserve it separately from local estimates, and surface it consistently through agent runtime, transcript/session usage, and status/tool output.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence verification path: OpenRouter chat uses the shared OpenAI-compatible parser, but current runtime/session metadata carries token counters and local estimated cost only.

## Solution Assessment

Unclear for the exact public API shape. The maintainable direction is an additive provider-reported cost contract with explicit source and units while preserving existing estimated-cost fields.
