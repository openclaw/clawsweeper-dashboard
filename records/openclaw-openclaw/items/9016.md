## Summary

Keep open: current main and v2026.6.6 only partially handle OpenRouter billing by reconciling assistant transcript cost totals; they still do not expose provider-reported OpenRouter cost, source/units, or upstream cost details through agent runtime metadata or session_status.

## What This Changes

The issue requests exposing OpenRouter provider-reported per-response usage cost to agent runtime/session metadata so agents can report exact costs in replies.

## Best Possible Solution

Define a provider-neutral reported-cost record separate from local estimates, capture OpenRouter raw usage at the OpenAI-compatible response boundary, and expose per-turn and cumulative values through agent metadata and session_status before adding channel presentation.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.6 can be inspected to show OpenRouter billed totals may reach assistant message usage while agent metadata and session_status still expose only token and estimated-cost surfaces.

## Solution Assessment

Unclear pending maintainer direction. A Telegram-only footer or reusing estimatedCostUsd would be the wrong layer; the maintainable path is a generic provider-reported cost contract consumed by runtime metadata, status tools, and channels.
