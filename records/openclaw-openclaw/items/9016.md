## Summary

Keep open: current main and the latest release still convert OpenRouter response usage through the shared token/local-estimate path and do not expose provider-reported `usage.cost` or `cost_details` to agent runtime or session/status metadata. Related open items cover resolved-model, structured session usage, and billed-total reconciliation, but none has merged this exact runtime metadata contract.

## What This Changes

Request to capture OpenRouter response cost fields and expose exact per-turn or cumulative costs to agents/session status so channels can render them.

## Best Possible Solution

Define additive provider-reported usage/cost metadata at the OpenAI-compatible/OpenRouter boundary, preserve source and units separately from local estimates, and expose per-turn plus cumulative values through agent runtime and session/status output.

## Reproduction Assessment

Yes. This is a source-level feature-gap check: OpenRouter uses the shared parser, while current runtime/session metadata carries token counters and local estimated cost rather than provider-reported OpenRouter cost fields.

## Solution Assessment

Unclear for the exact public API shape. The maintainable direction is additive provider-reported cost metadata with explicit source and units while preserving existing estimated-cost fields.
