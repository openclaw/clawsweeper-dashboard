## Summary

Keep open: current main still has estimated usage cost plumbing, but source inspection shows no OpenRouter provider-reported `usage.cost` or `cost_details.upstream_inference_cost` contract exposed to agent runtime/session metadata. The related estimated-cost and resolved-model items are adjacent rather than replacements for this exact OpenRouter runtime cost metadata request.

## What This Changes

Request to capture OpenRouter per-response cost fields and expose per-turn and cumulative cost data to agent runtime surfaces such as `session_status` or a dedicated usage tool.

## Best Possible Solution

Define an additive provider-reported cost contract, capture OpenRouter cost fields at the OpenAI-compatible response/stream boundary, keep them distinct from local `estimatedCostUsd`, and thread them through transcript/session usage plus agent runtime metadata for tools or channel rendering.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main still lacks OpenRouter raw provider-cost fields on the runtime, session, and footer surfaces.

## Solution Assessment

Unclear until maintainers settle the public cost metadata contract. The maintainable direction is additive provider-reported cost metadata, not overloading current estimated-cost fields or adding a Telegram-only formatter.
