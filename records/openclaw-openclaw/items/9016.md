## Summary

Keep open: current main has adjacent estimated-cost reporting, but it still does not carry OpenRouter's provider-reported `usage.cost` or `cost_details` into agent runtime metadata, `session_status`, or reply footers.

## What This Changes

Expose provider-reported OpenRouter per-response cost metadata to agent runtime/session status so channels such as Telegram can append exact cost to replies.

## Best Possible Solution

Add a provider-reported cost metadata path at the OpenAI-compatible response/stream boundary, keep it separate from `estimatedCostUsd`, and expose per-turn plus cumulative values through agent metadata and `session_status` with estimated-cost fallback behavior documented.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that the requested exact OpenRouter cost metadata is still missing from the current runtime/session surfaces.

## Solution Assessment

Unclear until maintainers choose the API shape. The narrowest maintainable direction is to preserve provider-reported cost as a distinct metadata object rather than overloading the existing estimated USD fields.
