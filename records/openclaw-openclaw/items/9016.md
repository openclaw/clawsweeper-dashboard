## Summary

Keep open: current main and the latest release still expose normalized token usage and local estimated USD cost, but do not carry OpenRouter provider-reported per-response `usage.cost` or `cost_details` into agent runtime, session status, or reply footer metadata.

## What This Changes

Expose OpenRouter provider-reported per-message cost data to agents and session/status surfaces so replies can show exact OpenRouter usage cost.

## Best Possible Solution

Define an additive provider-reported usage-cost contract at the OpenAI-compatible/OpenRouter boundary, keep it separate from `estimatedCostUsd`, and thread per-turn and cumulative values through agent metadata, session/status tooling, transcripts, and optional channel footers.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high-confidence proof that current main does not expose OpenRouter `usage.cost` or `cost_details` through agent meta, session state, or response footers.

## Solution Assessment

Yes on the direction, but not as a footer-only patch. The maintainable fix is an additive provider-cost metadata contract that preserves local estimated USD separately from exact OpenRouter charges and leaves display policy to maintainer review.
