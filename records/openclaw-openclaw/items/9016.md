## Summary

Keep open: current main and v2026.6.6 have partial OpenRouter billed-cost reconciliation in assistant message usage, but they still do not expose OpenRouter provider-reported `usage.cost`, `cost_details.upstream_inference_cost`, or source/units through agent runtime metadata or `session_status`.

## What This Changes

Request to capture OpenRouter per-response usage cost and make it available to agents/session tooling so channels such as Telegram can display exact per-reply cost.

## Best Possible Solution

Define a provider-neutral reported-cost record separate from local estimates, capture OpenRouter raw usage at the OpenAI-compatible response boundary, and expose per-turn and cumulative values through agent metadata and `session_status` before adding channel presentation.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.6 can be inspected to show OpenRouter billed total may reach assistant message usage, while agent metadata and `session_status` still expose only token/estimated-cost surfaces.

## Solution Assessment

No, a Telegram-only footer or reusing `estimatedCostUsd` would be the wrong layer. The maintainable path is a generic provider-reported cost contract consumed by runtime metadata, status tools, and channels.
