## Summary

Keep open: current main still has local estimated-cost and session usage surfaces, but it does not expose OpenRouter provider-reported per-response `usage.cost` or `cost_details.upstream_inference_cost` through agent runtime or session metadata.

## What This Changes

The issue requests capturing OpenRouter per-response cost fields and surfacing them to agents/session status so replies can include exact provider cost.

## Best Possible Solution

Add an additive provider-reported usage/cost metadata contract at the OpenAI-compatible/OpenRouter response boundary, preserve it separately from local estimated USD, carry it through transcript/session usage and `EmbeddedPiAgentMeta`, and expose it through `session_status` or a dedicated usage/cost tool.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence verification path: OpenRouter chat uses the shared OpenAI-compatible parser, but current runtime/session metadata carries token counters and local estimated cost only.

## Solution Assessment

Unclear for the exact API shape. The maintainable direction is an additive provider-reported cost contract with explicit source and units while preserving existing estimated-cost fields.
