## Label Justifications

- `P1`: The report describes a reproducible channel spam loop that posted 80+ duplicate messages and degraded gateway liveness during real operations.
- `impact:message-loss`: The affected behavior is duplicated user-visible outbound channel messages from repeated message-tool sends.
- `impact:session-state`: The reported loop depends on transcript delivery-mirror state being interpreted as fresh model context, even though current main now filters that replay path.
- `impact:crash-loop`: The incident included degraded liveness and high event-loop delay, with the report calling out potential gateway OOM if the loop continues.

## Summary

Keep open: current main has useful partial coverage for this incident family, but it does not yet implement the requested default gateway-level content dedupe, low-threshold per-turn abort, or a maintainer-approved cross-agent stop policy.

## What This Changes

The issue requests structural protections against repeated outbound message-tool loops, including content-based duplicate suppression, model-hidden delivery mirrors, a duplicate-call abort guard, and operational agent-to-agent stop access.

## Best Possible Solution

Keep the existing delivery-mirror replay filter, then split the remaining work into maintainer-approved core changes for safe outbound duplicate suppression or loop abort semantics, with any ag-manudis access handled through explicit operator configuration or the per-agent session-access design in https://github.com/openclaw/openclaw/pull/59170.

## Reproduction Assessment

Yes for the remaining source-level gap, but not for the full live incident. Current main still has no default 30s content-based outbound message dedupe or ERR_LOOP_GUARD_TRIPPED-style per-turn abort, while the original Discord/kimi-code loop was not replayed live in this review.

## Solution Assessment

Unclear as a single bundled fix. The delivery-mirror replay part is already handled on current main, but default send suppression and cross-agent stop access need a maintainer product/security decision before implementation.
