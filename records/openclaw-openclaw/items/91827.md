## What This Changes

Issue #91827 requests structural protection against repeated message-tool delivery loops, including content-based message deduplication, hiding delivery-mirror entries from model replay, a duplicate-call guard, and an operational `agentToAgent` allowlist update.

## Best Possible Solution

Keep this as the canonical P1 tracking item for the remaining structural work: decide the gateway/send dedup contract, keep transcript-only replay filtering intact, and split the `agentToAgent` operational access change into an explicit security-reviewed config decision if needed.

## Solution Assessment

The structural direction is valid, but bundling gateway dedup, replay filtering, per-turn abort behavior, and cross-agent default access is not a safe automatic fix; maintainers should split runtime safety from the authorization/default decision.
