## Summary

Keep open. Current main and v2026.6.5 already cover important pieces, including model-replay filtering for `delivery-mirror` rows and stable loop hashes for send results with volatile IDs, but the requested default content-based duplicate-send guard and cross-agent abort-control policy remain unresolved.

## What This Changes

The issue requests structural protection against repeated `message` tool delivery loops, model-hidden delivery mirrors, a per-turn duplicate-call guard, and an `agentToAgent` allowlist path for operational aborts.

## Best Possible Solution

Keep this issue as the structural tracker, preserve the shipped partial protections, and split any remaining work into maintainer-approved PRs for default duplicate-send policy, live regression proof, and cross-agent abort authorization.
