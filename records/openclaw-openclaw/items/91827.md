## Summary

Keep open: current main already addresses the delivery-mirror replay problem and part of the volatile-messageId loop-detection failure, but the issue still asks for broader gateway/runtime loop prevention that is not fully implemented or cleanly owned by an existing merged change.

## What This Changes

The issue requests structural mitigation for model self-echo message loops: content-based message deduplication, model-hidden delivery mirrors, a duplicate-call loop guard, and downstream agent-to-agent stop access.

## Best Possible Solution

Keep the issue as the umbrella for the remaining generic gateway/runtime spam-loop design, while treating delivery-mirror replay filtering as already done and moving downstream ag-manudis access to local configuration or a separate product decision.

## Reproduction Assessment

Yes, source inspection supports the remaining class: current main lacks the requested gateway content-dedup and hard duplicate-call budget, though the exact Argus incident was not replayed live.

## Solution Assessment

Partly: hiding delivery mirrors from replay and normalizing volatile send results are the right structural pieces already on main, but the remaining gateway dedupe and abort guard need maintainer design to avoid suppressing legitimate messages or duplicating #9912.
