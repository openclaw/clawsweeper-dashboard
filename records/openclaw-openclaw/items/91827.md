## Summary

## What This Changes

The issue asks for generic protections against repeated message-tool delivery loops: content-based duplicate suppression, hiding delivery mirrors from model replay, a per-turn duplicate-call guard, and operational A2A stop/abort access.

## Best Possible Solution

Use this as a maintainer-reviewed tracker for the remaining generic protections: decide whether a default delivery-layer duplicate-send gate belongs in core, keep delivery mirrors model-hidden, and route stop/abort access through the generic A2A policy work rather than a downstream hardcoded agent id.
