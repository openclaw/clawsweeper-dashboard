## Summary

Current `main` already fixes the reported coexistence failure: npm plugin installs now update the managed npm-root manifest and run `npm install` from that manifest, so installing Discord no longer prunes already managed sibling plugins such as lossless-claw.

## What This Changes

Issue reports that installing the npm Discord plugin and the third-party lossless-claw plugin on OpenClaw 2026.5.2 causes the previously installed plugin to disappear from plugin discovery/config.

## Best Possible Solution

Keep the current manifest-driven npm install path and ship it in the next release; no Discord- or lossless-claw-specific core change is needed.

## Reproduction Assessment

Yes. The issue logs and the earlier fixed report #76571 give a high-confidence sequential npm-plugin install reproduction on v2026.5.2; I verified the current-main source path and regression test rather than running installer commands in this read-only review.

## Solution Assessment

Yes. Installing from the managed npm-root manifest is the narrow generic fix because it preserves all recorded plugin package dependencies without special-casing Discord, lossless-claw, or any package scope.
