## Summary

Current main already replaces the v2026.5.2 npm-plugin install behavior with a managed npm root that preserves existing plugin dependencies and merged install records, so the reported alternating Discord/lossless-claw disappearance is addressed on main but not yet in a shipped release.

## What This Changes

The issue asks for npm-installed lossless-claw and Discord plugins to remain installed and enabled together instead of alternating stale plugin warnings after each install.

## Best Possible Solution

Keep the current managed npm-root implementation and let the next release carry it to users who hit this in v2026.5.2.

## Reproduction Assessment

Yes for the reported release path: the issue logs show alternating stale plugin warnings after npm plugin installs, and the v2026.5.2 tree lacks the managed npm-root implementation. I did not establish a failing current-main reproduction because current main contains the replacement path.

## Solution Assessment

Yes. The current fix is the narrow maintainable path: preserve all managed npm dependencies, verify the installed package-lock entry, and merge plugin install records instead of changing Discord or lossless-claw behavior.
