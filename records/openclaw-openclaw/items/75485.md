Keep open. Current main still has a packaging/docs/runtime mismatch: the npm package allowlist omits the sandbox setup assets while user-facing sandbox docs, runtime errors, and doctor repair flow point npm-installed users at those paths. The report is valid, but the requested “ship all scripts” direction is broader than the maintainable fix.

## What This Changes

The issue asks for the published npm package to include missing repository scripts so documented sandbox setup and package script commands work after a global npm install.

## Best Possible Solution

Provide one supported npm-installed sandbox setup path. Either ship only the documented sandbox setup scripts, their helper script, Dockerfiles, and browser entrypoint with release-pack tests, or move image setup behind a packaged openclaw sandbox/doctor command and update docs/runtime hints accordingly.
