## Summary

Keep open. Current main still lacks `openclaw cron add --dry-run`, and this PR is the canonical implementation path, but it still needs contributor-provided real CLI behavior proof before merge and should document the new flag.

## What This Changes

The branch adds `--dry-run` to `openclaw cron add`, prints resolved params through runtime output helpers before the `cron.add` RPC, and adds CLI tests for text, JSON, and no-RPC dry-run paths.

## Best Possible Solution

Land the narrow CLI-only preview after real CLI proof is posted and the cron docs mention the flag, without changing scheduler or gateway contracts.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main has no `cron add --dry-run` flag and always calls `cron.add` after params construction.

## Solution Assessment

Mostly yes. Building the existing params object, printing through runtime output helpers, and returning before the RPC is the narrow maintainable implementation; docs and real behavior proof remain before merge.
