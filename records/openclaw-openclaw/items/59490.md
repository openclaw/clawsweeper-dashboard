## What This Changes

The branch adds `--dry-run` to `openclaw cron add`, prints resolved params as text or JSON before the `cron.add` RPC, and adds CLI tests for no-RPC dry-run output.

## Best Possible Solution

Land the narrow CLI-only preview after real CLI proof is posted and the cron reference documents the flag, without changing scheduler or gateway contracts.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main has no `cron add --dry-run` flag and always calls `cron.add` after params construction.

## Solution Assessment

Mostly yes. Building the existing params object, printing through runtime output helpers, and returning before the RPC is the narrow maintainable shape; docs and real behavior proof remain the merge blockers.
