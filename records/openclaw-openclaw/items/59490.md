## Summary

Keep open: this focused feature PR is still useful because current main lacks `openclaw cron add --dry-run`, and the PR implements the intended no-RPC preview path. It should not merge until the contributor adds real CLI behavior proof and the cron docs mention the new flag.

## What This Changes

The PR adds a `--dry-run` flag to `openclaw cron add` that prints resolved create params as text or JSON and returns before `cron.add`, with CLI tests for both dry-run modes.

## Best Possible Solution

Land the narrow CLI-only dry-run implementation after real CLI proof is posted and the cron reference documents the new flag, without changing scheduler or gateway contracts.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main has no `cron add --dry-run` flag and always calls `cron.add` after params construction.

## Solution Assessment

Mostly yes. Building the existing params object, printing it through the runtime output helpers, and returning before the RPC is the narrow maintainable implementation; docs and real behavior proof are the remaining merge blockers.
