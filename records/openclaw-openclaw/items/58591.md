## Summary

Close as implemented on main. Current main solves the observable Feishu reply-attachment failure through shared reply-media normalization and managed-media staging before the Feishu dispatcher receives the payload, and that implementation shipped in v2026.4.15.

## What This Changes

The PR imports agent-scoped media root resolution into the Feishu reply dispatcher, forwards those roots to reply attachment media sends, and adds a focused unit test for that forwarding.

## Best Possible Solution

Keep the shipped shared reply-media staging design and close this Feishu-specific PR as redundant. Any remaining Feishu account-level `mediaLocalRoots` configuration semantics should stay with the separate config-support work rather than adding duplicate dispatcher plumbing.

## Reproduction Assessment

Yes. The original reproduction path is a Feishu auto-reply carrying a local media path from an agent workspace; current tests cover that by staging an absolute workspace media path before delivery, and current Feishu production replies run through that normalizer.

## Solution Assessment

No for merging this PR now. Its narrow root-threading approach is reasonable, but current main’s shared normalizer is the better maintained solution because it centralizes local-file validation and stages safe managed media before channel-specific dispatch.
