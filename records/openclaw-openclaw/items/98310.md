## Summary

Keep open: current main and v2026.6.11 still dedupe compaction user turns by normalized text and timestamp only, so distinct group participants with identical long text can still collapse unless a trusted sender discriminator is added.

## What This Changes

The issue reports that compaction duplicate-user-message pruning can drop a distinct group participant's identical long user turn because the dedup key omits sender identity.

## Best Possible Solution

Approve and land a durable non-phone sender metadata path, then make compaction duplicate-user dedup sender-aware while preserving same-sender retry pruning and adding focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible: current main's duplicateSignature keys only on normalized user text and timestamp, and both compaction consumers use it. I did not run a live group-channel compaction scenario in this read-only review.

## Solution Assessment

Partly: a sender-aware key is the right compaction behavior, but the durable sender discriminator and privacy semantics should be resolved first. A helper-only patch would be incomplete while persisted user turns lack sender identity.
