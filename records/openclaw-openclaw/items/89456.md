## Summary

## What This Changes

This PR forwards `plugins.load.paths` into trusted channel catalog discovery, makes rejected catalog fallback origin-aware, blocks untrusted non-bundled scoped setup-only channel imports, and expands regression coverage.

## Best Possible Solution

Land the hardening after maintainers explicitly accept the local-origin trust compatibility change and the current head has green focused and required validation; otherwise preserve config/global compatibility behind a deliberately chosen strict path.

## Solution Assessment

Yes, this is the right owner-boundary shape: trusted catalog resolution must see configured paths, and the loader needs a backstop so lower-level scoped setup loads cannot bypass trust. The unresolved part is not code shape but maintainer acceptance of the stricter upgrade behavior and green current-head checks.
