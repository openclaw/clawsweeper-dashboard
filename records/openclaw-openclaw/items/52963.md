## Summary

Keep open. Current main still lacks Slack user-level `message.im` event subscription, user-token inbound identity, and xoxp reply dispatch for monitor replies; the request is valid Slack-plugin work but needs maintainer product/security approval before implementation.

## What This Changes

Support Slack Socket Mode `message.im` events delivered on behalf of a configured user token and route replies through that user token.

## Best Possible Solution

Implement this in the bundled Slack plugin after approval: generate/document `user_events`, resolve the configured xoxp user identity, distinguish bot-level from user-level DMs, select the user token only for the approved opt-in path, prevent self-loops, and add focused unit plus live Slack validation.

## Reproduction Assessment

Yes for source-level reproduction: current main has no `user_events` manifest output, no xoxp user identity in monitor context, and monitor replies use `ctx.botToken`. A live Slack Socket Mode run is still needed to verify Slack's exact user-level event envelope.

## Solution Assessment

No automated fix path is best yet. The narrow implementation belongs in the bundled Slack plugin, but xoxp impersonation and inbound identity routing need maintainer product/security approval and live validation first.
