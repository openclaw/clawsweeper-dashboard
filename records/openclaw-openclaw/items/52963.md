## Summary

Keep open. Current main still lacks Slack user-level `message.im` subscription, xoxp inbound identity, and user-token reply dispatch for Slack monitor DMs, so the request remains valid bundled Slack plugin feature work that needs maintainer product/security approval before implementation.

## What This Changes

The issue requests Slack Socket Mode support for user-level `message.im` events and replies through the configured xoxp user token for DMs to a regular Slack user.

## Best Possible Solution

Implement this in the bundled Slack plugin after approval: add `user_events` manifest/docs, resolve the configured xoxp user identity, select the user token only for the approved opt-in path, prevent self-loops, and add focused unit plus live Slack validation.

## Reproduction Assessment

Yes for source-level reproduction: current main emits only Slack `bot_events`, builds monitor identity from the bot token, and monitor replies call `deliverReplies` with `ctx.botToken`. A live Slack workspace is still needed for the exact user-level Socket Mode envelope.

## Solution Assessment

Unclear for automation. The narrow implementation belongs in the Slack plugin, but user-token authorship and inbound identity routing should be approved by maintainers before a fix PR is attempted.
