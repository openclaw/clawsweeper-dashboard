## Summary

Keep open. Current main and the latest release still lack the Slack user-level `message.im` subscription, xoxp inbound identity, and user-token reply dispatch needed for this DM-routing feature, and the remaining work needs maintainer product/security approval before implementation.

## What This Changes

Requested support for Slack user-level Socket Mode events so DMs to a configured Slack user can route inbound and reply through that user token.

## Best Possible Solution

After approval, implement this in the bundled Slack plugin by adding `user_events` manifest/docs support, resolving the configured xoxp user identity, selecting the user token only for the approved opt-in path, guarding self-loops, and adding focused unit plus live Slack validation.

## Reproduction Assessment

Yes for source-level reproduction: current main lacks `user_events` manifest output and has no xoxp inbound identity or reply-token selector in the monitor path. I did not run a live Slack workspace reproduction, so the exact Socket Mode envelope remains unverified.

## Solution Assessment

Unclear for automation. The narrow implementation belongs in the Slack plugin, but using a user token for inbound identity and outbound authorship should be explicitly approved before a fix PR is attempted.
