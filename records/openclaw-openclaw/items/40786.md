## Summary

This issue is still necessary: current main has some built-in backup filtering for volatile state and installed plugin dependencies, but it still does not provide the requested user-controlled `openclaw backup create` exclude patterns, and the open linked implementation PR needs review rather than pre-merge issue cleanup.

## What This Changes

The issue requests .gitignore-like `openclaw backup create` exclude controls, including repeatable patterns, an exclude file, and sensible defaults for bulky or sensitive paths.

## Best Possible Solution

Review and either land, narrow, or replace the open backup exclude-pattern PR with a restore-safe native backup design that preserves critical recovery paths, documents the CLI contract, and proves both default and opt-in exclusion behavior.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source inspection: current main exposes only coarse backup scope flags and no `--exclude` or `--exclude-file` option, so the requested CLI behavior is absent.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers decide the permanent backup contract; generic exclude patterns are useful, but the best core solution needs protected restore-critical paths, sensible defaults, and upgrade-safe JSON/manifest behavior.
