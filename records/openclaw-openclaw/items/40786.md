## Summary

Keep open. Current main still lacks native `.gitignore`-style exclusions for `openclaw backup create`; Keep My Claw is useful prior art and a workaround, but it does not close the first-party CLI gap, and open PR #44288 already targets this issue with closing syntax.

## What This Changes

Request repeatable backup exclude patterns and an exclude-file path for `openclaw backup create`, with defaults for dependency, cache, log, and secret-like files.

## Best Possible Solution

Resolve the first-party backup CLI gap by reviewing, narrowing, landing, or replacing #44288 with focused tests and public backup docs; Keep My Claw can remain an optional workaround.

## Reproduction Assessment

Yes. Source inspection on current main shows no CLI option or typed archive API for user exclude patterns, while tests and docs cover only the existing coarse flags and installed-plugin `node_modules` filtering.

## Solution Assessment

Unclear for the exact implementation because #44288 makes default-exclusion and protected-path policy choices. The best tracking outcome is to keep this issue open and handle the active closing PR before starting another repair path.
