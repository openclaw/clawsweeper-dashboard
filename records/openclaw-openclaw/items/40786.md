## Summary

Keep open. Current main and the latest release still do not expose user-supplied backup exclude patterns, while an open pull request with closing syntax already owns the native CLI implementation path; the Keep My Claw ClawHub plugin is useful prior art and workaround, but it does not close the first-party backup CLI gap.

## What This Changes

Requested .gitignore-style exclude patterns, exclude-file support, and sensible exclusions for `openclaw backup create`.

## Best Possible Solution

Resolve the native backup exclusion design through https://github.com/openclaw/openclaw/pull/44288 or a narrower replacement, with CLI forwarding, archive filtering, restore-safe defaults, manifest/docs updates, and focused regression coverage.
