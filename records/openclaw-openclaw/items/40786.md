## Summary

Keep open: current `main` and `v2026.5.22` still lack native gitignore-style backup exclude controls, while https://github.com/openclaw/openclaw/pull/44288 is the open linked implementation path. The KeepMyClaw ClawHub plugin is useful workaround context, but it does not replace a first-party `openclaw backup create` filtering surface.

## What This Changes

The issue requests repeatable `--exclude` and exclude-file support for `openclaw backup create`, with sensible exclusions for large, noisy, or sensitive files.

## Best Possible Solution

Review and either land a corrected https://github.com/openclaw/openclaw/pull/44288 or replace it with a narrower native backup exclude design covering CLI flags, archive filtering, protected recovery-critical paths, docs, and regression coverage.
