## Summary

Keep open because current main and the latest release still lack first-party backup exclude patterns, and an open implementation PR already owns the native CLI path.

## What This Changes

The issue requests .gitignore-style `openclaw backup create` exclusions through repeatable patterns, an exclude file, and possible default skips for large or sensitive files.

## Best Possible Solution

Review and either land a corrected https://github.com/openclaw/openclaw/pull/44288 or replace it with a narrower native backup exclude implementation with CLI forwarding, archive filtering, protected defaults, manifest/docs updates, and focused regression coverage.
