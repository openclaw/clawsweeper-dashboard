## Summary

Keep open. Current main and the latest release still lack user-supplied backup exclude patterns, while an open implementation PR already owns the native CLI path; Keep My Claw is useful ClawHub prior art and workaround, but it does not close the first-party backup CLI gap.

## What This Changes

The issue requests gitignore-style `--exclude` and `--exclude-file` support for `openclaw backup create` so users can omit large, noisy, or sensitive files from native backups.

## Best Possible Solution

Resolve the native backup exclusion design through https://github.com/openclaw/openclaw/pull/44288 or a narrower replacement, with CLI forwarding, archive filtering, restore-safe defaults, manifest/docs updates, and focused regression coverage.
