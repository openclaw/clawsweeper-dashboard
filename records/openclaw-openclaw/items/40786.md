## Summary

Keep open. Current main and the latest release still lack native `.gitignore`-style exclusions for `openclaw backup create`; Keep My Claw is useful prior art and a workaround, but it does not close the first-party backup CLI gap, and open PR #44288 already targets this issue with closing syntax.

## What This Changes

Add repeatable backup exclude patterns and exclude-file support so users can omit dependency, cache, log, and secret-like files from `openclaw backup create` archives.

## Best Possible Solution

Keep this as the user-facing tracker for native backup exclusions, and resolve it by reviewing, narrowing, landing, replacing, or closing PR #44288 with tests and public backup docs updated.

## Reproduction Assessment

Yes. This is a high-confidence missing-feature check: current source, tests, and docs show no user-supplied exclude-pattern path, while existing coverage confirms only installed-plugin dependency trees are skipped.
