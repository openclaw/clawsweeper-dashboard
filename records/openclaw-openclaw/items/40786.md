## Summary

Keep open: current `main` and the latest release still lack native backup exclude-pattern support, and an open linked PR is the active implementation path but remains unresolved. The KeepMyClaw ClawHub plugin is useful workaround context, but it does not replace a first-party `openclaw backup create` filtering surface.

## What This Changes

The issue requests .gitignore-style exclude patterns for `openclaw backup create`, including repeatable `--exclude` flags, an `--exclude-file`, and default exclusions for bulky or sensitive files.

## Best Possible Solution

Review and either land a corrected https://github.com/openclaw/openclaw/pull/44288 or replace it with a narrower native backup exclude design covering CLI flags, archive filtering, protected recovery-critical paths, docs, and regression coverage.
