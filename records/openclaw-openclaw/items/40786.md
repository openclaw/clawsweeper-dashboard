## Summary

Keep open: current main and latest v2026.6.10 still expose only coarse backup scope flags plus fixed internal filters, while https://github.com/openclaw/openclaw/pull/44288 remains the open implementation candidate and still needs maintainer/security review. The Keep My Claw ClawHub plugin linked in the discussion is a useful workaround, but it does not close the native backup CLI gap.

## What This Changes

The issue requests gitignore-style `--exclude` and `--exclude-file` support for `openclaw backup create` so users can omit large, noisy, or sensitive files from archives.

## Best Possible Solution

Review, rebase, and either land or replace https://github.com/openclaw/openclaw/pull/44288 with a maintainer-approved native exclude design that preserves required recovery assets and documents the security tradeoffs.

## Reproduction Assessment

Yes, source-level. Current main and v2026.6.10 register no `--exclude` or `--exclude-file` flags, and the archiver only applies fixed internal filters.

## Solution Assessment

No, closing is not the best solution. The maintainable path is a reviewed native backup-exclude design, with the existing ClawHub plugin treated as a workaround rather than a replacement for the core CLI gap.
