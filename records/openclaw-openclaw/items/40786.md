## Summary

Keep open. Current main and v2026.4.27 still lack native `.gitignore`-style exclusions for `openclaw backup create`; the only archive filter skips installed-plugin `extensions/*/node_modules`. The Keep My Claw plugin linked in the discussion is useful prior art and a workaround, but it does not close the first-party backup CLI gap, and open PR #44288 already targets this issue with closing syntax.

## What This Changes

Request native repeatable exclude patterns and exclude-file support for `openclaw backup create`, with default exclusions for bulky or sensitive files.

## Best Possible Solution

Keep this as the user-facing tracker for native backup exclusions. The desired end state is a core backup implementation with repeatable gitignore-style patterns, exclude-file parsing, clear default and protected-path semantics, manifest behavior that avoids leaking sensitive filenames, focused CLI/archive regression tests, and updated public docs at https://docs.openclaw.ai/cli/backup.

## Reproduction Assessment

Yes. This is a high-confidence missing-feature check: current source, tests, and docs show no user-supplied exclude-pattern path, while existing coverage confirms only installed-plugin dependency trees are skipped and root `state/node_modules` remains included.

## Solution Assessment

Unclear for the exact implementation, but keeping this issue open is the right tracking outcome. The maintainable path is to review, narrow, land, replace, or close open PR #44288 before starting another fix path.
