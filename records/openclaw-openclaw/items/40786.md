## Summary

Keep open. Current main and the latest release still lack user-supplied backup exclude patterns, and https://github.com/openclaw/openclaw/pull/44288 is still the open first-party implementation path; the Keep My Claw ClawHub plugin is useful prior art and workaround, but it does not close the native backup CLI gap.

## What This Changes

The issue requests gitignore-style exclusion support for `openclaw backup create`, including repeatable patterns, an exclude file, and recommended default skips for large, noisy, or secret-like files.

## Best Possible Solution

Resolve the native backup exclusion design through https://github.com/openclaw/openclaw/pull/44288 or a narrower replacement with CLI forwarding, archive filtering, restore-safe defaults, manifest/docs updates, and focused regression coverage.
