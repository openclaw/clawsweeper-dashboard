## Summary

Keep open. Current main and latest release still lack native `.gitignore`-style backup exclusions; Keep My Claw is useful prior art/workaround, but it does not close the first-party CLI gap, and open PR #44288 already targets this issue with closing syntax.

## What This Changes

Add native `.gitignore`-style exclude patterns, exclude-file support, and possible defaults to `openclaw backup create`.

## Best Possible Solution

Keep this as the user-facing tracker for native backup exclusions and resolve it through PR #44288 or a maintainer-approved narrower replacement with CLI, archive, manifest, tests, changelog, and public docs aligned.

## Reproduction Assessment

Yes. This is a high-confidence missing-feature check from source: current main has no exclude flags, no exclude fields in `BackupCreateOptions`, and only a hard-coded installed-plugin `node_modules` filter.

## Solution Assessment

Unclear for the exact implementation. Native pattern-aware backup exclusions belong in the backup CLI, but the safest next step is maintainer review of PR #44288 before starting another repair path.
