## Summary

Keep open because current main still lacks first-party `.gitignore`-style backup exclude controls, and an open implementation PR already references this issue with closing syntax. The linked KeepMyClaw ClawHub plugin is useful workaround context for recovery-focused backups, but it does not replace the requested native `openclaw backup create` capability.

## What This Changes

Add `.gitignore`-style backup exclude controls such as repeatable `--exclude` patterns and an `--exclude-file` option to `openclaw backup create`.

## Best Possible Solution

Review and either land a corrected https://github.com/openclaw/openclaw/pull/44288 or replace it with a narrower native backup exclude implementation covering CLI forwarding, archive filtering, protected restore-critical paths, docs, and focused regression coverage.
