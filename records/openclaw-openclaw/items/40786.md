## Summary

Current `main` still lacks fine-grained backup exclusions, and an open linked PR already owns the implementation attempt; keep the issue open until maintainers decide the recovery-safety contract and land or reject that PR.

## What This Changes

Add repeatable `.gitignore`-style patterns and an exclude-file input to `openclaw backup create`, potentially with default exclusions for bulky or sensitive files.

## Best Possible Solution

Adopt one canonical, opt-in exclusion design in the native backup pipeline with protected restore-critical assets, deterministic pattern precedence, manifest/verification semantics, and explicit documentation of confidentiality versus full-recovery guarantees.
