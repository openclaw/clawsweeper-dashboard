## Summary

Keep open because current main and the latest release still lack first-party backup exclude patterns, and an open implementation PR already owns the native CLI path. The KeepMyClaw ClawHub plugin linked in the discussion is a useful workaround, but it does not replace the requested OpenClaw backup CLI capability.

## What This Changes

The issue requests .gitignore-style exclude patterns and exclude-file support for `openclaw backup create`.

## Best Possible Solution

Review and either land a corrected https://github.com/openclaw/openclaw/pull/44288 or replace it with a narrower native backup exclude implementation that includes CLI forwarding, archive filtering, protected restore-critical paths, docs, and focused regression coverage.
