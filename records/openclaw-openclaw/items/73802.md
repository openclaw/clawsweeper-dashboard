## Summary

Keep open. Current main still rejects Discord native approval handling when the request is bound to a different Discord account than the configured native approval client, which matches the reported manual-approval-works-but-card-does-not-deliver setup.

## What This Changes

The issue requests restoration of native Discord exec approval cards/buttons for a multi-account setup where manual `/approve` still succeeds.

## Best Possible Solution

Define the Discord multi-account native approval contract, then add routing and regression coverage for explicitly configured approver accounts while preserving command-disclosure boundaries.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Discord approval request with `turnSourceChannel: "discord"` and `turnSourceAccountId` set to a different account than the enabled approver account; current main rejects native handling while manual approver authorization can still pass.

## Solution Assessment

Unclear. Removing the account gate is the narrow mechanical change, but the maintainable fix needs maintainer policy on cross-account Discord card delivery before changing behavior.
