## Summary

Keep open. Current main still matches the reported Discord multi-account native approval gap: the native delivery path rejects requests whose source account differs from the configured native approver account, while the manual `/approve` path can still authorize that approver.

## What This Changes

Discord exec approvals should deliver native approval cards/buttons for enabled approver accounts instead of forcing users to type manual `/approve` commands.

## Best Possible Solution

Define the Discord multi-account native approval contract, then implement the chosen routing behavior with regression coverage for cross-account approver delivery or a documented origin-account-only fallback.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Discord approval request with `turnSourceChannel: "discord"` and `turnSourceAccountId` set to a different account than the configured native approver account; current main rejects native delivery before resolving origin or approver-DM targets.

## Solution Assessment

Unclear. Relaxing the account gate is the narrow mechanical fix, but the maintainable solution needs an explicit policy for cross-account command disclosure and approval routing before code changes.
