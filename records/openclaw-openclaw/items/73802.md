## Summary

Keep open. Current main still has the same Discord multi-account native approval gap: the native card path rejects a request before origin or approver-DM targets can be resolved when the request source account differs from the configured Discord approval account, while manual `/approve` remains authorized through the command path.

## What This Changes

Discord native exec approval cards/buttons should be delivered for configured Discord approvers instead of requiring manual `/approve` in the reported multi-account setup.

## Best Possible Solution

Define the Discord multi-account native approval contract, then implement either explicit approver-account card routing or documented origin-account-only delivery with reliable fallback notices and regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction is an approval request with `turnSourceChannel: "discord"` and `turnSourceAccountId` set to a different account than the configured native approval account; current main rejects native delivery before target resolution.

## Solution Assessment

Unclear. Removing the account gate would be the narrow mechanical fix for cards, but the safer maintainable solution needs maintainer policy on cross-account command disclosure and approval authority first.
