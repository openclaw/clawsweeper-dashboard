## Summary

Keep open. Current main still preserves the Discord multi-account native approval gap: native delivery is gated by the request's source account before origin or approver-DM targets are resolved, so the reporter's configured approver can still approve manually while the native card/buttons are skipped.

## What This Changes

Restore Discord native exec approval cards/buttons for configured approvers when an exec approval is created in multi-account Discord setups.

## Best Possible Solution

Define the Discord multi-account native approval contract, then either route cards only to explicitly configured approver accounts with regression tests or document origin-account-only native delivery while keeping the manual fallback reliable.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Discord approval request with turnSourceChannel set to discord and turnSourceAccountId set to a different account than the configured native approver account; current main rejects native handling before approver-DM target resolution, while /approve remains authorized for configured approvers.

## Solution Assessment

Unclear. Relaxing the account gate is the narrow mechanical route to native cards, but the best fix needs maintainer judgment on cross-account command disclosure and authorization before changing routing.
