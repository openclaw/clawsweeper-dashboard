## Summary

Keep open. Current main still matches the reported multi-account Discord approval gap: a native approval request is rejected when the request source account differs from the Discord account configured for native approvals, while the manual `/approve` path can still authorize the configured approver.

## What This Changes

The issue reports that Discord exec approvals can be approved manually but no native approval card/buttons are delivered in a multi-account Discord configuration.

## Best Possible Solution

Define the intended Discord multi-account native approval contract, then add regression coverage for either explicit approver-account card routing or documented origin-account-only delivery with a reliable same-chat fallback.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an approval request with `turnSourceChannel: "discord"` and `turnSourceAccountId` set to a different account than the account configured for native approvals; current main rejects native delivery before approver target resolution.

## Solution Assessment

Unclear. Removing the account gate is the narrow mechanical way to restore cards, but the maintainable fix needs maintainer policy on cross-account command disclosure and authorization before changing delivery behavior.
