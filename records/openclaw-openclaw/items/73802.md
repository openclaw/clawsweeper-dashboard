## Summary

Keep open. The latest same-account retest shows Discord native approvals can work again, but current main and the latest release still fail closed for the original multi-account routing pattern, and changing that needs an explicit approval-routing/security decision.

## What This Changes

The issue requests Discord native exec approval cards/buttons to reach the configured approval surface instead of requiring a manual `/approve` command.

## Best Possible Solution

Define the Discord multi-account approval-routing contract, then either implement safe cross-account approver delivery or document and enforce origin-account-only behavior with clear fallback coverage.

## Reproduction Assessment

Yes, at source level. A Discord approval request with `turnSourceChannel: "discord"` and `turnSourceAccountId` different from the configured native approval account is rejected before origin or approver-DM delivery planning.

## Solution Assessment

Unclear. The same-account path appears fixed, but the remaining cross-account behavior needs an explicit security and product policy before a maintainable code change can be chosen.
