## Summary

Keep open: the simple same-account Discord approval path now works in the reporter's retest, but the original multi-account setup remains source-reproducible because current main still rejects native delivery when the approval request is bound to a different Discord account than the native approval runtime account.

## What This Changes

The issue requests restored native Discord exec approval card/button delivery for a multi-account setup where manual Discord approval commands still resolve the approval.

## Best Possible Solution

Define the intended Discord multi-account native approval contract, then either implement safe cross-account approver delivery or document and enforce origin-account-only behavior with a reliable manual fallback and regression coverage.

## Reproduction Assessment

Yes, at source level. A request with turnSourceChannel set to discord and turnSourceAccountId set to a different account than the configured native approval account is rejected before origin or approver-DM native delivery planning.

## Solution Assessment

Unclear. Relaxing the account gate is mechanically narrow, but the maintainable fix needs an explicit security policy for command-bearing Discord approval cards crossing account boundaries.
