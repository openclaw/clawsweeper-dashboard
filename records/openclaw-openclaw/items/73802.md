## Summary

Keep open. Current main still matches the reported Discord multi-account native approval gap: the native card path rejects a request when the source Discord account differs from the configured native approver account, while the manual approval command can still authorize that approver.

## What This Changes

The issue requests restoration of Discord native exec approval cards/buttons when an approval is created, instead of requiring the operator to type `/approve` manually.

## Best Possible Solution

Define the Discord multi-account native approval routing contract, then implement that contract with regression coverage for cross-account approver delivery or a documented origin-account-only fallback.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Discord approval request with `turnSourceChannel: "discord"` and `turnSourceAccountId` set to a different account than the configured native approver account; current main rejects native delivery before target resolution.

## Solution Assessment

Unclear. Relaxing the account gate is the narrow mechanical change, but the best maintainable fix needs an explicit maintainer policy for cross-account command disclosure and approval routing.
