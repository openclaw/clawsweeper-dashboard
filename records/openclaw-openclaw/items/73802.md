## Summary

Keep open. Current main and the latest release still reject Discord native approval delivery when the request source account differs from the configured native approver account, while manual approval authorization can still accept that configured approver.

## What This Changes

Discord native exec approval cards/buttons should be delivered for enabled Discord exec approval configurations instead of requiring the operator to type `/approve` manually.

## Best Possible Solution

Define and document the Discord multi-account native approval routing contract, then update the Discord gate/target resolvers and regression coverage so the configured approver surface gets a card or an explicit safe fallback.

## Reproduction Assessment

Yes. A source-level reproduction is a Discord approval request with `turnSourceChannel: "discord"`, `turnSourceAccountId` set to another account, and native approvals configured on the approver account; current main rejects native delivery before origin or approver-DM target resolution.

## Solution Assessment

No. Current main preserves the failure, and simply relaxing the account gate is not enough without a clear policy for cross-account command disclosure and approval routing.
