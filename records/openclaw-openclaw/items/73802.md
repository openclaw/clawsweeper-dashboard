## Summary

Keep open. Current main still matches the reported multi-account Discord approval failure: native card delivery is rejected by the request source-account gate before origin or approver-DM targets can be resolved, while the manual `/approve` path can still authorize the configured approver.

## What This Changes

The issue reports that Discord exec approvals can be approved manually, but native Discord approval cards/buttons are not delivered in a multi-account setup.

## Best Possible Solution

Define the Discord multi-account native approval contract, then add regression coverage for either explicit approver-account card routing or documented origin-account-only delivery with a reliable manual fallback.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an approval request with `turnSourceChannel: "discord"` and `turnSourceAccountId` set to a different account than the configured native approver account; current main rejects native delivery before approver-target resolution.

## Solution Assessment

Unclear. Removing the account gate is the narrow mechanical route to native cards, but the maintainable fix needs maintainer policy on cross-account command disclosure and authorization before changing delivery behavior.
