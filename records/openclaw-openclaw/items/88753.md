## Summary

Keep open: current main and the latest release still lack a context-engine output lane for lower-authority historical/reference data, while the linked lossless-claw mitigation explicitly leaves compacted summaries as synthetic user-role messages until OpenClaw adds a provider-safe host contract.

## What This Changes

The issue requests a provider-safe context-engine output channel for untrusted historical/reference context so engines do not serialize compacted summaries as fresh user messages.

## Best Possible Solution

Design a narrow plugin SDK reference-context contract, render it below system/developer instructions and above the current user request across harnesses/providers, and add regression coverage plus lossless-claw coordination.

## Reproduction Assessment

Yes, from source inspection: current main's AssembleResult has no reference/historical context field, and the embedded runner consumes only messages plus systemPromptAddition. I did not run a live prompt-injection exploit, but the linked lossless-claw issue confirms its post-mitigation summary path remains upstream-gated on this contract.

## Solution Assessment

Yes, the host/plugin SDK boundary is the right layer for the remaining role-authority gap. The exact field shape and provider rendering semantics are still a maintainer security/product decision.
