## Summary

Keep open: current main and the latest release still lack a host-owned lower-authority reference/historical context lane. The public context-engine contract only exposes messages, promptAuthority, systemPromptAddition, and contextProjection, and the embedded/Codex paths still render assembled messages as model history or prompt text, so this remains a real plugin SDK and prompt-authority design item.

## What This Changes

Requests a context-engine AssembleResult output lane for untrusted historical/reference context so plugins do not have to replay compacted summaries as fresh user messages.

## Best Possible Solution

Design a narrow plugin SDK reference-context lane plus host-owned rendering semantics for embedded and Codex/native provider paths, with docs and regression coverage for lossless-claw-style compacted summaries.

## Reproduction Assessment

Yes. Source inspection shows the public AssembleResult shape has no separate reference/historical data lane, and the embedded runner consumes returned messages as model history.

## Solution Assessment

Yes at the boundary level: the host/plugin SDK is the right place for this authority contract. The exact field shape and provider/Codex rendering semantics still need maintainer security/product design before implementation.
