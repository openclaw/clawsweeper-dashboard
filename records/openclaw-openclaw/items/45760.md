## Summary

Keep open for maintainer judgment, but the patch should not merge as-is: it changes the documented delegated-session memory boundary and is stale against current main because the lowercase memory filename constant is gone.

## What This Changes

The PR adds root memory filenames to `MINIMAL_BOOTSTRAP_ALLOWLIST` in `src/agents/workspace.ts` so subagent and cron sessions receive `MEMORY.md`.

## Best Possible Solution

Keep root `MEMORY.md` out of minimal delegated bootstrap unless maintainers approve an explicit opt-in or on-demand memory path, with tests and docs updated for the chosen boundary.

## Reproduction Assessment

Yes, from source for the reported current behavior: `filterBootstrapFilesForSession` filters subagent and cron sessions through an allowlist that excludes `MEMORY.md`, and tests/docs define that exclusion. That does not establish a bug because current docs describe this as the expected private-session boundary.

## Solution Assessment

No. Adding root memory to the minimal allowlist changes default private-context exposure; an explicit opt-in or tool-mediated memory retrieval design would be safer if maintainers want delegated continuity.
