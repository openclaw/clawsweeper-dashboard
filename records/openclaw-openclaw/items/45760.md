## Summary

Keep open for maintainer judgment: current main intentionally keeps root `MEMORY.md` out of minimal subagent and cron bootstrap through code, tests, and docs, while this PR would broaden that privacy boundary and is stale against the current uppercase-only root-memory policy.

## What This Changes

The PR adds root memory filenames to `MINIMAL_BOOTSTRAP_ALLOWLIST` in `src/agents/workspace.ts` so subagent and cron sessions would receive root memory bootstrap files.

## Best Possible Solution

Keep root `MEMORY.md` out of minimal bootstrap by default; if continuity is needed, design an explicit opt-in or on-demand retrieval path with tests, docs, and security review.

## Reproduction Assessment

Yes, from source. Applying the PR additions would make `filterBootstrapFilesForSession` retain `MEMORY.md` for `agent:*:subagent:*` and `agent:*:cron:*`, while current tests assert that filtered subagent/cron bootstrap excludes it.

## Solution Assessment

No. The two-line allowlist change conflicts with the documented main-session-only memory boundary and should be replaced by an explicit opt-in or on-demand memory design if maintainers want this capability.
