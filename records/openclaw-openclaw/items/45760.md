## Summary

Keep open for maintainer handling; the PR is not safe to merge as-is because it widens default root-memory injection into subagent and cron sessions against the current tests, docs, and prior security review.

## What This Changes

The PR adds root memory filenames to `MINIMAL_BOOTSTRAP_ALLOWLIST` in `src/agents/workspace.ts` so minimal subagent and cron bootstrap would include workspace memory.

## Best Possible Solution

Keep root `MEMORY.md` out of default minimal bootstrap unless maintainers design an explicit opt-in sharing policy or retrieval flow with tests, docs, and security review updated together.

## Reproduction Assessment

Yes. Source inspection shows the PR additions make `filterBootstrapFilesForSession` retain root memory for `agent:*:subagent:*` and `agent:*:cron:*`, while current main tests assert that `MEMORY.md` is excluded.

## Solution Assessment

No. The two-line allowlist change is not the narrowest maintainable fix because it silently changes a documented main-session-only memory boundary; opt-in sharing or on-demand memory tools are safer directions.
