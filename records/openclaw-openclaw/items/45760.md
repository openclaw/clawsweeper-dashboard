## Summary

Keep this PR open for explicit maintainer handling: current main documents and tests root memory as main/private-session-only, while this patch would broaden that boundary for delegated/background runs and also references a retired lowercase-memory runtime constant.

## What This Changes

The PR adds `DEFAULT_MEMORY_FILENAME` and `DEFAULT_MEMORY_ALT_FILENAME` to `MINIMAL_BOOTSTRAP_ALLOWLIST` in `src/agents/workspace.ts` so subagent and cron sessions receive root memory files during bootstrap.

## Best Possible Solution

Keep root `MEMORY.md` main/private-session-only by default; if delegated continuity is desired, design a maintainer-approved opt-in or on-demand memory path aligned with the existing bootstrap boundary.

## Reproduction Assessment

Yes, from source. Current tests include `MEMORY.md` in the bootstrap fixture and assert that subagent and cron filtering excludes it, so applying this PR would change the asserted current-main behavior.

## Solution Assessment

No. The two-line allowlist change is not the best fix because it bypasses the documented main-session-only memory boundary; an explicit opt-in or on-demand memory design would be safer.
