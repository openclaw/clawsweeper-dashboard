## Summary

Keep open, but the PR should not merge as-is: current main intentionally keeps root memory out of subagent/cron bootstrap, tests and docs encode that boundary, and the patch also carries a stale lowercase-memory symbol.

## What This Changes

The branch adds `DEFAULT_MEMORY_FILENAME` and `DEFAULT_MEMORY_ALT_FILENAME` to `MINIMAL_BOOTSTRAP_ALLOWLIST` in `src/agents/workspace.ts` so subagent and cron sessions receive root memory files.

## Best Possible Solution

Keep root `MEMORY.md` out of minimal bootstrap by default; if continuity is needed, design an explicit opt-in, tiered, or on-demand memory path aligned with the existing bootstrap-selection work.

## Reproduction Assessment

Yes, from source. Applying the two allowlist additions would make `filterBootstrapFilesForSession` retain `MEMORY.md` for subagent and cron session keys, while current tests assert that filtered subagent/cron bootstrap excludes it.

## Solution Assessment

No. The two-line default allowlist change conflicts with the documented and tested main-session-only memory boundary; an explicit opt-in or on-demand design would be the safer maintainable path.
