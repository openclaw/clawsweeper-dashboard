## Summary

Keep this PR open for maintainer judgment, but it should not merge as-is: current main documents and tests delegated bootstrap as excluding root memory, and this patch broadens private durable context exposure for subagent and cron runs.

## What This Changes

The PR adds `DEFAULT_MEMORY_FILENAME` and `DEFAULT_MEMORY_ALT_FILENAME` to `MINIMAL_BOOTSTRAP_ALLOWLIST` in `src/agents/workspace.ts`.

## Best Possible Solution

Keep root `MEMORY.md` out of minimal delegated bootstrap unless maintainers approve an explicit opt-in or memory-tool retrieval design, with tests and docs updated to match that policy.

## Reproduction Assessment

Yes. Source inspection shows subagent and cron sessions are filtered through an allowlist that excludes `MEMORY.md`, but current docs and tests define that exclusion as intended behavior rather than a bug.

## Solution Assessment

No. Adding root memory to the default minimal allowlist is not the narrowest maintainable solution because it changes private-context exposure; an explicit opt-in or tool-mediated retrieval path would be safer if maintainers want this behavior.
