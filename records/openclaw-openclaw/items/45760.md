## Summary

Keep open for maintainer review: current main documents and tests a main-session-only root-memory boundary, while this patch would broaden default memory exposure in delegated/background sessions and is now stale against the uppercase-only memory policy.

## What This Changes

The branch adds `DEFAULT_MEMORY_FILENAME` and `DEFAULT_MEMORY_ALT_FILENAME` to `MINIMAL_BOOTSTRAP_ALLOWLIST` in `src/agents/workspace.ts` so subagent and cron minimal bootstrap would retain root memory files.

## Best Possible Solution

Keep root memory out of default minimal bootstrap and design any delegated-memory sharing as an explicit opt-in or on-demand retrieval policy with tests, docs, and security review updated together.

## Reproduction Assessment

Yes. Source inspection is enough: the PR additions make `filterBootstrapFilesForSession` retain root memory for `agent:*:subagent:*` and `agent:*:cron:*`, while current main tests assert that `MEMORY.md` is excluded.

## Solution Assessment

No. The two-line allowlist change silently changes the default memory-sharing boundary; an explicit opt-in policy or on-demand memory retrieval flow would be narrower and safer.
