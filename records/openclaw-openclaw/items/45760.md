## Summary

Keep open for maintainer review, but do not merge as-is: current main intentionally excludes root memory from subagent and cron bootstrap in source, tests, and docs, while this PR broadens default private context exposure and still carries the retired lowercase memory path.

## What This Changes

The PR adds `DEFAULT_MEMORY_FILENAME` and `DEFAULT_MEMORY_ALT_FILENAME` to `MINIMAL_BOOTSTRAP_ALLOWLIST` so subagent and cron sessions receive root memory files.

## Best Possible Solution

Keep root `MEMORY.md` out of minimal delegated bootstrap unless maintainers approve an explicit opt-in or on-demand memory retrieval path, with tests and docs updated for that policy.

## Reproduction Assessment

Yes, source inspection reproduces the reported current behavior: subagent and cron sessions are filtered through an allowlist that excludes `MEMORY.md`. That exclusion is currently documented and tested, so the source proof does not establish a bug.

## Solution Assessment

No. Adding root memory to the minimal allowlist changes default private-context exposure; an explicit opt-in or tool-mediated memory retrieval design would be safer if maintainers want delegated continuity.
