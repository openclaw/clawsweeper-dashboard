## Summary

Keep open, but the patch should not merge as-is. The discussion correctly identified that the minimal allowlist is the gate, but current main treats root `MEMORY.md` as main-session-only and the branch broadens that private-context boundary by default.

## What This Changes

The branch adds root memory filenames to `MINIMAL_BOOTSTRAP_ALLOWLIST` in `src/agents/workspace.ts` so minimal subagent and cron bootstrap would include memory files.

## Best Possible Solution

Keep root `MEMORY.md` out of default minimal bootstrap unless maintainers design an explicit opt-in memory-sharing or retrieval flow with docs, tests, and security review updated together.

## Reproduction Assessment

Yes. Current main source shows subagent and cron sessions are filtered through `MINIMAL_BOOTSTRAP_ALLOWLIST`, and the tests assert that `MEMORY.md` is excluded; the PR diff would make those sessions retain root memory files.

## Solution Assessment

No. The two-line allowlist change silently changes a private-context boundary; a maintainable fix would need explicit sharing policy or on-demand retrieval semantics instead of default injection.
