## Summary

Keep open, but this PR should not merge as-is: current main intentionally keeps root memory out of delegated/background bootstrap, and the docs/tests define that as a privacy boundary rather than a missing allowlist entry.

## What This Changes

The PR adds canonical and legacy root memory filenames to `MINIMAL_BOOTSTRAP_ALLOWLIST` in `src/agents/workspace.ts` so subagent and cron bootstrap filtering would include workspace memory files.

## Best Possible Solution

Preserve the default exclusion; if maintainers want delegated or cron access to root memory, add an explicit opt-in sharing policy or on-demand retrieval path with tests, docs, and security review updated together.

## Reproduction Assessment

Yes. A source-level check of `filterBootstrapFilesForSession` with `agent:<id>:subagent:<name>` or `agent:<id>:cron:<name>` shows `MEMORY.md` is currently filtered out, and `src/agents/workspace.test.ts` asserts that exclusion.

## Solution Assessment

No. Adding memory filenames to the global minimal allowlist silently changes a documented security boundary; an explicit opt-in policy or memory retrieval flow is the narrower maintainable direction.
