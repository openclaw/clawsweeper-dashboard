## Summary

Keep this PR open for maintainer follow-up, but the patch is not safe to merge as-is. Current main still excludes `MEMORY.md` from subagent and cron minimal bootstrap, while tests and workspace-template docs encode that as a main-session privacy boundary; the PR would widen long-term memory exposure in delegated/background sessions.

## What This Changes

The PR adds canonical and legacy root memory filenames to `MINIMAL_BOOTSTRAP_ALLOWLIST` so subagent and cron bootstrap filtering would include workspace memory files.

## Best Possible Solution

Keep the default main-session memory boundary unless maintainers explicitly decide otherwise. If delegated or cron memory sharing is desired, implement it as an explicit opt-in policy with source, tests, docs, and security review updated together, and keep canonical root-memory handling aligned with current bootstrap behavior.

## Reproduction Assessment

Yes. A focused check of `filterBootstrapFilesForSession` with a subagent or cron session key reproduces the current exclusion path, and `src/agents/workspace.test.ts` already asserts that `MEMORY.md` is filtered out for both session types.

## Solution Assessment

No. The PR is a narrow code change, but it bypasses the documented main-session-only memory boundary; the safer solution is an explicit opt-in sharing policy or continued on-demand memory retrieval, not default bootstrap injection.
