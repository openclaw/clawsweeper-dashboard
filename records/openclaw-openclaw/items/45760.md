## Summary

Keep open for maintainer judgment, but the patch should not merge as written: current main intentionally keeps root memory out of subagent/cron minimal bootstrap, and the branch also references a lowercase memory constant that was retired on main.

## What This Changes

The PR adds `DEFAULT_MEMORY_FILENAME` and `DEFAULT_MEMORY_ALT_FILENAME` to `MINIMAL_BOOTSTRAP_ALLOWLIST` in `src/agents/workspace.ts` so subagent and cron sessions receive root memory files.

## Best Possible Solution

Keep `MEMORY.md` main/private-session-only by default; if delegated continuity is desired, add a maintainer-approved opt-in or on-demand memory path aligned with the bootstrap privacy boundary.

## Reproduction Assessment

Yes, from source. Current main’s filter and tests show subagent/cron sessions exclude `MEMORY.md`, so applying this PR would change an asserted and documented behavior boundary.

## Solution Assessment

No. Adding root memory to the minimal allowlist bypasses the current privacy boundary; an explicit opt-in or on-demand memory design would be safer if maintainers want this capability.
