## Summary

Keep open: the PR is not ready to merge because it widens the root-memory privacy boundary for delegated/background sessions, and the branch is now stale against current main's uppercase-only memory filename policy.

## What This Changes

The branch adds `DEFAULT_MEMORY_FILENAME` and `DEFAULT_MEMORY_ALT_FILENAME` to `MINIMAL_BOOTSTRAP_ALLOWLIST` in `src/agents/workspace.ts` so minimal subagent and cron bootstrap would include root memory files.

## Best Possible Solution

Keep root `MEMORY.md` out of minimal bootstrap by default; if continuity is needed, design an explicit opt-in or on-demand retrieval path with tests, docs, and security review.

## Reproduction Assessment

Yes. Source inspection shows the PR additions would make `filterBootstrapFilesForSession` retain root memory for `agent:*:subagent:*` and `agent:*:cron:*`, while current tests assert that `MEMORY.md` is excluded.

## Solution Assessment

No. The two-line allowlist change silently changes a documented main-session-only memory boundary and is stale against current main's removal of the lowercase memory filename constant.
