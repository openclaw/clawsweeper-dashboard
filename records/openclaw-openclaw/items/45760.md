## Summary

Keep this PR open for maintainer follow-up, but do not merge it as-is. Current main treats root `MEMORY.md` as private main-session context, and the existing filter/tests/docs support keeping it out of delegated and background bootstrap by default.

## What This Changes

The PR adds canonical and legacy root memory filenames to `MINIMAL_BOOTSTRAP_ALLOWLIST` in `src/agents/workspace.ts` so subagent and cron bootstrap filtering would include workspace memory files.

## Best Possible Solution

Preserve the current default boundary. If maintainers want delegated or cron runs to receive root memory, make it an explicit opt-in policy with source, tests, docs, and security review updated together rather than broadening minimal bootstrap silently.

## Reproduction Assessment

Yes. A focused check of `filterBootstrapFilesForSession` with `agent:<id>:subagent:<uuid>` or `agent:<id>:cron:<name>` reproduces the current exclusion path, and `src/agents/workspace.test.ts` asserts `MEMORY.md` is filtered out for both session types.

## Solution Assessment

No. The PR is mechanically small, but default bootstrap injection is not the narrowest maintainable solution for private root memory; an explicit opt-in sharing policy or on-demand memory retrieval is safer.
