## Summary

Keep open. Current `main` still omits the requested timing information from the shared `/status`/`session_status` card and from the visible response usage footer; related open items cover pieces of the work, but there is no merged implementation that fully supersedes this broader report.

## What This Changes

Restore elapsed or response-time visibility in the shared `session_status`/`/status` card and related response footer, especially for Feishu operators.

## Best Possible Solution

Land one coordinated timing update, or explicitly split it across the open shared-status, response-footer, and Feishu-card trackers so each visible surface gets a clear owner and test coverage.

## Reproduction Assessment

Yes. Source inspection on current `main` shows `/status` and `session_status` both delegate to the shared status renderer, whose session line only includes the key plus update age, while the response footer formatter has no duration input even though diagnostics record `durationMs`.

## Solution Assessment

Unclear. The narrow implementation points are clear, but this issue overlaps open shared-status, response-footer, and Feishu-card customization work, so maintainers should coordinate the split before landing another implementation.
