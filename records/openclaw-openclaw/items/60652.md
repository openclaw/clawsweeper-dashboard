## Summary

Keep this issue open. Current main still does not advertise `upload-file` for Discord message actions, while the Discord `send` path still accepts file inputs; the related open PR #60808 is the canonical implementation candidate and should be reviewed or replaced before this issue closes.

## What This Changes

Discord should advertise a discoverable `upload-file` message action so agents can send files through the existing Discord attachment path.

## Best Possible Solution

Keep the issue open and converge on one Discord-owned fix: add `upload-file` to Discord message-action discovery under the chosen gate, route it through the existing media send backend or an equivalent dedicated handler, and update Discord action contract/unit coverage so discovery and dispatch stay aligned.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: configure a token-backed Discord account with `messages` enabled and inspect `describeDiscordMessageTool`; it can return `read`, `edit`, and `delete` but not `upload-file`.

## Solution Assessment

Yes on the fix boundary, unclear on the exact gate/handler shape. The narrow maintainable solution belongs in the Discord plugin discovery/dispatch tests, not in core action-name registration, because core already supports the action name and schema plumbing.
