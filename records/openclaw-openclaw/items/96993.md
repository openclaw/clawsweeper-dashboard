## Summary

Keep open: current main still has the unbounded proxy SSE tail-buffer path, but this PR is not merge-ready because it enforces the 64 KiB cap before draining complete SSE lines, so valid coalesced transport chunks can fail.

## What This Changes

Adds a 64 KiB `streamProxy` SSE buffer cap and a regression test for a delimiter-free oversized proxy response.

## Best Possible Solution

Keep the parser-tail guard, but drain complete SSE lines first and enforce the cap only on the leftover unterminated buffer while tracking the broader total-byte cap separately.

## Reproduction Assessment

Yes by source inspection. Current main has an unterminated proxy SSE buffer growth path, and the PR-introduced coalesced-chunk failure follows directly from checking the full buffer before line draining.

## Solution Assessment

No. The cap is the right mitigation, but the best narrow fix is to process complete SSE lines first and cap only the leftover unterminated tail.
