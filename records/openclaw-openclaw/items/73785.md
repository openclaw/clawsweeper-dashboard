## Summary

Keep open. The exact-sentinel and exec-ack suppression goal is valid, but this PR is not merge-ready because it lacks real behavior proof, is currently dirty against main, and the diff would make cron/system-event `HEARTBEAT_OK` summaries deliverable by switching them to message-mode stripping.

## What This Changes

The PR adds shared exact no-op sentinel detection for heartbeat replies, applies it in heartbeat-runner normalization, removes exec fallback delivery of stripped acknowledgements, and adds heartbeat-runner regression tests.

## Best Possible Solution

Rebase the exact sentinel suppression onto current main, remove exec ack fallback for no-op tokens, and preserve heartbeat-mode ack suppression for cron/no-op summaries while keeping real exec summaries deliverable.

## Reproduction Assessment

Yes, source-level. Current main lacks the proposed custom sentinel helper, and the relayable exec path can restore a stripped pure `HEARTBEAT_OK` before outbound delivery.

## Solution Assessment

No, not as written. The exact-sentinel helper is a good narrow direction, but the `systemEvent` message-mode change should not apply to cron/no-op summaries because it bypasses the current heartbeat ack suppression semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86
