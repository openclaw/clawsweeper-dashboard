Keep open. The PR targets a real current-main no-op heartbeat leak, but it is stale against current main and the proposed system-event message-mode switch can regress short `HEARTBEAT_OK` no-op summary suppression.

## Best Possible Solution

Land a rebased exact-sentinel suppression that preserves heartbeat `ackMaxChars` suppression and relayable exec output, with regression tests and a changelog entry.

## Solution Assessment

No, not as-is. Central exact-sentinel suppression is the right direction, but the patch should be rebased and should not bypass heartbeat-mode `ackMaxChars` suppression for short no-op summaries.
