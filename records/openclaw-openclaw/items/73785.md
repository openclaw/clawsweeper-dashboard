## Summary

Keep open: this PR targets a valid current-main heartbeat/system-event no-op leak, but latest head still moves cron and non-relayable system-event replies out of heartbeat-mode ack suppression, and the external proof is still a mocked focused test rather than live runtime/channel proof.

## What This Changes

The PR adds exact no-op sentinel detection for heartbeat/system replies, applies it during heartbeat normalization and exec fallback, and adds heartbeat-runner regression tests.

## Best Possible Solution

Keep the exact sentinel suppression, but bound message-mode stripping to relayable exec summaries, preserve heartbeat-mode `ackMaxChars` suppression for cron and non-relayable no-op events, then require redacted live runtime proof before merge.

## Reproduction Assessment

Yes, source-level. Current main can restore an exact `HEARTBEAT_OK` through the relayable exec fallback, and the PR regression is visible from `systemEvent: hasExecCompletion || hasCronEvents` combined with heartbeat-vs-message stripping behavior; no live command was run in this read-only review.

## Solution Assessment

No, not as written. The exact sentinel helper is a narrow fit for the exec leak, but the mode switch should be limited to relayable exec summaries so cron and non-relayable no-op events keep heartbeat-mode short-ack suppression.
