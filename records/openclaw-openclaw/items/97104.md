Full review comments:

- **[P2] Load selected documents before prompt filtering:** `src/projects/project-store.ts:601`
  - body: The UI saves role/chat-specific extra document IDs, and `projectDocumentList` is written to include selected IDs even when `includeInContext` is false.   - body: Every agent prompt now calls project context assembly, which can call `summarizeProjectDocument`; cache misses read local files and use `spawnSync` with 10s timeouts for PDF/DOCX. A few selected documents can block the prompt path before the agent turn starts, so extraction/summary refresh should move to import, save, or background work and prompt assembly should inject cached summaries only.
  - confidence: 0.87
