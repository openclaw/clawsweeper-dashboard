## Summary

Current main already implements the documented Codex harness route: forced `agentRuntime.id: "codex"` is resolved from agent policy, the selected harness is pinned into the attempt, and regression coverage now verifies `openai/*` with Codex OAuth.

## What This Changes

The issue reports that `agents.list[].agentRuntime.id: "codex"` is ignored for `openai/gpt-5.5`, causing the run to use the raw OpenAI provider instead of the Codex harness.

## Best Possible Solution

Keep the current main implementation and ship it in the next release with the existing regression coverage for the documented Codex setup.

Yes. The report gives a concrete v2026.5.2 config and failure trajectory, and current source now has regression coverage for the same `openai/*` plus Codex runtime/auth path; I did not run it because `pnpm` is unavailable in this read-only environment.
