Full review comments:

- **[P1] Restrict the reset target before trashing it:** `src/commands/workspace.ts:52-54`
  - body: `--workspace` is a user-supplied path, but this command accepts it as the trash/reseed target without proving it is an OpenClaw workspace. For example, `openclaw workspace reset --workspace ~/.ssh --yes` would move that directory and then reseed it with OpenClaw bootstrap files. Reuse the current cleanup target guards and/or require the target to be the configured agent/dev workspace before calling `moveToTrash`.
  - confidence: 0.95
- **[P1] Clear workspace attestations before reseeding:** `src/commands/workspace.ts:110-115`
  - body: After `moveToTrash(workspaceDir)`, current main's `ensureAgentWorkspace` refuses to recreate a recently attested missing workspace. The existing full reset path removes matching workspace attestation files after deleting workspace dirs; this command needs the same invariant or it can leave the workspace in Trash and fail before reseeding.
  - confidence: 0.9
- **[P3] Clean up the workspace reset temp directory:** `src/commands/workspace.test.ts:67-68`
  - body: `tempRoot` is created in `beforeEach` and never removed, so every test in this file leaves a temp tree behind. Add an `afterEach` that removes `tempRoot` to keep `--isolate=false` test runs clean.
