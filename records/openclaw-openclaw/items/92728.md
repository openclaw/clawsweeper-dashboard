## Summary

Keep open. Current main and v2026.6.8 still keep multi-character CJK trigram tokens on the MATCH path and only use the LIKE fallback on MATCH exceptions; the related open PRs cover adjacent or partial fixes but do not yet settle this issue's CJK term-decomposition semantics.

## What This Changes

The issue requests a memory-core keyword fallback that handles multi-character CJK trigram queries by matching relevant CJK terms instead of requiring the exact full CJK phrase.

## Best Possible Solution

Land one maintainer-approved CJK term decomposition and empty-MATCH fallback strategy in memory-core, with regression coverage and real redacted memory_search proof, then close this issue after the selected fix merges.

## Reproduction Assessment

Yes, at source level. Current main still routes long CJK trigram tokens through MATCH and only builds LIKE fallback on MATCH exceptions; this read-only review did not run a live Ollama-backed memory index reproduction.

No landed solution is best yet. The best path is a maintainer-selected split-term and fallback scoring strategy rather than closing on the current exact long-CJK path or relying on a blocked partial PR.
