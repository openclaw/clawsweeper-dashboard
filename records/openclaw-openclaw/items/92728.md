## Summary

Keep open. Current main and the latest release still route long CJK trigram queries through exact phrase matching or exact whole-token LIKE behavior, and the related open PR at https://github.com/openclaw/openclaw/pull/92341 has not landed or proven the narrower split-term fallback requested here.

## What This Changes

The issue asks memory-core to make multi-character CJK keyword fallback match relevant split terms instead of only the exact full CJK phrase.

## Best Possible Solution

Fold a focused split-term CJK fallback with regression coverage into https://github.com/openclaw/openclaw/pull/92341 or a narrow replacement, then close this issue after the fix lands.

## Reproduction Assessment

Yes. Current source shows the exact long-CJK MATCH and exception-only LIKE fallback path, and a focused SQLite trigram probe reproduced exact-phrase-only behavior versus split-term recall.

## Solution Assessment

No current solution is landed. The related open PR is a plausible repair vehicle, but its whole-token CJK LIKE approach needs review or adjustment before it is the best fix for this narrower issue.
