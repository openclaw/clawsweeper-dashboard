## Summary

Keep open. Current main preserves Feishu slash-command text and normalizes mention-prefixed commands, but it still denies group control commands when `commands.useAccessGroups=true` and no group/top-level command allowlist is configured; open PR #49958 is the paired closing implementation candidate.

## What This Changes

Feishu group-chat slash commands such as `/new`, `/status`, and `/model` should either execute as OpenClaw control commands or reach the agent as normal text instead of disappearing.

## Best Possible Solution

Land PR #49958 or an equivalent narrow shared command-gating fix that authorizes ordinary group commands when no command allowlist is configured, while preserving configured allowlist enforcement and Feishu parser coverage.

## Reproduction Assessment

Yes. A source-level reproduction is clear on current main: a Feishu group command with no per-group or top-level `allowFrom` passes an unconfigured authorizer into the shared gate, which current code and tests resolve to `false`.

## Solution Assessment

Yes. The narrowest maintainable direction is a shared command-gating fix with regression coverage; Feishu-specific parser work alone would not address the remaining current-main denial.
