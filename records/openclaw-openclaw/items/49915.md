## Best Possible Solution

Land a narrow maintainer-approved Feishu command-authorization fix, or an explicitly approved shared command-gating contract, that authorizes already-admitted senders only when no explicit command allowlist exists while preserving explicit allowlist denial and adding live Feishu proof.

Yes at source level, but not as a live Feishu run. Current main still carries Feishu group slash commands into CommandAuthorized=false when no command allowlist matches, and the shared reply path then returns no reply for unauthorized whole-message control commands.

## Solution Assessment

No for closing now. The best fix direction is narrow and clear, but the authorization contract is security-sensitive and the open linked PRs are unmerged, so maintainers still need to choose and validate the repair.
