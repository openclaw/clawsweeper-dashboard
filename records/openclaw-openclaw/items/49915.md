## What This Changes

The issue reports that Feishu group-chat slash commands such as /new, /status, and /model fail under group-bound routing and may be dropped as empty or metadata-only messages instead of executing or reaching the agent.

## Best Possible Solution

Land a narrow Feishu command-authorization fix, or an explicit shared command-gating contract with security coverage, so admitted Feishu senders can use slash commands when no command allowlist is configured while explicit allowlists still deny unauthorized senders.

## Reproduction Assessment

Yes, at source level. Current main can carry a Feishu group slash command with no explicit command allowlist into CommandAuthorized=false, and the shared reply path then returns no visible reply for unauthorized whole-message control commands.

## Solution Assessment

Yes for keeping this issue open. The best implementation should fix Feishu command authorization narrowly, or intentionally update the shared command-gating contract with parity tests and security proof; the closed target PRs have not resolved the issue.
