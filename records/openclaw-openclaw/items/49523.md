## Summary

Keep open: current main and latest release v2026.6.10 still allow mirrored transcript header creation to fall back to ambient `process.cwd()` when no explicit cwd is supplied, and the linked closing PR remains open and conflicting rather than merged.

## What This Changes

The issue requests that session transcript mirroring write header `cwd` from the target agent workspace instead of ambient process cwd.

## Best Possible Solution

Land, rebase, or replace https://github.com/openclaw/openclaw/pull/50454 with a current-main patch that resolves the target agent workspace before transcript header creation.

## Reproduction Assessment

Yes. By source inspection, create a stored session entry without `spawnedCwd`, run `appendAssistantMessageToSessionTranscript()` while ambient cwd differs from the target workspace, and header creation reaches `process.cwd()`.

## Solution Assessment

Yes. Passing the explicit target workspace into header creation is the narrow maintainable fix for this path; removing all process-global cwd usage is broader follow-up work.
