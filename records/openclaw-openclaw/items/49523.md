## Summary

Keep open. Current `main` and the latest release still allow mirrored transcript header creation to fall back to process-global `process.cwd()` when the session entry lacks `spawnedCwd`, and the linked closing PR remains open and conflicting.

## What This Changes

The issue asks for mirrored session transcript headers to record the target agent workspace explicitly instead of reading process-global cwd.

## Best Possible Solution

Land or replace the linked narrow fix so transcript header creation receives explicit target-agent workspace context and regression coverage for normal and spawned mirrored sessions.

## Reproduction Assessment

Yes by source inspection: create a mirrored transcript for a normal session entry without `spawnedCwd` while ambient `process.cwd()` points elsewhere, and the header factory records that global cwd fallback. I did not run a write-based repro because this review is read-only.

## Solution Assessment

Yes. Threading explicit target workspace context into header creation is the narrow maintainable fix; current main's `spawnedCwd` mitigation is useful but too narrow for normal session entries.
