## Summary

Keep open. Current main and v2026.6.6 still let mirrored transcript header creation fall back to `process.cwd()` whenever the session entry lacks `spawnedCwd`, and the linked closing fix remains open, unmerged, and conflicting at https://github.com/openclaw/openclaw/pull/50454.

## What This Changes

The issue requests that mirrored session transcript headers record the target agent workspace instead of ambient process cwd.

## Best Possible Solution

Update or replace https://github.com/openclaw/openclaw/pull/50454 with a current-main-compatible fix that resolves the effective target agent workspace before transcript header creation and covers both normal and spawned session entries.

## Reproduction Assessment

Yes by source inspection: create a mirrored transcript for a normal session entry without `spawnedCwd` while ambient `process.cwd()` points elsewhere, and the header factory records that global cwd fallback. I did not run a write-based repro because this review is read-only.

## Solution Assessment

Yes. Threading explicit target workspace context into header creation is the narrow maintainable fix; current main's `spawnedCwd` mitigation is useful but too narrow for normal session entries.
