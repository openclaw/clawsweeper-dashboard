## Summary

Keep open: current main and latest release still let mirrored transcript header creation fall back to ambient `process.cwd()` when the session entry lacks `spawnedCwd`, and the linked fix candidate is open rather than merged.

## What This Changes

The issue asks for session transcript mirroring to write header `cwd` from the explicit target agent workspace instead of ambient `process.cwd()`.

## Best Possible Solution

Land, rebase, or replace https://github.com/openclaw/openclaw/pull/50454 with a current-main fix that resolves the target agent workspace before transcript header creation and keeps focused `process.cwd()` isolation coverage.

## Reproduction Assessment

Yes. By source inspection, create a stored session entry without `spawnedCwd`, change the process cwd to another workspace, then call `appendAssistantMessageToSessionTranscript()`; the header path has no explicit cwd and falls back to `process.cwd()`.

## Solution Assessment

Yes. Passing an explicit target workspace into header creation is the narrow maintainable fix; the current spawned-cwd mitigation is useful but does not cover ordinary mirrored transcript creation.
