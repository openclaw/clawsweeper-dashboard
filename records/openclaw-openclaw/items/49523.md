## Summary

Keep open. Current main and v2026.5.12 still create mirrored transcript headers from `process.cwd()`, and https://github.com/openclaw/openclaw/pull/50454 still uses closing syntax while https://github.com/openclaw/openclaw/pull/56133 overlaps the same workspace-header problem.

## What This Changes

Make mirrored session transcript headers record the target agent workspace instead of the process-global cwd.

## Best Possible Solution

Land one consolidated current-main fix that derives a single effective agent id and workspace from runtime config before any create-if-missing transcript header write, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a brand-new mirrored transcript while `process.cwd()` differs from the target agent workspace, and the current header helper writes that process-global cwd.

## Solution Assessment

Yes. The explicit-workspace direction is the narrow maintainable fix, but it should use runtime config and one effective agent id rather than adding process cwd fallbacks.
