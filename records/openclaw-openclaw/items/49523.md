## Summary

Keep open. Current main and the latest release still create transcript headers from `process.cwd()`, and open PRs with closing syntax still need maintainer consolidation before this issue can close.

## What This Changes

The issue requests that mirrored session transcript headers record the target agent workspace instead of reading the process-global current working directory.

## Best Possible Solution

Land one current-main-compatible fix that derives a single effective agent id and workspace from runtime config for create-if-missing transcript header writes, with focused regression coverage and redacted runtime proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a brand-new mirrored transcript while `process.cwd()` differs from the target agent workspace, and the header helper writes the process-global cwd.

## Solution Assessment

Yes. The explicit-workspace direction is the narrow maintainable fix, but it should use runtime config and one effective agent id/workspace consistently rather than ambient cwd or config reload fallbacks.
