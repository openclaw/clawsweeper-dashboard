## Summary

Keep open. Current main still creates mirrored transcript headers from process-global `process.cwd()`, the latest release still contains the same behavior, and an open PR uses closing syntax for this issue.

## What This Changes

The issue asks for mirrored session transcript headers to record the target agent workspace instead of `process.cwd()`.

## Best Possible Solution

Land one current-main-compatible fix that threads the effective agent workspace into transcript header creation and adds focused regression coverage for cwd isolation.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a brand-new mirrored transcript while `process.cwd()` differs from the target agent workspace, and the header helper writes that process-global cwd.

## Solution Assessment

Yes. The explicit-workspace direction is the narrow maintainable fix, using the existing per-agent workspace resolver rather than adding another process cwd fallback.
