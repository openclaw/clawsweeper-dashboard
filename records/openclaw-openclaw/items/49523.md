## Summary

Keep open: current main and latest release still allow mirrored transcript header creation to fall back to ambient `process.cwd()` when the session entry has no explicit `spawnedCwd`, and the open closing PR remains conflicting rather than merged.

## What This Changes

The issue requests that transcript mirroring write the session header `cwd` from the target agent workspace instead of ambient `process.cwd()`.

## Best Possible Solution

Land, rebase, or replace https://github.com/openclaw/openclaw/pull/50454 with a current-main patch that passes the explicit target workspace into transcript header creation; keep broader `process.chdir()` cleanup separate.

## Reproduction Assessment

Yes. By source inspection, create a stored session entry without `spawnedCwd`, run `appendAssistantMessageToSessionTranscript()` while ambient cwd differs from the target workspace, and header creation reaches `process.cwd()`.

## Solution Assessment

Yes. Passing an explicit target workspace into header creation is the narrow maintainable fix for this path; removing all process-global cwd usage is broader follow-up work.
