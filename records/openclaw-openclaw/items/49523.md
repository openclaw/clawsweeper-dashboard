## Summary

Keep open: current main and latest release v2026.6.8 still allow mirrored transcript header creation to fall back to ambient `process.cwd()` when the session entry has no explicit `spawnedCwd`, and the linked fix candidate is still open rather than merged.

## What This Changes

The issue requests that mirrored session transcript headers record the target agent workspace instead of process-global `process.cwd()`.

## Best Possible Solution

Land, rebase, or replace https://github.com/openclaw/openclaw/pull/50454 with a current-main patch that resolves the target agent workspace before header creation and keeps focused process.cwd isolation coverage.

## Reproduction Assessment

Yes. By source inspection, create a stored session entry without `spawnedCwd`, change the process cwd to another workspace, then call `appendAssistantMessageToSessionTranscript()`; the header path has no explicit cwd and falls back to `process.cwd()`.

## Solution Assessment

Yes. Passing an explicit target workspace into header creation is the narrow maintainable fix; the current spawned-cwd mitigation is useful but does not cover ordinary mirrored transcript creation.
