## Summary

Keep open. Current main and v2026.6.6 still let mirrored transcript headers fall back to ambient process.cwd() when the session entry lacks spawnedCwd, and https://github.com/openclaw/openclaw/pull/50454 remains the open closing fix lane.

## What This Changes

The issue asks for mirrored session transcript headers to use the target agent workspace explicitly instead of process.cwd().

## Best Possible Solution

Update or replace https://github.com/openclaw/openclaw/pull/50454 with a current-main-compatible fix that resolves one effective agent workspace before transcript header creation and adds regression coverage for normal and spawned session entries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a new mirrored transcript for a session entry without spawnedCwd while process.cwd() points elsewhere, and the header helper records ambient cwd.

## Solution Assessment

Yes. Explicit workspace threading is the narrow maintainable fix; current main's spawnedCwd mitigation is too narrow for normal session entries.
