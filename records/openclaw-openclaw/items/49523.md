## Summary

Keep open. Current main still has the reported mirrored-transcript header bug, the latest shipped release still contains the same `process.cwd()` header write, and open PR #50454 uses closing syntax for this issue while #56133 covers an adjacent gateway-created transcript path.

## What This Changes

The issue asks OpenClaw to stop creating session transcript headers from process-global `process.cwd()` and instead record the resolved target agent workspace.

## Best Possible Solution

Keep this issue open and land or replace the canonical implementation so every session transcript header creation path records the resolved target agent workspace, with focused regression coverage proving temporary or concurrent `process.chdir()` changes cannot alter the header cwd.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: make a valid session-store entry, change `process.cwd()` to a different directory before the first mirrored append, and current `ensureSessionHeader()` will write that process-global directory into the new header.

## Solution Assessment

Yes. Threading an explicit resolved workspace/cwd into transcript header creation is the narrow maintainable fix for this bug; removing all runtime `process.chdir()` usage is a broader follow-up, and the adjacent gateway header path should be consolidated with #56133.

## Security Review

Status: needs_attention

Summary: The issue describes a real multi-agent isolation weakness in transcript metadata rather than a cosmetic-only mismatch.

Concerns:

- **[medium] Transcript cwd can cross agent workspace boundaries:** `src/config/sessions/transcript.ts:41`
  - body: New mirrored transcript headers read `process.cwd()`, so concurrent or temporary agent cwd changes can cause one agent's transcript metadata to point at another workspace. Downstream tooling that trusts the header cwd can weaken the documented per-agent workspace/session isolation model.
