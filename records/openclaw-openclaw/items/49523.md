## Summary

Keep open. Current main still has the ambient cwd fallback on transcript header creation when a mirrored session entry lacks explicit cwd metadata, and an open linked PR already owns the proposed fix lane.

## What This Changes

The issue asks for session transcript mirroring to write header cwd from the target agent workspace instead of process-global process.cwd().

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/50454 with a current-main-compatible fix that resolves the effective target agent workspace before transcript header creation and covers missing spawnedCwd metadata.

## Reproduction Assessment

Yes by source inspection, but not by a live concurrency run here. Create a mirrored transcript for a session entry without spawnedCwd while ambient process.cwd() points elsewhere, and current main records that fallback in the header.

## Solution Assessment

Yes. Explicit workspace context before header creation is the narrow maintainable fix; current main's spawnedCwd mitigation is useful but too narrow for normal or incomplete session entries.
