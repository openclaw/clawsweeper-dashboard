## Summary

Keep open. Current main and the latest release still write new mirrored transcript headers from `process.cwd()`, and an open PR still uses closing syntax for this issue.

## What This Changes

The issue requests that mirrored session transcript headers record the target agent workspace instead of the process-global cwd.

## Best Possible Solution

Land one current-main-compatible fix that resolves the effective agent workspace from runtime config before create-if-missing transcript header writes, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a brand-new mirrored transcript while `process.cwd()` differs from the target agent workspace, and the header helper writes the process-global cwd.

## Solution Assessment

Yes. The explicit-workspace direction is the narrow maintainable fix, but it should use one effective agent id/workspace from runtime config rather than adding new process cwd fallbacks.
