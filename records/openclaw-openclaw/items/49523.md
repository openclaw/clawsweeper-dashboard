## Summary

Keep open: current main and the latest release still allow a new mirrored transcript header to fall back to ambient `process.cwd()` when the session entry has no explicit cwd, and the linked fix candidate at https://github.com/openclaw/openclaw/pull/50454 is still open and conflicting rather than merged.

## What This Changes

The issue asks for session transcript mirroring to record the target agent workspace in the header `cwd` instead of reading process-global `process.cwd()`.

## Best Possible Solution

Rebase or replace https://github.com/openclaw/openclaw/pull/50454 with a current-main patch that resolves the target agent workspace before header creation and keeps a focused regression test for process.cwd isolation.

## Reproduction Assessment

Yes. By source inspection, create a stored session entry without `spawnedCwd`, change the process cwd to another workspace, then call `appendAssistantMessageToSessionTranscript()`; the header path has no agent workspace and falls back to `process.cwd()`.

## Solution Assessment

Yes. Passing an explicit target workspace into header creation is the narrow maintainable fix; the current spawned-cwd mitigation is useful but does not cover normal mirrored transcript creation.
