## Summary

Keep open. Current main and v2026.5.18 still create mirrored transcript headers from `process.cwd()`, and open fix PRs still need maintainer consolidation before this issue can close.

## What This Changes

The issue asks for mirrored session transcript headers to record the target agent workspace explicitly instead of `process.cwd()`.

## Best Possible Solution

Land one consolidated current-main-compatible fix that derives one effective agent id and workspace from runtime config before create-if-missing transcript header writes, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a brand-new mirrored transcript while `process.cwd()` differs from the target agent workspace, and the header helper writes the process-global cwd.

## Solution Assessment

Yes. The explicit-workspace direction is the narrow maintainable fix, but it should use runtime config and one effective agent id/workspace consistently rather than ambient cwd or config reload fallbacks.
