## Summary

Keep open. Current main and the latest release still allow mirrored transcript headers to fall back to ambient process cwd when the session entry lacks spawned cwd, and https://github.com/openclaw/openclaw/pull/50454 is still the open closing fix lane.

## What This Changes

The issue reports that session transcript mirroring can write a new transcript header cwd from process-global cwd instead of the target agent workspace.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/50454 with a current-main-compatible fix that resolves the effective agent workspace before transcript header creation and covers both normal and spawned entries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a new mirrored transcript for a session entry without spawnedCwd while process.cwd() points elsewhere, and the header helper records ambient cwd.

## Solution Assessment

Yes. Explicit workspace threading is the narrow maintainable fix; current main's spawnedCwd-only mitigation is too narrow for normal session entries, so the best path is to finish the linked explicit-workspace PR or an equivalent replacement.
