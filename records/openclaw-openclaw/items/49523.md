## Summary

Keep open. Current main and the latest release still create new mirrored transcript headers from `process.cwd()`, and an open PR still uses closing syntax for this issue, so this should stay open until a consolidated fix lands.

## What This Changes

The issue asks for mirrored session transcript headers to record the target agent workspace instead of `process.cwd()` when new transcript files are created.

## Best Possible Solution

Land one current-main-compatible fix that resolves one effective agent id and workspace from runtime config before create-if-missing transcript header writes, consolidating https://github.com/openclaw/openclaw/pull/50454 with the broader cwd coverage in https://github.com/openclaw/openclaw/pull/56133 if maintainers want the wider scope.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a brand-new mirrored transcript while `process.cwd()` differs from the target agent workspace, and `ensureSessionHeader()` writes the process-global cwd into the JSONL header.

## Solution Assessment

Yes for the requested direction: explicit workspace context is the narrow maintainable fix. The open fix path should use runtime config and one effective agent id/workspace consistently rather than reloading ambient config or preserving process cwd fallbacks.
