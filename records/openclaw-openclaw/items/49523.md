## Summary

Keep open. Current main and latest release v2026.6.6 still leave mirrored transcript headers on the ambient `process.cwd()` fallback whenever the session entry lacks `spawnedCwd`, and the open closing fix lane is still unmerged and conflicting.

## What This Changes

The issue reports a workspace-isolation bug where mirrored session transcript headers can record the wrong `cwd` because header creation falls back to ambient `process.cwd()` instead of explicit target-agent workspace context.

## Best Possible Solution

Update or replace https://github.com/openclaw/openclaw/pull/50454 with a current-main-compatible fix that resolves the effective target agent workspace before transcript header creation and covers both normal and spawned session entries.

## Reproduction Assessment

Yes by source inspection: create a mirrored transcript for a session entry without `spawnedCwd` while ambient `process.cwd()` points elsewhere, and the header factory records that global cwd fallback.

## Solution Assessment

Yes. Explicit workspace context before header creation is the narrow maintainable fix; current main's `spawnedCwd` mitigation is useful but too narrow for normal session entries.
