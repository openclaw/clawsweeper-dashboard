## Summary

Keep open. Current main and the latest release still allow mirrored transcript headers to fall back to ambient `process.cwd()` when the session entry lacks `spawnedCwd`, and https://github.com/openclaw/openclaw/pull/50454 is still the open closing fix lane for explicit agent-workspace header creation.

## What This Changes

The issue requests changing mirrored session transcript header creation so `cwd` comes from explicit target agent workspace context rather than ambient `process.cwd()`.

## Best Possible Solution

Update or replace https://github.com/openclaw/openclaw/pull/50454 with a current-main-compatible fix that resolves one effective agent workspace/cwd before header creation and adds regression coverage for both normal and spawned session entries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a new mirrored transcript for a session entry without `spawnedCwd` while `process.cwd()` points elsewhere, and the header helper records ambient cwd.

## Solution Assessment

Yes. Explicit workspace threading is the narrow maintainable fix; current main's `spawnedCwd` mitigation is too narrow for normal session entries, so the best path is to finish the linked explicit-workspace PR or an equivalent replacement.
