## Summary

Keep open: the PR addresses a real source-visible fallback/session routing problem, but the current head is not merge-ready because it still has a cleanup-ordering bug, a status auth regression, missing real behavior proof, and an unresolved policy choice against the open TTL alternative.

## What This Changes

The PR rolls back automatic fallback model overrides after successful fallback attempts, separates selected session routing from actual fallback usage accounting, updates status/docs/tests, and includes build/test-support churn.

## Best Possible Solution

Choose the fallback recovery policy explicitly, keep selected-vs-actual accounting, persist cleanup before dropping retryable in-memory state, keep status auth labels model-specific, drop unrelated build churn, and require redacted runtime proof before merge.

## Reproduction Assessment

Yes by source inspection: current main documents sticky auto fallback, returns successful fallback runs before success rollback, and writes actual fallback provider/model into session routing. I did not run a live fallback scenario in this read-only review.

## Solution Assessment

No as-is: selected-vs-actual accounting is a good direction, but immediate rollback changes documented policy while a TTL alternative is open, and this branch still needs the cleanup-ordering/status-auth fixes plus real behavior proof.
