## Summary

Keep open. Current main still only partially satisfies the documented OPENCLAW_HOME contract: state/config and some agent defaults use the effective home, but dev git checkout and installer bootstrap defaults still fall back to the OS account home.

## What This Changes

The issue requests that OPENCLAW_HOME drive OpenClaw state, config, git checkout, and related internal defaults unless more specific path environment variables are set.

## Best Possible Solution

Keep the effective-home helper as the contract, update the remaining internal git/install defaults to derive from it when explicit path env vars are absent, add focused regression tests, and document intentional OS-home exceptions.

## Reproduction Assessment

Yes, at source level. With OPENCLAW_HOME set and OPENCLAW_GIT_DIR unset, current main still resolves the dev checkout from os.homedir(), and the installers still derive git/workspace/bootstrap config defaults from HOME.

## Solution Assessment

Yes. The maintainable fix is a narrow repair in the existing effective-home, update-cli, and installer seams, plus tests/docs; replacing every OS-home lookup would risk breaking intentional host-account behavior.
