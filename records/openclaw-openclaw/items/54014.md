## Summary

Keep open: current main still only partially honors the documented OPENCLAW_HOME contract, and the issue now has an open unmerged closing PR at https://github.com/openclaw/openclaw/pull/85802 that should be reviewed instead of closing this issue early.

## What This Changes

The issue asks OpenClaw to derive internal state, config, git checkout, installer, and related default paths from OPENCLAW_HOME unless a more specific path environment variable is set.

## Best Possible Solution

Land an upgrade-safe version of https://github.com/openclaw/openclaw/pull/85802 that reuses the existing home/path helpers, preserves explicit override precedence, covers installer/update defaults with tests, and documents intentional OS-home exceptions.

## Reproduction Assessment

Yes, at source level. With OPENCLAW_HOME set and OPENCLAW_GIT_DIR/OPENCLAW_CONFIG_PATH unset, current main still derives update and installer defaults from os.homedir or HOME while docs and path helpers define OPENCLAW_HOME as the internal home override.

## Solution Assessment

Yes, the maintainable direction is a bounded repair through existing home/path helpers plus focused installer/update tests. The open linked PR is the right review target, but it must be checked for upgrade-safe prefix/config behavior before this issue can close.
