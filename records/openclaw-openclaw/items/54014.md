## Summary

Keep open. Current main partially honors OPENCLAW_HOME for central state/config and agent defaults, but it still does not satisfy the reported contract: the dev/git checkout default and installer bootstrap paths continue to use the OS HOME/os.homedir path, and the public docs still conflict about whether OPENCLAW_HOME covers all internal path resolution. Related open PRs are adjacent or narrower, not complete fixes for this issue.

## What This Changes

The issue requests that OPENCLAW_HOME drive OpenClaw's internal default paths, including state, config, and git directories, unless explicit path overrides are set.

## Best Possible Solution

Define the supported OPENCLAW_HOME contract for internal OpenClaw defaults, update the dev/git checkout and installer bootstrap paths to derive from the existing home-dir semantics unless explicit path overrides are set, add focused regression tests, and align docs with intentional OS-home exceptions.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: with OPENCLAW_HOME set and OPENCLAW_GIT_DIR unset, resolveGitInstallDir still derives ~/openclaw from os.homedir, and install.sh still derives git/config/workspace bootstrap paths from HOME.

## Solution Assessment

No, current main is not yet the complete solution. The narrow maintainable fix is to reuse the existing home-dir path-resolution seam for internal defaults and add shell-side installer equivalents, while leaving intentional OS-home host behavior and the separate sandbox tilde work scoped separately.
