## Summary

Keep open. Current main and the latest v2026.5.18 release still only partially honor the documented OPENCLAW_HOME contract: core state/config paths use the effective home in some cases, but the nested .openclaw edge case, default dev/git checkout, and installer bootstrap/onboarding defaults still source-reproduce.

## What This Changes

The issue asks OpenClaw to consistently derive internal state, config, git checkout, and related defaults from OPENCLAW_HOME unless more specific path environment variables are set.

## Best Possible Solution

Define the internal OPENCLAW_HOME contract in one place, add a narrow state-dir basename guard, update the remaining git/update/install defaults to use the effective home when explicit path env vars are absent, and document intentional OS-home exceptions.

## Reproduction Assessment

Yes, at source level. With OPENCLAW_HOME set and more specific path env vars absent, current main still derives some defaults from OPENCLAW_HOME but continues to append .openclaw in the explicit-state-dir edge case and uses OS HOME/os.homedir for git/install bootstrap paths.

## Solution Assessment

Yes for a bounded repair through the existing home/path helpers. The maintainable path is to reuse resolveRequiredHomeDir, add focused basename guards and tests, and document intentional OS-home exceptions rather than sweeping every HOME or os.homedir occurrence.

## Repair Work Prompt

Fix the OPENCLAW_HOME path-resolution bug tracked by this issue. Scope the repair to internal OpenClaw defaults: preserve current parent-home semantics, add a narrow guard so OPENCLAW_HOME values whose basename is an existing state dir name such as .openclaw or legacy state names are not nested again, update both resolveStateDir() and resolveConfigDir(), and make default git/update/install bootstrap paths derive from OPENCLAW_HOME when OPENCLAW_GIT_DIR, OPENCLAW_STATE_DIR, or OPENCLAW_CONFIG_PATH are not explicitly set. Keep the profile-precedence and sandbox-tilde PRs separate unless the central helper change naturally covers them. Add focused regression tests and user-facing docs/changelog coverage for the path behavior change.
