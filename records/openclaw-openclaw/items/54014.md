## Summary

Keep open. Current main only partially satisfies the documented `OPENCLAW_HOME` contract: state/config and some agent defaults use the effective home, but the dev git checkout and installer bootstrap defaults still use the OS account home, and related open PRs cover only adjacent slices.

## What This Changes

The issue requests `OPENCLAW_HOME` to drive OpenClaw state, config, git checkout, and installer defaults unless more specific path environment variables are set.

## Best Possible Solution

Keep the effective-home helper as the contract, update the remaining dev git checkout and installer bootstrap defaults to derive from it when explicit path env vars are absent, add focused regression coverage, and align docs around intentional OS-home exceptions.

## Reproduction Assessment

Yes, at source level. With `OPENCLAW_HOME` set and `OPENCLAW_GIT_DIR` unset, current main still resolves the dev checkout from `os.homedir()`, and the installers still derive git, workspace, and bootstrap config defaults from `$HOME`.

## Solution Assessment

Yes. The maintainable fix is a narrow repair in existing path-resolution and installer seams, plus docs/tests; replacing every OS-home use would risk breaking intentional host-account behavior.
