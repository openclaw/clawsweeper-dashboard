Keep open: current main still only partially honors the documented OPENCLAW_HOME contract. ## What This Changes

The issue requests OPENCLAW_HOME to drive OpenClaw state, config, git checkout, and other internal default paths unless a more specific path environment variable is set.

## Best Possible Solution

Centralize the OPENCLAW_HOME contract for internal defaults, update the remaining update/git/install/bootstrap paths to use it when explicit path env vars are absent, preserve documented parent-home semantics, add focused tests, and document intentional OS-home exceptions.

## Reproduction Assessment

Yes, at source level. With OPENCLAW_HOME set and more specific path env vars absent, current main still derives some defaults from the effective home but continues to use OS HOME/os.homedir for git/install bootstrap paths.

## Solution Assessment

Yes for a bounded repair through the existing home/path helpers. The maintainable path is to reuse the shared effective-home resolver for internal defaults, add focused tests, and document intentional OS-home exceptions instead of sweeping every HOME or os.homedir occurrence.
