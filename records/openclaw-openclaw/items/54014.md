## Summary

Keep open. Current main still only partially respects the documented `OPENCLAW_HOME` contract: state/config and agent defaults use the effective home in some paths, but default dev checkout, installer bootstrap/onboarding checks, and the explicit `.openclaw` nesting edge case still source-reproduce.

## What This Changes

The issue asks OpenClaw to derive internal state, config, git, installer, and related path defaults from `OPENCLAW_HOME` unless more specific path environment variables are set.

## Best Possible Solution

Define the internal `OPENCLAW_HOME` contract in one place, update the remaining git/update/install defaults to derive from it when explicit path env vars are absent, add a narrow state-dir basename guard, and document intentional OS-home exceptions.

## Reproduction Assessment

Yes, at source level. With `OPENCLAW_HOME` set and more specific path env vars absent, current main still resolves the default dev checkout and installer bootstrap paths from OS-home values, and the `.openclaw/.openclaw` state/config edge case is visible in the helpers.

## Solution Assessment

Yes for a narrow repair through the existing home/path seams. The safer path is to reuse `resolveRequiredHomeDir`, add focused guards/tests, and document intentional OS-home exceptions rather than replacing every `HOME` or `os.homedir()` occurrence.
