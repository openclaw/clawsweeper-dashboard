## Summary

Keep open. Current main still only partially satisfies the documented `OPENCLAW_HOME` contract: central state/config and agent defaults use the effective home, but the dev git checkout, installer git/default bootstrap paths, and public docs still fall back to the OS account home or describe that fallback.

## What This Changes

The issue asks for `OPENCLAW_HOME` to drive OpenClaw state, config, git checkout, and related default paths unless explicit path overrides are set.

## Best Possible Solution

Keep the existing effective-home helper as the contract, update dev git checkout and installer bootstrap defaults to derive from it unless explicit path env vars are set, add focused regression coverage, and align public docs with any intentional OS-home exceptions.

## Reproduction Assessment

Yes, at source level. With `OPENCLAW_HOME` set and `OPENCLAW_GIT_DIR` unset, current main still resolves the dev checkout from `os.homedir()`, and `scripts/install.sh` still resolves git, workspace, and bootstrap config defaults from `$HOME`.

## Solution Assessment

Yes. The best fix is a narrow repair in existing path-resolution and installer seams, not a broad replacement of every OS-home use or a new configuration flag.
