## Summary

Keep open: current main and the latest release still only partially honor the documented OPENCLAW_HOME contract. State/config and agent defaults now use the effective home, but update/git checkout and installer/onboarding defaults still use OS HOME/os.homedir when the more specific path vars are unset.

## What This Changes

The issue asks OpenClaw to derive internal state, config, git checkout, and related defaults from OPENCLAW_HOME unless a more specific path environment variable is set.

## Best Possible Solution

Centralize the OPENCLAW_HOME contract for internal OpenClaw defaults, update the remaining update/git/install/onboarding defaults to use it only when specific overrides are absent, preserve intentional OS-home exceptions, and align tests plus docs.

## Reproduction Assessment

Yes, at source level. With OPENCLAW_HOME set and OPENCLAW_GIT_DIR or OPENCLAW_CONFIG_PATH unset, current main still routes update/git and installer/onboarding defaults through os.homedir or $HOME.

## Solution Assessment

Yes, a bounded repair through the existing home/path helpers is the maintainable fix. The safer path is targeted default resolution plus tests and docs, not a broad sweep of every HOME or os.homedir use.
