## Summary

Keep open. Current main still only partially honors OPENCLAW_HOME: state/config and default agent paths use the effective OpenClaw home, but the dev/git checkout default, installer bootstrap workspace/config checks, and user-facing docs still use or document the OS account home. Related #65536 is adjacent sandbox tilde work and does not cover these remaining update/install/docs gaps.

## What This Changes

Request that OPENCLAW_HOME drive OpenClaw's default state, config, git checkout, setup/onboarding, and related internal paths unless more-specific path environment variables are set.

## Best Possible Solution

Define the supported OPENCLAW_HOME contract for internal OpenClaw defaults, make the dev/git checkout and installer bootstrap defaults derive from it unless explicit path overrides are set, document intentional OS-home exceptions, and add focused regression coverage for the update CLI, installer, state/config, and agent defaults.
