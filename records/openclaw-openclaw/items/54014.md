## Summary

Keep open. Current main only partially satisfies the requested OPENCLAW_HOME contract: state/config and agent defaults now honor it, but the dev/git checkout path, installer bootstrap defaults, and public docs still use or document the OS account home for internal OpenClaw defaults.

## What This Changes

The issue asks for OPENCLAW_HOME to drive OpenClaw state, config, git checkout, and other internal path defaults unless narrower override variables are explicitly set.

## Best Possible Solution

Define the supported OPENCLAW_HOME contract for internal OpenClaw defaults, update the dev/git checkout and installer bootstrap paths to derive from the existing home-dir helper unless explicit path overrides are set, add focused regression tests, and align docs with any intentional OS-home exceptions.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: with OPENCLAW_HOME set and OPENCLAW_GIT_DIR unset, resolveGitInstallDir still uses os.homedir, and install.sh still initializes git/config/workspace bootstrap paths from HOME.

## Solution Assessment

Yes, a narrow repair in the existing path-resolution seams is the best route. The safer boundary is to fix internal OpenClaw defaults and docs while leaving explicitly OS-home host behavior and the separate sandbox tilde work in #65536 alone.
