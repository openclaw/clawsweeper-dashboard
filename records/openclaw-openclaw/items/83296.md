## Summary

Keep open. ## What This Changes

The PR changes model provider normalization and planning so plaintext auth-profile API keys are not persisted to generated `models.json`, and adds regression tests for normalization and merge mode.

## Best Possible Solution

Land one rebased canonical Layer 1 fix that keeps plaintext secrets out of prompt-visible model catalogs while preserving provider model rows, env/SecretRef markers, plugin catalog splitting, and explicit upgrade behavior for existing keys.

## Solution Assessment

No, not as-is against current main. The safer fix is to keep plaintext out of `models.json` while writing a prompt-safe marker or adding an explicit, tested change to the writable-provider contract.
