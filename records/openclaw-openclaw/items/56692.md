## Summary

Keep open. Current main and the latest release still lack the requested per-message target-bot metadata in group history and Telegram reply context, while the linked implementation PR remains open and unmerged; the community comments and earlier closed PR support the same missing context-separation problem rather than showing a shipped fix.

## What This Changes

The issue asks OpenClaw to make shared group-chat context distinguish messages addressed to this agent, another agent, or ambient group history, especially in Telegram groups with multiple agents.

## Best Possible Solution

Resolve this through the linked PR or a rebased successor that adds minimal shared targeting metadata, wires Telegram population, and explicitly decides how unrelated ambient history should be exposed.

## Reproduction Assessment

Yes, at source level: current main stores and renders Telegram group history without per-message mentioned/replied bot metadata. I did not run a live multi-bot Telegram repro in this read-only review.

## Solution Assessment

Unclear. The linked PR is a plausible implementation path, but maintainers still need to decide whether annotation alone is enough or whether unrelated history should be filtered or summarized.
