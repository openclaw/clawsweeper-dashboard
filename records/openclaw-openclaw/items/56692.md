## Summary

Keep open. Current main and v2026.6.6 still lack per-message bot-targeting metadata in group history and reply context, and the open PR that closes this issue remains unmerged, so the issue should stay as the canonical tracker.

## What This Changes

The issue requests clearer per-message target metadata and context separation for multi-agent Telegram group chats.

## Best Possible Solution

Keep this issue open until the linked PR or a successor lands minimal shared targeting metadata, Telegram population, tests, and an explicit decision on ambient history behavior.

## Reproduction Assessment

Yes, at source level. Current main records and injects Telegram group history without per-message mentioned-bot or replied-to-bot fields, so a message addressed to another bot can later appear as plain history for this agent.

## Solution Assessment

Unclear. The linked PR is a plausible implementation path, but maintainers still need to decide whether annotation alone is enough or whether unrelated history should be filtered or summarized.
