## Summary

Keep this issue open: current main and v2026.6.8 still flatten recent group history without per-message bot-target attribution, and the open closing PR is an unresolved implementation candidate rather than a merged fix.

## What This Changes

The issue requests clearer per-message targeting and context separation for multi-agent group chats, especially Telegram groups with multiple bot agents.

## Best Possible Solution

Keep this issue as the canonical tracker until the linked implementation PR is rebased, reviewed for the sibling-bot identity source, proven in a real Telegram two-bot group, and landed or replaced by a narrower maintainer-approved fix.

## Reproduction Assessment

Yes, at source level: current main stores and renders Telegram group history without an addressed-to-current-agent versus addressed-to-other-agent signal. A live Telegram two-bot group scenario is still missing for transport-level proof.

## Solution Assessment

Unclear as final product direction. Explicit target metadata is a plausible repair, but maintainers still need to choose the sibling-bot identity source and decide whether prompt annotation, history filtering, or hard response gating is the durable fix.
