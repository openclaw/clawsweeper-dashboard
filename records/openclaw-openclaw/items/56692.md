## Summary

Keep open. Current main and v2026.6.6 still lack per-message bot-targeting metadata in group history/reply context, while the linked implementation PR is open, unmerged, and currently conflicting; the community comments also confirm real-world workaround patterns rather than a shipped OpenClaw fix.

## What This Changes

Request clearer per-message targeting and reply-chain context in shared group-chat history so Telegram multi-agent groups do not treat another bot's thread as instructions for the active agent.

## Best Possible Solution

Keep the issue open and resolve it through the linked PR or a rebased successor that adds minimal shared targeting metadata, wires Telegram population, and explicitly decides the history filtering or ambient-summary behavior.

## Reproduction Assessment

Yes, at source level. Current main records and injects group history without per-message mentioned-bot or replied-to-bot fields, so a Telegram turn that mentions another bot can later appear as plain history for this agent.

## Solution Assessment

Unclear. The linked PR is the plausible implementation path, but maintainers still need to decide whether annotation alone is enough or whether unrelated history should be filtered or summarized.
