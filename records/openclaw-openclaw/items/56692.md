- `impact:message-loss`: The report is about group messages being interpreted or answered as if they were targeted to the wrong agent.

## Summary

Keep this open because current main and v2026.5.18 still render flat group-chat history without per-message bot-target metadata, and the linked closing PR remains open rather than merged.

## What This Changes

The issue asks for clearer per-message target context in Telegram multi-agent group chats so an agent can distinguish messages addressed to itself, another bot, or ambient group history.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/72510; the desired end state is optional bot-target metadata carried through history storage, prompt rendering, and Telegram mention/reply parsing without broad multi-agent policy changes first.

## Reproduction Assessment

Yes by source inspection, but not by a live Telegram run in this review. Current main stores skipped Telegram group history and renders InboundHistory without per-message mentioned/replied bot metadata, so a multi-bot group prompt can include another bot's addressed message as flat context.

## Solution Assessment

Yes, keeping this issue open behind the linked implementation candidate is the best current cleanup outcome. If that PR is rejected, a replacement should stay focused on optional target metadata and an approved sibling-bot/source signal rather than broad group-chat orchestration.
