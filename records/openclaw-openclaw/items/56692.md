## Summary

Keep open. Current main and v2026.6.6 still lack per-message bot-targeting metadata in group history/reply context, while https://github.com/openclaw/openclaw/pull/72510 is the open implementation path and still needs maintainer review/proof.

## What This Changes

Request clearer per-message targeting context in Telegram multi-agent group history so agents distinguish messages addressed to themselves, other bots, or ambient chat.

## Best Possible Solution

Resolve this through the linked PR or a rebased successor that adds minimal shared targeting metadata, wires Telegram population, and documents the annotation/filtering decision.

## Reproduction Assessment

Yes, at source level. Current main records and injects Telegram group history without per-message mentioned-bot or replied-to-bot fields, so a message aimed at another bot can later appear as plain history for this agent.

## Solution Assessment

Unclear. The linked PR is the plausible implementation path, but maintainers still need to decide whether metadata annotation is sufficient or whether unrelated history should be filtered or summarized.
