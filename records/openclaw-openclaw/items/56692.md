## Summary

Keep this open. Current main and the latest release still carry flat group history without per-message bot-target metadata, and the open implementation candidate that closes this issue has not merged.

## What This Changes

Request clearer per-message target metadata and context separation for Telegram multi-agent group history so agents can distinguish messages addressed to themselves, other bots, or ambient chat.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/72510; the desired end state is optional bot-target metadata carried through history storage, prompt rendering, and Telegram mention/reply parsing.

No live end-to-end Telegram reproduction was run. Source inspection gives a high-confidence reproduction path because skipped group history is stored and rendered without per-message target metadata.

## Solution Assessment
