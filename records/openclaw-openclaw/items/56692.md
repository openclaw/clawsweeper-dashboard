## Summary

Keep this open: current main still records and renders Telegram group history without per-message bot-target metadata, and the linked closing implementation at https://github.com/openclaw/openclaw/pull/72510 remains open, conflicting, and proof-gated. The issue discussion adds useful production context for explicit addressed-to-me/other/ambient metadata, so this should stay as the canonical tracker until that implementation path is landed or rejected.

## What This Changes

Clarify group-chat history context so the current agent can distinguish messages addressed to itself, another agent, or only ambient group context.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/72510; the desired end state is target metadata carried through history storage, prompt rendering, and Telegram mention/reply parsing without broad multi-agent orchestration first.

Yes, by source inspection: current main stores skipped Telegram group history and renders InboundHistory without per-message mentioned/replied bot metadata. I did not run a live Telegram multi-bot scenario in this read-only review.

## Solution Assessment
