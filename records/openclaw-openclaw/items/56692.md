## Summary

This issue should stay open: current main and the latest release still do not carry per-message target attribution for group history, and the open closing PR is the active but unresolved implementation path rather than a reason to close the tracker.

## What This Changes

The issue requests clearer per-message target attribution and context separation in multi-agent group chats, especially Telegram groups with multiple bots.

## Best Possible Solution

Keep this issue as the canonical tracker until the linked implementation PR is rebased, reviewed for the sibling-bot identity source, proven in a real Telegram two-bot group, and either landed or replaced by a narrower maintainer-approved fix.

## Reproduction Assessment

Yes at source level: current main still carries Telegram group history without an addressed-to-current-agent versus addressed-to-other-agent signal. I did not run a live Telegram two-bot group scenario in this read-only review.

Unclear as a final product direction. Explicit target metadata is a plausible repair, but maintainers still need to choose the sibling-bot identity source and decide whether prompt annotation, history filtering, or hard response gating is the durable solution.
