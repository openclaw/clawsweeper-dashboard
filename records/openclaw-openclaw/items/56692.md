## Summary

Keep this open. Current main still drops per-message bot-targeting metadata from Telegram group history and prompt context, and PR #72510 is open/unmerged and explicitly positioned as the implementation for this issue; the linked community triage gate and multi-agent kit are useful adjacent workarounds, but they do not close the bundled Telegram/core context gap.

## What This Changes

Request clearer per-message targeting and reply-chain context in multi-agent Telegram group history so agents do not treat messages for other bots as instructions for themselves.

## Best Possible Solution

Keep this issue open until PR #72510 is reviewed and either landed or rejected; the maintainable end state is optional bot-target metadata carried from Telegram mention/reply parsing through HistoryEntry/InboundHistory into prompt rendering, with focused current-bot, other-bot, and ambient-history coverage.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection does show the reported gap: current main has current-turn mention/reply signals but flat prior group history with no per-message bot target metadata.

## Solution Assessment

Yes, the narrowest maintainable direction is to review the linked end-to-end implementation candidate rather than close the issue. If that PR is not accepted, the replacement should stay within optional context metadata and Telegram population rather than adding broad multi-agent policy changes first.
