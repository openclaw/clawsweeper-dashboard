## Summary

Keep this open: current main and v2026.5.22 still carry Telegram group history as flat sender/body/timestamp/message metadata without per-message bot-target attribution, and the linked closing implementation remains open and conflicting. The prior closed metadata proposal plus the community comments and external workaround references are useful evidence for the need, but they do not close the OpenClaw core/Telegram gap.

## What This Changes

The issue requests clearer per-message target metadata and context separation for multi-agent group chats, especially Telegram groups with multiple bots.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/72510; the desired end state is target metadata carried through history storage, prompt rendering, and Telegram mention/reply parsing without broad multi-agent orchestration first.

## Reproduction Assessment

Yes, by source inspection: current main stores skipped Telegram group history and renders `InboundHistory` without per-message mentioned/replied bot metadata. I did not run a live Telegram multi-bot scenario in this read-only review.

## Solution Assessment

Yes for triage: keeping this issue open behind the linked implementation candidate is the best current outcome. If that PR is rejected, a replacement should stay focused on optional target metadata and an approved sibling-bot/source signal rather than broad group-chat policy.
