## Summary

Keep this open: current main still stores and renders Telegram group history without per-message bot-target metadata, and the only linked closing implementation remains an open PR rather than a merged fix.

## What This Changes

The issue requests clearer per-message targeting context in Telegram multi-agent group chat history so agents can distinguish messages addressed to themselves, other bots, or ambient group context.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/72510; the desired end state is optional bot-target metadata carried through history storage, prompt rendering, and Telegram mention/reply parsing without broad multi-agent orchestration first.
