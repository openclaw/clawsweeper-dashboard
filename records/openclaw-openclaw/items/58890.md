## What This Changes

The issue reports that macOS auto-update can kill its own gateway parent during LaunchAgent reinstall, breaking piped stdout/stderr before a detached restart handoff can recover the service.

## Best Possible Solution

Implement or land a service-safe auto-update handoff that runs outside the live gateway process tree, preserving the manual package-update guard and adding regression plus redacted launchd/systemd proof.

## Reproduction Assessment

Yes for the current-main remaining failure path: source shows the gateway auto-updater still launches a child from the live gateway, while the package-update preflight blocks gateway-owned children before install. I did not reproduce the original EPIPE crash live because current main now fails earlier by design.

## Solution Assessment

No. Current main's fail-closed guard prevents the dangerous self-bootout class, but it is not the complete solution because managed-service auto-update is still blocked; the open env-strip PR is partial unless it also handles the ancestry guard or moves the updater outside the gateway process tree.
