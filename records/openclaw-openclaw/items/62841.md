## Summary

Keep open: the parser-side alias in this PR is useful and now matches `/activate`, but the production WhatsApp group path still depends on shared command detection, and the PR does not register `/activate` there.

## What This Changes

The PR changes the group activation parser to accept `/activate` alongside `/activation`, adds parser tests for the alias, and adds an Unreleased changelog fix entry for #62827.

## Best Possible Solution

Register `/activate` as a text alias for the activation command, keep parser compatibility for `/activation`, and add coverage for both parser behavior and `hasControlCommand('/activate mention')` in the WhatsApp group mention-gating path.

## Reproduction Assessment

Yes. Current main reproduces the linked bug because `/activate mention` is absent from both the parser and command registry; the current PR fixes the parser-level case but still leaves `hasControlCommand('/activate mention')` false for the group-gating bypass.

## Solution Assessment

No. The parser regex is a necessary part of the fix, but the narrow maintainable solution also needs the shared command registry/detection alias so the command reaches the handler before mention gating skips it.
