## What This Changes

The PR changes the embedded runner tool-result context estimator to ignore `toolResult.details` and adds focused regression coverage for large details payloads.

## Best Possible Solution

## Reproduction Assessment

Yes. On current main, a tool result with small visible `content` and very large `details` is charged through `estimateUnknownChars(details)`, and that estimate feeds the preemptive overflow guard even though `convertToLlm` strips details before provider submission.

## Solution Assessment

Yes, with one release-note gap. Ignoring details in this estimator is the narrow maintainable fix because the LLM-boundary and compaction token paths already strip tool-result details; the PR should add the required changelog entry before merge.
