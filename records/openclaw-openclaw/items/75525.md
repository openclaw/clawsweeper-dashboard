## What This Changes

The PR removes `toolResult.details` from tool-result context-budget estimation, adds focused guard regression tests for details-heavy results, and records the user-facing fix in `CHANGELOG.md`.

## Best Possible Solution

Land this narrow estimator change with its regression tests and changelog once exact-head CI is complete, preserving the contract that `details` remains runtime/UI/persistence metadata while only model-visible tool-result content affects prompt context accounting.

## Reproduction Assessment

Yes. On current main, a tool-result message with small `content` and large `details` exercises `estimateMessageChars`, which charges `details`; the guard then consumes that inflated estimate even though provider replay and compaction strip `details`.

## Solution Assessment

Yes. Excluding `details` in the shared tool-result estimator is the narrowest maintainable fix; changing persistence, session JSONL, provider replay, or truncation policy would be broader than the observed mismatch.
