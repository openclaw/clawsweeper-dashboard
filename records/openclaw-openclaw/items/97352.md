## What This Changes

The PR routes ACP task cancellation through Gateway `tasks.cancel`, fails ACP CLI cancellation when Gateway cancellation is unavailable, dedupes ACP task rows across late-derived flow ids, and cancels same-run ACP rows together.

## Best Possible Solution

Land the Gateway-first ACP cancellation fix only after a focused Gateway `tasks.cancel` ACP regression, redacted real ACP spawn/cancel/readback proof, and maintainer acceptance of the fail-closed Gateway-required operator behavior.

## Reproduction Assessment
