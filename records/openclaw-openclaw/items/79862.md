## Summary

Keep open for maintainer review: the Docker/local-provider pain point is plausible, but this patch broadens the SSRF trust boundary by letting hostname allowlists bypass private-host and DNS-answer protections without explicit private-network opt-in.

## What This Changes

The PR changes SSRF checks so `hostnameAllowlist` can bypass `.internal`, `.localhost`, and `.local` hostname blocks and resolved private-IP checks, with regression tests for reserved hostname suffixes.

## Best Possible Solution

Preserve `hostnameAllowlist` as target scoping, keep private-network bypasses behind explicit opt-in, and add a focused `host.docker.internal` regression if that documented opt-in still fails on current main.

## Reproduction Assessment

Yes for the no-private-opt-in path by source inspection: current main blocks `host.docker.internal` when only the auto-generated `hostnameAllowlist` is present. The linked issue's claimed `allowPrivateNetwork` path is unclear because the supplied config omits that documented field and current main appears to pass it through.

## Solution Assessment

No. The patch makes the demonstrated Docker flow work by treating hostname allowlists as private-network trust, but the safer maintainable path is explicit `request.allowPrivateNetwork` or a maintainer-approved contract change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep hostname allowlists from bypassing private DNS checks:** `src/infra/net/ssrf.ts:159-161`
  - body: Adding `hostnameAllowlist` to `shouldSkipPrivateNetworkChecks` skips `resolvePinnedHostnameWithPolicy`'s DNS-answer check. An allowlisted provider host can then resolve to RFC1918, link-local, or metadata addresses without `allowPrivateNetwork`; keep this skip gated by explicit private-network policy or an approved contract change.
  - confidence: 0.89
- **[P1] Keep private IP literals behind explicit opt-in:** `src/infra/net/ssrf.ts:270-271`
  - body: This early allowlist return runs before `isPrivateIpAddress`, so an auto-generated allowlist for a private `baseUrl` such as `http://10.0.0.5` would pass without `request.allowPrivateNetwork`. Preserve the private-IP check unless the explicit private-network flag is set.
  - confidence: 0.86
