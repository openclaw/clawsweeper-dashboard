## Summary

Keep open for maintainer security review. The PR addresses a real Docker/local-provider pain point, but it broadens `hostnameAllowlist` from host scoping into private-network trust, which changes the SSRF boundary and should not merge without an explicit maintainer policy decision.

## What This Changes

The PR changes `src/infra/net/ssrf.ts` so `hostnameAllowlist` can bypass blocked local hostname suffixes and resolved private-network checks, with focused SSRF unit tests.

## Best Possible Solution

Preserve explicit private-network access as the trust boundary; if `host.docker.internal` still fails with `request.allowPrivateNetwork: true`, add a scoped regression and fix that path without making hostname allowlists a resolved private-IP bypass.

## Reproduction Assessment

Yes for the PR-introduced security regression by source inspection: a matching `hostnameAllowlist` now skips the DNS-answer checks in `resolvePinnedHostnameWithPolicy`. The original issue's claimed failure with documented `request.allowPrivateNetwork: true` remains unclear because the shown repro config omits that nested option.

## Solution Assessment

No. The patch fixes the demonstrated Docker flow by broadening the security contract, but the narrower maintainable path is to keep private-network trust explicit or get maintainer approval for changing `hostnameAllowlist` semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep hostname allowlists from bypassing private DNS checks:** `src/infra/net/ssrf.ts:159-161`
  - body: Adding `hostnameAllowlist` to `shouldSkipPrivateNetworkChecks` lets any matching provider host skip the resolved-address check in `resolvePinnedHostnameWithPolicy`, so a host-scoped allowlist can resolve to RFC1918, link-local, or metadata addresses without `allowPrivateNetwork`. Keep this skip gated by explicit private-network policy or a maintainer-approved contract change, and add a regression for an allowlisted hostname resolving to a blocked IP.
  - confidence: 0.9
