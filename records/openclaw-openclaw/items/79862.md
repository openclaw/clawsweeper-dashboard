## Summary

Keep open for maintainer/security review: the branch fixes the reported Docker flow by treating `hostnameAllowlist` as private-network trust, but current main and docs keep that as an explicit `request.allowPrivateNetwork` boundary, and the author later confirmed that explicit opt-in works.

## What This Changes

The PR changes SSRF hostname policy handling so `hostnameAllowlist` can bypass `.internal`, `.localhost`, and `.local` hostname blocks and resolved private-network checks, with focused tests for hostname allowlist behavior.

## Best Possible Solution

Keep private DNS and private-IP access gated by explicit private-network opt-in, and only broaden `hostnameAllowlist` semantics if security owners approve and the contract is documented and tested.

## Reproduction Assessment

Yes for the no-private-opt-in path by source inspection: current main blocks `.internal` hostnames and still validates private DNS answers unless private-network policy is explicitly enabled. The original Docker use case with `request.allowPrivateNetwork: true` is no longer a high-confidence failing path because the author confirmed it works.

## Solution Assessment

No. The patch fixes the demonstrated Docker flow by broadening `hostnameAllowlist` into private-network trust, but the narrower maintainable direction is to keep private-network access explicit unless maintainers approve a security contract change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep hostname allowlists from bypassing private DNS checks:** `src/infra/net/ssrf.ts:159-161`
  - body: Adding `hostnameAllowlist` here makes any matching provider host skip the resolved-address check in `resolvePinnedHostnameWithPolicy`. Provider fetches auto-generate that allowlist from `baseUrl` without `allowPrivateNetwork`, so this would let an allowlisted hostname DNS-rebind to RFC1918, link-local, or metadata addresses; keep this bypass gated by explicit private-network policy or an approved contract change.
  - confidence: 0.89
