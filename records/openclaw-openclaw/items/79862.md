The PR changes SSRF policy and tests so hostnameAllowlist entries bypass reserved hostname suffix checks and resolved private-network checks.

## Solution Assessment

No. This PR's hostnameAllowlist bypass is broader than needed; the merged exact-origin approach fixes the local-provider flow while keeping fake-IP allowlisting and private-network trust as separate contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep hostname allowlists from bypassing private DNS checks:** `src/infra/net/ssrf.ts:159-161`
  - body: Adding hostnameAllowlist to shouldSkipPrivateNetworkChecks makes auto-generated provider host allowlists skip resolved private-address validation. That can turn a configured hostname into private-network trust without request.allowPrivateNetwork, so keep this boundary behind exact-origin trust or explicit private-network opt-in.
