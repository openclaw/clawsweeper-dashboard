## Summary

Keep open. Current `main` and the latest release still serialize both OpenAI-compatible HTTP endpoints through text-only assistant output, while the agent result and media parsing paths already carry structured media metadata; the remaining work needs maintainer product/security direction for the public HTTP media shape and authorization model.

## What This Changes

The issue reports that agent-authored `MEDIA:` directives on `/v1/responses` and `/v1/chat/completions` are returned as literal assistant text instead of converted into safe outbound media content.

## Best Possible Solution

Add a shared outbound-media response shaping layer for both OpenAI-compatible HTTP endpoints that strips raw `MEDIA:` from visible text, consumes `mediaUrl`/`mediaUrls`, validates local/remote references under the existing outbound media policy, and emits a documented authenticated media representation with streaming and non-streaming parity.

## Reproduction Assessment

Yes, at source level. Both HTTP handlers still read only `payloads[].text` or assistant text deltas while the agent result contract and shared media parser carry structured media metadata; I did not run a live Gateway request in this read-only review.

## Solution Assessment

Unclear as an implementation plan. Stripping raw `MEDIA:` text is necessary, but maintainers still need to choose the safe public HTTP media shape and authorization behavior before a repair PR changes the endpoint contract.
