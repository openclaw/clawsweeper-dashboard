- `impact:security`: The reported failure can expose host-local filesystem paths to HTTP clients as raw assistant text.

## Summary

Keep open. Current main and the latest release still serialize both OpenAI-compatible HTTP endpoints through text-only assistant output while the agent result contract carries media metadata, so the reported media loss and raw-path exposure remain real.

## What This Changes

The issue asks OpenAI-compatible HTTP responses to translate outbound MEDIA/file artifacts into safe response content instead of exposing raw MEDIA directives as assistant text.

## Best Possible Solution

Design one security-reviewed outbound media contract for the OpenAI-compatible HTTP surfaces: consume mediaUrl/mediaUrls, strip raw directives, expose only scoped URLs or data content, update schemas/docs, and cover text/image/file/denied-media cases.

Yes. Source inspection gives a high-confidence repro path: current main still serializes both HTTP endpoint outputs from payload text while ignoring payload media fields; I did not run a live Gateway client in this read-only review.
