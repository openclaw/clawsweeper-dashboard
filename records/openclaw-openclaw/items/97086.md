## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Ship the MXC SDK through a supported runtime path:** `extensions/mxc/package.json:10-12`
  - body: This private bundled plugin declares `@microsoft/mxc-sdk` only in the nested extension package, but the published root package excludes `dist/extensions/*/node_modules/**` and postinstall explicitly does not install plugin package dependencies. In an installed OpenClaw package, the launcher will try to import the SDK from the bundled plugin runtime without any supported install path, so enabling `sandbox.backend: "mxc"` can fail outside a source checkout. Make this an external installable plugin with install metadata, or make the root/package dist intentionally own and prove the runtime dependency.
  - confidence: 0.87
- **[P2] Use the stable MXC schema unless maintainers accept dev config:** `extensions/mxc/src/mxc-backend.ts:87`
  - body: The backend hard-codes `0.8.0-alpha`, but the inspected `@microsoft/mxc-sdk@0.7.0` docs mark that schema as dev/experimental and explicitly recommend `0.7.0-alpha` for new code. This first-party sandbox path is a security boundary, so it should default to the stable current schema or include an explicit maintainer-approved reason and compatibility proof for the dev schema.
  - confidence: 0.8
