Full review comments:

- **[P3] Use Telnyx's documented streaming event names:** `extensions/voice-call/src/providers/telnyx.ts:211-218`
  - body: Telnyx documents media-stream lifecycle webhooks as `streaming.started` / `streaming.stopped` / `streaming.failed`, so these `call.streaming.*` cases and tests will not run for the documented callbacks. Please handle the documented names and read `failure_reason` so stream failures surface instead of being silently ignored.
  - confidence: 0.87
- **[P3] Read Telnyx stream_id from the top-level frame:** `extensions/voice-call/src/webhook/stream-frame-adapter.ts:171-174`
  - body: Telnyx start frames put `stream_id` at the top level while `start` carries `call_control_id`; this reads `start.stream_id` and otherwise falls back to the provider call id. That loses the real carrier stream id in bridge/talk metadata and makes the test fixture drift from the upstream frame shape.
  - confidence: 0.86
