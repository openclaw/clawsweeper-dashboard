Full review comments:

- **[P2] Expose a supported feedback selector:** `extensions/memory-core/src/cli.ts:107`
  - body: The new feedback commands require a `<chunk-id>`, but the normal `memory search` flow only shows score plus `path:start-end` and does not give users a stable chunk id. As written, typical users have no supported argument for `recall` or `correct`, so the feature is effectively unusable without inspecting internals.
  - confidence: 0.91
- **[P2] Gate feedback on the active backend:** `extensions/memory-core/src/cli.runtime.ts:783-784`
  - body: These commands write a builtin `memory-weights.json` store directly, but QMD search is a separate backend and does not read those weights. A QMD-backed user can see “Recall recorded” or “Correction recorded” even though subsequent searches will not apply the feedback, so the command needs backend-aware support or a clear failure.
  - confidence: 0.86
- **[P2] Merge on-disk weights before flushing:** `extensions/memory-core/src/memory/weighted-recall.ts:123`
  - body: `flush()` writes the whole cached map back to disk, so two feedback commands that load the same old file can update different chunk ids and then the last writer drops the first update. Use a lock or read-merge-write/atomic update path so feedback state is not silently lost.
  - confidence: 0.84
