- **[P2] Expose a supported feedback selector:** `extensions/memory-core/src/cli.ts:107`
  - body: The new commands require `<chunk-id>`, but the supported `memory search` output and `MemorySearchResult` contract expose path, line range, score, snippet, and source, not a chunk id.   - body: These handlers write `memory-weights.json` directly under the agent state dir without checking which memory backend is active. On current main, QMD is selected ahead of the builtin manager when configured, and the QMD search path never reads `WeightedRecallStore`, so QMD users can see “Recall recorded” while subsequent searches ignore it. The command should either integrate with QMD or fail clearly when the active backend cannot apply weights.
  - confidence: 0.88
- **[P2] Merge on-disk weights before flushing:** `extensions/memory-core/src/memory/weighted-recall.ts:123`
  - body: `flush()` writes the store's entire cached map back to `memory-weights.json`, so two feedback commands that start from the same snapshot can overwrite each other's updates. Current recall persistence already uses a lock and has concurrent-write coverage; this new state path needs the same merge-safe write pattern before it stores user feedback.
  - confidence: 0.9
