## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Expose a supported feedback selector:** `extensions/memory-core/src/cli.ts:107`
  - body: These commands require `<chunk-id>`, but the supported `memory search` flow only shows score/path/snippet in text mode and the public result contract has no `id`; QMD results also omit ids. A normal operator cannot discover a valid argument, so the feedback path is unusable without inspecting internals. Please expose a stable selector/id in search output or accept an existing supported selector.
  - confidence: 0.9
- **[P2] Gate feedback on the active backend:** `extensions/memory-core/src/cli.runtime.ts:783-784`
  - body: This constructs `WeightedRecallStore` directly from the agent state directory regardless of the configured memory backend. When memory is using QMD, searches come from `QmdMemoryManager` and never read `memory-weights.json`, so `recall`/`correct` can print success while future ranking is unchanged. Check the active manager/backend and either support QMD weighting or fail clearly.
  - confidence: 0.88
- **[P2] Merge on-disk weights before flushing:** `extensions/memory-core/src/memory/weighted-recall.ts:123`
  - body: `flush()` serializes the whole cached weights map over `memory-weights.json`. Two feedback commands can both load the old file, update different chunks, and whichever flushes last drops the other update, silently losing memory feedback. Use file locking or an atomic read-merge-write strategy.
  - confidence: 0.87
