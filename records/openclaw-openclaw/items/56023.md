## What This Changes

This PR adds three SkillFoundry-derived bundled agent skills plus memory-core weighted recall scoring, feedback CLI commands, per-agent `memory-weights.json` state, tests, and changelog text.

## Best Possible Solution

Split the optional skills to ClawHub unless maintainers explicitly approve bundling, and redesign memory feedback around a stable result selector, explicit builtin/QMD semantics, merge-safe state updates, focused tests, docs, and redacted real CLI proof.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Expose a supported feedback selector:** `extensions/memory-core/src/cli.ts:107`
  - body: The new feedback commands require a `<chunk-id>`, but the normal `memory search` flow only shows score plus `path:start-end` and snippet, and the public result type does not expose an id. Typical users therefore cannot discover a valid value for `recall` or `correct` without inspecting internals.
  - confidence: 0.88
- **[P2] Gate feedback on the active backend:** `extensions/memory-core/src/cli.runtime.ts:783-784`
  - body: These commands write `memory-weights.json` directly, but `getMemorySearchManager()` may return a QMD-backed manager whose search path never reads that store. In a QMD setup the CLI can print success while future searches ignore the recorded feedback, so the command should either support QMD explicitly or fail clearly for that backend.
  - confidence: 0.86
- **[P2] Merge on-disk weights before flushing:** `extensions/memory-core/src/memory/weighted-recall.ts:123`
  - body: `flush()` writes the entire cached map back to `memory-weights.json`. If two feedback commands run at the same time, both can load the old file and the last writer will silently drop the other update, losing user feedback state.
  - confidence: 0.84
