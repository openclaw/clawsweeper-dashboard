## What This Changes

The PR adds three SkillFoundry-derived bundled agent skills plus memory-core weighted recall with feedback CLI commands, JSON weight persistence, scoring integration, tests, and changelog entries.

## Best Possible Solution

Split the optional skills to ClawHub unless bundled promotion is explicitly approved, and land memory weighted recall only with a stable feedback selector, backend-aware builtin/QMD semantics, merge-safe persistence, docs/tests, a clean rebase, and real CLI proof.

## Reproduction Assessment

Yes for the PR review findings by source inspection: current main lacks a public chunk-id selector, QMD search is routed through a separate manager, and the proposed flush path writes the cached weight map wholesale. I did not execute the PR because this review is read-only.

## Solution Assessment

No, not as submitted. Weighted recall may be useful, but the maintainable path needs a supported selector, backend parity or clear gating, merge-safe persistence, and a maintainer decision on whether the SkillFoundry skills belong in core.

  - body: The new commands require <chunk-id>, but the public search result contract and normal/json CLI search output expose path, lines, score, and snippet rather than chunk ids.   - body: These handlers write the builtin memory-weights.json store directly without checking the active memory backend. When memory.backend=qmd is selected, searches go through QmdMemoryManager, so the CLI can report feedback success even though QMD rankings never read those weights; either apply the same weighting to QMD or fail clearly for unsupported backends.
  - confidence: 0.84

  - body: flush() writes the cached this.data snapshot over memory-weights.json. Two feedback commands in separate processes can load the same old file, update different chunks, and the last writer silently drops the first update, so the feedback state needs a file lock or atomic read-merge-write path.
  - confidence: 0.82
