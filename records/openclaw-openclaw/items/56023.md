  - body: These commands always write `memory-weights.json`, but current main can use `QmdMemoryManager`, which returns ranked results without reading that store. In QMD mode the CLI can print success while recall/correction has no ranking effect; either support QMD or fail with a clear backend-aware message.
  - confidence: 0.9
- **[P2] Merge on-disk weights before flushing:** `extensions/memory-core/src/memory/weighted-recall.ts:123`
  - body: `flush()` writes the cached map wholesale. Two feedback commands that update different chunks from the same starting file can silently lose one update when the later process wins; use a lock or read-merge-write strategy.
  - confidence: 0.88
- **[P3] Align correction output with behavior:** `extensions/memory-core/src/cli.runtime.ts:828`
  - body: The command tells users that new entries start at `0.70`, but it only demotes the supplied chunk and never records or links a replacement entry. A manually added replacement starts at the normal default, so the CLI promises behavior it does not implement.
  - confidence: 0.8
