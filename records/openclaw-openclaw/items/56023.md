- **[P2] Expose a supported feedback selector:** `extensions/memory-core/src/cli.ts:107`
  - body: The new commands require a `<chunk-id>`, but normal `memory search` output does not show one and the public result type does not include `id`; hybrid results also strip IDs before returning.   - confidence: 0.88
- **[P2] Gate feedback on the active backend:** `extensions/memory-core/src/cli.runtime.ts:783-784`
  - body: These handlers write a builtin-only `memory-weights.json` sidecar directly from the agent directory, while current memory search may be backed by QMD. In that setup the command can print success even though QMD search never reads the weights, so this should either route through the active backend or fail clearly when unsupported.
  - confidence: 0.86
- **[P2] Merge on-disk weights before flushing:** `extensions/memory-core/src/memory/weighted-recall.ts:123`
  - body: `flush()` writes the cached map wholesale, so two feedback commands that load the same old file can each update different chunks and the last writer silently drops the other update. Use locking or a read-merge-atomic-write path so feedback state is not lost.
  - confidence: 0.84
