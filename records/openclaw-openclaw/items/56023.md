## Summary

Keep this PR open. Current main does not implement weighted recall or memory feedback commands, but the branch should not merge as-is because the new feedback CLI is not usable from supported search output, feedback writes can lose concurrent updates, and the bundled SkillFoundry skills need maintainer product/scope judgment against the current ClawHub-first guidance.

## What This Changes

The PR adds three repo-root agent skill documents plus memory-core weighted recall scoring, feedback CLI commands, persistence tests, and changelog entries.

## Best Possible Solution

Split the work before merge. Keep weighted recall as a focused memory-core design PR with a supported feedback selector, consistent ranking behavior, concurrency-safe state persistence, docs, and targeted tests; route the three SkillFoundry skills through ClawHub or explicit maintainer approval for repo-bundled agent skills.

## Reproduction Assessment

Yes for the review findings. On the PR head, the feedback commands require a chunk id that normal memory search output and MemorySearchResult do not expose, and the lost-update issue follows from two WeightedRecallStore instances loading the same file, modifying different ids, and flushing in last-writer-wins order.

## Solution Assessment

No, not as submitted. Weighted recall may be a useful memory-core feature, but the proposed branch needs a stable feedback-target contract, locked/atomic read-merge-write persistence, a current-main rebase, and a maintainer decision on the bundled skills.
