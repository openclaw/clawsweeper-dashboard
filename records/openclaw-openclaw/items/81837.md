## Summary

Keep open: current main still has the Dashboard-to-Gateway path that can clear a compatible auth profile override during a model switch, and the open closing PR at https://github.com/openclaw/openclaw/pull/81886 has not merged yet.

## What This Changes

Dashboard model switching should preserve a compatible session `authProfileOverride` instead of clearing it and causing provider auth failure or fallback.

## Best Possible Solution

Land or replace the open server-side `sessions.patch` fix so compatible auth overrides are preserved with provider-auth alias handling, then close this issue from the merge.

## Reproduction Assessment

Yes, at source level. The Control UI sends only a model patch, the server call reaches `applyModelOverrideToSessionEntry` without profile or preserve data, and the current gateway test asserts that this clears the auth override fields.

## Solution Assessment

Yes. The narrow maintainable fix belongs in server-side `sessions.patch` so Dashboard and SDK callers share the same compatible-auth preservation behavior; the open linked PR is the active implementation candidate.
