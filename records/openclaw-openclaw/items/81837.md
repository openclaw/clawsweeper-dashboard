## Summary

Keep open: current main still contains the Dashboard-to-Gateway path that clears session auth overrides, and the linked fix at https://github.com/openclaw/openclaw/pull/81886 is open rather than merged.

## What This Changes

Dashboard model switches through `sessions.patch` should preserve a compatible `authProfileOverride` instead of clearing it and triggering auth/provider fallback failures.

## Best Possible Solution

Land or replace the server-side `sessions.patch` fix in https://github.com/openclaw/openclaw/pull/81886 with compatible-auth preservation and cross-provider cleanup regression coverage, then close this issue after merge.

## Reproduction Assessment

Yes. A model-only Dashboard request reaches the current `sessions.patch` branch that calls `applyModelOverrideToSessionEntry` without auth preservation, and the current gateway test asserts that this clears same-provider auth override fields.

## Solution Assessment

Yes. The best fix belongs in the server-side `sessions.patch` model path so Dashboard and other RPC callers share the same compatible-auth preservation behavior; the linked PR is the active implementation candidate.
