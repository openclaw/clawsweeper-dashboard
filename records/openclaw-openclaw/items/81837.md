## What This Changes

The issue reports that Dashboard model switches call `sessions.patch` without preserving `authProfileOverride`, clearing session auth state and causing provider auth failure or fallback.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/81886 so `sessions.patch` preserves provider-compatible auth overrides and clears only incompatible ones.

## Reproduction Assessment

Yes, at source level. The Dashboard sends a model-only `sessions.patch`, the gateway calls the shared helper without profile or preserve data, and the current test asserts that a same-provider model patch clears the auth override fields.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the server-side `sessions.patch` model path so Dashboard and SDK callers share the same compatible-auth preservation behavior; the open closing PR is the right candidate to finish reviewing.
