The PR moves exec approval file/socket resolution to the OpenClaw state dir, adds a doctor migration for legacy home policies and socket rewrites, updates diagnostics/docs, and adds state-dir regression tests.

## Best Possible Solution

Keep exec approvals canonical to the state dir, but make the doctor migration refuse or warn on symlinked canonical state-dir components before writing or unlinking, with focused regression coverage for that upgrade path.

## Reproduction Assessment

## Solution Assessment

No, not yet. Resolving runtime state through resolveStateDir and using doctor for legacy migration is the maintainable shape, but the migration must enforce the same symlink boundary before it writes or deletes policy files.
