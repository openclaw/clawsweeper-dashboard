## Summary

Keep open: the PR improves one warning path, but the linked downgrade failure can still hit invalid-config handling before that warning runs, and the user-facing fix also needs a changelog entry.

## What This Changes

The PR appends doctor and manual value-format repair guidance to the newer-config warning emitted by `warnIfConfigFromFuture`.

## Best Possible Solution

Share a future-version remediation formatter between valid-config warnings and invalid-config diagnostics, using raw/source `meta.lastTouchedVersion` so downgrade guidance appears even when validation fails.

## Reproduction Assessment

Yes. The linked downgrade report provides concrete invalid config shapes, and current-main source shows validation failure exits before `warnIfConfigFromFuture`, so a config with future `meta.lastTouchedVersion` plus an invalid future key or value reproduces the missing guidance path.

## Solution Assessment

No. Adding hints only to `warnIfConfigFromFuture` is safe but incomplete; the maintainable fix is to surface the same future-version remediation from invalid-config reporting and cover both paths with focused tests.
