## Summary

Keep open. The PR targets a real downgrade-config diagnostic gap, but the patch only changes the valid-config future-version warning while the reported invalid downgrade cases can still fail before that warning runs.

## What This Changes

The PR appends `openclaw doctor --fix` and manual value-format correction guidance to the future config version warning in `src/config/io.ts`.

## Best Possible Solution

Revise and rebase so valid future-config warnings and invalid-config/snapshot diagnostics share precise downgrade remediation, while keeping Doctor claims limited to unknown-key stripping unless value-format migration is implemented.

## Reproduction Assessment

Yes, by source inspection. A future-stamped config with unknown keys or changed value shapes fails validation in `loadConfig` or `readConfigSnapshot` before `warnIfConfigFromFuture` can emit the proposed guidance.

## Solution Assessment

No. Updating only `warnIfConfigFromFuture` is too narrow because the reported invalid downgrade configs can return or throw before that function runs; remediation should also surface from invalid-config and snapshot diagnostics.
