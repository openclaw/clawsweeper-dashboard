## Summary

Keep open: the PR addresses a real downgrade-config diagnostic gap, but the patch only updates the valid-config warning path and does not reach the invalid downgrade configs described by the linked issue.

## What This Changes

The PR appends `openclaw doctor --fix` and manual value-format correction guidance to the newer-config warning in `src/config/io.ts`.

## Best Possible Solution

Revise and rebase the PR so downgrade remediation is shared by valid newer-config warnings and invalid-config/snapshot diagnostics, while keeping `doctor --fix` claims precise about unknown-key stripping versus manual value-format repair.

## Reproduction Assessment

Yes, by source inspection. A future-stamped config with unknown keys or changed value shapes fails in `loadConfig` or `readConfigSnapshot` before `warnIfConfigFromFuture` can emit the proposed guidance.

## Solution Assessment

No. Updating only `warnIfConfigFromFuture` is too narrow because the reported invalid downgrade configs can return or throw before that function runs; remediation should also surface from invalid-config and snapshot diagnostics.
