## Summary

Keep this PR open, but not mergeable as-is: the patch adds useful wording to one valid-config warning path while the downgrade cases in the linked report can still fail validation before that warning runs, and the external contributor has not provided real behavior proof.

## What This Changes

The PR modifies `warnIfConfigFromFuture` in `src/config/io.ts` to append `openclaw doctor --fix` and manual value-format correction hints to the newer-config warning.

## Best Possible Solution

Revise and rebase the PR so downgrade remediation is shared by valid newer-config warnings and invalid-config/snapshot diagnostics, while keeping `doctor --fix` claims precise about unknown-key stripping versus manual value-format repair.

## Reproduction Assessment

Yes, by source inspection. A future-stamped config with unknown keys or changed value shapes fails validation in `loadConfig` or `readConfigSnapshot` before `warnIfConfigFromFuture` can emit the proposed guidance.

## Solution Assessment

No. Updating only `warnIfConfigFromFuture` is too narrow because the affected invalid downgrade configs can return or throw before that function runs; the safer fix is shared remediation in invalid-config and snapshot diagnostics too.
