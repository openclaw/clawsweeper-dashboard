Keep open. Current main does not fully supersede this PR, and the PR head still has blocking static/runtime defects; the Feishu portion also overlaps open #39489 while ACPX has moved to a newer Codex ACP/auth-bridge baseline.

The PR branch changes heartbeat system-event handling, Feishu channel health reporting, exec/process terminal-status semantics, ACPX Codex launch behavior, and first-visible reply diagnostics/config/docs.

## Best Possible Solution

Split or replace the branch with current-main-sized changes: handle Feishu health through #39489 or a coordinated replacement, keep heartbeat event consumption aligned with main's selected-consumption model, decide exec non-zero semantics with compatibility coverage, and rework ACPX only on the current 0.12.0/auth-bridge baseline.
