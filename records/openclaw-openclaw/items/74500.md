## Summary

Keep open. Current main intentionally ignores whole-agent `agentRuntime` keys and supports provider/model-scoped Claude CLI routing, but this issue still needs a maintainer decision on whether legacy Claude CLI intent should be migrated, fail closed, or emit a stronger diagnostic before any alternate provider is used.

## What This Changes

Request to honor or visibly diagnose Claude CLI runtime selection on cold-start installs instead of silently routing to a fallback provider.

## Best Possible Solution

Define and prove the supported Claude CLI cold-start contract: model/provider-scoped runtime policy should invoke Claude CLI on first turn, while legacy whole-agent runtime config should be migrated by doctor or fail closed with an explicit diagnostic before alternate provider routing.

## Reproduction Assessment

Partially. Source inspection is high-confidence that whole-agent `agentRuntime` is ignored on current main and model-scoped `claude-cli` is supported, but I did not run the exact packaged Telegram cold-start reproduction.

## Solution Assessment

Unclear. The provider/model-scoped runtime policy is the maintainable current direction, but changing legacy whole-agent behavior or provider-substitution privacy semantics needs an explicit maintainer decision.
