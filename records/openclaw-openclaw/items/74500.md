## Summary

Keep open. Current main has the right source-level Claude CLI routing and diagnostics seams, but the report is concrete, security-sensitive, and still lacks packaged/live proof that the fresh-install Telegram cold-start provider substitution is fixed.

## What This Changes

The issue reports that a fresh `2026.4.26` install ignores `agents.defaults.agentRuntime.id: "claude-cli"` for an Anthropic primary model and silently routes a Telegram turn to OpenRouter's free fallback.

## Best Possible Solution

Current main or the next package should execute `claude-cli` for `anthropic/*` when `agentRuntime.id` is configured, or fail closed with visible CLI-skip and fallback diagnostics plus packaged/Telegram regression coverage.

Unclear. The issue gives detailed versioned steps and side-by-side `.24`/`.26` evidence, but I did not verify the current main package through a fresh npm install, Telegram turn, and Claude CLI OAuth environment.
