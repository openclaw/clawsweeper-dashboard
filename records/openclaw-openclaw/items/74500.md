## What This Changes

The issue reports that a fresh `2026.4.26` install ignores `agentRuntime.id: "claude-cli"` and silently routes the first Telegram turn to an OpenRouter fallback instead of Claude CLI.

## Best Possible Solution

Current main should either execute the configured Claude CLI runtime for the first Telegram turn on fresh state or fail closed with visible runtime-skip and model-fallback diagnostics, backed by packaged or QA coverage for that cold-start path.
