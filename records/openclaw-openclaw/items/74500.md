## Summary

Keep open. Current main supports the model-scoped Claude CLI route and has generic fallback diagnostics, but the reported legacy whole-agent runtime config is still intentionally ignored and the exact cold-start Telegram/provider-substitution path is not proven fixed.

## What This Changes

The issue reports that a cold-start install with `agents.defaults.agentRuntime.id = "claude-cli"` silently routes to the OpenRouter fallback instead of invoking Claude CLI.

## Best Possible Solution

Define and prove the supported Claude CLI cold-start path: model-scoped runtime policy should invoke Claude CLI on the first turn, and legacy whole-agent runtime configs should get a maintainer-approved migration or clear fail-closed diagnostic.

## Reproduction Assessment

No high-confidence live reproduction for the exact packaged Telegram cold-start path. Source inspection does show the reported legacy key is ignored on current main while the supported model-scoped runtime path routes to Claude CLI.

## Solution Assessment

Unclear. The current provider/model-scoped runtime direction is maintainable, but deciding whether to migrate, reject, or fail closed for legacy whole-agent runtime config is a product/privacy policy decision.
