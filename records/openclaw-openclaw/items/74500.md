The issue reports that a fresh 2026.4.26 install ignores `agents.defaults.agentRuntime.id: "claude-cli"` and silently routes Telegram turns to OpenRouter instead of the configured Claude CLI runtime.

## Best Possible Solution

Define and prove the supported cold-start Claude CLI path: model-scoped `agentRuntime.id: "claude-cli"` should run Claude CLI on the first Telegram turn, while legacy whole-agent runtime configs should get a clear doctor/runtime diagnostic or a maintainer-approved compatibility migration.
