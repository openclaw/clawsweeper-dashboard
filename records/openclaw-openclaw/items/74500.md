## Summary

Keep open. Current main has source-level support for canonical Anthropic models routed through `agentRuntime.id: "claude-cli"`, plugin auto-enable, external CLI auth overlay, and fallback-decision logging, but the reported product failure is a fresh packaged Telegram cold-start silently crossing an explicit provider boundary. I did not find enough packaged/live evidence or a merged PR that proves that exact 2026.4.26-style substitution is fixed on current main.

## What This Changes

The issue reports that a cold-start `2026.4.26` install ignores `agents.defaults.agentRuntime.id = "claude-cli"` and silently routes Telegram turns to the OpenRouter fallback instead of dispatching through Claude CLI.

## Best Possible Solution

Current main should either execute the explicitly configured `claude-cli` runtime for the first Telegram turn on a clean packaged install or fail closed with visible runtime-skip and model-fallback diagnostics before any OpenRouter substitution. The durable fix should include a packaged or QA lane covering fresh state, scoped external CLI auth, plugin auto-enable, canonical Anthropic model refs, and first-turn channel dispatch.

Unclear. The issue gives detailed versioned steps and prior side-by-side evidence, but this review did not live-reproduce a fresh packaged install with Telegram plus Claude CLI OAuth on current main; existing tests cover narrower source-level routing and auth pieces only.
