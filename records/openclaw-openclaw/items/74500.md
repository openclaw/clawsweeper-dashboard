## What This Changes

The issue reports that a cold-start `openclaw@2026.4.26` install ignores `agentRuntime.id: "claude-cli"` and silently routes Telegram traffic to OpenRouter instead of the configured Claude CLI runtime.

## Best Possible Solution

Current main should either execute the explicitly configured `claude-cli` runtime for the first Telegram turn or fail closed with visible runtime-skip and model-fallback diagnostics before any OpenRouter substitution. The durable fix should include packaged or QA coverage for a fresh state, scoped external CLI auth, plugin auto-enable, and first-turn channel dispatch.
