## Summary

Keep open. Current main has the expected source-level `claude-cli` runtime mapping, dispatch path, plugin auto-enable coverage, and external CLI auth overlay, but there is still no high-confidence packaged/live cold-start proof that the reported Telegram turn cannot silently route to OpenRouter. The issue is also provider-boundary sensitive, so it should not be auto-closed from source inspection alone.

## What This Changes

The issue reports that a cold-start `agentRuntime.id: "claude-cli"` configuration on OpenClaw 2026.4.26 is ignored and routes directly to an OpenRouter fallback without visible failover or CLI-backend diagnostics.

## Best Possible Solution

Current main should either execute the explicitly configured `claude-cli` runtime on the first channel turn or fail closed with visible runtime-skip and model-fallback diagnostics before any third-party provider substitution. The durable fix should include packaged or QA coverage for a fresh state with Claude CLI OAuth, plugin auto-enable, external CLI auth overlay, and first-turn channel dispatch.
