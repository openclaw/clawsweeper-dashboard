## What This Changes

The issue reports that a fresh 2026.4.26 install ignores a configured claude-cli agent runtime and silently routes Telegram traffic to the OpenRouter free fallback model.

## Best Possible Solution

Current main should either execute the explicitly configured claude-cli runtime on the first Telegram turn of a clean packaged install or fail closed with visible runtime-skip and model-fallback diagnostics before any OpenRouter substitution. The durable fix should include packaged/live coverage for fresh state, canonical Anthropic model refs, external CLI auth, plugin activation, and first-turn channel dispatch.
