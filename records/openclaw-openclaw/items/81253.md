## Summary

Keep open: current main still hard-codes English exec approval prompt text and labels, and the existing Telegram exec approval config only covers routing and approvers. Related approval UX work exists, but it does not provide the requested i18n/template surface.

## What This Changes

Request to add localized or operator-customizable templates for exec approval prompts, especially Telegram/native chat approval messages.

## Best Possible Solution

Add a versioned, validated approval-rendering contract that supports safe localized labels or templates while preserving canonical approval commands and shared tests across Telegram and fallback chat clients.

## Reproduction Assessment

Not applicable: this is a feature request, not a report of broken existing behavior. Source inspection confirms the requested locale/template knobs are absent from current main.

## Solution Assessment

Unclear until maintainers choose the product boundary. The maintainable direction is likely a shared, validated approval-rendering contract rather than ad hoc per-channel string replacement.
