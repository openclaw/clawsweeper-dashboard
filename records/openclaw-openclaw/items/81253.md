## Summary

Keep open: current main still hard-codes English shared native/chat exec approval text, Telegram reuses that renderer, and Telegram config only exposes routing and approver controls. The web Control UI has localized approval labels, but that does not satisfy the requested Telegram/native chat template and locale support.

## What This Changes

Request to add localized or operator-configurable templates for exec approval prompts, especially Telegram/native chat approval messages.

## Best Possible Solution

Add a shared, validated approval-rendering contract with locale/template support that preserves canonical /approve commands, required metadata, expiration, and consistent behavior across native chat clients.

## Reproduction Assessment

Not applicable: this is a feature/config request, not broken existing behavior. Source inspection confirms current main lacks Telegram/native chat locale or template knobs and hard-codes English prompt labels.

## Solution Assessment

Unclear until maintainers choose the safety contract. The safer direction is a versioned shared renderer with required fields and tests, not arbitrary per-channel string substitution.
