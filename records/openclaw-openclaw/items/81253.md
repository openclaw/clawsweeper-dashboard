## Summary

Keep open: current main and the latest release still render shared exec approval prompts from fixed English strings, Telegram native and forwarded approval paths reuse that renderer, and config/docs expose routing controls but no locale or template contract.

## What This Changes

Request to add localized or operator-customizable templates for exec approval prompts, especially Telegram/native chat approval messages.

## Best Possible Solution

Add a validated shared exec-approval rendering contract with localized labels/templates while preserving canonical /approve commands, approval id, command preview, allowed decisions, expiry, warnings, and channel parity.

## Reproduction Assessment

Not applicable: this is a feature/config request rather than a broken behavior report. Source inspection on current main shows fixed English exec approval rendering and no locale/template config path.

## Solution Assessment

Unclear until maintainers define the safety contract. The maintainable direction is a shared validated renderer for all chat approval clients, not unconstrained per-channel string replacement.
