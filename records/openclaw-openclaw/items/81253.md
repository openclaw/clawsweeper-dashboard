## Summary

Keep open: current main still hard-codes shared English exec approval text/action labels, Telegram native approvals reuse that shared renderer, and the Telegram config/schema/docs expose routing and approver controls but no locale/template contract. This is a real feature/config request for a safety-sensitive approval surface, so it needs maintainer product review rather than conservative cleanup closure.

## What This Changes

Request i18n or template customization for exec approval prompts, especially Telegram/native chat approval messages.

## Best Possible Solution

Add a versioned, validated shared approval-rendering contract with localized labels/templates while preserving canonical /approve commands and safety details across Telegram, native clients, and fallback chat flows.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection gives high confidence that current main lacks locale/template knobs and still uses fixed English prompt labels.

## Solution Assessment

Unclear until maintainers decide the product/security boundary. The maintainable direction is a shared, validated approval-rendering contract rather than per-channel free-form string replacement.
