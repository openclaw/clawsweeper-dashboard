## Summary

Keep open: current main and v2026.6.6 still use fixed English exec-approval copy, and Telegram approval delivery delegates to that shared renderer without locale/template config. The request is useful, but it is a security-facing prompt and config/API-surface decision rather than stale cleanup or a safe automated fix.

## What This Changes

The issue requests localized or operator-customizable exec approval prompt templates, especially for Telegram/native chat approval messages.

## Best Possible Solution

Define a shared typed approval presentation/i18n contract that preserves mandatory safety fields, then let native channel renderers apply localized labels or approved layout presets.

## Reproduction Assessment

Yes for the feature gap: source inspection shows fixed English prompt rendering and no locale/template field in Telegram exec approval config. This is not a broken-existing-behavior bug reproduction.

## Solution Assessment

No, the raw template config sketched in the request is not clearly the safest permanent shape. A typed shared presentation/i18n contract with mandatory safety fields is the more maintainable direction.
