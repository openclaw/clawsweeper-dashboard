## Summary

Keep open: current main and the latest release still render exec approval prompts with fixed English text, and Telegram/native approval delivery has no locale or template contract. The request is useful but security-facing, so it needs maintainer product/API direction rather than cleanup closure.

## What This Changes

Add configurable localized labels or templates for exec approval prompts, especially Telegram and native chat approval messages.

## Best Possible Solution

Define a shared typed approval presentation/localization contract that preserves mandatory safety fields, then let native channels render localized labels or approved layout presets from that contract.

## Reproduction Assessment

Yes for the feature gap: source inspection shows fixed English exec approval rendering and no Telegram locale/template config fields. This is not a broken-existing-behavior bug reproduction.

## Solution Assessment

No for the raw template sketch as-is: a free-form template could hide safety-critical fields. A typed shared presentation/i18n contract with mandatory fields is the safer maintainable direction.
