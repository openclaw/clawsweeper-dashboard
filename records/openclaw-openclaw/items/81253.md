## Summary

Keep open: current main still hard-codes shared English exec approval chat copy and action labels, Telegram native approval delivery reuses that shared renderer, and the Telegram config/schema/docs have no locale or template contract. This is a feature/config request for a safety-sensitive approval surface, so it needs maintainer product review rather than cleanup closure.

## What This Changes

Add locale/template customization for exec approval prompts, especially Telegram/native chat approval messages.

## Best Possible Solution

Add a versioned, validated shared approval-rendering contract with localized labels/templates while preserving canonical /approve commands and required safety details across Telegram, other native chat clients, and fallback chat flows.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection gives high confidence that current main lacks locale/template knobs and still uses fixed English chat approval labels.

## Solution Assessment

Unclear until maintainers decide the product/security boundary. The safer direction is a shared, validated approval-rendering contract rather than per-channel free-form string substitution.
