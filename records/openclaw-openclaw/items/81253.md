## Summary

Keep open: current main still hard-codes English shared exec approval prompt text and action labels, Telegram native approvals reuse that renderer, and the Telegram config/schema/docs expose routing and approver controls but no locale/template contract. This is a valid safety-sensitive feature/config request rather than a cleanup-close candidate.

## What This Changes

The issue requests configurable localized templates or labels for exec approval prompts, especially Telegram/native chat approval messages.

## Best Possible Solution

Add a versioned, validated shared approval-rendering contract with localized labels/templates while preserving canonical `/approve` commands, approval IDs, command text, expiry, and action semantics across native chat clients.

## Reproduction Assessment

Not applicable: this is a feature/config request, not broken existing behavior. Source inspection confirms current main lacks locale/template knobs and hard-codes English chat approval text/action labels.

## Solution Assessment

Unclear until maintainers decide the product/security boundary. The safer direction is a shared, validated renderer contract rather than per-channel free-form substitution.
