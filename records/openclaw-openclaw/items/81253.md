## Summary

Keep open: current main still renders exec approval prompts with fixed English text and labels, and Telegram/native approval config has no locale or template surface. The request is valid as product/design work, but it adds security-sensitive approval-copy and config/API surface, so it should stay in maintainer review rather than become an automatic fix PR.

## What This Changes

The issue requests configurable or localized exec approval prompt templates, especially for Telegram/native chat approval messages.

## Best Possible Solution

Define a maintainer-approved, typed approval prompt localization/customization contract that preserves the canonical approval id, command, decision set, expiry, and safety metadata before adding any Telegram or gateway config keys.

## Reproduction Assessment

Yes from source: current main still builds fixed English approval prompt text and fixed English action labels, and the strict Telegram execApprovals schema has no locale/template keys. No live Telegram run is needed to establish that the requested customization surface is missing.

## Solution Assessment

Unclear as proposed: Telegram-only freeform templates would be a risky one-off config surface for safety copy. A shared typed approval view/localization contract, with tests for preserved commands and metadata, is the safer maintainable direction if maintainers approve the feature.
