## Summary

Keep open: current main still uses fixed English text and action labels for shared exec approval chat prompts, Telegram native and forwarded approvals reuse that renderer, and the Telegram config/schema/docs expose routing and approver controls but no locale/template contract. The nearby open prompt-summary PR addresses length/readability, not i18n or templates.

## What This Changes

Request adding localized or customizable templates for chat exec approval prompts, especially Telegram/native approval messages.

## Best Possible Solution

Add a validated shared exec-approval rendering contract with localized labels/templates while preserving canonical /approve commands, approval ID, command preview, allowed decisions, expiry, warnings, and fallback behavior across Telegram and other chat flows.

Not applicable: this is a feature/config request, not a broken existing behavior report. Source inspection on current main is enough to show chat exec approval prompts still use fixed English renderer text and no locale/template config.

## Solution Assessment

Unclear as requested until maintainers define the product/security contract. The safer direction is a shared, validated approval-rendering contract rather than unconstrained per-channel string replacement.
