## Summary

Keep open: current main still renders exec approval prompts with fixed English text and fixed action labels, and Telegram native/forwarded approval paths reuse that shared renderer without a locale or template config surface. This is a clear feature/config request on a safety-sensitive approval surface, so it needs maintainer product/security direction rather than cleanup closure.

## What This Changes

Request locale or operator-template support for exec approval prompts, especially Telegram/native chat approval messages.

## Best Possible Solution

Add a shared, validated exec-approval rendering contract with localized labels/templates while preserving canonical /approve commands, approval ID, command preview, allowed decisions, expiry, and warnings across Telegram and fallback chat flows.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a broken behavior report. Source inspection on current main confirms the shared exec approval renderer and Telegram native/forwarded paths still use fixed English text with no locale/template config.

## Solution Assessment

Unclear until maintainers define the product/security contract. The safer direction is a shared validated approval-rendering contract rather than unconstrained per-channel string replacement that can obscure approval semantics.
