## Summary

Keep open: current main still has fixed English exec approval prompt/action text, Telegram native and forwarded approval paths reuse that shared renderer, and the Telegram config/schema/docs do not expose a locale or template contract. Adjacent open PRs improve prompt length, prompt clarity, or plugin approval language, but none owns exec approval i18n/template support.

## What This Changes

Requested configurable or localized exec approval prompt templates for chat approval messages, especially Telegram/native approval flows.

## Best Possible Solution

Add a validated shared exec-approval rendering contract with localized labels/templates while preserving canonical /approve commands, approval ID, command preview, allowed decisions, expiry, and warnings across Telegram and fallback chat flows.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a broken-behavior bug report. Source inspection on current main shows fixed English exec approval rendering and no locale/template config surface.

## Solution Assessment

Unclear until maintainers define the product/security contract. The safer direction is a shared validated renderer/config contract rather than unconstrained per-channel string replacement that can obscure approval semantics.
