## Summary

Keep open: current main still hard-codes shared English exec approval prompt text and action labels, Telegram native approvals reuse that renderer, and the Telegram config/schema/docs do not expose locale or template fields. This is a valid new safety/config feature request that needs maintainer product review, not a stale or already-implemented bug fix.

## What This Changes

Request to add localized or operator-customizable templates for exec approval prompts, especially Telegram/native chat approval messages.

## Best Possible Solution

Design a shared, versioned approval-rendering contract with validated localized labels/templates while preserving canonical `/approve` commands, ids, command text, decision semantics, expiry, and per-channel affordances.

## Reproduction Assessment

Not applicable: this is a feature/config request, not broken existing behavior. Source inspection confirms current main lacks locale/template knobs and uses fixed English approval labels and prompt text.

## Solution Assessment

Unclear until maintainers decide the product and safety boundary. The safer maintainable direction is a shared, validated renderer contract rather than ad hoc per-channel string substitution.
