## Summary

Keep open: current main still hard-codes English exec approval prompt text and action labels, Telegram native approvals reuse that shared renderer, and the config schemas do not expose locale/template fields. Related approval UX issues cover mobile flow, fallback hints, or channel-specific clarity, but not the requested i18n/template contract.

## What This Changes

Request configurable localized or templated exec approval prompts, especially for Telegram/native chat approval messages.

## Best Possible Solution

Add a versioned, validated approval-rendering/localization contract shared by native and fallback approval clients, keeping canonical /approve commands and decision semantics uncustomizable and covered by tests/docs.

## Reproduction Assessment

Not applicable: this is a feature request, not broken existing behavior. Source inspection on current main confirms the requested locale/template knobs are absent.

## Solution Assessment

Unclear until maintainers choose the product boundary. A shared validated approval-rendering contract is safer than ad hoc per-channel string replacement because this prompt carries security-critical decision text.
