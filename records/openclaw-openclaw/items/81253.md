## Summary

Keep open: current main still hard-codes English exec approval prompt text/action labels, Telegram native and forwarded approval paths reuse that shared renderer, and the Telegram config/schema/docs expose routing controls but no locale/template contract. This is a user-facing safety-surface feature that needs maintainer product/security direction rather than an automated narrow bug fix.

## What This Changes

The issue asks for configurable localized or templated exec approval prompts, especially for Telegram/native chat approval messages.

## Best Possible Solution

Add a shared, validated exec-approval rendering contract that supports localized labels/templates while preserving canonical /approve commands, approval IDs, command preview, allowed decisions, expiry, warnings, and fallback behavior across Telegram and other chat clients.

## Reproduction Assessment

Not applicable: this is a feature/config request rather than a broken existing behavior report. Source, schema, docs, and tests on current main show fixed English exec approval rendering with no locale/template config surface.

## Solution Assessment

Unclear until maintainers define the product and safety contract. A shared validated renderer is the safer direction than unconstrained per-channel string replacement because approval prompts are a safety decision surface.
