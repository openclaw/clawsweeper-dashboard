## Summary

Keep open: current main and the latest shipped release still use fixed English exec approval prompt text, and Telegram native/forwarded approval paths consume that shared renderer without a locale/template contract.

## What This Changes

Request i18n or operator-configurable templates for exec approval prompts, especially Telegram/native chat approval messages.

## Best Possible Solution

Define a validated shared exec-approval rendering contract with localized labels/templates while preserving canonical approval commands and required safety facts across Telegram and fallback chat clients.

## Reproduction Assessment

Not applicable: this is a feature/config request rather than a broken existing behavior. Source inspection on current main and v2026.5.12 shows fixed English rendering and no locale/template config path.

## Solution Assessment

Unclear until maintainers set the safety contract. A shared, validated approval-rendering contract is the maintainable direction; unconstrained per-channel string replacement would risk obscuring approval semantics.
