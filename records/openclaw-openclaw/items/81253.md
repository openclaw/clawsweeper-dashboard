## Summary

Keep open. Current main still uses fixed English approval prompt text and fixed action labels, and neither the shared approval forwarding config nor Telegram exec-approval config exposes locale or template fields; this is a new UX/config feature that needs maintainer design rather than cleanup closure.

## What This Changes

The issue requests configurable or localized templates for exec approval prompts, especially Telegram/native chat approval messages.

## Best Possible Solution

Design a documented safe localization/customization contract at the shared approval view-model or renderer boundary, then wire schema/help/tests and Telegram/native adoption without weakening approval command integrity or redaction.

Not applicable: this is a feature/config request rather than a broken existing contract. Source inspection confirms current shared and Telegram approval prompts use fixed English strings and current schemas lack locale/template fields.
