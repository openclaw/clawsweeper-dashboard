## Summary

Keep open: current main still hard-codes shared English exec approval prompt text and action labels, Telegram uses that shared renderer for native and forwarded approval prompts, and the Telegram config/schema/docs expose routing and approver controls but no locale/template contract. Adjacent open pull requests improve prompt length or plugin approval language, but none owns exec i18n/template support.

## What This Changes

Request to add operator-configurable localization or templates for exec approval prompts, especially Telegram/native chat approval messages.

## Best Possible Solution

Add a versioned, validated shared exec-approval rendering contract with localized labels/templates while preserving canonical /approve commands, approval ID, command preview, allowed decisions, expiry, and warnings across Telegram and fallback chat flows.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a broken behavior report. Source inspection on current main shows the exec approval renderer and Telegram native/forwarded approval paths still use fixed English labels with no locale/template config.

## Solution Assessment

Unclear until maintainers set the product/security contract. The safer direction is a shared validated approval-rendering contract rather than unconstrained per-channel string replacement that can obscure approval semantics.
