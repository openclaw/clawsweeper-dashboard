## Summary

Keep open: current main has a shared English exec-approval renderer and Telegram native approvals delegate to it, but there is no locale or operator template surface for approval prompts. The request is useful, but it adds security-facing UX and config surface, so it needs maintainer product direction rather than stale cleanup or an automated fix.

## What This Changes

Request to add localized or operator-customizable templates and labels for exec approval prompts, especially Telegram/native chat approval messages.

## Best Possible Solution

Define a safe shared approval presentation/i18n contract with required canonical fields, then let channel renderers apply localized labels or approved layout presets with focused docs and tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection shows the requested capability is absent because current prompt rendering is fixed English text and Telegram exec approval config has no locale/template fields.

## Solution Assessment

No, the raw template config sketched in the request is not obviously the best permanent shape. A typed shared presentation/i18n contract with mandatory safety fields would be safer than arbitrary prompt text replacement.
