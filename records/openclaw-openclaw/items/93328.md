## Summary

Keep open for maintainer review: the PR is a focused candidate fix for the linked Matrix E2EE forced-reset double-reset bug, but it changes compatibility-sensitive cross-signing, secret-storage, recovery-key, and message-decryption state and overlaps with the open MAS/UIA bootstrap work.

## What This Changes

The PR changes MatrixCryptoBootstrapper forced-reset sequencing and tests so password-backed resets repair SSSS before cross-signing, passwordless resets keep pre-UIA SSSS deferral, and forced reset no longer enters the SSSS repair retry that can call bootstrapCrossSigning twice.

## Best Possible Solution

Land this focused bootstrapper fix after exact-head CI and Matrix-owner coordination with the MAS/UIA work; keep broader MAS reset URL behavior in its own PR.

## Reproduction Assessment

Yes. Current main and v2026.6.8 are source-reproducible: forced reset defers initial SSSS bootstrap, calls bootstrapCrossSigning with secret-storage recreation enabled, and the repair branch can recreate SSSS then call setupNewCrossSigning again.

## Solution Assessment

Yes. The patch is in the right MatrixCryptoBootstrapper layer and removes the duplicate forced-reset retry while preserving the passwordless pre-UIA SSSS mutation guard; broader MAS/UIA handling remains separate.
