## Summary

Keep open: this PR is not merge-ready because the diff adds a quality gate only inside the basic HTML fallback, while current `web_fetch` still accepts non-empty Readability output and provider payloads before that helper runs; it also lacks real behavior proof.

## What This Changes

This PR adds a meaningful-content filter to basic HTML extraction and updates web_fetch fallback tests for SPA-shell rejection and simple-body extraction.

## Best Possible Solution

Implement a narrow guard where `runWebFetch` accepts Readability or provider results, falling back to basic HTML only for confirmed title/head-only output while preserving documented fallback behavior and adding real proof plus regression coverage.

No high-confidence real reproduction is present. Source inspection shows the relevant current-main path, but the linked issue and this PR do not provide a real URL, minimal served HTML, or after-fix `web_fetch` output.

## Solution Assessment
