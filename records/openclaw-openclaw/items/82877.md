## Summary

Keep open: the patch is not ready because the new raw-HTML quality gate can drop valid short pages, while the linked title/head-only Readability problem is better handled at the Readability acceptance/fallback point. The external PR also lacks after-fix real behavior proof.

## What This Changes

The PR adds a `hasMeaningfulBodyContent` filter to `extractBasicHtmlContent` and updates `web_fetch` fallback tests for title-only SPA shells and simple HTML extraction.

## Best Possible Solution

Fix title/head-only Readability acceptance in `web-fetch.ts`, then fall back to existing basic HTML cleanup without rejecting valid short body content, with one canonical PR carrying real behavior proof and regression coverage.

## Reproduction Assessment

Yes for the PR-introduced regression from source: the new 10-character cutoff rejects valid short body text when Readability returns null and no provider is available. The linked user report still lacks a concrete URL or HTML sample, so that original case was not live-reproduced.

## Solution Assessment

No. The patch gates the final raw-HTML fallback; the safer fix is to treat title/head-only Readability output as failed extraction and then use the existing fallback without a blanket body-length cutoff.
