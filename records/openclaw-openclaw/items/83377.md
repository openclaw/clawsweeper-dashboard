## Summary

Keep open: the PR is protected by the `maintainer` label and still introduces a delivery-state regression for explicit no-deliver cron jobs that successfully send through the message tool.

## What This Changes

The PR adds a shared internal `SourceDeliveryPlan`/`SourceDeliveryOutcome` contract and migrates isolated cron delivery planning, dispatch, and tests to use it instead of cron-local booleans.

## Best Possible Solution

Keep the shared plan direction, but make explicit no-deliver cron targets message-tool-owned with no runner fallback, add a regression test for delivered state and presentation-warning clearing, then provide redacted real cron proof or a maintainer proof override.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main treats a matched message-tool send as delivered, while the PR's `cron_none` plan cannot mark the same send as satisfying delivery.

## Solution Assessment

No. The central contract is a maintainable direction, but this implementation must preserve explicit no-deliver message-tool sends as completed visible delivery while still disabling runner fallback.
