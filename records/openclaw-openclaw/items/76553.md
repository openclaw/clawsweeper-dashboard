## What This Changes

The issue requests Windows Claude Code discovery for npm-global installs and stable Gateway behavior after attempting to add `%APPDATA%\npm` to the Scheduled Task PATH.

## Best Possible Solution

Resolve Windows CLI discovery through the existing `agents.defaults.cliBackends.claude-cli.command` contract or a Windows npm-global resolver, and handle Gateway restart stale-listener behavior through the existing restart-health path rather than adding `models.*` config.

Yes, at source level for the detection half: when the OpenClaw/Gateway process PATH lacks the npm global directory, current main probes only `claude` and has no Windows npm-global fallback. The restart loop is also source-plausible through current unknown-listener stale cleanup, but needs Windows logs to pin the exact loop.

## Solution Assessment

Unclear for the full issue, because it mixes command discovery with restart stability. The narrow maintainable path is to reuse `agents.defaults.cliBackends.claude-cli.command` and/or add Windows npm-global resolution, while coordinating restart cleanup with #50136.
