Fix the Windows Claude Code detection gap from https://github.com/openclaw/openclaw/issues/76553. On Windows, when `claude` is installed globally by npm under the user npm prefix but the OpenClaw/Gateway process PATH does not include that directory, setup/models should not report `Claude Code: not found` solely because the bare command lookup misses `%APPDATA%\npm\claude.cmd`. Reuse the existing `agents.defaults.cliBackends.claude-cli.command` contract or add a narrowly scoped Windows Claude CLI resolver; do not add a new `models.claude-code.path` config key, do not freeze arbitrary PATH into `gateway.cmd`, and do not weaken trusted internal system-binary resolution. Add regression coverage for the resolver/setup surface and update Windows-facing docs/doctor copy if behavior changes. The restart-loop half is related to #50136; only change restart-health code if you can prove a narrow source-level fix from current code, otherwise leave that path for the existing PR/log-driven follow-up. Add a user-facing changelog entry.
