## Summary

Keep open. Current main still uses PATH-only Claude Code probing for the setup/Crestodian model surface, while the Windows Scheduled Task restart-loop half is not proven fixed and overlaps an open, unmerged restart-stabilization PR.

## What This Changes

The issue reports that OpenClaw on Windows does not detect a globally npm-installed Claude Code binary and that a manual Gateway PATH workaround leads to a restart loop.

## Best Possible Solution

Add a narrow Windows-aware Claude CLI discovery path using known npm global locations or the existing `agents.defaults.cliBackends.claude-cli.command` contract, and keep the Gateway restart-loop diagnosis tied to the Windows Scheduled Task restart surface until live evidence separates it.

## Reproduction Assessment

Yes, at source level: on Windows, if the OpenClaw/Gateway process PATH lacks the npm global directory, `probeLocalCommand("claude")` cannot find `%APPDATA%\npm\claude.cmd`, so the reported `Claude Code: not found` state follows. The Gateway restart loop was not live-reproduced here.

## Solution Assessment

Yes for the detection half: the narrow maintainable fix is Windows command resolution or persisting a resolved path through `agents.defaults.cliBackends.claude-cli.command`. The restart-loop half is still unclear and should not be folded into a broad service rewrite without Windows proof.

## Repair Work Prompt

Fix the Windows Claude Code detection gap reported in https://github.com/openclaw/openclaw/issues/76553. On Windows, when `claude` is installed globally by npm under the user npm prefix but the OpenClaw/Gateway process PATH does not include that directory, `openclaw setup`/Crestodian `models` should detect Claude Code instead of reporting `Claude Code: not found`. Keep the fix narrow: add or reuse a Windows command resolver that checks known npm global locations such as `%APPDATA%\npm\claude.cmd` and `%LOCALAPPDATA%\npm\claude.cmd`, or persist the resolved command through the existing `agents.defaults.cliBackends.claude-cli.command` contract. Do not add a new `models.claude-code.path` config key, do not freeze arbitrary user PATH into `gateway.cmd`, and do not attempt broad Gateway restart rewrites in the same PR. Add focused regression coverage for the resolver/probe and the Crestodian model surface; update docs/changelog if user-facing behavior or diagnostics change. Treat the restart-loop half as related context and stop for maintainer review if reproducing it requires broader Windows Scheduled Task lifecycle changes.
