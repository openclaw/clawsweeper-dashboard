## What This Changes

This PR adds a default per-task timeout to the command queue, a timeout error type and enqueue option, warning diagnostics, and tests for recovering a blocked lane.

## Best Possible Solution

Keep the shipped caller-owned, progress-aware queue timeout on main and close this stale branch as superseded; handle any remaining reliability work in narrower follow-ups rather than merging the blanket default timeout.

## Reproduction Assessment

Yes, for the original behavior at source level: a never-settling lane task with maxConcurrent=1 keeps queued work behind it blocked until the active task is released. Current main now has regression coverage for that exact shape, so it is no longer a current-main failure.

## Solution Assessment

No, this PR branch is no longer the best way to solve it because the global 15-minute default can override caller-owned run budgets. The shipped main implementation keeps the queue primitive generic and lets agent/cron callers provide the correct timeout and progress signal.
