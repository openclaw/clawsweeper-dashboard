## Summary

Current main now implements the PR's intended lane liveness fix with a safer caller-owned timeout contract. The PR branch should not be merged as-is because it adds a default command-queue timeout for every task, while main preserves existing long/no-timeout agent and cron policies and applies lane timeouts from the owning runtime paths.

## What This Changes

The PR adds a command-lane task timeout error, a `taskTimeoutMs` enqueue option, timeout handling around active queue tasks, diagnostic logging, and command-queue regression tests.

## Best Possible Solution

Keep the current main implementation and close this PR as superseded by that mainline fix. The linked lane-liveness bug should be reviewed against commit `470098bd26f3a12d88d50b2d9ffc0219b3a69643` and closed separately if maintainers agree it covers the reported production failure.

## Reproduction Assessment

Yes. A high-confidence focused reproduction is a `maxConcurrent=1` lane with a never-settling first task and queued second task; current main covers that with a command-queue test using `taskTimeoutMs: 25` and verifies the second task drains.

## Solution Assessment

No for this PR branch; the hard default timeout is not the best solution. Current main's opt-in queue timeout plus caller-derived embedded-run and cron budgets is narrower and avoids false failures for legitimate long-running work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve caller-owned timeout budgets:** `src/process/command-queue.ts:142`
  - body: This default applies a 15-minute timeout to every queue entry that does not pass `taskTimeoutMs`. Existing agent runs can be configured for much longer or effectively no timeout, and cron has its own 10-minute/60-minute/disabled policy, so this can reject legitimate work while the original task keeps running and later lane work starts against shared state. The mainline fix avoids this by making queue timeouts caller-owned.
  - confidence: 0.9
