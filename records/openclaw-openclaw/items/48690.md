## Summary

## What This Changes

The PR adds a task-level timeout option and default timeout wrapper around command-queue lane tasks, plus timeout-focused command queue tests.

## Best Possible Solution

Keep the shipped caller-owned timeout implementation on main and close this branch instead of merging its global default-timeout policy.

## Reproduction Assessment

Yes. The queue timeout behavior is source-proven and covered by the current command-queue regression test; the PR branch defect is also source-reproducible at the hard default timeout assignment.

## Solution Assessment

No. The PR branch is not the best fix because it imposes a default timeout on callers that omit taskTimeoutMs; current main's caller-owned budget wiring is narrower and already shipped.
