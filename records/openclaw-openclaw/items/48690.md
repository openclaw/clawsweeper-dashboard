## What This Changes

This PR adds a task-level timeout option and timeout tests for the command lane so a never-settling task can release the lane and queued work can continue.

## Best Possible Solution

Keep the shipped caller-budget timeout implementation and close this stale branch instead of merging its hard default timeout path.

## Solution Assessment

Yes for current main, no for this PR branch. The shipped implementation is the narrower maintainable solution because it uses caller-owned timeout budgets and progress renewal rather than adding a hard queue-wide fallback timeout.
