## Summary

Keep this PR open: it addresses the wrapper-backed doctor/update repair path with useful tests and sufficient live proof, but the gateway start-repair path still does not pass existing ProgramArguments into the new preservation logic and can regenerate a direct command for wrapper-only services.

## What This Changes

The PR adds wrapper inference/preservation to gateway install planning and service audit, threads existing service argv through install and doctor repair callers, and adds wrapper-preservation regression coverage.

## Best Possible Solution

Land a complete gateway service repair that preserves or refuses wrapper-backed commands across install, doctor/update, audit, and start-repair paths while keeping SecretRef-loaded wrappers intact.

## Reproduction Assessment

Yes, as a source-level reproduction path: current main audits wrapper-only ProgramArguments as missing `gateway` and missing `PATH`, and its install plan ignores argv-only wrappers. The PR body also supplies redacted live macOS before/after output, though I did not run a live LaunchAgent update in this read-only review.

## Solution Assessment

No, not yet: the install/doctor/audit direction is the right maintainable fix, but the same preservation input needs to be threaded through start repair so all service rewrite paths share the same wrapper contract.
