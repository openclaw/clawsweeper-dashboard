## Summary

Keep open for maintainer review. Current main still suppresses fallback and fallback-cleared user notices when verbose mode is off, so the PR is not obsolete; however, it is currently dirty/unmergeable and overlaps with #69399’s broader opt-in config approach, so maintainers should choose the desired UX policy before merge.

## What This Changes

This PR changes the reply runner to surface existing model fallback and fallback-cleared notices for Telegram chats even when verbose mode is off, and expands reply-runner e2e helper/tests for Telegram and non-Telegram coverage.

## Best Possible Solution

Choose the intended fallback-notice UX policy, then land one coherent implementation that keeps fallback state/lifecycle handling unchanged, widens only the selected user-visible notice gate, includes focused reply-runner coverage for fallback and fallback-cleared notices, and documents/releases the user-facing behavior.
