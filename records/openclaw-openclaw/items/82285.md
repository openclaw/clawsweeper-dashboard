## Summary

Keep this PR open. The Telegram interleaving direction is useful, but the latest head still changes documented reasoning-stream cleanup semantics, double-formats reasoning output, and leaves transient draft retries able to skip the failed text.

## What This Changes

The PR changes Telegram dispatch and draft-stream behavior to render CLI tool progress, reasoning text, and a rolling timer together in one Telegram reasoning message.

## Best Possible Solution

Keep the Telegram renderer behind the existing stream semantics: consume the canonical CLI bridge after it lands, preserve `/reasoning stream` cleanup unless maintainers explicitly choose a new persistent mode, use the splitter output without reformatting it, and add focused Telegram dispatch/draft-stream coverage plus latest-head live proof.

## Reproduction Assessment

Yes for the PR defects: source inspection against current main docs/tests shows the reasoning stream persistence and double-formatting paths, and the draft-stream retry state follows from the shared loop contract. This is a feature PR rather than a current-main bug report.

## Solution Assessment

No. The rendering goal is reasonable, but the current patch should preserve the existing reasoning-stream lifecycle, reuse the formatted splitter output correctly, fix transient retry cache handling, and wait for the canonical CLI event bridge.
