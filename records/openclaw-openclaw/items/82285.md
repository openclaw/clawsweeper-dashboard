## Summary

Keep open for now: this is a fresh draft feature PR, but the current branch mixes the tool-marker change with the separate experimental Claude interactive backend, overlaps the focused CLI tool-progress PR at https://github.com/openclaw/openclaw/pull/80046, and has blocking correctness/security findings before maintainer review.

## What This Changes

The PR adds Claude stream-json tool-use tracking that injects a textual marker with a rolling timer and emits `stream: "tool"` events, while also carrying the experimental `claude-cli-interactive` backend/proxy changes and unrelated doctor/changelog edits.

## Best Possible Solution

Rebase or split this into a focused tool-progress change that preserves the sanitized bus contract, adds parser and auto-reply regression tests, supplies redacted real Telegram/log proof, and reconciles with https://github.com/openclaw/openclaw/pull/80046 and https://github.com/openclaw/openclaw/pull/81851 instead of bundling both efforts together.

## Reproduction Assessment

Yes for the review findings: PR-head source inspection shows raw tool args/details are surfaced, thinking events can be buffered behind title filtering, lightweight provider artifacts drift, and fallback binding stays hard-coded. The user-facing feature itself still lacks inspectable after-fix proof.

## Solution Assessment

No in the current shape. The narrower maintainable path is to preserve sanitized CLI tool-event parity, coordinate with the focused open tool bridge PR, and keep the experimental interactive backend/security decision separate unless maintainers explicitly accept bundling them.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Sanitize tool details before publishing:** `src/agents/cli-output.ts:484-506`
  - body: The new tracker pulls `args.command`, `file_path`, `query`, `url`, and similar raw fields into the assistant text marker and also forwards the raw `args` in `stream: "tool"`. Native tool events call `sanitizeToolArgs` before publication, so this can leak tokens or private command/URL details to channel text and external event consumers.
  - confidence: 0.9
- **[P2] Stream thinking events before title filtering:** `extensions/anthropic/interactive-proxy/wrapper.ts:191-192`
  - body: The wrapper starts buffering at `message_start` and routes every event through `emitOrBuffer`, so `thinking_delta` events that arrive before the first non-title `text_delta` are held until later text or `message_stop`. That defeats the live reasoning stream for extended-thinking turns whose thinking arrives before assistant text.
  - confidence: 0.91
- **[P2] Keep lightweight provider artifacts aligned:** `extensions/anthropic/register.runtime.ts:489`
  - body: The runtime provider now treats `claude-cli-interactive` as an Anthropic hook alias, but the lightweight provider discovery and policy artifacts still recognize only `claude-cli`. Metadata/control-plane paths that rely on those artifacts can miss synthetic auth and thinking policy for the new backend.
  - confidence: 0.86
- **[P2] Read the matching CLI session on fallback:** `src/agents/command/attempt-execution.ts:396`
  - body: This predicate now includes `claude-cli-interactive`, but the fallback prelude still reads the hard-coded `claude-cli` session binding. Fallbacks from the interactive backend will drop the prior CLI session context instead of carrying the matching session into the retry.
  - confidence: 0.83
