## Summary

Keep this PR open: the latest head fixed the earlier `toolCallId` and terminal tick replacement shape, but the inline marker privacy gate still does not match the current session-verbosity contract, and the real behavior proof is incomplete for the full changed behavior.

## What This Changes

This PR adds Claude stream-json tool-use tracking to the CLI JSONL parser, inline assistant markers with a rolling timer, and structured `stream: "tool"` events from Claude CLI execution paths.

## Best Possible Solution

Use the same live tool-verbosity resolver as `server-chat` at tool-emission time, keep structured events aligned with the existing `toolCallId` contract, add focused parser/runner tests, and refresh Telegram plus structured-event proof.

## Reproduction Assessment

Yes for the PR finding by source inspection: current `server-chat` re-checks session verbosity changes after run start, while this PR captures a one-shot inline marker flag before streaming begins. No current-main bug reproduction is required because this is a feature PR.

## Solution Assessment

No. The structured event direction is plausible, but inline assistant text must not use a weaker privacy gate than the existing tool-event delivery path; the narrow fix is to share or defer to the same dynamic resolver.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate inline markers on live session verbosity:** `src/agents/cli-runner/execute.ts:468-473`
  - body: The new inline marker flag is captured once before the Claude run and only considers the registered run context plus config default. Current tool-event delivery re-reads the session entry and lets a session verbose change after run start override the run context, so if a user turns verbose off during a long run, structured tool events stop reaching chat surfaces but these inline markers can still inject command/path/query details through assistant deltas. Move the inline decision to tool emission time or share the existing session-aware resolver.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: The diff can leak tool names and selected args through assistant text after a session-level verbose-off change because the inline marker gate is weaker than the existing tool-event privacy contract.

Concerns:

- **[medium] Inline tool details can bypass session verbose off:** `src/agents/cli-runner/execute.ts:473`
  - body: Tool args may include commands, paths, queries, or URLs. Because inline marker injection is decided once from run context/default, a later session verbose-off change that current `server-chat` respects for tool events will not stop the parser from sending those details as assistant deltas.
  - confidence: 0.9
