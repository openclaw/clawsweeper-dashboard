## Summary

Keep this PR open: the Telegram UX direction is useful, but the current patch is not merge-ready because the interleaved path bypasses existing tool-progress privacy settings, bypasses the Telegram reasoning splitter, and the transient-send change can drop an unchanged retry after a 429/5xx.

## What This Changes

The PR adds Telegram-side interleaving of reasoning text and CLI tool-progress lines with a rolling timer in one reasoning-lane draft message, plus transient-error handling in Telegram draft streams.

## Best Possible Solution

Keep the rendering layer, but make it consume the canonical CLI bridge, reuse the existing Telegram progress setting and reasoning-splitter contracts, and repair transient retry caching before merge.

## Reproduction Assessment

Yes for the patch defects: source inspection of the potential merge commit shows the new onToolStart path bypassing the existing progress gate, and the draft-stream retry issue follows from the sent-cache update before the transport call plus the loop's pending-text retry behavior. I did not run a live Telegram reproduction.

## Solution Assessment

No. The best path is to keep the Telegram-side idea, but wire it through the canonical https://github.com/openclaw/openclaw/pull/80046 bridge and preserve the existing Telegram progress, reasoning, and retry contracts instead of adding a competing rendering path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Honor the tool-progress opt-out before interleaving:** `extensions/telegram/src/bot-message-dispatch.ts:1765`
  - body: `injectToolLineIntoInterleave` runs before the existing `pushStreamToolProgress` gate, so when reasoning streaming is active it can render tool details even for users who set `streaming.preview.toolProgress: false` or `streaming.progress.toolProgress: false`. That setting is documented as keeping tool updates out of the edited Telegram preview, and this path can expose command/path/URL details anyway.
  - confidence: 0.94
- **[P2] Retry failed transient sends with a clean sent cache:** `extensions/telegram/src/draft-stream.ts:283-287`
  - body: The transient-error branch returns `false`, but `lastSentText` and `lastSentParseMode` were already updated before the transport call. The draft loop retries the same pending text after `false`, and that retry will hit the same-text fast path and be treated as delivered without calling Telegram again, so an unchanged preview or final flush can still be lost after a 429/5xx.
  - confidence: 0.88
- **[P2] Keep reasoning text on the existing splitter path:** `extensions/telegram/src/bot-message-dispatch.ts:1698-1705`
  - body: The new reasoning interleave slices raw `payload.text` and wraps each line in underscores, bypassing `splitTelegramReasoningText`. Providers that stream `<think>`-style wrappers or mixed answer/reasoning text can now show raw tags or malformed formatting in Telegram instead of the current tag-stripped reasoning message.
  - confidence: 0.86
