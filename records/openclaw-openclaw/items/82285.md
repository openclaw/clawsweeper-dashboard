## Best Possible Solution

Land one canonical CLI tool-event bridge first, then keep this Telegram rendering layer behind the existing tool-progress config contract, reuse the reasoning splitter, clear timers after queued work drains, and add focused Telegram dispatch coverage plus fresh live proof.

Full review comments:

- **[P1] Honor Telegram tool-progress settings:** `extensions/telegram/src/bot-message-dispatch.ts:1734-1738`
  - body: This renders interleaved tool lines whenever a reasoning draft exists, before `pushStreamToolProgress` can apply `streaming.preview.toolProgress` or `streaming.progress.toolProgress`.   - confidence: 0.9

  - body: The new reasoning branch appends the raw new text and italicizes each line, bypassing `splitTelegramReasoningText`. Existing reasoning payloads such as `<think>Thinking</think>` will display raw tags instead of the current `Thinking\n\n_Thinking_` rendering, so reuse the existing splitter before interleaving.
  - confidence: 0.88
- **[P2] Clear the timer after queued draft work drains:** `extensions/telegram/src/bot-message-dispatch.ts:1853`
  - body: `clearActiveTimer()` runs before `await draftLaneEventQueue`, but a queued `injectToolLineIntoInterleave` task can then call `startToolTimer()` after that cleanup. Move the final timer clear after the queue drains so ended or superseded turns do not leave an interval waking every 3 seconds.
  - confidence: 0.84
