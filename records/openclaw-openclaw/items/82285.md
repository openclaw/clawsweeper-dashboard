## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor Telegram tool-progress settings:** `extensions/telegram/src/bot-message-dispatch.ts:1734-1738`
  - body: This renders interleaved tool lines whenever a reasoning draft exists, before `pushStreamToolProgress` can apply `streaming.preview.toolProgress` or `streaming.progress.toolProgress`. Users who set those false can still see tool details in the edited Telegram preview, so gate interleaving on the same resolved tool-progress setting or fall back without rendering.
  - confidence: 0.9
- **[P2] Preserve the existing reasoning splitter:** `extensions/telegram/src/bot-message-dispatch.ts:1658-1675`
  - body: The new reasoning branch appends raw reasoning text and italicizes each line, bypassing `splitTelegramReasoningText`. Existing reasoning payloads such as `<think>Thinking</think>` will display raw tags instead of the current `Thinking\n\n_Thinking_` rendering, so reuse the existing splitter before interleaving.
  - confidence: 0.88
- **[P2] Clear the timer after queued draft work drains:** `extensions/telegram/src/bot-message-dispatch.ts:1853`
  - body: `clearActiveTimer()` runs before `await draftLaneEventQueue`, but a queued `injectToolLineIntoInterleave` task can then call `startToolTimer()` after that cleanup. Move the final timer clear after the queue drains so ended or superseded turns do not leave an interval waking every 3 seconds.
  - confidence: 0.84
