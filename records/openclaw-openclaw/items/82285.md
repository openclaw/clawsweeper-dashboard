## Review Findings

Full review comments:

- **[P1] Use the session-aware verbose gate for inline markers:** `src/agents/cli-runner/execute.ts:467-473`
  - body: The resolver only checks `getAgentRunContext` plus the config default. Current channel delivery also reloads the session entry when it changed after the run started, and `sessions.patch` persists verbose changes there, so turning verbose off during a long run can stop structured tool events while these assistant-delta markers keep exposing tool details.
  - confidence: 0.9
- **[P1] Redact inline marker details before emitting them:** `src/agents/cli-output.ts:471-475`
  - body: The marker detail is formatted from raw Claude args such as commands, paths, queries, descriptions, and URLs. Existing tool surfaces sanitize these values because they can contain tokens or headers; this path can send the raw value directly into chat text.
  - confidence: 0.88
- **[P1] Sanitize tool-event args before publishing them:** `src/agents/cli-runner/execute.ts:506`
  - body: The new `stream: "tool"` emitters forward `evt.args` directly. The embedded runtime sanitizes tool args before publishing the same event stream, so WS/session subscribers can receive secrets from Claude CLI events that the current runtime would mask.
  - confidence: 0.88
