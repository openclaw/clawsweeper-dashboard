## Summary

Keep open; this draft PR is useful but not mergeable because inline markers are still gated once at run start and can ignore later session verbose-off changes, and the real behavior proof does not cover the latest privacy and structured-event behavior.

## What This Changes

The PR adds Claude stream-json tool-use tracking to the CLI JSONL parser, optional inline assistant markers with rolling timer updates, and structured `stream: "tool"` events from Claude CLI execution paths.

## Best Possible Solution

Gate inline marker emission with the same session-aware verbose resolver used by `server-chat` at each tool start, keep structured tool events on `toolCallId`, add focused parser/runner coverage, then refresh Telegram plus structured-event proof.

## Reproduction Assessment

Yes for the PR finding by source inspection: start a run with verbose enabled, update the session to verbose off before a later Claude tool event, and the current PR head keeps inline marker injection enabled because the flag was computed once at run setup. No current-main bug reproduction is required because this is a feature PR.

## Solution Assessment

No. The structured event shape is closer after the follow-up commits, but the inline assistant-text path must use the same session-aware privacy contract as channel tool events before this is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Recheck verbose before emitting inline markers:** `src/agents/cli-runner/execute.ts:468-473`
  - body: The inline marker flag is resolved once before the turn starts. Current tool-event routing lets a session verbose change made after run start override the original run context, so a user who turns verbose off during a long Claude turn can still receive later tool names and args through assistant text.
