## Summary

Keep this PR open: the Telegram UX idea is useful, but the current branch still bypasses existing Telegram tool-progress controls and reasoning normalization, and the supplied proof does not yet show the full rolling-timer flow.

## What This Changes

The PR changes `extensions/telegram/src/bot-message-dispatch.ts` so Telegram reasoning drafts can include timestamped tool-start lines and a rolling elapsed timer in the same edited reasoning message.

## Best Possible Solution

Keep the feature in the Telegram dispatcher only after interleaving reuses the existing progress opt-out/filtering and reasoning normalization paths, then land it with redacted live Telegram proof of the timer tick and cleanup.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: enable reasoning streaming, set `streaming.preview.toolProgress: false`, then invoke `onToolStart`; the new interleave branch renders before the existing opt-out gate. A second focused source check is `onReasoningStream({ text: "<think>Thinking</think>" })`, which now bypasses the existing tag-stripping path.

## Solution Assessment

No. The Telegram dispatcher is the right surface, but the implementation should reuse existing progress gating and reasoning formatting instead of creating a parallel renderer that bypasses those contracts.
