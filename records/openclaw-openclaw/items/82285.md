## Summary

This PR should stay open, but the current patch is not merge-ready: the rescoped Telegram layer is focused, yet it still bypasses existing Telegram progress privacy settings, replaces the reasoning splitter with raw italicization, and can leak the rolling timer after queued draft work drains.

## What This Changes

The PR changes Telegram dispatch to interleave reasoning text, tool-progress lines, and a rolling timer into one reasoning-lane draft message, with fallback to the existing progress renderer when the reasoning lane is unavailable.

## Best Possible Solution

Land or explicitly depend on the canonical CLI tool-event bridge, then keep the Telegram interleave implementation behind the existing progress settings and reasoning splitter with timer lifecycle tests and refreshed live Telegram proof.

## Reproduction Assessment

Not applicable as a current-main bug report; this is feature work. The patch defects are source-reproducible from the PR head against current Telegram progress and reasoning contracts.

## Solution Assessment

No. The Telegram rendering idea is plausible, but this implementation should reuse the existing progress visibility gate and reasoning splitter instead of creating a parallel message path with different privacy and lifecycle behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Gate interleaved tool lines through progress settings:** `extensions/telegram/src/bot-message-dispatch.ts:1735`
  - body: The new interleave path runs before `pushStreamToolProgress`, and `injectToolLineIntoInterleave` only checks for a reasoning stream. That bypasses the existing `streaming.preview.toolProgress` / suppression checks, so a user who disabled tool progress can still get command, file path, or URL details rendered into the reasoning message.
  - confidence: 0.93
- **[P1] Reuse the Telegram reasoning splitter:** `extensions/telegram/src/bot-message-dispatch.ts:1671-1674`
  - body: This replaces `ingestDraftLaneSegments(payload, true)` with direct slicing of raw `payload.text` and underscore wrapping. A normal reasoning payload like `<think>Thinking</think>` will render the tags literally instead of the current `Thinking\n\n_Thinking_` output covered by the reasoning splitter tests.
  - confidence: 0.91
- **[P1] Clear the rolling timer after draft work drains:** `extensions/telegram/src/bot-message-dispatch.ts:1855-1856`
  - body: The finalizer calls `clearActiveTimer()` before `await draftLaneEventQueue`; a queued interleave task can then run during that await, call `startToolTimer()`, and leave a fresh interval with no later clear. Move the clear after the queue drains or make queued work unable to start the timer during shutdown.
  - confidence: 0.86
