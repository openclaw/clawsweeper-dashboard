## Summary

Keep open. The branch is a plausible Claude CLI tool-visibility feature, but the current diff still uses a weaker inline-marker privacy gate than the existing live tool-event path and publishes raw CLI tool args without the sanitizer used by the embedded runtime; the supplied proof also does not cover the rolling timer or structured event path.

## What This Changes

This PR adds Claude stream-json tool-use tracking to the CLI JSONL parser, inline assistant markers with a rolling timer, and structured `stream: "tool"` events from Claude CLI execution paths.

## Best Possible Solution

Land a coordinated Claude CLI tool-event bridge only after sharing the live verbosity contract, sanitizing event and inline payloads, adding focused parser/runner tests, and refreshing Telegram plus structured-event proof.

## Reproduction Assessment

Yes for the PR findings by source inspection: PR head computes inline marker permission once and publishes `evt.args` directly, while current main re-checks session verbosity for tool events and sanitizes embedded tool args. No current-main bug reproduction is required because this is a feature PR review.

## Solution Assessment

No. The parser callback direction is reasonable, but the current patch should not land until it reuses the existing dynamic privacy contract, sanitizes payloads, and is reconciled with the existing Claude CLI tool-event parity PR.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Re-check live verbosity before injecting markers:** `src/agents/cli-runner/execute.ts:468-473`
  - body: The inline marker flag is still captured before the Claude run and only looks at run context plus the config default. Current tool-event delivery re-reads the session entry when it changed after the run started, so a user can turn verbose off during a long run and stop structured tool chat events while these assistant deltas continue exposing command/path/query details. Move the inline decision to tool-emission time or share the session-aware resolver.
  - confidence: 0.9
- **[P1] Redact CLI tool args before publishing events:** `src/agents/cli-runner/execute.ts:505`
  - body: The new CLI `stream: "tool"` events put `evt.args` directly on the shared agent-event bus. Embedded tool events already run args through `sanitizeToolArgs` before publication because commands, URLs, headers, and nested fields can contain secrets; apply the same sanitizer at both new CLI emit sites before any WS or session subscriber can receive the payload.
  - confidence: 0.92

## Security Review

Status: needs_attention
