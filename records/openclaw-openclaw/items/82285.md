## Summary

Keep open. The feature direction is plausible and the PR includes partial live Telegram proof, but the current patch bypasses channel tool-verbosity/privacy controls, emits structured tool events with the wrong ID field for current consumers, and can leave the rolling timer in streamed chat text.

## What This Changes

The PR adds Claude stream-json tool-use tracking in the CLI JSONL parser, injects tool markers/timers into assistant deltas, and emits `stream: "tool"` events from Claude CLI execution paths.

## Best Possible Solution

Route Claude tool starts through the existing tool/progress event contracts, respect channel verbose/privacy settings for any user-visible text, emit the current `toolCallId` contract for structured consumers, and add focused parser plus live-chat merge regression coverage with refreshed real proof.

## Reproduction Assessment

Yes for the PR findings by source inspection: current main's channel and UI contracts require verbose-gated tool delivery and `data.toolCallId`, while the PR injects assistant text unconditionally and emits `itemId` only. No current-main bug reproduction is needed because this is a feature PR.

## Solution Assessment

No, not as currently implemented. The maintainable shape is to use the existing structured tool/progress path and live-chat replacement semantics instead of smuggling tool details through assistant deltas.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Respect channel verbosity for tool details:** `src/agents/cli-output.ts:597`
  - body: `onToolText` sends the tool name and selected args through assistant deltas, so every normal reply surface receives them even when tool verbose is off. Current channel delivery intentionally gates `stream: "tool"` details behind verbose; this bypass can expose commands, paths, queries, or URLs in chats and cannot be disabled by not subscribing to the new event.
  - confidence: 0.93
- **[P2] Emit a toolCallId for tool stream consumers:** `src/agents/cli-runner/execute.ts:492`
  - body: The new structured tool event only includes `itemId`, but current tool-stream consumers drop `stream: "tool"` events without `data.toolCallId`. That means dashboards/tool drawers will not render these new events; include the existing `toolCallId` field, possibly alongside `itemId`, at both call sites.
  - confidence: 0.9
- **[P2] Use replacement semantics when clearing the timer:** `src/agents/cli-output.ts:662`
  - body: After a timer tick, the next text delta strips the timer from the full `text` but keeps `delta` as the provider text. The live-chat merger appends non-empty deltas when the new text no longer starts with the previous value, so channel output can keep the stale `_ Ns ..._` timer and append the resumed text instead of replacing it.
  - confidence: 0.86
