## Summary

Keep open: the Telegram-only rescope is still potentially useful, but the current patch is not merge-ready because it bypasses existing Telegram tool-progress privacy/settings, bypasses the reasoning formatter, can leave the rolling timer alive after queued cleanup, and still depends on the separate CLI event bridge at https://github.com/openclaw/openclaw/pull/80046.

## What This Changes

This PR changes Telegram dispatch to interleave reasoning text, tool-progress lines, and a rolling timer into one reasoning-lane draft message, with fallback to existing progress rendering when the reasoning lane is inactive.

## Best Possible Solution

Land or otherwise establish the canonical CLI tool-event bridge first, then implement Telegram interleaving by composing the existing progress gates and reasoning splitter, with focused dispatch tests and refreshed Telegram proof.

## Reproduction Assessment

No current-main bug reproduction is applicable because this is feature work. The patch defects are source-reproducible from the diff against current Telegram dispatch code and the existing progress-setting and reasoning-formatting contracts.

## Solution Assessment

No, not in its current form. The Telegram rendering layer is a plausible place for the feature after the CLI bridge exists, but this implementation needs to reuse the existing settings, reasoning splitter, and timer lifecycle contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Honor disabled Telegram tool-progress settings:** `extensions/telegram/src/bot-message-dispatch.ts:1734-1738`
  - body: This renders `formatted` into the reasoning lane before `pushStreamToolProgress` can enforce `streaming.preview.toolProgress`, `streaming.progress.toolProgress`, or `streamToolProgressSuppressed`. Users who set those false can still see command/path/URL detail in the edited Telegram preview, so gate interleaving on the same checks or avoid rendering the line.
  - confidence: 0.9
- **[P2] Preserve the existing reasoning splitter:** `extensions/telegram/src/bot-message-dispatch.ts:1658-1675`
  - body: This branch slices raw `payload.text` and italicizes each line instead of going through `splitTelegramReasoningText`. Tagged streams such as `<think>Thinking</think>` will show raw tags or partial tag prefixes rather than the current `Thinking\n\n_Thinking_` contract, so feed the new reasoning text through the existing splitter/formatter before appending it.
  - confidence: 0.86
- **[P2] Clear the timer after queued draft work drains:** `extensions/telegram/src/bot-message-dispatch.ts:1853-1854`
  - body: Cleanup clears the timer before awaiting `draftLaneEventQueue`. A queued `injectToolLineIntoInterleave` can run during that await and call `startToolTimer()`, leaving an interval active while the lanes are being stopped; clear again after the queue drains or prevent timer starts once cleanup begins.
  - confidence: 0.82
