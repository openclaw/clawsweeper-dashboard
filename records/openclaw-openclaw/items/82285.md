## Summary

Keep open: this is an active draft PR that addresses a real Claude CLI tool-event gap related to https://github.com/openclaw/openclaw/issues/75846, but the current head still has a blocking verbose/privacy contract issue and incomplete real behavior proof.

## What This Changes

This PR adds Claude stream-json tool-use tracking in the CLI JSONL parser, optional inline assistant markers with a rolling timer, and `stream: "tool"` events from Claude CLI execution paths.

## Best Possible Solution

Thread the resolved run/session verbose level into the CLI parser, keep the structured event in the existing `toolCallId` contract, add focused parser/live-chat regression coverage, and refresh Telegram plus structured-event proof.

## Reproduction Assessment

Yes for the PR findings by source inspection: a run with global `verboseDefault: "on"` but session/run verbosity `off` will still enable inline markers from the PR's default-only gate. No current-main bug reproduction is required because this is a feature PR.

## Solution Assessment

No. The current follow-up fixes the earlier `toolCallId` and next-text replacement issues, but the inline marker gate still bypasses the resolved run/session verbosity contract and terminal timer cleanup can leave stale text visible.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate inline markers on the resolved run verbosity:** `src/agents/cli-runner/execute.ts:466-468`
  - body: The new `injectToolInlineMarkers` flag is derived only from `agents.defaults.verboseDefault`. If the default is `on` but the current session or run has verbose `off`, this still sends tool names and args through assistant deltas, bypassing the same per-run/session privacy gate that `server-chat` applies to tool events. Thread and use the resolved run verbosity instead of recomputing from the global default.
  - confidence: 0.91
- **[P2] Replace the timer when a result ends the stream:** `src/agents/cli-output.ts:648-650`
  - body: The `result` and `finish()` paths clear the interval and strip `assistantText` locally, but they do not emit a replacement assistant delta. If a rolling timer tick has already been sent and Claude ends without another text delta, live chat can keep showing the stale `_ Ns ..._` suffix after the turn is done.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The PR still has a concrete privacy issue because inline tool details are gated by the global default instead of the resolved session/run verbosity.

Concerns:

- **[medium] Inline tool details can bypass session verbose off:** `src/agents/cli-runner/execute.ts:466`
  - body: Tool names and selected args can include commands, paths, queries, or URLs. Because the PR enables inline assistant markers from `agents.defaults.verboseDefault`, a session or run explicitly set to verbose `off` can still receive those details when the global default is `on`.
  - confidence: 0.9
