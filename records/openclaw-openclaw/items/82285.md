Full review comments:

- **[P1] Honor disabled tool-progress settings before interleaving:** `extensions/telegram/src/bot-message-dispatch.ts:1735`
  - body: `injectToolLineIntoInterleave` succeeds whenever the reasoning stream exists, so `streaming.preview.toolProgress: false` and other suppression paths never reach `pushStreamToolProgress`'s checks. That can show command, path, or URL details to users who explicitly disabled tool-progress lines; gate the interleaved path through the same policy or return false when progress is disabled.
  - confidence: 0.94
- **[P2] Keep the reasoning splitter on the interleaved path:** `extensions/telegram/src/bot-message-dispatch.ts:1668-1675`
  - body: Replacing `ingestDraftLaneSegments(payload, true)` with raw italicization bypasses `splitTelegramReasoningText`, so `<think>` wrappers and partial reasoning tag prefixes can render in Telegram instead of being stripped or suppressed. Append the splitter's reasoning output to the interleaved body rather than raw `payload.text`.
  - confidence: 0.88
- **[P2] Reset the sent-text cache on transient send failures:** `extensions/telegram/src/draft-stream.ts:287`
  - body: The transient catch returns `false` after `lastSentText` has already been set. The draft loop keeps the same text pending on `false`, so the next retry of that exact text hits the duplicate guard and is treated as delivered even though Telegram never accepted it; clear the cache on transient failure or assign it only after a successful send/edit.
  - confidence: 0.86
