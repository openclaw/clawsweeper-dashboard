## Summary

Keep open: current `main` and latest `v2026.6.6` still register the Google memory adapter as `gemini` while lookup by `memorySearch.provider: "google"` does not resolve through `authProviderId`. The issue also has open closing PRs, so the right cleanup path is maintainer review of those fixes rather than closing before merge.

## What This Changes

The issue reports that `openclaw memory status --index` and `--deep` reject `memorySearch.provider: "google"` because the Google embedding adapter is registered under `gemini`.

## Best Possible Solution

Choose and document one memory provider-id contract, then land one focused fix that covers runtime lookup, cold startup/plugin-owner resolution, diagnostics, and regression tests while keeping https://github.com/openclaw/openclaw/issues/90801 separate for the watcher/status follow-up.
