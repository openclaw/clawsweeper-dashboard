The issue reports that `memorySearch.provider: "google"` is shown by status but fails during `memory status --index` and `--deep` because embedding initialization cannot resolve that provider id.

## Best Possible Solution

Keep this issue open and land one maintainer-approved provider-id contract fix that covers runtime lookup, cold startup/capability resolution, docs, diagnostics or migration, and regression tests.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers choose the contract. A resolver-level `authProviderId` fallback is cleaner than call-site scans, but any accepted solution must also cover manifest/startup/runtime/docs/tests or explicitly keep `gemini` canonical with migration or diagnostics.
