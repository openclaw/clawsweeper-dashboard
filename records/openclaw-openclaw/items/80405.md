## Summary

Keep open. Current main still routes forced Telegram MP4 sends through video-specific handling, while this PR applies the focused runtime/test/docs fix, has sufficient live proof, and has no blocking review findings; normal CI and maintainer merge handling should decide it.

## What This Changes

The PR routes Telegram video media through `sendDocument` when `forceDocument` is set and updates the related tests, docs, CLI/tool descriptions, prompt snapshots, and generated Swift protocol output.

## Best Possible Solution

Land this focused Telegram override fix once normal merge gates pass, and leave automatic fallback or exit-code behavior to a separate follow-up if it remains reproducible after the routing fix.

## Reproduction Assessment

Yes for the routing bug: current main source shows `forceDocument: true` video media still reaches `probeVideoDimensions` and the `sendVideo` branch. I did not live-send from current main in this read-only review.

## Solution Assessment

Yes. Guarding video-note selection, video probing, and `sendVideo` behind `!forceDocument` is the narrow maintainable fix for the explicit override; fallback-on-error is a separate behavior decision because ambiguous network failures can risk duplicate sends.
