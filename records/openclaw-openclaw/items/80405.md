## Summary

Keep open for normal review and merge handling. The PR fixes a still-present Telegram routing bug on current main, has focused coverage and live proof, and I found no blocking correctness or security findings.

## What This Changes

The PR routes Telegram video media through `sendDocument` when `forceDocument` is set, skips video-specific probing and typed video sends in that mode, and updates CLI/tool/docs snapshots.

## Best Possible Solution

Land this focused explicit-override fix after normal review and merge gates; keep automatic fallback and exit-code behavior as separate follow-up work if still reproducible.

## Reproduction Assessment

Yes for the routing bug: current main source inspection shows a `forceDocument: true` video still selects the video-specific path and probes dimensions. I did not run a live current-main Telegram send in this read-only review.

## Solution Assessment

Yes. Guarding video-note selection, video probing, and `sendVideo` behind `!forceDocument` is the narrow maintainable fix for the explicit override, while broader fallback behavior should stay separate.
