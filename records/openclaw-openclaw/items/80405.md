## Summary

This PR is a focused, plausible fix for a real Telegram media-routing bug that is still present on current main, and the updated PR body now supplies live redacted Telegram terminal proof. I found no blocking correctness or security findings in the proposed routing change.

## What This Changes

The PR routes Telegram video media through `sendDocument` when `forceDocument` is set, skips video probing/typed video sends in that mode, and updates CLI/tool/docs text plus snapshots.

## Best Possible Solution

Land this focused override fix once normal CI and maintainer merge gates pass, while leaving automatic fallback and exit-code behavior to separate tracked work such as https://github.com/openclaw/openclaw/issues/80389.

## Reproduction Assessment

Yes. Current main source inspection shows `video/mp4` still reaches `sendVideo` and `probeVideoDimensions` even when `forceDocument` is true, while the existing forceDocument test table covers only images and GIFs.

## Solution Assessment

Yes. Guarding video-note selection, video probing, and `sendVideo` with `!forceDocument` is the narrow maintainable fix for the explicit override; broader fallback and exit-code changes should remain separate.
