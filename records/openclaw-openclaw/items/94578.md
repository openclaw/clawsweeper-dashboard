## Summary

Keep this PR open: current main and the latest release still have the fixed 512-byte recent-session header probe, while this focused patch addresses the linked session-resume bug with no blocking code findings found in review.

## What This Changes

The PR replaces recent-session validation's fixed 512-byte JSONL header read with a bounded first-line reader and adds regression tests for long and oversized session headers.

## Best Possible Solution

Merge this bounded header-line scanner with regression coverage once required CI is green or confirmed unrelated, then let the linked issue close on merge.

## Reproduction Assessment

Yes for source-level reproduction. Current main can load a full valid JSONL transcript while findMostRecentSession rejects the same file after parsing only a 512-byte header prefix before continueRecent resumes.

## Solution Assessment

Yes. The bounded first-line reader is the narrowest maintainable fix; loading whole transcripts during recent-session scans would be broader and slower, and leaving the fixed 512-byte probe keeps the bug.
