Keep open: this PR is still the active focused fix candidate for a source-reproducible session resume bug that remains on current main and in the latest release, and I found no blocking patch defect in the bounded scanner or regression tests.

## What This Changes

The PR replaces the fixed 512-byte recent-session header probe with a bounded first-line reader and adds regression coverage for long and oversized JSONL session headers.

## Best Possible Solution

Land this bounded recent-session header scanner with the regression tests after maintainer review and CI rerun or unrelated-failure confirmation, then let the linked issue close on merge.

## Reproduction Assessment

Yes. Source inspection shows a header longer than 512 bytes can be loadable by `loadEntriesFromFile` but rejected by `findMostRecentSession` before `continueRecent` can resume it; I did not run tests because this review is read-only.

## Solution Assessment

Yes. The bounded first-line scan is the narrowest maintainable fix I found; loading whole transcripts during recent-session directory scans would be broader and slower, while keeping a cap avoids unbounded directory-scan reads.
