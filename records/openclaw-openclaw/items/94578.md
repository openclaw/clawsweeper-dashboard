## What This Changes

The PR replaces recent-session validation's fixed 512-byte JSONL header read with a bounded 64 KiB first-line reader and adds regression tests for long and oversized session headers.

## Best Possible Solution

Land this bounded first-line scanner with regression tests after required checks are green, then let the linked issue close through the merged PR.

## Reproduction Assessment

Yes. Source inspection shows a header longer than 512 bytes can be loadable by loadEntriesFromFile but rejected by findMostRecentSession before continueRecent decides whether to resume.

## Solution Assessment

Yes. The bounded first-line reader is the narrow owner-boundary fix; loading full transcripts during recent-session directory scans would be broader and slower.
