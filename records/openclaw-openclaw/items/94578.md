## Summary

Keep this PR open for maintainer review: the patch appears to be the focused fix for the linked session-resume bug, and current main plus v2026.6.8 still have the fixed 512-byte recent-session header scan. The only remaining blocker I found is validation state, because the live check rollup is unstable with unrelated-looking CI failures.

## What This Changes

This PR replaces recent-session validation's fixed 512-byte read with a bounded first-line JSONL header reader and adds regression tests for long and oversized session headers.

## Best Possible Solution

Land this bounded first-line scanner with regression coverage after maintainers resolve or accept the unrelated-looking CI failures, then let the linked issue close on merge.

## Reproduction Assessment

Yes. Source inspection shows a header longer than 512 bytes can be loadable by loadEntriesFromFile but rejected by findMostRecentSession before continueRecent decides whether to resume; I did not execute the repro because this review is read-only.

## Solution Assessment

Yes. The bounded first-line scan is the narrowest maintainable fix; loading whole transcripts during directory scans would be broader, and a sibling checkpoint reader already uses a 64 KiB session-header bound.
