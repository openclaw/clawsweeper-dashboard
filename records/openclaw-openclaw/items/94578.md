## What This Changes

The PR replaces recent-session validation's fixed 512-byte JSONL header read with a bounded first-line reader and adds regression tests for long and oversized session headers.

## Best Possible Solution

Land this bounded first-line scanner with regression coverage after the failed CI checks are resolved or confirmed unrelated, then let the linked issue close on merge.

## Reproduction Assessment

Yes. Source inspection shows a header longer than 512 bytes can be loadable by `loadEntriesFromFile()` but rejected by `findMostRecentSession()` before `continueRecent()` decides whether to resume.

## Solution Assessment

Yes. The bounded first-line scan is the narrowest maintainable fix; loading full transcripts during directory scans would be broader and slower, and a sibling checkpoint reader already uses a 64 KiB header bound.
