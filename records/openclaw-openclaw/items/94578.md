## What This Changes

The PR replaces recent-session validation's fixed 512-byte JSONL header read with a bounded 64 KiB first-line reader and adds regression coverage for long and oversized session headers.

## Best Possible Solution

Land this bounded first-line scanner with the regression tests after the queued checks pass, then let the linked issue close through the merged PR.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: a long first JSONL header can load through loadEntriesFromFile but fail the 512-byte isValidSessionFile scan before continueRecent resumes it.

## Solution Assessment

Yes. The bounded first-line scan is the narrowest maintainable fix; loading full transcripts during directory scans would be broader and slower, and a sibling checkpoint reader already uses a 64 KiB header bound.
