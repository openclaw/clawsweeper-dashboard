## What This Changes

The branch replaces the recent-session scanner's fixed 512-byte header read with a bounded first-line reader and adds regression tests for long and oversized JSONL session headers.

## Best Possible Solution

Land this bounded first-line reader with the regression coverage after required checks complete, keeping recent-session discovery aligned with the full JSONL loader without scanning whole transcripts.

## Reproduction Assessment

Yes by source inspection. Current main can parse the whole JSONL transcript with loadEntriesFromFile(), but recent-session discovery parses only the first 512 bytes before continueRecent() decides whether to resume.

## Solution Assessment

Yes. The fix belongs in the recent-session header validation helper; loading every transcript during directory scans or changing the session header shape would be broader than necessary.
