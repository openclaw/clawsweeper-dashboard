I found no line-level patch findings, but the failed CI checks still need rerun or resolution before merge.

## What This Changes

The PR replaces recent-session validation's fixed 512-byte JSONL header read with a bounded first-line reader and adds regression tests for long and oversized session headers.

## Best Possible Solution

Land this bounded first-line scanner with the regression coverage once required checks are green or confirmed unrelated, then let the linked issue close on merge.

## Reproduction Assessment

Yes. Source inspection shows a header longer than 512 bytes can be readable by `loadEntriesFromFile()` but rejected by `findMostRecentSession()` before `continueRecent()` decides whether to resume.

## Solution Assessment

Yes. The bounded first-line reader is the narrowest maintainable fix; loading full transcripts during directory scans would be broader and slower.
