## What This Changes

## Best Possible Solution

Land this bounded recent-session header scanner with regression coverage after maintainer review and CI rerun or unrelated-failure confirmation, then let the linked issue close on merge.

## Reproduction Assessment

## Solution Assessment

Yes. The bounded first-line scan is the narrowest maintainable fix I found; loading whole transcripts during recent-session directory scans would be broader and slower, while the 64 KiB cap matches an existing bounded-header pattern.
