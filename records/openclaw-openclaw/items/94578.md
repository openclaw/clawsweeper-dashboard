## What This Changes

The PR replaces the fixed 512-byte recent-session header probe with a bounded first-line reader and adds session manager regression tests for long and oversized JSONL headers.

## Best Possible Solution

Land this bounded header-line scanner with regression coverage after maintainer review and failed CI checks are resolved or confirmed unrelated, then let the linked issue close on merge.

## Reproduction Assessment

## Solution Assessment

Yes. The bounded first-line scan is the narrowest maintainable fix I found; loading whole transcripts during recent-session directory scans would be broader and slower, and a sibling checkpoint reader already uses a 64 KiB header bound.
