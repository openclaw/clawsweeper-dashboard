## Label Justifications

- `P1`: The PR addresses a real long-running gateway resource leak that can OOM active daemons under memory-heavy workloads.
- `merge-risk: 🚨 compatibility`: The patch adds default-on gateway-wide eviction and new config keys, changing upgrade behavior for existing gateway users.
- `merge-risk: 🚨 auth-provider`: Idle eviction closes memory managers and their embedding provider runtime state, so provider initialization/auth-adjacent behavior can be affected after idle periods.
- `merge-risk: 🚨 availability`: A bad eviction policy or missed in-flight guard could close managers during active gateway memory work or make memory search unavailable until re-open.

## Summary

Keep open for maintainer review: current main still lacks daemon-lifetime idle eviction for cached memory index managers, and the rebased PR now appears focused on that leak. I found no blocking line-level defect, but the default-on gateway-wide sweep is compatibility, availability, and provider-runtime sensitive enough to need explicit owner approval before merge.

## What This Changes

The PR adds busy-refcounted idle eviction for cached memory index managers, schedules it as a gateway-lifetime sidecar, exposes gateway-wide config/docs/help, and adds memory cache/idle-GC regression tests.

## Best Possible Solution

Land the bounded cache-level idle eviction if maintainers approve the default-on gateway-wide policy; otherwise adjust the policy to opt-in or defer to the narrower watcher-default work at https://github.com/openclaw/openclaw/issues/71335.
