- **[P2] Honor per-agent eviction settings:** `src/gateway/server-startup-post-attach.ts:189-191`
  - body: `resolveSyncConfig` now accepts `idleEvictMs`/`idleEvictScanMs` from agent overrides, but the gateway sidecar reads only `cfg.agents.defaults.memorySearch.sync` and then sweeps every cached manager with that one policy. Either make the sweep honor the resolved setting per manager, or make these knobs explicitly defaults-only and remove the per-agent resolution.
