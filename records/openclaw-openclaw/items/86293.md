## Summary

Keep open: the PR addresses a real long-running gateway memory-manager lifetime problem and has strong after-fix soak evidence, but the new public config contract is not safe enough yet because accepted agent-level settings are ignored and docs/help are not aligned.

## What This Changes

The PR adds idle tracking and busy-aware periodic eviction for memory-core `MemoryIndexManager` caches, wires it into gateway lifetime sidecars, and adds config/schema/test coverage for the new sweep.

## Best Possible Solution

Keep the idle eviction design, but make the config contract exact by either honoring each manager's resolved idle policy or exposing the knobs as default/global-only, then add matching schema help and memory config docs.

## Reproduction Assessment

Yes. Current main source shows a process-wide memory index manager cache with teardown only through explicit close paths, and the PR supplies production terminal soak output demonstrating the after-fix fd/RSS behavior; I did not run a local multi-hour soak in this read-only review.

## Solution Assessment

Not yet. The busy-aware idle sweep is a reasonable fix shape, but the implementation should not accept per-agent config that the gateway sidecar ignores, and the new public knobs need docs/help alignment.

Full review comments:

- **[P2] Honor the accepted agent-level eviction config:** `src/gateway/server-startup-post-attach.ts:189-191`
  - body: `MemorySearchSchema` is shared by both `agents.defaults.memorySearch` and `agents.list[].memorySearch`, so this PR accepts `sync.idleEvictMs`/`idleEvictScanMs` on individual agents and `resolveMemorySearchConfig` carries those values into each manager. The gateway sidecar only reads `cfg.agents.defaults.memorySearch.sync`, applying one default policy to every cached manager; an agent override such as `idleEvictMs: 0` would be silently ignored and that manager could still be closed on the default cadence. Please either make these knobs truly global/default-only in the contract or have idle eviction honor each manager's resolved policy.
  - confidence: 0.86

Add help/docs entries for `idleEvictMs`, `idleEvictScanMs`, defaults, scope, and the `0` disable behavior so config help and public docs match the schema/type/label updates.
