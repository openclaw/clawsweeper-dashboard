## Summary

Keep open for maintainer review: the current head addresses the prior docs/help finding and I found no blocking patch defect, but the default-on gateway idle eviction policy and optional memory runtime API extension remain compatibility-sensitive choices.

## What This Changes

The PR adds idle tracking, in-flight protection, and a gateway-lifetime idle eviction sidecar for cached memory index managers, plus gateway-wide config fields, docs, and regression tests.

## Best Possible Solution

Land the bounded idle-eviction design if maintainers accept the default-on gateway-wide policy, while keeping the separate `sync.watch` default discussion in https://github.com/openclaw/openclaw/issues/71335.

## Reproduction Assessment

Yes. Current main source shows cached memory managers only close through explicit teardown paths, and the PR supplies production soak output demonstrating the reported watcher and fd accumulation before the patch; I did not run a multi-hour daemon soak locally.

## Solution Assessment

Yes for code shape, pending maintainer policy acceptance. A periodic idle sweep with busy/revalidation guards is a maintainable fix for daemon-lifetime resource retention, but default-on gateway-wide behavior is a compatibility decision.
