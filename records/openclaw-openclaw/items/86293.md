## Label Justifications

- `P1`: The reported behavior is a long-running gateway fd/native-handle leak that can OOM production daemons within hours.
- `merge-risk: 🚨 compatibility`: Merging changes the default runtime contract for cached memory managers and adds an optional memory runtime hook/config surface.
- `merge-risk: 🚨 availability`: Merging introduces a default-on background sweeper that closes cached managers in live gateways, so a policy or race bug could affect daemon availability or memory-search continuity.

## Summary

Keep open for maintainer review: the patch is still useful and current main does not have runtime idle eviction for cached memory index managers. I found no blocking line-level code findings, but the default-on gateway-wide sidecar and new memory runtime hook remain compatibility and availability decisions.

## What This Changes

This PR adds idle tracking, in-flight protection, a gateway-lifetime idle-eviction sidecar, gateway-wide config/docs, and regression tests for cached memory index managers.

## Best Possible Solution

Land the idle-eviction design after owner approval of the default-on gateway-wide policy and a clean or explained CI rerun, while leaving the broader watch-default change tracked separately at https://github.com/openclaw/openclaw/issues/71335.
