## Summary

This PR should stay open: it targets a plausible process-lifecycle bug, but the proposed direct `runExec` await removes the current wall-clock settlement and depends on Node child-process close behavior after timeout, so it needs a bounded cleanup implementation plus real behavior proof before merge.

## What This Changes

The PR removes the outer 3-second `Promise.race` around the Tailscale binary `--version` probe in `src/infra/tailscale.ts` and relies only on `runExec(..., { timeoutMs: 3000 })`.

## Best Possible Solution

Use an owned bounded process probe for the binary check that preserves predictable setup latency, reliably terminates or reaps the child, and includes focused regression coverage or redacted runtime proof.

## Reproduction Assessment

No high-confidence live reproduction is present. Source inspection shows the lifecycle race and the PR's changed timeout dependency, but no real or fixture Tailscale-compatible hanging binary proof was supplied or run in this read-only review.

## Solution Assessment

No, not as written. Removing the outer race addresses the orphaned-process window but also removes the existing wall-clock bound; a small owned child-process probe that times out, kills, and reaps is safer.
