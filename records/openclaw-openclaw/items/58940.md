## Summary

Keep this PR open but not merge-ready: the current code still has the duplicate Tailscale binary timeout race, but the proposed direct `runExec` await can trade a leaked child for a setup hang when the child ignores `SIGTERM`, and the branch still lacks after-fix real behavior proof.

## What This Changes

The PR removes the outer 3-second `Promise.race` wrapper from the Tailscale binary `--version` probe and relies on `runExec`'s `execFile` timeout.

## Best Possible Solution

Use an owned bounded process probe for the binary check that preserves a wall-clock timeout, reliably terminates or reaps the child, and carries focused regression coverage or redacted runtime proof.

## Reproduction Assessment

No high-confidence live reproduction is present. Source inspection plus an inline Node `execFile` probe show the relevant timeout/close behavior, but no real Tailscale-compatible hanging binary artifact was supplied or run.

## Solution Assessment

No, not as written. The direct `runExec` await removes the outer wall-clock settlement and relies on `execFile` close after `SIGTERM`; a bounded helper that owns child cleanup is safer.
