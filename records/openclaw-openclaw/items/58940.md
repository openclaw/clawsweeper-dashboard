## What This Changes

The PR removes the manual `Promise.race` timeout from Tailscale binary detection so `runExec`/`execFile` owns the 3-second `--version` probe timeout.

## Best Possible Solution

Land the one-line cleanup after the contributor adds redacted real behavior proof showing a hanging version probe times out and leaves no child process behind.

## Reproduction Assessment

Yes at source level: current main races a manual 3-second timer against `runExec`, while `runExec` maps `timeoutMs` to Node `execFile` timeout. A live reproduction would use a hanging Tailscale-compatible binary and inspect child cleanup after timeout; I did not run that live path here.

## Solution Assessment

Yes for the code path. Removing the outer race is the narrowest maintainable fix because it preserves the command, timeout, and catch/false behavior while letting `runExec`/`execFile` own child termination.
