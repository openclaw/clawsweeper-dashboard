## Summary

Keep open: current main still has the redundant Tailscale binary-probe race and the patch is a narrow correct simplification, but this external PR lacks after-fix real behavior proof, so it should not merge yet.

## What This Changes

The PR changes `findTailscaleBinary` so `checkBinary` relies on `runExec(path, ["--version"], { timeoutMs: 3000 })` instead of wrapping it in a separate `Promise.race` timeout.

## Best Possible Solution

Land the one-line Tailscale probe cleanup after the contributor adds redacted real behavior proof, preferably terminal output showing a hanging version probe timing out and no child process left behind.

## Reproduction Assessment

Yes at source level, but not as live process proof. Current main races a separate 3-second timer against `runExec` while `runExec` already maps `timeoutMs` to `execFile` timeout; a live reproduction would use a hanging Tailscale-compatible binary and inspect cleanup after timeout.

## Solution Assessment

Yes for the code path. Removing the outer race is the narrowest maintainable fix because it preserves the command, timeout, and false-on-error behavior while letting `runExec`/`execFile` own child termination.
