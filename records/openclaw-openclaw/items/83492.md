## Summary

Keep open: the reported TTS bug is real, but this patch suppresses final TTS for the whole explicit command turn, which also covers model-forwarded skill slash commands that should continue to behave like normal assistant replies.

## What This Changes

The PR adds a dispatch-level final TTS bypass for explicit command turns and a regression test for native command replies such as `/active-memory status`.

## Best Possible Solution

Mark or otherwise identify replies produced by command handlers and skip final TTS only for those command-output payloads, while preserving normal configured TTS for model-forwarded skill command final answers.

## Reproduction Assessment

Yes. Source inspection shows current main routes final replies through the final TTS path when TTS is enabled, and the linked user report gives the `/active-memory status` command path; I did not run a live provider in this read-only review.

## Solution Assessment

No. The branch fixes the reported native command seam, but the turn-wide predicate is broader than the command-output contract; a payload-level or command-handler signal is the safer maintainable fix.
