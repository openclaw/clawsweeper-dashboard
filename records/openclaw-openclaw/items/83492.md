## Summary

Keep open: the linked bug is valid, but this PR is not merge-ready because the turn-level guard also suppresses TTS for model-backed skill command answers, and the external proof is mock-only.

## What This Changes

The branch adds a turn-level TTS bypass for explicit command final replies in `dispatch-from-config.ts` and a dispatch test for native command replies.

## Best Possible Solution

Mark or otherwise identify replies produced by command handlers and skip final TTS only for those payloads, while preserving normal TTS for model-forwarded skill commands and proving the result in a real setup.

## Reproduction Assessment

Yes, by source inspection: `/active-memory status` returns plain text, current main applies final TTS when auto mode resolves enabled, and the linked issue reports that exact path. I did not run a live TTS provider or channel bot in this read-only review.

## Solution Assessment

No. The branch fixes the reporter's native command seam, but the turn-wide predicate is broader than the command-output contract; a command-reply payload marker or equivalent narrow signal is safer.
