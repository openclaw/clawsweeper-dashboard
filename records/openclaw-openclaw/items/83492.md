## Summary

Keep open: this PR targets a real command-reply TTS regression that is still present on current main and v2026.6.9, but the branch is not merge-ready because the new guard skips TTS for the whole explicit command turn instead of only command-handler output.

## What This Changes

The PR adds a final-reply dispatch guard that skips TTS for explicit command turns and adds a regression test for `/active-memory status` with final TTS enabled.

## Best Possible Solution

Land a command-output-level TTS exclusion for terminal command replies, rebase it onto current main, and preserve configured final TTS for command continuations and normal replies.

## Solution Assessment
