## Summary

Keep open: the reported command-output TTS bug is valid, but this patch uses a turn-wide bypass that would also disable configured final TTS for model-forwarded skill slash commands.

## What This Changes

The branch changes final reply dispatch to bypass TTS whenever the inbound turn is an explicit command and adds a regression test for a native `/active-memory status` command reply.

## Best Possible Solution

Mark command-handler or direct command-output payloads and skip TTS only for that marker, while preserving configured final TTS for model-forwarded skill command final answers.

## Reproduction Assessment

Yes. Source inspection shows current main routes final replies through the final TTS path when TTS is enabled, and the linked user report gives the `/active-memory status` command path; I did not run a live provider in this read-only review.

## Solution Assessment

No. The branch fixes the reported native command seam, but the turn-wide predicate is broader than the command-output contract; a payload-level command-output signal is the safer maintainable fix.
