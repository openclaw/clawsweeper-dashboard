## Summary

This PR addresses a real command-output TTS bug, but the current patch is too broad because it disables final TTS for the entire explicit command turn, including commands that fall through into normal agent replies.

## What This Changes

The PR changes final reply dispatch so explicit command turns bypass final-reply TTS and adds a focused regression test for a native command reply.

## Best Possible Solution

Mark or otherwise identify command-handler-produced reply payloads and skip final TTS only for those command-output replies, while preserving configured TTS for explicit commands that continue as normal agent prompts.

## Reproduction Assessment

Yes. The linked issue gives a concrete /active-memory status path with TTS always enabled, and current main source routes final replies through maybeApplyTtsToReplyPayload; I did not run tests in this read-only review.

## Solution Assessment

No. The command-output bug should be fixed, but a turn-wide isExplicitCommandTurn guard is broader than the behavior being repaired.
