## Best Possible Solution

Carry a command-output marker through handler-produced command replies and skip TTS only for those payloads, with regression tests for command output and command-to-normal-prompt continuation.

Yes. The linked issue gives a concrete `/active-memory status` path with TTS always enabled, and current main source routes final replies through `maybeApplyTtsToReplyPayload`; I did not run tests in this read-only review.
