Keep this PR open: it contains useful context-overflow recovery work, but the truncation summary now promotes raw tool-result `details` references into model-visible content before the existing details redaction path runs.

## What This Changes

The PR hardens embedded-runner context overflow recovery, tool-result truncation, incomplete-turn messaging, and compaction summary safeguards with focused tests.

## Best Possible Solution

Keep the recovery hardening, but repair the truncation summary so any reference promoted from `details` is redacted or tightly allowlisted before becoming model-visible content.

## Reproduction Assessment

No high-confidence live failing reproduction was run. Source inspection shows current main misses `context_length_exceeded` and only considers assistant overflow text for `stopReason === "error"`, while the PR adds coverage for the `toolUse` overflow path.

## Solution Assessment

No, not as written. The recovery approach is narrow, but the truncation summary should not copy raw runtime/UI `details` references into model-visible content without redaction or a safe allowlist.
