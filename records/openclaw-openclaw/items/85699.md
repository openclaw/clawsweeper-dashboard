## Best Possible Solution

Resolve the wiki CLI failure by loading or registering the active memory plugin through the generic plugin capability/public-artifact contract before wiki status/import reads artifacts, and split unrelated skills onboarding work into its own PR.

Yes, at source level with medium confidence: wiki status/import call `listActiveMemoryPublicArtifacts`, and current main returns an empty list if the memory public-artifacts capability has not registered. I did not run the CLI because this review was read-only.

## Solution Assessment

No: the proposed fix solves the symptom by importing bundled memory-core internals from core, while the current SDK and tests point to capability registration or generic active-plugin loading as the maintainable path.
