## Summary

Keep open; the linked regression is plausible, but this PR needs revision because the proposed fallback bypasses the plugin capability contract and can read memory artifacts outside the approved activation path.

## What This Changes

The branch adds a memory-wiki bridge fallback that deep-imports memory-core public artifacts, and also includes a separate skills wizard empty-state note.

## Best Possible Solution

Fix the bridge/status path by making the active memory slot register or resolve its public-artifacts capability through the generic plugin runtime contract, while preserving plugin disablement and external memory-plugin behavior.

## Reproduction Assessment

Yes from source inspection, but not from an executed current-main scenario: the wiki bridge/status path ultimately depends on `listActiveMemoryPublicArtifacts`, which returns empty when no memory capability has registered. The linked issue supplies a concrete config and observed zero-artifact output, but this review did not run the live CLI flow.

## Solution Assessment

No; the proposed direct import is not the best way to solve it because it bypasses the memory plugin capability contract. A safer fix should ensure the active memory slot is loaded or resolved through the existing public-artifacts seam before the wiki bridge reads artifacts.
