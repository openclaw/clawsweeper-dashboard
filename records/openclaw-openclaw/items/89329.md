## Summary

## What This Changes

The PR adds a defensive projection for `/export-session` tool descriptors and a regression test that skips unreadable or cyclic schemas while preserving a healthy sibling descriptor.

## Best Possible Solution

Land the narrow export-boundary guard after maintainer draft and review gates complete, while keeping unreadable tool-name resolver handling on https://github.com/openclaw/openclaw/pull/89292.

## Reproduction Assessment

Yes. Source inspection shows current main directly reads tool descriptor fields and then JSON-stringifies the export payload, so a throwing `parameters` getter or cyclic schema can abort `/export-session`; I did not run tests in this read-only review.

## Solution Assessment

Yes. Guarding and JSON-projecting tool descriptors at the HTML export boundary is the narrowest maintainable fix for this crash, and it avoids changing the runtime tool contract or the prompt-bundle resolver ownership.
