## Summary

Keep open: current main and v2026.6.6 still directly read and serialize exported tool descriptors, while this maintainer-authored draft PR adds the missing export-boundary guard and has no blocking code findings from this review.

## What This Changes

The PR adds safe `/export-session` tool descriptor projection so unreadable or non-JSON-serializable schemas are skipped while serializable sibling tool metadata remains in the exported HTML data.

## Best Possible Solution

Land the narrow export-boundary guard once the maintainer marks the draft ready and ordinary review gates complete, while leaving resolver-level unreadable-name handling to the related open PR.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.6 directly read tool descriptor fields and then JSON-stringify the export payload, so a throwing `parameters` getter or cyclic schema can abort `/export-session`; I did not run tests in this read-only review.

## Solution Assessment

Yes. Guarding and JSON-projecting tool descriptors at the HTML export boundary is the narrowest maintainable fix for this crash, and it avoids changing runtime tool contracts or duplicating the separate unreadable-name resolver work.
