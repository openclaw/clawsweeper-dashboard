## What This Changes

The PR adds a defensive projection for `/export-session` tool descriptors so unreadable or non-JSON-serializable schemas are skipped while serializable sibling tools remain in exported HTML session data.

## Best Possible Solution

Land the narrow export-boundary guard after maintainer draft and review gates complete, while leaving resolver-level unreadable-name handling to https://github.com/openclaw/openclaw/pull/89292.

## Reproduction Assessment

Yes. Source inspection shows current main directly reads tool descriptor fields and JSON-stringifies the export payload, so a throwing `parameters` getter or cyclic schema can abort `/export-session`; I did not run tests in this read-only review.

## Solution Assessment

Yes. Guarding and JSON-projecting tool descriptors at the HTML export boundary is the narrowest maintainable fix for this crash and avoids changing the runtime tool contract or resolver ownership.
