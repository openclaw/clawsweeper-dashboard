## Summary

## What This Changes

The issue reports cron `agentTurn` sending unsupported OpenAI disabled reasoning despite `payload.thinking: minimal`.

## Best Possible Solution

Keep the shipped/current provider request-shaping behavior and close this issue as implemented, with any future recurrence reported as a fresh release-specific bug with outbound request evidence.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source/tag comparison and reporter logs: the old release path could send OpenAI Responses `reasoning.effort: none`, while current main no longer maps the explicit cron thinking request to disabled reasoning.

## Solution Assessment

Is this the best way to solve the issue? Yes: fixing provider request shaping and preserving cron payload thinking is the narrow maintainable solution, not a cron-only workaround.
