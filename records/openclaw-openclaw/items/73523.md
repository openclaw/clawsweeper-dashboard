## Summary

Keep this PR open: the original passive Chrome MCP spawn bug is source-reproducible and the cache-only probe direction is right, but the latest branch still introduces blocking regressions for explicit cold-cache start and documented Brave/Edge/non-macOS existing-session profiles.

## What This Changes

The PR adds Chrome MCP cache/file/port/HTTP health probes, rewires browser status/profile/tab/start availability paths, and updates browser tests plus CHANGELOG entries.

## Best Possible Solution

Merge a focused browser-plugin fix that keeps passive status/profile/tab probes non-spawning while preserving explicit start recovery and documented Chromium-based existing-session profiles.

## Reproduction Assessment

Yes, source-reproducible. On current main, passive Chrome MCP status/profile/tab paths can reach getSession(), and the MCP SDK stdio transport starts a subprocess when the cache is cold.

## Solution Assessment

No. Cache-only passive probing is the right direction, but this patch also blocks deliberate cold-cache attach and supported browser/OS cases; the safer fix separates passive non-spawning probes from explicit start recovery.

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Allow explicit start to warm a cold MCP cache:** `extensions/browser/src/browser/server-context.availability.ts:333-337`
  - body: When the probe returns HIGH from live file/port/HTTP signals after a gateway restart, `cacheAttached` is false and this branch throws before a deliberate start can create the MCP client. That contradicts the PR contract and leaves explicit `start` unable to recover the session.
  - confidence: 0.91
- **[P2] Recognize documented Brave and Edge port owners:** `extensions/browser/src/browser/chrome-mcp.ts:833-842`
  - body: The owner classifier only trusts Chrome/Chromium process names. Documented Brave and Edge existing-session profiles use different listener process names, so a healthy remote-debugging port is classified as `other` and refused instead of attaching.
  - confidence: 0.88
- **[P2] Keep non-macOS userDataDir attach startable:** `extensions/browser/src/browser/chrome-mcp.ts:1020-1021`
  - body: On non-darwin hosts this probe always returns `unknown`; with a valid `DevToolsActivePort`, that becomes MEDIUM and `decideStartGate()` refuses explicit start. Existing-session `userDataDir` support is documented as host-local, not macOS-only.
  - confidence: 0.84
- **[P3] Drop unrelated changelog carry-over:** `CHANGELOG.md:74-80`
  - body: The branch carries many unrelated release bullets that are already present on current main, so merging as-is would duplicate or reorder other fixes in the release notes. Keep only the browser entries for this PR.
  - confidence: 0.76
