The PR rotates the oldest per-config `.clobbered.*` snapshots when the 32-file cap is reached, adds focused config tests, extends recovery filesystem deps, and documents latest-32 retention.

## Best Possible Solution

Land the shared-helper rotation fix with focused recovery coverage and docs once maintainers accept newest-over-oldest retention for capped forensic snapshots.

Yes. Source inspection on current main shows the helper returns `null` at the clobber-snapshot cap, and the existing tests only assert the cap rather than preserving the newest repaired payload.

## Solution Assessment

Yes, with a policy caveat. Rotating inside the shared locked helper is the narrow maintainable fix for the observable failure while preserving the 32-file bound, but maintainers should explicitly accept discarding oldest artifacts.
