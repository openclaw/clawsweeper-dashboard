## Summary

Keep open: the PR addresses a real current-main logging bug, but it is not mergeable as-is and still introduces an info-level log of free-form announce labels, which undercuts the privacy/noise goal.

## What This Changes

The PR routes subagent announce delivery output and announce retry/error logging through an `agents/announce` subsystem logger, logging announce body text at debug with metadata at info, and adds targeted delivery test coverage.

## Best Possible Solution

Rebase or replace the PR on current `main`, keep announce info logs to stable metadata only, route/suppress empty announce replies through the same subsystem path, add a changelog entry, and land after targeted agent logging tests plus the changed gate.

## Reproduction Assessment

Yes. The issue has a high-confidence source reproduction: a `subagent_announce` agent call reaches `deliverAgentCommandResult()`, which on current `main` formats payload text and calls `runtime.log(output)`, and gateway console capture records that as info-level file log content.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the right approach, but the PR should remove free-form labels from info logs, cover empty announce payloads, rebase onto current `main`, and include the required changelog entry.
