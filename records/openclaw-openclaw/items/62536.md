## Summary

Keep this PR open: the announce logging fix is still useful, but the current branch should not merge as-is because it still writes free-form spawn labels to default info logs and its older base leaves an announce empty-reply path on `runtime.log`. The existing after-fix log sample is enough real behavior proof, so the remaining work is a narrow repair/rebase rather than a proof request.

## What This Changes

The PR adds an `agents/announce` subsystem logger for subagent announce delivery, routes announce payload bodies to debug with metadata at info, moves announce warn/error logs to that logger, and adds focused delivery coverage.

## Best Possible Solution

Land a rebased fix that keeps default announce logs to stable metadata only, moves full announce body/task text to debug, preserves non-announce delivery logging, and keeps announce runs away from `runtime.log` fallback paths.

## Reproduction Assessment

Yes. Current main source shows `sourceTool: "subagent_announce"` reaches `deliverAgentCommandResult()` and non-nested payload text still goes through `runtime.log(output)`, and the PR discussion includes copied after-fix gateway logs from a real local run.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the right narrow direction, but info logs must avoid free-form labels and the rebased fix should ensure announce runs never hit a `runtime.log` fallback.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep free-form labels out of info logs:** `src/agents/subagent-announce.ts:600`
  - body: `params.label` comes from the optional `sessions_spawn` label string, so writing it into the default `announce delivery` info line can persist user/model-authored task context or newline/control-character log pollution. Keep info logs to stable metadata and move any label/task text to sanitized debug output.
  - confidence: 0.88
- **[P3] Move announce detection before empty replies:** `src/agents/command/delivery.ts:299`
  - body: The announce discriminator is computed after the empty-payload branch on this PR head, so an announce run with no payload still writes `No reply from agent.` through `runtime.log` at info level. Compute the announce flag before that branch or preserve current-main silent empty-payload behavior when rebasing.
  - confidence: 0.72
