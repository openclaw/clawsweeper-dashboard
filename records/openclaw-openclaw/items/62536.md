## Summary

Keep this PR open: the underlying announce log-pollution bug is still source-reproducible on current main and the PR has useful proof, but the current patch is not merge-ready because it still writes free-form labels to default info logs and misses the empty announce reply path.

## What This Changes

This PR adds an `agents/announce` subsystem logger for subagent announce delivery, moves announce payload bodies to debug with metadata at info, and updates announcement warn/error logging plus delivery tests.

## Best Possible Solution

Land a rebased fix that keeps default announce logs to stable metadata only, moves free-form task/label/body text to sanitized debug output, handles empty announce replies through the same announce logger boundary, and preserves normal non-announce delivery logging.

## Reproduction Assessment

Yes. Current main source shows `sourceTool: "subagent_announce"` reaches delivery and non-nested payload text still goes through `runtime.log(output)`, while the PR discussion includes a real after-fix gateway log sample.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the narrow maintainable fix, but the PR should remove free-form label text from info logs and move announce detection ahead of the empty-payload branch.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep free-form labels out of info logs:** `src/agents/subagent-announce.ts:600`
  - body: `params.label` is an optional plain string from `sessions_spawn`, so writing it into the default `announce delivery` info line can persist user/model-authored task context or newline/control-character log pollution. Keep info logs to stable metadata and move any label/task text to sanitized debug output.
  - confidence: 0.88
- **[P3] Bypass runtime.log for empty announce replies:** `src/agents/command/delivery.ts:299`
  - body: The announce discriminator is computed after the empty-payload branch, so an announce run with no payload still writes `No reply from agent.` through `runtime.log` at info level. Compute the announce flag before this branch and handle empty announce replies through the announce logger boundary or silence them.
  - confidence: 0.72
