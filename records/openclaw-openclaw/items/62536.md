## Summary

Keep open. The PR addresses a real announce logging path that is still source-reproducible on current main, but the current head still writes an unconstrained spawn label into default info logs and is merge-conflicting with current main, so it needs a narrow repair or replacement before merge.

## What This Changes

The PR routes subagent announce delivery output and announce warning/error logs through an `agents/announce` subsystem logger and adds focused delivery test coverage.

## Best Possible Solution

Land a rebased or replacement fix that logs only stable announce metadata at info, keeps full bodies and task/label text debug-only, and preserves normal non-announce delivery logging.

## Reproduction Assessment

Yes, source-reproducible. Current main passes `sourceTool: "subagent_announce"` into the requester-agent delivery call, and `deliverAgentCommandResult()` still formats outbound payload text and calls `runtime.log(output)`, which console capture records as info-level file logs.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the right direction, but default info logs should not include raw `params.label`, and the branch needs to be rebased or reimplemented against current main.
