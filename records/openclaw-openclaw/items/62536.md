## Summary

Keep open. Current main still has the announce body log path, and this PR is useful source work, but the latest head should not merge as-is because it still puts free-form task labels in default info logs and is conflicting against current main.

## What This Changes

The PR routes subagent announce delivery output and announce warning/error logs through an `agents/announce` subsystem logger and adds focused delivery test coverage.

## Best Possible Solution

Land a rebased or replacement fix on current main that logs stable announce metadata at info, keeps announce bodies and task/label text debug-only, and preserves normal non-announce delivery logging.

## Reproduction Assessment

Yes, source-reproducible. A `subagent_announce` delivery path reaches `deliverAgentCommandResult()`, which still formats payload text and calls `runtime.log(output)` on current main, and console capture writes that path to info-level file logs.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the right fix direction, but the PR should remove free-form labels from info logs and be rebased or replaced on current main before merge.
