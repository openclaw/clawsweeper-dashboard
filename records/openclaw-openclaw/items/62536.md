## Summary

Keep this PR open as useful source work, but do not merge the current head. Current main already has a shipped gateway WebSocket log summarizer for part of the related issue, yet the announce delivery path still has a source-level `runtime.log(output)` route through console capture; the PR targets that remaining path but leaves a free-form label in default info logs and needs rebasing onto the current delivery flow.

## What This Changes

The PR adds an `agents/announce` subsystem logger, routes subagent announce delivery output to metadata-at-info and body-at-debug logging, updates announce retry/error logs, and adds delivery unit coverage.

## Best Possible Solution

Land a rebased fix or replacement PR that keeps announce info logs to stable metadata, moves full/free-form announce content to sanitized debug output, handles announce delivery through the current payload-plan flow, and preserves non-announce delivery behavior.

## Reproduction Assessment

Yes, source-level. Current main carries `sourceTool: "subagent_announce"` into delivery, gateway agent runs use `defaultRuntime`, and non-json delivery payload text still reaches `runtime.log(output)` before console capture writes info/file logs.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the right maintainable direction, but the PR should remove free-form labels from info logs, handle/rebase the empty-payload and current payload-plan delivery flow, and keep the shipped gateway event summarizer intact.
