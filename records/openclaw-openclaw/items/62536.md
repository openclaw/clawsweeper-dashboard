## Summary

This PR should stay open because the underlying announce-log leak is still source-reproducible on current main, but the submitted branch is stale and still has a privacy/log-noise regression in its info-level label logging.

## What This Changes

The PR routes subagent announce delivery output and announce retry/error logs through an `agents/announce` subsystem logger and adds focused delivery coverage.

## Best Possible Solution

Land a rebased announce-specific logging change on the current payload-plan delivery path, keeping info logs to stable metadata and moving full bodies plus free-form label/task text to sanitized debug logs.

## Reproduction Assessment

Yes, at source level. Direct announce calls carry `sourceTool: "subagent_announce"`, current delivery logging still sends formatted visible payload text to `runtime.log(output)`, and gateway console capture writes that path at info level.

## Solution Assessment

No, not as submitted. The subsystem logger boundary is the right direction, but the patch needs to target the current payload-plan delivery path and must keep free-form label/task text out of info logs.
