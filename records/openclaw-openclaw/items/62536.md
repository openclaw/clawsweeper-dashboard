## Summary

Keep open. This PR addresses a real announce log-pollution path, but it should not merge as-is because PR head still writes free-form spawn labels into default info logs and the branch needs a careful rebase onto the current delivery flow.

## What This Changes

The PR adds an `agents/announce` subsystem logger for subagent announce delivery, routes announce payload bodies away from `runtime.log` into debug logging with metadata at info, and updates announce-related warn/error logging plus delivery tests.

## Best Possible Solution

Land a rebased fix that detects `subagent_announce` in the current delivery payload-plan flow, keeps default announce logs to stable metadata only, moves body/task detail to sanitized debug output, and preserves normal non-announce logging.

## Reproduction Assessment

Yes. Current main source shows `sourceTool: "subagent_announce"` reaches delivery and non-nested payload text still goes through `runtime.log(output)`, while console capture forwards that output to info logs.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the narrow maintainable fix, but the PR should remove free-form label text from info logs and rebase onto current delivery code.

## Review Findings
