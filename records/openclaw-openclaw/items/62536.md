## Summary

Keep open: the logging bug is still source-reproducible on current main and this PR has the right narrow direction, but PR head still writes free-form labels to default info logs and misses the empty-announce path.

## What This Changes

The PR adds an `agents/announce` subsystem logger, routes subagent announce payload bodies to debug with metadata at info, updates announce warn/error logging, and adds delivery coverage.

## Best Possible Solution

Land a rebased fix that keeps announce-run info logs to stable metadata, sends full/free-form announce content only to debug with sanitization, covers empty announce replies, and preserves normal non-announce delivery logging.

## Reproduction Assessment

Yes, source-level. Current main carries `sourceTool: "subagent_announce"` into delivery, then logs non-json payload text through `runtime.log(output)`, and console capture forwards that output to info/file logs.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the right maintainable direction, but the PR should remove free-form labels from info logs, handle empty announce replies, and rebase onto current delivery code.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep free-form labels out of info logs:** `src/agents/subagent-announce.ts:600`
  - body: `params.label` is an optional plain string from `sessions_spawn`, so writing it into the default `announce delivery` info line can persist user/model-authored task context or newline/control-character log pollution. Keep info logs to stable metadata and move label/task text to sanitized debug output.
  - confidence: 0.88
- **[P3] Bypass runtime.log for empty announce replies:** `src/agents/command/delivery.ts:294`
  - body: The empty-payload branch runs before the announce discriminator is computed, so a zero-payload announce run still writes `No reply from agent.` through `runtime.log` at info level. Compute the announce flag before this branch or keep empty announce replies inside the announce logger boundary.
  - confidence: 0.68
