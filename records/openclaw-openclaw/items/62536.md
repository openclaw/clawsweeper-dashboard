## Summary

Keep this PR open: the linked logging problem is source-reproducible on current main and the subsystem-logger approach is a plausible narrow fix, but the PR head still has a concrete info-level logging privacy issue and is merge-conflicting with current main.

## What This Changes

The PR adds an `agents/announce` subsystem logger for subagent announce delivery, routes announce payload text to debug with metadata at info, moves announce warn/error logs off `defaultRuntime`, and adds delivery coverage.

## Best Possible Solution

Land a rebased fix that keeps default announce logs to stable metadata only, moves any free-form label/task/body text to sanitized debug output, and preserves normal non-announce delivery logging.

## Reproduction Assessment

Yes, source-reproducible. Current main passes `sourceTool: "subagent_announce"` into requester-agent delivery, and that delivery path still formats payload text and calls `runtime.log(output)`, which console capture forwards to info-level file logs.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the right direction, but the PR should remove raw `params.label` from info logs and move announce detection ahead of the empty-payload branch while rebasing onto current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep free-form labels out of announce info logs:** `src/agents/subagent-announce.ts:600`
  - body: `params.label` comes from `sessions_spawn` as an optional plain string, so writing it into the default `announce delivery` info log can persist user/model-authored task context or newline/control-character log pollution. Keep info logs to stable metadata and move any label/task text to sanitized debug output.
  - confidence: 0.88
- **[P3] Bypass runtime.log for empty announce replies:** `src/agents/command/delivery.ts:299`
  - body: The announce discriminator is computed after the empty-payload branch, so an announce run with no payload still writes through `runtime.log("No reply from agent.")` at info level without the `agents/announce` subsystem. Move announce detection before that branch and handle the empty announce case through the same metadata-only path or silence it.
  - confidence: 0.72
