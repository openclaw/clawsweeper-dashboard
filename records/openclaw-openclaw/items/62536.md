## Summary

Keep this PR open, but it needs changes before merge. The linked issue was closed based on the gateway WebSocket event summarizer, but current main still has a separate subagent announce path that reaches `deliverAgentCommandResult()` and writes formatted assistant output through `defaultRuntime.log()`, so the PR is still useful; PR head still leaks free-form labels into default info logs and is stale against current delivery code.

## What This Changes

The PR routes subagent announce delivery output through an `agents/announce` subsystem logger, logging metadata at info and full announce bodies at debug while adding focused delivery coverage.

## Best Possible Solution

Land a rebased logging fix that treats subagent announce delivery as a subsystem-owned path, keeps default info logs to stable metadata only, moves body and task/label text to sanitized debug output, and preserves non-announce delivery logging.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: subagent announce delivery calls the gateway `agent` method with `sourceTool: "subagent_announce"`, gateway runs it with `defaultRuntime`, and delivery still calls `runtime.log(output)` for formatted outbound payloads. I did not run a live gateway in this read-only review.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the narrow maintainable direction, but the PR should remove free-form label text from info logs and rebase the delivery change onto current main before merge.
