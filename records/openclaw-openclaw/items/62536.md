## Summary

Keep open. Current main still has the announce delivery payload-to-runtime.log path, and this PR has useful focused proof, but the PR head is conflicting with current main and still writes a free-form spawn label into default info logs.

## What This Changes

The PR adds an `agents/announce` subsystem logger for subagent announce delivery output, moves announce retry/error logging to that logger, and adds delivery unit coverage.

## Best Possible Solution

Land a rebased version that routes announce-run delivery through subsystem logging with metadata-only info entries, sanitized/debug-only free-form text, and regression tests preserving non-announce delivery behavior.

## Reproduction Assessment

Yes, source-level. Current main carries `sourceTool: "subagent_announce"` into delivery, gateway agent runs use `defaultRuntime`, and non-json delivery payload text still reaches `runtime.log(output)` before console capture writes info/file logs.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the right direction, but the PR should keep info logs to stable metadata and be rebased onto the current outbound payload-plan delivery code before merge.
