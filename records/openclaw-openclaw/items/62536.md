## Summary

Keep this PR open: it targets a real current-main logging bug, but the head is not merge-ready because it still logs free-form announce labels at default info level, leaves an empty-payload announce path on `runtime.log()`, lacks the required changelog entry, and is dirty against current `main`.

## What This Changes

The PR routes subagent announce delivery output and announce retry/error logging through an `agents/announce` subsystem logger, with metadata at info, body text at debug, and a targeted delivery unit test.

## Best Possible Solution

Land a rebased agent logging fix that keeps announce info logs to stable metadata only, moves full announce content to debug, preserves current-main empty-reply silence, adds regression coverage, and includes a changelog entry.

## Reproduction Assessment

Yes. A `subagent_announce` result reaches `deliverAgentCommandResult()`, which on current `main` formats the outbound payload and calls `runtime.log(output)`; gateway console capture records `console.log` as an info-level file log entry.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the right direction, but the PR should remove free-form labels from info logs, preserve the current empty-reply behavior, rebase onto current `main`, and add the required changelog entry.
