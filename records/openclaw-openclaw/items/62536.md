## Summary

Keep open: the PR addresses a real source-reproducible announce logging problem and has useful proof, but the current head still logs a free-form spawn label at default info level and is conflicting against main.

## What This Changes

The PR routes subagent announce delivery output and announce retry/error logs through an `agents/announce` subsystem logger and adds focused delivery test coverage.

## Best Possible Solution

Land a rebased replacement that logs only stable announce metadata at info, keeps full bodies and free-form task or label text debug-only and sanitized, and preserves normal non-announce delivery logging.

## Reproduction Assessment

Yes. Current main passes `sourceTool: "subagent_announce"` into the gateway agent call, runs that path with `defaultRuntime`, and still logs delivered payload text through `runtime.log(output)`, which console capture records as info-level file logs.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the right direction, but the PR head still writes `params.label` into default info logs and needs to be rebased or reimplemented against current main.

  - confidence: 0.88
