## Summary

Keep open: current main still has the announce payload log path this PR targets, but the PR head still needs changes before merge because it logs free-form labels at info level, leaves empty announce replies on `runtime.log`, lacks the required changelog entry, and is currently unmergeable against main.

## What This Changes

The PR routes subagent announce delivery output and announce retry/error logging through an `agents/announce` subsystem logger, logs announce bodies at debug with metadata at info, and adds targeted delivery unit coverage.

## Best Possible Solution

Land a rebased implementation that logs only stable announce metadata at info, keeps body/task text debug-only and sanitized, covers empty announce payloads, and records the user-visible log behavior change in the changelog.

## Reproduction Assessment

Yes, from source inspection. A `subagent_announce` result reaches `deliverAgentCommandResult()`, where current main formats outbound payload text and calls `runtime.log(output)`, and gateway console capture records `console.log` as info-level file log content.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the right fix direction, but the PR must remove free-form labels from info logs, handle empty announce payloads before `runtime.log`, add a changelog entry, and rebase or replace the dirty head.
