## Summary

Keep open: the reported leak still has a source-level path on current main, but this branch should not merge as-is because it keeps untrusted labels in info logs and targets the older delivery normalization path.

## What This Changes

The PR routes subagent announce output and announce warning/error logs through an `agents/announce` subsystem logger, with unit coverage for announce payload logging.

## Best Possible Solution

Land a rebased announce logging fix that keeps info logs to stable metadata only, moves announce bodies and free-form label/task text to debug or sanitized output, and preserves current outbound payload-plan semantics.

## Reproduction Assessment

Yes, source-level. Current main marks direct announce agent calls with `sourceTool: "subagent_announce"`, then non-json delivery output can still reach `runtime.log(output)` through gateway/defaultRuntime console capture.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the right direction, but the PR needs a current-main rebase and must remove free-form label text from default info logs.
