The branch adds an `agents/announce` subsystem logger for subagent announce delivery output, moves full announce bodies to debug, changes announce warning/error logging, and adds delivery tests.

## Best Possible Solution

Land a rebased delivery-layer fix that keeps announce info logs to stable metadata, moves body/task text to sanitized debug output, preserves non-announce delivery behavior, and covers empty announce replies.

## Reproduction Assessment

Yes, by source inspection. Current main passes `sourceTool: "subagent_announce"` through announce delivery into `deliverAgentCommandResult`, where visible non-json payload text still reaches `runtime.log(output)` and console capture writes it at info level.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the right narrow fix, but the PR should remove free-form label text from info logs and preserve current main's no-visible-payload handling when rebased.
