## Summary

Keep this PR open, but do not merge it as-is. Current main still has a source-level announce delivery path that can send non-json announce output through `runtime.log`, while the PR head still logs a free-form label at info level and is based on an older delivery flow.

## What This Changes

The PR routes subagent announce delivery output and announce warnings/errors through an `agents/announce` subsystem logger, with a unit test covering announce payload logging.

## Best Possible Solution

Land a rebased announce-specific logging fix that keeps info logs to stable metadata only, moves announce bodies and free-form task/label text to debug, and preserves current delivery payload-plan semantics.

## Reproduction Assessment

Yes, source-level. Current main marks direct announce agent calls with `sourceTool: "subagent_announce"`, then non-json delivery output can still reach `runtime.log(output)` through gateway/defaultRuntime console capture.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the right direction, but the PR needs a current-main rebase and must remove free-form label text from default info logs.
