## Summary

Keep open: the logging bug is real and the subsystem-logger direction is useful, but this branch needs changes before merge because it targets the older delivery flow and still allows caller-provided label text into default info logs.

## What This Changes

The PR routes subagent announce delivery output and announce retry/error logs through an `agents/announce` subsystem logger and adds focused delivery coverage.

## Best Possible Solution

Land a rebased announce-specific logging change on the current outbound payload-plan delivery path, keeping info logs to stable metadata and moving full bodies plus free-form task or label text to debug only.

## Reproduction Assessment

Yes, at source level. Direct announce calls carry `sourceTool: "subagent_announce"`, current delivery logging still sends formatted visible payload text to `runtime.log(output)`, and gateway console capture maps that to info-level file logging.

## Solution Assessment

No, not as submitted. The subsystem logger boundary is the right direction, but the patch must be rebased onto the current payload-plan delivery path and must not put free-form label/task text in info logs.
