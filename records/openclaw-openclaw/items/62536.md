## Summary

Keep open. Current main still has a source-visible announce payload path to `runtime.log(output)`, and this PR targets a real logging bug, but the branch is outdated against the payload-plan delivery flow and still writes free-form label text at info level.

## What This Changes

The PR routes subagent announce delivery output and announce warn/error messages through an `agents/announce` subsystem logger, with a focused delivery regression test.

## Best Possible Solution

Land a rebased announce-specific logging fix that keeps default info logs to stable ids, status, and counts only, moves body/task text to debug, and preserves current outbound payload-plan semantics.

## Reproduction Assessment

Yes, at source level. Direct announce calls mark `sourceTool: "subagent_announce"`, gateway agent execution uses `defaultRuntime`, and current delivery logging still sends visible formatted payload text to `runtime.log(output)`.

## Solution Assessment

No, not as submitted. The subsystem logger boundary is the right direction, but the patch must be rebased onto current payload-plan delivery and must remove free-form label/task text from info logs.
