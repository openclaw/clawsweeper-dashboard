## Summary

Keep this PR open: it addresses a real source-reproducible announce logging bug, and the linked issue being closed via gateway WS log summarization does not cover the remaining `runtime.log(output)` delivery path. The branch still needs a small repair before merge because it writes free-form spawn labels to default info logs.

## What This Changes

The PR routes subagent announce delivery output and related announce warning/error logs through an `agents/announce` subsystem logger with focused delivery test coverage.

## Best Possible Solution

Land a rebased or replacement fix that logs only stable announce metadata at info, keeps full bodies and task/label text debug-only or sanitized, and preserves normal non-announce delivery logging.

## Reproduction Assessment

Yes, source-reproducible. Current main passes `sourceTool: "subagent_announce"` into requester-agent delivery, and the delivery logger still formats payload text and calls `runtime.log(output)`, which console capture records as info-level file logs.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the right direction, but default info logs should not include raw `params.label`, and the empty-payload announce path should not fall back to `runtime.log`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep free-form labels out of announce info logs:** `src/agents/subagent-announce.ts:600`
  - body: `params.label` comes from `sessions_spawn` as an optional plain string, so writing it into the default `announce delivery` info log can persist user/model-authored task context or newline/control-character log pollution. Keep info logs to stable metadata and move any label/task text to sanitized debug output.
  - confidence: 0.88
- **[P3] Bypass runtime.log for empty announce replies:** `src/agents/command/delivery.ts:299`
  - body: The new announce discriminator is computed after the empty-payload branch, so an announce run with no payload still writes through `runtime.log("No reply from agent.")` at info level without the `agents/announce` subsystem. Move announce detection before that branch and handle the empty announce case through the same metadata-only path or silence it.
  - confidence: 0.72
