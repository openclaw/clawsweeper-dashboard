## Summary

Keep open: the current bug is still source-reproducible on main, and the PR’s subsystem-logger approach is useful but still has narrow blockers around info-level label logging, empty announce replies, rebase drift, and changelog coverage.

## What This Changes

The PR routes subagent announce delivery and announce retry/error logging through an `agents/announce` subsystem logger and adds targeted delivery test coverage.

## Best Possible Solution

Rebase or replace the PR on current main with announce info logs kept to stable metadata only, announce bodies and task text debug-only, empty announce replies covered, and a changelog entry added.

## Reproduction Assessment

Yes. The issue is source-reproducible: a `subagent_announce` provenance path reaches `deliverAgentCommandResult()`, which formats payload text and calls `runtime.log()`, and gateway console capture writes `console.log` output to root `info` file logs.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the right maintainable seam, but this PR should remove free-form labels from info logs, cover empty announce payloads, rebase onto current main, and add the required changelog entry.
