## Summary

Keep open: current main still lacks an announce-specific delivery logging boundary, and this PR has useful proof, but the current head logs a free-form spawn label at info level and needs a focused repair before merge.

## What This Changes

The PR routes subagent announce delivery output through an `agents/announce` subsystem logger, adds metadata/debug split coverage, and moves announce warning/error logs off `defaultRuntime`.

## Best Possible Solution

Land a rebased announce logging fix that keeps info logs to stable metadata, sends full announce content only to debug, preserves normal delivery logging, and keeps empty announce replies silent.

## Reproduction Assessment

Yes, source-level. Current main marks announce handoffs with `sourceTool: "subagent_announce"`, then the delivery layer logs non-json payload text through `runtime.log(output)`, and console capture forwards that output to info/file logs.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the right direction, but the PR should keep free-form label/task text out of info logs and rebase onto current delivery code.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep labels out of info logs:** `src/agents/subagent-announce.ts:600`
  - body: `params.label` comes from the optional `sessions_spawn` string field and is interpolated into the default `announce delivery` info log. That can persist user/model-authored task context or newline/control-character log pollution, undercutting the privacy/noise fix; keep info logs to stable metadata and move label/task text to debug only after sanitizing it.
  - confidence: 0.88
- **[P3] Bypass runtime.log for empty announce replies:** `src/agents/command/delivery.ts:294`
  - body: At PR head, the empty-payload branch runs before the announce discriminator is computed, so a zero-payload announce run still writes `No reply from agent.` through `runtime.log` at info level. Current main already keeps empty replies silent, so the rebase should preserve that behavior or handle empty announce replies inside the announce logger boundary.
  - confidence: 0.66
