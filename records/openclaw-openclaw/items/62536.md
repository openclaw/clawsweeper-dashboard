## Summary

Keep open: this is a real announce-log privacy/noise bug with sufficient after-fix runtime proof, but the PR branch now conflicts with current main and still logs caller-provided label text at info level.

## What This Changes

The PR routes subagent announce delivery output and announce retry/error messages through an `agents/announce` subsystem logger and adds a focused delivery test.

## Best Possible Solution

Land a rebased announce-only logging change on the current outbound payload-plan path, with info logs limited to stable metadata and full bodies or task labels only in sanitized debug logs.

## Reproduction Assessment

Yes, at source level. Current main still routes visible announce payloads through `runtime.log(output)`, and direct announce calls carry `sourceTool: "subagent_announce"`, which matches the reported gateway info-log leak path.

## Solution Assessment

No, not as submitted. The subsystem logger direction is right, but it must be rebased onto the current payload-plan delivery code and keep free-form label/task text out of info-level logs.
