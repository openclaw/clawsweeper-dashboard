## Summary

Keep open: current main still has a source-level announce body logging path through `defaultRuntime`, but this PR is not mergeable as-is because it targets an older delivery flow and leaves free-form label text in info logs.

## What This Changes

The branch routes subagent announce delivery output through an `agents/announce` subsystem logger, adds a delivery regression test, and moves announce warn/error logging away from `defaultRuntime`.

## Best Possible Solution

Land a rebased announce-specific logging fix that keeps info logs to stable ids, status, and counts only, moves announce body and task text to debug, and preserves current outbound payload-plan semantics.

## Reproduction Assessment

Yes, source-level. Direct announce calls mark `sourceTool: "subagent_announce"`, gateway agent execution uses `defaultRuntime`, and visible non-json, non-nested delivery output still reaches `runtime.log(output)` through console capture.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the right direction, but the PR must be rebased onto the current payload-plan path and must keep free-form label/task text out of info logs.
