## Summary

Keep open: the PR addresses a valid logging bug, but the branch is stale against current main’s payload-plan delivery path and still logs caller-provided label text at default info level.

## What This Changes

The PR routes subagent announce delivery output and announce warning/error logging through an `agents/announce` subsystem logger instead of `runtime.log`/`defaultRuntime`.

## Best Possible Solution

Land a rebased announce-specific logger change on the current payload-plan delivery path, keeping info logs to stable ids, status, and counts while moving body/task/label text to sanitized debug logs.

## Reproduction Assessment

Yes, at source level. Current main marks announce calls with `sourceTool: "subagent_announce"`, formats the visible outbound payload, and sends it to `runtime.log`, which console capture writes at info level.

## Solution Assessment

No, not as submitted. The subsystem logger boundary is the right direction, but the patch needs to target the current payload-plan delivery path and keep free-form label/task text out of info logs.
