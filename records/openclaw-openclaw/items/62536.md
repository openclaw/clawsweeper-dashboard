## Summary

Keep open: the PR addresses a real announce-log leak path, but the current head still has two concrete logging gaps and current main has not adopted the announce subsystem-logger boundary.

## What This Changes

The PR adds an `agents/announce` subsystem logger for subagent announce delivery, moves announce payload bodies to debug, updates announce warning/error logging, and adds delivery coverage.

## Best Possible Solution

Land a rebased fix that keeps default announce logs to stable metadata only, moves free-form task/label/body text to sanitized debug output, handles empty announce replies through the announce logger boundary, and preserves normal non-announce delivery logging.

## Reproduction Assessment

Yes. Current main source shows `sourceTool: "subagent_announce"` reaches delivery and non-nested payload text still goes through `runtime.log(output)`, while console capture forwards that output to info logs.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the narrow maintainable fix, but the PR should remove free-form label text from info logs and rebase onto current delivery code.
