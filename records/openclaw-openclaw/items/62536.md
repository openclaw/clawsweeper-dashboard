## Summary

Keep this PR open, but it still needs changes before merge. The logger split is the right direction for the announce log-pollution bug, but the current branch is stale against main's payload-plan delivery flow and still puts free-form label text into default info logs.

## What This Changes

The PR routes subagent announce delivery output through an `agents/announce` subsystem logger, moving full announce bodies to debug while keeping metadata at info.

## Best Possible Solution

Land a rebased announce-specific logging fix that keeps info logs to stable ids, status, and counts only, moves announce body and free-form task/label text out of default info logs, and preserves current outbound payload-plan semantics.

## Reproduction Assessment

Yes, source-level. Current main marks direct announce calls with `sourceTool: "subagent_announce"`, then visible non-json, non-nested delivery output can still reach `runtime.log(output)` through gateway `defaultRuntime` console capture.

## Solution Assessment

No, not as-is. The subsystem logger boundary is the right fix direction, but this branch needs a current-main rebase and must keep free-form label/task text out of default info logs.
