## Summary

Keep open: current main still has the announce log-noise path and this PR is directionally useful, but the latest patch is not merge-ready because it still writes free-form announce labels to default info logs and lacks the required user-facing changelog entry.

## What This Changes

The PR adds an `agents/announce` subsystem logger, routes subagent announce payload bodies to debug with metadata at info, moves announce warning/error logs off `defaultRuntime`, and adds delivery logging coverage.

## Best Possible Solution

Land a revised PR or replacement branch that keeps default info logs to stable announce metadata only, sends full bodies and any free-form task or label text to debug after sanitization, preserves normal non-announce delivery behavior, and documents the user-visible log change in the Unreleased changelog.

## Reproduction Assessment

Yes. The linked report gives concrete gateway-log reproduction steps, and the current source shows announce runs carrying `sourceTool: "subagent_announce"` into `deliverAgentCommandResult()`, where payload text is still sent through `runtime.log()` and console capture persists it at info level.

## Solution Assessment

No, not as written. The subsystem logger boundary is the right fix direction, but info-level announce logging should avoid all free-form label/task text, and the user-visible `openclaw logs` behavior change needs a changelog entry before merge.
