## Summary

Keep open: current main and the latest release still use UTC-only reset/delete archive suffixes, and the open linked PR is the live implementation candidate that needs maintainer review before this issue can close.

## What This Changes

Request archive/reset session filenames to include a local timezone date when a user timezone is configured, while preserving UTC behavior by default.

## Best Possible Solution

Review and land or reject the linked local-date-prefix PR after maintainers settle the persisted archive filename contract and upgrade-proof the affected archive readers.

## Reproduction Assessment

Yes at source level: current main formats reset/delete archives with UTC ISO file stamps and does not pass user timezone config into the archive writer. I did not run the reporter's Windows/Asia-Shanghai reset flow live.

## Solution Assessment

Unclear as a product decision: a local-date prefix while preserving the UTC suffix appears safer than replacing the suffix, but maintainers still need to approve the persisted filename contract and linked PR coverage.
