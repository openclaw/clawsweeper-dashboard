## Summary

## What This Changes

The issue requests local-date-aware session reset/archive filenames, preferably by using the configured user timezone or adding a local-date prefix while preserving UTC compatibility.

## Best Possible Solution

Review the linked implementation candidate and land a compatibility-preserving local-date prefix design only after archive readers and real reset/delete proof cover both old and new filename shapes.

## Reproduction Assessment

Yes at source level: current main formats reset/delete archive names with UTC ISO file stamps and does not pass user timezone config into the archive writer. I did not run the reporter's Windows/Asia-Shanghai reset flow in a live setup.

## Solution Assessment

Unclear: adding a local-date prefix while keeping the UTC suffix appears safer than replacing the suffix, but the persisted filename contract and sibling archive consumers need maintainer approval and proof before this is the best fix.
