## Summary

Keep open: current main and the latest release still generate session archive filenames with UTC-only archive timestamps, and the remaining work is already represented by an open closing PR. The filename format is a persisted session-state contract, so this should stay open for maintainer review of the linked PR rather than being auto-closed.

## What This Changes

The issue requests that reset/deleted session archive filenames include a local-date component derived from the configured timezone instead of relying only on the UTC timestamp.

## Best Possible Solution

Land a compatibility-preserving local-date prefix through the open candidate PR if maintainers accept the filename contract change, while keeping the UTC suffix parseable for existing cleanup and usage accounting paths.

## Reproduction Assessment

Yes from source inspection, but not from a live Windows/Asia-Shanghai run here: current main formats archive suffixes with a UTC file stamp and does not thread configured timezone data into the reset/delete archive path.

## Solution Assessment

Unclear until maintainer review: the open PR's local-date prefix while preserving the UTC suffix looks like the safer compatibility shape, but the repository still needs an explicit product decision on changing the persisted filename convention.
