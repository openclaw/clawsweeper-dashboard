## Summary

Keep this open: current main mitigates doctor’s orphan comparison with realpath-based comparison, but the resolver still returns and persists realpath-backed `sessionFile` values, so the portability part of the report remains and an open linked PR is the active fix path.

## What This Changes

The issue reports that symlinked OpenClaw state directories cause persisted session transcript paths to use real filesystem paths, breaking portability and previously causing doctor orphan false positives.

## Best Possible Solution

Land a narrow resolver fix that returns caller-visible logical session paths while retaining realpath containment checks and regression coverage for legacy realpath entries plus doctor orphan detection.

## Reproduction Assessment

Yes. Source inspection shows the current resolver can still return a path rooted at the real sessions directory and `resolveAndPersistSessionFile` persists that value into `sessions.json`; I did not run a live symlinked-home repro in this read-only review.

## Solution Assessment

Yes, the active PR direction is the right layer: preserve logical returned paths while keeping realpath containment, then prove legacy realpath entries and doctor orphan detection still work.
