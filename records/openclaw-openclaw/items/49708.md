## What This Changes

The issue asks OpenClaw to preserve logical session transcript paths for symlinked OpenClaw state directories and avoid false doctor orphan reports.

## Best Possible Solution

Adopt or land the open linked fix after confirming legacy realpath normalization, or close that PR and keep this issue as the canonical bug for a narrow replacement.

## Reproduction Assessment

Yes, at source level: on current main, generated transcript paths go through a resolver that realpaths the sessions directory and the result is persisted to session metadata. I did not run a live filesystem repro in this read-only review.

## Solution Assessment

Yes, the right fix belongs in session path resolution plus doctor legacy comparison handling. Current main only partially mitigates doctor orphan comparison and does not preserve logical persisted paths.
