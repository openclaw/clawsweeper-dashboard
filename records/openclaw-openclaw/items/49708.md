## Summary

Keep open: current main and v2026.6.6 mitigate the doctor orphan comparison, but the session path resolver still returns realpath-backed transcript paths and persistence writes that value into `sessions.json`; the active linked PR remains the closing path for the remaining portability fix.

## What This Changes

The issue asks OpenClaw to avoid storing symlink-resolved `sessionFile` paths and to stop `openclaw doctor` from reporting live transcripts as orphans when the state directory is symlinked.

## Best Possible Solution

Land a narrow fix that preserves caller-visible session paths while keeping realpath containment checks and legacy realpath remapping, then close this issue once that behavior is on main.

## Reproduction Assessment

Yes, at source level for the remaining behavior: with a symlinked sessions directory, the resolver anchors returned paths to `realBase` and the persistence helper stores that value. The specific doctor orphan false-positive is mitigated by current realpath comparison, so I did not mark it reproduced live.

## Solution Assessment

Yes: preserving caller-visible paths while retaining realpath containment checks and legacy realpath remapping is the narrow maintainable fix. A doctor-only mitigation is already present but does not resolve the non-portable stored metadata.
