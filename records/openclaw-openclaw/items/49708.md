## Summary

Keep this issue open. Current main still persists symlinked session transcript paths through the real sessions directory; doctor now canonicalizes comparisons, which mitigates false orphan reports while files still resolve, but the portability half of the report remains unresolved and an open closing PR is already paired with this issue.

## What This Changes

The issue asks OpenClaw to preserve logical symlinked state paths for session transcripts and avoid doctor orphan false positives from realpath-backed `sessionFile` values.

## Best Possible Solution

Review, rebase, and land or replace the linked PR with a patch that preserves logical session paths while retaining realpath containment checks and regression coverage.

## Reproduction Assessment

Yes, from source inspection rather than a live run. A symlinked sessions directory flows through `resolvePathWithinSessionsDir`, which still returns `realBase`, and `resolveAndPersistSessionFile` persists that returned value into `sessions.json`.

## Solution Assessment

Yes. The narrow maintainable direction is to keep realpath-based containment checks while returning and persisting caller-visible logical paths, with doctor's canonical comparison retained as defense in depth.
