## Summary

Keep this issue open. Current main still resolves and persists new session transcript paths through the real sessions directory, so the portability half of the report remains unresolved; doctor has a canonical comparison mitigation for orphan detection, but the linked open PR #49796 is still the implementation candidate and uses closing syntax for this issue.

## What This Changes

The issue asks for session transcript paths under symlinked OpenClaw state directories to remain logical and portable, while preventing doctor orphan checks from misclassifying referenced transcripts.

## Best Possible Solution

Land or supersede PR #49796 with a fix that keeps realpath-based containment checks for safety, but returns and persists caller-visible logical session paths, normalizes legacy realpath-backed entries, and covers symlinked state directories plus doctor orphan detection in regression tests.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: with a symlinked sessionsDir, the resolver uses realBase from realpath and returns realBase plus the transcript filename, then resolveAndPersistSessionFile writes that path into sessions.json.

## Solution Assessment

Yes, the linked PR’s direction is the right maintainable shape: preserve realpath containment for safety while storing logical paths and keeping doctor’s canonical comparison as defense in depth. Current main is not sufficient because it only mitigates the orphan warning side.
