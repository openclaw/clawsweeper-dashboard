## Summary

Keep this issue open. Current main mitigates the doctor orphan comparison by canonicalizing paths, but new session paths still resolve through the real sessions directory before being persisted, so the portability half of the report remains unresolved; open PR #49796 is the existing closing implementation candidate.

## What This Changes

The issue requests that symlinked OpenClaw state directories preserve logical session transcript paths and avoid false doctor orphan reports.

## Best Possible Solution

Land or supersede PR #49796 with a fix that keeps realpath containment checks, persists caller-visible logical session paths, normalizes legacy realpath-backed entries, and covers symlinked state directories plus doctor orphan detection.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: a symlinked sessionsDir is realpath-resolved in resolvePathWithinSessionsDir, and resolveAndPersistSessionFile writes that returned real path into sessions.json.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix: preserve realpath-based safety checks while storing logical paths and keeping doctor’s canonical comparison as defense in depth; current main is only a partial mitigation.
