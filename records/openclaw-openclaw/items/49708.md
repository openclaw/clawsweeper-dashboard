## Summary

Keep this issue open: current main still persists new `sessionFile` values through a realpath-backed sessions directory, so the doctor orphan comparison is mitigated but the portability bug remains. The open closing PR is the right implementation path rather than a cleanup close.

## What This Changes

The issue reports that symlinked OpenClaw home/session paths are persisted as real paths, breaking portability and causing false-positive doctor orphan checks.

## Best Possible Solution

Review, rebase, and land or replace the linked PR with a patch that preserves logical `sessionFile` paths while retaining realpath containment checks and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a symlinked sessions directory flows through `realBase` in `resolvePathWithinSessionsDir`, and `resolveAndPersistSessionFile` persists that returned value into `sessions.json`.

## Solution Assessment

Yes, with the linked implementation direction. Returning caller-visible logical paths while keeping realpath containment checks is the narrow maintainable fix; current main only mitigates doctor comparison and does not fix persisted path portability.
