## What This Changes

The issue requests logical or relative `sessionFile` storage for symlinked OpenClaw homes and doctor orphan checks that do not flag referenced transcripts as orphaned.

## Best Possible Solution

Land or replace the linked PR with a current-main patch that keeps realpath containment checks, persists caller-visible logical session paths, normalizes legacy realpath entries, and proves the symlinked upgrade path.
