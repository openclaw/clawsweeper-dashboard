## Summary

Keep this issue open: current main still persists new `sessionFile` values through the realpath-backed sessions directory, so doctor orphan comparison is mitigated but the reported portability bug remains. The open closing PR is still the right implementation path rather than a duplicate cleanup close.

## What This Changes

The issue asks OpenClaw to preserve logical symlinked session transcript paths and avoid false-positive `openclaw doctor` orphan reports.

## Best Possible Solution

Review, rebase, and land or replace the linked PR with a patch that preserves logical `sessionFile` paths while retaining realpath containment checks and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a symlinked sessions directory flows through `realBase` in `resolvePathWithinSessionsDir`, then `resolveAndPersistSessionFile` writes that returned path into `sessions.json`.

## Solution Assessment

Yes, with the linked implementation direction. Returning caller-visible logical paths while keeping realpath containment checks is the narrow maintainable fix; current main only mitigates doctor comparison and does not fix persisted path portability.
