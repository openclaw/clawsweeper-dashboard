## Summary

Keep this issue open: current main still resolves newly persisted session transcript paths through the realpath-backed sessions directory, while the open closing PR remains the focused implementation path for the remaining portability bug.

## What This Changes

The issue requests that symlinked OpenClaw session paths remain logical/portable and stop causing false `openclaw doctor` orphan transcript reports.

## Best Possible Solution

Land or replace the linked PR with a patch that preserves caller-visible logical `sessionFile` paths, keeps realpath containment checks, normalizes legacy real paths, and adds regression coverage for session persistence plus doctor orphan detection.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a symlinked sessions directory flows through `realBase` in `resolvePathWithinSessionsDir`, and `resolveAndPersistSessionFile` persists that returned value into `sessions.json`.

## Solution Assessment

Yes, with the linked implementation direction. Returning logical paths while retaining realpath containment is the narrow maintainable fix; current main only mitigates doctor comparison and does not fix persisted path portability.
