## Summary

Keep this issue open: current main still persists session transcript paths through the realpath-backed sessions directory, while doctor only canonicalizes comparisons to reduce false orphan warnings. The linked open PR remains the right implementation path for the unresolved portability bug.

## What This Changes

This issue asks OpenClaw to avoid storing symlink-resolved session transcript paths and keep doctor orphan detection correct when the OpenClaw home/state directory is symlinked.

## Best Possible Solution

Review, rebase, and land or replace the linked PR with a patch that preserves logical `sessionFile` paths while retaining realpath containment checks and regression coverage.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a symlinked sessions directory flows through `realBase` in `resolvePathWithinSessionsDir`, and `resolveAndPersistSessionFile` persists that value into `sessions.json`. I did not run the live symlink setup in this read-only review.

## Solution Assessment

Yes, the best direction is the linked PR's narrow approach: keep realpath-based containment checks, return caller-visible logical paths for persistence, and keep doctor's canonical comparison as defense in depth.
