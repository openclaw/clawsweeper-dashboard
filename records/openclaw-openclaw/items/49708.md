## Label Justifications

- `P2`: This is a source-reproducible session metadata and doctor false-positive bug with a limited but real blast radius for symlinked state directories.
- `impact:session-state`: The report concerns persisted `sessionFile` metadata and doctor’s interpretation of active transcript session state.

## Summary

Keep open: current main only mitigates doctor’s orphan comparison path, while the session resolver still returns and persists realpath-backed `sessionFile` values. The remaining fix is already paired with an open closing PR, so this issue should stay open until that PR lands or is abandoned.

## What This Changes

The issue asks OpenClaw to preserve logical symlinked session paths, or otherwise compare session transcript paths canonically, so symlinked state directories do not produce false doctor orphan warnings.

## Best Possible Solution

Review and land https://github.com/BingqingLyu/openclaw/pull/994 if its compatibility handling and proof are acceptable; otherwise close that PR and keep this issue as the canonical narrow bug.

## Reproduction Assessment

Yes, at source level: with a symlinked OpenClaw state directory, `resolvePathWithinSessionsDir` realpaths the sessions base and returns that namespace, and `resolveAndPersistSessionFile` persists it. I did not run a live filesystem repro in this read-only review.

## Solution Assessment

Yes, the best fix is to return logical caller-visible session paths while retaining realpath containment checks, plus doctor canonicalization for legacy realpath entries. The open linked PR follows that shape and is the active implementation path.
