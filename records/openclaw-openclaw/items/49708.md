## What This Changes

The issue reports that symlinked OpenClaw home/session paths are persisted as real paths, making doctor misclassify valid transcripts as orphaned.

## Best Possible Solution

Review and land https://github.com/BingqingLyu/openclaw/pull/994 if its compatibility handling and proof are acceptable; otherwise close that PR and keep this issue as the canonical narrow bug.

## Reproduction Assessment

Yes, at source level: with a symlinked sessions directory, the current resolver anchors the returned transcript path to `realBase`, and the persistence helper stores that returned value. I did not run a live filesystem repro in this read-only review.

## Solution Assessment

Yes, the maintainable fix is to preserve caller-visible session paths while retaining realpath containment checks, with regression coverage for legacy realpath-backed entries and doctor orphan detection.
