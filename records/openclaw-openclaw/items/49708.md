## Summary

Keep open: current main and the latest release mitigate the doctor orphan comparison, but the session resolver still returns and persists realpath-backed `sessionFile` values under a symlinked state directory. An open linked PR, https://github.com/BingqingLyu/openclaw/pull/994, targets the remaining resolver and regression-test work, so this issue should stay open until that candidate is merged or closed.

## What This Changes

The issue requests preserving symlink-visible session transcript paths and preventing doctor from misclassifying referenced transcripts as orphans when the OpenClaw state directory is symlinked.

## Best Possible Solution

Review and land a narrow resolver fix that preserves caller-visible session paths while retaining realpath containment checks and the existing doctor canonical comparison coverage.

## Reproduction Assessment

Yes, source-level: with a symlinked sessions directory, current main resolves and persists `sessionFile` through `realBase`; the original doctor orphan comparison is already covered by a current symlink regression test. I did not run a live symlinked-home CLI flow in this read-only review.

## Solution Assessment
