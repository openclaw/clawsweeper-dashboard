## Summary

Keep open: the related bug is credible, but this patch is not mergeable because it references private/unimported memory state and also changes the failed-register rollback path so failed plugin supplements would remain active.

## What This Changes

The PR adds a changelog entry and changes plugin loader memory-state restoration to merge previous and current corpus/prompt supplements during non-activating and failed register paths.

## Best Possible Solution

Land a narrow loader fix with regression coverage that preserves service-start memory supplements across non-activating lazy reloads while keeping failed-register rollback exact.

## Reproduction Assessment

Yes. Source inspection shows the PR introduces an unresolved `memoryPluginState` reference, and the failed-register leak follows directly from applying the merge in the catch rollback against existing test coverage.

## Solution Assessment

No. The loader is the right area, but the fix should use exported snapshot/list accessors and limit preservation to the non-activating restore path instead of changing failed-register rollback semantics.
