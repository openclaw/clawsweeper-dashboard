## Summary

Keep open: the PR targets a real plugin-loader memory supplement bug, but the current diff applies the preservation merge to the failed-register rollback path too, which would let failed plugin memory supplements remain active.

## What This Changes

The PR changes `src/plugins/loader.ts` to merge prior and current memory corpus/prompt supplements during loader restore and adds a `CHANGELOG.md` entry for #77039.

## Best Possible Solution

Keep the preservation merge for successful non-activating lazy re-registers, keep failed-register rollback as an exact snapshot restore, and cover both paths with focused loader regression tests.

## Reproduction Assessment

Yes. The PR defect is source-reproducible from the diff plus the existing failing-register test that expects memory prompt/corpus state to be cleared after `register()` throws.

## Solution Assessment

No. The requested fix direction is right for the non-activating success path, but applying the same merge in the catch path is not the narrowest maintainable solution because failed registrations must be rolled back exactly.
