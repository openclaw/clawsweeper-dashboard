## Summary

Keep open. Current main still carries the separate subagent announce message as raw inter-session prompt text into the requester agent path, and the linked fix PR is open but not merged.

## What This Changes

The issue asks OpenClaw to keep subagent announce internal runtime-context payloads out of Control UI/WebChat display and session JSONL for separate inter-session completion messages.

## Best Possible Solution

Review and land the focused transcript-prompt plumbing in https://github.com/openclaw/openclaw/pull/79094, or close that PR and replace it with an equivalent backend fix that preserves hidden runtime context while persisting only clean transcript text.

## Reproduction Assessment

Yes. The current-main source path is clear: direct and queued subagent announce delivery pass raw trigger text into the Gateway agent call, and the embedded runner is not given the clean `transcriptBody`; a follow-up comment also reports reproduction on v2026.5.7.

## Solution Assessment

Yes. The narrowest maintainable fix is to carry the already-computed clean transcript body through the existing embedded-runner `transcriptPrompt` seam, leaving UI/history filters as defense in depth rather than the primary fix.
