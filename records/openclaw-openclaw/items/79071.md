## Summary

Keep open. Current main still matches the reported backend leak boundary, and a matching open implementation PR explicitly closes this issue, so the issue should stay open until that PR is merged or closed.

## What This Changes

The issue reports that inter-session subagent announce handoffs can still expose internal runtime-context blocks or wrapper text in Control UI and persisted session history after an earlier partial fix.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/79094 first, then open a narrower follow-up only if Control UI still needs a structured operator-facing subagent result renderer.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: direct and queued subagent announce delivery still route the trigger message through the inter-session Gateway agent path, while the embedded attempt call does not pass the sanitized transcript prompt.

## Solution Assessment

Yes. The narrow maintainable fix is backend-side transcript/runtime-context separation in the announce or embedded attempt path, with UI/history stripping only as defense in depth; the linked PR is the active candidate for that fix.
