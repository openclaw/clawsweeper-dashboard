## Summary

Keep open. Current `main` still leaves the backend subagent announce path able to pass the raw inter-session trigger into the embedded runner, and the issue is already paired with the open fixing PR at https://github.com/openclaw/openclaw/pull/79094.

## What This Changes

The issue reports that separate inter-session subagent announce messages can expose internal runtime-context blocks in Control UI/WebChat and persist them to session history after an earlier partial fix.

## Best Possible Solution

Land or replace the focused backend fix in https://github.com/openclaw/openclaw/pull/79094, then handle any remaining sanitized-wrapper readability as a separate Control UI rendering follow-up.

## Reproduction Assessment

Yes. Source inspection on current `main` shows subagent announce delivery passes raw `triggerMessage` into the Gateway agent path while the embedded runner is not given the already-computed clean `transcriptBody`.

## Solution Assessment

Yes. The narrowest maintainable fix is to pass the existing clean transcript body into the embedded runner's existing `transcriptPrompt` seam, with UI/history stripping kept as defense in depth rather than the primary fix.
