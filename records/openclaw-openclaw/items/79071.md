## Summary

Keep open. Current main still has the backend subagent announce handoff sending the raw internal-context trigger as a normal inter-session agent message, while the newer Control UI/history stripping is only downstream defense in depth.

## What This Changes

The issue reports a regression where subagent completion announce internal runtime-context blocks can still reach Control UI/webchat and JSONL history as normal inter-session user messages.

## Best Possible Solution

Move the fix to the backend announce or agent persistence boundary so internal completion payloads are delivered as internal event/custom context or sanitized transcript data before any JSONL append, keeping UI/history filters as defense in depth.

## Reproduction Assessment

Yes, from source inspection on current main: direct and queued subagent announce paths pass the raw internal-context trigger into the Gateway agent call, and the session persistence guard does not strip or suppress it by default. I did not run the live Control UI flow, so this is source-reproducible rather than directly reproduced.

## Solution Assessment

Yes. The narrowest maintainable fix is backend-side in the subagent announce or persistence boundary, not another UI-only filter and not a new configuration option.
