## Summary

Keep open: the force-pushed patch now covers the main pre-DNS and resolved-DNS paths, but the SSRF hardening is still incomplete because pinned dispatcher overrides can still route an otherwise public guarded fetch to the cloud metadata IP under private-network opt-in.

## What This Changes

The PR changes the SSRF classifier and pinned-DNS guard to keep cloud metadata hostnames and IPs blocked when private-network access is enabled, adds focused regression tests, and adds a changelog entry.

## Best Possible Solution

Revise the patch so cloud metadata hostnames and metadata IPs are rejected before every private-network skip path, including `pinnedHostname` dispatcher overrides, then keep the classifier, pinned DNS, dispatcher, and guarded-fetch regression coverage aligned.

## Reproduction Assessment

Yes. A focused reproduction can call `fetchWithSsrFGuard` for a public hostname with a public mocked DNS result, `policy: { allowPrivateNetwork: true }`, and a `dispatcherPolicy.pinnedHostname.addresses` override of `169.254.169.254`; the current PR does not reject that override before fetch dispatch.

## Solution Assessment

No. The current direction is close, but the implementation is not yet the best safe fix because the always-blocked metadata rule is applied to hostnames and DNS answers, not to dispatcher override addresses that can replace the pinned target.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Reject metadata IPs in pinned dispatcher overrides:** `src/infra/net/ssrf.ts:367-373`
  - body: The new always-blocked helper only runs on DNS lookup results. In the strict guarded-fetch path, `createPinnedDispatcher` can still receive `dispatcherPolicy.pinnedHostname.addresses`, and `resolvePinnedDispatcherLookup` skips override validation whenever `allowPrivateNetwork` is true, so an override to `169.254.169.254` remains reachable. Reuse this always-blocked check for override records before the private-network skip.
  - confidence: 0.88
