## Summary

Keep open; the PR fixes the current-main SSRF bypass directionally, but the latest diff still breaks documented exact `allowedHostnames` exceptions and needs docs alignment before merge.

## What This Changes

The PR hardens SSRF guarded fetch, pinned-DNS, and dispatcher override paths so cloud metadata and `.internal` destinations remain blocked when private-network access is enabled.

## Best Possible Solution

Land the SSRF hardening after splitting always-blocked metadata/IP policy from exact non-metadata host exceptions and updating the user-facing SSRF docs/tests.

## Reproduction Assessment

Yes. Current main source shows the private-network skip bypasses pre-DNS hostname classification, and the PR body supplies before/after live output for the production resolver path.

## Solution Assessment

No. The hardening direction is right, but the implementation should preserve documented exact non-metadata `allowedHostnames` exceptions while keeping metadata hosts and IMDS IPs closed.
