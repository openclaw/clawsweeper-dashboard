## Summary

Keep open: the SSRF hardening addresses a real bypass and has sufficient live proof, but this PR still breaks documented exact `allowedHostnames` exceptions and currently leaves `check-lint` red with an unused helper.

## What This Changes

The PR tightens SSRF hostname/IP policy so cloud metadata and `.internal` destinations stay blocked under private-network opt-in, adds pinned-DNS and dispatcher regression tests, and updates the changelog.

## Best Possible Solution

Land the hardening after separating always-blocked metadata and `.internal` policy from documented exact non-metadata host exceptions, cleaning the lint failure, and aligning SSRF docs with the tightened behavior.

## Reproduction Assessment

Yes. Current main source shows the private-network skip bypasses pre-DNS hostname classification, and the PR body supplies before/after live output for the metadata and `.internal` cases.

## Solution Assessment

No. The hardening direction is right, but the implementation should preserve documented exact `allowedHostnames` exceptions for non-metadata hosts while keeping metadata, IMDS, and `.internal` protections closed.
