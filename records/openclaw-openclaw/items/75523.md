## Summary

Keep open: the PR is useful SSRF hardening with sufficient real behavior proof, but it still breaks the documented exact-host exception contract by running the general blocked-hostname classifier before `allowedHostnames`, and the public SSRF docs are not updated for the behavior tightening.

## What This Changes

The PR changes shared SSRF host/IP classification and pinned-dispatcher validation so metadata and `.internal` hosts stay blocked under private-network opt-in, adds SSRF regression tests, and adds a changelog entry.

## Best Possible Solution

Land the metadata/IMDS hard-deny after separating non-overridable metadata/IP checks from exact non-metadata hostname exceptions and updating the public SSRF docs/tests to match the final policy.

## Reproduction Assessment

Yes. Current main source shows the private-network skip bypasses pre-DNS hostname classification, and the PR body supplies before/after live output for the production resolver path; the PR regression against exact `agent.internal` exceptions is also source-reproducible from the changed call order.

## Solution Assessment

No. The metadata hard-deny direction is right, but the current implementation applies the broad blocked-host classifier before preserving exact non-metadata exceptions; the safer fix is to hard-block only metadata/IMDS before the skip path and keep documented exact exceptions working.
