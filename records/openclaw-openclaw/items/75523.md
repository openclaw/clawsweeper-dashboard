## Summary

Keep open: the SSRF hardening direction and supplied runtime proof are strong, but the current diff still regresses documented exact `allowedHostnames` exceptions for non-metadata blocked hosts such as `localhost`.

## What This Changes

This PR changes SSRF hostname/IP classification so metadata and `.internal` destinations remain blocked under private-network opt-in, adds pinned-DNS and dispatcher regression tests, and updates the changelog.

## Best Possible Solution

Land the hardening after splitting always-blocked metadata and `.internal` policy from exact non-metadata host exceptions, with focused regression coverage for both paths.

## Reproduction Assessment

Yes. Current main source shows the private-network skip bypasses hostname classification for `allowPrivateNetwork` and exact `allowedHostnames`, and the PR body supplies before/after runtime output for the metadata and `.internal` bypass cases.

## Solution Assessment

No. The hardening should stay, but the implementation needs a narrower classifier path that preserves documented exact `allowedHostnames` exceptions for non-metadata hosts while keeping metadata, IMDS, and `.internal` protections closed.
