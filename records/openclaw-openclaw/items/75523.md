## Summary

Keep open: the SSRF hardening direction is valid and the PR supplies real runtime proof, but the current patch still regresses exact non-metadata `allowedHostnames` exceptions that current tests and docs define.

## What This Changes

The PR makes SSRF guarded fetch, pinned-DNS resolution, and dispatcher overrides keep metadata and `.internal` destinations blocked under private-network opt-in, with tests and a changelog entry.

## Best Possible Solution

Land the hardening after separating non-overridable metadata/IP denies from exact non-metadata hostname exceptions and updating the public SSRF docs/tests.

## Reproduction Assessment

Yes. Current main source shows the private-network skip bypasses pre-DNS hostname classification, and the PR body supplies before/after live output for the production resolver path.

## Solution Assessment

No. The hardening should keep metadata and IMDS targets always blocked, but the current implementation applies the general blocked-host classifier before preserving exact non-metadata exceptions.
