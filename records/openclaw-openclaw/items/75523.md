## Summary

Keep open: this fixes a real SSRF bypass and the supplied live proof is sufficient, but the current head still breaks documented exact `allowedHostnames` exceptions for non-metadata blocked hostnames.

## What This Changes

The PR moves SSRF hostname and cloud-metadata checks ahead of private-network skips, adds pinned-DNS and dispatcher regression coverage, and records the hardening in the changelog.

## Best Possible Solution

Land the SSRF hardening after splitting always-blocked metadata/IP policy from exact non-metadata host exceptions and aligning docs/tests.

## Reproduction Assessment

Yes. Current main source shows the private-network skip bypasses pre-DNS hostname classification, and the PR body supplies before/after live output for the production resolver path.

## Solution Assessment

No. The hardening direction is right, but the implementation should preserve documented exact non-metadata `allowedHostnames` exceptions while keeping metadata hosts and IMDS IPs closed.
