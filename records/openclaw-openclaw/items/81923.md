## What This Changes

The issue reports that upgrading to OpenClaw 2026.5.12 broke a custom OpenAI-compatible Xiaomi MiMo provider through tool-schema normalization and fallback auth handling.

## Best Possible Solution

Keep the Xiaomi/MiMo owner-plugin fixes, then decide a narrow public custom-provider compat path or endpoint-detected schema cleanup for providers that reject empty `items`, while routing the Telegram/status auth-source display fragment through the narrower related issue.
