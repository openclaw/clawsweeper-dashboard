## Summary

This should stay open: the report is recent, externally reported, and current main still contains the globally applied array-schema normalization plus the missing-auth fallback path implicated by the issue. The auth behavior has been narrowed on main since the 2026.5.12 release, but the exact custom-provider failure still needs a redacted config/provider repro and maintainer judgment rather than cleanup closure.

## What This Changes

The issue reports that upgrading to OpenClaw 2026.5.12 broke a custom OpenAI-compatible provider through injected array `items` schemas, then hid the primary schema error behind an Ollama missing-auth fallback message.

## Best Possible Solution

Keep the issue open, collect a redacted provider/model config plus provider response proof, then decide a targeted schema-compat or diagnostic fix while preserving the SecretRef credential boundary.

## Solution Assessment
