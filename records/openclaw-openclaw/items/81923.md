## What This Changes

The issue reports that the 2026.5.12 upgrade broke a custom OpenAI-compatible provider through injected array `items` tool schemas and changed provider API-key resolution.

## Best Possible Solution

Keep the security-motivated SecretRef contract, add or document a targeted per-model/provider schema-compat path for backends that reject injected `items`, and improve fallback summaries so the primary schema failure remains visible.

## Reproduction Assessment

## Solution Assessment
