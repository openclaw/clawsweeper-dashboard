## What This Changes

The issue asks OpenClaw to inject `OPENCLAW_SESSION_KEY` and `OPENCLAW_AGENT_ID` into exec child process environments so scripts can route callbacks back to the active agent session.

## Best Possible Solution

Define the public exec session-context env contract, including non-auth semantics, override behavior, and whether `OPENCLAW_SESSION_ID` is included, then land the linked PR or a replacement with docs, tests, and real host proof.

## Reproduction Assessment

## Solution Assessment

Unclear. Exec env injection is the likely right layer, but the best final shape depends on maintainer decisions about env names, `OPENCLAW_SESSION_ID`, override precedence, and non-auth security wording.
