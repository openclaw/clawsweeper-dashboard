## Summary

Keep open. Current main still does not inject either requested variable into exec child-process environments, and the issue is already paired with open PR #73349, which uses closing syntax but still needs review/revision before this issue can be resolved.

## What This Changes

Expose trusted OPENCLAW_SESSION_KEY and OPENCLAW_AGENT_ID values to child processes spawned by the exec tool so scripts can route callbacks back to the active agent session.

## Best Possible Solution

Keep this issue open while #73349 is reviewed. The final fix should inject the trusted session and agent values after env sanitization for gateway, sandbox, PTY, TypeScript node-host, and macOS node-host paths, prevent user-supplied values from spoofing them, update the runtime env docs, and add regression tests that prove the final child process can read both variables.

## Reproduction Assessment

Yes. A source-level reproduction path is high-confidence: current main can invoke exec, but the inspected env-construction paths only add OPENCLAW_SHELL and never add OPENCLAW_SESSION_KEY or OPENCLAW_AGENT_ID.

## Solution Assessment

Unclear for the current open PR as-is. The PR is the right implementation lane, but the existing review finding means the safest solution is to revise it so trusted runtime env is injected after node-host sanitizer handling rather than only through normal requested env overrides.
