## Summary

Keep open. Current main still lacks the requested exec child-process env markers, and the open implementation candidate with closing syntax is https://github.com/openclaw/openclaw/pull/73349.

## What This Changes

Expose the current exec session key and agent id to child processes as `OPENCLAW_SESSION_KEY` and `OPENCLAW_AGENT_ID`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73349 or a narrow successor that injects trusted session and agent env values across gateway, sandbox, PTY, node-host, and macOS paths with docs and regression tests.

## Reproduction Assessment

Yes for source-level verification. Current env construction never adds the requested variables, so an exec command that prints them would only see existing inherited/request env plus `OPENCLAW_SHELL=exec`.

## Solution Assessment

Yes. Carrying trusted agent/session values into the existing child-env composition points, after sanitization and with override prevention, is the narrow maintainable fix; the active PR is the right lane to review.
