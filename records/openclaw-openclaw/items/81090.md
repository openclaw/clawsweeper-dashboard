## What This Changes

The branch adds a `before_route_inbound_message` plugin hook with redirect/suppress results, wires it into channel-turn and legacy inbound dispatch paths, and adds focused channel-turn tests.

## Best Possible Solution

Reconcile this with the paired issue and overlapping `pre_route` design, then implement one documented typed routing seam in the correct channel-turn/dispatch path with guardrails, regression tests, and real setup proof.

## Reproduction Assessment

Not applicable as a user bug reproduction: this PR adds a new plugin API capability. Source inspection gives high-confidence patch failures because the added channel-turn code reads properties that current main’s types do not expose.

## Solution Assessment

No. The current patch is not the best path as-is because it is not type-correct, leaves legacy redirects ineffective for finalized contexts, and should be reconciled with the broader pre-route API/security design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Read hook inputs from the resolved turn:** `src/channels/turn/kernel.ts:470-482`
  - body: `RunChannelTurnParams` does not carry `ctxPayload`, and `ChannelTurnResolved` does not expose `sessionKey`, so this added block will not typecheck on current main. Build the event from `resolved.ctxPayload` and `resolved.routeSessionKey` or introduce a typed pre-route facts object before this path.
  - confidence: 0.94
- **[P2] Update the compat context when redirecting:** `src/plugin-sdk/inbound-reply-dispatch.ts:202-205`
  - body: The legacy helper rewrites only `params.route.sessionKey`; it still passes the original `ctxPayload` into `recordChannelMessageReplyDispatch`, and the prepared turn path prefers `ctxPayload.SessionKey`. Normal finalized messages will keep using the original session, so redirects on this path are ignored.
  - confidence: 0.9
- **[P3] Call the matcher and assert the redirected session:** `src/channels/turn/kernel.test.ts:952`
  - body: This line only reads the `toHaveBeenCalledOnce` matcher instead of calling it, and the test never checks the recorded or dispatched session key. As written, the redirect test would not catch a route-only redirect being ignored.
  - confidence: 0.86
