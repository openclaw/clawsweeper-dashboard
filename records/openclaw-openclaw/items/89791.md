## Summary

Keep open: current main and v2026.6.8 still source the generated launchd env file without clearing launchd markers, and still generate gateway/node service envs with `OPENCLAW_LAUNCHD_LABEL`. The open docs PR is useful partial mitigation, but it does not settle the runtime, doctor, or install policy for this dual LaunchAgent loop.

## What This Changes

The issue asks OpenClaw to document and/or harden macOS launchd dual gateway/node LaunchAgent installs so launchd markers do not trigger supervisor respawn loops and blocked message dispatch.

## Best Possible Solution

Have maintainers choose the durable launchd policy, land the docs runbook if acceptable, and keep runtime wrapper generation plus doctor/install hardening tracked until macOS upgrade proof is available.
