## Summary

Keep open: the report describes a fresh regression with multiple independent confirmations, and current source plus the published Weixin 2.4.1 package show this sits on the external channel runtime contract rather than a stale support request.

## What This Changes

The issue reports that `@tencent-weixin/openclaw-weixin@2.4.1` stops processing inbound messages after upgrading OpenClaw to 2026.5.4 because channel runtime helpers are missing at message handling time.

## Best Possible Solution

Add a regression around an installed external channel plugin that depends on `api.runtime.channel` and `ctx.channelRuntime`, then preserve full channel runtime injection across gateway startup/cache paths or coordinate a Weixin plugin update if the public contract changed.

## Reproduction Assessment

No live high-confidence reproduction was run because it requires a Weixin account and valid session token. The source path is still strong: the affected package dereferences full channel runtime helpers and current comments provide matching runtime errors from multiple users.

## Solution Assessment

Unclear. The likely fix is narrow, but maintainers should first decide whether core must harden the external channel runtime contract or whether the official external plugin should switch to the gateway-provided `ctx.channelRuntime` path and republish.
