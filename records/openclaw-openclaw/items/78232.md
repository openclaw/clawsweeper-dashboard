Keep open: the report has credible user logs from multiple platforms and matches the current Weixin package's inbound processing code, but the exact OpenClaw-side runtime object that becomes partial was not live-captured in this read-only review, so this should not be closed or auto-fixed without maintainer/plugin-owner validation.

## What This Changes

The issue reports that `@tencent-weixin/openclaw-weixin@2.4.1` crashes on inbound messages after upgrading to OpenClaw 2026.5.4 because expected channel runtime helpers are undefined.

## Best Possible Solution

Restore the documented external-channel runtime contract for Weixin in a real configured setup, or coordinate a Weixin plugin release/catalog update that consumes the current Gateway `channelRuntime` correctly and verifies inbound message routing end to end.

## Reproduction Assessment

No high-confidence live reproduction was established because the Weixin account/session path was not available. Source and npm-package inspection do show that a partial `channelRuntime` would throw at the exact helper families named in the report.

## Solution Assessment

Unclear. A narrow core compatibility fix might be possible, but the safer solution needs maintainer choice between restoring a synchronous/runtime compatibility path in OpenClaw and updating the external Weixin plugin plus catalog/docs.
