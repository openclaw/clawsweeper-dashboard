Overall confidence: 0.91

Full review comments:

- **[P1] Pass resolved Slack config into search helpers:** `extensions/slack/src/actions.ts:333-347`
  - body: `resolveUserToken()` calls `loadConfig()` inside the action helper, bypassing the resolved config/account already built in `handleSlackAction()`. This breaks the current SecretRef/account selection boundary; search should receive the resolved token/config from runtime.
  - confidence: 0.94
- **[P2] Return Slack search hits in the results envelope:** `extensions/slack/src/action-runtime.ts:494`
  - body: `handleSlackAction()` returns Slack search data as top-level `messages`/`total`, but the shared CLI search formatter reads `payload.results.messages`. Successful Slack searches will not render the normal search-results table unless this uses the shared results payload shape.
  - confidence: 0.9
- **[P2] Scope search discovery to the selected account:** `extensions/slack/src/message-actions.ts:55-63`
  - body: Discovery adds `search` when any filtered Slack account has a user token and search enabled, but invocation without `accountId` resolves the selected/default account. A non-default search-capable account can advertise an action the default invocation cannot run.
  - confidence: 0.86
- **[P2] Report channel-list truncation:** `extensions/slack/src/action-runtime.ts:479`
  - body: `channel-list` stops at the default 500-channel cap, but the runtime returns only `channels`. Larger workspaces are silently truncated, leaving callers unable to know whether discovery is complete or whether another cursor/page is needed.
  - confidence: 0.82
