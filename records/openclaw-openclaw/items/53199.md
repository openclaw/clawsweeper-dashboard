Full review comments:

- **[P1] Use the resolved Slack account for search tokens:** `extensions/slack/src/actions.ts:333-347`
  - body: The new `resolveUserToken()` calls `loadConfig()` from the Slack action helper, but the diff does not import that symbol and current Slack action helpers receive resolved `cfg`/account data from `handleSlackAction()`. Search should use the already selected account/user token through `SlackActionClientOpts` instead of re-reading ambient config.
  - confidence: 0.95
- **[P2] Return search hits in the shared results envelope:** `extensions/slack/src/action-runtime.ts:494`
  - body: The CLI formatter for `search` reads `payload.results.messages`; this return places `messages` and `total` at the top level, so successful Slack searches render as a generic object instead of search-result rows. Wrap the Slack hits under `results` or update the shared formatter/tests consistently.
  - confidence: 0.9
- **[P2] Scope search discovery to the selected account:** `extensions/slack/src/message-actions.ts:55-63`
  - body: Discovery adds `search` when any Slack account has a user token and search enabled, but current main supports account-scoped discovery and invocation without `accountId` resolves the selected/default account. A search-capable non-default account can expose an action the selected account cannot run.
  - confidence: 0.86
- **[P2] Expose channel-list truncation metadata:** `extensions/slack/src/action-runtime.ts:479`
  - body: `listSlackChannels()` stops at the default 500-channel cap, but the runtime returns only `channels`. Larger workspaces are silently incomplete and callers have no `hasMore`, `nextCursor`, or limit metadata to continue discovery.
  - confidence: 0.82
