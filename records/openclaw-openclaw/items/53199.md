## Summary

Keep open: current main still lacks the requested Slack `channel-list` and `search` action surface, but this PR is not merge-ready because its head predates current Slack runtime/config boundaries, returns search data outside the shared formatter contract, can advertise search for the wrong account context, and silently truncates channel-list output.

## What This Changes

The PR adds Slack `channel-list` and `search` message actions, `channelList` config/schema metadata, generated config baselines, and Slack dispatch/discovery wiring.

## Best Possible Solution

Implement the Slack-owned actions through current `extensions/slack` runtime boundaries with account-scoped discovery, shared search result payloads, explicit pagination/truncation metadata, focused tests, and Slack docs.

## Reproduction Assessment

Yes for the PR blockers: source inspection of the PR head and current main shows the raw config reload, formatter-envelope mismatch, account-context discovery mismatch, and silent channel-list cap without needing a Slack token. For the feature request itself, current-main absence is verified by source and docs inspection.

## Solution Assessment

No. The feature direction is useful, but this branch is not the best merge path today because it needs rebasing onto current Slack runtime boundaries and coordination with the narrower open Slack search PR before channel-list can be safely landed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Pass resolved Slack config into search helpers:** `extensions/slack/src/actions.ts:333-347`
  - body: `resolveUserToken()` calls `loadConfig()` inside the Slack action helper, bypassing the resolved config/account already built in `handleSlackAction()`. Current Slack helpers require config to be passed through the runtime boundary so SecretRef/account selection keeps working; search should receive the resolved account/user token from runtime instead of re-reading raw config.
  - confidence: 0.94
- **[P2] Return Slack search hits in the results envelope:** `extensions/slack/src/action-runtime.ts:494`
  - body: `handleSlackAction()` returns Slack search data as top-level `messages`/`total`, but the shared CLI search formatter reads `payload.results.messages`. Successful Slack searches will miss the normal search-results table unless this uses the shared `results` payload shape.
  - confidence: 0.9
- **[P2] Scope search discovery to the selected account:** `extensions/slack/src/message-actions.ts:55-63`
  - body: Discovery adds `search` when any filtered Slack account has a user token and search enabled, but invocation without `accountId` resolves the selected/default account. A non-default search-capable account can therefore advertise an action that the default invocation cannot run.
  - confidence: 0.86
- **[P2] Expose channel-list truncation metadata:** `extensions/slack/src/action-runtime.ts:479`
  - body: `channel-list` stops at the default 500-channel cap, but the runtime returns only `channels`. Larger workspaces are silently truncated, leaving callers unable to know whether discovery is complete or whether they need another page/cursor.
  - confidence: 0.82
