## Summary

Keep open: this is the active Feishu URL-rendering fix candidate, and current main still sends raw post markdown, but the submitted helper still mutates a valid existing Markdown-link shape and lacks real Feishu after-fix proof.

## What This Changes

The PR adds a Feishu-local helper that wraps bare http(s) URLs as explicit Markdown links before serializing post `md` payloads, with helper and send-path regression tests.

## Best Possible Solution

Land one Feishu plugin-local URL-boundary fix that preserves full clickable bare URLs without mutating existing Markdown/link/code regions, with focused regression coverage and redacted real Feishu after-fix proof.

## Reproduction Assessment

Yes, source-level: current main still passes `messageText` into the Feishu post `md` tag unchanged, and the linked user report provides concrete send steps plus a screenshot. I did not establish a live current-main Feishu-client reproduction during this read-only review.

## Solution Assessment

No, not as currently submitted. A Feishu-local wrapper at `buildFeishuPostMessagePayload` is the right boundary, but the mask must preserve nested or escaped existing Markdown links before this is merge-ready.

## Review Findings

Overall correctness: patch is incorrect
