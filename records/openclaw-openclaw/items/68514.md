## Summary

Keep this PR open for repair rather than cleanup. Current main still sends Feishu post text as raw `md`, so the underlying URL-with-underscore bug is real and unimplemented, but this branch is not merge-ready because its Markdown-link masking misses valid nested-bracket link text and the user-facing fix lacks the required changelog entry.

## What This Changes

The branch adds a Feishu-local helper that wraps bare http(s) URLs as Markdown links in post payload text, wires it into the Feishu post payload builder, and adds helper plus send-path regression tests.

## Best Possible Solution

Land one Feishu-local fix that preserves full clickable bare URLs in post messages while robustly leaving existing Markdown links, image links, autolinks, inline code, and fenced code untouched; include the required changelog entry, then close the duplicate Feishu URL PRs and the shared bug report from that canonical merge.

## Reproduction Assessment

Yes. The user report in #41860 gives a Feishu send command and screenshot, current main can be inspected to show raw `md` payload text, and the remaining PR defect is reproducible with helper input like `[foo [bar]](https://example.com/a_b)`.

## Solution Assessment

No, not as currently written. A Feishu-local wrapping helper is a narrow maintainable direction, but the masking layer must parse or otherwise protect existing Markdown links with nested/escaped link text before this can safely merge.
