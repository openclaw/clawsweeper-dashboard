## Summary

Keep open: the PR targets a real Feishu URL-rendering bug at the right plugin-local payload boundary, but the current head still has a nested Markdown-link masking defect and lacks real after-fix Feishu proof.

## What This Changes

The branch adds a Feishu helper that wraps bare http(s) URLs in post Markdown payloads as explicit Markdown links, plus helper and send-path regression tests.

## Best Possible Solution

Land a Feishu plugin-local URL-boundary fix in the post payload builder that preserves bare URL clickability without mutating existing link/code regions, backed by focused tests and redacted real Feishu proof.

## Reproduction Assessment

Yes, at source level: the linked bug gives exact Feishu send steps and a screenshot, and current main still routes that text into a raw Feishu post `md` element. I did not establish a live Feishu-client reproduction in this read-only review.

## Solution Assessment

No, not as currently submitted. The Feishu-local payload-builder boundary is right, but the Markdown-link mask needs to preserve nested or escaped link text and the PR still needs real after-fix Feishu proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Mask nested-bracket Markdown links before wrapping URLs:** `extensions/feishu/src/markdown-urls.ts:8`
  - body: The Markdown-link mask only supports link text without `]`, so an existing link like `[foo [bar]](https://example.com/a_b)` is not protected before the bare-URL pass. The helper then rewrites the URL inside an already-linked destination, breaking content it promises to leave untouched.
  - confidence: 0.91
