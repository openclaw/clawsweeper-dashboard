## Summary

Keep open: the Feishu URL bug is still present on current main, and this PR is a narrow useful direction, but the latest diff still mutates valid existing Markdown links with nested bracket text and is missing the required changelog entry for a user-facing fix.

## What This Changes

The PR adds a Feishu-local URL-wrapping helper used by post message payloads and adds helper/send-path regression tests for underscore-containing bare URLs.

## Best Possible Solution

Land one Feishu-local implementation that preserves full clickable bare URLs in post `md` payloads while robustly leaving existing Markdown links, image links, autolinks, inline code, and fenced code untouched, with a changelog entry and coordinated closure of the duplicate Feishu URL PRs.

## Reproduction Assessment

Yes. The original bug has concrete Feishu send steps and a screenshot in #41860, current main shows raw `md` payload text, and the PR-specific defect is reproducible with helper input like `[foo [bar]](https://example.com/a_b)`.

## Solution Assessment

No, not as currently submitted. A Feishu-local wrapper is the narrow maintainable direction, but the masking logic needs to handle nested or escaped Markdown link text before wrapping bare URLs, and the user-facing fix needs a changelog entry.
