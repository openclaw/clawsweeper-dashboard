## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve default preamble progress:** `extensions/telegram/src/bot-message-dispatch.ts:2236-2240`
  - body: Current main documents Codex preamble/commentary text as part of Telegram's default tool-progress preview, and the existing onItemEvent path sends every item event through pushStreamToolProgress. This branch returns early for `payload.kind === "preamble"` and only calls `pushCommentaryProgress`, which is restricted to `streamMode === "progress"` plus the new `streaming.progress.commentary` opt-in, so default partial-preview users lose the existing preamble progress signal. Keep the existing preview/default path and make the new commentary option additive, or get an explicit maintainer decision to break that behavior.
  - confidence: 0.94
- **[P2] Reset status dedupe after reasoning appends:** `extensions/telegram/src/bot-message-dispatch.ts:1369-1381`
  - body: `appendStatusLine` suppresses a line when it equals `interleavedPrevStatusLine`, but `appendInterleavedReasoning` appends new reasoning text without clearing that previous status. A sequence such as tool Read, reasoning text, tool Read will therefore drop the second visible status line even though it is no longer consecutive in the rendered transcript. Reset the status dedupe after successful reasoning appends or compare against the actual rendered tail.
  - confidence: 0.87
