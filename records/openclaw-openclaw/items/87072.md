Overall correctness: patch is incorrect

Full review comments:

- **[P2] Honor the commentary opt-in in progress mode:** `extensions/telegram/src/bot-message-dispatch.ts:2242-2252`
  - body: When `streaming.mode` is `progress` and `streaming.progress.commentary` is left at its documented default `false`, `pushCommentaryProgress()` returns false and this preamble event falls through to the normal progress renderer. That still stores the preamble in `streamToolProgressLines`, so a later tool event or gate start can render commentary the user did not opt into. Suppress preamble/commentary in progress mode unless the new commentary option is true, and only keep the fallback for the non-progress preview/interleaved lanes that preserve released behavior.
  - confidence: 0.86
