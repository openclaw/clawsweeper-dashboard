## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve default preamble progress:** `extensions/telegram/src/bot-message-dispatch.ts:2236-2240`
  - body: This branch intercepts `payload.kind === "preamble"`, calls `pushCommentaryProgress`, and then returns. `pushCommentaryProgress` only does anything in `streamMode === "progress"` when `streaming.progress.commentary` is enabled, so default Telegram configs and all partial-mode configs skip the current `pushStreamToolProgress` path entirely. Current main sends these item events through the normal tool-progress preview, and the docs say Codex preamble/commentary progress is enabled by default; please keep that default path unless maintainers explicitly accept the breaking opt-in behavior.
  - confidence: 0.94
- **[P2] Reset status dedupe after reasoning appends:** `extensions/telegram/src/bot-message-dispatch.ts:1346`
  - body: `appendStatusLine` dedupes against `interleavedPrevStatusLine`, but `appendInterleavedReasoning` never clears that value. A turn like `tool: Read` -> reasoning text -> `tool: Read` will suppress the second tool line even though it is no longer consecutive duplicate status, so the Telegram Thinking lane can omit real repeated work. Clear the status-line dedupe marker whenever reasoning/body text is appended and add a regression test for that sequence.
  - confidence: 0.9
