## Summary

Keep open. Current main still gates the onboarding Homebrew recommendation with a Windows-only exclusion, so FreeBSD can still enter the prompt path when a selected missing skill has a brew installer and `brew` is absent; the issue is also paired with open closing PRs that should be reviewed or resolved before the issue is closed.

## What This Changes

The issue reports that `openclaw onboard` shows Homebrew setup guidance on FreeBSD.

## Best Possible Solution

Land one canonical narrow onboarding fix that allows the Homebrew prompt only on Homebrew-supported platforms, adds a FreeBSD regression test proving the prompt and `detectBinary("brew")` path are skipped there, updates changelog credit as needed, and closes or supersedes the duplicate open PRs after the chosen fix lands.

## Reproduction Assessment

Yes. A source-level reproduction is clear: on `freebsd`, selecting a brew-backed missing skill with no `brew` binary makes `process.platform !== "win32"` true, so `needsBrewPrompt` can emit the Homebrew note.

## Solution Assessment

Yes for the FreeBSD bug. A small Homebrew platform allowlist plus focused regression coverage is the narrowest maintainable fix; WSL messaging should be handled separately if maintainers want Linux-in-WSL behavior to differ.
