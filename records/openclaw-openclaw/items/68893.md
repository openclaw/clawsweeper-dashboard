## Summary

Keep open: current main and the latest release still use a non-Windows Homebrew prompt gate, and the linked closing PR is open/conflicting rather than merged.

## What This Changes

The issue reports that `openclaw onboard` recommends Homebrew on FreeBSD even though Homebrew is not a supported FreeBSD install path.

## Best Possible Solution

Land one canonical macOS/Linux Homebrew prompt allowlist with unsupported-platform regression coverage, then close this issue through the merged fix.

## Reproduction Assessment

Yes, at source level. On current main, `process.platform = "freebsd"`, a selected missing brew-backed skill, and no `brew` satisfy the existing non-Windows gate and reach the Homebrew note.

## Solution Assessment

Yes. A `darwin`/`linux` allowlist before the Homebrew prompt is the narrow maintainable fix, and this issue should stay open until https://github.com/openclaw/openclaw/pull/69002 or an equivalent fix merges.
