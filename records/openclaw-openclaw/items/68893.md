## Summary

Keep this open: current main and the latest release still gate the Homebrew onboarding prompt on only `win32`, so FreeBSD can still see the Homebrew recommendation after selecting a brew-backed skill. There is already an open linked fix PR that owns the implementation path, so this issue should stay open until that PR lands or maintainers choose a clean replacement.

## What This Changes

Running `openclaw onboard` on FreeBSD should not show the Homebrew recommendation or install command for brew-backed skills.

## Best Possible Solution

Land a narrow fix that allows the Homebrew recommendation only on supported Homebrew platforms, keeps macOS/Linux behavior intact, and adds a FreeBSD/BSD regression test; review or replace https://github.com/openclaw/openclaw/pull/69002 as the implementation path.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, from source: on FreeBSD with brew absent, selecting a brew-backed skill such as `video-frames` reaches a prompt guarded only by `process.platform !== "win32"`.

## Solution Assessment

Is this the best way to solve the issue? Yes: a small platform allowlist around the Homebrew recommendation plus a FreeBSD regression test is the narrow maintainable fix, while the broader WSL comment should stay separate unless verified.
