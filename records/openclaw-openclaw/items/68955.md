## Summary

Keep open. Current main still leaves Telegram native slash-command handlers outside the feedback-enabled regular message path, and the linked fork PR is open and unmerged, so upstream still needs a reviewed fix or replacement.

## What This Changes

The issue requests Telegram slash commands such as `/new` and `/reset` to emit the same ack reaction and typing indicator as regular inbound Telegram messages.

## Best Possible Solution

Implement the feedback inside the bundled Telegram plugin by threading the existing ack-reaction scope and send-chat-action handler into native command registration, then emit best-effort ack and typing after skip/auth checks for both built-in and plugin-backed commands. Add focused Telegram native-command tests plus a short docs/changelog note if the behavior changes user-visible UX.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available: regular Telegram messages build a context with ack and typing dependencies, while native slash commands execute in separate `bot.command(...)` handlers that do not receive or invoke those feedback hooks.

## Solution Assessment

Yes. The narrowest maintainable fix is an explicit Telegram command feedback helper after skip/auth checks; routing slash commands through the full regular-message pipeline would risk coupling command parsing, menus, and plugin command execution to unrelated message dispatch behavior.
