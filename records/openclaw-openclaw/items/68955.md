## Summary

Keep open. Current main and v2026.6.8 still route Telegram slash commands through dedicated native/plugin command handlers without the regular message-path ack reaction and early typing cue, and the only linked implementation is an open conflicting fork PR rather than an upstream fix.

## What This Changes

Request slash commands such as `/new` and `/reset` in Telegram to emit the same ack reaction and typing feedback as regular inbound messages.

## Best Possible Solution

Implement Telegram-local command feedback after auth for both native and plugin slash-command handlers, with focused tests for ack scope, typing, auth rejection, thread IDs, and disabled reaction settings.

## Reproduction Assessment

Yes, at source level. Current main shows regular messages entering the message-context feedback path, while Telegram native and plugin slash-command handlers bypass those ack/typing hooks; I did not run a live Telegram account reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers choose the command feedback lifecycle. The narrowest maintainable shape appears to be a Telegram-local helper invoked after command auth in both native and plugin command handlers, rather than rerouting commands through the full message pipeline or landing the broad conflicting fork PR.
