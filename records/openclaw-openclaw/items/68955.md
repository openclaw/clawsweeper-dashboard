## Summary

Keep open. Current main and the latest release still route Telegram native slash commands through separate handlers that do not receive or call the regular inbound-message ack reaction and typing plumbing, and the only linked fix is an open external fork PR rather than merged upstream work.

## What This Changes

The issue requests Telegram slash commands such as `/new` and `/reset` to emit the same ack reaction and typing indicator as regular inbound messages.

## Best Possible Solution

Add a Telegram-owned post-auth native-command feedback helper that reuses the existing ack gate and shared chat-action handler while preserving the native command fast path.

## Reproduction Assessment

Yes, at source level: regular Telegram messages build a feedback-capable context and call typing/ack logic, while native slash commands enter separate `bot.command` handlers with no feedback dependencies or calls. I did not run live Telegram in this read-only review.

## Solution Assessment

Yes, if maintainers accept the UX change: explicit post-auth feedback in the Telegram native-command path is narrower and safer than forcing slash commands through the full regular-message pipeline. The external fork can inform the patch, but a smaller upstream change is likely the maintainable route.
