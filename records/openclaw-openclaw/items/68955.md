## Summary

Keep open. Current main still wires Telegram ack reactions and typing cues through the regular message path, while native slash-command handlers lack the feedback dependencies or calls; an open fork PR already proposes a fix and should be reviewed, upstreamed, replaced, or closed before this issue is resolved.

## What This Changes

Request Telegram slash commands such as `/new` and `/reset` emit the same ack reaction and typing indicator as regular inbound messages.

## Best Possible Solution

Add a Telegram-owned post-auth slash-command feedback helper for built-in and plugin native commands that reuses the existing ack gate and sendChatAction handler, with focused tests, docs, and a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available: regular Telegram messages build a feedback-capable context and call typing/ack paths, while native slash commands enter separate `bot.command(...)` handlers with no feedback wiring.

## Solution Assessment

Yes. Explicit post-auth feedback in the Telegram native command path is the narrowest maintainable fix; routing commands through the full regular-message pipeline would risk unrelated command parsing and session behavior.
