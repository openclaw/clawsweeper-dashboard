## Summary

Keep open. Current main still gives Telegram ack reaction and typing feedback only to the regular inbound-message path, while built-in and plugin native slash commands run through separate handlers without those feedback dependencies or calls; the linked fork PR is open and not merged upstream.

## What This Changes

Request Telegram slash commands such as `/new` and `/reset` to emit the same ack reaction and typing indicator feedback as regular inbound messages.

## Best Possible Solution

Add a Telegram-owned post-auth native-command feedback helper that reuses the existing ack gate and shared chat-action handler for built-in and plugin slash commands, with focused tests and real Telegram proof.

## Reproduction Assessment

Yes, at source level: regular Telegram messages call the existing typing and ack paths, while native slash commands enter separate `bot.command` handlers without those dependencies or calls. I did not run a live Telegram E2E reproduction in this read-only pass.

## Solution Assessment

Yes, if maintainers accept the UX change: explicit post-auth feedback in the Telegram native-command path is narrower and safer than forcing slash commands through the full regular-message pipeline.
