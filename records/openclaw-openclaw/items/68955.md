## Summary

Keep open. Current main still wires Telegram ack reactions and typing only through the regular inbound-message path, while native slash-command handlers remain on a separate fast path without those feedback dependencies or calls; the linked fork PR is useful context but is open outside upstream and has not landed here.

## What This Changes

Request that Telegram slash commands such as `/new` emit the same ack reaction and typing cues as regular inbound messages when feedback config enables them.

## Best Possible Solution

Add a Telegram-owned post-auth native-command feedback helper that reuses the existing ack gate and shared chat-action handler, with focused unit coverage and live Telegram proof, while preserving the native-command fast path.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: regular Telegram messages build a feedback-capable context and call typing/ack logic, while native slash commands enter separate `bot.command` handlers without those feedback dependencies or calls. I did not run live Telegram in this read-only review.

## Solution Assessment

Yes, if maintainers accept the UX change: explicit post-auth feedback in the Telegram native-command path is narrower and safer than forcing slash commands through the full regular-message pipeline. The linked fork can inform the implementation, but a smaller upstream patch is likely the maintainable path.
