## Summary

Keep open. Current main still gives Telegram ack reaction and typing feedback only to the regular inbound-message path, while native slash-command handlers remain separate and do not receive or call those feedback hooks; the linked fork PR is open and unmerged.

## What This Changes

The issue requests Telegram native slash commands such as `/new` and `/reset` to emit the same ack reaction and typing feedback as regular inbound messages.

## Best Possible Solution

Add a Telegram-owned post-auth native-command feedback helper that reuses the existing ack gate and shared chat-action handler while preserving the native command fast path.

## Reproduction Assessment

Yes, at source level: regular Telegram messages call the existing typing and ack paths, while native slash commands enter separate `bot.command` handlers without those dependencies or calls. I did not run a live Telegram E2E reproduction.

## Solution Assessment

Yes, if maintainers accept the UX change: explicit post-auth feedback in the Telegram native-command path is narrower than forcing slash commands through the full regular-message pipeline. The linked fork can inform the fix, but a clean upstream patch should stay focused on Telegram command feedback.
