## Summary

Keep open. Current main still wires Telegram ack reactions and typing cues through the regular message path only, while native slash-command handlers lack the feedback dependencies or calls, and the linked fork PR is open and unmerged.

## What This Changes

Request Telegram native slash commands such as `/new` and `/reset` to emit the same ack reaction and typing indicator as regular inbound messages.

## Best Possible Solution

Add a Telegram-owned post-auth slash-command feedback helper that reuses the existing ack gate and typing handler for built-in and plugin native commands, with focused tests and a changelog note.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available: regular Telegram messages build a feedback-capable context and call typing/ack paths, while native slash commands enter separate `bot.command(...)` handlers with no feedback wiring.

## Solution Assessment

Yes. Explicit post-auth feedback in the Telegram native command path is the narrowest maintainable fix; routing commands through the full regular-message pipeline would risk unrelated parsing and session behavior.
