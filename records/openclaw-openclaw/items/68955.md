## Summary

Keep open. Current main and v2026.6.8 still keep Telegram slash commands on separate native/plugin `bot.command` handlers without the regular message-path ack reaction and typing dependencies, and the only linked implementation is an external fork PR that is open, conflicting, and not an upstream fix path.

## What This Changes

Request that Telegram slash commands such as `/new` emit the same ack reaction and typing feedback as regular inbound messages.

## Best Possible Solution

Land a narrow Telegram-owned command feedback path after command auth and before slow native or plugin command work, preserving native command routing while adding focused tests and real Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main still builds ack reactions and typing only in the regular message context path, while native and plugin slash commands run through separate `bot.command` handlers without those feedback dependencies.

## Solution Assessment

Unclear pending maintainer direction. A small helper invoked after command auth appears to be the narrow owner-boundary fix, while routing commands through the full message pipeline would be broader and risk command-routing regressions.
