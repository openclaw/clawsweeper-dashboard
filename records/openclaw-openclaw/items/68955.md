## Summary

Keep open. Current main still leaves Telegram native slash-command handlers outside the regular Telegram message feedback path, and the only linked implementation is an open, unmerged fork PR rather than a shipped upstream fix.

## What This Changes

Telegram slash commands such as `/new` and `/reset` should emit the same ack reaction and typing indicator as regular Telegram messages.

## Best Possible Solution

Add a Telegram-owned feedback helper for native commands that runs after skip/auth checks, reuses the existing ack-reaction gate and send-chat-action handler for built-in and plugin commands, and covers the behavior with focused Telegram tests plus a changelog/docs note.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available: regular messages build a feedback-capable context, while Telegram native slash commands execute in separate `bot.command(...)` handlers without feedback dependencies or calls.

## Solution Assessment

Yes. The narrowest maintainable direction is explicit Telegram slash-command feedback after authorization; routing commands through the full regular-message pipeline would couple command parsing and plugin-command execution to unrelated message dispatch behavior.
