## Summary

Keep open. Current main and the latest release still keep Telegram native slash commands on separate `bot.command` handlers without the regular message-path ack reaction or typing plumbing, and the only linked implementation is an open external fork PR rather than an upstream fix.

## What This Changes

The issue requests ack reactions and typing indicators for Telegram slash commands such as `/new`, matching regular inbound message feedback.

## Best Possible Solution

Add Telegram-owned command feedback after auth and before potentially slow command work, while preserving native command fast-path behavior and regular-message feedback semantics.

## Reproduction Assessment

Yes. Source inspection shows regular Telegram messages emit typing and ack reactions from `buildTelegramMessageContext`, while native and plugin slash commands enter separate `bot.command` handlers without those feedback dependencies or calls.

## Solution Assessment

Unclear pending maintainer direction. A small helper called after command auth looks like the narrow owner-boundary fix, while routing commands through the full message pipeline would conflict with the fast-path command design.
