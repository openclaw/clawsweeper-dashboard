## Summary

Keep open. Current main still wires Telegram ack reactions and typing through the regular message path only; native slash commands remain on a separate fast path without those feedback dependencies or calls, and the linked fork PR is open outside upstream rather than merged here.

## What This Changes

The issue requests Telegram slash commands such as `/new` and `/reset` to emit the same ack reaction and typing feedback as ordinary Telegram messages.

## Best Possible Solution

Add a Telegram-owned post-auth native-command feedback helper that reuses the existing ack gate and shared chat-action handler, with focused unit coverage and live Telegram proof while preserving the native command fast path.

## Reproduction Assessment

Yes at source level: regular Telegram messages build feedback-capable context and call typing/ack logic, while native slash commands enter separate `bot.command` handlers without those dependencies or calls. I did not run live Telegram in this read-only review.

## Solution Assessment

Yes, if maintainers accept the UX change: explicit post-auth feedback in the Telegram native-command path is narrower than forcing slash commands through the full regular-message pipeline. The fork PR can inform the implementation, but a smaller upstream patch is likely the maintainable path.
