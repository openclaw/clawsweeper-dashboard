## Summary

Keep open. Current main still leaves Telegram native slash-command handlers outside the regular Telegram message feedback path, and the only linked implementation is an open, unmerged fork PR rather than an upstream fix.

## What This Changes

Request Telegram slash commands such as `/new` and `/reset` to emit the same ack reaction and typing indicator as regular inbound Telegram messages.

## Best Possible Solution

Add a Telegram-owned feedback helper after skip/auth checks for built-in and plugin native commands, thread the existing ack and typing dependencies from bot core, and cover it with focused Telegram tests plus changelog/docs notes.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available: normal messages build a feedback-capable context, while Telegram native slash commands execute in separate `bot.command(...)` handlers without feedback dependencies or calls.

## Solution Assessment

Yes. The narrowest maintainable direction is explicit Telegram slash-command feedback after authorization; routing commands through the full regular-message pipeline would couple command parsing and plugin-command execution to unrelated message dispatch behavior.
