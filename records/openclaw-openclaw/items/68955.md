## Summary

Keep open. Current main still keeps Telegram native slash commands outside the feedback-enabled regular message path, and the only linked implementation is an open, unmerged fork PR rather than an upstream fix.

## What This Changes

The issue requests Telegram slash commands such as `/new` and `/reset` to emit the same ack reaction and typing indicator as regular inbound Telegram messages.

## Best Possible Solution

Add a Telegram-owned feedback helper after skip/auth checks for built-in and plugin-backed native commands, reusing the existing ack scope gate and send-chat-action handler with focused tests and a changelog/docs note.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available: regular Telegram messages build a feedback-capable context, while native slash commands execute in separate handlers that neither receive nor invoke the feedback hooks.

## Solution Assessment

Yes. The narrowest maintainable fix is explicit Telegram command feedback after authorization; routing commands through the full regular-message pipeline would couple command parsing and plugin command execution to unrelated dispatch behavior.
