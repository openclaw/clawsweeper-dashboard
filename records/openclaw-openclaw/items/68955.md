## Summary

Keep open. Current main still wires Telegram ack reactions and typing cues through the regular inbound-message processor, while built-in and plugin native slash-command handlers remain on a separate path without those feedback dependencies or calls; the linked implementation is an open external fork PR, not a merged upstream fix.

## What This Changes

The issue requests Telegram slash commands such as /new and /reset to emit the same ack reaction and typing feedback as regular inbound messages.

## Best Possible Solution

Add a Telegram-owned post-auth slash-command feedback helper that reuses the existing ack gate and typing handler for built-in and plugin native commands, with focused coverage and live Telegram proof when a PR is reviewed.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is available: regular Telegram messages build feedback-capable context and call typing/ack paths, while native slash commands enter separate bot.command handlers with no feedback wiring.

## Solution Assessment

Yes. Explicit post-auth feedback in the Telegram native command path is the narrowest maintainable direction; routing commands through the full regular-message pipeline would conflict with the current fast-path native-command boundary.
