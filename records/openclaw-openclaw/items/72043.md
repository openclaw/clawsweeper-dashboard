## Summary

Current main has moved last-known-good repair out of Gateway startup/reload and into `openclaw doctor --fix`, so the reported automatic recovery notice no longer gets generated for the customer-facing path that motivated this config knob.

## What This Changes

Request to add `gateway.systemEvents.recoveryNotice` so operators can suppress Gateway recovery notices in prompt and outbound channel surfaces.

## Best Possible Solution

Keep the shipped fail-closed and doctor-owned recovery model; revisit a system-event policy only if automatic Gateway recovery notices are reintroduced later.

## Reproduction Assessment

No. On current main, the reported startup/reload last-known-good auto-recovery path no longer exists: startup throws invalid-config and hot reload skips invalid snapshots instead of restoring and notifying.

## Solution Assessment

Yes. Moving last-known-good restore to `openclaw doctor --fix` is now the narrower maintainable answer for the reported leak, because Gateway no longer creates the recovery notice that the proposed knob would gate.
