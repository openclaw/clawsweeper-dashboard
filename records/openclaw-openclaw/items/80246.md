## Summary

Keep open: this PR addresses a valid missing cron alert event-time feature and has sufficient Telegram proof, but the current patch mislabels cron jobs without `--tz` as UTC instead of using the existing gateway-host timezone fallback.

## What This Changes

The PR adds a shared cron alert timestamp formatter, threads `eventTimeMs` through cron failure alerts and webhook payloads, and updates focused cron/gateway tests.

## Best Possible Solution

Land the formatter/plumbing approach after it shares cron's existing schedule timezone resolution, keeping alert routing, schemas, and UTC fallback for non-cron or invalid-timezone cases unchanged.

## Reproduction Assessment

Yes for source-level behavior: current main alert builders omit event-time text while run metadata is available, and the PR's formatter fallback can be checked directly against the existing scheduler/docs timezone contract.

## Solution Assessment

No, not as written. The formatter-plus-plumbing shape is maintainable, but it must reuse cron's existing no-`tz` timezone fallback before the PR is mergeable.
