## Summary

Keep open: the PR addresses a real current-main gap, but the branch still labels cron jobs without `tz` as UTC even though the scheduler and docs use the gateway host timezone. The fix is narrow, but this draft is not merge-ready as written.

## What This Changes

The PR adds a cron alert event-time formatter and threads `eventTimeMs` through repeated failure, gateway failure-destination, webhook, and auto-disable notifications with focused tests.

## Best Possible Solution

Land a narrow version that reuses the existing cron timezone semantics, keeps routing and schema unchanged, and includes event-time text across repeated alerts, failure-destination/webhook alerts, and auto-disable notifications.

## Reproduction Assessment

Yes for the source-level gap: current main still builds the relevant alert bodies without event-time text while run metadata is available nearby. The PR blocker is also source-reproducible from the new formatter's UTC fallback versus the scheduler/docs host-timezone fallback.

## Solution Assessment

No, not as written. The formatter-plus-plumbing approach is the right shape, but missing cron `tz` must resolve the same way the scheduler does before the PR is mergeable.
