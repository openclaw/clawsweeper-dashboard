## Summary

Keep open: this draft PR addresses the right cron alert gap and has sufficient real behavior proof, but the current head still formats cron schedules without `tz` as UTC even though the scheduler and docs use the gateway host timezone.

## What This Changes

Adds a cron alert event-time formatter, threads `eventTimeMs` through cron failure alert delivery and webhook payloads, and updates cron/gateway tests for timestamped alert text.

## Best Possible Solution

Land one canonical implementation that shares cron schedule timezone resolution for alert formatting, keeps routing/schema changes additive, and resolves https://github.com/openclaw/openclaw/issues/77497.

## Reproduction Assessment

Yes for source-level behavior: current main alert builders omit event-time text while cron run metadata such as `runAtMs`/`startedAt` exists nearby. The PR blocker is also source-reproducible because the new formatter returns UTC for missing cron `tz` while scheduler/docs use the gateway host timezone.

## Solution Assessment

No, not as written. The formatter-plus-plumbing approach is the right shape, but it must reuse the cron scheduler timezone fallback before this PR is mergeable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
