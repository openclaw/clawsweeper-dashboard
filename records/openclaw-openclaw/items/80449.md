## What This Changes

The PR threads `idleTimedOut` through agent failover policy, assistant failover handling, and focused tests so LLM idle silence can rotate profiles, advance model fallback, or surface a timeout error.

## Best Possible Solution

Merge a narrow failover-policy fix that carries idle timeout through decision and retry bookkeeping, with focused tests plus redacted real after-fix gateway or log proof showing fallback advances.

## Reproduction Assessment

Yes for the source-level policy gap and reporter log pattern, but not for a live MiniMax gateway freeze in this review. Current main has idle-timeout state and timeout fallback contracts, while the reporter supplied current idle-timeout traces where configured fallback was not observed.

## Solution Assessment

Mostly yes. Threading `idleTimedOut` through the failover policy is a narrow maintainable direction, but the submitted patch should also preserve timeout retry bookkeeping and needs real after-fix runtime proof.
