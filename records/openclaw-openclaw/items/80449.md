## What This Changes

The PR threads `idleTimedOut` through assistant failover decisions, retry bookkeeping, timeout messaging, logging, and policy tests so model silence can rotate profiles or advance model fallback.

## Best Possible Solution

## Solution Assessment

Yes. Threading `idleTimedOut` through the existing failover decision and retry bookkeeping is the narrowest maintainable fix for the documented timeout fallback path, without adding a new retry policy or config surface.
