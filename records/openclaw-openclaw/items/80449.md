## Summary

Keep open. The PR is a plausible narrow bug fix for idle-timeout failover, but it still needs contributor after-fix real behavior proof and a small retry-reason cleanup before it should merge.

## What This Changes

The PR adds `idleTimedOut` to agent failover decisions so LLM silence can rotate auth profiles, advance model fallback, and surface timeout errors.

## Best Possible Solution

Rebase onto current main, preserve existing failover safeguards, carry idle timeout through both the decision and retry-reason bookkeeping, then require real after-fix gateway or redacted log proof before merge.

## Reproduction Assessment

Yes for the source-level failure mode: current main has idle-timeout state but the policy helper does not accept it directly, and reporter logs show idle timeouts not reaching the configured fallback. I did not reproduce a live MiniMax gateway freeze in this read-only review.

## Solution Assessment

Mostly yes. Routing `idleTimedOut` through the failover policy is narrow and matches the documented timeout fallback contract, but the retry-reason bookkeeping should also treat idle-only timeouts as timeout failures.
