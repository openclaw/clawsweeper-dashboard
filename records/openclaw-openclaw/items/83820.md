## Summary

Keep open: current main and the latest shipped release still rely on model-discretionary skill discovery, and this report raises a real product/API gap rather than a cleanup-close candidate. The related open slash-command injection issue is narrower and does not fully cover natural-language skill activation.

## What This Changes

The issue requests deterministic description or trigger-based skill activation when a natural-language user query clearly matches a skill.

## Best Possible Solution

Define a deliberate skill activation contract, likely around explicit trigger metadata or a bounded deterministic matcher, with docs and regression coverage for natural-language and explicit slash-command paths.

## Reproduction Assessment

No high-confidence live reproduction was established for the exact aihot/Kimi transcript. Source inspection does confirm the reported class of behavior: skills are exposed through prompt text and the model decides whether to read SKILL.md.

## Solution Assessment

Unclear as a code fix: hard triggers, hybrid scoring, and guaranteed injection are all plausible but change the public skill activation contract. Maintainers should choose the contract before a focused implementation PR.
