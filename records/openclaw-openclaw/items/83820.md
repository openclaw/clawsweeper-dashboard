## Summary

Keep open: current main still exposes skills to the model through description-based prompt text and does not provide deterministic natural-language trigger matching, while the linked slash-command issue covers only explicit invocation. The existing ClawSweeper review already identified the right boundary: this is a real skill activation contract gap that needs maintainer product/API design before implementation.

## What This Changes

The issue requests deterministic or hybrid skill activation so semantically matching natural-language queries load the intended skill instead of relying only on the model to choose from description metadata.

## Best Possible Solution

Define an explicit skill activation contract, likely trigger metadata or bounded deterministic matching layered on the existing description prompt, then document it and add regression coverage for natural-language discovery plus the separate slash-command path.

## Reproduction Assessment

No high-confidence live reproduction was established for the exact aihot/Kimi transcript. Source inspection does confirm that current main relies on model-discretionary skill selection from description prompt text, which can produce the reported class of failure.

## Solution Assessment

Unclear as a concrete patch because the suggested hard keywords, hybrid scoring, and guaranteed injection each change the public skill activation contract. The best next step is a maintainer design decision before a focused implementation PR.
