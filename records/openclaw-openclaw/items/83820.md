## Summary

Keep open: current main still exposes skills through prompt text and relies on the model to decide whether to read SKILL.md, so this report describes a real skill activation/product contract gap rather than something already implemented or stale. The linked slash-command injection issue covers an explicit invocation path, but it does not fully cover natural-language description matching for ClawHub skills.

## What This Changes

The issue requests more reliable natural-language skill activation when a user's query semantically matches a skill description, such as loading the ClawHub aihot skill for Chinese AI-news queries.

## Best Possible Solution

Define a deliberate skill activation contract, likely trigger metadata or a bounded matcher with docs and regression tests, while keeping explicit slash-command injection tracked in the narrower related issue.

## Reproduction Assessment

No high-confidence live reproduction was established for the exact aihot/Kimi transcript. Source inspection does show the reported class of behavior: skills are presented as description text and the model decides whether to read SKILL.md.

## Solution Assessment

Unclear as a specific fix: hard triggers, hybrid scoring, and guaranteed injection are all plausible, but natural-language skill activation changes a public skill contract and should be chosen deliberately before coding.
