## Summary

Keep open: the PR targets a real memory-core auto-selection failure mode, but the proposed heuristic still keys local-vs-remote behavior off adapter id instead of the adapter transport contract and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR adds a fallback auth/config-error heuristic in `extensions/memory-core/src/memory/embeddings.ts` so auto memory embedding provider selection continues when an adapter omits `shouldContinueAutoSelection`.

## Best Possible Solution

Use the adapter transport contract for the heuristic, add focused omitted-callback regression tests, and require redacted runtime proof before merge.

## Reproduction Assessment

Yes, from source inspection rather than a live run: current main fail-closes when an auto-selected adapter omits `shouldContinueAutoSelection`, before the FTS-only provider-null result can be returned.

## Solution Assessment

No as proposed. The fix direction is narrow, but the implementation should use the adapter `transport` contract, add omitted-callback regression coverage, and include real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate auth-error soft-skip on adapter transport:** `extensions/memory-core/src/memory/embeddings.ts:78`
  - body: This branch treats every adapter id except `local` as remote. A third-party adapter can declare `transport: "local"` with its own id; if it omits `shouldContinueAutoSelection`, an auth/config-looking setup error would now be swallowed instead of surfacing the local adapter failure. Gate the heuristic on `adapter.transport !== "local"`.
