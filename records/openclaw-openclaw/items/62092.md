## What This Changes

The PR adds a memory-core heuristic that treats auth/config-looking errors from auto-selected embedding adapters without explicit skip guidance as soft failures.

## Best Possible Solution

Use the adapter transport contract for the heuristic, add omitted-callback regression tests for remote and local adapters, and require redacted runtime proof before merge.

## Reproduction Assessment

Yes by source inspection. Current main returns false when an auto-selected adapter omits `shouldContinueAutoSelection`, so the catch path throws before the documented FTS-only fallback can be returned.

## Solution Assessment

No as proposed. The fix direction is narrow, but the local-vs-remote guard should use `adapter.transport`, and the PR needs omitted-callback tests plus redacted real-behavior proof.

- **[P2] Gate auth-error soft-skip on adapter transport:** `extensions/memory-core/src/memory/embeddings.ts:78`

Gate the heuristic on `adapter.transport !== "local"`.
