## Summary

Keep open. The reported main-branch behavior is real and the PR is close, but the new heuristic currently keys remote-ness off the adapter id instead of the public transport contract and still needs release-note coverage.

## What This Changes

The PR changes memory-core auto embedding provider selection so auth/config-looking errors from adapters without explicit auto-selection guidance are treated as soft failures.

## Best Possible Solution

Land the behavior after gating the heuristic with `adapter.transport !== "local"`, adding focused regression tests for omitted adapter guidance, and adding a one-line active changelog fix entry.

## Reproduction Assessment

Yes. Register an auto-select remote memory embedding adapter that omits `shouldContinueAutoSelection` and throws a missing-key error before a later viable adapter; current main throws because the helper returns `false` instead of continuing.

## Solution Assessment

No as proposed. The direction is narrow and maintainable, but the implementation should use the public `transport` contract rather than the adapter id, with regression coverage for remote omitted guidance and local adapters.
