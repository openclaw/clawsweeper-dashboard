## What This Changes

The PR adds a fallback heuristic in memory-core auto embedding provider selection so auth/config-looking errors from adapters without `shouldContinueAutoSelection` continue to the next candidate instead of aborting.

## Best Possible Solution

Use the adapter transport contract for the heuristic, add focused omitted-callback regression coverage, and require redacted runtime proof before merge.

## Reproduction Assessment

Yes. Source inspection shows current main returns false when an auto-selected adapter omits `shouldContinueAutoSelection`, then throws before the documented FTS-only fallback can be returned.

## Solution Assessment

No as proposed. The fix direction is narrow, but the heuristic should use the adapter transport contract and needs focused tests plus redacted real-behavior proof.
