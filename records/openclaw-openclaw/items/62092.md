## Summary

Keep open: the bug is source-reproducible and the PR is narrowly aimed, but it still lacks contributor real-behavior proof and the proposed id-based guard can misclassify third-party local embedding adapters.

## What This Changes

This PR adds a fallback heuristic so memory-core auto embedding provider selection treats auth/config-looking errors from adapters without `shouldContinueAutoSelection` as soft failures.

## Best Possible Solution

Use the adapter transport contract for the heuristic, add focused omitted-callback regression coverage, and require redacted runtime proof before merge.

## Reproduction Assessment

Yes, by source inspection. Current main returns false when an auto-selected adapter omits `shouldContinueAutoSelection`, then throws before the FTS-only no-provider fallback can be returned.

## Solution Assessment

No as proposed. The fix direction is narrow, but the branch should use the adapter `transport` contract and needs focused tests plus redacted real-behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate auth-error soft-skip on adapter transport:** `extensions/memory-core/src/memory/embeddings.ts:78`
  - body: This branch treats every adapter id except `local` as remote. A third-party adapter can declare `transport: "local"` with its own id; if it omits `shouldContinueAutoSelection`, an auth/config-looking setup error would now be swallowed instead of surfacing the local adapter failure. Gate the heuristic on `adapter.transport !== "local"`.
