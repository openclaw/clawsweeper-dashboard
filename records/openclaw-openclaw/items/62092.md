## Summary

Keep open: the bug-fix direction is useful and current main still hard-fails auto-select adapters that omit skip guidance, but this PR currently has a blocking transport/id predicate issue and lacks contributor-supplied real behavior proof.

## What This Changes

The PR changes memory-core auto embedding provider selection to soft-skip common auth/config-looking errors when an adapter omits explicit auto-selection guidance.

## Best Possible Solution

Land the behavior after changing the heuristic to use adapter.transport !== "local", adding focused omitted-guidance regression coverage, and collecting redacted runtime proof.

## Reproduction Assessment

Yes, from source inspection. Registering an auto-select adapter with no shouldContinueAutoSelection that throws a missing-key error makes current main return false and throw before the provider:null FTS-only fallback.

## Solution Assessment

No as proposed. The fix direction is narrow, but the soft-skip predicate should use the public transport contract rather than the adapter id.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Check adapter transport before soft-skipping auth errors:** `extensions/memory-core/src/memory/embeddings.ts:78`
  - body: This branch treats every adapter id except local as remote. A third-party adapter can declare transport: "local" with its own id; if it omits shouldContinueAutoSelection, an auth/config-looking setup error would now be swallowed instead of surfacing the local adapter failure. Gate the heuristic on adapter.transport !== "local".
  - confidence: 0.88
