## Summary

Keep this PR open because the reported auto-selection failure is still present on current main and the fix direction is useful, but the submitted patch should not merge as-is: it soft-skips based on adapter id instead of the public transport contract and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR changes `extensions/memory-core/src/memory/embeddings.ts` so auto provider selection soft-skips auth/config-looking errors when an embedding adapter omits `shouldContinueAutoSelection`.

## Best Possible Solution

Land the behavior after changing the heuristic to use `adapter.transport !== "local"`, adding focused omitted-guidance regression coverage, and collecting redacted runtime proof.

## Reproduction Assessment

Yes. From source inspection, registering an auto-select adapter with no `shouldContinueAutoSelection` that throws a missing-key error makes current main return false and throw before the `provider: null` FTS-only fallback.

## Solution Assessment

No as proposed. The fix direction is narrow and maintainable, but the predicate should use the adapter transport contract rather than the adapter id string.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Check adapter transport before soft-skipping auth errors:** `extensions/memory-core/src/memory/embeddings.ts:78`
  - body: This branch treats every adapter id except `local` as remote. A third-party adapter can declare `transport: "local"` with its own id; if it omits `shouldContinueAutoSelection`, an auth/config-looking setup error would now be swallowed instead of surfacing the local adapter failure. Gate the heuristic on `adapter.transport !== "local"`.
  - confidence: 0.88
