## Summary

Keep open: source inspection supports the reported memory-core auto-selection bug, but this PR still has a provider-routing defect and lacks required after-fix real behavior proof.

## What This Changes

The PR adds an auth/config-error heuristic so memory-core auto embedding provider selection can continue when an adapter omits `shouldContinueAutoSelection`.

## Best Possible Solution

Use the adapter `transport` contract for the heuristic, add omitted-callback regression tests, and merge only after redacted runtime proof shows memory search falling back to FTS-only.

## Reproduction Assessment

Yes by source inspection. Current main returns false when an auto-selected adapter omits `shouldContinueAutoSelection`, then throws before the documented FTS-only memory search path can be used.

## Solution Assessment

No as proposed. The fix direction is narrow, but it should gate on `adapter.transport !== "local"`, add focused omitted-callback tests, and include redacted after-fix runtime proof.

## Review Findings

Overall confidence: 0.88

Full review comments:

- **[P2] Gate auth-error soft-skip on adapter transport:** `extensions/memory-core/src/memory/embeddings.ts:78`
  - body: This branch treats every adapter id except `local` as remote. A third-party adapter can declare `transport: "local"` with its own id; if it omits `shouldContinueAutoSelection`, an auth/config-looking setup error would now be swallowed instead of surfacing the local adapter failure. Gate the heuristic on `adapter.transport !== "local"`.
