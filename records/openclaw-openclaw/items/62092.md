## Summary

Keep open. The PR addresses a real current-main memory-core gap, but the proposed fallback heuristic can hide auth/config-looking failures from third-party local adapters and needs focused regression coverage plus the required changelog entry before merge.

## What This Changes

The PR changes memory-core auto embedding provider selection so common auth/config errors from adapters without explicit auto-selection guidance are treated as soft failures.

## Best Possible Solution

Land the behavior after guarding the heuristic with `adapter.transport !== "local"`, adding regression tests for omitted adapter guidance, and adding a one-line Unreleased/Fixes changelog entry.

## Reproduction Assessment

Yes. On current main, register an auto-select remote memory embedding adapter that omits `shouldContinueAutoSelection` and throws a missing-key error before a later viable adapter; `createEmbeddingProvider({ provider: "auto" })` returns `false` from the helper and throws before lexical fallback.

## Solution Assessment

No as proposed. The direction is the narrow maintainable fix, but the implementation should use the public `transport` contract rather than the adapter id and should be covered by focused tests.
