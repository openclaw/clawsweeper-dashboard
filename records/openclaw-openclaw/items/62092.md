## Summary

Keep open: current main still has the source-reproducible memory-core auto-selection failure, but this PR needs real behavior proof and a small correctness fix before it is merge-ready.

## What This Changes

The PR adds an auth/config-error heuristic so memory-core auto embedding provider selection can continue when an adapter omits `shouldContinueAutoSelection`.

## Best Possible Solution

Use the adapter transport contract for the heuristic, add focused omitted-callback regression coverage, and collect redacted runtime proof before merge.

## Reproduction Assessment

Yes, source-reproducible: on current main, an auto-selected adapter that omits `shouldContinueAutoSelection` and throws a missing-key/config error returns false from the helper and throws before the FTS-only fallback. I did not run tests because this was a read-only review.

## Solution Assessment

No as proposed. The fix direction is narrow, but the predicate should use `adapter.transport !== "local"` and include regression coverage for omitted guidance.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate auth-error soft-skip on adapter transport:** `extensions/memory-core/src/memory/embeddings.ts:78`
  - body: This branch treats every adapter id except `local` as remote. A third-party adapter can declare `transport: "local"` with its own id; if it omits `shouldContinueAutoSelection`, an auth/config-looking setup error would now be swallowed instead of surfacing the local adapter failure. Gate the heuristic on `adapter.transport !== "local"`.
  - confidence: 0.88
