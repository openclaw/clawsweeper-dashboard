## Summary

Keep open. The PR addresses a real current-main auto-selection abort, but the proposed heuristic should be tightened before merge and the user-facing fix needs a changelog entry.

## What This Changes

The branch changes memory-core auto embedding provider selection so common auth/config errors from adapters without explicit guidance are treated as soft failures.

## Best Possible Solution

Land the behavior after switching the heuristic to the public `transport` contract, adding focused regression coverage for omitted adapter guidance, and documenting the user-facing fix in the active changelog.

## Reproduction Assessment

Yes. Register an auto-select remote memory embedding adapter that omits `shouldContinueAutoSelection` and throws a missing-key error before a later viable adapter; current main returns `false` from the helper and throws before reaching the FTS-only result path.

## Solution Assessment

No as proposed. The direction is narrow and maintainable, but the predicate should use `adapter.transport !== "local"` and the PR should include regression tests plus a changelog entry.
