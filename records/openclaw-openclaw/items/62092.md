## Summary

Keep open: the bug and fix direction are source-backed, but the patch should not merge yet because it keys the heuristic on adapter id instead of transport and lacks real behavior proof and focused omitted-callback tests.

## What This Changes

The PR changes memory-core auto embedding provider selection to treat auth/config-looking errors from adapters without `shouldContinueAutoSelection` as skippable soft failures.

## Best Possible Solution

Use the adapter transport contract for the heuristic, add focused omitted-callback regression coverage, and require redacted runtime proof before merge.

## Reproduction Assessment

Yes, from source inspection. Current main throws when an auto-selected adapter omits `shouldContinueAutoSelection`, and the documented FTS-only fallback only happens after skippable provider failures are exhausted.

## Solution Assessment

No as proposed. The fix direction is narrow, but the predicate should use `adapter.transport !== "local"`, and the PR still needs omitted-callback tests plus redacted real-behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate auth-error soft-skip on adapter transport:** `extensions/memory-core/src/memory/embeddings.ts:78`
  - body: This branch treats every adapter id except `local` as remote. A third-party adapter can declare `transport: "local"` with its own id; if it omits `shouldContinueAutoSelection`, an auth/config-looking setup error would now be swallowed instead of surfacing the local adapter failure. Gate the heuristic on `adapter.transport !== "local"`.
