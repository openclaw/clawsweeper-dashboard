## Summary

Keep open: the PR addresses a real source-reproducible memory-core fallback bug, but the current patch can swallow errors for third-party local adapters and still lacks contributor-supplied real behavior proof.

## What This Changes

The PR adds a memory-core heuristic that soft-skips common auth/config failures during auto embedding provider selection when an adapter omits explicit skip guidance.

## Best Possible Solution

Land the behavior after changing the heuristic to use `adapter.transport !== "local"`, adding focused regression coverage for omitted adapter guidance, and collecting redacted runtime proof.

## Reproduction Assessment

Yes, from source inspection. On current main, an auto-select adapter that omits `shouldContinueAutoSelection` and throws a missing-key error makes the helper return false, so `createEmbeddingProvider` throws before reaching the FTS-only fallback result.

## Solution Assessment

No as proposed. The direction is narrow and maintainable, but the soft-skip predicate should use the public transport contract rather than the adapter id.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Check adapter transport before soft-skipping auth errors:** `extensions/memory-core/src/memory/embeddings.ts:78`
  - body: This branch treats every adapter id except `local` as remote. A third-party adapter can declare `transport: "local"` with its own id; if it omits `shouldContinueAutoSelection`, an auth/config-looking setup error would now be swallowed instead of surfacing the local adapter failure. Gate the heuristic on `adapter.transport !== "local"`.
  - confidence: 0.88
