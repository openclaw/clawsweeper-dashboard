## Summary

Keep open: the PR targets a real memory-core auto-selection bug, but the proposed patch still has a correctness blocker and the external PR lacks after-fix real behavior proof.

## What This Changes

The PR changes memory-core auto embedding provider selection to treat common auth/config errors as soft failures when an adapter omits explicit auto-selection guidance.

## Best Possible Solution

Land the behavior after changing the heuristic to use the adapter transport contract, adding focused regression coverage for omitted adapter guidance, and collecting redacted runtime proof.

## Reproduction Assessment

Yes, from source inspection. On current main, an auto-select adapter that omits shouldContinueAutoSelection and throws a missing-key/config error makes the helper return false, so createEmbeddingProvider throws before the FTS-only fallback result.

## Solution Assessment

No as proposed. The direction is narrow, but the soft-skip predicate should use adapter.transport !== "local" instead of adapter.id !== "local".

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Check adapter transport before soft-skipping auth errors:** `extensions/memory-core/src/memory/embeddings.ts:78`
  - body: This branch treats every adapter id except `local` as remote. A third-party adapter can declare `transport: "local"` with its own id; if it omits `shouldContinueAutoSelection`, auth-looking setup errors would now be swallowed during auto-selection instead of surfacing the local adapter failure. Gate the heuristic on `adapter.transport !== "local"`.
  - confidence: 0.88
