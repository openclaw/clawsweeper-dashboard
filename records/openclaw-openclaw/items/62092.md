## Summary

Keep this PR open: it targets a real memory-core fallback problem that still exists on current main, but it is not merge-ready because the proposed heuristic uses adapter id instead of the transport contract and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR changes memory-core auto embedding provider selection so auth/config-looking failures from adapters without explicit skip guidance continue selection instead of aborting.

## Best Possible Solution

Use the adapter transport contract for the heuristic, add omitted-callback regression tests for remote and local adapters, and require redacted runtime proof before merge.

## Reproduction Assessment

Yes by source inspection. Current main returns false when an auto-selected adapter omits `shouldContinueAutoSelection`, so the catch path throws before the documented FTS-only fallback can be returned.

## Solution Assessment

No as proposed. The fix direction is narrow, but the local-vs-remote guard should use `adapter.transport`, and the PR needs omitted-callback tests plus redacted real-behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate auth-error soft-skip on adapter transport:** `extensions/memory-core/src/memory/embeddings.ts:78`
  - body: This branch treats every adapter id except `local` as remote. A third-party adapter can declare `transport: "local"` with its own id; if it omits `shouldContinueAutoSelection`, an auth/config-looking setup error would now be swallowed instead of surfacing the local adapter failure. Gate the heuristic on `adapter.transport !== "local"`.
