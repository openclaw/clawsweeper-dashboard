## Summary

Keep open: the PR targets a real memory-core provider-selection bug, but the current patch is not merge-ready because the heuristic is keyed on adapter id instead of the transport contract and the external contributor has not provided real behavior proof.

## What This Changes

The PR adds an auth/config error heuristic to memory-core auto embedding provider selection for adapters that omit `shouldContinueAutoSelection`.

## Best Possible Solution

Use the adapter transport contract for the heuristic, add omitted-callback regression coverage for remote and local adapters, and land only after redacted runtime proof shows FTS-only fallback behavior.

## Reproduction Assessment

Yes by source inspection. Current main returns false when an auto-selected adapter omits `shouldContinueAutoSelection`, then throws before the documented FTS-only fallback can be returned.

## Solution Assessment

No as proposed. The fix direction is narrow, but the heuristic should use `adapter.transport !== "local"` and needs focused tests plus redacted real-behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate auth-error soft-skip on adapter transport:** `extensions/memory-core/src/memory/embeddings.ts:78`
  - body: This branch treats every adapter id except `local` as remote. A third-party adapter can declare `transport: "local"` with its own id; if it omits `shouldContinueAutoSelection`, an auth/config-looking setup error would now be swallowed instead of surfacing the local adapter failure. Gate the heuristic on `adapter.transport !== "local"`.
  - confidence: 0.88
