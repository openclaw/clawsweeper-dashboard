## Summary

Keep this PR open: the reported current-main bug is source-reproducible and the fix direction is useful, but the proposed id-based heuristic can misclassify third-party local adapters and the external contributor still has not provided after-fix real behavior proof.

## What This Changes

The PR changes memory-core auto embedding provider selection to continue after auth/config-looking errors when an adapter omits `shouldContinueAutoSelection`.

## Best Possible Solution

Use the adapter transport contract for the heuristic, add focused omitted-callback regression coverage, and require redacted runtime proof before merge.

## Reproduction Assessment

Yes, by source inspection. Current main returns false when an auto-selected adapter omits `shouldContinueAutoSelection`, then throws before the documented FTS-only fallback can be returned.

## Solution Assessment

No as proposed. The fix direction is narrow, but the heuristic should use the adapter transport contract and needs focused tests plus redacted real-behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate auth-error soft-skip on adapter transport:** `extensions/memory-core/src/memory/embeddings.ts:78`
  - body: This branch treats every adapter id except `local` as remote. A third-party adapter can declare `transport: "local"` with its own id; if it omits `shouldContinueAutoSelection`, an auth/config-looking setup error would now be swallowed instead of surfacing the local adapter failure. Gate the heuristic on `adapter.transport !== "local"`.
  - confidence: 0.88
