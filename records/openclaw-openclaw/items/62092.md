## Summary

Keep open. Current main still aborts auto provider selection unless an adapter explicitly opts into continuing, so this PR addresses a real memory-core gap; however the proposed fallback should use the adapter transport contract before it is safe to merge.

## What This Changes

The PR changes memory-core auto embedding provider selection to treat common auth/config errors from adapters without explicit auto-selection guidance as soft failures.

## Best Possible Solution

Land the memory-core behavior after changing the heuristic to exclude any `transport: "local"` adapter, preserving explicit adapter guidance, and add focused regression coverage plus the required Unreleased/Fixes changelog entry.

## Reproduction Assessment

Yes. On current main, register an auto-select remote memory embedding adapter that omits `shouldContinueAutoSelection` and throws a missing-key error before a later viable provider; `createEmbeddingProvider({ provider: "auto" })` will throw because the helper returns `false`.

## Solution Assessment

No as proposed. The direction is the narrow maintainable fix, but the implementation should guard on `adapter.transport !== "local"` and add tests so third-party local adapters are not silently skipped.
