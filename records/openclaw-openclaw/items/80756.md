## Summary

Keep this issue open: current main and the latest tagged release still lack a silent `registerCommand` result contract, and the open linked PR is the canonical implementation path with Telegram proof attached.

## What This Changes

Request a `registerCommand` handler result such as `suppressReply` or `handled` so Telegram plugin commands that deliver their own response do not also trigger the empty-response fallback.

## Best Possible Solution

Review and either land or explicitly reject https://github.com/openclaw/openclaw/pull/80928 with documented SDK semantics, focused Telegram regression coverage, and the attached Telegram proof, then close this issue after that decision.

## Reproduction Assessment

Yes. Source inspection plus the existing Telegram test show undefined command results become `{}` and then the Telegram adapter sends the empty-response fallback; I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Mostly yes. An explicit silent command-result flag is the narrow maintainable direction, but the public SDK scope should be settled through the already-open canonical PR rather than closing this issue now.
