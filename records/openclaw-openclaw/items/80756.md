## Summary

Keep this issue open: current main still lacks a silent `registerCommand` result contract, and the linked closing PR is open rather than merged. Closing now would lose the issue-to-fix link before the SDK/Telegram change lands or is explicitly rejected.

## What This Changes

Add a `registerCommand` result such as `suppressReply` or `handled` so Telegram plugin commands that deliver through their own transport can suppress the empty-response fallback.

## Best Possible Solution

Review and land or explicitly reject https://github.com/openclaw/openclaw/pull/80928 with SDK semantics, docs/tests, and Telegram proof, then close this issue after that decision.

## Reproduction Assessment

Yes. Current source converts undefined plugin command results to `{}` and Telegram substitutes the empty-response fallback; an existing test asserts that path, and the linked PR includes Mantis Telegram Desktop before/after proof on the same flow.

## Solution Assessment

Mostly yes. An explicit silent command-result flag is the narrow maintainable direction, but it expands public SDK behavior, so the linked PR needs maintainer approval on naming and channel scope before this issue closes.
