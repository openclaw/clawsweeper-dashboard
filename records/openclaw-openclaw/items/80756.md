## Summary

Keep open: current main still has no command-result contract for out-of-band plugin delivery, Telegram still maps non-renderable command results to the empty fallback, and an open implementation PR is already linked as the closing path.

## What This Changes

Add a first-class `registerCommand` result such as `suppressReply` or `handled` so Telegram plugin commands that send their own response can avoid the empty-response fallback.

## Best Possible Solution

Review and land or reject the linked implementation PR; the issue should close only after the accepted command-result contract ships on `main`.

## Reproduction Assessment

Yes, source-reproducible on current main: undefined handler results become `{}`, and Telegram native commands replace non-renderable results with the empty-response fallback. I did not run a live Telegram/Windows session during this read-only review.

## Solution Assessment

Unclear: an explicit silent result flag is a narrow plausible fix, but maintainers still need to settle the public SDK naming and whether the semantics are Telegram-only or generic across native command adapters.
