## Summary

Keep open: current main still lacks a silent plugin-command result contract, the Telegram native command path still falls back for non-renderable command results, and an open linked PR already owns the proposed implementation path.

## What This Changes

Add a command handler result such as `suppressReply` or `handled` so Telegram plugin commands that already sent their own response can skip the extra fallback reply.

## Best Possible Solution

Land a documented, backward-compatible silent command-result contract through the linked implementation PR or an equivalent maintainer-approved fix, then close this issue after it reaches main.

## Reproduction Assessment

Yes. Current source shows an undefined handler result becomes `{}`, and Telegram native command delivery maps that non-renderable result to the empty-response fallback; I did not run a live Telegram/Windows reproduction.

## Solution Assessment

Unclear. A `suppressReply` flag is a narrow plausible fix, but maintainers still need to accept the public SDK contract scope and review the already-open implementation PR before this issue is resolved.
