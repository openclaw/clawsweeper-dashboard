This should stay open: current main still has no `registerCommand` result that means “handled with no OpenClaw reply,” and the linked open implementation PR explicitly closes this issue. Closing the issue before https://github.com/openclaw/openclaw/pull/80928 merges would drop the canonical tracking item.

## What This Changes

The issue requests a silent command-handler result so Telegram plugin commands that send their own response can suppress the fallback reply.

## Best Possible Solution
