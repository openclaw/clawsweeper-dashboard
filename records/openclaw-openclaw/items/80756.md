## Summary

Keep open: current main and the latest release still lack a silent `registerCommand` result, and the linked closing PR is the active implementation path rather than a reason to close this issue before merge.

## What This Changes

Request a first-class `registerCommand` result such as `suppressReply` or `handled` so Telegram plugins that send their own response do not receive an extra empty-response fallback.

## Best Possible Solution

Review and land or explicitly reject https://github.com/openclaw/openclaw/pull/80928 with the SDK semantics documented, then close this issue after that decision.

## Reproduction Assessment

Yes. Current main source shows undefined command results become `{}` and Telegram sends the empty-response fallback; existing test coverage asserts that path, and linked Mantis proof reproduced the visible Telegram symptom on the PR baseline.

## Solution Assessment

Mostly yes. An explicit `suppressReply` result is a narrow maintainable contract, but maintainers still need to approve its public SDK/channel scope before landing the linked PR.
