## Summary

Keep open: current main and the latest release still lack a `registerCommand` suppression result, and the linked open implementation PR is the correct place to finish the fix rather than closing this issue early.

## What This Changes

Plugin command handlers need a first-class silent result to suppress Telegram's empty-response fallback after the plugin has delivered its own reply.

## Best Possible Solution

Land a narrow command-result suppression contract and Telegram adapter branch if maintainers accept the public API shape; otherwise choose a different explicit suppression seam and update the open implementation PR accordingly.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: return `undefined` from a Telegram plugin command handler, which becomes `{}` and then falls through to `EMPTY_RESPONSE_FALLBACK`. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

Unclear until maintainers choose the API shape. An explicit suppression result is the narrow maintainable direction, but the open PR still needs maintainer approval for putting `suppressReply` on the exported `PluginCommandResult` contract.
