## Summary

Keep this open: current main and v2026.5.12 still lack a silent plugin command result, and Telegram native commands still turn empty plugin command results into the fallback message. The open linked PR is the active implementation candidate, so the issue should stay open until that PR lands or is rejected.

## What This Changes

Add a plugin command result such as `suppressReply` or `handled` so a handler that sends through its own Telegram transport can suppress the empty-response fallback.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80928, or an equivalent maintainer-approved PR, with a documented silent command-result contract, Telegram adapter handling, regression coverage, and accepted Telegram proof.

## Reproduction Assessment

Yes. Current main shows `executePluginCommand` normalizes undefined to `{}`, Telegram native commands convert that non-renderable result to the fallback, and a focused test asserts the fallback behavior.

## Solution Assessment

Unclear. A silent command-result flag is a narrow plausible fix, but maintainers should choose the public SDK name and channel scope; the open linked PR is the right vehicle for that decision.
