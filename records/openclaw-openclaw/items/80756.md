## Summary

Keep this issue open: current main still has no silent `registerCommand` result contract, and an open linked PR is already the canonical implementation path. The issue should close only after that PR lands or maintainers explicitly decide not to add this SDK/channel behavior.

## What This Changes

The issue requests a `registerCommand` result such as `suppressReply` or `handled` so Telegram plugin commands that send their own response can avoid the fallback reply.

## Best Possible Solution

Review and land or explicitly reject https://github.com/openclaw/openclaw/pull/80928 with SDK docs, focused tests, and Telegram proof, then close this issue after that decision.

## Reproduction Assessment

Yes. Current source shows undefined command results become `{}` and Telegram replaces non-renderable results with the empty-response fallback; an existing test asserts that path, though I did not run live Telegram in this read-only review.

## Solution Assessment

Mostly yes. An explicit silent command-result flag is a narrow solution, but maintainers still need to settle the public SDK scope and land the already-open canonical PR rather than closing the issue now.
