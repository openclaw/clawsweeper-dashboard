## What This Changes

The issue requests a silent plugin command result so Telegram handlers that deliver replies themselves can suppress the empty-response fallback.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80928 or an equivalent documented SDK/Telegram change that adds a terminal silent result, preserves focused tests, and records real Telegram proof.
