## Summary

Keep open. Current main and the latest release still silently rewrite the generated macOS LaunchAgent env wrapper, and the active closing fix is the still-open PR at https://github.com/openclaw/openclaw/pull/90537.

## What This Changes

The issue asks OpenClaw to warn when reinstall/update/doctor flows overwrite customized generated gateway env wrappers and to clarify status diagnostics for secret-provider wrapper setups.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/90537, keeping `--wrapper` / `OPENCLAW_WRAPPER` as the durable operator path and splitting any remaining auth-profile migration problem into a narrower issue.

## Solution Assessment

Yes. The best fix is the focused diagnostics path in https://github.com/openclaw/openclaw/pull/90537: detect customized generated wrappers at the launchd write site and clarify status output without adding provider-specific support or new config.
