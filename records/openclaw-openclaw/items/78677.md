## Summary

Keep open: current main still routes remote inbound media through a hardcoded `/usr/bin/scp`, and the only adjacent open iMessage staging PR preserves the remote raw-path contract rather than fixing SCP binary resolution.

## What This Changes

Remote inbound media staging should invoke SCP portably instead of hardcoding `/usr/bin/scp` when `MediaRemoteHost` is set.

## Best Possible Solution

Resolve the SCP executable portably in the remote staging helper while preserving the current sanitized argument contract and add targeted regression coverage for the spawned command.

## Reproduction Assessment

Yes from source inspection: a remote attachment with `MediaRemoteHost` reaches `scpFile`, which calls `spawn('/usr/bin/scp', ...)`. On Windows or any host without that absolute path, Node will fail before staging even when `scp` is available on `PATH`.

## Solution Assessment

Yes: a narrow portable resolver or platform-aware `scp` command in `stage-sandbox-media.ts` is the maintainable fix. The safer path is to keep the existing no-shell spawn, normalized host/path, strict host-key checking, and allowed-root policy unchanged.
