## Summary

Current main still contains the hardcoded SCP executable path, so this is a valid open bug; it should stay open while the linked fix PR is reviewed and either merged or replaced.

## What This Changes

Remote inbound media staging should resolve the `scp` executable portably instead of spawning `/usr/bin/scp`.

## Best Possible Solution

Land a narrow fix that resolves SCP through a platform-aware trusted resolver or gives a clear preflight error, with regression coverage for Windows/OpenSSH and non-FHS layouts.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: set `MediaRemoteHost`, pass an allowed remote media path, and current main reaches `spawn("/usr/bin/scp", ...)`; a host without that absolute binary will fail before staging.

## Solution Assessment

Unclear for the linked PR's exact approach. The issue direction is correct, but the safest final implementation should reconcile portability with the existing trusted binary-resolution policy instead of only relying on ambient PATH if that widens command-resolution trust.
