Keep open: current main still lacks the requested always-visible Control UI topbar display for active model plus token/context usage. The data exists in adjacent Debug, Usage, chat, CLI, and docs surfaces, but the topbar remains navigation/search plus chat mobile controls and theme mode, so this is still a product/UI placement decision rather than cleanup to close.

## What This Changes

The issue requests an always-visible Control UI topbar status area showing the active model and token or context usage.

## Best Possible Solution

Keep this open until maintainers either add a focused topbar status component or explicitly decline the placement; the component should derive the active model from session/default model state, derive fresh context/usage from session usage data instead of debug-only polling, and handle responsive and stale/unavailable states.

## Solution Assessment
