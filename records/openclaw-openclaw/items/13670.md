Keep open: current main still does not provide the requested always-visible Control UI topbar model and token/context status. The data exists in session, chat, Usage, Debug, CLI, and TUI-adjacent surfaces, but the topbar remains navigation/search plus existing controls, and the recent maintainer comment already supports keeping this as follow-up work.

## What This Changes

The issue requests an always-visible Control UI topbar status area showing the active model and token/context usage.

## Best Possible Solution

Add a focused, responsive Control UI topbar status component if maintainers accept that placement.
