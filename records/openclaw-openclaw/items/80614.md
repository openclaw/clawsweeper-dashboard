## Summary

Keep open: the status cold-path direction is source-backed, but the PR head is not merge-ready because it regresses shared session-store aggregation and the added task-health protocol surface is not kept in sync with generated/native client docs. The branch is also a draft and needs complete real behavior proof for the user-visible task-health changes.

## What This Changes

The PR changes status session/model rendering to avoid manifest and CLI-provider cold-path scans, reuses loaded session rows, and also adds stale task health to CLI, Gateway protocol, SDK types, and docs.

## Best Possible Solution

Land a scoped status cold-path fix that rebuilds global rows once per unique store without overrides; either split task-health or update all protocol/client/docs artifacts with real output proof.

## Reproduction Assessment

Yes, source-backed: current main still lets status model parsing enter manifest normalization with only plugin normalization disabled, and the PR body plus an independent comment provide 2026.5.7 timing/microcheck proof. I did not run a live timing reproduction in this read-only review.

## Solution Assessment

No, not as submitted: the status fast-path pieces are the right shape, but the global session aggregation change breaks shared stores and the task-health protocol addition is incomplete. The safer solution is to repair aggregation and keep protocol/docs/generated clients synchronized or split that feature out.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Deduplicate shared session stores:** `src/commands/status.summary.ts:264`
  - body: When `session.store` lacks `{agentId}`, every agent resolves to the same store file. Appending rows already built with `agentIdOverride` makes global `sessions.count` and `sessions.recent` include each stored session once per configured agent, with shared rows stamped as the current agent. Build the global rows once per unique store without the override, or dedupe by store/key.
  - confidence: 0.92

  - body: Adding `health` to the public `TaskSummary` schema makes the Gateway return a new task field, but the generated Swift `TaskSummary` model and public protocol docs still omit it. Regenerate/update the native protocol artifact and protocol docs with this schema change, or split the task-health surface out of this PR.
  - confidence: 0.86
