## Summary

Keep open: the status cold-path fix addresses a source-backed performance bug, but the PR still introduces a shared-session-store aggregation regression and mixes in a separate task-health API/docs feature.

## What This Changes

The PR threads status-only manifest-normalization controls through model resolution, adds status-scope CLI-provider caching, reuses session rows for global status sessions, and adds stale task health fields to CLI, gateway, SDK, and docs surfaces.

## Best Possible Solution

Land the status cold-path optimization after repairing shared-store global aggregation and coordinating or splitting the overlapping task-health and `isCliProvider` changes.

## Reproduction Assessment

Yes, source-backed. Current main still has the manifest-normalization and per-row CLI-provider paths described by the PR, and the PR body plus an independent comment provide before/after timing data from affected installs.

## Solution Assessment

No, not as submitted. The manifest-normalization direction is narrow, but the global-session reuse change should rebuild rows once per unique store without `agentIdOverride`, and the task-health surface should be explicitly accepted or split.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
