## Summary

Keep open: the PR targets a valid `openclaw status` cold-path performance bug and includes after-fix timing proof, but the session-row reuse hunk would duplicate and inflate global recent sessions for supported shared `session.store` configurations. The useful fix should keep the shallow status path while building global rows once per unique store without per-agent overrides.

## What This Changes

The PR threads `allowManifestNormalization` through persisted model resolvers, skips manifest normalization in status summary model parsing, caches CLI-provider detection for status rendering, reuses session rows, and adds focused status summary tests.

## Best Possible Solution

Land the manifest/CLI-provider cold-path improvements after replacing the global recent-session reuse with a unique-store/no-override row cache and adding shared-store regression coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-backed yes: the linked report and PR body give timing commands/profiles, and current code shows the status summary paths that re-enter manifest normalization; I did not run the live timing command in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? Not yet: the manifest-normalization and CLI-provider caching direction is narrow, but the global session reuse should cache rows per unique store without `agentIdOverride` so shared stores remain correct.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep shared session stores deduplicated:** `src/commands/status.summary.ts:264`
  - body: When `session.store` omits `{agentId}`, multiple agents resolve to the same `sessions.json`. This line appends the per-agent row list for every agent into the global list; because `buildSessionRows(..., { agentIdOverride })` returns every entry in the store with the current agent id, a shared store containing `agent:main:*` and `agent:ops:*` will make `sessions.count` and `sessions.recent` show each stored session once per agent. Cache/build the global rows once per unique store without the override, or dedupe by store/key.
  - confidence: 0.9
