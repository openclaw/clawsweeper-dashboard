## Summary

Keep open: current main still lets a positive isolated-cron `timeoutSeconds` become the embedded model idle watchdog, so the reported 120s idle-timeout failure can still occur even though cron runs without an explicit timeout now disable the default watchdog.

## What This Changes

The issue asks OpenClaw to restore isolated cron runs with `openai/gpt-4o-mini` after a regression where cron `timeoutSeconds` does not prevent model idle-timeout failures and the error points at confusing provider timeout configuration.

## Best Possible Solution

Separate cron `timeoutSeconds` from model stream idle timing: keep it as the outer job runtime limit, and use configured provider request timeout for the model idle/HTTP guard with focused regression coverage.

## Reproduction Assessment

Yes, at source level. A positive isolated cron `payload.timeoutSeconds` flows into `runEmbeddedPiAgent` as `timeoutMs`, then into `resolveLlmIdleTimeoutMs` as `runTimeoutMs`, where it is capped into the model idle watchdog before the cron-disable branch runs.

## Solution Assessment

No, current main is not the best solution yet. The narrow maintainable fix is to distinguish cron outer runtime timeout from model idle timeout while preserving provider-level `models.providers.*.timeoutSeconds` behavior.

## Repair Work Prompt

Fix the isolated cron model idle-timeout regression described at https://github.com/openclaw/openclaw/issues/76331. On current main, `payload.timeoutSeconds` is resolved as the cron outer runtime timeout and passed to `runEmbeddedPiAgent`; `attempt.ts` then forwards it as `runTimeoutMs` to `resolveLlmIdleTimeoutMs`, which caps it at the default 120s model idle watchdog before the cron-disable branch. Keep cron `timeoutSeconds` as the outer job limit only; only configured provider request timeout or existing non-cron agent timeout semantics should set the model idle watchdog. Add regression coverage for a cron run with positive `payload.timeoutSeconds` not enabling the implicit idle watchdog while preserving provider `models.providers.*.timeoutSeconds` and non-cron behavior. Add a single-line `CHANGELOG.md` fix entry with appropriate human reporter credit; do not change auth profile structure, provider catalogs, or unrelated cron model-selection behavior.
