## Summary

Keep open: current main still has the isolated-cron timeout path, and the related closed PR only targeted the confusing hint text rather than the runtime timeout behavior.

## What This Changes

The issue reports isolated cron sessions timing out with `openai/gpt-4o-mini` after upgrading from 2026.4.24 to 2026.4.26, plus confusing timeout-config guidance.

## Best Possible Solution

Keep cron `timeoutSeconds` as the outer wall-clock deadline and prevent it from selecting the model idle watchdog, with the timeout-hint cleanup handled as a companion fix.

## Reproduction Assessment

Yes, at source level. A positive isolated-cron `payload.timeoutSeconds` reaches `runEmbeddedPiAgent` as `timeoutMs`, then `resolveLlmIdleTimeoutMs` converts it to a capped idle watchdog before the cron no-default-watchdog branch can run.

## Solution Assessment

No. The closed related PR only improves the error hint; the narrow maintainable fix is to separate cron’s outer runtime timeout from model idle-timeout selection.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/76331. Current main lets a positive isolated cron `payload.timeoutSeconds` flow through `src/cron/isolated-agent/run.ts` and `run-executor.ts` into `runEmbeddedPiAgent` as `timeoutMs`; `src/agents/pi-embedded-runner/run/attempt.ts` then passes it to `resolveLlmIdleTimeoutMs`, causing the model idle watchdog to cap at the default before the cron trigger disables the default watchdog. Keep cron `timeoutSeconds` as the cron-owned outer wall-clock deadline and avoid using that cron timeout as model idle-timeout input. Add regression coverage for an isolated cron agent-turn with positive `timeoutSeconds` so the embedded run still uses cron outer cancellation without enabling the default model idle watchdog. Include or preserve the provider-id timeout hint improvement from closed #76422 only as a companion fix if it stays narrow. Do not change provider `models.providers.*.timeoutSeconds` semantics, global `agents.defaults.timeoutSeconds`, or non-cron embedded-run timeout behavior. Add a user-facing changelog entry for the bug fix.
