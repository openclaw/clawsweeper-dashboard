## What This Changes

Requests a new opt-in Matrix `threadBindings.bypassMentionInBoundThreads` setting so in-thread replies with active per-thread session bindings can bypass `requireMention`.

## Best Possible Solution

## Solution Assessment

Unclear until maintainer review of the linked PR. An opt-in default-false setting is the safest direction because current tests intentionally pin the existing mention-gating contract.
