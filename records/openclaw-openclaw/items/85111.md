## What This Changes

Request an opt-in Matrix `threadBindings` setting that lets replies inside an already-bound thread bypass `requireMention` without re-mentioning the bot.

## Best Possible Solution

Review and refresh https://github.com/openclaw/openclaw/pull/85112, then land a default-off Matrix thread-continuation policy only if maintainers accept the config contract.

## Reproduction Assessment

Yes, at source level. Current main resolves Matrix thread/session routing, then drops unmentioned room replies on the `no-mention` path, and an adjacent test pins that bound thread replies without a mention do not dispatch or touch the binding.

## Solution Assessment

Unclear until maintainer review completes. A default-off opt-in looks safer than an unconditional bypass because it preserves current mention gating for existing users, but the new config surface needs explicit product acceptance.
