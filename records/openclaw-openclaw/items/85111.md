## Summary

Keep open. Current main still enforces Matrix `requireMention` for unmentioned in-thread replies even after resolving a runtime thread binding, and an open linked PR is already the implementation path, so this issue should stay open until that PR lands or is rejected.

## What This Changes

Request an opt-in Matrix config flag that lets unmentioned replies inside actively bound Matrix threads bypass room-level `requireMention`.

## Best Possible Solution

Review the linked Matrix PR and either land a default-off, documented opt-in with regression coverage, or close that PR and leave strict mention gating unchanged.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a room message with `requireMention: true`, no mention, an in-thread relation, and a runtime binding reaches the mention gate and is dropped because only control commands bypass it. I did not run a live Matrix repro in this read-only review.

## Solution Assessment

Unclear until the linked PR is reviewed, but the proposed default-off opt-in is the safer solution shape than changing all Matrix rooms. It preserves the current strict mention contract while giving operators an explicit thread-conversation mode.
