## What This Changes

The issue requests a new Matrix `threadBindings.bypassMentionInBoundThreads` opt-in that would let replies inside already-bound Matrix threads bypass room-level `requireMention`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/85112 or a maintainer-approved variant that keeps the existing default behavior, documents the opt-in, and proves the Matrix runtime behavior.

## Reproduction Assessment

Yes. Current source resolves Matrix thread bindings before mention gating, and the existing handler test shows a bound in-thread reply without a mention is dropped before `recordInboundSession` or binding `touch`.

## Solution Assessment

Unclear until maintainer review of the linked PR. A default-false opt-in is the safest direction because current tests intentionally preserve mention enforcement for bound threads by default.
