## Label Justifications

- `P1`: The PR targets a real Gemini CLI auth-provider workflow where selected OpenClaw OAuth profiles can be ignored or misrouted at runtime.
- `merge-risk: 🚨 compatibility`: The patch changes CLI session auth epochs and Gemini credential storage behavior that can affect existing Gemini CLI users during upgrade.
- `merge-risk: 🚨 auth-provider`: The diff changes auth profile selection, refresh, and forwarding for the `google-gemini-cli` runtime.
- `merge-risk: 🚨 security-boundary`: The diff forwards OAuth token material across the plugin and external CLI boundary and currently risks native keychain/global credential contamination.

## Summary

Keep this PR open: the bug fix direction is useful, but the current bridge targets Gemini CLI's legacy plaintext OAuth file while Gemini CLI 0.40.1 now loads token storage through its hybrid keychain/file store first, so existing keychain users can still run with ambient credentials or have OpenClaw credentials written into global Gemini storage.

## What This Changes

The PR adds opt-in CLI backend auth-profile forwarding, refreshes selected OAuth profiles before CLI execution, and stages Gemini OAuth credentials under an isolated `GEMINI_CLI_HOME` for the `google-gemini-cli` backend.

## Best Possible Solution

Keep the PR open, switch the Gemini bridge to an isolated current Gemini token-storage path, document or narrow the token-forwarding SDK contract, then rebase and prove an auth-consuming Gemini run uses the selected OpenClaw profile.

## Reproduction Assessment

Yes, source-reproducible: current main suppresses auth-profile forwarding for CLI execution and the current Gemini CLI backend has no `prepareExecution` auth bridge. I did not run a live Gemini model call in this read-only review.

## Solution Assessment

No: the owner-boundary direction is plausible, but the current implementation targets Gemini's legacy OAuth file instead of isolating the current hybrid token store, and the new public token-forwarding SDK contract is not yet settled.
