## What This Changes

The PR adds a private `google-gemini-cli` auth bridge that selects and refreshes OpenClaw auth profiles, stages OAuth/API-key credentials into isolated Gemini CLI homes, clears inherited Google auth environment, rejects Gateway profiles on this backend, and adds regression tests.

## Best Possible Solution

Land only after provider/auth owners accept the upgrade behavior and relevant checks remain green; keep the bridge private rather than adding a public SDK credential-forwarding API.

## Reproduction Assessment

## Solution Assessment

Yes, with maintainer signoff. A backend-owned private `prepareExecution` bridge is the narrowest owner boundary; a public plugin SDK credential-forwarding API would be broader than this bug fix.
