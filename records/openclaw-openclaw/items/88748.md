## Label Justifications

- `P1`: The PR affects selected-provider authentication and can make real Gemini CLI runs use the wrong or unusable credential source.
- `merge-risk: 🚨 compatibility`: The diff changes `google-gemini-cli` upgrade behavior by clearing inherited auth environment and rejecting Gateway profiles on this backend.

- `merge-risk: 🚨 security-boundary`: The bridge stages OAuth/API-key material into a temporary external CLI home and subprocess environment, so credential shape and cleanup must be exact.

## Summary

Keep open: the PR is aimed at the right owner boundary, but the current patch still replaces Gemini CLI's raw OAuth access token with OpenClaw's provider-formatted Google OAuth API-key string before writing `oauth_creds.json`, so selected OAuth profiles can fail in the real execution path.

## What This Changes

The branch adds a private `google-gemini-cli` auth bridge that selects OpenClaw auth profiles, stages OAuth/API-key credentials in an isolated `GEMINI_CLI_HOME`, clears conflicting auth environment, and adds focused tests.

## Best Possible Solution

Keep the private Google-owned bridge, but preserve raw OAuth credential fields for Gemini CLI staging, add a regression for the refreshed OAuth shape, and refresh live proof after maintainers accept the auth precedence change.

## Reproduction Assessment

Yes, source-reproducible: current main has no Gemini CLI `prepareExecution` bridge, and the linked bug shows selected OpenClaw profiles can diverge from ambient Gemini CLI credentials. I did not run a live Google account flow in this read-only review.

## Solution Assessment

No. The private Google plugin bridge is the right layer, but this implementation still corrupts OAuth credential shape and needs maintainer signoff for the compatibility-sensitive auth precedence changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve the raw OAuth token for Gemini CLI:** `src/agents/cli-runner/prepare.ts:336`
  - body: `resolveApiKeyForProfile` returns the provider-formatted API key string; for Google OAuth that is JSON, not the raw access token. Replacing `authCredential.access` with that value means the bridge can write JSON into `oauth_creds.json.access_token`, so selected OAuth runs may fail. Keep refreshed raw credential fields separate from the formatted API-key value before calling `prepareExecution`.
  - confidence: 0.93
- **[P3] Clean up temp homes when setup fails:** `extensions/google/cli-backend.ts:147-158`
  - body: `createIsolatedGeminiCliHome` creates the temp home before chmod, mkdir, and settings writes, but callers only enter their cleanup try/catch after this helper returns. If one of those setup steps throws, the directory remains under the OpenClaw temp area; wrap the helper's setup so partial homes are removed before rethrowing.
  - confidence: 0.86
