## Label Justifications

- `P1`: The PR targets an auth-provider runtime bug affecting selected Gemini CLI OAuth profiles, but the remaining blockers are security and auth-routing sensitive.
- `merge-risk: 🚨 compatibility`: The PR changes the public CLI backend plugin contract and adds new opt-in behavior that external backend plugins could depend on after release.
- `merge-risk: 🚨 auth-provider`: The runtime auth plan can use different provider-alias inputs than the selected auth profile path, risking wrong or missing auth-profile forwarding.
- `merge-risk: 🚨 security-boundary`: The diff exposes OAuth access, refresh, and ID token fields to CLI backend plugin code through a new public SDK context field.

## Summary

Keep open: the Gemini OAuth bridge is a plausible fix, but it is not merge-ready because it adds a token-bearing public CLI backend SDK contract, still lets runtime auth planning use different alias inputs than profile selection, and the supplied real proof only starts `gemini --version` rather than an authenticated Gemini request.

## What This Changes

The PR forwards selected `google-gemini-cli` OAuth profiles into CLI execution by adding CLI backend auth-profile forwarding flags/types, refreshing/loading credentials in CLI preparation, and writing an isolated Gemini CLI home for the subprocess.

## Best Possible Solution

Land a bounded Gemini OAuth bridge only after preserving alias-plan inputs, keeping raw OAuth material out of any unapproved public SDK contract or explicitly approving/documenting it, and adding redacted authenticated Gemini request proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source: current main blocks normal auth-profile forwarding for CLI providers and the current Gemini CLI backend has no `prepareExecution` credential bridge.

## Solution Assessment

Is this the best way to solve the issue? Not yet; the bridge shape is plausible, but the best fix must preserve existing auth-alias inputs and settle the token-bearing SDK boundary before merge.
