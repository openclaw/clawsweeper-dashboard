## Summary

Keep open: current main still lacks the Gemini CLI selected-profile bridge, but this PR is not merge-ready because the bridge can still lose to Gemini CLI native keychain state, one auth-routing call drops alias snapshot inputs, the branch expands the public raw-token SDK surface, and the supplied proof does not exercise an authenticated Gemini CLI command.

## What This Changes

The PR adds opt-in CLI auth-profile forwarding for `google-gemini-cli`, refreshes and forwards the selected OAuth profile to `prepareExecution`, and writes Gemini-compatible OAuth credentials into an isolated `GEMINI_CLI_HOME`.

## Best Possible Solution

Land a rebased Gemini-owned bridge that forces Gemini CLI to consume the isolated selected profile, preserves auth-alias snapshot semantics, narrows or explicitly approves the credential-forwarding SDK contract, drops unrelated changes, and includes redacted auth-consuming Gemini CLI proof.
