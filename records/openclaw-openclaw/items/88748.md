## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve the raw OAuth token for Gemini CLI:** `src/agents/cli-runner/prepare.ts:336`
  - body: `resolveApiKeyForProfile` returns the provider-formatted OpenClaw API key string for OAuth profiles; for Google OAuth that is JSON, not the raw access token. Assigning it back to `authCredential.access` means the Gemini bridge can write JSON into `oauth_creds.json.access_token`, so selected OAuth profiles may fail or authenticate with the wrong material. Keep the refreshed raw credential fields separate from the formatted API-key value before calling `prepareExecution`.
  - confidence: 0.91
- **[P3] Remove temp homes when setup fails:** `extensions/google/cli-backend.ts:143-158`
  - body: `createIsolatedGeminiCliHome` creates the temp home before chmod, mkdir, and settings writes, but cleanup is only returned after those steps finish. If any setup step throws, the directory remains under the OpenClaw temp area; wrap the setup block so the partially-created home is removed before rethrowing.
  - confidence: 0.87
