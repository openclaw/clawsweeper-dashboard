## Summary

## What This Changes

This PR adds a private `google-gemini-cli` auth bridge for selected OAuth/API-key profiles, clears conflicting Gemini auth environment variables, rejects Vercel Gateway profiles on that backend, and adds focused regression tests.

## Best Possible Solution

Keep the backend-owned bridge shape, fix OAuth forwarding so `oauth_creds.json` receives raw refreshed token fields, retain the focused tests, and require refreshed real-run proof plus maintainer security/compatibility signoff before merge.
