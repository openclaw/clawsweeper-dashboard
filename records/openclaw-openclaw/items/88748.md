## Summary

Keep open: the owner-local bridge is a useful direction, but the current head still forwards provider-formatted JSON as the OAuth access token on the full prepare path, and the supplied proof does not cover that path after this source-level bug.

## What This Changes

The branch adds a private Google Gemini CLI auth bridge that stages selected OAuth or API-key OpenClaw profiles into an isolated `GEMINI_CLI_HOME`, clears conflicting Gemini auth environment, rejects Vercel Gateway profiles on this backend, and adds focused tests.

## Best Possible Solution

Keep the private Google-owned bridge, preserve raw refreshed OAuth fields for OAuth staging, add full-path regression coverage, then require refreshed redacted live proof plus maintainer security and compatibility signoff before merge.
