## Summary

Keep open. Current main still has the hard timeout path after a successful `agent` start, and the open same-author PR #68196 with closing syntax is the right place to resolve it before closing this issue.

## What This Changes

Change `sessions_send` so a nonzero `agent.wait` deadline after an accepted send is reported as `accepted` or `ok` with a late reply instead of a hard timeout.

## Best Possible Solution

Review, land, or supersede #68196 with a narrow fix at the `sessions_send` and run-wait boundary that compensates wait-window expiry without changing unrelated gateway recovery behavior.

## Reproduction Assessment

Yes. A focused mock where `agent` returns an accepted run id and the first `agent.wait` returns `timeout` reproduces the current branch: the helper returns timeout without a history read, then `sessions_send` returns `status: "timeout"`.

## Solution Assessment

Yes. The requested compensation boundary is the narrow maintainable direction for this specific misclassification; broader gateway-routing recovery should remain separate unless it proves to share the same root cause.
