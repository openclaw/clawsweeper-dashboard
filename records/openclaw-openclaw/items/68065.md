## Summary

Keep open. Current main and the latest release have already softened the original hard-timeout classification by returning `accepted` for nonterminal first-wait timeouts, but they still do not implement the stronger post-timeout late-reply compensation contract, and the same reporter has an open closing PR for that exact work.

## What This Changes

The issue requests that `sessions_send` distinguish accepted-but-not-yet-replied agent runs from hard delivery timeouts after the first `agent.wait` timeout.

## Best Possible Solution

Resolve https://github.com/openclaw/openclaw/pull/68196, then close this issue once main either lands the post-timeout compensation contract or maintainers explicitly decide that the shipped `accepted` plus A2A follow-up behavior is the intended contract.

## Reproduction Assessment

No live current-main reproduction was run in this read-only review. Source and regression-test inspection show the current partial behavior clearly: nonterminal first-wait timeouts become `accepted`, but timeout paths still do not perform a post-timeout late-reply compensation read.

## Solution Assessment

Unclear until the linked PR is resolved. The proposed run-wait/sessions_send compensation path is a plausible narrow fix, but maintainers need to decide whether inline late-reply `ok` is better than the currently shipped `accepted` plus delayed A2A follow-up contract.
