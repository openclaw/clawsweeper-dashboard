## Summary

Keep open. Current main no longer treats a soft `agent.wait` window expiry as a hard timeout, but it still lacks the requested inline late-reply compensation, and the same reporter has an open closing PR that should be reviewed, rebased, or superseded before this issue is closed.

## What This Changes

The issue asks `sessions_send` to distinguish accepted-but-not-yet-replied runs from true send failures after the first nonzero-timeout `agent.wait` expires.

## Best Possible Solution

Review, rebase, land, or supersede https://github.com/openclaw/openclaw/pull/68196 while preserving current terminal-timeout handling and delayed A2A delivery.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source-level yes for the remaining gap: make the agent start succeed, make the first `agent.wait` return a non-terminal timeout, then add a post-baseline reply before the follow-up history read; current `run-wait` returns before reading history. The original hard-timeout symptom is no longer the current-main path for non-terminal waits because `sessions_send` returns `accepted` with pending delivery.

## Solution Assessment

Is this the best way to solve the issue? Unclear as currently proposed in the open PR: post-timeout compensation is the right narrow boundary, but it must be rebased onto current main and must not hide terminal timeouts or drop the delayed A2A follow-up.
