## Summary

Keep open. Current main and v2026.6.6 already mitigate the original hard-timeout classification by returning `accepted` for nonterminal waits and preserving delayed A2A follow-up, but the requested post-timeout inline compensation is still absent and the linked closing PR remains open.

## What This Changes

The issue asks for `sessions_send` to distinguish accepted-but-not-ready agent runs from hard delivery timeouts, including late-reply compensation after the first `agent.wait` timeout.

## Best Possible Solution

Resolve the linked compensation PR: land a rebased fix that preserves current timeout diagnostics, or explicitly supersede it if maintainers decide the shipped `accepted` plus A2A follow-up contract is the intended behavior.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. Source inspection shows the remaining compensation gap clearly: current main does not read history after a timeout wait, while current behavior already maps nonterminal waits to `accepted`.

## Solution Assessment

Unclear as a final close path. The linked PR is the right owner for the stronger compensation proposal, but maintainers still need to decide whether inline late-reply compensation should replace the shipped `accepted` plus delayed A2A behavior.
