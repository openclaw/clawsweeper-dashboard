## Summary

Current main implements the reported fix: soft `agent.wait` window timeouts after a successful `sessions_send` start now return `accepted` with delayed A2A delivery instead of a hard timeout, with regression coverage and release provenance in `v2026.5.7`.

## What This Changes

The issue requests that `sessions_send` treat a successful agent start followed by a soft `agent.wait` timeout as accepted or late delivery rather than a hard send failure.

## Best Possible Solution

Keep the shipped `v2026.5.7` sessions_send behavior and close this issue; handle the older open implementation PR separately as superseded by the merged fix.

## Reproduction Assessment

No current-main failure path remains from source inspection: a soft wait-window timeout now returns `accepted` and starts delayed A2A delivery, with regression tests covering that path. The original failure was source-reproducible on older main by mocking `agent` accepted plus a first bare `agent.wait` timeout.

## Solution Assessment

Yes. The merged fix is narrower than the proposed post-timeout history compensation because it preserves terminal timeout results, keeps the delayed A2A delivery path alive, and reuses existing baseline-filtered reply reads.
