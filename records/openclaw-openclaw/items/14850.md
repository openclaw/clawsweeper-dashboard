## Summary

Keep open. Current main still only supports `bindings[].session.dmScope` for direct-message session scoping; non-DM guild/channel peers remain explicitly channel-isolated in source, tests, and docs. Related work such as https://github.com/openclaw/openclaw/pull/47234 is open but not merged or settled, so this needs maintainer product and security design rather than conservative cleanup.

## What This Changes

The issue requests a binding-level session key, alias, or scope override so selected non-DM channel bindings can share an agent session across Discord and Telegram surfaces.

## Best Possible Solution

Add an explicit opt-in binding-level session target or alias contract for safe agent-owned targets, preserve default group/channel isolation, and update schema/help/docs plus routing regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection on current main gives high confidence that the requested binding-level non-DM session target cannot be expressed today and that channel peers remain isolated.

## Solution Assessment

Unclear as proposed. A raw `sessionKey` override is likely too permissive; the safer maintainable direction is a typed binding session target or alias with validation, privacy documentation, and unchanged isolated defaults.
