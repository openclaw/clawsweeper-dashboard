## Summary

Keep open. Current main still lacks a binding-level session target or alias for routing Discord guild/channel bindings into an agent main or DM session, and the existing schema, resolver, tests, and docs continue to preserve channel isolation.

## What This Changes

Request adding an opt-in binding-level session target or alias so Discord guild/channel route bindings can share continuity with an agent's main or DM session.

## Best Possible Solution

Add an explicit opt-in, typed route-binding session target or alias contract, validate it to safe canonical agent-owned targets, preserve default group/channel isolation, and update schema/help/docs plus regression coverage.

## Reproduction Assessment

Yes. Static inspection and current tests show route-binding session config only accepts `dmScope`, and channel peers resolve to isolated channel keys rather than `agent:<agentId>:main`.

## Solution Assessment

Unclear as proposed. A raw binding `sessionKey` is likely too permissive; a typed, validated target or alias contract is the safer maintainable direction.
