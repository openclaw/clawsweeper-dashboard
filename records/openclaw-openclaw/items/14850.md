## Summary

Keep open. Current main still supports only `bindings[].session.dmScope` for direct-message routing, while guild/channel peers remain isolated and the requested cross-surface unification needs an explicit product and security policy.

## What This Changes

Request to add a binding-level session target or alias so a Discord guild channel and Telegram DM bound to the same agent can share one session.

## Best Possible Solution

Expose a typed, validated binding session target or alias contract for safe agent-owned targets, preserve default group/channel isolation, and update schema/help/docs plus regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main has no binding-level session target beyond `dmScope`, and current tests/docs preserve channel isolation.

## Solution Assessment

Unclear as proposed. A raw `sessionKey` override is likely too permissive; a typed target or alias with validation and unchanged defaults is the safer maintainable direction.
