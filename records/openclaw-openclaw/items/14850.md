## Summary

Keep open. Current main still lacks a binding-level session target for regular route bindings; the only binding session override is DM-only `dmScope`, and guild/channel peers remain intentionally isolated in source, tests, and docs.

## What This Changes

Request an optional binding-level session key, alias, or scope so a Discord guild/channel binding can share an agent session with the same agent's direct-message surfaces.

## Best Possible Solution

Expose a typed, validated binding session target or alias contract for safe agent-owned targets, preserve default group/channel isolation, and update schema/help/docs plus regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives a high-confidence current-main path showing no binding-level session target beyond `dmScope`, and tests/docs preserve non-DM channel isolation.

## Solution Assessment

Unclear as proposed. A raw `sessionKey` override is likely too permissive; a typed target or alias with validation, warnings, and unchanged defaults is the safer maintainable direction.
