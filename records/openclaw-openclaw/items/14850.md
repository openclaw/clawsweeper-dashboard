## Summary

Keep open. Current main still only supports binding-level `dmScope` for direct-message peers, while Discord guild channels remain intentionally isolated and the requested sharing of channel traffic into a main/private session needs maintainer policy.

## What This Changes

Request adding a binding-level session key, alias, or scope override so a Discord guild channel and Telegram DM routed to the same agent can share one session.

## Best Possible Solution

Add an explicit typed route-binding session target or alias contract, validate it to safe agent-owned targets, preserve default group/channel isolation, and update schema/help/docs plus regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows binding session config only accepts `dmScope`, and the session-key helper always builds isolated keys for non-direct peers; current tests assert the same behavior.

## Solution Assessment

Unclear as proposed. A raw binding `sessionKey` would be too permissive; a typed, validated target or alias contract is the safer maintainable direction.
