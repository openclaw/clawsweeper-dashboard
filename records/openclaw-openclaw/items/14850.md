## Summary

Keep open. Current main still has only a direct-message `dmScope` route-binding override; guild/channel peers remain explicitly isolated in source, tests, and docs, so the requested cross-channel unification is real but needs a deliberate product/security design.

## What This Changes

Request adding a binding-level session target or alias so configured Discord guild/channel bindings can share an agent's main or DM session across channels.

## Best Possible Solution

Expose a typed, validated binding session target or alias contract for safe agent-owned targets, preserve default group/channel isolation, and update schema/help/docs plus regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence current-main path showing no binding-level session target beyond `dmScope`, and tests/docs preserve channel isolation.

## Solution Assessment

Unclear as proposed. A raw `sessionKey` override is likely too permissive; a typed target or alias with validation and unchanged defaults is the safer maintainable direction.
