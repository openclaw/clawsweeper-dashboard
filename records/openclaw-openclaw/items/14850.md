## Summary

Keep open. Current main still only supports binding-level `dmScope` for direct peers; Discord guild/channel route bindings still resolve to isolated channel session keys, and the requested shared session target needs a new config/API contract plus maintainer policy.

## What This Changes

Request an opt-in binding-level session target or alias so a Discord guild channel route can share an agent main/DM session with Telegram or other DMs.

## Best Possible Solution

Add an explicit typed route-binding session target or alias contract, validate it to safe agent-owned targets, preserve default group/channel isolation, and update schema/help/docs plus regression coverage.

## Reproduction Assessment

Yes. Source inspection shows route-binding session config only accepts `dmScope`, and the session-key helper always builds isolated keys for non-direct peers; current tests assert that DM and channel keys remain distinct.

## Solution Assessment

Unclear as proposed. A raw binding `sessionKey` would be too permissive; a typed, validated target or alias contract is the safer maintainable direction.
