## Summary

Keep open. Current main still lacks a binding-level session target for regular route bindings; the only binding session override is DM-only `dmScope`, and channel/guild peers remain intentionally isolated in source, tests, and docs.

## What This Changes

The issue requests a binding-level `sessionKey`, `sessionAlias`, or `sessionScope` override so a routed Discord guild channel can share the same agent session as a Telegram DM.

## Best Possible Solution

Add an explicit opt-in route-binding session target or alias contract for safe agent-owned targets, preserve default group/channel isolation, and update schema/help/docs plus regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives a high-confidence current-main path showing no binding-level session target beyond DM-only `dmScope`, with tests and docs preserving non-DM channel isolation.

## Solution Assessment

Unclear as proposed. A raw `sessionKey` override is likely too permissive; a typed target or alias with validation, warnings, and unchanged defaults is the safer maintainable direction.
