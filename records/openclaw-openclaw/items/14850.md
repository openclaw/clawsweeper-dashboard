## Summary

Keep open. Current main still has no binding-level session target or alias for regular route bindings; matched bindings only carry DM scoping, while Discord guild/channel peers remain intentionally channel-isolated in source, tests, and docs. This is a valid feature/config request, but it needs maintainer product and security design rather than conservative cleanup or autonomous repair.

## What This Changes

Request an optional binding-level session target or alias so a Discord guild/channel route binding can share an agent's main/DM session across Telegram and Discord.

## Best Possible Solution

Add an explicit opt-in binding-level session target or alias contract for safe agent-owned targets, preserve default group/channel isolation, and update schema/help/docs plus routing regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection on current main gives high confidence that the requested binding-level session target cannot be expressed today and that non-DM channel peers remain isolated.

## Solution Assessment

Unclear as proposed. A raw `sessionKey` override is likely too permissive; the safer maintainable direction is a typed target or alias with validation, warnings, and unchanged isolated defaults, coordinated with the related open implementation work.
