## Summary

Keep open. Current main still has no binding-level session target or alias for regular route bindings; matched bindings can only override DM scoping, while non-DM channel/guild peers remain intentionally isolated in source, tests, and docs.

## What This Changes

The issue requests a binding-level session target or alias so a matched Discord guild/channel binding can share an agent session with direct-message surfaces.

## Best Possible Solution

Add an explicit opt-in route-binding session target or alias contract for safe agent-owned targets, preserve default group/channel isolation, and update schema/help/docs plus regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main lacks a binding-level session target beyond DM-only `dmScope`, with tests and docs preserving non-DM channel isolation.

## Solution Assessment

Unclear as proposed. A raw `sessionKey` override is likely too permissive; a typed target or alias with validation, warnings, and unchanged defaults is the safer maintainable direction.
