## Summary

Current main already addresses the reported WhatsApp QR packaging failure by routing WhatsApp QR rendering through the core media-runtime QR helpers and by removing the staged bundled-plugin node_modules link that caused plugin-runtime dependency roots to resolve packages away from the OpenClaw package dependency tree. The fix is on main only; no shipped release tag contains it yet.

## What This Changes

Restore WhatsApp QR pairing when the packaged 2026.4.29 runtime fails to render QR codes because `qrcode` cannot be resolved from the WhatsApp plugin runtime dependency tree.

## Best Possible Solution

Keep the current main packaging direction: QR rendering stays in the core media runtime with the root `qrcode` dependency, bundled plugin runtime wrappers resolve the canonical package dist graph, and the fix ships in the next release.

## Reproduction Assessment

Yes. The report gives a concrete npm v2026.4.29 path: run WhatsApp login or click Show QR and QR rendering fails while importing `qrcode` from a legacy `plugin-runtime-deps` root. This review did not run the packaged flow because the checkout had to remain read-only, but the source and history match the failure signature.

## Solution Assessment

Yes. Resolving bundled plugin QR helpers through the core package runtime is the narrow maintainable fix; adding `qrcode` to the WhatsApp plugin would duplicate a core QR dependency and would not address stale legacy runtime roots.
