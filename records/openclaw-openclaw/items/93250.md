## What This Changes

The issue requests agent-visible browser download completion feedback when a browser click action saves a file in the managed downloads directory.

## Best Possible Solution

Land one reviewed browser-plugin fix that reuses the existing managed download primitives, preserves download-root and existing-session limits, and then close or supersede duplicate candidate PRs.

## Reproduction Assessment

Yes. Source inspection shows current main has managed download routes and helpers, but `act:click` returns no download payload and the browser tool schema has no `download` or `waitfordownload` action.

## Solution Assessment

Unclear until maintainers choose the API shape. The best path is one canonical browser-plugin fix that preserves the existing download security constraints and either reports click-triggered downloads or exposes a deliberate download action contract.
