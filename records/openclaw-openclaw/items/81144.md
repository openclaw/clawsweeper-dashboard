## Summary

Keep open: the patch has a blocking channel-boundary bug because WebChat message-tool-only runs would render every successful messaging-tool text back into WebChat, including explicit external sends. The PR also still needs maintainer choice against the open automatic-delivery alternative and stronger real behavior proof beyond focused tests.

## What This Changes

The PR adds a WebChat same-session message-tool success path, renders Codex messaging-tool text as source-suppression-safe embedded-run payloads, and adds focused tests for sanitized telemetry and payload metadata.

## Best Possible Solution

If maintainers choose this renderer option, carry an explicit same-session WebChat delivery signal from the message-tool result into Codex telemetry and render only those payloads; otherwise land https://github.com/openclaw/openclaw/pull/81110 as the canonical fix for https://github.com/openclaw/openclaw/issues/81109.

## Reproduction Assessment

Yes, from source inspection: an explicit external message send from a WebChat message_tool_only run records messagingToolSentTexts, and the new renderer uses those texts without checking the target/provider. I did not execute a live PR-head WebChat reproduction.

## Solution Assessment

No. The renderer should distinguish same-session WebChat tool results from external channel sends before bypassing source-reply suppression; the open automatic-delivery alternative is also still a maintainer product choice.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Filter WebChat rendering to same-session message sends:** `src/agents/pi-embedded-runner/run.ts:279-283`
  - body: The new renderer turns every `messagingToolSentText` into a WebChat payload whenever the source is WebChat and `message_tool_only` is active. An explicit external send from WebChat, such as `channel: "telegram"`, still records `didSendViaMessagingTool` and the sent text in Codex telemetry, so this path would also queue that external message back to WebChat even though the PR says external delivery behavior is unchanged. Carry a same-session delivery marker from the tool result or filter the recorded targets before rendering these texts.
  - confidence: 0.9
