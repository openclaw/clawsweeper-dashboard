## Summary

Keep open; the PR still has a blocking sanitizer/security regression. The renderer path turns raw Codex messaging-tool telemetry into visible WebChat payloads, while the current message tool sanitizer only strips reasoning tags from a copied params object before execution.

## What This Changes

The PR treats same-session WebChat `message(action="send")` calls in `message_tool_only` mode as visible WebChat replies and adds payload metadata/tests for that path.

## Best Possible Solution

If maintainers choose this renderer over the automatic-reply alternative, keep the WebChat-only scope but render sanitized same-session message-tool output and preserve the source-suppression metadata behavior.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: Codex telemetry stores original message-tool args, while the message tool strips reasoning tags only from its copied params. A focused regression with `<think>hidden</think>Visible reply` would exercise the leak without needing broad setup.

## Solution Assessment

No, not yet. The renderer approach may be acceptable if maintainers choose it, but the PR should render sanitized result text or sanitize telemetry before turning it into WebChat-visible payloads.
