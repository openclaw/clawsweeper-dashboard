## Summary

Keep open: the retry-plus-filter direction is reasonable for the tar EOF race, but the current PR head still misses current live session and cron log path contracts, and the supplied proof does not exercise those paths.

## What This Changes

The PR adds volatile backup path filtering, tar EOF retry/backoff, enriched backup errors, skippedVolatileCount reporting, command logger plumbing, and backup tests.

## Best Possible Solution

Update the branch so the built-in volatile filter covers current agent-scoped session transcripts and cron JSONL run logs, keeps JSON callers clean, and documents/changelogs the changed backup contents.

## Reproduction Assessment

Yes. The remaining blocker is reproducible by source inspection: current main writes sessions under stateDir/agents/<agentId>/sessions/*.jsonl and cron logs under cron/runs/<jobId>.jsonl, while the PR does not exclude those paths.

## Solution Assessment

No, not yet. Retry-plus-filter is a narrow maintainable approach for the node-tar EOF race, but it must match current storage contracts and preserve/document the backup output contract before merge.
