## Summary

Keep this PR open. Current main still lacks the requested backup resilience, and the PR is directionally useful, but the latest diff is not mergeable because its volatile filter still misses OpenClaw's current live session and cron run-log paths, so the main reported EOF culprits remain in the archive input.

## What This Changes

The PR branch adds backup volatile-file filtering, tar EOF retry/backoff cleanup, enriched failure messages, skippedVolatileCount reporting, command logger plumbing, and focused backup tests.

## Best Possible Solution

Revise this PR or a narrow replacement so the volatile filter covers current agents/<agentId>/sessions/*.jsonl and cron/runs/<jobId>.jsonl files, keeps machine-readable JSON output clean for command callers, documents the new backup semantics, adds the required changelog entry, and leaves ENOENT deletion races and configurable exclusion design to #67417 and #67990.

## Reproduction Assessment

Yes. The PR-introduced path bug is reproducible by evaluating the PR's isVolatileBackupPath against {stateDir}/agents/main/sessions/live.jsonl and {stateDir}/cron/runs/job.jsonl; both remain false even though current source and docs identify those as live append paths.

## Solution Assessment

No, not as currently written. Retry-plus-filter is a plausible narrow solution for the tar EOF race, but the filter must match current storage contracts and the user-facing backup content/API change needs docs, changelog, and JSON-channel hygiene before merge.
