## Summary

Keep open. Current main still lacks retry/volatile filtering for the backup tar EOF race, but this PR remains incorrect because its filter does not match the current live session and cron run-log paths, and it still needs JSON-output and user-facing docs/changelog cleanup before merge.

## What This Changes

The PR adds backup volatile-file filtering, tar EOF retry/backoff handling, enriched final failures, skippedVolatileCount reporting, and focused backup tests.

## Best Possible Solution

Update the branch so the volatile filter matches current storage contracts, operational retry/skip notices cannot corrupt JSON callers, and the backup content/API change is documented and changelogged while leaving ENOENT deletion races and configurable excludes to their existing follow-ups.

## Reproduction Assessment

Yes. A static reproduction is enough: check the PR predicate against {stateDir}/agents/main/sessions/live.jsonl and {stateDir}/cron/runs/job.jsonl; current main shows those are live append paths, but the PR rules do not match them.

## Solution Assessment

No, not as currently written. Retry-plus-filter is a reasonable narrow fix for the node-tar EOF race, but the branch must first align with current path contracts and preserve JSON/docs/changelog expectations.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Match current session and cron paths:** `src/infra/backup-volatile-filter.ts:69-75`
  - body: Current session transcripts live under `stateDir/agents/<agentId>/sessions/*.jsonl`, and cron run logs are `cron/runs/<jobId>.jsonl`. This filter only covers `stateDir/sessions` and cron `*.log`, so the files most likely to trigger the reported live-write EOF race remain archived.
  - confidence: 0.94

  - body: `backupCreateCommand` wires retry and skip notices to `runtime.log` even when `opts.json` is true. CLI startup may route the default logger away from stdout, but direct command callers and tests that use a plain `RuntimeEnv` also use `runtime.log` as the JSON fallback, so retry notices can precede the JSON payload.
  - confidence: 0.8
- **[P3] Document the backup content change:** `src/infra/backup-create.ts:518-524`
  - body: This changes what `openclaw backup create` includes and adds `skippedVolatileCount`, but the branch does not update the backup docs or changelog. Users need to know live session/cron/log tails are skipped and how that count appears before this user-facing backup change ships.
  - confidence: 0.86
