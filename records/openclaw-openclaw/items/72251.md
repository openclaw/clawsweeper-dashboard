## Summary

Keep open: the PR targets a real backup EOF race, but the current head is not merge-ready because the volatile filter still misses the current session and cron run-log paths, and the user-facing output/docs follow-through is incomplete.

## What This Changes

This PR adds backup tar EOF retry/backoff, built-in volatile-file filtering, skippedVolatileCount reporting, backup command logging plumbing, and focused backup tests.

## Best Possible Solution

Repair the branch in place so volatile filtering covers current session/cron/log paths, JSON callers receive one clean payload, and backup docs/changelog describe skipped content and skippedVolatileCount.

## Reproduction Assessment

Yes. Source inspection against PR head shows isVolatileBackupPath only anchors sessions at {stateDir}/sessions and cron runs at .log, while current main writes sessions under agents/<agentId>/sessions and cron run logs as .jsonl.

## Solution Assessment

No, not as currently written. Retry-plus-filter is a reasonable narrow direction for the node-tar EOF race, but it must match current storage contracts, preserve JSON output, and update docs/changelog before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Match current live log paths:** `src/infra/backup-volatile-filter.ts:69-75`
  - body: Current session transcripts live under stateDir/agents/<agentId>/sessions/*.jsonl, and cron run logs are cron/runs/<jobId>.jsonl. This filter only covers stateDir/sessions and cron *.log files, so the files most likely to trigger the reported live-write EOF race remain archived.
  - confidence: 0.95
- **[P3] Keep retry notices out of JSON output:** `src/commands/backup.ts:23-26`
  - body: backupCreateCommand now wires retry and skip notices to runtime.log even when opts.json is true. Direct command callers that pass a plain RuntimeEnv also use runtime.log as writeRuntimeJson's fallback sink, so a retry or skip notice can be emitted before the JSON payload and break machine-readable output.
  - confidence: 0.84
- **[P3] Document the backup content change:** `src/infra/backup-create.ts:518-524`
  - body: This changes what openclaw backup create includes and adds skippedVolatileCount to the result, but the branch does not update the backup docs or changelog. Users need to know live session/cron/log tails are skipped and how the count appears before this user-facing backup change ships.
  - confidence: 0.87
