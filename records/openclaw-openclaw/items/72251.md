## Summary

Keep open: this is a plausible backup bug fix, but the current PR still has merge-blocking correctness issues in what it skips and what live paths it actually covers, and the supplied proof does not exercise the latest/current path contracts.

## What This Changes

The PR adds backup tar EOF retries, volatile-file filtering, enriched backup errors, skipped-volatile reporting, and focused backup tests.

## Best Possible Solution

Revise the PR so volatile skips are scoped to known state/runtime paths, current agent session and cron JSONL paths are covered, JSON mode stays machine-readable, docs describe the new backup exclusions, and latest-head proof shows the real backup path passing.

## Reproduction Assessment

Yes. Source inspection is enough here: current main writes sessions under stateDir/agents/<agentId>/sessions/*.jsonl and cron logs under cron/runs/<jobId>.jsonl, while the PR filter does not match those paths and globally skips meaningful workspace *.lock files.

## Solution Assessment

No, not yet. Retry-plus-filter is a maintainable direction for the node-tar EOF race, but the filter must be scoped to state/runtime volatility and aligned with current storage contracts before this is the best fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Don't skip workspace lock files globally:** `src/infra/backup-volatile-filter.ts:60-62`
  - body: `isVolatileBackupPath` returns true for `.sock`, `.pid`, `.tmp`, and `.lock` before checking any stateDir anchor, and `createBackupArchive` applies this filter to every asset. Since workspace directories are included by default, this silently drops valid workspace files such as `Cargo.lock` or `poetry.lock` from backup archives. Scope these suffix skips to known state/runtime locations instead of all backup sources.
  - confidence: 0.92
- **[P2] Match current live session and cron paths:** `src/infra/backup-volatile-filter.ts:70-76`
  - body: Current session transcripts live under `stateDir/agents/<agentId>/sessions/*.jsonl`, and cron run logs are `cron/runs/<jobId>.jsonl`. This filter only covers `stateDir/sessions/**/*.{jsonl,log}` and cron `*.log`, so the main files likely to trigger the EOF race remain archived.
  - confidence: 0.95
- **[P3] Keep retry notices out of JSON output:** `src/commands/backup.ts:27`
  - body: Defaulting the new logger to `runtime.log` means retry and skip notices can be printed before `writeRuntimeJson` when `opts.json` is true, especially for plain `RuntimeEnv` callers that use the same log sink for JSON. Suppress the default logger in JSON mode unless a caller explicitly passes one.
  - confidence: 0.8
