## Summary

Keep open: the branch targets a real backup EOF race, but it still misses current live session and cron run paths and needs JSON-output plus docs/changelog cleanup before merge.

## What This Changes

This PR adds backup volatile-file filtering, tar EOF retry/backoff handling, enriched backup failures, skippedVolatileCount reporting, command logger plumbing, and focused backup tests.

## Best Possible Solution

Revise the branch so the built-in volatile filter matches current storage contracts, operational retry/skip notices cannot corrupt JSON callers, and the backup content/API change is documented and changelogged while leaving ENOENT deletion races and configurable excludes to their existing follow-ups.

## Reproduction Assessment

Yes. The main blocker is reproducible by checking the branch predicate against `{stateDir}/agents/main/sessions/live.jsonl` and `{stateDir}/cron/runs/job.jsonl`; current main source shows those are the live append paths, but the PR rules return false.

## Solution Assessment

No, not as currently written. Retry-plus-filter is a maintainable narrow approach for the node-tar EOF race, but it must match current path contracts and preserve JSON/docs/changelog expectations before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Match current session and cron paths:** `src/infra/backup-volatile-filter.ts:69-75`
  - body: Current session transcripts live under `stateDir/agents/<agentId>/sessions/*.jsonl`, and cron run logs are `cron/runs/<jobId>.jsonl`. This filter only covers `stateDir/sessions` and cron `*.log`, so the files most likely to trigger the reported live-write EOF race remain archived.
  - confidence: 0.94
- **[P3] Keep retry notices out of JSON output:** `src/commands/backup.ts:23-26`
  - body: `backupCreateCommand` wires retry and skip notices to `runtime.log` even when `opts.json` is true. CLI startup routes the default console logger to stderr, but direct command callers and tests that use a plain `RuntimeEnv` also use `runtime.log` as the JSON sink, so retry notices can precede the JSON payload.
  - confidence: 0.8
- **[P3] Document the backup content change:** `src/infra/backup-create.ts:519-524`
  - body: This changes what `openclaw backup create` includes and adds `skippedVolatileCount`, but the branch does not update the backup docs or changelog. Users need to know live session/cron/log tails are skipped and how that count appears before this user-facing backup change ships.
  - confidence: 0.86
