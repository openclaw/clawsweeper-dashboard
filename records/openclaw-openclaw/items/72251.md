## Summary

Keep open: the earlier Greptile counter/reset and EOF-classifier concerns appear addressed, but the branch still misses the current live session and cron run-log paths that cause the reported backup race, and the JSON/docs/changelog follow-through is incomplete.

## What This Changes

This PR adds backup volatile-file filtering, tar EOF retry/backoff handling, enriched backup failures, skippedVolatileCount reporting, command logger plumbing, and focused backup tests.

## Best Possible Solution

Revise the branch so the built-in volatile filter matches current storage contracts, operational notices cannot corrupt JSON callers, and the backup content/API change is documented and changeloged while leaving ENOENT deletion races and configurable excludes to their existing follow-ups.

## Reproduction Assessment

Yes. Source inspection reproduces the main blocker by checking the PR predicate against `{stateDir}/agents/main/sessions/live.jsonl` and `{stateDir}/cron/runs/job.jsonl`; current main shows those are the live append paths, but the PR rules return false.

## Solution Assessment

No, not as currently written. Retry-plus-filter is a maintainable narrow fix for the node-tar EOF race, but it must match current path contracts and preserve JSON/docs/changelog expectations before merge.
