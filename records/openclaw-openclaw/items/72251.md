## Summary

Keep this PR open: the earlier Greptile counter and EOF-classifier issues appear addressed, but the current head still misses OpenClaw's actual live session and cron log paths and needs JSON-output and documentation/changelog fixes before merge.

## What This Changes

The PR adds backup volatile-file filtering, tar EOF retry/backoff handling, enriched backup failures, skippedVolatileCount reporting, command logger plumbing, and focused backup tests.

## Best Possible Solution

Revise the branch so the built-in volatile filter matches current storage contracts, retry/skip notices cannot corrupt JSON output, and the backup content/API change is documented and changeloged while leaving ENOENT deletion races and configurable excludes to their existing follow-ups.

## Reproduction Assessment

Yes. Source inspection reproduces the remaining PR blocker by comparing the new predicate against current main's agents/<agentId>/sessions/*.jsonl and cron/runs/<jobId>.jsonl paths, which the proposed rules do not match.

## Solution Assessment

No, not as currently written. Retry-plus-filter is a narrow maintainable fix for the node-tar EOF race, but it must cover the actual live-write paths and preserve JSON/docs/changelog expectations before merge.
