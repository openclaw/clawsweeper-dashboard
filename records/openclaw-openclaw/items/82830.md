## Summary

Current OpenClaw already tears down stdio MCP process trees and reaps cached session MCP runtimes; the remaining reported problem is QClaw bundling an older OpenClaw version.

## What This Changes

The issue reports leaked stdio MCP child processes on Windows when heartbeat/session reconnects spawn new servers without terminating old ones.

## Best Possible Solution

Keep the shipped OpenClaw MCP cleanup implementation and leave the QClaw version bump to the linked QClaw packaging tracker.

Yes for the affected older releases: the report matches prior closed MCP leak reports and the pre-fix history added explicit cleanup commits. I did not run a live Windows repro against current main because source and shipped-release evidence show the cleanup path is now present.

## Solution Assessment
