## Summary

Close as cannot_reproduce: current main shows `denyCommands` is a documented exact node command-ID filter, and source inspection does not confirm the claimed `canvas.eval` payload-to-shell bypass.

## What This Changes

The issue asks for `gateway.nodes.denyCommands` to block equivalent shell execution passed through permitted command payloads rather than only exact command names.

## Best Possible Solution

Keep the documented command-ID policy boundary and use exec approvals, tool policy, or a concrete platform-specific security report for payload-level execution concerns.

## Reproduction Assessment

No for the claimed bug: source inspection confirms exact matching, but the report does not provide a concrete payload and current main does not show a `canvas.eval` path that reaches host shell execution without `system.run` or exec policy.

## Solution Assessment

No; making `denyCommands` semantically inspect arbitrary command payloads would be a new policy capability, not a narrow bug fix to the documented command-ID filter.
