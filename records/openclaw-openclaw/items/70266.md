## Summary

Keep this open: current main and v2026.6.8 still route only `ui.seamColor` into native macOS Talk Mode and render the Circle-based `TalkOrbView`, while assistant-avatar resolution is wired into Control UI/gateway avatar surfaces rather than the native Talk overlay.

## What This Changes

Request native macOS Talk Mode overlay rendering of the configured assistant avatar while preserving the current orb fallback and voice-state animation.

## Best Possible Solution

Extend `talk.config` and native macOS state to carry a safe resolved assistant avatar into `TalkOverlayView`, then render it as the circular center layer inside the existing ring/fallback visuals after maintainer approval of source, caching, and auth behavior.

## Reproduction Assessment

Yes. Source inspection shows `TalkOverlayView` always builds `TalkOrbView`, and the macOS `talk.config` path only carries `ui.seamColor`, so configured assistant avatars cannot reach the native overlay on current main.

## Solution Assessment

Unclear until maintainer UX/API direction is confirmed. The narrow maintainable direction is to reuse the existing safe avatar resolver and preserve the current orb fallback, but native delivery, caching, and auth behavior need product approval.
