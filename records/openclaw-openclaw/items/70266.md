## Summary

Keep open. Current main and the latest shipped release still render macOS Talk Mode through the generic native orb path, while the related Control UI identity work does not feed assistant avatar data into this overlay.

## What This Changes

Use the configured assistant avatar in the macOS Talk Mode overlay, preserving the current animated voice-state visuals and orb fallback.

## Best Possible Solution

Add a native macOS identity path that reuses the resolved gateway assistant identity/avatar contract, renders a safe circular avatar inside the existing Talk overlay motion, and keeps the current orb fallback.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.5.12 shows an avatar configured in assistant identity cannot reach TalkOverlayView because the native path only carries seamColorHex into TalkOrbView.

## Solution Assessment

Yes as a product direction, but not as automated cleanup. The maintainable fix is to reuse agent.identity.get or an equivalent additive gateway identity payload, then render that validated avatar in the native overlay with the existing fallback preserved.
