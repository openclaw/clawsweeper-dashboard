## Summary

Keep open. Current main still renders the macOS Talk Mode overlay as a generic orb and only carries seam color into that native path; the existing Control UI assistant-avatar work does not cover this surface.

## What This Changes

Use the configured assistant avatar in the macOS Talk Mode overlay while preserving voice-state motion and falling back to the current orb.

## Best Possible Solution

Add a macOS native identity path that fetches resolved assistant identity/avatar via the existing gateway contract, renders it inside the Talk overlay motion, and keeps the current orb fallback.

## Reproduction Assessment

Yes. Source inspection on current main shows configuring ui.assistant.avatar and opening macOS Talk Mode still reaches TalkOverlayView -> TalkOrbView with only phase, level, accent, and pause state, while Talk config/runtime only stores seamColorHex.

## Solution Assessment

Yes as a direction, but not as automated cleanup. The maintainable path is to reuse agent.identity.get and gateway avatar resolution, then render a safe circular avatar within the existing Talk overlay motion with the orb fallback preserved.
