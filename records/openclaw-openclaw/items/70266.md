## Summary

Keep open. Current main and the latest release still route macOS Talk Mode through the generic native orb path, and the closed Control UI identity work does not feed assistant avatar data into this overlay.

## What This Changes

Allow the native macOS Talk Mode overlay to render the configured assistant avatar while preserving the existing animated orb fallback.

## Best Possible Solution

Add a native macOS identity path that reuses resolved gateway assistant avatar metadata, renders a safe circular avatar inside the existing Talk overlay motion, and keeps the current orb fallback.

## Reproduction Assessment

Yes. Source inspection on current main shows a configured assistant avatar cannot reach TalkOverlayView because the native path only carries seamColorHex into TalkOrbView.

## Solution Assessment

Yes as a product direction, but not as automated cleanup. The maintainable fix is to reuse agent.identity.get or an equivalent additive gateway identity payload, then render that validated avatar in the native overlay with the existing fallback preserved.
