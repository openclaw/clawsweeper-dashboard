## What This Changes

Request to render the configured assistant avatar in the macOS Talk Mode overlay while retaining the current orb animation and fallback behavior.

## Best Possible Solution

Extend the Talk config/native state path to carry a safe resolved assistant avatar into TalkOverlayView, then render it as the circular center layer inside the existing ring/fallback visuals after maintainer approval of source, caching, and auth behavior.

## Reproduction Assessment

Yes. Source inspection shows TalkOverlayView always builds TalkOrbView, and the macOS talk.config path only carries ui.seamColor, so configured ui.assistant.avatar cannot reach the native overlay on current main.

## Solution Assessment

Unclear until maintainer UX/API direction is confirmed. The narrow maintainable direction is to reuse the existing safe avatar resolver and preserve the current orb fallback, but the native delivery, caching, and auth contract needs product approval.
