## Summary

Keep this open: current main and the latest release still render the native macOS Talk Mode overlay as the generic Circle-based orb, while assistant avatar resolution exists only in gateway/Control UI paths and is not delivered to the native Talk overlay.

## What This Changes

Request the native macOS Talk Mode overlay to render the configured assistant avatar when available while preserving the existing orb fallback and voice-state animation.

## Best Possible Solution

Add an additive Talk config/native state path that reuses the existing safe avatar resolver, passes a renderable avatar reference to macOS, and layers it inside the current Talk overlay motion/fallback visuals.

## Reproduction Assessment

Yes, source-reproducible: the native Talk overlay has no avatar input, and the talk.config path only exposes ui.seamColor to macOS. I did not run a real macOS Talk Mode session during this read-only review.

## Solution Assessment

Unclear as an implementation because no PR exists and maintainers still need to choose native avatar delivery, caching/auth, and visual layering. The best direction is to reuse the existing safe resolver rather than creating a separate macOS avatar policy.
