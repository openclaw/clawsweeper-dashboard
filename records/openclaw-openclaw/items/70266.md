## Summary

Keep open. Current main still does not pass assistant identity/avatar state into the macOS Talk Mode overlay; the related Control UI identity work at https://github.com/openclaw/openclaw/issues/22199 does not cover this native overlay surface, and this remains a narrow new UX feature rather than an obsolete cleanup item.

## What This Changes

Use the configured assistant avatar in the macOS Talk Mode overlay while preserving the existing voice-state motion and orb fallback.

## Best Possible Solution

Add a macOS Talk Mode identity path that reuses agent.identity.get, renders a safe circular avatar inside the existing overlay motion, and falls back to the current orb when no renderable avatar is available.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: configure ui.assistant.avatar and open macOS Talk Mode; the overlay still constructs TalkOrbView from phase, level, accent, and pause state while the runtime stores only seamColorHex.

## Solution Assessment

Yes. Reusing agent.identity.get is the narrowest maintainable direction because it preserves existing identity precedence, avatar validation, and avatar-source metadata instead of duplicating config resolution in the macOS client.
