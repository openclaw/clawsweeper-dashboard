## Summary

Keep this open: current main and the latest release still render the native macOS Talk Mode overlay as the generic Circle-based orb, while the existing assistant-avatar resolution is wired through gateway and Control UI surfaces rather than the native Talk overlay.

## What This Changes

Request to have native macOS Talk Mode render the configured assistant avatar in the overlay while preserving the existing animated voice-state visuals and orb fallback.

## Best Possible Solution

Add an additive Talk config/native state path that reuses the existing safe avatar resolver and renders an auth-safe circular avatar inside the current Talk overlay state visuals with the existing orb fallback.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.10 have no avatar input or rendering branch in the native Talk overlay, and talk.config only carries ui.seamColor for native UI appearance.

## Solution Assessment

Unclear because there is no PR yet and maintainers still need to choose the native avatar delivery, caching/auth, and visual layering policy. The best direction is to reuse the existing safe resolver instead of creating a separate macOS avatar policy.
