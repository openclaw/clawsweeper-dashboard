## Summary

Keep this open: current main and v2026.6.8 still only pass `ui.seamColor` into native macOS Talk Mode and render the Circle-based `TalkOrbView`; avatar resolution exists for Control UI/gateway surfaces but is not delivered to or rendered by the native Talk overlay.

## What This Changes

Request to render the configured assistant avatar in the native macOS Talk Mode overlay while preserving the existing orb animation and fallback.

## Best Possible Solution

Extend `talk.config` and macOS Talk state to carry a safe resolved assistant avatar reference, then render it as the circular center layer inside the existing ring/fallback visuals after maintainer UX and security direction.

## Reproduction Assessment

Yes. Source inspection shows `TalkOverlayView` always builds `TalkOrbView`, and the macOS `talk.config` path only carries `ui.seamColor`, so configured assistant avatars cannot reach the native overlay on current main.

## Solution Assessment

Unclear until maintainer UX/security direction is confirmed. The narrow maintainable direction is to reuse the existing safe avatar resolver and preserve the current orb fallback, but native delivery, caching, and auth behavior need product approval.
