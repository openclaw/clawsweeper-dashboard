## Summary

Keep open, but do not merge as-is: current main still has the Windows-drive-letter session path bug, while this PR’s basename rewrite can bypass existing agent/root-preservation behavior and still lacks after-fix real environment proof.

## What This Changes

The PR adds POSIX-only Windows drive-letter session path detection, rewrites those paths to the local sessions directory by basename, adds a focused session-path test, and updates the changelog.

## Best Possible Solution

Parse Windows absolute session paths with Windows semantics, preserve valid `agents/<agentId>/sessions/<file>` structure through the existing containment and fallback rules, and cover same-agent plus cross-agent Windows-path cases.

## Reproduction Assessment

Yes, at source level: on POSIX, Node treats a drive-letter `sessionFile` as non-absolute, so current main can append it under the local sessions directory. I did not run the full Windows-host/POSIX-container flow.

## Solution Assessment

No. The branch fixes the simplest malformed-path symptom but bypasses existing realpath, containment, and agent-preservation logic, so the safer fix should preserve path structure instead of collapsing to basename.
