## Summary

Keep open: the PR targets a real current-main Windows-on-POSIX session path bug, but the submitted early-return fix is not mergeable because it can collapse cross-agent transcript paths to a local basename and persist that alias.

## What This Changes

The PR adds POSIX-only Windows drive-letter detection in session path resolution and resolves Windows-authored `sessionFile` values by placing their basename under the current sessions directory.

## Best Possible Solution

Revise this PR or replace it with a narrow fix that recognizes Windows absolute paths on POSIX, preserves agent/root context through the existing resolver fallback flow, rejects degenerate path shapes, and adds focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. A focused POSIX Node check reproduces the path-classification problem, and current main uses that same `path.isAbsolute` branch before resolving the session path.

## Solution Assessment

No. The submitted basename extraction fixes the simplest local filename case, but it bypasses existing containment, symlink normalization, and cross-agent fallback behavior that current tests already protect.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve agent context for Windows session paths:** `src/config/sessions/paths.ts:211`
  - body: This early return rewrites every Windows absolute `sessionFile` to `<sessionsDir>/<basename>` before the existing sibling/extracted-agent fallback can inspect `agents/<agentId>/sessions`. If the basename exists locally, `resolveAndPersistSessionFile()` can persist the current agent's transcript path for a Windows-authored path that belonged to another agent, so the Windows-path fix needs to normalize into the existing resolver flow instead of returning the basename directly.
  - confidence: 0.89
- **[P2] Reject Windows paths without a usable filename:** `src/config/sessions/paths.ts:208-213`
  - body: When the Windows-style path ends with a slash or backslash, `extractBasename()` returns an empty string and the guard falls through to the POSIX relative-path branch, reproducing the broken concatenated path this PR is trying to avoid. Reject the degenerate shape or normalize to a validated final segment before continuing.
  - confidence: 0.82
