## Summary

Keep this PR open. Current main still mishandles Windows drive-letter `sessionFile` values on POSIX, but the proposed basename-only early return bypasses the existing realpath and cross-agent session fallback logic, so it can persist a transcript path for the wrong agent.

## What This Changes

The PR adds a POSIX-only Windows drive-letter path detector in `src/config/sessions/paths.ts` and resolves matching `sessionFile` values by extracting the leaf filename under the current sessions directory.

## Best Possible Solution

Revise this PR or replace it with a narrow fix that recognizes Windows absolute paths on POSIX, normalizes them through `path.win32` or a shared helper, preserves `agents/<agentId>/sessions/<file>` context through the existing containment/fallback logic, rejects or safely falls back for degenerate paths, and adds focused regression tests for both separator styles plus cross-agent and symlink cases.

## Reproduction Assessment

Yes. A focused POSIX Node check reproduces the current-main path classification problem, and the PR diff shows the basename-only early return that creates the cross-agent aliasing risk.

## Solution Assessment

No. The proposed approach handles the simplest filename case, but it is not the best maintainable fix because it bypasses the resolver’s existing realpath and agent-isolation protections.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve agent context for Windows session paths:** `src/config/sessions/paths.ts:211`
  - body: This early return collapses every Windows absolute `sessionFile` to `<sessionsDir>/<basename>` before the existing cross-agent fallback can inspect `agents/<agentId>/sessions`. If a Windows-authored path points at another agent and the basename exists locally, `resolveAndPersistSessionFile()` can persist the current agent's transcript path and later writes can attach to the wrong session. Normalize the Windows path into the existing containment/fallback flow instead of returning the basename directly.
  - confidence: 0.89
