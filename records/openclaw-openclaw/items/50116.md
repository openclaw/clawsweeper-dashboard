## Summary

Keep this PR open. Current main still misclassifies Windows drive-letter sessionFile paths on POSIX, so the reported bug is real, but the proposed basename-only early return would bypass the existing agent-aware containment flow and can alias another agent's transcript into the current sessions directory.

## What This Changes

The PR adds a POSIX-only Windows drive-letter detector and basename extractor to resolve Windows-authored sessionFile values under the current sessions directory.

## Best Possible Solution

Revise this PR or replace it with a narrow fix that recognizes Windows absolute paths on POSIX, normalizes them with path.win32 or a shared helper, feeds the normalized result through the existing realpath and agent-aware fallback logic, rejects degenerate path shapes, and adds regression tests for backslash, forward-slash, cross-agent, and symlink cases.

## Reproduction Assessment

Yes. A focused POSIX Node check reproduces the current-main path classification problem, and the current resolver uses that same path.isAbsolute decision before resolving the candidate.

## Solution Assessment

No. The proposed basename extraction fixes the simplest local filename case, but it is not the best maintainable solution because it discards agent/root context and bypasses containment protections; normalizing into the existing resolver flow is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve agent context for Windows session paths:** `src/config/sessions/paths.ts:211`
  - body: This early return collapses every Windows absolute sessionFile to <sessionsDir>/<basename> before the existing cross-agent fallback can inspect agents/<agentId>/sessions. If a Windows-authored path points at another agent and the basename exists locally, resolveAndPersistSessionFile() can persist the current agent's transcript path and later writes can attach to the wrong session. Normalize the Windows path into the existing containment/fallback flow instead of returning the basename directly.
  - confidence: 0.89
