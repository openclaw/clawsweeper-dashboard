## Summary

Keep open: the PR addresses a real POSIX/Windows session path gap, but the current branch introduces a P1 transcript-isolation regression and still lacks required real behavior proof.

## What This Changes

The branch detects Windows drive-letter `sessionFile` values on POSIX, resolves them by basename into the local sessions directory, adds one regression test, and updates the changelog.

## Best Possible Solution

Parse Windows absolute session paths with Windows semantics, preserve valid `agents/<agentId>/sessions/<file>` structure through the existing containment and fallback rules, and cover same-agent plus cross-agent Windows-path cases.

## Reproduction Assessment

Yes, by source inspection and a focused Node probe: POSIX `path.isAbsolute` does not recognize Windows drive-letter `sessionFile` values, and the PR's basename rewrite can route a cross-agent Windows path into the current sessions directory. I did not run the full Windows-host/POSIX-container flow.

## Solution Assessment

No. The current PR fixes the simplest malformed-path symptom but bypasses the existing realpath, containment, and agent-preservation logic, so the maintainable fix should preserve path structure instead of collapsing to basename.
