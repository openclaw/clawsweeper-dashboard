## Summary

Keep open: this is a plausible Windows/POSIX session-path bug fix, but the current branch is not mergeable because it collapses Windows absolute paths to a basename, which can break cross-agent transcript isolation, and the external PR still lacks real after-fix behavior proof.

## What This Changes

The PR adds POSIX-only Windows drive-letter sessionFile detection, rewrites matching paths into the local sessions directory by basename, and adds a focused test plus changelog entry.

## Best Possible Solution

Parse Windows absolute paths with Windows semantics, preserve valid agents/<agentId>/sessions/<file> structure through the existing containment and fallback rules, and add same-agent plus cross-agent regression coverage with real behavior proof.

## Reproduction Assessment

Yes, at source level: current main uses POSIX path.isAbsolute, and Node on Linux reports false for a C:\\... sessionFile, so the resolver can append the drive-letter path under the current sessions directory. I did not run the full Windows-host/POSIX-container flow.

## Solution Assessment

No. The branch fixes the simplest malformed-path symptom but bypasses existing realpath, containment, and agent-preservation logic, so the safer fix should preserve path structure instead of collapsing to basename.
