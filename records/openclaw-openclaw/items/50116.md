## Summary

Keep this PR open, but it is not merge-ready: it targets a real Windows/POSIX session path bug, yet the current patch can collapse another agent's Windows-authored transcript path into the active agent's sessions directory and there is no after-fix real environment proof.

## What This Changes

The PR adds a POSIX-only Windows drive-letter guard in session transcript path resolution, a regression test, and a changelog entry.

## Best Possible Solution

Parse Windows absolute paths with Windows semantics, preserve agents/<agentId>/sessions/<file> context through the existing containment and fallback resolver, and provide redacted Windows-host/POSIX-container proof.

## Reproduction Assessment

Yes, at source level: current main uses POSIX path.isAbsolute and Node on Linux reports false for a Windows drive-letter sessionFile, so the resolver can append that value under the current sessions directory. I did not run the full Windows-host/POSIX-container flow.

## Solution Assessment

No. The current PR fixes the simplest same-agent basename case but bypasses the resolver's existing realpath, cross-agent, and structural fallback behavior; the safer fix should parse Windows paths and feed a valid translated path through those existing rules.
