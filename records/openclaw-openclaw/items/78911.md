## Summary

Keep open: current main and the latest release still omit the requested compaction paths from the agent-facing gateway config mutation allowlist, and the open linked PR is the right implementation candidate to review rather than closing this issue early.

## What This Changes

The issue requests making documented `agents.defaults.compaction` tuning fields writable through agent-driven `gateway config.patch` / `config.apply`, with a separate sessions-history archive fallback idea.

## Best Possible Solution

Review and land or close the linked fix PR for the narrow compaction allowlist change, then track the sessions-history archive fallback separately if maintainers want that behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows the requested compaction leaf paths are schema-backed but not matched by `ALLOWED_GATEWAY_CONFIG_PATHS`, so changing them through the agent-facing gateway tool reaches the protected-path error path; the `agents.defaults.model.fallbacks` portion is already allowed by the `agents.defaults.model` prefix.

## Solution Assessment

Yes for the compaction half, subject to maintainer approval: the linked PR's exact-path allowlist plus negative guard coverage is narrower than a broad advanced override. The sessions-history fallback should be split into its own issue or PR rather than solved here.
