## Summary

Keep open: current main already appears to allow `agents.defaults.model.fallbacks` through the existing `agents.defaults.model` allowlist, but the requested compaction fields remain outside the agent-facing gateway tool allowlist by design, so this needs maintainer product/security judgment rather than cleanup closure.

## What This Changes

Request to let gateway `config.patch` change selected `agents.defaults.compaction` knobs and `agents.defaults.model.fallbacks`, with a related request for better `sessions_history` recovery after transcript rotation.

## Best Possible Solution

Have maintainers choose the safe control-plane contract, then either add a narrow allowlist for approved compaction fields with regression coverage and docs, or design an explicit advanced operator path; track the transcript-archive fallback separately if it is still wanted.

## Reproduction Assessment

Yes, from source inspection for the compaction half: current main computes changed config paths and rejects unallowlisted `agents.defaults.compaction.*` paths. The `agents.defaults.model.fallbacks` part appears already allowed by the `agents.defaults.model` prefix allowlist.

## Solution Assessment

Unclear: removing protection globally is not obviously the best fix because current security docs intentionally keep agent-driven config writes fail-closed. A narrow maintainer-approved allowlist or explicit advanced operator mode would be safer.
