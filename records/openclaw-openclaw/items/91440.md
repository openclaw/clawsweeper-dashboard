## Summary

Keep open. Current main still does not settle the central P0 report: Dashboard Stop reaches `chat.abort`, source shows abort records buffered partials but not a proven graceful in-flight tool drain, and host exec remains intentionally full/off by default unless operators enable sandboxing or approvals.

## What This Changes

Fix Dashboard Stop so aborting in-flight tool runs cannot corrupt session JSONL, and add guardrails around host exec so one agent cannot destructively access another agent's state.

## Best Possible Solution

Treat this as maintainer-owned P0 hardening: prove graceful Stop/session recovery, then choose an explicit exec/isolation policy using existing sandbox and exec approval surfaces where possible with documented upgrade behavior.

## Reproduction Assessment

Unclear. The best solution is not one small patch yet because cancellation, transcript recovery, and host exec isolation/defaults cross session-state, security, and compatibility policy boundaries.
