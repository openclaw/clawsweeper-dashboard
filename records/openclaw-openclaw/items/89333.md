## Best Possible Solution

Keep the PR open and adjust it so `name` and `sortKey` are normalized before sorting, while `executor` is read only after availability confirms a descriptor is visible.

## Reproduction Assessment

Yes from source: current main directly re-reads descriptor fields during sorting, duplicate checks, and missing-executor reporting. The PR body also records red proof that the new regression tests fail when the planner and diagnostics changes are reverted.

## Solution Assessment

No as written: snapshotting descriptor names and sort keys at the planner boundary is the right shape, but executor should remain a visible-tool-only read so unavailable descriptors keep the existing hidden-tool behavior.
