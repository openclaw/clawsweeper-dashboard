Full review comments:

- **[P2] Keep Gemma tool text buffered until promotion:** `src/plugin-sdk/tool-payload.ts:168-185`
  - body: Adding Gemma parsing here makes final-message promotion possible, but LM Studio still flushes streamed text unless the buffered prefix looks like bracket or Harmony syntax. A response starting with `<|tool_call>` can therefore be delivered as raw text before the final `done` event can replace it with tool-call events.
  - confidence: 0.92
- **[P2] Parse each Gemma tool call independently:** `src/plugin-sdk/tool-payload.ts:309`
  - body: Using the last Gemma closing marker consumes through the final call in the remaining text. Gemma allows multiple calls concatenated without separators, while `parseStandalonePlainTextToolCallBlocks` expects each parser call to return the current block end so it can advance to the next call.
  - confidence: 0.9
- **[P2] Remove the production any from the parser:** `src/plugin-sdk/tool-payload.ts:323`
  - body: This production file is covered by the repo's `typescript/no-explicit-any: error` rule, and the live PR already has a failed lint check. Keep the match value typed as `string | undefined` or `unknown` and narrow it before assignment.
  - confidence: 0.95
