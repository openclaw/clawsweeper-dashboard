## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Parse Gemma argument values with the documented rules:** `src/plugin-sdk/tool-payload.ts:319-323`
  - body: The new regex only unwraps marker-quoted values when they stay on one line and leaves unquoted scalars as strings. Gemma calls can carry multiline string arguments and typed scalar values, so tools with content, numeric, or boolean parameters can receive malformed arguments even though the simple `ls -a` case works.
  - confidence: 0.86
- **[P2] Strip Gemma blocks from visible text:** `src/plugin-sdk/tool-payload.ts:63-64`
  - body: Adding the Gemma prefix to the parser is not enough: `stripPlainTextToolCallBlocks` still returns early unless the text looks bracketed or Harmony-shaped. Unsupported, unregistered, or unpromoted Gemma blocks can therefore remain visible to users; include the new prefix in that prefilter and cover the assistant-visible path.
  - confidence: 0.9
- **[P2] Remove the returning else branch:** `src/plugin-sdk/tool-payload.ts:306`
  - body: The current `check-lint` job fails with `Unnecessary else after return` on this parser shape. The bracket/Harmony branch already returns in all paths, so make the Gemma branch a separate `if` and drop the final `else` before merge.
  - confidence: 0.95
