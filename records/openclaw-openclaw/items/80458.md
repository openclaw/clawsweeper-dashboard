## Summary

Keep open: current main still has a source-reproducible channel-delivery path where one non-empty raw streamed assistant text wins over the phase-filtered final answer, and an open fixing PR already references this issue with closing syntax.

## What This Changes

The issue requests filtering commentary-phase assistant text out of embedded channel-delivery payloads for Codex and other phase-tagged providers.

## Best Possible Solution

Land a corrected phase-aware payload-builder fix that covers the single raw-stream case, preserves media-directive behavior, and includes focused regression coverage plus real channel proof.

## Reproduction Assessment

Yes. ## Solution Assessment

Yes, the issue’s phase-metadata direction is the right narrow fix, but the implementation should use the media-aware final-answer source when needed so attachments and directives are not lost.
