## Summary

Keep open: current main still permits the reported single-entry raw streamed assistant text path, and the open fixing PR that references this issue has not merged. The issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that phase-tagged provider commentary text can leak into channel-delivered replies because `buildEmbeddedRunPayloads` prefers raw streamed `assistantTexts` over the phase-filtered final answer.

## Best Possible Solution

Amend and land the open fix PR or an equivalent patch that uses the phase-filtered final answer whenever explicit phase metadata exists, preserves raw/media directive behavior, and adds regression coverage for the single streamed-text case.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence helper-level reproduction: one raw streamed assistant text containing commentary plus final text, combined with a phase-tagged `lastAssistant`, is returned before the phase-filtered fallback on current main.

## Solution Assessment

Yes, the phase-aware direction is the narrow maintainable fix, but the existing open PR needs to preserve the media-aware raw final-answer path before it is safe to land.
