## Summary

Keep open: current main still has a source-reproducible single streamed-text path that can deliver raw commentary, and an open linked fixing PR with closing syntax should be reviewed or superseded before this issue is closed.

## What This Changes

The issue reports that phase-tagged provider commentary can leak into channel-delivered replies because embedded run payloads prefer raw streamed assistant text over the phase-filtered final answer.

## Best Possible Solution

Land a corrected phase-aware payload fix that prefers the final-answer source for phase-tagged assistant messages while preserving media directives, then close this issue from the merged PR.

## Reproduction Assessment

Yes. A helper-level source reproduction is available: call `buildEmbeddedRunPayloads` with one non-empty raw `assistantTexts` entry containing commentary plus final text and a `lastAssistant` whose text blocks include `phase: "final_answer"`; current main falls through to the raw streamed entry.

## Solution Assessment

Yes, the phase-aware final-answer route is the narrow maintainable fix. It should be implemented through the existing media-aware final-answer source and covered by a regression for the single raw streamed-text case.
