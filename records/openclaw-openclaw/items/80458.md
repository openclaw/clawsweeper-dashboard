## Summary

Close: current main no longer lets a single raw streamed commentary-plus-final assistant text win over the canonical final answer, and the same behavior is present in the v2026.6.6 release.

## What This Changes

The issue reports channel delivery leaking phase-tagged `commentary` text because `buildEmbeddedRunPayloads` preferred raw streamed `assistantTexts` over phase-filtered final-answer content.

## Best Possible Solution

Keep the current canonical final-answer payload selection and regression coverage; no further OpenClaw core work is needed for this report.

## Reproduction Assessment

Yes. The historical failure is source-reproducible with one raw streamed `assistantTexts` entry containing commentary plus final text and structured assistant blocks carrying `commentary` and `final_answer`; current tests now lock that case down.

## Solution Assessment

Yes. The current implementation is the better fix because it prefers canonical structured final-answer text before raw accumulated assistant text while preserving existing media-directive behavior.
