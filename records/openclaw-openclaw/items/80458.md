## Summary

Keep open: current `main` still has the helper-level leak path for a single raw streamed assistant text entry, even though it partially canonicalizes multi-entry accumulated output.

## What This Changes

The issue reports that phase-tagged assistant commentary can be delivered to channels when raw streamed `assistantTexts` is preferred over the phase-filtered final answer.

## Best Possible Solution

Add a focused single-entry regression, then prefer the phase-filtered final answer whenever explicit phase metadata exists while preserving unphased providers and existing media-directive behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to call `buildEmbeddedRunPayloads` with one non-empty raw `assistantTexts` entry containing commentary plus final text and a `lastAssistant` whose text blocks include `phase: "final_answer"`; current `main` returns the raw streamed entry.

## Solution Assessment

Yes. The narrow maintainable fix is to use the already phase-filtered final answer when explicit phase metadata is present, with targeted preservation of raw media-directive behavior and unphased provider output.
