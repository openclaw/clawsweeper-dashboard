## Summary

Keep open: current main still has a source-reproducible path where one raw streamed assistant text can win before the phase-filtered final answer, and the linked fix PR is still open.

## What This Changes

The issue reports that phase-tagged provider commentary can leak to channel delivery because `buildEmbeddedRunPayloads` prefers raw streamed `assistantTexts` over final-answer extraction.

## Best Possible Solution

Land a focused payload-builder fix, likely through the linked PR after corrections, that prefers phase-filtered final-answer text for phase-tagged messages while preserving raw final-answer media directives.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence helper-level reproduction: pass one raw streamed assistant text containing commentary plus final text and a phase-tagged `lastAssistant`; current main returns the raw streamed entry before the final-answer fallback.

## Solution Assessment

Yes, the reported fix direction is the right narrow path, but it should be landed through a patch that preserves existing media-directive behavior and adds the missing single streamed-text regression coverage.
