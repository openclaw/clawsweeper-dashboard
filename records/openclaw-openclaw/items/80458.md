## Summary

Keep open: current main still has a source-reproducible channel-delivery bug, and an open closing PR already tracks the fix path, so the issue should stay open until that PR or an equivalent replacement lands.

## What This Changes

The issue reports that phase-tagged provider commentary can leak into channel-delivered replies because embedded run payload construction prefers raw streamed assistant text over the phase-filtered final answer.

## Best Possible Solution

Land a focused phase-aware payload fix that prefers the final-answer source when explicit phase metadata exists, preserves raw media-directive handling, and adds a regression for the single raw streamed-text case.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence helper-level reproduction: call `buildEmbeddedRunPayloads` with one non-empty raw `assistantTexts` entry containing commentary plus final text and a phase-tagged `lastAssistant`; current main returns the raw streamed entry because canonical fallback only activates for more than one streamed text.

## Solution Assessment

Yes in direction, but the active fix path should preserve the existing raw/media-aware final-answer source. The narrow maintainable solution is phase-aware final-answer selection plus focused regression coverage, not a new channel-specific parser.
