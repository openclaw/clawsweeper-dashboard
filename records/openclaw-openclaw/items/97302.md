## Summary

## What This Changes

Adds a throttled memory-core search-path recovery probe to restore the primary embedding provider after fallback activation, with helper tests and adjusted memory index tests.

## Best Possible Solution

Keep the search-path recovery idea, but make the probe primary-only or close every discarded probe provider, add regression coverage for persistent outage, and require redacted real Gateway recovery proof before merge.

## Reproduction Assessment

Yes, at source level: with a configured fallback and the primary still unavailable, the recovery probe can receive a fallback provider result and return false without closing it. I did not run the live Ollama/Gateway outage sequence.

## Solution Assessment

No. Search-time recovery is plausible, but the probe should avoid the normal fallback path or close discarded providers before this is the best fix.
