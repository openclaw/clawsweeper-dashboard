- **[P1] Add the missing after_agent_complete runner:** `src/auto-reply/reply/agent-runner-hooks.ts:64`
  - body: `getGlobalHookRunner()` returns `ReturnType<typeof createHookRunner>`, but this PR calls `hookRunner.runAfterAgentComplete(...)` without adding that method to the hook runner or its returned object. This will fail typechecking and would throw at runtime once a handler is registered.
  - confidence: 0.98
- **[P1] Gate reinjection as prompt injection:** `src/plugins/types.ts:1552`
  - body: The new hook can return `reinject` with `injectContext`, which rewrites a later prompt, but the PR registers `after_agent_complete` as an ordinary hook. Plugins configured with prompt injection disabled would still be able to modify prompts through this path once the runner exists.
  - confidence: 0.88
- **[P1] Require conversation access for response hooks:** `src/plugins/types.ts:1546`
  - body: The new event exposes the assistant `response`, but the hook is not classified with the existing conversation-access gated hooks. A non-bundled plugin could observe conversation content without the explicit `allowConversationAccess` opt-in required by comparable hooks.
  - confidence: 0.9
- **[P1] Run hook verdicts before streamed replies can flush:** `src/auto-reply/reply/agent-runner-hooks.ts:46-64`
  - body: The wrapper waits for `runAgentTurnWithFallback()` to finish before firing `after_agent_complete`, but block streaming can enqueue or send chunks inside that call. A later `suppress` or `reinject` result cannot reliably retract content already sent to the channel.
  - confidence: 0.88
- **[P2] Pass accountId through to agent_end hooks:** `src/agents/pi-embedded-runner/run/attempt.ts:2813`
  - body: The PR adds `accountId` to the shared agent hook context goal, but the `agent_end` call site still only adds `conversationId`. Plugins using `agent_end` for cleanup, metrics, or state updates would still be unable to distinguish two bot accounts in the same conversation.
  - confidence: 0.93
- **[P2] Use the originating target for completion hook context:** `src/auto-reply/reply/agent-runner.ts:384`
  - body: For native commands, `sessionCtx.To` can be a synthetic command target while `OriginatingTo` carries the real conversation. Building `channelKey` and `conversationId` from `sessionCtx.To` makes this new hook disagree with inbound and embedded-agent contexts for the same turn.
  - confidence: 0.84
