  - body: `runAgentTurnWithHooks` calls `hookRunner.runAfterAgentComplete(...)`, but the PR does not add that method to `createHookRunner` or its returned object. TypeScript will reject the call, and registered hooks would throw at runtime.
  - confidence: 0.98

  - body: The new event exposes assistant `response` text, but the hook is not classified with current conversation-access gated hooks. Non-bundled plugins should need `allowConversationAccess=true` before they can observe conversation content.
  - confidence: 0.91
- **[P1] Gate reinjection as prompt injection:** `src/plugins/types.ts:1551-1553`
  - body: The new hook result can set `reinject` with `injectContext`, which appends to a later prompt, but the hook is registered as an ordinary hook. Plugins with prompt injection disabled could still mutate prompts through this path.
  - confidence: 0.9
- **[P1] Run completion hooks before block replies can stream:** `src/auto-reply/reply/agent-runner-hooks.ts:47-52`
  - body: The wrapper waits for `runAgentTurnWithFallback()` to finish before calling the hook. In block-streaming setups, the inner run can already enqueue or send chunks, so later `suppress` or `reinject` cannot reliably retract or replace content.
  - confidence: 0.88
- **[P2] Pass accountId through to agent_end hooks:** `src/agents/pi-embedded-runner/run/attempt.ts:2813-2817`
  - body: The PR adds `accountId` to the shared agent hook context goal, but the `agent_end` call site still omits it. Plugins using `agent_end` for cleanup, metrics, or state updates would still be unable to distinguish multiple bot accounts.
  - confidence: 0.93
- **[P2] Use the originating target for completion context:** `src/auto-reply/reply/agent-runner.ts:380-383`
  - body: For native commands, `sessionCtx.To` can be a synthetic command target while `OriginatingTo` carries the real conversation. Building this hook's `conversationId` and `channelKey` from `sessionCtx.To` can make it disagree with other hook phases for the same turn.
