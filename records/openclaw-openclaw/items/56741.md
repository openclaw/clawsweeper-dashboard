## Summary

This PR targets a real Telegram forum-topic command bug, but it is not merge-ready because the one-line fix still appears to turn disabled topic/group overrides into visible denial replies instead of silently dropping non-owning bots, and after-fix Telegram proof is still missing.

## What This Changes

The branch changes Telegram native command authorization so slash commands evaluate topic and group `groupPolicy` overrides instead of only account-level policy.

## Best Possible Solution

Native Telegram commands should use topic/group-aware policy resolution, silently drop effective topic/group-disabled commands for non-owning forum topics, preserve intentional account-level disabled-command feedback where still intended, and land with focused tests plus redacted real Telegram proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a Telegram forum topic or inherited group override with `groupPolicy: "disabled"` while the account/default policy would otherwise allow commands, then send a native slash command; I did not run a live Telegram forum setup in this read-only review.

## Solution Assessment

No. Flipping the resolver flag is necessary, but the patch should also preserve silent disabled-topic behavior for non-owning bots and add focused regression coverage plus real Telegram proof.
