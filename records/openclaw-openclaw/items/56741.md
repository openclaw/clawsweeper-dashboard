## What This Changes

The PR changes Telegram native command authorization to evaluate topic and group `groupPolicy` overrides when resolving slash-command access.

## Best Possible Solution

Native commands should resolve effective topic/group policy, silently ignore commands when that effective topic/group policy is disabled, preserve intentional account-level disabled feedback if still desired, and land with focused regression tests plus redacted real Telegram proof.

## Reproduction Assessment

Yes, at source level. Configure a Telegram forum topic or inherited group override with `groupPolicy: "disabled"` while the account/default policy would otherwise allow commands, then send a native slash command; I did not run a live Telegram forum setup in this read-only review.

## Solution Assessment

No. Flipping the resolver flag is necessary, but the patch should also preserve silent disabled-topic behavior for non-owning bots and add focused regression coverage plus real Telegram proof.
