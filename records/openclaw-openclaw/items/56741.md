## Summary

Keep open for normal PR review: current main still has the Telegram native-command policy bypass, but this external PR lacks after-fix real Telegram proof and appears to leave the linked forum-topic noise expectation unresolved.

## What This Changes

The PR flips Telegram native-command group policy evaluation to use topic/group overrides for slash commands.

## Best Possible Solution

Make native Telegram commands use topic/group-aware policy resolution, settle the disabled-topic reply policy, add focused regression coverage for forum-topic `groupPolicy: "disabled"`, and attach real Telegram proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a forum topic or inherited group override with `groupPolicy: "disabled"` while account/default policy would allow commands, then send a native command; current main passes `useTopicAndGroupOverrides: false` and skips that override.

## Solution Assessment

Not yet. Flipping the flag is the narrow authorization fix, but the PR still needs the disabled-topic reply semantics resolved and real Telegram proof attached before it is the best merge path.
