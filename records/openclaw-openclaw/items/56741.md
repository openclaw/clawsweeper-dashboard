## What This Changes

The PR changes Telegram native command authorization so slash commands evaluate topic and group `groupPolicy` overrides in forum topics.

## Best Possible Solution

Native Telegram commands should use topic/group-aware policy resolution, silently drop effective topic/group disabled commands for non-owning forum topics, preserve intentional account-level disabled-command feedback, and land with focused tests plus redacted real Telegram proof.

## Reproduction Assessment

Yes, at source level. Configure a Telegram forum topic or inherited group override with `groupPolicy: "disabled"` while account/default policy would otherwise allow commands, then send a native slash command; current main passes topic config into the helper but disables override resolution.

## Solution Assessment

No. Flipping the resolver flag is necessary, but the patch should also preserve silent disabled-topic behavior for non-owning bots and add focused regression coverage plus real Telegram proof.
