## Summary

Keep open: current main still has the native-command policy gap and the PR's central flag flip is useful, but it is not merge-ready because it lacks required real Telegram proof and would turn disabled topic overrides into visible disabled-command replies rather than the silent drop requested by the canonical issue.

## What This Changes

The PR flips `useTopicAndGroupOverrides` to `true` in Telegram native command authorization so `/commands` evaluate topic/group `groupPolicy` overrides.

## Best Possible Solution

Make native Telegram commands use the topic/group-aware policy path, handle disabled topic/group policy with the same silent-drop semantics as regular messages unless maintainers explicitly choose otherwise, add focused regression coverage, and attach redacted live Telegram proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a forum topic or inherited group override with `groupPolicy: "disabled"` while account/default policy would allow commands, then send a native command; current main passes `useTopicAndGroupOverrides: false`.

## Solution Assessment

No. Flipping the flag is the right narrow policy-evaluation fix, but the PR should also resolve disabled-topic reply semantics and attach real Telegram proof before merge.
