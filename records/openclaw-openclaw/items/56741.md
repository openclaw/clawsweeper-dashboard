## Summary

Keep open. The PR targets a real source-reproducible Telegram authorization bug, but the one-line change is incomplete because disabled topic/group policy would still produce visible native-command replies, and the external PR still lacks after-fix real Telegram proof.

## What This Changes

The PR flips Telegram native command authorization to evaluate topic/group `groupPolicy` overrides for native and plugin slash commands.

## Best Possible Solution

Native commands should use topic/group-aware policy resolution, silently drop commands blocked by the effective disabled policy, add focused regression coverage, and include redacted real Telegram forum proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: configure a Telegram forum topic or inherited group override with `groupPolicy: "disabled"` while account/default policy would otherwise allow commands, then send a native slash command. Current main passes topic/group config into the helper but disables topic/group override resolution for native commands.

## Solution Assessment

No. Flipping the flag is necessary, but this PR still routes disabled topic/group overrides into a visible disabled-command reply and lacks focused regression coverage plus real Telegram proof.
