## Summary

Keep open. The flag change is the right policy-selection direction, but this PR still leaves disabled topic/group overrides on a visible command-reply path and lacks required real Telegram proof for the forum-topic behavior.

## What This Changes

The PR flips Telegram native command authorization to evaluate topic/group `groupPolicy` overrides in `extensions/telegram/src/bot-native-commands.ts`.

## Best Possible Solution

Native commands should use topic/group-aware policy resolution, silently drop commands blocked by effective topic/group disabled policy consistently with regular messages, add focused regression coverage, and include redacted real Telegram proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a Telegram forum topic or inherited group override with `groupPolicy: "disabled"` while account/default policy would otherwise allow commands, then send a native slash command; current `main` skips that override for native commands.

## Solution Assessment

No. Flipping the flag is necessary, but this branch still sends `Telegram group commands are disabled.` for disabled topic/group overrides where the linked report expects non-owning bots to stay silent, and it lacks focused coverage plus real Telegram proof.
