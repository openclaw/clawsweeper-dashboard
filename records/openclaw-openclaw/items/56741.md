## Summary

Keep this PR open: current main still skips topic/group `groupPolicy` overrides for Telegram native command authorization, and the one-line direction matches the shared regular-message policy path. It still needs maintainer review for release-note coverage and the linked issue's product nuance around disabled topic commands replying versus being silently dropped.

## What This Changes

This PR changes Telegram native command authorization to pass `useTopicAndGroupOverrides: true` into the shared group-policy resolver.

## Best Possible Solution

Land this change or an equivalent override-aware command-auth change with a focused regression test for topic/group `groupPolicy: "disabled"`, add the required changelog entry, and have maintainers decide whether disabled topic/group command overrides should reply or silently drop in multi-bot forum setups.

## Reproduction Assessment

Yes. Configure a Telegram forum topic or group override with `groupPolicy: "disabled"` while the account/default policy allows groups, then send `/reset` in that topic; current main resolves command auth with `useTopicAndGroupOverrides: false` and skips the override.

## Solution Assessment

Yes for the authorization bypass: using the existing resolver with `useTopicAndGroupOverrides: true` is the narrowest maintainable fix and matches regular Telegram message handling. The only unresolved solution choice is user-visible behavior for disabled override commands.
