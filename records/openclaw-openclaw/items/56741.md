## Summary

Keep open. The one-line flag change addresses the root policy lookup, but this external PR still lacks required real Telegram proof and leaves disabled-topic commands on a visible reply path while the linked report asks non-owning bots to ignore them.

## What This Changes

The PR changes Telegram native command authorization to evaluate groupPolicy with topic/group overrides by flipping `useTopicAndGroupOverrides` to true in `extensions/telegram/src/bot-native-commands.ts`.

## Best Possible Solution

Native commands should use topic/group-aware policy resolution, silently drop commands blocked by effective topic/group `groupPolicy: "disabled"` consistently with regular messages, add focused regression coverage, and include redacted real Telegram proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a Telegram forum topic or inherited group override with `groupPolicy: "disabled"` while account/default policy would otherwise allow commands, then send a native slash command; current main skips that override for native commands.

## Solution Assessment

No. Flipping the flag is the necessary policy-selection piece, but the branch still leaves disabled-topic commands on a visible reply path and lacks focused regression coverage plus required real Telegram proof.
