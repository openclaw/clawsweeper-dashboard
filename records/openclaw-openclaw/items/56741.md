## Summary

Keep open. The PR targets a real Telegram authorization bug and the flag change is directionally right, but it still produces visible disabled-command replies where the canonical report asks disabled topic commands to be ignored, and it lacks required real Telegram proof.

## What This Changes

The PR flips Telegram native command authorization to use topic/group-aware `groupPolicy` resolution for slash commands.

## Best Possible Solution

Native commands should use topic/group-aware policy resolution, drop disabled-topic command updates consistently with regular messages unless maintainers deliberately choose reply semantics, and include focused regression coverage plus redacted real Telegram proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a Telegram forum topic or inherited group override with `groupPolicy: "disabled"` while account/default policy would otherwise allow commands, then send a native slash command; current main skips that override.

## Solution Assessment

No. Flipping the flag is the right policy-selection piece, but this branch still routes disabled-topic commands into a visible reply path, lacks focused topic/group override coverage, and lacks real Telegram proof.
