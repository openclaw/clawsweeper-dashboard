## Summary

Keep open. The target bug is still present on current main and in v2026.5.22, but this PR is not merge-ready because the one-line flag flip would route effective disabled-topic policies into a visible denial reply and the PR still lacks real Telegram forum proof.

## What This Changes

The PR flips Telegram native command authorization from account/default-only groupPolicy checks to topic/group-aware checks for slash commands.

## Best Possible Solution

Native command auth should use effective topic/group policy and silently ignore commands where that effective topic/group policy is disabled, while preserving intentional account-level denial replies.

## Reproduction Assessment

Yes, at source level. Configure a Telegram forum topic or inherited group override with `groupPolicy: "disabled"` while the account/default policy would otherwise allow commands, then send a native slash command; I did not run a live Telegram forum setup in this read-only review.

## Solution Assessment

No. Flipping the resolver flag is necessary, but the patch should also preserve silent disabled-topic behavior for non-owning bots and add focused regression coverage plus real Telegram proof.
