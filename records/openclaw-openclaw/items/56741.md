## Summary

Keep open. Current main still has the underlying Telegram command-policy bug and this PR is a plausible narrow start, but it lacks real Telegram proof and appears to leave disabled-topic commands visibly replying instead of being silently ignored as requested.

## What This Changes

The PR changes Telegram native command authorization to consult topic/group `groupPolicy` overrides for slash commands in forum topics.

## Best Possible Solution

Native Telegram commands should use the topic/group-aware policy path, silently drop disabled-policy commands consistently with regular group messages unless maintainers intentionally choose reply semantics, and include focused regression coverage plus redacted real Telegram proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a forum topic or inherited group override with `groupPolicy: "disabled"` while account/default policy would otherwise allow commands, then send a native slash command; current main skips that override via `useTopicAndGroupOverrides: false`.

## Solution Assessment

No. Flipping the flag is the right policy-selection fix, but this branch should also resolve disabled-topic reply semantics, add focused coverage, and attach redacted real Telegram proof before merge.
