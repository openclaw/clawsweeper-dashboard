## Summary

Keep open: the PR targets a real current-main Telegram command authorization bug, but the branch is not ready because it still produces visible disabled-command replies for non-owning bots and has no after-fix real Telegram proof.

## What This Changes

The PR flips Telegram native command authorization to evaluate topic/group `groupPolicy` overrides by setting `useTopicAndGroupOverrides` to true.

## Best Possible Solution

Make native Telegram commands use topic/group-aware policy resolution, silently drop disabled topic/group commands for non-owning bots or record an explicit maintainer decision otherwise, add focused regression coverage, and attach real Telegram proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a forum topic or inherited group override with `groupPolicy: "disabled"` while account/default policy would allow commands, then send a native command; current main passes `useTopicAndGroupOverrides: false` and skips that override.

## Solution Assessment

No. Flipping the flag is the narrow root-cause fix, but the PR still needs disabled-command reply semantics resolved, regression coverage for topic overrides, and real Telegram proof before it is the best merge path.
