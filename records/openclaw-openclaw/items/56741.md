## Summary

Keep open. The PR targets a real Telegram native-command authorization bug, but as written it would still make non-owning bots send visible disabled-command replies in forum topics, and the external PR still lacks real Telegram proof.

## What This Changes

The PR changes Telegram native command authorization to use topic and group `groupPolicy` overrides when resolving command access.

## Best Possible Solution

Native commands should resolve effective topic/group policy, silently ignore disabled-topic commands for non-owning bots, and land with focused regression coverage plus redacted real Telegram proof.

## Reproduction Assessment

Yes, at source level. Configure a Telegram forum topic or inherited group override with `groupPolicy: "disabled"` while the account/default policy would otherwise allow commands; current main passes topic config into the helper but disables topic/group override resolution for native commands.

## Solution Assessment

No. Flipping the resolver flag is necessary, but the patch should also preserve silent disabled-topic behavior for non-owning bots and add focused regression coverage plus real Telegram proof.
