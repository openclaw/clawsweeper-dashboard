## Summary

Keep open. Current main still leaves Telegram native command authorization on the fallback-only group-policy path, while this PR is a narrow one-line fix that makes slash commands consult the same topic/group override policy used by regular Telegram messages; the remaining work is maintainer review, validation, and deciding whether override-disabled topic commands should be silently dropped rather than replying with the existing disabled-command message.

## What This Changes

The PR changes Telegram native command authorization to enable topic/group `groupPolicy` override resolution for slash commands in forum topics.

## Best Possible Solution

Land this patch or an equivalent current-layout change with a focused regression test for a forum topic or group where the override sets `groupPolicy: "disabled"` while account/default policy would otherwise allow commands. If maintainers want the linked issue's exact no-noise behavior, keep the authorization fix and additionally make topic/group override-disabled command results drop silently without creating sessions.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure a Telegram forum topic with `topicConfig.groupPolicy: "disabled"` while account/default policy allows groups, then send `/reset`; current main ignores that topic policy because the command path passes `useTopicAndGroupOverrides: false`.

## Solution Assessment

Yes for the authorization bypass: reusing the existing resolver with `useTopicAndGroupOverrides: true` is the narrowest maintainable fix and matches the regular-message path. The remaining uncertainty is whether disabled topic/group overrides should produce the existing command-disabled reply or be silently dropped for multi-bot forum setups.
