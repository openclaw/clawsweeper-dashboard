## Summary

Keep open. The flag flip is necessary but incomplete: it makes topic/group disabled policy reach native command authorization, but the existing rejection branch sends a visible disabled-command reply where the linked issue and regular-message path require non-owning bots to stay silent; the PR also lacks required real Telegram proof.

## What This Changes

The PR changes Telegram native command authorization to evaluate topic/group `groupPolicy` overrides by flipping `useTopicAndGroupOverrides` from false to true.

## Best Possible Solution

Native commands should resolve the effective topic/group policy, silently drop commands blocked by disabled topic/group policy for non-owning bots, add focused regression coverage, and include redacted real Telegram forum proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a Telegram forum topic or inherited group override with `groupPolicy: "disabled"` while account/default policy would otherwise allow commands, then send a native slash command.

## Solution Assessment

No. Flipping the resolver flag is necessary, but the final fix should also preserve silent disabled-topic behavior for non-owning bots and add focused regression coverage plus real Telegram proof.
