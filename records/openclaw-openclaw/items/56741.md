## Summary

## What This Changes

The PR flips Telegram native command authorization to evaluate topic/group `groupPolicy` overrides.

## Best Possible Solution

Native commands should use topic/group-aware policy resolution, silently drop commands blocked by effective topic/group disabled policy, add focused regression coverage, and include redacted real Telegram forum proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a Telegram forum topic or inherited group override with `groupPolicy: "disabled"` while account/default policy would otherwise allow commands, then send a native slash command.

## Solution Assessment

No. Flipping the resolver flag is necessary, but this PR should also preserve silent disabled-topic behavior for non-owning bots and add focused regression coverage plus real Telegram proof.
