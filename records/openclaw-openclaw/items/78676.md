## What This Changes

The issue requests fixing `openclaw doctor` so it emits one accurate Gateway runtime panel when system Node is too old but a supported version-manager Node is selected.

## Best Possible Solution

Doctor should emit one Gateway runtime diagnostic for this condition, preserving the selected supported Node path and only saying Node is not found when no supported daemon Node is available.

## Solution Assessment

Yes: consolidate the doctor Gateway runtime notes and gate the not-found text on absence of any selected supported daemon Node, while keeping the existing Node minimum and migration policy unchanged.
