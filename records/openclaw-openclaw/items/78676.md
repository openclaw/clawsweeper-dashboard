## Summary

Keep open: current main still contains the duplicate and misleading `Gateway runtime` diagnostic path, and two open PRs already use closing syntax for this issue, so the issue should stay open until a fix merges.

## What This Changes

`openclaw doctor` should emit one accurate `Gateway runtime` note when system Node is below 22.14 but a supported version-manager Node is found and used.

## Best Possible Solution

Review and land the best linked fix PR so doctor emits a single `Gateway runtime` diagnostic with the selected daemon Node context and only says Node is not found when no supported daemon Node is reachable.

## Reproduction Assessment

Yes, by current-source inspection: the install-plan warning can emit once with the selected version-manager Node path, then doctor can emit a contextless duplicate and the unconditional not-found note when only system Node is unsupported. The reporter also supplied before/after terminal output from the affected Ubuntu+nvm layout.

## Solution Assessment

Yes, the narrow maintainable fix is to consolidate the doctor runtime notes and gate the not-found message on absence of any supported daemon Node, while leaving the Node minimum and version-manager warning policy unchanged.
