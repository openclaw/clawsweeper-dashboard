## Summary

Keep open. Current main still lacks a shared `media.image` limit contract; only `agents.defaults.imageMaxDimensionPx` and the OpenAI-compatible chat-completions image budgets are configurable, while sanitizer, generic media, input-pixel, gateway attachment, sandbox, and ACP caps remain fixed.

## What This Changes

The issue requests a user-configurable image media limit contract covering byte limits, input-pixel caps, resize dimensions, and total request budgets across sanitizer, media, gateway, sandbox, OpenAI-compatible HTTP, ACP, and subagent paths.

## Best Possible Solution

Define a backwards-compatible, documented `media.image` limits contract with global defaults and scoped overrides, then thread resolved limits through each named image path while preserving existing aliases such as `agents.defaults.imageMaxDimensionPx`.

## Reproduction Assessment

Yes. Source inspection on current main shows `media.image.*` is absent from strict config, several named paths still use fixed caps, and the current guard test expects oversized input-pixel images to fail before resize.

## Solution Assessment

Unclear. A shared configurable image-limit contract is directionally appropriate, but schema shape, scoped precedence, compatibility aliases, and bounded resource ceilings need maintainer design before implementation.
