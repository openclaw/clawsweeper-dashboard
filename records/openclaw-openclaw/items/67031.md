## Summary

Keep open. Current main only partially addresses the report: `agents.defaults.imageMaxDimensionPx` and OpenAI-compatible chat-completion image budgets are configurable, but there is still no shared `media.image.*` contract and several sanitizer/media/sandbox limits remain fixed or split across older scoped knobs.

## What This Changes

The issue requests configurable image byte, input-pixel, dimension, and per-request limits across OpenClaw media sanitization, gateway, sandbox, ACP, and OpenAI-compatible HTTP layers.

## Best Possible Solution

Define a backwards-compatible, documented image-limit resolver with bounded global defaults and scoped overrides, preserve existing aliases, and thread resolved limits through sanitizer, media/image-ops, generic media validation, sandbox staging, ACP, gateway attachment parsing, and OpenAI-compatible HTTP paths.

## Reproduction Assessment

Yes for the configurability gap by source inspection: the strict config contract lacks `media.image.*`, and several named byte/input-pixel paths still use fixed constants or separate scoped settings on current main. No runtime reproduction was needed because this is primarily a public config and wiring request.

## Solution Assessment

Unclear. A shared configurable image-limit contract is the maintainable direction, but the schema shape, scoped precedence, compatibility aliases, and safety ceilings need maintainer design before implementation.
