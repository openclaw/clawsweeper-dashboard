## Summary

Keep open. Current main still only partially addresses the request: `agents.defaults.imageMaxDimensionPx` is documented and OpenAI-compatible endpoint image budgets are configurable, but the shared `media.image` byte/input-pixel/dimension contract is absent and several named sanitizer, media, gateway, sandbox, and ACP caps remain fixed.

## What This Changes

The issue requests configurable image byte, input-pixel, dimension, and total-byte limits across OpenClaw media sanitization layers, preferably through a shared `media.image` config block with scoped overrides.

## Best Possible Solution

Define a backwards-compatible, documented image-limit resolver with bounded global defaults, scoped overrides, and aliases for existing knobs, then thread resolved limits through the sanitizer, generic media validation, image-ops input guard, media-store/sandbox staging, ACP attachments, and OpenAI-compatible HTTP paths.

## Reproduction Assessment

Yes, from source inspection rather than a runtime repro: current main lacks the requested `media.image.*` config contract and still has fixed image byte/input-pixel caps in the named layers.

## Solution Assessment

Unclear. A shared configurable image-limit contract is the maintainable direction, but the exact schema, scoped precedence, compatibility aliases, and resource ceilings need maintainer design before implementation.
