## Summary

Keep open. Current main still only partially addresses the request: `agents.defaults.imageMaxDimensionPx` and OpenAI-compatible chat-completions image budgets are configurable, but the shared `media.image` byte/input-pixel/dimension contract is absent and several hard caps remain in sanitizer, media, gateway, sandbox, and ACP paths.

## What This Changes

Request a configurable global and scoped image-limit contract for byte size, input pixels, resize dimension, and per-request total limits across OpenClaw media layers.

## Best Possible Solution

Define a backwards-compatible, documented `media.image` limits resolver with bounded defaults, scoped overrides, and aliases for existing knobs, then thread resolved limits through each named image path.

## Reproduction Assessment

Yes. Source inspection on current main shows the proposed `media.image.*` config is absent, several fixed caps remain, and the current image input guard test expects oversized pixel inputs to fail before resize.

## Solution Assessment

Unclear. A shared configurable image-limit contract is the right direction, but the exact schema, scoped precedence, compatibility aliases, and resource ceilings need maintainer design before implementation.
