## Summary

Keep open. Current main still only partially addresses the report: `agents.defaults.imageMaxDimensionPx` and OpenAI-compatible HTTP image budgets are configurable, but the shared `media.image` contract is still absent and multiple sanitizer, media, gateway, sandbox, and ACP caps remain fixed.

## What This Changes

Requests a shared user-configurable image limit contract for byte, input-pixel, dimension, and per-request budgets across OpenClaw media sanitize layers.

## Best Possible Solution

Define a backwards-compatible, documented `media.image` limit contract with global defaults and scoped overrides, then thread the resolved limits through sanitizer, image-ops, generic media, sandbox/media-store staging, gateway attachments, ACP image reads, and OpenAI-compatible HTTP paths while preserving existing aliases.

## Reproduction Assessment

Yes. Source and test inspection on current main gives a high-confidence reproduction path: `media.image.*` is absent from the schema, an 8,000 by 4,000 image is expected to fail the input-pixel guard before resize, and gateway images above `MAX_IMAGE_BYTES` are rejected before offload.

## Solution Assessment

Unclear. A shared configurable image-limit contract is directionally appropriate, but the exact schema, defaults, scoped override precedence, and maximum resource ceilings need maintainer design before implementation.
