## Summary

Keep open. Current main still only partially addresses the report: `agents.defaults.imageMaxDimensionPx` and OpenAI-compatible HTTP image budgets are configurable, but there is no shared `media.image` contract and several sanitizer, media, gateway, sandbox, and ACP caps remain fixed.

## What This Changes

Request a documented, top-level image limit configuration contract with scoped overrides across image sanitization, media storage, gateway attachments, sandbox staging, ACP, and OpenAI-compatible HTTP paths.

## Best Possible Solution

Define a backwards-compatible, documented `media.image` limit contract with global defaults and scoped overrides, then thread resolved limits through each named image path while preserving existing aliases such as `agents.defaults.imageMaxDimensionPx`.

## Reproduction Assessment

Yes. Source inspection on current main shows `media.image.*` is absent from strict config, and the existing guard test expects an 8,000 x 4,000 image to fail the fixed input-pixel limit before resize.

## Solution Assessment

Unclear. A shared configurable image-limit contract is directionally appropriate, but the exact schema, scoped precedence, defaults, and maximum resource ceilings need maintainer design before implementation.
