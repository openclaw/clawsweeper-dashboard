## Summary

Keep open. Current main still only partially addresses the request: `agents.defaults.imageMaxDimensionPx` and OpenAI-compatible HTTP image budgets are configurable, but there is no shared `media.image.*` contract and several sanitizer, media, gateway, sandbox, and ACP caps remain fixed.

## What This Changes

Request to add configurable global and scoped image byte, input-pixel, dimension, and per-request budgets across media sanitization, gateway, sandbox, ACP, and OpenAI-compatible image paths.

## Best Possible Solution

Define a backwards-compatible, documented `media.image` limit contract with global defaults and scoped overrides, then thread the resolved limits through sanitizer, image-ops, generic media, sandbox staging, gateway attachments, ACP reads, and OpenAI-compatible HTTP paths while preserving existing aliases.

## Reproduction Assessment

Yes. Source inspection on current main shows `media.image.*` is absent from the config/schema, an 8,000 by 4,000 image is expected to fail the input-pixel guard before resize, and gateway images above `MAX_IMAGE_BYTES` are rejected before offload.

## Solution Assessment

Unclear. A shared configurable image-limit contract is directionally appropriate, but the exact schema, defaults, scoped override precedence, and maximum resource ceilings need maintainer design before a narrow implementation is safe.
