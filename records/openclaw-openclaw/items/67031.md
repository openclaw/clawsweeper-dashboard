## Summary

Keep open. Current main still lacks the shared `media.image.*` config contract and several named layers still use fixed caps; existing progress is limited to `agents.defaults.imageMaxDimensionPx` and scoped OpenAI-compatible endpoint image budgets.

## What This Changes

Request a user-configurable image limit contract for byte caps, input-pixel caps, resize dimensions, and per-request budgets across sanitizer, media, gateway, sandbox, ACP, and OpenAI-compatible paths.

## Best Possible Solution

Define a backwards-compatible, documented image-limit resolver with bounded global defaults and scoped overrides, preserve existing aliases, and thread resolved limits through sanitizer, media/image-ops, generic media validation, sandbox staging, ACP, gateway attachment parsing, and OpenAI-compatible HTTP paths.

## Reproduction Assessment

Yes. Source inspection is enough: `media.image.*` is absent from the strict config schema and the named image byte/input-pixel paths still read fixed constants on current main.

## Solution Assessment

Unclear. A shared configurable image-limit contract is the maintainable direction, but the exact schema, precedence, aliases, and resource ceilings need maintainer design before implementation.
