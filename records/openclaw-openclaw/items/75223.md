## What This Changes

The PR updates CLI model status and probe target construction to load auth profiles with scoped external CLI discovery and adds regression tests for those paths.

## Best Possible Solution

Yes. Current main provides a source-level reproduction: plain `models status` loads the auth store through `ensureAuthProfileStoreWithoutExternalProfiles`, while the documented credential semantics require scoped external CLI discovery for runtime-only CLI credentials.

## Solution Assessment

Yes, with a current-main refresh. Passing config-derived external CLI discovery into the status auth-store load is the narrowest maintainable fix; the probe-side work should align with main's newer `externalCliDiscoveryScoped` helper instead of reintroducing older plumbing.
