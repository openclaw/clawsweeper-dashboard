## Summary

Keep open: current main has already fixed the probe-target half, but plain `models status` still loads the auth store without external CLI discovery scope, so the PR remains relevant and needs maintainer review/refresh before merge.

## What This Changes

The PR scopes `models status` and `models status --probe` auth-store loading for external CLI credential discovery and adds focused regression coverage for both CLI paths.

## Best Possible Solution

Land a refreshed narrow status fix that uses config-derived external CLI discovery with Keychain prompting disabled, while preserving current main's already-scoped probe helper and adding regression coverage.

## Reproduction Assessment

Yes. ## Solution Assessment

Yes, with a current-main refresh. The maintainable fix is to align plain CLI status with the existing gateway config-status discovery helper and keep probe target construction on the current scoped discovery API.
