## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Preserve raw-mode cache redaction when resolving config:** `src/plugins/runtime/load-context.ts:90-93`
  - body: This resolves `${ENV_VAR}` values into `context.config`, but `buildPluginRuntimeLoadOptions()` still leaves `resolveRawConfigEnvVars` unset. The loader only disables caching and redacts plugin entry config from cache keys when that flag is true; otherwise a resolved `plugins.entries.<id>.config.apiKey` can be serialized into the registry cache key and printed by `PluginLoadReentryError`. Keep the loader raw-mode signal flowing or add equivalent redaction/no-cache behavior for context-resolved raw configs.
  - confidence: 0.9
- **[P2] Wire the raw-env opt-in into a real load path:** `src/plugins/runtime/runtime-registry-loader.ts:134-135`
  - body: Adding `resolveRawConfigEnvVars` to `ensurePluginRegistryLoaded` only creates an opt-in; the inspected production CLI, gateway, and plugin runtime callers never set it, and the gateway startup test now asserts the loader flag stays off. That means the normal runtime/gateway behavior described in the PR remains unproven and likely unchanged unless the actual raw-config entry point passes this option.
  - confidence: 0.84
