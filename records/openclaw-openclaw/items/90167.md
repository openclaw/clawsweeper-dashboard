## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve already-resolved plugin config values:** `src/plugins/runtime/load-context.ts:76-78`
  - body: `resolvePluginRuntimeLoadContext()` defaults `rawConfig` to `getRuntimeConfig()`, which config IO has already passed through env substitution. Running `resolveConfigEnvVars()` here again means a stored literal produced from `$${PIONEER_API_KEY}` can become the real env value before plugin handoff, regressing the single-pass contract from https://github.com/openclaw/openclaw/pull/88211. Please keep this path on already-resolved runtime config, or prove and test that only raw/source inputs are re-resolved.
  - confidence: 0.92
- **[P1] Avoid raw env resolution on prepared gateway configs:** `src/gateway/server-plugins.ts:722`
  - body: This gateway call receives `config: resolvedConfig`, which is prepared runtime config from startup, while `activationSourceConfig` carries the source-side view. Setting `resolveRawConfigEnvVars: true` makes the loader treat that prepared runtime config as raw and re-run substitution, so escaped placeholders in existing gateway plugin configs can be converted during startup. Preserve the source-vs-runtime boundary before enabling raw resolution here.
  - confidence: 0.89
