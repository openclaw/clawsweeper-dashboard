## Summary

Keep open. Current main still performs an LM Studio manual preload through `/api/v1/models/load` before LM Studio chat inference, while LM Studio's own JIT/TTL/Auto-Evict docs distinguish JIT-loaded models from manually loaded models.

## What This Changes

The issue asks for LM Studio provider behavior that allows native JIT model loading/unloading instead of always sending LM Studio's manual load API call.

## Best Possible Solution

Add a provider-owned, default-preserving LM Studio preload/JIT mode so users can skip manual `/api/v1/models/load` for chat inference while keeping the current auto-preload behavior available.

## Reproduction Assessment

Yes. The reporter provided deterministic LM Studio 0.4.12 steps, and current main has a focused static reproduction: the LM Studio stream wrapper calls `ensureLmstudioModelLoaded`, whose tests and source show the manual load endpoint is used before inference when the model is not already loaded.

## Solution Assessment

Yes, the narrowest maintainable fix is a default-on preload setting with an opt-out for native JIT/Auto-Evict users; removing preload globally would risk regressing users who depend on OpenClaw loading LM Studio models for them.
