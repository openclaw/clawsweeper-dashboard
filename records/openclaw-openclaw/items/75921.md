## Summary

Keep open. Current main still manually preloads LM Studio models through `/api/v1/models/load`, the report has a concrete JIT/TTL reproduction path, and two open PRs already target this issue with closing syntax.

## What This Changes

The issue asks OpenClaw to let LM Studio JIT loading/unloading work by avoiding the manual model-load API when users opt into that behavior.

## Best Possible Solution

Add a documented LM Studio provider-local opt-out that defaults to the current preload behavior, skips the native load call for both inference and embedding warmup when enabled, and lands through one consolidated PR with schema/help/docs/tests/changelog coverage.

## Reproduction Assessment

Yes. The issue gives detailed LM Studio JIT/TTL steps, and current main deterministically calls the manual load endpoint before LM Studio inference and embedding warmup.

## Solution Assessment

Yes. A provider-owned opt-out with the current behavior preserved by default is the narrowest maintainable fix; changing core routing or disabling preload globally would risk users who rely on explicit loading.
