## What This Changes

The PR adds an assistant-response acknowledgement option for queued subagent completion wakes and wires it into the direct active requester completion path with focused regression expectations.

## Best Possible Solution

Land a fix that applies one consumed-or-fallback acknowledgement invariant to both direct active wake and completion fallback steering, with a regression for direct-primary failure followed by steer fallback and redacted real runtime proof.

## Reproduction Assessment

Yes. The source path is clear: completion dispatch can fail direct delivery, enter fallback steering, queue with only transcript-commit proof, and map that queued outcome to delivered before any assistant consumption event.

## Solution Assessment

No. The assistant-acknowledgement helper is a good fix shape for the direct wake, but applying it only there is too narrow because completion fallback steering still has the original acknowledgement boundary.
