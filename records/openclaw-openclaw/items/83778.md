## Summary

Current main and the latest stable release already fix the reported cron announce false-success path by requiring concrete message-tool target evidence before suppressing direct delivery fallback.

## What This Changes

The issue reports cron announce jobs with tools enabled marking delivery successful while sending or falling back to the wrong destination instead of the configured group.

## Best Possible Solution

Keep the v2026.6.8/current-main behavior: cron announce only treats a message-tool send as satisfying the configured target when concrete target metadata proves it.

## Reproduction Assessment

Yes. Source inspection and the merged regression test cover the failing shape: didSendViaMessageTool without concrete target metadata no longer satisfies cron announce delivery.

## Solution Assessment

Yes. The merged fix is the narrow owner-boundary solution because it preserves tools while requiring target proof before cron suppresses direct fallback.
