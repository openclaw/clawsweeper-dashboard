## Summary

Keep open. The report is recent, concrete, and still conflicts with the documented cron announce contract: an explicit WhatsApp group announce target should either receive the fallback delivery or the run should report a delivery failure, not resolve to an owner DM with delivered=true.

## What This Changes

Cron announce jobs with tools enabled reportedly send or resolve scheduled WhatsApp group output to the owner DM after a failed message-tool attempt.

## Best Possible Solution

Cron announce should treat explicit `delivery.channel` and `delivery.to` as authoritative for fallback delivery, and failed or unverified message-tool sends should not rewrite the resolved target or mark the run delivered.

## Reproduction Assessment

No high-confidence live current-main reproduction yet. The issue has concrete CLI steps and current source has the relevant message-tool/direct-fallback path, but the exact WhatsApp group-to-DM target rewrite still needs a focused repro.

## Solution Assessment

Unclear until the target rewrite is reproduced. The likely best fix is a bounded cron/WhatsApp target-resolution or delivery-trace correction, not disabling tools for announce jobs.
