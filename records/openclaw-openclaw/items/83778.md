## Summary

Keep open. The report is recent and concrete, and current docs still define announce delivery as targeting the configured route; current source has adjacent message-tool/direct-fallback handling but no focused current-main proof that the WhatsApp group-to-DM misroute is fixed.

## What This Changes

Cron announce jobs targeting a WhatsApp group should deliver to the configured group even when tools are enabled, or report delivery failure instead of falling back to the owner DM.

## Best Possible Solution

Make explicit cron announce targets authoritative: failed or unverified message-tool sends should fall back to the configured WhatsApp group JID, and the run should report not-delivered if that direct send cannot be proven.

## Reproduction Assessment

No high-confidence current-main reproduction yet. The issue has concrete CLI steps and current source shows the relevant message-tool/fallback path, but this read-only pass did not prove the live WhatsApp group-to-DM resolved-target rewrite.

## Solution Assessment

Unclear until the exact failure mode is isolated. The likely fix is a bounded cron delivery/target-resolution regression fix, not disabling tools for all announce jobs.
