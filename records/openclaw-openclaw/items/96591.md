The PR adds `service.instance.id` to diagnostics-otel resources using `OTEL_SERVICE_INSTANCE_ID` with a hostname fallback and adds a unit test for that env override.

## Best Possible Solution

Implement `service.instance.id` by preserving the standard `OTEL_RESOURCE_ATTRIBUTES` override path and using an opaque unique fallback or documented maintainer-approved config surface, with focused tests, docs, and real collector/export proof.
