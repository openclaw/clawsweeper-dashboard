## Summary

Keep open. Current main still has the reported promotion path: channels without a `namedAccountPromotionKeys` surface move shared policy/default keys into `accounts.default`, while Slack and Discord named accounts are documented and tested to inherit those defaults from the channel root only.

## What This Changes

The issue requests fixing multi-account channel promotion so shared defaults such as `allowFrom`, `groupPolicy`, and `defaultTo` are not stripped from named accounts for channels without `namedAccountPromotionKeys`.

## Best Possible Solution

Preserve root policy and delivery defaults for channels whose named accounts inherit from the channel root, while keeping explicit `accounts.default` shared-default behavior for channels that intentionally support it, with focused regression coverage for Slack, Discord, Telegram, Matrix, WhatsApp, and Google Chat.

## Reproduction Assessment

Yes. Source inspection on current `origin/main` gives a high-confidence path: a Slack or Discord config with named accounts and root `allowFrom` is promoted into `accounts.default`, then named accounts no longer inherit it because tests and docs only give named accounts the root fallback.

## Solution Assessment

Unclear. The bug is real, but the best fix needs a maintainer choice between per-channel `namedAccountPromotionKeys`, a safer generic fallback, or an explicit shared-default-account opt-in for channels that intentionally inherit from `accounts.default`.
