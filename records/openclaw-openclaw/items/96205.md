## What This Changes

The issue requests a first-class attach surface that mints session-bound, scoped, TTL/revocable gateway MCP tool grants for human-driven external harness sessions.

## Best Possible Solution

Keep this as a maintainer-owned design issue and split any work around the core grant primitive first, with explicit security review and proof that existing MCP/ACPX/plugin paths cannot cover the attach use case.
