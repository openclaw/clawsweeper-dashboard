## Summary

Keep open: current main respects `contextPruning.mode: "off"` and has the adjacent safeguard-mode Pi auto-compaction fix, but it still has no supported compaction `off` or hard-limit-only policy, and the exact Anthropic 1M/default-mode compaction storm is not proven fixed.

## What This Changes

Requests clarification and a supported way to stop or hard-limit compaction when `contextPruning` is off in an Anthropic large-context setup.

## Best Possible Solution

Decide the public compaction policy first, then implement any approved hard-disable or hard-limit-only mode across schema, runtime guards, docs, and focused regression coverage for large-context Anthropic/default-mode sessions.

## Reproduction Assessment

No high-confidence live reproduction of the exact Anthropic API-key, multi-profile, 1M-context compaction storm was established. Source inspection does confirm the remaining gap: pruning-off is respected, while current public compaction modes do not include an off or hard-limit-only policy.

## Solution Assessment

No: closing as implemented would conflate pruning with compaction and stretch the merged safeguard-only fix beyond what it covers. The maintainable path is an explicit owner decision on compaction disable or hard-limit semantics before code changes.
