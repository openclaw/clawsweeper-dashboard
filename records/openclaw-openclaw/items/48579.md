## Summary

Keep open: current main respects `contextPruning.mode: "off"` as pruning-only config, but it still has no supported compaction `off` or hard-limit-only mode, and the exact Anthropic 1M-context low-usage compaction storm is not proven fixed.

## What This Changes

The issue asks for large-context Anthropic sessions with `contextPruning.mode: "off"` to avoid low-usage compactions and for a way to disable compaction except at a hard context limit.

## Best Possible Solution

Decide the public compaction disable or hard-limit-only semantics, then implement the approved schema, type, runtime, docs, and regression coverage with large-context Anthropic proof.

## Reproduction Assessment

No high-confidence live reproduction of the exact Anthropic API-key, multi-profile, 1M-context compaction storm was established. Source inspection does confirm the remaining gap: pruning-off is respected, while current public compaction modes do not include off or hard-limit-only behavior.

## Solution Assessment

No: closing as implemented would conflate pruning with compaction and overstate the adjacent safeguard-mode fixes. The maintainable path is an explicit runtime-owner decision on compaction disable or hard-limit semantics before code changes.
