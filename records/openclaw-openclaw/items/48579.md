## Summary

Keep open: current main and v2026.6.6 still treat pruning and compaction as separate systems, and there is still no accepted hard-limit-only or off compaction mode. The related low-context compaction report at https://github.com/openclaw/openclaw/issues/86684 overlaps on trigger metadata and unexpected compaction, but it does not fully own this Anthropic/cache-TTL configuration request.

## What This Changes

The issue reports repeated low-context compactions despite `contextPruning.mode: "off"` and asks for clarification plus a way to fully disable compaction except at a hard context limit.

## Best Possible Solution

Define the compaction trigger contract, persist enough trigger metadata to distinguish overflow/threshold/manual/background causes, and add a hard-limit-only or off mode only if maintainers accept that safety tradeoff.

## Reproduction Assessment

No for the exact Anthropic API-key/cache-TTL runaway compaction path on current main. Source inspection does confirm the durable behavior that `contextPruning.mode: "off"` disables pruning only and does not disable the independent compaction system.

## Solution Assessment

No for a pruning-defaults-only fix: explicit pruning off is already respected. The maintainable solution belongs in compaction trigger accounting, trigger metadata, and a maintainer decision on any hard-limit-only config mode.
