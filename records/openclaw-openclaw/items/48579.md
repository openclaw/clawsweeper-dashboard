## Summary

Keep open: current main respects `contextPruning.mode: "off"` and has an adjacent merged safeguard-mode Pi auto-compaction fix, but it still has no supported compaction off or hard-limit-only mode and the Anthropic 1M/default-mode scenario is not proven fixed.

## What This Changes

The issue asks OpenClaw to clarify and control unexpected compactions when context pruning is disabled, including a way to disable compaction except at hard context limits.

## Best Possible Solution

Decide whether OpenClaw should support a compaction hard-disable or hard-limit-only policy, then implement it in schema, runtime guards, docs, and focused regression coverage for the Anthropic 1M-context path.

## Reproduction Assessment

Unclear: the report has concrete config and symptoms, but I did not establish a current-main live reproduction of the Anthropic multi-profile 1M-context path. Source inspection does show that pruning-off is respected and that the requested compaction-off policy is still absent.

## Solution Assessment

No: closing as implemented would be premature because current main only fixes an adjacent safeguard-mode ownership path. The maintainable solution is an explicit maintainer decision on compaction policy, possibly informed by the older unmerged off-mode PR.
