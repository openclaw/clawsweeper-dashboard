## Summary

Keep open: current main preserves explicit `contextPruning.mode: "off"`, but compaction is still a separate two-mode system with no public hard-off or hard-limit-only mode; the exact Anthropic multi-profile compaction storm still needs live proof and the related compaction-modes PR is open and not merge-ready.

## What This Changes

The issue reports aggressive compactions despite `contextPruning.mode: "off"` and requests clearer compaction triggers plus a way to fully disable or hard-limit automatic compaction.

## Best Possible Solution

Settle the compaction contract first: keep pruning and compaction separate, document that distinction, and if maintainers want disable or hard-limit behavior, add an explicit compaction mode across schema, types, docs, runtime, tests, and live Anthropic proof.

## Reproduction Assessment

No high-confidence live reproduction exists for the exact Anthropic multi-profile compaction storm. Source inspection does reproduce the contract gap: `contextPruning.mode: "off"` disables pruning only, while current compaction has no public hard-off mode.

## Solution Assessment

Unclear as an implementation path. Treating pruning-off as an implicit compaction disable would conflate two contracts; the maintainable path is an explicit compaction policy decision with documented overflow behavior and tests.
