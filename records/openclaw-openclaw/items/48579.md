## Summary

Keep open: current main respects `contextPruning.mode: "off"` as a pruning-only setting, but OpenClaw still has no supported compaction `off` or hard-limit-only mode, and the reported Anthropic 1M-context low-usage compaction storm is not proven fixed or reproduced on current main.

## What This Changes

The issue asks OpenClaw to clarify why compaction continues when `contextPruning.mode` is `off` and to provide a way to disable compaction except near a hard context limit.

## Best Possible Solution

Define an explicit compaction disable or hard-limit-only policy, then implement the approved schema, type, runtime, docs, and regression coverage with large-context Anthropic proof.

## Reproduction Assessment

No high-confidence live reproduction of the exact Anthropic API-key, multi-profile, 1M-context compaction storm was established. Source inspection does confirm the remaining gap: pruning-off is respected, while current public compaction modes do not include `off` or hard-limit-only behavior.

## Solution Assessment

Unclear until maintainers choose the semantics. The maintainable path is an explicit runtime-owner decision on compaction disable or hard-limit behavior before code changes, because this affects overflow recovery.
