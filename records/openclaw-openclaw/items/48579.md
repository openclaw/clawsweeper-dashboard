## What This Changes

Request to stop or clarify premature Anthropic compactions when pruning is disabled and to provide a hard-limit-only compaction disable path.

## Best Possible Solution

Settle the compaction policy explicitly: keep pruning and compaction distinct, add or reject a documented hard-limit-only mode, and route the Anthropic low-context compaction storm through live proof or the narrower cached-token tracker.

## Reproduction Assessment

No high-confidence live reproduction exists for the exact Anthropic multi-profile compaction storm on current main. Source inspection does reproduce the contract gap: `contextPruning.mode: "off"` disables pruning only, while compaction has no public `off` or hard-limit-only mode.

## Solution Assessment

No, treating `contextPruning.mode: "off"` as an implicit compaction disable would conflate two separate contracts. The maintainable path is an explicit compaction mode/policy decision plus live or source proof for the premature Anthropic trigger.
