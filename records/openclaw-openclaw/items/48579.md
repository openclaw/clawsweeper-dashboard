## Summary

Keep open: current main still respects explicit pruning `off`, but that only disables in-memory tool-result pruning; compaction remains a separate subsystem with no hard-off or hard-limit-only mode, no persisted trigger metadata, and related low-context compaction reports still open.

## What This Changes

The issue requests that `contextPruning.mode: "off"` prevent unexpected low-context compactions, or that OpenClaw provide a clear way to disable compaction except at a hard context limit.

## Best Possible Solution

Define and prove the compaction trigger contract across overflow accounting, preflight thresholding, and persisted trigger metadata, then add a hard-limit-only/off mode only if maintainers explicitly want that config surface.

## Reproduction Assessment

Yes for the source-level behavior: current main separates pruning from compaction and still allows compaction outside `contextPruning.mode`. The exact Anthropic API-key/cache-TTL reproduction is not live-reproduced here.

## Solution Assessment

No as a pruning-defaults-only fix: explicit pruning `off` is already respected, so the maintainable fix belongs in compaction trigger accounting, metadata, and a maintainer decision on hard-limit-only config.
