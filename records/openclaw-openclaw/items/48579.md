## Summary

Keep open: current main still treats `contextPruning.mode: "off"` as pruning-only and exposes no public compaction off or hard-limit-only mode; the exact Anthropic multi-profile compaction storm still needs live proof and the remaining fix is a compaction policy decision.

## What This Changes

The issue asks OpenClaw to explain and control aggressive compaction when context pruning is off, including a way to disable compaction except at a hard context limit.

## Best Possible Solution

Settle the compaction contract first: document the pruning/compaction separation and, if desired, add an explicit compaction hard-limit/off mode with schema, types, docs, upgrade guidance, regression tests, and live Anthropic proof.

## Reproduction Assessment

No high-confidence live reproduction exists for the exact Anthropic multi-profile compaction storm. Source inspection does reproduce the contract gap: `contextPruning.mode: "off"` disables pruning only, while current compaction has no public hard-off mode.

## Solution Assessment

Unclear as an implementation path. Treating pruning-off as an implicit compaction disable would conflate two contracts; the maintainable path is an explicit compaction policy decision with documented overflow behavior and tests.
