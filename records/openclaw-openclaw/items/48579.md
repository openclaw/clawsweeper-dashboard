## Summary

Keep open: current main still treats `contextPruning.mode: "off"` as pruning-only, while public compaction config has no supported `off` or hard-limit-only mode. The exact Anthropic 1M-context compaction storm was not live-reproduced, but source inspection confirms the requested disable semantics are not implemented.

## What This Changes

The issue asks OpenClaw to clarify compaction triggers and provide a way to prevent unexpected compactions except at a hard context limit for large Anthropic sessions.

## Best Possible Solution

Define explicit public compaction disable or hard-limit-only semantics, then align schema, types, runtime guards, docs, and regression coverage while preserving safe overflow recovery.

## Reproduction Assessment

No high-confidence live reproduction of the exact Anthropic compaction storm is available. Source inspection does confirm the remaining gap: pruning-off is respected, but it does not disable the separate compaction engine and no public compaction-off mode exists.

## Solution Assessment

No, treating `contextPruning.mode: "off"` as an implicit compaction disable would conflate two separate contracts. The safer direction is an explicit compaction policy decision with documented overflow behavior.
