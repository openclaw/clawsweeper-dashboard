## Summary

Keep open: current main and v2026.6.6 still do not provide a compaction hard-off or hard-limit-only mode, and the low-context compaction family remains covered by active related reports rather than a merged fix.

## What This Changes

The issue asks OpenClaw to stop unexpected low-context compactions when context pruning is off, clarify Anthropic cache/profile interactions, and provide a hard-limit-only compaction behavior.

## Best Possible Solution

Define and prove the compaction trigger contract across overflow accounting, preflight thresholding, and persisted trigger metadata, then add a hard-limit-only mode only if maintainers explicitly want that config surface.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not for the exact Anthropic API-key setup on current main; the low-context compaction family is source-reproducible from current overflow/compaction paths and the newer related live report.

## Solution Assessment

Is this the best way to solve the issue? No as a pruning-defaults-only fix: contextPruning.mode off is already respected, so the maintainable solution belongs in compaction trigger accounting, metadata, and a maintainer decision on hard-limit-only config.
