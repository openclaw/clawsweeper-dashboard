## Summary

Keep open: current main and v2026.6.8 still treat context pruning and compaction as separate systems. Explicit `contextPruning.mode: "off"` is respected, but there is still no accepted hard-limit-only/off compaction mode, compaction entries still lack durable trigger-cause metadata, and the related low-context compaction reports only partially cover this Anthropic API-key/cache-TTL configuration.

## What This Changes

The issue reports compactions continuing despite context pruning being configured off and asks for clearer compaction triggers plus a way to disable compaction except at a hard context limit.

## Best Possible Solution

Define and document the compaction trigger contract, persist trigger metadata, fix any verified low-context trigger bug, and add hard-limit/off config semantics only if maintainers accept the safety tradeoff.

## Reproduction Assessment

No for the exact Anthropic API-key/cache-TTL runaway sequence on current main; this read-only review did not run a live Anthropic repro. Source inspection does confirm that `contextPruning.mode: "off"` disables pruning only and does not disable the independent compaction system.

## Solution Assessment

No for a pruning-defaults-only fix: explicit pruning off is already respected. The maintainable solution belongs in compaction trigger accounting, persisted trigger metadata, and a maintainer decision on any hard-limit-only/off config mode.
