## Summary

Keep open: current main respects `contextPruning.mode: "off"` as a pruning-only setting, but it still has no public `compaction.mode: "off"` or hard-limit-only compaction policy, so the remaining request is a real compaction contract/product decision with live Anthropic proof still missing.

## What This Changes

The issue asks for `contextPruning.mode: "off"` to stop unexpected low-usage compactions and for a way to disable compaction except near a hard context limit.

## Best Possible Solution

Settle the compaction contract first, then implement any hard-disable or hard-limit mode explicitly across config types, schema, docs, runtime overflow behavior, focused tests, and live Anthropic proof.

## Reproduction Assessment

No high-confidence live reproduction exists for the exact Anthropic multi-profile compaction storm. Source inspection does confirm the contract gap: `contextPruning.mode: "off"` disables pruning only, while current compaction has no public hard-off mode.

## Solution Assessment

Unclear as an implementation path. Treating pruning-off as an implicit compaction disable would conflate two contracts; the maintainable path is an explicit compaction policy decision with documented overflow behavior and tests.
