## What This Changes

The issue asks OpenClaw to stop unexpected low-context compactions when context pruning is off and clarify or add a hard-limit-only compaction path.

## Best Possible Solution

Define the compaction trigger contract, persist enough trigger metadata to distinguish overflow/threshold/manual/background causes, and add a hard-limit-only or off mode only if maintainers accept that safety tradeoff.

## Reproduction Assessment

Yes for the source-level behavior: current main separates pruning from compaction and still allows compaction outside contextPruning.mode. The exact Anthropic API-key/cache-TTL runaway compaction path was not live-reproduced here.

## Solution Assessment

No for a pruning-defaults-only fix: explicit pruning off is already respected. The maintainable solution belongs in compaction trigger accounting, persisted trigger metadata, and a maintainer decision on any hard-limit-only config mode.
