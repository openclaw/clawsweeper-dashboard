## What This Changes

The issue requests clarification and control for frequent low-usage compactions when Anthropic cache-ttl pruning is disabled, including a way to disable compaction except at a hard context limit.

## Best Possible Solution

Make an explicit compaction policy decision: document that pruning-off does not disable compaction, or add a public hard-disable/hard-limit compaction mode with schema, types, docs, upgrade guidance, and regression coverage.

## Reproduction Assessment

No high-confidence live reproduction exists for the exact Anthropic multi-profile compaction storm. Source inspection does confirm the remaining product gap: pruning-off is respected, compaction remains separate, and no public compaction-off or hard-limit-only mode exists.

## Solution Assessment

No, treating `contextPruning.mode: "off"` as an implicit compaction disable would conflate two separate contracts. The maintainable path is an explicit compaction policy decision with documented overflow behavior.
