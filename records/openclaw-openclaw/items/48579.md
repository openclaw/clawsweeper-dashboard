## Summary

Keep open. Current main confirms that pruning and compaction are separate and that explicit `contextPruning.mode: "off"` is respected, but OpenClaw still has no public `compaction.mode: "off"` or hard-limit-only policy, and the reported Anthropic 1M-context early-compaction scenario has not been proven fixed.

## What This Changes

The issue asks for aggressive compactions to stop when pruning is disabled, and for a supported way to disable compaction except at a hard context limit.

## Best Possible Solution

Decide and implement or explicitly reject a supported compaction hard-disable or hard-limit-only policy, with regression coverage for Anthropic API-key 1M-context budget propagation and Pi auto-compaction coordination.

## Reproduction Assessment

Unclear: the issue gives a concrete config and symptoms, but I did not establish a current-main live Anthropic multi-profile reproduction. Source inspection proves pruning-off behavior and the missing compaction hard-disable mode, not the full reported compaction rate.

## Solution Assessment

No: current main clarifies pruning versus compaction and includes adjacent budget fixes, but it does not satisfy the requested hard-disable or hard-limit-only behavior. The narrow maintainable path needs a maintainer decision on compaction policy before code changes.
