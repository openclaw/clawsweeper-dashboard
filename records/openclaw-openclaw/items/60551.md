## Summary

Keep open: this addresses a real reasoning-leak family, but the current PR head still has deterministic false positives that can delete user-facing fenced schema examples and ordinary first-person paragraphs, and it is missing the required changelog entry.

## What This Changes

The PR adds delivery-profile text sanitizer heuristics and tests in `src/shared/text/assistant-visible-text.ts` to strip untagged reasoning preambles and structural contamination from assistant-visible outbound text.

## Best Possible Solution

Land a narrower delivery sanitizer that strips only high-confidence untagged leak markers, preserves normal prose and code/config examples, adds focused regression tests, and leaves configurable output filtering to #45041.

## Reproduction Assessment

Yes. The PR-head regressions are source-reproducible: a fenced JSON manifest with a `schema` key matches the envelope regex, and two ordinary first-person setup lines reach the preamble strip threshold before a normal paragraph.

## Solution Assessment

No. The delivery sanitizer seam is plausible, but the current regex and scoring rules are broader than the safest maintainable fix; explicit leak markers and stronger preamble evidence are needed before merge.
