Full review comments:

- **[P1] Narrow conversational preamble scoring:** `src/shared/text/assistant-visible-text.ts:634`
  - body: The new scoring treats normal openers such as `for the record` and `now I` as suspicious and adds an extra point when two lines match. A reply starting with those two user-facing lines followed by a normal third line reaches the `>= 3` threshold in the line-split pass and returns only the suffix, so channels silently lose valid opening content. Remove conversational cues from the scored regex or require an explicit reasoning marker.
  - confidence: 0.9
- **[P1] Protect fenced schema examples from envelope stripping:** `src/shared/text/assistant-visible-text.ts:766`
  - body: This pattern removes any fenced `json` block containing `"schema"` before code-region checks run. Because the PR enables the delivery profile in `sanitizeUserFacingText` and existing Discord/WhatsApp paths already call `sanitizeAssistantVisibleText`, ordinary plugin manifest examples can disappear from outbound replies. Require an explicit leakage marker or metadata shape and add a fenced manifest pass-through test.
  - confidence: 0.91
