## Summary

Keep open: the reasoning-leak problem is real and current main does not already contain this PR's last-mile preamble cleaner, but the current patch still has deterministic over-stripping risks and lacks inspectable real behavior proof before merge.

## What This Changes

The PR adds delivery-profile heuristic stripping for leaked reasoning preambles and structural contamination in the shared assistant-visible text sanitizer, expands sanitizer tests, and removes an unused command-analysis barrel.

## Best Possible Solution

Revise toward a narrower high-confidence outbound sanitizer with pass-through coverage for normal prose, fenced code, and config examples, then add redacted real Telegram or terminal proof before merge.

## Reproduction Assessment

Yes for the PR-introduced regressions by source inspection: the PR-head regex removes fenced JSON schema blocks, and the scoring rules can strip ordinary first-person opening lines. I did not run live Telegram or Gemini commands in this read-only review.

## Solution Assessment

No. The last-mile shared sanitizer seam is plausible, but the current regex and scoring rules are broader than the narrow maintainable fix; require explicit leak markers or stronger evidence plus pass-through coverage for normal examples.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Gate schema-envelope stripping on leak markers:** `src/shared/text/assistant-visible-text.ts:765-766`
  - body: `CONTAM_ENVELOPE_RE` matches any fenced `json` block containing a `schema` key, and the remover strips those matches unconditionally. A normal answer showing an OpenClaw plugin manifest or config snippet in a fenced block will lose the example, so require leak-specific metadata instead of every schema block.
  - confidence: 0.9
- **[P2] Narrow ordinary first-person preamble cues:** `src/shared/text/assistant-visible-text.ts:633-634`
  - body: The scorer treats ordinary openings like `I need to ...`, `I should ...`, and `Need to ...` as leaked-reasoning evidence. Two such lines plus the multi-line bonus reach the strip threshold, so normal instructional replies can lose their opening paragraph before send.
  - confidence: 0.86
