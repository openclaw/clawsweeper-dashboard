## Summary

Keep open: the latest head fixes the prior `let me think` gap for prose answers, but source inspection still shows a deterministic false positive for ordinary `I should...` plus `Need to...` openings and a false negative when the visible answer starts with a fenced code block.

## What This Changes

The PR adds a delivery-profile leaked-reasoning preamble detector to the shared assistant-visible sanitizer, invokes it from `sanitizeUserFacingText`, and adds sanitizer regression tests.

## Best Possible Solution

Keep the delivery sanitizer narrow: strip concrete planning leaks before prose or fenced-code visible answers, preserve ordinary instructional prose, and lock both strip and preserve boundaries in shared and `sanitizeUserFacingText` tests.

## Reproduction Assessment

Yes. By source inspection on the PR head, `I should` plus `Need to` reaches the strip threshold and fenced-code suffixes prevent stripping a strong leaked preamble; I did not run tests because this review is read-only.

## Solution Assessment

No. Routing delivery text through the shared sanitizer is the right boundary, but the heuristic needs narrower first-person scoring and a safe fenced-code suffix path before it is the best fix.
