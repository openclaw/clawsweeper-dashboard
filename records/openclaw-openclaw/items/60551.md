## Summary

Keep open: the underlying leaked-reasoning problem is valid and current main still lacks this PR's cleaner, but the current patch has source-reproducible false-positive regressions and still lacks inspectable real behavior proof.

## What This Changes

The PR adds delivery-profile leaked reasoning-preamble and structural-contamination scrubbing to the shared visible-text sanitizer, expands sanitizer tests, and removes an unused command-analysis barrel.

## Best Possible Solution

Revise toward a narrower high-confidence outbound sanitizer with pass-through coverage for normal prose, fenced code, and config examples, plus redacted real behavior proof; keep broader configurable filters under https://github.com/openclaw/openclaw/issues/45041.

## Reproduction Assessment

Yes for the PR-introduced regressions by source inspection: the PR-head regex removes fenced JSON schema blocks, and the scoring rules can strip ordinary first-person opening lines. The underlying leak has user reports, but I did not run live or test commands in this read-only review.

## Solution Assessment

No. The last-mile sanitizer seam is plausible, but the current regex and scoring rules are broader than the narrow maintainable fix; require explicit leak markers or stronger evidence and add pass-through coverage for normal examples.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

- **[P2] Gate envelope stripping on explicit leak markers:** `src/shared/text/assistant-visible-text.ts:765-766`
  - body: `CONTAM_ENVELOPE_RE` matches any fenced `json` block containing a `schema` key, and the remover strips those matches unconditionally. A normal answer showing an OpenClaw plugin manifest or config snippet in a fenced block will lose the example, so require leak-specific metadata instead of every schema block.
  - confidence: 0.9
