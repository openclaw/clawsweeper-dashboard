## Summary

Keep open: the underlying reasoning-leak problem is valid, but this PR still introduces deterministic content-loss regressions in outbound sanitizer behavior and needs a narrower repair before merge.

## What This Changes

The PR adds delivery-profile sanitizer logic and tests to strip untagged leaked reasoning preambles and structural contamination from assistant visible text.

## Best Possible Solution

Land a narrower, code-aware delivery sanitizer that strips only high-confidence untagged reasoning preambles, preserves normal examples and prose, and leaves configurable output filtering to the #45041 product path unless maintainers choose that scope now.

## Reproduction Assessment

Yes. A focused unit test can reproduce the PR regressions by passing a fenced JSON manifest example with a schema key, or normal first-person instructional opening lines followed by a paragraph, through sanitizeAssistantVisibleText on the PR head.

## Solution Assessment

No. The delivery sanitizer seam is plausible, but the proposed broad schema-envelope scrubber and first-person scoring need tighter leak sentinels before this is the best maintainable fix.
