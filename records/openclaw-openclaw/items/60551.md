## Summary

Keep open: the PR targets a real outbound leak class, but the current branch should not merge because the sanitizer can silently remove legitimate fenced JSON schema examples and ordinary first-person prose, and the external proof gate is not satisfied.

## What This Changes

The PR adds shared visible-text sanitizer stages for leaked reasoning preambles and structural contamination, extends sanitizer tests, and removes an unused command-analysis barrel.

## Best Possible Solution

Land a narrower last-mile sanitizer only for high-confidence leak markers, preserve normal prose and code/config examples, add focused pass-through regressions, and prove the outbound behavior with redacted real logs or a Telegram recording.

## Reproduction Assessment

Yes, for the PR-introduced regressions by source inspection: the PR-head regex removes fenced JSON schema blocks, and the scoring rules can strip ordinary first-person opening lines. I did not run tests because this was a read-only checkout review.

## Solution Assessment

No. The last-mile sanitizer seam is plausible, but the current regex and scoring rules are broader than the narrow maintainable fix; require explicit leak markers or stronger evidence and add pass-through coverage for normal examples.
