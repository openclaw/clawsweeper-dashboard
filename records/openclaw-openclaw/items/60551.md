## Summary

Keep open: the PR is an active bug-fix candidate, but the latest head still introduces broad outbound text stripping that can remove legitimate user-visible content, and the external contributor has not supplied inspectable real behavior proof.

## What This Changes

The PR adds delivery-profile leaked reasoning-preamble and structural-contamination scrubbing to the shared visible-text sanitizer, expands sanitizer tests, and removes an unused command-analysis barrel.

## Best Possible Solution

Revise toward a narrower high-confidence outbound sanitizer with pass-through coverage for normal prose, fenced code, and config examples, then add redacted real behavior proof before merge.

## Reproduction Assessment

Yes for the PR-introduced regressions by source inspection: the PR-head regex removes fenced JSON schema blocks, and the scoring rules can strip ordinary first-person opening lines. I did not run live Telegram or Gemini commands in this read-only review.

## Solution Assessment

No. The last-mile sanitizer seam is plausible, but the current regex and scoring rules are broader than the narrow maintainable fix; require explicit leak markers or stronger evidence and add pass-through coverage for normal examples.
