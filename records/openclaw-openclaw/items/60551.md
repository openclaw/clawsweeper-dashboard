## Summary

Keep open. Current main still does not include this PR's plain-text reasoning-preamble cleaner, and related user reports remain open, but the proposed patch has blocking false-positive content-loss risks and overlaps with newer mainline metadata/runtime-context cleanup.

## What This Changes

This PR adds delivery-profile sanitizer logic and tests in `src/shared/text/assistant-visible-text.ts` to strip leaked reasoning preambles and structural contamination from user-visible assistant text.

## Best Possible Solution

Land a narrower, code-aware delivery sanitizer that strips only high-confidence untagged reasoning preambles, relies on the existing targeted metadata/runtime-context strip helpers for copied OpenClaw envelopes, preserves examples and code blocks, and leaves configurable output filtering tracked by #45041 unless maintainers choose that product direction now.

## Reproduction Assessment

Yes. A focused unit test can reproduce the PR regressions by passing a fenced JSON plugin-manifest example with a `schema` key, or two normal lines such as `I need to...` / `I should...` followed by a paragraph, through `sanitizeAssistantVisibleText` on the PR head.

## Solution Assessment

No. The delivery sanitizer seam is plausible, but this patch mixes broad structural scrubbing with phrase heuristics that are not specific enough; the safer fix is to narrow the preamble detector and remove or explicitly gate the structural cleanup on known OpenClaw metadata sentinels.

  - confidence: 0.9

If a normal paragraph follows, the sanitizer drops those user-facing opening lines, so common instructional phrasing needs stronger leak evidence before stripping.
  - confidence: 0.84
