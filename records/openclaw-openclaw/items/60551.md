## Summary

Keep open: the reported leak class is plausible and current main does not contain this PR's plain-preamble cleaner, but the PR head still has deterministic false-positive sanitizer regressions and lacks inspectable real behavior proof.

## What This Changes

The PR adds delivery-profile leaked-preamble and structural-contamination scrubbing to `sanitizeAssistantVisibleText`, expands sanitizer tests, and removes the unused `src/infra/command-analysis/index.ts` barrel.

## Best Possible Solution

Revise toward a narrower high-confidence outbound sanitizer with pass-through coverage for normal prose, code, and config examples, plus redacted real behavior proof; keep broader configurable filters under https://github.com/openclaw/openclaw/issues/45041.

## Reproduction Assessment

Yes for the PR-introduced regressions by source inspection: the PR-head regex removes fenced JSON schema blocks, and the scoring rules can strip ordinary first-person opening lines. The underlying leak has user reports, but I did not run live or test commands in this read-only review.

## Solution Assessment

No. The last-mile sanitizer seam is plausible, but the current regex and scoring rules are broader than the narrow maintainable fix; require explicit leak markers or stronger evidence and add pass-through coverage for normal examples.
