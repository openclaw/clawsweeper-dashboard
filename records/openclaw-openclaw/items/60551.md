## What This Changes

The PR adds leaked-reasoning preamble and structural-contamination cleanup to the shared assistant visible-text sanitizer, expands sanitizer tests, and removes an unused command-analysis barrel.

## Best Possible Solution

Land a narrower shared outbound sanitizer that only strips high-confidence leak markers, preserves normal prose and code/config examples, adds pass-through regressions, and includes redacted live or terminal proof.

## Reproduction Assessment

Yes, for the PR-introduced regressions by source inspection: a fenced JSON schema example is removed by the envelope regex, and ordinary first-person opening lines can reach the preamble strip threshold. I did not run tests because this was a read-only review.

## Solution Assessment

No. The last-mile sanitizer seam is plausible, but the current regex and scoring rules are broader than the narrow maintainable fix; they should be gated on explicit leak markers or stronger evidence.
