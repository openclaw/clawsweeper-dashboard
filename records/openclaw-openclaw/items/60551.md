## Summary

Keep open: this is a focused and still-useful outbound sanitizer bug fix, but the current head still has blocking sanitizer-boundary issues that can either suppress legitimate reply text or leave explicit leaked preambles visible before code-block answers.

## What This Changes

The PR wires the shared assistant-visible delivery sanitizer into `sanitizeUserFacingText` and adds heuristic stripping plus regression tests for leaked plain-text reasoning preambles.

## Best Possible Solution

Keep the narrow delivery-sanitizer direction, but make broad first-person cues weak unless paired with stronger leak markers and allow explicit preambles to strip before fenced visible answers while preserving the fence and body.

## Reproduction Assessment

Yes for the review findings by source inspection: `I should` plus `Need to` reaches the strip threshold, and fence-start suffixes prevent stripping a strong `Let me think` preamble. I did not run tests because this review is read-only.

## Solution Assessment

No. The current implementation is close, but the scorer and fence suffix rule need a narrower boundary before this is the safest maintainable fix.
