## Summary

Keep open: the leak family is real and current main does not implement this PR's cleaner, but the current PR head still has deterministic user-visible truncation cases and lacks the required changelog entry.

## What This Changes

Adds delivery-profile leaked-preamble and structural-contamination stripping in `src/shared/text/assistant-visible-text.ts` with shared sanitizer tests.

## Best Possible Solution

Land a narrower, code-aware delivery sanitizer for high-confidence leaked-reasoning sentinels, and leave configurable output filtering to #45041 unless maintainers intentionally expand scope.

## Reproduction Assessment

Yes. The PR-head behavior can be reproduced with focused sanitizer inputs: two first-person setup lines followed by a visible paragraph, suspicious lead-in lines followed by a `~~~` fence, or a fenced JSON schema example passed through the new delivery cleaner.

## Solution Assessment

No. The delivery sanitizer seam is plausible, but the current regex/scoring rules are broader than the safest maintainable fix; explicit leak markers, stronger preamble evidence, and regression tests are needed before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate schema-envelope stripping on leak markers:** `src/shared/text/assistant-visible-text.ts:765-766`
  - body: `CONTAM_ENVELOPE_RE` removes any fenced `json` block containing a `schema` key, and the delivery profile runs this scrubber on normal outbound replies. User-facing plugin manifest or config examples in a code fence will be deleted before send, so require an explicit leaked-metadata marker instead of every fenced schema example.
  - confidence: 0.91
- **[P2] Require stronger evidence for first-person preambles:** `src/shared/text/assistant-visible-text.ts:633-634`
  - body: The suspicious-line pattern counts ordinary openings such as `I need to ...` and `I should ...`; two such lines plus the two-line bonus reach the strip threshold before a normal paragraph. That drops legitimate setup or checklist prose from outbound replies, so these cues need a stronger leak marker or narrower scoring.
  - confidence: 0.87
- **[P2] Reject tilde fences as visible suffix starts:** `src/shared/text/assistant-visible-text.ts:688`
  - body: `looksUserFacingStart` blocks backtick fences but not `~~~` fences. After two heuristic-triggering lead-in lines, a valid tilde-fenced Markdown example can become the suffix and the opening explanation is stripped, so treat tilde fences the same as backtick fences here.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/shared/text/assistant-visible-text.ts:853-854`
  - body: This enables a user-facing outbound delivery sanitizer, but the PR head has no `CHANGELOG.md` entry for the fix. Repo policy requires a changelog entry for user-visible bug fixes before merge.
  - confidence: 0.92
