## Summary

Keep open: the PR targets a real outbound reasoning-leak family that current main still does not cover, but the latest head can still delete legitimate fenced schema examples and normal first-person prose, and it lacks the required user-facing changelog entry.

## What This Changes

The PR adds delivery-profile sanitizer stages and tests in `src/shared/text/assistant-visible-text.ts` to strip untagged reasoning preambles and structural contamination from assistant-visible text.

## Best Possible Solution

Land a narrower delivery sanitizer that strips high-confidence leaked reasoning and explicit metadata contamination while preserving ordinary prose and code examples, with regression tests and a changelog entry; leave configurable output filters to #45041 unless maintainers choose that scope now.

## Reproduction Assessment

Yes. A focused unit test against `sanitizeAssistantVisibleText` on the PR head with a fenced `json` manifest/schema block, or two normal first-person instruction lines followed by a paragraph, would deterministically show the content loss from the current heuristics.

## Solution Assessment

No. The delivery sanitizer seam is plausible, but the current regex/scoring is too broad; the safer fix is explicit contamination markers plus stronger reasoning-leak evidence before stripping.
