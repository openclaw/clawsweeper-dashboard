## Summary

Keep open: current main does not include the proposed plain-text preamble cleaner, and the linked reports show a real leak class, but this PR head still has deterministic content-loss regressions plus a missing changelog entry before merge.

## What This Changes

Adds delivery-profile sanitizer logic and unit tests in `src/shared/text/assistant-visible-text.ts` to strip untagged reasoning preambles and structural contamination from assistant-visible text.

## Best Possible Solution

Land a narrower, code-aware delivery sanitizer that strips only high-confidence untagged reasoning leaks, preserves normal examples and prose, and leaves configurable output filtering to #45041 unless maintainers choose that scope now.

## Reproduction Assessment

Yes. A focused unit test on the PR head can reproduce the PR regressions by passing a fenced JSON plugin manifest with a `schema` key, or two ordinary first-person setup lines followed by a visible paragraph, through `sanitizeAssistantVisibleText`.

## Solution Assessment

No. The delivery sanitizer seam is plausible, but the current regex and scoring rules are broader than the safest maintainable fix; explicit leak sentinels and stronger preamble evidence are needed before merge.
