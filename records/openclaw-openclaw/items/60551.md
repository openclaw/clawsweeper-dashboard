## Summary

Keep this PR open: current main still lacks an untagged reasoning-preamble cleaner and the related Telegram/Gemini reports show a real user-visible leak, but the latest PR head still has deterministic false-positive paths and needs a changelog entry before merge.

## What This Changes

The PR adds delivery-profile leaked-reasoning preamble and structural-contamination cleanup with tests in `src/shared/text/assistant-visible-text.*`, and removes the unused `src/infra/command-analysis/index.ts` barrel.

## Best Possible Solution

Land a narrower delivery sanitizer that strips only high-confidence untagged reasoning leaks, preserves normal prose and code/config examples, adds focused regression tests and a changelog entry, and leaves configurable output filters to #45041.

## Reproduction Assessment

Yes. From PR-head source, a fenced JSON schema example is removed by `CONTAM_ENVELOPE_RE`, and two ordinary first-person setup lines can score 3 and be stripped before a visible paragraph.

## Solution Assessment

No. The last-mile sanitizer seam is plausible, but the current regex and scoring rules are broader than the safest maintainable fix; explicit leak markers and stronger preamble evidence are needed.
