## What This Changes

The PR adds delivery-profile sanitizer heuristics and tests in `src/shared/text/assistant-visible-text.ts` to strip untagged reasoning preambles and structural contamination from assistant-visible outbound text.

## Best Possible Solution

Land a narrower delivery sanitizer that strips only high-confidence untagged reasoning leaks, preserves normal prose and code/config examples, adds focused regression tests, and leaves configurable output filtering to #45041.

## Reproduction Assessment

Yes. The PR-head behavior is reproducible from source by passing a fenced JSON manifest with a `schema` key, or two ordinary first-person setup lines followed by a visible paragraph, through `sanitizeAssistantVisibleText`.

## Solution Assessment

No. The delivery sanitizer seam is plausible, but the current regex and scoring rules are broader than the safest maintainable fix; explicit leak markers and stronger preamble evidence are needed before merge.
