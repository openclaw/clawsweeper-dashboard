## Summary

Keep open. Current main still produces the reported nested workspace path, and the issue is paired with an open closing PR; maintainers need to resolve that PR’s upgrade-safety concern before the issue can close.

## What This Changes

Change non-default agent workspaces derived from `agents.defaults.workspace` from nested directories to hyphen-suffixed sibling directories.

## Best Possible Solution

Choose one canonical per-agent workspace layout and land the linked PR only with explicit upgrade handling for existing nested workspaces, preferably through the repository’s doctor migration boundary.

## Reproduction Assessment

Yes. Current main deterministically takes the nested fallback branch when `agents.defaults.workspace` is configured for a non-default agent without an override, and an adjacent test asserts that exact result.

## Solution Assessment

No, not as an unconditional path flip. The hyphenated layout resolves the documentation mismatch, but the maintainable fix must also define how users with the shipped nested layout retain their workspace state during upgrade.
