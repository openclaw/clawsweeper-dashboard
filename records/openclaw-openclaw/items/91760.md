## Summary

Keep open. Current main and the latest release still resolve non-default agents under `agents.defaults.workspace/<agentId>`, while docs and setup conventions point toward `workspace-<agentId>`; the linked fix PR was closed unmerged after the compatibility/migration concern was acknowledged.

## What This Changes

The issue asks to change non-default agent workspace fallback with `agents.defaults.workspace` from a nested path to a hyphenated per-agent path.

## Best Possible Solution

Choose one workspace contract, then make core, memory-host SDK, docs, tests, and any required doctor migration agree with that contract.

## Reproduction Assessment

Yes. Source inspection of current main and `v2026.6.6` shows the exact nested fallback, and adjacent tests assert the reported `/shared-ws/main` shape.

## Solution Assessment

No, not as a direct one-line resolver change. The hyphenated path may be the desired contract, but it needs a maintainer compatibility decision and likely doctor migration before runtime readers move shipped workspace-backed state.
