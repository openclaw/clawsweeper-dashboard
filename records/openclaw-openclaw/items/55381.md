## Summary

Keep open: current main still promotes assistant text from mixed assistant-plus-tool-call messages into subagent completion output, while the newer merged progress-only fix only changes downstream task terminal metadata. The issue remains a real final-output contract decision because current tests and docs still preserve the broader latest-assistant-text behavior.

## What This Changes

The issue requests that subagent completion capture ignore assistant text from messages that also contain tool calls, except for explicit final-output content.

## Best Possible Solution

Define a subagent final-output contract that separates tool-use progress from final child results, then update the selector, frozen-result tests, timeout partial-progress behavior, and docs together while preserving the newer blocked-task metadata from PR 85110.

## Reproduction Assessment

Yes, at source level. Current code and tests show a mixed assistant-plus-non-`sessions_yield` tool-call turn is expected to become `readSubagentOutput()` text, which the lifecycle path can persist as `frozenResultText`.

## Solution Assessment

Unclear until maintainers decide the output contract. If the stricter rule is accepted, the maintainable fix belongs in `summarizeSubagentOutputHistory()` with matching completion-capture tests and docs, not only downstream task-state blocking.
