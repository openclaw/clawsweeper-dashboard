## Label Justifications

- `P2`: The issue is source-reproducible and affects subagent completion quality, but it needs semantics review rather than emergency handling.
- `impact:message-loss`: The reported path can deliver planning/tool-scaffolding text as the child completion message instead of the intended final result.
- `impact:session-state`: The bug centers on persisted `frozenResultText` and subagent completion state used later by parent sessions.
- `impact:security`: The report describes internal planning or operational text being promoted into durable output that can surface in channels or logs.

## Summary

Keep open: current main and the latest shipped tag still allow mixed assistant-plus-tool-call text to become subagent completion output, and the existing test/docs contract currently preserves that behavior rather than resolving it.

## What This Changes

The issue asks OpenClaw to stop freezing assistant text from mixed assistant/tool-call subagent turns as the final child result, except for explicit final content.

## Best Possible Solution

Define the subagent final-output contract explicitly, then align `summarizeSubagentOutputHistory()`, completion-capture tests, timeout partial-progress behavior, and subagent docs to that contract.

## Reproduction Assessment

Yes, at source level. Current code and tests show a mixed assistant-plus-non-`sessions_yield` tool-call turn is expected to become `readSubagentOutput()` text, which the lifecycle path can persist as `frozenResultText`.

## Solution Assessment

Unclear until maintainers decide the output contract. If the stricter rule is accepted, the narrow maintainable fix belongs in `summarizeSubagentOutputHistory()` with matching completion-capture tests and docs, not only downstream announce routing.
