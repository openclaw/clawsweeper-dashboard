## Summary

Keep open: current main and the latest release still allow mixed non-`sessions_yield` assistant/tool-call text to become subagent completion output, while current tests and docs encode that broader contract. This needs a maintainer contract decision before a repair PR reverses the selector behavior.

## What This Changes

The issue asks OpenClaw to avoid freezing assistant text from mixed assistant/tool-call subagent messages as the final subagent completion result.

## Best Possible Solution

Define the subagent completion-selection contract explicitly, then align the selector, frozen-completion tests, and docs so final results come from assistant-only completion output or an accepted final marker while timeout partial progress remains separate.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current tests construct a mixed assistant plus non-`sessions_yield` tool-call message and expect `readSubagentOutput()` to return the assistant text. The lifecycle path can then freeze that selected text into `frozenResultText`.

## Solution Assessment

Unclear until maintainers confirm the product contract. If the stricter final-output rule is accepted, the narrowest maintainable fix is in `summarizeSubagentOutputHistory()` plus completion-capture tests and docs, not downstream announce routing.
