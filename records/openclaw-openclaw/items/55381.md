## Summary

Keep open: current main still promotes non-`sessions_yield` mixed assistant/tool-call text into subagent output, the selection path prefers that text, and the current regression test asserts the opposite of the requested behavior. The related fix PR #55510 is closed unmerged, so there is no landed canonical fix.

## What This Changes

The issue requests that subagent completion capture ignore assistant text from messages that still contain tool calls, except for an explicitly supported final-answer marker if maintainers choose one.

## Best Possible Solution

The desired end state is a documented completion-selection contract where frozen subagent results come from assistant-only final output, or from a deliberate final marker if adopted, while tool-use planning/progress text remains separate from `frozenResultText` and parent auto-announce payloads.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists in the current unit test shape: an assistant message containing both text and a non-`sessions_yield` tool call is expected to return that text from `readSubagentOutput()`, and the frozen-result path persists whatever that selector returns.

## Solution Assessment

Unclear until maintainers confirm the product contract. If the stricter final-output rule is accepted, the narrowest maintainable fix is in `summarizeSubagentOutputHistory()` plus completion-capture tests and docs, not in downstream announce routing.
