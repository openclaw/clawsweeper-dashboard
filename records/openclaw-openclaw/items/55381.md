## Summary

Keep open: current main still promotes assistant text from mixed assistant-plus-tool-call messages into subagent completion output, and the merged progress-only fix only changes downstream task terminal metadata rather than the selector that freezes `frozenResultText`.

## What This Changes

The issue asks OpenClaw to ignore assistant text from subagent messages that still contain tool calls when selecting the final child completion result, except for explicit final-result content.

## Best Possible Solution

Define a subagent final-output contract that separates tool-use progress from final child results, then update the selector, frozen-result tests, timeout/blocked-completion behavior, and docs together.

## Reproduction Assessment

Yes, at source level. Current code and tests show a mixed assistant-plus-non-`sessions_yield` tool-call turn is expected to become `readSubagentOutput()` text, and the lifecycle path can persist captured text as the completion result.

## Solution Assessment

Unclear until maintainers decide the output contract. If the stricter rule is accepted, the maintainable fix belongs in `summarizeSubagentOutputHistory()` with matching completion-capture tests and docs, not only downstream task-state blocking.
