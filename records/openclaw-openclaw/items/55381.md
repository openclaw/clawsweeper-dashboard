## Summary

Keep open: current main still promotes mixed non-`sessions_yield` assistant/tool-call text as subagent completion output, while current tests and docs preserve that broad contract, so this needs a maintainer decision rather than cleanup closure.

## What This Changes

The issue requests that subagent completion capture ignore assistant text from messages that also contain tool calls, except for explicit final content.

## Best Possible Solution

Define the subagent final-output contract explicitly, then align `summarizeSubagentOutputHistory()`, frozen-completion tests, timeout partial-progress behavior, and public subagent docs to that contract.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: current tests construct assistant text plus a non-`sessions_yield` tool call and expect `readSubagentOutput()` to return that text, and the lifecycle path can persist that selected text as `frozenResultText`.

## Solution Assessment

Unclear until maintainers decide the product contract. If the stricter rule is accepted, the narrow fix belongs in `summarizeSubagentOutputHistory()` with matching completion-capture tests and docs, not downstream announce routing alone.
