## Summary

Keep open: current main still selects mixed non-`sessions_yield` assistant/tool-call text as subagent completion output and can freeze it for announce delivery, while current tests and docs preserve that broad contract, so this needs maintainer semantics review rather than cleanup closure.

## What This Changes

The issue asks subagent completion capture to ignore assistant text from assistant/tool-call turns unless explicit final content is present.

## Best Possible Solution

Define the subagent final-output contract explicitly, then align `summarizeSubagentOutputHistory()`, frozen-completion tests, timeout partial-progress behavior, and public subagent docs to that contract.

## Reproduction Assessment

Yes. Current source and tests provide a high-confidence reproduction path: a mixed assistant plus non-`sessions_yield` tool-call turn is expected to become `readSubagentOutput()` text, and that selected text can be persisted as `frozenResultText`.

## Solution Assessment

Unclear until maintainers decide the output contract. If the stricter rule is accepted, the narrow maintainable fix belongs in `summarizeSubagentOutputHistory()` with matching completion-capture tests and docs, not downstream announce routing alone.
