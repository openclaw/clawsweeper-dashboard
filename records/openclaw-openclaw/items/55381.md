## Summary

Keep open: current main still lets mixed non-`sessions_yield` assistant/tool-call text become subagent completion output, and current tests/docs encode that broader contract, so changing it needs maintainer confirmation rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to ignore assistant text from subagent messages that also contain tool calls, except possibly explicit final-marked content, when freezing completion results.

## Best Possible Solution

Define the subagent final-output contract explicitly, then align `summarizeSubagentOutputHistory()`, frozen-completion tests, timeout partial-progress behavior, and the public subagent docs to that contract.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current tests construct an assistant message with visible text plus a non-`sessions_yield` tool call and expect `readSubagentOutput()` to return that text. The lifecycle path can then persist the selected text as `frozenResultText`.

## Solution Assessment

Unclear until maintainers decide the product contract. If the stricter rule is accepted, the narrow fix belongs in `summarizeSubagentOutputHistory()` with matching completion-capture tests and docs, not in downstream announce routing alone.
