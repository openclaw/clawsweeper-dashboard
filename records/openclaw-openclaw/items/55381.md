## Summary

Keep open: current main and latest release still let mixed assistant-plus-tool-call text become subagent completion output, and the current test/docs contract explicitly preserves that behavior. The merged progress-only task-state fix reduces false-success bookkeeping, but it does not stop `frozenResultText` or the announce event from carrying mixed tool-use text back to the parent.

## What This Changes

The issue asks OpenClaw to ignore assistant text from subagent messages that still contain tool calls when selecting durable completion output, except for an explicit final-output marker.

## Best Possible Solution

Define an explicit subagent final-output contract that separates tool-use progress from final child results, then update the selector, frozen-result persistence tests, timeout/error partial-progress behavior, and docs together.

## Reproduction Assessment

Yes, at source level. Current main has a focused test where a mixed assistant/toolCall message returns assistant text from readSubagentOutput(), and the lifecycle path can persist that selected text into frozenResultText and announce it.

## Solution Assessment

Unclear until maintainers decide the final-output contract. If the stricter rule is accepted, the maintainable fix belongs in summarizeSubagentOutputHistory() with matching completion-capture tests and docs, not only downstream announce routing.
