## Summary

Keep open: current main and the latest release still allow mixed assistant/tool-call text to become subagent completion output, the adjacent test currently expects that behavior, and the only direct fix PR was closed unmerged. The fix shape is narrow, but it needs maintainer/security review because it changes the final-result contract for mixed tool-use turns.

## What This Changes

Issue requests that subagent completion capture avoid freezing assistant text from mixed assistant/tool-call messages as the final child result.

## Best Possible Solution

Update subagent completion selection to separate final assistant-only or explicit-final text from in-flight tool-use progress, with regression tests for mixed-turn no-final text, explicit final content, timeout partial progress, and stale prior assistant text.

## Reproduction Assessment

Yes at source level: current `readSubagentOutput()` selects mixed assistant/tool-call text as output, and the adjacent test expects that behavior. I did not run a live subagent workflow because this review is read-only.

## Solution Assessment

Unclear pending maintainer decision: ignoring mixed-turn text is the right direction for final capture, but maintainers should choose the permanent contract for legitimate mixed final text, such as explicit `<final>` content or existing phase metadata.
