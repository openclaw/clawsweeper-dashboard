## Summary

Keep open: current `main` and `v2026.5.19` still allow mixed assistant-plus-tool-call text to become subagent completion output, and the current test/docs contract preserves that behavior. The related fix PR at https://github.com/openclaw/openclaw/pull/55510 was closed unmerged, while adjacent issues like https://github.com/openclaw/openclaw/issues/83863 track broader progress-only task success rather than replacing this exact final-output contract decision.

## What This Changes

The issue asks OpenClaw to stop promoting assistant text from messages that still contain tool calls into subagent completion output, except possibly explicit final-tagged text.

## Best Possible Solution

Define an explicit subagent final-output contract that separates tool-use progress from final child results, then update the selector, completion persistence tests, timeout partial-progress behavior, and docs together.

## Reproduction Assessment

Yes, at source level. Current code and tests show a mixed assistant-plus-non-`sessions_yield` tool-call turn is expected to become `readSubagentOutput()` text, which the lifecycle path can persist as `frozenResultText`.

## Solution Assessment

Unclear until maintainers decide the output contract. If the stricter rule is accepted, the narrow maintainable fix belongs in `summarizeSubagentOutputHistory()` with matching completion-capture tests and docs, not only downstream announce routing.
