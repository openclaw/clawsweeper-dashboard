## Summary

Keep open: current main still promotes non-`sessions_yield` mixed assistant/tool-call text into subagent completion output, freezes the selected text, and can pass it into the parent announce path. The related fix PR is closed unmerged, and changing this now conflicts with current tests/docs, so it needs a maintainer contract decision before a repair PR.

## What This Changes

The issue asks subagent completion capture to ignore assistant text from messages that still include tool calls, except possibly explicit final-marker content.

## Best Possible Solution

Define the completion-selection contract explicitly, then update the output selector, frozen-completion regression tests, and subagent docs so final results come from assistant-only final output or an accepted final marker while timeout partial progress stays separate.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: the current unit test constructs an assistant message containing both text and a non-`sessions_yield` tool call and expects `readSubagentOutput()` to return that text. The lifecycle path then freezes whatever the selector captures into `frozenResultText`.

## Solution Assessment

Unclear until maintainers confirm the product contract. If the stricter final-output rule is accepted, the narrowest maintainable fix is in `summarizeSubagentOutputHistory()` plus completion-capture tests and docs, not downstream announce routing.
