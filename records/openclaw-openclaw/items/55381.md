## Summary

Keep open: current main still promotes mixed non-`sessions_yield` assistant/tool-call text as subagent completion output, and the current test/docs contract preserves that behavior, so this needs maintainer product confirmation rather than cleanup closure.

## What This Changes

The issue requests stricter subagent completion selection so mixed assistant-plus-tool-call text is ignored unless it carries explicit final-result content.

## Best Possible Solution

Define the subagent final-output contract explicitly, then align `summarizeSubagentOutputHistory()`, frozen-completion tests, timeout partial-progress behavior, and public subagent docs to that contract.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current tests construct an assistant message with visible text plus a non-`sessions_yield` tool call and expect `readSubagentOutput()` to return that text. The lifecycle path can then persist the selected text as `frozenResultText`.

## Solution Assessment

Unclear until maintainers decide the product contract. If the stricter rule is accepted, the narrow fix belongs in `summarizeSubagentOutputHistory()` with matching completion-capture tests and docs, not downstream announce routing alone.
