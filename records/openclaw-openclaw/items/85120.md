## Summary

Keep open: current `origin/main` still has the immediate-parent-only macOS ancestor detector, the reporter added redacted live LaunchAgent logs matching the failure shape, and an open closing PR now owns the proposed fix. This issue should not close before that PR lands or is replaced.

## What This Changes

The issue reports that in-band `openclaw update` from a macOS LaunchAgent-managed OpenClaw session can stop the gateway that is supervising the command before a durable handoff or refusal exists.

## Best Possible Solution

Land or revise the linked fix so nested macOS updater descendants reliably refuse before stopping LaunchAgent, or implement an equivalent launchd-owned durable handoff with visible post-restart status.

## Reproduction Assessment

No high-confidence live reproduction was run in this review. The reporter supplied redacted live macOS logs, and source inspection on `origin/main` gives a medium-confidence path: a nested macOS updater can miss a gateway grandparent because only `process.ppid` is included.

## Solution Assessment

No, current `main` is not sufficient for the reported nested macOS shape. The linked PR is a plausible narrow repair path, but the issue should remain open until that fix lands or an equivalent durable handoff is proven.
