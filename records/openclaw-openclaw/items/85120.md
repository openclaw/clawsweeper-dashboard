## Summary

Keep open: current main has an intended in-band update refusal, but the macOS ancestor detector only includes the immediate parent, so a nested OpenClaw tool/chat process can still miss the LaunchAgent gateway ancestor and reach the service-stop path.

## What This Changes

Reports that `openclaw update` launched from an in-band macOS LaunchAgent-managed OpenClaw session can stop the gateway/session supervising the update before a durable handoff owns completion.

## Best Possible Solution

Preserve the existing safety contract by making macOS nested in-band updater processes reliably refuse before stopping the LaunchAgent, or by handing off to a launchd-owned updater with durable status before any gateway stop.

## Reproduction Assessment

No live macOS repro was run here. Source inspection gives a medium-confidence reproduction path: make the LaunchAgent gateway a grandparent rather than the immediate parent of `openclaw update` on macOS, and the current ancestor guard can miss it.

## Solution Assessment

No, the current shipped guard is not sufficient for the reported macOS/manual in-band shape because non-Linux ancestor detection stops at `process.ppid`. The narrow maintainable fix is to extend that safety check for nested macOS descendants or add an equivalent durable launchd-owned handoff before stopping the gateway.

## Repair Work Prompt

Fix the macOS in-band `openclaw update` safety gap reported in https://github.com/openclaw/openclaw/issues/85120. Current code refuses package updates when the active gateway PID is in `getSelfAndAncestorPidsSync()`, but on macOS that helper only returns the current process plus immediate parent, so a gateway -> tool host -> shell -> `openclaw update` shape can miss the LaunchAgent gateway and call `service.stop(...)`. Implement a bounded fix that refuses before stopping the gateway for nested macOS descendants, or otherwise hands off to a launchd-owned updater before any stop. Preserve the existing `update.run` managed-service handoff behavior where the helper waits for the gateway parent to exit, and do not add a config flag or broad product redesign. Add regression coverage for macOS nested ancestry or an equivalent guard, update the existing update CLI refusal tests, and include a changelog entry unless maintainers decide the release note is unnecessary.
