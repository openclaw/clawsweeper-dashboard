## Summary

Keep open: current main partially improves Workboard readability, but it does not yet provide approval-specific human-attention visibility, full-title hover/focus/tap behavior, or the requested in-place detail-panel navigation; no canonical duplicate or open fix PR owns the remaining UX decision.

## What This Changes

The issue requests better dense Workboard card readability, clearer human-approval/action visibility, optional prominent labels or summary counts, and smoother detail-panel navigation.

## Best Possible Solution

Expose full card titles and human-attention state directly in dense Workboard cards and summaries using existing status, labels, diagnostics, and blocked reasons where possible, then make a deliberate decision on whether the detail drawer should become non-modal or update in place.

## Reproduction Assessment

Yes, source inspection gives a medium-confidence reproduction path: open a Workboard detail drawer and the board becomes inert, and card hover uses the generic View details title rather than the full card title. I did not run a browser repro in this read-only review.

## Solution Assessment

Unclear as a final product shape: a narrow title/attention UI improvement is plausible, but configurable prominent labels and non-modal detail navigation need maintainer UX direction. The safest implementation should avoid adding config unless maintainers decide the existing status/label/diagnostic model is insufficient.
