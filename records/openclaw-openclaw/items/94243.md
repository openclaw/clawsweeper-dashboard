## Summary

Keep open: current main has partial Workboard scanability improvements, including the earlier title-collapse fix and blocked health counts, but it still lacks a distinct human-approval/action-required signal and the requested dense-card UX direction is not fully owned by another item.

## What This Changes

Improve dense Workboard card readability and make blocked or human-approval-required cards easier to identify without opening each card.

## Best Possible Solution

Define a Workboard-owned human-attention signal and pair it with a title-first or dense-list/card presentation, coordinating with https://github.com/openclaw/openclaw/issues/93053 so scanability and configurable display work do not diverge.
