- **[P1] Keep a stream ready for the next response:** `apps/ios/Sources/Voice/RealtimeTalkRelaySession.swift:629`
  - body: `finishOutputPlaybackStream()` clears `outputContinuation` on `audioDone`, but the old `outputTask` stays non-nil until `pcmPlayer.play` returns after draining. If a follow-up response starts during that local drain, this guard returns and the later `outputContinuation?.yield(data)` is a nil no-op, dropping the first audio chunks of the reply. Start or replace a fresh stream once the server response is done, or queue the next chunks until the previous task has fully closed.
  - confidence: 0.84
