## Summary

Keep open. Current main still routes the status-bar macOS chat through the native SwiftUI `OpenClawChatView`, and that view still programmatically writes the bottom scroll target without an active native scrollbar-drag guard; I did not find a linked or searched native companion fix.

## What This Changes

The issue requests fixing a macOS companion chat hang triggered by dragging the native scrollbar thumb in the status-bar chat window.

## Best Possible Solution

Add a macOS chat scroll guard or AppKit-backed adapter that pauses bottom-follow writes while the user is actively controlling the native scroll view, then resumes only after explicit bottom return, send, or new-bottom intent.

## Reproduction Assessment

No high-confidence live reproduction was established. The reporter supplied concrete steps, and current source still exposes the same native SwiftUI scroll-position path that plausibly fights native scrollbar-thumb dragging.

## Solution Assessment
