## Summary

Keep open. The report is recent, specific, and still matches the current native macOS chat surface: current main uses SwiftUI `ScrollView`/`LazyVStack` with a bound `scrollPosition` and repeated programmatic bottom-pinning, and I found no source, test, docs, changelog, or release evidence that the scrollbar-thumb hang has been fixed.

## What This Changes

The issue reports that dragging the scrollbar thumb in the macOS companion app chat view on v2026.4.22 freezes the UI and requires force-quit.

## Best Possible Solution

Fix `OpenClawChatView` so programmatic bottom-follow does not fight active scrollbar-thumb interaction, while preserving bottom-follow for initial load, sends, appended user messages, pending runs, and streaming assistant output. If SwiftUI `scrollPosition` remains unstable during thumb dragging, prefer a macOS-specific `NSScrollView`-backed seam over broad shared-chat churn.

## Reproduction Assessment

Yes for triage: the issue gives concrete companion-app steps against v2026.4.22 on macOS 26.4.1, and current main still uses the same native SwiftUI scroll architecture. This review did not independently live-reproduce the hang on macOS.

## Solution Assessment

Not applicable as a patch review because this issue has no proposed fix. The narrowest maintainable direction is a macOS-aware guard around bottom-pinning during active user scroll interaction, with an `NSScrollView` replacement only if the SwiftUI binding remains the source of the hang.
