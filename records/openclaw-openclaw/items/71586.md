## Summary

Keep open. The report is recent and current main still routes the macOS status-bar chat panel through the native SwiftUI chat view with repeated bottom-pinning writes and no visible guard for active native scrollbar-thumb dragging.

## What This Changes

Fix the macOS companion app chat view so dragging the native scrollbar thumb through chat history does not freeze the UI.

## Best Possible Solution

Add a macOS-aware chat scroll guard or adapter that defers bottom-follow writes while the user is actively controlling the native scroll view, then resumes only on explicit bottom return or intentional send/new-bottom conditions.
