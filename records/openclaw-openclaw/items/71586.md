## Summary

Keep open. Current main and the latest shipped release still route macOS Open Chat through the native SwiftUI chat view, whose message list continues to programmatically write the bottom scroll target without a visible guard for active native scrollbar-thumb dragging; no linked or searched fix owns this specific native companion-app hang.

## What This Changes

The issue reports that dragging the macOS companion app chat scrollbar thumb up and down can freeze the native chat UI.

## Best Possible Solution

Add a native macOS chat scroll guard or AppKit-backed adapter that pauses bottom-follow writes while the user is actively controlling the scroll view, then resumes on explicit bottom return, send, or new-bottom intent.

## Reproduction Assessment

No high-confidence live reproduction was established. The reporter provided concrete steps, and current source still exposes the same native SwiftUI scroll-position path that plausibly conflicts with native scrollbar-thumb dragging.
