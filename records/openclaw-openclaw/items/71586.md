## Summary

Keep open. The issue is recent, specific, and still matches current main: the macOS status-bar chat opens the native SwiftUI chat surface, and that surface still binds `scrollPosition` while repeatedly writing the bottom sentinel during load, sends, message-count changes, pending runs, and streaming updates. I found no source, test, doc, changelog, or recent-history evidence of a scrollbar-thumb drag fix.

## What This Changes

Fix the macOS companion chat so dragging the scrollbar thumb up and down in chat history does not freeze the app UI.

## Best Possible Solution

Fix `OpenClawChatView` so bottom-follow updates do not fight active user scrollbar-thumb interaction on macOS, while preserving bottom-follow for initial load, sent user messages, pending runs, and streaming assistant output. If SwiftUI `scrollPosition` remains unstable during thumb dragging, add a narrow macOS `NSScrollView`-backed seam rather than broad shared chat churn.
