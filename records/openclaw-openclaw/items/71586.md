The issue asks for the macOS companion chat UI to remain responsive when a user drags the chat scrollbar thumb up and down.

Fix the macOS chat scroll path so bottom-follow updates pause or defer while the user is actively dragging or otherwise controlling the scroll position, while preserving intentional bottom-follow for first load, sent user messages, pending runs, and streaming assistant output.
