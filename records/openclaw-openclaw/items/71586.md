## Summary

Keep open. Current main still routes the status-bar macOS chat through the native SwiftUI `OpenClawChatView`, whose scroll state repeatedly writes the bottom scroll target and has no visible guard for active native scrollbar-thumb dragging; related searches found only web Control UI scrollbar/follow-scroll work, not a native companion fix.

## What This Changes

Investigate and fix a macOS companion chat hang when dragging the native scrollbar thumb in the status-bar chat window.

## Best Possible Solution

Add a native macOS chat scroll guard or adapter that defers bottom-follow writes while the user is actively controlling the native scroll view, then resumes only on explicit bottom return or intentional send/new-bottom conditions.

No high-confidence live current-main reproduction was established. The reporter provided concrete macOS companion steps, and current source still exposes the same native SwiftUI scroll path that plausibly fights scrollbar-thumb dragging.
