## Summary

Keep open. The report is recent and specific, and current main still uses the same native SwiftUI chat scroll path with bound `scrollPosition` plus repeated programmatic bottom-pinning; I found no current source, test, docs, changelog, or release evidence that resolves scrollbar-thumb dragging hangs.

## What This Changes

The issue asks for the macOS companion chat window to remain responsive when users drag the scrollbar thumb quickly through chat history.

## Best Possible Solution

Fix the native macOS `OpenClawChatView` scroll behavior so programmatic bottom-pinning does not fight active user scrollbar-thumb interaction, while preserving bottom-follow behavior for initial load, sends, newly appended user messages, pending runs, and streaming assistant output.

## Reproduction Assessment

Yes for triage: the issue provides concrete companion-app steps against v2026.4.22 on macOS 26.4.1, and current main still uses the implicated SwiftUI scroll architecture. This review did not independently live-reproduce the hang on macOS.

## Solution Assessment

Not applicable as a patch review because this issue has no proposed fix. The narrowest maintainable direction likely belongs in `OpenClawChatView`, with a macOS-aware user-scroll interaction guard or an `NSScrollView`-backed replacement if SwiftUI `scrollPosition` remains unstable.
