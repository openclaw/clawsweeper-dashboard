## Summary

Keep open. The report is recent and specific, and current main still routes the macOS status-bar chat through the native SwiftUI chat view whose message list uses `scrollPosition` bottom-pinning with repeated programmatic writes and no visible guard for active scrollbar-thumb dragging; existing macOS tests only smoke show/close behavior.

## What This Changes

The issue reports that the macOS companion app's status-bar chat window freezes when the user drags the scrollbar thumb rapidly through chat history.

## Best Possible Solution

Fix the macOS chat scroll path so programmatic bottom-follow updates pause or defer while the user is actively controlling the scroll position, while preserving bottom-follow for first load, user sends, pending runs, and streaming assistant output. If SwiftUI `scrollPosition` remains unstable for thumb dragging, add a narrow macOS scroll adapter around the message list instead of changing gateway transcript or protocol behavior.

Yes for triage: the issue gives concrete version, macOS version, and status-bar chat steps, and current main still routes that workflow through the same SwiftUI scroll architecture. This review did not independently live-reproduce the hang.
