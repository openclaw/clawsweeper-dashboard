## Summary

Keep open. The report is recent and specific, and current main still routes the macOS status-bar chat to the same native SwiftUI chat surface where `ScrollView` is bound through `scrollPosition` and programmatically repinned during load, sends, message-count changes, pending runs, and streaming updates. I found no source, test, docs, or changelog evidence that a scrollbar-thumb drag hang has been fixed.

## What This Changes

The issue requests fixing a macOS companion Chat view hang that occurs when the user drag-scrolls the scrollbar thumb through chat history.

## Best Possible Solution

Fix `OpenClawChatView` so programmatic bottom-pinning does not fight active macOS scrollbar-thumb interaction, while preserving bottom-follow for initial load, new user sends, pending runs, and streaming assistant output. If SwiftUI `scrollPosition` remains unstable under thumb dragging, add a narrow macOS-specific `NSScrollView`-backed seam rather than broad shared chat churn.
