## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Resolve capability unions at every consumer:** `src/image-generation/types.ts:135`
  - body: Changing `capabilities` to also allow a resolver function means existing typed consumers can no longer read `.generate`, `.edit`, `.geometry`, or `.output` directly. The current `check-test-types` job fails with TS2339 in multiple image provider tests, so migrate all consumers/tests through the resolver or avoid widening the public field until that migration is complete.
  - confidence: 0.97
- **[P2] Resolve configured capabilities in provider lists:** `src/agents/tools/image-generate-tool.actions.ts:33-38`
  - body: These helpers resolve provider capabilities without the `cfg`/`agentDir` context that `createImageGenerateListActionResult` already has. A Comfy provider configured with `image.dimensions` can therefore still be summarized as if size/aspectRatio are unsupported, and list details/catalog paths can carry the unresolved function instead of the resolved capability object.
  - confidence: 0.88
- **[P2] Document dimensions under image.dimensions:** `docs/providers/comfy.md:236-240`
  - body: The table lists `widthNodeId`, `heightNodeId`, input names, and `baseSize` as direct image/video keys, but the manifest and runtime read them from nested `image.dimensions` and this PR only wires image generation. Users following this table would put the keys where the runtime ignores them.
  - confidence: 0.91
