## Best Possible Solution

Land only after maintainers choose the SDK direction: either fully materialize config-aware capabilities everywhere they are exposed, or keep Comfy dimensions provider-local while preserving static SDK capabilities.

## Reproduction Assessment

## Solution Assessment

No. The Comfy workflow injection is plausible, but the broad public dynamic-capabilities contract is not the best merge path until list/catalog/SDK compatibility is resolved or the feature is kept provider-local.
