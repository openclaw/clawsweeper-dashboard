The branch adds Comfy `image.dimensions` width/height injection from `size` or `aspectRatio`, widens image provider capabilities to allow runtime resolvers, and updates docs plus mocked provider tests.

## Best Possible Solution

Keep the Comfy dimension feature, but either keep capabilities static or finish a config-aware resolver migration across all image-generation metadata, list, catalog, test, and SDK consumers; align docs with `image.dimensions`; then require redacted real Comfy proof before merge.

## Reproduction Assessment

Yes for the review blockers: source inspection shows the PR makes Comfy capabilities a resolver while unchanged metadata, catalog, and live-test paths still treat capabilities as an object. No real Comfy runtime reproduction was provided or run in this read-only review.

## Solution Assessment

No. The Comfy injection path is plausible, but broadening the public image-provider capability contract without a complete migration is not the safest solution; a complete resolver migration or narrower Comfy-owned approach is safer.
