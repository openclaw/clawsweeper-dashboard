## Best Possible Solution

Land only after the Comfy dimension feature either materializes config-aware capabilities before every public/list/catalog payload with focused tests, or keeps the new behavior behind a provider-local design that does not widen the public capability contract.

## Solution Assessment

No. The Comfy workflow injection is plausible, but the current public dynamic-capabilities contract is not the best merge path until catalog/list/SDK compatibility is resolved or the feature is kept provider-local.
