## Summary

Keep this PR open: the feature direction is plausible, but the current patch has blocking typecheck/import defects and would advertise Comfy geometry support even when a workflow has no dimension node configuration.

## What This Changes

The PR adds optional Comfy image workflow dimension config, passes image `size` and `aspectRatio` into the workflow runner, and injects calculated width/height values into configured workflow nodes with mocked unit coverage.

## Best Possible Solution

Land this only after Comfy geometry support has honest config-aware semantics or a clear fail/ignore path, the public Comfy docs and manifest schema are aligned, typecheck blockers are fixed, and a redacted real Comfy run proves injected dimensions.

## Reproduction Assessment

Yes for the PR blockers from source inspection: current exports lack the imported SDK subpath, extension typecheck rejects unused locals, and normalization behavior depends on the capability flags being changed. I did not run tests because this review must keep the checkout read-only.

## Solution Assessment

No, not yet. Injecting dimensions in the Comfy plugin is a reasonable direction, but the current implementation needs build-safe imports, no unused locals, explicit behavior when `dimensions` is missing, docs, and real workflow proof before it is the maintainable solution.

Full review comments:

- **[P1] Do not advertise geometry support when dimensions are absent:** `extensions/comfy/image-generation-provider.ts:30-31`
  - body: The PR flips Comfy `supportsSize` and `supportsAspectRatio` to true globally, but `runComfyWorkflow` skips injection when `dimensions` is unset. The shared image-generation normalization only reports ignored overrides when capabilities are false, so existing Comfy workflows without dimension config would accept a size/aspectRatio request, leave the workflow unchanged, and no longer warn that the override was ignored. Gate support on usable dimensions, fail clearly, or preserve ignored-override reporting with runtime/tool coverage.
  - confidence: 0.92

  - body: `openclaw/plugin-sdk/image-generation/types` is not a package export or plugin SDK entrypoint on current main. This will fail extension resolution/typecheck; import `ImageGenerationRequest` from the existing `openclaw/plugin-sdk/image-generation` barrel instead.
  - confidence: 0.96

  - body: The new workflow code assigns `appliedDimensions` but never reads it. `tsconfig.extensions.json` enables `noUnusedLocals`, so the Comfy extension typecheck will fail until this variable is removed or intentionally returned/used.
  - confidence: 0.94
- **[P2] Document the new Comfy dimension mapping:** `extensions/comfy/openclaw.plugin.json:235-245`

  - confidence: 0.88
