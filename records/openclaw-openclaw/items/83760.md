## Summary

The PR is still useful but not merge-ready: it adds the requested Comfy dimension injection, but the public image-provider capabilities contract is widened without a complete config-aware migration, docs still describe the config at the wrong level, and real Comfy proof is missing.

## What This Changes

The branch adds Comfy `image.dimensions` width/height injection from `size` or `aspectRatio`, changes image provider capabilities to allow runtime resolvers, and updates related provider tests/docs.

## Best Possible Solution

Keep the Comfy dimension feature only after completing a config-aware capability resolver migration across list/catalog/live-test/SDK surfaces, correcting the docs to `image.dimensions`, and adding redacted real ComfyUI or Comfy Cloud proof.

## Reproduction Assessment

Yes for the review blockers: source inspection of the PR head shows dynamic capabilities are resolved without config in list/help paths and left raw in shared catalog/details. No live Comfy workflow reproduction was provided or run in this read-only review.

## Solution Assessment

No. The Comfy injection path is plausible, but broadening the public capability contract without a complete migration is not the safest solution; a finished resolver migration or narrower Comfy-owned path is safer.
