## What This Changes

The PR adds optional Comfy image `dimensions` config, runtime width/height injection from `size` or `aspectRatio`, docs, schema changes, and mocked provider tests.

## Best Possible Solution

Implement a typed nested `image.dimensions` parser, keep unsupported geometry overrides honest for configs without dimension nodes, align schema/docs/tests around that contract, and land only with redacted real Comfy workflow proof.

## Reproduction Assessment

Yes for the PR blockers from source inspection: the latest head still references `cfg` outside scope and passes untyped `dimensions` values into helpers requiring `ComfyDimensionConfig`. I did not run tests because this review is read-only.

## Solution Assessment

No: the best path is one typed parser for the nested `image.dimensions` config plus honest config-aware capability semantics, not a top-level config check that disagrees with the schema and docs.
