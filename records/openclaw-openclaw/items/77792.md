## Summary

Keep open. Current main still rejects Xiaomi MiMo per-call TTS `voice` and `model` overrides in the standard CLI/Gateway path because Xiaomi registers voices/models and consumes provider overrides during synthesis, but does not expose the required `resolveTalkOverrides` hook.

## What This Changes

Support Xiaomi MiMo TTS per-call voice and model overrides through the existing OpenClaw TTS CLI and Gateway paths.

## Best Possible Solution

Add Xiaomi's provider-local `resolveTalkOverrides` mapping for `voiceId` and `modelId`, cover the CLI/Gateway resolver and Xiaomi request payload in focused tests, and leave VoiceDesign support to the separate feature request.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: CLI/Gateway `--voice` or `--model xiaomi/...` reaches `resolveExplicitTtsOverrides`, which throws before synthesis because the selected Xiaomi provider lacks `resolveTalkOverrides`.

## Solution Assessment

Yes. The narrowest maintainable fix is provider-local resolver plumbing in `extensions/xiaomi`, not a shared CLI/Gateway behavior change.
