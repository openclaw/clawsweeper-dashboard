## Summary

Keep open. Current main still rejects Xiaomi MiMo per-call TTS voice/model overrides in the standard CLI/Gateway path because the Xiaomi speech provider does not expose the `resolveTalkOverrides` hook required by the shared explicit override resolver.

## What This Changes

Request Xiaomi MiMo TTS support for per-call `voice` and `model` overrides through the standard CLI/Gateway TTS path.

## Best Possible Solution

Add a Xiaomi-provider-local `resolveTalkOverrides` implementation that maps `voiceId` to Xiaomi `voice` and `modelId` to Xiaomi `model`, with focused regression coverage for CLI/Gateway override resolution and Xiaomi request payloads.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: `--voice` or `--model xiaomi/...` reaches `resolveExplicitTtsOverrides`, which throws before synthesis because the selected Xiaomi provider lacks `resolveTalkOverrides`.

## Solution Assessment

Yes. The narrow maintainable fix is provider-local resolver plumbing in `extensions/xiaomi`, while VoiceDesign and Token Plan endpoint work should remain separate.
