## Summary

Current main still has the same source-proven Xiaomi MiMo TTS override bug: CLI/Gateway `--voice` or `--model xiaomi/...` enters the shared explicit override resolver, but the Xiaomi speech provider does not implement the provider hook required to translate those request-scoped overrides.

## What This Changes

Support standard per-call Xiaomi MiMo TTS `voice` and `model` overrides through the existing CLI and Gateway TTS conversion paths.

## Best Possible Solution

Add a Xiaomi-plugin-local `resolveTalkOverrides` implementation that maps `voiceId` to Xiaomi `voice` and `modelId` to Xiaomi `model`, with focused coverage for CLI/Gateway override resolution and Xiaomi request payloads.

## Reproduction Assessment

Yes at source level. On current main, `--voice` or `--model xiaomi/...` reaches `resolveExplicitTtsOverrides`, which throws before synthesis because the selected Xiaomi provider has no `resolveTalkOverrides` hook.

## Solution Assessment

Yes. The narrow maintainable fix is provider-local resolver plumbing in `extensions/xiaomi`; VoiceDesign and Token Plan endpoint work should remain separate.
