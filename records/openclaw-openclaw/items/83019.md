## Summary

Keep open: the diagnostics feature has useful proof, but PR head still emits raw web search/fetch provider identifiers into secrets diagnostics timeline spans, which conflicts with the PR's sanitizer boundary for support artifacts.

## What This Changes

The PR adds secrets diagnostics spans across SecretRef resolution, runtime secret preparation, web-tool provider resolution, gateway startup trace output, and gateway startup benchmark coverage.

## Best Possible Solution

Keep the diagnostics feature, but emit only counts/classes/hashes for web provider identifiers and add coverage that fails if tenant/custom provider ids appear in timeline artifacts.

## Reproduction Assessment

Yes for the review finding: PR-head source shows raw `configuredProvider` and `selectedProvider` strings written into `secrets.webTools.*` span attributes, and the timeline helper preserves string attributes. I did not execute the PR branch in this read-only review.

## Solution Assessment

No. The instrumentation seams are appropriate, but web provider identifiers should use the same hash/class/count sanitization pattern as `secrets.resolve.provider` before merge.
