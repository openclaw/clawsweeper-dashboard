## Summary

Keep open: this is a recent external feature PR with useful gateway/secrets observability work, but PR-head still writes raw web provider identifiers into the new secrets diagnostics spans and lacks sufficient visible real behavior proof.

## What This Changes

The PR adds secrets runtime diagnostics spans, gateway startup trace metrics, web-tool diagnostics, and benchmark/test coverage for secret-backed startup cases.

## Best Possible Solution

Keep the observability direction, but hash or classify all provider-like web-tool span attributes, fix the concrete lint failures, and attach redacted runtime output or an artifact proving the benchmark and leak audit.

Yes for the blocking review finding: PR-head source passes raw configured and selected provider ids into `secrets.webTools` span attributes when diagnostics are enabled. I did not execute the branch in this read-only review.

## Solution Assessment

No. The instrumentation belongs in the right runtime seams, but provider-like attributes should follow the existing hash/class/count pattern before merge.
