## Summary

Keep open: the diagnostics direction is useful and the updated PR body now supplies copied live benchmark output, but the PR still emits raw web provider identifiers into secrets diagnostics attributes despite the stated no-raw-provider boundary.

## What This Changes

The PR adds secrets runtime diagnostics spans, gateway startup trace metrics, web-tool diagnostics, and benchmark coverage for secret-backed gateway startup cases.

## Best Possible Solution

Keep the diagnostics timeline feature, but emit only source/class/count/hash attributes for provider-like values and add web-tool span coverage that proves raw provider ids do not appear.

## Reproduction Assessment

Yes for the review finding: source inspection of PR head shows raw `configuredProvider` and `selectedProvider` strings written into `secrets.webTools.*` span attributes. I did not execute the PR branch in this read-only review.

## Solution Assessment

No. The instrumentation belongs in the right runtime seams, but web provider identifiers should follow the same hash/class/count pattern already used for `secrets.resolve.provider` before merge.
