## Summary

Keep open: current main still lacks known Mantle context-window metadata, so the PR addresses a real provider gap, but the head should not merge as-is because the 200K fallback over-advertises omitted 128K models, Nemotron Super is lower than current repo evidence, and contributor real-behavior proof is missing.

## What This Changes

The PR adds a static known context-window table to Amazon Bedrock Mantle discovery and changes the default context window for unknown Mantle models from 32K to 200K.

## Best Possible Solution

Land a rebased Mantle-local metadata update that maps known open-weight IDs, keeps unknown fallback behavior conservative, aligns Nemotron Super with current evidence, preserves the Claude Anthropic Messages route, and adds focused regression coverage.

## Reproduction Assessment

Yes, from source inspection: the `discoverMantleModels` fetch seam can mock `/v1/models`, current main returns 32K for IDs such as `minimax.minimax-m2.5`, and this PR's 200K fallback would over-advertise omitted 128K IDs while still under-advertising Nemotron Super. I did not execute tests because this review is read-only.

## Solution Assessment

No, not as written. A static known-context-window table is the right narrow shape, but the PR needs conservative fallback semantics, corrected Nemotron metadata, current-main Claude-route preservation, and real behavior proof before merge.
