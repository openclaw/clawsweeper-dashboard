## Summary

Keep open: the PR targets a real Mantle discovery metadata gap, but it is not merge-ready because real behavior proof is missing and the patch introduces provider metadata regressions for current lower-window models.

## What This Changes

The branch adds a Mantle context-window lookup table, routes discovered model metadata through it, and raises the unknown discovery fallback to 200K in `extensions/amazon-bedrock-mantle/discovery.ts`.

## Best Possible Solution

Land a rebased Mantle-local metadata table with a conservative unknown fallback or explicit lower-window entries, corrected vendor-backed values, focused discovery coverage, and redacted live discovery proof.

## Reproduction Assessment

Yes. From source, current main maps every raw Mantle `/v1/models` entry to the 32K fallback, so a mocked Mantle model list containing GPT-OSS or Nemotron reproduces wrong discovered metadata without needing a live account.

## Solution Assessment

No, not as written. A Mantle-owned known-window table is the right boundary, but the 200K fallback and Nemotron Super value need correction or Mantle-specific proof before this is the safest fix.
