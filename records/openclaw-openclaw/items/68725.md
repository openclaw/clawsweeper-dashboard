## Summary

Keep open: the Mantle metadata update is directionally useful, but the current PR still overstates unlisted models with a 200K fallback, conflicts with adjacent Nemotron Super metadata, and lacks contributor after-fix real behavior proof.

## What This Changes

The PR adds a static context-window lookup to Amazon Bedrock Mantle discovery and changes unknown discovered models from a 32K fallback to 200K.

## Best Possible Solution

Land a Mantle-local metadata fix that keeps unknown fallback behavior conservative, corrects or justifies Nemotron Super, adds focused regression coverage, and includes redacted live discovery proof.

## Reproduction Assessment

Yes. Current main source shows discoverMantleModels maps raw Mantle /v1/models entries to DEFAULT_CONTEXT_WINDOW = 32000, and the existing fetch seam can demonstrate that with a mocked response; no tests were run in this read-only review.

## Solution Assessment

No, not as written. A Mantle-local known-context-window table is the right narrow shape, but the fallback and Nemotron Super value need correction or evidence before merge.
