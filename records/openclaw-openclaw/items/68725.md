## Summary

Keep this PR open: the underlying Mantle discovery bug is still present on current main, but the branch is not ready to merge because it lacks contributor real-behavior proof and still has two correctness issues in the proposed metadata.

## What This Changes

The branch adds a Mantle `KNOWN_CONTEXT_WINDOWS` table, resolves discovered model IDs through it, and raises the unknown-model fallback from 32K to 200K in `extensions/amazon-bedrock-mantle/discovery.ts`.

## Best Possible Solution

Land a rebased Mantle-local metadata update that maps known open-weight IDs, keeps unknown fallback behavior conservative, corrects Nemotron Super, preserves the Claude Anthropic Messages route, adds focused regression coverage, and includes real behavior proof.

## Reproduction Assessment

Yes from source inspection: current main maps every mocked Mantle `/v1/models` ID to the 32K fallback through `discoverMantleModels`, so a model such as `minimax.minimax-m2.5` is underreported. I did not execute tests because this is a read-only review.

## Solution Assessment

No, not as written. A static known-context-window table is the right narrow shape, but the 200K unknown fallback overstates omitted 128K models and Nemotron Super is underreported relative to current repo and AWS metadata.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep the Mantle fallback conservative:** `extensions/amazon-bedrock-mantle/discovery.ts:18`
  - body: The final commit raises every unlisted `/v1/models` result to 200K, but the branch history says GPT-OSS, Kimi, Mistral small, and similar 128K models were omitted because a 128K fallback covered them; AWS documents GPT-OSS 120B as 128K. This can advertise unknown lower-window models above their real limit and delay compaction until provider errors, so keep the fallback at 128K or add explicit lower-window entries. ([docs.aws.amazon.com](https://docs.aws.amazon.com/bedrock/latest/userguide/model-card-openai-gpt-oss-120b.html?utm_source=openai))
  - confidence: 0.91
- **[P2] Align Nemotron Super's context window:** `extensions/amazon-bedrock-mantle/discovery.ts:58`
  - body: This table assigns `nvidia.nemotron-super-3-120b` a 128K window, while current main's Bedrock table uses 256K and AWS's Bedrock Mantle model card lists a 256K context window for the same ID. If Mantle returns this ID, discovery still compacts too early; set the Mantle value to the known larger window or document Mantle-specific evidence. ([docs.aws.amazon.com](https://docs.aws.amazon.com/bedrock/latest/userguide/model-card-nvidia-nemotron-super-3-120b.html?utm_source=openai))
  - confidence: 0.86
