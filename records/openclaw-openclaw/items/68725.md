## Summary

Keep open: current main still gives raw Mantle-discovered models a 32K context window, so the central fix is useful, but this PR still has blocking metadata issues and lacks contributor real-behavior proof.

## What This Changes

The PR adds a static known context-window table to Amazon Bedrock Mantle discovery and raises the unknown-model fallback from 32K to 200K.

## Best Possible Solution

Land a rebased Mantle-local metadata update that maps known open-weight IDs, keeps unknown fallback behavior conservative, corrects Nemotron Super, preserves the Claude Anthropic Messages route, adds focused regression coverage, and includes real behavior proof.

## Reproduction Assessment

Yes. Source inspection shows current main returns 32K for raw Mantle-discovered IDs such as `minimax.minimax-m2.5`, and the existing `discoverMantleModels` fetch seam can reproduce this with a mocked `/v1/models` response; I did not execute tests because this review is read-only.

## Solution Assessment

No, not as written. A static known-context-window table is the right narrow shape, but the PR needs conservative fallback semantics, corrected Nemotron metadata, current-main Claude-route preservation, and contributor real-behavior proof before merge.
