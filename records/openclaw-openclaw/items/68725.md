## Summary

Keep this PR open, but it is not merge-ready: the Mantle metadata table is the right boundary for the reported 32K discovery problem, but the branch still lacks real Mantle proof and changes provider defaults in a way that can overstate unknown model context windows.

## What This Changes

The branch adds a known context-window lookup to `extensions/amazon-bedrock-mantle/discovery.ts` and raises the fallback discovered context window from 32K to 200K.

## Best Possible Solution

Land a Mantle-local metadata table with conservative unknown fallback behavior, corrected or explicitly justified model entries, focused regression coverage, and redacted live discovery proof.

## Reproduction Assessment

Yes, from source: current main maps every raw Mantle `/v1/models` result to the 32K fallback, so a mocked list containing a known larger-window model reproduces wrong discovered metadata. I did not run tests because this review is read-only.

## Solution Assessment

No, not as written. A provider-local metadata table is the right shape, but the fallback default and at least one known entry need correction or explicit Mantle-specific evidence before merge.
