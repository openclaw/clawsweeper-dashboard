## Summary

Keep open: current main still has the 32K Mantle discovery behavior, but this PR is not merge-ready because it lacks after-fix real behavior proof and has two concrete metadata correctness blockers.

## What This Changes

The PR adds a static context-window lookup to Amazon Bedrock Mantle discovery and changes unknown discovered Mantle models from a 32K fallback to 200K.

## Best Possible Solution

Land a Mantle-local context-window table with a conservative unknown fallback, corrected or justified Nemotron Super metadata, focused regression coverage, and redacted live or terminal proof of discovery output.

## Reproduction Assessment

Yes. Current main source shows raw Mantle /v1/models discovery maps every returned model to the 32K default, and the existing fetch seam can demonstrate that with a mocked response; no tests were run in this read-only review.

## Solution Assessment

No, not as written. A Mantle-owned metadata table is the right narrow shape, but the 200K fallback and Nemotron Super value need correction or Mantle-specific proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Keep the Mantle fallback conservative:** `extensions/amazon-bedrock-mantle/discovery.ts:18`
  - body: Setting unknown raw Mantle models to 200K conflicts with the branch's own earlier assumption that GPT-OSS, Kimi, Mistral small, and similar models were omitted because a 128K fallback covered them. Those models would compact too late unless they get explicit lower-window entries.
  - confidence: 0.91
- **[P2] Align Nemotron Super's context window:** `extensions/amazon-bedrock-mantle/discovery.ts:58`
  - body: This assigns nvidia.nemotron-super-3-120b a 128K window, while current Bedrock discovery maps the same ID to 256K and the bundled NVIDIA provider docs list 262,144. If Mantle returns that ID, discovery still compacts too early; use the larger value or provide Mantle-specific evidence.
  - confidence: 0.84
