## Summary

Keep this PR open: current main still underreports discovered Mantle model context windows, but the proposed head has blocking metadata issues and lacks required real behavior proof, so it should not merge as-is.

## What This Changes

The PR adds a static known context-window lookup to Amazon Bedrock Mantle discovery and changes the unknown-model fallback from 32K to 200K.

## Best Possible Solution

Land a rebased Mantle-local metadata update with conservative fallback behavior, corrected Nemotron Super metadata or documented Mantle-specific evidence, focused regression coverage, and redacted live proof.

## Reproduction Assessment

Yes. Source inspection shows current main maps every raw Mantle `/v1/models` entry, such as `minimax.minimax-m2.5`, to `DEFAULT_CONTEXT_WINDOW = 32000`; the existing `discoverMantleModels` fetch seam can reproduce that with a mocked response, though no tests were run in this read-only review.

## Solution Assessment

No, not as written. A Mantle-local known-context-window table is the right narrow shape, but the head should keep unknown fallback behavior conservative, align Nemotron Super, and include proof before merge.
