## Summary

Keep open: the Mantle metadata fix is still useful on current main, but the branch is not merge-ready because the 200K fallback can overstate unknown model windows, the Nemotron Super value conflicts with adjacent metadata, and contributor real-behavior proof is still absent.

## What This Changes

The branch adds a static Mantle context-window lookup for discovered model IDs and raises the discovery fallback from 32K to 200K in `extensions/amazon-bedrock-mantle/discovery.ts`.

## Best Possible Solution

Land a Mantle-local metadata table with a conservative unknown fallback, corrected or cited vendor-specific entries, focused regression coverage, and redacted live discovery proof.

## Reproduction Assessment

Yes, from source: a mocked Mantle `/v1/models` response on current main maps every raw discovered model to the 32K fallback. I did not run tests because this review was read-only.

## Solution Assessment

No, not as written. A Mantle-owned metadata table is the right narrow shape, but the 200K fallback and Nemotron Super value need correction or Mantle-specific proof before merge.
