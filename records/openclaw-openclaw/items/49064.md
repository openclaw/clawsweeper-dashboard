## Summary

Keep open: current main still lacks the proposed top-level CLI workflow, but this PR needs maintainer approval for the new command surface, real behavior proof, and fixes for date parsing, session aggregation, and output consistency before merge.

## What This Changes

Adds a top-level `openclaw usage` CLI command that reads transcript-backed usage summaries with date ranges, session/model breakdowns, JSON output, CLI registration, and focused helper tests.

## Best Possible Solution

If maintainers accept the CLI surface, rebase onto current CLI registration, make the command a thin wrapper over current gateway usage/date-range contracts and shared formatters, aggregate all in-range sessions before display limits, and add docs, regression coverage, and redacted real-run proof.

## Reproduction Assessment

Not applicable as a user bug report. For the PR defects, source inspection gives a high-confidence path: `resolveDateRange` mixes date parsing modes, and the session/model breakdown path limits sessions before full in-range aggregation.

## Solution Assessment

No. The CLI direction may be useful, but this patch should reuse current gateway date/range contracts and shared formatting, and the top-level command surface needs maintainer approval before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Parse custom start dates in the same date mode:** `src/cli/usage-cli.ts:46-47`
  - body: `--from` uses date-only `new Date(opts.from)`, which is UTC, while `--to` uses a local datetime string. In non-UTC zones custom ranges cross local-day boundaries and report wrong totals, so parse both endpoints through the same local or gateway date helper.
  - confidence: 0.91
- **[P2] Aggregate all in-range sessions before limiting output:** `src/cli/usage-cli.ts:321-323`
  - body: The command loads costs for only the 100 most recently modified sessions before computing `--sessions` and `--by-model`, so an older expensive in-range session can be missing from the totals. Use the selected range in discovery and apply `--limit` only to rendered rows.
  - confidence: 0.9
- **[P3] Reuse the shared token formatter:** `src/cli/usage-cli.ts:102-109`
  - body: This command duplicates `formatTokenCount` and emits uppercase `M`/`K`, while the shared usage formatter emits lowercase `m`/`k`. Import the shared helper so usage output stays consistent across CLI surfaces.
  - confidence: 0.84
