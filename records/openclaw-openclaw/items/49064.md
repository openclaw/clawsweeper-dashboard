## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Parse custom range starts in local date mode:** `src/cli/usage-cli.ts:43-44`
  - body: `--from` uses `new Date(opts.from)`, which parses `YYYY-MM-DD` as UTC, while `--to` is parsed as local end-of-day. Non-UTC users get shifted custom range totals.
  - confidence: 0.92
- **[P2] Reject conflicting range flags:** `src/cli/usage-cli.ts:69-79`
  - body: The `--days` block runs after the main range selection, so it silently overrides `--from`, `--to`, `--week`, or `--month`. Users combining flags get a different range than requested.
  - confidence: 0.86
- **[P2] Aggregate before applying the display limit:** `src/cli/usage-cli.ts:314-315`
  - body: The session/model breakdown limits the session list before loading costs, so an older in-range expensive session can be excluded from top-session and by-model totals. Treat `--limit` as a display cap after aggregation.
  - confidence: 0.9
- **[P3] Reuse the shared token formatter:** `src/cli/usage-cli.ts:93-101`
  - body: This command duplicates `formatTokenCount` with uppercase suffixes while the shared formatter outputs lowercase and is already used by usage surfaces. Reuse the shared helper to keep CLI output consistent.
  - confidence: 0.83
- **[P3] Add the required changelog entry:** `src/cli/program/subcli-descriptors.ts:128-132`
  - body: This is a user-facing `feat(cli)` change, but the diff has no `CHANGELOG.md` entry. Add a single-line entry before merge, with appropriate human attribution if the feature is accepted.
  - confidence: 0.82
