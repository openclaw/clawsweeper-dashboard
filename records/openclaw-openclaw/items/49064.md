## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Aggregate all in-range sessions before applying display limits:** `src/cli/usage-cli.ts:310-313`
  - body: `--limit` is described as a display limit, but this path first sorts by transcript mtime and scans only the first 100 sessions before computing costs. Any older-but-expensive session inside the requested range is never loaded, so `--sessions` and `--by-model` can report the wrong top sessions and model totals.
  - confidence: 0.91
- **[P2] Parse custom range boundaries in one timezone:** `src/cli/usage-cli.ts:44-45`
  - body: `new Date(opts.from)` parses a `YYYY-MM-DD` value at UTC midnight, while the `--to` value is parsed as local end-of-day. Users outside UTC get shifted custom ranges, so usage near the start boundary can be included or excluded from the wrong local day.
  - confidence: 0.88
- **[P3] Reuse the shared token formatter:** `src/cli/usage-cli.ts:93-101`
  - body: The new command defines a local `formatTokenCount` with uppercase `K`/`M` output while existing usage CLI surfaces use `src/utils/usage-format.ts`. Reusing the shared helper avoids inconsistent token formatting across `openclaw gateway usage-cost`, status, and the new command.
  - confidence: 0.78
