## Summary

## What This Changes

The PR adds a root `openclaw usage` command, tests, and CLI registration for date-filtered token/cost summaries with session, model, and JSON output.

## Best Possible Solution

If maintainers want a root usage command, build it on current descriptor-backed CLI registration, delegate to existing gateway usage contracts, share date parsing and formatting helpers, and apply limits only to rendered rows.

## Reproduction Assessment

Not applicable as a user bug report because this PR proposes a new CLI feature. The review findings are source-reproducible from the PR diff and a focused Node date parsing check.

## Solution Assessment

No, not as-is. The safer path is to reuse the existing gateway usage date/session contracts and shared token formatter, then aggregate all in-range sessions before applying display limits.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Parse custom range endpoints consistently:** `src/cli/usage-cli.ts:46-47`
  - body: `--from` parses a date-only string as UTC while `--to` appends a local end-of-day timestamp. In non-UTC environments the custom range can include or exclude hours from the wrong local day, so parse both endpoints through the same local or gateway date helper.
  - confidence: 0.94
- **[P2] Aggregate sessions before applying display caps:** `src/cli/usage-cli.ts:321-323`
  - body: The sessions/by-model path sorts discovered sessions by mtime and hard-caps to 100 before loading summaries. Older in-range expensive sessions can be omitted from top-session and per-model totals; load/filter the full in-range data first and apply `--limit` only to displayed rows.
  - confidence: 0.91
- **[P3] Reuse the shared token formatter:** `src/cli/usage-cli.ts:102-109`
  - body: This command duplicates `formatTokenCount` and emits uppercase `K`/`M`, while the shared usage formatter emits the existing lowercase style and handles rounding consistently. Import the shared helper so usage output matches the rest of OpenClaw.
  - confidence: 0.84
