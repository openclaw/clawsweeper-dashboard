- **[P2] Rebase usage registration onto the current CLI catalog:** `src/cli/program/register.subclis.ts:302`
  - body: This hunk adds `usage` to the old `register.subclis.ts` entries table, but current main now resolves subcommands through descriptor-backed command groups. Rebase the command registration onto the current catalog so the new root command can land cleanly.
  - confidence: 0.93
- **[P2] Parse custom date range endpoints consistently:** `src/cli/usage-cli.ts:46-47`
  - body: `--from` parses a date-only string as UTC while `--to` appends a local end-of-day timestamp. In non-UTC environments custom ranges can include or exclude hours from the wrong local day, so parse both endpoints through the same local/UTC date contract.
  - confidence: 0.94
- **[P2] Aggregate all matching sessions before limiting display:** `src/cli/usage-cli.ts:321-323`
  - body: The sessions/by-model path hard-caps the most recently modified sessions before loading summaries. Older in-range expensive sessions can be omitted from top-session and per-model totals, so load/filter the full in-range data first and apply `--limit` only to rendered rows.
  - confidence: 0.91
- **[P2] Keep JSON costs from rounding to zero:** `src/cli/usage-cli.ts:17`
  - body: The JSON path uses the two-decimal `roundCost` helper, so small but nonzero costs such as $0.004 serialize as `0`. Machine-readable output should preserve the underlying cost precision rather than applying display rounding.
  - confidence: 0.86
- **[P3] Reuse the shared token formatter:** `src/cli/usage-cli.ts:102-109`
  - body: This command duplicates token formatting and emits uppercase suffixes while the shared usage formatter already defines OpenClaw's existing style. Import the shared helper so CLI usage output stays consistent.
  - confidence: 0.84
