## Summary

Keep this PR open. Current main still lacks the proposed top-level `openclaw usage` command, so the contribution is not obsolete, but the submitted branch is not merge-ready: it is stale against the current subcommand registration structure and has correctness issues in date parsing and session aggregation. The burn0 discussion is useful context for future provider-accurate cost tracking, but it does not supersede this CLI surface.

## What This Changes

The PR adds a new top-level `openclaw usage` CLI command, registration entries, and tests for token/cost summaries, date ranges, session rankings, by-model breakdowns, and JSON output.

## Best Possible Solution

Accept the feature only after rebasing onto current main and implementing the CLI as a thin, documented surface over the existing usage RPC/session-cost contracts, with shared date parsing/formatting, complete aggregation before display limits, and focused tests that exercise production helpers rather than cloned logic.

## Reproduction Assessment

Not applicable as a user bug report, but there is a high-confidence verification path. Current-main absence of the command is reproducible by the descriptor/file/doc searches, and the PR defects are reproducible by code inspection of the submitted date parsing and pre-aggregation limit paths.

## Solution Assessment

No, the submitted patch is not yet the best way to solve the issue. The direction is reasonable, but the maintainable implementation should align with the existing Gateway usage semantics, current CLI registration architecture, and shared usage formatting/date utilities.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Parse custom range starts in local date mode:** `src/cli/usage-cli.ts:43-44`
  - body: `--from` is parsed as a date-only ISO string, which JavaScript treats as UTC midnight, while `--to` is parsed as local end-of-day. In non-UTC environments this shifts the start boundary and produces incorrect custom range totals.
  - confidence: 0.92
- **[P2] Reject conflicting range flags:** `src/cli/usage-cli.ts:69-79`
  - body: The `--days` branch runs after the main range selection, so it silently overrides `--from`, `--to`, `--week`, or `--month`. Users combining flags get a different range than the explicit date option requested.
  - confidence: 0.86
- **[P2] Scan sessions before applying display limits:** `src/cli/usage-cli.ts:314-315`
  - body: The session/model breakdown path slices to the most recent sessions before loading usage. An older in-range expensive session is never scanned, so `--sessions` and `--by-model` can report the wrong top sessions and model totals.
  - confidence: 0.9
- **[P3] Reuse the shared token formatter:** `src/cli/usage-cli.ts:93-101`
  - body: This command adds a local `formatTokenCount` with uppercase `K`/`M` output even though the shared formatter already exists and is used by other usage surfaces. That makes CLI usage output inconsistent.
  - confidence: 0.83
