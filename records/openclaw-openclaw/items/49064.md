## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Parse custom start dates in the same date mode:** `src/cli/usage-cli.ts:43-44`
  - body: `--from` uses date-only `new Date(opts.from)`, which is UTC, while `--to` uses a local datetime string. In non-UTC zones, custom ranges cross local-day boundaries and report wrong totals; parse both endpoints through the same local or gateway date helper.
  - confidence: 0.91
- **[P2] Aggregate all in-range sessions before limiting output:** `src/cli/usage-cli.ts:312-314`
  - body: The command scans only the 100 most recently modified sessions before loading costs, so an older but expensive in-range session can be missing from `--sessions` and `--by-model` totals. Use the date range in discovery and apply `--limit` only to rendered rows.
  - confidence: 0.9
- **[P3] Reuse the shared token formatter:** `src/cli/usage-cli.ts:92-101`
  - body: This command duplicates `formatTokenCount` and emits uppercase `M`/`K`, while the shared formatter already used by usage surfaces emits lowercase `m`/`k`. Import the shared helper so CLI usage output stays consistent.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/cli/program/subcli-descriptors.ts:128-132`
  - body: This is a user-facing `feat(cli)` change, but the diff does not update `CHANGELOG.md`. Add a single-line entry if maintainers accept the command.
  - confidence: 0.82
