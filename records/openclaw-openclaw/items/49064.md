## Summary

Keep open for maintainer review. The top-level CLI feature is not on current main, but the PR still has blocking correctness issues, lacks contributor real behavior proof, and overlaps an open product decision about usage-reporting surfaces.

## What This Changes

Adds a top-level `openclaw usage` command with date ranges, session/model breakdowns, JSON output, CLI registration, and helper tests.

## Best Possible Solution

If maintainers want the top-level CLI, rebase onto current descriptor-backed registration and make the command reuse gateway date/range/session contracts, shared formatters, complete aggregation, docs, focused tests, and real-run proof.

## Reproduction Assessment

Not applicable as a user bug report. For the PR defects, yes: source inspection and a `TZ=America/Los_Angeles` Node probe show the custom date boundary shift, and source inspection shows session/model summaries cap candidates before full in-range aggregation.

## Solution Assessment

No. The CLI direction may be useful, but this patch duplicates gateway date/formatting logic and limits candidate sessions before aggregation; a safer version should reuse current usage contracts and apply limits only to rendered rows.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Parse custom range endpoints consistently:** `src/cli/usage-cli.ts:46-47`
  - body: `--from` uses date-only `new Date(opts.from)`, which is UTC, while `--to` uses a local datetime string. In non-UTC zones the custom range starts on the wrong local boundary, so parse both endpoints through the same local or gateway date helper.
  - confidence: 0.91
- **[P2] Aggregate sessions before applying the display limit:** `src/cli/usage-cli.ts:321-323`
  - body: The command loads costs for only the 100 most recently modified sessions before computing `--sessions` and `--by-model`. Older in-range sessions, including old expensive ones, can be missing from the totals; discover/load the full range first and apply `--limit` only to rendered rows.
  - confidence: 0.9
- **[P3] Reuse the shared token formatter:** `src/cli/usage-cli.ts:102-109`
  - body: This command duplicates `formatTokenCount` and emits uppercase `M`/`K`, while the shared usage formatter emits lowercase `m`/`k`. Import the shared helper so usage output stays consistent across CLI surfaces.
  - confidence: 0.84
