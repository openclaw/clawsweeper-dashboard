## Summary

Keep this PR open for maintainer follow-up. Current main still lacks the proposed top-level `openclaw usage` command, so the contribution is not obsolete, but the submitted patch has correctness issues in date parsing and session/model aggregation that should block merge until reworked.

## What This Changes

The PR adds a top-level `openclaw usage` CLI command, CLI registration, and tests for token/cost summaries, date ranges, session rankings, by-model breakdowns, and JSON output.

## Best Possible Solution

Keep the PR as a useful implementation candidate, but rebase it onto current main and implement `openclaw usage` through the existing usage RPC/session-cost contracts with shared date parsing, shared formatting, complete aggregation before display limits, focused CLI tests against production helpers, and docs if the top-level command is accepted.

## Reproduction Assessment

Not applicable as a user bug report, but the PR defects have high-confidence focused checks: a non-UTC custom `--from/--to` range shifts boundaries, and a session store with more than 100 newer sessions can hide older in-range high-cost sessions from `--sessions`/`--by-model`. Current-main absence of the command is reproducible from the descriptor catalog, CLI docs, and missing `src/cli/usage-cli.ts`.

## Solution Assessment

No, the submitted patch is not yet the best way to solve the issue. The direction is reasonable, but the maintainable fix should reuse current Gateway usage date semantics and shared formatters, and avoid a separate partial scan path that can drift from Control UI and RPC behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Parse custom date starts in the same timezone:** `src/cli/usage-cli.ts:43-44`
  - body: `--from` is parsed as a date-only ISO string, which JavaScript treats as UTC midnight, while `--to` is parsed as local end-of-day. For users outside UTC, custom range totals include or exclude the wrong local-day hours, so the usage summary can be incorrect.
  - confidence: 0.92
- **[P2] Aggregate sessions before applying display limits:** `src/cli/usage-cli.ts:314-315`
  - body: The session/model path slices to the 100 most recently modified sessions before loading usage. An older in-range expensive session is never scanned, so `--sessions` and `--by-model` can return the wrong top sessions and model totals even though the global summary includes that usage.
  - confidence: 0.9
- **[P3] Reuse the shared token formatter:** `src/cli/usage-cli.ts:93-101`
  - body: This command adds a local `formatTokenCount` with uppercase `K`/`M` output while `src/utils/usage-format.ts` already exports the CLI formatter used by the gateway usage command. Keeping a second formatter makes the new command's output inconsistent with existing usage surfaces.
  - confidence: 0.83
