## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Parse custom range endpoints consistently:** `src/cli/usage-cli.ts:46-47`
  - body: `--from` parses a date-only string as UTC while `--to` appends a local end-of-day timestamp. In non-UTC environments that shifts the custom range boundary, so usage from the wrong local day can be included or excluded; parse both endpoints through the same local or gateway date helper.
  - confidence: 0.92
- **[P2] Aggregate in-range sessions before limiting output:** `src/cli/usage-cli.ts:321-323`
  - body: The sessions/by-model path sorts all discovered sessions by mtime and hard-caps to 100 before loading summaries. Older in-range expensive sessions can be omitted from top-session and per-model totals; filter/load the full in-range data first and apply `--limit` only to rendered rows.
  - confidence: 0.9
- **[P3] Reuse the shared token formatter:** `src/cli/usage-cli.ts:102-109`
  - body: This command duplicates `formatTokenCount` and emits uppercase `K`/`M`, while the shared usage formatter emits lowercase `k`/`m` and handles rounding consistently. Import the existing helper so usage output matches the rest of OpenClaw.
  - confidence: 0.84
