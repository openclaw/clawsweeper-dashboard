## Summary

Keep open: current main still lacks the proposed top-level CLI command, but this PR has correctness issues in date handling and session aggregation, lacks contributor real-run proof, and needs maintainer product review for the new CLI surface.

## What This Changes

This PR adds a top-level `openclaw usage` CLI command for date-ranged token and cost summaries, session/model breakdowns, JSON output, registration, and helper tests.

## Best Possible Solution

If maintainers want this command, rebase it onto current CLI registration, reuse existing gateway usage date/session contracts and shared formatters, aggregate full in-range data before limiting rendered rows, add focused tests/docs, and include redacted real-run proof.

## Reproduction Assessment

Not applicable as a user bug report. For the PR defects, yes: source inspection plus a `TZ=America/Los_Angeles` Node probe shows the custom date boundary shift, and source inspection shows session/model summaries cap candidate sessions before full aggregation.

## Solution Assessment

No. The CLI direction may be useful, but this patch duplicates date/formatting logic and limits candidate sessions before aggregation; the safer path is to reuse the gateway usage contracts and apply limits only to rendered rows.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Parse custom range endpoints consistently:** `src/cli/usage-cli.ts:46-47`
  - body: `--from` parses `YYYY-MM-DD` as UTC while `--to` parses a local end-of-day string. In non-UTC zones this shifts the custom range boundary and can include or exclude the wrong local-day usage, so parse both endpoints through the same local or gateway date helper.
  - confidence: 0.92
- **[P2] Aggregate sessions before applying display limits:** `src/cli/usage-cli.ts:321-323`
  - body: The sessions/by-model path hard-caps loading to the 100 most recently modified sessions before computing totals. Older in-range expensive sessions can be omitted from top-session and per-model summaries; discover/filter the full range first and apply `--limit` only to rendered rows.
  - confidence: 0.9
- **[P3] Reuse the shared token formatter:** `src/cli/usage-cli.ts:102-109`
  - body: This command duplicates `formatTokenCount` and emits uppercase `K`/`M`, while the shared usage formatter emits lowercase `k`/`m`. Import the shared helper so usage output stays consistent across CLI surfaces.
  - confidence: 0.84
