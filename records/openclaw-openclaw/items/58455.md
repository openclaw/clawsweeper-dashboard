- **[P1] Reuse the existing tool-result pruning policy:** `src/agents/pi-hooks/compaction-safeguard.ts:776-779`
  - body: Current main already has config-backed `agents.defaults.contextPruning` that soft-trims and hard-clears old tool results.   - body: This appends the local session JSONL path into the compacted summary that re-enters model context. That exposes filesystem/session details and invites raw transcript reads outside an approved redaction or retrieval boundary.
  - confidence: 0.88

  - body: The regex matches the first `<summary>...</summary>` anywhere in the model response. If the analysis block mentions literal summary tags, this can keep analysis text and drop the real final summary, silently damaging compacted session context.
  - confidence: 0.86

  - body: Current main imports Pi types from `@earendil-works/*`, but this new module still imports `@mariozechner/pi-agent-core`. After rebasing, this stale scope will either fail resolution or keep an obsolete dependency boundary alive.
  - confidence: 0.9
