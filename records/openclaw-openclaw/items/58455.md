## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Extract the final summary block:** `src/agents/compaction-analysis-strip.ts:42`
  - body: `stripAnalysisBlock` currently returns the first `<summary>...</summary>` anywhere in the model response. Because the new prompt explicitly asks the model to reason about analysis and summary tags, a literal `<summary>` example inside `<analysis>` can be selected and persisted instead of the final summary, silently losing compacted session context. Parse the summary after the analysis block, or select the last valid summary block, and add that regression case.
  - confidence: 0.9
- **[P1] Gate transcript path recovery before defaulting it on:** `src/agents/pi-hooks/compaction-safeguard.ts:943-945`
  - body: This appends the session JSONL path to every compaction summary, while current compaction behavior exposes the summary and kept messages rather than a direct pointer to the full prior transcript. That changes the privacy/security boundary by encouraging future model turns to read raw pre-compaction transcript data and local paths by default. Remove it, make it an explicit maintainer-approved option, or document and test the boundary before merge.
  - confidence: 0.82
