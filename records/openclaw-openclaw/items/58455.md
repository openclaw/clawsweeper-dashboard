## Summary

Keep open. Current main does not implement the PR's central compaction helpers, and the linked compaction architecture request remains open, but the patch still has merge-blocking security and correctness issues around raw transcript recall and summary extraction.

## What This Changes

The PR adds microCompact and analysis-scratchpad helper modules/tests, wires them into safeguard compaction, and appends a full-session JSONL transcript reference to compacted summaries.

## Best Possible Solution

A mergeable version should keep microCompact scoped to summarizable messages, use a robust final-summary extractor, replace raw JSONL self-service with sanitized recall unless maintainers explicitly approve it, and add the required changelog plus regression coverage.

## Reproduction Assessment

Yes for the parser defect: pass stripAnalysisBlock a response where the analysis text contains a literal <summary>...</summary> example before the real final summary block. The raw-transcript concern is source-inspectable from the new suffix and current sanitized compaction/session-history boundaries.

## Solution Assessment

No. The microCompact direction is plausible and the second commit addressed earlier ordering/path-construction feedback, but the raw transcript recovery path and first-summary parser are not the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Avoid prompting agents to read raw transcripts:** `src/agents/pi-hooks/compaction-safeguard.ts:943`
  - body: This suffix persists the exact JSONL path and tells future model context where to recover details from the raw transcript. Current compaction and sessions_history paths strip details, runtime context, and sensitive payloads before LLM-facing recall, so this bypass can reintroduce hidden or credential-like transcript data on a later read. Keep recovery behind sanitized recall/export unless maintainers explicitly approve the raw path.
  - confidence: 0.84
- **[P2] Match the final summary block:** `src/agents/compaction-analysis-strip.ts:39`
  - body: text.match extracts the first <summary>...</summary> pair anywhere in the response. If the analysis phase mentions or quotes summary tags before the real final section, compaction keeps that analysis fragment and drops the actual summary. Target the summary after </analysis> or the last summary block and add a regression test for that shape.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `src/agents/pi-hooks/compaction-safeguard.ts:685-689`
  - body: This changes user-facing compaction behavior, but the diff does not update CHANGELOG.md. OpenClaw policy requires user-facing feat/fix/perf changes to include an active-version changelog entry with valid contributor credit.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete transcript-data exposure concern by steering future agents toward raw session JSONL instead of the existing sanitized recall path.

Concerns:

- **[high] Raw transcript path bypasses sanitized recall:** `src/agents/pi-hooks/compaction-safeguard.ts:943`
  - body: Appending the full JSONL path to persisted compaction summaries can cause later model/tool turns to read raw transcript entries that current compaction and sessions_history deliberately sanitize, including toolResult.details and runtime-context custom messages. Use a sanitized recovery surface or require explicit maintainer approval for the raw path.
