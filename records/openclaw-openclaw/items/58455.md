## Summary

Keep open. Current main does not contain the proposed microCompact or analysis-scratchpad feature, and this PR still has merge blockers: the raw transcript reference needs maintainer/security judgment, the summary stripper can select the wrong block, and the user-facing behavior change lacks a changelog entry. Earlier review feedback on microCompact ordering and getSessionFile usage appears addressed in the second PR commit.

## What This Changes

The PR adds microCompact helper/tests, analysis-scratchpad stripping/tests, and compaction-safeguard wiring that appends a full-session transcript reference after compaction.

## Best Possible Solution

A mergeable version should keep microCompact scoped to summarizable messages, use a robust final-summary extractor, replace raw JSONL self-service with sanitized recall unless approved, add the required changelog entry, and rebase onto current main.

## Reproduction Assessment

Yes for the parser defect: pass stripAnalysisBlock a response whose analysis text contains a literal <summary>...</summary> example before the real final summary. The raw-transcript concern is source-inspectable from the new suffix and current sanitized compaction/session-history boundaries.

## Solution Assessment

No. The microCompact direction and the second commit’s ordering/path fixes are plausible, but the raw transcript recovery path and first-summary parser are not the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Avoid prompting agents to read raw transcripts:** `src/agents/pi-hooks/compaction-safeguard.ts:943-945`
  - body: This suffix persists the exact JSONL path and tells future model context where to recover details from the raw transcript. Current compaction and sessions_history paths strip details, runtime context, and sensitive payloads before LLM-facing recall, so this can reintroduce hidden or credential-like transcript data on a later read. Keep recovery behind sanitized recall/export unless maintainers explicitly approve the raw path.
  - confidence: 0.84
- **[P2] Match the final summary block:** `src/agents/compaction-analysis-strip.ts:42-44`
  - body: text.match extracts the first <summary>...</summary> pair anywhere in the response. If the analysis phase mentions or quotes summary tags before the real final section, compaction keeps that analysis fragment and drops the actual summary. Target the summary after </analysis> or the last summary block and add a regression test for that shape.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `src/agents/pi-hooks/compaction-safeguard.ts:692-695`
  - body: This changes user-facing compaction behavior, but the diff does not update CHANGELOG.md. OpenClaw policy requires user-facing feat/fix/perf changes to include an active-version changelog entry with valid contributor credit.
  - confidence: 0.78
