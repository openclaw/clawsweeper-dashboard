## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Do not put raw transcript file paths in compaction summaries:** `src/agents/pi-hooks/compaction-safeguard.ts:943-944`
  - body: This appends the actual session JSONL path into model-visible compacted context. Session transcripts can contain pasted secrets, file contents, and command output, while the supported session-history path is bounded and redacted; prompting future turns to read the raw JSONL can bypass that visibility/redaction boundary and leak local paths or private transcript contents. Use a scoped transcript-recall mechanism or remove this suffix until the security model is explicit.
  - confidence: 0.88
- **[P2] Extract the final summary block after analysis:** `src/agents/compaction-analysis-strip.ts:42`
  - body: The regex returns the first `<summary>...</summary>` pair anywhere in the response. If the model mentions literal summary tags while reasoning inside `<analysis>`, this extracts an analysis fragment and drops the real final summary, so the persisted compaction context can lose the required sections. Parse after the closing analysis block or select the last valid summary block and add a regression case.
  - confidence: 0.86
