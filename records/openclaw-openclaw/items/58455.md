## Summary

Keep open: the PR is not implemented on current main and still has concrete security/correctness blockers. The follow-up commit addressed the earlier microCompact ordering and transcript filename feedback, but the raw transcript suffix and first-summary parser remain unsafe for merge.

## What This Changes

The PR adds microCompact helper modules/tests, appends analysis-scratchpad instructions to compaction, strips analysis output from summaries, and appends a full-session transcript path to compacted summaries.

## Best Possible Solution

Keep the microCompact direction after preserved-turn splitting, extract the intended final summary block robustly, and use a sanitized recall/export surface unless maintainers explicitly approve raw transcript self-service.

## Reproduction Assessment

Yes. The parser defect is reproducible by passing stripAnalysisBlock a response whose analysis text contains a literal `<summary>...</summary>` example before the real final summary, and the raw-transcript concern is source-inspectable against current sanitized compaction and sessions_history boundaries.

## Solution Assessment

No. The microCompact placement is now plausible, but the raw JSONL recovery path and first-summary regex are not the narrowest maintainable solution for improving compaction quality.
