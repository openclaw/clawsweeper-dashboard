## What This Changes

Workboard columns with enough cards to overflow should scroll while preserving each card title, labels, and footer instead of squeezing cards until content disappears.

## Best Possible Solution

## Reproduction Assessment

Yes, source-level. Current main has the reported grid scroll container without explicit implicit-row sizing plus clipped card titles, and the issue supplies Playwright measurements; I did not run a fresh browser repro in this read-only pass.

## Solution Assessment

Yes. Setting implicit grid rows to content height on .workboard-column__cards is the narrow owner-boundary fix because the renderer already emits the card titles correctly and the failure is layout compression in the scroll container.
