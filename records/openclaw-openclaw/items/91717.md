Fix the Workboard Control UI bug where overflowing columns squeeze grid rows until card titles and metadata disappear. On current main, `.workboard-column__cards` in `ui/src/styles/workboard.css` is a grid scroll container with no implicit row sizing, and cards clip their own content. Add the narrow CSS repair so each implicit row keeps card content height and the column scrolls instead of shrinking cards. Add or update focused browser/E2E coverage that creates enough cards in one Workboard column to overflow the visible column height and asserts every rendered card title remains visible/nonzero height. Keep the fix scoped to Workboard layout; do not take over or merge the broad Workboard polish work in https://github.com/openclaw/openclaw/pull/90057, do not change Workboard data model/config, and do not edit CHANGELOG.md.
