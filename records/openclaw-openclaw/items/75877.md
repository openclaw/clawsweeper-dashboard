## Summary

Keep open. Current main still wires Slack typing callbacks to `undefined` whenever group/channel visible replies resolve to message-tool-only, even though the shared typing policy and docs expect instant typing feedback in that mode.

## What This Changes

Restore Slack in-flight feedback for message-tool-only group/channel turns without re-enabling automatic visible replies.

## Best Possible Solution

Keep tool-only final/preview delivery suppression, but wire Slack typing and `typingReaction` whenever the typing policy allows it; handle lifecycle status reactions consistently with the existing status-reaction controller and #71329.

## Reproduction Assessment

Yes. A source-level reproduction is clear: set a Slack channel/group turn to `message_tool_only`, then `dispatch.ts` omits the typing callbacks, so `setSlackThreadStatus` and `reactSlackMessage` for `typingReaction` cannot run.

## Solution Assessment

Yes for the typing portion: decoupling Slack typing callbacks from visible-reply suppression is the narrow maintainable fix. Preview/native/block streaming should stay gated unless a separate product decision changes tool-only semantics.
