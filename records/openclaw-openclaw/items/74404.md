## Summary

Keep open. Current main has several adjacent mitigations for slow Gateway control-plane calls, runtime-deps churn, Telegram reply latency, and hung provider abort cleanup, but there is still no live packaged Ubuntu/Telegram/Minimax profile proving the reported CPU saturation, stuck heartbeat processing, WS/RPC timeouts, and cron timeout regression is fixed end to end.

## What This Changes

Reports a Linux Gateway regression where versions after 2026.4.22 become CPU-saturated during Telegram/provider work, causing agents, WS/RPC commands, heartbeat sessions, and cron jobs to stall.

## Best Possible Solution

Keep this open as an active Gateway liveness/performance regression. A maintainer should capture a packaged Linux profile with Telegram traffic and the reported provider route, compare 2026.4.20, an affected release, and current main, then land a targeted fix in the responsible owner layer with regression coverage.
