## Summary

Current `main` still has the reported Feishu gap: the message-tool path builds a markdown-only card from presentation fallback text, while a separate Feishu outbound renderer already knows how to render presentation buttons as native card actions.

## What This Changes

Render Feishu `message` tool `presentation.blocks` button groups as native Feishu interactive card buttons instead of plain fallback labels.

## Best Possible Solution

Unify the Feishu message-tool presentation-card builder with the existing native outbound presentation renderer so text, context, divider, and button blocks render consistently while unsupported controls still degrade safely.

## Reproduction Assessment

Yes, by source inspection: Feishu `message` tool sends with `presentation.blocks` call `buildFeishuPresentationCard`, and the current test expects a buttons block to become `- Run help` markdown instead of a native `action` element. I did not run a live Feishu send or local tests because this review is read-only.

## Solution Assessment

Yes, the narrow maintainable fix is to reuse or align with the existing `extensions/feishu/src/outbound.ts` native presentation-card mapping instead of keeping a second fallback-only renderer in `channel.ts`.
