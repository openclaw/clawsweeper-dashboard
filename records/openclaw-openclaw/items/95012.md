## Summary

The PR should stay open: the route-lease design is a plausible fix for #92460 and the manual `runId` plumbing has improved, but the current proof scripts and several tests still bind the lease helpers to `process.env` while only opening the temp DB through an options-only env. ## What This Changes

The PR adds a SQLite-backed `task_route_leases` table/module and wires cron/task execution to use runId-based delivery-route fallback for isolated cron completion delivery.

## Best Possible Solution

Keep the PR open until all lease proof/tests bind the same temp state env as the lease helpers, the current-head proof is rerun, and maintainers explicitly accept the persistent generic route-lease lifecycle.
