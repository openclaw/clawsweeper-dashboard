## Summary

Keep open; this PR is still not merge-ready. The cron fallback direction is relevant to the linked delivery-loss bug, but the patch widens lease acquisition to generic background tasks without tying those leases to the common terminal-delivery path or a production pruning owner, which recreates the lifecycle leak that a prior maintainer review rejected.

## What This Changes

The PR adds a SQLite-backed task-route lease table/module, wires isolated cron delivery-target resolution to use a run-id fallback, and adds regression tests plus repro scripts.

## Best Possible Solution

Own route leases in the shared background-task delivery lifecycle with tested acquire, resolve, settle, and prune/delete behavior for every runtime that acquires them, or keep this change cron-only until that shared lifecycle exists.

## Reproduction Assessment

Yes. Source inspection shows current main resolves cron delivery by stored/thread/main session presence and the linked report gives a concrete isolated-cron channel-only failure; I did not run a live cron send in this read-only review.

## Solution Assessment

No. The cron routability fallback is plausible, but the PR is not the best complete fix while generic leases are acquired outside the shared terminal-delivery settlement and pruning lifecycle.
