Keep open: current main still has a source-backed Codex transcript branch-sync risk, and I did not find a regression proving consecutive Codex app-server turns keep the mirrored final assistant reply on the active branch. Related #68582 is closed, but this report narrows a remaining stale-manager/orphaned-branch mechanism rather than only the older WebChat flash-and-disappear symptom.

## What This Changes

The issue asks for Codex app-server final replies to be integrated into the live active transcript branch so later session messages parent after the assistant reply.

## Best Possible Solution

Persist Codex app-server final replies through a manager-aware transcript path, or refresh the active manager after mirroring, so later appends parent after the assistant reply and `chat.history` remains ordered.

No high-confidence live reproduction was established in this read-only review.
