Keep open: current main still matches the reported Podman credential gap, and the latest shipped tag also starts setup/gateway containers without exposing Claude Code credentials while the Anthropic Claude CLI path requires them under the container home.

## What This Changes

The issue requests that Podman setup reuse host Claude CLI authentication by making the host Claude credential state visible inside the onboarding container.

## Best Possible Solution

Update the Podman launcher so setup and gateway containers safely expose the Claude Code auth files needed for Claude CLI reuse, with SELinux-compatible handling and focused regression coverage.

## Reproduction Assessment

Yes, from source inspection: the Podman container HOME has no Claude credentials while the Anthropic Claude CLI setup path reads credentials from HOME and emits the reported error when absent. I did not run the full Fedora/rootless Podman flow live.

## Solution Assessment

Yes, the best fix is a narrow Podman launcher credential staging or mount change; relying on CLAUDE_CONFIG_DIR is not a good direction because OpenClaw’s Claude CLI backend intentionally clears that override for managed runs.
