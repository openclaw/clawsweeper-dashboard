## Summary

Keep open: current `main` and the latest shipped tag still match the report. The Podman launcher exposes only OpenClaw state/workspace to the setup and gateway containers, while the Anthropic Claude CLI setup path reads Claude Code credentials from the container home and throws the reported error when they are absent; no duplicate or linked fix was found.

## What This Changes

The issue reports that rootless Podman onboarding cannot reuse a host Claude CLI login because the container cannot see the host Claude credential directory.

## Best Possible Solution

Update the Podman launcher so setup and gateway containers safely expose only the Claude Code auth files needed for Claude CLI reuse, with SELinux-compatible handling and regression coverage.

## Reproduction Assessment

Yes, at source level. The Podman launcher sets `HOME=/home/node` without exposing `.claude`, and the Anthropic setup path reads Claude CLI credentials from that container home before throwing the exact reported error; no live Podman run was performed.

## Solution Assessment

Yes. The narrow maintainable fix is in the Podman launcher/auth staging path, not in Anthropic credential parsing or an env override that the Claude CLI backend intentionally clears.
