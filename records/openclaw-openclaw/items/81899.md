## Summary

Keep open: current main still matches the report. The Podman setup and gateway containers set HOME to /home/node but mount only OpenClaw state and workspace, while Anthropic Claude CLI setup reads Claude Code credentials from the container home and throws the reported authentication error when they are absent.

## What This Changes

The issue requests fixing Podman onboarding so Anthropic Claude CLI auth inside the container can reuse the host Claude Code login.

## Best Possible Solution

Update the Podman launch and Quadlet setup surfaces to safely expose the required Claude Code auth state to setup and runtime containers, with SELinux-compatible handling and live rootless Podman proof.

## Reproduction Assessment

Yes. Current source gives a high-confidence reproduction path: run Podman setup with host Claude CLI auth available, choose Anthropic Claude CLI, and the containerized setup cannot see `/home/node/.claude/.credentials.json` because the launcher never exposes it.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Podman launcher/Quadlet credential exposure path, not in changing the Anthropic auth error or relying on `CLAUDE_CONFIG_DIR`, which the current Claude CLI backend clears for managed runs.
