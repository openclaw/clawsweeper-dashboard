## Summary

Keep open: current main and the latest shipped tag still match the report. The Podman launcher runs setup and gateway containers with container HOME and mounts only OpenClaw state/workspace, while the Anthropic Claude CLI setup path reads Claude Code credentials from HOME and throws the reported error when they are absent.

## What This Changes

The issue asks for Podman setup to expose host Claude Code credentials to the onboarding container so Anthropic Claude CLI auth succeeds.

## Best Possible Solution

Update the Podman launcher so setup and gateway containers safely stage or mount the Claude Code auth files needed by Claude CLI reuse, with SELinux-compatible options and regression coverage.

## Reproduction Assessment

Yes. Current main gives a high-confidence source reproduction: the Podman container paths do not expose Claude Code auth state, and the Anthropic setup path reads credentials from the container HOME before throwing the reported error; no live Podman run was performed.

## Solution Assessment

Yes. The narrow maintainable fix is in the Podman launcher/auth staging path, not in Anthropic credential detection or an env override that the Claude CLI backend intentionally clears.
