## Summary

Keep open: current main and the latest release still start the Podman onboarding container with HOME=/home/node but without exposing Claude CLI credentials, while the Anthropic Claude CLI setup path reads ~/.claude/.credentials.json and emits the reported error when that file is absent.

## What This Changes

Make rootless Podman onboarding reuse host Claude CLI credentials by mounting or safely staging ~/.claude into the container.

## Best Possible Solution

Update the Podman launcher and Quadlet path so setup and gateway containers can see scoped Claude Code auth files under /home/node/.claude with SELinux-compatible handling and focused regression coverage.

## Reproduction Assessment

Yes, as a source-level reproduction: Podman sets HOME=/home/node without mounting .claude, and the Anthropic Claude CLI setup path reads credentials from that home before throwing the reported error. I did not perform a live Fedora/rootless Podman run.

## Solution Assessment

Yes, the narrow maintainable fix is in the Podman launcher/Quadlet credential exposure path, not in the Anthropic provider or a CLAUDE_CONFIG_DIR override that OpenClaw-managed Claude CLI runs intentionally clear.
