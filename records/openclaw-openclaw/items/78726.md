## Summary

Keep open: the patch observes `deliverOutboundPayloads`, but the reported Telegram `message(action=send)` path currently runs through the gateway plugin action handler and direct Telegram send runtime, so the proposed correlation does not cover the failing path.

## What This Changes

The PR adds Telegram inbound-turn delivery correlation from outbound message-sent success, wires it into Telegram dispatch and outbound delivery, adds focused tests, and records a changelog entry.

## Best Possible Solution

Mark the active Telegram inbound turn delivered from the actual successful Telegram `message.action` send path, or add a generic post-action delivery signal carrying session key, channel, target, and account, then cover that gateway path in regression tests.

## Reproduction Assessment

Yes, as a source-level path, but not as a live run in this review. Current main records inbound Telegram reply delivery, while Telegram `message(action=send)` succeeds through the gateway plugin action path without updating that tracker before the empty-final fallback guard runs.

## Solution Assessment

No. The proposed notify hook sits in `deliverOutboundPayloads`, but the Telegram message tool path is a gateway plugin action that sends via `sendMessageTelegram`, so the narrow maintainable fix needs to observe that actual success path.
