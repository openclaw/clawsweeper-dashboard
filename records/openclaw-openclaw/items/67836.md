## Summary

Keep open. Current main does not implement the Memory v2 sidecar/ingest/rerank foundation, and the PR still has blocking correctness plus privacy-review concerns before it can merge.

## What This Changes

The PR adds a default-off SQLite-backed Memory v2 sidecar to `memory-core`, with ingest, rerank, dreaming shadow-touch, sidecar CLI commands, manifest config, docs, and tests.

## Best Possible Solution

Keep the implementation under maintainer review, split or fix the sidecar schema/store, ingest, rerank, dreaming, and CLI slices, and merge only after the persistence/privacy contract plus CLI/ranking semantics are settled.

## Reproduction Assessment

Yes for the review findings by source inspection of the PR head: omitted `--agent` expands through `resolveAgentIds`, stats/list call an initializer-backed opener, rejected write paths only log, and shadow touch runs before final result slicing. Not yet. A default-off foundation is a plausible direction, but this PR is too broad and still has correctness and security-review blockers; narrower reviewed slices or focused fixes are the safer path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Scope sidecar writes to one default agent:** `extensions/memory-core/src/cli.runtime.ts:2255`
  - body: The mutating sidecar commands call `resolveAgentIds`, which expands an omitted `--agent` to every configured agent even though the new command help says it defaults to the default agent. A plain `pin`, `status`, `salience`, or `supersede` can therefore mutate multiple sidecar databases unintentionally.
  - confidence: 0.91
- **[P2] Keep sidecar stats and list read-only:** `extensions/memory-core/src/cli.runtime.ts:2000`
  - body: `stats` and `list` open existing databases through `openSidecarDatabase`, which always runs schema initialization and metadata writes. These inspection commands are documented as read-only, so they should use a no-migration read opener or otherwise avoid writes.
  - confidence: 0.88
- **[P2] Fail rejected sidecar ref resolutions:** `extensions/memory-core/src/cli.runtime.ts:2229-2231`
  - body: When a ref prefix is missing or ambiguous, the renderer logs the rejection and the command still exits successfully. Scripts cannot distinguish a skipped mutation from a successful write, so rejected write targets should set a non-zero exit code.
  - confidence: 0.9
- **[P2] Fail invalid sidecar arguments:** `extensions/memory-core/src/cli.runtime.ts:2283-2289`
  - body: Invalid arguments such as an unknown status or salience value only print a warning and return without setting `process.exitCode`. Automation will treat the command as successful even though no mutation happened.
  - confidence: 0.87
- **[P2] Touch only surfaced search hits:** `extensions/memory-core/src/memory-v2/rerank/wrapper.ts:85-100`
  - body: `shadowOnRecall` records touches for every candidate passed into the wrapper, but `memory_search` sorts and slices after this call. Rows that are dropped by `maxResults` can still get `last_accessed_at` refreshed, skewing later recency scoring toward content the user never saw.
  - confidence: 0.84
- **[P2] Add a deterministic duplicate-location tie-breaker:** `extensions/memory-core/src/memory-v2/rerank/lookup.ts:31-35`
  - body: Duplicate `location_id` rows with equal salience and `last_accessed_at` are selected according to SQLite row order because the query has no `ORDER BY` and `preferCandidate` has no final tie-break. Add stable ordering or compare a stable field so rerank output cannot drift between runs.
  - confidence: 0.82
- **[P2] Add the required changelog entry:** `extensions/memory-core/src/cli.ts:273-275`
  - body: This PR adds user-facing config and CLI behavior, but the changed-file list does not include `CHANGELOG.md`. Repo policy requires user-facing `feat`, `fix`, and `perf` changes to carry a changelog entry before merge.
  - confidence: 0.86

## Risks / Open Questions

- The PR persists conversation-derived memory candidates in a new SQLite sidecar; the current redaction is best-effort and needs explicit privacy/data-retention review.
- Sidecar CLI commands can unexpectedly fan out across all configured agents when `--agent` is omitted, including mutating commands.
- Commands described as read-only can initialize or migrate the sidecar database, so inspection can write to disk or fail in read-only contexts.
- Rerank shadow-touch currently records recency for pre-slice candidates and duplicate-location signal selection lacks a stable final tie-breaker.
- The change is size XL across storage, search ranking, dreaming, CLI, manifest config, docs, and tests; this needs maintainer product/security review rather than automated cleanup.
