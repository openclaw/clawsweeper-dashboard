- **[P1] Scope sidecar writes to one default agent:** `extensions/memory-core/src/cli.runtime.ts:2255`
  - body: These new sidecar commands call resolveAgentIds, which expands an omitted --agent to every configured agent, while the command docs/help say the default is the default agent.   - confidence: 0.93
- **[P2] Keep sidecar inspections read-only:** `extensions/memory-core/src/cli.runtime.ts:2033`
  - body: The stats/list collection path opens an existing sidecar with openSidecarDatabase, which runs ensureSidecarSchema and writes meta/schema updates. Commands documented as read-only can therefore mutate the DB and fail in read-only contexts; inspection should use a no-migration read path or clearly stop being read-only.
  - confidence: 0.89
- **[P2] Return failure for rejected write inputs:** `extensions/memory-core/src/cli.runtime.ts:2283-2289`
  - body: Invalid sidecar write inputs only log a warning and return, and unresolved or ambiguous ref prefixes are rendered without setting a non-zero exit code. Automation will treat typos or rejected writes as success even though no mutation ran; set process.exitCode or use commander parser failures for these paths.
  - confidence: 0.9
- **[P2] Touch only results that were shown:** `extensions/memory-core/src/memory-v2/rerank/wrapper.ts:90-100`
  - body: shadowOnRecall writes touches for every result passed into the wrapper, but tools.ts passes the merged pre-sort/pre-limit candidate list. That refreshes last_accessed_at for memory rows the user never received, which can bias later recency scoring toward unseen content; move the touch after final sorting/slicing or pass the final surfaced list.
