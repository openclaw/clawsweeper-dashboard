## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve balanced corpus selection around rerank:** `extensions/memory-core/src/tools.ts:310-317`
  - body: Current main balances memory and wiki candidates for `corpus=all` via `mergeMemorySearchCorpusResults`, but the PR globally sorts and slices the merged list after rerank. That can let one corpus starve the other and regresses the mixed-corpus behavior the current helper protects.
  - confidence: 0.9
- **[P1] Scope sidecar writes to one default agent:** `extensions/memory-core/src/cli.runtime.ts:2255`
  - body: The sidecar write commands resolve omitted `--agent` through `resolveAgentIds`, which expands to every configured agent even though the new CLI help says the option defaults to the default agent. A plain pin/status/salience/supersede command can therefore mutate multiple agents' sidecar DBs unintentionally.
  - confidence: 0.9
- **[P2] Keep sidecar stats and list read-only:** `extensions/memory-core/src/cli.runtime.ts:2000`
  - body: `stats` and `list` open existing databases through `openSidecarDatabase`, which runs schema initialization and metadata writes. Commands documented as read-only should inspect without migrations or writes, otherwise they can mutate state and fail on read-only storage.
  - confidence: 0.88
- **[P2] Fail rejected sidecar write inputs:** `extensions/memory-core/src/cli.runtime.ts:2229-2231`
  - body: Missing, invalid, ambiguous, or unresolved write inputs only log a warning or skipped resolution and return without setting a non-zero exit code. Scripts cannot distinguish rejected writes from successful mutations.
  - confidence: 0.87
- **[P2] Touch only surfaced search hits:** `extensions/memory-core/src/memory-v2/rerank/wrapper.ts:90-100`
  - body: `shadowOnRecall` records touches for every candidate passed into the wrapper, but `memory_search` slices final output after rerank. Rows dropped from the returned result set can get `last_accessed_at` refreshed, skewing later recency scoring toward content the user never saw.
  - confidence: 0.84
- **[P2] Add a stable duplicate-location tie-breaker:** `extensions/memory-core/src/memory-v2/rerank/lookup.ts:31-35`
  - body: The lookup query has no `ORDER BY`, and equal salience plus equal `last_accessed_at` falls back to whichever row SQLite returns first. Duplicate `location_id` rows can therefore select different pinned/status signals between runs.
  - confidence: 0.82
