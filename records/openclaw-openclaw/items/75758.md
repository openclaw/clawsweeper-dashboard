## Summary

Keep open. Current main still has a Feishu streaming-card finalization gap: CardKit final update/close calls do not verify success, and the dispatcher can then suppress later final-text delivery after it believes the streaming card closed.

## What This Changes

Ensure Feishu streaming cards still render the final result after the CardKit streaming session times out around 10 minutes.

## Best Possible Solution

Keep the fix inside the Feishu plugin: make final CardKit update/close return a verified success signal, and fall back to a static card or text reply when finalization fails or expires before suppressing later final delivery.

## Reproduction Assessment

Yes for a source-level reproduction: simulate a failed or expired CardKit final update/close while FeishuStreamingSession.close currently returns, then deliver the final payload and observe dispatcher suppression. The live 10-minute Feishu timeout still needs an external fixture or log sample.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix if scoped to Feishu finalization and fallback delivery; a broad core channel change is not justified unless other plugins need the same contract.
