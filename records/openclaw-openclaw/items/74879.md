## Summary

Keep open: current main still routes QuickStart onboarding through the shell-completion cache generator, and the related timeout PR is still open rather than shipped. New discussion also reports a manual `completion --write-state` hang on `2026.4.29` and onboarding stalling after channel selection, so the central regression is not resolved.

## What This Changes

The issue asks OpenClaw to restore fast onboarding and responsive shell completion after regressions observed in `2026.4.26+` Docker installs.

## Best Possible Solution

Keep full plugin-command completion available for explicit refreshes, but make automatic onboarding/doctor cache work core-only or deferred, and bound the manual full refresh with progress or a useful failure path.

## Reproduction Assessment

Yes for the source-level completion path: a fresh QuickStart with no installed completion awaits `setupWizardShellCompletion`, which calls `ensureCompletionCacheExists` and blocks on the 30-second `completion --write-state` subprocess. The separate channel-selection stall has credible user confirmation but still needs a focused trace.

## Solution Assessment

Unclear because there is no patch on this issue yet. #72910 may reduce some timeout failures, but by itself it does not prove onboarding stops blocking on cache generation or that the manual full-plugin completion path no longer hangs.
