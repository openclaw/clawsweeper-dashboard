## Summary

Keep open: current main still leaves fresh onboarding and doctor on a blocking completion-cache path, and the latest discussion adds shipped Docker evidence that direct manual cache generation can still hang silently.

## What This Changes

The issue reports a regression where Docker-based onboarding and shell completion cache generation became much slower or timed out after the 2026.4.14 release line.

## Best Possible Solution

Keep automatic onboarding and doctor cache work bounded, core-only, or deferred, while making explicit `openclaw completion --write-state` observable and safe when full plugin-command completion is slow.

## Reproduction Assessment

Yes. A source-level reproduction is a fresh QuickStart without installed completion reaching `setupWizardShellCompletion`, awaiting `ensureCompletionCacheExists`, and spawning `completion --write-state`; the comments add shipped Docker evidence for the manual hang.

## Solution Assessment

Unclear because there is no patch on this issue. Raising timeouts alone may reduce one ETIMEDOUT case, but the safer fix should avoid blocking onboarding on full plugin-command completion generation.
