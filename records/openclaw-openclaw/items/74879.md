## Summary

Keep open: current main has a partial completion-registration cache mitigation, but the source still leaves onboarding/doctor completion cache generation on a blocking `completion --write-state` path and the manual command still loads plugin commands by default.

## What This Changes

The issue reports a regression where onboarding and shell completion are much slower on 2026.4.26+ than on 2026.4.14.

## Best Possible Solution

Keep the issue open and converge on a bounded or deferred onboarding/doctor cache path while making manual completion refresh observable and safe without unintentionally dropping needed plugin completions.

## Reproduction Assessment

Yes for a source-level reproduction path: a fresh QuickStart without installed completion reaches `setupWizardShellCompletion`, which calls `ensureCompletionCacheExists` and blocks on `completion --write-state`. The discussion adds shipped Docker evidence for the manual completion hang, but this review did not live-run Docker timing.

## Solution Assessment

Unclear because there is no patch on this issue and #72910 only partially overlaps the reported behavior. A narrower end state is to keep automatic onboarding/doctor completion work bounded or deferred, then separately decide whether manual `completion --write-state` should load full plugin commands by default.
