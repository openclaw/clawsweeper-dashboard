## Summary

Keep open: current main still leaves the completion-cache/onboarding slowdown path unresolved, and the linked #72910 work is open and only covers part of the update/doctor timeout surface.

## What This Changes

The issue requests restoring fast onboarding and shell completion behavior after a regression observed in Docker installs on 2026.4.26 and newer.

## Best Possible Solution

Separate automatic onboarding/doctor cache refresh from full plugin-command completion loading, make manual cache generation bounded and observable, and collect channel-specific Docker timing if the selection stall persists.

## Reproduction Assessment

Yes for the shell-completion portion: source inspection shows QuickStart and doctor can still await completion-cache generation, and the manual command still loads plugin commands unless an internal env flag is set. The channel-selection stall is unclear because the follow-up lacks the selected channel, logs, or a narrowed current-main timing path.

## Solution Assessment

Unclear because there is no patch on this issue and #72910 is only a partial mitigation. The narrow maintainable direction is to bound automatic onboarding/doctor work first, then make an explicit product call on full plugin-command completion for manual cache rebuilds.
