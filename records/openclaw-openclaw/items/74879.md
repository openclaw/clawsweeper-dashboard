## Summary

Keep open: current main still routes fresh onboarding and doctor completion repair through cache generation, and manual `completion --write-state` still loads plugin CLI commands by default; the open related PR only addresses part of the timeout surface.

## What This Changes

The issue reports a regression where Docker onboarding and shell-completion cache generation became much slower in OpenClaw 2026.4.26+ than in 2026.4.14.

## Best Possible Solution

Make automatic onboarding and doctor completion refresh bounded, visible, and preferably core-only or deferred, then define an explicit full plugin-command rebuild path with progress and timeout behavior.

## Reproduction Assessment

Yes for the completion-cache path: fresh QuickStart can await `ensureCompletionCacheExists`, which spawns `completion --write-state`, and a follow-up Docker report reproduced the manual command hanging on v2026.4.29. Unclear for the channel-selection hang because the comment identifies the prompt but does not include logs or a minimal failing fixture.

## Solution Assessment

Unclear because there is no patch on this issue, and #72910 only covers the update/doctor timeout class. The safer direction is to separate automatic setup from full plugin-command completion and then decide how manual plugin completion should behave.
