## Summary

Keep open: current main still has the completion-cache work on onboarding/doctor paths and manual `completion --write-state` still takes the full plugin-command route; the Docker follow-up reproduces the manual hang, while #72910 is open and only covers part of the timeout surface.

## What This Changes

Report asks to restore fast onboarding and shell-completion behavior for Docker installs where 2026.4.26+ takes minutes or times out.

## Best Possible Solution

Decouple automatic onboarding/doctor/update cache refresh from full plugin-command completion loading, make manual cache generation bounded and observable, and collect channel-specific Docker timing if selection still stalls.

## Reproduction Assessment

Yes for the shell-completion portion: source inspection shows current main still awaits completion-cache generation and manual write-state still loads plugin commands unless an internal env flag is set. The channel-selection portion is unclear without the selected channel and timing/log boundary.

## Solution Assessment

Unclear because there is no patch on this issue and #72910 is only a partial timeout/progress mitigation. The safer direction is to bound automatic setup/doctor work first, then make an explicit maintainer call on full plugin-command completion for manual rebuilds.
