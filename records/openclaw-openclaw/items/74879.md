## Summary

Keep open: current main still has the onboarding and doctor paths awaiting completion-cache generation, while manual `completion --write-state` still loads plugin CLI commands unless an internal skip env var is set. The related timeout PR covers only part of the reported surface, and newer comments add live Docker evidence that the manual path can still hang.

## What This Changes

The issue reports a regression where 2026.4.26+ Docker onboarding takes minutes and shell completion generation times out or hangs compared with 2026.4.14.

## Best Possible Solution

Move automatic onboarding and doctor cache refreshes to a bounded core-only or deferred path, and make full plugin-command completion rebuilds explicit, observable, and recoverable without blocking first-run setup.

## Reproduction Assessment

Yes for the completion-cache path: a fresh QuickStart without an installed cache reaches `setupWizardShellCompletion`, then `ensureCompletionCacheExists`, then a spawned `completion --write-state`. The channel-selection hang has fresh user confirmation but still needs a narrower trace of which selected channel/plugin blocks.

## Solution Assessment

Unclear because there is no patch on this issue and #72910 is only a partial timeout mitigation. Increasing the timeout alone is not the best durable fix if automatic setup and manual completion still run full plugin CLI registration silently.
