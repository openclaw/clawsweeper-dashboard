## Summary

Keep open: current main still has a source-backed onboarding/completion slowdown path, and recent Docker comments report the manual completion command can still hang while a separate channel-selection stall remains unresolved.

## What This Changes

The issue reports a regression where Docker onboarding on recent OpenClaw releases takes minutes and shell completion times out or hangs compared with 2026.4.14.

## Best Possible Solution

Separate automatic onboarding/doctor cache refresh from full plugin-command completion loading, make manual cache generation bounded and observable, and collect targeted Docker channel-setup logs if the selection-stage stall remains after that fix.

## Reproduction Assessment

Partly: the shell-completion portion is source-reproducible because QuickStart awaits ensureCompletionCacheExists, which spawns completion --write-state, and that command loads plugin CLI unless a private env flag is set. The channel-selection stall is not yet high-confidence because the current comment lacks logs or a narrowed setup adapter path.

## Solution Assessment

Unclear: there is no patch on this issue, and open #72910 only addresses the timeout/update-doctor portion. The safer maintainer direction is to keep automatic onboarding bounded or core-only while deciding what manual full plugin-command completion should do by default.
