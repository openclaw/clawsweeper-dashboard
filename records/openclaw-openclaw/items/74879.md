## Summary

Keep open: current main still keeps completion-cache generation on the onboarding/doctor path, and the manual completion command still loads plugin CLI commands by default; the linked PR addresses only part of the timeout surface.

## What This Changes

The issue reports a regression where recent Docker onboarding and shell completion cache generation can take minutes or hang instead of completing quickly.

## Best Possible Solution

Make automatic onboarding and doctor cache refresh bounded, core-only, or deferred, and give explicit manual completion cache generation progress plus a safe way to avoid full plugin-command loading when that path is slow.

## Reproduction Assessment

Yes for the completion-cache path: fresh QuickStart without an installed cache reaches setupWizardShellCompletion and awaits ensureCompletionCacheExists, which spawns completion --write-state. The shipped Docker comments add a live manual-command reproduction, while the channel-selection hang still needs a narrower repro.

## Solution Assessment

Unclear because there is no patch on this issue and #72910 only covers the update/doctor timeout class. A durable fix should separate automatic setup from full plugin-command completion and make the manual path observable.
