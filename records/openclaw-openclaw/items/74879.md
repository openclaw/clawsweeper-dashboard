## Summary

Keep open: current main still leaves a blocking onboarding completion-cache path and the manual completion command can still load the full plugin command tree. The related #72910 is open but appears to cover timeout/progress behavior rather than the full onboarding/manual hang reported here.

## What This Changes

The report asks to fix a recent Docker/umbrelOS onboarding and shell-completion slowdown in 2026.4.26+ compared with 2026.4.14.

## Best Possible Solution

Use a bounded core-only or deferred cache path for automatic onboarding/doctor completion work, while separately making explicit `openclaw completion --write-state` observable and safe when full plugin-command completion is slow.

## Reproduction Assessment

Yes for a source-level reproduction path: a fresh QuickStart without installed completion reaches `setupWizardShellCompletion`, calls `ensureCompletionCacheExists`, and blocks on `completion --write-state`. Reporter comments add shipped Docker evidence for the manual completion hang on 2026.4.29, but this review did not run a live Docker timing repro.

## Solution Assessment

Unclear because there is no patch on this issue and #72910 appears to address timeout/progress behavior rather than removing the onboarding blocking path. A core-only or deferred automatic cache path looks narrower than only raising the timeout, because it avoids making onboarding wait on full plugin-command completion.
