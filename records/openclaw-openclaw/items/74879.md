## Summary

Keep open: current main still has unresolved onboarding/completion slowdown paths, with source-backed evidence for the completion-cache path and fresh Docker discussion confirming manual completion can still hang.

## What This Changes

The issue requests a fix for a regression where OpenClaw 2026.4.26+ onboarding and shell completion became much slower than earlier releases.

## Best Possible Solution

Separate automatic onboarding/doctor cache refresh from full plugin-command completion loading, make manual cache generation observable and bounded, and narrow the channel-selection stall with channel-specific Docker logs.

## Reproduction Assessment

Yes for the shell-completion portion: source inspection shows fresh QuickStart awaits ensureCompletionCacheExists, which spawns completion --write-state through the 30s doctor/setup path, and a Docker user reproduced manual write-state hanging past 60s. The separate channel-selection stall is still unclear because the comment lacks a narrowed current-main reproduction path.

## Solution Assessment

Unclear because there is no patch on this issue and open #72910 only addresses part of the update/doctor timeout surface. The safer direction is to keep automatic setup bounded or core-only while deciding whether manual completion should retain full plugin-command coverage by default.
