## Summary

Keep open. Current `main` and the latest release still enforce and document the remote plain-HTTP Control UI rejection, but startup logging does not emit the requested warning; the issue is also paired with the open closing PR https://github.com/openclaw/openclaw/pull/73163.

## What This Changes

Request a gateway startup warning when Control UI is enabled in a configuration that will reject non-local plain-HTTP browser connections without the device-auth bypass.

## Best Possible Solution

Land or replace the linked PR with a narrow startup-log warning and focused regression tests while preserving current Control UI device-identity enforcement.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: remote plain HTTP lacks `crypto.subtle`, the UI skips device identity, and the gateway rejects the device-less non-local Control UI connection with `CONTROL_UI_DEVICE_IDENTITY_REQUIRED`.

## Solution Assessment

Yes for the diagnostic direction. A startup warning is the narrow maintainable fix because it adds operator guidance without relaxing authentication; the linked PR is the right vehicle to finish or replace that work.
