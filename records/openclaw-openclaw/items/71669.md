## Summary

Keep open. Current main still enforces and documents the remote plain-HTTP Control UI rejection, but the gateway startup logger does not emit the requested targeted warning; open PR #73163 is the active implementation candidate and should be reviewed, landed, replaced, or closed first.

## What This Changes

Request a gateway startup warning when Control UI is enabled in a configuration where non-localhost plain-HTTP browsers will be rejected for missing device identity.

## Best Possible Solution

Land or replace #73163 with a narrow startup-log warning and regression tests while keeping the existing Control UI device-identity enforcement unchanged.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a remote plain-HTTP Control UI skips device identity, the gateway rejects it with CONTROL_UI_DEVICE_IDENTITY_REQUIRED, and current startup logging has no targeted warning for that configuration.

## Solution Assessment

Yes. A log-only startup warning is the narrowest maintainable fix because it improves operator diagnostics without relaxing authentication or changing WebSocket policy.
