The PR extends paired-device platform-version refresh handling so `openclaw-macos` can refresh same-family `macos ...` platform labels without a metadata-upgrade approval, and adds table cases for that behavior.

## Best Possible Solution

Ship the platform-family refresh approach once a real patched macOS app/gateway reconnect demonstrates that the approval loop is gone.

## Reproduction Assessment

Yes, this is likely the best code path: extending `resolvePinnedClientMetadata` prevents same-family OS-version drift from entering the repair-pairing loop, instead of stretching pending request TTLs or adding another approval workaround. The remaining blocker is after-fix real behavior proof.
