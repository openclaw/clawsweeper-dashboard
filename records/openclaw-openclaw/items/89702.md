## Summary

Keep open: current release/main still lets the macOS node-mode app hit metadata-upgrade on OS-version-only platform drift, and the patch follows the existing iOS/Android platform-family refresh pattern. I found no blocking code defect, but the external PR still needs after-fix real gateway/Mac proof and maintainer acceptance of the pairing-boundary change.

## What This Changes

The PR adds `openclaw-macos` to the gateway platform-family refresh normalization and extends `resolvePinnedClientMetadata` coverage for macOS OS-version changes.

## Best Possible Solution

Land a server-side macOS platform-family refresh once real patched-gateway proof shows node-mode Mac reconnect updates the paired platform without an approval loop and maintainers accept the version string as display metadata.

## Reproduction Assessment

Yes. Source inspection plus the PR's before-fix logs show `openclaw-macos` node-mode sends a macOS versioned platform label, current main does not normalize that family, and direct-local metadata-upgrade auto-approval is limited to native UI mode.

## Solution Assessment

Yes, if maintainers accept treating same-family macOS platform labels as display metadata like iOS/Android. The safer alternative is a locality-gated metadata-upgrade path for macOS node-mode, but that would be a different policy than the existing platform-family helper.
