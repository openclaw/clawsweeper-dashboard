## Summary

Keep open: the PR targets a valid source-reproducible stale external-CLI OAuth path, but the current diff can write credentials during read-only auth-store loads and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR marks stale external CLI OAuth overlays as persistable, writes them back through the auth-profile lock helper, and adds unit coverage for Claude CLI stale-profile write-back.

## Best Possible Solution

Persist fresher matching non-bootstrapOnly external CLI credentials only from write-authorized auth resolution paths, preserve read-only/status behavior, and include regression coverage plus redacted real runtime proof.

## Reproduction Assessment

Yes, source-reproducible: current main resolves a fresh external CLI credential when the stored `anthropic:claude-cli` OAuth profile is expired, returns it as a runtime overlay, and emits the reported bootstrap DEBUG path. I did not run a live Windows/Claude CLI refresh cycle.

## Solution Assessment

No, not as written: persisting fresh matching credentials is a plausible fix, but placing the write-back inside the generic overlay helper also mutates read-only/status/secrets paths. The safer implementation should thread write permission through the auth-store path or move persistence to a write-authorized caller.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Skip write-back from read-only auth loads:** `src/agents/auth-profiles/external-auth.ts:79-87`
  - body: The new write-back runs inside `overlayExternalAuthProfiles`, but that helper is also used by `loadAuthProfileStoreForRuntime(..., { readOnly: true })` and `loadAuthProfileStoreForSecretsRuntime`. A read-only status/secrets load with an expired external CLI profile can now call `upsertAuthProfileWithLock` and mutate `auth-profiles.json`, violating the existing read-only external-overlay contract. Thread a write-allowed flag through this path or move the persistence into a non-read-only caller.
  - confidence: 0.88

## Security Review

Status: needs_attention

That broadens where credential material is written and should be constrained to write-authorized paths.
