## Summary

Keep open; the PR is a plausible narrow fix shape, but it is not merge-ready because it only closes ACP children matched by `parentSessionKey` in the parent store while normal `sessions_spawn(runtime="acp")` rows are written with `spawnedBy` and can live in the target agent store.

## What This Changes

The PR adds gateway reset/delete cleanup that enumerates child ACP session entries by `parentSessionKey`, closes them concurrently, and adds reset/delete/concurrency tests plus a changelog entry.

## Best Possible Solution

Keep the concurrent best-effort cleanup shape, but enumerate relevant ACP session entries across agent stores and close direct children matched by either `spawnedBy` or `parentSessionKey`, with reset/delete tests and proof using actual `sessions_spawn` rows.

## Reproduction Assessment

Yes, source-reproducible: current main closes only the reset/delete target ACP entry, and actual ACP spawn writes child rows with `spawnedBy`. I did not run the multi-hour live process leak in this read-only review.

## Solution Assessment

No: the proposed implementation is too narrow for the real spawned-session contract. The maintainable fix should match both lineage fields and cover target-agent stores before merge.
