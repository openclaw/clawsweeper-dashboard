## Summary

Keep open. Current main still recognizes base OpenAI Codex `gpt-5.2` as a modern model, but the provider-owned xhigh thinking allowlist still omits that exact id, and the linked one-off PRs remain closed unmerged.

## What This Changes

Request to expose `xhigh` thinking for `openai-codex/gpt-5.2` by adding the base model id to the OpenAI Codex xhigh allowlist.

## Best Possible Solution

Make the OpenAI Codex provider policy explicit for base `gpt-5.2`: either allow `xhigh` with focused regression coverage, or document that only the codex-suffixed GPT-5.2 route supports it.

## Reproduction Assessment

Yes. Source inspection gives a deterministic reproduction path: `resolveOpenAICodexThinkingProfile("gpt-5.2")` is built from an allowlist that does not contain `gpt-5.2`, so `listThinkingLevels("openai-codex", "gpt-5.2")` will not expose `xhigh`.

## Solution Assessment
