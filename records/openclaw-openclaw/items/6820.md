## Summary

Keep open. Current main and `v2026.5.2` still omit base `gpt-5.2` from the OpenAI Codex xhigh profile while recognizing it as a modern Codex model; prior one-off fix PRs are closed unmerged, and the broader RFC is context rather than a replacement for this concrete policy decision.

## What This Changes

The issue requests exposing `xhigh` for base `openai-codex/gpt-5.2` by adding it to the OpenAI Codex xhigh thinking allowlist.

## Best Possible Solution

Make the OpenAI Codex provider policy explicit for base `gpt-5.2`: either allow `xhigh` with focused regression coverage, or document/reject it as high-only while keeping the broader forward-compat design in #37804 separate.

## Reproduction Assessment

Yes. Source inspection gives a deterministic path: `resolveOpenAICodexThinkingProfile("gpt-5.2")` is built from an xhigh allowlist that omits `gpt-5.2`, so `listThinkingLevels("openai-codex", "gpt-5.2")` will not expose `xhigh`.

## Solution Assessment

Unclear until maintainers confirm the provider capability policy. If confirmed, the narrowest maintainable fix is provider-owned: add `gpt-5.2` to `OPENAI_CODEX_XHIGH_MODEL_IDS` and cover both the exposed thinking levels and hint/profile behavior.
