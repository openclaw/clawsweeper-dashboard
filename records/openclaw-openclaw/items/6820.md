## Summary

Keep open: current main still omits base `openai-codex/gpt-5.2` from the OpenAI Codex xhigh thinking policy, while the same provider recognizes it as a modern/reasoning model. This is still a real provider-policy gap, but enabling it needs maintainer confirmation of the upstream capability.

## What This Changes

Request to expose the `xhigh` thinking level for `openai-codex/gpt-5.2` by adding it to the OpenAI Codex xhigh model policy.

## Best Possible Solution

Decide the provider capability policy, then either add `gpt-5.2` to `OPENAI_CODEX_XHIGH_MODEL_IDS` with focused regression coverage or document that base OpenAI Codex GPT-5.2 is intentionally high-only.

## Reproduction Assessment

Yes, from source inspection: `openai-codex/gpt-5.2` routes through the OpenAI Codex thinking profile, and that profile's xhigh allowlist lacks `gpt-5.2`, so `listThinkingLevels` will not include `xhigh`. I did not run tests because this review was read-only and source inspection already establishes the path.

## Solution Assessment

Unclear. The one-line allowlist addition plus a regression test is the narrow implementation if maintainers confirm upstream xhigh support; otherwise the maintainable solution is to leave the policy as-is and document the high-only behavior.
