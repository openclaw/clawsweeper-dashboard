## What This Changes

The PR guards plugin agent event subscription dispatch against unreadable plugin-owned subscription metadata and adds a regression proving healthy subscribers still run after a poisoned registry row.

## Best Possible Solution

Land the narrow dispatcher guard after maintainer readiness, preserving the plugin API shape while skipping only malformed subscription rows and continuing healthy subscribers.

## Reproduction Assessment

Yes from source: current main and v2026.6.8 read `registration.subscription.streams` and later `registration.subscription.id` without a metadata guard, so a throwing subscription getter can abort dispatch before later healthy subscribers run. I did not run tests in this read-only review.

## Solution Assessment

Yes. Capturing and validating the plugin-owned metadata once in `dispatchPluginAgentEventSubscriptions()` is the narrowest maintainable fix; moving this into registry validation alone would not protect already-active or otherwise poisoned runtime registry rows.
