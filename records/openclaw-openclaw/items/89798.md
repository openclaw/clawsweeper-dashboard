## Label Justifications

## Summary

Keep open: the bugfix is useful, but the current head still changes agent-event handlers from a subscription-object method call to a bare function call, which can break method-style JavaScript plugin callbacks that rely on `this`. The PR is also draft and maintainer-labeled, so it needs author or maintainer follow-up rather than cleanup closure.

## What This Changes

The PR guards plugin agent-event subscription metadata during dispatch and adds a contract regression for malformed subscription metadata.

## Best Possible Solution

Keep the metadata guard, retain the validated subscription object, call the handler with that subscription as receiver, add a method-style receiver regression, and refresh focused plugin contract validation before marking ready.

## Reproduction Assessment

Yes. Current main dereferences `registration.subscription.streams` before a guard, so a throwing subscription getter can abort dispatch before healthy subscribers run; the PR-head receiver regression is also source-reproducible with a method-style `handle()` that reads `this`.

## Solution Assessment

No. Guarding subscription metadata in `src/plugins/host-hook-runtime.ts` is the right layer, but the implementation should preserve the original subscription receiver while avoiding fragile re-reads.
