## Summary

Keep open: current main and the latest release still have unguarded agent-event subscription metadata reads, while this protected maintainer draft contains a focused guard plus regression coverage. I found no blocking patch defect; the remaining action is maintainer readiness and exact-head validation rather than conservative cleanup close.

## What This Changes

The PR guards plugin agent event subscription metadata access in the host dispatcher and adds a contract regression proving a poisoned subscription row does not block a healthy subscriber.

## Best Possible Solution

Land the narrow dispatcher guard after maintainer readiness, preserving the plugin API shape while skipping only malformed subscription rows and continuing healthy subscribers.

## Reproduction Assessment

Yes. Current main and v2026.6.8 read `registration.subscription.streams` before a per-registration guard, so a throwing subscription getter can abort dispatch before a later healthy subscription runs.

## Solution Assessment

Yes. Capturing and validating the plugin-owned metadata once inside `dispatchPluginAgentEventSubscriptions()` is the narrowest maintainable fix; registry validation alone would not protect an already-active poisoned registry row.
