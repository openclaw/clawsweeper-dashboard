## Summary

Keep open: current main still lacks the requested skill-driven plugin context visibility contract, and the related landed work only improves skills visibility/debuggability rather than implementing `metadata.openclaw.activates.plugins`. This remains a public skill/plugin API and request-time tool-planning design decision, not a cleanup close.

## What This Changes

The issue requests a new `metadata.openclaw.activates.plugins` skill metadata contract so matched skills can make selected plugin tool schemas visible per turn.

## Best Possible Solution

Define an additive public skill activation contract, parse and preserve it in skill snapshots, thread matched-skill activation into plugin tool planning, and document/test precedence with explicit tool deny/allow policy and plugin-bundled skills.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/API proposal. Source inspection gives a high-confidence current-main gap check: the metadata type, parser, snapshot, docs, and plugin tool assembly do not implement `activates.plugins`.

## Solution Assessment

Unclear as a final implementation plan. The proposed direction is plausible, but maintainers should first settle the public contract and compatibility rules for tool policy, plugin disablement, optional tools, and plugin-bundled skills.
