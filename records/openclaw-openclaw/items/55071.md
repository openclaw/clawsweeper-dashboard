## Summary

Keep open. Current main still lacks the requested skill-driven plugin context visibility: `metadata.openclaw.activates.plugins` is not represented in skill metadata parsing, snapshots, docs, or request-time plugin tool resolution, so this remains a public API/design request rather than cleanup.

## What This Changes

The issue requests a new skill metadata contract where matched skills can activate plugin tool visibility per turn through `metadata.openclaw.activates.plugins`.

## Best Possible Solution

Keep this open for a maintainer-owned, backward-compatible activation design that parses skill activation metadata, carries activated plugin ids through skill snapshots, applies them during request-time plugin tool planning, and documents/tests the precedence rules.
