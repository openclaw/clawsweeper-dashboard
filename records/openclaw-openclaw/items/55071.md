## Summary

Keep open: current main and the latest release still lack the requested skill-driven `metadata.openclaw.activates.plugins` contract. Existing Tool Search and the merged skills visibility work reduce adjacent token-pressure and observability problems, but they do not make matched skills activate plugin tool schemas or plugin-bundled skills per turn.

## What This Changes

The issue requests a public `metadata.openclaw.activates.plugins` skill metadata contract so matched skills can expose plugin tool schemas and bundled plugin skills only when relevant.

## Best Possible Solution

Keep this open for maintainer API design of one backward-compatible skill-to-plugin activation contract that threads parsed metadata through skill snapshots into request-time tool planning with docs and precedence tests.
