## Summary

Keep open: current main and the latest release still lack the requested `metadata.openclaw.activates.plugins` contract and any matched-skill input to plugin tool visibility. The merged skills-check work is useful groundwork, but the discussion explicitly says it does not implement this proposal.

## What This Changes

The issue requests a new SKILL.md metadata contract that lets matched skills activate selected plugin tools and bundled skills per turn.

## Best Possible Solution

Define a backward-compatible public skill activation contract, thread parsed activation metadata through skill snapshots into request-time plugin tool planning, and document/test deny/allow precedence, optional tools, plugin disablement, bundled skills, and prompt-cache behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source and docs inspection give a high-confidence gap check because current metadata, parser, snapshots, docs, and plugin tool planning paths do not carry `activates.plugins`.

## Solution Assessment

Unclear until maintainers settle the public API and compatibility semantics. The maintainable direction is an additive metadata contract wired into request-time tool planning, not a one-off filter after plugin tools are already resolved.
