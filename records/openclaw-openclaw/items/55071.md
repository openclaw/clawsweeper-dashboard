## Summary

Keep this open: current main and the latest shipped release still do not implement `metadata.openclaw.activates.plugins`, and the related merged work only improves skill visibility diagnostics rather than skill-driven plugin tool activation.

## What This Changes

Request adding `metadata.openclaw.activates.plugins` so matched skills can dynamically expose plugin tool schemas and bundled plugin skills per turn.

## Best Possible Solution

Define a stable skill activation contract, or explicitly choose Tool Search/dynamic tools as the supported path, then thread the chosen design through metadata parsing, skill snapshots/matching, request-time plugin tool planning, docs, and focused precedence tests.

## Reproduction Assessment

Not applicable; this is a feature request rather than a bug report. Source and docs inspection show current main has no `activates.plugins` contract to exercise.

## Solution Assessment

Unclear; the proposed field is plausible, but it changes the public skill/plugin contract and request-time tool visibility semantics. Maintainers should choose the API and precedence model before implementation.
