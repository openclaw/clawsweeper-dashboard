## Summary

Keep open: current main has generic Tool Search and skill visibility diagnostics, but it still lacks a `metadata.openclaw.activates.plugins` contract or request-time skill-driven plugin tool activation. This remains a public skill/plugin API decision rather than cleanup closure.

## What This Changes

The issue requests a new `metadata.openclaw.activates.plugins` skill metadata contract that would expose plugin tools and bundled skills only when a matched skill activates that plugin.

## Best Possible Solution

Keep this open for a maintainer product/API decision that either defines an additive skill-driven plugin activation contract integrated with existing tool policy and Tool Search, or explicitly consolidates the need under the broader conditional-injection tracker.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection gives a high-confidence gap check: current main has no `activates.plugins` metadata field, parser path, snapshot data, or request-time skill-driven plugin tool selection.

## Solution Assessment

Unclear until maintainers choose the product direction. The requested contract is plausible, but Tool Search already provides a generic opt-in tool-schema compaction path, so the best solution may be a smaller API extension or consolidation with the broader conditional-injection work.
