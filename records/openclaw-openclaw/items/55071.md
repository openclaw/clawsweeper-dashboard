## Summary

Keep open: current main still lacks the requested `metadata.openclaw.activates.plugins` contract, and the latest human follow-up says the merged skills visibility work is groundwork rather than an implementation.

## What This Changes

Add a public skill metadata contract so matched skills can activate plugin tool schemas per turn instead of exposing every eligible plugin tool.

## Best Possible Solution

Keep this open for maintainer design of a backward-compatible activation contract that parses skill metadata, threads matched-skill activation into plugin tool planning, and documents/tests deny/allow precedence.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature/API request; the high-confidence check is current-main source and docs inspection showing `activates.plugins` is not parsed, stored, documented, or consumed during plugin tool assembly.

## Solution Assessment

Is this the best way to solve the issue? Unclear. Plugin-level activation fits the existing manifest-first direction, but the exact public skill contract and security/precedence behavior need maintainer API review before implementation.
