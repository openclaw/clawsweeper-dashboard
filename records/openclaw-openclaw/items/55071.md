## Summary

Keep open: current `main` still lacks the requested skill-driven plugin context visibility contract, and the merged skill visibility work is explicitly described in the discussion as groundwork rather than an `activates.plugins` implementation.

## What This Changes

The issue requests a new `metadata.openclaw.activates.plugins` skill metadata contract that would expose plugin tool schemas and plugin-bundled skills only when matched skills activate those plugins.

## Best Possible Solution

Keep this open for maintainer design of a backward-compatible skill activation contract that parses activation metadata, threads it through skill matching/snapshots, filters request-time plugin tools safely, and documents/tests precedence rules.

## Reproduction Assessment

Not applicable. This is a feature/API request, and the high-confidence check is source and docs inspection showing the requested `activates.plugins` contract is absent on current `main`.

## Solution Assessment

Unclear. Plugin-level activation fits the existing manifest-first plugin boundary, but the exact public metadata schema, activation timing, and security precedence need maintainer API/product review.
