## Summary

Keep open: current main still lacks the requested `metadata.openclaw.activates.plugins` skill contract and request-time plugin tool visibility, and the merged visibility work at https://github.com/openclaw/openclaw/pull/75983 is explicitly described as groundwork rather than the implementation.

## What This Changes

Request adds an opt-in `metadata.openclaw.activates.plugins` skill metadata contract so matched skills can make plugin tools and bundled skills visible per turn.

## Best Possible Solution

Define a backward-compatible public skill activation contract, thread parsed activation metadata through skill snapshots into request-time plugin tool planning, and document/test deny/allow precedence, optional tools, plugin disablement, bundled skills, and prompt-cache behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection gives a high-confidence gap check because current metadata, parser, snapshots, docs, and plugin tool planning paths do not carry `activates.plugins`.

## Solution Assessment

Unclear until maintainers settle the public API and compatibility semantics. The maintainable direction is an additive metadata contract wired into request-time tool planning, not a one-off filter after plugin tools are already resolved.
