## Summary

Keep open: current main still does not implement the requested skill-driven plugin context visibility contract, and the latest related maintainer discussion says the merged skills visibility work is groundwork rather than an `activates.plugins` implementation.

## What This Changes

The issue requests a new `metadata.openclaw.activates.plugins` skill metadata contract that would expose plugin tools and plugin-bundled skills only when a matched skill activates that plugin for the current turn.

## Best Possible Solution

Keep this open for maintainer design of a backward-compatible skill activation contract that parses activation metadata, threads it through skill snapshots/matching, filters request-time plugin tool exposure safely, and documents/tests precedence rules.

## Reproduction Assessment

Not applicable. This is a feature/API request, and the high-confidence check is source and docs inspection showing the requested `metadata.openclaw.activates.plugins` contract is absent on current main.

## Solution Assessment

Unclear. Plugin-level activation fits the manifest-first plugin boundary, but the public schema, activation timing, failure mode, and security precedence need maintainer API/product review before implementation.
