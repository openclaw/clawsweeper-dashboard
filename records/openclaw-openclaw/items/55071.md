## What This Changes

The issue requests a new `metadata.openclaw.activates.plugins` SKILL.md metadata field so matched skills can control per-turn plugin tool visibility.

## Best Possible Solution

Define an additive, maintainer-approved core skill activation contract that parses activation metadata, threads it through skill snapshots into request-time plugin tool planning, preserves existing tool policy precedence, and documents/tests plugin skill behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. The high-confidence check is source and docs inspection showing the requested `metadata.openclaw.activates.plugins` contract is absent on current main.

## Solution Assessment

Unclear until maintainers decide the public API and request-time semantics. Plugin-level activation is plausible, but the solution must preserve current tool policy, plugin disablement, optional-tool behavior, Tool Search boundaries, and prompt-cache determinism.
