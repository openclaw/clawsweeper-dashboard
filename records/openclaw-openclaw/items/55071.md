## What This Changes

Request adds `metadata.openclaw.activates.plugins` so matched skills can expose specific plugin tool schemas and bundled skills to the model context per turn.

## Best Possible Solution

Define an additive skill activation contract that parses and carries activated plugin ids through skill snapshots into request-time tool planning, with docs and focused tests for deny/allow precedence, plugin disablement, optional tools, and plugin-bundled skills.

## Solution Assessment
