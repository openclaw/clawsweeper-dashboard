## Summary

Keep open: current main and the latest release still lack a `metadata.openclaw.activates.plugins` contract or any matched-skill path that turns plugin tool schemas or bundled plugin skills on per turn. The merged skills visibility work and Tool Search reduce adjacent observability/token-pressure problems, but neither supersedes this specific public skill/plugin context-visibility design.

## What This Changes

The issue requests a new SKILL.md metadata field that lets matched skills activate plugin tools and plugin-bundled skills for the current turn.

## Best Possible Solution

Keep this as a focused maintainer product/API design item for one generic skill-to-plugin context visibility contract that threads parsed metadata through skill snapshots into request-time tool planning, with docs and tests for policy precedence.
