## What This Changes

The issue requests a new SKILL.md metadata field that lets matched skills activate plugin tool schemas and plugin-bundled skills on a per-turn basis.

## Best Possible Solution

Keep this open for a maintainer-approved skill-to-plugin activation contract that threads parsed metadata through skill snapshots into request-time plugin tool planning, with docs and precedence tests.
