## Summary

Keep open: current main still does not implement the requested skill-driven plugin visibility contract, and the related merged visibility work is explicitly documented as groundwork rather than an `activates.plugins` implementation.

## What This Changes

The issue requests a new `metadata.openclaw.activates.plugins` SKILL.md contract that would expose plugin tools and plugin-bundled skills only when a matching skill activates that plugin.

## Best Possible Solution

Define an additive, maintainer-approved skill activation contract that parses activation metadata, threads it into request-time plugin tool planning, preserves existing tool policy precedence, and documents/tests plugin-bundled skill behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. The high-confidence check is source and docs inspection showing that `metadata.openclaw.activates.plugins` is absent on current main.

## Solution Assessment

Unclear until maintainers decide the public API and request-time semantics. Plugin-level activation is plausible, but the design must preserve existing deny/allow policy, plugin disablement, optional-tool behavior, Tool Search boundaries, and prompt-cache stability.
