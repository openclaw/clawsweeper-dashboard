Request a `metadata.openclaw.activates.plugins` SKILL.md contract that exposes plugin tools and plugin-bundled skills only when matched skills activate them per turn.

## Best Possible Solution

Define an additive public skill activation contract that threads parsed metadata through skill snapshots into request-time plugin tool planning, with docs and tests for compatibility defaults, deny precedence, optional tools, plugin disablement, and bundled plugin skills.
