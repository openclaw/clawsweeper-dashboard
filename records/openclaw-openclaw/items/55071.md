## Summary

Keep open; current main still lacks `metadata.openclaw.activates.plugins` and any matched-skill path that turns plugin tool schemas or bundled plugin skills on per turn. Tool Search and the merged skills visibility PR are useful adjacent groundwork, but neither implements skill-driven plugin context visibility, and no open related issue fully supersedes this exact request.

## What This Changes

The issue requests a new skill metadata contract where matched skills can activate plugin-owned tool and skill context for the current turn.

## Best Possible Solution

Keep this open as a focused product/API design item: define one generic skill-to-context visibility contract that reuses plugin ownership metadata, existing tool policy, and Tool Search/catalog mechanics, then update schema, docs, diagnostics, and focused tests together.

## Reproduction Assessment

No bug reproduction is applicable because this is a new feature request. Source inspection gives high confidence that current main and the latest release lack the requested `activates.plugins` contract and matched-skill plugin activation behavior.

## Solution Assessment

Unclear as an implementation proposal: the goal is valid, but the exact `activates.plugins` shape needs maintainer API design so it composes with existing plugin enablement, optional tools, agent skill allowlists, and Tool Search.
