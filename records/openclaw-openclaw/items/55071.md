## Summary

Keep open: current main and the latest release still do not define, parse, snapshot, or apply `activates.plugins`, and the related merged work only improves skill visibility diagnostics. This is a public skill/plugin context API decision, not conservative cleanup.

## What This Changes

The issue requests a new `metadata.openclaw.activates.plugins` skill metadata contract so matched skills can dynamically expose selected plugin tool schemas per turn.

## Best Possible Solution

Define an additive, maintainer-approved skill activation contract and thread it from frontmatter parsing through skill snapshots/status into request-time plugin tool planning, with docs and tests for deny/allow precedence, disabled plugins, optional tools, plugin skills, and prompt-cache behavior.
