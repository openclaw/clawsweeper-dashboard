## Summary

Keep open: current main still lacks the requested skill-driven plugin context visibility contract, and the human follow-up explicitly says the merged skills visibility work is groundwork rather than an `activates.plugins` implementation.

## What This Changes

The issue requests a new `metadata.openclaw.activates.plugins` skill metadata contract so matched skills can make selected plugin tools and bundled skills visible per turn.

## Best Possible Solution

Design a backward-compatible skill activation contract that parses `activates.plugins`, carries it through matched skill state, filters request-time plugin tools safely, and documents/tests deny/allow precedence plus plugin-bundled skill behavior.
