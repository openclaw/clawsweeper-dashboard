## What This Changes

The issue requests a new `metadata.openclaw.activates.plugins` skill metadata contract that would expose plugin tool schemas only when matching skills activate those plugins.

## Best Possible Solution

Define an additive public skill activation contract that threads parsed metadata through skill snapshots into request-time plugin tool planning, with docs and tests for allow/deny precedence, optional tools, plugin disablement, and plugin-bundled skill behavior.

## Solution Assessment
