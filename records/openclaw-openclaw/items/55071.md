Add a `metadata.openclaw.activates.plugins` skill metadata contract that lets matched skills expose selected plugin tools and plugin-bundled skills per turn.

## Best Possible Solution

Design a backward-compatible skill activation contract that parses activation metadata, threads it through skill snapshots/matching, filters request-time plugin tool exposure safely, and documents/tests precedence rules.
