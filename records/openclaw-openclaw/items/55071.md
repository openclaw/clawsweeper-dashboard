## Summary

Keep open: current main still lacks the requested `metadata.openclaw.activates.plugins` skill contract and any path from matched skills into request-time plugin tool selection. The merged visibility work at https://github.com/openclaw/openclaw/pull/75983 is useful groundwork, but its author explicitly noted that it does not implement this proposal.

## What This Changes

Request adds a `SKILL.md` `metadata.openclaw.activates.plugins` contract so matched skills can make plugin-owned tools and bundled skills visible per turn.

## Best Possible Solution

Define a backward-compatible public skill activation contract, thread parsed activation metadata through skill snapshots into request-time plugin tool planning, and document/test precedence and plugin-skill behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Source inspection gives a high-confidence gap check because current metadata, parser, snapshots, docs, and plugin tool planning paths do not carry `activates.plugins`.

## Solution Assessment

Unclear until maintainers settle the public API and compatibility semantics. The maintainable direction is an additive metadata contract wired into skill snapshot and request-time tool planning, with deny/allow precedence and plugin-skill behavior tested.
