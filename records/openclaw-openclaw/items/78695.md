## What This Changes

The issue asks OpenClaw to keep 16 Telegram polling accounts through a local proxy from starving the gateway event loop by adding staggered/capped polling, stronger backoff, ELU protection, and proxy-aware degradation controls.

## Best Possible Solution

Define and land a Telegram-owned polling/backpressure policy that bounds account startup, restart bursts, and active setup/getUpdates pressure, documents any operator-facing cap, and proves it with a multi-account proxy stress scenario.

## Solution Assessment
