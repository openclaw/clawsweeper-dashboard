## Summary

Keep open: current main and the latest release still unconditionally load the plugin CLI path from `registerNodesCli`, and the issue now has an open closing PR that should be reviewed or landed before the issue closes.

## What This Changes

The issue reports a regression where built-in `openclaw nodes status/list` commands load plugin CLI/runtime despite the no-plugin command policy, making node status checks slow.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/96702, or adjust it to preserve any intended plugin-overlap contract, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Source inspection shows current main still calls plugin CLI registration from `registerNodesCli` for `nodes status/list`, even though the command-path policy inherited by `nodes` is `loadPlugins: "never"`; I did not run the timed container command.

## Solution Assessment

Yes, the best path is a narrow guard in nodes registration that keeps built-in nodes commands on the lightweight path while preserving plugin-owned node subcommands; the open candidate PR implements that shape pending maintainer compatibility review.
