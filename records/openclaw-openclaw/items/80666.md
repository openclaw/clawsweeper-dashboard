## Summary

Keep open. The PR targets a valid memory-core bug and uses the right ingestion-boundary hook, but the current patch still lets lowercase `pass` bypass the assistant-process-chatter rejection before the `Assistant: Need` rule can run.

## What This Changes

The PR adds a Dreaming session-corpus hygiene helper and focused tests to drop assistant process chatter while preserving durable decision and verification signals.

## Best Possible Solution

Keep the ingestion-boundary filter, tighten durable-signal matching so only explicit uppercase PASS or stronger verification context bypasses rejection, and add focused regression coverage for the lowercase `pass` case.

## Reproduction Assessment

Yes, at source level. Current main renders assistant messages as `Assistant:` lines and Dreaming ingestion appends normalized lines above the minimum length without a process-chatter gate, so a controlled transcript containing the reported examples will enter the corpus.

## Solution Assessment

No, not quite. The ingestion hook is the narrow maintainable place to solve this, but the current durable-signal bypass is too broad because lowercase `pass` defeats the later `Assistant: Need` rejection.
