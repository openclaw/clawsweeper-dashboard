## Summary

Keep open: the PR targets a real source-reproducible memory-core hygiene bug and now includes sufficient terminal proof, but the patch has an overbroad rejection pattern and current CI lint annotations on the new helper.

## What This Changes

The PR adds a memory-core Dreaming session-corpus rejection helper, invokes it during session ingestion, and adds focused tests for assistant process-chatter examples.

## Best Possible Solution

Keep the ingestion-boundary filter, but scope rejection to assistant process chatter or standalone action tokens, add regression coverage for preserved non-chatter corpus lines, and satisfy repo lint.

## Reproduction Assessment

Yes. Source inspection shows current main renders assistant transcript text as `Assistant:` snippets and only applies length, day/lookback, and dedupe checks before adding them to the session-corpus batch; I did not run the harness in this read-only review.

## Solution Assessment

No for the patch as submitted. The ingestion boundary is the right place and the new terminal proof is sufficient, but the unscoped `poll` rejection can drop non-assistant content and lint currently fails; tightening the helper plus braced `if` blocks is the narrow repair.
