## Summary

Keep open: current main still has a source-level false-positive path where identity-less text/media outbound results can become `sent`, and an open fixing PR already references this issue with closing syntax.

## What This Changes

The issue requests cron `--announce` fallback delivery to report `delivered` only when the configured channel actually produces a platform delivery receipt.

## Best Possible Solution

Land a narrow shared outbound/direct-cron fix that filters identity-less delivery results, records `not-delivered` for empty receipts, adds regression coverage, and includes redacted live cron announce proof.

## Reproduction Assessment

Yes, at source level: a cron announce path whose channel adapter returns an identity-less text/media result can produce a non-empty outbound result array, which durable send and cron dispatch then treat as delivered. The reporter also provided a concrete live WeChat/Feishu reproduction on 2026.5.7.

## Solution Assessment

Yes, the best direction is to fix the shared outbound result-identity contract rather than patch WeChat or Feishu specifically. The open linked PR is the current candidate, but it still needs its review finding and real behavior proof handled before merge.
