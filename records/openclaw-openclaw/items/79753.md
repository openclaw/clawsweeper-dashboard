## Summary

Keep open: current main still has a source-level path where identity-less outbound text/media results can become `delivered: true`, and the open fix PR at https://github.com/openclaw/openclaw/pull/79811 references this issue with closing metadata. Later WeChat comments also describe possible stale context-token behavior that may extend beyond the cron false-success symptom.

## What This Changes

The issue reports cron `--announce` jobs on WeChat and Feishu showing `delivered: true` and `fallbackUsed: true` even though no channel message arrives.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/79811; the core fix should filter identity-less results in shared outbound delivery with cron regression coverage, while any remaining WeChat context-token expiry should be split to the channel/plugin owner if still reproducible.

## Reproduction Assessment

Yes, at source level: current main can append identity-less outbound text/media results, durable send treats non-empty results as sent, and cron maps non-empty results to delivered. I did not establish a live WeChat or Feishu reproduction in this read-only review.

## Solution Assessment

Yes for the false-success bug: shared outbound identity filtering is the narrow maintainable fix. The broader stale WeChat context-token comments may need a separate plugin/channel-specific follow-up after the core delivered-state fix lands.
