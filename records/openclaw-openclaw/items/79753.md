## Summary

Keep open: current main still has a source-level path where an identity-less outbound text result can be counted as sent and then marked delivered by cron, and an open linked fix PR targets this issue with closing metadata.

## What This Changes

Cron announce fallback delivery reports successful delivery for WeChat and Feishu even when the channel message never arrives.

## Best Possible Solution

Review, rebase, and land or reject https://github.com/openclaw/openclaw/pull/79811; the core fix should make durable outbound count only adapter results with a real platform identity, with any remaining WeChat cold-session reachability tracked separately.

## Reproduction Assessment

Yes, at source level: current main can append an identity-less text adapter result, durable send treats the non-empty result array as sent, and cron maps that non-empty result to delivered. I did not run a live WeChat or Feishu cron job.

## Solution Assessment

Yes for the false-success bug: shared outbound identity filtering is the narrow maintainable fix because cron and durable send already rely on the shared results contract. The newer WeChat cold-session reports may need a separate plugin/channel follow-up if delivery still fails after accurate receipts.
