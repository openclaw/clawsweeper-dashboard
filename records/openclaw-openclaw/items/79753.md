## Summary

Keep open: current main still has a source-level false-success path for cron announce/direct delivery, and an open linked fix PR references this issue with GitHub closing metadata. The later WeChat comments add useful evidence that the symptom may be broader than cron, but they do not make this safe to close before the linked fix lands.

## What This Changes

Cron announce fallback delivery reports `delivered: true` for WeChat and Feishu even when no channel message arrives.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/79811; if cold WeChat context-token expiry remains afterward, track that separately in the WeChat plugin/channel surface.

## Reproduction Assessment

Yes, at source level: current main can keep identity-less outbound text/formatted/media results, durable send maps non-empty results to `sent`, and cron maps those results to `delivered`. I did not run a live WeChat or Feishu cron job in this read-only review.

## Solution Assessment

Yes for the false-success bug: filtering identity-less delivery results in shared outbound runtime with cron/outbound regression coverage is the narrow maintainable direction. The WeChat cold-session details should be handled as a separate plugin/channel follow-up if they persist after that fix.
