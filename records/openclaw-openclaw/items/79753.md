## Summary

Keep open: current main still has a source-level path where identity-less outbound text/media adapter results can be treated as delivered, and an open linked fix PR already targets this issue.

## What This Changes

Cron `--announce` should deliver to configured WeChat/Feishu channels or report failure instead of marking identity-less fallback sends as delivered.

## Best Possible Solution

Land shared outbound result identity filtering with cron regression coverage, then close this issue once the fix is merged and current-main or release provenance is available.

## Reproduction Assessment

Yes, at source level. Current main can append identity-less text/formatted results, durable send treats any non-empty result array as sent, and cron maps non-empty send results to `delivered: true`; I did not run a live WeChat or Feishu cron job.

## Solution Assessment

Yes. Filtering identity-less results in shared outbound delivery is the narrow maintainable fix, and the linked PR is the active implementation path rather than a reason to close this issue before merge.
