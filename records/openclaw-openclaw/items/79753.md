## Summary

Keep open: current main still has a source-level path where cron announce/direct delivery can count an identity-less outbound text/media result as sent, and an open linked PR references this issue with closing metadata. Recent WeChat comments add useful live context, but they do not make the core cron false-positive delivery bug safe to close yet.

## What This Changes

Cron announce fallback delivery reports `delivered: true` for WeChat and Feishu even when the channel message never arrives.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/79811; if delayed WeChat replies still fail afterward, track that as a separate plugin-specific cold-session delivery issue with redacted logs.

## Reproduction Assessment

Yes, at source level: current main can append an identity-less text delivery result, mark the durable send as `sent`, and then mark cron delivery as `delivered`. I did not run a live WeChat or Feishu cron delivery in this read-only review.
