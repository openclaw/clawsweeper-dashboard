## What This Changes

Cron announce delivery should report failure instead of `delivered: true` when fallback delivery does not produce a real channel message.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/79811; the durable fix belongs in shared outbound identity filtering with cron regression coverage.
