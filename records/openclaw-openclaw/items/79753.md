## Summary

## What This Changes

Cron announce fallback delivery should deliver final text to explicit WeChat/Feishu targets or report not-delivered instead of success when no channel adapter send occurs.

## Best Possible Solution

Keep the current channel-agnostic cron direct-delivery implementation on main, and ask for a new current-version plugin-specific report only if WeChat or Feishu still drops messages after this reaches a release.

## Solution Assessment
