## Summary

Close: current main no longer uses internal announce routing for cron fallback and only reports `delivered` after a non-empty successful durable outbound send. The reported v2026.5.7 behavior predates the current-main proof, and no shipped tag contains that proof yet.

## What This Changes

Cron `--announce` fallback delivery should invoke the configured WeChat/Feishu channel adapter or report a clear delivery failure instead of returning `delivered: true` for a no-op.

## Best Possible Solution

Keep the current durable direct cron delivery path and ship it in the next release; users on v2026.5.7 should upgrade once that release is available.

## Reproduction Assessment

No current-main failure remains. The v2026.5.7 report is credible, but current main routes cron announce fallback through durable outbound send and only marks delivery successful after non-empty successful results.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix: keep cron channel-agnostic, use the existing plugin outbound contract, and make success depend on actual durable outbound send results.
