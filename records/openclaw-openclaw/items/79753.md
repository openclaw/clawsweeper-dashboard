## Summary

Keep open: current main still has a source-level false-positive path for identity-less outbound text/formatted/media receipts, and https://github.com/openclaw/openclaw/pull/79811 is an open closing PR for this issue. ## What This Changes

Cron announce fallback delivery should not report WeChat or Feishu sends as delivered when no channel message actually arrived.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/79811; if WeChat cold-session delivery still fails afterward, handle that as the separate session-token issue.

## Reproduction Assessment

Yes, at source level: current main can keep an identity-less outbound text/formatted/media result in a non-empty array, durable send maps that to `sent`, and cron maps non-empty send results to `delivered: true`. I did not run the live WeChat or Feishu cron setup in this read-only review.

## Solution Assessment

Yes, the linked PR’s shared outbound identity filtering plus cron regression coverage is the narrow maintainable direction. This issue should stay open until that PR lands or is closed.
