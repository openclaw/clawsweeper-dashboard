## Summary

Keep open: current main still has a source-level path where cron announce/direct delivery can treat identity-less outbound text/media adapter results as sent, and an open linked fix PR references this issue with closing syntax.

## What This Changes

Cron `--announce` should either deliver to the configured WeChat or Feishu channel, or report failure instead of `delivered: true` when no platform message arrives.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/79811; the maintainable fix belongs in shared outbound delivery identity filtering with cron regression coverage.

## Reproduction Assessment

Yes, at source level: current main appends identity-less text/media adapter results, `sendDurableMessageBatch` treats any non-empty result array as `sent`, and cron marks non-empty results as delivered. I did not run a live WeChat or Feishu cron job in this read-only review.

## Solution Assessment

Yes, the linked PR’s shared outbound identity filtering is the narrow maintainable direction. This issue should stay open until that PR lands or is closed.
