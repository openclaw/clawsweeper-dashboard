## Summary

Keep open: current main still has a source-reproducible path where identity-less text/media outbound results can make cron direct delivery report success, and the linked fix PR is open but unmerged.

## What This Changes

Cron `--announce` delivery should either send the configured channel message or report a delivery failure instead of `delivered: true` when no usable platform receipt exists.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/79811 after it filters identity-less text/media outbound results, covers the regression, and provides redacted real cron/channel proof.

## Reproduction Assessment

Yes, at source level: current main can return a non-empty outbound result array from identity-less text/media adapter sends, and cron direct delivery converts that to `delivered: true`. The issue also gives concrete WeChat and Feishu live steps, but I did not run a live channel job in this read-only review.

## Solution Assessment

Yes, the narrow maintainable direction is to only count outbound results with a real platform delivery identity before durable send and cron delivery mark success. The linked PR is the right fix path, but it still needs its review finding and real behavior proof resolved before merge.
