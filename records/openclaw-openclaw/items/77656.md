## Summary

This should stay open: current main has the expected Telegram and node approval plumbing, but the reporter’s v2026.5.3 evidence describes a newer host=node failure after the older direct-gateway Telegram approval issue was closed as fixed.

## What This Changes

The issue asks OpenClaw to make Telegram approvals reliably resolve macOS node-host exec requests, or clearly report why that approval path cannot work.

## Best Possible Solution

Preserve the existing gateway approval contract while fixing the node-host approval route so Telegram buttons and /approve resolve the same pending host=node record, then add targeted regression coverage and approval-id tracing.
