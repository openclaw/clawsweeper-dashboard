## Summary

Keep open: current main still has the chat-channel node approval replay gap, and https://github.com/openclaw/openclaw/pull/79781 is an open closing PR for the targeted fix.

## What This Changes

The issue reports that Telegram and other chat-channel approvals for `host=node` exec commands can be accepted by the user but rejected during backend replay after connection changes.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/79781, then close this issue as implemented with commit and release provenance after merge.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the source-level core path: current main rejects changed-connection no-device node approval replay, and the WeCom comment shows that exact `APPROVAL_CLIENT_MISMATCH` after chat approval.

## Solution Assessment

Is this the best way to solve the issue? Yes: carrying existing turn-source facts into the approved replay while preserving node, command, session, agent, and target bindings is narrower than relaxing the connection check globally.
