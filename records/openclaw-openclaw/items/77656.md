## What This Changes

The issue reports Telegram and other chat-channel host=node exec approvals failing after approval because replayed node.invoke calls are rejected as expired or not valid for the client.

## Best Possible Solution

Keep the current main implementation and ship it in the next release so chat-channel node approval replay no longer depends on transient websocket connIds.

## Solution Assessment
