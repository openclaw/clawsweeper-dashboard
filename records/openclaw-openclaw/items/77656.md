## What This Changes

The issue reports that chat-channel approvals for macOS host=node exec commands appear accepted in Telegram but fail or replay as denied/unknown/invoke-failed while gateway-hosted approvals work.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes for the active direction. Carrying stable turn-source facts through approved node invokes and checking trusted backend plus node/session/agent/channel bindings is narrower than relaxing the connection check globally.
