## Summary

Keep open for maintainer review and merge consideration. Current main has source-reproducible unbounded plugin-owned compaction awaits, and this PR’s shared timeout/abort wrapper plus regression coverage addresses the pi-embedded, CLI, and Codex ownsCompaction paths; I found no discrete correctness or security finding, but merge should explicitly accept the changed timeout semantics for plugin-owned session compaction.

## What This Changes

The PR wraps plugin-owned `ContextEngine.compact()` calls in pi-embedded, CLI, and Codex harness paths with a shared safety-timeout helper, adds optional `abortSignal` support, exports the helper through the agent-harness runtime surface, and adds focused tests.

## Best Possible Solution

Land the shared timeout/abort wrapper after focused merge-ref validation, with an upgrade note if maintainers want to document the timeout knob for long-running plugin-owned compactions.

## Reproduction Assessment

Yes by source inspection. Current main directly awaits plugin-owned `ContextEngine.compact()` in the affected lanes without a safety timeout or abort signal; I did not run a live repro because this review is read-only.

## Solution Assessment

Yes. A shared helper that reuses the existing compaction timeout and adds an optional abort signal is the narrow maintainable fix, as long as maintainers accept the intentional timeout semantics for plugin-owned compactions.
