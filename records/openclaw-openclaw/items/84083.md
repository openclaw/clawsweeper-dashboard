## Label Justifications

- `P1`: The PR addresses an agent turn hang where a plugin-owned compaction can leave real users waiting indefinitely.
- `merge-risk: 🚨 session-state`: As written, timed-out plugin compaction can keep running in the background after the host resumes, risking stale or overlapping session/transcript mutations.

## Summary

Keep open: this PR addresses a real agent availability bug, but it is not merge-ready because the plugin-visible abort signal is not aborted on host timeout, the new public contract overstates bounded host behavior while CLI compaction still awaits directly, and contributor real behavior proof is still missing.

## What This Changes

The PR adds a context-engine compaction safety-timeout wrapper, wires it into queued, timeout-recovery, and overflow ownsCompaction paths, extends ContextEngine.compact params with abortSignal, and adds focused tests.

## Best Possible Solution

Land a corrected host-side wrapper that aborts the plugin-visible signal on timeout or caller abort, keeps the ContextEngine contract true across host callers, and includes focused tests plus redacted runtime proof.

## Reproduction Assessment

Yes from source inspection: current main directly awaits contextEngine.compact() in queued, timeout-recovery, and overflow paths without the safety wrapper. I did not run a live plugin repro in this read-only review.

## Solution Assessment

No as written. Wrapping the ownsCompaction lanes is the right direction, but the patch must make timeout cancellation visible to plugins and keep the public ContextEngine contract aligned with all host compact callers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Abort the plugin compact signal on timeout:** `src/agents/pi-embedded-runner/compaction-safety-timeout.ts:125-127`
  - body: The signal passed into `contextEngine.compact()` is only the caller's abort signal. When the helper's own safety timeout fires, `withTimeout` rejects and runs cancellation internally, but the signal the plugin received is never aborted, so a cooperative engine can keep running and writing in the background after the host has already treated compaction as timed out.
  - confidence: 0.94
- **[P2] Keep the compact contract true for CLI turns:** `src/context-engine/types.ts:302-306`
  - body: This new contract says the host always bounds `ContextEngine.compact()` and aborts the signal on timeout, but `runCliTurnCompactionLifecycle` still reaches `params.contextEngine.compact()` directly in `src/agents/command/cli-compaction.ts`. A plugin-owned engine can still hang CLI turn compaction, so either wrap that caller too or narrow the contract before exposing this guarantee to plugin authors.
  - confidence: 0.88
