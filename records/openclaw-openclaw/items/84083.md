## Summary

Keep open. The PR addresses a real P1 agent hang and now includes useful terminal proof, but the patch still leaves timeout cancellation invisible to cooperating plugins and overstates the public compact contract while CLI budget compaction remains unbounded.

## What This Changes

The PR adds a shared safety-timeout wrapper for plugin-owned `ContextEngine.compact()` calls, wires it into pi-embedded and Codex ownsCompaction lanes, extends compact params with `abortSignal`, and adds focused tests.

## Best Possible Solution

Keep the shared timeout boundary, but make timeout cancellation plugin-visible and align the CLI compaction path or public contract before merge.

## Reproduction Assessment

Yes. Source inspection of current main shows the ownsCompaction lanes directly await `ContextEngine.compact()` without the safety wrapper, and a never-settling compact promise would block those awaits; the PR body also supplies after-fix terminal proof for the Codex path.

## Solution Assessment

No. The shared helper is the right shape, but the implementation needs plugin-visible timeout cancellation and either CLI coverage or a narrower public contract before this is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Abort the plugin-visible compact signal on timeout:** `src/agents/pi-embedded-runner/compaction-safety-timeout.ts:125-127`
  - body: This passes the caller's signal into `compact()` and relies on `withTimeout` only to reject the host await. When the safety timeout fires, a cooperative engine never sees abort unless the caller separately aborted, so it can keep mutating session state or spending provider work after OpenClaw has already returned failure. Create a per-call `AbortController`, pass that signal to `compact()`, and abort it from the timeout/cancel path.
  - confidence: 0.91
- **[P2] Keep the public compact contract true for CLI compaction:** `src/context-engine/types.ts:302-306`
  - body: The new contract says the host always bounds `compact()` with the safety timeout, but `compactCliTranscript` still awaits `params.contextEngine.compact()` directly on CLI budget compaction. That leaves one host path unbounded and makes the SDK promise false; route that path through the helper or narrow this contract to the wrapped lanes.
  - confidence: 0.88
