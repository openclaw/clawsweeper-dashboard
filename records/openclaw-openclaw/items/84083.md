## Summary

Keep this PR open: it targets a real source-proven agent hang, but the current patch still has two correctness gaps and no real ownsCompaction runtime proof after the fix.

## What This Changes

Adds a shared safety-timeout wrapper for plugin-owned ContextEngine.compact calls, wires it into pi-embedded and Codex ownsCompaction lanes, extends compact params with abortSignal, and adds focused tests.

## Best Possible Solution

Keep the shared timeout boundary, but make timeout cancellation plugin-visible, align or narrow the public compact contract for CLI compaction, and require redacted real ownsCompaction runtime proof before merge.

## Reproduction Assessment

Yes from source inspection: current main directly awaits plugin ContextEngine.compact in the reported ownsCompaction lanes without the safety wrapper. I did not run a live plugin repro in this read-only review.

## Solution Assessment

No, not yet: the helper is the right boundary, but this implementation does not abort the plugin-visible signal on host timeout and leaves CLI compaction outside the new public contract. The safer path is to fix those contract gaps and then verify a real ownsCompaction plugin run.

- **[P2] Abort the plugin compact signal on timeout:** `src/agents/pi-embedded-runner/compaction-safety-timeout.ts:125-127`
  - body: The helper only forwards an existing caller signal into compact(), while withTimeout's timeout signal only resolves the host-side race.   - confidence: 0.91

Plugin authors can rely on a guarantee that is false for CLI-budget compaction, and that path can still hang.
  - confidence: 0.88
