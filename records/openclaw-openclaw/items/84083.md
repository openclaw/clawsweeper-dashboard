## Summary

Keep this PR open: source inspection confirms the current ownsCompaction lanes are unbounded and the PR targets the right agent boundary, but it still needs real behavior proof and one timeout-cancellation fix before merge.

## What This Changes

This PR wraps plugin-owned ContextEngine.compact() calls in queued, timeout-recovery, and overflow compaction lanes with a shared safety-timeout helper, adds an optional abortSignal parameter to the ContextEngine contract, and extends related tests.

## Best Possible Solution

Land a corrected wrapper that bounds all three ownsCompaction lanes, aborts cooperating plugin work on host timeout or run abort, preserves clean ok:false handling, and includes focused tests plus real behavior proof.

## Reproduction Assessment

Yes from source inspection, though I did not run a live plugin repro in this read-only review. Current main awaits contextEngine.compact() directly in queued, timeout-recovery, and overflow lanes, while only native activeSession.compact() uses the safety timeout wrapper.

## Solution Assessment

No as written. Wrapping all three call sites is the right boundary, but the helper must pass a timeout-aborted composite signal to compact() before the new abortSignal contract is accurate.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Abort the plugin compact signal on timeout:** `src/agents/pi-embedded-runner/compaction-safety-timeout.ts:125-127`
  - body: The wrapper passes the caller's abortSignal straight through to contextEngine.compact(). When withTimeout fires, it aborts only its internal timeoutSignal and this wrapper supplies no onCancel or composite signal, so a cooperative engine will keep compacting in the background after the host has timed out. That contradicts the new ContextEngine contract and leaves runaway plugin work active after recovery; please pass a per-call signal that aborts on either host timeout or caller abort, and assert that timeout path in the tests.
  - confidence: 0.88
