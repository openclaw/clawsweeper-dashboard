## Label Justifications

- `P1`: The PR addresses an agent turn hang where plugin-owned compaction can leave real users waiting indefinitely.
- `merge-risk: 🚨 session-state`: As written, timed-out plugin compaction can continue in the background after the host resumes, risking stale or overlapping session/transcript mutations.

## Summary

Keep open: this PR targets a real unbounded plugin-owned compaction hang and includes focused tests, but the patch still overpromises timeout cancellation/coverage and lacks real behavior proof. The companion lossless-claw PR is useful plugin-side context, but the OpenClaw host-side patch still needs fixes before merge.

## What This Changes

The branch adds a safety-timeout wrapper for plugin-owned context-engine compaction, wires it into queued, overflow, and timeout-recovery lanes, adds an optional compact abortSignal, and adds focused tests.

## Best Possible Solution

Land a corrected host-side wrapper that aborts the plugin-visible signal on timeout or caller abort, keeps the ContextEngine contract true across host callers or narrows the docs, and includes redacted runtime proof.

## Reproduction Assessment

Yes from source inspection. Current main directly awaits contextEngine.compact() in queued, timeout-recovery, and overflow-recovery ownsCompaction paths without the safety wrapper; I did not run a live lossless-claw repro in this read-only review.

## Solution Assessment

No as written. Wrapping the ownsCompaction lanes is the right direction, but the patch must make timeout cancellation visible to plugins and keep the public ContextEngine contract aligned with all host compact callers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Abort the plugin compact signal on timeout:** `src/agents/pi-embedded-runner/compaction-safety-timeout.ts:125-127`
  - body: The signal passed into `contextEngine.compact()` is only the caller's abort signal. When the helper's own safety timeout fires, `withTimeout` rejects and runs cancellation internally, but the signal the plugin received is never aborted, so a cooperative engine can keep running and writing in the background after the host has already treated compaction as timed out.
  - confidence: 0.94
- **[P2] Keep the compact contract true for CLI turns:** `src/context-engine/types.ts:302-306`
  - body: This new contract says the host always bounds `ContextEngine.compact()` and aborts the signal on timeout, but `runCliTurnCompactionLifecycle` still reaches `params.contextEngine.compact()` directly in `src/agents/command/cli-compaction.ts`. A plugin-owned engine can still hang CLI turn compaction, so either wrap that caller too or narrow the contract before exposing this guarantee to plugin authors.
  - confidence: 0.88
