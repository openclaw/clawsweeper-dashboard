## Summary

Keep this PR open: current main still lacks the named-DM primitives, and the PR branch has unresolved correctness and policy-classification defects plus an infrastructure-first product decision that should be made by maintainers before any merge or replacement work.

## What This Changes

The PR adds named-DM session key helpers and tests, an optional `activeNamedSession` session-store field, gateway store helpers, session documentation, and repeated changelog entries without routing integration.

## Best Possible Solution

If named-DM session switching is still desired, land it through maintainer-approved gateway/session infrastructure with strict key parsing, fail-closed persisted-state handling, explicit `dm-named` precedence in chat-type and send-policy paths, focused regression tests, accurate contributor docs, and one active changelog entry.

## Reproduction Assessment

Yes, for the PR review defects by source inspection. The PR head delegates named-DM parsing through an empty-token-filtering parser, passes persisted `activeNamedSession` into a throwing builder, and checks `dm-named` after `group` and `channel` tokens.

## Solution Assessment

No. The infrastructure direction may be useful, but this branch is not the best merge shape until the parser, stored-state lookup, chat-type precedence, changelog placement, docs, and maintainer product decision are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Reject collapsed empty segments in named-DM parsing:** `src/sessions/session-key-utils.ts:180-184`
  - body: `parseNamedDmSessionKey` relies on `parseAgentSessionKey`, which drops empty `:` segments before this code validates the named-DM shape. A malformed key like `agent:main:dm-named:123::work` can therefore be accepted as peer `123` and name `work`; parse the raw suffix or reject empty segments before returning a valid key.
  - confidence: 0.92
- **[P2] Fail closed for malformed stored active names:** `src/gateway/session-utils.ts:862-870`
  - body: `activeNamedSession` is persisted session-store data, but this helper passes it straight into `buildNamedDmSessionKey`. If a stored row contains `:`, active-session resolution throws instead of returning `null`; validate or ignore malformed stored names before building the key.
  - confidence: 0.9
- **[P2] Prioritize dm-named before token chat-type checks:** `src/sessions/session-key-utils.ts:50`
  - body: Adding `dm-named` to the direct-token branch after `group` and `channel` still misclassifies valid named DM keys whose user-chosen name is `group` or `channel`. Since send policy derives chat type through this path, those DMs can pick up group/channel policy rules.
  - confidence: 0.91
- **[P2] Keep the changelog entry in the active section only:** `CHANGELOG.md:109`
  - body: The branch repeats the same named-DM changelog bullet in many historical release sections. That rewrites old release history and can confuse release-note tooling; keep a single entry in the active changelog section for this change.
  - confidence: 0.96
- **[P3] Correct the helper nullability docs:** `docs/concepts/session.md:79`
  - body: The docs say all three named-DM helpers accept `null` or `undefined`, but the builder requires a params object and reads its fields immediately. Narrow the statement to parser/predicate helpers or change the implementation and types to match.
  - confidence: 0.78
