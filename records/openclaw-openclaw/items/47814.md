The feature direction also needs maintainer product/design review before a repair or replacement branch should proceed.

## What This Changes

The PR adds named-DM session key helpers and tests, an active named-session store field and gateway helpers, session docs, and changelog entries without routing integration.

## Best Possible Solution

Land a rebased, maintainer-approved named-DM infrastructure slice with strict parsing, fail-closed stored-state handling, explicit direct-message classification, focused tests, accurate docs, and one active changelog entry.

## Reproduction Assessment

Yes, for the PR review defects by source inspection. The parser delegates through an empty-token-filtering parser, active-session lookup builds from persisted data without validating the stored name, and chat-type classification checks group/channel tokens before dm-named.

## Solution Assessment

No. The infrastructure direction may be useful, but this branch is not the best merge shape until parser strictness, persisted-state handling, chat-type precedence, docs/changelog cleanup, real behavior proof, and maintainer product direction are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject collapsed empty segments in named-DM parsing:** `src/sessions/session-key-utils.ts:184-185`
  - body: parseNamedDmSessionKey relies on parseAgentSessionKey, which drops empty ':' segments before this code validates the dm-named shape. A malformed key like agent:main:dm-named:123::work can be accepted as peer 123 and name work; parse the raw suffix or explicitly reject empty segments before accepting a named-DM key.
  - confidence: 0.92
- **[P2] Fail closed for malformed stored active names:** `src/gateway/session-utils.ts:866-870`
  - body: activeNamedSession is persisted session-store data, but getActiveNamedSessionKey passes it straight into buildNamedDmSessionKey. If a stored row contains ':', callers get an exception during active-session resolution instead of the advertised string | null behavior; validate the stored name and return null or repair it before building the key.
  - confidence: 0.9
- **[P2] Prioritize dm-named before token chat-type checks:** `src/sessions/session-key-utils.ts:50`
  - body: Adding dm-named to the direct-token branch after group and channel checks still misclassifies valid named DM keys whose user-chosen name is group or channel. Since send policy derives chat type through this helper on the branch, those direct-message sessions can pick up group/channel policy rules unless dm-named is recognized first.
  - confidence: 0.91
- **[P2] Keep the named-DM changelog entry in the active section:** `CHANGELOG.md:109`
  - body: The branch repeats the same named-DM changelog bullet in many historical release sections. That rewrites old release history and can confuse release-note tooling; keep a single credited entry in the active changelog section when the change is ready to land.
  - confidence: 0.96
- **[P3] Correct the helper nullability docs:** `docs/concepts/session.md:79`
  - body: The docs say all three helpers accept null or undefined, but buildNamedDmSessionKey requires a params object and immediately reads its fields. Narrow this statement to the parser/predicate helpers or change the implementation and types to match.
  - confidence: 0.78
