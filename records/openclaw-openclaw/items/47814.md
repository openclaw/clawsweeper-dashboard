## Summary

Keep open. Current main still lacks the named-DM session API/docs, and the PR branch is not a clean merge candidate because the added parser can accept malformed keys, active-session lookup can throw on malformed persisted state, chat-type derivation can apply group/channel policy to named DMs, and the changelog edits rewrite old release sections.

## What This Changes

The PR adds named-DM session key helpers and tests, an optional `activeNamedSession` field on `SessionEntry`, gateway helpers for active named sessions, contributor docs, and changelog entries without wiring routing behavior.

## Best Possible Solution

Keep the work open for maintainer review or a narrow rebased replacement. If named-DM switching is still wanted, land it with strict raw key parsing, fail-closed persisted-state handling, explicit `dm-named` direct classification before arbitrary token checks in current chat-type and send-policy paths, focused malformed-key and policy-collision tests, contributor docs that match the helper contracts, and a single active changelog entry.

## Reproduction Assessment

Yes. The problematic paths are deterministic from code inspection: `agent:main:dm-named:123::work` is accepted after empty-segment filtering, `activeNamedSession: "foo:bar"` can throw during lookup, and a named DM called `channel` or `group` can be classified through the wrong policy path.

## Solution Assessment

No. The infrastructure direction may be useful, but this patch is not the best mergeable solution until the parser validates raw segments, stored active names fail closed, named-DM classification has precedence, the changelog is cleaned up, and the branch is rebased onto current main.
