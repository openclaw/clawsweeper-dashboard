## Summary

Keep open: current main still lacks the named-DM primitives, and this PR has concrete correctness/security review defects plus missing real behavior proof before it can be merged.

## What This Changes

The branch adds named-DM key helpers/tests, an `activeNamedSession` session-store field, gateway helpers, session docs, and changelog entries without routing or `/resume` integration.

## Best Possible Solution

If named-DM switching is still desired, land a narrow rebased implementation with strict raw-key parsing, fail-closed persisted-state handling, explicit `dm-named` direct classification precedence, focused regression tests, and docs that clearly separate primitives from wired routing.

## Reproduction Assessment

Yes, for the PR review defects by source inspection: the parser collapses empty segments before validation, persisted active names can throw during lookup, and chat-type precedence can misclassify named DMs. I did not run tests because this was a read-only review.

## Solution Assessment

No. The infrastructure direction may be useful, but this branch is not the best merge shape until maintainers approve the product boundary and the PR fixes parser strictness, persisted-state behavior, policy classification, docs/changelog drift, and proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject collapsed empty segments in named-DM parsing:** `src/sessions/session-key-utils.ts:180-185`
  - body: `parseNamedDmSessionKey` delegates to `parseAgentSessionKey`, which drops empty `:` segments before this code validates the named-DM shape. A malformed key like `agent:main:dm-named:123::work` can be accepted as `{ peerId: "123", name: "work" }`; parse the raw scoped suffix or reject empty segments before returning a valid named-DM key.
  - confidence: 0.92
- **[P2] Fail closed for malformed stored active names:** `src/gateway/session-utils.ts:866-870`
  - body: `activeNamedSession` is persisted session-store data, but `getActiveNamedSessionKey` passes it straight into `buildNamedDmSessionKey`. If a store row contains `:`, callers get an exception during active-session resolution instead of the advertised `string | null` behavior; validate the stored name and return `null` or repair it before building the key.
  - confidence: 0.9
- **[P2] Prioritize dm-named before token chat-type checks:** `src/sessions/session-key-utils.ts:50`
  - body: Adding `dm-named` to the direct-token branch after `group` and `channel` still misclassifies valid named DM keys whose user-chosen name is `group` or `channel`. Since send policy can fall back to key-derived chat type, those sessions can pick up group/channel policy rules unless `dm-named` is recognized before scanning arbitrary name tokens.
  - confidence: 0.91
- **[P2] Keep the changelog entry in the active section only:** `CHANGELOG.md:109`
  - body: The branch repeats the same named-DM changelog bullet across many historical release sections. That rewrites old release history and can confuse release-note tooling; keep any changelog text to the active section only, or leave it for maintainer-owned landing work.
  - confidence: 0.96
- **[P3] Correct the helper nullability docs:** `docs/concepts/session.md:79`
  - body: The docs say all three helpers accept `null` or `undefined`, but `buildNamedDmSessionKey` requires a params object and immediately reads its fields. Narrow that statement to the parser/predicate helpers or change the implementation and types to match the documented contract.
  - confidence: 0.78
