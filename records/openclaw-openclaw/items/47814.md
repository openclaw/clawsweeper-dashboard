## Summary

Keep open: current main still lacks the named-DM infrastructure, and this PR has source-reproducible defects in malformed-key parsing, persisted active-name handling, and send-policy chat-type classification. It also needs maintainer product/design review and real behavior proof before merge.

## What This Changes

This PR adds named-DM key helpers and tests, an `activeNamedSession` `SessionEntry` field, gateway active-session helpers, session docs, and changelog entries without wiring routing behavior.

## Best Possible Solution

If named-DM switching remains desired, land a narrow rebased design with strict raw key parsing, fail-closed store handling, explicit `dm-named` direct classification in chat-type/send-policy paths, and docs that distinguish primitives from wired routing.

## Reproduction Assessment

Yes. The PR-specific issues are source-reproducible from the PR head: malformed keys are parsed after empty-token filtering, stored active names are passed into a throwing builder, and `dm-named` is checked after user-controlled `group`/`channel` tokens.

## Solution Assessment

No. The feature direction is plausible core gateway/session work, but this branch is not the best merge shape until the key, persisted-state, and policy-classification contracts are fixed and reviewed against current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject empty segments before accepting named-DM keys:** `src/sessions/session-key-utils.ts:184-185`
  - body: `parseNamedDmSessionKey` delegates to `parseAgentSessionKey`, which drops empty `:` segments before validation. A malformed key such as `agent:main:dm-named:123::work` is accepted as a valid named-DM key; validate the raw suffix or explicitly reject empty segments first.
  - confidence: 0.92
- **[P2] Fail closed for invalid stored active names:** `src/gateway/session-utils.ts:866-870`
  - body: `activeNamedSession` is persisted session-store data, but `getActiveNamedSessionKey` passes it straight into `buildNamedDmSessionKey`. If a stored value contains `:`, active-session resolution throws instead of returning `null`; validate the stored name and fail closed.
  - confidence: 0.9
- **[P2] Classify dm-named before user-controlled tokens:** `src/sessions/session-key-utils.ts:50`
  - body: The new `dm-named` direct classification runs after `group` and `channel` token checks. A valid named DM like `agent:main:dm-named:123:channel` is classified as `channel`, so send-policy fallback can apply channel rules to a direct message.
  - confidence: 0.91
- **[P3] Fix the null-input helper docs:** `docs/concepts/session.md:79`
  - body: The docs say all three helpers accept `null` or `undefined`, but `buildNamedDmSessionKey` takes a required params object and string fields. Narrow this sentence to the parser/predicate helpers or make the builder actually fail safely for nullish input.
  - confidence: 0.82
