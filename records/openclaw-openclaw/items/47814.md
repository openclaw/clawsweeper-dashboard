## Summary

Keep open: current main still lacks the named-DM infrastructure, and the related named-DM PRs are closed as superseded by this PR rather than merged. The branch remains useful but has source-proven blockers, stale branch shape, missing real behavior proof, and a product/design question about landing infrastructure before routing.

## What This Changes

This PR adds named-DM session key helpers, an optional `activeNamedSession` store field, gateway helper functions, tests, session docs, and changelog entries without wiring routing behavior.

## Best Possible Solution

Land a rebased, maintainer-approved named-DM infrastructure slice with strict key parsing, fail-closed stored-state handling, explicit direct-message classification, focused malformed-key and policy-collision tests, accurate contributor docs, and one active changelog entry.

## Reproduction Assessment

Yes, for the PR review defects by source inspection. The PR head delegates named-DM parsing through an empty-token-filtering parser, builds keys from persisted `activeNamedSession`, and classifies `dm-named` only after arbitrary `group`/`channel` token checks.

## Solution Assessment

No. The infrastructure direction may be useful, but this branch is not the best merge shape until the parser, persisted-state contract, chat-type precedence, docs/changelog cleanup, rebase, and maintainer product direction are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject empty segments before parsing named-DM keys:** `src/sessions/session-key-utils.ts:180-185`
  - body: `parseNamedDmSessionKey` validates `parsed.rest` after `parseAgentSessionKey` has already dropped empty `:` segments. A malformed key like `agent:main:dm-named:123::work` can therefore be accepted as peer `123` and name `work`; parse the raw suffix or explicitly reject empty segments before returning a valid named-DM key.
  - confidence: 0.92
- **[P2] Fail closed for malformed stored active names:** `src/gateway/session-utils.ts:862-870`
  - body: `activeNamedSession` is persisted session-store data, but `getActiveNamedSessionKey` passes it directly into `buildNamedDmSessionKey`. If a stored row contains `:`, active-session resolution throws instead of returning `null`; validate or ignore malformed stored names before building the key.
  - confidence: 0.9
- **[P2] Classify dm-named before arbitrary name tokens:** `src/sessions/session-key-utils.ts:44-50`
  - body: `deriveSessionChatType` checks `group` and `channel` tokens before the new `dm-named` token. A valid named DM such as `agent:main:dm-named:123:channel` can be classified as `channel`, which can make send-policy fallback apply the wrong policy family.
  - confidence: 0.91
- **[P2] Keep named-DM changelog out of old releases:** `CHANGELOG.md:109`
  - body: The same named-DM changelog bullet is repeated in many historical release sections. That rewrites old release history and can confuse release-note tooling; keep this entry only in the active changelog section when the change is ready to land.
  - confidence: 0.96
- **[P3] Correct the helper nullability docs:** `docs/concepts/session.md:79`
  - body: The docs say all three helpers accept `null` or `undefined`, but `buildNamedDmSessionKey` requires a params object and reads its fields immediately. Narrow the statement to parser/predicate helpers or change the implementation and types to match.
  - confidence: 0.78
