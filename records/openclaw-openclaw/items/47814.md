## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reject empty named-DM segments before parsing:** `src/sessions/session-key-utils.ts:184-185`
  - body: `parseNamedDmSessionKey` relies on `parseAgentSessionKey`, which drops empty `:` segments before validation. A malformed key such as `agent:main:dm-named:123::work` can be accepted as peer `123` and name `work`; validate the raw segment count or reject empty segments first.
  - confidence: 0.92
- **[P2] Fail closed for malformed stored active names:** `src/gateway/session-utils.ts:866-870`
  - body: `activeNamedSession` is persisted store data, but `getActiveNamedSessionKey` passes it directly into `buildNamedDmSessionKey`. If an existing row contains `:`, active-session resolution throws instead of returning `null`; validate or ignore malformed stored names before building the key.
  - confidence: 0.9
- **[P2] Prioritize dm-named before token chat-type checks:** `src/sessions/session-key-utils.ts:50`
  - body: Adding `dm-named` to the direct branch after `group` and `channel` checks still misclassifies valid named DMs whose user-chosen name is `group` or `channel`, which can send those sessions through the wrong send-policy class.
  - confidence: 0.91
- **[P2] Keep changelog history unchanged:** `CHANGELOG.md:109`
  - body: The branch repeats the same named-DM changelog bullet in historical release sections. That rewrites old release history and can confuse release-note tooling; keep one entry in the active section only.
  - confidence: 0.88
- **[P3] Correct the helper nullability docs:** `docs/concepts/session.md:79`
  - body: The docs describe all named-DM helpers as accepting `null` or `undefined`, but the builder requires a params object and reads its fields immediately. Narrow that statement to parser/predicate helpers or change the implementation and types to match.
  - confidence: 0.78
