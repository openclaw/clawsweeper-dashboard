## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reject collapsed empty segments in named-DM parsing:** `src/sessions/session-key-utils.ts:180-184`
  - body: `parseNamedDmSessionKey` delegates to `parseAgentSessionKey`, which drops empty `:` segments before this code validates the `dm-named` shape. A malformed key like `agent:main:dm-named:123::work` can therefore parse as a valid named DM; validate the raw suffix or reject empty segments before returning a result.
  - confidence: 0.92
- **[P2] Fail closed for malformed stored active names:** `src/gateway/session-utils.ts:866-870`
  - body: `activeNamedSession` is persisted session-store data, but `getActiveNamedSessionKey` passes it straight into `buildNamedDmSessionKey`. If an existing or manually edited row contains `:`, active-session resolution throws instead of returning `null`; validate the stored name and fail closed.
  - confidence: 0.9
- **[P2] Prioritize dm-named before token chat-type checks:** `src/sessions/session-key-utils.ts:50`
  - body: The new `dm-named` token is checked after `group` and `channel`, so a valid named DM whose user-chosen name is `group` or `channel` is classified as that chat type instead of direct. Since send policy falls back to key-derived chat type, this can apply the wrong policy rules.
  - confidence: 0.91
- **[P2] Keep the changelog entry in the active section only:** `CHANGELOG.md:109`
  - body: The same named-DM changelog entry appears in many historical release sections on the branch. That rewrites release history and can confuse release-note tooling; keep one entry in the active changelog section for this change.
  - confidence: 0.96
- **[P3] Correct the helper nullability docs:** `docs/concepts/session.md:79`
  - body: The docs say all three helpers accept `null` or `undefined`, but `buildNamedDmSessionKey` requires a params object and immediately reads its fields. Narrow that statement to the parser/predicate helpers or change the implementation and types to match.
  - confidence: 0.78
