Keep this open for maintainer review or a narrow rebased replacement; if named-DM switching is wanted, land strict key parsing, fail-closed stored-name handling, explicit `dm-named` direct precedence in chat-type and send-policy paths, focused regression tests, and one active changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reject collapsed empty segments in named-DM parsing:** `src/sessions/session-key-utils.ts:184-185`
  - body: `parseNamedDmSessionKey` relies on `parseAgentSessionKey`, which drops empty `:` segments before this code validates the `dm-named` shape. A malformed key like `agent:main:dm-named:123::work` is accepted as `{ peerId: "123", name: "work" }`; parse the raw suffix or reject empty segments before returning a valid named-DM key.
  - confidence: 0.92
- **[P2] Fail closed for malformed stored active names:** `src/gateway/session-utils.ts:866-870`
  - body: `activeNamedSession` is persisted session-store data, but `getActiveNamedSessionKey` passes it straight into `buildNamedDmSessionKey`. If an existing or manually edited row contains `:`, callers get an exception during active-session resolution instead of the advertised `string | null` behavior.
  - confidence: 0.9
- **[P2] Prioritize dm-named before token chat-type checks:** `src/sessions/session-key-utils.ts:50`
  - body: Adding `dm-named` after the `group` and `channel` checks still misclassifies valid named DM keys whose user-chosen name is `group` or `channel`. Recognize the `dm-named` key shape before scanning arbitrary name tokens so send policy cannot apply group/channel rules to DMs.
  - confidence: 0.91
- **[P2] Keep the changelog entry in the active section only:** `CHANGELOG.md:109`
  - body: The branch repeats the same named-DM changelog bullet in historical release sections. That rewrites old release history and can confuse release-note tooling; keep a single entry in the active changelog section for this change.
  - confidence: 0.94
