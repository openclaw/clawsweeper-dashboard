## Summary

Keep open. Current main still lacks the named-DM infrastructure, and the PR branch is not mergeable as-is because the proposed parser, active-name lookup, chat-type precedence, changelog, and docs have concrete defects.

## What This Changes

This PR adds named-DM session key helpers and tests, an optional `activeNamedSession` field, gateway store helpers, session docs, and changelog entries without wiring routing behavior.

## Best Possible Solution

Keep the work open for maintainer review or a narrow rebased replacement; if named-DM switching is still wanted, land strict key parsing, fail-closed stored-state handling, explicit `dm-named` direct classification, focused malformed-key and policy-collision tests, contributor docs that match the API, and one active changelog entry.

## Reproduction Assessment

Yes. The problematic paths are deterministic from the PR head: `agent:main:dm-named:123::work` is accepted after empty-segment filtering, `activeNamedSession: "foo:bar"` can throw during lookup, and a named DM called `channel` or `group` can be classified through the wrong policy path.

## Solution Assessment

No. The infrastructure direction may be useful, but this branch should not merge until it is rebased onto current main and the parser, persisted-state handling, chat-type precedence, changelog, and docs are tightened.
