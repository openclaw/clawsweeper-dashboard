## Summary

Keep open. Current main still lacks a configurable model API request-start delay, and the linked implementation candidate #49221 remains open rather than merged or shipped.

## What This Changes

Add a user-configurable delay between consecutive model API request starts to help providers with strict rate limits, especially GLM-style backends.

## Best Possible Solution

Keep the issue open until #49221 or an equivalent maintainer-approved PR lands validated global and per-provider model request pacing across normal runs and compaction.

## Reproduction Assessment

Not applicable as a strict runtime bug reproduction, but yes for source-level verification: current main lacks the config fields, schema acceptance, docs, and request-start pacing in the inspected model-call paths.

## Solution Assessment

Unclear until #49221 is reviewed. The narrow maintainable direction is validated global/per-provider delay config applied by provider id to normal embedded runs and compaction, with zero-disablement, override precedence, and abortable waits covered by tests.
