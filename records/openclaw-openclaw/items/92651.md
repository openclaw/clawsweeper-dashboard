## Summary

Keep open for maintainer handling: current and shipped code trust a self-declared requester session for the write-scoped HTTP kill path, and this PR cleanly narrows that path to admin scope with focused regression coverage. The protected maintainer label, intentional compatibility break, and test-only proof keep it out of cleanup/automerge treatment.

## What This Changes

The PR removes the requester-owned write-scoped HTTP session kill branch and requires `sessions.delete`/admin scope before using `killSubagentRunAdmin`.

## Best Possible Solution

Land the admin-only HTTP kill path after maintainer acceptance of the compatibility break and either live proof or an explicit proof override.

## Reproduction Assessment

Yes. Current main and `v2026.6.6` both show the trusted-proxy requester header selecting the write-scoped `sessions.abort` path, and the removed test case encoded that success path.

## Solution Assessment

Yes, with maintainer acceptance. Because the HTTP auth result has no authenticated binding between caller and requester session key, requiring the existing admin `sessions.delete` scope is the narrowest safe fix; a non-admin replacement would need a new proof mechanism.
