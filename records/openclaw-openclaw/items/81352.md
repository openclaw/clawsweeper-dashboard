The PR requires both task and caller owner keys to normalize to non-empty values before owner-scoped session task access succeeds, with regression tests for blank caller keys and restored blank-owner legacy tasks.

## Best Possible Solution

Land a focused owner-access hardening after real behavior proof is supplied, and decide whether the same non-empty guard should be mirrored in task-flow owner lookup in this PR or a follow-up.
