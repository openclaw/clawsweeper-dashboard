## Summary

Keep open: this is still paired with an open fixing PR, and current main still has the doctor repair path that rewrites legacy `openai-codex/*` refs without adding the model-scoped Codex runtime protection proposed in https://github.com/openclaw/openclaw/pull/80138.

## What This Changes

Prevent `openclaw update` / `openclaw doctor --fix` from turning intended Codex OAuth agent routes into direct OpenAI API-key billed model calls.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80138, or an equivalent doctor repair change, then close this issue as implemented after merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a config with `openai-codex/gpt-5.5` plus explicit OpenAI PI runtime still enters doctor route repair, rewrites the model to `openai/gpt-5.5`, and current tests assert no model-scoped `agentRuntime.id` is added.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable fix: keep the canonical `openai/*` migration but carry Codex auth intent with model-scoped runtime policy so it cannot fall through to direct API-key billing.
