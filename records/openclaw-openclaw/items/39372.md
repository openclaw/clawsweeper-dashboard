## Summary

Keep open: the central decoupling request is not implemented on current main, but this PR is not safe to merge as-is because it still lacks real behavior proof and leaves ACP inline delivery reachable for headless session spawns.

## What This Changes

The branch removes native subagent and ACP `mode="session"` thread-required guards, adds headless-session notes and cleanup, and updates spawn-hook coverage for unthreaded persistent sessions.

## Best Possible Solution

Land a maintainer-approved replacement that decouples persistence from thread binding only after ACP delivery uses an explicit bound-thread or parent-relay contract, with docs/tests and real non-threaded plus thread-bound proof.

## Reproduction Assessment

Yes. Source inspection on current main shows native and ACP `mode="session"` guards rejecting requests without `thread=true`, and current spawn-hook coverage asserts the native rejection path; I did not run a live channel repro in this read-only review.

## Solution Assessment

No, not as proposed. Removing the guards is plausible only after ACP inline delivery is tied to an actual binding or explicit parent relay so headless persistent sessions do not leak output into requester chats.
