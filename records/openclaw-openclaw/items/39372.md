## Summary

Keep open: current main still rejects the requested unthreaded persistent-session path, so the PR targets a real requested capability, but the branch is not merge-ready because the ACP guard removal exposes inline delivery without a bound thread and the contributor has not supplied real behavior proof.

## What This Changes

The PR removes native subagent and ACP `mode="session"`/`thread=true` guards, adds headless-session accepted notes and cleanup, and updates spawn-hook coverage for unthreaded persistent sessions.

## Best Possible Solution

Review or replace this branch under https://github.com/openclaw/openclaw/issues/53548 with a rebased implementation that decouples persistence from thread binding, gates ACP inline delivery on an actual binding, preserves `thread:true` behavior, and updates tests, docs, and changelog.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: native and ACP `sessions_spawn` reject `mode: "session"` without `thread: true`, while the PR diff makes that path reachable without changing ACP inline delivery planning.

## Solution Assessment

No, not as currently proposed. Decoupling may be the right direction if maintainers accept the product contract, but this PR must first prevent headless ACP inline delivery and provide real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Require a bound thread before ACP inline delivery:** `src/agents/acp-spawn.ts:291-294`
  - body: Removing the ACP `mode="session" && !thread` guard makes headless persistent ACP spawns reachable, but the delivery planner still enables inline delivery for any session-mode spawn with a requester target. That can send child ACP output into the requester conversation even though no thread or conversation binding was prepared, so gate inline delivery on an actual binding or keep headless sessions internal.
  - confidence: 0.88
