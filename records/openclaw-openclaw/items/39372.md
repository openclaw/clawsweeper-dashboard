## Summary

Keep open: current main has not implemented unthreaded session mode, but the branch is not merge-ready because the ACP path would still inline-deliver session-mode child runs without a proven thread binding, and the remaining contract change is tracked in #53548.

## What This Changes

The PR removes native subagent and ACP `mode="session"` thread guards, adds headless-session accepted notes and cleanup, and updates spawn-hook coverage for unthreaded persistent sessions.

## Best Possible Solution

If maintainers accept the decoupled contract, land a rebased patch that keeps headless sessions internal unless a real binding exists, with aligned docs, changelog, and native plus ACP regression coverage.

## Reproduction Assessment

Yes. Source inspection of current main shows both native and ACP `sessions_spawn` reject `mode="session"` without `thread:true`, and the native spawn-hook test asserts that rejection path.

## Solution Assessment

No, not as currently proposed. The decoupling direction may be maintainable, but ACP inline delivery must first require a real binding and the documented product contract needs to be updated or reaffirmed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Require a bound thread before ACP inline delivery:** `src/agents/acp-spawn.ts:291-294`
  - body: Removing the ACP `mode="session" && !thread` guard makes headless persistent ACP spawns reachable, but the delivery planner still enables `deliver: true` for any session-mode spawn with a requester target. That can send worker output into the requester conversation even though no thread or conversation binding was prepared; gate inline delivery on an actual binding or keep headless sessions internal.
  - confidence: 0.87
