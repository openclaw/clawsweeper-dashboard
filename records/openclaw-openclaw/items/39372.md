## What This Changes

The PR removes native subagent and ACP session-mode thread requirements, adds headless-session accepted notes and cleanup, and changes spawn-hook coverage to expect unthreaded session mode to proceed.

## Best Possible Solution

Land a maintainer-approved update or replacement that decouples session persistence from thread binding, makes ACP delivery explicit for headless sessions, updates docs/tests, and proves both non-threaded and thread-bound behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows both native and ACP guards rejecting unthreaded `mode="session"`, and existing tests assert the native rejection path; no live channel repro was run in this read-only review.

## Solution Assessment

No, not as proposed. Removing the guards is plausible, but the PR must also fix ACP inline-delivery semantics so headless persistent sessions do not write to requester chats as if a thread were bound.
