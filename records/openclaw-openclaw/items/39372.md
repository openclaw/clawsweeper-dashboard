## Summary

This PR is still potentially useful but should not merge as-is: current main still enforces the documented thread-bound session contract, and the PR leaves ACP session-mode delivery able to inline output without a real thread binding.

## What This Changes

The PR removes native subagent and ACP `mode="session"`/`thread=true` guards, adds headless-session accepted notes and cleanup, and updates spawn-hook coverage for unthreaded persistent sessions.

## Best Possible Solution

If maintainers accept #53548's decoupled contract, rebase the PR and land it with ACP inline delivery gated on an actual binding plus aligned native/ACP tests, docs, and changelog.

## Reproduction Assessment

Yes. Source inspection on current main shows both native and ACP `sessions_spawn` reject `mode="session"` without `thread:true`, and the PR diff makes ACP headless session mode reachable while the delivery planner remains keyed only on `spawnMode === "session"`.

## Solution Assessment

No, not as currently proposed. The decoupling direction may be maintainable, but the PR must first prevent headless ACP inline delivery and update the documented session/thread contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Require a bound thread before ACP inline delivery:** `src/agents/acp-spawn.ts:291-294`
  - body: Removing the ACP `mode="session" && !thread` guard makes headless session-mode spawns reachable, but `spawnAcpDirect` still enables inline delivery later for any session-mode spawn with a requester target. That can send child ACP output into the requester conversation even though no thread or conversation binding was prepared, so gate inline delivery on an actual binding or keep headless sessions internal.
  - confidence: 0.88
