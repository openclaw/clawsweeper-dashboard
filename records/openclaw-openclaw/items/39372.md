## Summary

Keep open: current main still rejects unthreaded session mode, and this PR remains the active implementation candidate, but it is not merge-ready because ACP inline delivery is still keyed to session mode and real behavior proof is missing.

## What This Changes

The branch removes native subagent and ACP `mode="session"` plus `thread=true` guards, adds headless-session notes and cleanup, and updates spawn-hook coverage for unthreaded persistent sessions.

## Best Possible Solution

Land a maintainer-approved version that decouples persistence from thread binding, gates ACP inline delivery on an explicit binding or relay contract, updates docs/tests, and proves both non-threaded and thread-bound behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows both native and ACP guards returning errors for `mode="session"` without `thread=true`, and existing spawn-hook coverage asserts the native rejection path.

## Solution Assessment

No, not as proposed. Removing the guards is plausible only after the ACP delivery contract is tightened so headless persistent sessions cannot use thread-bound inline delivery semantics by accident.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require bound ACP threads before inline delivery:** `src/agents/acp-spawn.ts:291-294`
  - body: Removing the ACP `mode="session" && !thread` guard makes headless session spawns reachable, but the delivery planner still sets `useInlineDelivery` from `spawnMode === "session"`. In a chat-backed requester this passes `deliver: true` without a bound thread, so child output can be sent to the requester conversation instead of staying headless.
  - confidence: 0.9
