## Summary

Keep open: the branch targets a real limitation that current main and the latest release still have, but it is not mergeable as-is because ACP headless session delivery can still inline worker output into the requester chat and the PR has no after-fix real behavior proof.

## What This Changes

The PR removes native subagent and ACP session-mode thread-required guards, adds headless-session accepted notes and cleanup, and changes spawn-hook test expectations.

## Best Possible Solution

Land a maintainer-approved update or replacement that decouples persistence from thread binding, gates ACP delivery on an explicit binding or parent-relay contract, updates docs/tool text, and proves live headless plus thread-bound behavior.

## Reproduction Assessment

Yes, by source inspection against current main: native and ACP guards reject unthreaded session-mode spawns, and the existing spawn-hook test asserts the native rejection path. I did not run a live channel reproduction in this read-only review.

## Solution Assessment

No, not as proposed. Decoupling persistence from thread binding is plausible, but the patch also needs explicit ACP delivery semantics, docs updates, and real behavior proof before it is the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require bound ACP delivery before inline delivery:** `src/agents/acp-spawn.ts:292-295`
  - body: Removing the ACP `mode="session" && !thread` guard makes headless session spawns reach the existing delivery planner, where inline delivery is still enabled for any session-mode spawn with a requester target and no parent stream. A chat-backed ACP worker can therefore send output into the requester conversation even though no thread was bound; gate inline delivery on an actual binding or explicit parent relay before relaxing this guard.
  - confidence: 0.9
