## Summary

Keep open: current main still enforces the documented thread-bound session contract, and this PR is a useful implementation candidate, but it is not mergeable because the ACP path can inline-deliver headless session output into the requester conversation. The remaining product decision is tracked more broadly by #53548.

## What This Changes

The PR removes native subagent and ACP `mode:"session"` plus `thread:true` guards, adds headless-session accepted notes and failed-start cleanup, and updates spawn-hook coverage for unthreaded persistent sessions.

## Best Possible Solution

If maintainers accept headless persistent sessions, land a rebased implementation that decouples persistence from thread binding in native and ACP paths, makes ACP delivery bound-only or explicitly internal for headless sessions, and updates tests, docs, and changelog while keeping #53548 as the canonical product thread.

## Reproduction Assessment

Yes. Source inspection is enough: current main rejects unthreaded session mode in both native and ACP paths, and the ACP delivery planner would inline-deliver any session-mode ACP spawn if this PR removes the guard without adding a binding check.

## Solution Assessment

No, not as currently proposed. The decoupling direction may be maintainable, but the patch must first prevent ACP inline delivery without a real binding and align the documented product contract.
