## Summary

Keep open: current main and the latest release still reject unthreaded `mode="session"`, but this PR is not mergeable as proposed because it leaves ACP session-mode inline delivery enabled for headless spawns and still lacks after-fix real behavior proof.

## What This Changes

The PR removes native subagent and ACP guards requiring `thread=true` for `mode="session"`, adds headless-session accepted notes and cleanup, and updates spawn-hook coverage.

## Best Possible Solution

Land a maintainer-approved implementation that decouples session persistence from thread binding while making ACP delivery explicit, with updated tests, docs, and real proof for both headless and thread-bound paths.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.5.7` shows native and ACP guards rejecting unthreaded `mode="session"`, and current tests assert the native rejection path; I did not run a live channel repro in this read-only review.

## Solution Assessment

No, not as proposed. Removing the guards is plausible only after ACP delivery is tied to an actual bound thread or an explicit parent-relay contract so headless persistent sessions do not leak output into requester chats.

## Review Findings

Overall correctness: patch is incorrect
