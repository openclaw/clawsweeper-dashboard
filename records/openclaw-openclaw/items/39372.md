## Summary

Keep open: this PR targets a real requested capability, but it is not safe to merge as-is because the ACP path can still inline-deliver headless session output to the requester chat, the public session/thread contract still needs maintainer approval, and the contributor has not provided real after-fix behavior proof.

## What This Changes

The PR removes native subagent and ACP `mode="session"`/`thread=true` guards, adds headless-session accepted notes and cleanup, and changes spawn-hook coverage to expect unthreaded persistent sessions to proceed.

## Best Possible Solution

If maintainers accept https://github.com/openclaw/openclaw/issues/53548, land a rebased implementation that decouples persistence from thread binding while gating ACP inline delivery on an actual binding, preserving `thread:true` behavior, and updating tests, docs, and changelog.

## Reproduction Assessment

Yes. Source inspection on current main shows native and ACP `sessions_spawn` reject `mode:"session"` without `thread:true`, and the PR diff makes ACP headless session mode reachable while delivery planning remains keyed only on session mode.

## Solution Assessment

No, not as currently proposed. Decoupling may be the right maintainable direction if maintainers accept the product contract, but this PR must first prevent headless ACP inline delivery and align docs/tests with the new behavior.
