## Summary

Keep open: the requested decoupling is not implemented on current main, but this PR is not mergeable as-is because it still leaves ACP session-mode inline delivery keyed to session mode rather than an actual binding, and the external contributor has not provided real after-fix behavior proof.

## What This Changes

The branch removes native subagent and ACP `mode="session"` plus `thread=true` guards, adds headless-session accepted notes and cleanup, and updates spawn-hook coverage for unthreaded persistent sessions.

## Best Possible Solution

Land a maintainer-approved version of the decoupling that gates ACP inline delivery on an explicit binding or relay contract, keeps existing `thread:true` behavior intact, updates docs/tests, and includes redacted live proof for both headless and thread-bound flows.

## Reproduction Assessment

Yes. Source inspection on current main shows both native and ACP guards reject `mode:"session"` without `thread:true`, and the existing spawn-hook test asserts the native rejection path.

## Solution Assessment

No, not as currently proposed. Decoupling persistence from thread binding may be the right product direction, but this patch removes the guard before the ACP delivery contract is made safe for headless sessions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Require a bound thread before ACP inline delivery:** `src/agents/acp-spawn.ts:291-294`
  - body: Removing the ACP `mode="session" && !thread` guard makes headless persistent ACP spawns reachable, but the delivery planner still enables inline delivery for every session-mode spawn with a requester target. Gate inline delivery on an actual binding or explicit relay contract so child output is not sent to the requester conversation from a supposedly headless session.
  - confidence: 0.88
