## Summary

Keep open: current main still rejects unthreaded persistent native and ACP spawns, and this branch changes a documented contract while leaving a concrete ACP inline-delivery blocker unresolved.

## What This Changes

The PR removes native subagent and ACP `mode="session"`/`thread:true` guards, adds headless-session accepted notes and cleanup, and updates spawn-hook coverage to expect unthreaded persistent sessions to proceed.

## Best Possible Solution

If maintainers accept the public contract, land a refreshed implementation that decouples persistence from thread binding, gates ACP inline delivery on an actual binding or explicit parent-stream path, preserves `thread:true` behavior, and updates tests, docs, and changelog.

## Reproduction Assessment

Yes. Source inspection on current main shows native and ACP `sessions_spawn` reject `mode:"session"` without `thread:true`, and existing tests assert the native rejection path.

## Solution Assessment

No, not as currently proposed. Removing the guards is the useful direction only if maintainers accept the contract, but the PR must also prevent unbound ACP inline delivery and refresh docs/tests against current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Require a bound thread before ACP inline delivery:** `src/agents/acp-spawn.ts:291-294`
  - body: Removing the ACP `mode="session" && !thread` guard makes headless persistent ACP spawns reachable, but the delivery planner still enables inline delivery for session-mode spawns with a requester target. That can send child ACP output into the requester conversation even though no thread or conversation binding was prepared, so gate inline delivery on an actual binding or keep headless sessions internal.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The patch has no supply-chain changes, but it introduces a concrete ACP message-routing exposure by allowing headless sessions while inline delivery remains session-mode based.

Concerns:

- **[medium] Headless ACP output can reach requester chat:** `src/agents/acp-spawn.ts:291`
  - body: By allowing ACP session mode without thread binding while leaving inline delivery keyed only on `spawnMode === "session"`, the patch can route child ACP output to a user-facing requester conversation instead of keeping the headless session internal.
