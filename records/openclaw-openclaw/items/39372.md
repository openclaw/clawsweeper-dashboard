## Summary

Keep open: the requested capability is still absent from current main, and this PR still has a blocking ACP routing/security issue plus missing after-fix real behavior proof.

## What This Changes

The PR removes native subagent and ACP `mode="session"`/`thread=true` guards, adds headless-session accepted notes and cleanup, and updates spawn-hook coverage.

## Best Possible Solution

Land a maintainer-approved version that decouples session persistence from thread binding while making ACP delivery explicit, with tests, docs, and real proof for both headless and thread-bound paths.

## Reproduction Assessment

Yes. Source inspection on current main shows native and ACP guards rejecting `mode="session"` without `thread=true`, and the existing spawn-hook test asserts the native rejection path; I did not run a live channel repro in this read-only review.

## Solution Assessment

No, not as proposed. Removing the guards is plausible only after ACP delivery is tied to an actual bound thread or an explicit parent-relay contract so headless persistent sessions do not leak output into requester chats.
