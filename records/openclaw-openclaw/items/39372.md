## Summary

Keep open. Current main still enforces the session/thread coupling, and this PR targets the right area, but the branch leaves an acknowledged ACP delivery bug and still lacks after-fix real behavior proof.

## What This Changes

The PR removes the `mode="session"` plus `thread=true` requirement in native subagent and ACP spawn paths, adds headless-session accepted notes, and updates one spawn-hooks test.

## Best Possible Solution

Land a hardened version that decouples session persistence from thread binding while preserving controlled ACP delivery, continuation metadata, cleanup, docs, and regression coverage for both headless and thread-bound sessions.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the current-main behavior: native and ACP `sessions_spawn` calls with `mode:"session"` and no `thread:true` return explicit thread-required errors before starting the child session.

## Solution Assessment

No. Removing the guards is directionally useful, but the branch is not the best safe fix until ACP inline delivery is gated on actual thread binding and covered by focused tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Gate ACP inline delivery on thread binding:** `src/agents/acp-spawn.ts:291-294`
