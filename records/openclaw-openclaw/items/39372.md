## Summary

Keep open: the underlying decoupling request is still real on current main, but this branch is not mergeable as proposed because it lacks after-fix real behavior proof and leaves ACP headless session output on a thread-bound inline-delivery path.

## What This Changes

The branch removes native subagent and ACP `mode="session"` thread-required guards, adds headless-session acceptance notes and cleanup, and updates spawn-hook coverage for unthreaded persistent sessions.

## Best Possible Solution

Land a maintainer-approved replacement that decouples persistence from thread binding while gating ACP inline delivery on an explicit binding or parent relay, updating docs/tests, and proving both non-threaded and thread-bound behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows native and ACP guards rejecting `mode="session"` without `thread=true`, and existing spawn-hook coverage asserts the native rejection path; I did not run a live channel repro in this read-only review.

## Solution Assessment

No, not as proposed. Removing the guards is plausible only after the ACP delivery contract is tightened so headless persistent sessions cannot accidentally use thread-bound inline delivery semantics.
