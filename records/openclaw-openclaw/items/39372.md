## Summary

Keep open: current main still lacks the requested decoupling, but this PR is not merge-ready because removing the ACP guard makes unbound session-mode spawns hit inline delivery semantics and the external PR still has no real after-fix behavior proof.

## What This Changes

The PR removes the native subagent and ACP `mode="session"` thread requirement, adds headless-session accepted notes and cleanup, and updates spawn-hook coverage.

## Best Possible Solution

Land a maintainer-approved update or replacement that decouples session persistence from thread binding, makes ACP delivery explicit for headless sessions, updates docs/tests, and proves both non-threaded and thread-bound behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows both native and ACP guards rejecting unthreaded `mode="session"`, and existing tests assert the native rejection path; no live channel repro was run in this read-only review.

## Solution Assessment

No, not as proposed. Removing the guards is plausible, but the PR must also fix ACP inline-delivery semantics and align docs/proof before this is the narrow maintainable solution.
