## Summary

Keep open: current main still rejects the requested unthreaded persistent session mode, but this PR is not merge-ready because it conflicts with main, lacks real behavior proof, and leaves ACP delivery plus model-facing tool contract gaps unresolved.

## What This Changes

The PR removes the `mode="session"` plus `thread=true` requirement from native subagent and ACP spawning, adds headless-session accepted notes and cleanup, and updates one spawn-hook test to expect the unthreaded session path to proceed.

## Best Possible Solution

Land a rebased, maintainer-approved decoupling that makes headless native and ACP session delivery explicit, updates model-facing schema/docs/tests, and proves both non-threaded and thread-bound flows.

## Reproduction Assessment

Yes. Source inspection on current main shows native and ACP guards rejecting `mode="session"` without `thread=true`, and `src/agents/sessions-spawn-hooks.test.ts` still asserts that rejection path.

## Solution Assessment

No, not as proposed. Decoupling session persistence from thread binding is plausible, but the PR must first gate or redefine ACP headless delivery, update the schema/docs contract, rebase, and provide real behavior proof.
