## Summary

Keep open: the PR targets a real persistent-session gap, but it should not merge as-is because it removes the ACP thread-required guard while the delivery planner still treats every session-mode ACP spawn as inline-deliverable, and the external PR still lacks real behavior proof.

## What This Changes

The branch removes native subagent and ACP `mode:"session"`/`thread:true` guards, adds headless-session accepted notes and cleanup, and updates spawn-hook coverage for unthreaded persistent sessions.

## Best Possible Solution

Land a hardened version under https://github.com/openclaw/openclaw/issues/53548 that decouples persistence from thread binding, keeps headless sessions internal and continuable via `sessions_send`, preserves `thread:true` behavior, and updates tests plus public docs.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: current main rejects native and ACP `sessions_spawn` with `mode:"session"` and no `thread:true`, and current tests assert that rejection before hooks or gateway calls run.

## Solution Assessment

No, not as currently proposed. Decoupling may be the right direction, but the PR must first gate ACP inline delivery on a real binding or keep headless sessions internal, then provide real Telegram/headless and Discord/thread-bound proof.
