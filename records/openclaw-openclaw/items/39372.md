## Summary

Keep open: current main still rejects unthreaded session mode, but this PR is not mergeable as-is because the ACP inline-delivery contract remains unsafe, real behavior proof is missing, and the live branch is conflicting. The remaining feature is also tracked by https://github.com/openclaw/openclaw/issues/53548.

## What This Changes

The branch removes subagent and ACP session-mode thread guards, adds headless-session notes and cleanup, and updates spawn-hook coverage for unthreaded persistent sessions.

## Best Possible Solution

Land a maintainer-approved replacement that decouples persistence from thread binding while gating ACP inline delivery to a bound thread or explicit parent relay, updating docs/tests, and proving non-threaded plus thread-bound behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows both native and ACP guards reject `mode="session"` without `thread=true`, and existing spawn-hook coverage asserts the native rejection path; I did not run a live channel repro in this read-only review.

## Solution Assessment

No, not as proposed. Removing the guards is plausible only after the ACP delivery contract is tightened so headless persistent sessions cannot accidentally use thread-bound inline delivery semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

- **[P1] Require bound ACP threads before inline delivery:** `src/agents/acp-spawn.ts:289-292`
  - body: Removing the ACP `mode="session" && !thread` guard makes headless session spawns reachable, but `spawnAcpDirect` still treats session-mode spawns as inline-deliverable later.
