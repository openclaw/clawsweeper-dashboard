## Summary

Keep open: the requested decoupling is still not implemented on current main, but this PR is not correct as-is because ACP headless sessions can still use inline delivery semantics and the external contributor has not supplied real behavior proof.

## What This Changes

The PR removes native subagent and ACP `mode="session"`/`thread=true` guards, adds headless-session accepted notes and cleanup, and updates spawn-hook coverage.

## Best Possible Solution

Land a maintainer-approved implementation or replacement PR that decouples session persistence from thread binding while making ACP delivery explicit, updating docs/tests, and proving both headless and thread-bound paths.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.5.7` shows native and ACP guards rejecting unthreaded `mode="session"`, and current tests assert the native rejection path; no live channel repro was run in this read-only review.

## Solution Assessment

No, not as proposed. Removing the guards is plausible only after ACP delivery is tied to an actual bound thread or an explicit parent-relay contract so headless persistent sessions do not leak output into requester chats.
