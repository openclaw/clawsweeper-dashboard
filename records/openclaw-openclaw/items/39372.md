## Summary

Keep this PR open. Current main still rejects unthreaded session-mode native subagent and ACP spawns, so the proposed behavior is not already implemented, but the PR is not safe to merge as shown because the acknowledged ACP inline-delivery problem remains.

## What This Changes

The PR removes the native subagent and ACP `mode="session"` plus `thread=true` guards, adds headless-session accepted notes and failed-start cleanup, and updates spawn-hook coverage for unthreaded persistent sessions.

## Best Possible Solution

If maintainers want headless persistent sessions, land a rebased implementation that decouples persistence from thread binding in both native and ACP paths, keeps `thread:true` behavior intact, disables ACP inline delivery unless a real binding exists, and updates tests plus public docs. If maintainers prefer the current thread-bound-only contract, keep the remaining product discussion on #53548.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: on current main, `spawnSubagentDirect` or `spawnAcpDirect` with `mode:"session"` and no `thread:true` returns the thread-required error before dispatch, and tests assert the native rejection path.

## Solution Assessment

No, not as currently shown. The direction may be maintainable if explicitly accepted, but the PR must first fix ACP inline delivery for headless sessions and rebase over current cleanup/note refactors.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Require thread binding before ACP inline delivery:** `src/agents/acp-spawn.ts:291-294`
  - body: Removing the ACP `mode="session"` guard makes headless sessions reach the delivery planner, but that planner still treats any session-mode spawn with a requester target as `deliver: true`. A headless ACP worker can then write raw output into the requester chat instead of staying internal; gate inline delivery on an actual prepared/bound conversation or disable it for headless sessions.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff does not touch supply-chain files, but it introduces a concrete message-delivery leakage risk in ACP headless session mode.

Concerns:

- **[medium] Headless ACP output can leak into requester chat:** `src/agents/acp-spawn.ts:291`
  - body: By making ACP session mode reachable without thread binding while leaving inline delivery keyed only on session mode, the patch can route worker output into a user-facing requester conversation. This should require an actual bound conversation/thread or stay internal.
  - confidence: 0.86
