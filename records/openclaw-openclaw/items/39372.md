## Summary

Keep open: current main has not implemented headless persistent session mode, and this PR is not mergeable as-is because ACP inline delivery still follows thread-bound session semantics while live proof and maintainer routing-contract review are missing.

## What This Changes

The PR removes native subagent and ACP `mode="session"`/`thread=true` guards, adds headless-session accepted notes and cleanup, and changes spawn-hook coverage to expect unthreaded session spawns to proceed.

## Best Possible Solution

Land a maintainer-approved update or replacement that decouples persistence from thread binding, makes unbound ACP delivery explicit, updates docs/tests, and proves both headless and thread-bound flows.

## Reproduction Assessment

Yes. Source inspection on current main shows both native and ACP guards rejecting unthreaded `mode="session"`, and existing tests assert the native rejection path; no live channel repro was run in this read-only review.

## Solution Assessment

No, not as proposed. Removing the guards is plausible, but the PR must first fix ACP inline-delivery semantics and update the documented routing contract for headless persistent sessions.
