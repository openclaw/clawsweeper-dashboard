## Summary

Keep open: this PR addresses a real, still-open session-persistence gap discussed in https://github.com/openclaw/openclaw/issues/53548, but it should not merge as-is because the ACP headless-session delivery path can still inline-deliver child output to the requester conversation and the PR lacks required real behavior proof.

## What This Changes

The PR removes native subagent and ACP `mode="session"` thread-required guards, adds headless-session accepted notes and cleanup, and changes spawn-hook test expectations.

## Best Possible Solution

Land a maintainer-approved update or replacement that decouples persistence from thread binding, makes ACP delivery explicit, updates docs/tests, and proves both headless and thread-bound behavior.

## Reproduction Assessment

Yes, by source inspection. Current main rejects native and ACP `mode="session"` without `thread=true`, and the existing spawn-hooks test asserts the native rejection path; I did not run a live channel reproduction.

## Solution Assessment

No, not as proposed. Decoupling is plausible, but the PR must also fix ACP inline-delivery semantics, align docs/tests, and include real behavior proof before it is the narrow maintainable solution.
