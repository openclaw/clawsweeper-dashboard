## Summary

Keep open: current main still enforces the documented thread-bound session contract, and this PR's decoupling direction remains incomplete because ACP delivery and the public tool schema still assume session mode is thread-bound.

## What This Changes

The PR removes native subagent and ACP `mode:"session"` plus `thread:true` guards, adds headless-session notes and failed-start cleanup, and updates spawn-hook coverage for unthreaded persistent sessions.

## Best Possible Solution

If maintainers accept #53548, rebase and implement the decoupling end-to-end across direct guards, ACP delivery planning, tool schema/descriptions, docs, changelog, and regression tests; otherwise close the PR in favor of the canonical product issue.

## Reproduction Assessment

Yes. Source inspection of current main shows both native and ACP `sessions_spawn` reject `mode:"session"` without `thread:true`, and the native spawn-hooks test asserts that rejection path.

## Solution Assessment

No. The decoupling direction may be maintainable if accepted, but this patch must first prevent ACP inline delivery without a real binding and update the tool/docs contract that still hides or rejects headless session mode.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require binding before ACP inline delivery:** `src/agents/acp-spawn.ts:291-294`
  - body: Removing the ACP `mode:"session" && !thread` guard makes headless persistent ACP spawns reachable, but the delivery planner still enables `deliver: true` for any session-mode spawn with a requester target. Gate inline delivery on an actual prepared binding so headless sessions do not write child output into the requester conversation.
  - confidence: 0.88
- **[P2] Expose session mode without thread support:** `src/agents/subagent-spawn.ts:262-265`
  - body: The runtime guard is removed, but `createSessionsSpawnToolSchema` still emits only `mode: ["run"]` whenever the current channel lacks thread-binding support. On the non-thread channels this PR is meant to enable, the model-facing `sessions_spawn` contract will still hide or reject `mode:"session"` unless the schema, descriptions, and tests are updated too.
  - confidence: 0.82
