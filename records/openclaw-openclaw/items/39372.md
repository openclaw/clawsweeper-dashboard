## Summary

This PR should stay open: the requested decoupling is still not implemented on current main, but the branch is not safe to merge because ACP inline delivery is still tied to session mode rather than a real thread binding, and the contributor has not provided real after-fix behavior proof.

## What This Changes

The branch removes native subagent and ACP `mode="session"` plus `thread=true` guards, adds headless-session accepted notes and cleanup, and updates spawn-hook coverage for unthreaded persistent sessions.

## Best Possible Solution

Implement the decoupling under a maintainer-approved contract: headless persistent sessions should stay internal or relay through controlled parent/session metadata, ACP inline delivery should require a real binding, existing `thread:true` behavior should remain intact, and docs/tests should move with the runtime change.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: current main still rejects native and ACP `sessions_spawn` with `mode:"session"` and no `thread:true`, and the existing native spawn-hook test asserts that rejection path.

## Solution Assessment

No, not as currently proposed. Decoupling persistence from thread binding may be the right direction, but this patch leaves ACP inline delivery reachable without a bound thread and needs real channel proof plus docs follow-through.
