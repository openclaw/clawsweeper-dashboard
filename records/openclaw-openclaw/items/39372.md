## Summary

Keep this PR open, but the current patch is not mergeable: it targets a real feature gap tracked by https://github.com/openclaw/openclaw/issues/53548, but it removes the ACP session/thread guard while the ACP delivery planner still treats every session-mode spawn as inline-deliverable, and it lacks real behavior proof.

## What This Changes

The PR removes native subagent and ACP thread-required guards for session-mode spawns, adds headless-session accepted notes and cleanup, and updates spawn-hook coverage for unthreaded persistent sessions.

## Best Possible Solution

Harden or replace this branch under https://github.com/openclaw/openclaw/issues/53548 so persistence and thread binding are decoupled deliberately, ACP inline delivery requires an actual binding or stays internal, `thread:true` behavior is preserved, and docs/tests/changelog reflect the accepted contract.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: current main rejects native and ACP `sessions_spawn` with `mode:"session"` and no `thread:true`, and the native spawn-hook test asserts that rejection before hooks or gateway calls run.

## Solution Assessment

No, not as currently proposed. Decoupling may be the right direction, but the PR must first gate ACP inline delivery on a real binding or keep headless sessions internal, then provide real Telegram/headless and Discord/thread-bound proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Require a bound thread before ACP inline delivery:** `src/agents/acp-spawn.ts:289-294`
