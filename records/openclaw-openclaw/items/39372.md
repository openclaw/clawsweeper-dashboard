## Summary

Keep open: current main still enforces the thread-bound session contract, and this PR’s decoupling direction has a blocking ACP delivery issue acknowledged in the review discussion. The related product decision remains tracked in #53548.

## What This Changes

The branch removes native subagent and ACP `mode:"session"` plus `thread:true` guards, adds headless-session accepted notes and failed-start cleanup, and updates spawn-hook coverage for unthreaded persistent sessions.

## Best Possible Solution

If maintainers accept decoupling, land a rebased implementation that updates current native and ACP spawn paths, gates ACP inline delivery on an actual binding, and aligns tests, docs, and changelog with the new contract.

## Reproduction Assessment

Yes. Source inspection of current main shows both native and ACP session-mode spawns reject missing `thread:true`, and the native spawn-hook test asserts that rejection path.

## Solution Assessment

No, not as currently proposed. The decoupling direction may be valid, but the patch must first prevent ACP inline delivery without a real binding and update the documented contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Require a real ACP binding before inline delivery:** `src/agents/acp-spawn.ts:291-294`
