## What This Changes

The issue asks for Telegram inline-button callback turns to wait for or reuse bootstrapped session context on the first tap.

## Best Possible Solution

## Solution Assessment

Yes at the fix direction level. Prewarming the mirrored outbound session after metadata persistence is narrower than changing callback payload semantics or delaying all Telegram callbacks, but this issue should remain open until the linked PR or equivalent lands.
