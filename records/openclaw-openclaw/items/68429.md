## What This Changes

The issue reports that a Telegram inline-button callback on a fresh outbound session can reach the agent before bootstrap/skills context has been loaded.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68433, or replace it with an equivalent narrow outbound-session prewarm that proves the first Telegram callback sees warm bootstrap/skills context.

## Reproduction Assessment

Yes. The issue gives concrete Telegram send-with-buttons steps and transcript ordering, and current main shows callbacks entering the agent path while outbound sends only create metadata before reply-time skill snapshot loading.

## Solution Assessment

Yes at the fix direction level. Prewarming the outbound mirrored session after metadata persistence is narrower and more maintainable than Telegram-only delays or callback payload changes, but it still needs PR review, conflict resolution, and real Telegram proof.
