## What This Changes

Ensure Telegram inline-button callbacks in freshly created outbound delivery sessions see full bootstrap context before the first agent response.

## Best Possible Solution

Review and land PR #68433, or an equivalent narrow fix, so outbound mirrored sessions are warmed with the same bootstrap and skills context expected by the first Telegram callback-driven turn.

## Reproduction Assessment

Yes. The issue gives concrete `openclaw message send --channel telegram --buttons` steps, and current source confirms callbacks enter as synthetic inbound text while outbound mirrored session setup only records metadata before delivery.

## Solution Assessment

Unclear until PR #68433 is reviewed, but the proposed direction is the right boundary: warm the outbound mirrored-session/bootstrap path rather than adding Telegram-only delays or changing callback payload semantics.
