## Best Possible Solution

Review, rebase, validate, and land the linked prewarm PR or an equivalent narrow outbound-session prewarm so the first Telegram callback sees the same skills/bootstrap context as a normal warmed turn.

Yes, source-reproducible with high confidence. ## Solution Assessment

Yes, the linked prewarm direction looks like the narrow maintainable solution. Reusing the existing session skills snapshot path after outbound metadata persistence preserves the existing owner boundary better than adding Telegram-specific callback routing policy.
