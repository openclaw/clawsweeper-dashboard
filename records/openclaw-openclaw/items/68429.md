## Summary

Keep open. An open, unmerged PR (#68433) explicitly closes this issue, and current main still shows outbound session metadata/mirroring plus Telegram callback-to-message dispatch without an equivalent prewarm step before the first callback turn.

## What This Changes

The issue reports that Telegram inline-button callbacks on freshly mirrored outbound sessions can reach the agent before the session bootstrap context is available.

## Best Possible Solution

Review and land #68433, or an equivalent narrow fix, so outbound mirrored sessions are warmed with the same bootstrap and skills context expected by the first Telegram callback-driven turn, with regression coverage for outbound send followed by immediate callback.

## Reproduction Assessment

Yes. The issue provides concrete Telegram message-send and immediate inline-button steps, and current source confirms callback payloads are routed as inbound text while outbound mirrored session creation remains metadata-only.

## Solution Assessment

Unclear until #68433 is reviewed. The direction is appropriate: fix the outbound mirrored-session/bootstrap boundary rather than adding Telegram-only delays or changing callback payload semantics.
