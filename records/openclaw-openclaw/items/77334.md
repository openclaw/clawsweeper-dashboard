## Repair Work Prompt

Fix the Brave web_search cancellation gap behind the hang reported in https://github.com/openclaw/openclaw/issues/77334. Current main passes an AbortSignal from createWebSearchTool into runWebSearch and runWebSearch forwards it to provider execute, but extensions/brave/src/brave-web-search-provider.ts ignores the provider execution context and extensions/brave/src/brave-web-search-provider.runtime.ts does not pass a signal into withTrustedWebSearchEndpoint or withSelfHostedWebSearchEndpoint. Keep the fix extension-owned: pass context?.signal through createBraveToolDefinition into executeBraveSearch, then into both runBraveWebSearch and runBraveLlmContextSearch endpoint calls. Add focused regression coverage showing the Brave provider forwards the supplied signal to the guarded endpoint/fetch path. Do not change provider selection, credentials, Discord behavior, Gateway restart policy, or add config options. Add a single-line CHANGELOG.md fix entry because this is user-facing crash/hang behavior.
