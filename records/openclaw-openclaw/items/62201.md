## Summary

This PR should stay open: current main still lacks the stale diagnostic tool-result replay mitigation, but the latest PR head is not merge-ready because pending replay metadata uses delimiter-parsed string keys that collide with valid colon-containing toolCallIds. The PR body also claims remote validation but does not include inspectable after-fix real behavior proof.

## What This Changes

The PR adds transient diagnostic tool-result replay metadata, tags persisted diagnostic tool results, omits stale or superseded diagnostic output during replay, and expands focused agent replay tests.

## Best Possible Solution

Keep the replay mitigation in the generic agent replay/session guard layer, but store pending replay metadata with collision-free tuple or nested-map keys, add colon-containing toolCallId regression coverage, and require inspectable real behavior proof before merge.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to record pending replay metadata for a session with a toolCallId like `functions.read:0`, run the session-level drain path, and observe that delimiter parsing extracts the wrong session key.

## Solution Assessment

No. The replay-time mitigation is a reasonable direction, but the pending metadata key storage should be collision-free instead of parsing a colon-delimited composite string.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Use collision-free pending replay metadata keys:** `src/agents/tool-result-replay-metadata.ts:42-53`
  - body: `drainPendingToolResultReplayMetadataForSession` derives the session key by splitting `${sessionKey}:${toolCallId}` at the last colon. Current OpenClaw preserves native Kimi IDs such as `functions.read:0`, so cleanup after blocked or aborted tool-call persistence can miss those entries and later apply stale metadata to a reused ID. Store pending metadata as a nested map keyed by session and toolCallId, or encode a tuple instead of parsing a delimiter.
  - confidence: 0.9
