## Summary

Keep open: this is a plausible bug-fix PR, but the current head still has a delimiter parsing bug for colon-containing toolCallIds, is stale against current main package imports, and lacks inspectable real behavior proof.

## What This Changes

The PR adds replay metadata for transient diagnostic tool results, tags exec/read outputs during persistence, omits stale or superseded diagnostic results from replay, and adds focused tests.

## Best Possible Solution

Rebase onto current main, store pending replay metadata in nested session/tool maps or another delimiter-safe tuple, keep the replay omission in sanitizeSessionHistory, add colon-ID coverage, and require redacted after-fix proof.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to record metadata for sessionKey `agent:main` and toolCallId `functions.read:0`, then drain `agent:main`; the PR's last-colon parser extracts `agent:main:functions.read`, so cleanup misses it.

## Solution Assessment

No. The replay-history location is reasonable, but pending metadata should not be stored in a delimiter-parsed composite string, the branch must use the current package namespace, and contributor proof is still missing.
