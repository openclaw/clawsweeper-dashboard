## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Do not re-add metadata after the write hook:** `src/agents/session-tool-result-guard.ts:248-251`
  - body: `before_message_write` is the last plugin hook that can replace or redact a message before persistence, and current tests rely on it for tool-result redaction. This branch applies/stamps replay metadata after that hook returns, so a hook that redacts or rebuilds a tool result can still persist the original diagnostic command/path in `__openclaw.diagnosticTarget`. Make the final metadata pass through the hook, or store only non-sensitive opaque classification data after the hook.
  - confidence: 0.88
- **[P2] Keep session and tool IDs out of the same delimited key:** `src/agents/tool-result-replay-metadata.ts:46-52`
  - body: `toolCallId` can legally contain `:` on current main, for example native Kimi IDs like `functions.read:0`. With this PR, a pending key such as `agent:main:functions.read:0` is later split at the last colon, so `drainPendingToolResultReplayMetadataForSession({ sessionKey: "agent:main" })` misses the orphan entry and can leave stale replay metadata behind for a reused ID. Store the session key separately from the tool call ID, or otherwise make the drain parse unambiguous.
  - confidence: 0.94
