## Summary

Keep open: current main and latest release still omit `replyPayloadSendingHook` from agent-command durable final delivery, and the open closing PR that addresses it is still conflicting.

## What This Changes

The issue reports that restart-recovered main-session replies are written without firing `reply_payload_sending`, so channel plugins can miss delivery.

## Best Possible Solution

Resolve or replace https://github.com/openclaw/openclaw/pull/90300 so current main threads the existing hook metadata through agent-command durable delivery while preserving restart-abort handling.

## Reproduction Assessment

Yes at source level: restart recovery re-enters agent delivery with `deliver: true`, and current main's agent-command durable send call does not include `replyPayloadSendingHook`. I did not run a live restart because this is a read-only review.

## Solution Assessment

Yes: reusing the existing durable `reply_payload_sending` hook field is the narrow owner-boundary fix. A new recovery event or last-reply API would duplicate the documented pre-delivery hook contract.
