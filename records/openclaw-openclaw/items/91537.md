## Summary

Keep open: current main still has a credible Codex app-server lifecycle gap where normal turns preserve the binding sidecar and release a shared-client lease without waiting for the stdio child process to exit, while isolated/manual disposal paths already use `closeAndWait` because the unref'd close path can leave orphans.

## What This Changes

The issue requests that Codex yolo-mode app-server sessions clean up their stdio app-server process and `.codex-app-server.json` sidecar bindings when the controlling OpenClaw session or app closes.

## Best Possible Solution

Keep this issue open and fix the Codex plugin lifecycle by shutting down shared stdio app-server clients with bounded `closeAndWait` on app/session disposal while preserving active or resumable bindings and sweeping only stale sidecars.

## Reproduction Assessment

No, not with live process proof in this read-only review. The source path is reproducible: normal completed turns preserve the binding and release the shared-client lease without closing the stdio app-server, while the code already documents that unref'd close can leave orphaned children.

## Solution Assessment

No, not exactly as blanket session-end binding deletion. The maintainable fix is lifecycle-owned shared-client shutdown on app/session disposal plus a careful stale-binding sweep that does not break intentional resume bindings.
