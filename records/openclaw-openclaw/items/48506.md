## Summary

Keep open: current main and the latest release still expose only the raw `message_sending` destination plus routing metadata, while the requested resolved target context remains a security-sensitive public plugin API decision. The related HELM OSS reference in https://github.com/openclaw/openclaw/issues/48503 is useful design input, but no merged or open canonical replacement owns this exact `message_sending` target-context contract.

## What This Changes

The issue requests optional resolved outbound target context on the `message_sending` plugin hook so security plugins can distinguish internal routing from external or cross-context messages.

## Best Possible Solution

Define a narrow optional `message_sending` target-context contract that reuses canonical outbound target resolution and cross-context policy facts, document the disclosure boundary, and add focused hook/delivery tests after maintainer approval.
