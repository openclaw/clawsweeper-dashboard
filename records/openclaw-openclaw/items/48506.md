## Summary

Keep open. Current `main` still exposes `message_sending` with raw routing fields only, while target resolution and cross-context policy remain internal; this is a new public Plugin SDK and security-policy contract that needs maintainer design.

## What This Changes

Request adding optional resolved target-context fields to `PluginHookMessageSendingEvent` so plugins can distinguish external or cross-context outbound messages.

## Best Possible Solution

Define an additive Plugin SDK target-context contract, populate it from resolver and outbound-policy paths where available, and document/test the routing metadata disclosure boundary.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an additive feature request. Source inspection gives high confidence the SDK gap remains on current `main` and in `v2026.5.12`: the hook receives raw routing fields while resolved target and cross-context data stay internal.

## Solution Assessment

Unclear. The nested optional `targetContext` direction is plausible, but maintainers should decide stable names, redaction/privacy boundaries, same-account semantics, and coordination with the related security-hook proposals before implementation.
