## What This Changes

The issue requests optional resolved target context on `PluginHookMessageSendingEvent` so policy plugins can distinguish external, cross-context, and target-type information without reimplementing OpenClaw routing.

## Best Possible Solution

Approve a narrow optional hook-context contract that reuses canonical target resolution and cross-context facts, documents the security boundary, and adds focused hook/delivery tests.
