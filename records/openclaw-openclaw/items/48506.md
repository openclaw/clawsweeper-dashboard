## Summary

Keep open: current main and v2026.6.8 still expose only the raw outbound destination plus basic routing metadata to `message_sending`, while the requested resolved target context remains a security-sensitive public plugin API design decision.

## What This Changes

Request to add optional resolved target context fields to the `message_sending` plugin hook event so policy plugins can distinguish outbound target type, channel, cross-context status, and externality.

## Best Possible Solution

Define a narrow optional `message_sending` target-context contract that reuses canonical outbound target resolution and cross-context policy facts, documents the disclosure boundary, and adds focused hook/delivery tests after maintainer approval.

## Solution Assessment

Unclear until maintainer design review. Optional nested fields are plausible, but the exact contract and sensitive disclosure boundary should be approved before implementation because this is a public plugin SDK/security surface.
