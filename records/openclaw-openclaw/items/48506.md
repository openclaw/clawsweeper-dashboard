## Summary

Keep open: current main and v2026.6.10 still expose only the raw outbound destination plus basic routing metadata to `message_sending`; adding resolved target context remains a public Plugin SDK and security-disclosure design decision.

## What This Changes

The issue requests optional resolved target context on the `message_sending` plugin hook so policy plugins can distinguish external, cross-context, channel, and same-account outbound destinations.

## Best Possible Solution

Define a narrow optional `message_sending` target-context contract after maintainer approval, reusing canonical outbound resolver and cross-context policy facts and documenting the plugin disclosure boundary.

## Reproduction Assessment

Not applicable as a runtime bug. This is a plugin API feature request, and source inspection confirms the requested target-context field is absent from current main and v2026.6.10.

## Solution Assessment

Unclear until maintainer design review. Optional nested fields are plausible, but the exact target-context contract and sensitive disclosure boundary should be approved before implementation because this is a public Plugin SDK/security surface.
