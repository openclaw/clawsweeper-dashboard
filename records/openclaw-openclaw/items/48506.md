## Summary

The request fits OpenClaw's plugin/security direction, but the public hook shape and disclosure boundary need maintainer review before implementation.

## What This Changes

The issue requests optional resolved target context fields on `PluginHookMessageSendingEvent` so policy plugins can distinguish internal, external, cross-context, channel, account, and target-type facts without reimplementing core routing.

## Best Possible Solution

Define a narrow optional `message_sending` target-context contract that reuses canonical outbound target resolution and cross-context policy facts, document the disclosure boundary, and add focused hook/delivery tests after maintainer approval.
