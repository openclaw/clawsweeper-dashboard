## Summary

Keep open. Current main and the latest release still expose `message_sending` with raw destination/content plus reply/thread metadata only, while target resolution and cross-context policy facts remain internal; this is an additive public Plugin SDK and security-metadata design request.

## What This Changes

Add optional resolved target context to `PluginHookMessageSendingEvent` so plugins can distinguish internal, external, cross-context, and target-type routing decisions.

## Best Possible Solution

Define a narrow additive Plugin SDK target-context contract populated from existing resolver and outbound-policy facts, with tests and docs that lock the metadata disclosure boundary.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an additive Plugin SDK feature request. Source and release inspection show the requested target-context fields are absent while resolver and cross-context policy facts remain internal.

## Solution Assessment

Unclear. A nested optional `targetContext` is plausible, but maintainers need to choose stable names, privacy/redaction boundaries, same-account semantics, allowlist status, and coordination with the related security-hook proposals.
