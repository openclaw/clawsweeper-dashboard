## Summary

Keep open: current main and latest release v2026.6.10 still expose only the raw outbound destination plus basic routing metadata to `message_sending`, while the requested resolved target context remains a security-sensitive public plugin API design decision.

## What This Changes

Add optional resolved target-context fields to the `message_sending` hook so plugins can make outbound destination, DLP, allowlisting, and cross-context policy decisions without reimplementing OpenClaw target resolution.

## Best Possible Solution

Define a narrow optional `message_sending` target-context contract after maintainer approval, reusing canonical outbound resolver and cross-context policy facts and documenting the plugin disclosure boundary.
