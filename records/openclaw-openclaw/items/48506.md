## Summary

Keep open. Current main and v2026.5.7 still expose `message_sending` with raw routing fields only, while resolved target and cross-context state remain internal; this is a new public plugin SDK and security-policy contract that needs maintainer design.

## What This Changes

Request adding optional resolved target context fields to the `message_sending` hook event for DLP, allowlisting, auditing, and cross-context policy plugins.

## Best Possible Solution

Define a backward-compatible optional target-context contract, populate it from the existing resolver/policy paths, and document/test the disclosure boundaries for plugin hooks.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main and v2026.5.7 confirms the requested SDK gap: `message_sending` receives raw routing fields while resolver and cross-context policy data remain internal.

## Solution Assessment

Unclear. The nested optional `targetContext` direction is plausible, but maintainers should decide field names, stability guarantees, and privacy boundaries before implementation.
