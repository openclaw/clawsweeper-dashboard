## Summary

Keep open. Current main and v2026.5.12 still expose `message_sending` with raw routing fields only, while target resolution and cross-context facts remain internal; adding them is a new public Plugin SDK/security disclosure contract that needs maintainer design.

## What This Changes

Request to add optional resolved target context fields to the `message_sending` hook event so plugins can make DLP, allowlisting, and audit decisions without reimplementing target resolution.

## Best Possible Solution

Define a narrow, additive Plugin SDK target-context contract populated from outbound resolver/policy paths, with tests and docs that lock the metadata disclosure boundary.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an additive feature request. Source inspection of current main and v2026.5.12 shows the requested target-context fields are absent while resolver and cross-context policy facts remain internal.

## Solution Assessment

Unclear. The nested optional `targetContext` direction is plausible, but maintainers need to decide stable names, privacy/redaction boundaries, same-account semantics, allowlist status, and coordination with related security-hook proposals.
