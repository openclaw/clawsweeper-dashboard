## Summary

Keep open: current main still has a credible gap between the embedded attempt’s owned-write context and ordinary inbound transcript appends, while the orphan-repair strategy can remove a trailing user leaf before the next provider submission. The report adds a distinct default-concurrency, concurrent-user-message case with alleged persisted-message loss beyond the broader open silent-takeover reports, and no merged fix or current-main proof closes that gap.

## What This Changes

Prevent legitimate concurrent inbound session writes from being treated as external takeovers that abort replies and remove user messages.

## Best Possible Solution

Make ordinary same-process inbound transcript appends participate in the canonical session ownership/publication contract, keep strict rejection for genuinely foreign mutations, and make orphan repair remove a user leaf only when it is proven to be the exact duplicated trigger for the attempt.

## Reproduction Assessment

Yes at source level, with medium confidence: current main still exposes the narrow fence classification, context-bound owned-write publication, and trailing-user repair needed for the reported sequence. A high-confidence live reproduction against current main is still missing.

## Solution Assessment

No, none of the broad alternatives is yet the best proven fix: global serialization sacrifices supported concurrency and a wider whitelist weakens takeover safety. The narrow maintainable direction is to connect inbound appends to existing ownership publication and make cleanup trigger-identity-safe.
