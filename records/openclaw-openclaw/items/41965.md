## Summary

Keep open: current main still lacks the requested `MEDIA:/path#doc` convention, and WhatsApp still chooses image/document delivery from detected media kind rather than a documented force-document signal.

## What This Changes

Request a way to force media URLs such as `MEDIA:/path/to/screenshot.png#doc` to be sent as document attachments instead of inline image media.

## Best Possible Solution

Define one documented document-delivery control, then thread it through WhatsApp media loading and payload selection while preserving default image sends.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main has no `#doc` parsing and no WhatsApp image-to-document override path.

## Solution Assessment

Unclear. The `#doc` fragment is plausible, but current main already has `forceDocument`/`asDocument` concepts with Telegram-only docs, so maintainers should choose the public API before implementation.
