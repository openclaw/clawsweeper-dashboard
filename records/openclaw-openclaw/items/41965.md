## Summary

Keep open: current main still lacks `MEDIA:/path#doc` parsing, WhatsApp still does not honor the existing document-forcing signal end to end, and the newer channel-level default request needs maintainer API/config judgment rather than cleanup closure.

## What This Changes

Request a way to force media attachments, originally via a `#doc` media fragment and later via channel-level defaults, to send as document attachments instead of compressed images.

## Best Possible Solution

Land a corrected per-call WhatsApp `forceDocument` implementation first, then make a separate maintainer decision on whether OpenClaw should also support a documented `#doc` media hint or channel-level default.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a strict bug reproduction because this is a feature/API request; source inspection gives high confidence that current main lacks `#doc` parsing, a channel-level default, and WhatsApp end-to-end document forcing for image media.

## Solution Assessment

Is this the best way to solve the issue? Unclear: the existing per-call `forceDocument`/`asDocument` surface is a narrower path than a new `#doc` fragment, while the requested channel-level default needs explicit config/API product judgment.
