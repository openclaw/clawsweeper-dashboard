## Summary

Keep open. Current main and `v2026.6.8` already ship per-call document delivery for Telegram and WhatsApp, but this issue still asks for a new public media-delivery surface such as `#doc` syntax or a channel-level default, which needs maintainer product/API direction.

## What This Changes

The issue requests a way to force media that would normally be sent as an image to be delivered as a document attachment so platform compression can be avoided.

## Best Possible Solution

Keep the shipped per-call controls and add a maintainer-approved media-delivery contract only if the chosen default or syntax can preserve per-call overrides, media safety checks, and channel-specific behavior.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.8` expose per-call document delivery, while targeted source searches found neither `#doc` media syntax nor a channel-level force-document default.

## Solution Assessment

Unclear. The shipped per-call implementation is the right baseline, but the best remaining solution depends on maintainer choice between a channel default, media fragment syntax, another API, or staying with explicit per-call controls.
