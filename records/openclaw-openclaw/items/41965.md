## Summary

Keep open: current main still lacks a documented media-level `#doc` hint, WhatsApp still does not honor the existing `forceDocument`/`asDocument` signal end to end, and the newer channel-level default request needs maintainer API/config judgment. The open WhatsApp PR is the right near-term path for one slice, but it is still open and conflicting.

## What This Changes

Request a way to force outbound media to be delivered as document attachments rather than compressed image/video media, originally via a `MEDIA:/path#doc` hint and later with a possible channel-level default.

## Best Possible Solution

Land a corrected per-call WhatsApp `forceDocument` implementation, then make a separate maintainer decision on whether OpenClaw should expose `#doc`, a channel-level default, or both with docs and regression coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction because this is a feature/API request; source inspection on current main shows no `#doc` parser, no WhatsApp end-to-end document forcing, and no channel-level default.

## Solution Assessment

Is this the best way to solve the issue? Unclear: the existing `forceDocument`/`asDocument` per-call surface is narrower than a new URL fragment, while the channel-level default proposal needs explicit config/API product judgment.
