## Summary

Keep open. Current main and v2026.6.8 cover per-call document delivery for Telegram and WhatsApp, but the discussion now asks for a new public channel-level default or alternate media syntax, which is still missing and needs maintainer API direction.

## What This Changes

The issue requests a way to force media that would normally be sent as images to be delivered as document attachments, originally via a `#doc` media fragment and later with channel-level defaults.

## Best Possible Solution

Choose a maintainer-approved public surface for default document delivery, likely channel-owned config with explicit per-send override semantics or a rejected alternative documented against the existing per-call knobs.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.8 expose per-call `--force-document`/`forceDocument`/`asDocument`, while the searched config schemas and media parser do not expose a channel-level default or `#doc` parser.

## Solution Assessment

Unclear. The existing per-call mechanism is the best current shipped behavior for explicit sends, but the remaining deployer-default or media-syntax shape needs maintainer API direction before it is safe to implement.
