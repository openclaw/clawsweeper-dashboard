## Summary

Keep open. Current main and v2026.6.6 cover per-call document delivery for Telegram and WhatsApp, but the unresolved request is a new deployer-level/media-syntax control such as `channels.telegram.forceDocument` or `#doc`, which needs maintainer product/API direction.

## What This Changes

The issue requests a way to force outbound media, especially images, to be delivered as document attachments so channel platforms do not compress them.

## Best Possible Solution

Keep the shipped per-call behavior and make a maintainer-approved media-delivery contract for any default or fragment syntax, preserving per-call overrides plus existing size, MIME, and local-media safety checks.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 expose per-call document delivery, while Telegram and WhatsApp config types have no channel-level force-document default and the media loader has no `#doc` fragment handling. No live channel send is needed to prove the remaining missing surface.

## Solution Assessment

Unclear. The shipped per-call implementation is the right baseline, but the best remaining solution depends on maintainer choice between a channel default, fragment syntax, or keeping only explicit per-call controls.
