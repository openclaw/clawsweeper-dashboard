Keep open: current main now implements the per-call `forceDocument`/`asDocument` path for WhatsApp and Telegram, but this issue still contains unresolved public API/config scope around a `#doc` media hint and a deployer-level channel default. ## What This Changes

Request a way to force media to be sent as document attachments, originally proposed as `MEDIA:...#doc`, to avoid platform compression and possibly default that behavior per channel.

## Best Possible Solution

Keep the merged per-call implementation, then make an explicit maintainer decision on whether the remaining public surface should be `#doc`, a channel-level default, both, or not planned.
