## Summary

Keep open: current main has a Telegram-only `--force-document` / `asDocument` path, but not the requested `MEDIA:...#doc` directive or a WhatsApp image-to-document override. The related #42044 PR was closed unmerged, and #55363 was related but does not make this feature available.

## What This Changes

Request support for a `#doc` media URL fragment that strips before media resolution and forces media to document delivery, especially for WhatsApp.

## Best Possible Solution

Define one supported document-delivery hint, document its channel scope, and thread it through parsing, media loading, outbound/gateway transport, and WhatsApp delivery only if maintainers want WhatsApp to support it.

## Reproduction Assessment

Not applicable as a feature request. Source inspection shows current main has no `#doc` directive parser or loader override, while docs only advertise Telegram `--force-document`.

## Solution Assessment

Unclear until maintainers choose the public API. A narrow implementation could work, but the safer direction is to decide whether `#doc` should become a supported `MEDIA:` directive or whether callers should use the existing action parameter surface.
