## Summary

Keep this open: current main has a Telegram-only `forceDocument`/`asDocument` path, but the requested `MEDIA:/path#doc` mechanism is not implemented and WhatsApp still routes image MIME types to image payloads rather than document payloads.

## What This Changes

The issue requests a way to force outbound media, especially WhatsApp screenshots, to be delivered as document attachments instead of compressed inline media.

## Best Possible Solution

Add a maintainer-approved canonical document-delivery control that threads through outbound parsing and WhatsApp media preparation, with docs explicitly naming supported channels and preserving existing Telegram behavior.

## Reproduction Assessment

Yes from source for the missing capability: `loadWebMediaInternal` does not strip or interpret `#doc`, and WhatsApp sends `image/*` media as image payloads. I did not run a live WhatsApp delivery test.

## Solution Assessment

Unclear. The requested `#doc` fragment is one possible interface, but current main already has `forceDocument`/`asDocument` and docs mark it Telegram-only, so maintainers should choose the public surface before implementation.
