## Summary

Keep open. Current main still does not implement the requested MEDIA `#doc` delivery hint, and the newer generic `forceDocument` parameter is not enough for this report because it is not carried through the gateway send schema/payload or consumed by the WhatsApp outbound adapter. Images still classify as images and WhatsApp still sends image MIME through image payloads, while docs continue to scope `--force-document` to Telegram.

## What This Changes

Request a way to force outbound media, especially WhatsApp images, to be delivered as document attachments rather than compressed inline media.

## Best Possible Solution

Add a shared outbound media delivery hint that is parsed before media resolution, strips only the reserved hint from the resolved path, preserves the original MIME and filename, and lets WhatsApp direct send plus auto-reply choose a document payload for otherwise image media. The fix should cover parser/loader behavior, gateway/direct message sending, WhatsApp payload selection, and user-facing docs.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: `MEDIA:/tmp/screenshot.png#doc` is currently treated as a plain URL/path string, and a successfully loaded PNG still classifies as `image` and follows WhatsApp image-send branches.

## Solution Assessment

Unclear for the exact `#doc` suffix as the final API, because URL fragments and literal `#` path characters need precise handling. The maintainable solution is a structured `forceDocument`/`asDocument` delivery hint carried through media parsing/loading and outbound delivery, with `#doc` supported only if it has safe reserved-suffix semantics.
