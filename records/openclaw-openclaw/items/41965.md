## Summary

Keep open. Current main and the latest release already support per-call document delivery for Telegram and WhatsApp, but the remaining request is a new public media-delivery surface such as a channel default or `#doc` syntax, which still needs maintainer product/API direction.

## What This Changes

Request to let users force media, especially screenshots and images, to be delivered as document attachments instead of compressed image/video payloads.

## Best Possible Solution

Keep the shipped per-call controls and add a maintainer-approved media-delivery contract only if the chosen default or syntax can preserve per-call overrides, media safety checks, and channel-specific behavior.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.6` expose per-call document delivery, while the inspected Telegram/WhatsApp config types have no channel-level force-document default and the shared media loader has no `#doc` handling.

## Solution Assessment

Unclear. The shipped per-call implementation is the right baseline, but the best remaining solution depends on maintainer choice between a channel default, media fragment syntax, another API, or staying with explicit per-call controls.
