## Summary

Keep this open: current `main` still lacks the requested `MEDIA:/path#doc` parsing, and WhatsApp outbound media still does not consume the existing `forceDocument` signal before choosing the Baileys image/document payload path.

## What This Changes

Request a way to force outbound media, especially WhatsApp images, to be sent as document attachments, with `MEDIA:/path#doc` suggested as the user-facing control.

## Best Possible Solution

Adopt one canonical document-delivery override, document supported channels, and thread it through media parsing plus WhatsApp outbound media preparation without changing unrelated channel behavior.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path for the missing capability: `loadWebMediaInternal` does not interpret `#doc`, and WhatsApp image MIME types still route to image payloads. I did not run a live WhatsApp send.

## Solution Assessment

Unclear. The requested `#doc` fragment is plausible, but current `forceDocument`/`asDocument` is explicitly Telegram-only, so maintainers should choose the public API before implementation.
