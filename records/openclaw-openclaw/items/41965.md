## Summary

Keep open. Current main and v2026.6.8 already support per-call document delivery for Telegram and WhatsApp, but the remaining request is a new public media-delivery surface such as a channel default or `#doc` syntax, which still needs maintainer product/API direction.

## What This Changes

Request a public way to force outbound media to be sent as document attachments, originally via `#doc` media syntax and later discussion of channel-level defaults.

## Best Possible Solution

Keep the shipped per-call controls and add a maintainer-approved media-delivery contract only if the chosen default or syntax preserves explicit overrides, safety checks, and channel-specific behavior.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.8 expose per-call document delivery, while the inspected Telegram/WhatsApp config types have no channel-level force-document default and the shared media loader has no `#doc` handling.

## Solution Assessment

Unclear. The shipped per-call implementation is the right baseline, but the best remaining solution depends on maintainer choice between a channel default, media fragment syntax, another API, or staying with explicit per-call controls.
