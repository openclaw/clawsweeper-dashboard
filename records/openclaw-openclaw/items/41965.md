## Summary

Keep open: current main and v2026.6.10 support per-call `forceDocument`/`asDocument` for Telegram and WhatsApp, but the requested durable `MEDIA:#doc` syntax or channel-level default is still absent and needs maintainer product/API direction.

## What This Changes

The issue requests a way to force media attachments to be delivered as document/file attachments rather than inline/compressed images, initially via a `MEDIA:` `#doc` fragment and later via channel-level defaults.

## Best Possible Solution

Choose one canonical delivery contract, then implement it consistently at the message-tool/channel boundary with docs, focused tests, and live transport proof for the affected channels.

## Reproduction Assessment

Not applicable as a strict bug reproduction, but source inspection gives high confidence that current main lacks both `MEDIA:#doc` handling and a channel-level force-document default. Per-call Telegram/WhatsApp `forceDocument` is present.

## Solution Assessment

No, the current per-call implementation is only a partial solution for explicit sends. The best fix still requires maintainer choice of the public syntax/default contract before adding code.
