## Summary

Keep open: current main still does not provide the requested `MEDIA:/path#doc` convention, and the WhatsApp path still lacks an end-to-end force-document payload override. There is an active focused PR for the per-call WhatsApp `forceDocument` path, but it remains open with a review finding, and the newer channel-level default request still needs maintainer API/product judgment.

## What This Changes

Request a way to force media sends to use document attachment delivery, initially via a `MEDIA:/path#doc` hint, so platforms such as WhatsApp and Telegram avoid image compression.

## Best Possible Solution

Review and repair the active WhatsApp `forceDocument` PR for the per-call path, then decide separately whether this issue should also add a documented `#doc` media hint or channel-level default.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/API request. Source inspection gives high confidence that current main lacks `#doc` parsing and lacks a WhatsApp force-document override for image MIME media.

## Solution Assessment

Unclear. The requested `#doc` fragment is plausible, but current main already has a `forceDocument`/`asDocument` signal and an active WhatsApp PR uses that path, so maintainers should choose the canonical public surface before expanding the feature further.
