## Summary

Keep open: current main still lacks the requested `MEDIA:/path#doc` convention, and WhatsApp media delivery still selects image/document payloads from detected MIME/kind rather than an explicit document-forcing signal. The request is useful, but it is a new user-facing media/channel behavior because the existing public `--force-document` surface is still documented as Telegram-only.

## What This Changes

Add a public way, such as a `#doc` fragment on `MEDIA:` URLs, to force media to be sent as a document attachment instead of an inline image.

## Best Possible Solution

Choose a canonical public document-delivery control, document supported channels, and thread it through WhatsApp media loading and payload selection without changing normal image sends.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main has no `#doc` parsing and no WhatsApp document-forcing path for image MIME media.

## Solution Assessment

Unclear. The requested `#doc` fragment is plausible, but current main already has `forceDocument`/`asDocument` with Telegram-only docs, so maintainers should choose the public API before implementation.
