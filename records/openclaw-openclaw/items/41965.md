## Summary

Keep open: current main still lacks the requested `MEDIA:/path#doc` parsing and WhatsApp still does not honor the existing per-call document-forcing signal end to end. The active WhatsApp PR is the right immediate implementation path for part of this, but it is still open and the newer channel-level default proposal needs maintainer API judgment.

## What This Changes

Request to let outbound media be sent as document attachments instead of compressed image/photo uploads, originally via a `MEDIA:` `#doc` hint and later also as a possible channel-level default.

## Best Possible Solution

Land a corrected per-call WhatsApp `forceDocument` implementation first, then make a separate maintainer decision on whether to add a documented `#doc` media hint or a channel-level default.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug reproduction because this is a feature/API request; source inspection gives high confidence that current main lacks both `#doc` parsing and WhatsApp end-to-end document forcing for image media.

## Solution Assessment

Is this the best way to solve the issue? Unclear: the existing per-call `forceDocument`/`asDocument` surface is a narrower path than a new `#doc` fragment, while the requested channel-level default needs explicit config/API product judgment.
