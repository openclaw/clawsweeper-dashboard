## Summary

The PR is a focused fix for the linked plugin discovery bug, but it remains compatibility-sensitive because it disables currently tested standalone-file discovery from auto-discovered global and workspace extension roots.

## What This Changes

The PR adds a `scanFiles` gate to plugin discovery so auto-discovered extension roots ignore top-level helper files while configured load-path directories and installed records can still discover standalone plugin files.

## Best Possible Solution

Before merge, either add upgrade handling and docs for existing implicit standalone files or have maintainers explicitly accept package-only auto-discovered roots with `plugins.load.paths` as the standalone-file path.
