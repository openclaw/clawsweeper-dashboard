## Summary

Keep open: current main and v2026.6.6 still pass configured Voice Call realtime tool schemas to providers while only binding the shared `openclaw_agent_consult` handler, so the requested `end_call` or explicit action-binding path remains unimplemented and needs a maintainer policy decision.

## What This Changes

Request that Voice Call either bind a built-in realtime `end_call` tool or provide an explicit action-binding path for configured `realtime.tools`.

## Best Possible Solution

Define an explicit Voice Call realtime call-control contract, then implement the chosen bounded handler path with regression tests and docs for which realtime tools can execute.

## Reproduction Assessment

Yes, from source inspection. Current main exposes configured realtime tool schemas to providers, binds only `openclaw_agent_consult`, and returns the missing-handler error for any advertised tool name without a registered handler.

## Solution Assessment

Unclear. The suggested direct `end_call` registration is plausible, but the best maintainable fix depends on whether maintainers want a narrow built-in call-control tool or a broader action-binding contract.
