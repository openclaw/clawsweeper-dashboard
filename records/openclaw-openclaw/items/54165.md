## Summary

Keep this PR open, but do not merge the current diff: current main still passes heartbeat payload text through, so the heartbeat fix is useful, but the PR also adds Telegram draft-stream `message:sent` emission that conflicts with current preview-finalized delivery semantics and can fire with partial or duplicate content.

## What This Changes

The PR adds heartbeat-mode regex stripping of `[TOOL_CALL]`/`[TOOL_RESULT]` bracket blocks in `buildReplyPayloads` and wires Telegram draft-stream `message:sent` hook emission with a dispatch test.

## Best Possible Solution

Land a narrow heartbeat sanitizer in `src/auto-reply/reply/agent-runner-payloads.ts` with focused regression coverage for `[TOOL_CALL]` and `[TOOL_RESULT]` heartbeat replies, and handle any Telegram streaming hook changes separately against the existing #50917 preview-finalization semantics.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildReplyPayloads` with `isHeartbeat: true` and a text payload containing `[TOOL_CALL]...[/TOOL_CALL]`; current main preserves that text because the heartbeat branch returns `params.payloads`.

## Solution Assessment

No. The heartbeat sanitizer is the right narrow direction, but this PR is not the best patch because it bundles an unrelated Telegram hook timing change that should be split, removed, or redesigned around finalization/deduplication.
