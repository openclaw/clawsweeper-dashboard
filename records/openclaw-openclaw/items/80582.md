## Summary

This issue should stay open: current `main` still has the placeholder path the report describes, and the fix is a narrow memory-core bug fix rather than a cleanup-close candidate.

## What This Changes

The issue asks memory-core to reject markdown placeholder snippets and avoid relocating promotion candidates onto structurally meaningless markdown ranges.

## Best Possible Solution

Add a shared unpromotable-snippet guard in memory-core, use it at store load, recording, ranking, apply/rehydration, and require meaningful content before substring relocation, with focused regression tests.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a short-term memory result or grounded candidate with snippet `-` passes recording, ranking, and apply filters because only empty and dreaming-contaminated snippets are rejected. A focused regression test in `extensions/memory-core/src/short-term-promotion.test.ts` should make this fail on current main.

## Solution Assessment

Yes, the proposed direction is the narrowest maintainable fix: centralize placeholder rejection next to the existing dreaming-contamination filter and tighten substring relocation instead of changing promotion thresholds or product behavior.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/80582. In `extensions/memory-core/src/short-term-promotion.ts`, add a shared helper that treats empty snippets, bare markdown bullets (`-`, `*`, `+`), and markdown skeleton ranges made only of headings plus empty bullets as unpromotable. Use it consistently where snippets enter or leave the short-term promotion pipeline: store normalization, `recordShortTermRecalls`, `recordGroundedShortTermCandidates`, `rankShortTermPromotionCandidates`, `applyShortTermPromotions`, and rehydration/relocation. Tighten substring relocation so broad `includes` matches only run when the candidate and window snippets have meaningful token content; preserve exact-match behavior for real snippets. Add focused tests in `extensions/memory-core/src/short-term-promotion.test.ts` for placeholder helper behavior, not recording/ranking placeholder-only grounded candidates, and not rehydrating real section summaries onto empty bullet placeholders. Add a `CHANGELOG.md` fix entry because the change affects user-visible memory promotion output. Do not change promotion thresholds, plugin boundaries, memory file formats, or unrelated dreaming behavior.
