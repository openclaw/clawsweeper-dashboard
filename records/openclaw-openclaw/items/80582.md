## Summary

Keep open: current main still lets placeholder-only snippets pass the short-term promotion recording, ranking, and apply filters, and the relocation matcher can match tiny markdown placeholders by exact or substring comparison.

## What This Changes

The issue asks memory-core to reject markdown placeholder-only short-term snippets and avoid relocating candidates onto structurally empty markdown ranges.

## Best Possible Solution

Add one shared unpromotable-snippet predicate for short-term promotion and use it consistently in store normalization, recall/grounded recording, ranking, apply, and relocation while preserving ordinary non-empty bullet content.

## Reproduction Assessment

Yes, as a source-level reproduction path: a '-' snippet can pass recording, ranking, and apply because current filters only reject empty or contaminated dreaming snippets, and relocation accepts exact or substring matches. I did not run a live filesystem test because this sweep was read-only.

## Solution Assessment

Yes. The proposed direction is the narrow maintainable fix: centralize placeholder filtering near the existing contamination filtering and add a meaningful-content guard before substring relocation.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/80582. In memory-core short-term promotion, prevent markdown placeholder-only snippets from being recorded, ranked, rehydrated, or applied. Add a shared predicate near the existing dreaming-contamination helper that treats empty snippets, bare markdown bullets ('-', '*', '+'), and skeleton ranges made only of headings plus empty bullets as unpromotable. Use it in store normalization, recordShortTermRecalls, recordGroundedShortTermCandidates, rankShortTermPromotionCandidates, applyShortTermPromotions, and relocation/rehydration. Also require meaningful content before substring relocation so tiny placeholders cannot match live daily-note ranges; preserve exact/substring relocation for real snippets such as ordinary bullet text. Add focused regression tests in extensions/memory-core/src/short-term-promotion.test.ts for placeholder classification, grounded placeholder recording/ranking, and not rehydrating a real candidate onto an empty bullet placeholder. Add a CHANGELOG.md fix entry if this is treated as user-visible memory behavior. Do not add config knobs or broaden dreaming policy.
