## Summary

Keep open: current main still has the short-term promotion placeholder gap, and an open PR with closing syntax already targets this issue, so this should close only after that fix or an equivalent one lands.

## What This Changes

The issue requests filtering markdown placeholder snippets from short-term memory promotion so empty bullets or heading skeletons cannot be promoted into long-term memory.

## Best Possible Solution

Add one shared unpromotable-snippet predicate across store load, recording, ranking, apply/rehydration, and relocation, with regression coverage for placeholder skeletons and preservation of real bullet content.

## Reproduction Assessment

Yes, as a source-level reproduction path: current main accepts non-empty markdown markers through the empty/dreaming-contamination gates, then ranking/apply and relocation can reuse them. I did not run a filesystem fixture because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is a shared unpromotable-snippet guard at every promotion boundary plus meaningful-content checks for relocation; the linked PR is the current implementation candidate.
