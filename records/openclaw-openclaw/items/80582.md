## Summary

Keep open: current main still has no markdown-placeholder filter in short-term promotion, and the issue discussion now includes sanitized field evidence that the placeholder promotions occurred in a real workspace.

## What This Changes

Prevent short-term memory promotion from storing, ranking, applying, or relocating markdown skeleton placeholders as durable memory candidates.

## Best Possible Solution

Add one shared unpromotable short-term snippet predicate and apply it consistently at store load, recall/grounded recording, ranking, apply, and relocation while preserving real short bullet content.

Yes, as a source-level path: '-' normalizes to a non-empty snippet, passes the current recording/ranking/apply gates, and can match live placeholder lines during relocation. I did not run a filesystem reproduction because this review is read-only.

## Solution Assessment

Yes. The proposed direction is the narrow maintainable fix: centralize placeholder filtering near the existing contamination filtering and require meaningful content before relocation substring matches.
