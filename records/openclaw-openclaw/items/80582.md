## What This Changes

The issue asks memory-core to reject markdown placeholder or skeleton snippets from short-term promotion recording, ranking, relocation, and apply paths.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82028 or an equivalent memory-core guard, then close this issue once the fix is on main with regression proof.

## Reproduction Assessment

Yes, as a source-level path: current main accepts non-empty markdown markers through the existing empty/dreaming-contamination gates, then ranking/apply and relocation can reuse them. I did not run a filesystem fixture because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is a shared unpromotable-snippet predicate at every promotion boundary plus a meaningful-content guard for relocation; the open PR follows that direction, so the issue should stay open until it lands or is replaced.
