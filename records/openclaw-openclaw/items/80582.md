## Summary

Keep open: current main still has no structural placeholder filter in the short-term promotion entry, ranking, apply, or relocation paths, and the reporter added sanitized workspace evidence that this produced real MEMORY.md pollution.

## What This Changes

The issue asks memory-core to reject markdown placeholder-only snippets during short-term promotion and to make relocation avoid matching onto meaningless placeholder ranges.

## Best Possible Solution

Add one shared unpromotable short-term snippet predicate and use it consistently across store load/normalization, recall and grounded recording, ranking, apply, and relocation while preserving ordinary bullet notes with real content.

## Reproduction Assessment

Yes, as a source-level reproduction: a '-' snippet is non-empty, passes the current empty/dreaming-contamination filters, can rank/apply, and relocation can match exact or substring placeholder ranges. I did not run a live workspace reproduction because this review is read-only.

## Solution Assessment

Yes. Centralizing placeholder/marker-only filtering near the existing contamination predicate and adding a meaningful-content guard for relocation is the narrow maintainable fix; downstream cleanup alone would leave the bad candidates entering the pipeline.
