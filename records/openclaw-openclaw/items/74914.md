It also needs the required changelog entry.

## What This Changes

The PR adds `google-vertex` to the model API enum/generated schema and adds tests for native Google Vertex config validation and Google provider normalization.

## Best Possible Solution

Revise this PR so `google-vertex` is accepted by the config schema and preserved through runtime provider/model API resolution, then add a focused inline-provider inheritance regression test and a single-line active changelog entry for the user-facing fix.

## Reproduction Assessment

Yes. ## Solution Assessment
