## Summary

Keep this PR open. Current main still rejects `api: "google-vertex"` in the config schema, and the PR is directionally useful, but it does not yet preserve provider-level native Vertex API inheritance through the inline-provider runtime path, so the accepted config can still miss the native Vertex stream selection unless every model repeats the API value.

## What This Changes

The PR adds `google-vertex` to the model API config enum/generated schema and adds tests for native Google Vertex config validation and Google provider normalization.

## Best Possible Solution

Extend this PR so `google-vertex` is treated consistently across schema validation, generated schema, provider-level inline API inheritance, and Google provider runtime selection, with focused regression coverage for a provider-level native Vertex config whose models omit per-model `api`.

## Reproduction Assessment

Yes. A high-confidence source reproduction is available by tracing a config with `models.providers.google-vertex.api = "google-vertex"` and a model without `api` through `buildInlineProviderModels`; current normalization drops the provider-level API before the Google provider can select the native Vertex stream.

## Solution Assessment

No. Adding the enum and generated schema value is necessary, but the narrow maintainable fix also needs the inline-provider runtime allowlist and a focused test proving provider-level `google-vertex` inheritance reaches the resolved model.
