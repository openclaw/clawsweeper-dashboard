## Summary

Keep open: current main interrupts registered active embedded/Codex runs before Gateway compaction, but it does not prove the narrower Codex-native liveness/write-lock requirement before `thread/compact/start` is fully handled.

## What This Changes

The issue asks for `sessions.compact` to reject, defer, or liveness-guard compaction when a Codex app-server session is actively writing a turn.

## Best Possible Solution

Keep native Codex compaction, but require the Gateway/Codex boundary to prove the target session is idle or return a retryable active-session/deferred result before starting native compaction.

## Reproduction Assessment

No high-confidence live current-main reproduction was run; source inspection shows broad active-run interruption exists, but the Codex native compact start path still lacks a native busy/write-lock response if reached.

## Solution Assessment

Unclear: a Codex-owned liveness/write-lock guard or retryable active-session/deferred result is the safer direction, but maintainers should decide whether abort-before-compact is sufficient.
