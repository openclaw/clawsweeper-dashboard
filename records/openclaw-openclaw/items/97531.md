## Summary

Keep open: the report is credible, and current main/latest release only prove a broad active-run interrupt before native Codex compaction, not that every active Codex app-server turn is rejected/deferred or protected from incomplete-turn fallback.

## What This Changes

The issue asks for `sessions.compact` to avoid mutating Codex app-server sessions while an active turn, tool chain, write lock, or sidecar lease is in progress.

## Best Possible Solution

Keep native Codex compaction, but require `sessions.compact` to prove the target Codex session is idle before `thread/compact/start` and add focused active-turn regression coverage.

## Reproduction Assessment

No high-confidence current-main reproduction: the issue has strong 2026.6.9 live evidence, but current main/release now has an active-run interrupt before no-`maxLines` compaction and I did not prove a remaining failing path.

## Solution Assessment

Unclear: a Codex liveness/write-lock guard plus regression test is the right direction, but maintainers should decide whether the current interrupt behavior is sufficient or should become reject/defer for active turns.
