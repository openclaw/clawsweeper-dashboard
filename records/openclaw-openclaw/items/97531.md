## Summary

Keep open: current main and the latest release include a broad active-run interrupt before Gateway compaction, but they do not prove that explicit Codex app-server `sessions.compact` rejects or defers every active Codex turn before starting native `thread/compact/start`.

## What This Changes

The issue asks for `sessions.compact` to avoid mutating or starting native compaction on an active Codex app-server turn and to reject or defer with clear retryable behavior.

## Best Possible Solution

Keep Codex native compaction ownership, but add a Codex-aware idle/liveness gate for `sessions.compact` plus regression coverage for an unfinished tool-use turn before starting `thread/compact/start` or truncating mirrors.

## Reproduction Assessment

No high-confidence current-main live reproduction was run; source inspection shows tracked embedded runs are interrupted, but there is still no explicit Codex sidecar/write-lock active-turn check while the issue provides concrete 2026.6.9 live evidence.

## Solution Assessment

Unclear: the current interrupt guard is a useful mitigation, but the best fix should define a deliberate reject, defer, or idle-check contract for Codex app-server compaction instead of relying only on broad abort behavior.
