## Summary

Keep open: current main and the latest tag have the broad active-run interrupt guard before `sessions.compact`, but the remaining Codex-native liveness/write-lock requirement is not fully proved because the manual Codex path still starts `thread/compact/start` once that guard is bypassed or misses a turn.

## What This Changes

The issue reports that `sessions.compact` can mutate a live Codex app-server session during an active turn and trigger an incomplete-turn fallback after tool work.

## Best Possible Solution

Keep native Codex compaction, but make the Gateway/Codex boundary prove the target session is idle or return a retryable active/deferred result before starting native compaction.

## Reproduction Assessment

Source-reproducible: current main has broad active-run interruption, but the manual Codex native compaction path still reaches `thread/compact/start` without a Codex-native idle/write-lock proof if the active-run registry misses the turn. No live current-main reproduction was run.

## Solution Assessment

No: current main is a plausible mitigation for registered runs, but it is not yet the best complete solution for the native sidecar/write-lock liveness gap. The safer end state is a retryable active/deferred result or explicit idle proof before native compaction starts.
