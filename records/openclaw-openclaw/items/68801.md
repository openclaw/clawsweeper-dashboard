## Summary

Keep this PR open, but the current patch should not merge: it addresses a real orphan-start cleanup concern by age-pruning starts, yet that can delete active-run start timestamps needed when terminal lifecycle events omit `startedAt`, and external real behavior proof is still missing.

## What This Changes

The PR adds a TTL-based sweep of `agentRunStarts` in `src/gateway/server-methods/agent-job.ts` using the existing `agentRunCache` TTL.

## Best Possible Solution

Use lifecycle-aware cleanup that can drain true orphan starts without deleting timestamps for runs that may still be active, with fake-timer coverage for both orphan cleanup and long active runs crossing the cache TTL.

## Reproduction Assessment

Yes. Against the PR diff, a lifecycle `start` with `startedAt`, a prune after the 10-minute TTL, and then a terminal `end` without `startedAt` loses the original start timestamp by source inspection.

## Solution Assessment

No. Reusing the completed snapshot cache TTL for active-run starts is not the narrowest maintainable fix because active runs can last up to 48 hours and terminal lifecycle events often rely on the fallback map.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Preserve start timestamps for active runs:** `src/gateway/server-methods/agent-job.ts:37-42`
  - body: This loop deletes `agentRunStarts` entries based only on age, but the map is also the fallback `startedAt` source for terminal lifecycle events that omit it. Active runs can exceed the 10-minute cache TTL, so a later `end` or `error` for that still-active run can make `agent.wait` return a snapshot without its original `startedAt`.
  - confidence: 0.94
