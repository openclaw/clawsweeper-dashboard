## Summary

Keep this PR open, but do not merge the current patch: it addresses a plausible orphaned-start leak, yet the proposed TTL sweep can delete active-run start timestamps that `agent.wait` still needs.

## What This Changes

The PR adds a TTL sweep of Gateway `agentRunStarts` entries inside `pruneAgentRunCache()`.

## Best Possible Solution

Use lifecycle-aware cleanup that drains true orphan starts without deleting timestamps for runs that may still be active, with fake-timer coverage for both orphan cleanup and long active runs crossing the cache TTL.

## Reproduction Assessment

Yes. Against the PR diff, a lifecycle `start` with `startedAt`, a prune after the 10-minute TTL, and then a terminal `end` without `startedAt` loses the original timestamp by source inspection.

## Solution Assessment

No. Reusing the completed snapshot cache TTL for active-run starts is not the narrowest maintainable fix because active runs can last up to 48 hours and terminal lifecycle events often rely on the fallback map.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Preserve start timestamps for active runs:** `src/gateway/server-methods/agent-job.ts:39-42`
  - body: This loop deletes `agentRunStarts` entries based only on age, but that map is also the fallback `startedAt` source for terminal lifecycle events that omit it. Active runs can exceed the 10-minute cache TTL, so a later `end` or `error` for a still-active run can make `agent.wait` return a snapshot without its original `startedAt`.
  - confidence: 0.94
