## Summary

Keep open: the underlying orphaned-start cleanup may be valid, but this PR's TTL-only sweep can remove `startedAt` for active long-running runs before their terminal lifecycle event arrives, and the external PR still lacks inspectable after-fix real behavior proof.

## What This Changes

The PR adds a TTL sweep for `agentRunStarts` inside `pruneAgentRunCache()` alongside the existing completed-run cache pruning.

## Best Possible Solution

Use lifecycle-aware cleanup that drains true orphan starts without deleting active-run start timestamps, with focused fake-timer coverage for orphan cleanup and long active runs crossing the cache TTL.

## Reproduction Assessment

Yes. By source inspection against the PR diff, a start event with `startedAt`, a later prune after the 10-minute TTL, and a terminal event without `startedAt` can make `agent.wait` return a snapshot without the original start timestamp.

## Solution Assessment

No. Reusing the completed snapshot cache TTL for `agentRunStarts` is not the narrowest maintainable fix because active runs can last much longer than 10 minutes and terminal events still rely on that fallback map.
