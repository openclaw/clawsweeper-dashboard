## Summary

Keep this PR open, but do not merge it as-is. The reported leak is real on current main, but the proposed age-only sweep would also delete `agentRunStarts` for legitimate active runs older than 10 minutes, breaking the existing fallback used when terminal lifecycle events omit `startedAt`.

## What This Changes

The PR adds a TTL sweep for `agentRunStarts` inside `pruneAgentRunCache()` in `src/gateway/server-methods/agent-job.ts`.

## Best Possible Solution

Revise the fix so orphan cleanup does not remove start timestamps for still-active runs. A maintainable direction is either to make terminal lifecycle snapshots self-contained by consistently carrying `startedAt`, then prune old starts conservatively, or to track active run ownership and only sweep starts that are known orphaned. Add focused fake-timer coverage for both orphan cleanup and an active run that crosses the TTL before ending.

## Reproduction Assessment

Yes. Apply the PR, emit a lifecycle `start` with `startedAt`, advance past 10 minutes and trigger `pruneAgentRunCache()`, then emit a terminal `end` without `startedAt`; the returned `agent.wait` snapshot can no longer preserve the start timestamp. Current tests already cover terminal events that omit `startedAt`, which makes this a high-confidence static reproduction path.

## Solution Assessment

No. The patch uses the completed-snapshot cache TTL as if it were an active-run timeout, but active agent runs can legitimately last far longer than 10 minutes. The safer solution needs lifecycle-aware pruning or terminal events that always include the start timestamp before this map is aged out.
