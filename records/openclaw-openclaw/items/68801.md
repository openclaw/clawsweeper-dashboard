## Summary

Keep open. The PR targets a real narrow leak, but the proposed age-only sweep can remove `startedAt` for still-active runs because current Gateway lifecycle handling relies on `agentRunStarts` as a fallback when terminal events omit `startedAt`, and normal agent runs may last far longer than the 10-minute cache TTL.

## What This Changes

The PR adds a TTL sweep for `agentRunStarts` entries inside Gateway `pruneAgentRunCache()`.

## Best Possible Solution

Revise the PR so orphan cleanup never removes start timestamps for runs that may still be active. A maintainable path is either to make all terminal lifecycle snapshots self-contained with `startedAt` before pruning old starts, or to track active run ownership and sweep only starts known to be orphaned, with fake-timer coverage for both orphan cleanup and a long active run crossing the TTL.

## Reproduction Assessment

Yes. Apply the PR, emit a lifecycle `start` with `startedAt`, advance beyond 10 minutes and trigger pruning, then emit terminal `end` without `startedAt`; the `agent.wait` snapshot loses the fallback start timestamp. Current tests already model terminal events without `startedAt`, so this is a high-confidence static reproduction path.

## Solution Assessment

No. Reusing the completed-snapshot cache TTL as an active-run timeout is not the narrowest safe fix because active runs can legitimately last up to 48 hours. The safer solution needs lifecycle-aware pruning or self-contained terminal events first.
