## Summary

Keep open: the PR targets a plausible Gateway bookkeeping leak, but the current patch can remove `startedAt` for active long-running agent jobs and still lacks inspectable real behavior proof.

## What This Changes

The PR adds a TTL sweep for `agentRunStarts` inside Gateway agent-run cache pruning.

## Best Possible Solution

Use lifecycle-aware cleanup that drains only starts known to be orphaned while preserving active-run start timestamps, with focused coverage for both paths.

## Reproduction Assessment

Yes, by source inspection: emit `start` with `startedAt`, let the PR's TTL sweep delete the map entry after 10 minutes, then emit a terminal event without `startedAt`; `agent.wait` can lose the original timestamp. I did not run tests because this review was read-only.

## Solution Assessment

No. Reusing the completed snapshot cache TTL for active-run starts is not the narrowest maintainable fix because active runs can last far longer and terminal events still rely on the fallback map.
