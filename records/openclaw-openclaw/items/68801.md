## Summary

Keep open: the PR targets a real Gateway bookkeeping leak, but the proposed age-only sweep can drop the only fallback `startedAt` for still-active runs that outlive the 10-minute cache TTL. The external PR also only reports checks/tests, so it still needs real behavior proof before merge.

## What This Changes

The PR adds a TTL sweep for `agentRunStarts` in Gateway `pruneAgentRunCache()`, using the existing 10-minute `agentRunCache` TTL.

## Best Possible Solution

Revise the cleanup so orphaned starts can drain without deleting start timestamps for runs that may still be active, and add fake-timer coverage for both orphan cleanup and long active runs crossing the TTL.

## Reproduction Assessment

Yes. With the PR applied, emit lifecycle `start` with `startedAt`, trigger the new prune loop after ten minutes, then emit terminal `end` without `startedAt`; source inspection shows `agent.wait` can no longer recover the original start timestamp.

## Solution Assessment

No. Reusing the completed snapshot cache TTL as an active-run-start TTL is not the narrowest maintainable fix because active runs can legitimately last up to 48 hours and terminal events often omit `startedAt`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92
