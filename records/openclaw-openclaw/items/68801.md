## Summary

Keep open: the PR targets a plausible Gateway bookkeeping leak, but the proposed TTL sweep can delete active-run start timestamps before terminal lifecycle events arrive, and the PR still lacks inspectable real behavior proof.

## What This Changes

The PR adds a TTL sweep for `agentRunStarts` inside `pruneAgentRunCache()` in `src/gateway/server-methods/agent-job.ts`.

## Best Possible Solution

Use lifecycle-aware cleanup that removes starts only for runs known to be orphaned or no longer active, with regression coverage for both orphan cleanup and active runs that cross the 10-minute cache TTL.

## Reproduction Assessment

Yes. Source inspection gives a concrete sequence: emit `start` with `startedAt`, let the PR's TTL sweep delete the map entry after 10 minutes, then emit a terminal event without `startedAt` and `agent.wait` can lose the original timestamp.

## Solution Assessment

No. Reusing the completed snapshot cache TTL for active-run starts is not the narrowest maintainable fix because active runs default to 48 hours and terminal events still rely on the fallback map.
