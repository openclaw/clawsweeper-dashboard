## Summary

Keep open: the PR addresses a plausible Gateway bookkeeping leak, but the age-only sweep can erase active run start metadata before terminal lifecycle events that omit `startedAt` are converted into `agent.wait` snapshots, and the PR still lacks inspectable real behavior proof.

## What This Changes

The branch adds a 10-minute TTL sweep for `agentRunStarts` inside `pruneAgentRunCache()`.

## Best Possible Solution

Use lifecycle-aware cleanup that removes starts only for runs known to be orphaned or no longer active, with regression coverage for both orphan cleanup and active runs that cross the 10-minute cache TTL.

## Reproduction Assessment

Yes, by source inspection: emit `start` with `startedAt`, let the PR's TTL sweep delete the map entry after 10 minutes, then emit a terminal event without `startedAt`; `agent.wait` can lose the original timestamp. I did not run the repro because this review is read-only.

## Solution Assessment

No. Reusing the completed snapshot cache TTL for active-run starts is not the narrowest maintainable fix because active runs can last far longer and terminal events still rely on the fallback map.
