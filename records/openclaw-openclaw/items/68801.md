## Summary

Keep open: the leak target is plausible, but this patch is not mergeable because it can delete active-run start timestamps before terminal lifecycle events arrive, and the external PR still lacks real behavior proof.

## What This Changes

The branch adds an age-based TTL sweep for Gateway `agentRunStarts` entries during `pruneAgentRunCache()`.

## Best Possible Solution

Implement lifecycle-aware cleanup that drains provably orphaned starts without deleting active-run timestamps, with fake-timer coverage for orphan cleanup and long active runs crossing the cache TTL.

## Reproduction Assessment

Yes, at source level. Against the PR diff, emit a lifecycle `start` with `startedAt`, let the new prune loop run after the 10-minute TTL, then emit terminal `end` or `error` without `startedAt`; the snapshot fallback map has been cleared.

## Solution Assessment

No. Reusing the completed snapshot cache TTL for active-run start metadata is not the narrowest maintainable fix because active runs can last up to 48 hours and terminal lifecycle events may omit `startedAt`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Preserve start timestamps for active runs:** `src/gateway/server-methods/agent-job.ts:39-42`
