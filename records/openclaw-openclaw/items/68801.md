## Summary

Keep this PR open, but the current patch should not merge as-is: it targets a plausible Gateway bookkeeping leak, yet the added TTL sweep can delete start timestamps for still-active long-running agent runs and the external PR still lacks real behavior proof.

## What This Changes

Adds a TTL-based `agentRunStarts` sweep inside `pruneAgentRunCache()` in `src/gateway/server-methods/agent-job.ts`.

## Best Possible Solution

Implement lifecycle-aware cleanup that drains true orphan starts without deleting timestamps for active runs, with focused fake-timer coverage for both orphan cleanup and long active runs crossing the cache TTL.

## Reproduction Assessment

Yes. By source inspection of the PR diff, a lifecycle `start` with `startedAt`, a prune after the 10-minute TTL, and then a terminal `end` without `startedAt` loses the original start timestamp.

## Solution Assessment

No. Reusing the completed snapshot cache TTL for active-run starts is not the narrowest maintainable fix because active runs can last up to 48 hours and terminal lifecycle events often rely on the fallback map.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Preserve start timestamps for active runs:** `src/gateway/server-methods/agent-job.ts:37-42`

Active runs can exceed the 10-minute cache TTL, so a later `end` or `error` can make `agent.wait` return a snapshot without its original `startedAt`.
