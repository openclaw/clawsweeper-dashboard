## What This Changes

The PR adds a Channels-table merge helper that appends rows from live `channels.status.channelAccounts` when read-only channel discovery returns no row, wires it into status scan overview, and adds regression tests plus a changelog entry.

## Best Possible Solution

Land a gateway-backed row merge only if maintainers want live snapshots to supplement setup fallback, with proof against a configured channel and regression coverage that preserves the fast-status boundary.

## Reproduction Assessment

Yes. Source inspection plus the linked reports give a high-confidence reproduction shape: a reachable gateway has `channels.status.channelAccounts`, read-only discovery yields no table rows, and current main returns the read-only rows without merging live accounts.

## Solution Assessment

Mostly yes for live status paths. The additive merge preserves richer read-only rows and fills gaps from the gateway snapshot, but it should be framed as a live-status augmentation because default fast status still relies on setup fallback rather than fetching `channels.status`.
