## Summary

Keep this PR open: the live gateway merge is useful for `--all`/`--deep`, but it does not fix the plain `openclaw status` regression it closes because the default command still disables both live `channels.status` and setup fallback rows. The PR also needs to preserve optional `connected` semantics from the channel status contract.

## What This Changes

The PR adds a helper that synthesizes missing Channels rows from the live gateway `channels.status` payload, wires it into status overview collection, adds tests, and updates the changelog.

## Best Possible Solution

Land a revised PR that keeps default status fast while showing configured channels, keeps live gateway augmentation for `--all`/`--deep`, and treats omitted gateway connection state as unknown.

## Reproduction Assessment

Yes. Current main structurally reproduces the fast-path gap: default `scanStatus({ json: false })` disables `channels.status` and setup fallback, and related reports show the channel appears in `channels status` or `--deep` but not plain status.

## Solution Assessment

No. The proposed gateway merge is a useful augmentation for live status paths, but the narrow maintainable fix also needs a fast default row source and optional `connected` handling that matches the existing channel status contract.
