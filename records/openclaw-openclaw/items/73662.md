## Summary

Keep this PR open: current main does not contain the proposed gateway-row synthesis, and the PR branch still has blocking correctness issues around the default fast status path, the setup-fallback option contract, and optional gateway snapshot fields.

## What This Changes

The PR adds a helper to synthesize missing `openclaw status` Channels rows from the live `channels.status` gateway snapshot, wires it into status overview collection, adds focused tests, and updates the changelog.

## Best Possible Solution

Land a revised PR that deliberately populates fast default status channel rows without regressing startup cost, treats omitted snapshot fields as unknown, and covers both `scanStatus({ json: false })` and live/deep merge paths.

## Reproduction Assessment

Yes. The linked reports provide command-level reproductions, and current main has a test preserving the default path where live channel status and setup fallback are disabled.

## Solution Assessment

No. The gateway merge helper is useful for live/deep paths, but this PR does not deliberately solve the default fast command path and misreads optional connection state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Pass the setup fallback option name:** `src/commands/status.scan-overview.ts:257`
  - body: This calls `buildChannelsTable` with `includeSetupRuntimeFallback`, but the helper only reads `includeSetupFallbackPlugins`. The caller's false value is ignored, so the fast-path setup fallback contract is no longer controlled by the scan option.
  - confidence: 0.93
- **[P2] Cover the default status path:** `src/commands/status.scan-overview.ts:259-261`
  - body: The merge only runs when `channelsStatus` is non-null, but the reported `openclaw status` path passes `includeLiveChannelStatus: false`. Once the fallback option is wired correctly, plain status can still render a header-only Channels table, so this needs a command-level regression and a deliberate fast row source.
  - confidence: 0.9
- **[P2] Treat omitted connected as unknown:** `src/commands/status-all/channels.ts:236`
  - body: `ChannelAccountSnapshot.connected` is optional, and the existing formatter only reports disconnected when the field is explicitly `false`. Requiring `connected === true` makes running/configured snapshots that omit the field synthesize a warning without a reported disconnect.
  - confidence: 0.88
