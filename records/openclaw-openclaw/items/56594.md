## What This Changes

Issue reports that running `openclaw doctor` updated the install to 2026.3.24 and caused all Telegram/WhatsApp model calls through OpenAI and Google to return `404 status code (no body)`.

## Best Possible Solution

Keep the shipped provider normalization, doctor write-back, and 404 fallback fixes; no separate source change is needed for this historical 2026.3.24 report.

## Reproduction Assessment

No, I did not establish the failing 2026.3.24 runtime path on current main because it requires the reporter's old doctor-mutated config and provider credentials. The related reporter comment plus current source/tests provide high-confidence fixed-path evidence instead.

## Solution Assessment

Yes. Current main addresses the maintainable owner-boundary fixes: normalize provider base URLs/transports at provider/model resolution, avoid doctor write-back contamination, and classify bare 404s for fallback rather than adding a special case for one old config shape.
