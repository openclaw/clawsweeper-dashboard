## Summary

Keep open: the PR is a focused patch, but it removes the shipped and documented beta-channel fallback contract rather than proving the linked stale-beta report requires raw persistent beta selection.

## What This Changes

The PR removes beta-to-latest fallback logic from npm channel resolution, updates resolver tests to expect raw beta selection, and deletes CLI dry-run/downgrade handling for that fallback.

## Best Possible Solution

Preserve the shipped beta fallback unless maintainers explicitly approve raw persistent beta selection; if policy changes, align docs and add fresh-install, missing-beta, newer-stable downgrade, `--yes`, and auto-update coverage.

## Reproduction Assessment

## Solution Assessment

No. Deleting the fallback is a compatibility policy change, not the narrowest repair for a stale beta result; the safer path is to preserve fallback and diagnose metadata freshness, or intentionally change the public contract with docs and upgrade proof.
