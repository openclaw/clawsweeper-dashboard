## Summary

Keep this PR open but do not merge it as-is: it removes a shipped and documented beta-channel fallback, and the linked stale-beta report has not been tied to this resolver rather than npm dist-tag publication or caching.

## What This Changes

The branch removes beta-to-latest fallback from npm update channel resolution, removes the CLI fallback note and downgrade bypass handling, and changes resolver tests to expect raw beta dist-tag selection.

## Best Possible Solution

Keep the shipped beta fallback unless maintainers explicitly approve raw persistent beta selection; if approved, align docs and add fresh-install, missing-beta, newer-stable-to-beta, downgrade-confirmation, and real registry proof.

## Reproduction Assessment

Partly yes: the beta-to-latest fallback and this PR's policy reversal are reproducible from source and tests. The linked beta-3-versus-beta-4 failure is not high-confidence reproduced because no historical npm dist-tags, before output, or logs were provided.

## Solution Assessment

No. Deleting the fallback is not the best fix without a maintainer policy decision, because current code, docs, tests, and the latest release all define --channel beta as fallback-capable while --tag beta is the raw dist-tag escape hatch.
