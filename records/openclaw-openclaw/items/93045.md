## Summary

Keep open: the PR is a narrow and source-supported fix for the linked gateway usage undercount, but it still needs real gateway/session-store proof before merge because the submitted proof is unit-test/mock-only.

## What This Changes

This PR changes `sessions.usage` to aggregate usage across all matched sessions while using `limit` only for returned session rows, and adds a focused regression test.

## Best Possible Solution

Merge this narrow gateway fix after redacted real gateway proof shows totals are stable across different `limit` values while returned session rows remain capped.

## Reproduction Assessment

Yes. Current main slices to `limitedEntries` before loading usage and building aggregate totals, so any mocked or real store with more matched sessions than `limit` undercounts totals while returning only the page.

## Solution Assessment

Yes. The PR fixes the right owner boundary by keeping `limit` scoped to returned rows and aggregating through the existing cached/background session summary path instead of adding a new config or duplicate reporting path.
