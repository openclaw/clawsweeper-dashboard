The PR changes the gateway `sessions.usage` handler and adds a regression test so aggregate usage is computed over all matched sessions while returned session rows remain limited.

## Best Possible Solution

## Solution Assessment

Yes. The PR keeps the fix in the gateway method that owns `sessions.usage`, preserves the cache-backed loader, and scopes `limit` to returned rows instead of adding config or a duplicate aggregation path.
