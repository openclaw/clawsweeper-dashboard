## Summary

Keep open. The PR targets a real source-reproducible thrown context-overflow reset gap, but the branch is not merge-ready because it resets on broad mixed fallback-summary matching, needs to rebase onto current main’s shared recovery text and embedded-overflow handling, and still has only mock/unit proof.

## What This Changes

The PR adds session reset handling and regression tests for thrown and final embedded context-overflow paths in `src/auto-reply/reply/agent-runner-execution.ts`.

## Best Possible Solution

Land a rebased narrow fix that resets only pure unrecoverable context-overflow session failures through the existing shared recovery helper, with focused regression coverage and redacted real Telegram/runtime proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: mock `runWithModelFallback` to throw a plain context-overflow error and current main classifies it as overflow but only resets when the message is also a compaction failure.

## Solution Assessment

No, not as currently written. Reusing the existing reset callback is the right direction, but the reset must be gated on pure overflow failures, reuse the shared recovery copy, and be rebased around current main’s embedded-overflow handling.
