## Summary

## What This Changes

The PR updates `runAgentTurnWithFallback` to reset active sessions after thrown and final embedded context-overflow failures, with regression tests for those paths.

## Best Possible Solution

Land a rebased, narrow runtime fix that resets only pure unrecoverable context-overflow session failures through the existing callback/shared recovery text, with focused tests and a single-line changelog entry.

## Reproduction Assessment

Yes. A focused unit test can mock `runWithModelFallback` to reject with a mixed `FallbackSummaryError` containing one context-overflow attempt plus a different terminal cause, then assert the reset callback is not called.

## Solution Assessment

No. Reusing the existing reset callback is the right direction, but this patch should classify pure overflow separately, reuse current shared recovery copy, and add the required changelog entry.
