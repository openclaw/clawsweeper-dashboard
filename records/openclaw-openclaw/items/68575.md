## Best Possible Solution

Refit the change onto current main as a narrow thrown-overflow recovery: reset only for an unambiguous overflow/compaction failure, reuse `buildContextOverflowRecoveryText`, keep the existing embedded-result reset path, and add tests for direct thrown overflow plus mixed fallback-summary no-reset behavior.
