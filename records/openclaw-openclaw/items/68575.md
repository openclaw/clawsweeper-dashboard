## Summary

Keep open. The bug this PR targets is still source-reproducible on current main, but the proposed patch is not ready to merge because it can reset sessions on mixed-cause fallback failures, overlaps current main's embedded-overflow reset path, and still lacks contributor-supplied real Telegram/runtime proof.

## What This Changes

The branch adds session resets for thrown and final embedded context-overflow errors in `runAgentTurnWithFallback` and adds mocked regression tests for both paths.

## Best Possible Solution

Rebase or replace this with a narrow fix that resets only pure unrecoverable context-overflow session failures through `buildContextOverflowRecoveryText`, with focused tests and redacted Telegram/runtime proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a thrown plain context-overflow error is classified as overflow on current main, but the reset helper is only called for compaction failures.

## Solution Assessment

No, not as currently written. Reusing the existing reset helper is the right direction, but the reset must be gated on pure overflow failures, use the shared recovery text, and be rebased around current main's embedded-overflow handling.
