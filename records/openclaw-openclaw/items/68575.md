## Summary

Keep open. Current main still has the plain thrown context-overflow reset gap, but this PR needs a rebase, a narrower pure-overflow reset guard, and contributor-supplied real Telegram/runtime proof before it is mergeable.

## What This Changes

The PR adds auto-recovery session resets for thrown and final embedded context-overflow paths in auto-reply execution, plus focused regression tests.

## Best Possible Solution

Rebase or replace this with a narrow fix that resets only pure unrecoverable context-overflow session failures through buildContextOverflowRecoveryText, with focused tests and redacted Telegram/runtime proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: mock runWithModelFallback to throw a plain context-overflow error and current main classifies it as overflow but only resets when the same message is also a compaction failure.

## Solution Assessment

No, not as currently written. Reusing the existing reset callback is the right direction, but the reset should be gated on pure overflow failures, use the shared recovery text helper, and be rebased around current main's embedded-overflow handling.
