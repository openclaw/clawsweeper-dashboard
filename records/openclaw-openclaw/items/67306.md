## Summary

Keep this open. Current main still lacks a first-class JSON Schema input contract on the requested CLI/Gateway inference surfaces, although the related raw `infer model run` behavior has landed and there is reusable schema-validation machinery elsewhere in the repo.

## What This Changes

Add first-class JSON Schema input, schema validation, defined retry/repair behavior, and machine-readable validation failures for OpenClaw inference CLI surfaces such as `openclaw infer model run` and eventually `openclaw agent`.

## Best Possible Solution

Implement a provider-neutral schema contract on `openclaw infer model run` first: accept schema from a file or inline JSON, parse and validate final JSON output with shared structured validation errors, keep local and Gateway behavior consistent, and document the retry/failure policy. Broader `openclaw agent` exposure can be a follow-up once the core inference contract is stable.

## Reproduction Assessment

Not applicable as a feature request, but the gap is reproducible by static inspection: the current command metadata, parser, execution path, Gateway schema, and CLI docs expose JSON output only, not schema input or schema validation semantics.

## Solution Assessment

Yes for a narrow first implementation. A provider-neutral `infer model run` contract is more maintainable than exposing provider-specific native schema modes as the CLI contract; provider-native support can later be an optimization behind the same validation envelope.

## Repair Work Prompt

Implement first-class JSON Schema support for `openclaw infer model run` only. Add a `--json-schema <path-or-inline>` or similarly named option to the command metadata/parser, parse either a JSON file path or inline JSON object, require/parse the model output as JSON when a schema is provided, validate it with the existing shared JSON Schema validation helper where appropriate, and return machine-readable success/failure envelopes. Define the invalid-output policy explicitly in code/docs: either no repair or a bounded repair/retry loop, but do not leave behavior implicit. Keep provider-native schema modes out of the public contract for this first PR. Preserve the current lean raw model-run path and do not broaden `openclaw agent` unless a minimal additive Gateway protocol field is required for `infer model run --gateway`. Update `docs/cli/infer.md`, focused CLI tests, and Gateway protocol/server tests if protocol fields change. Add a `CHANGELOG.md` user-facing feature entry crediting `Thanks @howardclaw26`. Do not change unrelated provider behavior, bundled plugin defaults, or broad agent runtime policy.
