## Summary

Keep this issue open. Current main still lacks a first-class JSON Schema input contract on the requested core CLI surfaces: `infer model run` only exposes JSON output, `openclaw agent` has no schema option, and the Gateway agent protocol has no schema/structured-output field for validation or schema-specific failure reporting.

## What This Changes

Request first-class JSON Schema input support for `openclaw agent` and `openclaw infer model run`, including validation and machine-readable invalid-output handling.

## Best Possible Solution

Define a narrow provider-neutral JSON Schema contract on `infer model run` first, using the existing validator where practical, with explicit invalid JSON/schema failure semantics and documented retry/no-retry behavior.

## Reproduction Assessment

Not applicable as a feature request. Static inspection gives high-confidence evidence of the gap because the CLI metadata, parser, execution path, Gateway schema, and docs expose JSON output but no caller-supplied schema contract.

## Solution Assessment

Yes, a narrow provider-neutral `infer model run` contract is the best first step, with provider-native schema modes treated later as an optimization behind the same validation envelope. Broad `openclaw agent` exposure should be a follow-up API decision unless the first contract requires a minimal additive Gateway field.
