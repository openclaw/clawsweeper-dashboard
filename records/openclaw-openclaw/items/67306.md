## Summary

Keep this issue open. Current main still has no first-class JSON Schema contract on `openclaw infer model run`, `openclaw agent`, or the Gateway agent protocol; the adjacent `llm-task` and media-understanding structured APIs are useful precedent but do not satisfy the requested core CLI surface.

## What This Changes

Request first-class JSON Schema input, schema validation, optional repair/retry behavior, and machine-readable failure reporting for `openclaw agent` and `openclaw infer model run`.

## Best Possible Solution

Define a provider-neutral JSON Schema contract on `openclaw infer model run` first, accepting file or inline schema input, validating final JSON with structured errors, and documenting retry/no-retry semantics before broadening to `openclaw agent`.

## Reproduction Assessment

Not applicable as a feature request. Static inspection gives high-confidence evidence of the gap because current CLI metadata, parser, execution path, Gateway schema, and docs expose JSON output but no caller-supplied schema contract.

## Solution Assessment

Yes, the narrowest maintainable direction is to add a provider-neutral contract to `infer model run` first. Broad `openclaw agent` exposure and provider-native schema modes should be follow-up design decisions unless the first contract needs a minimal additive Gateway field.
