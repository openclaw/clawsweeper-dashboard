## Summary

Keep this open. Current main still has JSON output and an adjacent optional `llm-task` tool with schema validation, but the requested caller-supplied JSON Schema contract, validation path, repair/retry policy, and schema-specific failure envelope are not present on the `openclaw infer model run` or `openclaw agent` CLI/Gateway surfaces.

## What This Changes

Add first-class caller-supplied JSON Schema support to agent/inference CLI surfaces such as `openclaw agent` and `openclaw infer model run`, ideally including validation and invalid-output repair semantics.

## Best Possible Solution

Define an additive provider-neutral JSON Schema contract on `openclaw infer model run` first, with file/inline schema input, final-output validation, explicit retry or no-retry semantics, structured errors, docs, and focused CLI/protocol tests.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Static inspection gives high-confidence evidence of the gap because the current CLI metadata, parser, execution path, Gateway schema, and docs expose JSON output but no caller-supplied schema contract.

## Solution Assessment

Partly: first-class schema validation is the right direction, but the exact surface and repair semantics need maintainer API/product decisions. A narrow provider-neutral `infer model run` implementation is the most maintainable first step.
