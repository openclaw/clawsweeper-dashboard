## Summary

Keep this issue open. Current main has raw `infer model run` behavior, but it still does not expose a JSON Schema input contract, validate final model output against a caller schema, or define schema-specific machine-readable failure semantics on the requested CLI/Gateway surfaces.

## What This Changes

Add first-class JSON Schema support to OpenClaw inference CLI surfaces such as `openclaw infer model run` and possibly `openclaw agent`.

## Best Possible Solution

Implement a provider-neutral JSON Schema contract on `openclaw infer model run` first, with schema parsing from file or inline JSON, final-output JSON parsing and validation, structured validation errors, documented retry behavior, and local/Gateway parity where the protocol needs an additive field.

## Reproduction Assessment

Not applicable as a feature request, but the gap is high-confidence by static inspection: command metadata, parser options, execution paths, Gateway schema, and docs expose JSON output only, not schema input or validation semantics.

## Solution Assessment

Yes, a narrow provider-neutral `infer model run` contract is the best first step. Provider-native schema modes can later be an implementation optimization behind the same CLI validation envelope, and broader `openclaw agent` exposure can be a follow-up.

## Repair Work Prompt

Implement first-class JSON Schema support for `openclaw infer model run` for https://github.com/openclaw/openclaw/issues/67306. Add a provider-neutral CLI contract such as `--json-schema <path-or-inline>` on `infer model run`, parse schema input safely, require/parse final model text as JSON when a schema is supplied, validate with the repo's existing AJV-style structured validation behavior or a shared helper factored out of plugin-specific code, and return machine-readable schema/JSON failure details in the existing JSON envelope. Define the invalid-output policy explicitly; a bounded no-repair first version is acceptable if documented, but do not leave behavior implicit. Preserve local model-run behavior and Gateway raw-run behavior; add only an additive Gateway protocol field if Gateway parity needs it. Do not expose provider-specific native schema modes as the CLI contract, do not deep-import `extensions/llm-task/src/**`, and do not broaden to the full `openclaw agent` surface unless a minimal shared protocol field is necessary. Update `docs/cli/infer.md` and `CHANGELOG.md`; update `docs/cli/agent.md` only if the agent surface changes.
