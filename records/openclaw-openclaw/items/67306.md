## Summary

Keep open. Current main and v2026.5.18 still expose JSON output, adjacent JSON Schema validators, and provider-native response_format plumbing, but not a caller-supplied JSON Schema contract, validation/repair policy, or schema-specific failure envelope on the requested CLI surfaces.

## What This Changes

Request first-class JSON Schema support on `openclaw infer model run` and possibly `openclaw agent`, including validation and defined invalid-output handling.

## Best Possible Solution

Define an additive, provider-neutral JSON Schema contract on `openclaw infer model run` first, using the shared validator, documented validation/retry semantics, structured errors, focused CLI/protocol tests, and an explicit follow-up decision for broader `openclaw agent` exposure.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Static inspection gives high confidence that the requested schema flag, validation, repair/retry, and schema-failure envelope are absent from current main and v2026.5.18.

## Solution Assessment

Partly: first-class JSON Schema validation is the right direction, and starting with `infer model run` is the narrowest maintainable path. The exact retry policy, failure shape, and whether `openclaw agent` should be included need maintainer API/product decisions first.
