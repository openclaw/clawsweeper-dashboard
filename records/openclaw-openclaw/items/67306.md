## Summary

Keep this open. Current main and the latest shipped release still expose JSON output plus adjacent plugin/schema-validator infrastructure, but they do not expose a caller-supplied JSON Schema contract, schema validation, repair/retry policy, or schema-specific machine-readable failure envelope on the requested CLI/Gateway surfaces.

## What This Changes

The issue requests first-class caller-supplied JSON Schema support for OpenClaw inference CLI surfaces, including validation and defined repair or failure behavior.

## Best Possible Solution

Define an additive provider-neutral JSON Schema contract on `openclaw infer model run` first, using the shared validator, documented validation/retry semantics, structured errors, focused CLI/protocol tests, and a separate decision for broader `agent` exposure.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Static inspection gives high-confidence evidence of the gap because current CLI metadata, parser, execution, docs, tests, and Gateway protocol expose JSON output but no caller-supplied JSON Schema contract.

## Solution Assessment

Partly: first-class schema validation is the right direction, and a narrow provider-neutral `infer model run` implementation is the safest first step. The exact retry policy, failure shape, and whether `openclaw agent` should be included need maintainer API/product decisions.
