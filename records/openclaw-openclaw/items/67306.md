## Summary

Keep this open. Current main still has JSON output and an adjacent optional `llm-task` tool, but the requested caller-supplied JSON Schema contract, schema validation, repair/retry policy, and schema-specific failure envelope are still absent from the requested CLI/Gateway surfaces.

## What This Changes

Requests first-class JSON Schema input, validation, and defined invalid-output handling for `openclaw infer model run` and related agent CLI surfaces.

## Best Possible Solution

Define an additive provider-neutral JSON Schema contract on `openclaw infer model run` first, with file/inline schema input, final-output validation, explicit retry/no-retry semantics, structured errors, docs, and focused CLI/protocol tests.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Static inspection gives high-confidence evidence of the gap because the current CLI metadata, parser, execution path, Gateway schema, and docs expose JSON output but no caller-supplied schema contract.

## Solution Assessment

Partly: a first-class schema contract is the right direction, but exposing it across both `agent` and `infer` plus repair semantics needs maintainer API/product decisions. A narrow provider-neutral `infer model run` implementation is the most maintainable first step.
