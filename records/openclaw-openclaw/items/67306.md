## Summary

Keep this open. Current main still exposes JSON output and lean model-run behavior, but it does not provide a caller-supplied JSON Schema contract, schema validation, repair/retry policy, or schema-specific failure envelope on the requested CLI/Gateway surfaces.

## What This Changes

Add a first-class JSON Schema input contract to `openclaw infer model run` and possibly `openclaw agent`, with validation, optional repair/retry semantics, and machine-readable failure output.

## Best Possible Solution

Define a provider-neutral JSON Schema contract on `openclaw infer model run` first, with file/inline schema input, final-output validation, explicit retry/no-retry semantics, structured errors, docs, and focused CLI tests.

## Reproduction Assessment

Not applicable as a feature request. Static inspection gives high-confidence evidence of the gap because the current CLI metadata, parser, execution path, Gateway schema, and docs expose JSON output but no caller-supplied schema contract.

## Solution Assessment

Partly. A provider-neutral `infer model run` contract is the narrowest maintainable first step, while broad `openclaw agent` exposure and provider-native schema modes should wait for explicit API/product decisions.
