## Summary

Keep this open. Current main still lacks caller-supplied JSON Schema support on the requested CLI and Gateway surfaces, and the remaining work is a public feature/API design rather than an already-implemented cleanup item.

## What This Changes

Request first-class JSON Schema input, output validation, repair or retry semantics, and machine-readable schema failures for `openclaw agent` and `openclaw infer model run`.

## Best Possible Solution

Define an additive provider-neutral JSON Schema contract on `openclaw infer model run` first, backed by shared validation, documented retry or no-retry semantics, structured errors, focused CLI tests, and a separate decision for broader `openclaw agent` exposure.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Static inspection of current main gives high-confidence evidence that the requested schema input and validation path are absent from the relevant CLI and Gateway surfaces.

## Solution Assessment

Partly: first-class schema validation is a reasonable direction, but the retry policy, failure shape, provider-native mapping, and whether `openclaw agent` should be included need maintainer API/product decisions before implementation.
