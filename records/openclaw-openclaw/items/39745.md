## What This Changes

The PR stores timeout handles and clears them in finally blocks for Promise.race timeout paths in the Synology Chat, Twitch, and Zalouser plugins.

## Best Possible Solution

Refresh this work so Zalouser keeps the timer cleanup, Twitch is dropped as already implemented, and Synology either clears the timer immediately after the delivery race or is resolved through the configured-timeout fix, with focused regression proof.
