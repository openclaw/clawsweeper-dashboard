## Summary

Keep open. Current main still has the Zalouser and Synology Chat cleanup opportunities, but this cross-repo PR now conflicts with main, repeats a Twitch cleanup already present on main, and still lacks required real after-fix behavior proof.

## What This Changes

This PR updates Synology Chat, Twitch, and Zalouser bundled plugin timeout races to retain timeout handles and clear them in finally blocks.

## Best Possible Solution

Refresh the PR or replace it with a narrow branch that keeps the Zalouser cleanup, leaves Twitch unchanged, resolves the Synology timer scope with the open configured-timeout bug in mind, and adds focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main still has inline timeout promises without `clearTimeout` in Zalouser and Synology Chat, while Twitch already has the cleanup pattern.

## Solution Assessment

Partially. The handle-plus-finally pattern is the right narrow cleanup for Zalouser, but the Synology timer should be cleared immediately after the delivery race or coordinated with https://github.com/openclaw/openclaw/issues/59966, and the stale Twitch hunk should be dropped.
