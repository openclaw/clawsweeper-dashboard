## Summary

## What This Changes

The PR stores timeout handles and clears them in `finally` blocks for Promise.race timeout patterns in the Synology Chat, Twitch, and Zalouser plugins.

## Best Possible Solution

Refresh the branch to keep the Zalouser cleanup, leave Twitch unchanged, clear the Synology timer immediately after the delivery race or coordinate it with the configured-timeout issue, and add focused proof or regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main still has inline Promise.race timeout promises without clearTimeout in Zalouser and Synology Chat, while Twitch already has the cleanup pattern.

## Solution Assessment

Partially. The handle-plus-finally cleanup is the right narrow pattern, but the Synology timer should be cleared as soon as the delivery race settles and the PR should be refreshed because Twitch is already fixed on main.
