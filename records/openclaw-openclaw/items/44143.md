## Summary

Keep this PR open, but it is not merge-ready. The ordering race is plausible and still not implemented on current main, but the proposed patch locks too late, uses a non-canonical route key, lacks real affected-channel proof, and carries an unrelated Mattermost dependency change.

## What This Changes

The PR adds an in-memory outbound delivery serializer for same channel/account/recipient sends, serializer unit tests, and a Mattermost dependency plus lockfile entry.

## Best Possible Solution

Implement serialization at the durable outbound intent boundary using normalized channel/account/recipient/thread route identity, preserve abort and timeout behavior, remove the unrelated dependency, add regression coverage, and collect real-channel proof.

## Reproduction Assessment

No high-confidence live reproduction is established. Source inspection and the author's isolated mock signal-cli run support the concurrent-send ordering race, but this PR still lacks real affected-channel before/after proof.

## Solution Assessment

No. The current PR is not the best fix because it locks after durable queue persistence and derives the mutex key from raw inputs instead of canonical route identity.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Move the ordering lock before queue persistence:** `src/infra/outbound/deliver.ts:526-528`
  - body: The serializer starts only after `enqueueDelivery` has completed. If the earlier send is slower writing the durable queue, a later same-target send can reach the mutex first and deliver first, so FIFO by send initiation is still not guaranteed.
  - confidence: 0.9
- **[P2] Use canonical route identity for the serializer key:** `src/infra/outbound/deliver.ts:523`
  - body: The key is built from raw channel/account/to values and omits `threadId`. Account and target identity are normalized later, and threaded channels route by thread metadata, so equivalent chats can bypass the lock while distinct threads can block each other.
  - confidence: 0.88
