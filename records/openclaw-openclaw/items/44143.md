## Summary

Keep open: the PR targets a plausible outbound ordering bug, but the current patch is still blocked by mock-only proof, lock placement after queue I/O, a raw non-canonical destination key, and an unrelated Mattermost dependency change.

## What This Changes

The branch adds an in-memory outbound DeliverySerializer with tests, wraps core outbound delivery with it, and adds a Mattermost TypeBox dependency plus lockfile entry.

## Best Possible Solution

Move ordering to the durable outbound intent boundary with a canonical adapter-resolved destination key, drop unrelated dependency changes, and verify it with focused regression tests plus one real chat-channel proof run.

## Reproduction Assessment

No, not at high confidence. Source inspection and the author’s mock signal-cli run support the race, but there is still no current-main real Telegram, Discord, Signal, or Mattermost reproduction/proof path.

## Solution Assessment

No. The current PR locks after queue persistence and keys by raw inputs; the narrower maintainable fix is to serialize earlier using canonical account, destination, and thread identity.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Acquire the ordering lock before queue writes:** `src/infra/outbound/deliver.ts:526-528`
  - body: The new lock is entered only around `deliverOutboundPayloadsCore`, but `deliverOutboundPayloadsInternal` has already awaited `enqueueDelivery`. If an earlier same-target send has slower queue I/O, a later send can reach this serializer first and deliver first, so the PR still does not guarantee FIFO by send initiation.
  - confidence: 0.9
- **[P2] Key serialization by the canonical destination:** `src/infra/outbound/deliver.ts:523`
  - body: The mutex key uses raw `accountId` and `to` and omits thread identity. Existing adapters normalize account IDs, Signal strips `signal:`, and Discord routes threaded sends via `threadId`, so equivalent destinations can bypass the lock while unrelated threads can share it.
  - confidence: 0.88
