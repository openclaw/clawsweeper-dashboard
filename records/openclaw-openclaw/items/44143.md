- **[P1] Serialize before durable enqueue:** `src/infra/outbound/deliver.ts:1278`
  - body: The lock is acquired only after `enqueueDelivery(...)` resolves, so two same-target sends can still invert when the later call finishes the write-ahead queue first and reaches the mutex first. Move ordering to the durable intent boundary or otherwise preserve enqueue-order before platform send.
  - confidence: 0.9
- **[P2] Use canonical route identity for the mutex key:** `src/infra/outbound/deliver.ts:1272`
  - body: The key is built from raw `channel`, `accountId`, and `to`, but the delivery path normalizes accounts, canonicalizes targets, and routes threaded destinations through thread/topic identity. Equivalent chats can bypass the lock while distinct threads can be serialized together.
  - confidence: 0.86
- **[P2] Make queued waits abort and timeout aware:** `src/infra/outbound/deliver.ts:1278`
  - body: A send that is canceled while waiting behind an earlier same-target delivery will not observe `abortSignal` until it reaches `deliverOutboundPayloadsCore`; if the process restarts first, the durable entry can replay without the original abort signal. The wait path needs cancellation/timeout handling before merge.
  - confidence: 0.82
