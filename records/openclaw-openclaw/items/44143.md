## Summary

Keep open: the ordering bug is plausible and current main does not implement this serializer, but this PR is not merge-ready because the lock sits after durable queue persistence, uses a raw route key, lacks abort/timeout-aware queue boundaries, adds unrelated Mattermost dependency churn, and only mock proof is posted.

## What This Changes

Adds an in-memory DeliverySerializer around outbound deliveries with focused serializer tests, plus a Mattermost TypeBox dependency and lockfile entry.

## Best Possible Solution

Serialize at the durable outbound intent boundary using canonical channel/account/target/thread identity, with abort/timeout-aware release behavior and no unrelated dependency churn.

## Reproduction Assessment

No high-confidence live reproduction is established. Source inspection and the author's mock signal-cli run support the race, but no redacted real affected-channel before/after proof is posted.

## Solution Assessment

No. The PR serializes too late in the durable delivery flow and keys the lock from raw route strings, so it is not the narrowest maintainable fix for ordered delivery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Move the lock ahead of queue persistence:** `src/infra/outbound/deliver.ts:527-528`
  - body: The serializer is entered only after `enqueueDelivery`. If an earlier same-target send is slower writing its durable queue entry, a later send can reach the mutex first and deliver first, so FIFO by send initiation is still not guaranteed.
  - confidence: 0.9
- **[P2] Derive the key from canonical route identity:** `src/infra/outbound/deliver.ts:523`
  - body: The key uses raw `channel`, `accountId`, and `to` while omitting thread identity. Account/target normalization and thread routing happen later, so equivalent conversations can bypass the lock while unrelated threads can block each other.
  - confidence: 0.88
- **[P2] Preserve abort and timeout boundaries inside the queue:** `src/infra/outbound/deliver.ts:527-528`
  - body: A send waiting behind the serializer is not abort-aware, and the queued operation has no bounded release. A stalled platform send can pin every later same-key delivery until restart, and canceled work may still replay from the durable queue.
  - confidence: 0.78
