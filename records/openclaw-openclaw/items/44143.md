## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Serialize before durable queue I/O:** `src/infra/outbound/deliver.ts:524-526`
  - body: The serializer is entered only after enqueueDelivery has already completed, so two same-target sends can still invert if the earlier call has a slower queue write and the later call reaches the lock first. Move the ordering boundary before durable intent creation or make the durable queue itself preserve per-route FIFO before platform sends.
  - confidence: 0.9
- **[P2] Derive the key from canonical route identity:** `src/infra/outbound/deliver.ts:522`
  - body: The key uses raw accountId and to and omits thread/topic identity, while current adapters normalize account IDs and rewrite targets later. That lets equivalent destinations bypass serialization and can block unrelated Discord threads or Telegram topics together.
  - confidence: 0.86
- **[P2] Make queued waits abort or timeout aware:** `src/infra/outbound/delivery-serializer.ts:37-42`
  - body: A per-recipient chain means one never-settling platform request can wedge all later sends for that key. Current Discord webhook sends use fetch without an AbortSignal or timeout, so the serializer needs a bounded wait/abort strategy before wrapping shared delivery paths.
  - confidence: 0.82
- **[P2] Remove the unrelated Mattermost dependency:** `extensions/mattermost/package.json:7`
  - body: The Mattermost package and lockfile change is unrelated to outbound delivery ordering and adds runtime package surface to this PR. Split it to its own dependency fix or drop it from this branch.
  - confidence: 0.88
