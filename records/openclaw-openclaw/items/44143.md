## Summary

Keep open. Current main does not already implement this outbound ordering fix, but the PR is not merge-ready because it locks after queue persistence, uses a raw non-canonical route key, lacks abort/timeout-aware queue release behavior, has only mock proof, and includes unrelated Mattermost dependency churn.

## What This Changes

The branch adds an in-memory outbound DeliverySerializer around deliverOutboundPayloadsCore, serializer tests, and a Mattermost TypeBox dependency/lockfile change.

## Best Possible Solution

Serialize at the durable outbound intent boundary using canonical channel/account/target/thread identity, with abort/timeout-aware release behavior and no unrelated dependency churn.

## Reproduction Assessment

No high-confidence live reproduction is established. Source inspection and the author's mock signal-cli run support the race, but no redacted real affected-channel before/after proof is posted.

## Solution Assessment

No. The proposed serializer is placed after durable queue persistence and uses a raw route key, so the safer fix is to order at the durable intent boundary with canonical route identity and bounded release behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Move the lock ahead of queue persistence:** `src/infra/outbound/deliver.ts:526-528`
  - body: The serializer is entered only after `enqueueDelivery`. If the earlier same-target send is slower persisting its durable entry, a later send can acquire the mutex first and still deliver first, so FIFO by send initiation is not guaranteed.
  - confidence: 0.9
- **[P2] Derive the key from canonical route identity:** `src/infra/outbound/deliver.ts:523`
  - body: The key uses raw channel/account/to and omits thread identity. Accounts and targets can be normalized later and threaded channels route by thread, so equivalent chats can bypass the lock while unrelated threads block each other.
  - confidence: 0.88
- **[P2] Preserve abort and timeout boundaries inside the queue:** `src/infra/outbound/deliver.ts:526-528`
  - body: A send waiting behind the serializer is not abort-aware, and the queued operation has no bounded release. A stalled platform send can pin every later same-key delivery until restart.
  - confidence: 0.78
