## Summary

Keep open, but the current patch should not merge as written: current main still lacks outbound delivery serialization, yet this diff locks after durable queue writes and keys on raw destination inputs, so it does not reliably preserve same-recipient send order.

## What This Changes

The PR adds an in-memory DeliverySerializer around outbound core sends, serializer unit tests, and a Mattermost TypeBox dependency plus lockfile entry.

## Best Possible Solution

Implement serialization at the durable outbound intent boundary using a canonical adapter-resolved key that includes account, recipient, and thread identity, while preserving abort/timeout behavior and removing unrelated dependency changes.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection and the author's mock signal-cli run support the race, but no current-main Telegram, Discord, Signal, or Mattermost reproduction/proof path was provided.

## Solution Assessment

No. The proposed lock is placed too late and uses raw destination inputs; the safer fix is to serialize before durable queue ordering can invert, using canonical destination identity.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Acquire the ordering lock before queue writes:** `src/infra/outbound/deliver.ts:526-528`
  - body: The new lock is entered only around `deliverOutboundPayloadsCore`, after `enqueueDelivery` has already completed. If an earlier same-target send has slower queue I/O, a later send can reach the serializer first and deliver first, so this does not guarantee FIFO by send initiation.
  - confidence: 0.9
- **[P2] Build the mutex key from the canonical destination:** `src/infra/outbound/deliver.ts:523`
  - body: The mutex key uses raw `accountId` and `to` and omits thread identity. Account IDs and adapter targets are normalized later, so equivalent conversations can bypass the lock while distinct threaded conversations can be serialized together.
  - confidence: 0.88
