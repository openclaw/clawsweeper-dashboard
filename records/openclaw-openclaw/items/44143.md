## Summary

Keep open: the ordering bug is real and current main still lacks a generic per-destination outbound serializer, but this patch enters the lane after queue/claim work and keys it from raw destination fields, so it is not merge-ready yet.

## What This Changes

The PR adds an in-memory DeliverySerializer around outbound payload delivery plus unit tests for same-key ordering, cross-target concurrency, errors, synchronous throws, and cleanup.

## Best Possible Solution

Wrap the durable queue, active claim, platform send, and cleanup in one bounded per-destination lane keyed from canonical account, target, and thread/topic identity.

## Reproduction Assessment

Yes at source/proof level, but not by a fresh local current-main run. Mantis Telegram Desktop proof shows the visible ordering problem, and current main still has no generic outbound delivery serializer.

## Solution Assessment

No as submitted. The PR is a plausible mitigation, but the best fix must serialize before queue persistence and claims, use canonical destination facts, and bound abort or stalled-send behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Acquire the delivery lane before queue work can reorder sends:** `src/infra/outbound/deliver.ts:1278-1279`
  - body: The serializer is entered only after `enqueueDelivery` and `withActiveDeliveryClaim` have awaited. Two same-target sends can therefore reach the lock in queue-write or claim-completion order instead of caller order, so the later reply can still deliver first.
  - confidence: 0.93
- **[P1] Key delivery lanes from canonical channel targets:** `src/infra/outbound/deliver.ts:1274`
  - body: `serializerKey` uses raw `accountId` and `to` and omits thread/topic identity, while adapters normalize accounts and destinations later. Equivalent destinations can bypass the serializer, and distinct threads can block each other.
  - confidence: 0.9
- **[P2] Make queued lanes abort- and stall-aware:** `src/infra/outbound/deliver.ts:1278-1279`
  - body: Every later same-key delivery now waits for the adapter promise, but the lane has no abort or timeout/release path. A canceled or never-settling send can wedge later replies to that conversation until restart.
  - confidence: 0.82
