## Summary

Keep open. Current main still appends Codex ACP `-c model=...` and `-c model_reasoning_effort=...` overrides without checking whether the configured Codex ACP command already contains those keys, and the latest shipped tag has the same helper. Open PR #74339 explicitly references this issue as its fix, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that ACPX Codex worker runs fail with `AcpRuntimeError: Internal error` when the configured `codex-acp` command includes model and reasoning-effort overrides.

## Best Possible Solution

Land a narrow ACPX Codex command-normalization fix that preserves configured adapter args while skipping or validating runtime `model` and `model_reasoning_effort` keys already present in the command, with tests for full overlap, partial overlap, wrapper commands, and the reported `sessions_spawn` path.

## Reproduction Assessment

Yes. The issue gives concrete operator steps and current main statically reproduces the risky path: configured Codex ACP args are preserved, then `sessions_spawn` model/thinking overrides can append the same `-c` keys before startup.

## Solution Assessment

Yes. The narrow maintainable solution belongs in the ACPX Codex command-normalization/runtime override path; `sessions_spawn` is correctly forwarding generic ACP runtime options.
