## What This Changes

The issue reports that ACPX Codex worker runs fail when a configured Codex ACP command and `sessions_spawn` model/thinking overrides both supply model and reasoning settings.

## Best Possible Solution

Review and land PR #74339 or an equivalent narrow ACPX fix that preserves configured Codex ACP adapter args while skipping or validating duplicate `model` and `model_reasoning_effort` startup keys, with regression tests for full overlap, partial overlap, wrapper commands, and the reported `sessions_spawn` path.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the ACPX Codex command-normalization/runtime override path; `sessions_spawn` is correctly forwarding generic ACP runtime options.
