## Summary

Keep open. Current main still blindly appends Codex ACP `-c model=...` and `-c model_reasoning_effort=...` startup args without checking whether the configured command already contains those keys, and open PR #74339 references this issue with closing syntax, so this should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that ACPX Codex worker runs fail with `AcpRuntimeError: Internal error` when Codex model and reasoning overrides are configured in the adapter command.

## Best Possible Solution

Land PR #74339 or an equivalent ACPX Codex normalization change that preserves configured command args, appends only missing runtime model/thinking keys, and covers full overlap, partial overlap, wrapper commands, and the reported `sessions_spawn` path.

## Reproduction Assessment

Yes. The issue provides concrete operator steps and current main statically reproduces the risky construction path: configured Codex ACP `-c` args are preserved while runtime `model` and `thinking` can append the same keys again.

## Solution Assessment

Yes. The narrow maintainable fix belongs in ACPX Codex startup-command normalization; generic `sessions_spawn` should continue forwarding ACP runtime model/thinking options to adapters that support them.
