## Summary

Keep open. Current main still has the duplicate Codex ACP startup-config append path, and the open fix PR references this issue with closing syntax, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Fix ACPX Codex worker crashes when configured Codex ACP model/thinking `-c` overrides combine with runtime model/thinking overrides.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/74339 with extension-local Codex command normalization that avoids duplicate same-key `-c` entries while preserving explicit runtime override semantics and regression coverage.

## Reproduction Assessment

Yes, by source inspection rather than a live ACP run. Current main forwards `sessions_spawn.model` and `thinking` into ACPX Codex startup config while preserving configured Codex `-c` args, so the same keys can be duplicated on the launched command.

## Solution Assessment

Yes, the narrow maintainable fix is in ACPX Codex command normalization. Generic `sessions_spawn` forwarding is documented behavior and should remain in place.
