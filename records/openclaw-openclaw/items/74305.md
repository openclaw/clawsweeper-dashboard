Keep open. Current main still has the append-only ACPX Codex startup-config path, and the live fixing PR that references this issue with closing syntax remains open, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports ACPX Codex worker runs failing with `AcpRuntimeError: Internal error` when configured Codex `-c model` and `-c model_reasoning_effort` args are used with runtime model/thinking overrides.

## Best Possible Solution

Keep the issue open until an ACPX-local normalization fix lands that removes or replaces duplicate same-key Codex `-c` entries while preserving explicit runtime override semantics and regression coverage.

## Reproduction Assessment

Yes, by source inspection rather than a live ACP run. Current main appends runtime Codex model/thinking startup config to a command that can already contain the same configured `-c` keys, matching the reporter's confirmed workaround.

## Solution Assessment

Yes for the direction, but the open PR still needs review or repair because earlier review found it can discard conflicting explicit runtime overrides. The maintainable fix belongs in ACPX Codex command normalization, not in removing generic `sessions_spawn` forwarding.
