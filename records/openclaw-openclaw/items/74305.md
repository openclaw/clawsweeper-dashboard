## Summary

Keep open. Current main still appends Codex ACP `-c model=...` and `-c model_reasoning_effort=...` runtime overrides without detecting keys already present in the configured command, and open PR #74339 explicitly references this issue as the focused fix candidate. The issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that ACPX Codex worker runs fail when Codex ACP commands already include model and reasoning overrides and `sessions_spawn` forwards matching runtime overrides.

## Best Possible Solution

Land a narrow ACPX Codex command-normalization fix that preserves configured adapter args while skipping or validating runtime `model` and `model_reasoning_effort` keys already present in the command, with coverage for full overlap, partial overlap, wrapper commands, and the reported `sessions_spawn` path.

## Reproduction Assessment

Yes. The issue gives concrete operator steps, and current main statically reproduces the risky path: configured Codex ACP args are preserved, then `sessions_spawn` model/thinking overrides can append the same `-c` keys before startup.

## Solution Assessment

Yes. The narrow maintainable fix belongs in ACPX Codex command normalization; `sessions_spawn` is correctly forwarding generic ACP runtime options, and open PR #74339 is already the focused implementation candidate.
