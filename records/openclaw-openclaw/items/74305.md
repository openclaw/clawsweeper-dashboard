## Summary

Keep open. Current main still preserves configured Codex ACP adapter args and then appends runtime `model`/`thinking` values as additional `-c` config args without checking whether those keys are already present; open PR #74339 explicitly references this issue with closing syntax and is the focused implementation candidate.

## What This Changes

The report requests that ACPX Codex worker spawns tolerate or clearly validate configured Codex `model` and `model_reasoning_effort` overrides instead of failing with `AcpRuntimeError: Internal error`.

## Best Possible Solution

Land PR #74339 or an equivalent ACPX Codex command-normalization fix that detects existing `-c model` and `-c model_reasoning_effort` keys in configured commands, skips or validates duplicate runtime overrides, and keeps generic `sessions_spawn` forwarding unchanged. Coverage should include full overlap, partial overlap, wrapper commands, and the reported ACP spawn path.

## Reproduction Assessment

Yes. The reporter provided concrete steps and current main statically reproduces the risky construction path: configured Codex ACP args are preserved, then `sessions_spawn` model/thinking overrides can append the same `-c` keys before startup.

## Solution Assessment

Yes. The narrow maintainable solution belongs in ACPX Codex command normalization; `sessions_spawn` is correctly forwarding generic ACP runtime options, and the linked PR is already the focused implementation candidate.
