## Summary

Keep open. Current main still preserves configured Codex ACP `-c` args and appends runtime model/thinking overrides without checking for pre-existing config keys, and open PR #74339 already references this issue as its closing fix candidate.

## What This Changes

The issue reports ACPX Codex worker failures when configured Codex `-c model` and `-c model_reasoning_effort` overrides are combined with `sessions_spawn` model/thinking overrides.

## Best Possible Solution

Review and land PR #74339, or an equivalent ACPX normalization change that detects existing Codex `-c` keys, skips duplicates, appends only missing overrides, and covers configured, partial, wrapper-command, and `sessions_spawn` cases.

## Reproduction Assessment

Yes. The issue gives concrete 2026.4.26 steps and logs, and current main statically reproduces the duplicate-prone path by forwarding `sessions_spawn` model/thinking into ACPX Codex startup config that appends `-c` overrides regardless of pre-existing keys.

## Solution Assessment

Yes. The narrow maintainable fix belongs in ACPX Codex startup-command normalization; generic `sessions_spawn` should continue forwarding ACP runtime model/thinking options to adapters that support them.
