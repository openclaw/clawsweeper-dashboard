## Summary

Keep open. Current main still has the ACPX Codex duplicate-config construction path, and open PR #74339 is already the focused fix candidate with closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports ACPX Codex worker failures when configured Codex `-c model` and `-c model_reasoning_effort` startup overrides overlap with runtime model/thinking overrides from `sessions_spawn`.

## Best Possible Solution

Review and validate PR #74339, or land an equivalent ACPX Codex normalization change that detects pre-existing Codex `-c` config keys, skips duplicates, appends only missing overrides, and covers full overlap, partial overlap, wrapper commands, and the reported `sessions_spawn` path.

## Reproduction Assessment

Yes. The issue provides concrete 2026.4.26 operator steps, and current main statically reproduces the risky path: configured Codex ACP `-c` args are preserved while `sessions_spawn` model/thinking can append the same keys again before startup.

## Solution Assessment

Yes. The narrow maintainable fix belongs in ACPX Codex startup-command normalization; generic `sessions_spawn` should continue forwarding ACP runtime model/thinking options to adapters that support them.
