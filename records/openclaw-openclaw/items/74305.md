## Summary

Keep open. Current main still has the duplicate-prone ACPX Codex startup path: configured Codex ACP command args are preserved, and `sessions_spawn` model/thinking overrides can still append another `-c model=...` and `-c model_reasoning_effort=...`. Open PR #74339 explicitly references this issue as the focused fix candidate, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Ensure ACPX Codex worker runs either succeed with configured Codex model/reasoning overrides or fail with a clear validation error instead of `AcpRuntimeError: Internal error`.

## Best Possible Solution

Land or reject the focused fix PR after validation. The maintainable end state is ACPX Codex command normalization that preserves configured adapter args while skipping duplicate `model` and `model_reasoning_effort` appends, with coverage for full overlap, partial overlap, wrapper commands, and the reported `sessions_spawn` path.

## Reproduction Assessment

Yes. The issue gives concrete 2026.4.26 operator steps, and current main statically reproduces the risky path: configured Codex ACP `-c` args can be preserved, then `sessions_spawn` model/thinking overrides can append the same keys before startup.

## Solution Assessment

Yes. The narrow fix belongs in ACPX Codex startup-command normalization; `sessions_spawn` should continue forwarding generic ACP runtime model/thinking options, and PR #74339 is already the focused candidate.
