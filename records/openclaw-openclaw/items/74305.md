## Summary

Keep open. Current main still has the duplicate-prone ACPX Codex startup path, and open PR #74339 already references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that Codex ACP worker runs fail when configured `-c model=...` and `-c model_reasoning_effort=...` overrides are present alongside runtime model/thinking overrides.

## Best Possible Solution

Review and land PR #74339, or an equivalent ACPX Codex normalization change that detects existing Codex `-c` config keys, skips duplicates, appends only missing overrides, and covers full overlap, partial overlap, wrapper commands, and the reported `sessions_spawn` path.

## Reproduction Assessment

Yes. The issue provides concrete 2026.4.26 operator steps, and current main statically reproduces the risky path: configured Codex ACP `-c` args are preserved while `sessions_spawn` model/thinking can append the same keys again before startup.

## Solution Assessment

Yes. The narrow maintainable fix belongs in ACPX Codex startup-command normalization; generic `sessions_spawn` should continue forwarding ACP runtime model/thinking options to adapters that support them.
