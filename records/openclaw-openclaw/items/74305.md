## What This Changes

The issue reports that ACPX Codex workers fail when Codex ACP is configured with model and reasoning-effort overrides.

## Best Possible Solution

Review and land PR #74339, or an equivalent ACPX Codex normalization fix that preserves configured command args and appends only missing runtime model/thinking keys with overlap tests.

## Reproduction Assessment

Yes. The issue gives concrete operator steps, and current main statically reproduces the risky command construction path: configured Codex ACP `-c` args are preserved while runtime model/thinking overrides can append the same keys again.

## Solution Assessment

Yes. The narrow maintainable solution belongs in ACPX Codex startup-command normalization; generic `sessions_spawn` should continue forwarding ACP runtime model/thinking options to adapters that support them.
