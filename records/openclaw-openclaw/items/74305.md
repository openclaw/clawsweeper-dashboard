## Summary

Keep open. Current main still has the duplicate Codex ACP config append path, and PR #74339 is still the open closing implementation candidate, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Fix ACPX Codex worker startup so configured codex-acp model/thinking overrides do not fail when runtime model/thinking overrides are also passed.

## Best Possible Solution

Land PR #74339 or an equivalent ACPX Codex command-normalization fix that skips already-configured `model` and `model_reasoning_effort` keys while preserving configured adapter args.

## Reproduction Assessment

Yes. The issue gives concrete operator steps, and current main statically reproduces the risky construction path by preserving configured Codex ACP `-c` args while appending runtime model/thinking keys again.

## Solution Assessment

Yes. The narrow maintainable fix belongs in ACPX Codex startup-command normalization; generic `sessions_spawn` should continue forwarding model/thinking options to ACP runtimes that support them.
