## Summary

Keep open. Current main still has the duplicate Codex ACP startup-config append path, and open PR #74339 is the linked implementation candidate rather than a reason to close before merge.

## What This Changes

The report asks OpenClaw to let ACPX Codex workers run with supported model/thinking overrides or fail with a specific validation error instead of `AcpRuntimeError: Internal error`.

## Best Possible Solution

Review and land PR #74339, or an equivalent ACPX Codex command-normalization fix that appends only missing `-c` keys and covers configured command, partial-overlap, wrapper-command, and `sessions_spawn` paths.

## Reproduction Assessment

Yes, by source inspection. The issue provides concrete operator steps, and current main still preserves configured Codex ACP `-c` args while runtime model/thinking can append the same keys again.

## Solution Assessment

Yes. The narrow maintainable solution is in ACPX Codex startup-command normalization; generic `sessions_spawn` should continue forwarding ACP runtime model/thinking options to adapters that support them.
