## Summary

Keep open. Current main still appends Codex ACP model/reasoning `-c` args without detecting keys already present in the configured command, and open PR #74339 references this issue with closing syntax, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Ensure ACPX Codex worker runs support, or clearly reject, model/thinking overrides when operator-configured `codex-acp -c` overrides are already present.

## Best Possible Solution

Review and land PR #74339, or an equivalent ACPX Codex command-normalization fix that skips already configured `model` and `model_reasoning_effort` keys while appending only missing runtime overrides.

## Reproduction Assessment

Yes. The issue provides concrete 2026.4.26 steps and logs, and current main statically preserves the same risky construction path where configured Codex ACP `-c` args can be duplicated by runtime model/thinking overrides.

## Solution Assessment

Yes. The narrow maintainable fix belongs in ACPX Codex startup-command normalization; generic `sessions_spawn` should continue forwarding ACP runtime model/thinking options to adapters that support them.
