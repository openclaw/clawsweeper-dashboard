## Summary

Keep open. Current main still has the duplicate-prone ACPX/Codex startup path, and the linked open PR is the active implementation candidate that should be reviewed or landed rather than closing this issue early.

## What This Changes

The issue asks OpenClaw to let ACPX Codex workers run when configured with Codex model/reasoning overrides, or fail with a specific validation error instead of a generic ACP runtime error.

## Best Possible Solution

Review and land PR #74339, or an equivalent narrow ACPX fix, so configured Codex ACP adapter args are preserved while runtime model and reasoning overrides skip or validate already-present `-c` config keys.

## Reproduction Assessment

Yes. The issue gives concrete 2026.4.26 operator steps, and current main statically reproduces the risky construction path: configured Codex ACP args are preserved and runtime model/thinking overrides can append the same `-c` keys before startup.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the ACPX Codex command-normalization/runtime override path; `sessions_spawn` is correctly forwarding generic ACP runtime options and should not learn Codex-specific command-argument policy.
