## Summary

Keep open. Current main still has the duplicate Codex ACP startup-config append path, and the open fix PR #74339 references this issue with closing syntax, so this should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw ACPX Codex spawns to work, or fail with a specific validation error, when configured Codex ACP model and reasoning overrides are present.

## Best Possible Solution

Land PR #74339 or an equivalent ACPX command-normalization fix that skips already configured Codex -c keys while still appending missing runtime overrides.

## Reproduction Assessment

Yes, by source inspection. A configured Codex ACP command that already contains -c model and -c model_reasoning_effort can receive the same keys again when sessions_spawn passes model/thinking into the Codex ACP override scope.

## Solution Assessment

Yes. The narrow maintainable fix belongs in ACPX Codex startup-command normalization; generic sessions_spawn forwarding should remain because the docs define model/thinking overrides for ACP runtimes.
