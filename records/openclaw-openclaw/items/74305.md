## Summary

Keep open. Current main still appends Codex ACP model/thinking startup config without checking for configured `-c` keys, and open PR #74339 references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that ACPX Codex workers fail with `AcpRuntimeError: Internal error` when configured `codex-acp` model/thinking args overlap with runtime model/thinking overrides.

## Best Possible Solution

Review and land PR #74339, or an equivalent ACPX Codex command-normalization fix that appends only missing runtime config keys and adds duplicate-key regression coverage.

## Reproduction Assessment

Yes, by source inspection. The reported configured command args are preserved, `sessions_spawn` forwards model/thinking, and current ACPX runtime appends the same Codex ACP `-c` keys again without deduplication.

## Solution Assessment

Yes. The narrowest maintainable solution is to dedupe Codex ACP startup config keys in ACPX command normalization while leaving generic ACP `sessions_spawn` model/thinking forwarding intact.
