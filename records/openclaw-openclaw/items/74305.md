## Summary

Keep open. Current main still appends Codex ACP model/thinking startup config without detecting pre-existing `-c` keys, and open PR #74339 is already the focused closing fix candidate, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw ACPX Codex spawns to support configured Codex model/thinking overrides without failing, or to return a specific validation error when unsupported.

## Best Possible Solution

Review and validate PR #74339, or land an equivalent ACPX Codex normalization change that skips already-configured Codex `-c` keys, appends only missing overrides, and covers full overlap, partial overlap, wrapper commands, and the reported `sessions_spawn` path.

## Reproduction Assessment

Yes. The issue provides concrete 2026.4.26 operator steps, and current main statically reproduces the risky path: configured Codex ACP `-c` args are preserved while `sessions_spawn` model/thinking can append the same keys again before startup.

## Solution Assessment

Yes. The narrow maintainable fix belongs in ACPX Codex startup-command normalization; generic `sessions_spawn` should continue forwarding ACP runtime model/thinking options to adapters that support them.
