## Summary

Keep open. Current main still has the ACPX Codex command path that appends runtime `model` and `model_reasoning_effort` config without checking for the same `-c` keys already present in the configured command, and open PR #74339 explicitly tracks the narrow fix for this issue.

## What This Changes

The issue reports that ACPX Codex worker runs fail when the configured Codex ACP command already includes model/thinking overrides and `sessions_spawn` also supplies model/thinking runtime overrides.

## Best Possible Solution

Land a narrow ACPX Codex command-normalization fix that preserves configured adapter args while skipping or validating runtime `model` and `model_reasoning_effort` keys already present in the command, with focused regression tests for full overlap, partial overlap, wrapper commands, and the reported spawn path.

## Reproduction Assessment

Yes. The issue gives concrete operator steps, and current main statically reproduces the duplicate-prone path from `sessions_spawn` model/thinking input through ACP runtime startup into Codex ACP command construction.

## Solution Assessment

Yes. The narrow maintainable fix belongs in ACPX Codex command normalization; `sessions_spawn` should continue forwarding generic ACP runtime options rather than learning Codex-specific command parsing.
