## Summary

Keep open. Current main still preserves configured Codex ACP command arguments and then blindly appends runtime `model` and `model_reasoning_effort` overrides for Codex ACP startup, so the reported duplicate-override failure path remains plausible. Open PR #74339 explicitly closes this issue and is the focused implementation candidate, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to make ACPX Codex worker runs handle configured Codex model and reasoning overrides without failing with an internal ACP runtime error.

## Best Possible Solution

Keep this issue open until PR #74339 or an equivalent ACPX fix lands. The desired end state is Codex ACP command normalization that preserves configured adapter args while skipping duplicate `model` and `model_reasoning_effort` startup config keys, with regression coverage for full overlap, partial overlap, wrapper commands, and the `sessions_spawn` path.
