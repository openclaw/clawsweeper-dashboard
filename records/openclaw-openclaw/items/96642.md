## What This Changes

This PR wraps Telegram bot rich-message reply delivery with a plain-text `sendMessage` fallback for selected `RICH_MESSAGE_*_INVALID` errors and adds focused delivery tests.

## Best Possible Solution

If maintainers still want a generic rich-entity safety net, make it a clearly scoped Telegram rich-send fallback that complements the current builder sanitizers, covers the intended sibling surfaces, and has live Telegram proof.
