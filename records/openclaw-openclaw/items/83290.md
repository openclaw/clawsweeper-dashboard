## Summary

Keep open: current main still has the reported source-level mismatch, but applying channel plans during startup would change auth/pairing migration semantics that docs and tests currently keep in the doctor migration path, so this needs maintainer product judgment before a fix is queued.

## What This Changes

The issue asks for `autoMigrateLegacyState` to apply detected channel legacy-state migration plans and include their changes and warnings in the returned result.

## Best Possible Solution

Make the startup migration contract explicit: either keep channel credential migrations doctor-only and avoid collecting them in the auto helper, or intentionally add channel plan execution to `autoMigrateLegacyState` with matching docs and regression coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source inspection: a channel-only detection sets `detected.channelPlans.hasLegacy`, but `autoMigrateLegacyState` only gates on sessions and agent dir and only merges those results. I did not run a filesystem repro because this review must keep the checkout read-only.

## Solution Assessment

Is this the best way to solve the issue? Unclear: adding channel plans to the auto path is mechanically narrow, but it changes startup behavior for auth and pairing state that current docs/tests reserve for doctor, so maintainers should decide the contract first.
