## Summary

Keep open: current main has the source-level channel-only early return described, but automatically running channel credential/pairing migrations at startup would change the documented doctor-owned migration boundary and needs maintainer product judgment first.

## What This Changes

Requests that startup auto legacy-state migration include detected channel legacy migration plans and merge their changes and warnings into the returned result.

## Best Possible Solution

Align the migration contract explicitly, then either keep channel credential migrations doctor-only and avoid collecting them in the startup helper, or update docs/tests and `autoMigrateLegacyState` to run channel plans with focused regression coverage.

## Reproduction Assessment

Yes from source inspection: with `channelPlans.hasLegacy` true and no legacy sessions or agent dir, `autoMigrateLegacyState` returns before any channel plan runner is called. I did not execute a filesystem repro because this review is read-only.

## Solution Assessment

Unclear: including channel plans in the auto path is mechanically narrow, but it changes the documented doctor-only boundary for channel auth and pairing migrations. A maintainer should decide that contract before a fix is queued.
