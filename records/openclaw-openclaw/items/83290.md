- `impact:session-state`: The report concerns legacy state migration and whether detected on-disk state is left unmigrated.
- `impact:auth-provider`: The skipped channel plans include auth or pairing state such as WhatsApp credentials and Telegram allowFrom files.

## Summary

Keep open: current main still has the source-level path the report describes, but the suggested startup auto-migration would change the documented boundary for channel auth and pairing state, so maintainer product/owner judgment is needed first.

## What This Changes

The issue asks `autoMigrateLegacyState` to apply detected bundled channel legacy migration plans instead of returning no changes when only channel plans exist.

## Best Possible Solution

Make the migration contract explicit: either keep channel auth and pairing migrations doctor-only and avoid collecting them in the startup helper, or update docs/tests and `autoMigrateLegacyState` to run channel plans deliberately.

## Reproduction Assessment

Yes from source inspection: with `channelPlans.hasLegacy` true and no legacy sessions or agent dir, `autoMigrateLegacyState` returns before any channel plan runner is called. I did not execute a filesystem repro because this review is read-only.

## Solution Assessment

Unclear: including channel plans in the auto path is mechanically narrow, but it changes the documented doctor-only boundary for channel auth and pairing migrations, so maintainers should decide the intended contract first.
