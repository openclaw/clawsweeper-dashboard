## Summary

Current main still constructs ordinary OpenAI Responses requests with the system prompt inside `input`, and multiple open PRs explicitly reference this issue. Keep the issue open until maintainers select, validate, and merge one provider-compatible fix.

## What This Changes

Move OpenAI Responses system prompts from input messages to the top-level `instructions` request field.

## Best Possible Solution

Use one canonical Responses request-building policy that sends native OpenAI instructions through `instructions`, explicitly defines compatible-provider and Azure behavior, and adds regression coverage for every affected route.

## Reproduction Assessment

Yes at source level. Current main deterministically inserts the system prompt into converted input for ordinary `openai-responses` requests and does not populate top-level `instructions`; no live API call was performed during this read-only review.

## Solution Assessment

Unclear. Moving native OpenAI prompts to `instructions` is contract-aligned, but the best repository-wide fix requires a maintainer decision about Azure, legacy transport, and generic OpenAI-compatible endpoint behavior.
