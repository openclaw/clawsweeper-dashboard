## Summary

Keep open: current main still has the reported SDK/package boundary, and open fix PRs already target this issue with closing syntax. The issue should stay open until one of the linked fixes lands and package-level proof confirms the installed subpath works.

## What This Changes

The issue reports that `openclaw/plugin-sdk/zod` can publish a bare runtime `zod` re-export that fails under pnpm global installs and breaks plugin registration when `zod` is not resolvable from the OpenClaw package root.

## Best Possible Solution

Land one narrow fix that makes `openclaw/plugin-sdk/zod` self-contained or otherwise resolvable from the installed package, then add package-artifact smoke coverage before closing this issue.

## Reproduction Assessment

Yes, from source and package-path inspection, though not from a live pnpm global install in this review. Current main still exports `./plugin-sdk/zod`, the source is a bare `export * from "zod"`, and the build predicate does not always-bundle `zod`.

## Solution Assessment

Yes, the best direction is to fix the SDK subpath artifact by bundling or shimming `zod`; postinstall or doctor repair would only patch some installations after publishing a fragile runtime export.
