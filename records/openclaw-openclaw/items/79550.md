## Summary

Keep open. This is a scoped bug-fix PR for behavior that is still missing on current main; the patch gates ACP runtime metadata on both persisted `entry.acp` metadata and ACP-shaped session keys, preserves bridge-session behavior, and I found no blocking functional or security finding. Exact-head CI still has a plugin-contract shard failure outside the touched session/runtime files, and the real proof is seeded CLI proof rather than a live Gateway/ACP provider run.

## What This Changes

The PR adds an ACP runtime metadata overlay, threads ACP session metadata through sessions/status/Gateway emit paths, broadens the runtime source type, adds focused ACP runtime metadata tests, and adds a changelog entry.

## Best Possible Solution

Land the scoped overlay only for ACP-keyed rows with persisted ACP metadata, preserve ACP-shaped bridge rows, and report the stored backend id when present.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main computes `entry.acp` in the sessions command but the runtime resolver has no ACP metadata input, so ACP control-plane rows fall through to implicit model-runtime metadata. The PR body also includes pre-fix Docker output showing ACP rows reporting `auto/implicit`.

## Solution Assessment

Yes. The PR keeps the existing model-runtime policy path intact and adds the narrow ACP overlay at the resolver boundary, gated by both session key shape and persisted session metadata, which is safer than key-only classification or reverting the runtime resolver stub.
