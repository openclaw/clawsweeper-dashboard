## Summary

Keep this PR open: it addresses a valid sessions metadata bug and the updated PR body now includes after-fix real CLI proof, but the patch needs one correctness fix because the runtime overlay keys only on ACP-shaped session keys and would misclassify ACP bridge sessions that intentionally lack persisted ACP metadata.

## What This Changes

The branch adds an ACP runtime metadata overlay for session-keyed rows, threads stored ACP backend IDs through sessions/status/Gateway emitters, widens runtime source types, adds regression tests, and updates the changelog.

## Best Possible Solution

Keep the focused runtime metadata fix, but make the overlay depend on persisted ACP runtime metadata as well as key shape and cover the bridge-session control before merging.

## Reproduction Assessment

Yes. Source inspection shows ACP-keyed session rows reach resolveModelAgentRuntimeMetadata on current main without ACP classification, and the PR body includes before-fix live output plus after-fix CLI proof for the intended metadata surface.

## Solution Assessment

No. The resolver-level overlay is close, but key-shape-only classification is not the best final shape because current main documents ACP bridge sessions that use ACP-shaped keys without SessionAcpMeta; the maintainable fix should carry an explicit ACP runtime metadata signal from the session entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Gate ACP runtime overlay on persisted ACP metadata:** `src/agents/acp-runtime-overlay.ts:38-40`
  - body: Current main already distinguishes ACP control-plane sessions from ACP bridge sessions: bridge rows can use `agent:<id>:acp:*` keys without `entry.acp` and should keep normal configured runtime/model metadata. This overlay fires on key shape alone and falls back to `acpx` when no backend is passed, so those bridge rows would be misreported as `acpx`/`session-key`. Thread an explicit `entry.acp != null`/`acpRuntime` signal into the overlay and add the matching bridge-session runtime regression.
  - confidence: 0.91
