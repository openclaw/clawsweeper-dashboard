## Summary

Keep open. Current main still has a finite Slack Socket Mode reconnect cap and no Slack config/docs override, while the issue is paired with open closing PR #73162 and recent discussion still points to Slack SDK pong-stall and reconnect-window loss modes that need validation before cleanup.

## What This Changes

The issue asks OpenClaw to stop silently losing Slack Socket Mode delivery after a previously working Slack connection becomes non-responsive over time.

## Best Possible Solution

Keep this issue open until #73162 or a replacement Slack-owned fix lands with coverage for repeated recoverable Socket Mode disconnect/start failures. Then validate the Slack SDK pong-stall behavior and split any remaining reconnect-window message loss into a narrower follow-up.
