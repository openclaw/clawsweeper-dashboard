## Summary

Keep open. Current main and the latest release still cap Slack Socket Mode reconnects at twelve attempts, and open PR #73162 remains the linked implementation candidate for this issue.

## What This Changes

The issue reports Slack silently stopping replies after several days while the OpenClaw web interface continues to work.

## Best Possible Solution

Land #73162 or an equivalent Slack plugin fix that makes recoverable Socket Mode retries unlimited while preserving fail-fast auth errors, then validate transient not-ready delivery loss as a follow-up if it still reproduces.

## Reproduction Assessment

Yes. Source inspection on current main shows a deterministic failure path after twelve consecutive Slack Socket Mode disconnect/start failures, and the discussion includes current-main reproduction notes plus matching live logs.

## Solution Assessment

Yes for the permanent dead-Slack state. Removing the recoverable reconnect cap is the narrowest maintainable fix; broader transient delivery loss during reconnect windows should remain separate validation or follow-up work.
