## Summary

Keep open. Current main has the Slack SDK pong/reconnect update that the reporter confirmed made Slack work after a restart, but the OpenClaw-owned Socket Mode retry loop still has a hardcoded twelve-attempt cap that can produce the permanent silent-dead state described here.

## What This Changes

The issue reports Slack Socket Mode silently stopping delivery after a long-running connection while the gateway and web UI continue to work.

## Best Possible Solution

The Slack plugin should retry recoverable Socket Mode disconnects indefinitely while continuing to fail fast for non-recoverable auth errors, with separate follow-up only if transient reconnect-window delivery loss still reproduces.

## Reproduction Assessment

Yes. Source inspection on current main shows Slack Socket Mode still throws after twelve reconnect attempts, and the issue discussion includes matching current-main analysis plus live logs for the silent-dead symptom.

## Solution Assessment

Yes for the permanent dead-Slack state. Removing the recoverable reconnect cap in the Slack plugin is the narrowest maintainable fix; the upstream SDK update helps one trigger but does not remove OpenClaw's own cap.
