## Summary

Keep open: current main and v2026.5.7 still cap Slack Socket Mode reconnect/start retries at 12, the reporter posted a recurrence after the Slack SDK update, and the linked open fix PR remains the active implementation path.

## What This Changes

Slack Socket Mode can stop replying after long-running reconnect or pong failures while web chat continues working.

## Best Possible Solution

Review and land the linked Slack plugin fix or an equivalent current-main change that makes recoverable Socket Mode retries unlimited while preserving fail-fast auth errors; handle the no-event zombie WSS tracker separately.

## Reproduction Assessment

Yes. Source inspection shows repeated recoverable Slack disconnects or start failures increment to a hardcoded twelve-attempt cap and throw; field comments include matching finite-retry logs and a recurrence after the Slack SDK update.

## Solution Assessment

Yes for the permanent dead-Slack state. Removing the recoverable retry cap is the narrowest maintainable fix because the loop already has zero-as-unlimited guard/log conventions, while transient message loss and no-event zombie behavior should stay separate follow-up scope.
