## Summary

Keep open. Current main and latest release still contain the hardcoded 12-attempt Slack Socket Mode reconnect cap, and open PR #73162 references this issue with closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Slack Socket Mode can stop replying after repeated recoverable disconnects because the plugin gives up after a finite reconnect cap while the rest of the gateway remains usable.

## Best Possible Solution

Keep this issue open until #73162 or an equivalent Slack plugin change lands and ships; then validate transient reconnect-window message loss separately if it still reproduces.

## Reproduction Assessment

Yes, from source inspection rather than a live Slack workspace: current main still has `maxAttempts: 12`, and both disconnect and start-failure paths can throw after repeated recoverable attempts. The issue discussion also includes current-main reproduction notes and matching live logs.

## Solution Assessment

Yes for the permanent dead-Slack state: making recoverable Socket Mode retries unlimited while preserving fail-fast auth errors is the narrowest maintainable fix. Transient message loss while reconnecting should remain a separate validation or follow-up path.
