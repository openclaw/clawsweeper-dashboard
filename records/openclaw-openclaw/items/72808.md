## Summary

Keep open. Current main includes the newer Slack socket-mode dependency that matches the reporter's restart recovery, but the OpenClaw Slack plugin still has a hardcoded twelve-attempt reconnect cap and #73162 is the open closing PR for the remaining permanent-dead-state fix.

## What This Changes

The issue reports Slack Socket Mode becoming silently non-responsive after days of use while the gateway and web chat continue working.

## Best Possible Solution

Land #73162 or an equivalent Slack plugin change that makes recoverable Socket Mode retries unlimited while keeping nonrecoverable auth failures fail-fast; track transient reconnect-window delivery loss separately if it still reproduces.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to force repeated recoverable Slack Socket Mode disconnects until the current twelve-attempt cap is reached; the discussion includes matching ping/pong and reconnect logs from real installs.

## Solution Assessment

Yes. Removing the recoverable reconnect cap, as #73162 proposes, is the narrowest maintainable fix for the permanent dead-Slack state; the Slack SDK 2.0.7 update addresses one upstream trigger but leaves OpenClaw's cap in place.
