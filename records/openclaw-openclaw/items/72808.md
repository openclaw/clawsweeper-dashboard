## Summary

Keep open. Current main and the latest release still cap Slack Socket Mode fallback reconnects at twelve attempts, and an open linked PR directly targets that remaining failure mode.

## What This Changes

Slack Socket Mode can silently stop replying after long-running reconnect instability while the web interface continues to work.

## Best Possible Solution

Review, rebase, or replace the linked Slack reconnect PR so recoverable Socket Mode outages keep retrying; track reconnect-window message loss separately if it still reproduces.

## Reproduction Assessment

Yes. Source inspection on current main shows repeated Slack disconnect or start-failure events can reach the twelve-attempt guard and throw; field comments report matching retry warnings, but I did not run a live multi-day Slack loop.

## Solution Assessment

Yes. Keeping this open until the linked PR lands is the narrowest maintainable path; the existing zero-as-unlimited convention can fix the cap while disconnected-status hardening remains follow-up scope.
