## Summary

Keep open. Current main and v2026.5.3 still contain OpenClaw's hardcoded Slack Socket Mode reconnect cap, while PR #73162 remains the open closing implementation candidate for the permanent silent-Slack failure path.

## What This Changes

The issue asks OpenClaw's Slack Socket Mode connection to keep recovering and report health instead of silently stopping delivery after working for days.

## Best Possible Solution

Land PR #73162 or an equivalent Slack plugin fix that makes recoverable Socket Mode retries unlimited while preserving fail-fast handling for non-recoverable auth errors.

## Reproduction Assessment

Yes. Source inspection shows current main still reaches a hardcoded twelve-attempt Slack Socket Mode cap and throws, and the issue discussion includes matching current-main reproduction notes and live logs.

## Solution Assessment

Yes for the permanent dead-Slack state. Removing the recoverable reconnect cap is the narrowest maintainable fix; the Slack SDK 2.0.7 update addresses a likely upstream pong trigger but not OpenClaw's own finite retry exit.
