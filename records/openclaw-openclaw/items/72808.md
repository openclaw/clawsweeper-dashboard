## Summary

Keep open. Current main and v2026.5.2 still ship the hardcoded twelve-attempt Slack Socket Mode reconnect cap, and #73162 remains an open closing PR for this issue, so closing before merge would be premature.

## What This Changes

The issue reports Slack Socket Mode becoming permanently unresponsive after several days while web chat continues to work.

## Best Possible Solution

Land #73162 or an equivalent Slack plugin fix that makes recoverable Socket Mode retries unlimited while preserving fail-fast auth errors, then validate transient reconnect-window message loss as follow-up if it still reproduces.

## Reproduction Assessment

Yes. Source inspection on current main shows recoverable Slack socket disconnects increment against a hardcoded `maxAttempts: 12` policy and throw at exhaustion; the issue discussion includes matching current-main reproduction notes and live logs.

## Solution Assessment

Yes for the permanent dead-Slack state. Removing the recoverable reconnect cap in the Slack plugin is the narrowest maintainable fix, while the upstream Slack SDK pong fix and reconnect-window delivery loss remain related validation/follow-up work.
