## Summary

Keep open. Current main and the latest release still cap Slack Socket Mode reconnects at 12 attempts, while the live closing PR that removes the cap is open and unmerged, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports Slack Socket Mode becoming permanently non-responsive after a long-running gateway silently loses its Slack connection while web chat still works.

## Best Possible Solution

Land the linked Slack reconnect fix, or an equivalent narrow change, so recoverable Slack Socket Mode disconnects keep retrying while non-recoverable auth errors still fail fast and health status remains accurate.

## Reproduction Assessment

Yes from source inspection, but not from a live multi-day run in this review. Current main still caps Slack Socket Mode reconnects at 12 and throws at exhaustion, matching the reporter and follow-up logs.

## Solution Assessment

Yes, keeping this issue open is the right cleanup outcome because the linked closing PR is still open. The narrow maintainable fix is to remove the recoverable reconnect cap while preserving fail-fast handling for revoked or invalid Slack credentials.
