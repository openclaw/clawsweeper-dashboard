## Summary

Keep open. Current main still has a finite Slack Socket Mode reconnect cap, and the unmerged PR #73162 is the implementation candidate that references this issue with closing syntax.

## What This Changes

The issue reports that Slack can stop replying after a long-running Socket Mode connection fails while the web interface continues to work.

## Best Possible Solution

Land #73162 or an equivalent Slack plugin fix that makes recoverable Socket Mode retries unlimited while preserving fail-fast handling for non-recoverable auth errors.

## Reproduction Assessment

Yes. By source inspection, repeated recoverable Slack Socket Mode disconnects or start failures reach the hardcoded cap and throw; after enough channel-level restarts, the gateway also gives up restarting that account.

## Solution Assessment

Yes. Removing the recoverable Slack Socket Mode attempt cap is the narrowest maintainable fix for the long-running lost-connection path; transient reconnect-window delivery loss should be validated separately if it still reproduces.
