## Summary

Keep open: current main still contains the finite Slack Socket Mode reconnect cap and throw path, and the open linked fix PR uses closing syntax for this issue, so the issue should remain open until that PR or an equivalent maintainer fix lands.

## What This Changes

The issue reports Slack Socket Mode silently stopping replies after running for days while the gateway and web interface remain usable.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73162, or an equivalent maintainer replacement that removes the permanent Slack reconnect exhaustion path while keeping non-recoverable auth failures fail-fast.

## Reproduction Assessment

Yes. Current main has a source-visible reproduction path: repeated recoverable Slack socket disconnects or start failures increment the retry counter until the fixed 12-attempt cap throws, and the issue discussion includes matching finite-retry logs plus an unlimited-retry mitigation log.

## Solution Assessment

Yes. The linked PR's `maxAttempts: 0` direction is the narrowest maintainable fix for the permanent dead-state because the loop already treats zero as unlimited, keeps bounded backoff and abort handling, and still fails fast for permanent Slack auth errors.
