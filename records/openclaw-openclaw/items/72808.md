## Summary

Keep open. Current main and the latest stable tag still carry the finite Slack Socket Mode reconnect cap, the provider still throws when that cap is reached, and the linked open pull request remains the direct fix path for the reported silent Slack outage.

## What This Changes

The issue reports Slack Socket Mode going permanently silent after days of use while the web interface continues to work.

## Best Possible Solution

Review, rebase, or replace the linked reconnect-cap pull request so recoverable Slack Socket Mode outages cannot permanently exhaust the channel; track zombie/no-event and transient delivery-loss paths separately if they still reproduce.

## Reproduction Assessment

Yes. Source inspection on current main shows repeated Slack disconnect or start-failure events increment `reconnectAttempts` until the 12-attempt guard throws; issue comments include matching pong/retry logs and a May 11 recurrence after the Slack SDK fix, though I did not run a live multi-day Slack loop.

## Solution Assessment

Yes. Keeping this issue open while https://github.com/openclaw/openclaw/pull/73162 or an equivalent replacement lands is the narrowest maintainable path; closing now would lose an active, source-backed regression that is not implemented on current main.
