## Summary

Keep open. Current main and the latest release still cap Slack Socket Mode reconnects at 12 attempts, while open PR #73162 is the paired implementation candidate that references this issue with closing syntax.

## What This Changes

The issue asks OpenClaw to keep Slack responsive instead of silently dropping Slack traffic after long-running Socket Mode reconnect failures.

## Best Possible Solution

Review and land #73162 or an equivalent Slack plugin fix that makes recoverable Socket Mode retries unlimited while preserving fail-fast auth errors, then track transient reconnect-window delivery loss separately if it still reproduces.

## Reproduction Assessment

Yes. Source inspection on current main shows twelve recoverable Slack Socket Mode disconnects or start failures reach the hardcoded cap and throw; issue comments provide matching current-main reproduction and live logs.

## Solution Assessment

Yes for the permanent silent-dead path. Landing #73162 or the same Slack plugin change is the narrowest maintainable fix; Slack SDK pong handling and transient not-ready delivery loss should remain validation or follow-up work.
