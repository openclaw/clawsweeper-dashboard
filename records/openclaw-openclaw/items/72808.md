## Summary

Keep open. Current main still hardcodes a 12-attempt Slack Socket Mode reconnect cap and throws after exhaustion, while #73162 is an open closing PR for the exact cap and maintainer discussion is still validating Slack SDK pong-stall and transient delivery behavior.

## What This Changes

The issue reports Slack Socket Mode becoming non-responsive after days while OpenClaw web chat remains usable, with Slack DMs and channel mentions ignored until recovery.

## Best Possible Solution

Keep this issue open until #73162 or a replacement lands with Slack-owned coverage for repeated disconnect/start-failure recovery, then validate the current Slack SDK pong-stall behavior. If reconnect-window message loss remains after that, split it into a narrower follow-up.

## Reproduction Assessment

Yes. Repeated Socket Mode disconnect or start-failure events deterministically increment the hardcoded 12-attempt policy until the provider throws, and the discussion includes current-main reproduction plus live logs for the same failure family.

## Solution Assessment

Unclear whether the current open uncap is the complete solution. It is the narrowest maintainable fix for permanent reconnect exhaustion, but the Slack SDK pong-stall path and transient reconnect-window delivery failures still need maintainer validation.
