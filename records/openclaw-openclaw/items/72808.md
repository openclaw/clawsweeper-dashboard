## Summary

Keep open. Current main still has a hardcoded Slack Socket Mode reconnect cap and throws after exhaustion, while open PR #73162 is the unmerged implementation candidate that explicitly references this issue.

## What This Changes

Slack Socket Mode can silently stop responding after repeated reconnect failures while the gateway and web UI remain usable.

## Best Possible Solution

Land #73162 or an equivalent Slack plugin fix that makes recoverable Socket Mode retries unlimited while preserving fail-fast auth errors, then track transient reconnect-window delivery loss separately if it still reproduces.

## Reproduction Assessment

Yes. By source inspection, twelve recoverable Slack Socket Mode disconnects or start failures hit `maxAttempts: 12` and throw; the issue discussion includes matching live pong-timeout and reconnect-loop logs.

## Solution Assessment

Yes for the permanent dead-Slack state. Removing the recoverable reconnect cap is the narrowest maintainable fix, while transient not-ready delivery loss should remain a separate validation or follow-up path.
