## Summary

Keep open. Current main and latest release still cap Slack Socket Mode reconnects at 12 attempts, while the linked fix PR https://github.com/openclaw/openclaw/pull/73162 is open and unmerged.

## What This Changes

The issue reports Slack Socket Mode going permanently silent after days of use while the gateway and web chat remain healthy.

## Best Possible Solution

Land a focused Slack plugin fix that makes recoverable Socket Mode reconnects indefinite while preserving fail-fast behavior for nonrecoverable auth errors and existing disconnected-status updates.

## Reproduction Assessment

Yes. Current main still has `maxAttempts: 12` and provider code that throws when that cap is reached; I marked this source-reproducible rather than reproduced because I did not run a live Slack multi-day repro.

## Solution Assessment

Yes, the narrowest maintainable fix is the linked cap-removal path or equivalent, while keeping nonrecoverable auth errors fail-fast. A separate ClawSweeper repair PR would duplicate the open implementation candidate.
