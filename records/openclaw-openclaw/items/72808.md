## Summary

Keep open. Current main and the latest release still retain the hardcoded Slack Socket Mode reconnect cap, and open PR #73162 is the linked implementation candidate with closing syntax for this issue.

## What This Changes

The issue reports Slack Socket Mode becoming silently unresponsive after running successfully for days while the web interface remains usable.

## Best Possible Solution

Land #73162 or an equivalent Slack plugin fix that makes recoverable Socket Mode retries unlimited while preserving fail-fast behavior for nonrecoverable auth errors; track transient reconnect-window message loss separately if it still reproduces.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Slack Socket Mode uses OpenClaw's local retry loop, twelve consecutive recoverable disconnect/start failures hit the hardcoded cap, and the provider exits that monitor path.

## Solution Assessment

Yes for the permanent silent-dead state. Removing the recoverable reconnect cap is the narrowest maintainable fix, while the upstream Slack SDK pong fix and any transient not-ready delivery loss are separate validation paths.
