## Summary

Keep open. Current main still has the hardcoded 12-attempt Slack Socket Mode reconnect cap, the latest tagged release still contains the same cap and throw path, and open PR #73162 references this issue with closing syntax, so the issue should stay open until that PR or a replacement lands and is validated.

## What This Changes

The request is to prevent Slack Socket Mode from going permanently silent after days of operation while the rest of OpenClaw, including web chat, still works.

## Best Possible Solution

Keep this issue open until #73162 or a replacement lands with Slack-owned coverage for repeated disconnect and start-failure recovery, then validate the Slack SDK pong-stall path on current main. If reconnect-window delivery loss remains, track that as a narrower follow-up.

## Reproduction Assessment

Yes. Repeated Socket Mode disconnect or start-failure events deterministically increment the current hardcoded 12-attempt policy until the provider throws, and the issue discussion includes current-main reproduction plus live logs for the same failure family.

## Solution Assessment

Unclear whether #73162 alone is the complete solution. It is the narrowest fix for permanent reconnect exhaustion, but the maintainer discussion also points to upstream Slack SDK pong-stall behavior and transient delivery failures that need validation or follow-up.
