## Summary

Keep open. Current main still has the finite Slack Socket Mode reconnect cap and throws after twelve recoverable failures, while the linked open PR remains the direct fix path for this issue.

## What This Changes

Slack Socket Mode can silently stop responding after a long-running gateway hits reconnect exhaustion while web chat remains healthy.

## Best Possible Solution

Review, rebase, or replace https://github.com/openclaw/openclaw/pull/73162 so recoverable Slack Socket Mode outages cannot leave Slack permanently silent, then track transient reconnect-window delivery loss separately if it still reproduces.

## Reproduction Assessment

Yes. Source inspection shows a high-confidence path where repeated Slack disconnect or start-failure events reach the twelve-attempt cap and throw; issue comments include matching field logs, but I did not run a live long-duration Slack loop.

## Solution Assessment

Yes. Keeping this issue open for the linked implementation PR, or an equivalent narrow replacement, is the right path because the fix has not merged and the PR body uses closing syntax for this issue.
