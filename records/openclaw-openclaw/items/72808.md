## Summary

Keep open. Current main and the latest v2026.5.2 tag still cap Slack Socket Mode reconnects at 12 attempts, while open PR #73162 already targets the narrow cap removal and the discussion still has reconnect-window/upstream Slack SDK follow-up context to validate.

## What This Changes

The issue asks OpenClaw to stop silently losing Slack connectivity after a long-running Socket Mode connection stops delivering messages.

## Best Possible Solution

Land #73162 or an equivalent Slack-owned change that makes recoverable Socket Mode retries unlimited while preserving fail-fast auth errors, then track transient not-ready delivery loss separately if it still reproduces.

## Reproduction Assessment

Yes. Current source gives a high-confidence path by forcing repeated recoverable Slack Socket Mode disconnects or start failures until the twelfth attempt, and the issue discussion includes current-main reproduction notes plus matching live logs.

## Solution Assessment

Unclear. Removing the reconnect cap is the narrowest fix for the permanent dead-Slack state, but the comments indicate transient reconnect-window delivery failures and upstream pong-stall behavior still need validation or follow-up.
