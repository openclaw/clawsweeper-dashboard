## Summary

Keep open. Current main still has a hardcoded Slack Socket Mode reconnect cap, the latest release still ships that cap, and open PR #73162 remains the linked implementation candidate for the permanent dead-Slack failure mode.

## What This Changes

The issue reports Slack Socket Mode becoming unresponsive after days while web chat remains healthy and the Slack bot still appears online.

## Best Possible Solution

Land #73162 or an equivalent Slack plugin fix that makes recoverable Socket Mode retries unlimited while preserving fail-fast auth errors, then track transient reconnect-window delivery loss separately if it still reproduces.

## Reproduction Assessment

Yes. Source inspection on current main shows repeated recoverable Slack Socket Mode disconnect/start failures hit a hardcoded twelve-attempt cap and throw; issue comments include current-main reproduction notes and matching live logs.

## Solution Assessment

Yes for the permanent dead-Slack state. Removing the recoverable reconnect cap is the narrowest maintainable fix, while transient reconnect-window delivery loss should remain a separate validation or follow-up path.
