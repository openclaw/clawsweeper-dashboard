## Summary

Keep open. Current main and the latest tagged release still contain the finite Slack Socket Mode reconnect cap, and open PR #73162 already references this issue with closing syntax while maintainers are still validating whether the upstream Slack SDK pong-stall fix covers the broader failure family.

## What This Changes

The issue reports that Slack Socket Mode can stop responding while the OpenClaw gateway and web UI remain usable and Slack still shows the bot online.

## Best Possible Solution

Keep this issue open until #73162 or an equivalent Slack-owned fix lands with coverage for repeated disconnect/start-failure recovery, then validate the `@slack/socket-mode` pong-stall update in this repository. If reconnect-window delivery loss remains afterward, split that into a narrower follow-up instead of treating it as solved by the reconnect cap alone.

## Reproduction Assessment

Yes. Repeated Slack Socket Mode disconnect or start-failure events deterministically increment the current 12-attempt policy until the provider throws, and the discussion includes current-main reproduction plus live logs matching that failure family.

## Solution Assessment

Unclear. Removing the reconnect cap is the narrowest maintainable fix for permanent reconnect exhaustion, but maintainer discussion and later logs point to upstream Slack SDK pong-stall behavior and transient delivery failures that still need validation.
