## Summary

Keep open. Current main and the latest v2026.5.2 tag still cap Slack Socket Mode reconnects at 12 attempts, and open PR #73162 is the linked implementation path for the narrow reconnect-cap fix.

## What This Changes

The issue reports Slack becoming permanently non-responsive after previously working, while the OpenClaw web interface still works.

## Best Possible Solution

Land #73162 or an equivalent Slack-owned change that keeps recoverable Socket Mode reconnects indefinite while still failing fast on auth errors; track transient not-ready delivery loss separately if it remains.

## Reproduction Assessment

Yes, from source inspection and discussion logs. Current main can be driven through repeated recoverable Slack Socket Mode disconnects or start failures until the twelfth attempt, after which the provider exits.

## Solution Assessment

Yes for the main permanent-silence failure. Removing the retry cap is the narrowest maintainable fix because the code already treats `maxAttempts: 0` as unlimited and still fails fast on non-recoverable auth errors.
