## Summary

Keep open. Current main still has a finite Slack Socket Mode reconnect cap, the latest shipped release still has that cap, and open PR #73162 remains the linked implementation candidate for this issue.

## What This Changes

The issue asks OpenClaw's Slack plugin to recover instead of silently stopping after Slack Socket Mode disconnects.

## Best Possible Solution

Land #73162 or an equivalent Slack plugin change that makes recoverable Socket Mode retries unlimited while preserving fail-fast auth errors and disconnected status reporting.

## Reproduction Assessment

Yes. Source inspection on current main shows twelve recoverable Slack Socket Mode disconnects can still reach a hardcoded cap and throw, and the issue discussion includes matching live logs.

## Solution Assessment

Yes for the permanent dead-Slack state. Removing the recoverable reconnect cap is the narrowest maintainable fix; transient not-ready delivery loss should be validated separately after that lands.
