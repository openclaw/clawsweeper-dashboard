## Summary

Keep open. Current main has a generic bounded channel auto-restart mitigation, but Slack Socket Mode still uses a hardcoded 12-attempt reconnect cap with no Slack config override; the open closing PR targets that cap, and discussion still has unresolved pong-stall and reconnect-window loss concerns.

## What This Changes

Bug report asks Slack Socket Mode to keep replying or recover visibly when Slack silently stops responding while WebChat still works.

## Best Possible Solution

Review and land #73162 or a replacement Slack-owned fix if unlimited Socket Mode reconnects are the intended policy, then split any remaining reconnect-window message loss into a narrower follow-up.

## Reproduction Assessment

Yes. Current source still has OpenClaw-owned Socket Mode reconnects with a 12-attempt cap, and the issue comments include current-main reproduction notes plus live retry logs; a focused repro can force repeated disconnect/start-failure events through that path.

## Solution Assessment

Unclear. Removing the Slack cap is the narrowest fix for permanent finite-retry exhaustion, but maintainers should validate it alongside the current generic channel restart fallback and the Slack SDK 2.0.7 pong-stall behavior.
