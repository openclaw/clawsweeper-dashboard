## Summary

Keep open. Current main still has the Slack Socket Mode finite reconnect cap, and the issue is already paired with open PR #73162 while maintainer discussion continues around upstream pong stalls and reconnect-window delivery loss.

## What This Changes

The issue reports Slack Socket Mode going silent after a previously working long-running connection, while web chat continues to work.

## Best Possible Solution

Land #73162 or an equivalent Slack-owned change that makes recoverable Socket Mode retries unlimited while preserving fail-fast auth errors, then track transient not-ready delivery loss separately if it still reproduces.

## Reproduction Assessment

Yes. Current source gives a high-confidence path by forcing recoverable Slack Socket Mode start/disconnect failures until the retry cap is reached, and the issue discussion includes matching current-main notes and live logs.

## Solution Assessment

Unclear. Removing the cap is the narrowest fix for the permanent dead-Slack state, but the comments indicate reconnect-window delivery failures and upstream pong-stall behavior still need validation.
