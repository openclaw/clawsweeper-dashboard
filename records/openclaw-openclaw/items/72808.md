## Summary

Keep open. Current main and v2026.5.2 still cap Slack Socket Mode reconnects at 12 attempts, while open PR #73162 uses closing syntax and targets the narrow permanent-dead-channel failure; comments also leave transient reconnect-window behavior to validate.

## What This Changes

Slack Socket Mode stops answering after a long-running connection silently exhausts reconnect handling, even though the gateway and web UI remain usable.

## Best Possible Solution

Land #73162 or an equivalent Slack-owned change that makes recoverable Socket Mode retries unlimited while preserving fail-fast auth errors, then track reconnect-window message loss separately if it still reproduces.

## Reproduction Assessment

Yes. Source inspection shows current main retries Slack Socket Mode until `SLACK_SOCKET_RECONNECT_POLICY.maxAttempts` reaches 12 and then throws; issue comments provide matching current-main and live-log evidence for repeated ping/pong disconnects.

## Solution Assessment

Unclear for the whole issue. Removing the reconnect cap is the narrowest fix for the permanent dead-Slack state, but comments indicate transient not-ready delivery loss during reconnect windows may need a separate follow-up after #73162.
