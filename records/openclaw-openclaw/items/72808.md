## Summary

Keep open. Current main and the latest release still hardcode the Slack Socket Mode reconnect cap at twelve attempts, and the provider still exits when that cap is reached; the linked open implementation PR is the right path but has not merged yet.

## What This Changes

The issue reports that Slack Socket Mode stops responding after running for days while the web interface continues to work.

## Best Possible Solution

Review, rebase, or replace https://github.com/openclaw/openclaw/pull/73162 so recoverable Slack Socket Mode outages cannot leave Slack permanently silent, then track transient reconnect-window delivery loss separately if it still reproduces.

## Reproduction Assessment

Yes. Source inspection on current main shows repeated Slack socket disconnect or start-failure events can increment to the hardcoded twelve-attempt cap and throw; the issue comments include matching field logs and a recurrence after https://github.com/slackapi/node-slack-sdk/pull/2565, though I did not run a live long-duration Slack loop.

## Solution Assessment

Yes. Keeping this open for the linked reconnect-cap PR or an equivalent replacement is the narrowest maintainable path; current main and the latest release do not yet implement the fix.
