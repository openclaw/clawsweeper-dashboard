## Summary

Keep open: current `main` and the latest release still contain the default heartbeat template footer and predicate behavior that make the seeded file look actionable. The same author has an open closing PR, so this issue should stay open until that PR or another canonical fix lands.

## What This Changes

The issue reports that the default seeded `HEARTBEAT.md` template is treated as non-empty, causing unnecessary heartbeat model API calls on fresh installs.

## Best Possible Solution

Land one canonical narrow fix that treats only docs-only scaffold/footer content as non-actionable, preserves user-authored heartbeat tasks, and adds regression coverage for the actual shipped template plus link/list task cases.

Yes at source level: the current default template seeds the Related link, and the current predicate still returns false on that line before the heartbeat preflight can skip as `empty-heartbeat-file`. I did not run a live Docker/provider wait loop in this read-only review.
