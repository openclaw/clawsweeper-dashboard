## What This Changes

The issue reports that the default seeded HEARTBEAT.md contains a docs-only Related link that prevents empty-heartbeat skipping and causes recurring heartbeat model calls.

## Best Possible Solution

Land one narrow fix, likely https://github.com/openclaw/openclaw/pull/84677 or https://github.com/openclaw/openclaw/pull/75047, that makes the generated template empty by default while preserving real link/task content, then close this issue and fold in https://github.com/openclaw/openclaw/issues/85854 as the duplicate corroborating report.

Yes, at source level: the current seeded template includes the Related link, the detector treats that link as actionable, and the runner only skips the model call when the detector returns true. I did not run the full Docker/provider wait-loop in this read-only review.

Yes: the best fix is a narrow template/classifier regression repair with tests that exercise the actual generated HEARTBEAT.md and preserve user-authored actionable tasks. The open linked PRs are implementation candidates, not a reason to close this issue before merge.
