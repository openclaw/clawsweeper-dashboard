## Summary

This looks like a valid current-main regression: the default seeded HEARTBEAT.md now includes a docs-only Related link, and the empty-heartbeat gate treats that link as actionable content, so the heartbeat preflight will not skip the model call.

## What This Changes

The issue asks OpenClaw to treat the current default HEARTBEAT.md template as effectively empty so default heartbeats do not spend model tokens.

## Best Possible Solution

Make the seeded default HEARTBEAT.md non-actionable again, ideally by stripping docs-only template footer content or by testing the actual current template through the empty-heartbeat preflight while preserving user-authored tasks.

## Reproduction Assessment

Yes at source level: the current template after frontmatter stripping returns false in the same predicate that gates empty-heartbeat skips. I did not perform the full Docker/Claude wait-loop, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

Unclear until a patch exists, but the maintainable direction is narrow: keep the documented default skip behavior and add regression coverage for the actual shipped template, not a new heartbeat mode or config knob.
