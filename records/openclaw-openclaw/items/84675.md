## Summary

This should stay open: current main and the latest release still treat the default HEARTBEAT.md docs footer as non-empty, and an open same-author PR with closing syntax is already the active fix path.

## What This Changes

The issue reports that the default HEARTBEAT.md template can trigger recurring heartbeat model API calls even though the seeded file is intended to be effectively empty.

## Best Possible Solution

Land one narrow fix such as https://github.com/openclaw/openclaw/pull/84677 after confirming link-scope compatibility, then close this issue as implemented.

## Reproduction Assessment

Yes, at source level: the current default template leaves a Related link after frontmatter stripping, and the current empty-content predicate treats that line as actionable content. I did not run a live Docker/Claude heartbeat loop, so the live token-usage symptom remains inferred from the preflight path.

## Solution Assessment

Unclear until a PR is merged, but the best direction is narrow: make the seeded default template non-actionable again while preserving user-authored heartbeat tasks, especially link-only tasks outside the docs footer.
