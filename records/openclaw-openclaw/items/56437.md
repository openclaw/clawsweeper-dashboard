## Summary

Keep open: current main still lacks a native fallback-only runbook config field and injection path. Adjacent observability and limited model-ref validation exist, but they do not solve the requested fallback-only identity/tool-rule prompt surface.

## What This Changes

The issue requests a native fallback-only runbook prompt in agent model config plus diagnostics for invalid fallback model refs.

## Best Possible Solution

Add a typed, documented fallback-only prompt field for default and per-agent text model config, inject it only for fallback candidates, and add conservative diagnostics for provably invalid fallback refs.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, as source-level reproduction: the strict model schema/type surface has no fallbackRunbook key, and the fallback execution path has no fallback-candidate-specific prompt injection point.

## Solution Assessment

Is this the best way to solve the issue? Unclear. The requested direction is plausible, but the exact config shape, prompt precedence, cache/session behavior, and provider-catalog warning semantics need maintainer API/product direction first.
