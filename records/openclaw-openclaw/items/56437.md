## Summary

Keep open: current main still lacks a native fallback-only runbook config/injection path, and the remaining work is a public config and prompt-ordering feature that needs maintainer API direction.

## What This Changes

Requests a native `fallbackRunbook` agent model config field that is injected only during model fallback, plus startup/doctor diagnostics for invalid fallback model refs.

## Best Possible Solution

Add a typed, documented fallback-only prompt field for default and per-agent text model config, inject it only when the active candidate is a fallback, and add non-blocking diagnostics for fallback refs that are provably unresolved.

## Reproduction Assessment

Yes, as a source-level reproduction rather than a runtime bug repro: `agents.defaults.model.fallbackRunbook` is outside the strict schema/type surface, and the fallback execution path has no injection point for fallback-only runbook text.

## Solution Assessment

Unclear. The requested direction is plausible, but the exact schema placement, prompt precedence, and provider-catalog warning semantics need maintainer approval before implementation should proceed.
