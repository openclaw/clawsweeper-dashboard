## What This Changes

The issue requests making browser doctor Chrome version detection tolerate macOS Gatekeeper cold-cache latency instead of warning that a current Chrome version is unknown.

## Best Possible Solution

Land or replace the canonical browser plugin fix in https://github.com/openclaw/openclaw/pull/85460 with plist-first macOS version detection, a bounded fallback probe, regression coverage, and real macOS doctor proof.

## Reproduction Assessment

Yes. Current-main source plus a focused Node timeout probe show that a Chrome `--version` child process exceeding 2000 ms returns null through `execText`, and doctor then emits the reported unknown-version warning.

## Solution Assessment

Yes. A bounded browser plugin change that reads macOS app-bundle metadata before spawning Chrome, then keeps a slower fallback probe, is the narrow maintainable solution; this issue should stay open until the linked fix PR lands or is replaced.
