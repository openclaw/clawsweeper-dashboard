## What This Changes

The issue reports that `openclaw doctor --deep` can falsely warn about Chrome version detection on macOS when the 2 second Chrome `--version` probe times out during Gatekeeper cold-cache verification.

## Best Possible Solution

Land the open plist-first browser plugin fix, or a replacement with the same behavior and real macOS proof, then close this issue as implemented on main.

## Reproduction Assessment

Yes by source inspection: current main turns a Chrome `--version` call that exceeds 2000 ms into null, and doctor emits the reported unknown-version warning. I did not run a live macOS Gatekeeper cold-cache reproduction in this Linux review environment.

## Solution Assessment

Yes, the plist-first macOS bundle metadata approach in the open linked PR is the narrowest maintainable fix shape because it avoids the cold app spawn and keeps a bounded fallback for non-bundle paths.
