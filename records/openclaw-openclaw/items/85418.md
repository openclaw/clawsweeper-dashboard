## What This Changes

The issue asks doctor’s Chrome MCP readiness check to avoid false unknown-version warnings when macOS cold-cache Gatekeeper latency makes Chrome `--version` exceed 2 seconds.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/85460, or an equivalent browser-plugin fix that reads macOS bundle metadata before a bounded slower fallback, then close this issue after merge.
