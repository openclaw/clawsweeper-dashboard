The observable bug is that `openclaw infer image generate --model openai/gpt-image-2 ... --json` fails with `TypeError: fetch failed` when invoked through packaged `openclaw.mjs`/`dist/entry.js`, while direct `runCli(argv)`, direct provider invocation, guarded `postJsonRequest(...)`, and raw `fetch()` succeed on the same host. Keep the fix narrowly scoped to the launcher/entry/bootstrap, CLI run-main, proxy/dispatcher, or OpenAI image-provider boundary as evidence requires. Add regression coverage that distinguishes packaged entry/main invocation from direct `runCli()` and provider-level paths; if live credentials are available, include the packaged OpenAI Codex OAuth image smoke, not only the current Google `scripts/run-node.mjs` smoke. Include a user-facing `CHANGELOG.md` fix entry unless the final patch is test-only/internal.
