## Repair Work Prompt

Fix the packaged-entry Codex OAuth image-generation parity regression tracked by https://github.com/openclaw/openclaw/issues/72087. The observable bug is that `openclaw infer image generate --model openai/gpt-image-2 ... --json` and direct `node dist/entry.js ...` fail with `fetch failed`, while direct `runCli(argv)`, direct OpenAI image provider, guarded `postJsonRequest`, and raw fetch succeed on the same host. Investigate the packaged `openclaw.mjs`/`src/entry.ts` main prelude versus direct `runCli()` path, especially process-wide env normalization, compile-cache respawn, argv/profile handling, proxy capture, global undici dispatcher, and operator-managed proxy routing. Keep the fix narrow; do not weaken SSRF, guarded-fetch, proxy, auth, or provider request policy. Add regression coverage that proves packaged launcher or `dist/entry.js` parity for the OpenAI Codex OAuth image path with mocked network/auth where possible, and extend live coverage to exercise the packaged entry path when OpenAI Codex OAuth credentials are configured. Add a user-facing changelog fix entry. Stop and ask for maintainer review if the fix requires changing proxy security policy or adding a config knob.
