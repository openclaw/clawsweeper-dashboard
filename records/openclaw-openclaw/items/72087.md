Keep open. Current main still has a distinct packaged-entry bootstrap path with no packaged Codex OAuth image-generation parity proof, and the macOS follow-up shows the report is not Linux-only; adjacent Codex OAuth transport work does not cover this boundary.

Make packaged `openclaw infer image generate --model openai/gpt-image-2` behave like direct `runCli()` and provider invocation when using OpenAI Codex OAuth.

## Best Possible Solution

Fix the packaged entry/main bootstrap so it preserves direct `runCli()` OpenAI Codex OAuth image-generation behavior, then add packaged-entry regression coverage and a live Codex smoke path when credentials are configured.
