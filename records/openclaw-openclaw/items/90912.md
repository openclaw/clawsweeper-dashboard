## Summary

The patch is a coherent, focused fix for a source-reproducible resumed CLI watchdog mismatch: it preserves the inherited 180-second guard while allowing deliberate 600-second run settings to produce a 480-second no-output window. No discrete correctness or security defect was found, but this member-authored PR requires maintainer judgment because the longer watchdog intentionally increases stalled-process retention and the linked custom Anthropic report covers a broader, unverified path.

## What This Changes

The PR propagates explicit CLI timeout intent and configured positive agent-default timeout presence into resumed CLI no-output watchdog selection, with regression coverage for explicit, configured, inherited, and cron cases.

## Best Possible Solution

Land the single timeout-intent path after a maintainer explicitly accepts the bounded stalled-process retention tradeoff, keep inherited resumes on the 180-second guard, and leave the broader custom-provider report open until its execution path is independently reproduced or fixed.

## Reproduction Assessment

Yes. Current main deterministically resolves a user-triggered resumed CLI run with a 600-second effective timeout to the 180-second resume profile, and the PR’s resolver probe demonstrates the corrected 480-second result.

## Solution Assessment

Yes for the confirmed CLI-backed branch. Propagating timeout intent through the existing dispatch boundary is narrower and more maintainable than inferring explicitness from resolved values, but a maintainer must accept the intentional longer stalled-process window.
