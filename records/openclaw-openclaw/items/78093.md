## What This Changes

The issue asks OpenClaw to restore sibling `workspace-<id>` defaults for non-default agents so custom default workspaces do not nest new agents under the main workspace.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/78113 or choose a maintainer-approved migration/compatibility path that keeps the default agent honoring `agents.defaults.workspace` while preventing new non-default agents from nesting under it.

Yes. Source inspection on current `main` shows `resolveAgentWorkspaceDir` joins a non-default agent id under `agents.defaults.workspace`, and the current test suite asserts that nested fallback.
