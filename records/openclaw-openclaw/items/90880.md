## Summary

Keep open. Current main has overlapping primitives for subagents, cron, task records, generated-media background jobs, and completion events, but not the requested general assistant-facing post-response enqueue API tied to an originating WebChat/conversation callback.

The issue requests a runtime-level post-response async-action queue so agents can acknowledge queued work immediately and later deliver or resume with confirmed results.

## Best Possible Solution
