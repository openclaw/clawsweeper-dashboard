## Summary

Keep open. Current main has adjacent task ledger, Task Flow, cron, hook, subagent-yield, and completion-delivery primitives, but it still does not provide the requested general post-response enqueue/callback contract tied to the originating conversation; the related on-exit cron PR is open and only covers one narrower trigger.

## What This Changes

Add first-class runtime support for agents to enqueue post-response asynchronous work, persist its status, and deliver or resume the originating conversation when it completes.

## Best Possible Solution

Define a maintainer-owned runtime/API extension that reuses the task ledger, Task Flow, cron/session wake, and channel delivery primitives while specifying status, cancellation, idempotency, and callback semantics before implementation.

## Reproduction Assessment

Not applicable; this is a feature request. Current main source and docs prove adjacent primitives and the absence of a general post-response enqueue contract, but there is no broken current contract to reproduce.

## Solution Assessment

Unclear until maintainers choose the API boundary. The safest direction is to extend existing task, Task Flow, cron, session, and channel delivery primitives rather than add a parallel background-job stack.
