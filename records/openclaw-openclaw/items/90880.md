## Summary

Keep open. Current main and the latest release have adjacent exec backgrounding, cron, task-ledger, Task Flow, subagent-yield, and completion-delivery primitives, but they still do not provide the requested general post-response enqueue/callback contract tied to an originating conversation; the related on-exit cron PR is open and only covers one narrower trigger.

## What This Changes

Request a runtime-level post-response async action queue that persists jobs, lets an agent acknowledge queued work, and later delivers or resumes with confirmed results.

## Best Possible Solution

Define a maintainer-owned runtime/API extension that reuses existing task, Task Flow, cron, wake, session, and channel-delivery primitives while specifying status, cancellation, idempotency, execution policy, and callback semantics before implementation.

## Solution Assessment
