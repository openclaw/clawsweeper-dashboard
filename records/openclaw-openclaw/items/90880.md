## Summary

Keep open. Current main has adjacent task-ledger, Task Flow, cron, subagent-yield, and media-completion wake primitives, but it still lacks the requested general post-response enqueue/callback contract tied to the originating conversation; related PRs are open and unmerged rather than an implemented mainline fix.

## What This Changes

Request to add a runtime-level post-response async action queue so agents can acknowledge work immediately, execute persisted background work after the visible response, and send or resume with a later result.

## Best Possible Solution

Design this as a maintainer-owned runtime/API extension that reuses the task ledger, Task Flow, subagent yield/completion delivery, cron wake, and channel/session routing primitives instead of adding a parallel background-job stack.

## Reproduction Assessment

Not applicable as a feature request. Source and docs show related primitives, but no current documented post-response enqueue contract that can be reproduced as broken behavior.

## Solution Assessment

Unclear until maintainers choose the API boundary. The safest direction is to extend existing task, flow, cron, session, and channel delivery primitives rather than add a separate queue with competing callback semantics.
