## Summary

Keep open. Current main has related subagent, yield, cron, task-ledger, and TaskFlow primitives, but it still lacks the requested general model-facing post-response enqueue and callback API for arbitrary work tied to the originating conversation.

## What This Changes

Request a runtime-level post-response async action queue that lets an agent acknowledge immediately, persist background work, and later deliver or resume with confirmed results.

## Best Possible Solution

Design an additive continuation contract that reuses the existing task ledger, TaskFlow/subagent yield, cron/session wake, and channel delivery infrastructure instead of introducing a parallel queue.

## Reproduction Assessment

Not applicable as a feature request. Source and docs show related primitives, but no current-main general post-response enqueue API to reproduce as broken existing behavior.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. The maintainable direction is to extend existing continuation infrastructure rather than add an independent background queue.
