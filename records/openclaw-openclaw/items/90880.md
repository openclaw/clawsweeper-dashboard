## Summary

Keep open. Current main has useful adjacent machinery for cron jobs, background task records, Task Flow, background exec, media completions, and subagent handoff, but it still does not expose the requested general post-response async enqueue and callback contract for normal agent turns.

## What This Changes

The issue requests a runtime-level way for agents to acknowledge work immediately, persist post-response async jobs, and later deliver or resume with confirmed results.

## Best Possible Solution

Design this as a maintainer-owned runtime/API extension that reuses the existing task ledger, Task Flow, cron scheduling, session wake, and channel delivery contracts instead of adding a parallel background-job stack.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers choose the API boundary. The safest direction is to extend the existing task, flow, cron, session wake, and delivery primitives rather than introduce a competing queue/callback model.
