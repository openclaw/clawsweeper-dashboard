## Summary

Keep open. Current main has adjacent primitives for task tracking, cron runs, subagent yield, and media-generation continuations, but it still lacks the requested general post-response enqueue and callback contract tied to the originating conversation.

## What This Changes

Request a runtime-level post-response async action queue that lets an agent acknowledge work immediately, execute the job after the visible reply, and deliver or resume with the confirmed result later.

## Best Possible Solution

Design this as a maintainer-owned runtime/API extension that reuses the task ledger, Task Flow, cron scheduling, subagent/session wake, and channel delivery contracts instead of adding a parallel background-job stack.

## Reproduction Assessment

Not applicable as a feature request. Source and docs show related primitives, but no current documented post-response enqueue contract that can be reproduced as broken behavior.

## Solution Assessment

Unclear until maintainers choose the API boundary. The safest direction is to extend existing task, flow, cron, and session continuation primitives rather than add a separate queue with competing callback semantics.
