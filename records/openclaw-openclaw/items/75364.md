## Summary

Keep open. The PR targets a real Feishu topic-thread routing bug and keeps the fix in the Feishu plugin boundary, but the patch is incomplete because media sends still do not carry the new thread semantics and the required user-facing changelog entry is missing.

## What This Changes

This PR updates the Feishu plugin action handler and channel tests so topic-scoped `message(action="send")` text/card sends can use the inbound message as a thread reply.

## Best Possible Solution

Land a Feishu-plugin fix that derives one topic auto-thread decision for `send` and applies it consistently to text, card, and media payloads, with focused regression coverage and a user-facing changelog entry.

## Reproduction Assessment

Yes. On current `main`, a mocked Feishu `send` action with a `group_topic` session key and `toolContext.currentMessageId` reaches the send branch with `replyInThread: false`; the PR’s text tests reproduce that path, and the remaining media gap is visible by tracing the media branch into `feishuOutbound.sendMedia`.

## Solution Assessment

Mostly yes, but incomplete. Keeping this behavior in the Feishu plugin is the narrow maintainable boundary, but the implementation should propagate the same auto-thread decision through media sends as well as text/card sends.
