## Summary

Keep open. The PR is aimed at a real Feishu topic-thread routing bug and uses the right plugin boundary, but the current patch is incomplete: it can force explicit cross-target sends back into the inbound topic, does not carry the topic-thread flag through media sends, and lacks the required changelog entry.

## What This Changes

The PR changes the Feishu message action handler to auto-thread `send` actions in `group_topic` and `group_topic_sender` sessions using `toolContext.currentMessageId`, with text-path regression tests.

## Best Possible Solution

Keep the Feishu-owned fix, but make it complete before merge: auto-thread only when the `send` target is the inbound topic/chat context, propagate the same threading decision through media sends, add focused tests for those paths, and include an Unreleased changelog fix entry.

## Reproduction Assessment

Yes. Current main can be reproduced by tracing a Feishu topic-scoped `send` action to `replyInThread: false`; the PR’s text tests cover part of that path, while cross-target and media regressions are visible by tracing the new `replyToMessageId`/`sendMedia` calls.

## Solution Assessment

No, not as currently patched. The Feishu plugin is the right place to solve this, but the implementation should respect explicit targets and apply the same topic-threading semantics to text, card, and media sends.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Guard auto-threading to the inbound target:** `extensions/feishu/src/channel.ts:717-723`
  - body: The new auto-thread id is applied to every `send` action in a topic-scoped session without checking the resolved `to` target. Since `sendMessageFeishu` uses the reply API whenever `replyToMessageId` is set, an explicit send to another Feishu target from a topic session would be posted back to the inbound topic instead of the requested destination. Restrict auto-threading to the current topic/chat target and leave explicit cross-target sends unthreaded.
  - confidence: 0.84
- **[P2] Pass topic threading into media sends:** `extensions/feishu/src/channel.ts:768-776`
  - body: The computed `replyInThread` flag is only wired into card and text sends. The media branch still calls `sendMedia` with `replyToId` only, and the Feishu outbound adapter drops the thread flag before `sendMediaFeishu`, so media sent from a topic-scoped `send` action can still fall out of the Feishu topic.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `extensions/feishu/src/channel.ts:714`
  - body: This is a user-facing Feishu bug fix, but the PR does not update `CHANGELOG.md`. Add an Unreleased `### Fixes` entry with appropriate contributor credit before merge.
