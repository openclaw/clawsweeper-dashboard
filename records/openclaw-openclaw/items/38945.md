## Summary

Keep open. The Unicode tokenizer bug is still source-reproducible on current main, but this PR cannot merge as-is because it edits removed legacy `src/memory` files instead of the active `memory-core` runtime and still lacks inspectable real behavior proof.

## What This Changes

The PR updates legacy `src/memory` MMR and FTS tokenizers/tests to add Unicode letter/mark/number matching with NFC normalization.

## Best Possible Solution

Port the Unicode/NFC handling into `extensions/memory-core` FTS query building, search planning, and MMR tokenization while preserving CJK unigrams/bigrams, then add focused regressions and redacted real behavior proof.

## Reproduction Assessment

Yes, at source level. Current `memory-core` FTS/search-planner regexes omit combining marks and NFC normalization, and the active MMR tokenizer still drops non-CJK Unicode letters such as Finnish diacritics from similarity tokens.

## Solution Assessment

No. The Unicode/NFC direction is right, but the proposed patch edits removed legacy paths and would need to be ported to active `memory-core` without losing current CJK unigram/bigram behavior.
