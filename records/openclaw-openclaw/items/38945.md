## Summary

Keep open: the Unicode memory-search bug still appears valid on current main, but the branch edits removed legacy memory files instead of the active memory-core plugin and lacks inspectable real behavior proof.

## What This Changes

The branch modifies legacy `src/memory` MMR and FTS tokenizer code and tests to use Unicode letter/mark/number token patterns with NFC normalization.

## Best Possible Solution

Port Unicode mark and NFC handling into `extensions/memory-core` FTS query building, search planning, and MMR tokenization while preserving CJK unigram/bigram behavior and adding focused regressions plus redacted runtime proof.

## Reproduction Assessment

Yes, at source level. Current memory-core FTS/search-planner patterns omit `\p{M}` and NFC normalization, and a regex probe shows combining-mark scripts split while MMR still lacks non-CJK Unicode word tokens.

## Solution Assessment

No. The Unicode/NFC direction is right, but the patch edits removed legacy `src/memory` files; the maintainable fix belongs in active `memory-core` and must preserve CJK token granularity.
