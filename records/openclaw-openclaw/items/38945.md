## Summary

Keep open: the Unicode tokenizer bug is still valid on current main, but this PR is stale because it edits removed legacy `src/memory` helpers instead of the active `memory-core` plugin paths. The useful source work should be ported or replaced, with the current CJK MMR behavior preserved.

## What This Changes

The PR changes legacy memory tokenizer helpers/tests under `src/memory` to NFC-normalize text and tokenize Unicode letters, marks, numbers, and underscores for MMR and FTS queries.

## Best Possible Solution

Land a narrow `memory-core` repair that normalizes before tokenization, adds Unicode letter/mark/number tokens for non-CJK scripts, preserves current CJK MMR tokens, updates all FTS planner token paths consistently, and adds focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes, as a source-level reproduction. Current-main regexes in active `memory-core` omit combining marks/NFC in FTS paths and omit general Unicode word tokens in MMR, and a read-only Node probe shows Thai/Hindi fragmentation and Finnish MMR token loss.

## Solution Assessment

No, not as currently proposed. The tokenizer direction is correct, but the patch edits removed legacy files and would need to be ported into active `memory-core` while preserving CJK-specific token behavior.
