## Summary

Keep open. The Unicode/NFC tokenizer bug is still valid, but this PR is stale against current main: active memory search now lives in the `memory-core` plugin, while this branch edits deleted legacy `src/memory` files. The right path is a narrow port/replacement, not closing or merging this patch as-is.

## What This Changes

This PR updates legacy `src/memory` MMR and hybrid FTS tokenizer helpers/tests to NFC-normalize input and match Unicode letters, marks, numbers, and underscores.

## Best Possible Solution

Land a narrow memory-core repair that normalizes before tokenization, treats Unicode letters/marks/numbers consistently in FTS and fallback planning, extends MMR for non-CJK scripts, preserves existing CJK unigram/bigram behavior, and adds focused regressions plus the required changelog entry.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: current active regexes still fragment NFD and combining-mark scripts in FTS paths, and active MMR still only covers ASCII plus CJK-specific tokens rather than general Unicode word/mark tokens.

## Solution Assessment

No, not as currently proposed. The tokenizer direction is right, but the maintainable fix belongs in `extensions/memory-core` and the shared search planner, not in the deleted legacy `src/memory` files.
