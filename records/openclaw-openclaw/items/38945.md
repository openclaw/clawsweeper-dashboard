## Summary

Keep open: the bug is still source-reproducible on current main, but this branch targets legacy `src/memory` files that are no longer the active memory runtime, so it needs a memory-core port and real behavior proof before merge.

## What This Changes

The PR changes legacy `src/memory` MMR/FTS tokenizers and tests to add Unicode letter/mark/number matching with NFC normalization.

## Best Possible Solution

Port the Unicode/NFC handling into `extensions/memory-core` FTS query building, search planning, and shared MMR tokenization while preserving CJK unigrams/bigrams, then add focused regressions and redacted real behavior proof.

## Reproduction Assessment

Yes, at source level. Current `memory-core` FTS/search-planner regexes omit combining marks and NFC normalization, and the active MMR tokenizer still drops non-CJK Unicode letters such as Finnish diacritics from similarity tokens.

## Solution Assessment

No. The Unicode/NFC direction is right, but the proposed patch edits removed legacy paths and would need to be ported to active `memory-core` without losing current CJK unigram/bigram behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95
