The PR updates legacy memory tokenizer code and tests under `src/memory` to use Unicode-aware token matching and NFC normalization for MMR and FTS queries.

## Best Possible Solution

Port Unicode mark and NFC handling into active memory-core FTS, search-planning, and MMR tokenization while preserving CJK unigram/bigram behavior, then add focused regressions and redacted runtime proof.

## Reproduction Assessment

Yes, at source level. Current memory-core regexes still split combining-mark scripts and NFD input, and the active MMR tokenizer still drops or splits non-CJK non-ASCII words while preserving CJK-specific tokens.

## Solution Assessment

No. The Unicode/NFC direction is right, but the patch targets legacy files and needs to be ported to active memory-core without replacing the CJK token granularity.
