## Summary

Keep open: the Unicode/NFC bug is still credible, but this PR edits obsolete legacy memory files and would not fix the active memory-core runtime on current main.

## What This Changes

The PR changes legacy `src/memory` MMR and FTS tokenizers/tests to add Unicode letter/mark/number tokenization and NFC normalization.

## Best Possible Solution

Port Unicode mark and NFC handling into active `extensions/memory-core` FTS, search-planning, and MMR tokenization while preserving CJK unigram/bigram behavior, then add focused regressions and redacted runtime proof.

## Reproduction Assessment

Yes, at source level. Current memory-core regexes still split combining-mark scripts and NFD input, and the active MMR tokenizer still drops or splits non-CJK non-ASCII words while preserving CJK-specific tokens.

## Solution Assessment

No. The Unicode/NFC direction is right, but this patch targets legacy `src/memory` files and should be ported to `extensions/memory-core` without replacing the active CJK token granularity.
