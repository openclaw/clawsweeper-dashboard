## Summary

Keep open: the Unicode/NFC bug is valid, but this branch edits legacy `src/memory` files that are no longer the active memory runtime, so the useful fix still needs to be ported to `memory-core` and backed by inspectable real behavior proof.

## What This Changes

The PR updates legacy memory MMR and FTS tokenizer code and tests to use Unicode letter/mark/number tokenization with NFC normalization.

## Best Possible Solution

Port Unicode mark and NFC handling into `extensions/memory-core` FTS/search-planning/MMR while preserving CJK unigram/bigram tokenization, then add focused regressions and redacted runtime proof.

## Reproduction Assessment

Yes, at source level. The active FTS/search-planning regexes split combining-mark scripts and NFD input, and the active MMR tokenizer still fragments non-CJK Unicode words.

## Solution Assessment

No. The Unicode/NFC direction is right, but this patch targets removed legacy files and would not preserve the active memory-core CJK token contract if ported directly.
