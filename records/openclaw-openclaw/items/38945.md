## Summary

Keep open: the Unicode tokenizer problem is still valid, but this PR edits legacy `src/memory` files that are no longer the active memory implementation, so it should not be merged as-is.

## What This Changes

The PR changes legacy `src/memory` MMR and FTS tokenizer helpers/tests to use Unicode property regexes and NFC normalization.

## Best Possible Solution

Port Unicode/NFC tokenization into active `memory-core` FTS, MMR, and search-planner paths while preserving CJK unigram/bigram behavior, then add focused tests and redacted real behavior proof.

## Reproduction Assessment

Yes, source-level: current `memory-core` FTS and search-planner regexes omit `\p{M}`/NFC, and a read-only Node probe shows Thai/Hindi fragmentation plus NFD accent loss. I did not run a full `openclaw memory` command.

## Solution Assessment

No. The tokenizer direction is right, but this branch edits legacy/deleted `src/memory` files; the maintainable fix belongs in active `extensions/memory-core` paths and must preserve current CJK matching semantics.
