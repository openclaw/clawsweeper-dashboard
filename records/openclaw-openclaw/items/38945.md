## Summary

Keep open: the Unicode tokenizer bug still exists in active memory-core, but this PR targets removed legacy src/memory files and a direct MMR port would regress current CJK token behavior.

## What This Changes

The PR updates legacy src/memory MMR and FTS tokenizer helpers/tests to NFC-normalize input and match Unicode letters, marks, numbers, and underscores.

## Best Possible Solution

Port the Unicode/NFC tokenizer fix into `extensions/memory-core` FTS/MMR paths, preserve CJK unigram/bigram MMR tokens, update shared search-planner tokenization, and add focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes, as a source-level reproduction. Active memory-core regexes still omit combining marks/NFC in FTS paths and general Unicode word tokens in MMR, and a read-only Node probe shows Thai/Hindi-style fragmentation plus Finnish/Thai MMR token loss.

## Solution Assessment

No. The tokenizer direction is right, but the patch targets removed legacy `src/memory` files; the maintainable fix belongs in active `extensions/memory-core` and must preserve existing CJK MMR behavior.
