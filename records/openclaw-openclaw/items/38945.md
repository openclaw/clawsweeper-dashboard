## Summary

Keep open: the Unicode tokenizer bug still exists in current `memory-core`, but this PR is stale against current main because it edits deleted legacy `src/memory` paths and would lose current CJK MMR behavior if ported literally.

## What This Changes

The PR updates legacy `src/memory` MMR and hybrid FTS tokenizer helpers/tests to NFC-normalize input and match Unicode letters, marks, numbers, and underscores.

## Best Possible Solution

Land a narrow `memory-core` repair that normalizes before tokenization, adds Unicode letter/mark/number tokens for non-CJK scripts, preserves CJK unigram/bigram MMR behavior, updates FTS planner paths consistently, and includes focused regressions plus a changelog entry.

## Reproduction Assessment

Yes, as a source-level reproduction. Current main's active tokenizers still omit combining marks/NFC in FTS paths and only support ASCII plus CJK-specific MMR tokens, and a read-only Node probe demonstrates the token fragmentation.

## Solution Assessment

No, not as currently proposed. The tokenizer direction is correct, but the patch needs to move from deleted legacy `src/memory` files into `extensions/memory-core` and the shared search planner while preserving current CJK behavior.
