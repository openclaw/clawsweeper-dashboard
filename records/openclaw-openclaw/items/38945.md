## Summary

Keep open: the reported Unicode memory-tokenization bug is still present in current main, but this PR edits legacy `src/memory` files that are no longer the active runtime path and it lacks inspectable real behavior proof.

## What This Changes

The PR changes legacy memory MMR and FTS tokenizer code and tests to use Unicode letter/mark/number patterns with NFC normalization.

## Best Possible Solution

Port Unicode mark and NFC handling into the active `memory-core` FTS/search-planning and MMR paths, preserve the current CJK token contract, add focused regressions, and require redacted runtime proof before merge.

## Reproduction Assessment

Yes, at source level. Current memory-core FTS/search-planner regexes omit `\p{M}` and NFC normalization, and a focused Node probe shows combining-mark scripts split while the active MMR tokenizer still fragments non-CJK Unicode words.

## Solution Assessment

No. The Unicode/NFC direction is right, but the patch edits removed legacy `src/memory` files and a direct port would lose the active memory-core CJK unigram/bigram behavior.
