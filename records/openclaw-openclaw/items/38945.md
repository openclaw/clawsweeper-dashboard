## Summary

Keep open, but do not merge as-is: the reported Unicode memory bug is still real in current memory-core, while this branch edits removed legacy paths and would miss the active FTS/search-planning runtime. A direct port also needs to preserve the current CJK unigram/bigram MMR contract and still needs inspectable real behavior proof.

## What This Changes

The PR changes legacy `src/memory` MMR and FTS tokenizers and tests to add Unicode letter/mark/number matching with NFC normalization.

## Best Possible Solution

Port Unicode mark and NFC handling into active `extensions/memory-core` FTS, search-planning, and MMR tokenization while preserving CJK unigram/bigram behavior, then add focused regressions and redacted runtime proof.

## Reproduction Assessment

Yes, at source level. Current memory-core regexes still split or drop combining-mark/NFD and non-CJK Unicode input, and a read-only Node probe against those regexes reproduced the tokenization failure.

## Solution Assessment

No. The Unicode/NFC direction is right, but this patch targets legacy `src/memory` files and should be ported to `extensions/memory-core` without replacing the active CJK token granularity.
