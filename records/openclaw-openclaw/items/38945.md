## Summary

Keep open: current main still has the Unicode and combining-mark tokenizer bug in the active memory-core paths, but this branch edits obsolete legacy files and would not fix the shipped memory plugin as-is. A literal MMR port would also lose existing CJK unigram/bigram behavior, and the contributor still needs to add after-fix real behavior proof.

## What This Changes

The PR updates legacy `src/memory` MMR and FTS tokenizer helpers/tests to NFC-normalize text and match Unicode letters, marks, numbers, and underscores.

## Best Possible Solution

Port Unicode/NFC tokenization into `extensions/memory-core` FTS/MMR and planner paths, preserve current CJK unigram/bigram behavior, and add focused regression tests plus a changelog entry.

## Reproduction Assessment

Yes, as a source-level reproduction rather than a full live memory flow. Current memory-core regexes still omit combining marks/NFC in FTS paths, and the current MMR tokenizer still loses non-CJK Unicode terms while preserving CJK unigrams/bigrams.

## Solution Assessment

No. The tokenizer direction is right, but the patch targets obsolete `src/memory` files; the maintainable fix belongs in active `extensions/memory-core` paths and must preserve existing CJK behavior.
