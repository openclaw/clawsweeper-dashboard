## Label Justifications

- `P1`: The report describes a published Docker gateway restart loop that would break real container deployments if it were still current.
- `impact:crash-loop`: The reported symptom is a gateway restart loop before the container reaches a healthy state.

## Summary

Current main and the currently published `latest` image no longer use the failing Pi package shape: OpenClaw is pinned to `@earendil-works/pi-coding-agent@0.75.4`, that package imports `minimatch` by name, and GHCR `latest` now points at the v2026.5.20 image rather than the reported v2026.4.24 digest.

## What This Changes

The issue reports that recent `ghcr.io/openclaw/openclaw:latest` images restart-loop because bundled Pi code default-imports `minimatch` while minimatch v10 exposes named ESM exports.

## Best Possible Solution

Keep the v2026.5.20-plus Pi runtime dependency path and avoid adding an image rewrite workaround for a stale digest.

## Reproduction Assessment

No for current main/current `latest`: the registry now points `latest` at v2026.5.20, and the pinned Pi package uses a named `minimatch` import. The older digest mentioned in the report resolves to v2026.4.24, not the current tag.

## Solution Assessment

Yes; updating the owned Pi runtime package to a version that matches minimatch v10's ESM contract is the maintainable fix, and it is already in the released image path.
