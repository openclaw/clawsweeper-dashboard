## Summary

Keep open: current main and v2026.6.10 still miss the dropped-mention path, so a Feishu reply @mention can be rejected before dispatch in mention-required groups. The reply-target half is now scope-aware for normal groups, so the remaining fix should preserve root/thread facts instead of clearing them during parsing.

## What This Changes

The issue reports that Feishu reply messages can fail bot mention detection and can route an explicit reply mention into a thread instead of the main group chat.

## Best Possible Solution

Keep this issue open as the canonical tracker and land or adopt a Feishu-local fix that detects bot-specific dropped reply mentions while preserving root/thread context for existing scope-aware reply targeting.

## Reproduction Assessment

Yes, at source level. A Feishu group reply payload with a non-empty mentions array that omits the bot leaves mentionedBot false, and mention-required group ingress can return before dispatch.

## Solution Assessment

No, not as originally proposed. The fix belongs in the Feishu plugin, but parse-time rootId clearing is less maintainable than preserving root/thread facts and deciding the reply target after group/topic scope is known.
