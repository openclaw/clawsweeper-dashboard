## Summary

Keep open: current main still has a source-level Feishu mention-detection gap, and the linked closing PR is open with unresolved review/proof concerns, so this issue should not be closed before that fix is landed or replaced.

## What This Changes

Detect Feishu bot @-mentions in reply messages even when Feishu omits the bot from the structured mentions array, and avoid unwanted thread routing for those replies.

## Best Possible Solution

Land a scoped Feishu-plugin fix that detects dropped bot mentions with bot-specific evidence and chooses the reply target after group session scope is known, preserving topic-mode root handling.

## Reproduction Assessment

Yes, at source level. A Feishu text reply with `parent_id`/`root_id`, an orphan `@_user_N` placeholder in text, and structured mentions that omit `botOpenId` leaves `mentionedBot=false`, so mention-required group ingress returns before dispatch.

## Solution Assessment

No, not as currently proposed in the linked PR. The fix belongs in the Feishu plugin, but it should avoid any-orphan mention inference and defer reply-target decisions until routing knows whether the group is normal or topic-scoped.
