## What This Changes

The issue requests Feishu reply-message @bot detection when Feishu preserves an @ placeholder but omits the bot from the structured mentions array, plus main-chat reply targeting for normal group replies.

## Best Possible Solution

Land or replace `https://github.com/openclaw/openclaw/pull/39569` with a Feishu-local fix that uses bot-specific dropped-mention evidence and chooses reply targets after group session scope is known.

## Reproduction Assessment

Yes, at source level. A Feishu group reply with a non-empty `mentions` array that omits `botOpenId` leaves `mentionedBot=false`, then mention-required group ingress can return before dispatch.

## Solution Assessment

No, not exactly as proposed. The fix belongs in the Feishu plugin, but broad parse-time `rootId` clearing and any-orphan placeholder detection are riskier than bot-specific mention evidence plus scope-aware routing.
