## Best Possible Solution

Implement a Feishu-local replacement fix against current `bot-content.ts` and `bot.ts`: detect the dropped bot mention from bot-specific evidence or a tightly constrained Feishu reply payload shape, then preserve root/thread context until the existing scope-aware reply routing chooses the target.

## Solution Assessment

No, the issue's proposed parse-time `rootId` clearing is not the best full fix on current main. The safer fix is bot-specific dropped-mention detection plus preserving root/thread context until the existing scope-aware routing decides normal group versus topic behavior.
