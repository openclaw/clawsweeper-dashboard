## Summary

Keep open. Current main still has the Feishu reply-mention source path reported here, and the linked implementation PR remains open with closing syntax, so this issue should not be closed ahead of that fix.

## What This Changes

Feishu should detect bot @mentions in reply messages even when Feishu omits the bot from structured mention metadata, and normal group reply mentions should not be routed into a topic thread.

## Best Possible Solution

Review and land PR #39569 or an equivalent Feishu plugin change with regression coverage for orphaned reply mention placeholders and normal-group reply routing, while preserving explicit topic/thread replies.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence synthetic path: a Feishu group reply event with `root_id`, a non-empty `mentions` array that omits the bot, and text containing the bot placeholder will leave `mentionedBot=false` and be dropped by the mention gate; the same `root_id` also drives `replyInThread=true`.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Feishu plugin: add fallback detection for orphaned reply mention placeholders and suppress unintended thread routing for explicit normal-group @mentions without changing configured topic/thread behavior.
