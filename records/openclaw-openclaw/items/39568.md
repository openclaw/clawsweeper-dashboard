Fix Feishu group replies that @mention the bot so they are detected even when Feishu omits the bot from structured mentions, and route replies to the intended main chat or thread.

## Best Possible Solution

Implement a Feishu-local fix in the current bot-content/bot routing split: use bot-specific dropped-mention evidence, then let the existing scope-aware reply target logic decide main-chat versus topic replies.

## Solution Assessment

No, not exactly as originally proposed. The fix belongs in the Feishu plugin, but broad parse-time rootId clearing should be avoided in favor of bot-specific mention evidence and scope-aware reply routing.
