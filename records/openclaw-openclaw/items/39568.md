## Summary

Keep open: current main still misses the reported Feishu reply-mention shape, and the linked closing PR remains open and conflicting rather than merged. Current main already handles the normal-group reply-target part, but the mention false negative can still block dispatch when `requireMention` is enabled.

## What This Changes

The issue reports that Feishu group reply messages with an @-mention can be missed when Feishu drops the bot from the structured mentions array, and that replies can be routed into a thread.

## Best Possible Solution

Land or replace the linked Feishu plugin fix with bot-specific dropped-mention detection and reply-target selection after group session scope is known, preserving topic-mode routing.

## Reproduction Assessment

Yes, at source level. A Feishu group reply with a non-empty `mentions` array that omits `botOpenId` leaves `mentionedBot=false`, then mention-required group ingress can return before dispatch.

## Solution Assessment

No, not exactly as proposed. The fix belongs in the Feishu plugin, but broad parse-time `rootId` clearing and any-orphan placeholder detection are riskier than bot-specific mention detection plus routing after session scope is known.
