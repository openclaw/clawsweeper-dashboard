## Summary

Keep open. Current main still has the reported Feishu reply-mention gap, and an open implementation PR with closing syntax already tracks the fix, so the issue should remain open until that PR or an equivalent patch lands.

## What This Changes

Feishu group reply messages should detect explicit bot @mentions even when Feishu omits the bot from structured mention metadata, and replies should stay in the main group chat unless topic/thread mode applies.

## Best Possible Solution

Review, rework, and land https://github.com/openclaw/openclaw/pull/39569 or an equivalent Feishu plugin patch with focused regression coverage for dropped bot mention metadata and scope-aware reply routing.

## Reproduction Assessment

Yes, by source inspection. A Feishu group reply with root_id, text containing the bot placeholder, and a non-empty mentions array that omits the bot leaves mentionedBot=false and is dropped by the requireMention gate on current main.

## Solution Assessment

Yes on direction, but the issue should not close until the linked PR or an equivalent owner-local Feishu plugin fix lands. The safest shape is bot-specific dropped-mention detection plus reply targeting that preserves topic and explicit thread modes.
