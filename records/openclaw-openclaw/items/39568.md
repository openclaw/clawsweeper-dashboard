## Summary

Keep open: current main still has the Feishu reply-mention false negative at source level, and the linked closing PR remains open rather than merged.

## What This Changes

The issue asks the Feishu plugin to detect bot @mentions omitted from structured reply-message mentions and avoid unintended thread routing for normal group replies.

## Best Possible Solution

Land or replace the linked Feishu plugin fix with bot-specific dropped-mention detection and reply-target selection after group session scope is known, while preserving topic-mode routing.

## Reproduction Assessment

Yes, at source level. A Feishu group reply with a non-empty `mentions` array that omits `botOpenId` leaves `mentionedBot=false`, then mention-required group ingress can return before dispatch.

## Solution Assessment

No, not exactly as proposed. The fix belongs in the Feishu plugin, but parse-time `rootId` clearing is too broad for topic-mode groups and dropped-mention detection should be bot-specific.
