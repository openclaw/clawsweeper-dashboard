## Summary

Keep open: current main still has the Feishu reply-mention false negative, and the linked closing PR is open rather than merged.

## What This Changes

Feishu should detect bot @mentions in group reply messages even when Feishu omits the bot from the structured mentions array, then reply in the intended main chat context.

## Best Possible Solution

Land or replace the Feishu plugin fix with bot-specific dropped-mention detection and reply-target selection after group session scope is known, preserving topic-mode routing.

## Reproduction Assessment

Yes, at source level. A Feishu group reply with a non-empty `mentions` array that omits `botOpenId` leaves `mentionedBot=false`, then mention-required group ingress returns before dispatch.

## Solution Assessment

No, not exactly as proposed. The fix belongs in the Feishu plugin, but broad parse-time `rootId` clearing would risk topic routing; reply target choice should happen after session scope is known.
