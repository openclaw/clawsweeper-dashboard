## Summary

Keep open. Current main still has the reported Feishu reply-mention gap, and there is an open unmerged implementation PR with closing syntax for this issue, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Feishu reply @mentions should be detected even when Feishu omits the bot from structured mentions, and normal group reply @mentions should answer in the main chat instead of being forced into a topic thread.

## Best Possible Solution

Keep the issue open until PR #39569 or an equivalent Feishu plugin change lands. The end state should add an orphaned reply-mention fallback, avoid forcing normal group reply @mentions into thread mode, and preserve explicit topic/thread behavior with regression tests.

## Reproduction Assessment

Yes. A focused synthetic Feishu group reply event with a non-empty `mentions` array that omits the bot but text containing the bot placeholder should make `checkBotMentioned()` return false, and a reply event carrying `root_id` should set `threadReply` and force `replyInThread`.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Feishu plugin: adjust mention detection and root/thread handling there, with tests for both the orphaned mention case and the normal-group reply-routing case.
