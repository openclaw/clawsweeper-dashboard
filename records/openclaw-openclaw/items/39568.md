## Summary

Keep open: current main and v2026.5.20 still short-circuit Feishu mention detection when a non-empty `mentions` array omits the bot, and the linked implementation PR remains open and conflicting rather than merged.

## What This Changes

The issue asks the Feishu channel to detect bot @mentions in reply messages when Feishu drops the bot from `mentions`, and to avoid routing those replies into unintended topic threads.

## Best Possible Solution

Land or replace the linked Feishu-local PR with bot-specific dropped-mention detection while preserving root context until scope-aware reply routing chooses the target.

## Reproduction Assessment

Yes, at source level. A Feishu group reply whose non-empty `mentions` array omits the bot open_id leaves `mentionedBot=false`, and mention-required group ingress can return before dispatch.

## Solution Assessment

No, not as originally proposed. The fix belongs in the Feishu plugin, but broad parse-time `rootId` clearing should be avoided in favor of bot-specific mention evidence and the existing scope-aware reply routing.
