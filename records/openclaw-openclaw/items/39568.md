## Summary

Keep open. Current main still misses the Feishu reply-mention case when the structured `mentions` array is non-empty but omits the bot, and the linked closing PR is still open, so this issue should stay open until that fix or a replacement lands.

## What This Changes

A Feishu group reply that @-mentions the bot should be detected even when Feishu omits the bot from structured mentions, and the response should avoid unwanted topic-thread routing.

## Best Possible Solution

Review, rework, and land https://github.com/openclaw/openclaw/pull/39569 or an equivalent Feishu plugin patch with focused tests and redacted live Feishu proof.

## Reproduction Assessment

Yes, at source level. A Feishu group reply with `root_id`, text containing an omitted bot placeholder, and a non-empty `mentions` array that lacks `botOpenId` makes `checkBotMentioned()` return false, then the require-mention gate returns before dispatch.

## Solution Assessment

Yes for keeping this issue open while the linked fix path is reviewed. The maintainable solution belongs in the Feishu plugin and should combine bot-specific dropped-mention detection with scope-aware reply routing.
