## Summary

Keep open: current main still misses the Feishu reply-mention case, and the linked closing PR is open with proof/review blockers, so this should wait for that fix to land or be replaced.

## What This Changes

Feishu group reply messages that @-mention the bot should trigger the bot even when Feishu omits the bot from structured mentions, and normal group replies should stay in the main chat.

## Best Possible Solution

Land a scoped Feishu-plugin fix that uses bot-specific dropped-mention evidence and chooses reply targets after group session scope is known, preserving topic-mode root handling.

## Reproduction Assessment

Yes, at source level. A Feishu group reply with a non-empty `mentions` array that omits `botOpenId` leaves `mentionedBot=false`, so mention-required group ingress returns before dispatch.

## Solution Assessment

No, not exactly as proposed. The fix belongs in the Feishu plugin, but parse-time `rootId` clearing and any-orphan mention inference are broader than the narrow maintainable repair.
