## Summary

Keep open: current main still has the Feishu mention-detection gap, and the open closing PR has unresolved proof/review concerns, so this issue should stay open until that fix lands or is replaced.

## What This Changes

Fix Feishu group reply handling so an explicit bot @-mention in a reply is detected even when Feishu omits the bot from `mentions`, without misrouting normal group replies into topic threads.

## Best Possible Solution

Land a scoped Feishu-plugin fix that detects dropped bot mentions with bot-specific evidence and chooses reply targets after group session scope is known, preserving topic-mode root handling.

## Reproduction Assessment

Yes, at source level. A Feishu group reply with an orphan bot placeholder in text and a non-empty `mentions` array that omits `botOpenId` leaves `mentionedBot=false`, so mention-required group ingress returns before dispatch.

## Solution Assessment

No, not as currently proposed in the linked PR. The fix belongs in the Feishu plugin, but it should avoid any-orphan mention inference and defer reply-target decisions until routing knows whether the group is normal or topic-scoped.
