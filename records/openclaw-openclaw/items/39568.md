Keep open. Current main still has the Feishu reply-mention source path, and an open implementation PR with closing syntax already tracks the fix, so this issue should remain open until that PR or an equivalent patch lands.

## What This Changes

The issue requests Feishu reply-message @mention detection when Feishu omits the bot from structured mentions, plus main-chat reply targeting instead of unintended topic-thread routing.

## Best Possible Solution

Review, rework, and land https://github.com/openclaw/openclaw/pull/39569 or an equivalent Feishu plugin patch with focused regression coverage for orphaned reply mention placeholders and scope-aware reply routing.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Feishu group reply with root_id, text containing the bot placeholder, and a non-empty mentions array that omits the bot leaves mentionedBot=false and is dropped by the requireMention gate.

## Solution Assessment

Yes on direction, but not necessarily the current PR shape. The fix belongs in the Feishu plugin and should add the strongest available bot-specific dropped-mention detection while preserving explicit topic and configured thread behavior.
