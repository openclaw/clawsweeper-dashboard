## What This Changes

The issue asks Feishu to detect bot @-mentions in group reply messages when Feishu omits the bot from the structured mentions array, and to avoid routing those explicit replies into an unintended topic thread.

## Best Possible Solution

Land or replace the linked Feishu-local PR with bot-specific dropped-mention detection while preserving root context until scope-aware reply routing chooses the final target.

## Reproduction Assessment

Yes, at source level. A Feishu group reply whose non-empty `mentions` array omits the bot open_id leaves `mentionedBot=false`, and mention-required group ingress can return before dispatch.

No, not as originally proposed. The fix belongs in the Feishu plugin, but broad parse-time `rootId` clearing should be avoided in favor of bot-specific mention evidence and the existing scope-aware reply routing.
