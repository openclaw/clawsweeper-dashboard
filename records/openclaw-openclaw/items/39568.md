## What This Changes

The issue asks the Feishu plugin to detect bot @mentions in reply messages when Feishu omits the bot from `mentions`, and to avoid routing explicitly addressed normal replies into unintended topic context.

## Best Possible Solution

Land a current-main Feishu plugin fix that uses bot-specific dropped-mention evidence, preserves root/thread facts through parsing, and lets the existing scope-aware reply-target logic choose main-chat versus topic replies.

## Reproduction Assessment

Yes, at source level. A Feishu reply payload whose non-empty `mentions` array omits the bot still leaves `mentionedBot=false`, and the group ingress path can return before dispatch when mention gating is required.

## Solution Assessment

No for the old PR shape. The fix belongs in the Feishu plugin, but it should require bot-specific dropped-mention evidence and preserve root/thread context until the current routing layer chooses the reply target.
