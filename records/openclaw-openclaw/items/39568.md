## What This Changes

The issue reports that Feishu group reply messages can @-mention the bot but be ignored because mention detection trusts the structured mentions array, while reply root context can route responses into a thread.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/39569 with a Feishu-local fix that uses bot-specific dropped-mention evidence and preserves root context until scope-aware reply routing chooses the target.

## Reproduction Assessment

Yes, at source level. A Feishu group reply whose non-empty mentions array omits the bot open_id leaves mentionedBot=false, and mention-required group ingress can return before dispatch.

## Solution Assessment

No, not exactly as originally proposed. The fix belongs in the Feishu plugin, but broad parse-time rootId clearing should be avoided in favor of bot-specific mention evidence and the existing scope-aware reply routing.
