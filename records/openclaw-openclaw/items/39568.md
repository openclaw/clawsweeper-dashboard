## Label Justifications

- `P2`: This is a valid user-visible Feishu channel bug with limited blast radius, causing mentioned group replies to be ignored or routed incorrectly.
- `impact:message-loss`: When Feishu drops the bot from the structured mentions array, OpenClaw can reject an explicitly mentioned reply before dispatch.
- `impact:session-state`: Preserved `rootId` can route an explicitly mentioned reply into a topic/thread context instead of the intended main group chat.

## Summary

Keep open: current main and the latest release still retain the Feishu reply-mention false-negative path and the rootId-based reply routing path, while the exact fix remains only in an unmerged upstream PR and an open conflicting fork PR.

## What This Changes

Fix Feishu reply messages so bot @mentions are detected when Feishu omits the bot from `mentions`, and explicitly mentioned replies route back to the main chat instead of a topic thread.

## Best Possible Solution

Land or replace a narrow Feishu fix that removes the structured-mentions blind spot, adds orphaned reply mention-key coverage, and suppresses rootId only for explicit reply @mentions without regressing topic sessions.

## Reproduction Assessment

Yes, source-reproducible rather than live-reproduced here. A text reply with `parent_id`, an orphaned `@_user_N` placeholder, and a nonempty `mentions` array lacking the bot follows the early-return path in `checkBotMentioned()`, and parsed context still keeps `rootId`.

## Solution Assessment

Yes, the right layer is the Feishu parser/mention detector plus focused regression coverage. The current open fork PR is not yet the best landing vehicle because it is conflicting and still needs upstream maintainer review or replacement.
