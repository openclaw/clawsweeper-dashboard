## Summary

Keep open. Current main still has the Feishu reply-mention gap in source, and the linked implementation PR #39569 is open and unmerged, so this issue should stay open until that fix or an equivalent Feishu plugin change lands.

## What This Changes

Feishu should detect bot @mentions in reply messages even when Feishu omits the bot from `message.mentions`, and explicit reply mentions should not be forced into unwanted topic/thread routing.

## Best Possible Solution

Land PR #39569 or an equivalent Feishu plugin fix with regression coverage for orphaned reply-mention placeholders and normal-group reply routing while preserving configured topic/thread replies.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence synthetic path: a Feishu group reply with a non-bot `mentions` array, bot `@_user_N` placeholder text, `root_id`, and `requireMention=true` will set `mentionedBot=false` and drop the event; if detected, the same `root_id` path still forces thread reply mode.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Feishu plugin: adjust mention detection and reply-root handling there, then add focused regression tests that preserve explicit topic/thread behavior.
