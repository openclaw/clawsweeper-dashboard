## Summary

Keep open. Current main still drops the reported Feishu reply-mention path, and an open PR with closing syntax is already tracking the implementation.

## What This Changes

Feishu should detect explicit bot @-mentions in reply messages even when Feishu omits the bot from structured mention metadata, and should avoid unintended topic-thread replies in normal groups.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/39569 or an equivalent Feishu plugin patch with focused regression coverage and scope-aware reply routing.

## Reproduction Assessment

Yes, by source inspection. A Feishu group reply with root_id, text containing the bot placeholder, and a non-empty mentions array that omits the bot leaves mentionedBot false and is rejected by the requireMention gate.

## Solution Assessment

Yes on direction, but not necessarily the current PR shape. The fix belongs in the Feishu plugin and should use the strongest available dropped-bot-mention signal while preserving topic and explicitly threaded routing.
