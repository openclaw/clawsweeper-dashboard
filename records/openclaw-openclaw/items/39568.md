## What This Changes

The issue reports that Feishu reply messages with an explicit bot @-mention can be missed when Feishu omits the bot from `mentions`, and that reply context can route the response into the wrong thread/session target.

## Best Possible Solution

Refresh or replace the linked Feishu plugin PR so dropped bot mentions are detected narrowly while root context is preserved until scope-aware routing chooses the reply target.

## Reproduction Assessment
