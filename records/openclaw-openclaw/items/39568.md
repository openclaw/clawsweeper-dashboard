## Best Possible Solution

Rework or adopt the linked Feishu plugin fix on current main so dropped bot mentions are detected with bot-specific evidence while root/thread facts remain available for scope-aware reply routing.

## Reproduction Assessment

Yes, at source level. A Feishu group reply whose non-empty mentions array omits the bot leaves mentionedBot=false, and the mention-required group ingress can return before dispatch.

## Solution Assessment

No, not as originally proposed. The fix belongs in the Feishu plugin, but parse-time rootId clearing is less maintainable than preserving root/thread context and choosing the reply target after group/topic scope is known.
