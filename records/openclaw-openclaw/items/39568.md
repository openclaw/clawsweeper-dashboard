## Summary

Keep open: current main still false-negatives the reported Feishu reply-mention shape, and the linked closing PR is open and conflicting rather than merged. The normal-group reply-target half is already partly protected on main, but the mention gate can still block dispatch when `requireMention` is enabled.

## What This Changes

Feishu group reply @-mentions should be detected even when Feishu omits the bot from the structured mentions array, and the bot should reply in the expected main-chat context.

## Best Possible Solution

Land or replace the linked Feishu plugin fix with bot-specific dropped-mention detection and reply-target selection after group session scope is known, preserving topic-mode routing.

## Reproduction Assessment

Yes, at source level. A Feishu group reply with a non-empty `mentions` array that omits `botOpenId` leaves `mentionedBot=false`, then mention-required group ingress can return before dispatch.

## Solution Assessment

No, not exactly as proposed. The fix belongs in the Feishu plugin, but broad parse-time `rootId` clearing and any-orphan placeholder detection are riskier than bot-specific mention evidence plus routing after session scope is known.
