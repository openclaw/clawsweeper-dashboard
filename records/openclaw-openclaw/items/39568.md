## Summary

Keep open: current main and the latest release still have the source-level mention-detection gap, and an open but conflicting fork PR is a fix candidate rather than a reason to close the issue.

## What This Changes

Feishu should detect reply-message @mentions when Feishu omits the bot from structured mentions and avoid routing explicit bot replies into the wrong thread context.

## Best Possible Solution

Land or adopt a narrow Feishu plugin fix that adds orphaned placeholder detection without weakening broadcast/alias guards, and preserves topic-mode behavior while preventing explicit bot reply mentions from being hidden in the wrong thread.

Yes, the right fix belongs in the Feishu plugin parser/routing path with focused regression tests. The linked PR shape is plausible, but it needs maintainer review or adoption because the open fork PR is conflicting and not maintainer-modifiable.
