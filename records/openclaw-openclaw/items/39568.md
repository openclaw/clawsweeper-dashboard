## Summary

Keep open. Current main still has the Feishu reply-mention gap in source, and the linked implementation PR remains open and unmerged, so this issue should not be closed ahead of that fix.

## What This Changes

The issue asks Feishu to detect bot @mentions in reply messages when Feishu omits the bot from `mentions`, and to avoid routing those normal group replies into a topic thread.

## Best Possible Solution

Land PR #39569 or an equivalent Feishu plugin change with regression coverage for orphaned reply-mention placeholders and normal-group reply routing, while preserving configured topic/thread replies.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence synthetic path: a Feishu group reply event with a non-bot `mentions` array, bot `@_user_N` text placeholder, and `root_id` will set `mentionedBot=false` and `threadReply=true` on current main.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Feishu plugin: adjust mention detection and root/thread handling there, then add focused regression tests; the open PR is the current candidate but needs normal review/rebase before closure.
