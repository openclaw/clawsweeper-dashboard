## Summary

Keep open: current main and v2026.5.12 still miss the source-level Feishu reply-mention shape, and the linked closing pull request remains open and conflicting rather than merged.

## What This Changes

The issue asks Feishu to detect @bot mentions in reply messages when Feishu omits the bot from `mentions[]`, and to avoid sending normal-group replies into topic threads.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/39569 with a Feishu-local fix that uses bot-specific dropped-mention evidence and selects reply targets after group session scope is known.

## Reproduction Assessment

Yes, at source level. A Feishu group reply with a non-empty `mentions` array that omits `botOpenId` leaves `mentionedBot=false`, then mention-required group ingress can return before dispatch.

## Solution Assessment

No for the currently proposed shape. The fix belongs in the Feishu plugin, but broad parse-time `rootId` clearing and any-orphan placeholder detection should be replaced with bot-specific mention evidence plus scope-aware routing.
