## Summary

Keep open. Current main still has the Feishu reply-mention drop path, and an open linked implementation PR is already tracking the fix but has unresolved review concerns.

## What This Changes

Feishu should detect bot @mentions in group reply messages when Feishu omits the bot from structured mentions, and avoid routing normal reply responses into an unintended topic thread.

## Best Possible Solution

Review, rework, and land https://github.com/openclaw/openclaw/pull/39569 or an equivalent Feishu plugin patch with focused regression coverage and scope-aware reply routing.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Feishu group reply with root_id, text containing the bot placeholder, and a non-empty mentions array that omits the bot leaves mentionedBot=false and is dropped by the requireMention gate.

## Solution Assessment

Yes on direction, but not necessarily the current PR shape. The fix belongs in the Feishu plugin and should use the strongest available bot-specific dropped-mention detection while preserving topic and explicit thread behavior.
