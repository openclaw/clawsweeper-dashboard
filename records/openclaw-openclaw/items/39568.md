## Summary

Keep open: current main and the latest release still miss the reported Feishu reply-mention shape, and the linked open pull request has not merged. Source inspection shows a non-empty Feishu `mentions` array that omits the bot leaves `mentionedBot=false`, which can block mention-required group dispatch.

## What This Changes

The issue reports that Feishu group reply @mentions can be missed when Feishu omits the bot from the structured mentions array, with normal-group replies expected to stay in the main chat.

## Best Possible Solution

Land or replace the linked Feishu plugin fix with bot-specific dropped-mention detection and reply-target selection after group session scope is known, preserving topic and explicit-thread routing.

## Reproduction Assessment

Yes, at source level. A Feishu group reply with a non-empty `mentions` array that omits `botOpenId` leaves `mentionedBot=false`, then mention-required group ingress can return before dispatch.

## Solution Assessment

No, not exactly as proposed. The fix belongs in the Feishu plugin, but broad parse-time `rootId` clearing and any-orphan placeholder detection are riskier than bot-specific mention evidence plus routing after session scope is known.
