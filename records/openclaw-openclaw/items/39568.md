Keep this issue open: current main still has a source-reproducible Feishu mention-gating failure, and the issue is paired with an open fix PR that uses closing syntax but has not merged.

## What This Changes

Feishu should detect bot @mentions in reply messages even when Feishu omits the bot from `mentions`, and should avoid sending explicitly addressed normal-group replies into an unintended thread.

## Best Possible Solution

Fix this in the Feishu plugin with bot-specific dropped-mention detection and reply-target selection after group session scope is known, then land or supersede the linked PR.

## Reproduction Assessment

Yes, source-level reproduction is high confidence. A Feishu group reply with a non-empty `mentions` array that lacks the bot leaves `mentionedBot=false`, and mention-required group ingress returns before dispatch.
