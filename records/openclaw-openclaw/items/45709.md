## Summary

Keep open because current main still renders both the shared shell Usage header and the local Usage view header. https://github.com/openclaw/openclaw/pull/67290 is the narrow open fix candidate, and https://github.com/openclaw/openclaw/pull/74238 overlaps with a broader header redesign, but neither has landed.

## What This Changes

The issue asks to fix the v2 Control UI Usage tab so it renders one Usage header instead of both the shared shell header and the local Usage view header.

## Best Possible Solution

Land or revise the narrow Usage-header fix, or deliberately supersede it with the broader header redesign, so the Usage tab has one canonical header owner plus regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the Usage tab gets the shared shell title/subtitle and then `renderUsage` emits a local `usage-page-header`; no browser run was needed to identify the duplicate composition path.

## Solution Assessment

Yes, the narrow maintainable fix is to remove one of the two Usage header sources without changing Usage metrics, data loading, routing, or unrelated tab layouts. Maintainers should reconcile that with the broader open header redesign before landing both.
