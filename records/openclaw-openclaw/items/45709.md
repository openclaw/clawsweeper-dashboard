## Summary

Keep this open because current main still renders both the shared shell header and the local Usage view header. Open PR #67290 targets the same duplicate-header bug, but it is not merged, so the issue should remain open until that fix or an equivalent one lands.

## What This Changes

The issue asks for the v2 Control UI Usage tab to render a single Usage header instead of duplicating the title and subtitle.

## Best Possible Solution

Land or revise #67290, or an equivalent narrow fix, so the Usage tab has one header owner plus focused regression coverage for that composition.

## Reproduction Assessment

Yes. Source inspection on current main shows the Usage route renders the shared shell title/subtitle and then `renderUsage` emits a local `usage-page-header`; I did not run a browser UI in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is to remove one of the two Usage header sources without changing Usage data loading, metrics, routing, or unrelated tab layouts; the open PR is the current implementation candidate.
