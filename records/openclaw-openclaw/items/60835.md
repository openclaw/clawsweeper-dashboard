Full review comments:

- **[P2] Use supported SecretRef syntax in the guide:** `docs/guides/multi-agent-security-ops.md:62`
  - body: The guide tells readers to use `${{ secrets.NAME }}` in config files, but current config only recognizes structured SecretRef objects or `${NAME}` env templates. Readers copying this will leave a literal string instead of configuring the SOPS exec provider.
  - confidence: 0.92
- **[P2] Return protocolVersion from the exec provider example:** `docs/guides/multi-agent-security-ops.md:95`
  - body: With `jsonOnly: true`, exec providers must emit `protocolVersion: 1` before the resolver reads `values`. This script emits only `values` in both paths, so copied providers fail with `protocolVersion must be 1`.
  - confidence: 0.94
- **[P2] Remove the unsupported docker env SecretRef claim:** `docs/guides/multi-agent-security-ops.md:148`
  - body: The guide says `sandbox.docker.env` values resolve from SecretRef at container creation, but current sandbox Docker env config is only literal `Record<string, string>` and container creation passes sanitized literal values. Operators following this will expect secret resolution that does not exist.
  - confidence: 0.88
- **[P3] Add docs-list routing metadata:** `docs/guides/multi-agent-security-ops.md:2-3`
  - body: New docs pages should include OpenClaw `summary` and `read_when` metadata for `docs:list` routing. This page only has `description` and `title`, so it misses the repository's docs discovery metadata.
  - confidence: 0.84
