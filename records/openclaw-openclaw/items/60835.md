Full review comments:

- **[P1] Keep host-exec routing out of core security guidance:** `docs/guides/multi-agent-security-ops.md:430-436`
  - body: The guide presents a custom fswatch/ops-request channel as the "only authorized channel" and lists host-side `run-script`/`read-file` actions, but current OpenClaw docs only define sandbox/tool-policy boundaries and do not implement this protocol. Publishing it as core security guidance can lead operators to add a competing host-exec boundary outside OpenClaw's audited controls; remove it or make it clearly external after security review.
  - confidence: 0.86
- **[P2] Emit protocolVersion in the exec resolver example:** `docs/guides/multi-agent-security-ops.md:95`
  - body: This resolver is configured with `jsonOnly: true` but writes only `{"values": ...}`. Current `src/secrets/resolve.ts` rejects exec-provider JSON unless `protocolVersion` is exactly `1`, so readers copying the snippet will get SecretRef resolution failures.
  - confidence: 0.93
- **[P2] Use the current SecretRef syntax:** `docs/guides/multi-agent-security-ops.md:48-62`
  - body: The guide says to use `${{ secrets.* }}` in config files, but current OpenClaw SecretRef input is either a structured `{ source, provider, id }` object or exact env shorthands like `${OPENAI_API_KEY}`. Replace these examples with supported SecretRef config so the documented setup actually resolves secrets.
  - confidence: 0.9
- **[P2] Remove the Docker env SecretRef claim:** `docs/guides/multi-agent-security-ops.md:145-150`
  - body: The guide says `sandbox.docker.env` values resolve from SecretRef at container creation, but current sandbox config types/docs treat `docker.env` as explicit string environment variables and warn that Docker metadata can expose them. This should not be documented as a SecretRef delivery path.
  - confidence: 0.82
- **[P3] Add docs-list routing metadata:** `docs/guides/multi-agent-security-ops.md:1-4`
  - body: New OpenClaw docs pages need the metadata that `scripts/docs-list.js` extracts. This page only has `description:` and `title:`, so `pnpm docs:list` will not surface its summary or read-when routing hints.
  - confidence: 0.86
- **[P2] Rebase the docs navigation before merging:** `docs/docs.json:1328-1404`
