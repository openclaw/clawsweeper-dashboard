## Summary

Keep open. Current main does not already contain an equivalent multi-agent security operations guide, but this PR is not merge-ready because its copyable SecretRef guidance conflicts with the current SecretRef syntax and exec-provider response contract.

## What This Changes

The PR adds `docs/guides/multi-agent-security-ops.md` and registers it in `docs/docs.json`, while also reformatting a large portion of the docs navigation JSON.

## Best Possible Solution

A mergeable version should use the current structured SecretRef or `${ENV_VAR}` syntax, return `{ "protocolVersion": 1, "values": ... }` from every exec-provider path, minimize `docs/docs.json` to the intended nav entry, and cross-link the existing security, secrets, sandbox, and multi-agent docs.

## Reproduction Assessment

Yes. The static reproduction is high-confidence: copy the PR's `jsonOnly: true` SOPS provider and it emits JSON without `protocolVersion: 1`, which current `parseExecValues` rejects; the documented `${{ secrets.NAME }}` syntax also does not match the current SecretRef parser.

## Solution Assessment

No. The requested docs direction may be useful, but this PR is not the best merge path until the copyable SecretRef examples match current behavior and maintainers decide the broad security-operations guide belongs in the main docs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Use supported SecretRef syntax in the guide:** `docs/guides/multi-agent-security-ops.md:62`
  - body: The guide tells readers to use `${{ secrets.NAME }}` in config files, but current SecretRef parsing only recognizes structured `{ source, provider, id }` refs or env templates matching `${NAME}`. Copying this syntax leaves a literal string instead of resolving through the configured provider, so the credential lifecycle example does not work.
  - confidence: 0.87
- **[P2] Return protocolVersion from the exec provider example:** `docs/guides/multi-agent-security-ops.md:95`
  - body: With `jsonOnly: true`, exec providers must output an object with `protocolVersion: 1` before the resolver reads `values`. This script emits only `{"values": ...}` in both the empty and normal paths, so copied SOPS providers fail with `protocolVersion must be 1`.
  - confidence: 0.93
