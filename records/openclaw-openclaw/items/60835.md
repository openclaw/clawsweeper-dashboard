Full review comments:

- **[P2] Use supported SecretRef syntax in the guide:** `docs/guides/multi-agent-security-ops.md:62`
  - body: The guide tells readers to use `${{ secrets.NAME }}` in config files, but current OpenClaw SecretRefs are structured `{ source, provider, id }` objects or `${NAME}` env templates. Copying this leaves a literal string instead of configuring the SOPS exec provider.
  - confidence: 0.94
- **[P2] Return protocolVersion from the exec provider:** `docs/guides/multi-agent-security-ops.md:80-95`
  - body: The SOPS provider config sets `jsonOnly: true`, and current resolver rejects JSON responses unless `protocolVersion` is `1`. This snippet emits only `values` in empty and normal paths, so copied providers fail before any IDs are resolved.
  - confidence: 0.95
- **[P2] Remove the unsupported docker env SecretRef claim:** `docs/guides/multi-agent-security-ops.md:148`
  - body: The guide says `sandbox.docker.env` resolves SecretRef values at container creation, but current config types and sandbox env plumbing treat Docker env as a literal string map. Operators would expect secret resolution that does not exist.
  - confidence: 0.9
- **[P3] Add docs-list routing metadata:** `docs/guides/multi-agent-security-ops.md:2-3`
  - body: New OpenClaw docs pages should carry `summary` and `read_when` metadata for docs-list routing. This page only has Mintlify `description` and `title`, so it misses repository discovery metadata used by adjacent docs.
  - confidence: 0.82
