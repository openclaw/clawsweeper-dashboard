## What This Changes

## Best Possible Solution

Land one canonical ACP projector fix that preserves the whole final_only turn-scoped buffer until the existing dispatch turn-level flush, with real ACP gateway or transport proof.

## Reproduction Assessment

Yes at source level. Current main flushes and resets final_only state on every ACP done/error while dispatch-acp.ts performs a later turn-level projector.flush(true), so pre-tool text and per-turn counters can be lost or split across tool-use boundaries.

## Solution Assessment

No. The PR preserves final text and counters, but it still drains pending tool/status deliveries on intermediate done events; the narrower maintainable fix is to preserve the whole final_only turn buffer until the existing turn-level flush, with error behavior handled explicitly.
