## What This Changes

The PR adds recovery-language matching to embedded-agent payload warning suppression and tests recovered versus unresolved command-failure wording.

## Best Possible Solution

Tighten the recovery matcher so it only suppresses warnings when the assistant clearly acknowledges the actual failed command/tool/action and add negative coverage for unrelated recovery prose.

## Reproduction Assessment

Yes, source-level. Current main lacks the recovery acknowledgement shape, and the PR-introduced failure is source-reproducible from the new regex matching generic `error ... script ... fixed/passed` prose while `resolveToolErrorWarningPolicy` uses that boolean to hide mutating warnings.

## Solution Assessment

No. Suppressing duplicate warnings for true command recovery is the right target, but this implementation is not the best fix because it treats generic task-recovery prose as acknowledgement of the actual failed mutating action.
