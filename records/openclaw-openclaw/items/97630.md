The PR lowercases normalized Windows-drive absolute paths in the shared media-core inbound path policy and adds a mixed-case Windows regression test.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes, this is the best fix location because media-core owns the shared allow-list normalization used by local media access, media understanding, iMessage, and SDK callers. Caller-specific case folding would duplicate the invariant, while making all POSIX-style paths case-insensitive would be too broad.
