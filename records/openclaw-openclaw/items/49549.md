## What This Changes

The branch adds a root and per-account WhatsApp `allowTo` config field for outbound-only target gating, with docs, tests, and partial action/cron resolver wiring.

## Best Possible Solution

Rebase onto the current WhatsApp plugin and typed delivery surfaces, settle outbound authorization semantics, wire `allowTo` through config/schema/account resolution/message actions/cron/directory, preserve group/newsletter behavior, and attach redacted live WhatsApp proof.

## Reproduction Assessment

## Solution Assessment

No, not as-is. The `allowTo` direction is plausible, but this branch targets stale paths and misses active adapter and directory flows; the maintainable fix should be rebased through the current extension-owned surfaces.
