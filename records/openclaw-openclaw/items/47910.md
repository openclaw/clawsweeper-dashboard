## Summary

Keep open. Current main has failure-class classification, profile cooldowns, and `models status` auth visibility, but it still does not implement the requested provider-level auth-broken quarantine, invalid-request fail-fast policy, or provider health/quarantine snapshot in normal `status` output.

## What This Changes

The issue requests failure-class-aware provider fallback with hard-auth provider quarantine, rate-limit/network-specific handling, invalid-request fail-fast behavior, and provider health in status output.

## Best Possible Solution

Define a documented provider/profile health model, apply hard-auth, rate-limit, network, and invalid-request classes consistently in fallback selection, and expose a redacted provider health summary in normal status while keeping deeper diagnostics in `models status`.
