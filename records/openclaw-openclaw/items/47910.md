## Summary

Keep open. Current main has a failure-class-aware foundation, profile cooldowns, and model-auth visibility, but it still does not fully implement provider-level auth-broken quarantine, invalid-request fail-fast semantics, or provider health/quarantine in normal status output.

## What This Changes

The issue requests failure-class-aware provider fallback that quarantines auth-broken providers, treats rate limits and network failures differently, fails fast on invalid requests, and exposes provider health in status output.

## Best Possible Solution

Define a documented provider/profile health model, apply hard-auth, rate-limit, network, and invalid-request policies consistently in fallback selection, and expose a redacted provider-health summary in normal status while keeping detailed auth diagnostics in models status.

## Reproduction Assessment

Not applicable as a strict bug reproduction. This is a feature/policy request, and source inspection shows current main is partial rather than a broken implementation of an already-defined contract.

## Solution Assessment

Unclear. The requested direction is reasonable, but implementing the exact table without maintainer decisions on hard-auth classification, invalid-request fail-fast behavior, and status redaction would be premature.
