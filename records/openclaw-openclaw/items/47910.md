## Summary

Keep open. Current main has useful failure-class classification, profile cooldowns, and `models status` auth visibility, but it still does not implement the requested provider-level hard-auth quarantine policy, invalid-request fail-fast behavior, or provider health/quarantine snapshot in normal status output.

## What This Changes

Request provider fallback behavior that quarantines auth-broken providers, handles rate-limit/network/invalid-request failure classes differently, and exposes provider health in status output.

## Best Possible Solution

Define a documented provider/profile health model that separates hard auth, transient refresh, rate limits, network failures, and invalid-request failures, then apply it consistently in fallback selection and expose a redacted status summary.

## Reproduction Assessment

Not applicable as a strict bug reproduction. This is a feature/policy request, and source inspection shows the current implementation is partial rather than a single broken current-main behavior with an already-defined expected result.

## Solution Assessment

Unclear. The requested direction is reasonable, but implementing the exact table without maintainer decisions on hard-auth classification, invalid-request fail-fast behavior, and status redaction would be premature.
