## What This Changes

The issue requests failure-class-aware provider fallback that quarantines auth-broken providers, treats rate limits/network/invalid-request differently, and exposes provider health in status output.

## Best Possible Solution

Keep this as the cross-provider policy thread until maintainers choose the canonical health-state owner, provider-versus-profile scope, recovery/reset triggers, primary-attempt semantics, and operator-visible status contract.
