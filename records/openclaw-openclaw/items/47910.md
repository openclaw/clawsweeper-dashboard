## Summary

Keep open. Current main has a solid failure-class-aware fallback foundation, and the closed foundation thread at https://github.com/openclaw/openclaw/issues/61328 covers part of the requested direction, but the requested provider-level auth quarantine and normal status health surface are not fully implemented and need maintainer policy choices.

## What This Changes

Request provider failover policy that classifies auth, rate-limit, network, and invalid-request failures, quarantines auth-broken providers, and exposes provider health in status output.

## Best Possible Solution

Define a documented provider/profile health model, separate hard-auth from transient refresh/rate-limit/network failures, apply it consistently in fallback selection, and expose a redacted provider health snapshot in status.
