## Summary

Keep open. Current main and v2026.5.22 have useful failure-class, profile cooldown, and models-status auth diagnostics, but they still do not provide the requested provider-wide hard-auth quarantine, full failure-class routing contract, or provider-health/quarantine state in normal status output.

## What This Changes

Request to classify provider fallback failures by class, quarantine auth-broken providers, treat transient failures differently, and expose provider health in status output.

## Best Possible Solution

Define a documented provider/profile health model, apply it consistently in fallback selection and auth-state persistence, and expose a redacted provider-health snapshot in status while leaving deeper probes in models status.

## Reproduction Assessment

Yes, at source level: current main shows failure-class classification and profile cooldown/skip foundations, while provider-wide hard-auth quarantine and normal status provider-health output remain absent. I did not replay a live Anthropic OAuth outage.

## Solution Assessment

Unclear until maintainers choose the provider-health policy. The direction is useful, but the permanent-auth boundary, Retry-After persistence, network cooldowns, invalid-request model-fallback behavior, and redacted status contract need an explicit decision.
