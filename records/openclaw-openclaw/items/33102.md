## Summary

Keep open: current main and the latest release still expose only the invocation-time `openclaw tui --deliver` flag, while adding a config-backed default would create a new public config and CLI precedence contract that needs maintainer approval.

## What This Changes

The issue requests an OpenClaw config option that can supply the default value for `openclaw tui --deliver`.

## Best Possible Solution

Define an approved opt-in TUI delivery-default config key with schema, docs, help text, focused tests, and explicit CLI override semantics that preserve off-by-default behavior unless users opt in.

## Reproduction Assessment

Yes for source-level reproduction: current main resolves omitted `--deliver` to `false`, forwards that value into TUI sends, and has no config key that can change the default. I did not run a live TUI session because this review is read-only.

## Solution Assessment

Unclear until maintainers choose the public config contract. A config-backed default matches the request, but the namespace, `--deliver`/`--no-deliver` precedence, and docs contract are product/API decisions rather than a mechanical bug fix.
