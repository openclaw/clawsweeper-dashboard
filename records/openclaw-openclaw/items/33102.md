## Summary

Keep open: current main still lacks a config-backed TUI delivery default, and the merged lifecycle fixes addressed adjacent stuck-spinner behavior without adding this config surface.

## What This Changes

Allow `openclaw tui` to read a persistent config default for the `--deliver` flag so users do not need to pass it every time.

## Best Possible Solution

Add an approved config-backed TUI delivery default with exported types, Zod schema/help/labels, docs, focused tests, and explicit `--deliver`/`--no-deliver` precedence.

## Reproduction Assessment

Yes for the current behavior. Source inspection on current main shows omitted `--deliver` becomes `false`, `sendChat` receives that static value, and config schema/types/docs expose no delivery default.

## Solution Assessment

Unclear. A config default is a plausible solution, but maintainers need to choose the public namespace and CLI/config precedence before implementation becomes mechanical.
