## Summary

Keep open: current main still lacks a config-backed TUI delivery default, and the related implementation PRs were closed unmerged. The linked stuck-spinner report adds useful severity context, but it does not make this config request implemented.

## What This Changes

Add an OpenClaw config setting that controls the default value used by `openclaw tui --deliver` when the flag is omitted.

## Best Possible Solution

Add a documented config-backed TUI delivery default with explicit CLI override precedence and regression coverage, or make an intentional maintainer decision to solve this through a different TUI delivery policy.

## Reproduction Assessment

Yes. Source inspection on current main shows `openclaw tui` still defaults delivery to false and has no config schema/type path for a delivery default; I did not run a live TUI session in this read-only review.

## Solution Assessment

Unclear. A config option is plausible and narrow, but the exact namespace, CLI precedence, Settings behavior, and relationship to the no-deliver spinner issue need maintainer direction before treating this as a mechanical bug fix.
