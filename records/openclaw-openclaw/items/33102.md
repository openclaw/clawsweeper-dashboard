## Summary

Keep open: current main and the latest release still have only the invocation-time `openclaw tui --deliver` flag, with no config-backed default. The stuck-spinner failure was handled separately; this remaining request is a public config and CLI precedence decision.

## What This Changes

Request to add a persistent OpenClaw config option that supplies the default for `openclaw tui --deliver`.

## Best Possible Solution

Define an approved TUI delivery-default config contract with schema/help/docs/tests and explicit CLI override semantics that preserve current behavior unless users opt in.

## Reproduction Assessment

Yes for source-level reproduction: current main resolves omitted `--deliver` to `false`, forwards that value into TUI sends, and has no config key that can change the default. I did not run a live TUI session because this review is read-only.

## Solution Assessment

Unclear. A config-backed default matches the request, but maintainers still need to choose the public config namespace, CLI override behavior, docs contract, and upgrade semantics before this becomes a mechanical fix.
