## Summary

Keep open: current main still only supports `--deliver` as an invocation flag, while the requested persistent config default is absent. The two related contributor PRs were closed without merge, and the closed stuck-spinner follow-up does not supersede the remaining config-contract request.

## What This Changes

Request to let users configure the default value used by `openclaw tui --deliver` so replies are delivered without passing the flag each time.

## Best Possible Solution

Add a maintainer-approved TUI delivery default config field, wire it into `openclaw tui` only when no explicit flag overrides it, and update schema, docs, tests, and changelog together.

## Reproduction Assessment

Not applicable as a feature/config request. Source inspection still gives high-confidence evidence that current main has only an invocation flag default and no persistent config key for this behavior.

## Solution Assessment

No, the current implementation is not the requested solution because it keeps delivery defaulting from CLI options only. The narrow maintainable path is a config-contract addition with explicit CLI-over-config precedence after maintainer approval of the public key name.
