## Summary

Keep open: current main still lacks a config-backed TUI delivery default. The prior config PRs are closed unmerged, and the merged lifecycle fix at https://github.com/openclaw/openclaw/pull/77199 explicitly treated this delivery-default request as separate/out of scope.

## What This Changes

Add a config-backed default for TUI delivery so users do not need to pass `openclaw tui --deliver` on every run.

## Best Possible Solution

Add an approved config-backed TUI delivery default with types, Zod schema/help, docs, focused tests, and explicit CLI override semantics.

## Reproduction Assessment

Yes. Source inspection on current main shows the CLI still defaults `--deliver` to false, forwards a boolean into `runTui`, and has no config type/schema key for a TUI delivery default.

## Solution Assessment

Unclear. A config-backed default is a plausible narrow solution, but maintainers need to choose the public namespace and precedence semantics before this is mechanical.
