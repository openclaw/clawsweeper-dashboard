## Summary

Keep this PR open. Current main still drops a deferred same-session `session.message` reload after a locally appendable final assistant payload, and still treats a visible role-less final payload as enough to suppress history reload; the PR branch addresses both focused Control UI gaps with tests and a changelog entry.

## What This Changes

This PR changes Control UI final chat-event reload logic to require an explicit renderable assistant role before suppressing reloads, replay deferred same-session `session.message` reloads after terminal events, and updates focused tests plus the changelog.

## Best Possible Solution

Land this PR or an equivalent focused Control UI patch that preserves local rendering for explicit assistant final payloads while replaying deferred same-session transcript reloads after terminal events. Keep #70348 separate because it covers Gateway history sanitization rather than this UI reload interleaving.

## Reproduction Assessment

Yes. A focused reproduction is already represented by current tests: emit `session.message` while `chatRunId` is active, then emit a final assistant payload and observe `loadChatHistory` is not replayed; separately, pass a visible role-less final payload to `shouldReloadHistoryForFinalEvent` and observe reload suppression on current main.

## Solution Assessment

Yes. The proposed fix is the narrow maintainable UI-side change because it updates the final-event reload predicate and deferred reload replay gate without coupling this behavior to the separate Gateway sanitizer path.
