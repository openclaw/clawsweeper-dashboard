## Summary

Keep this PR open. Current main still contains the deferred same-session `session.message` reload drop and still treats role-less visible final payloads as enough to suppress the safer final-event history reload; the PR branch is a focused UI/test/changelog patch for those remaining behaviors and no allowed cleanup close reason applies.

## What This Changes

This PR updates Control UI final chat-event reload logic to require an explicit renderable assistant payload before suppressing history reloads, replay deferred same-session `session.message` reloads after terminal chat events, and add regression coverage plus a changelog entry.

## Best Possible Solution

Land this PR or an equivalent focused Control UI patch that preserves final-payload local rendering while replaying deferred same-session transcript reloads and requiring an explicit assistant role before suppressing final-event history reloads. Keep #70348 separate because it covers Gateway history sanitization, not this UI reload interleaving.

## Reproduction Assessment

Yes. A focused reproduction is already represented by the current tests: emit `session.message` while `chatRunId` is active, then a final assistant payload, and observe `loadChatHistory` is not replayed; separately, pass a role-less visible final payload to `shouldReloadHistoryForFinalEvent` and observe it returns false on current main.

## Solution Assessment

Yes. The PR is the narrow maintainable fix for the observed UI behavior because it changes only the final-event reload predicate and deferred reload replay gate, with targeted regression tests and without coupling this to the separate Gateway sanitizer work.
