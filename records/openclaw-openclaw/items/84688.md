## What This Changes

The issue requests that local CLI/simple-completion inference for `google/gemini-flash-latest` stop sending legacy `thinkingBudget: -1` for adaptive thinking and use Gemini 3 thinking-level semantics instead.

## Best Possible Solution

Compose OpenClaw's shared Google thinking sanitizer into the plain pi-ai Google simple-completion path while preserving transport-aware configs, then close this issue after the fix merges.

## Reproduction Assessment

Yes, at source level: local `infer model run` reaches simple completion, current main does not wrap plain Google simple completions with the Google sanitizer, and pi-ai 0.75.4 can fall back to `thinkingBudget: -1` for `gemini-flash-latest`. I did not run a live Google request because this review is read-only and credential-dependent.

## Solution Assessment

Yes for the issue direction: the maintainable fix is to reuse provider-owned Google thinking normalization rather than adding a CLI-only special case. The open linked PR is the active candidate, but it must be reviewed and merged before this issue can close.
