## What This Changes

The issue reports that local CLI inference with `--thinking adaptive` sends `thinkingBudget: -1` for `google/gemini-flash-latest` instead of Gemini 3 thinking-level or dynamic-thinking semantics.

## Best Possible Solution

Land a narrow fix, likely via https://github.com/openclaw/openclaw/pull/84781 after repairs, that applies the shared Google thinking sanitizer to plain Google simple-completion calls while preserving transport-aware routing and Gemini 2.5 adaptive semantics.

## Reproduction Assessment

Yes. Source inspection shows the local CLI path reaches pi-ai simple completion without the Google sanitizer, and pi-ai's locked Google adapter falls back to `thinkingBudget: -1` for the reported alias.

## Solution Assessment

Yes in direction, but the current open PR still needs its transport-policy concern resolved. The maintainable solution is to reuse the existing provider-owned Google sanitizer in the local simple-completion path rather than add a CLI-only alias rule.
