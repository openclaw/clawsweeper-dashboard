## Summary

Keep open: the local `openclaw infer model run` path still delegates Google native simple-completion payload construction to `@earendil-works/pi-ai`, whose Gemini 3 Flash detection misses `gemini-flash-latest` and falls back to `thinkingBudget: -1`. Current OpenClaw has the right Google thinking sanitizer for provider-wrapped streams, but this lean local CLI path can bypass it.

## What This Changes

The issue reports that `openclaw infer model run --model google/gemini-flash-latest --thinking adaptive` sends a legacy Google `thinkingBudget` payload instead of Gemini latest alias thinking-level semantics.

## Best Possible Solution

Route local Google simple-completion model runs through the same provider-owned Google thinking normalization used by wrapped streams, while preserving Gemini 2.5 adaptive `thinkingBudget: -1` behavior.

## Reproduction Assessment

Yes from source inspection: local `model run` reaches pi-ai simple completion, and pi-ai 0.75.3 misses the `gemini-flash-latest` alias before falling back to a negative thinking budget. I did not run the live Google command because this review is read-only and credential-dependent.

## Solution Assessment

Yes: the narrow maintainable fix is to make the local simple-completion path reuse provider-owned Google thinking normalization, not add a CLI-only special case or change Gemini 2.5 semantics.
