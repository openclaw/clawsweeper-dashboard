## Summary

Keep open: current `main` still has a source-backed gap in the local `openclaw infer model run` path, where Google simple-completion payload construction can bypass OpenClaw's Google thinking sanitizer and leave `gemini-flash-latest` to pi-ai's older model detection.

## What This Changes

The issue reports that local CLI inference sends `thinkingBudget: -1` for `google/gemini-flash-latest` with adaptive thinking instead of using Gemini 3/latest `thinkingLevel` or dynamic-thinking semantics.

## Best Possible Solution

Route local Google simple-completion model runs through provider-owned Google thinking normalization so Gemini 3/latest aliases use dynamic or `thinkingLevel` semantics without changing Gemini 2.5 behavior.

## Reproduction Assessment

Yes, source-backed: local `model run` passes adaptive thinking into pi-ai simple completion, and pi-ai 0.75.3 misses `gemini-flash-latest` before falling back to `thinkingBudget: -1`. I did not run the credentialed live Google command in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to reuse provider-owned Google thinking normalization in the local simple-completion path, rather than adding a CLI-only alias workaround or changing Gemini 2.5 semantics.
