## Summary

Keep open: current main still routes the local infer model-run path through simple-completion/pi-ai without applying OpenClaw's Google thinking sanitizer, and the closing fix PR is still open rather than merged.

## What This Changes

The issue reports that local CLI inference for `google/gemini-flash-latest` with `--thinking adaptive` sends `thinkingBudget: -1` instead of Gemini 3 `thinkingLevel` or dynamic-thinking semantics.

## Best Possible Solution

Land a fix that composes OpenClaw's existing Google thinking sanitizer into the plain Google simple-completion path while preserving transport-aware Google routing and Gemini 2.5 adaptive budget semantics.

## Reproduction Assessment

Yes, at source level: local `infer model run` reaches pi-ai simple completion, and pi-ai 0.75.4 still falls back to `thinkingBudget: -1` for the `gemini-flash-latest` alias. I did not run a live Google request because this review is read-only and credential-dependent.

## Solution Assessment

Yes for the issue direction: the maintainable fix is to reuse provider-owned Google thinking normalization rather than add a CLI-only special case. The current linked PR is the active candidate, but it has not merged yet.
