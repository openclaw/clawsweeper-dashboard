## Summary

Keep this open. Current main and the latest shipped tag remain phase-only at the shared assistant text signature, OpenResponses schema, provider transport, and embedded delivery boundaries, so the requested public/internal assistant audience contract is still missing.

## What This Changes

Request to add a versioned public/internal assistant audience contract across assistant text signatures, provider/replay schemas, visible extraction, and channel delivery.

## Best Possible Solution

Define one versioned public/internal assistant audience contract in shared message content and provider/replay schemas, then enforce public-only selection at delivery-visible boundaries with a narrow legacy phase fallback.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a text block signed as {"v":1,"id":"x","phase":"final_answer","audience":"internal"} has its audience discarded by parseAssistantTextSignature and remains eligible for extractAssistantVisibleText.

## Solution Assessment

Yes at the design level, but no patch exists yet. A shared versioned audience gate is the maintainable fix; the exact compatibility and schema semantics need maintainer approval before implementation.
