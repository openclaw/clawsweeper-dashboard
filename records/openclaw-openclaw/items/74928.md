## Summary

Keep this open. Current main still has a phase-only assistant visibility contract: signatures parse `phase` but not `audience`, the shared visible extractor selects `final_answer` text without an audience gate, and delivery suppresses commentary only.

## What This Changes

The issue requests an explicit public/internal assistant audience contract at the shared visible-text and channel delivery boundaries.

## Best Possible Solution

Define one versioned `public | internal` assistant audience contract in shared message content, propagate it through provider replay/OpenResponses/history/UI projection, and enforce public-only delivery with a narrow legacy phase fallback.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a text block signed as `{ "v": 1, "id": "x", "phase": "final_answer", "audience": "internal" }` has its `audience` ignored and is selected by the current `final_answer` extractor.

## Solution Assessment

Yes for the requested direction. A shared audience contract is the maintainable fix; patching only one channel send path would leave shared extraction, replay, history, and UI surfaces undefined.
