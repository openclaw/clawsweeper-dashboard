## Summary

Keep this open. Current main still has a phase-only assistant visibility contract; it does not parse, expose, or enforce a public/internal assistant audience boundary at the shared visible-text or channel delivery layer.

## What This Changes

Request an explicit public/internal assistant audience contract so visible extraction and delivery suppress internal or invalid assistant text while preserving intentional legacy phase fallback.

## Best Possible Solution

Define one versioned assistant audience contract in shared message content, propagate it through provider parsing/replay, OpenResponses, history/UI projection, and channel delivery, then enforce public-only visible selection with a narrow documented legacy fallback.

## Reproduction Assessment

Yes. A synthetic assistant text block with `textSignature` containing `{ "v": 1, "id": "x", "phase": "final_answer", "audience": "internal" }` still follows the current final-answer visible path because the parser and extractor ignore `audience`.

## Solution Assessment

Yes. The shared-contract direction is the maintainable fix shape; a channel-only sanitizer would not protect the shared extractor, OpenResponses, replay/history projection, or UI surfaces.
