## Summary

Keep this open. Current main and the latest shipped tag remain phase-only: assistant text signatures, OpenResponses schema, shared visible extraction, and embedded delivery suppress `commentary` but do not parse or enforce a public/internal assistant audience boundary.

## What This Changes

The issue requests an explicit public/internal assistant audience contract for assistant text signatures, shared visible-text extraction, replay/schema surfaces, and channel delivery.

## Best Possible Solution

Define one versioned `public | internal` assistant audience contract in shared message content, propagate it through provider/replay/history surfaces, and enforce public-only selection at visible delivery boundaries with a narrow legacy phase fallback.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a text block signed as `{ "v": 1, "id": "x", "phase": "final_answer", "audience": "internal" }` has its audience discarded and remains eligible for final-answer extraction.

## Solution Assessment

Yes for the requested direction. A shared versioned audience gate is the narrow maintainable boundary; per-channel filtering would not consistently protect shared extractors, replay, history, or UI projection.
