## Summary

Keep this open. Current main still has a phase-only assistant visibility contract, with no parsed, encoded, schema-validated, or delivery-enforced public/internal assistant audience boundary.

## What This Changes

Define and enforce explicit public/internal assistant audience metadata across signature parsing, provider replay, visible extraction, delivery, and UI/history, with fail-closed invalid audience handling and silent-sentinel suppression.

## Best Possible Solution

Define one versioned `public | internal` assistant audience contract in shared message content, propagate it through provider parsing/replay and Gateway schemas, and enforce public-only selection at visible delivery boundaries with a narrow legacy phase fallback.

## Reproduction Assessment

Yes. A synthetic assistant text block signed as `{ "v": 1, "id": "x", "phase": "final_answer", "audience": "internal" }` follows the current final-answer path because the parser drops `audience` before shared visible extraction.

## Solution Assessment

Yes for the requested direction. A shared audience contract is the maintainable boundary; channel-local filtering or more sentinel scrubbers would leave replay, history, UI, and shared extractors inconsistent.
