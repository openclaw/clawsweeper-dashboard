## Summary

Keep open: current main does not expose the requested `contextFiles`/`inheritContext` API, and this PR is not safe to merge as-is because the new reader uses the target workspace for cross-agent spawns and bypasses existing bootstrap bounds.

## What This Changes

The PR adds `contextFiles` and `inheritContext` parameters to `sessions_spawn`, injects selected workspace files into the child system prompt, and adds an optional plugin logger transport type.

## Best Possible Solution

If maintainers want per-spawn file injection, implement it through the current bootstrap/context pipeline with explicit requester-versus-target semantics, strict allowlisting or bounded reads, docs, tests, and a changelog entry.

## Reproduction Assessment

Yes for the blocking PR bug: source review shows `contextFiles`/`inheritContext` resolve `requesterWorkspaceDir` with `targetAgentId`, so a cross-agent spawn reads the child workspace. No runtime test was run because this review is read-only.

## Solution Assessment

No. The direct `extraSystemPrompt` file reader bypasses current bootstrap loading, budgeting, and workspace policy; a safer solution would reuse those mechanisms if this API is approved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Read context files from the requester workspace:** `src/agents/subagent-spawn.ts:525`
  - body: When `agentId` targets a different agent, this resolves `requesterWorkspaceDir` with `targetAgentId`, so `contextFiles`/`inheritContext` load the child agent's workspace instead of the caller's. That silently skips or injects the wrong project context, contradicting the new API docs and PR body.
  - confidence: 0.95
- **[P2] Bound context file injection before reading files:** `src/agents/subagent-spawn.ts:532-533`
  - body: The new path reads every requested basename with `fs.readFile` and appends full contents to the system prompt, while the schema has no item cap and the existing bootstrap byte/budget guards are bypassed. A caller can pass many large workspace-root files and cause prompt or memory blowups, or expose files a child would not otherwise receive.
  - confidence: 0.9
