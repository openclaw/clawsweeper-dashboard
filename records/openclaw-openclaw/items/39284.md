## Summary

Keep this PR open. Current main partly addresses the linked workspace-context problem through automatic bootstrap injection and `sessions_spawn(lightContext)`, but it still does not implement the PR's explicit `contextFiles` or `inheritContext` API, and the proposed patch is not merge-ready because it has cross-agent workspace and resource-bound issues.

## What This Changes

The PR adds `contextFiles` and `inheritContext` options to native `sessions_spawn` and appends selected workspace files to the spawned sub-agent system prompt, plus an unrelated plugin logger type extension.

## Best Possible Solution

Keep the PR open for maintainer product judgment. If explicit spawn-time file selection is still desired, it should be rebuilt on the current bootstrap pipeline with documented requester/target workspace semantics, bounded schema and prompt budgets, same-agent and cross-agent tests, docs updates, and no unrelated plugin type changes.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to inspect current main's `sessions_spawn` schema, forwarding path, and `SpawnSubagentParams`: the requested fields are absent, while existing tests only cover `lightContext`.

## Solution Assessment

No. The proposed patch is not the best fix as-is because it bypasses current bootstrap bounds, reads from the wrong workspace in cross-agent cases, lacks tests/docs, and mixes an unrelated plugin logger type change; a safer solution should extend the established bootstrap context path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Read inherited files from the requester workspace:** `src/agents/subagent-spawn.ts:525`
  - body: When `agentId` targets a different agent, this resolves `requesterWorkspaceDir` with `targetAgentId`, so `contextFiles`/`inheritContext` read the child workspace instead of the requester workspace promised by the API. Cross-agent spawns will silently miss the caller's project rules or inject the wrong context.
  - confidence: 0.96
- **[P2] Bound context file injection before appending prompts:** `src/agents/tools/sessions-spawn-tool.ts:61-66`
  - body: `contextFiles` has no `maxItems` limit, and the implementation appends full file contents without per-file or total prompt caps. A caller can request many root files or very large files and inflate the child system prompt far beyond the bounded bootstrap path.
  - confidence: 0.9
