## Summary

Keep open: current main still lacks the explicit `contextFiles`/`inheritContext` spawn API, and the proposed patch has merge-blocking requester-workspace and unbounded prompt-injection problems that need maintainer API direction before replacement work.

## What This Changes

The PR adds `contextFiles` and `inheritContext` parameters to native `sessions_spawn`, injects selected workspace files into the child system prompt, and extends the plugin logger type.

## Best Possible Solution

Build any explicit subagent context-file API on top of the existing bounded bootstrap/context pipeline with clear requester/target workspace semantics, schema limits, docs, and same-agent plus cross-agent regression tests; otherwise leave the broader role-instruction design to #54504.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists from the PR diff: cross-agent `contextFiles` resolve through `targetAgentId`, and the schema/implementation add unbounded file-to-prompt injection while current main lacks the requested fields.

## Solution Assessment

No. The proposed patch is not the best solution as-is because it bypasses the bounded bootstrap path, reads the wrong workspace for the promised API, lacks limits/tests/docs, and mixes in an unrelated plugin logger type change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Read inherited files from the requester workspace:** `src/agents/subagent-spawn.ts:525`
  - body: When `agentId` targets a different agent, this resolves `requesterWorkspaceDir` with the child `targetAgentId`, so `contextFiles` and `inheritContext` read the child workspace instead of the requester workspace promised by the API. Cross-agent spawns can silently miss the caller's project rules or inject the wrong context.
  - confidence: 0.96
- **[P2] Bound context file injection before appending prompts:** `src/agents/tools/sessions-spawn-tool.ts:61-66`
  - body: `contextFiles` has no `maxItems` limit, and the implementation appends full file contents without per-file or total prompt caps. A caller can request many root files or very large files and inflate the child system prompt beyond the existing bounded bootstrap path.
  - confidence: 0.9
