## Summary

Keep this PR open for maintainer handling. Current main partially covers the linked workspace-context problem through automatic bootstrap injection, but it still does not expose the explicit `contextFiles` or `inheritContext` API, and the proposed patch has confirmed correctness and resource-bound issues before it could be merged.

## What This Changes

The PR adds `contextFiles` and `inheritContext` options to native `sessions_spawn`, appends selected workspace files to the child system prompt, and includes an unrelated plugin logger type extension.

## Best Possible Solution

If explicit spawn-time context file selection is still desired, rebuild it on top of the existing bootstrap/context pipeline with documented requester/target workspace semantics, prompt-size budgets, schema bounds, same-agent and cross-agent tests, docs, and no unrelated plugin type changes.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: inspect the PR diff to see the cross-agent workspace lookup and unbounded file append, and inspect current main to confirm the requested `contextFiles` and `inheritContext` API is still absent.

## Solution Assessment

No. The proposed patch is not the best solution as-is because it bypasses the bounded bootstrap path, reads from the wrong workspace for cross-agent spawns, lacks tests/docs, and mixes in an unrelated plugin type change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Read inherited files from the requester workspace:** `src/agents/subagent-spawn.ts:525`
  - body: When `agentId` targets a different agent, this resolves `requesterWorkspaceDir` with `targetAgentId`, so `contextFiles`/`inheritContext` read the child workspace instead of the requester workspace promised by the API. Cross-agent spawns will silently miss the caller's project rules or inject the wrong context.
  - confidence: 0.96
- **[P2] Bound context file injection before appending prompts:** `src/agents/tools/sessions-spawn-tool.ts:61-66`
  - body: `contextFiles` has no `maxItems` limit, and the implementation appends full file contents without per-file or total prompt caps. A caller can request many root files or very large files and inflate the child system prompt far beyond the bounded bootstrap path.
  - confidence: 0.9
