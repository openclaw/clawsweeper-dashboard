## Summary

Keep open because current main still treats a non-empty Discord guild `channels` map as an implicit channel allowlist, and changing that behavior would alter access-control compatibility for existing guild configs.

## What This Changes

The issue requests Discord guild `channels` entries to act as additive per-channel overrides while unlisted channels inherit guild-level defaults.

## Best Possible Solution

Keep this open as a Discord plugin access-control design task that separates additive overrides from explicit channel allowlisting, preserves safe compatibility, and updates runtime, docs, security guidance, and regression coverage together.

## Reproduction Assessment

Yes. A source-level reproduction is a Discord guild config with `groupPolicy: "open"`, guild-level `requireMention: false`, and a non-empty `guilds.*.channels` map that omits the incoming channel; current main resolves the omitted channel to `{ allowed: false }` and preflight drops it.

## Solution Assessment

Unclear. Additive inheritance solves the reported UX, but applying it globally is not clearly the safest fix because current documented behavior may be used as an access-control allowlist.
