## Summary

Keep open: the PR is still relevant and improves OpenShell mirror mkdir/rename/sync paths, but it is not merge-ready because direct bridge deletes still use raw remote `rm`, leaving the same remote-only symlink-parent escape for deletions. The submitted proof is also test-only and does not cover the live OpenShell SSH path.

## What This Changes

The PR adds an OpenShell pinned remote path mutation script, wires mirror mkdirp/rename and sync delete/upload-parent handling through it, extends backend types, and adds focused OpenShell tests.

## Best Possible Solution

Keep the OpenShell-local pinned mutation approach, but route every mirror remote mutation, including bridge remove, through the backend pinning helper and add proof for the delete case before merge.

## Reproduction Assessment

Yes from source inspection: current main and the PR head leave `OpenShellFsBridge.remove()` on raw remote `rm`, so a remote-only symlink parent such as `/sandbox/alias -> /outside` can still redirect a delete path. I did not execute a live OpenShell SSH reproduction in this read-only review.

## Solution Assessment

No, not yet. The pinned backend helper is the right owner boundary for mkdir/rename/sync handling, but the best fix also routes bridge deletes through the same pinned parent validation and proves that case.
