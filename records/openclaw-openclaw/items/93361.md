## Summary

Keep open: this is useful OpenShell mirror hardening, but the patch is not complete because direct bridge deletes still run raw remote shell removal against `target.containerPath`, leaving a remote-only symlink-parent delete path outside the new pinned helper.

## What This Changes

The PR adds an OpenShell pinned remote mutation shell helper, wires mirror mkdir/rename and mirror-sync delete/upload-parent paths through it, extends backend types, and adds focused OpenShell tests.

## Best Possible Solution

Expose a pinned backend remove operation that preserves recursive/force semantics, route `OpenShellFsBridge.remove()` through it, and add the remote-only symlink-parent delete regression before merge.

## Reproduction Assessment

Yes, from source inspection: create a remote-only symlink parent under an OpenShell managed root and call `OpenShellFsBridge.remove()` on a child path; the PR head still sends raw `rm`/`rmdir` to that unresolved remote path. I did not run a live OpenShell SSH sandbox in this read-only review.

## Solution Assessment

No. The PR is the right layer for mkdir, rename, and sync mutations, but it is not yet the best complete fix because direct bridge removal needs the same pinned-parent treatment used by the sibling remote shell bridge.
