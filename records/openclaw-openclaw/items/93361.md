## Summary

Keep this PR open: it usefully hardens OpenShell mirror mkdir, rename, and sync-side mutations, but direct mirror deletes still shell out with raw `rm`/`rmdir` against the unresolved remote path, so the sandbox-boundary fix is incomplete.

## What This Changes

The PR adds an OpenShell remote mutation helper, routes mirror mkdir/rename and mirror-sync delete/upload-parent paths through it, updates backend types, and adds focused OpenShell regression tests.

## Best Possible Solution

Expose a pinned backend remove operation that preserves recursive/force semantics, route `OpenShellFsBridge.remove()` through it, and add a remote-only symlink-parent delete regression before merge.

## Reproduction Assessment

Yes, from source inspection: create a remote-only symlink parent under an OpenShell managed root and call `OpenShellFsBridge.remove()` on a child path; PR head still sends raw `rm`/`rmdir` to that remote path. I did not run a live OpenShell SSH sandbox in this read-only review.

## Solution Assessment

No. The PR is the right layer for mkdir, rename, and sync mutations, but it is not the best complete fix until direct bridge removal gets the same pinned-parent treatment as the sibling remote shell bridge.
