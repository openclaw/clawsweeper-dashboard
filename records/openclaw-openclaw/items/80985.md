## What This Changes

The PR adds cron add/edit checks, a `WebchatNotDeliverableError` in outbound channel selection, tests, docs, and a CHANGELOG entry for rejecting announce delivery to webchat.

## Best Possible Solution

Reject effective announce delivery to normalized webchat at cron create/update and cron runtime boundaries before target normalization, keep valid deliverable channels unchanged, and document only runnable webchat-session patterns.

## Reproduction Assessment

Yes by source inspection, though I did not execute tests in this read-only review. Current cron add defaults isolated agent-turn delivery to announce, cron edit merges target-only patches into existing delivery, and raw cron runtime currently loses `webchat` before selector validation.

## Solution Assessment

No; rejecting webchat as an announce target is the right direction, but this implementation needs to check effective delivery mode, validate the cron runtime path before normalization, and remove the invalid docs example.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Reject implicit announce on cron add:** `src/cli/cron-cli/register.cron-add.ts:209-212`
  - body: Isolated `cron add` jobs default to `delivery.mode: "announce"` when neither `--announce` nor `--no-deliver` is supplied. Because this guard keys only on `hasAnnounce`, `openclaw cron add ... --session isolated --message hello --channel webchat` still builds `delivery: { mode: "announce", channel: "webchat" }` and reaches the gateway, so the create-time protection misses the default shape.
  - confidence: 0.93
- **[P2] Validate webchat after cron edit merges delivery:** `src/cli/cron-cli/register.cron-edit.ts:156-159`
  - body: `cron edit` supports delivery-target-only patches, and the gateway validates the merged job, so an existing announce job can still be edited to `--channel webchat` without passing `--announce`. This guard also misses the deprecated explicit `--deliver` spelling. Reject against the effective merged announce delivery, not only `opts.announce`.
  - confidence: 0.92
- **[P2] Reject cron webchat before session target normalization:** `src/infra/outbound/channel-selection.ts:221-227`
  - body: The new typed error only fires when `resolveMessageChannelSelection` receives `channel: "webchat"`, but cron runtime calls `resolveSessionDeliveryTarget` first; that helper treats non-deliverable webchat as no channel and then calls the selector with only `{ cfg }`. Raw `jobs.json` or RPC cron callers therefore still get the old generic no-configured-channels error instead of `WebchatNotDeliverableError`.
  - confidence: 0.9
- **[P2] Remove the invalid main-session no-deliver example:** `docs/automation/cron-jobs.md:189`
  - body: The new docs show `--session main --system-event ... --no-deliver`, but the current CLI rejects `--no-deliver` for main-session systemEvent jobs with the existing `--announce/--no-deliver require a non-main agentTurn session target` guard. Users following the documented default-agent example will hit an error; drop `--no-deliver` from that shape or adjust the CLI contract intentionally.
  - confidence: 0.88
