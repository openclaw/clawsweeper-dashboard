## Summary

Keep open: current main still pre-creates only the Docker state directory, while the documented Docker Compose mount layout also exposes workspace and auth-profile config paths as independent named-volume targets. Two open PRs already reference this issue with closing syntax, so the right cleanup path is to review and land one fix rather than close the issue early.

## What This Changes

The issue asks OpenClaw to pre-create all documented Docker named-volume mount points with node-owned permissions so fresh named-volume deployments do not fail first-run writes with EACCES.

## Best Possible Solution

Land one narrow Dockerfile/test repair that pre-creates every documented Docker mount target before USER node, with structural coverage and fresh named-volume proof, then close this issue through the merged PR.

Yes. The issue provides a concrete fresh named-volume Docker command and current main still lacks pre-created image paths for the workspace and config mounts, though I did not build and smoke-test a current-main image during this read-only review.

## Solution Assessment
