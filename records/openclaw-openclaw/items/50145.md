## Summary

Keep open. Current main still contains the reported stale replay path, and open PR #50147 is already the paired closing-syntax implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue asks OpenClaw to stop aborted `messages.queue.mode: "interrupt"` runs from replaying a previous assistant reply before the replacement turn answers.

## Best Possible Solution

Land PR #50147 or an equivalent narrow patch that prevents aborted empty-output interrupt runs from constructing or delivering payloads from prior session history, with focused regression coverage.

## Reproduction Assessment

Yes. The issue gives concrete interrupt-mode steps, and current-main source inspection shows the abort path can retain an older `lastAssistant`, convert it into a payload, and deliver it without an aborted-run gate.

## Solution Assessment

Yes. The narrow maintainable fix is to suppress the `lastAssistant` fallback for aborted empty attempts and skip reply delivery for aborted runs, while keeping legitimate non-aborted fallback behavior intact.
