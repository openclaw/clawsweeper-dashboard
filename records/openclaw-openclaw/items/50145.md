## Summary

Current main already fixes the interrupt-mode stale assistant replay by passing the aborted-run fact into embedded payload assembly and suppressing assistant answer/reasoning output for aborted runs; the fix is also included in v2026.6.6.

## What This Changes

The issue reports that interrupt queue mode can re-deliver a previous assistant reply when an active run is aborted before producing new assistant text.

## Best Possible Solution

Keep the shipped runAborted payload-builder fix, and track broader non-interrupt replay/recovery symptoms in their narrower reports or the duplicate/replay umbrella.
