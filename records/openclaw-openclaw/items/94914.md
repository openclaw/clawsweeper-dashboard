Keep this PR open but do not merge as-is: it targets a real replace-mode model-list bug, but the shared row-filter guard would also narrow explicit `openclaw models list --all` and provider-filtered catalog browsing, which are documented as full-catalog diagnostic paths. The PR also still lacks after-patch real CLI proof.

## What This Changes

The PR adds replace-mode provider allowlisting to model-list row filtering and adds a unit test for provider-catalog row filtering.

## Best Possible Solution

Scope replace-mode filtering to the default configured list leak, preserve explicit `--all` and provider-filtered catalog browsing, then land one canonical fix with real CLI proof.

## Reproduction Assessment

Yes at source level: current main appends authenticated catalog rows in the default model-list path without checking `models.mode`, and the canonical issue provides concrete replace-mode leaked-provider output. I did not run the live CLI in this read-only review.

## Solution Assessment

No, not as-is. The shared row-filter guard is too broad because it also affects explicit `--all` and provider-filtered catalog browsing; the safer fix is to gate only the default authenticated catalog append or another default-list-only source.
