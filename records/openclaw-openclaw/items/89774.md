## Summary

Keep open: current main still hides inactive `subagent:` and `spawnedBy` sessions from the chat picker, and there is no current-main reveal toggle. The request is a plausible Control UI ergonomics feature, but it should get maintainer product/UX judgment because it deliberately exposes internal transcripts that were recently hidden by default.

## What This Changes

Add an explicit Control UI chat session-switcher toggle to show subagent or spawnedBy sessions that are currently hidden by default.

## Best Possible Solution

Keep internal sessions hidden by default, and if maintainers accept the operator workflow add a clearly labeled default-off Control UI reveal/filter that makes subagent rows visually distinct and covered by focused UI tests.

## Reproduction Assessment

Yes by source inspection for the reported current behavior: current main skips inactive `subagent:` and `spawnedBy` rows from `resolveSessionOptionGroups`, and existing tests assert that behavior. I did not run a live browser session, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

Unclear pending maintainer product judgment. A default-off operator reveal toggle is a plausible narrow solution and safer than removing the filter, but the exact UX should account for the related internal-session leak concern.
