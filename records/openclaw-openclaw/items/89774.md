## Summary

Keep open: current main and the latest release still hide inactive `subagent:` and `spawnedBy` rows from the chat picker, and there is no reveal setting. The related session-kind-filter PR is still open, conflicting, proof-blocked, and not safe to treat as a resolved canonical path for this issue.

## What This Changes

The issue requests a default-off Control UI chat session switcher toggle or filter that reveals subagent and spawnedBy sessions for operator inspection.

## Best Possible Solution

Keep internal sessions hidden by default, and if maintainers accept the operator workflow, implement a clearly labeled localized reveal/filter with distinct subagent presentation and focused desktop/mobile picker coverage.

## Reproduction Assessment

Yes. Source inspection of current main and `v2026.6.6` shows inactive `subagent:` and `spawnedBy` rows are filtered from the chat picker and no reveal setting exists; no live browser run was performed.

## Solution Assessment

Unclear pending maintainer product judgment. A default-off operator reveal is plausible and narrower than removing the filter, while the current related broad filter PR is not the best solution as written until it rebases, preserves safe defaults, and proves the UI behavior.
