## Summary

Keep open. The reported silent subagent announce give-up behavior is source-reproducible on current main, but this PR still needs a rebase into the active lifecycle controller, clearer delivery-contract handling, and after-fix real channel proof before it is mergeable.

## What This Changes

This PR adds a `notifyAnnounceGiveUp` fallback in `src/agents/subagent-registry.ts` to call gateway agent delivery after subagent announce retry-limit give-up.

## Best Possible Solution

Rebase the fallback into `src/agents/subagent-registry-lifecycle.ts`, cover live and resumed retry-limit give-up, and make the delivery path explicitly match the documented agent-mediated contract or a reviewed raw outbound contract.

## Reproduction Assessment

Yes, at source level: current main reaches retry-limit give-up through `resolveDeferredCleanupDecision`, then lifecycle cleanup marks delivery failed and completes without a fallback requester notification. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No, not as-is. The fallback direction is reasonable, but the patch must move into the active lifecycle controller and settle the agent-mediated versus raw-delivery contract before merge.
