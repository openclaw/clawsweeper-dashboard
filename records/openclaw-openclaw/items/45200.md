## Summary

Keep open: the silent terminal give-up bug still exists on current main, but this PR is not merge-ready because its fallback sends recovered text through a new gateway agent turn instead of the existing direct outbound delivery seam.

## What This Changes

The PR adds a `notifyAnnounceGiveUp` helper that calls gateway `agent` from retry-exhaustion cleanup paths to notify the requester channel with recovered subagent output.

## Best Possible Solution

Implement terminal announce give-up recovery in the current lifecycle/delivery modules by factoring or reusing the direct `sendMessage` completion fallback, delivering raw recovered result text with per-run idempotency and focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. On current main, force `runSubagentAnnounceFlow` to return `false` until retry count reaches `3`, or restore a run with `announceRetryCount >= 3`; both terminal paths mark delivery failed and finish cleanup without a requester-channel send.

## Solution Assessment

No. A last-resort notification is the right direction, but this PR’s gateway-agent route is not the narrowest maintainable fix because it turns recovered text into requester-agent input instead of direct outbound channel content.
