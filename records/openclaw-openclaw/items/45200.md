## Summary

Keep open: the silent terminal give-up path is still source-reproducible on current main, but this PR should not merge as-is because it routes recovery through the requester-agent/model path and is stale against the current lifecycle/delivery split.

## What This Changes

This PR adds a subagent announce give-up notifier in `src/agents/subagent-registry.ts` that tries a last-resort gateway `agent` delivery after retry exhaustion.

## Best Possible Solution

Implement terminal announce give-up recovery in the current lifecycle and delivery modules by reusing direct outbound fallback delivery for raw recovered result text with per-run idempotency, regression tests, and a changelog entry.

## Reproduction Assessment

Yes, from source inspection. A restored run with `announceRetryCount >= MAX_ANNOUNCE_RETRY_COUNT` or a live cleanup decision of `give-up` reaches terminal cleanup on current main without a final requester-channel notification.

## Solution Assessment

No. A last-resort notification is the right direction, but this PR sends the notice through gateway `agent`, so the requester model mediates the content; the narrower current-main fix is direct outbound fallback delivery of the recovered text.
