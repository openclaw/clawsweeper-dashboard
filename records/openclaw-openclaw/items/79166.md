## Summary

Keep open: current main still lacks a `doctor --dry-run` or `--preview` flag, and repair mode still flows into config writes when pending changes exist. This is a valid core CLI feature request, not an already-implemented cleanup item.

## What This Changes

Request adding a `openclaw doctor --dry-run` or preview/diff mode so users can inspect `doctor --fix` changes before they are applied.

## Best Possible Solution

Add a first-class core doctor preview mode that reuses repair planning, guarantees no config or state writes, and documents whether output is human diff, structured JSON, or both.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high-confidence current-main evidence that the doctor command has no dry-run/preview option and repair mode still writes when pending changes exist.

## Solution Assessment

Unclear as proposed. A dry-run flag is the right product direction, but maintainers should decide the output contract and coverage across config, state, plugin, and service repairs before implementation.
