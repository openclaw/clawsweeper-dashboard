## What This Changes

The PR wraps async JSON reads in `src/infra/json-files.ts` with bounded retries for `File changed during read` races and adds retry success/exhaustion tests.

## Best Possible Solution

Land one central retry path for async JSON reads, preferably with a plan to remove the local wrapper once `@openclaw/fs-safe` ships the same contract, while preserving parse/corruption failures after bounded retries.

## Reproduction Assessment

Yes, source-reproducible: `@openclaw/fs-safe` throws `File changed during read`, current `main` exposes async JSON readers without retry, and pairing reads `paired.json` through `readJsonIfExists`. The PR's tests simulate the stable-target race directly.

## Solution Assessment

Mostly yes: the wrapper is a narrow mitigation that preserves strict reader contracts and keeps parse errors fail-closed. A dependency-level fix would be cleaner long term, but it has not shipped yet.
