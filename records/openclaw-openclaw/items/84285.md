## What This Changes

The PR wraps OpenClaw async JSON readers with bounded retry handling for transient `File changed during read` races and adds retry success/exhaustion tests.

## Best Possible Solution

Land one central retry path for async JSON reads, preferably dependency-level; if this OpenClaw wrapper lands first, keep the retry bounded and plan to remove it after the `@openclaw/fs-safe` contract ships.

## Reproduction Assessment

Yes, source-reproducible: `@openclaw/fs-safe` 0.2.4 throws `File changed during read`, current `main` exposes async JSON readers without retry, and pairing reads `paired.json` through `readJsonIfExists`. The PR tests simulate the stable-target race directly.

## Solution Assessment

Mostly yes: the wrapper is a narrow mitigation that preserves strict reader contracts and keeps parse/corruption failures fail-closed. A dependency-level fix is cleaner long term, but it has not shipped into OpenClaw yet.
