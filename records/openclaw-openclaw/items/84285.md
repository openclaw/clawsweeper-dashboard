## What This Changes

This PR wraps OpenClaw async JSON readers with bounded retry handling for transient `File changed during read` races and adds retry success/exhaustion coverage.

## Best Possible Solution

Land one bounded async JSON read retry path; preferably take it from `@openclaw/fs-safe` when available, or merge this OpenClaw wrapper now with a cleanup expectation.

## Reproduction Assessment

## Solution Assessment

Mostly yes: the wrapper is narrow, bounded, preserves strict reader contracts, and keeps parse/corruption errors fail-closed. A dependency-level fix is cleaner long term, so the maintainer decision is whether to merge this mitigation now or wait for the upstream package PR.
