## What This Changes

The PR wraps OpenClaw’s async JSON file helpers with bounded retry handling for transient `File changed during read` races and adds retry success/exhaustion tests.

## Best Possible Solution

Land the bounded OpenClaw wrapper after lint cleanup if maintainers accept it as an interim mitigation, then replace it with the dependency-level fs-safe retry once that release is available.

## Reproduction Assessment

Yes, source-reproducible: current main re-exports fs-safe async JSON readers without retry, fs-safe throws `File changed during read` on stat/open identity races, and device pairing reads `paired.json` through that helper. I did not run a new live race harness in this read-only review.

## Solution Assessment

Mostly yes: the local wrapper is a narrow mitigation that preserves strict reader contracts and fail-closed parse/read errors. A dependency-level fs-safe fix is cleaner long term, but it has not shipped into OpenClaw yet.
