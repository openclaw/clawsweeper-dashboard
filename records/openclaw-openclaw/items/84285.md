## What This Changes

The PR replaces async JSON helper re-exports in `src/infra/json-files.ts` with bounded retry wrappers for `File changed during read` and routes `tryReadJson` through `readJsonIfExists` while preserving strict read types.

## Best Possible Solution

Prefer the dependency-level `@openclaw/fs-safe` fix and bump OpenClaw once released; if urgency requires this local mitigation, add focused retry coverage and redacted after-fix gateway proof before merge.

## Reproduction Assessment

Yes, source-reproducible: fs-safe 0.2.4 can throw `File changed during read`, OpenClaw currently re-exports those readers without retry, and device-pairing reads `paired.json` through `readJsonIfExists`. The open fs-safe PR also demonstrates the same race with a real-disk concurrent read/write test.

## Solution Assessment

Unclear as the final merge path: the local retry wrapper is plausible and the strict-reader type issue is fixed, but the dependency-level fix with a typed `path-mismatch` error and regression tests is cleaner if it can ship quickly.
