## Summary

Keep open. Current main still drops Mattermost `post_edited` WebSocket events before post parsing or mention routing, and open PR #72513 is the active closing implementation candidate rather than a reason to close this issue now.

## What This Changes

Request Mattermost plugin handling for `post_edited` WebSocket events so @mentions added by editing a message can wake the tagged agent.

## Best Possible Solution

Review and land PR #72513, or an equivalent Mattermost-only fix, so `post_edited` events with updated post bodies reach the post parser and replay handling does not suppress edited-in mentions.

## Reproduction Assessment

Yes. A synthetic Mattermost WebSocket payload with `event: "post_edited"` follows the current source path to the event guard at `monitor-websocket.ts:358` and returns before `onPosted` or mention detection can run.

## Solution Assessment

Yes for the product boundary: this belongs in the Mattermost plugin WebSocket and replay path, not in core channel APIs. The implementation should be edit-aware, not only a parser allow-list change.
