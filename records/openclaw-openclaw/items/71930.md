## Summary

Keep open. Current main still filters Mattermost WebSocket post handling to `posted` only, so `post_edited` events cannot reach the downstream mention gate; the open linked PR is the active implementation path and should be reviewed rather than closing this issue early.

## What This Changes

The issue requests Mattermost `post_edited` WebSocket events be handled so @mentions added by editing an existing post can wake the tagged agent.

## Best Possible Solution

Land PR #72513 or an equivalent Mattermost-only fix that lets both `posted` and `post_edited` events carrying `data.post` reach the existing parsing and mention-gate flow, with edit-aware replay behavior so an originally ignored post can wake the agent after a mention is added.

## Reproduction Assessment

Yes. The issue gives concrete Mattermost steps, and current main has deterministic early returns in `monitor-websocket.ts` before `onPosted`, while mention detection is only downstream in `monitor.ts`.

## Solution Assessment

Yes, with one caveat. The narrow maintainable fix belongs in the Mattermost plugin WebSocket parser/dispatcher path, but it must also account for the existing account-plus-post-id replay guard so edited-in mentions are not suppressed as duplicates.
