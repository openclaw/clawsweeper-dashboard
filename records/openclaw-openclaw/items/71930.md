## Summary

Keep open. Current main still drops Mattermost `post_edited` WebSocket events before parsing or mention routing, and open PR #72513 is the active closing implementation candidate.

## What This Changes

Mattermost should reevaluate `post_edited` WebSocket posts so @mentions added by editing a message can wake the tagged agent.

## Best Possible Solution

Land PR #72513, or an equivalent Mattermost-only patch, so `posted` and `post_edited` events carrying `data.post` reach the same parser and mention-gate flow with edit-aware replay behavior.

## Reproduction Assessment

Yes. The reported Mattermost edit sequence maps directly to a synthetic `post_edited` WebSocket payload on current main, which returns at the event guard before `onPosted` can run.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Mattermost plugin WebSocket parser/dispatcher plus replay handling; I found no evidence that a core channel API change is required.
