## Summary

Keep open. Current main still drops Mattermost `post_edited` WebSocket payloads before post parsing and mention-gate handling, and open PR #72513 is the active closing implementation candidate, so the issue should remain open until that PR or an equivalent fix merges.

## What This Changes

The issue reports that Mattermost `post_edited` WebSocket events are ignored, so @mentions added by editing an existing message do not wake the agent.

## Best Possible Solution

Review and land PR #72513, or an equivalent Mattermost-only patch, so `posted` and `post_edited` events carrying `data.post` feed the same post parsing and mention-gate flow while preserving self-message filtering and edit-aware duplicate protection.

## Reproduction Assessment

Yes. A `post_edited` WebSocket payload on current main follows deterministic early returns in `monitor-websocket.ts` before `onPosted`, and the issue provides concrete Mattermost reproduction steps confirmed on April 25, 2026.

## Solution Assessment
