## Summary

Keep open. Current main still rejects Mattermost `post_edited` WebSocket events before post parsing and mention routing, and open PR #72513 is the active closing implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests Mattermost to re-evaluate edited posts so @mentions added by edit can wake the agent.

## Best Possible Solution

Land PR #72513 or an equivalent Mattermost-only fix that routes `post_edited` events through parsing and handles replay/dedupe so edited-in mentions are not suppressed.

## Reproduction Assessment

Yes. A synthetic Mattermost WebSocket payload with `event: "post_edited"` follows the current source path to the event guard at `monitor-websocket.ts:358` and returns before `onPosted` or mention detection can run.

## Solution Assessment

Yes for the product boundary: this belongs in the Mattermost plugin WebSocket and replay path, not in core channel APIs. The implementation should be edit-aware rather than only changing one string comparison.
