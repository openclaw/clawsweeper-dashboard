## Summary

Keep open. Current main still drops Mattermost `post_edited` WebSocket events before post parsing and mention detection, and the linked open PR #72513 is already the active closing implementation path rather than a reason to close this issue early.

## What This Changes

The issue requests handling Mattermost WebSocket `post_edited` events so @mentions added by editing a message re-enter the Mattermost mention and wake flow instead of being silently ignored.

## Best Possible Solution

Keep this issue open until PR #72513, or an equivalent Mattermost-only fix, lands. The desired end state is for `posted` and `post_edited` events carrying `data.post` to reach the same parsing and mention-gate flow while preserving self-message filtering and avoiding replay dedupe suppression of edited-in mentions.

## Reproduction Assessment

Yes. A `post_edited` WebSocket payload on current main follows deterministic early returns in `monitor-websocket.ts` before `onPosted`, and the issue provides concrete Mattermost steps confirmed on April 25, 2026.

## Solution Assessment

Yes, with one implementation caveat. The narrow maintainable fix belongs in the Mattermost plugin WebSocket parser/dispatcher path, plus any necessary edit-aware replay handling; the code and docs do not indicate a need for a core channel API change.
