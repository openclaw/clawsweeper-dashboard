## Summary

Keep open: current main and v2026.6.10 still reject Mattermost `post_edited` events before post parsing/dispatch, and the former linked fix PR is now closed unmerged.

## What This Changes

Mattermost should wake an agent when a user edits an existing post to add an @mention.

## Best Possible Solution

Land a focused Mattermost plugin fix that treats `posted` and `post_edited` as post-bearing events, makes replay dedupe edit-aware, and proves the edited @mention wake path with targeted tests plus real Mattermost evidence.

## Reproduction Assessment

Yes at source level: a current-main `post_edited` event returns at the posted-only WebSocket guard before `onPosted` and mention detection. I did not run a live Mattermost workspace reproduction in this read-only review.

## Solution Assessment

Yes, the best fix is to route edited post-bearing events through the existing Mattermost post path and make replay dedupe edit-aware. A WebSocket-only branch would be too narrow because the original post id can already be committed in the replay guard.
