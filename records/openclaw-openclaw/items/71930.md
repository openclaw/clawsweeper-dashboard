## Summary

Keep open. Current main still drops Mattermost `post_edited` WebSocket events before parsing or dispatch, so edited-in @mentions cannot reach the existing mention/wake path. The linked PR #72513 is open with closing syntax and is the active implementation candidate, so this issue should stay open until that PR or an equivalent Mattermost-only fix lands.

## What This Changes

Mattermost should re-evaluate edited posts that add @mentions so the tagged agent wakes as it would for a fresh `posted` message.

## Best Possible Solution

Keep this issue open until PR #72513, or an equivalent Mattermost-only patch, lands. The desired end state is for `posted` and `post_edited` events carrying `data.post` to reach the same parser and mention-gate flow while preserving self-message filtering and avoiding replay suppression of edited-in mentions.

## Reproduction Assessment

Yes. The issue's Mattermost edit sequence is credible, and a synthetic `post_edited` WebSocket payload on current main deterministically returns at the event guard before parsing or `onPosted` dispatch.

## Solution Assessment

Yes, the requested direction is the narrow maintainable fix. This should be solved inside the Mattermost plugin WebSocket parser/dispatcher and replay handling; I found no evidence that a core channel API change is required.
