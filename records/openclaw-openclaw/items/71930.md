## Summary

Keep open. Current main still filters Mattermost WebSocket handling to `posted` events before parsing and dispatch, so edited-in @mentions remain dropped; the linked PR is open and should be reviewed or landed before this issue is closed. 

## What This Changes

Mattermost should evaluate `post_edited` WebSocket events with updated post bodies so @mentions added by editing a message can wake the tagged agent.

## Best Possible Solution

Keep this issue open until PR #72513, or an equivalent Mattermost-only fix, lands. The desired end state is for `posted` and `post_edited` events carrying `data.post` to reach the same parser and mention-gate flow while preserving self-message filtering and avoiding replay-dedupe suppression of edited-in mentions.

## Reproduction Assessment

Yes. The issue's Mattermost edit sequence is credible, and a synthetic `post_edited` WebSocket payload on current main deterministically returns before parsing and `onPosted` dispatch.

## Solution Assessment

Yes for the requested direction. The narrow maintainable fix belongs in the Mattermost plugin WebSocket parser/dispatcher path, with edit-aware replay handling as needed; I found no evidence that a core channel API change is required.
