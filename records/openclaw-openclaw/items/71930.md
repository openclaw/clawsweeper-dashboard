## Summary

Keep open. Current main still drops Mattermost `post_edited` WebSocket events before post parsing or mention routing, and open PR #72513 is the active linked implementation candidate.

## What This Changes

Report that Mattermost `post_edited` WebSocket events are ignored, so mentions added by editing a message never wake the agent.

## Best Possible Solution

Land PR #72513, or an equivalent Mattermost-only patch, so `posted` and `post_edited` events carrying `data.post` reach the same parser, debounce, replay, and mention-gate flow with regression coverage.

## Reproduction Assessment

Yes. The reported sequence maps directly to a `post_edited` WebSocket frame, and current main returns at the non-`posted` event guard before `onPosted` can run.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Mattermost plugin WebSocket parser/dispatcher plus edit-aware replay coverage; I found no evidence that a core channel API change is required.
