## Summary

Keep open. Current main still drops Mattermost `post_edited` WebSocket events before parsing or mention routing, and open PR #72513 is the active closing implementation candidate rather than a reason to close this issue before merge.

## What This Changes

Mattermost should evaluate edited messages that add @mentions instead of silently dropping `post_edited` WebSocket events.

## Best Possible Solution

Review and land PR #72513, or an equivalent Mattermost-only fix, so `post_edited` events with updated post bodies reach mention routing without replay dedupe suppressing edited-in mentions.

## Reproduction Assessment

Yes. A synthetic Mattermost WebSocket payload with `event: "post_edited"` follows the current source path to the dispatcher guard at `monitor-websocket.ts:358` and returns before `onPosted` or mention detection can run.

## Solution Assessment

Yes for the product boundary: this belongs in the Mattermost plugin WebSocket and replay path, not in core channel APIs. The fix should be edit-aware rather than only broadening the parser allow-list.
