## Summary

Keep open. Current main still drops Mattermost `post_edited` WebSocket events before post parsing or mention routing, and open PR #72513 is the active closing implementation candidate rather than a reason to close this issue now.

## What This Changes

The issue asks for Mattermost edited posts that add an @mention to wake the tagged agent instead of being silently ignored.

## Best Possible Solution

Review and land PR #72513, or an equivalent Mattermost-only fix, so `post_edited` events with updated post bodies reach the existing post parser and replay handling is edit-aware.

## Reproduction Assessment

Yes. A synthetic Mattermost WebSocket payload with `event: "post_edited"` and `data.post` follows the current source path to the line 358 guard and returns before `onPosted`, debouncing, replay handling, or mention detection can run.

## Solution Assessment

Yes. The narrow maintainable solution belongs in the Mattermost plugin WebSocket/post replay path, not in core channel APIs; the fix should route edited posts through the existing post handling while preserving duplicate protection.
