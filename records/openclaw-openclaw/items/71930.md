## Label Justifications

- `P1`: A real Mattermost channel workflow can silently miss @mentions added by edit, leaving addressed agents asleep for affected users.
- `impact:message-loss`: The issue is about a channel event being dropped before mention routing, causing an expected wake message to be missed.

## Summary

Current main still drops Mattermost `post_edited` WebSocket events before they reach mention detection, so the reported wake failure remains valid. The issue should stay open because the linked implementation candidate is still open, merge-conflicting, and unmerged.

## What This Changes

The issue requests that Mattermost edited-post events be evaluated for new @mentions so an agent wakes when a mention is added by editing a message.

## Best Possible Solution

Review, update, and land `https://github.com/openclaw/openclaw/pull/72513` or an equivalent Mattermost-only fix that routes edited posts into mention handling with regression coverage for edit-aware wake behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Mattermost `post_edited` WebSocket event returns before parsing or `onPosted`, while mention detection runs only after posts reach `monitor.ts`.

## Solution Assessment

Yes for the direction. The narrow maintainable solution is to handle Mattermost post-bearing edit events in the plugin WebSocket dispatch and cover edit-aware replay behavior in Mattermost tests.
