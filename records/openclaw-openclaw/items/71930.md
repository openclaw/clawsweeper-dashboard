## Summary

Keep open. Current main still drops Mattermost `post_edited` WebSocket events before parsing or dispatch, and open PR #72513 is the active closing implementation candidate, so this issue should remain open until that PR or an equivalent Mattermost-only fix lands.

## What This Changes

The issue requests Mattermost `post_edited` events to be evaluated like new posts so @mentions added by editing a message can wake the agent.

## Best Possible Solution

Keep this issue open until PR #72513, or an equivalent Mattermost-only patch, lands and routes `posted` and `post_edited` payloads through the same parser and mention-gate flow while preserving self-message filtering and edit-aware replay behavior.

## Reproduction Assessment

Yes. A synthetic Mattermost WebSocket payload with `event: "post_edited"` deterministically returns at the current dispatcher guard before parsing or `onPosted`, matching the reported edited-in mention wake failure.
