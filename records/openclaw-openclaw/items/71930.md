## Summary

Keep open. Current main and the latest stable release still drop Mattermost `post_edited` events before mention routing, and the linked closing implementation is still open rather than merged.

## What This Changes

The issue asks Mattermost to route `post_edited` WebSocket events through mention and wake handling so @mentions added by editing a post can wake the agent.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/72513, or an equivalent Mattermost-only fix, so edited posts with updated bodies reach mention routing and replay dedupe keys are edit-aware.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Mattermost WebSocket payload with `event: "post_edited"` and `data.post` returns at the current dispatcher/parser before `onPosted`, while upstream Mattermost publishes edited posts with post data.

## Solution Assessment

Yes for the direction. The narrow maintainable fix belongs in the Mattermost plugin WebSocket and replay path, but this issue should stay open until the linked implementation or an equivalent fix is merged.
