## Summary

Keep open. Current main still drops Mattermost `post_edited` events before post parsing or mention routing, and the linked closing PR remains open and unmerged.

## What This Changes

Mattermost should evaluate edited post bodies from `post_edited` WebSocket events so @mentions added by edit can wake the agent.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/72513, or an equivalent Mattermost-only fix, so edited posts with updated bodies reach mention routing and replay dedupe keys are edit-aware.

## Reproduction Assessment

Yes, from source inspection. A synthetic Mattermost WebSocket payload with `event: "post_edited"` and valid `data.post` reaches the current dispatcher guard and returns before `onPosted`, while upstream Mattermost publishes edited posts with post data.

## Solution Assessment

Yes for the implementation direction. The narrow maintainable fix belongs in the Mattermost plugin WebSocket and replay path; the linked PR is the active candidate, but this issue should stay open until that fix merges.
