## Summary

Keep open. Current main still drops Mattermost `post_edited` events before post parsing or mention routing, and https://github.com/openclaw/openclaw/pull/72513 is an open, unmerged closing implementation candidate rather than a reason to close this issue now.

## What This Changes

The issue reports that Mattermost mentions added by editing an existing message do not wake the agent because `post_edited` WebSocket events are ignored.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/72513, or an equivalent Mattermost-only fix, so edited posts with updated bodies reach mention routing and replay dedupe keys are edit-aware.

## Reproduction Assessment

Yes from source inspection. A synthetic Mattermost WebSocket payload with `event: "post_edited"` and valid `data.post` reaches the current dispatcher guard and returns before `onPosted`, while upstream Mattermost publishes edited posts with post data.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Mattermost plugin WebSocket and replay path, not in core channel APIs, and the linked open PR is already the active candidate for that direction.
