## Summary

Keep open. Current main still drops Mattermost `post_edited` events before post parsing and mention routing, the latest stable release has the same guards, and the linked closing PR is still open and conflicting rather than merged.

## What This Changes

Handle Mattermost `post_edited` WebSocket events so edits that add @mentions can wake agents through the existing mention routing path.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/72513 or an equivalent Mattermost-only fix that forwards edited post payloads to mention routing and uses edit-aware replay dedupe.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a `post_edited` WebSocket payload with `data.post` returns at the current parser/dispatcher before `onPosted`, while Mattermost upstream publishes edited posts with serialized post data.

## Solution Assessment

Yes for the direction. The narrow maintainable fix belongs in the Mattermost plugin WebSocket and replay path; this issue should stay open until the linked implementation or an equivalent fix is merged.
