## Summary

Keep open. Current main still drops Mattermost `post_edited` WebSocket events before parsing or mention routing, and open PR #72513 is the active unmerged implementation candidate rather than a reason to close this issue now.

## What This Changes

Mattermost should evaluate edited posts that add @mentions so mention-gated channel agents wake the same way they do for newly posted messages.

## Best Possible Solution

Land PR #72513, or an equivalent Mattermost-only fix, so `post_edited` events with updated post bodies reach mention routing exactly once and replay handling is edit-aware.

## Reproduction Assessment

Yes from source inspection. A synthetic Mattermost WebSocket payload with `event: "post_edited"` and valid `data.post` reaches the current dispatcher guard and returns before `onPosted`, while upstream Mattermost publishes edited posts with post data.

## Solution Assessment

Yes on the product boundary: the fix belongs in the Mattermost plugin WebSocket and replay path, not in core channel APIs. The implementation should be edit-aware rather than only widening the parser allow-list.
