## Summary

Keep open. Current main still drops Mattermost `post_edited` WebSocket events before parsing or mention routing, the latest release still has the same `posted`-only guards, and the linked closing implementation remains open and conflicting rather than merged.

## What This Changes

Mattermost should evaluate edited posts that add an @mention so the addressed agent wakes like it would on an original mentioned post.

## Best Possible Solution

Review, rebase, prove, and land https://github.com/openclaw/openclaw/pull/72513 or an equivalent Mattermost-only fix that routes edited posts to mention handling and uses edit-aware replay dedupe.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a `post_edited` WebSocket payload with `data.post` returns before `onPosted`, while Mattermost upstream publishes edited posts with serialized `post` data.

## Solution Assessment

Yes for the direction. The narrow maintainable solution belongs in the Mattermost plugin WebSocket dispatch plus replay-key path, and this issue should stay open until the linked PR or an equivalent fix is merged.
