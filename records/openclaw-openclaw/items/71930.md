## Summary

Keep open: current main and the latest release still accept only Mattermost `posted` WebSocket events, so edited messages that add an @mention are still dropped before the normal wake path. An open linked PR owns a plausible fix path, but it is not merged and is currently conflicting.

## What This Changes

The issue asks OpenClaw's Mattermost plugin to process `post_edited` WebSocket events so @mentions added by editing a message can wake the agent.

## Best Possible Solution

Land a refreshed fix that treats `posted` and `post_edited` as post-bearing events and distinguishes edited revisions in the replay guard without changing normal posted-message behavior.

## Reproduction Assessment

Yes at source level: current main has posted-only guards in both parsing and WebSocket dispatch, and Mattermost publishes a `post_edited` event with a post body. I did not run a live Mattermost workspace reproduction in this read-only review.

## Solution Assessment

Yes, the maintainable fix is to route edited post-bearing events into the existing post path and make replay dedupe edit-revision-aware. A WebSocket-only fix would be too narrow because the current replay guard keys only by post id.
