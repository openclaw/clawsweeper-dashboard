## Summary

Keep open: current main and v2026.6.8 still drop Mattermost `post_edited` events before mention detection, and the open closing PR is an implementation candidate rather than a merged fix.

## What This Changes

The issue asks for Mattermost edited messages that add a bot @mention to trigger the same agent wake flow as original posted mentions instead of being silently ignored.

## Best Possible Solution

A refreshed Mattermost plugin fix should route `post_edited` through the existing post handling path and include edit-aware replay dedupe plus live Mattermost proof.

## Reproduction Assessment

Yes at source level: current main has posted-only guards in both parsing and WebSocket dispatch, while Mattermost emits `post_edited` events with post payloads. I did not run a live Mattermost workspace reproduction in this read-only review.

## Solution Assessment

Yes, with one caveat: forwarding `post_edited` into the existing post path is the right layer, but the landed fix also needs edit-aware replay keys so an edited post is not deduped against the original.
