## Summary

Keep open. Current main still drops Mattermost `post_edited` WebSocket events before parsing, and open PR #72513 is the active closing implementation candidate rather than a reason to close this issue before merge.

## What This Changes

Mattermost should wake the agent when a message edit adds an @mention, or at least log that such an edited mention was ignored.

## Best Possible Solution

Review and land PR #72513, or an equivalent Mattermost-plugin-only fix, so `post_edited` payloads with updated post bodies reach parsing, mention gating, and edit-aware dedupe handling.

## Reproduction Assessment

Yes. A synthetic Mattermost WebSocket payload with `event: "post_edited"` follows the current source path to the guard at `extensions/mattermost/src/mattermost/monitor-websocket.ts:358` and returns before `onPosted` or mention detection can run.

## Solution Assessment

Yes for the product boundary: this belongs in the Mattermost plugin WebSocket and replay path, not in core channel APIs. The implementation should be edit-aware rather than only widening the parser event check.
