## Summary

Keep open: the latest PR head still has source-backed correctness blockers in worker startup, date-range accounting, pricing-cache freshness, and one localized UI-string regression, plus a security-sensitive worker config handoff.

## What This Changes

The branch replaces request-path usage transcript scans with a per-agent durable aggregate cache, detached rebuild worker, append-only refresh path, cache-status response fields, and Control UI cache-state messaging.

## Best Possible Solution

Keep the durable-cache direction, but fix worker entry resolution, preserve range-aware session summaries, add pricing/config-aware invalidation, localize new UI text, and pass only minimal secret-free worker inputs before merge.

## Reproduction Assessment

Yes. Current main deterministically scans usage-counted transcript files on usage.cost cache misses and per-session summaries on sessions.usage; a large profile with reset/deleted archives can exercise the reported slow path, though I did not live-measure the 90-244s timings.

## Solution Assessment

No. A durable aggregate cache is a maintainable direction, but this PR head is not the best mergeable solution until worker startup, range correctness, pricing freshness, localization, and config-handoff issues are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Fix the worker URL so refreshes can start:** `src/infra/session-cost-usage.ts:1077-1082`
  - body: `requestCostUsageCacheRefresh` runs from `src/infra/session-cost-usage.ts`, so `./infra/session-cost-usage-cache-worker.js` resolves under an extra nested `infra/` directory. Since the build entry emits `infra/session-cost-usage-cache-worker`, the detached refresh worker will start with a missing file and stale caches cannot rebuild through the request path.
  - confidence: 0.95
- **[P2] Preserve date ranges when reading session cache:** `src/gateway/server-methods/usage.ts:669-683`
  - body: `sessions.usage` used to pass `startMs` and `endMs` into `loadSessionCostSummary`, but the cached path now loads a full-session summary and only checks session overlap. A long-running session with any activity in range will contribute lifetime tokens and cost to that selected range.
  - confidence: 0.93
- **[P2] Invalidate cached costs when pricing inputs change:** `src/infra/session-cost-usage.ts:297-309`
  - body: Cache freshness only compares transcript size/mtime and session-summary presence, but cached entries are produced through config-dependent cost resolution and tiered-pricing recomputation. Changing model pricing without touching transcripts will keep serving old costs indefinitely.
  - confidence: 0.88
- **[P3] Localize the usage cache warning text:** `ui/src/ui/views/usage.ts:718-719`
  - body: This adds a new static Control UI warning in English instead of routing it through the existing `t(...)` locale surface. Non-English locale bundles will not get this user-visible cache-state message.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: Needs attention: the PR adds a detached worker and passes full runtime config into the child process environment.

Concerns:

- **[medium] Avoid passing full config through the worker env:** `src/infra/session-cost-usage.ts:1089`
  - body: `OPENCLAW_USAGE_COST_CACHE_CONFIG` receives `JSON.stringify(params?.config ?? {})` for a detached child. OpenClaw config can include API keys, tokens, headers, TLS material, and secret references, so the worker should receive only the minimal secret-free pricing data it needs or load through an existing safer config path.
  - confidence: 0.78
