## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Batch session summary refreshes before starting the rebuild:** `src/infra/session-cost-usage.ts:1126-1130`
  - body: `sessions.usage` calls this helper once per listed session, but each cold/stale call schedules only the current file and the refresh scheduler de-dupes by agent. On a cold cache, the first session wins, later stale files are ignored, and the dashboard can return null usage/totals for the rest until repeated refresh cycles. Batch all stale session files for the request or merge pending `sessionFiles` before the refresh starts.
  - confidence: 0.9
- **[P2] Include every pricing source in the cache fingerprint:** `src/infra/session-cost-usage.ts:157-185`
  - body: The durable cache fingerprint only serializes `config.models.providers`, but `resolveModelCostConfig` also uses packaged `models.json` and the gateway model-pricing cache. If those bundled or remote pricing sources change without transcript mtime/size changing, cached entries still look fresh and can keep serving stale or zero costs indefinitely.
  - confidence: 0.88
- **[P2] Move refresh work off the gateway event loop:** `src/infra/session-cost-usage.ts:1173-1178`
  - body: The refresh is scheduled with `setTimeout` but still runs `refreshCostUsageCache` inside the gateway Node process. Large JSONL scans and session-summary recomputation can still consume the same event loop the PR is trying to protect from dashboard stalls and delayed channel replies. Use an actual worker/process/thread or a bounded chunked scanner before treating this as fixed.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/gateway/server-methods/usage.ts:350-354`
  - body: This is a user-visible `fix`/performance change for gateway Usage responses and Control UI cache status, but the PR does not update `CHANGELOG.md`. Repo policy requires an active-version changelog entry for this kind of change before merge.
  - confidence: 0.94
