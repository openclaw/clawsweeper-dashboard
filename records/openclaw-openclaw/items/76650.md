## Summary

Keep open. Current main still has source-reproducible request-path transcript scans for `usage.cost` and `sessions.usage`, and this PR is a relevant implementation, but the current patch has merge-blocking worker startup, range correctness, cache invalidation, append-only summary, and security-handoff issues.

## What This Changes

The PR adds a durable per-agent usage-cost aggregate cache, detached refresh worker, cache-status fields on gateway usage responses, and Control UI stale/loading messaging.

## Best Possible Solution

Keep the durable cache direction, but revise it so worker startup is correct, cached summaries preserve date-range and derived-detail semantics, cache entries invalidate when pricing inputs change, and the worker receives only minimal secret-free inputs.

## Reproduction Assessment

Yes. Current main is source-reproducible: `usage.cost` and `sessions.usage` still route through transcript enumeration and JSONL scanning on gateway request paths; only the exact live latency numbers were not reproduced here.

## Solution Assessment

No. A durable aggregate cache is a maintainable fix direction, but this PR is not the best mergeable solution until the worker path, range semantics, pricing invalidation, append-only detail merging, and secret-free worker handoff are corrected.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Fix the worker URL so refreshes can start:** `src/infra/session-cost-usage.ts:1077-1082`
  - body: `requestCostUsageCacheRefresh` runs from `src/infra/session-cost-usage.ts`, so `./infra/session-cost-usage-cache-worker.js` resolves to an extra nested `infra/` directory in source and dist. The detached child starts with a missing file, leaving stale caches unable to rebuild through the request path this PR depends on.
  - confidence: 0.95
- **[P2] Preserve the selected date range when using session cache:** `src/gateway/server-methods/usage.ts:669-683`
  - body: `sessions.usage` used to pass `startMs` and `endMs` into `loadSessionCostSummary`, but the cached path now loads a full-session summary and only checks session overlap. A long-running session with any activity in range will contribute lifetime tokens and cost to that selected range.
  - confidence: 0.93
- **[P2] Invalidate cached costs when pricing inputs change:** `src/infra/session-cost-usage.ts:297-309`
  - body: Cache freshness only compares transcript size/mtime and session-summary presence, but cached entries are produced through config-dependent cost resolution and tiered-pricing recomputation. Changing model pricing without touching transcripts will keep serving old costs indefinitely.
  - confidence: 0.88
- **[P2] Merge full session details on append-only refreshes:** `src/infra/session-cost-usage.ts:871-920`
  - body: The append-only path marks a grown transcript fresh after merging only totals and `dailyBreakdown` into `sessionSummary`. Message counts, tool/model usage, latency, daily model usage, and quarter-hour buckets remain from the previous cache entry, so `sessions.usage` can serve stale details indefinitely for normal appended turns.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: Needs attention: the PR adds detached worker execution and passes full runtime config through the child environment.

Concerns:

- **[medium] Avoid serializing full config into the worker environment:** `src/infra/session-cost-usage.ts:1089`
  - body: `OPENCLAW_USAGE_COST_CACHE_CONFIG` receives the full runtime config for a detached child. OpenClaw config can contain sensitive provider API keys, auth tokens, and headers, so the worker should receive only minimal secret-free pricing data or use a safer existing config-loading path.
  - confidence: 0.78
