## Summary

Keep open: current main and the latest release still carry the bundled-only keyed-store gate, while this active implementation PR needs maintainer SDK-policy review, a rebase over the current keyed-store API, and real behavior proof before merge.

## What This Changes

This PR opens plugin keyed state to non-bundled plugins by making stores runtime-plugin-owned, removing plugin-controlled namespaces, and updating channel callers, docs, tests, API baseline, and changelog.

## Best Possible Solution

Land a maintainer-approved runtime-owned durable state SDK contract that preserves `registerIfAbsent`, resolves the rebase conflicts, aligns docs/tests/API baseline, and includes real behavior proof from a community plugin setup.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature PR. Source inspection gives a high-confidence behavior check: current main and v2026.5.7 still reject workspace-origin keyed-store calls, while the PR changes that SDK boundary.

## Solution Assessment

Unclear for merge as-is. Binding store ownership to the runtime plugin id is safer than trusting plugin-supplied owner data, but the branch must be rebased to preserve current `registerIfAbsent` behavior and needs maintainer approval for the public SDK policy change.
