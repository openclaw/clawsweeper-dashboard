## Summary

Keep open. Current main still has the bundled-only `openKeyedStore` gate, and this PR changes the public Plugin SDK trust boundary by allowing community native plugins to opt into durable keyed state, so it needs maintainer policy review rather than conservative cleanup.

## What This Changes

The PR adds a `contracts.usesKeyedStore` manifest opt-in, permits opted-in community plugins to call `api.runtime.state.openKeyedStore` with a 500-entry namespace cap, adjusts contract filtering, and updates runtime docs, changelog, and tests.

## Best Possible Solution

Have a maintainer decide the Plugin SDK policy; if accepted, land the manifest opt-in with manifest-reference docs and focused coverage for manifest parsing, access control, and quota behavior.

## Reproduction Assessment

Not applicable as a feature PR rather than a bug report. Source inspection gives a high-confidence behavior check: current main rejects workspace-origin plugins, and this PR changes that access gate.

## Solution Assessment

Unclear until maintainer policy review. The implementation is mechanically narrow and follows the related request, but exposing restart-safe durable state to community native plugins is a public SDK and trust-boundary decision.
