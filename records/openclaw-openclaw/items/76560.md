## Summary

Keep open. Current main still rejects non-bundled plugins, and this PR changes a public Plugin SDK durable-state capability that needs maintainer policy review rather than cleanup closure.

## What This Changes

The PR removes the bundled-only `openKeyedStore` runtime gate, binds keyed stores to the runtime plugin id, removes plugin-controlled namespaces, adds default row-budget behavior, and updates docs, API baseline, tests, and bundled channel callers.

## Best Possible Solution

Have a maintainer decide the public SDK durable-state policy, then either land the runtime-bound single-store design with aligned PR text/docs/tests or revise it to the chosen opt-in and quota model.

## Reproduction Assessment

Not applicable as a feature PR. Source inspection gives a high-confidence behavior check: current main rejects workspace-origin plugins and has a test asserting that bundled-only rejection.

## Solution Assessment

Unclear until maintainer policy review. The current PR head is mechanically coherent from diff inspection, but it no longer matches the manifest opt-in design described in the PR body and expands a public SDK trust boundary.
