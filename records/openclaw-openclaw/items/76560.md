## Summary

Keep open. The PR addresses a real SDK capability gap, but it changes public durable-state policy and the current diff drops shipped namespace-backed state while the PR body still describes a different manifest opt-in design.

## What This Changes

The PR removes the bundled-only `openKeyedStore` gate, binds keyed stores to the runtime plugin id with no namespace option, adds defaults/types/docs/changelog/API baseline updates, and adapts bundled channel persistence callers and tests.

## Best Possible Solution

Have a maintainer decide the public SDK durable-state policy; if accepted, land the runtime-bound namespace-free contract with migration coverage for shipped namespaced rows, updated PR text, docs/changelog/API baseline, and focused plugin-state plus channel persistence validation.

## Reproduction Assessment

Not applicable as a feature PR. Source inspection of current main gives a high-confidence behavior check: workspace-origin plugins still hit the bundled-only `openKeyedStore` rejection path, and a runtime test expects that rejection.

## Solution Assessment

Unclear until maintainer policy review. The runtime-bound, namespace-free design is mechanically narrower than plugin-controlled namespaces, but it needs shipped-state migration coverage and the PR description must match the implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve shipped keyed-store namespaces during migration:** `src/plugin-state/plugin-state-store.ts:43`
  - body: Current releases already persist Discord and Slack restart state under explicit namespaces such as `discord.components`, `discord.modals`, and `slack.thread-participation`. Collapsing every plugin store to `default` makes those rows unreadable after upgrade, so active Discord interactions and Slack thread participation can be lost despite the restart-safe contract. Add a compatibility migration or read path for the shipped namespaces before removing namespace access.
  - confidence: 0.87
