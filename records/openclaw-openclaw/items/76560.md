## Summary

Keep open. Current main still rejects workspace-origin plugins, so this PR is not obsolete; the branch is a focused SDK change, but it needs maintainer approval because the final diff exposes durable keyed state to all loaded native plugins and no longer matches the manifest opt-in design described in the PR body.

## What This Changes

The PR changes plugin keyed state to be runtime-plugin-owned without plugin-controlled namespaces, removes the bundled-only runtime gate, adds default row-budget handling and SDK types, updates docs/changelog/tests, and adapts bundled channel callers.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Unclear until maintainer policy review. The runtime-bound, namespace-free design is mechanically narrower than plugin-controlled namespaces, but allowing non-bundled native plugins to use durable SQLite state is a public SDK and trust-boundary decision.
