## Best Possible Solution

Compose the Google thinking sanitizer only for the plain pi-ai Google simple-completion path while preserving explicit OpenClaw transport-aware routing, then add a regression test for Google transport requirements and redacted after-fix payload proof.

## Reproduction Assessment

Yes, at source level: pi-ai 0.75.3 returns thinkingBudget -1 for unrecognized Google aliases, current main does not wrap plain google-generative-ai simple completions, and the linked issue includes captured proxy payloads. I did not run a live Google request because this review is read-only and credential-dependent.

## Solution Assessment

No, not as written. Reusing the existing sanitizer is the right direction, but the unconditional Google early return before the transport-aware branch drops existing configured transport behavior.
