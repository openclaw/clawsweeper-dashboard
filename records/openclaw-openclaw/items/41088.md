## Summary

Keep open: current main and v2026.6.8 still omit the requested Chrome Optimization Guide/WasmTts disable flags from both local managed Chrome and the sandbox Chromium entrypoint, while `browser.extraArgs` only covers the local managed workaround. ## What This Changes

The issue asks OpenClaw to stop Chrome Optimization Guide and WasmTts model downloads in headless/server browser launches, either by default flags or a browser config option.

## Best Possible Solution

Make one browser-plugin and sandbox-launch policy decision, then either add a tested shared disabled-feature list to both launch paths or document the supported `browser.extraArgs` and custom-entrypoint workaround without adding a new config key.
