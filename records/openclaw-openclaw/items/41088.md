## Summary

This should stay open, but not as the original four-flag patch: current main still lacks a default suppression policy, while current Chromium shows one proposed flag was removed and the Wasm TTS control has a different name and newer lazy-install behavior. The issue needs a current live reproduction and a small maintainer product decision rather than stale closure or automatic repair.

## What This Changes

Prevent unnecessary Chrome on-device model and Wasm TTS downloads in OpenClaw-managed headless or server browser environments.

## Best Possible Solution

Reproduce with current stable Chrome using a fresh OpenClaw profile, then adopt the minimal current Chromium controls in the browser plugin and sandbox launcher with focused launch-argument tests and user-visible docs; do not add another config key because `browser.extraArgs` already covers exceptional flags.

## Reproduction Assessment

No; current source proves OpenClaw does not apply suppression, but a fresh-profile current Chrome run must show whether and when the Optimization Guide or Wasm TTS assets actually download.

## Solution Assessment

No, not exactly as proposed; `browser.disableModelDownloads` duplicates `browser.extraArgs`, `OptimizationGuideModelDownloading` was removed upstream, and current Wasm TTS uses a different kill switch with lazy-install behavior. A verified current default-argument policy is the better path.
