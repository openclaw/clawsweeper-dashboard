## Summary

Keep open: current main and the latest release still do not add the requested Chrome Optimization Guide/WasmTts disable flags to the managed browser or sandbox Chromium launch paths, and the linked implementation attempt is closed unmerged. The remaining decision is whether OpenClaw should make these Chrome feature disables a default for headless/server launches or keep them as documented operator-supplied `browser.extraArgs`.

## What This Changes

The issue requests disabling Chrome Optimization Guide and WasmTts model downloads for OpenClaw-managed headless/server browser launches, either by default or through a config option.

## Best Possible Solution

Make one browser-plugin and sandbox-launch decision, then implement a shared disabled-feature list with focused launch-args coverage and docs, without adding a new config key unless `browser.extraArgs` is proven insufficient.

## Reproduction Assessment

No. Source inspection shows current main does not pass the requested Chrome disable-feature flags, but I did not establish a live current-main reproduction of Chrome 145 downloading the models under WSL2/Xvfb or the sandbox browser.

## Solution Assessment

Unclear. The closed PR's direction is plausible, but the best fix depends on a maintainer policy decision about default Chrome launch behavior versus documenting the existing `browser.extraArgs` workaround.
