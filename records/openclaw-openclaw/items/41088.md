## Summary

Keep open: current main and the latest release still omit the requested Chrome Optimization Guide/WasmTts disable flags from both managed-browser and sandbox Chromium launch paths. The existing `browser.extraArgs` path is a valid operator workaround for local managed profiles, but it does not settle whether OpenClaw should add these disables by default for headless/server use.

## What This Changes

The issue asks OpenClaw to prevent Chrome Optimization Guide and WasmTts model downloads in headless/server browser launches, either by default flags or a new config option.

## Best Possible Solution

Make one browser-plugin and sandbox-launch policy decision, then either add a tested shared disabled-feature list to both launch paths or document the supported `browser.extraArgs`/custom-entrypoint workaround without adding a new config key.

## Reproduction Assessment

No. Source inspection shows current main does not pass the requested Chrome disable-feature flags, but I did not establish a live current-main reproduction of Chrome 145 downloading the models under WSL2/Xvfb or the sandbox browser.
