## Summary

Keep open. Current main still records fallback transitions but only prepends model fallback and fallback-cleared reply notices inside the verbose-mode path; the requested independent opt-in config is absent, and the issue is paired with open implementation PR #69399 using closing syntax.

## What This Changes

Add a default-false configuration option that allows model fallback and fallback-cleared notices to appear in normal non-verbose replies.

## Best Possible Solution

Review #69399 and either land it or request a revised implementation that adds a default-false agent-defaults option, updates config types/schema/help/docs, and lets runReplyAgent prepend fallback and fallback-cleared notices when verbose mode is enabled or the opt-in is true while preserving once-per-active-fallback behavior.

## Reproduction Assessment

Yes. Current e2e coverage mocks a fallback transition and explicitly asserts that verbose-off replies do not contain Model Fallback text, which reproduces the reported UX gap without live provider failures.

## Solution Assessment

Yes in overall direction: reusing the existing transition tracking and notice formatter behind a default-false opt-in is the narrowest maintainable path. The unresolved part is the maintainer-approved config key and placement.
