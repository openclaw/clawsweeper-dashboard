## Summary

Keep open. Current main still only prepends model fallback and fallback-cleared reply notices through the verbose notice path, the requested independent default-off config option is absent from config/docs, and open PR #69399 still uses closing syntax for this issue.

## What This Changes

The issue requests a default-off config option that surfaces existing model fallback notices even when verbose mode is disabled.

## Best Possible Solution

Land one canonical implementation, likely via #69399 or a maintainer-directed revision, that adds a default-false opt-in, updates config schema/help/docs/changelog, and preserves once-per-active-fallback notice behavior.

## Reproduction Assessment

Yes. Current main has a source-backed path and e2e coverage that mock a fallback transition and assert verbose-off replies do not contain Model Fallback text.

## Solution Assessment

Yes in direction. Reusing the existing transition tracking and formatter behind a default-false opt-in is the narrowest maintainable solution, pending maintainer approval of the config surface.
