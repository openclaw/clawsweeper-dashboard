## Summary

Keep open. Current main still records fallback transitions but only prepends the user-facing fallback and fallback-cleared notices inside the verbose path, the independent opt-in config is absent from config/docs, and open PR #69399 uses closing syntax for this issue.

## What This Changes

Add a default-off configuration option that allows existing model fallback and fallback-cleared notices to appear in normal non-verbose replies.

## Best Possible Solution

Land one canonical implementation that keeps default behavior unchanged, adds a clearly named default-false opt-in, updates config schema/help/docs/changelog, and preserves once-per-active-fallback notice behavior.

## Reproduction Assessment

Yes. Current e2e coverage mocks a fallback transition and explicitly asserts that verbose-off replies do not contain Model Fallback text, reproducing the reported UX gap without live provider failures.

## Solution Assessment

Yes in direction. Reusing the existing transition tracking and notice formatter behind a default-false opt-in is the narrowest maintainable path, pending maintainer approval of the config key and placement.
