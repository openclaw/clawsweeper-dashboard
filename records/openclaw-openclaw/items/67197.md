## Summary

Keep open: current main still hides fallback and fallback-cleared reply notices behind verbose mode, has no independent fallback-notice config surface, and the open closing implementation PR at https://github.com/openclaw/openclaw/pull/69399 should be reviewed or replaced before this issue is closed.

## What This Changes

Request a default-off config that lets users receive one-time model fallback notices without enabling verbose replies.

## Best Possible Solution

Review and land, revise, or replace https://github.com/openclaw/openclaw/pull/69399 as the canonical implementation, keeping default behavior unchanged and documenting the chosen config surface.

## Reproduction Assessment

Yes. Source inspection and current e2e assertions show that verbose-off fallback and fallback-cleared replies suppress the notice text while lifecycle events still emit.

## Solution Assessment

Unclear as a final implementation choice. A default-false opt-in reusing the existing fallback transition tracker is the narrow maintainable direction, but maintainers still need to settle the public config key and choose between the broad PR and the Telegram-specific alternative.
