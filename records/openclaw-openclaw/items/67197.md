## Summary

Keep open. Current main still tracks fallback transitions but only prepends user-visible fallback and fallback-cleared notices when verbose mode is enabled, the independent opt-in config is absent, and open PR #69399 already references this issue with closing syntax.

## What This Changes

Request a default-false config option that lets model fallback notices appear in normal, non-verbose replies while preserving once-per-active-fallback behavior.

## Best Possible Solution

Review #69399 and either land it or request a revised implementation that adds the opt-in config, updates config/schema/docs, and reuses the existing fallback transition notice path.

## Reproduction Assessment

Yes. The current e2e tests mock fallback transitions and explicitly assert that verbose-off replies omit Model Fallback and fallback-cleared text while lifecycle events still emit.

## Solution Assessment

Yes in direction: a default-false opt-in around the existing transition tracker and notice formatter is the narrowest maintainable solution. The unresolved part is the maintainer-approved config key and placement.
