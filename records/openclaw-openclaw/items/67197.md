## Summary

Keep open. Current main still only prepends the user-visible model fallback and fallback-cleared reply notices when verbose mode is enabled, the independent default-false notice config does not exist, and https://github.com/openclaw/openclaw/pull/69399 is still open with closing syntax for this issue.

## What This Changes

Add a default-false config option that lets normal, non-verbose replies show existing model fallback and fallback-cleared notices once per active fallback state.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/69399 or request a revised canonical implementation that adds a default-false config, updates config schema/help/docs, and preserves the existing once-per-active-fallback behavior.

## Reproduction Assessment

Yes, from source inspection. Current main gates the reply-visible fallback and fallback-cleared notices on verboseEnabled, while tests still assert verbose-off suppression with lifecycle events emitted.

## Solution Assessment

Yes in direction, but not yet landed. Reusing the existing fallback transition state and notice formatter behind a default-false opt-in is a narrow path, while the public config placement still needs maintainer agreement.
