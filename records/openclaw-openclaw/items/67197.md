## Summary

Keep open. Current main still tracks fallback transitions but only prepends fallback and fallback-cleared notices when verbose mode is enabled; the requested independent opt-in config is absent, and an open linked implementation PR still owns the direct fix path.

## What This Changes

The issue asks for a default-off config option that lets users see one-time model fallback notices without enabling verbose replies.

## Best Possible Solution

Land or replace one canonical implementation that keeps default behavior unchanged, adds a clearly named default-false opt-in, updates config schema/help/docs/changelog, and preserves once-per-active-fallback notice behavior.

## Reproduction Assessment

Yes, from source inspection. Current main gates both user-facing fallback notices on `verboseEnabled`, and current e2e tests assert that verbose-off replies suppress fallback and fallback-cleared notice text while lifecycle events still emit.

## Solution Assessment

Yes in direction. Reusing the existing transition tracking and notice formatter behind a default-false opt-in is the narrowest maintainable path, but maintainers still need to choose the public config placement and whether the fix should be broad or channel-specific.
