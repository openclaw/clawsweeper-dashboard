## Summary

Keep open. Current main and the latest release still route WhatsApp mention gating through an extractor that returns undefined when no `contextInfo.mentionedJid` array is found, while the merged adjacent fix only covers already-populated mention IDs.

## What This Changes

Issue reports that WhatsApp group native @-mentions can be silently ignored when mention extraction returns null despite a WhatsApp mention flag being present.

## Best Possible Solution

Capture a redacted native WhatsApp group mention payload that reproduces `mentionedJids: null`, then update the WhatsApp plugin extractor and regression tests to read the canonical Baileys field without leaking quoted-message mentions.

## Reproduction Assessment

No high-confidence live reproduction is available from current main because the raw WhatsApp/Baileys payload is missing. Source inspection does show that if native mention extraction returns undefined, the group mention gate will treat the message as unmentioned and skip it when mentions are required.

## Solution Assessment

Unclear as an implementation until a native payload is available. The likely layer is `extensions/whatsapp/src/inbound/extract.ts`, but choosing `groupMentions` or another field without payload proof would be guesswork.
