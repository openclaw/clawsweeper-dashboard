## Summary

Keep open. Current main fixes adjacent WhatsApp group mention cases when `mentionedJids` are already present, but this issue reports the extractor returning null; the current extractor still has a narrow hard-coded mention read path and there is no redacted raw payload proving the reported `+38m` mention metadata is now captured.

The issue reports that WhatsApp group tap-mentions do not wake the bot because `extractMentionedJids()` returns null even though the message appears to contain mention metadata.

## Best Possible Solution

Obtain a redacted current Baileys inbound payload for this mention shape, then add a focused extractor regression test and update `extractMentionedJids()` to read the canonical current-message mention metadata without leaking quoted-message mentions.
