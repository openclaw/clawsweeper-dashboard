Keep open: current main still has the reported extractor-null risk. The adjacent #49317 fix is present, but it only helps after inbound mention JIDs already exist on the queued WhatsApp message; this issue is about the earlier `extractMentionedJids()` path returning no JIDs.

## What This Changes

The issue asks OpenClaw to detect WhatsApp group @mentions when the inbound payload indicates a mention but `mentionedJids` is extracted as null.

## Best Possible Solution

Fix the WhatsApp-owned inbound mention extractor and add a regression test for the actual current-message mention metadata location, while preserving quoted-message exclusion and the #49317 group self-chat behavior.

Unclear for a high-confidence live reproduction because the exact raw Baileys payload shape is missing. Source inspection does reproduce the failure path: if `extractMentionedJids()` returns undefined, the inbound queue has no mention metadata and required-mention group gating skips the message.

## Solution Assessment
