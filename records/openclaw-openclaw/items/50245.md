## Summary

Keep open. Current main still produces and types `durationMs`, but the human-readable `openclaw health` path does not print the top-level gateway probe duration; the issue also has an open paired implementation PR with closing syntax, so it should not be closed before that PR lands.

## What This Changes

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/50250, or an equivalent narrow patch, so the text renderer prints `summary.durationMs` while preserving the JSON contract.

Yes. Source inspection can reproduce the gap by mocking `callGateway` to return a `HealthSummary` with `durationMs` and calling `healthCommand({ json: false })`; current main has no text-output path for that top-level duration.

## Solution Assessment

Yes. Printing the already-required `summary.durationMs` in the non-JSON health renderer is the narrow maintainable fix; no protocol, schema, gateway RPC, or config change is needed.
