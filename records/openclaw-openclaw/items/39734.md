## Summary

Keep open: current main and v2026.5.22 still lack a supported Bedrock config path for custom `anthropic_beta` values. The original 1M-context example appears partly obsolete, but the broader provider-owned beta passthrough remains unresolved and overlaps an open default-beta PR.

## What This Changes

The issue requests a configurable way to pass custom Anthropic beta flags through Amazon Bedrock Converse `additionalModelRequestFields.anthropic_beta`.

## Best Possible Solution

Define a Bedrock-owned, validated `params` contract for Anthropic beta values, merge and dedupe them into `additionalModelRequestFields.anthropic_beta`, compose with existing Bedrock payload wrappers, and document the supported config shape.

## Reproduction Assessment

Not applicable: this is a feature/config request rather than broken documented behavior. Source and release inspection give high confidence that current main and v2026.5.22 do not read or forward custom Bedrock beta flags from config.

## Solution Assessment

Unclear until maintainers choose the public API shape. A narrow Bedrock-owned `params.anthropicBeta`-style contract that mirrors direct Anthropic beta merging is safer than a broad unvalidated `providerFields` passthrough.
