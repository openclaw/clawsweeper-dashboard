## Summary

Keep open: current main and v2026.6.5 still lack a supported Bedrock config path for custom `anthropic_beta` values, and the related open Bedrock beta PR is conflicting and does not cover the requested per-model 1M-context flag. The report is valid at source level, but the final public option name and merge semantics need maintainer approval.

## What This Changes

Request to let Amazon Bedrock model configuration pass custom Anthropic beta flags into Converse `additionalModelRequestFields`.

## Best Possible Solution

Add a narrow, documented Amazon Bedrock per-model `params` option for Anthropic beta identifiers or request fields, merge it deterministically with provider-generated fields, and prove both the emitted Converse payload and a live 1M-context Bedrock request.

## Reproduction Assessment

Yes at source level: current main has no path from Bedrock model/plugin configuration to a custom `anthropic_beta` list, and AWS docs confirm this belongs in `additionalModelRequestFields`. I did not run a live long-context Bedrock request in this read-only review.

## Solution Assessment

Unclear: the reported need is valid, but the proposed generic `providerFields` object is broader than the existing per-model `params` seam. A typed provider-owned option with deterministic merging is the safer maintainable direction, pending maintainer approval.
