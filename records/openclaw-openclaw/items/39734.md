## Summary

Keep open: current main and the latest release still do not provide a custom Bedrock `anthropic_beta` or `additionalModelRequestFields` passthrough, and the open related PR only hardcodes default betas rather than solving this config contract.

## What This Changes

The issue requests a Bedrock model-config path for forwarding custom Anthropic beta flags into Bedrock Converse `additionalModelRequestFields`.

## Best Possible Solution

Define a Bedrock-plugin-owned config contract for custom beta/request fields, merge and de-duplicate user-supplied betas with generated fields, add focused unit coverage and Bedrock docs, then validate with redacted live Bedrock output.

## Reproduction Assessment

Yes for the source-level gap: current main and v2026.6.8 have no path from model config to Bedrock `additionalModelRequestFields.anthropic_beta`. I did not run a live Bedrock call in this read-only review.

No closeable solution is present yet. The requested config-driven shape is plausible, but maintainers still need to choose the public config contract and decide how it composes with generated Bedrock request fields.
