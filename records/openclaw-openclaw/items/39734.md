## Summary

Keep open: current main and v2026.6.8 still do not provide a Bedrock config path for custom `anthropic_beta` or generic `additionalModelRequestFields`, while the Bedrock plugin already advertises 1M context for the affected Claude refs. ## What This Changes

The issue requests a Bedrock model-config path for custom Anthropic beta flags in `additionalModelRequestFields`.

## Best Possible Solution

Define a Bedrock-owned request-field merge contract, add focused runtime/docs/tests for `anthropic_beta` de-duplication and 1M Bedrock behavior, and require redacted live Bedrock proof before landing.

## Reproduction Assessment

Yes, at source level: current main has no Bedrock config or extra-param path that can merge custom `anthropic_beta` into `additionalModelRequestFields`, while Bedrock discovery marks the reported Claude refs as 1M-capable. I did not independently rerun a live AWS Bedrock request in this read-only review.

## Solution Assessment

Unclear. A configurable Bedrock request-field path is likely the right product direction, but the exact surface should be maintainer-approved because generic `providerFields` expands public config more than a narrower typed `anthropicBeta` or `context1m` repair.
