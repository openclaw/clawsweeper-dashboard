## Summary

Keep open: current main still lacks a user-configurable Bedrock `anthropic_beta` passthrough. The 1M-context example may be obsolete for current Claude GA models and the related default-beta PR is open, but neither resolves the broader custom Bedrock beta config request.

## What This Changes

The issue requests a configurable way to pass Anthropic Bedrock beta flags into Bedrock Converse API `additionalModelRequestFields`.

## Best Possible Solution

Add a Bedrock-owned, validated params path that merges and dedupes configured betas into `additionalModelRequestFields.anthropic_beta`, composes with guardrails/default betas/app-profile traits, and documents the contract.

## Reproduction Assessment

Not applicable: this is a feature/config request, not broken documented behavior. Source inspection gives high confidence that current main lacks the requested Bedrock custom-beta passthrough.

## Solution Assessment

Unclear until maintainers choose the public config shape. A narrow Bedrock-owned params field looks safer than the proposed broad `providerFields` passthrough, especially with the related default-beta PR touching the same payload path.
