## Summary

Keep open. Current main still lacks a configurable Amazon Bedrock Converse passthrough for `anthropic_beta` or general additional model request fields; the 1M-context example may be less urgent now, but the broader provider-owned config request remains concrete and unimplemented.

## What This Changes

Allow Bedrock Claude model configuration to pass custom Anthropic beta flags into Bedrock Converse `additionalModelRequestFields` without patching installed provider code.

## Best Possible Solution

Add a Bedrock-plugin-owned, schema-validated passthrough on the existing model `params` surface, merge it into Bedrock Converse `additionalModelRequestFields`, and document the supported config while keeping the core model schema generic.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `providerFields` is not accepted by the strict model schema, and no existing Bedrock config or model `params` path is read into Converse `additionalModelRequestFields`.

## Solution Assessment

No for the exact `providerFields` proposal. The narrower maintainable solution is likely a Bedrock-owned `params` passthrough or explicitly documented provider config field, coordinated with the adjacent open default-beta PR.
