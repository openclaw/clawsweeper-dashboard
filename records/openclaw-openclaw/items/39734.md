## Summary

Keep open: current main and the latest release still lack a supported Amazon Bedrock config path for custom Anthropic beta flags. The original 1M-context example is less compelling after the GA-model discussion, but the broader configurable Bedrock beta passthrough remains unresolved and the related default-beta PR does not replace it.

## What This Changes

Request to let Amazon Bedrock model config provide custom Anthropic beta flags that are forwarded as `additionalModelRequestFields.anthropic_beta`.

## Best Possible Solution

Define a Bedrock-owned, validated `params.anthropicBeta`-style contract, merge and dedupe configured values into `additionalModelRequestFields.anthropic_beta`, compose with existing Bedrock wrappers, and document the supported config shape.

## Reproduction Assessment

Not applicable: this is a feature/config request rather than broken documented behavior. Source inspection gives high confidence that current main can pass provider params to plugins but the Bedrock plugin does not read or forward custom beta flags.

## Solution Assessment

Unclear until maintainers choose the public config API. The proposed `providerFields` name does not match the current config seam, so a narrow Bedrock-owned `params.anthropicBeta` contract looks safer than a broad unvalidated passthrough.
