## Summary

Keep open: current main does not expose configurable Bedrock `anthropic_beta` passthrough; the 1M-preview use case may now be obsolete, but the broader custom beta-flag feature is still unresolved and only partially overlaps with open #69491.

## What This Changes

Request to let Amazon Bedrock Claude model config specify custom `anthropic_beta` flags that are merged into Converse `additionalModelRequestFields`.

## Best Possible Solution

Have the Bedrock provider own a documented, narrow beta-flag config surface, likely via existing model `params`, and merge validated string-array values into `additionalModelRequestFields` without adding a broad raw request-body passthrough unless maintainers explicitly want that.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection does give high confidence that current main lacks a custom `anthropic_beta` config path for Bedrock Converse payloads.

## Solution Assessment

Unclear. The requested direction is plausible, but the proposed `providerFields` shape is not obviously the best fit because current OpenClaw already has `agents.defaults.models["provider/model"].params` and provider-owned `wrapStreamFn` hooks for this kind of request shaping.
