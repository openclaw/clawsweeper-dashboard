## Summary

Keep open: current main and the latest shipped release still lack a Bedrock-owned config path for custom `anthropic_beta` or generic `additionalModelRequestFields`. The original 1M-context example may no longer be the strongest use case, but the broader provider-beta passthrough request remains unresolved and needs maintainer API/product judgment.

## What This Changes

The issue requests a user-configurable way to forward Bedrock Anthropic beta flags from model config into Converse `additionalModelRequestFields`.

## Best Possible Solution

Define a Bedrock-owned, validated model/provider `params` contract that merges and dedupes configured beta flags into `additionalModelRequestFields.anthropic_beta`, composes with existing guardrail, service-tier, thinking, and app-profile payload wrappers, and documents the supported config shape.

## Reproduction Assessment

Not applicable: this is a feature/config request rather than a failing documented behavior. Source and shipped-release inspection give high confidence that OpenClaw currently does not read or forward custom Bedrock beta flags from config.

## Solution Assessment

Unclear until maintainers choose the public API shape. A narrow, validated Bedrock-owned `params` contract looks safer than a broad unvalidated `providerFields` passthrough.
