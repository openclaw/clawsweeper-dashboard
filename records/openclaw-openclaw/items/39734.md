## Summary

Keep open: current main and v2026.6.5 still lack a supported way to send a per-model 1M-context Anthropic beta through Bedrock Converse. The recent live report makes the remaining Bedrock-specific need concrete, while the related open PR only hardcodes other default betas and is not a viable replacement. Likely related people include Peter Steinberger for recent Bedrock request-field work and Vincent Koc for the plugin extraction, with high and medium confidence respectively.

## What This Changes

Add a supported per-model way to pass Anthropic beta flags through Amazon Bedrock Converse requests.

## Best Possible Solution

Add a narrow, documented Amazon Bedrock model parameter for Anthropic beta identifiers using the existing `params` surface, merge it deterministically with plugin-required beta values, and verify both the emitted Converse payload and a live 1M-context request.

## Reproduction Assessment

Yes at source level: current main has no path from per-model configuration to `anthropic_beta`, and the latest issue comment provides a concrete live Bedrock comparison showing the dated flag is accepted while an incorrect flag is rejected. This review did not independently execute a full long-context Bedrock request.

## Solution Assessment

No, the proposed generic `providerFields` object is broader than necessary. Reusing existing per-model `params` for a typed, plugin-owned beta list is the narrower maintainable direction, subject to maintainer approval of naming and merge semantics.
