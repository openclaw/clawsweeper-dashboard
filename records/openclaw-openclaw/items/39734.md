## Summary

Keep open: current main still does not provide a user-configurable Amazon Bedrock `anthropic_beta` passthrough. The original 1M-context example is partly obsolete for known Claude 4.6 Bedrock models, but the broader custom-beta config request remains real and needs a maintainer API decision.

## What This Changes

Request adding configurable Amazon Bedrock `anthropic_beta` flags for Claude Converse requests, originally to opt into the 1M context beta without patching installed provider code.

## Best Possible Solution

Add a provider-local, validated Bedrock parameter path that merges and dedupes configured betas into `additionalModelRequestFields.anthropic_beta`, composes with guardrails/default betas/app-profile traits, and documents the supported config.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks a Bedrock custom-beta passthrough, while AWS ConverseStream supports `additionalModelRequestFields` for model-specific request parameters. ([docs.aws.amazon.com](https://docs.aws.amazon.com/bedrock/latest/APIReference/API_runtime_ConverseStream.html?utm_source=openai))

## Solution Assessment

Unclear until maintainers choose the public config shape. A narrow Bedrock-owned `params.anthropicBeta`-style implementation looks more maintainable than the proposed broad `providerFields` passthrough, especially with the default-beta PR touching the same payload path.
