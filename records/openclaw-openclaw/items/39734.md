## Summary

Keep open: current main still lacks a configurable Bedrock `anthropic_beta` passthrough. The 1M-context use case may be obsolete for GA Claude 4.6 models, but the broader request for user-controlled Bedrock beta flags remains unimplemented and is only partially related to the open default-beta PR.

## What This Changes

Request configurable Anthropic beta flags for Amazon Bedrock model definitions so OpenClaw can forward them through Bedrock Converse `additionalModelRequestFields`.

## Best Possible Solution

Add a provider-local Bedrock implementation that mirrors the existing Anthropic `params.anthropicBeta` pattern, merges/dedupes into `additionalModelRequestFields.anthropic_beta`, documents the Bedrock-specific behavior, and preserves app-profile/guardrail payload composition.

## Reproduction Assessment

Not applicable as a feature request, but source inspection gives high confidence that the requested Bedrock passthrough is absent on current main. The existing runtime surfaces show where it would be implemented.

## Solution Assessment

Unclear until maintainers choose the public config shape. The narrowest maintainable direction appears to be Bedrock-owned `params.anthropicBeta` support rather than a broad arbitrary `providerFields` passthrough.
