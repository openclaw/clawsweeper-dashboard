## Summary

Keep open: current main and v2026.5.20 still do not provide a Bedrock config path for custom `anthropic_beta` values. The 1M-context example appears partly obsolete for current Claude 4.6 models, but the broader provider-owned beta passthrough remains unresolved and needs maintainer API judgment.

## What This Changes

The issue requests configurable Amazon Bedrock Anthropic beta flags that are forwarded into Converse `additionalModelRequestFields.anthropic_beta` without patching OpenClaw or pi-ai after upgrades.

## Best Possible Solution

Define a Bedrock-owned, validated `params` contract for Anthropic beta values, merge and dedupe them into `additionalModelRequestFields.anthropic_beta`, compose with existing Bedrock payload wrappers, and document the supported config shape.

## Reproduction Assessment

Not applicable: this is a feature/config request rather than broken documented behavior. Source and release inspection give high confidence that current main and v2026.5.20 do not read or forward custom Bedrock beta flags from config.

## Solution Assessment

Unclear until maintainers choose the public API shape. A narrow Bedrock-owned `params` contract that mirrors direct Anthropic beta merging looks safer than a broad unvalidated `providerFields` passthrough.
