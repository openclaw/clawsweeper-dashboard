## Summary

Keep open: current main does not expose a documented Amazon Bedrock `anthropic_beta` passthrough, and the related open PR only forwards default Anthropic betas for Bedrock tool streaming rather than solving configurable per-user beta flags.

## What This Changes

The issue asks for Bedrock model configuration to accept custom Anthropic beta flags and merge them into Converse API `additionalModelRequestFields`.

## Best Possible Solution

Define a Bedrock-owned, documented request-parameter surface that maps vetted user-configured Anthropic beta values into `additionalModelRequestFields` without hardcoding core behavior.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection confirms current main lacks the requested Bedrock custom beta passthrough.

## Solution Assessment

Unclear. The requested direction is plausible, but maintainers should first choose whether the right surface is existing model `params`, plugin config, or a narrower Bedrock-specific allowlist before implementation.
