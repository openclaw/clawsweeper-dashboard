## Summary

Keep open: current main still has no Bedrock-owned custom `anthropic_beta` or generic `additionalModelRequestFields` config path. The original 1M-context example may be obsolete for current Claude 4.6 Bedrock models, but the broader provider-beta passthrough request remains unresolved and needs a maintainer API/product decision.

## What This Changes

The issue requests configurable Bedrock Anthropic beta flags, likely from model config, merged into Converse `additionalModelRequestFields`.

## Best Possible Solution

Define a Bedrock-owned, validated model params contract that merges and dedupes configured beta flags into `additionalModelRequestFields.anthropic_beta`, composes with guardrails/default betas/app-profile traits, and documents the supported config shape.

## Reproduction Assessment

Not applicable: this is a feature/config request rather than a failing documented behavior. Source inspection on current main and `v2026.5.12` gives high confidence that Bedrock does not currently read or forward custom beta flags from config.

## Solution Assessment

Unclear until maintainers choose the public API shape. A narrow Bedrock-owned params field or structured `additionalModelRequestFields` contract looks safer than a broad unvalidated `providerFields` passthrough.
