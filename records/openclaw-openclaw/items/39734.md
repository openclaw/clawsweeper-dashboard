## Summary

Keep open: current main still lacks a configurable Bedrock `anthropic_beta` or `additionalModelRequestFields` passthrough. The original 1M-context example appears likely obsolete for current Claude 4.6 Bedrock models, but the broader provider-beta config request remains unresolved and needs maintainer API/product judgment.

## What This Changes

Request to let Bedrock model configuration provide custom Anthropic beta flags that OpenClaw merges into Converse `additionalModelRequestFields`.

## Best Possible Solution

Define a Bedrock-owned, validated model params contract that merges and dedupes configured betas into `additionalModelRequestFields.anthropic_beta`, composes with guardrails/default betas/app-profile traits, and documents the supported config shape.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a broken documented behavior. Source inspection gives high confidence that current main does not read or forward a custom Bedrock `anthropic_beta` config field.

## Solution Assessment

Unclear until maintainers choose the public config shape. A narrow Bedrock-owned params field or structured `additionalModelRequestFields` contract looks safer than the proposed broad `providerFields` passthrough.
