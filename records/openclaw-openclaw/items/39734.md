## Summary

Keep open: current main and the latest v2026.5.22 release still do not expose a supported Amazon Bedrock config path for custom `anthropic_beta` values. The original 1M-context example is partly obsolete now that GA Claude 4.x models are documented as 1M-capable without the retired beta, but the broader Bedrock beta passthrough remains an unresolved provider config/API decision and overlaps an open default-beta PR.

## What This Changes

Request to let Bedrock model config specify custom `anthropic_beta` values that OpenClaw forwards into Converse `additionalModelRequestFields`.

## Best Possible Solution

Define a Bedrock-owned, validated `params` contract for Anthropic beta values, merge and dedupe those values into `additionalModelRequestFields.anthropic_beta`, compose with existing Bedrock payload wrappers, and document the supported config shape.

## Reproduction Assessment

Not applicable: this is a feature/config request rather than broken documented behavior. Source and release inspection give high confidence that current main and v2026.5.22 do not read or forward custom Bedrock beta flags from config.

## Solution Assessment

Unclear until maintainers choose the public API shape. A narrow Bedrock-owned `params.anthropicBeta`-style contract that mirrors direct Anthropic beta merging is safer than a broad unvalidated `providerFields` passthrough.
