## Summary

Keep open: current main and the latest release still do not expose a Bedrock config path for custom `anthropic_beta` values. The original 1M-context example may be partly obsolete for current Claude models, but the broader provider-owned beta passthrough request remains unresolved and needs maintainer API judgment.

## What This Changes

Request configurable Bedrock Anthropic beta flags that are merged into Converse `additionalModelRequestFields.anthropic_beta` from model or provider config.

## Best Possible Solution

Define a Bedrock-owned, validated `params` contract for Anthropic beta values, merge and dedupe them into `additionalModelRequestFields.anthropic_beta`, compose with current guardrail/service-tier/profile/thinking payload wrappers, and document the supported config shape.

## Reproduction Assessment

Not applicable: this is a feature/config request rather than a failing documented behavior. Source inspection gives high confidence that current main does not read or forward custom Bedrock beta flags from config.

## Solution Assessment

Unclear until maintainers choose the public API shape. A narrow Bedrock-owned `params` contract that mirrors direct Anthropic beta merging looks safer than a broad unvalidated `providerFields` passthrough.
