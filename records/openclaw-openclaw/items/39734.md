## Summary

Keep open: current main still lacks any user-configurable Amazon Bedrock `anthropic_beta` passthrough. The original 1M-context motivation is partly obsolete for known Claude 4.6 Bedrock models, but the broader custom-beta request remains unimplemented and needs a maintainer API decision.

## What This Changes

Request to let Amazon Bedrock Claude model configuration pass custom `anthropic_beta` flags into Converse `additionalModelRequestFields`.

## Best Possible Solution

Add a provider-local Bedrock config path, likely mirroring `params.anthropicBeta`, that validates string or string-array beta values, merges and dedupes them into `additionalModelRequestFields.anthropic_beta`, composes with guardrails/app-profile traits, and documents the behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main has no Bedrock custom-beta passthrough even though the AWS SDK supports `additionalModelRequestFields`.

## Solution Assessment

Unclear until maintainers choose the public config shape. A narrow Bedrock-owned `params.anthropicBeta` implementation looks more maintainable than the proposed broad `providerFields` passthrough.
