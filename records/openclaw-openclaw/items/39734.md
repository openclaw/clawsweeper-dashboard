## Summary

Keep open: current main still lacks a user-configurable Bedrock `anthropic_beta` passthrough. The discussion usefully narrows the original 1M-context example, but the broader custom-beta config request remains unresolved.

## What This Changes

Request config-driven `anthropic_beta` flags for Amazon Bedrock Claude models, merged into Bedrock Converse `additionalModelRequestFields`.

## Best Possible Solution

Add a Bedrock-owned, validated params path that merges and dedupes configured betas into `additionalModelRequestFields.anthropic_beta`, composes with guardrails/default betas/app-profile traits, and documents the contract.
