## Summary

Keep open. Current main still lacks configurable Bedrock Converse `anthropic_beta` or general `additionalModelRequestFields` passthrough; the 1M preview use case may be less urgent per the issue discussion, but the broader config-driven request remains concrete and unimplemented.

## What This Changes

The issue requests a Bedrock model/provider configuration surface for custom Anthropic beta flags that are merged into Bedrock Converse `additionalModelRequestFields`.

## Best Possible Solution

Add a Bedrock-plugin-owned, validated passthrough using the existing per-model `params` surface where possible, merge it into `payload.additionalModelRequestFields` without breaking existing guardrail/cache/region/Opus-thinking composition, and document the supported Bedrock config in the provider docs.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: `providerFields` is rejected by strict model schema, while `params.anthropicBeta` or `params.additionalModelRequestFields` is not read by the Bedrock wrapper and therefore cannot reach Bedrock Converse today.

## Solution Assessment

No for the exact proposed `providerFields` shape. The narrower maintainable path is to use the existing provider-specific `params` contract, unless maintainers explicitly want a new core model-definition field.
