## Summary

Keep open: current main and v2026.5.19 still do not expose a Bedrock config path for custom Anthropic beta values. The 1M-context example may be partly obsolete for current Claude models, but the broader provider-owned passthrough request remains unresolved and needs maintainer API judgment.

## What This Changes

The issue requests a Bedrock model configuration path for custom Anthropic beta flags that can be forwarded into Bedrock Converse request fields.

## Best Possible Solution

Define a Bedrock-owned, validated `params` contract for Anthropic beta values, merge and dedupe them into `additionalModelRequestFields.anthropic_beta`, compose with the existing guardrail/service-tier/profile/thinking payload wrappers, and document the supported config shape.

## Reproduction Assessment

Not applicable: this is a feature/config request rather than a failing documented behavior. Source and release inspection give high confidence that current main and v2026.5.19 do not read or forward custom Bedrock beta flags from config.

## Solution Assessment

Unclear until maintainers choose the public API shape. A narrow Bedrock-owned `params` contract that mirrors direct Anthropic beta merging looks safer than a broad unvalidated `providerFields` passthrough.
