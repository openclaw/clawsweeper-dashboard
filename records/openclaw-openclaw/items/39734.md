## Summary

Keep open: current main and v2026.6.10 still do not expose a Bedrock config path for custom `anthropic_beta` or generic `additionalModelRequestFields`, while AWS documents `additionalModelRequestFields` as the ConverseStream mechanism for model-specific parameters and lists `context-1m-2025-08-07` as a valid Anthropic beta header. ([docs.aws.amazon.com](https://docs.aws.amazon.com/bedrock/latest/APIReference/API_runtime_ConverseStream.html))

## What This Changes

The issue asks for a Bedrock model/config path to pass custom `anthropic_beta` flags into Converse `additionalModelRequestFields`.

## Best Possible Solution

Define a Bedrock-owned request-field merge contract, add runtime/docs/tests for `anthropic_beta` merging and de-duplication, and require redacted live Bedrock proof before landing.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current config, extra-param, and Bedrock option paths do not carry custom `anthropic_beta` into `additionalModelRequestFields`. I did not run a live AWS Bedrock request in this read-only review.

## Solution Assessment

Unclear. A configurable Bedrock request-field path is likely the right direction, but maintainers should choose the narrow public contract before implementation because generic passthrough expands user-facing config.
