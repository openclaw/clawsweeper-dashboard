## Summary

Keep open: current main still lacks a configurable Amazon Bedrock `anthropic_beta` passthrough. The original 1M-context example may now be obsolete for current Claude GA models, but the broader config request remains unresolved and needs maintainer API/product judgment.

## What This Changes

The issue requests a configurable Amazon Bedrock path for custom Anthropic beta flags in model or provider configuration.

## Best Possible Solution

Add a Bedrock-owned, validated params path that merges and dedupes configured betas into `additionalModelRequestFields.anthropic_beta`, composes with guardrails/default betas/app-profile traits, and documents the contract.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a broken documented behavior. Source inspection gives high confidence that current main does not read or forward a custom Bedrock `anthropic_beta` config field.

## Solution Assessment

Unclear until maintainers choose the public config shape. A narrow Bedrock-owned params field or structured `additionalModelRequestFields` subshape looks safer than the proposed broad `providerFields` passthrough, especially with default-beta work touching the same payload path.
