## Summary

Keep this open: current main and the latest release still lack a Bedrock config path for custom `anthropic_beta` values, while AWS documents `additionalModelRequestFields` as the Converse/ConverseStream mechanism for model-specific parameters and lists `context-1m-2025-08-07` as a Claude Bedrock beta flag.

## What This Changes

The issue requests a configurable way to pass Anthropic Bedrock beta flags such as `context-1m-2025-08-07` through the Bedrock Converse request body.

## Best Possible Solution

Define a provider-owned, schema-backed Bedrock request-field mechanism that merges user beta flags without clobbering generated thinking fields, then document and test the chosen per-model/config behavior.

## Reproduction Assessment

Yes for the source-level gap: current Bedrock code has no configurable merge path into `additionalModelRequestFields.anthropic_beta`. I did not establish a live failing Bedrock call from this checkout.

## Solution Assessment

Unclear. A Bedrock-owned passthrough is plausible, but maintainers need to choose whether the permanent contract is a narrow `anthropicBeta`/`context1m` flag, a more general `additionalModelRequestFields` map, or no custom passthrough.
