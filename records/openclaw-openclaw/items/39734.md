## Summary

Keep open: current main still lacks a user-configurable Bedrock `anthropic_beta` passthrough. The 1M-context example may be obsolete for current Claude GA models, and https://github.com/openclaw/openclaw/pull/69491 only covers hard-coded default betas, so the broader custom Bedrock beta config request remains unresolved.

## What This Changes

Request to let Bedrock model configuration supply custom Anthropic beta flags that OpenClaw merges into Converse `additionalModelRequestFields.anthropic_beta`.

## Best Possible Solution

Add a Bedrock-owned, validated params path that merges and dedupes configured betas into `additionalModelRequestFields.anthropic_beta`, composes with guardrails/default betas/app-profile traits, and documents the contract.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a broken documented behavior. Source inspection of current main gives high confidence that Bedrock does not read or forward a custom `anthropic_beta` config field today.

## Solution Assessment

Unclear until maintainers choose the public config shape. A narrow Bedrock-owned params field or structured `additionalModelRequestFields` subshape looks safer than the proposed broad `providerFields` passthrough, especially with default-beta work touching the same payload path.
