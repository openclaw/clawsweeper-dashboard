## Summary

Keep open: current main and the latest release still lack a Bedrock config path for custom `anthropic_beta` or generic `additionalModelRequestFields`, and the newer discussion plus the closed hardcoded PR confirm this remains the canonical product-contract decision rather than a cleanup close.

## What This Changes

Allow Amazon Bedrock model configuration to pass custom `anthropic_beta` flags into Converse `additionalModelRequestFields`.

## Best Possible Solution

Define a Bedrock-owned request-field merge contract, document it, test `anthropic_beta` merging and de-duplication, and require redacted live Bedrock proof before landing.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current main only builds Bedrock `additionalModelRequestFields` from internal reasoning options, and neither `BedrockOptions` nor the plugin config schema exposes the requested custom beta/request-field path.

## Solution Assessment

Unclear. A configurable Bedrock request-field path is likely the right direction, but maintainers should choose the narrow public contract before implementation because this adds user-facing provider config surface.
