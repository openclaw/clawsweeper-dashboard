## Summary

Keep open: current main and the latest release still do not provide a Bedrock config path for custom `anthropic_beta` or `additionalModelRequestFields`, while current Bedrock discovery already treats the relevant Claude refs as 1M-capable. The issue needs a maintainer decision on the permanent Bedrock request-field contract rather than cleanup closure.

## What This Changes

Allow Bedrock Claude model configuration to pass custom `anthropic_beta` or equivalent additional request fields so users do not patch provider code for Bedrock-only beta features.

## Best Possible Solution

Define a Bedrock-owned request-field merge contract, add focused runtime/docs/tests for `anthropic_beta` de-duplication and 1M Bedrock behavior, and require redacted live Bedrock proof before landing.

## Reproduction Assessment

Yes, at source level: current main has no Bedrock config or extra-param path that can merge custom `anthropic_beta` into `additionalModelRequestFields`, while Bedrock discovery marks the reported Claude refs as 1M-capable. I did not independently rerun a live AWS Bedrock request in this read-only review.

## Solution Assessment

Unclear. A configurable Bedrock request-field path is likely the right product direction, but the exact surface should be maintainer-approved because generic `providerFields` would expand public config and a narrower `anthropicBeta` field may be safer.
