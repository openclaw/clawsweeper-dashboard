Explicit `--profile` gateway commands should resolve the profile-derived LaunchAgent plist instead of an inherited launchd label from another gateway process.

## Best Possible Solution

Land or replace the focused repair so explicit CLI profile selection clears stale OpenClaw-managed launchd labels while preserving intentional custom operator label overrides.
