## Summary

Keep open. Current main still leaves an inherited `OPENCLAW_LAUNCHD_LABEL` in place after explicit `--profile`, and https://github.com/openclaw/openclaw/pull/66000 is an open closing implementation candidate, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Explicit `--profile` gateway status should ignore a stale inherited `OPENCLAW_LAUNCHD_LABEL` so it resolves the target profile's LaunchAgent plist.

## Best Possible Solution

Land or replace the focused PR so explicit CLI profile selection wins over stale OpenClaw-managed service labels while preserving intentional custom launchd overrides.

## Reproduction Assessment

Yes, at source level. With `OPENCLAW_LAUNCHD_LABEL=ai.openclaw.batch` and an explicit root `--profile interactive gateway status` path, current main leaves the stale label intact before launchd resolution chooses it over `OPENCLAW_PROFILE`.

## Solution Assessment

Yes. Fixing the CLI profile-env boundary is the narrowest maintainable solution because `--profile` is parsed before gateway status resolves the LaunchAgent plist, while global launchd precedence can still preserve intentional custom overrides.
