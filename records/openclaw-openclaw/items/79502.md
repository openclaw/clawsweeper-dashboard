## Summary

Keep open: the fix direction matches a real Telegram transcript gap on current main, but this PR still has correctness blockers in the mirror placement and SDK baseline handling, and it lacks required real behavior proof.

## What This Changes

The PR adds a Telegram `transcriptMirror` callback, wires it from Telegram message dispatch, re-exports the transcript helper through the Plugin SDK, changes empty `routeReply` delivery handling, and adds changelog entries.

## Best Possible Solution

Mirror only actually delivered post-hook Telegram replies into the session transcript, update the Plugin SDK API baseline and focused tests, then require redacted real Telegram/CLI-backend proof before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main’s Telegram `deliverReplies` direct-send path bypasses the transcript append helper while carrying only message hook metadata. I did not run a live Telegram bot in this read-only review.

## Solution Assessment

No. The direction is right, but the proposed post-loop mirror records normalized inputs instead of delivered post-hook payloads, and the public SDK export is missing its baseline update.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Mirror only delivered post-hook replies:** `extensions/telegram/src/bot/delivery.replies.ts:907-918`
  - body: This post-loop mirror is built from `normalizedReplies`, but the loop above can cancel a reply or rewrite `reply.text` through `message_sending` hooks before delivery. With hooks enabled, the transcript can record content that was not actually delivered, so collect or invoke `transcriptMirror` only after a reply’s delivery count increases and after hook mutations have been applied.
  - confidence: 0.88
- **[P2] Commit the Plugin SDK baseline hash:** `src/plugin-sdk/agent-harness-runtime.ts:121`
  - body: Adding `appendAssistantMessageToSessionTranscript` to `agent-harness-runtime` changes a public Plugin SDK export, but this PR does not update `docs/.generated/plugin-sdk-api-baseline.sha256`. The repo’s API baseline check is designed to fail on that drift, so run the SDK API generator and commit the updated tracked hash for this intentional export change.
  - confidence: 0.9
