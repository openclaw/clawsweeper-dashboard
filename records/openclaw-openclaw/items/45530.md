## Summary

Keep open for maintainer review: the feature is not implemented on current main, but this PR is stale/conflicting against the current speech-core plugin runtime and still needs real behavior proof before it can merge.

## What This Changes

The PR adds a `persist=true` TTS directive that stores the selected voice in user prefs by provider and reuses it on later TTS calls.

## Best Possible Solution

Port the feature to the current speech-core runtime with provider-generic voice override persistence across synthesize, stream, telephony, and Discord voice paths, then land it with focused regression tests and real runtime proof.

## Reproduction Assessment

Yes for the review blockers: source inspection shows current main has no persisted voice prefs or `persist` parse result, and the live PR is conflicting against the active speech-core/Discord voice paths. Live TTS behavior was not reproduced because the PR lacks after-fix runtime proof and does not merge cleanly.

## Solution Assessment

No. The feature direction is reasonable, but the current patch is not the best landing shape because it targets stale files and hard-codes providers instead of using the current provider override model.

  - body: This adds prefs helpers to the old in-core TTS implementation, but current main routes `src/tts/tts.ts` through the speech-core plugin runtime and the real prefs/synthesis code is in `extensions/speech-core/src/tts.ts`; the PR is also merge-conflicting and still touches the removed Discord voice manager path.   - confidence: 0.9
- **[P2] Keep persisted voices provider-generic:** `src/tts/tts.ts:586-596`
  - body: `resolveVoiceOverrideForPersist` only returns values for `elevenlabs` and `openai`, so providers such as MiniMax, Google, Azure, or any plugin that parses a `voice` override will never persist even though the prefs shape is keyed by provider id. Persist should derive the selected provider's generic voice override from `voice`, `voiceId`, or `voiceName` instead of hard-coding two providers.
  - confidence: 0.86
