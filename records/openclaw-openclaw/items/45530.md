## Summary

Keep open. The feature is not implemented on current main, and the branch is not mergeable as-is because it targets pre-speech-core TTS and Discord surfaces that current main no longer uses.

## What This Changes

The branch adds a `persist=true` TTS directive, provider-keyed persisted voice preferences, persisted voice reads and writes in TTS flows, Discord voice handling, and tests.

## Best Possible Solution

If maintainers want durable model-driven voice switching, rebuild it on the active speech-core runtime with a typed persist signal, provider-keyed prefs/facade API, normal/telephony/Discord wiring, docs, changelog, and tests.

## Reproduction Assessment

Not applicable as a user bug report. The PR defects are source-reproducible: current main routes directive parsing through `src/tts/directives.ts` and Discord voice TTS through `extensions/discord/src/voice/tts.ts`, while the branch changes older paths.

## Solution Assessment

No. The proposed patch is not the best way to solve the feature on current main because it targets obsolete TTS and Discord surfaces instead of extending the active speech-core/plugin-SDK contract.
