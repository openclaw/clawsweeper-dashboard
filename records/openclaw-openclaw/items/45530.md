## Summary

Keep open. The feature is useful, but the branch is stale against current `main`, lacks contributor real-behavior proof, and the proposed provider-agnostic persistence still hard-codes voice extraction to OpenAI and ElevenLabs while current TTS voice directives are plugin-owned.

## What This Changes

The PR adds a `persist=true` TTS directive that stores a successful voice override in user prefs and reuses it on later TTS calls.

## Best Possible Solution

Port the feature to the current `speech-core` provider contract, make provider voice persistence generic through provider-owned override metadata, and require maintainer approval plus real TTS behavior proof before merge.

## Reproduction Assessment

Yes for the review finding by source inspection: current `main` routes provider voice directives through `providerOverrides`, while the PR persists only OpenAI and ElevenLabs override shapes. I did not run live TTS because this is a read-only review and the branch is conflicting.

## Solution Assessment

No. The direction may be valid, but the implementation should be rebuilt around current `speech-core` and provider-owned directive contracts instead of reviving the old hard-coded core TTS implementation.
