## Summary

Keep open. The branch is stale against current main's speech-core/plugin-SDK TTS architecture, so the proposed changes target old core and Discord paths and would not make `persist=true` observable after rebase.

## What This Changes

The PR adds `persist=true` parsing for TTS directives, provider-keyed voice preference helpers, persisted voice reads and writes, Discord voice-session persistence, and focused tests.

## Best Possible Solution

If maintainers want this capability, rebuild it on the active speech-core/plugin-SDK design with a typed persist signal, provider-keyed speech-core prefs, facade helpers as needed, normal and telephony voice resolution, best-effort writes after successful synthesis, Discord extension wiring, docs, and tests.

## Reproduction Assessment

Not applicable as a user bug report. The PR defects are source-reproducible: current main routes directive parsing through `src/tts/directives.ts` and Discord voice TTS through `extensions/discord/src/voice/tts.ts`, while the branch changes older paths.

## Solution Assessment

No. The proposed patch is not the best way to solve the feature on current main because it targets obsolete surfaces instead of extending the active speech-core runtime contract.
