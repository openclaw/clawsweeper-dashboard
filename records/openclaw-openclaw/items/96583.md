## What This Changes

The PR adds validation for DuckDuckGo numeric HTML entity code points and regression coverage for invalid numeric and surrogate-range entities.

## Best Possible Solution

Land the plugin-local decoder guard after maintainer review, keeping the fix limited to DuckDuckGo parsing without new core, SDK, config, or dependency surface.

## Reproduction Assessment

Yes. Current main routes DuckDuckGo title, URL, and snippet fields through numeric entity decoding that calls `String.fromCodePoint` on unchecked parsed values, and a focused Node probe shows out-of-range values throw while surrogate values produce a lone surrogate.

## Solution Assessment

Yes. A DuckDuckGo-local guard is the narrowest maintainable fix because the failure is in a private bundled-plugin parser, while adjacent Telegram work is a separate plugin-local decoder hardening path.
