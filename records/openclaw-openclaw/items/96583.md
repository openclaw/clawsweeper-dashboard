## What This Changes

The PR updates DuckDuckGo HTML entity decoding to preserve invalid numeric entities instead of throwing, with focused regression coverage.

## Best Possible Solution

## Reproduction Assessment

Yes from source inspection: DuckDuckGo result parsing sends title, URL, and snippet through decodeHtmlEntities, which currently calls String.fromCodePoint on unchecked numeric entity values; a Node primitive check confirms out-of-range code points throw RangeError.

## Solution Assessment

Yes. The helper-level guard is the narrow maintainable fix because all affected DuckDuckGo fields share this decoder, and moving to a core helper would cross the plugin boundary without evidence of a missing SDK seam.
