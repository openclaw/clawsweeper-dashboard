Full review comments:

- **[P1] Preserve memory adapters with baseUrl:** `extensions/memory-core/src/memory/embeddings.ts:151`
  - body: This guard is not limited to `openai`. Any memory-specific provider with a configured `baseUrl`, such as `memorySearch.provider: "ollama"`, now skips its memory adapter and falls through to `getEmbeddingProvider`, but Ollama registers only a memory embedding adapter. That makes existing configured local memory providers fail instead of using their configured endpoint.
  - confidence: 0.92
- **[P1] Resolve provider owners before returning undefined:** `src/agents/memory-search.ts:200`
  - body: When the direct adapter has a `baseUrl`, this returns `undefined` before checking `providerConfig.api`. A configured provider like `openai` with `api: "ollama"` and `baseUrl` would lose the Ollama memory adapter/default model even though the documented custom-provider contract allows provider entries to point at memory-specific owners.
  - confidence: 0.87
- **[P1] Keep gateway memory-specific adapters:** `src/gateway/embeddings-http.ts:164`
  - body: The gateway repeats the broad `baseUrl` skip, so `/v1/embeddings` also stops using memory-specific adapters for configured providers such as Ollama and then throws when no generic adapter exists. The gateway path should resolve the intended owner and still prefer memory adapters before generic fallback.
  - confidence: 0.9
- **[P2] Remove the unused crypto import:** `scripts/verify-baseurl-routing.mjs:11`
  - body: The added verification script imports `randomUUID` but never uses it, and the latest PR checks show `check-lint` failing. Dropping the unused import should unblock that concrete lint failure.
