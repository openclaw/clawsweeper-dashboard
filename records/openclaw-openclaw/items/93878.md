- **[P1] Normalize `baseURL` before runtime routing:** `src/plugins/embedding-provider-config.ts:58-60`
  - body: The patch reads `baseURL` in runtime resolvers, but current `ModelProviderSchema` is strict and only accepts canonical `baseUrl`. A reported config using `baseURL` can fail or be rejected before these branches run, and the new test bypasses parsing with a cast; normalize this alias in config/doctor/docs or keep runtime routing canonical-only.
  - confidence: 0.88
- **[P1] Preserve configured memory-provider adapters:** `extensions/memory-core/src/memory/embeddings.ts:156-161`
  - body: When `models.providers.openai` has a custom endpoint and `api: "ollama"`, this skip sends memory-core past the direct OpenAI adapter and then only tries generic embedding providers. Ollama is exposed as a memory embedding provider, so this can still fail instead of using the configured owner adapter.
  - confidence: 0.86
- **[P1] Route the gateway through the resolved memory adapter:** `src/gateway/embeddings-http.ts:168`
  - body: The gateway patch has the same one-sided fallback: after detecting a custom base URL it skips the direct memory adapter, but it does not try the configured memory-specific owner before `getGenericEmbeddingProvider`. `/v1/embeddings` can therefore miss providers such as Ollama that expose memory adapters but no generic embedding adapter.
  - confidence: 0.84
