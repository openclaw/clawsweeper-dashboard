## Summary

Close as a support/config item: the reported `queuedFinal=false, replies=0` group behavior matches OpenClaw's documented group visible-reply policy, not a Feishu send regression.

## What This Changes

The issue asks for Feishu group chats to post the agent's final assistant reply to the group when direct-message replies already work.

## Best Possible Solution

Keep the existing group-room safety default and use `messages.groupChat.visibleReplies: "automatic"` when an operator wants normal assistant finals posted back to Feishu groups.

## Reproduction Assessment

Yes, from source inspection: a group turn with unset `messages.groupChat.visibleReplies` resolves to `message_tool_only`, which suppresses automatic final delivery and matches the reported `queuedFinal=false, replies=0` log.

## Solution Assessment

Yes, the existing config knob is the narrowest solution; changing Feishu code would bypass a documented cross-channel group delivery policy.
