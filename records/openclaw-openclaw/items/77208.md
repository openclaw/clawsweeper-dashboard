Keep open: current main still has the Slack DM U-ID direct-post path while default message-tool routing and outbound session metadata can still use the D-prefixed DM channel as a channel-style target. That matches the source-backed attribution split described in the issue comments, even though the full live non-delivery symptom still needs Slack proof.

## What This Changes

Fix Slack DM reply delivery and Control attribution so live replies and message-tool sends converge on the same direct Slack session after the v2026.5.2 proactive-DM change.

## Best Possible Solution

## Solution Assessment

No, current main is not the final solution because it still preserves the split between direct user posts and D-channel session routing. The narrow maintainable fix is Slack-owned target/session convergence rather than a broad core rewrite or blind revert of #62042.
