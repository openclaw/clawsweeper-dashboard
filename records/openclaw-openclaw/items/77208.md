## Summary

Keep open: current main still contains the prior #59030 live-reply split, but it also still has the 2026.5.2 text-only Slack DM direct-post path and a Slack outbound route that does not classify `D...` IM channel targets as direct sessions, matching the reported Control attribution edge. The normal-reply delivery symptom still needs live Slack confirmation before closing or claiming a complete fix.

## What This Changes

Slack DMs should deliver normal assistant replies and attribute proactive/message-tool sends to the same direct Slack session after the 2026.5.2 direct-post change.

## Best Possible Solution

Fix Slack plugin outbound routing so `user:U...` and `channel:D...` DM targets converge to the same direct-session attribution, keep the documented direct-post fast path where safe, and add regression coverage before a live Slack verification pass for normal replies.

## Reproduction Assessment

No high-confidence live reproduction is available here. Source inspection does show a current-main attribution path: D-prefixed Slack IM targets are not classified as direct sessions while text-only user DMs can return U-prefixed channel IDs.

## Solution Assessment

Unclear for the full live delivery report, but the current implementation is not complete for attribution. The narrow maintainable fix is to align Slack DM target/session normalization without removing the documented direct text-post behavior.
