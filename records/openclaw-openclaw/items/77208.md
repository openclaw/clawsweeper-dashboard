Keep open: current main still contains the direct user-ID proactive DM send path, while the prior live-reply guard only proves the earlier #59030 split. ## What This Changes

Slack DMs on v2026.5.2 reportedly receive inbound messages, but normal replies and message-tool attribution still diverge after the earlier #59030 DM reply fix.

## Best Possible Solution

Keep the issue open and make the Slack plugin converge DM delivery targets, cache behavior, and outbound session metadata so live replies, default message-tool sends, and explicit user sends all mirror to the same direct Slack session.

No high-confidence live reproduction of the full issue is established. Source inspection does reproduce the attribution edge: a Slack DM can expose `D...` as the current target while outbound routing and direct user-post delivery still split between `D...` and `user:U...`.

## Solution Assessment

Unclear for the full live delivery symptom. The narrow maintainable path is to fix the Slack-owned DM target/session convergence and add regression tests, not to blindly revert the #62042 direct-post behavior.
