Full review comments:

- **[P1] Send Inworld LLM requests with Basic auth:** `extensions/inworld/provider-catalog.ts:18-22`
  - body: `buildInworldProvider()` returns a normal OpenAI-compatible provider without any request auth/header override, and `index.ts` only adds `apiKey`. OpenClaw then passes that key to the shared OpenAI client, whose default auth is `Authorization: Bearer <key>`, while the docs and live test require `Authorization: Basic <key>`. The live test direct-fetches with Basic auth, so it does not prove real OpenClaw agent/model calls.
  - confidence: 0.92
- **[P2] Use a known endpoint class or remove it:** `extensions/inworld/openclaw.plugin.json:19`
  - body: `endpointClass: "inworld"` is not in core's allowed manifest endpoint classes, so `readManifestProviderEndpoints()` skips this entry and `api.inworld.ai` will not get the native endpoint classification the manifest declares. Add an approved core endpoint class with tests, or omit `providerEndpoints` if custom/proxy classification is intended.
  - confidence: 0.86
