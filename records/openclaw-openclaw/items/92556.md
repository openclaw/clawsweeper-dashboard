  - confidence: 0.91

  - body: The manifest declares `endpointClass: "inworld"`, but core only accepts endpoint classes in `MANIFEST_PROVIDER_ENDPOINT_CLASSES` and skips unknown rows. That leaves `api.inworld.ai` attributed as a custom/proxy-like endpoint instead of the intended known provider endpoint, which changes OpenAI-compatible request defaults.
  - confidence: 0.9
- **[P2] Bound the live catalog response before parsing JSON:** `extensions/inworld/models.ts:176`
  - body: `discoverInworldModels` buffers the authenticated provider catalog with `response.json()`. Live model catalogs are provider-controlled; the repository's shared catalog helper caps response bodies with `readResponseWithLimit`, so this path should use the bounded helper or an equivalent cap before parsing.
  - confidence: 0.84
