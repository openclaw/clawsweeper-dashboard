## What This Changes

The branch changes package-manager detection and dependency status so npm-installed package roots with `npm-shrinkwrap.json` are treated as npm while pnpm workspaces with `pnpm-lock.yaml` stay pnpm.

## Best Possible Solution

Land the narrow shrinkwrap-aware detector/status fix with proof from the patched OpenClaw path and focused coverage for the deps-status fallback.

## Reproduction Assessment

Yes, source-reproducible: current main and v2026.5.27 read the `packageManager` field before lockfile markers, while the linked report shows an npm global package root with `npm-shrinkwrap.json` and no `pnpm-lock.yaml` reporting pnpm.

## Solution Assessment

Yes on the code direction: preferring shrinkwrap only when `pnpm-lock.yaml` is absent matches OpenClaw's published-package contract while preserving source checkout detection. The merge path still needs proof that the patched OpenClaw module or CLI reports npm in the affected install shape.
