## What This Changes

The branch adds local-file MIME detection to `infer image describe` and passes the detected MIME through configured and explicit-model image description calls with CLI regression tests.

## Best Possible Solution

Keep local PNG MIME propagation, but resolve MIME in the media-understanding runtime/cache owner path after safe-file and size guards; leave the configured path on cache-derived MIME unless a focused test proves a remaining gap.

## Reproduction Assessment

Yes, at source level: current `main` and `v2026.6.9` send explicit-model local image describe requests without a MIME value, while downstream image request construction defaults missing image MIME to JPEG.

## Solution Assessment

No. MIME propagation is the right behavior, but the PR puts byte sniffing in the CLI before media-understanding guard ownership instead of fixing the runtime/cache path.
