## Summary

Keep open. The PR targets a real source-reproducible explicit-model MIME propagation bug and includes sufficient real behavior proof, but the proposed helper reads whole local files before the existing configured media-understanding size guard, so it needs a small fix before maintainer landing.

## What This Changes

The PR adds MIME detection for local `infer image describe` inputs, passes the detected MIME through configured and explicit media-understanding calls, and adds PNG propagation regression tests.

## Best Possible Solution

Keep MIME propagation, but detect it with a bounded sniff or runtime/cache-backed path so PNGs get the correct MIME without bypassing local media size limits.

## Reproduction Assessment

Yes. Current main's explicit `--model` image describe path calls `describeImageFileWithModel` without `mime`, while the runtime forwards `params.mime` to providers when present; a strict provider with a local PNG can reject the mismatched or defaulted MIME.

## Solution Assessment

No as written. Threading the existing MIME parameter is the right narrow fix, but the helper should not perform an unbounded full-file read before the media runtime's existing size checks.
