## Summary

## What This Changes

The PR adds MIME detection for local `infer image describe` inputs and regression tests for configured and explicit-model image description paths.

## Best Possible Solution

Keep MIME propagation in the CLI, but derive MIME with bounded or cache-backed sniffing so existing media size and safe-file guards stay first in the local path.

## Reproduction Assessment

Yes, as source-reproducible: current main's image describe path omits `mime`, while downstream image payload construction defaults missing MIME to `image/jpeg`. The PR body also supplies after-fix live provider output for a PNG.

## Solution Assessment

No. Passing detected MIME is the right fix direction, but this implementation should avoid reading the whole local file before media guards run.
