## What This Changes

## Best Possible Solution

Keep the MIME propagation, but implement it with bounded or cache-backed sniffing so the existing media guard path still owns local file safety and size rejection.

## Reproduction Assessment

Yes, at source level. Current main omits `mime` for local image describe calls, and the downstream image context defaults missing MIME to `image/jpeg`; the PR body also includes after-fix live provider output for a PNG.

## Solution Assessment

No. Passing detected MIME is the right fix direction, but the implementation should avoid reading the whole local file before media guards run.
