## Summary

Keep open: the reported MIME propagation bug is credible and the PR has real after-fix proof, but the current head still reads each local image fully before the existing media size and safe-file checks can run.

## What This Changes

The branch detects MIME for local `infer image describe` inputs, passes it to configured and explicit media-understanding calls, and adds CLI regression tests.

## Best Possible Solution

Keep MIME propagation, but derive the MIME through bounded header sniffing or the existing media attachment contract so size and safe-file checks still precede any full local read.

## Reproduction Assessment

Yes. Current main's source path omits `mime` for local image describe calls while the image runtime defaults absent MIME to `image/jpeg`; the PR body also includes after-fix live provider output for a PNG.

## Solution Assessment

No. Passing detected MIME into the runtime is the right fix direction, but the implementation should not read the whole local file before the media-understanding size and local-file guards run.
