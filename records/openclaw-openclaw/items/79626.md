## Summary

Keep open: the PR fixes a real CLI MIME propagation gap, but the current implementation is not merge-ready because it adds an unbounded CLI file read before the media-understanding guard path can enforce size and canonical local-file checks.

## What This Changes

The branch adds MIME sniffing for local `infer image describe` inputs in the CLI, passes the MIME to configured and explicit-model image description calls, and adds CLI regression tests for PNG propagation.

## Best Possible Solution

Keep the MIME propagation fix, but resolve local image MIME inside the media-understanding owner path after bounded safe local-file checks; leave the configured path cache-owned and repair the explicit-model local path there.

## Reproduction Assessment

Yes, at source level. Current main's explicit-model local image describe path can reach image request construction without MIME, and missing MIME defaults to `image/jpeg`; I did not rerun a live failing provider in this read-only review.

## Solution Assessment

No. Passing MIME is the right fix direction, but doing it in the CLI is the wrong layer because the configured path already has guarded MIME detection and the explicit path should be repaired inside media-understanding runtime.
