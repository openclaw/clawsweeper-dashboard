## Summary

Keep open: the PNG MIME bug is still present on current main and the latest release, but this PR should not merge as-is because it reads local image files in the CLI before media-understanding guards run.

## What This Changes

The PR adds local-file MIME detection to `infer image describe` and `describe-many`, passes the detected MIME to configured and explicit-model image-description calls, and adds CLI regression tests.

## Best Possible Solution

Keep local PNG MIME propagation, but resolve MIME in the media-understanding runtime/cache owner path after safe-file and maxBytes guards.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.9` call the explicit image-description path without MIME, and the downstream image context defaults missing MIME to `image/jpeg`.

## Solution Assessment

No. Passing MIME is the right direction, but this implementation puts sniffing in the CLI before media guards instead of resolving it in the media-understanding owner path.
