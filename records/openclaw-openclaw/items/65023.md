## Summary

Keep open for maintainer review: the branch still conflicts with the current provider architecture and has source-reproducible build, runtime, and security blockers, while the remaining Vertex ADC work is already being handled in the Google plugin path.

## What This Changes

The PR adds a core `google-genai` provider backed by `@google/genai`, a new `google-genai-sdk` auth mode, Vertex project/location config, embedded-runner routing, live tests, and an auth guide.

## Best Possible Solution

Consolidate on the Google plugin's guarded transport and review or land https://github.com/openclaw/openclaw/pull/60860 or a successor for remaining Vertex ADC gaps, rather than adding a parallel core `google-genai` provider.

## Reproduction Assessment

Yes. Static inspection of the PR diff against current main reproduces the blockers: stale imports, direct unguarded SDK calls, missing tool/run-option forwarding, incomplete keyless auth coverage, and unplumbed Vertex config.

## Solution Assessment

No. A new core direct-SDK provider is not the narrowest maintainable fix; the safer direction is plugin-owned Google transport work or a deliberate provider API change with security review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the current Pi package scope:** `src/agents/google-genai-stream.ts:2-3`
  - body: Current main depends on `@earendil-works/pi-agent-core` and `@earendil-works/pi-ai`, but this new file imports the old `@mariozechner` packages. A rebase onto current main will fail to resolve these modules until the imports are updated.
  - confidence: 0.93
- **[P2] Keep Google requests on the guarded transport:** `src/agents/google-genai-stream.ts:39-50`
  - body: Constructing `GoogleGenAI` here sends model traffic through the SDK's internal network path instead of OpenClaw's guarded provider fetch. That loses the proxy, capture, timeout, and network-policy controls used by the current Google plugin transport.
  - confidence: 0.87
- **[P2] Forward tools and run options to Google GenAI:** `src/agents/google-genai-stream.ts:96-100`
  - body: The `generateContentStream` call only sends model, contents, and the optional system instruction. Tool definitions, tool choice, max tokens, temperature, and cancellation are dropped, so tool-dependent and bounded runs will not match other transports.
  - confidence: 0.91
- **[P2] Allow keyless auth outside embedded runs:** `src/agents/model-auth.ts:419-421`
  - body: This new auth mode can return no API key, but the branch only relaxes embedded-runner checks. Other surfaces such as `/btw` still call `requireApiKey` for anything except `aws-sdk`, so `google-genai-sdk` models fail before streaming.
  - confidence: 0.9
- **[P2] Plumb Vertex config into runtime models:** `src/config/types.models.ts:79-82`
  - body: Adding `models.providers.*.vertexai` to the config schema is not enough because current runtime model construction does not copy that provider-level object into the model passed to the stream. Users who set project/location in config still rely on environment variables.
  - confidence: 0.88
