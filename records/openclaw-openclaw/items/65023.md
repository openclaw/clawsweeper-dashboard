## Summary

Keep open for maintainer review: the contribution targets a real Google auth/provider pain point and the author addressed early bot feedback, but the branch is not mergeable as-is because it still has correctness/security blockers and lacks inspectable real behavior proof.

## What This Changes

The PR adds a core `google-genai` stream provider/auth mode, Vertex config fields, embedded-runner routing, live tests, and a Google GenAI auth guide.

## Best Possible Solution

Consolidate Google GenAI/Vertex support in the Google plugin guarded transport or an explicit provider SDK seam, preserving auth, config, tool, run-option, and transport-policy parity before exposing new config.

## Reproduction Assessment

Yes. Static inspection of PR head against current main reproduces the merge/build and runtime blockers: stale Pi imports, direct SDK transport bypass, dropped tool/run options, and incomplete keyless-auth caller coverage.

## Solution Assessment

No. A parallel core direct-SDK provider is not the narrowest maintainable solution; the safer path is plugin-owned Google transport work or a deliberately reviewed provider SDK seam.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the current Pi package scope:** `src/agents/google-genai-stream.ts:2-3`
  - body: Current main depends on `@earendil-works/pi-agent-core` and `@earendil-works/pi-ai`, but this new file imports the old `@mariozechner` packages. A rebase onto current main will fail to resolve these modules until the imports are updated.
  - confidence: 0.93
- **[P2] Keep Google requests on the guarded transport:** `src/agents/google-genai-stream.ts:40-54`
  - body: Constructing `GoogleGenAI` here sends model traffic through the SDK's internal network path instead of OpenClaw's guarded provider fetch. That skips the proxy, capture, timeout, and network-policy controls used by the current Google plugin transport.
  - confidence: 0.88
- **[P2] Forward tools and generation controls:** `src/agents/google-genai-stream.ts:96-100`
  - body: The `generateContentStream` call sends only model, contents, and an optional system instruction. Tool definitions, tool choice, max tokens, temperature, and cancellation are dropped, so tool-dependent and bounded runs will not match other transports.
  - confidence: 0.91
- **[P2] Allow keyless auth mode in every caller:** `src/agents/model-auth.ts:422-423`
  - body: This branch can return `google-genai-sdk` with no API key, but other runtime surfaces such as `/btw` still only exempt `aws-sdk` before calling `requireApiKey`. Those runs will fail before reaching the new stream path.
  - confidence: 0.86
