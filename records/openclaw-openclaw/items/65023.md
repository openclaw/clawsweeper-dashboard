## Summary

Keep open. Current main does not implement the proposed `google-genai` provider/auth mode, and the PR still has merge-blocking functional and security issues around runtime auth, provider transport policy, dependency ownership, and request/stream parity.

## What This Changes

The PR adds a core `google-genai` provider backed by `@google/genai`, a `google-genai-sdk` auth mode, Vertex project/location config, embedded-runner routing, live tests, and a Google GenAI auth guide.

## Best Possible Solution

Keep unified Google/Vertex model behavior in the bundled Google provider plugin or add a deliberate plugin/SDK seam first, while preserving guarded transport semantics, auth contracts, docs, changelog, and focused regression coverage.

## Reproduction Assessment

Yes. The blocking issues are reproducible by static inspection of the PR diff against current main: runtime models do not forward `vertexai`, `/btw` still requires an API key for non-`aws-sdk` modes, and the SDK is imported from core without root dependency ownership.

## Solution Assessment

No. A new core direct-SDK provider is not the narrowest maintainable fix; the safer path is plugin-owned Google transport work or targeted `google-vertex` ADC/proxy repairs that preserve current request policy controls.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Pass Vertex config into runtime models:** `src/agents/google-genai-stream.ts:35-36`
  - body: `model.vertexai` is read here, but current runtime model construction only forwards known provider fields such as `api`, `baseUrl`, headers, request config, context limits, params, and timeouts. A user who configures `models.providers.google-genai.vertexai.project` in `openclaw.json` still reaches this stream without those values unless the same project/location are also in the environment.
  - confidence: 0.9
- **[P1] Allow keyless auth on every run surface:** `src/agents/model-auth.ts:419-421`
  - body: This new auth mode returns no API key, but only the embedded-runner checks are updated to allow it. `/btw` still exempts only `aws-sdk` before calling `requireApiKey`, so a `google-genai-sdk` model fails before this stream can run there.
  - confidence: 0.91
- **[P1] Declare the SDK where core imports it:** `src/agents/google-genai-stream.ts:1`
  - body: This core file imports `@google/genai`, but current main declares that package only as the bundled Google plugin's runtime dependency. Published core code should not rely on a plugin-local dependency path unless the runtime stays plugin-owned or the dependency is deliberately moved to root ownership.
  - confidence: 0.87
- **[P2] Forward tools and run options to the SDK:** `src/agents/google-genai-stream.ts:96-100`
  - body: The request sent to `generateContentStream` includes only the model, contents, and optional system instruction. It drops `context.tools`, tool choice, max-token and temperature controls, and the SDK-supported abort signal, so tool-dependent runs and caller limits will not match other transports.
  - confidence: 0.92
- **[P2] Preserve provider request policy for SDK calls:** `src/agents/google-genai-stream.ts:40-54`
  - body: Creating `new GoogleGenAI` and using the SDK's internal request path bypasses the existing Google transport's guarded fetch path. That can skip configured proxy/TLS policy, capture hooks, timeouts, and OpenClaw's network controls for this provider.
  - confidence: 0.84
- **[P2] Map streaming finish and usage metadata:** `src/agents/google-genai-stream.ts:107-108`
  - body: The streaming loop consumes only `chunk.text` and never maps usage or finish metadata from the SDK chunks. Max-token, safety, and tool-use finishes will be misclassified as normal stops, and cost/usage accounting will stay zero for this provider.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: Security review needs attention because the PR adds a core Google SDK network path and changes dependency ownership without preserving the existing provider request controls.

Concerns:

- **[medium] Direct SDK calls bypass provider network policy:** `src/agents/google-genai-stream.ts:40`
  - body: `new GoogleGenAI(...)` and `ai.models.generateContentStream(...)` use the SDK's internal network stack instead of OpenClaw's guarded provider fetch path, so configured proxy/TLS policy, capture hooks, timeout handling, and network controls may not apply to this provider.
  - confidence: 0.84
- **[medium] Core import relies on plugin-local dependency:** `src/agents/google-genai-stream.ts:1`
  - body: The new core stream imports `@google/genai`, but current main declares that package under the bundled Google plugin rather than root dependencies; this changes the supply-chain boundary and can break published package resolution unless dependency ownership is deliberate.
  - confidence: 0.87
