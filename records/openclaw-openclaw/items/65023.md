## Summary

Keep open: this branch is not mergeable as-is because it adds a parallel core Google GenAI provider with stale Pi imports, bypasses the guarded Google transport, drops tool/run-option parity, does not fully propagate the new config/auth mode, and still lacks inspectable real behavior proof.

## What This Changes

The PR adds a core `google-genai` provider backed by `@google/genai`, a new `google-genai-sdk` auth mode, Vertex project/location config, embedded-runner routing, live tests, and an auth guide.

## Best Possible Solution

Consolidate Google Vertex/GenAI support in the Google plugin's guarded transport or a deliberate provider SDK seam, preserving tool/run-option parity and full keyless auth coverage before exposing new config.

## Reproduction Assessment

Yes. Static inspection of PR head against current main reproduces the blockers: stale imports, unguarded SDK requests, dropped tool/run options, incomplete keyless auth coverage, and unplumbed Vertex config.

## Solution Assessment

No. Adding a parallel core direct-SDK provider is not the narrowest maintainable path; the safer direction is Google-plugin-owned transport work or an explicit provider SDK seam with security review.

  - body: Current main depends on `@earendil-works/pi-agent-core` and `@earendil-works/pi-ai`, but this new file imports the old `@mariozechner` packages. A rebase onto current main will fail to resolve these modules until the imports are updated.

  - body: Constructing `GoogleGenAI` here sends model traffic through the SDK's internal network path instead of OpenClaw's guarded provider fetch. That loses the proxy, capture, timeout, and network-policy controls used by the current Google plugin transport.
  - confidence: 0.88
- **[P2] Forward tools and run options to Google GenAI:** `src/agents/google-genai-stream.ts:96-100`
  - body: The `generateContentStream` call only sends model, contents, and optional system instruction. Tool definitions, tool choice, max tokens, temperature, and cancellation are dropped, so tool-dependent and bounded runs will not match other transports.
  - confidence: 0.91
- **[P2] Allow the new keyless auth mode everywhere:** `src/agents/model-auth.ts:422-423`
  - body: This branch can return `google-genai-sdk` with no API key, but only embedded-runner checks were relaxed. Other runtime surfaces such as `/btw` still require an API key for every mode except `aws-sdk`.
  - confidence: 0.9
- **[P2] Plumb Vertex config into runtime models:** `src/config/types.models.ts:79-82`
  - body: Adding `models.providers.*.vertexai` to the config type does not make it available to the stream because current runtime model construction does not copy that provider-level object into the model.
