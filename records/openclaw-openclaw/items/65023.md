## Summary

Keep open for maintainer review: the branch still has blocking correctness and security findings, lacks inspectable real-behavior proof, and overlaps active Google Vertex work that should stay plugin-owned unless maintainers choose a new core SDK seam.

## What This Changes

The PR adds a core `google-genai` provider/auth mode using `@google/genai`, with Vertex project/location config, embedded-runner routing, docs, and live tests.

## Best Possible Solution

Consolidate Google AI Studio and Vertex support through the Google plugin's guarded transport, or through an explicitly reviewed SDK seam that preserves auth, config, tools/options, cancellation, diagnostics, and redacted live proof.

## Reproduction Assessment

Yes for the review findings: source inspection of PR head and current main shows the package-scope mismatch, direct SDK transport path, omitted run controls/tools, keyless-auth caller gap, and missing runtime plumbing for Vertex config. No for the claimed live Google behavior, because no inspectable after-fix proof artifact is attached.

## Solution Assessment

No. A parallel core direct-SDK provider is not the narrowest maintainable solution while current main keeps Google behavior in the plugin-owned guarded transport; the safer path is plugin-owned Vertex/GenAI transport work or a deliberate reviewed SDK seam.

  - body: Current main depends on `@earendil-works/pi-agent-core` and `@earendil-works/pi-ai`, but this new file imports the old `@mariozechner` packages. A rebase or merge onto current main will fail to resolve these modules until the imports are updated.

  - body: Constructing `GoogleGenAI` here sends model traffic through the SDK's internal request path instead of OpenClaw's guarded model fetch, losing proxy/TLS policy, capture, timeout, cancellation, and network controls used by the current Google plugin transport.
  - confidence: 0.9
- **[P2] Forward tools and run options to Google GenAI:** `src/agents/google-genai-stream.ts:96-100`
  - body: The `generateContentStream` request only sends model, contents, and optional system instruction. Tool declarations, tool choice, max tokens, temperature, headers, cached content, and request cancellation are dropped, so tool-dependent and bounded runs will not match other provider transports.
  - confidence: 0.91
- **[P2] Allow the keyless auth mode in all runtime callers:** `src/agents/model-auth.ts:422-423`
  - body: This branch can return `google-genai-sdk` without an API key, but only embedded-runner checks were relaxed. Other runtime surfaces such as `/btw` still call `requireApiKey` for every no-key mode except `aws-sdk`, so this auth mode fails outside the patched runner paths.
  - confidence: 0.88
- **[P2] Carry Vertex config into runtime model resolution:** `src/config/types.models.ts:79-82`
  - body: Adding `models.providers.*.vertexai` to config does not make it available to the stream: runtime model construction copies known provider fields but does not carry this new object into the model passed to `createGoogleGenAiStreamFnForModel`, so config-only project/location users can still hit the missing-project path.
  - confidence: 0.87
