# Coding Plan for openclaw/openclaw#83507: agent.send auto-rotates sessionId without emitting session_start plugin hook

Source report: [records/openclaw-openclaw/items/83507.md](records/openclaw-openclaw/items/83507.md)

## Summary

This should stay open: current main and the latest release still have the reported gateway session-lifecycle asymmetry, and the closed related PR was not merged. Source inspection shows `agent.send` rotates/persists a new session ID and emits `sessions.changed`, but the lifecycle hook helper used by reset/create paths is not called there.

## Plan

Fix https://github.com/openclaw/openclaw/issues/83507. In `src/gateway/server-methods/agent.ts`, make `agent.send` session auto-rotation emit the existing gateway session lifecycle hook path when `isNewSession` is true, without removing or changing the existing `sessions.changed` create/send events. Reuse helpers from `src/gateway/session-reset-service.ts`; do not invent a parallel hook implementation. For rotations that replace an existing session ID, evaluate whether `session_end` must fire before `session_start` with `resumedFrom`, `nextSessionId`, and `nextSessionKey` metadata so the active-session shutdown tracker is not left with stale entries. Add focused regression coverage in `src/gateway/server-methods/agent.test.ts` for caller-supplied replacement session IDs and stale daily or idle rotation; keep existing reset/create behavior intact. Include a changelog entry if maintainers treat this as user-visible plugin behavior during landing. Do not change plugin hook names, gateway protocol versions, or broader session reset policy.
