# Coding Plan for openclaw/openclaw#66532: [Feature]: Expose abortEmbeddedPiRun family on api.runtime.agent

Source report: [records/openclaw-openclaw/items/66532.md](records/openclaw-openclaw/items/66532.md)

## Summary

Keep open: current main still lacks the requested injected `api.runtime.agent` helper family. The public `agent-harness` SDK path covers part of the steer/abort need, but the requested runtime object still omits abort, wait, active-check, and queue helpers for plugins that should use injected host-side runtime APIs.

## Plan

Implement a narrow additive `api.runtime.agent` embedded-run control surface for this issue. Add lazy runtime bindings in `src/plugins/runtime/runtime-agent.ts` backed by `src/plugins/runtime/runtime-embedded-pi.runtime.ts`, declare the contract in `src/plugins/runtime/types-core.ts`, update `src/plugin-sdk/test-helpers/plugin-runtime-mock.ts`, add shape coverage in `src/plugins/runtime/index.test.ts`, and document the surface in `docs/plugins/sdk-runtime.md` plus migration guidance if needed. Keep the helpers sessionId-scoped; do not expose internal process-wide abort overloads unless explicitly approved. Keep this separate from the runId-based `api.runtime.agent.abort` discussion at https://github.com/openclaw/openclaw/issues/52412. Decide and document the queue helper semantics carefully because `queueEmbeddedPiMessage` is a deprecated immediate-eligibility boolean helper; prefer a wrapper or explicit docs over implying it awaits runtime acceptance. Add a changelog entry only if the maintainer landing process requires one for this user-visible plugin API addition.
