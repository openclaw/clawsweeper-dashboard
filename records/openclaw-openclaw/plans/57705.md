## Summary

Keep open: current main still has a real global sub-agent lane budget and an internal queue snapshot, but no agent-facing `session_status`, `sessions_capacity`, `subagents list`, or `sessions_spawn` response exposes the sub-agent limit, active count, queued count, or remaining slots.

## Plan

Implement a narrow additive sub-agent capacity readback for this issue. Prefer adding structured data to `session_status.details`, backed by `getCommandLaneSnapshot(CommandLane.Subagent)`, with fields for the subagent lane limit, active count, queued count, remaining immediate slots (`max(0, maxConcurrent - activeCount)`), and draining state if useful. Do not change `sessions_spawn` queueing semantics, do not reject globally saturated lanes, do not treat `agents.defaults.subagents.maxConcurrent` as a per-parent child gate, and do not add multi-lane or per-lane config from the broader related feature. Add focused tests for the status-tool payload and the queue snapshot math, update user-facing docs for the snapshot/race semantics, and add a changelog entry because this is a new agent-facing feature.
