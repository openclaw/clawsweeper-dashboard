## Summary

Keep open: current main still defaults Telegram polling into isolated ingress while the configured stall threshold is only consumed by the embedded polling branch. The linked spool-drain fix addressed a different isolated-ingress hang mode and did not wire the parent-worker liveness watchdog to `pollingStallThresholdMs`.

## Plan

Fix https://github.com/openclaw/openclaw/issues/83950.
