## Plan

Fix https://github.com/openclaw/openclaw/issues/76831. The bug is that WhatsApp `messages.tts.auto: "inbound"` can produce text-only output when a text-originated active run receives a steered audio inbound: the audio transcript reaches the model, but final TTS still gates on the original dispatch context.
