## Summary

Keep open. Current main still has a source-reproducible connect-ordering path where the gateway can send the automatic remote bin probe before a Swift app-backed node has started normal event dispatch, and the related open PR only addresses probe churn/backoff rather than this readiness race.

## Plan

Fix https://github.com/openclaw/openclaw/issues/40527 as a narrow bug. Current main registers node sessions and starts refreshRemoteNodeBins before hello-ok is sent, while the Swift app starts normal listen/event dispatch only after sendConnect returns; automatic system.which probes can time out even when manual system.which works. Implement a readiness/order fix: defer the automatic remote bin refresh until after the node can receive normal events, or buffer and dispatch pre-hello node events in the Swift channel. Add focused regression coverage for the ordering/readiness behavior and preserve existing remote skill eligibility behavior.
