## Plan

Fix the Bedrock memory credential auto-selection regression reported in https://github.com/openclaw/openclaw/issues/64891. Current main still calls `hasAwsCredentials()` from `extensions/amazon-bedrock/memory-embedding-adapter.ts`, and that helper still invokes the AWS SDK default credential chain, which can probe IMDS at `169.254.169.254` during startup. Implement a passive/no-network eligibility path for Bedrock memory auto-selection and status checks so credential-less sandboxed environments do not contact IMDS, while preserving the full AWS default credential chain for explicit Bedrock runtime/provider use. Add focused regression coverage around the no-network auto-selection path and keep the fix within `extensions/amazon-bedrock` and `extensions/memory-core` unless the existing plugin SDK contract forces a small generic seam.
