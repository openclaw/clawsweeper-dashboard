Keep open. Current main still does not have an `openclaw doctor` diagnostic for persisted `skillsSnapshot.prompt` paths that point at inactive, missing, or temp-backed runtime roots; the adjacent refresh-state work does not cover this doctor/reporting request.

## Plan

Implement the doctor/session stale skill snapshot diagnostic for https://github.com/openclaw/openclaw/issues/47534. On current main, add a focused `openclaw doctor` state-integrity check that resolves the active bundled skills directory, scans persisted session `skillsSnapshot.prompt` values for bundled skill locations under inactive/missing/temp-backed runtime roots such as `/opt/homebrew`, `/tmp/openclaw`, and `/private/tmp`, and reports this as stale cached session metadata while making clear that the live runtime root can be healthy. In `doctor --fix`, back up the session store and delete only the affected `skillsSnapshot` fields so normal runtime rebuild logic refreshes them; do not wipe sessions, rewrite transcripts, add a config option, or change the broader runtime refresh semantics being tracked by https://github.com/openclaw/openclaw/pull/71497 and https://github.com/openclaw/openclaw/issues/73765. Account for current-main behavior where `resolvedSkills` is runtime-only and stripped from persistence; inspect raw legacy `resolvedSkills` only if needed for old store cleanup. Add focused tests and docs/changelog for the user-visible doctor output.
