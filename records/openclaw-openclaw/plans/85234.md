## Summary

Keep open: current main and the latest shipped release still route the npm staged package swap through the hardlink-rejecting copy path, so a hardlinked dependency binary in the staging tree can abort the update with the reported error.

## Plan

Fix https://github.com/openclaw/openclaw/issues/85234 by making the npm global staged install swap tolerate npm-created hardlinked files inside the trusted staging package tree only. Keep hardlink rejection intact for untrusted boundary reads/writes such as workspace files, credentials, hooks, backups, and generic replacement callers. Likely change `src/infra/package-update-steps.ts` and/or `src/infra/replace-file.ts` so `swapStagedNpmInstall` can copy or otherwise accept hardlinked files from `params.stage.packageRoot`, then add focused regression coverage in `src/infra/package-update-steps.test.ts` and/or `src/infra/replace-file.test.ts` that creates a hardlinked file in the staged tree and proves the swap succeeds without weakening the default reject behavior. Do not implement the broader external updater from https://github.com/openclaw/openclaw/issues/77511 in this PR. Changelog entry is expected because this fixes a user-facing updater regression.
