## Summary

Keep open: current main and the latest stable release still send the OpenClaw runtime-context wrapper through Codex `turn/start.input` as text user input, and the reporter's structured logs show that same marker accumulating as model-visible `role=user` history.

## Plan

Fix #84662 in the Codex app-server harness. Current main still builds `OpenClaw runtime context for this turn:` in `extensions/codex/src/app-server/attempt-context.ts`, decorates the prompt in `run-attempt.ts`, and sends it through `thread-lifecycle.ts` as `turn/start.input` text. Keep current-turn OpenClaw reference context available, but prevent historical decorated wrappers from accumulating in native Codex user history, replay, or compaction. Do not promote user-editable workspace/MEMORY context to developer/system authority. Prefer a supported non-persistent/current-turn Codex path if direct inspection of `../codex` proves its authority and persistence semantics; otherwise sanitize/reproject only historical decorated user items while preserving the text after `Current user request:`. Add multi-turn same-native-thread and compaction/replay regressions that fail when prior copies of the marker accumulate, and update prompt snapshots that currently assert the wrapper is ordinary user input.
