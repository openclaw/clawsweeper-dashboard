## Summary

Keep open. Current main still catches Feishu inbound media download/save failures as log-only errors, returns an empty attachment list, and can continue normal agent dispatch with a nonblank media placeholder and no Feishu-visible warning.

## Plan

Fix https://github.com/openclaw/openclaw/issues/69028 by making Feishu inbound media download/save failures user-visible without adding a new config option. In `extensions/feishu`, return structured media-resolution failures from `resolveFeishuMediaList` or an equivalent plugin-local helper. When a media-only message loses all attachments, send a warning reply to the same chat/thread with the configured size limit or download failure context and do not dispatch a normal agent turn. For text-plus-media and rich-post partial failures, preserve successfully saved attachments and make the dropped media visible without hiding partial loss. Do not change core media size limits or plugin SDK contracts. Add focused regression tests for `saveMediaBuffer` throwing on file/image media, embedded post media failure, and successful media continuing unchanged. Add a single-line `CHANGELOG.md` fix entry because this changes user-visible Feishu behavior.
