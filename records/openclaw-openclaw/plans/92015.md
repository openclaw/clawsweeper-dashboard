## Summary

Keep open: current main still resolves and scaffolds an OpenClaw workspace before ACP session dispatch, and `ensureAgentWorkspace` still runs `git init` for brand-new workspaces. The related open opt-out PR only addresses `agents.create`/manual git-init control and does not cover this ACP runtime-managed default-workspace path.

## Plan

Fix the ACP/runtime-managed workspace bootstrap hardening reported in https://github.com/openclaw/openclaw/issues/92015. On current main, `prepareAgentCommandExecution` resolves `workspaceDirRaw` with `resolveAgentWorkspaceDir` and calls `ensureAgentWorkspace` before ACP session resolution, while `ensureAgentWorkspace` creates a new workspace, writes bootstrap templates, and runs `git init` for brand-new workspaces. Implement a bounded fix so an `agents.list[]` entry with `runtime.type === "acp"` and no explicit `workspace` does not auto-scaffold the default OpenClaw workspace for ACP-bound/runtime-managed execution; preserve behavior for explicit workspaces and normal embedded agents. Also harden `ensureGitRepo` so it does not initialize a nested repository when the target directory is inside an existing parent git repository. Do not add a new config/env option unless existing config and runtime ownership cannot express the behavior. Add focused regression tests for the ACP no-explicit-workspace case and the parent-git nested-init case. Changelog is release-owned; do not edit `CHANGELOG.md`.
