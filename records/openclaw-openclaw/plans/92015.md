# Coding Plan for openclaw/openclaw#92015: Hardening: Skip default workspace auto-scaffold & git init for runtime-managed (ACP) agents

Source report: [records/openclaw-openclaw/items/92015.md](records/openclaw-openclaw/items/92015.md)

## Summary

Keep open. Current `main` still resolves and ensures an agent workspace before ACP dispatch, and the shared seeder can write bootstrap files plus run `git init` in a brand-new target without checking whether the target sits under an existing repository.

## Plan

Fix https://github.com/openclaw/openclaw/issues/92015 with a narrow core hardening change. Preserve normal embedded-agent workspace behavior and do not add a new config option by default. Add a central guard in `src/agents/workspace.ts` so automatic `git init` does not create a nested repo when the brand-new workspace path is already inside an existing parent git repository. Also update the agent-command/runtime path so an `agents.list[]` entry with `runtime.type: "acp"` and no explicit `workspace` does not implicitly create or bootstrap `${agents.defaults.workspace}/<agent.id>` just because the ACP session is dispatched; explicit per-agent `workspace` values should still be honored. Check open PR https://github.com/openclaw/openclaw/pull/72807 for reusable intent, but do not merge its generated Swift/API surface or introduce an `agents.create` flag unless it is still necessary after the central hardening fix. Add focused regression coverage for the parent-git guard and ACP runtime-managed no-workspace case. No `CHANGELOG.md` edit.
