# Coding Plan for openclaw/openclaw#92015: Hardening: Skip default workspace auto-scaffold & git init for runtime-managed (ACP) agents

Source report: [records/openclaw-openclaw/items/92015.md](records/openclaw-openclaw/items/92015.md)

## Summary

Keep open. Current main still routes non-default agents with `agents.defaults.workspace` into a derived workspace, then both agent-command and auto-reply startup call the bootstrap seeder; the seeder writes templates and runs `git init` without checking parent repositories, while ACP runtime bindings already have their own `runtime.acp.cwd` contract.

## Plan

Fix https://github.com/openclaw/openclaw/issues/92015. Current main derives a default workspace for non-default agents from `agents.defaults.workspace/<agentId>` and then bootstraps it even when `agents.list[].runtime.type` is `acp` and no explicit `workspace` is configured. Keep explicit per-agent workspaces honored. For ACP runtime-managed agents without an explicit workspace, avoid full default workspace scaffold/git initialization and let `runtime.acp.cwd`/binding cwd own the runtime working directory. Independently harden `ensureGitRepo` so it does not run `git init` inside a directory whose ancestors already contain a git repository. Add focused regression tests for the ACP no-explicit-workspace path and parent-repo git-init guard. Do not edit `CHANGELOG.md`; put any release-note context in the PR body or commit message. Do not resolve the broader non-default sibling-vs-nested workspace debate from https://github.com/openclaw/openclaw/issues/78093 unless the narrow fix requires a shared helper.
