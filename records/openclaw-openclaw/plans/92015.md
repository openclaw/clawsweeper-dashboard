## Summary

Keep open. Current main still resolves a fallback agent workspace for ACP-configured agents and runs the normal workspace seeder before ACP handling, so the reported bootstrap-file and nested-git side effects remain plausible and source-traceable.

## Plan

Fix https://github.com/openclaw/openclaw/issues/92015. Current main resolves a fallback workspace for configured ACP agents that omit `agents.list[].workspace`, and `src/agents/agent-command.ts` calls `ensureAgentWorkspace` before ACP handling; `src/agents/workspace.ts` can then seed `AGENTS.md`/`BOOTSTRAP.md` and run `git init`. Add a narrow guard so agents with `runtime.type: "acp"` and no explicit per-agent workspace do not auto-scaffold a default workspace or initialize an inner git repo. Preserve explicit per-agent workspace bootstrapping, normal OpenClaw agent bootstrapping, and existing `agents.defaults.workspace` behavior for non-ACP agents. Avoid adding a config option unless maintainers explicitly choose one, and do not edit `CHANGELOG.md`. Add focused regression coverage for ACP runtime-managed agents with `runtime.acp.cwd` set elsewhere and no `workspace`.
