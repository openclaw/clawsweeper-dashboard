## Summary

Keep this open: current main still has a source-reproducible path where an ACP/runtime-managed configured agent without an explicit workspace resolves a default workspace and the shared bootstrap helper can seed files plus run `git init` there.

## Plan

Fix https://github.com/openclaw/openclaw/issues/92015. Current main still resolves a default workspace for agents without an explicit workspace, and ensureAgentWorkspace can seed bootstrap files plus run git init there. Add a narrow runtime-managed-agent policy for agents.list entries with runtime.type="acp" and no explicit workspace so runtime-managed ACP agents do not auto-scaffold the default workspace or initialize a nested git repo. Preserve existing embedded-agent behavior, preserve explicit workspace behavior for ACP agents that intentionally set workspace, and do not add a new config option unless source review proves it is necessary. Also harden ensureGitRepo or its caller so brand-new OpenClaw workspaces are not git-initialized inside an existing parent git repository by default. Add focused regression tests for ACP runtime entries without workspace, ACP entries with explicit workspace, normal embedded agents, and nested parent git repository behavior. Do not edit CHANGELOG.md; put any release-note context in the PR body.
