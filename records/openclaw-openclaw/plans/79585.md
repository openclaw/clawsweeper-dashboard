## Summary

Keep open: current main still has a real mismatch between the documented canonical `google/*` plus `google-gemini-cli` runtime split and the auth/status paths users hit after Gemini CLI OAuth login.

## Plan

Keep the fix in the Google provider/status/runtime-contract surfaces; do not add a new config option or core product policy. Add focused regression coverage for login config patch/migration/status output with both a `google-gemini-cli` OAuth profile and `GEMINI_API_KEY` present.
