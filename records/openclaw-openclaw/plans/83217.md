# Coding Plan for openclaw/openclaw#83217: [Feature]: Support WhatsApp interactive list context and list replies

Source report: [records/openclaw-openclaw/items/83217.md](records/openclaw-openclaw/items/83217.md)

## Summary

Keep open. Current main does not expose inbound WhatsApp `listMessage` rows as body or structured context, and the WhatsApp message action surface only advertises reactions and polls; the related merged content-gate work at https://github.com/openclaw/openclaw/pull/73823 is useful precedent but only covers list selections, not list prompts or outbound list replies.

## Plan

Implement WhatsApp interactive list support for https://github.com/openclaw/openclaw/issues/83217. Keep the change inside the WhatsApp plugin: parse direct and wrapper-contained `listMessage` prompts into readable inbound body text plus an untrusted structured context entry labeled `WhatsApp list`; preserve existing text/media/contact/location behavior; make list prompts pass the inbound content gate; add a channel-owned `list-reply` message action that validates a non-empty row id and sends the Baileys `listResponseMessage`/`singleSelectReply.selectedRowId` payload with optional quote context for the original list message. Add the corresponding action discovery/gating/schema/docs updates, focused tests for direct/wrapped inbound lists, context propagation, send payload shape, action gating, row-id validation, and no regressions to existing contact/location/media handling. Add a changelog entry because this is user-facing. Do not add core WhatsApp-specific policy or raw Baileys payload exposure to agents.
