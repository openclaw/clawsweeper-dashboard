## Summary

Keep open. Current main still matches the reported gap: builtin search only waits for a synchronous sync when the index has no content; a populated dirty or reindex-needed index can still be queried while search sync runs in the background, and the related open PR at https://github.com/openclaw/openclaw/pull/59137 is adjacent rollback/cache durability work rather than a query-path fix.

## Plan

Fix https://github.com/openclaw/openclaw/issues/56862 in the builtin memory-core query path. Current main only forces synchronous sync when the builtin index has no content, then starts dirty search sync in the background before querying populated indexes. Add a narrow preflight/guard so a populated dirty/session-dirty or full-reindex-needed builtin index is not queried as if fresh. Prefer awaiting a bounded existing sync/reindex path or surfacing an explicit transient indexing/search failure over returning an empty result caused by pending sync. Do not change QMD behavior, provider defaults, memory backend selection, or add a new config option. Add focused regression coverage for populated builtin indexes with dirty/session-dirty state and metadata that requires full reindex; update existing async-search/preflight tests that currently lock in non-blocking search sync.
