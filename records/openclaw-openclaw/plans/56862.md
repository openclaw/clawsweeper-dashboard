Source report: [records/openclaw-openclaw/items/56862.md](records/openclaw-openclaw/items/56862.md)

## Summary

Keep open. Current main and the latest release still only synchronously bootstrap builtin memory search when the index is empty; a populated but dirty or reindex-needed index can query immediately while search-triggered sync runs in the background, matching the reported empty-result failure mode.

## Plan

Fix the builtin memory_search stale populated-index behavior described in https://github.com/openclaw/openclaw/issues/56862. Add a search preflight in memory-core so dirty, session-dirty, or full-reindex-needed builtin state cannot return an ordinary empty result before catch-up/reindex has a chance to complete; prefer bounded wait or explicit transient unavailable semantics over a new config knob. Keep QMD behavior unchanged, do not add a first-class reindex/status feature in this repair, and coordinate with the adjacent rollback-recovery work in https://github.com/openclaw/openclaw/pull/59137 rather than duplicating it. Add focused regression coverage around populated dirty indexes and search-triggered sync behavior; include a changelog entry if the user-visible memory_search behavior changes.
