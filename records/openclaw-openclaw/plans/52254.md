## Summary

Keep open: current main and the merged localization repair fixed a large part of the zh-CN lazy-tab coverage, but the requested complete module localization is not done. Current source still has hardcoded English display copy in Skills, Nodes, exec approvals, channel status callouts, and at least one Sessions accessibility label.

## Plan

Fix the remaining Control UI zh-CN localization gaps reported here. Scope the PR to display-only UI copy in the reported lazy-loaded tabs, especially `ui/src/ui/views/skills.ts`, `ui/src/ui/views/nodes.ts`, `ui/src/ui/views/nodes-exec-approvals.ts`, `ui/src/ui/views/channels.ts`, and the remaining Sessions accessibility/display labels. Use `ui/src/i18n/locales/en.ts` as the source of truth, run the Control UI i18n generator, and commit generated locale/metadata output. Do not translate raw gateway method names, JSON payload keys, config keys, command examples, server-provided IDs/names, or branded product names such as OpenClaw/ClawHub/Gateway unless the existing locale conventions already translate that exact display label. Coordinate with https://github.com/openclaw/openclaw/issues/44551 for audit/tool/event label scope and avoid changing unrelated protocol behavior. Add focused render/i18n tests for any newly localized view copy; include a changelog entry because this is user-visible UI localization.
