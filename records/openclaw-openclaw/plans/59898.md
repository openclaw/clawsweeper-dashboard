## Plan

Repair PR https://github.com/openclaw/openclaw/pull/59898 without broad prompt redesign. Preserve the intended explicit no-tools behavior: when the effective first-class and hosted tool sets are empty, the rendered prompt must not include the skills catalog, tool-call style, OpenClaw exec/subagent/message guidance, provider tool-discipline stable prefixes, CLI quick reference, or tool-only docs/status hints; workspace notes and TTS/context-only guidance may remain. Make buildSystemPromptReport derive skills.promptChars and skills.entries from the rendered Skills section or the same effective skills prompt used to render the final prompt, not the raw skillsPrompt. Keep hosted OpenResponses tool-name validation compatible except for clearly unsafe control/format characters, unless docs/tests explicitly cover a maintainer-approved stricter contract. Add focused regression tests in the affected prompt/report/OpenResponses suites and do not change unrelated prompt wording.
