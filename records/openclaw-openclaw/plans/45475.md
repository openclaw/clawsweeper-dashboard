# Coding Plan for openclaw/openclaw#45475: Feature: channels.slack.ignoreSubtypes config key to suppress message_changed / message_deleted events

## Summary

Keep this open: current main and v2026.5.12 still route Slack `message_changed` and `message_deleted` through the subtype system-event path, and there is still no `ignoreSubtypes` Slack config key to suppress them before session wake.

## Plan

Implement https://github.com/openclaw/openclaw/issues/45475 as a narrow Slack plugin feature. Add an opt-in `channels.slack.ignoreSubtypes` config key, likely an enum array for `message_changed` and `message_deleted`, to Slack config types, zod schema, UI hints, generated config metadata, and docs. In the Slack message event path, short-circuit matching subtype system events before `authorizeAndResolveSlackSystemEventContext` / `enqueueSystemEvent` so ignored edit/delete events do not wake a session. Preserve the current default behavior when the key is absent, preserve `thread_broadcast` as a normal user message, and be deliberate about the existing Slack Assistant DM `message_changed` rehydration path added by `1915b29a3c48`; add tests that document the chosen semantics. Add focused regression tests proving ignored `message_changed` and `message_deleted` do not enqueue system events, plus schema/config tests. Add a changelog entry because this is user-visible config behavior. Do not move Slack behavior into core or add broad subtype policy beyond this Slack plugin config.
