# Coding Plan for openclaw/openclaw#14079: Linux: No desktop notifications support

Source report: [records/openclaw-openclaw/items/14079.md](records/openclaw-openclaw/items/14079.md)

## Summary

Keep open. Current main still lacks Linux desktop notification delivery for the core `system.notify` node surface: Linux policy allowlists the command, but the headless node host neither advertises nor dispatches it, while docs and CLI still scope notify behavior to macOS/headless exclusion.

## Plan

Implement Linux desktop notification support for https://github.com/openclaw/openclaw/issues/14079 in the core headless node-host `system.notify` path. Add a small `notify-send`-first backend that detects desktop-session availability (`DISPLAY` or `WAYLAND_DISPLAY`), passes title/body as argv values without shell interpolation, returns structured unavailable/not-authorized style failures when no usable backend/session exists, and leaves macOS, iOS, Android, Windows, and `system.run` behavior unchanged. Update node-host command advertisement, invoke dispatch, CLI/docs wording, and changelog. Add focused tests for command advertisement, invocation behavior, env sanitization/backends, CLI coverage, and gateway policy; do not add a new config option or broaden into D-Bus/`dunstify` fallback unless the existing design already has a narrow helper for it. Keep the related closed error-message issue https://github.com/openclaw/openclaw/issues/24616 as context only; it is not the implementation target.
