## Summary

Keep open. Current main still lacks Linux desktop notification delivery for the reserved `system.notify` node command: Linux can be allowlisted at the gateway layer, but the headless node host neither advertises nor handles the command, and the docs/CLI still scope notify to macOS/node-app behavior.

## Plan

Fix Linux desktop notification support for the reserved node-host `system.notify` command. Add a focused Linux path for the headless node host: advertise `system.notify` only when appropriate, detect desktop-session availability, invoke a notification backend safely with argv fields rather than shell interpolation, return structured unavailable errors for headless or missing-backend cases, and preserve existing macOS, iOS, Android, Windows, `system.run`, exec approval, and plugin behavior. Keep D-Bus/`dunstify` as follow-up unless the minimal design clearly requires them. Update focused tests, CLI/docs, and changelog. Do not move `system.notify` into a plugin or loosen the reserved-command boundary.
