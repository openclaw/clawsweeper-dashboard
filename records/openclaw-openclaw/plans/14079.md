# Coding Plan for openclaw/openclaw#14079: Linux: No desktop notifications support

Source report: [records/openclaw-openclaw/items/14079.md](records/openclaw-openclaw/items/14079.md)

## Summary

Keep open. Current main still does not provide Linux desktop delivery for the reserved `system.notify` node surface: Linux policy allows the command, but the node-host neither advertises nor handles it, and docs/CLI still describe notify as macOS/node-app-only.

## Plan

Implement Linux desktop notification support for the reserved core `system.notify` node command tracked by https://github.com/openclaw/openclaw/issues/14079. Keep the scope focused: update the headless node-host advertisement/dispatch path, add a Linux notify backend that uses `notify-send` with title/body passed as argv values, return structured unavailable/headless/backend-missing results when no desktop session or backend is available, and do not broaden into D-Bus or `dunstify` fallback unless it stays small and well-tested. Update CLI copy, nodes docs, and CHANGELOG for the user-visible Linux support. Preserve macOS, iOS, Android, Windows, `system.run`, and exec approval behavior. Related context: https://github.com/openclaw/openclaw/issues/24616 only fixed the rejection-message experience and should remain closed.
