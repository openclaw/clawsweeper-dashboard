# Coding Plan for openclaw/openclaw#14079: Linux: No desktop notifications support

Source report: [records/openclaw-openclaw/items/14079.md](records/openclaw-openclaw/items/14079.md)

## Summary

Keep open. Current main still lacks Linux desktop notification delivery for the core `system.notify` node surface: Linux policy allowlists the command, but the headless node host neither advertises nor dispatches it, and docs/CLI still scope notify away from Linux headless nodes.

## Plan

Implement Linux desktop notification support for the existing core `system.notify` node command. The fix should add a headless node-host notify path for Linux desktop sessions, advertise `system.notify` only when the backend is available or when a structured unavailable response is intentional, call notification tools with argv arrays rather than shell strings, gracefully report no-display/no-backend cases, and leave macOS, iOS, Android, and Windows behavior unchanged. Keep the first backend scope focused, preferably `notify-send` with safe detection; do not add broad D-Bus or daemon-specific behavior unless the existing design already has a narrow seam for it. Update CLI/docs and add a user-facing changelog entry because this changes Linux node behavior. Related refs: https://github.com/openclaw/openclaw/issues/14079 and https://github.com/openclaw/openclaw/issues/24616.
