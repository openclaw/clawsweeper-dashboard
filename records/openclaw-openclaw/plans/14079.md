# Coding Plan for openclaw/openclaw#14079: Linux: No desktop notifications support

Source report: [records/openclaw-openclaw/items/14079.md](records/openclaw-openclaw/items/14079.md)

## Summary

Keep open. Current main and the latest shipped release still lack Linux desktop delivery for the reserved `system.notify` node command: the headless node host neither advertises nor handles it, while docs and CLI still scope local notification delivery to macOS.

## Plan

Fix Linux desktop notification delivery for the existing `system.notify` / `openclaw nodes notify` surface. Implement a focused headless node-host path, likely in `src/node-host/invoke-system-notify.ts` plus `src/node-host/invoke.ts` and `src/node-host/runner.ts`, that advertises and handles `system.notify` on Linux only when appropriate, detects desktop-session/backend availability, uses `notify-send` via argv without shell interpolation, and returns structured unavailable/invalid-request failures for headless or missing-backend cases. Update `src/cli/nodes-cli/register.notify.ts`, `docs/nodes/index.md`, and relevant CLI docs/changelog as needed. Add regression coverage for command advertisement, invocation success/failure paths, and existing macOS/Android/iOS/Windows/system.run behavior staying unchanged. Do not add broad D-Bus or `dunstify` fallback scope unless it stays small and tested; leave plugin registry reservation semantics intact.
