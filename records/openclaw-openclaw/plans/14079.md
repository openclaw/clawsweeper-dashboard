# Coding Plan for openclaw/openclaw#14079: Linux: No desktop notifications support

Source report: [records/openclaw-openclaw/items/14079.md](records/openclaw-openclaw/items/14079.md)

## Summary

Keep open. Current main and the latest release still do not deliver Linux desktop notifications: Linux is allowlisted for `system.notify`, but the headless node host does not advertise or handle the command, and docs/CLI still describe notification delivery as macOS-only node behavior.

## Plan

Implement Linux desktop notifications for the existing core `system.notify` node surface. Add a focused Linux headless node-host path that advertises and handles `system.notify` when a desktop session is available (`DISPLAY` or `WAYLAND_DISPLAY`), invokes a supported backend such as `notify-send` with title/body passed as argv fields rather than through a shell, and returns structured unavailable-style errors when no display or backend is present; keep D-Bus, `dunstify`, package-install, and daemon-discovery scope out unless the implementation is still small and covered. Update CLI/docs to remove the mac-only Linux gap, add a `CHANGELOG.md` entry, preserve macOS/iOS/Android/Windows behavior, and do not loosen plugin command reservation or exec approval semantics.
