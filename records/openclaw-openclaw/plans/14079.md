# Coding Plan for openclaw/openclaw#14079: Linux: No desktop notifications support

Source report: [records/openclaw-openclaw/items/14079.md](records/openclaw-openclaw/items/14079.md)

## Summary

Keep open. Current main still lacks Linux desktop notification delivery for the core `system.notify` node surface: Linux policy allowlists the command, but the headless node host does not advertise or dispatch it, and the related closed report only improved the rejection error.

## Plan

Implement Linux desktop notification support for the core `system.notify` node surface on headless node hosts. Scope the fix to the node-host path: add a Linux notification backend, likely in a new `src/node-host/invoke-system-notify.ts`, detect desktop session/backend availability, invoke `notify-send` or the chosen backend with argv-only arguments and no shell interpolation, return structured unavailable/invalid failures for headless or missing-backend cases, and wire advertisement plus dispatch so Linux desktop node hosts can receive `openclaw nodes notify`. Do not change macOS, iOS, Android, plugin command reservation, exec approval behavior, or add a config option unless a maintainer explicitly asks. Update CLI copy, public docs, and CHANGELOG.md, and add focused regression tests for advertisement, invocation, sanitization/backend handling, and policy behavior.
