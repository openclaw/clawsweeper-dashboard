# Coding Plan for openclaw/openclaw#14079: Linux: No desktop notifications support

Source report: [records/openclaw-openclaw/items/14079.md](records/openclaw-openclaw/items/14079.md)

## Summary

Keep open. Current main still does not implement Linux desktop notification delivery for the reserved core `system.notify` node surface: Linux is policy-allowlisted, but the headless node host does not advertise or handle the command, and docs/CLI still scope notification delivery to macOS node behavior.

## Plan

Implement Linux desktop notification support for the existing core `system.notify` node command. Keep the change in core node-host paths because `system.notify` is reserved and plugins cannot register that exact command. Add a focused Linux headless node-host notify path with desktop-session/backend detection, argv-only spawning for title/body fields, structured unavailable handling when no desktop session or backend is available, and no changes to macOS/iOS/Android/Windows notify behavior. Prefer a small helper such as `src/node-host/invoke-system-notify.ts`; update node-host command advertisement/dispatch, CLI wording, nodes/headless docs, and `CHANGELOG.md`. Do not add broad D-Bus or `dunstify` fallback behavior unless the dependency contract is read and covered by focused tests.
