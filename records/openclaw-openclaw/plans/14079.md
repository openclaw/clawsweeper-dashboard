# Coding Plan for openclaw/openclaw#14079: Linux: No desktop notifications support

Source report: [records/openclaw-openclaw/items/14079.md](records/openclaw-openclaw/items/14079.md)

## Summary

Keep open. Current main still leaves Linux/headless node hosts unable to deliver desktop notifications: `system.notify` is allowed in the gateway policy, but the node host neither advertises nor dispatches it, and the exact command is reserved by core rather than externally claimable by a plugin.

## Plan

Implement Linux desktop notification support for the core headless node host `system.notify` command. Current main allows `system.notify` for Linux in gateway policy, but `src/node-host/runner.ts` does not advertise it and `src/node-host/invoke.ts` does not dispatch it. Add a focused Linux path, preferably a new `src/node-host/invoke-system-notify.ts`, that detects a desktop session (`DISPLAY` or `WAYLAND_DISPLAY`), finds a supported backend such as `notify-send`, passes title/body/sound/priority/delivery as argv values without shell interpolation, and returns structured unavailable/invalid failures when unsupported or headless. Update node-host advertisement, dispatch, CLI copy, docs, changelog, and focused tests. Do not change macOS, iOS, Android, Windows, or plugin command behavior beyond the shared core command contract; keep direct D-Bus/dunstify fallback out unless the implementation remains small and well-tested.
