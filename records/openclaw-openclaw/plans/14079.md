# Coding Plan for openclaw/openclaw#14079: Linux: No desktop notifications support

Source report: [records/openclaw-openclaw/items/14079.md](records/openclaw-openclaw/items/14079.md)

## Summary

Keep open. Current main still lacks Linux desktop delivery for the reserved `system.notify` node command: the Linux policy can allow the command, but the headless node host does not advertise or handle it, and docs/CLI still describe notification delivery as macOS/node-app-only.

## Plan

Implement Linux desktop notification support for the existing reserved `system.notify` node command. Keep the change focused on the headless node-host path: advertise `system.notify` only where the node host can attempt it, implement safe Linux desktop-session/backend detection with argv-only invocation such as `notify-send`, return structured unavailable/invalid errors for missing DISPLAY/WAYLAND_DISPLAY or missing backend, and leave macOS app, iOS, Android, Windows, exec approvals, and plugin command behavior unchanged. Update CLI/docs/changelog wording from mac-only to the supported platform set, and add focused tests for node-host command advertisement, invoke handling, env sanitization/security boundaries, and gateway policy interaction. Do not add broad D-Bus or dunst-specific fallback scope unless the existing code structure makes it trivial and testable in the same PR.
