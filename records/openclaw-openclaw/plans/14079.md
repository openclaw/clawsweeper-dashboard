# Coding Plan for openclaw/openclaw#14079: Linux: No desktop notifications support

Source report: [records/openclaw-openclaw/items/14079.md](records/openclaw-openclaw/items/14079.md)

## Summary

Keep open. Current main still lacks Linux desktop notification delivery: there is no `notify-send`/libnotify backend, the headless node host neither advertises nor handles `system.notify`, docs and CLI still scope notify to macOS, and the related closed issue only addressed the error experience.

## Plan

Implement Linux desktop notifications for the existing core node `system.notify` command on headless node hosts. Keep the first PR focused: add a `src/node-host/invoke-system-notify.ts` helper or equivalent, use safe argv-based process execution for `notify-send` with no shell interpolation, detect Linux desktop-session/backend availability (`DISPLAY` or `WAYLAND_DISPLAY`, plus backend presence), advertise or return structured `UNAVAILABLE` consistently with the node command contract, and leave macOS/iOS/Android behavior unchanged. Update `src/node-host/runner.ts`, `src/node-host/invoke.ts`, `src/cli/nodes-cli/register.notify.ts`, `docs/nodes/index.md`, `docs/cli/node.md`, and `CHANGELOG.md` as needed. Add regression tests for command advertisement and invocation behavior; stop and ask for maintainer direction if the implementation requires broad D-Bus/dunstify fallback design, a new config option, or shell execution.
