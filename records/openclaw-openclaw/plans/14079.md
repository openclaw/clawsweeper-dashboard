# Coding Plan for openclaw/openclaw#14079: Linux: No desktop notifications support

## Summary

Keep open. Current main still lacks Linux desktop delivery for the reserved `system.notify` node surface: Linux policy allowlists the command, but the headless node-host neither advertises nor handles it, and the CLI/docs still describe notify as macOS/node-app-only.

## Plan

Implement Linux desktop notification support for https://github.com/openclaw/openclaw/issues/14079. Add a Linux-only node-host `system.notify` path, preferably with a focused `src/node-host/invoke-system-notify.ts` helper, advertise `NODE_SYSTEM_NOTIFY_COMMAND` for Linux node-hosts when appropriate, and invoke `notify-send` using `spawn` argv fields only. Detect headless or unavailable backend states (`DISPLAY`/`WAYLAND_DISPLAY`/session bus and missing executable) and return structured unavailable-style errors rather than shelling out or silently claiming success. Do not change macOS app notification behavior, iOS/Android native notify behavior, Windows policy, exec approvals, or plugin command registration semantics. Update `openclaw nodes notify` wording, nodes docs, changelog, and focused tests for command advertisement, dispatcher behavior, headless/backend-missing cases, argv-safe title/body handling, and existing policy behavior. Treat https://github.com/openclaw/openclaw/issues/24616 as related context only; it covered error diagnostics, not Linux delivery.
