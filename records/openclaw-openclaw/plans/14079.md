# Coding Plan for openclaw/openclaw#14079: Linux: No desktop notifications support

Source report: [records/openclaw-openclaw/items/14079.md](records/openclaw-openclaw/items/14079.md)

## Summary

Keep open. Current main still lacks Linux desktop delivery for the reserved `system.notify` node surface: the Linux policy can allow the command, but the headless node host does not advertise or handle it, and docs/CLI still present notification delivery as macOS/node-app-only. The related closed item at https://github.com/openclaw/openclaw/issues/24616 improved the rejection message; it did not add Linux notification delivery.

## Plan

Implement Linux desktop notification support for the existing reserved `system.notify` node-host command from https://github.com/openclaw/openclaw/issues/14079. Add a Linux-safe notifier helper under `src/node-host` that uses `notify-send` through argv-only process spawning when a desktop/session environment is present, returns a structured unavailable result when headless or when no backend is available, and never shells/interpolates user-controlled title/body text. Wire the headless node host to advertise and handle `system.notify` for the supported Linux desktop path without changing macOS, iOS, Android, Windows, or exec behavior. Update `openclaw nodes notify` wording, nodes/CLI docs, and CHANGELOG. Add focused tests for advertisement, invocation success/failure, headless/backend-missing behavior, CLI command shape, and gateway policy expectations. Stop and ask maintainers before adding direct D-Bus or `dunstify` fallback support, new config options, or broader Linux companion-app work.
