# Coding Plan for openclaw/openclaw#14079: Linux: No desktop notifications support

Source report: [records/openclaw-openclaw/items/14079.md](records/openclaw-openclaw/items/14079.md)

## Summary

Keep open. Current main and the latest shipped release still do not provide Linux desktop delivery for `system.notify`: the headless node host omits the command, the dispatcher has no notify path, and docs/CLI still scope notification delivery to macOS while the related closed report only improved the error message.

## Plan

Implement Linux desktop notification support for the existing core `system.notify` node command. Add the command to the Linux/headless node-host advertised commands only when the node host can handle it, implement a safe argv-only backend path such as `notify-send` with desktop-session detection and structured unavailable behavior for headless/no-backend cases, and do not shell-interpolate title/body/sound values. Keep D-Bus or `dunstify` fallback support out of the first PR unless it stays small and well-tested. Update CLI wording, nodes docs, and changelog for Linux support; preserve existing macOS, Android, iOS, Windows, exec approvals, and plugin registry behavior. Include focused tests for command advertisement, invocation success/failure/degradation, env sanitization, CLI coverage, and gateway policy.
