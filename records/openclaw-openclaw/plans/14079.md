# Coding Plan for openclaw/openclaw#14079: Linux: No desktop notifications support

Source report: [records/openclaw-openclaw/items/14079.md](records/openclaw-openclaw/items/14079.md)

## Summary

Keep open. Current main and the latest release still do not deliver Linux desktop notifications for the reserved core `system.notify` node surface: Linux is policy-allowlisted, but the headless node host neither advertises nor handles the command, and docs/CLI still describe notification delivery as macOS/node-app-only.

## Plan

Implement Linux headless node-host support for the reserved `system.notify` command from https://github.com/openclaw/openclaw/issues/14079. Keep the change in core node-host ownership: advertise `system.notify` for Linux node hosts only when appropriate, add a dispatcher path that detects a desktop session (`DISPLAY` or `WAYLAND_DISPLAY`) and uses a small safe backend such as `notify-send` when available, and return a structured unavailable/permission-style failure when headless or missing the backend. Pass notification title/body/sound/priority/delivery as argv values only; do not use shell interpolation, do not route through `system.run`, and do not let plugins register the reserved command. Update CLI wording, nodes docs, and changelog for the Linux support, while preserving macOS, iOS, Android, Windows, exec approval, system.run, system.which, and plugin node-host behavior. Add focused regression tests for node-host command advertisement, `system.notify` invocation behavior, headless/missing-backend degradation, env sanitization, CLI coverage, and gateway policy forwarding.
