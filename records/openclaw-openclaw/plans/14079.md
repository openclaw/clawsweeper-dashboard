# Coding Plan for openclaw/openclaw#14079: Linux: No desktop notifications support

Source report: [records/openclaw-openclaw/items/14079.md](records/openclaw-openclaw/items/14079.md)

## Summary

Keep open. Current main and the latest release still have the Linux notification gap: Linux is allowlisted for `system.notify`, but the headless node host does not advertise or handle the command, while CLI/docs still describe notification delivery as macOS-node behavior. The closed related report at https://github.com/openclaw/openclaw/issues/24616 covered the rejection-message experience, not Linux notification delivery.

## Plan

Implement Linux support for the existing core `system.notify` node command. Keep the fix in the headless node-host/core command boundary; do not add plugin-owned duplicate command surfaces or broad package-install behavior. Add a focused Linux notification handler that detects desktop-session availability, invokes a backend such as `notify-send` without shell interpolation, returns structured unavailable/invalid-request errors when no display or backend is available, and leaves macOS app, Android, iOS, Windows, `system.run`, exec approvals, browser proxy, and plugin node-host behavior unchanged. Update node-host advertisement/dispatch, CLI wording, public node docs, and changelog as appropriate. Preserve the diagnostics-only fix from https://github.com/openclaw/openclaw/issues/24616.
