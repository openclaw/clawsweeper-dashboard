# Coding Plan for openclaw/openclaw#14079: Linux: No desktop notifications support

Source report: [records/openclaw-openclaw/items/14079.md](records/openclaw-openclaw/items/14079.md)

## Summary

Keep open. Current main and the latest release still lack Linux desktop delivery for the reserved core `system.notify` path: Linux is allowed by gateway policy, but the headless node host neither advertises nor handles the command, and CLI/docs still describe notification delivery as macOS-only.

## Plan

Implement Linux support for the existing reserved core `system.notify` path used by `openclaw nodes notify`. Keep the fix in the headless node-host/core boundary: advertise and handle Linux `system.notify`, use a safe `notify-send`/desktop notification backend without shell interpolation, return structured unavailable behavior when there is no desktop session or backend, and preserve existing macOS, Android, iOS, Windows, `system.run`, exec approvals, and plugin node-host behavior. Update CLI text, node docs, and changelog for the user-visible Linux support. Add focused regression coverage for command advertisement, dispatch success/unavailable cases, and gateway policy interaction. Do not move this into a plugin or allow plugins to claim the reserved `system.notify` command.
