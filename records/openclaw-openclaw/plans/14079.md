## Summary

Keep open. Current main and the latest shipped release still do not deliver Linux desktop notifications for the reserved core `system.notify` node command; the headless node host does not advertise or handle it, and the related diagnostic issue only improved the rejection error.

## Plan

Implement Linux desktop notification support for the reserved core `system.notify` node command requested at https://github.com/openclaw/openclaw/issues/14079. Keep the change focused in the headless node host: advertise `system.notify` only when the Linux node host can attempt desktop notification delivery, implement a Linux backend that avoids shell interpolation and passes title/body as argv or structured fields, gracefully return structured unavailable/not-authorized style errors when no desktop session or backend is available, and leave macOS, Android, iOS, Windows, exec, browser proxy, and plugin node-host behavior unchanged. Use https://github.com/openclaw/openclaw/issues/24616 only as diagnostic context; that closed item fixed error messaging, not Linux delivery. Update CLI/docs and add a user-facing changelog entry.
