## Summary

Keep open. Current main still lacks Linux desktop notification delivery for the reserved `system.notify` node surface: Linux is platform-allowlisted, but the headless node host neither advertises nor handles the command, and the docs/CLI still scope notification delivery away from Linux node hosts.

## Plan

Fix Linux desktop notification support for the existing core `system.notify` node command from https://github.com/openclaw/openclaw/issues/14079. Implement the repair in the headless node-host path, not as a plugin replacement: advertise `system.notify` only when appropriate for Linux desktop notification handling, dispatch it in `src/node-host/invoke.ts` through a small typed helper, and preserve current macOS, Android, iOS, Windows, exec, and plugin behavior. Prefer `notify-send` with argv-only execution and explicit desktop-session detection; return structured unavailable-style failures for headless or missing-backend cases. Do not add broad D-Bus or dunst fallback unless it stays tightly scoped and tested. Update CLI/docs/changelog wording to remove the mac-only Linux gap, and add focused regression tests for node-host advertisement, invoke behavior, gateway command policy, and CLI coverage.
