Source report: [records/openclaw-openclaw/items/14079.md](records/openclaw-openclaw/items/14079.md)

## Summary

Keep open. Current main still lacks Linux desktop delivery for the reserved core `system.notify` path: Linux is allowlisted in gateway policy, but the headless node host does not advertise or handle the command, and docs/CLI still describe notify as macOS-only.

## Plan

Implement Linux desktop notification support for the core `system.notify` node-host command requested by https://github.com/openclaw/openclaw/issues/14079. Keep the fix focused in the headless node-host path: advertise `system.notify` for Linux when appropriate, add an invoke handler that detects desktop session/backend availability, uses `notify-send` or a deliberately scoped backend without shell interpolation, returns structured unavailable errors for headless/no-backend states, and leaves macOS, Android, iOS, Windows, exec, and plugin behavior unchanged. Update CLI help/docs and add a changelog entry because this is user-visible. Do not move this into a plugin: `system.notify` is reserved by core. Preserve the related diagnostic improvement from https://github.com/openclaw/openclaw/issues/24616.
