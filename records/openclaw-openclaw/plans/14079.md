# Coding Plan for openclaw/openclaw#14079: Linux: No desktop notifications support

Source report: [records/openclaw-openclaw/items/14079.md](records/openclaw-openclaw/items/14079.md)

## Summary

Keep open. Current main and the latest shipped release still do not deliver Linux desktop notifications for the reserved `system.notify` node command: Linux policy can allow it, but the headless node host neither advertises nor handles it, and docs/CLI still scope notification delivery to macOS/node-app behavior.

## Plan

Implement Linux desktop notification delivery for the reserved `system.notify` command in the headless node host. Add a safe argv-only backend path, preferably starting with `notify-send` when `DISPLAY` or `WAYLAND_DISPLAY` indicates a desktop session, and return a structured unavailable result when no desktop session or backend is available. Do not shell-interpolate title/body/sound fields, do not change macOS/iOS/Android/Windows behavior, and keep D-Bus or `dunstify` fallback scope small unless it is cleanly testable. Update `openclaw nodes notify` copy, `docs/nodes/index.md`, `docs/cli/nodes.md` if needed, and `CHANGELOG.md`; add focused tests for node-host command advertisement, invocation success/unavailable behavior, env/backend sanitization, CLI/docs expectations, and gateway policy interaction. Preserve plugin boundary rules: `system.notify` stays core-owned.
