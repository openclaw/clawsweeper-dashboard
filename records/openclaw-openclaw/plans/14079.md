# Coding Plan for openclaw/openclaw#14079: Linux: No desktop notifications support

Source report: [records/openclaw-openclaw/items/14079.md](records/openclaw-openclaw/items/14079.md)

## Summary

Keep open. Current main and the latest shipped release still do not deliver Linux desktop notifications for the reserved core `system.notify` node command: the headless node host does not advertise or handle it, and the related diagnostic work only improved the rejection error.

## Plan

Implement Linux desktop notification support for the existing reserved `system.notify` node command. Keep the boundary in the core headless node host: advertise `system.notify` for Linux node hosts only when appropriate, add a focused handler that uses a safe argv-based backend such as `notify-send`, returns structured unavailable results for headless/no-backend cases, and preserves current macOS app, Android, iOS, Windows, exec, and plugin behavior. Do not shell-interpolate title/body/sound fields, do not add a new config option unless unavoidable, and keep D-Bus or dunstify fallback scope narrow unless maintainer guidance already exists. Update CLI/docs and add a changelog entry because this is user-visible. Include regression coverage for node advertisement, direct invocation behavior, unavailable/headless behavior, CLI wording, and gateway policy interaction.
