# Coding Plan for openclaw/openclaw#14079: Linux: No desktop notifications support

Source report: [records/openclaw-openclaw/items/14079.md](records/openclaw-openclaw/items/14079.md)

## Summary

Keep open. Current main still lacks Linux desktop notification delivery for the core `system.notify` node surface: Linux policy allowlists the command, but the headless node host neither advertises nor dispatches it, and docs/CLI still scope notify away from Linux headless nodes.

## Plan

Treat https://github.com/openclaw/openclaw/issues/14079 as the canonical report. Implement focused Linux headless node-host support for the existing reserved `system.notify` command: advertise the command from the headless node host only where appropriate, dispatch it safely on Linux using `notify-send` with title/body passed as argv values, gracefully return a structured unavailable result when there is no desktop session or backend, and preserve existing macOS, iOS, Android, and Windows behavior. Keep fallback scope narrow unless maintainers explicitly choose D-Bus or daemon-specific support. Update CLI text, public docs, changelog, and regression tests for advertisement, invocation, unavailable/headless behavior, and command policy. Do not move this to a plugin because `system.notify` is reserved core node-host surface; do not use shell interpolation for notification content.
