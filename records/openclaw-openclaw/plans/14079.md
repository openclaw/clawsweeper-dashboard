## Summary

Keep open. Current main and the latest shipped release still do not deliver Linux desktop notifications for the reserved `system.notify` node command: the headless node host neither advertises nor handles it, and the related closed error-message issue only improved diagnostics.

## Plan

Implement Linux desktop notification support for the reserved core `system.notify` node command on the headless node host. Keep the change focused: add a Linux node-host notify handler, advertise the command appropriately, detect desktop/no-display/no-backend cases cleanly, use `notify-send` via argv-only spawning with no shell interpolation, and return structured unavailable/authorization-style failures when notifications cannot be posted. Update CLI/docs and add a changelog entry. Preserve macOS, Android, iOS, Windows, `system.run`, `system.which`, exec approvals, plugin node-host commands, and gateway command policy behavior. Treat https://github.com/openclaw/openclaw/issues/24616 and PR https://github.com/openclaw/openclaw/pull/24656 as diagnostic context only; do not rework the already-landed error-message improvement unless the new Linux implementation needs a narrow adjustment.
