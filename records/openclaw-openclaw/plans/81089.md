## Summary

Keep open: current main still publishes the usage-cost cache lock with a hard link and only treats EEXIST as lock contention, so ENOTSUP from filesystems without hard-link support remains a real source-proven crash path.

## Plan

Fix https://github.com/openclaw/openclaw/issues/81089. Current main publishes the usage-cost cache refresh lock in src/infra/session-cost-usage.ts by writing a temp lock payload and hard-linking it to .usage-cost-cache.json.lock; acquireUsageCostCacheRefreshLock only handles EEXIST, so ENOTSUP/EOPNOTSUPP/EPERM from filesystems without hard-link support escapes as a crash. Make lock acquisition portable by using exclusive-create lock creation or by falling back to exclusive create when the hard-link publish step reports unsupported-link errors. Use src/infra/backup-create.ts as a local precedent for unsupported-link error codes, but do not change backup behavior, gateway protocol, pricing, cache scan semantics, or add config.
