## Plan

Fix the cron main-session `systemEvent` timeout bug reported at https://github.com/openclaw/openclaw/issues/50621. Current main gives all `systemEvent` jobs `DEFAULT_JOB_TIMEOUT_MS`, but `sessionTarget: "main"` plus `wakeMode: "now"` awaits `runHeartbeatOnce`, so long main-session agent work can be recorded as `cron: job execution timed out` even when work completes. Update the cron timeout policy so only the blocking main-session `systemEvent` wake-now path uses the agent-turn-scale safety timeout, while preserving the 10-minute default for `next-heartbeat` and preserving existing `agentTurn` explicit `timeoutSeconds` and no-timeout semantics. Add focused regression coverage in timeout-policy and the main wake-now cron path if needed. Do not add a broad new config option, do not make all `systemEvent` jobs unbounded, and do not change isolated agent cleanup behavior.
