## Summary

Keep open: current main and the latest release still route main-session `systemEvent` cron jobs through the generic cron watchdog while `wakeMode: "now"` waits on heartbeat work, so the false `cron: job execution timed out` state remains source-reproducible. The linked duplicate adds useful evidence that the timeout can diverge from real completed work, and the related timeout PRs were closed unmerged.

## Plan

Fix the cron runtime bug where a main-session `systemEvent` job with `wakeMode: "now"` can be marked `cron: job execution timed out` while the main session is still processing or has processed the event.
