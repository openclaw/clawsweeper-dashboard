## Plan

Fix https://github.com/openclaw/openclaw/issues/73780. Add focused regression coverage for `always`, `once`, and `silent` final error payload handling, keep the first useful error visible for `once`, and update Telegram docs to the current `always`/`once`/`silent` enum and cooldown default.
