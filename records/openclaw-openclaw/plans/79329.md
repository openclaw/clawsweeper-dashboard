## Summary

Keep open: current main still preflights only the resolved primary model for isolated cron runs and returns `skipped` before the configured fallback chain can reach the model fallback runner. The closed, unmerged fix attempt at https://github.com/openclaw/openclaw/pull/79335 remains useful repair context, and the later production comment confirms the same failure mode for another provider setup.

## Plan

Fix https://github.com/openclaw/openclaw/issues/79329. In isolated cron agent runs, an unreachable local primary currently makes `prepareCronRunContext` return `status: "skipped"` before configured fallbacks reach `runWithModelFallback`. Make cron model preflight evaluate the effective candidate chain and skip only after every preflight-relevant candidate is unavailable. Preserve payload `fallbacks` precedence, explicit `fallbacks: []`, configured agent/default fallback semantics, alias-aware model ref resolution, and local endpoint probe caching. Do not disable preflight or add a new config option. Add focused regression coverage for local primary unavailable plus cloud fallback available, all candidates unavailable, and no fallback configured. Include a changelog entry because this is user-visible cron behavior.
