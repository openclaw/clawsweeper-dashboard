## Summary

Keep open: current main still performs cron local-model preflight against only the resolved primary provider/model and returns `skipped` before the configured fallback chain reaches the executor. The linked direct fix was closed unmerged, and the open retry-config PR addresses cold-start retries rather than this chain-aware fallback bug.

## Plan

Fix https://github.com/openclaw/openclaw/issues/79329. In isolated cron agent runs, when local-model preflight marks the resolved primary unavailable, consult the same effective fallback chain used by cron execution before returning `skipped`. Prefer a chain-aware preflight helper or a small preflight-selection helper that reuses existing model reference resolution, alias handling, payload `fallbacks`, explicit `fallbacks: []`, subagent fallback semantics, and endpoint cache behavior. Do not add a new config option, disable preflight globally, or broaden unrelated provider failover behavior. Add focused regression coverage for unreachable local primary plus cloud fallback, all-local-unavailable skip, explicit empty fallback list, payload fallback override, and alias-aware fallback resolution.
