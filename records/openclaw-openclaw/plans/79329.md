## Summary

Keep open: current main still preflights only the resolved primary cron model and returns `skipped` on local-provider unavailability before the configured fallback chain reaches the executor. The direct fix attempt at https://github.com/openclaw/openclaw/pull/79335 is closed unmerged, and the newer retry-config PR at https://github.com/openclaw/openclaw/pull/82145 is related but deliberately does not solve chain-aware fallback preflight.

## Plan

Fix https://github.com/openclaw/openclaw/issues/79329 in a focused PR. In isolated cron `agentTurn` preparation, an unavailable local primary provider must not immediately return `status: "skipped"` when configured fallbacks exist. Reuse the existing cron fallback policy/model resolution helpers so payload `fallbacks`, explicit `fallbacks: []`, agent/subagent/default fallbacks, and model aliases keep their current semantics. Probe candidates in order and continue with the first candidate whose preflight is available or not preflight-relevant; return skipped only when every candidate that can be used is unavailable. Do not add a new config option, do not disable local-provider preflight, and coordinate with but do not depend on the separate retry-config work in https://github.com/openclaw/openclaw/pull/82145. Add focused regression coverage for local-primary unavailable plus cloud fallback, all-local-unavailable skip, explicit empty fallback list, and alias-aware fallback resolution. Add a changelog entry if the change is user-visible per repo policy.
