## Summary

Keep open. Current main and the latest release still have the core problem: `stopReason` can be carried on the primary pi-embedded terminal lifecycle path, but the explicit `aborted` boolean never reaches the subscribe-layer state, setter, or emitted `lifecycle:end` payload. The CoClaw plugin context is relevant because event-only observers cannot rely on the `agent()` RPC completion frame, and the related fix PR was closed unmerged after bundling unrelated changes.

## Plan

Fix the pi-embedded terminal lifecycle metadata bug from https://github.com/openclaw/openclaw/issues/66534. Propagate the runner's explicit aborted boolean through the terminal lifecycle metadata contract used by `setTerminalLifecycleMeta`, store it in subscribe state, and include it with `stopReason` in both `emitAgentEvent` and direct `onAgentEvent` terminal lifecycle payloads. Do not revive the broad/unmerged diff from https://github.com/openclaw/openclaw/pull/66574 or infer cancellation only from stopReason. Add focused regression coverage for an aborted run path and for non-aborted natural completion so downstream gateway/session and SDK normalization expectations remain aligned. Changelog is expected if the emitted public event payload changes. Keep the fix scoped to the lifecycle metadata path and avoid unrelated Telegram, UI, command, or session refactors.
