## Summary

Keep open. Current main and the latest release still propagate `stopReason` on the pi-embedded terminal path but do not carry an explicit `aborted` boolean through the lifecycle metadata state, setter, or emitted `lifecycle:end` event.

## Plan

Fix the pi-embedded lifecycle terminal metadata bug reported at https://github.com/openclaw/openclaw/issues/66534. Propagate the runner's explicit `aborted` boolean through the typed `setTerminalLifecycleMeta` contract and subscribe state, then include it in the primary `handleAgentEnd` `lifecycle:end` payloads alongside the existing `stopReason`, preserving liveness, replay, yielded, and fallback behavior. Do not derive cancellation solely from `stopReason`, do not broaden the abort RPC/product surface, and do not pull in unrelated changes from the closed mixed-scope PR https://github.com/openclaw/openclaw/pull/66574. Add focused regression coverage for the subscribe handler and any downstream session/SDK classification that depends on the boolean. Add a `CHANGELOG.md` fix entry if the change is user-visible to plugin/SDK consumers.
