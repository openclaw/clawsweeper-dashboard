## Summary

Keep open. Current main and the latest stable release still emit `lifecycle:end` from the pi-embedded subscribe layer without an explicit `aborted` field, while downstream lifecycle consumers already distinguish `aborted` and `stopReason`.

## Plan

Fix https://github.com/openclaw/openclaw/issues/66534. The pi-embedded terminal lifecycle event currently emits `stopReason` but cannot emit the runner's explicit `aborted` boolean. Extend the existing terminal lifecycle metadata contract from the runner into subscribe state and `handleAgentEnd`, then emit `aborted` alongside `stopReason` for both `emitAgentEvent` and `onAgentEvent` terminal lifecycle payloads. Do not infer `aborted` solely from `lastAssistant.stopReason`; forward the runner/attempt explicit abort result through `setTerminalLifecycleMeta`. Add focused regression coverage for an aborted run and a natural completion so downstream SDK/session lifecycle behavior remains distinct. Keep the fix scoped to lifecycle metadata and tests; do not add a new RPC, config option, channel behavior, or broad lifecycle refactor.
