## Summary

Keep this open: current main has partial bounded transcript reads and optional compaction rotation, but the startup/open path can still read or reopen a pathological active transcript before any default quarantine or rotation guard, and there is still no cap on repeated full-bootstrap marker appends.

## Plan

Fix https://github.com/openclaw/openclaw/issues/63998. Implement a default startup/open safety guard for oversized active session transcript JSONL files: before repairSessionFileIfNeeded, prewarmSessionFile, SessionManager.open, compaction open, or readTranscriptFileState can materialize a pathological file, stat and stream enough of the active transcript to rotate/quarantine it into a bounded successor or mark it degraded without loading the full file. Add an idempotent/capped path for repeated openclaw:bootstrap-context:full markers so crash/restart loops cannot append unbounded bootstrap records. Keep this scoped to recovery/load safety; do not change agents.defaults.contextInjection defaults, do not broaden session.maintenance as part of this PR, and preserve session headers, file mode, active branch semantics, compaction references, malformed-line tolerance, and concurrent append safety. Add regression tests with synthetic oversized JSONL files proving the guard runs before full-file repair/open and that repeated bootstrap markers are capped. Add a single CHANGELOG.md fix entry unless the final change is test-only.
