## Plan

The fix should stat and stream/bound-read active transcripts, rotate or quarantine oversized/corrupt/pathological transcripts into a bounded successor that preserves a valid session header and enough compacted/recent context, and prevent repeated openclaw:bootstrap-context:full marker accumulation after crash/restart cycles. Do not introduce a new user-facing config knob unless maintainers explicitly require one; do not revive the raw session.maintenance transcriptRotateBytes/transcriptMaxLines API from the superseded PR; do not weaken SessionManager parentId chains or append raw Pi message records for current-format transcripts; do not broaden into unrelated memory leaks or backup-retention fixes except where the same boundary naturally prevents the loop.
