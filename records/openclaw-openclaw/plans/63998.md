# Coding Plan for openclaw/openclaw#63998: Session transcript doomloop: crash-restart cycle inflates transcript until gateway OOMs

Source report: [records/openclaw-openclaw/items/63998.md](records/openclaw-openclaw/items/63998.md)

## Summary

Keep open: current main and the latest release still let oversized active session JSONL files reach whole-file repair, dependency-backed `SessionManager.open()`, and whole-file transcript-state reads before any default quarantine or bounded successor recovery; the full-bootstrap marker path is improved for continuation-skip, but still lacks a hard cap for repeated successful full-bootstrap markers.

## Plan

Fix the session transcript doomloop tracked by https://github.com/openclaw/openclaw/issues/63998. Add a default pre-open safety boundary for active session JSONL files before `repairSessionFileIfNeeded`, `prewarmSessionFile`, `SessionManager.open()`, `readTranscriptFileState()`, compaction rotation, or manual compaction helpers can materialize the whole file. The boundary should stat and stream/tail-inspect the transcript, preserve the old file as an archive/quarantine when it is pathologically large, create or select a bounded successor transcript that can recover the session without reloading the entire history, and add regression coverage for oversized active files. Also cap or make idempotent `openclaw:bootstrap-context:full` marker persistence so repeated successful full-bootstrap cycles cannot append unbounded markers. Keep the fix in core session/agent ownership; do not add plugin-specific policy, do not require users to opt into a new config option for crash-loop recovery, and avoid lossy deletion of the archived transcript. Include a changelog entry if user-visible recovery behavior changes.
