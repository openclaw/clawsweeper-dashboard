## Summary

Keep open. Current main still has a source-backed duplicate path for model-authored WebChat MEDIA replies: WebChat renders raw MEDIA directives from the Pi-persisted assistant turn, while the gateway also appends a separate `gateway-injected` `:assistant-media` assistant transcript supplement for media-bearing agent-run payloads. The reporter added a MEDIA-only confirmation and another user confirmed the issue, and the earlier closed stale-media duplicate does not cover this raw MEDIA case.

## Plan

Fix https://github.com/openclaw/openclaw/issues/78885. Current main appends a gateway-injected `:assistant-media` transcript row for agent-run media payloads while Pi also persists the model assistant text containing `MEDIA:`, and WebChat renders that raw `MEDIA:` directive from transcript text. Implement a narrow fix so model-authored WebChat MEDIA directives produce exactly one displayed assistant message. Prefer using existing source-reply metadata or the persisted transcript/display contract to skip the supplemental agent-run media append when the Pi assistant turn already contains renderable MEDIA; preserve WebChat-managed media supplements for auto-TTS/audio, non-agent command replies, aborted/gateway-owned messages, and media broadcasts that have no Pi-owned assistant turn. Add regression coverage for agent-run MEDIA-only and MEDIA-plus-text outputs not producing a duplicate `gateway-injected` assistant-media row. Do not change channel/plugin APIs and do not remove UI parsing of raw MEDIA directives. Add a one-line changelog fix entry.
