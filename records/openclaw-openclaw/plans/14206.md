# Coding Plan for openclaw/openclaw#14206: Add Message Rate Limiting for Feishu Channel

Source report: [records/openclaw-openclaw/items/14206.md](records/openclaw-openclaw/items/14206.md)

## Summary

Keep open. Current main and the latest v2026.6.8 release have reactive Feishu send retries, but still do not provide the requested proactive outbound pacing/minimum-interval configuration before bursty chunks, media, comment replies, outbound messages, or streaming-card sends reach Feishu.

## Plan

Implement proactive Feishu outbound pacing for this issue. Add a Feishu-owned per-account outbound scheduler/minimum-interval layer that can be configured at channels.feishu and inherited/overridden by channels.feishu.accounts.<id>, without adding a core-wide rate-limit framework or changing unrelated channels. Cover normal reply chunks, comment replies, message-tool/outbound text, media/card/fallback sends, and streaming-card initial message sends where they hit Feishu message APIs; compose with the existing requestFeishuApi retry helper rather than replacing it. Update Feishu config schema, manifest metadata if needed, docs/channels/feishu.md, and focused tests. Do not edit CHANGELOG.md. Keep broader external API budgets/limits in https://github.com/openclaw/openclaw/issues/13615 and reactive retry behavior in the merged https://github.com/openclaw/openclaw/issues/70879 path.
