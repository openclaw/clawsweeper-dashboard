Keep open: current main still lacks a Feishu outbound pacing/minimum-interval config, and the Feishu reply, comment, outbound, media, and streaming-card delivery paths do not share a proactive per-account pacing helper. The related broader rate-limit and Feishu retry/backoff issues remain useful context, but they do not supersede this Feishu-specific proactive pacing request.

## Plan

Implement https://github.com/openclaw/openclaw/issues/14206 as a Feishu-plugin-owned outbound pacing feature. Add a top-level and per-account Feishu config option for a minimum interval between outbound Feishu delivery API calls, expose it in `extensions/feishu/src/config-schema.ts`, `extensions/feishu/openclaw.plugin.json`, and `docs/channels/feishu.md`, and add a changelog entry if the change is user-facing. Implement a shared per-account pacing helper inside the Feishu plugin and apply it to burst-prone delivery paths: normal chunked replies, comment-thread replies, message-tool outbound text/card sends, media sends/fallbacks, and initial/ongoing streaming-card delivery where appropriate. Keep the work Feishu-local; do not add a generic core rate limiter, do not solve the broader cross-provider tracker, and do not replace the separate retry/backoff work for Feishu rate-limit errors. Add focused regression tests using fake timers or injected wait hooks so the minimum interval is observable without slow sleeps.
