# Coding Plan for openclaw/openclaw#14206: Add Message Rate Limiting for Feishu Channel

Source report: [records/openclaw-openclaw/items/14206.md](records/openclaw-openclaw/items/14206.md)

## Summary

Keep open. Current main and the latest shipped tag still lack a Feishu outbound pacing or minimum-interval config, and the related broad rate-limit, retry/backoff, and streaming-throttle items do not cover this Feishu-specific proactive send pacing request.

## Plan

Implement configurable Feishu outbound pacing for https://github.com/openclaw/openclaw/issues/14206. Add a Feishu-owned per-account minimum-send-interval/pacing helper, expose the config at top level and per account in extensions/feishu/src/config-schema.ts and extensions/feishu/openclaw.plugin.json, document it in docs/channels/feishu.md, and add a user-facing CHANGELOG.md entry. Apply the pacing consistently to burstable delivery surfaces: chunked final replies, comment-thread replies, message-tool outbound text/cards, media/file/image sends, fallback sends, and initial streaming-card sends where they hit Feishu message APIs. Do not fold in broad cross-provider rate limiting from https://github.com/openclaw/openclaw/issues/13615 or reactive retry/backoff from https://github.com/openclaw/openclaw/issues/70879 unless maintainers explicitly expand scope; streaming update throttle work such as https://github.com/openclaw/openclaw/pull/31172 is not a substitute. Preserve existing default behavior unless a user opts into pacing, keep the implementation plugin-local, and add focused tests for schema inheritance/defaults and every paced send surface.
