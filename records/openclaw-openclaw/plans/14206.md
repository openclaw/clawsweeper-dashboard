# Coding Plan for openclaw/openclaw#14206: Add Message Rate Limiting for Feishu Channel

Source report: [records/openclaw-openclaw/items/14206.md](records/openclaw-openclaw/items/14206.md)

## Summary

Keep open: current main still lacks a Feishu proactive outbound pacing/minimum-interval config or scheduler; the merged retry work only reacts after Feishu returns a rate-limit response.

## Plan

Implement Feishu proactive outbound message pacing for https://github.com/openclaw/openclaw/issues/14206. Add a Feishu-owned scheduler around message create/reply sends so chunked replies, message-tool outbound text/cards, comment replies, media message sends, and initial streaming-card sends can enforce a configurable minimum interval without changing core channel APIs. Expose the setting in Feishu top-level and per-account config, manifest schema, and docs; preserve existing behavior by default unless maintainers choose a documented nonzero default. Compose with the existing requestFeishuApi retry helper from https://github.com/openclaw/openclaw/pull/89659 rather than replacing it. Do not implement the broader cross-provider rate-limit design from https://github.com/openclaw/openclaw/issues/13615, do not couple pacing to LLM segmentation, and do not add unrelated Feishu behavior changes.
