# Coding Plan for openclaw/openclaw#14206: Add Message Rate Limiting for Feishu Channel

Source report: [records/openclaw-openclaw/items/14206.md](records/openclaw-openclaw/items/14206.md)

## Summary

Current main still lacks a Feishu outbound minimum-interval or pacing configuration, so the issue remains valid. The merged Feishu retry work improves recovery after rate-limit responses but does not implement proactive spacing between Feishu sends.

## Plan

Implement configurable proactive outbound pacing for the Feishu plugin. Add a Feishu-owned per-account minimum-send-interval setting with conservative default behavior, expose it through the Feishu config schema, plugin manifest, and Feishu channel docs, and route all Feishu send surfaces that can burst API calls through one pacing helper: chunked replies, comment replies, outbound text/card/media/fallback sends, media message sends, and streaming-card sends. Preserve the existing retry/backoff behavior from https://github.com/openclaw/openclaw/pull/89659; do not add a core-wide rate-limit framework, cost controls, or unrelated provider/channel throttling. Add focused tests for config parsing/defaults and pacing across reply, comment, outbound, send/media, and streaming-card paths. Do not edit CHANGELOG.md.
