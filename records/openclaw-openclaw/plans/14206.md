# Coding Plan for openclaw/openclaw#14206: Add Message Rate Limiting for Feishu Channel

Source report: [records/openclaw-openclaw/items/14206.md](records/openclaw-openclaw/items/14206.md)

## Summary

Keep open. Current main and the latest shipped release still lack a Feishu outbound minimum-interval or pacing setting, while the related open trackers cover broader cross-provider rate limits and reactive retry/backoff rather than this proactive Feishu message pacing request.

## Plan

Implement Feishu outbound message pacing for https://github.com/openclaw/openclaw/issues/14206. Add an optional Feishu plugin config setting for outbound minimum interval/pacing at both top level and per-account scope, with a conservative default that preserves existing behavior unless configured. Validate it in `extensions/feishu/src/config-schema.ts`, expose it in `extensions/feishu/openclaw.plugin.json`, document it in `docs/channels/feishu.md`, and add a changelog entry because this is user-visible. Implement a Feishu-owned per-account sequential pacing helper and route every burst-capable delivery path through it: chunked normal replies, comment replies, message-tool outbound text/card finalization, media message create/reply, and streaming-card create/update/close where they issue Feishu API sends. Do not implement a global core rate limiter, cost controls, CLI `limits` commands, or retry/backoff policy; leave those to https://github.com/openclaw/openclaw/issues/13615 and https://github.com/openclaw/openclaw/issues/70879. Add focused tests proving configured pacing waits between multiple chunks/sends and that the default path is unchanged.
