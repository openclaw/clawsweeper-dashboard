# Coding Plan for openclaw/openclaw#14206: Add Message Rate Limiting for Feishu Channel

Source report: [records/openclaw-openclaw/items/14206.md](records/openclaw-openclaw/items/14206.md)

## Summary

Keep open: current main and the latest release include reactive Feishu send retry/backoff, but they still do not provide the requested proactive outbound minimum-interval or pacing configuration for Feishu message bursts.

## Plan

Implement Feishu-owned proactive outbound message pacing for https://github.com/openclaw/openclaw/issues/14206. Add a conservative top-level and per-account config option for minimum interval/outbound pacing in `extensions/feishu/src/config-schema.ts`, `extensions/feishu/openclaw.plugin.json`, and `docs/channels/feishu.md`; keep defaults backward-compatible/off or no added delay unless explicitly configured. Apply the pacing consistently to Feishu message delivery paths that can burst API calls: chunked normal replies, comment replies, message-tool/outbound text/card flows, media/fallback sends, and streaming-card send/start paths where relevant. Do not broaden this into the core-wide rate-limit design tracked by https://github.com/openclaw/openclaw/issues/13615, and do not replace the existing retry/backoff behavior from https://github.com/openclaw/openclaw/pull/89659. Add focused tests for config inheritance and paced dispatch ordering/delay using fake timers where practical.
