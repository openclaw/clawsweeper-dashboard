# Coding Plan for openclaw/openclaw#14206: Add Message Rate Limiting for Feishu Channel

Source report: [records/openclaw-openclaw/items/14206.md](records/openclaw-openclaw/items/14206.md)

## Summary

Keep open. Current main and v2026.6.8 have reactive Feishu send retry handling, but they still do not implement the requested proactive outbound minimum-interval or pacing configuration across chunked replies, comment replies, media/outbound sends, or streaming-card initial sends.

## Plan

Implement Feishu outbound proactive pacing for https://github.com/openclaw/openclaw/issues/14206. Add a Feishu-owned minimum outbound interval setting that is optional/default-off, works at top level and per account with clear inheritance, is represented in the Zod config schema, plugin manifest, and Feishu docs, and is applied consistently before Feishu message create/reply style sends that can burst: chunked normal/card replies, comment replies, outbound text/card/media/fallback sends, pairing notifications, and streaming-card initial sends. Compose with the existing requestFeishuApi reactive retry helper instead of replacing it. Keep the change inside the Feishu plugin/docs surfaces; do not add a broad core rate-limit framework, root rateLimits config, new dependencies, sidecar state files, or changelog edits. Add focused tests for config parsing/inheritance and for serialized spacing across representative text, comment, outbound/media, and streaming-card paths using injected timers or local helpers; include redacted live Feishu burst proof only if a maintainer provides credentials and it is feasible.
