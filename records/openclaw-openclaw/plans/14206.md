## Summary

Keep open. Current main and shipped v2026.5.20 still lack a Feishu outbound minimum-interval/pacing config, while chunked replies, comment replies, message-tool outbound, media, and initial streaming-card sends can call Feishu delivery APIs without a shared pacing gate; related broad rate-limit and reactive retry issues do not supersede this proactive Feishu-specific request.

## Plan

Add a Feishu-plugin-owned outbound pacing helper for this issue. Implement a configurable per-account minimum interval for Feishu outbound delivery, expose it in the Feishu config schema, plugin manifest, docs, and changelog, and apply it consistently to burstable Feishu message delivery surfaces: chunked replies, comment replies, message-tool outbound text/cards, media sends/fallbacks, and initial streaming-card create/reply sends where practical. Keep broader cross-provider rate limiting in https://github.com/openclaw/openclaw/issues/13615 and reactive retry/backoff in https://github.com/openclaw/openclaw/issues/70879 separate. Do not add a core-wide limiter or change default behavior unless the default is explicitly compatible/off. Validate with focused Feishu tests and formatting checks.
