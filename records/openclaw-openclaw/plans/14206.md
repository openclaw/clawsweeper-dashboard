# Coding Plan for openclaw/openclaw#14206: Add Message Rate Limiting for Feishu Channel

Source report: [records/openclaw-openclaw/items/14206.md](records/openclaw-openclaw/items/14206.md)

## Summary

Keep open. Current main and the latest shipped release still lack a Feishu-owned outbound minimum-interval/pacing configuration, and the related broad rate-limit and Feishu retry/backoff trackers do not supersede this proactive Feishu delivery request.

## Plan

Implement a narrow Feishu outbound pacing feature for https://github.com/openclaw/openclaw/issues/14206. Add a Feishu-owned per-account minimum-interval/pacing helper that is configurable at top level and per account, exposed in extensions/feishu/src/config-schema.ts, extensions/feishu/openclaw.plugin.json, docs/channels/feishu.md, and runtime config resolution. Apply the pacing consistently before burstable Feishu outbound delivery calls: chunked normal replies, comment-thread replies, message-tool outbound text/cards, media send sequences, sendMessageFeishu/sendCardFeishu, sendImageFeishu/sendFileFeishu, and initial streaming-card message create/reply where appropriate. Keep the default backward-compatible unless maintainers have already chosen an opt-in default; do not turn this into core-wide rate limiting, cost budgets, or retry/backoff. Leave https://github.com/openclaw/openclaw/issues/13615 for broad external API rate limiting and https://github.com/openclaw/openclaw/issues/70879 for retry/backoff after Feishu rate-limit errors. Add focused tests for schema/top-level/per-account inheritance and each paced delivery surface, update docs/manifest/config reference, and add a CHANGELOG.md entry because this is user-visible configuration.
