# Coding Plan for openclaw/openclaw#14206: Add Message Rate Limiting for Feishu Channel

Source report: [records/openclaw-openclaw/items/14206.md](records/openclaw-openclaw/items/14206.md)

## Summary

Keep open. Current main has reactive Feishu retry/backoff for rate-limit responses, but it still lacks the requested proactive Feishu outbound pacing/minimum-interval config and the chunked reply, comment reply, outbound, media, and card paths can still burst sends without a shared per-account limiter.

## Plan

Implement Feishu outbound message pacing for https://github.com/openclaw/openclaw/issues/14206. Add a Feishu-owned, opt-in minimum interval/pacing config accepted at both channels.feishu and channels.feishu.accounts.<id>, using the existing account config merge path. Apply the pacing consistently to Feishu outbound message creation/reply surfaces that can burst: chunked reply text/cards, comment replies, message-tool outbound text/cards, media sends/fallbacks, and streaming-card initial/final sends where applicable. Keep the scope inside the Feishu plugin; do not implement the broad core-wide limits/budget CLI from https://github.com/openclaw/openclaw/issues/13615, and do not undo the reactive retry behavior from https://github.com/openclaw/openclaw/pull/89659. Update Feishu docs and plugin metadata/schema as needed; do not edit CHANGELOG.md. Add focused tests proving top-level and per-account config parsing, account override/inheritance, ordered pacing between chunks/payloads, and no delay when the config is absent or disabled.
