# Coding Plan for openclaw/openclaw#14206: Add Message Rate Limiting for Feishu Channel

Source report: [records/openclaw-openclaw/items/14206.md](records/openclaw-openclaw/items/14206.md)

## Summary

Keep open: current main still lacks a Feishu outbound pacing/minimum-interval setting, and Feishu reply, comment, message-tool outbound, media, and streaming-card delivery paths still send through direct API calls without a shared per-account pacing helper. The broader rate-limit tracker and the Feishu retry/backoff tracker are useful related context, but neither supersedes this proactive Feishu-specific pacing request.

## Plan

Implement https://github.com/openclaw/openclaw/issues/14206 as a focused Feishu plugin enhancement. Add an optional Feishu outbound message pacing/minimum-interval configuration with top-level and per-account support, defaulting to current behavior unless configured. Keep the fix Feishu-owned; do not introduce a generic core rate limiter and do not fold in the retry/backoff behavior tracked by https://github.com/openclaw/openclaw/issues/70879 or the broad core design tracked by https://github.com/openclaw/openclaw/issues/13615. Apply the pacing helper consistently to burst-prone delivery surfaces: chunked normal replies, comment replies, message-tool/outbound text and cards, media sends and fallbacks, and streaming-card create/update paths where they count as Feishu outbound API calls. Update schema, manifest, Feishu docs, focused tests, and CHANGELOG.md if the behavior is user-facing.
