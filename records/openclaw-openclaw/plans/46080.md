Current main still supports custom `api: "anthropic-messages"` providers, but the empty non-visible-turn retry guard still omits `anthropic-messages`, so a successful post-tool empty `stop` turn can fall through to the incomplete-turn/error path instead of producing a recovered final answer or a completed-action acknowledgement.

## Plan

Fix https://github.com/openclaw/openclaw/issues/46080. On current main, custom providers using `api: "anthropic-messages"` can complete a tool_result round trip and then return a final assistant `stop` with empty content; `shouldApplyNonVisibleTurnRetryGuard` does not include `anthropic-messages`, and terminal payload building does not inline successful tool results.
