## Summary

Keep open. Current main implements the Gemini/Google side through the bundled Gemini web_search provider, but the requested ZAI/GLM web search support is still absent from manifest discovery, plugin registration, contract expectations, docs, and ZAI runtime files.

## Plan

Implement bundled ZAI/GLM web_search support for https://github.com/openclaw/openclaw/issues/17925. Add a ZAI-owned webSearchProviders contract entry, lazy provider runtime, registration in the ZAI plugin, tests, docs, and changelog. Reuse existing ZAI credential/base-url semantics and existing trusted web-search endpoint, cache, timeout, and untrusted-content wrapping patterns. Use Gemini, MiniMax, Kimi, and Grok web_search providers as implementation patterns, but do not add a core model-transport special case or a broad global nativeSearch policy knob. Verify current ZAI upstream docs/source for exact request body, endpoint family, optional parameters, and response/citation shape before coding. Preserve the existing Gemini provider behavior.
