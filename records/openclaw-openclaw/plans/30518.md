## Summary

Keep open: current main has Telegram delivery-pin support and low-level Bot API helpers, but it still does not expose standalone Telegram `pin`/`unpin` message actions through discovery, runtime aliases/dispatch, config gating, or docs; the related implementation PRs are closed unmerged.

## Plan

Implement Telegram `pin` and `unpin` message actions for https://github.com/openclaw/openclaw/issues/30518. Keep the change inside the Telegram plugin/action surface: add discovery entries in `extensions/telegram/src/channel-actions.ts`, runtime aliases and dispatch in `extensions/telegram/src/action-runtime.ts`, reuse `pinMessageTelegram` and `unpinMessageTelegram` from `extensions/telegram/src/send.ts`, and add a consistent Telegram action gate such as `channels.telegram.actions.pins` in the config type and zod schema. Add focused tests for discovery, account-scoped gating, runtime dispatch, helper behavior, and message-tool/schema integration. Update `docs/channels/telegram.md` and add a user-facing changelog entry if landing policy requires it.
