# Coding Plan for openclaw/openclaw#30518: [Feature]: pin/unpin message action for Telegram

Source report: [records/openclaw-openclaw/items/30518.md](records/openclaw-openclaw/items/30518.md)

## Summary

Keep open: current main has Telegram delivery pin support and low-level Bot API helpers, but it still does not expose standalone Telegram `pin`/`unpin` action discovery, runtime aliases, or a `pins` action gate; no open PR currently owns the remaining work.

## Plan

Implement Telegram message-tool `pin` and `unpin` actions for https://github.com/openclaw/openclaw/issues/30518. Keep the change Telegram plugin-owned: add discovery in `extensions/telegram/src/channel-actions.ts`, runtime aliases and dispatch in `extensions/telegram/src/action-runtime.ts`, reuse `pinMessageTelegram` / `unpinMessageTelegram` from `extensions/telegram/src/send.ts`, and add a consistent `channels.telegram.actions.pins` gate in config types/schema/tests. `unpin` should allow an optional message id because grammY's `unpinChatMessage` supports it. Update `docs/channels/telegram.md` and `CHANGELOG.md` for the user-visible feature. Do not add `list-pins` or pinned-message reads; that remains https://github.com/openclaw/openclaw/issues/38580. Do not move Telegram-specific policy into core beyond using the existing generic message action names and schema hooks. Include focused regression tests and note that live Telegram proof with a bot that can pin messages is needed before merge.
