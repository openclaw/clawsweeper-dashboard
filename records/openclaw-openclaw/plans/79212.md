## Summary

Keep open: current main still classifies Bedrock inference-profile-prefixed embedding IDs against the raw model string, so `global.cohere.embed-v4:0` misses the Cohere v4 path and can build the Titan request body reported in the issue. The related fix PR was closed unmerged, so there is no current implementation to close this against.

## Plan

Fix https://github.com/openclaw/openclaw/issues/79212 in the Amazon Bedrock plugin.
