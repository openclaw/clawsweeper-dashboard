## Summary

Keep open: current main still has the reported mismatch between restart timeout/drain behavior and marker-based main-session recovery. The ignored-SIGUSR1 in-flight token branch appears covered on main, but active sessions can still be left unmarked for immediate continuation after a forced restart.

## Plan

Fix https://github.com/openclaw/openclaw/issues/82433. Current main only recovers running main sessions when `abortedLastRun === true`, but config restart deferral timeout and run-loop drain timeout can force restart while active work remains without marking affected main sessions. Reuse the existing restart-aborted main-session recovery contract: before forced restart or drain-timeout shutdown, durably mark the known affected main sessions as interrupted/resumable, then let startup recovery consume the marker and preserve retry/failure semantics. Keep the already-fixed unauthorized SIGUSR1 token behavior intact and add/keep regression coverage if that path is touched. Keep scope narrow: do not add the broader restart manifest, durable inbound-message replay, new `gateway.restart` config surface, or channel ACK policy from the larger restart-recovery issues.
