# Coding Plan for openclaw/openclaw#81490: [Bug]: subagent completion spawns a fresh run on the parent's route instead of resuming the yielded session (supersedes #80310)

Source report: [records/openclaw-openclaw/items/81490.md](records/openclaw-openclaw/items/81490.md)

## Summary

Keep open: current main still lacks a durable paused-session binding for a yielded external-route parent, so subagent completion delivery can still fall back into a normal `agent` call that may create or overwrite the route session instead of resuming the yielded parent. The reporter's live Telegram/Bedrock evidence is consistent with the current source, and the later proposed fix commits referenced on the issue are not contained in current main or any local branch/tag.

## Plan

Fix https://github.com/openclaw/openclaw/issues/81490. When an embedded Pi run exits through `sessions_yield` with paused/yielded metadata, persist a bounded paused-session binding on the relevant route/session entry and make subagent completion delivery plus gateway `agent` route resolution prefer that paused session before any normal direct fallback can create a fresh session id. Preserve the existing direct fallback for truly dormant completed sessions, respect session freshness/reset semantics when expiring the binding, and do not change public plugin APIs or add config. Review the current behavior against https://github.com/openclaw/openclaw/pull/76927 and the merged routing work in https://github.com/openclaw/openclaw/pull/80242 so queue-first and originating-parent routing still behave correctly. Add focused regression coverage for yielded external-route parents and update the changelog if this lands as a user-visible fix.
