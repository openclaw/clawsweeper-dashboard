# Coding Plan for openclaw/openclaw#81490: [Bug]: subagent completion spawns a fresh run on the parent's route instead of resuming the yielded session (supersedes #80310)

## Summary

Keep open: current main still has a credible source-backed gap between the documented `sessions_yield` contract and completion routing for yielded parents, and the related open draft PR does not prove or directly fix the paused route-to-session binding described here.

## Plan

Fix https://github.com/openclaw/openclaw/issues/81490. Current docs require `sessions_yield` to let subagent completion events arrive as the next model-visible message, but current main parks the parent as `livenessState: "paused"` and completion delivery can fall through to a direct route `agent` run that overwrites the route session-store pointer. Implement a bounded canonical paused/yielded session binding for route entries, or an equivalent existing-contract mechanism, so subagent completion and inbound inter-session delivery for that route targets the yielded parent session id before normal dormant direct fallback. Preserve direct external delivery for truly dormant completed requesters, preserve the in-process handoff from https://github.com/openclaw/openclaw/pull/81383, and do not add a config option or channel-specific Telegram policy. Add regression coverage proving a yielded parent on a route receives the completion on the original session id and no fresh route session id is written.
