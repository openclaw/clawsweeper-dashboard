# Coding Plan for openclaw/openclaw#81490: [Bug]: subagent completion spawns a fresh run on the parent's route instead of resuming the yielded session (supersedes #80310)

Source report: [records/openclaw-openclaw/items/81490.md](records/openclaw-openclaw/items/81490.md)

## Summary

Keep open: current main still lacks a durable paused-session route binding for yielded external parents. The reporter’s live evidence matches the source path: yielded runs are marked paused/inactive, inactive embedded runs cannot be queued, and the gateway route resolver can allocate a new session when the stored route session is no longer considered reusable.

## Plan

Fix https://github.com/openclaw/openclaw/issues/81490 on current main. The observable bug is that a subagent completion handoff to a parent that called `sessions_yield` can go through the direct route path and create a fresh route session instead of resuming the yielded parent. Implement a bounded paused-session binding or equivalent canonical resolver owned by the session store/gateway/announce-delivery path: when an embedded run exits with `livenessState: "paused"` and `yielded: true`, persist enough route/session state to let subagent completion delivery target that paused session before freshness-based direct fallback allocates a new session. Clear or expire the marker after successful wake/resume and during store maintenance. Coordinate with the existing queue-first completion announce PR at https://github.com/openclaw/openclaw/pull/76927, but do not assume queue-first alone fixes this route binding bug. Useful source reference: cychen2021 fork commit 1bd49356e27329351b8ff075badf9dc791a6fa4a. Add focused regression tests for yielded route reuse, marker expiry/cleanup, and no hijack of unrelated stale sessions. Update CHANGELOG.md only if the maintainer workflow expects a user-facing fix entry. Do not add a config option, broad migration, or channel-specific Telegram policy.
