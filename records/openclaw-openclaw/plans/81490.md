# Coding Plan for openclaw/openclaw#81490: [Bug]: subagent completion spawns a fresh run on the parent's route instead of resuming the yielded session (supersedes #80310)

Source report: [records/openclaw-openclaw/items/81490.md](records/openclaw-openclaw/items/81490.md)

## Summary

Keep open: current main still has the reported gap. `sessions_yield` marks the parent run as paused, but completion routing resolves only the route entry `sessionId` plus active-run state, and direct `agent` dispatch can allocate a fresh session when that route entry is no longer fresh.

## Plan

Fix the yielded-parent route overwrite reported at https://github.com/openclaw/openclaw/issues/81490. Current main marks `sessions_yield` exits as paused/yielded, but subagent completion delivery only resolves the route entry `sessionId` plus active-run state; if direct handoff runs after the route entry is stale, gateway `agent` can allocate a fresh session and orphan the yielded parent. Implement a bounded internal paused-session binding for yielded route entries, likely in the session entry/lifecycle or agent handoff path, and make completion/inbound inter-session handoffs prefer that paused session before normal stale direct fallback. Preserve normal fallback for genuinely completed/dormant sessions, do not add a user config option, and do not duplicate the separate queue-first work in https://github.com/openclaw/openclaw/pull/76927, active-requester fallthrough in https://github.com/openclaw/openclaw/pull/79059, or Telegram parent-routing work in https://github.com/openclaw/openclaw/pull/80242. Add focused regression coverage for yielded route binding and update changelog for the user-visible bug fix.
