## Summary

Keep open: current main still does not preserve a yielded route-to-session binding for subagent completion handoffs. The reporter's live Telegram/Bedrock evidence matches the current source shape: `sessions_yield` parks the parent, inactive embedded runs cannot be queued, and completion delivery can continue into the direct `agent` path without a paused-session pointer.

## Plan

Fix https://github.com/openclaw/openclaw/issues/81490 on current main. Implement a bounded way to remember that an external route has a yielded/paused parent session after `sessions_yield`, and make subagent completion handoffs prefer resuming that paused session before direct `agent` fallback can allocate a fresh route session. Preserve existing direct fallback for truly dormant completed requester sessions, preserve active-run queue behavior, and do not treat https://github.com/openclaw/openclaw/pull/76927 as sufficient by itself. Add focused regression coverage for a yielded external parent whose completion announce would otherwise overwrite the route store entry with a new `sessionId`. Update docs only if the user-visible `sessions_yield` contract changes; otherwise no changelog should be added by the repair worker until maintainer landing.
