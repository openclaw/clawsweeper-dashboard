# Coding Plan for openclaw/openclaw#81490: [Bug]: subagent completion spawns a fresh run on the parent's route instead of resuming the yielded session (supersedes #80310)

Source report: [records/openclaw-openclaw/items/81490.md](records/openclaw-openclaw/items/81490.md)

## Summary

Keep open: current main still lacks a durable paused-session binding for yielded parent routes. The reporter's Telegram/Bedrock evidence matches the current source path: `sessions_yield` marks the run paused, inactive embedded runs are not queueable, and completion delivery can fall through to a direct gateway agent handoff that resolves the route without knowing it should resume the parked session. The adjacent queue-first work in https://github.com/openclaw/openclaw/pull/76927 and the merged origin-routing fix in https://github.com/openclaw/openclaw/pull/80242 do not cover the missing paused route/session reuse contract.

## Plan

Fix https://github.com/openclaw/openclaw/issues/81490. Current main can spawn a fresh route run instead of resuming a yielded parent after subagent completion because yielded sessions are marked paused/inactive but the route store does not persist or resolve a bounded paused session binding. Implement a generic, non-Telegram-specific fix: when an embedded run exits from `sessions_yield` with `livenessState: "paused"`/`yielded: true`, persist a bounded paused-for-subagent-completion marker on the route/session entry; have subagent completion activity resolution and gateway session reuse prefer that existing session while the marker is valid; clear the marker after successful wake and prune/ignore it after expiry. Preserve direct fallback for truly dormant non-paused requester sessions, do not add a public config option, and account for adjacent completion queue work in https://github.com/openclaw/openclaw/pull/76927. Add focused regression tests for the paused marker, gateway reuse, delivery wake path, and expiry/pruning. Add a `CHANGELOG.md` fix entry because this restores user-visible orchestrator behavior.
