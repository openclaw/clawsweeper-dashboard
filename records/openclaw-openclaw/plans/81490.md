## Summary

Keep open: current main still has a source-backed gap between the documented `sessions_yield` contract and route/session-store resolution for yielded parents. The related draft PR addresses completion queue/transport ordering, but it does not add a durable paused-session route binding for the failure reported here.

## Plan

Fix https://github.com/openclaw/openclaw/issues/81490 on current main. The observable bug is that a subagent completion announce to an external route can start a fresh parent run and overwrite the route's session-store `sessionId` after the original parent exited through `sessions_yield`; the completion should instead resume the yielded/paused parent session. Implement a bounded internal paused-session binding for yielded route entries and make completion/inbound inter-session route resolution prefer that paused session before normal dormant direct fallback. Preserve existing direct external delivery for truly dormant completed requesters, do not add a user config knob, do not raw-send child output to the external chat, and do not simply port https://github.com/openclaw/openclaw/pull/76927 unless it is extended to cover the paused-route binding. Add focused regression coverage for route binding reuse and existing fallback behavior; add a user-facing `CHANGELOG.md` fix entry because this repairs scheduled/orchestrated workflows.
