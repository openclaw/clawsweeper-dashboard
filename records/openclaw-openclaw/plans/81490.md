# Coding Plan for openclaw/openclaw#81490: [Bug]: subagent completion spawns a fresh run on the parent's route instead of resuming the yielded session (supersedes #80310)

Source report: [records/openclaw-openclaw/items/81490.md](records/openclaw-openclaw/items/81490.md)

## Summary

Keep open: current `main` still lacks a durable paused-session route binding, so a yielded parent can remain inactive while completion delivery falls back to an `agent` handoff on the route and risks creating a new run.

## Plan

Fix the yielded subagent completion route/session reuse bug from https://github.com/openclaw/openclaw/issues/81490. Current main can park a parent with `sessions_yield`, fail to queue the inactive session on completion, and then call the gateway `agent` route in a way that may create a fresh run and overwrite the route session pointer. Implement a bounded paused/yielded session binding or equivalent resolver so completion handoffs resume the yielded parent before direct route fallback can allocate a new session. Keep the fix in generic agent/session routing code; do not add Telegram-specific policy, user-facing config, startup migrations, or broad channel changes. Preserve direct fallback for truly dormant completed requester sessions and preserve busy-parent queue/deferral semantics. Add regression coverage for paused route binding persistence, stale binding cleanup, and subagent completion delivery resuming the original yielded parent.
