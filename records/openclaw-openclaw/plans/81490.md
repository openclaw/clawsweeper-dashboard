# Coding Plan for openclaw/openclaw#81490: [Bug]: subagent completion spawns a fresh run on the parent's route instead of resuming the yielded session (supersedes #80310)

Source report: [records/openclaw-openclaw/items/81490.md](records/openclaw-openclaw/items/81490.md)

## Summary

Keep open: current main still has a source-backed gap for yielded parent sessions. The code records `sessions_yield` as a paused run result, but announce routing still wakes only active embedded runs and otherwise falls through to an `agent` call on the external route, which can create or overwrite a route-bound session instead of resuming the yielded parent.

## Plan

Fix the yielded-parent subagent completion route overwrite described in https://github.com/openclaw/openclaw/issues/81490. On current main, `sessions_yield` exits with `livenessState: "paused"` and `yielded: true`, the parent run is no longer active, and completion delivery can fall through to a direct route `agent` call that starts or persists a different session for the route. Add a bounded durable binding for yielded parent sessions, likely on the route session entry or lifecycle metadata, and make subagent completion/inbound inter-session delivery prefer that paused session id before normal dormant direct fallback. Preserve existing direct external delivery for genuinely dormant completed requesters and preserve active-run queue/steer behavior. Check active adjacent PRs https://github.com/openclaw/openclaw/pull/76927 and https://github.com/openclaw/openclaw/pull/75662 first; reuse compatible lifecycle/paused-state work instead of duplicating it, but do not assume either PR fixes this route-binding issue. Add focused regression coverage around a yielded route-bound parent receiving a subagent completion without allocating or overwriting a fresh session id. Add a changelog fix entry if the final PR changes shipped user-facing behavior.
