# Coding Plan for openclaw/openclaw#81490: [Bug]: subagent completion spawns a fresh run on the parent's route instead of resuming the yielded session (supersedes #80310)

Source report: [records/openclaw-openclaw/items/81490.md](records/openclaw-openclaw/items/81490.md)

## Summary

Keep open: current main still lacks a durable paused-session binding for yielded route sessions, so a subagent completion can fall through to the normal route `agent` handoff and allocate a fresh session when the route entry is stale. Related open PRs address queue ordering or wrong-session routing, but none close this route-to-paused-session gap.

## Plan

Fix the subagent completion resume bug described in https://github.com/openclaw/openclaw/issues/81490. When an external-route parent session exits via `sessions_yield`, persist a bounded paused-session binding for that route and make subagent completion/inbound inter-session handoffs prefer resuming that paused session before the normal dormant route `agent` fallback can allocate a fresh session id. Preserve requester-agent mediation, existing queue/steer behavior, and direct fallback for genuinely dormant completed requesters. Do not add a user-facing config option. Add focused regression coverage for a route-bound parent that yields, ages past normal route freshness, receives a subagent completion, and resumes the original paused session rather than overwriting the route store with a new session id. Coordinate with, but do not fold in unrelated changes from, https://github.com/openclaw/openclaw/pull/76927, https://github.com/openclaw/openclaw/pull/79059, or https://github.com/openclaw/openclaw/pull/80242.
