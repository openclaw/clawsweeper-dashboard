## Summary

Keep open: current main and the latest release still lack the persisted paused-session route binding needed for yielded subagent completions to resume the waiting parent instead of falling through to direct handoff.

## Plan

Fix the yielded subagent completion route-resume bug described in https://github.com/openclaw/openclaw/issues/81490. Current main must persist a bounded paused/yielded session binding when an embedded run exits from `sessions_yield`, resolve that binding during subagent completion announce delivery for the requester route, queue or wake the paused parent session before direct fallback can start a fresh run, and clear or expire the binding so stale abandoned sessions do not hijack future route traffic. Use prior closed/dangling work such as https://github.com/openclaw/openclaw/pull/83091 and commits 1bd49356e27329351b8ff075badf9dc791a6fa4a / 7a490cefacf523d82da88ea58a7fcc89ebe5329d only as reference; implement against current main and preserve existing direct fallback for non-yielded dormant requester sessions. Add focused regression coverage for session-store paused metadata, yielded-run persistence, and subagent completion delivery choosing the paused parent.
