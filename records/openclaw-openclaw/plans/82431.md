## Summary

Keep open: this PR addresses a real generic plugin approval API gap, but the latest head still allows inconsistent approval action metadata that can turn plugin-owned command actions into native approval buttons.

## Plan

Repair https://github.com/openclaw/openclaw/pull/82431 by tightening plugin approval action metadata. Enforce that command actions cannot carry `decision` and decision actions must carry a valid decision, either through a discriminated TypeBox schema plus matching TypeScript types or through explicit normalization/rejection before storage. Update native adapter filters to render buttons only for `kind === "decision" && decision`, keep command actions visible as text, and add focused regression coverage for an invalid `{ kind: "command", decision: "allow-once" }` action. Do not broaden the public API beyond this PR or move AgentKit code into core. Verify protocol/API artifacts and the focused approval/channel tests listed for this PR.
