## Summary

Keep open: current main still has only partial spawned-session lineage and does not implement the requested durable root/trace session-chain contract. The direct implementation PR is closed and unmerged, and the newer related ACP metadata PR only serializes existing lineage fields rather than adding persisted rootSessionKey/traceId.

## Plan

Implement first-class session/task chain tracking for spawned sessions on current main. Add optional rootSessionKey and traceId to the persisted session model, derive them with parentSessionKey and spawnDepth for native subagent and ACP spawn paths, expose the fields through Gateway sessions.list rows and the agent-facing sessions_list output, and update docs/changelog for the user-facing contract. Preserve backward compatibility for existing sessions and do not add traversal APIs, UI graphing, descendant termination policy, provider/lifecycle-hook changes, or unrelated web-search/provider work from https://github.com/openclaw/openclaw/pull/11043. Check whether https://github.com/openclaw/openclaw/pull/73458 has landed before editing ACP serialization and avoid duplicating its _meta bridge work.
