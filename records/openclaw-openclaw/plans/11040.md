# Coding Plan for openclaw/openclaw#11040: Feature: First-class session/task chain tracking (parent/root/trace/depth)

Source report: [records/openclaw-openclaw/items/11040.md](records/openclaw-openclaw/items/11040.md)

## Summary

Keep open: current main and the latest release still provide only partial spawned-session lineage. `spawnedBy`, `parentSessionKey`, and `spawnDepth` exist in some paths, but the requested persisted `rootSessionKey` plus session-chain `traceId` contract is still missing from the session model, Gateway schema/projections, agent-facing list rows, and native/ACP spawn writes.

## Plan

Implement issue https://github.com/openclaw/openclaw/issues/11040 as a focused additive session-chain metadata contract. Add optional persisted `rootSessionKey` and session-chain `traceId` fields to the session model and Gateway protocol/schema, derive them consistently for native `sessions_spawn` and ACP spawn from the parent session (`parentSessionKey` is requester, `rootSessionKey` is parent root or parent key, `traceId` is parent trace or a new stable id, `spawnDepth` increments), and project the metadata through Gateway `sessions.list`/describe rows and agent-facing `sessions_list` rows. Preserve existing sessions and current permission/spawn policy behavior. Use closed PR https://github.com/openclaw/openclaw/pull/11043 only as source context; do not carry over unrelated lifecycle hooks, provider behavior, or broad UI changes. Keep existing ACP lineage projection from https://github.com/openclaw/openclaw/pull/73458 compatible and add/adjust docs for the new fields. Stop and ask for maintainer direction if the implementation would require reusing diagnostic tracing semantics instead of a session-chain trace id.
