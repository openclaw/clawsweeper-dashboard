## Summary

Keep open: current main still has only partial spawned-session lineage. It exposes `spawnedBy`, `parentSessionKey`, `spawnDepth`, child links, and ACP serialization for those existing fields, but it does not implement the requested persisted `rootSessionKey` or session-chain `traceId` contract in storage, Gateway rows, agent-facing `sessions_list`, or native/ACP spawn writes.

## Plan

Implement a focused replacement for https://github.com/openclaw/openclaw/issues/11040. Add optional first-class spawned-session chain metadata: `rootSessionKey?: string` and session-chain `traceId?: string` alongside existing `parentSessionKey`, `spawnedBy`, and `spawnDepth`. Persist and validate the new fields in the session model and `sessions.patch`; derive them in native subagent and ACP spawn writes as `parentSessionKey = requester session key`, `rootSessionKey = parent.rootSessionKey ?? parentSessionKey`, `traceId = parent.traceId ?? uuid`, and `spawnDepth = (parent.spawnDepth ?? 0) + 1`; expose them in Gateway `sessions.list` rows and agent-facing `sessions_list` rows. Preserve backward compatibility for old sessions and do not change spawn permissions, recursive spawn policy, delivery routing, or lifecycle/kill behavior. Use closed PR https://github.com/openclaw/openclaw/pull/11043 as source context but do not copy its unrelated provider, lifecycle-hook, or plugin changes. Treat merged PR https://github.com/openclaw/openclaw/pull/73458 and open PR https://github.com/openclaw/openclaw/pull/79962 as adjacent context only. Add focused regression coverage for multi-hop lineage and update relevant docs/protocol references; include a normal maintainer-owned changelog entry under active Changes without a human thanks line.
