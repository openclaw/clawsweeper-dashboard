# Coding Plan for openclaw/openclaw#11040: Feature: First-class session/task chain tracking (parent/root/trace/depth)

Source report: [records/openclaw-openclaw/items/11040.md](records/openclaw-openclaw/items/11040.md)

## Summary

Keep open: current main still implements only partial spawned-session lineage. The model and projections carry `spawnedBy`, `parentSessionKey`, and `spawnDepth` in several paths, and the merged ACP work exposes those existing fields, but the requested persisted `rootSessionKey` plus session-chain `traceId` contract is still absent from the core session model, Gateway schema/projections, agent-facing list rows, and spawn write paths.

## Plan

Implement the additive session-chain metadata requested in https://github.com/openclaw/openclaw/issues/11040. Add optional persisted `rootSessionKey` and session-chain `traceId` fields to the session model, derive them for native and ACP spawned sessions (`rootSessionKey = parent.rootSessionKey ?? parentSessionKey`, `traceId = parent.traceId ?? new uuid`, depth from parent depth), validate/persist them through Gateway patch/schema paths, and project them through Gateway `sessions.list`, agent `sessions_list`, and ACP metadata where appropriate. Preserve backward compatibility for older sessions and keep the repair narrowly on session lineage; do not bring over unrelated plugin/provider/lifecycle-hook changes from the closed source PR at https://github.com/openclaw/openclaw/pull/11043. Add focused multi-hop tests and documentation, and add a changelog entry if the final PR is user-visible.
