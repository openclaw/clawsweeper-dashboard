# Coding Plan for openclaw/openclaw#11040: Feature: First-class session/task chain tracking (parent/root/trace/depth)

Source report: [records/openclaw-openclaw/items/11040.md](records/openclaw-openclaw/items/11040.md)

## Summary

Keep open: current main still has only partial spawned-session lineage. It persists and exposes `spawnedBy`, `parentSessionKey`, and `spawnDepth` in some paths, and the merged ACP bridge now serializes those existing fields, but the requested `rootSessionKey` and session-chain `traceId` contract is still absent from storage, Gateway/tool projections, and spawn writes.

## Plan

Implement the Phase 1 session-chain metadata contract for https://github.com/openclaw/openclaw/issues/11040. Add optional `rootSessionKey` and `traceId` to the persisted session model and Gateway protocol/session row/tool row projections, derive them consistently for native `sessions_spawn` and ACP child-session spawn paths (`parentSessionKey = requester`, `rootSessionKey = parent.rootSessionKey ?? parentSessionKey`, `traceId = parent.traceId ?? crypto.randomUUID()`, `spawnDepth = (parent.spawnDepth ?? 0) + 1`), and preserve backward compatibility for existing sessions. Use https://github.com/openclaw/openclaw/pull/11043 only as source context; do not bring over unrelated lifecycle-hook, provider, web-search, or policy changes. Add focused regression tests for multi-hop reconstruction and old entries without the new fields, update user-facing docs for the new optional metadata, and add a changelog entry if this is treated as user-facing API/tool output.
