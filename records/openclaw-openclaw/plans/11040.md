# Coding Plan for openclaw/openclaw#11040: Feature: First-class session/task chain tracking (parent/root/trace/depth)

Source report: [records/openclaw-openclaw/items/11040.md](records/openclaw-openclaw/items/11040.md)

## Summary

Keep open: current main still only implements partial spawned-session lineage. `spawnedBy`, `parentSessionKey`, and `spawnDepth` exist in some storage and projection paths, and the merged ACP bridge serializes those existing fields, but the requested persistent `rootSessionKey` and session-chain `traceId` contract is still absent from the model, Gateway/tool rows, and spawn write paths.

## Plan

Implement a focused replacement for this issue: add optional first-class session-chain metadata for spawned sessions without changing spawn permissions or traversal APIs. Persist `rootSessionKey` and a session-chain `traceId` alongside existing `parentSessionKey` and `spawnDepth`, derive them in native and ACP spawn write paths from the requester session, expose them in Gateway `sessions.list` rows and agent-facing `sessions_list` rows, update docs/changelog for the user-visible contract, and add regression coverage for multi-hop reconstruction. Use https://github.com/openclaw/openclaw/pull/11043 only as source context; do not bring over its unrelated provider, plugin, or lifecycle-hook changes. Preserve backward compatibility for existing sessions with missing fields and keep authorization/visibility behavior unchanged.
