# Coding Plan for openclaw/openclaw#11040: Feature: First-class session/task chain tracking (parent/root/trace/depth)

Source report: [records/openclaw-openclaw/items/11040.md](records/openclaw-openclaw/items/11040.md)

## Summary

Keep open: current main still has only partial spawned-session lineage and does not implement the requested durable rootSessionKey/traceId session-chain contract. The direct implementation PR is closed unmerged, while the merged ACP lineage PR only serializes existing Gateway parent/depth metadata.

## Plan

Implement first-class spawned-session chain metadata for this issue. Add optional persisted `rootSessionKey` and session-chain `traceId` fields alongside the existing `parentSessionKey` and `spawnDepth` contract; derive them consistently for native `sessions_spawn` and ACP child sessions; expose them through Gateway `sessions.list`, sessions changed rows/events where appropriate, and the agent-facing `sessions_list` output. Preserve backward compatibility for existing entries, keep the change additive, and do not mix in lifecycle termination, provider/web-search, or broad sessionId-rotation lineage work. Add focused regression coverage for a multi-hop spawn chain proving parent/root/trace/depth reconstruction from current public/session-tool surfaces. Add/update user-visible docs and a changelog entry because this changes public session metadata. Stop and ask for maintainer review if the implementation would require changing spawn permissions, recursion policy, a new traversal method, or a different traceId meaning than a session-chain identifier.
