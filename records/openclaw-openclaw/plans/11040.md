# Coding Plan for openclaw/openclaw#11040: Feature: First-class session/task chain tracking (parent/root/trace/depth)

Source report: [records/openclaw-openclaw/items/11040.md](records/openclaw-openclaw/items/11040.md)

## Summary

Keep open: current main still only has partial spawned-session lineage. It persists and exposes spawnedBy, parentSessionKey, spawnDepth, and childSessions in some paths, but rootSessionKey and a session-chain traceId are still absent from the persistent model, Gateway schema/projections, agent-facing sessions_list rows, and native/ACP spawn writes.

## Plan

Implement first-class spawned-session chain metadata for https://github.com/openclaw/openclaw/issues/11040. Add optional rootSessionKey and traceId to the persisted SessionEntry contract, Gateway protocol/schema and row projections, and agent-facing sessions_list rows; preserve existing parentSessionKey, spawnedBy, and spawnDepth behavior. In native and ACP spawn paths, derive parentSessionKey from the requester, rootSessionKey from the parent's rootSessionKey or parent key, traceId from the parent's traceId or a newly generated stable id, and spawnDepth from the parent depth + 1. Keep this metadata-only and backward compatible for old sessions; do not change spawn permissions, lifecycle policy, provider behavior, unrelated hooks, or rotated usage-family semantics. Use https://github.com/openclaw/openclaw/pull/11043 only as context and avoid its unrelated changes. Add regression coverage for multi-hop reconstruction through Gateway and tool outputs, update relevant docs, and add a user-facing changelog entry if this lands through the maintainer repair lane.
