Source report: [records/openclaw-openclaw/items/11040.md](records/openclaw-openclaw/items/11040.md)

## Summary

Keep open: current main still has only partial spawned-session lineage. It persists and exposes `spawnedBy`, `parentSessionKey`, and `spawnDepth` in several paths, and the merged ACP work serializes those existing fields, but the requested persisted `rootSessionKey` and session-chain `traceId` contract is still absent; the direct implementation PR is closed and unmerged.

## Plan

Implement the Phase 1 session-chain metadata contract for https://github.com/openclaw/openclaw/issues/11040. Add optional persisted rootSessionKey and session-chain traceId fields to SessionEntry, Gateway protocol/patch validation, Gateway session list row types/builders, and agent-facing sessions_list rows. Derive parentSessionKey/rootSessionKey/traceId/spawnDepth consistently for native subagent spawning and ACP spawning: parentSessionKey is the requester session key, rootSessionKey is the parent's rootSessionKey or parent key, traceId is the parent's chain traceId or a new stable id for the chain, and spawnDepth increments from the parent. Preserve backward compatibility for existing sessions with missing fields, avoid changing spawn permissions/policy, and keep diagnostic request traceId semantics separate unless maintainers explicitly choose to bridge them. Use PR https://github.com/openclaw/openclaw/pull/11043 only as source context; do not copy its unrelated provider/lifecycle-hook changes. Include focused tests for multi-hop native and ACP spawn metadata, Gateway sessions.list projection, sessions.patch validation, and agent sessions_list output. Add docs only where the public session/Gateway/tool contract exposes the fields; changelog can be handled by maintainers at landing.
