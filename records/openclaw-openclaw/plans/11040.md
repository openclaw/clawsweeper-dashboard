# Coding Plan for openclaw/openclaw#11040: Feature: First-class session/task chain tracking (parent/root/trace/depth)

Source report: [records/openclaw-openclaw/items/11040.md](records/openclaw-openclaw/items/11040.md)

## Summary

Keep open: current main and latest release still only provide partial spawned-session lineage. The stored and projected contracts include `spawnedBy`, `parentSessionKey`, `spawnDepth`, and child summaries in some paths, but the requested `rootSessionKey` plus session-chain `traceId` are still missing from the persistent model, Gateway protocol/projections, agent-facing `sessions_list`, and native/ACP spawn writes.

## Plan

Implement a focused replacement for https://github.com/openclaw/openclaw/issues/11040. Add optional session-chain metadata only: persist `rootSessionKey` and a session-chain `traceId` on spawned session entries, derive them in both native subagent and ACP spawn paths from the requester parent (`rootSessionKey = parent.rootSessionKey ?? parentSessionKey`, `traceId = parent.traceId ?? new stable id`, `spawnDepth = parent depth + 1`), expose the fields through Gateway sessions.list/projection events and agent-facing `sessions_list`, and document the metadata. Use https://github.com/openclaw/openclaw/pull/11043 as source context only; do not carry over unrelated provider, lifecycle-hook, UI graph, traversal API, recursive listing, config, or CHANGELOG changes. Coordinate with https://github.com/openclaw/openclaw/issues/47320 by leaving recursive/depth listing out of this PR. Preserve backward compatibility for older entries where fields are absent.
