# Coding Plan for openclaw/openclaw#11040: Feature: First-class session/task chain tracking (parent/root/trace/depth)

Source report: [records/openclaw-openclaw/items/11040.md](records/openclaw-openclaw/items/11040.md)

## Summary

Keep open: current main still has only partial spawned-session lineage. It persists and exposes spawnedBy, parentSessionKey, spawnDepth, and childSessions in some paths, but it does not implement the requested first-class rootSessionKey/traceId contract; the direct source PR is closed unmerged, and the merged ACP work only serializes existing parent/depth metadata.

## Plan

Fix https://github.com/openclaw/openclaw/issues/11040 with a focused replacement for the useful session-chain part of https://github.com/openclaw/openclaw/pull/11043. Add optional durable session-chain metadata to the core session contract: rootSessionKey and traceId, while preserving existing parentSessionKey, spawnedBy, and spawnDepth behavior. Derive the fields in native subagent and ACP spawn paths so child.parentSessionKey is the requester session key, child.rootSessionKey is parent.rootSessionKey ?? requester session key, child.traceId is parent.traceId ?? a new stable chain id, and child.spawnDepth remains parent depth + 1. Expose the optional fields through Gateway protocol/schema validation, sessions.patch/store persistence, GatewaySessionRow/buildGatewaySessionRow, sessions.list, and the agent-facing sessions_list output; include spawnDepth in the agent-facing row if still absent. Add focused regression coverage for multi-hop native and ACP chains, list output, and compatibility with older entries that lack the new fields. Do not copy unrelated provider, web-search, lifecycle-hook, or plugin changes from PR 11043. Add docs/changelog only for the user-visible contract change.
