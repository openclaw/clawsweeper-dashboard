## Summary

Keep open: current main still only provides partial spawned-session lineage. The session model, Gateway schema/projections, agent-facing session list, and native/ACP spawn paths still lack the requested `rootSessionKey` plus session-chain `traceId` contract, and the only merged related work exposes existing lineage to ACP rather than adding root/trace persistence.

## Plan

Implement the Phase 1 session-chain metadata requested in https://github.com/openclaw/openclaw/issues/11040. Add optional persisted `rootSessionKey` and session-chain `traceId` fields alongside existing `spawnedBy`, `parentSessionKey`, and `spawnDepth`; derive them for native `sessions_spawn` and ACP child sessions using parent lineage; expose them through Gateway `sessions.list` rows and agent-facing `sessions_list` rows; update ACP lineage projection only as needed to carry the new fields; document the fields. Keep the change metadata-only: do not alter spawn permissions, recursive spawning policy, diagnostic tracing semantics, provider behavior, plugin web-search behavior, or lifecycle hooks. Use the closed PR https://github.com/openclaw/openclaw/pull/11043 only as source context, not as a branch to revive. Add focused regression coverage for multi-hop root/trace reconstruction and compatibility for existing entries without the new fields. Add a CHANGELOG.md entry if maintainers treat the new session metadata as user/API-visible.
