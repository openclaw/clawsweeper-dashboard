## Summary

Keep open: current main still has only partial spawned-session lineage. It exposes `spawnedBy`, `parentSessionKey`, `spawnDepth`, child rows, and ACP projection for those existing fields, but it does not persist or expose the requested `rootSessionKey` and session-chain `traceId`, and no open or merged PR appears to supersede that remaining core contract work.

## Plan

Implement the bounded Phase 1 session-chain metadata contract for https://github.com/openclaw/openclaw/issues/11040. Add optional `rootSessionKey` and session-chain `traceId` to the persisted session entry and Gateway/session-list contracts, derive them consistently when native `sessions_spawn` and ACP spawn create child sessions, and expose them in Gateway `sessions.list`, agent `sessions_list`, and ACP lineage metadata where appropriate. Preserve backward compatibility for older entries, keep the work metadata-only with no spawn permission or recursive policy changes, and do not pull in unrelated provider, lifecycle-hook, UI graph, or traversal-API changes from the closed source PR. Add focused tests for multi-hop root/trace derivation and projections; update docs/changelog if the user-visible API surface changes.
