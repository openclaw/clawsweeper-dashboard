Implement the Phase 1 session-chain metadata contract for https://github.com/openclaw/openclaw/issues/11040. Add optional durable rootSessionKey and traceId fields alongside existing spawnedBy/parentSessionKey/spawnDepth, validate and persist them through the Gateway sessions patch/create/list surfaces, derive them consistently for native subagent spawning and ACP spawning, and expose them through GatewaySessionRow and the agent-facing sessions_list output. Preserve backward compatibility for existing sessions and keep this scoped to metadata; do not add a traversal API, UI graph, lifecycle termination propagation, provider changes, or unrelated lifecycle hooks. Use closed PR https://github.com/openclaw/openclaw/pull/11043 only as source context and account for the already-merged ACP serialization work in https://github.com/openclaw/openclaw/pull/73458. Add focused tests for multi-hop reconstruction and old entries missing the new optional fields. Add docs/changelog if the user-facing session API output changes.
