# Coding Plan for openclaw/openclaw#11040: Feature: First-class session/task chain tracking (parent/root/trace/depth)

Source report: [records/openclaw-openclaw/items/11040.md](records/openclaw-openclaw/items/11040.md)

## Summary

Keep open: current main and latest release still only provide partial session lineage. `spawnedBy`, `parentSessionKey`, `spawnDepth`, and child summaries exist in some paths, but the requested persisted `rootSessionKey` plus session-chain `traceId` contract is still missing from the session model, Gateway protocol/projections, agent-facing `sessions_list`, and native/ACP spawn writes.

## Plan

Implement the Phase 1 session-chain metadata contract for https://github.com/openclaw/openclaw/issues/11040. Add optional persisted `rootSessionKey` and `traceId` fields to the current session model and Gateway protocol/schema, derive them for both native `sessions_spawn` and ACP spawn paths from the parent chain (`rootSessionKey = parent.rootSessionKey ?? parentSessionKey`, `traceId = parent.traceId ?? generated id`, `spawnDepth = parent depth + 1`), and expose `spawnedBy`, `parentSessionKey`, `rootSessionKey`, `traceId`, and `spawnDepth` through Gateway session rows and agent-facing `sessions_list` rows. Use https://github.com/openclaw/openclaw/pull/11043 only as source context; do not carry over its unrelated plugin/provider/web-search/lifecycle-hook changes. Keep the change metadata-only: no recursive list UI, no spawn permission policy changes, no new config/env option, no CHANGELOG edit. Preserve compatibility for existing sessions with missing optional fields, add focused multi-hop regression coverage, and update relevant docs for the new fields.
