## Summary

Keep open: current main still only implements partial spawned-session lineage. `spawnedBy`, `parentSessionKey`, `spawnDepth`, child rows, and ACP projection for those existing fields are present, but the requested persisted `rootSessionKey` and session-chain `traceId` contract is still absent from storage, Gateway schema/projections, agent `sessions_list` rows, and native/ACP spawn writes; the original implementation PR is closed and unmerged, and the merged ACP work is adjacent but not a replacement.

## Plan

Implement Phase 1 first-class session/task chain tracking for OpenClaw sessions. Add optional `rootSessionKey` and session-chain `traceId` metadata to the persisted session entry and Gateway/session-list contracts, derive them consistently when native `sessions_spawn` and ACP spawn create child sessions, and expose them through Gateway `sessions.list` rows plus agent `sessions_list` output. Preserve backward compatibility for existing sessions and do not add spawn-permission, recursive-spawn policy, provider, lifecycle-hook, web-search, or config-option changes. Use the closed source PR https://github.com/openclaw/openclaw/pull/11043 only as context; keep the replacement narrowly scoped to session-chain metadata. Add focused multi-hop regression coverage for storage, patch/schema/projection, native spawn, ACP spawn, and agent-facing listing. Add a changelog entry if the change is user-visible.
