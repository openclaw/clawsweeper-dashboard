# Coding Plan for openclaw/openclaw#11040: Feature: First-class session/task chain tracking (parent/root/trace/depth)

Source report: [records/openclaw-openclaw/items/11040.md](records/openclaw-openclaw/items/11040.md)

## Summary

Keep open: current main still provides partial spawned-session lineage only. It has `spawnedBy`, `parentSessionKey`, `spawnDepth`, child rows, and ACP projection for those existing fields, but it does not persist or expose the requested `rootSessionKey` and session-chain `traceId`, and neither native nor ACP spawn derives them.

## Plan

Implement first-class spawned-session chain metadata for https://github.com/openclaw/openclaw/issues/11040. Add optional persisted `rootSessionKey` and session-chain `traceId` fields alongside the existing `spawnedBy`, `parentSessionKey`, and `spawnDepth` metadata; derive them in native and ACP spawn paths as parent root/trace or parent session plus a new UUID; expose them through Gateway `sessions.list`/patch row contracts and agent `sessions_list` output; update ACP metadata only if the Gateway row contract carries the new fields; keep older session entries backward compatible; avoid unrelated provider, lifecycle-hook, UI graph, traversal API, or policy changes; add focused multi-hop regression coverage and a changelog entry for the user-visible API addition.
