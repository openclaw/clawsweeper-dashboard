# Coding Plan for openclaw/openclaw#11040: Feature: First-class session/task chain tracking (parent/root/trace/depth)

Source report: [records/openclaw-openclaw/items/11040.md](records/openclaw-openclaw/items/11040.md)

## Summary

Keep open: current main and v2026.6.5 still only provide partial spawned-session lineage. The requested persisted `rootSessionKey` and session-chain `traceId` contract is still missing from the session model, Gateway protocol/projections, agent-facing `sessions_list` rows, and native/ACP spawn writes; the linked implementation PR is closed unmerged, while the merged ACP lineage PR only exposes existing parent/depth metadata.

## Plan

Implement the Phase 1 first-class session-chain metadata contract for this issue. Add optional `rootSessionKey` and session-chain `traceId` to the persisted session model, Gateway protocol/session patch validation, Gateway session row projections, agent-facing `sessions_list` row type/builder, and SDK/API types where appropriate. Derive these fields in both native subagent spawn and ACP spawn: child `parentSessionKey` should be the requester session key, child `rootSessionKey` should inherit the parent's root or fall back to the parent key, child `traceId` should inherit the parent's chain trace or create a new stable id for a root chain, and child `spawnDepth` should remain `(parent.spawnDepth ?? 0) + 1`. Preserve backward compatibility for older entries with missing fields, do not add config/env options, do not change spawn permissions, do not implement recursive listing/UI traversal, and do not copy unrelated lifecycle/provider changes from the closed source PR. Add docs and focused regression coverage for multi-hop reconstruction through Gateway and tool listings. Do not edit CHANGELOG.md; put release-note context in the PR body or commit message.
