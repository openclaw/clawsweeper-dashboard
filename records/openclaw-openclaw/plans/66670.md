# Coding Plan for openclaw/openclaw#66670: [Feature] sessions_spawn should populate agent_id in task_runs SQLite

Source report: [records/openclaw-openclaw/items/66670.md](records/openclaw-openclaw/items/66670.md)

## Summary

Keep open. Current main still resolves a target child agent for native and ACP `sessions_spawn`, but the task-run creation path does not pass that target as `agentId`, so `task_runs.agent_id` can still be attributed to the requester instead of the executing child agent.

## Plan

Fix https://github.com/openclaw/openclaw/issues/66670. Native and ACP `sessions_spawn` must persist the executing target agent in `TaskRecord.agentId` and SQLite `task_runs.agent_id` for cross-agent spawned tasks. In `spawnSubagentDirect`, carry `targetAgentId` into `RegisterSubagentRunParams`; in `createSubagentRunManager`, pass that value to `createRunningTaskRun`. In `spawnAcpDirect`, pass `targetAgentId` in both `createRunningTaskRun` branches. Add focused tests for native subagent and ACP cross-agent spawns that assert the task record/listing and SQLite reload contain the target agent, not the requester. Do not move attribution into Mission Control, parse labels, or add plugin-specific policy. Add a single-line `CHANGELOG.md` fixes entry because this affects plugin-facing task attribution; do not add bot thanks.
