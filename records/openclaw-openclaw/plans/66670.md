## Summary

Keep open. Current `main` still knows the target child agent for native and ACP `sessions_spawn`, but the task-run creation path does not pass that target as `agentId`, so `task_runs.agent_id` and task-ledger agent filtering can still attribute spawned work to the requester or leave it unhelpfully unset.

## Plan

Add focused regression coverage for native subagent and ACP spawns plus task registry/SQLite persistence proving `TaskRecord.agentId` and `task_runs.agent_id` are the child/executing agent.
