## Summary

Keep open. Current main and v2026.5.22 still expose the A2A reverse-send path: the target run gets requester session context, nested reply/announce steps run with `sourceTool: "sessions_send"`, and the normal tool factory still includes `sessions_send` without a scoped A2A guard.

## Plan

Fix https://github.com/openclaw/openclaw/issues/39476. Add a scoped internal guard or denial path so nested A2A target/reply/announce runs cannot use `sessions_send` to call back to the requester and duplicate delivery. Do not add a new public delivery mode or config option, and do not implement the broader dispatch-only feature from https://github.com/openclaw/openclaw/issues/44309 or the private-delivery API from https://github.com/openclaw/openclaw/pull/50720. Preserve normal top-level `sessions_send`, documented reply-back loop behavior, thread-target rejection, baseline reply snapshots, per-session nested lanes, and parent-owned ACP/native A2A skips. Add focused regression coverage proving the nested A2A tool surface denies or omits `sessions_send` while ordinary top-level sends remain available.
