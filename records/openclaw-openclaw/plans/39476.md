## Summary

Keep open. Current main still exposes the requester session in A2A context and runs nested A2A reply/announce turns with normal tool availability, so there is no framework guard preventing a reverse `sessions_send`; the related fix PRs are closed unmerged and the newer related issues track adjacent delivery/dispatch semantics rather than replacing this bug.

## Plan

Fix https://github.com/openclaw/openclaw/issues/39476 by adding a narrow framework guard that prevents reverse `sessions_send` during nested A2A target/reply/announce processing. Start in `src/agents/tools/sessions-send-tool.ts`, `src/agents/tools/sessions-send-tool.a2a.ts`, `src/agents/tools/agent-step.ts`, and the gateway agent/tool-resolution contract if a scoped internal tool exclusion must be carried through RPC. Preserve ordinary cross-session `sessions_send`, existing conversation-mode A2A semantics, and the current parent-owned ACP/native subagent skip behavior. Do not implement the broader dispatch-only mode from https://github.com/openclaw/openclaw/issues/44309 and do not add a public config knob unless maintainers explicitly choose that. Add regression coverage showing that a target/nested A2A run cannot call `sessions_send` back to the requester, while normal non-nested sends still expose the tool. Add a user-facing changelog fix entry if the behavior changes externally.
