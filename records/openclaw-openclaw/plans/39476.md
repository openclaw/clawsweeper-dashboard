## Plan

Fix the A2A `sessions_send` duplicate-message bug reported at https://github.com/openclaw/openclaw/issues/39476. Current main exposes the requester session key to the target prompt and runs target/reply/announce nested agent steps without a scoped guard, so the target can call `sessions_send` back to the requester and duplicate channel output. Add an internal scoped tool-availability guard that hides or denies `sessions_send` during A2A target, reply, and announce runs while preserving inline `reply` return behavior and the existing peer A2A reply-back/announce lifecycle. Do not add a public config option, do not implement the broader dispatch-only/private-delivery feature, and do not remove normal A2A conversation semantics. Likely files: `src/agents/tools/sessions-send-tool.ts`, `src/agents/tools/sessions-send-tool.a2a.ts`, `src/agents/tools/agent-step.ts`, gateway agent request/schema if needed, and focused sessions tests. Add or update regression coverage proving the target/nested A2A run cannot use `sessions_send` back to the requester while ordinary non-A2A `sessions_send` remains available. Add a `CHANGELOG.md` fix entry if the repair changes user-visible delivery behavior.
