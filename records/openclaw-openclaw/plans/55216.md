# Coding Plan for openclaw/openclaw#55216: [Bug] Telegram media group: silent image loss when one photo fails to download

Source report: [records/openclaw-openclaw/items/55216.md](records/openclaw-openclaw/items/55216.md)

## Summary

Keep open. Current main still skips recoverable Telegram media-group download failures and forwards the reduced album without a user-visible Telegram warning; the media-group flush timer is now configurable, but that does not address the reported silent image-loss notification gap.

## Plan

Fix https://github.com/openclaw/openclaw/issues/55216 on current main. In the Telegram plugin media-group handler, count album items that fail with isRecoverableMediaGroupError and items where resolveMedia returns null, keep successful media flowing to processMessageWithReplyChain, and send exactly one user-visible Telegram warning for the affected album that states how many items could not be fetched. Preserve the partial-success behavior introduced by https://github.com/openclaw/openclaw/pull/20598. Do not move batching to the gateway, do not add a new config option, and do not broaden into the placeholder/body work tracked by https://github.com/openclaw/openclaw/issues/40991 or PR https://github.com/openclaw/openclaw/pull/41048 unless a tiny shared helper is unavoidable. Use existing Telegram API error logging/reply-parameter patterns, include allow_sending_without_reply, and avoid unbounded per-photo warning spam. Add focused regression coverage for thrown recoverable failures and null resolveMedia results in media groups. Add a changelog entry because this is user-visible.
