## Summary

Keep open. Current main still hard-resets after context-overflow or compaction failure and only replays a bounded user/assistant transcript tail, so the requested reset reason, task goal, and progress handoff are still missing.

## Plan

Implement a narrow recovery handoff for hard context resets. When context overflow or compaction failure calls resetSessionAfterCompactionFailure, keep the hard reset safety valve but append a bounded agent-visible record to the newly rotated session that states the reset reason, the last user instruction or task goal when safely available, and safe recent progress context. Do not replay tool/system/compaction records verbatim, do not broaden provider or bootstrap retry policy, and keep the separate repeated-reset loop in https://github.com/openclaw/openclaw/issues/63216 out of scope. Add or update focused tests for session reset, transcript replay, and agent-runner execution paths, plus a changelog entry if the user-visible recovery behavior changes.
