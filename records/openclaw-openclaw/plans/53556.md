## Plan

Fix the remaining Feishu legacy config compatibility gap from https://github.com/openclaw/openclaw/issues/53556. Current main already supports `groupPolicy: "allowall"` and legacy `topicSessionMode`; do not broaden this into a generic startup migration system. Add Feishu-owned support for legacy top-level and per-account `botId` as an alias for `appId`, with explicit `appId` taking precedence when both are present. Update Feishu schema/account/setup readiness behavior and focused docs so upgraded users know to run doctor. Add regression coverage for schema acceptance or migration, credential resolution, setup readiness, and doctor compatibility; do not change unrelated Feishu group policy, session routing, webhook defaults, or plugin SDK surfaces.
