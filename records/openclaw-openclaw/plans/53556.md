Keep open: current main still handles the reported `allowall` and `topicSessionMode` compatibility cases, but legacy Feishu `botId` is still neither accepted by the schema nor read by credential/setup paths, so the concrete upgrade break remains actionable.

## Plan

Fix https://github.com/openclaw/openclaw/issues/53556 by adding a Feishu-owned backward-compatibility path for legacy `botId` config. Accept or normalize legacy top-level `channels.feishu.botId` and per-account `channels.feishu.accounts.<id>.botId` to canonical `appId`, keep current `appId` precedence when both are present, and expose an idempotent Feishu doctor `normalizeCompatibilityConfig` hook so `openclaw doctor --fix` migrates on-disk config instead of adding startup-time core migration behavior. Preserve the existing `topicSessionMode` and `groupPolicy: "allowall"` compatibility; do not broaden core channel migration policy or add new user-facing config knobs. Update focused Feishu tests, doctor migration tests, Feishu docs, and changelog if user-visible behavior changes.
