## Summary

Keep open. Current main still has the Feishu-specific activation gap: `/activation mention` stores session activation, but Feishu's pre-dispatch mention gate still decides admission from config-derived `requireMention` before the session activation state can hard-block unmentioned group messages.

## Plan

Fix https://github.com/openclaw/openclaw/issues/50490. In the Feishu plugin, make `/activation mention` actually affect group admission by resolving the effective target session's persisted `groupActivation` before the Feishu hard mention gate. Keep group admission, sender allowlists, broadcast dedupe, topic-session routing, and config-derived `requireMention` defaults intact. Add focused regression coverage for a Feishu group with `requireMention: false` or `groupPolicy: open`: after `/activation mention`, an unmentioned ordinary group message must not dispatch to the agent; `/activation always` should restore admission. Cover normal groups and, if the existing session-routing seams make it practical, broadcast/topic-scoped sessions. Update Feishu/groups docs only where needed to align the supported `/activation` behavior, and add a changelog entry because this is user-visible. Stop and report back if the implementation would require a new config option or a broader product decision.
