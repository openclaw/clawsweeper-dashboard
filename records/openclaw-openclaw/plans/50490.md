## Summary

Keep open. Current main still has the Feishu-specific split described in the issue discussion: `/activation mention` persists session activation state, but Feishu's pre-dispatch activation gate still derives `requireMention` from config only, so unmentioned group messages can be admitted when config allows them.

## Plan

Fix https://github.com/openclaw/openclaw/issues/50490. In Feishu group chats, after an authorized `/activation mention` command persists `sessionEntry.groupActivation = "mention"`, later unmentioned group messages must be blocked or recorded as pending history before agent dispatch even when config would otherwise allow unmentioned messages. Implement this inside the Feishu plugin by resolving the effective routed session early enough to apply persisted group activation as an override to config-derived `requireMention`, while preserving group admission, sender allowlists, broadcast behavior, topic/group-session scopes, and pending-history recording. Do not add a new config option and do not fold in unrelated `@all` parsing behavior from https://github.com/openclaw/openclaw/issues/77383. Add focused regression tests and update Feishu/groups docs plus `CHANGELOG.md` if the user-visible support surface changes.
