## Summary

Keep open. Current main and the latest release still have the reported Feishu gap: `/activation mention` persists group activation state, but Feishu group admission still decides whether to dispatch from config-derived `requireMention` before that session state can hard-stop unmentioned messages.

## Plan

Fix https://github.com/openclaw/openclaw/issues/50490. On current main, `/activation mention` persists `sessionEntry.groupActivation`, but Feishu group admission uses config-derived `requireMention` before session state can affect dispatch. Implement the fix in the owning Feishu plugin: resolve/read the target group or topic session before the hard sender activation gate, treat `groupActivation: "mention"` as effective mention-required and `groupActivation: "always"` as effective no-mention-required, and preserve existing group admission, sender allowlist, broadcast, topic-session, card-action callback, and config-default behavior for sessions without an override. Do not add a new config option or broad core policy. Add regression coverage for an unmentioned Feishu group message with `groupPolicy: "open"` or `requireMention: false` after `groupActivation: "mention"`, a mentioned message that still dispatches, and `groupActivation: "always"` restoring unmentioned dispatch. Update Feishu/groups docs and changelog if behavior/docs are changed.
