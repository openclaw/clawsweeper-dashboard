## Summary

Keep open. Current main still exposes terminal-looking session `status` values through persisted session entries, gateway session rows, and agent-facing `sessions_list`, while `sessions.send` can still target and reactivate existing sessions.

## Plan

Fix https://github.com/openclaw/openclaw/issues/64103 without renaming or removing the existing `status` values. Add backward-compatible metadata to the gateway session row and agent-facing `sessions_list` output so agents can tell that `status` is a last-run/turn state and whether the session can still be resumed with `sessions_send`; update type contracts, row mapping, tool descriptions/system prompt guidance, docs, and focused regression tests. Do not add a config option, do not make a breaking protocol change, and stop for maintainer input if the implementation would require replacing the legacy status union rather than adding metadata. Add a changelog entry because this changes user-visible agent/tool behavior.
