Keep open. Current main still returns terminal-looking `status` values to agent-facing session list consumers without explicit resumability or status-scope metadata, while `sessions.send` can reactivate completed subagent sessions.

## Plan

Fix https://github.com/openclaw/openclaw/issues/64103 by making session resumability and status scope explicit to agents without renaming or removing the legacy `status` values.
