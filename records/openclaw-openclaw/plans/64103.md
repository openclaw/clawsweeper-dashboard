# Coding Plan for openclaw/openclaw#64103: Session `status` field values (\"failed\", \"timeout\", \"done\") mislead agents into spawning duplicate sessions

Source report: [records/openclaw-openclaw/items/64103.md](records/openclaw-openclaw/items/64103.md)

## Summary

Keep open. Current main still exposes terminal-looking session statuses to agents without explicit resumability or status-scope metadata, while the same sessions can still be targeted by `sessions_send`, so the reported session-state confusion remains actionable.

## Plan

Fix https://github.com/openclaw/openclaw/issues/64103 by preserving the existing session `status` values for compatibility while adding explicit agent-visible metadata/guidance that tells agents whether a listed session is targetable/resumable and that `status` is a last-turn/run outcome, not a session lifecycle verdict. Update the gateway/session row types and builder, agent-facing `sessions_list` row handling, relevant tool descriptions/system prompt guidance, public docs, and focused tests. Do not do a breaking rename of `running`, `done`, `failed`, `killed`, or `timeout`; do not add a config option. Add a changelog entry because this changes user-visible agent/tool behavior.
