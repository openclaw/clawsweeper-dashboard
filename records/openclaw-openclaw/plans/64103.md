# Coding Plan for openclaw/openclaw#64103: Session `status` field values (\"failed\", \"timeout\", \"done\") mislead agents into spawning duplicate sessions

Source report: [records/openclaw-openclaw/items/64103.md](records/openclaw-openclaw/items/64103.md)

## Summary

Keep open. Current main still exposes terminal-looking session `status` values through persisted session entries, gateway session rows, and the agent-facing `sessions_list` output, while `sessions.send` can still target an existing session and reactivate completed sub-agent sessions.

## Plan

Fix the session status/resumability contract reported at https://github.com/openclaw/openclaw/issues/64103. Preserve the existing `running | done | failed | killed | timeout` `status` values for compatibility; do not implement the breaking rename. Add backward-compatible metadata that makes status scope and resumability explicit in persisted/gateway/agent-facing session rows, likely fields such as `resumable` and/or `statusScope`, and ensure `sessions_list` returns it to agents. Update the sessions tool descriptions, default prompt guidance, docs, focused regression tests, and `CHANGELOG.md`. Keep the fix scoped to session list/send semantics; do not change unrelated subagent scheduling, polling, compaction recovery, or provider behavior.
