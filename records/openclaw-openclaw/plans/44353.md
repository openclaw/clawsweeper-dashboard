## Summary

Keep open: current main has fixed the isolated-cron subagent fallback gap from the related closed report, but normal spawned subagent runs still persist only an auto model override and then resolve fallback candidates from regular agent/default model config, not from the selected `subagents.model.fallbacks`; the current Bedrock provider hook also does not visibly classify the exact inference-profile-required error text.

## Plan

Fix the existing runtime fallback behavior reported in https://github.com/openclaw/openclaw/issues/44353. Preserve `subagents.model.fallbacks` for normal `sessions_spawn` child runs when the child model was auto-selected from `agents.defaults.subagents.model` or `agents.list[].subagents.model`; explicit `sessions_spawn.model` user overrides must keep disabling automatic fallbacks, and explicit empty fallback arrays must keep disabling fallback. Also add or verify exact classification for the Bedrock on-demand-throughput / inference-profile-required provider error so it advances through configured model fallbacks. Coordinate with the already-merged isolated-cron fix from https://github.com/openclaw/openclaw/pull/82302, but do not change cron preflight behavior from https://github.com/openclaw/openclaw/issues/79329 and do not add a new config option.
