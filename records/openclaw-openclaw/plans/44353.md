## Summary

Keep open: current main has the generic fallback runner behavior needed to continue when fallback candidates exist, and the isolated cron subagent path was fixed separately, but normal spawned subagents still do not pass the selected `subagents.model.fallbacks` chain into the runner. The remaining bug is narrow fallback-policy routing for auto-selected spawned-subagent models.

## Plan

Fix the remaining spawned-subagent fallback-policy bug for https://github.com/openclaw/openclaw/issues/44353. Current main can fall back when candidates are supplied, but normal spawned subagents selected from `agents.defaults.subagents.model` or per-agent `subagents.model` only persist the selected model as an auto session override; later `agent-command` resolves fallback candidates from `agents.list[].model` or `agents.defaults.model`, not the selected `subagents.model.fallbacks`. Thread the selected subagent fallback policy into normal spawned-subagent execution so `runWithModelFallback` receives the same fallback chain that selected the subagent primary. Preserve strict behavior for explicit `sessions_spawn.model` and user `/model` overrides, preserve explicit empty `fallbacks: []`, and do not add a new config option. Add focused regression coverage for normal spawned subagents; update cron tests only if shared fallback helpers are touched. Add a user-facing changelog fix entry unless maintainers explicitly decide this is internal-only.
