# Coding Plan for openclaw/openclaw#11703: [Feature]: Batch WhatsApp messages by messageTimestamp instead of receive time

Source report: [records/openclaw-openclaw/items/11703.md](records/openclaw-openclaw/items/11703.md)

## Summary

Keep open. Current main still preserves WhatsApp `messageTimestamp` as inbound metadata, but batching remains keyed to account, conversation, sender, and receive-time debounce windows; there is no timestamp coalescing behavior or config/docs surface yet.

## Plan

Implement the request in https://github.com/openclaw/openclaw/issues/11703 as a narrow WhatsApp plugin change. Add timestamp-aware coalescing for text-only inbound WhatsApp messages from the same account, conversation, and sender when their WhatsApp server timestamps are identical or within a very small documented tolerance. Preserve current media, location, reply-context, access-control, self-message, control-command, dedupe, and append-history behavior. Prefer keeping the shared inbound debouncer contract unchanged; add only an additive helper/hook if the WhatsApp path cannot stay local. If a config knob is added, update WhatsApp schema/types/UI hints/docs/generated config docs and add a user-facing changelog entry. Do not broaden this into global channel debounce policy.
