## Summary

Keep open. Current main still unconditionally sends `stream_options.include_usage` for streaming OpenAI Chat Completions payloads even when model compat resolves `supportsUsageInStreaming: false`, so the Mindie-style strict-parameter 422 remains source-reproducible.

## Plan

Fix https://github.com/openclaw/openclaw/issues/45706 by making OpenAI Chat Completions streaming payloads respect `models.providers.<provider>.models[].compat.supportsUsageInStreaming: false`. In `buildOpenAICompletionsParams()`, omit `stream_options` for models whose resolved compat disables streaming usage, while keeping `stream_options: { include_usage: true }` for native/known compatible endpoints and explicit `supportsUsageInStreaming: true` custom endpoints. Do not hardcode Mindie provider ids, do not remove the existing strict message-key/string-content/tool-disable compat behavior, and do not regress the local usage-tracking scenario from https://github.com/openclaw/openclaw/pull/68746. Add focused regression tests for explicit false omission and existing positive paths; update docs/changelog only if public behavior or config guidance changes.
