# Coding Plan for openclaw/openclaw#79140: update-check hardcodes registry.npmjs.org, not respecting npm mirror config

Source report: [records/openclaw-openclaw/items/79140.md](records/openclaw-openclaw/items/79140.md)

## Summary

Keep open: current main and the latest release still hardcode the public npm registry for update metadata, so mirror-configured installs can still time out before the package-manager path runs.

## Plan

Fix https://github.com/openclaw/openclaw/issues/79140. Current update metadata helpers hardcode `https://registry.npmjs.org/openclaw/<target>` and ignore npm registry mirror configuration. Add a central registry-base resolver for OpenClaw npm metadata fetches, use it from `fetchNpmPackageTargetStatus`/tag/channel helpers, honor existing npm registry environment such as `npm_config_registry` and `NPM_CONFIG_REGISTRY` first, and support bounded npmrc-derived registry config if it can be done without an unbounded package-manager subprocess in Gateway startup. Normalize mirror URLs with the URL API, handle trailing slashes, reject unsafe/malformed values by falling back to `https://registry.npmjs.org/`, and avoid logging credentials. Do not add a new OpenClaw config option or change package-manager install arguments. Add focused regression tests in `src/infra/update-check.test.ts` for default registry, env mirror, uppercase env mirror, trailing slash normalization, and malformed fallback; update startup/update CLI tests only if the public helper signature changes. Add a user-facing `CHANGELOG.md` Fixes entry with `Thanks @sixerLiu`.
