## Plan

Implement the remaining work for https://github.com/openclaw/openclaw/issues/50186 as a bounded Linux systemd diagnostics change. Extend `readSystemdServiceRuntime` and its parser to request and carry `KillMode`, `TasksCurrent`, `MemoryCurrent`, and `MainPID` through `GatewayServiceRuntime`; surface those values in `openclaw status --deep` human/JSON output and `openclaw doctor` notes/warnings when conservative combinations indicate degraded service hygiene. Do not change the generated systemd unit default, add a migration, or implement process reaping in this PR; related sidecar lifecycle issue https://github.com/openclaw/openclaw/issues/84413 remains separate.
