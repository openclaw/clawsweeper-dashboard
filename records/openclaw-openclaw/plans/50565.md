## Summary

Keep open. Current main no longer has the old Feishu-specific `tool call` text, but the embedded tool lifecycle logs still inherit a static `agent/embedded` logger because the subscription layer is not given the originating channel identity.

## Plan

Fix the logging attribution bug reported in https://github.com/openclaw/openclaw/issues/50565. Thread a typed, normalized channel origin or logger from `runEmbeddedAttempt` into `subscribeEmbeddedPiSession`; tool lifecycle start/end debug logs should use `gateway/channels/<channel>` for real message-channel runs and retain `agent/embedded` for channel-less/internal runs. Do not add a config option, do not add channel-specific branches in tool handlers, and preserve sanitized tool logging so raw args, exec command text, and sensitive tool output are not exposed. Add focused regression coverage for subscriber logger selection and for the attempt-to-subscription handoff. Add a single-line `CHANGELOG.md` fix entry because this changes operator-visible diagnostics.
