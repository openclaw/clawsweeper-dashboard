## Summary

Keep open. Current main no longer has the old Feishu-specific `tool call` wording, but embedded tool start/end debug logs still inherit a static `agent/embedded` logger because the subscription path is not given the originating channel identity.

## Plan

Fix https://github.com/openclaw/openclaw/issues/50565 by carrying the originating message channel into embedded tool lifecycle logging. Add a typed field or logger seam from `runEmbeddedAttempt` / `buildEmbeddedSubscriptionParams` into `SubscribeEmbeddedPiSessionParams`, derive a safe `gateway/channels/<normalized-channel>` logger for valid message-channel runs, and keep `agent/embedded` for channel-less/internal runs. Cover tool start and tool end attribution in focused tests, and preserve existing tool argument/result redaction. Do not add channel-specific branches in the tool handlers, do not log raw exec commands or secret-shaped tool data, and add a `CHANGELOG.md` fix entry for the diagnostic logging behavior.
