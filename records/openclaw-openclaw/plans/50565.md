# Coding Plan for openclaw/openclaw#50565: Tool call logs always tagged under feishu subsystem regardless of actual channel

Source report: [records/openclaw-openclaw/items/50565.md](records/openclaw-openclaw/items/50565.md)

## Summary

Keep open. The old Feishu-specific `tool call` text is gone, but current `main` and the latest release still attribute embedded tool lifecycle debug logs to a static `agent/embedded` logger because the originating channel is not passed into the subscription layer.

## Plan

Fix the tool lifecycle logging attribution bug reported in https://github.com/openclaw/openclaw/issues/50565. Current main still creates a static `agent/embedded` logger in `subscribeEmbeddedPiSession`, while `runEmbeddedAttempt` has `messageChannel`/`messageProvider` but does not pass channel origin into `buildEmbeddedSubscriptionParams`. Add a typed, normalized channel-origin or logger seam from the runner into subscription, derive `gateway/channels/<channel>` only for valid message-channel runs, and keep `agent/embedded` as the fallback for channel-less runs. Cover tool start/end attribution in focused tests and preserve existing redaction/sanitization behavior; do not add channel-specific branches or log raw tool args, exec command text, or tool output. Add a changelog fix entry if maintainers treat diagnostic log attribution as user-visible.
