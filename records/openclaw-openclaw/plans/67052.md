## Summary

Keep open. Current `main` and latest shipped TUI still use watchdog and stale-terminal recovery mitigations, but they do not provide the requested prompt content-complete or finalizing transition after visible assistant content stops.

## Plan

Fix the TUI streaming indicator timing for https://github.com/openclaw/openclaw/issues/67052. Current main sets `activityStatus` to `streaming` on chat deltas and clears it only on chat `final`/terminal handling or the 30s watchdog; lifecycle final can be delayed by post-response flush work. Implement a narrow backward-compatible content-complete or finalizing transition across the gateway/TUI boundary, or an equivalently safe TUI state-machine path, so visible assistant content completion stops presenting as active streaming promptly while the run may still be finalizing. Preserve the existing watchdog as recovery. Do not make a broad idle shortcut that breaks concurrent active runs, local /btw, reconnect recovery, hidden/thinking output, tool-only output, or late final deltas. Update typed protocol/TUI types if adding a wire state, add focused regression tests, and add a user-facing changelog fix entry.
