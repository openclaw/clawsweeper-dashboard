# Coding Plan for openclaw/openclaw#70928: [Bug]: `message_sent` / `before_message_write` hooks never fire for webchat and openclaw-tui outbound paths

## Summary

Keep open. Current main still routes WebChat and Gateway-backed TUI through `chat.send`, where final UI delivery and Gateway-owned assistant transcript appends bypass the existing outbound/write hook seams.

## Plan

Fix https://github.com/openclaw/openclaw/issues/70928 on current main. WebChat and Gateway-backed TUI use `chat.send`, but `src/gateway/server-methods/chat.ts` creates a raw `createReplyDispatcher(...)`, calls `dispatchInboundMessage(...)`, and finalizes with `broadcastChatFinal(...)` without the hook-aware `message_sending`/`message_sent` seams; Gateway-owned assistant transcript appends also reach `appendSessionTranscriptMessage(...)` without `before_message_write`. Add a narrow Gateway chat lifecycle hook seam for WebChat/TUI final delivery and Gateway-owned assistant transcript appends, reusing existing hook runner/message hook contracts and preserving exactly-once behavior. Do not add new hook names, config options, or a broad outbound observer architecture; leave the umbrella architecture to https://github.com/openclaw/openclaw/issues/50126. Add focused regression tests for `chat.send`/WebChat final delivery and transcript append behavior, update docs only if the effective contract changes, and include a changelog entry for the user-visible bug fix.
