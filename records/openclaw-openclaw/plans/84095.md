# Coding Plan for openclaw/openclaw#84095: Bug: Feishu plugin tools (feishu_doc) not injected into agent tool list in DM sessions (v2026.5.12, fix #49496 claimed in 4.22 but still broken)

Source report: [records/openclaw-openclaw/items/84095.md](records/openclaw-openclaw/items/84095.md)

## Summary

Keep open: current main still gates Feishu plugin tools on top-level `channels.feishu.appId/appSecret`, while documented and runtime-supported Feishu multi-account configs can place credentials only under `channels.feishu.accounts.<id>`. The reporter’s follow-up confirms an account-scoped manifest hotpatch made the tools callable in Feishu DM sessions, and the linked packaging PR does not address this account-aware availability gap.

## Plan

Fix https://github.com/openclaw/openclaw/issues/84095 by making Feishu plugin tool availability account-aware. Preserve the plugin/core boundary and existing credential gating: Feishu tools should be available when `channels.feishu.appId/appSecret` are configured or when any enabled named account under `channels.feishu.accounts.<id>` has usable `appId/appSecret`, and should remain hidden when no usable credentials exist. Prefer a generic manifest availability extension for account maps or an equivalent plugin-owned metadata path; do not hard-code reporter account names, do not loosen all Feishu tools unconditionally, and do not change Feishu runtime credential resolution. Add focused regression coverage for plugin tool availability with account-scoped Feishu credentials and for the no-credentials hidden case. Keep the open packaging-guard PR separate unless maintainers explicitly ask to combine it. Add a changelog fix entry if the repair lane owns landing.
