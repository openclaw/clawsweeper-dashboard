Keep open: current main still has a source-backed Feishu account-scoped tool availability gap, and the latest release keeps the same top-level-only manifest metadata that can hide Feishu tools before session tool assembly.

## Plan

Fix https://github.com/openclaw/openclaw/issues/84095 by making Feishu plugin tool availability account-aware. Preserve the plugin/core boundary, do not expose Feishu tools when no usable credentials exist, keep tool deny/allow policy authoritative, and add regression coverage for top-level credentials, named-account-only credentials, and no-credential hiding.
