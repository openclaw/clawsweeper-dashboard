Keep this open. Current main and the latest release still support multi-account Slack Socket Mode but do not stagger initial per-account WebSocket startup, so the reported startup fanout remains only partially mitigated by longer ping timeout and reconnect backoff.

## Plan

Keep the fix inside the Slack plugin or the smallest generic channel lifecycle seam needed; do not add connection pooling, new config knobs, broad gateway startup policy changes, or unrelated Slack behavior.
