# Coding Plan for openclaw/openclaw#78269: [Feature] Feishu channel: implement reactionLevel config to auto-add emoji reactions

Source report: [records/openclaw-openclaw/items/78269.md](records/openclaw-openclaw/items/78269.md)

## Summary

Keep open: current main exposes Feishu manual `react`/`reactions` actions behind `actions.reactions`, but it does not define, document, or read a Feishu `reactionLevel` setting or add reaction-guidance prompt behavior for Feishu.

## Plan

Implement Feishu `reactionLevel` support for issue https://github.com/openclaw/openclaw/issues/78269. Keep the change Feishu-owned: add `reactionLevel: off | ack | minimal | extensive` to `extensions/feishu/src/config-schema.ts` for top-level and account config, add a small Feishu resolver using `resolveReactionLevel` from `openclaw/plugin-sdk/text-runtime`, wire `agentPrompt.reactionGuidance` in `extensions/feishu/src/channel.ts`, and gate agent `react`/`reactions` discovery/handling so `off`/`ack` disable agent-controlled reactions while `minimal`/`extensive` enable guidance. Preserve existing `actions.reactions` behavior as the hard action gate, do not move Feishu-specific behavior into core, and avoid solving unrelated Feishu reaction API bugs already tracked by #66406/#66478 and #34528/#77653. Add focused tests for schema acceptance, account override behavior, discovery/gating, and prompt guidance. Update Feishu user docs and add a user-facing changelog entry if behavior ships. If the implementation would require inventing new automatic ACK emoji semantics beyond the existing reaction-level helper, stop and ask for maintainer product input.
