## Summary

Keep open: current main and the latest tagged release still lack Feishu `reactionLevel` config, metadata, prompt guidance, and runtime gating; the related open Feishu reaction PRs address API reliability or typing-reaction behavior, not this policy/config feature.

## Plan

Implement Feishu `reactionLevel` parity for this issue. Add Feishu-owned config support for `off | ack | minimal | extensive` at top-level and account-level config, expose it in plugin metadata/docs, add a Feishu reaction-level resolver using the shared `openclaw/plugin-sdk/status-helpers` resolver, wire `agentPrompt.reactionGuidance` so `minimal` and `extensive` guide agent reactions, and gate Feishu `react`/`reactions` action availability so `off` and `ack` disable agent-controlled reactions while preserving `actions.reactions` as the hard action gate. Define `ack` carefully around existing `typingIndicator` behavior so the bot does not duplicate typing reactions. Coordinate with the related Feishu reaction API PRs but do not fold their unrelated fixes into this feature PR. Add focused tests for schema parsing, account override behavior, message-tool discovery/prompt guidance, disabled action behavior, and docs/changelog updates for the user-facing config.
