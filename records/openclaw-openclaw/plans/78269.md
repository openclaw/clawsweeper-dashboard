# Coding Plan for openclaw/openclaw#78269: [Feature] Feishu channel: implement reactionLevel config to auto-add emoji reactions

Source report: [records/openclaw-openclaw/items/78269.md](records/openclaw-openclaw/items/78269.md)

## Summary

Keep open: current main still does not expose or consume Feishu `reactionLevel`, and the related Feishu reaction threads are API reliability fixes rather than this requested auto-reaction policy.

## Plan

Implement `channels.feishu.reactionLevel` for the bundled Feishu plugin. Add top-level and per-account schema support plus plugin metadata for `off | ack | minimal | extensive`; add a Feishu reaction-level resolver that reuses `resolveReactionLevel`; wire `feishuPlugin.agentPrompt.reactionGuidance` so `minimal` and `extensive` reach the system prompt; keep `actions.reactions` as the explicit tool gate and make `reactionLevel` disable agent-initiated reactions for `off`/`ack` consistently with the existing channel precedent. Do not change core plugin architecture, do not solve unrelated Feishu reaction API issues except to interoperate with current helpers, and do not couple `reactionLevel=ack` to the existing Feishu `typingIndicator` behavior unless existing tests/docs already require it. Update focused tests, `docs/channels/feishu.md`, `docs/tools/reactions.md`, and add a changelog entry because this is user-visible config behavior. Coordinate with the still-open Feishu reaction API fixes at https://github.com/openclaw/openclaw/pull/66478 and https://github.com/openclaw/openclaw/pull/77653 if they have landed before implementation.
