# Coding Plan for openclaw/openclaw#81232: [Feature] Discord: fetch a single message by channelId + messageId (or URL)

Source report: [records/openclaw-openclaw/items/81232.md](records/openclaw-openclaw/items/81232.md)

## Summary

Keep open. Current main has an internal Discord `fetchMessage` runtime path, but the public `message` tool still does not expose or map a `fetch` action, so the requested single-message-by-ID-or-URL surface remains missing.

## Plan

Implement Discord `message` tool support for fetching one message. Add a public `action: "fetch"` to the shared message-action contract and Discord discovery when the existing `channels.discord.actions.messages` gate is enabled. In `extensions/discord/src/actions/handle-action.ts`, map `action: "fetch"` to the existing lower-level `fetchMessage` runtime, accepting `channelId` + `messageId` and a Discord message URL via `url` or `messageLink`; preserve current account/gate/media option plumbing and use existing normalization so the return shape is compatible with `read` consumers. Reuse `fetchMessageDiscord` / `getChannelMessage`; do not implement bulk fetch, cross-guild access, or the granular action-gate redesign tracked separately. Update focused tests for discovery, adapter mapping, runtime URL parsing, and docs; add a concise changelog `### Changes` entry for the user-facing action if the landing workflow expects one.
