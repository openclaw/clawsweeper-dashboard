## Summary

This PR is still needed because current main and the latest release still choose `--before` from npm's computed `before` value in hosted installers, but the submitted fix has one remaining compatibility gap: the new raw-key probe misses npm's resolved default `userconfig` path when `HOME` is unset.

## Plan

Fix PR https://github.com/openclaw/openclaw/pull/84749 without changing the broader installer freshness policy. Add effective userconfig discovery to `npm_config_has_raw_key` in `scripts/install.sh` and `scripts/install-cli.sh` by querying `npm config get userconfig --global` with freshness env vars unset, then resolve and scan that path before choosing `--before`. Add the analogous `Invoke-NpmCommand -Arguments @("config", "get", "userconfig", "--global")` probe in `scripts/install.ps1`. Preserve the current exclusion of project `.npmrc` for global installs and keep globalconfig probing. Add focused HOME-unset regression coverage for the shell installer fixtures and PowerShell static tests, then run the existing focused installer validation. Do not alter package selection, managed TypeScript helper runtime behavior, or unrelated installer flows.
