# Coding Plan for openclaw/openclaw#60509: Feature Request: Expose user message in agent:bootstrap hook context for query-conditioned context compression

Source report: [records/openclaw-openclaw/items/60509.md](records/openclaw-openclaw/items/60509.md)

## Summary

Keep open. Current main still does not expose the triggering prompt/user message to `agent:bootstrap`; prompt-aware plugin hooks and context-engine assembly exist, but they run through different surfaces and cannot let a bootstrap hook query-condition `bootstrapFiles` before injection.

## Plan

Implement a narrow backward-compatible API extension for https://github.com/openclaw/openclaw/issues/60509. Add an optional current user message field to `AgentBootstrapHookContext` (prefer the issue's `userMessage?: string` unless surrounding naming strongly argues otherwise), thread it through `applyBootstrapHookOverrides`, `resolveBootstrapFilesForRun`, and `resolveBootstrapContextForRun`, and pass the current prompt from embedded and CLI user-triggered run paths where available. Leave it undefined for raw model runs, heartbeat/background paths where the value is not a user prompt, and any caller that lacks a prompt. Do not replace the existing context-engine or `before_prompt_build` hooks, and do not add a config option. Add focused regression tests proving the field is propagated, remains undefined by default/unavailable callers, and does not break existing mutable `bootstrapFiles` behavior. Update `docs/automation/hooks.md` and add a changelog entry because this changes a user-facing hook contract. Related prior closed source attempts are https://github.com/openclaw/openclaw/pull/9677 and https://github.com/openclaw/openclaw/pull/15081; use them only as context, not as a branch to merge blindly.
