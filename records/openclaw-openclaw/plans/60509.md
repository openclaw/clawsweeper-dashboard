## Summary

Keep open. Current main still does not expose the triggering user prompt to `agent:bootstrap`; prompt-aware hooks and context-engine assembly exist, but they run through later or different surfaces that cannot mutate `bootstrapFiles` before Project Context injection.

## Plan

Implement the feature requested by https://github.com/openclaw/openclaw/issues/60509. Add a backward-compatible optional current user prompt/message field to the `agent:bootstrap` hook context, thread it from embedded and CLI run paths only where the current prompt is already available, and leave it undefined for heartbeat, cron, compaction, raw model, startup, or other runs without a real user prompt. Keep the change limited to the bootstrap hook contract and call sites; do not replace context-engine assembly or broaden `before_prompt_build`. Add regression coverage proving hooks can read the field and mutate `bootstrapFiles` based on it, and that unavailable-run paths omit it safely. Update public hooks docs and add a concise changelog entry if this user-facing hook API change requires one.
