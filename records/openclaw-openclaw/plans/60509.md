# Coding Plan for openclaw/openclaw#60509: Feature Request: Expose user message in agent:bootstrap hook context for query-conditioned context compression

Source report: [records/openclaw-openclaw/items/60509.md](records/openclaw-openclaw/items/60509.md)

## Summary

Keep open. Current main still does not expose the triggering user prompt to `agent:bootstrap`; the prompt-aware surfaces run later or through context-engine assembly, so they cannot mutate `bootstrapFiles` before Project Context injection.

## Plan

Implement https://github.com/openclaw/openclaw/issues/60509 as a backward-compatible hook API addition. Add an optional current user prompt/message field to `AgentBootstrapHookContext`, pass it through `applyBootstrapHookOverrides`, `resolveBootstrapFilesForRun`, and `resolveBootstrapContextForRun`, and thread it from CLI and embedded-runner paths only where the run prompt is already available. Leave the field undefined for heartbeat, cron, raw model, startup, and other non-user-prompt runs. Add regression tests that a bootstrap hook can read the prompt and mutate `bootstrapFiles`, and that promptless/lightweight runs remain safe. Update `docs/automation/hooks.md` and any exported hook type docs; add a changelog entry because this is a user-visible hook API change. Do not move query-conditioned compression into core, add a config option, or change `before_prompt_build` semantics.
