## Summary

Keep open. Current main and the latest release still do not expose the triggering user prompt on `agent:bootstrap`, while the prompt-aware hook/context-engine paths are later surfaces that cannot mutate `bootstrapFiles` before Project Context injection.

## Plan

Implement a focused feature for https://github.com/openclaw/openclaw/issues/60509. Add one backward-compatible optional field to `AgentBootstrapHookContext` for the triggering user message/query, thread it through `applyBootstrapHookOverrides`, `resolveBootstrapFilesForRun`, and `resolveBootstrapContextForRun`, and populate it from run paths where `params.prompt` is already available. Leave it undefined for heartbeats, raw model/provider probes, gateway startup, and other non-user runs. Do not add a config option or vendor SMELT/core compression logic. Preserve existing `before_prompt_build` and context-engine behavior. Add/update regression tests for bootstrap hook propagation, bootstrap resolver propagation, embedded-runner context injection, and CLI prepare where applicable. Update `docs/automation/hooks.md` and add a concise changelog entry.
