## Plan

Fix https://github.com/openclaw/openclaw/issues/84697 on current main. The observable bug is that an OpenAI-compatible custom provider configured with a baseUrl missing its required path prefix can return HTTP 200 text/html, onboarding treats the probe as valid, and runtime may surface only an incomplete terminal response. Keep the fix bounded: validate OpenAI-compatible custom-provider verification responses by content type and/or parsed Chat Completions JSON for stream:false probes; reject text/html or other non-JSON success with a clear hint that many OpenAI-compatible APIs require baseUrl to include /v1. Do not auto-append /v1 or rewrite stored baseUrl because valid custom prefixes differ.
