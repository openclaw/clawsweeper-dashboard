# Coding Plan for openclaw/openclaw#79030: Regression: private model-provider requests still use env proxy when NO_PROXY is CIDR/IP-wildcard only

Source report: [records/openclaw-openclaw/items/79030.md](records/openclaw-openclaw/items/79030.md)

## Summary

Keep open: current main still has a source-provable path where private model-provider fetches can select trusted env-proxy mode, while OpenClaw's NO_PROXY matcher intentionally mirrors Undici exact-host/suffix semantics and does not match CIDR or IPv4 wildcard entries.

## Plan

Fix https://github.com/openclaw/openclaw/issues/79030. Current main lets model-provider fetches without an explicit dispatcher policy enter trusted env-proxy mode when `shouldUseEnvHttpProxyForUrl(url)` is true, but `matchesNoProxy` does not match CIDR entries such as `100.64.0.0/10` or narrowly scoped IPv4 wildcard entries such as `100.64.*`. Add target-aware CIDR and canonical IPv4 wildcard handling to the OpenClaw env-proxy bypass decision, preferably reusing `src/shared/net/ip.ts` helpers such as `isIpInCidr`. Preserve existing Undici-compatible exact host, suffix, optional port, wildcard-dot domain, and bare `*` behavior. Do not change managed proxy mode, explicit proxy policy, or default all `allowPrivateNetwork` providers to direct transport without maintainer approval. Add regression tests for `100.64.0.0/10`, `100.64.*`, exact host and host:port, nonmatching ranges, public host nonmatches, and provider guarded-fetch no-dispatcher behavior; add a single-line user-facing changelog fix entry.
