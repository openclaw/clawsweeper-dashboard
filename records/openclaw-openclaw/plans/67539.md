## Plan

Implement provider-specific TTS prompt hints for https://github.com/openclaw/openclaw/issues/67539. Add a backwards-compatible speech-provider hook that can return a short runtime prompt hint with resolved provider config, model/voice/target, and model override policy context. Have speech-core append only the active provider's hint when TTS is enabled, while preserving the existing generic hint. Start with a Google/Gemini hint for square-bracket audio tags; do not hard-code Google wording in speech-core, do not advertise inactive providers, and do not bypass modelOverrides policy. Update public SDK/types/API baseline and relevant plugin docs if the seam is exported. Add focused tests for active-provider-only injection and disabled/off-policy controls.
