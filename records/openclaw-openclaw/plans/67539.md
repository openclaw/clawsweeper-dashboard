## Summary

Keep open: current main still has only generic TTS prompt guidance, the speech-provider contract has no provider-owned prompt-hint hook, and the related open auto-emotion PR addresses provider mapping rather than this active-provider runtime prompt seam.

## Plan

Implement provider-specific TTS prompt hints for https://github.com/openclaw/openclaw/issues/67539. Add an optional backwards-compatible SpeechProviderPlugin hook for short runtime prompt guidance with resolved provider config/model/voice/target/modelOverrides context; have speech-core append only the selected active provider hint from buildTtsSystemPromptHint when TTS is enabled. Do not hard-code Gemini guidance in speech-core and do not advertise disabled/off-policy controls. Add Google/Gemini wording owned by extensions/google that mentions square-bracket cues inside spoken [[tts:text]] blocks. Update the Plugin SDK exports/types, contract/API baseline checks, relevant docs, and focused tests. Coordinate with, but do not implement, the separate autoEmotion work in https://github.com/openclaw/openclaw/pull/75043. Add a changelog entry if the project policy requires one for the user-visible prompt/runtime SDK change.
