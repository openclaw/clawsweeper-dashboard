## Summary

Keep open: current main has partial ACP `startTurn` failure handling, but legacy ACP done events and subagent `ok` finalization can still leave progress-only or undelivered completion work represented as successful durable task state.

## Plan

Fix https://github.com/openclaw/openclaw/issues/83863. Current main partially handles failed ACP startTurn results, but legacy ACP text_delta+done and subagent ok outcomes with missing or failed required final delivery can still leave durable task state looking successful. Carry structured final-deliverable and completion-delivery state through ACP and subagent finalization; for parented/required-completion child tasks, map missing final output, progress-only terminal output, or exhausted required delivery to the existing blocked/failed terminal outcome instead of plain succeeded. Add focused regressions for legacy ACP progress-only done, subagent progress-only frozen result, delivery retry exhaustion, and valid final report success.
