## Summary

Keep open. Current main partially addresses this class by honoring structured ACP `startTurn` failures and by crediting already-delivered subagent mirrors, but legacy ACP `done` events and subagent `ok` finalization can still make durable task state look successful while the required final artifact or completion delivery is missing.

## Plan

Fix the false-success completion contract reported at https://github.com/openclaw/openclaw/issues/83863. Current main partially handles structured ACP startTurn failures and requester delivery mirrors, but legacy ACP `done` events and subagent `ok` finalization can still record progress-only output or failed required completion delivery as succeeded. Keep the fix bounded: carry structured final-deliverable/required-delivery state through ACP and subagent task finalization, map missing final output or exhausted required completion delivery to the existing blocked/failed task/flow surface, preserve legitimate final text-only completions as succeeded, and avoid adding a user-facing config option. Do not solve the full broad async-delivery architecture tracked by https://github.com/openclaw/openclaw/issues/44925 in this PR. Add or update regression coverage for progress-only ACP, legacy done-without-final, subagent completion delivery retry exhaustion, and valid final output; add a changelog entry because this is user-visible task/delivery behavior.
