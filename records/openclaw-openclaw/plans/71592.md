## Summary

Keep open: current main still advertises shared `/status` and `/compact` commands in TUI slash completion, while local embedded TUI command handling still falls through to `sendMessage(raw)` and then into the embedded agent path. The related open PRs cover `/model` parsing and autocomplete-only behavior, not this local embedded command execution bug.

## Plan

Fix https://github.com/openclaw/openclaw/issues/71592. Add a local embedded TUI command boundary in `src/tui/tui-command-handlers.ts` or a shared command adapter so known advertised shared commands either execute supported shared behavior or render an explicit local-mode unsupported response before any embedded send occurs. Preserve gateway-connected fallback for truly unknown slash commands. Add regression coverage for local `/status` and `/compact` not calling `sendChat`, and keep existing gateway-mode `/status` behavior covered. Add a `CHANGELOG.md` fix entry for the user-visible TUI behavior change.
