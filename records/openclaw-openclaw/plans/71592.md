## Summary

Current main and the latest release still have the reported local TUI fallthrough: slash submissions reach the TUI command switch, `/status` and `/compact` are advertised through shared-command help/autocomplete, and unhandled command names still call `sendMessage(raw)`, which the embedded backend passes to the agent ingress path. The issue should stay open because the two directly linked fix PRs are closed unmerged and no current-main commit contains their central fix.

## Plan

Fix https://github.com/openclaw/openclaw/issues/71592. In `openclaw tui --local`, advertised shared slash commands such as `/status` and `/compact` must not be sent to the local embedded model as ordinary user text. Keep the boundary in `src/tui/**`: align local-mode `getSlashCommands()` and `helpText()` with commands that local mode can execute, and make `createCommandHandlers()` reject known-but-unsupported shared commands with an explicit local system response instead of calling `sendMessage(raw)`. Preserve gateway-mode behavior, local `/goal`, `/stop`, `/btw`, `/side`, model/session/settings commands, and unknown slash text behavior unless focused tests prove a safer narrow adjustment. Add regression coverage for `/status` and `/compact` not calling `sendChat`/`agentCommandFromIngress` in local mode, and extend the local PTY smoke when feasible. Do not change provider routing, config, migrations, plugin APIs, or docs unless directly required. No `CHANGELOG.md` edit for the repair PR; put release-note context in the PR body.
