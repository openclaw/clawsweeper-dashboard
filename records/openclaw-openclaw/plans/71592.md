Current `main` advertises shared `/status` and `/compact` through `getSlashCommands()`, but `createCommandHandlers().handleCommand()` has no `status` or `compact` branch, so in `openclaw tui --local` both commands reach `sendMessage(raw)` and then `EmbeddedTuiBackend.sendChat()` / `agentCommandFromIngress()`. Add a narrow local-mode boundary so advertised shared commands either execute reusable shared behavior or render an explicit local-mode unsupported response before any embedded send occurs. Preserve gateway-connected recovery for commands that intentionally flow through shared command handling, and do not broaden this into a full command-system refactor. Add focused regression tests proving local `/status` and `/compact` do not call `sendChat` / `agentCommandFromIngress`, and update autocomplete/handler coverage if needed.
