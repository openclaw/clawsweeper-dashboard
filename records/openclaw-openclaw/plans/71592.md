## Summary

Keep open: current main still advertises `/status` and `/compact` in TUI completion/help, but `handleCommand()` has no local cases for either command and its default path sends the raw slash text as a user message. In `--local`, that send path enters `EmbeddedTuiBackend.sendChat()` and then `agentCommandFromIngress()`, so the reported local embedded fallthrough remains source-provable.

## Plan

Fix https://github.com/openclaw/openclaw/issues/71592 with a narrow TUI local-mode command boundary. In `openclaw tui --local`, advertised shared commands such as `/status` and `/compact` must not fall through to `sendMessage(raw)` / `EmbeddedTuiBackend.sendChat()` / `agentCommandFromIngress()` as ordinary user prompts. Prefer reusable shared command execution if an existing safe adapter fits; otherwise render an explicit local-mode unsupported response for commands that cannot yet run locally. Preserve gateway-connected behavior for unknown slash commands and do not remove shared command autocomplete broadly. Add focused regression coverage showing local `/status` and `/compact` do not call `sendChat` or the embedded ingress path. Add a user-visible `CHANGELOG.md` fix entry. Do not introduce a config option or broad command-router refactor unless the current API already supports it cleanly.
