# Coding Plan for openclaw/openclaw#80380: [Feature]: updating openclaw to use gemini-3.1-flash-lite instead of gemini-3.1-flash-lite-preview

Source report: [records/openclaw-openclaw/items/80380.md](records/openclaw-openclaw/items/80380.md)

## Summary

Keep open: current main and the latest shipped release still rewrite configured Gemini 3.1 Flash-Lite GA IDs to the preview ID, while Google's Gemini API release notes say `gemini-3.1-flash-lite` is GA and `gemini-3.1-flash-lite-preview` is shutting down on May 25, 2026. ([ai.google.dev](https://ai.google.dev/gemini-api/docs/changelog?hl=en))

## Plan

Fix the Google/Gemini Flash-Lite model lifecycle bug reported at https://github.com/openclaw/openclaw/issues/80380 and related https://github.com/openclaw/openclaw/issues/81556. Google's release notes say `gemini-3.1-flash-lite` is GA and `gemini-3.1-flash-lite-preview` is shutting down, so OpenClaw should no longer rewrite the GA ID to the preview ID. Update the Google plugin manifest/helper, shared SDK normalizer, provider model templates/fallbacks, default aliases, and user-facing docs/tests so explicit `google/gemini-3.1-flash-lite`, bare `gemini-3.1-flash-lite`, and the `gemini-flash-lite` alias resolve to the GA ID where appropriate. Preserve intentional compatibility for unrelated Gemini 3 Pro/Flash preview aliases and older retired IDs; do not add a new config option or move Google-owned policy into core. Add or update focused regression coverage and add a single `CHANGELOG.md` Fixes entry if this lands as a user-facing fix.
