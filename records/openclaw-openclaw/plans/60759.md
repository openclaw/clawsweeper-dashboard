## Summary

Keep open: current main and the latest shipped release still nest plugin `$defs` under plugin config schemas while refs remain document-root `#/$defs/...`, so editor schema validation can still break. The target-repo fix PRs I found are closed unmerged, and the only open linked PR is in a fork rather than a mergeable target-repo PR.

## Plan

Fix https://github.com/openclaw/openclaw/issues/60759 in the target repo. Current `buildConfigSchema` nests plugin/channel config schemas under `plugins.entries.<id>.config` or `channels.<id>` without hoisting `$defs` or rewriting `#/$defs/...` refs, so plugin manifests such as qqbot, webhooks, and Feishu produce unresolved editor refs. Implement a generic, collision-safe JSON Schema composition fix in `src/config/schema.ts`: hoist or namespace local plugin/channel `$defs` at the root and rewrite local refs consistently, including nested refs inside definitions. Do not solve this by hand-inlining only individual plugin manifests. Add regression tests in `src/config/schema.test.ts` for both plugin and channel schemas, nested `$ref` inside `$defs`, and conflicting definition names. Include a changelog entry if the fix is user-visible. Preserve existing plugin manifest ownership and existing config schema lookup behavior outside the `$defs` repair.
