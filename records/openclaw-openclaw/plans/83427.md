## Summary

## Plan

In current main, embedded CLI gap-fill dedupes against `readTailAssistantTextFromSessionTranscript`, but that reader returns undefined when the physical transcript tail is a `type: "custom"` `openclaw.cache-ttl` entry, so it can append a duplicate CLI assistant mirror after the canonical embedded assistant turn. Patch the narrow transcript/gap-fill boundary so cache-TTL and other metadata-only custom rows do not block dedupe against the immediately preceding assistant, while preserving the existing behavior that a real user/tool tail starts a new turn and preserving real CLI backend transcript persistence. Add regression coverage for assistant -> cache-ttl custom -> embedded gap-fill with identical text, and keep the existing repeated-reply-after-user-tail test passing. Do not globally filter all `api: "cli"` transcript rows and do not change cache-TTL append semantics unless the focused fix proves that is necessary.
