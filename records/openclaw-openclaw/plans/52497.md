## Summary

Keep open. Current main and the latest release still have a source-proven path where `sessions_send` can return a timeout without the target run's lifecycle/model error, because `agent.wait` can observe a lifecycle error but let the wait deadline win before the deferred error snapshot is returned.

## Plan

Fix https://github.com/openclaw/openclaw/issues/52497. Current `sessions_send` already forwards `result.error` when `agent.wait` supplies it, but `agent.wait` can return a bare timeout after observing a lifecycle/model error because `waitForAgentJob` defers lifecycle errors through retry grace and the wait deadline can resolve `null` first. Keep the scope narrow: preserve bounded, sanitized pending lifecycle/model error context through `waitForAgentJob` or `agent.wait` so `sessions_send` returns the existing timeout `error` field. Do not implement a full `sessions_send` failure taxonomy, add config, or change protocol statuses unless a maintainer explicitly asks. Preserve current accepted-vs-terminal timeout and A2A delivery semantics. Add focused regression coverage for the short-wait lifecycle-error case and the `sessions_send` forwarding path. Add a single `CHANGELOG.md` Fixes entry because the result is user-visible.
