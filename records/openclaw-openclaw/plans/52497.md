## Summary

Keep open. Current main still has a source-proven path where a lifecycle/model error is observed during `agent.wait`, but `waitForAgentJob` holds it behind retry grace and the gateway can return a bare timeout before `sessions_send` has an error string to forward; the related soft-timeout work in https://github.com/openclaw/openclaw/issues/43095 does not cover this model-error reason gap.

## Plan

Fix https://github.com/openclaw/openclaw/issues/52497 narrowly. Preserve the existing `sessions_send` behavior that forwards timeout-side `error` when `agent.wait` provides one, and update the gateway wait path so a lifecycle/model error observed during the caller wait is not collapsed to a bare timeout solely because retry-grace outlives the caller timeout. Keep the complete failure taxonomy out of scope. Bound and sanitize any error text before crossing session boundaries; do not expose credentials, prompts, private endpoints, or raw provider payloads. Add focused regression coverage for `waitForAgentJob`/`agent.wait` when a lifecycle error arrives before a short wait expires, plus `sessions_send` coverage proving the caller receives the existing `error` field for terminal model/lifecycle timeout context while non-terminal soft wait timeouts still return accepted/pending. Do not change `timeoutSeconds: 0` fire-and-forget semantics, target run timeout policy, or unrelated A2A delivery behavior.
