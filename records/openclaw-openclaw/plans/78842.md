# Coding Plan for openclaw/openclaw#78842: [Feature]: Feishu plugin HTTP requests ignore proxy environment variables (fetchWithSsrFGuard defaults to strict mode)

Source report: [records/openclaw-openclaw/items/78842.md](records/openclaw-openclaw/items/78842.md)

## Summary

Keep open: current main still leaves the Feishu streaming-card and app-registration HTTP calls in default strict guarded-fetch mode, so raw `HTTP_PROXY`/`HTTPS_PROXY` environments still do not match the Feishu WebSocket proxy path. The exact fix attempt in https://github.com/openclaw/openclaw/pull/78849 was closed unmerged after the author could not provide live Feishu proxy proof.

## Plan

Implement the narrow Feishu guarded-fetch proxy parity requested by https://github.com/openclaw/openclaw/issues/78842. Add a Feishu-local helper or direct options so every fetchWithSsrFGuard call in extensions/feishu/src/streaming-card.ts and the fetchFeishuJson wrapper in extensions/feishu/src/app-registration.ts uses the existing trusted env-proxy mode for Feishu/Lark API hosts while preserving current hostname allowlists, audit contexts, release handling, timeouts/signals, and User-Agent behavior. Add focused tests that assert mode propagation for streaming-card token/create/update/note/close and app-registration requests; create an app-registration test if needed. Add a single-line changelog entry because this is user-visible Feishu proxy behavior. Do not change src/infra/net/fetch-guard.ts defaults, Feishu WebSocket proxy behavior, Lark SDK createFeishuClient Axios policy, package dependencies, or add a new config option; those are covered by separate Feishu proxy items such as https://github.com/openclaw/openclaw/issues/48949, https://github.com/openclaw/openclaw/issues/65799, and https://github.com/openclaw/openclaw/issues/72595. Validate with the targeted Feishu tests plus fetch-guard proxy tests, and note that live Feishu proxy-only proof is still desirable before landing.
