## Plan

Fix https://github.com/openclaw/openclaw/issues/76670. On current main, `mattermostSetupWizard.textInputs` defines the `httpUrl` prompt, but `src/channels/plugins/setup-wizard.ts` runs credential steps first and the default credential apply calls the Mattermost setup adapter with only `{ botToken }`; `extensions/mattermost/src/setup-core.ts` then rejects the incomplete input because it requires both `botToken`/`token` and `httpUrl`. Implement a narrow fix so interactive `openclaw setup --wizard` can collect the Mattermost bot token and base URL and persist both without throwing before the URL prompt. Add regression coverage that simulates entering a Mattermost token and then a base URL through the wizard and asserts both `botToken` and `baseUrl` are written. Add a single-line user-facing `CHANGELOG.md` fix entry if behavior changes.
