## Summary

Keep open: current main still has a source-visible heartbeat deferral gap for webchat agent runs that are accepted and tracked by Gateway before they enter the embedded session lane; the older busy-lane fix does not cover that active-run window.

## Plan

Teach `runHeartbeatOnce` to skip with `HEARTBEAT_SKIP_REQUESTS_IN_FLIGHT` when the resolved target session has an active Gateway chat/agent run before it is visible in the embedded session lane; prefer a canonical active-run/session-busy helper over channel-specific webchat filtering. Coordinate with the adjacent active-reply race in https://github.com/openclaw/openclaw/issues/64810 and the closed unmerged https://github.com/openclaw/openclaw/pull/64823, but do not replace this with broad non-blocking heartbeat architecture from https://github.com/openclaw/openclaw/issues/52046.
