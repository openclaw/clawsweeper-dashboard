# Coding Plan for openclaw/openclaw#51664: WebChat copy button doesn't work in Edge browser

Source report: [records/openclaw-openclaw/items/51664.md](records/openclaw-openclaw/items/51664.md)

## Summary

Keep open: current main and v2026.5.22 still route WebChat code-block Copy through `navigator.clipboard.writeText` with an empty rejection handler, so an Edge Clipboard API rejection remains a no-op. The direct fallback PRs were closed unmerged, and the open rendered-text PR addresses a different copy-payload bug.

## Plan

Fix WebChat code-block Copy in Edge by adding a shared Control UI clipboard helper that first tries `navigator.clipboard.writeText` and then falls back to a hidden textarea plus `document.execCommand("copy")` when the Clipboard API is missing or rejects. Wire `ui/src/ui/views/chat.ts` code-block Copy through the helper, consider reusing it from `ui/src/ui/chat/copy-as-markdown.ts` without changing message-level copy semantics, add focused Vitest coverage for rejected writes and fallback cleanup, and include a single-line `CHANGELOG.md` fix entry. Compose with https://github.com/openclaw/openclaw/pull/73633 if it lands first; do not solve the separate rendered-text payload issue here except to avoid conflicts.
