# Coding Plan for openclaw/openclaw#51664: WebChat copy button doesn't work in Edge browser

Source report: [records/openclaw-openclaw/items/51664.md](records/openclaw-openclaw/items/51664.md)

## Summary

Keep open: current main and v2026.5.20 still route WebChat code-block Copy through a bare `navigator.clipboard.writeText` call with an ignored rejection, so Edge/permission failures remain a no-op. The two direct fallback PRs were closed unmerged, and the open rendered-text PR is useful adjacent work but does not add a Clipboard API fallback.

## Plan

Fix https://github.com/openclaw/openclaw/issues/51664: WebChat code-block Copy should still copy when `navigator.clipboard.writeText` rejects or is unavailable in Edge/Windows-style browser contexts. Add a shared Control UI clipboard helper, likely `ui/src/ui/chat/clipboard.ts`, that returns a boolean, tries `navigator.clipboard.writeText(text)` first, then falls back to a temporary readonly offscreen textarea with `select()`, `setSelectionRange(0, value.length)`, `document.execCommand("copy")`, and unconditional cleanup in `finally`. Wire `ui/src/ui/views/chat.ts` code-block Copy through the helper and only show copied feedback on success; consider reusing the helper from `ui/src/ui/chat/copy-as-markdown.ts` so message-level copy gets the same fallback. Preserve or compose with the rendered `<pre><code>` textContent direction from https://github.com/openclaw/openclaw/pull/73633 if it lands first; do not change core/plugin architecture or add config. Add focused tests for Clipboard API success, rejected write fallback success, missing Clipboard API fallback success, fallback failure, and no DOM leaks; update the changelog because this is user-visible.
