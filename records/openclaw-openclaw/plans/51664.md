# Coding Plan for openclaw/openclaw#51664: WebChat copy button doesn't work in Edge browser

Source report: [records/openclaw-openclaw/items/51664.md](records/openclaw-openclaw/items/51664.md)

## Summary

Keep open: current `main` still renders WebChat code-block Copy buttons and handles them with a direct `navigator.clipboard.writeText` call plus an empty rejection handler, so Edge Clipboard API failures remain a no-op. The two linked fallback implementations, https://github.com/openclaw/openclaw/pull/51746 and https://github.com/openclaw/openclaw/pull/52010, were closed unmerged.

## Plan

Fix the WebChat code-block Copy button failure in Microsoft Edge. On current `main`, `ui/src/ui/views/chat.ts` calls `navigator.clipboard.writeText(code)` directly for `.code-block-copy` and swallows rejected writes, so Edge permission/secure-context failures are silent no-ops. Add a small Control UI clipboard helper, preferably under `ui/src/ui/chat/clipboard.ts` unless an existing local helper is a better fit, that tries Clipboard API first and falls back to a hidden textarea plus `document.execCommand('copy')` with cleanup in `finally`. Wire WebChat code-block Copy through it and consider reusing it from `ui/src/ui/chat/copy-as-markdown.ts` to avoid duplicated chat clipboard behavior. Add focused tests for success, rejected Clipboard API fallback, and total failure/no false copied state. Do not broaden this into unrelated clipboard surfaces outside chat, do not change markdown rendering semantics beyond the copy path, and add a user-facing changelog fix entry if the final patch is not test-only.
