# Coding Plan for openclaw/openclaw#61834: [Feature]: expose QMD no-rerank for memory.qmd query mode

Source report: [records/openclaw-openclaw/items/61834.md](records/openclaw-openclaw/items/61834.md)

## Summary

Keep open. Current main still lacks a QMD no-rerank config or request surface, while upstream QMD v2.1.0 explicitly added `qmd query --no-rerank` and `rerank: false` for the MCP query tool.

## Plan

Implement issue https://github.com/openclaw/openclaw/issues/61834 as a narrow config-level QMD no-rerank toggle. Add `memory.qmd.noRerank?: boolean` to the QMD config type, strict schema, labels/help/generated config docs, and resolved config. When `searchMode` is `query` and `noRerank` is true, append `--no-rerank` in the direct QMD CLI query path and pass `rerank: false` in the unified mcporter/MCP `qmd.query` payload. Ignore the toggle for `search` and `vsearch`. Do not add a request-level `memory_search.rerank` parameter in this PR. Add focused coverage for config resolution/schema/docs help and both direct CLI and MCP query builders. Update QMD docs and add a changelog entry because this is user-visible config behavior.
