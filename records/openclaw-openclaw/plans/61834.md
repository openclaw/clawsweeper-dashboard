# Coding Plan for openclaw/openclaw#61834: [Feature]: expose QMD no-rerank for memory.qmd query mode

Source report: [records/openclaw-openclaw/items/61834.md](records/openclaw-openclaw/items/61834.md)

## Summary

Keep open. Current main still lacks a QMD no-rerank config or request surface, and the existing direct CLI and mcporter/MCP query paths do not pass QMD's upstream no-rerank controls.

## Plan

Implement a config-level QMD no-rerank toggle for https://github.com/openclaw/openclaw/issues/61834. Add `memory.qmd.noRerank?: boolean` to the exported memory config types, strict zod schema, config labels/help/docs, and resolved QMD config. When `searchMode` is `query` and `noRerank` is true, append `--no-rerank` to the direct `qmd query` CLI args and include `rerank: false` in the unified QMD MCP `query` tool arguments. Ignore the option for `search` and `vsearch`; do not change the default `searchMode`, QMD fallback behavior, collection scoping, mcporter version fallback, or add a request-level `memory_search.rerank` parameter in this PR. Add focused regression tests for config resolution/schema and both direct CLI and mcporter query argument construction. Update QMD memory docs, generated config help/reference docs, and add a single changelog entry because this is user-visible.
