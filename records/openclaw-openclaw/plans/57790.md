# Coding Plan for openclaw/openclaw#57790: Gateway shutdown/restart does not fire `session_end` for active sessions

Source report: [records/openclaw-openclaw/items/57790.md](records/openclaw-openclaw/items/57790.md)

## Summary

Keep open. Current main still treats gateway shutdown/restart as gateway lifecycle only; the signal close path does not enumerate active sessions or call `runSessionEnd`, and the related umbrella issue was already closed while leaving this narrower shutdown hook gap as the remaining canonical item.

## Plan

Fix https://github.com/openclaw/openclaw/issues/57790. Current main emits `session_end` for reset/delete/new-session/compaction paths, but Gateway stop/restart only emits gateway lifecycle hooks and closes services. Add a bounded gateway-owned finalization path that snapshots active sessions and emits plugin `session_end` for each active session during SIGTERM/SIGINT/SIGUSR1/restart shutdown, with backward-compatible typed reasons such as `shutdown` and `restart`. Keep `gateway_stop` as the plugin service lifecycle hook; do not ask plugins to infer per-session endings from it. Avoid double-firing sessions already finalized by reset/delete/compaction/replacement or by work drained during restart. Update hook types/docs and add focused regression coverage for server close/run-loop behavior plus existing reset/delete coverage. Add a changelog entry because this changes user-visible plugin hook behavior.
