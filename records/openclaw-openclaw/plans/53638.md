## Summary

Keep open. Current main and the latest release have the shared `channels.modelByChannel` surface for channel/group overrides and wildcard direct-chat defaults, but they still do not support stable per-DM target keys for ordinary direct conversations.

## Plan

Fix persistent per-DM model routing through the existing `channels.modelByChannel` surface. Add stable direct-conversation candidate inputs to `resolveChannelModelOverride` and its reply/status/harness callers without adding provider-specific nested `channels.<provider>.*.model` fields. Derive and normalize direct peer/session candidates from finalized context, session entry/origin data, native target fields, and direct session keys while preserving existing precedence: session `/model` override beats channel config, specific group/thread/direct keys beat wildcard, and channel/group behavior must not regress. Document accepted DM key forms in `docs/gateway/config-channels.md`. Add focused tests for the generic resolver plus representative Slack, WhatsApp, Discord, and Telegram direct contexts where feasible. Do not edit `CHANGELOG.md`; capture release-note context in the PR body if user-visible behavior changes.
