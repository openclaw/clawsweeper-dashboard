# Coding Plan for openclaw/openclaw#53638: Feature: per-channel / per-group / per-DM model override in config

Source report: [records/openclaw-openclaw/items/53638.md](records/openclaw-openclaw/items/53638.md)

## Summary

Keep open. Current main and the latest release cover channel/group-style model pins through `channels.modelByChannel`, but ordinary direct-message matching and the public schema/docs surface for the shared key are still incomplete.

## Plan

Implement the remaining per-DM portion of https://github.com/openclaw/openclaw/issues/53638 using the existing shared `channels.modelByChannel` surface. Do not add provider-specific nested `model` fields under Slack or WhatsApp. Extend `src/channels/model-overrides.ts` so direct conversations can build normalized candidate keys from trusted finalized context/session data such as current session key, `NativeDirectUserId`, `NativeChannelId`, `OriginatingTo`, `To`, `From`, sender ids, and parent/session refs while preserving precedence: explicit/session override > per-target `modelByChannel` > wildcard/channel/default model. Wire the extra inputs from normal reply and dispatch paths, expose the kernel-owned `channels.modelByChannel` key in the public config schema without dumping all channel plugin schemas, document accepted DM key forms in `docs/gateway/config-channels.md`, and add focused tests for resolver behavior, reply/dispatch wiring, and config schema output. Add a changelog entry because this changes user-visible config behavior.
