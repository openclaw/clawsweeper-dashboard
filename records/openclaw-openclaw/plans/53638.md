## Summary

Keep open. Current main and v2026.6.6 cover the generic channel/group/wildcard surface through `channels.modelByChannel`, but target-specific direct-message keys still lack resolver inputs and documented key forms.

## Plan

Implement the remaining direct-conversation support for persistent model overrides tracked by https://github.com/openclaw/openclaw/issues/53638. Use the existing `channels.modelByChannel` surface; do not add provider-specific `channels.whatsapp.groups.*.model`, `channels.whatsapp.dms.*.model`, or `channels.slack.channels.*.model` fields. Extend `src/channels/model-overrides.ts` and its callers so ordinary DMs can produce stable normalized candidate keys from finalized context, typed session/origin metadata, native direct peer identifiers, and direct session keys while preserving precedence: explicit run override and stored session override stay ahead of channel config, and channel/group/thread behavior must not regress. Document accepted DM key forms in `docs/gateway/config-channels.md`, add focused resolver/reply/agent regression coverage, and stop for maintainer review if the implementation needs a new config surface or product choice about provider-specific identifier policy.
