# Coding Plan for openclaw/openclaw#53638: Feature: per-channel / per-group / per-DM model override in config

Source report: [records/openclaw-openclaw/items/53638.md](records/openclaw-openclaw/items/53638.md)

## Summary

Keep open. Current main and the latest release have the shared `channels.modelByChannel` schema, docs, channel/group matching, and direct-chat wildcard fallback, but target-specific DM keys still cannot match because the resolver does not receive or parse ordinary direct peer ids.

## Plan

Fix persistent per-DM model overrides for the existing `channels.modelByChannel` config surface. Do not add provider-specific nested `channels.<provider>.*.model` fields or a new config option unless maintainers explicitly ask. Extend the generic resolver/callers so ordinary direct conversations can match stable peer/session target keys while preserving session `/model` precedence, parent/thread inheritance, and wildcard fallback. Document accepted DM key forms in `docs/gateway/config-channels.md` and add focused regression coverage for resolver-only and normal reply/dispatch paths, including at least one direct peer key and one wildcard direct fallback.
