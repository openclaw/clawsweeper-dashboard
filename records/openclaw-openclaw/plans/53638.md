# Coding Plan for openclaw/openclaw#53638: Feature: per-channel / per-group / per-DM model override in config

Source report: [records/openclaw-openclaw/items/53638.md](records/openclaw-openclaw/items/53638.md)

## Summary

Keep open. Current main and the latest release have the shared channels.modelByChannel surface for channel/group targets, but ordinary per-DM matching still lacks resolver inputs/candidate construction, and the public schema/docs do not expose stable DM key forms.

## Plan

Fix the open per-channel/per-group/per-DM model override issue by completing the existing generic channels.modelByChannel path, not by adding provider-specific nested model fields. Add direct-conversation candidate inputs to src/channels/model-overrides.ts and its call sites so Slack, WhatsApp, Discord, Telegram, and generic channel plugins can match stable DM keys derived from finalized context, stored SessionEntry.origin, native direct/channel ids, OriginatingTo/To/From, and direct session keys while preserving current precedence: explicit session /model overrides win, then specific target, then provider wildcard, then global default. Update the public config schema path as appropriate so channels.modelByChannel is discoverable despite the stripped channel schema payload, document accepted channel/group/DM key forms in docs/gateway/config-channels.md, and add focused regression tests. Include a user-facing changelog entry. Do not add new provider-specific channels.<provider>.*.model fields or change existing channel/group modelByChannel semantics.
