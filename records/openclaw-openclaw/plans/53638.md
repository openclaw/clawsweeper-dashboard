# Coding Plan for openclaw/openclaw#53638: Feature: per-channel / per-group / per-DM model override in config

Source report: [records/openclaw-openclaw/items/53638.md](records/openclaw-openclaw/items/53638.md)

## Summary

Keep open. Current main and v2026.5.22 have the shared `channels.modelByChannel` config, schema, docs, and channel/group matching, but specific per-DM matching remains unfinished because the resolver still receives group/thread-style inputs and does not accept ordinary direct session keys.

## Plan

Finish `channels.modelByChannel` support for direct conversations. Keep the existing generic `channels.modelByChannel` config direction; do not add provider-specific nested `channels.<provider>.*.model` fields. Add normalized DM/direct candidate support to `src/channels/model-overrides.ts` and the auto-reply/status call sites so ordinary direct sessions can match a configured target while preserving current precedence: explicit session `/model` override wins, then per-target channel model override, then global default. Derive candidates from finalized message context, persisted session/origin metadata, direct session keys, and plugin-owned conversation binding hooks where available. Document accepted DM key forms in `docs/gateway/config-channels.md` and add focused tests for generic direct sessions plus Slack, WhatsApp, and Discord-style direct identifiers. Stop and ask for maintainer direction if the fix requires a new public config key or a provider-specific schema field.
