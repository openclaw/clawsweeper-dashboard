## Summary

Keep open: current main and the latest release only partially address related history-image attachment handling, and attached prior images still do not get explicit sent-time and thread-position annotations. The closed Slack-only candidate and the merged recent-history-image PR do not supersede the requested shared metadata feature.

## Plan

Implement image freshness annotations for recent inbound history images in the shared auto-reply path. Extend the recent-history image metadata used by `appendRecentHistoryImageContext` so notes include the image sender, message id when available, sent timestamp or relative age derived from the owning `InboundHistory` entry, and position within the bounded history window. Keep the change default-on, do not add a new config option, do not strip old images as the primary fix, and preserve current behavior that avoids reattaching history images when current-turn image/media should take precedence. Cover the shared path so Slack, Discord, Telegram, and other channel histories benefit without deep channel-specific imports; only touch channel-specific code if a producer is missing timestamp/message-id data needed by the shared helper.
