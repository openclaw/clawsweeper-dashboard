## Summary

Keep open. Current main and v2026.5.22 still only give recent history images a sender/message-id note, while Slack thread history has no prompt-visible per-image sent-time or position annotation; the only implementation attempt, https://github.com/openclaw/openclaw/pull/48725, was closed unmerged.

## Plan

Fix https://github.com/openclaw/openclaw/issues/48321 by adding compact prompt-visible timestamp, sender, and history-position metadata to images included from prior thread history. Extend the existing `src/auto-reply/reply/history-media.ts` path so resolved recent history images carry the source entry timestamp and position within the bounded InboundHistory window, and update `appendRecentHistoryImageContext` to produce deterministic notes without exposing local paths. Update Slack thread history in `extensions/slack/src/monitor/message-handler/prepare-thread-context.ts` to annotate image/file placeholders using available `SlackThreadMessage` `ts` and `files` data. Use https://github.com/openclaw/openclaw/pull/48725 as prior art but avoid its reviewed problems: do not claim an exact total thread count when only a bounded history window is known, and do not emit duplicate indistinguishable lines for multiple images. Do not change image retention/pruning, do not add a broad channel API refactor, and do not add a config option unless maintainers explicitly request one. Add focused regression tests and update Slack docs plus changelog if the final behavior is user-visible.
