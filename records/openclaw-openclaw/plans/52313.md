## Summary

Keep open: current main and the latest release still send update respawns through the raw `process.execPath` plus `process.execArgv`/`process.argv.slice(1)` path, so the unstable pnpm realpath failure remains. Related fixes improved lifecycle loading, supervisor detection, and service install paths, but none replace the unmanaged update respawn entrypoint with a stable wrapper.

## Plan

Fix https://github.com/openclaw/openclaw/issues/52313. Current `respawnGatewayProcessForUpdate()` still reaches `spawnDetachedGatewayProcess()`, which spawns `process.execPath` with `process.execArgv + process.argv.slice(1)` and can reuse pnpm versioned OpenClaw realpaths after package updates.
