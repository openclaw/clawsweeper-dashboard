# Coding Plan for openclaw/openclaw#59299: [Bug]: The Gateway daemon of OpenClaw crashes on the laptop when the device switches from being powered by the AC adapter to battery power.

Source report: [records/openclaw-openclaw/items/59299.md](records/openclaw-openclaw/items/59299.md)

## Summary

Keep open: current main and the latest release still create/update the Windows Gateway Scheduled Task without overriding Task Scheduler battery defaults, and the related merged Windows task PR preserved existing settings but did not add the missing battery-power repair.

## Plan

Fix the Windows Gateway Scheduled Task battery-power behavior reported at https://github.com/openclaw/openclaw/issues/59299. Current main creates new tasks with `schtasks /Create /F /SC ONLOGON /RL LIMITED /TN ... /TR ...` and updates existing tasks with `schtasks /Change /TN ... /TR ...`, but never sets `StopIfGoingOnBatteries=false` or `DisallowStartIfOnBatteries=false`; Microsoft defaults both to true. Add a focused install/update repair for the OpenClaw Gateway Scheduled Task that sets both flags false for newly created tasks and already-installed tasks. Preserve the existing `/Change` behavior from https://github.com/openclaw/openclaw/pull/59335, task user/run-level/script behavior, access-denied and hung-`schtasks` Startup-folder fallback behavior, restart behavior, and Node/Bun compatibility. Add focused tests in the scheduled-task install tests for both create and existing-task repair paths. Add a single changelog fix entry for the user-visible Windows daemon crash fix.
