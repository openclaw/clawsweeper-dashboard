# Coding Plan for openclaw/openclaw#41826: Feature Request: Add custom client display name for TUI

Source report: [records/openclaw-openclaw/items/41826.md](records/openclaw-openclaw/items/41826.md)

## Summary

Keep open. Current main still lacks the requested TUI client display-name option: the CLI exposes no `--client-name`/`--username` flag, `TuiOptions` cannot carry it, and the Gateway TUI client still sends the hard-coded `openclaw-tui` display name even though the Gateway protocol already has a `displayName` field.

## Plan

Implement https://github.com/openclaw/openclaw/issues/41826 as a focused TUI feature. Add a user-facing `openclaw tui` display-name flag, preferably `--client-name <name>` unless nearby CLI naming strongly suggests a better name, thread it through `TuiOptions`, `runTui`, `GatewayChatClient.connect`, and `GatewayChatClient` into the existing `clientDisplayName` GatewayClient option. Preserve `clientName: GATEWAY_CLIENT_NAMES.TUI` as the stable client id. Sanitize/trim/length-bound the user-supplied label and strip terminal control/ANSI sequences before sending it to Gateway surfaces. Update CLI/help docs and focused tests for CLI option parsing, GatewayChatClient option propagation/default behavior, and any launch path touched. Do not implement broader session labels, auto-generated labels, or unrelated presence/session UI changes. Add a CHANGELOG.md entry because this is user-visible.
