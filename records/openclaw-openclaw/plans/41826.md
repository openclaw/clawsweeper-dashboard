# Coding Plan for openclaw/openclaw#41826: Feature Request: Add custom client display name for TUI

Source report: [records/openclaw-openclaw/items/41826.md](records/openclaw-openclaw/items/41826.md)

## Summary

Keep open. Current main still lacks the requested TUI display-name option: the CLI has no client-name/username flag, `TuiOptions` cannot carry such a value, and the Gateway TUI client still sends the hard-coded `openclaw-tui` display name even though the Gateway client already supports `clientDisplayName`.

## Plan

Implement a focused `openclaw tui` display-name option for https://github.com/openclaw/openclaw/issues/41826. Add a user-facing flag such as `--client-name`, carry it through `TuiOptions` and the TUI Gateway connect path, and pass a sanitized display label into the existing `GatewayChatClient` / `GatewayClientOptions.clientDisplayName` contract. Preserve the stable `openclaw-tui` client id and avoid protocol changes. Trim, length-bound, and strip ANSI/control sequences from user-controlled display text before it can appear in Gateway/session/presence surfaces. Before docs edits, run `pnpm docs:list` and update the relevant TUI docs only. Add focused tests for CLI option parsing/options propagation, TUI relaunch args if needed, and Gateway client connect behavior; add a CHANGELOG entry if the change is user-facing.
