# Coding Plan for openclaw/openclaw#57376: sessions.list returns phantom store-key entry with null updatedAt and no sessionId

Source report: [records/openclaw-openclaw/items/57376.md](records/openclaw-openclaw/items/57376.md)

## Summary

Keep open. Current main and the latest release still lack a server-side listability guard for `sessions.list`; current normalization may coerce the reported `updatedAt: null` to `0`, but a no-`sessionId` store placeholder can still flow into the RPC, and the linked fix PR was closed unmerged.

## Plan

Fix https://github.com/openclaw/openclaw/issues/57376: `sessions.list` must not return store-level placeholder entries such as `agent:<id>:sessions` with no real `sessionId`. Add a server-side listability guard in the Gateway session-list selection path so invalid persisted `SessionEntry` rows are excluded before row construction; do not solve this in the Control UI, and do not rely only on `updatedAt == null` because current load normalization can coerce missing/null timestamps to `0`. Add focused regression coverage for `listSessionsFromStore`/`listSessionsFromStoreAsync` and Gateway `sessions.list` with a per-agent store template whose raw store contains `sessions: {}` plus a real session. Preserve session-store canonicalization and existing configured-agent behavior. Add a changelog entry only if maintainers treat the visible RPC/UI behavior fix as user-facing.
