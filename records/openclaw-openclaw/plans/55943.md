# Coding Plan for openclaw/openclaw#55943: subagent sessions don't trigger session-memory hook

Source report: [records/openclaw-openclaw/items/55943.md](records/openclaw-openclaw/items/55943.md)

## Summary

Keep open. Current main and the latest release still keep the bundled session-memory hook command-gated to `/new` and `/reset`, while subagent completion emits `subagent_ended` plugin lifecycle hooks and does not reach the memory writer; the related session_end PR is still open and not a merged fix.

## Plan

Fix the session-memory subagent completion gap tracked at https://github.com/openclaw/openclaw/issues/55943. Treat this as a narrow user-visible feature/gap, not a broad memory-system redesign. Current main only registers and runs `session-memory` for `command:new` and `command:reset`; native `sessions_spawn` completion emits `subagent_ended` and does not create a child workspace memory artifact. Implement a lifecycle bridge so completed native child sessions can resolve the child workspace, session key/session entry, and transcript file, then reuse the existing session-memory writer to create one dated Markdown artifact. Preserve existing `/new` and `/reset` behavior, keep capture off the reply hot path, and add idempotency so `/new`/`/reset`, `session_end`, and subagent completion cannot duplicate one ended session. Coordinate with https://github.com/openclaw/openclaw/pull/56672 if that PR has landed or changed the `session_end` API; prefer a clean shared lifecycle event over an ad hoc second writer. Do not add a new config option unless maintainers explicitly request it. Update docs for the new trigger and add a changelog entry under the active version because this is user-visible; thank `@Tiantian225650` if the entry credits the reporter.
