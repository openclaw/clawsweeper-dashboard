# Coding Plan for openclaw/openclaw#60812: fix(channels): extend user-visible error fallback to Discord, Slack, Signal, WhatsApp

Source report: [records/openclaw-openclaw/items/60812.md](records/openclaw-openclaw/items/60812.md)

## Summary

Keep open. Current main still has Telegram/LINE-style user-visible fallback behavior only in some paths; Discord, Slack, Signal, and WhatsApp still log, react, rethrow, or return without sending the requested fallback message when reply dispatch fails or produces no visible response.

## Plan

Fix https://github.com/openclaw/openclaw/issues/60812 by adding user-visible fallback delivery for Discord, Slack, Signal, WhatsApp, and the existing LINE hardcoded fallback path when dispatch fails or no visible non-silent reply is delivered. Keep the change plugin-safe: prefer a small helper or SDK/runtime seam for selecting fallback text and delivery state, avoid core deep imports from bundled plugins, and preserve Telegram's existing behavior. Do not send fallback messages after any visible reply, partial stream fallback, durable delivery, media-only send, abort/stop/superseded turn, or intentional silent NO_REPLY/silent-token success. Add focused per-channel regression tests and update shared helper tests if a new seam is added. No config option or product-policy expansion should be introduced.
