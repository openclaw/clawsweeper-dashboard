Current main still has the Telegram fallback pattern, but Discord, Slack, Signal, and WhatsApp either log, mark status reactions as error, or return false after failed/no-visible dispatch without sending a user-visible fallback reply; the linked Telegram/model PR is open and does not implement this cross-channel work.

## Plan

Fix https://github.com/openclaw/openclaw/issues/60812. Preserve channel-specific status reaction, ack cleanup, sourceReplyDeliveryMode/message_tool_only, group-history, and visible-partial-delivery guards so the fix does not create duplicate or spurious replies.
