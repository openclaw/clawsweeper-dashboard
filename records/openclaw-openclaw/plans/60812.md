Current main still has the shared failed-delivery signal and Telegram fallback anchor, but Discord, Slack, Signal, WhatsApp, and LINE's typed-routing slice do not yet send a user-visible fallback reply on final dispatch failure; related search still points to this issue as the canonical tracker.

## Plan

Fix the cross-channel dispatch-error fallback tracked by https://github.com/openclaw/openclaw/issues/60812. Add a small plugin-safe helper for choosing user-visible fallback text from dispatch errors and empty/no-visible-response state, then wire Discord, Slack, Signal, WhatsApp, and LINE's typed-routing slice through it without changing channel policy, config defaults, or room-event/message-tool-only suppression semantics. Preserve guards that prevent duplicate fallback replies after any visible, durable, queued final, streamed, partial, media-only, or message-tool delivery.
