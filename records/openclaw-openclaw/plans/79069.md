## Summary

Keep open: current main still drops the silent-empty flag before the embedded cron runner call, while the embedded runner only accepts empty assistant stops when that flag is explicitly set. The closed related PR attempted the right narrow fix but was not merged, so the bug remains present on current main.

## Plan

Fix https://github.com/openclaw/openclaw/issues/79069. Current main resolves `deliveryPlan.mode === "none"` in `src/cron/isolated-agent/run.ts`, but `executeCronRun` and `createCronPromptExecutor` only receive target fields and `suppressExecNotifyOnExit`, so the embedded `runEmbeddedPiAgent` call never receives `allowEmptyAssistantReplyAsSilent`. Add a clear boolean derived from the resolved delivery plan, thread it through the cron executor params, and pass it to `runEmbeddedPiAgent` so explicit no-delivery cron jobs can treat clean empty assistant stops as silent success.
