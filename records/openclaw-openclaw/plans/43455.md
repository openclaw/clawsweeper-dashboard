## Summary

Keep open. Current main still documents Browserbase as a static WebSocket CDP profile and still routes it through the generic CDP attach path, while Browserbase's current docs center session creation through the Sessions API and a returned connection URL.

## Plan

Fix https://github.com/openclaw/openclaw/issues/43455 by adding Browserbase session lifecycle support inside the browser plugin only. Preserve generic remote CDP behavior for Browserless, local Chrome, and explicit `ws`/`wss` URLs. Implement Browserbase detection/config handling narrowly enough to create a Browserbase session through `POST https://api.browserbase.com/v1/sessions` with the API key from the existing documented profile flow or an owner-approved equivalent, attach Playwright to the returned `connectUrl`, cache/reuse consistently with existing `connectBrowser()` semantics, and release/close only sessions that OpenClaw created. Add focused mocked-contract tests for session creation, attach fallback, failure/redaction, and cleanup; update `docs/tools/browser.md` and `CHANGELOG.md`. Do not add broad core migrations, do not weaken SSRF checks, and stop for maintainer review if the fix requires a new public config option or SecretRef policy decision.
