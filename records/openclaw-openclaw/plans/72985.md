## Summary

Keep open. Current main still has the cron isolated model fallback path accepting embedded incomplete-turn error payloads without the existing result classifier, then delivery and persistence can still record a fatal cron run independently from a successful channel send.

## Plan

Fix https://github.com/openclaw/openclaw/issues/72985 as a narrow cron isolated-agent bug. Reuse `classifyEmbeddedPiRunResultForModelFallback` in the cron `runWithModelFallback` path so embedded incomplete/empty/reasoning-only fallback results continue through configured model fallbacks or fail before being treated as cron output. Ensure fatal incomplete-turn/error payloads are not persisted as successful cron content delivery (`lastRunStatus: "error"` should not imply `lastDelivered: true` for the fatal output), while preserving the intentional distinction between channel-send diagnostics and successful cron content delivery. Do not add a new config option or broad alert policy; keep the fix in the cron runtime/result-classification boundary and add focused regression tests plus a changelog entry if user-visible behavior changes.
