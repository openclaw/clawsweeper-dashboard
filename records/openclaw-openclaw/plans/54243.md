## Summary

Keep open. Current main and the latest release still include the broad state asset and only drop separate workspace candidates from the plan, so nested `~/.openclaw/workspace*` contents are still archived despite the CLI/docs promise.

## Plan

Fix https://github.com/openclaw/openclaw/issues/54243 as a narrow backup flag correctness bug. Make `openclaw backup create --no-include-workspace` materially exclude known workspace roots even when they are nested under the `state` asset, by carrying workspace exclusion roots into archive creation and filtering those roots from the state-asset tar traversal when `includeWorkspace` is false. Preserve `--only-config`, invalid-config partial backup behavior, manifest placement, installed-plugin `extensions/*/node_modules` pruning, volatile-state filtering, output containment checks, and current archive path remapping. Do not implement generic `--exclude`/`--exclude-file` patterns here; that remains tracked by https://github.com/openclaw/openclaw/issues/40786 and https://github.com/openclaw/openclaw/pull/44288. Add focused regression coverage proving nested `~/.openclaw/workspace` and profile-style workspace roots are absent from a real archive or filter path with `includeWorkspace: false`, while ordinary state and non-workspace files still archive. Update CLI docs/help if needed so the documented behavior matches the implementation, and add a changelog entry for the user-visible backup flag fix.
