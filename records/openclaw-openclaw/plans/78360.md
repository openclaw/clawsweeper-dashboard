## Summary

Keep open: current main and v2026.5.12 still let the TUI watchdog clear an active run to idle and emit the reported prompt based on stream silence, while the TUI does not carry the gateway's existing active-run signal. The related timeout/config and stale-notice PRs do not supersede this active-run correctness bug.

## Plan

Send another message to continue.` solely because no stream delta arrived if the current session still has an authoritative active run. Thread the existing gateway `sessions.list.hasActiveRun` signal through the TUI session-list/session-info path or add a narrow helper, refresh/check it on watchdog expiry, and rearm/keep `streaming` while the backend reports the run active.
