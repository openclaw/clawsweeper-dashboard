## Summary

Keep open. Current main still writes and bootstraps the macOS Gateway LaunchAgent from the resolved HOME path, so an external-APFS home still routes launchctl to a /Volumes/... plist and the reported error remains source-backed and actionable.

## Plan

Fix the macOS Gateway LaunchAgent external-HOME bootstrap bug from https://github.com/openclaw/openclaw/issues/60398. Add a shared resolver for the LaunchAgent plist path that keeps the normal HOME/Library/LaunchAgents behavior for ordinary homes, but uses a launchd-safe boot-volume /Users/<account>/Library/LaunchAgents path when HOME resolves to an external APFS volume. Wire the resolver through install/stage, legacy cleanup, bootstrap repair, restart fallback, detached launchd handoff, update restart helper, service audit/status inspection, and the macOS app LaunchAgent snapshot or its CLI-backed equivalent.
