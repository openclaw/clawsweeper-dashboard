## Summary

Keep open. Current main and the latest release include important Dreaming cleanup hardening, especially the merged post-run scrubber from https://github.com/openclaw/openclaw/pull/88549, but the Gateway startup path still only reconciles the managed Dreaming cron and does not run that scrubber after a restart interrupts the in-process cleanup.

## Plan

Fix the remaining Gateway-restart Dreaming orphan cleanup path for https://github.com/openclaw/openclaw/issues/78458. Current `extensions/memory-core/src/dreaming-narrative.ts` has a bounded `scrubDreamingNarrativeArtifacts` path that runs after `generateAndAppendDreamNarrative`, and https://github.com/openclaw/openclaw/pull/88549 proved it reclaims aged transcript-backed `dreaming-narrative-*` rows. The missing piece is that `extensions/memory-core/src/dreaming.ts` handles `gateway_start` by reconciling the managed Dreaming cron only, so a Gateway process killed before the narrative `finally` block can leave rows until a later narrative run. Add a narrow plugin-owned startup or deferred-startup cleanup that reuses/factors the same scrubber semantics, preserves the existing 5-minute fresh-run threshold, does not delete unrelated sessions, does not grant broad admin powers, and adds focused regression coverage. Do not add a config/env option or changelog entry; stop for maintainer review if the implementation requires changing the permanent plugin SDK or session-store contract.
