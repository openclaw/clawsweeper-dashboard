## Summary

Keep open. Current `main` still leaves Signal without the provider-backed audio preflight path that the discussion asks for, and the existing Signal mention gate can skip audio-only group messages before attachments are fetched or transcribed.

## Plan

Fix https://github.com/openclaw/openclaw/issues/70439 by adding a Signal-owned provider-backed audio preflight for inbound voice notes. Reuse `transcribeFirstAudio` through a Signal-local lazy runtime wrapper before mention gating, so audio-only Signal group messages with `requireMention` can be transcribed and checked for mention patterns before the no-mention return. Thread the transcript into the inbound context/body and mark the transcribed media index so downstream media understanding does not transcribe the same attachment again. Keep the boundary narrow: use the existing `tools.media.audio` provider/CLI contract and do not add `channels.signal.audio.preflight_cmd`, arbitrary shell execution, new global config, or broader product policy.
