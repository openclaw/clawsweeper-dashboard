## Summary

Keep open. Current main and v2026.5.7 still lack the Signal audio preflight path used by other bundled channels, so mention-gated Signal voice notes can be skipped before transcription despite the shared audio contract.

## Plan

Fix https://github.com/openclaw/openclaw/issues/70439. Signal group voice notes with requireMention currently evaluate mention gating before fetching/transcribing attachments, so audio-only messages can be skipped before the shared media preflight can detect a spoken mention. Implement a Signal-owned provider-backed audio preflight using the existing `transcribeFirstAudio` helper through the plugin SDK/runtime boundary before the mention decision, and pass the transcript plus transcribed-media index state downstream so main media understanding does not duplicate STT. Keep the fix scoped to Signal and shared existing media contracts. Do not add `channels.signal.audio.preflight_cmd`, arbitrary shell execution, new provider auth policy, or broad channel config surface. Add focused Signal tests for audio-only mention-gated groups, transcript propagation, fallback/no-transcript behavior, and duplicate-STT avoidance where practical; update docs/changelog only if the user-visible Signal behavior or documented support matrix changes.
