## Summary

Keep open. Current main and the latest release still lack the Signal audio preflight path used by other bundled channels, so mention-gated Signal voice notes can be dropped before transcription despite the shared audio contract.

## Plan

Fix https://github.com/openclaw/openclaw/issues/70439 by adding Signal audio preflight for voice notes before mention gating. Use the existing shared transcribeFirstAudio/media-understanding contract rather than adding a new channels.signal.audio.preflight_cmd command-execution feature. The expected boundary is Signal-owned attachment fetch/preflight wiring: for audio-only Signal group messages that require mentions, fetch enough attachment metadata/media to run provider-backed preflight, use the transcript for mention detection and agent-facing body when accepted, and thread Transcript/MediaTranscribedIndexes so downstream media understanding does not transcribe the same audio twice. Preserve current access-control behavior and do not broaden sender authorization. Add focused regression tests for mention-gated Signal audio, ordinary Signal audio context, and no duplicate STT. Add a one-line CHANGELOG.md Fixes entry if this is user-facing under the active version.
