## Summary

Keep open: current main and v2026.5.22 still treat Ollama `message.content` as visible text, while existing reasoning fixes cover native fields, tagged thinking, Slack delivery, and OpenCode Go replay payloads rather than this Ollama Kimi inline-content leak.

## Plan

Fix the Ollama Kimi cloud inline reasoning leak reported in https://github.com/openclaw/openclaw/issues/86129. Keep the change provider-owned in the Ollama plugin: for `ollama` Kimi cloud models such as `kimi-k2.5:cloud` and `kimi-k2.6:cloud`, prevent inline reasoning text before the known Kimi boundary delimiter from being emitted as visible text when thinking is disabled. The fix must handle streaming deltas as well as the final assistant message so channels do not receive pre-delimiter text before final cleanup. Prefer a narrow sanitizer in `extensions/ollama/src/stream.ts` or a small local helper with tests in `extensions/ollama/src/stream.test.ts` and `extensions/ollama/src/stream-runtime.test.ts`; do not add a global gateway heuristic, new config option, or behavior for unrelated providers/models. Preserve native `message.thinking`/`message.reasoning` handling and existing garbled-text/tool-call behavior. Validate with focused Vitest wrapper commands and `git diff --check`; add a changelog entry during landing if maintainers treat this as user-visible.
