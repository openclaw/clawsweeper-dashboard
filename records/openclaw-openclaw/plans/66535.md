## Summary

Keep open. Current main still starts manual `/compact` without passing a cancellation signal or registering the compaction as an active embedded run, so `/stop`, `chat.abort`, and direct embedded-run abort cannot reliably discover or interrupt it; the linked fix PRs are closed and unmerged.

## Plan

Fix https://github.com/openclaw/openclaw/issues/66535 on current main. Manual `/compact` in `src/auto-reply/reply/commands-compact.ts` must participate in existing embedded-run cancellation: create a local compaction `AbortController`, propagate `params.opts?.abortSignal` into it one-way if present, make the active-run handle abort that controller, pass the local signal as `abortSignal` to `compactEmbeddedPiSession`, register the handle with `setActiveEmbeddedRun(sessionId, handle, params.sessionKey)`, and always call `clearActiveEmbeddedRun` in `finally`. Export only the needed active-run helpers/types through the existing facades. Add focused tests proving the command passes an abort signal, registers during the in-flight promise, `abortEmbeddedPiRun(sessionId)` aborts the manual compaction, cleanup happens on success and failure, and the previous active-run drain behavior remains. Do not mix in launchd/profile changes, do not solve the broader lane-starvation issue, and avoid aborting the parent reply signal from compaction timeout/failure. Add a user-facing changelog entry for the cancellation fix.
