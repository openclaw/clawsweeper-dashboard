# Coding Plan for openclaw/openclaw#7476: Feature Request: WhatsApp sticker send support

Source report: [records/openclaw-openclaw/items/7476.md](records/openclaw-openclaw/items/7476.md)

## Summary

Keep open. Current main still lacks outbound WhatsApp sticker support, and both related implementation PRs are closed unmerged while the active WhatsApp send path still treats WebP as ordinary image media.

## Plan

Implement outbound WhatsApp sticker send support for https://github.com/openclaw/openclaw/issues/7476. Keep the change within the bundled WhatsApp plugin and existing channel/message-action boundaries. Prefer an explicit sticker intent such as a WhatsApp-supported sticker action or sendAsSticker/asSticker option; avoid caption-based implicit WebP auto-detection unless the current codebase already has a clear matching pattern. Thread the intent from the message tool/channel action/outbound payload through sendMessageWhatsApp and ActiveWebSendOptions into createWebSendApi, preserve intended sticker WebP/media before generic image optimization only when that intent is set, and send through Baileys as { sticker: buffer }. Add focused tests for send-api payload selection, send option threading, action discovery/dispatch if an action is added, and shared media preservation behavior. Update docs/channels/whatsapp.md and CHANGELOG.md for the user-facing capability. Do not broaden behavior for other channels or globally preserve all WebP images without explicit WhatsApp sticker intent. Use https://github.com/openclaw/openclaw/pull/7502 and https://github.com/openclaw/openclaw/pull/19592 as context, not as code to apply blindly.
