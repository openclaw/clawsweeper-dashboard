# Coding Plan for openclaw/openclaw#58450: Agent can promise a later follow-up without starting any actual follow-up action

Source report: [records/openclaw-openclaw/items/58450.md](records/openclaw-openclaw/items/58450.md)

## Summary

Keep open. Current main has meaningful mitigations for the broader plan-only and incomplete-turn regression, but it still does not enforce the specific invariant that a final reply promising a later follow-up must be backed by a same-turn cron, subagent, queued task, or equivalent mechanism.

## Plan

Fix https://github.com/openclaw/openclaw/issues/58450 by adding conservative runtime enforcement for final replies that promise a later follow-up without starting one. Build on the existing reminder guard in src/auto-reply/reply/agent-runner-reminder-guard.ts and reply assembly in src/auto-reply/reply/agent-runner.ts. Detect a small set of unbacked future-follow-up promise shapes such as “I'll come back”, “I'll send a follow-up”, and close variants, including the reported one memory_search/read-style action followed by future-work prose. Suppress the guard only when the same turn actually created a follow-up mechanism such as successful cron.add, sessions_spawn/subagent background work, or another explicit queued/background task; do not treat arbitrary file edits, reads, memory_search, or generic side effects as follow-up mechanisms. Add focused regression coverage in src/auto-reply/reply/agent-runner.misc.runreplyagent.test.ts and, if the planning-only path is adjusted, src/agents/pi-embedded-runner/run.incomplete-turn.test.ts. Keep wording American English, avoid a config option, and keep the matcher conservative enough to avoid ordinary status summaries or completed-result replies.
