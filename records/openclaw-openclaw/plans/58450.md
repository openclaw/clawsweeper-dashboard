# Coding Plan for openclaw/openclaw#58450: Agent can promise a later follow-up without starting any actual follow-up action

Source report: [records/openclaw-openclaw/items/58450.md](records/openclaw-openclaw/items/58450.md)

## Summary

Keep open. Current main has useful reminder-note and incomplete-turn mitigations, but it still does not enforce the broader invariant that a final reply promising to come back later must be backed by a real follow-up mechanism in the same turn.

## Plan

Fix https://github.com/openclaw/openclaw/issues/58450 with a narrow runtime reply-pipeline guard for unbacked future-follow-up promises. Extend or add to the existing guard near src/auto-reply/reply/agent-runner-reminder-guard.ts and its runReplyAgent integration so final user-visible text that clearly promises a later follow-up, for example coming back later with a message or sending a follow-up, is annotated or rewritten when the same turn did not create a real follow-up mechanism. Preserve the existing reminder/cron behavior and false-positive suppression. Treat successful cron.add, existing same-session cron coverage where appropriate, sessions_spawn/subagent background work, queued follow-up delivery, and committed messaging delivery as evidence that can back the promise if the runtime exposes it. Do not add a new config option, do not implement broad commitment gating, and do not change unrelated strict-agentic retry behavior. Add focused regression coverage in runReplyAgent tests and, only if needed for evidence boundaries, the incomplete-turn helper tests. Add an active CHANGELOG Fixes entry because this is user-visible.
