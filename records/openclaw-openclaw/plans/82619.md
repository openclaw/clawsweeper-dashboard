## Summary

Keep open: the bug and proof look credible, but the patch still has a correctness gap in the new LRU lookup. It computes cache keys from one last-written registry-state hint, so alternating workspace-scoped and unscoped derived snapshots can still miss cached entries when their registry fingerprints differ.

## Plan

Fix PR https://github.com/openclaw/openclaw/pull/82619 so the plugin metadata snapshot LRU still hits when workspace-scoped and unscoped callers cache derived snapshots with different registry fingerprints. In `src/plugins/plugin-metadata-snapshot.ts`, do not compute every lookup key from only the last `pluginMetadataSnapshotRegistryStateMemo`; use entry-specific/candidate registry state or another bounded lookup strategy that preserves existing freshness checks, clone-on-hit behavior, and the 8-entry cap.
