## Summary

Keep open. Current main and the latest release still derive the macOS LaunchAgent stdout log path from the OpenClaw state directory, so the independently confirmed symlinked/external-volume launchd `EX_CONFIG` failure remains actionable.

## Plan

Fix https://github.com/openclaw/openclaw/issues/40207 by changing the macOS Gateway LaunchAgent stdout log path so generated and rewritten plists do not point `StandardOutPath` into `resolveGatewayStateDir(env)/logs` when the state directory may be symlinked or external. Keep the change focused on LaunchAgent stdout logging and dependent diagnostics/docs/tests; do not fold in the separate stdin hang from https://github.com/openclaw/openclaw/issues/46153 or the external-home plist bootstrap resolver from https://github.com/openclaw/openclaw/issues/60398. Update focused tests and docs/status output, and add a changelog entry because this is user-facing macOS Gateway startup behavior.
