# Coding Plan for openclaw/openclaw#80498: Subagent completion announcements can be premature or duplicated after tool-use turns

Source report: [records/openclaw-openclaw/items/80498.md](records/openclaw-openclaw/items/80498.md)

## Summary

Keep open: current main still treats non-`sessions_yield` assistant tool-use text as selectable completion output, and the stable announce id is only passed down as an idempotency key rather than guarded in `runSubagentAnnounceFlow` itself. The report is source-reproducible and overlaps existing subagent-announce work, but no canonical open item fully covers this exact tool-use-plus-terminal-idempotency combination.

## Plan

Fix https://github.com/openclaw/openclaw/issues/80498 on current main. Scope the PR to subagent completion announcement output selection and terminal announce idempotency. In `src/agents/subagent-announce-output.ts`, treat assistant messages containing any tool call as progress rather than terminal completion output; after non-`sessions_yield` tool use, keep waiting for a later assistant message without tool calls before selecting final output, and do not promote tool/toolResult text while waiting. Preserve timeout partial-progress behavior. In `src/agents/subagent-announce.ts`, add bounded in-memory idempotency for terminal announces keyed by the existing stable announce id from `buildAnnounceIdFromChildRun({ childSessionKey, childRunId })`, so duplicate calls for the same terminal child result do not deliver twice while still allowing cleanup/failure paths to proceed. Add focused regression tests in `src/agents/subagent-announce-output.test.ts` and `src/agents/subagent-announce.test.ts`. Check the overlapping open work at https://github.com/openclaw/openclaw/pull/80110, https://github.com/openclaw/openclaw/pull/73783, https://github.com/openclaw/openclaw/pull/74113, and https://github.com/openclaw/openclaw/pull/74340, but do not broaden this PR into provider parsing, compaction policy, parent fingerprint dedupe, or unrelated raw-output cleanup unless required for this issue.
