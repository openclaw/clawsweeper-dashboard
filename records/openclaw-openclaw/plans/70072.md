Keep open. Current main still lacks the requested deep-dreaming introspection surfaces: the scheduled deep report remains aggregate-only, verbose scoring is logger-only, rejected entries are filtered before callers can explain them, and the memory CLI still has only single-candidate `promote-explain` rather than an aggregate dreaming diagnostic. ## Plan

Implement the additive diagnostics requested by https://github.com/openclaw/openclaw/issues/70072. Add a shared deep-promotion evaluation/explain helper that preserves current scoring and promotion defaults while exposing bounded considered candidates, score components, thresholds, pass/fail/promoted state, and rejection reasons. Reuse that helper to extend `memory/dreaming/deep/YYYY-MM-DD.md`, persist structured verbose scoring when `verboseLogging` is enabled, and add an aggregate `openclaw memory dreaming doctor` or equivalent memory CLI diagnostic. Keep output privacy-conscious and bounded; do not alter promotion policy, default thresholds, or unrelated dreaming behavior. Update focused tests and user docs; add a changelog entry because this is user-visible diagnostics work.
