## Summary

Current main and the latest release still handle normal explicit skill slash commands by rewriting the turn into a model prompt instead of loading the selected SKILL.md, so there is no conservative cleanup close reason. The issue remains a clear, bounded feature gap with source-level reproduction and a reasonable repair path.

## Plan

Fix https://github.com/openclaw/openclaw/issues/52473 by making explicit non-tool skill slash invocations deterministically provide the selected SKILL.md instructions to the model. Scope the change to `/skill <name> [input]` and direct user-invocable skill commands only; do not change natural-language skill matching, skill visibility/allowlist filtering, authorization checks, Claude bundle `promptTemplate` commands, or `command-dispatch: tool` behavior. Prefer reusing `SkillEntry.skill.filePath` / the existing workspace skill loader or hydrated snapshot contract instead of adding a second discovery source. Add focused regression coverage showing that an explicit workspace skill command loads/injects that skill's SKILL.md before the user args, that tool-dispatch skills still bypass the model, and that bundle command templates still expand as before. Update `docs/tools/slash-commands.md` and `docs/tools/skills.md` for the new explicit-invocation behavior, and add a `CHANGELOG.md` Changes entry because this is user-visible.
