## Summary

Keep open: current main still resolves explicit non-tool skill slash commands, then rewrites the turn into advisory model text instead of deterministically injecting the invoked SKILL.md content. The related deterministic tool-dispatch work solves a different path and does not supersede this request.

## Plan

Implement deterministic SKILL.md context injection for explicit non-tool skill slash commands. When `/skill-name [args]` or `/skill <name> [args]` resolves to an eligible normal workspace skill, ensure the agent receives that skill's full instructions before the user args, without depending on the model voluntarily reading SKILL.md. Preserve `command-dispatch: tool`, Claude bundle prompt templates, natural-language skill matching, authorization checks, and existing skill filters. Avoid broad hot-path rediscovery/freshness polling; prefer carrying already-loaded eligible skill metadata or lifecycle-owned snapshot data into the invocation path. Update focused tests for direct skill command and generic `/skill` entrypoint, docs for slash-command behavior, and add a maintainer-owned changelog entry if this lands as user-visible behavior.
