## Summary

Keep open. Current main and the latest shipped release still route explicitly invoked normal skill commands through a model-mediated rewrite instead of loading and injecting the selected SKILL.md, so the requested deterministic activation path remains unimplemented.

## Plan

Implement deterministic SKILL.md injection for explicitly invoked normal skill slash commands. Scope this to `/skill <name> [input]` and direct user-invocable skill commands that currently fall through to the model rewrite path; do not change natural-language skill matching, command registration, sender authorization, or `command-dispatch: tool` behavior. Reuse the eligible/visible skill command or skill snapshot data so the selected skill respects existing filters, load the selected SKILL.md content, and inject it into the agent context before the user args using an existing safe context mechanism. Add focused regression coverage for `/skill <name>`, direct skill command invocation, tool-dispatch bypass preservation, and filtered/non-visible skills. Update docs for the changed slash-command behavior and add a changelog entry under the active `### Changes` section if the behavior is user-facing. Stop and ask if the fix would require a new config option or a broader product policy change.
