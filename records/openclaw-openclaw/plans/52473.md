## Summary

Current `main` still has the model-mediated skill slash-command behavior described by the issue, so there is no cleanup close reason yet; the requested deterministic SKILL.md injection path remains useful and source-provable.

## Plan

Implement deterministic SKILL.md injection for explicit skill slash-command invocation. When an authorized user invokes a normal skill via `/skill <name> [input]` or a direct per-skill slash command, resolve the selected eligible skill using the existing skill command/snapshot visibility rules, load the selected `SKILL.md` content from disk, and inject it into the model context before the user args. Keep natural-language skill matching advisory, preserve Claude bundle `promptTemplate` commands, and leave `command-dispatch: tool` skill commands on their direct tool path. Do not add a config option. Add focused regression tests for direct and `/skill` invocation proving the selected skill content is available without relying on the model to read it, plus tests that tool-dispatch and bundle commands keep their existing behavior. Update user-facing docs if the default slash-command behavior changes; include a changelog entry if policy requires it for the user-visible feature.
