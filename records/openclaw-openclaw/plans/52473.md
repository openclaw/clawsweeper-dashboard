## Summary

Keep open: current main and the latest release still route explicit non-tool skill slash commands into an advisory model prompt instead of loading the selected SKILL.md, and the remaining work is not superseded by a canonical fix.

## Plan

Implement deterministic SKILL.md injection only for explicit skill slash invocations tracked by https://github.com/openclaw/openclaw/issues/52473. Preserve natural-language skill matching, existing `/skill <name> [input]` parsing, sender authorization checks, skill visibility/agent filters, Claude bundle promptTemplate behavior, and `command-dispatch: tool` direct dispatch. Reuse the existing visible skill entry/snapshot/loader contracts to locate and read the selected eligible SKILL.md, inject it as model-only context before the user's args, and ensure the injected content is not sent back to channels, normal command help, UI command lists, or logs. Add focused regression tests for direct `/skill-name args`, `/skill <name> args`, tool-dispatch bypass, bundle command templates, unauthorized senders, and skills filtered out of the current session. Update user-facing docs and include a concise CHANGELOG.md entry because this changes skill command behavior.
