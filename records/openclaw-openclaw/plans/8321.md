# Coding Plan for openclaw/openclaw#8321: Feature: Per-job cache control for cron jobs

Source report: [records/openclaw-openclaw/items/8321.md](records/openclaw-openclaw/items/8321.md)

## Summary

Keep open: current main still lacks a per-cron-job cache/context-pruning override, and the related direct-exec cron work is complementary rather than a replacement for agent-run cron jobs.

## Plan

Implement per-cron-job context pruning for https://github.com/openclaw/openclaw/issues/8321. Add an additive top-level contextPruning?: AgentContextPruningConfig field to cron job create/update/list contracts using the existing mode: "off" | "cache-ttl" config shape, not a one-off disableCache boolean. Thread it through cron normalization, persistence, gateway protocol schemas, CLI add/edit, Control UI types/controller/form handling, agent cron tool schema, docs, and the isolated cron runner so job.contextPruning overrides global/per-agent agents.defaults.contextPruning only for that job before the embedded/CLI runner receives cfgWithAgentDefaults. Add regression coverage proving a job-level mode: "off" disables cache-ttl when global defaults enable it, and that cache-ttl still applies when no job override is set. Do not implement direct exec/script cron payloads; keep that separate from https://github.com/openclaw/openclaw/issues/18160. Add a changelog entry because this is a user-facing cron/config feature.
