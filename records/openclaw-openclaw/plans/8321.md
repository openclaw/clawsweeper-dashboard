## Plan

Implement per-cron-job cache/context-pruning control for https://github.com/openclaw/openclaw/issues/8321. Add a top-level `contextPruning?: AgentContextPruningConfig` field to cron jobs using the existing `mode: "off" | "cache-ttl"` contract, not a separate `disableCache` boolean. Thread it through cron create/update/list types, shared persisted base, normalization, service patch/build paths, gateway protocol schema, CLI add/edit, Control UI types/controllers, agent cron tool schema/recovery, docs, and changelog. In isolated cron execution, merge the job override into the effective `cfgWithAgentDefaults.agents.defaults.contextPruning` before the embedded runner receives config; do not implement direct exec/script cron mode here, which is a separate path from https://github.com/openclaw/openclaw/issues/24597. Add regression coverage proving a job-level `mode: "off"` overrides global or per-agent `agents.defaults.contextPruning.mode: "cache-ttl"` for isolated cron while preserving existing jobs without the field.
