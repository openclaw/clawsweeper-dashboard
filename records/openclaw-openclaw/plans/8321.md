# Coding Plan for openclaw/openclaw#8321: Feature: Per-job cache control for cron jobs

Source report: [records/openclaw-openclaw/items/8321.md](records/openclaw-openclaw/items/8321.md)

## Summary

Keep open. Current main still has no per-cron-job `contextPruning` or `disableCache` field, the isolated cron runner still derives pruning from effective agent defaults, and the earlier implementation attempt at https://github.com/openclaw/openclaw/pull/8341 was closed unmerged.

## Plan

Implement https://github.com/openclaw/openclaw/issues/8321 as an additive per-cron-job context pruning override. Use a top-level `contextPruning?: AgentContextPruningConfig` field on cron jobs, not a separate `disableCache` boolean unless a maintainer redirects. Thread the field through cron shared/types, normalization/persistence, gateway TypeBox schemas for cron add/update/list, CLI add/edit/show/list surfaces as appropriate, Control UI types/controllers/forms, agent cron tool schema/recovery, and docs. In isolated cron execution, merge a job-level override into the effective `cfgWithAgentDefaults.agents.defaults.contextPruning` before invoking the embedded runner, while preserving existing global/per-agent behavior when the job field is absent. Add focused tests proving a job-level `mode: "off"` overrides global/per-agent `mode: "cache-ttl"` for an isolated agent-turn cron job and proving add/update round-trips. Do not implement direct script/no-LLM cron execution here; that is separate work tracked at https://github.com/openclaw/openclaw/issues/18160. Add a user-facing changelog entry.
