# Coding Plan for openclaw/openclaw#8321: Feature: Per-job cache control for cron jobs

Source report: [records/openclaw-openclaw/items/8321.md](records/openclaw-openclaw/items/8321.md)

## Summary

Keep open: current main still has only payload-level `lightContext` for cron jobs, while cache/context pruning remains driven by agent defaults. The earlier implementation PR is closed unmerged, and the related direct-script cron work is complementary rather than a replacement for per-job pruning on agent-turn cron jobs.

## Plan

Implement per-cron-job context-pruning control for agent-turn cron jobs. Add an additive top-level `contextPruning?: AgentContextPruningConfig` field using the existing `mode: "off" | "cache-ttl"` contract, not a new `disableCache` boolean. Thread it through cron create/update/list types, normalization, persisted jobs, gateway protocol schemas, CLI add/edit/show/list paths, Control UI cron forms/types/controllers, the agent cron tool schema, docs, and CHANGELOG.md. In isolated cron execution, merge the job-level override into the effective `cfgWithAgentDefaults` before the embedded runner receives config, and add regression coverage proving a job-level `mode: "off"` overrides global or per-agent `agents.defaults.contextPruning.mode: "cache-ttl"`. Do not implement the separate direct script/no-LLM cron feature or the per-job elevated exec scoping feature in this PR.
