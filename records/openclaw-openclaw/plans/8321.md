# Coding Plan for openclaw/openclaw#8321: Feature: Per-job cache control for cron jobs

Source report: [records/openclaw-openclaw/items/8321.md](records/openclaw-openclaw/items/8321.md)

## Summary

Keep open: current main and the latest release still lack per-cron-job cache/context-pruning control, the prior implementation PR is closed unmerged, and the direct-exec cron discussion is complementary rather than a replacement for agent-run cron jobs.

## Plan

Implement issue https://github.com/openclaw/openclaw/issues/8321 as an additive per-cron-job context-pruning override. Add top-level `contextPruning?: AgentContextPruningConfig` to cron job create/update/list/storage/protocol surfaces using the existing `mode: "off" | "cache-ttl"` contract; avoid adding a separate `disableCache` boolean unless a maintainer explicitly redirects. Thread the field through cron normalization, persistence, Gateway schemas, CLI add/edit, Control UI types/controllers/views, docs, and the agent cron tool schema. In isolated cron execution, merge the job override into the effective `cfgWithAgentDefaults.agents.defaults.contextPruning` before the embedded runner builds context-pruning behavior. Add regression coverage proving a job-level `mode: "off"` overrides global or per-agent `agents.defaults.contextPruning.mode: "cache-ttl"`; do not implement direct exec/script cron behavior here because that is tracked separately. Add a changelog entry during landing because this is a user-facing config/API feature.
