# Coding Plan for openclaw/openclaw#8321: Feature: Per-job cache control for cron jobs

Source report: [records/openclaw-openclaw/items/8321.md](records/openclaw-openclaw/items/8321.md)

## Summary

Keep open: current main still lacks per-cron-job `contextPruning` or `disableCache`, the closed implementation attempt at https://github.com/openclaw/openclaw/pull/8341 was not merged, and isolated cron still derives pruning from effective agent defaults.

## Plan

Implement per-job cron context-pruning control for https://github.com/openclaw/openclaw/issues/8321. Add an additive top-level `contextPruning?: AgentContextPruningConfig` field using the existing `mode: "off" | "cache-ttl"` contract, not a separate `disableCache` boolean or new `disabled` mode. Thread it through cron types, persisted/shared base, normalization/create/update/patch merge, gateway TypeBox schemas and generated protocol artifacts, cron CLI add/edit, agent cron tool schema/recovery, Control UI form/types, and public docs. In isolated cron, merge the job-level override into the effective `cfgWithAgentDefaults` before invoking the embedded runner so `buildContextPruningFactory` sees the job override. Add regression coverage proving a job-level `mode: "off"` overrides global or per-agent `agents.defaults.contextPruning.mode: "cache-ttl"`, while preserving existing `lightContext` behavior. Do not fold this into direct script/no-LLM cron execution; that is a complementary path tracked elsewhere. Add a user-facing changelog entry.
