## Summary

Keep open: current main still lacks per-cron-job cache/context-pruning control, the prior implementation attempt is closed unmerged, and the isolated runner still derives pruning from effective agent defaults.

## Plan

Implement per-job cron context pruning for https://github.com/openclaw/openclaw/issues/8321. Add an optional top-level `contextPruning?: AgentContextPruningConfig` cron job field using the existing `mode: "off" | "cache-ttl"` contract; do not add a separate `disableCache` boolean and do not implement direct exec/script mode here. Thread the field through cron types, shared UI types, normalization/persistence/update handling, Gateway TypeBox schemas and generated protocol clients as needed, CLI add/edit/show/list surfaces, Control UI add/edit/clone forms, the agent cron tool schema, docs, and changelog. In isolated cron, merge a job-level override into the effective agent defaults before embedded runner/model/pruning setup so `mode: "off"` overrides global/per-agent `cache-ttl` for that job only. Add focused regression coverage proving create/update round-trip and a job-level `mode: "off"` disables pruning when global defaults use `mode: "cache-ttl"`, while jobs without the field keep existing defaults.
