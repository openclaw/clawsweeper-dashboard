# Coding Plan for openclaw/openclaw#8321: Feature: Per-job cache control for cron jobs

Source report: [records/openclaw-openclaw/items/8321.md](records/openclaw-openclaw/items/8321.md)

## Summary

Keep open: current main still lacks a per-cron-job cache/context-pruning override, the matching implementation PR was closed unmerged, and direct-exec cron work is complementary rather than a replacement for agent-run cron jobs.

## Plan

Implement per-cron-job contextPruning overrides for agent-run cron jobs. Add a top-level contextPruning?: AgentContextPruningConfig field to cron job create/patch/list/storage types using the existing mode: "off" | "cache-ttl" contract; thread it through normalization, persistence, Gateway TypeBox schemas, cron tool schema/recovery, CLI add/edit, Control UI form/state/types, docs, and tests. In isolated cron execution, merge the job-level override into the effective agent defaults before cfgWithAgentDefaults reaches model selection and embedded/CLI runner paths. Add regression coverage proving a job-level mode "off" overrides global/per-agent mode "cache-ttl" for an isolated agentTurn job, while unchanged jobs still inherit defaults. Do not implement direct exec/script payloads or per-job elevated trust in this PR; those are separate trackers. Include a user-facing changelog entry if this is promoted for landing.
