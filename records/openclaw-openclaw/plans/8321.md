# Coding Plan for openclaw/openclaw#8321: Feature: Per-job cache control for cron jobs

Source report: [records/openclaw-openclaw/items/8321.md](records/openclaw-openclaw/items/8321.md)

## Summary

Keep open: current main still lacks per-cron-job `contextPruning` or `disableCache`, the gateway API would reject those fields, and the isolated runner still derives pruning from effective agent defaults.

## Plan

Implement per-job cron context pruning for https://github.com/openclaw/openclaw/issues/8321. Add an additive top-level `contextPruning?: AgentContextPruningConfig` cron job override, not a one-off `disableCache` boolean, using the existing `mode: "off" | "cache-ttl"` contract. Thread it through cron types/shared persistence, normalization, service create/update/list round-tripping, gateway protocol schemas and generated artifacts, CLI add/edit/list/show surfaces, Control UI cron types/controllers/form display, agent cron tool schemas, docs, and CHANGELOG.md. In isolated cron execution, merge `input.job.contextPruning` into the effective per-run agent defaults before creating/using `cfgWithAgentDefaults` so job-level `mode: "off"` overrides global or per-agent `mode: "cache-ttl"`. Do not conflate this with direct exec/script cron mode and do not add owner-specific policy. Add regression coverage proving create/update round-trip and runtime override behavior.
