# Coding Plan for openclaw/openclaw#8321: Feature: Per-job cache control for cron jobs

Source report: [records/openclaw-openclaw/items/8321.md](records/openclaw-openclaw/items/8321.md)

## Summary

Keep open: current main still lacks a per-cron-job cache/context-pruning override. The cron contract, protocol schema, CLI/UI surfaces, and isolated runner only expose payload-level `lightContext` while pruning remains driven by agent defaults; the earlier fixing PR is closed unmerged, and direct script execution is complementary rather than a replacement.

## Plan

Implement https://github.com/openclaw/openclaw/issues/8321 as a focused feature PR. Add an additive top-level `contextPruning?: AgentContextPruningConfig` cron job override using the existing `mode: "off" | "cache-ttl"` contract, not a separate `disableCache` boolean. Thread it through cron types, normalization/persistence, gateway protocol schemas/validators, CLI add/edit/show/list JSON behavior, Control UI cron form/types/controllers, agent cron tool schema, docs, and changelog. In isolated cron execution, merge the job override into the effective `cfgWithAgentDefaults.agents.defaults.contextPruning` before the embedded runner/model path uses the config, while leaving jobs without the field on existing global/per-agent defaults. Add regression coverage proving a job-level `mode: "off"` overrides global or per-agent `agents.defaults.contextPruning.mode: "cache-ttl"` for that cron run only. Do not implement direct exec/script cron mode or per-job elevated scoping in this PR; those are separate issues.
