# Coding Plan for openclaw/openclaw#8321: Feature: Per-job cache control for cron jobs

Source report: [records/openclaw-openclaw/items/8321.md](records/openclaw-openclaw/items/8321.md)

## Summary

Keep open: current main still lacks per-cron-job `contextPruning` or `disableCache`, the closed implementation attempt was not merged, and isolated cron still derives pruning from the effective agent defaults.

## Plan

Implement https://github.com/openclaw/openclaw/issues/8321 as a narrow additive cron feature. Add a top-level `contextPruning?: AgentContextPruningConfig` field, not a one-off `disableCache` boolean, to cron job create/update/list/persisted shapes; validate it with the existing `mode: "off" | "cache-ttl"` contract; thread it through gateway protocol schemas/generated clients, CLI add/edit, Control UI types/controller/form, and the agent cron tool schemas; merge the per-job value into the effective `cfgWithAgentDefaults.agents.defaults.contextPruning` before isolated cron calls the embedded runner. Add regression coverage proving a job-level `mode: "off"` overrides global or per-agent `agents.defaults.contextPruning.mode: "cache-ttl"` for that isolated cron run while leaving other jobs/defaults unchanged. Update docs and add a user-facing CHANGELOG entry. Do not implement direct exec/script cron mode or per-job elevated scoping in this PR; those are separate features.
