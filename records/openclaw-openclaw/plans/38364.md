## Summary

Keep open. Current main still lacks a structured reply-target path from Control UI or Gateway chat sends into agent-visible context; the silent/no-output display side is partly covered, but the central reply metadata request remains unresolved.

## Plan

Fix https://github.com/openclaw/openclaw/issues/38364 with a narrow additive implementation. Add a backward-compatible structured reply-target field to the Gateway chat.send and sessions.send protocol schemas, forward it from sessions.send to chat.send, and populate the existing MsgContext reply fields such as ReplyToId, ReplyToIdFull, ReplyToBody, ReplyToSender, and ReplyToIsQuote when a Control UI reply target is supplied. Update Control UI chat send plumbing to carry a sanitized selected parent message id/body/sender summary where available, and coordinate with https://github.com/openclaw/openclaw/issues/16896 so UI-only quote replies and structured metadata do not diverge. Preserve the current NO_REPLY/silent-output suppression behavior; do not reintroduce generic success/no-text assistant messages. Add focused regression coverage and a user-facing CHANGELOG.md entry if behavior changes are visible.
