Source report: [records/openclaw-openclaw/items/59164.md](records/openclaw-openclaw/items/59164.md)

## Summary

Keep open. Current `main` still routes plugin `api.runtime.subagent.run()` through the generic Gateway `agent` path, which creates/finalizes a CLI task but does not register a `SubagentRunRecord`; the `subagent_ended` hook remains tied to registry entries created by the normal subagent spawn paths.

## Plan

Fix https://github.com/openclaw/openclaw/issues/59164. Current `api.runtime.subagent.run()` in `src/gateway/server-plugins.ts` dispatches Gateway method `agent`, and `src/gateway/server-methods/agent.ts` tracks that as a CLI task without registering a `SubagentRunRecord`, so plugin `api.on("subagent_ended")` handlers never fire. Add a narrow bridge so plugin-created subagent runs enter the existing subagent registry completion lifecycle and emit `subagent_ended` once when the run ends. Keep scope tight: do not implement the broader automatic external deferred-delivery semantics from https://github.com/openclaw/openclaw/issues/55517 unless the existing ended-hook lifecycle requires a field; prefer `expectsCompletionMessage: false` or equivalent if needed to avoid unexpected announcement delivery. Add regression coverage proving a plugin runtime `subagent.run()` completion invokes a registered `subagent_ended` hook. Likely files: `src/gateway/server-plugins.ts`, `src/gateway/server-methods/agent.ts`, `src/agents/subagent-registry.ts`, `src/agents/subagent-registry-completion.ts`, and tests under `src/gateway/`, `src/plugins/runtime/`, or `src/agents/`.
