## Summary

Keep open. Current main and v2026.5.19 still route plugin-runtime subagent runs through the generic Gateway agent method, so no `SubagentRunRecord` exists for the registry lifecycle that emits `subagent_ended`.

## Plan

Fix the plugin SDK subagent lifecycle gap reported in https://github.com/openclaw/openclaw/issues/59164. Current `api.runtime.subagent.run()` dispatches Gateway `agent` and returns a run id, but the run is never registered with the subagent registry, so `api.on("subagent_ended")` handlers do not fire. Add the narrow bridge needed for plugin-runtime subagent runs to emit `subagent_ended` on completion, preserving existing `deliver` and final completion delivery defaults; do not solve the broader `subagent_delivery_target` or automatic external delivery semantics from https://github.com/openclaw/openclaw/issues/55517 in this PR. Add regression coverage showing a plugin runtime subagent run registers enough lifecycle metadata and calls the ended hook once on ok/error completion, while existing `sessions_spawn` behavior and generic Gateway `agent` task tracking remain compatible. Include a user-facing changelog fix entry unless maintainers classify the change as internal-only.
