# Coding Plan for openclaw/openclaw#59164: subagent_ended hook does not fire for runs created via api.runtime.subagent.run()

Source report: [records/openclaw-openclaw/items/59164.md](records/openclaw-openclaw/items/59164.md)

## Summary

Keep open. Current main still routes plugin `api.runtime.subagent.run()` through the generic Gateway `agent` path without registering a `SubagentRunRecord`, while `subagent_ended` is emitted only from registry-tracked subagent records; the broader delivery policy remains tracked separately at https://github.com/openclaw/openclaw/issues/55517.

## Plan

Fix the plugin SDK lifecycle bug where runs started with `api.runtime.subagent.run()` never fire `api.on("subagent_ended", ...)`. Keep the scope narrow: register plugin-runtime subagent runs, or introduce a small shared registration wrapper, so the existing subagent registry can emit `subagent_ended` when the run finishes. Do not enable automatic final external delivery or change `subagent_delivery_target` semantics; leave that broader contract to https://github.com/openclaw/openclaw/issues/55517. Add focused regression coverage for a plugin runtime run that returns a run id, completes, and emits `subagent_ended`; preserve existing behavior for `sessions_spawn`, direct subagent spawn, idempotency keys, plugin owner tagging, and `deliver: false`. Add a changelog entry if the fix is user-visible to plugin authors.
