# Coding Plan for openclaw/openclaw#59164: subagent_ended hook does not fire for runs created via api.runtime.subagent.run()

## Summary

Keep open. Current main still routes plugin `api.runtime.subagent.run()` through the generic Gateway `agent` path without creating a `SubagentRunRecord`, while `subagent_ended` is emitted only from registry-tracked subagent records; the broader final-delivery behavior remains separately tracked at https://github.com/openclaw/openclaw/issues/55517.

## Plan

Fix https://github.com/openclaw/openclaw/issues/59164 only. Current `api.runtime.subagent.run()` in `src/gateway/server-plugins.ts` dispatches Gateway `agent` and returns a `runId`, while `src/gateway/server-methods/agent.ts` only creates a generic running task and never registers a `SubagentRunRecord`; as a result `subagent_ended` hooks cannot fire. Add a narrow bridge so plugin-runtime subagent runs are registered with the existing subagent registry and emit `subagent_ended` on completion. Preserve current provider/model override authorization and plugin-owned session cleanup semantics. Do not implement automatic final external delivery or broaden `subagent_delivery_target`; keep that product behavior separate under https://github.com/openclaw/openclaw/issues/55517. Add focused regression coverage for plugin runtime run registration and ended-hook emission, plus coverage that the narrow fix does not implicitly enable final completion delivery. Add a changelog fix entry if the repo's active changelog policy expects one for a user-visible plugin SDK bug fix.
