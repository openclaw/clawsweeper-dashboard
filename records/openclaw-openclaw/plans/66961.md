## Summary

Keep open. Current main still has a source-reproducible path where an unresolved custom provider with custom models can be serialized into generated `models.json`, and session model admission still reports catalog-visible but non-allowlisted models with a terse `model not allowed` error.

## Plan

Fix https://github.com/openclaw/openclaw/issues/66961. Keep `agents.defaults.models` as the session/subagent allowlist; do not make `models list --all` imply runtime admission. In `models.json` planning, omit or otherwise neutralize non-built-in custom provider rows that have custom `models` but no resolvable `apiKey` so the pinned Pi registry can still load unrelated valid providers. Preserve existing SecretRef marker behavior and local/synthetic provider auth behavior. Improve the `sessions.patch`/subagent model rejection path so a catalog-visible but disallowed model explains that `agents.defaults.models` is the allowlist and that `models list --all` is diagnostic. Add focused regression coverage for an unresolved custom provider poisoning generated `models.json` and for the clearer session model error; verify exact Codex provider spelling before adding expectations. Add a `CHANGELOG.md` fix entry if the final change is user-visible under repo policy.
