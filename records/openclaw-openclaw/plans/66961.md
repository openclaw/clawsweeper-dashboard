## Summary

Keep open. Current main still has both implicated paths: generated models.json can serialize unresolved non-built-in custom providers with models but no apiKey, and session model admission still reports a terse allowlist rejection for otherwise catalog-visible model refs.

## Plan

Fix https://github.com/openclaw/openclaw/issues/66961. Keep the existing agents.defaults.models allowlist policy. In models.json generation, skip or otherwise neutralize only non-built-in custom-provider rows that define models but still have no resolvable apiKey before serializing the generated models.json, matching @earendil-works/pi-coding-agent 0.74.0 ModelRegistry validation. Do not drop valid built-in provider overrides, provider rows without custom models, or source-managed secret markers. Also improve session/subagent model rejection so a catalog-visible but non-allowlisted model reports that it is blocked by agents.defaults.models and gives actionable add/clear/provider/* guidance; do not automatically admit every models.list --all row. Add focused regression coverage for unresolved custom providers and the session/subagent allowlist error. Add a user-facing changelog fix entry if this repair branch is landed by maintainers.
