## Summary

Keep open. Current main still only accepts typing indicator mode/cadence at `session` and `agents.defaults`; strict `agents.list[]` schema, generated types, agent resolution, reply typing, and heartbeat typing still lack per-agent `typingMode` and `typingIntervalSeconds` support.

## Plan

Implement per-agent typing mode/cadence support for https://github.com/openclaw/openclaw/issues/62725. Add `typingMode` and `typingIntervalSeconds` to the `agents.list[]` config contract and strict Zod schema, thread them through `resolveAgentConfig`, and resolve typing precedence as `session` over selected agent over `agents.defaults`. Update reply typing and heartbeat typing paths so the selected agent can override defaults without changing existing global/session behavior. Add focused regression coverage for schema acceptance/rejection, resolved agent config precedence, reply typing mode/cadence, and heartbeat typing behavior. Update user-facing docs and generated config/help/schema as needed, and add a changelog entry because this is user-facing config surface. Do not implement or rename `typingTtlSeconds`, and avoid conflating this with the adjacent TTL PR at https://github.com/openclaw/openclaw/pull/72009.
