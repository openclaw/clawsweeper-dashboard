## Summary

Keep open. Current main still exposes `typingMode` and `typingIntervalSeconds` only through session/default agent config; `agents.list[]` remains strict without those keys, and reply plus heartbeat typing paths do not resolve per-agent values.

## Plan

Implement per-agent `typingMode` and `typingIntervalSeconds` support for `agents.list[]`. Add the fields to `AgentConfig`, `AgentEntrySchema`, and `ResolvedAgentConfig`; resolve selected-agent typing values with precedence `session` over `agents.list[]` over `agents.defaults`; update reply typing cadence/mode and heartbeat typing paths without changing unrelated TTL behavior; document the new precedence in typing/config docs; add a user-facing changelog entry; and stop if the implementation needs a broader product decision or changes unrelated typing TTL semantics. Be aware of adjacent TTL work in https://github.com/openclaw/openclaw/pull/72009.
