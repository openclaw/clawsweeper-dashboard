## Summary

Keep open. Current main and the latest shipped release still limit `typingMode` and `typingIntervalSeconds` to `agents.defaults` or `session`, while the strict `agents.list[]` schema rejects the requested per-agent keys; the related TTL PR at https://github.com/openclaw/openclaw/pull/72009 does not supersede this per-agent mode/cadence request.

## Plan

Fix https://github.com/openclaw/openclaw/issues/62725 by adding per-agent `typingMode` and `typingIntervalSeconds` support for `agents.list[]`. Update the per-agent schema/types/resolved config so `agents.list[]` accepts the fields, preserve existing `session` and `agents.defaults` behavior, and apply precedence as `session` over selected agent over defaults. Thread the selected agent typing config through both normal reply typing and heartbeat typing; do not implement `typingTtlSeconds` or change TTL defaults, because that is covered by https://github.com/openclaw/openclaw/pull/72009.
