# Coding Plan for openclaw/openclaw#53932: [Feature]: Disable image optimization when images are uploaded to Slack

Source report: [records/openclaw-openclaw/items/53932.md](records/openclaw-openclaw/items/53932.md)

## Summary

Keep open: current main and the latest release still do not expose or pass a Slack-owned image optimization opt-out. The related implementation attempt was closed unmerged and only added an inert stub, while Slack uploads still call the shared media loader on its default optimizing path.

## Plan

Implement a default-compatible Slack upload image optimization opt-out for this issue. Add a Slack config field for preserving original image uploads or disabling image optimization, with top-level and per-account behavior consistent with existing Slack config patterns. Thread the resolved value through `extensions/slack/src/send.ts` into `loadOutboundMediaFromUrl`, and extend `src/plugin-sdk/outbound-media.ts` so plugin callers can pass `optimizeImages` through to `buildOutboundMediaLoadOptions`. Keep the default behavior unchanged, do not bypass `loadOutboundMediaFromUrl`, and preserve existing `mediaMaxMb`, local media read, and SSRF protections. Add focused tests for default optimized behavior and opt-out preserve-original behavior, update Slack config schema/docs, and add a user-facing changelog entry.
