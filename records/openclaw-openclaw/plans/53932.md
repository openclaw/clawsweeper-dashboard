# Coding Plan for openclaw/openclaw#53932: [Feature]: Disable image optimization when images are uploaded to Slack

Source report: [records/openclaw-openclaw/items/53932.md](records/openclaw-openclaw/items/53932.md)

## Summary

Keep open: current main and v2026.5.18 still route Slack media uploads through the shared image-optimizing loader with no Slack config or plugin-SDK option to disable optimization. The newer workaround discussion is useful, but source inspection shows `message(action="upload-file")` still reaches the same Slack send/upload path, and the only related Slack opt-out PR was closed unmerged with inert code.

## Plan

Implement a default-compatible Slack upload image optimization opt-out for https://github.com/openclaw/openclaw/issues/53932. Add a Slack config setting to preserve original uploaded image bytes, defaulting to current optimized behavior; thread it through Slack config types/schema/account resolution, `extensions/slack/src/send.ts`, and `src/plugin-sdk/outbound-media.ts` into the existing `src/media/load-options.ts` / `src/media/web-media.ts` `optimizeImages` option. Do not bypass `loadOutboundMediaFromUrl`, SSRF guards, media size caps, local-root/read-file protections, or change default Slack upload behavior. Add focused tests for default optimized uploads and opt-out raw uploads, update Slack docs/config docs and changelog if user-visible, and treat https://github.com/openclaw/openclaw/pull/40829 as adjacent generic PNG-preservation work rather than a replacement for the Slack config opt-out.
