# Coding Plan for openclaw/openclaw#53932: [Feature]: Disable image optimization when images are uploaded to Slack

Source report: [records/openclaw-openclaw/items/53932.md](records/openclaw-openclaw/items/53932.md)

## Summary

Keep open: current main and v2026.5.12 still route Slack media uploads, including `upload-file`, through the shared outbound media loader with default image optimization and no Slack-owned opt-out. The closed related PR only added an unused config stub, and the newer workaround note is not supported by the current `upload-file` source path.

## Plan

Implement a default-compatible Slack image upload optimization opt-out for this issue. Current Slack `sendMessage`/`upload-file` media sends flow through `extensions/slack/src/action-runtime.ts` -> `extensions/slack/src/actions.ts` -> `extensions/slack/src/send.ts` -> `loadOutboundMediaFromUrl`, while `src/plugin-sdk/outbound-media.ts` cannot pass `optimizeImages` even though `src/media/load-options.ts` and `src/media/web-media.ts` already support it. Add a Slack root/account config option with a clear default that preserves existing optimized behavior, add schema/types/UI hints/docs, thread the resolved setting into `uploadSlackFile`, extend `OutboundMediaLoadOptions` to include and forward `optimizeImages`, and test both default optimized uploads and preserve-original uploads. Do not bypass the shared loader, weaken max-size/local-read/SSRF protections, or implement a broad agent policy that forces message-tool routing unless a maintainer explicitly asks for that separate product change. Add a changelog entry if the resulting change is user-facing.
