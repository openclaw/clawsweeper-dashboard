## Summary

Keep open: current main and the latest release still route Slack media uploads through the shared image-optimizing media loader with no Slack-owned opt-out. The linked implementation attempt is closed unmerged and inert, and the newer workaround comment is useful context but does not match current source because Slack `upload-file` still calls the same default optimizing loader path.

## Plan
