# Coding Plan for openclaw/openclaw#55686: [Feature Request] Make command options clickable buttons in Feishu (e.g., /reasoning)

Source report: [records/openclaw-openclaw/items/55686.md](records/openclaw-openclaw/items/55686.md)

## Summary

Keep open: current main and the latest release still render `/reasoning` option choices as plain `Options:` text in Feishu. Feishu has native card/button delivery plumbing, and Slack already uses the shared reply-transform seam for this pattern, but Feishu does not yet transform command-option replies into clickable buttons; the related Feishu callback-field PR is adjacent and does not implement this UI.

## Plan

Implement clickable Feishu command-option buttons for simple status/menu replies such as `/reasoning` without changing core directive text or Slack behavior. Keep the fix Feishu-owned, preferably through `messaging.transformReplyPayload` and existing `ReplyPayload` interactive/presentation contracts, then let Feishu outbound render native card buttons with structured interaction envelopes. Do not execute arbitrary assistant text: only convert tightly validated simple trailing `Options:` lists from command/menu replies, and leave complex option lines as plain text. Preserve plain text fallback, avoid adding a config knob, add focused Feishu transform/outbound/dispatcher coverage, and add a user-facing changelog entry. Treat https://github.com/openclaw/openclaw/pull/42908 as adjacent callback parsing context, not the implementation for this issue, and avoid conflicting with it.
