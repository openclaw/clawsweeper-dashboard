## Summary

Keep open: current main still emits command-option replies such as `/reasoning` as plain `Options:` text, and Feishu's normal reply path does not transform those lines into clickable card buttons. Existing Feishu presentation-button and quick-action plumbing makes this a narrow feature candidate, while the adjacent callback-field work at https://github.com/openclaw/openclaw/pull/42908 does not implement this UX path.

## Plan

Fix https://github.com/openclaw/openclaw/issues/55686 by making Feishu render validated command-option replies such as `/reasoning` as clickable card buttons. Reuse the shared message presentation or channel-owned interactive reply transform contract plus the existing Feishu `ocf1` quick-action callback path; do not add a broad parser that turns arbitrary agent-authored `Options:` text into executable commands. Preserve plain text fallback for unsupported contexts and keep streaming, threaded replies, media replies, and duplicate-final behavior intact. Coordinate with, but do not fold in, the callback-field work at https://github.com/openclaw/openclaw/issues/42754 and https://github.com/openclaw/openclaw/pull/42908 unless strictly necessary. Add focused tests for `/reasoning` and at least one other simple command-option reply through Feishu rendering/callback behavior, plus a user-facing changelog entry if the behavior changes.
