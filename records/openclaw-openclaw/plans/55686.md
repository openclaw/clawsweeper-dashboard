# Coding Plan for openclaw/openclaw#55686: [Feature Request] Make command options clickable buttons in Feishu (e.g., /reasoning)

Source report: [records/openclaw-openclaw/items/55686.md](records/openclaw-openclaw/items/55686.md)

## Summary

Keep open: current main still has the Feishu-specific UX gap. `/reasoning` has shared `argsMenu: "auto"`, but Feishu outbound does not normalize the plain `Options:` reply into presentation buttons the way Slack does, and the latest release still has the same behavior.

## Plan

Fix the Feishu command-options UX tracked by https://github.com/openclaw/openclaw/issues/55686. Add a Feishu-owned outbound normalization path that detects safe/simple trailing command option replies such as `Current reasoning level: off.\nOptions: on, off, stream.` and converts them into `presentation` buttons whose values apply the selected command option, while preserving the plain text fallback and avoiding arbitrary executable actions from untrusted free-form text. Keep the change inside the Feishu plugin/outbound presentation boundary unless a small shared helper is clearly warranted; do not change `/verbose` policy or the separate Feishu callback-field contract tracked by https://github.com/openclaw/openclaw/issues/42754. Add focused tests for Feishu rendering and fallback, keep Slack behavior unchanged, and include a user-facing changelog entry if maintainers want this queued as a release-facing feature.
