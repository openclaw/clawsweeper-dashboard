# Coding Plan for openclaw/openclaw#55686: [Feature Request] Make command options clickable buttons in Feishu (e.g., /reasoning)

Source report: [records/openclaw-openclaw/items/55686.md](records/openclaw-openclaw/items/55686.md)

## Summary

Keep open: current main still has a Feishu-specific gap for clickable command options. `/reasoning` now has shared `argsMenu: "auto"`, but Feishu does not consume `resolveCommandArgMenu`; the no-arg command reply still produces a plain `Options: on, off, stream.` line, while Feishu only renders buttons when a payload already contains presentation blocks.

## Plan

Implement clickable Feishu command-option buttons for trusted simple slash-command option replies such as `/reasoning`. Prefer a Feishu-owned outbound `normalizePayload` path mirroring Slack's simple `Options:` transform, or a tiny shared helper only if it keeps channel ownership clean. Preserve the plain text fallback, do not change command semantics, do not claim to fix Feishu `option`/`options`/`form_value` callback extraction, and keep comment/document targets that cannot render cards falling back to text. Add focused tests for Feishu outbound card rendering and callback dispatch from generated buttons, plus regression coverage that plain/complex Options text stays text-only.
