## Summary

Keep open. Current main still has a provider-id canonicalization contract, but the Usage/Control UI path continues to derive visible labels, provider filters, query suggestions, query matching, CSV export values, and aggregate keys from raw provider strings, so supported aliases such as `z.ai`/`z-ai` and `aws-bedrock` can still display or group separately from `zai` and `amazon-bedrock`.

## Plan

Fix https://github.com/openclaw/openclaw/issues/47840 by canonicalizing supported provider aliases in the Usage/Control UI derived surfaces.
