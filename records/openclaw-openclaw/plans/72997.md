## Summary

Keep open. Current main has automatic config-driven channel hot reload and separate start/stop/logout RPCs, but it still lacks the requested one-step manual CLI/Gateway reload surface for a single channel.

## Plan

Implement a targeted manual channel reload surface for this issue. Add an admin-scoped `channels.reload` Gateway RPC plus `openclaw channels reload --channel <name> [--account <id>]` (or the repo's closest existing argument style) that disconnects and reconnects only the selected channel/account by composing the existing `stopChannel`/`startChannel` lifecycle. Preserve current config-driven hot reload behavior, active-work deferral semantics, secrets/reload contracts, and provider/plugin boundaries; do not add provider-specific Telegram or WhatsApp policy. Update protocol schemas/types, method descriptors/scopes, CLI routing/help, docs, regression tests, and a user-facing changelog entry. Check overlap with the adjacent account-scoped and deferred-reload PRs before editing.
