## Summary

Keep open. Current main still lacks the requested one-step manual channel reload CLI command or Gateway RPC; the existing config-driven hot reload and health-monitor restart paths are automatic lifecycle behavior, not an operator surface.

## Plan

Implement the operator-facing channel reload feature for https://github.com/openclaw/openclaw/issues/72997. Add an additive admin-scoped Gateway RPC, preferably `channels.reload`, with `{ channel, accountId? }` params and a result that reports the channel/account and stopped/started or reloaded state. Add `openclaw channels reload --channel <name> [--account <id>]` using the existing plural channels CLI style. Reuse the existing channel stop/start lifecycle and account resolution from `channels.start` and `channels.stop`; do not add provider-specific policy, a new config option, or a full Gateway restart shortcut. Preserve existing active-work deferral/restart fencing semantics from the automatic reload path where applicable. Update protocol schema/types/validators/method list/admin scopes, docs, focused tests, and a changelog entry because this is user-facing.
