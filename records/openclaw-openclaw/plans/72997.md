# Coding Plan for openclaw/openclaw#72997: Feature request: channel reload <provider> command

Source report: [records/openclaw-openclaw/items/72997.md](records/openclaw-openclaw/items/72997.md)

## Summary

Keep open. Current main still lacks the requested one-step manual channel reload CLI command or Gateway RPC; the existing automatic config/secrets-driven channel restarts and the earlier closed config-hot-reload report do not provide this operator surface.

## Plan

Implement the manual channel reload surface requested in https://github.com/openclaw/openclaw/issues/72997. Add an additive admin-scoped `channels.reload` Gateway RPC that reloads one channel/account by stopping and starting only that channel target, then expose it as `openclaw channels reload --channel <name> [--account <id>]` using the existing plural `channels` command style. Reuse the existing channel lifecycle helpers where possible; preserve admin authorization, existing active-work safety/deferral semantics where applicable, and existing start/stop/logout behavior. Do not add a new config option and do not attempt to solve watcher debounce, gateway overload, or deferred-reload race issues except where directly necessary for the reload command. Update protocol schema/types/validators, method registry/scopes, handler tests, CLI routing/help/docs, and add a user-facing changelog entry if repo policy requires it.
