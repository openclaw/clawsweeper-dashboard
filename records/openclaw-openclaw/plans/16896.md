Keep open. Current main still lacks the requested Dashboard/Control UI webchat reply workflow: no message context menu, no Reply action in the bubble toolbar, no selected reply state or quote preview in the composer, and no UI send path for a selected reply target.

## Plan

Implement a narrow Control UI Dashboard/WebChat quote-reply feature for https://github.com/openclaw/openclaw/issues/16896. Add an accessible message-level Reply control plus a right-click menu in the chat message surface, track the selected message in chat/composer state, focus the textarea, show a dismissible quote preview, and prepend a safe Markdown quote block to the outgoing draft/message on send.
