## Summary

Keep open: current main has configurable provider timeouts, SDK Retry-After sleep caps, and a progress-aware idle watchdog, but it still does not carry provider Retry-After metadata through failover into OpenAI-compatible HTTP/SSE errors.

## Plan

Fix the remaining Retry-After propagation gap for https://github.com/openclaw/openclaw/issues/75648.
