## Plan

Implement the narrow lane-wait diagnostic threshold request from https://github.com/openclaw/openclaw/issues/14747. Add a documented positive-integer `diagnostics.laneWaitWarnMs` config field that defaults to the current 2,000 ms behavior when unset, applies to future queue entries without explicit overrides, and preserves all explicit `warnAfterMs` call-site overrides such as manual cron runs. Update schema/types/labels/help/generated docs, config reload planning/tests for no-restart future-entry behavior, queue docs, and focused regression coverage.
