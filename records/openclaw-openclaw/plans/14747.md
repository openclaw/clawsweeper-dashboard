Keep open. Current main still stores a hardcoded 2,000 ms default on queued entries, diagnostics config/docs do not expose a lane-wait threshold, the prior implementation PR is closed unmerged, and the broader session-concurrency tracker does not supersede this narrower logging-control request.

## Plan

Implement the narrow lane wait diagnostic threshold feature for this issue.
