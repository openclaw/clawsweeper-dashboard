## Summary

Keep open. Current main still hardcodes the queued-entry lane wait warning default, the diagnostics config and docs do not expose a lane-wait threshold, and the closed implementation attempts plus broader concurrency tracker do not supersede this narrower logging-control request.

## Plan

Implement a narrow config-backed lane wait diagnostic threshold for https://github.com/openclaw/openclaw/issues/14747. Add a positive integer millisecond diagnostics.laneWaitWarnMs config field that applies to future queue entries when opts.warnAfterMs is unset, preserves the existing 2,000 ms fallback when unset, and preserves explicit per-call overrides including the cron manual-run warnAfterMs: 5_000 path. Do not implement per-lane thresholds, adaptive timeouts, session concurrency, or auto-yield from https://github.com/openclaw/openclaw/issues/53023. Update schema/types/labels/help/generated config docs and queue docs, add focused tests for default fallback, configured value, config reload/future-entry behavior, and override precedence, and add a user-facing changelog entry.
