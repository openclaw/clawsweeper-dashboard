## Plan

Repair the Google Chat sessionThread PR without broadening the feature. Preserve the PR's intended default behavior unless the existing account config explicitly says flat replies. Add focused coverage in `extensions/googlechat/src/monitor.reply-delivery.test.ts` for the fallback being suppressed in off mode and for explicit `payload.replyToId` still winning. Keep broader streamed/follow-up/message-tool thread routing with https://github.com/openclaw/openclaw/pull/74235 and https://github.com/openclaw/openclaw/issues/69422; do not duplicate that architecture here.
