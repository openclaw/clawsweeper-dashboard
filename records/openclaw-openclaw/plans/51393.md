## Summary

Keep open: current main and the latest shipped release still parse managed browser evaluate input as a parenthesized expression, so statement bodies with declarations fail. The related browser exec/evaluate issue was closed with this issue explicitly left as the narrower follow-up, and the open force-click issue does not own this statement-body request.

## Plan

Fix https://github.com/openclaw/openclaw/issues/51393 by extending browser evaluate to accept JavaScript statement/function-body input such as `const x = 1; return x` while preserving current expression, function-source, IIFE, async promise, ref/page, batch, timeout, abort, and navigation/SSRF guard behavior. Keep the work inside the browser plugin contract and execution paths; do not add a separate browser exec command, do not change `browser.evaluateEnabled` defaults or security policy, and do not fold in the separate force-click request from https://github.com/openclaw/openclaw/issues/60381. Update focused tests for managed Playwright and existing-session Chrome MCP parity or an explicit limitation, update CLI/tool/docs wording if the public contract changes, and add a user-visible changelog entry if behavior changes.
