# Coding Plan for openclaw/openclaw#51393: Feature: browser tool evaluate should support statements (var/let/const)

Source report: [records/openclaw-openclaw/items/51393.md](records/openclaw-openclaw/items/51393.md)

## Summary

Keep open. Current main and the latest release still parse managed browser evaluate input as a parenthesized expression, so `var`, `let`, and `const` statement bodies fail while the reported IIFE workaround succeeds; the broader browser exec/evaluate request was closed with this narrower statement-body issue explicitly left as follow-up.

## Plan

Implement statement-body support for `browser act kind=evaluate` / `openclaw browser evaluate` for https://github.com/openclaw/openclaw/issues/51393. Preserve existing expression and function-source behavior, `ref` evaluation, returned values, async promise timeout bounds, `browser.evaluateEnabled`, abort/dialog handling, and navigation/SSRF guards. Treat statement bodies with `var`, `let`, `const`, and `return` naturally via an owned async wrapper or equivalent parser fallback; do not add a separate browser exec command or a new config option. Check existing-session Chrome MCP behavior explicitly and either implement parity or return/document a clear unsupported path. Update focused tests and user-facing docs, and add a changelog entry because this is a user-visible browser feature.
