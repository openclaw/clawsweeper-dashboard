## Summary

Keep open. Current main and the latest release still parse managed browser evaluate input as a parenthesized expression, so statement bodies with `var`, `let`, or `const` still require the reported IIFE workaround; the related browser exec/evaluate issue was closed while explicitly leaving this statement-body request as separate follow-up.

## Plan

Fix https://github.com/openclaw/openclaw/issues/51393 by adding compatibility-preserving statement-body support to browser evaluate. Keep the existing `kind: "evaluate"` surface and `browser.evaluateEnabled` gate; do not add a separate `browser.exec` command. Preserve current expression and function-source behavior, ref argument behavior, async promise timeout handling, abort/dialog cleanup, and navigation guard semantics. Cover both page and ref Playwright evaluate paths, and make an explicit existing-session Chrome MCP decision: either align statement support there or return/document a clear unsupported error instead of silently diverging. Update focused browser docs and add a changelog entry if the change is user-facing.
