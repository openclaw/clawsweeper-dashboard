# Coding Plan for openclaw/openclaw#51393: Feature: browser tool evaluate should support statements (var/let/const)

Source report: [records/openclaw-openclaw/items/51393.md](records/openclaw-openclaw/items/51393.md)

## Summary

Keep open. Current main and the latest release still parse managed browser evaluate input as a parenthesized expression, so statement bodies with `var`, `let`, or `const` still fail while the reported IIFE workaround succeeds; the closed browser exec/evaluate thread explicitly left this narrower request as follow-up.

## Plan

Implement statement-body support for `browser act kind=evaluate` in the browser plugin. Preserve existing function-source and expression behavior, including `(el) => ...` ref evaluation, direct expression results, IIFE compatibility, async promise timeout handling, `browser.evaluateEnabled`, and existing navigation/SSRF guards. Add statement-body handling for page and ref evaluate paths, and make existing-session Chrome MCP behavior either parity-tested or explicitly rejected with a clear contract. Update CLI/tool/docs wording and changelog for the user-visible feature. Do not add a separate `browser.exec` action and do not loosen the existing evaluate security gate.
