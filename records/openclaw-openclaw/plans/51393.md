## Summary

Keep open. Current main and the latest shipped release still parse managed browser evaluate input as a parenthesized expression, so declaration statements like `var`, `let`, and `const` remain unsupported without the reported IIFE workaround.

## Plan

Implement statement-body support for browser `act`/CLI evaluate in the browser plugin. Preserve existing expression and function-source behavior, return values, async promise handling, timeout/abort behavior, evaluateEnabled gating, target/ref behavior, navigation guard, and SSRF guard. Prefer shared parsing/fallback logic used by both page and ref evaluate paths; confirm whether existing-session Chrome MCP evaluate already supports statement bodies or document/handle parity. Add focused regression coverage for `var`, `let`, and `const` statement bodies plus existing expression/IIFE behavior. Update CLI/browser docs and CHANGELOG if the behavior is user-visible.
