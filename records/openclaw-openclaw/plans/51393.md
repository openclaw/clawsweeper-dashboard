# Coding Plan for openclaw/openclaw#51393: Feature: browser tool evaluate should support statements (var/let/const)

Source report: [records/openclaw-openclaw/items/51393.md](records/openclaw-openclaw/items/51393.md)

## Summary

Keep open: current main and the latest release still parse browser evaluate input as a parenthesized expression, so var/let/const statement bodies remain unsupported. The related browser exec/evaluate request at https://github.com/openclaw/openclaw/issues/55598 was closed with this narrower behavior explicitly left as follow-up, and no open PR or canonical duplicate now owns it.

## Plan

Implement statement/function-body support for browser act kind=evaluate in a compatibility-preserving way. Keep current expression/function/IIFE inputs working, keep browser.evaluateEnabled gating, preserve evaluate timeout/abort/navigation/SSRF/dialog behavior, and do not add a separate browser exec tool or new config option. Cover both page and ref evaluate paths, and either provide parity for the Chrome MCP existing-session evaluate path or document and test the explicit limitation. Update CLI/docs and add a user-facing changelog entry because this changes browser tool behavior.
