# Coding Plan for openclaw/openclaw#51393: Feature: browser tool evaluate should support statements (var/let/const)

Source report: [records/openclaw-openclaw/items/51393.md](records/openclaw-openclaw/items/51393.md)

## Summary

Keep open: current main and latest v2026.5.22 still parse browser evaluate input as a parenthesized expression, so `var`/`let`/`const` statement bodies still fail; no merged PR or canonical duplicate has taken over this narrower request.

## Plan

Fix https://github.com/openclaw/openclaw/issues/51393. Current browser evaluate parses `fn` with `eval("(" + fnBody + ")")`, so declaration/function-body inputs such as `const x = 1; return x` fail. Implement compatibility-preserving statement/function-body support for `browser act kind=evaluate` and `openclaw browser evaluate`: keep existing expression, function-source, IIFE, async promise, ref, batch, timeout, abort, dialog, navigation/SSRF guard, and `browser.evaluateEnabled` behavior. Cover managed Playwright page/ref evaluate and either provide tested existing-session Chrome MCP parity or explicitly test/document a limitation if Chrome MCP cannot support the same semantics. Do not add a separate `browser.exec`, do not implement force-click from the related issue, and do not change evaluate security defaults. Update CLI/docs wording and add a changelog entry for the user-visible browser tool improvement.
