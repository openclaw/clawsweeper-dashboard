# Coding Plan for openclaw/openclaw#51393: Feature: browser tool evaluate should support statements (var/let/const)

Source report: [records/openclaw-openclaw/items/51393.md](records/openclaw-openclaw/items/51393.md)

## Summary

Keep open. Current main and v2026.5.19 still parse managed browser evaluate input as a parenthesized expression, so statement bodies with `var`, `let`, or `const` still fail and the closed broader evaluate issue explicitly left this as a separate follow-up.

## Plan

Implement statement-body support for `browser act kind=evaluate` and `openclaw browser evaluate --fn` without breaking existing expression and function-source behavior. Keep the work inside the browser plugin: managed Playwright evaluate page/ref paths, batch dispatch, existing-session Chrome MCP behavior or explicit parity handling, action normalization/schema/types, CLI/docs, and focused regression tests. Preserve the existing `browser.evaluateEnabled` gate, SSRF/navigation guards, dialog abort behavior, timeout Promise.race behavior, and current IIFE/function-expression compatibility. Add a user-facing changelog entry because this expands browser evaluate behavior. Do not add a separate browser exec command or a new config option unless maintainers explicitly ask for it.
