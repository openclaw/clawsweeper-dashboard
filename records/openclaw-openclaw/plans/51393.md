# Coding Plan for openclaw/openclaw#51393: Feature: browser tool evaluate should support statements (var/let/const)

Source report: [records/openclaw-openclaw/items/51393.md](records/openclaw-openclaw/items/51393.md)

## Summary

Keep open: current main and v2026.5.20 still parse browser evaluate input as a parenthesized expression, so declaration statement bodies with var, let, or const fail. The broader browser exec/evaluate issue is closed as implemented but explicitly leaves this narrower statement-body request open, and the other related open issue tracks force-click behavior rather than this gap.

## Plan

Implement compatibility-preserving statement-body support for browser evaluate. Keep existing expression, function-source, async function, promise, and IIFE behavior working; add support for bodies such as `const x = 1; return x` for both page and ref targets. Do not add a separate browser.exec command, do not change browser.evaluateEnabled defaults, do not weaken SSRF/navigation guards, and keep timeout/abort behavior bounded. Update the browser action/CLI/tool docs only as needed, add focused regression coverage, and add a user-facing CHANGELOG entry because this changes browser automation behavior.
