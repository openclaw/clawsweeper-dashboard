# Coding Plan for openclaw/openclaw#51393: Feature: browser tool evaluate should support statements (var/let/const)

Source report: [records/openclaw-openclaw/items/51393.md](records/openclaw-openclaw/items/51393.md)

## Summary

Keep open: current main and v2026.5.22 still parse browser evaluate input as a parenthesized expression, so `var`/`let`/`const` statement bodies remain unsupported. The broader browser exec/evaluate request at https://github.com/openclaw/openclaw/issues/55598 was closed with this narrower behavior explicitly left as follow-up, and no open PR now owns it.

## Plan

Implement compatibility-preserving statement-body support for browser `act:evaluate` / `openclaw browser evaluate --fn`. Preserve existing expression, function-source, IIFE, async promise, return-value, timeout, abort, dialog, navigation, SSRF, and `browser.evaluateEnabled` behavior. Do not add a separate `browser.exec` command, do not change the evaluate default security posture, and do not work on force-click behavior from https://github.com/openclaw/openclaw/issues/60381. Cover managed Playwright evaluate and Chrome MCP existing-session evaluate parity, or document a deliberate limitation if parity is impossible. Update focused browser docs and add a maintainer-owned changelog entry if the change is user-visible.
