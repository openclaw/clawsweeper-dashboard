# Coding Plan for openclaw/openclaw#51393: Feature: browser tool evaluate should support statements (var/let/const)

Source report: [records/openclaw-openclaw/items/51393.md](records/openclaw-openclaw/items/51393.md)

## Summary

Keep open: current main and latest v2026.5.22 still parse browser evaluate input as a parenthesized expression, so declaration statements like var/let/const still fail; the related browser exec/evaluate request at https://github.com/openclaw/openclaw/issues/55598 explicitly left this narrower behavior as follow-up, and no open PR or canonical duplicate now owns it.

## Plan

Implement statement/function-body support for browser act kind=evaluate and openclaw browser evaluate. Preserve current expression/function/IIFE behavior, async promise handling, abort behavior, timeout Promise.race, dialog handling, navigation/SSRF guards, browser.evaluateEnabled gating, and existing security wording. Prefer a narrow compatibility helper in the browser plugin evaluate path rather than a new browser exec command or a new config option. Cover both page and ref evaluate; for Chrome MCP existing-session evaluate, either provide tested parity or document and test the explicit limitation. Update docs/CLI guidance and add a single-line user-facing changelog entry if behavior changes. Do not broaden evaluate defaults, remove the IIFE workaround, or weaken browser-control security guidance.
