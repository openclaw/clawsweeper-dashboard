# Coding Plan for openclaw/openclaw#51393: Feature: browser tool evaluate should support statements (var/let/const)

Source report: [records/openclaw-openclaw/items/51393.md](records/openclaw-openclaw/items/51393.md)

## Summary

Keep open: current main still evaluates browser input by parenthesizing the submitted source before `eval`, so declaration statements such as `var`, `let`, and `const` remain invalid. The closed browser exec/evaluate request at https://github.com/openclaw/openclaw/issues/55598 explicitly left this narrower statement-body behavior as follow-up, and duplicate search did not find an open PR or better canonical item owning it.

## Plan

Fix browser evaluate so `browser act kind=evaluate` / `openclaw browser evaluate` accepts JavaScript statement/function bodies with declarations such as `var`, `let`, and `const`, while preserving current expression, function-source, IIFE, async promise, `ref`, timeout, abort, dialog, navigation guard, SSRF guard, and `browser.evaluateEnabled` behavior. Do not add a separate browser exec command or loosen evaluate defaults. Check the ChromeDevTools MCP `evaluate_script` function-declaration contract before changing existing-session behavior; either provide tested parity or document a clear limitation. Add focused regression tests for managed Playwright evaluate with declaration statements plus existing expression/function inputs, include CLI/route coverage where appropriate, and update docs/changelog only for the user-visible contract change. Related refs: https://github.com/openclaw/openclaw/issues/51393, https://github.com/openclaw/openclaw/issues/55598, and https://github.com/openclaw/openclaw/issues/60381.
