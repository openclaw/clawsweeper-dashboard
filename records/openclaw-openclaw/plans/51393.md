# Coding Plan for openclaw/openclaw#51393: Feature: browser tool evaluate should support statements (var/let/const)

Source report: [records/openclaw-openclaw/items/51393.md](records/openclaw-openclaw/items/51393.md)

## Summary

Keep open. Current main still parses managed Playwright browser evaluate input as a parenthesized expression, so `var`/`let`/`const` bodies reproduce the reported syntax failure; the closed related browser exec request explicitly left this statement-support gap as a separate follow-up.

## Plan

Implement statement-style JavaScript support for browser act/evaluate in the browser plugin. Preserve existing expression and function-source behavior first; add a shared compatibility fallback that accepts statement bodies with declarations and explicit `return`, and apply it to both page and ref evaluate in `extensions/browser/src/browser/pw-tools-core.interactions.ts`. Do not add a separate `browser.exec` command and do not move browser-specific behavior into core. Preserve evaluateEnabled checks, timeout Promise.race behavior, abort disconnect, target handling, and navigation/SSRF guards. Add regression coverage for page evaluate and ref evaluate declaration bodies plus existing expression/function inputs. Update CLI/tool docs and `CHANGELOG.md` if the behavior is user-visible.
