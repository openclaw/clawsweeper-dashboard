# Coding Plan for openclaw/openclaw#51393: Feature: browser tool evaluate should support statements (var/let/const)

Source report: [records/openclaw-openclaw/items/51393.md](records/openclaw-openclaw/items/51393.md)

## Summary

Keep open: current main and the latest release still reject statement-style evaluate bodies because the managed browser path parses input as a parenthesized expression, and the broader browser exec thread explicitly left this narrower request as follow-up.

## Plan

Implement statement-body support for browser evaluate in https://github.com/openclaw/openclaw/issues/51393. Keep existing expression, function, async function, and IIFE inputs working; add support for statement bodies with declarations such as `var`, `let`, and `const`, including natural `return` values where appropriate. Keep the work inside the browser plugin evaluate surface; do not add a new `browser.exec` command, do not implement force-click from https://github.com/openclaw/openclaw/issues/60381, and do not broaden `browser.evaluateEnabled` or navigation/SSRF defaults. Preserve timeout Promise.race behavior, abort cleanup, page/ref/batch paths, and either make existing-session Chrome MCP behavior match or document/test a deliberate limitation. Update browser CLI/tool docs and changelog if the behavior becomes user-visible.
