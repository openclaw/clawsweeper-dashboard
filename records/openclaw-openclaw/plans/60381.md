## Summary

Keep open. Current main already implements and gates the evaluate half, but the requested opt-in force-click parameter is still absent from the browser act contract, normalizer, CLI/docs, and Playwright click forwarding path.

## Plan

Implement the remaining force-click portion of https://github.com/openclaw/openclaw/issues/60381. Do not change the existing evaluate semantics except for docs that mention it is already available and gated by `browser.evaluateEnabled`. Add an explicit `force?: boolean` for `kind: "click"` through `extensions/browser/src/browser-tool.schema.ts`, `extensions/browser/src/browser/client-actions.types.ts`, `extensions/browser/src/browser/routes/agent.act.normalize.ts`, route dispatch, and `clickViaPlaywright`, forwarding `{ force: true }` only when explicitly requested for `locator.click` and `locator.dblclick`. Add `openclaw browser click --force` and update browser docs plus the browser-automation skill/tool wording so agents try normal click first and use force only after recording the failed normal-click reason and target ref/selector; do not make force the default. For existing-session Chrome MCP, either implement an equivalent only after proving the Chrome MCP contract supports it, or return/document a clear unsupported-action error rather than ignoring `force`. Add focused regression tests for schema/normalization, batch forwarding, Playwright click/dblclick options, CLI forwarding, docs/skill guidance where appropriate, and existing-session behavior. Add a changelog entry because this is user-visible.
