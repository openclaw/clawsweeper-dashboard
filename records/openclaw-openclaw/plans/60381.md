## Summary

Keep open. Current `main` and latest release already expose `evaluate`, but the requested opt-in `force` click path is still missing from the browser action schema, request type, normalizer, Playwright forwarding, CLI/docs, and existing-session behavior.

## Plan

Fix https://github.com/openclaw/openclaw/issues/60381 by adding explicit opt-in `force?: boolean` support for browser `act` click actions. `evaluate` is already implemented, so do not replace it, loosen `browser.evaluateEnabled`, or make evaluate the default fallback. Thread `force` through `extensions/browser/src/browser-tool.schema.ts`, `extensions/browser/src/browser/client-actions.types.ts`, `extensions/browser/src/browser/routes/agent.act.normalize.ts`, `extensions/browser/src/browser/pw-tools-core.interactions.ts`, the `executeActViaPlaywright` click dispatch, CLI click input, browser docs, and `extensions/browser/skills/browser-automation/SKILL.md`. Preserve current default behavior when `force` is absent or false, forward true to Playwright `locator.click` and `locator.dblclick`, and make the existing-session Chrome MCP path either support force or return a clear unsupported message. Add focused regression coverage for schema/normalizer/route forwarding and Playwright options; document force as an explicit audited fallback after normal click failure, not a default path.
