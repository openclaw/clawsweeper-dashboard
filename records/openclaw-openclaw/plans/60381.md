## Summary

Keep open: current main already has the requested gated `evaluate` action, but the `click` contract still has no opt-in `force` field or forwarding path. The FSB comments are useful prior art for making forced clicks explicit and auditable rather than a default interaction mode.

## Plan

Implement the remaining force-click support for https://github.com/openclaw/openclaw/issues/60381. Keep the existing gated browser `evaluate` implementation unchanged and do not fold in statement-body evaluate support from https://github.com/openclaw/openclaw/issues/51393. Add `force?: boolean` to the browser click action schema, legacy flattened tool keys, `BrowserActRequest`, normalization, route forwarding, managed Playwright click execution, CLI click command, docs, and browser-automation skill guidance. Forward force to Playwright `locator.click` and `locator.dblclick` only when explicitly true; normal clicks must remain the default. For Chrome MCP existing-session clicks, either implement a tested equivalent if the driver supports it or return/document a clear unsupported error instead of silently ignoring `force`. Add focused regression tests for normalization, route forwarding, Playwright option forwarding, CLI input, and existing-session behavior. Add a changelog entry because this is a user-facing browser tool capability.
