## Summary

Keep open: current main implements the gated browser `act:evaluate` path, but the requested `force?: boolean` click option is still missing from the browser action schema, request type, normalizer, CLI, docs, and Playwright forwarding path.

## Plan

Implement the remaining force-click part of https://github.com/openclaw/openclaw/issues/60381. Add an explicit `force?: boolean` field for `kind: "click"` through `extensions/browser/src/browser-tool.schema.ts`, `extensions/browser/src/browser/client-actions.types.ts`, `extensions/browser/src/browser/routes/agent.act.normalize.ts`, `extensions/browser/src/browser/routes/agent.act.ts`, `extensions/browser/src/browser/pw-tools-core.interactions.ts`, and the CLI click command. Forward it to Playwright `locator.click`/`locator.dblclick` as an opt-in option, keep the default unchanged, and handle existing-session Chrome MCP clicks explicitly: support force if the underlying driver has a real option, otherwise return a clear unsupported error instead of dropping it. Update browser docs and `extensions/browser/skills/browser-automation/SKILL.md` to say normal click remains the default and force is a last-resort, auditable fallback after stale-ref/snapshot/highlight/scroll recovery. Add focused contract/unit coverage for schema/normalization, Playwright forwarding, batch forwarding if applicable, CLI request construction, and existing-session behavior. Do not change evaluate semantics or take on the separate statement-body evaluate request at https://github.com/openclaw/openclaw/issues/51393. Add a single user-facing changelog entry under the active version.
