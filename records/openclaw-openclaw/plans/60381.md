# Coding Plan for openclaw/openclaw#60381: browser tool: add force parameter for click and expose evaluate action

Source report: [records/openclaw-openclaw/items/60381.md](records/openclaw-openclaw/items/60381.md)

## Summary

Keep open. Current main already implements the requested gated `evaluate` action, but the `force` click option is still absent from the browser action schema, request type, normalizer, CLI/docs, and both managed Playwright and existing-session click paths; the May 12 issue comment gives clear guardrails that force clicks should remain explicit and auditable.

## Plan

Implement the remaining force-click half of https://github.com/openclaw/openclaw/issues/60381. Add optional boolean `force` to browser `click` actions only, default false, and thread it through `BrowserActSchema`, `BrowserActRequest`, `normalizeActRequest`, managed Playwright `clickViaPlaywright`/`dblclick` options, CLI `openclaw browser click --force`, docs, and `extensions/browser/skills/browser-automation/SKILL.md`. Preserve the existing gated `evaluate` implementation, SSRF/navigation guards, JavaScript navigation block, and normal-click default; do not make force automatic. For existing-session Chrome MCP clicks, either implement an equivalent explicit/auditable force behavior or reject/document unsupported force instead of silently ignoring it. Add regression coverage for route normalization/forwarding, Playwright click/dblclick force options, existing-session behavior, CLI flag construction, and docs/skill guidance. Add a single-line changelog entry because this is user-visible.
