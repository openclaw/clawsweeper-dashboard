# Coding Plan for openclaw/openclaw#60381: browser tool: add force parameter for click and expose evaluate action

Source report: [records/openclaw-openclaw/items/60381.md](records/openclaw-openclaw/items/60381.md)

## Summary

Keep open: current main and the latest release already implement the gated browser `evaluate` action, but the requested opt-in `force` click parameter is still missing from the schema, normalized action type, managed Playwright click forwarding, and docs. The FSB comments are useful prior art for making forced clicks explicit and auditable rather than a default interaction mode.

## Plan

Add an explicit opt-in browser `click` force parameter without changing default click behavior. Update the browser act schema, normalized `BrowserActRequest` type, route normalization, managed Playwright `clickViaPlaywright` forwarding, CLI/tool docs, and focused Vitest coverage for normal-click default behavior and `force` forwarding. Preserve the implemented `evaluate` action and its `browser.evaluateEnabled` gate. For Chrome MCP existing-session profiles, either add a tested equivalent only if the dependency supports it or return a clear unsupported message when `force` is requested. Do not make force the default, do not add broad retry policy, and keep wording explicit that force is a last-resort fallback after normal user-like input paths fail. Add a `CHANGELOG.md` entry if maintainers treat this as a user-facing feature during landing.
