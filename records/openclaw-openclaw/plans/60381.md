## Plan

Implement the remaining force-click half of https://github.com/openclaw/openclaw/issues/60381. Keep the existing `evaluate` implementation and `browser.evaluateEnabled` gating unchanged; do not address statement-style evaluate bodies, which are tracked separately. Add an explicit `force?: boolean` to browser `click` actions across schema, types, normalization, managed Playwright execution, CLI option/docs, and browser automation skill guidance. Forward force to `locator.click` and `locator.dblclick` for managed Playwright clicks, default it to false/undefined, and preserve the existing navigation/SSRF guard behavior. For existing-session Chrome MCP clicks, either implement a proven equivalent or return/document a clear unsupported error rather than silently ignoring force. Add focused tests showing force is accepted, normalized, forwarded, not defaulted on ordinary clicks, and handled intentionally for existing-session profiles. Include a changelog entry because this is a user-visible browser tool capability.
