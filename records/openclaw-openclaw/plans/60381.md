## Summary

Keep open: current main already has the requested gated `act:evaluate` surface, but the `force` click option is still absent from the browser action schema, request type, normalizer, CLI, and Playwright forwarding path.

## Plan

Implement the remaining force-click part of https://github.com/openclaw/openclaw/issues/60381. Current main already has gated `act:evaluate`; preserve that implementation and do not take on statement-body evaluate support from https://github.com/openclaw/openclaw/issues/51393. Add an explicit opt-in `force?: boolean` to browser `act:click` across the tool schema, `BrowserActRequest`, `/act` normalizer, Playwright execution, batch/executor tests, and `openclaw browser click --force`. Forward it to Playwright `locator.click`/`locator.dblclick` only when requested, leaving the default false/omitted. For Chrome MCP/existing-session clicks, either implement an equivalent audited forced click if the driver supports it or return a clear unsupported error instead of silently ignoring the field. Update `docs/tools/browser-control.md`, `docs/tools/browser.md`, and the `browser-automation` skill so force is documented as a fallback after normal click, adjacent-target, keyboard, and scroll attempts, and add a single-line changelog entry because this changes a user-visible tool surface. Do not change core/plugin boundaries or broaden evaluate behavior.
