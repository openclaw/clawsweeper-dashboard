# Coding Plan for openclaw/openclaw#25621: [Feature]: Error: Gateway service install not supported on openbsd

Source report: [records/openclaw-openclaw/items/25621.md](records/openclaw-openclaw/items/25621.md)

## Summary

Keep open: current main and the latest v2026.5.20 release still throw on unsupported service platforms before the read-only status fallback paths can render, and the focused implementation PR was closed unmerged.

## Plan

Fix the unsupported-platform Gateway service status crash reported in https://github.com/openclaw/openclaw/issues/25621. On platforms outside darwin/linux/win32, `openclaw status` and `openclaw gateway status` should render degraded read-only status instead of throwing from `resolveGatewayService()`. Keep lifecycle operations (`install`, `stage`, `uninstall`, `stop`, `restart`, and start flows that require managed service control) explicitly unsupported with clear errors; do not add OpenBSD/FreeBSD service-manager support and do not implement the broader generic shell-template feature in this PR. Prefer either a conservative unsupported-platform `GatewayService` adapter with safe read-only methods or narrowly guarded status call sites, following existing service/status patterns. Add regression coverage for unsupported `process.platform` on both plain status and gateway status, preserve supported darwin/linux/win32 behavior, and add a maintainer-owned changelog entry if policy requires it during landing.
