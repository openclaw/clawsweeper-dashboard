## Summary

Keep open. Current main still lets the setup wizard apply provider auth with default-model promotion but without the existing preservation flag, and heartbeat runs without `heartbeat.model` still resolve through the configured default model, so the reported paid-background-usage path remains source-reproducible.

## Plan

Fix https://github.com/openclaw/openclaw/issues/64129 by preventing setup/provider auth from replacing an existing configured primary model before heartbeat can inherit it. Reuse the existing `preserveExistingDefaultModel` provider-auth seam in `src/wizard/setup.ts` when an existing primary is present, while preserving first-run setup default selection and explicit model-picker choices. Add focused regression coverage for Google Gemini API-key setup with an existing primary plus heartbeat enabled without `heartbeat.model`; the expected result is that Google becomes available/enabled but the existing primary remains selected unless the user explicitly picks a new model. Do not special-case only Google if the generic setup provider-auth contract covers all providers, do not add a new config option, and do not change heartbeat runtime model inheritance globally. Add a user-facing changelog entry if the repair changes setup behavior.
