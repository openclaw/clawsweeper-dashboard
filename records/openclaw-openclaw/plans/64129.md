# Coding Plan for openclaw/openclaw#64129: [Bug]: Configuring a paid Google Gemini key implicitly turned existing heartbeat traffic into paid background usage

Source report: [records/openclaw-openclaw/items/64129.md](records/openclaw-openclaw/items/64129.md)

## Summary

Keep open: current main and the latest release still let setup apply provider auth with `setDefaultModel: true` without the existing-primary preservation flag, so configuring Google auth can still promote the paid Gemini default that heartbeat may inherit.

## Plan

Fix https://github.com/openclaw/openclaw/issues/64129 by preventing setup/onboard provider auth from silently replacing an existing `agents.defaults.model.primary` with a provider default. Reuse the existing `preserveExistingDefaultModel` contract already used by `src/commands/configure.gateway-auth.ts`: pass it through the `runSetupWizard` `applyAuthChoice` call when setup applies provider auth, preserving first-run behavior when no prior primary exists and preserving explicit later model-picker choices. Add a focused regression test in `src/wizard/setup.test.ts` asserting setup forwards `{ setDefaultModel: true, preserveExistingDefaultModel: true }` while still forwarding provider-specific auth opts. Do not special-case Google, do not change heartbeat runtime model resolution, do not add a config/env option, and do not edit `CHANGELOG.md`. Leave broader paid-preview consent policy to https://github.com/openclaw/openclaw/issues/74395.
