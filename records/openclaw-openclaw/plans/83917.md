## Summary

Keep open: current main and the latest release still keep a module-level diff-viewer controller list and append during each hydration, so the reported stale-controller lifecycle remains plausible and unfixed. The issue is low blast radius but narrow enough for a focused repair with a double-hydration regression test.

## Plan

Fix https://github.com/openclaw/openclaw/issues/83917 in the diffs plugin only. The observable bug is that `extensions/diffs/src/viewer-client.ts` keeps `controllers` at module scope and `hydrateViewer()` appends controllers without rebuilding the active list, so two hydrations in one document can retain stale `FileDiff` instances. Add a focused jsdom regression in `extensions/diffs/src/viewer-client.test.ts` that renders a card, calls `hydrateViewer()` twice from the same imported module without manually clearing `controllers`, and asserts the controller count equals the number of cards rather than doubling. Implement the narrowest fix in `viewer-client.ts`, preferably rebuilding/resetting the controller collection at the hydration boundary. Do not add config, new plugin APIs, or broad lifecycle abstractions. Be careful with `@pierre/diffs`: it exposes `cleanUp()`, not `destroy()`, and default cleanup can remove the host container, so only add cleanup behavior if the test proves it is safe. No CHANGELOG edit.
