## Summary

Keep open: checked-out main still uses nullish account-groups fallbacks in both Telegram account merging and the raw runtime group-policy resolver, so an explicit empty account groups map can still suppress root Telegram groups. Two linked fix PRs were closed unmerged, so the issue is still actionable.

## Plan

Fix https://github.com/openclaw/openclaw/issues/79427. In single-account Telegram configs, explicit empty `accounts.<id>.groups: {}` should be treated as absent so root `channels.telegram.groups` is inherited; non-empty account groups must still override root groups, and multi-account Telegram configs must keep the isolation behavior from https://github.com/openclaw/openclaw/issues/30673. Update both `mergeTelegramAccountConfig` and the raw gateway/group-policy path used by `resolveChannelGroupPolicy` so they agree. Add focused regression coverage for account merging, raw policy resolution, and a Telegram bot/gateway authorization path if practical.
