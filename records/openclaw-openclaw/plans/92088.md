## Summary

Current main still permits a transient managed-update failure to become a sustained outage: after the gateway exits, every update-command failure path records an error but performs no service recovery. This is a distinct, actionable reliability bug rather than work already covered by the earlier handoff fixes.

## Plan

Fix https://github.com/openclaw/openclaw/issues/92088 on current main. Extend `src/infra/update-managed-service-handoff.ts` so every failed path after the gateway parent exits makes a best-effort attempt to restore the same managed gateway through canonical supervisor and service identity. Cover update-child spawn errors, nonzero exits, signal exits, and helper exceptions.
