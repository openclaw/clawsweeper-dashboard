## Plan

Fix the message tool omitted-account path for channels with multiple accounts where only one enabled/configured account can send. Implement a shared outbound message-account resolver after channel selection, preserving explicit `accountId`, `input.defaultAccountId`, and target-bound agent bindings first; then use the selected channel plugin config adapter to prefer an enabled configured default account, auto-select exactly one enabled configured account, and throw clear zero-enabled or multiple-enabled account errors. Do not hard-code WhatsApp-only routing policy into core. Add focused regression coverage for the shared resolver/runner and WhatsApp omitted-account send path, update routing/message docs, and add a user-facing changelog fix entry.
