## Plan

Fix https://github.com/openclaw/openclaw/issues/20756. Implement a shared outbound message-account resolver after channel selection for the message tool. Preserve existing precedence for explicit `params.accountId`, `input.defaultAccountId`, and target-bound agent bindings. Do not hard-code WhatsApp-specific policy into core. Add shared runner/channel-selection tests, WhatsApp regression coverage for disabled `default` plus one enabled named account, routing/message docs updates, and a user-facing changelog fix entry.
