## Summary

Keep open. Current main still has the reported account-selection gap: omitted message-tool account IDs come only from explicit/default/bound sources, and WhatsApp omitted sends still fall through to the default-account helper even when that account is disabled.

## Plan

Fix https://github.com/openclaw/openclaw/issues/20756. Implement a shared outbound message-account resolver after channel selection, using the selected channel plugin config adapter rather than a WhatsApp-only helper. Preserve explicit `accountId`, `input.defaultAccountId`, and target-bound account IDs before auto-selection; when no account is provided, prefer an enabled/configured default account, auto-select exactly one enabled/configured account, and return clear errors when zero or multiple accounts are enabled. Add shared channel-selection/runner tests and WhatsApp regression coverage for disabled `default` plus one enabled named account. Update routing/message docs if behavior changes and add a user-facing changelog fix entry. Do not add a new config option.
