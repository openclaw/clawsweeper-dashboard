## Summary

Keep open: current main still fixes adjacent click-triggered/managed-profile downloads, but direct browser `/navigate` attachment URLs remain on a URL-only path that rethrows Playwright's download-starting abort and cannot return download metadata.

## Plan

Fix the browser plugin bug where direct `/navigate` to a URL that returns `Content-Disposition: attachment` still fails with Playwright's `Download is starting` abort instead of returning success with download metadata. Keep the fix localized to `extensions/browser/src/browser`: arm or observe a real download around `navigateViaPlaywright`/`gotoPageWithNavigationGuard`, treat the download-starting abort as success only after a download is captured, save through the existing guarded downloads directory, return additive metadata from `/navigate`, and preserve SSRF/navigation-policy fail-closed behavior. Do not add a new config option, do not reintroduce `BrowserContext.setDefaultDownloadPath`, and do not double-save when explicit `waitfordownload`/`download` waiters own the event. Add focused regression coverage for direct attachment navigation and policy-denied navigation. Add a changelog entry if the branch is prepared for landing.
