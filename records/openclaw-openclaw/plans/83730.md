## Summary

Keep open. Current main and the latest release still route top-level Feishu p2p final replies through the raw p2p `chatId` after reply metadata is suppressed, and the two explicit fix PRs for this issue are closed unmerged.

## Plan

Fix https://github.com/openclaw/openclaw/issues/83730 with one focused Feishu plugin PR. The bug is that non-thread p2p DM final replies compute `feishuTo = user:<open_id>` but the dispatcher is constructed with `chatId: ctx.chatId`, so after `skipReplyToInMessages` clears reply metadata the visible reply sends to the raw `oc_*` chat target and can fail with `SUBSCRIPTION_NOT_FOUND`. Add a separate delivery target or equivalent narrow owner-boundary change so top-level p2p visible replies use `user:<open_id>`/`open_id` delivery while group/topic and direct-message thread replies keep their existing chat/thread targets.
