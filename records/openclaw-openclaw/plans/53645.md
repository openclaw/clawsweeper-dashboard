## Summary

Keep open: current main and the latest release still drop explicit Feishu `replyTo` targets on ordinary message-tool `send` actions before they reach the Feishu send helpers. Core exposes and resolves `replyTo`, and the lower Feishu outbound adapter can send it, so this remains a narrow Feishu plugin mapping bug.

## Plan

Fix https://github.com/openclaw/openclaw/issues/53645. In the Feishu plugin action handler, make ordinary `send` honor explicit and core-applied reply targets (`replyTo`, `reply_to`, and the existing message-tool shape) as inline reply targets for text/card/media sends. Keep `thread-reply` as the explicit `reply_in_thread` path that requires a message id, preserve topic auto-threading for `group_topic` sessions, preserve the missing-thread fallback behavior from https://github.com/openclaw/openclaw/pull/80306, and do not add a new config option or change core outbound routing. Add focused regression coverage in `extensions/feishu/src/channel.test.ts` for ordinary text, presentation/card, and media sends with `replyTo`, plus no-regression coverage for topic auto-threading. Add a user-facing changelog entry.
