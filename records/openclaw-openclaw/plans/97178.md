# Coding Plan for openclaw/openclaw#97178: [Bug]: openclaw gateway install adds a duplicate gui-domain LaunchAgent over an existing system LaunchDaemon (restart storm)

## Summary

Keep open. Current main and v2026.6.10 still treat the managed macOS gateway as a gui-domain LaunchAgent, while system LaunchDaemon discovery is only an advisory deep scan; the reporter's live follow-up adds concrete evidence for the duplicate-manager restart storm.

## Plan

Fix the macOS gateway duplicate-manager bug reported in https://github.com/openclaw/openclaw/issues/97178. Keep the scope narrow: before `openclaw gateway install`, forced install, or doctor repair writes/bootstraps a gui-domain LaunchAgent, detect an existing same-label system-domain LaunchDaemon or `/Library/LaunchDaemons/ai.openclaw.gateway.plist` that appears to run OpenClaw gateway on the target port, refuse to install a second manager, and print clear recovery/external-owner guidance. Make `gateway status`/doctor diagnostics avoid saying the service is simply not installed when a same-label system daemon is found; surface it as an external/system owner. Do not add a full `--system` LaunchDaemon install mode, sudo/root plist writes, new config/env options, or broad lifecycle support; leave that to https://github.com/openclaw/openclaw/issues/51860 and https://github.com/openclaw/openclaw/issues/14251. Add focused regression tests around launchd detection, install refusal, status/doctor messaging, and the existing Linux external-service guard should remain intact. No changelog edit.
