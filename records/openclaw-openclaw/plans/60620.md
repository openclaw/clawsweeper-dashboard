## Summary

Keep open. Current main still reloads the full startup plugin scope after detecting any deferred configured channel plugin, so unrelated startup plugins can still run registration twice across setup-runtime startup and post-listen full-runtime reload.

## Plan

Fix https://github.com/openclaw/openclaw/issues/60620. Current main loads the full startupPluginIds in setup-runtime when a deferred configured channel plugin exists, then post-listen calls reloadDeferredGatewayPlugins with the same full startupPluginIds. Narrow the post-listen full-runtime reload to only deferredConfiguredChannelPluginIds, preserve or merge the already-attached non-deferred startup plugin registry records/handlers, and add regression coverage with one non-deferred startup plugin counter plus one deferred configured channel plugin proving the non-deferred plugin is not re-registered. Preserve full-runtime promotion for deferred channel plugins. Do not add config knobs or broad plugin-specific idempotency workarounds. Add a CHANGELOG.md fix entry if the final change is user-visible.
