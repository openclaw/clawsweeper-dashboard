# Coding Plan for openclaw/openclaw#60620: Plugins initialized twice per gateway restart (duplicate register/load cycle)

Source report: [records/openclaw-openclaw/items/60620.md](records/openclaw-openclaw/items/60620.md)

## Summary

Keep open. Current main still reloads the full startup plugin scope after detecting any deferred configured channel plugin, so unrelated startup plugins can still run registration twice across the setup-runtime load and the post-listen full-runtime reload.

## Plan

Fix https://github.com/openclaw/openclaw/issues/60620. Current main reloads the full startupPluginIds scope after listen whenever deferredConfiguredChannelPluginIds is non-empty, which can re-register non-deferred startup plugins. Change the Gateway post-listen deferred reload path so only deferredConfiguredChannelPluginIds are loaded in full-runtime mode, and preserve the existing attached registry/runtime for non-deferred startup plugins. Add focused regression coverage with one non-deferred startup plugin that increments a register counter plus one configured deferred channel plugin; assert the non-deferred plugin registers once while the deferred channel still gets promoted to full runtime. Do not add a config option or broaden plugin discovery. Add a CHANGELOG.md fix entry because this changes user-visible startup/log behavior.
