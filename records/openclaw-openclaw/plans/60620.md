# Coding Plan for openclaw/openclaw#60620: Plugins initialized twice per gateway restart (duplicate register/load cycle)

Source report: [records/openclaw-openclaw/items/60620.md](records/openclaw-openclaw/items/60620.md)

## Summary

Keep open. Current main still reloads the full startup plugin scope after detecting any deferred configured channel plugin, so non-deferred startup plugins can still run registration twice during one Gateway restart.

## Plan

Fix https://github.com/openclaw/openclaw/issues/60620. Current main loads all startupPluginIds in setup-runtime mode when any configured deferred channel plugin exists, then after Gateway listen calls reloadDeferredGatewayPlugins with pluginIds: startupPluginIds again. Narrow the post-listen reload to deferredConfiguredChannelPluginIds so unrelated startup plugins do not re-run registration, and preserve/merge the already attached non-deferred registry records, handlers, method list, HTTP route/channel pins, and discovery state. Use https://github.com/openclaw/openclaw/pull/48616 only as historical context; do not reintroduce stale code blindly. Add regression coverage for a mixed startup plan with one non-deferred startup plugin and one deferred configured channel plugin, proving the non-deferred plugin is not reloaded while the deferred channel plugin is promoted to full runtime. Likely files are src/gateway/server.impl.ts, src/gateway/server-plugin-bootstrap.ts, src/gateway/server-startup-plugins.ts, src/gateway/server-plugins.ts, src/gateway/server-plugins.test.ts, src/gateway/server-startup-plugins.test.ts, src/gateway/server-startup-post-attach.test.ts, and possibly src/plugins/loader.test.ts. Add a CHANGELOG.md fix entry unless maintainers classify the change as internal-only. Do not add plugin-specific idempotency guards or new config flags.
