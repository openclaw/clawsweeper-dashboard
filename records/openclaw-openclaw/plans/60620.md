## Summary

Keep open. Current main still reloads the full startup plugin scope after detecting a deferred configured channel plugin, so a non-deferred startup plugin can still register once during setup-runtime startup and again during the post-listen full-runtime reload.

## Plan

Fix https://github.com/openclaw/openclaw/issues/60620. Scope the post-listen reload to deferredConfiguredChannelPluginIds while preserving the already attached non-deferred startup plugin runtime and still promoting deferred channel plugins to full runtime after listen. Add focused regression coverage with one non-deferred startup plugin and one configured deferred channel plugin proving the non-deferred plugin is not re-registered and deferred full-runtime activation still happens.
