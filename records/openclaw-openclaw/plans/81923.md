## Plan

Fix the remaining custom Xiaomi Token Plan schema compatibility gap from https://github.com/openclaw/openclaw/issues/81923. Add a narrow regression that builds an OpenAI-completions tool payload for a custom provider id pointed at `https://token-plan-cn.xiaomimimo.com/v1` with an array property missing `items`, and ensure the final payload omits only synthetic empty `items: {}` while preserving typed/boolean/null/literal items.
