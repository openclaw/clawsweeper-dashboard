# Coding Plan for openclaw/openclaw#93219: [Bug]: Gateway never loads externally-installed npm plugins (installed-plugin load stage skipped) — version/layout/clean-rebuild independent

Source report: [records/openclaw-openclaw/items/93219.md](records/openclaw-openclaw/items/93219.md)

## Summary

Keep open: current main still has a plausible core startup-planning gap for the documented WeChat/plugin-entry install flow. The gateway runtime loads only the startup plugin-id plan, and source inspection shows an installed external channel plugin enabled only via `plugins.entries.<id>.enabled=true` can remain in metadata without being attempted at runtime.

## Plan

Fix https://github.com/openclaw/openclaw/issues/93219 as a generic OpenClaw plugin startup bug. Current main documents a WeChat install flow where `openclaw plugins install @tencent-weixin/openclaw-weixin` is followed by `openclaw config set plugins.entries.openclaw-weixin.enabled true`, but gateway runtime loading is scoped to `PluginLookUpTable.startup.pluginIds`; an installed external channel plugin enabled only through `plugins.entries.<id>.enabled=true` can be present in `installed_plugin_index` and still never be passed to `loadOpenClawPlugins`. Add focused regression coverage for an installed/global external channel plugin with a manifest channel and no `activation.onStartup`, configured only by `plugins.entries.<id>.enabled=true`, then update `src/plugins/gateway-startup-plugin-ids.ts` and the minimal supporting startup/activation handoff so the plugin is included for registration while preserving `plugins.enabled`, allow, deny, explicit disable, workspace trust, and untrusted ambient env rules. Check the adjacent drafts https://github.com/openclaw/openclaw/pull/88497 and https://github.com/openclaw/openclaw/pull/88379 first to avoid duplicating their `channels.<id>.enabled=true` fix; this issue's required behavior is the plugin-entry-only documented flow. Do not add a new config option, do not special-case `openclaw-weixin`, do not move WeChat runtime logic into core, and do not edit `CHANGELOG.md`.
