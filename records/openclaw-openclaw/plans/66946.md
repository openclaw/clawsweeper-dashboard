## Summary

Keep open: current main and the latest release still encode the duplicated Control UI bootstrap config path, so the reported 404 remains source-reproducible; the two related fix PRs were closed without merging.

## Plan

Fix https://github.com/openclaw/openclaw/issues/66946. Current main defines `CONTROL_UI_BOOTSTRAP_CONFIG_PATH` as `/__openclaw/control-ui-config.json`; when the Control UI is opened under `/__openclaw__/`, the inferred base path `/__openclaw__` makes the frontend request `/__openclaw__/__openclaw/control-ui-config.json`, which the gateway does not treat as the bootstrap endpoint and returns 404. Make the bootstrap config path contract consistently base-path-relative across `src/gateway/control-ui-contract.ts`, `src/gateway/control-ui.ts`, `ui/src/ui/controllers/control-ui-bootstrap.ts`, and `ui/vite.config.ts`. Update focused tests in `src/gateway/control-ui.http.test.ts`, `ui/src/ui/controllers/control-ui-bootstrap.test.ts`, and `ui/src/ui/navigation.test.ts` or `src/gateway/control-ui-routing.test.ts` as needed, including coverage for `gateway.controlUi.basePath: "/__openclaw__"`. Update `docs/web/control-ui.md` and `CHANGELOG.md` because this is user-visible. Preserve bootstrap auth checks, plugin/API passthrough, gateway probe behavior, static asset fallback semantics, and existing no-base-path compatibility or a deliberate short-lived alias if needed. Do not change unrelated Control UI routing or gateway auth policy.
