## Summary

Keep open. Current main still exposes `subagents list` as a direct-child view only; depth-2 workers are summarized as descendants but not returned as rows with their own labels, status, runtime, model, or parent metadata, and the corroborating user comment confirms this is a real nested-orchestrator workflow gap.

## Plan

Implement recursive/depth-limited `subagents list` for https://github.com/openclaw/openclaw/issues/47320. Preserve current behavior by default as direct children only, add a bounded `depth` option and/or `recursive` flag for the `subagents` tool and `/subagents list`, and return/render flat rows that include each descendant's label, status, runtime, model, session key, depth, and parent session key. Reuse existing latest-run, descendant-count, liveness, and stale-child filtering; do not extend descendant `kill`, `send`, or `steer` control unless a maintainer explicitly asks. Keep adjacent issue https://github.com/openclaw/openclaw/issues/11040 as context for session-chain metadata but do not turn this into a broader root/trace session contract. Add focused regression tests for tool output and slash-command output, update docs/tools/subagents.md, and add a user-facing changelog entry.
