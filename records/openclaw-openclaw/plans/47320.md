## Summary

Keep open. Current main still exposes `subagents list` as a direct-child inspection surface: nested workers can appear as descendant summaries, but they are not returned as list rows with their own status, runtime, model, task, depth, or parent metadata.

## Plan

Implement recursive/depth listing for the `subagents` tool and `/subagents list` slash command. Preserve the current direct-child default, add bounded options such as `depth` and/or `recursive`, return flat rows with `depth` and parent/controller session metadata, and reuse existing latest-run, descendant traversal, liveness, and stale-child filtering. Do not expand descendant `kill`, `send`, or `steer` semantics in this PR. Treat https://github.com/openclaw/openclaw/issues/11040 and https://github.com/openclaw/openclaw/issues/54578 as adjacent context only; do not implement root/trace metadata or a new CLI tree command here. Update focused tests, docs for the public subagents command/tool surface, and add a changelog entry because this is user-visible.
