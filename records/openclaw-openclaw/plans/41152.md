## Summary

Keep open. Current main still routes async gateway and node exec approval completion text through the compact notify formatter, so multiline output over 400 characters is tailed and whitespace-normalized before the agent follow-up prompt sees it; the linked focused fix PR is closed unmerged.

## Plan

Fix https://github.com/openclaw/openclaw/issues/41152. Current async exec approval completion delivery reuses compact notification formatting: gateway completions call `normalizeNotifyOutput(tail(..., DEFAULT_NOTIFY_TAIL_CHARS))`, and node completions call `normalizeNotifyOutput(combined.slice(-DEFAULT_NOTIFY_TAIL_CHARS))`, causing >400-character multiline output to lose its prefix and newlines before `buildExecApprovalFollowupPrompt` sees it. Implement a narrow formatter for approval-completion results that preserves captured multiline output up to the existing exec capture cap and explicitly marks real capture-cap truncation. Apply the same ownership boundary to gateway and node async approval completion paths if both still share the bug. Do not change generic background `notifyOnExit` compact summary behavior, exec approval policy/security semantics, or the existing capture cap.
