Keep open. Current main and the latest release still use slash-command-only debounce bypasses for the shared inbound debounce path, while the abort-aware helper and docs already cover standalone abort phrases; the related fix PR at https://github.com/openclaw/openclaw/pull/48916 is closed unmerged.

## Plan

Fix the current-main bug where bare abort phrases such as `stop`, `abort`, and `wait` are delayed or swallowed by inbound debounce. Use abort-aware command detection for `src/channels/inbound-debounce-policy.ts` and for plugin-local debounce predicates that currently call `hasControlCommand()` directly, including Feishu, Mattermost, Microsoft Teams, and WhatsApp. Preserve existing behavior for normal text, media flushes, slash commands, command authorization, mention-gating, and the documented same-sender DM coalescing exception. Add focused regression tests proving bare abort phrases bypass debouncing while normal text still debounces. Include a user-facing changelog entry if the repair PR lands.
