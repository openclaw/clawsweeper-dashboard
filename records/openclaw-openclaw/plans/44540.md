# Coding Plan for openclaw/openclaw#44540: Hybrid search BM25 component penalizes multimodal (image/audio) results

Source report: [records/openclaw-openclaw/items/44540.md](records/openclaw-openclaw/items/44540.md)

## Summary

Keep open. Current main and the latest stable release still use a modality-blind weighted hybrid merge, while persisted memory rows do not carry image/audio modality, so the reported ranking penalty remains a valid memory-core bug.

## Plan

Fix https://github.com/openclaw/openclaw/issues/44540. Current hybrid memory scoring multiplies vector-only image/audio hits by `vectorWeight` because stored chunks/search rows have no modality. Carry explicit multimodal modality from indexing into persisted chunk/FTS/search rows with a backward-compatible schema path, treat missing modality as text, and update `mergeHybridResults` so non-text multimodal vector candidates are scored by vector score rather than BM25-dampened global weights. Preserve existing weighted BM25+vector behavior for markdown/text chunks, keyword-only fallback/minScore behavior, and default weights; do not add a config option or infer modality from filenames at merge time. Add regression coverage for image/audio modality persistence and vector-only multimodal hybrid ranking, plus a `CHANGELOG.md` fix entry. Stop for maintainer review if this turns into a provider API or product policy change.
