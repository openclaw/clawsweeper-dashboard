# Coding Plan for openclaw/openclaw#44540: Hybrid search BM25 component penalizes multimodal (image/audio) results

Source report: [records/openclaw-openclaw/items/44540.md](records/openclaw-openclaw/items/44540.md)

## Summary

Keep open. Current main still loses image/audio modality before hybrid ranking, so vector-only media hits are merged with `textScore: 0` and diluted by the default weighted BM25 component.

## Plan

Fix https://github.com/openclaw/openclaw/issues/44540. Current main classifies image/audio memory files before indexing, but persisted chunks/search rows do not retain modality, so hybrid ranking scores vector-only media hits as vectorWeight * vectorScore + textWeight * 0 under default BM25+vector weights. Add a narrow modality contract to the memory index so image/audio modality survives persistence and search result construction, then update hybrid merge/ranking so non-text media vector hits are not diluted by missing BM25 text score while text chunks keep current weighting. Do not change global default weights, do not infer modality from path extensions at rank time, and preserve existing fallback lexical ranking behavior from manager-search. Add or update migration/reindex handling for existing rows that lack modality, plus focused regression tests for schema/indexing/search rows/hybrid media scoring and existing text hybrid behavior.
