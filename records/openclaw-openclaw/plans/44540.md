## Summary

Keep open. Current main still drops image/audio modality before builtin memory search ranking and applies the same weighted vector-plus-BM25 formula to every hybrid candidate, so multimodal vector hits with no lexical match remain penalized under the documented defaults.

## Plan

Fix https://github.com/openclaw/openclaw/issues/44540. Current builtin memory hybrid search drops multimodal modality before ranking, so image/audio vector hits with no BM25 text match are scored as vectorWeight * vectorScore under the default 0.7/0.3 weights. Carry an explicit chunk modality signal from multimodal file entry/indexing through the SQLite schema, chunk writes, search row/result contracts, and mergeHybridResults. Score non-text multimodal vector candidates fairly, likely using their vector score when there is no meaningful BM25 component, while preserving existing text hybrid scoring, keyword-only fallback behavior, MMR, temporal decay, and default weights. Add migration/ensureColumn handling for existing stores and focused regression coverage for multimodal vector-only candidates competing with text hybrid candidates. Do not infer modality from filenames at merge time and do not change global defaults.
