# Coding Plan for openclaw/openclaw#44540: Hybrid search BM25 component penalizes multimodal (image/audio) results

Source report: [records/openclaw-openclaw/items/44540.md](records/openclaw-openclaw/items/44540.md)

## Summary

Keep open. Current main still classifies multimodal files before indexing but drops the modality before persistence/search results, and the builtin hybrid ranker still applies the same vector-plus-BM25 weighted formula to all candidates, so image/audio vector hits with no lexical match remain downweighted under the documented defaults.

## Plan

Fix https://github.com/openclaw/openclaw/issues/44540 by carrying `MemoryMultimodalModality` or equivalent chunk modality from file classification through the builtin memory index schema, persistence, search result contracts, and `mergeHybridResults`. Rank image/audio candidates so a vector hit is not penalized by a missing BM25 text score, while preserving normal hybrid BM25 + vector scoring for text chunks. Avoid changing global default weights, avoid adding a config option, and avoid relying only on filename-extension inference at ranking time when persisted modality can be carried. Add focused regression coverage showing multimodal vector results compete fairly under default 0.7/0.3 hybrid weights, and add a user-facing CHANGELOG.md fix entry.
