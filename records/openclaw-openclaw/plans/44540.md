## Summary

Keep open. Current main still drops image/audio modality before ranking and applies the same global vector-plus-BM25 weighted score to every hybrid result, so the reported multimodal penalty remains source-reproducible.

## Plan

Fix https://github.com/openclaw/openclaw/issues/44540. Current main indexes multimodal image/audio files but stores and searches them without modality, so hybrid scoring weights media vector-only hits by vectorWeight and penalizes them versus text chunks. Carry explicit chunk modality from MemoryFileEntry through the persisted chunk/search-row path, including any safe schema migration needed, and make hybrid ranking score image/audio candidates with vector-only semantics while preserving existing text hybrid behavior. Add regression coverage showing a multimodal vector candidate with no keyword/BM25 hit is not demoted below a weaker text candidate solely because textWeight is nonzero. Do not add a config knob, do not weaken default BM25 weights globally, and avoid filename inference at merge time if a stored modality is available. Add a changelog entry because this changes user-visible memory search ranking.
