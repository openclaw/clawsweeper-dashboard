## Summary

Keep open. Current main and the latest release still use the fixed 100,000-token parent-fork cap, log oversized parent skips, and start the child isolated without auto-compaction, model-aware sizing, or a channel-visible warning.

## Plan

Implement the narrow visible-warning slice for https://github.com/openclaw/openclaw/issues/63141. When initSessionState skips parent inheritance because resolveParentForkDecision returns status=skip/reason=parent-too-large, surface a one-time or rate-limited user-visible notice to the originating channel/thread explaining that the child is starting with isolated context because the parent context is too large. Keep the existing 100,000-token cap and isolated fallback. Do not reintroduce session.parentForkMaxTokens, do not add a new config option, do not auto-compact parents, and do not change model-context sizing policy. Add focused regression coverage for the visible warning and dedupe behavior, preserve the current subagent forkFallbackNote behavior, update user-facing docs if the warning semantics need documenting, and add a changelog entry because this changes user-visible behavior.
