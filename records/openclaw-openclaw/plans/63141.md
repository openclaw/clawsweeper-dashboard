# Coding Plan for openclaw/openclaw#63141: [Feature] Auto-tune or warn on session.parentForkMaxTokens instead of silent skip

Source report: [records/openclaw-openclaw/items/63141.md](records/openclaw-openclaw/items/63141.md)

## Summary

Keep open. Current main still skips oversized parent-session inheritance for channel/thread session initialization by logging a warning and starting the child isolated; it does not auto-compact the parent, derive the cap from model context, or emit a channel-visible warning for the reported cold starts.

## Plan

Fix https://github.com/openclaw/openclaw/issues/63141 by adding a one-time, channel-visible warning when `initSessionState()` skips parent-session inheritance because the parent is too large. Keep the scope narrow: do not reintroduce `session.parentForkMaxTokens`, do not auto-compact the parent, and do not change the 100k threshold or model-budget policy. Prefer carrying an explicit skip notice out of the session-init path into the normal reply/channel surface instead of sending ad hoc messages from low-level session code. Rate-limit or persist the notice so each child session warns once and subsequent turns do not repeat it. Update focused regression coverage for oversized parent channel/thread sessions and preserve existing subagent fork fallback-note behavior. Add/update user-facing docs and a changelog entry if the warning reaches users.
