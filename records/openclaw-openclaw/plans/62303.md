## Summary

Keep open. Current main and the latest stable release still gate Telegram DM-topic auto-renaming on `!entry?.systemSent`, so a topic whose session entry is already marked bootstrapped can skip `generateTopicLabel` and `editForumTopic` and keep the fallback title.

## Plan

Fix the Telegram DM topic auto-topic-label regression reported at https://github.com/openclaw/openclaw/issues/62303. In current main, `extensions/telegram/src/bot-message-dispatch.ts` only attempts `generateTopicLabel` and `bot.api.editForumTopic` when `isFirstTurnInSession` is true, and that flag is derived from `!entry?.systemSent`; repair this inside the Telegram plugin by tracking an idempotent per-DM-topic label attempt independently of core `systemSent`. Preserve default-on `autoTopicLabel`, per-DM-over-account precedence, `RawBody` input, the 500-character prompt bound, fire-and-forget reply delivery, and the Codex-compatible label generator behavior from https://github.com/openclaw/openclaw/pull/78450. Do not change global core session first-turn or `systemSent` semantics. Add a focused dispatch regression test where the DM-topic session store entry already has `systemSent: true` but the first visible user message still triggers `generateTopicLabel` and `editForumTopic`; keep the existing negative config override coverage. Add a user-facing changelog fix entry if the repair changes runtime behavior.
