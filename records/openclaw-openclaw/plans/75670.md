## Summary

Keep open. Current main still has the reported Matrix/session boundary mismatch: Matrix preserves mixed-case thread IDs during inbound routing, but shared session construction, recording, and store lookup still lowercase opaque room/event IDs before persistence and lookup.

## Plan

Fix https://github.com/openclaw/openclaw/issues/75670. Current main preserves Matrix thread IDs in `extensions/matrix/src/matrix/monitor/threads.ts` and `route.ts`, but shared routing/session/store paths still lowercase Matrix room IDs and `:thread:$eventId` suffixes. Add a generic, extension-safe way to preserve provider-opaque session-key components for Matrix room/event IDs across route construction, `recordInboundSession`/last-route updates, session-store lookup/maintenance, delivery metadata extraction, transcript/session lookup, and outbound Matrix thread relation recovery. Do not add Matrix-specific hard-coded policy to broad core hot paths if a typed generic seam or channel-owned metadata can carry the provider-exact value. Do not remove intentional lowercase canonicalization for existing generic/direct/webchat behavior. Include regression tests that show a mixed-case Matrix room ID and mixed-case thread event ID produce exactly one session entry and that outbound `m.relates_to.event_id` uses the original event ID. Account for the prior rejected builder-only direction in https://github.com/openclaw/openclaw/pull/57337, and do not conflate this with the distinct MCP cron loopback issue https://github.com/openclaw/openclaw/issues/75261. Add a changelog entry because this is user-visible Matrix delivery behavior.
