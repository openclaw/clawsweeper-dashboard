## Summary

Keep open: current main and the latest release still allow `sessions.delete` to remove a row while a normal agent-run finalizer can later recreate the same `sessionKey` from stale in-memory metadata.

## Plan

Fix https://github.com/openclaw/openclaw/issues/40840 by making `sessions.delete` a reliable boundary against stale post-run session metadata persistence. Current main deletes the durable row in `src/gateway/server-methods/sessions.ts`, but `src/agents/agent-command.ts` later calls `updateSessionStoreAfterAgentRun`, and `src/agents/command/session-store.ts` can recreate `store[sessionKey]` from a stale in-memory snapshot if the row was deleted after active-run tracking cleared. Keep the fix narrow: do not add a new public hard-kill API, config option, or broad session-management UI; preserve legitimate fresh session creation after deletion. Add regression coverage for the deleted-row/stale-finalizer interleaving, likely in `src/agents/command/session-store.test.ts` and the Gateway delete lifecycle tests. Add a `CHANGELOG.md` fix entry if the repair changes user-visible behavior.
