## Summary

Keep open: current main and the latest release still omit `docker-compose.override.yml` from `_clawdock_compose()`, so `clawdock-start` can recreate the gateway from only the base and extra Compose files. Docker’s Compose docs confirm that explicit `-f` files define the merge set/order, while default override auto-loading is the no-`-f` path. ([docs.docker.com](https://docs.docker.com/compose/how-tos/multiple-compose-files/merge/))

## Plan

Fix https://github.com/openclaw/openclaw/issues/49909. Add focused regression coverage for Compose file argument ordering without requiring Docker to run; prefer stubbing the `docker` command or extracting a tiny testable helper if that keeps the shell helper clean.
