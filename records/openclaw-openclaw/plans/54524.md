Keep open. Current main still permits the reported overlap: session transcript indexing and session-memory hook Markdown output can both feed memory search, and the public docs still do not warn users that combining them can duplicate session-derived content and embedding work.

## Plan

Add concise public guidance explaining that `memorySearch.experimental.sessionMemory` with `sources: ["memory", "sessions"]` overlaps with the bundled `session-memory` hook because the hook writes session-derived Markdown into `<workspace>/memory/`, which the normal memory source indexes.
