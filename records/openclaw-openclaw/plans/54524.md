Current main and the latest shipped tag still allow the reported overlap: session transcript indexing can run beside workspace memory indexing, the bundled hook still writes session-derived Markdown into that memory directory, and the public docs still do not warn users about duplicate session-derived results.

## Plan

Add guidance in the public memory search/config docs and session-memory hook docs explaining that `memorySearch.experimental.sessionMemory` with `sources: ["memory", "sessions"]` overlaps with the bundled `session-memory` hook because the hook writes session-derived Markdown into `<workspace>/memory/`.
