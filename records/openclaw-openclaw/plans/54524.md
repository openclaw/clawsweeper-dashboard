Keep open. Current main and the latest release still allow memory search to index both workspace memory summaries and raw session transcripts, and the public docs still lack the requested overlap warning.

## Plan

Fix https://github.com/openclaw/openclaw/issues/54524 with a docs-only PR. Add concise guidance to the public memory search/config docs and session-memory hook docs explaining that `memorySearch.experimental.sessionMemory` with `sources: ["memory", "sessions"]` overlaps with the bundled `session-memory` hook because the hook writes session-derived Markdown under `<workspace>/memory/`.
