## Summary

Keep open: current main still builds Feishu group session keys from the group peer only, so multiple Feishu apps in the same group can share one session despite account-scoped multi-account routing support.

## Plan

Fix Feishu group multi-account session identity collision reported at https://github.com/openclaw/openclaw/issues/40194. Current main routes Feishu group sessions by chat-scoped peer id while `buildAgentPeerSessionKey` ignores `accountId` for non-direct peers, so multiple Feishu apps in the same group can converge on one session. Implement a Feishu-owned or opt-in account-aware group routing/session-key path; preserve legacy single-account group session continuity where feasible, and do not change unrelated channel group key behavior.
