## Summary

Keep open: current main still has a source-clear restart-recovery/WebChat gap. Restart recovery can resume through an internal `agent` run and the embedded runner can return the existing incomplete-turn error payload, but the WebChat/chat final path still only emits buffered assistant text or reloads persisted transcript history, so a recovered tool-only turn can still end without a visible terminal error.

## Plan

Fix https://github.com/openclaw/openclaw/issues/77883. Current main resumes interrupted main sessions through `src/agents/main-session-restart-recovery.ts` using the Gateway `agent` method with `deliver: false`; when the resumed embedded run ends with an incomplete-turn `isError` payload and no assistant delta, Control UI can receive tool rows and/or an empty final/history reload without a visible terminal error. Add a narrow bridge so restart-recovery agent runs surface or persist the existing incomplete-turn error as a visible WebChat terminal row. Do not change provider empty-response fallback behavior, do not broadly enable `deliver: true`, do not re-execute tools, and avoid duplicating normal Pi assistant transcript entries. Coordinate with adjacent restart/session lifecycle work in https://github.com/openclaw/openclaw/pull/74687 and https://github.com/openclaw/openclaw/pull/75662. Add focused regression coverage for a restart-recovery `agent` run returning an `isError` payload with no assistant text and for Control UI/history rendering of the terminal error. Add a changelog entry if the final patch is user-visible.
