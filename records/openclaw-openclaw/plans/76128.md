## Summary

Keep open. Current `main` still lets `openclaw models set` update and report the global default while an explicit default-agent `agents.list[].model.primary` remains authoritative for the active agent, and the later thread comment is correct that the same pinned model can flow into spawned subagents through the caller-model fallback.

## Plan

Fix https://github.com/openclaw/openclaw/issues/76128 with a narrow diagnostic change. When `openclaw models set <model>` updates `agents.defaults.model.primary`, detect whether the configured default agent has an explicit `agents.list[].model` primary that will still take precedence over the new default and print a clear warning after the successful update. Preserve existing per-agent override precedence; do not add flags, do not rewrite/remove agent overrides automatically, and do not change subagent model precedence. Sanitize config-controlled values before terminal output using the existing log sanitizer. Add focused regression coverage for object-form and string-form agent model overrides, no-warning cases, and the existing default write behavior.
