# Coding Plan for openclaw/openclaw#29707: [Feature Request]: Native Prompt Caching Support for Amazon Nova (Bedrock)

Source report: [records/openclaw-openclaw/items/29707.md](records/openclaw-openclaw/items/29707.md)

## Summary

Keep open: current main still drops explicit cache retention for direct Amazon Nova Bedrock model IDs and forces non-Anthropic Bedrock streams to `cacheRetention: "none"`. The earlier Nova rejection history in https://github.com/openclaw/openclaw/issues/20815 and https://github.com/openclaw/openclaw/pull/20866 explains why the guard exists, but AWS now documents Nova-compatible prompt caching, so focused provider work remains.

## Plan

Fix https://github.com/openclaw/openclaw/issues/29707 by adding native Amazon Nova prompt-cache support in the bundled Amazon Bedrock provider. Recognize supported direct, region-prefixed, and global Nova model IDs; allow explicit `cacheRetention` to reach the provider for Nova; inject only AWS/Nova-compatible `cachePoint` checkpoints on supported Bedrock payload fields; keep unsupported non-Anthropic Bedrock models on the no-cache path; and do not map `cacheRetention: "long"` to Nova `ttl: "1h"` unless AWS docs or a live probe proves that exact model supports it. Update focused unit tests, public prompt-caching/Bedrock docs, and `CHANGELOG.md`; avoid core policy changes outside the cache-retention/provider seam. If AWS credentials are available, add or run a focused Nova ConverseStream live probe showing payload acceptance plus cache read/write accounting; otherwise document that live proof was unavailable.
