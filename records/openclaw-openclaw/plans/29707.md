# Coding Plan for openclaw/openclaw#29707: [Feature Request]: Native Prompt Caching Support for Amazon Nova (Bedrock)

Source report: [records/openclaw-openclaw/items/29707.md](records/openclaw-openclaw/items/29707.md)

## Summary

Keep open: current main still drops explicit `cacheRetention` for direct `amazon.nova-*` Bedrock IDs and then forces those models through `cacheRetention: "none"`. The prior discussion usefully explains why the old guard existed via https://github.com/openclaw/openclaw/issues/20815 and https://github.com/openclaw/openclaw/pull/20866, but current AWS documentation now describes Nova prompt caching and recommends explicit prompt caching for cost savings.

## Plan

Implement native Amazon Nova prompt caching for the bundled Amazon Bedrock provider. Recognize AWS-supported direct, region-prefixed, and global Nova text model IDs such as `amazon.nova-*`, preserve explicit `cacheRetention` where valid, and emit only Bedrock/Nova-compatible explicit cache checkpoints for Converse/ConverseStream when AWS docs support them; do not send Claude-only `ttl: "1h"` for Nova unless the model-specific AWS contract proves it. Keep unsupported non-Anthropic Bedrock models on the existing no-cache path. Update focused tests for `resolveCacheRetention`, provider wrapping/payload patching, and config defaults as appropriate, plus `docs/reference/prompt-caching.md`, `docs/providers/bedrock.md`, and `CHANGELOG.md`. Do not move provider-specific policy into core or add a new config option.
