# Coding Plan for openclaw/openclaw#29707: [Feature Request]: Native Prompt Caching Support for Amazon Nova (Bedrock)

Source report: [records/openclaw-openclaw/items/29707.md](records/openclaw-openclaw/items/29707.md)

## Summary

Keep open: current main and v2026.5.19 still exclude direct Amazon Nova Bedrock model IDs from cache-retention eligibility and force them through the Bedrock no-cache wrapper, while current AWS Bedrock docs describe Nova prompt caching and Nova-compatible cachePoint usage. ([docs.aws.amazon.com](https://docs.aws.amazon.com/bedrock/latest/userguide/prompt-caching.html))

## Plan

Implement native Amazon Nova prompt-caching support for the bundled Amazon Bedrock provider. Keep the fix provider-owned: teach the cache-retention family/resolver and Bedrock wrapper to recognize supported Nova model ID forms such as amazon.nova-*, region-prefixed, and global-prefixed Nova IDs; preserve explicit cacheRetention only for Nova where AWS supports it; inject Bedrock/Nova-compatible cachePoint checkpoints for system/messages without sending Anthropic-only cache metadata; avoid mapping cacheRetention "long" to ttl "1h" for Nova unless AWS documents that exact support; and keep unsupported non-Anthropic Bedrock models on the cacheRetention "none" path. Update focused tests, docs/reference/prompt-caching.md, docs/providers/bedrock.md if needed, and CHANGELOG.md for the user-visible provider change. Coordinate with related resolver PRs but do not broaden every openai-completions or non-Anthropic Bedrock provider into cache eligibility.
