## Summary

Keep open: current main and the latest release still keep direct Amazon Nova Bedrock model IDs off the cache-retention path and force them through the no-cache wrapper, while AWS documents Nova prompt caching support. This remains a focused Amazon Bedrock provider enhancement with a clear implementation and validation path.

## Plan

Fix Amazon Nova prompt caching for the bundled Amazon Bedrock provider. Add capability-gated Nova prompt-cache support for direct, region-prefixed, and global Nova model IDs documented by AWS; preserve explicit `cacheRetention` only for eligible Nova models; inject only Bedrock/Nova-compatible `cachePoint` checkpoints on supported `system` and `messages` blocks; avoid blindly applying Claude 1-hour TTL semantics to Nova; and keep unsupported non-Anthropic Bedrock models on the existing no-cache path.
