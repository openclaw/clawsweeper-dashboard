# Coding Plan for openclaw/openclaw#29707: [Feature Request]: Native Prompt Caching Support for Amazon Nova (Bedrock)

Source report: [records/openclaw-openclaw/items/29707.md](records/openclaw-openclaw/items/29707.md)

## Summary

Keep open: current main still drops direct Amazon Nova Bedrock cache-retention settings and then routes direct non-Anthropic Bedrock models through a no-cache wrapper, while the current AWS Bedrock contract documents Nova prompt caching support.

## Plan

Implement native prompt caching support for Amazon Nova on the bundled Amazon Bedrock provider. Keep the fix provider-owned: recognize supported direct, region-prefixed, and global Nova model IDs such as amazon.nova-micro-v1:0, amazon.nova-lite-v1:0, amazon.nova-pro-v1:0, amazon.nova-premier-v1:0, and amazon.nova-2-lite-v1:0; allow valid explicit cacheRetention to reach the stream path; inject Bedrock/Nova-compatible cachePoint checkpoints only on supported Converse payload fields; keep unsupported non-Anthropic Bedrock models on cacheRetention: "none"; do not reuse Claude-only 1-hour TTL behavior for Nova unless AWS docs or a live probe confirms it. Update focused tests, docs/reference/prompt-caching.md, docs/providers/bedrock.md if needed, and CHANGELOG.md. Preserve the earlier fix for https://github.com/openclaw/openclaw/issues/20815 so Nova no longer receives Anthropic-specific incompatible metadata.
