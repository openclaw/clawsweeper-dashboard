# Coding Plan for openclaw/openclaw#29707: [Feature Request]: Native Prompt Caching Support for Amazon Nova (Bedrock)

Source report: [records/openclaw-openclaw/items/29707.md](records/openclaw-openclaw/items/29707.md)

## Summary

Keep open: current main and the latest release still exclude direct Amazon Nova Bedrock model IDs from cache-retention eligibility and force them through the no-cache Bedrock wrapper, while AWS documents Nova prompt caching as supported. This remains a focused Amazon Bedrock provider enhancement, not cleanup to close.

## Plan

Fix native prompt caching for supported Amazon Nova models on the bundled Amazon Bedrock provider. Add provider-owned Nova cache eligibility for supported direct, region-prefixed, and global Nova model IDs; preserve explicit `cacheRetention` only where AWS supports Bedrock prompt caching; inject only Bedrock/Nova-compatible `cachePoint` checkpoints on supported fields; avoid blindly applying the Claude-only 1-hour TTL mapping to Nova; keep unsupported non-Anthropic Bedrock models on the no-cache path; and update focused tests, public prompt-caching/provider docs, and CHANGELOG.md. Do not add a new config option or broaden caching for every non-Anthropic Bedrock model without explicit AWS support. If AWS credentials are available, run a redacted Nova Converse/ConverseStream probe for accepted cachePoint payloads and cache read/write accounting.
