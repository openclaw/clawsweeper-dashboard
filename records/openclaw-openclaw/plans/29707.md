## Summary

Keep open: current main and v2026.6.5 still classify Bedrock prompt caching as Claude-only and force direct Amazon Nova models through the no-cache wrapper, while current AWS documentation supports explicit prompt caching for Nova. No open replacement PR owns the remaining work, and the correct boundary is a focused change in the existing Amazon Bedrock plugin.

## Plan

Implement Amazon Nova prompt caching in the existing Amazon Bedrock provider plugin for https://github.com/openclaw/openclaw/issues/29707. Update cache-capability detection to recognize only AWS-documented cache-capable Nova text models and preserve the no-cache wrapper for unsupported non-Anthropic models. Add Nova-compatible Bedrock Converse `cachePoint` shaping only in supported `system` and `messages` fields, respect `cacheRetention: "none"`, do not enable Claude-only tool caching or unsupported one-hour TTL, and keep existing Claude/application-profile behavior unchanged. Add focused tests for direct and inference-profile Nova IDs, unsupported Bedrock models, none/short retention, pre-existing cache points, and Claude regressions. Validate with the focused extension test and redacted live AWS Nova proof showing cache write then cache read token metrics on repeated requests. Do not add a config or environment variable, edit CHANGELOG.md, or broaden caching to all non-Anthropic Bedrock models.
