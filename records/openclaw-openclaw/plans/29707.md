## Summary

Keep open: current main still treats direct Amazon Nova Bedrock IDs as non-Anthropic and forces them to `cacheRetention: "none"`, while AWS now documents Nova prompt caching and explicit `cachePoint` support.

## Plan

Fix https://github.com/openclaw/openclaw/issues/29707 by adding native Amazon Nova prompt-caching support in the bundled Amazon Bedrock provider. Keep core plugin-agnostic and avoid blanket-enabling unsupported Bedrock models. Recognize supported direct, region-prefixed, and global Amazon Nova model IDs; preserve explicit cache retention where valid; inject Bedrock/Nova-compatible `cachePoint` markers on supported payload fields without sending Anthropic-only metadata; keep unsupported non-Anthropic Bedrock models on the no-cache path; and avoid mapping `cacheRetention: "long"` to `ttl: "1h"` for Nova unless AWS docs or a live probe proves that TTL is supported. Update focused tests, docs/reference prompt caching, Bedrock provider docs if needed, and CHANGELOG.md for the user-visible provider feature.
