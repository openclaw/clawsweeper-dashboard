## Summary

Keep open: current main still blocks direct Amazon Nova Bedrock prompt caching even though current AWS Bedrock docs document Nova cache checkpoints and 5-minute TTL support.

## Plan

Implement native prompt caching support for AWS-supported Amazon Nova models in the bundled `amazon-bedrock` provider. Recognize documented direct, regional, and global Nova text model IDs such as Nova Micro, Lite, Pro, Premier, and Nova 2 Lite after checking current AWS model cards; preserve explicit `cacheRetention` where valid; inject Bedrock/Nova-compatible `cachePoint` checkpoints into supported `system` and `messages` payload fields; keep unsupported non-Anthropic Bedrock models on the no-cache path; and do not send `ttl: "1h"` to Nova unless current AWS docs or live proof confirm support. Update regression tests for `resolveCacheRetention`, Bedrock `wrapStreamFn`, payload cache-point injection, and docs/changelog. Related refs: this issue, https://github.com/openclaw/openclaw/issues/20815, https://github.com/openclaw/openclaw/pull/60370, https://github.com/openclaw/openclaw/pull/69953, and https://github.com/openclaw/openclaw/issues/21986.
