# Coding Plan for openclaw/openclaw#29707: [Feature Request]: Native Prompt Caching Support for Amazon Nova (Bedrock)

Source report: [records/openclaw-openclaw/items/29707.md](records/openclaw-openclaw/items/29707.md)

## Summary

Keep open: current main still forces direct Amazon Nova Bedrock model IDs onto the no-cache path, while AWS now documents Nova prompt caching and Nova-compatible `cachePoint` examples. The older Nova `cachePoint` crash fix is useful history, but it does not implement native Nova prompt-cache support. ([docs.aws.amazon.com](https://docs.aws.amazon.com/bedrock/latest/userguide/prompt-caching.html))

## Plan

Implement native Amazon Nova prompt caching for the bundled Amazon Bedrock provider. Recognize supported direct, region-prefixed, and global Nova model IDs as cache-eligible; preserve explicit `cacheRetention` where valid; inject only AWS Bedrock/Nova-compatible `cachePoint` checkpoints into `system` and `messages`; do not send Anthropic `cache_control`; keep unsupported non-Anthropic Bedrock models on the no-cache path; and avoid mapping `cacheRetention: "long"` to `ttl: "1h"` for Nova unless current AWS docs or a live call proves it is supported. Update focused tests, public prompt-caching and Bedrock docs, and add a user-facing changelog entry. Validate with targeted Vitest via `node scripts/run-vitest.mjs`; use Testbox for `pnpm check:changed`; run a live Bedrock Nova ConverseStream probe for payload acceptance and cache read/write metrics if AWS credentials are available.
