## Summary

Keep open: current main and the latest shipped release still exclude direct Amazon Nova Bedrock models from prompt-cache eligibility, so this remains a valid focused provider enhancement rather than cleanup that can be closed.

## Plan

Implement native prompt caching support for supported Amazon Nova models in the Amazon Bedrock plugin. Keep the change provider-owned and narrow: recognize AWS-documented Nova direct IDs, region-prefixed IDs, and global IDs where appropriate; preserve explicit cacheRetention only for supported Nova cache modes; inject or preserve only Bedrock/Nova-compatible cachePoint payloads; avoid Claude-only ttl: "1h" behavior unless AWS documents it for the target Nova model; and keep unsupported non-Anthropic Bedrock models on createBedrockNoCacheWrapper. Update focused tests for resolver behavior, Bedrock stream wrapping/payload patching, unsupported non-Anthropic models, public docs for Bedrock prompt caching, and CHANGELOG.md if this becomes user-facing. Do not broaden cacheRetention passthrough for all Bedrock models and do not change unrelated provider routing.
