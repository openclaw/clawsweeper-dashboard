# Coding Plan for openclaw/openclaw#29707: [Feature Request]: Native Prompt Caching Support for Amazon Nova (Bedrock)

Source report: [records/openclaw-openclaw/items/29707.md](records/openclaw-openclaw/items/29707.md)

## Summary

Keep open: current main and the latest release still force direct Amazon Nova Bedrock models out of prompt-cache retention, while the current AWS Bedrock contract documents Nova prompt caching and recommends explicit prompt caching for cost savings.

## Plan

Fix native prompt caching for Amazon Nova on Bedrock. In the Amazon Bedrock provider path, recognize AWS-documented Nova cache-capable model IDs such as direct, region-prefixed, and global `amazon.nova-*` refs; allow explicit valid `cacheRetention` to reach the Bedrock Converse stream; inject only Nova-compatible `cachePoint` checkpoints for `system` and `messages`; do not reuse Claude-only `ttl: "1h"` for Nova unless AWS documents it for that model; and keep unsupported non-Anthropic Bedrock families on the no-cache path so the older `cachePoint` rejection does not regress. Update focused tests and public prompt-caching/provider docs; do not edit `CHANGELOG.md` for a normal PR, but include user-visible release-note context in the PR body.
