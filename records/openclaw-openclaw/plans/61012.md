Keep open: current main and the latest release still bypass Telegram's default-account resolver on token-gated message actions, while the linked repair PRs are closed unmerged.

## Plan

Fix https://github.com/openclaw/openclaw/issues/61012 by making omitted-account Telegram message actions honor channels.telegram.defaultAccount. In extensions/telegram/src/action-runtime.ts, route all token-gated actions that currently call resolveTelegramToken(cfg, { accountId }) through the existing resolveTelegramAccount resolver or an equivalent small helper, so omitted accountId uses the configured default account while explicit accountId behavior is unchanged. Add focused regression coverage in extensions/telegram/src/action-runtime.test.ts for omitted accountId plus defaultAccount across sendMessage and the other token-gated Telegram actions, learning from closed PRs https://github.com/openclaw/openclaw/pull/61140 and https://github.com/openclaw/openclaw/pull/61144.
