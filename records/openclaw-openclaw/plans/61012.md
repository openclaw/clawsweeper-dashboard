Keep open: current `origin/main` still has the Telegram message-action bug where omitted-account outgoing actions bypass the default-account resolver and resolve tokens as the implicit `default` account.

## Plan

Fix the Telegram message-action default-account routing bug for https://github.com/openclaw/openclaw/issues/61012. On current `origin/main`, `extensions/telegram/src/action-runtime.ts` resolves tokens for token-gated actions with `resolveTelegramToken(cfg, { accountId })`; when `accountId` is omitted this bypasses `channels.telegram.defaultAccount` and uses the implicit `default` account. Update the Telegram action runtime to resolve omitted-account actions through `resolveTelegramAccount` and pass the resolved token and account ID consistently to send, poll, react, delete, edit, sticker, and forum-topic actions. Preserve explicit `accountId` behavior, env/default-account fallback behavior, and the unknown-account no-fallback guard that prevents wrong-bot sends. Add focused regression coverage in `extensions/telegram/src/action-runtime.test.ts` for `channels.telegram.defaultAccount = "work"` with only `accounts.work.botToken` and no top-level/default/env token, covering all token-gated Telegram actions. Add or keep explicit-account and missing/unknown-account safety assertions. Include a `CHANGELOG.md` Fixes entry because this is user-visible. Do not broaden into duplicate-token validation, startup warnings, or unrelated multi-agent routing changes unless a failing test proves they are required for this bug.
