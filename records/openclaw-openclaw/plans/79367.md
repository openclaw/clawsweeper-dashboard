## Summary

Keep open. Current main includes the older `timeoutSeconds` mitigation for resumed cron runs, but the requested per-cron `payload.noOutputTimeoutMs` API is still absent from the cron protocol, normalizer, CLI runner params, and watchdog resolver.

## Plan

Implement the requested per-cron resumed CLI no-output watchdog override for https://github.com/openclaw/openclaw/issues/79367. Add a bounded `payload.noOutputTimeoutMs` field to the Gateway cron payload schema, cron TypeScript types, cron normalizer/create/update/top-level-copy paths, and cron tool/docs guidance. Thread the field from isolated cron `agentPayload` into `runCliAgent`, add a typed CLI runner param, and update `resolveCliNoOutputTimeoutMs` so resumed CLI sessions may honor a higher per-job value bounded by 1_800_000 ms and the total run timeout guard. Preserve the current `timeoutSeconds` behavior from the existing #76289 fix and do not change the separate poisoned-resume cascade tracked at https://github.com/openclaw/openclaw/issues/79365. Add focused regression coverage for schema/normalization, executor forwarding, and resolver behavior; include a user-facing changelog entry if the final PR changes the public cron payload contract.
