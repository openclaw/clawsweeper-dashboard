# Coding Plan for openclaw/openclaw#83748: Telegram inbound stickers are not hydrated as agent-readable media

Source report: [records/openclaw-openclaw/items/83748.md](records/openclaw-openclaw/items/83748.md)

## Summary

Keep open: current main and v2026.6.6 still expose Telegram cached media to the model as `telegram:file/...` refs without a hydrated media path in conversation context, and this issue has sticker-specific static WEBP behavior that is not fully owned by the broader photo issue.

## Plan

Fix Telegram inbound static sticker hydration so agent/model-visible context gets an image-tool-readable path (`media/inbound/*`, absolute staged path, or canonical `media://inbound/*`) instead of only `telegram:file/<file_id>`. Keep the fix inside the Telegram plugin/context handoff; do not teach the core image tool to resolve Telegram file IDs. Preserve existing static WEBP support and keep animated TGS/video WEBM sticker skip policy unchanged unless a maintainer explicitly expands scope. Add regression coverage for a static sticker current turn and a cached/reply/conversation-context path that currently exposes only `media_ref`, and include https://github.com/openclaw/openclaw/issues/87401 as the related broader Telegram media-ref cluster. Do not edit CHANGELOG.md.
