## Summary

Keep open. Current main still selects heartbeat-scoped model and thinking values for execution, but session_status for semantic current is built from stored session/default state and lacks a transient current-run status context.

## Plan

Fix https://github.com/openclaw/openclaw/issues/77493. In a heartbeat run, session_status({"sessionKey":"current"}) should report the active per-run heartbeat model and current run thinking level, not the stored/default session values. Keep the fix transient: do not persist heartbeat model or inline thinking into SessionEntry, and do not change explicit non-current session lookups. Likely approach: carry current-run status context from get-reply/runEmbeddedAttempt through createPiTools/createOpenClawTools into createSessionStatusTool, apply it only when the request is semantic current, and leave session model override mutation behavior unchanged. Use https://github.com/openclaw/openclaw/pull/77553 as prior art for the model side, but extend coverage for thinking because the reporter added May 7 evidence. Coordinate with https://github.com/openclaw/openclaw/issues/74217 so the fix does not reintroduce heartbeat model leakage. Add a user-facing CHANGELOG.md fix entry.
