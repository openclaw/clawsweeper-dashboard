# Coding Plan for openclaw/openclaw#56818: [Bug] Feishu card action callback causes streaming reply to fail with Invalid open_message_id

Source report: [records/openclaw-openclaw/items/56818.md](records/openclaw-openclaw/items/56818.md)

## Summary

Keep open. Current main and v2026.5.22 still normalize Feishu card-action `open_message_id` by taking the top-level value before `context.open_message_id`, so a temporary `card-action-c-...` callback id can still become the streaming reply target; the related card-update PR is adjacent but does not close or solve this exact parser precedence bug.

## Plan

Fix https://github.com/openclaw/openclaw/issues/56818. In the Feishu plugin, make card-action callback normalization prefer a real `context.open_message_id` over temporary top-level callback ids such as `card-action-c-*`; keep SDK-flat real top-level `open_message_id` support, and make temporary-only or no-real-id callbacks plain-send without reply metadata. Add focused lifecycle regression coverage for a payload containing both top-level `open_message_id: "card-action-c-..."` and `context.open_message_id: "om_original"`, plus a temporary-only fallback case if not already covered. Keep the fix within `extensions/feishu`, do not pull in the in-place card update feature from https://github.com/openclaw/openclaw/pull/64027, and add a user-visible changelog fix entry.
