## Summary

Keep open. Current main and the latest release still select a top-level Feishu card-action `open_message_id` before `context.open_message_id`, so the reported temporary callback ID can still become the streaming reply target.

## Plan

Fix the Feishu card-action callback reply-target bug from https://github.com/openclaw/openclaw/issues/56818. Current main parses `open_message_id` with top-level precedence, so a temporary `card-action-c-*` value can override the real `context.open_message_id` and then reach streaming replies as `replyToMessageId`. Keep the fix inside the Feishu plugin boundary: prefer the real context ID when the top-level ID is synthetic/temporary, preserve SDK-flat real top-level IDs, and preserve no-real-ID suppress/plain-send fallback. Add focused regression coverage for a payload with top-level `open_message_id: "card-action-c-temp"` plus `context.open_message_id: "om_original"` proving the dispatcher uses `om_original`. Do not change core reply dispatch or the unrelated in-place card-update feature path from https://github.com/openclaw/openclaw/pull/64027. Add a user-facing changelog fix entry.
