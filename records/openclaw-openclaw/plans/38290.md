## Plan

Repair the Gateway exact custom-scheme origin PR without broadening `gateway.controlUi.allowedOrigins`. Keep the incoming-origin fallback for hosted non-special schemes such as `chrome-extension://<id>`, `app://localhost`, `tauri://localhost`, and `electron://localhost`, but preserve configured allowlist entries as trimmed/lowercased exact strings rather than parsing them through `URL.origin`. Add regression coverage that an exact `chrome-extension://...` or custom-scheme entry matches, a mismatched one rejects, literal `Origin: null` still rejects, wildcard behavior is unchanged, and a configured standard URL with a path such as `https://trusted.example/path` does not authorize `https://trusted.example`. Include the Gateway WebSocket handshake coverage already proposed here or an equivalent consolidated test. Do not add scheme wildcard support; leave that to https://github.com/openclaw/openclaw/issues/39739. Coordinate with https://github.com/openclaw/openclaw/pull/73511 so only one normalization contract lands. Changelog is maintainer-owned during landing if this becomes user-facing release work.
