# Coding Plan for openclaw/openclaw#72927: feat(tts): support MiMo v2.5 TTS VoiceDesign

Source report: [records/openclaw-openclaw/items/72927.md](records/openclaw-openclaw/items/72927.md)

## Summary

Keep open. Current main still has Xiaomi MiMo TTS wired only for built-in voices plus style text, while Xiaomi's current public docs define VoiceDesign as a separate `mimo-v2.5-tts-voicedesign` request path that OpenClaw does not parse, serialize, test, or document.

## Plan

Implement Xiaomi MiMo VoiceDesign support for https://github.com/openclaw/openclaw/issues/72927. Keep the change scoped to the bundled Xiaomi speech provider and docs/tests. In `extensions/xiaomi/speech-provider.ts`, add support for the official `mimo-v2.5-tts-voicedesign` model and a documented `voiceDesign` config shape, for example `voiceDesign.prompt` plus optional `voiceDesign.optimizeTextPreview`, mapping the prompt to the required user-role message and the text to the assistant-role message. For VoiceDesign requests, send `audio.format` and optional `audio.optimize_text_preview` and do not send a preset `audio.voice`; preserve existing `mimo-v2.5-tts` and `mimo-v2-tts` voice/style behavior unchanged. Do not add core TTS API changes or implement VoiceClone/per-call override behavior unless it is strictly required for this feature. Add focused unit tests for config normalization, request payload, backward compatibility, and invalid/missing prompt behavior. Update `docs/providers/xiaomi.md`, `docs/tools/tts.md`, and `CHANGELOG.md` for the user-facing feature. If credentials are available, add or run a live VoiceDesign check with redacted proof; otherwise keep live proof optional and explain the missing credential.
