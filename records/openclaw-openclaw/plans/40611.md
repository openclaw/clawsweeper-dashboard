## Summary

Keep open: current main and v2026.5.19 still route retryable heartbeat busy skips through the shared 1 second wake retry, so the active-reply guard does not resolve the post-turn heartbeat race reported here.

## Plan

Fix the heartbeat busy-skip retry race reported in https://github.com/openclaw/openclaw/issues/40611. Current main returns retryable busy skip reasons for main/session/active-reply work, and `src/infra/heartbeat-wake.ts` retries all retryable busy skips after `DEFAULT_RETRY_MS = 1_000`, letting heartbeat runs win the race immediately after a user turn releases the session. Add an internal typed debounce/backoff or session-idle signal for request/session busy reasons so heartbeat does not immediately grab the session after active conversations, while preserving fast retry for cron/error/transient paths and preserving the no-drift behavior from the earlier scheduling fix. Do not add a public `heartbeat.retryDelayMs` config option unless maintainers explicitly ask; keep the broader activity-based idle-timeout feature in https://github.com/openclaw/openclaw/issues/14051 separate. Update focused tests for wake retry timing, scheduler behavior, and session-lane/active-reply busy skips; update docs only if user-visible heartbeat skip behavior wording changes.
