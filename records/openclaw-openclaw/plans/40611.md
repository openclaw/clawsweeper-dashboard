## Plan

Fix the heartbeat busy-skip retry regression reported in https://github.com/openclaw/openclaw/issues/40611. Current main returns retryable busy skip reasons for main/session/agent busy pressure and heartbeat-wake retries them through DEFAULT_RETRY_MS=1000, which can fire heartbeat work immediately after an active conversation drains. Add a heartbeat-owned reason-specific debounce/backoff for conversation/session busy skips, preserving PR 39182's no-drift behavior and preserving fast retry for unrelated thrown handler errors. Keep the change bounded to heartbeat wake/runner scheduling and focused tests; do not implement the broader activity-based idle-timeout feature from https://github.com/openclaw/openclaw/issues/14051 or the non-blocking heartbeat architecture from https://github.com/openclaw/openclaw/issues/52046.
