## Plan

Implement opt-in Gemini web search support for thinking and URL context in the bundled Google plugin. Keep the boundary inside the Google plugin and shared public plugin contracts; do not move provider-specific behavior into core. Add typed/config parsing for `plugins.entries.google.config.webSearch` fields such as a numeric thinking budget or level and a boolean URL-context toggle, update `extensions/google/openclaw.plugin.json` schema/ui hints, and preserve current default behavior unless maintainers explicitly choose a default-on `url_context`. In `extensions/google/src/gemini-web-search-provider.runtime.ts`, include `{ url_context: {} }` only when enabled and include `generationConfig.thinkingConfig` only when configured; include the new effective options in the search cache key. Add focused tests in `extensions/google/web-search-provider.test.ts`, update `docs/tools/gemini-search.md`, and add a user-facing changelog entry. Stop and ask for maintainer direction if the implementation would require changing defaults, adding a cross-provider web-search API, or broadening SSRF/network policy.
