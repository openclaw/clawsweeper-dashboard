# Coding Plan for openclaw/openclaw#37842: Feature: Graph-aware loop detection for sessions_send (triangular/polygonal cascades bypass maxPingPongTurns)

Source report: [records/openclaw-openclaw/items/37842.md](records/openclaw-openclaw/items/37842.md)

## Summary

Keep open: current main and the latest release still lack graph-aware `sessions_send` chain provenance, and the only focused implementation PR was closed unmerged. The source still models A2A turns as an immediate requester/target pair, so triangular or polygonal cascades remain source-reproducible.

## Plan

Implement graph-aware cycle detection for `sessions_send` without changing the existing pairwise `maxPingPongTurns` contract. Add an optional, typed, normalized chain field to `InputProvenance` and the gateway protocol schema; thread existing incoming provenance into `createSessionsSendTool`; append the canonical requester agent ID on outbound inter-session sends; reject delivery when the canonical target agent ID already appears in the chain; preserve legacy/non-agent session behavior and the current A2A reply-back flow. Use the closed PR https://github.com/openclaw/openclaw/pull/37945 as source context only, and address its normalization concern instead of copying raw-ID comparison. Add regression coverage for A->B->C->A, self-send or A->B->A revisits, absent provenance, malformed chain values, and non-agent session keys. Add a changelog entry only if maintainers treat the new user-visible guard as release-note worthy.
