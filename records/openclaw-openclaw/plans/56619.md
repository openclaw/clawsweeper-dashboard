## Summary

Keep open. Current main and the latest release still expose the generic pairing approval notification contract, but WhatsApp does not register an approval notifier and the CLI/generic notifier path still drops account scope, so the approved sender receives no WhatsApp confirmation even when `--notify` is used.

## Plan

Fix https://github.com/openclaw/openclaw/issues/56619 by making WhatsApp honor the existing `openclaw pairing approve ... --notify` contract. Thread `--account` through the pairing approval notification path, pass `accountId` into `notifyPairingApproved`, and add a WhatsApp `notifyApproval` implementation that sends the standard approval text to the approved sender through the existing WhatsApp outbound path. Add focused regression coverage for CLI account forwarding, generic notifier account forwarding/no-op behavior, and WhatsApp approval notification sending. Do not make notification default-on, add a new config option, or replay the originally blocked message in this PR; call those out as separate product decisions if needed. Add a user-facing changelog fix entry.
