## Summary

Keep open: current main and the latest release still expose `pairing approve --notify`, but WhatsApp has no approval notifier and the CLI notification path still drops `--account`, so the approved sender does not receive the promised confirmation.

## Plan

Fix the WhatsApp pairing approval notification gap for the existing `openclaw pairing approve --notify` contract. Keep the scope narrow: pass `--account` through `src/cli/pairing-cli.ts` into `notifyPairingApproved`, include `accountId` in `src/channels/plugins/pairing.ts` when calling the adapter, and add a WhatsApp pairing notifier in `extensions/whatsapp/src/channel.ts` that sends the standard approval message through the existing WhatsApp outbound/send path for the resolved account. Add focused tests for CLI account forwarding, generic notifier account forwarding/no-op behavior, and WhatsApp notifier delivery using mocked outbound/listener dependencies. Do not add default-on behavior, new config keys, custom message text, replay of the originally blocked message, or broad pairing management commands in this PR.
