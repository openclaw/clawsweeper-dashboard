## Summary

Keep open. Current main and the latest release still normalize malformed Anthropic SSE JSON into a stable transport sentinel, but that sentinel is not classified as a retryable failover signal, so the reported failover recovery gap remains.

## Plan

Fix the Anthropic malformed streaming JSON failover gap for https://github.com/openclaw/openclaw/issues/60593. Current main normalizes malformed Anthropic SSE JSON to `OpenClaw transport error: malformed_streaming_fragment`, but `classifyFailoverReason` and `isFailoverAssistantError` do not classify that sentinel, so embedded assistant failover can still surface the error with `failoverReason=null`. Add a narrow classifier path for the normalized sentinel as a retryable transport-style reason, add focused coverage for the classifier and embedded failover path, and preserve the existing behavior that arbitrary raw JSON.parse diagnostics and provider request-validation diagnostics are not broadly rewritten or retried. Do not add a config option, do not make all JSON.parse messages retryable, and do not change Anthropic streaming parsing beyond what is needed for failover classification. Add a user-facing changelog fix entry if this repair is promoted to a PR.
