## Summary

Keep open: current main and v2026.5.20 still route Windows node service lifecycle through shared Scheduled Task Gateway cleanup, so a node service can treat its remote Gateway connection port as a local listener to reclaim.

## Plan

Fix https://github.com/openclaw/openclaw/issues/85289 by preventing Windows node service lifecycle commands from reclaiming or waiting on the node service's remote Gateway connection port. Keep Gateway Scheduled Task stale-listener cleanup intact, but use existing service-kind metadata such as `OPENCLAW_SERVICE_KIND` so node services skip `terminateScheduledTaskGatewayListeners`, `waitForGatewayPortRelease`, and fallback busy-listener termination. Add regression tests for node stop/restart/start-through-restart so a node task using `node run --host 127.0.0.1 --port 18789` does not inspect or kill listeners on 18789, and keep existing Gateway cleanup tests passing. Do not add WSL-specific process-name filters, do not add a config option, and do not weaken Gateway service cleanup.
