## Plan

On current main, `GatewaySession.connectOnce` assigns `currentConnection` before `conn.connect()` completes, while `sendNodeEvent` sends through any non-null current connection; `NodeRuntime` forwards notification events without checking the Boolean result, and `DeviceNotificationListenerService` emits posted notifications once. Keep the Gateway protocol unchanged. Implement an Android-side readiness guard or publish `currentConnection` only after the connect ACK, and add bounded retry/queue/reconciliation so `notifications.changed` events arriving during reconnect are delivered after reconnect without unbounded memory growth or duplicate wakes.
