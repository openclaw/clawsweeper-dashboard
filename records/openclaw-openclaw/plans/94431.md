Keep open: this is a focused fix for a real shipped CLI regression, but the new hoist helper has one blocking parser edge case where a parent value option with no value is moved before the subcommand and can consume the subcommand or next flag as its value.

## Plan

Repair PR https://github.com/openclaw/openclaw/pull/94431 without broad refactors. Keep the parent-option hoist in `src/cli/program/helpers.ts`, but change `consumeHoistableOption` so value options are hoisted only when the value is present in the same `--flag=value` token or the next token is a hoistable value. A bare required/optional parent option after the lazy subcommand must remain in place so Commander reports the normal missing/unknown-option path and does not consume the subcommand name or another flag as the value.
