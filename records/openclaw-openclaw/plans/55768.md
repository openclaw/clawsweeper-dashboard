## Summary

Keep open. Current main still makes anonymous OpenAI-compatible Chat Completions a normal agent run with a generated session key, while public docs still do not clearly tell uptime monitors to use the lightweight health/readiness probes instead.

## Plan

Fix the monitoring guidance for https://github.com/openclaw/openclaw/issues/55768 with a focused docs-only PR. Document that external uptime monitors should use `/health` or `/healthz` for liveness and `/ready` or `/readyz` for readiness, and should not use `/v1/chat/completions` because that endpoint executes a normal agent run, creates/uses an agent session key, can persist transcript/session state, and may call an LLM. Update `docs/gateway/health.md` and, if useful, `docs/gateway/openai-http-api.md`; keep wording aligned with the existing `/health`/`/healthz` liveness and `/ready`/`/readyz` readiness split. Do not implement runtime warnings, metrics, rate limiting, passthrough mode, session persistence changes, or config schema changes in this PR. No changelog entry is expected for a docs-only fix.
