Current main still sends anonymous `/v1/chat/completions` calls through generated OpenAI session keys and the normal agent execution path, while the public docs still lack explicit uptime-monitor guidance warning users to use the lightweight HTTP probes instead; the related docs PR was closed unmerged and the broader no-persistence relay request remains separate.

## Plan

Prepare a narrow docs-only PR for https://github.com/openclaw/openclaw/issues/55768. Add public gateway monitoring guidance that tells external uptime monitors to use `GET /health` or `/healthz` for liveness and `GET /ready` or `/readyz` for readiness, and explicitly warns not to use `POST /v1/chat/completions` as a health check because it runs the normal agent path, may call the model, and persists a generated session when no `user` or `x-openclaw-session-key` is supplied. Prefer updating `docs/gateway/health.md` and adding a short cross-reference/warning in `docs/gateway/openai-http-api.md`; include practical examples for common HTTP monitors without changing runtime behavior.
