# Coding Plan for openclaw/openclaw#55768: Health check bloat: uptime monitors must use /health, not /v1/chat/completions

## Summary

Keep open. Current `main` and the latest shipped release still route anonymous OpenAI-compatible Chat Completions through the normal agent/session path, and the public health/OpenAI HTTP docs still do not clearly warn uptime monitors away from that endpoint; the closed related docs PR attempted the right idea but was not merged.

## Plan

Create a docs-only PR for https://github.com/openclaw/openclaw/issues/55768. Add public guidance in `docs/gateway/health.md` and `docs/gateway/openai-http-api.md` that uptime monitors should use `GET /health` or `/healthz` for liveness and `/ready` or `/readyz` for readiness, and should not use `POST /v1/chat/completions` because it runs the normal agent path, creates/persists sessions for anonymous requests, and may call an LLM. Touch deployment docs only where they already discuss probes. Do not change gateway runtime behavior, add metrics/rate limits, or introduce passthrough/ephemeral mode; point broader no-persistence product work to https://github.com/openclaw/openclaw/issues/62436 if mentioned. Keep examples generic and do not add a changelog for a docs-only fix.
