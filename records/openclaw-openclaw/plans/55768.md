## Summary

Keep open. Current main and the latest release still route anonymous `/v1/chat/completions` calls through generated `openai:<uuid>` agent session keys and the normal persisted agent path, while the public health docs still do not clearly warn uptime monitors to use `/health` or `/healthz` instead.

## Plan

Fix OpenClaw issue https://github.com/openclaw/openclaw/issues/55768 with a docs-only change. Add focused public guidance that external uptime monitors must use the Gateway probe endpoints (`/health` or `/healthz` for liveness, `/ready` or `/readyz` when readiness is required) and must not use `/v1/chat/completions` as a health check because that endpoint runs the normal agent path and can create persisted sessions for anonymous requests.
