# Coding Plan for openclaw/openclaw#79366: Track and log pending MCP tool calls in claude-live-session for post-mortem visibility

Source report: [records/openclaw-openclaw/items/79366.md](records/openclaw-openclaw/items/79366.md)

## Summary

Keep open: current main still has generic `claude-live-session` no-output and failure logs with no pending tool-call state, and the related resume-recovery and trajectory issues do not provide the requested per-tool attribution.

## Plan

Implement pending MCP/native tool-call diagnostics for `claude-cli` live sessions without changing recovery behavior. In `src/agents/cli-runner/claude-live-session.ts`, track pending tool calls for the current turn from the actual Claude stream-json event shapes, clear them on the matching completion/result/end events and on turn/session cleanup, and emit a `cliBackendLog.warn` when a pending tool exceeds a bounded threshold. Include provider, model, sanitized tool name, sanitized tool-use id, ageMs, and turn/session context where already available; do not log raw tool arguments, raw payloads, auth data, file contents, or secrets. Add focused regression coverage in `src/agents/cli-runner.spawn.test.ts` or a colocated cli-runner test that simulates a tool start, advances timers past the warning threshold, verifies one warning, then verifies the warning is cleared/suppressed after the tool result and after fail/close cleanup. Coordinate with resume-recovery work so this PR stays diagnostic only; do not alter stored session invalidation, fallback, watchdog timeout, or config contracts. Add a changelog entry only if maintainers consider the new diagnostic user-facing.
