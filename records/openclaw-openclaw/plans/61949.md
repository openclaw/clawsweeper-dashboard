## Summary

Keep open. Current main and the latest release still render model aliases with guidance to prefer aliases while listing every configured alias, so an LLM can replace an exact provider/model before runtime preservation logic sees it; adjacent alias reports are closed or cover different resolver paths.

## Plan

Fix https://github.com/openclaw/openclaw/issues/61949. Source-proven bug: the agent system prompt currently tells models to prefer configured aliases and lists all configured aliases, which can cause `sessions_spawn` model arguments to swap provider when an exact provider/model was requested. Keep the fix bounded to prompt/alias presentation and regression tests: exact `provider/model` strings must be described as pass-through values, aliases should be framed only for unqualified requests, and overlapping providers/fallbacks must not be implied as interchangeable. Do not change provider fallback policy, auth routing, model resolver compatibility, or add a config option unless source review proves the prompt-only fix is insufficient. Adjacent refs `https://github.com/openclaw/openclaw/pull/57966` and `https://github.com/openclaw/openclaw/issues/82680` are related alias-resolver work but do not cover this prompt-choice defect.
