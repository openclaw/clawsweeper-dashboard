# Coding Plan for openclaw/openclaw#85521: [Bug]: saveAuthProfileStore overwrites runtime auth-profile snapshot with external-CLI-filtered view, dropping OAuth credentials from in-process state

## Summary

Current main still has a source-backed auth-profile snapshot bug: `saveAuthProfileStore` writes a persistence-filtered `localStore` back into the active runtime snapshot, and embedded-agent auth paths can consume snapshot-backed stores without external CLI profiles. This should stay open as a narrow fix candidate rather than being closed as stale or already implemented.

## Plan

Fix https://github.com/openclaw/openclaw/issues/85521. Current main lets `saveAuthProfileStore` build a persistence-filtered `localStore` and then install that same filtered store as the active runtime auth-profile snapshot, which can drop external CLI OAuth profiles from in-process state. Keep external CLI credentials out of persisted local auth files, but ensure runtime snapshots preserve or re-overlay runtime-safe external CLI auth profiles before snapshot-backed embedded-agent auth reads use them. Add focused regression coverage for a runtime snapshot containing an external CLI OAuth profile, a save path that filters it for disk, and a subsequent embedded/runtime auth read that still sees the external profile. Do not persist runtime-only external CLI secrets to `auth-profiles.json`, do not add a config option, and do not change plugin SDK/API contracts. Add a concise `CHANGELOG.md` fix entry unless maintainers decide this is already covered by release notes.
