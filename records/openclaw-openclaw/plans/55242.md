## Summary

Keep open. Current `main` and the latest release still interpolate raw `identity.emoji` into Feishu structured-card headers in reply and outbound paths, so prose-plus-emoji identity values can be shown visibly in card titles.

## Plan

Fix https://github.com/openclaw/openclaw/issues/55242. Current Feishu card header construction leaks prose from `identity.emoji` into structured-card titles in `extensions/feishu/src/reply-dispatcher.ts` and `extensions/feishu/src/outbound.ts`. Add a Feishu-local helper for identity-derived card titles and use it consistently for streaming replies, static replies, presentation-card identity fallbacks, and outbound card sends.
