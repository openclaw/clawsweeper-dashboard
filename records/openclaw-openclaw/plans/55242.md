## Summary

Keep open. Current main and v2026.5.18 still pass raw `identity.emoji` text into Feishu structured-card headers in reply and outbound paths, so mixed prose-plus-emoji values can be rendered visibly; the adjacent header-hide PR does not sanitize identity text.

## Plan

Fix the Feishu card header leak reported in https://github.com/openclaw/openclaw/issues/55242.
