# Coding Plan for openclaw/openclaw#55242: [Bug] Feishu card message header leaks identity.emoji field containing non-emoji text

Source report: [records/openclaw-openclaw/items/55242.md](records/openclaw-openclaw/items/55242.md)

## Summary

Keep open. Current main and the latest release still pass raw `identity.emoji` text into Feishu card headers, so mixed prose-plus-emoji identity values can become visible card titles; the adjacent header-hide PR does not sanitize this value.

## Plan

Fix https://github.com/openclaw/openclaw/issues/55242 as a narrow Feishu plugin bug. Add a shared Feishu card-header title resolver for identity-derived headers so `identity.emoji` contributes only when it is a pure emoji sequence; mixed prose such as `根据心情/语气自由切换 😊😅` must not appear in card headers. Apply the resolver consistently in `extensions/feishu/src/reply-dispatcher.ts` streaming/static paths and `extensions/feishu/src/outbound.ts` presentation/direct outbound paths, while preserving explicit presentation titles and existing generic-main header suppression. Add focused regression coverage for mixed prose-plus-emoji, pure emoji, empty/missing identity name, and existing explicit title behavior. Do not turn this into the broader header/footer config work tracked by https://github.com/openclaw/openclaw/issues/60748 or the header-hide option in https://github.com/openclaw/openclaw/pull/54979. Add a changelog entry because this is user-visible.
