# Coding Plan for openclaw/openclaw#55242: [Bug] Feishu card message header leaks identity.emoji field containing non-emoji text

Source report: [records/openclaw-openclaw/items/55242.md](records/openclaw-openclaw/items/55242.md)

## Summary

Keep open. Current main and the latest release still pass raw `identity.emoji` text into Feishu card headers, so a mixed prose-plus-emoji value can be serialized into the visible card title; the related header-hide PR does not sanitize identity text.

## Plan

Fix the Feishu card header identity leak reported at https://github.com/openclaw/openclaw/issues/55242. Add a shared Feishu-local helper for identity-derived card header titles and use it in `reply-dispatcher.ts` streaming/static card paths plus `outbound.ts` presentation fallback and direct outbound card paths. Preserve explicit presentation titles, keep a pure emoji sequence only when safely detected, otherwise fall back to the identity name or non-main agent id, and do not change global identity parsing or add a new config option. Add focused tests for mixed CJK/prose plus emoji, pure emoji, name-only identity, generic `main` header suppression, and presentation-title precedence. Include a user-facing changelog fix entry.
