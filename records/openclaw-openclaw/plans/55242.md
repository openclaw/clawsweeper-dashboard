# Coding Plan for openclaw/openclaw#55242: [Bug] Feishu card message header leaks identity.emoji field containing non-emoji text

Source report: [records/openclaw-openclaw/items/55242.md](records/openclaw-openclaw/items/55242.md)

## Summary

Keep open. Current `main` and `v2026.5.22` still compose Feishu card headers from raw `identity.emoji` in reply/streaming, payload/presentation, and direct outbound card paths, so mixed prose-plus-emoji identity values can still leak into visible card titles.

## Plan

Fix the Feishu card header identity leak reported in https://github.com/openclaw/openclaw/issues/55242. Add a Feishu-plugin-local helper for identity-derived card header titles and use it in `extensions/feishu/src/reply-dispatcher.ts` and `extensions/feishu/src/outbound.ts` so reply/streaming, payload/presentation, and direct outbound structured-card paths do not display non-emoji prose from `identity.emoji`. Preserve explicit presentation titles and agent names; keep pure emoji sequences working; do not add a new public config option and do not change global identity parsing unless a maintainer explicitly asks. Be aware of related but separate header-config work in https://github.com/openclaw/openclaw/pull/54979 and broader UX tracking in https://github.com/openclaw/openclaw/issues/60748. Add focused regression tests for mixed prose-plus-emoji and pure emoji identity values, and add a `CHANGELOG.md` fix entry because this is user-visible.
