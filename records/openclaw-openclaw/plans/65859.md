# Coding Plan for openclaw/openclaw#65859: [Feature]: Add audio upload and playback support in webchat

Source report: [records/openclaw-openclaw/items/65859.md](records/openclaw-openclaw/items/65859.md)

## Summary

Keep open. Current main has partial WebChat audio support, but source inspection still shows two remaining gaps: explicit audio files with `.webm`/`.mp4` names can be filtered out during upload, and local TTS audio is emitted as base64 `type: "audio"` content blocks while the Control UI render path only turns `attachment` blocks into playable audio cards.

## Plan

Fix the remaining WebChat audio upload classifier gap for https://github.com/openclaw/openclaw/issues/65859. Current main already sets `CHAT_ATTACHMENT_ACCEPT` to include `audio/*`, but `isSupportedChatAttachmentFile` rejects `.mp4` and `.webm` filenames even when the browser reports an explicit `audio/*` MIME type. Update `ui/src/ui/chat/attachment-support.ts` so explicit audio MIME is accepted before the video-container extension blacklist, while preserving rejection for `video/*` and true video/container uploads with missing or video MIME. Add focused coverage in `ui/src/ui/views/chat.test.ts` for at least `audio/webm` or `audio/mp4` acceptance and existing `video/mp4` rejection. Do not change gateway TTS playback, assistant-media routing, or the active playback PR surfaces tracked by https://github.com/openclaw/openclaw/pull/81203 and https://github.com/openclaw/openclaw/pull/81204. Add a concise `CHANGELOG.md` fix entry because this is user-visible.
