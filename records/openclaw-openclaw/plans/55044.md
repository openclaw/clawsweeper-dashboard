## Summary

Keep open: current main still uses the same durable `abortedLastRun` marker for explicit stop/no-action aborts and restart-interruption recovery, so a stopped running main session with a resumable transcript tail can still receive the synthetic restart-resume turn. The related open PR only covers channel-backed restart recovery and does not resolve explicit user-abort intent.

## Plan

Fix https://github.com/openclaw/openclaw/issues/55044 so explicit `/stop` and natural-language no-action aborts cannot later trigger the main-session restart-recovery synthetic `agent` resume turn. Split durable explicit user-abort intent from restart-interruption recovery state, or add an equivalent restart-owned marker, so `markRestartAbortedMainSessionsFromLocks` can still mark true restart interruptions while `persistAbortTargetEntry`, fast aborts, and lifecycle aborted/killed updates do not make an explicit stop eligible for `recoverRestartAbortedMainSessions`. Preserve existing abort cutoff semantics, subagent/cron/ACP skip behavior, pending-final-delivery recovery for true restart-aborted sessions, and non-channel restart recovery. Add focused regression coverage showing an explicit stop/abort-marked running main session with a resumable transcript tail is not resumed, while a restart-marked session still is. Coordinate with open PR https://github.com/openclaw/openclaw/pull/74687 because it touches the same recovery module, but do not treat channel-only replay prevention as the fix for this issue.
