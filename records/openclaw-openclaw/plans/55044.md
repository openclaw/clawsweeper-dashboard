# Coding Plan for openclaw/openclaw#55044: [Bug]: ignored an explicit stop/no-action instruction

Source report: [records/openclaw-openclaw/items/55044.md](records/openclaw-openclaw/items/55044.md)

## Summary

Keep open: current main still uses the same `abortedLastRun` flag for explicit stop/no-action aborts and restart-owned recovery, so a stopped running main session with a resumable transcript tail can still receive the hidden continuation turn. The related open PR is useful context but is scoped to channel-backed restart replay and does not resolve the explicit user-abort/restart-recovery conflation.

## Plan

Fix https://github.com/openclaw/openclaw/issues/55044 by separating explicit user abort state from restart-owned main-session recovery eligibility. Keep `/stop` and standalone stop/no-action phrases durable, but ensure they cannot make `recoverRestartAbortedMainSessions` inject a hidden continuation turn; preserve automatic recovery for true gateway-restart interruptions by setting and checking a restart-owned marker in the restart-marker paths. Do not add a config option, disable true restart recovery, or treat the channel-only behavior in https://github.com/openclaw/openclaw/pull/74687 as the whole fix. Add focused regression coverage for explicit `/stop` and fast abort/no-action paths with a resumable transcript tail, plus coverage that a true restart marker still resumes. Update `SessionEntry` typing/schema-adjacent tests as needed and add a single-line changelog fix entry if this becomes a maintainer landing PR.
