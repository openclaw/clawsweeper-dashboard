## Summary

Keep open: current main and the latest release still let explicit stop/no-action aborts set the same `abortedLastRun` marker that restart recovery later treats as permission to inject a hidden continuation turn.

## Plan

Fix https://github.com/openclaw/openclaw/issues/55044 on current main. Preserve the existing public stop contract: `/stop`, `chat.abort`, and standalone abort/no-action phrases must terminate user-requested work and must not make restart recovery synthesize a continuation turn later. Split restart-recovery eligibility from explicit user-abort intent in the session state model or equivalent internal marker, migrate current internal callers so gateway restart/interrupted-work paths still recover, and make explicit stop/no-action paths terminal for synthetic recovery. Coordinate with the open stale-session recovery work at https://github.com/openclaw/openclaw/pull/84157; do not add protocol/config flags, do not broaden plugin/API surface, and do not change normal active-run abort semantics beyond preventing explicit aborts from being treated as restart-interrupted work.
