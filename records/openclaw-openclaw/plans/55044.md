## Summary

Keep open: current main and the latest release still use `abortedLastRun` for both explicit stop/no-action aborts and restart-recovery eligibility, so a running session with a resumable transcript tail can still receive a hidden synthetic recovery turn after the user asked it to stop.

## Plan

Fix the session restart-recovery bug where explicit user stop/no-action aborts can be treated as restart-recovery eligible. Preserve automatic recovery for true restart interruptions, but persist or infer explicit user-abort intent separately from restart cleanup so `recoverRestartAbortedMainSessions` does not inject the hidden recovery message after /stop or natural-language no-action aborts. Add regression coverage for /stop and no-action abort phrases. Coordinate with stale-running-row work such as https://github.com/openclaw/openclaw/pull/84157, but keep this fix within session lifecycle, abort persistence, and restart recovery; do not add a new user-facing config option or weaken real restart recovery.
