# Coding Plan for openclaw/openclaw#55044: [Bug]: ignored an explicit stop/no-action instruction

Source report: [records/openclaw-openclaw/items/55044.md](records/openclaw-openclaw/items/55044.md)

## Summary

Keep open: current main and latest release still use `abortedLastRun` for both explicit stop/no-action aborts and restart-recovery eligibility, so a running session with a resumable transcript tail can still receive a hidden synthetic recovery turn after the user asked it to stop.

## Plan

Fix https://github.com/openclaw/openclaw/issues/55044. Current main persists explicit `/stop` and standalone no-action abort phrases by setting `abortedLastRun`, and main-session restart recovery also treats any running entry with `abortedLastRun: true` as eligible for synthetic resume. Introduce a separate restart-recovery eligibility marker/source, or an equivalent explicit-user-abort guard, so explicit user stops are never resumed by `recoverRestartAbortedMainSessions`; preserve automatic recovery for sessions marked by real restart cleanup. Add regression coverage for `/stop` and natural-language abort/no-action with a running session and resumable transcript tail proving recovery does not call the gateway, plus coverage that true restart-marked interrupted sessions still recover. Coordinate with https://github.com/openclaw/openclaw/pull/84157, https://github.com/openclaw/openclaw/pull/83803, and https://github.com/openclaw/openclaw/pull/74687 if they touch the same session-state semantics. Do not add a user config flag and do not weaken existing stop/subagent abort behavior. Add a `CHANGELOG.md` Fixes entry if this is prepared as a maintainer/ClawSweeper fix PR.
