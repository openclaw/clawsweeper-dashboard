# Coding Plan for openclaw/openclaw#68916: [Bug]: ACP oneshot sessions leave orphaned processes — session reset does not clean up child ACP session keys

Source report: [records/openclaw-openclaw/items/68916.md](records/openclaw-openclaw/items/68916.md)

## Summary

Keep open. Current main still closes only the reset/delete target ACP runtime and does not enumerate ACP child session entries whose `spawnedBy` or `parentSessionKey` points at the parent, so the reported reset cleanup leak remains source-reproducible.

## Plan

Fix https://github.com/openclaw/openclaw/issues/68916. In `src/gateway/session-reset-service.ts`, add parent-owned ACP child cleanup for session reset/delete: enumerate ACP session entries with the existing ACP session metadata listing seam, find entries whose `entry.spawnedBy` or `entry.parentSessionKey` matches the reset/delete parent key or its canonical/legacy aliases, and close each child through the existing bounded ACP cancel/close helper. Preserve the existing target-session close behavior and avoid changing ACP completion cleanup, PID exposure, startup orphan reaping, remote execution policy, or unrelated ACP manager capacity behavior. Add focused regression coverage in `src/gateway/server.sessions.reset-cleanup.test.ts` and `src/gateway/server.sessions.delete-lifecycle.test.ts` for a parent session plus ACP child entries, asserting child cancel/close calls occur and unrelated ACP sessions are not closed. Add a concise `CHANGELOG.md` fix entry if this user-visible cleanup change requires one under repo policy.
