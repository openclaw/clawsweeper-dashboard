## Plan

Fix https://github.com/openclaw/openclaw/issues/80582 by preventing markdown placeholder-only snippets from entering or being reused by memory-core short-term promotion. In extensions/memory-core/src/short-term-promotion.ts, add a shared predicate that treats empty snippets, bare list markers, marker-only markdown fragments, fence-only/quote-only/table-marker-only fragments, and heading-plus-empty-bullet skeletons as unpromotable, while preserving short bullets that contain real letters or numbers. Use it in store normalization/load filtering, recordShortTermRecalls, recordGroundedShortTermCandidates, rankShortTermPromotionCandidates, applyShortTermPromotions, rehydration, and relocation matching. For relocation, keep normal exact matching for meaningful snippets but do not let placeholder-only exact or substring matches relocate a candidate; add a meaningful-content guard before substring matching. Add focused tests in extensions/memory-core/src/short-term-promotion.test.ts for rejecting '-', '* ', '- -', '>', fence-only, marker-only, and heading-plus-empty-bullet skeletons; rejecting placeholder snippets before store write; rejecting already-stored placeholders during rank/apply; avoiding placeholder relocation; and preserving ordinary bullet content with real letters/numbers. Do not add a config option or downstream cleanup-only fix. Add a changelog entry if the repair is user-visible under the active Fixes section.
