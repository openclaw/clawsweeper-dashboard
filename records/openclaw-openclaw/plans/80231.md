## Summary

Keep open: current main still has the native iOS group-chat update gap. The Gateway already emits and documents `session.message`, but Aight's iOS transport does not subscribe to or decode that event, and the shared view model currently ignores assistant `session.message` payloads.

## Plan

Fix https://github.com/openclaw/openclaw/issues/80231. Aight iOS group chats do not show new agent replies until the user leaves and re-enters.
