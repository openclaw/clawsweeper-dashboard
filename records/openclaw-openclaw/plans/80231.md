## Summary

Keep open: current main still has a clear iOS/shared chat event plumbing gap for group-chat transcript updates; the gateway exposes and emits subscribed `session.message` events, but iOS does not subscribe/decode them and the shared view model drops assistant payloads.

## Plan

Fix the iOS group-chat real-time update bug reported at https://github.com/openclaw/openclaw/issues/80231. On current main, `IOSGatewayChatTransport.setActiveSessionKey` is a no-op, its event mapper drops `session.message`, and `OpenClawChatViewModel.handleSessionMessageEvent` only appends `user` messages. Implement the existing Gateway contract: subscribe the active iOS chat session with `sessions.messages.subscribe`, map `session.message` to `.sessionMessage`, and update the shared view model so active-session assistant transcript payloads become visible without leaving and re-entering the chat. Add focused Swift coverage in `ChatViewModelTests` and `IOSGatewayChatTransportTests`; include a user-facing changelog entry if repo policy treats this as a shipped iOS fix.
