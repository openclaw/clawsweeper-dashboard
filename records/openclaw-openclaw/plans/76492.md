# Coding Plan for openclaw/openclaw#76492: `openclaw agent` CLI silently falls back to embedded mode, masking gateway behavior in diagnostic probes

Source report: [records/openclaw-openclaw/items/76492.md](records/openclaw-openclaw/items/76492.md)

## Summary

Keep open: current main still automatically runs the embedded agent after Gateway transport failures or final-response timeouts, and there is still no hard-fail diagnostic mode for probes that must prove Gateway behavior.

## Plan

Implement an additive Gateway-only/no-fallback mode for `openclaw agent` for https://github.com/openclaw/openclaw/issues/76492. Add `--no-fallback` and, if consistent with local CLI env patterns, `OPENCLAW_AGENT_NO_FALLBACK=1`, causing Gateway transport failures and Gateway final-response timeouts to return an error instead of invoking embedded `agentCommand`. Preserve the current default fallback behavior. Improve fallback diagnostics so the stderr preamble clearly says embedded runs use the current shell environment, not the running Gateway environment. Add regression coverage for transport failure with no-fallback, timeout with no-fallback, default fallback unchanged, and option forwarding/help. Update `docs/cli/agent.md`, `docs/tools/agent-send.md`, and `CHANGELOG.md`. Do not change Gateway authorization error handling, do not flip the default fallback policy, and stop for separate triage if fixing stale accepted Gateway tasks requires broader lifecycle changes.
