# Coding Plan for openclaw/openclaw#57091: [Feature]: Improve Workspace Skill Loading: Add Frontmatter Parsing Error Visibility and Validation for SKILL.md

Source report: [records/openclaw-openclaw/items/57091.md](records/openclaw-openclaw/items/57091.md)

## Summary

Keep open. Current main still silently drops malformed or incomplete workspace skill candidates before the status payload, CLI, or Control UI can report them, and the related `oc-path` work does not add `openclaw skills validate` or loader diagnostics.

## Plan

Fix https://github.com/openclaw/openclaw/issues/57091 by adding visible diagnostics for workspace skill candidates that are skipped because `SKILL.md` frontmatter is missing, malformed, invalid YAML, missing description, oversized, or otherwise fails metadata loading. Implement a shared discovery diagnostic model in the skills loader/status path and surface it through `buildWorkspaceSkillStatus` / `skills.status`, `openclaw skills list`, `openclaw skills check`, and a new or clearly equivalent `openclaw skills validate` command. Update the Control UI to render invalid/skipped skill diagnostics from the status payload. Preserve existing directory-name fallback for missing `name`; report that as a warning unless maintainers explicitly choose to make it fatal. Add/update focused tests for missing opening fence, unclosed fence, invalid YAML, missing description, missing name fallback warning, oversized files, and grouped skill roots. Update `docs/cli/skills.md`, `docs/tools/skills.md`, and `docs/tools/creating-skills.md`, plus a user-facing changelog entry. Do not change skill execution semantics, auto-repair user files, or relax YAML parsing rules.
