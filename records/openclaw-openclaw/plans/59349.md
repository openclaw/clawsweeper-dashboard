## Summary

Keep open. Current main still builds and dispatches exec approval follow-ups with only `sessionKey`, while `/new` and `/reset` rotate the current `sessionId` under that key and there is no closing PR or duplicate that owns the exact fix.

## Plan

Fix https://github.com/openclaw/openclaw/issues/59349. Capture the original expected session id when creating an exec approval follow-up target, carry it through the shared target/runtime handoff/dispatch path, and before sending either an agent continuation or direct fallback verify that the session key still maps to that same session id; if it does not, log a concise stale-followup diagnostic and drop the completion. Cover both gateway and node exec approval hosts. Add focused regression tests for pending approval -> `/new` or `/reset` -> late approve/deny, including a case that proves the stale follow-up does not call the gateway agent and does not fall back to external direct delivery. Keep the change scoped to exec approval follow-up/session reset contracts; do not add a config option or change normal successful follow-up behavior. Add a maintainer-owned changelog fix entry only if this repair lane is expected to ship as a user-visible fix.
