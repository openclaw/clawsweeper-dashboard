## Summary

Current main still routes Telegram message-tool edits through text-message editing, so the reported media-caption and button-only edit failure remains valid and narrowly fixable in the Telegram plugin.

## Plan

Fix the Telegram message-tool edit action for media messages and presentation-only edits. In `extensions/telegram/src/action-runtime.ts`, allow `edit`/`editMessage` requests with presentation/buttons but no text to proceed, preserving inline button scope checks. In `extensions/telegram/src/send.ts`, preserve the existing `editMessageText` behavior for normal text edits, route button-only edits through `editMessageReplyMarkupTelegram`/`api.editMessageReplyMarkup`, and add an `editMessageCaption` path for media messages, preferably by handling Telegram's "there is no text in the message to edit" error while preserving HTML parse fallback, `MESSAGE_NOT_MODIFIED` success handling, retry/logging behavior, link-preview settings for text edits, and existing reply markup semantics. Use https://github.com/openclaw/openclaw/pull/21898 as prior art but adapt to the current `extensions/telegram` layout.
