## Summary

Keep open. Current main covers the managed default and some DevToolsActivePort guidance, but the canonical existing-session diagnostics gaps remain: profile/doctor output still drops attach failure reasons, and Chrome MCP tab selection can still treat browser-internal pages as selectable.

## Plan

Fix the canonical browser existing-session attach diagnostics issue without adding a new config option or changing the managed `openclaw` default. Preserve the existing managed default and DevToolsActivePort guidance, but add structured Chrome MCP attach/page failure reason plumbing so `/status`, `/profiles`, `browser doctor`, and CLI output can distinguish transport OK vs page tools unavailable vs missing Chrome/DevToolsActivePort/approval/npx/list_pages failures. Also filter or deprioritize Chrome MCP browser-internal pages such as `chrome://`, `devtools://`, `brave://`, `edge://`, `vivaldi://`, and `chrome-untrusted://` before tab aliases and default target selection, matching the existing CDP target filter. Do not duplicate https://github.com/openclaw/openclaw/pull/80143's cdpUrl resolver change or https://github.com/openclaw/openclaw/issues/70529's Desktop packaging/npx policy work. Add focused regression tests and a CHANGELOG.md fix entry if the final behavior is user-visible.
