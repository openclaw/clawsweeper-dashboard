## Summary

Keep open. Current main still lacks a structured accepted-sessions_spawn terminal-success signal in the incomplete-turn classifier, so a pure relay can still be reported as an empty failed turn after a successful child-session spawn.

## Plan

Fix the gateway false-negative for pure-relay agents that end a turn after a successful sessions_spawn. Preserve structured accepted child-run metadata from tool completion into embedded attempt metadata, and have resolveIncompleteTurnPayloadText suppress the incomplete-turn warning only when sessions_spawn returned status "accepted" with non-empty runId and childSessionKey. Do not add a broad config bypass, do not treat arbitrary successful tools as complete, and preserve existing warnings for failed spawns, tool errors, incomplete tool-use terminals, and generic empty turns. Add focused tests for accepted sessions_spawn suppression and failed/missing-id cases; update CHANGELOG.md because this is user-visible runtime behavior.
