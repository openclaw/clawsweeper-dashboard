## Summary

Keep open. The prior ClawSweeper finding still matches current main: the incomplete-turn classifier has no structured accepted-`sessions_spawn` success fact to distinguish a pure relay from a generic zero-payload stop, so the false warning can still fire.

## Plan

Fix https://github.com/openclaw/openclaw/issues/72541. Current main treats a pure relay agent that successfully calls `sessions_spawn` and then stops with no text as an incomplete turn because the accepted child-run result is not preserved for `resolveIncompleteTurnPayloadText`. Preserve structured success metadata from `sessions_spawn` tool completion into the embedded attempt result, and suppress the incomplete-turn warning only when the result proves `status: "accepted"` with non-empty `runId` and `childSessionKey`. Do not add a user-facing config option, do not bypass empty-turn warnings for generic tools, and do not change failed/partial/untracked spawn handling. Add focused regression coverage for the zero-payload accepted-spawn terminal-stop case, plus coverage that failed or incomplete spawn results still warn. Add a user-facing changelog fix entry if the repair changes shipped behavior.
