## Summary

Keep open: current `main` still has a source-reproducible path where the old updater parent can read config during restart verification after the updated child has stamped `meta.lastTouchedVersion`; the later stale-wrapper diagnostic work improves the warning text but does not suppress this update-specific false positive.

## Plan

Fix https://github.com/openclaw/openclaw/issues/78652. A successful package self-update can run post-core config/plugin writes in the updated child, stamp `meta.lastTouchedVersion` with the new version, then have the old updater parent read config during final restart health probing and emit a misleading newer-config warning. Implement a narrow update-scoped suppression or update-aware/silent config read for the restart/probe handoff only. Preserve the normal newer-config warning for non-update stale CLIs and do not weaken `resolveFutureConfigActionBlock` or service mutation safeguards. Add focused regression coverage for package-update restart verification with a newer `lastTouchedVersion` and no warning from the old parent. Add a user-facing changelog fix entry if behavior changes.
