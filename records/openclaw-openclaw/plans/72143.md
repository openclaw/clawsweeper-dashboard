## Summary

Keep open. Current main still exposes the internal resume-failure prefix for denied and non-success direct fallbacks, and the session follow-up path still makes only one agent resume attempt before direct fallback.

## Plan

Fix https://github.com/openclaw/openclaw/issues/72143 on current main. In `src/agents/bash-tools.exec-approval-followup.ts`, add bounded short retry/backoff around the existing `callGatewayTool("agent")` session resume/follow-up call before direct fallback, using test-injectable delays and no new config option. Remove the user-facing `Automatic session resume failed, so sending the status directly.` prefix for all direct fallback cases, including denied and non-success completions, while preserving sanitized command result/status content. Update focused regression tests for retry success, retry exhaustion with prefix-free direct fallback, denied prefix-free fallback, and non-success prefix-free fallback.
