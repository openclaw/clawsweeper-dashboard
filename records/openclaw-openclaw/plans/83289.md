## Summary

The report is valid on current main: the SSH config and tunnel helpers still hard-code `/usr/bin/ssh`, while OpenClaw already has a trusted system-binary resolver intended for this class of spawn. This is narrow enough for a focused fix PR, but it should stay open until the helpers resolve SSH through the trusted resolver and cover Windows OpenSSH explicitly.

## Plan

Fix https://github.com/openclaw/openclaw/issues/83289. Do not add a config option or weaken trust to user-writable directories. This is user-visible behavior, so include a changelog entry if the repair lane is producing a mergeable PR.
