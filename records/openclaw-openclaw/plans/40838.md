The related open PR https://github.com/openclaw/openclaw/pull/72510 addresses group bot-targeting history signals, not this current-sender bot flag.

## Plan

Fix https://github.com/openclaw/openclaw/issues/40838 by exposing Telegram sender bot classification to the agent-visible inbound context. Derive an optional boolean from `msg.from?.is_bot`, pass it through the existing `SenderFacts.isBot` contract, add an optional `MsgContext` field such as `SenderIsBot`, and render it in the sender/inbound metadata with stable omission when unknown. Add focused tests for Telegram payload construction, `buildChannelInboundEventContext`, and `buildInboundUserContextPrefix`.
