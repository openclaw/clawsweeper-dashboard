# Coding Plan for openclaw/openclaw#40838: [Feature]: requirement: Expand Telegram message field information for the agent.

Source report: [records/openclaw-openclaw/items/40838.md](records/openclaw-openclaw/items/40838.md)

## Summary

Keep open. Current main still has the Telegram sender source data and a shared `SenderFacts.isBot` field, but the Telegram turn builder, finalized message context, and inbound prompt metadata expose sender name/id/username without the sender bot classification; the broader Telegram bot-to-bot design in https://github.com/openclaw/openclaw/issues/79077 is adjacent, not a replacement for this focused metadata gap.

## Plan

Implement the focused feature requested in https://github.com/openclaw/openclaw/issues/40838. Source Telegram sender bot status from `msg.from?.is_bot`, pass it as a bounded `sender.isBot` fact in `extensions/telegram/src/bot-message-context.session.ts`, carry it through `src/channels/turn/context.ts` into a typed agent-visible context field, and render it in inbound sender metadata using a stable boolean name. Add focused tests for Telegram payload construction, shared turn context mapping, and inbound metadata rendering. Update relevant SDK/channel docs and add a user-facing changelog entry. Do not expose raw Telegram `msg.from`, do not add config options, and do not change Telegram bot-to-bot routing, allowlist, loop protection, or the broader design tracked in https://github.com/openclaw/openclaw/issues/79077.
