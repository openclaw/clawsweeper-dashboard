## Summary

Keep open: current main and the latest v2026.5.12 release still accept `Display Name` and `@Display Name` visible Matrix mention labels, but not `@[Display Name]`, so a valid `m.mentions.user_ids` event can still be dropped as `no-mention` in require-mention rooms.

## Plan

Fix Matrix inbound mention detection for formatted_body anchors whose href targets the bot MXID and whose visible label is `@[Display Name]`. Preserve the current strict policy that metadata-only `m.mentions.user_ids` is not enough, and do not change `m.mentions.room` behavior or add config. Update `extensions/matrix/src/matrix/monitor/mentions.ts` so visible label candidates and mention-prefix stripping accept `@[${displayName}]` when `displayName` is known. Add focused regression coverage in `extensions/matrix/src/matrix/monitor/mentions.test.ts` and `extensions/matrix/src/matrix/monitor/handler.test.ts` for a room message with `body: "@[Display Name] hello"`, matching `formatted_body` matrix.to anchor, `m.mentions.user_ids` for the bot, and `requireMention` room handling. Add a user-facing changelog fix entry. Stop and ask for maintainer guidance if the fix appears to require trusting metadata-only mentions or changing Matrix mention policy.
