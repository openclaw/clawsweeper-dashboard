# Coding Plan for openclaw/openclaw#83210: buildCliAgentSystemPrompt invoked with tools: [], suppresses plugin-side prompt sections like Memory Recall

Source report: [records/openclaw-openclaw/items/83210.md](records/openclaw-openclaw/items/83210.md)

## Summary

Keep open: current main still builds the CLI runner system prompt with an empty prompt-visible tool list even when bundle MCP is enabled, while the loopback MCP server exposes policy-filtered gateway tools. That makes plugin prompt sections gated on available tools, including memory-core Memory Recall, source-proven missing for claude-cli runs.

## Plan

Fix https://github.com/openclaw/openclaw/issues/83210. In src/agents/cli-runner/prepare.ts, compute prompt-visible tools for bundle-MCP CLI runs from the same gateway-scoped loopback tool surface used by McpLoopbackToolCache/resolveGatewayScopedTools, respecting sessionKey, message provider/channel, accountId, inbound event kind, senderIsOwner, surface: "loopback", and native tool exclusions. Pass those tools to buildCliAgentSystemPrompt and buildSystemPromptReport instead of [] so memory-core and other plugin prompt sections see the tools the CLI can call. Do not hardcode memory_search/memory_get and do not advertise native tools the CLI cannot invoke. Because claude-cli uses systemPromptWhen: "first", include the prompt tool surface or full generated prompt in CLI session reuse invalidation so existing sessions refresh when this guidance changes. Add focused regression coverage for bundle-MCP CLI runs including Memory Recall when memory_search/memory_get are exposed, and for disabled-tools/override paths that must not advertise loopback tools. Add a user-facing changelog fix entry during landing if repository policy requires it.
