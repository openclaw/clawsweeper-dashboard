# Coding Plan for openclaw/openclaw#83210: buildCliAgentSystemPrompt invoked with tools: [], suppresses plugin-side prompt sections like Memory Recall

Source report: [records/openclaw-openclaw/items/83210.md](records/openclaw-openclaw/items/83210.md)

## Summary

Keep open. Current main and v2026.5.12 still build the CLI-backed agent system prompt with an empty tool list while the MCP loopback exposes gateway-scoped tools, so plugin-owned prompt sections that depend on `availableTools`, including Memory Recall, can be suppressed for CLI agents.

## Plan

Fix the CLI-backed agent system prompt so bundle-MCP CLI runs receive the same scoped tool names exposed by the OpenClaw MCP loopback. In `src/agents/cli-runner/prepare.ts`, resolve the loopback gateway tool surface when bundle MCP is enabled and tools are not disabled, keep native CLI tools excluded like `src/gateway/mcp-http.runtime.ts`, preserve sender/account/session/message-channel/inbound-event and owner-only policy inputs, and pass those tools into `buildCliAgentSystemPrompt` and `buildSystemPromptReport` instead of `[]`. Add focused tests proving a CLI bundle-MCP run with `memory_search`/`memory_get` available includes the Memory Recall guidance, and that `disableTools` or no loopback does not advertise unavailable tools. Do not add a config option or move plugin-owned memory policy into core.
