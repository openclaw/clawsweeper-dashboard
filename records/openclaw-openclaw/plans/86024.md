Current main includes the earlier topic-name support from https://github.com/openclaw/openclaw/pull/65973 for normal inbound Telegram forum messages, but the reliability gap remains in native slash-command context, recovered topic dispatch context, and create/edit topic cache writes, so the latest main and release do not solve the central user problem.

## Plan

Fix the Telegram forum topic-name metadata propagation bug reported in https://github.com/openclaw/openclaw/issues/86024. Keep the change scoped to carrying already-known current-topic names into trusted metadata before prompt construction: persist/update the existing topic-name cache after successful createForumTopic/editForumTopic/rename/auto-topic-label paths, resolve TopicName for native slash-command contexts, and preserve or recover TopicName when dispatch reconstructs topic context from session state. Add focused regression coverage for normal cached inbound, native command metadata, recovered/system dispatch context, and create/edit cache writes.
