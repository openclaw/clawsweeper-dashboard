## Plan

Fix https://github.com/openclaw/openclaw/issues/79109 as a narrow Discord-owned bug fix. Use the existing plugin target resolver seam or equivalent Discord-owned path so `resolveActionTarget` preserves allowFrom-configured DM user IDs. Add regression coverage for the full `runMessageAction` dry-run send path plus Discord target resolver/normalizer tests, and add a single-line user-facing `CHANGELOG.md` fix entry.
