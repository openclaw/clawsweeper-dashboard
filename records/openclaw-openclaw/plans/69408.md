# Coding Plan for openclaw/openclaw#69408: [agents/loop-detection]: downgrade Loop warning log level to info for isolated sessions with lifetime <= cron interval

Source report: [records/openclaw-openclaw/items/69408.md](records/openclaw-openclaw/items/69408.md)

## Summary

Keep open: current main and the latest release still emit same-run generic tool-loop diagnostics at warning level, and the cron-to-tool-hook path still lacks typed bounded-isolated-run metadata for a safe info-level demotion. The related run-scoped cron session-key fix is useful context, but it only fixed cross-run accumulation, not repeated exec calls inside one bounded isolated run.

## Plan

Fix https://github.com/openclaw/openclaw/issues/69408 by reducing false-positive warning-level loop-detection noise for bounded isolated cron runs. Thread typed metadata from cron scheduling/execution into the agent before-tool loop-detection path so the hook can identify sessionTarget="isolated", the schedule interval or next-run boundary, and the observed run lifetime; do not use a hardcoded interval proxy. Update the diagnostic/tool-loop contract so non-critical bounded isolated-cron generic_repeat warnings can be emitted/logged at info while preserving critical blocks, non-isolated warnings, known-poll no-progress behavior, ping-pong behavior, and current runId scoping. Add focused regression coverage for bounded isolated-cron demotion and for preserved warning/critical behavior. Add a changelog fix entry if the log-level change is treated as user-visible.
