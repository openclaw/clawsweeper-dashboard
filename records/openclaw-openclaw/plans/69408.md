## Summary

Keep open. Current main and the latest stable tag still emit same-run generic loop detections at warning level, while the cron-to-agent hook path does not carry the isolated-run schedule/lifetime metadata needed for a conservative info-level demotion.

## Plan

Fix the isolated-cron loop-detection log severity noise reported in https://github.com/openclaw/openclaw/issues/69408. Preserve current warning/critical behavior for interactive, custom-session, non-isolated, unbounded, and critical loop detections. Add a typed metadata path from cron execution into the before-tool-call/tool-loop diagnostic path that can prove an isolated cron run is bounded by its actual schedule/run timing, then demote only non-critical bounded isolated-cron loop warnings to info/diagnostic info. Do not use a hardcoded cron interval proxy.
