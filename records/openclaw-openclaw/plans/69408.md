# Coding Plan for openclaw/openclaw#69408: [agents/loop-detection]: downgrade Loop warning log level to info for isolated sessions with lifetime <= cron interval

Source report: [records/openclaw-openclaw/items/69408.md](records/openclaw-openclaw/items/69408.md)

## Summary

Keep open. Current main still treats repeated generic tool calls inside the same run as warning-level loop diagnostics, and the isolated cron path still does not pass session target, schedule interval, or bounded-lifetime metadata into that decision.

## Plan

Fix https://github.com/openclaw/openclaw/issues/69408 by adding conservative context-aware severity for tool-loop warnings from isolated cron runs. Thread typed cron/session metadata from `src/cron/isolated-agent/run-executor.ts` through embedded run params and the before-tool-call hook so the loop-warning decision can know: trigger is cron, session target is isolated, the run is bounded, and the run lifetime is within the schedule interval. Demote only non-critical bounded isolated-cron generic repeat diagnostics to info/audit level; preserve all critical blocks and all main/interactive/session-target warning behavior. Do not use a hardcoded five-minute interval proxy, do not add a new config knob unless maintainers explicitly request one, and keep diagnostic event/log contracts, stability aggregation, and tests aligned. Add focused regression coverage for same-run isolated cron demotion, non-isolated warning preservation, critical block preservation, and schedule-bound edge cases. Add a changelog entry if the final change is treated as a user-visible fix.
