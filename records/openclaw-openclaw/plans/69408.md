# Coding Plan for openclaw/openclaw#69408: [agents/loop-detection]: downgrade Loop warning log level to info for isolated sessions with lifetime <= cron interval

Source report: [records/openclaw-openclaw/items/69408.md](records/openclaw-openclaw/items/69408.md)

## Summary

Keep open. Current main and the latest stable release still emit same-run loop-detection warnings at warning level, and the cron/agent hook path still lacks isolated-run schedule and lifetime metadata needed for a conservative info-level demotion.

## Plan

Fix https://github.com/openclaw/openclaw/issues/69408. Current main already scopes isolated cron runs by run key, so do not reintroduce idle-gap history resets or broad diagnostic-state changes. Instead, pass typed isolated-cron boundedness metadata from the cron run context into the tool-loop warning path, then emit info-level diagnostics only for non-critical loop warnings when the run is sessionTarget="isolated" and its measured/allowed lifetime is bounded by the real schedule interval. Preserve warning/error behavior for critical loops, non-isolated sessions, unknown-tool and no-progress blocks, and cases where boundedness cannot be proven. Do not add a public config option or a hardcoded five-minute proxy. Add focused regression coverage for the isolated bounded info case and for unchanged non-isolated/critical warning behavior. Add a single-line Fixes changelog entry if the repair changes user-visible logging behavior.
