# Coding Plan for openclaw/openclaw#69408: [agents/loop-detection]: downgrade Loop warning log level to info for isolated sessions with lifetime <= cron interval

Source report: [records/openclaw-openclaw/items/69408.md](records/openclaw-openclaw/items/69408.md)

## Summary

Keep open. Current main still emits repeated tool-call detections as warning-level logs/diagnostics, and isolated cron execution does not pass target, schedule interval, or run-lifetime context into the loop-warning decision.

## Plan

Fix the isolated-cron loop-detection log-noise issue. Carry typed cron execution context from the isolated cron run path into the before-tool-call loop detection/logging path: sessionTarget, job id, run start/elapsed time, and a conservatively derived schedule interval where available. Add an info-level non-critical loop diagnostic/log path and demote only warning-level loop detections for sessionTarget="isolated" cron runs proven to still be within their schedule bound. Preserve critical blocking, normal session warnings, and unknown/unbounded cron warnings. Do not add a config option or hardcode a 5-minute proxy unless there is no schedule-bound helper and the fallback is explicit with tests. Update focused unit/e2e diagnostics coverage, cron run-context coverage, docs if the public logging contract changes, and changelog if the behavior is user-visible.
