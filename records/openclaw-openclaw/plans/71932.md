Keep open. Current main still has a source-visible Google provider path where a configured `google/gemini-*` model id can survive into native Gemini URL construction as `models/google/...`, and the overlapping prefix-fix PRs are closed unmerged rather than shipped.

## Plan

Fix https://github.com/openclaw/openclaw/issues/71932 on current main. The observable bug is that Google provider configured model entries such as `google/gemini-3.1-flash-lite-preview` or `google/gemini-3-flash-preview` can remain provider-prefixed through sub-agent/session-reset model resolution and reach native Google Generative AI transport as `models/google/...`, causing 404s. Implement a narrow Google/provider-owned normalization fix that strips only redundant active Google provider prefixes before configured-model resolution and native Google transport URL construction. Preserve valid nested/proxy model ids such as OpenRouter, NVIDIA, Hugging Face, MLX, and provider-local namespaced ids; do not add a generic same-provider strip. Add regression coverage for configured Google provider models, sub-agent spawn model resolution, and session reset model recomputation where practical. Related closed unmerged PRs: https://github.com/openclaw/openclaw/pull/41398 and https://github.com/openclaw/openclaw/pull/41517.
