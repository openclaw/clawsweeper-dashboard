# Coding Plan for openclaw/openclaw#55249: Feature: Session labels / nicknames for easier identification

Source report: [records/openclaw-openclaw/items/55249.md](records/openclaw-openclaw/items/55249.md)

## Summary

Keep open. Current main and v2026.5.12 still have backend session-label primitives, but the requested user-facing workflow is not implemented: there is no `openclaw sessions label` command, no `/label` chat/TUI command, and local `openclaw sessions` text/JSON output still drops stored labels. Related items either point back to this issue as canonical or cover narrower auto-label/status display work.

## Plan

Implement the canonical session-label workflow for https://github.com/openclaw/openclaw/issues/55249. Reuse the existing Gateway `sessions.patch({ key, label })` / `parseSessionLabel` contract; do not add a separate mutable `displayName` or `subject` mutation path. Add a local `openclaw sessions label <session-key> <label>` command plus a clear/remove form, expose stored `label` in `openclaw sessions --json`, render a sanitized friendly label/name in text output while keeping the raw key visible for copy/paste, and add a `/label` chat/TUI command for current-session show/set/clear. Coordinate, but do not fully solve, WebUI auto-generated dashboard labels tracked in https://github.com/openclaw/openclaw/issues/76848. Update `docs/cli/sessions.md` and add a user-facing `CHANGELOG.md` entry if this becomes a feature PR. Keep terminal/TUI output single-line sanitized and preserve resolvable keys.
