## Summary

Keep open. Current main has the session-store and Gateway label primitives, plus label preservation across reset, but the requested user-facing workflow is still missing: no `openclaw sessions label` command, no `/label` chat/TUI command, and local `openclaw sessions` text/JSON output still projects opaque keys instead of stored labels.

## Plan

Implement manual session labels for this issue. Add a user-facing `openclaw sessions label <session-key> <label>` flow with clear support, reuse Gateway `sessions.patch` and existing label validation rather than mutating `subject`/`displayName` directly, and update local `openclaw sessions` text and JSON output to carry sanitized `label`/`displayName` while preserving a resolvable key. Add a `/label` chat/TUI workflow for the current session: `/label <name>` sets, `/label` shows the current label, and `/label --clear` clears. Do not implement auto-generated labels or channel-specific defaults in this PR. Keep the status-display work in https://github.com/openclaw/openclaw/pull/51257 separate unless current main already absorbed it. Update docs and add a user-facing changelog entry.
