## Summary

Keep open: current main still maps default Gateway-backed ACP bridge sessions to metadata-free `acp:<uuid>` keys, while ACP dispatch still treats any ACP-shaped key without persisted ACP metadata as a stale ACPX runtime session and returns `ACP_SESSION_INIT_FAILED`.

## Plan

Fix https://github.com/openclaw/openclaw/issues/38907 as a narrow ACP bridge/runtime-boundary bug.
