# Coding Plan for openclaw/openclaw#64236: Feature request: support per-model reasoningDefault configuration

Source report: [records/openclaw-openclaw/items/64236.md](records/openclaw-openclaw/items/64236.md)

## Summary

Keep open. Current main and the latest stable release still lack a `reasoningDefault` field on configured model entries, and the merged Telegram/Feishu preview fix only covers per-agent/global channel fallback behavior, not per-model config.

## Plan

Implement per-model reasoning visibility defaults for https://github.com/openclaw/openclaw/issues/64236. Add `reasoningDefault?: "off" | "on" | "stream"` to `AgentModelEntryConfig` and both default/per-agent model-entry schemas, update schema help/labels/generated config docs as needed, and resolve the selected model entry's default after inline directive, session state, and per-agent `agents.list[].reasoningDefault`, but before `agents.defaults.reasoningDefault` and catalog `reasoning: true` fallback. Preserve the existing authorization gate in `get-reply-directives.ts`, keep catalog fallback behavior for compatibility, and do not change the broader implicit API-thinking/default-cost policy tracked in https://github.com/openclaw/openclaw/issues/73182. Add focused regression coverage for schema acceptance/rejection, precedence, `stream`, explicit session/directive overrides, and catalog fallback. Update user docs and add a changelog entry if this is user-facing.
