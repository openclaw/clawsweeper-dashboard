# Coding Plan for openclaw/openclaw#64236: Feature request: support per-model reasoningDefault configuration

Source report: [records/openclaw-openclaw/items/64236.md](records/openclaw-openclaw/items/64236.md)

## Summary

Keep open. Current main still supports per-agent and global `reasoningDefault`, plus catalog-derived `on`/`off` fallback, but it does not support the requested per-model `reasoningDefault` under `agents.defaults.models[...]`.

## Plan

Implement https://github.com/openclaw/openclaw/issues/64236 as a narrow additive config feature. Add a documented top-level `reasoningDefault?: "off" | "on" | "stream"` to `AgentModelEntryConfig` and the strict Zod/generated schema/help/labels as needed. Resolve the selected model entry's default after inline directive, session override, and per-agent default, but before `agents.defaults.reasoningDefault` and catalog `reasoning: true` fallback. Preserve existing authorization gating for configured reasoning defaults and keep catalog fallback compatible. Update docs at `docs/gateway/config-agents.md` and `docs/tools/thinking.md`, add focused regression tests for selected-model defaults including `stream`, explicit `off`, slash-containing model ids, and precedence over global/catalog fallback. Do not change unrelated Telegram/Feishu preview behavior except where tests prove the new core default is consumed through existing contracts; do not remove existing global or per-agent defaults.
