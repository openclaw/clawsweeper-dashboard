# Coding Plan for openclaw/openclaw#64236: Feature request: support per-model reasoningDefault configuration

Source report: [records/openclaw-openclaw/items/64236.md](records/openclaw-openclaw/items/64236.md)

## Summary

Keep open. Current main and the latest release still support global and per-agent `reasoningDefault`, but not the requested per-model `reasoningDefault` on configured model entries; the merged Telegram/Feishu work is useful adjacent context but only covers channel preview fallback behavior.

## Plan

Implement per-model reasoning defaults for configured agent model entries. Add optional `reasoningDefault: "off" | "on" | "stream"` to `AgentModelEntryConfig` and the Zod schemas for model entries used by `agents.defaults.models` and `agents.list[].models`; update schema help/labels/docs so the public config contract is aligned. Resolve the selected model entry's value after inline directive, session override, and per-agent `agents.list[].reasoningDefault`, but before `agents.defaults.reasoningDefault` and catalog-derived `reasoning: true` fallback. Preserve the existing authorization gate for configured reasoning defaults, keep catalog fallback as `on`/`off`, add regression tests for `stream` and `off`, and include a maintainer-owned changelog entry for the user-visible config feature. Do not change provider reasoning effort semantics or unrelated thinking defaults.
