## Summary

Keep open. Current main and the latest release still support per-agent and global `reasoningDefault`, plus catalog-derived model fallback, but they do not support the requested `reasoningDefault` field inside `agents.defaults.models[...]`.

## Plan

Implement https://github.com/openclaw/openclaw/issues/64236 as an additive config feature. Add `reasoningDefault?: "off" | "on" | "stream"` to `AgentModelEntryConfig` and the `agents.defaults.models.*` Zod/schema/help/label surfaces; resolve the selected model entry's value after inline directive, session override, and per-agent `agents.list[].reasoningDefault`, but before global `agents.defaults.reasoningDefault` and catalog `reasoning: true` fallback. Preserve the existing owner/authorized-sender/operator-admin gate for configured reasoning defaults, keep catalog fallback compatible, cover `stream`, and update docs/changelog for the user-facing config surface. Do not change provider-specific thinking params or replace `thinkingDefault`; that closed nearby issue is a different effort-level control.
