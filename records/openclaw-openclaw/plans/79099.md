Keep open. The no-auth backend subcase has been fixed, but current main still has source-visible paths where a WebSocket connection that fails before `hello-ok` remains classified as unreachable/not-running instead of reachable-but-errored.

## Plan

Fix the gateway probe/doctor diagnostic contradiction reported in https://github.com/openclaw/openclaw/issues/79099. Current main still treats a WebSocket connection that is accepted but fails before `hello-ok` as unreachable because `src/gateway/probe.ts` only records connect latency at `onHelloOk`, and gateway-status/doctor then collapse protocol/auth/pre-hello closes into unreachable or not-running output. Implement a narrow diagnostic-state fix that distinguishes: no listener/connect failure, listener accepted but handshake/auth/protocol/read diagnostics failed, and fully healthy/reachable. Preserve existing auth and pairing semantics; do not make failed auth/protocol handshakes count as operator access. Keep the existing auth-none backend self-pairing fix from https://github.com/openclaw/openclaw/pull/75781 intact. Add or update focused tests for probe classification, gateway-status text/JSON warnings, doctor gateway health/daemon contradiction, and protocol/auth pre-hello close behavior. Add a changelog fix entry because this changes user-visible CLI diagnostics.
