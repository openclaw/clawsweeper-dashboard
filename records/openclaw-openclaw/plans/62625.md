## Summary

Keep open. Current main still skips session transcript sync after restart or missed events when no in-process session dirty state exists, while the docs define session transcripts as searchable memory content; the related observability work does not close the indexing gap.

## Plan

Fix https://github.com/openclaw/openclaw/issues/62625. In memory-core, add a bounded reconciliation path for the sessions source that lists the agent's session transcript files, compares them with durable DB file state, marks only missing or changed session files dirty, and allows/schedules a session sync without requiring --force. Preserve targeted, forced, and full-reindex semantics; do not turn every session-start/watch hot path into an unconditional full transcript scan. Add focused regression coverage for a clean restart or missed-event state where sessionsDirty=false and sessionsDirtyFiles is empty but a session file is missing or changed on disk. Coordinate with https://github.com/openclaw/openclaw/pull/42014 for status wording only; that PR is not the indexing fix. Add a changelog entry if the fix changes user-visible memory indexing reliability.
