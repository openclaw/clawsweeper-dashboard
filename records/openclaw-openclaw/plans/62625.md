## Summary

Keep open. Current main still lacks a startup/session catch-up reconciliation path, and clean `session-start` or `cli` syncs still skip the `sessions` source when no current-process dirty files are recorded; the open observability PR at https://github.com/openclaw/openclaw/pull/42014 does not fix the indexing gap.

## Plan

Fix https://github.com/openclaw/openclaw/issues/62625 in memory-core. Add a bounded session transcript reconciliation path during non-status/non-CLI manager startup or equivalent bootstrap: list `sessions` files with `listSessionFilesForAgent`, compare them against durable `files` rows for source `sessions` including path plus mtime/size/hash as appropriate, mark missing or changed files in `sessionsDirtyFiles`, set `sessionsDirty`, and schedule or allow a normal sync to index them. Preserve the existing hot-path `session-start`/`watch` skip semantics; do not turn every clean session start or watch event into an unconditional full transcript scan. Add focused regression tests for clean restart or missed event state where existing indexed session rows fall behind and non-forced `memory index`/session bootstrap catches up. Coordinate but do not depend on https://github.com/openclaw/openclaw/pull/42014 for status wording; add a changelog fix entry if the repair changes user-visible memory behavior.
