## Plan

Fix https://github.com/openclaw/openclaw/issues/68066. Implement a conservative OpenRouter-owned reconciliation for completed streaming chat completions only: when the route is verified `https://openrouter.ai/api/v1`, the stream completes normally, a response/generation id is available, and a real OpenRouter bearer token/header is available, call `GET https://openrouter.ai/api/v1/generation?id=<id>` with a bounded timeout, validate `data.total_cost` as finite and nonnegative, and update the assistant message usage cost total before final transcript persistence/done handling.
