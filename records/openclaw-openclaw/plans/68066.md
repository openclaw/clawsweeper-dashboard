## Summary

Keep open. Current main and the latest release still persist OpenRouter streaming usage through the shared OpenAI-compatible parser and local cost calculation, with no OpenRouter `/generation` reconciliation for authoritative billed `total_cost`; the adjacent tiered-pricing and usage-aggregation work does not fix this transcript write path.

## Plan

Fix the OpenRouter billed-cost undercount reported in https://github.com/openclaw/openclaw/issues/68066. Implement a conservative OpenRouter-owned reconciliation path for completed streaming chat completions: when the route is the real OpenRouter base URL/provider, a response id is available, the request was not aborted, and a real OpenRouter credential is available, call `GET https://openrouter.ai/api/v1/generation?id=<responseId>` after the stream completes, validate a finite numeric `data.total_cost`, and update the assistant message’s authoritative `usage.cost.total` before the final done message is emitted/persisted. Preserve the existing local estimate on missing ids, custom base URLs, missing auth, non-2xx responses, invalid payloads, timeouts, aborts, or lookup failures. Keep the change scoped to OpenRouter/provider stream behavior and the existing usage-cost transcript path; do not change cross-agent aggregation, agent.wait metadata, pricing-cache merge policy, or unrelated usage dashboard behavior. Add focused unit coverage for successful reconciliation, fallback on lookup failure, and no lookup for custom OpenAI-compatible endpoints. Add a user-facing changelog fix entry if this repair lane owns release notes.
