# Coding Plan for openclaw/openclaw#68066: Gateway writes streamed usage.cost.total for OpenRouter calls — up to 4× under actual billed amount on tier-priced models

Source report: [records/openclaw-openclaw/items/68066.md](records/openclaw-openclaw/items/68066.md)

## Summary

Keep this open. Current main still records OpenRouter streamed usage through the shared OpenAI-compatible parser and local pricing path, while OpenRouter exposes an authenticated generation metadata endpoint with authoritative `total_cost`; the adjacent tiered-pricing and dashboard work does not update the persisted assistant message cost from that endpoint.

## Plan

Fix https://github.com/openclaw/openclaw/issues/68066 by adding OpenRouter billed-cost reconciliation for streamed chat completions. Keep the fix provider-owned when possible: prefer `extensions/openrouter/stream.ts` or a small generic stream-wrapper seam over hardcoding OpenRouter policy in core. After a verified OpenRouter streaming completion finishes and before the final assistant message is persisted/emitted, use the retained response id plus the request credential to call OpenRouter `GET /generation?id=<responseId>`, validate `data.total_cost` as a finite nonnegative USD number, and update `message.usage.cost.total` to that authoritative billed value. Preserve the existing locally calculated estimate on aborts, missing response ids, missing credentials, custom non-OpenRouter endpoints, lookup errors, nonnumeric responses, and non-OpenRouter providers. Do not implement the broader OpenRouter runtime metadata requests from https://github.com/openclaw/openclaw/issues/7006 or https://github.com/openclaw/openclaw/issues/9016, do not change `agent.wait` metadata from https://github.com/openclaw/openclaw/pull/49502, and do not change the multi-agent dashboard aggregation scope from https://github.com/openclaw/openclaw/pull/70205. Add focused regression coverage for successful reconciliation, lookup failure fallback, non-OpenRouter/custom route no-op, missing id/key no-op, and downstream transcript cost reading if needed. Add a changelog entry if the repair is user-visible.
