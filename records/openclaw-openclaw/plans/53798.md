# Coding Plan for openclaw/openclaw#53798: [Bug]: Feishu voice message sent via message tool does not display audio duration

Source report: [records/openclaw-openclaw/items/53798.md](records/openclaw-openclaw/items/53798.md)

## Summary

Keep open. Current main and the latest release still route Feishu Ogg/Opus voice media through `uploadFileFeishu` without computing or passing `duration`, while the upload helper and upstream SDK contract support the field and the overlapping duration PRs are closed unmerged.

## Plan

Fix the Feishu voice duration bug reported in https://github.com/openclaw/openclaw/issues/53798. In `extensions/feishu/src/media.ts`, compute a finite millisecond duration for the final bytes that will be uploaded after `prepareFeishuVoiceMedia` has run, and pass it to `uploadFileFeishu` only for Feishu audio/video upload types that accept duration. Preserve existing routing, `audioAsVoice` transcode behavior, file fallback behavior when duration probing fails, media size limits, and plugin boundary rules. Prefer existing SDK media-runtime helpers such as `runFfprobe`/bounded parsers or a small plugin-local helper over shell-string execution; do not broaden audio format support, add a config option, or move Feishu-specific policy into core. Add focused regression tests in `extensions/feishu/src/media.test.ts` for native Ogg/Opus, transcoded voice-intent audio, video if supported by the helper, and duration-probe failure fallback. Add a single user-visible `CHANGELOG.md` fix entry for the active release section.
