## Summary

Keep open: the PR is focused and useful, and its text proof is sufficient for the status/CLI path, but the new ClawHub trust check can be bypassed by a local skill that ships a forgeable origin marker.

## Plan

Repair https://github.com/openclaw/openclaw/pull/80016 for the linked security issue https://github.com/openclaw/openclaw/issues/57535. Keep the change scoped to skill status trust reporting, CLI/UI rendering, config/docs/tests, and do not change skill loading, precedence, eligibility, or execution semantics. Fix the warning bypass: do not treat a self-contained `.clawhub/origin.json` inside an arbitrary local skill as trusted by itself. Require non-self-asserted ClawHub install provenance and/or strict marker validation that at least rejects mismatched slug/path and non-ClawHub registry values; if the existing provenance contract is insufficient, leave the skill untrusted and document that explicit `skills.load.trustedDirs` suppresses warnings. Add regression tests proving forged or mismatched origin metadata remains `untrustedLocalSource: true`, while real ClawHub-installed or explicitly trusted directories remain trusted. Preserve the changelog/docs intent but update wording if the trust model changes.
