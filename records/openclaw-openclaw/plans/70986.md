## Summary

Keep open: current main and the latest release still expose the two central Memory Wiki lint bugs. The `.md` relative-target false positive is fixed on main, but wikilink extraction still scans raw Markdown and broken-link lint still does not accept frontmatter titles or aliases; the related resolver PRs are open/partial rather than a terminal fix.

## Plan

Fix only the Markdown-aware wikilink extraction half of https://github.com/openclaw/openclaw/issues/70986. In `extensions/memory-wiki/src/markdown.ts`, make `extractWikiLinks` ignore `[[...]]` and native Markdown link syntax inside fenced code blocks using backticks or tildes, inline code spans, YAML frontmatter, and HTML comments before applying the existing link extraction, while preserving real body wikilinks and native Markdown links. Add focused regression coverage in `extensions/memory-wiki/src/markdown.test.ts` and/or `extensions/memory-wiki/src/lint.test.ts` for fenced shell `[[ "$x" ]]`, inline `[[reply_to_current]]`, frontmatter/comment examples, and a real `[[real-target]]` still being captured. Do not implement title/alias/slug resolver normalization in this PR; coordinate that half with https://github.com/openclaw/openclaw/pull/80915, https://github.com/openclaw/openclaw/pull/73722, and https://github.com/openclaw/openclaw/issues/73574. Add a single-line `CHANGELOG.md` fix entry unless this is intentionally batched with the resolver fix during landing.
