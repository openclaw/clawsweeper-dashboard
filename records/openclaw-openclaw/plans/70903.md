## Summary

Keep open. Current main still leaves the reported recovery gap for billing-disabled same-provider fallback chains, and the configure/plugin auth path can still upsert fresh credentials without clearing stale usage state.

## Plan

Fix https://github.com/openclaw/openclaw/issues/70903 on current main. The bug is that persisted auth-state.json cooldown/disabled state can keep recovered users blocked for hours: billing-disabled same-provider/same-auth fallback chains are not probed far from expiry, and configure/plugin auth refresh paths can upsert fresh credentials without clearing stale usage stats. Keep persistent cooldown protection; do not make disabledUntil in-memory only, do not add a new config option, and do not broaden WHAM/overloaded/failure-policy behavior from https://github.com/openclaw/openclaw/pull/74909. Add targeted recovery so a billing-disabled chain with no real alternate auth path can make a throttled recovery probe, and clear matching provider/profile usage state after successful configure/plugin auth refresh or returned profile upsert. Add focused regression tests for model-fallback billing same-auth fallback behavior and provider-auth-choice/configure stale-state cleanup.
