# Coding Plan for openclaw/openclaw#65329: bug(msteams): DM inline images and file attachments silently dropped

Source report: [records/openclaw-openclaw/items/65329.md](records/openclaw-openclaw/items/65329.md)

## Summary

Keep open. Current main still leaves MS Teams personal-DM non-HTML media without a Bot Framework fallback after direct download returns no media, so pasted images or direct file attachments can still be represented only as unresolved media placeholders.

## Plan

Fix https://github.com/openclaw/openclaw/issues/65329 in the bundled MS Teams plugin. In `resolveMSTeamsInboundMedia` and its caller, preserve the original Bot Framework personal-DM conversation id separately from the Graph chat id, and after direct download returns no media, attempt the Bot Framework attachment fallback for real non-HTML media/file candidates in personal DMs. Do not make mention-only HTML messages trigger fallback calls; preserve the guard from https://github.com/openclaw/openclaw/issues/58617. Coordinate with the narrower file-download URL rewrite in https://github.com/openclaw/openclaw/issues/67177 but do not fold broad SDK migration work from https://github.com/openclaw/openclaw/pull/76262 into this fix. Add focused regression tests for the non-HTML personal-DM media path and keep docs/changelog aligned if the user-visible behavior changes.
