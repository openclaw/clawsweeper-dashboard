## Plan

Fix https://github.com/openclaw/openclaw/issues/56068. Current main validates `ui.seamColor` and returns it from `talk.config`, but the web Control UI never receives or applies it. Add an optional `seamColor` field to `ControlUiBootstrapConfig`, include it from `config.ui?.seamColor` in the authenticated Control UI bootstrap response, consume it in the frontend bootstrap path, and apply normalized accent CSS variables only when a valid configured color is present.
