# Coding Plan for openclaw/openclaw#69054: [Feature] Add \"cron lint\" subcommand and require delivery.accountId on announce-mode crons in multi-account channel configs

Source report: [records/openclaw-openclaw/items/69054.md](records/openclaw-openclaw/items/69054.md)

## Summary

Keep open. Current main still lacks the requested `openclaw cron lint` command and still allows multi-account announce cron jobs to be created or edited without an explicit `delivery.accountId`; the related logging and explicit-invalid-account threads are complementary rather than replacements.

## Plan

Implement https://github.com/openclaw/openclaw/issues/69054 as a focused staged cron lint/validation PR. Add a shared cron lint helper that reports `CRON_ANNOUNCE_ACCOUNT_ID_REQUIRED` when `delivery.mode === "announce"`, the resolved/static channel has more than one configured account, and `delivery.accountId` is missing. Expose `openclaw cron lint` with JSON/human output and `--warn-only`, add gateway startup warning-only reporting after cron jobs load, and reject new/edited announce jobs missing accountId for multi-account channels in gateway cron.add/cron.update validation. Use lightweight read-only channel configured-account metadata such as `listAccountIds`; do not load full channel plugin runtimes from the gateway hot path. Do not implement strict startup failure, waivers, auto-inference from agentId, or the companion delivery telemetry request. Be aware of https://github.com/openclaw/openclaw/issues/69513 and https://github.com/openclaw/openclaw/pull/69562 for adjacent explicit invalid-account validation, but do not duplicate their branch if it is still open; share or supersede the helper cleanly. Add focused tests and docs; add a changelog entry because this is user-visible CLI/validation behavior.
