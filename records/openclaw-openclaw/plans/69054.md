## Summary

Keep open. Current main still lacks the requested `openclaw cron lint` command and account-aware validation for multi-account announce delivery; the linked logging issue remains complementary rather than a replacement.

## Plan

Implement the feature requested in https://github.com/openclaw/openclaw/issues/69054 as a narrow OpenClaw cron change. Add a shared cron lint helper, likely `src/cron/lint.ts`, that reports `CRON_ANNOUNCE_ACCOUNT_ID_REQUIRED` when an announce-mode cron job targets a channel whose plugin config lists more than one account and the job lacks `delivery.accountId`; single-account channels must remain a no-op. Use the channel plugin account-list contract, not raw Telegram/Discord config assumptions. Expose `openclaw cron lint [--warn-only]` with the requested exit behavior and JSON-ish finding fields, add gateway startup warning-only reporting, and reject new/edited announce cron jobs missing accountId on multi-account channels through the existing cron add/update validation path. Do not infer accountId from agentId, do not auto-migrate jobs, do not implement fail-closed startup or a strictness env var without maintainer direction, and keep https://github.com/openclaw/openclaw/issues/69055 out of scope except for comments/tests that clarify it is separate telemetry work. Update focused docs and add a user-facing changelog entry because this adds CLI/runtime validation behavior.
