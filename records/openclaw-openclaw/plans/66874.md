# Coding Plan for openclaw/openclaw#66874: [Feature]: Official Docker image should ship with `claude` CLI pre-installed (or document persistent install path)

Source report: [records/openclaw-openclaw/items/66874.md](records/openclaw-openclaw/items/66874.md)

## Summary

Keep this open. Current main and the latest shipped release still treat the Anthropic Claude CLI backend as an external `claude` command, while the official Docker image does not install Claude Code and the Docker/provider/backend docs still do not give Docker users a persistent binary plus `~/.claude` credential recipe.

## Plan

Create a docs-only fix for https://github.com/openclaw/openclaw/issues/66874. Do not install `@anthropic-ai/claude-code` in the official Docker image and do not change Docker release workflows. Add Docker-specific Claude CLI guidance that explains where to install `claude` in persistent Docker state, how to persist or mount `~/.claude` credentials safely, how to set `agents.defaults.cliBackends.claude-cli.command` to an explicit path, how to verify onboarding with `openclaw models auth login --provider anthropic --method cli --set-default`, and how to use the Anthropic API-key path as a fallback. Prefer `docs/install/docker.md` as the main home and add concise cross-links from Anthropic/provider or CLI-backend docs if needed. No changelog is expected for this docs-only guidance unless the maintainer landing policy requires one.
