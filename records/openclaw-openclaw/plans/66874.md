## Summary

Keep this open: current main still treats Claude Code as an external `claude` command, the official Docker image path does not install it, and Docker docs still lack a Claude-specific persistent install plus `~/.claude` credential recipe. A docs-only Option B repair remains the narrow safe path; preinstalling Claude Code in the image is a separate release artifact and supply-chain decision.

## Plan

Add a docs-only fix for Docker users who want to use the existing Anthropic Claude CLI backend. Do not edit Dockerfile, release workflows, package manifests, lockfiles, or install @anthropic-ai/claude-code into the official image. Update docs/install/docker.md and, if needed for cross-linking, docs/providers/anthropic.md, docs/gateway/cli-backends.md, or docs/gateway/authentication.md to explain a persistent Docker setup: where to put a persisted npm prefix or binary path for Claude Code, how to persist Claude Code's ~/.claude credentials separately from OpenClaw auth profiles, how to set agents.defaults.cliBackends.claude-cli.command to the persisted binary, how to verify with claude auth status --text and openclaw models auth login --provider anthropic --method cli --set-default, and when to use an Anthropic API key fallback. Keep examples generic, redact-safe, and Mintlify-compatible; do not promise that the official image includes claude. No CHANGELOG.md update.
