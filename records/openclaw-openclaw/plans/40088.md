## Summary

Keep open. Current main has partial watcher hardening, but the reported fallback gap remains: the default interval is still disabled and an interval tick still does not make memory-file sync pass the dirty gate when the watcher has stopped firing.

## Plan

Fix https://github.com/openclaw/openclaw/issues/40088. Current main already has direct memory-directory watching and watcher error logging, but default `memorySearch.sync.intervalMinutes` is still `0` and interval sync still calls `sync({ reason: "interval" })` without setting `dirty`, so a dead watcher leaves memory files stale. Implement a bounded memory-core fallback: update `src/agents/memory-search.ts` so default users get a nonzero periodic safety net while preserving explicit `intervalMinutes: 0` opt-out; update `extensions/memory-core/src/memory/manager-sync-ops.ts` so interval sync performs a lightweight file scan by marking memory dirty or otherwise bypassing the dirty gate without forcing a full reindex; keep existing watcher error handling. Add focused regression tests in `src/agents/memory-search.test.ts` and `extensions/memory-core/src/memory/manager.watcher-config.test.ts`. Update config help/labels/docs if the public default or help surface changes, and add a changelog entry because this changes user-visible freshness behavior. Do not pull in broader recursive-memory, QMD watcher, or dirty-first-search changes from https://github.com/openclaw/openclaw/pull/72748; only coordinate if touched paths overlap.
