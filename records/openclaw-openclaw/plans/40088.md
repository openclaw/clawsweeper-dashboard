## Summary

Keep open. Current main and v2026.5.19 still leave the reported dead-watcher fallback unresolved: default interval sync is disabled, and a configured interval still calls sync without marking memory files dirty, so the memory-file sync gate can skip after watcher events stop. The earlier watcher directory fix and error listener are useful partial mitigations, but https://github.com/openclaw/openclaw/pull/42042 and https://github.com/openclaw/openclaw/pull/65755 closed unmerged, and https://github.com/openclaw/openclaw/pull/72748 covers adjacent first-search freshness rather than the dead-watcher dirty=false fallback.

## Plan

Fix https://github.com/openclaw/openclaw/issues/40088 in the memory-core/agent memory search sync path. Current main already watches the memory directory directly and logs watcher errors; do not reintroduce glob watch targets, awaitWriteFinish polling, QMD watcher changes, provider changes, or unrelated async first-search work from https://github.com/openclaw/openclaw/pull/72748. Implement a fallback for dead chokidar events by setting a conservative non-zero default for agents.defaults.memorySearch.sync.intervalMinutes while respecting explicit intervalMinutes: 0, and make interval-triggered sync mark memory files dirty or otherwise run the lightweight file listing/hash comparison path without force/full reindex. Preserve session-only retry semantics and avoid leaving dirty permanently true for managers without memory-file sources. Add regression coverage for default/override/explicit-zero interval config, interval ticks after dirty is false causing memory file sync to run, watcher error listener behavior, and any session-only/no-memory guard.
