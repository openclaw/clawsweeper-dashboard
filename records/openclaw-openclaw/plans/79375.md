## Summary

Keep open: current main still has a real Linux systemd duplicate-service gap. The code blocks duplicate user-service installation only when the user service is missing, while the extra-service scanner filters canonical `openclaw-gateway.service` units, so a stale canonical user unit plus a system unit can still evade diagnostics.

## Plan

Fix https://github.com/openclaw/openclaw/issues/79375 on current main.
