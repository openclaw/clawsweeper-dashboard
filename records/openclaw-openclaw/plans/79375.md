## Summary

Keep open: current main still has a source-level gap for the canonical user-systemd plus system-systemd gateway pair. The shipped guard prevents installing a second user service only when the user service is missing, while a stale loaded/restartable user unit can still bypass duplicate detection.

## Plan

Fix https://github.com/openclaw/openclaw/issues/79375 as a focused Linux gateway/systemd bug. Current main only blocks a system-level OpenClaw gateway service when the user service is not loaded, and findExtraGatewayServices filters the canonical openclaw-gateway.service unit, so a stale canonical user unit plus a canonical system unit can remain undetected and let service-mode stale-pid cleanup kill the peer process. Add detection for the canonical user+system systemd pair for the same profile/port in doctor/status deep output, and route any stop/disable/remove behavior through existing confirmed service repair policy. Do not silently delete operator-owned units, do not change the default Linux installer away from the systemd user service, and do not add a new config option. Add focused regression tests for both scopes existing at once and update CHANGELOG.md with a user-facing fix entry.
