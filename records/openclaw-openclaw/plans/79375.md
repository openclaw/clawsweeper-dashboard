## Plan

Fix https://github.com/openclaw/openclaw/issues/79375. On Linux, when both the canonical user unit (~/.config/systemd/user/openclaw-gateway.service) and a canonical system unit (/etc/systemd/system/openclaw-gateway.service or other system systemd dirs) exist for the same gateway/profile/port, current doctor/status paths fail to report or clean up the duplicate supervisor. Add detection for this canonical user+system systemd pair in doctor/status. Route any stop/disable/remove/archive behavior through the existing confirmed service-repair policy; do not silently delete operator-owned units, do not change the default Linux user-service installer model, and do not add a config knob. Add regression coverage for loaded/enabled user service plus system service, and for deep doctor/status visibility. Only touch stale-pid cleanup if the sibling-supervisor loop still needs fail-fast hardening after duplicate detection. Add a user-facing changelog entry.
