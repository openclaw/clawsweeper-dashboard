## Summary

Keep open: current main still has the reported Codex app-server bridge gap. Declined native tools are recognized as blocked tool events, but the projector result still does not populate `lastToolError`, so the existing terminal payload error path can be bypassed for an aborted empty turn.

## Plan

Fix https://github.com/openclaw/openclaw/issues/83093 with a narrow Codex app-server adapter change. In `extensions/codex/src/app-server/event-projector.ts`, record non-success Codex-native tool completions such as declined/blocked or failed `commandExecution` as an `EmbeddedRunAttemptResult.lastToolError` with the existing normalized tool name, error text from `itemToolError`, meta where available, and mutating-action metadata where appropriate. Do not add Telegram-specific behavior, new config, or a new payload policy; reuse the existing `buildEmbeddedRunPayloads` `lastToolError` path. Add focused regression coverage showing a declined native command produces `lastToolError`, and that an aborted empty turn with that error emits a visible error payload. Preserve existing diagnostic/native-tool event behavior from https://github.com/openclaw/openclaw/pull/80217.
