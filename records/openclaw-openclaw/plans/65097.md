# Coding Plan for openclaw/openclaw#65097: Feature: Add language/locale config for Dreaming narrative generation

Source report: [records/openclaw-openclaw/items/65097.md](records/openclaw-openclaw/items/65097.md)

## Summary

Keep open. Current main still lacks a Dreaming language/locale config path: the Dream Diary prompt is fixed English, the narrative runner and phase callers carry only timezone/model, the strict memory-core schema rejects language/locale keys, and docs only document the existing settings. Related https://github.com/openclaw/openclaw/issues/69354 was closed as a duplicate of this issue, while https://github.com/openclaw/openclaw/issues/69533 and https://github.com/openclaw/openclaw/issues/79223 cover broader prompt-customization variants rather than superseding this narrower canonical request.

## Plan

Implement the canonical language/locale portion of https://github.com/openclaw/openclaw/issues/65097 as a focused memory-core feature. Add an explicit `plugins.entries.memory-core.config.dreaming.language` string config, resolve and normalize it through the memory-host Dreaming config path, pass it through light/REM/deep Dream Diary narrative generation, and inject a concise output-language directive into both the subagent prompt path and request-scoped fallback behavior without changing unrelated Dreaming ranking or storage policy. Update `extensions/memory-core/openclaw.plugin.json`, docs, and focused tests. Do not implement automatic locale inference from SOUL.md/USER.md/OS locale and do not implement full prompt-template/custom diary prompt support; leave that broader work to https://github.com/openclaw/openclaw/issues/69533. Include a changelog entry because this adds user-facing config.
