## Summary

Keep open. Current main still sends validation-shaped mutating tool failures through the user-facing warning path, and the related retryability PR does not implement chat-surface suppression for this issue.

## Plan

Fix the user-facing chat leak for tool-parameter validation failures. On current main, mutating validation-shaped lastToolError values still produce final warning payloads such as "Message failed" or "Write failed". Implement a typed validation/internal-only classification from parameter validation through adapter normalization and ToolErrorSummary/handleToolExecutionEnd, then update buildEmbeddedRunPayloads so only that class is suppressed from final chat surfaces. Preserve visible warnings for real unresolved mutating side-effect failures, especially failed write/edit/exec/message actions that the assistant did not acknowledge. Coordinate with https://github.com/openclaw/openclaw/pull/68070 if it has landed or changed the validation error contract, but do not turn this into a retryability-only fix. Add focused regression coverage for the suppressed validation case and for preserving ordinary mutating failure warnings. Add a user-facing CHANGELOG.md fix entry if policy still requires one for this AI-maintained fix.
