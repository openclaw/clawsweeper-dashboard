## Plan

Fix https://github.com/openclaw/openclaw/issues/79390 on current main. The narrow bug is that pre-run maintenance is not isolated enough from the parent reply operation: `runPreflightCompactionIfNeeded` passes `params.replyOperation.abortSignal` to `compactEmbeddedPiSession` and awaits it before the main turn, and `runMemoryFlushIfNeeded` passes both the parent signal and `replyOperation` into the maintenance embedded run. Add focused regression coverage that simulates preflight compaction timeout/rejection and memory-flush abort/failure, proving that auxiliary maintenance failures are logged/downgraded and the parent turn continues unless the parent/user aborts. Implement the smallest isolation needed so parent abort still cancels maintenance, but maintenance timeout/failure cannot fail/abort or attach mutable backend state to the parent reply operation.
