## Summary

Current main still does not implement the requested provider quota summary in the per-response `/usage` footer. `/status` has the provider-usage window plumbing and GitHub Copilot exposes Premium/Chat windows, but the reply footer path still formats only local turn/session usage, so the issue remains valid and narrow.

## Plan

Implement the feature requested by https://github.com/openclaw/openclaw/issues/7379: when `/usage tokens` or `/usage full` is enabled for a response and the active provider exposes provider usage windows, append a compact provider usage-window summary such as `Premium 86% left · Chat 100% left` to the response usage footer. Keep provider-specific fetching inside plugins; use the existing generic provider usage loader/formatter (`loadProviderUsageSummary`, `formatUsageWindowSummary`) rather than hardcoding GitHub Copilot behavior in the reply runner. Coordinate with the broader response-footer work in https://github.com/openclaw/openclaw/pull/69618 but do not add a new config surface or implement unrelated response footer templating. The provider usage lookup must be timeout-bounded and nonblocking for reply delivery. Update focused tests for the reply-runner/footer path and provider usage formatting; update token-use docs if the user-visible footer behavior changes. Do not edit `CHANGELOG.md` for a normal PR.
