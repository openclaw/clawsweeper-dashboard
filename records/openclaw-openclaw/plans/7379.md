## Summary

Keep open. Current main and latest release still separate provider quota-window display from the per-response `/usage` footer: `/status` can show active-provider Premium/Chat windows, but the reply footer only formats token/cache/cost/session data. The direct fix PR is closed unmerged, and the broader open footer PR is conflicting and does not own this specific quota-window request.

## Plan

Fix https://github.com/openclaw/openclaw/issues/7379 by adding a generic provider usage-window suffix to enabled response usage footers. When `/usage tokens` or `/usage full` is active and the active provider has usage windows, load the active provider usage summary through the existing provider-usage contract, format at most two windows with `formatUsageWindowSummary`, and append it to the existing reply footer without disrupting `appendUsageLine` metadata preservation. Keep provider-specific fetching inside plugins, do not hardcode GitHub Copilot in core, do not add config/env/options, and make provider usage loading bounded/nonblocking so timeout or error is a no-op for reply delivery. Coordinate against the overlapping but currently conflicting https://github.com/openclaw/openclaw/pull/69618. Update `docs/reference/token-use.md`, add regression coverage for Premium/Chat windows and timeout/error no-op behavior, and do not edit `CHANGELOG.md`.
