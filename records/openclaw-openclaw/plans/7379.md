Keep open. Current main and the latest release still format the per-response `/usage` footer from token/cache/cost/session data only, while `/status` already loads provider usage windows and the GitHub Copilot plugin already exposes Premium/Chat windows through the generic provider-usage contract.

## Plan

Implement the request from https://github.com/openclaw/openclaw/issues/7379. Add a generic reply-footer enhancement so when `/usage tokens` or `/usage full` is enabled and the active provider has provider usage windows, the footer appends a compact `formatUsageWindowSummary` result such as `Premium 86% left · Chat 100% left`. Keep provider-specific fetching inside provider plugins; do not add Copilot-specific network calls or policy to core. Use the existing provider usage loader/formatter with a bounded, best-effort timeout so usage fetch failures never fail or delay normal reply delivery beyond the intended bound.
