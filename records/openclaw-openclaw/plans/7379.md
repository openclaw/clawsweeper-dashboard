Source report: [records/openclaw-openclaw/items/7379.md](records/openclaw-openclaw/items/7379.md)

## Summary

Current main still lacks the requested bridge between provider quota windows and the per-response `/usage` footer. The reply path only formats token/cache/cost/session data, while `/status` already loads generic provider usage windows and GitHub Copilot already exposes Premium/Chat windows through the plugin usage hook.

## Plan

Fix https://github.com/openclaw/openclaw/issues/7379 by adding provider usage-window summaries to the per-response `/usage` footer. Keep the implementation generic: when `/usage tokens` or `/usage full` is enabled and the active provider exposes provider usage windows, append compact `formatUsageWindowSummary` output to the existing footer; do not add GitHub Copilot-specific footer logic in core. Reuse existing provider-usage loading/formatting contracts, keep provider-specific fetching inside plugins, keep provider quota loading bounded and nonblocking, and coordinate with nearby open footer work in https://github.com/openclaw/openclaw/pull/69618. Add regression coverage showing Premium/Chat quota text appears when provider windows are available and remains absent when unavailable. Update `docs/reference/token-use.md` and add a user-facing changelog entry unless maintainers decide to handle changelog during landing.
