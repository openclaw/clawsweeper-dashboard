# Coding Plan for openclaw/openclaw#7379: Feature Request: Add premium request % to /usage footer

Source report: [records/openclaw-openclaw/items/7379.md](records/openclaw-openclaw/items/7379.md)

## Summary

Keep open. Current main still does not bridge provider quota windows into the per-response `/usage` footer: the reply path only formats turn token/cache/cost/session data, while `/status` and the GitHub Copilot plugin already expose the Premium/Chat usage-window data.

## Plan

Resolve this issue by adding provider quota windows to the per-response `/usage` footer generically. When `/usage tokens` or `/usage full` is enabled and the active provider has usage windows, load the active provider's usage summary through the existing provider-usage plumbing, format it with `formatUsageWindowSummary`, and append it compactly to the existing footer while preserving current token/cache/cost/session behavior. Do not hard-code GitHub Copilot behavior in core, do not add a new config option, and keep provider usage fetching timeout-bounded and nonblocking. Coordinate with https://github.com/openclaw/openclaw/pull/69618 because it touches related footer plumbing; https://github.com/openclaw/openclaw/pull/43098 is useful prior art but is closed unmerged. Update `docs/reference/token-use.md` and add a user-facing `CHANGELOG.md` entry.
