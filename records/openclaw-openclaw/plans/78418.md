## Summary

Keep open: current main no longer auto-restores invalid external edits, but it still starts the runtime config watcher and reads/validates changes even when `gateway.reload.mode="off"`. The public docs now say `off` disables file watching, so the remaining scope is a narrow behavior/docs-contract bug rather than stale cleanup.

## Plan

Fix https://github.com/openclaw/openclaw/issues/78418 by making `gateway.reload.mode="off"` actually disable the runtime config watcher/validation loop. Preserve startup validation and doctor-owned config recovery. Prefer honoring the existing documented reload mode over adding a new `gateway.fileWatch` config field or `OPENCLAW_DISABLE_CONFIG_WATCHER` env var. Cover both initial startup with mode=off and a live accepted change that switches mode to off; after off is active, external file changes should not trigger chokidar-driven reads, validation warnings, hot reload, restart, or last-known-good promotion until the gateway is manually restarted. Do not fold in the separate recovery-notice policy request tracked by https://github.com/openclaw/openclaw/issues/72043. Add focused tests, update docs only if the chosen behavior differs from the existing docs, and add a user-facing CHANGELOG.md fix entry.
