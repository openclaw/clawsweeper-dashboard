## Plan

Fix https://github.com/openclaw/openclaw/issues/46023 on current main. The bug is that Telegram reply-media delivery still omits the configured media cap: `mediaMaxBytes` is computed in bot setup and direct sends pass it to `buildOutboundMediaLoadOptions`, but `createTelegramMessageProcessor`/`dispatchTelegramMessage`, native-command delivery, `deliverReplies`, and `deliverMediaReply` do not carry it into `loadWebMedia`. Implement the narrow parameter threading only: add optional `mediaMaxBytes` where needed, pass it from bot setup through normal reply dispatch and native-command reply delivery, and call `buildOutboundMediaLoadOptions({ mediaLocalRoots, maxBytes: mediaMaxBytes })` for reply media.
