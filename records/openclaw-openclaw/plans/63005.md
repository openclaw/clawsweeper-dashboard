## Summary

Keep open. Current main still has the raw-vs-canonical session-key mismatch: explicit inbound keys can remain raw while Gateway/session-tool paths canonicalize the same key to an agent-scoped key and then prune the raw candidate.

## Plan

Fix https://github.com/openclaw/openclaw/issues/63005 by making stable raw external session keys resolve consistently across inbound session initialization, Gateway sessions.resolve/sessions.send, nested agent activity, and migration/pruning. Implement a generic alias/lookup rule so a raw key with an existing active entry and its agent-scoped canonical form share one durable session entry, or preserve a raw alias that future explicit inbound SessionKey lookups can resolve. Do not add a narrow Open WebUI-only UUID exception as the generic fix, and do not relax normal canonicalization for standard agent/channel keys. Add regression coverage for a structured raw key such as conversation:pair:user_a::user_b:space:space_123, including the sequence raw inbound row -> Gateway/session-tool resolution/migration -> later raw inbound reuse with the same sessionId. Check whether the same fix covers https://github.com/openclaw/openclaw/issues/80064; if not, leave that as a separate follow-up. Add a user-facing CHANGELOG.md fix entry if the behavior changes.
