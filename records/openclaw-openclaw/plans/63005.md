## Summary

Keep open. Current main still canonicalizes arbitrary non-agent raw keys into the default agent namespace and can prune the original raw key, while inbound explicit session handling still looks up the raw key directly; the UUID and Voice Call related items cover narrower subsets, not this structured external-key case.

## Plan

Fix https://github.com/openclaw/openclaw/issues/63005 on current main. Preserve stable raw external session keys such as `conversation:pair:user_a::user_b:space:space_123` across Gateway `sessions.resolve`, `sessions_send`, nested agent handling, and subsequent inbound `SessionKey` lookups. Add a generic session-store alias/lookup rule so existing raw rows are either durable aliases or resolve to the same row before pruning; do not solve this with a UUID-only special case, do not remove existing main-session alias behavior, case-normalization cleanup, or deleted-agent checks, and coordinate with the narrower Open WebUI UUID work in https://github.com/openclaw/openclaw/pull/55351. Add focused regression coverage for a structured raw key seeded in the store, Gateway migration/prune touching it, and a later inbound lookup reusing the same `sessionId`.
