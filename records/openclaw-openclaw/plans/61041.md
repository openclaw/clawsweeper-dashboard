## Summary

Keep open. Current `main` and v2026.5.19 still reject shared gateway image attachments over `MAX_IMAGE_BYTES` before any server-side resize/recompress path can run; the merged iOS work only preprocesses iOS PhotosPicker attachments client-side.

## Plan

Fix oversized shared gateway image attachments by adding a bounded resize/recompress path before the image `MAX_IMAGE_BYTES` rejection in `parseMessageWithAttachments`. Add focused regression coverage for an oversized-but-resizable image, an unreducible image, and the existing non-image/offload paths.
