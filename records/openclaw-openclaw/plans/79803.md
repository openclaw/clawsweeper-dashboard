## Summary

Keep open: current main still carries the WebChat live-state gap, where delayed lifecycle errors for active `chat.send` runs without a chat link suppress the terminal chat error and leave the UI dependent on refresh/history reconciliation. The two related fix PRs were closed unmerged, and a later user comment reports the symptom after upgrading to `2026.5.12`.

## Plan

Fix the WebChat stuck-polling bug for provider 429/idle-timeout or no-content assistant failures. Current main preserves `skipChatErrorFinal` through `scheduleTerminalLifecycleError`, and the active `chat.send` no-link test expects no terminal chat error after the grace timer. Update the gateway lifecycle finalization so, after the retry grace expires and no fallback/end event has replaced the error, WebChat receives a terminal error or otherwise reloads the persisted failed assistant turn without requiring refresh. Preserve the intent of https://github.com/openclaw/openclaw/pull/60043: do not surface per-attempt fallback errors early when a fallback can still succeed. Use https://github.com/openclaw/openclaw/pull/79817 and https://github.com/openclaw/openclaw/pull/79858 as prior context, not as a blind patch. Add or update focused regression tests in `src/gateway/server-chat.agent-events.test.ts` and any needed UI tests around terminal error reconciliation.
