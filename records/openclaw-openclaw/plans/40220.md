# Coding Plan for openclaw/openclaw#40220: [Bug]: 日志时间字段不一致（_meta.date 用 UTC，time 用本地时间）

## Summary

Keep open: current main still computes a local-offset top-level `time` while serializing tslog's nested `_meta.date` unchanged, so raw JSON file logs can still show two different timestamp zones. The direct fix PRs are closed unmerged, and the related open logs item tracks CLI display defaults rather than this raw-field consistency bug.

## Plan

Fix https://github.com/openclaw/openclaw/issues/40220 by aligning raw JSON file-log `_meta.date` with the already computed local-offset `time` field. In `src/logging/logger.ts`, keep the existing top-level `time` calculation, clone the tslog `_meta` object when present, set `_meta.date` to `time`, and serialize the cloned record so `JSON.stringify` cannot emit the original Date as UTC. Add focused regression coverage in `src/logging/logger-timestamp.test.ts` that writes through `getLogger()` to a temp file and asserts `lastLine._meta.date === lastLine.time` and that the timestamp is offset-formatted. Do not change `openclaw logs` default display behavior, do not reinterpret `agents.defaults.userTimezone` as logging policy, and leave the CLI default/local-time product decision to https://github.com/openclaw/openclaw/issues/46748. Add a concise `CHANGELOG.md` fix entry unless the maintainer landing flow chooses to handle changelog separately.
