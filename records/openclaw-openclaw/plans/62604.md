## Summary

Keep open. Current main still lets text/plain or text/html `data:image/...;base64,...` clipboard payloads fall through to the composer because the paste handler only intercepts clipboard items whose MIME starts with `image/`; https://github.com/openclaw/openclaw/issues/74960 is useful related media-pipeline context, but it does not replace this distinct data-URL text-injection case.

## Plan

Fix the Control UI composer paste path for https://github.com/openclaw/openclaw/issues/62604.
