# Coding Plan for openclaw/openclaw#62604: [Bug]: Control UI image paste flow can inject massive base64 text into the composer; send/draft handling is hard to patch safely in installed dist bundles

Source report: [records/openclaw-openclaw/items/62604.md](records/openclaw-openclaw/items/62604.md)

## Summary

Keep open. Current main still only intercepts clipboard file items whose MIME starts with `image/` and returns without `preventDefault()` for text/plain or text/html data-image payloads, while no merged fix or test covers the raw composer base64 failure.

## Plan

Fix https://github.com/openclaw/openclaw/issues/62604 by hardening the Control UI WebChat composer paste path. In `ui/src/ui/views/chat.ts`, update `handlePaste` so clipboard payloads containing `data:image/...;base64,...` in text/plain or text/html do not fall through into the textarea as raw draft text; convert supported image data URLs into the existing `ChatAttachment`/payload-store path or reject them cleanly without inserting the huge base64 string. Preserve existing file-backed image paste, drop, file-picker behavior, attachment payload-store semantics, and send/draft restoration in `ui/src/ui/app-chat.ts`; do not move restore logic into UI button wrappers and do not address installed dist/source-map hotpatch policy in this PR. Add focused regression coverage in `ui/src/ui/views/chat.test.ts` for text/plain and/or text/html data-image paste, and touch `ui/src/ui/controllers/chat.test.ts` or `ui/src/ui/app-chat.test.ts` only if serialization or send-flow code changes. Add a changelog entry if the fix is user-visible under repo policy.
