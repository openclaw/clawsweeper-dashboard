## Summary

Keep open: current main and the latest release still reach `lsof` in the macOS/Unix gateway port paths implicated by the SMB stat-timeout hang, so the reported bug is not fully fixed.

## Plan

Fix https://github.com/openclaw/openclaw/issues/75767. On Unix/macOS, gateway restart and `gateway --force` should not run `lsof` for routine busy/free port checks before a socket probe can classify the port. Make the implementation socket-probe-first in `src/cli/ports.ts`, `src/infra/ports-inspect.ts`, and `src/infra/restart-stale-pids.ts`; reserve `lsof` or other process enumeration only for terminating verified OpenClaw gateway listener PIDs. Preserve Windows netstat/PowerShell behavior, existing EADDRINUSE/TIME_WAIT safeguards, and the OpenClaw-only PID attribution guard before killing processes. Add or update focused regression tests proving the macOS/Unix busy/free paths avoid `lsof` when a socket probe is sufficient and still safely handles verified OpenClaw process termination. Add a single `CHANGELOG.md` Fixes entry because this changes user-visible gateway restart behavior.
