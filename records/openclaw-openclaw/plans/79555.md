# Coding Plan for openclaw/openclaw#79555: fix(xai): context-aware thinking profile for reasoning-capable models

Source report: [records/openclaw-openclaw/items/79555.md](records/openclaw-openclaw/items/79555.md)

## Summary

Keep open: the updated PR fixes the earlier policy-only gap for `grok-4.3`, but it now treats every xAI model with `reasoning: true` as accepting configurable effort values. That would re-enable reasoning payloads for existing xAI reasoning routes outside the documented configurable-effort surface and risks reintroducing the invalid-effort failures current `main` deliberately avoids.

## Plan

Repair https://github.com/openclaw/openclaw/pull/79555 without broadening core policy. The PR should make `xai/grok-4.3` support existing `/think low|medium|high` on the wire, but must not send `reasoning`, `reasoningEffort`, or `reasoning_effort` to xAI reasoning routes that do not explicitly support configurable effort (for example `grok-4-1-fast`, `grok-4.20-beta-latest-reasoning`, and `grok-code-fast-1`). Add a provider-owned adjustable-effort predicate/helper in the xAI plugin, use it consistently in `resolveThinkingProfile`, `applyXaiRuntimeModelCompat`, and the stream wrapper safety gate, and keep non-adjustable reasoning models on the current all-null/off-only behavior. Update the existing changelog entry if its wording is too broad. Do not add a new core config option or change non-xAI provider behavior. Related issue: https://github.com/openclaw/openclaw/issues/79210.
