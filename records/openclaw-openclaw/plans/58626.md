## Summary

Keep open. Current main handles current-package bundled path aliases and the older `extensions/<id>` layout, but it still does not recognize a removed old-version pnpm/npm package path like `openclaw@2026.3.28.../dist/extensions/acpx`, so the reported gateway-start failure remains source-reproducible.

## Plan

Fix https://github.com/openclaw/openclaw/issues/58626. Extend the existing bundled plugin load-path doctor repair so `openclaw doctor --fix` can safely handle missing old-version OpenClaw package paths such as `.../.pnpm/openclaw@2026.3.28.../node_modules/openclaw/dist/extensions/acpx` after pnpm/npm global upgrades. Keep the boundary conservative: only remove or rewrite a missing path when it can be proven to be a bundled OpenClaw plugin path for a plugin that exists in the current bundled set; do not remove arbitrary custom `plugins.load.paths` entries and do not add startup-time config mutation. Add focused regression coverage for the old-version `dist/extensions/<id>` case, preserve the existing legacy `extensions/<id>` behavior, and add a user-facing changelog fix entry.
