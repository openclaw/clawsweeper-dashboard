## Summary

Keep open. Current main has retry classification for the reported transport errors, but live memory-search query embeddings still bypass the retry loop and call the provider once; the related open PR is batch-indexing scoped and does not close this live query gap.

## Plan

Fix https://github.com/openclaw/openclaw/issues/71784 by adding bounded retry/backoff for live memory-search query embeddings. Keep the change in `extensions/memory-core` if possible: `MemoryManager.search()` should still call a query embedding helper, but `embedQueryWithTimeout()` or a sibling helper should use `runMemoryEmbeddingRetryLoop()` with `isRetryableMemoryEmbeddingError` for transport/socket failures. Preserve fail-fast behavior for auth/configuration and non-retryable provider errors; do not add a config knob, change provider selection, tune global Undici dispatcher defaults, or fold in QMD SQLite lock work from https://github.com/openclaw/openclaw/issues/66339. Add focused regression coverage with a mocked provider whose `embedQuery()` fails once with `fetch failed | other side closed` or `ECONNRESET` and then succeeds, proving `MemoryManager.search()` returns results after retry.
