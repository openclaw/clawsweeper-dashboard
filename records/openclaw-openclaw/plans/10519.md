## Summary

Keep open. Current main and the latest shipped release still drop Markdown fence language metadata before Telegram HTML rendering, so Telegram fenced code blocks cannot emit the requested native language class.

## Plan

Fix https://github.com/openclaw/openclaw/issues/10519 by preserving Markdown fenced-code language identifiers for Telegram HTML output. Prefer adding sanitized optional language metadata to `MarkdownStyleSpan`/Markdown IR and carrying it through `sliceMarkdownIR`, `chunkMarkdownIR`, render-aware chunking, and `renderMarkdownWithMarkers`; use a Telegram-local fence-aware renderer only if the shared IR change is too broad. Emit `class="language-X"` only for fenced code blocks with a nonempty sanitized language, not for indented code blocks, and preserve existing Telegram escaping, file-reference wrapping, list-boundary handling inside fences, HTML fallback behavior, and chunk-limit balancing. Do not add a config option or change WebChat/UI copy-button behavior. Add focused regression coverage for Telegram formatter output and chunking behavior, plus shared IR tests if the IR contract changes. Add a changelog entry because this is user-visible Telegram behavior.
