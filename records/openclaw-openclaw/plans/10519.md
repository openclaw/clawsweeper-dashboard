## Summary

Keep open. Current main and the latest tagged release still render Telegram fenced code blocks as plain `<pre><code>` without preserving the fence language, and the related WebChat language-class/copy-button path is a separate UI renderer rather than Telegram Bot API output.

## Plan

Implement Telegram fenced-code language preservation for https://github.com/openclaw/openclaw/issues/10519. Current main drops markdown fence `token.info` in `src/markdown/ir.ts` and Telegram renders code blocks with static `<pre><code>` in `extensions/telegram/src/format.ts`; preserve a sanitized first fence-language token through the Markdown IR/render-aware chunking path, make the renderer able to emit `class="language-X"` for safe fenced blocks, and keep unlabeled or invalid fences unchanged. Do not allow arbitrary HTML attributes through the Telegram sanitizer, do not change unrelated channel formatting, and preserve existing escaping, file-reference wrapping, list-boundary handling, and chunk splitting behavior. Add focused tests for language fences, unlabeled fences, invalid fence info, sanitizer behavior, and chunking. Changelog can be handled by maintainers during landing if needed.
