# Coding Plan for openclaw/openclaw#10519: Telegram: Add language class to code blocks for native copy button support

Source report: [records/openclaw-openclaw/items/10519.md](records/openclaw-openclaw/items/10519.md)

## Summary

Keep open. Current main and the latest release still render Telegram fenced code blocks with a static `<pre><code>` marker, and the shared Markdown IR still drops the fence language before Telegram rendering; the related WebChat copy-button work is a separate UI path.

## Plan

Implement Telegram fenced-code language preservation for https://github.com/openclaw/openclaw/issues/10519. Preserve the optional first Markdown fence info token through either the shared Markdown IR/render-aware chunking path or a narrowly scoped Telegram renderer, sanitize it for an HTML class attribute, and emit `<pre><code class="language-js">` for fenced code blocks with a language. Keep indented code blocks and fences without a language classless, preserve existing Telegram HTML escaping, file-reference wrapping, link handling, chunk-limit behavior, and plain-text fallback behavior. Add focused tests for language fences, no-language fences, escaped/invalid language input, and chunked output. Update docs/changelog only if the shared Markdown/SDK contract or user-facing behavior requires it. Do not change the WebChat copy-button implementation except as context.
