## Summary

Keep open: current main and v2026.5.20 still have the WebChat reconnect-window delivery gap. Normal sends can return before queueing while disconnected, pending browser RPCs are rejected on socket close without method-aware replay, and the existing `chat.send` ACK/idempotency contract is not yet wired into a Control UI outbound queue.

## Plan

Fix https://github.com/openclaw/openclaw/issues/45952 by adding a focused Control UI outbound-send queue for normal WebChat `chat.send` submissions during transient WebSocket reconnects. Capture the message and attachment snapshot before transport, preserve the same run/idempotency key until the server ACKs, retry closed/not-connected or lost-ACK cases after reconnect hello, and show bounded failed-delivery feedback after retry exhaustion. Keep the fix scoped to transient reconnect delivery; do not broaden into browser-refresh history/draft persistence, do not change the gateway protocol unless the existing `chat.send` ACK/idempotency contract proves insufficient, and re-check the open adjacent PRs before choosing the final branch shape.
