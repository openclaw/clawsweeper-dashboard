# Coding Plan for openclaw/openclaw#45952: Webchat: messages lost during WebSocket reconnect (no client-side queue/ACK)

Source report: [records/openclaw-openclaw/items/45952.md](records/openclaw-openclaw/items/45952.md)

## Summary

Keep open: current main and the latest shipped tag still have a source-proven WebChat reconnect-window delivery gap. Normal sends are not captured while disconnected, pending browser RPCs are rejected on socket close without method-aware replay, and the existing `chat.send` ACK/idempotency contract is not yet wired into a Control UI outbound queue.

## Plan

Fix https://github.com/openclaw/openclaw/issues/45952 in a focused Control UI PR. Current main drops/rejects normal WebChat `chat.send` work during transient WebSocket reconnect because `handleSendChat` returns while disconnected, `GatewayBrowserClient` rejects pending RPCs on close, and no normal-send queue preserves the original idempotency key for replay. Add a bounded outbound-send queue for normal sends only: capture the payload before transport, generate and retain one stable run/idempotency key until server ACK, retry only transient closed/not-connected failures after reconnect hello, preserve/restore user-visible draft/error state, and show failed-delivery feedback after bounded retries. Keep busy-run `chatQueue` and slash/abort behavior compatible, do not add a config option, do not solve broader refresh/history/draft persistence from https://github.com/openclaw/openclaw/issues/51549, and avoid durable persistence of large attachment payloads without an explicit maintainer decision. Inspect nearby open PRs https://github.com/openclaw/openclaw/pull/73154 and https://github.com/openclaw/openclaw/pull/75776 before editing to avoid duplicating or conflicting with their narrower/broader WebChat reliability work. Add focused regression coverage for disconnected submit, in-flight socket close before ACK, reconnect flush with the same idempotency key, duplicate-ACK/in-flight handling, and bounded failure feedback. Add a concise user-facing Fixes changelog entry if the repair branch is maintainer-owned.
