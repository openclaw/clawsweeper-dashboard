# Coding Plan for openclaw/openclaw#44810: Codex ACP run can stall silently when gateway child environment lacks proxy/network access

Source report: [records/openclaw-openclaw/items/44810.md](records/openclaw-openclaw/items/44810.md)

## Summary

Keep open. Current main still has the OpenClaw-visible failure mode: the ACP parent relay emits the generic interactive-input stall notice when no assistant output arrives, while the current ACP agent bridge still drops non-text runtime/status events that could distinguish prompt-submitted/no-first-output from a real input wait.

## Plan

Fix https://github.com/openclaw/openclaw/issues/44810 with a focused current-main ACP diagnostics patch. Preserve the current packaged-ACPX boundary; do not revive the old in-repo event-parser PR. Carry prompt-submitted and runtime-activity/no-first-output signals from the ACP runtime/control-plane through `agent-command` and ACP dispatch into `acp-spawn-parent-stream`, so a child run that has accepted/submitted a prompt but produces no assistant output reports a concrete no-response/runtime-stall diagnostic instead of `may be waiting for interactive input`. Keep proxy environment diagnostics redacted to presence/shape only, and do not change proxy inheritance or permission policy. Add focused regression coverage for the parent relay classification and event propagation paths; include a concise CHANGELOG.md Fixes entry for the user-visible diagnostic change unless maintainers decide it is internal-only.
