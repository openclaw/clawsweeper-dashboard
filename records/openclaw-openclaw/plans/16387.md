# Coding Plan for openclaw/openclaw#16387: Feature: Dynamic header templating for custom LLM providers (enables Fireworks.ai prompt caching)

Source report: [records/openclaw-openclaw/items/16387.md](records/openclaw-openclaw/items/16387.md)

## Summary

Keep open. Current main still forwards configured provider headers and request headers as literal strings, the Fireworks plugin does not add session-affinity headers, and the linked dynamic-header implementation PRs remain closed unmerged while no open replacement PR is tracking the fix.

## Plan

Implement dynamic header templating for https://github.com/openclaw/openclaw/issues/16387. Add a narrow resolver for configured LLM provider headers from `models.providers.*.headers` and `models.providers.*.request.headers` that supports only `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` at model request dispatch. Preserve static header behavior, SecretRef marker stripping/resolution, header case-insensitive merge behavior, provider/model/request/caller precedence, auth override injection, protected attribution headers, provider-owned `wrapStreamFn` and `resolveTransportTurnState`, retries, and both normal turns and compaction. Carry the already-known sessionKey, agentId, and runId into the resolver; do not hardcode Fireworks-only behavior, add eval-style templating, broaden request-time plugin discovery, or change Pi packages unless a typed OpenClaw boundary absolutely requires it. Add focused regression coverage for provider headers, request.headers, normal run wrappers, compaction, and OpenAI-family transport, update docs for the placeholder contract, and add a user-facing changelog entry with safe contributor attribution.
