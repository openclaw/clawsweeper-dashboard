## Summary

Keep open: current main and v2026.6.5 still treat configured provider headers as literal static strings, while the closed implementation attempts were not merged and no canonical open replacement owns the remaining dynamic header placeholder work.

## Plan

Implement a narrow dynamic header placeholder resolver for `models.providers.*.headers` and `models.providers.*.request.headers`. Support only explicit placeholders `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` at model-request dispatch for normal turns and compaction; preserve literal static headers, SecretRef resolution, auth injection, attribution headers, retry/request policy behavior, caller header precedence, and provider-plugin stream hooks. Do not add broad templating, new environment variables, or Fireworks-only policy. Update focused docs for the header placeholder contract and add regression coverage for provider headers, request headers, normal run, compaction, and OpenAI transport/header precedence.
