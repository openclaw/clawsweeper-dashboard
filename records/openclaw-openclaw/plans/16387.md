## Summary

Keep open: current main still treats configured provider headers as literal strings, the Fireworks plugin does not add session-affinity headers, the exact dynamic-templating implementation attempts are closed unmerged, and the adjacent open work at https://github.com/openclaw/openclaw/pull/79990 adds static per-agent/job header scopes rather than runtime placeholder interpolation.

## Plan

Implement runtime placeholder interpolation for configured LLM provider headers requested by https://github.com/openclaw/openclaw/issues/16387. Add a small allowlisted resolver for `models.providers.*.headers` and `models.providers.*.request.headers` that supports only `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}`; leave unknown placeholders literal or reject them only if the existing config contract already supports that style. Thread the required runtime facts through normal model dispatch and compaction without hardcoding Fireworks, without changing provider plugin ownership, and without broad request-time rediscovery. Preserve existing static header behavior, SecretRef resolution, auth injection, provider attribution headers, request transport policy, retries, and provider-owned stream wrappers. Use the closed attempts at https://github.com/openclaw/openclaw/pull/16448 and https://github.com/openclaw/openclaw/pull/18213 only as background, not as a blind cherry-pick. Watch for overlap with https://github.com/openclaw/openclaw/pull/79990 if it lands first. Add focused regression tests and update the public provider config docs; add a changelog entry because this is user-facing.
