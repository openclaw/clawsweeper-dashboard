# Coding Plan for openclaw/openclaw#16387: Feature: Dynamic header templating for custom LLM providers (enables Fireworks.ai prompt caching)

Source report: [records/openclaw-openclaw/items/16387.md](records/openclaw-openclaw/items/16387.md)

## Summary

Keep open: current main still normalizes and merges configured provider headers as literal strings, while provider-owned turn-state hooks and prompt_cache_key support do not provide the requested generic config placeholder contract. The related implementation PRs are closed unmerged, and the provider/config gap remains narrow and source-confirmed.

## Plan

Implement the dynamic provider-header templating feature requested in https://github.com/openclaw/openclaw/issues/16387. Add a narrow allowlisted resolver for configured `models.providers.*.headers` and `models.providers.*.request.headers` at model-request dispatch time, supporting only `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}`. Preserve existing static header behavior, SecretRef marker/resolution behavior, auth overrides, protected attribution headers, provider/model/request/caller precedence, retries, provider-owned stream hooks, and compaction behavior. Use the closed PRs https://github.com/openclaw/openclaw/pull/16448 and https://github.com/openclaw/openclaw/pull/18213 only as background, not as code to copy wholesale. Do not broaden this into arbitrary template evaluation or request-body metadata; that broader attribution/privacy design is tracked separately at https://github.com/openclaw/openclaw/issues/63901. Add focused tests and docs for the new placeholder contract, and add a user-facing changelog entry.
