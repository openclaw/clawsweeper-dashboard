## Plan

Implement a narrow documented resolver for dynamic model-provider header placeholders. Scope: resolve only allowlisted placeholders such as {{session.key}}, {{agent.id}}, and {{run.id}} in models.providers.*.headers and models.providers.*.request.headers at model-request dispatch time for both normal turns and compaction. Preserve existing static header behavior, SecretRef handling, auth injection, OpenClaw attribution/protected headers, retry behavior, provider-plugin turn headers, and caller-header precedence. Do not add broad template evaluation, arbitrary environment expansion, provider-specific Fireworks policy in core, or Pi SDK changes unless a small typed seam is strictly required. Use https://github.com/openclaw/openclaw/issues/16387 as the canonical issue and treat closed PRs https://github.com/openclaw/openclaw/pull/16448 and https://github.com/openclaw/openclaw/pull/18213 only as background. Add focused tests for literal/static headers, configured placeholders, unknown placeholders, missing values, SecretRef markers, request.headers, normal-run context, compaction context, and provider-plugin turn headers. Update public docs and schema help for the placeholder contract; add a changelog entry if the repo landing workflow requires one.
