## Plan

Implement dynamic provider-header templating for the existing custom provider header config. Add a narrow allowlisted resolver for `models.providers.*.headers` and `models.providers.*.request.headers` that supports `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` at model-request dispatch for normal runs and compaction. Preserve existing static header behavior, SecretRef source-marker handling, auth override behavior, provider/model/request/caller header precedence, protected attribution headers, retries, and provider-owned `resolveTransportTurnState`/`wrapStreamFn` semantics. Do not add arbitrary template evaluation, provider-specific Fireworks logic, or broad plugin API churn unless the existing runtime context is insufficient. Update focused tests and user-facing docs, add a changelog entry for the feature, and stop for maintainer review if the fix requires choosing a broader public plugin API direction.
