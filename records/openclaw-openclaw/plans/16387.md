## Summary

Keep open: current main still normalizes and forwards configured provider headers as literal strings, so the requested session/run-aware interpolation is not implemented. The linked implementation PRs are closed unmerged, and the current provider-owned hook surface does not replace a documented custom-provider config resolver.

## Plan

Implement dynamic provider-header interpolation for openclaw/openclaw issue https://github.com/openclaw/openclaw/issues/16387. Add a narrow allowlisted resolver for `models.providers.*.headers` and `models.providers.*.request.headers` supporting `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` at model-request dispatch time. Preserve static header behavior, SecretRef marker/resolution behavior, auth injection, protected attribution headers, caller/provider/model/request precedence, retries, provider-owned stream hooks, and both normal-run and compaction paths. Do not add arbitrary template evaluation or provider-specific Fireworks-only logic. Add focused regression coverage and update public config docs plus CHANGELOG because this is user-visible config behavior.
