# Coding Plan for openclaw/openclaw#16387: Feature: Dynamic header templating for custom LLM providers (enables Fireworks.ai prompt caching)

## Summary

Keep open: current main still treats configured provider headers as literal strings, Fireworks does not add session-affinity headers itself, and the prior dynamic-header PRs are closed unmerged with no generic replacement tracking the requested behavior.

## Plan

Implement a focused OpenClaw feature for https://github.com/openclaw/openclaw/issues/16387: configured model-provider headers should support a small explicit placeholder allowlist at dispatch time, at least `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}`, for both `models.providers.*.headers` and `models.providers.*.request.headers`. Preserve current static header behavior, SecretRef resolution/redaction, auth injection, protected attribution headers, caller/turn header precedence, provider-owned stream hooks, retries, and compaction behavior. Do not add Fireworks-only core logic, do not move this into Pi, and do not introduce broad runtime rediscovery or a large plugin API. Use the closed implementation attempts https://github.com/openclaw/openclaw/pull/16448 and https://github.com/openclaw/openclaw/pull/18213 only as background; build against current main. Add focused tests for literal passthrough, supported placeholders, unknown placeholders, header precedence, normal runs, and compaction. Update the public config docs and schema/help text, and add a user-facing changelog entry if the behavior changes.
