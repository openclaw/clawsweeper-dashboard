# Coding Plan for openclaw/openclaw#16387: Feature: Dynamic header templating for custom LLM providers (enables Fireworks.ai prompt caching)

Source report: [records/openclaw-openclaw/items/16387.md](records/openclaw-openclaw/items/16387.md)

## Summary

Keep open: current main still forwards configured provider headers as literal strings, the latest release has the same static-header behavior, Fireworks does not add session-affinity headers, and the two related implementation PRs are closed unmerged.

## Plan

Implement dynamic provider-header templating for this issue in OpenClaw core, not in a Fireworks-only plugin. Add an allowlisted resolver for models.providers.*.headers and models.providers.*.request.headers that supports {{session.key}}, {{agent.id}}, and {{run.id}} at model request dispatch, preserves static headers and unknown/literal text behavior intentionally, and keeps SecretRef resolution, auth injection, protected attribution headers, retries, provider plugin turn-state headers, normal turns, and compaction safe. Use https://github.com/openclaw/openclaw/pull/16448 and https://github.com/openclaw/openclaw/pull/18213 only as background, not as code to merge blindly. Add focused tests for provider-level headers, request.headers, precedence, SecretRef markers, normal runs, and compaction. Add a changelog entry because this is user-visible config behavior. Do not broaden plugin SDK APIs unless the narrow core dispatch path cannot carry sessionKey/runId cleanly.
