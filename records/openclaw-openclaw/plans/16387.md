# Coding Plan for openclaw/openclaw#16387: Feature: Dynamic header templating for custom LLM providers (enables Fireworks.ai prompt caching)

Source report: [records/openclaw-openclaw/items/16387.md](records/openclaw-openclaw/items/16387.md)

## Summary

Keep open: current main and the latest release still treat configured provider headers as literal strings, and the closed related implementation PRs were not merged. The provider-plugin turn-state hook can support provider-owned dynamic headers, but it does not provide the requested custom-provider config placeholders for Fireworks, LiteLLM, or similar proxies.

## Plan

Implement the dynamic provider-header templating feature requested in https://github.com/openclaw/openclaw/issues/16387. Add a narrow allowlisted resolver for models.providers.*.headers and models.providers.*.request.headers at model-request dispatch time, supporting {{session.key}}, {{agent.id}}, and {{run.id}} only. Preserve existing static header behavior, SecretRef marker/resolution behavior, auth injection, caller header precedence, retry behavior, hidden attribution headers, provider-plugin resolveTransportTurnState headers, normal turns, and compaction. Do not add Fireworks-only core policy, a broad templating engine, or logging of resolved header values. Update focused tests and public docs; add a concise changelog entry if the final PR is user-facing.
