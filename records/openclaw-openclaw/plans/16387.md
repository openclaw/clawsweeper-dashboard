# Coding Plan for openclaw/openclaw#16387: Feature: Dynamic header templating for custom LLM providers (enables Fireworks.ai prompt caching)

## Summary

Keep open: current main still treats configured provider headers as literal strings, so the requested session/agent/run-aware header templating is not implemented. The newer provider-owned transport identity hook helps native provider plugins, but it does not resolve user-configured custom-provider header values; the earlier implementation PRs are closed unmerged and no open generic fix PR owns this exact work.

## Plan

Implement the narrow feature requested in https://github.com/openclaw/openclaw/issues/16387: add an allowlisted dispatch-time resolver for `models.providers.*.headers` and `models.providers.*.request.headers` supporting only `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}`. Preserve existing static headers, SecretRef marker/resolution behavior, auth/request-header precedence, provider attribution headers, retries, provider-owned stream hooks, and compaction behavior. Do not add arbitrary template evaluation, do not hardcode Fireworks-only behavior, and do not take over the broader OpenRouter `user`/`trace` metadata design tracked separately. Add focused tests for literal/static headers, each placeholder, unknown placeholders, SecretRef headers, normal turns, and compaction; update the public config docs and add a changelog entry if the repair lane owns user-facing changelog work.
