## Summary

Keep open: current main still treats configured provider/request headers as literal config values, while native session-affinity support and the adjacent Fireworks PR only cover known provider behavior rather than arbitrary dynamic header templating for custom providers.

## Plan

Implement explicit allowlisted header placeholder resolution for models.providers.*.headers and models.providers.*.request.headers at provider request dispatch. Support only documented placeholders such as {{session.key}}, {{agent.id}}, and {{run.id}}; preserve static headers, SecretRef resolution, auth/provider attribution headers, provider hooks, caller override precedence, and existing provider-native sendSessionAffinityHeaders behavior. Add focused tests and docs, and stop for maintainer review if the implementation requires broader templating or non-opt-in exposure of session identifiers.
