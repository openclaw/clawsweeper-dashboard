# Coding Plan for openclaw/openclaw#16387: Feature: Dynamic header templating for custom LLM providers (enables Fireworks.ai prompt caching)

## Summary

Keep open: current main and the latest release still pass configured provider headers through as literal strings, while the related implementation PRs are closed unmerged. Existing provider-plugin turn hooks can cover vendor-owned dynamic headers, but they do not provide the requested custom-provider config placeholders.

## Plan

Implement dynamic provider header templating for this issue. Add a narrow allowlisted resolver for models.providers.*.headers and models.providers.*.request.headers that resolves {{session.key}}, {{agent.id}}, and {{run.id}} at model-request dispatch time for normal turns and compaction. Preserve existing static header behavior, SecretRef resolution/marker stripping, auth injection, request.headers precedence, protected attribution headers, caller/turn headers, retries, provider-plugin resolveTransportTurnState behavior, and cache/prompt-cache behavior. Use the prior closed PRs only as background, not as code to apply wholesale. Add focused regression coverage for literal static headers, resolved placeholders, unknown placeholders or missing values per the chosen contract, request.headers, compaction, and header precedence. Update user-facing docs and add a changelog entry because this is a user-visible config feature.
