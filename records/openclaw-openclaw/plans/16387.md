# Coding Plan for openclaw/openclaw#16387: Feature: Dynamic header templating for custom LLM providers (enables Fireworks.ai prompt caching)

Source report: [records/openclaw-openclaw/items/16387.md](records/openclaw-openclaw/items/16387.md)

## Summary

Keep open: current main still treats configured provider headers and request headers as literal strings, Fireworks does not add session-affinity headers itself, and no open linked fix PR covers session/run header templating.

## Plan

Implement dynamic provider-header interpolation for https://github.com/openclaw/openclaw/issues/16387. Add a narrow resolver for `models.providers.*.headers` and `models.providers.*.request.headers` at LLM request dispatch, supporting only explicit placeholders such as `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}`. Preserve existing static-header behavior, SecretRef resolution/redaction, auth injection, attribution headers, caller/turn header precedence, provider-owned stream hooks, retries, and compaction. Carry prepared session/run identity facts into normal turns and compaction without broad request-time plugin/provider rediscovery. Do not add per-agent or per-cron header configuration in this PR; that adjacent surface is tracked by https://github.com/openclaw/openclaw/issues/80026 and https://github.com/openclaw/openclaw/pull/79990. Update docs and add a user-facing changelog entry.
