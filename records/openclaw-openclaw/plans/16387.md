# Coding Plan for openclaw/openclaw#16387: Feature: Dynamic header templating for custom LLM providers (enables Fireworks.ai prompt caching)

Source report: [records/openclaw-openclaw/items/16387.md](records/openclaw-openclaw/items/16387.md)

## Summary

Keep open: current main still treats configured provider headers as static strings and has no dispatch-time context for `{{session.key}}` or `{{run.id}}`; prior exact PRs are closed unmerged, and the adjacent open static-header PR does not cover runtime interpolation.

## Plan

Implement dynamic provider-header templating for this issue. Add an allowlisted dispatch-time resolver for configured LLM provider headers in `models.providers.*.headers` and `models.providers.*.request.headers`, supporting only `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}`. Preserve existing static header behavior, SecretRef resolution/markers, auth injection, protected attribution headers, caller/turn header precedence, retries, provider-owned stream wrappers, and both normal-run and compaction paths. Wire runtime context from the embedded run/compaction paths to the point where outbound provider headers are built; do not add broad plugin APIs or resolve templates at config-load time. Include focused regression tests and update user-facing docs; add a changelog entry because this is a user-visible config feature.
