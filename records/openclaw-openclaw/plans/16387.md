# Coding Plan for openclaw/openclaw#16387: Feature: Dynamic header templating for custom LLM providers (enables Fireworks.ai prompt caching)

Source report: [records/openclaw-openclaw/items/16387.md](records/openclaw-openclaw/items/16387.md)

## Summary

Keep open: current main and the latest release still pass configured provider headers through as literal strings, while the newer provider turn-state hook is provider-owned and does not implement the requested generic `{{session.key}}`/`{{agent.id}}`/`{{run.id}}` config contract. The two implementation attempts found in related GitHub search are closed unmerged, and there is no open closing PR.

## Plan

Implement dynamic provider header templating for https://github.com/openclaw/openclaw/issues/16387. Add a narrow allowlisted resolver for `models.providers.*.headers` and `models.providers.*.request.headers` values at model-request dispatch, supporting `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` only. Preserve existing static header behavior, SecretRef resolution, auth/header precedence, attribution headers, retries, provider-owned stream hooks, and current `prompt_cache_key` behavior. Cover normal turns and compaction. Update focused docs for provider headers and add tests showing static headers remain literal unless they contain supported placeholders, unknown placeholders are handled intentionally, and Fireworks/LiteLLM-style headers resolve to stable session/agent/run values. Do not change Pi packages or add arbitrary template evaluation.
