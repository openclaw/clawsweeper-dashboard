# Coding Plan for openclaw/openclaw#16387: Feature: Dynamic header templating for custom LLM providers (enables Fireworks.ai prompt caching)

## Summary

Keep open: current main still treats configured provider headers as static strings, while the adjacent provider turn-state hook and `prompt_cache_key` support do not provide the requested generic placeholder contract for custom provider headers. The two implementation PRs linked from the discussion are closed unmerged, and no newer canonical fix owns the remaining work.

## Plan

Implement dynamic provider-header placeholder resolution for https://github.com/openclaw/openclaw/issues/16387. Add a small allowlisted resolver for `models.providers.*.headers` and `models.providers.*.request.headers` that replaces exact placeholders `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` at model-request dispatch time. Preserve existing static header behavior, SecretRef resolution, auth override/injection, attribution headers, retries, provider-owned stream hooks, and header precedence. Do not add arbitrary template evaluation, broad fallback stacks, Fireworks-only core policy, or startup-time migration. Update public docs for provider headers and request headers, add focused regression tests for static values, exact placeholders, unknown/static brace strings, SecretRef/static merge behavior, normal runs, and compaction where the context is available. Add a concise changelog entry because this is user-visible config behavior.
