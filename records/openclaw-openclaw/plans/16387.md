## Summary

Keep open: current main still normalizes configured provider headers as literal strings, has no dispatch-time placeholder context for `{{session.key}}` or `{{run.id}}`, and the exact dynamic-header PR attempts at https://github.com/openclaw/openclaw/pull/16448 and https://github.com/openclaw/openclaw/pull/18213 remain closed unmerged. The adjacent open work at https://github.com/openclaw/openclaw/pull/79990 adds static per-agent/job header scopes, not runtime interpolation.

## Plan

Implement dynamic provider-header templating for https://github.com/openclaw/openclaw/issues/16387. Add a narrow, documented, allowlisted resolver for configured `models.providers.*.headers` and `models.providers.*.request.headers` values so `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` resolve at model request dispatch. Preserve literal static headers, SecretRef handling, auth injection, request header precedence, protected attribution headers, retries, provider-owned stream hooks, and current Node/Bun paths. Do not add provider-specific Fireworks policy in core beyond the generic placeholder resolver. Coordinate with adjacent static-scoping work in https://github.com/openclaw/openclaw/pull/79990 if it lands first. Add focused regression coverage for provider/model/request headers, OpenAI transport dispatch, normal turns, and compaction. Update user-facing docs for the placeholder contract and include a changelog entry because this is user-facing config behavior.
