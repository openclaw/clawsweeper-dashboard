## Summary

Keep open: current main still normalizes and merges configured provider headers as literal strings, and the newer provider turn-state hook is plugin-owned rather than a generic `{{session.key}}`/`{{agent.id}}`/`{{run.id}}` config placeholder contract. The exact implementation attempts are closed unmerged, and live search did not find an open canonical replacement PR.

## Plan

Implement dynamic provider header templating for custom model providers. Add an allowlisted resolver for `models.providers.*.headers` and `models.providers.*.request.headers` that supports `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` at model-request dispatch time. Preserve existing static header behavior, SecretRef resolution/marker stripping, auth injection, attribution headers, provider request policies, retries, provider plugin turn-state hooks, normal turns, and compaction. Do not add arbitrary template evaluation or Fireworks-only core policy. Add focused regression coverage for provider headers, request headers, normal run dispatch, and compaction; update the public config docs; add a concise `CHANGELOG.md` entry for the user-visible config behavior.
