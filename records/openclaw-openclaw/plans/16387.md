# Coding Plan for openclaw/openclaw#16387: Feature: Dynamic header templating for custom LLM providers (enables Fireworks.ai prompt caching)

Source report: [records/openclaw-openclaw/items/16387.md](records/openclaw-openclaw/items/16387.md)

## Summary

Keep open: current main still treats configured provider headers and provider request headers as literal strings, the Fireworks plugin does not add session-affinity itself, and the prior dynamic-templating PRs are closed unmerged. Adjacent open PRs for agent headers or per-agent/job static headers do not cover session/run placeholder interpolation for arbitrary configured headers.

## Plan

Fix https://github.com/openclaw/openclaw/issues/16387. Implement a narrow documented dispatch-time resolver for configured `models.providers.*.headers` and `models.providers.*.request.headers` so explicit placeholders such as `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` are resolved per model request. Preserve static header behavior, SecretRef resolution, auth injection, forbidden-header filtering, attribution header precedence, caller/turn header precedence, provider runtime turn-state headers, compaction behavior, and provider-owned stream hooks. Do not add Fireworks-specific defaults, do not broaden per-agent/cron header surfaces from https://github.com/openclaw/openclaw/pull/79990, and stop for maintainer guidance if a requested placeholder cannot be provided without a new public product decision. Add focused regression tests for provider headers and request.headers, normal run and compaction where applicable, unknown placeholders, missing values, and static headers. Add/update user-facing docs and a changelog entry because this changes provider config behavior.
