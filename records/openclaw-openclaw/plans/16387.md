# Coding Plan for openclaw/openclaw#16387: Feature: Dynamic header templating for custom LLM providers (enables Fireworks.ai prompt caching)

Source report: [records/openclaw-openclaw/items/16387.md](records/openclaw-openclaw/items/16387.md)

## Summary

Keep open: current main still forwards configured provider headers as literal strings, so the requested session/agent/run-aware header templating is not implemented. The provider-owned turn-state hook helps native provider plugins, but it does not provide a documented resolver for user-configured custom-provider headers; the two related implementation PRs are closed unmerged and no open PR currently owns the generic fix.

## Plan

Implement dynamic provider-header templating for https://github.com/openclaw/openclaw/issues/16387. Add a narrow allowlisted resolver for `models.providers.*.headers` and `models.providers.*.request.headers` supporting only `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` at model-request dispatch time. Preserve literal static headers, SecretRef resolution/markers, auth override precedence, protected attribution headers, provider-owned stream/transport hooks, retries, and existing provider/model/header merge order. Cover normal turns and compaction; do not add arbitrary template evaluation, do not move the feature into Pi, and do not make this Fireworks-only. Update focused docs and changelog for the user-visible config contract.
