## Summary

Keep open: current main and the latest release still pass provider header values through literally, while the related dynamic-header implementation PRs are closed unmerged. The existing provider-owned turn-state hook can add vendor headers, but it does not resolve custom config placeholders for Fireworks, LiteLLM, or similar proxy use cases.

## Plan

Fix the dynamic provider-header templating gap for https://github.com/openclaw/openclaw/issues/16387 in one focused PR. Add a documented, allowlisted resolver for configured `models.providers.*.headers` and `models.providers.*.request.headers` values at model request dispatch, supporting `{{session.key}}`, `{{agent.id}}`, and `{{run.id}}` only. Preserve existing static header behavior, SecretRef handling, auth injection, retries, attribution/protected headers, provider-plugin turn headers, and normal provider/model header precedence. Cover both normal turns and compaction paths; stop and report back if the implementation would require a broad plugin API redesign or an unconditional provider-specific policy. Include focused regression tests and a user-facing changelog entry.
