# Coding Plan for openclaw/openclaw#79897: OpenAI-compatible streaming with llama.cpp saves zero usage (stream closed before final usage chunk)

Source report: [records/openclaw-openclaw/items/79897.md](records/openclaw-openclaw/items/79897.md)

## Summary

Keep open: current main still has a source-reproducible usage-loss path for llama.cpp-style OpenAI-compatible streams whose final usage arrives after `[DONE]`. The related closed PR was not merged, and the current compat gate does not add a raw-SSE or fallback path that would preserve late usage into saved assistant/session usage.

## Plan

Fix https://github.com/openclaw/openclaw/issues/79897 in a focused OpenAI-completions streaming PR. Current main records usage only from SDK-yielded chunks, and OpenAI Node skips SSE after `[DONE]`; llama.cpp can emit final usage/timings after that point, so saved assistant/session usage stays zero or unknown. Implement the narrow transport or ingestion fix that lets final llama.cpp usage/timings reach `parseTransportChunkUsage` / `normalizeUsage` / embedded session persistence. Do not hardcode a provider-specific llama.cpp policy if a generic OpenAI-compatible stream boundary works, do not broadly remove `stream_options.include_usage` for local backends that need it, and preserve the strict-provider compatibility concern from https://github.com/openclaw/openclaw/issues/45706. Add focused regression coverage for post-DONE or final-done usage/timings and for no double-counting. Changelog is expected if user-visible token accounting changes.
