Keep open: current main still has a source-reproducible OpenAI-compatible streaming usage loss for llama.cpp-style streams. OpenClaw requests streamed usage, but the consumer only sees OpenAI SDK-yielded chunks, and OpenAI Node v6.37.0 skips SSE messages after `[DONE]`; the related one-line gating PR was closed without merge and would not preserve the metadata the reporter needs.

## Plan

Fix https://github.com/openclaw/openclaw/issues/79897 by preserving llama.cpp-style final streaming usage/timings for OpenAI-compatible Chat Completions streams. Investigate `src/agents/openai-transport-stream.ts` and `src/agents/provider-transport-fetch.ts` so usage/timings that arrive after `[DONE]` are surfaced before the OpenAI SDK drops them, or otherwise carried into `processOpenAICompletionsStream` and the embedded session persistence path.
