## Summary

Keep open: current main and the latest release still have the documented and implemented node-host exec approval commands, but the gateway node command policy does not allow them through pairing/reconnect advertisement, so `nodes describe` and the Control UI target picker can still miss valid exec-approvals-capable Windows headless nodes.

## Plan

Fix the Windows/headless node exec approvals advertisement mismatch for https://github.com/openclaw/openclaw/issues/57775. Add the shared `NODE_EXEC_APPROVALS_COMMANDS` to the desktop node command policy path that pairing/reconnect uses so `nodes describe` and the Control UI can see `system.execApprovals.get/set` when a headless node declares them.
