## Summary

Keep open: current main and v2026.5.19 still filter `system.execApprovals.get/set` out of the advertised desktop node command surface even though the docs and node host implementation expect those commands to exist.

## Plan

Fix this issue by aligning gateway node command policy with the documented and implemented headless node exec approval commands. Add `NODE_EXEC_APPROVALS_COMMANDS` to the desktop/headless command allowlist path so Windows/Linux/macOS headless node hosts that declare `system.execApprovals.get` and `system.execApprovals.set` retain those commands through pairing/reconnect normalization and appear as Control UI exec-approval targets. Keep the existing `node.invoke` block for `system.execApprovals.*`; users should still use `exec.approvals.node.*`/`openclaw approvals --node`. Do not add a config flag or widen unrelated dangerous command defaults. Add regression coverage for gateway policy/reconnect/approval and UI target discovery, and add a single changelog fix entry because this is user-visible.
