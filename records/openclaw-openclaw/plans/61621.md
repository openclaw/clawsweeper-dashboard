## Summary

Keep open: current main and the latest release still build Feishu agent text with a standalone `[message_id: ...]` line plus a sender label, and the outbound user-facing sanitizer does not strip that Feishu scaffold if the assistant echoes it.

## Plan

Fix the Feishu visible metadata leak reported at https://github.com/openclaw/openclaw/issues/61621. Keep `[message_id: ...]` and sender IDs available to agent context, but prevent copied Feishu scaffolding like `[message_id: ...]\nou_...: ...` from reaching channel-visible replies. Prefer a narrow user-facing sanitizer/shared helper used by `normalizeReplyPayload` or a Feishu-specific outbound transform; do not add a default-visible `showMetadata` config option. Add regression coverage for standalone message-id lines plus Feishu sender-label lines, quote/reply context, and preserving inline user-authored `[message_id: ...]` text. Add a `CHANGELOG.md` fix entry because this changes user-visible Feishu output.
