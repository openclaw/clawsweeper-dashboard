## Summary

Keep open. Current main and the latest tagged release still run the hooks CLI report/toggle path through full plugin diagnostics, which can enter runtime plugin loading; the broader bootstrap hang was closed separately and explicitly left the hooks-specific case to this issue.

## Plan

Fix the Windows hooks CLI timeout tracked by https://github.com/openclaw/openclaw/issues/70451. Current main still has `src/cli/hooks-cli.ts` `buildHooksReport` call `buildPluginDiagnosticsReport`, which enters `src/plugins/status.ts` runtime plugin loading via `loadOpenClawPlugins`. Replace the hooks CLI report/toggle dependency on full runtime plugin diagnostics with a metadata/snapshot/light-descriptor path so `openclaw hooks list`, `info`, `check`, bare `hooks`, `enable`, and `disable` do not load the full runtime plugin registry. Preserve workspace hook enable/disable config writes, plugin-managed hook visibility as `plugin:<id>`, and the rule that plugin-managed hooks cannot be enabled or disabled from the hooks CLI. Add focused regression coverage proving the hooks CLI report path avoids runtime plugin loading while keeping plugin-managed hook display and toggle restrictions. Add a `CHANGELOG.md` fix entry if the repair changes user-visible behavior. Do not broaden this into a general Windows loader refactor or add a new config option.
