# Coding Plan for openclaw/openclaw#58765: Feature: Support output dimensionality truncation for local GGUF embedding models

Source report: [records/openclaw-openclaw/items/58765.md](records/openclaw-openclaw/items/58765.md)

## Summary

Keep open: current main and v2026.5.22 still carry `memorySearch.outputDimensionality` through config resolution, but the built-in local GGUF provider never reads it and still returns full normalized vectors. The closed related dimension issue solved the OpenAI-compatible path, not this local provider path.

## Plan

Fix local GGUF memory-search output dimensionality for https://github.com/openclaw/openclaw/issues/58765. Reuse the existing `agents.defaults.memorySearch.outputDimensionality` / `agents.list[].memorySearch.outputDimensionality` field; do not add a new config key. In the local `node-llama-cpp` embedding provider, truncate raw vectors to the requested positive dimension before normalization for both `embedQuery` and `embedBatch`. Make the local memory-core provider runtime cache/reindex identity include `outputDimensionality` so changing it forces a full reindex and separates cached embeddings. Add focused regression tests for query and batch truncation plus provider-key/reindex behavior, update memory config docs, add a changelog entry, and leave `queryInstruction`/Qwen instruction support out of scope. Do not touch unrelated providers or broad memory architecture.
