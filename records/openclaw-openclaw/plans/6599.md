Keep open. Current main still lacks a primary-skipping fallback-chain probe, the direct implementation PR is closed unmerged, and the broader per-model health-check request does not replace this targeted command.

## Plan

Implement a focused fallback-chain probe for https://github.com/openclaw/openclaw/issues/6599. Add `/models test-fallback` and, if consistent with current CLI patterns, `openclaw models test-fallback`, reusing real model/auth execution and fallback resolution rather than the existing `models status --probe` auth-profile planning. The probe should intentionally skip the configured primary, test the first configured fallback by default, support an all-chain mode (`all` or `--all`), report which provider/model answered and any failures, and avoid letting later fallbacks mask the candidate being tested. Respect active agent/session fallback overrides where the slash-command surface has that context. Keep broad per-model health checks out of scope; that remains tracked at https://github.com/openclaw/openclaw/issues/63145. Update user-facing docs and CHANGELOG for the new command, and do not change unrelated fallback ordering or auth-profile probe semantics.
