## Plan

Implement https://github.com/openclaw/openclaw/issues/8812 as a focused enhancement. Add safe clickable URL rendering for tool output in Control UI while preserving JSON/plain-text formatting: escape arbitrary tool output before creating anchors, link only safe http/https URLs, handle trailing punctuation and whitespace, and cover expanded output, raw details, collapsed previews, short inline cards, grouped tool-message paths, and sidebar/raw views. If native WebChat remains in scope, add equivalent SwiftUI URL rendering with AttributedString or Link handling around ToolResultCard/ToolResultTextFormatter. Do not apply markdown wholesale to arbitrary tool output, do not use unsafeHTML without escape-first construction, and do not add a config option. Add a user-facing changelog entry if behavior changes ship.
