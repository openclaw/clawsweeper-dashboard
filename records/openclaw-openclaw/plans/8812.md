# Coding Plan for openclaw/openclaw#8812: [Feature Request] Auto-linkify URLs in tool output cards (Control UI & WebChat)

Source report: [records/openclaw-openclaw/items/8812.md](records/openclaw-openclaw/items/8812.md)

## Summary

Keep open. Current main and the latest release still render URL-bearing tool output as escaped text/code in Control UI and plain SwiftUI Text in native WebChat, while the directly related implementation at https://github.com/openclaw/openclaw/pull/8880 was closed unmerged.

## Plan

Implement clickable http/https URL rendering for tool output without changing tool-result semantics. Treat https://github.com/openclaw/openclaw/pull/8880 as prior art only: add an escape-first URL renderer/helper, apply it consistently to Control UI expanded output, raw details, collapsed previews, inline short cards, grouped tool-card/JSON paths, and sidebar/raw views where tool output is shown, and preserve JSON/plain-text formatting. Do not use unsafeHTML on unescaped tool output; only emit safe http/https anchors with target="_blank" and noopener/noreferrer. If native WebChat is included, render URLs with SwiftUI URL-capable text such as AttributedString or Link rather than plain Text. Add focused regression coverage for URL linkification, HTML escaping, trailing punctuation/whitespace, non-http schemes, JSON URL fields, and the key render paths. Include a concise CHANGELOG.md entry if this worker produces the user-facing feature PR.
