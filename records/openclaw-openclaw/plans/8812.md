Current main and v2026.5.12 still render URL-bearing tool output as escaped text or code in Control UI and as plain SwiftUI Text in native WebChat; the directly related implementation at https://github.com/openclaw/openclaw/pull/8880 was closed unmerged, so there is no shipped fix to close against.

## Plan

Implement safe clickable URL rendering for tool output cards requested by https://github.com/openclaw/openclaw/issues/8812. Treat https://github.com/openclaw/openclaw/pull/8880 as prior art only. Add a shared Control UI helper that escapes arbitrary tool output before creating anchors, only linkifies safe http/https URLs, handles trailing punctuation/whitespace, and preserves JSON/plain-text formatting. If changing native WebChat, use SwiftUI URL-capable rendering such as AttributedString or Link handling rather than plain Text.
