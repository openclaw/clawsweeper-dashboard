# Coding Plan for openclaw/openclaw#8812: [Feature Request] Auto-linkify URLs in tool output cards (Control UI & WebChat)

Source report: [records/openclaw-openclaw/items/8812.md](records/openclaw-openclaw/items/8812.md)

## Summary

Keep open: current main still renders URL-bearing tool output as plain text in Control UI tool-card paths and native WebChat, while normal markdown linkification does not cover those tool-card renderers. The related implementation at https://github.com/openclaw/openclaw/pull/8880 was closed unmerged, so it remains useful prior art rather than shipped behavior.

## Plan

Implement safe URL linkification for tool output cards for https://github.com/openclaw/openclaw/issues/8812, using https://github.com/openclaw/openclaw/pull/8880 only as prior art. Keep the fix in the UI/native owner surfaces: add an escape-first helper for Control UI tool output that only creates safe http/https anchors with target=_blank and rel=noopener/noreferrer, handles trailing punctuation/whitespace, and preserves JSON/plain-text formatting. Apply it to expanded output blocks, raw details, collapsed previews, inline short cards, grouped tool-message paths, and sidebar/raw views as applicable. If native WebChat remains in scope, add equivalent SwiftUI URL rendering using AttributedString/Link behavior rather than plain Text. Do not change the tool-output protocol, add a config option, or use unsafeHTML on unescaped tool output. Add focused regression tests and a user-facing changelog entry if the rendered behavior changes.
