## Summary

Current main still leaves the requested behavior unresolved: Control UI tool-card output and native WebChat tool-result output render URL strings through text-only paths, while the normal markdown linkifier is separate from those cards. The earlier related PR is useful prior art, but it was closed unmerged and is not shipped behavior.

## Plan

Fix https/http URL rendering in Control UI tool-output cards for https://github.com/openclaw/openclaw/issues/8812. Add a shared safe renderer/helper that escapes arbitrary tool output before creating anchors, supports only safe http/https URLs, handles trailing punctuation/whitespace, and preserves JSON/plain-text formatting. Apply it across expanded output, raw details, collapsed previews, short inline cards, grouped tool-message paths, and sidebar/raw views as applicable. Treat https://github.com/openclaw/openclaw/pull/8880 as prior art only; do not introduce raw HTML injection over unescaped tool output. If native WebChat remains in scope, add equivalent SwiftUI URL rendering with AttributedString or Link rather than plain Text. Add focused regression coverage and avoid unrelated plugin/channel behavior changes.
