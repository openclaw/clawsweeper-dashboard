## Summary

Keep open. Current main still renders URL-bearing tool output through plain text/code paths in Control UI and native WebChat, while the only issue-closing PR found for this request is closed and unmerged.

## Plan

Implement safe URL linkification for tool output cards on current main. Start with Control UI: add or reuse a shared helper/renderer that escapes arbitrary tool output before creating anchors, only links http/https URLs, handles trailing punctuation and whitespace, preserves JSON/plain-text formatting, and does not route untrusted tool output through broad markdown/HTML injection. Apply it consistently to expanded output, raw details, collapsed previews, inline short cards, grouped tool-message paths, and sidebar/raw views. Treat https://github.com/openclaw/openclaw/pull/8880 as prior art but do not copy its partial surface coverage. Coordinate with still-open overlapping tool-output UI PRs if they remain open. If native WebChat is included, use SwiftUI `AttributedString`/`Link`-style rendering rather than raw HTML. Add focused regression tests for safe escaping, safe schemes, trailing punctuation, multiple URLs, JSON URL fields, and no anchors for non-http schemes. Add a single-line CHANGELOG.md entry because this is user-visible.
