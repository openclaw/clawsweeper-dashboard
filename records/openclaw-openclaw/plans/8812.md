# Coding Plan for openclaw/openclaw#8812: [Feature Request] Auto-linkify URLs in tool output cards (Control UI & WebChat)

Source report: [records/openclaw-openclaw/items/8812.md](records/openclaw-openclaw/items/8812.md)

## Summary

Keep open. Current main still renders URL-bearing tool output as text/code in Control UI and plain SwiftUI text in native WebChat, and the directly related implementation was closed unmerged, so this remains a focused queueable UI feature.

## Plan

Implement clickable URL rendering for URL-bearing tool output in Control UI. Add or reuse a shared helper in ui/src/ui/chat/tool-helpers.ts or an adjacent chat UI helper that escapes arbitrary text before wrapping safe http/https URLs with anchors using target="_blank" and rel="noopener noreferrer", handles trailing punctuation/whitespace, and preserves JSON/plain-text readability. Apply it across expanded tool data blocks, collapsed previews, short inline cards, grouped tool-message JSON/tool-card paths, raw details/sidebar content as appropriate. Treat https://github.com/openclaw/openclaw/pull/8880 as prior art, but do not copy an unsafe or partial approach. If native WebChat remains in scope, use AttributedString or Link in ChatMessageViews/ToolResultTextFormatter with focused Swift tests. Do not link non-http(s) schemes or render arbitrary tool output through unsafeHTML without an escape-first guard. Include focused regression tests and a user-facing changelog entry if landing as maintainer-owned feature work.
