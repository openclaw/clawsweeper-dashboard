# Coding Plan for openclaw/openclaw#8812: [Feature Request] Auto-linkify URLs in tool output cards (Control UI & WebChat)

## Summary

Keep open: current main still renders URL-bearing tool outputs as text in the Control UI tool-card paths and native WebChat, while the prior implementation at https://github.com/openclaw/openclaw/pull/8880 was closed unmerged.

## Plan

Implement https://github.com/openclaw/openclaw/issues/8812. Make http/https URLs in tool output clickable while preserving JSON/plain-text formatting and treating tool output as untrusted. Treat https://github.com/openclaw/openclaw/pull/8880 as prior art, not a branch to merge wholesale. Add an escape-first URL rendering helper for Control UI tool outputs that supports only safe http/https schemes, handles trailing punctuation and whitespace, sets target="_blank" plus rel="noopener noreferrer", and covers expanded blocks, raw details, collapsed previews, short inline cards, grouped tool-message paths, and sidebar/raw views. Add the native WebChat equivalent with SwiftUI AttributedString/Link handling if the existing ToolResultCard path remains in scope. Add focused regression tests and a user-facing changelog entry. Do not use unsafeHTML with unescaped tool output, do not linkify non-http schemes, and do not refactor unrelated chat rendering.
