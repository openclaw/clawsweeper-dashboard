# Coding Plan for openclaw/openclaw#8812: [Feature Request] Auto-linkify URLs in tool output cards (Control UI & WebChat)

Source report: [records/openclaw-openclaw/items/8812.md](records/openclaw-openclaw/items/8812.md)

## Summary

Keep open: current main still renders URL-bearing tool output as escaped/plain text in the affected Control UI tool-card paths and native WebChat, while the linked implementation attempt at https://github.com/openclaw/openclaw/pull/8880 was closed unmerged.

## Plan

Implement safe URL linkification for tool output cards from https://github.com/openclaw/openclaw/issues/8812. In Control UI, add or reuse an escape-first renderer that preserves JSON/plain-text formatting, emits anchors only for safe http/https URLs, handles trailing punctuation/whitespace, and covers expanded output, raw details, collapsed previews, inline short cards, and grouped tool-message paths. Do not render arbitrary tool output as unsafe HTML before escaping. For native WebChat, either add equivalent SwiftUI URL rendering using AttributedString/Link handling for ToolResultCard/ToolResultTextFormatter output, or leave a narrowly documented follow-up if the native scope is intentionally deferred. Use https://github.com/openclaw/openclaw/pull/8880 as prior art, but do not copy incomplete behavior that only covers previews/inline cards. Add focused regression tests for URL linkification, HTML escaping, safe-scheme filtering, trailing punctuation, and the covered render paths. Add a user-facing changelog entry because this changes UI behavior.
