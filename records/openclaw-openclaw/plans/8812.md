# Coding Plan for openclaw/openclaw#8812: [Feature Request] Auto-linkify URLs in tool output cards (Control UI & WebChat)

Source report: [records/openclaw-openclaw/items/8812.md](records/openclaw-openclaw/items/8812.md)

## Summary

Keep open. Current main and v2026.5.12 still render URL-bearing tool output through plain text bindings in Control UI and native WebChat, and the only directly linked implementation, https://github.com/openclaw/openclaw/pull/8880, was closed unmerged.

## Plan

Implement safe URL linkification for tool result output in Control UI. Add a helper that escapes all text before producing anchors, only emits safe http/https links with target _blank and noopener/noreferrer, handles trailing punctuation and whitespace, preserves JSON/pre formatting and newlines, and routes expanded blocks, raw details, collapsed previews, inline cards, grouped tool messages, and sidebar/raw views through the same safe behavior. Treat https://github.com/openclaw/openclaw/pull/8880 as prior art, but adapt it to the current outputText APIs and cover all current paths. If native WebChat stays in scope, add AttributedString or Link handling for ToolResultCard without changing ToolResultTextFormatter's summarization contract. Add focused regression tests and a user-visible changelog entry. Do not change tool execution, message normalization, gateway protocol, or plugin behavior.
