## Summary

Keep open: current main and v2026.6.10 still compose exec/bash failure warnings as a compact command-like summary plus `failed`, and the exec display helper can still produce `run python3 <script>` as that summary, so the reported ambiguity remains a narrow display bug.

## Plan

Fix the misleading exec tool failure warning format so framework-generated summaries cannot be mistaken for the literal command.
