# Coding Plan for openclaw/openclaw#69778: [Bug]: Gateway restart resurrects ancient interrupted CLI subagent tasks regardless of age — old prompts auto-execute

Source report: [records/openclaw-openclaw/items/69778.md](records/openclaw-openclaw/items/69778.md)

## Summary

Keep open. Current main still auto-recovers restart-aborted subagent sessions from `abortedLastRun` without any stale-age cutoff, and the closed wedge-loop fix only bounds rapid repeated recovery, not old prompt replay.

## Plan

Fix https://github.com/openclaw/openclaw/issues/69778 by adding an age-bound stale restart recovery guard for interrupted CLI/subagent work. Implement a conservative `agents.defaults.taskRecovery.maxAgeMinutes` config surface, defaulting around 120 minutes unless existing config conventions suggest a better value, and wire schema/types/help/docs consistently. In `recoverOrphanedSubagentSessions` and the task/session reconciliation path, skip replaying `abortedLastRun` child sessions older than the cutoff, mark the associated run/task lost with a clear reason, and log/count the skipped stale recoveries. Preserve the existing rapid re-wedge tombstone behavior from https://github.com/openclaw/openclaw/issues/74864 and do not broaden into side-effectful-tool policy. Add regression coverage that an old restart-aborted run is lost/skipped instead of calling the gateway, while fresh restart-aborted recovery still works. Update public docs and changelog if user-visible behavior/config changes.
