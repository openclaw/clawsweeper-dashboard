## Summary

Keep open. Current main still resumes stale restart-aborted subagent work solely from `abortedLastRun` plus the rapid-retry tombstone state, with no recovery-age cutoff or `agents.defaults.taskRecovery.maxAgeMinutes`; the closed related wedge-loop issue only fixed repeated rapid resurrection, not ancient prompt replay.

## Plan

Fix https://github.com/openclaw/openclaw/issues/69778 by adding an age-bound stale recovery cutoff for restart-aborted subagent/CLI task recovery. Keep the scope narrow: add `agents.defaults.taskRecovery.maxAgeMinutes` or the smallest equivalent config contract with a safe default around 120 minutes, skip automatic orphan recovery when the run/session is older than the cutoff, mark the corresponding run/task lost or otherwise terminal without replaying the original prompt, and log/report the skipped count. Preserve the rapid re-wedge tombstone behavior from https://github.com/openclaw/openclaw/issues/74864. Add regression coverage for an old `abortedLastRun: true` subagent run that no longer calls `agent`/`agent.wait`, update docs for subagent/task recovery semantics, and add a changelog entry. Do not bundle a separate side-effectful-tool policy unless a maintainer explicitly asks.
