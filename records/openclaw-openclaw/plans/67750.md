## Summary

Keep open. Current `main` still tracks completed compaction separately from the terminal embedded-run failure formatter, so a post-compaction timeout plus unavailable-fallback exhaustion can still collapse to the generic `/new` message when verbose details are off.

## Plan

Fix https://github.com/openclaw/openclaw/issues/67750 on current `main`. When an embedded run has completed auto-compaction and the retried run later fails with a `FallbackSummaryError` containing a timeout plus unavailable/exhausted fallback attempts under verbose-off external chat output, do not return the generic `/new` fallback. Add a narrow sanitized post-compaction failure formatter in `src/auto-reply/reply/agent-runner-execution.ts` that uses completed-compaction state plus structured fallback attempts to say compaction succeeded, the retried model call timed out, and fallbacks were unavailable/exhausted. Do not change fallback candidate policy, session reset policy, compaction retry counts, provider auth behavior, or unrelated timeout reset PR behavior. Add focused regression coverage in `src/auto-reply/reply/agent-runner-execution.test.ts` for completed compaction followed by timeout plus billing/auth/unavailable fallback exhaustion with `resolvedVerboseLevel: "off"`; assert the reply is specific and does not leak raw provider billing/account details. Add a `CHANGELOG.md` fix entry because this changes user-visible error text.
