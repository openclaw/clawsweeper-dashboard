## Summary

Keep open: current main still only clears and fresh-retries a persisted CLI session binding for `session_expired`, while real `timeout` and `unknown` CLI `FailoverError` paths can still rethrow from a resumed binding and leave the stale session in place. The two related contributor PRs are closed unmerged, so there is no active implementation candidate covering the remaining bug.

## Plan

Fix the stale CLI session recovery bug reported in https://github.com/openclaw/openclaw/issues/77089. On current main, a resumed CLI session with a persisted binding only clears and retries fresh for `FailoverError.reason === "session_expired"`; `timeout` and `unknown` leave the stale binding in place. Implement a narrow hook-aware repair: clear the persisted binding for any active resumed-session `FailoverError`; retry fresh exactly once only for recoverable reasons such as `session_expired`, `timeout`, and `unknown`; clear then rethrow nonrecoverable reasons such as auth, billing, and rate-limit without retrying. Preserve a single coherent `agent_end` lifecycle result, so do not add an outer successful retry after the CLI runner has already emitted a failed final hook event. Add or update focused regressions for timeout, unknown, and nonrecoverable stale-binding clearing. Use the existing `clearCliSessionInStore` and session binding helpers; do not add config knobs, provider policy, or broad session-store refactors. Add a changelog entry if the final branch is user-facing and maintainer-owned.
