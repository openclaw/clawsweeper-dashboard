## Summary

Keep open: current main still passes configured ACP spawn model options into the acpx runtime before the harness has proven model-control support, and acpx 0.10.0 throws the reporter's exact unsupported-model error when no ACP model list is advertised.

## Plan

Fix https://github.com/openclaw/openclaw/issues/95869. Current `sessions_spawn(runtime: "acp")` resolves configured/default subagent or runtime=acp agent models in `src/agents/acp-spawn.ts` and passes them through `extensions/acpx/src/runtime.ts` to acpx `sessionOptions.model`; acpx 0.10.0 then throws when the harness advertises no ACP `models`. Implement a narrow fix so automatic/default ACP model choices do not make no-model-capability harnesses fail; the ACP harness should keep its own configured model in that case. Preserve explicit `sessions_spawn.model` behavior: if the caller explicitly asks for a model and the ACP backend cannot apply it, keep a clear unsupported-model error. Preserve Codex/Claude adapter-specific startup model handling and capability-supported generic model selection. Add regression coverage for a non-Codex ACP agent with no advertised models and an automatic configured model, plus explicit model failure coverage if the existing tests do not already prove it. Do not add a new config option, do not weaken ACP sandbox/policy checks, and do not edit CHANGELOG.md.
