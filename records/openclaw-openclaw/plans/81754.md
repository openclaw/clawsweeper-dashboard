## Summary

This is a valid, narrow performance feature request rather than a stale or already-implemented bug: current main serves Control UI/WebChat static assets as raw buffers and has no gateway-side `Accept-Encoding` negotiation.

## Plan

Implement gateway compression for Control UI/WebChat static assets requested in https://github.com/openclaw/openclaw/issues/81754. Keep the change scoped to `src/gateway/control-ui.ts` and tests. Negotiate `Accept-Encoding` for compressible text-like static assets and `index.html` using Node built-ins only; prefer brotli when accepted, otherwise gzip, and leave already-compressed/binary assets such as png/jpg/webp/ico uncompressed. Set `Content-Encoding` and `Vary: Accept-Encoding` on compressed GET responses, preserve existing `Content-Type`, `Cache-Control`, CSP, auth, route classification, path-safety, symlink/hardlink behavior, and make HEAD behavior coherent without writing a body. Avoid adding a config option unless a maintainer asks for one. Add focused regression coverage in `src/gateway/control-ui.http.test.ts` for gzip and brotli negotiation, no compression without `Accept-Encoding`, no compression for binary assets, and HEAD/no-body behavior. Add a single-line changelog entry under the active version because this is user-visible gateway performance behavior.
