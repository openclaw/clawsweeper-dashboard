## Summary

Keep open. Current main preserves provider `stopReason: "length"` and propagates it in metadata, but embedded `mode:"run"` sessions still have no configurable fail/continue policy and can reach the normal terminal path with partial visible output.

## Plan

Implement a conservative Phase 1 fix for https://github.com/openclaw/openclaw/issues/63188. Scope: embedded `mode:"run"` / subagent sessions only. Add a documented config surface for length truncation that preserves current default behavior, but supports an explicit `fail` mode that turns terminal `stopReason:"length"` attempts into an error payload/failure signal for the caller instead of normal success with partial output. Add config types, strict zod schema, help/docs, and focused regression coverage. Do not implement automatic `continue`/replay in this PR; stop and ask for maintainer direction if the fix requires replaying model output, repeating tool calls, or changing main interactive-session truncation policy already tracked by https://github.com/openclaw/openclaw/issues/63210. Include a changelog entry because this is user-visible runtime behavior.
