## Summary

Keep open. The PR is the right direction, but it is not merge-ready because the shared resolver can still write zsh/fish completion to a HOME-backed profile when an env-scoped profile should be created, and it regresses bash no-profile installs from the prior .bash_profile fallback to .bashrc.

## Plan

Repair PR https://github.com/openclaw/openclaw/pull/81298 on the PR branch or a narrow replacement branch. Keep the shared resolver in `src/cli/completion-runtime.ts`, but change fallback semantics so zsh with `ZDOTDIR` set and fish with `XDG_CONFIG_HOME` set choose the env-scoped profile path even when that file does not exist yet; do not fall back to an existing HOME default in those env-var cases. Preserve bash install behavior: when `.bashrc` is absent, choose `.bash_profile` even if neither profile exists, while still letting status/slow-pattern checks find an existing `.bash_profile`. Add focused tests in `src/cli/completion-runtime.profile-path.test.ts` for ZDOTDIR set + missing `$ZDOTDIR/.zshrc` + existing HOME `.zshrc`, XDG_CONFIG_HOME set + missing XDG fish config + existing HOME fish config, and bash with neither `.bashrc` nor `.bash_profile` selecting/creating `.bash_profile`. Do not change completion script generation, cache location, PowerShell support, dependencies, docs, or changelog in the contributor repair pass.
