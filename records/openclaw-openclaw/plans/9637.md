# Coding Plan for openclaw/openclaw#9637: Add accessibility config option to disable emojis and unicode symbols in TUI

Source report: [records/openclaw-openclaw/items/9637.md](records/openclaw-openclaw/items/9637.md)

## Summary

Keep open: current main still lacks a documented TUI plain-text accessibility mode, and the config, schema, docs, tool summaries, and TUI render paths still emit emoji or Unicode glyphs without an opt-out.

## Plan

Implement the TUI plain-text accessibility mode requested in https://github.com/openclaw/openclaw/issues/9637. Add a default-off `ui.accessibility.plainText` or equivalent config field to OpenClawConfig, strict zod validation, generated config help/labels/schema, and the public configuration docs; add a user-facing changelog entry. Load the flag once at TUI startup and thread it through TUI components rather than reading config during render. Apply ASCII alternatives consistently to tool summaries, ToolExecutionComponent headers/previews, searchable/filterable selectors, settings cursors, waiting/status text, and other TUI-owned emoji/Unicode glyphs. Check the pi-tui selection/theming API before adding wrappers or local overrides. Treat https://github.com/openclaw/openclaw/pull/9685 as prior art, but do not resurrect stale code blindly and do not change unrelated channel/plugin behavior.
