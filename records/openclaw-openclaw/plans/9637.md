## Summary

Keep open: current main still lacks a TUI plain-text accessibility mode, and config/schema/docs plus TUI/tool display paths still hard-code emoji or Unicode glyphs without a switch.

## Plan

Implement https://github.com/openclaw/openclaw/issues/9637 as a core TUI/config accessibility feature. Add a default-off `ui.accessibility.plainText` or equivalent documented key to `OpenClawConfig`, zod validation, generated labels/help, config docs, and any schema baselines generated by the repo scripts. Load the option once at TUI startup and thread it through existing TUI component/options seams instead of reading config during render. When enabled, replace decorative emoji and Unicode glyphs in tool summaries, `ToolExecutionComponent`, chat log/tool rendering, searchable/filterable selectors, settings cursor/theme, waiting/status text, separators, bullets, ellipses, and any pi-tui selection wrapper seam with deterministic ASCII alternatives. Inspect the pi-tui SelectList API/source before changing dependency-backed selection behavior. Use https://github.com/openclaw/openclaw/pull/9685 only as prior art; do not assume its branch is current. Add focused regression tests for config acceptance, docs/schema metadata, tool display formatting, selector/theme rendering, waiting/status formatting, and TUI component propagation. Add a user-facing `CHANGELOG.md` entry and credit human reporters/contributors as appropriate, such as @robin24 and any reused prior-PR work from @mudrii. Do not add startup migrations or broad UI rewrites beyond this default-off accessibility mode.
