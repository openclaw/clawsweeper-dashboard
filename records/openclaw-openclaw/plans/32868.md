## Plan

Fix https://github.com/openclaw/openclaw/issues/32868. Current main still queues same-channel block replies through `dispatcher.sendBlockReply()` without waiting for actual channel delivery before tool execution continues. Implement a backward-compatible awaitable completion for accepted same-channel block replies, or otherwise wait for the accepted block/dispatcher drain at the existing `onBlockReply`/`onBlockReplyFlush` boundary.
