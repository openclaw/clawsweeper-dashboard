## Plan

Fix https://github.com/openclaw/openclaw/issues/32868 with a narrow same-channel block reply delivery barrier. Current main awaits routed/cross-channel `sendPayloadAsync`, but same-channel `onBlockReply` only calls `dispatcher.sendBlockReply(normalizedPayload)` while `sendBlockReply` returns boolean after enqueueing async delivery. Update the dispatcher contract or same-channel path so an accepted block reply can be awaited through actual `options.deliver()` completion before tool execution proceeds. Preserve dropped/silent payload semantics, ordering, humanDelay behavior, abort/cancellation behavior, and existing public plugin API expectations; do not implement a broad followup-runner idle wait. Add focused regression coverage for same-channel delivery ordering before tool start, keep cross-channel behavior intact, and add a user-visible changelog fix entry if the repair lane owns changelog updates.
