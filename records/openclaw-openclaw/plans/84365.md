## Summary

Keep open: current main has the managed Codex startup path and legacy Codex CLI migration, but the malformed Windows `appServer.command` path is still source-reproducible and no doctor/runtime diagnostic catches the combined `node C:\...\codex.js` shape before startup fails.

## Plan

Fix the Codex Windows startup diagnostic gap reported in https://github.com/openclaw/openclaw/issues/84365. Add targeted validation and/or doctor warnings for `plugins.entries.codex.config.appServer.command` and `OPENCLAW_CODEX_APP_SERVER_BIN` values that combine an executable and script path such as `node C:\\...\\codex.js`, directing users to remove the override, set the executable only, or put arguments in `appServer.args`. Keep managed Codex startup as the default, do not add broad shell splitting or shell fallback behavior, and preserve current `appServer.args` semantics. Also review the managed binary missing message so it does not encourage a misleading combined command on Windows. Add focused regression coverage and a `CHANGELOG.md` fix entry if behavior or user-facing diagnostics change.
