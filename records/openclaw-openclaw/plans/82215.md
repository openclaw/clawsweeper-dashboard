# Coding Plan for openclaw/openclaw#82215: doctor misclassifies junction-backed source checkout as non-git install

Source report: [records/openclaw-openclaw/items/82215.md](records/openclaw-openclaw/items/82215.md)

## Summary

Keep open: current main still uses raw path equality in doctor’s pre-update git checkout detector, so a linked package root can be classified as `not-git` even when it resolves to the same checkout as Git’s top-level path.

## Plan

Fix https://github.com/openclaw/openclaw/issues/82215. In `src/commands/doctor-update.ts`, make doctor’s git checkout detector treat the package root and `git rev-parse --show-toplevel` as the same checkout when their canonical filesystem paths match, including Windows junction/symlink installs. Preserve the existing `unknown` behavior for git command failures and the package-manager guidance for real non-git installs. Add focused regression coverage, likely in a new `src/commands/doctor-update.test.ts`, for a linked package root whose git top-level resolves to a different raw string but the same real path, plus a real non-git package root case. Prefer reusing or mirroring the existing realpath comparison pattern from update code; do not broaden core update policy or change package-manager install detection. Add a concise `CHANGELOG.md` fix entry because this is user-visible CLI behavior. Validate with `node scripts/run-vitest.mjs src/commands/doctor-update.test.ts`, `pnpm oxlint src/commands/doctor-update.ts src/commands/doctor-update.test.ts`, and `git diff --check`.
