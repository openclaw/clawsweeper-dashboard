Current main still has only implicit Web UI chat run state and coarse session liveness; the composer toolbar does not render the requested compact `In progress` / `Interrupted` / `Done` status chip, and the broader related activity/status issues remain open rather than superseding this narrower UI request.

## Plan

Implement https://github.com/openclaw/openclaw/issues/71630 as a narrow Web UI composer status chip. Add a compact, accessible indicator beside the existing attach/Talk controls in `ui/src/ui/views/chat.ts` with stable layout in `ui/src/styles/chat/layout.css`; support at least in-progress, interrupted, and done/idle-ready states using explicit `chatSending`, `chatRunId`, terminal chat events, abort/error, reconnect cleanup, and existing `hasActiveRun`/session status signals where already available. Add focused tests for render state, terminal/abort/reconnect transitions, responsive layout, and no transcript pollution. Stop and return for maintainer review if the solution requires protocol changes, a new config option, or product taxonomy beyond the three requested states.
