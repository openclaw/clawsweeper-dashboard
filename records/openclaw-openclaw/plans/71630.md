# Coding Plan for openclaw/openclaw#71630: [Feature]: Add a compact runtime status indicator beside the Web UI composer actions

Source report: [records/openclaw-openclaw/items/71630.md](records/openclaw-openclaw/items/71630.md)

## Summary

Keep open. Current main still lacks the requested compact In progress/Interrupted/Done chip beside the Control UI composer actions; it has local run state, action buttons, Talk-only composer status text, compaction/fallback notices, and silent reconnect cleanup instead. A prior unmerged prototype in https://github.com/openclaw/openclaw/pull/69571 covered this idea, but it was closed as split/superseded and the UI run-state slice is not present on main.

## Plan

Implement https://github.com/openclaw/openclaw/issues/71630 as a focused Control UI feature. Add a compact runtime status chip in the chat composer toolbar near the attach/Talk controls for In progress, Interrupted, and Done; keep it out of the transcript, session-scoped, accessible via aria-label/title, stable on desktop/mobile, and non-flickery for fast turns. Drive the first version from existing local chat lifecycle signals: send/run start, final/error/abort handling, explicit abort/cancel, disconnect/reconnect/seq-gap/orphaned run cleanup, and current session active-run rows where already available. Do not add the broader sessions.activity gateway API in this PR; if that becomes necessary, stop and point back to https://github.com/openclaw/openclaw/issues/39127. You may use https://github.com/openclaw/openclaw/pull/69571 as reference for the UI run-state idea, but do not bring over its unrelated bootstrap/auth/setup/runtime/plugin changes. Add/update English i18n source and regenerate locale outputs, update focused tests for view rendering, run controls, app chat/gateway/controller lifecycle, add CSS for stable desktop/mobile layout, update docs/web/control-ui.md if user-visible behavior changes, and add a changelog entry because this is user-facing.
