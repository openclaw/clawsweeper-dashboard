# Coding Plan for openclaw/openclaw#95135: Nix-mode write guard blocks auth-profile and session-state writes that never touch openclaw.json

## Summary

Keep open: current main and v2026.6.8 still fail Nix-mode doctor repair at command entry before knowing whether the repair touches openclaw.json. Auth login is partly improved, but provider config patches and paste-token/API-key paths still couple credential writes to config writes, so the central state-versus-config boundary is not fully solved.

## Plan

Fix https://github.com/openclaw/openclaw/issues/95135 by scoping Nix-mode refusal to actual openclaw.json/config mutations, not all mutable state writes. Preserve the low-level config write guard in config/mutate and config/io, and keep setup/onboarding/plugin lifecycle/update/gateway-token/config writes blocked in Nix mode. Allow auth-profile writes and session-state-only doctor repairs to complete when they do not write openclaw.json; for provider auth results that include configPatch/default-model changes, persist the credential when safe and clearly report that the config/default write was skipped or refused by Nix mode. Add focused tests for doctor state-only repair under OPENCLAW_NIX_MODE=1, blocked config/token repair under OPENCLAW_NIX_MODE=1, and models auth login/paste-token/paste-api-key behavior. Update docs only where behavior changes; do not edit CHANGELOG.md.
