## Summary

Keep open. Current main still has no one-shot `openclaw agent` SIGTERM/SIGINT bridge into the Gateway request, the accepted Gateway run, or the embedded fallback path, and the public docs still do not document `timeout -k` as the cron-supervisor backstop.

## Plan

Fix https://github.com/openclaw/openclaw/issues/71710. Add a bounded shutdown path for one-shot `openclaw agent` so SIGTERM/SIGINT from cron/coreutils `timeout` cancels in-flight Gateway and embedded agent work instead of leaving the CLI alive. Scope the fix to `openclaw agent`: add a CLI-owned AbortController/signal bridge, let `callGateway` accept caller cancellation and expose or callback the accepted `runId`, use the existing Gateway `agent` abort registration (`chat.abort`/`sessions.abort` compatible seam) once the run is known, wire the same signal into embedded fallback/local `agentCommand`, and ensure shutdown exits within a bounded grace window without breaking existing session-lock cleanup handlers. Also document `timeout -k <delay> <duration> openclaw agent ...` as the recommended external cron wrapper backstop until supervisors can escalate. Add focused regression coverage; do not broaden exec-tool supervisor behavior beyond what this issue needs. Add a user-visible changelog entry.
