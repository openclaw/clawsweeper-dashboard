# Coding Plan for openclaw/openclaw#71710: [Bug]: openclaw-agent ignores SIGTERM under cron, accumulates hung process chains and exhausts host RAM/swap

Source report: [records/openclaw-openclaw/items/71710.md](records/openclaw-openclaw/items/71710.md)

## Summary

Keep open. Current main still lacks a one-shot `openclaw agent` SIGTERM/SIGINT bridge that can cancel the Gateway request, abort the accepted Gateway run, or abort the embedded/local fallback path; the newer Codex no-signal hang data point is related but does not resolve the external-supervisor SIGTERM case.

## Plan

Fix the external-supervisor SIGTERM/SIGINT leak for one-shot `openclaw agent` runs. Add a bounded shutdown path in the CLI command layer that creates a top-level AbortController, registers SIGTERM/SIGINT handlers, forwards cancellation into `callGateway` and embedded/local `agentCommand` runs, and, once the Gateway accepted run id is known, invokes the existing Gateway abort surface for that run. Extend `callGateway`/`GatewayClient.request` only as needed to support caller cancellation and an accepted-run callback; keep protocol changes additive if any are needed. Preserve the hard liveness fallback: do not replace SIGKILL escalation with SIGTERM-only behavior. Add focused regression coverage for Gateway expect-final cancellation, accepted-run abort, local/embedded abortSignal forwarding, and session-lock cleanup interaction. Update `docs/cli/agent.md` and `docs/automation/cron-jobs.md` to mention `timeout -k <delay> <duration>` or equivalent supervisor escalation as an external backstop until/when applicable. Add a changelog entry because this is user-visible reliability behavior. Do not broaden this into the separate Codex raw-completion hang tracked by https://github.com/openclaw/openclaw/pull/82403 or the general exec supervisor escalation tracked by https://github.com/openclaw/openclaw/issues/66399.
