## Summary

Keep open. Current main still lacks a one-shot `openclaw agent` SIGTERM/SIGINT bridge into the Gateway request, accepted agent run, or embedded fallback path, and the docs only cover Gateway-managed cron timeouts rather than external `timeout` wrappers.

## Plan

Fix https://github.com/openclaw/openclaw/issues/71710 on current main. Implement a bounded one-shot `openclaw agent` SIGTERM/SIGINT shutdown path for external supervisors such as cron plus GNU `timeout`: when the CLI is waiting on a Gateway `agent` request, capture the accepted run id, forward process termination into the existing Gateway abort surface, stop waiting within a small bounded grace window, and make the embedded fallback/`--local` path honor the same caller abort signal. Prefer extending `callGateway`/`GatewayClient.request` with caller-owned cancellation and an accepted-frame callback instead of adding scattered polling. Document `timeout -k <delay> <duration>` as an external-supervisor backstop in the agent/cron docs. Do not broaden this into the exec supervisor SIGTERM-to-SIGKILL work tracked by https://github.com/openclaw/openclaw/issues/66399, and do not fold in Codex raw-completion natural-exit hangs from https://github.com/openclaw/openclaw/issues/82343 / https://github.com/openclaw/openclaw/pull/82403. Add focused regression coverage for Gateway wait cancellation, accepted-run abort, embedded fallback signal wiring, and session-lock cleanup interaction. Add a changelog fix entry because this is user-visible reliability behavior.
