# Coding Plan for openclaw/openclaw#71710: [Bug]: openclaw-agent ignores SIGTERM under cron, accumulates hung process chains and exhausts host RAM/swap

Source report: [records/openclaw-openclaw/items/71710.md](records/openclaw-openclaw/items/71710.md)

## Summary

Keep open. Current main still leaves the one-shot `openclaw agent` CLI without a SIGTERM/SIGINT bridge into the pending Gateway request or embedded fallback path; Gateway-side abort plumbing exists, but the CLI cannot currently drive it when an external supervisor sends SIGTERM.

## Plan

Fix the one-shot `openclaw agent` SIGTERM/SIGINT shutdown bug reported in https://github.com/openclaw/openclaw/issues/71710. Keep the boundary narrow: add a bounded shutdown path for CLI `openclaw agent` runs that forwards SIGTERM/SIGINT into both Gateway-backed and embedded fallback execution, lets the CLI abort the accepted Gateway run once the run id is known, and exits within a small grace window. Preserve existing Gateway `expectFinal`, timeout, auth/scope, `chat.abort` authorization, and embedded fallback behavior. Add focused regression coverage for a pending Gateway `agent` call receiving SIGTERM/SIGINT, accepted-frame/run-id capture, abort RPC or equivalent cleanup, and embedded fallback receiving an abort signal. Update public docs to warn external cron/coreutils wrappers to use `timeout -k <delay> <duration>` or an equivalent supervisor escalation backstop. Add a changelog entry because this is user-visible reliability behavior.
