## Summary

Keep open. Current main still lacks a one-shot `openclaw agent` SIGTERM/SIGINT bridge into the pending Gateway request, the accepted Gateway run, or the embedded fallback path, while the cron docs only cover OpenClaw-managed scheduled tasks rather than external `timeout` wrappers.

## Plan

Fix https://github.com/openclaw/openclaw/issues/71710. Current main leaves `openclaw agent` one-shot CLI processes launched by external cron wrappers alive after SIGTERM because `agent-via-gateway.ts` does not bridge process signals into `callGateway`, the accepted Gateway `agent` run, or embedded fallback. Implement a bounded SIGTERM/SIGINT shutdown path for one-shot `openclaw agent`: add a caller-owned AbortSignal/cancel path through `src/gateway/call.ts` and `src/gateway/client.ts` as needed, expose/remember the accepted run id while still waiting for the final response, and on process signal abort the known Gateway run through the existing `chat.abort`/`sessions.abort` seam or an equivalent existing contract. Ensure embedded/local fallback receives the same abort signal and exits within a small bounded grace window. Add focused regression coverage for the CLI Gateway path, call wrapper/client cancellation, accepted-run abort wiring, embedded fallback signal propagation, and session-lock signal cleanup interactions. Update `docs/cli/agent.md` and the scheduled-task docs with `timeout -k <delay>` or equivalent external supervisor guidance as a backstop, and add a changelog entry if the fix is user-visible. Do not broaden this into the exec supervisor work tracked by https://github.com/openclaw/openclaw/issues/66399 except for shared helper reuse if it stays small.
