# Coding Plan for openclaw/openclaw#29100: [Feature]: Cost metadata support for auto-discovered Amazon Bedrock models

Source report: [records/openclaw-openclaw/items/29100.md](records/openclaw-openclaw/items/29100.md)

## Summary

Keep open. Current main and the latest tagged release still lack a supported cost overlay for auto-discovered Amazon Bedrock models, the direct implementation PR was closed unmerged, and the remaining display behavior for `aws-sdk` cost output is separately tracked.

## Plan

Implement https://github.com/openclaw/openclaw/issues/29100 as a narrow Amazon Bedrock plugin feature. Add a plugin-owned `plugins.entries.amazon-bedrock.config.discovery.costs` map keyed by Bedrock model ID with optional nonnegative `input`, `output`, `cacheRead`, and `cacheWrite` values in USD per 1M tokens. Do not add a top-level `models.costs` overlay without maintainer approval. Apply the overlay when converting discovered foundation models and inherited inference profiles, keep unspecified cost fields at zero, and include a deterministic normalized cost fingerprint in the discovery cache key. Preserve legacy `models.bedrockDiscovery` migration behavior by moving or merging any legacy `costs` object into the plugin-owned discovery config if present. Update Bedrock docs and focused tests. Keep visible `/status` and `/usage` auth-display changes out of scope unless explicitly coordinating with https://github.com/openclaw/openclaw/issues/53286.
