# Coding Plan for openclaw/openclaw#29100: [Feature]: Cost metadata support for auto-discovered Amazon Bedrock models

Source report: [records/openclaw-openclaw/items/29100.md](records/openclaw-openclaw/items/29100.md)

## Summary

Keep open. Current main still lacks a supported cost overlay for auto-discovered Bedrock models: the plugin discovery schema and shared type omit pricing, discovery assigns zero-cost metadata, and the direct implementation PR was closed unmerged.

## Plan

Implement a focused Bedrock discovery cost metadata overlay for https://github.com/openclaw/openclaw/issues/29100. Add `costs` under `plugins.entries.amazon-bedrock.config.discovery`, keyed by discovered model id, using the existing model cost shape (`input`, `output`, `cacheRead`, `cacheWrite`) with nonnegative optional fields defaulting to zero. Apply the overlay to discovered foundation models and inference profiles, include a deterministic normalized cost-map component in the discovery cache key, and preserve legacy `models.bedrockDiscovery` migration behavior. Update the Amazon Bedrock plugin schema/ui hints, `BedrockDiscoveryConfig`, docs, and focused tests. Do not add a global `models.costs` overlay unless a maintainer explicitly asks, and do not broaden OAuth/token cost-display policy; if status or usage display code is touched, coordinate with https://github.com/openclaw/openclaw/issues/53286 and add the relevant display tests. Add a user-facing changelog entry.
