# Coding Plan for openclaw/openclaw#29100: [Feature]: Cost metadata support for auto-discovered Amazon Bedrock models

Source report: [records/openclaw-openclaw/items/29100.md](records/openclaw-openclaw/items/29100.md)

## Summary

Keep open. Current main and the latest release still lack a supported cost overlay for auto-discovered Amazon Bedrock models; the direct implementation PR was closed unmerged, and the remaining `aws-sdk` cost-display behavior is tracked separately.

## Plan

Fix https://github.com/openclaw/openclaw/issues/29100 by adding a narrow Amazon Bedrock discovery cost overlay. Add `plugins.entries.amazon-bedrock.config.discovery.costs` keyed by exact model ID with optional nonnegative `input`, `output`, `cacheRead`, and `cacheWrite` values in USD per 1M tokens; keep `models.bedrockDiscovery` only as the legacy compatibility path that migrates into the plugin-owned config. Apply the overlay when building discovered foundation models and inference profiles, with exact profile ID overrides taking precedence and base-model inherited costs preserved where appropriate. Include a deterministic normalized costs component in the discovery cache key. Update `extensions/amazon-bedrock/openclaw.plugin.json`, `src/config/types.models.ts`, discovery/config-compat code, focused tests, and `docs/providers/bedrock.md`. Do not add a top-level `models.costs` map, do not change unrelated provider discovery behavior, and keep visible `aws-sdk` cost display policy coordinated with https://github.com/openclaw/openclaw/issues/53286 rather than mixing that change into this PR.
