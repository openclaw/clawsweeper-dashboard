# Coding Plan for openclaw/openclaw#29100: [Feature]: Cost metadata support for auto-discovered Amazon Bedrock models

Source report: [records/openclaw-openclaw/items/29100.md](records/openclaw-openclaw/items/29100.md)

## Summary

Keep open. Current main and v2026.5.12 still lack a supported Bedrock discovery cost overlay; the related implementation pull request was closed unmerged, and the separate aws-sdk cost display issue tracks a different remaining gate.

## Plan

Implement https://github.com/openclaw/openclaw/issues/29100 as a focused Amazon Bedrock plugin-owned config feature. Add `plugins.entries.amazon-bedrock.config.discovery.costs`, keyed by discovered model ID, with values using OpenClaw's documented USD per 1M token fields (`input`, `output`, `cacheRead`, `cacheWrite`, and only include tiered pricing if it already fits the local model cost contract). Update the Bedrock plugin manifest schema/ui hints, shared BedrockDiscoveryConfig type, discovery runtime, and tests so foundation models receive configured cost metadata and inference profiles either match an explicit profile ID cost or inherit an overridden base model cost. Include the costs map in the discovery cache key with deterministic ordering. Preserve legacy compatibility behavior but do not add a new top-level `models.costs` map and do not attempt to solve the separate aws-sdk display-policy issue at https://github.com/openclaw/openclaw/issues/53286. Update https://docs.openclaw.ai/providers/bedrock source docs and add a single changelog entry if the feature lands. Keep the change scoped to Bedrock discovery metadata and avoid broad provider/pricing refactors.
