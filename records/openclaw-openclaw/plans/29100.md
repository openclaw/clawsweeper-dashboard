# Coding Plan for openclaw/openclaw#29100: [Feature]: Cost metadata support for auto-discovered Amazon Bedrock models

Source report: [records/openclaw-openclaw/items/29100.md](records/openclaw-openclaw/items/29100.md)

## Summary

Keep open. Current main still lacks any supported cost overlay for auto-discovered Amazon Bedrock models: the plugin-owned discovery schema and shared discovery type omit pricing, discovery still assigns zero costs to discovered models, and the only direct implementation PR was closed unmerged; the related aws-sdk display issue tracks a separate remaining gate.

## Plan

Implement https://github.com/openclaw/openclaw/issues/29100 as a narrow Amazon Bedrock plugin feature. Add an optional `costs` map under `plugins.entries.amazon-bedrock.config.discovery`, keyed by Bedrock model id, with `input`, `output`, `cacheRead`, and `cacheWrite` USD-per-1M-token fields. Preserve existing behavior when the map is absent, validate nonnegative numbers in the plugin manifest/schema surface, update `BedrockDiscoveryConfig`/SDK exports, migrate legacy `models.bedrockDiscovery.costs` into the plugin-owned discovery config if present, apply overrides to discovered foundation models and inference profiles, and include a deterministic overlay fingerprint in the discovery cache key. Keep auth, model routing, and provider selection unchanged; do not add a global `models.costs` overlay unless a maintainer explicitly asks. Update Bedrock/token-cost docs and CHANGELOG.md. Coordinate with, but do not silently absorb, the separate aws-sdk display-gate work tracked by https://github.com/openclaw/openclaw/issues/53286.
