# Coding Plan for openclaw/openclaw#82002: Telegram group replies to bot messages can lose reply context under group allowlist

Source report: [records/openclaw-openclaw/items/82002.md](records/openclaw-openclaw/items/82002.md)

## Summary

Keep open: current main still drops Telegram bot-authored reply targets when `contextVisibility: "allowlist"` is active and the bot id is not in the group sender allowlist, even though Telegram reply metadata and bot-reply implicit mentions are otherwise supported.

## Plan

Fix https://github.com/openclaw/openclaw/issues/82002 in a focused Telegram PR. In `extensions/telegram/src/bot-message-context.session.ts`, treat reply/quote targets authored by the current Telegram bot (`primaryCtx.me.id`) as allowed supplemental quote context when evaluating group `contextVisibility: "allowlist"`; do not relax group sender authorization for triggering messages, other users, other bots, or forwarded origins. Add regression coverage in `extensions/telegram/src/bot.test.ts` for an allowed group with `contextVisibility: "allowlist"`, `requireMention: false`, an allowlisted human sender, and a `reply_to_message` from `ctx.me` whose text reaches `ReplyToBody`/reply-chain context. Preserve existing allowlist filtering for non-allowlisted human reply targets and forwarded origin redaction. Only add outbound Telegram message-cache persistence if a live Telegram repro proves Telegram returns insufficient parent text without it. Add a `CHANGELOG.md` fix entry because this is user-visible Telegram behavior. Validate with the targeted Telegram Vitest wrapper and, before merge, provide real Telegram proof showing a group reply to a bot-sent message reaches the agent with reply context under an allowlist configuration.
