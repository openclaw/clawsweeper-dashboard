## Plan

Fix https://github.com/openclaw/openclaw/issues/80963. In the OpenAI Codex OAuth wrapper, handle local mode with 127.0.0.1:1455 already occupied before the automatic browser opener runs. Add or reuse a local port probe for localhost:1455, but do not make the callback port configurable and do not change token exchange, refresh, remote/VPS behavior, TLS preflight behavior, or the fixed OpenAI redirect URI. On EADDRINUSE, explain that the callback port is in use, include `lsof -nP -iTCP:1455 -sTCP:LISTEN`, mention that `Invalid or expired OAuth state` can be caused by the wrong local process handling the callback, do not call `openUrl`, and prompt for manual redirect/code immediately. Preserve a visible/logged authorization URL for manual opening; this was the blocker in https://github.com/openclaw/openclaw/pull/80971. Mirror the behavior in the bundled OpenAI runtime copy or refactor only if the plugin SDK boundary remains valid. Add focused regression coverage for occupied-port behavior, no openUrl call, immediate manual prompt, diagnostic text, manual URL availability, normal free-port local flow, and remote/VPS skip. Add a changelog entry because this is user-visible auth UX.
