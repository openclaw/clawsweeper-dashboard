# Coding Plan for openclaw/openclaw#66512: [Feature] systemd sd_notify / Type=notify support for gateway readiness

Source report: [records/openclaw-openclaw/items/66512.md](records/openclaw-openclaw/items/66512.md)

## Summary

Keep open. Current main still has no native systemd readiness-notify contract: the generated gateway unit lacks `Type=notify`, the ready boundary only updates OpenClaw state/logging, and update/restart flows still fall back to OpenClaw polling.

## Plan

Implement systemd readiness notification for https://github.com/openclaw/openclaw/issues/66512. Add a best-effort Linux/systemd notifier that detects `NOTIFY_SOCKET` and sends `READY=1` after the existing post-attach gateway-ready boundary (`sidecars.ready` / `gateway ready`) without failing startup when not under systemd or when notification delivery fails. Update generated systemd units from `openclaw gateway install` to use `Type=notify` and explicit `NotifyAccess=main` when this version supports readiness notifications. Add focused tests for the unit renderer and the gateway ready notification boundary. Update public gateway/Linux docs and add a user-facing changelog entry. Do not implement periodic `WATCHDOG=1` pings or `WatchdogSec` in this PR; keep that separate for https://github.com/openclaw/openclaw/issues/11973 unless maintainers explicitly broaden scope.
