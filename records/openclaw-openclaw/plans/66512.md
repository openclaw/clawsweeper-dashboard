# Coding Plan for openclaw/openclaw#66512: [Feature] systemd sd_notify / Type=notify support for gateway readiness

Source report: [records/openclaw-openclaw/items/66512.md](records/openclaw-openclaw/items/66512.md)

## Summary

Keep open. Current main still lacks the native systemd readiness-notify contract requested here: the generated gateway unit has no `Type=notify` or `NotifyAccess=main`, the gateway ready boundary only updates OpenClaw state/logging, and restart/update paths still use OpenClaw polling rather than systemd readiness notification.

## Plan

Implement systemd sd_notify readiness support for the OpenClaw gateway. On Linux, when `NOTIFY_SOCKET` is present, send `READY=1` at the existing post-attach gateway-ready boundary after startup sidecars/plugins are ready and the WebSocket listener is accepting connections. Update generated systemd units to include `Type=notify` and `NotifyAccess=main`. Keep notify delivery best-effort and non-fatal when `NOTIFY_SOCKET` is absent or delivery fails. Do not add watchdog pings unless maintainers explicitly request that follow-up. Add focused tests for unit rendering, notify emission/no-op behavior, and existing restart/update polling compatibility; update user-facing gateway/VPS docs and changelog if the change is user-visible.
