# Coding Plan for openclaw/openclaw#24196: [Feature]: Allow max allowed file attachments on discord messages

Source report: [records/openclaw-openclaw/items/24196.md](records/openclaw-openclaw/items/24196.md)

## Summary

Keep open. Current main has multi-media payload plumbing, but Discord still converts normal sends to scalar `mediaUrl` uploads or sequential one-file sends, so native Discord multi-attachment delivery remains unimplemented.

## Plan

Implement native multi-file attachment sending for Discord message delivery. Preserve existing single `media`/`mediaUrl`/`path`/`filePath`/`filename` behavior, existing media access checks, `mediaMaxMb`, replyTo handling, silent sends, text chunking, forum/thread behavior, component file behavior, and voice-message special handling. Add an ordered multi-attachment path for normal Discord sends and payload sends: load each file through existing outbound media helpers, send up to the verified Discord per-message file cap in one `files` array, and split overflow into follow-up messages. Do not rewrite shared outbound fan-out for all channels unless a minimal additive seam is needed. Update focused Discord tests, docs/channels/discord.md, and CHANGELOG.md for the user-visible feature.
