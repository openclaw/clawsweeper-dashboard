# Coding Plan for openclaw/openclaw#24196: [Feature]: Allow max allowed file attachments on discord messages

Source report: [records/openclaw-openclaw/items/24196.md](records/openclaw-openclaw/items/24196.md)

## Summary

Keep open. Current main and the latest release still route Discord outbound uploads through scalar `mediaUrl`/`filename` fields and one-file sends, while shared `mediaUrls` delivery fans out into repeated media sends instead of one native Discord multi-file message.

## Plan

Implement native multi-file Discord outbound sending for https://github.com/openclaw/openclaw/issues/24196. Keep the change Discord-owned where possible: preserve existing single `media`/`path`/`filePath`/`filename` behavior, accept ordered multi-attachment input from the message/outbound payload paths, load every file through existing outbound media helpers and access checks, send up to Discord's verified 10-file limit in one `files` array, and split overflow into follow-up messages. Do not apply batching to `audioAsVoice` voice-message sends. Avoid changing other channels' media fan-out semantics unless a minimal additive shared seam is necessary. Add focused Discord/shared outbound tests, update `docs/channels/discord.md`, and add a changelog entry because this is user-visible.
