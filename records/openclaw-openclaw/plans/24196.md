# Coding Plan for openclaw/openclaw#24196: [Feature]: Allow max allowed file attachments on discord messages

Source report: [records/openclaw-openclaw/items/24196.md](records/openclaw-openclaw/items/24196.md)

## Summary

Keep open. Current main still accepts Discord uploads through scalar `mediaUrl`/`filename` paths and delivers multiple shared `mediaUrls` as repeated one-file sends, so the requested native Discord multi-file message behavior remains unimplemented.

## Plan

Implement native multi-file Discord message uploads for https://github.com/openclaw/openclaw/issues/24196. Preserve existing single-file `media`/`path`/`filePath`/`mediaUrl` and `filename` behavior. Add an ordered multi-attachment input path for the bundled Discord plugin, load each file through existing outbound media helpers and access checks, send up to the verified Discord per-message attachment limit in one `files` array, split overflow into follow-up messages, and keep text chunking, reply-to, silent sends, components/embeds, forum/thread behavior, voice-message handling, and `mediaMaxMb` semantics intact. Prefer a Discord-owned implementation; add only a minimal additive shared/plugin seam if the current outbound adapter cannot receive ordered batches. Update focused tests, Discord docs, and CHANGELOG.md for the user-facing feature. Do not change unrelated channel behavior or add broad cache layers.
