# Coding Plan for openclaw/openclaw#24196: [Feature]: Allow max allowed file attachments on discord messages

Source report: [records/openclaw-openclaw/items/24196.md](records/openclaw-openclaw/items/24196.md)

## Summary

Keep open. Current main still carries multiple media references through shared send payloads, but Discord delivery reduces them to one scalar `mediaUrl` or repeated one-file sends instead of one native multi-file Discord message.

## Plan

Implement native multi-file Discord message attachments for this issue. Preserve existing single `media`/`mediaUrl`/`path`/`filePath`/`filename` behavior, accept ordered multi-attachment input from shared `mediaUrls`/structured attachments where it already exists, load each file through the existing outbound media helpers and access checks, send batches up to the verified Discord per-message attachment cap in one `files` array, and split overflow into follow-up messages. Keep the fix Discord-owned unless a minimal additive plugin-facing seam is required; do not rewrite generic outbound fan-out behavior for other channels. Update focused Discord/shared outbound tests, docs, and changelog. Verify the Discord attachment cap from current Discord docs/API before hard-coding it, and preserve media size caps, reply-to behavior, text chunking, components/file blocks, and voice-message restrictions.
