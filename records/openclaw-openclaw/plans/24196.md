## Summary

Keep open. Current main still has scalar Discord upload paths and one-file media fan-out, so the requested native multi-file Discord message batching remains unimplemented.

## Plan

Fix https://github.com/openclaw/openclaw/issues/24196 by adding native multi-file attachment batching for ordinary Discord message sends. Preserve existing scalar `media`/`mediaUrl`/`path`/`filePath`/`filename` behavior, but add an ordered multi-attachment input path for Discord-owned sends and outbound `mediaUrls` payloads. Load each file through the existing outbound media helpers and access checks, apply the existing Discord media size cap per file, send up to Discord's verified 10-file per-message limit in one `files` array, and split overflow into follow-up messages. Keep voice-message `audioAsVoice` single-audio behavior unchanged. Avoid broad changes to non-Discord channel fan-out unless a minimal additive plugin-facing seam is required. Update focused Discord/shared outbound tests, `docs/channels/discord.md`, and add a user-facing changelog entry.
