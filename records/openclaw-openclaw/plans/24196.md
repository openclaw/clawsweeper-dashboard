# Coding Plan for openclaw/openclaw#24196: [Feature]: Allow max allowed file attachments on discord messages

Source report: [records/openclaw-openclaw/items/24196.md](records/openclaw-openclaw/items/24196.md)

## Summary

Keep open. Current main still exposes Discord message uploads through scalar `mediaUrl`/`filename` paths and fans shared `mediaUrls` out as repeated one-file sends, so the requested native multi-file Discord message behavior remains unimplemented.

## Plan

Implement native multi-file Discord message sending for https://github.com/openclaw/openclaw/issues/24196. In the bundled Discord plugin, accept ordered multi-attachment input while preserving existing single `media`/`path`/`filePath`/`mediaUrl`/`filename` behavior. Load every file through the existing outbound media helpers, access checks, and `mediaMaxMb`; send batches up to the verified Discord create-message attachment limit in one `files` array; split overflow into follow-up messages. Preserve reply-to consumption, silent sends, text chunking, forum/thread behavior, voice-message handling, component/file attachment behavior, and other channels' existing media fan-out. Prefer a Discord-owned implementation; add only a minimal additive shared seam if Discord cannot receive ordered batches from current outbound payloads. Add focused tests, update `docs/channels/discord.md`, and add a user-facing changelog entry.
