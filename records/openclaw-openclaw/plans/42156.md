# Coding Plan for openclaw/openclaw#42156: Telegram callback queries time out when agent turn is queued behind sequentialize

Source report: [records/openclaw-openclaw/items/42156.md](records/openclaw-openclaw/items/42156.md)

## Summary

Keep open. Current main and the latest release still answer ordinary Telegram callback queries inside the downstream callback handler after `sequentialize`, so same-chat/topic callbacks can miss Telegram's acknowledgement window; the related timing-fix PRs at https://github.com/openclaw/openclaw/pull/42181 and https://github.com/openclaw/openclaw/pull/42516 were closed unmerged, while https://github.com/openclaw/openclaw/pull/76975 addresses acknowledgement text, not this queueing path.

## Plan

Fix https://github.com/openclaw/openclaw/issues/42156. Move Telegram callback acknowledgement timing ahead of `sequentialize` for accepted callback updates only: use the existing `updateTracker.beginUpdate(ctx)` middleware in `extensions/telegram/src/bot-core.ts` or an equivalent non-consuming accepted-update seam, answer `ctx.callbackQuery` before awaiting `next()`, and keep action routing in `extensions/telegram/src/bot-handlers.runtime.ts` ordered downstream. Do not add a config option or broad UX policy change. Do not let skipped/duplicate/unauthorized updates drive extra outbound API calls; preserve `withTelegramApiErrorLogging({ runtime, operation: "answerCallbackQuery" })`-style sanitized logging and existing Telegram API throttling. Remove or avoid duplicate handler-level acknowledgements after the early ACK path is in place. Add focused regression coverage proving the ACK happens before a blocked same-chat/topic sequential handler can run, while ordinary callback routing still happens in order and approval callbacks keep their existing separate lane. Add a user-facing changelog entry. Related context: closed unmerged timing-fix attempts at https://github.com/openclaw/openclaw/pull/42181 and https://github.com/openclaw/openclaw/pull/42516, adjacent ack-text PR at https://github.com/openclaw/openclaw/pull/76975, and approval-lane precedent at https://github.com/openclaw/openclaw/pull/64979.
