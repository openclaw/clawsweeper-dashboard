## Summary

Keep open: current main and the latest release still acknowledge ordinary Telegram callback queries inside the downstream callback handler after `sequentialize`, so inline-button callbacks can still wait behind an active same-chat/topic turn. The merged approval-lane fix only covers approval payloads, and the open callback-toast work is a different UX/config request.

## Plan

Fix https://github.com/openclaw/openclaw/issues/42156 by moving Telegram callback-query acknowledgement timing before `sequentialize` while keeping callback action routing ordered. The observable bug is that ordinary inline-button callbacks still call `answerCallbackQuery` inside `bot.on("callback_query")`, downstream of `botRuntime.sequentialize(getTelegramSequentialKey)`, so same-chat/topic callbacks can time out behind an active agent turn. Add a pre-sequentialize acknowledgement path after accepted-update gating/dedupe, make the acknowledgement non-blocking or safely error-logged, remove or avoid duplicate handler-level acknowledgement, and preserve authorization, accepted-update skipping, callback payload routing, and normal chat/topic ordering for the actual action. Do not add a new config option, do not change approval callback lane behavior from https://github.com/openclaw/openclaw/pull/64979, and do not solve the toast-text request tracked by https://github.com/openclaw/openclaw/issues/76622 / https://github.com/openclaw/openclaw/pull/76975.
