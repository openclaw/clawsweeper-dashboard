# Coding Plan for openclaw/openclaw#71709: [Bug]: slug-generator HTTP 400 misclassified as profile-wide billing failure (5h cooldown), kills all agents on profile

## Summary

Keep open. Current main mitigates the default path by making LLM slugs opt-in, but the opt-in slug helper still uses the shared embedded-run auth failure path, so a helper-only Anthropic extra-usage billing error can still disable the whole auth profile.

## Plan

Fix https://github.com/openclaw/openclaw/issues/71709 narrowly. Current main already makes LLM slug generation opt-in, but when `hooks.internal.entries.session-memory.llmSlug: true` is enabled, `src/hooks/llm-slug-generator.ts` still calls `runEmbeddedPiAgent` on `sessionKey: "temp:slug-generator"` without `authProfileFailurePolicy: "local"`; Anthropic extra-usage text then classifies as `billing` and can mark the shared auth profile disabled for about five hours. Add `authProfileFailurePolicy: "local"` to the slug-helper embedded run, add/update `src/hooks/llm-slug-generator.test.ts` to assert the helper passes the local policy, and keep the existing global billing matcher behavior unchanged. Do not add a new slug model/config option or broaden the scope into https://github.com/openclaw/openclaw/issues/33962. Add a changelog entry because this changes user-visible outage behavior.
