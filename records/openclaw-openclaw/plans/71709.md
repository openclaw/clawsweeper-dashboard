## Summary

Keep open. Current main mitigates the default path by making LLM slugs opt-in, but with `llmSlug: true` the slug helper still uses the shared embedded-runner auth failure policy, and the reported Anthropic extra-usage text still resolves to `billing` and can persist a profile-wide cooldown.

## Plan

Fix https://github.com/openclaw/openclaw/issues/71709 by keeping LLM slug-generation auth failures local to the helper. In `src/hooks/llm-slug-generator.ts`, pass `authProfileFailurePolicy: "local"` into the `runEmbeddedPiAgent` call for `sessionKey: "temp:slug-generator"`. Add or update `src/hooks/llm-slug-generator.test.ts` to assert the helper passes local auth failure policy while preserving provider/model/timeout and cleanup behavior. If needed, add a focused auth-policy regression in `src/agents/pi-embedded-runner/run/auth-profile-failure-policy.test.ts` showing local policy suppresses billing. Do not globally weaken billing classification or shared cooldown behavior for normal user-facing runs. Add a `CHANGELOG.md` fix entry because this is user-visible outage prevention. Watch for overlap with open PRs that touch slug generation, especially https://github.com/openclaw/openclaw/pull/65134 and https://github.com/openclaw/openclaw/pull/64181.
