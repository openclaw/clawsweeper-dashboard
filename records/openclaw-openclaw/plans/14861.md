## Summary

Keep open. Current main still lacks the requested structured Gateway boot outcome summary: startup has point logs, readiness gating, startup trace timings, and plugin failure warnings, but no started/skipped/failed collector for sidecars or hooks; the Gmail watcher missing-account path is still silent during normal startup.

## Plan

Implement https://github.com/openclaw/openclaw/issues/14861 as a narrow Gateway startup observability enhancement. Add a structured, redacted startup outcome collector around the existing sidecar flow so normal startup emits one concise summary after sidecars settle, covering at least Gmail watcher, internal hooks, channels, plugin services, memory scheduling, recovery sidecars, and gateway_start hooks. Preserve current readiness/failure semantics: optional skipped or failed sidecars should be reported but not made fatal unless they already are. Do not log account identifiers, command arguments, local paths, tokens, provider secrets, or unredacted hook metadata. Check overlap with https://github.com/openclaw/openclaw/pull/61704 before editing Gmail watcher lifecycle wording, and do not try to solve this by restoring only the old decorative startup format from https://github.com/openclaw/openclaw/issues/73187. Add focused regression coverage and a user-facing CHANGELOG.md entry.
