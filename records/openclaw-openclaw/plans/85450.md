# Coding Plan for openclaw/openclaw#85450: [Bug] Structured card action callback filtered by requireMention in group chats

Source report: [records/openclaw-openclaw/items/85450.md](records/openclaw-openclaw/items/85450.md)

## Summary

Keep open: current `main` and the latest release still appear to drop Feishu structured card-action callbacks in `requireMention: true` groups because the synthetic callback message is parsed as not mentioning the bot and then reaches the existing group mention gate.

## Plan

Fix Feishu structured card-action callbacks being dropped in group chats with `requireMention: true`. Current main builds the synthetic card-action `FeishuMessageEvent` in `extensions/feishu/src/card-action.ts` without `mentions` or an activation fact, so `parseFeishuMessageEvent`/`checkBotMentioned` sets `mentionedBot=false` and `handleFeishuMessage` drops it at the group activation ingress with `message in group ... did not mention bot`. Implement the narrow fix in the Feishu plugin so card callbacks from bot-sent cards are treated as bot-addressed for activation, while preserving `groupPolicy`, group allowlists, sender allowlists, and existing DM behavior. Do not disable `requireMention` for the whole group, do not add a new config option, and do not bypass authorization gates. Add regression coverage for a structured group card action with `requireMention: true` reaching dispatch, plus a negative/guard case proving sender/group gates still apply where practical. Include a `CHANGELOG.md` Fixes entry if this becomes a maintainer repair PR.
