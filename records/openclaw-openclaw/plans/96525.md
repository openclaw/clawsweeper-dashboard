## Summary

Keep open: current `main` still has the cron-specific gap. The shared fallback loop and embedded classifier already support result-level failures, and the interactive/auto-reply callers pass that classifier, but isolated cron does not, so a configured fallback can still be skipped for returned embedded error results.

## Plan

Fix https://github.com/openclaw/openclaw/issues/96525. Current isolated cron `agentTurn` calls `runWithModelFallback` without `classifyResult` or `mergeExhaustedResult`, so returned embedded fallback-safe `incomplete_turn`, `empty`, or `reasoning-only` results are treated as success and the configured fallback model is not called. Wire the cron embedded path to reuse `classifyEmbeddedAgentRunResultForModelFallback` and `mergeEmbeddedAgentRunResultForModelFallbackExhaustion`, matching `src/agents/agent-command.ts` and `src/auto-reply/reply/agent-runner-execution.ts`; preserve CLI-provider behavior and do not add config, docs, or changelog unless the implementation changes the documented contract. Add focused cron regression coverage using the existing test harness or a narrow test that simulates a primary fallback-safe incomplete/reasoning-only result and a healthy fallback, proving cron attempts the fallback and returns/delivers the fallback result. Validate with `node scripts/run-vitest.mjs src/cron/isolated-agent/run.payload-fallbacks.test.ts` plus `node scripts/run-vitest.mjs src/agents/outcome-fallback-runtime-contract.test.ts`; add the nearest cron test file if the new regression lives elsewhere.
