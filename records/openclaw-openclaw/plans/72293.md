## Summary

Keeping this open: current main still has a source-level path where a lifecycle `end` carrying `stopReason: "aborted"` but no `aborted: true` is classified as successful by `agent.wait` and the subagent registry, so the parent completion path can still receive an `ok` outcome instead of a failure/timeout signal.

## Plan

Fix https://github.com/openclaw/openclaw/issues/72293. Current main can emit a lifecycle `end` event with `stopReason: "aborted"` and no `aborted: true`; `agent.wait` then returns `status: "ok"`, and the subagent registry completes the run with `outcome: { status: "ok" }`. Normalize `stopReason: "aborted"` into a non-success completion path for `agent.wait`, subagent registry completion, and parent announcement. Prefer existing timeout/killed/error semantics already used by `src/gateway/session-lifecycle-state.ts`; do not add an auto-retry policy or a new public status without maintainer direction.
