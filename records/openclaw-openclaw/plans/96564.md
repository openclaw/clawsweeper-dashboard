## Plan

Fix https://github.com/openclaw/openclaw/issues/96564. Current main's claude-cli backend uses live warm stdin, then `isCliBindingFlushed` probes the native Claude project transcript and `buildCliRunResult` clears the session binding when the probe returns false; for headless warm-stdin claude-cli the reporter shows that native transcript may never exist, so every successful turn can wipe continuity and disarm OpenClaw raw-transcript reseed. Implement a narrow fix in `src/agents/cli-runner.ts`, `src/agents/command/attempt-execution.helpers.ts`, and/or `src/agents/cli-runner/prepare.ts` so pure post-turn flush misses for this backend mode do not destroy usable session continuity. Prefer backend-aware validation against OpenClaw's just-persisted session transcript or a non-destructive miss that preserves the binding/reseed path; preserve true stale/phantom binding invalidation for cases with a real invalidation reason. Do not add config/env options and do not edit `CHANGELOG.md`. Add focused regression tests for a successful claude-cli turn with a false native-transcript probe and for the existing stale-binding/phantom-session behavior that must still clear when appropriate.
