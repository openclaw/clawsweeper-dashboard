# Coding Plan for openclaw/openclaw#59509: feat: support per-model location override for google-vertex provider

Source report: [records/openclaw-openclaw/items/59509.md](records/openclaw-openclaw/items/59509.md)

## Summary

Keep open. Current main and the latest shipped release still only resolve Google Vertex location from transport options or `GOOGLE_CLOUD_LOCATION`, while per-model `params.location` is read by config but not mapped into the Vertex transport path.

## Plan

Implement per-model Google Vertex location override for `google-vertex` without adding a top-level model schema field. Add a Google provider `extraParamsForTransport` hook that accepts a non-empty string `agents.defaults.models["google-vertex/<model>"].params.location`, maps it to transport `location`, and preserves the existing transport option and `GOOGLE_CLOUD_LOCATION` fallback behavior. Do not add preview-model smart defaults unless maintainers explicitly request them. Add focused coverage proving a `google-vertex` model with `params.location: "us-central1"` produces regional Vertex transport options while another model can still use the env/global fallback, update Google provider docs, and add a changelog entry if this is user-facing.
