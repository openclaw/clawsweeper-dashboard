Keep open. Current main still has the provider-level/env Vertex location path, while the existing per-model params seam is not mapped into Google Vertex transport options.

## Plan

Implement per-model Google Vertex location override for https://github.com/openclaw/openclaw/issues/59509. Use the existing `agents.defaults.models["provider/model"].params` and provider `extraParamsForTransport` seam: add a Google provider mapper that accepts a non-empty string `params.location` for `google-vertex` models and patches the transport `location`, while preserving current `options.location` and `GOOGLE_CLOUD_LOCATION` fallback behavior. Do not add automatic preview-model smart defaults and do not add a new top-level model field unless maintainers explicitly ask. Add regression coverage for mixed `google-vertex/gemini-3.1-pro-preview` using `global` and `google-vertex/gemini-2.5-pro` using `us-central1`, plus invalid/non-string location ignored or warned consistently with provider params patterns. Update the Google provider docs and add a user-facing changelog entry if this lands as a feature/fix.
