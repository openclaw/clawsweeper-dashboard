# Coding Plan for openclaw/openclaw#59509: feat: support per-model location override for google-vertex provider

Source report: [records/openclaw-openclaw/items/59509.md](records/openclaw-openclaw/items/59509.md)

## Summary

Keep open. Current main and the latest release still resolve Google Vertex location from transport options or `GOOGLE_CLOUD_LOCATION` only, while per-model params are collected but no Google-owned hook or wrapper forwards a per-model location into Vertex request options.

## Plan

Fix https://github.com/openclaw/openclaw/issues/59509 by adding explicit per-model Google Vertex location override support. Use the existing per-model `agents.defaults.models["google-vertex/<model>"].params.location` path or an equally bounded Google-plugin-owned params path; do not add automatic preview/global smart defaults, do not rework ADC/auth/onboarding, and avoid duplicating the broader work in https://github.com/openclaw/openclaw/pull/60860. Ensure a model with `params.location: "us-central1"` builds a regional Vertex URL while another model can still use `GOOGLE_CLOUD_LOCATION=global`. Add focused tests for the params-to-transport path and Vertex URL selection, update `docs/providers/google.md`, and add a changelog entry because this is user-visible provider behavior.
