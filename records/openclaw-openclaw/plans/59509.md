# Coding Plan for openclaw/openclaw#59509: feat: support per-model location override for google-vertex provider

Source report: [records/openclaw-openclaw/items/59509.md](records/openclaw-openclaw/items/59509.md)

## Summary

Keep open. Current main still resolves Google Vertex location from transport options or `GOOGLE_CLOUD_LOCATION`, while the existing per-model `params` seam is not mapped into Vertex transport options, so mixed global/regional Vertex models remain unsupported.

## Plan

Implement per-model Google Vertex location override for https://github.com/openclaw/openclaw/issues/59509. Use the existing per-model `agents.defaults.models["google-vertex/<model>"].params.location` seam and add a Google provider `extraParamsForTransport` mapper that copies a non-empty string `location` into transport options for `google-vertex` models. Preserve the existing provider/env fallback behavior, do not add preview-model smart defaults, and do not add a new top-level model schema key unless a maintainer explicitly asks. Add focused tests for params precedence and Vertex URL selection, update `docs/providers/google.md`, and add a user-facing changelog entry if this lands as provider config behavior.
