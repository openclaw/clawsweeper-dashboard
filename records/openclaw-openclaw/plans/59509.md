## Summary

Keep open. Current main still has a source-level gap: Google Vertex location can be supplied through transport options or `GOOGLE_CLOUD_LOCATION`, but per-model params are not mapped into those transport options.

## Plan

Implement Google Vertex per-model location override for this issue. Support `agents.defaults.models["google-vertex/<model>"].params.location` by adding a Google provider-owned `extraParamsForTransport` mapper that normalizes a non-empty string `location` into Vertex transport options, preserving the existing fallback to provider/transport options and `GOOGLE_CLOUD_LOCATION`. Do not add hard-coded preview-model smart defaults or a new top-level model schema key. Add focused tests for mixed `global` and regional URLs, update Google provider docs, and add a changelog entry if the change is user-visible.
