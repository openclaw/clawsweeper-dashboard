## Summary

Keep open. Current main still resolves Google Vertex location only from transport options or `GOOGLE_CLOUD_LOCATION`, while per-model `params` are accepted but not mapped into Vertex transport options; the requested mixed global/regional model setup remains a valid provider feature gap.

## Plan

Fix https://github.com/openclaw/openclaw/issues/59509 by adding an explicit Google Vertex per-model location override through the existing params seam. Implement support for `agents.defaults.models["google-vertex/<model>"].params.location` by adding a Google provider-owned `extraParamsForTransport` mapping that forwards a non-empty string `location` into Vertex transport options, preserving the existing fallback to transport `options.location` and `GOOGLE_CLOUD_LOCATION`. Do not add a top-level per-model `location` schema key, do not hard-code preview-model defaults, and do not change unrelated Google endpoint/auth behavior. Add focused tests for the extra-params transport patch and Vertex URL construction, update Google provider docs, and add a changelog entry if this lands as user-facing behavior.
