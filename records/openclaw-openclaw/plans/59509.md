## Summary

Keep open. Current main and v2026.5.18 still resolve Google Vertex location from transport options or `GOOGLE_CLOUD_LOCATION`; per-model `params.location` is accepted by config but not mapped into the Vertex transport request.

## Plan

Fix https://github.com/openclaw/openclaw/issues/59509 by adding explicit per-model Google Vertex location overrides through the existing provider params seam. Implement a Google provider `extraParamsForTransport` mapper that accepts a non-empty string `agents.defaults.models["google-vertex/<model>"].params.location` and patches transport `location` only for `google-vertex` models, preserving the existing transport option and `GOOGLE_CLOUD_LOCATION` fallback. Do not add preview-model smart defaults or a new top-level model schema field unless a maintainer asks. Coordinate with the broader open Google Vertex PRs and stop/report if an open branch already implements the same `params.location` behavior. Add focused tests for mixed global/regional Vertex models, update Google/provider docs, and add a changelog entry because this is user-visible config behavior.
