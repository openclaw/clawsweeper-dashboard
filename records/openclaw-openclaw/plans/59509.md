## Summary

Keep open. Current main and the latest release still have only transport-level Vertex `options.location` plus the generic provider transport-param seam; the Google plugin does not expose or forward a per-model Google Vertex location, so mixed global/regional model use under one project remains a valid feature gap.

## Plan

Implement per-model Google Vertex location override for https://github.com/openclaw/openclaw/issues/59509. Use the existing per-model `params` contract, specifically `agents.defaults.models["google-vertex/<model>"].params.location`; do not add a provider-specific top-level model key and do not add preview-model smart defaults in this PR. Add a Google provider hook that maps a trimmed nonempty `location` param into the Vertex transport patch so `resolveGoogleVertexLocation()` receives it, preserving the existing `GOOGLE_CLOUD_LOCATION` fallback. Add focused tests showing two google-vertex models can carry different locations through the extra-params/transport path, and update Google provider docs plus changelog for the user-visible config. Check current overlap with https://github.com/openclaw/openclaw/pull/60860 before editing and adapt if that PR lands first.
