## Summary

Keep open: current main and latest release still lack Telegram pinned-message reads, while the existing shared `list-pins` action and the closed unmerged implementation PRs show a clear plugin-owned path that has not landed.

## Plan

Implement Telegram pinned-message reads for https://github.com/openclaw/openclaw/issues/38580. Add a Telegram plugin-owned on-demand `list-pins` message action that uses grammY/Bot API `getChat` to read the current chat-level `pinned_message`, returns a `pins` array with zero or one normalized pinned message, and supports explicit targets plus current chat context consistently with existing message actions. Add `channels.telegram.actions.pins` as the top-level/per-account gate, update strict config schema/types/docs, and add a maintainer-owned changelog entry because this is user-visible. Keep automatic session-start pinned-context injection out of scope. Do not add core Telegram-specific policy outside the plugin boundary and do not bypass existing Telegram token/account/gateway-scope handling. Include redacted live Telegram proof or Telegram E2E proof showing `list-pins` returning the pinned message from a real bot/chat.
