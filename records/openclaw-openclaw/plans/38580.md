# Coding Plan for openclaw/openclaw#38580: Feature request: Read pinned Telegram messages via bot API

Source report: [records/openclaw-openclaw/items/38580.md](records/openclaw-openclaw/items/38580.md)

## Summary

Keep open: current main still lacks an on-demand Telegram pinned-message read surface, while the related implementation attempts were closed unmerged. The existing Telegram plugin can pin sent messages, but it does not advertise or execute `list-pins`/pinned-message reads.

## Plan

Implement Telegram pinned-message reads as a narrow on-demand message action. Add `list-pins` support to the Telegram plugin action discovery and runtime, backed by grammY/Bot API `getChat` on the resolved chat target; return a `pins` array containing zero or one current pinned message because Telegram `ChatFullInfo.pinned_message` exposes only the most recent pinned message. Add an `actions.pins` gate in Telegram config/types/schema/docs if needed, defaulting consistently with other safe message actions. Do not add automatic session-start pinned-context fetching, persistence, scheduling, or broad memory/session behavior. Preserve plugin boundary rules under `extensions/telegram`, update user-facing docs, and include focused tests for discovery, runtime success, no pin, action-disabled behavior, and target resolution.
