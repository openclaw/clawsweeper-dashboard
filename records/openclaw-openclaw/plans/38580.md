## Summary

Keep open: current main and v2026.6.5 still lack a Telegram pinned-message read action. The two linked implementation attempts are closed unmerged, and the related open pin/unpin issue tracks a distinct write capability rather than this read path.

## Plan

Add Telegram support for reading the current pinned message through the existing generic `message` tool `list-pins` action. Keep the change inside the Telegram plugin/channel action boundary. Use Telegram Bot API `getChat` and return zero or one pinned message in a generic `pins` result shape. Do not add automatic session-start fetching, do not promise pin history or topic-specific pin lists, do not bundle the separate pin/unpin write request, and do not edit `CHANGELOG.md`. Add focused Telegram discovery/runtime/send tests and update Telegram docs only for the new on-demand read action. Include redacted live Telegram proof before landing if feasible.
