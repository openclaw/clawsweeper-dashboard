# Coding Plan for openclaw/openclaw#38580: Feature request: Read pinned Telegram messages via bot API

Source report: [records/openclaw-openclaw/items/38580.md](records/openclaw-openclaw/items/38580.md)

## Summary

Keep open: current main and latest release still lack Telegram pinned-message reads. The shared `list-pins` action exists, and Telegram has adjacent write-side pin helpers plus `getChat` usage, but the Telegram plugin does not advertise, gate, or dispatch a pinned-message read action; the two related implementation PRs were closed unmerged.

## Plan

Implement Telegram pinned-message reads for https://github.com/openclaw/openclaw/issues/38580. Add a Telegram plugin-owned on-demand `list-pins`/`listPins` message action backed by grammY/Bot API `getChat`, returning a `pins` array with zero or one current pinned message; add a `channels.telegram.actions.pins` gate consistent with existing action gates; update Telegram docs, CLI/message docs if channel support changes, config schema/types, and focused tests. Keep the scope to read-only pinned-message listing; do not add automatic session-start context injection, scheduled refresh, storage/cache behavior, or standalone write-side pin/unpin actions from https://github.com/openclaw/openclaw/issues/30518 unless maintainers explicitly expand scope. Use the closed attempts https://github.com/openclaw/openclaw/pull/38613 and https://github.com/openclaw/openclaw/pull/38718 only as reference, not as an unreviewed bulk apply. Add a changelog entry because this is user-visible, and include redacted live Telegram proof showing a real bot/chat returning the current pinned message.
