# Coding Plan for openclaw/openclaw#38580: Feature request: Read pinned Telegram messages via bot API

Source report: [records/openclaw-openclaw/items/38580.md](records/openclaw-openclaw/items/38580.md)

## Summary

Keep open: current main and v2026.6.5 still do not expose a Telegram pinned-message read action, and the two linked implementation attempts are closed unmerged. The related open pin/unpin request is a distinct write capability, so it does not supersede this read-only session-context feature.

## Plan

Implement this issue as a narrow Telegram plugin feature. Add an on-demand read-only `list-pins` message action for Telegram using the existing generic `CHANNEL_MESSAGE_ACTION_NAMES` entry and the Telegram Bot API `getChat` contract. Keep ownership in `extensions/telegram`: update discovery/action mapping, runtime dispatch, and a small helper that resolves the target chat using existing target normalization/getChat patterns, then returns a `pins` array with zero or one current pinned message from `pinned_message`. Do not add automatic session-start injection, persistence/cache, topic pin history, a new core action name, or unrelated pin/unpin write support. Avoid adding a new config option; if a config gate becomes unavoidable, stop for maintainer decision instead of inventing a new default. Update Telegram docs and focused tests. Treat https://github.com/openclaw/openclaw/pull/38613 and https://github.com/openclaw/openclaw/pull/38718 as design context only because both are closed unmerged; https://github.com/openclaw/openclaw/issues/30518 is related but covers write pin/unpin actions and should remain separate. No CHANGELOG.md edit.
