# Coding Plan for openclaw/openclaw#38580: Feature request: Read pinned Telegram messages via bot API

Source report: [records/openclaw-openclaw/items/38580.md](records/openclaw-openclaw/items/38580.md)

## Summary

Keep open: current main and the latest v2026.5.22 release still do not expose a Telegram pinned-message read action, and the two related implementation PRs were closed unmerged. The remaining work is a focused Telegram plugin feature, distinct from the separate open pin/unpin write-action request.

## Plan

Implement the open Telegram pinned-message read feature as an on-demand Telegram plugin message action. Add list-pins/listPins handling that uses the Telegram Bot API getChat result pinned_message, returns a pins array with the current pinned message or an empty array when none exists, supports explicit targets and current chat context consistently with existing Telegram message actions, adds a channels.telegram.actions.pins gate in config types/schema/docs, updates message tool discovery/docs, and adds focused tests. Do not implement automatic session-start pinned-context injection, persistent storage, scheduled refresh, or topic pin history in this PR. Include a user-facing CHANGELOG.md entry because this is a Telegram feature.
