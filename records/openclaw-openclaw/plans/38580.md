## Plan

Implement Telegram pinned-message reads for https://github.com/openclaw/openclaw/issues/38580. Add a Telegram plugin-owned `list-pins` message action backed by Telegram Bot API `getChat` / `pinned_message`, return `{ ok: true, pins: [...] }` compatible with existing message-format rendering, handle no pinned message as an empty list, and surface Telegram API errors clearly. Add a strict `channels.telegram.actions.pins` gate consistent with Slack/Discord. Update Telegram action discovery/runtime aliases, config types/schema/help as needed, docs at `docs/channels/telegram.md`, focused tests, and a changelog entry if the change is user-facing. Do not add automatic session-start pinned-context injection in this PR; keep that as a separate product decision. Do not move Telegram-specific behavior into core except for existing generic action contracts. Coordinate with https://github.com/openclaw/openclaw/issues/30518 only if the same `actions.pins` work naturally exposes `pin`/`unpin`; otherwise keep this PR focused on `list-pins`.
