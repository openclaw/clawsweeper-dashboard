Keep open: current main and the latest release still lack a Telegram pinned-message read action or `/getpinned` path, while two related implementation PRs were closed unmerged and the remaining work has a clear Telegram plugin-owned shape.

## Plan

Implement a narrow Telegram pinned-message read feature for https://github.com/openclaw/openclaw/issues/38580. Add a Telegram plugin-owned on-demand `list-pins` message action backed by Bot API `getChat`, returning a `pins` array with zero or one normalized current pinned message. Wire discovery and runtime dispatch in `extensions/telegram/src/channel-actions.ts` and `extensions/telegram/src/action-runtime.ts`, add or reuse a helper in the Telegram send/API layer without broad core changes, add `channels.telegram.actions.pins` to TypeScript config and zod schema, and document that Telegram exposes only the most recent pinned message. Do not implement automatic session-start context injection in this PR. Reuse the existing generic `list-pins` action name; do not add a new core action vocabulary term. Add focused tests for discovery, runtime success/no-pin/error handling, config gating, and shared message action behavior. Include a user-facing changelog entry if landing policy requires one. Require redacted live Telegram proof before merge showing a real bot/chat where `list-pins` returns the current pinned message, with private chat IDs, tokens, phone numbers, IP addresses, and non-public endpoints redacted.
