## Summary

Keep open: current main still lacks a Telegram pinned-message read action. The generic `list-pins` action name exists, but Telegram does not advertise or route it, and both related implementation attempts are closed unmerged.

## Plan

Implement Telegram pinned-message reads for https://github.com/openclaw/openclaw/issues/38580 as a narrow Telegram plugin feature. Add a Telegram-owned `list-pins` message action backed by grammY/Bot API `getChat`, returning an empty `pins` array when no current pinned message is available and one normalized pinned-message entry when present. Wire discovery, runtime dispatch, config schema/types, docs, and focused tests; use a consistent `channels.telegram.actions.pins` gate if following existing action-gate patterns. Do not implement automatic session-start context injection, scheduled refresh, or unrelated Telegram pin/unpin write actions from https://github.com/openclaw/openclaw/issues/30518. Include a user-facing changelog entry if required by landing policy, and include redacted live Telegram proof or a Telegram E2E artifact showing `list-pins` returns the current pinned message from a real bot/chat.
