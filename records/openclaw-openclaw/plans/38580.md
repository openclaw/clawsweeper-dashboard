## Summary

Keep open: current main and v2026.5.20 still lack a Telegram pinned-message read action or /getpinned path, while the two related implementation PRs were closed unmerged and the remaining work is a focused Telegram plugin feature.

## Plan

Implement this issue as a narrow Telegram plugin feature. Add an on-demand list-pins message action for Telegram that calls grammY/Bot API getChat and returns zero or one current pinned message from ChatFullInfo.pinned_message. Keep automatic session-start context injection out of scope. Add a channels.telegram.actions.pins gate, default it consistently with other safe read actions, document that Telegram exposes only the most recent pinned message, and keep the implementation inside the Telegram plugin boundary. Do not revive the old closed PR structure under legacy src/telegram paths; use current extensions/telegram files and public plugin SDK seams. Include focused tests for discovery, runtime success/no-pin/error-disabled cases, config schema, and shared message-action discovery. Add a changelog entry because this is user-visible. Validation should include targeted Vitest via node scripts/run-vitest.mjs and a live or Telegram E2E proof with private details redacted.
