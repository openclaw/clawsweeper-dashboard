## Plan

Implement https://github.com/openclaw/openclaw/issues/38580 as a narrow Telegram plugin feature. Add an on-demand Telegram `list-pins`/`pins` message action that fetches the current chat's `pinned_message` via grammY/Bot API `getChat`, returns an empty pins array when no pin exists, and handles Telegram target parsing consistently with existing Telegram send/read helpers. Add `channels.telegram.actions.pins` config/schema/docs gating, defaulting consistently with other read-style message actions. Do not implement automatic session-start pinned-context injection, scheduled refresh, storage, or pinned-message history beyond what Telegram exposes. Update Telegram plugin tests, shared message-action tests if needed, public docs, and add a user-facing changelog entry if this ships as a feature.
