# Coding Plan for openclaw/openclaw#38580: Feature request: Read pinned Telegram messages via bot API

Source report: [records/openclaw-openclaw/items/38580.md](records/openclaw-openclaw/items/38580.md)

## Summary

Keep open: current main still lacks Telegram pinned-message reads. The shared `list-pins` action exists, Telegram already has write-side pin helpers and Bot API `getChat` support, but the Telegram plugin does not advertise, gate, or dispatch a read action; both related implementation PRs are closed unmerged.

## Plan

Implement Telegram pinned-message reads for this issue as a narrow on-demand message action. Add Telegram plugin-owned `list-pins` / `listPins` discovery and runtime dispatch backed by Bot API `getChat`, returning a `pins` array with zero or one current pinned message using the existing cross-channel result shape. Add a `channels.telegram.actions.pins` gate consistent with other Telegram action gates, update strict config schema/types/docs, and add focused tests for discovery, gating, no-pin, pinned-message, and unsupported/disabled behavior. Reuse current Telegram target parsing/account/token/retry helpers and do not implement automatic session-start context injection or standalone pin/unpin write actions unless maintainers explicitly expand scope. Include a user-facing changelog entry if the repair lane is preparing a mergeable feature PR. Validate with focused Vitest and formatting checks, and include redacted live Telegram proof when possible; redact bot tokens, chat IDs if private, phone numbers, IPs, and non-public endpoints.
