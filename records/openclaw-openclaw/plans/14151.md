## Summary

Current main still lacks any config, runner parameter, or subscriber state that can classify a wholly untagged assistant text stream as reasoning; native reasoning events, ordinary tagged streams, and orphan closing tags are handled, but they do not cover the reported no-delimiter behavior. The only linked implementation PR closed unmerged, so the feature remains necessary and should stay open as a focused, low-priority enhancement.

## Plan

Implement https://github.com/openclaw/openclaw/issues/14151 against the current embedded-agent architecture. Add one default-off typed and documented config contract for treating the opening portion of an otherwise untagged assistant text stream as reasoning, with explicit global/per-model precedence. Resolve it before the embedded-agent run, thread it through the current runner and subscriber parameters, and apply it exactly once before existing reasoning extraction and block-tag stripping. Do not hardcode provider or model names. Preserve native structured thinking events; do not synthesize an opening when the message already starts with a recognized reasoning tag; reset state for every assistant message or stream item; and cover streaming, final-text, native-tag, native-thinking-event, and orphan-close behavior.
