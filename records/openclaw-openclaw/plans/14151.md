## Summary

Keep open: current main and the latest release still lack the requested default-off synthetic `<think>` prepending config surface, and the only related implementation PR is closed unmerged.

## Plan

Implement the requested default-off synthetic think-tag prepend option for local reasoning models. Add a documented config contract, likely `agents.defaults.forcePrependThinkTag?: boolean` plus per-model override on `agents.defaults.models["provider/model"].forcePrependThinkTag?: boolean`, with clear precedence over the global default. Resolve the effective value before embedded Pi runs, thread it through `RunEmbeddedPiAgentParams`, `EmbeddedRunAttemptParams`, `buildEmbeddedRunBaseParams`, `subscribeEmbeddedPiSession`, and the message handler context/state. Apply `<think>` exactly once per assistant message before existing reasoning extraction/stripping, but do not prepend when the stream/final assistant text already starts with a native opening reasoning tag. Cover streaming deltas, final/non-streaming text, disabled default behavior, enabled global behavior, per-model override, and native-tag guard. Update generated/config docs and help/labels if the schema contract requires it. Use the closed source PR https://github.com/openclaw/openclaw/pull/14323 as context, but do not copy its broad type duplication; follow current main file shapes. Do not change provider-specific `params` semantics, do not add startup migrations, and keep the default false.
