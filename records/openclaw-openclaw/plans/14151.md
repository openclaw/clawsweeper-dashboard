## Summary

Keep open: current main still lacks the requested default-off synthetic `<think>` prepend option, and the related implementation PR is closed unmerged. The current code accepts and documents reasoning visibility, parses existing reasoning tags, and enforces final tags, but it does not expose or thread a config flag that injects an opening tag before parsing.

## Plan

Implement the requested default-off synthetic `<think>` prepend option for local reasoning models. Add a documented config contract for global `agents.defaults` and per-model `agents.defaults.models.*` settings, resolve precedence before embedded Pi runs, thread the resolved flag through `src/auto-reply/reply/agent-runner-run-params.ts`, `src/agents/pi-embedded-runner/run/params.ts`, `run/types.ts`, and `run/attempt.ts`, and apply it in the embedded subscriber before existing reasoning extraction/stripping. Inject exactly once per assistant message, do not prepend when the model already starts with a native `<think>` opening tag, preserve `enforceFinalTag` behavior, and cover both streaming deltas and final message-end paths. Include docs/config schema help and a user-facing changelog entry; do not change provider defaults or make the behavior default-on.
