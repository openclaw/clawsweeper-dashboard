## Summary

Keep open: current main still lacks a default-off synthetic `<think>` opening-tag option, and the directly related implementation PR is closed unmerged. The existing maintainer and ClawSweeper comments remain consistent with current source inspection.

## Plan

Implement https://github.com/openclaw/openclaw/issues/14151 as a narrow OpenClaw feature. Add default-off `forcePrependThinkTag` support using existing camelCase config conventions, with a global default under `agents.defaults` and a per-model override under `agents.defaults.models[model]`. Resolve precedence before embedded Pi runs, thread the effective value through the auto-reply runner params, embedded runner attempt params, and `SubscribeEmbeddedPiSessionParams`, then apply it once before existing reasoning extraction/stripping for both streaming deltas and final message-end text. Guard against native opening tags so models that already emit `<think>` do not produce `<think><think>...`. Do not make the behavior default-on, do not change provider defaults or prompt templates, and do not add startup/load-time legacy migration. Update docs/config help/labels and add a user-facing changelog entry crediting @LisaMacintosh if appropriate.
