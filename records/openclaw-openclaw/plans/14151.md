## Plan

Implement the requested default-off synthetic `<think>` opening-tag option for local reasoning models. Add a documented config contract for a global default and per-model override, keep default behavior unchanged, resolve precedence before embedded Pi runs, thread the resolved flag through `buildEmbeddedRunBaseParams`, embedded runner params/types, `runEmbeddedPiSession`, and `subscribeEmbeddedPiSession`, and apply the synthetic opening tag exactly once before existing reasoning extraction/stripping only when the assistant message does not already begin with a native `<think>` tag.
