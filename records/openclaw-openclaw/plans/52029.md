# Coding Plan for openclaw/openclaw#52029: Feature Request: heartbeat.tools option to disable tools during heartbeat

Source report: [records/openclaw-openclaw/items/52029.md](records/openclaw-openclaw/items/52029.md)

## Summary

Keep open: current main still lacks a heartbeat-scoped config option to disable tools, while ordinary heartbeat turns only pass `disableTools` for due-commitment delivery. The request is a narrow feature/config addition with clear runtime, schema, docs, and test surfaces.

## Plan

Implement `heartbeat.tools: false` for `agents.defaults.heartbeat` and `agents.list[].heartbeat`. Keep the default tool-capable behavior unchanged. When the resolved heartbeat config has `tools: false`, pass `disableTools: true` and `skillFilter: []` into the heartbeat `getReplyFromConfig` options for ordinary heartbeat turns; preserve the existing due-commitment no-tools behavior. Update the Zod schema, public config types, schema labels/help if applicable, heartbeat config-honor inventory, docs in `docs/gateway/heartbeat.md` and `docs/gateway/config-agents.md`, and add a changelog entry if maintainer policy requires one. Add focused tests proving schema acceptance, defaults/per-agent merge behavior, ordinary heartbeat `tools: false` disables tools, default heartbeats still leave tools enabled, and due-commitment behavior still disables tools. Do not change global agent tool policy, do not make heartbeats no-tools by default, and do not add a heartbeat allowlist in this PR unless an existing schema pattern makes it unavoidable.
