# Coding Plan for openclaw/openclaw#52029: Feature Request: heartbeat.tools option to disable tools during heartbeat

Source report: [records/openclaw-openclaw/items/52029.md](records/openclaw-openclaw/items/52029.md)

## Summary

Keep open: current main and the latest release still lack a heartbeat-scoped tool toggle, and ordinary heartbeat turns only set `disableTools` for due-commitment delivery. The issue is a narrow feature/config addition with a clear existing runtime seam.

## Plan

Implement support for `agents.defaults.heartbeat.tools` and `agents.list[].heartbeat.tools` where `false` disables all tools for heartbeat turns. Add the boolean to the heartbeat schema, public types, schema labels/help, config-honor inventory, and heartbeat/config docs. In `runHeartbeatOnce`, after defaults and per-agent heartbeat config are merged, map `heartbeat.tools === false` to `disableTools: true` and `skillFilter: []` in reply options for ordinary heartbeat turns. Preserve the current default tool-capable heartbeat behavior, preserve due-commitment safety behavior, and do not add a tool allowlist or change global tool policy in this PR. Add regression coverage for default and per-agent config, schema acceptance/rejection, ordinary heartbeat reply options disabling tools, and no regression to due-commitment/heartbeat response-tool behavior. Add a changelog entry if required for the new user-facing config option.
