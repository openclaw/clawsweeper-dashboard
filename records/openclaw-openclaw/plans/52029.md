# Coding Plan for openclaw/openclaw#52029: Feature Request: heartbeat.tools option to disable tools during heartbeat

Source report: [records/openclaw-openclaw/items/52029.md](records/openclaw-openclaw/items/52029.md)

## Summary

Keep open. Current main and the latest release still lack a heartbeat-scoped tool-disable config; ordinary heartbeat turns only pass `disableTools` for due-commitment heartbeats, while the schema, public types, docs, and inventory have no `heartbeat.tools` option.

## Plan

Implement the heartbeat-scoped tool-disable option for https://github.com/openclaw/openclaw/issues/52029. Add a documented boolean `heartbeat.tools` option for both `agents.defaults.heartbeat` and `agents.list[].heartbeat`; when it is exactly `false`, pass the existing `disableTools: true` run option for ordinary heartbeat turns so no tool definitions are included. Preserve current behavior when the option is omitted or true, preserve due-commitment heartbeat behavior, and do not add an allowlist or broader tool-policy redesign in this PR. Update schema/types/labels/help as needed, the heartbeat config-honor inventory, docs, focused tests, and CHANGELOG.md for the user-facing config addition.
