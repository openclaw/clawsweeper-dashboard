# Coding Plan for openclaw/openclaw#10467: [Feature Request]: Multi-lane concurrency support for sub-agents via sessions_spawn

Source report: [records/openclaw-openclaw/items/10467.md](records/openclaw-openclaw/items/10467.md)

## Summary

Keep open. Current main and the latest shipped tag still expose one fixed `subagent` lane for `sessions_spawn`; the tool schema has no lane field, native and ACP dispatch hard-code that lane, and config/docs still describe one scalar sub-agent concurrency cap.

## Plan

Implement multi-lane concurrency support for `sessions_spawn` for https://github.com/openclaw/openclaw/issues/10467. Add an optional validated `lane` parameter to the `sessions_spawn` tool that defaults to the existing `subagent` lane, forward the resolved lane through both native subagent and ACP spawn paths, and add compatible per-lane subagent concurrency configuration without changing existing callers or the default `agents.defaults.subagents.maxConcurrent` behavior. Update TypeScript config types, strict zod schema, config labels/help if needed, docs, and focused tests. Do not broaden the scope into per-parent caps, CLI final-output waiting, per-agent inbound lanes, or queue status tools; those are separate adjacent issues. Add a changelog entry because this is a user-visible tool/config feature.
