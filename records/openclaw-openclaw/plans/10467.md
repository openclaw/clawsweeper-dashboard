## Summary

Keep open. Current main still lacks caller-selectable `sessions_spawn` lanes: the tool schema has no `lane` input, native and ACP spawns still dispatch through the fixed `subagent` lane, and config/docs/tests still model one scalar sub-agent concurrency cap.

## Plan

Implement caller-selectable sub-agent queue lanes for https://github.com/openclaw/openclaw/issues/10467. Add an optional `lane` parameter to `sessions_spawn` that defaults to `subagent`, validate/normalize lane names with a bounded safe pattern, forward the resolved lane through both native and ACP spawn paths, and apply compatible per-lane subagent concurrency config while preserving existing `agents.defaults.subagents.maxConcurrent` behavior for callers that omit `lane`. Update schema/types/help/docs and focused tests. Do not change unrelated agent/provider/channel routing semantics, do not remove the existing scalar default, and coordinate with adjacent capacity/per-parent issues only by avoiding conflicting semantics, not by broadening this PR.
