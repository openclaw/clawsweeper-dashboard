# Coding Plan for openclaw/openclaw#10467: [Feature Request]: Multi-lane concurrency support for sub-agents via sessions_spawn

Source report: [records/openclaw-openclaw/items/10467.md](records/openclaw-openclaw/items/10467.md)

## Summary

Keep open. Current main still has one fixed `subagent` lane for `sessions_spawn`: the tool schema has no lane field, both native and ACP dispatch hard-code the subagent lane, and config/docs still expose only one scalar sub-agent concurrency cap.

## Plan

Implement named sub-agent lanes for `sessions_spawn` without changing existing callers. Add an optional validated `lane` parameter that defaults to `subagent`, forward it through `createSessionsSpawnTool`, `spawnSubagentDirect`, and `spawnAcpDirect`, and apply compatible per-lane subagent concurrency config while preserving the existing scalar `agents.defaults.subagents.maxConcurrent` behavior. Update TypeScript config types, strict zod schema, docs, and focused tests. Do not fold in the separate per-parent cap work from https://github.com/openclaw/openclaw/issues/73870 or CLI liveness behavior from https://github.com/openclaw/openclaw/issues/76145. Add a changelog entry because this is a user-visible feature/config surface.
