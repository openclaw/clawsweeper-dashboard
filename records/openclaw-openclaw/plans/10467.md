## Summary

Keep open. Current main and the latest release still expose `sessions_spawn` as one fixed `subagent` queue lane with no caller-selected lane or per-lane subagent concurrency config.

## Plan

Implement the feature requested in https://github.com/openclaw/openclaw/issues/10467. Add a validated optional `sessions_spawn.lane` parameter that defaults to the current `subagent` lane, forward the resolved lane through both native `spawnSubagentDirect` and ACP `spawnAcpDirect`, and add compatible per-lane subagent concurrency config without changing existing callers. Keep lane names bounded and do not let agent-controlled arbitrary strings create unbounded queue-state growth or bypass reserved lane policy. Preserve the existing scalar `agents.defaults.subagents.maxConcurrent` behavior for the default lane. Update types, strict zod schema, config resolver/help where applicable, docs, and focused tests. Do not edit `CHANGELOG.md` for this normal fix PR.
