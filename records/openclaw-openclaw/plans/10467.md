# Coding Plan for openclaw/openclaw#10467: [Feature Request]: Multi-lane concurrency support for sub-agents via sessions_spawn

Source report: [records/openclaw-openclaw/items/10467.md](records/openclaw-openclaw/items/10467.md)

## Summary

Keep open. Current main still routes `sessions_spawn` through one fixed `subagent` lane, with no tool `lane` field, no native/ACP forwarding, and only scalar subagent concurrency config; the recent maintainer/ClawSweeper comments remain accurate.

## Plan

Implement the feature requested in https://github.com/openclaw/openclaw/issues/10467. Add an optional `sessions_spawn.lane` field that defaults to `subagent`, validate/normalize lane names with bounded length/character rules, pass the resolved lane through both native `spawnSubagentDirect` and ACP `spawnAcpDirect`, and apply per-lane subagent concurrency config while preserving `agents.defaults.subagents.maxConcurrent` as the default `subagent` cap. Update config types, zod schema/help/generated docs as required, public docs for subagents and ACP sessions_spawn, focused regression tests, and a user-facing changelog entry. Do not solve adjacent per-parent throttling, per-agent inbound lanes, or `openclaw agent --json` foreground lifecycle behavior in this PR.
