Keep open. Current main and the latest release still expose one fixed `subagent` lane for `sessions_spawn`; no caller-selected lane or per-lane subagent concurrency config exists yet.

## Plan

Add an optional validated `lane` field to `sessions_spawn` that defaults to `subagent`, forward the resolved lane through both native `spawnSubagentDirect` and ACP `spawnAcpDirect`, and apply compatible per-lane subagent concurrency config while preserving `agents.defaults.subagents.maxConcurrent` and existing omitted-lane behavior. Update config types/schema/help/baselines if required by the repo pattern, docs under `docs/tools/subagents.md` and `docs/tools/acp-agents.md`, and focused regression tests.
