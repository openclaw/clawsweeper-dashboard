# Coding Plan for openclaw/openclaw#10467: [Feature Request]: Multi-lane concurrency support for sub-agents via sessions_spawn

Source report: [records/openclaw-openclaw/items/10467.md](records/openclaw-openclaw/items/10467.md)

## Summary

Keep open. Current main still lacks a caller-selectable `sessions_spawn` lane: the tool schema has no `lane` input, both native and ACP spawn paths dispatch through the fixed `subagent` lane, and gateway/config concurrency remains a single scalar subagent cap.

## Plan

Implement multi-lane subagent spawning for https://github.com/openclaw/openclaw/issues/10467. Add an additive `lane` option to `sessions_spawn` that defaults to `subagent`, validate lane names, forward the resolved lane through both native and ACP spawn paths, and add backward-compatible per-lane subagent concurrency config while preserving `agents.defaults.subagents.maxConcurrent` for existing users. Update schema/help/docs and regression tests. Do not change unrelated per-agent inbound lanes, per-parent caps, or session completion behavior; keep core/plugin boundaries intact. Add a changelog entry only if maintainers require it at landing.
