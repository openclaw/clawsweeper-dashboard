# Coding Plan for openclaw/openclaw#10467: [Feature Request]: Multi-lane concurrency support for sub-agents via sessions_spawn

Source report: [records/openclaw-openclaw/items/10467.md](records/openclaw-openclaw/items/10467.md)

## Summary

Keep open. Current main still exposes `sessions_spawn` through one fixed `subagent` lane with no caller `lane` field, no native/ACP lane forwarding, and only scalar subagent concurrency config; the recent maintainer-side comments remain accurate.

## Plan

Implement https://github.com/openclaw/openclaw/issues/10467 as a narrow, backward-compatible feature. Add an optional `lane` parameter to `sessions_spawn` that defaults to `subagent`, validate lane names, forward the resolved lane through both native subagent and ACP spawn paths, and apply compatible per-lane subagent concurrency configuration while preserving `agents.defaults.subagents.maxConcurrent` as the default lane cap. Update config types, zod schema/help/metadata, docs, changelog, and focused regression tests. Do not change per-agent inbound lanes, session lane concurrency, per-parent spawn caps, CLI completion waiting, or unrelated subagent lifecycle behavior.
