# Coding Plan for openclaw/openclaw#10467: [Feature Request]: Multi-lane concurrency support for sub-agents via sessions_spawn

Source report: [records/openclaw-openclaw/items/10467.md](records/openclaw-openclaw/items/10467.md)

## Summary

Keep open. Current main still lacks caller-selectable `sessions_spawn` lanes: the tool schema has no `lane` input, native and ACP spawns still dispatch through the fixed `subagent` lane, and config/docs/tests still model one scalar sub-agent concurrency cap.

## Plan

Fix https://github.com/openclaw/openclaw/issues/10467 by adding additive multi-lane support for `sessions_spawn`. Add a validated optional `lane` input that defaults to `subagent`; propagate it through `src/agents/tools/sessions-spawn-tool.ts`, `src/agents/subagent-spawn.ts`, and `src/agents/acp-spawn.ts`; add compatible per-lane subagent concurrency config while preserving `agents.defaults.subagents.maxConcurrent` as the default/fallback; update config types/schema/help, docs, tests, and a user-facing changelog entry. Keep this scoped to sub-agent spawn lanes; do not fold in per-parent throttling, capacity/status tools, or per-agent inbound lanes except where compatibility tests require coordination. Preserve existing callers and default behavior when `lane` is omitted.
