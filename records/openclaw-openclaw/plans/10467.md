## Summary

Keep open. Current main still exposes `sessions_spawn` with one fixed `subagent` lane and one scalar subagent concurrency limit, while adjacent open issues cover capacity visibility and per-parent throttling rather than this named-lane feature.

## Plan

Implement named subagent queue lanes for `sessions_spawn` in one focused PR. Add an optional `lane` input that defaults to `subagent`, validate lane names with a conservative documented pattern, and forward the resolved lane through both native `spawnSubagentDirect` and ACP `spawnAcpDirect` gateway `agent` calls. Add compatible per-lane subagent concurrency configuration under the existing agents defaults surface while preserving `agents.defaults.subagents.maxConcurrent` for the default lane and existing configs. Update schema/types/help/docs/tests and add a single-line CHANGELOG entry. Do not solve capacity reporting from https://github.com/openclaw/openclaw/issues/57705 or per-parent throttling from https://github.com/openclaw/openclaw/issues/73870 unless a tiny shared helper is required.
