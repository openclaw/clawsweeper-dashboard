# Coding Plan for openclaw/openclaw#10467: [Feature Request]: Multi-lane concurrency support for sub-agents via sessions_spawn

Source report: [records/openclaw-openclaw/items/10467.md](records/openclaw-openclaw/items/10467.md)

## Summary

Keep open. Current main and v2026.5.22 still lack caller-selected `sessions_spawn` lanes; native and ACP spawned sessions are both pinned to the fixed `subagent` lane, and no open PR owns the exact requested fix.

## Plan

Implement multi-lane subagent spawning for https://github.com/openclaw/openclaw/issues/10467. Add an additive `sessions_spawn.lane` string parameter with validation and default `subagent`; thread the resolved lane through `src/agents/tools/sessions-spawn-tool.ts`, native `spawnSubagentDirect`, and ACP `spawnAcpDirect` so both gateway `agent` calls use the caller-selected lane. Add compatible per-lane subagent concurrency config while preserving `agents.defaults.subagents.maxConcurrent` as the default `subagent` lane behavior for existing users. Update exported config types, strict zod schema/help where applicable, docs for `sessions_spawn`, and focused regression tests for schema, native dispatch, ACP dispatch, gateway lane concurrency, default compatibility, and invalid lane names. Add a user-facing changelog entry. Do not change unrelated subagent lifecycle, completion routing, thread binding, or ACP runtime behavior.
