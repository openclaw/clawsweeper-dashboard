## Plan

Fix https://github.com/openclaw/openclaw/issues/10467 with a narrow additive implementation. Add an optional `lane` parameter to the `sessions_spawn` tool that defaults to `subagent`, validate lane names with a bounded allowlist-style format/length, forward the resolved lane through both native `spawnSubagentDirect` and ACP `spawnAcpDirect`, and make gateway lane concurrency apply compatible per-lane subagent limits while preserving `agents.defaults.subagents.maxConcurrent` as the default `subagent` behavior.
