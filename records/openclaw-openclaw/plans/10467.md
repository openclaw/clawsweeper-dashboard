## Summary

Keep open. Current main still has a fixed `subagent` lane for `sessions_spawn`, and the related open lane-concurrency PRs cover session or per-agent lanes rather than caller-selected sub-agent lanes.

## Plan

Implement additive multi-lane support for `sessions_spawn` from https://github.com/openclaw/openclaw/issues/10467. Add a validated optional `lane` parameter that defaults to `subagent`, thread the resolved lane through both native `spawnSubagentDirect` and ACP `spawnAcpDirect` gateway dispatch, and add compatible per-lane subagent concurrency config while preserving `agents.defaults.subagents.maxConcurrent` and all existing omitted-lane behavior. Keep lane names bounded and server-safe; do not change unrelated per-agent/session lane semantics or solve per-parent caps in this PR. Update schema/types/help/docs and add focused regression tests plus a changelog entry for the user-visible config/tool feature.
