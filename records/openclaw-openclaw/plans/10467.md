## Summary

Keep open. Current main still lacks caller-selectable `sessions_spawn` lanes: the tool schema has no `lane` input, native and ACP dispatch remain pinned to `subagent`, and config/docs/tests still model one scalar sub-agent concurrency cap.

## Plan

Implement multi-lane `sessions_spawn` support for https://github.com/openclaw/openclaw/issues/10467. Add an optional validated `lane` parameter to the `sessions_spawn` tool that defaults to `subagent`; forward the resolved lane through both native `spawnSubagentDirect` and ACP `spawnAcpDirect`; apply compatible per-lane subagent concurrency configuration while preserving `agents.defaults.subagents.maxConcurrent` for the default lane and existing callers. Keep lane-name validation bounded so model-controlled input cannot create arbitrary unbounded queue state or bypass intended caps. Update config types, strict zod schema/help/docs, and focused tests. Do not solve adjacent capacity-status, per-parent throttling, or CLI final-output issues in this PR unless a small compatibility test is required. Add a user-facing changelog entry under the active version if repo policy expects it for this feature.
