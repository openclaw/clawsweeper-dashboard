## Summary

Keep open. Current main still routes every `sessions_spawn` child run through one fixed `subagent` lane, and the related open items cover adjacent capacity/per-agent/per-parent scheduling rather than this caller-selectable lane request.

## Plan

Implement https://github.com/openclaw/openclaw/issues/10467 as a focused, backward-compatible feature. Add an optional `lane` parameter to `sessions_spawn` that defaults to `subagent`, validates lane names, and forwards the resolved lane through both native `spawnSubagentDirect` and ACP `spawnAcpDirect` into the gateway `agent` call. Add compatible per-lane subagent concurrency config under the existing subagent defaults namespace, apply it on gateway startup and config reload, and preserve `agents.defaults.subagents.maxConcurrent` as the default `subagent` lane behavior.
