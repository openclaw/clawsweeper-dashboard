## Summary

Keep open. Current main still exposes `sessions_spawn` through one fixed `subagent` lane with no caller `lane` field, no native/ACP forwarding, and only scalar subagent concurrency config; the recent maintainer-side comments remain accurate.

## Plan

Implement https://github.com/openclaw/openclaw/issues/10467 by adding additive multi-lane support for `sessions_spawn`. Add an optional `lane` parameter that defaults to the existing `subagent` lane, validate lane names with bounded length/characters and reserved-lane behavior, forward the resolved lane through both native `spawnSubagentDirect` and ACP `spawnAcpDirect`, and add compatible per-lane subagent concurrency config while preserving `agents.defaults.subagents.maxConcurrent` as the scalar fallback. Update exported config types, strict zod schema/help/docs, public subagent and ACP docs, changelog if user-facing policy requires it, and regression tests. Do not fold in per-agent queue lanes, per-parent caps, queue-status tooling, CLI liveness/final text behavior, or broad scheduler refactors; treat those related items as separate unless maintainers explicitly ask to combine scope.
