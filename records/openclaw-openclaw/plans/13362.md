## Summary

Keep open. Current main still has no native `openclaw doctor --dry-run`, and the related global dry-run issue and `--check-config` PR cover different layers rather than this doctor repair side-effect surface.

## Plan

Implement https://github.com/openclaw/openclaw/issues/13362 as a focused native `openclaw doctor --dry-run` feature. Register and type the doctor `--dry-run` flag, thread it through the prompter, config flow, health context, and doctor repair/contribution paths, and render the same repair plan without prompts or writes. Dry-run must guard more than `replaceConfigFile`: cover cron store normalization, configured plugin install/update and install-record writes, gateway service install/start/restart, and other doctor repair side effects found during implementation. Keep https://github.com/openclaw/openclaw/issues/41418 and https://github.com/openclaw/openclaw/pull/47302 separate; do not implement global agent dry-run or `--check-config` as part of this PR. Update `docs/cli/doctor.md` and add a user-facing `CHANGELOG.md` entry.
