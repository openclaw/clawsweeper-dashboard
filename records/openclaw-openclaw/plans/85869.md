# Coding Plan for openclaw/openclaw#85869: Plugin install should guard WhatsApp plugin/core version compatibility

Source report: [records/openclaw-openclaw/items/85869.md](records/openclaw-openclaw/items/85869.md)

## Summary

Keep open: current main and latest release still only enforce `openclaw.install.minHostVersion` for direct npm/package installs, while the WhatsApp package advertises a newer plugin API/OpenClaw peer requirement than that install floor.

## Plan

Fix https://github.com/openclaw/openclaw/issues/85869. Direct npm/package plugin installs must fail closed before persisting a plugin when downloaded package metadata declares an OpenClaw/plugin API or OpenClaw peer requirement newer than the running host. Keep the fix generic in the plugin install/package metadata path; do not special-case WhatsApp auth recovery. Preserve existing `openclaw.install.minHostVersion` behavior, reuse the ClawHub/plugin-package compatibility contract where practical, and add regression coverage showing a WhatsApp-like package with `minHostVersion >=2026.4.25` but `openclaw.compat.pluginApi`/`peerDependencies.openclaw >=2026.5.24` is rejected on an older host such as `2026.5.10-beta.1`. Add or update docs/changelog only if the user-visible CLI install error contract changes.
