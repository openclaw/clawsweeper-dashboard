## Summary

Keep open. Current main still treats spawned child sessions as `non-main` sandboxed runs, accepts the spawn before the child runner resolves Docker sandbox setup, and has no merged PR closing the broader startup-stall behavior; the overlapping Docker failure report at https://github.com/openclaw/openclaw/issues/57992 is still open.

## Plan

Fix sandboxed native subagent startup when `agents.defaults.sandbox.mode: "non-main"` is enabled. Current main accepts `sessions_spawn`, registers a generated child session key, and then the async child runner resolves Docker sandbox setup before prompt progress; failures or stalls can remain invisible to `subagents list` and session metadata. Implement a narrow lifecycle/error-surfacing fix so the child either reaches transcript/token progress or transitions to a visible terminal error with sanitized logging. Coordinate Docker create/start failure handling with https://github.com/openclaw/openclaw/issues/57992. Do not disable `non-main` sandboxing, weaken sandbox inheritance, or add a new config knob. Add focused regression coverage and a changelog fix entry unless maintainers classify the final patch as internal-only.
