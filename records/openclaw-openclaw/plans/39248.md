## Summary

Keep open. Current main still supports `agents.defaults.sandbox.mode: "non-main"`, still classifies generated native subagent sessions as sandboxed, and still acknowledges `sessions_spawn` before async child dispatch; the related Docker diagnostics PR was closed unmerged after being narrowed away from this lifecycle path.

## Plan

Fix https://github.com/openclaw/openclaw/issues/39248. Current main accepts `sessions_spawn` under `agents.defaults.sandbox.mode: "non-main"` but a native sandboxed child can remain running with no transcript/messages/tokens and no gateway diagnostics. Keep the public `sessions_spawn` accepted response contract and the documented `non-main` semantics. Make post-ack sandbox/container/startup failures transition the child run/session to a terminal failed state, surface an actionable error in `subagents list`/session state/logs, and avoid zombie active runs. Add focused regression coverage around native `sessions_spawn` plus sandbox startup failure handling, and include Docker-backed proof by forcing Docker create/start or setupCommand failure. Do not fold in https://github.com/openclaw/openclaw/issues/69327 Docker env/setup inheritance policy, do not disable `non-main`, and do not add a new config option.
