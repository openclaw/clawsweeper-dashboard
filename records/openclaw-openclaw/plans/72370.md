## Summary

Keep open. Current main still preserves the documented no-override rule for workspace hooks, but an eventless managed hook can still win the same-name collision, register no handlers, and be reported as loadable because status only checks config and requirements.

## Plan

Fix https://github.com/openclaw/openclaw/issues/72370. Preserve the documented rule that workspace hooks cannot override same-name managed, bundled, or plugin hooks. Instead, make same-name collisions and eventless winning hooks actionable: carry enough collision state from hook resolution into hook status, treat hooks with empty metadata events as not loadable/eligible with a clear blocked reason, and show that reason in `openclaw hooks list`, `openclaw hooks check`, `openclaw hooks info`, and JSON output. Keep the existing loader warning. Add regression coverage for a managed/workspace same-name collision where the managed hook has no events, eventless hook status, and CLI formatting/JSON. Update docs if the status shape or CLI text changes, and add a `CHANGELOG.md` fix entry. Do not add a new config option, implement machine-wide policy hooks, or allow workspace hooks to override managed hooks.
