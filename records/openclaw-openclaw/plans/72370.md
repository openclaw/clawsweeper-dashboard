## Summary

Keep open. Current main still preserves the documented no-override policy for workspace hooks, but an enabled managed hook with empty metadata events can win the collision, register no handlers, and still appear ready in hook status/check surfaces.

## Plan

Fix https://github.com/openclaw/openclaw/issues/72370 by preserving the documented no-override policy while making hook collisions and eventless winners actionable. Do not allow workspace hooks to override managed, bundled, or plugin hooks, and do not add a new config option. Carry enough collision/eventless state through the hook discovery/status path so enabled hooks with no metadata events are not reported as ready/loadable, and so same-name workspace hooks rejected behind managed/bundled/plugin hooks are visible in `openclaw hooks list`, `openclaw hooks info`, and `openclaw hooks check` with a clear blocked/actionable reason. Add focused regression tests for managed-vs-workspace collisions, eventless managed hook readiness, and CLI JSON/text output. Update docs under hooks CLI/automation if the status fields or wording change, and add a changelog entry because this changes user-visible diagnostics.
