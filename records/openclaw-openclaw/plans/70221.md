# Coding Plan for openclaw/openclaw#70221: Feature request: Add width parameter to [embed] directive

Source report: [records/openclaw-openclaw/items/70221.md](records/openclaw-openclaw/items/70221.md)

## Summary

Keep open. Current main still has height-only `[embed]` parsing/rendering, while the shipped `chatMessageMaxWidth` setting only changes the global chat message column and does not provide a per-embed width contract.

## Plan

Implement per-embed width support for the Control UI `[embed]` directive. Add a typed, bounded `preferredWidth` hint parsed from `width="..."` on embed shortcodes, preserve it through normalized canvas preview content and Control UI preview types, and apply it in the iframe rendering/CSS so a single embed can request a wider responsive preview without changing the global `gateway.controlUi.chatMessageMaxWidth` behavior. Keep the existing iframe sandbox modes, URL allow rules, and external embed policy unchanged. Do not add arbitrary style passthrough or general custom CSS injection. Update public docs, system-prompt examples, focused UI/parser tests, and add a user-facing changelog entry under the active Changes section.
