# Coding Plan for openclaw/openclaw#70221: Feature request: Add width parameter to [embed] directive

Source report: [records/openclaw-openclaw/items/70221.md](records/openclaw-openclaw/items/70221.md)

## Summary

Keep open. Current main still has only a height hint for Control UI `[embed ...]`; the parser, normalized UI type, renderer, CSS, docs, and tests do not carry or apply a per-embed width value.

## Plan

Implement width support for Control UI `[embed ...]` directives as a bounded typed presentation hint. Add `preferredWidth` end to end in `src/chat/canvas-render.ts`, `ui/src/ui/types/chat-types.ts`, `ui/src/ui/chat/message-normalizer.ts`, `ui/src/ui/chat/tool-cards.ts`, and any sidebar canvas content path that preserves preview sizing. Parse numeric `width` from shortcode/structured preview using a conservative validator analogous to `preferredHeight`; do not allow arbitrary inline style passthrough, do not loosen external URL handling, and do not change iframe sandbox defaults. Update CSS so a preferred width can render wider than the chat column while remaining responsive. Add focused regression tests for shortcode parsing, normalization, grouped render/tool-card rendering, and prompt/docs examples. Update public docs and add a user-facing changelog entry because this adds a Control UI directive capability.
