## Summary

Keep open. Current main still lets `channels login` inherit the parent `channels` configured-channel plugin preload before the selected plugin auth flow runs, so the reported host-side pre-QR hang remains source-reproducible.

## Plan

Fix https://github.com/openclaw/openclaw/issues/62120 by keeping `openclaw channels login` and `openclaw channels logout` off broad pre-action configured-channel plugin preload. The fix should be channel-agnostic: add exact startup policy for login/logout or an equivalent command-policy change so explicit-channel auth uses `runChannelLogin()`/`runChannelLogout()` and `resolveInstallableChannelPlugin()` to load only the selected auth-capable channel, while preserving the existing implicit auto-selection behavior for omitted `--channel`. Do not add Weixin-specific core behavior or change the external `@tencent-weixin/openclaw-weixin` plugin. Add focused regression coverage for command path/startup/bootstrap behavior and channel-auth resolution, update the changelog because this is user-visible CLI behavior, and include live Weixin QR proof when credentials/device access are available.
