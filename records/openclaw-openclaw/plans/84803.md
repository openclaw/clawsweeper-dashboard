# Coding Plan for openclaw/openclaw#84803: WhatsApp group history drops media from unmentioned messages before later mention

Source report: [records/openclaw-openclaw/items/84803.md](records/openclaw-openclaw/items/84803.md)

## Summary

Keep open: current main and the latest release still record skipped WhatsApp group history as text-only, even though the shared channel-history media contract can carry bounded local image media into later turns.

## Plan

Fix the WhatsApp requireMention group-history media loss in `extensions/whatsapp/src/auto-reply/monitor`. When a group message is skipped for missing mention, record recent local image media from `WebInboundMsg.mediaPath`/`mediaType` using the existing channel-history media contract, then preserve that media through `resolveVisibleWhatsAppGroupHistory()` and `buildWhatsAppInboundContext()` into `InboundHistory`. Do not add a new retention mode, do not pass remote URLs or arbitrary non-image media into history, and keep allowlist/supplemental-context filtering in front of model visibility. Add focused regression tests for an unmentioned group image followed by a later mentioned text turn, plus filtering coverage that blocked group-history media does not reach the agent. Add a user-facing fix changelog entry if landing policy requires it.
