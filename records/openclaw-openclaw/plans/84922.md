# Coding Plan for openclaw/openclaw#84922: Bug: Gateway silently strips cron jobs with non-systemEvent/agentTurn payload.kind

Source report: [records/openclaw-openclaw/items/84922.md](records/openclaw-openclaw/items/84922.md)

## Summary

This should stay open: current `main` still has a source-level path where unsupported cron payload kinds are filtered out of the loaded in-memory store and later persistence can rewrite `jobs.json` without those jobs. The report is credible and narrow enough for a focused repair PR.

## Plan

Fix https://github.com/openclaw/openclaw/issues/84922. Current `main` filters persisted cron jobs whose `payload.kind` is not `systemEvent` or `agentTurn` during `ensureLoaded`, and later timer persistence can rewrite `jobs.json` without those records. Make runtime load/persist non-destructive for unsupported persisted payload kinds: it may skip/quarantine them for execution and warn, but it must preserve their raw job definitions on disk until `openclaw doctor --fix` or explicit user action migrates/removes them. Keep core runtime support limited to canonical cron payloads; do not implement broad arbitrary payload execution or solve the separate main-session shell-command issue at https://github.com/openclaw/openclaw/issues/63107. Add focused regression tests for an invalid/legacy payload kind surviving force reload plus a subsequent persist path, and update `CHANGELOG.md` if the repair is user-visible.
