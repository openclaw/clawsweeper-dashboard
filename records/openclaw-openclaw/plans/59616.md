# Coding Plan for openclaw/openclaw#59616: Config hot-reload does not watch $include-referenced files

Source report: [records/openclaw-openclaw/items/59616.md](records/openclaw-openclaw/items/59616.md)

## Summary

Keep open. Current main and latest release still watch only the active config path, while `$include` resolution and recursive include scanning exist but are not wired into the gateway reloader; the closed unmerged source PR at https://github.com/openclaw/openclaw/pull/59632 remains useful context.

## Plan

Fix https://github.com/openclaw/openclaw/issues/59616 by making `startGatewayConfigReloader` watch valid `$include`-referenced config files in addition to the active root config file. Reuse `collectIncludePathsRecursive()` rather than adding a new config option or broad directory watcher. Keep include path security semantics unchanged, update the dynamic Chokidar set after initial load and after accepted snapshots, unwatch removed include paths, and handle in-process-write/hash-dedupe cases so include-only edits still reload. Preserve the existing reload planning behavior and avoid redundant reload storms from `watcher.add()` emitting `add` events. Add focused tests in `src/gateway/config-reload.test.ts` for adding include watches, unwatching removed includes, nested includes or scanner failure tolerance, and the known feedback/dedupe edge cases from https://github.com/openclaw/openclaw/pull/59632. Update user-facing docs only if the behavior becomes documented, and add a `CHANGELOG.md` fix entry for the user-visible gateway config behavior. Do not implement the broader debounce/scoped-reload requests from https://github.com/openclaw/openclaw/issues/74122 in this PR.
