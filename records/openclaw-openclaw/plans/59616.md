# Coding Plan for openclaw/openclaw#59616: Config hot-reload does not watch $include-referenced files

Source report: [records/openclaw-openclaw/items/59616.md](records/openclaw-openclaw/items/59616.md)

## Summary

Keep open. Current `main` and the latest release still watch only the active root config file, while `$include` resolution and recursive include scanning exist but are not wired into the gateway config reloader.

## Plan

Fix https://github.com/openclaw/openclaw/issues/59616. Current `src/gateway/config-reload.ts` watches only `opts.watchPath`, so edits to `$include`-referenced config files do not trigger gateway config reload. Implement dynamic include-file watching inside `startGatewayConfigReloader` using `collectIncludePathsRecursive()` from `src/config/includes-scan.ts` and Chokidar `add` / `unwatch`. Derive watched include paths only from valid/accepted config snapshots, preferably `snapshot.parsed`, so include path guardrails from `src/config/includes.ts` are preserved. Sync on startup/initial accepted state, after successful external reload snapshots, and after in-process config writes where a current snapshot is available. Preserve the existing root-file hash dedupe behavior, but do not let the dedupe early-return prevent include-watch synchronization. Avoid a broad directory watcher and do not add a new config option. Account for the closed source PR discussion at https://github.com/openclaw/openclaw/pull/59632: avoid startup races, suppress or ignore dynamic `add` feedback loops, and cover include removal. Add focused regression tests in `src/gateway/config-reload.test.ts`; add a single-line `CHANGELOG.md` Fixes entry if this is a ClawSweeper-owned repair PR.
