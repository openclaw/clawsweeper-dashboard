# Coding Plan for openclaw/openclaw#84707: deleteAfterRun: true does not clean up isolated cron session files

Source report: [records/openclaw-openclaw/items/84707.md](records/openclaw-openclaw/items/84707.md)

## Summary

Keep open: current main still treats `deleteAfterRun` cleanup as a delivery-path side effect, so isolated cron runs that skip delivery or end through terminal error paths can leave transcript/session files for delayed retention cleanup instead of deleting them at run completion.

## Plan

Fix https://github.com/openclaw/openclaw/issues/84707. For isolated `agentTurn` cron jobs with `deleteAfterRun: true`, make transcript/session cleanup run when the isolated run reaches a terminal state, including `delivery.mode: "none"`, skipped/best-effort delivery paths, and runner error/abort paths after a transcript is created. Keep cleanup scoped to cron-owned isolated run/base session keys; do not add an ephemeral public config mode, do not change one-shot job deletion semantics, and preserve descendant/subagent safety from the direct-delivery cleanup surface. Add focused regression coverage for no-delivery and terminal-error cleanup; update adjacent delivery cleanup tests if the helper moves. Add a single user-visible changelog Fixes entry if policy requires it, without bot thanks.
