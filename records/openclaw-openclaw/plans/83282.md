# Coding Plan for openclaw/openclaw#83282: [Bug]: encoded trajectory export request can erase an explicit session key

Source report: [records/openclaw-openclaw/items/83282.md](records/openclaw-openclaw/items/83282.md)

## Summary

Keep open: current main still has the reported option-merge bug. The decoder materializes absent encoded fields as own properties, including `sessionKey: ""`, and then spreads the decoded object after explicit CLI options, so a partial encoded request can erase a direct session key before the required-session check.

## Plan

Fix https://github.com/openclaw/openclaw/issues/83282. In `src/commands/export-trajectory.ts`, prevent `decodeExportTrajectoryRequest` or `resolveExportTrajectoryOptions` from letting absent or blank decoded fields erase explicit direct CLI options. Preserve current behavior where non-empty encoded `sessionKey`, `output`, `store`, `agent`, or `workspace` values can set/override options, but do not materialize empty placeholders or `undefined` own properties that overwrite direct values. Add a focused regression in `src/commands/export-trajectory.test.ts` for direct `sessionKey` plus `requestJsonBase64` containing only another option such as `output`; it should continue with the explicit key and reach the existing missing-session lookup path, not the `--session-key is required` path. Keep the slash-command request builder unchanged unless the fix requires a compatible type adjustment. Add a concise `CHANGELOG.md` fix entry if landing policy treats this CLI behavior as user-visible.
