# Coding Plan for openclaw/openclaw#39037: WhatsApp channel: expose ctwa_clid from Click-to-WhatsApp ads in session metadata

Source report: [records/openclaw-openclaw/items/39037.md](records/openclaw-openclaw/items/39037.md)

## Summary

Keep open. Current `origin/main` still reaches WhatsApp `contextInfo` but has no CTWA or `externalAdReply` normalizer, while the existing untrusted structured metadata path can carry this data without a new core session schema.

## Plan

Implement the WhatsApp CTWA metadata feature requested in https://github.com/openclaw/openclaw/issues/39037. Add a bounded normalizer in the WhatsApp plugin that extracts Click-to-WhatsApp referral fields from Baileys `contextInfo.externalAdReply`, especially `ctwaClid`, plus safe string fields such as `sourceUrl`, `sourceType`, `sourceId`, `title`/`headline` when present. Carry the normalized payload through `WebInboundMessage.untrustedStructuredContext` so prompt assembly exposes it as untrusted metadata. Keep this plugin-owned; do not add durable session-level storage, config options, core migrations, or a broad `contextInfo` redesign. Add focused tests for extraction, WhatsApp dispatch forwarding, and prompt rendering. Consider but do not conflate adjacent open metadata work at https://github.com/openclaw/openclaw/issues/49099 and https://github.com/openclaw/openclaw/issues/69374. Add a `CHANGELOG.md` entry because this is user-visible WhatsApp behavior.
