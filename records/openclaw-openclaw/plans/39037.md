# Coding Plan for openclaw/openclaw#39037: WhatsApp channel: expose ctwa_clid from Click-to-WhatsApp ads in session metadata

Source report: [records/openclaw-openclaw/items/39037.md](records/openclaw-openclaw/items/39037.md)

## Summary

Keep open. Current main still drops WhatsApp Click-to-WhatsApp referral metadata: Baileys exposes the CTWA click-id fields under `contextInfo.externalAdReply`, OpenClaw extracts raw `contextInfo`, but the WhatsApp inbound normalization and prompt-context carrier never read or forward that payload.

## Plan

Fix https://github.com/openclaw/openclaw/issues/39037 by adding a narrow WhatsApp-owned Click-to-WhatsApp referral metadata pass-through. Read Baileys `contextInfo.externalAdReply` from inbound messages via the existing `extractContextInfo` path, and when `ctwaClid` is present, append an `untrustedStructuredContext` entry such as label `WhatsApp Click-to-WhatsApp referral`, source `whatsapp`, type `ctwa_referral`, and a sanitized payload with `ctwa_clid`, `source_url`, `source_type`, `source_id`, and title/headline/body fields where available. Preserve existing contact structured context entries instead of replacing them. Do not add durable session storage, a new core schema, plugin hook API fields, or logging of CTWA identifiers unless a maintainer explicitly asks for that broader design. Add focused regression tests for extraction, WhatsApp inbound dispatch, and prompt rendering, and add a user-facing changelog entry.
