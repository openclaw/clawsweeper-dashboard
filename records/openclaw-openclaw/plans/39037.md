# Coding Plan for openclaw/openclaw#39037: WhatsApp channel: expose ctwa_clid from Click-to-WhatsApp ads in session metadata

Source report: [records/openclaw-openclaw/items/39037.md](records/openclaw-openclaw/items/39037.md)

## Summary

Keep open. Current main and the latest release still extract WhatsApp Baileys contextInfo but do not normalize or expose Click-to-WhatsApp ad referral fields, and the related forwarding-metadata issue/PR do not cover ctwaClid.

## Plan

Implement WhatsApp CTWA referral metadata exposure for https://github.com/openclaw/openclaw/issues/39037. In the WhatsApp plugin, extract a bounded payload from Baileys proto.IContextInfo.externalAdReply when CTWA fields are present, including ctwaClid plus sourceUrl, sourceType, sourceId, title/headline/body/ref only when present. Thread it through the existing QueuedInboundMessage.untrustedStructuredContext path with a WhatsApp-specific label/source/type so buildWhatsAppInboundContext and buildInboundUserContextPrefix render it as untrusted metadata. Preserve existing WhatsApp contact structured context, mentions, replies, media, and location behavior. Do not add durable session storage, new config, core session schema, plugin hook API changes, or forwardingScore/isForwarded changes; those are separate from https://github.com/openclaw/openclaw/issues/69374 and https://github.com/openclaw/openclaw/pull/46291. Add focused tests for extraction, WhatsApp dispatch, and prompt rendering, and add a user-facing changelog entry for the active release section.
