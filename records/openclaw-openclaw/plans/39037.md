## Plan

Implement a bounded WhatsApp CTWA referral metadata pass-through for https://github.com/openclaw/openclaw/issues/39037. Add an extractor in the WhatsApp inbound code for `contextInfo.externalAdReply` fields including `ctwaClid`, `sourceUrl`, `sourceType`, `sourceId`, `title`/headline, and `body` where present; normalize/truncate strings conservatively, keep the payload explicitly untrusted, and do not log click IDs or ad URLs. Do not add a config option, core session schema migration, durable session storage, or plugin hook API change in this PR. Add focused regression tests for extraction, WhatsApp inbound dispatch carrier behavior, and prompt rendering; add a maintainer-owned changelog entry because this is user-facing metadata exposure.
