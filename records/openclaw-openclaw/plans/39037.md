## Summary

Keep open. Current main still has the same WhatsApp CTWA metadata gap: raw Baileys `contextInfo` is available, but no CTWA-specific reader or forwarding path puts `externalAdReply.ctwaClid` into the existing untrusted structured-context carrier for agents.

## Plan

Implement a narrow WhatsApp CTWA referral metadata feature for https://github.com/openclaw/openclaw/issues/39037. Extract `contextInfo.externalAdReply` from inbound Baileys messages when it contains CTWA ad referral data, especially `ctwaClid`, and include sanitized fields such as `ctwaClid`, `sourceUrl`, `sourceType`, `sourceId`, `title`/headline, and `body`/description in a WhatsApp-owned `untrustedStructuredContext` entry. Keep the data untrusted, avoid logging click IDs or URLs, append to existing contact structured context rather than overwriting it, and do not add a durable session metadata schema or config option. Add focused tests for extraction, WhatsApp inbound context forwarding, and prompt rendering; add a user-facing changelog entry under the active Changes section if the feature is implemented.
