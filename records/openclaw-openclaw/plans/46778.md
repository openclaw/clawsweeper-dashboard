## Summary

Keep open. ## Plan

Keep the change Feishu-local: update `extensions/feishu/src/dedupe-key.ts` or a nearby helper so non-media text messages can produce a conservative logical retry key from stable fields such as account namespace at call sites, sender id, chat/thread/root, `create_time`, message type, and normalized text content; preserve the current media-aware `message_id + file_key/image_key` behavior for audio/image/file/video/post media. Ensure receive-time claims, debounce suppression, and final persistent dedupe all use the same key. Add focused regression coverage for two same-content p2p text events with different `message_id` values and a guard that distinct media variants still remain distinct.
