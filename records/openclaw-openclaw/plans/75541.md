Source report: [records/openclaw-openclaw/items/75541.md](records/openclaw-openclaw/items/75541.md)

## Summary

Keep open. Current main still trusts a truthy context-engine assemble result without validating that `messages` is an array, so a malformed plugin result can still replace session history with `undefined` and crash later prompt diagnostics; the linked RFC is related but broader and does not supersede this narrow crash fix.

## Plan

Fix https://github.com/openclaw/openclaw/issues/75541. Current main trusts `ContextEngine.assemble()` output and can assign `assembled.messages` into session state even when it is not an array, causing prompt diagnostics to crash on `.length`. Add a generic runtime validator or guard in the shared context-engine assemble boundary and the Codex/embedded-runner consumption paths as needed so malformed assemble results are warned about and ignored, preserving the original pipeline messages without mutating session state. Add regression coverage with a fake context engine returning a truthy result whose `messages` field is undefined or non-array. Keep the public context-engine API backward-compatible; do not add lossless-claw-specific handling, a new config option, or the broader provenance/event API from the related RFC. Add a user-facing changelog fix entry unless the final patch is test-only/internal.
