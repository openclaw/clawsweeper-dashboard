## Summary

Keep open. Current main still accepts a truthy context-engine assemble result without checking that `messages` is an array, so malformed plugin output can still replace session history with `undefined` and later crash prompt diagnostics.

## Plan

Fix https://github.com/openclaw/openclaw/issues/75541 by validating context-engine assemble results before they can replace session history. Add a generic guard, preferably in `assembleHarnessContextEngine()` or the immediate shared runner boundary, that treats a missing/non-array `messages` field as an assemble failure, logs a warning, and keeps the original pipeline messages. Add focused regression coverage with a fake context engine returning a truthy result whose `messages` is `undefined`, covering the embedded runner path and Codex harness projection if the shared helper is changed. Do not implement the broader metadata/provenance contract from https://github.com/openclaw/openclaw/issues/82137 in this repair. Add a user-facing changelog fix entry.
