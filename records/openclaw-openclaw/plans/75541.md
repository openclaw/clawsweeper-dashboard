## Summary

Keep open. Current `main` still trusts a truthy context-engine assemble result without validating that `messages` is an array, so a malformed plugin result can still corrupt the session message state and crash later in prompt diagnostics.

## Plan

Fix https://github.com/openclaw/openclaw/issues/75541 as a narrow plugin-boundary bug. Add a generic runtime guard for context-engine `assemble()` results so a malformed result with missing or non-array `messages` does not replace session state or crash prompt diagnostics; warn and fall back to the original pipeline/baseline messages. Prefer a shared validator near `src/agents/harness/context-engine-lifecycle.ts` or the immediate consumers so both embedded runner and Codex harness behavior are covered. Do not add lossless-claw-specific handling, do not add a config option, and do not change the public context-engine contract except for defensive runtime validation/error text. Add regression coverage with a fake context engine returning malformed `messages`, and include a user-facing changelog entry because the fix prevents live runs from failing before provider calls. Related context: https://github.com/openclaw/openclaw/pull/74255 fixed prompt-authority overflow behavior but did not validate assemble-result shape; https://github.com/openclaw/openclaw/issues/66591 was an older factory-validation issue with a different root cause.
