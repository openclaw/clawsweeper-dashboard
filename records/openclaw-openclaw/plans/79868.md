## Summary

Keep open. Current main still has a source-level path where an auto-persisted fallback selection can become the next embedded run's starting model, so the configured primary is bypassed without producing the existing failed-primary `model fallback decision` log.

## Plan

Fix https://github.com/openclaw/openclaw/issues/79868 by adding explicit telemetry when an embedded run starts from a persisted auto fallback selection and thereby bypasses the configured primary. Keep the existing fallback persistence behavior, do not reorder model candidates, and do not log explicit `modelOverrideSource: "user"` or legacy user overrides as fallback. Prefer a small source-level change in the model fallback or auto-reply runner boundary that emits `model fallback decision` or equivalent `model.fallback_step` evidence for reused auto fallback selections, including requested/configured primary and actual candidate when available. Add regression coverage for an auto fallback override starting on the fallback model with no failed primary attempt, plus coverage that user overrides stay strict and unreported as fallback.
