## Summary

Keep open. Current main still has a source-level path where an auto-persisted fallback provider becomes the first embedded-run candidate, so the existing failed-primary `model fallback decision` log path never fires even though the configured primary is being bypassed.

## Plan

Fix https://github.com/openclaw/openclaw/issues/79868 by adding explicit gateway fallback telemetry when an embedded auto-reply run starts from a session model override whose `modelOverrideSource` is `auto` and whose provider/model differs from the configured primary. Keep fallback persistence and routing behavior unchanged. Do not log explicit user or legacy model overrides as fallback. Prefer reusing the existing model fallback decision / fallback-step observation shape so operators can grep `model fallback decision` or an equivalent structured event and trajectory consumers can reconstruct that an auto fallback was reused. Keep logs sanitized; do not persist raw provider errors, tokens, request IDs, or secrets. Add focused regression coverage for: auto fallback reuse emits telemetry, user/legacy overrides do not, normal primary success remains quiet, and existing failed-candidate fallback logging still works. Add a changelog entry if the change is considered operator-visible.
