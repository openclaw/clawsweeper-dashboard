## Summary

Keep open. Current main has improved fallback visibility for no-reply outcomes, but a persisted auto fallback can still start a later embedded run on the fallback provider without emitting a `model fallback decision` gateway log for a successful visible reply.

## Plan

Fix https://github.com/openclaw/openclaw/issues/79868 by adding gateway-visible fallback telemetry when an embedded run uses an auto-persisted fallback provider/model as the active candidate and the selected/configured origin differs. Keep the change narrow: do not change fallback candidate ordering, sticky fallback persistence, user `modelOverrideSource: "user"` behavior, or the open rollback/TTL policy in https://github.com/openclaw/openclaw/pull/75270. Reuse the existing `model-fallback/decision` logger or an equivalent existing fallback observation path with sanitized, bounded fields. Add regression coverage for a persisted auto fallback session that produces a visible reply and currently has no failed-primary attempt, plus coverage that explicit user overrides do not emit automatic fallback telemetry. No changelog is needed unless the user-visible logging contract changes materially.
