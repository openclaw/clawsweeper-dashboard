## Summary

Keep open. Current main still has the direct OpenAI-compatible penalty passthrough gap: config accepts these params and preserves them, but the stream wrapper and OpenAI-compatible payload only map temperature, top_p, token caps, and response format; the prior implementation PRs are closed unmerged.

## Plan

Implement https://github.com/openclaw/openclaw/issues/32496. Support direct `frequency_penalty` and `presence_penalty` from `agents.defaults.params`, `agents.defaults.models[provider/model].params`, and `agents.list[].params` for OpenAI-compatible completions providers. Add finite-number extraction/canonicalization in `src/agents/pi-embedded-runner/extra-params.ts` and carry those values through `BaseStreamOptions`/`buildOpenAICompletionsParams` in `src/agents/openai-transport-stream.ts`. Preserve existing `params.extra_body` final-precedence behavior, keep runtime option precedence intact, and do not broaden scope to `top_k`, `repetition_penalty`, or raw passthrough without maintainer approval. Add focused regression coverage and update docs/changelog for the user-facing config support.
