# Coding Plan for openclaw/openclaw#32496: [Feature]:  Support frequency_penalty and presence_penalty Parameters for OpenAI-Compatible Providers

Source report: [records/openclaw-openclaw/items/32496.md](records/openclaw-openclaw/items/32496.md)

## Summary

Keep open. Current main still accepts arbitrary model `params`, but it does not directly normalize or forward `frequency_penalty` or `presence_penalty` on the OpenAI-compatible transport path; `params.extra_body` is only a workaround, and the linked implementation PRs were closed unmerged.

## Plan

Fix https://github.com/openclaw/openclaw/issues/32496 as a narrow OpenAI-compatible model-params passthrough. Implement finite-number forwarding for `params.frequency_penalty` and `params.presence_penalty` from `agents.defaults.params`, `agents.defaults.models["provider/model"].params`, and `agents.list[].params` into OpenAI-compatible completions payloads. Use the existing extra-params wrapper/payload-patch style, preserve `params.extra_body` as the vendor-specific escape hatch and final override, and do not add broad `top_p`, `top_k`, `repetition_penalty`, or a new `generation` schema in this PR. Add regression coverage for direct pass-through, non-finite/non-number rejection, extra_body override ordering, and non-OpenAI-compatible route non-leakage. Update user-facing docs and CHANGELOG because this adds documented params support.
