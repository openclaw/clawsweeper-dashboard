## Summary

Keep open. Current main still accepts arbitrary model `params` but does not directly normalize or forward `frequency_penalty` or `presence_penalty` on the OpenAI-compatible transport path; users can only work around this today with `params.extra_body`, and the linked implementation PRs were closed unmerged.

## Plan

Fix https://github.com/openclaw/openclaw/issues/32496 with a narrow direct passthrough for OpenAI-compatible penalty params. Add finite-number handling for `frequency_penalty` and `presence_penalty` from accepted model/agent `params` so OpenAI-compatible Completions payloads receive those exact snake_case keys. Preserve existing precedence from `resolveExtraParams`, preserve `extra_body` as the lower-level escape hatch, and do not add a new public `generation` block or broaden into `top_k`/`repetition_penalty` unless the code already has a tightly scoped helper that makes this unavoidable. Add/update regression tests for direct-key passthrough, invalid non-number rejection, precedence/override behavior, and interaction with existing `extra_body` payload patching. Update docs/changelog if the behavior is user-facing. Do not change provider-specific Ollama native option behavior except for tests proving it remains separate.
