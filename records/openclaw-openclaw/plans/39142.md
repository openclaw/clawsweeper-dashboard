# Coding Plan for openclaw/openclaw#39142: Configurable user-facing message when all models are exhausted

Source report: [records/openclaw-openclaw/items/39142.md](records/openclaw-openclaw/items/39142.md)

## Summary

Keep open: current main has safer built-in fallback text for terminal billing/rate-limit exhaustion, but it still does not expose a supported configurable `messages.modelExhausted` override. I also did not find an open PR or canonical issue that implements this exact config surface.

## Plan

Implement https://github.com/openclaw/openclaw/issues/39142 as a focused feature. Add an optional `messages.modelExhausted?: string` config key across message config types, strict zod schema, help/labels, docs, and changelog. In `src/auto-reply/reply/agent-runner-execution.ts`, use the configured text only when a terminal `FallbackSummaryError` exhausts all attempts for billing/rate-limit style reasons; preserve existing built-in billing, rate-limit, context, role-ordering, Control UI, and generic text when the key is unset or failures are unrelated. Do not implement the broader proactive retry/failover/recovery notification policy from https://github.com/openclaw/openclaw/issues/8130 and do not depend on the plugin-hook PR at https://github.com/openclaw/openclaw/pull/70990. Add regression tests for pure billing, pure rate-limit, mixed billing/rate-limit, and unrelated fallback failures, proving raw `All models failed` details remain hidden.
