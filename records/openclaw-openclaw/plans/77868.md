## Summary

Keep open: current main still lacks a generic plugin API compatibility gate for npm/local installs and non-bundled runtime loading, so a newer official plugin can pass the older `minHostVersion` check even when `openclaw.compat.pluginApi` requires a newer core. The Discord helper mismatch is source-backed, the invalid `visibleReplies` part is already covered by schema validation, and the follow-up msteams comment points to the same remaining cross-plugin gap.

## Plan

Fix https://github.com/openclaw/openclaw/issues/77868 by enforcing `openclaw.compat.pluginApi` for non-ClawHub plugin paths. Add a generic compatibility check to npm/local plugin install and non-bundled manifest registry loading so packages whose plugin API range exceeds the current OpenClaw runtime fail closed with a clear diagnostic before runtime code loads. Reuse the existing ClawHub plugin API range semantics, including CalVer correction normalization, rather than adding a Discord-specific fallback.
