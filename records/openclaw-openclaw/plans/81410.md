# Coding Plan for openclaw/openclaw#81410: Gateway lifecycle commands target stale SUDO_USER scope from root shell instead of root systemd user service

Source report: [records/openclaw-openclaw/items/81410.md](records/openclaw-openclaw/items/81410.md)

## Summary

Keep open: current main and the latest release still preserve the effective-root plus non-root `SUDO_USER` path that targets the sudo caller's user manager, while the report is specifically about a root-owned user unit under `/root`. The June 15 follow-up comment adds useful before/after live evidence, and the linked fix PRs for this issue are closed unmerged.

## Plan

Fix this issue in OpenClaw core: gateway lifecycle/status commands run as effective root with `HOME=/root`, root user-bus env, and a root-owned user unit under `/root/.config/systemd/user` must not target a stale non-root `SUDO_USER` systemd user manager. Keep the change bounded to the systemd user-scope resolver and immediate lifecycle/status callers as needed. Preserve valid sudo-to-root behavior when the sudo caller's user unit is genuinely the intended target; do not add a config flag or environment variable, and do not weaken the existing system-scope unit detection from https://github.com/openclaw/openclaw/pull/87618. Add focused regression coverage for the reported root-owned user-unit stale-sudo case, for a valid sudo-user unit case, and for status/runtime or lifecycle command paths that currently call `execSystemctlUser`. Do not edit `CHANGELOG.md`.
