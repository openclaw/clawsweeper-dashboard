Current main still exposes Telegram outbound interactive controls as inline keyboards only, while native reply keyboards and reply-keyboard removal are absent from the Telegram send/action/message-tool surfaces; the related closed items cover inline-button behavior, not this separate capability.

## Plan

Implement Telegram-native reply keyboard support for https://github.com/openclaw/openclaw/issues/64715.
