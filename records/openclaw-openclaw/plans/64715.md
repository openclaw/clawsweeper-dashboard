# Coding Plan for openclaw/openclaw#64715: Telegram: add native reply keyboard support to the message/agent send surface

Source report: [records/openclaw-openclaw/items/64715.md](records/openclaw-openclaw/items/64715.md)

## Summary

Keep this open. Current main still routes Telegram outbound controls through inline keyboards only; the send/action/message-tool surfaces and docs do not expose Telegram native reply-keyboard rows or keyboard-removal options, and the related closed items are about inline buttons or callback behavior rather than this separate capability.

## Plan

Implement native Telegram reply keyboard support for the agent/message outbound send surface. Add an additive, typed Telegram-owned reply-keyboard/reply_markup input covering rows/buttons, keyboard removal, resize_keyboard, and one_time_keyboard; wire it through Telegram message action discovery/schema, action runtime, outbound adapter/send path, and tests without changing existing inline-button behavior. Update docs for when to use inline buttons versus reply keyboards/removal. Do not broaden the portable message presentation contract unless the existing implementation makes that necessary; keep the feature owned by the Telegram plugin and avoid touching unrelated channel behavior.
