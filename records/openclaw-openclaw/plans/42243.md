## Summary

Keep open. Current main still converts fatal isolated cron failures into `isError` delivery payloads and passes them through normal announce delivery, while the separate failure-notification path does not suppress the completion announce payload.

## Plan

Fix https://github.com/openclaw/openclaw/issues/42243. Current main turns fatal isolated cron outcomes from `resolveCronPayloadOutcome` into `isError` delivery payloads, then `runCronIsolatedAgentTurn` still passes those payloads into `dispatchCronDelivery`, where they can be sent as normal announce output. Add a structured fatal-run skip before normal completion announce dispatch, likely by passing explicit fatal/error state into the dispatch boundary and returning a failed-run result with no normal result delivery. Preserve run status, run history, diagnostics, `delivery.failureDestination`, global `cron.failureDestination`, and primary-channel failure notifications. Do not implement a raw-text regex classifier for provider errors. Add a regression test proving fatal error payloads are not normal-announced, plus coverage that successful output and separate failure notifications still work. Add a changelog fix entry under the active version unless maintainer policy says the landing flow will add it.
