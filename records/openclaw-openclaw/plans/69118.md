## Summary

Keep open. Current main still puts turn-scoped group intro and other static prompt overlays into the Claude CLI session-reuse hash, and the resolver still drops the saved session when that stored hash differs from the next turn; recent comments confirm the same user-visible reset on v2026.5.7.

## Plan

Fix the canonical `extraSystemPromptHash` false-invalidation bug tracked by https://github.com/openclaw/openclaw/issues/69118. Preserve strict invalidation for auth profile/epoch and genuinely durable runtime identity such as stable prompt context, model/workspace/live-session/MCP boundaries. Add focused regression tests for first-to-second group-channel reuse and subagent announce/static-prompt drift; do not add a user-facing config option or weaken auth/MCP checks.
