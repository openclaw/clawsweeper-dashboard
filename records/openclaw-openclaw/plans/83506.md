# Coding Plan for openclaw/openclaw#83506: [Bug]: QMD archived session hits can be dropped by visibility filter due to normalized path mapping

Source report: [records/openclaw-openclaw/items/83506.md](records/openclaw-openclaw/items/83506.md)

## Summary

Keep open. Current main has the visibility filter and prior dotted archive handling, but the shared transcript-hit helper still treats a QMD-normalized archived Markdown basename as an ordinary `.md` stem, so the later session-key lookup can fail and drop the hit; the broader related report remains open, while this narrower path-mapping bug is actionable on its own.

## Plan

Fix https://github.com/openclaw/openclaw/issues/83506. Current bug: QMD can return a session hit path like `qmd/sessions-main/<session-id>-jsonl-reset-<timestamp>.md` for an exported archive named `<session-id>.jsonl.reset.<timestamp>.md`; `extractTranscriptIdentityFromSessionsMemoryHit()` treats that as a non-archived Markdown stem, `resolveTranscriptStemToSessionKeys()` finds no session key, and the visibility filter drops the hit. Implement canonical parsing in `src/plugin-sdk/session-transcript-hit.ts` or a narrowly adjacent helper so QMD-normalized reset/deleted archived stems map back to the original session id. Preserve existing dotted archive handling, owner metadata, fail-closed behavior, `tools.sessions.visibility`, sandbox clamps, QMD export defaults, and agent-to-agent policy. Add regression tests for the shared helper and `filterMemorySearchHitsBySessionVisibility()`; update memory-wiki coverage if the shared helper change affects that caller. Do not turn this into a product change for broader session recall. Add a user-facing changelog entry during landing because this restores archived session recall.
