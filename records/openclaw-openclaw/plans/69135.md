# Coding Plan for openclaw/openclaw#69135: `gateway probe`: false positive \"multiple reachable gateways\" when SSH tunnel + `remote.url` hit the same gateway

Source report: [records/openclaw-openclaw/items/69135.md](records/openclaw-openclaw/items/69135.md)

## Summary

Keep open. Current main and the latest shipped release still build both the SSH tunnel and configured remote targets, then emit `multiple_gateways` from reachable target count alone, so the reported false positive remains a narrow CLI warning bug.

## Plan

Fix https://github.com/openclaw/openclaw/issues/69135. In `openclaw gateway probe`, do not emit the `multiple_gateways` warning when an SSH tunnel target and the configured remote URL both reach the same logical gateway identity. Prefer fixing `buildGatewayStatusWarnings` or a small helper around it using `GatewayStatusProbedTarget.self` from `pickGatewaySelfPresence`; preserve the existing warning when reachable targets have distinct non-null identities or when identity is missing/mixed enough that distinct gateways cannot be ruled out. Do not skip the configured remote target, because the direct-path check is useful. Add focused regression coverage in `src/commands/gateway-status/output.test.ts` for two reachable targets with the same `self` identity not producing `multiple_gateways`, and for distinct identities still producing it. Update docs only if warning copy changes. Add a changelog entry because this is a user-visible CLI warning fix.
