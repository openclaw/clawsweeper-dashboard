# Coding Plan for openclaw/openclaw#69135: `gateway probe`: false positive \"multiple reachable gateways\" when SSH tunnel + `remote.url` hit the same gateway

Source report: [records/openclaw-openclaw/items/69135.md](records/openclaw-openclaw/items/69135.md)

## Summary

Keep open. Current main still emits `multiple_gateways` from the count of reachable probe targets, so an SSH tunnel plus configured direct URL can still warn about multiple gateways even when both probes report the same logical gateway identity.

## Plan

Fix https://github.com/openclaw/openclaw/issues/69135 by making `openclaw gateway probe` distinguish multiple transports to the same gateway from multiple distinct gateways. In `src/commands/gateway-status/output.ts`, classify reachable probes by a conservative non-null `entry.self` identity such as normalized host plus IP; suppress or downgrade `multiple_gateways` when all reachable entries share that identity, and keep warning conservatively when identities are distinct or unavailable. Do not skip the configured remote target when an SSH tunnel is active. Add focused tests in `src/commands/gateway-status/output.test.ts` for same-identity `sshTunnel` plus `configRemote`, distinct identities, and null identity behavior; update `docs/cli/gateway.md` and `CHANGELOG.md` if the user-visible warning text changes.
