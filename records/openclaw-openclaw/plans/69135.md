# Coding Plan for openclaw/openclaw#69135: `gateway probe`: false positive \"multiple reachable gateways\" when SSH tunnel + `remote.url` hit the same gateway

Source report: [records/openclaw-openclaw/items/69135.md](records/openclaw-openclaw/items/69135.md)

## Summary

Keep open. Current main still emits the warning from the count of reachable probe targets, so an SSH tunnel plus configured direct URL can still be misclassified as multiple gateways even when both probes identify the same logical gateway.

## Plan

Fix https://github.com/openclaw/openclaw/issues/69135. In `openclaw gateway probe`, avoid the false `multiple_gateways` warning when multiple reachable targets are different transports to the same gateway identity. Use the existing `GatewayStatusProbedTarget.self` data conservatively: suppress or downgrade the warning only when all reachable targets have the same non-null identity; preserve `multiple_gateways` for distinct identities and for unknown/null identities unless a clearly safer policy is justified. Do not skip the configured remote probe merely because an SSH tunnel is active. Add focused regression coverage for same-identity reachable probes and for distinct/unknown identities still warning. Update user-facing gateway docs if warning wording changes, and add a `CHANGELOG.md` fix entry for the CLI warning behavior. Avoid dependency/config changes.
