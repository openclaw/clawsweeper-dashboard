# Coding Plan for openclaw/openclaw#69135: `gateway probe`: false positive \"multiple reachable gateways\" when SSH tunnel + `remote.url` hit the same gateway

Source report: [records/openclaw-openclaw/items/69135.md](records/openclaw-openclaw/items/69135.md)

## Summary

Keep open. Current main and the latest release still classify `multiple_gateways` from the count of reachable probe targets, so an SSH tunnel plus configured direct URL can still be reported as multiple gateways even when both probes identify the same logical gateway.

## Plan

Fix https://github.com/openclaw/openclaw/issues/69135. `openclaw gateway probe` currently emits `multiple_gateways` when two reachable probe targets are the SSH tunnel and configured remote URL for the same logical gateway. In `src/commands/gateway-status/output.ts`, classify reachable targets by a conservative non-null gateway identity from `GatewayStatusProbedTarget.self` and suppress or downgrade the multiple-gateways warning when all reachable targets share that identity. Preserve the warning when identities are distinct or when identity is missing/insufficient. Do not skip the configured remote target solely because an SSH tunnel is active; the direct-path probe is useful. Add focused regression coverage for same-identity multi-transport, distinct identities, and null/unknown identity behavior. Update docs only if warning copy changes, and add a `CHANGELOG.md` fix entry because this is user-visible CLI diagnostic behavior.
