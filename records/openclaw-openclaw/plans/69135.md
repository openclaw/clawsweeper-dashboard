# Coding Plan for openclaw/openclaw#69135: `gateway probe`: false positive \"multiple reachable gateways\" when SSH tunnel + `remote.url` hit the same gateway

Source report: [records/openclaw-openclaw/items/69135.md](records/openclaw-openclaw/items/69135.md)

## Summary

Keep open. Current main still classifies `multiple_gateways` by the number of reachable probe targets, while SSH tunnel probing keeps a differently addressed configured remote URL, so the reported one-gateway/two-transport case can still get a misleading warning.

## Plan

Fix the false `openclaw gateway probe` `multiple_gateways` warning when an SSH tunnel target and configured `gateway.remote.url` are both reachable but report the same logical gateway identity. Keep probing both transports; update `buildGatewayStatusWarnings` so reachable targets sharing a conservative non-null `self` identity do not produce the multiple-gateways warning, while distinct identities and unknown/null identities still preserve the existing warning behavior. Add focused regression coverage in `src/commands/gateway-status/output.test.ts` and, if needed, command-level coverage in `src/commands/gateway-status.test.ts`. Update user-facing docs or warning copy only if behavior/copy changes, and add a changelog fix entry. Do not add a config option, skip the direct remote probe, or weaken real multi-gateway detection.
