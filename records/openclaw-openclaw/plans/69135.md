# Coding Plan for openclaw/openclaw#69135: `gateway probe`: false positive \"multiple reachable gateways\" when SSH tunnel + `remote.url` hit the same gateway

Source report: [records/openclaw-openclaw/items/69135.md](records/openclaw-openclaw/items/69135.md)

## Summary

Keep open. Current main still emits `multiple_gateways` solely from the count of reachable probe targets, so the reported SSH tunnel plus configured direct URL path can still warn about multiple gateways even when both targets identify the same logical gateway.

## Plan

Fix https://github.com/openclaw/openclaw/issues/69135. In `openclaw gateway probe`, avoid the false `multiple_gateways` warning when multiple reachable targets are just different transports to the same gateway. Keep the existing SSH tunnel plus configured remote probing behavior. Implement a conservative identity-aware classification in `src/commands/gateway-status/output.ts`: if all reachable targets with usable non-null `self` identity resolve to one gateway identity, suppress or downgrade the warning; keep `multiple_gateways` when reachable identities are distinct, unknown, or mixed in a way that cannot prove sameness. Add focused tests in `src/commands/gateway-status/output.test.ts` for same-identity suppression and distinct/unknown identity warning preservation. Update `docs/cli/gateway.md` warning-code copy and add a user-facing fix entry to `CHANGELOG.md`. Do not add a new config option or drop the configured remote target when SSH tunnel probing is active.
