## Summary

Keep open. Current main and the latest release still classify `multiple_gateways` by reachable probe target count, while SSH tunnel probing can keep a differently addressed configured remote URL to the same gateway, so the reported false positive remains actionable.

## Plan

Fix https://github.com/openclaw/openclaw/issues/69135. `openclaw gateway probe` should not emit the `multiple_gateways` warning when multiple reachable targets are only different transports to the same gateway identity, such as an SSH tunnel URL plus `gateway.remote.url`. Keep probing both transports; do not remove the configured remote target. In `src/commands/gateway-status/output.ts`, classify reachable targets by a conservative non-null `self` identity from `pickGatewaySelfPresence` and keep warning when identities are distinct or unknown. Add regression coverage for same-identity reachable targets and for distinct/unknown identities still warning. Update CLI/gateway docs only if the warning semantics or copy changes, and add a `CHANGELOG.md` fix entry for the user-visible CLI diagnostic change. Do not change gateway auth, target resolution, tunnel setup, or provider/channel behavior.
