# Coding Plan for openclaw/openclaw#54409: [Bug]: Feishu plugin: per-chat serial queue prevents messages.queue.mode = \"collect\" from batching queued messages

Source report: [records/openclaw-openclaw/items/54409.md](records/openclaw-openclaw/items/54409.md)

## Summary

Keep open. Current main and the latest release still route ordinary same-chat Feishu messages through the Feishu sequential queue before the shared active-run queue can observe them, so `messages.queue.mode: "collect"` can still miss mid-run followups and process them as separate turns.

## Plan

Fix https://github.com/openclaw/openclaw/issues/54409 by making ordinary same-chat Feishu messages that arrive while a prior same-key agent run is active reach the shared active-run followup/collect queue instead of waiting behind the full Feishu sequential task. Keep the fix inside the Feishu plugin or the documented channel reply seam; do not change global queue policy unless a focused failing test proves the shared policy is wrong. Preserve Feishu FIFO for normal idle cadence, dedupe/processing-claim behavior, inbound debounce merging, `:control` and `:btw` out-of-band lanes, and the task-timeout guard added for https://github.com/openclaw/openclaw/issues/70133. Add focused regression coverage for collect-mode active-run followups and update changelog if the user-visible behavior changes.
