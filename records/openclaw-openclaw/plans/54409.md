# Coding Plan for openclaw/openclaw#54409: [Bug]: Feishu plugin: per-chat serial queue prevents messages.queue.mode = \"collect\" from batching queued messages

## Summary

Keep open. Current main and the latest shipped release still route ordinary same-chat Feishu messages through the sequential queue before the gateway can observe active-run state, so collect-mode follow-ups can still be seen as separate `run-now` turns; the merged Feishu timeout fix explicitly scoped this behavior out.

## Plan

Fix https://github.com/openclaw/openclaw/issues/54409. Reproduce the source path where Feishu queues ordinary same-chat messages behind a full `handleMessage`/agent turn, causing `messages.queue.mode = "collect"` follow-ups to reach the gateway after the active run is already inactive. Implement the fix in the Feishu ingress/dispatch boundary so ordinary same-chat messages sent during an active run can reach the shared follow-up collect queue, while preserving Feishu FIFO for normal cadence, dedupe, inbound debounce merging, `:control` and `:btw` out-of-band lanes, and the task-timeout guard from https://github.com/openclaw/openclaw/pull/76687. Do not change global queue semantics or add a new config option unless the existing contract proves insufficient. Add focused regression coverage and a user-facing changelog fix entry.
