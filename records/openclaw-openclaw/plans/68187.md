## Summary

Keep open. Current main and the latest release still route SSE-backed bundled MCP tool calls through a cached client with no stale-session catch, reconnect, or retry path, and the related open PRs cover streamable-http recovery or unexpected transport-close eviction rather than the SSE POST `Session not found` failure.

## Plan

Fix https://github.com/openclaw/openclaw/issues/68187. In `createSessionMcpRuntime`, add bounded stale-session recovery for `transportType === "sse"`: classify known MCP SDK SSE POST lost-session errors such as `Error POSTing to endpoint (HTTP 400/404): ... Session not found`, dispose the stale client/transport, rebuild that server session through the existing `resolveMcpTransport` and `connectWithTimeout` path, refresh catalog metadata for that server when needed, serialize concurrent reconnects, and retry the original `callTool` once. Keep stdio unchanged, avoid broad generic HTTP 4xx retry behavior, and do not add startup backoff/circuit policy in this fix.
