## Summary

Keep open: current main and the latest release still advertise raw `buttons=[[...]]` inline-button syntax to agents, while the current Telegram message-tool schema exposes `presentation.blocks[].buttons` and explicitly omits a top-level `buttons` property. No canonical merged or closing PR fixes this; the open group-prompt PR is useful adjacent work but only changes group reply guidance.

## Plan

Update the focused prompt tests to assert the new wording and keep Slack-specific interactive hints untouched.
