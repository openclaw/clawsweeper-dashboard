## Summary

Keep open: current main and the latest release still route `sessions_send` target turns as generic `inter_session` data, still expose broad skip-token exits, and do not give explicit allowed channel-delivery requests a higher-precedence path over `REPLY_SKIP`, `ANNOUNCE_SKIP`, or `NO_REPLY`.

## Plan

Fix https://github.com/openclaw/openclaw/issues/49223 as a narrow sessions_send bug. Current main routes target turns as `inter_session`, gives the receiver broad `REPLY_SKIP` / `ANNOUNCE_SKIP` / `NO_REPLY` exits, and lacks a guard that makes explicit allowed channel-delivery requests produce a normal visible message. Update the existing sessions_send prompt/runtime contract so explicit delivery requests to the target channel outrank generic coordination skip heuristics while coordination-only chatter can still use skip tokens. Add focused regression coverage around sessions_send A2A delivery intent and forbidden control-token suppression. Do not add a public `mode`/schema/API field unless maintainers explicitly choose that product direction; prefer a bounded internal prompt/runtime guard.
