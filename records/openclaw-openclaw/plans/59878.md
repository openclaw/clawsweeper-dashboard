## Summary

Keep open. Current main has fixed adjacent diagnostic recovery and active-restart marking, but the exact stale-row path still remains: `sessions.abort` can return `no-active-run` without healing the persisted `running` row, and restart recovery still skips running rows that were never marked `abortedLastRun`.

## Plan

Fix the stale session-row recovery gap tracked by https://github.com/openclaw/openclaw/issues/59878. Implement conservative reconciliation so `sessions.abort` no-active outcomes can heal a persisted `running` row when there is no live run for that session, and add bounded startup/restart recovery for markerless stale running rows only when active-run and lane facts prove the work is not live. Preserve the existing active restart marker fix from https://github.com/openclaw/openclaw/issues/82433 and the diagnostic recovery behavior from https://github.com/openclaw/openclaw/issues/71127; do not fold in broader model/tool stall policy or the deferred lifecycle-error issue tracked by https://github.com/openclaw/openclaw/issues/63819 unless the exact path is proven. Add focused regression coverage for the no-active `sessions.abort` path and markerless restart skip becoming safe recovery, and include a user-facing changelog fix entry.
