# Coding Plan for openclaw/openclaw#54565: Feature request: per-group debounceMs for WhatsApp (and other channels)

Source report: [records/openclaw-openclaw/items/54565.md](records/openclaw-openclaw/items/54565.md)

## Summary

Keep open: current main and the latest release still do not accept or resolve WhatsApp group-level `debounceMs`, and the related composing-aware debounce PR does not cover this requested per-group override.

## Plan

Implement WhatsApp-only per-group `debounceMs` support for `channels.whatsapp.groups.<jid>` and account-level group maps. Add `debounceMs` to the WhatsApp group config type, strict zod schema, UI/config metadata, generated docs, and focused schema tests. In runtime, resolve debounce per inbound message using exact group entry before wildcard, respecting existing account/root group map replacement semantics and existing account/channel/global fallback order. Use the shared debouncer's item-level `resolveDebounceMs` hook instead of adding a second debounce system. Preserve existing channel/account defaults and do not implement Telegram/Discord parity in this PR. The reported image/image/text use case must not drop earlier attachments: either preserve all batched media/context through the inbound message context or leave media bypass behavior unchanged and document that limitation in the PR for maintainer review. Add a user-facing changelog entry because this adds config behavior.
