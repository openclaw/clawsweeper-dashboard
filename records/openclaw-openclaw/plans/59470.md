# Coding Plan for openclaw/openclaw#59470: Bug: openclaw gateway call times out for plugin gateway methods even when the method executes successfully

Source report: [records/openclaw-openclaw/items/59470.md](records/openclaw-openclaw/items/59470.md)

## Summary

Keep open. Current main and the latest release still await plugin gateway handlers without adapting returned values into RPC responses, so a return-style plugin method can complete its side effect while `openclaw gateway call` waits until its timeout.

## Plan

Fix the plugin gateway method return-value timeout reported in https://github.com/openclaw/openclaw/issues/59470. In `src/gateway/server-methods.ts`, adapt plugin gateway handler return values into successful RPC responses when the handler returns a JSON-compatible value and did not already call `respond(...)`; preserve all existing callback-style `respond(...)` behavior and accepted/final multi-response flows. Update the public handler type/docs if needed so the contract is explicit, add focused regression coverage for a return-style plugin handler invoked through `handleGatewayRequest` or the closest existing gateway/plugin RPC test seam, and add a user-facing changelog entry. Do not change CLI timeout behavior, Discord plugin behavior, auth scopes, or plugin registration ownership.
