# Coding Plan for openclaw/openclaw#59470: Bug: openclaw gateway call times out for plugin gateway methods even when the method executes successfully

Source report: [records/openclaw-openclaw/items/59470.md](records/openclaw-openclaw/items/59470.md)

## Summary

Keep open. Current `main` and the latest release still invoke plugin gateway handlers without adapting a returned value into a Gateway RPC response, so a return-style plugin method can complete its work while `openclaw gateway call` waits until timeout.

## Plan

Fix https://github.com/openclaw/openclaw/issues/59470. In current main, plugin gateway handlers that return a payload without calling `respond(...)` can complete successfully while `openclaw gateway call` times out because `handleGatewayRequest` ignores the handler return value. Add a backward-compatible adapter in the gateway method dispatch path so a JSON-compatible returned value sends a successful RPC payload when the handler has not already responded. Preserve existing callback-style `respond(...)` handlers and avoid double responses, including accepted/final or multi-response flows. Update the public handler type and plugin SDK docs if the return-value contract becomes supported. Add focused regression coverage for return-style handlers and existing respond-style handlers. Add a `CHANGELOG.md` fix entry if the repair changes user-visible gateway CLI behavior. Do not change gateway timeouts or add Discord-specific logic.
