## Plan

Repair https://github.com/openclaw/openclaw/pull/89587 by fixing the sanitizer bypass in `src/plugins/schema-validator.ts`. The current PR adds `runSchemaOperation`, but it rethrows any Error whose message starts with `invalid schema:` before sanitizing; a schema Proxy can throw that prefix plus terminal controls and leak raw plugin-controlled text. Use an internal branded/sentinel error or re-sanitize caught errors so only OpenClaw-created invalid-schema errors can avoid double wrapping.
