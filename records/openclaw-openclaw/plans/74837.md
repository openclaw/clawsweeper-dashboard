## Summary

Keep open. Current main still accepts and reports an explicit subagent model after persistence without validating that the resolved provider has a runnable CLI backend; the open related PRs are useful but complementary rather than a complete fix for the latest reproduced failure mode.

## Plan

Fix https://github.com/openclaw/openclaw/issues/74837 by adding a narrow spawn-time validation for explicit resolved native subagent model providers. In `src/agents/subagent-spawn.ts`, after `resolveSubagentModelAndThinkingPlan` resolves a caller-provided `sessions_spawn.model` and before returning `modelApplied` or accepting the child run, split the resolved model provider and use the existing `resolveCliBackendConfig(provider, cfg, { agentId: targetAgentId })` contract to verify that the provider has a runnable CLI backend. If it does not, clean up the provisional child session like other pre-start failures and return a clear error that names the resolved model/provider and the missing `agents.defaults.cliBackends.<provider>` setup. Do not duplicate the `modelApplied` flag-only changes from https://github.com/openclaw/openclaw/pull/74892, and do not take over the broader default-precedence/parent-override work from https://github.com/openclaw/openclaw/pull/72984. Add focused regression coverage for explicit model override with missing backend, and keep docs/changelog changes only if the user-visible error text or behavior contract needs a release note.
