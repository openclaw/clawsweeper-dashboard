Keep open: current main still has a source-reproducible path where media-store IDs containing `---<uuid>` become `WebMediaResult.fileName`, and Telegram passes that value directly to `grammy` `InputFile` for outbound delivery.

## Plan

Fix the Telegram outbound filename suffix leak reported at https://github.com/openclaw/openclaw/issues/96538. Keep the fix in the shared media filename normalization layer, preferably reusing `extractOriginalFilename`, so all channel callers get the caller-facing filename without duplicating media-store internals in the Telegram plugin.
