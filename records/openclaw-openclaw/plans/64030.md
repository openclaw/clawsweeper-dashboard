## Plan

Fix https://github.com/openclaw/openclaw/issues/64030 with a narrow cron-boundary repair. Current main rejects cron `session:<id>` targets when the runtime session key contains `/` or `\`, even though channel-native IDs may contain those characters and cron job IDs are UUID-backed. Accept explicit `session:<id>` targets and resolved `current` session keys as opaque nonempty, non-NUL session keys. Keep `assertSafeCronRunLogJobId` and run-log path traversal protections on actual job IDs. Do not normalize or change `buildAgentSessionKey`, and do not store an encoded session key unless all read, delivery, failure-notification, and execution paths decode to the original raw session key before lookup. Update focused tests for session-target helpers, normalization, create/patch validation, gateway add/update validation, and run-log safety.
