## Plan

Fix the source-reproducible builtin `memory_search` freshness bug tracked by this issue and the overlapping https://github.com/openclaw/openclaw/issues/56862. Current builtin default managers are cached for gateway use; after file watcher dirty state, `MemoryIndexManager.search()` starts `startAsyncSearchSync()` without awaiting it, so the current query can return stale rows or deleted paths. Keep the fix inside the builtin memory manager/search sync boundary; do not change QMD behavior except to preserve existing tests, and do not duplicate the docs-only wording work in https://github.com/openclaw/openclaw/pull/85899. Add regression coverage for first search after create/update/delete of memory files and for stale-row pruning before results are returned. If a full await can hang on slow embedding/reindex work, use existing bounded timeout or transient-unavailable semantics rather than silently returning stale results.
