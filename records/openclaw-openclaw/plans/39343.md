# Coding Plan for openclaw/openclaw#39343: Feature Request: Image batching / media group buffering at gateway layer

Source report: [records/openclaw-openclaw/items/39343.md](records/openclaw-openclaw/items/39343.md)

## Summary

Keep open. Current main and the latest release have the Telegram album/config half, but LINE still builds and dispatches one agent turn per image event with no LINE media batching option or docs surface.

## Plan

Implement LINE media batching for this issue inside the LINE plugin only. Add an optional LINE media batching delay/config surface at top level and per account, prefer `message.imageSet.id` for LINE image sets when present, fall back to account/conversation/sender-scoped short time proximity only when metadata is absent, flush on timer and before same-key text or non-media processing, and aggregate media into one `LineInboundContext`. Preserve existing access gating, replay dedupe, media size handling, reply-token/push fallback behavior, group history behavior, Telegram media-group logic, and gateway core contracts. Add focused tests for same imageSet batching, fallback same-sender batching, text flush, separate sender/conversation/account isolation, disabled/default config behavior, and config schema/types/docs. Update docs and changelog for the user-facing LINE option. Do not change Telegram issue https://github.com/openclaw/openclaw/issues/55216 in this PR.
