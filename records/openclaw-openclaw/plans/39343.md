# Coding Plan for openclaw/openclaw#39343: Feature Request: Image batching / media group buffering at gateway layer

Source report: [records/openclaw-openclaw/items/39343.md](records/openclaw-openclaw/items/39343.md)

## Summary

Keep open. Current main covers Telegram album buffering and its config key, but the LINE plugin still processes each downloadable media webhook event as its own agent turn and has no LINE batching config or docs surface.

## Plan

Implement the remaining LINE media batching behavior for this issue. Add an optional LINE plugin config key, likely `mediaBatchDelayMs`, defaulting to current behavior unless explicitly enabled or using a conservative documented default approved by existing patterns. Scope buffering by LINE account, conversation/peer, and sender; buffer downloadable media events, flush on timer and before same-key text/non-media dispatch, aggregate media into one `LineInboundContext`, and preserve replay dedupe, access gating, group history, reply-token freshness, and media download error handling. Do not add a generic gateway-layer buffer and do not change Telegram behavior. Update schema/types/generated config metadata/docs/changelog and add focused tests for multiple rapid LINE image events, text-triggered flush, separate sender/conversation/account isolation, and existing single-image behavior.
