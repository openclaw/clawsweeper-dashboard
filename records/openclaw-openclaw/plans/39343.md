## Summary

Keep open. Current main covers the Telegram album buffering/config half, but LINE media events are still processed one at a time with no same-sender media buffer or LINE config/docs surface; the remaining work belongs in the LINE plugin rather than a generic gateway layer.

## Plan

Implement the remaining LINE media batching from https://github.com/openclaw/openclaw/issues/39343 inside the LINE plugin boundary. Do not add generic gateway batching and do not change Telegram behavior. Add an optional LINE media batch delay config surface, buffer downloadable LINE media by accountId + conversation peer + sender, flush on timer and before same-key text/non-media processing, and aggregate the saved media into one LineInboundContext. Preserve LINE access gating, replay dedupe, group history behavior, reply-token/push fallback behavior, and existing single-media behavior. Update schema/types/generated config metadata/docs and add focused regression tests; add a CHANGELOG.md entry because this is user-visible.
