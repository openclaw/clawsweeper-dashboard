# Coding Plan for openclaw/openclaw#78697: Weixin channel can drop standalone MEDIA directives when payload.mediaUrl is absent

Source report: [records/openclaw-openclaw/items/78697.md](records/openclaw-openclaw/items/78697.md)

## Summary

Keep open: current main still has a legacy/plugin reply dispatcher seam where standalone `MEDIA:` text can reach `deliver` without being promoted to `mediaUrl`/`mediaUrls`. The normal agent-runner, block-reply, and durable outbound paths already parse directives, so this remains a narrow consistency bug rather than a new feature request.

## Plan

Fix https://github.com/openclaw/openclaw/issues/78697. Current main leaves standalone `MEDIA:` directives in direct legacy/plugin reply dispatcher payloads when callers use `createReplyDispatcher`/`createReplyDispatcherWithTyping` with text-only payloads, so external plugins such as @tencent-weixin/openclaw-weixin see no `mediaUrl`/`mediaUrls`. Add a channel-agnostic core normalization step before the legacy/plugin `deliver` callback, reusing existing `parseReplyDirectives`/`normalizeReplyPayloadDirectives` behavior. Preserve existing silent/heartbeat handling, response prefixes, `transformReplyPayload`, `beforeDeliver`, ordering, block-streaming, and already-normalized final/durable paths. Do not add Weixin-specific logic. Add focused regression coverage for dispatcher delivery of `caption\nMEDIA:/tmp/a.png` and compatibility delivery through the plugin SDK bridge where appropriate. Include a user-facing changelog fix entry unless the maintainer lane decides to batch changelog at landing.
