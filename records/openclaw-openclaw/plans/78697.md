## Summary

Keep open: current main still has a source-reproducible reply-dispatcher path where standalone `MEDIA:` text can reach channel delivery without being promoted to `mediaUrl` or `mediaUrls`. The closed related reports cover parser pollution and generic outbound message delivery, but they do not fix this legacy reply-dispatch boundary.

## Plan

Fix https://github.com/openclaw/openclaw/issues/78697. Current main can deliver standalone `MEDIA:` directives as plain text through `createReplyDispatcher`/`createReplyDispatcherWithTyping` because `normalizeReplyPayload` runs before `options.deliver` but `normalizeReplyPayloadDirectives` does not. Add a narrow shared-boundary fix, likely in `src/auto-reply/reply/reply-dispatcher.ts`, so text directives such as `caption\nMEDIA:/tmp/a.png` are promoted to `mediaUrl`/`mediaUrls` before channel/plugin delivery while preserving already-normalized payload media fields, silent reply handling, heartbeat stripping, response-prefix behavior, `beforeDeliver`, delivery ordering, and media dedupe semantics. Add focused regression coverage for dispatcher final/block/tool delivery as appropriate. Do not add a Weixin-specific parser to OpenClaw core.
