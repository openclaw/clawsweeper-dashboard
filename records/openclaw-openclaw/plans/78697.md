## Summary

Keep open: current main still has a source-reproducible reply-dispatcher path where standalone `MEDIA:` text can reach a channel `deliver` callback without being promoted to `mediaUrl` or `mediaUrls`. The related polluted-path parser issue at https://github.com/openclaw/openclaw/issues/75182 was fixed separately and does not cover this delivery-boundary normalization gap.

## Plan

Fix the Weixin-observed reply delivery normalization gap by making the shared dispatcher or legacy plugin delivery boundary promote standalone `MEDIA:` directives from `payload.text` into `mediaUrl`/`mediaUrls` before channel `deliver` callbacks run. Reuse `normalizeReplyPayloadDirectives` rather than adding Weixin-specific parsing in core, and preserve already-normalized media fields, silent reply handling, heartbeat stripping, response prefix behavior, `beforeDeliver`, queued delivery ordering, and block/final media dedupe. Add focused regression coverage showing `createReplyDispatcher` or the inbound plugin dispatch path delivers `{ text: "caption\nMEDIA:/tmp/a.png" }` as text `caption` with media fields populated, while existing silent/media cases still pass.
