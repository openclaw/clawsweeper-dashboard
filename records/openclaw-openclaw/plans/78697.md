# Coding Plan for openclaw/openclaw#78697: Weixin channel can drop standalone MEDIA directives when payload.mediaUrl is absent

Source report: [records/openclaw-openclaw/items/78697.md](records/openclaw-openclaw/items/78697.md)

## Summary

Keep open: current main still has a source-reproducible legacy/channel delivery path where a final payload containing standalone `MEDIA:` text can reach a plugin deliverer without being promoted to `mediaUrl` or `mediaUrls`. The related polluted-path parser fix at https://github.com/openclaw/openclaw/issues/75182 is distinct and does not close this delivery-boundary normalization gap.

## Plan

Fix the reply delivery contract so standalone `MEDIA:` directives in final reply text are normalized into `mediaUrl`/`mediaUrls` before external channel/plugin delivery. Prefer a shared, channel-agnostic fix at `src/auto-reply/reply/reply-dispatcher.ts` or the legacy plugin delivery bridge using `normalizeReplyPayloadDirectives`; do not add Weixin-specific parsing to OpenClaw core. Preserve existing `mediaUrl`/`mediaUrls`, silent reply, heartbeat, response-prefix, `beforeDeliver`, and media dedupe behavior. Add focused regression coverage for a final dispatcher payload such as `caption\nMEDIA:/tmp/a.png`, keep existing block/agent-runner directive behavior intact, and add a user-facing `CHANGELOG.md` fix entry.
