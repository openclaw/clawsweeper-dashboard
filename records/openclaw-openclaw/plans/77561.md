## Summary

Keep open: current main still routes `ownsCompaction` sessions away from the normal tool-result guard and the opt-in mid-turn precheck, so the reported owned-engine tool-loop gap remains source-reproducible even though I did not run a live 20+ tool-call Codex session.

## Plan

Fix https://github.com/openclaw/openclaw/issues/77561 by adding a bounded last-resort mid-turn context-pressure safeguard for embedded Pi sessions whose active context engine has `info.ownsCompaction === true`. Preserve the context-engine ownership contract: do not blindly install the normal tool-result guard in a way that destructively truncates engine-owned context, and do not add a new config option. Prefer routing an owned-engine pressure signal through the existing precheck/overflow recovery path so the active engine's `compact()` handles compaction and retry. Add regression coverage showing that `midTurnPrecheck.enabled: true` can trigger recovery for `ownsCompaction: true` sessions after tool results, while non-owned sessions keep existing behavior. Update docs/changelog only if the user-visible compaction behavior changes; stop and return to maintainer review if the fix requires changing the public context-engine contract.
