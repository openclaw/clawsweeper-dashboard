# Coding Plan for openclaw/openclaw#77561: ownsCompaction tool-result context guard bypass still present in v2026.5.3 (Codex sessions have no mid-turn compaction protection)

## Summary

Keep open: current main still routes `ownsCompaction` sessions away from the normal mid-turn tool-result precheck, and the owned-engine loop hook does not raise the existing mid-turn recovery signal. The related Codex sidecar rotation work is useful adjacent context-window hardening, but it does not close this core owned-compaction gap.

## Plan

Fix https://github.com/openclaw/openclaw/issues/77561. Current main routes `ownsCompaction` sessions through `installContextEngineLoopHook`, while only `installToolResultContextGuard` receives `midTurnPrecheck`; `shouldDisablePiAutoCompaction` also disables Pi internal auto-compaction for owned engines. Add an owned-engine-safe mid-turn pressure path that detects prompt pressure after new tool results and before the next model call, then reuses `MidTurnPrecheckSignal`/`handleMidTurnPrecheckRequest` or the active context-engine compaction path. Do not unconditionally install the normal tool-result guard for `ownsCompaction` sessions, do not destructively truncate engine-owned context outside the existing recovery flow, and do not add a new config option unless maintainer review explicitly asks for one. Update focused tests so `ownsCompaction: true` with `midTurnPrecheck.enabled: true` is covered by the new behavior instead of asserting the option is absent; preserve behavior for sessions without a context engine and for `ownsCompaction !== true`. Add a `CHANGELOG.md` fix entry unless maintainers classify the final patch as internal-only.
