## Summary

Keep open: current main captures and persists Telegram forum topic names for inbound context, but the known topic name is still not carried into stored session display metadata, gateway session rows, or the Control UI session selector, so the central dropdown problem remains.

## Plan

Fix the Telegram forum-topic session dropdown labeling gap for this issue. Current main already captures cached TopicName for Telegram forum inbound context, but does not persist or expose it as session display metadata. Add a narrow generic thread/display-label path: have Telegram forum-topic context populate an explicit ThreadLabel or equivalent session display label from cached TopicName, persist it through session metadata without changing non-Telegram labels, expose it in gateway sessions.list rows, and make the Control UI selector prefer the explicit label. Do not add a config option, do not hardcode Telegram-specific display formatting in generic core, and coordinate with the adjacent UI cleanup in https://github.com/openclaw/openclaw/pull/68850 if it is still open. Add focused regression coverage across Telegram context, session metadata/store normalization, gateway session rows, and Control UI display.
