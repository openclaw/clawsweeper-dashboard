## Summary

Current main still knows Telegram forum topic titles but does not carry them into the stored display metadata consumed by the Control UI selector. This remains the canonical, vision-aligned tracker; the direct implementation PRs are closed unmerged and no open PR currently owns the fix.

## Plan

Implement human-readable Telegram forum topic labels for the Control UI session selector. In the Telegram inbound context path, populate the existing generic `ThreadLabel` from the already-resolved forum `topicName` only for forum-topic sessions; rely on the canonical session path that promotes `ThreadLabel` to `displayName`, and confirm `sessions.list` exposes that value to `ui/src/ui/session-display.ts`. Preserve current labels for non-Telegram sessions, Telegram DMs, non-forum chats, and forum topics whose name is unknown. Do not add a config option, a parallel `topicName` session field, raw-key name inference, Telegram-specific formatting in core, or a CHANGELOG.md entry. Add focused regression coverage for Telegram context, session persistence/gateway rows, and selector display. Provide redacted live proof from a real Telegram forum topic and the Control UI selector; stop if the implementation requires a new product policy or config surface.
