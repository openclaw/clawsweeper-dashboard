# Coding Plan for openclaw/openclaw#7406: [Feature]: Human-readable Telegram topic names in session dropdown

Source report: [records/openclaw-openclaw/items/7406.md](records/openclaw-openclaw/items/7406.md)

## Summary

Keep open: current main still learns Telegram forum topic names, but it does not carry them into persisted session display metadata, gateway session rows, or the Control UI selector label path. The earlier implementation PRs were closed unmerged, and the open UI cleanup PR is adjacent rather than a complete backend-to-UI fix.

## Plan

Fix https://github.com/openclaw/openclaw/issues/7406 by adding a generic session thread/display-label path for known thread or topic names. Reuse the existing Telegram topic-name cache and TopicName source; have the Telegram plugin populate a generic ThreadLabel or explicit session display-label field for forum-topic sessions, persist it through session metadata, expose it in GatewaySessionRow/sessions.list, and update the Control UI selector to prefer the explicit label while preserving existing non-Telegram label behavior. Coordinate with the UI-only cleanup in https://github.com/openclaw/openclaw/pull/51124; do not hard-code broad Telegram session-key parsing in core as the primary fix. Add focused regression tests across Telegram context, session metadata, gateway row construction, and UI selector behavior. Add a changelog entry because the behavior is user-facing.
