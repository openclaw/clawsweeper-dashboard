## Summary

Keep open. Current main has a persisted Telegram forum topic-name source and exposes `TopicName` in inbound context, but that value still is not propagated into durable session display metadata, gateway session rows, or the Control UI session selector.

## Plan

Fix the Telegram forum topic session display-label gap for https://github.com/openclaw/openclaw/issues/7406. Current main already caches Telegram forum topic names and exposes `TopicName` in inbound context, but session metadata, `sessions.list`, and the Control UI selector do not consume it. Add a narrow generic display/thread label path, have the Telegram plugin populate it from the cached topic name for forum-topic sessions, expose it through gateway session rows, and make the UI selector prefer that explicit label while preserving existing non-Telegram labels.
