# Coding Plan for openclaw/openclaw#7406: [Feature]: Human-readable Telegram topic names in session dropdown

Source report: [records/openclaw-openclaw/items/7406.md](records/openclaw-openclaw/items/7406.md)

## Summary

Keep open: current main and the latest release learn Telegram forum topic names, but that value still stops at Telegram context metadata and is not carried into stored session display metadata, gateway session rows, or the Control UI selector.

## Plan

Fix the session dropdown label gap for Telegram forum topics. Current main already learns cached Telegram TopicName from forum_topic_created/forum_topic_edited service messages; carry that known thread/topic label through a generic session display-label or thread-label metadata path, persist it in session metadata for topic sessions, expose it in gateway sessions.list rows, and make the Control UI session selector prefer that explicit label while preserving existing non-Telegram labels and session selection semantics. Do not add a new config option, do not re-fetch topic names on UI/gateway hot paths, and do not make a Telegram-only UI string parser the source of truth. Coordinate with the adjacent UI-only cleanup PR if needed. Add focused regression tests and attach redacted real behavior proof showing a known Telegram forum topic name in the Control UI picker.
