## Summary

Keep open: current main still learns Telegram forum topic names but only exposes them as inbound context metadata; the stored session row and Control UI session selector still use generic group labels instead of the topic name.

## Plan

Fix the open Telegram forum-topic session-label gap. Current main caches Telegram TopicName from forum service messages and exposes it to inbound context, but stored session metadata, gateway sessions.list rows, and the Control UI session selector still do not display the human topic name. Add a narrow generic session thread/display-label path rather than a Telegram-only UI formatter; have the Telegram plugin populate it from the cached TopicName for forum-topic sessions; expose it through gateway session rows; make the Control UI prefer the explicit label while preserving existing non-Telegram labels. Do not add a per-topic config name field, new env/config option, or broad session-label rewrite. Add focused regression coverage and include redacted real behavior proof showing a known Telegram forum topic name in the Control UI picker.
