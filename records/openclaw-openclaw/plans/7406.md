## Summary

Keep open: current main still learns Telegram forum topic names, but it does not persist or expose that value as session display metadata for the gateway row or Control UI selector. The related duplicate reports are already closed in favor of this canonical tracker, and the open UI cleanup PRs are adjacent rather than a complete metadata bridge.

## Plan

Fix https://github.com/openclaw/openclaw/issues/7406. Add a narrow generic session thread/display-label metadata path for known conversation thread/topic labels; have the Telegram plugin populate it from the existing cached TopicName for forum-topic sessions; persist it in SessionEntry/session metadata; expose it through Gateway sessions.list rows and UI types; and make the Control UI session selector prefer that explicit label while preserving current non-Telegram labels. Do not parse Telegram session-key strings in core as the primary fix, do not add a new config option, and do not fold unrelated synthetic-label cleanup from https://github.com/openclaw/openclaw/pull/51124 or https://github.com/openclaw/openclaw/pull/68850 into this change except where needed to avoid conflicts. Add focused regression coverage and a user-facing CHANGELOG.md entry.
