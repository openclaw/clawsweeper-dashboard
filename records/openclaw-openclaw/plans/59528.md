# Coding Plan for openclaw/openclaw#59528: [Bug]: The bundled plugins openshell does not work correctly since version 2026.03.13

Source report: [records/openclaw-openclaw/items/59528.md](records/openclaw-openclaw/items/59528.md)

## Summary

Keep open. Current main still lets `openclaw sandbox list` inspect OpenShell registry entries without loading the full plugin registration that provides the OpenShell backend manager, so the persistent stopped-status symptom remains source-reproducible.

## Plan

Fix OpenShell sandbox CLI status/lifecycle commands so backend-managed entries are inspected after the providing plugin registers its sandbox backend manager. Add a generic sandbox CLI plugin/backend activation path for `openclaw sandbox ...` before registry inspection, without hardcoding OpenShell in core and without changing the separate split-module registry fix tracked by https://github.com/openclaw/openclaw/issues/67610 and https://github.com/openclaw/openclaw/pull/67623. Preserve lightweight startup behavior for unrelated CLI commands. Add or update regression coverage around command path policy/startup/bootstrap, sandbox list/recreate behavior, sandbox manager lookup, and OpenShell manager behavior. This is user-visible sandbox CLI behavior, so include a changelog entry unless maintainers decide it is landing-only maintenance.
