## Plan

Fix the OpenShell sandbox status bug reported at https://github.com/openclaw/openclaw/issues/59528. Current main lets `openclaw sandbox list` and related sandbox lifecycle commands run without loading plugin-provided sandbox backend managers, so an `openshell` registry entry is rendered as stopped when the OpenShell plugin has not registered its manager. Add a generic sandbox CLI backend activation path before list/recreate/status-style registry inspection; do not hardcode OpenShell-specific behavior in core.
