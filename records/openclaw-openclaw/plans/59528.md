## Summary

Keep open. Current main and the latest shipped release still have the source-level path described in the prior ClawSweeper review: `openclaw sandbox list` can inspect an OpenShell registry entry without loading the plugin that registers the OpenShell backend manager, so OpenClaw reports the runtime as stopped instead of asking OpenShell for status.

## Plan

Fix https://github.com/openclaw/openclaw/issues/59528. Current main lets `openclaw sandbox list` and `openclaw sandbox recreate` run without loading plugin-provided sandbox backend managers; an OpenShell registry entry then has no registered manager and is rendered as stopped. Add a generic sandbox-backend activation path for sandbox CLI lifecycle/status operations, preserving the core/plugin boundary and avoiding OpenShell-specific hardcoding in core. Add focused regression coverage for an OpenShell-style backend entry or plugin-provided sandbox manager so list/recreate can query/remove through the manager. Include a user-facing Fixes changelog entry if the final patch changes shipped behavior. Do not add a new config option or change unrelated sandbox policy.
