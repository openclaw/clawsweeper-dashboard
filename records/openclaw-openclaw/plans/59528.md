## Summary

Keep open. Current main still does not load plugins for `openclaw sandbox` commands, while OpenShell only registers its backend manager during full plugin registration; that leaves `openclaw sandbox list` able to read OpenShell registry entries but unable to ask the OpenShell manager for live status, so entries can still be shown as stopped.

## Plan

Fix https://github.com/openclaw/openclaw/issues/59528 on current main. The observable bug is that `openclaw sandbox list` can read an OpenShell registry entry but does not load the full plugin registration that supplies the OpenShell sandbox backend manager, so the entry is reported stopped. Implement a generic sandbox CLI backend activation path for sandbox lifecycle/status commands before registry inspection; do not hardcode OpenShell in core, do not add startup/load-time config migration, and do not conflate this with the separate split-module backend registry work in https://github.com/openclaw/openclaw/issues/67610 or https://github.com/openclaw/openclaw/pull/67623.
