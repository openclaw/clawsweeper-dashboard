# Coding Plan for openclaw/openclaw#59528: [Bug]: The bundled plugins openshell does not work correctly since version 2026.03.13

Source report: [records/openclaw-openclaw/items/59528.md](records/openclaw-openclaw/items/59528.md)

## Summary

Keep open. Current main and the latest stable tag still have the OpenShell sandbox CLI status path described in the earlier ClawSweeper review: `openclaw sandbox list` can inspect an OpenShell registry entry without loading the plugin that registers the OpenShell backend manager, so the entry is rendered as stopped.

## Plan

Fix https://github.com/openclaw/openclaw/issues/59528. Observable bug: OpenShell sandbox registry entries can show as stopped in `openclaw sandbox list` because sandbox CLI lifecycle/status commands run without loading the plugin that registers the OpenShell backend manager. Implement a generic sandbox-backend activation path for sandbox CLI list/recreate/status-style registry inspection so configured plugin-provided backend managers are available before `listSandboxContainers()` and removal paths inspect entries. Do not hardcode OpenShell behavior in core and do not change the OpenShell CLI dependency or plugin config schema unless a source-level need appears. Add or update regression tests around command path/startup policy, CLI bootstrap plugin loading, sandbox list/recreate manager availability, and OpenShell backend registration. Add a user-facing changelog fix entry if this becomes a PR.
