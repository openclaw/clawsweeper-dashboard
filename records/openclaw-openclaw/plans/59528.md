## Summary

Keep open. Current main and the latest stable release still allow `openclaw sandbox list` to inspect OpenShell registry entries without loading the plugin that registers the OpenShell backend manager, so those entries can still render as stopped.

## Plan

Fix https://github.com/openclaw/openclaw/issues/59528. The observable bug is that `openclaw sandbox list` and related sandbox lifecycle/status commands can inspect an OpenShell registry entry without loading the plugin that registers the `openshell` sandbox backend manager, causing the entry to render as stopped. Implement a generic sandbox CLI startup/activation path for configured sandbox backend plugins before registry inspection; do not hardcode OpenShell in core and do not broaden into the separate split-module registry issue tracked elsewhere. Add focused regression coverage for sandbox command path plugin loading, OpenShell manager registration availability, and the existing missing-manager stopped fallback.
