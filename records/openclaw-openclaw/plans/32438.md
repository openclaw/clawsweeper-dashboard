## Summary

Keep open. Current main and v2026.5.20 still write only the configured JSONL log file, while CLI and Control UI formatting happens at read/display time; the earlier duplicate and same-author implementation PRs are closed without a merged sidecar, and the merged adjacent log-discovery fix does not provide the requested companion file.

## Plan

Implement a focused formatted log sidecar for OpenClaw file logging. Preserve the existing JSONL `logging.file` contract for CLI, Control UI, scripts, and log-tail consumers; derive the sidecar path from the resolved active file path by inserting `-formatted` before the extension; write redacted plain-text lines such as `YYYY-MM-DD HH:mm:ss [level] [subsystem] message` whenever the JSONL file is written. Define and test date-derived path behavior, size-cap/rotation behavior, pruning expectations, and redaction parity. Update operator logging docs and CHANGELOG.md for the user-visible behavior. Do not add a new config option unless maintainers explicitly choose that product direction.
