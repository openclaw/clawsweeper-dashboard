## Summary

Keep open. Current main and v2026.5.12 still persist Gateway file logs as a single JSONL stream; CLI and Control UI readability is produced at read time, and the direct formatted-sidecar PRs were closed unmerged.

## Plan

Implement a focused logger enhancement for https://github.com/openclaw/openclaw/issues/32438. Add a human-readable companion file derived from the active `logging.file` path, inserting `-formatted` before the extension, while preserving the existing JSONL file format and all `logs.tail`, CLI, Control UI, and script consumers. Write one redacted plain-text line per successful JSONL log event using `YYYY-MM-DD HH:mm:ss [level] [subsystem] message`; handle child logger bindings without duplicating raw JSON binding strings in the message. Define and test behavior for default rolling dated log files, date changes, JSONL rotation, pruning, and max-size handling so the formatted sidecar does not grow or drift unexpectedly. Do not add a new config key unless maintainers explicitly decide one is needed. Update operator docs and add a user-facing changelog entry.
