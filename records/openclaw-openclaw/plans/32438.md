## Summary

Keep open. Current main and v2026.5.12 still persist Gateway file logs as a single JSONL stream, while CLI and Control UI formatting happens only at read time; the direct implementation attempts at https://github.com/openclaw/openclaw/pull/27446 and https://github.com/openclaw/openclaw/pull/32442 were both closed unmerged.

## Plan

Implement the formatted log sidecar requested in https://github.com/openclaw/openclaw/issues/32438. Add a logger-layer companion file derived from the active `logging.file` path by inserting `-formatted` before the extension, write one redacted plain-text line per JSONL event, and keep the existing JSONL path, format, `logs.tail`, CLI, Control UI, and script consumers unchanged. Handle default dated rolling files, date changes, rotation/pruning, and size-cap behavior deliberately; do not add a new config key unless the existing request becomes impossible without maintainer approval. Reuse the existing redaction path before persistence, update operator docs and `CHANGELOG.md`, and add regression coverage for sidecar creation, redaction, unchanged JSONL consumers, rolling path/date behavior, and size-cap/rotation cleanup. Treat https://github.com/openclaw/openclaw/pull/27446 and https://github.com/openclaw/openclaw/pull/32442 as useful prior art, but avoid the known formatted-line prefix/message duplication issue called out on the later PR.
