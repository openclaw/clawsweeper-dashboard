## Summary

Keep open. Current main and v2026.5.12 still persist Gateway file logs as one JSONL stream, while CLI and Control UI readability is produced by parsing that JSONL at read time; the related formatted-sidecar PRs are closed unmerged, and the merged date-boundary log discovery PR is adjacent rather than a replacement.

## Plan

Implement a focused formatted-log sidecar for https://github.com/openclaw/openclaw/issues/32438. Preserve the existing JSONL `logging.file` contract and all CLI/Control UI/script consumers. In `src/logging/logger.ts`, derive a human-readable companion path from the active JSONL log path by inserting `-formatted` before the extension, write one redacted plain-text line per JSONL event, and keep the JSONL append path unchanged. Reuse the existing redaction path before writing the sidecar. Handle default rolling log dates, size-cap rotation, and pruning deliberately; do not add a config key unless maintainers explicitly ask. Use the closed PRs https://github.com/openclaw/openclaw/pull/27446 and https://github.com/openclaw/openclaw/pull/32442 as context only; do not reintroduce child-logger prefix noise in the formatted message. Update focused tests for resolved settings, redaction, file rotation, default rolling date changes, and unchanged JSONL consumers. Update `docs/logging.md` and any gateway logging/config docs that describe file-log behavior. Add a changelog entry because this is user-visible. Stop and ask if the implementation would need a new config option or broader logging policy change.
