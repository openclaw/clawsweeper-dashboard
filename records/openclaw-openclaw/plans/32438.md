# Coding Plan for openclaw/openclaw#32438: [Feature]: Add a formatted log file when JSON logs are hard to read and Control UI is unusable

Source report: [records/openclaw-openclaw/items/32438.md](records/openclaw-openclaw/items/32438.md)

## Summary

Keep open. Current main and the latest release still persist Gateway file logs as a single JSONL stream, while CLI and Control UI readability is produced by parsing that JSONL at read time; the formatted-sidecar PRs are closed unmerged, and nearby log timestamp/UI fixes do not supersede this request.

## Plan

Implement a focused formatted-log sidecar for https://github.com/openclaw/openclaw/issues/32438. Preserve the existing logging.file JSONL file format and all CLI/Control UI/script consumers. When file logging is enabled, derive a companion path in the same directory by inserting -formatted before the extension and append one redacted plain-text line per JSONL event, using the requested shape YYYY-MM-DD HH:mm:ss [level] [subsystem] message. Do not add a new config option unless maintainer guidance changes. Reuse the existing file-log redaction path, keep logging failures non-blocking, and define behavior for default dated rolling paths, size rotation, and pruning so the formatted file does not grow or drift unexpectedly. Use the closed sidecar PRs only as reference, especially https://github.com/openclaw/openclaw/pull/32442 and https://github.com/openclaw/openclaw/pull/27446; do not copy their bugs blindly. Update operator docs and add regression coverage for unchanged JSONL output, companion path derivation, child/subsystem formatting, redaction, size/rotation behavior, and CLI/UI read paths remaining JSONL-based. Add a user-visible changelog entry because this adds an operator-facing log artifact.
