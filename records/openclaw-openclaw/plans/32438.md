# Coding Plan for openclaw/openclaw#32438: [Feature]: Add a formatted log file when JSON logs are hard to read and Control UI is unusable

Source report: [records/openclaw-openclaw/items/32438.md](records/openclaw-openclaw/items/32438.md)

## Summary

Keep open. Current main and the latest tagged release still persist Gateway file logs as a single redacted JSONL stream, while CLI and Control UI formatting happens only when reading that JSONL; the duplicate report and same-author implementation PRs are closed without a merged formatted sidecar.

## Plan

Fix https://github.com/openclaw/openclaw/issues/32438 by adding a focused formatted log sidecar in the logger layer. Derive the companion path from the resolved active `logging.file` by inserting `-formatted` before the extension, write one redacted plain-text line per JSONL event, preserve the existing JSONL file path/format and all CLI/Control UI consumers, and avoid adding a new config key unless the existing contract forces it. Treat the closed PRs https://github.com/openclaw/openclaw/pull/27446 and https://github.com/openclaw/openclaw/pull/32442 as context only; do not copy the known bug where the child-logger JSON binding prefix leaks into the formatted message. Explicitly handle dated rolling paths, JSONL rotation/pruning/maxFileBytes behavior, and redaction for the sidecar. Add/update focused regression tests and user-facing docs, plus a CHANGELOG entry for the user-visible logging change.
