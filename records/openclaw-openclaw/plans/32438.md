# Coding Plan for openclaw/openclaw#32438: [Feature]: Add a formatted log file when JSON logs are hard to read and Control UI is unusable

Source report: [records/openclaw-openclaw/items/32438.md](records/openclaw-openclaw/items/32438.md)

## Summary

Keep open. Current main and the latest release still persist Gateway file logs as one JSONL file, while CLI and Control UI formatting happens only when those consumers read the JSONL; the earlier duplicate and same-author implementation PRs are closed without a merged fix.

## Plan

Implement the formatted Gateway file-log sidecar requested by https://github.com/openclaw/openclaw/issues/32438. Keep `logging.file` as the unchanged JSONL source of truth and do not add a new config option. Derive a readable companion path from the active file path by inserting `-formatted` before the extension, including dated rolling files. Write one redacted plain-text line per emitted JSONL record using the existing logger transport/redaction behavior; preserve CLI, Control UI, `logs.tail`, JSONL parsers, and existing file-size rotation behavior. Define and test lifecycle behavior for date changes, rotation/pruning, silent logging, and redaction. Use the closed PRs https://github.com/openclaw/openclaw/pull/27446 and https://github.com/openclaw/openclaw/pull/32442 as source context, but fix the prior review concern about raw child-logger binding JSON leaking into the formatted message. Update operator docs and changelog for the new user-visible file surface.
