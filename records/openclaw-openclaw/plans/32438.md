## Summary

Keep open. Current main and v2026.5.19 still write only the Gateway JSONL file; CLI and Control UI parse that file at read time, and the linked formatted-log PRs are closed unmerged, so no close reason applies.

## Plan

Implement the formatted file-log companion requested by https://github.com/openclaw/openclaw/issues/32438. In `src/logging/logger.ts`, derive a human-readable companion path from the resolved active `logging.file` path by inserting `-formatted` before the extension, and append one redacted plain-text line per JSONL event without changing the existing JSONL file, `logs.tail`, CLI, Control UI, or scripts that consume JSONL. Handle dated rolling paths, JSONL rotation/maxFileBytes, and pruning deliberately so the companion file does not grow or drift unexpectedly; do not add a config option unless a maintainer explicitly asks. Reuse the existing redaction path before persistence. Add/update focused tests for settings, rotation/date rollover, redaction, logger transport output, and unchanged JSONL consumers. Update `docs/logging.md`, `docs/gateway/logging.md`, `docs/gateway/configuration-reference.md`, and add a user-facing changelog entry. Preserve the merged active-log date-boundary behavior from https://github.com/openclaw/openclaw/pull/42904 and do not revive the raw message-assembly bug called out on https://github.com/openclaw/openclaw/pull/32442.
