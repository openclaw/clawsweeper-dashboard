# Coding Plan for openclaw/openclaw#32438: [Feature]: Add a formatted log file when JSON logs are hard to read and Control UI is unusable

Source report: [records/openclaw-openclaw/items/32438.md](records/openclaw-openclaw/items/32438.md)

## Summary

Keep open. Current main and latest release v2026.5.12 still persist Gateway file logs as a single JSONL stream, while CLI and Control UI readability is produced by parsing that file at read time; the two same-author formatted-sidecar PRs were closed unmerged, and the merged date-boundary log discovery work is adjacent rather than a replacement.

## Plan

Implement the formatted log sidecar requested in https://github.com/openclaw/openclaw/issues/32438 as a focused logger-layer feature. Preserve the existing JSONL file log path, format, rotation semantics, and consumers; derive a companion path from the active JSONL file by inserting `-formatted` before `.log`; write redacted plain-text `YYYY-MM-DD HH:mm:ss [level] [subsystem] message` lines whenever a JSONL line is written; handle default rolling date changes, rotation/pruning, and main-file write failures deliberately; do not add a new config option unless maintainers explicitly choose one. Update operator docs and add a user-facing changelog bullet. Use https://github.com/openclaw/openclaw/pull/27446 and https://github.com/openclaw/openclaw/pull/32442 as context only, and avoid the raw child-logger bindings leak noted in the later PR discussion.
