## Summary

Keep open: current main still has no read-only gateway RPC for image-generation provider inventory, while the existing image provider registry and CLI/tool list paths make this a bounded additive gateway feature.

## Plan

Implement a read-only `image.providers` gateway RPC for https://github.com/openclaw/openclaw/issues/78330. Reuse the existing image-generation provider registry/runtime list; do not add config or a second provider registry. Return secret-free metadata such as provider id, label/name, configured state, selected/default model info, models, and capabilities. Register the method under `operator.read` in the core gateway descriptor table and lazy handler list, add protocol schemas/validators where appropriate, update public gateway docs, and add focused tests covering descriptor auth, successful provider listing, configured-state behavior, and rejection of unexpected params. Preserve current CLI `openclaw infer image providers --json` behavior and do not expose API keys, SecretRefs, raw provider configs, or auth profile payloads.
