## Summary

Keep open: current main and the latest release still cache the Twitch `ChatClient` immediately after `connect()` and log success before Twurple emits connect/auth lifecycle events. The report is valid, but the suggested `await client.connect()` shape is not the right fix because Twurple 8.1.4 exposes `connect(): void`.

## Plan

Fix the Twitch `TwitchClientManager.getClient` lifecycle bug reported at https://github.com/openclaw/openclaw/issues/83885. Do not implement `await client.connect()` because `@twurple/chat` 8.1.4 declares `ChatClient.connect(): void`; instead register Twurple lifecycle handlers before `connect()` and make cache/log state follow `onConnect`, `onAuthenticationFailure`, and `onDisconnect` semantics. Preserve Twurple reconnect behavior, avoid duplicate retrying clients, and make sure a terminal failed setup does not leave a broken client returned from the `clients` map.
