# Coding Plan for openclaw/openclaw#61612: perf(cli): add help fast path for built-in and plugin subcommands

## Summary

Keep open: the performance goal is plausible, but the PR is not safe as-is because the new fast path can misread pass-through `--help` arguments and the metadata-only plugin help fallback can drop supported command-only plugin CLIs.

## Plan

Treat https://github.com/openclaw/openclaw/pull/61612 as source work. Rebuild the CLI subcommand help fast path on current main without reverting existing root/browser/gateway fast paths or current command-registration policy. Use `resolveCliArgvInvocation`/`isHelpOrVersionInvocation` semantics so flags after `--` do not trigger help fast paths. For plugin subcommand help, use metadata-only loading only when manifest or descriptor ownership proves the target command is covered; preserve the full non-activating loader fallback for supported commands-only `api.registerCli` plugin CLIs. Avoid a hard-coded bundled plugin id map in core when manifest/registry ownership can resolve the command. Add focused regressions for pass-through `--help`, descriptor-backed plugin help, commands-only plugin CLI fallback, and the built-in help fast path. Include a changelog entry only if the replacement PR lands as a user-visible CLI performance change.
