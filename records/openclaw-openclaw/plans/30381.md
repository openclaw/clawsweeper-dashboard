## Summary

Keep open. Current main still routes by `x-openclaw-agent-id` first, but then the shared OpenAI-compatible model override helper rejects a generic request body `model` before chat dispatch; the related open PR addresses unknown agent IDs and does not supersede this compatibility bug.

## Plan

Fix https://github.com/openclaw/openclaw/issues/30381.
