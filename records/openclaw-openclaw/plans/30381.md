Keep open. Current main still lets `x-openclaw-agent-id` select the agent, then rejects a generic OpenAI-compatible request-body `model` before dispatch, so the reported production compatibility problem remains valid and narrowly fixable.

## Plan

Fix https://github.com/openclaw/openclaw/issues/30381. In the OpenAI-compatible gateway paths, when an explicit valid `x-openclaw-agent-id` or `x-openclaw-agent` header selects the agent, do not reject the request just because the request-body `model` is a generic OpenAI/provider model such as `gpt-4`; treat that body field as SDK compatibility input on the accepted-header path.
