# Coding Plan for openclaw/openclaw#13870: Feature Request: Human-friendly device names in paired devices list

Source report: [records/openclaw-openclaw/items/13870.md](records/openclaw-openclaw/items/13870.md)

## Summary

Keep open: current main still lacks the requested durable, operator-owned paired-device naming workflow. The existing code stores and displays client-supplied `displayName` metadata or the raw device id, but there is no separate label field, paired-device rename method, CLI command, or Control UI action.

## Plan

Implement a durable paired-device naming workflow for https://github.com/openclaw/openclaw/issues/13870. Add an operator-owned label field separate from client-supplied `displayName`, expose an additive Gateway method/schema/handler with existing paired-device self/admin authorization semantics, add `openclaw devices rename --device <id> --name <label>` (including JSON/error behavior consistent with devices commands), add a Control UI rename action, document display precedence, and add a user-facing changelog entry. Keep this separate from broader device metadata display work in https://github.com/openclaw/openclaw/issues/23916 and do not change `node.rename` semantics. Preserve token redaction, `approvedScopes` redaction, and terminal/UI sanitization for device-controlled fields.
