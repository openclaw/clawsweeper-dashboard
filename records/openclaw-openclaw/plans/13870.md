## Summary

Keep open: current main and latest release still lack a durable operator-owned paired-device label plus the rename protocol/schema/handler, CLI command, and Control UI action. Related metadata-display work is useful context, but it does not replace this end-to-end naming workflow.

## Plan

Implement a durable paired-device rename workflow for https://github.com/openclaw/openclaw/issues/13870. Add an operator-owned paired-device label/friendly-name field separate from client-supplied displayName so reconnect metadata cannot overwrite it. Expose an additive Gateway protocol/schema/method and server handler, add an `openclaw devices rename --device <id> --name <label>` CLI flow, add Control UI display/action support, document display precedence, and add regression coverage. Keep node.rename and broader metadata display work separate; do not fold in timestamp/browser/OS display changes except where needed to show the new label. Preserve existing paired-device authorization boundaries and add authz coverage for rename.
