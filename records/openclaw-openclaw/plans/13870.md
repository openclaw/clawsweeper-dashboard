## Summary

Keep open: current main still lacks the durable, operator-owned paired-device label and rename workflow requested here. Existing code can show client-supplied `displayName` or raw `deviceId`, and reconnects can overwrite `displayName`, so the remaining work is not covered by the related metadata-display issue or open display-only PR.

## Plan

Implement durable human-friendly names for paired devices. Add an operator-owned label field separate from reconnect/client `displayName` metadata, an additive Gateway method/schema/handler for renaming a paired device, CLI support such as `openclaw devices rename --device <id> --name <name>`, Control UI display/action support, docs for label precedence, and focused regression/authz tests. Preserve existing `--json` compatibility, sanitize all device-controlled terminal output, do not fold in broader metadata/table-column work from https://github.com/openclaw/openclaw/issues/23916 or https://github.com/openclaw/openclaw/pull/57449, and add a changelog entry because this is user-visible.
