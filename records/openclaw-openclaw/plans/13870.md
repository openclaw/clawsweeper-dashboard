## Plan

Implement human-friendly paired-device labels for https://github.com/openclaw/openclaw/issues/13870. Add a durable operator-owned label field separate from client reconnect displayName, expose an additive Gateway protocol/schema/handler method for renaming paired devices, add `openclaw devices rename --device <id> --name <label>`, render the label with clear precedence in `openclaw devices list` and the Control UI devices/nodes panel, and update docs. Preserve existing paired-device self/admin authorization boundaries, do not fold in broader metadata-display work from https://github.com/openclaw/openclaw/issues/23916, sanitize device-controlled terminal output, and keep JSON output backwards-compatible except for additive fields. Add focused tests for storage, gateway method/authz, CLI command/list output, and Control UI rendering.
