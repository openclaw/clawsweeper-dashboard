# Coding Plan for openclaw/openclaw#13870: Feature Request: Human-friendly device names in paired devices list

Source report: [records/openclaw-openclaw/items/13870.md](records/openclaw-openclaw/items/13870.md)

## Summary

Keep open: current main still lacks a durable, operator-owned paired-device label or rename workflow. The existing code stores and displays client-supplied `displayName` metadata or the raw device id, and the related display-only issues/PRs do not replace this rename feature.

## Plan

Add a durable human-friendly paired-device label workflow for https://github.com/openclaw/openclaw/issues/13870. Store an operator-owned label separately from client-supplied reconnect metadata; do not reuse `displayName` as the durable override. Add an additive Gateway protocol/schema/method such as `device.pair.rename` with existing paired-device management authz semantics, wire it through `src/gateway/server-methods/devices.ts`, `src/gateway/server-methods-list.ts`, and `src/gateway/method-scopes.ts`, expose `openclaw devices rename --device <id> --name <label>`, and add a Control UI rename action in the Devices/Nodes panel. Document display precedence and keep broader metadata columns tracked by related display issues/PRs separate. Preserve token redaction and sanitize every user/device-controlled value in terminal/UI output. Add changelog coverage because this is user-facing.
