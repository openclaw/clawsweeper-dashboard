# Coding Plan for openclaw/openclaw#13870: Feature Request: Human-friendly device names in paired devices list

Source report: [records/openclaw-openclaw/items/13870.md](records/openclaw-openclaw/items/13870.md)

## Summary

Keep open: current main and the latest shipped tag still lack the durable operator-owned paired-device label and rename workflow needed for human-friendly device names. Related device-list metadata work is useful context, but it does not replace the requested end-to-end naming feature.

## Plan

Implement human-friendly names for paired devices. Add a durable operator-owned paired-device label separate from reconnect-managed displayName in src/infra/device-pairing.ts, expose an additive Gateway protocol/schema/handler/method descriptor such as device.pair.rename with the same self/admin authorization shape as paired-device management, add openclaw devices rename --device <id> --name <label> and display precedence in the human table while preserving --json compatibility, and add a Control UI rename action in the Devices/Nodes panel. Update docs/cli/devices.md and docs/gateway/protocol.md, add focused tests for storage, Gateway method/authz, CLI output/command behavior, and UI rendering/action wiring, and add a user-facing changelog entry if the repair lane handles changelog. Do not fold in the broader metadata columns from https://github.com/openclaw/openclaw/issues/23916, do not change node.rename semantics, and do not store user labels in reconnect displayName.
