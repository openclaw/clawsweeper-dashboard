# Coding Plan for openclaw/openclaw#13870: Feature Request: Human-friendly device names in paired devices list

Source report: [records/openclaw-openclaw/items/13870.md](records/openclaw-openclaw/items/13870.md)

## Summary

Keep open: current main still lacks the durable, operator-owned paired-device label and rename workflow requested here. Existing surfaces can display client-supplied `displayName` or a raw `deviceId`, while related metadata-display work at https://github.com/openclaw/openclaw/issues/23916 and https://github.com/openclaw/openclaw/pull/57449 does not implement user-managed paired-device names.

## Plan

Implement the paired-device naming workflow requested in https://github.com/openclaw/openclaw/issues/13870. Add a durable operator-owned label/name field separate from client reconnect `displayName`, add an additive Gateway protocol/schema/handler method for renaming paired devices with the same self/admin authorization shape as existing paired-device management, expose `openclaw devices rename --device <id> --name <label>` plus JSON behavior, add a Control UI action to edit the paired-device label, and document display precedence. Keep this separate from broader metadata-display work in https://github.com/openclaw/openclaw/issues/23916 and do not change `node.rename` semantics. Preserve token redaction and terminal sanitization, update focused regression tests, and add a user-facing changelog entry.
