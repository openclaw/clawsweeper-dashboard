# Coding Plan for openclaw/openclaw#13870: Feature Request: Human-friendly device names in paired devices list

Source report: [records/openclaw-openclaw/items/13870.md](records/openclaw-openclaw/items/13870.md)

## Summary

Keep open: current main still has no operator-owned paired-device label or rename workflow. The existing implementation displays client-supplied `displayName` or `deviceId`, while the related metadata-display issue and open display-only PRs do not replace this user-settable naming request.

## Plan

Implement human-friendly paired-device labels for https://github.com/openclaw/openclaw/issues/13870. Add a durable operator-owned label field separate from client-supplied `displayName`; do not reuse reconnect metadata as the label. Expose an additive Gateway method such as `device.pair.rename` with schema, method list, method scopes, handler, and tests that mirror existing self/admin authorization boundaries. Add `openclaw devices rename --device <id> --name <name>` and a clear/remove-label behavior if appropriate. Update the Control UI Devices panel with a rename action and display precedence of label, then client displayName, then deviceId. Update docs and changelog for the user-facing workflow. Keep `node.rename` and broader metadata-display work in https://github.com/openclaw/openclaw/issues/23916 separate; do not fold in the open display-only PRs.
