# Coding Plan for openclaw/openclaw#13870: Feature Request: Human-friendly device names in paired devices list

Source report: [records/openclaw-openclaw/items/13870.md](records/openclaw-openclaw/items/13870.md)

## Summary

Keep open: current main still lacks the durable, operator-owned paired-device label and rename workflow requested here. The existing code can display client-supplied displayName metadata or a raw deviceId, but there is no separate label field, Gateway method, CLI command, or Control UI action for user-managed names.

## Plan

Implement the paired-device naming workflow requested at https://github.com/openclaw/openclaw/issues/13870. Add a durable operator-owned label/name field separate from client-supplied displayName/reconnect metadata; add an additive Gateway protocol schema, advertised method, handler, and authorization checks for paired-device rename/label updates; add openclaw devices rename --device <id> --name <name> and render label/displayName/deviceId with documented precedence; add a Control UI rename action for paired devices; update docs and a user-facing changelog entry. Keep broader metadata display from https://github.com/openclaw/openclaw/issues/23916 and https://github.com/openclaw/openclaw/pull/57449 separate, preserve --json compatibility, sanitize all device-controlled terminal output, and do not expose token secrets or approvedScopes.
