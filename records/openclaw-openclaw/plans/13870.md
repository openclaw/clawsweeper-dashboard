# Coding Plan for openclaw/openclaw#13870: Feature Request: Human-friendly device names in paired devices list

Source report: [records/openclaw-openclaw/items/13870.md](records/openclaw-openclaw/items/13870.md)

## Summary

Current main still lacks the requested operator-owned paired-device naming workflow: it can display client-supplied displayName metadata or the deviceId, but there is no durable user label, rename command, Gateway method, or Control UI rename action.

## Plan

Implement human-friendly paired-device labels for https://github.com/openclaw/openclaw/issues/13870. Add a durable operator-owned paired-device label field separate from client displayName metadata, expose an additive Gateway device.pair rename method with schema/scope/server handler support, add an openclaw devices rename command, and add a Control UI rename action in the Devices/Nodes panel. Preserve existing device-token self/admin authorization boundaries and do not merge this with node.rename or the broader metadata-display work in https://github.com/openclaw/openclaw/issues/23916. Document precedence rules for label, client displayName, and deviceId; do not edit CHANGELOG.md. Keep device-controlled metadata sanitized in CLI/UI output.
