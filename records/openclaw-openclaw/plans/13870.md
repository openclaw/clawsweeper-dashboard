## Summary

Keep open: current main and the latest shipped release still lack a durable operator-owned paired-device label, rename protocol/handler, CLI rename command, and Control UI rename action. The related metadata-display work remains separate and unmerged, so closing this would lose the canonical rename workflow.

## Plan

Implement https://github.com/openclaw/openclaw/issues/13870 as a narrow paired-device rename/label workflow. Add a durable operator-owned paired-device label field separate from client reconnect metadata/displayName, expose an additive Gateway method/schema/handler such as `device.pair.rename`, add `openclaw devices rename --device <id> --name <label>` with JSON behavior preserved, and add a Control UI action for paired devices. Document label/displayName/deviceId precedence in CLI and protocol docs, sanitize all user/client-controlled terminal output, preserve token redaction and self/admin authorization boundaries, and do not fold in the broader metadata columns/detail-mode work from https://github.com/openclaw/openclaw/issues/23916 or the stale/conflicting display PR https://github.com/openclaw/openclaw/pull/57449. Add focused regression tests and a user-facing changelog entry if release policy requires it.
