## Summary

Keep open: current main still lacks a durable operator-owned paired-device label and any rename path in storage, Gateway protocol, CLI, or Control UI. The related metadata-display issue https://github.com/openclaw/openclaw/issues/23916 and open display-only PR https://github.com/openclaw/openclaw/pull/57449 do not supersede this rename workflow.

## Plan

Implement the paired-device human-friendly naming workflow for https://github.com/openclaw/openclaw/issues/13870. Add a durable operator-owned label field separate from client-supplied `displayName`, expose an additive `device.pair.rename` Gateway method/schema/handler with existing self/admin authorization semantics, add `openclaw devices rename --device <id> --name <label>` plus display precedence in `devices list`, wire a Control UI edit/rename action in the Devices panel, document the command and precedence rules, and add a user-facing `CHANGELOG.md` entry. Keep this narrow: do not fold in the broader metadata display work from https://github.com/openclaw/openclaw/issues/23916 or https://github.com/openclaw/openclaw/pull/57449 except where needed for label precedence, preserve `--json`, and sanitize all device-controlled terminal output.
