# Coding Plan for openclaw/openclaw#13870: Feature Request: Human-friendly device names in paired devices list

Source report: [records/openclaw-openclaw/items/13870.md](records/openclaw-openclaw/items/13870.md)

## Summary

Current main still lacks a durable operator-owned paired-device name and an end-to-end rename workflow; existing `displayName` data is client-owned metadata, while the related items only cover broader metadata or timestamps.

## Plan

Implement human-friendly paired-device names as a durable operator-owned label separate from client-supplied `displayName`. Add an additive Gateway protocol schema and method using existing self/admin authorization semantics; persist and return the label; add `openclaw devices rename --device <id> --name <name>` and a Control UI rename action; and define display precedence as operator label, then client display name, then device ID. Add regression coverage proving reconnect metadata cannot overwrite the operator label and authorization prevents unauthorized renames. Update focused CLI and Gateway protocol documentation. Keep https://github.com/openclaw/openclaw/issues/23916 and https://github.com/openclaw/openclaw/pull/55973 out of scope, do not introduce a config option, and do not edit CHANGELOG.md.
