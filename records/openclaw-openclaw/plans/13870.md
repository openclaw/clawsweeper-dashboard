## Summary

Keep open: current main and the latest release still lack a durable operator-owned paired-device label, rename RPC/schema/handler, CLI command, and Control UI action. The related metadata-display issue and open CLI-only PR are useful context, but they do not replace this end-to-end naming workflow.

## Plan

Implement the paired-device naming workflow for https://github.com/openclaw/openclaw/issues/13870. Add a durable operator-owned label/name field separate from reconnect-owned displayName, add an additive Gateway protocol schema and handler such as device.pair.rename with the same self/admin authorization boundaries as paired-device management, expose it through `openclaw devices rename --device <id> --name <label>`, render the label in Control UI Devices/Nodes, and document display precedence. Do not fold in the broader metadata/table-column work tracked by https://github.com/openclaw/openclaw/issues/23916 or rely on client displayName as the user label. Add focused storage, gateway, CLI, and UI tests, update docs and changelog for the user-facing feature, and preserve JSON output plus terminal sanitization for device-controlled fields.
