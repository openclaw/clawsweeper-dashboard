# Coding Plan for openclaw/openclaw#13870: Feature Request: Human-friendly device names in paired devices list

Source report: [records/openclaw-openclaw/items/13870.md](records/openclaw-openclaw/items/13870.md)

## Summary

Keep open: current main and v2026.5.20 still lack a durable operator-owned paired-device label, paired-device rename protocol/schema/handler, CLI rename command, and Control UI rename action. The broader metadata issue at https://github.com/openclaw/openclaw/issues/23916 and timestamp PR at https://github.com/openclaw/openclaw/pull/55973 are related display work, but they do not replace this naming workflow.

## Plan

Implement durable human-friendly labels for paired devices. Add a separate operator-owned paired-device label field rather than reusing reconnect-owned displayName; add an additive Gateway protocol schema and handler such as device.pair.rename with the same paired-device management authorization boundaries as remove/token management; expose it through `openclaw devices rename --device <id> --name <label>`; add a Control UI rename action in the Devices/Nodes panel; document label precedence and command usage. Keep this separate from broader metadata display in https://github.com/openclaw/openclaw/issues/23916 and timestamp work in https://github.com/openclaw/openclaw/pull/55973. Add focused tests for storage persistence, server authz, CLI command behavior, UI rendering/action wiring, and docs; include a user-facing changelog entry if the fix PR changes behavior.
