# Coding Plan for openclaw/openclaw#13870: Feature Request: Human-friendly device names in paired devices list

Source report: [records/openclaw-openclaw/items/13870.md](records/openclaw-openclaw/items/13870.md)

## Summary

Keep open: current main and v2026.5.22 still lack a durable operator-owned paired-device label, paired-device rename protocol/schema/handler, CLI rename command, and Control UI rename action. The related metadata work at https://github.com/openclaw/openclaw/issues/23916 and timestamp PR at https://github.com/openclaw/openclaw/pull/55973 do not replace this naming workflow.

## Plan

Implement human-friendly paired-device labels for this issue. Add a durable operator-owned label field separate from reconnect-owned `displayName`; add an additive Gateway protocol/schema/method and handler for renaming a paired device with the same self/admin authorization shape as paired-device remove/rotate/revoke; add `openclaw devices rename --device <id> --name <label>`; show label precedence in `openclaw devices list` and the Control UI Devices/Nodes panel without changing `--json` compatibility except for the additive field; add/update focused tests for storage, Gateway methods/authz, CLI, and Control UI rendering; update `docs/cli/devices.md` and `docs/gateway/protocol.md`; do not fold in broader metadata/timestamp display work from https://github.com/openclaw/openclaw/issues/23916 or https://github.com/openclaw/openclaw/pull/55973, and do not reuse `displayName` as the operator label.
