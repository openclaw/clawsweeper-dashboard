## Summary

This should stay open: current main and the latest release still lack a durable operator-owned paired-device label, rename protocol/schema/handler, CLI command, and Control UI action. The related metadata-display issue and open CLI-only PR do not replace the requested end-to-end naming workflow.

## Plan

Implement human-friendly paired-device names as a durable operator-owned label, separate from client-supplied reconnect metadata. Add storage support in `src/infra/device-pairing.ts`, additive Gateway protocol/schema/method handling for paired-device rename, CLI support such as `openclaw devices rename --device <id> --name <label>`, and a Control UI rename action/display precedence that shows the operator label while retaining the hash ID for destructive commands. Preserve existing self/admin authorization boundaries, do not fold in the broader metadata display work from https://github.com/openclaw/openclaw/issues/23916 or the CLI-only PR https://github.com/openclaw/openclaw/pull/57449, and update docs for the new command and protocol method. Add focused regression coverage for storage, gateway method/authz, CLI output/rename behavior, and UI controller/view behavior; include a changelog entry only if the maintainer landing workflow requires one.
