## Summary

Keep open: current main and the latest release still lack a durable operator-owned paired-device label, a paired-device rename protocol/schema/handler, a CLI rename command, and a Control UI rename action. The broader device metadata discussion at https://github.com/openclaw/openclaw/issues/23916 is related but does not replace this naming workflow.

## Plan

Implement a durable human-friendly paired-device label workflow for https://github.com/openclaw/openclaw/issues/13870. Add a separate operator-owned label field to paired-device storage rather than reusing reconnect-owned displayName; add an additive Gateway protocol schema/method/handler for renaming paired devices with the same self/admin authorization shape as paired-device management; add `openclaw devices rename --device <id> --name <name>`; add Control UI support to rename paired devices; document display precedence in the CLI and Gateway protocol docs; add/update focused tests. Do not fold in the broader metadata-display columns tracked by https://github.com/openclaw/openclaw/issues/23916 or the timestamp PR at https://github.com/openclaw/openclaw/pull/55973, and do not expose token secrets or approvedScopes. Include a user-facing changelog entry if this lands as a user-visible repair PR.
