# Coding Plan for openclaw/openclaw#13870: Feature Request: Human-friendly device names in paired devices list

Source report: [records/openclaw-openclaw/items/13870.md](records/openclaw-openclaw/items/13870.md)

## Summary

Current main still lacks the requested operator-owned paired-device naming workflow; existing `displayName` is client metadata, not a durable user label, and there is no rename protocol, CLI command, or Control UI action.

## Plan

Implement human-friendly paired-device labels for https://github.com/openclaw/openclaw/issues/13870. Add an additive operator-owned label field to the canonical paired-device record, separate from client-supplied `displayName`; make display precedence operator label, then client `displayName`, then `deviceId`; expose one authenticated Gateway rename method with schemas/validators/descriptors/handlers using the existing paired-device self/admin authorization model; add `openclaw devices rename --device <id> --name <label>` and Control UI rename action; update docs for CLI and Gateway protocol. Do not fold in the broader metadata-display work from https://github.com/openclaw/openclaw/issues/23916, do not reuse `displayName` as the operator label, do not add config/env options, and do not edit CHANGELOG.md.
