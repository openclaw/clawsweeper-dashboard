# Coding Plan for openclaw/openclaw#13870: Feature Request: Human-friendly device names in paired devices list

Source report: [records/openclaw-openclaw/items/13870.md](records/openclaw-openclaw/items/13870.md)

## Summary

Keep open: current main still lacks a durable, operator-owned paired-device label and rename workflow. The existing CLI and Control UI only render client-supplied `displayName` or the raw `deviceId`, and the related metadata-display issue/PR do not implement editable paired-device names.

## Plan

Fix https://github.com/openclaw/openclaw/issues/13870 by adding a durable operator-owned paired-device label workflow. Add a label/friendly-name field separate from client reconnect metadata in paired-device storage, an additive Gateway protocol/schema/method such as `device.pair.rename` with existing paired-device self/admin authorization semantics, a CLI command like `openclaw devices rename --device <id> --name <label>`, and a Control UI action/display update. Document label/display precedence and keep https://github.com/openclaw/openclaw/issues/23916 metadata-display work separate. Preserve `--json`, do not expose token secrets or `approvedScopes`, sanitize all device-controlled terminal output, and add focused regression coverage plus a user-facing changelog entry.
