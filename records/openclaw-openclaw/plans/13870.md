## Summary

Keep open: current main still lacks the durable, operator-owned paired-device naming workflow requested here. The existing surfaces can show client-supplied `displayName` metadata or the raw device id, while the related open display work does not implement a rename/label feature.

## Plan

Implement the paired-device naming feature requested at https://github.com/openclaw/openclaw/issues/13870. Add a durable operator-owned paired-device label/name stored separately from client reconnect `displayName` metadata; do not reuse client `displayName` as the editable label. Expose the feature additively through Gateway protocol/schema, server handlers and method scopes, `openclaw devices rename --device <id> --name <label>` or an equivalent command, and the Control UI Devices/Nodes panel. Preserve existing self/admin paired-device authorization rules, keep `--json` backward-compatible, sanitize all device-controlled terminal/UI output, avoid exposing token secrets or `approvedScopes`, and keep broader metadata display at https://github.com/openclaw/openclaw/issues/23916 and https://github.com/openclaw/openclaw/pull/57449 separate. Add focused regression tests and update public docs plus changelog if the change is user-facing.
