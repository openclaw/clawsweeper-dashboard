# Coding Plan for openclaw/openclaw#13870: Feature Request: Human-friendly device names in paired devices list

Source report: [records/openclaw-openclaw/items/13870.md](records/openclaw-openclaw/items/13870.md)

## Summary

Keep open: current main still lacks a durable operator-owned paired-device label and has no rename workflow in storage, Gateway protocol, CLI, Control UI, or docs; existing client `displayName` metadata is not a safe substitute because reconnect metadata can rewrite it.

## Plan

Implement #13870 as a narrow paired-device label workflow. Add a durable operator-owned label field separate from client `displayName`, with display precedence label > client displayName > deviceId. Expose an additive Gateway protocol/schema/handler such as `device.pair.rename` or the maintainer-approved equivalent, mirror existing paired-device self/admin authorization boundaries, add `openclaw devices rename --device <id> --name <label>` support, surface rename in the Control UI devices panel, and document CLI/Gateway behavior. Do not fold in the broader metadata/detail work from #23916, and do not reuse client `displayName` as the operator-owned label.
