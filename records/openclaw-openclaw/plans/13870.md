# Coding Plan for openclaw/openclaw#13870: Feature Request: Human-friendly device names in paired devices list

Source report: [records/openclaw-openclaw/items/13870.md](records/openclaw-openclaw/items/13870.md)

## Summary

Keep open: current main still lacks a durable operator-owned paired-device label, paired-device rename protocol/schema/handler, CLI rename command, and Control UI rename action. Related display work covers metadata/timestamps, but it does not replace this canonical naming workflow.

## Plan

Implement a durable human-friendly paired-device label workflow for this issue. Add an operator-owned label/friendlyName field separate from client-refresh `displayName`; add additive Gateway protocol schema/validator/descriptors and a server handler such as `device.pair.rename` with the same self/admin authorization shape as paired-device management; add `openclaw devices rename --device <id> --name <label>`; add a Control UI action/button for paired device rename; display label precedence clearly without changing `--json` secrets behavior; document CLI and Gateway protocol changes; add/update regression tests. Do not fold in broad metadata columns, timestamp display, or `--full` work from https://github.com/openclaw/openclaw/issues/23916 or https://github.com/openclaw/openclaw/pull/55973. Add a `CHANGELOG.md` entry under the active version because this is user-visible.
