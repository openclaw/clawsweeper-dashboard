# Coding Plan for openclaw/openclaw#13870: Feature Request: Human-friendly device names in paired devices list

Source report: [records/openclaw-openclaw/items/13870.md](records/openclaw-openclaw/items/13870.md)

## Summary

Keep open: current main and v2026.6.8 still lack a durable operator-owned paired-device label and rename workflow; the existing `displayName` is client metadata that reconnects can refresh, and the related metadata issue does not replace this naming workflow.

## Plan

Implement https://github.com/openclaw/openclaw/issues/13870 as a narrow paired-device label workflow. Add a durable operator-owned label separate from client `displayName`, define label-over-client-displayName-over-deviceId display precedence, expose an additive Gateway protocol method/schema/handler with self/admin authorization matching existing paired-device management, add `openclaw devices rename --device <id> --name <label>`, add a Control UI rename action, and update docs. Do not fold in the broader device metadata/detail work from https://github.com/openclaw/openclaw/issues/23916, do not change `--json` shape except for the additive label field, and preserve token redaction and terminal sanitization for device-controlled metadata.
