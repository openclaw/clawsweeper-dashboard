# Coding Plan for openclaw/openclaw#13870: Feature Request: Human-friendly device names in paired devices list

Source report: [records/openclaw-openclaw/items/13870.md](records/openclaw-openclaw/items/13870.md)

## Summary

Keep open: current main and v2026.6.8 still lack a durable operator-owned paired-device label and any paired-device rename flow. The existing `displayName` is client-supplied metadata and reconnect paths can refresh it, so it is not a safe substitute for the requested user label.

## Plan

Implement a durable operator-owned paired-device label workflow for https://github.com/openclaw/openclaw/issues/13870. Add a label field separate from client-controlled displayName, make display precedence explicit as operator label over client displayName over deviceId, expose an additive Gateway protocol/schema/handler method for renaming paired devices, add `openclaw devices rename --device <id> --name <label>`, add a Control UI action in the Devices/Nodes panel, and update docs. Preserve existing token secret redaction, terminal-output sanitization, `--json` behavior, and paired-device self/admin authorization boundaries. Keep this separate from broader metadata/detail display in https://github.com/openclaw/openclaw/issues/23916 and do not add config/env options or changelog edits.
