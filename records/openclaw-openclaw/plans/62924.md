Keep open. Current `main` and the latest release still carry media-understanding provider/model telemetry internally, but the assistant-visible inbound body is formatted without that telemetry, so the guessed-model failure remains source-reproducible and narrowly fixable.

## Plan

Fix openclaw/openclaw issue https://github.com/openclaw/openclaw/issues/62924. The media-understanding runtime already records `MediaUnderstandingOutput.provider` and optional `model`, but `formatMediaUnderstandingBody` omits that telemetry before `applyMediaUnderstanding` forces the formatted body into `BodyForAgent`. Add a narrow formatter/apply behavior that renders `Analysis model:` from real successful current-turn outputs, handles missing model values explicitly, and does not expose `openclaw/synthetic-empty-audio` or any guessed conversation/default model as actual analysis telemetry. Update focused formatter/apply tests and add a user-facing changelog entry.
