Source report: [records/openclaw-openclaw/items/60078.md](records/openclaw-openclaw/items/60078.md)

## Summary

Keep open: current main still has the reported model-routing gap. Direct subagent announce delivery delegates model selection to shared agent ingress without provider/model overrides, and that ingress path resolves the agent default, stored overrides, and explicit run overrides but does not apply the existing `channels.modelByChannel` resolver that normal reply and status paths use.

## Plan

Fix https://github.com/openclaw/openclaw/issues/60078 by wiring `channels.modelByChannel` into shared agent ingress for channel-backed direct subagent announce delivery. Reuse `resolveChannelModelOverride` rather than adding announce-specific model policy; preserve precedence so explicit run overrides and stored session overrides still win over channel overrides, and preserve alias normalization, model visibility/allowlist checks, auth-profile validation, and auto-fallback probe behavior. Add focused regression tests for direct subagent announce using a channel-pinned parent session, explicit override precedence, stored override precedence if not already covered, and no-channel fallback to the agent default. Include a user-facing changelog entry unless maintainers say this will be handled during landing. Do not change unrelated subagent delivery routing or provider fallback policy.
