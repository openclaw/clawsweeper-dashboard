# Coding Plan for openclaw/openclaw#60078: [Bug]: Announce delivery ignores modelByChannel, always uses agent default model

Source report: [records/openclaw-openclaw/items/60078.md](records/openclaw-openclaw/items/60078.md)

## Summary

Keep open: current main and the latest release still have the reported model-selection gap for direct subagent announce delivery. The existing `channels.modelByChannel` resolver is used by normal reply/status paths, but the announce path enters shared agent ingress without provider/model overrides and that ingress path still falls back to the agent default before stored or explicit overrides.

## Plan

Fix https://github.com/openclaw/openclaw/issues/60078 by making direct subagent announce delivery honor `channels.modelByChannel` for the parent/requester session channel. Reuse the existing `resolveChannelModelOverride` contract in shared agent ingress or an equally shared model-selection helper; do not add an announce-only duplicate resolver or change the public config shape. Preserve precedence so explicit run overrides and stored session overrides win over channel overrides, and channel overrides win over the agent default. Preserve alias normalization, model visibility checks, auth-provider validation, and auto-fallback probe behavior. Add regression coverage for direct subagent announce using a channel-pinned model, explicit override precedence, stored override precedence, and no-channel fallback. Add a single-line Fixes changelog entry if the repair is user-visible.
