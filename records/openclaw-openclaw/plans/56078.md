## Summary

Keep this issue open. Current main and the latest shipped tag still route default Control UI and backend isolated agent-turn cron jobs through implicit announce delivery, so a local/no-channel setup can still hit the reported delivery-target error unless the job explicitly selects no delivery or a valid delivery target.

## Plan

Fix https://github.com/openclaw/openclaw/issues/56078. In local/no-channel setups, default or otherwise implicit isolated agent-turn cron jobs must not fail solely because fallback announce delivery resolves to delivery.channel="last" with no configured channel; the agent should be able to complete and write requested files when no explicit delivery target was chosen. Keep explicit announce/webhook delivery strict, preserve explicit delivery.mode="none" semantics, and do not globally weaken resolveMessageChannelSelection for normal send paths. Review the active related PR https://github.com/openclaw/openclaw/pull/80438 before editing to avoid conflicting with admission-time implicit-last validation. Add focused regression coverage for the UI-created default path and backend-created isolated agent-turn path, plus target/dispatch behavior proving explicit misconfigured delivery still reports useful errors. Update docs or CHANGELOG only if the user-visible cron delivery behavior/workaround changes.
