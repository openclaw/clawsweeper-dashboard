## Plan

Fix https://github.com/openclaw/openclaw/issues/56078. Current main still defaults Control UI and backend-created isolated `agentTurn` cron jobs to strict `announce` delivery against `last`; in a local/no-channel setup this reaches `Channel is required (no configured channels detected)` and dispatch treats the unresolved target as fatal. Implement a narrow fix so implicit cron-owned no-channel isolated runs do not require fallback announce delivery unless the user explicitly chooses a deliverable target. Preserve strict behavior for explicit `delivery.mode="announce"` with explicit channel/target and for webhook delivery.
