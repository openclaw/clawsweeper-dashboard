# Coding Plan for openclaw/openclaw#62056: Feature: Expose Feishu bot added/removed events as hook events

## Summary

Keep open. Current main and the latest release still only parse and log the Feishu bot-added and bot-deleted subscription events, and neither the hook type contract nor public hook docs expose a Feishu channel hook event family.

## Plan

Implement Feishu bot membership hook events for https://github.com/openclaw/openclaw/issues/62056. Keep the change inside the Feishu plugin boundary plus the minimal generic hook contract/docs needed for additive event keys. In `extensions/feishu/src/monitor.account.ts`, dispatch internal hook events when `im.chat.member.bot.added_v1` and `im.chat.member.bot.deleted_v1` arrive; do not add polling, startup migrations, or core Feishu policy. Add `channel` to `InternalHookEventType` if needed so hooks can subscribe to `channel:feishu:bot-added` and `channel:feishu:bot-removed`. Include context for `chatId`, `accountId`, and a timestamp; include `chatName` only if confirmed by the Feishu/Lark event payload or fetched through an existing safe helper without new polling. Add focused regression tests for both handlers, update `docs/automation/hooks.md`, `docs/channels/feishu.md` if appropriate, and add a changelog entry. Stop and ask for maintainer guidance if the implementation would require reviving the broader closed plugin hook-runner API from earlier PRs.
