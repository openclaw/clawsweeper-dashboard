# Coding Plan for openclaw/openclaw#38721: [Bug] gateway shutdown timed out: node report shows active child process handle; leaves dirty shutdown + lane wait

Source report: [records/openclaw-openclaw/items/38721.md](records/openclaw-openclaw/items/38721.md)

## Summary

Keep open: current main and v2026.5.12 have important shutdown mitigations for service-managed children, but the gateway close path still does not explicitly cancel or drain active exec/process sessions, which is the child-handle blocker reported here. The related session-wide process-control feature remains open and should not absorb this narrower shutdown cleanup bug.

## Plan

Fix https://github.com/openclaw/openclaw/issues/38721 with a narrow internal shutdown cleanup. During gateway shutdown/restart, active exec/process child handles should not keep shutdown waiting until the watchdog logs `shutdown timed out; exiting without full cleanup`. Reuse or expose a small internal helper around the bash process registry/process supervisor to cancel active supervised process sessions and fall back to process-tree kill when only a pid is available; call it from the gateway close path with a bounded wait/warning. Do not add public `sessions_kill`, `exec-list`, CLI/RPC/API surface, and do not change `sessions.abort` semantics; that broader operator work belongs to https://github.com/openclaw/openclaw/issues/12554. Preserve current service-managed child behavior and systemd `KillMode=control-group`. Add focused regression coverage for shutdown cleanup with active foreground/background exec/process sessions and ensure existing websocket/HTTP shutdown timeout coverage still passes. Add a changelog entry if the repair is prepared as a maintainer-owned user-visible fix.
