## Summary

Keep open because current main still lacks a Telegram config-level outbound topic default per agent. The existing topic config remains inbound/session routing only, strict validation rejects the proposed extra topic field, and outbound threading only resolves an explicit thread/topic target or the current Telegram tool context.

## Plan

Implement the requested Telegram per-agent outbound topic default for proactive sends. Add typed topic-level outbound-default metadata to Telegram config, schema, generated config metadata/docs, and tests. Update the channel threading seam only as an additive optional input if needed so Telegram can resolve the sending agent's configured default; do not hard-code Telegram topic policy in generic outbound core. Preserve precedence: explicit `threadId` or `:topic:` target first, current Telegram tool context second, configured per-agent default last, with deterministic handling or validation for duplicate defaults. Add a user-facing CHANGELOG entry because this is a Telegram config feature.
