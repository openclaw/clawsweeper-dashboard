## Plan

Implement the focused REST session-management surface requested by https://github.com/openclaw/openclaw/issues/20934. Add authenticated Gateway HTTP endpoints for session reset, status, compact, model, and reasoning controls over the existing `sessions.list`/`sessions.describe` or equivalent status helper, `sessions.reset`, `sessions.compact`, and `sessions.patch` semantics. Add route/auth/request/response tests, update public Gateway HTTP docs, add a changelog entry because this is a user-facing API feature, and coordinate naming/shape with https://github.com/openclaw/openclaw/issues/47826 without collapsing the two trackers.
