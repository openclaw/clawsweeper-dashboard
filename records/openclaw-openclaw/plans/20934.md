## Summary

Keep open. Current main still lacks REST-native session reset/status/compact/model/reasoning endpoints for HTTP API integrations, and the related closed PRs did not land an HTTP session-management surface.

## Plan

Implement the REST session-management surface requested in https://github.com/openclaw/openclaw/issues/20934. Add authenticated Gateway HTTP routes for session reset, status, compact, model override, and reasoning override, reusing the existing `sessions.reset`, `sessions.list`/`sessions.describe`, `sessions.compact`, and `sessions.patch` semantics rather than duplicating business logic. Keep `/v1/chat/completions` slash-command handling default-deny; do not set `CommandAuthorized` for HTTP chat messages. Coordinate the read-only/status response shape with https://github.com/openclaw/openclaw/issues/47826 but keep this PR focused on session management. Update public docs for the OpenAI-compatible/Gateway HTTP surface and add a user-facing changelog entry. Do not add plugin-only APIs as a substitute for REST endpoints, and do not resurrect the closed https://github.com/openclaw/openclaw/pull/23941 implementation without fixing its known status/schema and route-prefix issues.
