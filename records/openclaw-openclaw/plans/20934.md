## Plan

Implement authenticated REST-native Gateway session-management endpoints for https://github.com/openclaw/openclaw/issues/20934. Add a focused HTTP handler, likely `src/gateway/sessions-http.ts`, and register it from `src/gateway/server-http.ts` for `/v1/sessions/reset`, `/v1/sessions/status`, `/v1/sessions/compact`, `/v1/sessions/model`, and `/v1/sessions/reasoning` or an equivalent documented shape that maps cleanly to existing `sessions.reset`, `sessions.describe`/`sessions.list`, `sessions.compact`, and `sessions.patch` semantics. Reuse existing Gateway HTTP bearer/trusted-proxy auth helpers and request/response utilities; do not re-enable slash-command interception or set `CommandAuthorized` in `/v1/chat/completions`. Keep business logic in existing session RPC/shared helpers rather than duplicating reset/patch/compact behavior. Coordinate status/read-only naming with https://github.com/openclaw/openclaw/issues/47826, but keep this issue focused on session management for HTTP integrations. Add route/auth/request/response tests, update `docs/gateway/openai-http-api.md` and `docs/gateway/index.md`, and add a user-facing changelog entry.
