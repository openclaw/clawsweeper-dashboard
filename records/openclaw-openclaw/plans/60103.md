# Coding Plan for openclaw/openclaw#60103: [Feishu] Auto-normalize @mention syntax across message formats (card/text/post)

Source report: [records/openclaw-openclaw/items/60103.md](records/openclaw-openclaw/items/60103.md)

## Summary

Keep open. Current main and the latest release still pass Feishu agent-authored mention tags through outbound post, card, streaming-card, and proactive-send paths without converting them to the format-specific Feishu syntax; the related open PR only covers part of the post-message structured-mention path, and the streaming notification issue remains separate.

## Plan

Fix Feishu outbound mention-tag normalization for issue https://github.com/openclaw/openclaw/issues/60103. Add a plugin-local helper that converts explicit agent-authored Feishu mention tags to the target outbound format at the send boundaries: post/text payloads should use native Feishu post `at`/text elements or the existing supported post contract, card and structured-card markdown should convert text-style `<at user_id="ou_xxx">Name</at>` to card-style `<at id=ou_xxx></at>`, and text/post paths should not leak card-style tags as literal text. Apply the helper in `extensions/feishu/src/send.ts`, `extensions/feishu/src/reply-dispatcher.ts`, and `extensions/feishu/src/outbound.ts` as needed. Preserve the no-automatic-mention-cascade behavior from https://github.com/openclaw/openclaw/pull/71396: do not reattach inbound `mentionTargets` to ordinary replies, and only normalize explicit tags present in the outbound body. Coordinate with the partial post-message work in https://github.com/openclaw/openclaw/pull/49983; do not create a second public helper/export surface unless it is intentionally part of the Feishu API. Add focused regression tests for post/text, markdown/structured card, streaming close/update, proactive outbound card routing, escaping/literal text preservation, and explicit structured `mentionTargets`. Add a changelog entry because this changes user-visible Feishu message behavior. Stop and ask for maintainer guidance if the fix would require changing streaming notification semantics tracked by https://github.com/openclaw/openclaw/issues/73469.
