## Summary

Keep open: current main and the latest release still do not normalize agent-authored Feishu `<at user_id="...">name</at>` tags at the post, card, proactive outbound, or streaming-card boundaries, and the linked open PR only addresses part of the post-message structured-mention problem.

## Plan

Fix Feishu outbound mention normalization for https://github.com/openclaw/openclaw/issues/60103. Add a Feishu-owned helper that converts explicit agent-authored mention tags between post/text/card forms at the outbound boundaries without reintroducing automatic inherited mention cascades. Cover sendMessageFeishu post payloads, sendMarkdownCardFeishu/sendStructuredCardFeishu card content, proactive outbound routing in extensions/feishu/src/outbound.ts, and streaming card update/close paths in extensions/feishu/src/reply-dispatcher.ts and extensions/feishu/src/streaming-card.ts. Coordinate with https://github.com/openclaw/openclaw/pull/49983 rather than duplicating its partial native-post direction, preserve allowTopLevelReplyFallback behavior, and do not add a new config option. Add focused regression tests for text-style tags flowing into card/post/streaming paths and for preserving the no-automatic-mention-cascade behavior. Add a changelog entry only if this becomes the landing PR.
