# Coding Plan for openclaw/openclaw#60103: [Feishu] Auto-normalize @mention syntax across message formats (card/text/post)

Source report: [records/openclaw-openclaw/items/60103.md](records/openclaw-openclaw/items/60103.md)

## Summary

Keep open: current main and the latest release still route agent-authored Feishu text-style `<at user_id="...">name</at>` tags through post, card, proactive outbound, and streaming-card paths without a format-specific outbound normalizer.

## Plan

Fix Feishu outbound mention syntax normalization for https://github.com/openclaw/openclaw/issues/60103. Add a Feishu-owned helper that detects explicit agent-authored mention tags in message body text and normalizes them for the actual outbound format: text/post should not leave card-only syntax broken, card/streaming-card should convert text-style `<at user_id="ou_xxx">Name</at>` into card-compatible `<at id=ou_xxx></at>`, and post serialization should use a Feishu-supported structured shape when converting body mentions. Apply the helper at sendMessageFeishu/buildFeishuPostMessagePayload, sendStructuredCardFeishu, sendMarkdownCardFeishu, proactive outbound card routing, and FeishuStreamingSession update/close boundaries as needed. Preserve the existing no-automatic-mention-cascade behavior: do not reattach `mentionTargets` to ordinary replies unless the agent explicitly wrote tags in the body. Reconcile overlap with https://github.com/openclaw/openclaw/pull/49983 and keep https://github.com/openclaw/openclaw/issues/73469 as a separate notification-strategy problem. Add focused regression tests for post, markdown/structured card, proactive outbound, streaming-card update/close, and the existing no-cascade dispatcher behavior. Add a user-facing changelog entry if the repair PR changes behavior.
