Current main still lacks outbound Feishu mention-tag normalization: agent-authored text-style `<at user_id="...">...</at>` can still be sent through card, streaming-card, post/text, and proactive outbound paths without conversion to the format-specific Feishu syntax, so the previously identified source-level bug remains actionable.

## Plan

Fix Feishu outbound mention normalization for issue https://github.com/openclaw/openclaw/issues/60103. Apply it at the outbound send boundaries in `extensions/feishu/src/send.ts`, `extensions/feishu/src/reply-dispatcher.ts`, and `extensions/feishu/src/outbound.ts` without changing core channel APIs or adding config.
