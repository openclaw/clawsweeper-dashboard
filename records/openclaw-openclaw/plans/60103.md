# Coding Plan for openclaw/openclaw#60103: [Feishu] Auto-normalize @mention syntax across message formats (card/text/post)

Source report: [records/openclaw-openclaw/items/60103.md](records/openclaw-openclaw/items/60103.md)

## Summary

Keep open. Current main still lacks a Feishu outbound normalizer for agent-authored mention tags, so text-style `<at user_id="...">...</at>` can flow unchanged into card, outbound card, and streaming-card content.

## Plan

Fix https://github.com/openclaw/openclaw/issues/60103 by adding Feishu-owned outbound mention syntax normalization. Implement a helper in `extensions/feishu/src/mention.ts` that converts agent-authored text-style tags like `<at user_id="ou_xxx">Name</at>` to card-style `<at id=ou_xxx></at>` for Feishu card/streaming markdown, and normalizes card-style tags back to the supported post/text form for the non-card send path. Wire it into `sendMessageFeishu`, `sendStructuredCardFeishu`, `sendMarkdownCardFeishu`, outbound `sendText`/payload card routing as appropriate, and streaming updates/close before content reaches Card Kit. Preserve explicit `mentions` prepending, do not convert escaped display text such as `&lt;at ...&gt;`, and keep existing card/presentation sanitization intact. Add focused regression tests for post/text, static card, outbound card, and streaming close/update paths. Add a single `CHANGELOG.md` fix entry because this is user-visible Feishu behavior.
