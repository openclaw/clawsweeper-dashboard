Keep open: current main and the latest shipped release still lack the requested WhatsApp config field and Baileys `ephemeralExpiration` plumbing, and the only linked implementation attempts are closed and unmerged.

## Plan

Implement WhatsApp disappearing-message expiration support for outbound replies requested by https://github.com/openclaw/openclaw/issues/71157. Add optional `channels.whatsapp.disappearingMessagesSeconds` and `channels.whatsapp.accounts.<accountId>.disappearingMessagesSeconds` config with account-level override semantics; positive finite values should be forwarded to Baileys as `MiscMessageGenerationOptions.ephemeralExpiration`, while `0`, `null`, omitted, negative, or non-finite values should mean no OpenClaw override. Keep the change local to WhatsApp plugin/config surfaces; do not add a new core channel abstraction or change dependency versions. Update types/schema/docs/tests and include changelog coverage if the branch is prepared for landing. Cover normal text, quoted replies, media/voice-note trailing text, polls or send-payload paths where applicable, and unset behavior; only add the optional credential-default fallback if Baileys source clearly exposes a stable value to read.
