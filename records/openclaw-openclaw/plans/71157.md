Keep open: current main and the latest stable release still lack the requested WhatsApp config surface and Baileys `ephemeralExpiration` plumbing, while the only implementation attempts are closed and unmerged.

## Plan

Implement WhatsApp disappearing-message expiration support for this issue. Add `channels.whatsapp.disappearingMessagesSeconds` and `channels.whatsapp.accounts.<accountId>.disappearingMessagesSeconds` to the WhatsApp config type/schema/docs, resolve account-level override over channel-level default in the WhatsApp plugin account config path, and pass positive integer values to Baileys as `MiscMessageGenerationOptions.ephemeralExpiration` across text, quoted reply, media, trailing voice-note text, poll/tool/channel send paths where applicable. Preserve existing behavior when omitted, null, or 0; do not add a new core channel abstraction or change non-WhatsApp defaults. Use https://github.com/openclaw/openclaw/pull/73080 and https://github.com/openclaw/openclaw/pull/73083 as implementation context, but rebase the fix cleanly on current main. Add focused regression tests, update WhatsApp docs/config docs as needed, and add a user-facing changelog entry.
