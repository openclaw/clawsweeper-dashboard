Keep open: current main still lacks the requested WhatsApp config field and outbound `ephemeralExpiration` plumbing. The two related implementation PRs are closed and unmerged, and no open duplicate or replacement currently owns the work.

## Plan

Implement https://github.com/openclaw/openclaw/issues/71157 as a WhatsApp plugin-local feature. Add optional `disappearingMessagesSeconds?: number` to WhatsApp shared channel/account config types and strict zod schema, resolve account-level override semantics through the existing account merge/resolution flow, normalize positive finite seconds to a whole-number Baileys expiration while treating 0/null/omitted/non-finite/non-positive as no OpenClaw override, and pass it as `MiscMessageGenerationOptions.ephemeralExpiration` on WhatsApp outbound `sock.sendMessage` paths for normal text, quoted replies, media, the voice-note trailing text payload, polls if the current listener path supports message-generation options, and channel/tool sends. Preserve existing behavior when unset, keep the change in the bundled WhatsApp plugin/config/docs surfaces rather than adding a core abstraction, and use closed PRs https://github.com/openclaw/openclaw/pull/73080 and https://github.com/openclaw/openclaw/pull/73083 only as source context. Add focused tests for channel-level config, account override, unset/invalid values, quoted/media/voice-note send paths, and update `docs/channels/whatsapp.md`, generated config docs/metadata as required, plus `CHANGELOG.md` because this is user-facing.
