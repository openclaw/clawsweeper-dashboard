# Coding Plan for openclaw/openclaw#71157: [Feature]: Support WhatsApp disappearing-message expiration for outbound replies

Source report: [records/openclaw-openclaw/items/71157.md](records/openclaw-openclaw/items/71157.md)

## Summary

Keep open: current main and the latest shipped release still lack the requested WhatsApp config key and Baileys outbound `ephemeralExpiration` plumbing, while the two implementation attempts linked from the discussion are closed and unmerged.

## Plan

Implement WhatsApp disappearing-message expiration for outbound replies. Add optional `channels.whatsapp.disappearingMessagesSeconds` and per-account `channels.whatsapp.accounts.<accountId>.disappearingMessagesSeconds` config, with account-level override and positive integer values forwarded as Baileys `MiscMessageGenerationOptions.ephemeralExpiration`; treat 0, null, omitted, non-finite, or negative values as no explicit override. Keep the change inside the WhatsApp plugin/config contract, preserve existing behavior unless configured, merge the option with quoted-message send options, and cover normal text, media, voice-note trailing text, polls, tool/channel sends, and quoted reply paths. Use the closed PRs as context but produce a fresh current-main implementation. Add docs and a changelog entry for the user-facing config; do not move WhatsApp-specific policy into core abstractions. If implementing the optional Baileys credential-default fallback would require uncertain dependency behavior, leave it out or gate it behind clearly proven source support.
