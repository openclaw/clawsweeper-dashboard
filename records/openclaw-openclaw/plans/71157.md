## Plan

Implement support for WhatsApp outbound disappearing-message expiration for https://github.com/openclaw/openclaw/issues/71157. Keep the fix inside the WhatsApp plugin/config contract; do not add a core-wide channel abstraction. Add `channels.whatsapp.disappearingMessagesSeconds` and `channels.whatsapp.accounts.<accountId>.disappearingMessagesSeconds` as optional config, with account-level override, positive whole-second values forwarded to Baileys as `MiscMessageGenerationOptions.ephemeralExpiration`, and `0`/`null`/omitted/non-positive values preserving current behavior. Thread the resolved value through resolved account state and outbound send options so normal text, quoted replies, media, voice-note trailing text, and poll sends are covered where Baileys accepts generation options. Verify the Baileys dependency contract from source/types before using the option; only implement credential-default fallback if the auth-state source clearly supports it. Update schema/types, docs, focused tests, generated config docs if needed, and a single-line changelog entry. Use https://github.com/openclaw/openclaw/pull/73080 and https://github.com/openclaw/openclaw/pull/73083 as reference only; do not merge stale branch code blindly.
