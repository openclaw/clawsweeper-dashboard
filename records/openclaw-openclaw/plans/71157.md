## Plan

Implement WhatsApp outbound disappearing-message expiration for this issue. Add `channels.whatsapp.disappearingMessagesSeconds?: number` and `channels.whatsapp.accounts.<accountId>.disappearingMessagesSeconds?: number` to the WhatsApp config contract/schema/docs; resolve channel-level default plus account-level override in the WhatsApp plugin account state; forward positive finite whole seconds to Baileys as `MiscMessageGenerationOptions.ephemeralExpiration` across normal text, quoted replies, media sends, trailing voice-note text, and poll sends where the current send API supports generation options. Preserve current behavior when the value is omitted, null, zero, negative, or non-finite. Use the closed unmerged PRs as implementation context, but verify the current Baileys contract directly before adding any credential-default fallback. Keep the change plugin-local; do not add a new core channel abstraction. Add focused regression coverage for schema acceptance/rejection, account override precedence, unset behavior, quoted option merging, media/voice-note text, and poll coverage if options are plumbed there. Add docs and a maintainer-owned changelog entry because this is a user-visible config feature.
