# Coding Plan for openclaw/openclaw#71157: [Feature]: Support WhatsApp disappearing-message expiration for outbound replies

Source report: [records/openclaw-openclaw/items/71157.md](records/openclaw-openclaw/items/71157.md)

## Summary

Keep open: current main still lacks the requested WhatsApp config surface and Baileys `ephemeralExpiration` plumbing, and the only matching implementation PRs are closed and unmerged.

## Plan

Implement WhatsApp disappearing-message expiration for outbound replies. Add `channels.whatsapp.disappearingMessagesSeconds?: number` and `channels.whatsapp.accounts.<accountId>.disappearingMessagesSeconds?: number` to the typed config/schema/docs surfaces, with account-level override over channel-level config. Resolve a positive finite integer expiration on the effective WhatsApp account; treat 0, null, omitted, negative, and non-finite values as no OpenClaw override, and floor fractional values if accepted. Pass the effective value as Baileys `MiscMessageGenerationOptions.ephemeralExpiration` across outbound WhatsApp sends, including normal text, quoted replies, media/voice-note trailing text, polls, and auto-reply/tool/channel paths. Merge expiration with existing quoted options rather than replacing them. Keep the change plugin/local-config scoped; do not add a generic core channel policy or package-manager/dependency changes. Use https://github.com/openclaw/openclaw/pull/73080 and https://github.com/openclaw/openclaw/pull/73083 as source context, but re-review coverage for auto-reply monitor paths. Update docs and add a user-facing changelog entry.
