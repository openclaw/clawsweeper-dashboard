## Summary

Keep open. Current main still lacks the requested privacy-safe, per-attempt `cron.delivery.sent` record; the newer cron diagnostic helpers add cron fields for stuck-session logs only, not outbound send success/error logs with returned platform message IDs.

## Plan

Implement https://github.com/openclaw/openclaw/issues/69055 as a focused observability PR. Add a cron-owned, privacy-safe structured `cron.delivery.sent` record for each outbound cron delivery attempt from `dispatchCronDelivery`/durable send results: include cronJobId, cronRunId/sessionId, agentId, accountId, channel, to, status, returned platform message IDs, and a bounded error string on failure. Do not log payload text, prompt text, bot tokens, PHI, or hook content. Prefer threading explicit telemetry/callback data through `sendDurableMessageBatch` or wrapping the cron direct send path rather than using content-bearing `message_sent` hooks. Cover success, suppressed/no identity, best-effort partial failure, and thrown send failure behavior with focused tests; add a changelog entry if maintainers treat this as user-visible observability.
