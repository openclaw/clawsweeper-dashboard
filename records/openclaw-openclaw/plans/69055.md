## Summary

Keep open. Current main still lacks the requested privacy-safe `cron.delivery.sent` record; cron delivery code has the needed cron/job/session context and durable send results, but no cron-owned per-attempt success/error log with returned platform message IDs.

## Plan

Implement https://github.com/openclaw/openclaw/issues/69055. Add a privacy-safe structured `cron.delivery.sent` log emitted exactly once for each outbound cron delivery attempt, success or error. Cover `src/cron/isolated-agent/delivery-dispatch.ts` and the cron announce/failure notification boundary in `src/cron/delivery.ts` if needed for every outbound cron delivery. Include cron job id, cron run id when available, agent id, account id, channel, target, platform message id fields or a clearly mapped message id, status, and sanitized error. Do not log message body, prompt text, tokens, credentials, or PHI. Prefer a cron-owned wrapper/telemetry parameter around `sendDurableMessageBatch` and its `DurableMessageBatchSendResult.results` / `OutboundDeliveryResult.messageId`; do not reuse content-bearing `message_sent` hook payloads for the log record. Add focused tests for success, failed send, partial/bestEffort behavior, and no-content/suppressed outcomes where applicable. Add a changelog entry because this is user-visible observability.
