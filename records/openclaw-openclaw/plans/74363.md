# Coding Plan for openclaw/openclaw#74363: Subagent runs can be falsely marked failed/lost after clean gateway close or pending wait

Source report: [records/openclaw-openclaw/items/74363.md](records/openclaw-openclaw/items/74363.md)

## Summary

Keep open. Current main still has the reported source-level gaps: clean gateway close 1000 is not classified as recoverable for subagent waits, pending waits return without rearming supervision, and a stale active run with a still-running backing session can still be finalized as lost context.

## Plan

Fix the subagent lifecycle false-failure bug tracked by https://github.com/openclaw/openclaw/issues/74363. Keep the patch focused: in src/agents/run-wait.ts, classify `gateway closed (1000...)` agent.wait errors as recoverable without making `gateway timeout` recoverable; in src/agents/subagent-registry-run-manager.ts, when waitForSubagentCompletion receives `wait.status === "pending"`, keep supervision alive by scheduling a guarded follow-up wait for the same current, unended entry; in src/agents/subagent-registry.ts, make stale active sweep reattach or re-wait when the backing session entry still has `status: "running"` instead of completing as `subagent run lost active execution context`. Preserve fail-closed behavior for missing session entries, missing session ids, terminal session entries, abortedLastRun recovery, stale/drifted entries, and truly orphaned runs. Add focused regression tests for close 1000 recovery, pending wait rearming, running-session reattach, and the fail-closed cases. Check adjacent open PRs https://github.com/openclaw/openclaw/pull/74131 and https://github.com/openclaw/openclaw/pull/75462 before editing to avoid duplicating their scope.
