## Summary

Keep open: current main still sends Slack native assistant-thread status only at reply start and clear. Tool-start events update status reactions and draft-preview progress, but they do not update `assistant.threads.setStatus`; the prior implementation PR was closed unmerged, while related open items cover customization and broader Slack assistant lifecycle work.

## Plan

Implement Slack native assistant-thread tool status updates for https://github.com/openclaw/openclaw/issues/33413. Use the existing `onToolStart` payload in `extensions/slack/src/monitor/message-handler/dispatch.ts` and the existing `ctx.setSlackThreadStatus` wrapper to set concise human-readable labels while a Slack thread status is active. Preserve status reactions, draft-preview tool progress, typing start/stop cleanup, error cleanup, and suppressed-typing behavior; dedupe repeated status labels so long runs do not spam Slack. Do not add custom `statusMessages`, `loading_messages`, or broad Slack Agents lifecycle support; those belong to https://github.com/openclaw/openclaw/issues/67129 and https://github.com/openclaw/openclaw/issues/80787. Add focused Slack tests that cover initial `is typing...`, a tool status update, duplicate-label suppression if implemented, and final clear. Update Slack docs and CHANGELOG if the behavior is user-visible.
