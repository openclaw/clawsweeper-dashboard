# Coding Plan for openclaw/openclaw#39588: [Bug]: macOS app 'Launch at login' stops sticking / appears to turn itself off after app updates

Source report: [records/openclaw-openclaw/items/39588.md](records/openclaw-openclaw/items/39588.md)

## Summary

Keep open. Current main and the latest shipped release still bind the macOS Launch at login toggle to live LaunchAgent health rather than durable user intent, so a failed or stale launchd registration can still make the setting appear off after an update.

## Plan

Fix the macOS app Launch at login persistence bug reported in https://github.com/openclaw/openclaw/issues/39588. Persist the user's Launch at login intent separately from LaunchAgent health, initialize the UI from that durable intent, and when intent is enabled reconcile/re-register ai.openclaw.mac if the plist or launchctl registration is missing or stale after app update. Preserve the current manual-quit behavior: do not reintroduce KeepAlive for the app LaunchAgent. Likely files are apps/macos/Sources/OpenClaw/AppState.swift, apps/macos/Sources/OpenClaw/LaunchAgentManager.swift, apps/macos/Sources/OpenClaw/Constants.swift, apps/macos/Sources/OpenClaw/GeneralSettings.swift or OnboardingView+Pages.swift only as needed, and focused tests under apps/macos/Tests/OpenClawIPCTests. Add a user-facing Fixes changelog entry if this becomes the landing PR. Do not change gateway/node LaunchAgent ownership or CLI doctor behavior; related gateway issues are separate.
