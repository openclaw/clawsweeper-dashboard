# Coding Plan for openclaw/openclaw#39588: [Bug]: macOS app 'Launch at login' stops sticking / appears to turn itself off after app updates

Source report: [records/openclaw-openclaw/items/39588.md](records/openclaw-openclaw/items/39588.md)

## Summary

Keep open. Current main and the latest release still derive the visible macOS Launch at login toggle from live launchd status instead of durable user intent, so the reported post-update toggle drift is still plausible from source inspection.

## Plan

Fix https://github.com/openclaw/openclaw/issues/39588 by making the macOS app's Launch at login toggle represent durable user intent rather than only live launchd health. Persist the user's intent in app defaults, initialize the visible toggle from that intent, and when intent is enabled reconcile/rewrite/bootstrap the `ai.openclaw.mac` LaunchAgent for the current `Bundle.main.bundlePath` if the plist is missing, stale, or unloaded. Do not reintroduce `KeepAlive` for the desktop app LaunchAgent, and do not change the separate gateway LaunchAgent ownership work tracked by https://github.com/openclaw/openclaw/issues/67631. Add focused Swift tests around intent persistence and plist/status reconciliation where practical, update docs only if the user-facing behavior text changes, and include a changelog entry if this becomes a user-facing fix.
