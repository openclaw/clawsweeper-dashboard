## Summary

Keep open. Current main and v2026.5.19 still initialize the UI-bound `launchAtLogin` state from live LaunchAgent status, with no durable user-intent key, so launchd/update drift can still make the setting appear off after the user enabled it.

## Plan

Fix the macOS app Launch at login persistence drift reported in https://github.com/openclaw/openclaw/issues/39588. Keep the boundary in the desktop app: persist the user's Launch at login intent separately from live launchd health, initialize the settings/onboarding UI from durable intent, and when intent is enabled reconcile or re-register `ai.openclaw.mac` if the plist/job is missing, unloaded, or points at the wrong bundle path.
