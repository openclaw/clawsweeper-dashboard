## Summary

Keep open. Current main still derives the macOS Launch at login toggle from live launchd status, so an unloaded or unhealthy `ai.openclaw.mac` LaunchAgent can make the setting appear disabled and can trigger the disable path again.

## Plan

Fix https://github.com/openclaw/openclaw/issues/39588 by separating the macOS app Launch at login user intent from live launchd health. Add a durable internal intent state for the existing setting, initialize the UI from that intent, and reconcile or repair the `ai.openclaw.mac` LaunchAgent when intent is enabled but the plist/job is missing, unloaded, or stale. Keep the fix scoped to the macOS app LaunchAgent path; do not change gateway LaunchAgent ownership, do not reintroduce `KeepAlive` for the app LaunchAgent, and preserve manual quit behavior. Add focused Swift regression coverage for intent persistence/reconciliation and update `CHANGELOG.md` because this is user-facing. Validate with the macOS Swift tests below and use Testbox for broader changed checks if Swift runtime behavior changes.
