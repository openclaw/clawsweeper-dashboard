# Coding Plan for openclaw/openclaw#39588: [Bug]: macOS app 'Launch at login' stops sticking / appears to turn itself off after app updates

Source report: [records/openclaw-openclaw/items/39588.md](records/openclaw-openclaw/items/39588.md)

## Summary

Keep open. Current main and the latest release still derive the macOS Launch at login toggle from live LaunchAgent health rather than durable user intent, so a stale or unloaded `ai.openclaw.mac` job can still make the setting appear off after an update.

## Plan

Fix https://github.com/openclaw/openclaw/issues/39588 in the macOS app only. Persist the user's Launch at login intent separately from `LaunchAgentManager.status()`, load that intent into `AppState`, and when intent is enabled reconcile/rewrite/bootstrap `ai.openclaw.mac` with the current `Bundle.main.bundlePath` if launchd health is missing or stale. Preserve the no-`KeepAlive` manual-quit behavior from https://github.com/openclaw/openclaw/pull/40213. Do not change gateway LaunchAgent recovery or CLI service behavior. Add focused Swift tests for intent persistence, status reconciliation, and generated plist shape; include a changelog entry if maintainers treat the fix as user-facing.
