# Coding Plan for openclaw/openclaw#69536: [Bug]: memory file naming format inconsistency between session-memory hook and dreaming system

Source report: [records/openclaw-openclaw/items/69536.md](records/openclaw-openclaw/items/69536.md)

## Summary

Keep open. Current main partially loads date-prefixed session-memory artifacts for first-turn startup context, but memory-core rem-backfill, live daily ingestion, historical seeding, REM harness, and Gateway Dream Diary backfill still use exact `YYYY-MM-DD.md` matchers, so the reported slugged-file visibility bug remains.

## Plan

Fix the memory filename mismatch reported at https://github.com/openclaw/openclaw/issues/69536. Add a shared parser/helper for daily memory artifact names that accepts `YYYY-MM-DD.md` and bounded same-directory `YYYY-MM-DD-*.md` session-memory artifacts, extracts the day deterministically, and excludes unrelated Markdown such as `DREAMS.md`, `memory/dreaming/**`, and arbitrary non-date files. Reuse the helper in `extensions/memory-core/src/cli.runtime.ts`, `extensions/memory-core/src/dreaming-phases.ts`, `extensions/memory-core/src/rem-harness.ts`, and `src/gateway/server-methods/doctor.ts`; keep the existing startup-context slugged-file behavior compatible. Add focused regression coverage for rem-backfill/rem-harness path input, live daily ingestion or historical seeding, Gateway Dream Diary backfill, and session-memory writer compatibility. Update user-facing docs and CHANGELOG.md if behavior is user-visible. Do not add a new config option and do not fold in the separate configurable inline daily-note location request at https://github.com/openclaw/openclaw/issues/68501.
