## Summary

Keep open. Current main still has the reported mismatch: the session-memory hook writes dated files with a slug, while memory-core historical discovery, live daily ingestion, grounded seeding, REM harness scanning, and Gateway Dream Diary backfill still accept only exact `YYYY-MM-DD.md` daily files. The related open memory issue is about configurable daily-note locations, not this slugged-file ingestion bug.

## Plan

Fix https://github.com/openclaw/openclaw/issues/69536 by making memory-core daily-memory discovery accept both canonical `YYYY-MM-DD.md` files and session-memory hook artifacts named `YYYY-MM-DD-*.md`. Prefer a shared parser/helper for dated memory filenames and reuse it in `extensions/memory-core/src/cli.runtime.ts`, `extensions/memory-core/src/dreaming-phases.ts`, `extensions/memory-core/src/rem-harness.ts`, and `src/gateway/server-methods/doctor.ts`. Keep the accepted set limited to dated daily/session-memory artifacts; do not ingest arbitrary `memory/*.md`, `DREAMS.md`, or `memory/dreaming/**` reports. Preserve deterministic ordering, correct day attribution, and existing canonical `YYYY-MM-DD.md` behavior. Add regression coverage for rem-backfill/rem-harness, live daily ingestion or historical seeding, Gateway Dream Diary backfill, and session-memory hook filename compatibility. Update memory/dreaming/hook docs and add a changelog fix entry if behavior changes are user-visible. Do not implement the separate configurable daily-memory path request tracked at https://github.com/openclaw/openclaw/issues/68501.
