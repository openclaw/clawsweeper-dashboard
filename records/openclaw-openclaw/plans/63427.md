## Summary

Keep open. Current main and the latest stable tag still create a fresh `GatewayClient` for each `probeGateway()` call with no per-URL `device-required` suppression; the exact cache fix only exists in closed, unmerged https://github.com/openclaw/openclaw/pull/63446, while open https://github.com/openclaw/openclaw/pull/68280 covers adjacent local probe auth fail-fast behavior rather than repeated rejection dampening.

## Plan

Fix this issue by adding a narrow `probeGateway()`-boundary dampener for repeated `DEVICE_IDENTITY_REQUIRED` WebSocket closes. Implement a per-gateway-URL module-level cache in `src/gateway/probe.ts` with a small threshold and TTL, short-circuit repeated device-required probe attempts before constructing `GatewayClient`, clear the cache on successful connection, and return the same rejected probe shape with a clear hint. Add focused coverage in `src/gateway/probe.test.ts` for threshold behavior, no premature short-circuit, per-URL independence, non-device-required closes not populating the cache, and success clearing the cache. Use https://github.com/openclaw/openclaw/pull/63446 as source work but avoid its reviewed threshold bug: recovery after pairing must not be TTL-only. Do not add an unauthenticated HTTP `/health` endpoint, do not weaken gateway device-auth policy, and do not broaden status/restart auth behavior beyond this probe-boundary cache. Add a single-line `CHANGELOG.md` fix entry because this is user-visible CLI/gateway behavior.
