## Summary

Keep open. Current main still treats native subagent waits as a parent turn ending plus a background `agent.wait`, and neither the Control UI nor TUI has a requester-scoped waiting/progress state to show while the child run is active.

## Plan

Fix https://github.com/openclaw/openclaw/issues/58067 by adding a minimal requester-scoped progress/wait state for native subagent waits after `sessions_yield`. Scope the change to signalling that the parent session is waiting on active child work and rendering that state in Control UI and TUI; do not implement full native `streamTo="parent"`, raw child token forwarding, tool-output forwarding, or a new config/API policy. The signal should start when a parent run yields with active native child work and clear on child completion, error, kill, pause/replacement, or parent resume. Add focused regression coverage for sessions_yield/subagent registry state and the Control UI/TUI rendering/clearing behavior. Changelog is expected because the fix is user-visible.
