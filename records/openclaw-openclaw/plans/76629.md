# Coding Plan for openclaw/openclaw#76629: [Bug] replay should prune internal/no-op control messages before provider context

Source report: [records/openclaw-openclaw/items/76629.md](records/openclaw-openclaw/items/76629.md)

## Summary

Keep open. Current main has partial replay cleanup, but it still lacks the broader transient replay filter for standalone silent assistant payloads, copied runtime-context assistant text, and pending-final-delivery recovery text before those paths can affect model or channel context.

## Plan

Fix https://github.com/openclaw/openclaw/issues/76629 as a narrow replay/recovery hygiene bug. Add or reuse a core helper near `src/agents/pi-embedded-runner/replay-history.ts` / `src/agents/internal-runtime-context.ts` that removes only OpenClaw-owned nonsemantic replay artifacts before provider conversion: historical `openclaw.runtime-context` custom messages, standalone assistant silent-token payloads (`NO_REPLY` / `no_reply` when the whole visible assistant text is the silent token), synthetic stream-error placeholders where provenance shows they are generated/empty-error turns, and assistant text whose visible content is only OpenClaw runtime-context/event scaffolding. Also sanitize `pendingFinalDeliveryText` before it is interpolated by `buildResumeMessage()` and before the heartbeat pending-final-delivery replay return in `src/auto-reply/reply/get-reply.ts`, using the existing `stripInternalRuntimeContext`, `stripInboundMetadata`, and `stripEnvelopeFromMessage` contracts where appropriate. Preserve real user-authored text, mixed assistant content, media payload behavior, current-turn runtime context needed by the active model turn, and provider-specific tool/reasoning repairs. Do not implement the broad declarative registry from https://github.com/openclaw/openclaw/issues/69541 or add a config flag unless maintainers explicitly ask. Add focused tests for replay history, LLM-boundary normalization, restart recovery, and heartbeat pending-final-delivery replay; add a changelog entry if the repair is prepared for landing.
