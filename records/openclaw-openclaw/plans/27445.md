# Coding Plan for openclaw/openclaw#27445: [Feature]: `announceTarget` option for sub-agent completion announce routing

Source report: [records/openclaw-openclaw/items/27445.md](records/openclaw-openclaw/items/27445.md)

## Summary

Keep open. Current main still lacks a native `announceTarget` parameter or `agents.defaults.subagents` default, and completion routing still depends on existing depth/thread delivery behavior rather than an explicit parent-routing option.

## Plan

Implement https://github.com/openclaw/openclaw/issues/27445 as a focused, backward-compatible native subagent announce-routing feature. Add `announceTarget: "channel" | "parent"` to native `sessions_spawn` and `agents.defaults.subagents`; default to `channel`; when set to `parent`, inject the completion payload into the requester/parent session with `deliver: false`, trigger exactly one parent turn, and suppress direct final channel announce. Preserve current behavior for omitted/`channel`, keep ACP `streamTo` semantics separate, and do not add a broader workflow engine. Cover thread-bound `sessions_spawn(thread:true, mode:"session", announceTarget:"parent")` so the child active run can stay in its bound thread while completion wakes the parent only. Update schema/types, generated config docs/help, user docs, tests, and changelog for the user-facing feature.
