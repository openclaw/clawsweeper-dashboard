# Coding Plan for openclaw/openclaw#71865: Auth login blocked by size-drop guard when openclaw.json was created by PowerShell (verbose/BOM format)

Source report: [records/openclaw-openclaw/items/71865.md](records/openclaw-openclaw/items/71865.md)

## Summary

Keep open. Current main still sends models auth login through the shared config writer, which compares canonical two-space output bytes against the previous raw file bytes and can still reject a PowerShell/BOM-formatted config as a destructive size drop.

## Plan

Fix the false-positive config size-drop rejection for formatting-only shrink from PowerShell/BOM or otherwise verbose `openclaw.json` files. Keep the fix in the shared config writer rather than adding an auth-only bypass. Preserve fail-closed behavior for real destructive writes, especially `gateway-mode-removed`, and do not implement the separate explicit large-delete feature requests. Add focused regression coverage showing a semantically equivalent but verbose/BOM existing config can be rewritten canonically during a normal config mutation without `CONFIG_WRITE_REJECTED`, plus models-auth or models-shared coverage that the auth login persistence path still uses the protected writer. Add a user-facing changelog entry for the bug fix.
