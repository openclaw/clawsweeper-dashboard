## Plan

Current main invokes `runMemoryFlushIfNeeded()` only before the reply turn; embedded Pi in-turn auto-compaction is counted after execution in `agent-runner-execution.ts`, `agent-runner.ts`, and `followup-runner.ts` without a memory-flush retry or `memoryFlushCompactionCount` update. Implement a shared, one-per-compaction-cycle fix so enabled memory flush cannot be bypassed by actual runtime compaction, covering both main replies and queued followups.
