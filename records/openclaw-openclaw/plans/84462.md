## Summary

Keep open. Source inspection shows current main can still drop a generic `/command@TargetBot` group message behind `requireMention` because Telegram mention detection only accepts standalone `@username` text or `mention` entities, while the existing slash-command bypass only covers known authorized OpenClaw control commands.

## Plan

Fix https://github.com/openclaw/openclaw/issues/84462 with a narrow Telegram mention-gating change. Treat Telegram `bot_command` text of the form `/command@<current bot username>` as an explicit mention for the current bot before applying `requireMention`, including generic/non-native commands such as `/ping@TargetBot`. Preserve existing behavior for `/command@OtherBot`, native OpenClaw command authorization, reply-to-bot implicit mentions, and configured mentionPatterns. Prefer a small helper in `extensions/telegram/src/bot/body-helpers.ts` or the adjacent inbound-body path, with focused tests in Telegram body/message-context tests and command-control tests as needed. Do not implement the broader bot-to-bot sender policy, peer mapping, loop protection, or history-target metadata tracked by related issues/PRs. Changelog: user-visible Telegram fix; maintainers can add the release entry without forcing contributor credit.
