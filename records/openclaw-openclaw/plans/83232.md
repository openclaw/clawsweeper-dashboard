# Coding Plan for openclaw/openclaw#83232: [Bug]: install.sh unconditionally runs install_homebrew on macOS even when adequate Node is already present

Source report: [records/openclaw-openclaw/items/83232.md](records/openclaw-openclaw/items/83232.md)

## Summary

Keep open: current main and the latest release still run macOS Homebrew setup before Node detection, so the installer can fail for a no-admin/no-brew user before it checks an already-supported Node on PATH.

## Plan

Fix https://github.com/openclaw/openclaw/issues/83232. In `scripts/install.sh`, do not call `install_homebrew` before Node detection; on macOS call it only immediately before brew-dependent installs, including `install_node` when `check_node` fails and `install_git` when Git is missing. Preserve Linux behavior, nvm loading before `check_node`, and the package/runtime Node >=22.16 contract; do not add config flags or broaden runtime support. Add focused regression coverage in `test/scripts/install-sh.test.ts` proving a macOS no-brew/non-admin fixture with supported Node on PATH does not invoke Homebrew/admin failure, and proving missing or outdated Node still invokes Homebrew before the Node install. Update `docs/install/installer.md` if the flow text no longer matches, and add a changelog entry because installer user behavior changes.
