## Plan

Fix https://github.com/openclaw/openclaw/issues/82337. Current main can replay OpenClaw inbound metadata from historical user-role messages after `currentTurnContext` was prefixed into the submitted model prompt. Add a narrow replay-history sanitization step for historical user-role messages only, at or before `normalizeAssistantReplayContent` / `sanitizeSessionHistory` in `src/agents/pi-embedded-runner/replay-history.ts`. Reuse and extend `src/auto-reply/reply/strip-inbound-meta.ts` so it strips all generated inbound prefix components now emitted by `buildInboundUserContextPrefix`, including Delivery hints, fenced JSON metadata blocks, reply-chain/location blocks, and textual `chat_window` blocks, without stripping legitimate user-authored text. Cover both string user content and array text-block user content; preserve non-text blocks, assistant turns, tool results, current-turn context, transcriptPrompt behavior, and persisted transcript bytes.
