Keep open: the PR addresses a real WebChat transcript persistence gap with strong proof, but the current patch still adds a gateway durable user write in a helper used by SessionManager-backed agent runs, which can duplicate or mis-chain persisted user turns.

## Plan

Add focused regression coverage for a full `chat.send` embedded-agent path proving exactly one persisted user row and for the gateway-owned/no-agent path proving the user row remains persisted with parentId and hook behavior. Do not add a config option, change plugin APIs, remove SessionManager guard semantics, or broaden the PR beyond WebChat transcript persistence.
