## Summary

Keep open. Current main still lacks a caller-visible lane-busy or lane-wait-timeout result for blocked same-session `openclaw agent` dispatch; the queue records wait facts internally and the CLI has generic Gateway-timeout fallback, but lane, queueAhead, and waitedMs are not returned to the caller.

## Plan

Fix the blocked same-session `openclaw agent` CLI relay path so a queued request that waits too long surfaces an actionable lane-busy/lane-wait-timeout result instead of only generic Gateway timeout/fallback. Preserve same-session serialization and the existing accepted/final response flow. Add an additive contract from the command queue through Gateway agent dispatch to CLI text and JSON output carrying lane, queueAhead, waitedMs, and guidance. Do not add a new `--force-queue` or broader wait-policy flag unless maintainers explicitly request it. Add focused regression coverage in queue, Gateway agent, and CLI Gateway tests, and add a user-facing changelog entry if behavior changes.
