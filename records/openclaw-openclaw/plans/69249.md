## Summary

Keep open: current main and latest release still recover restart-aborted main sessions through a hidden `deliver: false` turn, while the Discord abort path exits without a same-channel recovery notice. Related recovery PRs are open but do not yet close this focused Discord-visible gap.

## Plan

Fix https://github.com/openclaw/openclaw/issues/69249. Implement a channel-agnostic, idempotent post-restart interruption notice for externally routable main sessions using persisted `deliveryContext` or last-route metadata, preserving hidden resume for internal/non-routable sessions.
