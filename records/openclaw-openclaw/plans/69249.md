## Summary

Keep open: current main has restart-aborted main-session recovery, but it still resumes with hidden delivery and the Discord abort path returns without posting a same-channel recovery notice. This remains a source-reproducible user-visible message-loss/session-state bug.

## Plan

Fix https://github.com/openclaw/openclaw/issues/69249. Current main marks restart-aborted main sessions and resumes them internally with `deliver: false`; Discord abort handling also returns without a same-channel recovery notice. Implement a generic, idempotent post-restart interruption notice for externally routable main sessions using persisted delivery context or last-route metadata, while preserving hidden resume for internal/non-routable sessions. Avoid a Discord-only core special case and avoid changing provider fallback, markerless startup scanner policy, or durable generation checkpoint semantics. Add focused regression coverage for marked restart-aborted sessions with routable Discord-style delivery metadata and for non-routable/internal sessions staying hidden. Add a user-facing changelog entry because this changes visible channel recovery behavior.
