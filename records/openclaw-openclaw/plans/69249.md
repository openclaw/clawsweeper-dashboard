## Summary

Keep open: current main still has the Discord-visible restart interruption gap. Startup can mark and internally resume restart-aborted main sessions, but the recovery path is still hidden delivery and the Discord abort branch still exits without a same-channel notice.

## Plan

Fix the gateway-restart interruption gap for externally delivered main sessions. On current main, startup marks stale-lock main sessions as `abortedLastRun`, then `recoverRestartAbortedMainSessions` resumes via `callGateway` with `deliver: false`; Discord processing also returns on abort without a same-channel notice. Implement a generic, idempotent recovery notice or safely delivered recovery turn for routable main sessions using persisted `deliveryContext`/last-route metadata. Preserve hidden recovery for internal or non-routable sessions, avoid a Discord-only core special case, avoid adding a config option, and add focused regression coverage. Include a user-facing changelog fix entry unless maintainers decide this remains internal-only.
