## Summary

Keep open: current main still has the Discord-visible restart interruption gap. The code persists enough Discord route metadata to notify the original channel, but the abort path returns silently and the main-session restart recovery path still resumes with hidden `deliver: false` instead of posting a same-channel recovery notice.

## Plan

Fix https://github.com/openclaw/openclaw/issues/69249 on current main. Implement a generic, deduplicated same-channel recovery notice for external channel-backed main sessions interrupted by gateway restart, using persisted `deliveryContext`/`lastChannel`/`lastTo` or inbound route metadata. Do not add a config option, do not implement a Discord-only branch, and do not broaden into Dashboard reconnect recovery or high-load Discord interaction AbortError handling. Preserve automatic hidden resume for safe local/internal sessions, and reconcile the design with https://github.com/openclaw/openclaw/pull/74687 so channel-backed sessions do not silently replay hidden turns without a user-visible recovery path. Add focused regression coverage around `abortedLastRun` external-route sessions and Discord route persistence; add a one-line `CHANGELOG.md` Fixes entry because the behavior is user-visible.
