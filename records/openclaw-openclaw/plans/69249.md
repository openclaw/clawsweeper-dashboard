## Summary

Keep open: current main and v2026.5.18 still recover restart-aborted main sessions through a hidden `deliver: false` turn, while the Discord abort path exits without a same-channel recovery notice. Related restart-recovery PRs are open and do not yet close this focused Discord-visible gap.

## Plan

Fix https://github.com/openclaw/openclaw/issues/69249. Current main resumes restart-aborted main sessions with `deliver: false`, so Discord-backed interrupted turns can recover internally without any visible same-channel recovery notice. Add a generic, idempotent post-restart interruption notice for externally routable main sessions using persisted `deliveryContext`/last-route metadata; keep hidden resume behavior for internal or non-routable sessions. Do not add a Discord-only policy workaround or a new config option. Coordinate with the open manual-recovery direction in https://github.com/openclaw/openclaw/pull/74687 so channel-backed sessions do not get unsafe duplicate replays. Add focused regression coverage for restart recovery with a Discord/channel-backed route, delivery false/current hidden path, and the Discord abort cleanup path. Add a CHANGELOG entry because the fix is user-visible.
