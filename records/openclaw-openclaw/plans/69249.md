# Coding Plan for openclaw/openclaw#69249: Gateway restart can silently abort an in-flight Discord turn, with no automatic recovery message to the user

Source report: [records/openclaw-openclaw/items/69249.md](records/openclaw-openclaw/items/69249.md)

## Summary

Keep open: current main still has the Discord-visible restart interruption gap. The Discord route is persisted, but the abort path returns silently and restart recovery resumes internally with `deliver: false`, so the original channel still gets no automatic recovery notice.

## Plan

Fix https://github.com/openclaw/openclaw/issues/69249 on current main. Add a generic, idempotent post-restart interruption notice for external channel-originated main sessions using persisted deliveryContext/lastRoute metadata, so a Discord-backed in-flight turn interrupted by gateway restart gets a visible same-channel recovery message instead of only hidden `deliver: false` internal recovery. Preserve automatic hidden resume for local/internal WebChat sessions, do not replay channel-backed turns blindly, and do not add Discord-only core policy when the generic channel route contract is sufficient. Reconcile or supersede the behavior proposed in https://github.com/openclaw/openclaw/pull/74687 before editing the same decision point. Add focused regression coverage for Discord-shaped channel-backed main sessions, internal WebChat/local sessions, non-resumable tails, idempotency/dedupe, and startup scheduling. Add a single-line changelog entry because the fix is user-visible.
