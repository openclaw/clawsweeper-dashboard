## Summary

Keep open: current main still has the Discord-visible restart interruption gap. Restart-aborted main sessions are resumed through a hidden `deliver: false` agent turn, and the Discord abort path still returns without posting a recovery notice to the originating channel.

## Plan

Fix https://github.com/openclaw/openclaw/issues/69249. Current main can silently drop a Discord-visible turn when gateway restart aborts an in-flight main session: the Discord route is persisted, the Discord abort path returns silently, and `recoverRestartAbortedMainSessions` resumes with `deliver: false`. Implement a generic, idempotent post-restart interruption notice for external deliverable channel-originated main sessions using persisted `deliveryContext` or last-route metadata. Reconcile behavior with https://github.com/openclaw/openclaw/pull/74687 and avoid conflicting with the dashboard-only restart transcript work in https://github.com/openclaw/openclaw/pull/41067.
