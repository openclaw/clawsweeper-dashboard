## Summary

Keep open: current main still recovers restart-aborted main sessions through a hidden `deliver: false` turn, and the Discord abort branch exits without posting a same-channel recovery notice. The related open PRs cover adjacent restart or dashboard behavior but do not yet ship this focused Discord-visible recovery path.

## Plan

Fix the source-reproducible Discord restart ghosting bug: when a restart-aborted externally routable main session is detected, OpenClaw should either safely resume with delivery or post one idempotent recovery/interruption notice to the original channel. Keep the implementation generic around persisted delivery context or last-route metadata; do not add a Discord-only workaround, do not add a new config option, and do not implement broad checkpointing. Coordinate with the policy in https://github.com/openclaw/openclaw/pull/74687 so channel-backed sessions are not silently hidden or incorrectly auto-replayed. Add focused regression coverage for routable external sessions, internal/non-routable sessions, hidden resume preservation, and Discord abort cleanup behavior. Add a `CHANGELOG.md` fix entry unless maintainers decide the landing flow will own it.
