## Plan

Keep the fix narrow: preserve/finalize the already visible Telegram text message and deliver late TTS audio without changing the text message order or timestamp. Do not implement the broader word-highlight or global text-then-audio feature from https://github.com/openclaw/openclaw/issues/80630. Likely files: src/auto-reply/reply/dispatch-from-config.ts, extensions/telegram/src/lane-delivery-text-deliverer.ts, extensions/telegram/src/bot-message-dispatch.ts, extensions/telegram/src/*delivery*.test.ts, and extensions/speech-core/src/tts.ts only if the payload contract must be split. Add regression coverage for final-mode auto-TTS with an existing Telegram preview and a late media/voice result.
