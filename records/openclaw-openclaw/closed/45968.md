## Summary

Current `main` already preserves the selected auth profile through compaction and resolves OAuth credentials from that profile, so the reported `No API key found for provider "openai-codex"` path is implemented on `main` even though it does not appear to be in the latest tagged release yet.

## Evidence

- **Run path now carries auth profile into compaction:** Both timeout-recovery and overflow compaction build runtime context with `authProfileId: lastProfileId`, so compaction receives the active session auth profile instead of dropping it.
