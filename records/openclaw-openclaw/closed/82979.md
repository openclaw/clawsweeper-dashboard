## Best Possible Solution

Keep the current main Codex auto-routing docs, and accept only a narrow model-override error-message improvement if it is split from the incorrect docs changes and proven with focused assertion coverage plus redacted real CLI output.

No high-confidence current-main reproduction of the claimed PI fallback is established. Current source and regression tests show official `openai/*` agent runs select Codex when the harness is available; the disallowed model-override path is source-visible but not proven after this patch.
