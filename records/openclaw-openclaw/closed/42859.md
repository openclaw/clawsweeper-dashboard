## Summary

Close this PR as superseded by #47216. Current main still has the Control UI sender-metadata prompt problem, but this PR's blanket removal of `SenderId`/`SenderName`/`SenderUsername` from all gateway `chat.send` contexts is too broad because sender identity is still used for command attribution and sender-bound ownership checks. #47216 tracks the narrower prompt-rendering fix that preserves those internal identity semantics.

## Best Possible Solution

Close this PR as superseded and focus review on #47216 or an equivalent narrow prompt-rendering fix. The best code direction is to preserve trusted gateway caller identity in `MsgContext` for command auth, run attribution, delivery context, and sender-bound ownership checks, while suppressing synthetic direct-webchat/TUI sender metadata only at the prompt-visible `buildInboundUserContextPrefix` layer with regression coverage.
