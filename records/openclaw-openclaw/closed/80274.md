## Summary

Keep this PR open: it is a useful fix path for stale managed external CLI OAuth credentials, but the latest head still lets no-external-discovery auth paths read external CLI credentials and persist tokens.

## What This Changes

The branch persists fresher managed external CLI OAuth credentials into `auth-profiles.json`, keeps bootstrap-only Codex CLI credentials runtime-only, and updates auth-store coverage plus related test helper typing.

## Best Possible Solution

Keep the persisted non-bootstrap CLI refresh behavior, but make no-external loaders pass `syncExternalCli: false` and add regression coverage for disabled external discovery.

## Reproduction Assessment

Yes. Source inspection plus the PR's terminal proof establish the stale `anthropic:claude-cli` resync path, and the remaining regression is source-reproducible through `ensureAuthProfileStoreWithoutExternalProfiles()` reaching the new sync path.

## Solution Assessment

No. Persisting managed non-bootstrap CLI refreshes is the right direction, but the implementation must first preserve callers that explicitly disabled external CLI discovery.
