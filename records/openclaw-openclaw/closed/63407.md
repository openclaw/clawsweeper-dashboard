## Summary

Keep this PR open. Current `main` still has the Dockerfile-side `OPENCLAW_INSTALL_BROWSER` implementation, but the Podman setup script does not forward that build arg and the Podman docs still do not mention it, so the PR's intended change remains useful and unsuperseded.

## Best Possible Solution

Keep the PR open for maintainer review and likely merge after current rebase/checks. The best code direction is the PR's focused change: add the guarded `OPENCLAW_INSTALL_BROWSER` passthrough in `scripts/podman/setup.sh` and document it in `docs/install/podman.md`, with a small shell-level or Podman build proof before landing.
