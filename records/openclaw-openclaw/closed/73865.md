## Summary

Keep open. Current main still has a credible pre-listen hang path: Gateway startup awaits `runtime.early` before binding HTTP, and `runtime.early` awaits gateway discovery services without a generic timeout. Recent Bonjour fixes reduce one likely failure mode, but they do not prove this reported pre-sidecar, no-`/healthz` startup hang is fixed.

## Best Possible Solution

Keep this open and make pre-listen Gateway startup robust at the generic discovery seam. The focused fix is to move local discovery off the blocking pre-bind path or wrap each `gatewayDiscoveryServices[].advertise()` call in a short bounded timeout that logs the plugin/service id and continues. Add a regression test with a never-resolving discovery service and ask the reporter to retry current main/latest image with `OPENCLAW_DISABLE_BONJOUR=1` plus startup trace to isolate the suspected path.
