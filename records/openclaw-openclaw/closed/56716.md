## Summary

Current `main` already includes LaunchAgent recovery for the exact unloaded-service case: `openclaw gateway restart` now re-bootstrapstraps an installed-but-unloaded LaunchAgent instead of requiring a fresh install, and the macOS app also auto-enables the LaunchAgent when returning to local mode.

## Evidence

- **launchd restart path re-bootstrapstraps unloaded service:** `restartLaunchAgent` clears disabled state, tries `launchctl kickstart`, and if launchd reports the job as not loaded it runs `bootstrap` and retries. This directly addresses the reported `launchctl unload` / fully booted-out case.
  - file: [src/daemon/launchd.ts:656](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/daemon/launchd.ts#L656)
  - command: `nl -ba src/daemon/launchd.ts | sed -n '640,690p'`
  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **CLI restart falls back to LaunchAgent repair instead of requiring install:** `runDaemonRestart` calls `recoverInstalledLaunchAgent({ result: "restarted" })` when the service is not loaded, and that helper repairs bootstrap state if the plist still exists.
  - file: [src/cli/daemon-cli/lifecycle.ts:203](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/cli/daemon-cli/lifecycle.ts#L203)

  - sha: [56fe2aab9c87](https://github.com/openclaw/openclaw/commit/56fe2aab9c87596515f2280c8060dc740ea64a88)
- **macOS app now auto-enables unloaded LaunchAgent in local mode:** When the app enters local mode it schedules `ensureLaunchAgentEnabledIfNeeded()`, which checks whether the LaunchAgent is loaded and runs the enable/install path when it is not.
