## Summary

The proposal is useful, but on current `main` it fits the plugin/ClawHub layer rather than OpenClaw core. Core now ships durable Task Flow/task-ledger primitives and a plugin runtime seam for long-running orchestration, while `VISION.md` explicitly pushes new skills and optional orchestration features out of core.
