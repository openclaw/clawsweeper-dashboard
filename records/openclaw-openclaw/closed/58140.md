## Summary

Current main and the shipped release `v2026.4.23` already contain the relevant mitigation: the published Windows installer script does not create the reported `system.bat`/Run-key persistence, and OpenClaw now pins `axios` plus blocks known malicious install-time packages during plugin/install scanning.

## Evidence

- **Published Windows installer only performs expected install steps:** The official `install.ps1` path is documented as the published Windows installer, and the script itself only ensures PowerShell/Node/Git, runs `npm install -g openclaw@...`, and updates user PATH. There is no code in the installer that writes `C:\ProgramData\system.bat` or creates a `CurrentVersion\Run` key.
