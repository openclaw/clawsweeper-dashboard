## What This Changes

The PR branch adds a cron expression validation helper, calls it from the `cron.add` and `cron.update` gateway handlers, and adds cron validation tests.

## Best Possible Solution

Keep the current service-boundary fix on main as the canonical solution; future gateway-level validation can be added only as UX polish while preserving the service-layer mutation guard.

## Reproduction Assessment

Yes. The linked bug has a concrete CLI reproduction, and current main has a focused service regression that exercises the invalid disabled-job edit and corrupt-job enable paths without relying on the gateway handler.

## Solution Assessment

Yes. The current main implementation is the narrower maintainable fix because it protects all cron update callers at the service boundary before stored state is replaced, while the PR branch only validates selected gateway requests.
