## Summary

Keep open: the proposed timeout flag is narrowly scoped and the supplied live output is sufficient, but the branch still leaves the canonical capability metadata and docs out of sync with the new CLI surface.

## What This Changes

The PR adds a Gateway-only `--timeout-ms` option to `openclaw infer model run` and forwards it to both the Gateway client wait budget and agent timeout seconds.

## Best Possible Solution

Keep the Gateway-only timeout flag, then sync `model.run` metadata, docs, and focused CLI coverage before merge.

## Reproduction Assessment

Not applicable: this PR adds a new Gateway-only CLI option rather than reporting broken existing behavior. Current main source shows Gateway model-run calls still use a fixed `120_000` ms client wait budget.

## Solution Assessment

Mostly yes: forwarding one finite positive millisecond budget to the existing Gateway client timeout and seconds-level agent timeout is a narrow maintainable design. The patch should also update canonical metadata and docs so inspect output and user guidance match the new flag.
