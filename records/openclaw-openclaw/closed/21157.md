## Close Comment

Thanks for the report. Codex review on current `main` (a958b6e72328d1d922a6d133b1a70e866faf4013) found that OpenClaw no longer matches the issue’s core premise: the agent loop resolves the selected session/provider model in `src/agents/pi-embedded-runner/run.ts:285-327`, and tool execution in the same loop is driven by that same runtime model in `src/agents/pi-embedded-runner/run/attempt.ts:572-573` and `src/agents/pi-embedded-runner/run/attempt.ts:1308-1345`, not a Claude-only hardcoded path. For custom routing, current main already exposes the plugin seam intended for this kind of logic: `before_model_resolve` is documented as the preferred provider/model override hook in `docs/concepts/agent-loop.md:91-99` and `docs/plugins/architecture.md:96-101`, and the runner applies it in `src/agents/pi-embedded-runner/run/setup.ts:58-95`. Given `VISION.md:52-60` (“core stays lean; optional capability should usually ship as plugins”), this is better scoped as a ClawHub/plugin router than a new core config surface, so I’m closing this core issue on that basis.
