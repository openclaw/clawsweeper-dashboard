## Summary

Close as configuration/support: the reported setup uses `anthropic` as an agent runtime, but current code and docs define Anthropic as a provider/plugin id, not a registered agent harness.

## What This Changes

The issue asks to fix Anthropic gateway use in 2026.5.7 after an enabled Anthropic setup fails with `Requested agent harness "anthropic" is not registered`.

## Best Possible Solution

Keep Anthropic as a provider/CLI-backend plugin, and have affected users remove `agentRuntime.id: "anthropic"` or use the documented model-scoped `claude-cli` runtime only when they intend Claude CLI execution.

## Reproduction Assessment

No for the claimed Anthropic plugin load failure. Source inspection shows `anthropic` is not an agent harness id, and the quoted error is the expected fail-closed result for an invalid explicit runtime.

## Solution Assessment

No, registering `anthropic` as an agent harness would blur the provider/runtime boundary. The maintainable path is to correct local config to the documented Anthropic API-key or `claude-cli` runtime form.
