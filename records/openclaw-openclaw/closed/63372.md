## Summary

Current main and the shipped v2026.4.26 release implement the requested `plugins.entries.memory-core.config.dreaming.model` override. The memory-core manifest accepts the field, the host resolver propagates it into dreaming execution config, Dream Diary narrative runs receive the resolved model for light/REM/deep paths, and tests cover override forwarding plus fallback to the session default when the configured model is unavailable.

## Best Possible Solution

Close this as implemented by the shipped `dreaming.model` Dream Diary subagent override in v2026.4.26. Keep the current trust-gated behavior, documented fallback semantics, and existing tests; broader dreaming batching, provider availability, or cost-control ideas should be tracked separately if needed.
