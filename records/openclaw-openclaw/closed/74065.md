## Summary

Keep this PR open. Current main still lacks the final-aware block-reply flush this PR adds: `stripBlockTags` can hold a trailing `<final>` prefix in `pendingTagFragment`, while the current `onBlockReply` text-end and chunked message-end drains do not pass `final: true`. The PR is still open and not merged; it is the implementation candidate, with one low-severity memory-review concern to resolve before merge.

## Best Possible Solution

Keep this PR open as the implementation candidate.
