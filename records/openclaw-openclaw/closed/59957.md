## Summary

Current main addresses the reported v2026.4.2 symptoms: package auto-update is opt-in and has an incident-recovery kill switch, and the read/tool-result output path now preserves bounded output instead of dropping fresh read results during overflow recovery.

## Best Possible Solution

Close this as implemented on main and keep the shipped update controls plus tool-result overflow handling. If a current release still produces an unknown tool-output error, the follow-up should be a fresh bug with the exact provider route, model id, update config, transcript/log excerpt, and read-tool output size.
