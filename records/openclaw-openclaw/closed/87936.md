## Summary

Keep open: current main and the latest release still contain source-visible lossy tool-display truncation in the shared agent/TUI display path. The model-context leakage claim is not live-reproduced here, but the display bug and possible user-facing summary leakage are actionable.

## What This Changes

The issue asks OpenClaw to stop TUI tool-call path and command display from replacing copy-sensitive text with lossy ellipses that can later be reused incorrectly.

## Best Possible Solution

Keep TUI/tool summaries bounded while making copy-sensitive paths and command tails lossless or tail-preserving, and add tests proving full tool arguments remain separate from display-only summaries.

## Reproduction Assessment

Partially: source inspection shows current-main TUI/tool-display details truncate long commands and strings with ellipses. I do not have a high-confidence live reproduction of the later model-context leakage claim.

## Solution Assessment

Yes: the maintainable fix belongs in the shared tool-display/TUI path, with regression coverage for long exec commands, long path-like detail values, and separation between display summaries and model-visible tool arguments.
