## Summary

Current main already fixes the reported dispatcher mismatch by routing SSRF-guarded requests through the bundled Undici runtime fetch whenever a dispatcher is attached and no real caller-provided fetch implementation is present; regression tests cover the ambient-global-fetch case.

## What This Changes

Request to stop `fetchWithSsrFGuard` from pairing bundled Undici dispatchers with Node's ambient `globalThis.fetch` when no `fetchImpl` is supplied.

## Best Possible Solution

Keep the current-main runtime-fetch path, and let the next release carry this fix rather than adding another fallback shape.

## Reproduction Assessment

No current-main failing reproduction remains: source inspection shows dispatcher-attached calls now use bundled Undici fetch, and regression coverage asserts ambient global fetch is not called in that case. I did not run a live Node 25 gateway scenario.

## Solution Assessment

Yes. The current implementation is narrower than the suggested fallback because it reuses the existing `fetchWithRuntimeDispatcher` path while preserving explicit caller fetch implementations and test mocks.
