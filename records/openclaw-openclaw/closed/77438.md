## Summary

Current main does not match the reported hook-runner failure: `before_prompt_build` is registered into the live typed-hook registry, `hasHooks()` reads that live array, prompt building invokes the hook when present, and the linked media/cache failure modes are already covered or resolved.

## What This Changes

The issue reports that `before_prompt_build` hooks registered by path-loaded plugins do not run during prompt construction.

## Best Possible Solution

Keep the current core hook implementation; if this still fails for `memory-bridge`, reopen with a minimal plugin fixture, exact OpenClaw version/SHA, Gateway restart path, and logs from the agent turn that should invoke the hook.

## Reproduction Assessment

No: the current-main source path and regression tests contradict the reported core failure, and the issue does not include a minimal fixture that reproduces against this SHA. I did not run a live Gateway because this review was read-only.

## Solution Assessment

Not applicable: there is no patch here, and the existing core design is the narrow maintainable solution: register hooks into the live registry, rebind the global runner on activation/cache hits, and use non-activating provider snapshots for secondary provider discovery.
