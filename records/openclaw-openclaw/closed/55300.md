## Summary

Current `main` already contains the TUI send/history reconciliation changes for this report, with regression tests and changelog evidence tied to PR #59800. The current code sends new editor text as a fresh optimistic message, defers local run binding until the first gateway event, and avoids the history reload path that previously wiped or replayed pending sends.

## Evidence
