## Summary

Current main already preserves WhatsApp group participant identity and sender name through the inbound pipeline. Group inbound messages store Baileys `participant` as `senderJid`, map it into agent-facing `SenderId`/`SenderName`, and forward the same metadata to hooks/plugins as `senderId`/`senderName`. The latest release `v2026.4.22` also contains this behavior.
