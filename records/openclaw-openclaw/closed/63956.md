## Summary

Keep open: current main and v2026.6.10 still render each live stream segment through its own assistant group, and the focused closing PR is open but unmerged.

## What This Changes

The issue reports that Control UI streaming replies appear as multiple assistant chat bubbles during an in-flight response and collapse into one completed message afterward.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/92063 if its grouped-stream UX is accepted, or replace it with an equivalent render-layer fix that keeps one in-flight assistant presentation for a turn.

## Reproduction Assessment

Yes at source level: current main emits multiple stream items for segmented live replies and renders each through a helper that creates a full assistant group. I did not run a live browser/model reproduction in this read-only review.

## Solution Assessment

Yes, with one maintainer UX decision: the clean fix boundary is the Control UI render layer, but maintainers should decide whether one assistant group with segment bubbles is sufficient or whether the live text must be merged into one literal bubble.
