## Summary

Keep open: current main still re-arms the opencode-go stalled-stream idle timer only on delta events, so post-delta block-boundary events can be forwarded without refreshing liveness, and the open candidate PR has not merged.

## What This Changes

The issue requests that opencode-go block-boundary SSE events refresh stalled-stream liveness so live provider streams are not aborted between token deltas.

## Best Possible Solution

Land a provider-wrapper fix that refreshes idle liveness for real post-delta boundary events while preserving the longer first-event window, with focused regression coverage and real wrapper proof.

## Reproduction Assessment

Yes, at source level. Current main forwards boundary events but only re-arms idle liveness for delta events, matching the supplied failing timing sequence; I did not run the test in this read-only review.

## Solution Assessment

Yes, the best direction is a narrow opencode-go provider-wrapper fix. The fix should count real post-delta block boundaries as liveness without letting pre-delta boundary events shorten the first-event timeout contract.
