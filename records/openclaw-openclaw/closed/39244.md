- **[P1] Add nsSandbox to the strict exec config schema:** `src/config/types.tools.ts:253-258`
  - body: This advertises `tools.exec.nsSandbox` as a supported config field and threads it through `pi-tools`, but `ToolExecBaseShape` in `src/config/zod-schema.agent-runtime.ts` still omits it while both global and per-agent exec schemas are strict. A normal config file using the documented key will fail validation instead of enabling bwrap, so the feature is not usable through the supported config path.
  - confidence: 0.93
- **[P2] Preserve network for unclassified safeBins:** `src/infra/exec-bwrap-sandbox.ts:304-313`
  - body: `processGatewayAllowlist` always supplies `commandBins`, so `shouldShareNetwork` now disables networking for any custom safeBin not in this hardcoded list. That makes legitimate opt-in safeBins such as custom profiled CLIs fail in bwrap mode even though they worked under the existing safeBins policy; keep network shared by default unless a command is explicitly known local-only or make network policy configurable and documented.
  - confidence: 0.88
- **[P1] Remove unrelated native build policy changes:** `package.json:435-439`
  - body: This exec sandbox PR adds `@discordjs/opus` and `@tloncorp/tlon-skill` to root `onlyBuiltDependencies`, but the repository docs explicitly keep `@discordjs/opus` disabled for default installs so unrelated tests do not compile a native addon. That supply-chain/install-policy change is unrelated to bwrap and should be dropped or handled in a dedicated owner-reviewed PR.
  - confidence: 0.9
- **[P2] Mount the exact selected shell path:** `src/agents/bash-tools.exec-host-gateway.ts:207-219`
  - body: `makeBwrapParams` passes only `path.basename(getShellConfig().shell)` into the mount builder, but `runExecProcess` later executes the full selected shell path inside bwrap. If `$SHELL` is a valid shell outside `/bin` or `/usr/bin` and not listed in `safeBinTrustedDirs`, bwrap cannot exec that path and every sandboxed safeBins command fails; pass and mount the actual resolved shell path or force a known mounted shell in bwrap mode.
  - confidence: 0.82
