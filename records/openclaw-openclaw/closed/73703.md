## Summary

Keep this PR open. Current main still does not provide the requested `openclaw/plugin-sdk/discord` package subpath, and the PR is tied to the still-open Discord regression #73685. It is not a cleanup close candidate; the maintainer decision is whether to accept this as an explicit compatibility shim or fix/release the external Discord plugin to use the existing generic SDK subpaths.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is either to revise it into an explicit compatibility shim for the published external Discord package, updating SDK contract tests and docs accordingly, or to fix and release the external Discord plugin so it imports the existing generic SDK subpaths instead of depending on `openclaw/plugin-sdk/discord`.
