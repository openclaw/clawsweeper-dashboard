## Summary

Current main already implements provider-specific thinking handling for the reported providers: MiniMax Anthropic-compatible streams force `thinking: { type: "disabled" }` unless explicitly overridden, and Z.AI exposes binary thinking with a default of `off`. The same behavior is present in the shipped `v2026.4.22` tree, so the original regression report is obsolete against current releases.

## Evidence
