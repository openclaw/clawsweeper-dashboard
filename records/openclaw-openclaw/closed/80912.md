## Summary

Current main already covers the reported runaway path: plain tool-shaped text is treated as text, malformed blank structured calls are rewritten before dispatch, unknown-tool repeats are capped, and real structured calls go through name lookup and schema validation.

## What This Changes

The issue asks OpenClaw to avoid executing accidental XML-like model output as real tool calls and to stop recursive tool-call amplification.

## Best Possible Solution

Keep the current main and beta behavior: plain tool-shaped model text remains text, malformed or unknown structured calls are stopped, and true tool calls continue through name and schema validation.

## Reproduction Assessment

No current-main failing reproduction is established. Source, tests, and dependency contracts show the reported text-shaped XML path does not execute as a tool call and malformed structured calls are guarded before they can amplify indefinitely.

## Solution Assessment

Yes. The current layered boundary is the narrow maintainable solution: providers may emit structured tool calls, but OpenClaw validates names and arguments at the agent boundary and treats plain tool-shaped assistant text as text.
