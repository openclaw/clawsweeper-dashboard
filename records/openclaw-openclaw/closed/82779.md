## Summary

Current main already coalesces same-session deferred context-engine turn-maintenance scheduling with focused regression coverage; the proof commit is on main but not in the latest tagged release.

## What This Changes

The issue asks OpenClaw to avoid queuing another deferred context-engine turn-maintenance job for a session while one for that session is already queued or running.

## Best Possible Solution

Keep the current per-session coalescing behavior and ship it in the next release so older 2026.4.x/2026.5.12 installs receive the fix.

## Reproduction Assessment

No current-main failure path was found: source inspection shows repeated same-session requests return the active deferred-maintenance promise. A focused regression test covers the reported pile-up shape with one queued task plus one follow-up run.

## Solution Assessment

Yes. The current per-session active-run map is the narrow maintainable fix because it prevents queue growth while still preserving one latest-params follow-up after the active maintenance run completes.
