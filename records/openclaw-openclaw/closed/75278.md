## Summary

Close as duplicate of #74047. Current main has partial updater and installer guardrails, but the remaining user/system split-install problem is already tracked by the older open canonical issue with broader gateway-service path evidence and related implementation work; this report adds useful `/usr/lib` plus NVM evidence without pointing to a distinct root cause.

## What This Changes

The issue asks OpenClaw upgrade/install flows to replace, remove, or clearly warn about stale global installs when system and NVM/user paths can resolve different OpenClaw versions.

## Best Possible Solution

Keep #74047 as the canonical tracker and solve the remaining behavior there: update/doctor/status should make split user/system installs visible, identify the active CLI and managed-service entrypoints, and provide a safe migration or cleanup path without deleting independently managed installs unexpectedly.
