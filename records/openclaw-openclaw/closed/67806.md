## Summary

Current `main` already implements the Control UI behavior this issue asked for: sent messages are appended optimistically, assistant finals are appended from live chat events, and `session.message` reloads are deferred during active runs to avoid the exact mid-run race that could hide messages until refresh. The same code is present in the shipped `v2026.4.23` release.
