## Summary

Keep open: current main still has a source-backed path where a stale live-switch flag plus canonical `openai/*` default selection can be treated as different from an active `openai-codex/*` Codex runtime turn, causing the active attempt to abort.

## What This Changes

The issue asks OpenClaw to stop falsely live-switching active Codex runtime sessions when the selected canonical OpenAI model and the active `openai-codex` runtime provider refer to the same model route.

## Best Possible Solution

Teach the live-switch resolver/comparison to preserve or equivalently compare the active runtime alias for canonical OpenAI Codex routes, then add regression coverage for the stale pending default-selection case.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction: a pending live switch with runtime fields cleared resolves to canonical `openai/gpt-5.5`, while the active run remains `openai-codex/gpt-5.5`, and the strict provider comparison returns a switch. I did not run a live gateway turn under the read-only constraints.

## Solution Assessment

Yes, the narrow maintainable fix is in live model-switch selection/comparison, using the existing runtime model-ref equivalence or preserving the active runtime provider for equivalent canonical OpenAI Codex routes. A broader config migration or new setting is not needed.

Fix the live model switch false positive reported in https://github.com/openclaw/openclaw/issues/87226. Current main can resolve a pending session selection to canonical `openai/gpt-5.5` after runtime fields are cleared while the active attempt is still `openai-codex/gpt-5.5`; `hasDifferentLiveSessionModelSelection` then compares provider strings directly and returns a switch. Keep the fix focused on the live-switch selection/comparison path: preserve the active runtime provider or use the existing runtime model-ref equivalence so `openai-codex/<same model>` and `openai/<same model>` do not trigger a switch, while real model changes, auth-profile changes, and non-equivalent provider changes still switch.
