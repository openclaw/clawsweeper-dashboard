## Summary

Keep open. Current main still lacks the requested cron-add missing-agent warning, and the PR remains the focused implementation path for the linked user problem. The related merged cron-tool work in #72326 is adjacent but does not supersede this CLI warning because it changes agent-tool session inference, while `openclaw cron add` still accepts omitted or blank `--agent` without warning.

## What This Changes

This PR adds a stderr warning when `openclaw cron add` creates an agent-turn job without a nonblank `--agent`, expands cron CLI tests for warning/stdout edge cases, and adds a changelog fix entry crediting the contributor.

## Best Possible Solution

Land this PR or an equivalent focused patch that warns after payload classification, writes the warning to stderr, warns for omitted or blank `--agent` on agent-turn jobs including session-key jobs, stays quiet for system events and explicit nonblank agents, preserves machine-readable stdout, and keeps the contributor changelog credit.
