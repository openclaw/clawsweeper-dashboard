## Summary

Close this PR as superseded rather than merge it. The QMD `--mask` default and the `memory-alt-main` default-collection failure path are already handled on current main, while the remaining stale/unverified QMD collection-name conflict is tracked by #71783 with active PR #72297; this branch's pruning approach also has blocking correctness regressions.

## What This Changes

The branch prunes QMD managed search collection names after `ensureCollections()`, switches QMD collection-add pattern preference from `--glob` to `--mask`, and updates memory-core QMD tests.

## Best Possible Solution

Keep current main's QMD compatibility and default-collection fixes, close this stale unsafe branch, and route the remaining stale collection-name work through #71783/#72297 with warning-only or verified metadata-gated repair behavior.

## Reproduction Assessment

Yes. The PR body gives a concrete QMD sidecar path where a configured collection fails to materialize and `memory_search` queries it, and the QMD manager test harness can model the same list/add/search behavior without live QMD.

## Solution Assessment

No. This PR's in-place pruning is not the best solution because it drops usable names when collection listing is unavailable and can retry a repaired search with an empty filter; the safer direction is the current main split plus #71783/#72297 for the remaining conflict case.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve names when QMD collection listing is unavailable:** `extensions/memory-core/src/memory/qmd-manager.ts:464-466`
  - body: The new `existing.has(name)` filter trusts `listCollectionsBestEffort()`, but that helper intentionally returns an empty map when older QMD builds cannot list collections. If `collection add` then reports an existing collection without metadata, this drops usable managed names and later searches skip configured QMD collections.
  - confidence: 0.9
- **[P1] Recheck scope after collection repair before retrying:** `extensions/memory-core/src/memory/qmd-manager.ts:468-473`
  - body: `search()` captures `collectionNames` before the first attempt, and this patch mutates that same managed names array during `ensureCollections()`. If the repair path prunes it to empty, the retry can run without `-c` constraints instead of failing closed.
  - confidence: 0.86
