## Summary

Current `main` already fixes this by staging inbound media into the resolved active sandbox workspace and rewriting prompt-visible paths to `media/inbound/*` before the agent reply runs.

## Evidence

  - file: [extensions/slack/src/monitor/media.ts:273](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/slack/src/monitor/media.ts#L273)

  - file: [extensions/slack/src/monitor/message-handler/prepare.ts:696](https://github.com/openclaw/openclaw/blob/86f8c826e20a01d1bba67c0ba6e73182fbf0455f/extensions/slack/src/monitor/message-handler/prepare.ts#L696)
