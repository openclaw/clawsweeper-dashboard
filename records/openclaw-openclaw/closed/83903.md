## Summary

Keep open: current main still has the one-shot CLI banner module guard with no reset path, while nearby cached CLI helpers already expose `testing`/`__testing` reset hooks used by tests. This is a narrow P3 testability cleanup, not a user-facing runtime bug.

## What This Changes

The issue requests a test-only reset helper for the module-level CLI banner emission guard so repeated tests can reset cached state.

## Best Possible Solution

Add a `testing`/`__testing.resetBannerEmittedForTests()` helper and a focused `src/cli/banner.test.ts` regression that stubs TTY/write, emits, resets, and emits again without changing production banner behavior.
