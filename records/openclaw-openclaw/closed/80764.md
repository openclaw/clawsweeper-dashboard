## What This Changes

The issue reports that Discord message read, search, and channel-info actions can return raw gzip-compressed response bytes instead of parsed JSON.

## Best Possible Solution

Review and land or explicitly close https://github.com/openclaw/openclaw/pull/80788, then let this issue close from the merged fix.

## Reproduction Assessment

Yes at source level, but not with a live Discord account in this read-only pass. If raw gzip bytes reach `RequestClient`, current main parses text with no gzip decode and can return a string into the read/search paths.

## Solution Assessment

Yes. The linked PR’s plugin-local REST decode plus read/search shape guards is the narrow maintainable fix, and the issue should remain open until that PR merges or maintainers choose another implementation.
