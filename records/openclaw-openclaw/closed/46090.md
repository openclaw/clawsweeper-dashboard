## Summary

Current `main` already contains targeted fixes for this `profile="user"` / Chrome MCP attach path: it now surfaces `DevToolsActivePort` failures as a specific existing-session connectivity error instead of a generic tab-wait timeout, and the Chrome MCP session layer now resets poisoned existing-session sessions on timeout/transport failure so retries reconnect cleanly. Those fixes are covered by regression tests and shipped in releases after the report.
