The issue reports that Telegram direct-chat image and sticker messages reach the agent only as `<media:image>` placeholders instead of usable inbound media file paths.

## Best Possible Solution

Keep the existing Telegram `MediaPaths` plus sandbox staging and media-note pipeline, while leaving the separate Telegram group-history and captionless group activation issues tracked in their own open reports.
