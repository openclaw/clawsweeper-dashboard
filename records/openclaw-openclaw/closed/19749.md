## Summary

Close as `clawhub`. Current main explicitly directs optional channel integrations to the plugin ecosystem rather than OpenClaw core, and the repo already documents channel plugins plus ClawHub/npm distribution for this exact use case.

## Close Comment

Closing this as a `clawhub` item after Codex review. Current `main` treats optional channel integrations as plugins rather than OpenClaw core work: `VISION.md:54` says optional capability should usually ship as plugins and that the bar for adding optional plugins to core is intentionally high, `docs/plugins/building-plugins.md:16` says new plugins do not need to be added to this repository and should be published to ClawHub or npm, and `docs/plugins/sdk-channel-plugins.md:11` already documents how to build a messaging-platform channel plugin. The user-facing plugin docs also show external channel plugins are a supported path today (`docs/tools/plugin.md:11`, `docs/tools/plugin.md:87`). For Session specifically, the right fit is a Session channel plugin distributed through ClawHub/npm, not a new core cleanup item in `openclaw/openclaw`. If someone runs into a concrete SDK gap that blocks a Session plugin, please open a focused issue for that gap.
