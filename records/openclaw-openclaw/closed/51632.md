## Summary

Current main already handles `cron.run` asynchronously: the gateway method enqueues the job and returns an immediate `{ ok: true, enqueued: true, runId }` acknowledgement instead of keeping the gateway request open. That behavior is documented, regression-tested, and recorded in the changelog as shipped in `v2026.3.7`.
