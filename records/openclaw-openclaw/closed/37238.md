## Summary

Keep open. Current `main` still populates the Control UI chat session dropdown from an unbounded `sessions.list` call, so completed/killed historical subagent sessions remain eligible for display unless a caller explicitly supplies an `activeMinutes` filter. There is now separate cleanup UI on the Sessions page, but that does not resolve the reported dropdown clutter/default behavior.

## Evidence

- **Chat session dropdown requests all sessions:** `refreshSessionOptions()` calls `loadSessions(..., { activeMinutes: 0, limit: 0, includeGlobal: true, includeUnknown: true })`, so the picker refresh path explicitly disables active/recent filtering.
  - file: [ui/src/ui/chat/session-controls.ts:74](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/ui/src/ui/chat/session-controls.ts#L74)
