## Summary

Current `main` no longer uses the old ACPX CLI argument-construction path that put `--format` before the agent subcommand. The reported buggy path existed in `v2026.4.2`, but the bundled `acpx` plugin was reworked to embed `acpx/runtime` directly and that change shipped in `v2026.4.5`.

## Evidence

- **Buggy path existed in the reported release:** In `v2026.4.2`, `buildPromptArgs()` and `buildVerbArgs()` prefixed ACPX commands with `--format json --json-strict --cwd ...` before the agent verb/subcommand, matching the issue report about `unknown option '--format'`.

  - sha: [d74a12264aa5](https://github.com/openclaw/openclaw/commit/d74a12264aa5fb0598605e8f04e1864b7239ddd5)
- **Reported release pinned the older ACPX version:** The `v2026.4.2` bundled plugin depended on `acpx` `0.4.0`, which matches the issue body's environment details.
  - file: [extensions/acpx/package.json:7](https://github.com/openclaw/openclaw/blob/d74a12264aa5fb0598605e8f04e1864b7239ddd5/extensions/acpx/package.json#L7)

  - file: [extensions/acpx/src/service.ts:43](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/extensions/acpx/src/service.ts#L43)
