## Summary

Keep open: current main still freezes the file-log hostname at module load, while this PR fixes the affected write path with focused regression coverage and no concrete review findings.

## What This Changes

The PR replaces file-log module-load hostname caching with lazy first-real hostname recovery, aligns `_meta.hostname`, and adds regression coverage.

## Best Possible Solution

Land this focused logger fix after required checks and maintainer review, then close the linked bug and retire the older duplicate at https://github.com/openclaw/openclaw/pull/87301 if this path is chosen.

## Reproduction Assessment

Yes for source-level reproduction: current main computes `HOSTNAME` once at module load and every file record reuses it, so an empty first `os.hostname()` remains `unknown`. I did not run a live macOS timing repro in this read-only review.

## Solution Assessment

Yes. Resolving in the file-log transport, caching only the first real hostname, and copying that value into `_meta.hostname` is narrower than import-order or startup bootstrap workarounds.
