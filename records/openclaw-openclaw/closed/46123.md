## Summary

Current main already normalizes inbound `chat.send` attachments before message parsing, and the gateway test suite now covers successful WebSocket `chat.send` requests with both nested `source` image attachments and direct base64 `content` attachments. The same attachment-normalization implementation is present in release `v2026.4.22`, so this issue appears fixed on shipped builds newer than the reported `2026.3.11` environment.

## Evidence

- **`chat.send` now normalizes RPC attachments before processing:** The `chat.send` handler calls `normalizeRpcAttachmentsToChatAttachments(p.attachments)` before validating empty message/attachment cases and before attachment parsing.
  - file: [src/gateway/server-methods/chat.ts:2113](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/gateway/server-methods/chat.ts#L2113)
