## Summary

Current `main` treats proactive compaction, cross-session recall, and memory persistence as plugin-owned capabilities. This PR adds an optional core archive/tool surface that the repo's current vision and context-engine architecture explicitly steer toward a context-engine or memory plugin, with ClawHub as the preferred distribution path for new skills.

## Risks / Open Questions

- There is real user interest in proactive compression and recall, but current main already provides plugin seams for shipping that outside core.
- A future maintainer could still choose to add a narrower core seam if plugin authors hit a concrete blocker not covered by the existing context-engine or memory APIs.
