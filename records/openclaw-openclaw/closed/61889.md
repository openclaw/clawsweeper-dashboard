## Summary

Current main already fixes the iMessage self-chat/from-me classification problem reported against 2026.4.2. The shipped implementation uses normalized `sender`, `chat_identifier`, and explicit `destination_caller_id` to distinguish true self-chat from normal/ambiguous DM outbound rows, drops normal `is_from_me=true` rows before dispatch, and has regression coverage for true self-chat dispatch, agent self-chat echoes, blank/absent destination rows, and reflected ambiguous echoes.

## Best Possible Solution

Keep the shipped 2026.4.10 iMessage behavior: explicit `destination_caller_id`-based self-chat classification, conservative `from me` drops for normal outbound rows, and reflected-echo suppression for ambiguous rows. If a new live imsg payload still drops a legitimate self-chat message, it should be filed as a fresh issue with sanitized raw fields for `sender`, `chat_identifier`, `destination_caller_id`, `is_from_me`, `created_at`, and `text`.
