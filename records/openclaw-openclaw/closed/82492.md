## Summary

Keep this PR open, but do not merge the current patch: it addresses a valid WhatsApp profile-isolation bug, yet the current diff breaks the exported auth-dir string contract, weakens unrelated cron validation, and still lacks required real profile-gateway proof.

## What This Changes

The PR defers WhatsApp auth-dir resolution for profile state, adds lazy-dir regression coverage, adjusts channel-add catalog lookup, and changes cron persisted-job validation/tests.

## Best Possible Solution

Land a focused WhatsApp-owned fix that keeps public auth-dir exports primitive or moves callers to a resolver, resolves credential paths from active profile env/state at runtime, drops unrelated cron churn, and includes focused tests plus redacted real profile-gateway or CLI proof.

## Reproduction Assessment

Yes, from source inspection: current main eagerly assigns WA_WEB_AUTH_DIR while resolveOAuthDir is env-aware, and the linked issue gives concrete profile-gateway steps and logs. I did not run a live WhatsApp/Baileys pairing, so this is source-reproducible rather than fully reproduced here.

## Solution Assessment

No. Lazy resolution is the right direction, but a string-like object is not a maintainable public string contract, and the cron persisted-shape change is unrelated and weakens validation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep WA_WEB_AUTH_DIR a real string:** `extensions/whatsapp/src/auth-store.ts:62`
  - body: WA_WEB_AUTH_DIR is exported through the WhatsApp runtime APIs and the web-channel loader types it as a string. This change returns a class instance cast as string; Node path APIs reject string-like objects before coercion, so consumers of the exported auth dir can throw on the credential-isolation path.
  - confidence: 0.93

  - body: The cron change removes the trim check that filters whitespace-only agentTurn.message values during persisted-store hydration. Those jobs can then reach runIsolatedAgentJob with an empty prompt instead of being skipped for repair, and this is unrelated to the WhatsApp fix.
  - confidence: 0.82
