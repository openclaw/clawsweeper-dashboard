## Summary

Close as implemented. The merged aggregate tool-result guard now bounds provider-visible prompt history and recovery truncation on current main, and the same implementation is present in the v2026.6.6 release.

## What This Changes

The issue reports that repeated long-running `toolResult` messages can accumulate in persisted session history until short follow-up turns exceed the model context window.

## Best Possible Solution

Keep the shipped aggregate prompt-history and recovery guard; reopen or file a new issue only if v2026.6.6 or later still shows context overflow from repeated tool results.

## Reproduction Assessment

Yes, source-level for the original released bug: repeated sub-cap `toolResult` messages could exceed the prompt budget before the aggregate guard. Current main includes the aggregate guard, so I did not establish a failing current-main path.

## Solution Assessment

Yes. The shipped fix bounds provider-visible prompt history and recovery truncation without rewriting canonical session state by default, which is narrower than the original persistence rewrite while addressing the user-visible context overflow.
