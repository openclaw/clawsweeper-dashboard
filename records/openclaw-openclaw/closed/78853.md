## Summary

Keep open: current main has opt-in `/verbose full`, internal `command_output` events, Control UI delivery, and compact channel progress previews, but it still does not define automatic default/full-result delivery across channels. The remaining work is a product/security-sensitive visibility policy, not a broken documented contract.

## What This Changes

The issue requests automatic user-visible delivery of tool results, or an equivalent status/result visibility policy, so users are not dependent on the agent to summarize tool output.

## Best Possible Solution

Define an explicit channel-safe tool-result visibility policy that reuses `/verbose full`, structured agent events, and progress drafts, defaults to compact sanitized progress, and exposes raw/full output only through deliberate session/channel policy.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/policy request, not a broken documented contract. Source and docs verify that current main provides opt-in `/verbose full` and compact progress previews, not default automatic full-result posting.

## Solution Assessment

No: blanket auto-push of every tool result is not the safest or most maintainable solution. A sanitized, opt-in visibility policy layered on the existing event and verbose/progress plumbing is the safer path.
