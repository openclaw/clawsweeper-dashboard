## Summary

Issue #58104 is a request to add a new bundled Feishu skill, not a packaging bug on current `main`. OpenClaw scope now keeps new skills on ClawHub by default, and the Feishu plugin currently bundles only the four skill folders present in `extensions/feishu/skills/`.

## Evidence

- **Scope policy excludes new core skills when ClawHub is sufficient:** `VISION.md` says bundled skills are only for baseline UX, that new skills should go to ClawHub first, and explicitly lists new core skills that can live on ClawHub under 'What We Will Not Merge (For Now)'.
  - file: [VISION.md:68](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/VISION.md#L68)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Feishu plugin bundles whatever is in its local `./skills` folder:** The Feishu plugin manifest points `skills` at `./skills`, so the bundled set is defined by directories committed under `extensions/feishu/skills/`, not by an omitted dist copy step.
  - file: [extensions/feishu/openclaw.plugin.json:12](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/extensions/feishu/openclaw.plugin.json#L12)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Current main only has four bundled Feishu skills:** Listing `extensions/feishu/skills/` on current `main` shows only `feishu-doc`, `feishu-drive`, `feishu-perm`, and `feishu-wiki`; there is no `feishu-sheet` directory to stage or publish from core.
  - command: `find extensions/feishu/skills -maxdepth 1 -mindepth 1 -type d | sort`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Feishu plugin docs/metadata describe a narrower built-in tool scope:** The package metadata advertises the Feishu channel as 'enterprise messaging with doc/wiki/drive tools' and separately marks the plugin for ClawHub/npm publishing, which fits shipping extra Feishu capability as external/plugin-distributed work instead of extending the core bundled set.
  - file: [extensions/feishu/package.json:33](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/extensions/feishu/package.json#L33)
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
