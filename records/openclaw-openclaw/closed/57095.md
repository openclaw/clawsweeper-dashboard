## Summary

Current `main` already avoids this CLI-only crash path, and the shipped `v2026.4.23` release contains both the fix behavior and an explicit regression test for Gemini multimodal memory without a registered runtime adapter.

## Evidence

- **Config resolution no longer fails when the CLI process has no registered Gemini adapter:** `resolveMemorySearchConfig` only throws the multimodal capability error when a provider adapter is actually present in the runtime registry. If the CLI process has not registered Gemini yet, `multimodalProvider` is `undefined` and the config resolves instead of crashing `openclaw status`.
