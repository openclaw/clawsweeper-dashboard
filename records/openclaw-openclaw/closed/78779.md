## Summary

Keep this PR open for maintainer review, but do not merge it yet: the updater change is plausible, while the supplied proof only exercises a small helper-style script and the current code already has an owning-npm lookup path for normal nvm installs.

## What This Changes

The PR changes npm updater root resolution to prefer the global root inferred from `pkgRoot` when it differs from `npm root -g`, adds a regression test, and adds a changelog entry.

## Best Possible Solution

Accept a narrow updater fix once real after-fix updater proof is added and maintainers are comfortable treating `pkgRoot` as the fallback source of truth when the owning npm executable cannot be found.

## Reproduction Assessment

Partially. Source inspection shows the failure path when a package root has an npm global-root shape, the inferred npm executable is unavailable, and bare `npm root -g` returns a different unwritable root; the PR does not provide live launchctl/OpenClaw updater reproduction after the fix.

## Solution Assessment

Unclear. Preferring `pkgRoot` is plausible for staged updates, but current main already uses the `pkgRoot`-inferred npm binary when present, so the remaining behavior should be reviewed as an explicit fallback rather than treated as fully proven.
