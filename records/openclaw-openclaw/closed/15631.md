## Summary

Current main already implements Mattermost outbound media/file uploads, resolves ambiguous DM targets to a real DM channel before upload, and includes regression coverage for both direct sends and reply-media delivery. The same implementation is present in the shipped v2026.4.22 release commit.

## Close Comment

Closing as implemented on current OpenClaw. Codex review against `main` found that Mattermost outbound sends now upload media through the Mattermost files API and attach the returned `file_ids` when `mediaUrl` is present (`extensions/mattermost/src/mattermost/send.ts:395`, `extensions/mattermost/src/mattermost/client.ts:495`, `extensions/mattermost/src/mattermost/client.ts:557`). The DM-channel bug described in the April 6 reproduction is also addressed: bare 26-character Mattermost IDs are resolved user-first, then the send path creates/uses the DM channel before uploading (`extensions/mattermost/src/mattermost/target-resolution.ts:53`, `extensions/mattermost/src/mattermost/send.ts:345`). There is explicit regression coverage for both outbound media upload and the bare-user-ID DM upload case (`extensions/mattermost/src/mattermost/send.test.ts:224`, `extensions/mattermost/src/mattermost/send.test.ts:294`), and the inline reply/media path routes media through the same send function (`extensions/mattermost/src/mattermost/reply-delivery.ts:51`, `extensions/mattermost/src/mattermost/reply-delivery.test.ts:134`). I also verified that the shipped `v2026.4.22` release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5` contains this implementation, so this request appears fixed in release.
