## Summary

Keep open: this PR targets a real Feishu WebSocket proxy bug with a narrow owner-local code change, but it still needs patched-branch real behavior proof and has a concrete test-env cleanup bug plus missing operator-facing docs/changelog.

## What This Changes

The PR makes Feishu WebSocket proxy-agent creation opt in via `OPENCLAW_FEISHU_WS_USE_PROXY=1` and updates Feishu client proxy tests for the new default and opt-in behavior.

## Best Possible Solution

Land the Feishu-owned default change after adding env cleanup, a changelog and docs entry, and patched-branch live proof while preserving managed proxy routing through the existing global-agent path.

## Reproduction Assessment

Yes. Source inspection on current main shows Feishu WebSocket creation resolves an ambient proxy agent from HTTP(S) proxy env and passes it to the Lark WS client; the linked issue and PR body provide live failure/workaround logs, but I did not run a live Feishu gateway.

## Solution Assessment

Yes, with completion work. Changing the Feishu-owned `getWsProxyAgent()` path is the narrow maintainable fix, and dependency/source review shows managed proxy mode still routes WebSocket traffic through global-agent when active.
