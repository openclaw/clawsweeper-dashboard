## Summary

Keep #69132 open. Current main no longer hard-codes `/api/experimental/web_search`, but it still sends a single request to the resolved Ollama host at `/api/web_search`; the issue's repro already shows that endpoint also returns 404 on the affected local Ollama host. The open #69222 path remains the concrete candidate fix for local 404 fallback plus cloud retry/auth scoping.

## Best Possible Solution

Keep #69132 open as the canonical bug. The best maintainer path is to land or rework the #69222-style fix in `extensions/ollama/src/web-search-provider.ts`: handle local `/api/web_search` 404s, preserve/decide whether to keep legacy `/api/experimental/web_search`, retry `https://ollama.com/api/web_search` only with correctly scoped `OLLAMA_API_KEY`, update docs/schema as needed, and add regression coverage for the dual-local-404 and auth-boundary cases. Retitling the issue to mention current `/api/web_search` local 404s would reduce confusion.
