## Summary

Current main and the shipped v2026.4.27 release already solve the user-facing npm global update failure with a broader staged-update flow: npm installs into a temporary prefix, OpenClaw verifies the staged package, then swaps the package tree and copies the npm-generated bin shim into the real prefix. This makes the PR’s post-hoc package.json bin repair obsolete, and the PR also adds an unscoped cleanup path that should not be merged as-is.

## What This Changes

This PR adds post-update npm global bin repair helpers that parse package.json bin metadata, delete stale `.openclaw-*` bin entries, recreate missing Unix symlinks, and cover that behavior with unit tests.

## Best Possible Solution

Keep the shipped staged npm global update path. If maintainers still want cosmetic cleanup of historical hidden bin leftovers, it should be a separate narrowly-scoped doctor/update cleanup that proves npm ownership before unlinking files.

## Reproduction Assessment

Yes. #63719 provides a concrete Linux npm global install failure mode, and the current-main unit test exercises the replacement path by staging a package update and asserting that the target `openclaw` bin shim is present after the swap.

## Solution Assessment

Yes for current main; no for this PR as the merge vehicle. The staged-prefix design is narrower and more maintainable because npm creates the canonical shim in isolation, while this PR reparses package metadata and deletes live bin-directory entries after the fact.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Constrain cleanup before unlinking bin entries:** `src/infra/update-global.ts:479`
  - body: The cleanup loop removes any non-directory file or symlink in the global bin directory named `.openclaw-*`. A user or tool-owned hidden entry with that prefix would be silently deleted during update, so the repair should only unlink entries it can prove are npm reify leftovers for this package.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The PR is obsolete, and its proposed live global-bin cleanup has a concrete low-severity filesystem integrity concern.

Concerns:

- **[low] Unscoped global-bin cleanup:** `src/infra/update-global.ts:479`
  - body: `repairGlobalBinLinks` unlinks any non-directory `.openclaw-*` entry under the npm global bin directory. Because that directory is user-writable in common global-prefix setups, the patch could delete unrelated hidden files or symlinks that happen to share the prefix.
  - confidence: 0.84
