## Summary

Current main already addresses this request well enough: Anthropic Claude 4.6 no longer defaults to interleaved thinking, the transport skips the interleaved-thinking beta for adaptive-capable Claude 4.6 models, and users can already disable or override thinking via existing global, per-agent, and per-model config.

## Close Comment

Closing as implemented on `main` after a Codex review. Current `main` no longer uses the old interleaved-thinking path for Anthropic Claude 4.6 by default: `src/agents/model-thinking-default.ts:64` resolves configured Claude 4.6 models to `adaptive`, and `src/agents/anthropic-transport-stream.ts:117` plus `src/agents/anthropic-transport-stream.ts:529` only enable the `interleaved-thinking-2025-05-14` beta for models that do not support adaptive thinking. Users also already have config/runtime controls to turn thinking off or override it via `agents.defaults.thinkingDefault`, per-agent `thinkingDefault`, per-agent `reasoningDefault`, and `/think off`, as documented in `docs/tools/thinking.md:25` and `docs/gateway/configuration-reference.md:1291` / `docs/gateway/configuration-reference.md:1818`. The main proof commit is `59a8afe6fa030f18af662e125e7dd826ce9a53ca`. I do not see a released tag containing that commit yet, so there is no confirmed shipped release to cite.
