## Summary

Close as duplicate/superseded by #45698. This report is part of the same v2026.3.12 Control UI responsiveness regression cluster, and the discussion here narrowed it to overview/dashboard fan-out rather than a separate session-bloat or channel-probe-only bug. Current main still has that overview aggregation surface, so remaining work should be consolidated in the still-open #45698 thread.

## Best Possible Solution

Use #45698 as the canonical Control UI v2026.3.12 responsiveness thread. The best fix path is to instrument Overview RPC latency, lazy-load or defer heavier Overview panels such as debug, usage, logs, and model-auth status, and verify slow provider/status paths cannot make the dashboard feel frozen.
