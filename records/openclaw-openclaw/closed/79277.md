## Summary

Codex review found that current OpenClaw does not implement Feishu inbound mark-read, but the cited Feishu SDK method is a read-status query for bot-sent messages rather than a mutation that can mark inbound user messages as read, so there is no OpenClaw source change to make without a supported Feishu/Lark API.

## What This Changes

Automatically mark inbound Feishu/Lark messages as read after OpenClaw processes them.

## Best Possible Solution

Keep the current Feishu typing-indicator feedback path and reconsider inbound read receipts only if Feishu/Lark exposes an official mark-inbound-read API for bot messages.

## Reproduction Assessment

No live Feishu tenant reproduction was established, but source inspection gives high confidence that current main has no inbound mark-read implementation. The suggested SDK path was verified against the current dependency and is not a mark-read mutation.

## Solution Assessment

No: calling client.im.message.readUsers is not the right fix because the SDK exposes it as a GET query for read users of bot-sent messages. The maintainable path is to use the existing typingIndicator feedback unless Feishu provides a supported mark-inbound-read API.
