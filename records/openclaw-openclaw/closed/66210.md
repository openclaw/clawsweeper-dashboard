## Summary

Keep open. Current main still does not implement the Teams thread copilot approval flow, and I cannot confirm it is fully buildable as an external plugin with today’s Microsoft Teams public surfaces. The M365/Outlook pieces overlap the ClawHub/plugin direction from related cleanup, but the Teams-specific interactive dispatch and thread action seams need maintainer judgment rather than automated closure.

## Best Possible Solution

Keep this PR open for maintainer judgment, but require a split. Move the M365/Outlook service integration to a ClawHub/npm plugin or separate product discussion. Narrow any core-worthy work to additive, documented Teams/plugin SDK seams such as Teams plugin interactive dispatch for Adaptive Cards, thread list/send-thread capability, and focused tests that preserve current file-consent, SSO, dependency, and test-script behavior.
