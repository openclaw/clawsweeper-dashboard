## Label Justifications

- `P1`: The reported missing runtime dependencies disabled real Telegram and Slack channel workflows for existing installs.
- `impact:message-loss`: The issue says Telegram alerts were silently dead and Slack had the same missing-runtime dependency shape.

## Summary

Close as implemented: the report targets old 2026.4.2-era packaging, while current main and the shipped v2026.5.19 package publish Telegram runtime deps directly and route Slack through the official external plugin package that owns `@slack/web-api`.

## What This Changes

The issue asks OpenClaw to stop Telegram and Slack channels from loading runtime imports whose packages are absent from the installed package.

## Best Possible Solution

Keep the shipped packaging model: Telegram runtime deps stay in the core package, Slack remains an official external plugin with its own deps, and doctor/package checks catch missing configured plugin installs.

## Reproduction Assessment

Yes for the reported old npm-global install: the issue includes version, install method, config condition, and concrete gateway errors. I did not rerun the old package because current main and the latest release have the packaging contract fixed.

## Solution Assessment

Yes. The current solution is maintainable: Telegram deps are direct published `openclaw` dependencies, while Slack is an official external plugin that declares its own Slack SDK dependency and has repair coverage.
