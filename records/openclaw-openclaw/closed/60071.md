## What This Changes

The issue asks for the bundled Twitch channel to remain running after a successful IRC connection instead of triggering immediate gateway auto-restarts.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/81853 with the same Twitch-local passive lifecycle wrapper and regression coverage, then close this issue after merge.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction follows Twitch startAccount awaiting a monitor that returns { stop }, after which the gateway supervisor restarts any non-aborted settled account task.

## Solution Assessment

Yes. Wrapping Twitch monitor startup with runStoppablePassiveMonitor is the narrow established fix and avoids changing generic gateway restart semantics for third-party channels.
