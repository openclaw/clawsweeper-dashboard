## Summary

Current main already fixes the reported write/read mismatch for native `sessions_spawn.model`: the spawn writer now persists `modelOverride` and `providerOverride`, the agent-command path reads those fields when selecting the child model, and regression coverage verifies the child session stores the requested model before the first run starts. The fix is on current `main`; it is not present in the latest shipped `v2026.4.26` tag.

## Best Possible Solution

Close this issue as implemented on current main, keep the override-field implementation and regression tests, and let the fix ship with the next OpenClaw release after `v2026.4.26`.
