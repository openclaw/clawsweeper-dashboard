## Summary

Current main already implements this PR's intended ACP stdout isolation fix: `serveAcpGateway()` routes logs to stderr before reading runtime config or bootstrapping the Gateway client, and the linked bug was recorded as fixed in `v2026.4.26`. The open PR diff is therefore obsolete rather than needed for main.

## Best Possible Solution
