## Summary

Current main already fixes the Feishu inbound runtime gap by passing the full gateway channel runtime into Feishu startup; the fix is main-only because no release tag contains it yet.

## What This Changes

The issue reports Feishu WebSocket inbound messages reaching the channel but failing dispatch because the runtime `run` method is undefined.

## Best Possible Solution

Keep the current full-runtime startup fix on main, ship it in the next release, and close this duplicate fixed-on-main report.

## Reproduction Assessment

Yes for the affected release path: the duplicate reports and history show Feishu startup was using a runtime surface without `channel.inbound.run`. Current main no longer reproduces that source path because the full channel runtime is passed into startup.

## Solution Assessment

Yes. Passing the existing full gateway channel runtime is the narrow maintainable fix because Feishu handlers already depend on the channel runtime contract; defensive null checks would only hide the broken startup surface.
