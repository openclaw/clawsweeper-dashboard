## Summary

Keep open. Current main still has the reported behavior path: active-memory sets an AbortController timeout, then directly awaits runRecallSubagent, so the hook can only return timeout after the embedded run observes abort and settles. The linked PR #71687 appears to target the exact bug, but it is open/unmerged in the provided GitHub context and its fix commit is not contained in current main, so the observable bug remains on main.

## Evidence

  - file: [extensions/active-memory/index.ts:1786](https://github.com/openclaw/openclaw/blob/a5f5608d060b23477b4522c1ba4effab2f7eb1c6/extensions/active-memory/index.ts#L1786)
