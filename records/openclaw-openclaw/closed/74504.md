## Summary

Keep open: current main and the latest release still use the old Matrix cross-signing UIA fallback that ignores MAS reset stages, UIA sessions, and reset URLs. The same reporter's open implementation PR is the right tracked fix path, but it remains unmerged and has review/CI blockers.

## What This Changes

Report that Matrix cross-signing bootstrap on MAS-fronted Synapse with PAT auth fails to publish keys and shows a misleading password-UIA fallback instead of a MAS reset URL.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/74509 with a Matrix plugin-owned fix that parses UIA challenges, preserves UIA sessions, surfaces MAS reset URLs, supports the stable OAuth reset shape, keeps the MSC3967 upload path available, and updates focused tests plus operator docs.

## Reproduction Assessment

Yes, at source level: current main's UIA callback can be exercised with no configured password while null and dummy requests reject, which throws the reported password fallback error without inspecting the MAS UIA body, reset URL, or session. I did not repeat the live MAS homeserver run.

## Solution Assessment

Yes in direction: the stage-aware Matrix plugin fix described by the issue is the narrow maintainable path. The issue should stay open until the active PR lands or is superseded because current main and the latest release still have the old behavior.
