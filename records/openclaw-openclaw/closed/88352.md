Add a separate bounded continuity path, or an explicit context-unavailable failure, only for fresh/transient starts that cannot resume native Codex history and lack active context-engine projection.

## Solution Assessment

Yes, the best fix is narrow: restore continuity for non-resumed fresh/transient starts under a separate name while keeping the empty parallel history behavior for actual native resumes.
