## Summary

Current main and the latest shipped release already cover the central user problem: OpenClaw has scheduled heartbeat agent turns that can use a separate heartbeat model, consult HEARTBEAT.md/custom prompts, run in lightweight or isolated context, notify only when useful, and use local model services for on-machine models. The exact dual-brain framing is a product architecture preference rather than a missing core capability.

## What This Changes

Request a second continuous local drive model that observes agent state between user turns and wakes the existing executor with internal directives.

## Best Possible Solution

Keep using the shipped heartbeat, cron, task-flow, and local-model-service surfaces for ambient autonomous work instead of adding a parallel default brain architecture to core.

## Reproduction Assessment

Not applicable; this is a feature/product architecture request, not a reported defect. Source and docs inspection show shipped heartbeat/local-model behavior rather than a current-main failure to reproduce.

## Solution Assessment

Yes; the current heartbeat, cron, and local-model-service approach is the maintainable solution because it reuses existing scheduler and model contracts without adding a heavy default agent-hierarchy architecture.
