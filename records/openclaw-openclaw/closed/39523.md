## What This Changes

The PR adds a Feishu `mediaLocalRoots` config field and makes `sendMediaFeishu` fall back to that account config when loading media, including an `"any"` mode.

## Best Possible Solution

Close this stale branch and resolve any remaining Feishu local-root config or failure-reporting behavior through https://github.com/openclaw/openclaw/issues/51347 with a Feishu-owned, security-reviewed update.

## Reproduction Assessment

Yes, at source level for the review findings: current `loadWebMedia` rejects `localRoots: "any"` without `readFile`, and current Feishu send paths already pass runtime roots that would suppress this PR's account-config fallback.

## Solution Assessment

No. The branch is not the best fix because the linked user failure is already handled on main, and the remaining config/fallback behavior needs the canonical security/product decision tracked in https://github.com/openclaw/openclaw/issues/51347.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Route `any` through host-read media loading:** `extensions/feishu/src/media.ts:459`
  - body: `loadWebMedia` rejects `localRoots: "any"` unless a read-file override is provided. Forwarding configured `"any"` directly makes the advertised setting fail with `unsafe-bypass` instead of enabling local sends through the shared safe helper path.
  - confidence: 0.95
- **[P2] Respect configured roots in the main send path:** `extensions/feishu/src/media.ts:450`
  - body: Regular Feishu outbound sends already pass runtime `mediaLocalRoots` into this function. Because the fallback uses `mediaLocalRoots ?? account.config?.mediaLocalRoots`, injected runtime roots suppress the configured account roots, so the new config still would not apply in the main flow it targets.
  - confidence: 0.9
- **[P2] Regenerate config metadata for the new key:** `extensions/feishu/src/config-schema.ts:165`
  - body: Adding the schema field without updating generated channel config metadata leaves the public Feishu config surface out of sync, so config/help surfaces would omit the new setting.
  - confidence: 0.82
