## Summary

Current main already addresses the report. The onboarding stall/freezing symptom from v2026.4.24 is covered by the post-release cold-path setup/auth/model fixes recorded under 2026.4.26. MiniMax CN API-key onboarding is wired through the shared provider API-key flow and persists the secret to the per-agent auth profile store; `openclaw.json` intentionally only records `auth.profiles` metadata, so an `apiKey` value missing there is expected rather than a failed secret write.

## Best Possible Solution

Close the issue as fixed on current main. Keep the shipped 2026.4.26 onboarding cold-path fixes and the existing credential split: `openclaw.json` stores auth-profile metadata, while API keys live in per-agent `auth-profiles.json`. If a user can still reproduce a native-Windows terminal freeze on 2026.4.26 or newer, they should file a new issue with the exact step that hangs plus logs or screenshots.
