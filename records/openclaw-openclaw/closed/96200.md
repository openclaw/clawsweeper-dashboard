## Summary

## What This Changes

The PR changes Windows PowerShell exec shell arguments to prepend UTF-8 initialization and error-suppression commands, plus updates the Windows shell-config test expectation.

## Best Possible Solution

Keep the shipped shared Windows output decoder as the canonical fix and close this redundant PR rather than adding global PowerShell error suppression.

## Solution Assessment

No; the best fix is the existing shared Windows output decoder, which handles legacy and UTF-8 output without changing PowerShell error semantics. The PR is a plausible shell-boundary idea, but it is redundant and less safe because it suppresses errors globally.
