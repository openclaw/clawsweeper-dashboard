## Summary

Current `main` preserves and applies configured provider `baseUrl` values again during provider merge and runtime model resolution, and Z.AI coding-endpoint coverage is present in code and tests. This looks fixed on `main`, but I could not tie it to a shipped release tag from the available local history, so `fixedRelease` is `null`.

## Evidence

- **Provider merge now preserves existing baseUrl:** `shouldPreserveExistingBaseUrl(...)` keeps a non-empty existing provider `baseUrl` unless the provider API surface changed, which addresses upgrade/regeneration paths that would otherwise overwrite a Coding Plan endpoint.

  - sha: [db8f7478b435](https://github.com/openclaw/openclaw/commit/db8f7478b43550d2949c04b2a9c9ca7fd474dad3)
