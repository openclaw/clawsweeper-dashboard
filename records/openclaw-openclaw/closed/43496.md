## Summary

Current main implements ACP session action support through the dedicated `/acp` control surface and ACP-aware `sessions_send` behavior. The exact `subagents(action=steer)` path remains native-subagent-only, but docs and implementation now provide ACP-specific steer/status/cancel/close paths, which solves the issue's core request.
