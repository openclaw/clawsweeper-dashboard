Keep this PR open. Current main still does not implement `gateway.handshakeTimeoutMs` as a config-file setting; the shipped path only supports the environment override plus the 10s default. The PR is still aimed at a relevant gateway timeout problem, but the submitted diff is incomplete and stale against the current handshake-timeout implementation.

Keep the PR open and request a rebase plus complete implementation against current main: add `gateway.handshakeTimeoutMs` to `GatewayConfig`, the strict Zod schema, labels/help, generated schema/docs surfaces, and focused tests for env > config > default precedence. Wire runtime behavior through the current `src/gateway/handshake-timeouts.ts` path using an existing gateway startup/runtime config snapshot, and consider overlap with the still-open gateway timing report #73592.
