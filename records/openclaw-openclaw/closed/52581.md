## Summary

Current main already implements the requested subagent spawn/completion lifecycle hook surface through public plugin hooks: `subagent_spawned` for accepted `sessions_spawn` runs, `subagent_ended` for completion/cleanup outcomes, and adjacent `subagent_spawning` / `subagent_delivery_target` hooks for thread binding and completion routing. The hook catalog and release tag v2026.4.24 both include this surface, so the issue's central request is implemented even though the final event names are not `agent:spawn` / `agent:complete`.
