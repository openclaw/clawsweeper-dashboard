## Summary

Current main still has the hook-completion routing bug this PR targets: `dispatchAgentHook` computes the completion/error event session from the default main session, while the request handler already passes a normalized target `agentId` into the dispatch payload. The PR is a focused security fix with matching tests and no apparent supply-chain or unrelated infrastructure changes, so it should stay open for maintainer security review and landing rather than be closed by cleanup.

## Best Possible Solution

Keep this PR open. The best maintainer path is to review and land this focused fix, or apply an equivalent change on main that resolves the completion/error event session with `resolveAgentMainSessionKey` when `value.agentId` is present, while keeping the added success/error regression tests and integration checks that drain the target agent session. After merge, #24693 can close as implemented.
