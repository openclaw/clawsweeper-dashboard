## Summary

Current main already treats `~/.openclaw/workspace` as the agent workspace for file operations, and the existing `read`/`write`/`edit` tools are wired against that workspace root. The issue appears to conflate the workspace (`~/.openclaw/workspace`) with the separate agent state directory under `~/.openclaw/agents/<agentId>/agent`.

## Evidence

- **Workspace is the file-tool root:** Docs state the workspace is the only working directory used for file tools and defaults to `~/.openclaw/workspace`.
  - file: [docs/concepts/agent-workspace.md:9](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/docs/concepts/agent-workspace.md#L9)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Workspace vs agent dir are distinct:** Docs distinguish `~/.openclaw/workspace` from `~/.openclaw/agents/<agentId>/agent`, which addresses the issue’s core premise.
  - file: [docs/concepts/multi-agent.md:55](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/docs/concepts/multi-agent.md#L55)
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
