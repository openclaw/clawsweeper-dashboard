## Summary

Current `main` routes `exec` through a supervisor path that does not attach a drain-based no-output watchdog to normal exec runs. Pending stdout/stderr caps only trim buffered poll output, so the reported SSH burst-triggered SIGKILL behavior is already addressed on current `main` and matches the latest release code for the relevant files.

## Evidence

- **Exec path no longer passes a no-output watchdog:** The `exec` tool now calls `runExecProcess(...)` with `timeoutSec` only; this call site does not pass any `noOutputTimeoutMs` or drain-related liveness signal.
  - file: [src/agents/bash-tools.exec.ts:1692](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/agents/bash-tools.exec.ts#L1692)
  - command: `nl -ba src/agents/bash-tools.exec.ts | sed -n '1650,1715p'`
  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Supervisor only kills for no-output when explicitly configured, and it keys off raw stdout/stderr arrival:** `createProcessSupervisor()` arms the no-output timer only when `input.noOutputTimeoutMs` is set, and `touchOutput()` is called directly from `adapter.onStdout(...)` and `adapter.onStderr(...)`. That means bursty SSH output resets liveness when bytes arrive; it does not depend on later `process poll` drains.
  - file: [src/process/supervisor/supervisor.ts:96](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/process/supervisor/supervisor.ts#L96)

  - sha: [5dfc1b90e176](https://github.com/openclaw/openclaw/commit/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213)
- **Pending-output caps are buffer trimming only, not process-kill logic:** The exec session registry appends stdout/stderr into pending buffers, caps those buffers in memory, and drains them on poll. There is no kill/watchdog path here; overflow only marks output truncated and trims buffered text.
  - file: [src/agents/bash-process-registry.ts:108](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/agents/bash-process-registry.ts#L108)
