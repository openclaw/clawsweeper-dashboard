## Summary

Keep open. Current `main` has partial exec-notification plumbing, but the cron isolated-agent path still does not thread `deliveryRequested`/`notifyOnExit` suppression into embedded exec runs, and the exec session registry still does not retain termination reasons needed to suppress no-output `manual-cancel` completion wakes.

## Evidence

- **Cron flow distinguishes delivery.mode=none upstream:** `resolveCronDeliveryContext()` already computes `deliveryRequested: deliveryPlan.requested`, so current `main` does know when cron delivery was not requested.
  - file: [src/cron/isolated-agent/run.ts:322](https://github.com/openclaw/openclaw/blob/cc0992564bc565791810e6a560d12b430a8544ee/src/cron/isolated-agent/run.ts#L322)
  - sha: [cc0992564bc5](https://github.com/openclaw/openclaw/commit/cc0992564bc565791810e6a560d12b430a8544ee)
- **Cron executor does not receive deliveryRequested:** `runCronIsolatedAgentTurn()` calls `executeCronRun()` without any `deliveryRequested` argument, so the executor cannot currently disable exec exit notifications for silent cron runs.
  - file: [src/cron/isolated-agent/run.ts:952](https://github.com/openclaw/openclaw/blob/cc0992564bc565791810e6a560d12b430a8544ee/src/cron/isolated-agent/run.ts#L952)
  - sha: [cc0992564bc5](https://github.com/openclaw/openclaw/commit/cc0992564bc565791810e6a560d12b430a8544ee)
- **Embedded cron runs have no notify override seam:** `createCronPromptExecutor()` calls `runEmbeddedPiAgent()` without an `execOverrides.notifyOnExit`-style override, and `PiRunParams.execOverrides` only allows `host | security | ask | node` on current `main`.
  - file: [src/cron/isolated-agent/run-executor.ts:160](https://github.com/openclaw/openclaw/blob/cc0992564bc565791810e6a560d12b430a8544ee/src/cron/isolated-agent/run-executor.ts#L160)
  - sha: [cc0992564bc5](https://github.com/openclaw/openclaw/commit/cc0992564bc565791810e6a560d12b430a8544ee)
- **Exec override type cannot carry notify suppression:** `execOverrides?: Pick<ExecToolDefaults, "host" | "security" | "ask" | "node">` means the current embedded runner contract cannot express `notifyOnExit: false` for cron runs.
  - file: [src/agents/pi-embedded-runner/run/params.ts:112](https://github.com/openclaw/openclaw/blob/cc0992564bc565791810e6a560d12b430a8544ee/src/agents/pi-embedded-runner/run/params.ts#L112)
  - sha: [cc0992564bc5](https://github.com/openclaw/openclaw/commit/cc0992564bc565791810e6a560d12b430a8544ee)
- **Process sessions do not retain termination reason:** `ProcessSession`/`FinishedSession` record exit code and signal only, and `markExited()` accepts no termination-reason argument, so current `main` lacks the state needed to distinguish `manual-cancel` from other failures when deciding whether to emit completion noise.
  - file: [src/agents/bash-process-registry.ts:29](https://github.com/openclaw/openclaw/blob/cc0992564bc565791810e6a560d12b430a8544ee/src/agents/bash-process-registry.ts#L29)
  - sha: [cc0992564bc5](https://github.com/openclaw/openclaw/commit/cc0992564bc565791810e6a560d12b430a8544ee)
- **Manual-cancel still loses cause before notify:** `runExecProcess()` cancels background runs with `managedRun?.cancel("manual-cancel")`, but after `wait()` it calls `markExited(..., outcome.status)` and then `maybeNotifyOnExit()` without preserving `exit.reason`, so no-output manual cancels are still plausible on current `main`.
  - file: [src/agents/bash-tools.exec-runtime.ts:818](https://github.com/openclaw/openclaw/blob/cc0992564bc565791810e6a560d12b430a8544ee/src/agents/bash-tools.exec-runtime.ts#L818)
  - sha: [cc0992564bc5](https://github.com/openclaw/openclaw/commit/cc0992564bc565791810e6a560d12b430a8544ee)
