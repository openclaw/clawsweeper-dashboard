## Summary

The report targets the external `aight-utils` community plugin, not OpenClaw core. Current `main` documents native plugin lifecycle hooks as restart-only and routes community plugin ownership/discovery to ClawHub or the plugin’s own repository, so this is best closed as a plugin-maintainer item.

## Evidence

## Risks / Open Questions

- If OpenClaw core is actually re-invoking external plugin `register(api)` without a process restart on current main, that would be a core bug; I did not run a live reproduction because this review was read-only and the reported plugin is not in the repo.
- The external plugin may still need an idempotency guard for its own bootstrap registration even though that fix belongs outside this repository.
