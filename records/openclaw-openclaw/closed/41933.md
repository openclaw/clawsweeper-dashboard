## Summary

Close as implemented. Current main exposes configured agents through the WebChat agent selector sourced from `agents.list`, and selecting an agent with no stored session falls back to its `agent:<id>:main` session; the relevant fix shipped in `v2026.5.28`.

## What This Changes

The issue requests that the Control UI WebChat selector expose all configured agents, including agents without stored session rows.

## Best Possible Solution

Keep the current split control: `agents.list` drives agent selection, while the session picker remains scoped to real sessions for the active agent.

## Reproduction Assessment

No current-main failing reproduction remains. Source and regression coverage show configured agents are now selectable and a configured agent without a stored row targets `agent:<id>:main`.

## Solution Assessment

Yes. The current implementation solves the user problem through a dedicated `agents.list`-backed agent selector, which is cleaner than injecting synthetic agents into the session-row picker.
