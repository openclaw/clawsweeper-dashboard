## Summary

Current `main` already contains the Telegram reasoning-stream fix: hidden `<think>` content is split away from answer text, reasoning previews are only shown when reasoning visibility is explicitly enabled, and this behavior was shipped in `v2026.4.5` and remains present on current `main` (`c1383680405879a3c3f43dceb20a694f54d3c13c`).

## Evidence

- **Shipped changelog entry:** `v2026.4.5` includes a Telegram reasoning fix: only create a Telegram reasoning preview lane when the session is explicitly `reasoning:stream`, so hidden `<think>` traces from streamed replies do not surface as chat previews on normal sessions.
