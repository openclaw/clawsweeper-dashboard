## Summary

The report describes a custom skill-sidecar scheduler feature, not a supported core regression on current main. OpenClaw intentionally treats skills as `SKILL.md` prompt assets, while custom routing/orchestration belongs in plugins/ClawHub per `VISION.md` and the plugin docs.

## Evidence

- **Skill loader contract is `SKILL.md` only:** The local skill loader builds each skill from `<skillDir>/SKILL.md`, reads that file, parses frontmatter, and returns the skill metadata from it. There is no sidecar JSON read in this path.
  - file: [src/agents/skills/local-loader.ts:50](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/agents/skills/local-loader.ts#L50)
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Workspace discovery only recognizes folders containing `SKILL.md`:** Workspace skill discovery checks for `SKILL.md` at the root or in immediate child folders and loads only those entries. This is the current main discovery boundary for skills.
  - file: [src/agents/skills/workspace.ts:400](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/src/agents/skills/workspace.ts#L400)
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Docs define skills as `SKILL.md` instruction folders:** The user-facing skills docs say a skill is a directory containing a `SKILL.md` with instructions, and plugin-shipped skills are still just skill directories loaded through the same mechanism.
  - file: [docs/tools/skills.md:9](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/docs/tools/skills.md#L9)
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Skill watching only refreshes the skill snapshot:** The skills config docs state watcher support refreshes the skill snapshot and picks up changes on the next agent turn; it does not describe arbitrary runtime scheduler config execution from sidecar files.
  - file: [docs/tools/skills-config.md:89](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/docs/tools/skills-config.md#L89)
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Project scope prefers plugins for optional capability:** `VISION.md` says core should stay lean, optional capability should usually ship as plugins, and bundle-style plugins are specifically for skills, MCP servers, and related configuration.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/VISION.md#L54)
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Requested routing/orchestration is explicitly out of default core scope:** `VISION.md` lists agent-hierarchy frameworks and heavy orchestration layers that duplicate existing agent/tool infrastructure among the things OpenClaw will not merge into core by default.
  - file: [VISION.md:106](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/VISION.md#L106)
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
- **Supported extension path is external plugins via ClawHub or npm:** The plugin docs say new capability should be built as a plugin and published outside the core repo, with install/distribution through ClawHub or npm.
  - file: [docs/plugins/building-plugins.md:11](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/docs/plugins/building-plugins.md#L11)
  - sha: [7e52223d3275](https://github.com/openclaw/openclaw/commit/7e52223d327555c611e8c91ee2500897e9eab0bf)
