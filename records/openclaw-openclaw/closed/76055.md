## Summary

Keep open. The Feishu message-tool bug is still present on current main, but this PR needs changes before merge because it drops mixed message text, crosses the lazy runtime boundary, and currently lacks after-fix real Feishu proof.

## What This Changes

The PR exports Feishu outbound card-element helpers and uses one from the Feishu message-tool presentation-card builder so buttons and selects can render as card elements.

## Best Possible Solution

Extract or share a lightweight Feishu presentation-card helper for message actions and outbound payloads, preserve mixed text before block elements, keep runtime-heavy imports lazy, update focused regression tests, and attach redacted real Feishu proof.

## Reproduction Assessment

Yes. A Feishu message-tool send with presentation.blocks containing a buttons block enters buildFeishuPresentationCard, and current tests assert the button renders as markdown fallback text rather than a native Feishu action element.

## Solution Assessment

No. Reusing the native renderer is the right direction, but this implementation loses message text when any block renders, imports outbound runtime code into the channel entry, and leaves unused imports that violate the extension typecheck settings.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve message text when rendering blocks:** `extensions/feishu/src/channel.ts:156-164`
  - body: The new builder only uses fallbackText when no block produces an element. Feishu message-tool sends can include both message/text and presentation, so adding any button, select, or text block now drops the actual message text; prepend the escaped text like buildFeishuPayloadCard does or share that behavior directly.
  - confidence: 0.9
- **[P2] Keep card rendering out of the channel entry:** `extensions/feishu/src/channel.ts:32-35`
  - body: Importing buildFeishuCardElementForBlock from ./outbound.js makes the channel entry eagerly load outbound send/client/media code that is otherwise behind loadFeishuChannelRuntime. Move the pure card rendering helper to a lightweight shared module and use it from both paths.
  - confidence: 0.84
- **[P2] Remove unused imports before typecheck:** `extensions/feishu/src/channel.ts:29-34`
  - body: MessagePresentationBlock, MessagePresentationButton, and buildFeishuPayloadButton are imported into channel.ts but never used. tsconfig.extensions.json enables noUnusedLocals, so the extension typecheck will fail until those imports are removed or the helper split uses them.
  - confidence: 0.95
