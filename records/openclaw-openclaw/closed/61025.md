## Summary

Current main already routes IDENTITY.md avatar paths through the Control UI avatar endpoint, filters assistant avatar values to renderable URLs/blob URLs, and renders the avatar as a separate chat-side element rather than inline in message text. The changelog shows related Control UI avatar fixes shipped in 2026.4.21.

## Evidence

- **Chat renderer places avatar beside the message group, not inside bubble content:** `renderMessageGroup()` renders `renderAvatar(...)` before `.chat-group-messages`, so the assistant avatar is a sibling element in the chat row rather than part of the message body.
  - file: [ui/src/ui/chat/grouped-render.ts:327](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/ui/src/ui/chat/grouped-render.ts#L327)

  - file: [ui/src/ui/chat/grouped-render.ts:700](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/ui/src/ui/chat/grouped-render.ts#L700)
