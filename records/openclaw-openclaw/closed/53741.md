## Summary

Keep this PR open. Current main still lacks the requested Discord voice `autoJoin.sessionChannelId` routing and text delivery behavior, but the PR is not merge-ready because the branch enables Discord delivery without a target, leaves validation-failure cleanup and same-guild validation gaps, and overlaps with the broader open voice-as-text-session design in #73699.

## What This Changes

This PR adds optional Discord voice `autoJoin.sessionChannelId` config/schema/generated-baseline support, routes auto-joined voice sessions through that channel, enables delivered Discord text replies for routed voice turns, and adds session-channel validation tests.

## Best Possible Solution

Land a revised narrow implementation only if maintainers want this config option: update the current split `manager.ts` and `segment.ts` structure, keep the config/schema/generated-docs surface aligned, document the option in the Discord channel docs, validate a same-guild messageable target with cleanup on failure, pass an explicit Discord delivery target/run context, keep text delivery best-effort relative to voice playback, preserve routing during DAVE recovery, and cover those paths with focused Discord voice and delivery tests. If #73699 becomes the chosen product direction, supersede this PR with that broader bridge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Pass an explicit Discord delivery target:** `extensions/discord/src/voice/manager.ts:663`
  - body: When `sessionChannelId` is set, this flips `deliver` on but still does not pass `to`, `replyTo`, or `runContext.currentChannelId`. The delivery path uses those fields to resolve the Discord recipient, so routed voice turns can fail target normalization before TTS playback is queued.
  - confidence: 0.93
- **[P1] Destroy connections on validation failure:** `extensions/discord/src/voice/manager.ts:416-426`
  - body: The new `sessionChannelId` validation runs after the voice connection reaches Ready. These error returns report failure without destroying the connection or registering it in `this.sessions`, leaving an unmanaged live voice connection that later status/leave flows cannot clean up.
  - confidence: 0.9
- **[P1] Reject session channels without a guild:** `extensions/discord/src/voice/manager.ts:425-426`
  - body: The same-guild check only rejects a mismatched `guildId`; channel objects with no guild id can still become the routed session target. Require a guild channel and then compare it to the joined voice guild before using the configured id.
  - confidence: 0.86
- **[P2] Preserve routed sessions on recovery rejoin:** `extensions/discord/src/voice/manager.ts:322`
  - body: Adding a distinct `sessionChannelId` means recovery paths must carry that value forward. The existing decrypt-failure recovery rejoin still calls `join()` with only the voice channel, so a routed session falls back to voice-channel routing after recovery.
  - confidence: 0.88
- **[P2] Keep text delivery best effort:** `extensions/discord/src/voice/manager.ts:663`
  - body: With delivery enabled for routed voice sessions, a transient Discord send failure can throw out of the agent command before TTS is synthesized or queued. Text mirroring should not silence the voice reply when Discord posting fails.
  - confidence: 0.82
