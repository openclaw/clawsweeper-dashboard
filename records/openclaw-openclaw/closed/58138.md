## Summary

Current `main` contains the cron-specific timeout changes that address the reported false timeout path: unconfigured cron-triggered runs no longer inherit the default LLM idle watchdog, isolated cron runs pass the `cron` trigger into the embedded runner, and `agentTurn` cron jobs now use a longer safety timeout while honoring explicit `timeoutSeconds`. The original ~120s timeout behavior described against `2026.3.28` is implemented as fixed.

## Evidence

- **Cron trigger disables default idle watchdog:** `resolveLlmIdleTimeoutMs` returns `0` for `trigger: "cron"` when no explicit LLM idle timeout or agent timeout is configured, removing the old default idle-timeout abort path.

  - file: [src/cron/isolated-agent/run-executor.ts:160](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/cron/isolated-agent/run-executor.ts#L160)
