## Summary

Current main and the latest stable tag already implement the central Feishu streaming-card behavior requested here: streaming Feishu replies register tool-start callbacks and write a transient tool status into the live card while preserving the final answer without the transient status. The exact sample text and a separate `showToolCalls` field are not present, but the observable blank-wait problem is covered by shipped code and tests.

## What This Changes

The issue requests Feishu channel streaming cards to show live tool-call progress, ideally with start/done status text and an optional display toggle.

## Best Possible Solution

Keep the shipped Feishu extension implementation as the canonical behavior for this request. If someone later needs persistent completion/duration rows or a Feishu-specific visibility flag, that should be a narrower follow-up rather than keeping this broad request open.

## Reproduction Assessment

Yes. A focused code-level reproduction is covered by `extensions/feishu/src/reply-dispatcher.test.ts`: start a Feishu streaming-card reply, invoke `onToolStart` with a tool name, and verify the streaming update includes that tool status before the final close.

## Solution Assessment

Yes. The implementation keeps Feishu-specific card behavior inside the Feishu plugin, uses the existing reply-options tool event, and avoids changing core channel contracts for a channel-owned UX detail.
