## Summary

Current main, also shipped in v2026.4.24, runs Windows gateway exec through PowerShell directly as an argv vector. The reported `$_` corruption requires an outer shell to expand the command first; that is not the current gateway exec path. The exact `shell: "powershell"` API was not added, but the observable pipeline-variable corruption is addressed for the reported native Windows gateway surface.

## Evidence

- **Windows exec selects PowerShell:** `getShellConfig()` uses PowerShell on `win32`, preferring `pwsh` and falling back to Windows PowerShell, with `-NoProfile -NonInteractive -Command`.
  - file: [src/agents/shell-utils.ts:54](https://github.com/openclaw/openclaw/blob/dcfd5913fd24a7fce8fe804bf687f4d4856a6652/src/agents/shell-utils.ts#L54)
  - command: `nl -ba src/agents/shell-utils.ts | sed -n '1,190p'`
  - sha: [dcfd5913fd24](https://github.com/openclaw/openclaw/commit/dcfd5913fd24a7fce8fe804bf687f4d4856a6652)
- **Exec passes command as argv, not through an outer shell string:** `runExecProcess()` builds `childArgv = [shell, ...shellArgs, execCommand]` and forwards that argv to the supervisor. This preserves `$_.LastWriteTime` until PowerShell parses it.
  - file: [src/agents/bash-tools.exec-runtime.ts:708](https://github.com/openclaw/openclaw/blob/dcfd5913fd24a7fce8fe804bf687f4d4856a6652/src/agents/bash-tools.exec-runtime.ts#L708)
  - command: `nl -ba src/agents/bash-tools.exec-runtime.ts | sed -n '520,735p'`
  - sha: [dcfd5913fd24](https://github.com/openclaw/openclaw/commit/dcfd5913fd24a7fce8fe804bf687f4d4856a6652)
- **Supervisor spawns argv directly:** The child adapter ultimately calls Node `spawn(argv[0], argv.slice(1), options)` with no shell wrapper, so there is no bash/cmd interpolation layer that can strip `$`.
  - file: [src/process/spawn-utils.ts:62](https://github.com/openclaw/openclaw/blob/dcfd5913fd24a7fce8fe804bf687f4d4856a6652/src/process/spawn-utils.ts#L62)
  - command: `nl -ba src/process/spawn-utils.ts | sed -n '1,110p'`
  - sha: [dcfd5913fd24](https://github.com/openclaw/openclaw/commit/dcfd5913fd24a7fce8fe804bf687f4d4856a6652)
- **Local PowerShell reproduction check:** Direct `-Command` argv preserved `$_` and returned `1` and `2`; a deliberately bash-wrapped variant expanded `$_` before PowerShell and failed with the same class of corruption. This matches the code-path distinction above.
