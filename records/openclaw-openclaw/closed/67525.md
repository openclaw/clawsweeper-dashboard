## Summary

Current `main` already fixes the reported `openclaw doctor` crash by hydrating legacy plugin interactive state before cleanup clears dedupe caches, and the fix is covered by a regression test and shipped in `v2026.4.23`.

## Evidence

- **Legacy state is hydrated before cleanup:** `hydrateInteractiveState` rebuilds missing `callbackDedupe` and `inflightCallbackDedupe` fields from older partial global state, and `getState` replaces the legacy singleton with the hydrated shape before callers use it.
