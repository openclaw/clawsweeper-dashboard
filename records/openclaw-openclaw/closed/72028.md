## Summary

Keep open. Current main still duplicates the native subagent task in both the standing subagent system prompt and the initial child message, so the linked bug is not implemented on main. This PR is an active, focused fix candidate, but it still represents a maintainer design choice because it makes the kickoff message the canonical task source while related #72053 keeps the task in the system prompt instead.

## Best Possible Solution

Keep this PR open for maintainer review and consolidation with #72053. The final fix should keep the full delegated task in exactly one authoritative model-visible location, preserve multiline and structured task text, avoid ambiguous task binding for forked or persistent sessions, and include regression coverage for the final `message` plus `extraSystemPrompt` gateway payload.
