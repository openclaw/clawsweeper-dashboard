## Summary

Close as duplicate/superseded by #75591. This report is part of the same v2026.4.29 embedded-run prep-stage stall cluster already tracked by #75591, with corroborating open reports #75642 and #75650; the WebSocket handshake failures are a downstream symptom of the same event-loop starvation rather than a separate repository action.

## What This Changes

Reports that embedded agent initialization in v2026.4.29 blocks the Gateway event loop for roughly 33 seconds, causing WebSocket handshake timeouts, stuck sessions, and CPU spikes.

## Best Possible Solution

Use #75591 as the canonical root-cause tracker for the v2026.4.29 repeated plugin manifest/tool discovery embedded-run latency, fold this report's WebSocket handshake and liveness metrics into that investigation, and fix the embedded-run hot path rather than adding another parallel issue.
