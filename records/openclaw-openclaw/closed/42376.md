## Summary

Current main and the v2026.4.24 release no longer use the reported abort-on-next-message behavior for normal inbound reply turns. The generic reply pipeline defaults to collect-mode followups, enqueues same-session messages while a run is active, preserves account/thread routing metadata for multi-instance plugins, and dedupes queued redeliveries by route and message id.
