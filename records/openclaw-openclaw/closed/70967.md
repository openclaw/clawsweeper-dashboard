## Summary

#70967 reports a real remaining mismatch on current main: gateway session rows/defaults still expose provider/model-derived `thinkingDefault`, while runtime chat/status paths use the config-aware resolver. The issue can be closed as superseded because open PR #71779 now tracks the same remaining work with the broader fix path and regression coverage; earlier duplicate PR attempts #70981, #71033, and #66655 are also directly cross-referenced in the issue context.

## Best Possible Solution

Close #70967 as superseded by #71779 and let maintainers review/land the canonical PR that changes gateway session rows/defaults to use the effective config-aware thinking default while preserving provider-owned thinking level options.

## Evidence

- **Current session defaults still use provider/model fallback:** `getSessionDefaults()` builds the Control UI defaults object and sets `thinkingDefault` from `resolveThinkingDefaultForModel({ provider, model })`, which does not receive `cfg` and therefore cannot honor `agents.defaults.thinkingDefault`.
  - file: [src/gateway/session-utils.ts:1041](https://github.com/openclaw/openclaw/blob/c74fb781943f3bad7d6fec5256729c861d285cab/src/gateway/session-utils.ts#L1041)

  - sha: [c74fb781943f](https://github.com/openclaw/openclaw/commit/c74fb781943f3bad7d6fec5256729c861d285cab)
- **Current session rows still use provider/model fallback:** `buildGatewaySessionRow()` similarly sets each row's `thinkingDefault` from `resolveThinkingDefaultForModel({ provider: thinkingProvider, model: thinkingModel })`, so the UI row data can still disagree with configured defaults.
  - file: [src/gateway/session-utils.ts:1432](https://github.com/openclaw/openclaw/blob/c74fb781943f3bad7d6fec5256729c861d285cab/src/gateway/session-utils.ts#L1432)
  - command: `nl -ba src/gateway/session-utils.ts | sed -n '1380,1450p'`
  - sha: [c74fb781943f](https://github.com/openclaw/openclaw/commit/c74fb781943f3bad7d6fec5256729c861d285cab)
- **Runtime resolver is config-aware:** The runtime resolver `resolveThinkingDefault()` reads per-model `params.thinking` and then `params.cfg.agents?.defaults?.thinkingDefault`, matching the issue's description that runtime behavior can be correct while the session-row label is not.
  - file: [src/agents/model-thinking-default.ts:39](https://github.com/openclaw/openclaw/blob/c74fb781943f3bad7d6fec5256729c861d285cab/src/agents/model-thinking-default.ts#L39)

  - sha: [c74fb781943f](https://github.com/openclaw/openclaw/commit/c74fb781943f3bad7d6fec5256729c861d285cab)
- **UI label consumes gateway-provided defaults:** The web chat selector formats `Default (${defaultLevel})` from `activeRow?.thinkingDefault` or `state.sessionsResult?.defaults?.thinkingDefault` before falling back locally, so fixing the gateway session row/default payload is the right product path.
  - file: [ui/src/ui/chat/session-controls.ts:213](https://github.com/openclaw/openclaw/blob/c74fb781943f3bad7d6fec5256729c861d285cab/ui/src/ui/chat/session-controls.ts#L213)
