## Summary

Current `main` already prevents the managed browser action path from silently succeeding on covered targets: click/hover go through Playwright locators with normal actionability checks, covered-element failures are rewritten to an explicit `hidden or covered` error, and there is test coverage for the intercepted-pointer-events case. The issue’s proposed occlusion guard is therefore already satisfied well enough on `main`, though as default behavior rather than a separate `verifyOcclusion` flag.

## Evidence
