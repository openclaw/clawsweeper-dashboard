## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Validate xcode_path before selecting Xcode:** `.github/workflows/build-ios-unsigned.yml:50`
  - body: `xcode_path` is substituted into the generated shell script before bash runs it. A crafted dispatch value can break out of the quoted argument and execute commands on the macOS runner, so pass the input through `env`, validate an allowlisted Xcode path in bash, and then quote the variable.
  - confidence: 0.9
- **[P1] Pass build_number through env before invoking the script:** `.github/workflows/build-ios-unsigned.yml:68`
  - body: The digits-only validation in `scripts/ios-write-version-xcconfig.sh` happens too late because the workflow expands `build_number` into the shell command first. Build the `--build-number` argument after validating an environment variable instead of interpolating the expression into `run`.
  - confidence: 0.91
- **[P2] Limit the workflow token before running build scripts:** `.github/workflows/build-ios-unsigned.yml:36`
  - body: This workflow runs package install and build scripts after checkout, but it has no explicit read-only `permissions` block and `actions/checkout@v4` persists credentials by default. Add `permissions: contents: read` and `persist-credentials: false` to match the hardened sibling iOS workflows.
  - confidence: 0.82
