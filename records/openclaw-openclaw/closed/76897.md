## Summary

Current main and the v2026.5.2 release already preserve mixed Telegram text plus `MEDIA:` replies through parsing, final-payload filtering, and Telegram caption/follow-up delivery, so the 2026.4.x symptom is covered by shipped behavior.

## What This Changes

Report requests Telegram direct replies containing visible text plus an inline `MEDIA:` attachment to deliver both the text and the file.

## Best Possible Solution

Keep the shipped mixed text/media Telegram handling; reopen with fresh v2026.5.x logs only if the caption/follow-up path still drops text in a current build.

## Reproduction Assessment

No current-main failing reproduction is available from this read-only review. The old 2026.4.x report is clear, but current source and regression tests route the text to a Telegram caption or follow-up instead of dropping it.

## Solution Assessment

Yes. The current solution is in the narrow Telegram/outbound reply boundaries: parse `MEDIA:`, preserve unsent mixed payload parts, and let Telegram delivery attach short text as a caption or send long text separately.
