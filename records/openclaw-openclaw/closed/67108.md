## Summary

Keep this PR open. Current main and the latest release still lack the proposed chmod-error revalidation path, the related temp-dir race report remains open, and this PR is a narrow implementation candidate that needs maintainer review and CI rather than cleanup closure.

## What This Changes

This PR changes temp-dir writable-bit repair to revalidate after EPERM/EACCES/ENOENT chmod failures and adds a Vitest regression for silent success when another process has already tightened permissions.

## Best Possible Solution

Land or refine a narrow temp-dir repair fix that treats concurrent repair as success only after revalidating ownership, symlink status, mode, and access, with regression coverage for both the chmod-error path and any already-secure race maintainers decide is in scope.
