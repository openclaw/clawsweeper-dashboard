## Summary

Close as implemented on current main: the reported all-channel failure depended on WhatsApp/Baileys being bundled into the core package/runtime-deps path, and main now treats WhatsApp as an on-demand external plugin excluded from the core npm package. The related Baileys/libsignal fix PRs were closed unmerged, but the core-bundle failure path is gone on main.

## What This Changes

The issue reports that OpenClaw 2026.5.2 bundled Baileys/WhatsApp runtime dependencies whose GitHub-sourced libsignal subdependency made pnpm reject installs and prevented Discord/Codex replies.

## Best Possible Solution

Keep the current main packaging model that excludes WhatsApp/Baileys from core installs, and track any remaining standalone WhatsApp plugin install failure as a narrower plugin-specific issue.

## Reproduction Assessment

No against current main. The historical 2026.5.2 failure is plausible, but current main no longer ships WhatsApp in the core package or installs plugin dependencies during packaged postinstall.

## Solution Assessment

Yes for the reported all-channel core-bundle failure. Externalizing WhatsApp/package-local dependency ownership is narrower than adding a global pnpm override to core, while any remaining standalone WhatsApp install problem can be handled separately.
