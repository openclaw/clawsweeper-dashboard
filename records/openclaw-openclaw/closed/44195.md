## Summary

This is a locale-specific privacy/filtering feature request that fits OpenClaw's documented plugin/ClawHub scope, not core maintainer cleanup on `main`. Current core only provides generic configurable log/tool-summary redaction, while the project vision explicitly says optional capability should usually ship as plugins and community distribution belongs on ClawHub.

## Evidence

## Risks / Open Questions

- If maintainers later decide locale-specific PII filtering must be a core security default across all model traffic, this issue could become in-scope again.
- Current core does offer `logging.redactPatterns`, but that is limited to documented redaction surfaces and is not the same as full bilingual traffic filtering/prompt-injection detection.
