## Summary

Current main already hardens macOS gateway restart against the reported LaunchAgent removal path: it re-bootstraps deregistered LaunchAgents, recovers installed-but-unloaded LaunchAgents during `gateway start`/`restart`, and adds restart-health/stale-PID recovery. The changelog says these macOS restart fixes shipped in 2026.4.5.

## Evidence

## Close Comment

Closing from a Codex review because the reported macOS restart/LaunchAgent failure path is already implemented on current `main` and was shipped in 2026.4.5. The release notes call out the exact fixes in `CHANGELOG.md:1255`-`1257` and the related shutdown hardening in `CHANGELOG.md:1411`. In current code, macOS restart now falls back from `launchctl kickstart -k` to `launchctl bootstrap` + retry in `src/infra/restart.ts:403`, and `openclaw gateway start` / `restart` explicitly recover an installed-but-unloaded LaunchAgent in `src/cli/daemon-cli/launchd-recovery.ts:14` and `src/cli/daemon-cli/lifecycle.ts:197`. There is regression coverage for both start and restart recovery in `src/cli/daemon-cli/lifecycle.test.ts:210` and `src/cli/daemon-cli/lifecycle.test.ts:386`. If someone can still reproduce this on a current build, please open a fresh issue with a new diagnostics bundle and exact macOS/install details.
