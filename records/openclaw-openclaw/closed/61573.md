## Summary

Current `main` already gives isolated cron `payload.model` higher precedence than stored session overrides, and the persisted `sessions.json.model` field is treated as runtime metadata rather than as the next-run selector. The reported permanent fallback pinning is therefore already addressed on `main`.

## Evidence

- **Cron selection prefers job payload over stored session state:** `resolveCronModelSelection()` applies `payload.model` first and only consults stored session override fields when no payload model is present; it does not read the persisted runtime `model` field for selection.
  - file: [src/cron/isolated-agent/model-selection.ts:104](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/src/cron/isolated-agent/model-selection.ts#L104)
