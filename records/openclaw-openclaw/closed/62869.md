## Summary

Current `main` fixes the two concrete regressions reported in #62869: heartbeat isolated-session keys are normalized back to a single `:heartbeat` suffix, and `forceNew` isolated-session rollover now clears stale `sessionFile` state so new heartbeat runs stop appending into the prior transcript file.

## Evidence

- **Repeated `:heartbeat` suffixes are normalized:** `resolveIsolatedHeartbeatSessionKey` collapses repeated trailing `:heartbeat` suffixes back to a single isolated key and preserves the base-session marker.
  - file: [src/infra/heartbeat-runner.ts:375](https://github.com/openclaw/openclaw/blob/e6d04682d33c9576edc4ad77b031949f84407bec/src/infra/heartbeat-runner.ts#L375)
  - command: `nl -ba src/infra/heartbeat-runner.ts | sed -n '375,430p'`
  - sha: [e6d04682d33c](https://github.com/openclaw/openclaw/commit/e6d04682d33c9576edc4ad77b031949f84407bec)
- **Nested stale isolated keys are removed and archived:** `runHeartbeatOnce` now detects stale nested isolated heartbeat session keys, deletes the stale store entry, and archives any removed transcript before continuing.
  - file: [src/infra/heartbeat-runner.ts:878](https://github.com/openclaw/openclaw/blob/e6d04682d33c9576edc4ad77b031949f84407bec/src/infra/heartbeat-runner.ts#L878)
  - command: `nl -ba src/infra/heartbeat-runner.ts | sed -n '875,920p'`
  - sha: [e6d04682d33c](https://github.com/openclaw/openclaw/commit/e6d04682d33c9576edc4ad77b031949f84407bec)
- **Fresh isolated sessions clear old transcript path:** `resolveCronSession(... forceNew: true)` explicitly clears `sessionFile` when rolling to a fresh session, removing the reported cause of reusing the same transcript file across heartbeat runs.
  - file: [src/cron/isolated-agent/session.ts:68](https://github.com/openclaw/openclaw/blob/e6d04682d33c9576edc4ad77b031949f84407bec/src/cron/isolated-agent/session.ts#L68)
  - command: `nl -ba src/cron/isolated-agent/session.ts | sed -n '65,95p'`
  - sha: [e6d04682d33c](https://github.com/openclaw/openclaw/commit/e6d04682d33c9576edc4ad77b031949f84407bec)
- **Session-file resolution allocates a new path for a new sessionId:** When the stored `sessionId` changes and no fallback path is carried forward, `resolveAndPersistSessionFile` resolves the transcript path with `sessionFile: undefined`, which yields a fresh transcript file for the new session.
  - file: [src/config/sessions/session-file.ts:21](https://github.com/openclaw/openclaw/blob/e6d04682d33c9576edc4ad77b031949f84407bec/src/config/sessions/session-file.ts#L21)
  - command: `nl -ba src/config/sessions/session-file.ts | sed -n '12,38p'`
  - sha: [e6d04682d33c](https://github.com/openclaw/openclaw/commit/e6d04682d33c9576edc4ad77b031949f84407bec)
- **Regression tests cover both reported behaviors:** The heartbeat runner tests assert already-suffixed and deeply accumulated wake keys converge to `<base>:heartbeat`, and the cron isolated-session tests assert `forceNew` clears a stale `sessionFile`.
  - file: [src/infra/heartbeat-runner.isolated-key-stability.test.ts:102](https://github.com/openclaw/openclaw/blob/e6d04682d33c9576edc4ad77b031949f84407bec/src/infra/heartbeat-runner.isolated-key-stability.test.ts#L102)
