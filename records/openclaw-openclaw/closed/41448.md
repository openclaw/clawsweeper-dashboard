## Summary

Current main already supports this fix through the shipped `compat.requiresStringContent` path for string-only `openai-completions` backends, with transport code, embedded-runner payload wrapping, tests, and public docs covering the exact failure mode described in the PR.

## Risks / Open Questions

- The shipped fix is opt-in via `compat.requiresStringContent`, not an unconditional mutation for every `openai-completions` backend. If a specific provider should auto-enable this compat flag by default, that would be a narrower follow-up than this PR.
- This review is based on current/released source and tests in the checkout. I did not run live reproductions in this read-only pass.
