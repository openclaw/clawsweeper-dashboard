## Label Justifications

- `impact:message-loss`: The affected image content blocks are part of the inbound channel message and were reported as missing from the provider request.

## Summary

The reported 2026.5.19 image-loss path is covered by current main and shipped in v2026.5.20: Anthropic Messages now uses OpenClaw's boundary-aware transport, gateway image routing inlines through an image-capable session model, and the Anthropic payload converter preserves image blocks when the runtime model advertises image input.

## What This Changes

The issue reports that Slack image content blocks reach session storage but disappear before bifrost-routed Anthropic Messages provider requests when relying on active-model vision routing.

## Best Possible Solution

Keep the v2026.5.20+ boundary-aware Anthropic transport path; open a narrower follow-up only if v2026.5.20 or newer still loses image blocks or if transport block-count diagnostics are wanted as separate observability work.
