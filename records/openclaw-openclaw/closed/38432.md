## Summary

Current main already addresses this well enough: chat history filters out the synthetic transcript-repair tool result before rendering, OpenAI/Codex replay paths no longer synthesize missing tool results, and teardown now waits for idle or clears pending tool-call state to avoid restart/long-command races.

## Evidence

- **History view hides the synthetic repair marker:** `loadChatHistory()` filters `toolResult` entries whose text matches the synthetic transcript-repair message before storing `chatMessages`.
  - file: [ui/src/ui/controllers/chat.ts:61](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/ui/src/ui/controllers/chat.ts#L61)
- **History filtering is covered by regression tests:** Tests verify the synthetic transcript-repair tool result is removed from visible history while real tool results remain.
  - file: [ui/src/ui/controllers/chat.test.ts:565](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/ui/src/ui/controllers/chat.test.ts#L565)
- **OpenAI/Codex replay no longer synthesizes missing tool results:** Synthetic tool-result insertion is limited to Anthropic/Gemini/Bedrock APIs; OpenAI-compatible transports are excluded.
  - file: [src/agents/transport-message-transform.ts:3](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/src/agents/transport-message-transform.ts#L3)
