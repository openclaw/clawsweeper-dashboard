## Summary

Close #71975 as ClawHub/plugin scope. The request is for optional LTX model/provider curation, presets, and a possible ComfyUI workflow or dedicated Lightricks provider. Current main already has the generic `video_generate` capability, a bundled fal video provider that can route configured `fal/<model>` refs through fal's queue endpoint, and a public plugin SDK surface for third-party video-generation providers. OpenClaw's documented product boundary says optional capabilities should usually ship as plugins, with plugin discovery/promotion in ClawHub rather than core docs. No maintainer-authored/protected-label signal requires keeping this as a core tracker.

## Best Possible Solution

Keep OpenClaw core focused on the generic `video_generate` contract and existing provider/plugin surfaces. The best next step is an LTX-focused ClawHub/native plugin or bundle that documents fal/Replicate/Lightricks model refs, presets, and optional ComfyUI workflow assets; if that work uncovers a concrete missing video-generation SDK capability, open a narrower core SDK issue.
