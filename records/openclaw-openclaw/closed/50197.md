## Summary

Current `main` already fixes the stale-provider-prefix model-switch behavior in the Control UI. The picker now submits the selected provider-qualified model value directly, the UI re-derives the active model ref from session data/catalog instead of separate prefix state, backend session merging guards against stale provider carry-over, regression tests cover these cases, and the changelog says the broader session/model-selection fix shipped in `v2026.4.10`.
