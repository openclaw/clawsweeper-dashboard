## Summary

Close as `cannot_reproduce`. The reported root cause does not match current `main` or the latest shipped release: OpenClaw's image-understanding path reads local media into Buffers, passes Buffers into provider calls, and serializes image inputs as base64/data URLs. The bundled ZAI provider delegates to that shared implementation. The related timeline cross-reference (#69171) also points to a separate image-model routing bug that is already called out in the changelog.

## Evidence

- **Attachment cache reads local files into memory:** `MediaAttachmentCache.getBuffer()` resolves a local attachment path, reads it with `readLocalBuffer(...)`, and returns `buffer`, `mime`, and `fileName` for downstream provider use rather than keeping a raw filesystem path in the image payload.
  - file: [src/media-understanding/attachments.cache.ts:87](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/media-understanding/attachments.cache.ts#L87)
  - command: `nl -ba src/media-understanding/attachments.cache.ts | sed -n '68,150p'`
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Image runner passes buffer objects to providers:** The image capability branch calls `params.cache.getBuffer(...)` and builds `imageInput` with `buffer`, `fileName`, and `mime`; that object is passed into `describeImage(...)`. No local path is forwarded on this provider call path.
  - file: [src/media-understanding/runner.entries.ts:528](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/media-understanding/runner.entries.ts#L528)

  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Shared image transport serializes base64:** `buildImageContext()` converts each image buffer to `data: image.buffer.toString("base64")`, which is the generic image payload used by shared provider transport.
  - file: [src/media-understanding/image.ts:164](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/media-understanding/image.ts#L164)
  - command: `nl -ba src/media-understanding/image.ts | sed -n '164,173p'`

- **MiniMax-specific branch explicitly builds a data URL:** The provider-specific MiniMax vision branch constructs `data:${mime};base64,...` from the image buffer before sending the request, reinforcing that this subsystem uses encoded image bytes rather than local paths.
  - file: [src/media-understanding/image.ts:217](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/media-understanding/image.ts#L217)
