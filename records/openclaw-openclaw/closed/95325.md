## Summary

Keep open: current main and v2026.6.8 still preserve stale channel-keyed origin fields when the next channel omits them, and open candidate fixes already reference this issue.

## What This Changes

The issue asks OpenClaw to clear channel-specific session origin fields when a shared DM session switches channels and the new inbound omits those fields.

## Best Possible Solution

Land one narrow fix in `src/config/sessions/metadata.ts` that clears channel-keyed origin fields on provider or surface change while preserving same-channel sparse metadata, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection shows `deriveSessionMetaPatch` merges the existing origin with sparse inbound metadata, so a Slack-provided `nativeChannelId` survives a later Telegram direct turn that omits `NativeChannelId`.

## Solution Assessment

Yes. The narrow maintainable fix belongs in `mergeOrigin`, because both the runtime session initializer and store update paths already centralize inbound origin updates there.
