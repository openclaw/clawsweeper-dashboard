## Summary

Current `main` already ships the task-centric runtime pieces this proposal asks for: a durable Task Flow layer above background tasks, revisioned waiting/blocked state for resumable work, plugin-facing task-flow APIs, and first-class exec/plugin approval request objects with channel/UI approval surfaces. The relevant work is documented and released in `v2026.4.2`.
