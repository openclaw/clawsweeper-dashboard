## Summary

Keep open for maintainer handling: the item has a protected `maintainer` label and a `MEMBER` author, and the proposed patch is not merge-ready because the export HTML hamburger remains hidden on mobile due to a CSS specificity regression. Current main does not already contain the requested aligned affordance, so this is not an implemented-on-main or duplicate cleanup case.

## What This Changes

The PR adds a shared `sidebar-menu-trigger` affordance to the Control UI mobile nav toggle, restyles the exported session HTML hamburger to the same three-line trigger pattern, and adds CSS/DOM contract tests for both surfaces.

## Best Possible Solution

Keep the shared Control UI affordance direction, but adjust the export HTML CSS so the mobile display override has equal or higher specificity than the hidden base rule, then keep focused tests that would catch the exported hamburger being visible and interactive on mobile.
