## What This Changes

The issue asks for isolated cron jobs with `deleteAfterRun: true` to delete their run session transcript files when the run finishes, including no-delivery and terminal-error paths.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/84794 or an equivalent runner-owned cleanup that deletes isolated cron run sessions in the terminal run path while preserving the cron-owned session guard.

## Reproduction Assessment

Yes. Source inspection shows isolated runs create transcript-backed run sessions, while `deleteAfterRun` cleanup only happens inside delivery-dispatch paths; bare `delivery.mode: "none"` skips the requested-delivery path.

## Solution Assessment

Yes. Moving the existing cron-session cleanup into a runner-final terminal path, as the linked PR does, is the narrow maintainable fix; changing global retention or adding a new option would not address the contract directly.
