## Summary

Keep #67953 open. Current main still writes file log JSONL by stringifying the raw log object and appending that payload to disk without passing the line through `redactSensitiveText`. The issue is concrete, recent, security-relevant, and the linked replacement PR #69651 is still open rather than merged, so this is not implemented on main and should not be cleanup-closed yet.

## Best Possible Solution

Keep #67953 open until #69651 or an equivalent maintainer-reviewed patch lands. The fix should apply redaction at the core file-log transport boundary, preserve `logging.redactSensitive: "off"`, add regression coverage for direct file logs, child/subsystem loggers, and console capture relay, and update logging docs if file-log redaction semantics change from the current documented console/tool-only behavior.
