## Summary

Current main already fixes the stale subsystem date-rolled log-file bug. The shipped implementation removes the cached subsystem file logger and resolves rolling log files at write time, with regression tests and a changelog entry covering #62381. PR #62458's generation-counter approach is now obsolete, and its own review discussion flagged sequencing/performance issues that the main implementation avoids.

## Best Possible Solution

Close this PR as implemented on main. Keep the cd89adf0 implementation because it solves the user-visible stale rolling-file behavior directly, avoids the generation-counter hot-path concerns raised on #62458, and already has focused regression coverage and changelog/docs updates.
