- **[P2] Publish Krea aspect limits before fallback:** `extensions/fal/image-generation-provider.ts:546-547`
  - body: The Krea schema narrows aspect ratios inside the Fal provider, but provider capabilities still expose only the fal-wide aspect ratio list and no `aspectRatiosByModel` entries for the Krea model ids. Runtime normalization runs before the provider and fallback catches provider errors, so `fal/krea/v2/...` with an unsupported ratio like `4:1` can leave Krea and generate with a later fallback provider instead of being rejected or normalized for Krea.
  - confidence: 0.9
- **[P2] Keep Krea unsupported options out of fallback:** `extensions/fal/image-generation-provider.ts:209-210`
  - body: `supportsCount: false` and `supportsOutputFormat: false` are only provider-private schema flags; the public capabilities still advertise Fal-wide count, output format, and resolution support. With configured fallbacks, Krea requests using `count: 2`, `outputFormat: "jpeg"`, or `resolution: "2K"` can fail inside the provider and then run on a different provider, so these limits need runtime/tool preflight or a non-failover validation path plus fallback tests.
  - confidence: 0.88
