## Summary

Current main and the shipped release already provide the requested provider-aware opt-in: proxy-like OpenAI Responses routes still strip prompt-cache fields by default for compatibility, but `compat.supportsPromptCacheKey: true` preserves them for compatible providers.

## What This Changes

The issue requests preserving `prompt_cache_key` and `prompt_cache_retention`, or adding a config option to preserve them, for OpenAI-compatible `openai-responses` providers that support provider-side prompt caching.

## Best Possible Solution

Keep the conservative default strip for unknown proxy-like Responses endpoints and use `compat.supportsPromptCacheKey: true` on model configs for providers known to forward prompt-cache fields.

## Reproduction Assessment

Yes. Source and tests show the default proxy-like `openai-responses` path strips prompt-cache fields, while `compat.supportsPromptCacheKey: true` disables that strip for compatible providers.

## Solution Assessment

Yes. The existing opt-in is the best fit because changing the default would regress providers like Volcengine that reject `prompt_cache_key`, while the compat flag lets known-good proxies preserve caching.
