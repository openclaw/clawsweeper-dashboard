## Summary

Current main partially addresses the report: the exact `node-domexception@1.0.0` and `glob@10.5.0` entries are gone, and `node-domexception` is overridden to `@nolyfill/domexception`. I would not close this yet because the lockfile still contains deprecated `glob@7.2.3` through `rimraf@3.0.2`, so the broader “glob deprecation warning on update” problem is not fully proven fixed. The linked PR #47896 is still open and had compatibility concerns around a broad glob override.

## Best Possible Solution

Keep the issue open but narrow it to the remaining dependency-warning gap: keep the shipped `node-domexception` nolyfill override, avoid a broad unverified glob major override, and either upgrade/remove the `@discordjs/node-pre-gyp -> rimraf@3.0.2 -> glob@7.2.3` path or close only after a current-main Windows/npm update check shows no glob deprecation warning.
