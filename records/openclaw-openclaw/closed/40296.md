## Summary

Keep this PR open. Current main has added a formal system-prompt cache boundary and already places Group Chat Context below it, but the PR's core remaining behavior is still not implemented: runtime/channel-sensitive Messaging and Reactions guidance is assembled before stable Project Context and before the cache boundary, so it can still invalidate the reusable prefix. The PR discussion remains useful, including the linked tracker #40256 and the author's follow-up fixing the test's fragile `indexOf` checks.

## Best Possible Solution

Refresh this PR against current main or implement the linked #40256 tracker directly. Keep stable Project Context in the reusable prefix, move or split volatile Messaging and Reactions guidance below `SYSTEM_PROMPT_CACHE_BOUNDARY`, preserve stable messaging rules where appropriate, and add a current regression test in `src/agents/system-prompt.test.ts` proving the cache boundary and Project Context precede channel/session-sensitive guidance.
