## Summary

Current `main` already contains an idle-aware fix for late-arriving tool results during session cleanup, plus a regression test that covers the exact failure mode where a synthetic `missing tool result` marker used to win the race over the real tool result.

## Evidence

- **Idle-aware pending-tool-result flush now waits before inserting synthetic repair output:** `flushPendingToolResultsAfterIdle()` races `agent.waitForIdle()` against a timeout, and only flushes/clears pending tool calls after that wait. This prevents immediate synthetic transcript-repair insertion while a real tool result is still in flight.
