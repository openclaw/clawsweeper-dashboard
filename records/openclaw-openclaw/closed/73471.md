## Summary

Keep open. Current main still treats a valid older primary transcript as an orphan whenever it is not the current `sessions.json` target; interactive confirmation and `.deleted` memory indexing reduce damage but do not fix the doctor/session-history semantics.

## What This Changes

The issue asks OpenClaw to stop `doctor --fix` from archiving valid historical session transcripts as orphans and to harden doctor repair recommendations.

## Best Possible Solution

Define a shared semantic session-artifact classifier that preserves valid historical primary transcripts, then handle restore UX, corrupt-store diagnostics, cleanup pruning, and doctor-recommendation CI hardening as focused follow-ups.

## Reproduction Assessment

Yes, at source level. A sessions directory with `sessions.json` pointing at current `A.jsonl` and a valid older `B.jsonl` will still flag `B.jsonl` because current main only excludes paths present in `referencedTranscriptPaths`.

## Solution Assessment

No, not as one automated sweep. The narrow maintainable fix is semantic classification and preservation tests first, with recovery and CI hardening deliberately scoped by maintainers.
