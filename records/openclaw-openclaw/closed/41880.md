This PR changes Feishu post payload construction so a trimmed single bare HTTP(S) URL is emitted as an explicit rich-text link while other text keeps the Markdown path, and adds regression tests for the URL and normal Markdown cases.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

No. This PR is a narrow maintainable fix for URL-only messages, but #68514 is the better canonical path because it also handles bare URLs embedded in normal Markdown while preserving existing linked/code regions.
