## Summary

Current main already suppresses silent isolated cron announce output before Telegram delivery, including exact and trailing `NO_REPLY` cases and stale interim narration; the same implementation is present in shipped release `v2026.4.22`.

## Evidence

- **Cron delivery normalizes silent text:** `normalizeSilentReplyText` strips exact `NO_REPLY`, leading silent tokens, and trailing `NO_REPLY` wrappers before delivery, returning `undefined` for silent output.
  - file: [src/cron/isolated-agent/delivery-dispatch.ts:44](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/cron/isolated-agent/delivery-dispatch.ts#L44)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Direct announce payloads are filtered before outbound send:** Direct cron announce maps each payload through the silent-text normalizer and drops payloads with no remaining user-visible content; if nothing remains it finishes as a silent delivery instead of sending.
  - file: [src/cron/isolated-agent/delivery-dispatch.ts:525](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/cron/isolated-agent/delivery-dispatch.ts#L525)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Interim narration is suppressed for descendant/subagent cases:** When cron orchestration only has stale interim parent text such as 'on it, pulling everything together', delivery is suppressed and marked attempted to avoid fallback leakage.
  - file: [src/cron/isolated-agent/delivery-dispatch.ts:752](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/cron/isolated-agent/delivery-dispatch.ts#L752)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Silent expected runs suppress streamed visible output upstream:** The agent runner skips partial/block/final streaming text entirely when `silentExpected` is set, and also suppresses reasoning-stream forwarding for those runs.
  - file: [src/auto-reply/reply/agent-runner-execution.ts:785](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/auto-reply/reply/agent-runner-execution.ts#L785)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Regression tests cover the reported behaviors:** Tests assert that direct cron delivery never sends bare `NO_REPLY`, trailing `NO_REPLY` summaries are suppressed, narrated silent-turn partials are dropped, and Telegram group sessions suppress exact `NO_REPLY`.
  - file: [src/cron/isolated-agent/delivery-dispatch.double-announce.test.ts:821](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/cron/isolated-agent/delivery-dispatch.double-announce.test.ts#L821)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
