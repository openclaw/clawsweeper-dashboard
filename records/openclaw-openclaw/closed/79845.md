## Summary

Keep this PR open, but it is not merge-ready: the new progress broadcasts use an event name current main denies by default, the completed-step progress bar is one step behind, and the contributor evidence does not show the real Control UI update flow.

## What This Changes

Adds gateway `update.progress` broadcasts during `update.run` and Control UI state/rendering for an inline update progress panel.

## Best Possible Solution

Land a typed, admin-scoped update progress event with broadcaster coverage, UI event/render tests, corrected progress math, and redacted real behavior proof from an actual update run.

## Reproduction Assessment

Yes for the PR defect: source inspection shows `update.progress` is not registered in `EVENT_SCOPE_GUARDS`, so current main drops those frames. The requested feature still lacks high-confidence real behavior proof from a gateway/browser update flow.

## Solution Assessment

No: the direction is reasonable, but this patch is not the best implementation until the event is explicitly admin-scoped, covered by tests, progress math is fixed, and the real UI behavior is shown.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Register the update progress broadcast event:** `src/gateway/server-methods/update.ts:75`
  - body: The new `context.broadcast("update.progress", ...)` calls use an event name that the gateway broadcaster drops because it is not in `EVENT_SCOPE_GUARDS` and unknown non-plugin events return false. As written, the Control UI listener will never receive these progress frames; add the event with the appropriate admin scope and coverage.
  - confidence: 0.94
- **[P3] Advance the completed-step progress width:** `ui/src/ui/app-render.ts:1589-1592`
  - body: `index` is documented as 0-based, but the progress bar width uses `index / total`. On `step-complete`, step 1 of 10 still renders 0% and every completed step is one behind; derive the completed count as `index + 1` for completed steps.
  - confidence: 0.86
