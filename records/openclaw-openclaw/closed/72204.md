## Summary

Current main already fixes the Windows Telegram/WhatsApp channel startup regression by keeping bundled plugin Jiti loads off the native ESM import path on win32. The reported v2026.4.24 behavior is confirmed by the release-tag source and has since been inverted on main with regression coverage; the related #71749/#72040/#72142 reports are the same root cause, with #71749 explicitly credited in the changelog fix entry.

## Best Possible Solution

Close #72204 as implemented on current main. Keep the current win32 non-native Jiti behavior for bundled plugin/channel runtime loading; users hitting the v2026.4.24 crash should update to a build containing the current main fix, and any new reproduction on that build should be filed with fresh Windows logs and exact version details.
