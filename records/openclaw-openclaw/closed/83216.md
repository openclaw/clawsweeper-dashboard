## Summary

Keep open. Current main still has a credible privacy-sensitive display gap: Telegram chat-window context is formatted as `Conversation context` lines, then the web display path does not clearly strip that sentinel or suppress hidden runtime-context entries before chat history reaches the browser.

## What This Changes

Hide injected Telegram or other non-web conversation context from the web gateway Chat UI instead of rendering it as visible chat history.

## Best Possible Solution

Treat conversation context as internal at the transcript/display boundary: keep or convert it to hidden runtime metadata, filter hidden/custom context from `chat.history` and `session.message`, and strip legacy textual `Conversation context` blocks from user-role display with regression tests.

## Reproduction Assessment

Yes for a source-level reproduction: construct a Telegram `chat_window` context, observe it formats as `Conversation context` lines, and pass it through the current web display stripper/projection path where that sentinel is not recognized. A live Telegram/browser run was not performed in this read-only review.

## Solution Assessment

Yes, keeping this open is the right outcome. The maintainable fix belongs in the shared transcript/display projection and metadata stripping path, not as a Telegram-only visual workaround.
