## Summary

Keep open. The bug-fix direction is plausible, but this PR cannot merge as-is because the new public SDK subpath is not wired through the SDK entrypoint manifest/build contracts, the SDK facade exposes global drain/test helpers that plugins should not be able to call during steady state, and the PR has no after-fix real gateway or Telegram restart proof.

## What This Changes

The branch adds a generic in-process restart hook registry, exposes it through `openclaw/plugin-sdk/lifecycle-restart-hooks`, runs it from the gateway restart loop, and registers Telegram polling leases for restart-boundary cleanup with regression tests.

## Best Possible Solution

Land a revised fix that keeps the restart drain core-owned, exposes only the minimal plugin registration seam if a public SDK subpath is needed, aligns the SDK entrypoint manifest/package exports/API contracts, and includes redacted real restart proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partly: the package/SDK review finding is source-reproducible from current-main contracts, and the Telegram lease leak has a clear source path, but I did not establish a live in-process gateway restart reproduction in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? Not yet: restart-boundary cleanup is the right general shape, but the PR should use the existing SDK manifest/build contracts and avoid making global drain/test helpers public plugin APIs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Add the SDK subpath to the entrypoint manifest:** `package.json:343-346`
  - body: The new `openclaw/plugin-sdk/lifecycle-restart-hooks` subpath is exported from `package.json` and imported by Telegram, but the plugin SDK build and contract tests derive public entrypoints from `scripts/lib/plugin-sdk-entrypoints.json`. Without adding the subpath there, the package export sync check fails and packaged builds can point at a `dist/plugin-sdk/lifecycle-restart-hooks.js` artifact that was never generated.
  - confidence: 0.93
- **[P2] Expose only registration through the SDK:** `src/plugin-sdk/lifecycle-restart-hooks.ts:6-10`
  - body: This public SDK facade lets plugin code call `runInProcessRestartHooks()` or `clearInProcessRestartHooksForTests()`, even though only the gateway restart boundary should drain global lifecycle cleanup and tests should not be a production plugin API. Please keep the run/clear helpers core-internal or test-only and export only the registration capability plugins need.
  - confidence: 0.84
