## Summary

Current `main` and the shipped `v2026.4.23` snapshot already cover the reported multi-provider Ollama routing path: native Ollama streams now receive the selected provider's configured `baseUrl`, and explicit Ollama hosts are allowlisted with private-network access for the fetch guard.

## Evidence

- **Selected Ollama provider baseUrl is passed into native stream creation:** The Ollama plugin's `createStreamFn` resolves the active provider config and forwards that provider's `baseUrl` into `createConfiguredOllamaStreamFn`, instead of hardwiring the default Ollama host.
