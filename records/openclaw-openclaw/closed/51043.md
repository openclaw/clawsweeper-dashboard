## Summary

Current main no longer rewrites configured fallback chains during live model switches. Model switches are stored as session-scoped overrides, fallback resolution still uses the configured chain, and the regression is covered by tests; the current repo also has no `freeride` command surface.

## Evidence

- **Session switches are session-scoped overrides, not config fallback rewrites:** `applyModelOverrideToSessionEntry` updates `providerOverride`, `modelOverride`, `modelOverrideSource`, and optional `liveModelSwitchPending` on the session entry; it does not edit `agents.defaults.model.fallbacks`.
  - file: [src/sessions/model-overrides.ts:10](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/sessions/model-overrides.ts#L10)
- **Session schema distinguishes user model switches from runtime fallbacks:** `SessionEntry` tracks `modelOverrideSource` and `liveModelSwitchPending`, with comments explicitly saying system-initiated fallbacks do not set the live-switch flag.
  - file: [src/config/sessions/types.ts:177](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/config/sessions/types.ts#L177)
- **Fallback resolution keeps configured chain after a session model override:** `resolveEffectiveModelFallbacks` returns the agent override or default fallback chain when `hasSessionModelOverride` is true, instead of replacing it with an auto-generated chain.
  - file: [src/agents/agent-scope.ts:166](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/agents/agent-scope.ts#L166)
- **Live switch path feeds preserved fallback chain into runtime failover:** `agent-command` computes `effectiveFallbacksOverride` and passes it to `runWithModelFallback` for each run attempt, including after `LiveSessionModelSwitchError` retries.
  - file: [src/agents/agent-command.ts:871](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/agents/agent-command.ts#L871)
- **Regression tests cover keeping the fallback chain on override and after a live switch:** `agent-scope.test.ts` asserts configured fallbacks remain available when `hasSessionModelOverride` is true, and `agent-command.live-model-switch.test.ts` asserts a switch flips fallback resolution from `(false, true)` rather than dropping the chain.
  - file: [src/agents/agent-command.live-model-switch.test.ts:518](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/agents/agent-command.live-model-switch.test.ts#L518)
- **Current repo no longer has a `freeride` command surface:** Repository search for `freeride` returned no matches on `main`.
  - command: `git grep -n "freeride" HEAD -- .`
