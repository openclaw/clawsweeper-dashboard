- **Changelog entry matches the reported symptom:** `CHANGELOG.md:2731` under the `2026.3.11` section says: Brave `llm-context` grounding snippets are treated as plain strings so `web_search` no longer returns empty snippet arrays in LLM Context mode.
  - file: [CHANGELOG.md:2731](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/CHANGELOG.md#L2731)
- **Current mapper preserves Brave snippet strings:** `mapBraveLlmContextResults` reads `grounding.generic` and keeps non-empty `entry.snippets` values instead of dropping them.
  - file: [extensions/brave/src/brave-web-search-provider.shared.ts:216](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/extensions/brave/src/brave-web-search-provider.shared.ts#L216)
- **Current runtime returns mapped snippets in tool output:** The `llm-context` branch calls `runBraveLlmContextSearch`, then writes `snippets: entry.snippets.map(...)` into each `web_search` result payload.
  - file: [extensions/brave/src/brave-web-search-provider.runtime.ts:307](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/extensions/brave/src/brave-web-search-provider.runtime.ts#L307)
