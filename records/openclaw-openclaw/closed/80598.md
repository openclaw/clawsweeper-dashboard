## Summary

Keep open: this active external PR targets a valid backup leak, but latest head still breaks signal cleanup, reports success on interruption, and can fail publishing when system temp and output are on different filesystems; the supplied proof does not show the interrupted CLI path after the latest push.

## What This Changes

The PR moves backup archive temp files into the staging temp directory, adds CLI SIGINT/SIGTERM cleanup plumbing, updates backup docs/changelog, and adds a temp-path regression test.

## Best Possible Solution

Land a corrected narrow backup fix that keeps active temp archives out of the output directory, exposes a real synchronous cleanup closure, preserves signal exit semantics, handles cross-filesystem publication, and proves the interrupted CLI path.

## Reproduction Assessment

Yes, source-backed: current main writes the active tarball beside `outputPath` and only removes it during normal JavaScript unwinding. Running `openclaw backup create` long enough to enter `tar.c`, then terminating it before unwinding, matches the orphaned `.tmp` report, though I did not live-run it in this read-only pass.

## Solution Assessment

No. Moving the temp archive into a staging temp root is directionally right, but this PR still needs corrected cleanup wiring, nonzero signal exit codes, cross-filesystem publish fallback, and real interrupted-run proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Pass the cleanup handler itself to onCleanup:** `src/infra/backup-create.ts:504`
  - body: `onCleanup` should receive the synchronous cleanup closure, but this passes a setter-shaped callback. The CLI stores that callback and calls it with no argument on SIGINT/SIGTERM, so it never removes `tempArchivePath` or `tempDir`; the imported `rmSync` also remains unused under `noUnusedLocals`.
  - confidence: 0.96
- **[P2] Preserve signal-derived exit codes:** `src/commands/backup.ts:30`
  - body: The backup signal handler exits with `0` for both SIGINT and SIGTERM, so an interrupted backup can look successful to shell scripts or supervisors even when no archive completed. Use the usual signal-derived codes, matching the existing CLI shutdown path: 130 for SIGINT and 143 for SIGTERM.
  - confidence: 0.88
- **[P2] Fallback to copy when hard links cross filesystems:** `src/infra/backup-create.ts:499`
  - body: After this change, the temp archive can live under `os.tmpdir()` while the requested output is on another mount. `fs.link` fails with `EXDEV` across filesystems, but the existing fallback only handles ENOTSUP/EOPNOTSUPP/EPERM, so these backups fail instead of using the exclusive copy fallback.
  - confidence: 0.89
