## Summary

Keep open. Current main still has the plausible Google streaming liveness gap, and an open unmerged PR (#76080) explicitly closes this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to stop timing out Gemini 3.1 Pro Preview agent runs when Google streams thinking/signature-only activity before visible output.

## Best Possible Solution

Review and land #76080 or an equivalent Google-provider liveness fix that treats non-visible Gemini thinking/signature SSE chunks as model activity without exposing hidden thinking content, then close this issue after merge and validation.

## Reproduction Assessment

Unclear. The report gives detailed production steps and a direct-API control, but a high-confidence repo reproduction should feed signature-only Gemini SSE chunks through the Google transport under streamWithIdleTimeout.

## Solution Assessment

Yes in direction, but not complete on main. A provider-local liveness event or chunk-activity signal is narrower than changing global timeout semantics, and the open PR targets that boundary.
