## Summary

Current `main` no longer relies on session transcript persistence for outbound replay in the way described by #60110. Outbound sends now use a dedicated write-ahead delivery queue with atomic ack cleanup, startup recovery re-reads queue state before replaying, and regression tests cover crash recovery and duplicate-send suppression. The changelog shows the delivery-queue recovery work shipped in `v2026.4.14`.

## Evidence
