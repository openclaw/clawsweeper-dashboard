## Summary

Current `main` already makes `/status` derive its denominator from the active provider/model context window rather than only the global session/default `contextTokens`, with regression coverage for model-switch and fallback cases.

## Evidence

- **Status recomputes displayed context from selected/active model:** `buildStatusMessage` resolves both selected and active model context windows, then chooses the active/provider-specific limit for the rendered `Context:` line instead of blindly echoing the stored/global token cap.
  - file: [src/status/status-message.ts:594](https://github.com/openclaw/openclaw/blob/f04a3dced01bba230390f570f2369e3f2b63b3e7/src/status/status-message.ts#L594)

- **Provider-qualified model lookup prefers per-model config:** `resolveContextTokensForModel` directly scans `models.providers[provider].models[].contextWindow/contextTokens` and prefers provider-qualified cache keys so `/status` gets the correct per-model window.
  - file: [src/agents/context.ts:435](https://github.com/openclaw/openclaw/blob/f04a3dced01bba230390f570f2369e3f2b63b3e7/src/agents/context.ts#L435)
