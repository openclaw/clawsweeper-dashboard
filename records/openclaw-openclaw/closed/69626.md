## Summary

Current `main` already fixes the isolated-heartbeat exec-event mismatch the report describes, and it now has regression coverage for both the isolated-session routing path and the empty-output prompt fallback. The latest release in the provided context predates that `main` change, so this looks fixed on `main` but not yet shipped in `v2026.4.23`.

## Evidence

- **Heartbeat preflight now guards isolated wake inspection against queue/session mismatches:** `src/infra/heartbeat-runner.ts:564` peeks pending events from the resolved session, and `src/infra/heartbeat-runner.ts:571-585` only treats wake-time pending events as inspectable for isolated heartbeats when the active queue is the same isolated `:heartbeat` session. That directly addresses the parent-session vs isolated-session mismatch described in the issue.
  - file: [src/infra/heartbeat-runner.ts:564](https://github.com/openclaw/openclaw/blob/07cf1dd65c937c75cf48a1645ea580404f5056b4/src/infra/heartbeat-runner.ts#L564)
  - command: `nl -ba src/infra/heartbeat-runner.ts | sed -n '560,760p'`
  - sha: [07cf1dd65c93](https://github.com/openclaw/openclaw/commit/07cf1dd65c937c75cf48a1645ea580404f5056b4)
- **Exec-event prompts no longer rely on "system messages above":** `src/infra/heartbeat-events-filter.ts:43-71` embeds the exec completion details directly into the heartbeat prompt and falls back to `HEARTBEAT_OK` when no command output exists, instead of emitting the old 'shown in the system messages above' wording that produced the confusion in the report.
  - file: [src/infra/heartbeat-events-filter.ts:43](https://github.com/openclaw/openclaw/blob/07cf1dd65c937c75cf48a1645ea580404f5056b4/src/infra/heartbeat-events-filter.ts#L43)

- **Regression tests cover the isolated-session exec-event path:** `src/infra/heartbeat-runner.isolated-key-stability.test.ts:295-337` explicitly verifies that `hook:wake` exec events queued on the active isolated `:heartbeat` session are classified as `exec-event` and run with the isolated session key. This is the same routing class the report calls out.
  - file: [src/infra/heartbeat-runner.isolated-key-stability.test.ts:295](https://github.com/openclaw/openclaw/blob/07cf1dd65c937c75cf48a1645ea580404f5056b4/src/infra/heartbeat-runner.isolated-key-stability.test.ts#L295)

- **Prompt tests cover the empty-output fallback:** `src/infra/heartbeat-events-filter.test.ts:72-76` asserts that empty exec-completion prompts say 'no command output was found' and 'Reply HEARTBEAT_OK only', and explicitly do not contain 'system messages above'.
  - file: [src/infra/heartbeat-events-filter.test.ts:72](https://github.com/openclaw/openclaw/blob/07cf1dd65c937c75cf48a1645ea580404f5056b4/src/infra/heartbeat-events-filter.test.ts#L72)
