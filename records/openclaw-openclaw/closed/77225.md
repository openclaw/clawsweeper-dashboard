Full review comments:

- **[P1] Route title generation through provider runtime:** `src/gateway/session-auto-title.ts:239-248`
  - body: `generateAiTitleWithFetch` posts transcript excerpts directly to an OpenAI-compatible endpoint with an ad hoc bearer header.   - confidence: 0.94
- **[P2] Preserve session-scoped model and auth selection:** `src/gateway/session-auto-title.ts:174-184`
  - body: `resolveTitleProviderConfig` only uses the global default model/provider and environment key. Sessions using agent defaults, session model overrides, runtime-selected models, aliases, or auth profiles can generate titles with the wrong backend or fail even though the session itself works.
  - confidence: 0.9
- **[P2] Notify subscribers after persisting generated titles:** `src/gateway/server-chat.ts:382`
  - body: This starts title generation in the background, then immediately sends the normal `sessions.changed` event before `aiTitle` can be persisted. There is no later post-persist event, and the UI event merge whitelist still lacks `aiTitle`, so the live sidebar will not update until a reload or unrelated event.
  - confidence: 0.88
