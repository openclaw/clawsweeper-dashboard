## Summary

Current `main` already addresses this Discord gateway crash path. The Discord monitor now wraps gateway errors in a supervisor that classifies `"Max reconnect attempts"` failures, buffers early errors, and suppresses late teardown/dispose errors instead of letting them crash the process; startup cleanup also disposes that supervisor, and the gateway plugin now sets `reconnect.maxAttempts` to `50` rather than `0`. Targeted tests on `main` cover the exact `Max reconnect attempts (0)` error string and assert it does not throw during teardown/startup-failure cleanup.

## Evidence

- **Gateway plugin no longer uses zero reconnect attempts:** `createDiscordGatewayPlugin` builds the Discord gateway with `reconnect: { maxAttempts: 50 }`, so the reported `maxAttempts=0` default path is not the current OpenClaw configuration.

  - file: [extensions/discord/src/monitor/gateway-supervisor.ts:89](https://github.com/openclaw/openclaw/blob/4d1ee3a73ed19063d16ac627d3071b7637fda82d/extensions/discord/src/monitor/gateway-supervisor.ts#L89)
