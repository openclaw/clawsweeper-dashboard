## Summary

Current main already fixes the user-visible duplicate `/new` verbose session notice by removing the separate standalone reset notice for reply-producing reset turns and handling bare `/new`/`/reset` as an in-band startup prompt. PR #55543's `resetNoticeDelivered` plumbing is no longer needed, and its review concern about suppressing all notices when delivery fails is avoided because agent-runner still emits the normal verbose `🧭 New session` notice.

## Best Possible Solution

Close PR #55543 as implemented on main; keep the shipped in-band bare reset flow and regression coverage instead of merging the older `resetNoticeDelivered` plumbing.
