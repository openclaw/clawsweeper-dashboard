## Summary

Keep #50003 open. Current main still does not provide a non-interactive `openclaw onboard` path for importing a GitHub Copilot token: the Copilot provider only exposes the interactive device flow, the bundled manifest does not advertise `--github-copilot-token`, and the non-interactive plugin-auth dispatcher explicitly errors when a provider auth method lacks `runNonInteractive`. The PR is stale/conflicted and needs rebase work, but the central feature remains unimplemented rather than obsolete or out of scope.

## Best Possible Solution

Keep the PR open, but ask for a current-main rebase that drops obsolete generated metadata changes and implements the feature through today’s surfaces: add Copilot flag metadata to `extensions/github-copilot/openclaw.plugin.json`, add a `runNonInteractive` implementation to the GitHub Copilot provider auth method, preserve/refine token normalization and env fallback behavior, set the current Copilot default model consistently with interactive auth, and add focused non-interactive onboard tests for explicit flag, inferred auth choice, env fallback, missing token, existing-profile reuse, and config/default-model behavior.
