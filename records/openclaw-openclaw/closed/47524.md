## Summary

Current main already blocks the reported ambiguous multi-channel `delivery.mode="announce"` cron configuration at `cron.add`/`cron.update`, and current cron surfaces now expose unresolved delivery routes and delivery-specific status instead of leaving the condition hidden behind only `lastError`.

## Evidence

- **Validation rejects ambiguous multi-channel announce delivery:** `assertConfiguredAnnounceChannel` throws `delivery.channel is required when multiple channels are configured...`, and both `cron.add` and `cron.update` call that validation before persisting the job.
  - file: [src/gateway/server-methods/cron.ts:40](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/gateway/server-methods/cron.ts#L40)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Regression coverage for add/update:** Gateway tests configure Telegram + Slack and assert that ambiguous `delivery: { mode: "announce" }` is rejected on both add and update with an error containing `delivery.channel is required`.
  - file: [src/gateway/server-methods/cron.validation.test.ts:82](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/gateway/server-methods/cron.validation.test.ts#L82)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Unresolved routes are surfaced in cron list/show:** `cron.list` attaches `deliveryPreviews`, and unresolved `channel: "last"` previews explicitly report `will fail-closed` instead of hiding the routing problem.
  - file: [src/cron/delivery-preview.ts:23](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/cron/delivery-preview.ts#L23)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Delivery outcome is tracked separately from run outcome:** Cron state now includes `lastDeliveryStatus` and `lastDeliveryError`, and tests assert `not-delivered` plus preserved delivery error text for failed delivery attempts.
  - file: [src/cron/types.ts:160](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/cron/types.ts#L160)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
