## Summary

Keep open: current main and the latest release still expose `sessions.compact` only through raw Gateway/SDK paths, while the `openclaw sessions` CLI/docs do not provide a first-class compact command and the CLI `/compact` workaround is not explicitly rejected with guidance.

## What This Changes

The issue requests a documented `openclaw sessions compact <session-key>` command and clear non-zero handling for CLI-originated `/compact` messages.

## Best Possible Solution

Add a thin documented sessions CLI command backed by the existing `sessions.compact` operator-admin RPC, and make CLI-originated `/compact` fail clearly with that command as the recovery path.

## Reproduction Assessment

Yes, source-reproducible: current source shows no `openclaw sessions compact` command or CLI docs entry, while the raw `sessions.compact` RPC exists. The CLI slash workaround path is also traceable in source: Gateway `agent` only special-cases `/new` and `/reset`, and `/compact` is gated as an authorized chat command rather than a CLI maintenance command.

## Solution Assessment

Yes: a first-class CLI wrapper over the existing RPC plus an explicit `/compact` CLI rejection is the narrowest maintainable fix. Executing slash commands through `agent --message` would be a riskier auth-boundary change.
