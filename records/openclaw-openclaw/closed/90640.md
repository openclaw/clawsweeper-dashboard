## What This Changes

The issue requests a documented `openclaw sessions compact <session-key>` CLI surface and a non-zero diagnostic when `openclaw agent --message '/compact ...'` cannot execute the slash command.

## Best Possible Solution

Land or replace the open linked PR so the supported recovery path is a documented `sessions compact` command and CLI `/compact` fails loudly instead of giving false success.
