## Summary

Close #69931 as a duplicate of the broader open tracker #11460. The requested ability to read/query WhatsApp reactions on specific or historical messages is already covered by #11460, and open PR #46881 is a partial implementation candidate for forwarding WhatsApp reaction events to the agent. Current main still has only outbound/ack WhatsApp reaction support, so this should not close as implemented or ClawHub-only.

## Best Possible Solution

Close #69931 as a duplicate of #11460. Keep the actual product/API work tracked on #11460, with #46881 as a partial implementation candidate: add WhatsApp reaction event ingestion, expose a documented reaction read/query surface with target message id, reacting actor, emoji, account/chat/session correlation, and decide whether historical retrieval comes from direct provider queries, local persistence, or both.
