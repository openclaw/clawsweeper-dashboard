## Summary

Current main already resolves the reported old-release streaming failure: the OpenAI-compatible chat endpoint flushes an initial SSE chunk before the agent run settles, and embedded pi text deltas are emitted onto the same assistant event stream the endpoint forwards.

## What This Changes

The issue asks for true SSE streaming from `/v1/chat/completions` when OpenAI-compatible requests run through embedded pi-ai providers instead of CLI or ACP backends.

## Best Possible Solution

Keep the current-main generic event-bus implementation and let it ship in the next release; reopen only with a fresh main build reproduction showing embedded provider deltas still buffering.

## Reproduction Assessment

No, not as a current-main failure. The supplied curl is concrete for the older released image, but current main source and tests show an early SSE role chunk plus embedded `text_delta` assistant events before final fallback buffering.

## Solution Assessment

Yes. The current implementation fixes the shared boundary by emitting embedded assistant deltas onto the existing agent event bus and letting the OpenAI-compatible endpoint forward that stream, without adding provider-specific HTTP behavior.
