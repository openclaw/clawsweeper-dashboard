## Summary

Close as duplicate/superseded by the older open #29132, which tracks the same remaining public pre-model tool-result hook gap. Current main still does not implement the requested public hook; the remaining product/API decision should stay consolidated in #29132 and the open implementation candidate #64907 rather than in a second parallel issue.

## Best Possible Solution

Close this duplicate and keep the remaining API/product decision in #29132. Review #64907, or a successor PR, against that canonical issue and decide whether the public hook should be synchronous, async, text-only, arbitrary-result, or remain bundled-only behind the existing trusted middleware.
