## Summary

Keep open: current main and v2026.5.27 still derive the sidebar Recent list from loaded session rows without a current-agent filter, while the chat session picker already builds an agent-scoped sessions.list request.

## What This Changes

Request the Control UI Dashboard sidebar Recent sessions list to show only sessions for the currently selected agent, matching the top-bar session picker.

## Best Possible Solution

Filter the sidebar Recent list by the selected agent using the existing session-key/agent scoping helpers or an agent-scoped sessions load, and add a mixed-agent sidebar regression test; leave an optional All agents toggle for a separate product choice.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: populate sessionsResult with mixed-agent direct rows and render the sidebar while one agent is selected; the current resolver has no selected-agent predicate. I did not run a browser test in this read-only review.

## Solution Assessment

Yes, the narrow maintainable path is to reuse the existing agent scoping already used by the chat picker/session APIs and cover mixed-agent rows in the sidebar test. Adding an All agents toggle is optional and should not block the core filter fix.
