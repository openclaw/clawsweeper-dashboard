## Summary

Keep open: current main still drops the silent-empty opt-in before the embedded cron runner, and https://github.com/openclaw/openclaw/pull/81374 is an open implementation candidate that references this issue for closure.

## What This Changes

The issue reports that isolated cron jobs with `delivery.mode: "none"` do not forward `allowEmptyAssistantReplyAsSilent` into `runEmbeddedPiAgent`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81374 or an equivalent narrow fix that forwards the resolved `deliveryPlan.mode === "none"` state into `runEmbeddedPiAgent` with regression coverage.

## Reproduction Assessment

Yes, source-level. Current main resolves `delivery.mode: "none"`, but the embedded cron call does not pass the silent-empty opt-in and the embedded classifier treats an omitted value as false.

## Solution Assessment

Yes for the implementation direction. Carrying the already-resolved cron no-delivery state to the embedded runner is narrower than re-reading raw job config at the call site or relaxing incomplete-turn classification globally.
