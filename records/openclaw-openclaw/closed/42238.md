## Summary

Current `main` implements a generic fix for channel-session image sends on text-only models: inbound images are offloaded to `media://inbound/*` refs instead of being sent inline, those refs are reloaded during prompt assembly, and the Codex media path now rejects text-only app-server models before starting an image turn.

## Close Comment

Closing as implemented on `main` after Codex review. Current `main` now checks the effective session model before parsing inbound attachments (`src/gateway/server-methods/chat.ts:2204`) and, for text-only channel/session models, offloads image uploads to `media://inbound/*` refs instead of sending inline image payloads that can fail at the provider boundary (`src/gateway/chat-attachments.ts:376`). The embedded runner resolves those refs back into image content during prompt assembly (`src/agents/pi-embedded-runner/run/images.ts:356`), and there is direct regression coverage for the text-only channel-session path in `src/gateway/server-methods/chat.directive-tags.test.ts:2074` and `src/gateway/chat-attachments.test.ts:146`. The Codex media path also now preflights `model/list` and rejects text-only app-server models before starting an image turn (`extensions/codex/media-understanding-provider.ts:169`, `extensions/codex/media-understanding-provider.test.ts:222`), which avoids the mid-turn `server_error` pattern reported here. This appears fixed on current `main` via commit `4737a86071e35c409b0af9d7849bad05d7cb387f`. No tagged release contains that commit yet, so `fixedRelease` is unknown/not shipped at review time.
