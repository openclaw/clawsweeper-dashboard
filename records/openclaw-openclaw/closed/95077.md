## Summary

Keep open: the PR is a useful narrow mitigation for the linked local TUI command fallthrough, but it is not merge-ready because it only handles bare command names, misses shared colon-form inputs such as `/compact: focus`, and still lacks real after-fix local TUI proof.

## What This Changes

The branch adds `status` and `compact` cases to the TUI command switch so local mode prints an unsupported-command system message while gateway mode forwards the raw command.

## Best Possible Solution

Normalize local TUI command parsing against the shared slash-command contract, cover bare and colon-form `/status` and `/compact` with focused tests or local PTY proof, and require redacted real local TUI evidence before merge.

## Reproduction Assessment

Yes. From source, slash input enters `handleCommand`, `parseCommand` splits only on whitespace, unhandled names hit `sendMessage(raw)`, and local `sendChat` reaches `agentCommandFromIngress`; I did not run artifact-producing PTY tests in this read-only pass.

## Solution Assessment

No. Explicit switch cases are a plausible partial mitigation, but the best fix should normalize the TUI command token or reuse shared normalization so colon-form shared commands cannot continue into the model path.
