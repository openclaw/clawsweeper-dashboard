Keep open. The PR targets a valid current-main transcript-state hardening gap, but PR head still lets a malformed duplicate id poison a previously accepted entry, so later valid children can be detached from the accepted parent chain.

## What This Changes

The PR adds TranscriptFileState JSONL entry validation/repair, regression tests, and a changelog entry for malformed persisted transcript rows.

## Best Possible Solution

Keep the validation hardening, but make rejected-id bookkeeping occurrence-safe and add duplicate-id regression coverage before merge.

## Solution Assessment

No. TranscriptFileState is the right layer, but PR head should not let a malformed duplicate occurrence mark an already accepted id as rejected.
