## Summary

Current main addresses the reported orphaning path: embedded session transcripts are now resolved through the session store before run execution, and later maintenance archives or prunes removed/unreferenced artifacts through the existing session cleanup flow. The fix is present on main but I could not prove it has shipped in a release tag yet.

## What This Changes

The issue asks for failed or interrupted agent runs to avoid leaving partial session transcript JSONL files outside the agent's `sessions.json` index.

## Best Possible Solution

Keep the current main behavior: persist every active transcript path through the session store, archive transcripts when entries are removed, and use `sessions cleanup` only for older historical unreferenced artifacts.

## Reproduction Assessment

No, I did not run a live Anthropic/API-failure reproduction in this read-only review. Source inspection of current main shows the reported orphan path is no longer expected for embedded runs because the transcript path is persisted before the run opens the session file.

## Solution Assessment

Yes. Persisting the transcript path in `sessions.json` before execution is narrower and safer than running broad cleanup on every failure, because interrupted transcripts remain recoverable while existing maintenance handles stale or removed entries.
