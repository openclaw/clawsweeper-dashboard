## Summary

Keep this PR open: the nested-metadata bug is valid and the patch is targeted, but it still needs after-fix runtime proof and a maintainer/security decision on parsing approval wrappers from raw exec output.

## What This Changes

This PR replaces regex-based exec approval denied/finished parsing with balanced metadata parsing, adjusts emitted denial metadata, broadens sandbox-blocked classification, and adds parser/followup tests plus a changelog entry.

## Best Possible Solution

Land the nested-metadata fix after approval result classification is tied to trusted structured lifecycle data, with spoof-guard regression coverage and redacted runtime proof of the denied followup path.

## Reproduction Assessment

Yes, at source level: current main emits `approval-timeout (allowlist-miss)` and the current `[^)]*` parser cannot match that nested metadata as a denied result. I did not run a live gateway flow because this review is read-only.

## Solution Assessment

Not yet. Balanced parsing fixes the nested metadata symptom, but the merge-safe version should source approval-result classification from trusted structured lifecycle data or an explicitly accepted marker before merge.
