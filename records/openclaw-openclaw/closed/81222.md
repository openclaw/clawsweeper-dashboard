## Summary

Close as duplicate/superseded: current main intentionally keeps orphan transcript archiving interactive-only, and the remaining operator need for audited orphan cleanup is already tracked by https://github.com/openclaw/openclaw/issues/77941.

## What This Changes

Request that `openclaw doctor --fix --yes` archive orphan transcript files non-interactively and optionally include related trajectory side-files.

## Best Possible Solution

Keep the remaining design work on https://github.com/openclaw/openclaw/issues/77941 and define an explicit audited cleanup/archive path instead of weakening doctor’s interactive safety gate.

## Reproduction Assessment

Yes. Source inspection and existing regression coverage show that orphan transcript archival requires an interactive confirmation and non-interactive repair mode leaves the file untouched; I did not run a write-path doctor invocation because this review is read-only.

## Solution Assessment

No. Making `--yes` override `requiresInteractiveConfirmation` would reverse the documented safety policy introduced for https://github.com/openclaw/openclaw/issues/73106; the safer path is the canonical audited cleanup design in https://github.com/openclaw/openclaw/issues/77941.
