## Best Possible Solution

Land an agent-aware Recent/session-picker fix only after maintainers accept the scoped `sessions.list` contract or narrow the change to sidebar-specific data while preserving existing broad session surfaces.

## Solution Assessment

Unclear as a merge-ready path. Filtering Recent by selected agent is the right end state, but reusing shared `sessionsResult` by making several refresh paths agent-scoped is compatibility-sensitive; a narrower sidebar data path or explicit maintainer acceptance would be safer.
