## Summary

Current `main` already prevents Discord gateway `error` events from crashing the whole process via the newer gateway-supervisor flow, so this PR's safety-net listener is superseded.

## Evidence

- **Early gateway error listener now exists before lifecycle attach:** `createDiscordGatewaySupervisor()` installs `emitter.on("error", onGatewayError)` and buffers events by phase, so unhandled gateway `error` events are intercepted instead of reaching Node's default uncaught-emitter path.
