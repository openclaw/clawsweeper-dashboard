## Summary

## What This Changes

## Best Possible Solution

Yes. The current implementation separates run context (`session`/`isolatedSession`) from delivery (`target`/`to`) and provides `heartbeat_respond` for structured notify/no-notify behavior, which is the maintainable framework-level shape for this request.
