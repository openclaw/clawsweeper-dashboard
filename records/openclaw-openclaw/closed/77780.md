## Summary

Keep open: current main and the latest shipped release still use a 20k compaction reserve floor, below the requested 25k-30k headroom, and the reported long-session transcript-tail summary failure has not been live-reproduced or otherwise proven fixed.

## What This Changes

The issue requests raising the default compaction reserve headroom and optionally warning when compaction produces a transcript-tail summary instead of structured state.

## Best Possible Solution

Validate summary quality on realistic 200k-token sessions, choose a higher absolute floor or relative reserve policy, update source/tests/docs together, and treat transcript-tail detection as a follow-up observability guard if needed.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection confirms the 20k floor and roughly 180k-token trigger point on a 200k window, but not the exact transcript-tail summary failure on current main.

## Solution Assessment

Unclear. Raising the floor to 25k-30k is plausible, but the exact default or relative-reserve policy should be chosen after validating summary quality and compaction frequency on realistic long sessions.
