## Summary

Current main already supports custom `models.providers.<id>` entries for OpenAI-compatible backends, request-level auth header overrides that cover Basic auth, and opt-in private-network access for trusted self-hosted model endpoints. The requested dedicated `owl` provider is not necessary for the underlying capability, and these surfaces were already present in release `v2026.4.23`.

## Evidence

- **Custom provider entries are already a first-class config surface:** Docs describe `models.providers` as a custom provider map keyed by provider id, with configurable `api`, `baseUrl`, `headers`, `request.auth`, `request.proxy`, `request.tls`, and `request.allowPrivateNetwork`.
  - file: [docs/gateway/config-tools.md:401](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/docs/gateway/config-tools.md#L401)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Basic auth is covered by generic header auth overrides:** Provider request auth supports `mode: "header"` plus `headerName`, `value`, and optional `prefix`; the resolver concatenates `prefix + value`, so `Authorization` with prefix `Basic ` is supported without adding a dedicated provider.
  - file: [src/agents/provider-request-config.ts:452](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/provider-request-config.ts#L452)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **The config schema includes private-network opt-in for model providers:** `ConfiguredModelProviderRequest` includes `allowPrivateNetwork`, and schema help documents it as the operator opt-in for trusted self-hosted OpenAI-compatible endpoints whose DNS resolves to private/CGNAT ranges.
  - file: [src/config/types.provider-request.ts:45](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/config/types.provider-request.ts#L45)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Self-hosted OpenAI-compatible backends are already documented:** Docs explicitly describe custom/self-hosted OpenAI-compatible setups, including `inferrs` as a custom backend and the broader local-model guidance for custom OpenAI endpoints.
  - file: [docs/providers/inferrs.md:10](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/docs/providers/inferrs.md#L10)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **The capability shipped in the latest release:** The same request-auth and self-hosted OpenAI-compatible docs/config surfaces are present in tag `v2026.4.23`, confirming this was already shipped.
