## Summary

Keep this PR open. Current main still does not implement the PR's intended behavior: unset `requireMention` still falls back to `true`, and Telegram falls through to that shared resolver after topic/group-specific overrides. The linked bug report is closed, but this PR is unmerged and the observable behavior is still missing on main.

## Best Possible Solution

Keep the PR open and either rebase/land it or port its core idea onto current main: make the shared group mention resolver aware of the resolved group policy mode so `groupPolicy: "open"` defaults unset `requireMention` to false, while `allowlist`/unset policy still defaults true. Keep the PR's intended regression coverage in both the shared config tests and Telegram path, plus a docs/changelog update with proper attribution.
