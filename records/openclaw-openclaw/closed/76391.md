Current main now repairs configured external channel plugin installs from npm during `openclaw doctor --fix` before stale cleanup, with npm-first catalog entries for Discord and Zalo plus regression coverage. The fix is on `main` after `v2026.5.2` and is not yet proven shipped in a release tag.

## What This Changes

The issue asks for an upgrade migration or fallback so existing Discord and Zalo channel configs continue working after the 2026.5.2 external-plugin cutover.

## Best Possible Solution

Keep the npm-first doctor repair as the migration path and ship it in the next patch release so configured Discord/Zalo installs recover without relying on ClawHub package availability.

## Reproduction Assessment

Yes, by source inspection: a packaged install with configured `channels.discord` or `channels.zalo` and missing external plugin payloads follows the stale-plugin path reported in the issue, and current main now repairs those configured channel plugins from npm before stale cleanup. ## Solution Assessment

Yes. The current main fix is a narrow maintainable repair path: keep Discord/Zalo externalized, use official npm-first catalog metadata, install missing configured plugin payloads during doctor repair, and leave explicit ClawHub installs to ClawHub artifact readiness.
