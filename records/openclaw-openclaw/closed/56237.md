## Summary

Current main already avoids passing unsupported geometry overrides for image edits, and the MiniMax image provider only sends edit-safe fields. The changelog shows this shipped in release v2026.4.5.

## Evidence

- **Tool stops inferring resolution for edit flows that do not support it:** `image_generate` now leaves `resolution` undefined when a size was supplied or the selected edit capability reports `supportsResolution === false`, which covers MiniMax edit mode.
  - file: [src/agents/tools/image-generate-tool.ts:640](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/agents/tools/image-generate-tool.ts#L640)

  - sha: [7d1891e6e63d](https://github.com/openclaw/openclaw/commit/7d1891e6e63dc35dd9dcb688ba308049e68fe695)
- **Normalization drops unsupported size/resolution overrides in edit mode:** The shared image-generation normalization layer translates unsupported `size` to aspect ratio when possible and otherwise records it as ignored; unsupported `resolution` is always dropped instead of reaching the provider request.
  - file: [src/image-generation/normalization.ts:74](https://github.com/openclaw/openclaw/blob/7d1891e6e63dc35dd9dcb688ba308049e68fe695/src/image-generation/normalization.ts#L74)

## Close Comment

Closing as implemented. Codex review against current `main` found that `image_generate` now suppresses unsupported edit-mode geometry overrides before provider dispatch (`src/agents/tools/image-generate-tool.ts:640`, `src/image-generation/normalization.ts:74`), and the MiniMax provider itself declares edit `supportsResolution: false` / `supportsSize: false` and only sends `aspect_ratio` plus `subject_reference` in the API body (`extensions/minimax/image-generation-provider.ts:66`). There is also regression coverage asserting edit calls do not forward inferred resolution when edit-mode resolution is unsupported (`src/agents/tools/image-generate-tool.test.ts:781`). This behavior was already shipped per the v2026.4.5 changelog entry noting that OpenClaw stopped inferring unsupported resolution overrides for image edits.
