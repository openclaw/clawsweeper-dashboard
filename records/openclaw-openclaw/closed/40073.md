## Summary

Current `main` already implements the requested behavior: the gateway health monitor only treats a connected channel as a stale socket when explicit transport-liveness data exists, and Discord does not publish transport-liveness timestamps for ordinary inbound-message idleness. The requested health-monitor disable override is also present on `main` for Discord at both channel and account scope.

## Evidence

- **Health policy no longer treats idle app events as socket failure:** `evaluateChannelHealth` only enters stale-socket logic when `connected === true` and `lastTransportActivityAt != null`; otherwise stale app-event age alone is ignored.
