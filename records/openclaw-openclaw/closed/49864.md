## Summary

Keep open. Current main and v2026.5.3 still resolve Telegram persisted activation before explicit topic `requireMention`, while the docs promise topic overrides, and linked PR #49980 is still the open implementation candidate.

## What This Changes

The issue requests that Telegram forum-topic `requireMention: false` override group-level mention requirements and persisted activation state for that topic.

## Best Possible Solution

Telegram inbound handling should make an explicit topic `requireMention` value outrank persisted activation and group defaults, with regression coverage for topic-over-activation, topic-over-group, and the no-activation thread-id path discussed in comments.

## Reproduction Assessment

Yes for the activation-precedence path: source inspection shows `groupActivation: "mention"` resolves to `true`, wins before topic `requireMention: false`, and then reaches the `no-mention` skip. The no-activation variant is unclear from current source because the existing test suite already covers the simple topic-over-group case without activation.

## Solution Assessment

Yes: changing the local Telegram precedence so explicit topic config wins is the narrowest maintainable fix for the confirmed bug. PR #49980 appears to be the right implementation candidate, with final review needed for the no-activation commenter’s path.
