## Summary

Current `main` no longer gives the health monitor a path to restart or emit events for channels/accounts removed from config. The runtime snapshot is rebuilt from the live config on each check, the monitor iterates only that snapshot, and health-monitor config keys now exist for global and per-channel/account opt-out.

## Evidence

- **Health monitor only scans current runtime snapshot:** `startChannelHealthMonitor()` calls `channelManager.getRuntimeSnapshot()` and iterates `snapshot.channelAccounts`; it does not walk any persistent stale registry.
  - file: [src/gateway/channel-health-monitor.ts:111](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/src/gateway/channel-health-monitor.ts#L111)
