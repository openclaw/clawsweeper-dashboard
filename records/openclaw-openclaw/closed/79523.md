## Summary

Close as implemented on current main: WhatsApp now preserves the raw user message for command parsing while keeping the wrapped/enveloped body for agent context, and regression coverage exercises wrapped WhatsApp group `/status` extraction. This appears main-only after `v2026.5.7`.

## What This Changes

The issue reports that WhatsApp group text commands such as `/status` and `/commands` should trigger from the raw user-authored message even when group context prefixes are added for the prompt.

## Best Possible Solution

Keep the current main behavior that separates raw command text from prompt/envelope context, and include it in the next release.

## Reproduction Assessment

No current-main failure was reproduced. Source and regression-test inspection show current main already routes a wrapped WhatsApp group message through raw `/status` command extraction.

## Solution Assessment

Yes. Preserving raw `CommandBody`/`BodyForCommands` while leaving the enveloped body for agent context is the narrow maintainable fix for this report.
