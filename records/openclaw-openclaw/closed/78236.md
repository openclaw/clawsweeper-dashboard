## Summary

The reported validation failure does not match current `main` or the published v2026.5.4 package: `plugins.bundledDiscovery` is a documented, schema-backed config key and doctor intentionally writes it for legacy allowlist migration.

## What This Changes

The issue reports that upgrading to v2026.5.4 leaves `plugins.bundledDiscovery` in config and causes config validation or Gateway startup to fail instead of validating or being repaired by doctor.

## Best Possible Solution

Keep `plugins.bundledDiscovery` as the documented compatibility setting; any remaining report should provide exact active binary path, package version, config validation output, and Gateway logs showing a package/source mismatch.

## Reproduction Assessment

No. Current main and the published v2026.5.4 npm tarball both include `plugins.bundledDiscovery` in the strict config schema, so the reported `Unrecognized key` path does not reproduce from source/package inspection.

## Solution Assessment

No patch is applicable. Removing the key would be the wrong fix because current docs, schema, and doctor migration define it as the compatibility setting; a fresh report should focus on stale install or package mismatch proof if the error persists.
