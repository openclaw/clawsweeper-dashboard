## Summary

Keep open: this is a useful but not merge-ready feature because the new backend option is missing from the strict config/schema surfaces, and the raw-transcript replay behavior needs explicit maintainer security/product approval.

## What This Changes

The PR adds a default-off CLI backend option to reseed fresh CLI sessions from raw transcript history when no usable compaction summary exists, plus tests for that opt-in path.

## Best Possible Solution

If maintainers accept the safety tradeoff, complete the option as a full config contract with zod schema, generated schema/help/docs coverage, validation tests, and a changelog entry while keeping the default off.

## Reproduction Assessment

Yes for the PR blocker: source inspection shows the proposed config key is absent from the strict `CliBackendSchema` and generated schema. A focused reproduction would be validating the PR's example config against the schema.

## Solution Assessment

No. The runtime fallback is narrow, but a user-facing backend option must be wired through the config/schema contract and the raw transcript replay policy should be accepted before merge.
