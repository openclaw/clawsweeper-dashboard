## Summary

The PR targets a Teams file-send regression reported in #60003, but the reporter closed that linked bug after identifying the root cause as the external Teams app manifest, not OpenClaw. Current main already documents the required `bots[].supportsFiles: true` manifest field and contains the Teams file-send implementation paths; the PR is therefore a speculative source change for an external setup problem, and its diff has unresolved functional review concerns.

## Best Possible Solution

Close this PR as not actionable in the OpenClaw repository. Keep the current documented Teams manifest requirement and ask for a new, focused bug report if Teams file sending still fails with `bots[].supportsFiles: true` and the documented Graph/SharePoint setup in place.
