## Summary

Current main already exchanges GitHub Copilot auth at runtime and replaces the runtime request base URL with the token-derived Copilot endpoint, so the misconfigured `models.json` `baseUrl` from the report no longer controls the actual request path.

## Evidence

- **GitHub Copilot provider prepares runtime auth from exchanged Copilot token:** The bundled `github-copilot` provider implements `prepareRuntimeAuth`, calling `resolveCopilotApiToken(...)` and returning both `apiKey: token.token` and `baseUrl: token.baseUrl`. That means the provider computes the endpoint from the Copilot token instead of trusting static config.
  - file: [extensions/github-copilot/index.ts:168](https://github.com/openclaw/openclaw/blob/a94763ee224e858e29dd177cd6f7f4daa438c2f1/extensions/github-copilot/index.ts#L168)
  - command: `nl -ba extensions/github-copilot/index.ts | sed -n '160,178p'`
  - sha: [a94763ee224e](https://github.com/openclaw/openclaw/commit/a94763ee224e858e29dd177cd6f7f4daa438c2f1)
- **Runner overwrites model baseUrl with provider-prepared runtime auth baseUrl:** The embedded runner applies `preparedAuth?.baseUrl` onto `runtimeModel` before making requests. A wrong provider `baseUrl` in config is therefore superseded at run time when runtime auth returns the Copilot endpoint.
  - file: [src/agents/pi-embedded-runner/compact.ts:399](https://github.com/openclaw/openclaw/blob/a94763ee224e858e29dd177cd6f7f4daa438c2f1/src/agents/pi-embedded-runner/compact.ts#L399)

- **Runtime auth test covers provider-transformed GitHub Copilot credentials and endpoint:** The runtime auth test asserts that a `github-copilot` model receives provider-prepared auth with `baseUrl: https://api.individual.githubcopilot.com`, proving the runtime path uses the transformed Copilot endpoint.
  - file: [src/plugins/runtime/runtime-model-auth.runtime.test.ts:41](https://github.com/openclaw/openclaw/blob/a94763ee224e858e29dd177cd6f7f4daa438c2f1/src/plugins/runtime/runtime-model-auth.runtime.test.ts#L41)
