## Summary

Current `main` no longer matches the reported behavior: default OpenAI GPT-5 routing now goes through the Responses API and sends `max_output_tokens`, while the remaining OpenAI-compatible completions path now reads `compat.maxTokensField` and defaults OpenAI-family requests to `max_completion_tokens` instead of `max_tokens`. The gap described in the April 6 discussion appears fixed on `main`, but I could not tie it to a released tag newer than the provided latest release `v2026.4.23`.

## Evidence
