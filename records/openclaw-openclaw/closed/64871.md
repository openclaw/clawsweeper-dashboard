## Summary

Keep #64871 open. Current main still has the session-keyed bootstrap snapshot behavior the PR targets: `getOrLoadBootstrapFiles` returns the cached array before reloading from disk, and the existing unit test asserts that the second same-session call does not call `loadWorkspaceBootstrapFiles` again. Session-backed agent runs use that cache path, and the cache is only explicitly cleared on session rollover, so same-session bootstrap/context file updates can still remain stale. The PR is a focused bug fix with relevant regression coverage; no conservative close reason applies.

## Best Possible Solution

Keep the PR open for maintainer review. The best path is to land this PR or an equivalent small patch that reloads the workspace bootstrap file set for existing session keys, preserves object identity when unchanged, refreshes when path/content/missing state changes, and adds regression tests for unchanged and changed same-session snapshots. If per-turn filesystem checks are considered too expensive, refine the implementation with an mtime/stat gate or short TTL rather than closing the bug-fix PR.
