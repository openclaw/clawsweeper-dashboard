## Summary

Issue #59456 should close as superseded by the active canonical help-text PR #69975. Current main still has the old `--tz` wording in both `cron add` and `cron edit`, so this is not implemented-on-main; the same remaining help-text-only work is explicitly tracked by #69975, which the reporter identified as the replacement for #59487 and which says it closes #59456.

## Best Possible Solution

Close #59456 as superseded by PR #69975, and keep the remaining work concentrated there: update both `cron add` and `cron edit` `--tz` help text to mention `--cron` plus offset-less `--at` datetime values, while leaving the already-tested runtime behavior intact.
