Full review comments:

- **[P1] Apply log-file before route-first commands:** `src/cli/program/preaction.ts:101`
  - body: `OPENCLAW_LOG_FILE` is only set from the Commander preAction hook, but `runMain` tries `tryRouteCli()` and returns before Commander parses routed commands. As a result, documented invocations like `openclaw --log-file /tmp/x status` still won't promote the flag for `status`, `gateway status`, or routed `config get/unset`; parse/promote the root option before route-first dispatch and cover that path directly.
  - confidence: 0.93
- **[P2] Resolve status log file from daemon env:** `src/logging/log-file-path.ts:42`
  - body: The new log-path helper reads `OPENCLAW_LOG_FILE` from the current CLI process, but daemon status already builds `mergedDaemonEnv` from the managed service command. When the service and invoking shell have different `OPENCLAW_LOG_FILE` values, status will report the wrong file sink; thread an explicit env into the resolver and use the daemon env for daemon status.
  - confidence: 0.87
