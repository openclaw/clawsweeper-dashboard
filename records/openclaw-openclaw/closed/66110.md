Full review comments:

- **[P2] Apply --log-file before route-first dispatch:** `src/cli/program/preaction.ts:96-98`
  - body: The new flag is only promoted in Commander preAction, but this PR also teaches root-option parsing to consume `--log-file`, so `openclaw --log-file /tmp/x status` can still match `tryRouteCli` and return before Commander ever runs. That leaves `OPENCLAW_LOG_FILE` unset for the exact status/config fast paths the PR claims to support; parse and export the root log-file override before route-first/gateway fast paths or have routed bootstrap apply the same root option.
  - confidence: 0.9
- **[P2] Resolve status log files with daemon env:** `src/logging/log-file-path.ts:42`
  - body: `resolveConfiguredLogFilePath()` reads `process.env`, and `gatherDaemonStatus()` returns the log file from `cliCfg` even though it already computes `mergedDaemonEnv` and `daemonCfg` from the managed service. If the service has a different `OPENCLAW_LOG_FILE`, profile, or config path than the invoking CLI, `openclaw status` can display the wrong file; thread an env parameter through the helper and report the daemon context for service status.
  - confidence: 0.86
