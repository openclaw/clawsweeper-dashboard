## Summary

Keep open. Current `main` still rejects `diagnostics.otel.captureContent`, strips it as an unknown config key during doctor cleanup, and the OTel docs/runtime do not expose any replacement under `diagnostics.otel`.

## Evidence

- **Strict config schema still excludes the key:** `diagnostics.otel` only allows `enabled`, `endpoint`, `protocol`, `headers`, `serviceName`, `traces`, `metrics`, `logs`, `sampleRate`, and `flushIntervalMs`, then calls `.strict()`. `captureContent` is not accepted on current `main`.
