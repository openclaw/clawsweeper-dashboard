## Summary

Close as implemented. Current main and release v2026.5.2 now route runtime text-transform reads through the active plugin registry and merge those transforms into the CLI and provider prompt paths; the linked #65331 is superseded by this shipped implementation.

## What This Changes

The issue asks for plugin-registered textTransforms to be applied in the main process instead of resolving an empty registry through loadPluginRuntime().

## Best Possible Solution

Keep the shipped active-registry implementation and treat any remaining work as follow-up test hardening or cleanup of the now-superseded #65331.

## Reproduction Assessment

Yes. The original source-level failure path is clear from the issue, and current-main verification follows resolveRuntimeTextTransforms() through getActiveRuntimePluginRegistry() into the CLI and provider prompt consumers.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix because the generic active-runtime registry seam imports runtime state directly without adding plugin-specific behavior or changing unrelated plugin loading.
