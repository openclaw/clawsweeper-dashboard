## Summary

Keep open: current main still has the owner-scoped CLI cron path, and the PR head appears to address the earlier scoped-token overgrant without introducing a blocking review finding.

## What This Changes

The branch makes CLI-backed isolated cron runs use non-owner loopback authorization while threading a current-job-only cron self-removal grant through the MCP loopback path.

## Best Possible Solution

Merge after normal required CI or changed-gate proof so embedded and CLI-backed isolated cron runs share the same non-owner, job-scoped cron tool boundary.

## Reproduction Assessment

Yes. Source inspection of current main shows CLI-backed isolated cron runs call `runCliAgent` with `senderIsOwner: true`, while embedded isolated cron already uses `senderIsOwner: false` plus the cron allowlist.

## Solution Assessment

Yes. The PR carries the existing embedded policy to the CLI path and threads current-job scope through loopback token, request, cache, and tool resolution; that is narrower than preserving owner auth.
