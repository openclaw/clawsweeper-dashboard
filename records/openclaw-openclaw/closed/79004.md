## Summary

Keep open: current main and the latest release still import Memory Wiki bridge artifacts and refresh deterministic indexes, but they do not run any agent-driven synthesis step that turns new sources into syntheses, entities, or concepts. The prior Codex review remains consistent with current code; this is a feature/product gap, not stale cleanup.

## What This Changes

The issue requests an automated path from Memory Wiki bridge-imported sources to generated syntheses, entities, or concepts without manual wiki_apply.

## Best Possible Solution

Design an opt-in Memory Wiki synthesis automation path that runs after bridge import or dreaming, uses existing wiki search/get/apply primitives, records source provenance, and exposes explicit privacy, cost, scheduling, and approval controls.

## Reproduction Assessment

Not applicable as a strict bug reproduction. Source inspection gives high confidence that current main imports and compiles bridge sources, while structured synthesis still requires explicit wiki_apply, wiki.apply, or CLI apply calls.

## Solution Assessment

Unclear until maintainers choose the product shape. A cron, hook, or import-integrated flow could work, but the maintainable solution should keep deterministic imports separate from opt-in model-driven writes with provenance and resource controls.
