## Summary

Keep open. Current main still catches every phase-signal store read/parse failure as an empty store, the latest release still ships that behavior, and three open PRs already reference this issue with closing links, so the issue should close only after one fix lands.

## What This Changes

The issue requests that `readPhaseSignalStore` preserve dreaming phase-signal history by returning an empty store only for missing or corrupt JSON and rethrowing operational I/O errors.

## Best Possible Solution

Land one focused memory-core fix that rethrows non-recoverable phase-signal read errors while preserving missing-file and corrupt-JSON recovery, with regression coverage, then close this issue and the duplicate fix PRs.

## Reproduction Assessment

Yes. Source inspection provides a high-confidence path: make `memory/.dreams/phase-signals.json` fail with a non-ENOENT read error during `recordDreamingPhaseSignals`, and current main converts that failure into an empty store that can be written back.

## Solution Assessment

Yes. The narrow maintainable fix is to rethrow operational read errors from `readPhaseSignalStore` so the existing dreaming workspace catch path logs the failure and avoids overwriting stored signals.
