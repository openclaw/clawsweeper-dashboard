## Summary

Keep this PR open: the change is focused and appears correct, but it adds a new Discord callback-lifetime config contract that needs maintainer acceptance rather than cleanup closure.

## What This Changes

This PR adds bounded Discord `agentComponents.ttlMs` config/schema/docs/generated metadata, deep-merges Discord account `agentComponents`, and forwards the resolved TTL into component registry registration for sends and edits.

## Best Possible Solution

Land this implementation if maintainers accept the 24-hour opt-in callback lifetime; otherwise lower the bound and update schema, docs, generated metadata, tests, and proof together.

## Reproduction Assessment

Yes, at source level: current main has optional registry TTL support but no supported Discord `agentComponents.ttlMs` config field or send/edit propagation for a configured value. I did not run tests because this is a read-only review.

## Solution Assessment

Yes, conditionally: reusing the existing registry TTL plumbing while preserving the current default is the narrow maintainable approach. The remaining decision is whether 24 hours is the right public maximum.
