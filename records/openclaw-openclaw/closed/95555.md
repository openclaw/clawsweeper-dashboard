## Summary

Current main and the latest stable release already address the reported v2026.6.8 Telegram rich-text font/readability problem by keeping standard Telegram text delivery as the default and making Bot API rich messages explicit opt-in.

## What This Changes

The issue reports smaller Telegram rich-text rendering on v2026.6.8 and asks for matching font size or a config toggle to fall back to plain text.

## Best Possible Solution

Keep the v2026.6.9/current-main behavior: standard Telegram text is the compatibility default, and Bot API 10.1 rich messages remain an explicit opt-in for clients that render them acceptably.

## Reproduction Assessment

Yes at source level for the reported v2026.6.8 behavior: related reports and release-source evidence show that normal Telegram text was forced through rich delivery, while current main and v2026.6.9 gate that path behind richMessages === true. I did not run a live Telegram font-size repro.

## Solution Assessment

Yes. Restoring standard sendMessage/HTML delivery as the default is the narrowest maintainable fix because OpenClaw cannot control Telegram client font metrics for native rich messages, while the opt-in preserves rich formatting for compatible clients.
