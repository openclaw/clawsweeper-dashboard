## Summary

Keep open. Current main still has a source-level path where `session_status({ sessionKey: "current" })` resolves through the sandbox/requester session key, and #76995 is an open implementation PR with closing syntax that should be reviewed or closed before this issue is resolved.

## What This Changes

The issue reports that `session_status({ sessionKey: "current" })` can report a stale Telegram direct session/model instead of the live `agent:main:main` session used by `/status`.

## Best Possible Solution

Review and land or close #76995; the end state should pass the live run session identity to `session_status current` separately from sandbox/policy identity and cover the stale Telegram direct row regression.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: construct tools with a Telegram default direct `sandboxSessionKey` while the live run key is `agent:main:main`, then call `session_status` with `sessionKey: "current"` against a store containing both rows.

## Solution Assessment

Yes. The narrow maintainable solution is to thread live run session identity separately from sandbox policy identity and add regression coverage; the linked open PR is the active candidate for that direction.
