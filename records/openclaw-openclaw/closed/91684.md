## Summary

Keep open: this protected-label Mattermost PR is still needed because current main and the latest release still drop existing Mattermost thread roots when `replyToMode` is `off`, despite the channel docs saying default replies should stay in an existing thread. The patch is focused, has no blocking review findings, and should proceed through maintainer merge handling rather than cleanup closure.

## What This Changes

The PR changes Mattermost reply target resolution and regression tests so existing channel/group thread roots are preserved under default `replyToMode: "off"` while top-level messages and DMs stay unthreaded.

## Best Possible Solution

Land the focused Mattermost helper/test change after maintainer handling so documented default behavior preserves existing channel/group threads without starting new threads for top-level posts or DMs.

## Reproduction Assessment

Yes, at source level: on current main, a channel/group call with `replyToMode: "off"` and a non-empty `threadRootId` returns `undefined`, which sends the reply to the channel root instead of the existing Mattermost thread. I did not run a live Mattermost instance in this read-only review.

## Solution Assessment

Yes. The PR fixes the single shared Mattermost reply/session helper used by the affected runtime paths and keeps the top-level and direct-message exceptions covered by tests; adding a new config option or patching only send delivery would be broader and less consistent with the docs.
