The issue reports a Control UI regression where left-sidebar recent session labels show truncated raw session IDs, with no practical way to expand or identify them at a glance.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/84753 as the canonical label/readability track, and handle any separate resizable or collapsible sidebar choice as a UI product decision alongside https://github.com/openclaw/openclaw/issues/84216.

## Reproduction Assessment

Yes at source level: current main still renders the Recent sidebar in a fixed-width column with ellipsized visible names, while opaque IDs fall through when no better label or displayName exists. I did not run a live browser session in this read-only review.

## Solution Assessment

No, keeping this as a separate issue is not the best path because the remaining root problem is already covered by the open canonical display-name issue. A resize handle may be a separate UX choice, but it should not split the same truncated-ID report into another tracker.
