## What This Changes

The issue reports that outbound transcript mirror appends ignore configured `session.store` when callers pass `config` but no explicit `storePath`.

## Best Possible Solution

Review and land the linked fix PR if it preserves explicit `storePath` priority, then honors configured `session.store`, and covers fixed plus `{agentId}` templated stores.

## Reproduction Assessment

Yes from source inspection: outbound mirror callers pass `config` without `storePath`, while the append fallback still reads `resolveDefaultSessionStorePath(params.agentId)`. I did not run a live outbound channel repro because this review is read-only.

## Solution Assessment

Yes, the best narrow fix is to resolve the transcript append fallback in the session transcript helper using explicit `storePath`, then configured `session.store`, then the existing default. The linked PR appears to target that owner boundary rather than adding store-path plumbing to every outbound caller.
