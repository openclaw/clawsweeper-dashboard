## Summary

Current `main` already provides the requested capability in practice: built-in automatic compaction with preserved recent context, configurable compaction-only model/provider and memory flush, plus a second provider-side compaction layer for OpenAI Responses and plugin-owned context-engine compaction hooks.

## Risks / Open Questions

- The implementation is distributed across core compaction, context engines, and provider integrations rather than a single feature named "dual-layer context compression".
- Provider-side server compaction is currently explicit for supported backends such as OpenAI Responses, not a universal gateway-side layer for every provider.
