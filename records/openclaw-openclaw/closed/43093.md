## Summary

Current main already fixes the user-facing cross-date `openclaw logs --follow` / `logs.tail` failure. The implementation now moves long-lived rolling loggers to the current-day file at write time and routes `logs.tail` through shared log-tail resolution with newest-rolling-file fallback, so this helper-based PR is obsolete.

## Best Possible Solution

Close this PR as implemented by current main, keep the shared logger/log-tail implementation, and handle any remaining `openclaw channels logs` dated-file behavior through the separate channel-log follow-up rather than merging this now-obsolete helper path.
