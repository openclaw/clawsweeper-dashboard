## Summary

Current main already fixes the reported Claude CLI tool-event gap: Claude stream-json tool-use records are parsed, converted into `stream: "tool"` agent events, and threaded through both one-shot and live Claude CLI execution paths. The merged implementation is current-main-only after the latest release, so users on `v2026.5.27` still need a newer release to receive it.

## What This Changes

The issue reports that Claude CLI stream-json `tool_use` records are dropped before reaching `onAgentEvent`, so iMessage and webchat do not receive `stream:tool` activity events.

## Best Possible Solution

Keep the merged parser-to-agent-event bridge on main and leave any remaining iMessage watchdog behavior to the separate linked watchdog issue.

## Reproduction Assessment

Yes. The source-level reproduction is to feed Claude stream-json `tool_use` and `content_block_*` records through the CLI JSONL parser and observe whether `stream: "tool"` reaches the agent event bus; current main now has parser and runner coverage for that path.

## Solution Assessment

Yes. The merged fix uses the existing `emitAgentEvent`/`onAgentEvent` contract and shared CLI parser paths instead of adding channel-specific workarounds.
