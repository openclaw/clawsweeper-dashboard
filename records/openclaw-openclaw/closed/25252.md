## Summary

Issue #25252 is a reasonable idea, but current OpenClaw main treats this kind of optional tool/protocol integration as plugin scope rather than core scope. The repo docs explicitly steer optional capabilities and tool integrations toward plugins published via ClawHub or npm, and current main contains no UTCP-specific implementation.

## Close Comment

Thanks for the suggestion. I reviewed this on current `main` as a Codex maintainer pass, and I’m closing it as a ClawHub/plugin-scope request rather than a core OpenClaw item.

The current project direction is that optional capability should usually ship as plugins, not core (`VISION.md:52-59`), and the plugin docs already support adding agent tools, hooks, commands, routes, and other runtime integrations as external plugins published via ClawHub or npm (`docs/plugins/building-plugins.md:11-19`, `docs/plugins/building-plugins.md:145-168`; `docs/tools/clawhub.md:10-17`). I also checked current `main` at `d3f6783b16ecc10c67b101bf019ca4294453206a` and there is no UTCP-specific implementation in-tree.

Given that, UTCP looks like a better fit as an external OpenClaw plugin distributed through ClawHub, instead of expanding OpenClaw core.
