## Summary

Keep open: current main still maps fully permissive OpenClaw exec policy to Claude CLI bypass mode without a root-aware branch, and the active control-request work only covers the timeout/stall subset. The remaining root permission policy, docs, and any higher-level config surface need maintainer security/product judgment.

## What This Changes

The issue requests root-aware Claude CLI permission handling, clearer root/workaround documentation, better blocked-turn diagnostics, and easier Claude backend configuration for fallback use.

## Best Possible Solution
