## Summary

Keep open. Current main still has the reported memory-core leak, and this PR is a useful focused fix, but the patch still lets selected dreaming marker lines pass because the new fence helper skips marker lines before testing overlap.

## What This Changes

The PR broadens memory-core dreaming contamination detection, adds a rehydration-time dreaming-fence overlap guard, and covers those paths in short-term promotion tests.

## Best Possible Solution

Keep the inline-metadata detector, then fix the rehydration guard so managed marker lines count as overlapping or replace that part with the managed-block redaction approach from https://github.com/openclaw/openclaw/pull/80702.

## Reproduction Assessment

Yes. Source-level probing shows current main misses the reported inline status/confidence/evidence/recalls Candidate shape, and current rehydration returns relocated snippets without a dreaming-fence check; I did not run a live promotion cycle in this read-only review.

## Solution Assessment

No, not as currently written. The widened detector is a narrow fit for the leak, but the new fence helper must count marker lines as managed content or use the stronger managed-block redaction path before merge.
