## Summary

Close as `clawhub`: native Sentry DSN wiring is a vendor-specific observability integration, while current `main` keeps core observability generic through diagnostics/stability plus the bundled OTLP exporter and routes optional integrations to plugins/ClawHub. No protected labels or maintainer author signal are present.

## Best Possible Solution

Keep OpenClaw core focused on generic diagnostics, stability bundles, and OTLP export. Put Sentry DSN wiring in a ClawHub/npm code plugin or preload package; if implementation reveals a generic early crash-hook gap, open a focused plugin API issue for that hook rather than adding Sentry-specific core behavior.
