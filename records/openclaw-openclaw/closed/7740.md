## Summary

Issue #7740 requests bundling an externally maintained skill, but current OpenClaw main explicitly routes new skills to ClawHub/community distribution instead of core.

## Evidence

- **Vision: new skills go to ClawHub first:** Current project vision says bundled skills remain baseline UX only; new skills should be published to ClawHub first, and core skill additions should be rare.
  - file: [VISION.md:68](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/VISION.md#L68)
- **Vision: not merging new core skills:** The roadmap guardrails explicitly list 'New core skills when they can live on ClawHub' under items the project will not merge for now.
  - file: [VISION.md:101](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/VISION.md#L101)
- **Plugin docs: community/external distribution is first-class:** User-facing plugin docs describe external community plugins and a ClawHub-first install resolver instead of bundling optional capabilities into core.
  - file: [docs/tools/plugin.md:11](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/docs/tools/plugin.md#L11)
- **Skills docs: ClawHub is the public skill registry:** Current skills docs direct users to ClawHub for skill discovery/install/update, and to the separate `clawhub` CLI for publish/sync workflows.
  - file: [docs/tools/skills.md:109](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/docs/tools/skills.md#L109)
- **Skill authoring guide: contributors should use ClawHub:** The skill creation guide tells authors to use ClawHub and links it as the public skill registry rather than suggesting new bundled-skill additions to core.
  - file: [docs/tools/creating-skills.md:99](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/docs/tools/creating-skills.md#L99)
- **ClawHub guide: official registry for skills and plugins:** The ClawHub docs state that ClawHub is the public registry for OpenClaw skills and plugins, with native OpenClaw install/search/update flows and separate publish/sync tooling.
  - file: [docs/tools/clawhub.md:10](https://github.com/openclaw/openclaw/blob/cec34821754ad935c780805f3116f99449d46ac2/docs/tools/clawhub.md#L10)
