## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Derive shrinkwrap checks from reviewed state:** `scripts/generate-npm-shrinkwrap.mjs:235-239`
  - body: `generateShrinkwrap()` runs a fresh `npm install --package-lock-only` in a temp directory, so `--check` depends on current registry resolution rather than a stable reviewed source. That can make unchanged release commits fail or pull in newly resolved transitive versions during the check path; derive the check from checked-in lock state or document and test live resolution as an intentional release policy.
  - confidence: 0.9
- **[P2] Include package overrides during normalization:** `scripts/generate-npm-shrinkwrap.mjs:190-198`
  - body: `packageJsonForShrinkwrap()` merges package-local overrides into the temporary manifest, but `normalizeShrinkwrapOverrides()` only builds enforcement rules from `pnpm-workspace.yaml`. A package-only override can still be hidden inside a dependency-owned shrinkwrap subtree, leaving the emitted lockfile out of policy for that package.
  - confidence: 0.84
- **[P2] Hydrate bundled deps when overlays are unchanged:** `scripts/lib/plugin-npm-package-manifest.mjs:592-596`
  - body: This early return skips `installPackageLocalBundledDependencies()`. If a plugin package is already normalized but bundling is enabled, the callback can run with `bundledDependencies` metadata but no package-local `node_modules`, so `npm pack` can emit a tarball whose manifest and contents disagree.
  - confidence: 0.88
- **[P2] Install root optional bundled dependencies:** `scripts/lib/plugin-npm-package-manifest.mjs:227-229`
  - body: The portable optional scan only discovers missing optional dependencies after reading an already-installed package. A root optional dependency listed in `bundledDependencies` but skipped by the first npm install is ignored here, so cross-platform packs can silently omit a dependency that the manifest says is bundled.
  - confidence: 0.78
