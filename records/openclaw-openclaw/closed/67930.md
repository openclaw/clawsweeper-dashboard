## Summary

Current main already supports llama.cpp through OpenClaw's generic local OpenAI-compatible provider path. The docs explicitly say self-hosted `llama.cpp` is supported, and runtime code has llama.cpp-specific compatibility handling, so this request is already satisfied without adding a separate bundled `llama.cpp` provider id.
