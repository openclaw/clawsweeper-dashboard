## Summary

This is a coherent feature request, but it is asking for a broad optional orchestration/planning layer rather than a missing OpenClaw core primitive. Current main already exposes the pieces a planner can build on through plugins/skills: inbound and agent-turn hooks, prompt/model hooks, tool approval gates, subagent hooks, plugin commands/CLI, runtime subagent helpers, Task Flow/background task state, and documented ACP/subagent execution paths. The project vision explicitly keeps core lean, sends optional capabilities to plugins/ClawHub, and lists default agent-hierarchy/heavy orchestration layers as out of scope without a stronger product/security rationale.

## Best Possible Solution

Close this core issue and encourage the idea as a ClawHub plugin or skill bundle first: implement a read-only `route`/explain command, optional prompt guidance, approval-aware tool hooks, and Task Flow/background-task status integration using the existing plugin SDK. Reopen or file a narrower core API issue only if that external implementation proves a specific missing hook, runtime helper, or UI descriptor contract is needed.
