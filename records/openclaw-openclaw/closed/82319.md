## Summary

Close as superseded: current main already has `/stop`, `chat.abort`, and `sessions.abort` wired to abort active run signals, while the remaining active-turn stop gaps are already tracked by https://github.com/openclaw/openclaw/issues/82162, https://github.com/openclaw/openclaw/pull/82298, and the broader run-invalidation tracker https://github.com/openclaw/openclaw/issues/70319.

## What This Changes

Requests a user-facing way to cancel a single active LLM stream mid-generation without killing the session.

## Best Possible Solution

Keep the existing `/stop` and abort APIs as the user-facing cancellation path, then land or reject the narrower text-stop fix and broader run-invalidation work in their canonical threads.

## Reproduction Assessment

Not applicable for this feature-shaped report: `/steer` being model-boundary-only is documented behavior, while current main already provides `/stop`, `chat.abort`, and `sessions.abort`. The concrete remaining failure mode has a source-level path in the canonical text `/stop` regression report.

## Solution Assessment

Yes. Closing this as superseded is the best cleanup path because a separate broad tracker would duplicate the existing abort implementation, the open text-command regression, and the broader run-invalidation issue.
