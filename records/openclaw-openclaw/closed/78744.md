## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Clear the orphan SIGKILL timer on child exit:** `extensions/acpx/src/codex-auth-bridge.ts:291-292`
  - body: When orphan cleanup starts, the child exit handler returns before clearing `forceKillTimer`. If the child exits after SIGTERM, the delayed callback can still call `process.kill(-child.pid, "SIGKILL")`; after PID/PGID reuse that can terminate an unrelated process group.
  - confidence: 0.88
- **[P2] Run legacy cleanup when lease verification skips:** `extensions/acpx/src/runtime.ts:777`
  - body: `cleanupProcessTreeForRecord` returns after every lease cleanup attempt, even when the result terminated nothing. For legacy or migrated records, a selected stale lease can be marked `lost` and prevent the record PID fallback from reaping the live ACPX tree.
  - confidence: 0.82
