## Summary

Current `main` already routes Telegram media downloads through the same proxy-aware transport path as Bot API calls, and the changelog records this fix in version `2026.3.13`. The issue describes behavior on `2026.3.12`, which is older than the shipped fix.

## Evidence

- **Proxy-aware transport is created from Telegram bot options:** `createTelegramBotCore` resolves `telegramTransport` from `opts.proxyFetch` plus Telegram network config, so the bot has a shared transport object for API calls and related fetches.
