## Summary

Current main implements the intended runtime-dependency behavior for this report. Discord's reported packages are plugin-local dependencies, packaged/global installs intentionally keep bundled plugin node_modules out of the global package tree, and the loader repairs, registers, mirrors, and imports from the external runtime stage. The supporting docs/tests are in current main; I did not find a shipped release tag for this yet, so fixedRelease is null.

## Best Possible Solution

Keep the current external runtime-deps stage and mirror design rather than copying Discord deps into the global package tree. Ship the current main fix in the next release; if Discord still fails after that, track it as a new report with the exact plugin load error and stage/root paths.
