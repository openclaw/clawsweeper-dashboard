## Summary

The report does not match current `main`: the Nextcloud Talk plugin is now a webhook-bot channel, with inbound reads handled by signed webhook delivery rather than a `/chat/{token}` polling/fetch path. On current `main`, I found no `lookIntoFuture`, `/chat/`, or notification-polling code in the Nextcloud Talk plugin, so the reported 998 chat-read failure is obsolete against today's implementation.
