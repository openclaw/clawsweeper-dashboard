## Summary

Current main already routes browser-plugin config loading through the shared OpenClaw config runtime, and that shared schema accepts object-form channel streaming configs for Discord. The shipped v2026.4.23 release contains the same implementation, so the reported browser-plugin rejection is obsolete.

## Evidence

- **Browser plugin now uses shared config runtime:** The browser plugin's config entrypoint is only a re-export of the shared browser config runtime, instead of a separate validator/loader.
  - file: [extensions/browser/src/config/config.ts:1](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/browser/src/config/config.ts#L1)
