## Summary

Current main already implements the requested framework-level dedupe for hook-adjusted bootstrap files. Hooks can still mutate `context.bootstrapFiles`, but `resolveBootstrapFilesForRun` now sanitizes the hook-adjusted list by resolving each path against the workspace and skipping duplicate workspace-relative paths before prompt context and `systemPromptReport` are built. Regression coverage exercises duplicate hook paths and confirms only one injected context file remains. The same implementation is present in the `v2026.4.26` release tag.

## Best Possible Solution

Close this issue as implemented and keep the framework-level dedupe in `resolveBootstrapFilesForRun`; any remaining context-list label fallback work should stay in the narrower display/reporting issues rather than this accumulation report.
