## Summary

Current `main` does not match the reported behavior: Ollama remote hosts are wired through `models.providers.<id>.baseUrl`, requests prefer that configured provider URL over localhost, and `baseURL` is not a supported config key in the strict schema.

## Evidence

- **Strict config schema only accepts `baseUrl`:** `ModelProviderSchema` defines `baseUrl` and is `.strict()`, so `baseURL` is not a valid provider field.
  - file: [src/config/zod-schema.core.ts:346](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/config/zod-schema.core.ts#L346)
  - command: `nl -ba src/config/zod-schema.core.ts | sed -n '340,390p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Config validation rejects unknown fields:** Repo tests assert strict config validation rejects unknown fields rather than silently ignoring them.
  - file: [src/config/config-misc.test.ts:588](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/config/config-misc.test.ts#L588)
  - command: `nl -ba src/config/config-misc.test.ts | sed -n '588,602p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Ollama stream setup reads provider `baseUrl` from config:** The Ollama plugin passes `resolveConfiguredOllamaProviderConfig(...).baseUrl` into stream creation.
  - file: [extensions/ollama/index.ts:161](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/extensions/ollama/index.ts#L161)

  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Runtime prefers configured provider host over localhost:** `resolveOllamaBaseUrlForRun` returns `providerBaseUrl` first and only falls back to the loopback default when no provider/model URL is configured.
  - file: [extensions/ollama/src/stream.ts:43](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/extensions/ollama/src/stream.ts#L43)

- **Tests cover remote provider host behavior:** Tests assert non-default remote `baseUrl` is treated as explicit config and that requests go to `http://provider-host:11434/api/chat`, not `127.0.0.1`.
  - file: [extensions/ollama/index.test.ts:272](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/extensions/ollama/index.test.ts#L272)
