## What This Changes

The issue reports that Slack account-level `streaming.mode: "off"` is ignored when channel-level Slack streaming/block defaults remain enabled, causing partial draft snapshots to be posted as separate messages.

## Best Possible Solution

Land a focused Slack regression fix that makes account-level `mode: "off"` suppress the reported inherited block-streaming path while preserving or explicitly documenting any compatibility-sensitive explicit block opt-in behavior.

## Reproduction Assessment

Yes. Source inspection shows account `streaming.mode: "off"` can merge with inherited `streaming.block.enabled: true`, then Slack dispatch leaves block streaming enabled because the helper does not receive the resolved mode.

## Solution Assessment

Yes, the fix belongs in the Slack streaming dispatch/config path. The issue should stay open until the linked repair PR, or an equivalent narrow fix with regression coverage, lands on main.
