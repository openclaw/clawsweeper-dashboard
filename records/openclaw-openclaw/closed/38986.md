## Summary

Current `main` already implements an opt-in fix for this fake-IP/TUN DNS case: `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange` allows `web_fetch` to pass the RFC 2544 `198.18.0.0/15` range instead of failing unconditionally, and the latest shipped release `v2026.4.22` contains that implementation.
