## Summary

Close as duplicate/superseded by the older canonical #64121 tracker. Current main still lacks a first-class configured-agent chat selector, but #64121 covers the same remaining split into Agent and Agent Session controls, and open PR #68926 is already the implementation candidate for that surface.

## What This Changes

The issue requests a first-class Control UI chat agent switcher that lists configured agents, switches between them, and can start or target a session for any configured agent.

## Best Possible Solution

Keep #64121 as the canonical tracker for the split Agent and Agent Session controls, and review, rebase, or replace #68926 so the fix lands once in the current Control UI chat surfaces.

## Reproduction Assessment

Yes. A source-level reproduction is clear: with agents.list containing a non-default agent that has no stored session row, the current chat controls only render options from sessions.list plus the current session key, so that configured agent is not directly selectable.

## Solution Assessment

Yes. Closing this duplicate while preserving #64121 as the canonical issue is the narrowest maintainer path; the grouped session select on main is not sufficient, and the fix should land through the canonical tracker or #68926.
