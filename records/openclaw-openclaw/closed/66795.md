Keep this PR open. Current main still has the Telegram picker fallback bug, and the PR’s functional change is narrow and correct, but the user-facing fix needs a required changelog entry and current-head validation before merge.

## What This Changes

This PR changes Telegram model-picker callback session reconstruction to use the routed agent’s effective default model and adds a regression test for an agent-specific default.

## Best Possible Solution

Land this patch or an equivalent small change after adding the changelog entry and rerunning the Telegram and changed-surface validation.

## Reproduction Assessment

Yes. Configure a routed Telegram agent whose `agents.list[].model` differs from `agents.defaults.model`, leave the session without a stored override, and trigger `mdl_list_<provider>_1`; current main reconstructs the global default, so the agent default is not checkmarked.

## Solution Assessment

Yes. Reusing `resolveDefaultModelForAgent()` at the Telegram callback fallback is the narrow maintainable fix; caching the resolved default once inside the function would be a small cleanup, not a different solution.
