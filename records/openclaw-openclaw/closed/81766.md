## Summary

Keep this PR open: the Feishu plugin is the right boundary for legacy `/docs/:token` reads, but the current branch still has blocking error-handling defects and lacks required real behavior proof from a real Feishu legacy document.

## What This Changes

The PR adds Feishu legacy document read support by detecting old doc v2 metadata and adding REST helpers for `/raw_content` and `/content` alongside the existing docx read path.

## Best Possible Solution

Keep the legacy read path in the Feishu plugin, but scope fallback to metadata only, keep raw reads working when rich content fails, update `/docs/:token` guidance, and land with focused tests plus redacted live proof.

## Reproduction Assessment

Yes for the review findings via source inspection: PR head awaits `readOldDoc` inside the metadata fallback catch and makes raw text depend on `/content` in one `Promise.all`. I did not establish a live Feishu legacy-document reproduction for the feature itself.

## Solution Assessment

No. The plugin boundary is appropriate, but this patch is not the best final solution until old-doc read errors propagate correctly, rich content is best-effort for plain-text reads, and caller-facing legacy token and permission guidance is updated.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Move old-doc reads out of the meta catch:** `extensions/feishu/src/docx.ts:856`
  - body: `readOldDoc` still runs inside the `try` whose `catch` falls back to docx. If metadata identifies a legacy doc but `/content` or `/raw_content` fails, the real old-doc error is swallowed and retried as docx with the legacy token, producing a misleading failure. Only the metadata lookup should be inside this fallback catch.
  - confidence: 0.91
- **[P2] Make rich old-doc content best-effort:** `extensions/feishu/src/old-doc.ts:155-158`
  - body: `readOldDoc` returns plain text from `/raw_content`, but the `Promise.all` makes `/content` a hard dependency. A rich-content failure currently discards raw text that was already readable; fetch or handle rich block content best-effort so plain legacy reads still work when only block metadata fails.
  - confidence: 0.83
- **[P3] Update legacy document guidance:** `extensions/feishu/src/docx.ts:856`
  - body: This exposes `read` for legacy `/docs/:token` documents, but the tool schema and bundled skill still tell callers to extract `/docx/` tokens and list only docx scopes. Add legacy token extraction and old-doc permission guidance so agents and operators can use the new path.
  - confidence: 0.78
