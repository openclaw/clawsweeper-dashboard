## Review Findings

Full review comments:

- **[P1] Probe the configured storage roots instead of a writable fallback:** `src/gateway/server.impl.ts:894`
  - body: This wires the probe with empty options, so `resolveDefaultProbeDir()` picks the first directory where it can create `.openclaw/.health`. If the PVC-backed OpenClaw home/config root is full or unwritable, that mkdir can fail and the resolver can fall through to `HOME` or `process.cwd()`, returning `{ ok: true }` for an unrelated filesystem while the actual workspace remains broken. Bind the probe to the critical state/workspace roots and fail readiness when those cannot be written.
  - confidence: 0.91
- **[P2] Keep source-root discovery scoped to requested plugin IDs:** `src/plugins/plugin-metadata-snapshot.ts:741`
  - body: This appends every discovered source-root plugin whose ID is not in the indexed registry, even when `loadPluginMetadataSnapshot()` was called with a scoped `pluginIds` set. Current code filters installed-index candidates by that scope and records it in `snapshot.pluginIds`; widening the manifest registry with unrequested source plugins can make setup, provider, or runtime callers see plugins they explicitly did not ask for. Apply the same `pluginIds` scope to `sourceRootDiscovery.candidates` or keep this change separate with regression coverage.
  - confidence: 0.86
