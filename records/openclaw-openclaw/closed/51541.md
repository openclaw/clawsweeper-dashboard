## Summary

Close as cannot reproduce. The reporter later confirmed the failure was caused by a stale `google:default` key in `auth-profiles.json`, not by `.env` loading. Current `main` still intentionally resolves auth profiles before env vars for provider auth, Gemini embeddings use that shared path, both `GEMINI_API_KEY` and `GOOGLE_API_KEY` are accepted, and `.env` loading for `~/.openclaw/.env` remains implemented.

## Evidence

- **Reporter identified local root cause:** In the issue follow-up on 2026-03-21, the reporter confirmed the problem was a stale `google:default` API key in `~/.openclaw/agents/main/agent/auth-profiles.json`; after updating that stored key, Gemini embeddings worked and reindexing completed.
- **Current auth resolution is profile-first:** `resolveApiKeyForProvider()` iterates stored auth profiles before falling back to env vars, unless a caller explicitly requests `credentialPrecedence: "env-first"`. That matches the reporter's observed behavior with a stale stored Google key shadowing env credentials.
  - file: [src/agents/model-auth.ts:496](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/agents/model-auth.ts#L496)
  - command: `nl -ba src/agents/model-auth.ts | sed -n '412,565p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Gemini embeddings use the shared Google auth resolver:** The Gemini memory embedding client resolves provider auth via `resolveApiKeyForProvider({ provider: "google" ... })`, so it is not using a separate `.env`-bypassing path.
  - file: [extensions/google/embedding-provider.ts:315](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/extensions/google/embedding-provider.ts#L315)

  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Google provider accepts both env var names:** The bundled Google plugin advertises `GEMINI_API_KEY` and `GOOGLE_API_KEY` as provider auth env vars, and the provider docs say both are accepted.
  - file: [extensions/google/openclaw.plugin.json:7](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/extensions/google/openclaw.plugin.json#L7)
