## Summary

Keep open: current main still lists and accepts explicit allowlist-only sub-agent ids, and the recent wildcard hardening PR deliberately preserved that compatibility path, so this needs maintainer/security policy review rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to reject or clearly warn on stale explicit `allowAgents` entries whose agent config has been removed.

## Best Possible Solution

Define the explicit `allowAgents` registry contract, then either fail closed for stale allowlist-only targets with migration/docs/doctor coverage or keep compatibility while making `agents_list` and doctor/security warnings unambiguous.

Unclear pending maintainer/security decision. Fail-closed registry validation is the cleaner security boundary, but it would reverse a recently preserved compatibility behavior for explicit allowlist entries.
