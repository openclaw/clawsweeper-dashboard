## Summary

Current main contains a new claude-cli integration path that matches the reporter’s working `claude -p` flow instead of the failing `--bare` path, hardens Anthropic env handling to avoid host-managed billing routing, and adds doctor/docs/live-test coverage for headless Claude auth and setup-token/subscription flows. The specific #64308 failure no longer matches current main.

## Evidence

- **Bundled claude-cli backend now uses `claude -p` flow, not `--bare`:** The current Anthropic CLI backend invokes `claude` with `-p`, `--output-format stream-json`, `--setting-sources user`, live-session support, and session resume flags. There is no `--bare` flag in the configured backend path.
