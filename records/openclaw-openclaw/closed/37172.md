## Summary

Keep open. Current main still ties Telegram group sender authorization to whether the message is processed at all: unauthorized/group-override-blocked messages return before any pending-history buffering, while pending history is only recorded on the mention-gating skip path. The config/docs still expose only `groupPolicy`/`groupAllowFrom` plus `contextVisibility` for supplemental context filtering; there is no separate trigger-vs-context control like the issue requests.
