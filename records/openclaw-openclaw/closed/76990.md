## Summary

Current main now implements the central missing-assistant transcript repair: embedded runs are gap-filled into the session transcript without re-mirroring the user prompt, and duplicate assistant tails are skipped. The broader stale-reply/orphan-user branch-drift reports remain tracked separately, so this assistant-side report can close as implemented.

## What This Changes

The issue requests that successful embedded Pi assistant replies captured in run artifacts be persisted as visible assistant turns in the active session transcript before later prompts are assembled.

## Best Possible Solution

Keep the shipped embedded assistant gap-fill path and continue handling the remaining orphan-user branch-drift work under the narrower related issues.

## Reproduction Assessment

Yes for the original affected releases: successful embedded runs could produce assistantTexts while the durable transcript lacked the visible assistant turn. Current main should no longer reproduce the central assistant-side gap because embedded results now append a deduped assistant text entry to session JSONL before the next turn path resumes.

## Solution Assessment

Yes. The merged shared persistence path is a narrow maintainable solution because Pi still owns the user prompt and embedded gap-fill adds only the missing visible assistant tail when it is not already present.
