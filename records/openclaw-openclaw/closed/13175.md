## Summary

Current main now implements the central Feishu inline-button request through the shared message `presentation`/`interactive` payload path: Feishu outbound declares native presentation button support, renders buttons as Feishu Schema 2.0 action elements, wraps value buttons in the existing `ocf1` card-action envelope, sends the card through `sendCardFeishu`, and routes quick-action callbacks back through `handleFeishuCardAction`. This is not the exact legacy `[label](btn:action)` syntax from the original proposal, but it solves the observable user problem with the repository's current shared presentation contract. The fix is present on current main and documented in the changelog as current-main/unreleased; no local release tag contains the proof commit yet.

## Best Possible Solution

Close as implemented on current main. Keep the shared message presentation implementation as the product direction for Feishu buttons instead of adding a Feishu-specific `inlineButtons` config or markdown-only shortcut; any future work should be a narrower docs or compatibility request if maintainers still want a legacy syntax alias.
