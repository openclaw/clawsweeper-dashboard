## Summary

Close as `cannot_reproduce`: the current `main` implementation no longer matches the reported architecture. `google-gemini-cli` is now wired as a local Gemini CLI backend, while the provider plugin owns OAuth setup/usage metadata only; the issue describes an older direct OpenClaw OAuth adapter path with embedded 60s stalls between tool-call steps.

## Evidence

- **`google-gemini-cli` is a CLI backend now:** The bundled Google plugin registers `google-gemini-cli` as a CLI backend that shells out to local `gemini` with JSON output and `--resume` support.

- **Plugin registration wires the CLI backend:** The Google plugin registers the Gemini CLI backend alongside provider registration.
  - file: [extensions/google/index.ts:152](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/extensions/google/index.ts#L152)

  - file: [docs/providers/google.md:65](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/docs/providers/google.md#L65)
