## Summary

Current `main` contains post-report session-model logic that stops transient fallback selections from permanently pinning a chat to the fallback model, while preserving explicit user `/model` choices. That matches the persistent GPT-5.4 -> Claude Sonnet drift described here, so this is implemented on `main` but not in the latest tagged release yet.

## Evidence

- **Auto fallback selections are now transient and self-heal back to the configured primary:** `createModelSelectionState` explicitly clears `modelOverrideSource: "auto"` and resets the turn back to the configured default provider/model before continuing, instead of leaving the session pinned to the fallback.
  - file: [src/auto-reply/reply/model-selection.ts:173](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/auto-reply/reply/model-selection.ts#L173)
  - command: `nl -ba src/auto-reply/reply/model-selection.ts | sed -n '173,199p'`
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
