## Summary

Close as implemented on current main: the DeepSeek provider now advertises `xhigh` and `max` for V4 models and the shared DeepSeek V4 payload wrapper maps both to `reasoning_effort: "max"`. The proof commit is on `main` after `v2026.5.2`, so the reported release is behind current source.

## What This Changes

Expose DeepSeek V4 Pro `/think xhigh` and `/think max` in model thinking profiles and map both to DeepSeek `reasoning_effort: "max"`.

## Best Possible Solution

Keep the current provider-owned implementation and let the next release carry the fix; no additional core change is needed for this report.

## Reproduction Assessment

No for current main: the reported 2026.5.2 behavior is obsolete because current source and regression tests show DeepSeek V4 Pro exposes xhigh/max and maps xhigh to reasoning_effort max. The original release predates the proof commit.

## Solution Assessment

Yes. The current solution is the narrow maintainable path because DeepSeek owns its model-specific thinking profile while the shared DeepSeek V4 wrapper handles the OpenAI-compatible payload mapping.
