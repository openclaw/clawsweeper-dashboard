## Summary

Close as ClawHub/external integration work. PR #65406 is a docs-only guide for a third-party Claude subscription billing proxy, and current main already has the generic config/plugin surfaces needed to point Anthropic-compatible traffic at a custom base URL or package a provider/plugin guide externally. OpenClaw project scope says optional capability and plugin promotion should live in ClawHub rather than core docs, and the PR discussion still includes a privacy-wording concern for a proxy that forwards request content upstream to Anthropic.

## Best Possible Solution

Close this core-docs PR and route the guide to the linked openclaw-billing-proxy project or a ClawHub skill/plugin/bundle entry. Keep OpenClaw source docs focused on the Anthropic provider, generic baseUrl/custom-provider behavior, and the provider plugin SDK; open a narrow core issue only if an external implementation proves a missing plugin or provider API.
