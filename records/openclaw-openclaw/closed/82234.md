## Summary

Current main already fixes the reported Telegram isolated-ingress regression: spooled updates now drain per Telegram sequential lane, so control-lane stop/status messages can dispatch while an older regular turn remains active; the fix is current-main-only and not in v2026.5.12.

## What This Changes

The issue reports that Telegram stop/abort messages in polling mode wait behind the isolated ingress spool instead of interrupting active runs promptly.

## Best Possible Solution

Keep the current per-lane isolated-ingress spool drain on main and let the next release carry the Telegram control-lane bypass to users on v2026.5.12.

## Reproduction Assessment

No, there is no high-confidence failing reproduction path on current main because the source and tests already show the bypass behavior. The reported v2026.5.12 path is source-backed by the isolated spool design, but this review did not run live Telegram.

## Solution Assessment

Yes. The current main solution is the narrow maintainable fix: keep isolated ingress, but drain spooled updates by Telegram control/session lane so stop/control messages are not blocked by an active ordinary turn.
