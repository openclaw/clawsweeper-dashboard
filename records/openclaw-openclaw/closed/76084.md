## Summary

Keep open: the PR is a plausible QQ Bot dispatcher fix, but a fresh member review raised a concrete quote-semantics bug and showed live C2C image traffic on the PR head using top-level `attachments`, so the current branch should not land as the claimed fix without another update or proof.

## What This Changes

The PR maps QQ Bot C2C and group media nested under `msg_elements` into queued inbound attachments when top-level attachments are absent, adds dispatcher tests, and records a changelog entry.

## Best Possible Solution

Update the dispatcher fallback only after either a redacted live payload confirms current-message `msg_elements` media still occurs, or rescope it as defensive normalization with an ext-based quote/ref guard and coverage for the non-103 quote/ref shape.

## Reproduction Assessment

Partially. A source-level payload with media only under `msg_elements` reproduces the current-main gap, but the latest live QQ Bot C2C image run on the PR head delivered media through top-level `attachments`, so the original live symptom still needs a redacted current op=0 payload.

## Solution Assessment

No. The dispatcher is the right layer if the live payload shape is real, but this patch needs a broader quote/ref guard using `message_scene.ext` and should not keep claiming the linked issue is fixed without live payload proof.
