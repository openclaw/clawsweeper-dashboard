## Summary

Current main fixes the central model-routing problem: stale runtime-only `modelProvider`/`model` snapshots are no longer treated as session selections, and explicit user-pinned sessions are surfaced with clear status/docs guidance. The fix is present on `main` only and is not in the latest tagged release yet.

## What This Changes

The issue requests discoverability and a supported way to recover when existing channel sessions appear pinned to an older provider/model after the configured default changes.

## Best Possible Solution

Keep the current contract: configured defaults drive new and unpinned sessions, explicit user selections stay strict, and status/docs direct users to `/model` or `/reset` instead of silently rewriting session state.

## Reproduction Assessment

No current-main failure path was found. Source and tests show runtime-only `modelProvider`/`model` snapshots are not treated as user-pinned selections, while explicit user pins are detected and reported.

## Solution Assessment

Yes. Preserving explicit session choices while ignoring stale runtime snapshots and warning through status is safer than automatically rewriting user session state.
