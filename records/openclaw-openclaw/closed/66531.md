## Summary

Current `main` already gives external clients a supported way to cancel `agent()` runs by routing them through the existing `chat.abort`/`sessions.abort` surfaces, so the uncancelable embedded-run gap described in #66531 is no longer present even though a separate `agent.abort` RPC was not added.

## Evidence

- **Agent runs now register into the shared abort registry before dispatch:** `agent()` creates a `chatAbortControllers` entry with `kind: "agent"` and passes that abort signal into the dispatched run, which closes the original gap where agent RPC runs were not externally abortable.
  - file: [src/gateway/server-methods/agent.ts:879](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/gateway/server-methods/agent.ts#L879)
  - command: `nl -ba src/gateway/server-methods/agent.ts | sed -n '876,900p'`
  - sha: [9a0b26cafca7](https://github.com/openclaw/openclaw/commit/9a0b26cafca76d267dd231b5eaceadb071ec749d)
- **Public RPC contract already exposes the needed abort inputs:** `chat.abort` is a typed Gateway surface with `sessionKey` plus optional `runId`, so out-of-process clients have a supported RPC instead of relying on the `globalThis[Symbol.for(...)]` side door.
  - file: [src/gateway/protocol/schema/logs-chat.ts:54](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/gateway/protocol/schema/logs-chat.ts#L54)

- **chat.abort now aborts tracked runs by session or specific run:** The handler accepts `{ sessionKey, runId? }`, aborts all active runs for the session when `runId` is omitted, and aborts the specific tracked run when `runId` is present.
  - file: [src/gateway/server-methods/chat.ts:1956](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/gateway/server-methods/chat.ts#L1956)

- **Regression tests cover agent-run cancellation through chat.abort:** Tests assert that agent RPC runs are registered in `chatAbortControllers`, that `chat.abort` by `runId` aborts the agent run signal, and that session-scoped `chat.abort` also cancels the active agent run.
  - file: [src/gateway/server-methods/agent.test.ts:1570](https://github.com/openclaw/openclaw/blob/9a0b26cafca76d267dd231b5eaceadb071ec749d/src/gateway/server-methods/agent.test.ts#L1570)

## Risks / Open Questions

- A dedicated `agent.abort` RPC still does not exist; clients need to use `chat.abort` or `sessions.abort` rather than a new agent-specific method.
- The implementing behavior is present on `main`, but the latest tagged release in this checkout is `v2026.4.23`, so this appears unreleased so far.
