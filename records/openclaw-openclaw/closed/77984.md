## Summary

Current main still matches the reported failure shape: the Codex app-server completion watchdog is a hard-coded 60s timer, status notifications can become the last activity reason, and plugin config/docs do not expose a turn-completion idle timeout knob.

## What This Changes

The issue asks OpenClaw to avoid or make configurable Codex app-server turn-completion idle timeouts that fire after a status-only `rawResponseItem/completed` notification.

## Best Possible Solution

Add a documented Codex plugin `appServer` turn-completion idle timeout setting and richer timeout diagnostics while preserving stale-turn protection by default.

## Reproduction Assessment

Yes. A focused source reproduction is clear: after an armed turn-scoped request response, a `rawResponseItem/completed` notification updates `lastActivityReason`, and 60s of silence triggers `turn.completion_idle_timeout`; I did not run the live Codex app-server scenario in this read-only review.

## Solution Assessment

Yes for a first repair. Exposing the existing test-only timeout through Codex plugin config and improving diagnostics is narrower and safer than changing raw response item semantics without more protocol evidence.
