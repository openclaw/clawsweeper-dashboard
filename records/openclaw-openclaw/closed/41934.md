## Summary

Current `main` no longer uses `launchctl kickstart -k` for config-triggered/SIGUSR1 restarts under macOS launchd. The restart path now exits cleanly and lets launchd `KeepAlive` relaunch the gateway, with regression tests and changelog entries covering the behavior.

## Evidence

- **Launchd restart path is supervisor-owned:** `restartGatewayProcessWithFreshPid()` detects launchd supervision and returns `supervised`; it only calls `triggerOpenClawRestart()` for Windows scheduled-task restarts, so macOS config/SIGUSR1 restarts no longer go through `launchctl kickstart -k`.
  - file: [src/infra/process-respawn.ts:30](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/infra/process-respawn.ts#L30)
  - sha: [7ac35b4f6900](https://github.com/openclaw/openclaw/commit/7ac35b4f69009490180a35bf404638d4280a4865)
- **SIGUSR1 restart exits cleanly for launchd:** After shutdown, the gateway treats a launchd-supervised restart as `full process restart (supervisor restart)`, waits briefly to avoid launchd crash-loop throttling, then exits `0` so `KeepAlive` can relaunch it.
  - file: [src/cli/gateway-cli/run-loop.ts:82](https://github.com/openclaw/openclaw/blob/7ac35b4f69009490180a35bf404638d4280a4865/src/cli/gateway-cli/run-loop.ts#L82)
