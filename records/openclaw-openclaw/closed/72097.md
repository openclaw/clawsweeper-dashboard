## Summary

Keep open. Current main still refreshes plugin registry metadata after install while active agent runs build concrete plugin-tool arrays and Pi tool allowlists at setup time, and #75976 is an open linked PR intended to address this issue.

## What This Changes

After plugin install, active agent sessions should refresh their model-facing plugin toolset without requiring a gateway restart.

## Best Possible Solution

Land or replace #75976 with a generic runtime/session refresh path that re-resolves plugin tools after install, enable, disable, or registry refresh through the existing policy pipeline at a safe turn boundary.

## Reproduction Assessment

Yes. The issue gives concrete install -> immediate use -> restart steps, and current main source confirms install refreshes registry metadata while model-facing tool arrays and Pi allowlists are setup-time snapshots.

## Solution Assessment

Unclear because no fix is merged on current main. The maintainable direction is turn-boundary re-resolution through existing tool policy and cache invalidation rather than arbitrary mid-stream mutation of active session state.
