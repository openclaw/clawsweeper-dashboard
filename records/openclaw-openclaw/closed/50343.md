## Summary

Current main already carries a targeted fix for first-call local gateway handshake failures on CLI RPCs, and the changelog shows the handshake-timeout fix shipped after the reporter's 2026.3.13 build.

## Evidence

- **`nodes pending` still uses the generic CLI gateway RPC path:** `openclaw nodes pending` calls `callGatewayCli("node.pair.list", ...)`, which flows into the shared `callGateway(...)` transport used for local loopback CLI gateway requests.
  - file: [src/cli/nodes-cli/register.pairing.ts:17](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/cli/nodes-cli/register.pairing.ts#L17)
- **Current main adds an event-loop yield before opening the WebSocket:** `src/gateway/call.ts` now does `await new Promise<void>((r) => setImmediate(r));` immediately before creating the per-call `GatewayClient`, with an inline comment that this avoids starving `connect.challenge` processing and causing handshake timeouts on heavy CLI startup paths.
  - file: [src/gateway/call.ts:479](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/gateway/call.ts#L479)
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **Current main uses a 10s pre-auth handshake timeout on the server side:** The gateway handshake timeout now defaults to `10_000` ms in `src/gateway/handshake-timeouts.ts`, and `src/gateway/server/ws-connection.ts` applies that value to the pre-auth close timer that logs `handshake timeout ...` before closing unauthenticated sockets.
  - file: [src/gateway/handshake-timeouts.ts:1](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/src/gateway/handshake-timeouts.ts#L1)
  - sha: [9b7f6250f43b](https://github.com/openclaw/openclaw/commit/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f)
- **The shipped release notes match this exact failure mode:** `CHANGELOG.md` for release `2026.3.22` says: `Gateway/WS handshake: raise the default pre-auth handshake timeout to 10 seconds ... so busy local gateways stop dropping healthy CLI connections at 3 seconds.` That is the same first-call local CLI handshake failure pattern described in this issue.
  - file: [CHANGELOG.md:2446](https://github.com/openclaw/openclaw/blob/9b7f6250f43bbc1816cdaf26e8c07116dfa4975f/CHANGELOG.md#L2446)
- **The latest shipped release still contains the handshake-yield fix:** `git show 00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5:src/gateway/call.ts` shows the same pre-connect `setImmediate(...)` yield in the released `v2026.4.22` tree, so this is not just an unreleased main-branch change.
  - file: [src/gateway/call.ts:479](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/gateway/call.ts#L479)
