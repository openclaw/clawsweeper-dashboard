## Summary

Current `main` already changes image-routing behavior in the area reported here: it now prefers `agents.defaults.imageModel` over the active reply model for auto image understanding, disables the native-vision skip when explicit image config exists, and has regression tests for both behaviors. The latest release `v2026.4.23` predates these `main`-only changes, so this looks fixed on `main` but not yet shipped in a release.

## Evidence

- **Auto image routing now prefers configured imageModel:** `resolveAutoEntries()` now returns `agents.defaults.imageModel` entries before trying the active reply model for image understanding, which directly matches the reported `ollama/qwen3.5:cloud` plus global `agents.defaults.imageModel.primary` setup.
