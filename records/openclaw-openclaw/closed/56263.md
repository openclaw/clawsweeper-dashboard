## What This Changes

Requests a configurable or opt-out file and directory permission policy so multi-user group/ACL deployments can preserve 0640/0750-style access instead of repeated owner-only chmod hardening.

## Best Possible Solution

Define a maintainer-approved sensitive-state permission policy that keeps owner-only defaults, supports explicit operator-managed group/ACL deployments, and applies consistently across config, state DBs, security repair, fs-safe private stores, transcripts, approvals, and docs/tests.
