## Summary

The requested shared bootstrap/reference directory is useful, and the discussion shows real multi-agent demand, but it fits the current plugin/ClawHub extension model rather than a new core config key. Current main keeps core workspace bootstrap reads guarded to the workspace, while native plugins can register internal agent:bootstrap hooks and mutate bootstrapFiles, which is enough for a trusted shared-dir plugin to read configured external paths and inject them into prompt context.
