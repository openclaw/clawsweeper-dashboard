## Summary

Keep open: the PR targets a real notification-delivery gap, but the current patch is not safe to merge because it broadens the shared `deliver` contract. Exec approval followups would send instruction prompt text directly to external chat instead of running the agent and delivering the synthesized reply.

## What This Changes

The PR adds a direct-delivery early return in `agentCommandInternal`, updates `--deliver` help text, and adds CLI/gateway tests for direct delivery behavior.

## Best Possible Solution

Add an explicit direct-delivery contract for the CLI notification path while preserving generic `deliver` as delivery of agent-produced replies, with regression coverage for exec followups and subagent internal-event announcements.

## Reproduction Assessment

Yes. Source inspection is enough: current main has exec approval followups call the gateway with `deliver: true`, an instruction prompt, and no `internalEvents`, so the PR early return would deliver that prompt before any agent turn runs.

## Solution Assessment

No. The PR fixes the direct notification case by broadening the shared `deliver` contract; the safer solution is a separate direct-delivery signal from the CLI path so gateway/internal followups still synthesize user-facing replies.
