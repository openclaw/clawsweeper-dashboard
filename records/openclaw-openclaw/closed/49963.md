## Summary

Current main already fixes the user-visible direct/inline memory embedding batch timeout problem with a newer, documented `sync.embeddingBatchTimeoutSeconds` config path. The older PR branch is no longer the best change to merge because its review discussion identified real regressions, while main now has source, schema, docs, and regression-test coverage for the corrected behavior.

## Best Possible Solution

Close #49963 as implemented on main. Keep the shipped main design: use `agents.defaults.memorySearch.sync.embeddingBatchTimeoutSeconds` for direct/inline embedding batches, keep `remote.batch.timeoutMinutes` scoped to provider batch APIs, and avoid merging the older PR branch with its dead-fallback and timeout-clamp regressions.
