## Summary

Keep open. Current main still has the source-reproducible Active Memory bug, and the linked open PR is an implementation candidate that should be reviewed or replaced before this issue is closed.

## What This Changes

Active Memory should not pass Telegram forum-topic conversation IDs as embedded recall run channel/plugin IDs when group chat recall is enabled.

## Best Possible Solution

Review and land or replace #76719 so embedded recall uses a runnable provider/channel id while preserving the Telegram forum-topic session key as conversation context.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Telegram topic session key parses to channelId '-100123:topic:77', Active Memory prioritizes that over messageProvider 'telegram', and embedded-run channel discovery then reaches bundled plugin dirName validation.

## Solution Assessment

Yes. The narrow maintainable solution is to keep the topic id as session/conversation context but resolve the embedded recall run's runnable channel/provider to 'telegram'; the linked open PR targets that boundary.
