## What This Changes

The PR classifies Telegram isolated-ingress missing-agent-harness failures as non-retryable, dead-letters those spooled updates via the existing failed-tombstone path, adds wrapped-error regression tests, and records the fix in the changelog.

## Best Possible Solution

## Reproduction Assessment

Yes, source-reproducible: current main requeues `bot.handleUpdate` failures and the lane drain can leave later same-lane updates behind a failing head. The linked issue and PR body also include main-branch Crabbox breakage proof, but I did not rerun live Telegram in this read-only pass.

## Solution Assessment

Yes: adding a narrow non-retryable classifier in `polling-session.ts` and reusing `failTelegramSpooledUpdateClaim` preserves transient retries while unblocking same-lane spool drain. The remaining maintainer choice is whether tombstoning the poison update is sufficient or whether a user-facing slash fallback should be added separately.
