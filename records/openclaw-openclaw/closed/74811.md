## Summary

Keep open. Current main does not auto-disable the Bonjour advertiser on WSL2, and the PR is a plausible narrow fix using an existing public runtime-env helper. The patch should not be closed as obsolete; it mainly needs normal maintainer review plus a small docs follow-up because the Bonjour docs still describe auto-disable as container-only behavior.

## What This Changes

The branch imports `isWSL2Sync` into the Bonjour advertiser, skips mDNS advertising on detected WSL2 unless `OPENCLAW_DISABLE_BONJOUR=0` is set, adds WSL2 advertiser tests, and adds a changelog fix entry.

## Best Possible Solution

Land the narrow WSL2 Bonjour auto-disable if maintainers accept that default, update the Bonjour/discovery docs to describe WSL2 alongside containers and the `OPENCLAW_DISABLE_BONJOUR=0` opt-in, then keep #74209 open for the broader bundled-plugin startup policy.
