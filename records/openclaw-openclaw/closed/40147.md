## Summary

Keep this PR open. Current main still implements the opposite of the PR's central Feishu behavior: `thread_id`-only group messages still force `replyInThread` even when config disables it, and current tests explicitly assert that behavior. The submitted PR is also stale against the current file split and has an unresolved functional review concern around forcing `root_id` thread replies outside the intended group-routing path.

## Best Possible Solution

Keep this PR open for maintainer product review or refresh it against current main. A mergeable fix should update `extensions/feishu/src/bot-content.ts` and `extensions/feishu/src/reply-dispatcher.ts` together, decide the intended `root_id` versus `thread_id` semantics relative to `replyInThread: "disabled"`, scope any forced `root_id` behavior to the intended group surfaces, preserve or explicitly change DM/quote behavior, and add regression tests for thread-id-only disabled groups, root-id groups, topic sender routing, and p2p quote replies.
