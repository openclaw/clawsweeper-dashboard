## Summary

Current main no longer matches this report: WhatsApp inbound policy explicitly allows same-phone DMs, inbox coverage verifies those messages reach the inbound handler, and routing coverage still resolves WhatsApp account/peer bindings instead of silently dropping them.

## Evidence

- **Same-phone DM admission is explicit in current policy:** WhatsApp inbound policy treats the linked self number as an allowed DM sender via `isSamePhone(sender) || isNormalizedSenderAllowed(...)`. That means a message from the linked WhatsApp number is not rejected by DM allowlist/pairing checks on current main.
  - file: [extensions/whatsapp/src/inbound-policy.ts:110](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/extensions/whatsapp/src/inbound-policy.ts#L110)
  - command: `nl -ba extensions/whatsapp/src/inbound-policy.ts | sed -n '104,126p'`
- **Access-control test covers owner/same-phone delivery:** The WhatsApp access-control test asserts that same-phone DMs are allowed even when `allowFrom` is restrictive, and that no pairing reply is emitted for that path.
  - file: [extensions/whatsapp/src/inbound/access-control.test.ts:172](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/extensions/whatsapp/src/inbound/access-control.test.ts#L172)
  - command: `nl -ba extensions/whatsapp/src/inbound/access-control.test.ts | sed -n '172,206p'`

That directly covers the 'linked and connected but inbound never arrives' symptom for same-phone DMs.
