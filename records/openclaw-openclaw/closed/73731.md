## Summary

Current main removes the reported built-in model suppression runtime-hook path. Built-in suppression now resolves from manifest `modelCatalog.suppressions`; the old `resolveProviderBuiltInModelSuppression` fallback that iterated catalog-hook provider plugins in v2026.4.26 is gone, with tests and docs updated to match. The broader cache/load reports linked by the issue remain separate work.

## Best Possible Solution

Keep the manifest-backed built-in model suppression implementation on main and leave broader provider/cache performance follow-ups to the separate linked reports (#73728 and #73729).
