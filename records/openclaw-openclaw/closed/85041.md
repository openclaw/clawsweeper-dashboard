## What This Changes

The issue reports that Talk mode using OpenAI fails because the gateway still uses a deprecated OpenAI Realtime beta session-token endpoint shape.

## Best Possible Solution

Keep the shipped GA Realtime flow and ask for a fresh issue only if a current `v2026.5.19` or newer install still reaches `beta_api_shape_disabled` with versioned logs.

## Reproduction Assessment

No: current main and the shipped `v2026.5.19` tag no longer contain the reported `/v1/realtime/sessions` endpoint path. Source inspection shows Talk now mints `/v1/realtime/client_secrets` and posts WebRTC SDP to `/v1/realtime/calls`.

## Solution Assessment

Yes: the current implementation already follows the GA client-secret plus WebRTC call flow, which is the narrow maintainable fix for this report. No repo change is needed unless a fresh current-build repro shows a different failing path.
