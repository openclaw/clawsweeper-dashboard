## Summary

Close #51614 as duplicate/superseded. The generic gateway stale threshold is already configurable globally via `gateway.channelStaleEventThresholdMinutes`, but the remaining WhatsApp-specific low-traffic reconnect problem is still real and is now tracked by canonical open issue #70678 with active maintainer PR #71466. Current main still does not expose `channels.whatsapp.healthMonitor.staleEventThresholdMs` or `web.messageTimeoutMs`, and the WhatsApp extension-local watchdog still uses an internal 30-minute inbound-silence default.

## Best Possible Solution

Close this older mixed tracking item as superseded by canonical open issue #70678 and maintainer PR #71466. Keep the shipped global `gateway.channelStaleEventThresholdMinutes` behavior, let #71466 resolve the WhatsApp quiet-session reconnect bug, and use #63939 or a focused follow-up only if maintainers still want an explicit `web.messageTimeoutMs` or account-scoped watchdog config after the default behavior is fixed.
