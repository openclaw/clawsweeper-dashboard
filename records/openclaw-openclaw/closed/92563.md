## Summary

Keep open: current main and v2026.6.9 still copy every assistant text row into session-memory output, and the three linked closing PRs remain unmerged candidate fixes rather than landed implementation.

## What This Changes

The issue reports duplicate assistant lines in session-memory memory files when thinking-enabled transcripts contain both raw assistant output and a cleaned mirror row.

## Best Possible Solution

Land one canonical consumer-layer session-memory filter that skips only transcript mirror duplicates while preserving legitimate repeated or standalone assistant content, then close this issue after merge.

## Reproduction Assessment

Yes, at source level. Current main extracts and pushes text from every assistant message row, so a raw thinking row followed by a stripped delivery-mirror copy yields duplicate assistant lines.

## Solution Assessment

No, the issue's suggested storage-layer deletion is not the safest default. A consumer-layer semantic filter is narrower because replay and history surfaces intentionally keep persisted transcript bookkeeping rows.
