## Best Possible Solution

Keep the current main recovery path and include it in the next OpenClaw release so release users get session recovery without a gateway restart.

Yes. The current main fix is narrow and maintainable: treat final assistant output as enough to release the OpenClaw session after a short idle guard, preserve the reply, and clean up the dangling Codex turn best-effort.
