## Summary

## What This Changes

The PR expands `openclaw doctor` to warn on and repair legacy `openai-codex/*` refs across config and session stores, updates model-ref scanning, docs, changelog, and doctor tests.

## Best Possible Solution

Keep the broad config rewrite, but narrow session-store mutation to entries with actual legacy `openai-codex/*` provider/model/auth/fallback state and preserve valid native Codex harness pins unless the generic owner-scoped doctor proves they are stale.

## Reproduction Assessment

Yes, source-level: a session entry with only `agentHarnessId: "codex"` is reported stale when the repair runtime resolves to `pi`, then the repair writes the session harness fields to `pi`. I did not execute it because this review is read-only.

## Solution Assessment

No: the legacy route rewrite direction is maintainable, but the current session repair predicate is broader than the route state being fixed. The safer repair is to require actual legacy provider/model/auth/fallback evidence before changing session harness pins.
