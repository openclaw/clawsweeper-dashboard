## Summary

Current main already implements the PR's central behavior through the shared gateway model-catalog visibility path: default `models.list` no longer returns the full catalog, `view: "all"` is the explicit full-catalog escape hatch, and provider availability now goes through the broader runtime auth checker. The submitted PR branch should not be merged as-is because its local auth helper misses supported Bedrock and local-provider auth paths that current main now covers.

## What This Changes

The PR changes gateway `models.list` so the default response filters catalog entries to providers with recognized auth when no explicit model allowlist is configured, while preserving a full-catalog fallback and adding gateway tests.

## Best Possible Solution

Close this PR and keep the current shared main implementation. Any remaining product work, such as configurable Control UI filter modes or model-switch preflight behavior, should continue in the broader model-selector follow-up rather than by merging this branch.
