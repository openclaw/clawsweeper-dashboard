## Summary

OpenClaw already exposes a generic `agent_end` plugin hook with `success` and `error` fields, fires it on failed agent runs, documents it as a supported lifecycle hook, and has a regression test covering a `rate limit exceeded` failure path.

## Evidence

- **Public hook exists:** `agent_end` is part of the public plugin hook surface, and its event payload includes `success` plus optional `error` fields for generic failure handling.
  - file: [src/plugins/hook-types.ts:63](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/plugins/hook-types.ts#L63)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)
- **Hook payload includes failure data:** `PluginHookAgentEndEvent` is defined with `messages`, `success`, `error?`, and `durationMs?`, which matches the request for a hook on rate-limit or other errors.
  - file: [src/plugins/hook-types.ts:204](https://github.com/openclaw/openclaw/blob/a958b6e72328d1d922a6d133b1a70e866faf4013/src/plugins/hook-types.ts#L204)
  - sha: [a958b6e72328](https://github.com/openclaw/openclaw/commit/a958b6e72328d1d922a6d133b1a70e866faf4013)

## Close Comment

Closing as implemented on main after Codex review. OpenClaw already exposes a generic plugin lifecycle hook for this case: `agent_end` is part of the public hook API in `src/plugins/hook-types.ts:63` and its payload includes `success` plus `error` in `src/plugins/hook-types.ts:204`. The embedded runner fires that hook on failed runs and passes the formatted error text in `src/agents/pi-embedded-runner/run/attempt.ts:2569`. The docs list `agent_end` as a supported lifecycle hook in `docs/concepts/agent-loop.md:97`, and there is a regression test showing a `rate limit exceeded` failure triggers `agent_end` with `success: false` and `error: "rate limit exceeded"` in `src/agents/cli-runner.reliability.test.ts:542`. The current release `v2026.4.22` also contains this hook surface, so there is not a remaining core gap to track here.
