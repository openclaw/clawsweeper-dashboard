## Summary

Current main intentionally keeps `cron` in the `coding` profile, suppresses the old built-in profile warning path that this PR was addressing, and has regression coverage that depends on `cron` remaining re-allowable under `tools.profile: "coding"`. The PR's one-line removal no longer matches main's behavior or docs.
