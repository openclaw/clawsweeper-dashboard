## Summary

Current `main` already eliminates the duplicate `/new` notice through a broader refactor: reply-producing bare `/new`/`/reset` turns are handled as in-band reset prompts and no longer send a separate standalone reset notice, so the extra `resetNoticeSent` plumbing from PR #55605 is no longer needed.
