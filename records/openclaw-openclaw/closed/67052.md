## Summary

Current main and the latest release now solve the reported UX problem by separating content completion from terminal finalization: the agent emits a lifecycle `finishing` phase after the model turn finishes, and the TUI changes from `streaming` to `finishing context` until the final lifecycle/chat cleanup completes.

## What This Changes

The issue requests that the TUI stop showing the response as actively streaming long after visible assistant content has finished.

## Best Possible Solution

Keep the shipped lifecycle `finishing` to TUI `finishing context` path, and ask for a fresh issue with v2026.6.6-or-newer evidence if the status still stays on `streaming` after visible content completes.

## Reproduction Assessment

Yes. The original failure was source-reproducible: older handling kept the visible status on `streaming` until terminal chat/lifecycle completion, while current source and tests show a separate `finishing` lifecycle path before finalization.

## Solution Assessment

Yes. The shipped solution is the maintainable path because it uses an explicit lifecycle signal from the agent runtime instead of making the TUI guess from delta silence, and it preserves terminal final/idle behavior for cleanup completion.
