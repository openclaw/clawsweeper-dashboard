## Summary

Current `main` has the trace-carrier groundwork and emits separate `openclaw.run`, `openclaw.model.call`, and `openclaw.tool.execution` spans, but it still does not wire them as child spans under `openclaw.message.processed`. The merged follow-up PR #70924 explicitly scoped itself to adding the shared trace carrier and said it made "no runtime telemetry behavior changes yet," so this issue remains open. ([github.com](https://github.com/openclaw/openclaw/pull/70924))
