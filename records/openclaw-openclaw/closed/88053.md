## Summary

Keep this PR open, but it is not merge-ready: removing the Feishu schema check alone conflicts with the current shared DM ingress runtime, tests, setup copy, and security guidance, so the proposed config would load while the runtime can still block the sender.

## What This Changes

The PR removes the Feishu config schema validation that requires `channels.feishu.allowFrom` to include `"*"` when `channels.feishu.dmPolicy` is `"open"`, and also adds a repo-root PR-description markdown file.

## Best Possible Solution

Either keep the wildcard requirement as the current explicit public-DM contract, or intentionally change the Feishu DM-open contract across schema, runtime, tests, setup/doctor guidance, and docs with maintainer security review before merge.

## Reproduction Assessment

Yes. Source inspection reproduces the merge blocker: current main's shared ingress gate and tests still require a wildcard or explicit allowlist match for `dmPolicy="open"`, so the PR's schema-only change cannot make wildcard-free Feishu DMs dispatch.

## Solution Assessment

No. Removing only the Feishu schema guard is not the best fix because it accepts configs that the current runtime, tests, setup flow, and docs still treat as blocked unless the wildcard is present.
