## Summary

Current `main` already implements sub-agent model selection for both `agents.defaults.subagents.model` and explicit `sessions_spawn.model`, and it persists the resolved child runtime model so `session_status` can report the spawned session correctly. The fix appears present on `main` after the latest tagged release, so this issue is obsolete against current `main` but may still match older builds such as 2026.3.13.

## Evidence
