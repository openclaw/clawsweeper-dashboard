## Summary

Current `main` already routes `openai-codex` agent runs through the shared OpenAI Responses transport and explicitly serializes inbound user turns for both HTTP/SSE and WebSocket paths. The relevant transport/message-conversion code landed after the issue was opened and is covered by focused tests, so the reported drop-on-dispatch behavior is implemented/fixed on `main`.
