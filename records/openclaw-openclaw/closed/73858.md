## Summary

Current main already fixes the reported custom-provider image capability path. Explicit `models.providers.<id>.models[]` rows are merged into the gateway model catalog with the parent provider id, and `resolveGatewayModelSupportsImages` now looks them up through the shared catalog matcher before deciding whether to offload images.

## Best Possible Solution

Close this issue as implemented on current main. The best follow-through is to let the next OpenClaw release carry commit `4bd3d258`; users on `v2026.4.26` should use a newer main build or wait for the next tagged release.
