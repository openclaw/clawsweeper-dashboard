## Summary

Keep open. Current main only partially covers the diagnostic gap by preserving a sanitized error message, while this PR's stack/cause goal is not implemented on main and the proposed branch still has blocking diagnostics privacy and sanitizer issues.

## What This Changes

This PR extends diagnostic stability-bundle error metadata with message, stack, and cause fields plus a sanitizer, and adjusts stability-bundle tests to expect persisted error messages.

## Best Possible Solution

Rebase and revise this PR so generated startup/restart failure bundles capture useful stack and bounded structured cause metadata while preserving current secret redaction and keeping imported bundle/support-export output within the documented privacy model.

## Reproduction Assessment

Yes. Current main has a focused source and test path: startup-failure bundles go through `readSafeErrorMetadata`, which persists sanitized `message` but not `stack` or `cause`; the support-export path for imported bundles is also traceable in source.

## Solution Assessment

No. The feature direction is useful, but sharing one serializer for generated bundles and imported support-export bundles is not the narrowest maintainable fix; write-time capture should be separated from import/export sanitization.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Redact secrets before persisting error text:** `src/logging/diagnostic-stability-bundle.ts:127-136`
  - body: `sanitizeForBundle` only redacts home paths, strips ANSI escapes, and truncates text. Error messages and stacks commonly contain tokens, API keys, config snippets, or user payloads, so this can persist and later export sensitive text that current diagnostics redaction is supposed to remove.
  - confidence: 0.86
- **[P1] Keep imported error text out of support exports:** `src/logging/diagnostic-stability-bundle.ts:144-155`
  - body: `readSafeErrorMetadata` is also used while parsing imported stability bundles for `gateway stability --bundle ... --export`. Adding `message`, `stack`, and `cause` here lets arbitrary imported text flow into shareable support zips, conflicting with the documented payload-free export model.
  - confidence: 0.88
- **[P2] Serialize cause as bounded metadata:** `src/logging/diagnostic-stability-bundle.ts:152-155`
  - body: `String(cause)` collapses an `Error.cause` into text and drops the safe fields the PR promises, such as cause name, message, and code. Serialize one bounded cause object without recursion so the bundle remains useful without unbounded growth.
  - confidence: 0.82
- **[P2] Strip NUL bytes from bundle text:** `src/logging/diagnostic-stability-bundle.ts:127-136`
  - body: The sanitizer does not remove embedded NUL bytes even though the PR describes NUL stripping as part of the safety contract. Add an escaped `\x00` replacement and cover message, stack, and cause text.
  - confidence: 0.76
