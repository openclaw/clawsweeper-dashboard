## Summary

Keep open: current main and the latest release still render exec approval chat prompts from fixed English shared text, Telegram native and forwarded approvals reuse that renderer, and there is no locale/template config contract for this safety-sensitive prompt surface.

## What This Changes

Request to add localized or operator-customizable templates for exec approval prompts, especially Telegram/native chat approval messages.

## Best Possible Solution

Add a validated shared exec-approval rendering contract with localized labels/templates while preserving canonical /approve commands, approval IDs, command previews, allowed decisions, expiry, and warnings across Telegram and fallback chat flows.

## Reproduction Assessment

Not applicable: this is a feature/config request, not a broken behavior report. Source inspection confirms current main still uses fixed English chat approval text with no locale/template config.

## Solution Assessment

Unclear until maintainers define the product and security contract. The safer path is a shared validated renderer contract rather than unconstrained per-channel string replacement.
