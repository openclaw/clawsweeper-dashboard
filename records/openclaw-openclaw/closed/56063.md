## Summary

Current main already carries Feishu-specific target normalization and session-route handling that make heartbeat `target: "last"` resolve to valid Feishu delivery ids, while explicit Feishu ids belong in `heartbeat.to` rather than `heartbeat.target`. This appears fixed on main after the latest release.

## Evidence

- **Feishu inbound now records routable `To` targets:** Feishu inbound messages set `feishuTo` to `chat:<chatId>` for groups and `user:<openId>` for DMs, then pass that into inbound context as both `To` and `OriginatingTo`.
  - file: [extensions/feishu/src/bot.ts:575](https://github.com/openclaw/openclaw/blob/7130e56f872aea96577ac9fcbda0c7555a9ecc69/extensions/feishu/src/bot.ts#L575)
