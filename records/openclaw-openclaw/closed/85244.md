## What This Changes

The PR deletes the auth-profiles.json watcher and removes its gateway startup wiring while keeping startup prewarm and auth-failure rewarm behavior.

## Best Possible Solution

Stop the self-write rewarm loop while preserving external credential invalidation, for example by filtering OpenClaw-owned/no-op auth-profile writes or comparing the persisted credential payload before scheduling a rewarm.

## Reproduction Assessment

Yes. Source inspection shows the watcher fires on every auth-profiles.json event and OpenClaw auth success/failure paths write the store; the PR body also includes repeated rewarm log lines, though I did not run a live gateway repro.

## Solution Assessment

Unclear. Deleting the watcher is the narrowest way to stop the feedback loop, but the more maintainable end state is to suppress self-writes while keeping external credential invalidation unless maintainers explicitly accept restart-required behavior.
