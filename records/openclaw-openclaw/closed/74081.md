Full review comments:

- **[P1] Keep env substitution string-preserving until schema-aware:** `src/config/env-substitution.ts:208`
  - body: `resolveConfigEnvVars` runs before config validation, so returning `asArray` here changes every exact `${VAR}` string field, not only array-typed fields. Existing env-backed string settings whose value happens to be a JSON array literal can become arrays and fail validation or auth handling; keep substitution string-only or gate coercion on a schema/path that proves the target accepts `string[]`.
  - confidence: 0.88
- **[P2] Preserve env placeholders when coerced arrays are written back:** `src/config/env-substitution.ts:208`
  - body: The config write path restores `${VAR}` only for string leaves, and the map restore walks arrays by element path. With this return, `allowFrom: "${ALLOW_FROM}"` loads as `string[]`; a later config write persists the literal array instead of the env placeholder, so container deployments lose the env-backed source unless whole-value array refs are preserved.
  - confidence: 0.82
