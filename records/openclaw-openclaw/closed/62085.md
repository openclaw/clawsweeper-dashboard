## Summary

Current `main` already fixes the reported WebChat image-loss path: the UI still sends image attachments, the gateway no longer drops them for text-only models, plugin-bound sessions are treated as image-capable, and regression tests now cover both offloaded text-only images and bound-session image forwarding. The latest release tag still had the old drop-on-text-only behavior, so this appears fixed on `main` but not yet in `v2026.4.23`.

## Evidence
