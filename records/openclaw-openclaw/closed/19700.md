## Summary

Issue #19700 proposes a location-based powpow integration as an OpenClaw skill/plugin. Current main scope explicitly routes optional integrations and new skills to external plugins/ClawHub rather than core, and current docs already support installing external/community plugins through ClawHub or local/npm packages.

## Close Comment

Closing as `clawhub` after a Codex review. Current `main` is explicit that optional capability should usually ship as plugins, that external plugins should live in their own repositories, and that new skills should go to ClawHub first rather than OpenClaw core (`VISION.md:54`, `VISION.md:69`, `VISION.md:101`). Current docs also already support external/community plugin installs, including ClawHub installs and local/npm plugin packages (`docs/tools/plugin.md:11`, `docs/tools/plugin.md:26`, `docs/tools/plugin.md:51`). That makes the proposed powpow map integration a better fit as a ClawHub/community plugin than as a core OpenClaw issue.
