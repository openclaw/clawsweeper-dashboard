## Summary

Current main addresses the reported root cause: plugin manifests are deduplicated before runtime loading, duplicate diagnostics now include the losing and winning sources, tracked installed global plugins can outrank bundled duplicates, and shipped `plugins.installs` records are migrated out of `openclaw.json` so doctor/config writes do not reintroduce that legacy duplicate state. The related #53967 PR was closed unmerged, but main now has an independent implementation. The #53926 env-file issue mentioned in the comment is a separate open config-consistency report.

## Best Possible Solution

Close this broad root-cause issue as implemented on main. Keep the shipped manifest-registry/config-index behavior, leave #53926 and any independent Feishu gateway crash/restart reports to their narrower issue paths, and track a general `doctor --fix --dry-run` UX request separately if maintainers want that workflow.
