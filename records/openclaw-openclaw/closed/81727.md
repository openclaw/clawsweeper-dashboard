## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Honor delivery-mode fallback before dropping replies:** `src/infra/heartbeat-runner.ts:1692-1693`
  - body: `resolveSourceReplyDeliveryMode` can downgrade requested `message_tool_only` delivery to `automatic` when message-tool delivery is unavailable, but this gate still drops non-tool payloads solely because heartbeat-tool mode was requested. In that downgrade path the heartbeat becomes `ok-empty` instead of delivering the automatic fallback reply selected by the delivery contract.
  - confidence: 0.86
- **[P2] Accept valid Git worktrees as existing checkouts:** `scripts/install.sh:2356`
  - body: Normal Git worktrees store `.git` as a file, not a directory. This new guard rejects those existing checkouts as non-git directories before `git -C "$repo_dir"` can validate them, so source installs into worktrees now fail immediately.
  - confidence: 0.84
- **[P2] Track the preferred plugin instance by path:** `src/infra/npm-managed-root.ts:442`
  - body: `preferredPackageName` is a global name match while the scan walks nested `node_modules`. A nested older copy of the same package can still be treated as preferred and overwrite the top-level plugin's peer range, so managed peer pins can drift to a stale spec.
  - confidence: 0.82
