## Summary

Current main already implements the reported OpenCode ACP compatibility behavior: rejected optional timeout config writes are treated as best-effort and an OpenCode-shaped regression test covers the `ACP -32602 Invalid params` path. The fix is current-main-only relative to v2026.5.12, so it should close as implemented on main rather than as shipped in the latest release.

## What This Changes

The issue asks OpenClaw to stop failing OpenCode ACP sessions when `session/set_config_option` rejects the `timeout` runtime option.

## Best Possible Solution

Keep the generic optional-timeout compatibility behavior on main and ship it in the next release, rather than adding a growing per-agent timeout skip list.

## Reproduction Assessment

Yes. The affected-release path is the reported `sessions_spawn({ runtime: "acp", agentId: "opencode" })` flow, and current main has source-level regression coverage for an OpenCode ACP timeout config rejection returning `ACP -32602 Invalid params`.

## Solution Assessment

Yes. The current implementation is broader and safer than an OpenCode-only skip list because it treats only optional timeout-control rejections as best-effort while preserving failures for required config options and real backend errors.
