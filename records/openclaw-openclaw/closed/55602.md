## Summary

Current main already solves the PR's central user problem: users can set `logging.file` to a custom directory using the documented `openclaw-YYYY-MM-DD.log` rolling filename pattern, and the logger re-resolves the active dated file on each write. The exact `%DATE%` spelling is not implemented, but the product behavior requested in the PR body, custom log directory plus date-based rolling, is covered without the broader custom-basename pruning risks raised in PR review comments.

## Best Possible Solution

Close this PR and keep the current main implementation: use the documented `openclaw-YYYY-MM-DD.log` rolling filename pattern in any desired directory. If arbitrary custom rolling basenames become a product goal, track that as a new, narrower logging feature with explicit retention/pruning tests.
