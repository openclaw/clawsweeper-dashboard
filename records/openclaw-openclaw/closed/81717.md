## What This Changes

Report asks OpenClaw to auto-discover `sqlite-vec-darwin-arm64/vec0.dylib` under a global Homebrew npm install instead of requiring `memorySearch.store.vector.extensionPath`.

## Best Possible Solution

Keep the current main fallback, ship it in the next release, and route any remaining warning-only cases after that build to the separate misleading-warning diagnostic issue.

## Reproduction Assessment

Yes for the pre-fix failure: the merged PR reproduced the missing-meta-package layout with only a platform sqlite-vec variant installed, and v2026.5.7 source lacks the fallback. Current main source now resolves the same platform-variant path for `darwin-arm64`.

## Solution Assessment

Yes. Resolving the published platform package's exported `vec0` subpath from the OpenClaw module location is the narrow maintainable fix, and it preserves explicit `extensionPath` priority without adding broad filesystem search.
