## Summary

Keep open. Current main still drops model compatibility before OpenAI tool-schema normalization, and #75476 is an open closing PR, so this issue should remain the tracker until that PR lands or is replaced.

## What This Changes

The issue asks OpenAI-compatible provider tool-schema normalization to honor `compat.unsupportedToolSchemaKeywords` and Fireworks `kimi-k2p5-turbo` metadata so rejected JSON Schema keywords such as `not` are stripped before requests.

## Best Possible Solution

Review and land or reject #75476; the durable end state is generic OpenAI tool-schema normalization honoring `ModelCompatConfig`, Fireworks catalog metadata, and regression coverage.

## Reproduction Assessment

Yes, by source inspection. Use an OpenAI-compatible model with `compat.unsupportedToolSchemaKeywords: ["not"]` and a tool schema containing `{ "not": {} }`; current main drops compat before `normalizeToolParameterSchema`, so the keyword remains.

## Solution Assessment

Yes. The narrow maintainable fix is to thread existing model compatibility into OpenAI tool-schema normalization and keep provider-specific unsupported-keyword data in the Fireworks plugin catalog.
