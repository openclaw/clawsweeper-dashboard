## Summary

Keep open. Current main still uses ASCII-only slug normalization in the active group display-name path, so the reported non-Latin group-name loss remains reproducible by inspection. There are also open same-surface PRs with closing syntax, including the reporter's paired PR, so the issue should stay open until one is reviewed and merged or replaced.

## Best Possible Solution

Review and land a focused fix, likely one of the existing open PRs or a maintainer replacement, that preserves Unicode letters/numbers in `normalizeHyphenSlug` and `normalizeAtHashSlug` or otherwise preserves raw group labels in the session display-name path. Add regression tests for CJK, Cyrillic, Arabic, and mixed-script group labels before closing this issue.
