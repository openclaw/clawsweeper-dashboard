## What This Changes

The issue reports that Discord `streaming.mode: "progress"` renders Codex reasoning deltas as replacement snapshots, leaving only the latest tiny reasoning chunk visible.

## Best Possible Solution

Merge raw reasoning delta/snapshot semantics before display formatting, or carry explicit delta/snapshot metadata, then format the stable Discord progress line once while preserving final-answer and tool-progress behavior.

## Solution Assessment

Yes: the narrow maintainable fix is to keep raw reasoning semantics separate from display formatting, with regression tests for both fragment deltas and true full snapshots.
