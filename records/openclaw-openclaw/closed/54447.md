## Summary

Close as superseded. Current main has moved PDF fallback extraction out of the old `src/media/pdf-extract.ts` implementation that this PR edits and into the bundled `document-extract` plugin. The same remaining `standardFontDataUrl` bug is still real, but it is now tracked by #51455 and the newer plugin-level PR #70936, which targets the active code path.

## Best Possible Solution

Close this stale old-path PR as superseded and keep the remaining product fix focused on #70936/#51455. The implementation should land in `extensions/document-extract/document-extractor.ts`, resolve the installed `pdfjs-dist/standard_fonts` directory from the package root, pass a trailing-slash filesystem `standardFontDataUrl` to `getDocument`, and add plugin-level regression coverage.
