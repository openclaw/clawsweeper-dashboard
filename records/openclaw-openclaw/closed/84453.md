## Summary

Keep open: the central bug fix is useful and the PR includes solid CLI proof, but the new dynamic-import state currently treats nested strings inside non-literal import expressions as specifiers and skips real import expressions inside template interpolations, so it needs a focused parser repair before merge.

## What This Changes

The PR replaces the package dist import scanner heuristic with a forward state machine and adds regression tests for static imports, re-exports, dynamic imports, and negative import-like cases.

## Best Possible Solution

Keep the forward-scanner direction, but gate merge on dynamic-import parsing that only records actual literal specifiers and on focused regression plus package/dist validation.

## Reproduction Assessment

Yes. A current-main node probe reproduces the original destructured-import miss, and PR-head probes reproduce both the intended fix and the new dynamic-import edge-case regressions.

## Solution Assessment

No, not yet. The forward scanner is the right place to fix this, but the proposed dynamic import handling needs to distinguish direct literal arguments from expression internals before it is the maintainable solution.

Either scan interpolation expressions or add a narrowly justified unsupported-boundary test.
