## Summary

Current main and the shipped v2026.4.24 release address the Telegram native command menu invisibility symptom through process-local command-sync caching, restart re-registration, payload fitting, and BOT_COMMANDS_TOO_MUCH retry behavior. The linked follow-ups (#66699 and #66950) provided useful evidence for stale cache and payload-overflow variants; current code covers those paths even though earlier logging/cache PRs did not land verbatim.
