## Summary

Keep PR #47232 open. The fix idea is still valid, but the PR is stale: it targets the old `src/tts/tts-core.ts` Edge TTS location while current `main` routes Microsoft/Edge speech through `extensions/microsoft/tts.ts`. Current main still lacks the requested empty-text guard and one-time retry after a zero-byte output, so this is not implemented or superseded.

## Best Possible Solution

Keep the PR open but require a rebase onto the current Microsoft speech plugin: add a whitespace/empty text guard at the top of `extensions/microsoft/tts.ts` before dependency loading or `EdgeTTS` construction, retry `ttsPromise` once when the first output file is zero bytes, use an explicit final error such as `Edge TTS produced empty audio file after retry`, and add regression tests in `extensions/microsoft/tts.test.ts` for whitespace input and a successful retry after an empty first write.
