## Summary

Keep open: this targets a real duplicate `Off` dropdown bug, but the patch is too broad and removes the explicit `Off` override whenever a session inherits a non-off default; the claimed screenshot proof is also not visible in the PR body or comments.

## What This Changes

The PR changes the chat thinking dropdown builder to skip backend `off` levels whenever no session override is set and updates chat UI tests to expect no explicit `Off` entry in inherited states.

## Best Possible Solution

Thread the inherited default level into option building and suppress duplicate `off` only when the inherited default normalizes to `off`, while keeping explicit `Off` for non-off inherited defaults and covering both cases with tests and visible proof.

## Reproduction Assessment

Yes. Current main source and tests show the duplicate `Off` path for default-off options, and the PR regression is source-reproducible because empty `currentOverride` also represents inherited non-off defaults.

## Solution Assessment

No. The patch keys off `currentOverride === ""`, but that means inherit any default; the maintainable fix should compare against the normalized inherited default before hiding explicit `off`.
