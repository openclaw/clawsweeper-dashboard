## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Do not reuse clients with stale request callbacks:** `src/gateway/call.ts:58-59`
  - body: `GatewayClient` stores the `onHelloOk` and `onClose` closures supplied when it is constructed. Returning an existing client here means a later `callGateway` invocation does not wire its own method, params, signal, or promise settlement; reconnects can replay the first request or leave the later call waiting until timeout.
  - confidence: 0.91
- **[P1] Include the full connection context in the pool key:** `src/gateway/call.ts:53-54`
  - body: The key only includes method, token, and password, so calls to a different gateway URL, TLS fingerprint, scope set, approval token, device identity, protocol range, or client mode can reuse the wrong authenticated connection. That crosses the security and routing boundaries that `callGateway` currently resolves per call.
  - confidence: 0.88
- **[P2] Keep OAuth sidecar migration before shadow cleanup:** `src/commands/doctor/repair-sequencing.ts:137-150`
  - body: Current main has a test requiring sidecar OAuth migration to run before stale OAuth shadow cleanup. Running both in `Promise.all` lets the shadow scan observe the pre-migration sidecar shape and skip cleanup, so one `doctor --fix` run can leave the stale shadow behind until a later run.
  - confidence: 0.84
