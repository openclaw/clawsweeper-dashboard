Because this would add a new gateway config surface and startup behavior, it needs maintainer product review before implementation.

## What This Changes

Requests an optional Tailscale Serve service identifier for OpenClaw Control UI exposure while leaving the current device-hostname Serve behavior and Funnel behavior unchanged.

## Best Possible Solution

If maintainers accept the product direction, implement a Serve-only service-name option that preserves current defaults, validates the service identifier, passes the Tailscale service flag only for Serve, updates the displayed URL/docs/schema/help, and proves non-service Serve plus Funnel behavior still work.
