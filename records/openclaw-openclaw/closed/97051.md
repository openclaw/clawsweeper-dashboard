## Summary

Current main already solves the central reported regression: provider/model `agentRuntime.id: "openclaw"` keeps the OpenClaw embedded runtime path, and runtime allowlists containing `read`, `exec`, or `process` now materialize the base and shell tool families instead of dropping them. The fix/proof is current-main-only relative to the latest release context, so release users on v2026.6.10 can still see the old behavior until the next release.

## What This Changes

The issue reports that after `embeddedHarness` deprecation, native OpenAI sessions configured with `agentRuntime.id: "openclaw"` no longer expose allowlisted `read`, `exec`, and `process` tools.

## Best Possible Solution
