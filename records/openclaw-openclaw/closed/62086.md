## Summary

Issue #62086 does not hold as a current OpenClaw CLI regression. The reporter’s own follow-up narrowed it to the exec-tool/runtime wrapper path (`OPENCLAW_SHELL=exec`), and current main code/docs still define that path as a 10s auto-background boundary with `process` follow-up, not a raw CLI timeout.

## Evidence

- **Reporter follow-up narrowed the bug away from the CLI:** In the provided GitHub discussion, the reporter stated on 2026-04-06 that direct container execution (`docker compose exec ... sh -lc 'node dist/index.js config get gateway.mode'`) worked and that the root cause was the exec-wrapper path, not a CLI regression.
  - command: `Provided issue comment by B34RDY on 2026-04-06`
- **Exec default yield window is 10 seconds:** Current main sets the default exec background/yield window to 10,000 ms via `defaultBackgroundMs`, matching the behavior described in the report/comments.

  - command: `nl -ba src/agents/bash-tools.exec.ts | sed -n '1310,1415p'`
