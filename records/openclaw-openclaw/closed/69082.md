## Summary

Current main implements the browser read-then-click path that #69082 reported as broken in 2026.4.14/2026.4.15. Snapshot refs are now preserved across calls and `axN` refs resolve through stored DOM markers instead of falling through to a locator that can wait until timeout. Browser act calls also get a configured/default action timeout with transport slack, so healthy clicks/waits are not cut off at the browser-control request boundary.

## Best Possible Solution

Close as implemented on current main. Keep the shipped ref-preservation and action-timeout behavior; if a newer build still has universal click timeouts, the follow-up should be a fresh bug with gateway logs, Chrome version, snapshot format/ref used, profile driver, and exact act request details.
