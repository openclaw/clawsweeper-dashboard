## Summary

Current main already rebuilds stale `skillsSnapshot` entries for existing sessions when the snapshot version changes, and the skills watcher bumps that version on `SKILL.md` add/change/unlink events. The stale-path described in the issue (`isNewSession || !sessionEntry?.skillsSnapshot`) is no longer the live code.

## Evidence

- **Agent sessions now refresh stale snapshots:** `agent-command.ts` loads the current skills snapshot version, compares it with the cached session snapshot via `shouldRefreshSnapshotForVersion(...)`, and rebuilds/persists the snapshot when stale instead of only rebuilding on new sessions.

## Risks / Open Questions

- There is still no dedicated manual `sessions refresh-skills` CLI path in the inspected tree; that appears to be separate feature work, not the stale-snapshot bug itself.
- The auto-refresh path depends on version invalidation from watched skill roots or other explicit bump sources, so edge cases outside those paths may still need separate follow-up.
