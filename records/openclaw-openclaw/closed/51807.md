## Summary

Current `main` already ships a stronger fix for `config.openFile`: it resolves a platform-specific command/argv tuple, calls `execFile` without shell interpolation, preserves failure reporting, and includes tests for Linux/macOS/Windows quoting and shell-metacharacter paths. This was already present in release `v2026.4.22`, so the open PR is obsolete.
