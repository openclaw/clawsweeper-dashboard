## Summary

Keep open: this maintainer-labeled draft moves in the right direction, but the guard still leaves tokenless remote node hosts able to write OAuth credentials locally and also reimplements loopback detection too narrowly.

## What This Changes

The PR adds an early node-host config check to `openclaw models auth login`, blocks remote-node OAuth writes, adds regression coverage, and updates the changelog.

## Best Possible Solution

Land a focused guard that blocks local OAuth writes whenever the node-host gateway is non-loopback, regardless of stored token, using the shared loopback classifier and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows the PR still returns for `{ gateway.host: remote, token: undefined }`, while current main persists provider OAuth profiles to the local `agentDir`.

## Solution Assessment

No. Blocking remote node-host OAuth login is the right narrow mitigation, but this implementation should remove the token precondition and reuse the shared loopback classifier before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Block remote node hosts without requiring a saved token:** `src/commands/models/auth.ts:620`
  - body: `NodeHostConfig.token` is optional, and node-host auth can come from env/config/password paths while `node.json` still records a remote `gateway.host`. Returning here when the token field is absent leaves that remote-node setup able to run OAuth login and write the refreshed credentials into the node-local `agentDir`, so the bug remains reachable.
  - confidence: 0.93
- **[P2] Reuse the shared loopback host classifier:** `src/commands/models/auth.ts:611-615`
  - body: The new helper only allows empty host, `localhost`, `127.0.0.1`, and `::1`, but OpenClaw already treats other loopback forms such as `localhost.`, 127/8, `[::1]`, and IPv4-mapped loopback as local in `src/gateway/net.ts`. A valid local node-host config using one of those forms would now be rejected as a remote node host.
  - confidence: 0.88
