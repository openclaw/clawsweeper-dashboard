## Summary

Current `main` does not have the reported `/tmp` -> `/private/tmp` symlink-validation bug in the image tool. The local-media validator canonicalizes both the candidate path and each allowed root with `fs.realpath`, so the issue’s claimed `require("fs").realpathSync()`/ESM failure path is obsolete. A plain `/tmp/img.png` rejection on current `main` is explained by the active allowlist policy: the image tool allows the managed temp root (`/tmp/openclaw`) plus workspace/state/config roots, not arbitrary `/tmp`.

## Evidence

- **Validator canonicalizes candidate and roots:** `src/media/local-media-access.ts:44-83` resolves `mediaPath` and every root with `fs.realpath(...)` before checking containment, which covers symlinked paths such as `/tmp` -> `/private/tmp`.
  - file: [src/media/local-media-access.ts:44](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/media/local-media-access.ts#L44)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Image tool uses the shared allowlist, not arbitrary source parents:** `src/agents/tools/image-tool.ts:472-490` passes `mediaLocalRoots` into `loadWebMedia`, and `src/agents/tools/media-tool-shared.ts:376-386` builds those roots from default local roots plus optional workspace root. The third `mediaSources` parameter is unused here, so `/tmp/img.png` is not implicitly widened into an allowed root.
  - file: [src/agents/tools/image-tool.ts:472](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/agents/tools/image-tool.ts#L472)

  - file: [src/media/local-roots.ts:30](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/media/local-roots.ts#L30)
