## Summary

Current `main` already surfaces model failover state: fallback transitions emit lifecycle events, verbose sessions prepend a human-readable `Model Fallback` notice with the active model and reason, and `/status` shows both configured fallbacks and the active fallback model/reason. The implementation landed on `main` at commit `59a8afe6fa030f18af662e125e7dd826ce9a53ca`, which is newer than the latest tagged release `v2026.4.22`, so there is no released tag to cite yet.
