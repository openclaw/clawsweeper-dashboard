## Summary

Current `main` and shipped release `v2026.4.23` already clear `sessionFile` on fresh isolated-session rollover and rotate the transcript path when the `sessionId` changes, matching the heartbeat isolation docs and covering the reported reuse bug with regression tests.

## Evidence

- **Fresh isolated sessions clear persisted transcript path:** `resolveCronSession` now clears `sessionFile` alongside delivery-routing fields whenever `isNewSession` is true, so a forced/fresh isolated session cannot carry the old transcript file forward.
  - file: [src/cron/isolated-agent/session.ts:68](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/cron/isolated-agent/session.ts#L68)

- **Session-file resolver rotates on sessionId change:** `resolveAndPersistSessionFile` now treats `baseEntry.sessionId !== sessionId` as a transcript rotation case and resolves a new path instead of reusing the persisted `sessionFile`.
  - file: [src/config/sessions/session-file.ts:21](https://github.com/openclaw/openclaw/blob/fd65caf4b0bad1b0ac27a82d349c22e92167e7bc/src/config/sessions/session-file.ts#L21)
