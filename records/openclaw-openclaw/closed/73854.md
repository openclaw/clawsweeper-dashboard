## Summary

Keep this PR open. Current main still has the GitHub skill example using `gh pr create --body "Description"`, and the OpenClaw PR maintainer skill only covers comment bodies and heredocs, not the broader `--body-file` guidance this PR proposes. The PR is docs-only and still addresses a real gap in current skill guidance.

## Best Possible Solution

Keep this PR in review. The best path is to land or refine the docs-only update so the GitHub and OpenClaw PR maintainer skills consistently require real multiline body files or heredocs for PR bodies, comments, reviews, issue bodies, and release notes, with a final pass on the temp-file examples.
