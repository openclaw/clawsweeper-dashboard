## Summary

Close as implemented on current main. The runtime catalog now appends explicit `models.providers.<provider>.models[]` entries, preserving `input` metadata, so the existing Gateway image-capability resolver sees configured custom vision models even when discovery did not provide them. The proof commit is on main after `v2026.4.26`, so this is current-main fixed but not yet proven shipped in a release.

## What This Changes

The issue requests that Gateway runtime image-capability resolution honor `models.providers[provider].models[].input` for custom models missing from built-in discovery/catalog metadata.

## Best Possible Solution

Keep the current catalog-level implementation, because it makes explicit provider model definitions first-class runtime catalog rows without adding provider-specific special cases to the Gateway resolver. Leave the separate onboarding/default-modality enhancement tracked by #61716 for any remaining UI or setup flow work.
