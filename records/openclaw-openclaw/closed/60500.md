## Summary

Close as implemented on main. Current main now covers this PR’s PDF-specific goal as part of broader WebChat/Control UI non-video file attachment support: the UI accepts PDFs, serializes non-image uploads as files, Gateway accepts and offloads them, routes them through MediaPaths/MediaTypes for the agent, and the PDF extraction path is backed by document extractors. The original PR branch should not be merged as-is because review found an unrelated lockfile hunk and an obsolete marker-only PDF path.

## Best Possible Solution

Close this PR and keep the current main implementation. Main implements the intended PDF upload support through the broader non-video attachment pipeline with staging, media-understanding hooks, tests, and changelog coverage; future work should be tracked against narrower remaining document-format gaps rather than this stale branch.
