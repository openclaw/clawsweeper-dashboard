## Summary

The proposal is useful, but current OpenClaw scope puts optional security/integration capabilities like this in the plugin/ClawHub layer rather than in core. The issue already points to a working external implementation (`aip-agent-guard`), and `main` documents ClawHub/plugin distribution as the preferred path for this kind of feature.

## Evidence

## Risks / Open Questions

- This close decision is scope-based, not a claim that the underlying security problem is solved in core.
