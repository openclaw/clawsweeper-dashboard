## Summary

Keep open: the report has concrete Discord and Cove message-loss evidence on a v2026.6.9 build, while current main only shows adjacent post-tool empty-turn guards and narrower Discord draft-order tests, not exact proof that a non-zero exec progress event can no longer suppress a produced final reply.

## What This Changes

The issue asks OpenClaw core to deliver the final channel reply even when an exec/tool progress event reports a non-zero exit code as failed.

## Best Possible Solution

Prove the current-main `false` or exit-1 exec path with a focused runner/dispatch regression, then fix the core runner or reply-dispatch boundary so failed progress cannot satisfy or suppress final assistant delivery while preserving intentional silence.

## Reproduction Assessment

No high-confidence current-main reproduction yet. The reporter supplied strong v2026.6.9 Discord and Cove evidence, but this read-only review did not run a live or focused current-main repro and existing tests do not exercise the exact failed-exec-to-final-delivery path.
