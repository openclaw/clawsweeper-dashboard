## Summary

Current `main` already addresses this stale plugin-registry/hot-reload problem with a broader architecture: gateway startup pins the HTTP-route and channel registries, webhook registration/dispatch resolve through those pinned surfaces, and config schema loads no longer reactivate plugin runtime state. That supersedes this PR’s channel-lifecycle rebinding approach.

## Evidence
