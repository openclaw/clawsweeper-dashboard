## Summary

Current main already covers the PR's user-visible fix: explicit `/new` and `/reset` chat session rollovers create a fresh session entry without carrying stored CLI backend session IDs, and regression tests verify both modern `cliSessionBindings`/`cliSessionIds` and legacy `claudeCliSessionId` are cleared while user-selected model/auth overrides remain. The fix is also present in the v2026.4.24 release tree and documented for CLI backends.

## Best Possible Solution

Close this PR as already implemented on main. Keep the shipped session-initialization/reset behavior and its focused regression coverage; do not merge the older PR branch or its larger now-redundant agent-command test patch.
