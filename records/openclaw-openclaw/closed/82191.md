## Summary

Current main already fixes the reported v2026.5.12 regression by trusting the exact configured custom/local provider `baseUrl` origin while keeping broader SSRF protections in place. The proof commit is on `main` and not contained in `v2026.5.12`, so this is a current-main-only fix pending release.

## What This Changes

The issue asks OpenClaw to restore model-provider requests to Tailscale/CGNAT `baseUrl` endpoints after v2026.5.12 blocked them unless `request.allowPrivateNetwork` was set.

## Best Possible Solution

Keep the current exact-origin provider `baseUrl` trust behavior and ship it in the next release; broader CGNAT allowlisting or automatic private-network migration is unnecessary for the reported configured-origin case.

## Reproduction Assessment

Yes for the reported release regression, using a custom provider `baseUrl` on a Tailscale/CGNAT IP and then making a model call. On current main, the same path is not expected to reproduce because the configured custom/local origin is converted into a narrow SSRF allow policy.

## Solution Assessment

Yes. The current main fix is the narrowest maintainable solution because it trusts only the configured `scheme://host:port` origin and keeps different ports, metadata, and link-local pivots blocked.
