## Summary

Current main already implements this PR’s intended LINE inbound media persistence. `downloadLineMedia` now saves the downloaded buffer through the shared `saveMediaBuffer(..., "inbound", ...)` media-store path, passes that saved path to LINE message handling, and has focused tests and docs coverage. The main implementation is also stricter than the PR diff because it avoids the temp-file fallback and propagates media-store failures.

## Best Possible Solution

Close this PR as implemented on main and keep the shipped `downloadLineMedia` implementation, which persists LINE inbound media through the shared media store and propagates persistence errors instead of silently falling back to temporary files.
