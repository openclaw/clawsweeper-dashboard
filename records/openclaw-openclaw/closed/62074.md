## Summary

Current main already routes root-help/CLI metadata through a dedicated non-activating plugin metadata path, skips bundled full runtime entries during metadata capture, and ships that behavior in v2026.4.23, so the reported build-time plugin startup problem is already addressed.

## Evidence

- **Root help now uses plugin CLI descriptors instead of full runtime registration:** `renderRootHelpText()` builds root help from `getPluginCliCommandDescriptors(...)`, which is the metadata-only descriptor path.
