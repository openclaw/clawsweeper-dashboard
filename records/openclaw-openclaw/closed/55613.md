## Summary

Current main has already removed the risky `skills list` loading path from `v2026.3.24`. The old code synchronously `readFileSync`'d every discovered `SKILL.md`; main now routes those reads through guarded helpers that reject symlinked/non-regular paths and added regression coverage. The fix landed in commit `b7b46ad185392f53a995124f6861d2b356f84976` and is included in release tag `v2026.3.31`.

## Evidence

- **Old blocking read path in v2026.3.24:** The `v2026.3.24` snapshot built `skillEntries` by calling `fs.readFileSync(skill.filePath, "utf-8")` directly for every discovered skill. That is the exact `openclaw skills list` path and can hang on malformed/non-regular `SKILL.md` targets.

  - sha: [cff6dc94e307](https://github.com/openclaw/openclaw/commit/cff6dc94e30794a269eb7805b6e636c3634a088c)
- **Main uses guarded frontmatter reads:** Current main no longer does a raw read there; it calls `readSkillFrontmatterSafe(...)` while mapping merged skills, so unsafe paths are skipped instead of synchronously read.
  - file: [src/agents/skills/workspace.ts:600](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/agents/skills/workspace.ts#L600)
