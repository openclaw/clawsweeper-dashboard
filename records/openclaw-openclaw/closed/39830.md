## Summary

Current main no longer uses the old standalone reset-notice flow that could inject an assistant-only message before the first post-`/new` user turn. The present `/new` path is covered by tests that assert no routed reset notice is sent and that bare `/new` still proceeds into the normal startup greeting flow.

## Evidence
