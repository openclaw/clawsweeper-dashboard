## Summary

Current `main` already handles the reported WhatsApp MP3/PTT bug class by transcoding non-native audio to Ogg/Opus before sending the PTT payload, and the exact MP3 send path is covered by tests and docs.

## Evidence

- **Outbound send path normalizes media before WhatsApp delivery:** `sendMessageWhatsApp` now routes outbound media through `prepareWhatsAppOutboundMedia(...)` before calling the active WhatsApp sender, so the CLI `message send --media` path no longer forwards raw MP3 bytes unchanged.
