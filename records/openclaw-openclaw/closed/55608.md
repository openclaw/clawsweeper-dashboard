## Summary

Current `main` already routes `openclaw memory status` and `openclaw memory search` through the resolved runtime config, then merges `agents.defaults.memorySearch` into the actual provider/model/remote request path. The reported CLI divergence is no longer present, and the same implementation is already in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Evidence

- **CLI now loads resolved memory config before status/search:** `extensions/memory-core/src/cli.runtime.ts:93-100` defines `loadMemoryCommandConfig()` using `loadConfig()` plus `resolveCommandSecretRefsViaGateway(...)`. `runMemoryStatus()` and `runMemorySearch()` call that helper at `extensions/memory-core/src/cli.runtime.ts:667-669` and `extensions/memory-core/src/cli.runtime.ts:1170-1178`, so the CLI no longer bypasses config resolution.
