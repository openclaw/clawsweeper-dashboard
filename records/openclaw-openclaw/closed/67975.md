## Summary

Keep open. Current `main` still defaults the ACPX startup probe to `codex` unless `plugins.entries.acpx.config.probeAgent` is set explicitly; there is no current code path or test coverage that derives the probe agent from `acp.allowedAgents`, so the issue's `allowedAgents` half is not implemented on the checked-out `main`.

## Evidence

- **Runtime still hardcodes codex as fallback probe agent:** `resolveProbeAgentName` only reads `options.probeAgent` and otherwise falls back to `"codex"`; it does not inspect `acp.allowedAgents` or any allowlist-derived value.
  - file: [extensions/acpx/src/runtime.ts:190](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/extensions/acpx/src/runtime.ts#L190)
