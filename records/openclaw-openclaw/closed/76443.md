## Summary

Keep open. Current main still has a source-visible `sessions_send` path where an accepted target run can return `timeout` before launching the existing A2A follow-up, and the report also raises sender-session visibility that is not fully covered by the related timeout-only work.

## What This Changes

The issue requests that cross-agent `sessions_send` replies be delivered back to the sender session instead of only appearing in the target agent session or inline tool result.

## Best Possible Solution

Consolidate this with the existing timeout/A2A work so accepted target runs preserve or launch the sender follow-up path, including late replies, and add regression coverage for sender-session visibility.

## Reproduction Assessment

## Solution Assessment
