## Label Justifications

- `P1`: The original bug could break active agent/channel workflows by blocking compaction and causing context-limit resets.
- `impact:session-state`: The failure mode was inconsistent compaction/session state where persisted transcript content was not represented in the compaction input.
- `impact:data-loss`: The reported resets and placeholder summaries could drop or fail to preserve useful conversation history.

## Summary

Close: current main and v2026.6.5 now handle the reported empty-preparation/no-compactable compaction path with branch-message fallback, harmless preflight skips, stale-token snapshot reset, and regression coverage.

## What This Changes

The issue reports compaction guards treating empty or unpopulated message arrays as no real conversation, causing budget/manual compaction to skip and sessions to hit context-limit resets instead of summarizing.

## Best Possible Solution

Keep the v2026.6.5+ compaction behavior and ask for a new report only if fresh logs on that release still show real transcript content being skipped.

## Reproduction Assessment

No current-main failing reproduction remains from source review; the historical failure path is well supported by multiple issue comments and current regression tests now cover the relevant no-compactable and stale-token cases.

## Solution Assessment

Yes. The implemented path addresses the maintainable core failure classes directly: recover real branch messages when preparation is empty, skip harmless no-compactable preflight results, and clear stale token accounting before retrying.
