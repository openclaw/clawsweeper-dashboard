## Summary

Current `main` already resolves `contextWindow` by explicit provider+model instead of sharing a merged value across providers, and it has regression tests covering both provider-specific context lookup and matching fallback model metadata. The fix appears to be on `main` after the latest tagged release, so there is no confirmed shipped release tag yet.

## Evidence

- **Provider-specific context lookup in current main:** `resolveConfiguredProviderContextTokens` scans `cfg.models.providers` for the explicitly selected provider and exact model id before any shared cache lookup, then `resolveContextTokensForModel` returns that configured window first when `provider` is explicit.
