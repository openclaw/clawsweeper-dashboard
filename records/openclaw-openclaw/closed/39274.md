## Summary

Current main no longer blocks `MESSAGE_CREATE` on the Carbon listener path: `DiscordMessageListener.handle()` fire-and-forgets the work, timeout ownership is in the queued inbound worker, and regression tests cover immediate return/concurrent dispatch. The changelog records the Discord inbound timeout isolation fix in `2026.3.13`, so the 80s listener-timeout report against `2026.3.2` is obsolete on main.

## Evidence
