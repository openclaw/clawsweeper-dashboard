## Summary

Keep open. Current main still has the slash-form alias bypass, the linked issue #73573 is still open, and the PR remains a focused implementation candidate rather than obsolete work. The diff is limited to model-selection source and tests, with no security- or supply-chain-sensitive files touched.

## Best Possible Solution

Review and land this PR or an equivalent patch that checks the alias index before provider/model parsing in the shared model-selection path, keep the focused regression tests, and let #73573 close through the merge. If slash-form aliases are intended globally, add or follow up with coverage for the separate subagent-spawn alias helper.
