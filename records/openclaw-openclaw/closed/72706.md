## Summary

Current main already handles the reported Telegram partial-stream ghost deletion path: text-only finals edit the existing preview when possible, and fallback delivery sends the completed final before deleting the old preview. The valid streaming modes remain `off | partial | block | progress`; `final` is not needed for this fix. The behavior is present in v2026.4.26 and current main.

## Best Possible Solution

Keep the shipped Telegram preview finalization behavior: edit the same preview message for normal text finals, send the completed final before cleaning up stale/complex previews, and leave any broader single-preview coalescing changes to the existing Telegram coalescing follow-up rather than changing the default to an undocumented `final` mode.
