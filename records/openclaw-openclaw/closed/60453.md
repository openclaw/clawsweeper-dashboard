## Summary

This report is about the external `openclaw-gmail` channel, not an in-tree OpenClaw channel. Current main has no Gmail chat channel or `channels.openclaw-gmail` surface, while core already has generic plugin channel config-schema and status APIs. The fix belongs in the Gmail plugin/ClawHub path: publish a valid `channelConfigs.<id>.schema` for Control UI and wire account configured/running status through the channel plugin status/lifecycle hooks.

## Best Possible Solution

Keep OpenClaw core generic. The `openclaw-gmail` plugin should be fixed or published through ClawHub with a valid manifest `channelConfigs` schema for `channels.openclaw-gmail` and a status adapter/lifecycle implementation that reports each account's configured and running state through the existing SDK hooks. Only open a core issue if that documented plugin API is insufficient.
