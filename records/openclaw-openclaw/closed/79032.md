## What This Changes

The issue requests that successful sub-agent completion output reliably return to the parent agent instead of failing after Gateway WebSocket announce-back timeouts.

## Best Possible Solution

Keep the in-process same-process handoff on main and leave broader Gateway timeout tuning to the separate transport/status trackers.

## Reproduction Assessment

No, not as a failing current-main path. The original source path and logs were clear, and current main now exercises the fixed path with live coverage that forbids the loopback `agent` RPC.

## Solution Assessment

Yes. Routing same-process subagent completion handoffs through the in-process Gateway dispatcher is narrower and more maintainable than extending loopback WebSocket timeouts for this path.
