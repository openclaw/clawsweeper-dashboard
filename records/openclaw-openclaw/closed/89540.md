## Summary

Keep open: current main still has a credible Telegram message-delivery bug where a trailing verbose Active Memory diagnostic is appended as a second final payload and can be handled as answer text after the assistant reply has already finalized.

## What This Changes

The issue requests that Telegram keep the assistant answer visible when `/v on` emits trailing Active Memory/plugin diagnostic status.

## Best Possible Solution

Keep the issue open and fix the existing final-status boundary: mark inline plugin diagnostics as status notices and have Telegram send post-answer final status notices as separate durable messages without editing the finalized answer preview.
