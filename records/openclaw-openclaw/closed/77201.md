## Summary

Keep open: this maintainer-labeled PR is still useful, but the latest head still has a P2 correctness issue where search filtering can make non-consecutive duplicate transcript messages collapse as if they were adjacent.

## What This Changes

The PR refines Control UI chat session controls, responsive chat layout, duplicate-message display, docs, changelog, and related tests.

## Best Possible Solution

Preserve original transcript adjacency for duplicate counts, add a search-filter regression test, then continue normal maintainer review of the responsive Control UI changes.

## Reproduction Assessment

Yes for the review finding: source inspection of the latest PR head shows `searchOpen` removes non-matching messages before duplicate collapse runs. A focused regression would use two identical text-only messages separated by a non-matching message while search is open.

## Solution Assessment

No. The shared-control and responsive-layout direction is maintainable, but duplicate collapse should run against original transcript adjacency or carry a reset marker through search filtering so search results cannot fabricate consecutive duplicates.
