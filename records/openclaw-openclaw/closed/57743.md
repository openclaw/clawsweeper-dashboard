## Summary

Current `main` already hard-times out wedged Telegram `getUpdates` requests and rebuilds the transport on polling/network recovery, which addresses the reported 90-110s stuck-poll behavior.

## Evidence

- **Hard timeout for getUpdates:** Telegram request timeouts now assign `getupdates` a 45,000 ms hard timeout instead of waiting on undici/socket defaults.
  - file: [extensions/telegram/src/request-timeouts.ts:6](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/extensions/telegram/src/request-timeouts.ts#L6)

  - file: [extensions/telegram/src/bot-core.ts:181](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/extensions/telegram/src/bot-core.ts#L181)
