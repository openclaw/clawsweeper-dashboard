## Summary

Current `main` already resolves direct `google/*` Gemini ids through the bundled Google provider’s runtime model hook, and Google auth now comes from plugin-manifest env var metadata rather than the old core implicit-provider special case. The requested behavior is therefore already implemented and shipped.

## Evidence

- **Core resolution now defers to provider runtime hooks before generic fallback:** `resolveModelWithRegistry()` first asks provider runtime hooks for a dynamic model and only falls back to generic configured-provider behavior afterward, so provider-owned Google handling is on the main path.
