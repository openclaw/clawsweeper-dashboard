## Summary

Keep open: current source shows a credible config-write validation mismatch where CLI dry-run schema validation still uses bundled channel schemas directly, while plugin-aware writes can resolve external `channelConfigs` ownership for the same channel.

## What This Changes

Allow OpenClaw config writes to accept openclaw-lark-owned `channels.feishu.replyMode` and `footer` fields instead of failing bundled Feishu strict-schema validation.

## Best Possible Solution

Unify config write preflight around plugin-aware channel schema resolution, then cover external Feishu ownership with `replyMode` and `footer` regression tests while preserving strict bundled Feishu validation when no external owner is active.

Yes, source inspection gives a high-confidence reproduction path for the validation mismatch: a JSON config patch dry-run under `channels.feishu` is checked with bundled Feishu validation that ignores external plugin `channelConfigs`. A live command was not run in this read-only review.

## Solution Assessment

Yes, the narrowest fix is to make config patch/apply preflight use the same plugin-aware channel metadata path already used by persisted writes, without importing plugin runtime code or relaxing bundled schemas globally.

## Repair Work Prompt

Fix the config validation mismatch reported in #77887. Source inspection shows `openclaw config patch --dry-run` routes schema checks through `collectDryRunSchemaErrors` in `src/cli/config-cli.ts`, which calls `validateConfigObjectRaw(..., validateBundledChannels: true)` and therefore applies the generated strict bundled Feishu schema instead of plugin-aware `channelConfigs` ownership. Update the CLI config patch/set dry-run validation, and any adjacent config apply/patch preflight that has the same raw bundled-channel behavior, to use plugin-aware channel schema resolution without importing plugin runtime code. Preserve SecretRef policy checks and keep bundled Feishu strict when no external owner is active. Add regression coverage with a mocked external `openclaw-lark`/configured plugin manifest owning `channels.feishu` via `channelConfigs.feishu.schema` and accepting `replyMode` plus `footer`; also assert the bundled-only Feishu schema still rejects unknown keys. Add a user-facing changelog fix entry. Do not change external plugin package metadata, bundled Feishu runtime behavior, or global JSON schema strictness beyond the plugin-owned channel validation path.
