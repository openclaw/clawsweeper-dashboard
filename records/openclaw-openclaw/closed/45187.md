## Summary

Current main already addresses both reported paths: the chat view shows a loading skeleton instead of the welcome/empty-state while history is loading, constrains the logo/avatar sizing so it cannot fill the viewport, and clears/flushed queued chat state on terminal events and reconnect recovery.

## Evidence

- **Loading path no longer shows the welcome empty-state:** `renderChat` computes `isEmpty` as `chatItems.length === 0 && !props.loading`, renders `.chat-loading-skeleton` when `props.loading` is true, and only renders `renderWelcomeState(props)` when `isEmpty` is true. That prevents the empty-state/logo from appearing during chat-history loading.
