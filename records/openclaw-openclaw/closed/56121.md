## Summary

Current `main` does not show a full fix for the reported cron run-session lifecycle bug. Isolated cron runs still create a distinct `:run:` session key, but lifecycle persistence is wired to the base cron session key, and the shared persist helper writes the in-memory session entry back to both keys without merging terminal lifecycle fields. The secondary unbounded-growth complaint is partly mitigated on `main` because old cron run-session entries are now reaped from `sessions.json` by age.

## Evidence
