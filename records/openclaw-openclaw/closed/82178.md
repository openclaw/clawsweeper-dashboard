## What This Changes

The issue asks maintainers to decide whether finite-timeout `exec` backgrounding should remain available when the `process` follow-up tool is hidden by policy.

## Best Possible Solution

Decide the restricted-tool policy on the linked PR; if accepted, land a bounded patch that preserves finite timeout safety, provides observable completion without `process`, and keeps both docs pages and regression tests aligned.

## Reproduction Assessment

Yes for the current-main behavior: source, docs, and an existing test show that denying `process` makes `exec` run synchronously despite `yieldMs`. This is not a strict bug reproduction because current docs define that behavior today.

## Solution Assessment

Unclear until maintainer review: the linked PR is the focused implementation path, but it changes a documented restricted-tool contract and still needs the completion-visibility and docs-sync concerns resolved before it is the best solution.
