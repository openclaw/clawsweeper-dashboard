## Summary

Keep open. Current main still maps each chat-only projection snapshot to `data.delta`, while this PR changes the SDK fallback to preserve full `text` and emit only the incremental `delta`; I did not find a blocking code-review or security finding in the patch.

## What This Changes

The PR updates SDK chat-only projection normalization to emit `text` plus incremental `delta` or `replace` data, and extends the SDK unit test for append and replacement chat frames.

## Best Possible Solution

Land the narrow SDK normalization fix after final CI, keeping Gateway chat projections as full buffered chat frames while making `Run.events()` expose canonical assistant-shaped data for app clients.

## Reproduction Assessment

Yes. A high-confidence repro is the fake SDK transport scenario that emits chat-only deltas with `hello` followed by `hello again`; current main exposes both as appendable `data.delta`, while the docs and live e2e consumer append `delta`.

## Solution Assessment

Yes. The SDK per-run normalization boundary is the right place to compute incremental fallback deltas because raw Gateway chat frames remain available through `rawEvents()` and chat projection semantics do not need to change.
