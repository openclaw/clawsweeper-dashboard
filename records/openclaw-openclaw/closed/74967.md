## Summary

Close as duplicate/superseded by the older open Discord role-mention tracker. Current main still has the missing role-mention activation behavior, but #7717 already tracks the same remaining Discord preflight/config work, and #59151 was previously closed in favor of that canonical item.

## What This Changes

Report asks Discord auto-reply to treat the bot's own managed role mention as a bot-addressing mention when Discord autocomplete inserts a role mention instead of a user mention.

## Best Possible Solution

Keep #7717 as the canonical implementation tracker and make sure that fix explicitly covers the bot-managed-role autocomplete case from this report. The implementation should stay in the bundled Discord plugin, preserve the existing sender `roles` allowlist semantics, and decide there whether bot-managed roles are automatic wake mentions or configured trigger roles.

## Reproduction Assessment

Yes. The issue’s Discord payload shape maps directly to current source: `mentions[]` empty, `mention_roles[]` populated, and `requireMention` true leaves `wasMentioned` false and reaches the `no-mention` drop path.

## Solution Assessment

Yes. Closing this duplicate is the best cleanup because #7717 already owns the broader role-mention trigger design; fixing only this issue separately would risk splitting the same Discord mention semantics across two trackers.
