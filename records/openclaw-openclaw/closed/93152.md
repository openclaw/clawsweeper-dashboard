## Best Possible Solution

Use the actual workspace or project boundary for ancestor context discovery, preserve nested in-workspace context files, exclude parent/home/system files, and add focused regression coverage for those cases.

## Reproduction Assessment

Yes from source inspection: with a workspace below the user home, the PR’s condition still allows the loop to load `~/AGENTS.md`, one of the linked report’s outside-workspace examples. I did not run a live agent session because this review was read-only.

## Solution Assessment

No; the home-directory cap is a partial mitigation, not the best fix for a workspace-boundary issue. The safer fix is to pass or derive the real workspace/project root and stop the walk there.
