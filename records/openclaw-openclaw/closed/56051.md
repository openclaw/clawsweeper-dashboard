## Summary

Current `main` already preserves account-scoped requester routes for multi-account agent sessions and delivers subagent/session completions back through the external requester route, so the reported Discord multi-bot possession problem is implemented without adding a new `agents.list[].discord.accountId` field.

## Evidence

- **Documented supported model: account-scoped bindings per channel account:** The multi-agent docs explicitly describe `accountId` as the channel-account selector, note that `accountId` bindings route one channel account to one agent, and show Discord bots-per-agent using `bindings: [{ agentId, match: { channel: "discord", accountId } }]` rather than an agent-level Discord field.
  - file: [docs/concepts/multi-agent.md:243](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/concepts/multi-agent.md#L243)
