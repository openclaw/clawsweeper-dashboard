Full review comments:

- **[P1] Remove the broad workspace file reader:** `src/gateway/server-methods/agents.ts:478-488`
  - body: The new `agents.files.read` method accepts any relative path under the agent workspace, follows it through `fs.stat`, then reads it directly. Existing `agents.files.get/set` deliberately restrict reads/writes to known bootstrap and memory filenames and use fs-safe/lstat checks, so this would expose arbitrary workspace files or symlink targets through the gateway.
  - confidence: 0.92
- **[P2] Keep the existing agent file editor:** `ui/src/ui/views/agents.ts:1491-1507`
  - body: The Agents panel currently lets users edit and save bootstrap/memory files. This change replaces that section with a read-only markdown preview and removes the Save/Reset/textarea controls, so an existing Mission Control editing workflow disappears as part of an unrelated Slack fix.
  - confidence: 0.86
