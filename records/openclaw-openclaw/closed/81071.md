## Summary

Keep open: the hardening is plausible and mostly narrow, but this PR has a protected maintainer label, lacks external real behavior proof, and still leaves the public node exec-event contract out of sync.

## What This Changes

The PR adds gateway-issued node `system.run` run IDs, records authorized run provenance in `NodeRegistry`, rejects unmatched node exec lifecycle events, preserves run IDs in the macOS node, and adds focused TypeScript and Swift tests.

## Best Possible Solution

Keep the narrow registry-backed provenance gate, document the node exec-event contract in the public gateway protocol docs, and require redacted live or terminal gateway-node proof before maintainer merge.

## Reproduction Assessment

Yes. Source inspection and current tests show current main can enqueue node exec lifecycle payloads without a pending gateway `system.run` provenance check; I did not establish a live gateway/node replay.

## Solution Assessment

Mostly yes. Binding accepted lifecycle events to the issuing node connection, run id, and session in `NodeRegistry` is the narrow maintainable fix, but the public docs and real behavior proof still need to be completed before merge.
