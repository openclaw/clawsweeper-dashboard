## Summary

Keep open. Current main still has the reported Bedrock thinking-profile gap, so the PR is useful, but the patch should not merge as-is because it hand-rolls an Opus 4.7 profile that remains out of sync with the shared Claude contract: current main’s shared helper and docs include `xhigh`, `adaptive`, and `max` for Claude Opus 4.7, while this branch only exposes `xhigh` and `adaptive`.

## What This Changes

The PR updates the Amazon Bedrock provider thinking profile and adds a Bedrock plugin test so Claude Opus 4.7 Bedrock model IDs advertise `xhigh` and `adaptive` without defaulting to adaptive.

## Best Possible Solution

Align the Bedrock provider with the shared Claude thinking-profile contract, preferably by reusing `resolveClaudeThinkingProfile` through a small Bedrock model-id normalization adapter. The regression test should assert Opus 4.7 exposes `xhigh`, `adaptive`, and `max`, keeps `defaultLevel` aligned with the shared helper, and preserves the existing Claude 4.6 adaptive default.
