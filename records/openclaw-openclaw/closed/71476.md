## Summary

Current main already covers the reported WebChat/Control UI HTTP 401 failure mode: protected Control UI HTTP reads accept paired operator device tokens, the browser prefers the WebSocket hello device token for avatar/media fetches, and bootstrap config fetches retry alternate credentials on 401. The same relevant implementation is present in v2026.4.24, while the report was against v2026.4.22.

## Best Possible Solution

Close this item as implemented on main and in v2026.4.24. Keep the shipped HTTP-auth/device-token behavior; leave the broader compaction/session-rotation problem to the separate canonical tracker #70472 if it still reproduces independently of these 401s.
