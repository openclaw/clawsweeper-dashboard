## Summary

Current main already validates `config.patch`/`config.apply` before persisting config, and the shipped `v2026.4.22` release contains the same behavior, so the reported crash-loop path is already addressed.

## Evidence

- **`config.patch` validates merged config before write:** The gateway handler parses the patch, merges it with the current snapshot, restores redacted values, runs `validateConfigObjectWithPlugins`, and returns `INVALID_REQUEST` on validation failure before any write. `writeConfigFile(...)` happens only after validation succeeds.
