## Summary

Keep open. Current main still resolves outbound transcript mirror appends through the default session store when no explicit storePath is supplied, while this PR moves the central fallback to the existing configured session.store and adds focused custom-store coverage; no blocking code findings surfaced, but the external PR still needs real behavior proof before merge.

## What This Changes

The PR changes session transcript append fallback resolution to honor config.session.store when storePath is absent, threads canonical agent ownership into transcript persistence and hook context, and adds focused regression tests.

## Best Possible Solution

Land this PR or an equivalent narrow patch after redacted real behavior proof shows an outbound mirrored assistant reply written to the configured session.store while explicit storePath priority remains intact.

## Reproduction Assessment

Yes. The current-main source path is enough to reproduce by seeding a configured session.store, calling the outbound mirror append without storePath, and observing the fallback resolve the default store instead of the configured store.

## Solution Assessment

Yes. Fixing the transcript append fallback is the narrowest maintainable owner-boundary fix because the outbound callers already pass runtime config, while adding storePath plumbing to each caller would duplicate session-store resolution.
