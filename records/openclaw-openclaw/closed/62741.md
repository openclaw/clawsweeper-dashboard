## Summary

Current main already implements the intended defensive behavior for memory-lancedb embeddings. The OpenAI-compatible embedding path no longer directly dereferences `response.data[0].embedding`; it optional-chains the response and routes missing or malformed vectors through `normalizeEmbeddingVector`, which throws a descriptive error covered by tests.

## Best Possible Solution

Close this PR as implemented on current main and keep the shipped normalization path, which handles empty or missing embedding vectors more generally than the proposed bounds check.

## Evidence
