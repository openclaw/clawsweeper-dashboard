## Summary

Keep open: current main still lets persisted auth-profile `api_key` and `token` strings pass through profile resolution with only trimming before runtime auth injection, and Node still throws the reported ByteString error for a Bearer token containing U+2026.

## What This Changes

The issue asks OpenClaw to stop subagent startup from crashing when an agent-scoped `auth-profiles.json` stores a masked API key containing U+2026.

## Best Possible Solution

Normalize or reject resolved persisted profile credentials at the auth-profile resolution boundary, cover locked writes too, and add regression tests proving U+2026 cannot reach runtime Authorization header construction.

## Reproduction Assessment

Yes, by source inspection plus a direct Node header check. Current main can carry a stored U+2026 profile key through `resolveApiKeyForProfile` into runtime auth injection, and Node throws the reported ByteString error for that header value.

## Solution Assessment

Yes: the narrow maintainable fix is to reuse the existing secret normalizer at auth-profile credential resolution/read and locked upsert boundaries, not add OpenRouter-specific handling or a new config option.
