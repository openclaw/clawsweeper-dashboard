## Summary

Current `main` already routes `claude-cli` session usage through fresh context-snapshot accounting instead of cumulative run totals, and the Control UI notice renders `totalTokens` from that snapshot rather than inflated raw input totals. The reported multi-turn inflation from `2026.4.19-beta.1` is covered by current code and tests on `main`.

## Evidence

- **CLI runs persist fresh context snapshots, not cumulative usage:** `runReplyAgent` passes both `lastCallUsage` and `usageIsContextSnapshot: isCliProvider(...)` into session persistence. `persistSessionUsageUpdate` then prefers `lastCallUsage` for `totalTokens` and latest cache counters, with comments explicitly calling cumulative per-call usage an overstatement of true context.
