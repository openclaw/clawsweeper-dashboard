## Summary

Close: current main already prevents `EmbeddedAttemptSessionTakeoverError` and session write-lock errors from walking the model fallback chain; the fix is present on main but not in the latest tagged `v2026.5.18` release.

## What This Changes

The issue asks OpenClaw to stop treating a local embedded session-file takeover race as a provider failure that consumes cross-provider fallback candidates.

## Best Possible Solution

Keep the current main behavior that classifies these as local coordination errors and prevents provider fallback consumption; consider a separate targeted issue only for bounded same-candidate retry semantics.

## Reproduction Assessment

No for current main: source and regression coverage show the takeover error is rethrown before the fallback loop can try another model. The `v2026.5.18` source lacks that guard, which matches the release behavior described in the report.

## Solution Assessment

Yes: the current main fix addresses the core bug by preventing a local session coordination error from consuming provider fallback candidates. Retrying the same candidate after settle would be a separate enhancement rather than required to close this fallback-chain issue.
