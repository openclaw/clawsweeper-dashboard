## Summary

Current `main` already persists provider auth failures into auth-profile unavailable state and the model-fallback loop skips providers whose profiles are unavailable for `auth`/`auth_permanent`, which addresses the reported Moonshot 401 fallback loop.

## Evidence

  - file: [src/agents/auth-profiles/usage.ts:559](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/agents/auth-profiles/usage.ts#L559)

## Close Comment

Closing this as implemented after Codex review.

Current `main` already persists provider auth failures into auth-profile unavailable state and the model-fallback loop skips providers whose profiles are unavailable for `auth`/`auth_permanent`, which addresses the reported Moonshot 401 fallback loop.

What I checked:

- **Embedded runner persists auth failures:** Non-timeout failover reasons are converted into auth-profile failure state and written via `markAuthProfileFailure`, so a 401-classified auth failure is not treated as a one-off transient. (`src/agents/pi-embedded-runner/run.ts:633`)
- **Auth failures become durable unavailable state:** `markAuthProfileFailure` records failure counts and cooldown/disable windows; billing and `auth_permanent` use the disabled lane, while `auth` still records a profile-wide unavailable cooldown. (`src/agents/auth-profiles/usage.ts:559`)
- **Fallback skips auth-broken providers:** When all profiles for a candidate provider are unavailable and the inferred reason is `auth` or `auth_permanent`, fallback returns `skip` for that provider instead of probing it again. (`src/agents/model-fallback.ts:663`)
- **Skipped providers are removed from the current fallback pass:** The fallback loop records a `skip_candidate` attempt and continues to the next provider/model candidate, preventing immediate switch-back to the failing provider in the same pass. (`src/agents/model-fallback.ts:780`)
- **Release tag already contains the auth-skip logic:** The `v2026.4.23` tag contains the same `auth`/`auth_permanent` skip logic in [`src/agents/model-fallback.ts`](https://github.com/openclaw/openclaw/blob/9b18ae877018/src/agents/model-fallback.ts), so this is not only on current `main`. ([`src/agents/model-fallback.ts`](https://github.com/openclaw/openclaw/blob/9b18ae877018/src/agents/model-fallback.ts), [9b18ae877018](https://github.com/openclaw/openclaw/commit/9b18ae877018))

So I’m closing this as already implemented rather than keeping a duplicate issue open.
