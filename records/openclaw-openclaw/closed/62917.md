## Summary

Keep this issue open. Current main still resolves exec approvals JSON and socket paths from the home-based `~/.openclaw` defaults without consulting `OPENCLAW_STATE_DIR`, and #62938 remains an open, unmerged implementation candidate for this exact issue.

## Best Possible Solution

Keep this issue open and review, land, or close #62938. The best fix is to update `src/infra/exec-approvals.ts` so the approvals file and default socket resolve under `OPENCLAW_STATE_DIR` / the shared state-dir resolver before the home fallback, add focused regression coverage in `src/infra/exec-approvals-store.test.ts`, preserve the existing symlink-hardening behavior, and update the exec approvals docs if storage wording changes.
