## Summary

Keep this PR open. Current main has Microsoft Teams thread-routing/session code that parses `conversation.id;messageid=`, but the `requireMention` implicit-mention gate still only checks `activity.replyToId` against the sent-message cache, and the send path only records returned sent message IDs. The PR remains a focused plugin fix for the reported Teams thread-reply behavior.

## Best Possible Solution

Keep this PR open for Microsoft Teams maintainer review. The best path is to adapt or rebase the focused plugin fix onto current main, keep the production implicit-mention helper covered by tests, record a purpose-named thread anchor or document the cache semantics, and verify with targeted Teams tests plus live RSC coverage if available. Do not close as implemented until main's mention gate uses the `conversation.id;messageid=` fallback and the send path caches the matching thread-root anchor.
