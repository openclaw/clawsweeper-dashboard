Full review comments:

- **[P1] Keep Claude runtime ownership in ACPX:** `src/agents/runtime-dispatch.ts:46-54`
  - body: Current main already provides Claude Agent SDK execution through the plugin-owned ACPX backend, and `extensions/acpx/AGENTS.md` explicitly assigns reusable ACP runtime behavior to ACPX. Adding `runAgent` as a second core dispatcher creates competing session, auth, tool, and lifecycle implementations; rework this around ACPX and add only a demonstrated missing generic seam.
  - confidence: 0.95
- **[P1] Fail closed when native policy assembly fails:** `src/agents/claude-sdk/run.ts:526-535`
  - body: If `buildNativeMcpServers` throws, the run continues with SDK Bash/Read/Edit-style built-ins. Those tools do not pass through the OpenClaw native inventory’s owner, requester-role, plugin, and sandbox policy gates, so an inventory failure broadens capability; abort the run or disable all tools instead.
  - confidence: 0.98
- **[P1] Report structured tool failures as MCP errors:** `src/agents/claude-sdk/native-tools-adapter.ts:183-203`
  - body: `agentResultToMcp` always returns `isError: false`, but OpenClaw tools can return structured failure results without throwing. Claude will treat failed message/session/cron operations as successful and continue from a false state; derive the MCP error flag from the tool result contract.
  - confidence: 0.97
- **[P1] Do not persist partial assistant frames:** `src/agents/claude-sdk/session-mirror.ts:226-230`
  - body: The projector treats `partial_assistant` exactly like a finalized assistant message, and every projected record is appended to the canonical transcript. Streaming deltas therefore become duplicate incomplete turns that contaminate future context and transcript consumers; persist only finalized assistant messages or coalesce deltas first.
  - confidence: 0.98
- **[P1] Remove the runtime transcript sidecar:** `src/agents/claude-sdk/session-mirror.ts:150-178`
  - body: This creates `<session>.claude-sdk.jsonl` as persistent runtime state alongside the canonical session file. Repository policy requires one canonical store and prohibits new sidecar state paths; keep runtime provenance in the canonical session/database owner rather than introducing dual writes and lifecycle drift.
  - confidence: 0.96
- **[P1] Preserve the Claude Code preset on normal runs:** `src/agents/claude-sdk/run.ts:488-496`
  - body: When `extraSystemPrompt` is absent, `systemPrompt` is omitted rather than selecting the `claude_code` preset. Normal runs therefore lose the intended Claude Code instruction baseline while only customized runs receive it; always pass the preset and append extra text conditionally.
  - confidence: 0.9
- **[P2] Translate OpenClaw allow-list IDs before SDK dispatch:** `src/agents/claude-sdk/run.ts:498-501`
  - body: `toolsAllow` contains OpenClaw tool IDs, while SDK built-ins and MCP tools use different names such as `Bash` and `mcp__openclaw__...`. Forwarding the list verbatim can silently hide required tools in constrained cron or agent runs; map IDs through the same canonical tool-name contract used by the bridge.
  - confidence: 0.92
