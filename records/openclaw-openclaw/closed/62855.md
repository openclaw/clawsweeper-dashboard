## Summary

The false `invalid config` / `failed to initialize` path described in #62855 appears addressed on current `main` and was already present in the shipped `v2026.4.23` release. Current code no longer cold-loads memory embedding providers during `status`/`doctor` config resolution, and current tests cover both the unregistered-runtime case and the explicit-empty-scope runtime-loading case raised in the follow-up discussion.

## Evidence
