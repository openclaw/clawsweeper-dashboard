## Summary

Current `main` already installs an env-proxy-aware global Undici dispatcher for the OpenAI Codex OAuth login flow and for Codex token refresh/API-key retrieval paths, which matches the issue request.

## Close Comment

Closing as implemented on `main` after Codex review. The current code already forces Codex OAuth onto the env-proxy-aware Undici dispatcher before the PI OAuth login flow (`src/plugins/provider-openai-codex-oauth.ts:130`), and it does the same before Codex token refresh / OAuth API-key retrieval (`extensions/openai/openai-codex-provider.runtime.ts:7-18`). The dispatcher helper installs `EnvHttpProxyAgent` when `HTTP_PROXY` / `HTTPS_PROXY` are present (`src/infra/net/undici-global-dispatcher.ts:91-116`). This behavior is also covered by tests in `src/infra/net/undici-global-dispatcher.test.ts:207-259` and `extensions/openai/index.test.ts:308-324`. The shipped `v2026.4.22` release (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`) already contains that implementation, so this issue no longer needs separate maintainer action.
