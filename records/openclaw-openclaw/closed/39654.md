This PR is obsolete. The focused PR discussion produced a reasonable commands-only coercion patch with regression tests, but the linked user problem in #39500 has since been handled on main by last-known-good config recovery, rejected-write protection, and updated docs that show the supported provider-keyed object shape for `commands.allowFrom`.

## Best Possible Solution
