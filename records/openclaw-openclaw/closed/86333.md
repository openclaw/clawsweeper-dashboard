Full review comments:

- **[P1] Use the real SavedMedia shape:** `src/gateway/server-methods/chat-rewrite-media.test.ts:37-39`
  - body: `mockSavedMedia` returns `localPath`, `mediaType`, and `sha256`, but `SavedMedia` is `{ id, path, size, contentType? }` and the production rewrite reads `.path`. This should fail strict type checking, and if bypassed it rewrites `undefined` media paths, so the added regression tests do not prove the fix.
  - confidence: 0.96
- **[P1] Avoid timestamp-only transcript rewrites:** `src/gateway/server-methods/chat.ts:1141-1145`
  - body: The new fallback accepts any user transcript entry within five seconds even when the text does not match. In rapid or overlapping WebChat sends, the newest no-media user row can receive this turn's `MediaPath`, corrupting persisted history; use a stable row id or a narrower normalized media-marker match instead.
  - confidence: 0.84
- **[P2] Cover the media URI resolver half before claiming the fix:** `src/gateway/server-methods/chat.ts:1128-1132`
  - body: The linked bug also reports `media://inbound/...` refs being resolved under `workspaceDir`; current `image-tool.ts` still has that branch, and this patch only changes transcript matching. Either include the managed-media resolver repair or narrow this PR so the linked issue remains open for the missing half.
  - confidence: 0.82
