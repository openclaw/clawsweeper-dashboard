## Close Comment

Closing as `clawhub` after Codex review. Current `main` already exposes the right extension points for this as a policy plugin rather than an OpenClaw core feature: `before_tool_call` can require human approval for risky actions (`docs/plugins/building-plugins.md:182-194`, `src/plugins/hook-types.ts:324-336`, `src/agents/pi-tools.before-tool-call.ts:219-257`), `after_tool_call` can inspect tool outputs (`docs/concepts/agent-loop.md:99-113`, `src/plugins/hook-types.ts:339-343`), and `message_sending` can cancel or rewrite outbound public content before it leaves OpenClaw (`docs/concepts/agent-loop.md:102-113`, `src/plugins/hook-message.types.ts:42-53`). `VISION.md:52-70` also sets the scope: optional capability should usually ship as plugins, and new skills belong on ClawHub first. Because this request is a policy layer for outbound/public behavior rather than a missing core primitive, it belongs as a ClawHub plugin/skill instead of an OpenClaw core issue.
