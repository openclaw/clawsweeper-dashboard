## Summary

Current `main` already implements `subagents.requireAgentId` for `sessions_spawn`, including both agent-specific and default config knobs, rejects omitted `agentId` when that knob is enabled, and also rejects malformed `agentId` values before normalization/fallback. The best proving commit in this checkout is `56fe2aab9c87596515f2280c8060dc740ea64a88`, which is after the latest tagged release `v2026.4.23`, so this appears fixed on `main` but not yet in a shipped release tag.

## Evidence

- **Subagent runtime enforces explicit agentId:** `spawnSubagentDirect` now reads per-agent/default `requireAgentId` and returns a forbidden error when `agentId` is omitted; it also validates malformed `agentId` strings before normalization.
  - file: [src/agents/subagent-spawn.ts:559](https://github.com/openclaw/openclaw/blob/56fe2aab9c87596515f2280c8060dc740ea64a88/src/agents/subagent-spawn.ts#L559)
