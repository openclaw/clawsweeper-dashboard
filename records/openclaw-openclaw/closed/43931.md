## Summary

Current `main` already implements the requested fix. Root bootstrap loading now only includes `MEMORY.md` when that exact uppercase entry exists, instead of probing both `MEMORY.md` and `memory.md`, and the memory host code follows the same canonical-root policy. Regression tests cover the lowercase-fallback and single-entry behavior, and the original fix shipped in `v2026.3.12` via merge commit `a3eed2b70f5055074ca96a802be1637604ada3db`.

If you still see duplicate root-memory injection on a newer build, please open a new issue with the exact OpenClaw version and a concrete bootstrap dump/repro from that environment.
