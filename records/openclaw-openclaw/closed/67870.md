## Summary

Current `main` supersedes this PR: the pre-commit hook is now formatting-only, no longer runs `pnpm check`, and the added integration tests explicitly verify that `pnpm` is not invoked from the hook path. That makes the Corepack-specific `pnpm check` fallback in this PR unnecessary on current `main`.

## Evidence

- **Pre-commit hook no longer runs repo-wide validation:** `git-hooks/pre-commit` only gathers staged files, filters format targets, runs `oxfmt`, and re-stages files. There is no `pnpm check`, `corepack`, or `FAST_COMMIT` branch left in the hook.
