## Summary

Current `main` already implements per-agent model fallback control via `agents.list[].model` object overrides. Agent-level `{ primary, fallbacks }` is accepted, runtime resolution preserves explicit `fallbacks: []` to disable inherited global fallbacks, and docs/tests describe that behavior. This appears to have landed on `main` in commit `bff212822cf73c7a37fa7a77d9a725c2d8895b1e`, which is newer than the latest tagged release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Risks / Open Questions

- The implementation matches the issue body's proposed `{ primary, fallbacks }` override model. A bare string `agents.list[].model: "provider/model"` still inherits global fallbacks unless the config is changed to object form with `fallbacks: []`.
