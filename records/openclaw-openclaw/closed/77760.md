## Summary

Close: current main and released v2026.5.26+ no longer use the bundled Sharp-backed media image path, so the nested sharp dependency gap reported here is gone.

## What This Changes

The issue reports that packaged/global installs can miss sharp because media-understanding-core declared it only in a nested bundled extension manifest while runtime image ops imported it.

## Best Possible Solution

Retain the released rastermill-backed media image stack; any new image failures should be filed as fresh backend-specific reports rather than reopening the old nested-sharp packaging bug.

## Reproduction Assessment

Yes. The old failure mode is source-reproducible from the historical package shape, and current main plus v2026.5.26 source show that the sharp import and nested sharp manifest no longer exist for this path.

## Solution Assessment

Yes. Replacing the shipped Sharp-backed path with a root-declared image backend is a cleaner durable fix than mirroring sharp into root optionalDependencies.
