## Summary

Keep open: current main and the latest release still lack root `sharp` package metadata while the bundled media image runtime imports `sharp`; the earlier fixing PR was closed unmerged, so the package-boundary bug remains.

## What This Changes

The issue asks OpenClaw to declare the bundled media runtime's `sharp` dependency in the root published package graph so npm global installs can resolve it.

## Best Possible Solution

Add exact-pinned `sharp: 0.34.5` to root `optionalDependencies` with the matching lockfile importer, keep the nested media package dependency, preserve graceful fallback when optional native install fails, and pass dependency ownership/SBOM checks.

## Reproduction Assessment

Yes, by source and package metadata. A packaged/global install without root `sharp` loads bundled `media-understanding-core/image-ops.js`, which dynamically imports `sharp`; the issue and comments include matching real Linux/macOS global-install failures.

## Solution Assessment

Yes, with the exact version pinned. Root `optionalDependencies` is the narrow package-boundary fix for a bundled runtime import while allowing platforms without usable Sharp binaries to keep the existing fallback behavior.
