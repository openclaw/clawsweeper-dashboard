## Summary

Current `main` routes OpenAI-compatible STT uploads through a runtime FormData normalization layer that rebuilds multipart bodies into the runtime undici FormData and clears stale multipart headers before dispatch, which directly addresses the reported 422 parsing failure.

## Evidence
