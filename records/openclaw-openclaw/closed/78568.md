## Summary

Close as ClawHub/plugin work: current main already has the `before_dispatch` hook semantics needed to claim selected inbound messages and suppress model dispatch, while a reusable inbound-to-webhook forwarder is optional integration code that belongs outside OpenClaw core.

## What This Changes

The issue asks whether a third-party plugin should use `before_dispatch` to forward selected inbound WhatsApp group messages to an authenticated external webhook and skip the agent reply.

## Best Possible Solution

Keep the core hook API as-is, publish the inbound webhook forwarder as an external ClawHub/npm plugin, and handle any generic `before_dispatch` wording improvement as a small docs-only follow-up.

## Reproduction Assessment

Not applicable; this is a feature/guidance request rather than a bug report. Source inspection and focused tests give high-confidence proof of the current hook behavior.

## Solution Assessment

Yes; `before_dispatch` is the narrow maintainable primitive for a hook-only code plugin to claim selected inbound messages and suppress model dispatch. Bundling a generic webhook forwarder in core is not the best fit under the current plugin and ClawHub policy.
