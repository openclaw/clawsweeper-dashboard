## Summary

Keep this PR open. It is a draft maintainer-labeled MEMBER PR, so it should not be auto-closed, and current main does not yet carry StorePack metadata through ClawHub install/update records. The patch is directionally useful, but I found a blocking version-scoping issue in the new StorePack metadata fallback before it should merge.

## What This Changes

The PR adds ClawHub StorePack response types, persists StorePack digest/spec/manifest/size fields in plugin install records, propagates them through ClawHub install and update flows, preserves them in installed plugin index records, and adds tests plus a changelog entry.

## Best Possible Solution

Persist StorePack fields only when the metadata is tied to the exact resolved ClawHub package version, then keep the install/update/index propagation and tests so stored artifact facts can safely support later registry refresh and download verification.

## Reproduction Assessment

Yes for the review finding: mock a pinned ClawHub install where `fetchClawHubPackageVersion` returns the requested version without `storepack`, while `fetchClawHubPackageDetail` contains package-level StorePack metadata for the latest version. The PR would persist that package-level digest on the pinned install record.

## Solution Assessment

No as written. The persistence path is the right place for this feature, but the package-detail fallback should be removed or guarded by an explicit version match so stored verification facts remain version-scoped.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Keep StorePack metadata tied to the resolved version:** `src/plugins/clawhub.ts:696`
  - body: This fallback can persist package-level StorePack facts when the selected version response does not include `storepack`. For a pinned install such as `clawhub:pkg@old`, package/detail metadata can describe the current package/latest artifact, so the install record would store a digest and manifest hash for the wrong version. Since the PR says later refresh/download verification will reuse these fields, persist only version-scoped StorePack metadata or guard the fallback with an explicit resolved-version match.
  - confidence: 0.78

## Security Review
