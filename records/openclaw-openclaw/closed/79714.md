## Summary

Current main already supports user-defined session labels that satisfy the requested custom session naming: labels can be patched through the Gateway/API, edited in the Control UI Sessions view, and are preferred by chat session display/resolution.

## What This Changes

Request that persistent chat sessions be manually renamed or labeled with a human-friendly custom title.

## Best Possible Solution

Keep the existing session-label implementation as the rename mechanism; a `/rename` alias would be optional UX polish rather than required core work.

## Reproduction Assessment

Not applicable; this is a feature request rather than a bug report. Source and tests on current main show the requested naming behavior is already available through session labels.

## Solution Assessment

Yes; using `sessions.patch` plus the Control UI Sessions label field is a narrow, maintainable solution that avoids adding a separate title store. The exact `/rename` command proposed in the issue is not present, but the requested outcome is covered.
