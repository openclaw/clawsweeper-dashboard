## Summary

Current `main` does expose plugin tools on `POST /tools/invoke`, and the shipped Lobster/LLM Task docs explicitly document `openclaw.invoke --tool llm-task` with a required `main`-agent allowlist. The reported 404 is more consistent with session targeting or missing allowlist/config than with `llm-task` being unavailable on the HTTP tool surface.

## Evidence

- **HTTP tools surface resolves plugin tools and defaults missing sessionKey to main:** `POST /tools/invoke` accepts an optional `sessionKey`; when omitted it resolves to `resolveMainSessionKey(cfg)`. Tool resolution then calls `resolveGatewayScopedTools(...)` with plugin tools enabled for non-core tool names and returns 404 only if the tool is absent from that resolved set.

## Risks / Open Questions
