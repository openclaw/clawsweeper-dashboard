## Summary

Current `main` already contains a Feishu-specific replay/dedup pipeline that matches the reported duplicate-reply failure mode from `2026.3.23-2`: persistent per-account message dedupe, duplicate-event dropping before debounce, post-success commit of processed message IDs, retry-safe release on pre-delivery failure, and reply-once regression coverage. The checkout version is `2026.4.24`, which is newer than the reported affected version.
