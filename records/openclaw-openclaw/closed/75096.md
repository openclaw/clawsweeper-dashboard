## Summary

This should close for OpenClaw source cleanup because the failing queue and transport path belongs to the external Yuanbao channel plugin, not code carried in this repository. Current main only documents and catalogs the plugin, while the core SDK already exposes generic routing/session/reply helpers for direct and group channel runtimes.

## What This Changes

The issue reports Yuanbao group @mention replies being generated but queued to the direct outbound session instead of the group outbound session.

## Best Possible Solution

Have the Yuanbao plugin maintainers fix the runtime queue/session selection in YuanbaoTeam/yuanbao-openclaw-plugin, then update OpenClaw catalog metadata or docs only if a new plugin release needs to be pinned or called out.

Yes. The report gives a concrete Yuanbao group @mention path and logs showing group inbound detection, empty group queue flush, direct queue push, and C2C send; live reproduction is outside this checkout because the Yuanbao runtime is external.
