## Summary

Keep open. Current main still does not emit `ProcessType=Interactive` in the generated macOS gateway LaunchAgent plist. The WorkingDirectory part has partially changed on main, defaulting macOS installs to the OpenClaw state directory rather than `/`, but the central reported launchd classification fix is not present. An open PR, #58070, explicitly closes this issue, so the best maintainer path is to review/land that PR or an equivalent fix before closing.

## Best Possible Solution

Review and land #58070 or an equivalent targeted change that adds `ProcessType=Interactive` to the gateway LaunchAgent plist builder and regression tests. Confirm the desired macOS default WorkingDirectory, then close this issue through the merged PR once main contains the launchd classification fix.
