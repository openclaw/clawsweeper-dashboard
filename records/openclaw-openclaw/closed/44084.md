## Summary

Current main, and the latest release tag v2026.4.24, already prevent the reported persisted cron shape from crashing `openclaw cron list`. The list command reads jobs through the Gateway cron list path, persisted jobs are normalized before display, and non-string `payload.model` values are removed because current cron payloads store `model` as a string with `fallbacks` as a sibling array.
