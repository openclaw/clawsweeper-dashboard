## Summary

Keep open. Current `main` still hardcodes the 256 KiB Claude live stdout buffer cap and 2 MiB per-turn raw cap, aborts the session when those thresholds are exceeded, and does not expose config knobs for either limit. The latest release `v2026.4.23` ships the same behavior, and the issue discussion adds cross-platform confirmation plus a second trigger path for Telegram image turns.

## Evidence

## Risks / Open Questions

- Raising only the stdout buffer may leave the separate `CLAUDE_LIVE_MAX_TURN_RAW_CHARS` abort path in place for image-heavy or schema-heavy turns.
- Any fix will need config/schema/docs/tests alignment and should consider memory-abuse tradeoffs before simply lifting limits globally.
