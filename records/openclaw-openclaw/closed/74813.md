## Summary

Keep open. The draft PR targets a real continuation-skip subcase, but the current diff is not mergeable because it treats any assistant transcript row as safe completion evidence; heartbeat-only `HEARTBEAT_OK` rows are persisted without full-bootstrap markers, and the PR lacks real behavior proof.

## What This Changes

The PR changes continuation-skip transcript classification so assistant messages without a full-bootstrap marker count as completed bootstrap evidence, and updates bootstrap regression tests.

## Best Possible Solution

Revise the helper to accept ordinary non-heartbeat continuation evidence while preserving heartbeat/lightweight and post-compaction reinjection, then add redacted runtime proof for continuation-skip.

## Reproduction Assessment

Yes for source-level reproduction: a session JSONL containing only a heartbeat prompt and `HEARTBEAT_OK` assistant reply, with no full-bootstrap marker, would make the PR branch return completion for the next normal continuation-skip run. I did not run tests because this was a read-only review.

## Solution Assessment

No. The helper is the right place for the missing-marker subcase, but the proposed any-assistant fallback should ignore recognized heartbeat-only pairs or require ordinary non-heartbeat continuation evidence.
