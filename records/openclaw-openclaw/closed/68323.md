## Summary

Keep #68323 open. Current main has per-agent TTS overrides and provider-specific style knobs, but it still does not implement the requested provider-agnostic TTS persona contract, active persona selection, persona rewrite stage, or provider persona mapping semantics. The related #70748 PR is the closest implementation path, but it is open and unmerged; the maintainer comments in the issue explicitly identify #68323 as the canonical feature request.

## Best Possible Solution

Keep #68323 open as the canonical feature request. The best implementation path is to refine or replace #70748 with a durable config/API contract: add `messages.tts.persona` and `messages.tts.personas`, implement persona resolution in speech-core before synthesis, add provider mapping/preparation semantics for bundled speech providers, expose `/tts persona`/CLI/gateway/status controls, and cover the schema, resolver, provider mappings, and UX surfaces with targeted tests and docs.
