## Summary

Current main already fixes this packaged channel crash by staging mirrored root runtime dependencies, including `json5`, and by treating materialized runtime-deps manifests as supersets instead of pruning them during later plugin activation. The fix is on main only; `v2026.4.29` predates it, which matches the reported release-specific failure.

## What This Changes

The issue asks OpenClaw to ensure `json5` and other bare `dist/` imports are explicit bundled runtime dependencies so channel plugins do not crash after upgrading to `2026.4.29`.

## Best Possible Solution

Keep the generic package-level runtime-deps mirror and superset materialization behavior, then ship it in the next release rather than adding Telegram-specific dependency handling or rewriting shared dist imports.

## Reproduction Assessment

Yes. The issue gives a concrete `2026.4.29` packaged install failure path with the missing `json5` bare import, and current main has focused regression coverage for the package-level runtime-deps behavior that would otherwise prune or omit the dependency.

## Solution Assessment

Yes. The current main approach is the narrow maintainable fix because it stages root dist dependencies through the generic mirrored-root dependency contract and prevents later bundled plugin activation from breaking that staged superset.
