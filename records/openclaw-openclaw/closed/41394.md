## Summary

Close #41394 as a duplicate/superseded tracker. The same strict-write/tolerant-read config validation mode request is already tracked by open issue #40317, while current main only has partial mitigations: strict write validation, last-known-good startup/reload recovery, and doctor repair for unknown keys. The remaining product work should stay consolidated on #40317, with #40268 still covering the narrower read-path bug.

## Best Possible Solution

Close #41394 as a duplicate of #40317. Keep #40317 open for maintainer product judgment and future implementation of an explicit config validation policy that preserves strict writes, defines tolerant read/startup behavior and logging, and adds regression tests for unknown root and nested config keys. Keep #40268 available as the narrower read-path bug reference.
