The exact behavior landed in commit 5cc06c69a9c61e079f6ce0f40c780556f43c17db and is contained in release v2026.4.26, so the open PR is now redundant with shipped main behavior.

## What This Changes

The PR changes Discord explicit-target parsing from returning a bare target id to returning the normalized target string, with regression tests for explicit user and channel targets.
