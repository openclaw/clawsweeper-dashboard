## Summary

Current main addresses the reported packaged runtime-deps crash path: `json5` is included in the mirrored root dependency set, and the runtime-deps materialization check now accepts already-staged package-level dependency supersets so later lazy plugin activation does not reinstall and prune the shared stage. The fix is on main after `v2026.4.29`, so release users still need a newer build.

## What This Changes

The issue reports that OpenClaw 2026.4.29 npm installs crash-loop Telegram and Discord startup because staged bundled runtime dependencies lose `json5` for root dist chunks.

## Best Possible Solution

Keep the current main implementation: package-level staging should continue to mirror root dist dependencies such as `json5`, and per-plugin activation should treat a staged superset as converged instead of pruning it. The next release should carry this fix to npm users.

## Reproduction Assessment

Yes. The issue's gateway-start path maps to the unit-level reproduction now covered by `src/plugins/bundled-runtime-deps.test.ts`, where a package-level staged superset must not trigger a per-plugin reinstall; live gateway smoke was not run because this review is read-only.

## Solution Assessment

Yes. Fixing the runtime-deps convergence logic is narrower and more maintainable than patching `json5` metadata, because the failure is caused by staged dependency pruning/reinstall behavior rather than by a missing `json5` export contract.
