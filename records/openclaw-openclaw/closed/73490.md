## Summary

The proposed security fix is unnecessary on current main. The alleged argument-injection path does not reproduce because `debug.logs` derives `pid` from `android.os.Process.myPid()` as an `Int`, passes logcat arguments with list-form `ProcessBuilder`, and is gated to debug builds. The PR only adds digit filtering to an already numeric value and does not add supply-chain, dependency, workflow, permission, or secret-handling risk.

## Best Possible Solution

Close the PR rather than merge it. Keep the current implementation unless a future report shows a concrete path where attacker-controlled input reaches the logcat argv; that report should include a reproduction or failing test tied to the Android debug command surface.
