## What This Changes

The issue requests suppressing `plugins.metadata.scan` diagnostics spans for plugin metadata cache hits while keeping real metadata scans instrumented.

## Best Possible Solution

Refresh and land or replace the linked PR so memo hits return the cached snapshot directly while real metadata scans keep diagnostics coverage.

## Solution Assessment

Yes. The narrow maintainable fix is to return the memoized snapshot directly on cache hits while leaving real metadata scan instrumentation on misses; the linked PR targets that path but needs refresh/proof before merge.
