## Summary

Current `main` already routes Slack `message read` requests with `threadId` through the threaded replies path, and the shipped `v2026.4.22` release contains the same implementation and regression tests.

## Evidence

- **Slack threaded reads use `conversations.replies`:** `readSlackMessages` switches to `client.conversations.replies({ channel, ts: threadId, ... })` when `opts.threadId` is present and filters out the parent message before returning results.
  - file: [extensions/slack/src/actions.ts:235](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/extensions/slack/src/actions.ts#L235)
  - command: `nl -ba extensions/slack/src/actions.ts | sed -n '220,290p'`
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **`message read` forwards `threadId` into Slack read action:** The Slack channel action adapter enables read-thread forwarding, and the channel test asserts that a `read` action with `threadId` is converted into `readMessages` with the same `threadId`.
  - file: [extensions/slack/src/channel.test.ts:259](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/extensions/slack/src/channel.test.ts#L259)

- **Runtime passes `threadId` through to the Slack reader:** `handleSlackAction` reads the `threadId` param for `readMessages` and forwards it to `readSlackMessages`; the runtime test covers that exact path.
  - file: [extensions/slack/src/action-runtime.ts:353](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/extensions/slack/src/action-runtime.ts#L353)

- **Regression test covers threaded Slack reads:** `actions.read.test.ts` verifies that a threaded read calls `conversations.replies`, not `conversations.history`, and excludes the parent thread root from returned messages.
  - file: [extensions/slack/src/actions.read.test.ts:19](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/extensions/slack/src/actions.read.test.ts#L19)
