## Summary

## What This Changes

Correct embedded-agent failure reporting so an HTTP client disconnect is classified as external cancellation rather than a configured run timeout.

## Best Possible Solution

Land the narrow abort-signal classification fix in https://github.com/openclaw/openclaw/pull/90936, then close this issue; keep the distinct resumed-CLI watchdog work in https://github.com/openclaw/openclaw/pull/90912 separate.

Yes. Current source gives a high-confidence path: an HTTP socket close calls `AbortController.abort()` without a timeout reason, and embedded-run handlers apply broad timeout heuristics to the resulting cancellation reason.

## Solution Assessment

Yes. The linked PR’s focused signal-reason discriminator is narrower and more maintainable than changing timeout ceilings, adding configuration, or weakening general provider-error classification.
