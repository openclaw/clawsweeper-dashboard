## Summary

Keep open: current main has a directive-based media path for documented `MEDIA:` lines and explicit payload media, but it still does not provide the requested generic final-mile guard for arbitrary raw path prose, structured assistant blocks, configurable generated-artifact roots, success-gated stripping, and safe failure payloads. The related open root-policy PRs overlap part of the trust-root problem but do not supersede this broader product/security design request.

## What This Changes

The issue requests a channel-neutral outbound guard that converts validated generated artifact references into real channel attachments and removes unsafe raw path/directive text only after delivery succeeds.

## Best Possible Solution

Keep this as the canonical design issue until maintainers define a central outbound artifact-normalization policy with a narrow grammar, explicit trust roots, existing media access checks, send-success-dependent stripping, and observable failure semantics.

## Reproduction Assessment

Not applicable: this is a feature/design request rather than a bug against an already-defined OpenClaw contract. Source inspection gives high-confidence gap evidence because documented `MEDIA:` and explicit payload media are handled, while arbitrary raw path claims are not.

## Solution Assessment

Unclear: the requested direction is plausible, but grammar, allowed roots, hook location, and failure payload behavior need maintainer product/security decisions. The safest implementation direction is an additive central helper that reuses existing media staging and delivery outcomes instead of channel-specific parsing.
