## Summary

Close as superseded: the merged broader fix in https://github.com/openclaw/openclaw/pull/80812 covers the same Claude ACP timeout failure at both the control-plane emission layer and the ACPX wrapper layer, while this PR only patches the wrapper path.

## What This Changes

This PR changes ACPX to skip `timeout` and `timeout_seconds` config-option forwarding and flips the Claude ACP runtime unit test to expect no forwarding.

## Best Possible Solution

Keep the merged https://github.com/openclaw/openclaw/pull/80812 implementation as the canonical fix, with timeout filtering at the advertised-key control-plane boundary and ACPX Claude suppression as defense in depth.

## Reproduction Assessment

No current-main failure reproduces now: the timeout pair is no longer emitted when Claude's advertised config keys omit timeout. The historical failure is high-confidence from source inspection, dependency source, and multiple live repro comments on the PR.

## Solution Assessment

No. This PR's wrapper-only change is not the best fix because `applyManagerRuntimeControls` can reject an unsupported auto-injected timeout before `AcpxRuntime.setConfigOption` is reached; the merged two-layer fix is the narrower maintainable solution.

## Review Findings
