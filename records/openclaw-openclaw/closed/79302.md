## Summary

Current main already carries parsed `MEDIA:` directives into outbound payload media and then through plugin/channel delivery, with regression coverage for the reported path. The proof commit is on current main only; no release tag contains it yet.

## What This Changes

The issue reports that `MEDIA:` directive text is stripped but the parsed media URLs are not forwarded to plugin outbound delivery payloads.

## Best Possible Solution

Keep the current generic outbound payload normalization and delivery fanout; adapters that need full structured payloads should use the existing `sendPayload`/`payload.mediaUrls` contract, while ordinary media sends continue through `sendMedia`.

## Reproduction Assessment

No. Current-main source and regression tests show `MEDIA:` directives are parsed into `mediaUrls` and delivered as media, so the reported dropped-media state does not reproduce from source inspection.

## Solution Assessment

Yes. The current implementation solves this at the shared outbound payload layer, which keeps core plugin-agnostic while preserving both structured `payload.mediaUrls` and per-media adapter sends.
