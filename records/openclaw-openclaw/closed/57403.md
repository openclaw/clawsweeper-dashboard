## Summary

Current `main` already implements the protocol-side mitigation for this report: Telegram webhook callbacks return early with a 5s timeout and `200` ACK while work continues, and duplicate Telegram updates are deduped by `update_id`/update key. The shipped `v2026.4.23` changelog explicitly records this fix in #70146.
