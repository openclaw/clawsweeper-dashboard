## Summary

Current main and the latest release already route the reported bare npm package install through the managed npm installer instead of the archive extraction path that produced the Windows EISDIR error, so the central user problem is already implemented for this command.

## What This Changes

The issue reports that `openclaw plugins install @wecode-ai/weibo-openclaw-plugin` fails on Windows during archive extraction with an EISDIR realpath error.

## Best Possible Solution

Keep the shipped managed-npm install behavior and ask affected Windows users on older builds to upgrade to `v2026.5.22` or newer; separately triage any fresh current-version archive-source failure.

## Reproduction Assessment

No high-confidence current-main failing reproduction remains for the reported command: source and tests show that ordinary bare npm plugin specs no longer use the archive extraction path. I did not run a live Windows install in this read-only review.

## Solution Assessment

Yes. For the reported command, routing npm package installs through the managed npm root is the maintainable fix because it avoids treating registry packages as local archives while preserving explicit archive semantics for `npm-pack:` and local files.
