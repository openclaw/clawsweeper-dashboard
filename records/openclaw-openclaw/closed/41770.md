## Summary

PR #41770 is superseded by the merged/current-main memory-host SDK implementation from #71506. Current main routes shared remote memory embedding and batch HTTP calls into `TRUSTED_ENV_PROXY` only when `shouldUseEnvHttpProxyForUrl()` says Undici would actually use HTTP_PROXY/HTTPS_PROXY for that target, while preserving strict DNS-pinned behavior for non-proxied targets. That is narrower than this PR's earlier unconditional trusted-mode approach and addresses the linked proxy bug without keeping this stale PR open.

## Best Possible Solution

Close #41770 as superseded by merged #71506/current main. Keep the conditional memory-host SDK env-proxy implementation and leave separate status-probe or provider-specific proxy-policy work to focused follow-ups rather than reviving this stale PR.
