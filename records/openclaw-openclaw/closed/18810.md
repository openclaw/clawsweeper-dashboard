## Summary

Current `main` already ships `before_tool_call` veto handling and `after_tool_call` wiring through the tool wrapper + embedded handler path, with docs and tests covering the behavior. This PR's branch is superseded by the implementation now on `main`.

## Evidence

- **Tools are wrapped before execution with hook context:** `wrapToolWithBeforeToolCallHook(...)` is applied to agent tools with `agentId`, `sessionKey`, `sessionId`, and `runId` before they are handed to the adapter.
  - file: [src/agents/pi-tools.ts:704](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/agents/pi-tools.ts#L704)
- **Veto and context propagation are implemented in the live hook path:** `runBeforeToolCallHook(...)` invokes the global hook runner with normalized params plus tool context and returns a terminal blocked result when a plugin sets `block: true`; it also supports the newer `requireApproval` flow.
  - file: [src/agents/pi-tools.before-tool-call.ts:187](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/agents/pi-tools.before-tool-call.ts#L187)
- **after_tool_call is wired on tool completion:** `handleToolExecutionEnd(...)` emits `after_tool_call` with tool params, error/duration data, and `agentId`/`sessionKey`/`sessionId`/`runId` context.
  - file: [src/agents/pi-embedded-subscribe.handlers.tools.ts:1122](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/agents/pi-embedded-subscribe.handlers.tools.ts#L1122)
- **Regression tests cover veto behavior:** Current tests assert that `block: true` prevents tool execution and that hook context includes `agentId`, `sessionKey`, `runId`, and `toolCallId`.
  - file: [src/agents/pi-tools.before-tool-call.integration.e2e.test.ts:128](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/agents/pi-tools.before-tool-call.integration.e2e.test.ts#L128)
- **Regression tests cover after_tool_call wiring:** Current tests assert that `after_tool_call` fires from `handleToolExecutionEnd(...)` with the expected payload and context.
  - file: [src/plugins/wired-hooks-after-tool-call.e2e.test.ts:115](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/src/plugins/wired-hooks-after-tool-call.e2e.test.ts#L115)
