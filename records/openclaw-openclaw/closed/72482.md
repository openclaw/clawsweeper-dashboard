## Summary

Keep this PR open. Current main still has the Telegram non-streaming gap this PR targets: `streamMode: "off"` disables the answer draft stream, but Telegram still passes `suppressDefaultToolProgressMessages: false`, allowing generic tool/progress messages to be delivered. The PR is a narrow candidate fix with a matching regression-test update and no unrelated security-sensitive file changes in the provided diff.

## Best Possible Solution

Review and, if maintainers agree with final-only Telegram non-streaming semantics, land this narrow PR with its regression test. Before merge, adjust the closing relationship or confirm current reasoning-lane handling fully resolves the reasoning-token portion of #72363 so remaining work is not closed accidentally.
