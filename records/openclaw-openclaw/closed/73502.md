## Summary

Current main still matches the reported bundled-plugin mismatch: Active Memory hard-codes `memory_search`/`memory_get` in its prompt and runtime tool allowlist, while the bundled `memory-lancedb` memory-slot plugin registers `memory_recall`/`memory_store`/`memory_forget`. This remains an actionable OpenClaw bundled-plugin integration bug, not something to close as stale, duplicate, implemented, or ClawHub work.

## Best Possible Solution

Keep open. Align the bundled memory plugin boundary by making Active Memory use the selected memory slot's typed recall capability/tool, or by adding a compatibility adapter in `memory-lancedb` if `memory_search`/`memory_get` are intended to be the shared memory-slot contract. Add a regression test covering Active Memory with `plugins.slots.memory = "memory-lancedb"`, then update the Active Memory and Memory LanceDB docs to describe one supported integration path.
