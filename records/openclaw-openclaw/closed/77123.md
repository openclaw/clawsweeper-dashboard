## Summary

Keep open: the PR addresses a real unbounded `sessions.list` path, but the patch currently applies the new default bound inside a shared helper used by exact session resolution, which can truncate `spawnedBy` visibility checks and reject older exact-key child sessions.

## What This Changes

The PR adds a default 100-row bound to Gateway `sessions.list` responses, returns truncation metadata, and updates session types, tests, CLI docs, and the changelog.

## Best Possible Solution

Land a version that bounds only `sessions.list` responses, preserves unbounded internal exact-resolution visibility paths, and adds regression coverage for `sessions.resolve` with more than 100 spawned children.

## Reproduction Assessment

Yes. Current main source shows `sessions.list` can build every filtered row when no `limit` is supplied, and the PR-introduced regression is source-reproducible from the new default limit plus `sessions.resolve` using the same sync helper for exact-key visibility.

## Solution Assessment

No. Bounding `sessions.list` is the right direction, but applying the default inside the shared sync helper is not the narrowest safe fix because exact resolution also uses that helper.
