## Summary

Close as implemented on main: current `main` keeps plain `openclaw status` on the fast path by skipping the expensive security-audit, plugin-compatibility, memory-backend, and live channel checks unless `--all` or `--deep` is requested, and tests/docs now cover that behavior. The report’s `--deep` timing is still a heavier path by design, so this decision covers the default status latency that the issue centers on.

## What This Changes

The issue requests making default `openclaw status` table output complete close to JSON speed for a lightweight Gateway setup.

## Best Possible Solution

Keep the current fast-path behavior and ship it in the next release; if current `main` still shows a table-only delay independent of audit/probe work, that should be tracked as a narrower renderer performance issue.
