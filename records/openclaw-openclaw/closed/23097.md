## Summary

Current `main` already covers the reported Discord thread-context gap: it fetches the original thread starter and injects it as `ThreadStarterBody`, keeps thread messages on a stable thread session key, and can optionally fork the parent channel transcript into new thread sessions via `channels.discord.thread.inheritParent`. The issue’s `history_count: 0` symptom is no longer a reliable indicator that thread context is missing, because the starter message is now passed separately from short untrusted history metadata.
