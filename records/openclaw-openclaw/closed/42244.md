## Summary

Current `main` already fixes the announce/message-tool zero-delivery path for isolated cron runs: announce mode keeps the message tool enabled, matching message-tool sends are treated as the completed delivery path instead of being counted as a skipped duplicate, and the behavior is covered by tests and docs.

## Close Comment

Closing from a Codex review because current `main` already implements this behavior. `src/cron/isolated-agent/run.ts:852` detects when the message tool already sent to the resolved announce target and passes `skipMessagingToolDelivery`, and `src/cron/isolated-agent/delivery-dispatch.ts:439` then treats that matched tool send as the completed delivery path instead of falling into a silent zero-delivery state. The regression coverage in `src/cron/isolated-agent/run.message-tool-policy.test.ts:408` and `src/cron/isolated-agent/run.message-tool-policy.test.ts:457` explicitly checks that `announce` keeps the message tool enabled and skips cron fallback when the tool already sent to the same target. I also verified the relevant delivery code is unchanged between current `main` and release `v2026.4.22`, so this is already shipped.
