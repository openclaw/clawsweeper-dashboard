Restore Discord channel bot tokens configured as SecretRef objects after they stopped resolving for npm-installed Discord in 2026.5.3.

## Best Possible Solution

Keep the v2026.5.4/current-main external channel contract loader and Discord SecretRef registry behavior, and have affected npm installs upgrade both OpenClaw and `@openclaw/discord` before reloading secrets or restarting the gateway.

No high-confidence current-main failure path remains. The 2026.5.3 failure mode is source-explained by the missing external plugin `dist/` contract lookup, and current main/v2026.5.4 include the resolver and regression coverage.

## Solution Assessment

Yes. Loading compiled external channel secret-contract sidecars is the narrow maintainable fix because Discord already declares `channels.discord.accounts.*.token` through the plugin contract; no Discord-specific core migration is needed.
