## Summary

Current main still does what the report describes: the embedded runner applies context-engine assembly to `activeSession.messages`, but the pre-prompt overflow guard can replace that prompt estimate with the larger pre-assembly `unwindowedMessages` estimate and block submission. The companion LCM issue is correctly split: OpenClaw owns the host-side prompt-admission behavior, while Martian-Engineering/lossless-claw#534 tracks plugin-side cache/provenance hardening.

## Best Possible Solution

Keep this issue open and fix the OpenClaw host-side prompt-admission path. The best direction is to make successful context-engine assembly prompt-authoritative for the pre-prompt admission check, while preserving raw pre-assembly history pressure as a separate maintenance/debt signal for context-engine upkeep. Update the preemptive-compaction helper and tests so a small assembled prompt from a large raw history is allowed, but genuinely over-budget assembled prompts still compact or truncate before provider submission.

## Repair Work Prompt

Fix the host-side prompt-admission bug from openclaw/openclaw#74229. Current main applies context-engine assembly to `activeSession.messages`, but `shouldPreemptivelyCompactBeforePrompt(...)` can still prefer the larger pre-assembly `unwindowedMessages` estimate and block prompt submission. Make successful context-engine assembly prompt-authoritative for prompt admission: estimate the actual assembled messages plus system prompt/systemPromptAddition and current user prompt when deciding whether to submit, compact, or truncate. Preserve raw pre-assembly history pressure as a separate maintenance/debt signal if needed; do not delete useful observability or reintroduce the #71325 hidden-history maintenance gap. Keep the fix generic to the context-engine contract: no `lossless-claw`, LCM, provider, or plugin-specific IDs/policy in core. Update or replace the current test that asserts the larger unwindowed estimate wins; add a regression where a fake context engine assembles a small under-budget prompt from a large raw history and precheck allows submission. Also keep coverage that genuinely over-budget assembled prompts still trigger the existing compact/truncate route. Update docs if the prompt-admission/raw-history-debt distinction becomes a documented contract, and add a single-line changelog entry under the active version with `Thanks @electricsheepops`. Do not change unrelated compaction, provider limits, model selection, or plugin installation behavior.
