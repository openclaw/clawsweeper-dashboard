## Summary

Keep open. Current main still shallow-merges cron-supplied `heartbeat.target="last"` over configured heartbeat defaults, so an inherited `heartbeat.to` can remain an explicit destination; PR #73968 is the open unmerged fix candidate for this exact route.

## What This Changes

The issue reports that main-session cron `systemEvent` heartbeat wakes should route back to the bound Telegram forum topic instead of inheriting a global heartbeat DM destination.

## Best Possible Solution

Review and land PR #73968, or an equivalent narrow core fix that clears inherited explicit heartbeat destination fields only for cron-forced `target="last"` wakes while preserving the bound session route and topic metadata.

## Reproduction Assessment

Yes by source inspection and the reporter's concrete config/logs. The current path is cron `heartbeat: { target: "last" }` -> gateway shallow heartbeat-default merge -> heartbeat delivery treating inherited `to` as explicit.

## Solution Assessment

Yes. A narrow fix at the cron wake adapter boundary is the best maintainable solution; changing all heartbeat `target="last"` semantics would be broader than this bug requires.
