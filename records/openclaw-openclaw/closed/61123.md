## Summary

Current `main` already supports running multiple isolated agents on one Gateway, with per-agent workspace/state/auth/session scope, per-agent model/runtime overrides, channel/account bindings, and per-agent heartbeat/session behavior. The linked follow-up about memory-wiki vault scoping is a narrower enhancement, not evidence that single-gateway multi-agent support is missing.

## Evidence

- **Multi-agent docs explicitly match the request:** The user-facing docs say one running Gateway can host many isolated agents side-by-side, each with its own workspace, state directory (`agentDir`), session store, and per-agent auth profiles, and that bindings route channel accounts to the right agent.

  - file: [docs/gateway/config-agents.md:892](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/docs/gateway/config-agents.md#L892)
