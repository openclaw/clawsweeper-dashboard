## Summary

Keep this PR open. The reported `sessions_spawn` model override bug is valid on current `main`, but the proposed forwarding path is still incorrect because native subagent Gateway `agent` calls intentionally keep write scope, while the Gateway rejects `provider`/`model` params unless the caller has admin scope or `internal.allowModelOverride`.

## What This Changes

The PR forwards the resolved native subagent model as `provider`/`model` params in the `callSubagentGateway({ method: "agent" })` call and removes one blank line in `src/gateway/server-plugins.ts`.

## Best Possible Solution

Keep the focused `sessions_spawn` fix, but route the requested model through a Gateway-recognized native subagent trust path or persisted override field that `agentCommandFromIngress` actually consumes, while preserving least-privilege `agent` calls and adding a seam test that proves the requested model reaches runtime.
