## Summary

Current `main` already fixes the reported shadowed-install update path. The updater now derives the owning npm prefix from the active package root and uses that npm binary/root for global updates, instead of blindly following whatever `npm root -g` from PATH returns.

## Evidence

- **Updater resolves manager from active package root first:** `resolveGlobalManager(...)` checks `detectGlobalInstallManagerForRoot` for package installs before any presence-based fallback, so the update flow starts from the launched install root.

  - file: [src/infra/update-global.ts:371](https://github.com/openclaw/openclaw/blob/f85806b6d78698fb6b6060d09e16cb09ae40131b/src/infra/update-global.ts#L371)

  - file: [src/infra/update-global.ts:465](https://github.com/openclaw/openclaw/blob/f85806b6d78698fb6b6060d09e16cb09ae40131b/src/infra/update-global.ts#L465)

  - file: [src/cli/update-cli/update-command.ts:347](https://github.com/openclaw/openclaw/blob/f85806b6d78698fb6b6060d09e16cb09ae40131b/src/cli/update-cli/update-command.ts#L347)
