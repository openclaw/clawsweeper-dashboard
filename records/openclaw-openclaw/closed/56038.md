## Summary

Close as `clawhub`: the requested WeChat native TTS voice-bar behavior belongs to the existing external `@tencent-weixin/openclaw-weixin` channel plugin, not OpenClaw core. Current `main` documents WeChat as external, assigns outbound media delivery to channel plugins, and already exposes a generic `audioAsVoice` outbound seam for plugin implementations to use.

## Evidence

- **WeChat channel is explicitly external:** The WeChat docs say OpenClaw connects through Tencent's external `@tencent-weixin/openclaw-weixin` plugin, that the WeChat code does not live in the OpenClaw core repo, and that WeChat-specific runtime/media/API work is owned by the external plugin.
  - file: [docs/channels/wechat.md:10](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/channels/wechat.md#L10)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Outbound delivery is plugin-owned by design:** The channel-plugin SDK docs assign outbound sending of text, media, and polls to the channel plugin, while core keeps the shared message tool and dispatch.
  - file: [docs/plugins/sdk-channel-plugins.md:21](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/plugins/sdk-channel-plugins.md#L21)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Project vision prefers optional capability in plugins, not core:** VISION says core should stay lean, optional capability should usually ship as plugins, the preferred path is plugin distribution from a separate repository, and plugin promotion belongs in ClawHub rather than core.
  - file: [VISION.md:52](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/VISION.md#L52)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Current main already provides a generic audio-as-voice seam:** Channel outbound context includes optional `audioAsVoice`, giving plugins a core-supported way to request native voice-note style delivery where their transport supports it.
  - file: [src/channels/plugins/outbound.types.ts:16](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/channels/plugins/outbound.types.ts#L16)
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Reply payload delivery preserves `audioAsVoice`:** The shared reply-payload sender forwards `audioAsVoice` into media sends, and the regression test verifies that the flag is preserved across payload media fallback sends.
  - file: [src/plugin-sdk/reply-payload.ts:295](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/src/plugin-sdk/reply-payload.ts#L295)
