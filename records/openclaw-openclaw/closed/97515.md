## Summary

Keep open; this PR is not merge-ready because the proposed bounded reader is used only after `response.body` is known to be null, where the helper falls back to `arrayBuffer()` before checking size and also breaks the existing text-only foreign response path.

## What This Changes

The PR routes the MCP HTTP fetch bodyless `text()` fallback through `readProviderTextResponse` and adds an oversized foreign-response regression test.

## Best Possible Solution

Choose an explicit MCP bodyless-response policy: either preserve text-only OAuth-error compatibility with honest post-read limits, or fail closed before unbounded reads with updated tests and real proof.

## Reproduction Assessment

## Solution Assessment

No. Reusing `readProviderTextResponse` is appropriate for stream-backed responses, but this branch runs only when `body` is null, where the helper cannot enforce a streaming cap and changes the existing foreign-response contract.
