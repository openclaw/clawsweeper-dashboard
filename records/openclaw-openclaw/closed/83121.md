## Summary

Current main already restores authenticated CLI runtime providers such as `claude-cli` in the shared `/models` provider data that Telegram renders, so the reported 2026.5.12 behavior is fixed on main but not yet in a tagged release.

## What This Changes

The issue asks for Telegram `/models` to expose/select the legacy Claude CLI runtime-backed provider lane after the provider/runtime split.

## Best Possible Solution

Keep the current scoped picker/catalog fix on main and let the next release carry it; any richer two-step runtime picker can be tracked separately as UX work.

## Reproduction Assessment

No. The reported behavior is reproducible on the 2026.5.12 release described by the reporter, but current main now source-proves the opposite path: authenticated `claude-cli` providers are included in `/models` data and Telegram renders that list.

## Solution Assessment

Yes. The current main fix is the narrow maintainable solution for this report: restore authenticated CLI runtime provider entries in `/models` while keeping shared setup/default picker filtering and auth gating intact.
