## Summary

Current `main` does not implement the MiniMax TTS env-style `apiKey` resolution described in PR #62352. The code still treats `"env:MINIMAX_API_KEY"` as an ordinary string, that string wins over the real env fallback during synthesis, and the current tests do not cover the regression cases from the PR discussion. The PR discussion also shows the author addressed the review follow-ups, so there is not strong evidence for a cleanup close.

## Evidence

- **Secret-input normalization keeps marker strings literal:** `normalizeSecretInputString()` returns any non-empty string unchanged, and `resolveSecretInputString()` marks that string as `available` before any SecretRef coercion. That means `"env:MINIMAX_API_KEY"` is treated as a plain string, not resolved from process env.
