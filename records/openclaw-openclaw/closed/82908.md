## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Parse each lsof file set before filtering clients:** `src/infra/ports-inspect.ts:150-155`
  - body: `parseLsofConnectionFieldOutput` loops over `parseLsofFieldOutput`, but that helper stores only the first `n` address for a PID. If a process has multiple established sockets and the retained address is the server-side one, deep status/doctor filters it out and misses the stale client process.
  - confidence: 0.92
- **[P2] Force numeric netstat output for Windows connections:** `src/infra/ports-inspect.ts:579`
  - body: `readWindowsEstablishedConnections` runs `netstat -ano -p tcp` without `-n`, so Windows can emit host or service names. `parseNetstatConnections` then compares those names to numeric local addresses and can drop the stale local client entries this diagnostic needs to show.
  - confidence: 0.9
