## Summary

Current main already fixes the Windows `pnpm ui:build` failure when `pnpm.cmd` resolves under `Program Files`. The root script still routes `ui:build` through `scripts/ui.js`, and that wrapper now detects Windows `.cmd/.bat/.com` launchers, quotes the resolved command before both async and sync spawns, and installs the full UI dependency set before build. Focused regression coverage asserts the exact `C:\Program Files\nodejs\pnpm.cmd` quoting behavior. The linked #52291 and later duplicate #55101 correctly identified the same issue, but the current main implementation covers it.

## Best Possible Solution

Close #52282 as implemented on current main. Keep the shipped `scripts/ui.js` quoting behavior and regression test; handle the still-open #52291 separately as superseded by the main-branch implementation if maintainers want to clean up the duplicate PR.
