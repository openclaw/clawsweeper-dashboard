## Summary

Current main already supports the user-facing `/active_memory` path through shared underscore/hyphen command matching and Telegram command-name normalization while preserving the canonical `active-memory` plugin key. The PR branch is therefore redundant for the observable command behavior and also unsafe as-is because it renames the canonical key that tests and integrations still use.

## What This Changes

The PR changes Active Memory command help text and the registered plugin command name from `active-memory` to `active_memory`.

## Best Possible Solution

Keep the canonical `active-memory` plugin command key, rely on the existing underscore alias and Telegram normalization for `/active_memory`, and only consider a non-breaking help/docs update that lists both accepted forms if maintainers want that UX polish.

## Reproduction Assessment

Yes. The current-main reproduction path is source-backed: register a hyphenated plugin command and call `matchPluginCommand("/active_memory status")`, which is covered by `src/plugins/commands.test.ts`; the PR regression path is also clear because Active Memory tests look up `registeredCommands["active-memory"]`.

## Solution Assessment

No. Renaming the canonical command key is not the best fix because main already handles the underscore user-facing form; the maintainable path is to preserve the existing key and use aliases or copy-only documentation changes for surface-specific naming.
