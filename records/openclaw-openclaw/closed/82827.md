## Summary

Keep this open. Current main and v2026.5.12 still have source-level paths that lowercase Signal group IDs in routing/session keys, and there are open linked fix PRs that explicitly close this issue, so the right cleanup is to review and land or close those PRs rather than close the issue early.

## What This Changes

The issue reports a v2026.5.12 regression where Signal group auto-replies fail because shared session/routing/store normalization lowercases opaque mixed-case Signal group IDs.

## Best Possible Solution

Review and land one focused fix PR, preferably https://github.com/openclaw/openclaw/pull/82840 if maintainers approve it, then close this issue with the merged commit and any remaining live Signal follow-up split separately.

## Reproduction Assessment

Yes, from source inspection rather than a live Signal daemon: current main passes Signal group IDs into generic route/session-key code that lowercases non-DM peer IDs, full route keys, store keys, and explicit session keys.

## Solution Assessment

Yes. Preserving opaque Signal group ID casing in the shared session-key builder and key normalizers is the narrow maintainable fix; this issue should remain open until the linked implementation PR merges or is rejected.
