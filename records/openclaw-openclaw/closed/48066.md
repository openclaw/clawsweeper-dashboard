## Summary

Current main addresses the reported Feishu-loaded gateway failure path: Feishu WebSocket startup now has monitor-owned retry/backoff and heartbeat defaults, the gateway channel runner contains channel task failures instead of letting them kill the process, and the observed local gateway WebSocket handshake-timeout symptom was separately mitigated and shipped in v2026.3.22. The Feishu-specific fix is on current main after the latest provided release, so fixedRelease is null.

## Best Possible Solution

Close this issue as fixed on current main, keep the Feishu retry/heartbeat implementation from #72411, and let any remaining Feishu WebSocket liveness or doctor-status work continue under the narrower open follow-ups #42354 and #55532.
