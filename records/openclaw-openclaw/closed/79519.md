## Summary

Keep open: the provider wildcard visibility direction is useful and the PR includes real Gateway output, but the current head still has a blocking Gateway `models.list` regression that bypasses the existing bounded read-only catalog path for wildcard configs.

## What This Changes

Adds `provider/*` model visibility entries across model selection, Gateway `models.list`, `/models`, Discord model pickers, docs, generated baselines, and focused tests.

## Best Possible Solution

Keep the `provider/*` visibility contract, but preserve the read-only 750 ms timeout path for every non-`all` Gateway `models.list` view and apply wildcard filtering after the bounded catalog selection.

## Reproduction Assessment

Yes, by source inspection: configure `agents.defaults.models` with a `provider/*` entry and call default/configured `models.list` while provider discovery is slow or stalled. The PR head awaits `readOnly: false` before the timeout branch can run.

## Solution Assessment

No as submitted. `provider/*` is a reasonable compact contract, but Gateway model listing should keep the existing read-only timeout fallback and then filter wildcard providers.
