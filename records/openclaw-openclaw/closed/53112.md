## Summary

Keep this PR open. Current main still has only the softer Slack docs text saying channel allowlists should use stable IDs, and the troubleshooting section does not include the proposed allowlist/name-key failure callout. The PR is a focused docs candidate, but its wording should be revised to account for supported exceptions: startup name resolution can add ID entries when token access allows, and `dangerouslyAllowNameMatching` can explicitly enable name candidates.

## Best Possible Solution

Keep the PR open as a focused docs candidate. The best path is to revise the warning to say unresolved name-based keys are ignored for allowlist routing by default, recommend stable Slack channel IDs, and mention the two exceptions: startup resolution when token access can resolve names, and the break-glass `dangerouslyAllowNameMatching` setting. Then keep the troubleshooting callout and run the relevant docs check/render review.
