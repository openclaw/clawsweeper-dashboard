## Summary

Current `main` already has a more general synchronous `before_message_write` plugin hook that intercepts persisted transcript messages, including assistant messages, after assistant sanitization and before the raw session append. That covers the requested `before_assistant_persist` interception point well enough.

## Evidence

## Risks / Open Questions

- The current hook is generic (`before_message_write`) rather than assistant-specific and does not expose the exact optional token-estimate fields proposed in the issue. If maintainers consider those extra fields essential, a follow-up enhancement could still be reasonable, but the core interception capability already exists.

## Close Comment

Closing as implemented. Codex review checked current `main` and the shipped `v2026.4.22` tag, and OpenClaw already provides the requested write-time interception point via the synchronous `before_message_write` hook. That hook is part of the public plugin API in `src/plugins/hook-types.ts:74` and `src/plugins/hook-types.ts:715`, and the session persistence path calls it for assistant messages after `sanitizeToolCallInputs()` and before the raw append in `src/agents/session-tool-result-guard.ts:180` and `src/agents/session-tool-result-guard.ts:243`. Plugins can already rewrite or block persisted assistant messages there, which covers the session-corruption prevention use case from this report. There is also regression coverage for blocking/mutating persisted messages in `src/agents/session-tool-result-guard.test.ts:402` and assistant transcript mirroring in `extensions/codex/src/app-server/transcript-mirror.test.ts:95`.
