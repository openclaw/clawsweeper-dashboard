Overall confidence: 0.88

Full review comments:

- **[P1] Pin self-hosted discovery to the configured host:** `src/plugins/provider-self-hosted-setup.ts:71-72`
  - body: This still uses `allowPrivateNetwork: true` for whatever `baseUrl` the user supplied. In the SSRF guard, that flag skips private-network checks globally, so a value like a cloud metadata URL would still be allowed and receive the configured bearer token. Keep private-network support scoped to the configured self-hosted host, as current main does with a hostname allowlist and explicit metadata-host block.
  - confidence: 0.91
- **[P1] Do not silently clamp external hook priorities:** `src/plugins/registry.ts:963-978`
  - body: `api.on(..., { priority })` is documented as numeric ordering, and terminal hook behavior depends on that ordering. Clamping all external priorities above 100 silently changes existing plugin behavior and can make previously ordered hooks tie or run later, including approval/blocking hooks. This needs a maintainer-approved contract and migration path, or the cap should be removed from this PR.
  - confidence: 0.86
- **[P2] Match aliases after earlier import specifiers:** `src/security/skill-scanner.ts:203`
  - body: The `import-aliasing` pattern only matches when `exec`/`spawn` is the first specifier inside the braces. `import { readFile, exec as run } from "child_process"` still bypasses this new rule, and the later `run(...)` call will not match `dangerous-exec`. The rule needs to handle aliases anywhere in the import list.
  - confidence: 0.84
