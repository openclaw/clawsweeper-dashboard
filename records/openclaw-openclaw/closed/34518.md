## Summary

Current main already ships a different, documented solution for `delivery.mode="none"`: isolated cron still resolves the chat target so the agent can use the shared `message` tool, while runner fallback delivery stays disabled unless `deliveryRequested` is true. This PR's change would regress the current contract rather than fix a still-open bug on main.

## Evidence

- **Runtime contract on main:** `resolveCronToolPolicy` keeps the message tool enabled for `deliveryMode: "none"`, and `resolveCronDeliveryContext` still resolves non-webhook targets, including `channel ?? "last"`.
  - file: [src/cron/isolated-agent/run.ts:285](https://github.com/openclaw/openclaw/blob/9153598e659daf5a52c479731f5b4ab8a5bf16b7/src/cron/isolated-agent/run.ts#L285)
