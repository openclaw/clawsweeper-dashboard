Keep open. The PR is focused, has sufficient live proof for the main shortcut paths, and CI is green, but one correctness issue remains: `N` can still trigger when the New messages affordance is hidden during manual refresh.

## What This Changes

Adds a guarded document-level Control UI shortcut dispatcher for `/`, `N`, and Escape, with browser coverage for editable-target guards.

## Best Possible Solution

Land the dispatcher after matching the `N` shortcut to the existing visible-affordance predicate and adding a regression check for the manual-refresh no-op case.

Yes for the review finding: source inspection shows PR head checks only `chatNewMessagesBelow`, while current rendering hides New messages when `chatManualRefreshInFlight` is true. The broader PR is a feature request rather than a broken existing-behavior report.

## Solution Assessment

Mostly yes. A guarded app-shell dispatcher is the right shape, but the `N` path should reuse the same visibility predicate as the existing button.
