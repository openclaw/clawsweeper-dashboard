## Summary

Current `main` no longer sends text-only cron announce deliveries down a single unchunked Telegram `sendMessage` path. Cron delivery now routes text-only announce output through the outbound adapter stack, where Telegram advertises a chunker and limit, and the send layer still splits long Telegram text into multiple messages. Regression coverage was added with the same main-branch change.

## Evidence

- **Cron announce now uses outbound adapter for text-only delivery:** `dispatchCronDelivery` now explicitly finalizes text-only cron output and then sends it through direct outbound delivery instead of the old internal announce route. The branch comment says text-only cron runs should go through the real outbound adapter, and `finalizeTextDelivery()` ends by calling `deliverViaDirectAndCleanup(...)`.
