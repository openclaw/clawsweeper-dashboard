## Summary

Current main already fixes the reported v2026.4.24 Bonjour/CIAO crash-loop: the bundled Bonjour plugin wires scoped CIAO rejection/exception handlers into the gateway runtime, the global unhandled-rejection path honors those handlers before fatal exit, and the advertiser disables itself after repeated stuck probing/announcing restarts instead of crash-looping the gateway. The fix is in the Unreleased changelog, so no shipped release tag was identified yet.

## Best Possible Solution

Close this issue as implemented on main. Keep the current scoped Bonjour/CIAO process-handler implementation and stuck-advertiser self-disable behavior, ship it in the next release, and leave v2026.4.24 users on the documented OPENCLAW_DISABLE_BONJOUR=1 or plugin-disable workaround until they upgrade.
