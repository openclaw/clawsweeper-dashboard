## Summary

Current main and the shipped bootstrap budget code already separate per-file bootstrap limits from aggregate bootstrap limits, so the reported aggregate-stack false alert is no longer necessary to track as open core work.

## What This Changes

The issue requests per-file Sentinel bootstrap threshold measurement for multi-file SOUL.md stacks, plus separate aggregate warning and critical thresholds.

## Best Possible Solution

Keep the existing per-file plus total bootstrap budget model; operators should tune `bootstrapTotalMaxChars` for aggregate pressure instead of adding another measurement mode.

## Reproduction Assessment

No. Source inspection shows current main and the latest inspected release separate per-file and aggregate bootstrap limits, and the reported 48,101-character aggregate is below the current 60,000-character aggregate default.

## Solution Assessment

Yes. The existing `bootstrapMaxChars` plus `bootstrapTotalMaxChars` model is the narrow maintainable solution; adding `bootstrapMeasurement` would duplicate an already documented budget split.
