## Summary

Current main already addresses the reported exec-permissions failure path. OpenClaw now defaults fresh local setups back to `tools.profile: "coding"`, current tool-policy resolution re-exposes `exec`/`process` whenever `tools.exec` is configured even under `profile: "messaging"`, and current troubleshooting/docs describe the supported host-exec recovery path without requiring a profile switch or global sandbox disable.
