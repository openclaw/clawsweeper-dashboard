## What This Changes

The issue reports that an embedded subagent announce timeout can leak the parent session write lock in memory, blocking later writes until gateway restart.

## Best Possible Solution

Add a focused regression around direct subagent announce timeout and embedded-run/lane timeout cleanup, then make the timeout path explicitly abort or unwind the accepted run so the session write lock is released and announce retry state remains recoverable.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection does show the reported direct announce timeout path and a plausible gap between gateway wait timeout and accepted-run cleanup.

## Solution Assessment

Unclear. A finally-based lock release is directionally right, but the maintainable fix should use the existing EmbeddedAttemptSessionLockController, abort controller, and watchdog contracts rather than adding a separate ad hoc release path.
