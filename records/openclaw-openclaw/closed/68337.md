## Summary

Current main already implements the substance of PR #68337. The ACP agents docs now tell operators that harness adapters/auth must exist on the host and that the harness command must start on the Gateway host, while the sandbox compatibility section explicitly says ACP runs on the host runtime rather than the OpenClaw sandbox. Source and tests enforce the same host-side ACP boundary.

## Best Possible Solution

Close the draft PR as implemented on current main and keep the consolidated ACP agents docs as the canonical user-facing guidance. A future refinement can add Docker-specific container wording against the current page if maintainers want that extra explicitness, but the older five-line patch is no longer needed.
