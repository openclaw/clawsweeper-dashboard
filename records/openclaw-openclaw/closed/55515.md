## Summary

Current `main` no longer has the duplicate core `/new` notice path described in #55515. Bare reset/new turns are covered by a test that asserts no standalone reset notice is routed, while verbose mode still emits only the single `🧭 New session: …` payload.

## Evidence

- **Current verbose notice path is singular:** `runReplyAgent` prepends exactly one verbose new-session notice when `verboseEnabled && activeIsNewSession`.
  - file: [src/auto-reply/reply/agent-runner.ts:1493](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/auto-reply/reply/agent-runner.ts#L1493)
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Standalone reset notices are now conditional on hook messages:** Reset-hook reply routing only happens when `hookEvent.messages.length > 0`; there is no unconditional built-in `/new` notice in this path.
  - file: [src/auto-reply/reply/commands-reset-hooks.ts:119](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/auto-reply/reply/commands-reset-hooks.ts#L119)
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Regression test covers bare `/new` turns:** The test named `does not send a standalone reset notice for reply-producing /new turns` asserts `routeReply` is not called for `/new` and that the turn stays threaded through the normal agent reply path.
  - file: [src/auto-reply/reply/get-reply-run.media-only.test.ts:503](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/auto-reply/reply/get-reply-run.media-only.test.ts#L503)
  - sha: [d3d9b5738f83](https://github.com/openclaw/openclaw/commit/d3d9b5738f83d92787f57db9186ddcebda60af9a)
- **Bare reset turns deny implicit extra threading:** `get-reply-run` forces `implicitCurrentMessage: "deny"` for bare session resets, which keeps the reset turn on the main reply path instead of emitting an extra standalone reply to the trigger message.
  - file: [src/auto-reply/reply/get-reply-run.ts:771](https://github.com/openclaw/openclaw/blob/d3d9b5738f83d92787f57db9186ddcebda60af9a/src/auto-reply/reply/get-reply-run.ts#L771)

## Risks / Open Questions

- Custom/workspace hooks can still push `event.messages` for `command:new` or `command:reset`, so a local hook/plugin could recreate duplicate notices even though core `main` no longer does.
