Fix Anthropic-on-Vertex requests that fail when redundant payload shaping produces more than four `cache_control` blocks.

## Best Possible Solution

Keep the merged single-pass transport design and include commit `0fc5a57a34409782c8e0c9260cedbf788ed382d8` in the next OpenClaw release.

## Reproduction Assessment

Yes. The pre-fix Vertex wrapper reapplied cache policy to a payload already budgeted by the shared transport, and the merged fix plus regression test directly cover that source-level five-marker path.

## Solution Assessment

Yes. Removing the duplicate plugin-level shaping pass is narrower and more maintainable than disabling caching, adding configuration, or duplicating the shared marker budget in the plugin.
