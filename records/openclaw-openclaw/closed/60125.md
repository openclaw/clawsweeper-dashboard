## Summary

The reported Control UI exec-approval overflow problem is already fixed on `main` and was shipped in `v2026.4.15` via the release-note entry for PR #67082.

## Evidence

- **Current modal layout bounds the approval card:** The current Control UI CSS caps the modal height with `max-height: calc(100dvh - 48px)` and keeps the card in a column layout, preventing the approval surface from growing past the viewport.
  - file: [ui/src/styles/components.css:2991](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/ui/src/styles/components.css#L2991)
