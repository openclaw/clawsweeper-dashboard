## Summary

Current main implements the PR’s intended default-agent `--to` session routing, with broader compatibility handling than the original patch. The later merged #72414 carried forward this PR’s analysis, and current source/tests now derive `agent:<default>:main`, use the default agent’s store when `--agent` is omitted, and preserve legacy main-store rows.

## Best Possible Solution

Close this PR as implemented on main. Keep the compatibility-preserving implementation from the later merged #72414, since it solves the original default-agent `--to` routing bug while also preserving legacy main-store session continuity.
