## Summary

Current main and the shipped v2026.5.22 release already propagate OpenAI TTS tool audio as Telegram voice-note media, so the reported 2026.5.20 text-only failure is fixed by upgrade.

## What This Changes

Fix OpenAI TTS tool output so Telegram receives a voice note instead of only a text reply when synthesis succeeds.

## Best Possible Solution

Keep the shipped v2026.5.22/current-main tool-result media propagation and ask for a fresh report only if OpenAI Telegram TTS still fails after upgrade.

## Reproduction Assessment

Yes for the reported 2026.5.20 behavior: the issue includes deterministic Telegram DM steps and logs comparing OpenAI text-only delivery with Google sendVoice. I did not reproduce a current-main failure because current source and v2026.5.22 show the failing handoff now propagates audioAsVoice.

## Solution Assessment

Yes. The shipped fix is provider-agnostic: OpenAI marks voice-note opus output voiceCompatible, the TTS tool converts that into structured audioAsVoice media, and Telegram consumes that flag through sendVoice without adding a new config path.
