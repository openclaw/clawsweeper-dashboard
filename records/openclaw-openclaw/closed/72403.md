## Summary

PR #72403 should close as plugin/ClawHub-scoped. It is a user-specific Bucky/JARVIS deployment bundle and daemon, with personal GCP/WhatsApp wiring and unsafe global core edits, while current main already provides plugin, bundle, skill, MCP, hook, and web-search extension surfaces for this kind of optional assistant integration.

## Best Possible Solution

Close the core PR and ask the author to extract Bucky/JARVIS as an external ClawHub package or private plugin/bundle using the documented skills, hooks, MCP, tool, channel, provider, and web-search surfaces. If that extraction reveals a missing generic OpenClaw API, it should come back as a narrow design/API PR without personal deploy state, credentials, or global provider/text behavior changes.
