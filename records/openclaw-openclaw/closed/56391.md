## Summary

Keep open: the linked blank-tool-name bug still exists on current main, but this PR's literal `_blank` sentinel is not safe to merge because it can break streamed ID recovery, collide with real tool names, and preserve malformed replay blocks.

## What This Changes

The PR changes embedded-runner tool-call normalization so blank or whitespace-only tool names become `_blank`, updates loop classification for that sentinel, and adjusts stream-normalization tests.

## Best Possible Solution

Revise blank tool-call handling so malformed blank names are non-dispatchable and loop-broken while preserving id inference, stream recovery, and replay pruning contracts.

## Reproduction Assessment

Yes. The linked issue has concrete provider logs for repeated blank tool names, and the PR regressions are reproducible from source through current stream mutation, allowlist collection, and replay sanitization paths.

## Solution Assessment

No. Replacing blanks with a normal `_blank` dispatch string is not the narrowest maintainable fix; the safer path is an internal invalid/blank classification that cannot dispatch as a real tool.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve ID recovery for blank streamed calls:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:206`
  - body: Writing `_blank` into the mutable stream block means a blank-name delta with a later usable id is no longer blank on the next normalization pass, so id-based inference is skipped and a recoverable call can become `Tool _blank not found`.
  - confidence: 0.91
- **[P2] Prevent `_blank` from reaching dispatch:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:206`
  - body: The fallback is a normal tool name, while registered and client tool names accept arbitrary trimmed strings. If `_blank` is allowlisted, malformed blank model output can execute that real tool instead of failing closed.
  - confidence: 0.88
- **[P2] Keep blank replay calls pruned:** `src/agents/pi-embedded-runner/run/attempt.tool-call-normalization.ts:206`
  - body: `resolveReplayToolCallName` accepts any non-empty normalized name when no allowlist is present, so this changes blank assistant replay tool calls from pruned malformed history into `_blank` replay blocks.
  - confidence: 0.82
