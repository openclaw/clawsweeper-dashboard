## Summary

Current main implements the reported Windows Bonjour/@homebridge/ciao ARP console-window fix. The bundled Bonjour advertiser now installs a scoped Windows wrapper before loading ciao, recognizes the exact `arp -a | findstr /C:"---"` command, and forwards it to `child_process.exec` with `{ windowsHide: true }`; a colocated test covers the active-advertiser behavior and cleanup. The fix shipped in `v2026.4.26` via commit `276291d399edb8043dc0fbeae91cd1d5eeea09d3`. The earlier duplicate #71188 was correctly kept pointed at this issue before the fix landed; the changelog entry references the same user-visible symptom through #70238.

## Best Possible Solution

Keep the shipped scoped Bonjour fix in OpenClaw and close this issue as implemented. A future upstream ciao PR would still be useful cleanup, but the current release already solves the reported user-visible Gateway window flash for the bundled ciao path.
