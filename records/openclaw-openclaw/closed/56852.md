## Summary

Current `main` already implements optional user-visible model-fallback surfacing. The reply runner now records fallback transitions, emits structured lifecycle events, adds inline fallback notices when verbose mode is enabled, and the UI renders a compact fallback toast plus `/status` fallback metadata.

## Evidence

- **Fallback notice formatting exists:** `buildFallbackNotice()` and `buildFallbackClearedNotice()` generate concise user-facing fallback messages, including selected vs active model and summarized reason.
  - file: [src/auto-reply/fallback-state.ts:83](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/auto-reply/fallback-state.ts#L83)
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **Runner emits structured fallback events and optional inline notices:** The reply runner persists fallback state, emits `lifecycle` events with `phase: "fallback"` / `"fallback_cleared"`, and prepends fallback notice payloads when verbose mode is enabled.
  - file: [src/auto-reply/reply/agent-runner.ts:1287](https://github.com/openclaw/openclaw/blob/68315792677f9628d1370d6869366442f1d3ecad/src/auto-reply/reply/agent-runner.ts#L1287)
  - sha: [68315792677f](https://github.com/openclaw/openclaw/commit/68315792677f9628d1370d6869366442f1d3ecad)
- **UI surfaces compact fallback indicator:** Chat event handling converts fallback lifecycle events into `FallbackStatus`, and the chat status indicator renders a compact `Fallback active:` / `Fallback cleared:` toast with reason/attempt details in the tooltip.
