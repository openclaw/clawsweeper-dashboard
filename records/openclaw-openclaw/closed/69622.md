## Summary

Current main already routes Feishu inbound images into the generic inbound-media/media-understanding path, and MiniMax image understanding now uses the plugin-owned MiniMax-VL-01 provider instead of sending image work to text-only MiniMax M2.7. The reported pruning marker is now documented as an old-history cleanup marker, not the current-turn image payload.

## Best Possible Solution

Close as implemented on main. Keep the current Feishu inbound media -> generic media understanding -> MiniMax-VL-01 routing, and ask for a fresh issue with logs/config only if the behavior still reproduces on v2026.4.24 or newer.
