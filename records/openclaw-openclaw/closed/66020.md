## Summary

Keep open. Current main still rejects existing `<qqmedia>` local files outside the guarded OpenClaw media roots, and the requested auto-mirroring would intentionally broaden QQ Bot local-file upload authority.

## What This Changes

Request that QQ Bot automatically copy existing local `<qqmedia>` files outside `~/.openclaw/media/qqbot/` into QQ Bot media storage before sending.

## Best Possible Solution

Define a security-preserving staging policy for QQ Bot local media, either keeping the explicit media-root requirement or adding a constrained mirror path limited to approved OpenClaw-owned generated-output roots with canonicalization, symlink/traversal protection, size checks, docs, and regression tests.

## Reproduction Assessment

Yes. Source tracing on current main shows `<qqmedia>` content reaches `resolveOutboundMediaPath`, and existing local files outside the approved media roots still return `Media path must be inside QQ Bot media storage`.

## Solution Assessment

No for unrestricted auto-mirroring. A maintainable fix needs an explicit allowlist or staging contract rather than copying any existing local path into upload storage.
