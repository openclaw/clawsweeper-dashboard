## Summary

Current main already solves the PR's observable user problem: subagent completion announce-back preserves the parent/requester delivery account even when the child agent has a different channel binding. The shipped implementation uses a safer split than the PR: child startup/thread binding may still use the target agent's bound account, while the registered completion requester origin stays on the parent route. The PR's one-character guard flip remains risky for same-agent account preservation and should not be merged as-is.

## Best Possible Solution

Close this PR as implemented by the shipped mainline fix. Keep the current split: child startup/thread binding can use the target agent's bound account, while completion bookkeeping and announce routing preserve the parent/requester route. Do not merge the proposed guard flip unless a maintainer intentionally changes the broader cross-agent child-start routing contract with new coverage.
