## Summary

Current main and the latest release already preserve `sessions_spawn` ACP routing payloads during transcript repair, so the reported `streamTo: "parent"` redaction failure is fixed. The linked merged fix removed the redaction path entirely and the release tag contains the fixed sanitizer plus regression coverage.

## What This Changes

The issue reports that `sessions_spawn({ streamTo: "parent" })` can be rewritten to a redaction sentinel during transcript replay, breaking ACP parent-stream observability.

## Best Possible Solution

Keep the shipped transcript-repair behavior that treats local `sessions_spawn` tool payloads as trusted operator transcript state and preserves ACP routing fields.

## Reproduction Assessment

Yes for source reproduction: the old redaction helper, replay-safety reference check, current fixed sanitizer, and preservation tests are all traceable in source and history. I did not run the live Telegram/ACP intermittent scenario.

## Solution Assessment

Yes; the merged fix addresses the owner boundary by stopping transcript repair from redacting local `sessions_spawn` payloads, while the live ACP tool path continues forwarding `streamTo` normally.
