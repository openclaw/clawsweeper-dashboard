## Summary

Keep open: current main still has the source-backed redaction/replay mismatch, and https://github.com/openclaw/openclaw/pull/82203 is an open, unmerged closing PR for this issue. This should not be closed before that implementation path lands or is explicitly rejected.

## What This Changes

The issue reports that ACP `sessions_spawn` calls with `streamTo: "parent"` can lose parent streaming because transcript redaction rewrites the routing field to `__OPENCLAW_REDACTED__`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82203 if the narrow idempotent redaction fix holds; after it merges, close this issue as implemented.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction path: an already-redacted `sessions_spawn` block with ACP routing fields is re-sanitized into a new object, causing the replay-safe reference check to fail and the thinking assistant turn to be dropped. I did not establish a live Telegram/ACP run in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable solution is to make ACP argument redaction idempotent for already-redacted fields while preserving first-pass redaction. The linked PR appears to target that boundary instead of disabling transcript redaction broadly.
