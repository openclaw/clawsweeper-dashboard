## Summary

Keep open: current main still has the source-level doctor/config mismatch, and open PR #76879 uses closing syntax for this issue, so the issue should remain open until that PR is reviewed and merged or replaced.

## What This Changes

Doctor should stop warning that channel-derived `plugins.allow` entries such as `discord` are stale when gateway auto-enable re-adds them for enabled channels.

## Best Possible Solution

Review and land or replace #76879 so config validation no longer warns for recognized channel ids that gateway auto-enable manages, while true unknown plugin ids still warn.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with `channels.discord.enabled` and `plugins.allow: ["discord"]` in a package layout lacking the Discord plugin manifest, validation warns for a missing plugin while auto-enable can preserve or re-add the channel id.

## Solution Assessment

Yes. The narrow maintainable fix is to align plugin allowlist validation with the channel-aware stale-plugin scan, or otherwise stop auto-enable from writing non-plugin channel ids; the linked PR appears to target that validation side but remains unmerged.
