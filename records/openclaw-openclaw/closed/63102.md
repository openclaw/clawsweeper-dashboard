## Summary

Keep open. Current `main` still logs plugin tool-name conflicts during normal plugin-tool resolution, and QQ Bot tool registration still uses the same `qqbot_channel_api` / `qqbot_remind` names that would collide with an installed standalone `openclaw-qqbot` plugin. The report is also plausibly current because docs now say QQ Bot is bundled, while community docs still advertise the standalone Tencent package, so duplicate installs remain a credible path.

## Evidence

This path is used by agent plugin-tool resolution, so the same conflict can be logged repeatedly across runs.
