## Summary

Keep open. Current main still routes every parsed Android assistant launch to Chat, while promptless `ACTION_ASSIST` is already distinguishable and the Voice tab has no one-shot assistant mic activation path.

## What This Changes

Promptless Android `ACTION_ASSIST` launches should open Voice and auto-enable the mic once, while prompt-bearing `ASK_OPENCLAW` App Actions continue to Chat/draft.

## Best Possible Solution

Implement Android-side promptless assist routing to Voice with one-shot mic permission/activation, preserve prompt-bearing App Actions on Chat/draft, and update focused tests plus docs/changelog.

## Reproduction Assessment

Yes. The source-level path is high confidence: `ACTION_ASSIST` parses as promptless assist, `MainActivity` forwards it, `handleAssistantLaunch` requests Chat, and `VoiceTabScreen` has no assistant-origin mic activation path.

## Solution Assessment

Yes. The narrow maintainable fix is to branch only promptless `source="assist"` launches in `MainViewModel` and consume a one-shot Voice activation flag in `VoiceTabScreen`, without changing manifest, gateway config, or App Actions metadata.

## Repair Work Prompt

Fix the Android assistant gesture routing for https://github.com/openclaw/openclaw/issues/65414. On promptless `ACTION_ASSIST` launches (`source == "assist"` and `prompt == null`), route to `HomeDestination.Voice`, clear stale chat launch state, and set a one-shot `pendingVoiceAutoActivate` flag. In `VoiceTabScreen`, consume that flag once from a `LaunchedEffect`: if mic permission is granted, call `setMicEnabled(true)`; if not, request `RECORD_AUDIO` and enable the mic only if the grant callback succeeds. Preserve prompt-bearing `ASK_OPENCLAW` App Actions on Chat/draft and do not re-enable auto-send from #70714. Do not change `AndroidManifest.xml`, App Actions metadata, gateway config/schema, or Talk Mode behavior. Add focused Android tests for promptless assist routing/one-shot consumption and prompt-bearing App Action preservation, then update `docs/platforms/android.md` and `CHANGELOG.md` because this is user-facing.
