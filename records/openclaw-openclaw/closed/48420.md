## Summary

Current main and the shipped v2026.4.26 tag implement the reported auth-header split. Anthropic sk-ant-oat credentials now go through the Bearer-token path, regression tests cover the old x-api-key failure mode, the direct Anthropic media/PDF bypass noted in the comments now delegates through the shared model transport, and OpenAI OAuth is separated into the documented openai-codex route rather than direct openai API-key auth.

## Best Possible Solution

Close this issue as implemented. Keep the shipped Anthropic Bearer-token transport split and the explicit OpenAI openai-codex OAuth route; any remaining desire for one-click Anthropic setup belongs in the separate feature discussion already tracked by #32515.
