## Summary

## What This Changes

The issue reports that empty REM dreaming results are written as candidate-like markdown and can later be promoted into long-term MEMORY.md.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80916, or apply the same narrow source and regression-test changes on main while coordinating with the broader placeholder-promotion guard work.

## Reproduction Assessment

Yes, source-level. Current main still writes the empty REM placeholder and the promotion filter does not reject that exact placeholder-shaped candidate before apply.

## Solution Assessment

Yes. The narrow maintainable fix is to stop producing candidate-like markdown for empty REM results and defensively filter already-written historical placeholders during promotion; the linked PR follows that path.
