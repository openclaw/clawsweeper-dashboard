## Summary

Close. Current `main` already prevents caller-provided `/v1/responses` tools from colliding with built-in agent tools by rejecting normalized name conflicts before session creation, and the gateway surfaces the condition as `400 invalid tool configuration`. The same behavior is present in release `v2026.4.23`, so the reported security gap is already fixed well enough on shipped code, albeit via rejection rather than the issue's proposed prefix-renaming approach.

## Risks / Open Questions

- The shipped fix rejects colliding caller tool names with `400 invalid tool configuration` rather than auto-renaming them with a `user_` prefix. That is sufficient to stop the built-in shadowing/bypass described in this issue, but it does not implement the exact UX proposed in the report.
