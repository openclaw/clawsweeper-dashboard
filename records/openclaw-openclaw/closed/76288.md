## Summary

Keep open: the PR addresses the linked `cron.run`/`cron.runs` correlation gap, but as submitted it reuses the public `manual:` acknowledgement ID as the internal task ledger ID and would bypass existing cron task recovery.

## What This Changes

The PR threads manual `cron.run` acknowledgement IDs through cron finished events, JSONL run history, gateway protocol/schema outputs, plugin hook typing, generated Swift models, regression coverage, SDK baseline hashes, and the changelog.

## Best Possible Solution

Carry the manual acknowledgement ID as a public history/correlation field while preserving the internal recoverable cron task ID, or add explicit manual-ID recovery semantics with regression coverage.

## Reproduction Assessment

Yes. Current main reproduces the original gap by source inspection: `cron.run` returns a `manual:...` ID, but finished events and JSONL history do not carry `runId`; the introduced blocker is reproducible by interrupting a manual cron task after task creation and before finalization, then running task maintenance.

## Solution Assessment

No, not as submitted. Propagating the public run ID is the right direction, but using it as the task ledger `runId` is not the narrowest safe fix because current maintenance recovery depends on `cron:<jobId>:<startedAt>` IDs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve recoverable cron task IDs:** `src/cron/service/ops.ts:512`
  - body: Passing the manual acknowledgement ID into `createRunningTaskRun` changes manual cron task records from the recoverable `cron:<jobId>:<startedAt>` form to `manual:...`. `parseCronExecutionId()` only recovers active cron tasks with `cron:` IDs, so a gateway restart before finalization can leave the task unrecovered and mark it lost even when cron history has a terminal entry. Keep the task ledger ID as the cron execution ID and carry the manual ID separately, or add equivalent manual-ID recovery coverage.
  - confidence: 0.87
