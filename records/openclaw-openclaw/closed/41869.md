## Summary

Current main already fixes the user-visible cron contradiction behind this PR, but at the isolated-run outcome layer instead of by clearing `lastError` inside `applyJobResult`. The review comments on this PR were useful: they warned that clearing `lastError` while keeping `lastStatus: "error"` would remove diagnostics and could interfere with retry counting, and the main implementation avoids that by returning `status: "ok"` with no error when the presentation warning was actually delivered.

## Best Possible Solution

Close this stale PR and keep the mainline implementation: classify delivered Message/Canvas presentation warnings correctly before `applyJobResult`, while preserving `lastError`, retry counting, and failure alerts for genuine error statuses. The linked bug can be handled against the mainline fix rather than merging this older diff.
