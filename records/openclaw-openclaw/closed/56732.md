## Summary

Current `main` already avoids the reported abort-path `maxAttempts=0` override and explicitly suppresses late Discord gateway `reconnect-exhausted` errors during teardown/dispose, with regression coverage for the reported `code 1005` path.

## Evidence

- **Abort path no longer rewrites reconnect attempts:** `waitForDiscordGatewayStop()` resolves aborts by disconnecting only; there is no `gateway.options.reconnect = { maxAttempts: 0 }` mutation in the current abort handler.
  - file: [extensions/discord/src/monitor.gateway.ts:54](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/discord/src/monitor.gateway.ts#L54)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Late gateway errors are suppressed instead of escaping:** `createDiscordGatewaySupervisor()` keeps an `error` listener installed and, in `teardown`/`disposed` phases, logs late `reconnect-exhausted` events instead of letting them become uncaught exceptions.
  - file: [extensions/discord/src/monitor/gateway-supervisor.ts:137](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/discord/src/monitor/gateway-supervisor.ts#L137)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Disconnect cleanup keeps listeners active through teardown:** `waitForDiscordGatewayStop()` intentionally removes lifecycle listeners only after `gateway.disconnect()` so late `error` events emitted during disconnect are still handled.
  - file: [extensions/discord/src/monitor.gateway.ts:36](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/discord/src/monitor.gateway.ts#L36)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Regression test covers reported crash shape:** Startup cleanup test emits `Error("Max reconnect attempts (0) reached after code 1005")` after disconnect and asserts it does not throw, and that the runtime logs a suppressed late reconnect-exhausted error.
  - file: [extensions/discord/src/monitor/provider.test.ts:281](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/discord/src/monitor/provider.test.ts#L281)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Supervisor test covers dispose-time code 1005 path:** Unit test emits `Max reconnect attempts (0) reached after close code 1005` after `dispose()` and asserts `emitter.emit("error", ...)` does not throw.
  - file: [extensions/discord/src/monitor/gateway-supervisor.test.ts:107](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/discord/src/monitor/gateway-supervisor.test.ts#L107)
  - sha: [6b618f063516](https://github.com/openclaw/openclaw/commit/6b618f063516aa9ba85903aa4565c5a204c055b2)
- **Default reconnect budget remains 50:** The Discord gateway plugin still initializes Carbon with `reconnect: { maxAttempts: 50 }`, matching the expectation in the report rather than forcing `0` on the abort path.
  - file: [extensions/discord/src/monitor/gateway-plugin.ts:377](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/extensions/discord/src/monitor/gateway-plugin.ts#L377)
