## Summary

Keep open. Current `main` still logs the full dashboard URL, including `#token=...`, for literal/non-SecretRef gateway tokens, and the captured log path plus `logs.tail` exposure still exists. The issue body's `logs.tail` file/line references are stale, but the underlying leak remains on current `main` at `ebd7f19a3b20001960474ec9ba9cc424e45f7b23`.

## Risks / Open Questions

- The issue body's old `src/gateway/server-methods/logs.ts` line references no longer match current `main`; log-tail now flows through `src/logging/log-tail.ts`, so a maintainer should evaluate the current implementation rather than the original file offsets.
- Exploitability still depends on the gateway using a literal or fallback shared token rather than a SecretRef-managed token, because SecretRef-managed tokens already take the non-tokenized branch.
- `logs.tail` now applies generic redaction on read, but the default rules do not appear to scrub URL-fragment tokens, so the read-side mitigation is insufficient for this case.
