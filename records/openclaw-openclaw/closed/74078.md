## Summary

Close as implemented on current main. The current agent/local-model path now resolves explicit Ollama-style model refs through target-provider runtime hooks while skipping PI discovery, and the logs CLI now falls back to the configured file log when implicit loopback Gateway RPC closes or times out during diagnostics. The fix is on main only; no tag currently contains the proof commit.

## What This Changes

The issue reports that OpenClaw 2026.4.26 on Ubuntu via npm hangs or disconnects after a few minutes when using a local Ollama model, with `openclaw logs` failing against the local Gateway websocket.

## Best Possible Solution

Keep the current main implementation, ship it in the next release, and leave the narrower `logs --follow` resilience work to the separate open tracker for that specific live-tail behavior.
