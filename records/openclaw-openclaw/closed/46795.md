## Summary

Current `main` already contains the ACP parent-stream relay path and the cross-chunk shared agent-event singleton needed for `sessions_spawn(runtime: "acp", streamTo: "parent")` to surface child progress back to the parent session. The same implementation is present in release `v2026.4.22`, so this looks fixed on released code.

## Evidence

- **Agent event bus is now a global singleton across chunk/module splits:** `getAgentEventState()` stores shared state behind `Symbol.for("openclaw.agentEvents.state")` via `resolveGlobalSingleton(...)`, so duplicated module instances share listeners, run context, and sequence state instead of silently splitting the bus.
  - file: [src/infra/agent-events.ts:127](https://github.com/openclaw/openclaw/blob/bd1b8448a523eabc757e9d2eaae3e9a324a6cb54/src/infra/agent-events.ts#L127)
  - sha: [bd1b8448a523](https://github.com/openclaw/openclaw/commit/bd1b8448a523eabc757e9d2eaae3e9a324a6cb54)
- **Regression test covers duplicate-module event bus sharing:** The test explicitly imports the module twice with cache-busting URLs and verifies that events emitted from one instance are received by listeners registered from the other instance, matching the reported cross-bundle/singleton-split failure mode.
  - file: [src/infra/agent-events.test.ts:161](https://github.com/openclaw/openclaw/blob/bd1b8448a523eabc757e9d2eaae3e9a324a6cb54/src/infra/agent-events.test.ts#L161)
  - sha: [bd1b8448a523](https://github.com/openclaw/openclaw/commit/bd1b8448a523eabc757e9d2eaae3e9a324a6cb54)
- **ACP spawn registers the parent relay before dispatch:** `spawnAcpDirect()` starts the parent relay before the gateway `agent` call, then rebinds it if the returned `runId` differs from the idempotency key. This closes the timing hole where fast child events could be missed before the relay subscribed.
  - file: [src/agents/acp-spawn.ts:1251](https://github.com/openclaw/openclaw/blob/bd1b8448a523eabc757e9d2eaae3e9a324a6cb54/src/agents/acp-spawn.ts#L1251)
  - sha: [bd1b8448a523](https://github.com/openclaw/openclaw/commit/bd1b8448a523eabc757e9d2eaae3e9a324a6cb54)
- **Parent relay forwards assistant/lifecycle events and has direct regression coverage:** `startAcpSpawnParentStreamRelay()` subscribes to `onAgentEvent(...)`, forwards assistant deltas plus lifecycle completion/error into parent-session system events, and the paired tests cover progress, no-output stall, resume, and completion behavior.
  - file: [src/agents/acp-spawn-parent-stream.ts:307](https://github.com/openclaw/openclaw/blob/bd1b8448a523eabc757e9d2eaae3e9a324a6cb54/src/agents/acp-spawn-parent-stream.ts#L307)
  - sha: [bd1b8448a523](https://github.com/openclaw/openclaw/commit/bd1b8448a523eabc757e9d2eaae3e9a324a6cb54)
