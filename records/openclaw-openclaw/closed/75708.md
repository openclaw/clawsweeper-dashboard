## Summary

Keep open because current main still has the reported per-task full-registry relist, and an open same-author PR explicitly targets this issue but has not merged.

## What This Changes

The issue requests eliminating a quadratic task-registry maintenance path by reusing the sweep's task snapshot instead of relisting during each per-task ACP cleanup check.

## Best Possible Solution

Review and land PR #75709, or an equivalent narrow patch, so terminal ACP cleanup reuses the initial maintenance snapshot while retaining the final fresh snapshot for orphaned parent-owned ACP sessions and adding the counting regression test.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction is to configure many active task records with a counting listTaskRecords runtime and run runTaskRegistryMaintenance; current main will call the full registry listing once for the initial sweep, once per processed task before ACP cleanup, and once for final orphan cleanup.

## Solution Assessment

Yes. Reusing the already captured tasks snapshot for cleanupTerminalAcpSession(current, tasks) is the narrowest maintainable fix because that helper only needs the supplied task array for active child-session checks and does not require a fresh global listing per task.
