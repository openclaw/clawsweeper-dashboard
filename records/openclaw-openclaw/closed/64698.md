## Summary

Current main, and the latest release v2026.4.24, already honor Telegram group-level `requireMention: false` for the reported multi-account/config shape. Account-scoped `groups` are resolved, the inbound path preserves explicit `false`, and the `no-mention` skip only runs when the resolved value is true. The narrower per-topic override concern remains separately tracked by #49864/#49980.

## Best Possible Solution

Close this issue as implemented on main and keep the shipped Telegram account/group mention-gating behavior. Leave the narrower per-topic override work to #49864/#49980 rather than keeping this broader group-level report open.
