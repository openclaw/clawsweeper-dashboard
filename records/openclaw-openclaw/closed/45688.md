## Summary

Keep open: current main still has the Control UI config discoverability regression. Advanced Settings renders through the accordion path, but that path still lacks the visible search control and still scopes form results to the active section before search matching.

## What This Changes

Restore v1-level config discoverability in the v2 Control UI by making Advanced Settings globally searchable/filterable again.

## Best Possible Solution

Advanced Settings should keep the intended v2 grouping while exposing search/tag filter controls in accordion mode and letting nonempty searches match settings across all included Advanced Settings sections.

## Reproduction Assessment

Yes. Static inspection against current main reproduces it: Advanced Settings selects accordion layout, the accordion branch omits the search input, and form filtering applies `activeSection` before search matching.

## Solution Assessment

Yes for a narrow repair. Restoring accordion-mode search/filter and making search global across Advanced Settings is smaller and safer than reverting the whole v2 settings redesign.
