## Summary

Close as duplicate: the Feishu failure is the same configured-channel/plugin-activation gap already tracked by https://github.com/openclaw/openclaw/issues/83212, where valid channel config can still leave the owning plugin disabled and invisible to Gateway startup.

## What This Changes

Feishu should be recognized and started by Gateway after the plugin is installed and `channels.feishu` is configured, instead of remaining `installed, not configured, disabled`.

## Best Possible Solution

Fix the shared channel-plugin activation path once in https://github.com/openclaw/openclaw/issues/83212 so configured Feishu and Discord channels either activate their owning plugin or fail with an actionable operator message while preserving explicit plugin disablement.

## Reproduction Assessment

Yes. The source path is clear: a configured channel whose owning plugin is not effectively activated is absent from `listChannelPlugins()`, so Gateway status returns no channel account snapshot.

## Solution Assessment

No, not as a separate issue. The narrow maintainable path is to solve the generic activation mismatch in the canonical open issue and use this Feishu report as additional channel-specific evidence.
