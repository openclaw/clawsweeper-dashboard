## Summary

Current `main` already handles the reported `toolResult.content[0].text` shape in Control UI, and the same implementation is present in the shipped `v2026.4.22` tag.

## Evidence

  - file: [ui/src/ui/chat/grouped-render.ts:1334](https://github.com/openclaw/openclaw/blob/53aac30f5104052db56da2b26bcee0f4e5140325/ui/src/ui/chat/grouped-render.ts#L1334)

- **Standalone tool results fall back to extracted message text:** When a message is a standalone tool/toolResult message and there are no inline `toolresult` blocks, `extractToolCards()` populates `outputText` from `extractTextCached(message)`. That matches the issue report's top-level `toolResult` WebSocket shape.
  - file: [ui/src/ui/chat/tool-cards.ts:191](https://github.com/openclaw/openclaw/blob/53aac30f5104052db56da2b26bcee0f4e5140325/ui/src/ui/chat/tool-cards.ts#L191)

- **Array-shaped `content[].text` is explicitly extracted:** `extractRawText()` handles `content` arrays by collecting every `{ type: "text", text: ... }` block and joining them. A tool result delivered as `content[0].text` therefore produces non-empty text instead of falling through to the empty-state path.
  - file: [ui/src/ui/chat/message-extract.ts:88](https://github.com/openclaw/openclaw/blob/53aac30f5104052db56da2b26bcee0f4e5140325/ui/src/ui/chat/message-extract.ts#L88)

With the extraction above, the reported payload shape no longer maps to "No output."
