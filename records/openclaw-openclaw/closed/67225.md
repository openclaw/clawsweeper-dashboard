## Summary

Keep open. Current `main` still computes ingress-run fallback thinking from session/global defaults only and does not honor `agents.list[].thinkingDefault`, so this PR appears to address a real behavior gap rather than duplicate existing code.

## Evidence

- **Ingress fallback path skips per-agent thinking default:** In `agentCommandInternal`, ingress/local runs initialize `resolvedThinkLevel` from explicit or persisted overrides, then fall back to `resolveThinkingDefault({ cfg, provider, model, catalog })`. There is no lookup of `resolveAgentConfig(cfg, sessionAgentId)?.thinkingDefault` before that fallback.
  - file: [src/agents/agent-command.ts:567](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/agents/agent-command.ts#L567)
  - command: `nl -ba src/agents/agent-command.ts | sed -n '560,805p'`
  - sha: [7875092f4ddc](https://github.com/openclaw/openclaw/commit/7875092f4ddc6f2dbe218bea8506c9ac1da87c39)
- **Shared thinking resolver only reads global defaults:** `resolveThinkingDefault` checks per-model config and `cfg.agents?.defaults?.thinkingDefault`, then falls back to model heuristics. It does not read any per-agent entry from `agents.list[]`.
  - file: [src/agents/model-thinking-default.ts:13](https://github.com/openclaw/openclaw/blob/7875092f4ddc6f2dbe218bea8506c9ac1da87c39/src/agents/model-thinking-default.ts#L13)
