## Summary

Keep #70403 open. Current main still has light and REM phases appending Dream Diary entries, the docs still describe diary generation after each phase, and current tests assert light/REM narrative writes. The related PRs address adjacent dreaming bloat, fallback, dedupe, and raw-dump issues, but none fully supersedes this one-entry-per-sweep fix.

## Best Possible Solution

Keep the PR open and adapt/merge the focused behavior change: remove light/REM `generateAndAppendDreamNarrative` calls and related cleanup, leave full-sweep diary generation on the deep path, update `docs/concepts/dreaming.md` and the dreaming phase/narrative tests, and resolve the minor type-only import review comment. If #70332 or #70523 lands first, rebase this PR and keep only the remaining one-entry-per-sweep delta.
