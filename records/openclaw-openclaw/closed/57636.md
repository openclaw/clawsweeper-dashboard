## Summary

Current `main` already lets plugin hooks distinguish main-agent runs from subagent runs: `PluginHookAgentContext` carries `sessionKey`, the harness forwards that field into agent hooks, and the public plugin SDK exports `isSubagentSessionKey(...)` on the supported `openclaw/plugin-sdk/routing` surface. The reported blocker does not reproduce on `main` at `c65aa1d2a6e573d49619c4dd0698a76010476e7d`.
