## Summary

Current main already fixes the reported Dreaming session-picker pollution. The same Dreaming narrative session leak was tracked in #66358 and fixed by merged PR #67023; current code now uses stable hashed narrative session keys, deletes the transient Dreaming session through the plugin runtime, defensively scrubs stale Dreaming artifacts, and has regression coverage plus changelog release evidence for v2026.4.20.

## Best Possible Solution

Close this as implemented on main and keep the shipped memory-core cleanup path. The right fix is at the source of the leak, not a Control UI-only filter: Dreaming narrative runs should remain transient, be deleted through `sessions.delete`, and have stale artifacts scrubbed. Any separate UX for browsing intentionally retained internal/system sessions should be tracked as a new product request, not under this bug.
