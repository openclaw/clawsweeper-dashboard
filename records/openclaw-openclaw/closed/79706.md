Close: current main no longer uses the reported external `plugin-runtime-deps` startup path for bundled plugins, and the package/runtime docs, root dependencies, postinstall cleanup, and regression tests now cover the read-only install shape.

## What This Changes

Ensure bundled plugins can resolve `openclaw/plugin-sdk/*` when OpenClaw is installed globally under a read-only, root-owned package directory.

## Best Possible Solution

Keep the current package-owned bundled plugin dependency model and ask affected users on v2026.4.22-era installs to upgrade to a build from current main or newer release once shipped.

## Reproduction Assessment

No high-confidence current-main failure path remains. The report gives credible v2026.4.22 steps, but current source no longer creates the external bundled `plugin-runtime-deps` load path that caused the missing `openclaw` package resolution.

## Solution Assessment

Yes. Eliminating the external bundled runtime-deps path and keeping packaged bundled dependencies under the OpenClaw package is narrower and more maintainable than adding a per-user symlink repair at startup.
