## Summary

Keep open. Current main still falls back to the legacy context engine without a transcript-size preflight or guard policy, and #76950 is an open closing PR for the same work, so this issue should remain open until that PR or an equivalent implementation lands.

## What This Changes

Request adds a startup preflight guard that warns, archives, or blocks oversized session transcripts when a configured context-engine plugin is missing or falls back to legacy.

## Best Possible Solution

Land a reviewed core implementation like #76950 after choosing conservative defaults and aligning config schema, docs, resolver fallback behavior, and session-recovery expectations.

## Reproduction Assessment

Yes, at source level: configure a non-legacy `plugins.slots.contextEngine` that is missing, disabled, or fails registration, and `resolveContextEngine` falls back to legacy without inspecting transcript sizes. I did not run a live gateway boot because this review was read-only.

## Solution Assessment

Unclear as proposed: the guard belongs in core, but `archive` as a default and the low threshold need maintainer product judgment. The narrow path is to review #76950 or an equivalent warn-first/opt-in archival design rather than spawn a duplicate repair PR.
