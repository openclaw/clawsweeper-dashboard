## Summary

Keep open: the linked bug is valid, but this PR is not mergeable because the cache-hit path trusts only package.json.version, bypasses the existing plugin validation and dependency scan, and the supplied proof is still unit-test output from a simulated install root.

## What This Changes

The PR adds an install-mode managed npm fast path that returns success when an existing plugin package version is equal to or newer than the requested npm version, plus focused npm-spec tests.

## Best Possible Solution

Keep the version-satisfied skip direction, but route cache hits through the existing installed-package validation, managed-root dependency scan, peer-link repair, and validated metadata return path.

## Reproduction Assessment

Yes, source-level: current main rejects an existing managed npm target in install mode before any installed-version comparison, and the existing duplicate-install test asserts that behavior. I did not run tests because this review must keep the checkout read-only.

## Solution Assessment

No. The requested direction is narrow, but this implementation returns before the validation/scanning path that currently establishes plugin identity, host compatibility, dependency safety, peer links, and result metadata.
