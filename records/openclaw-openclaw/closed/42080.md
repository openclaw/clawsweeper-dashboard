## Summary

Current main, and the shipped `v2026.4.22` release, already resolve an unqualified outbound message to a selected configured channel before contact-name lookup. That removes the reported fan-out to absent providers like WhatsApp.

## Evidence

- **Channel selection happens before target resolution:** `runMessageAction()` resolves the channel first and only then calls `resolveActionTarget()` with that single channel, so contact lookup is no longer attempted across unrelated providers.

## Close Comment

Codex review: closing as implemented on main. Current outbound message handling selects the channel before contact-name resolution (`src/infra/outbound/message-action-runner.ts:848`, `src/infra/outbound/message-action-runner.ts:911`), and the selector only considers configured channels via `listConfiguredMessageChannels()` / `resolveMessageChannelSelection()` (`src/infra/outbound/channel-selection.ts:131`, `src/infra/outbound/channel-selection.ts:146`). That means an unqualified `message <name>` no longer fans out to absent providers like WhatsApp; it now uses the explicit/tool-context/single-configured channel or fails cleanly when no unique configured channel exists. The same implementation is already shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), with regression coverage in `src/infra/outbound/channel-selection.test.ts:65` and `src/infra/outbound/channel-selection.test.ts:140`.
