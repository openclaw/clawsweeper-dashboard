## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate Python orchestration behind explicit exec policy:** `src/agents/pi-tools.ts:552`
  - body: This adds python_orchestrator whenever `tools.pythonOrchestrator.enabled` is not explicitly false, while the tool itself runs model-supplied Python via `spawn("python3", ...)`. That creates a new local code-execution path by default instead of routing through the existing exec approval/tool-policy model, so users who have not opted into this capability can still get host Python execution.
  - confidence: 0.92
- **[P1] Do not swallow uncaught exceptions:** `src/entry.ts:12-18`
  - body: Registering an `uncaughtException` handler that only logs suppresses Node's default fatal exit. After an uncaught exception the gateway can keep running in an undefined state, so this handler should either use `uncaughtExceptionMonitor` or perform cleanup and terminate with a failing exit code.
  - confidence: 0.86
- **[P2] Keep the UI browser test project enabled:** `ui/vitest.config.ts:20`
  - body: Removing the Playwright browser project means `ui/src/**/*.browser.test.ts` is no longer selected by any UI Vitest project. That silently drops browser coverage from the UI gate even though the dependency remains installed.
  - confidence: 0.9
- **[P2] Preserve HTTP status on postJson failures:** `src/memory/post-json.ts:47`
  - body: The new inner retry catches errors that intentionally carry `status` and then throws a fresh `failed after 3 attempts` error without that status. Callers such as `postJsonWithRetry` use `err.status` to retry only 429/5xx responses, so this breaks the existing retry contract and can change retry behavior for client errors.
  - confidence: 0.82
- **[P2] Keep config entries passthrough before logging unknown keys:** `src/config/zod-schema.ts:177-182`
  - body: Replacing `catchall(z.unknown())` with a plain object transform causes Zod to strip unknown keys before the transform runs, so the warning loop cannot see those keys and provider-specific MCP/talk fields are lost. Keep `catchall`/`passthrough` semantics and add logging in a way that preserves the current config contract.
  - confidence: 0.84
