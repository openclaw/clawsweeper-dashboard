## Summary

Keep open: current main and v2026.6.6 still mix OS-mismatched skills into ordinary doctor missing-requirement and doctor --fix targets, while https://github.com/openclaw/openclaw/pull/89260 is an open closing PR for this issue.

## What This Changes

The issue asks for openclaw doctor to separate OS-incompatible skills from genuinely missing skill requirements and avoid disabling them with doctor --fix.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/89260, or an equivalent shared-evaluator-based fix, so doctor rolls up platform-only mismatches separately and keeps genuine missing requirements actionable.

## Reproduction Assessment

Yes. Source inspection shows unsupported OS requirements populate missing.os, make the skill ineligible, and current doctor code counts and repairs those entries as ordinary unavailable skills; I did not run the Windows bundle scenario live.

## Solution Assessment

Yes. The linked shared-evaluator-based PR is the narrow maintainable path because it uses missing.os, preserves remote-platform eligibility, and updates both the doctor summary and doctor --fix target.
