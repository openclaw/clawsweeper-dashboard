## Label Justifications

## Summary

Keep open. Current `main` has related hook work for some dispatcher paths, but the WebChat/Gateway final-broadcast paths still bypass `message_sending`; the open related reports are broader umbrella/product-policy items rather than a merged or canonical replacement for this narrow source-proven gap.

## What This Changes

The issue requests that the `message_sending` plugin hook run for WebChat/WebSocket final assistant delivery, including rewrite and cancellation before Gateway chat broadcasts.

## Best Possible Solution

Approve and implement one Gateway WebChat pre-delivery hook boundary that covers both `chat.send` dispatcher finals and terminal lifecycle finals, with regression tests for rewrite, cancel, sessionKey context, and streaming-final behavior.

## Reproduction Assessment

Yes at source level: current `main` shows WebChat final delivery can broadcast without `runMessageSending`, while durable outbound delivery does run that hook. I did not run a live WebSocket repro because this was a read-only review.

## Solution Assessment

Unclear as proposed: adding a call only inside `emitChatFinal` is too narrow for current `main` because `broadcastChatFinal` is another WebChat final path. The maintainable fix should cover the shared Gateway pre-delivery boundary instead of only one final broadcaster.
