## Summary

Keep this PR open, but request changes before merge: current main still has the eager `sessions.list` reload, and the debounce direction is narrow, but the submitted patch leaves lifecycle cleanup, test coverage, and changelog handling incomplete.

## What This Changes

The PR adds Control UI timer state to debounce full `sessions.list` reloads for five seconds after `sessions.changed` push events while keeping the incremental UI update immediate.

## Best Possible Solution

Request changes on this PR: keep incremental session updates immediate, debounce only the full reconciliation reload, clear pending timers during teardown, add fake-timer coverage for delayed and coalesced reloads, initialize test fixtures, and add the active-version changelog entry if this remains user-facing.

## Reproduction Assessment

Yes. Current main has a source-level reproduction: dispatching any `sessions.changed` event reaches `applySessionsChangedEvent` and then immediately calls `loadSessions`, and the current unit test asserts that synchronous reload behavior.

## Solution Assessment

No, not as submitted. The debounce is a narrow maintainable direction for the fanout problem, but the patch needs lifecycle cleanup, updated tests, and changelog handling before it is the best fix.
