## Summary

Close as ClawHub/external-plugin scope: OpenClaw core already has the generic public-artifact contract and fallback guidance, while Honcho-specific export or sync belongs in the external Honcho plugin/package.

## What This Changes

The issue asks whether OpenClaw has a supported workaround or manual process for feeding Honcho memory artifacts into Wiki Bridge mode when Honcho does not export `publicArtifacts`.

## Best Possible Solution

Keep OpenClaw core on the generic `publicArtifacts` contract; publish a Honcho plugin update or ClawHub package that exports Honcho artifacts, while users use isolated/native wiki workflows until then.

## Reproduction Assessment

Yes from source inspection: bridge imports return zero when the active memory plugin has no `publicArtifacts.listArtifacts` provider. I did not run a live Honcho deployment, and this is not a core bug reproduction.

## Solution Assessment

Yes: the narrow maintainable solution is for the external Honcho memory plugin to implement the existing memory capability API, not for OpenClaw core to add a Honcho-specific bridge path.
