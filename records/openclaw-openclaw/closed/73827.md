## What This Changes

The issue asks the Slack plugin to wake mention-gated agents when a Slack user-group `<!subteam^...>` mention targets a group containing those bot users.

## Best Possible Solution

Keep the API-backed Slack user-group mention implementation on main and let the next release ship it.

## Reproduction Assessment

Yes. The current unit test constructs the reported `requireMention` channel with `<!subteam^S0AGENTS|agents>`, stubs Slack membership to include the bot, and verifies `WasMentioned=true`; the non-member case verifies the old silent-drop behavior remains for unrelated groups.

## Solution Assessment

Yes. The current implementation follows the maintainer-preferred API-backed membership check instead of a config-only allowlist, so OpenClaw wakes only bots Slack reports as actual members of the mentioned user group.
