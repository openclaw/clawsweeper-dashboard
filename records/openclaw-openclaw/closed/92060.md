## Summary

The reported cross-session reply is the shipped, bounded `sessions_send` conversation lifecycle rather than WebChat automatically routing arbitrary output; the remaining request for one-way delivery is already tracked by the canonical dispatch-mode issue.

## What This Changes

Prevent `sessions_send` replies from being re-injected across agent sessions when a one-way handoff is intended.

## Best Possible Solution

Keep the existing bounded conversation mode and track any explicit per-send, one-way handoff semantics in https://github.com/openclaw/openclaw/issues/44309.

## Reproduction Assessment

## Solution Assessment

No; WebChat provenance filtering or another per-agent setting would place policy at the wrong boundary, while the canonical dispatch-mode issue proposes explicit per-send semantics.
