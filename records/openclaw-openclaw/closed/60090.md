## Summary

Close PR #60090 as ClawHub/plugin-scope work rather than a new OpenClaw core compaction behavior. The idea is useful, but current main already exposes context-engine and compaction-provider plugin seams for alternate summarization, scratchpad stripping, micro-compaction, and transcript-aware context strategies. The submitted PR also had unresolved review/validation concerns around brittle tag parsing and check failures, so it is not the right core change to carry forward.

## Best Possible Solution

Close this PR and keep the shipped core compaction path stable. Anyone wanting hidden scratchpad parsing, tag-envelope extraction, or micro-compaction should publish it as a ClawHub context-engine or compaction-provider plugin; if that plugin hits a concrete missing API, open a focused plugin API issue instead of carrying this core PR forward.
