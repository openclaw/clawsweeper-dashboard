## Summary

Current `main` already implements host-backed Ollama onboarding: the wizard prompts for a custom Ollama base URL, discovers models from that configured host, and sends model pulls to that same host via `/api/pull`. The issue's requested behavior is therefore implemented on `main`.

## Evidence
