## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Handle system alias paths in the symlink guard:** `src/infra/exec-approvals.ts:245-247`
  - body: The new symlink check compares the literal target with a realpath-derived target and only normalizes selected Darwin prefixes. On Linux, valid state roots such as `/var/run/openclaw` commonly pass through `/var/run -> /run`, so `saveExecApprovals()` would throw before writing approvals. Normalize known system aliases or keep the guard scoped to a trusted root.
  - confidence: 0.82
