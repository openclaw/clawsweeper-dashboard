## Summary

Current main no longer sends the default OpenAI-compatible `tool_choice: "auto"` unless a non-empty tools array is present, and the shipped changelog records this vLLM/OpenAI-compatible provider fix in 2026.4.10. The old `tools.allow: []` reproduction also no longer matches current tool-policy semantics, where an empty allowlist is allow-all with deny exceptions rather than a no-tools policy.
