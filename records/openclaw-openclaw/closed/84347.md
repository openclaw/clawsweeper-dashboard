## Summary

Keep this PR open: the docs warning is useful and matches the iMessage runtime contract, but the added `socat` example is incomplete for a `cliPath` wrapper because it does not forward the `rpc` and optional `--db` arguments OpenClaw passes.

## What This Changes

Adds a Warning block to the iMessage Remote Mac over SSH docs explaining that `cliPath` wrappers must stream JSON-RPC stdio incrementally and naming the timeout symptoms.

## Best Possible Solution

Keep the new warning, but remove or rewrite the `socat` example so every shown wrapper forwards OpenClaw's arguments and keeps JSON-RPC stdout clean.

## Reproduction Assessment

Yes at source level. Current main lacks the warning, and `extensions/imessage/src/client.ts` shows the channel uses long-lived newline-framed JSON-RPC over the configured `cliPath` stdio.

## Solution Assessment

Mostly yes. Documenting the streaming requirement in the Remote Mac setup section is the right fix, but the added `socat` example should be complete or removed before merge.
