## Summary

Current main already handles the reported WebChat case: assistant text containing remote Markdown image syntax is parsed into media segments, normalized into assistant image attachments, and rendered in WebChat as image elements. The implementation keeps the generic Markdown sanitizer's remote-image hardening, but the assistant-response path now displays safe HTTPS Markdown image URLs through the attachment renderer.

## Best Possible Solution

Keep the shipped safe attachment pipeline: parse assistant `![alt](https://...)` outputs into media attachments and render those in WebChat, while preserving the sanitizer hardening that blocks arbitrary remote Markdown/HTML image tags outside the assistant media path.
