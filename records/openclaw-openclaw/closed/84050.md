## Summary

Keep this issue open: current main and the latest shipped release still derive the remote-media fallback filename from raw `URL.pathname`, and the issue is already paired with an open fix PR that closes it. The issue should close only after https://github.com/openclaw/openclaw/pull/84052 lands or is deliberately replaced.

## What This Changes

Decode valid percent escapes in remote media URL fallback basenames when no `Content-Disposition` filename or caller `filePathHint` is available.

## Best Possible Solution

Land a narrow fix through the linked PR that decodes the URL-derived basename after basename extraction, preserves malformed escapes, and adds focused regression coverage for the shared media fetch path.
