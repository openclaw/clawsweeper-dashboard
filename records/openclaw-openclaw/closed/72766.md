## Best Possible Solution

Keep the shipped writer-queued `updateSessionStore` implementation on main and close this now-redundant branch instead of merging a conflicting duplicate.

## Solution Assessment

Yes. The shipped main implementation uses the existing session-store writer/update helper for both targeted heartbeat writes, which is the narrow maintainable fix for this race without adding a new concurrency mechanism.
