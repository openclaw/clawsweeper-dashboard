## Summary

Close as duplicate. The remaining work is already tracked by open canonical issue #45608, which specifically covers adding a silent, agentic pre-reset memory/final-turn path before `/new`, `/reset`, daily reset, and idle reset archive the outgoing session; this issue adds useful framing but does not identify a distinct code path or separate fix.

## What This Changes

The issue requests an opt-in session-end agent turn with full history and tool access so the agent can run cleanup before session reset or rollover destroys the outgoing session.

## Best Possible Solution

Keep #45608 as the canonical maintainer design thread for reset-time agentic cleanup, fold this issue’s `sessions.onEnd.agentTurn` framing into that discussion, and keep #58818 as the complementary raw-context carry-forward tracker rather than splitting the same teardown behavior across multiple open issues.

## Reproduction Assessment

Not applicable as a defect reproduction; this is a feature request. The absence is high-confidence from current-main config/schema checks plus reset and rollover paths showing void lifecycle hooks and no reset-time agent-runner call.

## Solution Assessment

Yes for triage cleanup: closing this duplicate is the best way to keep design, security, and acceptance criteria concentrated in #45608. The eventual implementation shape is still unclear and should be decided there, including whether the public config should be `session.reset.memoryFlush`, `sessions.onEnd.agentTurn`, or another bounded policy.
