## Summary

Current `main` already provides Control UI voice input: a microphone dictation button backed by browser speech recognition, plus browser Realtime Talk with mic/WebRTC and Gateway-minted realtime sessions. That covers the requested user-facing feature well enough even though the implementation differs from the proposed Whisper/`/api/voice/transcribe` design.

## Risks / Open Questions

- The shipped implementation is not the exact architecture proposed in the issue: dictation uses browser-native speech recognition, and the live conversation path uses the configured realtime voice provider rather than a generic `/api/voice/transcribe` Whisper endpoint.
- Browser dictation availability depends on browser `SpeechRecognition` support; unsupported browsers still rely on the realtime Talk path or typed input.
