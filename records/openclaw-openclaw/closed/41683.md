## Summary

Current `main` now makes the behavior explicit and consistent: Anthropic 4.6 only gets the 1M beta header when `params.context1m: true` is set, OAuth/subscription tokens (`sk-ant-oat-*`) explicitly do not send that header, and provider-aware context/status lookup now reports the configured long-context window instead of the old stale fallback.

## Evidence

- **1M beta is opt-in for Anthropic 4.6:** `resolveAnthropicBetas()` adds `context-1m-2025-08-07` only when `extraParams?.context1m === true`; there is no auto-apply path for Opus 4.6.
  - file: [extensions/anthropic/stream-wrappers.ts:98](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/extensions/anthropic/stream-wrappers.ts#L98)
  - command: `nl -ba extensions/anthropic/stream-wrappers.ts | sed -n '82,130p'`
- **OAuth/subscription auth strips the 1M beta header:** `createAnthropicBetaHeadersWrapper()` detects `sk-ant-oat*` auth, removes the 1M beta, and logs that Anthropic rejects `context-1m` with non-API-key auth.
  - file: [extensions/anthropic/stream-wrappers.ts:115](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/extensions/anthropic/stream-wrappers.ts#L115)
  - command: `nl -ba extensions/anthropic/stream-wrappers.ts | sed -n '82,130p'`
- **Regression tests cover both cases:** Tests assert that OAuth token auth does not include `context-1m-2025-08-07`, while regular API-key auth does include it.
  - file: [extensions/anthropic/stream-wrappers.test.ts:87](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/extensions/anthropic/stream-wrappers.test.ts#L87)
  - command: `nl -ba extensions/anthropic/stream-wrappers.test.ts | sed -n '78,110p'`
- **Docs now answer the question directly:** Provider docs show `anthropic/claude-opus-4-6` requires `params.context1m: true`, warn that legacy/OAuth `sk-ant-oat-*` auth falls back to the standard window, and separately note that Opus 4.7 is 1M by default.
  - file: [docs/providers/anthropic.md:254](https://github.com/openclaw/openclaw/blob/350d3221800a18283915cd2f3b612ae04ddd490c/docs/providers/anthropic.md#L254)
  - command: `nl -ba docs/providers/anthropic.md | sed -n '248,276p'`
- **Status/context lookup no longer relies on the stale fallback:** `resolveContextTokensForModel()` returns 1M when Anthropic `context1m` is enabled and also has explicit provider-aware configured-window lookup before fallback; the test suite verifies 1M only when `context1m` is enabled and 200k otherwise.
