## Summary

Current `main` already strips Telegram `group:` namespace prefixes before parsing delivery targets, and the announce/session-target path resolves Telegram group session keys to bare numeric chat IDs. The released `v2026.4.22` tag also contains the same code and regression coverage.

## Evidence

- **Telegram target parsing strips legacy `telegram:group:` prefixes:** `stripTelegramInternalPrefixes()` removes `telegram:`/`tg:` and then removes `group:` when it is part of the legacy internal Telegram target form, so `telegram:group:-100123` normalizes to `-100123`.
  - file: [extensions/telegram/src/targets.ts:10](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/extensions/telegram/src/targets.ts#L10)
  - command: `nl -ba extensions/telegram/src/targets.ts | sed -n '1,220p'`
- **Telegram send path parses normalized targets before numeric chat-id validation:** `sendMessageTelegram()` calls `parseTelegramTarget(to)` and then resolves the chat id from `target.chatId`; `resolveChatId()` first runs `normalizeTelegramChatId()` before throwing `Telegram recipient must be a numeric chat ID`. That means prefixed Telegram group targets no longer fail on the raw `group:` form.
  - file: [extensions/telegram/src/send.ts:617](https://github.com/openclaw/openclaw/blob/7d9172d5e093435d37ed3e00e476a4eaf84a252b/extensions/telegram/src/send.ts#L617)
