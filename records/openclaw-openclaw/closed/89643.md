The branch adds plugin session extension preservation across session store writes, metadata merges, and first-turn skill snapshot writes, plus explicit cleanup/unset drop controls and a deadcode allowlist update.

## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection gives a high-confidence repro: preserve one namespace from the current store while a stale writer carries a different namespace for the same plugin, and the shallow plugin-id spread replaces the previous inner namespace record.

## Solution Assessment

No, not yet. The owner boundary is appropriate, but the best fix is a nested namespace merge across all preservation helpers rather than the current outer-level spread.
