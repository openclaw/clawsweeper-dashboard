- `P1`: The PR is for a live Talk-mode agent workflow where the spoken answer can conflict with the visible final Control UI answer.

## Summary

Keep open: the PR targets a valid Talk/Control UI divergence, but the patch still has a race-prone 250 ms no-text fallback before the Gateway source-reply final, and the supplied proof is jsdom/unit-test output rather than real Talk behavior.

## What This Changes

The PR changes Talk consult result selection so an empty final chat event waits briefly for a later canonical source-reply final text, with a regression test covering delivery-mirror text.

## Best Possible Solution

Resolve Talk consults from the same canonical source-reply final that Control UI displays, with the no-text fallback firing only after Gateway has made it clear no canonical reply will arrive.

## Reproduction Assessment

Yes. The linked issue has live repro evidence, and current main source shows Talk resolves the first final chat event while Gateway can emit the displayed delivery-mirror source reply separately afterward.

## Solution Assessment

No, not yet. Preferring a later text final is the right direction, but a fixed 250 ms fallback is not a reliable boundary for Gateway's async source-reply mirror path.
