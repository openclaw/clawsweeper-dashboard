## What This Changes

The PR changes the Codex app-server turn/start path so OpenClaw workspace runtime context is no longer prepended to user input and is instead appended to collaboration-mode developer instructions, with a MEMORY.md test updated for that routing.

## Best Possible Solution

Keep the clean `turn/start.input` goal, but route workspace/MEMORY fallback through an authority-safe transient/reference channel and keep collaboration developer instructions limited to trusted instruction content.

## Reproduction Assessment

Yes at source level: current main builds `openClawPromptContext`, decorates `codexTurnPromptText`, and passes it to `buildUserInput`, while the linked issue supplies structured logs showing that marker accumulating as user history. ## Solution Assessment

No: removing the wrapper from `turn/start.input` is the right goal, but moving fallback workspace/MEMORY reference content into `developer_instructions` is not the best authority boundary. The safer fix is a transient/reference transport such as Codex untrusted additional context, plus tests that prove no history accumulation.
