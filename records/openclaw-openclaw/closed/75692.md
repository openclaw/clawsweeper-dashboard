## Summary

Close as superseded by open PR #57276. Current main still does not provide the requested automatic system-level systemd restart or `--restart-via` override, but the same remaining system-scoped systemd lifecycle work is already tracked by the open PR and should be reconciled there rather than split into a second tracker.

## What This Changes

The issue requests that `openclaw update` detect and support system-level systemd gateway units, including explicit restart-mode override and loud failure when a user-scope restart would miss the active system unit.

## Best Possible Solution

Use #57276 as the canonical implementation thread: decide the product boundary for system-scoped units, then revise it so update detection finds active system units before the user-service gate, restarts only when privilege is explicit or already available, preserves user-unit behavior, and documents any override flag or manual guidance.

## Reproduction Assessment

Yes. A system-only unit under `/etc/systemd/system` with no matching user unit follows the current user-service state path, so package update restart either skips the service path or reaches only guidance rather than restarting the active system unit.

## Solution Assessment

Unclear for the exact proposal. Detecting system-scoped units and avoiding silent success is the right direction, but automatic `sudo systemctl restart` and a new `--restart-via` flag need to be settled in the canonical systemd-scope PR so privilege behavior stays explicit.
