## Summary

Keep open: current main still renders bare `/model` `Available:` values while `sessions.patch` validates provider-aware model refs; open PR #50141 remains the overlapping implementation candidate.

## What This Changes

The issue asks for `/model` `Available:` entries to be canonical, unambiguous values that can be pasted back into `/model <value>`.

## Best Possible Solution

Land or replace the overlapping Control UI fix so `/model` status prints provider-qualified values that `sessions.patch` accepts, with regression coverage for copying an `Available:` value back into `/model`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: no-argument `/model` renders bare catalog IDs, while `/model <value>` submits through `sessions.patch` and provider-aware allowlist resolution.

## Solution Assessment

Yes. The narrow maintainable fix is to use the existing Control UI model-ref qualification helper in the `/model` display path while preserving backend bare-name compatibility.
