## Summary

Current main no longer waits for `openclaw cron run` to finish the job under the old 30s CLI RPC default. Manual runs are enqueued asynchronously, the CLI treats queued runs as success, and the docs/tests/changelog all describe that behavior.

## Evidence

- **CLI no longer uses the old default wait behavior:** `cron run` checks whether the global `--timeout` option is still at its default and, in that case, raises it to `600000` ms before calling the gateway.
  - file: [src/cli/cron-cli/register.cron-simple.ts:165](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cli/cron-cli/register.cron-simple.ts#L165)

  - file: [src/cron/service/ops.ts:731](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/cron/service/ops.ts#L731)
