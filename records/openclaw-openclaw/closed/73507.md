## Summary

Current main already keeps the two named core modules off the deprecated plugin runtime `config.loadConfig()` API. The warning string is emitted only by the plugin-runtime compatibility shim when a caller invokes deprecated `api.runtime.config.loadConfig()`; docs and guardrails enforce that bundled/repo code does not use it, and the relevant guard scripts pass.

## Best Possible Solution

Keep the current implementation: status/audit should continue using boundary-appropriate best-effort reads or file snapshots, while any remaining warning should be traced to the specific plugin/runtime caller invoking deprecated `api.runtime.config.loadConfig()`, not to these core paths.
