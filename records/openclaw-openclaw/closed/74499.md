## Summary

Keep this PR open. The linked Ollama tool-dispatch bug is plausible and current main still lacks Ollama-local name normalization, but the current branch has a blocking correctness bug: the revised lookahead regex preserves the separator, so `functions.exec` becomes `.exec` instead of `exec`.

## What This Changes

The PR adds Ollama-local normalization for `functions.*`/`tools.*` tool-call names in stream conversion and assistant-message reconstruction, plus focused regression tests for prefixed and non-prefixed names.

## Best Possible Solution

Amend the Ollama-local normalizer to consume the prefix separator while preserving names such as `functionshell`, keep the new adapter-boundary tests, and then run the focused Ollama and runner normalization tests before deciding whether this PR fully resolves #74487.

- **[P2] Consume the prefix separator when normalizing tool names:** `extensions/ollama/src/stream.ts:598-599`
  - body: The lookahead verifies that `functions` or `tools` is followed by a separator, but the separator is not removed. As written, `functions.exec` normalizes to `.exec` and `tools/exec` normalizes to `/exec`, so the added tests expecting `exec` fail and dispatch still misses canonical tool names. Consume the delimiter with the prefix or strip it immediately after matching.
