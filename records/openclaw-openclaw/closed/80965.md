## Summary

This PR should stay open, but it still needs a narrow docs repair before merge. The resolver/test change matches the current-main bug and the contributor supplied before/after live output, but several user-facing docs would still say populated `groups` always acts as an allowlist even after explicit `groupPolicy: "open"` stops doing that.

## What This Changes

The PR changes the shared group-policy resolver so explicit `groupPolicy: "open"` treats configured `groups` as per-group overrides, adds regression coverage, and updates channel/security docs plus the changelog.

## Best Possible Solution

Land the resolver/test fix after qualifying every remaining groups-as-allowlist doc statement to apply only when `groupPolicy` is omitted or explicitly `"allowlist"`.

## Reproduction Assessment

Yes. Source inspection on current main shows explicit `groupPolicy: "open"` still enters allowlist mode whenever `groups` is populated, and the PR body includes matching before/after resolver output.

## Solution Assessment

Mostly yes. The resolver condition is the narrow maintainable fix and preserves omitted-policy compatibility, but the PR is not complete until the remaining user-facing allowlist docs are qualified.
