## Summary

Current `main` already removes managed memory dreaming from the global heartbeat path by scheduling it as an isolated `agentTurn` with `lightContext: true`, plus migration/repair for legacy `systemEvent` jobs. That addresses the issueâ€™s underlying need for lightweight dreaming bootstrap without changing global heartbeat behavior.

## Evidence

- **Managed dreaming now runs as isolated lightweight agent turn:** `buildManagedDreamingCronJob` creates the managed job with `sessionTarget: "isolated"`, payload `{ kind: "agentTurn", message: DREAMING_SYSTEM_EVENT_TEXT, lightContext: true }`, and `delivery: { mode: "none" }`, so dreaming no longer depends on main-session heartbeat bootstrap behavior.

## Risks / Open Questions

- Current `main` solves the scope problem by always using lightweight isolated managed dreaming, not by exposing a configurable `dreaming.lightContext` toggle. If someone specifically needs full bootstrap for managed dreaming, that is a different request.
- The latest release tag `v2026.4.23` still has the old heartbeat-backed shape, so the fix appears unreleased as of the repository state reviewed here.
