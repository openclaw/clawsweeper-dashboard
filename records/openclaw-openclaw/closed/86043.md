## What This Changes

The issue asks OpenClaw to prevent DeepSeek `reasoning_content` from being serialized as Gemini `thought_signature` when a session switches providers.

## Best Possible Solution

## Reproduction Assessment

No current-main failing reproduction is available from this read-only review. The reporter's logs explain the old-version failure, while current source and tests show the Gemini payload path no longer serializes foreign reasoning as `thoughtSignature`.

Yes. The maintainable fix boundary is the provider replay/transport layer, and current main already gates Gemini signatures by provider/api/model route instead of trusting cross-provider thinking metadata.
