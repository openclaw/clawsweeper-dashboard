## What This Changes

The issue reports that memory-wiki `wiki_apply` and `wiki_lint` fail with a YAML parse error when an unrelated vault page has malformed frontmatter containing markdown bold text and an inline colon.

## Best Possible Solution

Review and land one focused memory-wiki fix that catches malformed frontmatter at the vault-scan summary boundary, reports it through lint, and keeps direct edit-target parsing strict to avoid metadata loss.

## Reproduction Assessment

Yes. Source inspection on current main shows malformed frontmatter reaches `YAML.parse`, `toWikiPageSummary`, `readPageSummaries(errorMode: "stop")`, then `compileMemoryWikiVault`, which both `wiki_apply` and `wiki_lint` use.

## Solution Assessment

Yes. The best direction is a scan-boundary fix that degrades unrelated broken pages into lintable summaries while leaving direct edit-target parsing strict enough to avoid metadata loss.
