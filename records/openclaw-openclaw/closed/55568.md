## Summary

Current `main` already suppresses the Discord ACP duplicate-final fallback by treating visible block deliveries as satisfying the reply, with Discord-specific routing hooks and regression tests covering the reported behavior.

## Evidence

- **ACP final fallback now gated on visible delivery:** `finalizeAcpTurnOutput` only sends the extra final text when ACP block output was not already visibly delivered or that visible delivery failed, which directly addresses the duplicate-message shape from the report.
