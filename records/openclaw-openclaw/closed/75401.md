## Summary

Keep open for merge. Current main still has the key-existence guard that lets `properties: undefined` or `properties: null` bypass empty-schema normalization, and this PR is a narrow fix with matching regression coverage and changelog entry.

## What This Changes

The PR updates OpenAI/MCP tool-parameter schema normalization to treat null or undefined top-level `properties` as missing, adds regression tests, and records the user-facing fix in the changelog.

## Best Possible Solution

Land this narrow shared-normalizer change after CI passes; no provider-specific workaround or core API redesign is needed.

## Reproduction Assessment

Yes. Passing `{ type: "object", properties: undefined }` or `{ type: "object", properties: null }` through current main's tool-parameter normalization follows the guarded path that skips injecting `properties: {}`, matching the reported OpenAI 400 failure mode.

## Solution Assessment

Yes. Fixing the shared schema normalizer is the narrowest maintainable solution because all relevant OpenAI transports already consume that normalized schema path.
