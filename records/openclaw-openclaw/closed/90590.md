## Summary

Current main and the latest release already provide persistent custom session labels that cover the requested custom-title workflow through the Gateway session API and Control UI; the remaining requested slash-command spelling is an optional convenience rather than missing core capability.

## What This Changes

The issue asks for a user-set custom session title that persists, appears in session lists/sidebar, and can be set through API or UI instead of relying only on derived first-message titles.

## Best Possible Solution

Keep the existing session-label implementation as the canonical custom title path; a future `/title` alias can be considered separately as small UI/CLI polish if maintainers want it.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. The current-main source check confirms the requested custom-title behavior is represented by persisted session labels.

## Solution Assessment

Yes for the central request: a durable `label` field plus `sessions.patch` and Control UI editing is the narrow maintainable implementation. A separate slash command would be convenience UI rather than the core fix.
