## Summary

Current `main` already derives and forwards Feishu `receive_id_type` during outbound sends, with regression coverage for `group:` and `dm:` targets. The fix was shipped in `v2026.3.2`; the reported `message_defaults.receive_id_type` knob is not part of the current supported Feishu config surface.

## Close Comment

Closing as `implemented_on_main` based on a Codex review. Current `main` already preserves Feishu send routing hints in `extensions/feishu/src/send-target.ts:17` and forwards the resolved `receiveIdType` into outbound sends in `extensions/feishu/src/send.ts:454`, with regression coverage in `extensions/feishu/src/send-target.test.ts:35`. This behavior is recorded in the `2026.3.2` changelog section at `CHANGELOG.md:3368` and traces to commit `bcc3d9ba9e23755d9aad93ec1ea10b9c52c16f97` (`fix(feishu): preserve explicit target routing hints`). Current Feishu config also does not define `message_defaults.receive_id_type` (`extensions/feishu/src/config-schema.ts:155`), so the supported fix is automatic target-type resolution rather than that config override. If this still reproduces on a current build, please open a new issue with the exact outbound target shape and a fresh failing debug log from current `main`.
