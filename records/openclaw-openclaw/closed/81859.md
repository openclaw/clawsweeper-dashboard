## Summary

Keep open. Current main has useful LaunchAgent recovery and token-drift checks, but it still uses the LaunchAgent service-env wrapper path, and a focused search found no stale `ai.openclaw.update.*` submitted-job detection or cleanup; the related open self-update issue does not fully cover this stale-job plus wrapper/token failure mode.

## What This Changes

The issue asks OpenClaw to make macOS beta updates recover from stale launchd updater jobs, report them in gateway status/doctor, and handle LaunchAgent service-env wrapper/token drift safely.

## Best Possible Solution

Split this into macOS update resilience work: detect and clean/report stale submitted updater jobs, keep gateway status/doctor actionable, and make an explicit owner decision on service-env wrapper versus safe inline launchd environment handling with token preservation proof.
