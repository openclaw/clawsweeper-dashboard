## Summary

Current main still has the original Slack prompt mismatch. Slack object-form `capabilities: { interactiveReplies: true }` enables the Slack directive path and plugin hint, but the generic message-tool prompt still treats inline buttons as enabled only when runtime capabilities contain `inlineButtons` and can still tell Slack users to configure `slack.capabilities.inlineButtons`. The later streaming/final-update Block Kit loss appears addressed separately, so the remaining work is a narrow prompt-guidance fix.

## Best Possible Solution

Keep this issue open and land a narrow prompt fix. Prefer making the generic message-tool inline-buttons affordance channel-aware so Slack does not emit the Telegram-style `slack.capabilities.inlineButtons` disabled line; rely on Slack-specific `messageToolHints` or a typed prompt capability derived from Slack config for `interactiveReplies`. Add regression coverage for Slack object-form `{ interactiveReplies: true }` so the prompt does not contradict the Slack docs or plugin hint.
