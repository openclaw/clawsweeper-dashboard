## Summary

Current main already solves the reported Feishu/provider role-alternation problem by keeping inbound metadata in the current-turn context boundary and composing it with the actual prompt instead of emitting separate user-role metadata messages. The proof commit is present only on main and is not contained in the latest release tag checked here.

## What This Changes

The issue requests that Feishu DM conversation metadata and queued system-event context stop appearing as separate user-role messages ahead of the real user text.

## Best Possible Solution

Keep the generic current-main prompt-boundary implementation and let the next release carry it; reopen or file a new report only if a current-main Feishu payload still shows separate user-role metadata messages.

## Reproduction Assessment

No current-main failing reproduction remains from source inspection: Feishu inbound metadata now flows through `currentInboundContext` and is joined with the current prompt instead of becoming separate user-role turns. I did not run a live Feishu DM because this review is read-only and live credentials were not available.

## Solution Assessment
