## Summary

Current main already supersedes this PR with a different fix: heartbeat transcripts are intentionally append-only, and the replay pipeline now repairs malformed tool_use/tool_result history before provider calls. The same implementation is present in release v2026.4.22, so the prune-based PR is obsolete.
