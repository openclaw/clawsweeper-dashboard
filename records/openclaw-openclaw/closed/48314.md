## Summary

Current `main` already separates think effort from reasoning visibility in `/status`. The status formatter prints `Think` from the resolved think level and prints `Reasoning` separately, and the existing tests assert that behavior.

## Evidence

- **Status formatter uses think level, not reasoning visibility:** `/status` builds its options line with `Think: ${thinkLevel}` and a separate `Reasoning: ${reasoningLevel}` field when reasoning is enabled.
