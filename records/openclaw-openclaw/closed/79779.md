## Reproduction Assessment

Yes, at source level. Current main writes the staged manifest under os.tmpdir() and also passes recursive backup assets to tar, so TMPDIR under the state asset can produce two root manifest entries before verify rejects the archive.

## Solution Assessment

Yes. Moving only the manifest staging root when it overlaps a backup asset is narrower than loosening verification or filtering the manifest out, and the tar dependency source supports that filter-based dedupe is not reliable here.
