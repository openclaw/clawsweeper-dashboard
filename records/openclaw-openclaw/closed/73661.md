## Summary

Current main and the latest release already use a file-backed device identity for `openclaw tui`. The gateway still rejects identity-less UI clients by design, but the current TUI sends identity material through `GatewayClient`; the reporter's stated workaround of reinstalling latest matches the shipped/current behavior.

## Best Possible Solution

Keep the current shipped implementation: TUI should continue using the Node GatewayClient with file-backed device identity, and users seeing this from a stale version-mismatch install should update/reinstall to the current package. If it reproduces on the latest TUI and gateway together, open a new focused bug with `openclaw --version`, binary paths, gateway logs, and auth config shape.
