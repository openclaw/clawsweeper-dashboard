## Summary

Keep open: current main still does not classify `attempt_dispatch` as cron execution-started, and https://github.com/openclaw/openclaw/pull/81871 is an open closing PR for the focused fix.

## What This Changes

The issue reports isolated cron jobs reaching `attempt_dispatch` being killed by the 60s pre-execution watchdog instead of running until the configured job timeout.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81871 after normal gates pass, then close this issue as fixed by the merge.

## Reproduction Assessment

Yes. The source path is clear: an isolated cron run that emits `runner_entered` and then `attempt_dispatch` without a later recognized milestone leaves the pre-execution watchdog armed, and user comments provide live beta.4, beta.6, and beta.8 logs with that phase.

## Solution Assessment

Yes. Treating the existing `attempt_dispatch` phase as execution-started in the private cron classifier, with the focused regression from the open PR, is the narrowest maintainable fix.
