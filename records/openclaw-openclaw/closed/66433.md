## Summary

Keep open. Current main does not implement the requested upgrade-preserve behavior for `node-llama-cpp`; it codifies the opposite policy. Although the issue discussion says “Fixed, closing,” the linked PR #66586 proposed restoring `node-llama-cpp` as an optional dependency but is closed and unmerged, while current release metadata, runtime guidance, docs, and tests all treat the runtime as operator-installed.

## Best Possible Solution

Keep this open for maintainer judgment. If OpenClaw wants upgrade-preserved local embeddings, the package metadata, release check, tests, and npm global-upgrade validation need to change together. If the current operator-installed policy is intentional, close manually with a clear by-design/migration comment and consider improving `memory status`/docs to say that users upgrading from older bundled installs must run `npm i -g node-llama-cpp@3.18.1` once next to OpenClaw.
