## Summary

Keep open: this is a plausible loop-detection bug area, but the PR is not merge-ready because the patch can still leave null-param calls hashed inconsistently on the no-hook path, and the new test references an unimported constant.

## What This Changes

The PR normalizes non-object agent tool params before loop detection/recording and adds null-param no-progress loop coverage.

## Best Possible Solution

Use one effective params value for loop detection, call recording, hook/policy processing, execution, and outcome tracking, then cover the exact null-param path that fails on current main.

## Reproduction Assessment

No high-confidence runnable repro was established in this read-only review. Source inspection supports a plausible null-vs-{} hash mismatch in hook/policy-adjusted paths, but the PR's no-hook scenario is not yet proven against current main.

## Solution Assessment

No. The direction is maintainable, but this patch records normalized params without aligning every returned outcome path and its added test currently cannot compile.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep the recorded params aligned with the returned params:** `src/agents/pi-tools.before-tool-call.ts:435`
  - body: The patch records loop history with `normalizedParams`, but the no-policy/no-hook fast path still returns the original `params`. For null-param calls, that makes `recordToolCall()` store the `{}` hash while `recordLoopOutcome()` later hashes `null`, so outcomes will not attach to the recorded calls and the global breaker still cannot count no-progress repeats. Return the same normalized value from that path, or delay recording until the effective params are known.
  - confidence: 0.89
- **[P1] Import the global breaker threshold in the new test:** `src/agents/pi-tools.before-tool-call.e2e.test.ts:269`
  - body: The added test references `GLOBAL_CIRCUIT_BREAKER_THRESHOLD`, but this file still imports only `CRITICAL_THRESHOLD` from `tool-loop-detection.js`. The PR head will fail to compile until the new constant is imported.
  - confidence: 0.95
