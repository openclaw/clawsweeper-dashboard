## Summary

Current `main` already makes slug generation timeout configurable by reusing `agents.defaults.timeoutSeconds`. The slug helper still defaults to 15s when unset, but it now resolves and passes the configured agent timeout through to the embedded run, with tests covering both behaviors.

## Evidence

- **Slug generator now honors configured agent timeout:** `resolveSlugGeneratorTimeoutMs` reads `cfg.agents?.defaults?.timeoutSeconds`, falls back to `15_000` only when unset, and otherwise calls `resolveAgentTimeoutMs({ cfg })`. The resulting `timeoutMs` is passed into `runEmbeddedPiAgent` for `slug-gen` runs.
