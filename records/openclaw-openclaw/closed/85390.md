## Best Possible Solution

Keep the shipped shared proxy implementation and direct affected users on 2026.4.5 to upgrade, then ask for a fresh current-version report if Feishu still fails with the managed HTTP(S) proxy configuration.

## Reproduction Assessment

Yes at source level for the reported release: v2026.4.5 used a Feishu-local `HttpsProxyAgent` path for websocket proxy env handling. I did not run a live macOS Feishu/Lark proxy setup.

## Solution Assessment

Yes. The maintainable fix is the shipped shared ambient proxy helper plus the documented managed HTTP(S) proxy configuration, not another Feishu-only proxy implementation.
