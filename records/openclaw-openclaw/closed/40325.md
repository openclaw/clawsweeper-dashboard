## Summary

The request is a useful plugin-level orchestration idea, but current OpenClaw scope keeps optional multi-agent orchestration out of core and on the plugin/ClawHub path. Current main also closes the specific SDK gap mentioned in the issue by exposing `api.runtime.subagent` plus TaskFlow/webhook helpers to plugins.

- If maintainers want first-party supervised multi-agent UX later, this could be reopened as a narrower core product decision rather than a plugin-SDK gap.
