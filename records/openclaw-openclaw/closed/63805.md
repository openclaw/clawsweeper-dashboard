## Summary

Keep open. Current `main` still treats positive cron `payload.timeoutSeconds` for isolated `agentTurn` jobs as a hard wall-clock ceiling and aborts the run when that outer timer fires. `main` does include two mitigations the discussion identified: omitted `agentTurn` timeouts now fall back to a 60-minute safety ceiling, and `timeoutSeconds: 0` disables the outer timeout entirely. Those changes do not implement the progress-aware/overhead-tolerant behavior requested in #63805.

## Evidence

- **Outer cron timeout is still a hard abort race:** `executeJobCoreWithTimeout` wraps `executeJobCore(...)` in `Promise.race(...)` against `setTimeout(...)`; when the timer fires it aborts the run and rejects with `cron: job execution timed out`. There is no progress-sensitive extension or grace logic in this path.
