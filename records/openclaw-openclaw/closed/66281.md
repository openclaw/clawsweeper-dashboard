## Summary

Current main already covers the reported browser attach failures well enough: managed `openclaw` launches wait for CDP readiness before succeeding, and existing-session profiles now support explicit Chromium `userDataDir` routing with CLI/docs coverage for non-default Chrome/Edge/Brave profiles. The fix is on `main` via commit `30aa7e0d4d225650b58c2fba79e71bffa5627b08`, but it is not in a tagged release yet.

## Evidence
