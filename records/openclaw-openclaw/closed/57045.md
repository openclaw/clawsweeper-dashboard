## Summary

Current main already implements `openclaw cron add` as a direct `cron.add` RPC with top-level job fields, and the gateway normalization layer accepts both direct and wrapped job shapes. The same implementation is present in release `v2026.4.23`, so this report is obsolete on current OpenClaw.

## Evidence

- **CLI sends direct cron.add params:** `registerCronAddCommand` builds a top-level params object with `name`, `schedule`, `sessionTarget`, `wakeMode`, `payload`, and optional `delivery`, then calls `callGatewayFromCli("cron.add", opts, params)`.
  - file: [src/cli/cron-cli/register.cron-add.ts:219](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/cli/cron-cli/register.cron-add.ts#L219)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Gateway accepts direct add payloads:** The `cron.add` server method normalizes incoming params with `normalizeCronJobCreate(params, ...)` before validation, so the direct CLI shape is an accepted input path.
  - file: [src/gateway/server-methods/cron.ts:189](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/gateway/server-methods/cron.ts#L189)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Normalizer accepts wrapped or unwrapped job objects:** `unwrapJob` accepts `data` or `job` wrappers when present and otherwise uses the raw object, which means current code supports both the wrapped shape from the issue writeup and the direct top-level shape used by the CLI.
  - file: [src/cron/normalize.ts:309](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/cron/normalize.ts#L309)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Tests cover current cron add path:** CLI tests assert `cron add` sends top-level `delivery` and inferred `sessionTarget` values on the `cron.add` RPC, matching the current implementation rather than a nested `job` wrapper.
  - file: [src/cli/cron-cli.test.ts:291](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/cli/cron-cli.test.ts#L291)
