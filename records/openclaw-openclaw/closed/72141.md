## Summary

Current main already enforces isolated cron runs as fresh cron-scoped sessions and drops stale cron-row state that could carry prior transcript or routing context into the next isolated run.

## Best Possible Solution

Keep the current main implementation: isolated cron jobs should continue to use cron-scoped fresh sessions, sanitize old cron session rows, and leave persistent-context behavior to explicit `current` or `session:<id>` targets. If a similar leak is observed on current main, it should be filed as a new regression with logs from the fresh-session path and provider request diagnostics.
