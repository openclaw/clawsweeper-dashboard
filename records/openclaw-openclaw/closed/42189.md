## Summary

Current main already neutralizes the reported ACP env-leak path: ACP client spawn strips active skill/provider auth env keys before copying `process.env`, and the older ACPX queue-owner path described in the report is now explicitly legacy/ignored.

## Evidence

## Close Comment

Closing from a Codex review. The specific leakage path described here is already addressed on current `main`: ACP client spawn now strips active skill env keys and provider auth env vars before copying `process.env` (`src/acp/client.ts:128`, `src/acp/client-helpers.ts:157`), with regression coverage in `src/acp/client.test.ts:90`. Separately, the ACPX `queueOwnerTtlSeconds` setting is now explicitly documented as a legacy compatibility field for the older queue-owner path and is ignored (`extensions/acpx/openclaw.plugin.json:129`, `extensions/acpx/src/service.ts:63`). This behavior is also present in shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`), so this issue appears implemented on main.
