## Summary

Current main no longer takes the plugin-runtime path that would emit repeated `memory-lancedb: plugin registered` lines for `openclaw models list` or repeated config-schema reads; the relevant code now stays on manifest metadata and has a regression guard against re-activating the plugin registry.

## Evidence

- **Duplicate line comes from plugin runtime registration:** `memory-lancedb` emits the reported log from its registration path.
  - file: [extensions/memory-lancedb/index.ts:343](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/extensions/memory-lancedb/index.ts#L343)

  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **CLI defaults to not loading plugins:** The default CLI command policy is `loadPlugins: "never"`, so commands only preload plugins when explicitly opted in.
  - file: [src/cli/command-path-policy.ts:5](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/cli/command-path-policy.ts#L5)

  - file: [src/cli/command-catalog.ts:84](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/cli/command-catalog.ts#L84)

  - file: [src/config/runtime-schema.ts:11](https://github.com/openclaw/openclaw/blob/a52b50ef3c8bb05d4a18302aad6862c37278cc57/src/config/runtime-schema.ts#L11)
