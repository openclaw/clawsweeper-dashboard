## Summary

Keep open. Current main still routes Claude CLI `AskUserQuestion` through generic tool-call display, but lacks both pending picker state and a tool-result submission path back into the active Claude CLI turn.

## What This Changes

A claude-cli-backed Control UI session should render Claude Code `AskUserQuestion` choices and return the structured selection to the active assistant turn.

## Best Possible Solution

Define a small Claude CLI HITL/result contract, parse `AskUserQuestion` as pending input, render a Control UI picker, submit structured answers back to the active Claude CLI turn, and cover parser, UI, and live Claude CLI behavior.

## Reproduction Assessment

Yes by source inspection. A claude-cli `AskUserQuestion` tool event reaches generic tool rendering while the live-session stdin path can only send ordinary user messages, so the structured result cannot be returned on current main.

## Solution Assessment

No as a narrow UI-only rendering patch. The maintainable fix needs the picker UI plus a dependency-confirmed result-submission path across the Control UI, gateway/session, and Claude CLI live-stdio boundary.
