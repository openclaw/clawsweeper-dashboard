## Summary

Current main still does not satisfy #46602. The /status Voice line is present, but it only renders auto mode, provider id, text limit, and summary state. The TTS status snapshot does not carry model, voice, baseUrl, or a custom-backend/display name marker, even though the OpenAI speech provider and docs support custom TTS baseUrl/model/voice settings. The related PR #47353 targeted the right behavior but was closed unmerged, and the maintainer comment on #46602 correctly identifies the same remaining gap.

## Best Possible Solution

Keep #46602 open. Extend the /status TTS snapshot and formatter to include resolved provider config metadata for the active TTS provider: provider/display label, model or modelId, voice or voiceId, and a sanitized custom-backend marker for non-default baseUrl. Add focused tests covering an OpenAI-compatible custom backend such as messages.tts.providers.openai.baseUrl with custom model/voice, plus legacy direct openai config if still supported by the resolver.
