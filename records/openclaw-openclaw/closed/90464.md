## Summary

Keep this PR open: current main and the latest release still allocate the combined ANSI/OSC scanner inside `truncateToVisibleWidth`, while this branch is a narrow terminal-core performance cleanup with focused regression coverage and sufficient real-output proof.

## What This Changes

The PR hoists the ANSI/OSC truncation scanner to module scope, resets `lastIndex` for each scan, and adds a repeated-call regression test for styled, plain, and OSC-8 truncation.

## Best Possible Solution

Land the narrow scanner reuse after normal maintainer review and exact-head merge/check gates, preserving existing truncation semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a performance cleanup PR. Source inspection confirms current main has the per-call scanner allocation path, and the supplied proof/tests exercise output equivalence after the change.

## Solution Assessment

Yes; hoisting the combined scanner and resetting `lastIndex` before each scan is the narrowest maintainable fix. Replacing the parser would be broader, and leaving the scanner inside the function would preserve the allocation overhead.
