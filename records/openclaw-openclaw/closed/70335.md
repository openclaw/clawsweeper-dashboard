## Summary

Current main already fixes the PR's flattened/bundled logging-config problem via a direct active-config-path reader, so the hashed config chunk loader proposed in #70335 is obsolete. The fix is present in logging, console, and redaction paths, has regression coverage, and is recorded in the changelog, but it is not in the latest v2026.4.24 tag I inspected.

## Best Possible Solution

Close #70335 as implemented on current main and keep the direct active-config-path implementation in src/logging/config.ts. That path solves the user-visible bundled/flattened runtime problem without adding a hashed config chunk scanner; any remaining logging behavior should be tracked as narrower follow-up issues.
