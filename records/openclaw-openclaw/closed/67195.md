## Summary

Keep this PR open. Current main still lacks the MiniMax manifest auth aliases, while the MiniMax runtime still exposes the CN hook aliases and current auth resolution depends on manifest-backed providerAuthAliases before resolving env/config/profile credentials. The PR is also paired with same-author open #63823 and is a focused implementation candidate. Security review found no unexplained supply-chain or code-execution changes in the provided diff.

## Best Possible Solution

Review and land this PR, or apply an equivalent maintainer patch that adds providerAuthAliases for minimax-cn and minimax-portal-cn in extensions/minimax/openclaw.plugin.json plus focused manifest regression coverage in extensions/minimax/index.test.ts. After the fix merges, close #63823 as implemented.
