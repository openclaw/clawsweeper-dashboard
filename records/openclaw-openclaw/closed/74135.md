## Summary

Keep open. Current main contains several relevant mitigations: event-loop diagnostics in readiness/stability output, configurable preauth handshake budgets, #73853's metadata-only startup provider discovery, and #73908's external CLI credential scoping. Those do not fully implement the broader reported fix: ordinary gateway `models.list` and catalog loads can still synchronously reach `ensureOpenClawModelsJson`, non-live provider discovery is still broad when no provider ids are supplied, hot reloads still reset the model catalog cache without a stale-while-revalidate/background refresh path, and #73853 is not in the latest tagged release from the provided context.

## Best Possible Solution

Keep this issue open as the broader gateway responsiveness tracker. The best maintainer path is to retest current main or the next release containing #73853, then coordinate the remaining focused fixes: thread provider filters through full model catalog refreshes where possible, avoid broad runtime provider loading when configured hints exist, make catalog refresh stale-while-revalidate or otherwise non-blocking for latency-sensitive gateway paths, keep live provider discovery bounded and preferably cancellable, and use the new event-loop diagnostics to prove late preauth timers no longer occur under the reported load.
