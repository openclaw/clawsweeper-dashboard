## What This Changes

The issue asks OpenClaw to prevent in-band macOS LaunchAgent stop/restart paths from unloading the gateway without durable recovery or clear diagnostics.

## Best Possible Solution

Keep this as a core macOS gateway availability hardening item: prefer safe restart RPC/SIGUSR1 for responsive gateways, gate destructive in-band LaunchAgent unloads, and make status/doctor report incomplete launchd handoffs distinctly.

## Reproduction Assessment

No full live reproduction was established for the several-hour outage. Source inspection does show a current-main macOS `gateway stop` path that directly calls LaunchAgent `bootout`, and the exact browser-triggered caller chain remains unproven.

## Solution Assessment

Unclear until maintainers choose the lifecycle policy. The maintainable direction is core launchd lifecycle hardening rather than a browser-only workaround, but the in-band stop behavior needs an explicit product decision.
