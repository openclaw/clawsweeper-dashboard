## Summary

Keep open: current main still has the asymmetric legacy-session migration path that can save over an unreadable target store, and the linked fix PR is open rather than merged. The issue should close only after that PR or an equivalent guard lands.

## What This Changes

The issue asks OpenClaw to preserve a corrupt target `agents/{id}/sessions/sessions.json`, warn, and keep the legacy store when legacy session migration cannot read the target store.

## Best Possible Solution

Land the linked fix PR or an equivalent narrow change that refuses to save/delete when the target sessions store is unreadable, emits a warning, preserves legacy data, and adds the corrupt-target regression test.

## Reproduction Assessment

Yes. Source inspection shows a deterministic path: corrupt target JSON5 returns `{ store: {}, ok: false }`, then a readable legacy store makes the current save gate true and allows a legacy-only save over the target.

## Solution Assessment

Yes. The linked fix direction is narrow and maintainable: guard the save and legacy delete on target readability, warn the operator, preserve both stores, and prove the corrupt-target path with one regression test.
