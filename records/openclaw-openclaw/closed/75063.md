## What This Changes

The PR changes the Docker attestation verifier and its tests to accept buildx attestation manifests without artifactType by recognizing in-toto layer media types.

## Best Possible Solution

Keep the shipped verifier behavior from v2026.5.6 and handle any additional layer-mediaType hardening as a separate narrow issue if maintainers want it.

## Reproduction Assessment

No for current main. The cited GHCR image has attestation manifests without an artifactType property, and current main verifies that tag for linux/amd64 and linux/arm64 with no errors.

## Solution Assessment

Yes for the shipped implementation, not for this PR branch. Current main fixes the buildx output at the verifier boundary while preserving rejection of explicit bad artifactType values.
