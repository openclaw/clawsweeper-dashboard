## Summary

Close as `duplicate_or_superseded`: this is the same zero-LLM cron script/exec payload request already tracked by open canonical issue #18160. Current main still lacks direct script/exec cron payload support, and the related implementation PRs remain closed unmerged, so the remaining product and security discussion should stay consolidated on #18160.

## What This Changes

Request to add a zero-LLM cron script or command payload, plus a `--script` CLI path, so scheduled shell commands can run directly while retaining cron run tracking and delivery.

## Best Possible Solution

Consolidate this datapoint into #18160 and land one owner-reviewed, secops-reviewed additive cron command payload there, covering protocol, normalization, runtime dispatch, run history, delivery, CLI, Control UI, docs, and tests.

## Reproduction Assessment

Yes for the capability gap. On current main, a `payload.kind="script"` or `payload.kind="exec"` cron job has no accepted type/schema/CLI path, and isolated-like cron execution still requires `agentTurn`.

## Solution Assessment

Yes for cleanup: closing this duplicate in favor of open #18160 is the narrowest maintainable triage outcome. The implementation itself should be designed and reviewed on the canonical tracker, not split across duplicate issues.
