## Summary

Current main already implements the requested per-run `openclaw agent --model` behavior: CLI flag, Gateway protocol and handler plumbing, local embedded forwarding, non-persistent per-run model resolution, docs, generated Swift protocol models, and regression tests. The fix also shipped in `v2026.4.26` per the release changelog entry for restoring `openclaw agent --model` one-shot CLI runs.

## Best Possible Solution

Close this PR as implemented on main and keep the shipped `v2026.4.26` implementation. Any remaining refinement around global bare-model alias lookup should be handled as a separate focused issue or PR against the current `opts.model`/Gateway override path.
