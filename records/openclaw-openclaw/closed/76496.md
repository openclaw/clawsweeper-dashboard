## Summary

Keep open: current main and v2026.6.6 still build `sessions.usage` totals and aggregate buckets only from the limited page, and an open fix PR with closing syntax is already the implementation candidate.

## What This Changes

The issue requests that `sessions.usage` totals and aggregate buckets use all matched sessions while `limit` only caps the returned `sessions` array.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/93045, or an equivalent gateway fix that aggregates all matched sessions while limiting only returned rows.

## Reproduction Assessment

Yes, source-reproducible. Current main slices to `limitedEntries` before both usage loading and aggregate construction, so a mocked date range with more matched sessions than `limit` will undercount totals while returning only the page.

## Solution Assessment

Yes. The best fix is the narrow gateway change already proposed by the open PR: keep `limit` scoped to returned session rows while computing totals and aggregate buckets from the full matched session set through the existing cached/background loading path.
