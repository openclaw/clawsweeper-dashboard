## Summary

Current `main` already suppresses `NO_REPLY` for heartbeat/cron outbound delivery paths, including mixed-content replies where visible text would otherwise be followed by the silent token. The shared outbound pipeline used by heartbeat delivery drops exact silent payloads when visible content exists, and reply normalization strips leaked trailing `NO_REPLY` text before channel delivery. The changelog records the cron/announce fix as shipped in `v2026.4.15`.

## Evidence
