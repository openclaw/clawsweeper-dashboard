## Summary

Current `main` already bootstraps Undici's env-proxy dispatcher for CLI commands, so `openclaw skills install` no longer relies on a direct bare `fetch` connection when `HTTPS_PROXY`/`HTTP_PROXY` are set.

## Evidence

- **CLI bootstraps env-proxy before commands run:** `runCli()` calls `ensureGlobalUndiciEnvProxyDispatcher()` immediately after env normalization and before command execution, so `openclaw skills install ...` inherits the proxy-aware Undici dispatcher.
  - file: [src/cli/run-main.ts:187](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/cli/run-main.ts#L187)
  - command: `nl -ba src/cli/run-main.ts | sed -n '168,190p'`
- **Proxy bootstrap installs Undici EnvHttpProxyAgent:** `ensureGlobalUndiciEnvProxyDispatcher()` checks for configured HTTP/S proxy env vars and replaces the default global dispatcher with `new EnvHttpProxyAgent()`.
  - file: [src/infra/net/undici-global-dispatcher.ts:91](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/infra/net/undici-global-dispatcher.ts#L91)
  - command: `nl -ba src/infra/net/undici-global-dispatcher.ts | sed -n '70,120p'`
- **ClawHub requests still use global fetch:** ClawHub network calls go through `(params.fetchImpl ?? fetch)`, so once the CLI bootstrap swaps the global dispatcher, skill install/download traffic follows that proxy configuration.
  - file: [src/infra/clawhub.ts:382](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/infra/clawhub.ts#L382)
  - command: `nl -ba src/infra/clawhub.ts | sed -n '372,390p'`
- **Tests cover the exact env-proxy upgrade:** Unit coverage asserts that when env proxy configuration is present, the bootstrap installs `EnvHttpProxyAgent` instead of leaving the default agent in place.
  - file: [src/infra/net/undici-global-dispatcher.test.ts:207](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/infra/net/undici-global-dispatcher.test.ts#L207)
  - command: `nl -ba src/infra/net/undici-global-dispatcher.test.ts | sed -n '199,260p'`
- **Proxy env resolution matches HTTPS_PROXY/HTTP_PROXY semantics:** Proxy resolution explicitly prefers `https_proxy`/`HTTPS_PROXY` for HTTPS requests and falls back to `http_proxy`/`HTTP_PROXY`; tests cover those cases.
  - file: [src/infra/net/proxy-env.ts:28](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/infra/net/proxy-env.ts#L28)
  - command: `nl -ba src/infra/net/proxy-env.ts | sed -n '1,120p'`
