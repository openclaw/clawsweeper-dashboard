## Summary

Keep open. Current main does not already contain the requested provider runtime-deps repair, and source review of the PR head confirms concrete correctness gaps in the new provider-selection logic; the protected `maintainer` label also makes this explicit maintainer-review material.

## What This Changes

The PR centralizes bundled runtime-deps selection for configured bundled providers, updates related docs/tests/changelog, and makes external runtime mirror file refresh atomic.

## Best Possible Solution

Canonicalize both collected config provider refs and manifest provider refs with the shared provider normalizer, include `agents.defaults.subagents.model` in the configured-provider scan, add regression coverage for alias refs and subagent defaults, then rerun the targeted runtime-deps tests before maintainer merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Normalize provider IDs before matching:** `src/plugins/bundled-runtime-deps-selection.ts:202-203`
  - body: The new provider selection compares only lowercased strings, but provider routing canonicalizes aliases such as `bedrock`/`aws-bedrock` to `amazon-bedrock`. An alias-based config will not match the bundled manifest provider, so doctor and `plugins deps` can still skip the provider deps this PR is meant to stage.
  - confidence: 0.9
- **[P2] Include default subagent model refs:** `src/plugins/bundled-runtime-deps-selection.ts:175-181`
  - body: `agents.defaults.subagents.model` is a supported fallback for spawned subagents, but the new scan only reads per-agent `subagents.model`. If the provider is configured only through the global subagent default, runtime-deps repair will miss it until subagent execution tries to load the provider plugin.
  - confidence: 0.82
