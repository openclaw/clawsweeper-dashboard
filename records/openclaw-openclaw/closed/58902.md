## Summary

Current `main` already fixes the ACP duplicate-final-reply problem with a safer channel-specific visibility model, plus regression tests for Discord, Slack, Telegram, and fallback-preserving channels. This PR’s unconditional `kind === "block"` change is now obsolete and weaker than the implementation on `main`.

## Evidence
