## Summary

Current `main` already moved the bundled `claude-cli` path onto a warm stdio/session model with explicit non-TTY stream-json stdin handling, added regression coverage for that spawn path, and applies service-managed spawning behavior for launchd/systemd-style runtimes. The issue describes OpenClaw 2026.4.1 behavior, but the current implementation is materially different.

## Evidence

- **Claude CLI now uses warm stdio session args:** `buildClaudeLiveArgs` forces Claude live sessions onto `--input-format stream-json`, `--permission-prompt-tool stdio`, and `--replay-user-messages`, which is the non-interactive stdio path rather than the older one-shot shell-style invocation described in the report.
