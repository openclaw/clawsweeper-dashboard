Full review comments:

- **[P1] Remove the runtime @latest package repair:** `railway-start.sh:14`
  - body: This startup path installs `openclaw@latest` and `@openclaw/codex@latest` into a persistent cache when the container starts. That lets a reviewed image execute newer unpinned package code and drift from the image's bundled versions; use the built artifact or a pinned, verified package/version instead.
  - confidence: 0.94
- **[P1] Do not clobber configured Control UI origins:** `railway-start.sh:19`
  - body: This command replaces `gateway.controlUi.allowedOrigins` with only the Railway domain on every startup. Existing persisted origins in `/data/.openclaw/openclaw.json` would be silently removed, so the script should merge the Railway origin with the current array and preserve user settings.
  - confidence: 0.9
- **[P2] Keep the Docker cache mounts or update the contract:** `Dockerfile:68`
  - body: The PR removes the root Dockerfile pnpm and apt cache mounts, but current `src/docker-build-cache.test.ts` explicitly asserts those mounts for dependency and system-package install layers. This will break the existing Docker build-cache contract unless that contract is deliberately redesigned with matching tests and rationale.
  - confidence: 0.88
- **[P2] Keep the runtime image non-root by default:** `Dockerfile:279`
  - body: Changing the final image to `USER root` means overridden commands now run as root by default, reversing the current Dockerfile hardening that switches the runtime stage to `USER node`. If Railway needs ownership repair, use a narrow entrypoint that drops privileges without changing the general image default.
  - confidence: 0.82
