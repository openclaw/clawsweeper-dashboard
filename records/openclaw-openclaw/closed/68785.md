  - body: The default breaker is only added in enqueueCommand, but current embedded-agent runs enqueue the resolved global lane through enqueueCommandInLane directly.   - confidence: 0.92

  - body: The timeout wrapper rejects the queued promise but leaves entry.task running, and the catch path then frees the lane and pumps the next queued task. Lanes are used to serialize command/session side effects, so this can overlap later work with the original still-running task; preserve the current taskTimeoutMs/progress/abort/release design instead of reintroducing maxExecutionMs-only release.
  - confidence: 0.9
