## Summary

Current main and the latest shipped release already implement the memory-core promotion path; the reported token is an internal managed cron sentinel, not a PATH executable.

## What This Changes

The issue reports that the daily memory promotion token is missing as an executable command and prevents scheduled memory promotion.

## Best Possible Solution

Keep the shipped internal managed-cron sentinel design; users should use the documented memory CLI/status commands rather than adding a PATH executable for the sentinel.

## Reproduction Assessment

No; current main source and tests show the sentinel is handled as an internal cron/heartbeat payload, and the supported manual command is `openclaw memory promote`. The issue does not include a concrete current-main cron record or log path that bypasses that handler.

## Solution Assessment

Yes; the current implementation is the maintainable solution because memory-core owns the managed cron and trigger handling, while user-facing manual promotion stays on the documented `openclaw memory promote` CLI.
