## Summary

Keep open: current main has useful mitigations, and the old plugin-runtime-deps cache/SDK-file mismatch is now legacy cleanup, but the broader macOS/Homebrew runtime ownership problem remains unresolved in source and in the maintainer discussion.

## What This Changes

The issue reports OpenClaw becoming unavailable after macOS/Homebrew updates because the CLI, gateway service Node, OpenClaw package, plugin SDK, and plugin runtime state can drift apart.

## Best Possible Solution

Decide the macOS gateway runtime authority, then land or replace the focused runtime/path follow-ups so updates, LaunchAgent generation, plugin cleanup, and diagnostics enforce one coherent runtime boundary.

No high-confidence end-to-end reproduction is available here; the Homebrew ABI break requires an affected macOS/Homebrew state. Source inspection does confirm the remaining external Node/package boundary and the open bundled-node service path gap on current main.
