## Summary

Keep open: current main still overwrites the auto-managed `meta.lastTouchedVersion` and `meta.lastTouchedAt` fields after config CLI mutation handling, and an open linked fix PR already closes this issue so the issue should stay open until that PR lands or is rejected.

## What This Changes

The issue requests that `openclaw config set` stop reporting success for auto-managed `meta.lastTouched*` values that the config writer later overwrites.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80856 or an equivalent CLI-boundary rejection, then close this issue as implemented once the fix reaches main.

## Reproduction Assessment

Yes. The reproduction path is to run `openclaw config set meta.lastTouchedVersion BOGUS-NOT-A-VERSION` or `openclaw config set meta.lastTouchedAt 1999-01-01T00:00:00.000Z` against a config, then inspect the persisted JSON; source inspection shows the final writer overwrites those values, and the issue/PR comments include live terminal proof.

## Solution Assessment

Yes. The narrow maintainable fix is to reject mutations targeting these documented auto-managed fields at the CLI boundary while leaving `stampConfigVersion` as the system-owned writer.
