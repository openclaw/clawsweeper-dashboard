## Summary

Current `main` already contains targeted fixes for the reported isolated-cron Telegram failure mode: the gateway now pins the startup channel registry so later registry refreshes do not drop the Telegram outbound adapter, and Telegram polling now force-restarts and rebuilds its transport on recoverable network/stall conditions that match the report's long-running NAT/IPv6 symptoms.
