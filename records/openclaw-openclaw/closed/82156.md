## Summary

Close as implemented on current main: the active-memory path now uses backend-specific memory tool defaults and no longer includes `memory_recall` for the built-in memory-core path. The fix is current-main-only in this checkout and is not contained in the latest tagged release checked here.

## What This Changes

The issue reports that active-memory embedded live Telegram sessions fail on memory-core because the hidden sub-agent may include backend-specific memory tools in a strict allowlist.

## Best Possible Solution

Keep the backend-sensitive allowlist behavior on main and ship it in the next release so 2026.5.7 users move off the stale hardcoded tool set.

## Reproduction Assessment

No high-confidence live reproduction was run against current main. The suspected root cause is source-verifiable, and current main no longer includes `memory_recall` in the memory-core default embedded `toolsAllow`.

## Solution Assessment

Yes. The current implementation is the narrow fix: default memory-core to `memory_search`/`memory_get`, preserve `memory_recall` for `memory-lancedb`, and pass only that resolved list to the embedded run.
