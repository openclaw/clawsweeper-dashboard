## Summary

## What This Changes

Request bundle-MCP server initialization to apply per-server exponential backoff, jitter, and attempt caps for rate-limited streamable HTTP/SSE MCP startup failures.

## Best Possible Solution

Track the remaining bundle-MCP failed-start throttling work in #68527, while preserving this issue's narrower HTTP rate-limit case: transient `429` and `temporarily_unavailable` startup failures should cool down per server across session runtimes, reset on success or config change, and avoid hiding permanent auth failures such as `invalid_token`.
