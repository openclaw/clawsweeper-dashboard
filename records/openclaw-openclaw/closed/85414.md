## What This Changes

The branch routes systemd managed update handoffs through `systemd-run --user --scope --collect`, adds a 2s systemd restart grace, updates focused tests, and adds a changelog entry.

## Best Possible Solution

Land the transient user-scope handoff only after real user-systemd update proof shows `handoff.log` creation, version change, healthy gateway restart, and a final non-stuck update sentinel while preserving non-systemd paths.

## Reproduction Assessment

Yes: the linked issue gives a concrete Linux user-systemd upgrade failure, and current main still starts the helper as a plain detached Node child before scheduling the supervisor restart. I did not run the live upgrade in this read-only review.

## Solution Assessment

Yes, with proof: using a transient user scope is the narrow maintainable direction for escaping the systemd service cgroup while preserving launchd, schtasks, and non-supervised paths. The missing piece is real user-systemd upgrade evidence.
