## Summary

PR #41929 is a stale test-only PR superseded by the later merged Telegram status work in #54362 and by the current Telegram plugin contract on main. The PR’s proposed regression expectation treats unavailable tokenFile credentials as top-level configured, while current main deliberately requires a readable/resolved token for Telegram configured status and already tests the replacement behavior.

## Best Possible Solution

Close #41929 as superseded. Keep the shipped #54362 Telegram status contract: channel-level fallthrough works for single-bot binding-created account IDs, multi-bot unknown IDs are blocked, and configured-but-unavailable tokenFile/SecretRef credentials are reported as unavailable rather than counted as configured. Track any remaining current-main doctor/status false positive in #44539 or a new issue with current config, logs, and version details.
