Keep open: this PR addresses a source-reproducible prompt-cache regression with a narrow, plausible fix, but it is an open implementation PR with the protected `maintainer` label and should proceed through maintainer review and normal merge gates rather than cleanup closure.

## What This Changes

The PR removes ordinary-turn runtime context from the session system prompt, preserves current-turn hidden runtime context through Pi `nextTurn` custom messages, updates LLM-boundary filtering tests, adds a transcriptPrompt regression, and adds a changelog entry.

## Best Possible Solution

Land this PR, or an equivalent narrow fix, after maintainer review confirms current-turn runtime context remains model-visible while historical runtime context stays out of provider replay.

## Reproduction Assessment

Yes, by source inspection. Current main appends `promptSubmission.runtimeContext` into `systemPromptText` before the model turn, while the docs define `prependContext` as per-turn dynamic text rather than stable system context.

## Solution Assessment

Yes. The PR uses Pi's existing `nextTurn` custom-message path and dependency source shows those messages are appended after the current user message, so the historical-strip rule preserves the current turn while removing old runtime context on later turns.
