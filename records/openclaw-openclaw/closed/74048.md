## Summary

Close as superseded: current main already has the useful ACP alias/timeout fixes through merged PRs, while this branch still changes missing `configOptionKeys` into a strict empty allowlist that would silently drop existing runtime controls.

## What This Changes

The PR changes `applyManagerRuntimeControls` so ACP config options are sent only when advertised keys exist and are skipped entirely when no keys are advertised.

## Best Possible Solution

Keep the merged current-main alias and timeout fixes, close this superseded branch, and track any broader Claude ACP lifecycle or environment failures in https://github.com/openclaw/openclaw/issues/78546.

## Reproduction Assessment

Yes. The original timeout/thinking failures are source/report reproducible and now have merged replacement fixes; the PR regression is also source-reproducible because current main treats missing `configOptionKeys` as permissive and tests cover no-key forwarding.

## Solution Assessment

No. This branch solves the symptom by changing the no-key capability contract, while the merged fixes in https://github.com/openclaw/openclaw/pull/79926 and https://github.com/openclaw/openclaw/pull/80812 are the narrower maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Preserve no-key config-option semantics:** `src/acp/control-plane/manager.runtime-controls.ts:153-158`
