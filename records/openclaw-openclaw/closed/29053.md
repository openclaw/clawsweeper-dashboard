## Summary

Current main already has native outbound MCP server support: OpenClaw stores MCP server definitions under `mcp.servers`, supports stdio plus remote HTTP/SSE and `streamable-http`, auto-discovers tools from configured servers, materializes them into agent tools, and allows profile-level opt-out via `tools.deny: ["bundle-mcp"]`.

## Close Comment

Thanks for the request. Closing this as implemented after a Codex review of current `main`.

OpenClaw now has native outbound MCP server support through `mcp.servers`, with documented stdio, HTTP/SSE, and `streamable-http` transports in `docs/cli/mcp.md:356`. Current runtime code wires those definitions into agent execution in `src/agents/embedded-pi-mcp.ts:11`, connects and auto-discovers tools from configured servers in `src/agents/pi-bundle-mcp-runtime.ts:163`, and materializes them as callable agent tools in `src/agents/pi-bundle-mcp-materialize.ts:56`. Profile-level control is also present: `docs/tools/index.md:140` and `src/agents/pi-embedded-runner/effective-tool-policy.test.ts:122` show `tools.deny: ["bundle-mcp"]` hiding MCP tools per agent/profile.

The shipped release evidence is in `CHANGELOG.md:1636` under `2026.3.31`, which records both runnable MCP tools and remote `mcp.servers` URL support. If there is still a missing gap, please open a fresh issue for that specific limitation with a current repro path.
