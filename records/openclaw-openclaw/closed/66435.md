## Summary

Current `main` already implements the underlying fix in a safer form than this PR: gateway calls now clear the wrapper timeout, wait for client teardown before settling, and swallow `stopAndWait` failures via a fallback so cleanup-timeout errors do not mask successful responses.

## Evidence
