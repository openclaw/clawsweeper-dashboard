## Summary

Current main covers the reported single Claude ACP no-output hang path: the bundled acpx plugin is startup-activated with lazy backend registration, ships local Claude/Codex ACP adapter dependencies, normalizes known npx adapter commands through local wrappers, propagates ACPX permission settings into the runtime, and documents the non-interactive permission remedy. The useful follow-up comments identified the same two causes: non-interactive permissions and slow npx adapter startup. Remaining concurrent-spawn no-output behavior is tracked separately in #53319.

## Best Possible Solution

Close this issue as fixed on current main, keep the lazy ACPX startup/local adapter packaging and permission documentation, and continue any remaining concurrent no-output investigation under #53319.
