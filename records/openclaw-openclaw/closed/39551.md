## What This Changes

The PR changes `sessions_send` to prefer `sessionKey` or `sessionId` over redundant `label`/`agentId`, updates the tool description and changelog, and adds mixed-selector regression tests.

## Best Possible Solution

Finish the fix in one canonical branch, preferably #74009 or its successor: remove the mixed `sessionKey`+`label` rejection, keep the public selector contract narrow, avoid returning resolved canonical keys on denied sends, and isolate the new tests before running the focused sessions test and changed gate.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: call `sessions_send` with non-empty `sessionKey` and `label`, and the guard at `src/agents/tools/sessions-send-tool.ts:109` returns the reported error before target resolution.

## Solution Assessment

No as this branch stands. Preferring `sessionKey` is the right narrow direction, but this PR is superseded by #74009 and still needs the denied-key disclosure and persistent mock implementation issues resolved before any equivalent patch should merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Avoid echoing resolved keys on denied mixed-selector sends:** `src/agents/tools/sessions-send-tool.ts:103-108`
  - body: By making `sessionKey`/`sessionId` authoritative over `label`, a session-id-shaped mixed call now resolves to a canonical key before visibility or A2A denial. The existing denial response returns `sessionKey: displayKey`, so an unauthorized caller can confirm the session id and learn the resolved key. Return no key, or only the original caller-provided selector, on denied access before allowing this path.
  - confidence: 0.84
- **[P2] Reset gateway mock implementations between send-gating tests:** `src/agents/tools/sessions.test.ts:654`
  - body: The new mixed-selector tests install `callGatewayMock.mockImplementation(...)`, but this describe's `beforeEach` only calls `mockClear()`. That leaves the implementation in place for later tests, so send-gating coverage can depend on test order. Reset/restore the mock implementation in the local setup or use one-shot mock results in the new tests.
  - confidence: 0.86

## Security Review

Status: needs_attention
