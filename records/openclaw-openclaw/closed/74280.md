## Summary

Keep open. This is a fresh Windows restart regression report against v2026.4.26 where the reporter explicitly says the shipped c85065e policy-close fix is already present, but restart is still slow and logs still show startup delay plus loopback WebSocket closes/timeouts. Current main has the `connect failed` allowlist and adjacent startup-handshake retry work, but the Windows restart path still has long health-wait windows and there is no Windows 11 proof that current main resolves this specific report, especially with the external `moltbot-popo` plugin in the logs.

## What This Changes

The issue reports that `openclaw gateway restart` remains extremely slow on Windows 11 in v2026.4.26 even after the earlier `connect failed` restart-health fix shipped.

## Best Possible Solution

Keep this issue open and reproduce on Windows 11 with both v2026.4.26 and current main. Collect restart/status JSON, gateway logs, and startup trace, then repeat with `moltbot-popo` disabled and enabled. If core reproduces, fix the narrow Windows Scheduled Task, restart-health, or startup-handshake path with targeted tests; if only the external plugin reproduces, route the plugin manifest/runtime problem separately.
