- `impact:message-loss`: The bug can prevent Telegram media replies from being delivered when the configured media cap should allow them.

## Summary

Keep open. Current main still has the reported Telegram reply-media cap gap, and https://github.com/openclaw/openclaw/pull/83478 is an open implementation PR with closing syntax, so this issue should track that PR until it merges or is closed.

## What This Changes

Telegram final and native-command media replies should pass the configured `mediaMaxMb` limit into outbound media loading.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83478 if it continues to preserve the existing media loader options while adding `maxBytes`, then close this issue from the merged fix.

## Reproduction Assessment

Yes. A source-level reproduction is clear: configure `channels.telegram.mediaMaxMb`, trigger a Telegram final or native reply with `mediaUrl`/`mediaUrls`, and current main reaches `loadWebMedia` without passing `maxBytes`.

## Solution Assessment

Yes. Threading the already-computed `mediaMaxBytes` through the existing Telegram reply delivery options into `buildOutboundMediaLoadOptions({ maxBytes })` is the narrowest maintainable fix.
