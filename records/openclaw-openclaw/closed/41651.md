## Summary

Current main already fixes the reported false-positive skill symlink warning: managed and personal skill-directory symlinks are accepted while per-skill `SKILL.md` containment still fails closed, and that behavior has shipped.

## What This Changes

The issue asks OpenClaw to stop repeatedly warning when user-owned skill roots contain symlinks to trusted bundled or managed skills.

## Best Possible Solution

Keep the shipped loader behavior and docs; any new report should include a symlink layout that still warns on `v2026.5.18` or newer.

## Reproduction Assessment

No current-main failing reproduction remains. Source and regression tests show directory symlinks under managed or personal skill roots are accepted without warning while escaped `SKILL.md` files are still rejected.

## Solution Assessment

Yes. The shipped fix handles the user-owned managed/personal skill-manager symlink path at the loader boundary and is safer than only downgrading the warning.
