## Summary

Keep open. Current main has partial timeout summarization and parent-review guidance, but it still lacks the requested proactive checkpointing, structured named-deliverable verification, and resume/follow-up contract for long-running subagent work.

## What This Changes

The issue requests more reliable long-running subagent handling through proactive checkpointing, deliverable verification, explicit partial-progress timeout state, and resume guidance.

## Best Possible Solution

Define a bounded subagent completion contract that carries checkpoint/progress metadata, optional named deliverable expectations, verification results, and resume guidance through spawn, registry, announce, and task-ledger surfaces.

## Reproduction Assessment

No high-confidence live reproduction was established. The reporter's SQLite examples are concrete, and source inspection confirms the missing structured checkpoint, deliverable-verification, and resume mechanisms on current main.

## Solution Assessment

Unclear. The suggested direction is reasonable, but the maintainable fix should start with an explicit runtime contract for checkpoint and deliverable metadata before adding automatic resume or spawn-time warnings.
