## Summary

Keep this PR open. Current main still hardcodes file-backed memory behavior in the workspace templates, while memory-core's prompt builder only emits recall-tool guidance. The existing plugin state already supports active memory plugins owning prompt builders, so this PR's central refactor is still relevant and not implemented on main.

## Best Possible Solution

Review and either land this PR or an equivalent main-branch patch that removes backend-specific memory instructions from workspace templates, moves memory-core-specific persistence guidance into memory-core's promptBuilder, keeps the shared privacy rule in a plugin-agnostic template section, and adds focused prompt/template assertions if maintainers want a guardrail for the string relocation.
