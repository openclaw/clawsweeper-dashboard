## Summary

Keep open: this PR targets a real, source-reproducible cron announce leak linked to https://github.com/openclaw/openclaw/issues/85421, but it is not merge-ready because the patch still duplicates the existing control-reply classifier and the proof is unit-test only.

## What This Changes

The PR suppresses exact `REPLY_SKIP` and `ANNOUNCE_SKIP` replies in isolated cron announce delivery and adds core-channel direct-delivery regression coverage.

## Best Possible Solution

Move the control-reply classifier to a neutral shared module, reuse it from both gateway/chat and cron delivery, keep the focused regression tests, and add redacted real cron announce proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main normalizes only `NO_REPLY` in cron direct delivery, so exact `REPLY_SKIP` or `ANNOUNCE_SKIP` text would continue into the outbound send path. I did not run tests because this review is read-only.

## Solution Assessment

No, not as written: the PR fixes the right runtime path, but it reimplements an existing control-reply classifier instead of sharing the established suppression contract. A small shared-helper move is the safer maintainable fix.
