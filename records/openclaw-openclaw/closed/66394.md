## Summary

Current main and the v2026.4.26 release already implement the requested log rotation behavior: file logs rotate at logging.maxFileBytes, keep bounded numbered archives, and continue writing after rotation. The related PR #66457 was closed unmerged, but equivalent built-in behavior is present in source, tests, docs, and the shipped changelog.

## Best Possible Solution

Close this broad request as implemented. Keep the shipped built-in rotation behavior, and track archive compression or configurable archive retention separately only if maintainers want those extra knobs.
