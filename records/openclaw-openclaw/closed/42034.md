## Close Comment

Thanks for the idea. Codex review against current `main` points to this belonging in ClawHub rather than OpenClaw core. The project vision explicitly says optional capability should usually ship as plugins, and that new skills should go to ClawHub first (`VISION.md:54`, `VISION.md:69`, `VISION.md:101`). It also says setup is intentionally terminal-first so auth, permissions, and security decisions stay explicit, and that OpenClaw does not want convenience wrappers hiding those decisions (`VISION.md:87-91`). Current setup/reconfiguration already centers on `openclaw onboard` and `openclaw configure` (`docs/start/wizard.md:10-29`), while current `main` already supports ClawHub skill installation (`docs/tools/skills.md:109-115`, `src/gateway/server-methods/skills.ts:175-201`, `src/gateway/server-methods/skills.clawhub.test.ts:46-87`). Closing this core issue as a ClawHub-scope feature request; if someone wants to pursue it, the compatible path is an external ClawHub skill/plugin rather than a built-in core agent.
