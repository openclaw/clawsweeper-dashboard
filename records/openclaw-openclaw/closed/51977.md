## Summary

Current main still supports the core reported bloat path: `skillsSnapshot.prompt` is a required field on persisted session skill snapshots, normal reply and CLI paths still write that snapshot into session entries, reset and follow-up paths still copy it forward, and the session store still serializes and caches the full entry graph. Cron per-run duplication has been narrowed, but the durable rendered-prompt retention problem remains.

## Best Possible Solution

Keep this open and fix the root persistence model: make durable session skill snapshots metadata-only, avoid writing rendered `skillsSnapshot.prompt` to `sessions.json`, rebuild or share rendered prompts lazily by version/filter/hash when a run needs them, strip full snapshots from reset and queued follow-up payloads, preserve skill refresh semantics, and add regression coverage for session store serialization, normal reply, CLI, reset, follow-up, and isolated cron paths. Coordinate explicitly with #55334 if maintainers want one canonical tracker.
