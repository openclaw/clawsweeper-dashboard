## Summary

Keep open: current main and v2026.5.12 still let the TUI watchdog clear an active run to idle and emit the reported prompt based on stream silence alone. The related open PRs cover stale-notice dismissal or timeout configurability, but neither resolves the authoritative quiet-active-run state bug.

## What This Changes

The issue asks for quiet but still-active TUI chat runs to remain visibly working until authoritative run/session state says the run is terminal.

## Best Possible Solution

Before the watchdog clears a TUI run or prompts the user, refresh or consume session/run state such as sessions.list.hasActiveRun and rearm the working state while the current session still has an active run.

## Reproduction Assessment

Yes, at source level. A chat delta arms the watchdog, timer expiry clears the active run to idle and logs the reported prompt, and current tests assert that behavior without consulting gateway active-run state.

## Solution Assessment

No, the current implementation is not the best solution because it treats stream silence as enough to reset the UI. The narrower maintainable fix is to check authoritative active-run state before clearing TUI run state, while leaving timeout configurability to the separate config thread.

## Repair Work Prompt

Fix the TUI streaming watchdog false-idle bug tracked by https://github.com/openclaw/openclaw/issues/78360. On watchdog expiry, do not clear activeChatRunId, set idle, or add the taking-longer prompt solely because no recent chat delta arrived if authoritative current-session state still reports an active run. Prefer carrying sessions.list.hasActiveRun through the TUI session-list/session-info path or refreshing it from the watchdog before deciding; rearm or keep the visible working state while the backend run is active. Preserve recovery behavior for genuinely dead streams, lost final events, reconnect recovery, local run history reloads, and normal final/aborted/error cleanup. Do not add a new config option or change watchdog default policy here; that remains tracked by https://github.com/openclaw/openclaw/issues/68596 and https://github.com/openclaw/openclaw/pull/79997. Add focused regression coverage for a quiet active run and update existing watchdog tests accordingly. Changelog expected because this is user-visible TUI behavior.
