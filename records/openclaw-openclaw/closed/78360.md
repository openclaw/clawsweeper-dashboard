## Label Justifications

- `P2`: The report describes a normal-priority TUI state correctness bug affecting long-running agent sessions.
- `impact:session-state`: The affected behavior is TUI active-run/session state drifting from an actually active backend run.

## Summary

Close: current main and the latest release keep the TUI active run alive during normal watchdog silence, rearm the watchdog on tool/lifecycle proof-of-life, and dismiss stale watchdog notices once output resumes. The distinct delta-only live projection problem remains tracked separately at https://github.com/openclaw/openclaw/issues/82988.

## What This Changes

The issue reports that the TUI streaming watchdog can mark quiet but still-active long-running chat runs idle and show repeated continue prompts before later output arrives.

## Best Possible Solution

Keep the shipped v2026.5.28 watchdog behavior and use https://github.com/openclaw/openclaw/issues/82988 for the separate delta-only live projection problem if that still reproduces.

## Reproduction Assessment

Yes. The old watchdog source path cleared activeChatRunId and set idle on delta silence, and current regression tests now encode the quiet-active, tool-event, and late-output recovery cases; no live TUI run was attempted in this read-only review.

## Solution Assessment

Yes. The current implementation is the maintainable fix for this issue because it preserves active run state during ordinary quiet periods, limits idle cleanup to reconnect recovery, tracks proof-of-life events, and removes stale notices when output resumes.
