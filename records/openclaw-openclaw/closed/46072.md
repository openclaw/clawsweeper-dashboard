## Summary

Current `main` already changed `openclaw gateway status` to report connectivity separately from read-scope RPC health, which addresses the misleading `unreachable (missing scope: operator.read)` behavior described in #46072.

## Evidence

- **Default status probe is now connect-only:** `probeGatewayStatus()` uses `probeGateway(... includeDetails: false)` for normal `gateway status`, and only switches to a read-scope RPC check when `--require-rpc` is set.

- **Status output separates reachability from capability:** `printDaemonStatus()` prints `Connectivity probe:` or `Read probe:` based on probe kind, and prints `Capability:` on its own line instead of folding missing `operator.read` into an `unreachable` label.
