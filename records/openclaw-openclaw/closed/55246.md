## Summary

Issue #55246 is already implemented on current main, and the same watchdog-reset behavior is present in release v2026.4.22. The WhatsApp reconnect path creates a fresh live connection with a new `startedAt` timestamp and clears `lastInboundAt`, and the watchdog uses `lastInboundAt ?? startedAt` as its inactivity baseline. A dedicated regression test asserts that a reconnected listener gets a fresh watchdog window instead of immediately looping.

## Close Comment

Closing as implemented on main. Codex review found that the WhatsApp watchdog now resets its inactivity baseline on reconnect: each new live connection starts with `startedAt: Date.now()` and `lastInboundAt: null` in `extensions/whatsapp/src/connection-controller.ts:103`, and the watchdog compares inactivity against `connection.lastInboundAt ?? connection.startedAt` in `extensions/whatsapp/src/connection-controller.ts:563`. This implementation is already present in release `v2026.4.22` at `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`.
