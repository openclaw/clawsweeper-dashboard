## Summary

The failure is caused by the external `chrome-openclaw-sider` npm package being published without the built JavaScript runtime files that OpenClaw requires for installed plugins; current and shipped OpenClaw are enforcing the package contract correctly, and the extra hook error is only the CLI hook-pack fallback.

## What This Changes

The issue reports that installing the external `chrome-openclaw-sider` plugin fails on Windows because its npm package lacks built runtime files and is not a hook pack.

## Best Possible Solution

The Sider package publisher should republish the npm package with built JavaScript runtime artifacts and matching runtime metadata, while OpenClaw keeps enforcing the installed-plugin package boundary.

## Reproduction Assessment

Yes. The npm metadata and tarballs for the reported package versions show TypeScript source entries with no built runtime files, and OpenClaw's install validator rejects that package shape by design.

## Solution Assessment

Yes. The narrow maintainable solution is to fix the external package publication rather than weakening OpenClaw to execute TypeScript source from installed npm packages.
