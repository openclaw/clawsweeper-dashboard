## Summary

Current main removes the reported global-prefix hazard by externalizing Matrix from the root npm package and routing Matrix setup through the official external plugin catalog and managed plugin install paths. The latest stable release is still older, so this is current-main-only until the next release ships.

## What This Changes

The issue asks OpenClaw to stop Matrix configure dependency repair from running npm in a user's global npm prefix and pruning unrelated global packages.

## Best Possible Solution

Keep Matrix on the official external plugin path and ship the current-main package/catalog changes in the next release so configure no longer depends on root-package Matrix repair.

## Reproduction Assessment

Yes. The original v2026.5.7 report is source-plausible from the old bundled Matrix repair path, and current main removes that root-package path by externalizing Matrix; I did not execute the destructive npm-prefix scenario.

## Solution Assessment

Yes. Moving Matrix out of the root package and through the official managed plugin install path is the maintainable core fix because it avoids writing Matrix dependencies into the user's global npm prefix.
