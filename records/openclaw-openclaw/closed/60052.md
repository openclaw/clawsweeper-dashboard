## Summary

Current `main` makes this PR obsolete: the runtime image now exposes `openclaw` directly without relying on npm global installs as the non-root `node` user, and the supported Docker flows use `openclaw-cli`/`openclaw-gateway` or explicit per-session `NPM_CONFIG_PREFIX` + `PATH` setup when a Docker test intentionally does `npm install -g`.

## Evidence

- **Runtime image avoids npm-global dependency:** `Dockerfile:257` now documents and implements `openclaw` exposure 'without requiring npm global writes as non-root', then switches to `USER node` at `Dockerfile:263`. That covers the container CLI path this PR was trying to fix by changing npm prefix.

## Risks / Open Questions

- If the original intent was to support arbitrary third-party `npm install -g` inside the published runtime image, current `main` does not preconfigure that globally; it instead avoids needing it for OpenClaw itself and uses explicit prefix/PATH setup in targeted Docker workflows.
- `gh` inspection of issue `#59601` was unavailable in this environment because `GH_TOKEN` is not configured, so this decision relies on the provided PR context plus current `main` code/docs/tests.
