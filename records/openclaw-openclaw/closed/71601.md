## Summary

Current `main` still has the docs gap this PR fixes. The gateway configuration reference only documents `~` expansion for top-level `browser.executablePath`, while browser config normalization on `main` also expands per-profile `executablePath` and `existing-session` `userDataDir`. This is a fresh, docs-only PR with discussion already confirming the mapping, so there is no conservative basis to close it.
