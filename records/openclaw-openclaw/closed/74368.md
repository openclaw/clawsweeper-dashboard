## Summary

Current main already solves the onboarding friction: `openclaw onboard` routes default-model setup through the shared catalog-backed model picker, which presents selectable model options and keeps manual entry only as a fallback. The same implementation is present in the shipped `v2026.4.26` tag.

## What This Changes

The issue requests replacing or supplementing manual onboarding model entry with a dropdown-style prompt during `openclaw onboard` model selection.

## Best Possible Solution

Keep the current catalog-backed searchable model picker as the onboarding default and treat any future gaps as narrower provider/catalog bugs when a specific auth path still falls back to manual entry unexpectedly.
