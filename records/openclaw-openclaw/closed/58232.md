## Summary

Current `main` already implements fallback behavior for session-level model overrides, including same-provider overrides that differ from the configured primary, and preserves explicit user `/model` selections while using fallback candidates during failover.

## Evidence

- **Fallback chain now includes configured fallbacks for same-provider session overrides:** `resolveFallbackCandidates` always keeps the full configured fallback chain when the active session model is on the same provider, even if the override differs from the configured primary. That addresses the reported case where a session-level override would otherwise keep retrying the same model.
  - file: [src/agents/model-fallback.ts:492](https://github.com/openclaw/openclaw/blob/0783090da0090982c19f7803a9daf9a2ec29538e/src/agents/model-fallback.ts#L492)
  - sha: [0783090da009](https://github.com/openclaw/openclaw/commit/0783090da0090982c19f7803a9daf9a2ec29538e)
- **Regression tests cover the reported scenario:** Tests explicitly verify that when the session model differs from config within the same provider and the first attempt rate-limits, OpenClaw advances to the configured fallback model instead of retrying only the overridden model.
  - file: [src/agents/model-fallback.test.ts:1473](https://github.com/openclaw/openclaw/blob/0783090da0090982c19f7803a9daf9a2ec29538e/src/agents/model-fallback.test.ts#L1473)
  - sha: [0783090da009](https://github.com/openclaw/openclaw/commit/0783090da0090982c19f7803a9daf9a2ec29538e)
- **User-selected `/model` overrides are preserved across turns:** Model-selection logic documents and enforces that auto-failover overrides are transient but user-selected overrides are preserved, so fallback retries can occur without clobbering the user's explicit session choice.
  - file: [src/auto-reply/reply/model-selection.ts:173](https://github.com/openclaw/openclaw/blob/0783090da0090982c19f7803a9daf9a2ec29538e/src/auto-reply/reply/model-selection.ts#L173)
  - sha: [0783090da009](https://github.com/openclaw/openclaw/commit/0783090da0090982c19f7803a9daf9a2ec29538e)
- **Execution tests protect explicit user override behavior during fallback:** Regression coverage confirms fallback selection is not persisted over a user-selected session override (`modelOverrideSource: "user"`), which matches the intended `/model` semantics while still allowing the fallback run to proceed.
  - file: [src/auto-reply/reply/agent-runner-execution.test.ts:2560](https://github.com/openclaw/openclaw/blob/0783090da0090982c19f7803a9daf9a2ec29538e/src/auto-reply/reply/agent-runner-execution.test.ts#L2560)
  - sha: [0783090da009](https://github.com/openclaw/openclaw/commit/0783090da0090982c19f7803a9daf9a2ec29538e)
