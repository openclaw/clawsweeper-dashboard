## Review Findings

Overall confidence: 0.9

Full review comments:

- **[P1] Base wrapping on ingress provider:** `src/auto-reply/reply/get-reply-run.ts:270-274`
  - body: `OriginatingChannel` is reply-routing metadata, not ingress trust metadata. Current heartbeat/cron/exec-event runs can carry `OriginatingChannel` for delivery while `Provider` remains the trusted system-event source, so this wraps trusted scheduler prompts as external channel text and can distort scheduled actions. Use the ingress provider/event kind for the trust decision and keep route metadata separate.
  - confidence: 0.9
- **[P1] Preserve scheduler providers as trusted:** `src/auto-reply/reply/inbound-text.ts:7-18`
  - body: `isExternalChannelProvider` treats every provider except `node`, `cli`, and `exec-event` as external, but current main uses `heartbeat` and `cron-event` as internal providers. When those system-event runs have no `OriginatingChannel`, this patch fences their trusted automation prompts and can degrade reminder/heartbeat behavior; centralize or extend the trusted system-event provider check and cover it with tests.
  - confidence: 0.88
