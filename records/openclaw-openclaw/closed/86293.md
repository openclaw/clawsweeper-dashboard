## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve metadata for replacement managers during close:** `extensions/memory-core/src/memory/manager.ts:1093-1097`
  - body: `closeIdleManagedCacheEntries` removes the cache slot before awaiting `entry.close()`, so a concurrent `MemoryIndexManager.get()` can create and use a replacement under the same key while the old manager is still closing. This `finally` then unconditionally deletes `lastAccessAt`/`inflightCount` for that key; if the replacement finished its operation before the old close completes, its watcher can remain cached but invisible to future idle sweeps, bringing back the daemon-lifetime leak this PR is meant to bound. Only clear these maps when the key still belongs to the same closing instance, or otherwise preserve replacement metadata.
  - confidence: 0.86
