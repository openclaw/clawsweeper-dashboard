## What This Changes

The PR adds `src/shared/fs-atomic-fix.ts`, a new synchronous atomic text writer helper that catches `chmodSync` failures before renaming a temp file.

## Best Possible Solution

Keep the fs-safe-backed secrets writer from current main and treat later auth-profile read-only cascade reports, such as https://github.com/openclaw/openclaw/issues/62099, as separate work.

## Reproduction Assessment

Yes for the PR defect: source inspection shows the new helper has no callers and the PR head leaves the active `writeTextFileAtomic` implementation unchanged. No live Windows/Docker mount was needed to prove that merging this branch would not affect the reported runtime path.

## Solution Assessment

Yes for closing, no for merging this diff. The current fs-safe-backed implementation is the narrower maintainable fix because it changes the active secrets writer and centralizes best-effort chmod behavior instead of adding an orphan helper.
