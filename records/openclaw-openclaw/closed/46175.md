## Summary

Current `main` already implements this Anthropic signed-thinking replay fix in a stricter form: replay sanitization enables immutable-thinking handling only for Anthropic-compatible replay, preserves signed-thinking assistant turns only when every sibling tool call is already replay-safe, and otherwise drops the whole assistant turn instead of mutating provider-owned content. Current tests cover the exact unsafe cases from this PR, including disallowed tool names and `sessions_spawn` attachment exposure, and the fix is documented as shipped in `v2026.4.12`.
