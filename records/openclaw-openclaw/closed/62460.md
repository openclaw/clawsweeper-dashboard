## Summary

Close #62460 as superseded by open #63626. Current main still lacks the requested `memory.qmd.update.embedOnBoot` behavior, but #63626 is a later, broader same-author tracker for the same QMD embed-disable problem: `embedInterval` only disables the periodic timer, while forced/first-run update paths still run `qmd embed`. Keeping #63626 open as the canonical issue is the better cleanup path.

## Best Possible Solution

Close #62460 as superseded by open #63626. Keep #63626 as the canonical maintainer-facing issue to design and implement a documented QMD embed-disable control that covers boot, first-run, forced/manual sync, and periodic embed paths while preserving `qmd update`/BM25 refresh behavior.
