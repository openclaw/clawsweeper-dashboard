## Summary

Current main no longer matches the reported failure mode where config validation errors repeatedly occur during agent hot paths and runs finish silently with no model call. Runtime config is pinned after the first successful load, invalid config reads fail closed or surface through startup/recovery, and the v2026.3.31 changelog records the runtime config pinning that stops hot paths from reparsing openclaw.json between watcher swaps.

## Best Possible Solution

Keep the shipped runtime-config snapshot/fail-closed implementation. If a current release still has a specific AJV stack overflow with a concrete config, it should be reopened as a fresh bug with the minimized config schema or plugin/channel entry that still overflows on current main.
