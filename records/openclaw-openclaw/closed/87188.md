## Summary

Current main and the latest release already satisfy the supported OpenClaw install path: community ClawHub/npm/local plugin installs persist `plugins.entries.<id>.enabled=true`, which satisfies skill `requires.config` gates.

## What This Changes

The issue asks OpenClaw to make community plugin installs activate plugin-shipped skills gated on `plugins.entries.<id>.enabled` when the manifest opts into default enablement.

## Best Possible Solution

Keep the shipped OpenClaw installer behavior; if a separate ClawHub-side installer bypasses OpenClaw config mutation, track that in the ClawHub/install-tool surface with exact command evidence.

## Reproduction Assessment

No for current main: the supported `openclaw plugins install` paths already persist `plugins.entries.<id>.enabled:true`, which satisfies the reported skill gate.

## Solution Assessment

Yes. The current install-persistence path is the narrow maintainable solution; changing default activation semantics for arbitrary community manifests would be a broader policy change and is not needed for OpenClaw-managed installs.
