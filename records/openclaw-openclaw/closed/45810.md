## Summary

Current `main` already documents and enforces `NO_REPLY` for cron delivery suppression. The cron docs now state that isolated runs returning only `NO_REPLY`/`no_reply` suppress both direct outbound delivery and the fallback queued summary, and the current cron delivery code and tests match that behavior.
