## Summary

Current `main` still drops top-level `browser.ssrfPolicy` before starting the sandbox browser bridge, so issue #45153 is not implemented or obsolete yet.

## Evidence

- **Sandbox bridge config still omits `ssrfPolicy`:** `buildSandboxBrowserResolvedConfig` returns a `ResolvedBrowserConfig` with no `ssrfPolicy` field, and `ensureSandboxBrowser` passes only `controlPort`, `cdpPort`, `headless`, and `evaluateEnabled` into that builder before calling `startBrowserBridgeServer`.
