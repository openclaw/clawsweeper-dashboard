## Summary

Current `main` no longer reproduces the reported UI regression. The model picker is fed by `models.list`, and that path now includes the configured default model and fallbacks even when `agents.defaults.models` omits them; the picker also preserves the current override and default locally. The remaining mismatch is wording in `models status`, which still prints only the explicit allowlist under `Configured models`.

## Evidence

- **Docs define `agents.defaults.models` as the picker allowlist:** The CLI docs explicitly describe `agents.defaults.models` as the allowlist used for `/model` and the model picker, so the `Configured models` line is about the explicit catalog/allowlist, not a separate runtime registry.
