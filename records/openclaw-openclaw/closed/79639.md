## What This Changes

The issue requests restoring agent workspace image/PDF sends through message actions or MEDIA attachments instead of limiting outbound local files to managed media roots.

## Best Possible Solution

Keep the merged gateway dispatch fix and direct affected users to a build containing v2026.5.10-beta.3 or later rather than changing trustedLocalFileRoots.

## Reproduction Assessment

Yes. Source and merged PR evidence show the pre-fix gateway path forwarded agentId without mediaLocalRoots while Telegram media sends require those roots; current main has a focused regression test for that path.

## Solution Assessment

Yes. Passing getAgentScopedMediaLocalRoots at gateway message-action dispatch is the narrowest maintainable fix because it reuses the existing media-root contract and leaves plugin-owned send behavior unchanged.
