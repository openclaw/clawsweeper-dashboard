## Summary

Current `main` and the latest shipped release already return startup-gated `chat.history` errors with `retryable: true`, and the Control UI is served by that same gateway build, so the missing-`retryable` path in this PR does not match current or shipped behavior.

## Evidence

- **Gateway startup contract on current main:** Startup-gated gateway methods return `UNAVAILABLE` with `retryable: true`, `retryAfterMs: 500`, and `details.method`, which is the exact contract the UI retry loop expects.
