## Summary

Keep open. Current main still lacks a user-configurable Undici TCP connect timeout for Ollama/model-provider requests, and the assistant failover policy still has explicit timeout paths that can surface an error instead of entering the model fallback chain depending on runtime state.

## Best Possible Solution

Keep the issue open. A good fix would add a typed, documented model-provider or global network connect timeout field, validate it in config/schema/help, propagate it into Undici Agent connect.timeout for global and guarded provider dispatchers, and add regression coverage for Ollama remote-host connect timeouts. The fallback side should either make timeout failover deterministic for configured fallback chains or document and test the exact surface_error conditions such as external abort, compaction timeout, missing fallback config, and timeout-payload synthesis.
