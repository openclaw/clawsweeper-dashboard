## Summary

Keep open: current main still reuses a host-built `skillsSnapshot` after switching non-rw sandbox runs to the sandbox workspace, so the reported host-only skill path can still be exposed. The issue is already paired with open PR #38789 as the focused implementation candidate.

## What This Changes

Sandboxed non-main sessions should expose sandbox-readable skill paths instead of host-only skill paths in the skills prompt.

## Best Possible Solution

Review and land #38789 or an equivalent root fix that rebuilds or remaps skills prompts from sandbox-readable entries for non-rw sandbox attempt and compaction runs while preserving normal snapshot reuse elsewhere.

## Reproduction Assessment

Yes, as a source-level reproduction. A host-built `skillsSnapshot` with `resolvedSkills` can enter a non-rw sandbox run; current main then switches to the sandbox workspace, skips live sandbox skill loading, and returns the stored host prompt unchanged.

## Solution Assessment

Yes. The narrow maintainable fix is to refresh or remap the skills prompt at the sandbox runtime boundary; prompt wording alone, such as the related #74161 hardening, cannot repair a stale host path already stored in the snapshot.
