## Summary

Current main already preserves runtime `extraSystemPrompt` when `systemPromptOverride` is configured, with focused regression coverage and merged fix provenance from the earlier maintainer PR; the reported 2026.5.5 installed-dist behavior is obsolete on main and in the beta release that contains the fix.

## What This Changes

The issue requests preserving generated subagent extra system context when a spawned target agent also has a `systemPromptOverride`.

## Best Possible Solution

Keep the merged prompt-composition fix from https://github.com/openclaw/openclaw/pull/78356; broader CLI or compaction override follow-ups can continue separately without keeping this native subagent bug open.

## Reproduction Assessment

No as a current-main failure: the historical bug was source-reproducible before the fix, but current main composes the override with runtime extra context and has a regression test for the subagent task marker.

## Solution Assessment

Yes. The merged fix is the narrow maintainable solution for this issue because it preserves custom override prompts while appending generated subagent context in the same attempt prompt builder used by native `sessions_spawn`.
