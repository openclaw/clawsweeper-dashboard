## Summary

Close as implemented on main. The report is against OpenClaw 2026.4.2, and current main already contains the v2026.4.25 fix from #71933/#71864: parent `--agent` is forwarded to `models auth` write commands, `models auth login` resolves the requested configured agent directory, and returned OAuth profiles are written to that agent store instead of the default agent.

## Best Possible Solution

Close this bug as implemented and point affected users on OpenClaw 2026.4.2 to upgrade to v2026.4.25 or newer. Keep #40402 and #43681 open as separate multi-profile/profile-rotation feature requests; they are not needed for the fixed `--agent` write-target behavior.
