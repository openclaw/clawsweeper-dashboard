## Summary

Close as superseded. Current main already solves the user-visible WebChat leak with a non-empty internal reset marker, and the merged #72863 follow-up explicitly replaced this PR’s empty-transcript approach because it can break fresh Responses runs that need a submitted input.

## What This Changes

The PR changes bare WebChat `/new` and `/reset` handling to keep startup instructions in the runtime prompt while using an empty or reset-tail transcript prompt, with tests and UI filtering for empty user transcript placeholders.

## Best Possible Solution

Keep the current main implementation: startup instructions remain runtime-only, reset notes remain visible when provided, and bare reset turns use a non-empty internal marker instead of an empty transcript prompt.

## Reproduction Assessment

Yes. The source path and current tests give a high-confidence reproduction/proof path: bare `/new` runs through `runPreparedReply`, keeps the startup prompt in the runtime body, and persists `[OpenClaw session new]` instead of the raw startup instruction.

## Solution Assessment

No, this PR is no longer the best way to solve the issue. The current marker approach is narrower and safer because it hides the startup instruction without creating the empty first-turn submission that #72863 fixed.
