## Summary

Current main already treats `agents.list[].subagents.allowAgents` as a supported per-agent config field and `sessions_spawn` reads it when enforcing cross-agent spawn policy, so the reported clobber/forbidden behavior from 2026.3.11 is obsolete on main.

## What This Changes

Preserve per-agent subagent allowlist config across Gateway restarts so `sessions_spawn` can spawn the configured agents.

## Best Possible Solution

Keep the current main implementation that models subagent permissions as first-class per-agent config and preserves source-shaped config writes.
