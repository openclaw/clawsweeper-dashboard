## Summary

This report matches the already-open Control UI CSP fix in #71886: current main documents and implements browser-side tweakcn fetching, but the served CSP does not allow `https://tweakcn.com`. The canonical PR is open and already reviewed with the remaining merge blockers, so this issue should close as superseded rather than create a second fix thread.

## What This Changes

The issue requests that Appearance theme imports from tweakcn URLs fetch and apply valid theme payloads instead of failing with a browser NetworkError.

## Best Possible Solution

Use #71886 as the canonical fix path: merge a rebased CSP change that allows both `https://api.openai.com` and `https://tweakcn.com`, updates CSP assertions, and adds the required changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows the browser importer fetches `https://tweakcn.com/r/themes/<id>` while the served Control UI CSP omits `https://tweakcn.com`, which is enough to reproduce the browser-side NetworkError path without a live UI run.

## Solution Assessment

Yes, with the existing PR review constraints. The narrow maintainable fix is to allow the exact tweakcn HTTPS origin in `connect-src` while preserving the existing OpenAI origin; a server-side proxy is not needed for valid tweakcn registry payloads that already return permissive CORS headers.
