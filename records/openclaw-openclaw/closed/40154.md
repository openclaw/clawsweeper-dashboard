## Summary

Current main already normalizes `~`-based workspace paths before skill discovery/root checks, and only compacts skill locations back to `~/...` for prompt display after canonical absolute paths have been resolved. The same implementation is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Evidence

- **Agent workspace paths are expanded before use:** `resolveAgentWorkspaceDir()` passes configured and default workspace values through `resolveUserPath()`, so `~/.openclaw/workspace` becomes an absolute path before any skill loading path checks run.
  - file: [src/agents/agent-scope-config.ts:149](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/agent-scope-config.ts#L149)

  - file: [src/utils.ts:125](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/utils.ts#L125)
