## Summary

Close as implemented: the report is against openclaw 2026.4.25, while current main and shipped v2026.4.27 strip stale trailing assistant replay turns before Anthropic provider submission. The existing regression test covers the provider-facing assistant-tail shape, and the related #72226 cleanup report confirms the same systemEvent/heartbeat failure mode is covered.

## What This Changes

Prevent persisted heartbeat/session histories that end with an assistant turn from causing Anthropic request-format failures and silent fallback on the next request.

## Best Possible Solution

Keep the shipped provider-boundary sanitizer and have affected users upgrade from 2026.4.25 to v2026.4.27 or later; any remaining post-upgrade Anthropic fallback should be filed with fresh logs and the exact provider payload path.

## Reproduction Assessment

Yes. The issue gives a deterministic persisted heartbeat/session path on 2026.4.25, and current regression coverage reproduces the provider-facing shape by sending Anthropic replay history ending in assistant text through the stream wrapper.

## Solution Assessment

Yes. Repairing at the provider replay boundary is narrower and more complete than a heartbeat-only synthetic user turn because it covers heartbeat, systemEvent, context-engine, and other callers that can reach the same invalid Anthropic request shape while preserving trailing assistant tool-use turns.
