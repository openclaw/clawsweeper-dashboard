## Summary

Keep open: current main has retention-based unreferenced-artifact pruning, but it still lacks the requested explicit audited orphan-transcript archive/prune mode with per-file metadata, hashes, manifest/quarantine support, and deliberate destructive semantics.

## What This Changes

The issue requests a native `openclaw sessions cleanup` mode to audit, archive, or prune transcript files that exist on disk but are absent from `sessions.json`.

## Best Possible Solution

Define an explicit audited `sessions cleanup` orphan-transcript submode with per-file metadata and hashes, manifest-backed quarantine/archive, and clear confirmation rules for destructive prune.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, and related-thread checks give a high-confidence verification path that the requested audited archive/prune mode is still absent on current main.

## Solution Assessment

Unclear until maintainers choose the UX and safety boundary. Extending `sessions cleanup` is plausible, but the current retention prune and doctor interactive archive paths do not satisfy the requested audit/archive contract.
