## Summary

Close as implemented on main. Current macOS node/companion code already includes automatic WebSocket reconnect handling after gateway disconnects/restarts, and the same implementation is present in the latest shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Evidence

- **Gateway channel auto-reconnects after socket failures:** `GatewayChannelActor` handles receive failures by marking the channel disconnected and calling `scheduleReconnect()`, which retries with backoff. It also runs a watchdog loop that periodically reconnects while disconnected.
  - file: [apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayChannel.swift:267](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayChannel.swift#L267)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Reconnect logic is explicit on receive failure:** The receive-failure path calls `disconnectHandler`, fails pending requests, and schedules reconnect immediately.
  - file: [apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayChannel.swift:724](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayChannel.swift#L724)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Node session is designed to survive reconnects:** `GatewayNodeSession` keeps the underlying channel, resets connection state on disconnect, and re-emits connected state after a fresh snapshot arrives post-reconnect.
  - file: [apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayNodeSession.swift:219](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/apps/shared/OpenClawKit/Sources/OpenClawKit/GatewayNodeSession.swift#L219)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **macOS companion runs a persistent connection loop:** `MacNodeModeCoordinator` continuously calls `session.connect(...)` inside a retry loop with exponential backoff, so the macOS node keeps trying to reconnect without requiring a machine reboot.
  - file: [apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift:32](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/apps/macos/Sources/OpenClaw/NodeMode/MacNodeModeCoordinator.swift#L32)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Automated test covers reconnect after receive failure:** `GatewayNodeSessionTests.emitsSyntheticSeqGapAfterReconnectSnapshot` simulates a socket receive failure, then asserts a second socket is created (`snapshotMakeCount() >= 2`), which is direct proof of reconnect behavior.
  - file: [apps/shared/OpenClawKit/Tests/OpenClawKitTests/GatewayNodeSessionTests.swift:523](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/apps/shared/OpenClawKit/Tests/OpenClawKitTests/GatewayNodeSessionTests.swift#L523)
