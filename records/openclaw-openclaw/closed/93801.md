The issue requests a per-task external model-router command in agent dispatch with fail-open fallback to the configured primary model.

## Best Possible Solution

Keep the core hook seam as-is and route external policy-command routers to a ClawHub plugin built on `before_model_resolve`; add a core `modelRouter` config only if maintainers later choose that product surface.
