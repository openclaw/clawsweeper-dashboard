## Summary

Keep open. Current main intentionally treats `qwen/qwen3.6-plus` as unavailable on Qwen Coding Plan endpoints, but the reporter's residual complaint is still valid: the Qwen plugin defines a descriptive Coding Plan error, while the final unknown-model error path can still drop the config/baseUrl needed to surface that message and fall back to plain `Unknown model`.

## Best Possible Solution

Keep #64238 open until the Qwen Coding Plan error path is fixed. The narrow code fix likely belongs in `src/agents/pi-embedded-runner/model.ts`: preserve/pass `cfg` and `providerConfig?.baseUrl` into `buildSuppressedBuiltInModelError`, then add a regression test proving `qwen/qwen3.6-plus` on `coding.dashscope.aliyuncs.com` surfaces the Qwen-specific Standard-endpoint guidance. Keep #63654/#63987 as the broader canonical thread for changing whether Coding Plan should allow the model at all.
