## Summary

Current `main` already hardens supervised gateway restarts against the reported port-release race: managed service runs mark themselves as supervised, clear stale gateway listeners before rebinding, wait for the port to be released, and retry transient `EADDRINUSE` binds. The report's requested `ThrottleInterval` bump is no longer required for the failure mode described.
