## Summary

Keep open. Current main still has a source-level path where a configured Feishu channel can be emitted as a plugin auto-enable candidate and appended to a restrictive `plugins.allow` even when no installed/discovered Feishu plugin manifest claims it, which matches the recurring missing-plugin warning.

## What This Changes

Stop startup auto-enable from persisting a catalog-only configured Feishu channel into `plugins.allow` and triggering stale plugin warnings.

## Best Possible Solution

Gate channel-derived writes to `plugins.allow` on an installed or discovered plugin manifest owner, while still allowlisting a real `@openclaw/feishu` plugin when it is present.

## Reproduction Assessment

Yes, at source level. A config with `channels.feishu` and a restrictive `plugins.allow` can emit `feishu` as a `channel-configured` auto-enable candidate, append it to the allowlist, and then hit the existing missing-plugin validation warning when no Feishu manifest is installed.

## Solution Assessment

Yes for a narrow code fix, but not for the literal “never add Feishu” framing. The maintainable solution is to skip allowlist mutation for catalog-only or unclaimed channel ids, while preserving allowlisting for a real installed Feishu plugin manifest.
