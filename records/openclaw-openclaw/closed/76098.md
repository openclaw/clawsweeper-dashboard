## Summary

Close as not actionable in this repo: the reported WeChat/iLink media bundling behavior belongs to the external `@tencent-weixin/openclaw-weixin` runtime, while current OpenClaw core only projects the media facts handed to it by a channel plugin.

## What This Changes

A user reports that openclaw-weixin sends an unrelated old image along with a single WeChat camera photo, causing the agent to receive two images in one message.

## Best Possible Solution

Track and fix the stale or over-broad media selection in `@tencent-weixin/openclaw-weixin`, keeping OpenClaw core unchanged unless plugin maintainers identify a missing generic channel API.

## Reproduction Assessment

Yes, the report gives a concrete reproduction path: OpenClaw 2026.4.22 with openclaw-weixin/iLink, send one WeChat camera photo, then inspect the session record and Gateway logs for two images under one message id. It is not locally executable from this source checkout because the WeChat bridge implementation is external.

## Solution Assessment

Yes for repo cleanup: moving this to the external plugin is the narrowest maintainable resolution. A core change would be premature because the inspected core path only preserves the channel-supplied media array.
