## Summary

Current main and the latest stable release already defer non-interactive update-mode Linux systemd service rewrites, so the reported update-triggered service rewrite is fixed.

## What This Changes

The issue reports that `openclaw update` runs doctor in update mode and silently rewrites gateway systemd service config when entrypoint drift is detected.

## Best Possible Solution

Keep the shipped update-mode systemd deferral and use `openclaw gateway install --force` only when an operator explicitly wants OpenClaw to replace the service definition.

## Reproduction Assessment

Yes, by source inspection of v2026.4.22: update-mode doctor reached `confirmAutoFix` and `service.stage` for entrypoint drift. Current main has a focused regression test proving that Linux update-mode repair now reports drift without staging or installing the service.

## Solution Assessment

Yes. The current fix is the narrow path: defer only non-forced Linux systemd update-mode service rewrites while leaving explicit `openclaw gateway install --force` available for intentional replacement.
