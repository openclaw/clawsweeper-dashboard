## Summary

Keep open. Current main still has the reported Telegram preflight path where the voice note transcript reaches the agent but the echo delivery path is skipped, and an open PR (#73866) is already the active implementation candidate.

## What This Changes

Restore Telegram `tools.media.audio.echoTranscript` so preflight-transcribed voice-note transcripts are sent back to the originating chat before agent processing.

## Best Possible Solution

Review and land PR #73866, or an equivalent small fix, so Telegram passes the raw preflight transcript through a generic media-understanding echo path and adds a regression test proving both framed agent context and outbound echo delivery happen once.

## Reproduction Assessment

Yes. The source path is enough to reproduce at high confidence: Telegram preflight frames the transcript and marks the media index transcribed, then shared audio selection skips the attachment and the only echo call remains behind the new-audio-output branch.

## Solution Assessment

Unclear until PR #73866 is reviewed, but its stated direction is the right narrow boundary: reuse the existing echo helper from the media-understanding pipeline with a carried preflight transcript instead of rerunning STT or importing private core behavior into Telegram code.
