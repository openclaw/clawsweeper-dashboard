## Summary

Current main now avoids the reported Feishu onboarding trap: Feishu setup defaults to manual credentials, the QR scan path remains optional, and shared terminal QR rendering no longer uses compact mode by default.

## What This Changes

The issue reports that Feishu QR-code onboarding in `openclaw onboard` shows a QR code that the Feishu app does not scan on Windows 10.

## Best Possible Solution

Keep the current main behavior: manual Feishu setup as the default, optional QR scan-to-create as best-effort, and full-block terminal QR rendering by default.

## Reproduction Assessment

No high-confidence current-main reproduction remains: the earlier compact/default QR path is gone from source. The original end-to-end symptom still needs Windows 10 plus a Feishu mobile client to test live.

## Solution Assessment

Yes. The current combination of manual-default Feishu setup plus full-block terminal QR rendering is a narrow maintainable fix without adding a config knob or moving channel policy into core.
