## Summary

Close as implemented: current main replaces the old pre-model-only watchdog with execution milestone tracking and forwards runner progress from the affected Codex, CLI, and Pi paths, with regression coverage for the 60s false-timeout case. The fix is on current main only; it is not in the latest stable release I found.

## What This Changes

The issue reports isolated cron agent runs being killed by the 60s pre-model watchdog because active runner progress was not clearing the startup guard.

## Best Possible Solution

Keep the current execution-milestone implementation and ship it in the next release or beta; no separate repair branch is needed for this report.

## Reproduction Assessment

No for current main: the reported beta path is credible, but current main clears the startup watchdog on explicit execution milestones and has regression coverage showing a run survives the 60s check after turn_accepted.

## Solution Assessment

Yes: tracking runner execution milestones and forwarding those phases is the narrow maintainable fix, avoiding a broad timeout policy or config workaround while preserving a guard for true pre-execution stalls.
