## Best Possible Solution

Merge only after the branch uses one shared ANSI stripping strategy, normalizes context before parsing, preserves raw export lines, restores parser regression coverage, and removes or separately justifies dependency metadata changes.

## Solution Assessment

No. Parser-boundary display normalization is the right place, but this patch should share ANSI handling, strip context before JSON parsing, restore existing parser coverage, and keep dependency metadata out of the UI fix.
