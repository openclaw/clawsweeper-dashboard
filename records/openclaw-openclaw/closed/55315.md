## Summary

Current main already implements the `sessions_send` inter-session delivery/persistence path and includes gateway/test coverage for it, so the reported 2026.3.23-2 behavior appears obsolete on newer code.

## Evidence

- **sessions_send injects inter-session message into target run:** The tool sends the requested `message` to the resolved target `sessionKey` via the gateway and marks it as `inputProvenance.kind = "inter_session"`, which is the expected path for transcripted cross-session delivery.
