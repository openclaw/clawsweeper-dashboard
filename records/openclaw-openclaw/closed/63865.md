## Summary

#63865 should close as a duplicate of the existing open pre-compaction memory-flush UI leak tracker #58956. Its underlying prompt-as-user-message/session-contamination concern is also already tracked by open #54408, and current main does not contain a fix that would make this implemented-on-main.

## Best Possible Solution

Close #63865 as a duplicate of #58956, with #54408 covering the underlying persisted-prompt contamination. Fix the remaining work in the canonical trackers by making memory flush truly internal or hidden from the user-visible transcript, then add regression coverage that chat.history and session JSONL do not expose or replace user-visible messages with the flush directive.
