Current main already aligns the plugin entry TypeScript type and strict Zod schema: `plugins.entries.<id>.hooks.allowConversationAccess` is accepted as an optional boolean, covered by schema tests, documented, and used by the runtime hook gate. The same schema/test/doc behavior is present in the latest release tag `v2026.4.24`, so the reported `2026.4.23` mismatch has already shipped fixed.

## Best Possible Solution
