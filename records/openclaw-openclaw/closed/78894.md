## What This Changes

The issue asks memory-core’s isolated dream-promotion cron to stop logging missing operator.admin during narrative session cleanup.

## Best Possible Solution

Keep the shipped owner-gated plugin cleanup fix; handle any remaining Dreaming session or trajectory residue under https://github.com/openclaw/openclaw/issues/77723 and the focused follow-up PR rather than reopening the missing-scope bug.

## Reproduction Assessment

Yes, historically: the old plugin fallback deletion path used a write-scoped synthetic client while sessions.delete requires operator.admin, matching multiple logs and prior reports. Against current main, the failing path is fixed by plugin owner metadata plus owner-scoped synthetic admin cleanup.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix because it preserves operator.admin for arbitrary session deletion while allowing a plugin to clean up only sessions it created.
