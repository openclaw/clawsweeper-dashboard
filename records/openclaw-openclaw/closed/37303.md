## Summary

Issue #37303 is already implemented on current `main` and in the shipped `v2026.4.22` release: Telegram/Slack/Discord onboarding status now uses read-only inspection paths that treat unresolved SecretRefs as `configured_unavailable` instead of eagerly resolving and throwing, and the related CLI/status fallback paths also resolve or degrade safely instead of crashing.

## Evidence

- **Telegram onboarding no longer eagerly resolves tokens in setup status:** `telegramSetupWizard.status.resolveConfigured` now checks `inspectTelegramAccount(...).configured` rather than resolving the runtime token during onboarding status refresh.
  - file: [extensions/telegram/src/setup-surface.ts:31](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/extensions/telegram/src/setup-surface.ts#L31)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Telegram read-only inspection treats unresolved SecretRefs as unavailable, not fatal:** `inspectTokenValue` returns `tokenStatus: "configured_unavailable"` for unresolved SecretRefs in read-only inspection paths instead of throwing.
  - file: [extensions/telegram/src/account-inspect.ts:67](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/extensions/telegram/src/account-inspect.ts#L67)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Slack onboarding status is inspect-based:** Slack setup status now uses `inspectSlackAccount(...).configured`, avoiding the old eager resolution path during onboarding status checks.
  - file: [extensions/slack/src/setup-core.ts:164](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/extensions/slack/src/setup-core.ts#L164)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Slack inspection marks SecretRef credentials as configured-but-unavailable:** `inspectSlackToken` maps configured SecretRef inputs to `status: "configured_unavailable"` for read-only/account-status flows.
  - file: [extensions/slack/src/account-inspect.ts:42](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/extensions/slack/src/account-inspect.ts#L42)
