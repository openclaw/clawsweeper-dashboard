The issue reports that context usage percentage can remain based on the previous smaller model window after switching to DeepSeek V4 Flash, especially when persisted DeepSeek V4 metadata still has `contextWindow: 200000` instead of 1M.

## Best Possible Solution

Review and land the linked narrow doctor/config migration that rewrites only known stale DeepSeek V4 200K generated metadata to 1M while preserving intentional user overrides.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a configured DeepSeek V4 Flash row with `contextWindow: 200000` is returned as the active limit, and the UI percentage is computed from that resolved limit.

## Solution Assessment

Yes. The best solution is a focused `openclaw doctor --fix` repair for known stale generated DeepSeek V4 metadata, not a broad runtime override that would overwrite intentional model context-window settings.
