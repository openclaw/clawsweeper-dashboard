## Summary

## What This Changes

The issue requests fixing native Windows Gateway startup failures caused by bundled plugin runtime-dependency installs hitting npm/pnpm EPERM and MAX_PATH limits in v2026.4.29.

## Best Possible Solution

Keep heavy optional channel dependencies in external plugins and keep Gateway startup free of package-manager dependency repair.

## Solution Assessment

Yes. Externalizing WhatsApp and removing bundled dependency staging from startup is a more maintainable fix than patching Windows rename behavior in the old staging path.
