## Summary

Keep open: current main has generic sessions, labels, thread-aware channel routing, and plugin session extensions, but it does not implement first-class skill-declared project context, project UI switching, project-scoped history, or cross-project lookup. This is a broad feature/API request that needs maintainer product design rather than cleanup closure.

## What This Changes

The issue requests first-class project/thread awareness for skills, including project metadata on sessions, Control UI display and switching, project-scoped history, and cross-project queries.

## Best Possible Solution

Define a generic project-context/session metadata contract, likely through plugin SDK/session projection plus Control UI and history support, so external project-management skills or plugins can implement workflows like Project Atlas without hardcoding a single product flow into core.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source and docs inspection show existing sessions, labels, thread-aware channel routing, and plugin extension state, but not the requested project-aware skill/session/UI/history contract.
