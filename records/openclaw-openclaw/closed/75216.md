I found no concrete missing lifecycle field or runtime hook that would make a rolling/prioritized FIFO policy require core work.

## What This Changes

The issue requests an optional rolling/prioritized session context policy as an alternative to the existing compaction behavior.

## Best Possible Solution

Keep core focused on the context-engine extension point and let a rolling/prioritized FIFO context policy ship as a ClawHub code plugin; open a narrower core API issue only if plugin implementation identifies a specific missing lifecycle value or runtime helper.
