## Summary

Current main and the shipped v2026.4.26 release already prevent a malformed plugin tool schema from crashing Anthropic runs. Plugin tool materialization now rejects tools without a `parameters` object with plugin diagnostics, Anthropic payload conversion also skips direct/custom tools without object parameters before reading `properties`, and regression coverage plus the release changelog explicitly cover the reported outage.

## Best Possible Solution

Close as shipped in v2026.4.26. Keep the current core behavior: malformed plugin tools are isolated with diagnostics, Anthropic conversion is defensive for direct/custom tool arrays, and plugin authors should use the documented `parameters` field. Any future aliasing, install-time gate, or richer status UX should be tracked as a separate narrower enhancement if maintainers want it.
