## Summary

Current main does not support plugin-defined fields directly under `plugins.entries.<id>`; that wrapper is intentionally limited to `enabled`, `hooks`, `subagent`, and `config`. Plugin manifest `configSchema` is merged and validated under `plugins.entries.<id>.config.*`, which matches the docs and tests, so the reported `qdrantUrl` rejection is consistent with the current contract rather than a reproducible main-branch bug.

## Evidence
