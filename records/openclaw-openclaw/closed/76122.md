## Summary

Keep open for maintainer handling: the patch changes the shared idle-timeout fallback for every provider, while current main routes slow local/self-hosted models through provider-scoped timeout configuration and the linked Ollama regression lacks end-to-end proof.

## What This Changes

## Best Possible Solution

Keep the shared fallback conservative unless maintainers explicitly choose a global behavior change; handle slow Ollama/local models through provider-scoped request timeout defaults, config guidance, tests, and changelog coverage.

Unclear: source inspection proves current main can use a 120-second implicit idle watchdog, but the linked report only gives version/model steps and no logs or deterministic transcript. The scoped provider-timeout path is covered by unit tests, not a live Ollama/qwen3.6 reproduction.
