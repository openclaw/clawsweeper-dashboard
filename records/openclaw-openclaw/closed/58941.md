## Summary

Current main and shipped release notes show the Discord exec-approval regression/migration gap has been addressed: native chat approvals now auto-enable when supported channels can infer approvers from owner config, and the Discord docs now spell out the exact config and fallback behavior.

## Evidence

- **Release note: native chat approvals auto-enable from owner config:** v2026.4.2 release notes state: "Exec approvals/channels: auto-enable DM-first native chat approvals when supported channels can infer approvers from existing owner config, while keeping channel fanout explicit and clarifying forwarding versus native approval client config."
  - command: `web search/open GitHub releases for v2026.4.2`
  - sha: [d74a122](https://github.com/openclaw/openclaw/commit/d74a122)
- **Release note: Discord native approval disabled/timeout states fixed:** v2026.4.1 release notes state that Slack and Discord native approval handling was aligned with inferred approvers and real channel enablement so remote exec stops falling into false approval timeouts and disabled states.
  - command: `web search/open GitHub releases for v2026.4.1`
  - sha: [da64a97](https://github.com/openclaw/openclaw/commit/da64a97)
- **Current Discord docs document auto mode and migration boundary:** The Discord channel docs on main document `channels.discord.execApprovals.*`, auto-enabling when `enabled` is unset or `"auto"`, fallback from explicit `execApprovals.approvers` or `commands.ownerAllowFrom`, and explicitly say Discord does not infer approvers from channel `allowFrom`, legacy `dm.allowFrom`, or direct-message `defaultTo`.
  - file: [docs/channels/discord.md:923](https://github.com/openclaw/openclaw/blob/2e0ae56b1a580103f71b9b7f79b3d638e3c3698d/docs/channels/discord.md#L923)

- **Current configuration reference documents default auto behavior:** The public configuration reference documents `channels.discord.execApprovals.enabled` as `true | false | "auto"` with auto mode activating when approvers resolve from `approvers` or `commands.ownerAllowFrom`.
  - file: [docs/gateway/configuration-reference.md:512](https://github.com/openclaw/openclaw/blob/2e0ae56b1a580103f71b9b7f79b3d638e3c3698d/docs/gateway/configuration-reference.md#L512)
