## Summary

Current main and the latest available release contain the plugin-based web_search resolution that fixes the reported missing-key behavior for Brave and Perplexity. The original reporter confirmed the Docker env fallback worked in v2026.4.1; current code still resolves provider env vars and the providers themselves fall back to BRAVE_API_KEY, PERPLEXITY_API_KEY, and OPENROUTER_API_KEY before returning missing-key errors.

## Best Possible Solution

Close this issue as implemented. Keep the current plugin-owned Brave/Perplexity env fallback and runtime provider-selection behavior; handle any remaining observability improvements or a fresh v2026.4.5+ reproduction as separate, narrower work with exact config and debug logs.
