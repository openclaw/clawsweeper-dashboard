## Summary

Current main still trusts raw audio transcription output for Telegram voice notes. The Telegram inbound path assigns the preflight transcript directly to `bodyText` and then to `BodyForAgent`, and the shared audio transcription path still copies transcript text into message bodies without any suspicious-transcript or subtitle-credit filter. I found no implementation on `main` that guards known Whisper hallucination strings like subtitle/watermark credits.
