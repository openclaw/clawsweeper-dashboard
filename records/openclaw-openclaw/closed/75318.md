## Summary

Close as a duplicate of #73525. This report describes the same still-open regression: default `openclaw status` can render an empty Channels table while the live/deep status paths show configured channel accounts, and the canonical issue already retains the code discussion plus open fix PR #73662.

## What This Changes

The issue requests that default `openclaw status` show configured channels in its Channels table when the same Telegram channel is visible in `openclaw status --deep`.

## Best Possible Solution

Keep #73525 as the canonical open tracker and fix the behavior there by making default `openclaw status` populate the Channels table from the reachable gateway account snapshot or equivalent read-only metadata, while preserving the fast fallback for unreachable gateways and adding focused regression coverage.

## Reproduction Assessment

Yes. The issue gives concrete steps and two independent environments, and current source/test inspection shows the default path skips the live/deep channel data path that would surface the Telegram account.
