## Summary

Current main already addresses the reported Control UI freeze on loading historical chat/session data: chat.history is limited to recent messages, text fields are truncated, oversized nested payloads are replaced with a placeholder, final JSON payload size is enforced, and the UI only renders the last 200 history messages. The shipped v2026.4.24 tag contains the same bounded chat.history behavior, and current tests cover large history payloads and oversized nested messages.

## Best Possible Solution

Keep the shipped bounded chat.history implementation, UI render windowing, and documented Control UI history contract. If a current-release user can still reproduce a freeze, the best follow-up is a new issue with the offending transcript shape or sanitized logs so maintainers can add a narrower regression test.
