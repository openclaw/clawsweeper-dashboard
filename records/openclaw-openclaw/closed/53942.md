## Summary

Close as duplicate. The same remaining product decision is already tracked by #25493, which asks to make fixed OpenClaw-owned system prompt sections editable; current main still lacks section-level user overrides, and the prior implementation attempt for this exact area was closed unmerged.

## What This Changes

The issue requests a config or trust-level mechanism for overriding selected OpenClaw-owned system-prompt sections such as Safety, Tool Call Style, and self-update guidance while preserving safer defaults.

## Best Possible Solution

Keep the remaining design and implementation discussion on #25493. If maintainers approve the capability, add a schema-backed or workspace-file-backed section override model with explicit safety docs and prompt regression tests; otherwise document whole-prompt replacement, hooks, and provider overlays as the supported customization paths.
