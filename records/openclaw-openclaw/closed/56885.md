## Summary

Current `main` already exposes public plugin tool hooks (`before_tool_call` and `after_tool_call`) with typed registration, error reporting, and latency context, so the issue's core claim that OpenClaw lacks granular tool-execution hooks no longer matches the codebase. Skills are documented as instructional `SKILL.md` content used for prompt building/discovery rather than a separate runtime execution primitive.

## Evidence

- **Public plugin hook API documents tool hooks:** `api.on(...)` is the typed lifecycle hook surface, and the SDK docs explicitly list `before_tool_call` semantics in the public registration API.
  - file: [docs/plugins/sdk-overview.md:225](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/docs/plugins/sdk-overview.md#L225)

## Risks / Open Questions
