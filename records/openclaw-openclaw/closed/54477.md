## Summary

Current main already implements this via the optional `active-memory` plugin: recall runs from a `before_prompt_build` hook on eligible user turns, builds its query from the latest prompt plus recent turns, and injects the recalled context before the main reply. The feature shipped in `v2026.4.12`.

## Risks / Open Questions

- This is implemented through the optional `active-memory` plugin, so the operator must enable that plugin for the target agent.
- Default `allowedChatTypes` is `direct`; Feishu group chats require explicit opt-in if that was the intended surface.
