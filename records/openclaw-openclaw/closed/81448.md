## What This Changes

The issue reports that an installed and enabled `openclaw-weixin` plugin is visible to CLI commands but omitted from Gateway startup on OpenClaw 2026.5.7 when configured with `channels.openclaw-weixin.enabled true`.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/81323 and https://github.com/openclaw/openclaw/pull/81331 as the canonical fix path, carry over this issue's Gateway-skip evidence, and land a shared channel-presence regression that makes enabled-only external channel config load at Gateway startup without Weixin-specific special cases.

## Reproduction Assessment

Yes, at source level: current main ignores `{ enabled: true }` in `src/channels/config-presence.ts`, while Gateway startup depends on that configured-channel list to select external channel plugins. I did not live-run the packaged Weixin plugin in this read-only review.

## Solution Assessment
