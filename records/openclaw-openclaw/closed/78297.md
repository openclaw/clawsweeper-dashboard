## Summary

Close as a duplicate of #78296: the reported cron failures are a downstream effect of the same agents.defaults.models allowlist shrink, while current cron behavior intentionally fails closed with explicit diagnostics rather than silently falling back.

## What This Changes

The issue asks cron jobs whose configured models disappear from agents.defaults.models to keep running or fail visibly instead of producing cryptic allowlist rejection errors.

## Best Possible Solution

Use #78296 as the canonical source fix for preserving agents.defaults.models; keep cron’s explicit fail-closed validation unless maintainers choose a separate fallback-policy change.

## Reproduction Assessment

Yes, at source level: an isolated cron agentTurn with payload.model outside agents.defaults.models reaches resolveAllowedModelRef and returns a cron-preflight error. I did not run a live cron job because this review is read-only and live cron would write run artifacts.

## Solution Assessment

Yes for cleanup: closing this satellite in favor of #78296 is the narrowest maintainable path because fixing the model-tree shrink removes the observed cron failures without changing cron’s documented fail-closed model policy.
