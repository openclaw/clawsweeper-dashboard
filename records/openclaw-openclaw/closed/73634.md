## Summary

This issue is a duplicate of the already-open #47311, which tracks the same remaining WebChat server-side STT / MediaRecorder-to-Gateway voice input work. Current main still lacks the requested recorder/mic dictation button; it only exposes Talk mode plus generic non-video attachments, so the right cleanup is to keep #47311 as the canonical implementation thread.

## Best Possible Solution

Close this issue as a duplicate and keep #47311 as the canonical thread. The best implementation belongs there: add a MediaRecorder capture UI/state to WebChat, validate and upload browser audio through an authenticated Gateway path, run the configured tools.media.audio/local STT pipeline, return the transcript to the composer or send path, and update Control UI docs/tests to distinguish browser dictation, server-side STT, generic audio attachments, and Talk mode.
