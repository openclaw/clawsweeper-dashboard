## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Handle wrapper-backed gateway service commands:** `src/cli/update-cli/update-command.ts:986`
  - body: `findGatewayServiceEntrypoint` assumes the token before `gateway` is the dist entrypoint, but supported wrapper installs use `[wrapperPath, "gateway", ...]`. That makes the guard compare the wrapper path to `dist/index.js` and allows a same-install gateway to keep running during the git update.
  - confidence: 0.9
- **[P1] Restart the gateway after failed or skipped git updates:** `src/cli/update-cli/update-command.ts:3356-3358`
  - body: This stops the gateway before `runGitUpdate`, but the later `error` and `skipped` paths only recover `prePackageServiceStop`, which is package-update state. A dirty checkout, failed fetch, or failed rebase can therefore leave the previously running gateway down even though no update was applied.
  - confidence: 0.88
- **[P2] Remove restart from the no-restart unblock guidance:** `src/cli/update-cli/update-command.ts:3344-3349`
  - body: In the `--no-restart` block, telling users to run `openclaw gateway restart` before rerunning the update leaves the gateway running, so the same preflight blocks again. The guidance should say to stop first, update, then restart after the update if desired.
  - confidence: 0.86
