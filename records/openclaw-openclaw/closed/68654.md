## Summary

Close: current main already shipped the central lazy CLI plugin-loading reduction, and the remaining branch-only config snapshot reuse would regress finalized runtime config semantics.

## What This Changes

The PR adds metadata-only plugin CLI entry loading, routes lazy core-command registration through it, reuses the validated config snapshot, and adds focused plugin CLI tests.

## Best Possible Solution

Keep the shipped command-owner based lazy-loading behavior from https://github.com/openclaw/openclaw/pull/76379 and leave validated plugin CLI config loading on finalized runtime config instead of merging snapshot.config reuse.

## Reproduction Assessment

Yes. Source inspection reproduces the branch regression: PR head returns snapshot.config from validated config loading, then the plugin runtime context treats that supplied config as rawConfig and skips getRuntimeConfig finalization.

## Solution Assessment

No for the PR branch. The maintainable solution is the already-shipped current-main command-owner policy plus finalized runtime config, not the branch's snapshot.config shortcut.
