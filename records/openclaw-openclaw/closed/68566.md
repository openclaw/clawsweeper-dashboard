## What This Changes

The PR skips auth-profile cooldown mutation for `format` failures and any HTTP 400 by adding status plumbing, direct `markAuthProfileFailure` guards, unit tests, and a changelog entry.

## Best Possible Solution

Keep the merged resolver-level approach: filter only confirmed request-shape `format` failures before shared auth cooldown, while preserving cooldown or disable behavior for classified auth, billing, rate-limit, and model failures.

## Reproduction Assessment

Yes, at source level. The diff would skip `markAuthProfileFailure` for a classified `billing` or `rate_limit` failure if a caller also forwarded raw HTTP 400, while current main preserves those classifications and only filters confirmed `format` failures.

## Solution Assessment

No. The branch is too broad; the merged resolver-level fix in https://github.com/openclaw/openclaw/pull/77280 is the narrower maintainable solution because it stops `format` cooldown cascades without treating every raw 400 as harmless.

  - confidence: 0.78
