## What This Changes

The issue reports that concurrent native Codex app-server turns for two agents can close another agent's in-flight client, causing `<turn_aborted>` and a stuck session until gateway restart.

## Best Possible Solution

Review and land the narrow keyed shared-client pool in https://github.com/openclaw/openclaw/pull/79513 under the canonical tracking issue https://github.com/openclaw/openclaw/issues/79495, preserving this report as additional reproduction evidence.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to start overlapping native Codex app-server turns for two agents with different `agentDir` values; current main computes different keys and clears the single shared client slot.

## Solution Assessment

Yes. Pooling shared clients by the existing compatibility key is the narrow maintainable fix because it preserves per-agent auth and `CODEX_HOME` isolation while preventing unrelated agent eviction; broader session-isolation work can remain separate.
