Full review comments:

- **[P2] Inspect every find start path before allowing workspace exemption:** `src/agents/bash-tools.exec.ts:344-345`
  - body: `find` can traverse multiple roots, but this guard records only the first path-like token. A command such as `find . / -type f` from a workspace resolves the first root inside `workdir` and returns uncapped even though it still walks `/`, which defeats the stated broad-root guard.
  - confidence: 0.9
- **[P2] Handle brace-expanded HOME roots:** `src/agents/bash-tools.exec.ts:286-289`
  - body: The resolver handles `$HOME` and `$HOME/...` but not `${HOME}` or `${HOME}/...`. Bash expands those to the home directory, so `find ${HOME} ...` is treated as a relative literal and skips the 20-second cap.
  - confidence: 0.88
