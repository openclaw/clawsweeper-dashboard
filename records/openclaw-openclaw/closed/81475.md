## What This Changes

Report that isolated cron jobs using `sessionTarget: isolated` and `payload.kind: agentTurn` fail around 60 seconds with a false `attempt-dispatch` stall before the first model call.

## Best Possible Solution

Yes, by source inspection and canonical live repro: isolated `agentTurn` cron runs with a configured timeout can remain at `attempt_dispatch`, which current `main` does not treat as execution-started, so the 60-second pre-execution watchdog can abort before the configured timeout.
