## Summary

Current main already implements the requested native `sessions_spawn` behavior: the task is rendered under `## Your Role` in the child system prompt and the first user message only points the child back to that system section. The same source is present in the latest v2026.5.2 tag, so the reported observation most likely came from a different runtime path or external bridge behavior rather than the default native subagent implementation.

## What This Changes

Ensure `sessions_spawn.task` is injected into the child agent system prompt `Your Role` section instead of being delivered only as an initial user message.

## Best Possible Solution

Keep the shipped native `sessions_spawn` implementation, and track any ACP or external bridge task-delivery issue separately with explicit runtime and prompt evidence.

## Reproduction Assessment

No. Current main and the v2026.5.2 tag show the native `sessions_spawn` path injecting `task` into `extraSystemPrompt` under `## Your Role`, while the first user message does not contain the task text.

## Solution Assessment

Yes for the native subagent path. The current implementation is the narrow maintainable solution because it single-sources the task in the system prompt and avoids duplicating long task text in the first user message.
