## Summary

This PR is superseded by current main. The proposed launchctl fallback extends the old service-env proxy persistence path, but current main intentionally removed proxy variables from generated gateway/node service environments and added doctor audit/docs/tests that treat embedded proxy values as something to clean up, not add more of. The prior review comments about lowercase/uppercase proxy precedence are still useful context, and current main avoids that specific conflict by not persisting either casing into supervisor metadata.

## Best Possible Solution

Close this stale PR as superseded by the main-line service environment policy. Keep the shipped behavior that avoids persisting ambient proxy variables in supervisor metadata, and track any future macOS service proxy support as a new explicit runtime/managed-env design if maintainers want it.
