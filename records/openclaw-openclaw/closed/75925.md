## Summary

Current main already solves the LM Studio JIT preload problem through the supported `models.providers.lmstudio.params.preload: false` setting, with source, regression test, docs, changelog, and provenance evidence. The PR’s proposed `skipModelLoad` key is no longer the path to merge.

## What This Changes

The PR adds a `models.providers.lmstudio.skipModelLoad` flag, gates LM Studio inference preload and embedding warmup on it, and wires config schema/help plus one stream regression test.

## Best Possible Solution

Keep the current `params.preload: false` LM Studio option as the supported surface and close this PR as superseded by the main implementation.

## Reproduction Assessment

Yes. The linked bug has concrete LM Studio v0.4.12 JIT/TTL model-switching steps, and current main has a focused regression test showing the preload call is skipped when the provider parameter is disabled.

## Solution Assessment

Yes. The current-main provider-local `params.preload` guard is the narrower maintainable solution because it avoids adding a second LM Studio-specific top-level config key while preserving default preload behavior.
