Keep open: current main still writes the same REM h3 body under both inline and standalone outputs, while this PR fixes the standalone report heading shape without reintroducing the inline collision flagged in review.

## What This Changes

This PR keeps REM preview body headings at h3 for inline daily notes and lowers them to h2 only when writing standalone REM reports, with focused memory-core tests.

## Best Possible Solution

Land the writer-boundary REM heading normalization with the focused regression tests; maintainer-owned changelog handling can happen during landing if needed.

## Solution Assessment

Yes. Adjusting heading depth only for REM standalone report lines is the narrowest maintainable fix because it preserves the inline preview contract while correcting the report wrapper output.
