## Summary

## What This Changes

The branch adds a native resolver for `openclaw/plugin-sdk/*` and `@openclaw/plugin-sdk/*` during plugin module loading, plus resolver regression tests and a changelog entry.

## Best Possible Solution

Scope native SDK alias targets to their registered OpenClaw/package root, add a multi-root regression test, and include redacted real external-plugin runtime proof before merge.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only pass. Source inspection supports the native `createRequire()` gap because current main only aliases the Jiti/plugin-loader path and still relies on a dist alias tree writer.

## Solution Assessment

No, not as-is. A guarded native resolver is a plausible fix direction, but the implementation should keep alias targets scoped to the parent root and needs real external-runtime proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Scope SDK aliases by registered root:** `src/plugins/plugin-sdk-native-resolver.ts:139-140`
  - body: `pluginSdkNativeAliases` is one global map keyed only by specifier, while `allowedParentRoots` keeps accumulating every plugin root. If one process loads plugins from two OpenClaw package roots, the later install overwrites targets like `openclaw/plugin-sdk/channel-message` for all earlier allowed roots, so earlier plugins can resolve against the wrong dist files. Store aliases per registered root, or bind each allowed parent root to the target map created for that root.
  - confidence: 0.88
