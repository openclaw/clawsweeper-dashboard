## Summary

Keep open. Current main does not already contain the OpenShell platform deployment guide, and the existing OpenShell docs cover a different host-managed sandbox-backend flow. The PR is useful in scope, but its current head still has blocking docs/security issues around proxy bypass guidance, process-wide WebSocket monkey-patching, duplicate glossary entries, and advanced binary replacement instructions, so there is no conservative auto-close reason.

## What This Changes

The PR adds an OpenShell sandbox deployment guide, registers it in the Platforms docs navigation, and adds OpenShell-related zh-CN glossary entries.

## Best Possible Solution

Keep the contribution under maintainer docs/security review, then either merge a dedicated `platforms/openshell` guide after rewriting the proxy sections around `proxy.enabled` / `OPENCLAW_PROXY_URL` and current Slack proxy support, or fold the durable operator guidance into the canonical OpenShell docs. The final version should preserve strict SSRF boundaries, avoid destination bypass env guidance, scope advanced supervisor replacement to trusted test builds with rollback framing, and deduplicate the glossary entries.

## Reproduction Assessment

Not applicable; this is a docs PR rather than a runtime bug report. The high-confidence review path is source/diff inspection: current main lacks the platform page, while current main docs/source already define managed proxy routing and Slack proxy-agent support that the PR has not fully aligned with.

## Solution Assessment

No; the current PR is not the best way to solve the documentation gap as-is. A safer approach is to document the supported proxy lifecycle and current Slack WebSocket proxy behavior first, then clearly label any remaining OpenShell-specific workarounds as temporary, scoped, and security-sensitive.

## Security Review

Status: needs_attention

Summary: The diff is docs-only, but it introduces security-sensitive operator guidance for proxy bypasses, process-wide preload hooks, and runtime binary replacement.

Concerns:

- **[medium] NO_PROXY bypass weakens proxy egress boundary:** `docs/platforms/openshell.md:140`
  - body: The launcher example exports `NO_PROXY` for loopback and private ranges even though current managed proxy mode clears those bypass variables to prevent destinations from skipping the filtering proxy. Operators following the guide could unintentionally weaken the intended egress boundary.
  - confidence: 0.86
- **[medium] Custom preload and module monkey-patch guidance:** `docs/platforms/openshell.md:328`
  - body: The guide instructs operators to use `NODE_OPTIONS` to preload JavaScript that installs a global proxy dispatcher and monkey-patches `Module._load` for `ws`. That is process-wide code-execution and egress-routing guidance, and it conflicts with OpenClaw's supported proxy lifecycle.
  - confidence: 0.88
- **[low] Supervisor binary replacement instructions:** `docs/platforms/openshell.md:744`
  - body: The advanced section documents replacing the OpenShell supervisor binary inside the cluster container with `docker cp` and `kill -9`. That can be useful for upstream testing, but it normalizes swapping a trusted runtime binary outside a release process without enough trust, source, or rollback framing.
  - confidence: 0.78
