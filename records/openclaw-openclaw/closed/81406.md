## Summary

This PR should close because the useful defaults-inheritance part is already covered by a narrower active fix path, while this branch also re-enables legacy per-agent/default runtime pins in a way that conflicts with current main’s documented runtime policy.

## What This Changes

The branch adds `agents.list[].agentRuntime` and `agents.defaults.agentRuntime` to model runtime policy resolution, extends runtime source types, and updates harness tests to expect those legacy keys to select runtimes.

## Best Possible Solution

Keep runtime selection provider/model-scoped unless maintainers explicitly choose a narrower defaults-inheritance change, and route that discussion through https://github.com/openclaw/openclaw/pull/81421.

## Reproduction Assessment

Unclear as a bug: source and tests show current main ignores whole-agent runtime keys, but docs and doctor rules define that as expected behavior. This PR does not include live subagent or billing-lockout reproduction, only targeted test claims.

## Solution Assessment

No. This branch is not the best fix because it revives legacy per-agent/default runtime pins and places them before provider/model policy; the safer path is a narrower defaults-only policy decision with docs, doctor, and tests aligned.
