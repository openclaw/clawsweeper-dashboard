## Summary

Keep this PR open. Current main still does not classify `application/msword` as binary in media-understanding, so the legacy `.doc` false-positive path described by the PR remains plausible. The PR is narrow and security-safe, but its patch is stale against current main and needs a rebase/manual test placement before review or landing.

## Best Possible Solution

Keep this PR open. The best path is to rebase it onto current main, keep the minimal `application/msword` addition in `isBinaryMediaMime`, place the regression test near the current binary/PDF media-understanding tests, then run the relevant changed/test lane before merge. If the branch remains stale, a maintainer could apply the same two-file fix directly.
