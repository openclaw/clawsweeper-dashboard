## Summary

Current `main` already uses `session.totalTokens` for the WebChat/Control UI context warning, and the shipped `v2026.4.22` release contains the same implementation plus a regression test that matches the reported bad numbers.

## Evidence

- **Context warning now uses current prompt tokens:** The Control UI context notice reads `session.totalTokens` and compares that against `contextTokens`, so it no longer bases the warning on accumulated `inputTokens`.

## Close Comment

Closing from a Codex review because current `main` already implements this fix, and it is shipped in `v2026.4.22`. In `ui/src/ui/chat/context-notice.ts:56`, the Control UI warning now uses `session.totalTokens` instead of accumulated `inputTokens`, which matches the TUI-side notion of current context usage. There is also a regression test in `ui/src/ui/chat/context-notice.test.ts:32` that covers the reported failure mode directly: `inputTokens: 757_300` with `totalTokens: 46_000` no longer shows the warning, while `190_000 / 200_000` renders `95% context used` and not `757.3k / 200k`. If you still see this on `v2026.4.22` or later, please open a new issue with current screenshots and exact reproduction steps.
