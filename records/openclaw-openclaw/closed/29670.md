## Summary

Current main still accepts Tailscale Serve identity as Control UI WebSocket authentication, but it does not use that authenticated Tailscale operator path to skip first-time Control UI pairing. The issue also has recent maintainer guidance explicitly saying to keep it open until a narrower fix with regression tests lands.

## Best Possible Solution

Keep #29670 open. The landable fix belongs in the gateway WebSocket path: add a narrowly scoped Tailscale-authenticated Control UI operator pairing bypass alongside the trusted-proxy bypass, keep device identity enforcement for device-less clients, pass the Tailscale auth state through message-handler/connect-policy, and add regression tests for the positive device-present Tailscale path plus the negative missing-device path. Reconcile the public Control UI/Tailscale docs after the behavior lands.
