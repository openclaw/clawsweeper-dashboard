## Summary

Keep this open. Current main still allows the macOS direct config writer to replace the live config before treating `gateway-mode-removed` as an audit anomaly, while open PR #75815 is the focused fix candidate for this exact path.

## What This Changes

The issue reports macOS/Dashboard config writes dropping required config envelope fields such as `meta` or `gateway.mode`.

## Best Possible Solution

Land PR #75815 or an equivalent macOS-side pre-commit guard that rejects destructive `gateway.mode`/envelope removal, preserves required fields, records rejected payloads, and makes direct-save callers handle rejection.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: start with an existing config containing `gateway.mode`, then invoke the Swift direct `saveDict` path with a root payload whose `gateway` object omits `mode`; current main writes before auditing `gateway-mode-removed`.

## Solution Assessment

Yes. The narrow maintainable fix is to harden the macOS direct writer before atomic replacement and propagate rejection to callers, matching the existing TypeScript config writer behavior; a broader migration to patch-only writes can remain follow-up work.
