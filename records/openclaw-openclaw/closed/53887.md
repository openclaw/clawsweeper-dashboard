## Summary

This is a useful Feishu/Lark channel enhancement, but it does not need OpenClaw core tracking. Current main does not ship Feishu lifecycle status reactions; however, the generic status-reaction controller is already exposed through the plugin SDK, Feishu already has reaction add/remove/list primitives and typing-reaction plumbing, and Feishu is packaged as the @openclaw/feishu plugin with ClawHub/npm publishing metadata. The best path is a scoped Feishu plugin/ClawHub enhancement rather than a core issue.

## Best Possible Solution

Close this core issue and route the work to the Feishu plugin/ClawHub path: implement a Feishu StatusReactionAdapter in @openclaw/feishu that maps lifecycle states to FeishuEmoji values, uses the existing add/remove/list reaction wrappers, preserves the current Feishu backoff handling, and documents the plugin behavior.
