## Summary

Current `main` already keys the Docker dependency-install layer on root manifest inputs, so dependency changes invalidate the cached install layer. The same implementation is present in release `v2026.4.22`, with regression coverage in `src/docker-build-cache.test.ts`.

## Close Comment

Closing from a Codex review: current OpenClaw already invalidates the Docker dependency-install layer when root dependency manifests change. In `Dockerfile:64-75`, the build copies `package.json`, `pnpm-lock.yaml`, `pnpm-workspace.yaml`, and `.npmrc` before `pnpm install --frozen-lockfile`, so a `package.json` dependency change busts that layer. `src/docker-build-cache.test.ts:16-25` also keeps a regression check on that layout, and release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`) already contains the same behavior. One note: the report mentions `package-lock.json`, but this repo uses `pnpm-lock.yaml`.
