## Summary

Keep open with changes requested: the latest PR head still drops adjacent validation keywords when a local `$ref` is resolved, which weakens schemas before tool-argument validation. The `maintainer` label also makes cleanup closure inappropriate.

## What This Changes

The PR adds local `$ref` inlining to tool parameter schema normalization, applies it when materializing bundled MCP tools, adds Notion-style regression coverage, and updates the changelog.

## Best Possible Solution

Keep the shared normalizer and bundled MCP materialization boundary, but compose resolved refs with adjacent validation siblings and cover that behavior with a focused regression test.

## Solution Assessment

No. Normalizing local refs at the shared tool-schema boundary is the right shape, but the resolver must preserve or compose adjacent validation siblings rather than returning only the referenced schema plus metadata.
