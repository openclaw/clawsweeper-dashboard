## Summary

Current `main` already contains a Discord gateway teardown path that keeps the gateway error listener attached through `disconnect()`, classifies late reconnect-exhausted events, and suppresses them during teardown/dispose instead of letting them surface as uncaught exceptions. The same implementation is present in the latest release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
