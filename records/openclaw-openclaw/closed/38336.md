## Summary

Issue #38336 does not match current OpenClaw main: the repo now documents and implements a single per-agent `auth-profiles.json` auth store, and the reported `auth-profiles.provisioned.json` / `safe-mode-recovery.sh` path is not present in this checkout.

## Evidence

- **OAuth storage docs name only one primary store:** Current docs describe `auth-profiles.json` as the per-agent token sink and only mention legacy `credentials/oauth.json` as an import source; there is no documented `auth-profiles.provisioned.json` golden copy.
  - file: [docs/concepts/oauth.md:43](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/docs/concepts/oauth.md#L43)
  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)
- **Gateway config reference matches single-store model:** The configuration reference says per-agent profiles are stored at `<agentDir>/auth-profiles.json` and does not describe any provisioned mirror file.
  - file: [docs/gateway/configuration-reference.md:706](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/docs/gateway/configuration-reference.md#L706)
  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)
- **Onboarding writes OAuth credentials into the agent auth store:** `writeOAuthCredentials()` builds an OAuth credential and persists it through `upsertAuthProfile()` for the resolved agent dir; this path writes the current auth store, not a separate provisioned mirror.
  - file: [src/plugins/provider-auth-helpers.ts:269](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/src/plugins/provider-auth-helpers.ts#L269)
  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)
- **Auth profile writes persist through the canonical store helper:** `upsertAuthProfile()` updates the in-memory store and calls `saveAuthProfileStore()`, which writes `auth-profiles.json` plus auth state for that agent.
  - file: [src/agents/auth-profiles/profiles.ts:46](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/src/agents/auth-profiles/profiles.ts#L46)
  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)
- **OAuth refreshes are mirrored into the main agent store:** Current refresh logic explicitly mirrors refreshed OAuth credentials into the main agent store when a non-main agent refreshes, which addresses stale-main-store drift within the present architecture.
  - file: [src/agents/auth-profiles/oauth-manager.ts:297](https://github.com/openclaw/openclaw/blob/76a4c167f7ef65040fd5b00d206c59b8bb28fd49/src/agents/auth-profiles/oauth-manager.ts#L297)
  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)
- **Repo search found no reported provisioned/recovery artifacts:** A repo-wide search for `auth-profiles.provisioned.json`, `safe-mode-recovery.sh`, `build-full-config.sh`, `try-full-config.sh`, and `/var/lib/init-status/safe-mode` returned no matches in the checkout.
  - command: `rg -n "auth-profiles\.provisioned\.json|provisioned auth|golden copy|safe-mode-recovery|build-full-config\.sh|try-full-config\.sh|/var/lib/init-status/safe-mode" -S .`
  - sha: [76a4c167f7ef](https://github.com/openclaw/openclaw/commit/76a4c167f7ef65040fd5b00d206c59b8bb28fd49)

## Risks / Open Questions

- An external installer, host script, or out-of-repo deployment tool could still maintain its own `auth-profiles.provisioned.json` mirror, but that behavior is not present or verifiable in `openclaw/openclaw` current main.
