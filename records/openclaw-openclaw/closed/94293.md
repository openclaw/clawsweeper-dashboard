## Summary

Close as a duplicate of the existing gateway channel lifecycle tracker: the reported failure is real on current main and in the latest release, but the same stale task/no-op health-monitor recovery class is already being tracked with broader live evidence and active related fix discussion.

## What This Changes

The issue reports that the channel health monitor skips stale task cleanup when a channel status is already stopped, leaving a wedged account unrecoverable until gateway restart.
