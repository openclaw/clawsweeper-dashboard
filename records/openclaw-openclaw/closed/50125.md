## Summary

Current main already supports the central vault/exec auth-profile use case through `auth-profiles.json` `keyRef`/`tokenRef` SecretRefs. The exact proposed `auth.profiles.<id>.apiKey` field is not the current config contract; `auth.profiles` remains profile metadata, while static credentials live in the auth profile store and can be SecretRef-backed.
