## Summary

Close as implemented on current main: after v2026.6.1, commit 6a0fdea90ad053ef977214b4cadcf9898c100b3c removed suffix-only Feishu streaming card updates and now sends cumulative content, so the reported overwrite regression is fixed on main but not yet in the latest release.

## What This Changes

Feishu streaming cards in v2026.6.1 were reported to show only the latest streamed characters instead of cumulative card text.

## Best Possible Solution

Keep the current-main cumulative Feishu streaming-card update behavior and ship it in the next release; users on v2026.6.1 can keep using the documented non-streaming workaround until that release is available.

## Reproduction Assessment

Yes for the reported release/source path: v2026.6.1 sent suffix-only payloads on Feishu streaming-card updates, matching the overwrite-visible symptom. I did not reproduce it against current main because current main no longer has that suffix path.

## Solution Assessment

Yes. The current-main fix is the narrow owner-boundary solution because it changes the Feishu streaming session to send cumulative content, while leaving dispatcher callers and user configuration unchanged.
