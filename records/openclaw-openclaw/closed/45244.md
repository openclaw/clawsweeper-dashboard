## Summary

Current `main` no longer documents `/tts always|inbound|tagged` as valid slash-command actions. The implementation and docs agree that `/tts` supports `on|off|status|provider|limit|summary|audio|help`, while `always|inbound|tagged` are configuration values for `messages.tts.auto`. The documentation mismatch described in the issue is therefore obsolete on current `main`.
