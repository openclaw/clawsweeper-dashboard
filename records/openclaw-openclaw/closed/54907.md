## Summary

Current `main` already routes `openclaw browser storage local|session get/set/clear` through the selected `--browser-profile`, and the browser route layer resolves `query.profile` before falling back to defaults. This behavior is present in the code shipped by `v2026.4.22`.

  - file: [extensions/browser/src/cli/browser-cli.ts:24](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/extensions/browser/src/cli/browser-cli.ts#L24)

## Close Comment

Closing as implemented on current `main` after a Codex review. The current browser CLI defines `--browser-profile` on `openclaw browser` (`extensions/browser/src/cli/browser-cli.ts:24`), the storage `get/set/clear` commands forward that selected profile in their request query/body path (`extensions/browser/src/cli/browser-cli-state.cookies-storage.ts:140`), and the browser route layer resolves `req.query.profile` before falling back to the default profile (`extensions/browser/src/browser/routes/utils.ts:23`). There is also regression coverage for `--browser-profile` parsing in `extensions/browser/src/cli/browser-cli.test.ts:27`, and the behavior is documented in `docs/cli/browser.md:24`. This shipped in `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
