## Summary

Current main already supports the requested tree-structured memory organization through the bundled `memory-wiki` plugin, while intentionally keeping `memory-core` daily capture in `memory/YYYY-MM-DD.md` files.

## Evidence

- **Default memory is intentionally split into durable root + daily notes:** The memory overview documents `MEMORY.md` for long-term memory and `memory/YYYY-MM-DD.md` for daily notes, and explicitly directs users who want a maintained knowledge base to the bundled `memory-wiki` plugin.
