## Summary

Current main implements the reported fix. Bare `/new` and `/reset` no longer produce an empty transcript prompt, and the `openclaw.runtime-context` custom entry is stripped before messages are converted for the provider boundary, so it should not serialize into an empty OpenAI Responses/Codex user input. This also addresses the related runtime-context custom-message path previously tracked in #72386.

## Best Possible Solution

Close this issue as implemented on main. Keep the current design: bare `/new` and `/reset` submit a non-empty session marker, runtime context remains hidden/prompt-local, and `openclaw.runtime-context` custom entries are stripped before provider conversion. Ship this with the next tagged release if it has not already been released.
