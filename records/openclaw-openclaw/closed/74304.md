## Summary

This issue reports a real current-main gap, but the same automatic daily/idle reset hook work is already tracked by open issue #11960 and active implementation PRs #61675 and #71792. Current main still does not fire `before_reset` or the bundled `session-memory` hook on lazy daily rollover, so this should close only as duplicate/superseded, not as implemented.

## What This Changes

Request to fire the plugin `before_reset` hook and preserve session memory during automatic daily session resets, with automatic-reset context for hook consumers.

## Best Possible Solution

Close this duplicate and consolidate discussion on #11960 plus the active implementation PRs #61675 and #71792. The eventual fix should make automatic daily and idle stale rollovers invoke the same effective reset persistence surface as manual resets, pass the previous session entry/transcript to session-memory, fire `before_reset` if that remains the plugin contract, and preserve existing `session_end`/`session_start` lifecycle behavior.
