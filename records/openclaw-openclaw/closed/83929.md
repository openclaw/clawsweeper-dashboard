## Summary

Current main has a post-v2026.5.18 final-delivery routing refresh that covers the reported transcript-only final-reply path by retrying stale/no-target session routing against fresh session state before delivery; the fix is on main only and is not contained in the v2026.5.18 release tag.

## What This Changes

The issue reports that Discord group/channel turns on v2026.5.18 can record final assistant replies in the transcript without visibly posting them when automatic group replies are enabled.

## Best Possible Solution

Keep the current-main routing refresh and ship it in the next release; reopen or file a new report only if Discord still misses final replies on a build containing the fix SHA.

## Reproduction Assessment

No live high-confidence reproduction was run. The release/main source diff gives a source-level path: v2026.5.18 lacked fresh-session final routing, while current main retries from fresh session state and has a focused regression test.

## Solution Assessment

Yes. Refreshing persisted session routing immediately before final delivery is the narrowest maintainable fix because it keeps the existing durable delivery path instead of adding Discord-specific fallback delivery logic.
