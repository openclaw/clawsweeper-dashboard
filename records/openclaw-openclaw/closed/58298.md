## Summary

This issue should close as a duplicate/superseded item rather than as fully implemented-on-main. PR #72667 merged a real Feishu native-card payload path and explicitly listed this issue as covered, but current main still has a separate Feishu message-action path where `message(action=send)` presentation buttons are rendered as markdown fallback labels. The remaining agent-authored Feishu interactive button/card work is already tracked by the open canonical issue #13175; #69754 is a separate fallback-guidance issue.

## Best Possible Solution

Close this issue as covered by the open canonical Feishu interactive-buttons tracker #13175. The best implementation path is to finish Feishu-owned rendering for shared `presentation` on the message-action path, generate structured `ocf1` envelopes for value buttons, preserve safe fallbacks, and route callbacks through the existing Feishu card-action handler; keep #69754 separate for fallback-copy clarity.
