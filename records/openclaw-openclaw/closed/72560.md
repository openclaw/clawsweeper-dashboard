## Summary

Close as ClawHub/plugin scope. Current main does not ship the requested context-staleness classifier, but it now has generic plugin hooks that are sufficient for an external implementation to record the prior assistant output and short-circuit a later inbound turn with clarification or silence. That matches the VISION guidance to keep optional capability out of core when plugin surfaces are enough.

## What This Changes

The issue proposes an optional pre-turn context-staleness check that compares the prior assistant question, timestamps, and message intent before treating the newest user message as an answer.

## Best Possible Solution

Publish and maintain this as a ClawHub code plugin or skill bundle that reuses the linked heuristic module, records the last assistant question per session/conversation, and uses before_dispatch to clarify or ignore ambiguous delayed messages.

## Reproduction Assessment

Not applicable as a feature/API proposal rather than a concrete shipped regression. The verification path is source and docs inspection: current main has no built-in classifier, but it has the hooks needed for an external implementation.

## Solution Assessment

Yes for conservative repository cleanup. The narrow maintainable direction is an external plugin/skill using the current generic hooks, leaving core responsible for stable extension points rather than one heuristic policy.
