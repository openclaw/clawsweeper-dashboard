## Summary

Keep open. Current main still classifies `multiple_gateways` by the number of reachable probe targets, while SSH tunnel probing keeps a differently addressed configured remote URL, so the reported one-gateway/two-transport case can still get a misleading warning.

## What This Changes

The issue requests identity-aware `openclaw gateway probe` warning behavior so SSH tunnel and direct remote URL probes to the same logical gateway are not reported as multiple gateways.

## Best Possible Solution

Classify reachable probe targets by a conservative non-null gateway self identity, suppress or downgrade same-identity multi-transport results, and keep the existing warning for distinct or unidentifiable gateways.

## Reproduction Assessment

Yes, at source level. Construct two reachable `GatewayStatusProbedTarget` entries with the same non-null `self`; current main still emits `multiple_gateways` because it only checks `reachable.length > 1`.

## Solution Assessment

Yes. Identity-aware warning classification is the narrowest maintainable fix because it keeps the useful direct-path probe while avoiding a false multi-gateway warning; skipping the configured remote target would lose diagnostic coverage.
