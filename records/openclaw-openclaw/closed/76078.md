## Summary

Keep open. Current main still hides Z.AI GLM-5 `reasoning_content` as thinking, and this PR is a narrow fix that uses the disabled-thinking signal while preserving the enabled-thinking path; I found no blocking review findings.

## What This Changes

The PR updates OpenAI-compatible completion stream parsing to treat Z.AI disabled-thinking `reasoning_content` as visible text, adds regression tests, and records the fix in the changelog.

## Best Possible Solution

Land the scoped stream parser fix so disabled-thinking Z.AI GLM-5 responses reach users as visible text while preserved thinking stays hidden and replayable.

## Reproduction Assessment

Yes. On current main, a Z.AI OpenAI-completions stream chunk with empty `content` and nonempty `reasoning_content` is classified as a thinking block, and visible reply extraction only includes text blocks.

## Solution Assessment

Yes. The PR uses the already-mutated Z.AI request payload plus the existing compat signal, which is narrower and safer than treating every Z.AI `reasoning_content` field as visible text.
