## Summary

Current main already covers the reported MCP child-process leak path: stdio MCP transports are closed with process-tree termination, run-mode subagents and nested `sessions_send` retire bundled MCP runtimes by session key, and idle session-scoped runtimes have a shipped TTL sweep. The related prior cleanup for #70364 is reflected in the same current code path rather than leaving separate remaining work here.

## What This Changes

The issue requests gateway-owned MCP server child-process trees to be reaped when agent or subagent sessions end so filesystem/github MCP servers do not accumulate memory.

## Best Possible Solution

Keep the shipped MCP cleanup design: direct retirement for completed run-mode/nested work, process-tree kill on stdio transport close, and idle TTL cleanup for session-scoped runtimes. A new issue would only be needed if someone reproduces surviving MCP trees on v2026.4.27 or newer with session keys and timings showing they outlive completion plus the configured TTL.
