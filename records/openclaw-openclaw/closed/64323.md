## Summary

Close as `clawhub`: current `main` already has the needed core seams for isolated subagent runs, agent-specific prompts, and direct slash-command entrypoints, so a specialized `/rename` or `/extract` flow belongs in a plugin/skill package rather than new OpenClaw core routing.

## Evidence

- **Repo vision keeps optional capability in plugins/ClawHub:** `VISION.md` says core should stay lean, optional capability should usually ship as plugins, and new skills should go through ClawHub first rather than core.

## Risks / Open Questions

- This issue asks for a built-in alias-binding UX (`/rename`) rather than composing the existing plugin/skill seams; if maintainers want that ergonomics in core despite the plugin-first vision, this close would be too aggressive.
