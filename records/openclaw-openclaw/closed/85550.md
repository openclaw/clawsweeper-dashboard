## Summary

Keep open: the billing/session-isolation symptom is serious and freshly reported, while current source confirms trajectory sidecars can grow with compiled prompt context but does not prove they are read back into provider input.

## What This Changes

The issue reports that trajectory sidecar files may be reused as context across new `openclaw chat --session <uuid>` sessions, causing runaway input-token usage and billing.

## Best Possible Solution

Capture a Windows/Gemini request-payload or provider-token repro for fresh TUI sessions, then fix the exact runtime path: block sidecars from model context if proven, or adjust trajectory/session defaults, limits, and docs if the tokens come from normal compiled prompt/tool context.
