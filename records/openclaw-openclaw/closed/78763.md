## Summary

The current official package train already fixes the reported packaging defect: the old 2026.5.2-beta.2 tarball is source-only, but npm latest @openclaw/acpx@2026.5.6 ships dist output and runtimeExtensions, and the remaining warning is for stale local installs that need update or reinstall.

## What This Changes

The issue requests that the official @openclaw/acpx package ship compiled JavaScript runtime output instead of only TypeScript source so doctor and gateway startup stop warning.

## Best Possible Solution

Keep the fixed @openclaw/acpx package train and have users update or reinstall stale 2026.5.2-beta.2 installs rather than suppressing the scanner warning.

## Reproduction Assessment

Yes for the stale package path: @openclaw/acpx@2026.5.2-beta.2 has index.ts and no dist, and current scanner code warns for global TypeScript package entries with no built runtime candidate. The latest @openclaw/acpx@2026.5.6 package does not reproduce because it ships dist/index.js and runtimeExtensions.

## Solution Assessment

Yes; rebuilding and republishing the official package with dist plus runtimeExtensions is the narrowest maintainable fix, while keeping the scanner warning for stale source-only installs protects users from bad package artifacts.
