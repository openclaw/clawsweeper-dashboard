The PR changes `src/media/web-media.ts` and tests to allow host-read `.html` files only when they are generated under the resolved OpenClaw temp root, while rejecting outside-root, symlink, hardlink, non-HTML, and binary cases.

## Best Possible Solution

Keep the restricted generated-HTML behavior, but bind approval to the exact opened file or use a safe open/read flow that rejects path swaps before allowing `text/html`.

## Solution Assessment

No. The restricted temp-root HTML allowance is the right shape, but this implementation should bind trust to the opened file before it is the best security-boundary fix.
