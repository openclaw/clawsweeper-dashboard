## Summary

Current `main` treats new messaging channels as plugin work rather than core issue backlog. The repo vision, plugin docs, channel-plugin SDK docs, and existing external-channel precedent all point to XChat belonging in the ClawHub/plugin lane, not as a core OpenClaw issue.

## Evidence

- **Vision: optional capability should usually ship as plugins:** `VISION.md` says core stays lean, optional capability should usually ship as plugins, channels are a code-plugin use case, the bar for adding optional plugins to core is intentionally high, and plugin discovery/promotion belongs in ClawHub.

  - file: [docs/tools/plugin.md:11](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/docs/tools/plugin.md#L11)

- There is no linked external XChat plugin yet, so closing moves the idea to the plugin lane without a concrete implementation to hand off to.
