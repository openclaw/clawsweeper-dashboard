## Summary

Close as duplicate/superseded. The reported Telegram symptom maps to the same shared channel lifecycle bug already tracked by open issue #71412, and open PR #72912 is the active implementation candidate for the remaining stop-timeout recovery work.

## What This Changes

The issue asks for Telegram polling-stall and health-monitor recovery to start a fresh channel after stop() times out instead of leaving the channel wedged until a gateway restart.

## Best Possible Solution

Route the remaining shared lifecycle fix through #71412 and review/land #72912 or an equivalent forced-retirement implementation; preserve this report's Telegram logs as linked evidence, then split any post-fix Telegram transport failure into a narrower follow-up.
