## Summary

Current main preserves legacy Claude CLI runtime intent by backfilling provider/model-scoped `agentRuntime: { id: "claude-cli" }` entries for older OAuth configs, so the reported `pi` fallback and misleading billing error path is fixed on main but not yet in a tagged release.

## What This Changes

The issue asks OpenClaw to stop silently dropping legacy `agents.defaults.agentRuntime` configs during update and avoid falling back to the `pi` runtime with a misleading billing error.

## Best Possible Solution

Keep the current-main runtime backfill and let the next release carry it; `openclaw doctor --fix` should remain the persistent cleanup path for removing stale legacy keys.

## Reproduction Assessment

Yes. The v2026.5.12 source path only seeded empty Anthropic model entries for legacy Claude CLI selection, while current main adds the missing per-model runtime policy and matching regression coverage.

## Solution Assessment

Yes. The current fix keeps runtime selection provider/model-scoped while preserving older Claude CLI OAuth configs at runtime, which is narrower and safer than startup-time config rewriting.
