## Summary

Keep this PR open. Current main still has the explicit `openclaw crestodian` no-TTY path logging the error and returning normally, while the CLI wrapper only exits non-zero when the action throws. The PR remains a focused implementation candidate for the open bug it references, not obsolete cleanup.
