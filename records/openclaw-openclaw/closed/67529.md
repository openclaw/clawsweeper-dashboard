## Summary

Reporter already shipped VK as a working third-party channel plugin, and current OpenClaw `main` explicitly scopes optional channel integrations like this to community plugins/ClawHub rather than OpenClaw core.

## Evidence

- **Vision scope is plugin-first, not core-first, for optional channel capability:** `VISION.md` says core should stay lean, optional capability should usually ship as plugins, plugin authors should host and maintain plugins in their own repositories, the bar for adding optional plugins to core is intentionally high, and plugin promotion belongs in ClawHub.

- **Current docs define community channels/plugins as ClawHub-or-npm installs:** `docs/plugins/community.md` defines community plugins as third-party packages that can add channels, says ClawHub is the canonical discovery surface, and says plugin discoverability should happen via ClawHub instead of opening repo docs requests. It also states plugins should be publishable/installable via `openclaw plugins install <package-name>`.
  - file: [docs/plugins/community.md:9](https://github.com/openclaw/openclaw/blob/7e52223d327555c611e8c91ee2500897e9eab0bf/docs/plugins/community.md#L9)
