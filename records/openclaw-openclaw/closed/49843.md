## Summary

Current main already prevents the Control UI session selector from rendering duplicate session options by key, disambiguates duplicate-looking labels, and replaces existing session rows on live updates instead of accumulating them. The prior discussion correctly pointed at the UI/session-list boundary; that path is now covered on main and is listed in the 2026.3.22 changelog.
