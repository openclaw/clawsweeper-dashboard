## Summary

Current main already suppresses the extra `sessions_send` announce path for the parent->own background ACP child setup described in the issue, and that behavior is present in shipped release `v2026.4.22`.

## Evidence

- **Parent-owned ACP children are classified as background-only:** `resolveAcpSessionInteractionMode()` marks one-shot ACP sessions with `spawnedBy` or `parentSessionKey` as `parent-owned-background`, with comments stating they should report back through the parent task notifier instead of speaking on the user-facing channel.
