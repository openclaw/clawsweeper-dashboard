Full review comments:

- **[P1] Restrict prototype detection to mutations:** `src/security/skill-scanner.ts:198`
  - body: The new critical rule matches any bracket access, including read-only checks such as `if (obj["__proto__"])`, but critical code-safety findings block plugin/skill installs by default. Require a write or mutation shape before emitting this critical finding.
  - confidence: 0.9
- **[P2] Cover direct prototype-pollution writes:** `src/security/skill-scanner.ts:198`
  - body: The same rule only covers bracket notation and `constructor["prototype"]`, so direct writes like `obj.__proto__.polluted = true` or prototype mutation helpers bypass the new prototype-pollution coverage. Either include those mutation forms or narrow the rule name/tests to the limited bracket-only scope.
  - confidence: 0.86
- **[P2] Match credential paths on readFile arguments:** `src/security/skill-scanner.ts:226-227`
  - body: The credential rule checks for a credential path anywhere in the source plus any `readFile*` anywhere else, so a help string mentioning `.ssh` and an unrelated `readFileSync("./config.json")` still warns. Tie the credential path to the readFile/readFileSync argument or at least the same expression/line.
  - confidence: 0.9
- **[P2] Constrain listener context to Node server APIs:** `src/security/skill-scanner.ts:191-192`
  - body: The network-listener context matches tokens like `https` anywhere in the file, so a local `createServer()` helper can warn merely because the file also contains an HTTPS URL string. Require an actual http/net/tls import/require or a qualified server API call.
  - confidence: 0.85
- **[P3] Tie symlink warnings to fs calls:** `src/security/skill-scanner.ts:204-205`
  - body: The symlink rule also uses a source-wide `fs` context, so an unrelated `readlink()` helper can warn when the file mentions or imports fs elsewhere. Match `fs.readlink*`/`fs.symlink*` or a known imported binding rather than any same-file token.
  - confidence: 0.78
