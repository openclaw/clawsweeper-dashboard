## Summary

Current `main` already blocks the reported metadata-theme leakage in the dreaming pipeline: inbound sender/conversation metadata is stripped before session-corpus ingestion, chat scaffolding terms like `assistant` and `user` are stop words during concept-tag extraction, and REM reflection generation blacklists those tags before producing theme summaries.

## Evidence

- **Session ingestion strips inbound metadata before dreaming export:** `stripInboundMetadataForUserRole()` removes OpenClaw-injected inbound metadata for user-role messages before normalization, and `extractSessionText()` uses that sanitized text for session export consumed by dreaming ingestion.
  - file: [src/memory-host-sdk/host/session-files.ts:338](https://github.com/openclaw/openclaw/blob/24e9924d6abd93560fb7bf721909257de533e57a/src/memory-host-sdk/host/session-files.ts#L338)
  - sha: [24e9924d6abd](https://github.com/openclaw/openclaw/commit/24e9924d6abd93560fb7bf721909257de533e57a)
- **Regression test covers the reported sender/conversation envelope path:** The regression test builds a user message with `Conversation info (untrusted metadata)` and `Sender (untrusted metadata)` blocks and asserts the exported content keeps only the real user text, not `untrusted metadata`, `message_id`, or fenced JSON.
  - file: [src/memory-host-sdk/host/session-files.test.ts:179](https://github.com/openclaw/openclaw/blob/24e9924d6abd93560fb7bf721909257de533e57a/src/memory-host-sdk/host/session-files.test.ts#L179)
  - sha: [24e9924d6abd](https://github.com/openclaw/openclaw/commit/24e9924d6abd93560fb7bf721909257de533e57a)
- **Concept-tag extraction now drops chat scaffolding tokens:** Current `main` adds `assistant`, `user`, `system`, and `subagent` to concept-tag stop words, and the matching test verifies meaningful tags survive while scaffold words are excluded.
  - file: [extensions/memory-core/src/concept-vocabulary.ts:15](https://github.com/openclaw/openclaw/blob/24e9924d6abd93560fb7bf721909257de533e57a/extensions/memory-core/src/concept-vocabulary.ts#L15)
  - sha: [24e9924d6abd](https://github.com/openclaw/openclaw/commit/24e9924d6abd93560fb7bf721909257de533e57a)
- **REM reflection generation blacklists scaffold tags before theme summaries:** REM theme generation now skips blacklisted tags including `assistant`, `user`, `system`, `subagent`, and `the` before producing `Theme:` lines, preventing those labels from surfacing as recurring themes.
  - file: [extensions/memory-core/src/dreaming-phases.ts:194](https://github.com/openclaw/openclaw/blob/24e9924d6abd93560fb7bf721909257de533e57a/extensions/memory-core/src/dreaming-phases.ts#L194)
  - sha: [24e9924d6abd](https://github.com/openclaw/openclaw/commit/24e9924d6abd93560fb7bf721909257de533e57a)
- **Release comparison shows the main-only theme-filter additions:** Comparing `v2026.4.23` to current `main` shows the `concept-vocabulary.ts`, `concept-vocabulary.test.ts`, and `dreaming-phases.ts` changes that add the scaffold stop words and REM blacklist, so the issue is implemented on `main` but I could not verify a tagged release containing the full fix set from this checkout.
