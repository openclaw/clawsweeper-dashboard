## Summary

Keep open: current main avoids QMD semantic mode by default and can fall back after QMD search failures, but it still has no Alder Lake-N/native-Windows guard, no 0xC0000005-specific diagnostic, and no documented limitation for the reported node-llama-cpp crash path.

## What This Changes

The issue reports that QMD semantic memory paths crash on native Windows Alder Lake-N systems when node-llama-cpp loads GGUF models.

## Best Possible Solution

Keep QMD semantic modes from hard failing on affected native-Windows CPUs by adding owner-approved detection, diagnostics, fallback behavior, and documentation while coordinating any binary compatibility fix upstream.

## Reproduction Assessment

No, not with high confidence in this review: reproducing the access violation requires native Windows on Alder Lake-N hardware with QMD semantic mode. Source inspection does confirm the relevant QMD embed/query paths and the absence of hardware-specific handling on current main.

## Solution Assessment

Unclear. BM25 defaults and builtin fallback are useful mitigations, but the best durable fix requires an owner decision between upstream binary coordination, native-Windows diagnostics, forced fallback, source-build guidance, or docs-only mitigation.
