## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Scope idempotency per outbound send unit:** `src/infra/outbound/deliver.ts:256`
  - body: This adds one `idempotencyKey` to the shared outbound base context, but the delivery loop later calls `handler.sendText` once per chunk and `handler.sendMedia` once per attachment with the same context. Since Telegram caches by account/chat/idempotency key, the second and later sends in a long reply or media sequence can return the first result instead of sending, silently dropping content.
  - confidence: 0.92
- **[P2] Encode Telegram idempotency key components:** `extensions/telegram/src/send.ts:208`
  - body: The Telegram cache key is built by joining `accountId`, `chatId`, and `idempotencyKey` with `:`, but two of those values are strings that can contain that separator. Encode the components or use a tuple-safe key so distinct sends cannot collide and suppress each other.
  - confidence: 0.85
