## Summary

Current main solves the unsafe part of #70869: delivery-mirror transcript rows are still stored as user-visible transcript records, but they are explicitly filtered out of in-memory model replay before Anthropic/strict-provider requests are built, so they no longer create Anthropic alternation failures.

## Best Possible Solution

Keep the current transcript-only mirror plus replay filtering. It preserves user-visible channel history while preventing delivery-mirror rows from reaching Anthropic or other strict provider payloads. If an external context-engine or LCM plugin still ingests raw transcript rows unsafely, track that as a plugin/context-engine integration bug rather than reopening this provider alternation issue.
