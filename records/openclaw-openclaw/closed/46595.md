## Summary

Current main already resolves cross-agent `sessions_spawn` workspace inheritance from the target agent, persists the child workspace on the spawned session, and forwards that stored workspace into the child agent runtime so bootstrap files come from the effective spawned workspace.

## Evidence

- **Target agent workspace is now selected for cross-agent spawns:** `resolveSpawnedWorkspaceInheritance` prefers `targetAgentId` over the requester session agent before falling back, which addresses the issue's reported root cause.
