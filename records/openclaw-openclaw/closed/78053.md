## Summary

Close as implemented: current main and the v2026.5.4 release source keep volatile inbound IDs/timestamps out of the system prompt and prepend `Conversation info` to the latest user message, addressing the OpenAI-compatible cache-busting symptom reported against 2026.5.3.

## What This Changes

The issue requests moving volatile per-turn inbound metadata out of the system prompt so OpenAI-compatible local providers can reuse stable prompt prefixes.

## Best Possible Solution

Keep the current source behavior and point affected 2026.5.3 users at v2026.5.4 or newer for this specific inbound-metadata cache-stability fix.

## Reproduction Assessment

We do not have a current-main failing reproduction; source inspection gives a high-confidence path instead. The reported volatile `message_id` and `timestamp` fields are now built into the latest user-role context, while the system prompt builder excludes them.

## Solution Assessment

Yes. Moving volatile inbound metadata to the current user turn is the narrow maintainable fix for OpenAI-compatible prefix caches because it preserves message targeting context without changing the reusable system prompt bytes.
