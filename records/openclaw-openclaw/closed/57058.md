## Summary

Current `main` already clears cron running locks on timeout/error completion, auto-recovers stale `runningAtMs` markers during maintenance and manual rerun preflight, and clears/replays interrupted recurring jobs on startup. The indefinite `already-running` lock described in #57058 is covered by existing code and regression tests.

## Evidence

- **Completion clears the lock:** `applyJobResult()` unconditionally sets `job.state.runningAtMs = undefined` before recording final status/error metadata, so timeout and error completions do not leave the job marked running.
  - file: [src/cron/service/timer.ts:416](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/cron/service/timer.ts#L416)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Maintenance auto-unlocks stale markers:** Cron maintenance treats a `runningAtMs` older than the stuck-run threshold as stale and clears it with a warning.
  - file: [src/cron/service/jobs.ts:399](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/cron/service/jobs.ts#L399)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Manual rerun preflight repairs stale locks before `already-running`:** `cron run` preflight calls `recomputeNextRunsForMaintenance()` before checking `runningAtMs`, explicitly to avoid stale markers blocking manual triggers.
  - file: [src/cron/service/ops.ts:510](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/cron/service/ops.ts#L510)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Startup recovery clears stale markers:** Cron startup clears persisted `runningAtMs` markers and persists the repaired state before running restart catch-up.
  - file: [src/cron/service/ops.ts:95](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/cron/service/ops.ts#L95)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
- **Regression tests cover timeout and stale-lock recovery:** Existing regression coverage asserts that timed-out manual runs end with `runningAtMs` cleared, and that stale `runningAtMs` no longer blocks a manual rerun.
  - file: [src/cron/service/ops.regression.test.ts:225](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/cron/service/ops.regression.test.ts#L225)
  - sha: [3a64aa49a901](https://github.com/openclaw/openclaw/commit/3a64aa49a9014518fa1977fbc023f3e0b978ae5d)
