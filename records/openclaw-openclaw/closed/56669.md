## Summary

Current main no longer uses the old message-tool root-union shape that this PR patches, and the reported top-level `buttons` validation failure no longer matches the shipped schema.

## Evidence

- **Message tool schema is now a flat object, not a root anyOf/oneOf merge case:** `createMessageTool` builds `MessageToolSchema` from `buildMessageToolSchemaFromActions`, which returns `Type.Object({ action, ...props })`. That removes the old top-level variant-merging path this PR targeted for `message` actions.
