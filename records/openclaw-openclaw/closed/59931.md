## Summary

Current main implements source-authored config writes for runtime-derived config updates, which addresses both the original legacy Brave web search config failure and the later skills/update reproduction described in the issue comments.

## Evidence

- **Runtime-derived writes are projected back onto source config:** The exported writeConfigFile wrapper now detects when both runtime and source snapshots exist, computes a merge patch from the active runtime snapshot to the caller's config, and applies that patch to the source snapshot before persisting. That prevents materialized runtime-only fields such as provider-owned web search config from being written back to openclaw.json.
  - file: [src/config/io.ts:32](https://github.com/openclaw/openclaw/blob/d6ef1fcf24eea3a5985d653a3baf400e32aa1b75/src/config/io.ts#L32)

- **Runtime and source snapshots are tracked separately:** The config module keeps runtimeConfigSnapshot and runtimeConfigSourceSnapshot separately, exposes projectConfigOntoRuntimeSourceSnapshot, and loadConfig stores the runtime snapshot instead of reparsing hot paths. This is the supporting mechanism for safe writes from loadConfig-derived objects.
