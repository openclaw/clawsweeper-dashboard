## Summary

Current `main` already contains explicit WhatsApp replay/echo protections that address the reported post-credential-recovery duplication loop: stale replayed `append` history is skipped, recent gateway-sent `fromMe` echoes are dropped, and inbound message ids are deduped before dispatch.

## Evidence

- **Outbound self-echo suppression:** `normalizeInboundMessage` drops `fromMe` inbound events when their message id matches a recently sent outbound WhatsApp message for the same account/chat, which directly addresses the reported loop where the agent receives its own reply back as new input.
