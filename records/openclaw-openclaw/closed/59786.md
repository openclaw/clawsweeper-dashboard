## Summary

Close as duplicate/superseded: current main does not implement the requested cross-platform subcommand/argument allowlist behavior, but the same remaining work is already tracked by the open canonical argument-level allowlist issue and the open implementation PR attempting all-platform `argPattern` support.

## What This Changes

The issue requests subcommand-level exec allowlisting so safe commands such as `openclaw cron list` can be allowed without allowing every `openclaw` subcommand.

## Best Possible Solution

Keep the remaining product/security discussion centralized in #60427 and review or replace #60996 with a narrower, well-tested cross-host argument-pattern implementation that updates source, CLI/UI editing surfaces, protocol/schema docs, and macOS parity.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: on non-Windows, a matching executable path ignores `argPattern`, and the macOS matcher has no argument predicate, so the reported binary-level limitation remains on current main.

## Solution Assessment

Yes for the cleanup decision. The best immediate solution is not a separate tracker for this duplicate issue, but canonical review of #60427 and the open implementation PR because the change affects the host-exec approval security boundary.
