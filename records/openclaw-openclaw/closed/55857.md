## Summary

Current main already contains the missing runtime-auth forwarding for provider-owned transports, plus committed regression coverage for that path. The reported Anthropic OAuth symptom depends on `options.apiKey` being absent in the transport layer; current main explicitly bridges `authStorage`/resolved runtime auth into `options.apiKey`, so this issue is already implemented on main and present in the latest release snapshot I could inspect.

## Evidence
