## Summary

Keep open. Current `main` still has the unguarded tool-result paths this PR is trying to harden, although the original `undefined` tool-return trigger is partly mitigated by newer result normalization. The PR's file paths are stale after the `pi-extensions` -> `pi-hooks` move, but the underlying fix is not implemented on `main`.
