## Summary

Current `main` does not drop forwarded text-only Discord messages in the path described by this PR. Empty `baseText` only disables batching; it does not filter the message out. The item still flushes immediately into Discord preflight, and preflight resolves forwarded snapshot text before applying the empty-content drop.

Because the bug described here does not match current `main`, and the proposed batching change would be unsafe with the existing synthetic-message snapshot handling, I’m closing this stale PR.
