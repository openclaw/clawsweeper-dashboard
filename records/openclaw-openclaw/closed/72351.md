## Summary

Keep open: the reported TUI bug is valid, but the PR is not merge-ready because it lacks required after-fix real behavior proof and introduces two current-main blockers in the test import and session-remembering path.

## What This Changes

The PR adds a TUI zero-history branch that skips the gateway `chat.history` call when `historyLimit === 0` and adds a regression test.

## Best Possible Solution

Keep the TUI-side zero-history guard, but land it on current main with the existing session-remembering side effect, current `@earendil-works/pi-tui` test import, regression coverage, and redacted terminal/live proof.

## Reproduction Assessment

Yes, by source inspection: the CLI preserves `0`, TUI forwards it as `limit: 0`, and the gateway `chat.history` schema rejects limits below `1`. I did not run the TUI because this review is read-only.

## Solution Assessment

No, not as written. Handling zero before calling `chat.history` is the right boundary, but the patch must preserve session remembering and use the current pi-tui package before it is the narrow maintainable fix.
