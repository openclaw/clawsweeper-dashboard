## Summary

Close as duplicate/superseded: the PR targets the same degraded macOS sqlite-vec symptom already routed to open issue #66977, and its stated root cause is not supported by the current dependency contract. The published sqlite-vec helper already delegates to DatabaseSync.loadExtension(), so this one-file change does not address the remaining node:sqlite/macOS extension-loading boundary.

## What This Changes

The PR removes the default-path sqliteVec.load(db) branch from src/memory-host-sdk/host/sqlite-vec.ts, drops load from the local module type, and always calls params.db.loadExtension(extensionPath).

## Best Possible Solution

Keep #66977 as the canonical macOS sqlite-vec/node:sqlite load tracker and solve that issue by validating a supported macOS runtime, changing the SQLite binding/loading strategy, or documenting the limitation and fallback behavior; this PR should not be the fix path.
