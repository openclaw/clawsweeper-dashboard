## Summary

Current main fixes the reported packaged loader fallback by resolving built read-only chunks from the installed package `dist` root, with a regression test for the exact `node_modules/openclaw/dist/read-only-B4EkEtUx.js` path shape. The bug is real in `v2026.4.27`, but the fix is already on main and not contained in a shipped tag yet.

## What This Changes

The issue reports a startup crash after upgrading to `2026.4.27` when an external channel plugin triggers the read-only channel setup fallback and the packaged chunk looks for a missing sibling `plugins/loader.js`.

## Best Possible Solution

Keep the current main implementation and let the next release ship it; no global `node_modules/plugins` shim or postinstall mutation is needed.

## Reproduction Assessment

Yes. The issue body gives concrete upgrade and external-channel-plugin steps, `v2026.4.27` source confirms the bad source-relative loader candidates, and current tests model the installed `dist/read-only-B4EkEtUx.js` path shape.

## Solution Assessment

Yes. Resolving built fallback candidates from the OpenClaw package `dist` root is the narrow maintainable fix because it uses existing packaged loader entries instead of creating a fragile sibling `plugins/loader.js` shim outside the package.
