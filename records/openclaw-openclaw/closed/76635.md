## Summary

Current main already fixes the packaging path that made the external WhatsApp package load TypeScript at runtime; the fix is on main after the latest release and should ship with the next package publish.

## What This Changes

Installing the external WhatsApp plugin on OpenClaw 2026.5.2 loads its TypeScript entry and can starve Windows cron model calls.

## Best Possible Solution

Keep the current package-local runtime build and publish the next WhatsApp/OpenClaw release so installed packages load dist JS instead of TypeScript.

## Reproduction Assessment

Yes for the packaging root cause: the published 2026.5.2 WhatsApp package metadata points OpenClaw at ./index.ts without runtimeExtensions. I did not live-run the Windows cron stall, but the source and registry path match the reported load behavior.

## Solution Assessment

Yes. Publishing package-local dist runtime entries is the narrow maintainable fix; it avoids runtime TypeScript compilation without special-casing WhatsApp or changing cron/model-call behavior.
