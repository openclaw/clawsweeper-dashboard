## Summary

Current `main` resolves the `local` memory embedding provider through bundled capability discovery and includes follow-up bundled-plugin loader fixes for packaged installs, so the reported `openclaw memory status` failure is implemented on `main` even though it is not yet in the latest tagged release.

## Evidence

- **`local` is a declared bundled memory embedding provider:** The bundled `memory-core` plugin advertises the `local` embedding provider via its manifest contract, so provider discovery no longer depends on prior eager registration in the CLI path.
  - file: [extensions/memory-core/openclaw.plugin.json:4](https://github.com/openclaw/openclaw/blob/bf0221c5b3de3f290a934a314814426c05b59b84/extensions/memory-core/openclaw.plugin.json#L4)

- **Memory provider lookup now falls back to capability discovery:** `listMemoryEmbeddingProviders` and `getMemoryEmbeddingProvider` merge registered adapters with bundled capability-provider discovery keyed by config, which covers CLI calls that have config but did not pre-register `local`.
  - file: [src/plugins/memory-embedding-provider-runtime.ts:17](https://github.com/openclaw/openclaw/blob/bf0221c5b3de3f290a934a314814426c05b59b84/src/plugins/memory-embedding-provider-runtime.ts#L17)
