## Summary

The request is for rubric-specific task quality scoring on top of the generic background-task ledger. Current main has a generic `openclaw tasks` surface and plugin-first architecture; this kind of organization-specific observability belongs in a plugin/ClawHub package, not OpenClaw core.

## Evidence

- **Vision is plugin-first for optional capability:** `VISION.md` says core should stay lean and optional capability should usually ship as plugins, with a high bar for adding optional plugins to core.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/VISION.md#L54)
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Current core surface is `tasks`, not `workq`:** The public CLI/docs describe `openclaw tasks` as the durable background-task ledger and Task Flow surface; there is no documented `openclaw workq` command on main.
  - file: [docs/cli/tasks.md:1](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/docs/cli/tasks.md#L1)
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Core command catalog only exposes `tasks`:** The root command descriptors include `tasks` and do not include `workq`.
  - file: [src/cli/program/core-command-descriptors.ts:88](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/cli/program/core-command-descriptors.ts#L88)
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Core task ledger is a generic SQLite task store:** Main persists background tasks in `tasks/runs.sqlite`, and the `TaskRecord` / `task_runs` schema contains generic lifecycle fields only; there is no built-in quality rubric column.
  - file: [src/tasks/task-registry.paths.ts:24](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/tasks/task-registry.paths.ts#L24)
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)
- **Task schema is generic, not rubric-specific:** `TaskRecord` includes task id, runtime, status, summaries, and delivery metadata, but no task-quality score/note fields.
  - file: [src/tasks/task-registry.types.ts:53](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/tasks/task-registry.types.ts#L53)

  - file: [src/tasks/task-registry.store.sqlite.ts:368](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/src/tasks/task-registry.store.sqlite.ts#L368)
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)

  - file: [docs/plugins/sdk-overview.md:149](https://github.com/openclaw/openclaw/blob/c1383680405879a3c3f43dceb20a694f54d3c13c/docs/plugins/sdk-overview.md#L149)
  - sha: [c13836804058](https://github.com/openclaw/openclaw/commit/c1383680405879a3c3f43dceb20a694f54d3c13c)

## Risks / Open Questions

- If maintainers later want a generic core task-annotation facility for all detached work, this plugin-first close would need reevaluation.
- A plugin can annotate and report on tasks today, but it would likely use its own CLI/report surface rather than extending `openclaw tasks list --json` directly.
