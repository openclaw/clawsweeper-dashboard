## Summary

Keep open. Current main already has the renderer-side `has-copy` overlap fix, but this PR still proposes a distinct CSS hover/pointer change and the patch needs revision plus real behavior proof before it can merge.

## What This Changes

The PR changes grouped chat CSS so action padding applies on hover and always-visible chat action buttons only apply under `(hover: none) and (pointer: coarse)`.

## Best Possible Solution

Keep the merged renderer fix, revise this CSS change to gate always-visible actions on no hover-capable input, and require real desktop, hybrid, and touch proof before merge.

## Reproduction Assessment

Yes at source level, but not as a live device reproduction. Current main still uses `@media (hover: none)` for always-visible actions, while the PR replacement still keys off primary pointer state.

## Solution Assessment

No. The proposed selector improves one case, but `(hover: none) and (pointer: coarse)` can still match hybrid devices whose primary pointer is coarse/no-hover; `any-hover: none` or equivalent device-proven gating is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Gate touch-only actions on any-hover:** `ui/src/styles/chat/grouped.css:282`
  - body: The new `@media (hover: none) and (pointer: coarse)` rule still keys off the primary input, so hybrid devices whose primary pointer is coarse/no-hover can keep matching the always-visible action rule even when another input can hover.
