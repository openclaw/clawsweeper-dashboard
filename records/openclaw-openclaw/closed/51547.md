## Summary

PR #51547 is superseded by the still-open canonical bug #50519 and the newer PR #56538, which targets the same Windows GBK/CJK exec-output problem with the required buffer-plus-TextDecoder approach. Current main does not implement #51547, and #51547 itself should not be merged because it passes "gbk" as a Node BufferEncoding, which is unsupported.

## Best Possible Solution

Close #51547 as superseded. Keep the underlying user problem tracked in #50519 and focus maintainer review/rework on #56538 or another buffer-plus-TextDecoder implementation that covers both runExec and runCommandWithTimeout without using unsupported BufferEncoding values.
