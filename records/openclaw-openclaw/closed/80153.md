## Summary

Current main and the latest release already honor provider request timeouts in the embedded model idle watchdog, and the lane timeout is derived from the resolved agent run timeout plus grace rather than a fixed 210s ceiling.

## What This Changes

The issue asks for Ollama provider `timeoutSeconds` to control embedded-runner and lane watchdog behavior for slow cold-start model calls.

## Best Possible Solution

Keep the v2026.5.7/current-main provider-timeout wiring, and use https://github.com/openclaw/openclaw/issues/77744 for any separate reproducible current-release local-model cancellation.

## Reproduction Assessment

No high-confidence current-main reproduction path was established. The report is against 2026.5.2, while current main and v2026.5.7 source show the reported fixed 120s/210s ceilings are no longer the active path.

## Solution Assessment

Yes. The current implementation is the narrow maintainable solution: carry provider request timeout into the idle watchdog and keep whole-run/lane deadlines governed by the agent run timeout.
