## Summary

Current `main` already treats explicit exec-approval denial as a terminal stop, returns before any exec path, and sends denial-specific follow-up instructions that tell the agent not to rerun the command or reuse earlier output.

## Evidence

- **Explicit deny becomes terminal user denial:** `resolveBaseExecApprovalDecision` maps `decision === "deny"` to `{ approvedByAsk: false, deniedReason: "user-denied" }`, distinguishing an explicit user denial from timeout/fallback handling.

  - file: [src/agents/bash-tools.exec-host-gateway.ts:361](https://github.com/openclaw/openclaw/blob/5fe333ada85afd0def9e85985ce57e36aa00108a/src/agents/bash-tools.exec-host-gateway.ts#L361)
