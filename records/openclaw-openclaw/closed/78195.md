## What This Changes

The PR adds `cfg.media` host-read MIME allowlist/policy fields, passes them through some outbound attachment loaders, updates attachment media tests, and adds session-list benchmark scripts.

## Best Possible Solution

## Solution Assessment

No, not as submitted. The current PR adds a config knob but does not restore default ZIP behavior or thread that knob to the normal channel send media loaders, and the policy/security scope needs maintainer approval.
