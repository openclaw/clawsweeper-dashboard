## What This Changes

The issue asks for `maybeWakeNodeWithApns` to avoid retaining `nodeWakeById` entries for node IDs that have no APNs registration.

## Best Possible Solution

Delete speculative wake state on the `no-registration` path, or route that branch through an equivalent helper cleanup, with a focused regression test proving repeated unregistered wake attempts leave no `nodeWakeById` entries.

Yes. A focused unit test can mock `loadApnsRegistration` to return null, call `maybeWakeNodeWithApns` with distinct unregistered node IDs, and inspect the exported wake-state map; current source leaves entries behind.
