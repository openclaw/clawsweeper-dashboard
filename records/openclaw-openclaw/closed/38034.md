## Summary

Keep open. Current `main` has partial Perplexity/OpenRouter compatibility, but it does not implement this PR's first-class `openrouter` web-search provider, and it still does not auto-reroute `PERPLEXITY_API_KEY=sk-or-*` to OpenRouter.

## Evidence

- **No first-class OpenRouter web_search provider on main:** The bundled OpenRouter plugin manifest exposes `providers`, `mediaUnderstandingProviders`, and `imageGenerationProviders`, but no `webSearchProviders` contract. So current `main` still lacks the PR's requested dedicated OpenRouter web-search provider.
  - file: [extensions/openrouter/openclaw.plugin.json:23](https://github.com/openclaw/openclaw/blob/627c19c5cbbe8856a11effe1871c442ac87d56c9/extensions/openrouter/openclaw.plugin.json#L23)
  - command: `nl -ba extensions/openrouter/openclaw.plugin.json | sed -n '1,120p'`
  - sha: [627c19c5cbbe](https://github.com/openclaw/openclaw/commit/627c19c5cbbe8856a11effe1871c442ac87d56c9)
- **Auto-detect still resolves OpenRouter env to `perplexity`:** Current tests assert that when only `OPENROUTER_API_KEY` is present, `resolveSearchProvider({})` returns `"perplexity"`, not `"openrouter"`. That is a different behavior than this PR's explicit `openrouter` provider path.
  - file: [src/config/config.web-search-provider.test.ts:487](https://github.com/openclaw/openclaw/blob/627c19c5cbbe8856a11effe1871c442ac87d56c9/src/config/config.web-search-provider.test.ts#L487)
  - command: `nl -ba src/config/config.web-search-provider.test.ts | sed -n '482,490p'`
  - sha: [627c19c5cbbe](https://github.com/openclaw/openclaw/commit/627c19c5cbbe8856a11effe1871c442ac87d56c9)
- **`PERPLEXITY_API_KEY` env values are not prefix-inspected:** `resolvePerplexityApiKey()` treats any non-empty `PERPLEXITY_API_KEY` as `perplexity_env`, and `resolvePerplexityBaseUrl()` maps that source directly to `https://api.perplexity.ai`. There is no `sk-or-*` env-prefix check on this path, so the PR's legacy env migration case is not implemented on `main`.
  - file: [extensions/perplexity/src/perplexity-web-search-provider.runtime.ts:75](https://github.com/openclaw/openclaw/blob/627c19c5cbbe8856a11effe1871c442ac87d56c9/extensions/perplexity/src/perplexity-web-search-provider.runtime.ts#L75)
  - command: `nl -ba extensions/perplexity/src/perplexity-web-search-provider.runtime.ts | sed -n '75,117p'`
  - sha: [627c19c5cbbe](https://github.com/openclaw/openclaw/commit/627c19c5cbbe8856a11effe1871c442ac87d56c9)
- **Main only ships partial compatibility under `perplexity`:** Docs and runtime on `main` support OpenRouter compatibility only through the `perplexity` web-search provider: `OPENROUTER_API_KEY` or an `sk-or-...` key in `plugins.entries.perplexity.config.webSearch.apiKey` switch Perplexity to the chat-completions compatibility path. That overlaps with the PR's goal, but it is not the same implementation the PR proposes.
  - file: [docs/tools/perplexity-search.md:14](https://github.com/openclaw/openclaw/blob/627c19c5cbbe8856a11effe1871c442ac87d56c9/docs/tools/perplexity-search.md#L14)
