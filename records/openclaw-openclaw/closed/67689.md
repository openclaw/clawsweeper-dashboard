## Summary

Current main already addresses the observable problem behind #67689. Dreaming narrative sessions are cleaned up automatically, stale dreaming entries/orphan transcripts are scrubbed, and the Control UI Sessions tab already supports keyword filtering, page selection, bulk delete, and transcript archiving through `sessions.delete`. A narrower optional hide-system toggle remains separately tracked by open PR #67155 / issue #67058, but this broad issue no longer needs to stay open as the manual `sessions.json` cleanup workflow is covered.

## Best Possible Solution

Close this broad issue as implemented on main. Keep the shipped automatic dreaming cleanup and Control UI search/bulk-delete behavior; use #67155/#67058 as the focused thread if maintainers still want a default hide-system toggle or a non-destructive visibility filter.
