## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Wire chained reads into chat.history:** `src/gateway/session-utils.fs.ts:281`
  - body: `chat.history` uses `readRecentSessionMessagesAsync`, but the branch only routes sync `readSessionMessages` through the chained resolver. The gateway RPC named in the repro would still read the live transcript segment only.
  - confidence: 0.93
- **[P1] Preserve active-branch filtering per segment:** `src/gateway/session-utils.fs.ts:287-288`
  - body: The new per-file loop parses every JSONL line directly, bypassing the active-branch selection current readers use for append-only tree transcripts. Abandoned rewrite branches can be reintroduced into history.
  - confidence: 0.91
- **[P1] Respect reset and restore epochs before replaying snapshots:** `src/gateway/session-utils.fs.ts:212-223`
  - body: The loop prepends every stored checkpoint snapshot. Because reset and checkpoint restore preserve checkpoint metadata, default history can resurface pre-reset content or discarded restore-era content.
  - confidence: 0.92
- **[P1] Trim checkpoint snapshots at compaction boundaries:** `src/gateway/session-utils.fs.ts:212-223`
  - body: Checkpoints record `firstKeptEntryId` because the successor transcript retains an unsummarized tail. Concatenating full snapshots with the successor can duplicate retained turns and produce wrong sequence counts.
  - confidence: 0.86
- **[P2] Constrain checkpoint paths before reading:** `src/gateway/session-utils.fs.ts:196-198`
  - body: `pushFile` accepts stored paths after `path.resolve()` and existence checks only. Stale or corrupted session-store metadata can make the reader ingest unrelated JSONL files outside the active session store.
  - confidence: 0.9
- **[P2] Prefer the canonical store row before returning:** `src/gateway/session-utils.fs.ts:125-140`
  - body: The store scan returns the first matching `sessionId` and optional `sessionFile`, so duplicate legacy rows can select a stale or non-main entry before the canonical main row with the checkpoint chain is considered.
  - confidence: 0.82
