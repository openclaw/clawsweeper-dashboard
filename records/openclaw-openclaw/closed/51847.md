## Summary

Current `main` already implements a broader fix than this PR: `hooks.mappings[].channel` is no longer hardcoded to a small enum, accepts runtime plugin channel ids in config validation, and is validated against registered channel plugins at hook dispatch time.

## Evidence

- **Hook mapping type is open-ended on main:** `HookMappingConfig.channel` now allows `"last" | (string & {})` with an explicit comment that runtime channel ids, including plugin channels, are allowed.
  - file: [src/config/types.hooks.ts:25](https://github.com/openclaw/openclaw/blob/c1b9aa7d9630efe79f366b98564854aa935d00f4/src/config/types.hooks.ts#L25)

- **Zod schema no longer hardcodes channel literals:** `HookMappingSchema.channel` is now `z.string().trim().min(1).optional()` with comments stating plugin channels should not be hardcoded in config schema.
  - file: [src/config/zod-schema.hooks.ts:52](https://github.com/openclaw/openclaw/blob/c1b9aa7d9630efe79f366b98564854aa935d00f4/src/config/zod-schema.hooks.ts#L52)

  - file: [src/config/config.hooks-module-paths.test.ts:80](https://github.com/openclaw/openclaw/blob/c1b9aa7d9630efe79f366b98564854aa935d00f4/src/config/config.hooks-module-paths.test.ts#L80)

  - file: [src/gateway/hooks.ts:234](https://github.com/openclaw/openclaw/blob/aa3a5f9ff7907c2a41d6259d24b529bda41f293a/src/gateway/hooks.ts#L234)
