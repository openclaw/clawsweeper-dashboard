## Summary

Current main has already superseded this PR with a shipped systemd env-file implementation: it renders EnvironmentFile entries, writes dotenv-backed service secrets to a mode-0600 gateway.systemd.env file, omits those values from inline Environment lines, and audits EnvironmentFile-backed values correctly. The older PR should not be merged over that implementation, especially because review comments flagged an env-file escaping/read-back issue in the PR diff.

## Best Possible Solution

Close this stale PR as superseded by the shipped #66249 systemd env-file work. Any remaining broader service-secret or uninstall-cleanup questions should stay with narrower current issues rather than landing this older diff over main.
