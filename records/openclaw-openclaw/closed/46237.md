## Summary

Current `main` already contains multiple Telegram-side protections that cover this stale-result resurfacing class: replay dedupe for successful deliveries, preserved thread/topic routing for subagent completion delivery, explicit suppression of superseded stale Telegram replies, and startup cleanup of stale ACP thread bindings before they can be reused.
