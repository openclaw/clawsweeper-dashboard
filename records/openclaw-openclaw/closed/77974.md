The issue reports intermittent claude-cli conversation context loss when OpenClaw invalidates a reusable session as missing-transcript before the Claude transcript is flushed.

## Best Possible Solution

Keep the current main core runtime fix, and use narrower trackers for distinct remaining transcript-writer or long-lived lifecycle failures such as https://github.com/openclaw/openclaw/issues/78273 and https://github.com/openclaw/openclaw/issues/84593.

## Solution Assessment

Yes. The current implementation fixes the issue at the core CLI runtime boundary by retrying/probing transcript visibility, preventing unflushed bindings from being persisted, and using bounded OpenClaw-history reseed rather than adding Telegram-specific behavior.
