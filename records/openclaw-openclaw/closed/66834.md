## Summary

Close as `clawhub`: the warning is coming from an external Discord plugin command description, not from an OpenClaw core command. Current `main` already guards Discord's 100-character limit by truncating plugin-provided descriptions, and OpenClaw's scope guidance says optional capability should live in separately maintained plugins/ClawHub rather than core.

## Evidence

- **Project scope keeps optional capability in plugins:** OpenClaw's vision says optional capability should usually ship as plugins, plugin authors should host and maintain them in their own repositories, and plugin discovery/promotion belongs in ClawHub rather than core.

- **Discord native commands ingest plugin-owned command metadata:** Discord command registration appends plugin command specs by reading `getPluginCommandSpecs("discord")` and pushing each plugin command's `name` and `description` into the native command list. That means the `/lossless` description text is owned by the plugin that registered it, not by core.
  - file: [extensions/discord/src/monitor/provider.ts:188](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/extensions/discord/src/monitor/provider.ts#L188)
