## Summary

Keep open. Current main does not contain the proposed Prompt Inspector integration, so this is not obsolete, but the PR is not merge-ready because it targets stale hook wiring and introduces a concrete plaintext detector endpoint risk while also needing maintainer/secops judgment on the vendor, privacy, and capability boundary.

## What This Changes

The PR adds a Prompt Inspector HTTP client with PMTINSP environment controls and wires detection into hook dispatch, cron external-content wrapping, and web-tool result scanning.

## Best Possible Solution

Keep the security hardening work open, but land it only after the hook integration is rebased onto the live handler and the maintainer team chooses between a reviewed core content-inspection capability with provider plugins or a narrowly documented Prompt Inspector integration with explicit privacy, config, tests, and secure endpoint policy.
