## Summary

Current `main` already routes the initial `sessions.create` message through the normal `chat.send` persistence path, and the same implementation plus regression coverage is present in the shipped `v2026.4.23` tag.

## Evidence

- **Current main persists the initial `sessions.create` turn through `chat.send`:** `sessions.create` resolves the initial `task`/`message`, computes `messageSeq` from the transcript, and then calls `chatHandlers["chat.send"]` after `ensureSessionTranscriptFile(...)` succeeds, which puts the first user turn on the standard write/dispatch path instead of a separate silent path.
