## Summary

Keep this PR open. Current main still lacks the requested user-message color env vars, and the PR remains open and unmerged, but the proposed implementation has a concrete readability bug for background-only overrides that should be addressed before merge.

## What This Changes

The PR branch adds `OPENCLAW_TUI_USER_BG` and `OPENCLAW_TUI_USER_TEXT` TUI palette overrides with strict hex parsing, theme tests, and environment/TUI documentation updates.

## Best Possible Solution

Land the customization only after the TUI theme layer preserves readable user-message text for single-variable overrides, defines malformed and low-contrast override behavior, and keeps the environment and TUI docs aligned with that policy.
