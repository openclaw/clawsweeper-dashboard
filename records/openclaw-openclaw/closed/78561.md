## Summary

Keep open. Current main fixed the narrower Discord partial-draft finalization failure, but this broader report is not conclusively resolved because Discord preview-off, block delivery, and verbose tool/progress delivery still travel through separate source paths with an unresolved policy boundary.

## What This Changes

The issue asks OpenClaw to prevent misleading Discord mid-turn partial, block, or progress fragments when streaming is disabled or configured to avoid partial output.

## Best Possible Solution

Add a focused Discord runtime regression that separates preview streaming, explicit block streaming, inherited block defaults, and verbose tool/progress delivery, then document or implement the chosen final-only policy for `streaming.mode=off`.

## Reproduction Assessment

No, not as a high-confidence live reproduction. Source inspection shows plausible current-main paths where Discord preview streaming is off but separate block/progress delivery can still emit mid-turn messages, but the issue lacks exact config, fragments, timestamps, or logs.

## Solution Assessment

Unclear. A narrow fix may exist, but maintainers should first decide whether Discord `streaming.mode=off` means preview-only off or all mid-turn block/progress output off when other controls are enabled.
