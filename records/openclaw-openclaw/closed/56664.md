## Summary

Current `main` no longer matches the PR's reported failure mode. Discord lifecycle failures can still reject, but channel startup is now wrapped by the gateway channel manager, which catches account-task failures, records `lastError`, marks the account stopped, and schedules restart/backoff instead of letting the rejection crash the gateway process.
