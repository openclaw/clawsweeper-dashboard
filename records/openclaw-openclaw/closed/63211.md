## Summary

Keep open. Current main does not provide strong evidence that #63211 is fixed, and static inspection still shows a plausible isolated-cron gap: interactive/heartbeat reply flows compute and pass exec overrides from per-agent/session defaults into embedded runs, but the isolated cron embedded-run path does not. The exec tool still accepts stricter per-call `security`/`ask` values from tool calls, which matches the reporters' contradictory approval-path symptoms. Combined with three independent reproductions in the issue thread, this is not strong enough for conservative closure.

## Evidence

- **Issue thread has multiple independent reproductions:** The provided issue context includes three separate confirmations across WhatsApp, Discord, Telegram, stable, and beta builds, all describing isolated async runs prompting for exec approval despite configured `tools.exec.ask="off"`/broader policy.
  - command: `provided GitHub context for issue #63211`
- **Interactive and heartbeat flows resolve exec overrides from per-agent/session defaults:** Reply handling computes `execOverrides` from directives, session state, and `agentEntry?.tools?.exec`, which is the mechanism that tells embedded runs the current exec defaults.
