## Summary

Current `main` already contains a targeted fix for the reported `browser.request`/CLI unavailability case caused by restrictive plugin allowlists: the bundled browser plugin now auto-enables whenever `browser` config or browser tool policy references it, gateway startup persists that auto-enable, regression tests cover the behavior, docs call out the exact symptom, and the changelog records the fix as shipped in `v2026.4.5`.
