## What This Changes

The PR changes boot-md startup runs to use an isolated boot session key, suppress persistence of the generated BOOT.md prompt, and update boot tests around session-store mapping restoration.

## Best Possible Solution

Land this focused boot isolation after redacted real gateway proof shows repeated startup with BOOT.md leaves the canonical main transcript and LCM history clean while the boot mapping remains transient.

## Reproduction Assessment

Yes at source level: current main sends the boot-md synthetic prompt through the canonical session key without suppressPromptPersistence, and the linked report gives concrete installed-gateway restart evidence. I did not run a live gateway restart in this read-only review.

## Solution Assessment

Yes for the forward fix: using agent:<id>:boot plus suppressPromptPersistence is the narrow maintainable path for keeping startup prompts out of user-facing history. Existing polluted transcript cleanup can remain a separate doctor/follow-up decision if maintainers want it.
