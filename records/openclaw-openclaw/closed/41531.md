## Summary

Current `main` already contains targeted fixes and regression coverage for Telegram background/topic routing. The outbound heartbeat/delivery resolver now keeps chat and thread paired, avoids inheriting stale thread IDs for heartbeat/background sends, preserves explicit topic-qualified Telegram targets, and rewrites legacy username-based cron/default targets to numeric chat IDs without dropping `:topic:` suffixes.

## Evidence

- **Heartbeat/background routing no longer inherits stale thread IDs:** `resolveHeartbeatDeliveryTarget` only forwards `turnSourceThreadId` from an explicit event context and explicitly avoids falling back to session-stored thread IDs for heartbeat mode, preventing stale topic reuse in background sends.
  - file: [src/infra/outbound/targets.ts:128](https://github.com/openclaw/openclaw/blob/a903438707f11de28f65c33511ac1da9ee0641cb/src/infra/outbound/targets.ts#L128)
  - command: `nl -ba src/infra/outbound/targets.ts | sed -n '95,140p'`
  - sha: [a903438707f1](https://github.com/openclaw/openclaw/commit/a903438707f11de28f65c33511ac1da9ee0641cb)
- **Chat/thread pairing stays coupled during last-target resolution:** `resolveSessionDeliveryTarget` parses comparable targets for both session state and turn-source state, clears inherited thread IDs when the channel/route changes, and only reuses a thread when it still belongs to the same resolved route.
  - file: [src/infra/outbound/targets-session.ts:67](https://github.com/openclaw/openclaw/blob/a903438707f11de28f65c33511ac1da9ee0641cb/src/infra/outbound/targets-session.ts#L67)
  - command: `nl -ba src/infra/outbound/targets-session.ts | sed -n '67,160p'`
  - sha: [a903438707f1](https://github.com/openclaw/openclaw/commit/a903438707f11de28f65c33511ac1da9ee0641cb)
- **Isolated/background heartbeat path explicitly avoids stale turn-source reuse:** `runHeartbeatOnce` passes no turn-source delivery context for isolated heartbeat runs, with an inline note that reusing base-session turn-source routing could pin later isolated runs to stale channels/threads.
  - file: [src/infra/heartbeat-runner.ts:804](https://github.com/openclaw/openclaw/blob/a903438707f11de28f65c33511ac1da9ee0641cb/src/infra/heartbeat-runner.ts#L804)
  - command: `nl -ba src/infra/heartbeat-runner.ts | sed -n '798,812p'`
  - sha: [a903438707f1](https://github.com/openclaw/openclaw/commit/a903438707f11de28f65c33511ac1da9ee0641cb)
- **Regression tests cover stale Telegram topic misrouting in background runs:** Tests verify isolated wake runs do not reuse stale turn-source routing and that exec-event delivery stays pinned to the original Telegram topic even when session route state drifted to another topic.
  - file: [src/infra/heartbeat-runner.ghost-reminder.test.ts:492](https://github.com/openclaw/openclaw/blob/a903438707f11de28f65c33511ac1da9ee0641cb/src/infra/heartbeat-runner.ghost-reminder.test.ts#L492)

- **Explicit Telegram topic targets are parsed and preserved for heartbeat delivery:** Tests verify `heartbeat.to` values like `-100111:topic:42` resolve to chat `-100111` plus `threadId: 42`, so background delivery keeps the exact `(chat_id, message_thread_id)` pair instead of collapsing to the group root.
  - file: [src/infra/heartbeat-runner.returns-default-unset.test.ts:478](https://github.com/openclaw/openclaw/blob/a903438707f11de28f65c33511ac1da9ee0641cb/src/infra/heartbeat-runner.returns-default-unset.test.ts#L478)

  - file: [extensions/telegram/src/send.ts:623](https://github.com/openclaw/openclaw/blob/a903438707f11de28f65c33511ac1da9ee0641cb/extensions/telegram/src/send.ts#L623)
