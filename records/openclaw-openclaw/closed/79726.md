## Summary

Keep open: the latest PR head still has a P1 transport-policy regression, and the PR carries the protected `maintainer` label. The PI-native happy path is proven, but custom Codex baseUrl/request-policy routes can bypass OpenClaw's boundary-aware guarded fetch.

## What This Changes

The PR routes explicit OpenAI Codex Responses runs through PI native Codex Responses streaming, removes OpenClaw's custom OpenAI Responses WebSocket stack and warm-up config, updates PI packages to 0.73.1, and refreshes related tests, docs, changelog, and snapshots.

## Best Possible Solution

Gate PI-native Codex Responses to known native ChatGPT/Codex endpoints with no custom request-policy requirements, and keep custom, proxied, TLS-overridden, or private-network-sensitive routes on OpenClaw's boundary-aware transport.

## Reproduction Assessment

Not applicable for the refactor itself, but the blocking regression is source-reproducible: the PR returns PI-native streaming for default Codex Responses models before checking baseUrl or request-policy compatibility.

## Solution Assessment

No, not as-is. The narrow maintainable fix is to preserve the PI-native route only for known native Codex endpoints and fall back to `createBoundaryAwareStreamFnForModel` when OpenClaw's request boundary is needed.
