## Summary

Close as ClawHub/plugin-scope. The request is an optional Discord-specific provisioning capability. Current main does not expose a general `webhook-create` Discord message action; it only has internal thread-binding webhook creation. The current plugin system already supports optional agent tools/code plugins, and `VISION.md` directs optional capabilities to plugins and ClawHub rather than adding one-off core/bundled behavior.

## Best Possible Solution

Keep the core/shared message tool and bundled Discord action set unchanged for now. Implement this as an optional ClawHub Discord webhook provisioning plugin or MCP/tool with explicit enablement, permission documentation, and secret-safe output. If that implementation proves a concrete missing SDK seam, open a narrow plugin API issue for that seam.

## Risks / Open Questions

- A maintainer could still choose to promote this into the bundled Discord plugin later if there is strong product demand; this cleanup decision only says the current best default is external plugin distribution.
- A ClawHub implementation must handle Discord bot credentials and returned webhook URLs as secrets, and should require explicit tool allowlisting plus Discord Manage Webhooks permission.
