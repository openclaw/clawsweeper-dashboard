## Summary

Current `main` already auto-watches memory files and schedules SQLite memory reindexing on external file changes; the reported `2026.3.11` behavior is obsolete on current releases.

## Evidence

- **SQLite memory manager now watches memory paths:** `ensureWatcher()` builds watched paths for `MEMORY.md`, the `memory` directory, and extra memory paths, then starts a `chokidar` watcher and marks the index dirty on `add`/`change`/`unlink`.
  - file: [extensions/memory-core/src/memory/manager-sync-ops.ts:367](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/extensions/memory-core/src/memory/manager-sync-ops.ts#L367)
