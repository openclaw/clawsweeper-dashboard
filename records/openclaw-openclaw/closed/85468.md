The branch bumps the root `packageManager` pin to pnpm 11.2.2, routes workflow pnpm setup through package.json via a pinned `pnpm/action-setup` wrapper, removes separate `PNPM_VERSION` plumbing, and adds exact `minimumReleaseAge` exclusions plus workflow tests.

## Best Possible Solution

Keep packageManager-based pnpm version resolution, but preserve OpenClaw's current best-effort pnpm store cache semantics and keep the release-age exclusions exact and auditable.
