## Summary

Current `main` already implements the Control UI/WebChat image-attachment path end to end: pasted/selected images are turned into `chat.send` attachments in the UI, the gateway parses and persists them, and current tests cover the prior silent-drop behavior for text-only session models by offloading images instead of dropping them.

## Evidence

- **Control UI collects pasted/selected images as attachments:** The chat view handles paste, file-pick, and drag/drop image inputs by reading them with `FileReader` into `ChatAttachment` objects.
  - file: [ui/src/ui/views/chat.ts:208](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/ui/src/ui/views/chat.ts#L208)
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Control UI sends images through `chat.send` RPC:** The Control UI converts attachment data URLs to base64 image payloads and sends them as `attachments` on the gateway `chat.send` request; there is no separate browser POST upload path in this implementation.
  - file: [ui/src/ui/controllers/chat.ts:213](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/ui/src/ui/controllers/chat.ts#L213)
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Gateway parses and accepts chat attachments:** `chat.send` normalizes RPC attachments, rejects only empty message+attachment submissions, and parses attachment images for either inline image handling or offloaded media handling.
  - file: [src/gateway/server-methods/chat.ts:2113](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/gateway/server-methods/chat.ts#L2113)
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Gateway stores attachment results as media fields:** User transcript updates are built from saved media using `MediaPath`/`MediaPaths` and `MediaType`/`MediaTypes`, which explains why current behavior is not centered on an `attachments` metadata array.
  - file: [src/gateway/server-methods/chat.ts:741](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/gateway/server-methods/chat.ts#L741)
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
- **Current tests cover the old silent-drop path:** The gateway tests now assert that text-only session models offload image attachments to `media://inbound/...` instead of dropping them, and that agent-scoped text-only sessions do the same.
  - file: [src/gateway/server-methods/chat.directive-tags.test.ts:2080](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/gateway/server-methods/chat.directive-tags.test.ts#L2080)
  - sha: [fa976e5b93b3](https://github.com/openclaw/openclaw/commit/fa976e5b93b3819f9e137fe08635764edee38290)
