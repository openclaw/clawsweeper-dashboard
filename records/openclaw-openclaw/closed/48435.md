## Summary

Keep open. Current main still cannot express or send the requested Slack unfurl controls: the strict Slack config schema/type lack the fields, the only Slack `chat.postMessage` payload builder omits the API arguments, and related PR attempts are closed unmerged.

## What This Changes

Add Slack `unfurlLinks` and `unfurlMedia` configuration so OpenClaw can pass `unfurl_links` and `unfurl_media` to Slack `chat.postMessage` for bot messages.

## Best Possible Solution

Implement the controls in the bundled Slack plugin/config surface with top-level and per-account inheritance, preserve Slack defaults when unset, pass configured booleans through normal text/block/follow-up sends, and update docs, generated config metadata, changelog, and regression tests.

## Reproduction Assessment

Yes. For this feature gap, source inspection is enough: the proposed config keys are absent from the strict Slack schema/type on current main, and outbound `postSlackMessageBestEffort` never sends `unfurl_links` or `unfurl_media`.

## Solution Assessment

Yes. A config-driven bundled Slack change is the narrowest maintainable solution for this request; per-message tool controls could be a separate feature, but they are not required to solve the reported bot-output problem.
