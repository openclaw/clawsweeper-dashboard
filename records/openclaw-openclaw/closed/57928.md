## Summary

Current `main` already routes `session.tool` and `session.message` websocket events with the originating child session key for subagent sessions, and has direct regression tests covering both behaviors.

## Evidence

- **`session.tool` uses the event/run session key and mirrors it to session subscribers:** The gateway agent-event handler resolves `sessionKey` from the event/run context and emits `session.tool` with that same key plus session snapshot metadata.
  - file: [src/gateway/server-chat.ts:891](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/gateway/server-chat.ts#L891)
  - command: `nl -ba src/gateway/server-chat.ts | sed -n '888,970p'`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Regression test for subagent-style `session.tool` payloads:** The test asserts `session.tool` carries `sessionKey: "session-1"` along with spawned-session metadata when mirrored to session subscribers.
  - file: [src/gateway/server-chat.agent-events.test.ts:730](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/gateway/server-chat.agent-events.test.ts#L730)
  - command: `nl -ba src/gateway/server-chat.agent-events.test.ts | sed -n '728,790p'`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **`session.message` broadcasts use the transcript update session key:** Transcript updates resolve the owning session key and broadcast `session.message` with that key to both global session subscribers and per-session message subscribers.
  - file: [src/gateway/server-session-events.ts:89](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/gateway/server-session-events.ts#L89)
  - command: `nl -ba src/gateway/server-session-events.ts | sed -n '74,130p'`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)

  - file: [src/gateway/session-message-events.test.ts:366](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/src/gateway/session-message-events.test.ts#L366)
