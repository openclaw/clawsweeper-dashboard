## Summary

Current main already implements the requested diagnostic split: sqlite-vec load failures include the underlying error, while the no-dimensions path reports semantic vector embeddings as unavailable instead of blaming sqlite-vec. The same behavior is present in the latest release tag, so this issue can close as implemented.

## What This Changes

The issue asks OpenClaw to stop emitting a blind recurring sqlite-vec warning and surface the actual memory-vector blocker when chunks_vec is not updated.

## Best Possible Solution

Keep the shipped split-warning behavior and route any remaining platform-specific sqlite-vec failures to the narrower existing reports.

## Reproduction Assessment

No current-main failing reproduction remains from source inspection. The reported 2026.5.7 warning shape is covered by current tests that now distinguish sqlite-vec load errors from missing vector dimensions.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix: preserve the one-time degraded warning, include loadError when present, and name the embedding-dimensions blocker when sqlite-vec is not the failing component.
