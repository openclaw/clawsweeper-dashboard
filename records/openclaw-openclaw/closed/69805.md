## Summary

Current main already includes a shipped fix for hanging one-shot gateway RPC CLI commands, and the reported `status`, `cron list`, and doctor gateway-probe paths all run through that shared code.

## Evidence

- **2026.4.23 changelog entry:** The `2026.4.23` changelog explicitly says OpenClaw now waits for one-shot gateway RPC clients to finish WebSocket teardown before the CLI exits, reducing hangs where commands like `openclaw status` stayed alive until an external timeout killed them.
  - file: [CHANGELOG.md:434](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/CHANGELOG.md#L434)

  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Shared gateway teardown helper:** `src/gateway/call.ts` now uses `client.stopAndWait({ timeoutMs: 1_000 })` before falling back to `client.stop()`, which is the core fix for one-shot CLI teardown hangs.
  - file: [src/gateway/call.ts:95](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/gateway/call.ts#L95)

- **Gateway request resolves only after teardown:** The request path clears its timer and only resolves or rejects after `stopGatewayClient(client)` finishes, so the CLI no longer exits while the WebSocket client is still tearing down.
  - file: [src/gateway/call.ts:487](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/gateway/call.ts#L487)

- **Regression tests for the hang:** `src/gateway/call.test.ts` adds coverage that `callGateway()` waits for lingering teardown to finish and does not let the wrapper timeout fire while waiting.
  - file: [src/gateway/call.test.ts:844](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/src/gateway/call.test.ts#L844)
