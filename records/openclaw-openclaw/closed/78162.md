## Summary

Keep open: current main still has a source-reproducible `models auth login` path where a provider `configPatch` can change `agents.defaults.model` before the `--set-default` gate, and the linked closing PR is still open.

## What This Changes

The issue asks OpenClaw to preserve the existing default model when adding or refreshing providers unless the user explicitly opts into changing it.

## Best Possible Solution

Land or replace the linked fix so `models auth login` restores the previous `agents.defaults.model` unless `--set-default` is passed, while leaving tier labels as separate follow-up work.

## Reproduction Assessment

Yes at source level: `models auth login` applies provider config patches before the explicit `--set-default` check, and OpenAI/Google provider helpers can put a primary model in those patches. I did not run a live provider login in this read-only review.

## Solution Assessment

Yes: preserving or restoring the prior `agents.defaults.model` around the `models auth login` config-patch merge is the narrowest maintainable fix. Changing provider catalogs or adding cheap/balanced/premium tier labels is useful but broader than the bug fix.
