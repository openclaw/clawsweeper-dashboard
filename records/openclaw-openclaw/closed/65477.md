## What This Changes

Request adds the active model/provider label to per-response `/usage` footers in messaging channels.

## Best Possible Solution

Land or consolidate a narrow implementation that renders the actual resolved provider/model in the existing usage-footer path while preserving token, cache, cost, and session output.

## Solution Assessment

Is this the best way to solve the issue? Yes, adding the resolved provider/model to the existing usage-footer path is the narrow maintainable direction; the open PRs need consolidation around exact wording and scope.
