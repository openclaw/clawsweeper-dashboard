## Summary

Current main already fixes the reported MiniMax remaining-vs-used inversion: MiniMax remaining fields are normalized to internal usedPercent, and the status renderer displays 100 - usedPercent as `% left`. The relevant fixes are merged and shipped.

## What This Changes

The issue asks OpenClaw to stop showing a MiniMax usage percentage as `X% left` when the value represents consumed quota.

## Best Possible Solution

Keep the current normalized usedPercent contract and `% left` renderer; only add another parser case if a redacted MiniMax payload shows an uncovered response shape.

## Reproduction Assessment

No current-main failing reproduction is evident. Source and regression tests show the reported 98 remaining / 2 used shape now normalizes to usedPercent 2 and renders as 98% left.

## Solution Assessment

Yes. Normalizing provider-specific remaining fields into the existing usedPercent contract is the narrow maintainable fix, and it avoids adding a provider-specific display mode or config option.
