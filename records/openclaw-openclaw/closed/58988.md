## Summary

Current `main` already supersedes this PR with a broader Discord gateway shutdown/lifecycle rewrite: abort-time disconnects now flow through `waitForDiscordGatewayStop()`, late `Max reconnect attempts ...` errors are classified and suppressed by the gateway supervisor, and regression tests cover the crash path this PR targeted.
