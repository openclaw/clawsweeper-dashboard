## Summary

Close as duplicate/superseded. Current main still lacks the requested DeepSeek V4 thinking-profile exposure, but the same remaining work is already tracked by canonical issue #72893 and open PR #73008, which covers the DeepSeek-owned profile, tests, and docs path.

## What This Changes

Expose `xhigh` and `max` thinking levels for native DeepSeek V4 Flash/Pro models by adding a DeepSeek-owned provider thinking profile that maps those levels to DeepSeek max reasoning.

## Best Possible Solution

Keep the remaining work centralized on #72893 and review #73008, or an equivalent DeepSeek-local patch, so only `deepseek-v4-flash` and `deepseek-v4-pro` expose `off|minimal|low|medium|high|xhigh|max` while older DeepSeek models keep their existing behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: resolve or list thinking levels for `deepseek/deepseek-v4-pro` on current main and `max` will be absent because the DeepSeek provider has no profile hook.

## Solution Assessment

Yes. Closing this duplicate while keeping #72893/#73008 as the canonical path is the cleanest maintainer outcome; the actual code fix should remain a provider-local `resolveThinkingProfile`, not a broad core resolver change.
