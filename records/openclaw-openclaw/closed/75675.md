## Summary

Current main already implements the PR's intended memory embedding proxy behavior through the merged #71506 path, and the implementation is narrower than the proposed helper-level env check because it preserves strict SSRF handling for targets Undici would not proxy.

## What This Changes

The PR changes the memory remote HTTP helper to pass `trusted_env_proxy` mode when proxy environment variables are configured, with an optional caller-supplied guarded-fetch mode override.

## Best Possible Solution

Close this PR and keep the shipped #71506 implementation as the maintained path; future memory-host proxy changes should build on `shouldUseEnvHttpProxyForUrl` and keep the core and package SDK helper surfaces aligned.

## Reproduction Assessment

Yes. The historical failure path is remote embedding or batch HTTP behind `HTTP_PROXY`/`HTTPS_PROXY`; current main has focused helper and proxy-selection tests that verify trusted env-proxy mode is selected only for proxied targets.

## Solution Assessment

Yes for current main, not for this stale PR branch. The shipped implementation is the narrower maintainable solution because it uses the target-aware proxy helper in both memory-host surfaces and preserves strict DNS-pinned SSRF checks for `ALL_PROXY`-only and `NO_PROXY` bypass cases.
