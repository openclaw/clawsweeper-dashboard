## Summary

This should stay open: current main still has the source-level regression, and the linked open PR is the active targeted fix path rather than a reason to close the issue early.

## What This Changes

The issue reports that an external WeChat channel plugin stopped initializing on OpenClaw 2026.5.4 because runtime injection no longer reaches its setup-runtime startup path.

## Best Possible Solution

Land #77799 or an equivalent loader fix that forwards non-bundled setup-entry `setChannelRuntime`, with regression coverage and a changelog entry, then close this after it is implemented.

## Reproduction Assessment

Yes, source-reproducible on current main: the non-bundled setup-entry resolver drops `setChannelRuntime`, and the loader only invokes the setter if that field survived resolution. I did not run a live WeChat bot.

## Solution Assessment

Yes, based on current source and the related PR context, forwarding the existing setup-entry setter through `resolveSetupChannelRegistration` is the narrowest maintainable fix. Updating only the external plugin would not repair the OpenClaw SDK compatibility regression for other affected channel plugins.
