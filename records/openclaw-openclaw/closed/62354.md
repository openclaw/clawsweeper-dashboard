## Summary

Current `main` already handles the reported empty-error session failure mode by retrying zero-output assistant errors in-place and by cleaning errored replay state before subsequent turns, so the reported stuck-until-`/new` behavior is implemented away on `main`.

## Evidence

- **In-place retry for empty assistant error turns:** `runEmbeddedPiAgent` now detects assistant turns with `stopReason === "error"`, zero output tokens, and empty content, then resubmits the same prompt/session instead of immediately surfacing an incomplete-turn failure.
  - file: [src/agents/pi-embedded-runner/run.ts:2086](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/agents/pi-embedded-runner/run.ts#L2086)
  - sha: [51dd4f288f67](https://github.com/openclaw/openclaw/commit/51dd4f288f67ef4df4c7a121f078f508535df1fd)
- **Regression coverage matches the reported failure shape:** New regression tests describe the silent `stopReason="error"` + zero-output symptom and assert automatic retry, capped retry exhaustion, and model-agnostic behavior.
  - file: [src/agents/pi-embedded-runner/run.empty-error-retry.test.ts:13](https://github.com/openclaw/openclaw/blob/51dd4f288f67ef4df4c7a121f078f508535df1fd/src/agents/pi-embedded-runner/run.empty-error-retry.test.ts#L13)
