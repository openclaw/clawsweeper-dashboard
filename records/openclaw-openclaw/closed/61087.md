## Summary

Current main does not implement this PR's intended Discord upload-file discoverability fix. Discord message-tool discovery still advertises send/read/edit/delete under the messages gate but not upload-file, while the Discord send path and runtime already accept media/path/filePath and can upload attachments. The PR remains an implementation candidate for open bug #60652 and should be revised rather than closed as obsolete.

## Best Possible Solution

Keep this PR open as an implementation candidate for the open Discord discoverability bug. The best path is to pick one canonical PR among the duplicate candidates, add upload-file to Discord action discovery, add a Discord upload-file dispatch branch that preserves the same session, agent, and reply metadata as send, and add focused discovery and dispatch tests before merge.
