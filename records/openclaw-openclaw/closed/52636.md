  - confidence: 0.91
- **[P1] Enforce the loopback ceiling before normal lockout:** `src/gateway/auth-rate-limit.ts:214-215`
  - body: Loopback failures still hit the normal `maxAttempts` lockout at this line, and later `recordFailure()` calls return while locked. Because `check()` bypasses `lockedUntil` for loopback until attempts reach `maxAttempts * 10`, the counter never reaches the advertised ceiling and localhost failures can continue indefinitely.
  - confidence: 0.9
- **[P2] Reject malformed webhook signatures without throwing:** `src/security/shield/webhook-dispatch.ts:63`
  - body: This compares hex string lengths before decoding, then passes decoded buffers to `timingSafeEqual()`. A same-length non-hex signature decodes to a shorter buffer and makes verification throw instead of returning `false`, turning a bad signature into an exception path.
  - confidence: 0.86
- **[P2] Expire stale Shield health scores when metrics age out:** `src/security/shield/gateway-shield.ts:265-287`
  - body: `computeHealthScores()` filters metrics to the last five minutes but only updates functions present in the new aggregate. Functions with no recent metrics keep old `CRITICAL` or `CIRCUIT_OPEN` scores, so `shouldThrottle()` can continue throttling idle or recovered functions from stale state.
  - confidence: 0.82
