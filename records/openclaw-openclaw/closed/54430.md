## Summary

The reported Feishu proactive-send bug is already gone on current `main` and in the latest shipped release: the `send`/`thread-reply` path no longer accepts a raw `params.card` object, so `card={}` cannot reach `sendCardFeishu` there.

## Evidence

- **Current send path no longer reads raw `params.card`:** In the Feishu action handler, `send`/`thread-reply` derive `card` only from `params.presentation`, then require `text/message`, `media`, or that derived card before sending. There is no `ctx.params.card` read in this path.
  - file: [extensions/feishu/src/channel.ts:695](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/extensions/feishu/src/channel.ts#L695)
