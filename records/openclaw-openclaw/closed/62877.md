## Summary

Keep open: current main still emits the raw `/btw <side question>` usage text, and the current plain-text delivery sanitizer removes that tag-shaped placeholder before direct channel delivery.

## What This Changes

The issue asks OpenClaw to stop emitting a tag-shaped `/btw <side question>` usage placeholder that markdown/plain-text channel paths can strip.

## Best Possible Solution

Use a non tag-shaped `/btw` usage placeholder at the producer, add regression coverage proving the delivered/sanitized usage keeps the placeholder visible, and add a user-facing changelog entry.

## Reproduction Assessment

Yes. On current main, `/btw` without a question reaches the empty-question branch returning `BTW_USAGE`, and the current sanitizer path strips `<side question>` from that text.

## Solution Assessment

Yes. A producer-owned usage-string fix is the narrowest maintainable path; changing the global sanitizer would risk the existing autolink and HTML-stripping contract, and backticks alone are not enough if the sanitizer still sees `<side question>`.
