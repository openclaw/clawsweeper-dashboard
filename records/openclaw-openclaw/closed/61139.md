## Summary

Current `main` has a dedicated Kimi Coding provider path that rewrites Kimi's tagged Anthropic-message tool output into structured OpenClaw tool calls, includes targeted `k2p5` regression tests, and this shipped in `v2026.4.23`.

## Evidence

- **Kimi stream wrapper rewrites tagged tool output into real tool calls:** `parseKimiTaggedToolCalls` recognizes Kimi's `<|tool_calls_section...|>` markup, converts it into structured `toolCall` blocks, and `rewriteKimiTaggedToolCallsInMessage` flips `stopReason` from `stop` to `toolUse`, which is the missing dispatch handoff needed for tool execution instead of hanging.
  - file: [extensions/kimi-coding/stream.ts:75](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/extensions/kimi-coding/stream.ts#L75)
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Bundled Kimi provider always applies the wrapper:** The bundled `kimi` provider on current `main` registers `wrapKimiProviderStream`, so the Kimi-specific tool-call rewrite is applied on the provider path used by `kimi/...` models rather than relying on the generic transport path alone.
  - file: [extensions/kimi-coding/index.ts:98](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/extensions/kimi-coding/index.ts#L98)
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Regression tests cover Anthropic-messages k2p5 tool dispatch:** Targeted tests exercise `api: "anthropic-messages"`, `provider: "kimi"`, `id: "k2p5"` and assert that tagged `functions.read` and multi-tool payloads become structured tool calls; companion tests also verify Kimi-specific thinking-field normalization for this transport.
  - file: [extensions/kimi-coding/stream.test.ts:35](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/extensions/kimi-coding/stream.test.ts#L35)
  - sha: [59523e66da03](https://github.com/openclaw/openclaw/commit/59523e66da03591bc7bcc183a11f690fa0aae641)
- **Docs explicitly preserve the reporter's legacy model id:** The Kimi Coding docs now state that legacy model ref `kimi/k2p5` remains accepted as a compatibility id, matching the model reported in this issue.
  - file: [docs/providers/moonshot.md:188](https://github.com/openclaw/openclaw/blob/59523e66da03591bc7bcc183a11f690fa0aae641/docs/providers/moonshot.md#L188)
