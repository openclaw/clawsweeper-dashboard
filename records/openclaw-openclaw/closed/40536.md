## Summary

This issue should close as a duplicate of open issue #8441. Current main still lacks per-skill model override support, but #8441 already tracks the same remaining `skills.entries.<skill>.model` work, plus the closely related `thinking` override, and prior implementation attempts (#8455 and #30488) are closed unmerged.

## Best Possible Solution

Close this issue as a duplicate and keep #8441 as the canonical design and implementation thread. The implementation should add a validated `skills.entries.<skill>.model` contract, define precedence against explicit user/session/agent defaults, thread the resolved skill key through invocation, apply the override before the model run, and update tests plus public skills config docs.
