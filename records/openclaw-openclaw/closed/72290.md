The PR adds a content-hash fallback to inbound dedupe, adds an idempotencyKey to chat.send user transcript update payloads, and changes Control UI final/aborted chat event ordering with a regression test.

## Best Possible Solution

Implement a narrow current-main gateway/auto-reply fix for #72176 with reachable retry-safe dedupe semantics and focused tests; keep the Control UI render-order fix in #72004 unless a maintainer intentionally chooses this PR as the canonical UI vehicle.
