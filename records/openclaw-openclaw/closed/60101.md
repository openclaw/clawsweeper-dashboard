## Summary

Current `main` and shipped `v2026.4.23` already preserve Ollama `tool_calls` alongside streamed `thinking`/`reasoning`, so the reported adapter-drop path does not match the checked-in implementation.

## Evidence

- **Streaming loop keeps tool calls after thinking:** The native Ollama stream path reads `message.thinking`/`message.reasoning`, then separately accumulates `message.tool_calls` before breaking on `done`, and copies the accumulated tool calls onto the final response object.
  - file: [extensions/ollama/src/stream.ts:740](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/ollama/src/stream.ts#L740)
  - command: `nl -ba extensions/ollama/src/stream.ts | sed -n '730,855p'`
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Final assistant message preserves thinking plus toolUse:** `buildAssistantMessage()` adds a thinking block when present, appends tool calls, and sets `stopReason` to `toolUse` when any tool calls exist.
  - file: [extensions/ollama/src/stream.ts:519](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/ollama/src/stream.ts#L519)
  - command: `nl -ba extensions/ollama/src/stream.ts | sed -n '500,560p'`
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Regression tests cover the shipped chunk shapes:** Tests explicitly cover tool calls arriving in an intermediate `done:false` chunk and thinking-only streaming that survives into the final assistant message.
  - file: [extensions/ollama/src/stream-runtime.test.ts:497](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/ollama/src/stream-runtime.test.ts#L497)
  - command: `nl -ba extensions/ollama/src/stream-runtime.test.ts | sed -n '466,567p'`
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Thinking-path behavior is already shipped:** The same Ollama streaming logic is present in the latest release commit `a9797214`, including the thinking accumulation and intermediate `tool_calls` handling.
  - file: [extensions/ollama/src/stream.ts:740](https://github.com/openclaw/openclaw/blob/a9797214338ba31c52c796adbb75afb16e0684a9/extensions/ollama/src/stream.ts#L740)
  - command: `git show a9797214338ba31c52c796adbb75afb16e0684a9:extensions/ollama/src/stream.ts | sed -n '740,840p'`
  - sha: [a9797214338b](https://github.com/openclaw/openclaw/commit/a9797214338ba31c52c796adbb75afb16e0684a9)
- **Reported raw trace matches the handled code path:** The April 7, 2026 comment from `@twistdroach` shows thinking deltas in multiple `done:false` chunks, then a later `done:false` chunk carrying `message.tool_calls`, then a final empty `done:true` chunk. That is exactly the shape the current parser handles.
