## Summary

Keep open. Current main does not implement `OPENCLAW_WRAPPER` or a `--wrapper` install option, and the daemon install path still derives LaunchAgent/systemd commands from the Node/Bun entrypoint resolver. The PR addresses a real core daemon/service-install gap from #69400, though it needs changes for installer-env persistence and executable validation before it is merge-ready.

## Best Possible Solution

Keep PR #69425 open and revise it in the core daemon install path: add env-aware wrapper resolution shared by gateway and node service plans, validate that the wrapper is a regular executable file, decide how `OPENCLAW_WRAPPER` or a `--wrapper` option is persisted through reinstall/update, and add focused tests for gateway/node plus LaunchAgent/systemd behavior. #69400 should remain the linked problem statement rather than replacing this PR.
