## Summary

Close as implemented on current main: the hard-coded Control UI chat session active-window bug is fixed by removing the chat picker recency/current-agent request filters while keeping bounded configured-agent discovery. The fix is not in v2026.5.20, but it is present on main with regression coverage.

## What This Changes

The issue reports that Control UI chat/session selection hides valid older cron-backed sessions because chat refreshes request only recently active sessions.

## Best Possible Solution

Keep the current-main chat picker request shape: recency-free and not current-agent-filtered, with bounded configured-agent discovery; revisit the explicit hide-cron default only as a separate UX decision if maintainers want cron sessions shown by default.

## Reproduction Assessment

Yes for the shipped regression: v2026.5.20 source still sends activeMinutes:120 from chat refreshes. No current-main failing path remains because 8df350030dd03fc726a2fc3afaa877071f482cb3 sets the chat picker active window to 0 and loadSessions omits activeMinutes when the value is not positive.

## Solution Assessment

Yes. Removing the recency/current-agent request filters at the chat picker boundary while preserving limit and configuredAgentsOnly is the narrow maintainable fix for hidden older sessions without making the picker an unbounded full-history scan.
