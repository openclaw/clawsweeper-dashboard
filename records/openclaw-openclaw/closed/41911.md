## Summary

Current OpenClaw already contains the memory-search protections that match this report: first-search bootstrap after a fresh/empty index, FTS-only fallback when embeddings are unavailable, and relaxed hybrid scoring so exact keyword hits are not dropped. The same implementation is present in the released `v2026.4.22` tag.

## Evidence

## Close Comment

Closing from a Codex review because this behavior is already implemented in current OpenClaw, and the same implementation is present in released `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`). The active search path now bootstraps a fresh/empty index before returning no results (`extensions/memory-core/src/memory/manager.ts:300`), keeps degraded FTS-only search working when embeddings are unavailable (`extensions/memory-core/src/memory/manager.ts:341`), and preserves exact keyword hits that would otherwise be filtered out by hybrid `minScore` (`extensions/memory-core/src/memory/manager.ts:420`). Committed regression coverage for these paths is in `extensions/memory-core/src/memory/index.test.ts:387` and `extensions/memory-core/src/memory/manager.fts-only-reindex.test.ts:99`. If you still hit this on `v2026.4.22` or newer, please open a new issue with a minimal repro plus `openclaw memory status --deep` output so we can isolate what is different in that environment.
