## Summary

Keep open. Current main and the latest release still allow gateway-started Feishu accounts to reach the per-account bot identity probe concurrently, while the focused closing PR is still open rather than merged.

## What This Changes

The issue reports Feishu multi-account startup bot identity probes timing out because several accounts request Feishu tenant tokens and bot info concurrently.

## Best Possible Solution

Review, refresh, and land the focused Feishu startup serialization fix in https://github.com/openclaw/openclaw/pull/63481 or an equivalent narrow replacement, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes from source, not from a live Feishu setup: gateway startup can still run multiple Feishu account starts concurrently, and each per-account path can call fetchBotIdentityForMonitor without a shared queue. The external API timeout pattern is supported by the report logs but was not live-tested here.

## Solution Assessment

Yes, the best fix shape is a plugin-owned serialization or equivalent concurrency guard around the shared Feishu startup identity probe, because both gateway per-account startup and monitor recovery call that helper. Current main only serializes the all-accounts monitor path, which is not enough for the reported gateway path.
