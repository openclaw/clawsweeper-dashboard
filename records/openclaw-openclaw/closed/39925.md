## Summary

Current `main` already addresses this multi-agent auth failure mode: `openclaw agents add <id>` can copy the default agent's `auth-profiles.json` into the new agent and can also launch auth setup for that new agent, while the runtime error and docs now explicitly explain that auth is per-agent.

## Evidence

- **Agent add now offers auth-profile copy:** The interactive `agents add` flow checks the default agent's auth store and prompts to copy it into the new agent when the destination store is empty.
  - file: [src/commands/agents.commands.add.ts:247](https://github.com/openclaw/openclaw/blob/86fa8eeb68c64eb0ae25e1eb8c6196061862a010/src/commands/agents.commands.add.ts#L247)

  - file: [src/commands/agents.commands.add.ts:269](https://github.com/openclaw/openclaw/blob/86fa8eeb68c64eb0ae25e1eb8c6196061862a010/src/commands/agents.commands.add.ts#L269)
