## Summary

Close as duplicate; the requested output-verification hook is a specific use case of the open post-subagent completion/pre-announcement API design tracked at https://github.com/openclaw/openclaw/issues/22358.

## What This Changes

Request adding an optional `sessions_spawn` `postCompletionHook` that runs a verifier script after subagent completion and can abort or warn before delivery.

## Best Possible Solution

Fold the output-verification use case into the canonical post-subagent completion hook/API design at https://github.com/openclaw/openclaw/issues/22358, with explicit trust and delivery semantics.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main lacks `postCompletionHook`, and the same post-completion/pre-announcement hook gap is already tracked in the canonical open issue.

## Solution Assessment

No, a separate `sessions_spawn` script hook is not clearly the best core API. The safer path is to route this use case through the existing maintainer API design thread for a typed plugin/lifecycle contract.
