## Summary

Current main and the latest release still drop `memorySearch.outputDimensionality` before OpenAI inline and batch embedding requests, and an open unmerged closing PR already targets the fix, so this issue should stay open.

## What This Changes

Honor `memorySearch.outputDimensionality` for OpenAI memory embeddings so configured OpenAI `dimensions` affects vector size and cache/reindex identity.

## Best Possible Solution

Update the OpenAI memory embedding provider to carry the resolved dimension through inline requests, batch JSONL bodies, and cache identity, with focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction follows the accepted `memorySearch.outputDimensionality` field through config resolution and memory-core provider requests, then shows the OpenAI inline and batch request builders never send `dimensions`.

## Solution Assessment

Yes. The narrow maintainable solution is provider-local: thread the resolved dimension into the OpenAI embedding client, request bodies, and provider cache key without changing unrelated providers or adding a new config surface.
