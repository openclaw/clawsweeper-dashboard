## Summary

Keep open: current main and the latest release still finalize and persist completed voice-call records without a Telegram/session notification path. The prior same-gap report was closed stale, and the broader outbound task-call issue does not supersede this narrower post-call delivery gap.

## What This Changes

The issue requests a post-call notification or summary delivery path for completed inbound voice calls with persisted transcripts.

## Best Possible Solution

Define an opt-in voice-call post-completion contract that can summarize transcript-bearing calls, deliver to an approved session or configured channel, surface delivery failures, and document the behavior with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows completed inbound/realtime calls persist transcript-bearing records and finalize through call.ended without any notification path, and the reporter provided live Twilio plus Telegram evidence; no live call was run in this read-only review.

## Solution Assessment

Unclear until maintainers choose the delivery policy. A plugin-owned post-call delivery contract is the maintainable direction; a Telegram-only special case or external calls.jsonl scanner would be brittle.
