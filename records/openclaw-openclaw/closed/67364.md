## Summary

Keep open. Current main still does not fix the reported Control UI path: heartbeat runs build a normal user-turn body without `InputProvenance`, `chat.history` still sanitizes only assistant-side control replies, and Control UI still loads `chat.history` then maps any surviving `role: "user"` row to the local user label. The linked discussion correctly points to the deeper transcript-shape bug in #66496, and the open PR #68262 describes the right provenance-shaped direction, but that fix is not present in this checkout.

## Best Possible Solution

Keep the issue open and land a provenance-based fix: heartbeat/cron/exec-event runs should set `InputProvenance.kind = "internal_system"` at origin, transcript persistence should retain that provenance, `chat.history` should omit or classify those rows, and Control UI should keep a matching defense-in-depth filter. Add regression tests for heartbeat prompt plus `HEARTBEAT_OK` on the actual `chat.history`/Control UI path, and coordinate with #66496/#68262 so the backend transcript shape and UI symptom close together.
