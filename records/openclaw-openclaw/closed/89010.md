## What This Changes

The PR lowers the Control UI tweakcn theme ID minimum from 8 to 4 characters and adds regression tests for short raw IDs, registry/editor URLs, and malformed short IDs.

## Best Possible Solution

Land the narrow validator and regression-test fix after normal maintainer and CI gates, keeping the existing bounded tweakcn fetch/schema/CSS sanitization path as the single import path.

## Reproduction Assessment

Yes by source inspection: current main rejects 4-7 character tweakcn IDs before fetch, while docs and the UI import path accept raw/default theme IDs. Live registry checks confirmed at least `claude` and `twitter` are valid short IDs returning JSON.

## Solution Assessment

Yes. Changing the central validator and adding focused tests is the smallest maintainable fix; special-casing built-in theme names or adding a second import path would create drift.
