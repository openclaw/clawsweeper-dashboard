## Summary

The request is a real workflow need, but the evidence points to a repo-specific DishPics orchestration layer that should live as a ClawHub/plugin/skill extension rather than new OpenClaw core behavior. Current main already provides the generic ACP spawn/task-tracking primitives; what is missing is custom repo prep and registry glue.

## Evidence

- **Current `openclaw tasks` surface is inspection/maintenance only:** The public CLI docs list `list`, `show`, `notify`, `cancel`, `audit`, `maintenance`, and `tasks flow {list,show,cancel}`. There is no native `tasks start` or repo-bootstrap command on current main.

- **Command registry confirms no start subcommand:** The CLI wiring registers `maintenance`, `show`, `notify`, `cancel`, and `flow {list,show,cancel}` under `tasks`; there is no `start` handler to bridge repo prep, worktree creation, ACP spawn, and registry writes.
