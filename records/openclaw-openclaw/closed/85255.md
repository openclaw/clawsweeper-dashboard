## Summary

Keep open: current `main` still has the reported locale blind spot in the Windows Scheduled Task fallback path, while the docs promise a Startup-folder fallback when task creation is denied.

## What This Changes

The issue reports that localized Windows `schtasks /Create` access-denied output is not recognized, so gateway install/update throws instead of using the Startup-folder login item.

## Best Possible Solution

Make the `/Create` fallback locale-independent for denied or blocked Scheduled Task creation, add non-English regression coverage, and preserve existing behavior for non-create failures such as `/Run` errors.

## Reproduction Assessment

Yes, at source level: code 1 with `错误: 拒绝访问。` reaches the create-failure branch but does not satisfy the current fallback predicate. I did not run a live Chinese Windows install in this read-only review.

## Solution Assessment

No, the current locale-specific predicate is not the best fix because it requires adding languages one by one. A narrow locale-independent check for the `/Create` fallback path, with localized regression tests, is more maintainable.
