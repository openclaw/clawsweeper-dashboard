## Summary

Current main still has the reported schema mismatch: the runtime writes/preserves `source: "allow-always"` on exec allowlist entries, while the gateway protocol schema for `exec.approvals.set` rejects unknown allowlist fields. The related comments correctly identify the root cause, and open PRs such as #60064/#60073 appear to be fix attempts rather than shipped replacements.

## Best Possible Solution

Keep the issue open and land a protocol-compatible fix: add `source: Type.Optional(Type.Literal("allow-always"))` to `ExecApprovalsAllowlistEntrySchema`, keep `commandText` stripped by normalization unless maintainers intentionally want it on the wire, align the Control UI type, and add a gateway protocol validator regression test for `exec.approvals.set` and `exec.approvals.node.set` with allow-always metadata.
