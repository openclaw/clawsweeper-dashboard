## Summary

Current `main` already contains the Telegram/SSRF transport work needed for this report: guarded hostname resolution now uses `dns.lookup`-style resolution with IPv4 preference, pinned dispatchers preserve caller transport hints such as `autoSelectFamily`, Telegram media downloads reuse the same direct/proxy/IPv4 fallback policy as API calls, and the fetch guard now allows trusted explicit proxies after hostname-policy checks.

## Evidence

- **SSRF hostname pinning uses lookup results and prefers IPv4:** `resolvePinnedHostnameWithPolicy` resolves via `lookupFn(..., { all: true })`, validates the answers, and prefers IPv4-first address ordering before creating the pinned lookup.
  - file: [src/infra/net/ssrf.ts:365](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/infra/net/ssrf.ts#L365)
- **Pinned dispatchers preserve transport hints instead of discarding them:** `createPinnedDispatcher` merges the pinned lookup into existing `connect` or proxy TLS options; the regression test asserts `autoSelectFamily` and `autoSelectFamilyAttemptTimeout` survive the override.
  - file: [src/infra/net/ssrf.dispatcher.test.ts:88](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/infra/net/ssrf.dispatcher.test.ts#L88)
- **Telegram media/API transport now shares direct, IPv4-only, and pinned-IP fallback policies:** Telegram builds dispatcher policies from `channels.telegram.network` settings and exposes a three-step fallback chain for media/API fetches: default transport, sticky IPv4-only retry, then pinned Telegram IP retry.
  - file: [extensions/telegram/src/fetch.ts:159](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/extensions/telegram/src/fetch.ts#L159)
- **Trusted explicit proxy DNS is now allowed for Telegram media downloads after hostname checks:** `fetchWithSsrFGuard` has an explicit trusted-proxy path, and the regression test covers `https://api.telegram.org/file/...` succeeding through `http://localhost:6152` with `allowPrivateProxy: true`.
  - file: [src/infra/net/fetch-guard.ssrf.test.ts:1084](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/infra/net/fetch-guard.ssrf.test.ts#L1084)
- **Release notes record the shipped Telegram media proxy fix:** The changelog entry under `2026.4.14` says Telegram media downloads now trust an operator-configured explicit proxy for target DNS resolution after hostname-policy checks.
  - file: [CHANGELOG.md:658](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/CHANGELOG.md#L658)
