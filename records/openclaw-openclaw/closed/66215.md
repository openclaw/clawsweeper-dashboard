## Summary

Current `main` no longer hands Telegram a transient workspace file for generated images. The image-generation path now persists outputs into the OpenClaw media store and emits those saved paths for delivery, which addresses the workspace-cleanup race described in #66215. This landed on `main` after the report and after `v2026.4.23`, so it appears fixed on `main` but not yet in a tagged release.

## Evidence

- **Generated images are persisted before delivery:** `image_generate` now saves each generated buffer via `saveMediaBuffer(..., "tool-image-generation", ...)` and emits the saved path in both `MEDIA:` text and `details.media.mediaUrls`, so Telegram receives a persisted file path rather than a transient workspace path.
  - file: [src/agents/tools/image-generate-tool.ts:725](https://github.com/openclaw/openclaw/blob/628f0e80553fe6d0d1d755eb2fef4679b0e2732b/src/agents/tools/image-generate-tool.ts#L725)
  - command: `nl -ba src/agents/tools/image-generate-tool.ts | sed -n '720,757p'`
  - sha: [628f0e80553f](https://github.com/openclaw/openclaw/commit/628f0e80553fe6d0d1d755eb2fef4679b0e2732b)
- **Saved path is under the media store, not the workspace:** `saveMediaBuffer` writes to `path.join(resolveConfigDir(), "media", subdir)`, creating a persistent media-store location for generated outputs. That does not match the issue's old `~/.openclaw/workspace/...` path shape.
  - file: [src/media/store.ts:15](https://github.com/openclaw/openclaw/blob/628f0e80553fe6d0d1d755eb2fef4679b0e2732b/src/media/store.ts#L15)

- **Telegram still reads the provided path at send time:** Telegram reply delivery calls `loadWebMedia(mediaUrl, buildOutboundMediaLoadOptions(...))` immediately before upload, so the fix is specifically that the upstream tool now provides a durable stored path.
  - file: [extensions/telegram/src/bot/delivery.replies.ts:273](https://github.com/openclaw/openclaw/blob/fe930b987e33f4e1acd93b5ef1ea76c8b7260047/extensions/telegram/src/bot/delivery.replies.ts#L273)

That means the reported failure mode would still surface if a path vanished, but the current image-generation path no longer uses the transient workspace file pattern from the report.
