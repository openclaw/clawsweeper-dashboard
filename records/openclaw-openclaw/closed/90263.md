## Summary

Keep open for maintainer review: the Discord hydration fix is well scoped and source-supported, but the PR also adds two documented public plugin hook fields that should be explicitly accepted as API contract before merge.

## What This Changes

This PR hydrates Discord reply messages missing `referenced_message`, threads `replyToIdFull` and `replyToIsQuote` through plugin hook payloads, and updates hook docs and tests.

## Best Possible Solution

Land the hydration fix and additive hook fields only after a maintainer accepts `replyToIdFull` and `replyToIsQuote` as stable public plugin hook metadata.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, source-reproducible: current main builds Discord quote context only from `message.referencedMessage`, while the Discord type contract allows reply payloads with `message_reference` but no `referenced_message`.

## Solution Assessment

Is this the best way to solve the issue? Yes for the hydration path: fetching the current reply message through Discord's existing channel-message route is narrower than directly fetching the referenced target and keeps existing visibility handling; the additive hook fields still need maintainer API acceptance.
