## Summary

PR #52592 is superseded. Its linked Feishu status repro (#52572) has already been closed against current main, and the proposed patch still has unresolved correctness gaps. Current main does not include registry-side tool dedupe, so the remaining per-plugin tool-registration work should stay with the newer canonical PR #56574.

## Best Possible Solution

Close #52592 as superseded. Keep current main's metadata/snapshot and runtime-registry fixes, and review or rebase #56574 for any remaining per-plugin tool-registration dedupe and factory-context behavior.

## Evidence

- **Current main still has append-only registerTool behavior:** `registerTool()` normalizes any explicit names and unconditionally appends a new `registry.tools` entry. That means #52592 was not implemented on main as a registry-level dedupe fix, and any remaining dedupe work needs a live canonical thread rather than this older flawed patch.

  - sha: [06d409dc2738](https://github.com/openclaw/openclaw/commit/06d409dc2738034f6005ede426726e830ea48679)
- **PR review identified unresolved correctness gaps:** The PR discussion notes that a `registry.tools` guard is local to one registry instance, so it cannot cover cross-registry startup reloads, and that name-based dedupe misses factory registrations that omit explicit `name`/`names` options. The author acknowledged the cross-registry concern and left it for maintainer judgment.
  - sha: [783a3b165754](https://github.com/openclaw/openclaw/commit/783a3b165754c76d3971ab619bac4b96b95f448f)

The PR's dedupe key is built before factory execution from explicit `opts.name`/`opts.names`, so it would not cover this class of Feishu tool registration.

This supports the prior close of linked issue #52572 as not reproducible for `status --verbose`.
  - file: [src/plugins/status.ts:207](https://github.com/openclaw/openclaw/blob/06d409dc2738034f6005ede426726e830ea48679/src/plugins/status.ts#L207)
