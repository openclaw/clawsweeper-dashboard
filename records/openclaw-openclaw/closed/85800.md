## Summary

Keep open: current main and v2026.5.22 already support a provider-owned Perplexity default model config, but the requested per-call model escalation is still absent and needs an explicit API/product decision.

## What This Changes

Allow `web_search` users to configure the Perplexity model default and optionally override the model per tool call.

## Best Possible Solution

Keep the shipped plugin-owned default model config, then decide whether per-call escalation should be a provider-scoped tool parameter, a named search preset, or intentionally unsupported.

## Reproduction Assessment

Yes. Source inspection shows the current generic and Perplexity-specific `web_search` schemas do not accept a per-call `model` argument, while plugin config covers only the default.

## Solution Assessment

Unclear. The provider-owned default config is the maintainable owner boundary for the shipped default, but a per-call model override would be a public tool/API contract decision.
