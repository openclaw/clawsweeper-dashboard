## Summary

Current main now implements the intended Agents model-selector behavior with a safer shape than this PR: the default agent selects the configured primary model on initial render, inherited non-default agents keep the empty inherited placeholder, and missing explicit models retain the Current label. The PR branch is obsolete and still carries the inherited-default selection regression called out in review.

## What This Changes

The PR changes the Agents UI model-option helper to fall back to `agents.defaults.model.primary`, mark matching options selected, and prepend a missing configured model option labeled as Default.

## Best Possible Solution

Close this PR and keep the current main implementation, which preserves the distinction between available/effective model options and the actually selected value. Any further selector regressions should be handled as focused follow-up tests or fixes against the current Agents Overview and shared config-form select paths.

Full review comments:

- **[P2] Preserve inherited model placeholder selection:** `ui/src/ui/views/agents-utils.ts:600`
  - body: For non-default agents without an explicit model, the surrounding `<select>` intentionally has value `""` so it shows `Inherit default (...)`. Marking the effective default option as selected here can override that empty selection and make an inherited setting look like an explicit concrete model.
  - confidence: 0.87
- **[P3] Keep missing explicit models labeled as current:** `ui/src/ui/views/agents-utils.ts:591`
  - body: When `current` is an agent-specific model missing from the option list, this labels it as `Default (...)`. That regresses the previous `Current (...)` label and misrepresents an explicit per-agent selection as the global default.
  - confidence: 0.82
