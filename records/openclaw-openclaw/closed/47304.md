## Summary

Current main already contains the google-vertex ADC handling needed for this report, and the latest shipped release `v2026.4.22` includes it as well. OpenClaw now resolves ADC for `google-vertex` to a synthetic non-secret marker instead of a real API key, and the bundled `@mariozechner/pi-ai` `0.70.0` explicitly ignores both that marker and placeholder values like `"<authenticated>"`, falling back to ADC/project/location auth instead of Vertex API-key auth.

## Evidence

- **OpenClaw resolves google-vertex ADC to a marker, not a literal key:** `resolveEnvApiKey("google-vertex")` returns `GCP_VERTEX_CREDENTIALS_MARKER` when project + location + ADC credentials are present, with source `gcloud adc`.
  - file: [src/agents/model-auth-env.ts:27](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/model-auth-env.ts#L27)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Marker is classified as non-secret auth:** `GCP_VERTEX_CREDENTIALS_MARKER` is `gcp-vertex-credentials`, and `isNonSecretApiKeyMarker(...)` treats it as a synthetic marker rather than a real credential.
  - file: [src/agents/model-auth-markers.ts:9](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/model-auth-markers.ts#L9)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Regression coverage exists for google-vertex ADC env snapshots:** Tests assert that `resolveEnvApiKey("google-vertex")` accepts ADC credentials from an env snapshot and resolves to `gcp-vertex-credentials` with source `gcloud adc`.
  - file: [src/agents/model-auth.profiles.test.ts:842](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/src/agents/model-auth.profiles.test.ts#L842)
  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
- **Bundled pi-ai version is newer than the broken one from the report:** OpenClaw ships `@mariozechner/pi-ai` `0.70.0`, not the older `0.58.0` mentioned in the issue comment.
  - file: [package.json:1585](https://github.com/openclaw/openclaw/blob/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5/package.json#L1585)
