## Label Justifications

- `P1`: The report describes a broken Anthropic agent workflow where one MCP tool result can make later session turns fail until history is reset.
- `impact:session-state`: The malformed or unsupported tool-result block is persisted in conversation history and can keep poisoning future provider requests.

## Summary

Keep open: current main still lets MCP content blocks wider than OpenClaw's text/image tool-result contract reach the Anthropic conversion paths, where unsupported blocks are dropped in text-only payloads and can be coerced into malformed image blocks in mixed image payloads.

## What This Changes

The issue asks OpenClaw to safely convert MCP `resource_link`, `resource`, `audio`, and unknown tool-result blocks before sending Anthropic tool-result content.

## Best Possible Solution

Normalize MCP tool-result content at the MCP materialization boundary into OpenClaw-supported text/image blocks, and add defensive Anthropic conversion coverage so existing resource/audio blocks in session history cannot become empty or malformed provider payloads.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source inspection gives a high-confidence reproduction path: raw MCP `resource_link`/`resource`/`audio` content can be cast into tool-result content and then mishandled by the Anthropic converters; a live Anthropic 400 was not run in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? The requested provider-side guard is necessary for bad existing histories, but the best fix should also normalize raw MCP content in `toAgentToolResult` so unsupported blocks do not enter the runtime's text/image-only tool-result contract.

Normalize `CallToolResult.content` in `src/agents/agent-bundle-mcp-materialize.ts` before it becomes `AgentToolResult.content`: keep valid text blocks, keep valid image blocks only when `data` and image `mimeType` are present, convert MCP `resource_link` to safe text using title/name/uri, convert embedded `resource` to safe text using text or URI/blob metadata, convert `audio` to a short text marker with MIME when present, and stringify or summarize unknown public blocks without leaking huge binary payloads. Add focused tests for pure resource_link, mixed image+resource_link, resource, audio, and valid image behavior; preserve existing text/image behavior and do not add config, fallback knobs, changelog edits, or provider-specific hardcoded service names.
