## Summary

Current main now persists sub-agent model selections as the runtime override fields (`modelOverride`, `providerOverride`, `modelOverrideSource`) before the child run starts, and the reply runtime selects models from those override fields. The report matches older shipped behavior, but the current main tree fixes the observable problem for explicit `sessions_spawn.model` and configured sub-agent model defaults.

## Best Possible Solution

Close as implemented on main, keep the current override-field persistence path, and ship it in the next release so users on 2026.4.25/2026.4.26 receive the fix.
