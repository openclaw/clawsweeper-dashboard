## Summary

Keep open. Current `main` has related remote-CDP hardening for availability checks and stop-time cleanup, but I did not find a fix for the reported failure mode where a cached Playwright browser/context survives long enough to make `browserContext.newPage()` throw after a remote browser restart.

## Evidence

- **Tab creation still uses cached Playwright connection/context without retry:** `createPageViaPlaywright` gets the cached browser via `connectBrowser()`, reuses `browser.contexts()[0]`, and calls `context.newPage()` directly. There is no catch/reconnect/retry path around `newPage()` here.
  - file: [extensions/browser/src/browser/pw-session.ts:1122](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/extensions/browser/src/browser/pw-session.ts#L1122)
  - command: `nl -ba extensions/browser/src/browser/pw-session.ts | sed -n '1112,1127p'`
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Cache invalidation depends on transport disconnect, not stale-context errors:** `connectBrowser` returns a cached browser when present and only drops that cache on the browser `disconnected` event. If the remote restart leaves OpenClaw holding a stale browser/context handle until the next `newPage()` call fails, this path does not recover it.
  - file: [extensions/browser/src/browser/pw-session.ts:430](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/extensions/browser/src/browser/pw-session.ts#L430)
  - command: `nl -ba extensions/browser/src/browser/pw-session.ts | sed -n '428,466p'`
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Retry-on-disconnect exists for navigation, not for tab creation:** `navigateViaPlaywright` explicitly catches retryable detached/closed-target errors, force-disconnects Playwright, and retries once. I did not find equivalent recovery around remote tab creation.
  - file: [extensions/browser/src/browser/pw-tools-core.snapshot.ts:240](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/extensions/browser/src/browser/pw-tools-core.snapshot.ts#L240)
  - command: `nl -ba extensions/browser/src/browser/pw-tools-core.snapshot.ts | sed -n '238,253p'`
  - sha: [b45329334965](https://github.com/openclaw/openclaw/commit/b453293349657dedd069d291fa5b078d910067b1)
- **Shipped changelog entries are adjacent but narrower than this report:** The changelog mentions retrying remote-CDP websocket reachability after restart and disconnecting cached Playwright CDP sessions when stopping attach-only/remote profiles, but nothing here shows automatic recovery from `browserContext.newPage()` on a stale cached remote context after an external browser restart.
  - file: [CHANGELOG.md:1154](https://github.com/openclaw/openclaw/blob/b453293349657dedd069d291fa5b078d910067b1/CHANGELOG.md#L1154)
