## Summary

Current main addresses the v2026.3.12 low-memory startup/OOM regression without adding the exact Dockerfile NODE_OPTIONS workaround: startup now uses built plugin artifacts instead of source plugin trees in packaged/Docker paths, the 2026.4.15 release slimmed packaged plugin/runtime payloads, and release/install checks now block a repeat of the oversized 2026.3.12 package that was explicitly tied to low-memory startup/doctor OOM reports. This also matches the already-closed duplicate #45962.
