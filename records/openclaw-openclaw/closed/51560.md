## Summary

Current `main` has already landed part of this PR's intent: ACP config no longer hard-allowlists `discord|telegram|feishu`, tests now accept arbitrary channel ids, and plugin-facing routing helpers are exported from the public conversation runtime. But the feature is not fully implemented for "any channel": configured ACP bindings still only work for channel plugins that provide a `bindings` adapter, and several plugin channels called out by the PR rationale are still missing that surface. That is not strong enough evidence to close this PR as implemented.
