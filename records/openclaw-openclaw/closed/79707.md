## Summary

Current main already honors `agents.defaults.imageModel` before the active DeepSeek text model in both the explicit image tool and automatic media-understanding path, with regression coverage for the routing behavior.

## What This Changes

The issue requests that image analysis route to the configured `agents.defaults.imageModel` even when the primary conversation model is a non-image provider such as DeepSeek.

## Best Possible Solution

Keep the current imageModel-first routing; if a current release still fails, handle it as a new report with the exact image-tool error, resolved model id, provider auth state, and logs.

## Reproduction Assessment

No, current main does not show a failing reproduction path: source and regression tests show `agents.defaults.imageModel` is selected before the DeepSeek primary/active model for image routing. I did not run live provider calls, but the routing failure described here is contradicted by the current code path.

## Solution Assessment

Yes, the current implementation is the narrow maintainable solution: it resolves the configured image model at the image-tool and media-understanding boundaries and passes the qualified provider/model into the fallback runner without adding another config knob.
