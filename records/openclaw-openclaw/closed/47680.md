## Summary

Current `main` already avoids the reported `sessions_send` serialization by routing nested agent work onto per-target-session lanes instead of the single global `nested` lane, so this PR's `CommandLane.Nested` concurrency patch is stale against the current architecture.

## Evidence

- **`sessions_send` uses a session-scoped nested lane:** Current `sessions_send` dispatch sets `lane: resolveNestedAgentLaneForSession(resolvedKey)`, so sends to different target sessions no longer share one global lane.
  - file: [src/agents/tools/sessions-send-tool.ts:281](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/agents/tools/sessions-send-tool.ts#L281)
