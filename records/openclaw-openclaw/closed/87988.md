## Best Possible Solution

Land this as a narrow WhatsApp hot-reload fix after maintainers either accept broad `channels.whatsapp.*` restart semantics or ask for a narrower account-enabled trigger, and after current-head after-fix proof is added.

## Solution Assessment

Unclear as the final merge shape. Removing the noop prefix is the smallest source fix for the reported disabled-account path, but maintainers need to decide whether all WhatsApp config changes should restart active sessions or whether the trigger should be narrower.
