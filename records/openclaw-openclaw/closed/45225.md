## Summary

Current `main` already implements the underlying workspace cleanup this issue asks for: workspace files are separated from config/credentials/session state, agents can have isolated workspaces and agent state directories, and shared skills already live in standardized shared roots.

## Evidence

- **Workspace is already separated from system/state files:** The workspace docs explicitly say the agent workspace is separate from `~/.openclaw/`, which stores config, credentials, and sessions, and list managed skills under `~/.openclaw/skills` as outside the workspace.
  - file: [docs/concepts/agent-workspace.md:12](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/docs/concepts/agent-workspace.md#L12)
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Per-agent isolation is documented and supported:** Multi-agent docs describe each agent as having its own workspace, `agentDir`, auth profiles, and session store, with paths under `~/.openclaw/agents/<agentId>/...`.
  - file: [docs/concepts/multi-agent.md:8](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/docs/concepts/multi-agent.md#L8)
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Per-agent workspace and state paths are implemented in code:** `resolveAgentWorkspaceDir` resolves a configured per-agent workspace or falls back to isolated defaults, and `resolveAgentDir` resolves per-agent state under `agents/<id>/agent`.
  - file: [src/agents/agent-scope-config.ts:149](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/src/agents/agent-scope-config.ts#L149)
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Shared vs per-agent skill locations are already standardized:** Skills docs define a precedence stack with per-workspace skills, project-agent skills, personal-agent skills, and shared managed skills in `~/.openclaw/skills`, including explicit multi-agent sharing guidance.
  - file: [docs/tools/skills.md:13](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/docs/tools/skills.md#L13)
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
- **Existing CLI already manages isolated agents/workspaces:** `openclaw agents add` supports `--workspace` and `--agent-dir`, and the docs show multiple isolated workspace examples such as `~/.openclaw/workspace-work` and `~/.openclaw/workspace-ops`.
  - file: [docs/cli/agents.md:20](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/docs/cli/agents.md#L20)
  - sha: [4630ce3d9e2f](https://github.com/openclaw/openclaw/commit/4630ce3d9e2f089062ccd386b0ae241325961d1a)
