## Summary

Keep open. Current main still collects configured model strings as raw `id` values in both security audit model collectors, then applies weak-tier and small-parameter checks to those raw values. An alias such as `gpt-prev` can therefore be warned on before it is resolved through `agents.defaults.models`, and the existing resolver path is available but not used by the audit collectors.

## What This Changes

Make `openclaw security audit --deep` resolve configured model aliases through `agents.defaults.models` before weak-tier and small-model checks while preserving the original report source paths.

## Best Possible Solution

Add a shared audit model collection/resolution helper that builds the alias index once, resolves every collected model ref with the existing model-selection resolver, evaluates hygiene and parameter-size checks on the canonical provider/model id, and preserves the original source path in reported findings.
