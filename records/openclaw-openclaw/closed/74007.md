## Summary

Keep this PR open. Current main still uses raw `${provider}/${modelId}` in `resolveExtraParams`, so provider-prefixed model IDs such as `openrouter/auto` still miss canonical per-model params. The PR is a narrow replacement for the closed, unmerged source PR #44319 and its diff matches the intended fix by using `modelKey` plus `legacyModelKey` fallback with focused regression coverage. Security review found no unexplained workflow, dependency, install, release, generated, or secret-handling changes in the PR file list.

## Best Possible Solution

Keep this PR open for normal maintainer review and merge if CI, targeted tests, and the pending security check are clean. The best fix is the narrow PR approach: use `modelKey` for canonical extra-params lookup, preserve `legacyModelKey` fallback for existing double-prefixed config entries, keep the `openrouter/auto` regression coverage, and retain the changelog credit for #44319 without adding unrelated workflow or dependency changes.
