## Summary

Current main already implements the reported OpenRouter fallback fix. OpenRouter-specific "Key limit exceeded" is classified as a billing failure, HTTP 401/403 preserves that billing classification over generic auth, billing failures feed the model-fallback path, and the same behavior is present in release v2026.4.22.
