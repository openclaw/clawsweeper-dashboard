## Summary

Current main no longer contains the Pi RPC client that this PR patches. The docs-i18n tool now uses a Codex exec runner with buffered stdio and `exec.Cmd.Run()`, so the partial `StdinPipe`/`StdoutPipe`/`StderrPipe` initialization leak path is gone. The PR diff is also security-neutral: it only touched one Go script file and did not change workflows, dependencies, package resolution, release scripts, or secret handling.

## Best Possible Solution

Close this PR and keep the current main implementation: docs-i18n has moved to the Codex exec runner, so merging the obsolete Pi RPC pipe cleanup is no longer useful. If a pipe-based long-lived client is ever reintroduced, add explicit failure-path cleanup plus a regression test in that owner module.
