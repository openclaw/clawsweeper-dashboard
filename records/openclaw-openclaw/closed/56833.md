## Summary

Current `main` already handles the Discord gateway abort/shutdown path without surfacing the late `Max reconnect attempts ... code 1005` crash. The relevant shutdown/supervisor code and regression tests are present on `main` in commit `68315792677f9628d1370d6869366442f1d3ecad`, but I could not verify a tagged release that shipped it from the local checkout.

## Evidence

- **Abort shutdown keeps lifecycle attached through disconnect:** `waitForDiscordGatewayStop` now calls `gateway.disconnect()` before detaching lifecycle handlers, with an explicit comment that late `error` events during disconnect must still be handled instead of becoming uncaught.
