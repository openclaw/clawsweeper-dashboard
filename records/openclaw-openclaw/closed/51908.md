## Summary

Current `main` no longer matches the reported regex-based pipe-to-shell false positive. The approval path now gates on exec policy, allowlist-plan limits, heredocs, and strict inline-eval, while `||` is parsed as a shell chain operator before pipeline parsing.

## Evidence

- **Current gateway approval logic has no obfuscation regex gate:** `processGatewayAllowlist()` only forces approval via `requiresExecApproval(...)`, unsupported allowlist-plan execution, heredocs, or strict inline-eval; there is no pipe-to-shell regex branch in this path.
