## Summary

Current `main` now has explicit restart recovery for crash-interrupted main sessions: startup maps stale transcript locks back to `running` sessions, then either resumes a resumable `toolResult`/`tool`/`user` tail or marks the session failed so the concurrency slot is freed.

## Evidence

- **In-flight runs still hold a transcript lock for the full attempt:** The embedded runner acquires `acquireSessionWriteLock(...)` before the run proceeds, so a crash during a mid-turn leaves a stale `<session>.jsonl.lock` that startup can detect.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:623](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/agents/pi-embedded-runner/run/attempt.ts#L623)
  - command: `nl -ba src/agents/pi-embedded-runner/run/attempt.ts | sed -n '618,632p'`
  - sha: [dc7c70342549](https://github.com/openclaw/openclaw/commit/dc7c7034254971fa6e9769e99a18da88f432f8c7)
- **Gateway startup now marks interrupted running sessions from stale locks:** On startup, `startGatewaySidecars()` cleans stale session locks and, when any were removed, calls `markRestartAbortedMainSessionsFromLocks(...)`; it also schedules delayed main-session recovery.
  - file: [src/gateway/server-startup-post-attach.ts:149](https://github.com/openclaw/openclaw/blob/dc7c7034254971fa6e9769e99a18da88f432f8c7/src/gateway/server-startup-post-attach.ts#L149)
  - command: `nl -ba src/gateway/server-startup-post-attach.ts | sed -n '149,176p;365,369p'`
  - sha: [dc7c70342549](https://github.com/openclaw/openclaw/commit/dc7c7034254971fa6e9769e99a18da88f432f8c7)
- **Main-session recovery resumes or fails stuck sessions:** `main-session-restart-recovery.ts` marks matching `running` sessions `abortedLastRun: true`, then scans those entries on restart. If the transcript tail ends in `user`/`tool`/`toolResult`, it re-dispatches the same session via `agent` with `deliver: false`; otherwise it marks the session `failed`, which frees the blocked slot instead of leaving it indefinitely `running`.
  - file: [src/agents/main-session-restart-recovery.ts:101](https://github.com/openclaw/openclaw/blob/9ab51bb66e0d75893de49863b8d1bb7fc9442dcd/src/agents/main-session-restart-recovery.ts#L101)
  - command: `nl -ba src/agents/main-session-restart-recovery.ts | sed -n '1,240p'; nl -ba src/agents/main-session-restart-recovery.ts | sed -n '240,320p'`
  - sha: [9ab51bb66e0d](https://github.com/openclaw/openclaw/commit/9ab51bb66e0d75893de49863b8d1bb7fc9442dcd)
- **Regression tests cover the reported recovery shape:** Tests verify that cleaned stale locks mark only main running sessions, that a marked session with a `toolResult` tail is resumed, and that a non-resumable interrupted session is marked `failed`.
  - file: [src/agents/main-session-restart-recovery.test.ts:61](https://github.com/openclaw/openclaw/blob/9ab51bb66e0d75893de49863b8d1bb7fc9442dcd/src/agents/main-session-restart-recovery.test.ts#L61)
