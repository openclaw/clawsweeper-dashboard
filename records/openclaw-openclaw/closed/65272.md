## Summary

Current main already fixes the reported pre-compaction memory-flush context pollution. The flush instructions are still sent to the maintenance run, but with an empty transcript prompt, so the `Pre-compaction memory flush.` text is treated as runtime-only system context instead of a normal persisted user turn that can appear mid-context or in `chat.history`.

## Best Possible Solution

Close this as implemented on main and keep the runtime-only design: the memory flush can still write durable memory, but its maintenance prompt should not be persisted or replayed as a normal user turn. No separate first-position transcript insertion is needed because the contaminating transcript entry is gone.
