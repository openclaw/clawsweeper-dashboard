## Summary

Keep open. Current main and the latest shipped release still drop the already computed outbound `sessionKey` before the plugin `message_sent` event/context mappers run, and the open linked fix PR has not merged yet.

## What This Changes

The issue asks OpenClaw to pass the already available delivery session key into the canonical plugin `message_sent` context.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/73706 or an equivalent narrow patch that forwards `sessionKeyForInternalHooks` into the sent-message canonical context and adds regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: deliver through `deliverOutboundPayloads` with `params.session.key` or `params.mirror.sessionKey`, and the plugin `message_sent` event/context will lack `sessionKey` because the emitter omits it before the mappers run.

## Solution Assessment

Yes. Passing the already computed `sessionKeyForInternalHooks` into `buildCanonicalSentMessageHookContext` is the narrowest maintainable fix because the public types and mapper functions already support the field.
