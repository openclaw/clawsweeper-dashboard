## Summary

Current main already prevents standalone HTML/Cloudflare provider responses from falling through to the DNS transport copy, and the fix was already shipped in `v2026.4.23` with a regression test covering the exact `dns-prefetch` HTML case.

## Evidence

- **Released HTML-page detector:** Release tag `v2026.4.23` includes standalone HTML/Cloudflare detection in the shared formatter. That lets raw `<html>...</html>` challenge pages be treated as upstream HTML errors even without a leading HTTP status prefix.
