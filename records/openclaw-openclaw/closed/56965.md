## Summary

Current `main` already implements selective Discord voice response by reusing Discord guild/channel allowlists and sender `users`/`roles` checks for voice ingress, and it drops unauthorized speakers before transcription. This behavior is documented and is present in the shipped `v2026.4.23` release.

## Evidence

## Risks / Open Questions

- This closes on the existing selective-response implementation via Discord allowlists/users/roles; it does not add a separate wake-word or mention-only voice activation mode.
- The feature is documented under the existing Discord allowlist model rather than a dedicated `channels.discord.voice.activation` config surface, so discoverability may still be imperfect.
