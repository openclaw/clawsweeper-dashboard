Keep open: the direction is useful and the Windows terminal proof is enough for this test-only change, but the new module-load probes still need a small cleanup fix because they can leave temp files or links when setup or cleanup throws.

## What This Changes

The branch replaces three unconditional Windows skips in the browser wait-for-download save tests with symlink/hardlink capability probes and Windows junction symlink creation.

## Best Possible Solution

Keep the dynamic capability-check approach, but make the probes allocate isolated temp paths and always clean them up before merge.

## Reproduction Assessment

## Solution Assessment

No, not quite yet: replacing broad Windows skips with capability checks is the right layer, but the probes should be hermetic by using isolated temp allocation and finally cleanup before this merges.
