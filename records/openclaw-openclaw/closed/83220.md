## Summary

Keep open. Current main still replays Copilot Responses reasoning item IDs unchanged, and this issue has both a protected `maintainer` label and an open linked fix PR, so cleanup should wait for explicit maintainer review or merge.

## What This Changes

The issue reports that Copilot native Responses replay still sends overlong persisted reasoning item IDs after tool-call ID repair, causing resumed sessions to be rejected before generation.

## Best Possible Solution

Land the narrow linked fix that strips only invalid Copilot replay-visible reasoning IDs while preserving valid reasoning replay and existing tool-call ID normalization, with focused transport regression coverage.

## Reproduction Assessment

Yes, this is source-reproducible: build a `github-copilot` `openai-responses` request from same-model history containing a JSON reasoning `thinkingSignature` with an overlong `id`, and current main will push that ID into the outbound Responses input. I did not run the private live Copilot session.

## Solution Assessment

Yes, the linked PR’s direction is the narrow maintainable fix: remove invalid Copilot reasoning item IDs during request construction while leaving non-Copilot replay and tool-call pairing alone. The issue should remain open until that PR is reviewed and merged or closed by a maintainer.
