Full review comments:

- **[P1] Honor skill-level MCP readiness denials:** `src/agents/skills/workspace.ts:175-180`
  - body: If `readiness.skills[entry.skill.name]` exists but is not callable, this code still falls through to `readiness.servers[mcp.server]`. A server-level `callable: true` can therefore expose a skill that the skill-level readiness record explicitly denied, which breaks the gate's main invariant.
  - confidence: 0.9
- **[P2] Run global mcporter checks from the user home:** `scripts/mcp-readiness-smoke.mjs:59-62`
  - body: For the global `~/.mcporter/mcporter.json`, `workspaceDir` becomes OpenClaw home rather than the user home, so `mcporter config list` runs from the wrong cwd and can audit a different global invocation than the one this smoke script is meant to verify.
  - confidence: 0.78
- **[P2] Redact mcporter command output before printing it:** `scripts/mcp-readiness-smoke.mjs:47-49`
  - body: The script copies `mcporter config list --output json` stdout and stderr into its JSON rows. MCP config output can include environment-backed credentials or private endpoints, so this creates a credential-disclosure footgun when contributors paste smoke output as PR proof.
  - confidence: 0.66
