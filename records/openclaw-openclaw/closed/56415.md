## Summary

Current main has superseded this branch with a narrower phone-control authorization split: scoped gateway callers need `operator.admin`, while native Telegram callers still use channel sender authorization instead of absent gateway scopes. The remaining PR diff would reintroduce a known native Telegram regression and still lacks real Telegram proof.

## What This Changes

The PR makes phone-control `arm` and `disarm` require `operator.admin` scopes on every channel and updates tests to reject Telegram callers without that scope.

## Best Possible Solution

Keep current main's split authorization model: gateway-scope callers need `operator.admin`, while any stricter native-channel phone-control policy should be a new maintainer-approved change with live Telegram proof.

## Reproduction Assessment

Yes for the PR-introduced regression by source inspection: native Telegram command execution does not pass `gatewayClientScopes`, so this patch rejects authorized native `/phone arm` and `/phone disarm` callers. No high-confidence current-main bypass remains for scoped gateway callers because current tests cover the `operator.write` denial path.

## Solution Assessment

No. The current helper is the narrower maintainable fix; the PR conflates internal gateway client scopes with native channel sender authorization.
