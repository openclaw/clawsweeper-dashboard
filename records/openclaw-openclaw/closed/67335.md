## Summary

Current main implements the requested macOS LaunchAgent lifecycle reconciliation. Merged PR #72616 changed restart ordering so installed-but-unloaded LaunchAgents are re-bootstrapped before unmanaged listener restart fallback, marks plist-present/launchd-missing states distinctly, adds status and doctor repair guidance, and includes regression coverage for the split-brain and raw SIGTERM supervision cases.

## Best Possible Solution

Close this issue as implemented on main by PR #72616. Keep the shipped main implementation and treat any future macOS launchd failures as narrower regression reports with fresh status, launchctl, and gateway logs.

## Evidence
