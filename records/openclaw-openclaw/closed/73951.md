## Summary

Close as superseded by the open canonical Feishu fix PR #39507. Current main still shows the reported behavior, but that PR already tracks the same `replyInThread: "disabled"` contract and should be the place to review or replace the fix.

## What This Changes

The issue asks for Feishu `replyInThread: "disabled"` to prevent `reply_in_thread: true` from being sent even when inbound reply/thread metadata is present.

## Best Possible Solution

Use #39507 as the canonical work item: review, land, or replace it with an equivalent narrow Feishu extension fix that makes disabled mode suppress thread replies at the resolver/dispatcher boundary and updates the regression tests to match the documented contract.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: the Feishu bot test configures `replyInThread: "disabled"` with inbound `thread_id` and expects thread mode to be forced downstream.

## Solution Assessment

Yes for triage cleanup: closing this as superseded by #39507 preserves the report while avoiding two open trackers for the same Feishu bug. The code solution itself should land through that PR or a narrow replacement, not through a separate ClawSweeper repair job.
