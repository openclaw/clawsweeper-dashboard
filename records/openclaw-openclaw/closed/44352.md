## Summary

Close as a duplicate of #62328. Current main has not fixed the underlying FTS5-unavailable behavior separately for Debian; it still creates the builtin memory FTS table with SQLite FTS5, records/logs the unavailable state when that fails, and skips keyword/BM25 search unless FTS is available. The broader open #62328 tracks the same remaining node:sqlite/FTS5 fallback problem.
