## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Defer TTL handler invocation before catching failures:** `src/auto-reply/reply/get-reply.ts:127`
  - body: `Promise.resolve(ttlHandler())` invokes the handler before `.catch(...)` is attached, so a synchronous throw escapes the timer callback instead of being logged and contained. Start the chain with `Promise.resolve().then(() => ttlHandler())` or use guarded `try/catch`.
  - confidence: 0.91
- **[P2] Drop the stale Tlon Git dependency bump:** `extensions/tlon/package.json:7`
  - body: This branch changes a GitHub-sourced Tlon dependency even though current main no longer declares `@tloncorp/api` in the Tlon package. Keep package-resolution changes out of this stale auto-reply PR.
  - confidence: 0.86
- **[P2] Avoid flipping global connection mode during remote probe:** `apps/macos/Sources/OpenClaw/OnboardingView+Pages.swift:473`
  - body: Setting `state.connectionMode = .remote` during the onboarding probe can trigger app-wide connection-mode observers and real lifecycle work, such as stopping the local gateway, even though the user only clicked a connection check.
  - confidence: 0.78
