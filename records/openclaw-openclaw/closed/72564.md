## Summary

Close as superseded: this PR targets the BlueBubbles TTS voice-memo failure from #72506, but that issue was already closed as fixed on main and the later merged #72586 shipped the maintained solution through channel TTS capabilities plus CAF pre-transcoding. Current main has source, tests, docs, and release-note evidence for that path, and this PR should not be merged as-is because its new tool schema introduces a provider-facing `anyOf` shape.

## What This Changes

The PR derives ElevenLabs output file extensions from `outputFormat` and threads an optional TTS `target` override through the SDK runtime, speech core, `tts` agent tool, and `tts.convert` RPC with tests.

## Best Possible Solution

Keep the shipped channel-capability and CAF pre-transcode solution from #72586 for BlueBubbles voice memos. If a manual target override is still desired, design it as a separate SDK/RPC/tool API change with provider-safe schemas, documented channel semantics, and tests that cover delivery behavior, not only parameter forwarding.

## Reproduction Assessment

Yes. The original bug has a concrete v2026.4.24 reproduction in #72506, and current main has source/test proof plus #72586's recorded macOS BlueBubbles/iOS end-to-end validation showing the failure path is now covered.

## Solution Assessment

No, this PR is no longer the best way to solve the issue. Current main solves the user-visible BlueBubbles problem without forcing a provider `voice-note` target, and the PR adds a public target override with a provider-schema compatibility bug that should not be merged as-is.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Use a flat enum schema for target:** `src/agents/tools/tts-tool.ts:15-18`
  - body: `Type.Union([Type.Literal(...)])` emits an `anyOf` schema for this new tool parameter. Core agent tool schemas are sent to providers that reject nested `anyOf`, and nearby repo guidance/tools use flat string enums for this case. Please use a provider-safe `type: "string", enum: [...]` shape so adding `target` does not make the `tts` tool invalid for those providers.
  - confidence: 0.86
