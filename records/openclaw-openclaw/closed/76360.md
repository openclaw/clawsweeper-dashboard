## Summary

Close as implemented on current main. The observable cold-start problem is addressed after v2026.5.2 by main-only Gateway changes that make Control UI `models.list` use a read-only persisted catalog fast path and make `sessions.list` rows lightweight instead of doing heavy per-row transcript work.

## What This Changes

The issue asks for the Gateway Control UI `models.list` and `sessions.list` paths to avoid >20s first-call latency immediately after a v2026.5.2 gateway restart.

## Best Possible Solution

Keep the narrower current-main request-path fixes and ship them in the next release, rather than adding startup prewarming that could move heavy work into the post-ready window.

## Reproduction Assessment

No, not against current main. I did not run a live restart benchmark, and source/history show the v2026.5.2 cold full-catalog and heavy session-row paths have been replaced on main.

## Solution Assessment

Yes. The current main solution is the narrower maintainable fix because it makes the first Control UI calls cheaper on demand instead of shifting catalog/session work into startup prewarming.
