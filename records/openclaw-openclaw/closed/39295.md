## Summary

Current main already routes subagent completion announces back to the requester chat context instead of a stale last-active channel, so the misrouting problem behind this feature request is already addressed without adding a `--channel` parameter to `sessions_spawn`.

## Evidence

- **Docs say subagents announce back to requester chat:** `docs/tools/subagents.md` documents that sub-agents announce their result back to the requester chat channel and that `sessions_spawn` posts the announce reply to the requester chat channel.

  - file: [docs/tools/subagents.md:107](https://github.com/openclaw/openclaw/blob/4d7c4b3298920c0e6e2c5cc8bd7a1546e8122abe/docs/tools/subagents.md#L107)
