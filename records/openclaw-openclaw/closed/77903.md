## What This Changes

The PR adds a `knownPluginToolNames` parameter to the tool policy pipeline and direct regression tests for suppressing unknown-allowlist warnings for declared plugin tools.

## Best Possible Solution

Resolve enabled plugin manifest tool names in the real tool-policy call path and use them to suppress only declared plugin-tool warnings while preserving warnings for typos, with regression coverage and a changelog entry.

## Reproduction Assessment

Yes, at source level. Current main classifies allowlist entries as unknown when they are absent from `pluginGroups.all`, and the reported `llm-task` case is an optional manifest-declared tool that can be referenced by `tools.alsoAllow` before it is present in the catalog seen by a policy pass.

## Solution Assessment

No. The proposed seam is plausible, but this PR does not connect it to production callers or enabled plugin manifest metadata, so the real warning path remains unchanged.
