## Label Justifications

- `P2`: The report describes a real but already-fixed privacy/safety bug in cron announce delivery with limited blast radius.
- `impact:security`: The affected behavior could expose internal model reasoning or silent-control output to external chat channels.

## Summary

Close: current main and the latest release source already suppress the reported cron announce shape by treating trailing `NO_REPLY` as silent delivery and dropping reasoning-marked outbound payloads before channel send.

## What This Changes

The issue reports that isolated cron `delivery.mode: "announce"` can forward model reasoning or chain-of-thought text to chat channels instead of suppressing silent/internal output.

## Best Possible Solution

Close as fixed and direct affected users to upgrade to `v2026.6.10` or newer.

## Reproduction Assessment

No current-main repro: source and release-tree evidence show the reported trailing-`NO_REPLY` announce payload is now suppressed before channel delivery.

## Solution Assessment

Yes: the implemented fix is in the generic cron/outbound delivery path rather than Feishu-specific code, and it preserves the existing silent-token contract without adding a new delivery mode.
