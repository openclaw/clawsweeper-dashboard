## Summary

This is a valid feature idea, but the requested WeChat voice-bubble send path belongs in the external `@tencent-weixin/openclaw-weixin` channel plugin / ClawHub lane rather than OpenClaw core.

## What This Changes

Request `openclaw-weixin` to send TTS/audio outputs as native WeChat voice-bubble messages instead of generic file attachments.

## Best Possible Solution

Keep OpenClaw core channel-agnostic and implement WeChat native voice-bubble delivery in the external `@tencent-weixin/openclaw-weixin` plugin using the existing `sendMedia` and TTS voice-note capability surfaces.

Not applicable: this is a feature request for external WeChat plugin outbound behavior, not a current OpenClaw core bug. Source and docs confirm the WeChat runtime lives outside this repository.
