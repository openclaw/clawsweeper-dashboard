## Summary

Current `main` already exposes `group:memory` as read-only in practice: the active `memory-core` plugin registers only `memory_search` and `memory_get`, the core tool catalog puts only those two tools in the memory section/group, the prompt/docs only describe recall tools, and there are no `memory_write`, `memory_upsert`, or `group:memory-write` symbols on `main`. The behavior this PR is trying to correct does not reproduce against the current architecture.
