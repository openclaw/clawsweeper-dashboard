## Summary

Current main and v2026.5.4 already address the reported regression: external channel SecretRef contracts load from compiled plugin `dist/` artifacts, and Discord token resolution uses the active runtime snapshot for named accounts with regression coverage.

## What This Changes

Restore Discord bot startup for SecretRef-backed account tokens after the 2026.5.3 external-plugin upgrade regression.

## Best Possible Solution

Keep the v2026.5.4 implementation and ask affected 2026.5.3 operators to upgrade and restart rather than replacing SecretRefs with literal tokens.

## Reproduction Assessment

Yes for the reported 2026.5.3 release path: the issue gives concrete SecretRef config steps, and current source/changelog identify the missing compiled `dist/` contract sidecar path. I did not run a live Discord bot in this read-only sweep; current main is verified by source, tag, and regression-test inspection.

## Solution Assessment

Yes. Loading compiled external plugin secret-contract sidecars from `dist/` and keeping Discord token reads on the active runtime snapshot is the narrow maintainable fix for this regression.
