## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Restore fallbacks on init service mounts:** `docker-compose.yml:13-14`
  - body: `openclaw-init` uses `${OPENCLAW_CONFIG_DIR}` and `${OPENCLAW_WORKSPACE_DIR}` without the fallbacks used by the gateway and CLI volumes. With no setup-generated env file, `docker compose config` now fails with an empty-source volume before the documented manual flow can start, while current main succeeds. Give the init service the same fallback sources or keep it out of the base file until setup writes the env.
  - confidence: 0.93
- **[P1] Verify Homebrew before extracting it:** `Dockerfile:356`
  - body: This layer downloads the Homebrew tag archive and pipes it directly into `tar`, then later runs `brew --version` from that extracted code. Pinning the tag is not an integrity check for official Docker images; add a pinned SHA256 or another auditable verification step before extracting/executing it.
  - confidence: 0.87
- **[P2] Keep apt upgrades opt-in for pinned image builds:** `Dockerfile:243-244`
  - body: The Dockerfile still says release builds consume a reviewed pinned base snapshot, but this new default-on `apt-get upgrade` makes the runtime image depend on whatever Debian publishes at build time. That can change package versions or break builds without a Dockerfile diff; make the upgrade opt-in or provide release-build policy and proof for the new default.
  - confidence: 0.78
