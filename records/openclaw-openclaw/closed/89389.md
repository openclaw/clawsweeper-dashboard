## Summary

Current main already fixes the reported MiniMax Global OAuth failure by sending the device-code and token requests directly to the MiniMax account OAuth host. The fix is not in the latest tagged release, so this can close as implemented on main rather than as shipped.

## What This Changes

MiniMax Global OAuth in v2026.5.28 is reported to call the outdated `api.minimax.io/oauth/code` endpoint and fail before browser/device login starts.

## Best Possible Solution

Keep the current provider-owned OAuth endpoint split and ship it in the next release so MiniMax account OAuth endpoints are used while API resource URLs remain unchanged.

## Reproduction Assessment

Yes. v2026.5.28 source uses `api.minimax.io/oauth/code`, live probing shows that path now redirects, and current main changes the same flow to `account.minimax.io/oauth2/device/code`.

## Solution Assessment

Yes. The current main implementation is the narrow maintainable repair because it updates only the OAuth code/token host while preserving the MiniMax API resource base URL used by model and usage paths.
