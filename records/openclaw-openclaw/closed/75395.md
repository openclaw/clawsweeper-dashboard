Keep open. Current main still does not apply the shared Anthropic thinking-prefill cleanup on the OpenRouter `openai-completions` path for Anthropic model IDs, while direct Anthropic and Cloudflare Anthropic routes do; the report is recent and the only linked fixing PR is closed unmerged.

The issue requests stripping trailing assistant prefill turns when Anthropic models are sent through OpenRouter's OpenAI-compatible route.

## Best Possible Solution

Add an OpenRouter-owned wrapper or shared route-aware helper that detects Anthropic model IDs on verified OpenRouter OpenAI-compatible routes, treats OpenRouter reasoning as thinking-enabled, strips only trailing non-tool assistant prefill turns, and locks it with focused tests plus a changelog entry.
