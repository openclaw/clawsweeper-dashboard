## What This Changes

The PR maps Active Memory's user-visible `empty` recall status to `none` in the status line and updates active-memory tests to expect `status=none`.

## Best Possible Solution

Keep the current main implementation from https://github.com/openclaw/openclaw/pull/80015, which emits precise statuses and avoids caching no-summary recall results.

## Reproduction Assessment

Yes for the original bug on the pre-fix code path: source and tests showed `NONE` and related no-summary paths surfacing as `status=empty`. Current main no longer reproduces it because the landed fix asserts `status=no_relevant_memory`, `status=unavailable`, and `status=failed` separately.

## Solution Assessment

No for this branch as the best solution: mapping all internal `empty` statuses to `none` is too broad. The landed main fix is narrower and more maintainable because it classifies the underlying outcomes before formatting the user-visible line.
