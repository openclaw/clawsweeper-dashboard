## Summary

Current main already wires Codex app-server tool and item progress into Mattermost progress drafts; the reported v2026.5.12 build predates the main-only commit that added the missing Mattermost item-progress path.

## What This Changes

The issue requests Mattermost progress drafts to render Codex app-server native tool activity during agent turns when `streaming.mode` is `progress`.

## Best Possible Solution

Keep the current main implementation and let it ship in the next release; reopen or file a new issue only if a post-v2026.5.12 build still shows no Mattermost draft updates in a live Codex run.

## Reproduction Assessment

No, not as a failing current-main live reproduction. The reported v2026.5.12 path is source-plausible, and current main now has the Mattermost callbacks and Codex event bridge that should prevent the silent draft behavior.

## Solution Assessment

Yes. The maintainable fix is to project Codex app-server progress through the existing agent event stream and consume it in Mattermost’s existing draft stream, rather than adding a Mattermost-specific Codex path.
