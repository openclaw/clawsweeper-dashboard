## Summary

Current `main` already supports least-privilege subagent tool sandboxing through explicit child agent selection (`agentId`), per-agent tool/sandbox policies, `subagents.allowAgents`/`requireAgentId` enforcement, and a dedicated `tools.subagents.tools` clamp. The implementation differs from the issue's proposed inline `toolsAllow` parameter, but the requested capability is present and shipped.

## Evidence

- **`sessions_spawn` already selects a child agent profile:** The tool schema includes an optional `agentId` field for spawned sessions, which lets callers target a specific configured agent instead of relying on the parent's default profile.

## Risks / Open Questions

- The supported shape is config-driven (`agentId` + agent/subagent policy) rather than the issue's proposed per-call `toolsAllow`/`toolProfile` arguments, so this is a capability match more than an API-shape match.
- If maintainers want a more discoverable inline shorthand on top of the existing mechanism, that would be a follow-up ergonomics request rather than missing core functionality.
