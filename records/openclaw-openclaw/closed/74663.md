## Summary

Keep open: the PR addresses the legacy Telegram outbound adapter, but the normal registered Telegram plugin delivery path still drops the voice-note hint, so merging this branch would not fully fix the linked TTS Telegram behavior.

## What This Changes

The PR forwards `audioAsVoice` as `asVoice` in `extensions/telegram/src/outbound-adapter.ts` and adds legacy outbound adapter tests for `sendMedia` and `sendPayload`.

## Best Possible Solution

Propagate Telegram voice-note intent through the registered plugin implementation: carry `ctx.audioAsVoice` or `payload.audioAsVoice` into `buildTelegramSendOptions` as `asVoice`, keep the legacy adapter aligned, and add a focused regression test that exercises `telegramPlugin.outbound` rather than only the legacy adapter.
