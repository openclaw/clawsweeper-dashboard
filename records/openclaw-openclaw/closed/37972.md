## Summary

Keep open. Current `main` still contains the cross-provider guard that drops configured fallbacks unless the requested model is already one of those fallbacks, and `main` tests explicitly assert that override failures skip configured fallbacks and jump straight to the configured primary. This PR is therefore not implemented on `main` and is not stale-close material.
