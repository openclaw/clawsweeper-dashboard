## Summary

Automerge review finds this PR focused and likely correct: it separates recovered middleware tool warnings from fatal cron output while preserving them in diagnostics. ## What This Changes

The PR adds metadata for non-terminal middleware tool-error warnings, propagates it from embedded tool handling into cron payload outcome and diagnostics, and adds focused tests for fatal versus diagnostic warning behavior.

## Best Possible Solution

Land the metadata-based classification once maintainers accept the `middlewareError` contract and the automerge lane confirms the exact head remains green.

## Solution Assessment

Yes, conditionally: metadata on the recovered warning is a narrow maintainable fix if `middlewareError` is the right upstream signal. The safer alternative is a more explicit recovered-warning marker if that contract is broader than the PR assumes.
