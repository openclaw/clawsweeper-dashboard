## Summary

Current `main` no longer leaves the initial `Wake up, my friend!` Ollama hatch stuck indefinitely. The run loop now bounds no-token stalls with the LLM idle watchdog, retries one silent startup idle timeout, and then surfaces a clear error pointing at `agents.defaults.llm.idleTimeoutSeconds`; the changelog records this under `v2026.4.10`.

## Risks / Open Questions

- This does not prove Gemma4/Ollama first-token latency itself became fast; it proves current `main` no longer hangs indefinitely on that path and now fails/retries with bounded timeout behavior.
- One commenter had `idleTimeoutSeconds = 0`, which still disables the watchdog by design; the implemented fix covers the default/current hatch behavior, not explicit timeout disablement.
