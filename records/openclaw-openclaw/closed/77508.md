## Summary

Keep open: current main still lets `channels login` save auth before runtime reconciliation and does not have a missing-channel-config preflight, so the reported misleading post-save gateway error remains plausible.

## What This Changes

Request clearer `channels login` handling when the selected channel config block is missing, either by failing before saving auth or creating minimal channel config.

## Best Possible Solution

Add an explicit login preflight for absent or disabled `channels.<name>` config that aborts before auth persistence and prints actionable setup guidance, with focused CLI/gateway tests and a changelog entry.

## Reproduction Assessment

No live current-main reproduction was run. Source inspection gives a clear path: `runChannelLogin` saves auth before calling `channels.start`, and the gateway start path does not preflight an absent channel config block.

## Solution Assessment

Yes. The narrowest maintainable fix is to fail before auth is saved with explicit `channels.<name>` setup guidance, rather than implicitly writing channel config in the auth-only login flow.
