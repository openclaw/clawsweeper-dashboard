## Summary

Current main has moved past the v2026.4.29 failure mode: bundled plugin dependency staging is now treated as legacy cleanup state, bundled channel loading resolves from package bundled roots, and root-owned dist imports are guarded against undeclared runtime dependencies. The fix is present on main but not in a release tag yet.

## What This Changes

The issue reports a v2026.4.29 npm-global regression where Slack and Telegram probes fail because the bundled plugin runtime staging path cannot resolve the bare `json5` import.

## Best Possible Solution

Keep the current main direction: bundled channels should load from the packaged bundled plugin graph, old user-state runtime-deps staging should stay legacy cleanup only, and the postpublish guard should continue blocking root-owned dist imports that lack root package dependencies. The remaining operational step is shipping this in the next release.

## Reproduction Assessment

Yes. The report gives a concrete npm-global upgrade path from 2026.4.26 to 2026.4.29 plus `openclaw channels status --probe` output showing Slack and Telegram failing on `json5` under `plugin-runtime-deps`. I did not run the live channel probe in this read-only review, but the reported path maps directly to the old staging surface now removed on main.

## Solution Assessment

Yes. Removing the user-state bundled dependency staging path and guarding installed root dist imports against undeclared dependencies is the narrowest maintainable fix for this class of `Cannot find package` failures; Slack and Telegram do not need channel-specific work for this root runtime resolution issue.
