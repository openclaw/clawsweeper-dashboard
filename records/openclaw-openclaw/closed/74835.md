## Summary

Keep open. Current main has per-provider request overrides plus exact-origin/fake-IP SSRF improvements, but it still lacks an inherited/default model-provider request policy for proxy and `allowPrivateNetwork`; adding that surface needs config and security design.

## What This Changes

Request a global/default model-provider request policy so proxy and `allowPrivateNetwork` settings can be configured once and overridden per provider.

## Best Possible Solution

Design a models-scoped request-defaults layer that merges before provider-local `request`, preserves explicit overrides and opt-outs, defines local-provider handling, and updates schema, docs, redaction, effective config, and SSRF tests together.

## Reproduction Assessment

Yes for the current config-surface gap: source inspection on current main shows only provider-local `models.providers.<id>.request` is accepted and passed into runtime request resolution. A live fake-IP proxy environment is not needed to verify the inherited defaults surface is missing.

## Solution Assessment

Unclear. A model-provider defaults layer is plausible, but the key shape, merge precedence, local-provider defaults, audit output, and inherited SSRF semantics need maintainer/security design before implementation.
