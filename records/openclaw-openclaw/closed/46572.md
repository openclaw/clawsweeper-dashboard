## Summary

Current `main` already sanitizes user chat-history entries before the TUI renders them, stripping the injected inbound metadata and timestamp/envelope text that made WhatsApp/Telegram user turns look like internal context.

## Evidence

- **`chat.history` sanitizes returned messages:** The gateway history handler now runs all history entries through `stripEnvelopeFromMessages(...)` before returning them to clients, so the TUI no longer receives raw inbound-context text.
