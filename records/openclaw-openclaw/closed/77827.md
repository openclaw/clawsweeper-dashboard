## Summary

Keep this PR open, but it still needs changes before merge. The bug is valid and the LM Studio proof is convincing, but the current patch can still clear the active Responses reasoning block before `response.output_item.done` records replay metadata.

## What This Changes

The PR adds `response.reasoning_text.done` handling to the OpenAI Responses stream parser plus focused regression tests for LM Studio done-only reasoning text.

## Best Possible Solution

Merge a refreshed parser fix that supports LM Studio done-only reasoning text while preserving replayable reasoning-item metadata and avoiding duplicate thinking events for standard Responses streams.

## Reproduction Assessment

Yes. Current main has no `response.reasoning_text.done` branch in `processResponsesStream`, the upstream event is documented in OpenAI types, and the PR supplies inspected before/after LM Studio screenshots showing the missing and restored thinking block.

## Solution Assessment

No, not yet. The direction is right, but the standard path should keep or transfer reasoning item state through `response.output_item.done` so `thinkingSignature` remains available for replay.
