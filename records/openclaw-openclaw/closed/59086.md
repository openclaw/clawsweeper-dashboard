## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Preserve the reasoning visibility gate:** `src/agents/pi-embedded-subscribe.ts:50-52`
  - body: Setting `streamReasoning` from `reasoningMode === "stream"` alone drops the current-main `thinkingLevel !== "off"` visibility gate. A session configured to hide thinking could still emit live reasoning, so keep the visibility check when decoupling the callback.
  - confidence: 0.86
- **[P1] Gate thinking chat deltas before delivery:** `src/gateway/server-chat.ts:607-608`
  - body: The new helper broadcasts thinking as chat content to session listeners without an opt-in capability or show-thinking boundary. That can expose reasoning to clients that did not request or should not display it, so route it through an explicit visibility/capability contract before sending.
  - confidence: 0.78
- **[P2] Suppress hidden thinking-only stream rows:** `ui/src/ui/views/chat.ts:1014-1020`
  - body: When thinking is hidden, a stream item backed only by `thinking` blocks still reaches the renderer, but the renderer is called with reasoning hidden. That leaves an empty assistant row/footer until answer text arrives, so skip thinking-only stream rows when `showThinking` is off.
  - confidence: 0.75
- **[P3] Avoid stripping double Markdown markers:** `ui/src/ui/chat/message-extract.ts:125-133`
  - body: The normalizer treats any line that starts and ends with `_` or `*` as single-wrapped, so `**bold**` becomes `*bold*` before being rewrapped. Exclude double markers before slicing to avoid corrupting deliberately formatted reasoning text.
  - confidence: 0.83
