## Summary

Keep open: the warning suppression is aimed at a real minimal-build problem, but the patch disables the whole static plugin asset phase for `gatewayWatch` and `cliStartup`, which also drops source-backed ACPX runtime helpers rather than only generated diffs assets.

## What This Changes

The branch adds a `runtime-postbuild` static-asset skip env override for `gatewayWatch` and `cliStartup`, plus regression tests for profile env propagation and skip behavior.

## Best Possible Solution

Keep minimal profiles quiet for generated missing diffs assets while still copying existing source-backed runtime assets such as ACPX MCP helpers; full and CI artifact builds should remain strict.

## Reproduction Assessment

Yes. Source inspection shows current minimal profiles run `runtime-postbuild` without generated diffs assets, and the PR body includes before/after Crabbox evidence for the noisy warning path; I did not run commands because this review is read-only.

## Solution Assessment

No, not as written. The proposed env guard solves the warning noise too broadly by bypassing source-backed static assets; a narrower asset-class or missing-generated-source guard would be more maintainable.
