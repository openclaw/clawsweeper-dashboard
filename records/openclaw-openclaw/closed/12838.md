## Summary

Current `main` already auto-sanitizes replayed image payloads before provider submission, including downscaling/recompressing oversized images to fit byte limits, dropping images that still cannot fit, and converting residual provider image-size failures into a friendly user-facing message instead of dumping the raw upstream error.
