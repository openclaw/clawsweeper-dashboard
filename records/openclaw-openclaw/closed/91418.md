## Summary

Close: current main and shipped releases already route Discord guild text-channel channelConfig.systemPrompt into GroupSystemPrompt and append it to the agent system prompt, including the parent-channel config path used by thread/auto-thread flows.

## What This Changes

The issue asks for Discord per-channel systemPrompt configuration to be injected for text-channel turns instead of being limited to voice transcript turns.

## Best Possible Solution

Keep the shipped runtime path; if a current install still drops this prompt, the next report should include prompt-report or verbose Discord preflight evidence showing where GroupSystemPrompt becomes empty.

## Reproduction Assessment

No; source inspection shows current main resolves Discord guild channelConfig.systemPrompt, stores it as GroupSystemPrompt, and appends it to the agent prompt for text turns. I did not run a live Discord gateway repro because this cleanup review is read-only, but current tests cover the helper path.

## Solution Assessment

Yes; the existing implementation is the maintainable owner-boundary fix because the Discord plugin resolves channel config and the shared reply runner injects the resulting GroupSystemPrompt. The remaining docs wording in the voice section can be clarified separately, but the runtime behavior requested here is already present.
