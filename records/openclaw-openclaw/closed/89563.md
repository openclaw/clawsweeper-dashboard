## Summary

Current main has already removed the reported voice-call JSONL append path and persists call records through SQLite-backed plugin state, so the symlinked state-directory failure is fixed on main but not yet in the latest release.

## What This Changes

The issue reports voice-call call-record persistence failing when the OpenClaw state directory is reached through a symlink.

## Best Possible Solution

Keep the current SQLite-backed plugin-state persistence plus doctor migration, and let a future release carry the current-main fix to users still on v2026.5.27.

## Reproduction Assessment

Yes for the shipped v2026.5.27 path: the issue stack matches the appendRegularFile JSONL path removed by the merged storage migration. Current main no longer has that failing persistence path.

## Solution Assessment

Yes. Moving voice-call call records to SQLite-backed plugin state, with a doctor migration for legacy calls.jsonl, is the cleaner repository-aligned fix than allowing broader symlink traversal in fs-safe.
