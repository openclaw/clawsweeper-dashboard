## Summary

The report is real on current main: Signal still builds external daemon URLs under `/api/v1/...`, while bbernhard/signal-cli-rest-api documents root `/v2/send` and `/v1/receive/{number}` endpoints. Close as duplicate/superseded because open PR #16085 already tracks the same bbernhard REST/WebSocket container support, and #10240 was already closed to keep that work centralized on that PR.

## Best Possible Solution

Close this issue as duplicate/superseded and keep PR #16085 as the canonical implementation candidate. The best product path is to review, fix, and land #16085 or an equivalent Signal adapter that preserves native JSON-RPC/SSE mode while adding bbernhard REST/WebSocket container mode, config/schema/docs, and focused regression tests.
