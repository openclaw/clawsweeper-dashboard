## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Scope the draft cache beyond sessionKey:** `ui/src/ui/app-render.helpers.ts:495-502`
  - body: The module-level draft map is keyed only by sessionKey, then restored by nextSessionKey. If the Control UI reconnects or switches gateway/workspace contexts with a reused key like main, a stale unsent draft can reappear in the wrong context. Scope the cache to the active connection/session namespace or clear it on gateway/auth changes.
  - confidence: 0.87
- **[P2] Narrow the generated group-name filter:** `ui/src/ui/app-render.helpers.ts:677-678`
  - body: The new generated-name filter treats every <provider>:g-* displayName as internal, but the Gateway emits that format for normal channel groups. This drops legitimate labels such as Discord or Slack group names and falls back to generic text.
  - confidence: 0.9
- **[P2] Use the last subagent segment for short IDs:** `ui/src/ui/app-render.helpers.ts:686-688`
  - body: extractShortSessionId takes the first UUID-like segment in the full key. Nested subagent keys can contain both parent and child segments, so child sessions can display the parent's short ID and become indistinguishable.
  - confidence: 0.86
- **[P3] Assert the Stop button against null:** `ui/src/ui/views/chat.test.ts:661`
  - body: querySelector returns null, not undefined, so this presence assertion would pass if the Stop button were absent. Use not.toBeNull() to make the test's intent and failure message accurate.
  - confidence: 0.82
