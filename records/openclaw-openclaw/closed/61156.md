## Summary

Close PR #61156 as already implemented on current `main`: Discord ack/status reaction handling now builds a runtime reaction context with `cfg`, `rest`, and `accountId`, and both reaction handlers use that context directly, which avoids the old fallback-to-`default` account path described in the PR.
