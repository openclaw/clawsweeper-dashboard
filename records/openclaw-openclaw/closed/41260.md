## Summary

Current main no longer has the recursive 50ms drain-poll loop that this PR patches. `waitForActiveTasks()` was replaced on main with an event-driven waiter model backed by task-completion notifications and a single deadline timer, with tests covering drain success and timeout behavior, so the PR is obsolete as written.

## Evidence

- **`waitForActiveTasks()` is no longer a polling loop:** On current main, `waitForActiveTasks()` snapshots active task IDs, registers an `ActiveTaskWaiter`, and sets one timeout deadline. The old recursive `setTimeout(check, POLL_INTERVAL_MS)` loop from the PR description is not present.
  - file: [src/process/command-queue.ts:381](https://github.com/openclaw/openclaw/blob/29f7a2f441c6464df740ccebb24bae18783eb12a/src/process/command-queue.ts#L381)

- **Drain completion is event-driven and clears the timeout:** `resolveActiveTaskWaiter()` removes the waiter and `clearTimeout()`s its deadline, while `notifyActiveTaskWaiters()` resolves `drained: true` once the tracked task IDs are gone.
  - file: [src/process/command-queue.ts:135](https://github.com/openclaw/openclaw/blob/29f7a2f441c6464df740ccebb24bae18783eb12a/src/process/command-queue.ts#L135)
