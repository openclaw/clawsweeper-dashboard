## Summary

Keep open. The PR targets a credible duplicate plugin-load symptom, but the proposed cache-key collapse is too broad because current main intentionally separates default, explicit-subagent, and gateway-bindable plugin runtime registries.

## What This Changes

The PR changes `src/plugins/loader.ts` to omit `runtimeSubagentMode` from `buildCacheKey`, with an explanatory comment and no test or changelog updates.

## Best Possible Solution

Implement a narrow asymmetric reuse path or call-site normalization that avoids redundant plugin loads while never serving a default-built registry to gateway-bindable or explicit-subagent requests, then add regression tests and a changelog entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to apply the PR and inspect the existing cache-mode tests: current main expects gateway-binding mode to remain compatibility-shaping, and the PR removes that discriminator without changing the tests.

## Solution Assessment

No. Removing `runtimeSubagentMode` from the cache key globally is not the best fix because `allowGatewaySubagentBinding` changes the runtime object provided to plugins; a narrower asymmetric reuse path is safer.
