## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/87732. The observable bug is that a globally npm-installed OpenClaw package can report `packageManager: "pnpm"` and check `<packageRoot>/pnpm-lock.yaml` because the published package keeps root `package.json.packageManager: pnpm@...` while npm installs ship `npm-shrinkwrap.json`. Keep the change focused on update/status package-manager and dependency-lock detection. Treat npm package roots with `npm-shrinkwrap.json` as npm install evidence when the pnpm lockfile is absent, and make npm dependency status use `npm-shrinkwrap.json` as the lock marker when present rather than requiring `package-lock.json`. Preserve source checkout behavior where `pnpm-lock.yaml` exists, and do not add a config option, migration, changelog edit, or broad update-command refactor. Add focused regression coverage for the published package root shape and keep existing git/source checkout cases green.
