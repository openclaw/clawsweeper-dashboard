## Summary

Current `main` still appears susceptible to this bug: the resolved daily log path updates by date, but long-lived subsystem loggers cache a child logger once and keep writing through that original logger instead of rebinding after the date changes.

## Evidence

- **Daily path is date-based and re-resolved:** `resolveSettings()` uses `cfg?.file ?? defaultRollingPathForToday()`, and `getLogger()` rebuilds only when the resolved settings differ. That means status/doctor can report a new daily file after midnight.
  - file: [src/logging/logger.ts:127](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/logging/logger.ts#L127)
  - command: `nl -ba src/logging/logger.ts | sed -n '100,320p'`
  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
- **Logger transport writes to a fixed file from the build-time settings:** `buildLogger()` attaches a transport that appends to `settings.file`; that file path is captured when the logger is built.
  - file: [src/logging/logger.ts:176](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/logging/logger.ts#L176)
