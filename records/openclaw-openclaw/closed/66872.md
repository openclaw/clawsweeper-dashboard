## Summary

Current `main` already treats plugin-owned synthetic auth as valid for `models status --json`, so `codex` should no longer appear under `auth.missingProvidersInUse` when the Codex harness owns auth. The same implementation and regression test are present in the shipped `v2026.4.23` release snapshot.

## Evidence

- **Status command excludes synthetic-auth providers from missing list:** `models status` builds `syntheticAuthByProvider` from plugin runtime refs, passes that through auth overview resolution, and filters `missingProvidersInUse` to exclude providers present in synthetic auth.
