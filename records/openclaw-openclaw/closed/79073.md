## Summary

Current main contains the generic external-channel sidecar fix that lets npm-published Discord contribute its SecretRef contract before startup, and that fix shipped in v2026.5.7 after the reported 2026.5.3-1 failure.

## What This Changes

Allow `channels.discord.token` SecretRef values, including exec-backed refs, to resolve before Discord channel startup instead of leaving the channel `not configured`.

## Best Possible Solution

Keep the shipped v2026.5.7 external channel contract discovery fix; remaining embedded reply/action discovery work should stay with the separate open SecretRef thread.

## Reproduction Assessment

No current-main failing reproduction remains from this review: the old 2026.5.3-1 dist-sidecar failure is source- and PR-proven, and current main now includes the loader and regression test that close that path.

## Solution Assessment

Yes. The merged generic sidecar discovery fix is narrower and more maintainable than a Discord-only token fallback because it restores the plugin SecretRef contract before startup for npm-published external channel plugins.
