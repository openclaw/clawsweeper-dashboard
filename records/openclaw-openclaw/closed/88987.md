Keep open: current main and the latest release still reject valid short tweakcn theme IDs before fetch, while the Control UI docs already say raw theme IDs, registry URLs, and default theme names are supported.

## What This Changes

The issue reports that the Control UI tweakcn theme importer rejects valid short built-in tweakcn theme names such as `claude`, `zinc`, and `slate`.

## Best Possible Solution

Relax only the theme ID length validation to match live/documented tweakcn IDs, keep the existing host/path/fetch/CSS safety checks, and add focused regression tests for short raw IDs and registry URLs.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main's validator rejects `claude` before fetch, and the live tweakcn registry returned a valid JSON theme for that ID. I did not run the full UI because this read-only review cannot install missing dependencies or create test artifacts.

## Solution Assessment
