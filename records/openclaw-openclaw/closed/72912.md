## Summary

Keep open for maintainer review: the PR addresses a real current-main channel restart wedge, but it is maintainer-labeled and deliberately leaves a lifecycle-overlap security/product tradeoff for human approval.

## What This Changes

The PR adds health-monitor-only force retirement on channel stop timeouts, lifecycle-gated cleanup, scoped runtime and approval shutdown hardening, regression tests, and a changelog entry.

## Best Possible Solution

Maintainers should either explicitly accept and document the health-monitor-only recovery boundary or add a generation fence, hard-stop contract, or plugin abort policy before replacement lifecycles can process inbound events.

## Reproduction Assessment

Yes. Current main has a focused unit reproduction in `src/gateway/server-channels.test.ts:304`, and related reports describe the same stop-timeout wedge for Telegram and Slack after sleep or stale transport recovery.

## Solution Assessment

Unclear. The PR is a narrow repair for the observed manager bookkeeping wedge, but it is not clearly the best final fix until the plugin-owned transport overlap is accepted or fenced.
