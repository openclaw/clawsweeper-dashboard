## What This Changes

The issue asks for QQBot cron delivery targets to normalize redundant provider prefixes and avoid duplicated `qqbot:` prefixes in `cron list` previews.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/78916, then close this issue as fixed by that merge; if that PR is rejected, keep this issue as the canonical QQBot cron target-normalization bug.

## Solution Assessment

Yes. The linked PR's approach is the narrow maintainable fix: use QQBot-owned explicit target parsing/normalization and a small prefix-aware cron preview guard rather than adding broad cron storage special cases.
