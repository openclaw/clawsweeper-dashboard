## Summary

Current `main` no longer has Telegram/Discord bundled plugins importing `../../../src/...` from the Docker runtime. Both channel entrypoints now use the packaged `openclaw/plugin-sdk/outbound-runtime` export, which is shipped from `dist`, while the Docker runtime image still intentionally omits `/app/src`. That makes the reported `/app/src`-missing failure path obsolete on current `main` and in the shipped `v2026.4.22` release.
