## Summary

Keep open: current `main` still resolves the CLI compaction context engine before initializing built-in context engines, and https://github.com/openclaw/openclaw/pull/79484 is an open closing PR for this issue. The issue should stay open until that PR merges or is abandoned.

## What This Changes

The issue reports failed CLI transcript persistence because the default `legacy` context engine is resolved before any built-in context engine registration.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79484 as the canonical fix, then close this issue and dedupe the older duplicate reports/PRs.

## Reproduction Assessment

Yes, at source level: a persisted CLI session with `sessionFile` and `contextTokens` enters `runCliTurnCompactionLifecycle`, which resolves the default context engine before any built-in initializer. I did not run the live `openclaw agent` command in this read-only review.

## Solution Assessment

Yes. Calling the existing idempotent initializer before `resolveContextEngine` is the narrowest maintainable fix and matches adjacent subagent paths; the open closing PR implements that direction.
