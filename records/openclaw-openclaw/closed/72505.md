## Summary

Current main addresses this gateway-startup/runtime-deps failure class. The affected Google Chat minimal config uses the bundled runtime-dependency staging path, and current main now reuses an existing external `openclaw-*` stage root when a mirrored bundled plugin root is inspected again instead of deriving a second-generation `openclaw-unknown-*` root. Regression coverage models the mirrored-stage loop and fails if runtime deps reinstall into the nested root. The latest release noted in the report, `v2026.4.24`, lacks that existing-root check, so this is implemented on main but not tied to a shipped release tag in this checkout.

## Best Possible Solution

Close this as implemented on current main. Keep the external runtime-deps stage-root reuse, mirrored-root regression test, and startup/runtime-deps hardening in place; if a build containing current main still loops before HTTP bind, it should be filed as a fresh narrower report with the new version, startup trace, stability bundle, and CPU/strace sample.
