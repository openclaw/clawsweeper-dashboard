## Summary

Current main already implements the PR’s intended direct-first `sudo -u` systemd behavior, with a more conservative scope resolver than the PR branch and regression coverage for stale `SUDO_USER`, no-medium user-bus fallback, and target-user machine-scope retry. The shipped v2026.4.26 changelog also references #44417, so this PR is now redundant rather than something to merge as-is.

## Best Possible Solution

Close this PR and keep the shipped main implementation. It solves the linked `sudo -u` systemd problem with narrower sudo-to-root handling, target-user fallback, and regression tests; any remaining Linux user-bus edge case should be filed as a new focused issue with exact environment, command, and `systemctl` output.
