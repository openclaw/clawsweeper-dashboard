## Summary

Current main implements the requested transcript isolation. Heartbeat runs still send the internal heartbeat/system-event prompt to the runner, but they pass an empty visible transcript prompt; queued System: event blocks are kept in runtime prompt context and excluded from transcriptCommandBody; the embedded runner treats empty transcript prompts as runtime-only/system context instead of user-authored visible text. Regression tests cover both heartbeat prompts and system event blocks staying out of the transcript prompt. The linked PRs #43189 and #43191 correctly diagnosed the same pollution class but were closed unmerged; main now addresses the central #43168 behavior through a different implementation.

## Best Possible Solution

Close #43168 as implemented on current main, keep the transcriptPrompt/runtime-only context separation, and leave broader system-event audience/preemption design to the canonical follow-up issues such as #69492 and #64810. Do not revive the closed unmerged PR attempts unless a new regression shows the current implementation misses a concrete path.
