## Summary

Current main already implements safe heredoc allowlist handling, and the same behavior is present in the shipped `v2026.4.23` release: quoted heredocs are parsed as literal stdin input, unquoted expansion forms are rejected, and allowlist-mode prompts only happen on analysis or allowlist misses.

## Evidence

- **Quoted heredoc accepted in shipped tests:** The shipped `v2026.4.23` test suite explicitly accepts `"/usr/bin/cat <<'EOF'\n$(id)\nEOF"` as a safe heredoc form, proving quoted heredocs are intended to pass shell analysis without an automatic approval gate.

  - file: [src/infra/exec-approvals-analysis.ts:208](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/infra/exec-approvals-analysis.ts#L208)
