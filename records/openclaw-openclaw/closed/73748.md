## Summary

Current main and the shipped v2026.4.26 release already cover the OpenClaw side of this report: non-bundled plugins must explicitly opt into conversation-content hooks, the allowConversationAccess key is accepted by config/schema/CLI paths, blocked hooks produce plugin diagnostics, and the docs describe the exact MemOS-style fix. The remaining action is for the external MemOS plugin/user config to set the trusted opt-in and restart the Gateway, which the issue body says resolved the symptom.

## Best Possible Solution

Keep the shipped security opt-in in OpenClaw core. For MemOS and similar external plugins, document or automate setting plugins.entries.<plugin-id>.hooks.allowConversationAccess=true in the plugin's own setup flow, then restart the Gateway. A separate OpenClaw issue would only be needed for a narrower doctor/upgrade UX improvement.
