## Summary

Close as `implemented_on_main`: current `main` already supports explicit session-store isolation via `session.store` with `{agentId}` templating, while `agents.defaults.sessionsDir` is not a valid config key under the strict `agents.defaults` schema.

## Evidence

- **`agents.defaults.sessionsDir` is not a supported schema key:** `agents.defaults` is validated by a strict object schema, and `AgentDefaultsSchema` does not define any `sessionsDir` field. That makes the reported `Unrecognized key: "sessionsDir"` error expected rather than a missing runtime hookup.
  - file: [src/config/zod-schema.agents.ts:7](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/config/zod-schema.agents.ts#L7)

  - sha: [11cffb230013](https://github.com/openclaw/openclaw/commit/11cffb230013c2d8bd1e0e18da04b0b985b8abd5)
- **OpenClaw already supports relocating the session store:** `resolveStorePath()` uses `session.store` as the supported override and expands `{agentId}` into per-agent session-store paths, which covers the isolation use case described in the issue.
  - file: [src/config/sessions/paths.ts:280](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/src/config/sessions/paths.ts#L280)

  - file: [docs/gateway/config-agents.md:1120](https://github.com/openclaw/openclaw/blob/11cffb230013c2d8bd1e0e18da04b0b985b8abd5/docs/gateway/config-agents.md#L1120)
