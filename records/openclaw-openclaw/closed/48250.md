## Summary

Current main already implements the session-yield lifecycle plumbing the report asks for: `sessions_yield` ends the parent turn cleanly, gateway agent lifecycle events are broadcast to subscribed WebSocket clients as `sessions.changed`, and the Control UI subscribes to those events, clears active run state on terminal chat events, and flushes queued messages. The latest release commit also contains the same relevant lifecycle/session files as current main.
