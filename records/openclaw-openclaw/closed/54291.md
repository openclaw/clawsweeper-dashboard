## Summary

Issue #54291 is implemented on current main. The duplicated unsafe describeUnknownError serializers were first wired through the central redacted formatter and then removed, and current formatter/redaction tests cover the credential leakage path.

## Best Possible Solution

Close #54291 as implemented. Keep the shipped central formatErrorMessage/redactSensitiveText path as the core solution, and require any future reports about other serializers to name a specific current-main call path and user-facing/logging sink.
