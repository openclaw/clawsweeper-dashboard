## Summary

Keep open. Current main still writes JSON only after successful Gateway RPCs for the reported commands, and the open closing PR at https://github.com/openclaw/openclaw/pull/79233 should land or be closed before this issue is resolved.

## What This Changes

The issue asks gateway-backed CLI commands with `--json` to emit a structured stdout error on gateway transport failures instead of stderr-only text.

## Best Possible Solution

Land a rebased shared `GatewayTransportError` JSON serializer through https://github.com/openclaw/openclaw/pull/79233 or a narrow replacement, then close this issue through the merge.

## Reproduction Assessment

Yes, source-level. Current main awaits Gateway RPC calls before stdout JSON writes in `health`, `gateway health`, and `devices list`, and the issue provides concrete unavailable-Gateway commands for the failing paths.

## Solution Assessment

Yes. A shared serializer for typed `GatewayTransportError` failures in JSON-mode command wrappers is the narrow maintainable fix, while preserving existing human stderr behavior outside JSON mode.
