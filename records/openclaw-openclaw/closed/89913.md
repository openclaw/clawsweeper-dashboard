## Summary

Keep open. Current main still sends QQBot outbound text without a channel sanitizer, while shared delivery only sanitizes channels that opt in and sibling channels already sanitize assistant-visible text; MiniMax explicit thinking can still expose reasoning content per current source/docs, so the reported QQBot leak remains actionable.

## What This Changes

Request to stop MiniMax/Anthropic-compatible reasoning or thinking text from being delivered as visible QQBot replies in OpenClaw 2026.5.28.

## Best Possible Solution

Make QQBot outgoing text pass through the canonical assistant-visible sanitizer before platform send, preserving final answers, media tags/captions, streaming, chunking, and reply metadata.

## Reproduction Assessment

Yes from source, but not live-reproduced here: MiniMax can emit reasoning content when thinking is explicit, and QQBot has no outbound sanitizer before native send.

## Solution Assessment

Yes, the best first fix is plugin-owned QQBot outbound sanitization using the canonical assistant-visible text path; a generic default sanitizer would need broader sibling-channel proof before changing all unsanitized channels.
