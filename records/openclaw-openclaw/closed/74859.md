## Summary

Keep open. Current main still evaluates `activeSession.prompt(...)` before the abort wrapper can reject an already-aborted run signal, and the active closing path remains the open, conflicting PR at https://github.com/openclaw/openclaw/pull/74979.

## What This Changes

The issue reports that pre-aborted embedded-run signals can still start `activeSession.prompt()`, leaving a background Agent loop running after the attempt exits.

## Best Possible Solution

Fix the embedded prompt boundary so pre-aborted and mid-preflight aborts cannot start or leave a Pi Agent run behind, add focused regression coverage, and keep the optional per-run LLM-call cap as separate hardening.

## Reproduction Assessment

Yes. The deterministic pre-aborted `AbortSignal` path is source-reproducible on current main because the external abort is observed, but `activeSession.prompt(...)` is still evaluated before the abort wrapper can reject.

## Solution Assessment

Yes for the bug boundary, with one caveat. The narrow maintainable solution is an abort-aware handoff before Pi can start `Agent.prompt()` plus escaped-run cleanup; a configurable max-call cap is useful defense-in-depth but not required for the primary fix.
