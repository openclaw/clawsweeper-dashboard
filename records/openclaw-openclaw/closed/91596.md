## Summary

Current main already solves the local `modelPath` dirty-status problem through a provider-owned index identity hook and alias support, so this conflict-marked PR is no longer needed as a landing branch.

## What This Changes

This PR changes memory-core status-only index identity to expect `local.modelPath` or the local default model for local embedding providers and adds a regression test for custom local model paths.

## Best Possible Solution

Keep current main's provider-owned identity implementation and close this duplicate branch; users need a later release or a main build until the fix ships.

## Reproduction Assessment

Yes, source-level: the original bug is the status-only identity comparison before provider initialization, while local embedding metadata is written from the provider's effective model identity. I did not run a live Docker/GGUF repro in this read-only pass.

## Solution Assessment

Yes for current main: the provider-owned `resolveIndexIdentity` path is the better fix because each adapter declares its identity and aliases instead of memory-core hard-coding all local transports to `local.modelPath`.
