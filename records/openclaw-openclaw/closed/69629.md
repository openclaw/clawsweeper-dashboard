## Summary

Keep open. Current main still publishes Discord native slash commands at application or guild scope and only enforces configured channel policy after the user selects a command, while the issue now has a second user report confirming the same multi-bot picker clutter.

## What This Changes

The issue requests per-channel Discord slash-command visibility so multiple bot accounts in one guild only show commands in their assigned channels.

## Best Possible Solution

Design and document a supported Discord per-account, per-channel native-command visibility model, then add targeted unit coverage and live Discord multi-account proof for the chosen behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure multiple Discord accounts in one guild with separate allowed channels and native commands enabled; current main deploys each account's command set at application/guild scope and rejects wrong-channel use only after selection.

## Solution Assessment

Unclear. The requested UX is valid, but the best implementation needs a maintainer choice between Discord permission reconciliation, deriving visibility from bindings/channel config, adding explicit native-command visibility config, or documenting a text-only/isolation workaround.
