## What This Changes

The branch explicitly creates `/home/node/.config` as `node:node 0755` in the Docker runtime image setup and extends the Dockerfile test to assert that parent ownership.

## Best Possible Solution

Land this narrow Dockerfile fix after normal CI and maintainer-owned changelog handling, keeping `.config/openclaw` private at `0700` while making the standard `.config` parent node-owned.

## Reproduction Assessment

Yes, from source and dependency behavior rather than a local full image run: current main creates only the nested OpenClaw config directory, and GNU `install -d` gives missing parents default attributes. The linked release report supplies concrete Docker output for the shipped symptom.

## Solution Assessment

Yes. Explicitly creating the standard parent directory with node ownership before the private OpenClaw directories is narrower and safer than a broad recursive `chown`, and the PR adds a focused guard assertion.
