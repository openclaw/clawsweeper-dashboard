## Summary

Current `main` already covers both reported failure modes: dangling Anthropic tool calls are stripped using a helper that recognizes `toolCall`, and errored/empty assistant turns are filtered out before Anthropic payload conversion so they do not corrupt replayed request history.

## Evidence

- **Dangling tool-call stripping now matches `toolCall`:** `stripDanglingAnthropicToolUses` no longer compares against the wrong literal; it calls `isToolCallBlock()`, which returns true for `toolUse`, `toolCall`, and `functionCall`, and the filter path uses that helper.
  - file: [src/agents/pi-embedded-helpers/turns.ts:14](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/src/agents/pi-embedded-helpers/turns.ts#L14)
- **Regression coverage for Anthropic replay stripping:** Current tests cover `validateAnthropicTurns` preserving/stripping modern `toolCall`-style replay blocks, including standalone `toolResult` matching before the next assistant turn.
  - file: [src/agents/pi-embedded-helpers.validate-turns.test.ts:498](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/src/agents/pi-embedded-helpers.validate-turns.test.ts#L498)
- **Errored assistant stubs are excluded from transport replay:** Anthropic request building first runs all transcript messages through `transformTransportMessages(...)` before converting them to Anthropic payload blocks.
  - file: [src/agents/anthropic-transport-stream.ts:257](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/src/agents/anthropic-transport-stream.ts#L257)
- **Transport transform drops errored assistant turns:** `transformTransportMessages` explicitly skips assistant messages with `stopReason === "error"` or `"aborted"`, so persisted empty error stubs are not sent back to Anthropic on replay.
  - file: [src/agents/transport-message-transform.ts:116](https://github.com/openclaw/openclaw/blob/bae7b54a85b17d180e4bfc28973a7e4fd1646a93/src/agents/transport-message-transform.ts#L116)
- **Shipped in latest tagged release:** The `v2026.4.22` tag contains the same `toolCall` handling in `src/agents/pi-embedded-helpers/turns.ts` and the same error-turn skip in `src/agents/transport-message-transform.ts`, so this behavior is not only on HEAD.

  - sha: [00bd2cf7a376](https://github.com/openclaw/openclaw/commit/00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5)
