## Summary

Current main still exposes Feishu plugin tools and their schemas to embedded runs, and the strongest follow-up reproduction in the issue resolved as Feishu app permission/publish misconfiguration rather than an OpenClaw tool-injection bug.

## Evidence

- **Feishu tools still register concrete schemas:** `feishu_doc` and `feishu_wiki` are registered with explicit `parameters` schemas, not name-only stubs, in the current main code.
  - file: [extensions/feishu/src/docx.ts:1429](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/extensions/feishu/src/docx.ts#L1429)

- **Plugin tools are appended into the agent tool set:** `createOpenClawTools()` resolves bundled/plugin tools through `resolveOpenClawPluginToolsForOptions()` and returns them alongside core tools for agent runs.
  - file: [src/agents/openclaw-tools.ts:325](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/src/agents/openclaw-tools.ts#L325)

- **Embedded runner uses the effective tool set:** Embedded attempts build tools via `createOpenClawCodingTools()`, keep them in `effectiveTools`, and record/send the resulting tool definitions for the run.
  - file: [src/agents/pi-embedded-runner/run/attempt.ts:510](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/src/agents/pi-embedded-runner/run/attempt.ts#L510)

- **Tool definitions forward schemas:** The tool-definition adapter forwards each tool's `parameters` schema into the runtime tool definition instead of dropping it.
  - file: [src/agents/pi-tool-definition-adapter.ts:216](https://github.com/openclaw/openclaw/blob/bfa6708c03f24c7caa5d0a96a298c292be0d85c1/src/agents/pi-tool-definition-adapter.ts#L216)
  - command: `nl -ba src/agents/pi-tool-definition-adapter.ts | sed -n '216,225p'`
  - sha: [bfa6708c03f2](https://github.com/openclaw/openclaw/commit/bfa6708c03f24c7caa5d0a96a298c292be0d85c1)
- **Generic empty-params failure mode was separately fixed on main:** Current main includes `coerceParamsRecord()` so streamed JSON-string tool args are parsed instead of silently degrading to `{}` before dispatch.
  - file: [src/agents/pi-tool-definition-adapter.ts:278](https://github.com/openclaw/openclaw/blob/86fa8eeb68c64eb0ae25e1eb8c6196061862a010/src/agents/pi-tool-definition-adapter.ts#L278)
