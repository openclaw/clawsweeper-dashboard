## Summary

Keep this open: current main still has source-level paths where unmarked Gemini/plain-text reasoning preambles are treated as user-visible reply text, and PR #60551 is open as the current implementation candidate rather than proof of a shipped fix.

## What This Changes

The issue asks OpenClaw to stop Gemini reasoning or planning text from leaking into channel-delivered assistant replies.

## Best Possible Solution

Keep this as the Gemini/plain-text reasoning leak tracker and review PR #60551 or a replacement that adds conservative shared outbound sanitization with regression coverage for the reported Gemini examples, localized headers, normal prose/code pass-through, and Telegram plus generic delivery paths.

## Reproduction Assessment

Yes for a source-level path: an unmarked Gemini `part.text` containing `Thinking Process:` or a localized reasoning header is emitted as normal text and neither the shared sanitizer nor Telegram splitter removes it. I did not establish a live Gemini/Telegram reproduction in this read-only pass.

## Solution Assessment
