## Summary

Keep open. The PR targets a real Control UI draft replay path, but the host-side text-equality guard can reject a legitimate same-text edit after send while the composer mirror believes it was committed, and the external proof is still jsdom-only.

## What This Changes

The branch adds a one-shot `lastSubmittedDraftClear` guard around Control UI chat draft changes plus a focused regression test for replaying the just-submitted draft after send clears the composer.

## Best Possible Solution

Keep the host draft and composer mirror in sync while ignoring only the actual stale replay, then show browser Control UI proof that sent text clears and an immediate same-text repeat still sends.

## Reproduction Assessment

Yes, source inspection and the added regression show the stale draft replay path at the host-state level. A live browser reproduction was not established in this read-only review.

## Solution Assessment

No. The proposed host-side equality guard is a plausible mitigation, but it is too broad because the composer mirror updates before host acceptance and can diverge when the host ignores a same-text edit.
