## Summary

Keep open. Current `main` still does not support `timeoutSeconds` on `systemEvent` cron payloads, and the submitted PR remains incomplete because it updates only the timeout resolver/types/tests without wiring the field through validation and normalization.

## Evidence

- **Type surface still excludes the field on `systemEvent`:** `CronPayload` and `CronPayloadPatch` still define `systemEvent` as text-only, while `timeoutSeconds` remains part of the agent-turn-only field set.
  - file: [src/cron/types.ts:112](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/cron/types.ts#L112)
  - command: `nl -ba src/cron/types.ts | sed -n '70,136p'`
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **Gateway schema rejects `systemEvent.timeoutSeconds`:** `CronPayloadSchema` and `CronPayloadPatchSchema` allow only `{ kind: "systemEvent", text }` with `additionalProperties: false`, so create/update requests cannot carry `timeoutSeconds` for system events.
  - file: [src/gateway/protocol/schema/cron.ts:130](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/gateway/protocol/schema/cron.ts#L130)
  - command: `nl -ba src/gateway/protocol/schema/cron.ts | sed -n '130,156p'`
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **Normalization strips the field from system events:** `coercePayload()` explicitly deletes `timeoutSeconds` when `next.kind === "systemEvent"`, so the value is removed before execution even if present in raw input.
  - file: [src/cron/normalize.ts:227](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/cron/normalize.ts#L227)
  - command: `nl -ba src/cron/normalize.ts | sed -n '220,238p'`
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **Execution logic still honors explicit timeout only for agent turns:** `resolveCronJobTimeoutMs()` reads `timeoutSeconds` only when `job.payload.kind === "agentTurn"`; `systemEvent` continues to fall back to the default 10-minute timeout.
  - file: [src/cron/service/timeout-policy.ts:16](https://github.com/openclaw/openclaw/blob/938b53698ec17934a5d1a6cffb279c584f64d9f3/src/cron/service/timeout-policy.ts#L16)
  - command: `nl -ba src/cron/service/timeout-policy.ts | sed -n '16,24p'`
  - sha: [938b53698ec1](https://github.com/openclaw/openclaw/commit/938b53698ec17934a5d1a6cffb279c584f64d9f3)
- **Existing review already identified the same missing wiring:** The PR review comment on head commit `237aa77803d494b2898d9dff4315e75611661d00` points out that the schema and normalization paths still reject/strip `systemEvent.timeoutSeconds`, which matches the current `main` code inspection.
  - sha: [237aa77803d4](https://github.com/openclaw/openclaw/commit/237aa77803d494b2898d9dff4315e75611661d00)
