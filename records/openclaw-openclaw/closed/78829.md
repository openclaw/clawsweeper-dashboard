## What This Changes

The issue reports that the external `@tencent-weixin/openclaw-weixin` channel plugin logs successful runtime registration but times out waiting for the Weixin runtime when the Gateway starts the channel.

## Best Possible Solution

Keep the main-branch generic loader fix and let the next release carry it to stable users; WeChat-specific runtime/API issues should remain owned by the external plugin.

## Reproduction Assessment

Yes, at the source level: the current regression test models a configured external deferred channel exporting `{ plugin, setChannelRuntime }` and asserts the runtime setter runs before the channel starts. I did not run a live WeChat/iLink setup in this read-only sweep.

## Solution Assessment

Yes. The current fix is the narrow generic core behavior: carry and invoke the external setup entry's runtime setter without adding Weixin-specific logic to core.
