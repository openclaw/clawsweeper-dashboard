## Summary

Keep open: current main and the latest release still run the reported 30-minute synchronous TRUNCATE checkpoint path, and an open linked PR now specifically targets this issue with closing syntax. The issue should remain open until that PR is reviewed and merged or explicitly rejected.

## What This Changes

The issue reports that periodic SQLite WAL TRUNCATE checkpoints run synchronously on the gateway event loop and can stall all SQLite-backed gateway stores.

## Best Possible Solution

Review and land the linked hybrid PASSIVE-periodic/TRUNCATE-explicit fix if maintainers accept the checkpoint tradeoff, then close this issue from the merged PR and monitor any remaining idle-stall reports separately.

## Reproduction Assessment

No for a full live reproduction in this review: I did not run a 30-minute idle gateway with a large WAL and capture an 8-12s delay. Yes at source level: current main and v2026.5.12 schedule synchronous TRUNCATE checkpoints on the event loop, and the related closed symptom report has matching logs.
