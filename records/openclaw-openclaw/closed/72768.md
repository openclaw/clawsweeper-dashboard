## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Diff the previous and next config snapshots:** `src/gateway/server-runtime-subscriptions.ts:102`
  - body: This listener compares `evt.sourceConfig` with `evt.runtimeConfig`, but current write notifications describe the committed source config and the active runtime config for the same write. When a normal edit leaves source and runtime equal, `changes` is empty and no `config.changed` event is sent; when runtime resolution adds derived values, the event reports those instead of the user's hot-reload diff. Use the reload/write path's previous-to-next config diff instead.
  - confidence: 0.92
- **[P1] Deep-redact changed values before broadcasting:** `src/gateway/config-diff.ts:35`
  - body: The helper only redacts by the immediate key before recursion. If a non-sensitive parent changes type or hits the depth cap while containing a nested `apiKey`, `token`, or similar field, the raw object is placed in `before` or `after` and sent to every admin-scoped client. Every emitted value needs recursive redaction before it enters the payload.
  - confidence: 0.9
- **[P2] Register the schema in the protocol package:** `src/gateway/protocol/schema/config-diff.ts:1`
  - body: Current main's protocol schemas live under `packages/gateway-protocol/src/schema` and are exported through `schema.ts`/`protocol-schemas.ts` for validators and generated clients. Adding `src/gateway/protocol/schema/config-diff.ts` creates an orphan schema file, so the new public event payload will not be part of the canonical protocol contract.
  - confidence: 0.87
