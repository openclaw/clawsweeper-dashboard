## What This Changes

Control UI/WebChat should respect the existing `agents.defaults.timeFormat` setting when rendering chat/session timestamp and duration-style time labels.

## Best Possible Solution

Reuse the existing `agents.defaults.timeFormat` contract, thread its resolved value into Control UI timestamp helpers, and update the WebChat/session display tests without adding a new config key.

## Solution Assessment

Yes, a narrow fix should reuse the existing `agents.defaults.timeFormat` setting and centralize UI hour options. Adding `gateway.controlUi.timezone` belongs to the separate open timezone feature request, not this bug fix.
