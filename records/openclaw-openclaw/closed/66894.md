## Summary

Keep open. The underlying cancellation bug is still present on current main and this PR is relevant, but the patch is incomplete: it misses the gateway chat abort signal, leaves harness-owned compactions outside active-run registration, and bundles a launchd/profile change that conflicts with existing custom-label override behavior.

## What This Changes

The PR wires queued embedded compaction into active-run cancellation for `/compact`, adds a previous-run drain guard, exports active-run helpers, and changes CLI profile handling for `OPENCLAW_LAUNCHD_LABEL`.

## Best Possible Solution

Land a focused compact-cancellation fix that forwards the gateway reply abort signal, registers an abortable handle for queued and harness-owned compaction only while work is active, cascades same-session reply abort when needed, cleans up in `finally`, adds targeted cancellation coverage, and handles the launchd/profile issue separately.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main calls manual `/compact` without `abortSignal` or active-run registration, while the existing abort path only reaches registered active handles or active reply runs.

## Solution Assessment

No. The PR is a useful start, but it is not the best complete fix because it misses the chat abort signal, does not cover harness-owned compaction, and mixes an unrelated launchd/profile behavior change into the same branch.
