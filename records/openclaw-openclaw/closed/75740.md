## Summary

Keep open. Current main still has the reported no-progress overflow retry shape: precheck overflow routes to compaction, a compacted result is enough to retry, and the regression harness still expects three compaction retries before final overflow rather than stopping after an unchanged oversized prompt.

## What This Changes

Prevent embedded/Paperclip overflow recovery from retrying unchanged oversized prompts after auto-compaction reports success.

## Best Possible Solution

Add a narrow overflow-recovery progress gate that refuses another retry unless compaction brings the next prompt under budget or materially reduces the prompt estimate.

## Reproduction Assessment

Yes, at source level. Current main’s overflow-loop harness still models repeated successful compactions followed by repeated overflow, and the runner code lacks a prompt-size progress check before retrying.

## Solution Assessment

Yes. The maintainable fix is a post-compaction budget or shrink check with a clear `context_overflow` result, not a startup archive policy or new config knob.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/75740 with a narrow embedded-runner overflow recovery change. Current main routes aggregate precheck overflow to `compact_only`, treats `contextEngine.compact().compacted` as enough to retry, and can retry the same over-budget prompt three times. Add a progress gate after overflow/precheck compaction: retry only when the next prompt is under budget or the estimated prompt size/tokens-after signal shows material reduction; otherwise return a clear `context_overflow` result and actionable log without consuming identical retries. Preserve legitimate multi-step compaction retries when each attempt makes progress or later succeeds. Do not add startup session-size guards, transcript auto-archive policy, new config keys, or context-engine ownership changes. Add/update regression coverage in the overflow loop tests, and add a one-line `CHANGELOG.md` Fixes entry if the user-visible behavior changes.
