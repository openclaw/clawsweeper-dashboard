## Summary

Current main has not implemented this PR's intended docs fix: the creating-skills guide still creates `hello-world` but shows `name: hello_world` and describes the `name` field as `snake_case`. The PR is narrow docs-only work and remains useful, though the final wording should match the Agent Skills constraints more precisely than just `kebab-case`.

## Best Possible Solution

Keep this PR open and ask for or apply a tiny docs refinement: use `name: hello-world` in the starter example, and describe `name` as a lowercase alphanumeric/hyphen identifier that must match the skill directory name. Land that docs fix once wording is precise; track runtime validation separately only if maintainers want OpenClaw to enforce the external spec.
