## Summary

Keep open. Current `main` does not show a maintainer-safe fix for #55445: Slack Bolt app creation still does not pass any self-filter override, and `message_changed` sender resolution still only trusts `message.user` / `previous_message.user` / `bot_id`, which matches the reported misrouting failure mode rather than resolving it.
