## Summary

Current main already fixes the webchat/control-ui provider-prefix drift in model switching, and the same implementation is present in the shipped `v2026.4.22` release.

## Evidence

- **Picker renders provider-qualified option values:** The control-ui model selector reads `options` from `resolveChatModelSelectState(state)` and uses each `entry.value` directly for the `<option value=...>` payload, rather than rebuilding a model ref from the current session provider.
  - file: [ui/src/ui/chat/session-controls.ts:83](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/ui/src/ui/chat/session-controls.ts#L83)
  - command: `nl -ba ui/src/ui/chat/session-controls.ts | sed -n '70,115p'`
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Model switch RPC forwards the selected value unchanged:** `switchChatModel` stores `createChatModelOverride(nextModel)` and sends `model: nextModel || null` in `sessions.patch`. There is no concatenation with the previously active provider in this path.
  - file: [ui/src/ui/chat/session-controls.ts:234](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/ui/src/ui/chat/session-controls.ts#L234)
  - command: `nl -ba ui/src/ui/chat/session-controls.ts | sed -n '232,264p'`
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **State resolution prefers catalog-qualified refs over stale provider state:** `resolveChatModelOverrideValue` and `resolvePreferredServerChatModelValue` normalize cached/server model values through the catalog, including slash-containing refs, and explicitly avoid producing nonsense double prefixes under stale providers.
  - file: [ui/src/ui/chat-model-ref.ts:106](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/ui/src/ui/chat-model-ref.ts#L106)
  - command: `nl -ba ui/src/ui/chat-model-ref.ts | sed -n '1,160p'`
  - sha: [9ece33c50506](https://github.com/openclaw/openclaw/commit/9ece33c50506ba031df382f69bde65dfda19bea5)
- **Regression tests cover stale-provider drift cases:** Tests assert that stale providers are corrected to the catalog-backed provider and that bad values like `zai/openai/gpt-5-mini` are excluded from picker state.
  - file: [ui/src/ui/chat-model-select-state.test.ts:76](https://github.com/openclaw/openclaw/blob/9ece33c50506ba031df382f69bde65dfda19bea5/ui/src/ui/chat-model-select-state.test.ts#L76)
