## Summary

Close as implemented on current main: the Control UI chat history loader now preserves visible local user/assistant tail messages when a final `chat.history` refresh returns a stale or empty snapshot, with regression coverage for the reported disappearance path.

## What This Changes

The issue reports WebChat assistant replies disappearing after a tool-run final event triggers a stale `chat.history` reload that replaces local chat messages.

## Best Possible Solution

Keep the current optimistic-tail history merge behavior; distinct stream-segment or markdown rendering bugs should remain in focused reports with their own reproductions.

## Reproduction Assessment

No current-main failing reproduction is available from source inspection: the former stale-history replacement path now preserves local assistant/user tails, and tests cover stale and empty `chat.history` responses.

## Solution Assessment

Yes. Preserving only the local optimistic tail is the narrow maintainable fix for this race because it avoids wiping visible replies while still letting persisted `chat.history` replace local state once it catches up.
