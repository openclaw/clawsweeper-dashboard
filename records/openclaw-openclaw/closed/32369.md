## Summary

Current main already supports changing models without re-entering provider credentials: the configure wizard has a dedicated `model` section, its auth flow allows `skip` and then goes directly to default-model/allowlist pickers, and the docs/FAQ explicitly direct users to `openclaw configure --section model` or `openclaw models set ...` for model-only changes. The same implementation is present in release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).

## Risks / Open Questions

- The exact implementation is a model-only flow (`skip` auth / `configure --section model` / `models set`) rather than auto-detecting stored auth profiles inside every provider auth prompt, but it satisfies the requested user outcome of changing models without retyping keys.

## Close Comment

Closing as implemented on current `main` after Codex review. OpenClaw already supports changing models without retyping provider credentials: `openclaw configure --section model` goes through the model flow, and `src/commands/configure.gateway-auth.ts:106` includes a `skip` auth choice that jumps straight to the default-model picker (`promptDefaultModel(...)`) instead of asking for a new API key. The user-facing docs also point to `openclaw models set ...` and `openclaw configure --section model` as the supported ways to switch models without rewriting config (`docs/cli/configure.md:12`, `docs/help/faq.md:2254`). This behavior is already present in the shipped release `v2026.4.22` (`00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5`).
