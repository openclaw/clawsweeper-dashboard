## Summary

Keep open: this targets a real source-reproducible config CLI bug, but the current patch can break documented bracket-index array writes and the supplied proof does not exercise the patched CLI against an on-disk config.

## What This Changes

The PR changes config set/patch path mutation calls to always pass `numericObjectKeys: true` and adds a config set regression test for a Discord guild snowflake key.

## Best Possible Solution

Carry bracket/index intent through parsed path segments, or make missing-container creation schema-aware, so Discord snowflake record paths become objects while documented `agents.list[index]` paths remain arrays.

## Reproduction Assessment

Yes, at source level: current main still creates array parents for numeric-looking set-path segments while `channels.discord.guilds` is a string-keyed record. I did not execute the CLI because this review is read-only.

## Solution Assessment

No. The fix direction is right for Discord snowflake keys, but forcing all path writes to object-key semantics is not the narrowest maintainable solution because it loses bracket-index array intent.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve bracket-index array creation:** `src/cli/config-cli.ts:1751`
  - body: With this change, every path write passes `numericObjectKeys: true`, but `parsePath` has already collapsed bracket notation into plain string segments. Keep bracket-index intent or consult schema when choosing missing containers.
