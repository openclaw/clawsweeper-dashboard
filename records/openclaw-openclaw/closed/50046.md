## Summary

Keep this PR open. Current main still does not pass a Discord gateway WebSocket `handshakeTimeout`, and the pinned Carbon 0.16.0 implementation does not supply one upstream. The PR’s bug remains plausible, but the March patch shape is stale against the newer OpenClaw wrapper and should be rebased instead of applied as-is.

## Best Possible Solution

Keep the PR open, but rebase or replace the patch for current main: add a plugin-owned 30s `handshakeTimeout` to the existing WebSocket constructor options, widen `DiscordGatewayWebSocketCtor` and test mocks to accept `handshakeTimeout?: number`, preserve the injected constructor, proxy agent, capture hooks, transport-liveness events, and `binaryType` assignment, then update focused direct and proxy WebSocket tests to assert the timeout.
