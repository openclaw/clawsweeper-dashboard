## Summary

Keep open. Current `main` still exits on every `uncaughtException`, so the PR is not obsolete, but the proposed patch is not safe to auto-close or treat as implemented: it would add transient suppression on the uncaught-exception path without preserving the fatal-error precedence that current unhandled-rejection handling relies on.
