## Summary

Current main and the latest release already align Android's advertised canvas capability with concrete canvas/A2UI node commands and dispatcher handlers, so the reported 2026.5.25 client behavior is obsolete.

## What This Changes

The issue reports that an Android node advertises the `canvas` capability while exposing no `canvas.*` commands, causing canvas requests and settings-canvas restores to fail.

## Best Possible Solution

Keep the current Android registry/dispatcher contract and have affected users update from Android client `2026.5.25` to `v2026.5.28` or newer.

## Reproduction Assessment

No current-main reproduction path remains. Source inspection shows Android `node.describe` metadata now includes both the `canvas` capability and concrete `canvas.*` command handlers; the reported client version was older than the checked release tag.

## Solution Assessment

Yes. Fixing the Android node registry and dispatcher is the right owner boundary because callers can now trust the advertised command surface instead of adding fallback logic for a stale client mismatch.
