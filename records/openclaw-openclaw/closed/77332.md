## Summary

Close as ClawHub scope: this is a useful and well-specified provider variant, but current main still lacks Token Plan Xiaomi choices and the existing provider plugin surfaces can express the requested auth choices, base URLs, protocols, and catalog outside OpenClaw core.

## What This Changes

Add Xiaomi MiMo Token Plan regional OpenAI-compatible and Anthropic-compatible onboarding choices for China, Singapore, and Europe endpoints.

## Best Possible Solution

Keep OpenClaw core and the bundled standard Xiaomi provider as-is, and implement Token Plan regional variants as a ClawHub provider plugin unless that work exposes a missing generic provider-plugin API.

## Solution Assessment

Yes for this cleanup decision: the current plugin SDK already supports provider auth choices, catalogs, base URLs, and onboarding metadata, so an external ClawHub provider plugin is the narrowest maintainable path unless maintainers explicitly choose bundled promotion.
