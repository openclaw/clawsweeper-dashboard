## Summary

Close as implemented_on_main. Current OpenClaw already ships Gemini-backed web search using native Google Search grounding; the user-facing setup is selecting the Gemini `web_search` provider rather than a separate per-model toggle.

## Evidence

- **Google plugin registers Gemini web search:** The bundled Google plugin registers a Gemini web-search provider on current `main`, so this capability is part of the shipped plugin surface rather than an unimplemented idea.
  - file: [extensions/google/index.ts:163](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/google/index.ts#L163)
  - command: `nl -ba extensions/google/index.ts | sed -n '155,170p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Gemini web-search provider is first-class config surface:** The provider is exposed as `Gemini (Google Search)` with its own credential path and tool factory, which matches the requested 'use Gemini search instead of local search' behavior even though the UX is provider selection rather than a model checkbox.
  - file: [extensions/google/src/gemini-web-search-provider.ts:54](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/google/src/gemini-web-search-provider.ts#L54)
  - command: `nl -ba extensions/google/src/gemini-web-search-provider.ts | sed -n '50,80p'`
  - sha: [95b7a85f0607](https://github.com/openclaw/openclaw/commit/95b7a85f0607784d8e33b4ce7e42617174196490)
- **Runtime uses native Google Search grounding:** The Gemini search runtime calls Gemini `generateContent` and sends `tools: [{ google_search: {} }]`, which is the exact native grounding mechanism requested in the issue.
  - file: [extensions/google/src/gemini-web-search-provider.runtime.ts:68](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/extensions/google/src/gemini-web-search-provider.runtime.ts#L68)

- **User-facing docs describe Gemini grounding setup:** Docs explicitly say OpenClaw supports Gemini with built-in Google Search grounding and show the config to set `tools.web.search.provider` to `gemini`.
  - file: [docs/tools/gemini-search.md:10](https://github.com/openclaw/openclaw/blob/95b7a85f0607784d8e33b4ce7e42617174196490/docs/tools/gemini-search.md#L10)

## Risks / Open Questions

- The current UX is not a Cherry Studio-style per-model toggle; users enable the behavior by choosing the Gemini `web_search` provider, so someone looking specifically for a model-level switch may still perceive a gap in ergonomics.
