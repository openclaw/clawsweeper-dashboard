## Summary

Current main handles this at the group/channel delivery layer. Group and channel turns now default to `message_tool_only`, which suppresses automatic partial/block/tool/final source delivery while still letting the agent post visible output with `message(action=send)`. This covers the reported Slack multi-agent group-channel noise without adding the exact proposed `suppressIntermediateText` key.

## Best Possible Solution

Close this as implemented on current main. Keep the current `messages.groupChat.visibleReplies: "message_tool"` default and message-tool-visible output model; a future narrower issue can ask for an opt-in mode that preserves automatic final replies while dropping only pre-tool narration.
