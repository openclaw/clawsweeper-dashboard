- **[P2] Keep snapshot helper bindings:** `src/process/command-queue.test.ts:31`
  - body: This merge result removes the `getCommandLaneSnapshot` and `getCommandLaneSnapshots` bindings, but `expectLaneSnapshotFields()` still calls `getCommandLaneSnapshot()` and the stable-order test still calls `getCommandLaneSnapshots()`.   - confidence: 0.96
- **[P2] Notify waiters after rejected tasks:** `src/process/command-queue.ts:395`
  - body: On the rejection path, `completeTask()` removes the task id from the active set, but the PR no longer calls `notifyActiveTaskWaiters()` before pumping/rejecting. A `waitForActiveTasks()` caller that captured this task can now wait until its timeout when the last active task rejects.
  - confidence: 0.9
- **[P2] Preserve suspended dynamic lanes:** `src/process/command-queue.ts:545`
  - body: Current main supports `maxConcurrent=0` as an intentional suspended-lane state used by session suspension. Pruning dynamic lanes with `maxConcurrent <= 1` in idle clear/reset paths can delete that suspended state, so later work recreates the lane at the default concurrency and can run before the suspension is resumed.
  - confidence: 0.88
