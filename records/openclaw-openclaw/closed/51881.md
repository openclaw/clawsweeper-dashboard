## Summary

Keep open. Current main still lets a sniffed `image/apng` MIME flow through gateway attachment parsing into Anthropic image payloads, and open PR #51964 already uses closing syntax for this issue.

## What This Changes

Normalize PNG/APNG attachment MIME handling so Claude-compatible vision payloads use `image/png` instead of `image/apng`.

## Best Possible Solution

Land or replace #51964 with a current-main patch that normalizes `image/apng` to `image/png` in shared MIME detection, with focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes, from source inspection. `file-type` v22 can return `image/apng`, OpenClaw returns that MIME unchanged, the gateway prefers it as `finalMime`, and Anthropic transport forwards it as `media_type`.

## Solution Assessment

Yes. Shared `detectMime` normalization is the narrowest maintainable fix because gateway attachments and other media helpers already consume that detector; provider-specific handling would leave other paths exposed.
