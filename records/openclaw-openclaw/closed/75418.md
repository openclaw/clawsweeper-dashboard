## Summary

Close as implemented on current main. The reported v2026.4.29 Telegram embedded-agent cold-start regression matches the stage instrumentation in the runner, and current main contains the post-release #75360 latency fix cluster that moves optional generation/provider/catalog work and typing startup off the reply critical path; the fix is not yet in a release tag, so users on npm 2026.4.29 still need a newer package.

## What This Changes

The issue reports that embedded agent startup on Linux/Node 24 regressed to about 70 seconds per Telegram-triggered run after updating to OpenClaw 2026.4.29.

## Best Possible Solution

Keep the current main hot-path changes and ship them in the next package release; if comparable startup timings persist after a post-2026.4.29 build, the follow-up should include fresh stage summaries from that build and the active channel/provider config.

## Reproduction Assessment

Yes, for triage purposes: the report includes precise stage timings whose names match the embedded-runner instrumentation, and the current-main fix targets those exact startup/prep stages. I did not run a live Telegram/Moonshot reproduction in this read-only review.

## Solution Assessment

Yes. The current main direction fixes the maintainable boundary by avoiding optional provider/catalog/generation work and nonessential typing awaits on the reply critical path rather than adding a Telegram-specific workaround.
