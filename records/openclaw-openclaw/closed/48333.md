## Summary

Current `main` still returns full cron job objects through the agent-facing `cron` tool, with no compact list mode and no tool-level `get` action for detail-on-demand. The issue remains valid and should stay open.

## Evidence

- **Cron tool still lacks a detail-on-demand action:** `CRON_ACTIONS` only includes `status`, `list`, `add`, `update`, `remove`, `run`, `runs`, and `wake`; there is no `get` action in the agent tool surface.
