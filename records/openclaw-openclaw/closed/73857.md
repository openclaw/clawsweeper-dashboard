## Summary

Current main implements the central Slack Socket Mode fix: OpenClaw now exposes `channels.slack.socketMode.clientPingTimeout`, `serverPingTimeout`, and `pingPongLoggingEnabled`, defaults the Slack SDK pong timeout to 15s, passes those options into the Slack SocketModeReceiver, documents the tuning, and has tests for config validation and receiver pass-through. The proof commit is on current main but is not contained in any tag I could find, and v2026.4.26 lacks the new fields, so this is main-only until the next release.

## Best Possible Solution

Close as implemented on current main. Keep the typed Slack Socket Mode transport config and 15s default, ship it in the next release, and only open a narrower follow-up for worker-thread Slack transport if real current-main reports still reproduce with a 15-30s `channels.slack.socketMode.clientPingTimeout`.
