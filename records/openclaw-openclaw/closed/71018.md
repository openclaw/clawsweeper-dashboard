## Summary

Current main already wires Feishu/Lark tool activity into the reply path and exposes detailed tool-call data through the plugin hook path. The current Feishu plugin renders live tool status on streaming cards, and core now forwards tool lifecycle events plus after_tool_call payloads with params/result/duration/session context.

## Best Possible Solution

Keep the current main implementation: core forwards tool lifecycle events generically, after_tool_call carries the detailed data needed by plugins, and the Feishu/Lark plugin displays live tool status in streaming cards. If maintainers want a persistent final-card step list, track that as a narrower Feishu UX enhancement.
