## Summary

Current main already fixes the Control UI model-picker behavior through the UI path: picker options keep alias/friendly labels but submit canonical provider-qualified model refs, and the /model flow preserves the requested qualified ref when the backend returns a stale provider. That shipped in v2026.4.22, so this PR is redundant.
