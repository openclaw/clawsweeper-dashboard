## Summary

Current main still routes `openclaw memory index` and `memory status --index` through watcher-backed memory managers. The builtin backend starts Chokidar for non-status managers, and the QMD backend uses a full manager for non-status requests that also starts Chokidar. The related #49851 remains an open, unmerged implementation candidate for a watcher-free one-shot CLI path, so this issue should stay open until that or an equivalent fix lands.

## Best Possible Solution

Keep this issue open and review, rebase, or replace #49851 with an equivalent watcher-free one-shot memory manager mode. The fix should route `openclaw memory index` and `memory status --index` through that mode for both builtin and QMD backends, preserve normal long-lived Gateway/runtime watching, and add regression tests proving the one-shot CLI paths do not start Chokidar while normal managers still watch.
