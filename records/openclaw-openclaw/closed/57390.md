## Summary

Current `main` no longer declares `node-llama-cpp` as an optional peer dependency and now documents/runtime-guides a sibling global install (`npm i -g node-llama-cpp@3.18.1`) instead of a package-local install under `openclaw/node_modules`. That addresses the npm global-update pruning pattern described in #57390. The latest tagged release available in this checkout, `v2026.4.23`, still had the old optional-peer setup, so this appears fixed on `main` but not yet attributable to a released tag from the available evidence.
