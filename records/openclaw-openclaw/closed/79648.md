## Summary

Close as ClawHub scope: Lemonade support is useful and the branch includes credible real behavior proof, but current scope docs, plugin docs, and provider SDK surfaces support this as a standalone provider plugin rather than a bundled OpenClaw core plugin; prior Lemonade provider work was closed on the same community-plugin path, and this branch is also not mergeable as-is.

## What This Changes

The PR adds a bundled Lemonade model-provider plugin that reuses Ollama-compatible helper code, registers Lemonade onboarding/discovery metadata, and adds labeler/lockfile workspace entries.

## Best Possible Solution

Publish Lemonade as a standalone provider plugin through ClawHub or npm, reusing the public provider SDK and shared self-hosted/Ollama-compatible seams; reopen core only for a missing generic SDK API or explicit official bundled-provider ownership.

## Reproduction Assessment

Not applicable: this is a feature PR, not a current-main bug report. The submitted screenshots are credible after-change proof for the branch, but there is no broken existing behavior to reproduce on main.

## Solution Assessment

No: bundling a new optional Lemonade provider in OpenClaw core is not the narrowest maintainable solution under the current plugin and ClawHub direction. The safer path is an external provider plugin unless maintainers explicitly choose official bundled Lemonade ownership.
