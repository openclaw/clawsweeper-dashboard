## Summary

Current main already repairs the reported poisoned assistant-turn shape before session load and before provider replay. The same repair path is present in the v2026.4.26 release, and OpenAI-compatible replay policy also validates strict turn shape so consecutive repaired assistant tail entries are merged before Venice/OpenAI-compatible payload construction.

## Best Possible Solution

Close as implemented. Users on 2026.4.25 should update to v2026.4.26 or newer; if a body-less Venice 400 still reproduces after that, the next report should include a sanitized replay payload or cache-trace stage showing which remaining message shape is rejected.
