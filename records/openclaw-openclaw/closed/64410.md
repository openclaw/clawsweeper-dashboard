## Summary

Current `main` already batches queued busy-session messages in `collect` mode by default, exposes queue controls including `imessage` overrides, and has direct tests covering same-route batching versus separate-route delivery. The same implementation is present in the shipped `v2026.4.23` release.

## Evidence

- **Default busy-queue mode is collect:** Queue settings resolve to `collect` when no inline/session/channel override is set, so queued followups default to batched handling rather than one-by-one replay.
  - file: [src/auto-reply/reply/queue/settings.ts:7](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/auto-reply/reply/queue/settings.ts#L7)
