## Summary

Current `main` already fixes this. `openclaw node status` routes through the shared Windows Scheduled Task runtime reader, which now derives running/stopped from locale-invariant numeric `Last Run Result` codes instead of localized `Status` strings, and regression tests cover German/French localized states. The changelog records this as shipped in `2026.3.7`.

## Risks / Open Questions

- The current logic intentionally reports `unknown` when Windows does not provide a numeric `Last Run Result`; that is a separate edge case, not the reported German-locale false `stopped` bug.
