## Summary

Current main implements the relevant fix path: native Ollama chat now gets provider-owned OpenAI-compatible replay normalization, and auto-sourced fallback model overrides are treated as transient so long-lived sessions retry the configured primary instead of staying pinned to fallback.

## Best Possible Solution

Close as implemented on main. Keep the native Ollama replay-policy override and auto-fallback self-healing behavior; ship it in the next release. If the same symptom appears after current main, the follow-up should include native Ollama request/replay diagnostics from the affected long-lived session.
