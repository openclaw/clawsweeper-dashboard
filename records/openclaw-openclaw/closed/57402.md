## Summary

Current `main` already registers and handles Feishu `card.action.trigger` events, routes callback payloads into the normal Feishu message/command flow, and has dedicated coverage for card-action dispatch and replay handling. The implementation appears to have landed after the latest tagged release, so it is implemented on `main` but not yet tied to a shipped release tag.
