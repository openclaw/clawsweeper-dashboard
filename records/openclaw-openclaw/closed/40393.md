## Summary

Keep open. Current `main` fixed the ACP `model` half of this report, but `sessions_spawn(runtime:"acp")` still accepts `thinking` in the public tool schema and then silently drops it before the ACP spawn/runtime path.

## Evidence

- **Public tool schema still advertises `thinking` for `sessions_spawn`:** `SessionsSpawnToolSchema` exposes both `model` and `thinking`, and `execute()` reads both. In the ACP branch, only `model` is forwarded to `spawnAcpDirect`; `thinking` is not included in the ACP spawn params object.
  - file: [src/agents/tools/sessions-spawn-tool.ts:111](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/agents/tools/sessions-spawn-tool.ts#L111)
  - command: `nl -ba src/agents/tools/sessions-spawn-tool.ts | sed -n '100,120p;183,190p;245,267p'`
  - sha: [f00d65a30496](https://github.com/openclaw/openclaw/commit/f00d65a30496335fb068f01fe529dbba484d4312)
- **ACP spawn API still has no `thinking` field:** `SpawnAcpParams` includes `model` but no `thinking`. `initializeAcpSpawnRuntime()` likewise accepts only `model`, and the initial `agent` dispatch payload contains no thinking override metadata.
  - file: [src/agents/acp-spawn.ts:95](https://github.com/openclaw/openclaw/blob/f00d65a30496335fb068f01fe529dbba484d4312/src/agents/acp-spawn.ts#L95)

Runtime-option validation allowlists the same keys, so a thinking override cannot flow through this path today.
