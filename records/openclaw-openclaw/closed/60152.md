## Summary

Current `main` already supports config-based skill disabling via `skills.entries.<skillKey>.enabled: false`, plus agent-level skill allowlists via `agents.defaults.skills` / `agents.list[].skills`. Disabled skills are filtered before the `<available_skills>` prompt is built, and current docs also clarify that `SKILL.md` contents are loaded on demand rather than injected by default.

## Evidence

- **Config surface already has a per-skill disable flag:** `SkillConfig` includes `enabled?: boolean`, and `SkillsConfig` exposes per-skill overrides under `skills.entries` rather than a top-level `skills.disabled` array.
  - file: [src/config/types.skills.ts:3](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/config/types.skills.ts#L3)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Loader excludes disabled skills from runtime eligibility:** `shouldIncludeSkill()` resolves `skills.entries[skillKey]` and returns `false` immediately when `enabled === false`, before prompt assembly.
  - file: [src/agents/skills/config.ts:73](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/agents/skills/config.ts#L73)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Only eligible skills are rendered into `<available_skills>`:** `resolveWorkspaceSkillPromptState()` filters entries through `filterSkillEntries()` and builds the prompt from the remaining eligible entries only.

  - file: [docs/tools/skills-config.md:13](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/docs/tools/skills-config.md#L13)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)

## Risks / Open Questions

- The exact schema proposed in the issue (`skills.disabled: []`) does not exist; the equivalent shipped behavior uses `skills.entries.<skillKey>.enabled: false` and agent allowlists instead.
