## Summary

Current main has replaced the eager subcommand/plugin dispatch path described in #69526 with primary-command-only registration, lazy command groups, help/version config-bootstrap skips, and cold plugin-registry paths. The reported `cron --help` case no longer has to load all subcommand graphs or validated plugin CLI config on current main.

## Best Possible Solution

Close this issue as implemented on current main. Keep the current lazy primary-command dispatch and cold plugin-registry startup paths; if a user can still reproduce a specific slow command after the 2026.4.25 line ships, track that as a fresh, command-specific performance regression with a current CPU profile.

## Evidence
