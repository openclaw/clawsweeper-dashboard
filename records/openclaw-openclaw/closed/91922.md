## Summary

The requested core capability already exists and is shipped: OpenClaw plugins can inspect complete tool arguments before execution and block the call with a reason, allowing clawband to integrate through the existing plugin API.

## What This Changes

Requests a blocking, content-aware pre-tool hook that lets external policy programs allow or deny commands before execution.

## Best Possible Solution

Keep the released generic `before_tool_call` API as the single core policy boundary and implement clawband's stdin/stdout subprocess protocol in clawband or a third-party OpenClaw plugin.

## Reproduction Assessment

Not applicable as a bug reproduction; current source, documentation, tests, and release history directly verify the requested pre-execution blocking capability.

## Solution Assessment

Yes. The existing generic typed plugin hook is more maintainable than a duplicate command-specific lifecycle hook because it covers shell and other tools through one policy boundary.
