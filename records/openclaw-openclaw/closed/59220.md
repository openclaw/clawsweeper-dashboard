## Summary

Current main implements the requested cleanup well enough: web_search is no longer DuckDuckGo-only, supports multiple configurable API-backed providers, documents DuckDuckGo as an experimental key-free fallback, and the runtime falls through candidate providers when search is not explicitly pinned to one provider.

## Evidence
