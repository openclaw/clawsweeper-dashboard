## Summary

Keep this PR open. It is maintainer-authored/protected, and current main still has the two timeout propagation gaps the PR targets: image understanding uses the timeout for the outer abort wrapper but does not pass it into provider request options, and Ollama native streams ignore generic `options.timeoutMs` when resolving the guarded fetch timeout.

## Best Possible Solution

Keep this PR open for maintainer review and likely landing if CI/security checks confirm it. The right fix boundary is the PR's current scope: forward the resolved image timeout into provider request options, make Ollama native streams honor generic `timeoutMs` when no provider-specific timeout is set, and keep the focused regression tests. No docs or changelog entry appears necessary for this internal propagation fix unless maintainers want to call out improved timeout reliability.
