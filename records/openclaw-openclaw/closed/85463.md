## Summary

Keep open. The feature has useful real Telegram proof, but the implementation diverges from existing Telegram group config precedence in multi-account setups and needs a small fix before merge; the group-visible tool summary behavior also needs explicit maintainer acceptance because it can expose tool/progress details when enabled.

## What This Changes

The PR adds a per-Telegram-group `verboseToolSummaries` config option, schema/generated metadata, docs, and dispatch tests so opted-in Telegram groups can receive tool/progress summaries.

## Best Possible Solution

Keep the opt-in default-off on Telegram group config, but implement it through the existing group-config precedence rules with tests and docs for wildcard and multi-account behavior.

## Reproduction Assessment

Not applicable as a feature PR. Current main source shows normal Telegram groups suppress tool summaries, and the PR body supplies live after-fix Telegram proof for the new opt-in.

## Solution Assessment

No, not yet. The Telegram group config boundary is the right place for the opt-in, but the resolver should match existing multi-account group precedence or clearly document a deliberate per-account-only rule.
