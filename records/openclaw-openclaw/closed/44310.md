## Summary

Issue #44310 asks for per-model request/generation timeout controls in the model fallback chain. Current main does not implement that capability, but the same remaining work is already tracked by the open canonical issue #8724, while #65551 remains open for the broader local MLX/RAM-pressure cron behavior.

## Evidence
