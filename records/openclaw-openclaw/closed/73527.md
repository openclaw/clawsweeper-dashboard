## Summary

Current main already has the Gemini memory embedding path wired: the bundled Google plugin registers a `gemini` memory embedding adapter, its manifest declares the `memoryEmbeddingProviders` contract, and memory-core resolves providers through the config-aware runtime resolver before throwing the reported unknown-provider error. The named `3eb2a9d` shim cleanup was followed on main by `2ccdbc7d`, which restored the memory-host wildcard shims. I did not find a release tag containing that main-line state yet, so users on 2026.4.23 still need a newer build.

## Best Possible Solution

Keep the current main implementation and ship it in the next release; no further core change is needed unless someone can reproduce `provider: "gemini"` failing on a build that includes the current Google plugin registration and memory-host shim restoration.
