## Summary

Current `main` contains explicit fixes for the Windows/pnpm dashboard failure pattern from #38471: Control UI root resolution now handles symlink/realpath package layouts, bundled auto-detected Control UI roots are allowed to serve pnpm-style hardlinked assets, and `openclaw dashboard` now opens/copies a token-bootstrap URL via `#token=` instead of leaving auth to manual recovery. This appears fixed on `main` but not yet shipped in the latest tagged release `v2026.4.22`.

## Evidence

- **Control UI auto-detection now resolves package roots from argv/realpath candidates:** `resolveControlUiDistIndexPath()` now checks both the original entrypoint and its realpath, then falls back to locating the `openclaw` package root and `dist/control-ui/index.html`. That directly addresses pnpm/global-install path layouts.
