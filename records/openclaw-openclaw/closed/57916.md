## Summary

Current `main` already avoids the busy-parent direct gateway path for completion announces by detecting an active requester session and steering the completion into that active embedded run instead of calling the gateway agent lane first.

## Evidence

- **Busy requester sessions are detected before direct gateway delivery:** `sendSubagentAnnounceDirectly()` resolves requester activity and, when `expectsCompletionMessage` is true and the requester session is active, calls `queueEmbeddedPiMessage()` and returns `path: "steered"` without reaching `callGateway()`.
