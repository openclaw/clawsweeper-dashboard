## Summary

Keep open. Current main still serializes non-mcporter multi-collection QMD CLI searches: `runQueryAcrossCollections` awaits each per-collection `runQmd` inside a `for` loop, and `runCliCommand` has no `AbortSignal` support. The PR is an unmerged, active implementation candidate with focused tests for parallel fan-out and sibling abort behavior; it is not obsolete or implemented on main.

## Best Possible Solution

Keep this PR open as the current implementation candidate. Review it on top of current main, decide whether to add a concurrency cap and stronger abort/error cleanup, then run the memory-core changed gate before merge. Closing now would leave the observable non-mcporter multi-collection QMD search latency issue unresolved on main.
