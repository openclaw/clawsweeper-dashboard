## Summary

Keep open: current main still has the Discord `thread-reply` attachment fallback mismatch, and the linked open fix PR #43574 uses closing syntax, so this issue should stay open until that PR or a same-scope replacement lands.

## What This Changes

Discord `thread-reply` should accept attachment paths from `media`, `path`, and `filePath` the same way the Discord send path does.

## Best Possible Solution

Review and land #43574, or a same-scope replacement, to mirror the existing fallback chain in `thread-reply` and add regression coverage for `media`, `path`, and `filePath` precedence.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: call Discord `message` with `action: "thread-reply"` and `filePath` or `path`; those parameters are never read before the runtime receives `mediaUrl`.

## Solution Assessment

Yes. The narrow maintainable fix is to align the Discord `thread-reply` mapper with the existing `send` fallback chain and cover it with focused tests; no core API or config change is needed.
