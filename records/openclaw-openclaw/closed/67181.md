## Summary

Keep open. Current main still sends the exact user-visible resume-fallback banner during direct external fallback after a session resume failure, including for successful async exec results. The issue also has a Telegram confirmation, so this is a shared exec approval follow-up bug rather than a Discord-only channel issue.

## Best Possible Solution

Keep this issue open. Fix `src/agents/bash-tools.exec-approval-followup.ts` so successful direct fallback delivery sends only sanitized command output, with session resume diagnostics logged internally, then update `src/agents/bash-tools.exec-approval-followup.test.ts` to expect `all good` without the internal prefix for successful `code 0` fallback delivery. Review #68129 as a possible implementation candidate, and treat later #72143 as duplicate or adjacent unless maintainers choose it as the canonical tracker.
