## Summary

Keep open. Current main still intentionally renders Telegram tool-progress preview lines by default during partial streaming; users can opt out with config, but changing the default disclosure policy needs maintainer approval.

## What This Changes

Request that Telegram message streaming continue for assistant text while hiding tool calls, progress, and raw tool output from the Telegram preview by default or behind an explicit verbose mode.

## Best Possible Solution

Maintainers should choose the Telegram disclosure policy, then align the runtime default, focused Telegram tests, docs, config hints, and changelog around either hidden-by-default progress or an explicit opt-in activity mode.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: run Telegram dispatch with streamMode partial, leave streaming.preview.toolProgress unset, trigger onToolStart/onItemEvent, and current tests expect a visible Working… preview with tool and command text.

## Solution Assessment

Unclear. Hiding tool progress while keeping assistant text streaming is privacy-preserving, but current docs and tests intentionally define the opposite default, so the narrow implementation depends on maintainer policy.
