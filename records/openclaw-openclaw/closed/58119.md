## Summary

The reported daily `contextToken` expiry/reconnect problem belongs to the external `openclaw-weixin` WeChat plugin, not OpenClaw core on current `main`. This checkout only documents that integration and does not contain the WeChat runtime to fix or verify.

## Evidence

- **WeChat integration is external:** The WeChat docs state that OpenClaw connects through Tencent's external `@tencent-weixin/openclaw-weixin` channel plugin.
  - file: [docs/channels/wechat.md:10](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/docs/channels/wechat.md#L10)
- **Token and monitoring ownership stays in plugin:** The docs explicitly assign Tencent iLink API calls, context tokens, and account monitoring to the external plugin rather than OpenClaw core.
  - file: [docs/channels/wechat.md:41](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/docs/channels/wechat.md#L41)
- **Channel index marks WeChat as external:** The channel index describes WeChat as an external plugin.
  - file: [docs/channels/index.md:35](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/docs/channels/index.md#L35)
- **Project scope favors plugin ownership:** The vision doc says optional capability should usually ship as plugins and be hosted and maintained in their own repositories.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/VISION.md#L54)
- **No WeChat runtime source in this repo:** Searching tracked files for `weixin`/`wechat` only returns the WeChat docs page, with no in-repo runtime implementation to inspect or patch.
  - command: `rg --files | rg "weixin|wechat"`

## Risks / Open Questions

- `clawhub` is the closest available closure bucket for an external plugin ownership issue. If maintainers intentionally want third-party plugin bugs tracked in the core repo, this would be too aggressive.
