## Summary

Keep open: the reset-archive fix addresses a real source-visible gap, but the patch still makes deleted transcript archives indexed even though the agent visibility guard cannot authorize deleted sessions and the CLI search path is not filtered by that guard.

## What This Changes

The PR indexes reset/deleted session transcript archives, maps `.jsonl.reset.*` and `.jsonl.deleted.*` memory hits back to transcript stems, adds focused tests, and updates the changelog.

## Best Possible Solution

Land the reset archive indexing and stem-resolution fix with regression coverage, while keeping `.deleted` archives opaque unless maintainers define an explicit deleted-session recovery and visibility model.

## Reproduction Assessment

Yes. Source inspection shows current main skips archive content and drops archive hit paths, and also shows the PR's deleted-archive problem: deleted sessions lose their store key while CLI search is unfiltered.

## Solution Assessment

No. The reset archive portion is narrow and maintainable, but indexing `.deleted` archives is not the best fix without a reviewed visibility policy; the safer path is reset-only handling or a separate deleted-archive design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep deleted transcript archives out of memory search:** `packages/memory-host-sdk/src/host/session-files.ts:74`
  - body: This change treats `.jsonl.deleted.*` archives as usage-counted content, but `sessions.delete` removes the store entry the agent visibility guard needs and the CLI `memory search` path does not apply that guard. Please keep deleted archives skipped here or add an explicit deleted-session recovery and visibility model.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The patch can make deleted transcript content searchable through the unfiltered CLI memory search path.

Concerns:

- **[medium] Deleted archives become searchable outside the guard:** `packages/memory-host-sdk/src/host/session-files.ts:74`
  - body: Indexing `.jsonl.deleted.*` archives adds their contents to the memory index after the session store key has been removed. Agent `memory_search` drops those hits because no key can be authorized, but CLI memory search renders manager results directly, so deleted transcript content can surface locally without the claimed visibility guard.
  - confidence: 0.84
