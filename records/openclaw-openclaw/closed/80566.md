## Summary

Keep open: this active MEMBER PR has the protected `maintainer` label and is opted into automerge, but the current head still misses carrier-wrapped non-POSIX shell wrappers because the new guards inspect the immediate executable instead of the resolved shell transport argv.

## What This Changes

The PR suppresses generated exec approval command spans for non-POSIX shell wrappers and Windows gateway approval commands, preserves explicit spans, and adds focused tests plus a changelog entry.

## Best Possible Solution

Use the existing shell-wrapper transport resolver for generated span suppression in both the formatter and host argv shortcut, then cover direct and carrier-wrapped `pwsh`/`cmd.exe` cases while preserving explicit spans.

## Reproduction Assessment

Yes, by source inspection. The branch checks `command.executable` or `argv[0]`, while current main already has a resolver that unwraps transparent carriers before identifying the effective shell wrapper.

## Solution Assessment

No, not yet. Suppressing unsupported generated highlights is the right fix direction, but the PR should base the decision on `resolveShellWrapperTransportArgv` rather than immediate executable tokens.

## Review Findings
