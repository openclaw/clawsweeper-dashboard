## Summary

Current main still supports only fixed Telegram DM thread policies and defaults omitted DM thread handling to `off`; the requested `auto` default based on `getMe.has_topics_enabled` remains meaningful, but it is a config/default behavior change that needs maintainer product review rather than an automatic strict bug fix.

## What This Changes

Request to add `channels.telegram.dm.threadReplies: "auto"` and make it the omitted default, using Telegram bot topic capability to choose flat versus topic-scoped DM sessions.

## Best Possible Solution

Keep this open for a maintainer decision on whether Telegram's bot-level topic capability should become the default DM session policy, then implement it with schema/types/docs/tests and upgrade proof for both topic-enabled and topic-disabled bots.

## Reproduction Assessment

No, not as a broken existing contract; current source and tests do reproduce the present behavior that omitted `threadReplies` defaults to flat DM sessions and only explicit config enables DM topic sessions.

## Solution Assessment

Unclear until maintainer product review; `auto` is a plausible user-friendly policy, but the default change affects upgrade/session-key behavior and needs explicit compatibility expectations.
