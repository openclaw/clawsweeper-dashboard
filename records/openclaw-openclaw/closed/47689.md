## Summary

Current main already routes auth-profile exhaustion into the model fallback path instead of leaking an undefined auth value, and it has regression coverage for all-profiles-in-cooldown plus cross-provider fallback.

## Evidence

- **Inner runner turns auth exhaustion into failover:** When all auth profiles are unavailable and fallbacks are configured, `throwAuthProfileFailover()` throws `FailoverError` instead of continuing with missing auth, which prevents the undefined-auth crash path and signals the outer fallback layer.

- **Embedded runner regression test for all profiles in cooldown:** The e2e test explicitly asserts that `runEmbeddedPiAgent` rejects with `FailoverError` and makes no provider attempt when all profiles are in cooldown and fallbacks are configured.
  - file: [src/agents/pi-embedded-runner.run-embedded-pi-agent.auth-profile-rotation.e2e.test.ts:1201](https://github.com/openclaw/openclaw/blob/ed6b487e2033d87a85b5713a964cc677c70353f8/src/agents/pi-embedded-runner.run-embedded-pi-agent.auth-profile-rotation.e2e.test.ts#L1201)
