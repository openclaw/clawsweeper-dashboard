## Summary

Current main already exposes hyphenated plugin commands as underscore Telegram commands and routes them back to the canonical plugin command name, so this PR's explicit `nativeNames: { telegram: "active_memory" }` change is redundant.

## Evidence

- **Active Memory still registers the canonical hyphenated command:** The plugin registers `name: "active-memory"` with no Telegram-specific override on current `main`, so any Telegram alias behavior has to come from shared command plumbing rather than this PR.
