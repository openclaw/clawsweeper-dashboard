  - body: This code is added to the old `src/memory` manager path, but current main owns the active memory runtime through `extensions/memory-core` and `packages/memory-host-sdk`. As written, the branch conflicts and would not exercise the current plugin-owned memory manager/CLI boundary.
  - confidence: 0.88
- **[P1] Reject blank shared paths before allowing reads:** `src/memory/manager.ts:729-730`
  - body: `sharedPaths` accepts blank strings, and this prefix check treats a blank path plus `path.sep` as `/` on POSIX. A config typo can therefore allow `memory_get` to read arbitrary absolute `.md` paths instead of only explicitly shared directories.
  - confidence: 0.86
- **[P2] Avoid mkdir on file sharedPaths:** `src/memory/manager.ts:248-249`
  - body: The constructor creates every configured shared path as a directory even though memory extra paths can be individual markdown files. A not-yet-created `policy.md` entry would become a directory and block indexing the intended file until manual cleanup.
  - confidence: 0.8
