## What This Changes

The PR records streamed model chunks as diagnostic run progress, throttles emitted progress events, skips disabled-diagnostics bookkeeping, and adds regression tests for streamed versus silent model-call recovery.

## Best Possible Solution

Merge once maintainers accept the chunk/heartbeat progress policy, and leave broader configurable thresholds or graceful-degradation work to the linked issue if that scope still matters.

## Reproduction Assessment

Yes, from source inspection and the linked report: current main tracks model-call start/end activity but streamed chunks do not refresh diagnostic run progress, while active-abort recovery keys on stale lastProgressAgeMs. I did not run the tests because this review was read-only.

## Solution Assessment

Yes, tracking observed stream chunks is narrower than blanket-exempting local providers and preserves recovery for silent or non-streaming calls. A broader timeout/config design would be a separate product decision, not required for this focused fix.
