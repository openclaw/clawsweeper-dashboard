## Best Possible Solution

Land or replace the linked formatter fix so send and poll dry-runs check `result.dryRun` before success formatting, with focused formatter coverage and the existing no-delivery guard preserved.

## Reproduction Assessment

Yes. Current main still separates `dryRun: true` from `handledBy: "core"`, while the formatter only checks `handledBy === "dry-run"`, so the failure path is source-reproducible with high confidence.

## Solution Assessment

Yes. The narrow maintainable fix is to use dry-run state before send and poll success formatting; the open linked PR takes that direction with focused formatter tests.
