## Summary

Close as cannot_reproduce. Current main sends normal Discord outbound replies through the regular channel message API, rewrites cached @username text into Discord mention syntax, and still omits allowed_mentions. Discord's current API docs say omitted allowed_mentions on regular messages parses users, roles, and everyone by default, while webhooks/interactions have different defaults. The related #46083 patch was closed unmerged after a maintainer/Codex review found the premise did not reproduce and that parse: ["users"] would narrow existing mention behavior and interact poorly with silent sends.

## Best Possible Solution

Close this issue as not reproducible for the current regular Discord channel-message send path. Do not add allowed_mentions to normal outbound sends unless a new live reproduction proves the current Discord default no longer applies; any future change should preserve role/everyone behavior intentionally and keep silent/preview semantics explicit.
