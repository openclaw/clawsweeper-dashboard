## What This Changes

The report asks OpenClaw to stop long streamed agent replies from truncating mid-output under large-context provider timeout conditions when direct OpenAI calls complete normally.

## Best Possible Solution

Land a shared provider stream-timeout fix with regression coverage so active SSE streams refresh liveness and true timeouts produce explicit errors instead of partial replies.

## Reproduction Assessment

Yes. The issue provides a concrete live reproduction path, and source inspection shows the deterministic mocked path: a guarded provider stream timeout after non-empty `assistantTexts` can bypass the explicit timeout-payload guard.

## Solution Assessment

Yes. The narrowest maintainable path is the shared provider/runner timeout handling targeted by #76633, not Discord, Telegram, or model-specific channel delivery code.
