## Summary

Keep this PR open. Current main still lacks the Telegram command-menu 429 retry/backoff path, so the PR is not obsolete. The PR is focused on the right surface, but its current head still parses only top-level `error_code` plus description text, while existing Telegram retry helpers already handle structured and wrapped `retry_after` shapes; the current head also has a failing node test check.

## Best Possible Solution

Keep this PR open. Ask for a rebase onto current main, implement command-menu 429 backoff using the same structured and wrapped `retry_after` extraction shapes already used by Telegram retry helpers, add regression tests for those shapes, keep the process-local command hash behavior explicit, and clear the failing node test check before maintainer review or landing.
