Keep open. Current main and v2026.5.12 still preserve the systemd duplicate-start retry branch and tests that reattempt startup after a failed health probe; the newer warmup-timing issue is useful cluster context, but it does not fully cover this CLI-triggered duplicate-start report.

## What This Changes

The issue reports that read-only CLI commands in v2026.4.29 can trigger a systemd-managed gateway duplicate-start retry path that restarts or takes over an already-running gateway.

## Best Possible Solution

Repair the systemd gateway lifecycle so duplicate supervised starts cannot retry into or kill a running or warming gateway, while preserving read-only CLI RPC/local snapshot behavior and coordinating with the open systemd warmup issue.
