## Summary

Current main already fixes the reported systemd entrypoint drift: gateway/service entrypoint resolution now prefers `dist/index.js` over legacy `dist/entry.js`, doctor repair reinstalls or stages mismatched service configs, and regression tests cover both update and doctor repair flows. The changelog records this as shipped in 2026.4.14.

## Evidence

- **Gateway install entrypoint now prefers index.js first:** `GATEWAY_DIST_ENTRYPOINT_BASENAMES` orders `index.js`, `index.mjs`, then legacy `entry.js`, `entry.mjs`, and `buildGatewayInstallEntrypointCandidates()` uses that order for service/update resolution.
