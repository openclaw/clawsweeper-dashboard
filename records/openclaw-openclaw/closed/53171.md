## Summary

Current main does not definitely implement this PR's intended change. The root package still excludes `scripts/`, `scripts/ui.js`, and `ui/` from the npm `files` allowlist while retaining `ui:build`; the current release path instead validates built `dist/control-ui` assets. The PR itself is also not mergeable as-is because the provided head diff makes `package.json` invalid JSON and broadens the published package surface to all scripts.

## Best Possible Solution

Do not auto-close as implemented. A maintainer should either request changes or close/replace this PR manually: first remove the invalid JSON change, then decide the supported product behavior. If `pnpm ui:build` is meant to work inside packaged installs, the fix needs a deliberate package-surface design including the required UI sources and regression coverage. If packaged installs should rely only on built `dist/control-ui` assets, the better fix is to remove or guard unsupported packaged `ui:*` scripts and keep release/package validation focused on the built asset inventory.
