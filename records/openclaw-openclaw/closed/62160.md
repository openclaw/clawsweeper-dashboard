The PR adds `api.registerMcpServer(...)` to the Plugin SDK, stores native plugin MCP registrations in the plugin registry, merges them into embedded MCP runtime configuration, surfaces MCP ownership in plugin status/manifest metadata, documents the new API, and adds focused runtime/loader/status tests plus a native MCP fixture.

Land a rebased, maintainer-reviewed implementation that keeps the SDK seam generic and additive, validates managed stdio-only server config, preserves user-config precedence and workspace scoping, documents the public contract, updates the changelog, and keeps the focused registry/runtime/status coverage.
