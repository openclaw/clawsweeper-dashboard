Full review comments:

- **[P2] Buffer Harmony text before promoting it:** `src/plugin-sdk/tool-payload.ts:260-264`
  - body: Once this fallback makes `commentary to=... code` blocks parseable at done time, the LM Studio wrapper still flushes streamed text whenever the buffer does not start with `[`. Harmony deltas start with `commentary` or `<|channel|>`, so the raw tool-call text can be emitted as `text_delta` before done-time promotion replaces the final message. Update the stream prefix predicate and add coverage that asserts no visible text event is emitted.
  - confidence: 0.88
- **[P2] Strip Harmony blocks from visible assistant text:** `src/plugin-sdk/tool-payload.ts:112-113`
  - body: The new parser recognizes Harmony standalone blocks, but `stripPlainTextToolCallBlocks` still short-circuits on a bracket-only regex, and assistant-visible cleanup calls that helper. A Harmony block that reaches fallback or persisted text will remain visible unless the detection/stripper is updated and covered.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `CHANGELOG.md:115`
  - body: This is a user-facing LM Studio fix for runs that can currently exit successfully with zero tool invocations, so repo policy requires a single-line entry in `CHANGELOG.md` under the active release section.
  - confidence: 0.93
