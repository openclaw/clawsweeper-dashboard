## Summary

The requested global default flip should close as superseded: current main still defaults `commands.nativeSkills` to `"auto"`, but the exact default-change PR was reviewed and closed because current Discord/Telegram overflow paths degrade instead of hard failing, and the remaining actionable problem is the open diagnostic-copy PR.

## What This Changes

The issue requests changing the default `commands.nativeSkills` behavior from automatic native command registration to disabled native skill commands to avoid Telegram/Discord command quota exhaustion.

## Best Possible Solution

Keep the current auto default and graceful overflow behavior, then land or replace the narrower diagnostic PR so quota warnings point users to `commands.nativeSkills` without silently breaking users who rely on native skill commands.

## Reproduction Assessment

No, not for a current-main fatal startup failure. Source inspection and existing tests show Telegram caps/retries and Discord drops per-skill commands, while the exact crash claim would need fresh live logs or a stack trace.

## Solution Assessment

No, changing the default is not the best current solution based on the closed default-flip PR review. The maintainable path is the narrower diagnostic fix tracked by https://github.com/openclaw/openclaw/pull/50191 plus a new report only if a current-main crash is proven.
