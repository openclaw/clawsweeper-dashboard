## Summary

Current main already supports custom assistant display names in Control UI/WebChat. Gateway resolves an assistant identity from `ui.assistant.name`, then `agents.list[].identity.name`, then workspace `IDENTITY.md`, exposes it through bootstrap and `agent.identity.get`, and the chat UI renders that value instead of a hardcoded assistant label.
