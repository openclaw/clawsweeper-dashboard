## Summary

Keep open for maintainer revision: the fix targets the right update data-loss path, but the proposed post-core handoff writes the resolved full config to a temp file, which can materialize env-substituted secrets on disk.

## What This Changes

The PR adds a post-core source-config handoff and fallback to restore dropped channel config during update repair, persists channel-only restorations, and adds update CLI regression tests plus a changelog entry.

## Best Possible Solution

Use a minimized, SecretRef/env-preserving channel-only handoff that restores dropped channels through fresh-process and legacy resume paths without serializing the full resolved config.

## Reproduction Assessment

Yes for source-level reproduction: current main reads the post-doctor config in post-core repair and has no pre-update channel handoff, while the linked regression gives concrete upgrade steps. I did not run a live package upgrade in this read-only review.

## Solution Assessment

No, not as written: the restoration approach is narrow, but the handoff should avoid serializing the resolved full config because `sourceConfig` can include env-substituted secret values.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Avoid writing the resolved full config to the handoff file:** `src/cli/update-cli/update-command.ts:2058`
  - body: `preUpdateSourceConfig` comes from `readConfigFileSnapshot().sourceConfig`, which is already after `$include` and `${ENV}` substitution. Writing the entire object to `source-config.json` can put provider or channel secrets that were only in environment variables onto disk in the temp handoff directory until cleanup succeeds, and a crash would leave them behind. Please hand off only the unresolved/minimized channel data needed for restoration, with owner-only permissions.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: The PR introduces a concrete secret-handling concern by serializing the resolved full config into a temp handoff file.

  - body: The handoff writes `sourceConfig`, which config IO defines as post-include and post-env-substitution, so env-sourced secrets can be materialized on disk outside the normal config/secret storage path.
