## Summary

Keep open for normal maintainer review: current main still uses raw UTF-16 slicing in cron failure alerts, while this PR applies the existing safe helper at the alert assembly point and backs it with focused coverage and proof.

## What This Changes

The PR replaces cron failure-alert error `.slice(0, 200)` with `truncateUtf16Safe(..., 200)` and adds a CronService regression test for a surrogate-pair boundary.

## Best Possible Solution

Land the narrow helper reuse and regression test after maintainer review and required checks remain green.

## Reproduction Assessment

Yes. Source inspection shows current main slices the trimmed cron alert error at 200 UTF-16 code units, and a 199-code-unit prefix plus a non-BMP character reaches the alert text path; I did not run local tests in this read-only review.

## Solution Assessment

Yes. The alert text is assembled in `emitFailureAlert` before webhook and channel delivery, and reusing the existing dependency-free helper there fixes the cron alert path without adding configuration or channel-specific truncation logic.
