## Summary

Close this PR as superseded by the later open Feishu thread-follow-up PR #48945. This PR’s central goal is still valid, and the author addressed earlier review comments, but the latest patch still has a concrete mention-gate bypass path for ordinary rooted quote replies and lacks the public config/docs follow-through expected for a new Feishu setting.

## What This Changes

The PR adds a Feishu global and per-group `requireMentionInThread` option, wires thread context into Feishu reply-policy resolution, and adds policy tests for thread mention overrides.

## Best Possible Solution

Use #48945, or an equivalent maintainer-approved topic-scoped follow-up policy, as the canonical implementation path and make it land with bot logic, schema, docs, metadata, and tests aligned.

## Reproduction Assessment

Yes. A normal Feishu group event with `root_id`, no bot mention, `requireMention: true`, and `requireMentionInThread: false` follows the PR predicate through `groupSession.replyInThread` and bypasses the mention gate even though current main treats normal rooted replies as distinct from topic sessions.

## Solution Assessment

No. The boolean override is too broad because it relies on ambiguous `replyInThread` semantics; the later `threadFollowUp` direction in #48945 is a narrower and safer product shape because it can limit bypasses to topic-scoped or bot-activated follow-ups.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep ordinary quote replies behind the mention gate:** `extensions/feishu/src/bot.ts:421-425`
  - body: The new predicate includes `groupSession?.replyInThread`, but current session resolution makes that value true for any inbound `root_id` or `thread_id`. With `requireMentionInThread: false`, a normal group quote reply without a bot mention can therefore bypass the group mention gate.
  - confidence: 0.93
- **[P2] Document the accepted Feishu config key:** `extensions/feishu/src/config-schema.ts:146`
  - body: Adding `requireMentionInThread` to the strict Feishu schema makes it accepted user config, but the PR does not update the Feishu channel docs or static plugin metadata. Users and config UIs would drift from runtime validation for this public setting.
  - confidence: 0.82
