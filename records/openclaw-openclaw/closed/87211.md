## Summary

Keep open: current main still contradicts the documented fast read-only `openclaw status` contract in source-inspectable ways: default text status still includes channel secret-target discovery, update-check work, and full session-row hydration before slicing recent rows.

## What This Changes

The issue requests that default `openclaw status` stay on a fast path by avoiding plugin-aware config work, package update checks, and full historical session metadata hydration unless deep/all status is requested.

## Best Possible Solution

Make plain `openclaw status` use a lean default path: no channel/plugin secret-target discovery, no network update check, and cheap session counting plus hydration only for rendered recent rows, while keeping full diagnostics under `--deep` and `--all`.

## Reproduction Assessment

Yes from source inspection, not from a live timing run: current main clearly performs the three reported work classes on the default status path. The exact 12-second latency depends on the reporter's real config and session history.

## Solution Assessment

Yes, the requested fast-path boundary is the right general fix, with one caveat: update availability should move to explicit/deep status without losing cheap local update channel or SHA context from the overview.
