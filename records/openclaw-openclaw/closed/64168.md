## Summary

Keep this issue open. Current main still loads recent daily memory into bare `/new` and `/reset` prompts by default, and session-memory can persist assistant reset text into those daily memory artifacts. The transcript marker added on main reduces transcript pollution, but it does not prevent a seeded self-improvement note from being loaded into the first prompt and echoed by the model.

## Best Possible Solution

Keep this issue open. Add a focused regression test for a Telegram or native bare `/new`/`/reset` where startup memory contains `[note self-improvement (before reset): ...]`, then prevent internal hook/self-improvement artifacts from being saved into daily memory or loaded into the startup prelude. Likely touch points are `src/hooks/bundled/session-memory/transcript.ts`, `src/hooks/bundled/session-memory/handler.ts`, and `src/auto-reply/reply/startup-context.ts`; output-time suppression alone would be a weaker fallback because the model would still see the internal artifact.
