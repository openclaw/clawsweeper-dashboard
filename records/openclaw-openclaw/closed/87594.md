The PR adds OpenRouter Mistral-family route detection, applies strict9 replay tool-call-id sanitization for those routes while preserving passthrough policy elsewhere, adds unit and live catalog coverage, and updates one core schema-cache sort call for lint compliance.

## Best Possible Solution

Land the provider-owned OpenRouter replay-policy override if maintainers accept the prefix set and the boundary-level proof, while keeping future route additions covered by the centralized helper and live catalog test.

## Reproduction Assessment

## Solution Assessment

Yes, this is the narrow maintainable fix if maintainers want the behavior owned by the OpenRouter plugin: it composes the existing passthrough policy and only adds strict9 for centralized Mistral-family route matches. The safer alternative would be a shared helper only if owners want this proxy-provider rule reused beyond OpenRouter now.
