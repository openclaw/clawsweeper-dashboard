## Summary

Current main now implements charset-aware byte decoding for web_fetch: response bytes are decoded with a charset selected from Content-Type or early BOM/XML/HTML metadata before extraction, and the runtime accepts GBK/GB2312 labels. The proof/fix commit is on main after v2026.4.26 and is not contained in any release tag yet.

## Best Possible Solution

Close as implemented on current main and keep the shared byte-first decoder. If users find a concrete GBK/GB2312 page with no header or meta charset that still garbles, track that separately as a narrower heuristic-detection or manual-override enhancement.
