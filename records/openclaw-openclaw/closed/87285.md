## Summary

Close as implemented: current main addresses the reported v2026.5.22 Gateway restart, event-loop prewarm, and inactive SecretRef failure pattern with hot/no-op reload planning, worker-thread provider-auth warming, and inactive SecretRef diagnostics.

## What This Changes

The issue asks OpenClaw to reduce unnecessary Gateway restarts, avoid provider-auth prewarm event-loop starvation, and make disabled-plugin SecretRef failures non-fatal and diagnosable.

## Best Possible Solution

Keep the shipped worker-thread auth warm and hot/no-op reload behavior, and handle any remaining startup-prewarm policy or channel-specific reload parity through the narrower open items already tracking those surfaces.

No current-main failing reproduction remains from this review. The v2026.5.22 logs were concrete, but current source and release history show the relevant reload, provider-auth, and inactive SecretRef paths have changed.

Yes. Current main solves the central issue at the owner boundaries: config reload planning handles hot/no-op paths, provider-auth warm runs off the Gateway event loop, and inactive SecretRefs are diagnostics rather than batch-failing disabled plugin config.
