## Summary

Current `main` already has the `openai-completions` usage handling this issue asks for. The transport always sends `stream_options.include_usage` on chat-completions requests and parses OpenAI-compatible `prompt_tokens` / `completion_tokens` usage into OpenClaw's internal counters. The shipped changelog for `v2026.4.22` also explicitly calls out fixing 0% usage for local/custom OpenAI-compatible backends.
