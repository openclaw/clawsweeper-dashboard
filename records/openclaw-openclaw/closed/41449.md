## Summary

Close as implemented on current main. The original #41305 user problem, Ollama memory embeddings failing for localhost/private-network endpoints, is covered by the current embedding provider plus the shared SSRF policy: the configured Ollama host is added to `allowedHostnames`, and the SSRF resolver treats exact allowed hostnames as an opt-in that skips private-network blocking for that host. The PR's redirect-pinning idea is useful hardening, but it is not needed to fix the reported localhost/private-network failure and this PR is now an XL, stale branch with broad unrelated churn.

## Best Possible Solution

Close this stale PR as implemented for the reported private-network embedding failure. Keep the current exact-host SSRF policy behavior on main; if maintainers want the redirect-pinning hardening from the review discussion, it should be a new narrow PR against `extensions/ollama/src/embedding-provider.ts` that reuses or aligns with `buildOllamaBaseUrlSsrFPolicy` and adds focused redirect tests, without carrying this XL branch's unrelated churn.
