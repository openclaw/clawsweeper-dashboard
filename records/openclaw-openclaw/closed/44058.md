## Summary

Current main implements per-agent TTS overrides, but it does not implement the requested bot/channel/account-level TTS contract. Feishu account config remains strict without a `tts` field, TTS resolution only merges `messages.tts` with active `agents.list[].tts`, and outbound TTS call paths pass `agentId` rather than channel account context. The recent maintainer comment explicitly kept #44058 open while pointing to #63338 as the closest, still-open account-level implementation/design path.

## Best Possible Solution

Keep #44058 open as the user-facing tracker until a maintainer-approved channel/account-level design lands. The best path is to refine #63338 or a successor so Feishu/account defaults are owned by the channel/plugin layer or a generic extension hook, while preserving the shipped `agents.list[].tts` contract for voice identity and avoiding a Feishu hardcode in the generic speech-core TTS facade.
