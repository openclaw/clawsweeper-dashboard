## Summary

Keep open. Current main still has the reported silent scope drop: the parent `models --agent <id>` option is accepted before `set`, but the `set` and `set-image` actions do not resolve or reject it and still call the global default update path. The prior targeted fix PR is closed and unmerged, so the remaining work is a narrow CLI repair.

## What This Changes

The issue requests that `openclaw models --agent <id> set <model>` either update the named agent’s model override or fail clearly instead of silently changing the global default model.

## Best Possible Solution

Stop the silent global side effect on the current CLI surface: reject `--agent` for `models set` and `models set-image` with a clear error unless maintainers intentionally design per-agent model mutation for those commands.

## Reproduction Assessment

Yes. The source path is enough to reproduce with high confidence: parent `--agent` is accepted before `set`, the `set` action ignores it, and the command writes through the global `agents.defaults` updater.

## Solution Assessment

Yes for a focused repair. Rejecting unsupported `--agent` on these write commands is the narrowest maintainable fix; implementing per-agent writes should be a separate deliberate change if maintainers want that behavior.
