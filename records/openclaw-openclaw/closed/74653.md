## Summary

Keep open. Current main still has the false-positive path: the gateway can return a skipped embedding probe with `checked: false`, but the CLI adapter maps successful payloads to `checked: true` and the key-optional provider renderer still emits the warning. The latest PR patch targets that gap without a concrete blocking review finding, so the remaining action is maintainer review and final validation rather than closure or replacement.

## What This Changes

The PR propagates skipped doctor memory probe state through the CLI adapter, suppresses key-optional provider warnings only for intentional skips, adds regression coverage, and adds a changelog entry.

## Best Possible Solution

Merge the skipped-probe discriminator and renderer guard after final CI/format/test validation, preserving timeout and not-ready diagnostics while allowing linked bug #74608 to close on merge and keeping unrelated FTS5 doctor work such as #62338 separate.
