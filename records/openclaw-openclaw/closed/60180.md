## Summary

Current `main` already contains a Telegram-specific shutdown fix: Telegram polling fetches are aborted on shutdown, Telegram API calls are bounded, and gateway channel shutdown no longer blocks indefinitely on a stuck account task. The issue’s proposed `stopAccount()` root cause no longer matches the current Telegram plugin, which has no `gateway.stopAccount` hook.

## Evidence

- **Telegram plugin uses abort-driven shutdown, not stopAccount teardown:** The current Telegram channel plugin exposes `gateway.startAccount` and `logoutAccount`, but no `gateway.stopAccount`. Startup passes the gateway abort signal into the Telegram monitor, so shutdown is driven through abort propagation rather than a plugin-local stop hook.
  - file: [extensions/telegram/src/channel.ts:865](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/telegram/src/channel.ts#L865)
  - command: `nl -ba extensions/telegram/src/channel.ts | sed -n '860,970p'`
- **Telegram fetches are aborted on shutdown and bounded by per-method timeouts:** `createTelegramBotCore()` wraps Telegram API fetches so the gateway shutdown signal aborts in-flight requests immediately, and it applies explicit request timeouts (`getupdates` 45s; `deletewebhook`/`getme`/`setwebhook` 15s).
  - file: [extensions/telegram/src/bot-core.ts:181](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/telegram/src/bot-core.ts#L181)
  - command: `nl -ba extensions/telegram/src/bot-core.ts | sed -n '170,240p'`
- **Polling session explicitly aborts active fetches and cleans up shutdown listeners:** `TelegramPollingSession` aborts the active fetch when shutdown arrives, stops the runner/bot through bounded graceful-stop helpers, and removes the per-cycle abort listeners in `finally`, which is the Telegram-side shutdown handling this report was asking for.
  - file: [extensions/telegram/src/polling-session.ts:274](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/telegram/src/polling-session.ts#L274)

- **Gateway channel shutdown is itself bounded:** Gateway channel stop waits only 5 seconds after abort before continuing shutdown, rather than blocking indefinitely on a stuck channel task.
  - file: [src/gateway/server-channels.ts:29](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/gateway/server-channels.ts#L29)
  - command: `nl -ba src/gateway/server-channels.ts | sed -n '20,110p' && nl -ba src/gateway/server-channels.ts | sed -n '545,590p'`
