## Summary

Current `main` has moved web-search provider support behind the plugin registry/runtime, and Serper is not present as a bundled provider. This PR’s hardcoded core-provider approach is stale against that architecture; the right path is a plugin/ClawHub integration, not more core wiring.

## Evidence

- **Vision scope:** OpenClaw’s current scope says optional capability should usually ship as plugins, with a high bar for adding optional plugins to core.
  - file: [VISION.md:55](https://github.com/openclaw/openclaw/blob/b98a6a46fb54145a33e753101c02f0d7d7e0bc3b/VISION.md#L55)
