## Summary

Current main and the shipped v2026.4.26 tag implement the requested behavior: plugin-entry-only config invalidity is treated as degraded Gateway startup/reload state instead of a hard config crash, mixed core config invalidity remains fatal, plugin loader failures are isolated to the affected plugin, and legacy Brave web-search config is migrated to plugin-owned config paths.

## Best Possible Solution

Close as implemented. Keep the current plugin-local degraded startup/reload behavior, loader-level plugin failure isolation, Brave legacy web-search migration, and doctor quarantine path; any remaining post-startup plugin runtime crash should be filed as a narrower plugin/runtime bug with logs and the affected config path.
