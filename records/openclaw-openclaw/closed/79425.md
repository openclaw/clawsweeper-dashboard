## Summary

Current main and shipped releases already handle the reported fenced `HEARTBEAT.md` scaffold as empty, so the 2026.4.7 regression is obsolete after upgrading.

## What This Changes

The issue asks OpenClaw to avoid running heartbeat API calls when `HEARTBEAT.md` contains only comments, empty content, or the default doc-only Markdown fence scaffold.

## Best Possible Solution

Keep the shipped fence-aware empty-file gate; users on 2026.4.7 should upgrade to v2026.4.10 or later.

## Reproduction Assessment

No failing current-main reproduction remains. The exact fenced-template shape is covered by source and regression tests that assert `empty-heartbeat-file` skips with no reply or send call.

## Solution Assessment

Yes. The current fix is narrow: it ignores only doc-only fence marker lines in the emptiness predicate and keeps fenced real tasks or prose actionable.
