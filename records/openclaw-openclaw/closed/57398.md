## Summary

Current `main` no longer does the eager bundled-channel runtime loading described in #57398, and the plugin facade loader now has sentinel-based circular re-entry guards. The changelog shows a shipped startup/facade circular-load fix in `v2026.4.5`, so the reported regression is implemented on main and released.

## Evidence

- **Bundled channel discovery is metadata-only on current main:** `listBundledChannelMetadata()` now reads bundled plugin metadata and filters by manifest channel presence; it does not import every bundled channel runtime during discovery.
  - file: [src/channels/plugins/bundled.ts:324](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/channels/plugins/bundled.ts#L324)
  - command: `nl -ba src/channels/plugins/bundled.ts | sed -n '324,337p'`
- **Bundled channel runtime loads are lazy per requested id:** `getLazyGeneratedBundledChannelEntryForRoot()` resolves metadata for one channel id and only then loads that channel entry, instead of iterating all discovered bundled channels.
  - file: [src/channels/plugins/bundled.ts:384](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/channels/plugins/bundled.ts#L384)
  - command: `nl -ba src/channels/plugins/bundled.ts | sed -n '384,417p'`
- **Bootstrap registry requests one plugin at a time:** `getBootstrapChannelPlugin()` calls `getBundledChannelPlugin(resolvedId)` / `getBundledChannelSetupPlugin(resolvedId)` for the requested id only, after ids are listed separately via metadata/catalog.
  - file: [src/channels/plugins/bootstrap-registry.ts:117](https://github.com/openclaw/openclaw/blob/5dfc1b90e176a9e4d13b9e91ad6affec6b92f213/src/channels/plugins/bootstrap-registry.ts#L117)
