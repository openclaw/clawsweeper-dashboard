## Summary

Current main already supports Perplexity-backed agent models through the generic custom-provider flow for OpenAI-compatible endpoints, so the underlying request is implemented even though the bundled `perplexity` plugin remains web-search-only.

## Evidence

- **Perplexity plugin is still web-search-only:** The bundled Perplexity surface is explicitly documented as a web search provider, not a model provider, and its plugin contract only registers `webSearchProviders`.
  - file: [docs/providers/perplexity-provider.md:21](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/docs/providers/perplexity-provider.md#L21)
- **Bundled Perplexity contract has no model-provider registration:** `extensions/perplexity/openclaw.plugin.json` exposes Perplexity only under `contracts.webSearchProviders`, which matches the docs and explains why there is no dedicated bundled model provider.
  - file: [extensions/perplexity/openclaw.plugin.json:22](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/extensions/perplexity/openclaw.plugin.json#L22)
- **Current onboarding supports OpenAI-compatible custom providers:** The onboarding docs show `openclaw onboard --auth-choice custom-api-key --custom-compatibility openai` and later state that Custom Provider can connect any OpenAI- or Anthropic-compatible endpoint, including hosted providers not listed.
  - file: [docs/cli/onboard.md:34](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/docs/cli/onboard.md#L34)
- **Current config docs support explicit custom model catalogs:** `models.providers` is documented for custom/base-URL providers with `api: "openai-completions"`, custom `baseUrl`, and explicit `models[]` entries, which is the supported way to register unlisted compatible models such as Perplexity Sonar and avoid `Unknown model` catalog failures.
  - file: [docs/gateway/config-tools.md:356](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/docs/gateway/config-tools.md#L356)
- **Runtime tests cover non-catalog allowlisted model refs:** Model-selection coverage explicitly accepts allowlisted refs that are absent from the bundled catalog, which is the mechanism needed for compatible backends whose model ids are not in OpenClaw's built-in catalog.
  - file: [src/agents/model-selection.test.ts:750](https://github.com/openclaw/openclaw/blob/3a64aa49a9014518fa1977fbc023f3e0b978ae5d/src/agents/model-selection.test.ts#L750)
- **This was already shipped in the latest release:** The `v2026.4.23` tag contains the custom-provider onboarding docs and the Perplexity web-search-only docs, so the supported path was already present in the latest release provided in the review context.
  - command: `git show v2026.4.23:docs/cli/onboard.md | sed -n '20,70p'`
  - sha: [9b18ae877018](https://github.com/openclaw/openclaw/commit/9b18ae877018a34933e27bb7b2986cb97e9602aa)

## Risks / Open Questions

- There is still no dedicated bundled `perplexity` model provider; this closure relies on the generic custom-provider path being sufficient for the reported user need.
- The reporter's exact old setup using the built-in `openai` provider plus only a `baseURL` override is not the documented path on current main; the supported path is `models.providers` / Custom Provider onboarding.
