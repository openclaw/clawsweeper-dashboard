## Summary

Current main already routes `web_search` through the trusted web-tools fetch path that explicitly enables env-proxy mode, and both Brave and SearXNG use that shared helper. Tests cover the trusted env-proxy path, so the older `web_fetch works but web_search fails with fetch failed` behavior from 0.1.7 is obsolete on current main. The same implementation is present in release `v2026.4.22`.
