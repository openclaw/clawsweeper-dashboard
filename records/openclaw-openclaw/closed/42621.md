## Summary

Current main still does not implement targetable existing-session delivery for `/hooks/agent`: agent hooks are still dispatched as isolated cron agent turns, and the hook config/schema has no `sessionTarget` or `targetExistingSession` field. The same remaining webhook-agent session reuse problem is already tracked by open #11665, while #70268 is only a related wake-hook workaround and explicitly does not solve this broader agent-hook request.

## Best Possible Solution

Track the remaining `/hooks/agent` session reuse/sessionTarget design under #11665, likely by adding a policy-gated session target or persistent mode through hook mapping/request parsing and reusing the existing cron session plumbing. Keep #70268 scoped to the wake-hook workaround.
