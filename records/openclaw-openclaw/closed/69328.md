## Summary

Keep open: this PR still contains useful ACP and Control UI work that is not on current main, but the patch should not merge as written because the new verifier can misclassify real worktree changes as zero-diff and the external PR lacks after-fix real behavior proof.

## What This Changes

The PR adds an ACP worktree diff verifier plus Control UI live `session.message` append/dedupe behavior and regression tests.

## Best Possible Solution

Revise the PR so the verifier counts tracked, staged, unstaged, untracked, and unborn-HEAD worktrees, then provide redacted real behavior proof for the ACP and Control UI paths.

## Reproduction Assessment

Yes, at source level. The PR helper calls `git diff --stat HEAD` and maps empty output or git errors to `hasChanges: false`, then ACP terminal handling turns that result into blocked or failed runs.

## Solution Assessment

No. The direction is reasonable, but the verifier should use a worktree-state check that includes untracked files and handles unborn HEAD before this merge path is safe.
