## Summary

Close #55590 as superseded by the more precise open tracker #63990, with #43760 covering the narrower fallback-provider/model/remote config gap. Current main already supports per-agent memorySearch overrides and a single provider fallback, but it does not provide safe multi-model/multi-index embedding failover; that remaining product/code work is now tracked in #63990 to avoid split discussion.

## Best Possible Solution

Close #55590 as duplicate/superseded. Keep #63990 as the canonical tracker for safe multi-model memory embeddings via per-model indexes, active/fallback index routing, dimension validation, and observability. Keep #43760 as the narrower tracker for independent fallback provider/model/remote config. Future implementation should be backward-compatible with the current single provider/model/fallback config.
