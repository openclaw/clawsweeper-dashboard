## Summary

Close as duplicate of the older open canonical issue #43295. The current issue adds valuable incident detail, but it reports the same remaining product gap: prompt/memory/context rules are known to the agent yet are not enforced as a runtime rule-checking or approval gate before actions. Current main has optional Active Memory recall and prompt/context injection, but not the requested higher-priority binding rule layer or broad confirmation gate.

## Best Possible Solution

Close this issue as a duplicate and continue the product/runtime design in #43295. The best fix belongs in the canonical thread: a runtime-level rule/approval mechanism that can enforce high-priority workspace rules or confirmation gates without relying only on prompt text, memory recall, or model self-discipline.
