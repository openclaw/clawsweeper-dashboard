## Summary

Current `main` already fixes this cron startup crash through a broader load-time normalization path: persisted jobs are normalized before startup inspects `job.state.runningAtMs`, so this PR is superseded.

## Evidence

On current `main`, safety depends on the loader path rather than an inline guard here.
