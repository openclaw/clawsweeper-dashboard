Current main already has the narrower dispatcher delivery-error logging, while the remaining user-visible problem is the same unresolved gap tracked by #49225: model/session text can exist without a clear WhatsApp provider delivery state or operator-facing explanation.

## What This Changes

The issue asks for generated WhatsApp group auto-replies to be delivered back to the group instead of appearing only in the transcript and Control UI.

## Best Possible Solution

Consolidate the remaining work under #49225: make WhatsApp delivery state explicit across generated text, provider dispatch invocation, provider acceptance/error, and intentional group visible-reply suppression. Keep the current documented group default, and point operators who want legacy group auto-posting to messages.groupChat.visibleReplies: automatic or the message tool.
