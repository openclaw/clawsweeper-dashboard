## What This Changes

The PR makes QMD vector status probes use `memory.qmd.limits.timeoutMs`, adds checkpoint transcript export regression coverage, and includes SMS status plus timer/test stabilization changes.

## Best Possible Solution

Either explicitly accept `memory.qmd.limits.timeoutMs` as the QMD status-probe budget, or add a dedicated status-probe timeout/default while keeping the existing search-timeout semantics clear.

## Reproduction Assessment

Yes, from source inspection rather than a local run: current main hard-caps the QMD status probe at 5000 ms, so configs above that value can still time out early. The PR adds focused fake-timer regression coverage for the configured-timeout path.

## Solution Assessment

Unclear until maintainers decide the contract. Using the existing timeout is a narrow code change, but a dedicated status-probe timeout would be safer if status checks are expected to stay independently bounded.
