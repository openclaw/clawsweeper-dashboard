## Summary

This is a user-specific OpenAI API key/auth-profile setup problem rather than an actionable OpenClaw source bug. Current main documents and implements the expected behavior: direct OpenAI uses `OPENAI_API_KEY` or OpenAI API-key onboarding, stored per-agent auth profiles are the primary credential source, and `openclaw models status` / `openclaw models status --probe` expose the active source and live auth status.

## Best Possible Solution

Close this support-style issue. Keep the current profile-first credential contract, direct users to `openclaw models status`, `openclaw models status --probe`, OpenAI onboarding, and the per-agent auth-profile docs; ask for a new focused bug only if current main/current release still returns 401 with redacted logs and credential-source output.
