## Summary

Keep open. Current main still has the sandbox bridge mutation helper creating 0600 replacement files and atomically replacing existing workspace files without preserving the target mode, so the reported sandbox permission regression remains valid.

## What This Changes

The issue requests that sandboxed file-tool edits preserve sane workspace file permissions instead of rewriting existing files from 0644 to 0600.

## Best Possible Solution

Add a focused sandbox mutation-helper fix that preserves the existing target mode during atomic replacement, with regression coverage for sandbox Write/Edit and sandbox apply_patch, while keeping sensitive internal write defaults restrictive.

## Solution Assessment
