## Summary

Keep open. Current `main` still appears to leak queued system events across `sessions.reset`/`/new`: the live reset path moved into `src/gateway/session-reset-service.ts`, and that cleanup still clears reply queues without draining `system-events`. The PR's intent still matches a real gap, but its patch targets pre-refactor files and likely needs rebasing onto the reset-service path.

## Evidence
