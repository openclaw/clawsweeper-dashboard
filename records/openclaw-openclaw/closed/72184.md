## Summary

Current main already fixes the reported 2026.4.22 path: config.patch/config.apply restart sentinels preserve routing metadata but do not create a generic agent continuation, and delivery recovery replays the stored outbound payloads rather than reconstructing text from session context. The same behavior is present in v2026.4.24.

## Best Possible Solution

Keep the shipped v2026.4.24/main behavior: config.patch/config.apply restarts should write routing-only restart sentinels, while outbound delivery recovery should continue replaying persisted payloads. If a current-version report appears, investigate the producer that enqueued unsafe payload text and add a targeted quarantine/regression test there.
