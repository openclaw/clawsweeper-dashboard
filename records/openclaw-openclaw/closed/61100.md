## Summary

Keep this PR open. Current main still does not implement the PR's intended Discord `upload-file` action: Discord can send media through the existing `send` action, but the message tool still does not advertise or dispatch a dedicated `upload-file` action, and the current contract coverage has not been updated for that behavior.

## Best Possible Solution

Review or port this PR rather than close it. The best implementation is to add Discord `upload-file` discovery when `messages` is enabled, map the tool action through `handle-action.ts` and the Discord messaging runtime, and add current-main tests in `extensions/discord/src/channel-actions.contract.test.ts` plus focused dispatch/runtime coverage. If landed, it should resolve the linked Discord discoverability bug.
