## Summary

Keep open. Current main and the latest release still have the polluted-origin gap: origin-mismatched auto fallback pins are not eligible for the primary snap-back probe and are not cleared by the ordinary reply or direct-agent selection paths, while https://github.com/openclaw/openclaw/pull/92819 is the open linked fix candidate.

## What This Changes

The issue reports that auto fallback session model pins can persist indefinitely when fallback-origin metadata no longer matches the configured primary model.

## Best Possible Solution

Keep this issue open as the canonical bug until https://github.com/openclaw/openclaw/pull/92819 or an equivalent fix lands with reply and direct-agent stale-origin cleanup plus focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an auto fallback session entry with origin fields that differ from the current primary is ignored by the primary-probe guard while ordinary reply/direct selection can still honor the stored fallback override.

## Solution Assessment

Yes. The best fix is to finish the existing shared stale-origin cleanup in the reply and direct-agent selection boundaries, preserving user-owned overrides and avoiding a broader fallback-policy feature change.
