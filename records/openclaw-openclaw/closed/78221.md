## Summary

Current main now covers the Telegram `commands.native: false` case: plugin command handlers are registered and executed locally even when they are not published to Telegram's native menu, so the v2026.5.4 report is obsolete on main.

## What This Changes

The issue reports that Telegram text-routed plugin slash commands no longer stop agent execution when native Telegram commands are disabled.

## Best Possible Solution

Keep the current main behavior and ship it in the next release so v2026.5.4 users can upgrade past the Telegram native-disabled plugin command regression.

## Reproduction Assessment

No. I did not find a current-main failure path: current main registers Telegram plugin command handlers even when native menu registration is disabled, and the regression test covers that exact condition.

## Solution Assessment

Yes. The narrow maintainable solution is the current Telegram-owned handler registration plus the generic plugin command handler that only continues to the agent when continueAgent is explicitly true.
