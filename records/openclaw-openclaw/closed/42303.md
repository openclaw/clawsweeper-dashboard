## Summary

Keep open for human handling: current main still has the Docker missing-brew path, but this PR is not mergeable as-is because it conflicts with current Docker setup, edits a removed root setup script, adds installer quoting/security issues, and lacks inspectable real Docker proof.

## What This Changes

This PR adds optional `OPENCLAW_INSTALL_BREW` Dockerfile support, wires that env var plus `OPENCLAW_SKIP_BUILD` through root `docker-setup.sh`, and attempts to restore Linuxbrew availability for Docker skill installs.

## Best Possible Solution

Choose one documented Docker skill-dependency strategy for https://github.com/openclaw/openclaw/issues/14593; if Linuxbrew-in-Docker is selected, update `Dockerfile` and `scripts/docker/setup.sh`, harden installer execution, scope brew env to brew-enabled images, update docs, and require real Docker proof.

## Reproduction Assessment

Yes, from source inspection. Current Docker has no Linuxbrew build option, onboarding can offer a brew-only missing skill such as `openai-whisper`, and `installSkill` returns the brew-missing failure when no brew executable resolves; I did not run a live Docker build.

## Solution Assessment

No. Optional Linuxbrew-in-Docker may be a valid strategy, but this patch is not the best mergeable solution because it targets an obsolete setup script, has Dockerfile execution/env issues, and needs maintainer package-manager/security direction.

  - body: Current main and docs use `scripts/docker/setup.sh`, and root `docker-setup.sh` is absent. This branch only persists and passes `OPENCLAW_INSTALL_BREW` in the old root script, so the documented setup flow would still build without the brew arg after conflict resolution.

  - body: This expands the installer with `$(curl ...)` and passes the full script as a single-quoted `bash -c` string.   - confidence: 0.92
- **[P2] Point skip-build recovery at a real command:** `docker-setup.sh:418`
  - body: The skip-build missing-image path tells users to run `openclaw build`, but current CLI registration does not expose a top-level `build` command. Point users to rerun setup without `OPENCLAW_SKIP_BUILD` or to an explicit Docker build command.
  - confidence: 0.88
- **[P3] Scope Homebrew env to brew-enabled builds:** `Dockerfile:222-225`
  - body: These `ENV` instructions run for every image, so default builds get `HOMEBREW_*` values and PATH entries for directories that do not exist. Keep the runtime environment scoped to images built with brew enabled.
  - confidence: 0.86
