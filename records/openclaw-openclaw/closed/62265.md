## Summary

Current `main` already implements this gateway-crash fix with a stronger approach than the open PR: it proactively disables exec `onUpdate` emission when the exec run settles or the agent aborts, and it has regression coverage for backgrounded, exited, and aborted exec sessions. The changelog records the shipped-on-main fix as `#62821`, so this PR is now redundant.

## Evidence

- **Main code now suppresses late exec updates before stale listeners can be reached:** `runExecProcess` tracks `updatesDisabled` and makes `emitUpdate()` a no-op once the exec session is backgrounded, exited, or updates have been disabled. The inline comment explicitly ties this to the `"Agent listener invoked outside active run"` crash and explains why a local try/catch would not be effective because the failure is an async rejected Promise from `pi-agent-core`.
