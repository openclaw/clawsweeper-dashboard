## Summary

Current main already handles explicit silent-token replies without retrying or surfacing an incomplete-turn failure; this PR changes the shared predicate in the opposite direction and would restore the reported heartbeat fallback cascade.

## What This Changes

The PR changes embedded-agent turn classification so assistant text containing only the silent reply token is treated as empty for silent-response handling.

## Best Possible Solution

Keep current main’s explicit silent-token exclusion; if stronger proof is desired, add only a focused internal-heartbeat regression test without changing the runtime predicate.

## Reproduction Assessment

Yes. Source tracing shows current main completes NO_REPLY through its explicit silent-token guard, whereas the proposed predicate enters empty-response retry for an internal heartbeat where empty-assistant silence is not enabled.

## Solution Assessment

No. Current main already solves the observable failure with the narrower distinction between an explicit silent token and a genuinely empty reply; this PR removes that distinction in the retry predicate.
