## Summary

Keep open: current main still lacks a steering-specific plugin hook, and the active-run steering paths accept inbound and explicit `/steer` text without a per-steering block/rewrite gate. This is a security-sensitive public plugin API addition, so it needs maintainer/API design rather than cleanup closure or an automatic fix lane.

## What This Changes

The issue requests a new `before_steering` plugin hook that can inspect, rewrite, or block steering messages before they are injected into active agent turns.

## Best Possible Solution

Design and document a steering input-gate seam that covers normal inbound steering and explicit `/steer`, with clear failure policy, metadata contract, Pi/Codex behavior, and focused regression coverage.

## Reproduction Assessment

Yes, by source inspection: current main defaults inbound active-run behavior to `steer`, and both inbound active-run steering and explicit `/steer` pass text into the runtime queue without a steering-specific block/rewrite hook. I did not run a live exploit because the issue is a security-sensitive public API gap rather than a narrow failing command path.

## Solution Assessment

Unclear. A dedicated `before_steering` hook is plausible, but maintainers should decide the public API shape, failure policy, metadata exposure, and whether the gate belongs before enqueue or at a shared runtime steering boundary.
