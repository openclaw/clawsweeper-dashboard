## Summary

This PR is superseded by merged maintainer PR #72393, which fixed the same Windows exec/CJK output problem on current main with raw-buffer decoding and console-codepage-aware fallback instead of prepending `chcp 65001`. Current source, tests, and changelog all show the replacement implementation is present and credits this PR's contribution.

## Best Possible Solution

Close this PR as superseded by merged #72393 and keep the current raw-buffer, console-codepage-aware implementation on main. Any remaining Windows exec encoding regression should be filed against current main with a fresh reproduction rather than merging this `chcp 65001` branch.
