## Summary

Keep open. Current main still has a source-level path where pre-tool assistant text makes the terminal result look payload-bearing, so a missing post-tool final assistant message can bypass the incomplete-turn recovery path.

## What This Changes

The issue reports that WebChat tool-chain runs can drop the final assistant text after the last tool result, leaving both UI history and next-turn context without the conclusion.

## Best Possible Solution

Add a regression-backed embedded-runner guard that distinguishes pre-tool progress text from post-tool final assistant output and surfaces a retry or explicit incomplete-turn state when the final post-tool message is missing.

## Reproduction Assessment

Yes at source level, but not as a live WebChat run. A mocked attempt with pre-tool `assistantTexts`, tool activity, and no post-tool assistant message reaches `payloadCount > 0` and skips the incomplete-turn warning path, matching the reported transcript gap.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the embedded runner terminal/transcript path, with WebChat covered as a consumer that should render whatever the transcript and lifecycle events persist.
