## Summary

Current main contradicts the issue premise: announce-mode isolated cron jobs still have direct chat-send capability, and announce is only the runner fallback if the agent did not already send. The reported "no Slack API access in announce mode" behavior is obsolete on current main.

## Evidence

- **Cron docs define announce as fallback delivery:** The scheduled-tasks docs say `announce` fallback-delivers final text only if the agent did not send, and explicitly state isolated jobs can still use the `message` tool even with `--no-deliver`.
