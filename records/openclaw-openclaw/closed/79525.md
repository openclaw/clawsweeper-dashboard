## Summary

Current main and the shipped v2026.5.7 Gateway path already turn linked agent lifecycle errors into terminal chat error events, clear the active run state, and leave Stop or `/new` recovery unblocked.

## What This Changes

The issue asks for context-overflowed chat runs to surface an error, stop processing, and allow Stop or `/new` recovery instead of leaving the Control UI stuck running.

## Best Possible Solution

Keep the v2026.5.7/current-main terminal error handling and ask users on v2026.4.15 to upgrade, with a new report only if a current build still leaves chat stuck.

## Reproduction Assessment

No live Kilo overflow was run, but the source path is clear: the reported embedded lifecycle error maps to current Gateway lifecycle-error handling, and regression coverage simulates that terminal error for an active chat send.

## Solution Assessment

Yes. The current solution is the narrow maintainable one: Gateway emits a terminal chat error for linked lifecycle failures, the UI clears active run state on that terminal event, and Stop or `/new` remain local recovery controls.
