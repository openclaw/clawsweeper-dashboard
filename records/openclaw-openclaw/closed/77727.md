## Summary

Keep open: the shipped #76529 fix covers short final edits after visible non-preview output, but current main still has a plausible source-level gap for long/chunked finals that can reuse the old preview message and place the first final chunk above later progress output.

## What This Changes

Telegram should keep the final assistant reply after all tool/progress messages during streaming turns.

## Best Possible Solution

Keep the issue open and make Telegram finalization send any final answer, including long/chunked finals, after visible non-preview progress output, with focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes, by source inspection: create a Telegram streaming turn with an active preview, deliver a visible block/tool progress payload, then deliver a final answer longer than the draft limit. Current main can edit the old preview to the first final chunk instead of sending the whole final after the progress bubble.

## Solution Assessment

Yes, the narrow fix is to extend the existing fresh-final guard into the long/chunked final path rather than adding a new mode or config option.
