## What This Changes

The issue reports that subscription-limit cooldowns can remain blocked until a far-future upstream reset timestamp when no model fallbacks are configured, preventing recovery probes for the primary model.

## Best Possible Solution

Add a narrow recovery-probe path for subscription-limit/rate-limit primary cooldowns when no fallback candidates exist, with regression coverage that proves a throttled primary probe happens without adding new config.
