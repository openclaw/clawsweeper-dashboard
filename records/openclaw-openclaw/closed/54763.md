## Summary

Issue #54763 is already fixed on current `main` and the fix was shipped by `v2026.4.22`. Current main no longer contributes a required Discord `components` field to the shared message tool schema, and the Discord send runtime treats empty `components` objects as non-component sends.

## Evidence

- **Current shared message-tool schema does not expose Discord `components`:** The current schema-scoping test asserts `properties.components` is `undefined` for Discord, so plain `message(action="send", channel="discord", ... media=...)` sends cannot fail because `components` is required.
  - file: [src/agents/tools/message-tool.test.ts:447](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/src/agents/tools/message-tool.test.ts#L447)
  - command: `nl -ba src/agents/tools/message-tool.test.ts | sed -n '374,466p'`
  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Discord runtime only requires content when no component payload is present:** `handleDiscordMessagingAction` computes `componentSpec` only for non-empty component objects and then sets `content` required only when `!asVoice && !componentSpec && !components`, which does not make `components` globally required for media sends.
  - file: [extensions/discord/src/actions/runtime.messaging.ts:344](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/extensions/discord/src/actions/runtime.messaging.ts#L344)

  - sha: [a6aa151653ab](https://github.com/openclaw/openclaw/commit/a6aa151653ab478e9aa23e95c6446c41adc4fce1)
- **Regression test covers empty `components` on regular media sends:** The Discord action test sends `mediaUrl` with `components: {}` and verifies the normal send path is used and the component send path is not called.
  - file: [extensions/discord/src/actions/runtime.test.ts:486](https://github.com/openclaw/openclaw/blob/a6aa151653ab478e9aa23e95c6446c41adc4fce1/extensions/discord/src/actions/runtime.test.ts#L486)

- **Linked fix PR explicitly addressed this bug:** PR #54800 patch states the root cause was the Discord-contributed `components` schema being treated as required during merged message-tool schema construction, and fixes #54763 by making that schema optional.
