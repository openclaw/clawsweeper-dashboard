## Summary

Current main already implements the Slack Socket Mode inbound path described in #57844. Socket-mode startup now constructs Bolt with an explicit SocketModeReceiver, message and app_mention handlers are registered before socket start, channel/group messages are handled through Bolt's valid generic message event, and regression tests cover channel/group and app_mention dispatch. The same implementation is present in the v2026.4.24 tag.

## Best Possible Solution

Close this issue as already fixed on main and keep the shipped Slack implementation: explicit SocketModeReceiver construction, valid generic message/app_mention event registration, and existing regression coverage. Any new Slack Socket Mode failure after v2026.4.24 should be opened as a fresh issue with current version, config shape, Slack app event subscriptions, and logs.
