## Summary

Current `main` already has the end-to-end session update path needed for Telegram/WebChat conversation mirroring: Telegram DMs resolve to the shared main session, appended transcript messages emit `session.message` updates, the gateway broadcasts those updates, and Control UI reloads the active session when they arrive.
