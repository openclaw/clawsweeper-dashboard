The issue requests raising the launcher-enforced minimum Node.js minor version from 22.16 to the documented 22.19 minimum.

## Best Possible Solution

Keep the current main alignment between openclaw.mjs, package.json, runtime guard, and launcher regression coverage; the next stable release should carry the fix to users.

## Reproduction Assessment

Yes for the reported release from source: v2026.5.18 accepts Node 22.17 because openclaw.mjs has MIN_NODE_MINOR = 16 while package.json requires >=22.19.0. The same failure no longer reproduces on current main because the launcher now uses 22.19 and the E2E test covers a mocked below-floor version.

## Solution Assessment

Yes. Current main implements the narrow requested fix and adds a regression test tying the launcher guard to package engines and the runtime guard, so no broader product or config change is needed.
