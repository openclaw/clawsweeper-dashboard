## Summary

Current main fixes the reported npm sibling-pruning path by maintaining a managed npm-root manifest and running npm install from that manifest, with regression coverage for sequential plugin installs. The fix is on main after v2026.5.2 and is not proven shipped in a tagged release yet.

## What This Changes

Sequential npm-backed plugin installs should preserve previously installed OpenClaw plugins such as Discord when installing another plugin such as Brave.

## Best Possible Solution

Keep the current managed npm-root manifest behavior and ship it in the next release; no additional core change is needed for this report.

## Reproduction Assessment

Yes from source inspection: the reported v2026.5.2 path is npm pruning siblings in a shared root, and current main now installs from the managed root manifest. I did not run live npm installs because this review was read-only.

## Solution Assessment

Yes. Installing from the managed npm-root package.json after upserting all managed plugin dependencies is the narrowest maintainable fix; it avoids a new config option or plugin-specific special case.
