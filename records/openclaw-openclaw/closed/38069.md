## Summary

Keep this open. Current main still does not implement configurable delayed retry for failed recurring cron executions; it only has global one-shot transient retry plus a recurring error backoff floor that waits for the later of the next natural schedule and the backoff delay. The related implementation PR #69252 was closed unmerged, so there is no merged or shipped fix to close against.

## What This Changes

The issue requests configurable per-job delayed execution retry for failed recurring cron jobs before the next natural scheduled run.

## Best Possible Solution

Add an additive core scheduler feature for recurring execution retry: persist per-job retry policy and attempt state, classify retryable errors, schedule retries at endedAt plus the configured delay when that retry falls before the next natural slot, exhaust attempts cleanly, then resume normal scheduling. Wire it through Gateway validation, CLI add/edit flags, docs, and focused regression tests while keeping existing one-shot retry and failure-alert semantics intact.
