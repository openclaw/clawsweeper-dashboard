## Summary

Keep this PR open. Current main still uses full normalized-query string matching in the memory-wiki digest candidate gate and full-page scorer, and it has no token-overlap helpers or focused regression tests for the multi-term wiki search behavior. The linked user report was already closed as superseded by this PR, so this PR remains the canonical implementation candidate rather than something current main has made obsolete. The PR also mixes the memory-wiki fix with plugin-loader/trust, doctor, gateway/config, and browser test changes that need maintainer review or splitting.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to split or explicitly justify the unrelated loader/trust, doctor, gateway/config, and browser changes, then land this memory-wiki token-overlap fix or an equivalent focused patch that replaces full-string-only candidate gates, preserves exact-phrase priority, avoids zero-token matches, preserves fallback scanning behavior, and includes targeted memory-wiki regression tests.
