## Summary

Keep open. Current main still rejects Feishu Schema 2.0-style card callback context: the parser requires `context.open_id`, `context.user_id`, and `context.chat_id`, while the reported payload shape uses `context.open_chat_id` / `context.open_message_id` and operator user fields. This is a plausible active bug in the Feishu bundled plugin, not a stale or ClawHub-only request.

## Evidence
