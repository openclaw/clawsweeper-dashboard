## Summary

Chatwork is a plausible OpenClaw channel integration, but current main documents this class of optional channel support as an external/community plugin concern rather than an in-repo core feature request.

## Evidence

- **Vision routes optional integrations out of core:** OpenClaw's vision says optional capability should usually ship as plugins, that plugin authors should host and maintain them in their own repository, and that the bar for adding optional plugins to core is intentionally high.
  - file: [VISION.md:54](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/VISION.md#L54)

  - file: [docs/plugins/building-plugins.md:16](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/docs/plugins/building-plugins.md#L16)

## Close Comment

Closing after Codex review. This looks like a good plugin idea, but current OpenClaw direction is to keep optional channel integrations out of core unless there is a strong reason to bundle them. The repo vision says optional capability should usually ship as plugins maintained in their own repo (`VISION.md:54-61`), and the plugin docs say new plugins do not need to be added to this repository and should be published to ClawHub or npm instead (`docs/plugins/building-plugins.md:16-19`, `docs/plugins/building-plugins.md:125-133`). Current docs already treat some chat integrations this way, for example the external WeChat plugin (`docs/channels/wechat.md:10-29`), and the community plugin page lists other channel plugins like DingTalk and WeCom (`docs/plugins/community.md:9-22`, `docs/plugins/community.md:52-62`, `docs/plugins/community.md:115-126`). I also did not find any existing Chatwork code on current `main`. On that basis, this belongs in the ClawHub/community-plugin path rather than as an open core issue. If someone builds it, `openclaw plugins install <package>` is the intended distribution route.
