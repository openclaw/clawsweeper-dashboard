## Summary

Current `main` (`9a0b26cafca76d267dd231b5eaceadb071ec749d`) already carries the end-to-end WebChat image path: the UI sends attachments, gateway `chat.send` persists them into transcript media fields, the reply runner accepts image-only turns via `opts.images`, and the WebChat renderer rehydrates transcript `MediaPath`/`MediaPaths`. The supplied discussion was reviewed, including the confirming comment and the linked PR #61811 in the timeline.
