## Summary

Current main no longer has the unsafe default described in #41530. Omitted `sessions_spawn.cleanup` now resolves to `keep`, kept/session-mode subagents do not get `archiveAtMs`, and transcript-deleting cleanup is limited to explicit delete-mode archive paths with regression coverage and docs.

## Best Possible Solution

Close #41530 as implemented on current main: keep the shipped opt-in cleanup model where default subagent spawns preserve transcripts, and handle any future request to change explicit `cleanup: "delete"` retention semantics as a narrower product decision.
