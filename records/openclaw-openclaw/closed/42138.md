## Summary

Current main already provides the requested CLI-first auth/profile pool and ordered model fallback behavior through the existing models/auth command surface, runtime auth-profile rotation, cooldown/disabled profile handling, and ordered model fallbacks. The shipped namespace differs from the proposal (`models auth order` and `models fallbacks` rather than `auth-pool`), but the observable operational workflow is covered.
