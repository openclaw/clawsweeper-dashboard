## Summary

Current `main` already auto-discovers local skills from `~/.openclaw/skills`, `~/.agents/skills`, `<workspace>/.agents/skills`, and `<workspace>/skills`, and the skills watcher also monitors those roots. The shipped `v2026.4.22` release contains that implementation, matching the docs and test coverage.

## Evidence

- **Loader includes managed and workspace skill roots with documented precedence:** `loadSkillEntries()` loads `CONFIG_DIR/skills` (`~/.openclaw/skills` by default), personal `~/.agents/skills`, project `<workspace>/.agents/skills`, and `<workspace>/skills`, then merges them with precedence `extra < bundled < managed < personal < project < workspace`.
