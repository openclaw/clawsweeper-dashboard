## Summary

Keep open: this is a focused Discord plugin bug fix for gzip response parsing that is still missing on current main, with no blocking review findings. Current main has a newer read-shape guard, but the Discord REST client still reads `response.text()` with no gzip decoding and search can still surface malformed non-object results.

## What This Changes

The PR changes the Discord REST client to decode raw gzip response bytes before JSON parsing and adds message read/search response-shape guards with regression tests.

## Best Possible Solution

Land the plugin-local REST byte-decoding fix and shape guards after current-base checks, while leaving the broader cross-provider gzip investigation to the related canonical items.

## Solution Assessment

Yes. Decoding at the Discord `RequestClient` boundary is the narrow shared fix for read/search/channel REST callers, and the response-shape guards turn malformed data into clear errors without changing auth, endpoints, or request parameters.
