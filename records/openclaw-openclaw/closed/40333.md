## Summary

Current `main` already degrades malformed MCP transport config safely: MCP transport fields are parsed structurally, malformed header/env values are dropped with warnings, unsupported/invalid transports return `null`, and the session MCP runtime skips bad servers instead of aborting gateway startup. This behavior is present in commit `ac063568d354c14e0a0a43bcffb9725bb3fca651` on `main` and does not appear to be in the latest release `v2026.4.22`.

## Evidence

  - file: [src/agents/mcp-transport-config.ts:95](https://github.com/openclaw/openclaw/blob/ac063568d354c14e0a0a43bcffb9725bb3fca651/src/agents/mcp-transport-config.ts#L95)
