The proof is on main only; no release tag contains the proof SHA yet.

## What This Changes

Custom local plugin installs should keep explicit install or load-path precedence over bundled provider discovery when `plugins/installs.json` is regenerated across gateway restarts.

## Best Possible Solution

Keep the current main registry behavior and ship it in the next release; users on `2026.5.7` should upgrade once that release is available or use a main build if they need the fix immediately.

Yes, at source level: the reported restart failure maps to install-record duplicate precedence during registry rebuild, and current main has focused tests for installed globals beating bundled duplicates.
