## Summary

Keep open. Current main still does not accept the Ollama `baseURL` alias: the Ollama discovery, streaming, model-pull, embedding, and web-search paths still read only canonical `baseUrl`. The PR is stale/incomplete against current main, especially after the provider discovery fast path moved through `provider-discovery.ts` and `discovery-shared.ts`, but the underlying user problem remains real and useful to fix.

## Best Possible Solution

Keep the PR open, but require a current-main rework: add a small Ollama-owned helper that reads canonical `baseUrl` first and then own-property `baseURL`, use it in `discovery-shared.ts`, `stream` provider resolution, `setup.ts`, embedding, and web search, and add focused tests for discovery-entry, streaming, embedding, web search, and model-pull behavior with both spellings.
