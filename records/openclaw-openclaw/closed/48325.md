## Summary

This PR adds a new optional observe-only gateway telemetry plugin. Current main does not already implement `gateway-probe`, but the requested capability fits the documented plugin/ClawHub path and can be built with current public plugin hooks plus the diagnostic runtime subpath. No protected labels or maintainer author signal require keeping it open for automatic cleanup purposes.

## Best Possible Solution

Close this core-repo PR as ClawHub/standalone plugin work. Publish and iterate `gateway-probe` as an external plugin using current `api.on(...)` hooks plus `openclaw/plugin-sdk/diagnostic-runtime`; bring back a narrow SDK proposal only if that external implementation uncovers a real missing seam. Official bundling should require explicit maintainer product and security ownership.

## Risks / Open Questions

- If maintainers want an official bundled gateway telemetry exporter, this should be reopened or replaced with an owner-backed design and security review.
- A standalone plugin still needs careful redaction, data-minimization, and Kafka dependency/egress review before users enable it.
- The current PR code imports `registerLogTransport` from the root plugin SDK, but current main does not export that symbol; an external implementation should use `openclaw/plugin-sdk/diagnostic-runtime` for `log.record` events.
