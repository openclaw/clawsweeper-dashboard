## Summary

Current main implements the central Feishu card-action fixes from PR #38609: Schema 2.0 open_chat_id callbacks are normalized, real open_message_id values are used as reply targets without breaking synthetic callback dedupe, and chat type is resolved through the Feishu chat API with p2p fallback. The form_value/input pass-through portion remains separate and is already tracked by narrower Feishu follow-ups (#42908/#43202), so this broad PR is obsolete.

## Best Possible Solution

Close PR #38609 as implemented on main for the central context-field, reply-target, and chat-type fixes. Keep the shipped safer design that uses card-action tokens for synthetic inbound ids and real Feishu message ids only as reply targets, and continue any remaining form_value/option/input pass-through work in the canonical narrower Feishu follow-ups.
