## Summary

Current main already restores full operator scopes and owner semantics for shared-secret bearer auth on the OpenAI-compatible HTTP routes, so the reported `/v1/models` and `/v1/chat/completions` 403 regression is implemented/fixed in the codebase and present in the shipped `v2026.4.23` release.

## Evidence

- **Compat auth restores default operator scopes:** `resolveOpenAiCompatibleHttpOperatorScopes()` returns `CLI_DEFAULT_OPERATOR_SCOPES` for token/password bearer auth, explicitly restoring full operator defaults instead of trusting narrower request scopes.
  - file: [src/gateway/http-utils.ts:232](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/gateway/http-utils.ts#L232)

- **Compat auth restores owner semantics:** `resolveOpenAiCompatibleHttpSenderIsOwner()` returns `true` for shared-secret bearer auth on the compat surface, so chat-completions runs keep operator-owner behavior.
  - file: [src/gateway/http-utils.ts:255](https://github.com/openclaw/openclaw/blob/6b618f063516aa9ba85903aa4565c5a204c055b2/src/gateway/http-utils.ts#L255)
