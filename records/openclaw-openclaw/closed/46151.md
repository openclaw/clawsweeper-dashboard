## Summary

Current `main` already implements this fix path in a different form: gateway lifecycle errors are buffered as potentially retryable, chat-linked fallback runs keep their remapping alive across per-attempt errors, terminal errors still clean up correctly, and the regression is covered by focused tests on `main`. This PR is superseded.
