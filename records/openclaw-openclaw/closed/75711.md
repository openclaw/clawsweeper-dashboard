## Summary

Keep open: the branch is not obsolete on current main, but the patch should not merge as-is because source inspection shows wrong systemd-unit reporting, a test that can mutate the real user service file, missing CLI docs, missing real behavior proof, and a new public feature-lab command that needs maintainer direction.

## What This Changes

The PR adds an `openclaw feature-lab` CLI command that reports feature-lab root/install paths, deployed SHA, systemd service wiring, and unit coverage for that status reader.

## Best Possible Solution

Expose this diagnostic only after maintainer approval, backed by the existing gateway service contracts, temp-isolated tests, CLI docs, and redacted real output proof.

## Reproduction Assessment

Yes. Source inspection of the PR branch is enough to reproduce the review defects: the status reader hardcodes the default systemd unit path, while current main supports custom/profile-derived units, and the test writes to the real home service directory.

## Solution Assessment

No. The implementation should reuse the existing systemd resolver or reader, isolate HOME/service paths in tests, add CLI docs, attach real status output proof, and proceed only if maintainers want this as a public core command.
