## Summary

Current main covers the reported parent-session completion delivery problem: subagent completions now retry and fall back through the requester-agent path, timeout/error outcomes are explicit, and background exec exits have notify-on-exit delivery with documented quiet-success behavior.

## What This Changes

Report unreliable completion announcements from subagents and background `exec` sessions with `yieldMs` back to parent sessions.

## Best Possible Solution

Keep the current main implementation: subagent completions use the hardened requester-agent delivery path, while exec users who need quiet-success wakes can enable `tools.exec.notifyOnExitEmptySuccess` or poll the process session.

## Reproduction Assessment

No live Telegram reproduction was run in this read-only review. Source and regression tests show the reported 2026.5.7 delivery gaps are now covered on current main, including explicit timeout handling and exec notify-on-exit routing.

## Solution Assessment

Yes. The current approach keeps subagent completions on the requester-agent/session route and leaves quiet successful exec events behind an explicit config knob, which matches the documented exec contract.
