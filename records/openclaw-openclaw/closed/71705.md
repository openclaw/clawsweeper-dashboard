## Summary

Keep #71705 open. Current main still applies the shared 180000ms OpenAI image timeout to Azure image generation when no per-call or configured timeout is provided. A config-level timeout workaround exists, but the requested Azure-aware default behavior remains unimplemented; linked PR #71708 is closed/unmerged in the provided context and its head commit is not present in this checkout.
