## Summary

The report targets Lossless Claw's own message/FTS storage, but current `openclaw/openclaw` only treats Lossless Claw as an external context-engine plugin. OpenClaw core documents that plugin context engines own ingest, compaction, and their own data stores, so this should be tracked in the Lossless Claw plugin/repo rather than the core repo.

## Evidence

- **Lossless Claw is documented as an external plugin:** `main` documents Lossless Claw (LCM) as a separately installed plugin, with its own npm package and repository, not as bundled core code.
  - file: [docs/plugins/community.md:64](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/docs/plugins/community.md#L64)
