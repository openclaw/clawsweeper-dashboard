## Summary

Close. Current main already bounds the Control UI session-history load path and no longer returns heavyweight session-list rows, which covers the reported crash mode. The same protections are present in release `v2026.4.22`.

## Evidence

- **Control UI only requests a bounded history window:** `loadChatHistory()` requests `chat.history` with `limit: 200`, so the browser no longer asks Gateway to hydrate an unbounded transcript on refresh.
  - file: [ui/src/ui/controllers/chat.ts:152](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/ui/src/ui/controllers/chat.ts#L152)

- **Gateway truncates history by count and payload size:** `chat.history` caps to the last 200 messages by default, applies text truncation, replaces oversized messages with placeholders, and enforces a final byte budget before responding.
  - file: [src/gateway/server-methods/chat.ts:1902](https://github.com/openclaw/openclaw/blob/f03252aaf9cc946dd6a4f6189f30d5c5dcd64022/src/gateway/server-methods/chat.ts#L1902)
