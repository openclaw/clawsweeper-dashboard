## Summary

This PR should close because its useful behavior is superseded by the shipped loader-boundary fallback, while the remaining package-level `sharp` install gap is tracked separately; the branch’s direct `optimizeImageToJpeg` passthrough is less safe than current main.

## What This Changes

The PR replaces the terminal `optimizeImageToJpeg` failure with returning the original buffer and zero resize/quality metadata.

## Best Possible Solution

Keep the shipped loader-boundary fallback, track the root `sharp` package-install gap in the canonical issue, and close this unsafe branch rather than merging it.

## Reproduction Assessment

Yes. The historical missing-optimizer failure is source-reproducible through `loadWebMedia` into Sharp-backed image ops, and the PR’s defect is source-reproducible because it catches every resize failure before MIME and guard policy can be applied.

## Solution Assessment

No. The maintainable fix is the current loader-boundary fallback plus separate package-dependency tracking, not returning original bytes from `optimizeImageToJpeg` where MIME, cap, HEIC, and guard context is unavailable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve image input guard failures:** `src/media/web-media.ts:657-662`
  - body: The fallback returns `source` after every `resizeToJpeg` attempt throws. `resizeToJpeg` is where pixel-limit and unknown-dimension guard errors are enforced, so this can accept byte-small but unsafe images instead of failing closed.
  - confidence: 0.91
- **[P2] Preserve original MIME on fallback:** `src/media/web-media.ts:657-662`
  - body: For non-HEIC PNG/WebP inputs, `source` is still the original bytes, but callers wrap any `optimizeImageToJpeg` result as JPEG and emit `image/jpeg`. Consumers can receive PNG/WebP bytes labeled as JPEG.
  - confidence: 0.92
- **[P3] Keep optimizer diagnostics on failed fallbacks:** `src/media/web-media.ts:657-662`
  - body: When the original is still over cap and all resize attempts fail, this return path turns the detailed optimizer failure into a generic cap error from the caller. Preserve the underlying optimizer cause for unsendable originals.
  - confidence: 0.82
