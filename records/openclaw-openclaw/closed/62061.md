## Summary

Current `main` already short-circuits `before_dispatch` before the normal reply/model path, so a hook that returns `{ handled: true }` does not proceed into the session-writing flow described in the report. The separate ACP raw transcript append noted in the issue still exists, but that is a distinct path from the handled `before_dispatch` channel flow.
