## Summary

Current main already routes Telegram media downloads through the proxy-aware guarded fetch path, skips target DNS pinning for trusted explicit proxies, and documents the Telegram-only private-network opt-in for fake-IP/transparent proxy environments. The changelog says this shipped in 2026.4.5.

## Evidence

- **Media download path preserves explicit proxy transport metadata:** Telegram media downloads pass `transport.sourceFetch`, `dispatcherAttempts`, and `trustExplicitProxyDns` into `fetchRemoteMedia`, so media fetches use the proxy-aware transport instead of dropping back to an unrelated fetch path.
  - file: [extensions/telegram/src/bot/delivery.resolve-media.ts:235](https://github.com/openclaw/openclaw/blob/37c37eec/extensions/telegram/src/bot/delivery.resolve-media.ts#L235)
  - command: `nl -ba extensions/telegram/src/bot/delivery.resolve-media.ts | sed -n '232,248p'`
  - sha: [37c37eec](https://github.com/openclaw/openclaw/commit/37c37eec)
- **Guarded media fetch has trusted explicit-proxy mode:** `fetchRemoteMedia` switches to `withTrustedExplicitProxyGuardedFetchMode(...)` when `trustExplicitProxyDns` is set on an explicit-proxy attempt, which is the code path used by Telegram media downloads.
  - file: [src/media/fetch.ts:131](https://github.com/openclaw/openclaw/blob/37c37eec/src/media/fetch.ts#L131)
  - command: `nl -ba src/media/fetch.ts | sed -n '110,155p'`
  - sha: [37c37eec](https://github.com/openclaw/openclaw/commit/37c37eec)
- **SSRF guard now lets trusted explicit proxies resolve target DNS after hostname checks:** `fetchWithSsrFGuard` has `TRUSTED_EXPLICIT_PROXY` mode and, in that mode, skips target DNS pinning while still enforcing hostname policy, then builds the explicit proxy dispatcher without pinned target DNS.
  - file: [src/infra/net/fetch-guard.ts:43](https://github.com/openclaw/openclaw/blob/37c37eec/src/infra/net/fetch-guard.ts#L43)
  - command: `nl -ba src/infra/net/fetch-guard.ts | sed -n '38,50p;125,175p;345,385p'`

- **Regression tests cover Telegram explicit-proxy media case:** Tests explicitly verify Telegram media URLs work with an explicit proxy, and that trusted explicit-proxy mode skips target DNS pinning after hostname-policy checks.
  - file: [src/infra/net/fetch-guard.ssrf.test.ts:1102](https://github.com/openclaw/openclaw/blob/37c37eec/src/infra/net/fetch-guard.ssrf.test.ts#L1102)

  - file: [extensions/telegram/src/bot/delivery.resolve-media-retry.test.ts:365](https://github.com/openclaw/openclaw/blob/37c37eec/extensions/telegram/src/bot/delivery.resolve-media-retry.test.ts#L365)
