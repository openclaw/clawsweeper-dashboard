## Summary

Current `main` no longer sends every `sessions_send` call through one shared `nested` lane. The tool now scopes nested delivery lanes per target session, so different agent targets no longer serialize behind a single `maxConcurrent: 1` queue. The same implementation is present in shipped release `v2026.4.22`.

## Evidence

## Risks / Open Questions

- `src/gateway/server-lanes.ts` and `src/gateway/server-reload-handlers.ts` still do not call `setCommandLaneConcurrency(CommandLane.Nested, ...)`, so the original proposed fix did not land literally; the bug is addressed by per-session lane scoping instead.
- The remaining fallback case with no session key still uses the unscoped `nested` lane, but the reported multi-agent fan-out path in this issue targets concrete sessions and is covered by the scoped-lane implementation.
