## Summary

Close as superseded by the existing native conversation/sidebar tracking work. PR #61721 is a partial Control UI sidebar branch with unresolved review comments showing it would be non-functional, while current main already has a different session selector, a /new reset button, and the sessions.create backend surface needed for a fresh implementation. The remaining sidebar/product work is still tracked by #53032 and related issue #52266, so this PR should not remain the canonical implementation path.

## Best Possible Solution

Close PR #61721 as superseded and keep the canonical feature tracking in #53032/#52266. The best implementation path is a fresh, focused Control UI PR that uses current sessions.create/sessions.list/session subscription APIs, wires sidebar state through app-view-state and app-render, preserves the existing chat sidebar behavior, adds UI tests, and avoids unrelated build configuration changes.
