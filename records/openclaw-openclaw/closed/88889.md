## Summary

Close: current main already fixes the Feishu WebSocket inbound dispatch crash by carrying the injected channel runtime into the Feishu monitor path instead of relying on the uninitialized fallback; the fix is main-only and not in v2026.5.28.

## What This Changes

The issue reports Feishu WebSocket messages being received but failing to dispatch to the agent with `Cannot read properties of undefined (reading 'run')` on OpenClaw 2026.5.28.

## Best Possible Solution

Keep the current Feishu monitor runtime handoff on main and let the next release carry it; use the older Feishu dispatch reports as release-regression context if more verification is needed.

## Reproduction Assessment

Yes for the affected release: v2026.5.28 source omits the Feishu `channelRuntime` handoff while the reported log reaches `core.channel.inbound.run`. I did not establish a failing current-main path because current main carries the runtime through gateway startup.

## Solution Assessment

Yes. Passing the already-injected channel runtime through the Feishu gateway and monitor is the narrow owner-boundary fix; adding guards around `core.channel.inbound.run` would hide the bad handoff and still risk dropping messages.
