## Summary

Current main already preserves a JSON-only stdout contract for `--json` CLI commands by routing diagnostic/plugin-load console output to stderr, and the codebase has targeted tests plus docs for that behavior.

## Evidence

- **JSON-mode startup routes diagnostics off stdout:** `applyCliExecutionStartupPresentation` enables stderr routing whenever the command is in JSON output mode (`suppressDoctorStdout`).
  - file: [src/cli/command-execution-startup.ts:29](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/cli/command-execution-startup.ts#L29)
  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
- **Plugin bootstrap inherits JSON-mode stderr routing:** CLI bootstrap forwards `routeLogsToStderr: params.suppressDoctorStdout` into plugin registry loading, so plugin startup logs do not stay on stdout for `--json` commands.
  - file: [src/cli/command-bootstrap.ts:14](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/cli/command-bootstrap.ts#L14)
  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
- **Plugin loader flips console writes to stderr during registry load:** `ensureCliPluginRegistryLoaded` temporarily sets `loggingState.forceConsoleToStderr = true` around `ensurePluginRegistryLoaded(...)` when JSON-mode routing is requested.
  - file: [src/cli/plugin-registry-loader.ts:18](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/cli/plugin-registry-loader.ts#L18)
  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
- **Patched console implementation sends console.* output to stderr in JSON mode:** When `loggingState.forceConsoleToStderr` is set, the console capture layer writes formatted `console.*` output to `process.stderr` instead of calling the original stdout-backed console methods.
  - file: [src/logging/console.ts:276](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/logging/console.ts#L276)
  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
- **Regression tests cover both general JSON-mode routing and plugin-load routing:** Tests assert that `--json` commands route logs to stderr so stdout stays clean, and separately assert that plugin loading also runs with `forceConsoleToStderr=true` and restores state afterward.
  - file: [src/cli/program/preaction.test.ts:394](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/cli/program/preaction.test.ts#L394)
  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
- **Affected commands explicitly emit structured JSON on stdout:** `plugins list --json` uses a quiet logger and writes the JSON payload via `defaultRuntime.writeJson(...)`; `hooks` JSON output is also documented as writing structured JSON directly to stdout.
  - file: [src/cli/plugins-cli.ts:71](https://github.com/openclaw/openclaw/blob/7d471837369885c12d540d6bafa319ff41129b89/src/cli/plugins-cli.ts#L71)
  - sha: [7d4718373698](https://github.com/openclaw/openclaw/commit/7d471837369885c12d540d6bafa319ff41129b89)
- **Latest release tag already contains the same fix path:** Diffing the relevant routing files from `v2026.4.22` to `HEAD` shows no change in the CLI/plugin stderr-routing implementation; the latest tagged release already contains this behavior.
  - command: `git diff --stat v2026.4.22..HEAD -- src/cli/plugin-registry-loader.ts src/cli/command-execution-startup.ts src/cli/command-bootstrap.ts`
