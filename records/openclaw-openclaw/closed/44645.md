## Summary

Current main already preserves one-shot cron execution history through the cron run log and background task ledger, even when the job auto-deletes after a successful run.

## Risks / Open Questions

- The implementation does not use a single `history.json` file; it uses `~/.openclaw/cron/runs/<jobId>.jsonl` plus the task ledger, so this closes the issue as functionally implemented rather than as the exact proposed storage shape.
