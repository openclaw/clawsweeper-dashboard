Keep open: current main still exposes the Android reconnect race, and both linked fix PRs are closed unmerged, so notification events can still be lost before the Gateway connect handshake completes.

## What This Changes

Fix Android gateway reconnect so notification `node.event` frames wait for the connect handshake and notification events arriving during reconnect are queued or reconciled.

## Best Possible Solution

Fix Android-side connection readiness and bounded notification replay/reconciliation while keeping the Gateway connect-first protocol unchanged.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `currentConnection` is visible before `conn.connect()` completes, the notification sink can call `sendNodeEvent` in that window, and the Gateway rejects non-connect first frames.

## Solution Assessment

Yes. The narrow maintainable direction is Android-side: do not expose or use a `node.event`-capable connection until connect succeeds, and preserve notification events through bounded queueing or active-notification reconciliation after reconnect.

## Repair Work Prompt

Fix the Android gateway reconnect notification-loss bug tracked by https://github.com/openclaw/openclaw/issues/79552. Keep the Gateway protocol unchanged: the first WebSocket frame must remain `connect`. In Android, prevent `sendNodeEvent` and `sendNodeEventDetailed` from sending through a connection before the connect RPC/ACK has completed; prefer publishing `currentConnection` only after `conn.connect()` returns ready metadata, or use an explicit handshake-complete readiness guard if that better fits the existing lifecycle. Preserve notification events that arrive during reconnect with a bounded queue and/or active-notification reconciliation after reconnect; respect existing notification forwarding filters, avoid unbounded memory growth, and avoid duplicate agent wakeups. Add focused Android regression coverage for a delayed connect ACK and for notification events being retained/flushed or reconciled after reconnect.
