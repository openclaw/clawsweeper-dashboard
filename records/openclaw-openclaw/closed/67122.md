## Summary

Current main implements the suggested fix: the workspace-* blacklist in assertLocalMediaAllowed only applies when localRoots are omitted, while image and PDF tools now pass explicit roots that include the active agent workspace. The same implementation is present in the v2026.4.24 tag, and tests cover explicit workspace-* roots being allowed while preserving the implicit-default-root block.

## Best Possible Solution

Close this issue as implemented. Keep the current behavior: explicit agent-scoped localRoots should allow the active workspace-<agentId> directory, while default-root media reads should continue blocking arbitrary sibling workspace-* directories unless a caller provides explicit roots.
