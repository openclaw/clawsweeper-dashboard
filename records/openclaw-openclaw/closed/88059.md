## Summary

Keep open: the PR is useful and has real proof, but the new `annotations` metadata is not yet correct for device-emulated/high-DPI screenshots, and the scoped-label behavior still carries compatibility risk that maintainers should explicitly accept before merge.

## What This Changes

The PR updates the browser plugin so Playwright-backed labeled screenshots honor full-page/ref/element scopes, adds optional per-ref `annotations` metadata, and refreshes related docs/tests.

## Best Possible Solution

Land only after annotation boxes match saved-image pixels across device emulation and normalization, and after a maintainer explicitly accepts the scoped-label behavior; otherwise preserve the old viewport fallback or introduce a named opt-in.

## Reproduction Assessment

Yes. Source inspection confirms current main does not forward `fullPage`, `ref`, or `element` into the labeled Playwright screenshot helper; the new DPR mismatch is source-reproducible from OpenClaw device emulation plus Playwright screenshot defaults.

## Solution Assessment

No, not yet. The helper is a reasonable owner-boundary shape, but the current implementation returns CSS-pixel annotation boxes while screenshots default to device pixels; the narrower fix is to make the metadata and saved image use the same coordinate scale.
