## Summary

Current main contains the timeout-path fixes that address this MiniMax/Anthropic-compatible truncation regression, and they shipped in v2026.4.23.

## Evidence

- **Regression shape in v2026.4.5:** The v2026.4.5 tree called `ensureGlobalUndiciStreamTimeouts()` directly from `runEmbeddedAttempt` without threading the run timeout through the guarded transport path, matching the issue's post-4.5 regression window.

- **Embedded-run timeout floor added:** Current main routes embedded-run HTTP setup through `configureEmbeddedAttemptHttpRuntime()` and clamps the undici stream timeout to at least `DEFAULT_UNDICI_STREAM_TIMEOUT_MS`, preventing short attempt budgets from shrinking shared stream timeouts.
  - file: [src/agents/pi-embedded-runner/run/attempt-http-runtime.ts:7](https://github.com/openclaw/openclaw/blob/245451b6a9c529464e177afa65013adca0d63eb5/src/agents/pi-embedded-runner/run/attempt-http-runtime.ts#L7)
