## Summary

Keep open. Current `main` still shows the gateway/CLI ownership split the issue describes: the gateway service eagerly starts the voice-call runtime and webhook server, but the `voicecall` CLI still calls the same local `ensureRuntime()` path instead of routing through the existing gateway RPC handler. I did not find a main-branch code change that would make the CLI reuse a gateway-owned webhook server.
