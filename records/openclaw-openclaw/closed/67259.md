## Summary

Current main already solves the user-visible Control UI bug this PR targets. Live assistant text and pre-text activity are emitted as stream/reading-indicator chat items independent of `showToolCalls`, optimistic sends create immediate visible state, and pre-tool streamed text is preserved before tool cards are rendered or hidden. The linked bug #67250 is already closed as implemented, so this one-line guard PR is now obsolete rather than necessary to land.

## Best Possible Solution

Close this PR as already implemented on current main. Keep the current stream-item and reading-indicator architecture; if maintainers want extra hardening, add a focused regression test for `showToolCalls: false` plus empty active stream rather than landing this stale one-line guard as the primary fix.
