## Summary

Keep open. The Kimi endpoint switch is still useful because current main continues to advertise image-capable Kimi models while routing defaults through the Anthropic-compatible endpoint, but this external PR needs real behavior proof, a refresh against current main, and two small cleanup fixes before merge.

## What This Changes

The PR switches bundled Kimi Coding provider defaults, onboarding/tests, docs, and changelog from Anthropic Messages at `https://api.kimi.com/coding/` to OpenAI chat completions at `https://api.kimi.com/coding/v1`, and adds Kimi endpoint metadata.

## Best Possible Solution

Land a rebased Kimi-plugin-owned endpoint switch that preserves `kimi-for-coding`, removes or implements endpoint metadata, aligns Kimi docs, and includes redacted real Kimi image proof.

## Reproduction Assessment

Yes. Current main is source-reproducible because it advertises image-capable Kimi models while routing default Kimi requests through the Anthropic-compatible endpoint, and the linked issue supplies a concrete failing and working curl comparison.

## Solution Assessment

Mostly yes. Switching the provider-owned Kimi defaults to OpenAI completions at `/coding/v1` is the narrow maintainable fix, but this PR should first refresh against current main, remove or implement ineffective endpoint metadata, align docs, and add real after-fix proof.
