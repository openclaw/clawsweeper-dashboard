## What This Changes

The issue reports that persistent claude-cli live sessions can bind to a stale internal session id, causing false `missing-transcript` invalidation and cold resume fallback despite valid Claude transcripts under a different id.

## Best Possible Solution

Route this report’s concrete logs to https://github.com/openclaw/openclaw/issues/81042 and land or replace https://github.com/openclaw/openclaw/pull/81048 with last-wins Claude JSONL session-id capture plus transcript-probe regression coverage.

## Reproduction Assessment

Yes at source level: current main captures the first JSONL `session_id`, persists it as the next CLI binding, then invalidates that exact id when no matching Claude transcript exists. I did not run a live Claude CLI group-chat reproduction in this read-only review.

## Solution Assessment

Yes for cleanup routing: the maintainable path is to keep one canonical tracker and fix PR for this parser/transcript validation bug rather than split the same stale-binding failure across duplicate issues.
