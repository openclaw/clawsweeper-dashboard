## Summary

The PR is a focused security hardening with sufficient terminal proof, but it should stay open because it intentionally changes provider credential source precedence and the public docs still describe cwd `.env` as a normal env source without the new provider-credential exception.

## What This Changes

The PR adds registered provider credential env var names to the workspace dotenv blocklist and adds dotenv regression coverage that trusted global values win over workspace values.

## Best Possible Solution

Land the fail-closed provider credential rule only with maintainer approval and docs that direct users to shell/global `.env`, SecretRefs, or auth storage.

## Reproduction Assessment

Yes. Source inspection shows current main loads workspace `.env` before the global fallback while only blocking a subset of provider credential keys, and the PR body supplies before/after terminal output; I did not execute the scenario in this read-only review.

## Solution Assessment

Not yet. Blocking registered provider credentials from workspace `.env` is a narrow security fix, but the merge shape still needs maintainer acceptance of the compatibility break and matching docs/migration guidance.
