## Summary

Current main no longer matches the reported no-change Mac App save path: the macOS Config tab disables Save unless the draft is dirty, warns on invalid config snapshots, and the gateway withholds invalid raw/config payloads to avoid unsafe round trips. The example top-level `auth` field is not a valid current OpenClaw root key under strict config validation. For valid configs, the gateway write path uses the source-preserving merge logic that also resolved the broader config.write clobber class previously tracked in #55219, and the relevant behavior is present in v2026.4.26.

## Best Possible Solution

Keep the current dirty-gated macOS Config save flow and source-preserving gateway write path. If a user can still reproduce data loss with a valid schema path after making an actual dirty edit, track that as a new focused bug with the exact field path, before/after config, and whether the write went through gateway `config.set` or the local fallback.
