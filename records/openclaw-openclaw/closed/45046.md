## Summary

Current `main` already includes OpenAI-compatible tool-call compatibility fixes that cover the reported LiteLLM/vLLM symptom: proxy-style backends no longer get forced strict tool schemas, and vLLM-style completions no longer get unsolicited `tool_choice` payloads. The remaining reports in the thread were also traced to vLLM/Qwen runner configuration rather than a current OpenClaw bug.

## Evidence

- **Shipped fix in changelog:** `CHANGELOG.md:1907-1981` shows release `2026.3.28` and explicitly notes: `Models/openai-completions: default non-native OpenAI-compatible providers to omit tool-definition strict fields unless users explicitly opt back in, so tool calling keeps working on providers that reject that option. (#45497)`
  - file: [CHANGELOG.md:1907](https://github.com/openclaw/openclaw/blob/4630ce3d9e2f089062ccd386b0ae241325961d1a/CHANGELOG.md#L1907)

## Risks / Open Questions
