## Summary

Keep open. Current `main` still treats `--format ai` snapshots as `eN`-style refs rather than numeric refs, and the docs still promise numeric AI refs. The issue is not implemented on `main` or in the latest release evidence available here.

## Evidence

- **Explicit `format=ai` route still uses the AI snapshot path without any numeric-ref normalization:** The `plan.format === "ai"` branch calls `snapshotAiViaPlaywright(...)` and returns `...snap` directly. The only fallback is when Playwright lacks `_snapshotForAI`, in which case it falls back to a role snapshot. There is no conversion from `eN` refs to numeric refs before returning the AI snapshot.
  - file: [extensions/browser/src/browser/routes/agent.snapshot.ts:463](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/extensions/browser/src/browser/routes/agent.snapshot.ts#L463)
  - command: `nl -ba extensions/browser/src/browser/routes/agent.snapshot.ts | sed -n '463,539p'`
  - sha: [ecac696643c6](https://github.com/openclaw/openclaw/commit/ecac696643c604c6d854341bfe6424bfa31423e9)
- **AI snapshot implementation preserves `eN` refs:** `snapshotAiViaPlaywright` returns Playwright's snapshot text as-is, then calls `buildRoleSnapshotFromAiSnapshot(snapshot)` and stores those refs in `mode: "aria"`. There is no numeric-ref rewrite step in the AI snapshot path.
  - file: [extensions/browser/src/browser/pw-tools-core.snapshot.ts:61](https://github.com/openclaw/openclaw/blob/ecac696643c604c6d854341bfe6424bfa31423e9/extensions/browser/src/browser/pw-tools-core.snapshot.ts#L61)
