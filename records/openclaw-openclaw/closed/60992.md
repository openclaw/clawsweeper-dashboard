## Summary

Current main already ships an opt-in `web_fetch` fake-IP compatibility setting for RFC2544 proxy DNS responses (`198.18.0.0/15`), so the issue's claim that there is no working solution is now obsolete.

## Evidence

- **Config surface added:** `web_fetch` config now exposes `tools.web.fetch.ssrfPolicy.allowRfc2544BenchmarkRange`, with inline docs explicitly calling out fake-IP proxy compatibility for Clash TUN / Surge-style `198.18.0.0/15` responses.
