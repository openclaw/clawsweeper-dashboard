## Summary

Current main already fixes the reported subagent pairing-gate failure. `sessions_spawn` still reaches the Gateway RPC path, but internal loopback backend calls now use the `gateway-client`/`backend` shared-secret path without depending on the CLI paired-device baseline, and subagent dispatch no longer forces the `agent` call through admin scopes that trigger headless scope-upgrade pairing.

## Best Possible Solution

Close as implemented on current main. Keep the narrow direct-loopback backend shared-auth pairing bypass plus least-privilege subagent scope routing; treat a true in-process dispatch shortcut as a separate performance/refactor proposal if maintainers still want it.
