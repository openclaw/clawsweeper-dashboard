Full review comments:

- **[P1] Preserve skill eligibility before explicit filters:** `src/agents/skills/workspace.ts:79-81`
  - body: The explicit-name shortcut runs before `shouldIncludeSkill`, so `skills.entries.<key>.enabled === false`, bundled allowlists, and runtime `requires` checks can be bypassed by listing the skill in an agent filter.   - confidence: 0.92
- **[P2] Recreate containers when the env policy changes:** `src/agents/sandbox/docker.ts:370-372`
  - body: Changing create-time env filtering without changing the sandbox config hash leaves already-created containers on the old filtered env set, so users may not see the fix until manual recreation. The merged implementation solves this with an explicit env-policy hash epoch.
  - confidence: 0.86
