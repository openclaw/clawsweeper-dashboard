## Summary

Close as implemented on current main: the Telegram callback handler now sends `answerCallbackQuery` immediately, before the callback path reaches agent/model dispatch. The proof commit is present on `main`/`origin/main` but is not contained in a release tag I could verify.

## What This Changes

The issue asks for Telegram inline-keyboard callbacks to be acknowledged before invoking the model so slow model responses do not make `answerCallbackQuery` expire.

## Best Possible Solution

Keep the current Telegram-layer early-ACK behavior and let the next release carry it; any future report should include logs from a build after the proof commit if callback ACKs still expire.

## Reproduction Assessment

No for current main. Source inspection shows `answerCallbackQuery` is awaited at callback entry before the raw callback fallback reaches `runInboundReplyTurn`, so the reported late-ACK path no longer reproduces in current source.

## Solution Assessment

Yes. The current implementation is the narrow Telegram-owned fix: ACK the callback immediately and process the callback payload afterward without adding core behavior or configuration.
