## What This Changes

The PR changes gateway session listing so configured main-session keys return an explicit display name or `Main Session` instead of transient origin labels such as `heartbeat`.

## Best Possible Solution

Consolidate the heartbeat display-name repair in the active shared-builder follow-up, preserving explicit display names and adding regression coverage for both `sessions.list` and live `sessions.changed` rows.

## Reproduction Assessment

Yes, by source inspection: a main session row with no explicit `displayName` or `label` and `origin.label: "heartbeat"` flows through `buildGatewaySessionRow`, and the UI uses that row label before the `Main Session` key fallback. I did not run tests because this review is read-only.

## Solution Assessment

No. The intended behavior is right, but this PR changes a stale list-only location; the narrow maintainable fix is in `buildGatewaySessionRow` or a shared display-name helper, which #66544 already targets.
