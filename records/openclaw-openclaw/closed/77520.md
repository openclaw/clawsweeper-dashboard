## Summary

Keep open: current main still documents and passes the `automatic` mode, but the Discord path can set terminal `done` and treat queued/route-success signals as final dispatch without proof that a visible Discord reply landed, so this remains distinct from the earlier workaround issues.

## What This Changes

The issue asks for Discord guild/channel turns with `messages.groupChat.visibleReplies: "automatic"` to post normal final assistant text, or avoid terminal success status when delivery did not happen.

## Best Possible Solution

Keep the issue open and fix Discord/channel final-reply accounting so automatic turns either deliver a visible final reply or surface the failed/missing dispatch instead of showing terminal success.
