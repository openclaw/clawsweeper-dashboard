## Summary

Current main still routes Feishu card-action callbacks through a synthetic `card-action-${token}` message id and passes that id into the Feishu reply path. The existing fallback only handles withdrawn/not-found reply targets, not the reported `Invalid ids: [card-action-...]` case, so this is a plausible current bug rather than cleanup-close material.
