## Summary

Close as not reproducible on current main. The target write path already creates a random temp file with mode `0o600` before `rename()`, the reported `v2026.4.15` source had the same pattern, and an ephemeral Node filesystem check with a `0664` target plus `umask 0002` ended with final mode `0600`; the linked bug report and duplicate chmod-after-rename PRs were also closed unmerged after maintainer rechecks.

## What This Changes

This PR adds a best-effort `chmod(configPath, 0o600)` after the successful atomic rename in `src/config/io.ts` so hot-saved `openclaw.json` would be tightened after writes.

## Best Possible Solution

Close this PR and keep the current atomic temp-file behavior unless a fresh current-main reproduction identifies a different writer path that can persist loose config permissions; any future hardening should include an accurate comment and a regression test for the real path.
