Full review comments:

- **[P1] Make the documented pipe install usable:** `docs/install/vm.md:53`
  - body: The guide's primary install command pipes this script through `bash`, but the script now only succeeds when `BASH_SOURCE[0]` points at a local `scripts/virtualbox` checkout and otherwise exits with the clone-and-run error. Users following the quick path cannot install `openclaw-vm` from stdin.
  - confidence: 0.97
- **[P1] Parse VM config without sourcing passwords:** `scripts/virtualbox/openclaw-vm:396`
  - body: `VM_PASSWORD` is written from user input into `.env`, and `status` later sources that file. A password containing shell syntax such as command substitution will be evaluated on the host, so config display needs a parser that does not execute the file.
  - confidence: 0.98
- **[P1] Keep SSH forwarding private until LAN access is explicit:** `scripts/virtualbox/Vagrantfile:50`
  - body: This forwards guest SSH on `0.0.0.0:2222` for every VM, and the provisioner can enable password auth for the `vagrant` user. An official setup script should not widen the host network boundary by default without an explicit opt-in and owner-approved hardening.
  - confidence: 0.9
- **[P2] Configure gateway bind before promising host WebChat:** `docs/install/vm.md:134`
  - body: The guide says the host browser can open `http://localhost:18789`, but OpenClaw's gateway defaults to loopback inside the guest. With Vagrant port forwarding, host traffic reaches the guest interface, so the docs or setup must set a supported non-loopback bind with auth or use an SSH tunnel.
  - confidence: 0.88
- **[P2] Propagate reprovisioning failures:** `scripts/virtualbox/openclaw-vm:442-445`
  - body: `openclaw-vm provision` masks `vagrant provision` failures with `|| true` and then prints `Re-provisioning complete`, so users can get a false success after a failed setup or security-config change.
  - confidence: 0.9
