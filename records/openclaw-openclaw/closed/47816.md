Full review comments:

- **[P1] Make the documented piped installer work:** `scripts/virtualbox/install.sh:149`
  - body: The guide tells users to run this script with `curl ... | bash`, but stdin execution leaves no local checkout beside the script, so the installer reaches this hard-fail path instead of installing `openclaw-vm`.
  - confidence: 0.97
- **[P1] Initialize vagrant_status on successful create:** `scripts/virtualbox/openclaw-vm:323`
  - body: On the first-create path, a successful `vagrant up | track_provision_progress` never assigns `vagrant_status`. With `set -u`, the later expansion exits with an unbound-variable error after provisioning succeeds.
  - confidence: 0.96
- **[P1] Parse VM config without sourcing password text:** `scripts/virtualbox/openclaw-vm:396`
  - body: The setup flow writes `VM_PASSWORD` into `.env`, and `status` later sources that file. Passwords containing shell syntax or unmatched quotes can execute on the host or break parsing, so parse known keys without shell evaluation.
  - confidence: 0.98
- **[P2] Forward ssh subcommand arguments:** `scripts/virtualbox/openclaw-vm:570`
  - body: The headless docs use `openclaw-vm ssh -c "openclaw gateway"`, but the dispatcher still calls `cmd_ssh` without `"$@"`, so the wrapper opens an interactive SSH session instead of running the requested command.
  - confidence: 0.97
- **[P2] Configure gateway access before promising WebChat:** `docs/install/vm.md:134`
  - body: The guide starts the default gateway inside the VM and then tells users to open host `localhost:18789`. Current gateway docs say the default bind is loopback, so the VM instructions need a bind change or SSH tunnel before promising host browser access.
  - confidence: 0.88
- **[P2] Enter the cloned checkout before local install:** `docs/install/vm.md:60`
  - body: The fallback path clones `openclaw/openclaw` and then immediately runs `cd scripts/virtualbox` from the parent directory. That directory only exists after entering the cloned `openclaw` checkout.
  - confidence: 0.95
- **[P2] Propagate reprovision failures:** `scripts/virtualbox/openclaw-vm:442`
  - body: `provision` masks `vagrant provision` failures with `|| true` and always prints success, so failed package setup, firewall changes, or SSH provisioning can be hidden from users and automation.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff introduces security-sensitive host/guest setup scripts, LAN SSH exposure, and shell-evaluated password config that need attention before merge.

Concerns:

- **[high] Raw .env sourcing executes password text:** `scripts/virtualbox/openclaw-vm:396`
  - body: `VM_PASSWORD` is written from user input into `.env`, then `openclaw-vm status` sources that file. A password containing shell syntax can execute on the host.
  - confidence: 0.98
- **[medium] SSH is forwarded on all host interfaces:** `scripts/virtualbox/Vagrantfile:50`
  - body: The Vagrantfile exposes guest SSH on host port 2222 with `host_ip: "0.0.0.0"`, and provisioning can enable password auth for the `vagrant` user. Official scripts need owner-approved defaults before widening this boundary.
  - confidence: 0.9
- **[medium] Provisioning executes remote installer scripts:** `scripts/virtualbox/scripts/provision.sh:17`
  - body: The proposed provisioning downloads and runs external package/bootstrap scripts inside the official VM setup path; that may be acceptable, but it needs explicit ownership and clean-machine proof.
  - confidence: 0.82
