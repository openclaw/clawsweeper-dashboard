## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Move the Tauri build scripts to crate roots:** `apps/linux/src/build.rs:1-3`
  - body: Cargo only auto-discovers build.rs at the package root unless [package].build points elsewhere. Both new crates put tauri_build::build() under src/build.rs and omit a build override, so Tauri build-time generation is skipped.
  - confidence: 0.94
- **[P1] Add the missing Windows Vite entry files:** `apps/windows/package.json:6-8`
  - body: The Windows package exposes Vite dev/build scripts, but the PR does not add apps/windows/index.html, src/main.tsx, or an app component. A clean Windows app build therefore has no frontend entry to serve or bundle.
  - confidence: 0.93
- **[P1] Commit the referenced bundle icon assets:** `apps/linux/tauri.conf.json:31-37`
  - body: Both Tauri configs reference icons/32x32.png, icons/128x128.png, icons/128x128@2x.png, icons/icon.icns, and icons/icon.ico, but no icon files are added. Release packaging will fail or depend on untracked local assets.
  - confidence: 0.91
- **[P2] Return after the gateway handshake:** `apps/linux/src/gateway.rs:36-42`
  - body: connect() awaits the WebSocket receive loop inline, so it does not return while the socket remains open. Any Tauri command wired to this helper would hang instead of responding to the frontend.
  - confidence: 0.88
- **[P2] Wire gateway commands to real connection state:** `apps/linux/src/commands.rs:22-25`
  - body: connect_gateway reports success without using GatewayConnection, the provided URL, or the API key. The UI can show a connected state even when no gateway connection or pairing occurred.
  - confidence: 0.86
- **[P2] Register the new app package graphs:** `apps/linux/package.json:10-22`
  - body: The PR adds npm and Rust dependency graphs for first-party apps but does not add root workspace registration, lockfiles, or repo validation/release wiring. That leaves these apps outside the normal reproducible install and check path.
  - confidence: 0.84
- **[P2] Set a restrictive CSP before shipping WebViews:** `apps/linux/tauri.conf.json:40-42`
  - body: Both Tauri configs set security.csp to null. A first-party desktop app that talks to the Gateway should ship a restrictive policy for app resources and expected HTTP/WebSocket origins.
  - confidence: 0.84
