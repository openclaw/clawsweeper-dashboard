## Summary

Keep open: current main and the latest release still lack a silent `registerCommand` result contract, and the open linked PR is the canonical implementation path for this issue.

## What This Changes

Request a first-class `registerCommand` result such as `suppressReply` or `handled` so a Telegram plugin command that sends through its own transport does not receive the empty-response fallback.

## Best Possible Solution

Define and document a narrow public command-result suppression contract, land https://github.com/openclaw/openclaw/pull/80928 with Telegram regression coverage and proof, then close this issue after merge.

## Reproduction Assessment

Yes. Current source shows undefined command results become `{}` and Telegram replaces non-renderable results with the empty-response fallback; existing Telegram coverage asserts that fallback path.

## Solution Assessment

Mostly yes. An explicit silent command-result flag is a narrow maintainable solution, but maintainers still need to settle the public SDK scope and land or reject the already-open canonical PR.
