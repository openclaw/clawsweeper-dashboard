## Summary

Current main still has the reported Windows installer behavior. The documented PowerShell pipe path still runs scripts/install.ps1, and the failure path for a fresh Windows image with no Node/package manager still returns false from Install-Node and reaches hard exit 1 branches with no pause, Read-Host, Wait-BeforeExit, or wrapper around Main. The related fix PRs were closed unmerged, and the WebClaw comment is only a third-party workaround, not a core installer fix.

## Best Possible Solution

Keep #38054 open as the canonical Windows installer UX bug. The best fix belongs in scripts/install.ps1: replace or wrap the hard failure exits so interactive Windows installer sessions, especially the documented piped `iwr | iex` path, leave the error visible with a guarded prompt while still avoiding hangs in CI/noninteractive contexts. Add focused regression coverage where feasible and record Windows manual verification if full automation is impractical.
