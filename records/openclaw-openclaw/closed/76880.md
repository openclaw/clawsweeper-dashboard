## Summary

Keep open: current main still lacks the response-header normalization needed for this Google Chat regression. The issue now has an open implementation path in #76791, whose latest head commit adds the response interceptor, but that PR is still unmerged so the bug is not implemented on main.

## What This Changes

Google Chat inbound webhook auth fails after the earlier request-header fix because certificate response headers can still reach google-auth-library as plain objects without `Headers.get()`.

## Best Possible Solution

Land or rebase the response-header normalization in the Google Chat plugin transport with focused regression tests, then close this issue once that fix reaches main.

## Reproduction Assessment

Yes at source level: `app-url` verification calls `OAuth2Client.verifyIdToken` through the shared Gaxios transport, google-auth-library 10.6.2 reads `res?.headers.get('cache-control')`, and current main only normalizes request headers. I did not run a live Google Chat webhook in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is a Google Chat plugin response interceptor that restores `Headers` on Gaxios responses, with regression coverage. Core channel or gateway behavior does not need to change.
