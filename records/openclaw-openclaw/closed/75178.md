## Summary

Close as duplicate/superseded. The report matches the same group visible-reply behavior already handled in #74876 and the earlier WhatsApp duplicate #67394: current main keeps normal group/channel final replies private by default, leaves visible output to the message tool unless `messages.groupChat.visibleReplies` is set to `automatic`, and now emits an operator warning instead of making the behavior silent.

## What This Changes

The issue requests that generated WhatsApp group auto-replies be visibly delivered to the group the same way direct-chat replies and manual `message` tool sends are delivered.

## Best Possible Solution

Keep the current group/channel safety default and warning. Operators who want legacy group auto-posting should set `messages.groupChat.visibleReplies: "automatic"`; reports of provider failure after that setting or after explicit message-tool sends should be filed separately with delivery logs.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a group/channel turn with no visible-reply override resolves to `message_tool_only`, typing can still start, and the dispatch test confirms the final reply is not sent to the channel.

## Solution Assessment

Yes. Consolidating this under the already handled group visible-reply policy is the narrowest maintainable path; this is not evidence of a WhatsApp group send failure while the documented default intentionally suppresses normal final replies.
