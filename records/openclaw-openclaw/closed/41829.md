## Summary

Current `main` already handles the reported macOS LaunchAgent state correctly: installed-but-not-loaded services are restarted instead of being treated as missing, the launchd adapter explicitly clears persisted disabled state before bootstrap/restart, and there is dedicated launchd integration coverage for stop-then-start and stop-then-restart without reinstall.

## Evidence

## Close Comment

Closing this as implemented on main after a Codex review. Current OpenClaw already treats an installed-but-not-loaded LaunchAgent as restartable instead of missing (`src/daemon/service.ts:114`), and the macOS launchd adapter now explicitly clears persisted disabled state before bootstrap/restart (`src/daemon/launchd.ts:188`, `src/daemon/launchd.ts:656`). It also retries by bootstrapping the plist when `kickstart` sees a not-loaded service rather than requiring reinstall (`src/daemon/launchd.ts:671`). There is dedicated launchd integration coverage for the exact stop-then-start and stop-then-restart flows without reinstall (`src/daemon/launchd.integration.e2e.test.ts:190`). I also checked the latest release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5` and these daemon files are unchanged from current main, so this behavior is already shipped. If you still hit this on a newer build, please open a fresh issue with the exact OpenClaw version, `openclaw gateway status --deep`, and `launchctl print gui/$UID/ai.openclaw.gateway` output.
