## Summary

Keep this PR open: it fixes the right CLI/Gateway delivery boundary, but latest head still drops some literal direct-delivery messages during outbound payload planning, so one merge-blocking correctness issue remains.

## What This Changes

The branch adds a `directDelivery` CLI/Gateway protocol path so `openclaw agent --deliver` sends user-authored CLI text directly, while updating docs, Swift protocol models, delivery normalization, tests, and changelog.

## Best Possible Solution

Preserve every user-authored direct-delivery message before outbound planning, keep silent/status suppression only on agent-composed replies, and land after maintainer approval of the CLI semantics.

## Reproduction Assessment

Yes. Source inspection shows current main routes `--deliver` through the normal agent path, and PR head still has a clear source-level failing case where direct text matching relay-status placeholders is suppressed before outbound send.

## Solution Assessment

No, not yet. The direct-delivery shortcut is the right boundary, but the payload layer needs a literal direct-send mode instead of only preserving exact silent-token text.
