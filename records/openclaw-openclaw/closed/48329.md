## Summary

Close as superseded. PR #48329 no longer contains the awaited cleanup described in its body; its current head only adds `closeAllConnections()` to `stopFeishuMonitorState()` and still misses the abort-driven webhook cleanup path. The same remaining Feishu cleanup bug is already tracked by open issue #48183 and the more complete open PR #48588, which updates both `monitor.state.ts` and `monitor.transport.ts`.

## Best Possible Solution

Close #48329 as superseded by #48588, and keep #48183/#48588 as the canonical path. The eventual fix should use a shared awaited HTTP-server close helper with `server.close()`, `closeAllConnections()`, bounded timeout/error handling, apply it to both `stopFeishuMonitorState()` and `monitorWebhook()` abort cleanup, and update callers/tests to await cleanup where needed.
