## What This Changes

The issue requests hardening post-update.run self-restart so package-manager updates exit or respawn instead of dynamic-importing swapped bundle chunks.

## Best Possible Solution

Keep the shipped fresh-process update restart behavior and use the documented manual kickstart only for already-stale pre-fix processes.

## Reproduction Assessment

No current-main failing reproduction remains: the report gives concrete old-version steps, but current source routes package-swap update.run through immediate restart and fresh process respawn or supervisor exit. The historical stale dist chunk failure is plausible, and the current code is aimed directly at that path.

## Solution Assessment

Yes, the current solution is the narrow maintainable fix: package-swap updates bypass deferral/cooldown and replace the process module graph instead of re-importing from a rewritten dist tree. It matches the proposed direction without adding a new user-facing config knob.
