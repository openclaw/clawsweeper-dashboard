## Summary

Current main already implements the Feishu file-typed iOS video recovery through the merged replacement PR #73986, with source, tests, and changelog coverage. This PR should close rather than merge because its latest branch is obsolete and diverges from the landed `type=media` fallback.

## What This Changes

The PR branch adds Feishu message-resource retry logic, a video-file heuristic, parsed-content plumbing, and media tests for iOS video downloads that arrive as `file` messages.

## Best Possible Solution

Keep the current-main #73986 implementation as the canonical Feishu fix and close this PR as obsolete. Any further Feishu API tuning should happen in a fresh, narrow follow-up against the landed fallback.

## Reproduction Assessment

Yes. The linked bug has concrete reproduction steps for iOS photo-library videos arriving as Feishu `file` messages and failing with HTTP 502, and current main has focused unit coverage that simulates the HTTP 502 and verifies the `type=media` recovery path.

## Solution Assessment

No for this PR branch: its latest patch switched to same-type retry/backoff and still has retry-shape gaps. Yes for current main: the landed replacement is the narrower maintainable fix because it retries only `type=file` HTTP 502 as `media` and preserves the original error on fallback failure.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Retry the failed file download as media:** `extensions/feishu/src/media.ts:246`
  - body: The retry loop keeps calling `messageResource.get` with the original `type`, so a `type=file` 502 retries the same endpoint that the report says consistently fails for longer iOS photo-library videos. This leaves the requested media-endpoint fallback unimplemented on the PR branch.
  - confidence: 0.88
- **[P2] Handle object-shaped HTTP 502 errors:** `extensions/feishu/src/media.ts:291`
  - body: `isFeishuServerError` returns false unless the thrown value is an `Error`, so Axios/SDK failures shaped like `{ response: { status: 502 } }` skip the retry path even though the rest of the Feishu codebase already treats that as a common HTTP status shape.
  - confidence: 0.82
