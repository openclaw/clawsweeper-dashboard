## Summary

Current main already resolves isolated cron agentTurn runs from the target agent's configured model when payload.model is omitted, and that behavior is covered by regression tests and already shipped in the latest release tree.

## Evidence

- **Target agent config is merged before model resolution:** runCronIsolatedAgentTurn resolves the requested agent, builds agentConfigOverride from that agent entry, and folds it into agents.defaults as cfgWithAgentDefaults before model selection runs.
  - file: [src/cron/isolated-agent/run.ts:435](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/cron/isolated-agent/run.ts#L435)
  - sha: [0e7250f37bf3](https://github.com/openclaw/openclaw/commit/0e7250f37bf347c72790abf2a6692d5ec84e5ea1)
- **Cron model selection starts from merged agent defaults:** resolveCronModelSelection seeds provider/model from resolveConfiguredModelRef(cfgWithAgentDefaults); when payload.model is absent, only Gmail and stored session overrides can replace that base, so the target agent model is the default starting point rather than the global default.
  - file: [src/cron/isolated-agent/model-selection.ts:48](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/src/cron/isolated-agent/model-selection.ts#L48)
