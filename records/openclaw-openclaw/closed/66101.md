## Summary

Keep open: current main does not include this host-level self-inspection exception, and the PR is security-sensitive but not mergeable as written because the new boundary can be bypassed by pipe/Linux path forms and mutating doctor/memory flags.

## What This Changes

The PR adds a main-agent exception for selected host-level OpenClaw status, doctor, cron, tasks, and memory commands under gateway full-trust exec, with focused path tests.

## Best Possible Solution

If maintainers want this exception, implement it with segment-parser-based OpenClaw detection, platform-independent path recognition, a closed read-only argv policy, and regression coverage for pipes, Linux paths, wrapper invocations, doctor repair/token flags, and memory status repair/reindex flags.

## Reproduction Assessment

Yes. Static reproduction is high confidence from the PR diff and current CLI contracts: `echo ok | openclaw update`, `/home/user/.npm-global/bin/openclaw update`, `openclaw doctor --repair`, `openclaw doctor --generate-gateway-token`, and `openclaw memory status --fix` exercise the bypasses.

## Solution Assessment

No. The product direction may be acceptable after maintainer review, but this implementation is not the narrowest maintainable security boundary because enforcement starts with an incomplete regex and the argv allowlist accepts mutating CLI modes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Detect OpenClaw invocations after pipes:** `src/agents/bash-tools.exec.ts:178-181`
  - body: `assertMainHostLevelOpenClawBoundary` only runs when the trigger regex matches, but the regex does not treat a single `|` as a command boundary. A full-trust main command such as `echo ok | openclaw update` can skip the new guard and run a mutating OpenClaw command.
  - confidence: 0.92
- **[P1] Match host OpenClaw paths across platforms:** `src/agents/bash-tools.exec.ts:179-185`
  - body: The absolute-path patterns are hard-coded to `/Users/...`, so Linux paths such as `/home/user/.npm-global/bin/openclaw` or `/home/user/.openclaw/scripts/ops-host-openclaw.sh` do not trigger the boundary and can bypass the intended restriction.
  - confidence: 0.9
- **[P1] Reject mutating doctor modes:** `src/agents/bash-tools.exec.ts:219-222`
  - body: The doctor allowlist blocks only the exact `--fix` token. Current CLI registration also exposes `--repair`, `--force`, and `--generate-gateway-token`, so mutating doctor operations can pass this exception.
  - confidence: 0.95
- **[P1] Reject mutating memory status flags:** `src/agents/bash-tools.exec.ts:228-230`
  - body: The PR allows `openclaw memory status` with arbitrary trailing args, but the memory CLI treats `--fix` as repair work and `--index` as reindexing. Those flags should not be accepted by a read-only self-inspection exception.
  - confidence: 0.95
