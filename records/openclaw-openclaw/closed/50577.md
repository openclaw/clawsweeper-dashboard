## Summary

Current `main` does not implement the slash-containing bare model fix from #50509, and this PR still needs maintainer judgment because it bundles an unrelated streaming-compat change and introduces a provider-guessing path that conflicts with existing unique-inference behavior.

## Evidence

- **`sessions.patch` still does a single resolution attempt and returns `model not allowed` immediately:** Current `main` loads the catalog, calls `resolveAllowedModelRef` once, and returns `invalid(resolved.error)` on failure. There is no provider-prefix retry or unique-provider fallback in this path.
  - file: [src/gateway/sessions-patch.ts:412](https://github.com/openclaw/openclaw/blob/42514156e03c53dec6be6c581069d8c5b2d4cd42/src/gateway/sessions-patch.ts#L412)

  - sha: [42514156e03c](https://github.com/openclaw/openclaw/commit/42514156e03c53dec6be6c581069d8c5b2d4cd42)
- **Provider inference is intentionally limited to inputs without `/`:** `resolveAllowedModelRefFromAliasIndex` only runs `inferUniqueProviderFromConfiguredModels(...)` when `!trimmed.includes("/")`. Slash-containing inputs are therefore treated as explicit provider/model refs today, which leaves the `deepseek-ai/...` case unresolved on `main`.
