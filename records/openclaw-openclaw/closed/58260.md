## Summary

Current `main` already routes Discord REST traffic through the configured proxy path by passing `discordProxyFetch` into monitor startup and replacing Carbon's default REST client with a proxy-aware request client. The same implementation is present in release `v2026.4.23`.

## Evidence

- **Proxy fetch is threaded into Discord monitor startup:** `createDiscordMonitorClient` is called with `proxyFetch: discordProxyFetch`, so the resolved Discord proxy fetch is available at client construction time.
  - file: [extensions/discord/src/monitor/provider.ts:937](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/extensions/discord/src/monitor/provider.ts#L937)
  - command: `nl -ba extensions/discord/src/monitor/provider.ts | sed -n '930,955p'`
  - sha: [90877e0d4274](https://github.com/openclaw/openclaw/commit/90877e0d4274c53fd91bf5787265c4d6889dc9a7)
- **Startup swaps Carbon REST client for proxy-aware client:** When `params.proxyFetch` is present, monitor startup replaces `client.rest` with `createDiscordRequestClient(params.token, { fetch: params.proxyFetch })`, avoiding Carbon's default global fetch path for REST calls.
  - file: [extensions/discord/src/monitor/provider.startup.ts:180](https://github.com/openclaw/openclaw/blob/90877e0d4274c53fd91bf5787265c4d6889dc9a7/extensions/discord/src/monitor/provider.startup.ts#L180)
