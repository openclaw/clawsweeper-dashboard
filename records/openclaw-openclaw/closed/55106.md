## Summary

Current main already contains explicit WhatsApp guardrails that stop pairing/setup messages from being re-sent through outbound echo or historical catch-up DM paths, and those behaviors are covered by regression tests.

## Evidence

- **Gateway-sent WhatsApp echoes are dropped before inbound processing:** The inbound monitor now skips `fromMe` messages whose IDs match recent gateway-sent outbound messages, preventing the bot's own outbound traffic from re-entering the pairing path.
  - file: [extensions/whatsapp/src/inbound/monitor.ts:365](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/whatsapp/src/inbound/monitor.ts#L365)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Outbound DMs no longer trigger pairing replies:** Direct-message access control returns early for outbound `fromMe` DMs and does not create or send a pairing challenge.
  - file: [extensions/whatsapp/src/inbound/access-control.ts:113](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/whatsapp/src/inbound/access-control.ts#L113)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Historical catch-up DMs no longer send pairing codes on reconnect:** The same access-control path suppresses pairing replies when a DM timestamp is older than the reconnect grace window, blocking replayed history from sending fresh pairing/setup text.
  - file: [extensions/whatsapp/src/inbound/access-control.ts:131](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/whatsapp/src/inbound/access-control.ts#L131)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
- **Regression tests cover the historical-DM suppression:** Unit tests assert that historical DMs on connect do not upsert a pairing request and do not send a pairing reply, while live DMs still do.
  - file: [extensions/whatsapp/src/inbound/access-control.test.ts:82](https://github.com/openclaw/openclaw/blob/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534/extensions/whatsapp/src/inbound/access-control.test.ts#L82)
  - sha: [34896839ba22](https://github.com/openclaw/openclaw/commit/34896839ba22f3a71a3d5a8fb4ca5f4c6989a534)
