## Summary

Current main already fixes the reported Feishu DM dispatch failure by passing a full channel runtime with `inbound.run` through gateway channel startup; the latest tagged release still had the startup-only runtime path, so this is fixed on main but not yet in a release.

## What This Changes

Report that Feishu WebSocket DMs receive messages but fail to dispatch to the agent with `Cannot read properties of undefined (reading 'run')` under a single configured account.

## Best Possible Solution

Keep the current gateway-level full channel runtime handoff and ship it in the next release rather than adding Feishu-only defensive checks.

## Reproduction Assessment

Yes, by source for the affected release path: `v2026.5.28` could hand bundled channels only a startup runtime without `inbound.run`, matching the reported TypeError. I did not run a live Feishu account, and current main no longer follows that path.

## Solution Assessment

Yes. The current fix is at the shared gateway runtime handoff boundary, which is narrower and more maintainable than adding Feishu-only guards around a missing runtime method.
