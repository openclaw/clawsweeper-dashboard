## Summary

Close: current `main` and shipped `v2026.4.23` already provide dedicated filesystem `write` and `apply_patch` tools, so the requested way to bypass shell escaping already exists without going through `exec`.

## Evidence

- **Built-in filesystem tools are documented:** The tools docs list `read` / `write` / `edit` as built-in file I/O tools and `apply_patch` as a built-in patch tool available without installing plugins.

  - file: [src/agents/tool-catalog.ts:62](https://github.com/openclaw/openclaw/blob/11804a484ded89374afa55020813cfa265aa143a/src/agents/tool-catalog.ts#L62)
  - sha: [11804a484ded](https://github.com/openclaw/openclaw/commit/11804a484ded89374afa55020813cfa265aa143a)
