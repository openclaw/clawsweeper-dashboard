## Summary

Current main and the latest release already carry state-dir `.env` entries into macOS LaunchAgent gateway service env files, which covers the reported `GITHUB_PAT` placeholder path without requiring MCP header SecretRef objects.

## What This Changes

The issue requests that env vars referenced by `${VAR}` placeholders in `mcp.servers.<name>.headers.*` survive `openclaw daemon install --force` by being written into the gateway service env.

## Best Possible Solution

Keep the shipped durable service-env implementation and ask for a fresh redacted report only if this still reproduces on `v2026.5.19` or newer.

## Reproduction Assessment

No current-main failing path was established; source and shipped tests show state-dir `.env` values are retained for macOS LaunchAgent service env files. The issue body is a strong reproduction for older `2026.5.12` behavior, but the checked current/release code no longer matches that failure.

## Solution Assessment

Yes. Carrying durable state-dir `.env` values into the daemon install service env solves the documented MCP header placeholder path with less schema expansion than adding SecretRef objects to MCP headers.
