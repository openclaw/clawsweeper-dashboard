## Summary

Keep open: current main still has a source-proven path where Active Memory accepts a non-interactive dreaming narrative session as a webchat direct session and can dispatch recall work for it.

## What This Changes

The issue asks OpenClaw to prevent Active Memory from running on memory-core dreaming/cron session keys, preferably by tightening the existing eligibility gate.

## Best Possible Solution

Tighten Active Memory's existing eligibility/chat-type classification so only recognized interactive chat session-key shapes can pass, with a regression test for memory-core dreaming narrative keys.

## Reproduction Assessment

Yes, at source level. Current main shows dreaming narrative keys can be canonicalized into agent-scoped webchat runs, `runAgentAttempt` marks them as `trigger: "user"`, and Active Memory maps unknown webchat keys to `direct`.

## Solution Assessment

Yes. The narrowest maintainable fix is to reject non-interactive synthetic session keys in the existing Active Memory gate; adding an operator `excludeSessionKeyPatterns` config would be broader than this bug requires.
