## Summary

Current main advertises and handles macOS node camera.snap and camera.clip when the macOS Allow Camera setting is enabled. The issue was reported against v2026.3.31 and has no comments or timeline entries to preserve beyond the original reproduction.

## Evidence

- **macOS node command advertisement includes snap and clip:** MacNodeModeCoordinator builds the node command list from current capabilities and, when the camera capability is present, appends camera.list, camera.snap, and camera.clip. The same coordinator disconnects/reconnects when the Allow Camera setting changes, so the gateway receives an updated allowlist.
