## Summary

Yuanbao is documented and cataloged as an external/community channel plugin, not a bundled OpenClaw core plugin. Current main already has the generic `channelConfigs` manifest contract and the doctor diagnostic that reports missing metadata for non-bundled channel plugins. The missing `channelConfigs` field should be fixed and published in the external `openclaw-plugin-yuanbao` package/ClawHub listing rather than in this repository.

## Best Possible Solution

Keep OpenClaw core's generic manifest contract and doctor warning as-is. The best fix is for the Yuanbao plugin maintainers to publish a new `openclaw-plugin-yuanbao` release with `openclaw.plugin.json#channelConfigs.yuanbao` metadata, then update its ClawHub/npm package if needed.
