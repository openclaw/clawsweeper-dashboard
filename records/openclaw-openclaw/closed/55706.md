## Summary

Current `main` already fixes the restart-loop symptom this PR targets by deduping internal config writes in the gateway reloader, including the startup write path that previously re-triggered chokidar/SIGUSR1 loops. The PR's exact `skip no-op write` change is not on `main`, but the reported user-facing failure mode is already addressed well enough by a later implementation.
