## Summary

Close as ClawHub/plugin scope. Current main and the latest release intentionally do not execute Claude hooks automatically; Codex Stop is already represented through OpenClaw's before_agent_finalize path, while UserPromptSubmit-style prompt behavior can be implemented as an explicitly installed plugin over existing prompt/context hook surfaces.

## What This Changes

Request core support for Claude Code SDK-compatible UserPromptSubmit and Stop hooks from user and workspace .claude/settings.json during OpenClaw-mediated agent turns.

## Best Possible Solution

Keep core's no-automatic-Claude-hooks policy and move Claude hook compatibility into an explicitly installed and trusted ClawHub plugin or hook-pack; open a narrower core API issue only if that plugin hits a missing payload or decision contract.

## Reproduction Assessment

Yes, by source inspection for the missing core behavior: current main does not emit or accept a UserPromptSubmit native relay event, and docs say Claude hooks are not executed automatically. I did not run the shell-log repro because the source and shipped docs already establish the behavior.

## Solution Assessment

No, not as automatic core execution of .claude/settings.json hooks. The safer repository-level solution is an opt-in ClawHub/native plugin using OpenClaw prompt/context/finalize hooks, with a narrow core API request only if that plugin cannot express a required contract.
