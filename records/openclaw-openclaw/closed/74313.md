## Summary

Keep open. Current main still has the doctor/model-status mismatch, and #74438 is the open unmerged closing PR for this bug, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue asks `openclaw doctor` to stop presenting the live `agents/main` compatibility store as removable orphan state when the configured default agent is not `main`.

## Best Possible Solution

Land #74438 or an equivalent narrow patch that aligns omitted-agent `models status` and `--probe` with the configured default agent while exempting only the live `main` compatibility store from removable-orphan guidance.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `stateDir/agents/main/agent` on disk, and current main still classifies `main` as orphaned while omitted `models status` reads the compatibility auth store.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable path because it aligns command state resolution with the configured-default contract and limits the doctor change instead of suppressing all orphan warnings.
