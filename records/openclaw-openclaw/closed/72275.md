## Summary

Current main already keeps Telegram topic-bound sessions on the agent-specific model through compaction. The Telegram route preserves the topic agent in the session key, reply startup resolves defaults through that agent's configured model, and the compaction runtime receives and uses the active run provider/model unless an explicit compaction-only override is configured. The same relevant compaction model code is present in v2026.4.24, while the report was confirmed on 2026.4.23.

## Best Possible Solution

Close this report as fixed by the shipped v2026.4.24/main behavior. Keep the current agent-scoped model resolution and compaction runtime model propagation; if a user can reproduce on v2026.4.24 or newer, track that as a fresh bug with compaction logs and session-store model fields before and after compaction.
