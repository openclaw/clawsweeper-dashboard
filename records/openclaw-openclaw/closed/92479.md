## Summary

Keep open: current main and v2026.6.10 still lack a provider-owned OpenCode Zen model catalog, while https://github.com/openclaw/openclaw/pull/92495 is the open candidate fix for this regression.

## What This Changes

The issue asks OpenClaw to restore bundled OpenCode Zen model catalog/discovery support so `opencode/...` models work without manual `models.providers.opencode.models[]` registration.

## Best Possible Solution

Review and land, narrow, or explicitly replace the linked provider-boundary fix that restores Zen catalog rows, dynamic resolution, discovery, manifest metadata, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: choose an `opencode/<model>` ref beyond the default and current main has neither Zen catalog rows nor a dynamic resolver to satisfy it; the issue discussion also has a fresh v2026.6.10 confirmation.

## Solution Assessment

Yes on owner-boundary shape, but not fully settled on product policy. Restoring Zen catalog/discovery inside `extensions/opencode` is the maintainable fix; seed scope and shared credential fallback should be accepted through the linked PR review.
