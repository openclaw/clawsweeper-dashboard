## Summary

Current main already addresses the user-visible Telegram group-session display problem through gateway-provided group display names and UI selector disambiguation, and the same behavior is present in v2026.4.24. The unmerged PR also bundles unrelated gateway and postinstall changes that were called out in review, so it should not be merged as-is.

## Best Possible Solution

Close this PR as already handled on main. Keep the shipped gateway/UI session-display implementation; if maintainers want the exact prettier key-only fallback text from the PR, that should be a new small UI-only PR with focused tests and without the unrelated gateway/postinstall changes.
