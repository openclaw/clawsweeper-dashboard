The report is recent, concrete, and matches a risky release window: v2026.4.24 introduced the Pi package bump the reporter suspects, v2026.4.26 shipped several adjacent gateway/runtime-deps fixes, and current main has further unshipped gateway probe/channel-startup changes. ## What This Changes

The issue reports that Linux npm-global installs of v2026.4.24 and v2026.4.26 start the gateway HTTP listener but time out the 10s connectivity probe and never connect Telegram, WhatsApp, or Discord, while v2026.4.22 works.

## Best Possible Solution

Reproduce or instrument the failing Linux npm-global/systemd upgrade path, identify whether the blocking point is probe handshake timing, sidecar/channel startup, plugin-runtime-deps staging, or a Pi runner initialization hang, then land the narrow fix in the owning module with a regression test and a changelog entry if user-visible.
