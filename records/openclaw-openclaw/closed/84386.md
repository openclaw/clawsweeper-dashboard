## What This Changes

The issue reports that Codex OAuth relogin plus explicit auth order can still leave stale `lastGood.openai-codex` selecting `openai-codex:default` over the fresh named profile.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84403 or a narrower equivalent that makes explicit Codex auth order authoritative in the affected active-profile path, then close this issue after the fix is merged and verified.

## Reproduction Assessment

Yes at source level: with order `[fresh, default]` and `lastGood.openai-codex = default`, the current Codex account helper can return `default` before falling back to the first ordered profile. I did not perform a live OAuth round trip in this read-only review.

## Solution Assessment

Yes for the issue direction: the narrow maintainable fix is to honor explicit Codex order over stale `lastGood` in this active-profile path, or clear conflicting `lastGood` when setting the order. The existing broader primary-first policy request belongs to https://github.com/openclaw/openclaw/issues/83496, not this bug fix.
