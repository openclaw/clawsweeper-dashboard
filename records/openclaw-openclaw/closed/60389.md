## Summary

Current main implements the stale ACP binding cleanup requested here. Telegram restart routing now drops persisted ACP bindings whose target session entry is missing or terminal/error, and shared ACP dispatch cleanup unbinds stale bound conversations on missing-metadata or Resource not found init failures, covering the Discord follow-up as well.

## Best Possible Solution

Keep the current main implementation: remove stale conversation bindings during channel startup where possible and unbind target-session routes on stale ACP init failures, so dead ACP sessions cannot keep hijacking unrelated chat traffic. Re-spawn and re-bind the ACP session when the user still wants that conversation bound.
