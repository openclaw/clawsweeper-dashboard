## Summary

Close as implemented on `main`: the current codebase already supersedes this PR by isolating bundled runtime-dependency installs to an OpenClaw-owned cache under the install root instead of `~/.npm` or shared `/tmp`, and it already preserves explicit channel/plugin disablement during bundled runtime-dependency repair.

## Evidence

- **Current main uses an OpenClaw-owned cache inside the runtime-deps install root:** `installBundledRuntimeDeps()` now calls `createBundledRuntimeDepsInstallEnv()` with `cacheDir: path.join(installExecutionRoot, ".openclaw-npm-cache")`, which avoids the `~/.npm` `ProtectHome=true` failure mode and is more isolated than the PR's bare `os.tmpdir()` proposal.
