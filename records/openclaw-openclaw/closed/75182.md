## Summary

Keep open. Current main still accepts a same-line serialized JSON suffix as part of a slash-prefixed local MEDIA path, the exact fix PR #75202 is closed and unmerged, and adjacent PR #68338 addresses multiline MEDIA directives rather than this suffix-parsing bug.

## What This Changes

Harden shared MEDIA directive parsing so local file paths do not absorb trailing serialized JSON before channel delivery.

## Best Possible Solution

Fix the shared parser contract in src/media/parse.ts with focused regression coverage in src/media/parse.test.ts; channel-level checks can remain defense in depth but should not be the primary fix.

## Reproduction Assessment

Yes. Source inspection provides a high-confidence reproduction path: the regex captures the full same-line payload, cleanCandidate leaves the embedded JSON suffix intact, and isLikelyLocalPath accepts the resulting slash-prefixed candidate as media.

## Solution Assessment

Yes. A narrow parser-level sanitize-or-reject fix is the maintainable solution because it prevents polluted mediaUrls from reaching every outbound channel, including external channel plugins.
