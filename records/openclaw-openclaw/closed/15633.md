## Summary

Current `main` already implements Mattermost media send/receive plumbing strongly enough to cover this feature request: outbound replies upload media as Mattermost file attachments, inbound posts fetch attached files and classify audio, and the resulting media context flows into the shared audio-transcription pipeline.
