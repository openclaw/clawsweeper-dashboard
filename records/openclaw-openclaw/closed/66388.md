## Summary

Current `main` no longer uses the old heartbeat-only managed dreaming path. `memory-core` now creates an isolated cron `agentTurn`, accepts direct `cron` triggers, and has regression tests covering the missing-cron warning and cron-trigger execution, so the failure mode described in #66388 is implemented away on `main` even though it is not in the latest tagged release yet.

## Evidence

- **Managed dreaming cron is now an isolated cron turn:** On current `main`, `buildManagedDreamingCronJob` creates the managed dreaming job as `sessionTarget: "isolated"` with `payload.kind: "agentTurn"`, `lightContext: true`, and `delivery.mode: "none"` instead of the older main-session system-event wake.
