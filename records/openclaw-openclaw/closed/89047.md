## Summary

## What This Changes

The PR removes per-call `sessions_spawn` timeout fields from the model-facing schema, rejects stale timeout arguments, routes ACP spawns through configured subagent timeouts, and updates docs, tests, and Codex prompt snapshots.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes, with the compatibility caveat. Because Codex consumes the dynamic tool `input_schema` directly as model-visible tool parameters, removing the fields at `createSessionsSpawnToolSchema` while keeping the internal configured timeout path is the narrowest maintainable fix.
