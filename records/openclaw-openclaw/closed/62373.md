## Summary

Current main already implements the requested separation: `openai-codex/*` is documented and tested as the Codex OAuth PI route, the Codex app-server harness is a separate opt-in runtime, and the embedded PI runner registers OpenClaw-managed local tools for execution.

## Evidence

- **Docs separate Codex OAuth from native Codex app-server:** The OpenAI provider docs state that `openai-codex/*` is the Codex subscription route through PI, while native Codex app-server behavior requires `openai/*` plus `agents.defaults.embeddedHarness.runtime: "codex"`. The same docs say selecting `openai-codex/*` does not enable the bundled Codex app-server plugin.
