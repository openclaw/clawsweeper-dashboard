## Summary

Current main already handles the reported WSL2/systemd user-bus failure: the exact `No medium found` detail is classified as user-bus unavailable, systemd calls can fall back to machine-scoped user control, read-only status paths degrade instead of failing, and gateway status skips restart-health/service diagnostics when the gateway RPC probe is already OK.

## Best Possible Solution

Close this issue and keep the shipped generalized systemd user-bus handling. Any new WSL2 report should include current-version logs if `No medium found` still appears despite the gateway RPC probe succeeding.
