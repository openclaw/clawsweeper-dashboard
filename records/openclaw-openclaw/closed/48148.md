## Summary

Current `main` already computes the status cache percentage from prompt-side tokens (`input + cacheRead + cacheWrite`) instead of `input + output`, so the displayed cache hit rate is bounded correctly for the `openclaw status` paths implicated by the issue.

## Evidence

- **Detailed status path uses prompt-side denominator:** `formatCacheLine()` computes `totalInput = cacheRead + cacheWrite + input` and `hitRate = Math.round((cacheRead / totalInput) * 100)`, which cannot exceed 100% when values are non-negative.
