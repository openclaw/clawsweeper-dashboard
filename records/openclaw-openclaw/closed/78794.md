## Summary

Close as implemented: the gateway bootstrap hang was fixed by deferring startup plugin runtime loading out of the pre-listen `plugins.bootstrap` path, and release `v2026.5.6` descends from the fix commit and contains the fixed source tree.

## What This Changes

The issue reports that the `plugins.bootstrap` gateway hang also affects the 2026.4.25 Docker image and asks for a released build carrying the existing main-branch fix.

## Best Possible Solution

Keep the shipped `v2026.5.6` implementation and direct users on 2026.4.24 through 2026.4.29 to upgrade to a release at or after `v2026.5.6`.

## Reproduction Assessment

Yes for the affected released images: the report gives a deterministic Docker restart path for 2026.4.25 with the startup trace stopping before `plugins.bootstrap` returns, matching the related 2026.4.26 report. Current main and `v2026.5.6` no longer use that pre-listen plugin-runtime load path.

## Solution Assessment

Yes. Deferring plugin runtime/dependency loading until after HTTP bind is the narrow maintainable fix for the no-listener hang while preserving sidecar ordering through post-attach startup tests.
