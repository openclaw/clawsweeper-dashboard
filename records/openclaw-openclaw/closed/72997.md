## Summary

Keep open. Current main has automatic config-driven channel hot reload and separate start/stop/logout RPCs, but it still lacks the requested one-step manual CLI/Gateway reload surface for a single channel.

## What This Changes

Request a targeted operator command or Gateway action to disconnect and reconnect one named chat channel without restarting the Gateway or other channels.

## Best Possible Solution

Add a generic admin-scoped `channels.reload` RPC and `openclaw channels reload --channel <name> [--account <id>]` that reuses the existing channel stop/start lifecycle, active-work deferral semantics, docs, protocol schemas, and regression coverage.

## Reproduction Assessment

Yes, for the feature gap. Source and docs inspection on current main shows automatic config hot reload plus start/stop/logout surfaces, but no one-step manual channel reload CLI command or Gateway RPC.

## Solution Assessment

Yes. A generic reload operation layered over the existing channel lifecycle is the narrow maintainable direction; provider-specific recovery commands or full Gateway restarts would be broader than the request.
