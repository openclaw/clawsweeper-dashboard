## Summary

Current `main` already handles OpenAI-compatible `usage` payloads with `prompt_tokens` / `completion_tokens`, persists the resulting prompt-context snapshot into the session store, and the TUI status bar only falls back to `?` when that snapshot is absent. The fix appears to be present on `main` after the latest tagged release, so this issue is implemented on `main` but not yet tied to a shipped release from the provided context.

## Evidence

- **Usage normalization now accepts the reported alias shape:** `normalizeUsage` reads `prompt_tokens`, `completion_tokens`, and `total_tokens`, matching the exact payload shape from the issue and confirming comment.
  - file: [src/agents/usage.ts:114](https://github.com/openclaw/openclaw/blob/4f4288e3b5be2e55abfce5213f0bcf182a252f5e/src/agents/usage.ts#L114)
