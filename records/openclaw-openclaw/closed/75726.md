## Label Justifications

- `P1`: The report covered a real user-facing channel workflow leaking internal runtime context into assistant output across Discord, Slack, and Telegram surfaces.
- `impact:security`: The issue is about internal runtime-generated details that are explicitly marked private reaching user-visible channels.
- `impact:session-state`: The affected text is generated runtime context tied to session/message context and echoed back through assistant output.

## Summary

Current main and the shipped v2026.5.2 release contain the modern runtime-context preface sanitizer, and the corroborating issue comment validated the release against the captured leak corpus.

## What This Changes

The issue requests that OpenClaw strip the modern runtime-context wrapper before Discord, Slack, Telegram, and other user-facing channel delivery when a model echoes internal scaffolding.

## Best Possible Solution

Keep the shipped core sanitizer behavior and track malformed or typoed duplicate-wrapper variants separately only if new evidence shows they matter in normal outputs.

## Reproduction Assessment

Yes for affected versions: the pre-fix source lacked modern preface detection, and the issue plus corroborating Telegram corpus show production leaks. Current main no longer reproduces the central leak because the sanitizer strips that modern preface.

## Solution Assessment

Yes. The core sanitizer/delivery path is the right boundary, and the shipped line-based literal matcher avoids the over-stripping risk of a broad paragraph regex while matching the real corpus.
