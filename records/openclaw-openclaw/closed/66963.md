## Summary

Current `main` already contains the TDZ-safe poll-schema initialization referenced by the issue timeline, and the same implementation is present in shipped release `v2026.4.23`, so #66963 is fixed well enough to close.

## Evidence

- **Current poll param initialization is TDZ-safe:** `src/poll-params.ts` defines `SHARED_POLL_CREATION_PARAM_DEFS` first and derives `SHARED_POLL_CREATION_PARAM_NAMES` from that local constant before exporting the combined `POLL_CREATION_PARAM_DEFS`, which removes the startup-time read from a partially initialized combined export that would cause a TDZ `ReferenceError`.
  - file: [src/poll-params.ts:10](https://github.com/openclaw/openclaw/blob/cc0f3e0e40364023cc3a39227a411898ee68ce3b/src/poll-params.ts#L10)
