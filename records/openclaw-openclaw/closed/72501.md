Full review comments:

- **[P2] Record polls at the shared send boundary:** `extensions/telegram/src/action-runtime.ts:622-630`
  - body: This registry write only runs when the Telegram message-action handler sends the poll. The shared Gateway poll method still calls `outbound.sendPoll`, which calls `sendPollTelegram` directly, so a public Telegram poll sent through that supported path can return a `pollId` without ever creating a registry entry; the new `poll_answer` handler will then skip the vote as missing. Move the non-anonymous poll registry write into `sendPollTelegram` or the outbound adapter so every Telegram poll send path is covered.
  - confidence: 0.86
- **[P2] Use approved runtime state storage:** `extensions/telegram/src/poll-registry.ts:26-32`
  - body: This adds a new JSON registry under the OpenClaw state directory for live poll-to-session routing. Repository policy treats runtime registries/caches as SQLite/plugin-KV state, not new JSON sidecars, because these files become another state owner and upgrade surface. Please move this to the approved plugin/shared state store, or get an explicit maintainer exception with tests around the chosen lifecycle.
  - confidence: 0.78
- **[P3] Avoid shadowing the existing stateDir:** `extensions/telegram/src/action-runtime.test.ts:180`
  - body: The new outer `stateDir` variable shadows the existing local `const stateDir` in this test file, and the latest CI logs show both lint shards failing on `eslint(no-shadow)`. Rename either variable so `check-lint` and `check-additional-extension-bundled` can pass.
  - confidence: 0.99
