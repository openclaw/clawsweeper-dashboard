## Summary

Keep open. Current main still lets a raw `x-openclaw-session-key` win before agent-scoped session construction, while the public docs and related stable-session PR treat that header as a full routing override, so this needs a maintainer API-precedence decision rather than cleanup closure.

## What This Changes

Request that `/v1/chat/completions` keep routing to `model: "openclaw/<agentId>"` even when `x-openclaw-session-key` is supplied as the session identifier.

## Best Possible Solution

Define one documented OpenAI-compatible HTTP session precedence order, then update Chat Completions, Responses, docs, and focused regression tests consistently.

## Reproduction Assessment

Yes, at source level. Send `model: "openclaw/<agentId>"` with a raw `x-openclaw-session-key`; current main forwards the raw key and the command path resolves non-agent keys to the default agent.

## Solution Assessment

Unclear. Letting the model-selected agent always win matches this report, but it conflicts with the documented full-override semantics and the related stable-session PR's stated priority order.
