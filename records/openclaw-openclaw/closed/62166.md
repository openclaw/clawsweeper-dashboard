## Summary

Current `main` already implements this rollover fix more comprehensively than the PR by rebuilding fresh cron session entries from an allowlist, which drops `sessionFile` and other run-scoped fields whenever a new session is created.

## Evidence

- **Fresh-session rollover now sanitizes carried fields instead of spreading the old entry:** `sanitizeFreshCronSessionEntry` constructs a new `SessionEntry` from explicit allowlists and does not copy `sessionFile`. `resolveCronSession` uses that sanitized entry whenever `isNewSession` is true, covering both forced-new and stale-session rollovers.
