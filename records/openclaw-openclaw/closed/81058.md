## Summary

Keep open: current main still applies the reported OpenAI-compatible reasoning-strip default to SGLang’s generic provider path, no canonical duplicate or open fix PR was found, and the exception boundary needs provider-policy review.

## What This Changes

The issue reports empty user-visible responses from self-hosted OpenAI-compatible reasoning models after historical reasoning is stripped during replay.

## Best Possible Solution

Choose a provider-owned replay policy for SGLang or affected self-hosted reasoning models, with regression coverage that preserves both visible final text for Kimi-style reasoning and the prior Qwen/oMLX/vLLM stale-reasoning protection.

## Reproduction Assessment

No for high-confidence live reproduction. Source inspection gives a clear current-main path: SGLang uses openai-completions without a replay-policy opt-out, so the default strips historical reasoning before replay, but the reported empty response was not observed against a real SGLang/Kimi setup here.

## Solution Assessment

Unclear without provider-policy judgment. A targeted provider/model replay-policy opt-out is safer than a blanket revert, but the final fix must keep the prior OpenAI-compatible stale-reasoning protection that fixed the Qwen/oMLX/vLLM failures.
