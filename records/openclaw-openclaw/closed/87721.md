## Label Justifications

- `P1`: The report describes a current-release Discord/Codex channel workflow where valid agent replies are silently suppressed for an existing bound session.
- `impact:message-loss`: The affected behavior suppresses plugin-owned reply messages that should be delivered back to the Discord conversation.

## Summary

Keep open: current main still routes plugin-owned binding turns to the owning plugin under message-tool-only delivery, but a returned plugin reply goes through `sendBindingNotice()`, which exits on automatic source suppression and matches the reported silent Discord/Codex reply path.

## What This Changes

The issue asks OpenClaw to deliver Codex plugin-owned conversation binding replies in Discord group chats configured with message-tool-only visible replies while preserving normal final-reply suppression.

## Best Possible Solution

Adjust the plugin-binding reply delivery path so targeted `inbound_claim` replies are delivered under `message_tool_only`, while sendPolicy deny and ordinary automatic agent final suppression remain unchanged.

## Reproduction Assessment

Yes from source, not from a live run. A group or channel turn with `message_tool_only` sets `suppressAutomaticSourceDelivery`, then a handled plugin-owned `inbound_claim` reply is passed to `sendBindingNotice()` and returned before delivery.

## Solution Assessment

Yes. The narrow maintainable fix is to distinguish plugin-owned hook replies from normal automatic final replies in the binding notice path, without weakening sendPolicy deny or ordinary message-tool-only suppression.
