## Summary

Keep this PR open for maintainer review. Current main still lacks an explicit instruction-file precedence rule, but the proposed hierarchy is a trust-boundary decision because it ranks active skill instructions above workspace policy while current docs treat third-party skills as untrusted and the current prompt tells agents to pause when instructions conflict.

## What This Changes

The PR adds a one-line instruction-precedence directive to the Skills system-prompt section and adds a regression assertion for that directive.

## Best Possible Solution

Land a maintainer-approved, tested conflict-resolution rule in the Skills prompt section; if active skills should not outrank workspace policy, revise the hierarchy or scope it to trusted skills and align the docs/tests with that policy.

## Review Findings

- **[P2] Keep untrusted skills below workspace policy:** `src/agents/system-prompt.ts:171`
  - body: The added directive makes an active `SKILL.md` outrank `AGENTS.md`, `TOOLS.md`, and `SOUL.md`. Please scope this to trusted skills or keep workspace policy above skill instructions.
