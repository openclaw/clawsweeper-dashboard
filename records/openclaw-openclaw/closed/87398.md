## Summary

Keep open: the PR fixes the immediate root-version fast path, but the same argv is still classified as a help/version invocation by shared startup code, so subcommand `--version <value>` can run without normal dotenv, proxy, and bootstrap handling.

## What This Changes

The PR swaps the CLI help/version fast path to the root-only version helper, adds a regression test for `skills verify ... --version <value>`, and includes a test-only model directive stabilization.

## Best Possible Solution

Land after shared CLI invocation detection treats subcommand-scoped `--version <value>` as a normal command while preserving root `openclaw --version`, `-V`, and `-v` behavior.

## Reproduction Assessment

Yes. Source inspection shows current main exits from `configureProgramHelp` whenever nested argv contains `--version`, and the PR body gives an after-fix smoke for the target command; I did not run tests because this review is read-only.

## Solution Assessment

Not yet. Reusing `isRootVersionInvocation` is the right narrow fix for the immediate exit, but the shared help/version startup classifier needs the same root-only treatment for version flags before merge.
