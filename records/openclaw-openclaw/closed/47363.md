## Summary

Current main already ships Claude Code session continuation through the supported ACP path, so this PR's user-facing capability is implemented without adding a separate `runtime="claude-code"` branch.

## Evidence

- **Claude Code is intentionally routed through ACP on main:** The docs say requests to start Claude Code should go to the ACP runtime, not a native sub-agent runtime.
  - file: [docs/tools/acp-agents.md:17](https://github.com/openclaw/openclaw/blob/0e7250f37bf347c72790abf2a6692d5ec84e5ea1/docs/tools/acp-agents.md#L17)
