## Summary

The request is a reasonable UX idea, but on current `main` it fits the existing skill/plugin extensibility path rather than a new OpenClaw core `/alias` command. OpenClaw explicitly keeps optional capability in plugins/ClawHub, and the current skill system already exposes user-invocable slash commands that the TUI forwards to the gateway.

## Risks / Open Questions

- A ClawHub skill/plugin is not the same UX as a built-in `/alias` manager; users would still need to install or create the shortcut skill.
- TUI autocomplete/help currently enumerates built-in gateway commands, so custom skill-command discoverability may be weaker than a dedicated core `/alias` surface.
