## Summary

This is a useful Feishu/Lark UX enhancement, but it belongs in the Feishu plugin/ClawHub path rather than OpenClaw core. Current main already has the account-level fallback mentioned in the issue: Feishu `typingIndicator` is part of the top-level and per-account config schema, account config is merged by the Feishu plugin, and reply/comment typing reactions honor `account.config.typingIndicator`. The exact desktop/mobile selector is not a current core contract; if Feishu/Lark exposes a stable client-platform signal, resolving it is Feishu plugin-local config/event parsing work, not a missing core API.

## Best Possible Solution

Close this core issue and route the work to the Feishu plugin/ClawHub path. A focused Feishu plugin update should add a platform-aware `typingIndicator` policy only if Feishu/Lark exposes a stable client-platform signal; otherwise the current account-level `typingIndicator` support is the practical workaround for different bot/account use cases.
