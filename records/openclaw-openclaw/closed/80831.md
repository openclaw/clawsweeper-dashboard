## Summary

Current main already fixes the reported duplicate root manifest by staging the temporary manifest outside every selected backup source, and the fix is covered by regression tests plus the prior merged replacement PR for the same backup failure.

## What This Changes

The issue asks for `openclaw backup create --verify` to avoid duplicate root `manifest.json` entries when `TMPDIR` is under the backed-up state directory.

## Best Possible Solution

Keep the current containment-aware temp-manifest staging and direct stable users affected on v2026.5.7 to a release containing the fix or current main.

## Reproduction Assessment

Yes. The pre-fix path is source-reproducible from `v2026.5.7`, where backup creation stages `manifest.json` under `os.tmpdir()` and passes that path plus recursive source assets to `tar`; current main adds focused tests for `TMPDIR` under the state directory.

## Solution Assessment

Yes. Staging the temporary manifest in a root proven outside all selected backup sources is the narrowest maintainable fix and preserves the verifier's exactly-one-root-manifest invariant.
