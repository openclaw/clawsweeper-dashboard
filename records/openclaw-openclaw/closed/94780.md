## Label Justifications

- `P2`: The issue concerns a bounded scheduled automation/provider-routing capability with real cost impact but no current outage or security emergency.
- `impact:auth-provider`: The affected behavior is per-run provider/model selection for cron jobs.

## Summary

Close: current main and the latest release already honor `payload.model` for isolated `agentTurn` cron jobs, including allowlist validation, selected-model forwarding, persisted payload fields, and regression coverage; the remaining main-session and Bedrock-specific concerns are tracked separately.

## What This Changes

The issue requests per-cron model selection for isolated `agentTurn` jobs so automation can use a different model from the agent default.

## Best Possible Solution

Keep the current isolated `agentTurn` implementation, route main-session override design to the existing main-session issue, and keep Bedrock inference-profile behavior in the provider-specific issue.

## Reproduction Assessment

No for current main. Source inspection shows isolated `agentTurn` reads, validates, persists, and forwards `payload.model`; the same-root earlier fallback bug was closed by a merged fix PR.

## Solution Assessment

Yes. The existing payload-level model field is the narrowest maintainable contract already present in protocol, tool schema, persistence, and runtime, while broader main-session override semantics belong to the separate main-session issue.
