## Summary

Close as ClawHub/plugin work: the structural pre-spawn validation use case is valid, but current main already exposes a plugin hook that can inspect and block `sessions_spawn` before execution, and OpenClaw’s scope guidance keeps optional operator policies out of core when the plugin API can express them.

## What This Changes

Add a per-call `preSpawnHook` parameter to `sessions_spawn` that runs an operator validation script before spawning and aborts or warns on failure.

## Best Possible Solution

Keep core lean and implement this as an operator-owned ClawHub code plugin using `before_tool_call` to validate or block `sessions_spawn` before execution.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior report. Source and docs inspection confirms the relevant current-main surfaces are `sessions_spawn`, tool policy, and plugin `before_tool_call` hooks.

## Solution Assessment

No for the proposed per-call script parameter; the safer maintainable path is an operator-owned plugin hook, because it keeps script selection in operator-controlled code rather than model-supplied tool input.
