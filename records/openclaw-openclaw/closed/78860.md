## Summary

Current main no longer has Gateway startup/runtime dependency staging for `plugin-runtime-deps`; the reported v2026.4.26 cache path is now treated as legacy cleanup state, so the EFS rewrite loop is obsolete on main.

## What This Changes

The issue asks OpenClaw to stop an EFS-amplified infinite rewrite loop in bundled plugin runtime dependency staging under `plugin-runtime-deps`.

## Best Possible Solution

Keep the current install/update/doctor-owned dependency model and direct operators on old affected releases to upgrade, with `openclaw doctor --fix` available to clean legacy state.

## Reproduction Assessment

No, not against current main: I did not run the EFS/Docker repro, and current source no longer has the Gateway startup/runtime path that creates or validates `plugin-runtime-deps`. The supplied strace is strong evidence for the old v2026.4.26 behavior.

## Solution Assessment

Yes: removing runtime dependency staging from Gateway startup and treating `plugin-runtime-deps` as legacy cleanup is narrower and safer than adding special-case `ENOTDIR` handling to an obsolete validator.
