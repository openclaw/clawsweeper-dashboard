## Summary

Keep open: this is a valid current-main leak fix with sufficient terminal proof, but the current PR head can regress skills snapshot invalidation by deleting the workspace version and falling back to an older global version.

## What This Changes

The PR adds per-workspace skills snapshot version cleanup, opportunistic idle skills watcher eviction, regression tests, and changelog entries.

## Best Possible Solution

Keep the watcher and Map lifecycle cleanup, but make teardown and idle eviction preserve or advance an invalidation version so cached skills snapshots always refresh after watcher disposal.

## Reproduction Assessment

Yes. Source inspection shows current main writes workspace-scoped versions without a production cleanup path, and the paired issue plus PR body provide a concrete high-cardinality runtime proof shape.

## Solution Assessment

No, not yet. The cleanup boundary is right, but the current deletion-based implementation can lower the observed version and let a stale cached skills snapshot survive after idle eviction.
