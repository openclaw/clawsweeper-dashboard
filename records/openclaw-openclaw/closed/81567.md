## Summary

Keep open: current main still makes GPT-4o selectable, but the strict-agentic retry and blocked-exit contract remains scoped to GPT-5-family OpenAI/Codex runs, so the reporter's GPT-4o plan-only terminal path is not resolved or superseded.

## What This Changes

The issue asks OpenClaw to investigate and either support or clearly document GPT-4o's plan-only single-response behavior in agent sessions.

## Best Possible Solution

Choose the GPT-4o support boundary: either extend incomplete-turn or strict-agentic handling to GPT-4o with focused tests and live proof, or document and surface that GPT-4o is not supported for long multi-step agentic workflows.

## Reproduction Assessment

No high-confidence live reproduction was run on current main. Source inspection does show that if GPT-4o returns a stop/planning-only assistant turn, current main routes it through the default contract and skips the strict-agentic planning-only retry path.

## Solution Assessment

Unclear. Extending the strict-agentic/incomplete-turn lane to GPT-4o or documenting GPT-4o as unsupported for multi-step agentic workflows could both be maintainable, but maintainers need to choose the product boundary first.
