## Summary

Close as duplicate/superseded by #50248. This issue reports the same remaining cleanup safety bug: `sessions cleanup --fix-missing` can trust a stale persisted `sessionFile` and prune an entry even when the canonical `<sessionId>.jsonl` transcript exists. Current `main` still has the bug, but #50248 is the older canonical open issue and #69954 is the active implementation PR for the same fallback/repair behavior.

## Best Possible Solution

Close this issue as a duplicate of #50248 and keep the implementation discussion on #69954 or an equivalent PR. The eventual fix should make cleanup use transcript-candidate semantics, preserve explicit custom transcript paths, repair stale generated `sessionFile` metadata when the canonical transcript exists, and include cleanup regression tests for stale generated path plus valid canonical transcript, valid custom path, and truly missing transcripts.
