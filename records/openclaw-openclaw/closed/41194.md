## Summary

Current main no longer starts a new gateway for the reported non-gateway/no-service-installed case. The update path can still fall back to runDaemonRestart(), but runDaemonRestart() now returns false for a not-loaded service unless it finds an existing unmanaged gateway process or an installed macOS LaunchAgent to repair.

## Evidence

- **Update fallback path verified:** updateCommand still performs the reported pre-check: when shouldRestart is true it only prepares a restart script if resolveGatewayService().isLoaded() returns true, then passes restartScriptPath into maybeRestartService. maybeRestartService falls back to runDaemonRestart() when no restart script exists, so the remaining safety depends on the daemon lifecycle restart contract.
