## Summary

Current main implements the reset behavior for the reported stale session override path: auto-sourced fallback model/provider/auth-profile overrides are cleared on `/new` and `sessions.reset`, with regression coverage and a shipped changelog entry in v2026.4.9 via PR #63155.

## Evidence

- The current intended behavior still preserves explicit user-driven `/model` or `sessions.patch` selections across reset. This closes the stale auto-fallback/cron-style override problem described by the report, not a request to make `/new` erase deliberate user model pins.
