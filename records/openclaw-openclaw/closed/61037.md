## Summary

Current `main` already contains the Matrix self-message protections described in the report, and the shipped `v2026.4.23` tag contains the same guard. The Matrix monitor drops events from the bot's own Matrix user ID before ingress processing, routes verification/system messages away from the agent handler, and the docs already describe self-loop avoidance.

## Evidence

- **Current main drops self-authored Matrix events before ingress:** `readIngressPrefix()` resolves the Matrix self user ID and immediately returns before any further processing when `senderId === selfUserId`, preventing the bot's own outbound events from re-entering the Matrix inbound pipeline.
