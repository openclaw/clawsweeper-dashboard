## Summary

Current `main` already contains the Discord stale-socket reconnect crash fix that this PR targets, and it shipped in `v2026.3.28`. The remaining delta in this PR would broaden `reconnect-exhausted` suppression beyond current intended behavior by swallowing queued startup failures that `main` still treats as real errors.
