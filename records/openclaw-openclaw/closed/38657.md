## Summary

Current main intentionally preserves existing operator-owned provider credentials during generated models-state refresh, while the narrower stale baseUrl precedence defect was separately addressed; the broader claim still lacks the requested current-main onboarding before/after evidence after more than three months.

## What This Changes

Make explicit onboarding provider edits overwrite previously stored endpoint and credential values across canonical and per-agent configuration state.

## Best Possible Solution

Keep the current credential-preservation contract; if the problem persists, open a fresh issue with one exact current-version onboarding edit and redacted before/after values from openclaw.json, models.json, and auth-profiles.json so a narrow explicit-edit fix can be evaluated.

## Solution Assessment

No. Changing the shared merge helper to overwrite values unconditionally would risk replacing operator-owned credentials during setup and doctor flows; any future fix should be scoped to a proven explicit-edit operation.
