## Summary

Current main implements the runtime-mirror hardening needed for this v2026.4.26 report. The old release path rebuilt mirrored plugin trees by deleting and renaming them on load; current main uses fingerprinted freshness checks, skips unchanged mirrors, and refreshes changed mirrors in place. The fix is on main/2026.4.27 notes, not in the provided latest v2026.4.26 tag.

## Best Possible Solution

Keep the current main runtime-mirror implementation and let the 2026.4.27 package carry the fix. If the same CPU profile reproduces after upgrading past v2026.4.26, reopen with a fresh current-version sample and gateway/plugin-runtime-deps logs.
