## Summary

Current `main` already implements the requested Kimi two-round `web_search` flow. The Moonshot web-search runtime now sends Kimi's required `builtin_function` tool declaration, loops through follow-up rounds when `finish_reason === "tool_calls"`, replays the returned assistant `tool_calls` plus matching `tool` messages, preserves `reasoning_content`, and keeps native Kimi tool-call IDs intact for replay.
