## Summary

Current `main` already contains the gateway WebSocket handshake-timeout fix that matches this report, and the changelog shows it shipped in `v2026.3.22`. The reported symptom was local CLI commands timing out against the loopback gateway on `2026.3.12`; current code now uses a 10s pre-auth handshake timeout with an env override, plus regression coverage.
