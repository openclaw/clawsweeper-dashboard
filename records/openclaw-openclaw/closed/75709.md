## Summary

Keep this PR open, but it still needs changes before merge: current main still has the per-task full-registry relist, while the proposed snapshot reuse can make terminal ACP cleanup miss active work created after the sweep snapshot.

## What This Changes

The PR changes task registry maintenance to pass the initial sweep task snapshot into terminal ACP cleanup and adds a regression test counting maintenance `listTaskRecords()` calls.

## Best Possible Solution

Land a small fix that avoids full per-task relists with a current session-specific active-task lookup or equivalent fresh active-session view, preserves ACP close safety, adds focused regression coverage, and updates the changelog.

## Reproduction Assessment

Yes. A focused check can seed many task records, count `listTaskRecords()` during `runTaskRegistryMaintenance()`, and add a companion case where an active same-child ACP task appears after the initial snapshot but before cleanup.

## Solution Assessment

No, not as written. Reusing the initial snapshot fixes the call count but is not the narrowest safe fix because the ACP close guard needs current task state without cloning and sorting the whole registry.
