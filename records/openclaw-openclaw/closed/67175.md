## Summary

Current main already implements the bundled-runtime logging fix with a direct active-config-path reader, and regression tests cover both reading `logging.file` and writing logs to the configured path. This makes this PR's registration-seam approach obsolete.

## Best Possible Solution

Close this PR as obsolete on current main. Keep the shipped direct config-path logging reader and its regression tests; related open logging PRs should be reviewed against commit 5d3168c343a3af6b478e9eefc3ad9ccfcef136d1 rather than merged independently.
