## Summary

Current `main` already skips the doctor token warning when gateway auth resolves to `trusted-proxy`, and the same logic is present in released tag `v2026.4.23`.

## Evidence

- **Doctor auth check excludes trusted-proxy from token warning path:** `runGatewayAuthHealth` resolves the effective gateway auth mode, then returns early unless a token is actually needed. The `needsToken` guard explicitly excludes `auth.mode === "trusted-proxy"`, so the `Gateway auth is off or missing a token` note is not shown for trusted-proxy configs.
