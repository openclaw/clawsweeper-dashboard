## Summary

Keep open. Current main still has the same-line reasoning delta problem this PR targets, but the proposed underscore-skip fix is not merge-ready because it emits an append delta that leaves the previously emitted closing underscore in accumulated stream text.

## What This Changes

The PR changes `emitReasoningStream` to ignore a prior trailing italic underscore when computing reasoning `data.delta`, and adds a Vitest case for same-line reasoning appends.

## Best Possible Solution

Revise the reasoning stream so same-line deltas are computed on raw reasoning before markdown formatting, or make formatted reasoning updates explicit replacements/snapshots; coverage should verify that appending emitted deltas reconstructs the latest `data.text`.

## Reproduction Assessment

Yes. A deterministic source-level reproduction is two thinking snapshots, `Step 1` followed by `Step 1 and Step 2`, through `formatReasoningMessage` and the PR delta branch; the emitted delta does not reconstruct the second formatted snapshot when appended to the first.

## Solution Assessment

No. The proposed branch avoids the full-snapshot fallback, but it still treats markdown-wrapped formatted text as appendable even after a closing underscore was already emitted; raw-delta computation or explicit replacement semantics is the safer maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Compute thinking deltas before formatting:** `src/agents/pi-embedded-subscribe.ts:847`
  - body: For a same-line append from `Reasoning:\n_Step 1_` to `Reasoning:\n_Step 1 and Step 2_`, this branch emits `data.delta === " and Step 2_"`. Any client that appends deltas to the previous event text keeps the old closing underscore and renders `Reasoning:\n_Step 1_ and Step 2_`, so the delta no longer reconstructs `data.text`. Compute the delta on raw reasoning text or emit an explicit replacement/snapshot for this case.
  - confidence: 0.88
