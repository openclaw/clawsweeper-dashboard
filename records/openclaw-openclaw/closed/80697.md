## What This Changes

The issue reports that status and session/model RPC summaries repeatedly rebuild plugin manifest normalization metadata for each session row, causing multi-second event-loop stalls.

## Best Possible Solution

Use https://github.com/openclaw/openclaw/issues/78461 as the single tracker for the manifest-normalization metadata reuse fix, and fold this issue's CoClaw/session-row timing evidence into that implementation discussion.

## Reproduction Assessment

Yes, source-level: current main still maps status session rows through resolveSessionModelRef() into manifest model-id normalization and a metadata-snapshot fallback when a compatible current snapshot is not reused. I did not run a live 70-session timing profile in this read-only review.

## Solution Assessment

Yes for triage: closing this duplicate keeps the fix centered on the existing open canonical model-normalization performance issue. The implementation should prefer explicit current-snapshot or row-result reuse over a separate wall-clock fallback cache unless plugin owners choose that tradeoff.
