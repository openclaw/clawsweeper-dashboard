## Summary

Current main still contains the hardcoded GPT-5 reply brevity guard and tests that expect literal-ellipsis trimming, while an open linked PR explicitly proposes an opt-out for this same issue. Keep this open until maintainers decide the desired guard policy and either land or close the linked PR.

## What This Changes

The issue requests fixing or making configurable the GPT-5 chat reply brevity guard that can post-process final assistant text and append a literal ellipsis.

## Best Possible Solution

Decide the owner policy for GPT-family reply post-processing, then land a documented and tested fix via https://github.com/openclaw/openclaw/pull/82948 or a maintainer-approved alternative.
