## What This Changes

The PR changes cron timer task-ledger creation so isolated agent-turn cron jobs without an explicit session key record the derived stable cron session key, with a focused timer regression test.

## Best Possible Solution

Land the focused cron ledger fix after normal validation so isolated cron task rows point at the same stable backing session used by execution while explicit session keys and per-run transcript keys remain unchanged.

## Reproduction Assessment

Yes from source inspection: current main creates the detached task row before isolated execution returns and copies only `job.sessionKey`. An isolated agent-turn cron job without `sessionKey` therefore has a clear path to an unset `childSessionKey`.

## Solution Assessment

Yes. Reusing `resolveCronAgentSessionKey` for the stable backing cron session is the narrowest maintainable fix and preserves explicit `sessionKey` plus per-run `:run:<sessionId>` behavior.
