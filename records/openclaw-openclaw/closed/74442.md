## Summary

Keep open for maintainer review. Current main still strips `sessionKey` from hidden agent events at the shared event bus, while gateway terminal lifecycle persistence only writes final session state when it can resolve a session key. The PR is a narrow implementation candidate and the latest diff also suppresses hidden live agent/tool broadcasts, so I did not find a blocking correctness bug; the remaining question is maintainer acceptance of lifecycle `sessionKey` visibility plus CI validation.

## What This Changes

The PR preserves `sessionKey` on hidden lifecycle agent events, suppresses hidden live client broadcasts in `server-chat`, adds focused regression tests, and adds a changelog entry.

## Best Possible Solution

Land or adapt a narrow fix that lets gateway lifecycle persistence write terminal session state for hidden channel-routed runs while keeping hidden assistant, tool, generic error, and live client event traffic redacted; if lifecycle `sessionKey` on the shared event bus is too broad, carry the key through an internal persistence-only path instead.
