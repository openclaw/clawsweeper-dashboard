## Summary

Current main already contains Telegram polling recovery and same-chat follow-up protections that match this report, and the relevant polling fixes were shipped after `v2026.4.2` in `v2026.4.22`.

## Evidence

- **Release note: Telegram polling recovery:** `v2026.4.22` documents a Telegram polling fix that rebuilds the HTTP transport after `getUpdates` 409 conflicts so retries do not loop on a dead keep-alive socket.

- **Release note: channel health uses transport activity:** `v2026.4.22` also changed channel health to use provider-proven transport activity, explicitly covering Telegram stale-socket recovery instead of relying on quiet-chat event freshness.
  - file: [CHANGELOG.md:307](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/CHANGELOG.md#L307)
- **Current polling watchdog:** Current main tracks `getUpdates` liveness, logs polling stalls, stops the runner/bot, and restarts the polling cycle; recoverable network errors and 409 conflicts mark the Telegram transport dirty so the next cycle rebuilds it.
  - file: [extensions/telegram/src/polling-session.ts:246](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/telegram/src/polling-session.ts#L246)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Current bounded Telegram request timeouts:** Current main wraps Telegram fetches with per-method timeouts, including `getUpdates`, so wedged polling/control-plane requests fail over instead of hanging indefinitely.
  - file: [extensions/telegram/src/bot-core.ts:181](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/telegram/src/bot-core.ts#L181)
  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Regression test: second same-chat message is not stranded:** A regression test on current main covers the exact class of failure where a second message from the same Telegram chat arrives while the first run is still active and verifies both replies are delivered in order.
  - file: [extensions/telegram/src/bot.create-telegram-bot.test.ts:381](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/extensions/telegram/src/bot.create-telegram-bot.test.ts#L381)
