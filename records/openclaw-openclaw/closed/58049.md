## Summary

Current main already routes the reported unavailable-model/fallback path through the model failover chain: bare/minimal 404 assistant errors are classified as `model_not_found`, remaining-candidate unknown errors no longer abort traversal, and timeout/model-not-found fallback behavior is covered in current docs and tests.

## Evidence

- **404 assistant errors now enter fallback as model_not_found:** Current failover classification maps HTTP 404 responses to `model_not_found` unless the payload is clearly auth/billing/session/context-overflow related, which addresses the reported unavailable-model case instead of leaving it as a generic timeout/unknown failure.
