## Summary

This PR adds an optional, AlpaCore-specific local MCP bridge and VS Code workspace auto-start wiring. Current OpenClaw scope and implementation already route optional MCP/tool integrations through plugins, bundle-style plugins, and ClawHub rather than repo-local one-off workspace tooling, and current main has the generic MCP/bundle surfaces needed to package this outside core.

## Best Possible Solution

Close this core PR and route the AlpaCore bridge to an external ClawHub/plugin bundle or standalone MCP package. If packaging it outside the repo exposes a missing generic OpenClaw MCP/plugin API, open a narrow core API issue or PR for that generic gap rather than adding AlpaCore-specific workspace tooling.
