## Summary

Current main already handles the reported WebChat subagent announce case in the gateway display projection, so the internal inter-session user envelope is filtered before WebChat history/live updates and covered by a regression test. The proof commit is on current main only and is not included in v2026.5.7.

## What This Changes

WebChat inter-session `subagent_announce` messages should not appear in chat as user-attributed messages.

## Best Possible Solution

Keep the shared gateway display projection filtering so WebChat shows the clean assistant/subagent result without exposing the internal inter-session user envelope.

## Reproduction Assessment

Yes. The current regression test builds the reported `subagent_announce` inter-session envelope and source inspection shows current main filters it before WebChat display; I did not run a live UI session in this read-only review.

## Solution Assessment

Yes. Filtering in the shared gateway display projection is the narrowest maintainable fix because both history and live UI events use that projection, avoiding a WebChat-only styling workaround for internal envelopes.
