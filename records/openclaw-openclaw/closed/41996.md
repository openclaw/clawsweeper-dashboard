## Summary

Current `main` already implements the Feishu/Lark voice-send fix: outbound `.opus`/OGG audio is routed as `msg_type: "audio"`, video stays `media`, documents stay `file`, and the shipped `v2026.4.22` release contains the same behavior. The report matches an older Feishu send-type failure, but that path no longer matches current code.
