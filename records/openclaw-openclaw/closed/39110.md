## Summary

Current `main` already resolves the reported docs drift: `docs/concepts/session-tool.md` now documents the default no-nested-spawn policy and the `maxSpawnDepth >= 2` orchestrator exception, matching the subagents guide and the tested runtime behavior.

## Evidence

- **Session-tool docs now describe the exception:** `sessions_spawn` docs state that leaf sub-agents do not get session tools by default, and that when `maxSpawnDepth >= 2`, depth-1 orchestrators receive `sessions_spawn`, `subagents`, `sessions_list`, and `sessions_history`.
