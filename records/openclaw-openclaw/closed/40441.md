## Summary

#40441 targets a real current-main Control UI reload bug, but its implementation is too broad: it would clear `configFormDirty` inside the shared `loadConfig()` path used by both explicit reloads and passive refresh/reconnect flows. Open PR #40443 tracks the same linked bug (#40352) with the safer product shape: explicit reload discards stale local config state while passive refresh preserves unsaved edits.

## Best Possible Solution

Close #40441 as superseded and keep #40443/#40352 as the canonical path. The best fix is a scoped explicit-reload mechanism, such as a `loadConfig` option or separate reload handler, with regression coverage proving explicit Reload discards stale local config while passive refresh and reconnect preserve unsaved edits.
