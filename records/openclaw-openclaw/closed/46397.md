## Summary

Current `main` already fixes the reported restart behavior for `config.patch`: direct SIGUSR1 restarts are no longer unconditional, no-op patches short-circuit entirely, and the gateway reloader owns restart-vs-hot-reload decisions from the reload plan.

## Evidence

- **Direct restart is now gated by reload mode and reload plan:** `config.patch` calls `shouldScheduleDirectConfigRestart`, which resolves reload mode from the patched config and only forces a direct restart for `gateway.reload.mode === "off"` or for `hot` mode when the reload plan says a restart is required.
