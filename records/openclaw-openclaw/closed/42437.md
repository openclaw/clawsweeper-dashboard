## Summary

Current `main` already supports outbound message media from outside the workspace under the existing filesystem trust model. The exact knob is `tools.fs.workspaceOnly` rather than a new `tools.media.allowedRoots`, but the reported use case is implemented.

## Close Comment

Thanks for the report. Closing from a Codex review because current `main` already supports this workflow under the existing filesystem trust model.

The implementation now accepts outbound local media outside the workspace when `tools.fs.workspaceOnly` is `false`: `src/media/local-roots.ts:74` expands absolute and `~` media paths into allowed local roots, `src/media/read-capability.ts:29` enables host-local outbound media reads when root expansion is allowed, and the message send/action pipeline uses that media access in `src/infra/outbound/message-action-runner.ts:884` and `src/infra/outbound/outbound-send-service.ts:79`.

So the requested use case is implemented on current `main`, although the config surface is the existing `tools.fs.workspaceOnly` model rather than a new `tools.media.allowedRoots` key. Current reviewed `main` SHA: `7d9172d5e093435d37ed3e00e476a4eaf84a252b`.
