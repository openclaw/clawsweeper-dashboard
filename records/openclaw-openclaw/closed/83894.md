## What This Changes

The issue requests replacing config preflight's global `process.stdout.write` suppression with a scoped output-suppression path that does not drop unrelated concurrent stdout writes.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84439 if CI and maintainer review stay clean; otherwise keep this issue as the canonical bug until an equivalent scoped note-suppression fix lands.

## Reproduction Assessment

Yes, at source level: call `ensureConfigReady()` with `suppressDoctorStdout: true`, keep the mocked preflight pending, and write to stdout from a queued concurrent task before the `finally` restore. I did not execute the repro because this review is read-only.

## Solution Assessment

Yes. The maintainable fix is to scope suppression to OpenClaw preflight note output, as the linked PR does, instead of mutating process-global stdout or process-wide note state across an awaited preflight.
