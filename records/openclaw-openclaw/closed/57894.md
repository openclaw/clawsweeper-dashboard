## Summary

Close as `clawhub`: this is a useful context-policy idea, but current OpenClaw scope and plugin architecture point it to a context-engine plugin rather than new core behavior. Current `main` does not already implement time-based conversation filtering; the built-in pruning feature only trims old tool results.

## Risks / Open Questions

- No linked context-engine plugin is already posted on the issue, so closing this in core still leaves the implementation work to a plugin author.
- A plugin may need its own config surface for a rolling time window; if plugin authors hit a concrete API gap while implementing it, that narrower API-gap issue would still be valid.
