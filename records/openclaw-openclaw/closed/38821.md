## Summary

Current main already implements replay-side stale media pruning for the reported Discord vision failure: old image blocks and textual media markers are scrubbed before provider replay, while current Discord attachments still flow through MediaPaths into the prompt. The related #71868 root-cause report is closed and the 2026.4.25 changelog records the stale media rehydration fix.

## Best Possible Solution

Keep the shipped replay-pruning implementation and close #38821 as implemented. If unrelated text-context contamination still reproduces, track it separately against the context engine or compaction path with fresh reproduction data.
