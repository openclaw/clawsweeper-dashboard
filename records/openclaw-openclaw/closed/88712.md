## Summary

Current main already fixes the reported channel dispatch failure by passing the full plugin channel runtime into bundled channel startup; the affected v2026.5.28 code still had the context-only startup runtime, so this is main-only until the next release.

## What This Changes

The issue reports QQBot and Feishu external channel dispatch failing with `Cannot read properties of undefined (reading 'run')` after `openclaw gateway run --allow-unconfigured`.

## Best Possible Solution

Keep the current main full-runtime startup path and let the next release carry the fix to users on v2026.5.28.

## Reproduction Assessment

Yes from source comparison: v2026.5.28 could pass a context-only `channelRuntime` into bundled channel startup while Feishu and QQBot dispatch paths need `channel.inbound.run`. I did not run live QQBot or Feishu credentials in this read-only review.

## Solution Assessment

Yes; the current fix removes the bundled-channel partial startup runtime path and passes the full `PluginRuntimeChannel`, which is the narrow generic owner-boundary fix for both channels.
