## Summary

Issue #51509 targets the removed Chrome extension relay/browser-relay options flow. Current `main` has no browser relay subsystem to extend; OpenClaw migrated to Chrome MCP `existing-session`/named browser profiles instead.

## Evidence

- **Relay path removed in shipped release:** `CHANGELOG.md` records that `2026.3.22` removed the legacy Chrome extension relay path, bundled extension assets, `driver: "extension"`, and `browser.relayBindHost`.
  - file: [CHANGELOG.md:2211](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/CHANGELOG.md#L2211)
  - command: `nl -ba CHANGELOG.md | sed -n '2188,2222p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Runtime keeps extension relay as no-op:** The browser runtime still exposes `ensureExtensionRelayForProfiles`, but its body is an intentional no-op because the Chrome extension relay path has been removed.
  - file: [extensions/browser/src/browser/server-lifecycle.ts:9](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/extensions/browser/src/browser/server-lifecycle.ts#L9)
  - command: `nl -ba extensions/browser/src/browser/server-lifecycle.ts | sed -n '1,40p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Legacy configs are auto-migrated away from extension relay:** Doctor normalization deletes `browser.relayBindHost` and rewrites `browser.profiles.<name>.driver` from `extension` to `existing-session`.
  - file: [src/commands/doctor/shared/legacy-config-core-normalizers.ts:27](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/src/commands/doctor/shared/legacy-config-core-normalizers.ts#L27)
  - command: `nl -ba src/commands/doctor/shared/legacy-config-core-normalizers.ts | sed -n '20,55p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Docs describe the old extension path as removed:** Gateway doctor docs say configs pointing at the removed Chrome extension path are normalized to the current host-local Chrome MCP attach model.
  - file: [docs/gateway/doctor.md:205](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/docs/gateway/doctor.md#L205)
  - command: `nl -ba docs/gateway/doctor.md | sed -n '205,236p'`
  - sha: [608c08fc54da](https://github.com/openclaw/openclaw/commit/608c08fc54da2f86338366f8eeafd7e2a02713aa)
- **Current browser control uses named browser profiles, not relay connection profiles:** Browser docs describe current multi-profile support through named browser profiles, with `?profile=<name>` and `--browser-profile`, under the managed/remote/existing-session model.
  - file: [docs/tools/browser.md:370](https://github.com/openclaw/openclaw/blob/608c08fc54da2f86338366f8eeafd7e2a02713aa/docs/tools/browser.md#L370)
