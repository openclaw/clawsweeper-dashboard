## Summary

Keep open. Current main still has the source-visible path where a persisted skill snapshot can apply env overrides for a disabled `skills.entries.gh-issues` config, and the open linked PR already owns the focused fix with closing syntax.

## What This Changes

The issue reports that a disabled `skills.entries.gh-issues.apiKey` SecretRef is still evaluated during WhatsApp/embedded lane sessions, crashing turns despite secrets audit/reload passing.

## Best Possible Solution

Refresh/review and land https://github.com/openclaw/openclaw/pull/79173, or replace it with the same disabled-entry guard and regression test if that branch cannot be merged.

## Reproduction Assessment

Yes, source-level. A persisted snapshot containing `gh-issues`, current config with `skills.entries.gh-issues.enabled:false`, a raw SecretRef apiKey, and no host `GH_TOKEN` reaches `applySkillEnvOverridesFromSnapshot` without a disabled-entry guard.

## Solution Assessment

Yes. Honoring `enabled:false` before skill env injection in both live and persisted snapshot paths is the narrow maintainable fix, and the linked PR targets that boundary with regression coverage.
