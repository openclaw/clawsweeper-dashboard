## Summary

Current main and the shipped v2026.5.20 release already guard Discord progress drafts once final delivery starts, with regression tests for both late and pending-final progress callbacks.

## What This Changes

Prevent Discord tool-progress draft updates from appearing after or during final answer delivery for the same turn.

## Best Possible Solution

Keep the shipped Discord progress draft finalization behavior; any new report should include live evidence for a path not covered by the final-start and pending-final guards.

## Reproduction Assessment

Yes for the reported race shape: current Discord tests simulate command/progress callbacks both after final delivery and while final delivery is pending. There is no failing current-main reproduction because the guards are already present.

## Solution Assessment

Yes; the implemented fix keeps the state and cleanup inside the Discord draft/finalizer path instead of changing generic reply delivery semantics.
