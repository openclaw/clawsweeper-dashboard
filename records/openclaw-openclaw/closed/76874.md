## Summary

Keep open. Current main still has a source-reproducible stale Control UI run-state path, and the earlier implementation PR #76894 is closed without merge, so there is no open paired fix to wait on.

## What This Changes

Make the Control UI clear stale active-run state after a completed assistant reply so the Stop button returns to Send without a refresh.

## Best Possible Solution

Patch terminal Control UI chat lifecycle handling to reconcile the cached current-session active-run flag, with regression coverage for final, aborted, or error events after a row already has `hasActiveRun: true`.

## Reproduction Assessment

Yes, by source inspection: seed the current cached session row with `hasActiveRun: true`, set a matching `chatRunId`, then deliver a terminal `chat` event with a final assistant message. `chatRunId` clears, but the cached row remains active and keeps `hasAbortableSessionRun` true.

## Solution Assessment

Yes. The narrow maintainable fix is to clear or refresh the cached session active-run state on matching terminal chat lifecycle events; the closed PR #76894 proposed that boundary but did not land.
