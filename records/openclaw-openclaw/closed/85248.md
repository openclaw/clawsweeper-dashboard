## Summary

Keep open for maintainer review. The patch is focused and supported by source/proof, but merging it codifies a compatibility contract: accumulated OpenAI-compatible `delta.tool_calls` are trusted only when the final `finish_reason` maps to tool use.

## What This Changes

The PR adds finish-reason/tool-call guards to the OpenAI-compatible gateway transport and registered completions provider parser, plus regression tests for stripping non-toolUse toolCall blocks while preserving true tool-call finishes.

## Best Possible Solution

Land after maintainers explicitly accept `finish_reason` as the authoritative tool-use signal; otherwise revise the PR into an explicit provider compatibility path before merge.

## Reproduction Assessment

## Solution Assessment
