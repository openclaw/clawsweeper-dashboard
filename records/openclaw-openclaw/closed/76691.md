## Summary

Current main already cleans up Dream Diary narrative sub-sessions after accepted runs and includes regression coverage; the same implementation is present in shipped v2026.5.2.

## What This Changes

The issue requests automatic cleanup of memory-core Dream Diary `dreaming-narrative-*` subagent sessions after the narrative content is processed.

## Best Possible Solution

Keep the shipped cleanup path that deletes accepted Dream Diary subagent sessions and scrubs stale dreaming artifacts; future reports should include a concrete version and retained session-store entry sample if cleanup still fails.

## Reproduction Assessment

No. Current main has a source-backed non-reproduction path: accepted `dreaming-narrative-*` attempts are deleted in `finally`, and tests cover normal, retry, failure, and stale orphan cleanup behavior.

## Solution Assessment

Yes. The current implementation uses the existing plugin subagent `deleteSession` and gateway `sessions.delete` lifecycle, with a targeted stale-artifact scrubber instead of adding a new TTL/config surface.
