## Summary

Current `main` already suppresses the reported retry-noise case: mutating tool failures are recorded, then cleared when the same action later succeeds, and the user-facing warning path only runs if that failure state remains. The current release tag `v2026.4.22` already contains the same implementation and regression test coverage for the `edit` mismatch-then-retry scenario.

## Evidence
