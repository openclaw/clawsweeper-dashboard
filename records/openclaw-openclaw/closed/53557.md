## Summary

Keep open. The PR contains useful memory-search status UX work, but the patch is not merge-ready because it edits legacy core files that are absent on current main, while the shipped memory CLI/tool now live in `extensions/memory-core`; it also misses the promised successful-tool-response field.

## What This Changes

The PR adds memory status display rows and a more specific Ollama auto-mode unavailable-action hint in legacy memory CLI/tool files.

## Best Possible Solution

Port the behavior into `extensions/memory-core`: render search mode and provider issue in human `openclaw memory status`, consolidate fallback into one readable row, include `providerUnavailableReason` in successful `memory_search` results when present, and add the Ollama auto-mode unavailable-action hint without changing provider selection semantics.
