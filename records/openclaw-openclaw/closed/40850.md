## Summary

Current main already implements the central user-facing fix from PR #40850/#40839 through a maintainer-owned installer warning for duplicate OpenClaw global installs across Node/npm roots. The shipped implementation is more targeted than the PR: it warns when multiple actual OpenClaw installs exist, prints active node/npm/openclaw paths, lists package versions and roots, and has regression coverage.

## Best Possible Solution

Close PR #40850 as implemented/superseded by the maintainer-owned installer warning on main. Keep the current duplicate OpenClaw install detection and tests; if maintainers still want a proactive warning for merely having multiple Node managers, that should be a separate product decision with clear false-positive guidance.
