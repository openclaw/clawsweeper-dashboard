## Summary

This issue should stay open: current main still replaces the whole oversized trajectory event `data` block, and the active open closing candidate is https://github.com/openclaw/openclaw/pull/96811 rather than a merged fix.

## What This Changes

The issue reports that oversized `model.completed` trajectory rows discard `data.usage` during event-level truncation, causing heavy-turn token accounting to be lost.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/96811 with a narrow fix that preserves accounting-critical fields while keeping the event byte cap and redaction behavior intact.

## Reproduction Assessment

Yes, from source inspection: current main stores `usage` inside `model.completed.data`, and both event-level truncation helpers replace that whole object once the serialized row exceeds 262144 bytes. I did not run a write-path reproduction because this review is read-only.

## Solution Assessment

Yes in direction: preserving small structured accounting fields inside the truncation stub is the narrow runtime-layer fix. The open linked PR should be reviewed and landed or replaced before this issue is closed.
