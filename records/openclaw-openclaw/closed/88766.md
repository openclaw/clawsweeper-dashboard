## What This Changes

Restore Mattermost progress-preview updates during Codex message-tool-only turns without changing final reply delivery.

## Best Possible Solution

Rebase and land the narrow Mattermost reply-options fix that opts channel-owned progress callbacks into suppressed-source delivery only while draft tool progress is active.

## Reproduction Assessment

Yes, at source level. Mattermost passes progress callbacks but omits the allow flag, while dispatch suppresses those callbacks under message_tool_only unless the flag is true; the reporter also confirmed the symptom on v2026.6.5.

## Solution Assessment

Yes. The best fix is the narrow Mattermost replyOptions opt-in already proposed by the linked PR, because it preserves final message-tool delivery while restoring only the channel-owned progress draft path.
