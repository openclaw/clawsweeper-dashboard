## Summary

Keep open. Current main still has the reported write path: empty non-report wiki pages are summarized, sent through the managed-block helper, and written back as a Related-only file; the only direct fix PR found was closed unmerged by active-PR-limit automation.

## What This Changes

Prevent memory-wiki related-block refresh from rewriting empty or whitespace-only source pages into 107-byte managed Related stubs.

## Best Possible Solution

Land a narrow memory-wiki fix that skips zero-byte and whitespace-only pages before related-block writes, adds regression coverage, and leaves the broader imported-source validation work to #64696 unless the same small PR can safely validate stubs too.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create an empty `sources/*.md`, run `compileMemoryWikiVault`, and current main will summarize it, build the 107-byte Related block, and write it back through `refreshPageRelatedBlocks`.

## Solution Assessment

Yes. The narrowest maintainable fix is to guard `refreshPageRelatedBlocks` before calling the shared helper; changing the helper globally may affect create-from-scratch callers, while source-sync validation is the complementary #64696 scope.
