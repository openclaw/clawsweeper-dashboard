## Summary

Close as implemented: current main and v2026.5.20 resolve `sessions_spawn` model selection through the target agent, persist the child session model before the run, and have focused coverage for target-agent subagent models.

## What This Changes

The issue requests that `sessions_spawn` child sessions use the target agent's configured model instead of inheriting the parent agent's model.

## Best Possible Solution

Keep the shipped target-agent model resolution path; users on older builds should update to v2026.5.20 or newer and pass `agentId` for cross-agent spawns.

## Reproduction Assessment

No current-main failure reproduction is available from the inspected path. Current source and shipped v2026.5.20 source route explicit target-agent spawns through `targetAgentId` and persist the resolved child model before the agent run.

## Solution Assessment

Yes. The current implementation resolves the target agent's configured subagent or primary model during spawn, records it on the child session, and lets the runner derive the agent from the child session key.
