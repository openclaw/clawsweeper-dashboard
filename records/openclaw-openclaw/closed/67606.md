## Summary

Current `main` already routes Telegram photo turns through inbound media-understanding, prefers `agents.defaults.imageModel` over the active reply model for image analysis, and auto-registers user-configured image-capable providers. That addresses the reported path where Telegram photos were sent to a text-only primary model and failed with a provider format error.

## Evidence

- **Telegram still forwards inbound photo/file paths into session context:** Telegram message context stores `MediaPath`/`MediaPaths` and MIME types for inbound media, so photos reach the shared inbound-media pipeline instead of being dropped at the channel layer.
