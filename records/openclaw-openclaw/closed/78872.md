## Summary

Keep open: the reported Feishu topic-session split is valid and the contributor supplied useful real logs, but the current patch still regresses documented normal-group root-keyed sessions and broadens authenticated message reads before group admission checks.

## What This Changes

The PR changes Feishu inbound group-topic routing to hydrate `omt_*` thread IDs from root messages, suppress `rootId` during routing, and adds two Feishu bot regression tests.

## Best Possible Solution

Keep the fix Feishu-local, distinguish Feishu topic-message-format groups from OpenClaw-created reply threads, defer or avoid extra hydration until after admission gates, and document the shipped user-facing behavior.

## Reproduction Assessment

Yes. Current main can route a normal `group` topic first turn by `omt_*` when no root is present and a follow-up with both `root_id` and `thread_id` by `om_*`; the contributor also supplied redacted before/after Feishu WebSocket logs from OpenClaw 2026.5.6.

## Solution Assessment

No as-is. The Feishu plugin is the right surface, but this implementation treats all normal-group `om_*`/`omt_*` pairs as native topics and performs the new lookup before access checks.
