## Summary

Current `main` already prevents the duplicate-gateway outcome behind this report: managed `gateway start` routes through service restart/scheduling logic, and any extra foreground gateway process fails closed on lock/port checks instead of competing long-term.

## Evidence

  - file: [src/cli/daemon-cli/register-service-commands.ts:97](https://github.com/openclaw/openclaw/blob/fa976e5b93b3819f9e137fe08635764edee38290/src/cli/daemon-cli/register-service-commands.ts#L97)

- **Service start routes through restart semantics:** `startGatewayService()` reads current service state, then calls `service.restart(...)` and returns `outcome: "scheduled"` when appropriate instead of spawning an independent unmanaged gateway.
  - file: [src/daemon/service.ts:114](https://github.com/openclaw/openclaw/blob/306c0f73bff1338482dd6533f61a86905bf541b1/src/daemon/service.ts#L114)
