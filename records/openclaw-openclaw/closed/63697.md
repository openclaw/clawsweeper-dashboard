## Summary

Keep open. Current main still intercepts the native chat reset/compact slash commands locally, while this PR is a broad active implementation that introduces approval-scope and transcript-handling regressions before it can merge.

## What This Changes

The PR routes shared native iOS/macOS chat slash commands through `chat.send` and also adds broader composer, model picker, tool-result, splash, and iOS exec-approval UI changes.

## Best Possible Solution

Keep the branch open but split or narrow it to the native chat slash-routing behavior, preserving existing reset/compact transcript semantics and the iOS approval-scope compatibility contract.

## Reproduction Assessment

Yes. On current main, entering `/new` or `/compact` in `OpenClawChatViewModel` reaches the local `resetSession` or `compactSession` path, and the existing Swift tests assert no `chat.send` run id is produced.

## Solution Assessment

No. Routing slash commands through the gateway is the right product direction, but this PR is not the best mergeable shape because it adds unrelated UI/auth changes and introduces blocking regressions.
