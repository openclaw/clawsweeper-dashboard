## Summary

Current main fixes the narrow Telegram verbose-progress regression behind this report: default tool/progress messages are no longer suppressed when Telegram preview streaming is off, and partial preview mode shows short tool-progress lines. The broader durable Web UI-style activity stream remains tracked by #37816 and #12831.

## Best Possible Solution

Close this narrow bug as fixed on main by PR #71825. Keep the shipped suppression guard and regression tests, and continue any durable Telegram activity-message/Web UI parity work in #37816, with #12831 as the broader cross-surface observer-mode tracker.
