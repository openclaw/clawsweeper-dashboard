## Summary

The reported regression is already fixed on current `main` and in the latest release: `openclaw plugins update` now accepts `--dangerously-force-unsafe-install`, threads it through the update path, and has CLI/updater tests plus docs covering that behavior. The remaining allowlist/verification-bypass ideas were not adopted; current docs explicitly keep `plugins.allow` as an id allowlist rather than a scan bypass.
