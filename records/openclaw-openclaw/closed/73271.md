## Summary

Close as implemented on main. Current OpenClaw already persists conversation sessions by default through gateway-owned session rows and transcript files, includes restart recovery for interrupted resumable main sessions, and has durable task/Task Flow state for background work; the requested opt-in `sessionPersistence` flag is not needed for the reported user problem.

## What This Changes

Request to add optional per-agent session persistence that saves and restores conversation context and in-progress state across gateway restarts.

## Best Possible Solution

Keep the current gateway-owned persistence model as the product contract: session history and metadata persist by default, cleanup is controlled through `session.maintenance`, and durable background workflows use tasks or Task Flow rather than a new per-agent persistence toggle.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source/docs inspection plus the restart-recovery unit test showing persisted transcript state is reused after restart recovery.

## Solution Assessment

Yes. The current default persistence model is the maintainable solution because it avoids a split opt-in contract while still giving operators cleanup and retention controls.
