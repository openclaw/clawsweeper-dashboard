## Summary

Close as implemented on current main. The reported restart blocker is covered by current startup and session-maintenance behavior: the Gateway binds HTTP/WebSocket before sidecars and recovery work runs, missing transcript reads are non-fatal, and explicit cleanup exists for sessions.json entries whose transcript files are gone.

## What This Changes

The issue asks OpenClaw to survive shutdown crashes that leave missing trajectory/transcript artifacts referenced by session metadata without blocking the next Gateway restart.

## Best Possible Solution

Keep the current main behavior and ship it: startup should bind before optional sidecar/session recovery work, missing session artifacts should be diagnosed and repairable through the session cleanup path, and affected v2026.4.26 installs should upgrade or run the documented cleanup command once.

## Reproduction Assessment

Yes for the reported failure class: a sessions.json entry with a missing resolved transcript path exercises the same orphaned-reference path, and current main handles that without blocking listener startup. A live systemd stop/start was not run because this review is read-only.

## Solution Assessment

Yes. Binding before sidecars plus explicit missing-transcript cleanup is the narrow maintainable fix for the restart blocker; automatic load-time deletion of session rows would be riskier than the existing doctor/cleanup flow.
