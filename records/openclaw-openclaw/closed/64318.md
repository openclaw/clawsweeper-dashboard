Full review comments:

- **[P1] Update the lockfile for the Amazon package:** `extensions/amazon/package.json:7-10`
  - body: `extensions/amazon/package.json` adds three new AWS SDK dependencies, but the PR head lockfile has no `extensions/amazon` importer and no Polly or Transcribe Streaming entries. Frozen installs and `check-dependencies` will fail before build or tests can validate the plugin.
  - confidence: 0.97
- **[P1] Resample telephony input before Nova Sonic:** `extensions/amazon/nova-sonic/bridge.ts:79-80`
  - body: This only changes 8 kHz mu-law bytes into PCM samples, then sends them into a Nova stream declared as 16 kHz. Voice-call audio will arrive time-scaled/pitched instead of real 16 kHz speech unless it is resampled before `audioInput`.
  - confidence: 0.93
- **[P1] Downsample Nova output before emitting telephony audio:** `extensions/amazon/nova-sonic/bridge.ts:487-490`
  - body: The bridge requests 24 kHz PCM output from Nova, then converts it directly to mu-law and sends it to the voice-call sink, whose contract is 8 kHz mu-law. That produces wrong-speed playback and timing drift even though the codec is mu-law.
  - confidence: 0.93
- **[P1] Use a Polly-valid PCM sample rate:** `extensions/amazon/polly/speech-provider.ts:235-241`
  - body: `synthesizeTelephony` requests `outputFormat: "pcm"` with `sampleRate: "22050"`, but Polly only accepts 8000 or 16000 for PCM. Normal voice-call TTS through this provider will fail with a Polly validation error.
  - confidence: 0.96
- **[P2] Parse Nova tool events with the AWS schema:** `extensions/amazon/nova-sonic/bridge.ts:521-530`
  - body: Nova Sonic tool output uses `toolName` and JSON `content`, while this reads `name` and `input`; tool results are likewise sent as `textInput` instead of the documented `toolResult` event. Advertised tool calling will not work reliably until both directions match the AWS event schema.
  - confidence: 0.88
- **[P2] Read canonical Talk override keys:** `extensions/amazon/polly/speech-provider.ts:160-162`
  - body: Speech-core passes `voiceId` and `modelId` into `resolveTalkOverrides`, but this provider only reads `voice` and `engine`. Explicit Talk voice/model requests are ignored and can trip the existing "ignored override" error path.
  - confidence: 0.91
- **[P2] Normalize or detect FLAC sample rates:** `extensions/amazon/transcribe/stt.ts:110-112`
  - body: The FLAC path passes bytes through but declares 16000 Hz for every file. Amazon Transcribe requires the declared sample rate to match the input audio, so common 44.1 or 48 kHz FLAC files will fail or transcribe incorrectly.
  - confidence: 0.84
- **[P3] Add the Amazon provider page to docs navigation:** `docs/providers/amazon.md:1`
  - body: The PR adds `docs/providers/amazon.md`, but the provider navigation does not include `providers/amazon`, leaving the new provider documentation unreachable from the docs sidebar and consistent with the red `check-docs` status.
  - confidence: 0.78
