Full review comments:

- **[P1] Use a valid Polly PCM sample rate:** `extensions/amazon/polly/speech-provider.ts:237-243`
  - body: `synthesizeTelephony` requests `outputFormat: "pcm"` with `sampleRate: "22050"`, but Amazon Polly only accepts PCM sample rates of `8000` or `16000`. Voice-call synthesis through this provider will fail with an AWS validation error instead of returning telephony audio.
  - confidence: 0.95
- **[P1] Resample Nova output before telephony callbacks:** `extensions/amazon/nova-sonic/bridge.ts:491-494`
  - body: Nova Sonic is configured for 24 kHz PCM output, but `handleOutputEvent` converts those samples directly to mu-law and sends them to `onAudio`, which voice-call feeds to an 8 kHz telephony pacer. This produces wrong-speed audio unless the PCM is downsampled to 8 kHz before mu-law encoding.
  - confidence: 0.88
- **[P1] Resample caller audio before Bedrock input:** `extensions/amazon/nova-sonic/bridge.ts:74-82`
  - body: Incoming voice-call audio is 8 kHz mu-law, but `sendAudio` only changes the encoding to PCM before sending it into a content stream declared as 16 kHz PCM. That misdeclares the sample rate to Nova Sonic and can distort recognition and timing for live calls.
  - confidence: 0.85
- **[P1] Merge parent plugin defaults into resolved config:** `extensions/amazon/polly/speech-provider.ts:130-132`
  - body: `buildPollySpeechProvider` captures `pluginConfig.polly`, but `resolveConfig` normalizes only the provider raw config. When runtime selection resolves an empty provider config, operator settings under `plugins.entries.amazon.config.polly.*` are replaced by hardcoded defaults, so configured region/voice/engine values can be ignored.
  - confidence: 0.8
- **[P2] Read canonical Talk override keys:** `extensions/amazon/polly/speech-provider.ts:144-149`
  - body: Talk passes canonical `voiceId` and `modelId` override keys, but the Polly Talk adapter reads `voice` and `engine`. Explicit Talk voice/model overrides for Amazon Polly will be dropped or reported as unsupported instead of applying to synthesis.
  - confidence: 0.84
- **[P1] Update the lockfile for the Amazon workspace:** `extensions/amazon/package.json:7-10`
  - body: This new workspace declares three AWS SDK dependencies, but the PR does not change `pnpm-lock.yaml` and the branch lockfile has no `extensions/amazon` importer. Frozen installs and dependency checks cannot resolve this plugin reproducibly until the lockfile is updated.
  - confidence: 0.9
- **[P2] Do not hardcode FLAC sample rate:** `extensions/amazon/transcribe/stt.ts:110-112`
  - body: The FLAC pass-through path declares `MediaSampleRateHertz` as `16000`, but Amazon Transcribe requires the declared sample rate to match the input audio. Common 44.1/48 kHz FLAC files will be sent with a mismatched sample rate unless the code detects the header or normalizes through ffmpeg.
  - confidence: 0.83
