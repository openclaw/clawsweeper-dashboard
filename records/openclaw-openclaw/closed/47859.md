## Summary

Keep open. Current main still lacks the requested TUI image-attachment surface, but this PR is not mergeable as-is because it rewrites prompts, can leak raw local paths for attachment-only messages, and does not line up with the current shared Gateway/local TUI backend boundary.

## What This Changes

The PR adds a TUI image parser, a `/image` slash command, attachment forwarding for TUI `chat.send`, and parser/loader tests.

## Best Possible Solution

Land a maintainer-reviewed TUI implementation that preserves prompt text unless image tokens are removed, never sends raw local path tokens, threads attachments through the shared `TuiBackend` contract or gates unsupported modes, reuses Gateway media validation, and updates docs, tests, and changelog.

## Reproduction Assessment

Yes. The feature gap is reproducible by static inspection of current main: there is no `/image` command, no TUI image parser, and no attachment field in `TuiBackend`; the PR defects are reproducible by tracing the diff around `extractImagePaths` and the `cleanText || text` fallback.

## Solution Assessment

No. Reusing the existing Gateway `chat.send.attachments` seam is the right direction, but this patch is not the best mergeable implementation because it mutates ordinary prompt text, leaks attachment-only local paths, and does not address the current shared Gateway/local TUI backend boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve prompts when no image refs are present:** `src/tui/tui-image-extract.ts:59`
  - body: `extractImagePaths` always collapses repeated whitespace and trims `cleanText`, and `sendMessage` uses that result for every non-BTW message. A normal prompt with indentation, leading whitespace, or intentional double spaces is changed before it reaches the model even when no image token was extracted.
  - confidence: 0.95
- **[P2] Strip attachment-only paths before sending:** `src/tui/tui-command-handlers.ts:565`
  - body: For a message such as `@/tmp/screenshot.png`, `cleanText` is empty, so `message = cleanText || text` falls back to the raw local path token. That displays and sends the path instead of sending only the attachment payload or a neutral placeholder.
  - confidence: 0.93
- **[P2] Gate or implement image sends for local TUI mode:** `src/tui/commands.ts:116`
  - body: The new `/image` command is advertised unconditionally, but current main uses the same TUI command list for Gateway mode, embedded local mode, and Crestodian. Unless the shared backend contract and embedded implementation handle attachments, local users will see a command with no defined attachment path.
  - confidence: 0.84
- **[P3] Check the bytes that are actually read:** `src/tui/tui-image-extract.ts:88-101`
  - body: The loader checks `fs.stat` before `fs.readFile`, so the file can be replaced or enlarged after passing the 5 MB guard. Read the buffer first and enforce the limit on `buffer.byteLength` before base64 encoding.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `src/tui/commands.ts:116`
  - body: This is a user-facing TUI feature, but the PR does not update `CHANGELOG.md`. Add a single-line entry under the active version before merge.
  - confidence: 0.9
