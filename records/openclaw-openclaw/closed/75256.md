## Best Possible Solution

Implement Mattermost block-mode boundary splitting through the existing shared `streaming` contract, preserve `off`/`progress`/`preview`/`block` controls, add focused regression coverage, and include redacted real Mattermost proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main keeps one Mattermost draft post id and updates it across partial, reasoning, and tool callbacks without rotating at block-mode boundaries; I did not run a live Mattermost UI reproduction in this read-only review.

## Solution Assessment

No. Boundary splitting is the right Mattermost-local direction, but this PR should preserve the current shared streaming contract, remove the false preview-off workaround, and either redact raw tool args or document them as a sensitive opt-in before merge.
