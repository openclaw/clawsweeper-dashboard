## What This Changes

The issue requests that scheduled Discord agent turns stop rendering failed internal file-read/tool messages as normal visible chat output.

## Best Possible Solution

Review and land the focused heartbeat policy fix, or an equivalent change, so scheduled heartbeat runs use the same group/channel visible-reply and model-runtime policy as normal dispatch with focused regression coverage.

## Reproduction Assessment

Yes at source level: current main's heartbeat path ignores `messages.groupChat.visibleReplies` while normal dispatch honors it, and heartbeat only enables `message_tool_only` suppression after that missed decision. I did not run a live Discord scheduled-turn replay.

## Solution Assessment

Yes, the best direction is to align heartbeat response-tool selection with the existing group/channel visible-reply and runtime policy rather than adding Discord-specific output filtering. The open linked PR is the current focused implementation path, so the issue should stay open until that lands or is replaced.
