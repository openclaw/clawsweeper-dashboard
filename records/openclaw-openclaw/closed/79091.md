## Summary

Keep open: current main has manual split-brain troubleshooting and destructive-action protection, but the read-time config warning and gateway status path still do not automatically identify a stale local CLI wrapper while the gateway is already healthy on the newer version.

## What This Changes

Request to make CLI/gateway install skew visible by adding stale-wrapper hints to version-mismatch warnings and status/doctor/update diagnostics.

## Best Possible Solution

Add non-mutating diagnostics that make stale CLI-wrapper skew visible in the warning/status/doctor surfaces while leaving package-manager root and overwrite policy to the related updater issues.

## Reproduction Assessment

Yes at source level, not as a live current-main run: a config last touched by a newer version still produces a generic warning, and `gateway status` currently drops the probed server version needed for an automatic comparison. The full dual-global-install setup still needs a real host to reproduce end to end.

## Solution Assessment

Yes if scoped to diagnostics: add stale-wrapper/PATH hints and expose enough CLI, service, and gateway version/path data to identify skew. Broadly changing updater overwrite or npm-prefix behavior would be less targeted here and overlaps with https://github.com/openclaw/openclaw/issues/79087 and https://github.com/openclaw/openclaw/issues/65130.
