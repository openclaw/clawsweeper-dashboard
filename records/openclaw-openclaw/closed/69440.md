## Summary

Current main addresses the reported startup downgrade: dispatched replies load runtime plugins before thinking validation, the OpenAI Codex provider profile exposes xhigh for gpt-5.4, and fallback validation no longer persists a downgrade unless it is remapping an existing explicit session override. Related PRs #71541, #71543, and #71557 identified the same surface but were closed unmerged; the relevant safeguards are present on main.

## Best Possible Solution

Keep the current main implementation: provider-owned thinking profiles are activated before dispatched reply validation, OpenAI Codex gpt-5.4 advertises xhigh support through the plugin profile, and fallback remapping avoids persisting default-derived downgrades. Any future remaining issue should be a narrower regression report with a current cold-start reproduction path.
