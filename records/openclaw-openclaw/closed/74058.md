## Summary

Close as duplicate of the already-open canonical tracker #62364. The new report is useful and more specific about bundled runtime dependency installs and pricing timeouts, but the remaining actionable problem is the same Gateway/model-catalog hot path: `models.list` can still synchronously wait on full catalog/provider work. Current main has some CLI/provider-list fast paths, but the Gateway RPC path is not fully fixed, so this should not close as implemented.

## Best Possible Solution

Close this issue as a duplicate of #62364 and keep #62364 as the canonical tracker for the Gateway model-catalog latency fix. The fix should make `models.list` avoid full provider/plugin startup for configured/default rows, time-box or background slow provider catalog work, and keep dependency staging/pricing refresh off the request path.
