## Summary

Current main still does not implement this PR's intended root `openclaw/plugin-sdk` compatibility export. The reported external-plugin breakage is supported by the provided discussion, and this PR remains a plausible implementation candidate, but it is incomplete for the current contract/API-baseline shape and may need to cover the loader compat/root-alias path as well as `src/plugin-sdk/index.ts`.

## Best Possible Solution

Keep this PR open as an implementation candidate. If maintainers want root-entry backward compatibility, update it to cover the actual root import paths used by plugins, update the exact root export contract, regenerate the Plugin SDK API baseline, and add any small docs/changelog note needed for the promoted compatibility surface. If maintainers reject widening the deprecated root SDK, keep #68279 as the canonical migration/compat tracking item and close this PR only after that decision is documented.

## Risks / Open Questions

- As submitted, this PR appears incomplete for current main because the root SDK contract test and generated Plugin SDK API baseline would need to be updated if the root runtime surface is intentionally widened.
- An index-only re-export may not fix all affected runtime paths because source plugin loading aliases the root SDK through `root-alias.cjs` and the compat surface, which currently also lacks these helpers.
- There is a product-boundary decision here: docs and guardrails prefer focused subpaths, while the provided reports show backward-compatibility pressure for the deprecated root surface.
