## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Validate heartbeat intervals before scheduling:** `src/gateway/http-common.ts:147`
  - body: `Math.max(1000, options?.intervalMs ?? 30_000)` still lets `NaN`, `Infinity`, and `-Infinity` reach `setInterval`; Node normalizes those to a 1ms timer, so a malformed future override can create a high-frequency heartbeat per open stream. Normalize with `Number.isFinite` before clamping and fall back to the default when invalid.
  - confidence: 0.91
- **[P3] Restore real timers after heartbeat tests:** `src/gateway/http-common.test.ts:302-304`
  - body: This suite enables fake timers in `beforeEach` but never restores real timers. In shared-worker or `--isolate=false` runs, later tests can inherit fake timers and stop exercising real timer behavior, so clear pending timers and call `vi.useRealTimers()` in cleanup.
  - confidence: 0.72
