## Summary

Current `main` already implements the requested configurability and no longer treats quiet WhatsApp inbound silence as stale-socket evidence.

## Evidence

- **Config schema exposes a stale-threshold setting:** `gateway.channelStaleEventThresholdMinutes` is part of the gateway config schema and is validated alongside the health-check interval.
  - file: [src/config/zod-schema.ts:747](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/src/config/zod-schema.ts#L747)
  - sha: [40be5ad58187](https://github.com/openclaw/openclaw/commit/40be5ad58187fd20b8defe5fc30e649190c131a8)
- **Gateway runtime wires the config into the monitor:** `startGatewayChannelHealthMonitor` reads `cfg.gateway.channelStaleEventThresholdMinutes` and passes it to `startChannelHealthMonitor` as `staleEventThresholdMs`.
  - file: [src/gateway/server-runtime-services.ts:36](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/src/gateway/server-runtime-services.ts#L36)
  - sha: [40be5ad58187](https://github.com/openclaw/openclaw/commit/40be5ad58187fd20b8defe5fc30e649190c131a8)
- **Health policy no longer uses app-event freshness as socket liveness:** The health evaluator only considers stale-socket failures when `lastTransportActivityAt` is present, and the tests explicitly verify that stale app events without transport activity are ignored.
  - file: [src/gateway/channel-health-policy.ts:117](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/src/gateway/channel-health-policy.ts#L117)
  - sha: [40be5ad58187](https://github.com/openclaw/openclaw/commit/40be5ad58187fd20b8defe5fc30e649190c131a8)
- **WhatsApp snapshots do not publish transport-liveness timestamps:** The WhatsApp channel snapshot includes `lastInboundAt`, `lastMessageAt`, and `lastEventAt`, but not `lastTransportActivityAt`, so a quiet connected WhatsApp account is not classified stale by the current health policy. This matches the changelog note that stale-socket recovery now keys off provider-proven transport activity instead of inbound app-event freshness.
  - file: [extensions/whatsapp/src/channel.ts:243](https://github.com/openclaw/openclaw/blob/40be5ad58187fd20b8defe5fc30e649190c131a8/extensions/whatsapp/src/channel.ts#L243)
