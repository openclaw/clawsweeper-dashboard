## Summary

Keep this PR open. Current main still has the three user-visible behaviors the PR targets: suppressed group delivery drops TTS/tool voice media, OpenAI-compatible PCM speech responses are returned as raw requested format without WAV normalization, and WhatsApp PTT voice sends still duplicate visible text. The provided PR discussion and diff make this an active implementation candidate rather than obsolete cleanup.

## Best Possible Solution

Continue reviewing this PR as the implementation candidate. The best path is to land a hardened trusted-local-media voice exception with regression tests for final and pending tool media, keep untrusted/remote media suppressed, wrap valid declared PCM into a deliverable WAV container, remove duplicate WhatsApp voice-note text sends, and update the WhatsApp/TTS docs for accepted behavior changes.
