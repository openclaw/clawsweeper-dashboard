## Summary

Current main already avoids refreshing usable OAuth credentials, limits `probe-*` sessions to explicit CLI probe flows, and no longer surfaces raw OAuth refresh errors to external channel users; the latest release also contains that behavior.

## Evidence

- **Valid OAuth credentials short-circuit before refresh:** `resolveOAuthAccess` returns the existing effective OAuth credential when `hasUsableOAuthCredential(effectiveCredential)` is true, so a still-valid access token is not refreshed on the message path.
  - file: [src/agents/auth-profiles/oauth-manager.ts:505](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/agents/auth-profiles/oauth-manager.ts#L505)

  - sha: [a155b84cdade](https://github.com/openclaw/openclaw/commit/a155b84cdadef068de50274318952032cd867788)
- **Usable means unexpired and outside the refresh margin:** `hasUsableOAuthCredential` only returns false for missing access tokens or tokens that are expired/within the refresh margin; otherwise the stored OAuth credential remains usable.
  - file: [src/agents/auth-profiles/credential-state.ts:42](https://github.com/openclaw/openclaw/blob/a155b84cdadef068de50274318952032cd867788/src/agents/auth-profiles/credential-state.ts#L42)
