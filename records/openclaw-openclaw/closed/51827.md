## Summary

Current `main` already covers the concrete delivery failures described in this umbrella issue: async completion routing now preserves session delivery context, announce target resolution hydrates account/thread data from session state instead of defaulting, transcript mirroring deduplicates duplicate `delivery-mirror` writes, and Telegram target parsing accepts `telegram:group:<chatId>:topic:<id>` forms.

## Evidence
