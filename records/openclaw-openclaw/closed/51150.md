## Summary

Keep open. Current `main` still resolves image models through a bare registry lookup, so the reported custom-provider `Unknown model` path is not implemented on `main` or the latest shipped release. The PR also has active superseding work (`#71458`) and recent user reports confirming the bug still exists, so conservative cleanup should not close it.
