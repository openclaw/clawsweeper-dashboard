## Summary

Current `main` already supports `plugins.entries.memory-core.config.dreaming`, the Dreaming UI checks runtime schema before patching, and the same implementation is present in shipped release `v2026.4.23`.

## Evidence

- **memory-core manifest schema accepts dreaming config on current main:** `extensions/memory-core/openclaw.plugin.json` defines `configSchema.properties.dreaming` under `plugins.entries.memory-core.config`, including `enabled`, `frequency`, and nested dreaming settings.
  - file: [extensions/memory-core/openclaw.plugin.json:21](https://github.com/openclaw/openclaw/blob/fe930b987e33f4e1acd93b5ef1ea76c8b7260047/extensions/memory-core/openclaw.plugin.json#L21)
  - command: `nl -ba extensions/memory-core/openclaw.plugin.json | sed -n '1,120p'`
  - sha: [fe930b987e33](https://github.com/openclaw/openclaw/commit/fe930b987e33f4e1acd93b5ef1ea76c8b7260047)
- **Dreaming UI guards unsupported runtimes before writing the patch:** `updateDreamingEnabled` calls `ensureDreamingPathSupported`, which queries `config.schema.lookup` for `plugins.entries.<pluginId>.config` and refuses to patch when the schema disallows unknown properties and does not expose `dreaming`.
  - file: [ui/src/ui/controllers/dreaming.ts:978](https://github.com/openclaw/openclaw/blob/fe930b987e33f4e1acd93b5ef1ea76c8b7260047/ui/src/ui/controllers/dreaming.ts#L978)

  - file: [ui/src/ui/controllers/dreaming.test.ts:652](https://github.com/openclaw/openclaw/blob/fe930b987e33f4e1acd93b5ef1ea76c8b7260047/ui/src/ui/controllers/dreaming.test.ts#L652)
