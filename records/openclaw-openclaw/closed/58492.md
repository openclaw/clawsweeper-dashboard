## Summary

Close as ClawHub/plugin-track. The PR hard-codes `cli-orchestrator` prompt behavior into core, but current OpenClaw policy and existing skill/bundle surfaces already support putting that activation and clarification guidance in the owning skill or bundle command instead. The PR branch is also not mergeable as described in review comments because it contains unresolved conflict markers in production TypeScript.
