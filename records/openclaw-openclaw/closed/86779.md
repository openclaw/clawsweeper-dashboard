Keep the persisted-config rule strict: hand-written legacy `tools.web.search.<provider>` objects should still be rejected and migrated by doctor. Do not add a new config option, do not relax blocked object-key validation, and do not solve the broader generated-schema issue from https://github.com/openclaw/openclaw/issues/80536. Prefer a narrow fix around `mergeScopedSearchConfig` or the runtime validation boundary, plus regression coverage for canonical `tools.web.search.provider` with plugin-owned `webSearch`, disabled unrelated provider plugins, and the existing literal legacy rejection path. No CHANGELOG.md edit.
