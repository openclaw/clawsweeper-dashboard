- `impact:other`: The issue affects machine-readable CLI automation rather than message delivery, session state, auth, security, or availability.

## Summary

Keep open: the exact `sessions --json` path is route-first on current main, but the central bug remains because global startup progress can still start for non-routed `--json` commands before Commander parses and emits JSON.

## What This Changes

The issue asks for Clack startup/progress UI to be suppressed for machine-readable `--json` CLI invocations so stdout stays valid JSON.

## Best Possible Solution

Disable startup/progress UI centrally for JSON-output invocations and make spinner creation require both the intended progress stream and stdout to be TTY-safe; leave the optional config knob for separate product review.

## Reproduction Assessment

Yes, at source level: a non-routed `--json` command such as `openclaw mcp list --json` reaches the unguarded startup progress path, and Clack's spinner defaults to stdout. The exact `sessions --json` example is partly mitigated on current main by the route-first sessions path, so the remaining repro target is the central progress handling.

## Solution Assessment

Yes, the best fix is central progress suppression for machine-readable invocations, plus a stdout-safety check around Clack spinner use. The optional global progress config should not be bundled into the narrow bug fix without maintainer product approval.
