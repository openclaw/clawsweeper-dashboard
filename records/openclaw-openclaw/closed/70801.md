## Summary

## What This Changes

The issue requests changing macOS LaunchAgent restart ordering so the old gateway is stopped and confirmed gone before a new LaunchAgent instance starts.

## Best Possible Solution

Keep the current pre-kickstart port-postcondition fix; any future switch back to a bootout/bootstrap restart contract should be a separate owner decision backed by new macOS evidence.

## Reproduction Assessment

No, not against current main. Source and regression tests now show restart aborts before kickstart when the resolved gateway port remains busy, so the previously reported race path is not present in the current implementation.

## Solution Assessment

Yes. The merged fix addresses the user-visible port race with a narrower postcondition before kickstart, avoiding the bootout unload regressions that led to the current launchd restart contract.
