Keep this PR open for maintainer handling, but do not merge as-is. Split this contribution into small maintainer-owned decisions: keep only CI capabilities OpenClaw wants in core, remove or relocate the agent-dna-auditor bundle, and harden any accepted workflow to match current trusted-event, fork/secret, pinned-action, repo-wrapper, pipefail, and issue/PR filtering patterns.

## Security Review

Status: needs_attention

Summary: The PR introduces secret-bearing and write-token GitHub Actions workflows with unpinned third-party action refs and insufficient trusted-event, fork, and issue/PR guards.

Concerns:

- **[high] Unpinned third-party action receives secrets:** `.github/workflows/codex-pr-review.yml:42`
  - body: openai/codex-action@v1 is used in workflows that receive OPENAI_API_KEY, with the PR review workflow also holding PR write permission. Current main pins Codex automation to a full commit SHA; these workflows should do the same before secrets are passed to the action.
  - confidence: 0.9
- **[high] Autofix write token is not limited to trusted main:** `.github/workflows/codex-autofix.yml:22`
  - body: The workflow_run job grants contents and pull-requests write but gates only on failed push CI, not the repository, main branch, exact current main SHA, actor class, or secret availability. That is broader than the PR description and weaker than the hardened main-only Codex workflow pattern in current main.
  - confidence: 0.92
- **[medium] PR workflows can fail or misbehave for forks:** `.github/workflows/codex-pr-review.yml:25`
  - body: The AI review and bundle-size workflows run on pull_request while depending on unavailable secrets or write-token comment permissions. Without fork/permission guards, external contributor PRs can get red checks instead of skipped optional automation.
  - confidence: 0.86
- **[medium] Dead-code issue mutation can affect PRs:** `.github/workflows/dead-code-sweep.yml:88`
  - body: The dead-code sweep uses issue APIs to update or close matching open items without filtering pull requests returned by issue listing endpoints. A PR labeled dead-code could have its title/body overwritten or be closed by the sweep.
  - confidence: 0.87
