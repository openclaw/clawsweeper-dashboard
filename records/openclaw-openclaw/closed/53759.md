## Summary

Close #53759 as a duplicate of the still-open macOS node `system.run.prepare` reports, especially #57586 and #58356. Current main still has the same root mismatch: gateway node exec prepares with `system.run.prepare`, while the macOS companion advertises and handles `system.run`/`system.which` but not `system.run.prepare`.

## Best Possible Solution

Close #53759 as a duplicate and keep the root macOS node exec bug consolidated on #57586/#58356. The product fix should happen in the canonical issue: either add and advertise macOS companion support for `system.run.prepare`, or adjust the gateway/node exec contract so the gateway does not require a prepare command that the node cannot implement, with coverage for `exec host=node` and remote macOS skill execution.
