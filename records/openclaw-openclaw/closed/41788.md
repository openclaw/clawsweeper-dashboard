## Summary

Close #41788 as superseded by open PR #68967. Current main has some Google Chat thread support, but it is not a complete implemented-on-main close: inbound stores `message.thread?.name` as `ReplyToId`, the typing placeholder is sent in that thread, and the Chat API sender can include `thread.name` plus `messageReplyOption`; however session metadata only persists `MessageThreadId`, and Google Chat does not currently populate that field. The active PR #68967 directly tracks the remaining fix, explicitly says it closes #41788, and includes real Google Chat verification for replies landing in the inbound thread.

## Best Possible Solution

Keep the remaining Google Chat thread-continuity work centralized in #68967, where the implementation, tests, docs, and live Google Chat verification are already being reviewed. Do not close this as implemented on main until that PR or an equivalent fix lands.

## Evidence
