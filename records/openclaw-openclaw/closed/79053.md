## What This Changes

The issue reports dropped subagent completion announcements when an idle parent session has an active but non-consuming embedded Pi run.

## Best Possible Solution

Land a narrow fix through https://github.com/openclaw/openclaw/pull/79059 that lets active-but-non-consuming requesters reach the requester-agent handoff or queue fallback without raw direct sends, then close this issue from the merge.
