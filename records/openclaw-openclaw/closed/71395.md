## Summary

Current main already supports silent isolated cron outcomes via the built-in exact token `NO_REPLY` / `no_reply`, suppressing both direct delivery and queued fallback summaries for no-op monitoring runs.

## Evidence

- **Silent reply contract is documented:** The docs define the exact token `NO_REPLY` / `no_reply` as meaning 'do not deliver a user-visible reply.'
  - file: [docs/concepts/messages.md:156](https://github.com/openclaw/openclaw/blob/ec8dbc4595583816f71af2568df61dbe7c090e1f/docs/concepts/messages.md#L156)
