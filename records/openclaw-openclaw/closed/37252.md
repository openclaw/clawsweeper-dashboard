## Summary

Close as implemented on `main`: Telegram outbound no longer permissively parses partially numeric `replyTo` values, the outbound entry points route through the strict parser, and the shipped `v2026.4.22` release contains the same implementation and regression test.

## Evidence

- **Strict parser now gates outbound reply IDs before conversion:** `parseTelegramReplyToMessageId()` delegates to `normalizeTelegramReplyToMessageId()`, which trims the string and rejects anything that does not match the full integer regex before calling `Number.parseInt()`. That means `"123abc"` is no longer accepted.
