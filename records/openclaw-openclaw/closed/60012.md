## Summary

Close as `cannot_reproduce`. On current `main`, and in the latest shipped release commit `a9797214338ba31c52c796adbb75afb16e0684a9` (`v2026.4.23`), the Control UI Debug view still calls `status` as a Gateway WebSocket RPC, the browser client sends RPC frames over the open socket rather than fetching `/api/v1/status`, and the gateway still registers and serves the `status` method with coverage asserting it succeeds. The report’s claimed `status -> GET /api/v1/status` mapping does not match the current or latest shipped code.

## Evidence

- **Control UI Debug still calls `status` as RPC:** The Debug controller issues `state.client.request("status", {})` alongside `health`, `models.list`, and `last-heartbeat`; it does not fetch an HTTP `/api/v1/status` route.
