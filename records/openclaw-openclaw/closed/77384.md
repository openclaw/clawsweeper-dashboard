## Summary

Current `main` and the v2026.5.3 release already route registered plugin commands through the local command handler before an agent run, so the reported v2026.5.2 regression is addressed by upgrading.

## What This Changes

The issue reports that plugin-defined slash commands that previously short-circuited Telegram/Signal routing are now being sent into the LLM turn after upgrading from v2026.4.23 to v2026.5.2.

## Best Possible Solution

Keep the existing plugin-command pipeline and have affected v2026.5.2 users upgrade to v2026.5.3 or newer; any remaining external plugin failure should be reported with its `api.registerCommand({ name, ... })` registration and activation logs.

## Reproduction Assessment

No. I did not find a current-main failure path: current source and regression tests show plugin commands are matched and executed before agent dispatch on text and native command surfaces.

## Solution Assessment

Yes. The existing generic plugin command registry and first-position handler are the narrow maintainable fix; this does not need a new routing-layer API or config option.
