## Summary

Close as ClawHub/plugin work. The request is a Perplexity-specific history importer for memory-wiki, not a core OpenClaw behavior gap. Current main already has a Perplexity plugin for web search only, ChatGPT-specific memory-wiki import code, documented memory-wiki source/apply/bridge surfaces, and plugin APIs for external CLI/tool/gateway implementations. A Perplexity export parser can live as a separate ClawHub plugin or skill without bundling service-specific import logic in core.

## Best Possible Solution

Close this core issue and direct the work to ClawHub as a Perplexity history importer plugin or skill that parses user-provided exports into memory-wiki source pages via documented wiki ingest/apply workflows or plugin CLI/tool/gateway APIs. If authors later find a missing generic Dreaming or import-insights extension point, that should be opened as a narrow plugin API proposal rather than bundling Perplexity parsing in core.
