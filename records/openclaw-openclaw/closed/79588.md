## Summary

Keep open. Current main still seeds the compaction quality guard's identifier audit from a last-10, text-only message view, so opaque identifiers that appear only in tool-call arguments or earlier/pruned transcript content are not reliably checked; the linked implementation PR is still open and unmerged.

## What This Changes

Request that the compaction quality guard verify opaque identifiers from original messages, tool-call arguments, and tool results survive into strict post-compaction summaries.

## Best Possible Solution

Land or replace a reviewed compaction-safeguard fix that seeds identifiers from the full pre-compaction input, excludes stripped `toolResult.details`, validates the final returned summary, and adds regression plus real-behavior proof.

## Reproduction Assessment

Yes. Source inspection shows an opaque identifier that exists only in assistant tool-call arguments is not included in `identifierSeedText`, because current main extracts only string/text-block content from the last 10 sampled messages before auditing.

## Solution Assessment

Yes. The best path is to keep this issue open while https://github.com/openclaw/openclaw/pull/75336 is reviewed, fixed, landed, or replaced; closing the issue before that PR merges would preempt the active implementation path.
