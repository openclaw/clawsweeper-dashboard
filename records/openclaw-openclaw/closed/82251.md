## Summary

Current main already defers heartbeat wakes when the target session lane is busy and retries them later, so the reported interruption path is already covered by the runtime and regression tests.

## What This Changes

The issue asks for heartbeat polls to wait behind an active model response instead of interrupting the in-progress turn.

## Best Possible Solution

Keep the existing heartbeat runner and wake-layer deferral model, which serializes heartbeat work behind active session turns and retries without adding a new user-facing mode.

## Reproduction Assessment

No high-confidence failing current-main reproduction remains from this review. Source and regression tests show active session-lane work makes heartbeat return requests-in-flight and retry instead of dispatching a model turn.

## Solution Assessment

Yes. The current fix is at the runtime scheduling boundary, so all heartbeat wake sources share the same session-lane guard instead of each caller adding its own queueing behavior.
