## Summary

Close as a duplicate of the older open Windows installer tracker #38054. Current main still has the reported `install.ps1` hard-exit path, so this is not implemented; the work should stay consolidated on #38054 and the related open PRs rather than split across duplicate issues. The WebClaw comment is a useful third-party workaround, but it does not resolve the native OpenClaw installer bug.

## Best Possible Solution

Keep #38054 as the canonical tracker. The eventual fix should change `scripts/install.ps1` so failure paths inside `Main` return or throw through a guarded top-level wrapper when invoked via `iwr | iex`, while preserving non-zero exit codes for direct `powershell -File` or `pwsh -File` execution. Use #41931/#48983 as references, but land only with regression coverage or manual Windows evidence.
