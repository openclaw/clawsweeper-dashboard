## Summary

Keep open: this is a useful feature PR, but the current patch is not correct because the cron providers field is not accepted by the gateway protocol/normalization path, agent-only overrides miss providerConfig-free models, job overrides are dropped for default/no-list cron jobs, update helpers drop providers, and the Canvas hash change is unrelated.

## What This Changes

The PR adds per-agent and per-cron provider request override types/resolution, threads agent IDs into model resolution, adds provider/model tests, and modifies a Canvas A2UI bundle hash.

## Best Possible Solution

Implement the override contract end-to-end across model resolution, cron protocol/add/update/store paths, redaction/docs/tests, and remove unrelated generated output.

## Reproduction Assessment

Yes for the review defects: source inspection of PR head and current main shows the protocol rejection, providerConfig early return, agents.list-dependent cron merge, and missing update persistence. There is no current-main bug reproduction because this PR adds a new feature surface.

## Solution Assessment

No. Mutating cron agents.list and leaving cron protocol/update/redaction paths incomplete is not the narrow maintainable solution; the safer path is to pass resolved request overrides as prepared runtime facts and update the public contract consistently.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Apply agent request overrides before the providerConfig return:** `src/agents/pi-embedded-runner/model.ts:559-564`
  - body: The new agent/job request merge runs only after the no-providerConfig return, so a discovered provider with only agents.list[].providers.<id>.request skips the override entirely unless users also define a global models.providers.<id> block.
  - confidence: 0.91
- **[P2] Add providers to the cron protocol and normalization path:** `src/cron/types.ts:168-180`
  - body: The PR advertises payload.providers, but cron.add and cron.update still validate agentTurn payloads against a schema with additionalProperties: false and no providers field. API/CLI callers cannot create or update jobs with this field, so the cron feature is unreachable through the gateway contract.
  - confidence: 0.9
- **[P2] Pass job request overrides without requiring an agents.list row:** `src/cron/isolated-agent/run.ts:496-500`
  - body: This code merges job providers by patching a matching cfg.agents.list entry. Default-agent cron jobs with no list row, or jobs resolved from defaults/runtime config without a matching row, silently drop the job-level request override.
  - confidence: 0.88
- **[P2] Persist and clear cron payload providers on updates:** `src/cron/types.ts:178-180`
  - body: The type now allows payload.providers and null clearing, but the cron payload merge/build helpers do not copy, merge, or delete providers. Once the protocol accepts the field, update paths can still drop it at runtime.
  - confidence: 0.84
- **[P3] Remove the unrelated Canvas bundle hash change:** `extensions/canvas/src/host/a2ui/.bundle.hash:1`
  - body: This hash-only Canvas generated file change is unrelated to provider request headers and has no matching Canvas source or generated bundle change in the PR.
  - confidence: 0.86
