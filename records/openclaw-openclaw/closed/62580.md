## Summary

Keep open. Current `main` has added hardening around malformed/corrupted Kimi/OpenAI-compatible tool streams and documents the known shared-DM session leakage risk, but I did not find strong evidence that this mixed-language/cross-session contamination report itself is fixed on `main`. The related side-effect report referenced in discussion (#63617) also points to an unresolved class of failures rather than an obsolete one.

## Evidence

- **Shared-DM leakage is documented as a current risk, not a closed bug:** The security audit and docs still warn that when `session.dmScope` is `main`, multiple DM senders can share one session and leak context across users; the remediation is configuration (`per-channel-peer` / `per-account-channel-peer`), not evidence of a core fix for this report.
