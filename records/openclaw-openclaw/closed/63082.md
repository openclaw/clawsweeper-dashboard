## Summary

Keep open: this PR targets a real bug linked to #63069, but the current patch adds the environment-aware resolver in the generator module while the actual install/check/update/doctor runtime path still resolves hardcoded profile files.

## What This Changes

This PR adds a shell profile resolver and resolver tests in `src/cli/completion-cli.ts` and changes doctor completion messages to print that resolved path.

## Best Possible Solution

Move the profile resolver into `src/cli/completion-runtime.ts`, share it across install/check/slow-pattern detection and user-facing hints, preserve completion script/cache behavior, and cover zsh, fish, bash fallback, and PowerShell path cases with focused tests plus a changelog entry.

## Reproduction Assessment

Yes. A code-level reproduction is high-confidence: with `ZDOTDIR` or `XDG_CONFIG_HOME` set, the runtime installer/checker on current main still uses `$HOME`-based zsh/fish paths, and this PR does not change that runtime owner.

## Solution Assessment

No. The requested direction is right, but this patch puts the resolver in the wrong module and leaves the actual write/check surfaces on their old path logic; a shared runtime resolver is the narrower maintainable fix.
