The issue requests preserving DeepSeek `reasoning_content` through `opencode-native` proxy providers so tool-call follow-up requests do not fail after reasoning is stripped.

## Best Possible Solution

Keep the shipped suffix-normalized replay path and explicit compat opt-in; require fresh live/catalog evidence before adding any further global proxy aliases.

## Reproduction Assessment

No current failing reproduction remains: current main preserves `reasoning_content` for the reported OpenCode Zen `deepseek-v4-flash-free` path, and the old failure is covered by source regression plus an OpenCode live replay probe.

## Solution Assessment

Yes. The shipped fix is narrower than marking every `opencode-native` route as DeepSeek, and the compat flag covers custom proxy cases without hard-coding unproven aliases such as `big-pickle`.
