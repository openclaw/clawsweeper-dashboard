## Summary

Close. The remaining first-party work is already tracked by the open same-product migration issue and the open native backup/restore umbrella; current main has backup create/verify, provider-based imports, and ClawHub/plugin bundle distribution surfaces, so keeping this broad discussion open would split the same product decision.

## What This Changes

Request a standardized agent packaging and migration model that exports stable shareable agent files while excluding runtime state, logs, secrets, and machine-specific paths.

## Best Possible Solution

Consolidate first-party migration and restore design under https://github.com/openclaw/openclaw/issues/45993 and https://github.com/openclaw/openclaw/issues/13616, while using ClawHub/plugin bundles for distributable skills/plugins until maintainers approve any new agent-package format.
