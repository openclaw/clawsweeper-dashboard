## Summary

Keep open: the PR is still necessary because current main and the latest release still pass OpenClaw-wrapped note text to Clack with the real terminal width, letting Clack 1.4.0 hard-wrap copy-sensitive paths a second time. The patch is a narrow shared-terminal fix, the linked issue remains open, and I found no blocking diff defect.

## What This Changes

The branch changes `packages/terminal-core/src/note.ts` so `note()` wraps once with OpenClaw, then calls Clack with a widened virtual output stream to avoid Clack re-wrapping copy-sensitive tokens.

## Best Possible Solution

Land the shared terminal-core note rendering fix once maintainers are satisfied with the focused proof and any desired rendered-output regression coverage, then let the linked issue close through the PR merge.

## Reproduction Assessment

Yes for source-level reproduction: current main routes session-lock paths through the shared `note()` helper, and pinned Clack 1.4.0 hard-wraps note content using the output columns. I did not run a full `openclaw doctor` scenario in this read-only checkout.

## Solution Assessment

Yes: the shared terminal-core note helper is the right layer because doctor, setup, and provider notes all funnel through it; a session-lock-only workaround would leave the same second-wrap behavior elsewhere.
