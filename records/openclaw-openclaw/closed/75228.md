## Summary

Keep open: the PR is a plausible default-strict fix for https://github.com/openclaw/openclaw/issues/74848, but it still needs real macOS behavior proof, config-contract registration, and maintainer/security acceptance of system-trust-based stale-pin repair. Current main still keeps arbitrary public WSS hosts strict, and https://github.com/openclaw/openclaw/pull/81609 explicitly left this as a separate stale-pin repair-policy decision.

## What This Changes

The PR adds a default-off macOS `gateway.remote.allowSystemTrustedPinRepair` flag to auto-repair stale remote WSS TLS pins when macOS system trust accepts the new chain, plus Swift tests and a changelog entry.

## Best Possible Solution

If maintainers accept the opt-in policy, register and document the flag in the canonical config contract, keep defaults strict, and merge only after redacted real macOS runtime proof; otherwise keep the documented `tlsFingerprint` or Remote over SSH recovery path.

## Reproduction Assessment

Yes for source-level reproduction: current main has a trusted public WSS pin-mismatch test that is not auto-repaired, and the TLS delegate cancels stored-pin mismatches. A live reproduction needs a macOS app install with a stale stored pin and a rotated public certificate.

## Solution Assessment

No. A default-off opt-in is a plausible direction, but this patch is incomplete until the flag is registered across the config contract and maintainers accept the TLS trust boundary.
