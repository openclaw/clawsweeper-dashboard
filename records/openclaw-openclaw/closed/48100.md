- **[P1] Require conversation access for completion hooks:** `src/plugins/types.ts:1548`
  - body: `response` exposes the assistant's raw answer, but this hook is only added to `PROMPT_INJECTION_HOOK_NAMES`, not the conversation hook list enforced by `registry.ts`. A non-bundled plugin could register it without `plugins.entries.<id>.hooks.allowConversationAccess=true`, bypassing the existing raw-conversation access boundary used for `llm_output`, `before_agent_finalize`, and `agent_end`.
  - confidence: 0.91
- **[P1] Keep suppressed replies out of session history:** `src/auto-reply/reply/agent-runner-hooks.ts:86-87`
  - body: By the time this branch returns an empty final payload for `suppress`, `runAgentTurnWithFallback` has already completed the embedded turn and session snapshot work. That hides the reply from the user while leaving the unseen assistant answer in transcript/context state, so the next turn can be conditioned on content the user never received.
  - confidence: 0.87
- **[P1] Defer delivery before running the completion hook:** `src/auto-reply/reply/agent-runner-hooks.ts:46-50`
  - body: The wrapper calls `runAgentTurnWithFallback` before the hook verdict, but block/partial streaming can enqueue or send chunks during that call through the existing reply-delivery pipeline. A later `suppress` or `reinject` result cannot reliably honor the PR's pre-delivery contract and can leak stale first-attempt content.
  - confidence: 0.86
