## Summary

Keep open. Current `main` still allows an isolated cron with an explicit parent `sessionKey` like `agent:main:main` to reuse and persist back to that same store key, which can overwrite the parent session entry including `updatedAt`. The heartbeat path also still has no general session-freshness check; it only restores `updatedAt` on a few reply-shape branches, so heartbeat runs can still keep a stale main session looking fresh.

## Evidence

- **Explicit parent session keys are still preserved for cron runs:** `resolveCronAgentSessionKey()` keeps a canonical key like `agent:main:main` unchanged instead of forcing a cron-namespaced key, and the unit test explicitly asserts that behavior.
  - file: [src/cron/isolated-agent/session-key.ts:11](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/cron/isolated-agent/session-key.ts#L11)
  - command: `nl -ba src/cron/isolated-agent/session-key.ts | sed -n '1,24p'; nl -ba src/cron/isolated-agent/run.session-key.test.ts | sed -n '1,16p'`
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **Isolated cron namespacing only happens for default `cron:<jobId>` keys:** `run.ts` derives `baseSessionKey` from `input.sessionKey` when present. It only appends `:run:<sessionId>` when `baseSessionKey.startsWith("cron:")`; otherwise `runSessionKey` is just `agentSessionKey`. For an explicit `agent:main:main`, the run key stays the parent key.
  - file: [src/cron/isolated-agent/run.ts:459](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/cron/isolated-agent/run.ts#L459)
  - command: `nl -ba src/cron/isolated-agent/run.ts | sed -n '459,500p'`
  - sha: [6944d7025d9c](https://github.com/openclaw/openclaw/commit/6944d7025d9c0ba4891cbe646ab5dab039d504ca)
- **Persist step still overwrites the parent store entry:** `createPersistCronSessionEntry()` writes `cronSession.sessionEntry` directly to `store[agentSessionKey]`, and only writes a separate run key when `runSessionKey !== agentSessionKey`. With an explicit parent key, that means the parent session entry is replaced in-place.
  - file: [src/cron/isolated-agent/run-session-state.ts:33](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/cron/isolated-agent/run-session-state.ts#L33)

- **Heartbeat still lacks a general stale-session guard:** `src/infra/heartbeat-runner.ts` contains no `evaluateSessionFreshness` or `resolveSessionResetPolicy` call. It only restores `updatedAt` in four specific branches (`ok-empty`, `ok-token`, `duplicate`, `alerts-disabled`), so normal heartbeat deliveries still leave the touched `updatedAt` in place.
  - file: [src/infra/heartbeat-runner.ts:447](https://github.com/openclaw/openclaw/blob/6944d7025d9c0ba4891cbe646ab5dab039d504ca/src/infra/heartbeat-runner.ts#L447)
