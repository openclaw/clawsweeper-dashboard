## What This Changes

The branch adds a new bundled `extensions/session-search` Control UI plugin with transcript search/detail views, role filters, Show/Resume context injection, tests, package metadata, and a lockfile importer.

## Best Possible Solution

No high-confidence runtime reproduction was established for this feature PR. The merge blockers are source-verifiable from the branch diff: the plugin is bundled under `extensions/session-search`, imports core internals, and exposes mutating session actions without a separate write-scope gate.

## Solution Assessment

No. The maintainable direction is external plugin distribution through ClawHub plus narrow SDK seam work where needed, not a large optional bundled plugin that bypasses the extension boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require write-scoped authorization for session mutation:** `extensions/session-search/index.ts:21-27`
  - body: The entry point is advertised with only `operator.read`, but the same gateway-auth prefix route handles `POST /show-session`, creates resumed sessions, appends transcript markers, and queues prompt context. Since those writes are performed directly through runtime/session helpers, a read-scoped Control UI surface can become a write-capable session mutation path; split the mutating action behind an explicit write/admin check or a scoped host action before merging.
  - confidence: 0.86
- **[P2] Use public SDK seams instead of core internals:** `extensions/session-search/src/page.ts:4-10`
  - body: This plugin imports multiple `../../../src/**` modules from production code, which violates the extension boundary and makes the plugin unsuitable for external package/ClawHub distribution. Replace these with documented `openclaw/plugin-sdk/*` subpaths, or add a focused SDK seam first for any missing transcript/bootstrap behavior.
  - confidence: 0.93
