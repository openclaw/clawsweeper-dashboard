## Summary

Keep open. Current main still lacks Discord-specific Cloudflare/Error 1015 HTML 429 classification, fallback cooldown, and non-JSON body sanitization in the REST startup and allowlist paths, and #73395 is an open unmerged PR with closing syntax for this issue.

## What This Changes

The issue requests Discord channel reconnect hardening so Cloudflare HTML 429/Error 1015 responses trigger cooldown/backoff, sanitized logging, and recovery instead of repeated REST retries.

## Best Possible Solution

Add the fix in the Discord plugin REST startup/allowlist surface: classify Cloudflare/Error 1015 and other non-JSON HTML HTTP 429 responses, summarize non-JSON bodies before logging or throwing, honor `Retry-After` when present, apply a conservative fallback cooldown when absent, and cover `/users/@me/guilds` plus `/oauth2/applications/@me` with focused tests.
