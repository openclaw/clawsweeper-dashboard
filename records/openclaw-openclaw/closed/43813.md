## Summary

Current `main` already skips `node_modules`/`dist` during plugin discovery, applies that filter before recursion, and has regression tests covering nested dependency-tree scans. The same implementation is present in the tagged release `v2026.4.22`.

## Close Comment

Closing from a Codex review because the reported plugin-discovery recursion into dependency trees is already implemented on current `main` and shipped in `v2026.4.22`. In `src/plugins/discovery.ts:26`, the scanner now ignores `node_modules` and other build metadata directories; `src/plugins/discovery.ts:326` applies that filter, and `src/plugins/discovery.ts:879` checks it before recursing. Regression coverage in `src/plugins/discovery.test.ts:514` and `src/plugins/discovery.test.ts:556` verifies nested `node_modules` copies and workspace `dist`/`node_modules` trees are not discovered as plugins. I also verified the same code is present in the tagged release commit `00bd2cf7a376f1fba26291c6c4766f1f15cbdfa5` (`v2026.4.22`). If you are still seeing FD exhaustion on a newer repro path outside skipped dependency trees, please open a fresh issue with that exact layout and paths.
