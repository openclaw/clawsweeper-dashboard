## Summary

The request targets a backend/plugin-specific memory capability surface (`thememoria`/Memoria) that is not present in `main`. OpenClaw core already provides the generic plugin-tool and ACP/MCP bridge mechanisms for memory backends to expose write tools themselves, so this belongs in the backend plugin/ClawHub package rather than in core.

## Evidence

- **No in-tree thememoria backend on main:** A repo-wide search for `thememoria` returned no matches, so there is no current core integration for this backend to patch.
  - command: `rg -n "thememoria" .`

- **Vision keeps optional memory behavior plugin-owned:** The project vision says core should stay lean, optional capability should usually ship as plugins, and memory is an explicit plugin slot.
