## Summary

Current main already solves the PR's central problem: plugins no longer need to read OpenClaw JSONL transcript files directly to fetch recent session history. The shipped path is `api.runtime.subagent.getSessionMessages({ sessionKey, limit })`, backed by the Gateway `sessions.get` method, and it is present in `v2026.4.24`. The exact proposed `rt.channel.session.readRecentMessages` shape is not on main, but the supported generic runtime API covers the same storage-contract need for channel plugins.

## Best Possible Solution

Close this stale PR and use the shipped `api.runtime.subagent.getSessionMessages({ sessionKey, limit })` / Gateway `sessions.get` path as the supported plugin session-read API. If maintainers want a channel-namespaced helper or a smaller channel-visible text projection, that should be a focused follow-up against the current contract rather than this branch.

## Risks / Open Questions

- The shipped helper is documented under `api.runtime.subagent`, not `api.runtime.channel.session`, so channel-plugin docs could be clearer for authors using it outside subagent orchestration.
- The shipped helper returns raw Gateway session messages rather than the PR's simplified `{ role, content }` projection; plugins that need channel-visible plain text may still need a focused projection/sanitization helper later.
