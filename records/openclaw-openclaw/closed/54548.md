## Summary

Current `main` already implements inline media rendering in the Control UI/webchat via `MEDIA:`/structured attachment handling, including inline images and inline video players, with regression coverage. The relevant implementation appears to have landed after the latest tagged release, so it is on `main` but not yet in a shipped release tag.

## Evidence

- **Control UI normalizes `MEDIA:` lines into typed attachments:** `expandTextContent` parses `MEDIA:` segments, accepts renderable attachment URLs/paths, infers `image`/`audio`/`video` kinds from extension/MIME, and converts them into attachment items for the renderer.
  - file: [ui/src/ui/chat/message-normalizer.ts:160](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/ui/src/ui/chat/message-normalizer.ts#L160)

  - file: [ui/src/ui/chat/grouped-render.ts:1107](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/ui/src/ui/chat/grouped-render.ts#L1107)

## Risks / Open Questions

- Support is implemented through OpenClaw attachment handling (`MEDIA:` and structured media content), not arbitrary remote markdown/HTML `<img>` or `<video>` embedding.
