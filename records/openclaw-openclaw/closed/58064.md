## Summary

Current main already addresses the reported macOS LaunchAgent restart/channel-start failure. The managed restart path no longer uses the self-killing bootout/bootstrap flow, hands off safely when invoked from the current LaunchAgent, and keeps the startup channel registry pinned so Discord/Telegram channel plugins cannot disappear due to later registry churn.

## Best Possible Solution

Close this issue as implemented. Keep the shipped macOS launchd kickstart/handoff restart path and startup channel-registry pinning; if `openclaw status` still misreports an actually stopped channel, track that as a narrower diagnostics/status issue.

## Evidence
