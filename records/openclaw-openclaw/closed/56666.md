## Summary

Current `main` already implements the requested behavior through a newer channel-registry runtime split. The outbound adapter loader now reads from `getActivePluginChannelRegistry()`, which is explicitly pinned across active-registry replacement, and there is a regression test covering `loadChannelOutboundAdapter("telegram")` after an active registry swap.
