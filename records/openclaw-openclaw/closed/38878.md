## Summary

Current main already supports this through the pluggable memory architecture: memory is an explicit plugin slot, the documented Honcho memory plugin provides service-backed cross-session memory suitable for shared multi-device use, and OpenClaw also supports remote MCP servers over HTTP/SSE/streamable-http for custom remote memory services.

## Risks / Open Questions

- The built-in default memory backend is still local-first; users must choose and configure a service-backed memory plugin such as Honcho or a remote MCP-backed solution.
- The repository does not bundle the Honcho plugin source, so the evidence is architectural plus documented integration rather than an in-tree core implementation.
