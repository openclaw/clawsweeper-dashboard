## Summary

Keep open: current main still gates Feishu plugin tool visibility on top-level Feishu credentials, while the documented and runtime-supported multi-account configuration stores credentials under `channels.feishu.accounts.<id>`. The existing open PR linked from this issue addresses a different packaged-dist guard and does not repair the account-aware manifest availability gap confirmed by the reporter.

## What This Changes

The issue asks OpenClaw to expose Feishu plugin tools such as `feishu_doc` in Feishu DM sessions when the Feishu plugin is enabled and credentials are configured under documented named accounts.

## Best Possible Solution

Make Feishu tool availability account-aware by supporting top-level credentials or any configured named Feishu account, with regression coverage for the manifest gate and no new DM-specific injection path.

## Reproduction Assessment

Yes, from source inspection: documented `channels.feishu.accounts.<id>` credentials do not satisfy the current top-level-only Feishu manifest signals, so the plugin tools can be filtered before they reach the agent tool list. I did not run a live Feishu DM session.

## Solution Assessment

Yes, the narrow maintainable fix is to align Feishu manifest/tool availability with the existing named-account config contract rather than add another session-specific tool injection path.
