## Summary

Keep this PR open. Current main still calls `runCliAgent` directly in the auto-reply CLI path and returns the result after emitting events, without any `persistCliTurnTranscript` call. The direct CLI command path does persist CLI turns, so the PR is not obsolete and targets a real gap between the two paths. The review feedback about thread scoping was also addressed in the PR context by forwarding `originatingThreadId`, which is already part of the queued follow-up run metadata.

## Best Possible Solution

Keep this PR open for maintainer review. The best path is to rebase/update it if needed, preserve the successful-run best-effort `persistCliTurnTranscript` call with `originatingThreadId`, and land it with the targeted regression tests because current main has not implemented the behavior.
