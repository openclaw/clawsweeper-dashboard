## Summary

Keep this issue open. Current main still lacks the reported `openclaw/plugin-sdk/discord` export, the latest published `@openclaw/discord` package still imports that missing subpath, and open PR #73703 directly references this issue as an unmerged implementation candidate. A maintainer also noted that the current PR's four-symbol shim is not sufficient for the published package surface.

## Best Possible Solution

Keep this issue open and treat PR #73703 as the active implementation candidate, but revise or replace it before landing. The maintainer path is either a complete `openclaw/plugin-sdk/discord` compatibility shim with regression coverage and SDK docs/test updates for all imports used by `@openclaw/discord@2026.3.13`, or a new published `@openclaw/discord` release that migrates to supported generic SDK subpaths and is verified through the normal `openclaw plugins install @openclaw/discord` path.
