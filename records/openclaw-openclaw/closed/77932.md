## Summary

Current main already handles the reported bloat by treating `skillsSnapshot.resolvedSkills` as runtime-only, stripping it on session-store read/write, and rehydrating it from disk when needed; regression coverage and the changelog show this shipped in 2026.5.2.

## What This Changes

The issue requests reducing `sessions.json` bloat caused by duplicating the large `skillsSnapshot.resolvedSkills` payload in every session entry.

## Best Possible Solution

Keep the shipped runtime-only `resolvedSkills` design; legacy oversized stores should shrink after upgrading to 2026.5.2 or later and letting the Gateway perform a session-store save.

## Reproduction Assessment

No: current main does not reproduce the `resolvedSkills` duplication path because read/write normalization strips it, and source tests cover direct save, update mutator, legacy load/save, and scale behavior.

## Solution Assessment

Yes: the current implementation is the narrow option-1-style fix, preserving lightweight snapshot metadata while rehydrating the large parsed skill objects only when a runtime consumer needs them.
