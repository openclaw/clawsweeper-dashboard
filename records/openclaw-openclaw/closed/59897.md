## Summary

Close as implemented on current main. The reported Bedrock/Nova `/new` and `/reset` failure was caused by replaying OpenClaw transcript-only assistant records, and current main now removes `delivery-mirror` and `gateway-injected` assistant messages from the in-memory replay before provider policy runs; the same behavior is present in the v2026.4.27 release.

## What This Changes

The PR adds an Amazon Bedrock provider replay-policy override and tests so non-Claude Bedrock model IDs enable `applyAssistantFirstOrderingFix` while Claude model IDs keep the base Anthropic replay policy.

## Best Possible Solution

Keep the current central transcript-replay filter as the shipped fix: transcript-only OpenClaw assistant records should be removed before provider replay, while Bedrock-specific replay-policy changes should be reserved for a separately reproduced non-transcript assistant-first failure.

## Reproduction Assessment

Yes. The reported path is reproducible as a replay history where an OpenClaw `delivery-mirror` assistant record is first; current main has deterministic source and unit-test proof that such records are removed from provider replay before Bedrock sees them.

## Solution Assessment

No, the PR branch is no longer the best merge path. Current main solves the reported Bedrock/Nova startup failure more narrowly by filtering transcript-only artifacts for all affected providers instead of adding a Bedrock-only synthetic bootstrap policy.
