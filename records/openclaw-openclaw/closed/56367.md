## Summary

Keep open. Current `main` still has the reported mismatch: the voice-call plugin starts response generation with `agentId = "main"` but uses an unscoped `sessionKey` like `voice:<digits>`, and sandbox resolution derives the agent only from that session key. When `agents.list` omits `main`, the sandbox path falls back to the repo default agent (first/default listed agent), so another agent's sandbox config and bind mounts can still be applied to a voice run while the workspace path remains `main`'s.
