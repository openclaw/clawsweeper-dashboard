## Summary

Keep #60420 open. Current main still only unbinds stale ACP conversation bindings on missing-metadata/session-init failures; it does not implement this PR's session-store scrub for persisted routing fields (`deliveryContext`, `origin`, `lastChannel`, `lastTo`, `lastAccountId`, `lastThreadId`). The linked bug #60389 is still open in the provided context, and related #60720 was closed unmerged with a different missing-cwd/reset scope.

## Best Possible Solution

Keep the PR open for maintainer review or rebase. The best fix path is to land a narrowly scoped stale ACP cleanup that keeps the existing binding unbind behavior, forwards the active config, uses `updateSessionStoreEntry` under the store lock, and adds regression coverage proving stale ACP session rows lose persisted routing fields after missing-metadata init failures. If maintainers prefer not to take this PR, #60389 should remain the canonical open bug for the missing session-store scrub.
