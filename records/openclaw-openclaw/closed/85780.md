## What This Changes

The PR routes `openclaw node status` recovery hint lines for missing-unit and stopped-service states from stderr to stdout and adds focused tests plus a changelog entry.

## Best Possible Solution

Land this focused CLI fix if maintainers accept stdout as the consistent home for follow-up recovery hints while stderr remains reserved for the primary failure summary.

## Reproduction Assessment

Yes from source inspection: current `main` logs the unloaded-service hints through `defaultRuntime.log`, but the missing-unit and stopped-runtime hint loops call `defaultRuntime.error` in `src/cli/node-cli/daemon.ts`. The linked report also gives a concrete shell comparison using `openclaw node status 2>/dev/null`.

## Solution Assessment

Yes, if maintainers accept the CLI stream contract: the patch changes only the two affected hint loops, keeps terse failure summaries on stderr, and adds branch-specific tests. The safer alternative would be to pause for an explicit broader CLI stdout/stderr policy if maintainers do not want this command-specific contract change.
