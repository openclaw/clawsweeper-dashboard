## Summary

Keep open. The parent-death guard is a useful, focused fix for the linked launcher respawn bug, but the PR currently introduces two platform-sensitive test failures by expecting the Unix-only guard env on Windows.

## What This Changes

The PR adds parent-PID lifecycle guards to launcher, compile-cache, and CLI respawn children, plus regression tests and a changelog entry.

## Best Possible Solution

Make the two respawn env assertions platform-aware, then land this parent-death guard or an equivalent focused implementation with the included regression coverage.

## Reproduction Assessment

Yes, at source level: the PR helper intentionally skips guard-env injection on `win32`, while two changed tests now expect that env var without a platform branch. The original systemd scenario was not live-reproduced in this read-only pass.

## Solution Assessment

Yes for the runtime direction, with one test fix needed. The parent-PID guard is narrower than broadening systemd cleanup policy, but the tests should match the Unix-only contract before merge.
