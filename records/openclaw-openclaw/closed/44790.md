## Summary

Keep open. Current `main` does not show a landed OpenClaw-side fix for ACP/acpx process-tree cleanup, and the current docs still describe ACP sessions that stall after completion and require manual process cleanup.

## Evidence

- **Current docs still acknowledge stale ACP processes:** The ACP troubleshooting table on current `main` says completed ACP work can stall indefinitely because the harness process finished without reporting completion, and tells users to monitor `ps` and kill stale processes manually.
  - file: [docs/tools/acp-agents.md:548](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/docs/tools/acp-agents.md#L548)
  - command: `nl -ba docs/tools/acp-agents.md | sed -n '536,552p'`
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **acpx wrapper close path has no explicit process-tree teardown:** `AcpxRuntime.close()` delegates to the backend `close()` call and, on reset, only marks the session fresh in the session store. There is no OpenClaw-side SIGTERM/SIGKILL or process-group cleanup in this wrapper.
  - file: [extensions/acpx/src/runtime.ts:314](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/extensions/acpx/src/runtime.ts#L314)
  - command: `nl -ba extensions/acpx/src/runtime.ts | sed -n '314,324p'`
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Gateway ACP cleanup only waits for cancel/close and times out:** Session reset/delete cleanup races ACP cancel/close against a 15s timeout and returns an unavailable error if cleanup does not finish. This path does not add a force-kill fallback for lingering ACP child processes.
  - file: [src/gateway/session-reset-service.ts:247](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/src/gateway/session-reset-service.ts#L247)
  - command: `nl -ba src/gateway/session-reset-service.ts | sed -n '247,325p'`
  - sha: [38caa6832d4e](https://github.com/openclaw/openclaw/commit/38caa6832d4e1d344736d635b0c5fdad37e4430b)
- **Current bundled acpx dependency is visible but not proven to fix this bug:** The bundled plugin currently pins `acpx` to `0.5.3`, but the repo tree here does not contain evidence of a landed fix for orphaned Claude/ACP child processes, and the docs above still describe the failure mode as current.
  - file: [extensions/acpx/package.json:7](https://github.com/openclaw/openclaw/blob/38caa6832d4e1d344736d635b0c5fdad37e4430b/extensions/acpx/package.json#L7)
