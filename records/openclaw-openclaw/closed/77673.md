## Summary

Current main has the missing `plugin-config-runtime` SDK subpath wired into the source export map, build entry list, and post-build export check, so the npm memory-lancedb import path reported here is resolved on main. This is not proven shipped in `v2026.5.3`, so the bundled-plugin workaround remains the right release-side workaround until a release containing current main is available.

## What This Changes

The issue reports that `@openclaw/memory-lancedb@2026.5.3` imports `openclaw/plugin-sdk/plugin-config-runtime`, which was missing from the OpenClaw 2026.5.3-1 published `dist/plugin-sdk` tree and prevented the plugin from loading.

## Best Possible Solution

Keep the current main SDK subpath/export wiring and ship it in the next release; users pinned to `v2026.5.3` should keep using the bundled memory plugin workaround or upgrade once the fixed release is available.

## Reproduction Assessment

Yes. The reported release failure is source-reproducible from the exact `memory-lancedb` import and the missing published subpath condition described in the stack trace; current main removes that condition by exporting and building the subpath.

## Solution Assessment

Yes. Preserving `openclaw/plugin-sdk/plugin-config-runtime` as a real SDK subpath is the narrowest backward-compatible fix because it satisfies the existing external plugin import without changing memory-lancedb runtime behavior.
