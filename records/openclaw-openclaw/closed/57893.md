## Summary

Current `main` already routes Feishu text slash commands through account-scoped Feishu ingress and reply paths. The reported `accounts.<accountId>` multi-account failure mode does not match the present code: websocket events carry the receiving `accountId`, Feishu routing applies account-scoped bindings before dispatch, command context keeps the Feishu account/session metadata, and outbound replies resolve the same Feishu account.

## Evidence

- **WebSocket ingress keeps the receiving Feishu account:** The Feishu monitor passes its active `accountId` into `handleFeishuMessage(...)` for each received event, so multi-account websocket deliveries do not lose account context before command handling.
  - file: [extensions/feishu/src/monitor.message-handler.ts:183](https://github.com/openclaw/openclaw/blob/7e49cc87f9e9cdad47d0572331d7a7a47281a83e/extensions/feishu/src/monitor.message-handler.ts#L183)
