## Summary

Current main and shipped releases already cover the central workflow: agents can send a local file to Feishu through the generic message media path, and the Feishu plugin uploads it through the IM file API before sending the chat message.

## What This Changes

Request to add a Feishu IM file-upload tool that accepts a local path, returns an IM `file_key`, and enables sending local files with Feishu chat messages.

## Best Possible Solution

Keep using the shipped generic `message(action="send")` media path for Feishu file delivery, and avoid adding a duplicate Feishu-specific file-key tool unless a separate plugin API gap is proven.

## Reproduction Assessment

Not applicable: this is a new tool-surface request, not a broken existing contract. Source inspection confirms the observable send-local-file workflow is already available through the generic message media path.

## Solution Assessment

Yes: the shipped generic message media path is the better maintained solution because it keeps channel delivery behind the shared message abstraction while Feishu still performs the IM `file_key` upload internally.
