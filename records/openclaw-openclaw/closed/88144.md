## Best Possible Solution

Preserve the durable no-send contract, add a narrowly gated recovery only for confirmed missing final DM bubbles, and prove it with focused tests plus live Telegram DM evidence.

## Reproduction Assessment

No high-confidence live reproduction is present. Source inspection shows the proposed scenario: a DM final whose durable delivery returns `handled_no_send` leaves current main with `finalAnswerDeliveryStarted` true, `finalAnswerDelivered` false, and no visible DM fallback.

## Solution Assessment

No. Recovering transcript-backed final text is reasonable, but the PR needs to distinguish transient missing-visible-send from intentional durable suppression before replaying a final DM.
