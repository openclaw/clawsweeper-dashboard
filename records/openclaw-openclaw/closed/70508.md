## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Serialize boundary flushes with partial updates:** `extensions/mattermost/src/mattermost/monitor.ts:1702-1706`
  - body: `onDraftBoundary` awaits `draftStream.flush()` before resetting the draft, but assistant/reasoning boundaries and partial replies are fired without being serialized by the embedded subscriber. A new partial can arrive during that await and then be discarded or attached to the wrong preview post when `forceNewMessage()` runs; queue boundary and partial handling together or make the rotation synchronous.
  - confidence: 0.88
- **[P2] Preserve the shared tool-progress formatter:** `extensions/mattermost/src/mattermost/draft-stream.ts:59-71`
  - body: This replaces the current shared Mattermost progress formatter with a local icon/title mapper, which would drop the current `args`, `detailMode`, and `config` behavior used for raw command/detail and preview config handling after rebase. Keep the shared formatter contract and layer any Mattermost-specific display changes through that surface.
  - confidence: 0.86
