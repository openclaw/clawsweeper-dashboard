## Summary

Current main no longer leaves Telegram inbound images on a describer-only path. Telegram still records image files in `MediaPaths`, but the reply prompt now prepends `[media attached: ...]` refs from those paths, and both embedded and CLI-backed vision runs hydrate those refs into native image inputs before the model call. The same implementation is already present in release `v2026.4.23`.

## Evidence

- **Telegram still preserves inbound image file paths:** `resolveTelegramInboundBody()` keeps inbound media in `MediaPaths` and only uses `<media:image>` as fallback body text when there is no user text. That means the real file refs are still available downstream.
