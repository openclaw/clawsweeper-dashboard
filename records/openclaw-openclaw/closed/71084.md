## Summary

Close #71084 as superseded, not because the original gate-removal patch landed. Current main still keeps the CLI gates for embedded memoryFlush/preflightCompaction and documents memory flush as embedded-only, but merged PR #71916 added a CLI-specific compaction lifecycle that addresses the reported long-lived CLI context growth and stale resume behavior without the stale CLI binding problem flagged on #71081.

## Best Possible Solution

Keep the shipped CLI-specific compaction/reseed lifecycle from #71916 and do not apply the broad #71081 gate-removal patch. If operators still need a distinct CLI durable memoryFlush turn, track that as a focused design issue that accounts for CLI resume bindings and provider-owned session state.
