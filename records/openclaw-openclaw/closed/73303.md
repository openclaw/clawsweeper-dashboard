## Summary

Current main has shipped the maintainer-approved restart drain controls that the earlier review said were missing: operators can now force an immediate managed restart, set a per-restart wait budget, use the safe coordinated restart path, and see active task blockers before the drain waits.

## What This Changes

The issue asks for investigation and a patch or workaround for macOS LaunchAgent Gateway restarts that can sit in the shutdown drain for several minutes before the replacement process starts.

## Best Possible Solution

Keep the shipped restart controls and docs; if plain restart should change default semantics or Weixin/model startup remains slow, track that as a narrower follow-up with fresh logs on a current release.

## Reproduction Assessment

Yes for the historical mechanism: source and tests show managed restart can enter the active-work drain and wait up to the configured budget. I did not live-reproduce the exact macOS plus plugin timing from the report on current main.

## Solution Assessment

Yes. The shipped solution preserves compatibility for plain service-manager restarts while adding explicit `--force`, `--wait`, `--safe`, and `--safe --skip-deferral` paths plus blocker logging, which is the maintainable policy split the earlier review requested.
