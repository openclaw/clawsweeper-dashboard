## Summary

Current `main` already exposes supported seams for tool-call observation and tool-result persistence, so this RFC's requested host patch point is obsolete on `main`.

## Evidence

## Risks / Open Questions

- The exact RFC hook names (`before_tool_call_persist` / `after_tool_call_persist`) do not exist; the behavior is split across existing seams, so discoverability depends on docs.
- `api.registerAgentToolResultMiddleware(...)` is a bundled-plugin seam, not a general external-plugin API; external plugins rely on the documented hook surface instead.
