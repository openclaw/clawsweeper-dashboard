## Summary

Current `main` already implements the requested behavior: the shared `message` tool send schema does not require a `card`, Feishu accepts media-only sends, and the same behavior is present in the shipped `v2026.4.22` release.

## Evidence

- **Shared message-tool send schema has optional media/path/filePath/presentation fields and no required `card`:** `buildSendSchema()` defines `message`, `media`, `path`, `filePath`, and optional `presentation`; there is no `card` property in the shared send schema.
  - file: [src/agents/tools/message-tool.ts:90](https://github.com/openclaw/openclaw/blob/8f64cd3e4d8339a6aab3ca230d5e14944c91cfaf/src/agents/tools/message-tool.ts#L90)
