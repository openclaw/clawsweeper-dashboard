## Summary

The session-end hook half is already covered on current main, and the remaining periodic/session-end memory-save policy belongs as a ClawHub/plugin capability rather than another OpenClaw core scheduler/config knob. Related #41518 closed as implemented for lifecycle hooks, and related #11619 already closed the generic periodic memory autosave request as ClawHub scope.

## Best Possible Solution

Do not add another OpenClaw core memory scheduler/config knob for this issue. Keep the shipped pre-compaction flush behavior, use the existing `session_end` hook surface for session-bound cleanup, and move periodic/thread-specific memory flushing to a ClawHub plugin or memory-plugin design that can use plugin lifecycle hooks plus gateway cron. If a future plugin discovers a missing API, open a focused plugin-SDK issue for that seam.
