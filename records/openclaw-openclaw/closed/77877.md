- **[P2] Strip managed document links from fallback events:** `src/gateway/server-methods/chat.ts:460-468`
  - body: The new document block creation can add `/api/chat/media/outgoing-doc/...` attachment blocks, but the append-failure fallback still strips only managed image blocks. Since the new handler returns 404 unless the record is referenced by the transcript, a failed transcript append can broadcast a dead document link; strip managed document attachment blocks in that fallback path and add regression coverage.
  - confidence: 0.9
- **[P3] Gate the document media route before loading it:** `src/gateway/server-http.ts:750-753`
  - body: This registers `chat-managed-document-media` for every unhandled HTTP request. Unrelated Control UI/API paths will import and run the managed document module before falling through, while the existing image path is guarded by a route predicate; add an outgoing-doc predicate and only register this stage for matching paths.
  - confidence: 0.83
