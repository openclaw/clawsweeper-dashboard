## Summary

Current main already exposes Anthropic remaining-quota data through OpenClaw's normalized provider-usage surfaces: `session_status`/`/status`, the gateway `usage.status` API, CLI usage output, and UI usage hints. It does not pass through raw `anthropic-ratelimit-unified-*` headers verbatim, but the underlying user-facing capability requested in the issue is present.
