## Summary

Keep open. Current main still has the reported config-contract mismatch: the strict root browser schema and generated JSON schema omit browser.tabCleanup, while runtime resolution, public types, labels/help, and docs expose the setting. The issue is also paired with open implementation PRs, so it should remain open until maintainers choose and land one fix.

## What This Changes

Operators request that the config validator accept the documented browser.tabCleanup object so browser tab cleanup defaults can be tuned through openclaw.json.

## Best Possible Solution

Land one focused schema fix that adds a strict optional browser.tabCleanup object to the root config schema, regenerates the checked-in generated schema/hash outputs, and adds validation regressions for accepted documented keys plus rejected invalid nested keys; close this issue after the chosen PR merges.
