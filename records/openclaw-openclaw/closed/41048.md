## Summary

Keep open: current main still has the Telegram null-media warning gap, but this PR is not ready to merge because it drops current fallback metadata, misses all-failed media groups, adds an unthrottled warning path, and lacks real Telegram proof.

## What This Changes

The PR changes Telegram handling to send a warning when media resolution returns null and to omit generated media placeholders when no attachment resolves.

## Best Possible Solution

Adapt or replace the fix on current `extensions/telegram` code so null `getFile()` failures produce an accurate, rate-limited warning while preserving fallback media/file_id context and covering media groups with focused tests plus real Telegram proof.

## Reproduction Assessment

Yes, from source inspection: `getFile()` failures can return null, `resolveMedia()` propagates null, and the handler only warns for thrown media errors. Current tests also lock the empty-media fallback metadata behavior that this PR would remove.

## Solution Assessment

No. The warning direction is useful, but this exact patch drops fallback file_id context, misses all-failed media groups, lacks reply throttling, and has no real Telegram proof.

  - body: Current main intentionally prepends `<media:video> [file_id:...]` for captioned failed downloads. Making `placeholder` empty when `allMedia` is empty drops that metadata and leaves the agent with only caption text.
  - confidence: 0.9

  - confidence: 0.86
- **[P2] Throttle repeated media-failure replies:** `src/telegram/bot-handlers.ts:1031-1036`
  - body: This branch sends a Telegram warning for every null media resolution in an allowed chat. Add per-chat, per-sender, or media-group cooldown/dedupe so repeated failing media cannot spam chats or send capacity.
  - confidence: 0.82
- **[P3] Use reason-neutral warning text:** `src/telegram/bot-handlers.ts:1031`
  - body: `resolveTelegramFileWithRetry()` returns null for both Bot API size-limit failures and exhausted transient retries, but this message mostly points at file size. Make it generic unless the null reason is carried through explicitly.
  - confidence: 0.78
