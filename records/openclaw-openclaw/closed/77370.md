## Summary

Current main already implements and documents the underlying CLI-backend thinking behavior, so this PR is obsolete and would now make the docs inaccurate by saying `claude-cli` thinking is inert.

## What This Changes

The PR adds docs and changelog text stating that `/think` and `/reasoning` do not affect CLI backends because OpenClaw uses fixed argv.

## Best Possible Solution

Keep the current-main SDK seam and `claude-cli` mapper, and only add follow-up docs if they narrowly clarify CLI backends that still lack a mapper.

## Reproduction Assessment

Yes. Source inspection on current main shows non-off `thinkLevel` is passed through the CLI backend argv rewrite seam to Claude Code `--effort`, while the PR diff says the opposite for CLI backends.

## Solution Assessment

No. This PR is no longer the best way to solve the issue because current main implements the `claude-cli` behavior and documents the narrower contract; merging this wording would regress the docs.
