## Summary

Keep open. Current main has scoped external CLI discovery and Codex bootstrap protections, but it still has no global `auth.externalCliSync.enabled` or `OPENCLAW_AUTH_EXTERNAL_CLI_SYNC` equivalent; scoped Codex, Claude CLI, and MiniMax reads still run when those providers or profiles are in scope.

## What This Changes

Request a global config and environment opt-out that disables external CLI credential synchronization into auth profiles while leaving the default behavior enabled.

## Best Possible Solution

Add a documented central auth-level switch that disables only built-in external CLI credential sync before any external CLI reader is invoked, while preserving provider-managed plugin external auth overlays and keeping the default enabled for backward compatibility.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: call the current overlay path with Codex or Claude CLI in provider scope and it reaches the external CLI credential reader, and there is no config/env flag to stop that path globally.

## Solution Assessment

Yes. A central `auth.externalCliSync.enabled` plus env override is the narrowest maintainable direction because the behavior is shared auth-store hydration, not a provider-specific fallback.
