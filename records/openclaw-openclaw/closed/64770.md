## Summary

Facebook Messenger channel support fits OpenClaw's documented channel-plugin path and ClawHub distribution model, not a core cleanup item. Current docs and architecture explicitly route new messaging platforms through `api.registerChannel(...)`, the channel plugin SDK, and external/community plugin publishing; WeChat is already documented as an external channel plugin rather than core code.

## Risks / Open Questions

- There is no in-tree Facebook Messenger plugin today, so closing as `clawhub` assumes the request is best handled as external plugin work rather than bundled core work.
- If Messenger needs a missing SDK seam beyond the current channel-plugin contract, a narrower issue for that specific plugin API gap would still be valid.
