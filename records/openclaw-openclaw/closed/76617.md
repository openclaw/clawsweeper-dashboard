## Summary

Keep open: the reported Zalouser long-message problem is plausible, but this PR only changes a helper fallback that normal runtime delivery does not use, so the observable bot-reply path remains on the shared `length` default.

## What This Changes

The branch changes the Zalouser send helper default chunk mode to `newline` and adds a helper-level regression test for paragraph-boundary splitting.

## Best Possible Solution

Wire natural-boundary chunking through the Zalouser production reply and outbound adapter defaults while preserving explicit config overrides, then cover those runtime paths and provide redacted real delivery proof.

## Reproduction Assessment

Yes, by source inspection. With normal config, current main resolves Zalouser chunk mode to the shared `length` default and passes it explicitly into runtime sends, so changing only the helper fallback does not affect ordinary long bot replies.

## Solution Assessment

No. The PR changes a fallback used only when callers omit `textChunkMode`; the maintainable fix needs to update the Zalouser runtime delivery default/resolver path and preserve explicit overrides.
