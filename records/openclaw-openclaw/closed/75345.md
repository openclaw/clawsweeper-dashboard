## Summary

Keep open. Current main still lacks the requested stale-adapter guard, and the PR head still has a concrete blocker: explicit npm partial/range pins below Codex ACP 0.12.0 can bypass the new check and reach `session/new`.

## What This Changes

The PR adds an ACPX runtime guard, focused tests, docs, and a changelog entry for `openai-codex/gpt-5.5` with explicit `@zed-industries/codex-acp` adapter versions older than 0.12.0.

## Best Possible Solution

Keep the guard at the ACPX wrapper boundary, but make explicit npm version/range specs conservative so any spec that can resolve below Codex ACP 0.12.0 fails early for `gpt-5.5` while wrapper, local binary, unpinned, and 0.12.0+ paths continue to pass.

## Reproduction Assessment

Yes. Configure the Codex ACP command as a partial old npm spec such as `npx @zed-industries/codex-acp@0.11` or `@0.11.x`, then start a Codex ACP session with `openai-codex/gpt-5.5`; the PR parser cannot extract a full patch version and lets the old adapter receive the model.

## Solution Assessment

No as written. The runtime-boundary guard is the right shape, but it needs npm-range-aware parsing or conservative rejection for explicit specs that may resolve below 0.12.0.
