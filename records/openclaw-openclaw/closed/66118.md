## Summary

Current `main` already provides a config-backed way to suppress update notifications: `update.checkOnStart: false`. That setting disables the startup update check path that feeds gateway logs and the Control UI `update available` banner, and it is covered by tests and docs. The implementation appears to have landed after this issue was opened, and it does not appear to be in the latest tagged release `v2026.4.23` yet.

## Evidence

- **Config surface exists on main:** `OpenClawConfig` now includes `update.checkOnStart?: boolean` with the documented meaning "Check for updates on gateway start (npm installs only)."
