## Summary

Keep this PR open, but it is not merge-ready: the feature is still missing on current main, while the proposed implementation bypasses bounded history reads, duplicates archive sequence metadata, lacks public contract docs, and needs explicit approval for model-facing reset-archive visibility.

## What This Changes

The PR adds an optional `includeArchived` parameter to `chat.history` and `sessions_history`, reset-archive discovery/aggregation helpers, tests, and regenerated Swift protocol bindings.

## Best Possible Solution

Land a coordinated archive-history mode that keeps bounded async reads, stable segment metadata, public docs, generated clients, regression tests, and an approved reset-archive visibility contract.

## Reproduction Assessment

Yes for the review findings: static source inspection shows current main rejects `includeArchived` and uses bounded async history reads, while the PR branch routes the opt-in archive path through full sync readers and per-archive sequence counters.

## Solution Assessment

No, not as proposed. A read-only opt-in is plausible, but it should preserve the bounded read contract, keep monotonic segment metadata, document the public contract, and get approval for reset-archive visibility.
