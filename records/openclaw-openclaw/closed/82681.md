## What This Changes

The issue requests preventing Telegram/channel sessions from wedging indefinitely when a Codex app-server turn-scoped request keeps `activeAppServerTurnRequests` nonzero.

## Best Possible Solution

Keep the current main watchdog/recovery implementation; any desire for a shorter configurable terminal cap can be tracked separately after the unreleased fix ships.

Yes for the historical failure mode: the reported v2026.5.12 path is source-plausible from the request-counter guards and logs, but current main adds an independent progress idle abort and diagnostic recovery backstop. I did not establish a failing current-main live Telegram reproduction.
